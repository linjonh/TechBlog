    <h2>
     <a id="_10">
     </a>
     一、操作规则
    </h2>
    <h3>
     <a id="1next_11">
     </a>
     1、next()
    </h3>
    <p>
     <strong>
      返回空格/回车结束标识符前的字符串，但不处理标识符
     </strong>
     。如果以回车键作为结束标志，后面不能跟nextLine()

    <h3>
     <a id="2nextInt_35">
     </a>
     2、nextInt()
    </h3>
    <p>
     <strong>
      返回数字，但结束标志回车/空格还留在缓冲区
     </strong>
     ，以回车键作为结束标志，后面不能跟nextLine()

    <h3>
     <a id="3nextLine_46">
     </a>
     3、nextLine()
    </h3>
    <p>
     <strong>
      会把结束标识符前的内容连同标识符一同截取，但返回的时候不带标识符
     </strong>
     ，相当于丢弃了
    </p>

    <h3>
     <a id="4_59">
     </a>
     4、使用
    </h3>
    <p>
     <mark>
      next()
     </mark>
     开始读取时：不读取字符前后的空格/tab键，只读取字符，读取过程中直到遇到空格/tab键回车截至。
     <br/>
     <mark>
      nextInt()
     </mark>
     ：会读取数字，而将结束标志回车留在缓冲区，所以nextInt后面不跟nextLine()
     <br/>
     <mark>
      nextLine()
     </mark>
     ：读取字符前后的空格/tab键，直到遇到回车键结束。
     <strong>
      如果一开始就遇到回车键，那就还没开始就直接结束了。
     </strong>
    </p>
    <h2>
     <a id="_64">
     </a>
     总结
    </h2>
    <ol>
     <li>
      <strong>
       nextInt()和next()都不会处理结束标识符
      </strong>
      。
     </li>
     <li>
      但next()可以忽略开头的标识符，从字符开始读入，而nextInt()最开始就必须是数字，否则抛出异常。
     </li>
     <li>
      nextLine()会把标识符和字符串一同截取，但会把截取中最后的标识符丢弃。
     </li>
     <li>
      <strong>
       nextInt()后不跟nextLine()
      </strong>
     </li>
    </ol>
