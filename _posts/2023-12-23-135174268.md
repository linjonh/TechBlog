    <blockquote>
     <p>
      数据流图(DFD)是一种图形化技术,它描绘信息流和数据从输人移动到输出的过程中所经受的变换。
     </p>
    </blockquote>
    <p>
     首先给出一个数据流图样例
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e32f1d6bd895310a065b9b88f60ea948.png"/>
    </p>
    <h2>
     <a id="_6">
     </a>
     基本的四种图形
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f9745b8e6753d531efe218bcf50ff3b7.png"/>
    </p>
    <ul>
     <li>
      直角矩形:代表源点或终点,一般来说,是
      <font color="FF0000">
       人
      </font>
      ,如例图的仓库管理员和采购员
     </li>
     <li>
      圆形(也可以画成圆角矩形):是处理,一般来说,是
      <font color="FF0000">
       动作
      </font>
      ,是
      <font color="FF0000">
       动词+名词的形式
      </font>
      ,处理可以一步一步,再细化,也就是从最顶层,到第1层,到第2层.
     </li>
     <li>
      数据流:是数据的传递,数据的流向,一般来说,是
      <font color="FF0000">
       名词
      </font>
      ,是
      <font color="FF0000">
       我们向后传递的那个信息名称
      </font>
      .
     </li>
     <li>
      数据存储,可以是一个清单,一个文件,是我们要从中取数据或者存数据的地方.
     </li>
    </ul>
    <p>
     <font color="3366FF">
      注意
     </font>
     :
     <br/>
     当源点和终点相同时,我们要在直角矩形的右下角画一横线.
     <br/>
     不断的抽象,但是两头的数据流是不能变的
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9243913c67a06bd14354a668921776cb.png"/>
    </p>
    <blockquote>
     <p>
      现在可以开画了,注意的是,其实数据流图中,很多描述,同时,我们自己编的,让自己的图更有逻辑,所以,参考答案并不唯一
     </p>
    </blockquote>
    <h2>
     <a id="_19">
     </a>
     题目一
    </h2>
    <blockquote>
     <p>
      银行计算机储蓄系统的工作过程大致如下:储户填写的存款单或取款单由业务员键人系统,如果是存款则系统记录存款人姓名、住址(或电话号码),身份证号码,存款类型、存款日期、到期日期、利率及密码(可选)等信息,并打印出存款存单给储户;如果是取款而且存款时留有密码,则系统首先核对储户密码,若密码正确或存款时未留密码,则系统计算利息并打印出利息清单给储户。
      <br/>
      请用数据流图描绘本系统的功能.
     </p>
    </blockquote>
    <p>
     首先,画出最顶层数据流图,最顶层数据流图也就是找到源点终点,中间最抽象的一个处理,配上他们之间的数据流即可.
    </p>
    <p>
     本题中值得注意的是,业务员不是起点.
    </p>
    <p>
     分析如下:
     <br/>
     关键信息是储户存款或取款,若存款,打印存款单给储户,若取款,打印利息清单给储户,中间的抽象就是银行储蓄系统.
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3b0bd64f0780a8357190770a57b31b96.png"/>
    </p>
    <p>
     其次,将中间的抽象,细分,分为两步,存款和取款.(熟练后这步不需要写)
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c8ac89f2e5a019943556e9e6797d5400.png"/>
    </p>
    <p>
     最后,我们再把存款和取款的流程细分.
     <br/>
     根据题目信息,存款要先记录信息,打印存款单这两步取款要先核验密码,计算利息,打印利息清单这三步,不要忘记数据存储记录存款信息,取款时读出存款信息
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4f2b2f8c02b10b0ff43390b815132990.png"/>
    </p>
    <h2>
     <a id="_40">
     </a>
     题目二
    </h2>
    <blockquote>
     <p>
      高考录取统分子系统有如下功能
      <br/>
      ⑴计算标准分:根据考生原始分计算得到标准分，存入考生分数文件
      <br/>
      ⑵计算录取分数线︰根据标准分、招生计划文件中的招生人数计算录取线，存入录取线文件。
      <br/>
      根据要求画出该系统的数据流图。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f227746da639642b09df4129f8fe4a70.png">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a7aac58a17debe08790f9da1eb42a610.png">
        <br/>
        该图有一点不规范：
        <br/>
        因为外部实体不能与数据存储相连.
        <br/>
        修改思路如下:
        <br/>
        考生（外部实体）：
        <br/>
        先通过一个处理过程（如“提交成绩”或“报名申请”）再进入“评审标准”过程。
        <br/>
        招生简章（外部实体）：
        <br/>
        作为外部信息输入，流向“评审标准”或“录取标准”处理过程。
        <br/>
        评审标准（处理过程）：
        <br/>
        从外部实体（考生、招生简章）接收信息，处理后产生输出（如“成绩”）。
        <br/>
        录取标准（处理过程）：
        <br/>
        处理来自“评审标准”的结果，以及外部输入的信息，做出录取决策。
        <br/>
        数据存储：
        <br/>
        处理过程的结果会存入数据存储（如“考生数据存储”、“录取结果存储”）。
       </img>
      </img>
     </p>
    </blockquote>
