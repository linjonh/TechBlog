     Qt开发 - 使用 ODBC 连接数据库遇到的坑

    <h3>
     <a id="_3">
     </a>
     遇到的问题
    </h3>
    <p>
     学习了 Qt 中，正好学到了数据库的连接，看了点内容挥挥手去编代码，使用 ODBC ，结果连不上数据库。怀疑是驱动的问题，废了老大劲去编译 MySQL 数据库驱动，结果跟这玩意没有关系。归根到底还是 C++ 类的内核本质。

查找问题

驱动？（否）

<h4>
<a id="_104">
</a>
解决方法
</h4>
<p>
将数据库对象指针全换成普通的对象标识符。
</p>
<pre><code class="prism language-C++">#if 1
QSqlDatabase database = QSqlDatabase::addDatabase("QODBC");
database.setHostName("127.0.0.1");
database.setPort(3306);
database.setDatabaseName("MySQL");
database.setUserName("root");
database.setPassword("@mysql");

    if(database.open()){
    	QMessageBox::information(this,"恭喜","数据库连接成功");
    }else{
    	QMessageBox::warning(this,"提示","数据库连接失败");
    }
    qDebug()&lt;&lt;database.databaseName()&lt;&lt;endl;

#endif
</code></pre>
<p>
运行结果：
</p>
<p>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8a4b454469d1c29ce7b5167fc0fcc384.png#pic_center"/>
</p>
<p>
这里的
<code>
database.databaseName()
</code>
的内容是我在 ODBC 中设置的 MySQL 的，数据库连接成功了。
</p>
<h4>
<a id="_133">
</a>
为什么？
</h4>
<h5>
<a id="addDatabase_135">
</a>
<code>
addDatabase()
</code>
是静态方法
</h5>
<table>
<thead>
<tr>
<th>
[static] QSqlDatabase QSqlDatabase::addDatabase(QSqlDriver \*driver, const QString &amp;connectionName = QLatin1String(defaultConnection))
</th>
</tr>
</thead>
</table>
<p>
<code>
addDatabase()
</code>
是一个静态函数，它不属于任何一个
<code>
QSqlDatabase 对象
</code>
，而是属于
<code>
QSqlDatabase 类
</code>
。静态函数只能通过类名来调用，不能通过对象或指针来调用。这里应该把数据库对象定义为一个普通对象而不是对象指针，因为指针操作很复杂。
</p>
<pre><code class="prism language-c++">QSqlDatabase database = QSqlDatabase::addDatabase("QODBC"); // 定义为普通对象
</code></pre>
<pre><code class="prism language-c++">QSqlDatabase database = QSqlDatabase(); // 定义为指针
database = &amp;QSqlDatabase::addDatabase("QODBC");
</code></pre>
<p>
这里
<code>
addDatabase("QODBC")
</code>
产生的地址是一个临时变量，到头来
<code>
database
</code>
指向的是空指针，所以没有调用成功 ODBC 的驱动。
</p>
