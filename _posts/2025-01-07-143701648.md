    <h3 id="1">
     插件介绍
    </h3>
    <p>
     腾讯位置服务地图选点插件 可以让用户快速、准确地选择并确认自己的当前位置，并将相关位置信息回传给开发者。同时我们还提供搜索、关键词分类、POI主子点等辅助功能，让选点更轻松！
    </p>
    <p>
     请扫描二维码，在手机端体验地图选点插件Demo。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="430" src="https://img-blog.csdnimg.cn/img_convert/9e90cb90d29c9e3cfc5e4fb07f822ade.jpeg" width="430"/>
    </p>
    <p>
    </p>
    <h3 id="2">
     接入指引
    </h3>
    <p>
     1、插件申请接入：
    </p>
    <p>
     在腾讯微信公众平台中， “微信小程序官方后台-设置-第三方服务-插件管理” 里点击 “添加插件”，搜索 “腾讯位置服务地图选点” 申请，申请后小程序开发者可在小程序内使用该插件。
    </p>
    <p>
    </p>
    <p>
     2、引入插件包： 地图选点appId： wx76a9a06e5b4e693e （
     <a href="https://mp.weixin.qq.com/wxopen/plugindevdoc?appid=wx76a9a06e5b4e693e" rel="nofollow" title="点击浏览插件最新版本">
      点击浏览插件最新版本
     </a>
     ）
    </p>
    <pre><code>// app.json
{
    "plugins": {
        "chooseLocation": {
        "version": "1.0.12",
        "provider": "wx76a9a06e5b4e693e"
        }
    }
}  
</code></pre>
    <p>
     3、设置定位授权：
    </p>
    <p>
     地图选点插件需要小程序提供定位授权才能够正常使用定位功能：
    </p>
    <pre><code>// app.json
{
    "permission": {
        "scope.userLocation": {
        "desc": "你的位置信息将用于小程序定位"
        }
    }
}
</code></pre>
    <p>
     4、使用插件：
    </p>
    <p>
     插件页面调用示例：
    </p>
    <pre><code>const key = ''; //使用在腾讯位置服务申请的key
const referer = ''; //调用插件的app的名称
const location = JSON.stringify({
  latitude: 39.89631551,
  longitude: 116.323459711
});
const category = '生活服务,娱乐休闲';
 
wx.navigateTo({
  url: 'plugin://chooseLocation/index?key=' + key + '&amp;referer=' + referer + '&amp;location=' + location + '&amp;category=' + category
});
</code></pre>

    </p>
    <pre><code>const chooseLocation = requirePlugin('chooseLocation');
Page({
    // 从地图选点插件返回后，在页面的onShow生命周期函数中能够调用插件接口，取得选点结果对象
    onShow () {
        const location = chooseLocation.getLocation(); // 如果点击确认选点按钮，则返回选点结果对象，否则返回null
    },
    onUnload () {
        // 页面卸载时设置插件选点数据为null，防止再次进入页面，geLocation返回的是上次选点结果
        chooseLocation.setLocation(null);
    }

    <h3 id="%E6%8F%92%E4%BB%B6%E7%9A%84%E4%BD%9C%E7%94%A8">
     插件的作用
    </h3>
    <h4 id="添加插件" style="background-color:transparent;">
     添加插件
    </h4>
    <p>
     在使用插件前，首先要在小程序管理后台的“设置-第三方服务-插件管理”中添加插件。开发者可登录小程序管理后台，通过 appid 查找插件并添加。如果插件无需申请，添加后可直接使用；否则需要申请并等待插件开发者通过后，方可在小程序中使用相应的插件。
    </p>
    <h4 id="引入插件代码包" style="background-color:transparent;">
     引入插件代码包
    </h4>
    <p>
     使用插件前，使用者要在
     <code>
      app.json
     </code>
     中声明需要使用的插件，例如：
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>{
  "plugins": {
    "myPlugin": {
      "version": "1.0.0",
      "provider": "wxidxxxxxxxxxxxxxxxx"
    }
  }
}
</code></pre>
    <p>
     如上例所示，
     <code>
      plugins
     </code>
     定义段中可以包含多个插件声明，每个插件声明以一个使用者自定义的插件引用名作为标识，并指明插件的 appid 和需要使用的版本号。其中，引用名（如上例中的
     <code>
      myPlugin
     </code>
     ）由使用者自定义，无需和插件开发者保持一致或与开发者协调。在后续的插件使用中，该引用名将被用于表示该插件。
    </p>
    <h4 id="使用插件-2" style="background-color:transparent;">
     使用插件
    </h4>
    <p>
     使用插件时，插件的代码对于使用者来说是不可见的。为了正确使用插件，使用者应查看插件详情页面中的“开发文档”一节，阅读由插件开发者提供的插件开发文档，通过文档来明确插件提供的自定义组件、页面名称及提供的 js 接口规范等。
    </p>
    <h5 id="页面">
     页面
    </h5>
    <p>
     插件的页面从小程序基础库版本
     <a href="https://developers.weixin.qq.com/miniprogram/analysis/experience/compatibility.html" rel="nofollow" title="2.1.0">
      2.1.0
     </a>
     开始支持。
    </p>
    <p>
     需要跳转到插件页面时，
     <code>
      url
     </code>
     使用
     <code>
      plugin://
     </code>
     前缀，形如
     <code>
      plugin://PLUGIN_NAME/PLUGIN_PAGE
     </code>
     ， 如：
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>&lt;navigator url="plugin://myPlugin/hello-page"&gt;
  Go to pages/hello-page!
&lt;/navigator&gt;
</code></pre>
    <h5 id="js-接口">
     js 接口
    </h5>
    <p>
     使用插件的 js 接口时，可以使用
     <code>
      requirePlugin
     </code>
     方法。例如，插件提供一个名为
     <code>
      hello
     </code>
     的方法和一个名为
     <code>
      world
     </code>
     的变量，则可以像下面这样调用：
    </p>
    <pre><code>var myPluginInterface = requirePlugin('myPlugin');

myPluginInterface.hello();
var myWorld = myPluginInterface.world;
</code></pre>
    <p>
     基础库
     <a href="https://developers.weixin.qq.com/miniprogram/analysis/experience/compatibility.html" rel="nofollow" title="2.14.0">
      2.14.0
     </a>
     起，也可以通过插件的 AppID 来获取接口，如：
    </p>
    <pre><code>var myPluginInterface = requirePlugin('wxidxxxxxxxxxxxxxxxx');
</code></pre>
