    <blockquote>
     <p>
      在学习利用OneNET云平台的过程中，使用MQTT客户端——MQTT.fx来将设备当前所处的状态作为MQTT主题发送给IoT Hub，每个MQTT主题topic具有不同等级的名称，如“小区/区域/车位闲置” 。MQTT代理服务器将接收到的主题topic发送给给所有订阅的客户端。快速验证是否可以与IoT Hub 服务交流发布或订阅消息。
     </p>
    </blockquote>
    <p>
     MQTT.fx下载链接：
     <a href="http://www.jensd.de/apps/mqttfx/1.5.0/" rel="nofollow">

    </p>

    <h3>
     <a id="MQTTfxOneNET_12">
     </a>
     二、MQTT.fx连接OneNET
    </h3>
    <p>
     <strong>
      1、打开MQTT.fx-1.7.1客户端，点击下方的齿轮按钮
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d197d7240c8fa67f79bd9459b8c039f8.png#pic_center">
      <br/>
      <strong>
       2、在打开的对话框中进行相关参数配置
      </strong>
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/3bc9a796bc8f43c970d615308b445866.png#pic_center"/>
     </img>
    </p>
    <p>
     ① Profile Name：可以随便写入名称，支持中文
     <br/>
     ② Profile Type：保持默认设置，MQTT Broker
     <br/>
     ③ Broker Address：这个地址是设置MQTT服务器的地址，OneNET提供加密和非加密两种不同的IP地址，上图中是非加密地址，具体地址可以参考OneNET官方链接：
     <a href="https://open.iot.10086.cn/doc/mqtt/book/device-develop/manual.html" rel="nofollow">
      link
     </a>
     <br/>
     ④ Broker Port：服务器端口设置，加密和非加密的端口号是不同的；如果选用加密的服务器地址，则需要选择加密的服务器端口号，反之亦同；具体同样参考上述链接
     <br/>
     ⑤ Client ID：客户端ID，这里填入的是创建的MQTT套件中的设备名称；在MQTT物联网套件下的设备列表中选择需要模拟设备名称，我这里选择DHT11
    </p>
    <p>
     <strong>
      注：General 中的参数保持静默
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0abeed0083a15e99f12a9869fff4e88c.png#pic_center">
      <br/>
      <strong>
       3、点击User Credentials标签
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d4e4d43eaa630f682b0f25b6415e2dc4.png#pic_center"/>
     </img>
    </p>
    <p>
     ① User Name：用户名选择MQTT物联网套件中的产品ID；在产品概述中可以查阅，如下图所示：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/19ddf99b0983c52a338fa39e18c9da24.png#pic_center"/>
    </p>
    <p>
     ② Password：密码的获取需要用到Token，这里我们先暂时搁置密码的填写；讲解下Token的使用。
    </p>
    <p>
     <strong>
      4、密码计算工具Token
     </strong>
    </p>
    <p>
     Token的详细讲解可以参见链接：
     <a href="https://open.iot.10086.cn/doc/mqtt/book/manual/auth/token.html" rel="nofollow">
      link
     </a>
    </p>
    <p>
     OneNET提供了Token的上位机软件，可以方便下载并使用，参见链接：
     <a href="https://open.iot.10086.cn/doc/mqtt/book/manual/auth/tool.html" rel="nofollow">
      link
     </a>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/80482ffd0846a2960ee03bbfdc31eb71.png#pic_center"/>
     ① res：
     <strong>
      products/{pid}/devices/{device_name}
     </strong>
     ，由产品的ID和设备的名称组成
     <br/>
     pid和上述中的User Name是同一个号码，所以为349308；
     <br/>
     device_name为设备的名称，和上述中Client ID为同一个名称
    </p>
    <p>
     ② et：访问过期时间 expirationTime，unix时间；大家可以自行百度，查看当前的unix时间，设置的访问时间应大于当前的时间，当前Unix时间参考链接：
     <a href="https://time.is/Unix" rel="nofollow">
      link
     </a>
    </p>
    <p>
     ③ key：这里填入的key指的是选择设备的key；在MQTT物联网套件下的设备列表中选择需要模拟设备名称，点击详情，如下所示：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/77edcff02aa881f60e1b132bb96c6566.png#pic_center"/>
     <br/>
     res、et、key三个数据分别填写到token工具，单击右上角的Generate按键，自动生成我们需要的Password；见下图所示：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/40dbfb1ace64b63eff4002fec5b9e60e.png#pic_center"/>
     <br/>
     将上述计算工具Token产生的Password复制到第3步的②中;
    </p>
    <p>
     <strong>
      5、点击SSL/TLS标签
     </strong>
     <br/>
     可以选择加密方式进行数据的传输；如果选择加密方式进行数据传输，Broker Address和Broker Port也需要选择加密的地址和端口号；本示例中保持默认，选择不加密传输。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/603cf2553ea3fafc1eb55bbe603a39cc.png#pic_center"/>
     <br/>
     <strong>
      6、点击右下角的OK按钮，保存刚刚的一系列设置
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/be5f31cc80121a5928cf682eb22f8223.png#pic_center"/>
     <br/>
     <strong>
      7、单击下图所示的下拉三角，选择刚刚创建的Profile Name，单击Connet蓝色按钮，连接OneNET平台的MQTT服务器
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6414de91900953a74c332e1cbfe78252.png#pic_center"/>
     <br/>
     <strong>
      8、成功连接上MQTT服务器后，灰色的指示灯会变成绿色。如下图所示：
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/19ea9a2ec893b50ec3a52afa43211a28.png#pic_center"/>
     <br/>
     <strong>
      9、同样在OneNET平台设备列表可以看到当前的设备已经处于在线状态；如下所示：
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4ecd73b941109d7bd1b1b8b9479379bc.png#pic_center"/>
     <br/>
     <strong>
      10、连接完成
     </strong>
    </p>
