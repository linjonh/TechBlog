    <h2>
     <a id="_2">
     </a>
     绪论
    </h2>
    <p>
     <strong>
      APT
     </strong>
     ： 作为网络战时代的一个重要标志，高级持续性威胁（APT）快速发展。APT攻击是一种针对特定对象的长期，有计划的，有组织的网络攻击行为。
    </p>
    <h4>
     <a id="_4">
     </a>
     网络空间的保护对象
    </h4>
    <p>
     包括设施，数据，用户，操作。
    </p>
    <h4>
     <a id="_6">
     </a>
     网络空间的安全属性
    </h4>
    <p>
     安全属性主要包括
     <strong>
      机密性，完整性，可用性
     </strong>
     ，简称
     <strong>
      CIA
     </strong>
     <br/>
     随诊发展，又增加了不可否认性和可靠性、
     <br/>
     （可靠性指系统无故障的持续运行）
     <br/>
     总结：主要包括四个基本元属性：
     <strong>
      机密性，可鉴别性，可用性，可控性。
     </strong>
     <br/>
     机密性和可鉴别性属于以
     <strong>
      保护信息
     </strong>
     为主 的属性，可用性和可控性属于以
     <strong>
      保护信息系统
     </strong>
     为主的属性。
    </p>
    <h4>
     <a id="_12">
     </a>
     网络安全事件的作用空间
    </h4>
    <p>
     1.
     <strong>
      硬件层
     </strong>
     <br/>
     2.
     <strong>
      代码层
     </strong>
     <br/>
     3.
     <strong>
      数据层
     </strong>
     <br/>
     4.
     <strong>
      应用层
     </strong>
     <br/>
     （应用层的安全问题主要表现为数据挖掘，主要通过大数据处理手段，从公开信息中去挖掘脱敏信息所导致的信息泄露）
    </p>

    <h3>
     <a id="_65">
     </a>
     网络安全技术发展史
    </h3>
    <p>
     1.第一代安全技术，以
     <strong>
      保护
     </strong>
     为目的，
     <strong>
      划分明确的网络边界
     </strong>
     ，利用各种
     <strong>
      保护和隔离
     </strong>
     手段。
    </p>
    <p>
     2.第二代安全技术，以
     <strong>
      保障
     </strong>
     为目的，以
     <strong>
      检测技术
     </strong>
     为核心，以
     <strong>
      恢复技术
     </strong>
     为后盾，融合了
     <strong>
      防护
     </strong>
     ，
     <strong>
      检测
     </strong>
     ，
     <strong>
      响应
     </strong>
     ，
     <strong>
      恢复
     </strong>
     四大类技术。包括防火墙，入侵检测系统（IDS），虚拟专用网（VPN），公钥基础设施（PKI）
     <br/>
     第二代技术又称信息保障技术（IA）
    </p>
    <p>
     3.第三代安全技术，以
     <strong>
      顽存
     </strong>
     为目的，即系统在遭受攻击，故障或意外事故的情况下，在一定时间内仍然具有继续执行全部或关键使命的能力。
     <br/>
     核心：
     <strong>
      入侵容忍技术
     </strong>
    </p>

     网络协议脆弱性分析
    </h2>
    <h3>
     <a id="IP_75">
     </a>
     IP分析
    </h3>
    <h4>
     <a id="IPv4_76">
     </a>
     IPv4协议分析
    </h4>
    <p>
     没有认证机制
     <br/>
     没有加密机制
     <br/>
     无带宽控制
     <br/>
     解决方案：IPsec标准（但主要为IPv6设计的）
    </p>
    <h4>
     <a id="IPv6_81">
     </a>
     IPv6协议分析
    </h4>
    <p>
     <strong>
      IPv4 到 IPv6的过渡
     </strong>
     <br/>
     • 双协议栈
     <br/>
     • 隧道
     <br/>
     • 将IPv6的数据包封装在IPv4的数据中
     <br/>
     • 网络地址转换（NAT）
     <br/>
     <strong>
      安全隐患
     </strong>
     <br/>
     • IPv4 到 IPv6 过渡技术的安全隐患
     <br/>
     • 无状态地址自动配置的安全风险
     <br/>
     • IPv6中PKI管理系统的安全风险
     <br/>
     • IPv6编址机制的隐患
     <br/>
     • 安全机制给网络安全体系带来的安全风险
     <br/>
     • 对传统防火墙和入侵检测系统的不友好
    </p>
    <h3>
     <a id="ICMP_94">
     </a>
     ICMP分析
    </h3>
    <p>
     IPv4版本和IPv6版本
    </p>
    <h4>
     <a id="_96">
     </a>
     威胁
    </h4>
    <p>
     利用目的不可达报文发起拒绝服务攻击
     <br/>
     利用改变路由报文破坏路由表
     <br/>
     木马利用ICMP报文进行隐秘通信
     <br/>
     利用回送请求或回答报文进行网络扫描或拒绝服务攻击
    </p>
    <h3>
     <a id="ARP_101">
     </a>
     ARP分析
    </h3>
    <h4>
     <a id="_102">
     </a>
     网络嗅探
    </h4>
    <p>
     伪造arp响应发送给主机，修改arp缓存，从而重定向 IP数据流到攻击者主机
    </p>
    <h4>
     <a id="_104">
     </a>
     阻止目标的数据包通过网关
    </h4>
    <p>
     实际上原理也是修改被攻击主机的arp缓存
    </p>
    <h3>
     <a id="RIP_106">
     </a>
     RIP分析
    </h3>
    <h4>
     <a id="_107">
     </a>
     处理策略
    </h4>
    <p>
     仅与相邻路由器交换信息
     <br/>
     交换信息是自己的路由表
     <br/>
     按固定时间间隔交换信息
    </p>
    <h4>
     <a id="_111">
     </a>
     安全分析
    </h4>
    <p>
     RIPv1不支持认证并且使用不可靠的UDP作为传输协议，安全性极差
     <br/>
     • 导致攻击者可以轻易伪造RIP路由更新信息，向邻居路由发送
     <br/>
     RIPv2支持明文认证和MD5认证，认证以单向为主，但明文认证的安全性依然比较弱
     <br/>
     RIPng为IPv6环境下运行的RIP，利用IPsec提供的安全机制保证了交换路由信息的安全性
    </p>
    <h3>
     <a id="OSPF_116">
     </a>
     OSPF分析
    </h3>
    <h4>
     <a id="_117">
     </a>
     处理策略
    </h4>
    <p>
     向本系统所有路由器发送消息（泛洪法）
     <br/>
     发送的是与本路由器相邻的所有路由器的状态
     <br/>
     只有当链路状态发生改变时才发送消息
    </p>
    <h4>
     <a id="_121">
     </a>
     安全分析
    </h4>
    <h4>
     <a id="OSPF_122">
     </a>
     OSPF的三种认证方式
    </h4>
    <p>
     • 默认认证方式是不认证
     <br/>
     • 简单口令认证（明文传输）
     <br/>
     • MD5加密身份认证（使用非递减的加密序列一定程度防止重放攻击，但序列号从最大值回滚回初值或路由器重启后仍能进行重放攻击）
     <br/>
     • 容易遭受重放或伪造攻击
    </p>
    <h4>
     <a id="_127">
     </a>
     常见攻击手段
    </h4>
    <p>
     • 最大年龄攻击
     <br/>
     • 序列号加 1 攻击
     <br/>
     • 最大序列号攻击
     <br/>
     • 重放攻击
     <br/>
     • 篡改攻击
    </p>

    <h4>
     <a id="_135">
     </a>
     安全分析
    </h4>
    <p>

     面临因使用TCP而产生的安全问题

     管理员密码泄露导致路由表更改
     <br/>
     数字大炮，利用路由表更新机制造成主干网络路由器瘫痪
    </p>

     序号预测，TCP会话劫持
     <br/>
     网络扫描
    </p>
    <h3>
     <a id="DNS_144">
     </a>
     DNS安全分析
    </h3>
    <h4>
     <a id="_145">
     </a>
     域名欺骗
    </h4>
    <h5>
     <a id="ID_146">
     </a>
     事务ID欺骗：
    </h5>
    <p>
     通过网络监听或序列号预测发送相同数据包id的响应报文给目标主机，并是在正确的报文返回前就要发送给目标主机
    </p>
    <h5>
     <a id="_148">
     </a>
     缓存投毒：
    </h5>
    <p>
