    <p style="">
     <span style="">
      一、Java环境变量设置
     </span>
    </p>
    <p style="">
     <span style="">
      <strong>
       windows 下配置JDK环境变量：
      </strong>
     </span>
    </p>
    <p style="">
     1. 安装JDK，安装过程中可以自定义安装目录，例如我们选择安装目录为
     <span style="background-color:rgb(153,255,255)">
      C:\Program Files\Java\jdk1.7.0_40；
     </span>
    </p>
    <p style="">
     2. 安装完成后，右击“计算机”，点击“属性”;
    </p>
    <p style="">
     3. 点击“高级系统设置”，选择“高级”选项卡，点击“环境变量”;
    </p>
    <p style="">
     4. 在“系统变量”中，设置3项属性，
     <span style="background-color:rgb(153,255,255)">
      JAVA_HOME,PATH,CLASSPATH
     </span>
     (大小写无所谓),若已存在则点击“编辑”，不存在则点击“新建”;
    </p>
    <p style="">
     5.
     <span style="">
      <span style="">
       JAVA_HOME
      </span>
      指明JDK安装路径
     </span>
     ，就是刚才安装时所选择的路径
     <span style="font-family:'Lantinghei SC','Open Sans',Arial,'Hiragino Sans GB','Microsoft YaHei',STHeiti,'WenQuanYi Micro Hei',SimSun,sans-serif; font-size:16px; line-height:30px; background-color:rgb(153,255,255)">
      C:\Program Files\Java\jdk1.7.0_40
     </span>
     ，此路径下包括lib，bin，jre等文件夹(此变量最好设置，因为以后运行tomcat，eclipse等都需要依*此变量);
    </p>
    <p style="">
     6.
     <span style="">
      <span style="">
       Path
      </span>
      使得系统可以在任何路径下识别java命令
     </span>
     ，设为：
     <span style="background-color:rgb(153,255,255)">
      %JAVA_HOME%\bin；
     </span>
    </p>
    <p style="">
     <span style="font-family:'Lantinghei SC','Open Sans',Arial,'Hiragino Sans GB','Microsoft YaHei',STHeiti,'WenQuanYi Micro Hei',SimSun,sans-serif; font-size:16px; line-height:30px; background-color:rgb(255,255,255)">
      7.
     </span>
     <span style="">
      <span style="">
       CLASSPATH
      </span>
      为java加载类(classorlib)路径，只有类在classpath中，java命令才能识别
     </span>
     <span style="font-family:'Lantinghei SC','Open Sans',Arial,'Hiragino Sans GB','Microsoft YaHei',STHeiti,'WenQuanYi Micro Hei',SimSun,sans-serif; font-size:16px; line-height:30px">
      <span style="background-color:rgb(255,255,255)">
       ，设为.
      </span>
      <span style="background-color:rgb(204,255,255)">
       ;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar
      </span>
     </span>
     <br/>
    </p>
    <p style="">
    </p>
    <p style="">
     <span style="background-color:rgb(255,255,255)">
      8.“开始”-&gt;;“运行”，键入“cmd”;
     </span>
    </p>
    <p style="">
     <span style="background-color:rgb(255,255,255)">
      9. 键入命令“java-version”，“java”，“javac”几个命令，出现画面，说明环境变量配置成功;
     </span>
    </p>
    <span style="">
     <strong>
      <span style="font-size:18px">
       linux 下配置JDK环境变量：
      </span>
     </strong>
    </span>
    <br/>
    <p style="">
     <span style="">
     </span>
    </p>
    <p style="">
     <span style="background-color:rgb(255,255,255)">
      Linux下使用“变量名=变量值”设置变量，并使用export命令将其导出为环境变量。
      <span style="">
       为了使每一次登录都自动设置好这些变量，你需要在~/.bash_profile里或者~./bashrc里进行设置
      </span>
      ，如
     </span>
    </p>
    <p style="">
    </p>
    <p style="">
     <span style="font-size:14px">
      <span style="background-color:rgb(153,255,255)">
       export JAVA_HOME=/usr/local/jdk
      </span>
     </span>
    </p>
    <p style="">
     <span style="font-size:14px; background-color:rgb(153,255,255)">
      export PATH=$JAVA_HOME/bin:$PATH
     </span>
    </p>
    <p style="">
     <span style="font-size:14px; background-color:rgb(153,255,255)">
      export CLASSPATH=$JAVA_HOME/jre/lib/rt.jar:.
     </span>
    </p>
    <p style="">
     <span style="background-color:rgb(255,255,255)">
      设置PATH时用的$JAVA_HOME是指替换变量JAVA_HOME的值到$JAVA_HOME所在位置。如上句实际就是export PATH=/usr/local/jdk/bin:$PATH。这句中$PATH也是同样的作用，不过这里的PATH是指以前设置的PATH变量的值，而非本次设置PATH变量的值。
     </span>
    </p>
    <p style="">
     <span style="background-color:rgb(255,255,255)">
      注意，在CLASSPATH中包含了一个“当前目录(.)”。包含了该目录后，就可以到任意目录下去执行需要用到该目录下某个类的Java程序，即使该路径并未包含在CLASSPATH中也可以。原因很简单：虽然没有明确的把该路径包含在CLASSPATH中，但CLASSPATH中的“.”在此时就代表了该路径。
     </span>
    </p>
    <p style="">
     <span style="">
      二、Java命令行使用
     </span>
    </p>
    <p style="">
     <span style="">
      查看参数列表
     </span>
    </p>
    <p style="">
     虚拟机参数分为基本和扩展两类，在命令行中输入 JAVA_HOME/bin/
     <span style="">
      java -help
     </span>
     就可得到基本参数列表，在命令行输入 JAVA_HOME/bin/
     <span style="">
      java -X
     </span>
     就可得到扩展参数列表。
    </p>
    <p style="">
     <span style="">
      基本参数说明
     </span>
    </p>
    <p style="">
     <span style="">
      <span style="">
       javac –help
      </span>
     </span>
    </p>
    <p style="">
     <a href="http://static.oschina.net/uploads/img/201211/11215120_ewVd.png" rel="nofollow noopener noreferrer" style="" target="_blank">
      <span style="color:#000000">
       <img alt="image" height="494" src="https://i-blog.csdnimg.cn/blog_migrate/182868759c160504bc38d60b27cf1f83.png" style="" title="image" width="645"/>
      </span>
     </a>
    </p>
    <p style="">
     <span style="">
      <span style="">
       java -help
      </span>
     </span>
    </p>
    <p style="">
     <a href="http://static.oschina.net/uploads/img/201211/11210950_8UNe.png" rel="nofollow noopener noreferrer" style="" target="_blank">
      <span style="color:#000000">
       <img alt="image" height="393" src="https://i-blog.csdnimg.cn/blog_migrate/5c729e89b43bbe5a335779b898d0931c.png" style="" title="image" width="643"/>
      </span>
     </a>
    </p>
    <p style="">
     <span style="">
      1. -client，-server
     </span>
    </p>
    <p style="">
     这两个参数用于设置虚拟机使用何种运行模式，
     <span style="">
      client 模式启动比较快，但运行时性能和内存管理效率不如 server 模式
     </span>
     ，通常用于客户端应用程序。相反，
     <span style="">
      server 模式启动比 client 慢，但可获得更高的运行性能。
     </span>
     <br style=""/>
     <span style="">
      在 windows上，缺省的虚拟机类型为 client 模式
     </span>
     ，如果要使用 server模式，就需要在启动虚拟机时加-server 参数，以获得更高性能，
     <span style="">
      对服务器端应用，推荐采用 server 模式
     </span>
     ，尤其是多个 CPU 的系统。
     <span style="">
      在 Linux，Solaris 上缺省采用 server模式。
     </span>
    </p>
    <p style="">
     <span style="">
      2. -hotspot
     </span>
    </p>
    <p style="">
     <span style="">
      含义与 client 相同
     </span>
     ，jdk1.4 以前使用的参数，jdk1.4 开始不再使用，代之以 client。
    </p>
    <p style="">
     <span style="">
      3. -cp
     </span>
    </p>
    <p style="">
     运行Java要使用类的全名来运行。
     <span style="">
      如果遇到文件夹，则需要-cp设置到顶级包下面
     </span>
     ，例如：
    </p>
    <p style="">
     <a href="http://static.oschina.net/uploads/img/201211/11210951_hwRJ.png" rel="nofollow noopener noreferrer" style="" target="_blank">
      <span style="color:#000000">
       <img alt="200811121226497087275" height="339" src="https://i-blog.csdnimg.cn/blog_migrate/ed2de579591b9ab87c3121d6a306dbcb.png" style="" title="200811121226497087275" width="592"/>
      </span>
     </a>
    </p>
    <p style="">
     <span style="">
      4. -classpath
     </span>
    </p>
    <p style="">
     虚拟机在运行一个类时，需要将其装入内存，
     <span style="">
      虚拟机搜索类的方式和顺序如下：Bootstrap classes，Extension classes，User classes
     </span>
    </p>
    <p style="">
     <span style="">
      Bootstrap
     </span>
     是
     <span style="">
      虚拟机自带的 jar 或 zip 文件
     </span>
     ，虚拟机首先搜索这些包文件，
     <span style="">
      用System.getProperty("sun.boot.class.path")可得到虚拟机搜索的包名
     </span>
     。
     <br style=""/>
     <span style="">
      Extension
     </span>
     是
     <span style="">
      位于 jre/lib/ext 目录下的 jar 文件
     </span>
     ，虚拟机在搜索完 Bootstrap后就搜索该目录下的 jar 文件。
     <span style="">
      用System. getProperty("java.ext.dirs”)可得到虚拟机使用Extension 搜索路径
     </span>
     。
     <br style=""/>
     <span style="">
      User classes
     </span>
     搜索顺序为
     <span style="">
      当前目录、环境变量 CLASSPATH、-classpath
     </span>
     。
    </p>
    <p style="">
     <span style="">
      <span style="">
       告知虚拟机搜索目录名、jar 文档名、zip 文档名，之间用分号;分隔
      </span>
     </span>
     。例如当你自己开发了公共类并包装成一个 common.jar 包，在使用 common.jar 中的类时，就需要用-classpath common.jar 告诉虚拟机从 common.jar 中查找该类，否则虚拟机就会抛出 java.lang.NoClassDefFoundError异常，表明未找到类定义。在运行时可用 System.getProperty(“java.class.path”)得到虚拟机查找类的路径。
    </p>
    <p style="">
     <span style="">
      使用-classpath 后虚拟机将不再使用 CLASSPATH 中的类搜索路径，如果-classpath 和 CLASSPATH 都没有设置，则虚拟机使用当前路径(.)作为类搜索路径。推荐使用-classpath 来定义虚拟机要搜索的类路径，而不要使用环境变量 CLASSPATH 的搜索路径，以减少多个项目同时使用 CLASSPATH 时存在的潜在冲突。
     </span>
     例如应用 1 要使用a1.0.jar 中的类 G，应用 2 要使用 a2.0.jar 中的类 G,a2.0.jar 是 a1.0.jar 的升级包，当a1.0.jar，a2.0.jar 都在 CLASSPATH 中，
     <span style="">
      虚拟机搜索到第一个包中的类 G 时就停止搜索， 如果应用1应用2的虚拟机都从CLASSPATH中搜索，就会有一个应用得不到正确版本的类G。
     </span>
    </p>
    <p style="">
     <span style="">
      javac -classpath .;jar01.jar;jar02.jar;……;jarN.jar class_name.java ：使用-classpath选项，指定了在编译Java源代码时，用到类库的位置。一般用于该类库不在当前CLASSPATH的环境变量中。
     </span>
    </p>
    <p style="">
     <span style="">
      5. -D&lt;propertyName&gt;=value
     </span>
    </p>
    <p style="">
     <span style="">
      在虚拟机的系统属性中设置属性名/值对，运行在此虚拟机之上的应用程序可用。
     </span>
     当虚拟机报告类找不到或类冲突时可用此参数来诊断来查看虚拟机从装入类的情况。
    </p>
    <p style="">
     <span style="color:#000000">
      <a href="http://static.oschina.net/uploads/img/201211/11210953_JAax.png" rel="nofollow noopener noreferrer" style="" target="_blank">
       <img alt="200811121226497391728" height="499" src="https://i-blog.csdnimg.cn/blog_migrate/3e262935f66ee63f46149f2864f39706.png" style="" title="200811121226497391728" width="672"/>
      </a>
      <br style=""/>
      <span style="">
       7. -verbose:gc
      </span>
     </span>
    </p>
    <p style="">
     <span style="">
      在虚拟机发生内存回收时在输出设备显示信息
     </span>
     <span style="">
      ，格式如下： [Full GC 268K-&gt;168K(1984K), 0.0187390 secs]，该参数用来监视虚拟机内存回收的情况
     </span>
     。
     <br style=""/>
     <span style="">
      8. -verbose:jni
     </span>
    </p>
    <p style="">
     <span style="">
      在虚拟机调用 native 方法时输出设备显示信息，格式如下：[Dynamic-linking native method HelloNative.sum ... JNI] 该参数用来监视虚拟机调用本地方法的情况，在发生 jni错误时可为诊断提供便利。
     </span>
    </p>
    <p style="">
     <span style="">
      9. -version
     </span>
    </p>
    <p style="">
     显示可运行的虚拟机版本信息然后退出。
    </p>
    <p style="">
     <span style="">
      10. –showversion
     </span>
    </p>
    <p style="">
     显示版本信息以及帮助信息。&amp;#160;
    </p>
    <p style="">
     <br style=""/>
     <span style="">
      扩展参数说明
     </span>
    </p>
    <p style="">
     <span style="">
      javac –X
     </span>
    </p>
    <p style="">
     <a href="http://static.oschina.net/uploads/img/201211/11215124_lXM6.png" rel="nofollow noopener noreferrer" style="" target="_blank">
      <span style="color:#000000">
       <img alt="image" height="309" src="https://i-blog.csdnimg.cn/blog_migrate/15b91b52b1989c399e78286521094690.png" style="" title="image" width="644"/>
      </span>
     </a>
    </p>
    <p style="">
     <span style="">
      java -X
     </span>
    </p>
    <p style="">
     <span style="color:#000000">
      <a href="http://static.oschina.net/uploads/img/201211/11210954_UFAj.png" rel="nofollow noopener noreferrer" style="" target="_blank">
       <img alt="image" height="377" src="https://i-blog.csdnimg.cn/blog_migrate/1739b29c946ea130644d74c36054f68b.png" style="" title="image" width="640"/>
      </a>
      <br style=""/>
      <span style="">
       1. -Xmixed
       <br style=""/>
      </span>
      <span style="">
       设置-client 模式虚拟机对使用频率高的方式进行 Just-In-Time 编译和执行，对其他方法使用解释方式执行。该方式是虚拟机缺省模式。
      </span>
      <br style=""/>
      <span style="">
       2. -Xint
       <br style=""/>
      </span>
      <span style="">
       设置-client模式下运行的虚拟机以解释方式执行类的字节码，不将字节码编译为本机码。
      </span>
      <br style=""/>
      <span style="">
       3. -Xbootclasspath:path
       <br style=""/>
       4. -Xbootclasspath/a:path
       <br style=""/>
       5. -Xbootclasspath/p:path
       <br style=""/>
      </span>
      <span style="">
       改变虚拟机装载缺省系统运行包 rt.jar 而从-Xbootclasspath 中设定的搜索路径中装载系统运行类。除非你自己能写一个运行时，否则不会用到该参数。
      </span>
      <br style=""/>
      /a:将在缺省搜索路径后加上 path 中的搜索路径。
      <br style=""/>
      /p:在缺省搜索路径前先搜索 path 中的搜索路径。
      <br style=""/>
      <span style="">
       6. -Xnoclassgc
       <br style=""/>
      </span>
      <span style="">
       关闭虚拟机对 class 的垃圾回收功能。
      </span>
      <br style=""/>
      <span style="">
       7. -Xincgc
       <br style=""/>
      </span>
      <span style="">
       启动增量垃圾收集器，缺省是关闭的。
      </span>
      增量垃圾收集器能减少偶然发生的长时间的垃圾回收造成的暂停时间。但增量垃圾收集器和应用程序并发执行，因此会占用部分 CPU 在应用程序上的功能。
      <br style=""/>
      <span style="">
       8. -Xloggc:&lt;file&gt;
      </span>
      <br style=""/>
      <span style="">
       将虚拟机每次垃圾回收的信息写到日志文件中，文件名由 file 指定，文件格式是平文件，内容和-verbose:gc 输出内容相同。
      </span>
      <br style=""/>
      <span style="">
       9. -Xbatch
       <br style=""/>
      </span>
      <span style="">
       虚拟机的缺省运行方式是在后台编译类代码，然后在前台执行代码，使用-Xbatch参数将关闭虚拟机后台编译，在前台编译完成后再执行。
      </span>
      <br style=""/>
      <span style="">
       10. -Xms&lt;size&gt;
      </span>
      <br style=""/>
      <span style="">
       设置虚拟机可用内存堆的初始大小，缺省单位为字节，
      </span>
      该大小为 1024 的整数倍并且要大于1MB，可用 k(K)或m(M)为单位来设置较大的内存数。
      <span style="">
       初始堆大小为 2MB。
      </span>
      <br style=""/>
      例如：-Xms6400K，-Xms256M
      <br style=""/>
      <span style="">
       11. -Xmx&lt;size&gt;
      </span>
      <br style=""/>
      <span style="">
       设置虚拟机内存堆的最大可用大小，缺省单位为字节，
      </span>
      该值必须为 1024 整数倍，并且要大于 2MB。可用 k(K)或 m(M)为单位来设置较大的内存数。
      <span style="">
       缺省堆最大值为 64MB。
      </span>
      <br style=""/>
      例如：-Xmx81920K，-Xmx80M
      <br style=""/>
      当应用程序申请了大内存运行时虚拟机抛出 java.lang.OutOfMemoryError: Java heap space 错误，就需要使用-Xmx 设置。
      <br style=""/>
      <span style="">
       12. -Xss&lt;size&gt;
       <br style=""/>
      </span>
      <span style="">
       设置线程栈的大小，缺省单位为字节。
      </span>
      与-Xmx 类似，也可用 K 或 M 来设置较大的值。通常操作系统分配给线程栈的缺省大小为 1MB。
      <br style=""/>
      另外也可在 java 中创建线程对象时设置栈的大小，构造函数原型为 Thread(ThreadGroup group, Runnable target, String name, long stackSize)。
      <br style=""/>
      <span style="">
       13. -Xprof
       <br style=""/>
      </span>
      <span style="">
       输出 CPU 运行时的诊断信息
      </span>
      。
      <br style=""/>
      <span style="">
       14. -Xfuture
      </span>
      <br style=""/>
      <span style="">
       对类文件进行严格格式检查，以保证类代码符合类代码规范。为保持向后兼容，虚拟机缺省不进行严格的格式检查。
      </span>
      <br style=""/>
      <span style="">
       15. -Xrs
       <br style=""/>
      </span>
      <span style="">
       减少虚拟机中操作系统的信号（singals）的使用。该参数通常用在虚拟机以后台服务方式运行时使用（如 Servlet）。
      </span>
      <br style=""/>
      <span style="">
       16. -Xcheck:jni
       <br style=""/>
      </span>
      <span style="">
       调用 JNI 函数时进行附加的检查，特别地虚拟机将校验传递给 JNI 函数参数的合法性，在本地代码中遇到非法数据时，虚拟机将报一个致命错误而终止。使用该参数后将造成性能下降。
      </span>
     </span>
    </p>
    <p style="">
     <span style="">
      三、其他命令行使用
     </span>
    </p>
    <p style="">
     <span style="">
      1. jar
     </span>
    </p>
    <p style="">
     <a href="http://static.oschina.net/uploads/img/201211/11215126_ahDz.png" rel="nofollow noopener noreferrer" style="" target="_blank">
      <span style="color:#000000">
       <img alt="image" height="432" src="http://static.oschina.net/uploads/img/201211/11215126_lkTZ.png" style="" title="image" width="645"/>
      </span>
     </a>
    </p>
    <p style="">
     <span style="">
      jar命令行格式为：
     </span>
    </p>
    <p style="">
     jar {ctxu}[vfm0M] [jar-文件] [manifest-文件] [-C 目录] 文件名，其中{ctxu}是jar命令的子命令，每次jar命令只能包含一个子命令。
    </p>
    <p style="">
     <span style="">
      使用JDK的jar命令打包，会自动在压缩包中生成一个META-INF目录，其中有一个MANIFEST.MF文件。
     </span>
     使用jar命令进行如下压缩：
    </p>
    <p style="">
     <span style="">
      <span style="">
       jar cf test.jar test
      </span>
     </span>
    </p>
    <p style="">
     test.jar 压缩包中，包含一个test文件夹和一个META-INF文件夹，其中有一个MANIFEST.MF文件。
    </p>
