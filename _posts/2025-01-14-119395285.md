    <h3>
     <a id="_2">
     </a>
     基本概念
    </h3>
    <p>
     javac与java命令是我们最常用的Java命令。
     <br/>
     <strong>
      javac
     </strong>
     ：Java编译器。负责编译，将.java这个文本文件编译成.class字节码文件。
     <br/>
     <strong>
      java
     </strong>
     ：Java程序启动器。负责启动Java虚拟机(JVM)以运行Java程序。其主要用来载入字节码文件中的主类、执行jar文件等。
    </p>
    <p>
     我们可以通过
     <code>
      javac --help
     </code>
     与
     <code>
      java -help
     </code>
     来查看其帮助。
    </p>
    <p>
     执行
     <code>
      javac -help
     </code>
     显示：
     <strong>
      用法: javac
     </strong>
     <br/>
     其中的
     <strong>
      source files
     </strong>
     指的是源代码文件。比如
     <code>
      javac HelloWorld.java
     </code>
     ，生成HelloWorld.class。里面的主类是
     <code>
      HelloWorld
     </code>
     。
     <br/>
     其中的
     <strong>
      options
     </strong>
     指的是可选的其他参数。
    </p>
    <p>
     执行
     <code>
      java -help
     </code>
     显示：
     <strong>
      用法：java [options] &lt;主类&gt; [args…]
     </strong>
     <br/>
     可以看出，其必要参数是
     <strong>
      主类
     </strong>
     。注意，是
     <strong>
      主类而不是文件
     </strong>
     。因此正确执行命令是
     <code>
      java HelloWorld
     </code>
     而不是
     <code>
      java HelloWorld.class
     </code>
     。因为HelloWorld才是主类。
    </p>
    <p>
     我们会通过几个实验来简单讲解一下javac与java的基本用法。
    </p>

    <h3>
     <a id="1javacjava_35">
     </a>
     1.javac与java基本用法
    </h3>
    <p>
     执行如下命令：
    </p>
    <pre><code class="prism language-java">javac <span class="token operator">-</span>version
java <span class="token operator">-</span>version
javac <span class="token class-name">HelloWorld</span><span class="token punctuation">.</span>java
java <span class="token class-name">HelloWorld</span>
</code></pre>
    <p>
     说明：
    </p>
    <ol>
     <li>
      前两个命令用来查看当前命令行下执行的javac与java是什么版本的。
     </li>
     <li>
      编译
      <strong>
       当前目录下
      </strong>
      的HelloWorld.java文件并在
      <strong>
       当前路径下
      </strong>
      生成
      <strong>
       HelloWorld.class
      </strong>
      。
     </li>
     <li>
      启动JVM载入HelloWorld这个主类，从而运行该Java程序。再次强调一下java HelloWorld中的HelloWorld是类名而不是文件名。
     </li>
    </ol>
    <p>
     注意：一个系统上可能有多个javac与java命令文件。可以使用
     <code>
      where javac
     </code>
     或
     <code>
      where java
     </code>
     查看系统中的相应的文件。
    </p>

    <h3>
     <a id="2_javac_58">
     </a>
     2. javac的其他常用参数
    </h3>
    <p>
     这里只介绍几个常用的参数。
     <br/>
     <code>
      -d &lt;directory&gt;
     </code>
     ：在指定目录(directory)下生成相应的.class文件
     <br/>
     <code>
      -verbose
     </code>
     ：输出有关编译器正在执行的操作的消息。
    </p>
    <p>
     现在想将编译生成的.class文件自动放在d:\testjava下，如何实现呢？
    </p>
    <pre><code class="prism language-java">del <span class="token operator">*</span><span class="token punctuation">.</span><span class="token keyword">class</span>
del d<span class="token operator">:</span>\testjava\<span class="token operator">*</span><span class="token punctuation">.</span><span class="token keyword">class</span>

dir <span class="token operator">*</span><span class="token punctuation">.</span><span class="token keyword">class</span>
dir d<span class="token operator">:</span>\testjava\<span class="token operator">*</span><span class="token punctuation">.</span><span class="token keyword">class</span>
</code></pre>
    <p>
     说明:
    </p>
    <ol>
     <li>
      <code>
       del *.class
      </code>
      将当前目录下所有.class文件删除，
      <code>
       del d:\testjava\*.class
      </code>
      将
      <code>
       d:\testjava\
      </code>
      目录下所有.class文件删除。
     </li>
     <li>
      编译当前目录下的HelloWorld.java并将生成的.class文件放到d:\testjava目录。注意：d:\testjava可以不用事先建立，该条命令会自动建立。
     </li>
     <li>
      <code>
       dir *.class
      </code>
      查看当前目录下的.class文件，发现什么都没有
     </li>
     <li>
      <code>
       dir d:\testjava\*.class
      </code>
      ，查看d:\testjava\目录下的所有.class文件。
     </li>
    </ol>

     <code>
      dir *.class
     </code>
     也可进行验证。
    </p>
    <h3>
     <a id="4_cpclasspath_240">
     </a>
     4. 类路径参数：-cp或-classpath
    </h3>
    <p>
     类路径参数也是一个重要参数，格式为：
     <code>
      -classpath &lt;path&gt;
     </code>
     与
     <code>
      -cp &lt;path&gt;
     </code>
     。
     <br/>
     <code>
      -cp
     </code>
     是
     <code>
      -classpath
     </code>
     的缩写。
     <br/>
     该参数告诉java编译器，在编译某个文件时，去哪个路径去寻找该java代码文件所需要用到的类。因此称之为类路径。
    </p>
    <p>
     比如
     <code>
      d:\test
     </code>
     有如下代码：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">HelloWorld</span><span class="token punctuation">{<!-- --></span>
    <span class="token keyword">public</span> <span class="token keyword">static</span> <span class="token keyword">void</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token class-name">String</span><span class="token punctuation">[</span><span class="token punctuation">]</span> args<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token class-name">Test</span> test <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">Test</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
        <span class="token class-name">System</span><span class="token punctuation">.</span>out<span class="token punctuation">.</span><span class="token function">println</span><span class="token punctuation">(</span><span class="token string">"HelloWorld"</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     很明显，编译该文件需要一个Test类，然而Test类并未在该文件中有定义。
     <br/>
     如果直接执行
     <code>
      javac HelloWorld.java
     </code>
     ，那么其会在
     <code>
      当前目录下寻找Test.class
     </code>
     。如果没找到，会报错。
    </p>
    <p>
     有了
     <code>
      -cp
     </code>
     参数，我们就可以将类所需的其他.class放置到另外一个目录，编译的时候可以指定编译器的查找该路径。
    </p>
    <p>
     了解了这个原理，我们可以自己做一个实验：
    </p>
    <ol>
     <li>
      在
      <code>
       d:\testjava\
      </code>
      目录下新建一个Test.java，代码很简单
      <code>
       public class Test{}
      </code>
      。
     </li>
     <li>
      进行编译。生成的Test.class在d:\testjava\目录下。
     </li>
     <li>
      再执行
      <code>
       javac -cp d:\testjava\ HelloWorld.java
      </code>
      ，就可以顺利完成编译。
     </li>
    </ol>
    <p>
     事实上，即使你不手动编译Test.java，上面那条javac命令也会在d:\testjava\下寻找Test.java文件。如果有，就自动编译。
    </p>
    <h3>
     <a id="_272">
     </a>
     总结
    </h3>
    <ol>
     <li>
      javac可以将.java文件编译成字节码文件。
     </li>
     <li>
      Java 11中的javac编译时，使用到了模块系统。所有java程序都需要
      <code>
       java.base
      </code>
      模块。
     </li>
     <li>
      编译时会将程序所需
      <strong>
       类与接口
      </strong>
      全部导入。
     </li>
     <li>
      Java将类的信息都存储在.class文件中。启动Java程序时，会从.class文件中载入相关类。
     </li>
     <li>
      <code>
       -d
      </code>
      参数可以在指定目录(directory)下生成相应的.class文件。
     </li>
     <li>
      <code>
       -cp
      </code>
      参数可以帮java编译器寻找相应的类。这个参数很重要，应当理解。IDE中虽然不需要使用命令行，但如果用到第三方库，可能就需要对
      <code>
       Build Path
      </code>
      进行设置。这种设置实际上就是修改了相应的类路径。
     </li>
    </ol>
