     <p style="text-align:center;">
      —
      <strong>
       写在前面的话
      </strong>
      —
     </p>
     <p style="text-align:justify;">
      笔者从2013年开始进入智能家居领域，这些年磕磕碰碰掉了不少坑，一路走来几乎面对所有要解决的问题都找不到人可以指点，全凭自己交学费。
     </p>
     <p style="text-align:justify;">
      <strong>
       此篇文章的目标是希望能对一个完整涉及到互联网产业上下游的产品进行覆盖性讲解，力争通过尽量少的文字讲明白一个互联网产品各领域所涉及到的知识点。
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      笔者尽量在
      <strong>
       技术原理、行业知识、产品设计
      </strong>
      这三个方面之间做出权衡，不过分偏重技术但又避免表面知识，最基本的，会让读者明白未来准备涉及的相应领域要怎么入手。
     </p>
     <p style="text-align:right;">
      —— 柯鼠
     </p>
     <p>
      <strong>
       Now, let's make the cat out of the bag.
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="41f5de7427496bb8905bba0a4f245c89.png" src="https://i-blog.csdnimg.cn/blog_migrate/5a232e253e58840ba44af5c3d1db38df.png"/>
     </p>
     <p style="text-align:left;">
      <img alt="98d5550f2e08f9b23ff9db13e2bda999.png" src="https://i-blog.csdnimg.cn/blog_migrate/70af7e868ec302800a7e11a228528281.jpeg"/>
     </p>
     <p style="text-align:left;">
      <img alt="9df340bf5dc6394bc3f34985288818d8.png" src="https://i-blog.csdnimg.cn/blog_migrate/3c30941c3fe41c72e91c2445bde5a7ad.jpeg">
       <br/>
      </img>
     </p>
     <p style="text-align:center;">
      <strong>
       一、项目立项
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （一）客户背景
      </strong>
     </p>
     <p style="text-align:justify;">
      客户来自中国台湾，希望做一个智能语音机器人，并且带宝宝远程视频看护功能，对标大陆这边的智伴机器人和阿尔法蛋。
     </p>
     <p style="text-align:justify;">
      客户质量方面，其自身积累了大量人脉，与各种渠道商都建立有大量合作，同台湾的一些酒店连锁集团也建立有合作关系，而且在敬老院市场也有相关渠道入口，未来销量保底 80K/Year......
     </p>
     <p style="text-align:justify;">
      <strong>
       客户掏了开案费。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （二）需求确认
      </strong>
     </p>
     <p style="text-align:justify;">
      由于商务合作内容方面的关系，客户最终要求交 Turn-key 的形式，翻译过来就是：
      <strong>
       撒手不管但你最好按我脑子里想的要求去做。
      </strong>
     </p>
     <p style="text-align:justify;">
      <img alt="f3a5bc7f0bf582c56cc867f013218884.png" src="https://i-blog.csdnimg.cn/blog_migrate/349a7d1446938e2c4e5d470b9ba9ebeb.jpeg"/>
     </p>
     <p style="text-align:justify;">
      经多番友好沟通之后，客户最终还是给了一份需求文档：
      <strong>
       “你猜猜我要啥 &amp; 你看着设计呗.docx”
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （三）自身经验
      </strong>
     </p>
     <p style="text-align:justify;">
      一般从零开始做一整套互联网硬件产品，预期时间大约为一年，但是这次客户的期望时间是6个月。
     </p>
     <p style="text-align:justify;">
      <img alt="58072979a8737ae1b01a910f16fda409.png" src="https://i-blog.csdnimg.cn/blog_migrate/fa3450b4a509c3d075112b18c30c8443.jpeg"/>
     </p>
     <p style="text-align:justify;">
      作为一个纯粹的物联网云服务提供商，此前完全没有硬件及嵌入式开发经验，突然被要求跑去给客户做完整套硬件产品的确是不小的挑战，
      <strong>
       况且由于公司内部项目的安排，只有三个程序猿可供我虐待。
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      <strong>
       （四）Get the shit done
      </strong>
     </p>
     <p style="text-align:justify;">
      虽然公司业务为云服务，但好在云端系统从一开始就要求为模块化设计，类似现在流行的中台，内部代号为 CloudMeet 。模块化的好处是可以选择组合出各种业务产品云端服务形态。
     </p>
     <p style="text-align:justify;">
      <strong>
       （五）关键性问题
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="e873441987d9bbbc1b44a31aae30d2e2.png" src="https://i-blog.csdnimg.cn/blog_migrate/7856a2dc1bcb394447efb5a1b94d1535.jpeg">
       <img alt="baca3d0cac9a1f2852f1cc3f060b5481.png" src="https://i-blog.csdnimg.cn/blog_migrate/14eedbd56fa621fdf2f5507e3a6244a9.jpeg"/>
      </img>
     </p>
     <p style="text-align:center;">
      <img alt="c0da8148f696ab10ee9382ddb9aace00.png" src="https://i-blog.csdnimg.cn/blog_migrate/ef1ff6069d03b32eba37a57316b3c0de.png"/>
      <br/>
     </p>
     <p style="text-align:center;">
      <strong>
       二、工业设计（ID）
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （一）形象定位
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      产品定位为智能语音机器人，面对的是儿童教育市场，所以在产品形象上要考虑小朋友的喜好，无非几种：
      <strong>
       动物、卡通、人形机器人。
      </strong>
     </p>
     <p style="text-align:justify;">
      卡通形象没钱买IP，人形机器人则太直男，
      <strong>
       最后选定动物形象：狗子。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （二）产品需求确认
      </strong>
     </p>
     <p style="text-align:justify;">
      设计方面，找了深圳设计公司A的老王，俗称
      <strong>
       王工
      </strong>
      ；
     </p>
     <p style="text-align:justify;">
      <img alt="a56f05651a206467589bebfb1f2c106e.png" src="https://i-blog.csdnimg.cn/blog_migrate/d30f5ff7c39665133ad0d4c9092ebb79.jpeg"/>
     </p>
     <p style="text-align:justify;">
      初步面谈沟通后，告知了设计师产品立项的背景和目标；然后再提供正式的设计需求文档给到王工，说明产品的
      <strong>
       设计定位、形象要求、外观设计要求等基本要素。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （三）初步设计稿
      </strong>
     </p>
     <p style="text-align:justify;">
      在确认需求后，王工开始进行设计，一般工期大概为一周。
     </p>
     <p style="text-align:justify;">
      常规的设计公司合作方式，
      <strong>
       一般是一份设计合同提供三份不同的原型设计
      </strong>
      ，然后让客户从中挑选一个作为选定方案。
     </p>
     <p style="text-align:justify;">
      <img alt="ff3de1ad67e5ad2fca8d082253de671a.png" src="https://i-blog.csdnimg.cn/blog_migrate/93d640eb5ffc4f10890859619b2793f7.jpeg"/>
     </p>
     <p style="text-align:justify;">
      另外，
      <strong>
       比较贱的设计公司会让高水平的设计师设计一份稿件，再让公司实习生或者初级设计师做两份 Bullshit 凑数，
      </strong>
      从而让客户能产生一眼就相中的快感。
     </p>
     <p style="text-align:justify;">
      <strong>
       总之，设计套路深，多看别当真~
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （四）初次手板
      </strong>
     </p>
     <p style="text-align:justify;">
      选定了设计方案后，需要再跟王工进一步沟通细化调整，包括整体尺寸和颜色搭配调整，细节调整完成后，则开始安排制作手板。
     </p>
     <p style="text-align:justify;">
      手板的意思是验证模型，
      <strong>
       将产品的设计纯粹用塑料模型制作出来以便确认实际效果。
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="960aa27e5361e25726d7b900fcde248b.png" src="https://i-blog.csdnimg.cn/blog_migrate/33e5ad6a51ca69137615051bb6f1bad0.jpeg"/>
      <br/>
     </p>
     <p style="text-align:center;">
      CNC雕刻加上喷油可以得出非常漂亮的质感
     </p>
     <p style="text-align:justify;">
      这部分进行处理的是手板厂的程工，王工给程工提供设计文件，手板类型选择了程工说的最漂亮的CNC雕刻。
      <br/>
     </p>
     <p style="text-align:justify;">
      <strong>
       （五）直男白+中国红
      </strong>
     </p>
     <p style="text-align:justify;">
      手板出来后，
      <strong>
       跟同事一起左看右看，怎么看怎么好看。
      </strong>
     </p>
     <p style="text-align:justify;">
      为体现该设计符合市场目标客户的需求，特意带着手板去请一个咖啡店的老板娘鉴赏，她有一个六岁的小朋友，典型的目标客户。
     </p>
     <p style="text-align:justify;">
      一开场我便激昂而款款地谈论产品的设计创意和市场目标，
      <strong>
       终于在口水多过咖啡的交流过程中得出一个结论：
      </strong>
      <strong>
       直男设计。
      </strong>
     </p>
     <p style="text-align:justify;">
      值得反思的是，
      <strong>
       毕竟连女孩子的手都没牵过，不能正确理解母婴市场的产品设计原则应该也是情有可原。
      </strong>
     </p>
     <p style="text-align:justify;">
      于是再去做功课，研究了市场上儿童智能设备的外观设计比例、配色特点。人类视觉上对于“萌”的感觉有会一些关键特征。
     </p>
     <p style="text-align:justify;">
      比如
      <strong>
       ：动漫的角色大都具有大眼睛大脑袋短身子
      </strong>
      ，而人类的婴儿也是类似的特征。
     </p>
     <p style="text-align:center;">
      <img alt="5c784ae868510e6a087bd94f71383fe5.png" src="https://i-blog.csdnimg.cn/blog_migrate/bf26d33c7d2f70f6f407cf267a9b7622.jpeg"/>
     </p>
     <p style="text-align:center;">
      电影《神偷奶爸》剧照，图片来源网络
     </p>
     <p style="text-align:justify;">
      估计不按这种图纸比例生出来的婴儿，在远古时期容易养着养着就被大人撒点孜然烤了，不利于物种繁衍。
      <br/>
     </p>
     <p style="text-align:justify;">
      <strong>
       （六）比例调整
      </strong>
     </p>
     <p style="text-align:justify;">
      定出新的设计方向后，然后就是麻烦王工再通宵加班修改了。
     </p>
     <p style="text-align:justify;">
      首先是比例调整，这次没有再选CNC，而是选择了3D打印。
     </p>
     <p style="text-align:center;">
      <img alt="8701adc5acce3aaf2dbffd223d899802.png" src="https://i-blog.csdnimg.cn/blog_migrate/9bd471dcc09b6018d97e9af83bab1d60.jpeg"/>
     </p>
     <p style="text-align:center;">
      3D打印
     </p>
     <p style="text-align:justify;">
      对于外观及颜色的初步验证来说，这已经足够，关键是制作周期要短很多（便宜）。
     </p>
     <p style="text-align:justify;">
      <strong>
       （七）颜色调整
      </strong>
     </p>
     <p style="text-align:justify;">
      颜色部分遇到了一个问题，就是颜色的具体定义。我们重新选定了淡蓝色和淡粉色两个配色标准，但这两个描述对于设计师而言，
      <strong>
       就跟我描述女生的口红色号一样：
      </strong>
      <strong>
       不是姨妈红就是牛屎绿。
      </strong>
     </p>
     <p style="text-align:justify;">
      尝试在网上找色盘去选定色值参数，而王工进行上色设计后，怎么看都不对。反复折腾了王工之后终于还是办法比问题多：
      <strong>
       直接从网上买了一对杯子
      </strong>
      。
     </p>
     <p style="text-align:justify;">
      <img alt="95b11d0673734a93a4871d2f7b35fa1c.png" src="https://i-blog.csdnimg.cn/blog_migrate/4263fcf6ec0b0dedc7853c57e8a49d63.jpeg"/>
     </p>
     <p style="text-align:justify;">
      这两只杯子刚好一粉一蓝，而且颜色符合要求，王工直接对着杯子参考。
     </p>
     <p style="text-align:justify;">
      <strong>
       （八）关键性问题
      </strong>
     </p>
     <p style="text-align:justify;">
      <img alt="c874a99a13b990c6fff12debd3fe8ca8.png" src="https://i-blog.csdnimg.cn/blog_migrate/66598ff1b6b13c29959e105f61d2b7e1.jpeg"/>
     </p>
     <p style="text-align:left;">
      <img alt="eefdce4e45190b8957fc6034d98c9497.png" src="https://i-blog.csdnimg.cn/blog_migrate/eec2f92d8fba8890698d7beee0683afd.png"/>
      <br/>
     </p>
     <p style="text-align:center;">
      <strong>
      </strong>
      <strong>
       三、结构设计（MD)
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （一）设计需求
      </strong>
     </p>
     <p style="text-align:justify;">
      ID设计确认之后，下个阶段便是产品内部结构设计，这部分找的是深圳另一家设计公司B的
      <strong>
       刘工
      </strong>
      。
     </p>
     <p style="text-align:justify;">
      <img alt="4a49ccf59d477e573b9de5cc7c0852b5.png" src="https://i-blog.csdnimg.cn/blog_migrate/57cf483621d47e8ae23f8acd17c4112c.jpeg"/>
     </p>
     <p style="text-align:justify;">
      常规来说一般ID和MD都会选择同一家公司设计，一方面合并费用低一些，另一方面减少沟通成本。
      <br/>
     </p>
     <p style="text-align:justify;">
      不过由于一些原因，我最终将ID和MD设计分开，所以这阶段会再次产生设计需求的确认工作。
     </p>
     <p style="text-align:justify;">
      <strong>
       外观部分，ID工程师提供设计的渲染图及对应尺寸标注文件，并注明外观颜色要求；结构部分，电子工程师提供电子结构空间设计要求文档，告知所选用的关键电子元器件尺寸及散热、布局和避空要求。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （二）首次结构设计
      </strong>
     </p>
     <p style="text-align:justify;">
      因为整体结构比较大，发挥空间充足，
      <strong>
       所以首次结构并不需要考虑太多元器件冲突问题。
      </strong>
     </p>
     <p style="text-align:justify;">
      <img alt="3055874671926755fb699f341d153ba4.png" src="https://i-blog.csdnimg.cn/blog_migrate/31c5bc26bfa2b258a171a664b0132555.jpeg"/>
     </p>
     <p style="text-align:justify;">
      在ID确定之后，修改MD部分的时候一般由于实际的空间摆放、器件避空等要求，外观都会有调整，这部分由于刘工也做过ID，就一并修改了。
      <br/>
     </p>
     <p style="text-align:justify;">
      结构设计工作完成就可以进行3D打印做手板，用以确认外观变动、结构实际状况。
      <strong>
       一般首次结构实物确认都是体现设计问题，比如螺丝柱遗漏、按键骨架脆弱等。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       （三）初版结构确认
      </strong>
     </p>
     <p style="text-align:justify;">
      在调整结构设计问题之后，再进行3D打印的手板制作，经实物确认即可作为初版的结构定型。
     </p>
     <p style="text-align:justify;">
      下一步刘工会向电子工程师提供
      <strong>
       版框图
      </strong>
      ，电子工程师根据结构调整原理图及元器件选用，进行后续的电路Layout，电路部分样品制作完成将与结构进行配合验证。
     </p>
     <p style="text-align:justify;">
      <strong>
       （四）0.618
      </strong>
     </p>
     <p style="text-align:justify;">
      在第一次拿到电路板正式与外壳手板进行组装之后，总还是觉得外观有所欠缺。
     </p>
     <p style="text-align:center;">
      <strong>
       改 ！！！
      </strong>
     </p>
     <p style="text-align:justify;">
      首先是耳朵，感觉上有点偏大，然后按一个毫米的幅度反复调整了两三个版本，尾巴也为了卡哇伊缩短为柯基版本。而最麻烦的是身体比例，
      <strong>
       为了头部与身体的视觉协调，反复拜托刘工进行微调，以希望头身比例尽可能逼近黄金分割比。
      </strong>
     </p>
     <p style="text-align:justify;">

     <p style="text-align:justify;">
      <strong>
       （五）MIC位置确定
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       1.单 MIC
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      产品的麦克风主要作用的机器人唤醒（类似“喂，Siri”）、AI对讲和远程双向通话，物理上要考虑拾音、回声的问题。
     </p>
     <p style="text-align:justify;">
      一开始结构设计上考虑的是单MIC设计，这种情况下拾音效果最佳的位置是正对用户，
      <strong>
       在ID设计初期MIC开孔放在了机器人正面嘴巴的位置。
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="ee8cbee22ffb6f94741476b493e93d70.png" src="https://i-blog.csdnimg.cn/blog_migrate/1d724e1197556c3eaf86e7fafe6bab92.jpeg"/>
     </p>
     <p style="text-align:center;">
      单MIC位置
     </p>
     <p style="text-align:justify;">
      <strong>
       2.MIC 阵列
      </strong>
     </p>
     <p style="text-align:justify;">
      ID阶段制作了手板发现不大美观，所以在MD设计时就放在了铃铛的位置。感觉美观问题总算是解决了客户又提了个小需求：
      <strong>
       预留双MIC阵列，提升拾音和回音消除效果。
      </strong>
     </p>
     <p style="text-align:justify;">
      这个小需求怎么说呢，反正，客户是上帝。于是跑去咨询一个音频算法大牛，
      <strong>
       彭工
      </strong>
      。大牛表示
      <strong>
       要想实现双MIC阵列，在开孔的物理设计上有严格的要求，这会直接关系到拾音的能力、回音消除的效果。
      </strong>
     </p>
     <p style="text-align:justify;">
      贿赂了两包黄鹤楼之后，刘工终于又修改了结构，
      <strong>
       两个MIC的位置最终被放置在了机器人顶部。
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="2a8260f9d9d4d6155f23bbad261e6e9c.png" src="https://i-blog.csdnimg.cn/blog_migrate/f2fa3944756dc7e7d4894dce0f793770.jpeg"/>
     </p>
     <p style="text-align:center;">
      MIC阵列位置
     </p>
     <p style="text-align:justify;">
      <strong>
       （六）声音系统
      </strong>
     </p>
     <p style="text-align:justify;">
      放音喇叭的设计是一套比较系统的工程，牵涉的部分也比较多，关系到：
      <strong>
       共振、MIC输入干涉、放音品质、设备重心等一系列考量。
      </strong>
     </p>
     <p style="text-align:justify;">
      正常来说，为避免SPK放出的声音影响到MIC的输入，
      <strong>
       喇叭在设计要尽量远离麦克风并且最好呈背对形式。
      </strong>
      比如360小水滴、萤石C2W两种智能摄像头的设计。
     </p>
     <p style="text-align:justify;">
      <img alt="7a7b2ae215729ea038fe355adeac4cbd.png" src="https://i-blog.csdnimg.cn/blog_migrate/7e65a1bf891ffb6670355ae07eca68e7.jpeg"/>
     </p>
     <p style="text-align:justify;">
      <strong>
       1.SPK位置
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      单MIC正前方设计时，喇叭在机身后的位置影响不大，因为彼此是背对的。
     </p>
     <p style="text-align:justify;">
      但双MIC布局在脑袋顶部之后，就会发现喇叭和麦克风已经不能背对设计了，那么
      <strong>
       SPK就要考虑尽量远离MIC
      </strong>
      。
     </p>
     <p style="text-align:justify;">
      最开始考虑的是放在机身背部下方，刘工修改了之后看效果图还是觉得比较丑。再往上摆，处于脑袋正下方，但由于形状冲突（弧面过小），最终只能选择在脑袋正后方。
     </p>
     <p style="text-align:justify;">
      <strong>
       2.SPK选型
      </strong>
     </p>
     <p style="text-align:justify;">
      在测试声音播放品质过程中，遇到了不少问题。
     </p>
     <p style="text-align:justify;">
      首先是喇叭的选型，外磁的喇叭价格比较便宜但是重量大，内磁的喇叭轻一些但是价格高。
     </p>
     <p style="text-align:justify;">
      由于喇叭位置在脑袋正后方，重心靠外，所以为尽量平衡产品的重心问题，选用了一个内磁喇叭，
      <strong>
       然而音质测试并不理想。
      </strong>
     </p>
     <p style="text-align:justify;">
      在测试声音播放品质时，发现不管更换任何测试音频，都有一种声音被闷住无法输出的感觉。
     </p>
     <p style="text-align:justify;">
      在更换了三四个供应商提供的喇叭还是不满意之后，直接从阿里巴巴上找了十几家喇叭厂商，逐一购买了内磁、外磁的各型喇叭数十个。
     </p>
     <p style="text-align:justify;">
      在各种喇叭快装满一抽屉之后，
      <strong>
       想到了从喇叭开孔问题
      </strong>
      。因为不管换了多少种喇叭，声音都处于闷住状态，直接把喇叭开孔全部砸开就没问题，所以只能拜托刘工再从喇叭开孔入手。
     </p>
     <p style="text-align:justify;">
      <strong>
       3.开孔设计
      </strong>
     </p>
     <p style="text-align:justify;">
      在物理上，喇叭的开孔并不是随意而为之，而是有既定的开孔公式进行换算。一开始我们打算单纯增加开孔数量，但是过多的开孔数量会导致孔径变小，这会影响后续的模具问题。
     </p>
     <p style="text-align:justify;">
      权衡之下，只能尝试在
      <strong>
       小量增加开孔数量的同时，更改开孔排列形状
      </strong>
      ，然后把几个不同形状的喇叭开孔打了手板进行验证之后，但仍是差强人意。
     </p>
     <p style="text-align:justify;">
      <strong>
       这要上升到玄学的范畴了。
      </strong>
     </p>
     <p style="text-align:justify;">
      从结构设计师、喇叭供应商、电子工程师都已经无法理解到底是什么原因导致了声音无法出来。看着刘工怨念的眼神，我突然想到
      <strong>
       为何不能仿造别家产品的喇叭开孔？
      </strong>
     </p>
     <p style="text-align:justify;">
      <img alt="b45ef67ba13e29dc4a47890f111d74a6.png" src="https://i-blog.csdnimg.cn/blog_migrate/4025462de6d274309f40c39bb7b36f4c.jpeg"/>
     </p>
     <p style="text-align:justify;">
      于是立即下单买了个小米的米兔，带着一包玉溪又拜托刘工比对米兔的开孔设计修改了一次——声音终于是肯出来了，最后为了音质，还是选定了一个外磁的喇叭。
      <br/>
     </p>
     <p style="text-align:justify;">
      <strong>
       4.后音腔
      </strong>
     </p>
     <p style="text-align:justify;">
      声音能出来之后，虽然音质满意，但是洪量度不够，
      <strong>
       又双叒叕请刘工再调整设计，增加了后音腔。
      </strong>
     </p>
     <p style="text-align:justify;">
      但手板打出来之后测试发现没啥帮助，加上客户考虑到增加的模具费用，最终并未使用后音腔设计，但预留了装配空间。

       老陈

      </strong>
      之后，便是付款开模。

     <p style="text-align:justify;">

     </p>
     <p style="text-align:justify;">

     </p>
     <p style="text-align:justify;">

       （四）试模
      </strong>
     </p>
     <p style="text-align:justify;">
      模具制作完成，
      <strong>
       模具厂会先购买原料通过注塑机（啤机）试产小数量成品进行验证，
      </strong>
      包括外观、结构、颜色相关问题。
     </p>
     <p style="text-align:justify;">
      <strong>
       1.触控键
      </strong>
     </p>
     <p style="text-align:justify;">
      机器人头顶设计有一个触控按键，这个按键需要通过铜皮利用电容效应进行触发，为避免电路触发不够灵敏，所以在铜皮贴合位置的壳子部分进行了减胶做薄处理。
     </p>
     <p style="text-align:justify;">
      <img alt="d1741992773924215a16c93feec068f0.png" src="https://i-blog.csdnimg.cn/blog_migrate/6c8e6274c81c1e9050378b853910beb3.jpeg"/>
     </p>
     <p style="text-align:justify;">
      由于在壳子上挖了个正方形的矩形坑，导致生产时带来了
      <strong>
       缩水的问题
      </strong>
      ，最后解决的办法是将矩形坑做圆角处理，减轻注胶冷却时各区域收缩不一致问题。
     </p>
     <p style="text-align:justify;">
      <strong>
       2.电池仓
      </strong>
     </p>
     <p style="text-align:justify;">
      机器人支持电池使用模式，为追求终端用户体验设计选用了两颗18650电池组，就是特斯拉上的那种。
     </p>
     <p style="text-align:justify;">
      这种类型锂电池有较好的稳定性，但是
      <strong>
       缺点是偏重
      </strong>
      。固定电池的电池仓一开始设计的是两条腿固定，但是实际组装后进行摔落测试发现很容易出现断腿事件，讨论之后只能增加腿的数量到四条，改结构。
     </p>
     <p style="text-align:justify;">
      <strong>
       3.耳朵硬度
      </strong>
     </p>
     <p style="text-align:justify;">
      耳朵一开始为了小朋友摸起来舒服，将硬度设计得很低，这使得耳朵稍微使劲一拽就掉了。
     </p>
     <p style="text-align:justify;">
      让模厂调整材料，变成硬质形式，又导致机器人很不扛摔，多次修改尝试之后，
      <strong>
       最终选定了一个材料硬度系数在保证不被抠出来的同时又让耳朵尽量柔软。
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       4.颜色校正
      </strong>
     </p>
     <p style="text-align:justify;">
      外观颜色方面，虽然ID给了详细的数值，但是实际生产出来的效果仍然还是有细微差距，因为
      <strong>
       色粉的值
      </strong>
      并不是完全跟计算机图示效果一致，所以最终把王工手里的两个杯子给拿到了模厂进行比对配色。
     </p>
     <p style="text-align:justify;">
      <strong>
       （五）定型
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       1.签样
      </strong>
     </p>

      <strong>

       2.验收标准
      </strong>

      <img alt="ebde7f3333489378fe426ef58cd2fafd.png" src="https://i-blog.csdnimg.cn/blog_migrate/43d6d31ddca9198000f65fb78f7f4716.jpeg"/>

      <img alt="d92d09e984c4108188b12ecb9d7dfc3d.png" src="https://i-blog.csdnimg.cn/blog_migrate/c4ff8c8dbcdcb10740d27563592218e1.jpeg"/>

       天天把戒烟戒酒挂在嘴边，但身体从来都很诚实。
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="fc2fadcd8c7b460943c3b8d9dc3c4bac.png" src="https://i-blog.csdnimg.cn/blog_migrate/d619d3e71ec59148b112f06f42a61607.jpeg"/>
     </p>
     <p style="text-align:left;">
      在张工开始设计之前，我需要先编写产品电子规格文档交给张工，除了产品的功能描述和市场定位，还包括
      <strong>
       摄像头参数、点阵参数、电池容量、耳朵颜色
      </strong>
      等。
      <br/>
     </p>
     <p style="text-align:left;">
      <strong>
       （二）设计资料确认
      </strong>
     </p>
     <p style="text-align:left;">
      产品电子规格确认之后，需要组织CPU原厂、嵌入式工程师、电子设计工程师进行讨论，原厂的工程师会告知嵌入式系统的一些特性及电路设计上的要求。
     </p>
     <p style="text-align:left;">
      <strong>
       工程师之间的沟通往往比较质朴，闷头讲完画几个白板就是各自留邮箱和微信了，
      </strong>
      该阶段原厂这边会将电路设计用到的芯片资料和原理图资料发出来。
     </p>
     <p style="text-align:left;">
      （三）EVT阶段
     </p>
     <p style="text-align:left;">
      Engineering Verification Test，
      <strong>
       该阶段设计纯粹验证电路原理设计及Layout的设计。
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="b5550d156e85addf4ff1c76481a805d8.png" src="https://i-blog.csdnimg.cn/blog_migrate/ab5e2183d9a750091060e52d9d68544d.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       1.硬件设计
      </strong>
     </p>
     <p style="text-align:left;">
      张工拿到原厂给的资料首先会进行的是电路原理设计。
     </p>
     <p style="text-align:left;">
      <strong>
       该阶段硬件第一步目标首先要进行验证性研发
      </strong>
      ，既完整产品涉及到的所有硬件功能节点都设计到一块电路板上，主要用于原理设计及元器件选型的可行性验证。
     </p>
     <p style="text-align:left;">
      原理图设计完成张工会将设计文件发回给原厂，原厂的硬件工程师Evans会进行
      <strong>
       原理图的设计review
      </strong>
      ，以保证张工的原理设计是符合芯片技术要求。
     </p>
     <p style="text-align:left;">
      原理图修正并确认后，既可进行PCB Layout设计，将元器件布局到电路上。
     </p>
     <p style="text-align:left;">
      <strong>
       2.洗板
      </strong>
     </p>
     <p style="text-align:left;">
      所有设计经过确认之后，便可以交由洗板厂进行PCB洗板，洗板所花费的时间会受到PCB层数的影响，比如从二层板、四层板、六层板所需要的洗板时间会以此增加，费用也会更贵，当然对于硬件性能来说也会更好些。
     </p>
     <p style="text-align:left;">
      <strong>
       3.SMT
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       PCB洗板完成，物料备齐之后便是上机贴片（SMT）。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       SMT是表面贴装技术Surface Mounted Technology的缩写
      </strong>
      ，SMT贴片指的是在PCB基础上进行加工的系列工艺流程的简称。
     </p>
     <p style="text-align:left;">
      电路板通过钢网刷上锡膏之后，由高速贴片机(SMT)将相应元器件贴合至电路板上，该步骤得出的结果便是
      <strong>
       PCBA。
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="adbcabe32ae08026d6a97338af57f8b8.png" src="https://i-blog.csdnimg.cn/blog_migrate/747ea88f3fcc5c610b197597ec171fab.jpeg"/>
     </p>
     <p style="text-align:center;">
      这是机器的样子
      <br/>
     </p>
     <p style="text-align:left;">
      <strong>
       4.组装
      </strong>
     </p>
     <p style="text-align:left;">
      将外围器件，比如MIC、SPK、电池等相关器件，通过相应电子连线串接至PCBA上，便可得到一个完整的硬件设备，由于处于EVT阶段，硬件设计与结构无相关性，所以这里组装并不涉及外壳。
     </p>
     <p style="text-align:left;">
      <strong>
       5.测试
      </strong>
     </p>
     <p style="text-align:left;">
      该阶段硬件测试主要是电路的工作测试，比如上电之后CPU是否能正常点亮，各种模块的电流、电压是否正确等等。
      <strong>
       对于问题的修正，一般用飞线的形式进行操作验证。
      </strong>
     </p>
     <p style="text-align:center;">
      <img alt="ac1671b4742cee41ebbdeb54a159f3d8.png" src="https://i-blog.csdnimg.cn/blog_migrate/3c438176c7817f50ad56bf5369e4e2d7.jpeg"/>
     </p>
     <p style="text-align:center;">
      所有的元器件都集合到一个板子
     </p>
     <p style="text-align:left;">
      <strong>
       （四）DVT阶段
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       Design Verification Test，
      </strong>
      该阶段修正EVT阶段的电路原理设计，并根据外壳结构进行Layout验证。
     </p>
     <p style="text-align:left;">
      <strong>
       1.设计
      </strong>
     </p>
     <p style="text-align:left;">
      在EVT阶段验证通过之后，便可以进行下一阶段的DVT设计。
     </p>
     <p style="text-align:left;">
      张工根据EVT的测试结果，重新修正原理图设计并发给原厂review，并根据从设计公司B的刘工那给出的结构版框图重新进行layout，将元器件按正式的外壳结构进行布局。
     </p>
     <p style="text-align:left;">
      <img alt="63cde2a2eb74f51b85ec01e80843206e.png" src="https://i-blog.csdnimg.cn/blog_migrate/91a53a7ebb23af5dbae5d82b4bed8ce8.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       2.结构验证
      </strong>
     </p>
     <p style="text-align:left;">
      该阶段板子洗完并贴片后，即可与初版的外壳手板进行匹配验证。
      <strong>
       包括电路板与螺丝柱契合是否正确，结构与板型对于生产方面的简易程度、抗摔性等。
      </strong>
      外壳结构及电路布局设计都会进行微调以实现两者锲合度最佳化。
     </p>
     <p style="text-align:left;">
      <strong>
       3.软硬件功能配合开发
      </strong>
     </p>
     <p style="text-align:left;">
      电路设计的经过EVT的验证，在DVT阶段修改了layout之后，则开始配合嵌入式、单片机程序进行验证。
     </p>
     <p style="text-align:left;">
      单片机、嵌入式程度，对于电路硬件的验证会进行反复调整修正，以确定硬件设计是符合软件要求。同时OEM工厂会与天线工厂完成
      <strong>
       天线的位置测定。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       这个阶段是整个项目开发的主要工作，基本会占到整个项目开发时间的65%，
      </strong>
      硬件、嵌入式、单片机会在这个阶段完成到可试产状态。
     </p>
     <p style="text-align:left;">
      <img alt="c62e5f195c5aedd423df5811a61d9d1b.png" src="https://i-blog.csdnimg.cn/blog_migrate/cbf524a14f5a8106b6afb41a0042d2bb.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （五）NCC认证
      </strong>
      <br/>
     </p>
     <p style="text-align:left;">
      <strong>
       1.目的
      </strong>
     </p>
     <p style="text-align:left;">
      类似于大陆市场的消费类电子产品需要做3C认证，台湾市场的监管要求电子产品需要做NCC认证。
     </p>
     <p style="text-align:left;">
      <img alt="9b4446844e170be25f0d574934c19783.png" src="https://i-blog.csdnimg.cn/blog_migrate/51f6ff1706493c3535b9a8dfc0b811ae.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       对于本产品的认证测试包括：
      </strong>
     </p>
     <p style="text-align:left;">
      A.大于1GHz高频的EMC测试，主要是WiFi模块。
     </p>
     <p style="text-align:left;">
      B.低于1GHz低频的EMC测试，其他普通元器件。
     </p>
     <p style="text-align:left;">
      C.传导发射（Conducted Emission）测试，   通常也会被称为骚扰电压测试，主要是测试连接适配器时对供电系统的影响。
     </p>
     <p style="text-align:left;">
      D.静电测试
     </p>
     <p style="text-align:left;">
      <strong>
       2.高频EMC
      </strong>
     </p>
     <p style="text-align:left;">
      这里的高频是指大于1GHz的电磁辐射测试，设备中主要是WiFi模块会涉及这个部分测试。
     </p>
     <p style="text-align:left;">
      <strong>
       A. WiFi定频
      </strong>
     </p>
     <p style="text-align:left;">
      在开始之前首先要对WiFi模块进行定频。这个部分是由WiFi模组厂进行协助处理，负责对接的是彭工。
     </p>
     <p style="text-align:left;">
      我们将完整设备带到实验室，彭工首先进行
      <strong>
       WiFi功率测定，
      </strong>
      将模块的工作功率设定至合理水平，然后将WiFi工作频率和通道进行限定测试，通过之后会给出相应的操作指令文档。
     </p>
     <p style="text-align:left;">
      <strong>
       B.辐射测试
      </strong>
     </p>
     <p style="text-align:left;">
      这个部分我们找了深圳的一家测试实验室A，对方的测试工程师是个萌妹子，但她强烈要求我称她为
      <strong>
       温大侠
      </strong>
      。
     </p>
     <p style="text-align:left;">
      设备的辐射测试需要将设备
      <strong>
       放到一个巨大的屏蔽房
      </strong>
      ，然后将设备调整到正常运行状态，再测定各频率波段的辐射情况。
     </p>
     <p style="text-align:center;">
      <img alt="d46b5ca709e63dd7ebc9ca51daac826a.png" src="https://i-blog.csdnimg.cn/blog_migrate/c19144d98297d856cd171a5208f77f81.jpeg"/>
     </p>
     <p style="text-align:center;">
      屏蔽房
     </p>
     <p style="text-align:left;">
      由于电路在设计之初就考虑到了过认证测试的情况，所以预留了相应设计用于进行屏蔽修改，对于需要增加屏蔽的部分焊上相应参数的磁珠即可。
     </p>
     <p style="text-align:left;">
      过程不大顺利，虽然焊磁珠解决了部分问题，但是测试几次都仍然有不能通过的部分。而WiFi的定频测试比较顺利，设定几个频率接入测试设备后都通过了。
     </p>
     <p style="text-align:left;">
      <strong>
       3.低频EMC
      </strong>
     </p>
     <p style="text-align:left;">
      在测试实验室A多次修改之后，低频部分都测试不过，需要进行详细的整改。考虑到每个小时高达700软妹币的整改费用，于是我们
      <strong>
       靠友谊的友谊的友谊
      </strong>
      找到了另一家测试
      <strong>
       实验室B
      </strong>
      。
     </p>
     <p style="text-align:center;">
      <img alt="9b0e4373aa18da9cdf7519adb8700fd2.png" src="https://i-blog.csdnimg.cn/blog_migrate/e20f36b08bce4c8e62e45525aaae8181.jpeg"/>
     </p>
     <p style="text-align:center;">
      测试实验室B
     </p>
     <p style="text-align:left;">
      靠脸皮的关系拜托了
      <strong>
       梁工
      </strong>
      加班整改，在尝试了调整摄像头驱动等级，给电路板增加磁珠，给连接线增加磁环等各种方式后，对辐射波形进行读点操作终于符合了认证要求。
     </p>
     <p style="text-align:left;">
      <img alt="ce32c38f485d1a6035c7ed446675c36c.png" src="https://i-blog.csdnimg.cn/blog_migrate/3fefcaab95617e281d7913761af8ef49.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       4.传导发射
      </strong>
     </p>
     <p style="text-align:left;">
      传导发射（Conducted Emission）测试，通常也会被称为骚扰电压测试，主要是测试连接适配器时对供电系统的影响。这部分也是在测试实验室A完成的，相对比较走运测试一次既通过了。
     </p>
     <p style="text-align:left;">
      <strong>
       5.静电测试（ESD）
      </strong>
     </p>
     <p style="text-align:left;">
      Electro-Static discharge，在用户操作设备的过程中，有可能会因为自身的静电而击伤电路元器件，所以还
      <strong>
       需要对设备进行静电测试并作出整改以符合标准。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       6.领证
      </strong>
     </p>
     <p style="text-align:left;">
      在所有的测试都通过之后，实验室A、实验室B都会分别出具相应的
      <strong>
       整改报告
      </strong>
      给到OEM厂，由张工根据整改报告将设备进行整改之后，便可以将设备寄送至台湾NCC认证机构进行测试审核，通过之后便可颁发相应证书。
     </p>
     <p style="text-align:left;">
      <img alt="a5a6fb1cb2b9e9d0c164a3e9656d2cfa.png" src="https://i-blog.csdnimg.cn/blog_migrate/4818210bbe66b137bf0ded8f7c153376.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （六）PVT
      </strong>
     </p>
     <p style="text-align:left;">
      在DVT及NCC通过之后，便可以进行试产。
     </p>
     <p style="text-align:left;">
      Process Verification Test，该阶段的目的：一方面是开始向客户交付测试样机检验功能及稳定性，一方面是开始为批量生产的流程确定标准。
     </p>
     <p style="text-align:left;">
      <strong>
       1.签样
      </strong>
     </p>
     <p style="text-align:left;">
      在NCC整改确认之后，量产之前OEM厂首先会要求在生产之前进行
      <strong>
       电路板的签样
      </strong>
      ，以确定原理设计、layout、BOM符合客户要求。
     </p>
     <p style="text-align:left;">
      客户签字既锁定版本，OEM厂将以签样标准进行后续生产，如果由于市场原因需要更换物料，则要通知客户确认并重新进行测试签样。该部分签样由OEM厂的生产工程师李工负责对接。
     </p>
     <p style="text-align:left;">
      <img alt="e235f8d66c9b48ed4458b3f0ad04be0b.png" src="https://i-blog.csdnimg.cn/blog_migrate/c8b09651101bfd0910b4bcb7ecf54796.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       2.最终设计
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="754e508d9e5f41b81dc95bb9773b9840.png" src="https://i-blog.csdnimg.cn/blog_migrate/bac7146e4a71bcdf1bda1c685dc1300e.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （七）关键性问题
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="6b760d09e2388590d60dc113210baf70.png" src="https://i-blog.csdnimg.cn/blog_migrate/cedd251ab62f3084d4aeb6f4331fe63a.jpeg"/>

     <p style="text-align:center;">
      <strong>
       六、嵌入式系统
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       （一）需求确认
      </strong>
      <br/>
     </p>
     <p style="text-align:left;">
      在确认完硬件完整规格后，我这边还需要设计一份系统功能规格文档，用来给嵌入式开发的
      <strong>
       Danny
      </strong>
      和单片机开发的
      <strong>
       Talan
      </strong>
      ，并且需要开个小会面谈功能以确认功能的理解。
     </p>
     <p style="text-align:left;">

     <p style="text-align:left;">
      <strong>
       （二）原厂SDK验证
      </strong>
      <br/>
     </p>
     <p style="text-align:left;">
      从CPU原厂会拿到三样资料：
      <strong>
       SDK文件、开发资料文档、evb开发板。
      </strong>
     </p>
     <p style="text-align:left;">
      为确保后续开发不引入已知问题，Danny首先在开发板上进行SDK的功能验证，以确保原厂的SDK能正确工作。
     </p>
     <p style="text-align:left;">
      <strong>
       （三）协议及规格定制
      </strong>
     </p>
     <p style="text-align:left;">
      在正式开发之前，与开发确认完系统功能规格后，为减轻开发的工作量，我还需要定制通信的报文协议及表情、提示音内容。
     </p>
     <p style="text-align:left;">
      <strong>
       1.设备与APP交互指令
      </strong>
     </p>
     <p style="text-align:left;">
      该协议用于设备和APP之间进行交互控制，比如查看电量、控制静音等等。
     </p>
     <p style="text-align:left;">
      CloudMeet平台本身支持为各种类型的设备提供服务，此前为方便各种厂商设备接入我已经定制了一个通用的协议表，而机器人由于是新增设备，所以还需要增加一些协议，比如儿童锁、睡眠之类的控制。
     </p>
     <p style="text-align:left;">
      协议内容采用JSON格式，
      <strong>
       支持 HTTP 和 TCP 两种通信类型。
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="8950e65393fb1860bd1fedd7681b3a0a.png" src="https://i-blog.csdnimg.cn/blog_migrate/d6724868c30ff0828fecc3d2ed2933b4.jpeg"/>

       2.单片机与SoC交互协议
      </strong>

     <p style="text-align:left;">
      SoC处理的功能与单片机不同，所以两者之间也需要进行通信交互。
     </p>
     <p style="text-align:left;">
      由于两者走的是
      <strong>
       UART串口
      </strong>
      ，所以需要自己定义协议。这里我们基于16进制定义了报文头和数据段及报文长度。
     </p>
     <p style="text-align:left;">
      <img alt="e1eecfb2c9446a889c2a1db6144fc03f.png" src="https://i-blog.csdnimg.cn/blog_migrate/8520882b5a14cff1992bcc53c18e8da1.jpeg"/>

     <p style="text-align:left;">

       3.表情规格书
      </strong>
     </p>
     <p style="text-align:left;">

     </p>
     <p style="text-align:left;">
      由于表情的IC连接在单片机的引脚上，所以表情的执行都通过单片机，我
      <strong>

      </strong>
     </p>
     <p style="text-align:left;">
      编号：ID 001 ；含义：笑脸；场景：功能规格书指定；方式：逐列扫描

       4.提示音规格书

      <strong>
       （四）嵌入式开发
      </strong>
     </p>
     <p style="text-align:left;">
      嵌入式开发部分主要是基于Embedded Linux做开发，一些需求快速启动的产品则多半基于RTOS系统，前者是分时操作系统，后者是实时操作系统。
     </p>
     <p style="text-align:left;">
      RTOS硬件资源占比小很多，但是开发上限制也很多。Embedded Linux资源占比较大，但是开发难度要低许多，该部分由Danny开发为主。
     </p>
     <p style="text-align:left;">
      在正式开发之前Danny会先
      <strong>
       编写嵌入式功能设计技术文档，用以定义嵌入式的技术内容，以便未来其他人维护及开发，这会比代码写注释还要重要。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       1.联网模块
      </strong>
     </p>
     <p style="text-align:left;">
      该模块主要是解决设备的配网功能，首先通过APP将配网用的WiFi信息生成二维码，然后设备端调用摄像头采集图像并将二维码解码得出对应的WiFi信息。联网程序得到相关WiFi资料后执行联网操作，成功之后再将认证信息提交至CloudMeet系统。
     </p>
     <p style="text-align:left;">
      <strong>
       2.回音消除
      </strong>
     </p>
     <p style="text-align:left;">
      回音消除功能(Acoustic Echo Cancellation,AEC)，作用是避免喇叭播放出来的声音又经过麦克风录制到系统，形成回声。该功能模块需要电路设计配合，即硬回采设计。
     </p>
     <p style="text-align:left;">
      Codec模块将声音采集电路传入的音频提交到系统，系统再通过相应的AEC算法进行回声消除，从而得到单一的原始播放声音。这部分由SoC原厂的Kason配合算法部分开发。
     </p>
     <p style="text-align:left;">
      <strong>
       3.唤醒模块
      </strong>
     </p>
     <p style="text-align:left;">
      唤醒，也叫做“热词”，类似iPhone“喂，Siri”。这个用以设备待机时唤醒设备或者设备执行其他动作时进行打断。
     </p>
     <p style="text-align:left;">
      程序首选会进行VAD监测，发现有说话声音后再进入通过ASR进行热词识别，比如模型设定的是“啊猫啊猫”，你叫“啊狗啊狗”设备就不会鸟你。热词唤醒之后才会提交到云端AI进行处理。
     </p>
     <p style="text-align:left;">
      <strong>
       4.NLP模块
      </strong>
     </p>
     <p style="text-align:left;">
      AI模块主要是自然语言处理，既NLP。在热词唤醒之后将用户输入音频提交至AI云端，进行语音识别，云端再回复相关内容至设备或执行相应动作。
     </p>
     <p style="text-align:left;">
      <strong>
       5.点播模块
      </strong>
     </p>
     <p style="text-align:left;">
      点播模块的作用是用户从APP端H5页面点播内容时，经由CM IoT服务推送至设备端进行播放。
     </p>

      <strong>
       6.播放模块
      </strong>
     </p>
     <p style="text-align:left;">

      </strong>

       （一）CloudMeet

     <p style="text-align:left;">

      <strong>

      </strong>

      <strong>
       所有的模块都是解耦的，而选用不同功能模块则可以组成不同的云端服务能力。
      </strong>

     <p style="text-align:left;">
      <strong>
       （二）AI系统
      </strong>
     </p>
     <p style="text-align:left;">
      一个完整的AI对话系统大致包含四大模块：ASR、STT、NLP、TTS。
     </p>
     <p style="text-align:left;">
      <strong>
       1.ASR（Automatic Speech Recognition）
      </strong>
     </p>
     <p style="text-align:left;">
      语音识别，一般简称ASR，其作用是将声音转换为文字的过程。对于机器人而言，语音识别的主要应用方式是远场语音识别（Farfield Voice Recognition），这里我们选用了苏州的一家语音服务商，这部分由苏州的语音服务厂商刘工配合，该服务包含两个主要模块。
     </p>
     <p style="text-align:left;">
      <strong>
       （1）语音激活检测（VAD）
      </strong>
     </p>
     <p style="text-align:left;">
      Voice Active Detection，主要作用是在麦克风持续工作并输入音频的过程中，检测何时才是发生有效的声音输入，识别并消除长时间的静音期。
     </p>
     <p style="text-align:left;">
      <strong>
       （2）语音唤醒（KWS）
      </strong>
     </p>
     <p style="text-align:left;">
      Keyword Spotting，当输入的声音经过VAD处理后，进行语音识别。该识别会判断是否包含用户输入的语音中是否包含关键字，该关键字可认为是机器人设备的“名字”，例如iPhone的“Siri”、亚马逊Echo的“Alexa”等。如果检测语音中包含该关键字，则将设备唤醒。
     </p>
     <p style="text-align:left;">
      <strong>
       2.STT( Speech to Text)
      </strong>
     </p>
     <p style="text-align:left;">
      语音识别的一种应用类型，将音频转换成文字。这部分我们仍然选用了苏州的服务商。
     </p>
     <p style="text-align:left;">
      <strong>
       3.NLP(Natural Language Processing)
      </strong>
     </p>
     <p style="text-align:left;">
      自然语音处理，通俗的解释就是理解用户到底在说什么。
     </p>
     <p style="text-align:left;">
      <strong>
       用户输入的语音通过STT识别为文字时，系统是无法理解内容阐述的是什么，需要进行语义理解，分析出对话所要表达的内容，然后才能安排下一步的回应动作
      </strong>
      ，比如问答形式回复用户，或者是指令相关控制性指令。
     </p>
     <p style="text-align:left;">
      因为目标是台湾市场，所以我们选择了一家台湾的AI服务商，这部分由台湾的AI服务商
      <strong>
       Nick
      </strong>
      配合。
     </p>
     <p style="text-align:left;">
      <strong>
       4.TTS（Text to Speech）
      </strong>
     </p>
     <p style="text-align:left;">
      在NLP系统理解了用户的对话后，需要作出对话回复，该回复一般是即时文字内容生成，对于设备端而言需要播放的是音频，所以需要预先将回复的内容转换为音频再进行进行播放。
     </p>
     <p style="text-align:left;">
      一开始用的也是跟STT相同的苏州服务商，但是对方不具备台湾腔的语调。最后我们选用了KDXF的TTS服务，以实现台湾腔调的音频。
     </p>

      <strong>
       （三）AIoT系统
      </strong>

     </p>
     <p style="text-align:left;">
      <strong>
       （二）交互要求
      </strong>
     </p>
     <p style="text-align:left;">
      交互部分我一般会有两个基本的设计要求，分别是目标路径、目标成本。
     </p>
     <p style="text-align:left;">
      <strong>
       1.目标路径
      </strong>
     </p>
     <p style="text-align:left;">
      所谓目标路径既用户到达其目标的路径。举个例子，对比微信在iOS和原生Android两个系统下启用微信“位置权限”的设定的典型操作。
     </p>
     <p style="text-align:left;">
      <img alt="d50d12ac762356c24eb02fb1975887ce.png" src="https://i-blog.csdnimg.cn/blog_migrate/da59f1f140050e8933dc17bf7310c450.jpeg"/>
     </p>
     <p style="text-align:left;">
      工程师思维与用户交互思维往往会相左。
      <strong>
       工程师会希望保证工程（功能）的整洁性而倾向对功能模块进行收纳、归类、分组，但是这会导致用户操作的目标路径变深变长。
      </strong>
     </p>
     <p style="text-align:left;">
      而用户永远追求
      <strong>
       “一眼就看到”
      </strong>
      的使用需求，操作路径越短越好，但是这对交互设计而言又会使得功能模块过度扁平化形成层次逻辑混乱的焦虑。
     </p>
     <p style="text-align:left;">
      但就交互设计的目标而言，永远都是尽可能缩短目标路径。
     </p>
     <p style="text-align:left;">
      <strong>
       2.目标成本
      </strong>
     </p>
     <p style="text-align:left;">
      所谓目标成本，是指在用户在目标路径上操作时间成本的数学期望。
     </p>
     <p style="text-align:left;">
      <strong>
       做个假设：
      </strong>
      微信在未来除了提供普通群聊，还提供更高一级的高级群聊，则目标路径深度分别如下：
     </p>
     <p style="text-align:left;">
      <img alt="c8855a1228518a7d3ba879a578884999.png" src="https://i-blog.csdnimg.cn/blog_migrate/52f6ea6b0bfd3105af220ab6ef41ab8a.jpeg"/>
     </p>
     <p style="text-align:left;">
      我们假定微信所有的群创建类型都符合幂律分布，选取普通群:高级群=8:2，微信的每一步操作成本计量值为10，则得出目标成本计算公式：
     </p>
     <p style="text-align:left;">
      <img alt="cbf0fb7ece121769f56121138ee566c7.png" src="https://i-blog.csdnimg.cn/blog_migrate/15e8bf382a83058ed28d12b240b1f5ab.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       由此可见在不同的设计情况下，应用到用户实际场景中会带来不同的目标成本预期。
      </strong>
     </p>
     <p style="text-align:left;">
      所以在设计上，我们希望通过改变用户的交互形式来使得这个成本尽量变小。
     </p>
     <p style="text-align:left;">
      <strong>
       （三）APP配色风格
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       1.暖色调
      </strong>
     </p>
     <p style="text-align:left;">
      由于产品的目标人群是儿童，而APP的目标人群是父母，所以APP的配色风格一开始优先考虑的是暖色调。
     </p>
     <p style="text-align:left;">
      我们参考MIUI的设计情景，为了追求暖色调而大量使用橙色、黄色、红色这些配色，
      <strong>
       用户初期视觉接触感很好，用久了之后却会形成视觉压力从而造成使用者的视觉疲劳。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       2.安全色
      </strong>
     </p>
     <p style="text-align:left;">
      如果花点时间去研究下，我们会发现Facebook、WhatsApp、支付宝、App Store、饿了么、Safari，这些巨型应用的ICON或界面主色调多为蓝色/淡蓝色。而根据调查数据显示，大多数人都喜欢蓝色。在全球范围内来讲，
      <strong>
       蓝色也是最安全的颜色。
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="a7823c5b5842a1839b13233314289d48.png" src="https://i-blog.csdnimg.cn/blog_migrate/1c8369ab27cbebece9d27d163df7eeab.jpeg"/>

     </p>

      <strong>
       （四）视觉稿件

     </p>
     <p style="text-align:left;">
      对于设备的内部缓冲结构，一开始考虑的是珍珠棉，这玩意对于偏大型产品的保护效果好，又比较美观，但是找了好几个厂商后发现价格过高。
     </p>
     <p style="text-align:left;">
      又考虑过换成瓦楞盒，但是在样品测试摔落时表现不好，长得也比较廉价，再三考虑钱包问题，最后选用了吸塑。
     </p>
     <p style="text-align:left;">
      首先要将完整设备提供给材料厂商李工，用以评估产品重量并进行进行抄数。
     </p>
     <p style="text-align:left;">
      <strong>
       抄数的意思是对设备整个外形的尺寸数据进行测量，
      </strong>
      然后根据该参数进行制作对应的吸塑模具。
     </p>
     <p style="text-align:left;">
      <strong>
       2.彩盒
      </strong>
     </p>
     <p style="text-align:left;">
      外包装彩盒的设计比较简单，我这边先跟客户Mic讨论他的想法，然后Jennifer提供相应的设计图形。
     </p>
     <p style="text-align:left;">
      最后整理资料，包括
      <strong>
       图形文件、颜色及四面排版要求
      </strong>
      ，印刷厂的黄小姐即可将元素按照吸塑的外形重新排版设计。
     </p>
     <p style="text-align:left;">
      <strong>
       3.说明书
      </strong>
     </p>
     <p style="text-align:left;">
      说明书稍麻烦点，印刷厂黄小姐明确表示，说明书如果找他们做只包印刷不包设计。所以这边找了一直兼职配合我做文案的
      <strong>
       Jensen
      </strong>
      ，客户Mic先提供相关说明书内容，我这边准备App截图，然后请Jensen设计为PPT然后再交由黄小姐。
     </p>
     <p style="text-align:left;">
      <strong>
       4.组装说明文档
      </strong>
     </p>
     <p style="text-align:left;">
      缓冲结构、彩盒、说明书都确认之后，我这边会出具一份外包装组装说明文档给到工厂，产线将按照该标准进行最后的组装。
     </p>
     <p style="text-align:left;">
      <img alt="ada4d4ae6148b74f37d01bf25d67f185.png" src="https://i-blog.csdnimg.cn/blog_migrate/860d77d0dd12d6167e10ae37844afe13.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （二）外壳
      </strong>
     </p>
     <p style="text-align:left;">
      生产订单下给模具厂，排期大约两周可以交付。由模厂的品质总监李工按照验收标准文档进行质量检查，符合标准之后运送至OEM厂。OEM厂这边由品质工程师何工进行验收，符合标准之后完成验收并安排给产线生产工程师李工。
     </p>
     <p style="text-align:left;">
      <strong>
       （三）SMT
      </strong>
     </p>
     <p style="text-align:left;">
      Layout完成的电路设计会交由洗板工厂洗出印刷电路板（PCB），然后进行元器件贴片（SMT）得到PCBA，再将外围元器件（喇叭、电池等）与外壳进行组装后得到完整产品。
     </p>
     <p style="text-align:left;">
      <strong>
       （四）PCBA产测
      </strong>
     </p>
     <p style="text-align:left;">
      PCB进行贴片后，由于是批量生产，除了工厂自身的通电测试，还需要进行PCBA的批量测试。
     </p>
     <p style="text-align:left;">
      这个部分由于涉及到硬件端口的读写控制，所以由Talan与OEM厂的张工进行讨论后，编写相应测试程序并提供操作文档给到OEM厂的
      <strong>
       黄工
      </strong>
      ，黄工的责任为生产测试。
     </p>
     <p style="text-align:left;">
      <img alt="d9abecda8754439050c2e96385a985c2.png" src="https://i-blog.csdnimg.cn/blog_migrate/d4432e6a3452d5f461f22edc6cb69320.jpeg"/>
     </p>
     <p style="text-align:left;">
      黄工会在PCBA组装好之后单纯将板子联通到一起，然后通过测试程序自动逐一测试对应功能，如果有测试不通过则给出错误提示。
      <br/>
     </p>
     <p style="text-align:left;">
      <strong>
       （五）装配
      </strong>
     </p>
     <p style="text-align:left;">
      在PCBA完成测试之后，硬件交由组装产线进行
      <strong>
       PCBA+辅料+外壳
      </strong>
      的整体装配工序。
     </p>
     <p style="text-align:left;">
      这阶段工厂会由生产工程师制作并
      <strong>
       打印装配指导手册
      </strong>
      ，告知产线工人按照什么标准进行产品的组装。
     </p>
     <p style="text-align:left;">
      <img alt="b7e2a9376c1584a6452c4770591f0157.png" src="https://i-blog.csdnimg.cn/blog_migrate/c1100ef3e5401c1950b21e2085045a85.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （六）整机测试
      </strong>
     </p>
     <p style="text-align:left;">
      设备组装完毕，进入到整机批量测试工序。由Danny与OEM厂的张工进行讨论后，将系统应用功能做成一个自动化执行的程序，并提供相应的操作文档，由OEM厂的黄工安排工人按照自动测试流程对整机进行测试。
     </p>
     <p style="text-align:left;">
      同时测试通过之后，进入设备信息烧录，Danny提供烧录程序让产线工人将设备的SN号自动烧录到设备中，完成生产工作。
     </p>
     <p style="text-align:left;">
      <strong>
       （七）成品质检
      </strong>
     </p>
     <p style="text-align:left;">
      在成品按照相应标准组装完成之后，由OEM厂何工先安排工人先进行内部抽检，通过之后再交由客户（我）进行检查，PVT首次试产都要进行全检，后续生产则进行抽样检查，不良率一般可接受标准为 &lt;3%。
     </p>
     <p style="text-align:left;">
      <img alt="9cac9c0fc67edd31792e5fba6d4b3b04.png" src="https://i-blog.csdnimg.cn/blog_migrate/2a84a728fe3cdedcfa0759025b108e63.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （八）关键性问题
      </strong>
     </p>
     <p style="text-align:left;">
      <img alt="6b30828fe6076e7c3ffbee2eafcde78d.png" src="https://i-blog.csdnimg.cn/blog_migrate/8e6a3a8c32c01b21d104b1f02e13990f.jpeg"/>
     </p>
     <p style="text-align:left;">
      <img alt="c2d4b858dae0a3f4ec4e540f16fce8c1.png" src="https://i-blog.csdnimg.cn/blog_migrate/258efe57c10a083e60f36f53d043c595.png"/>
     </p>
     <p style="text-align:center;">
      <strong>
       十一、项目成果
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       （一）项目协调
      </strong>
      <br/>
     </p>
     <p style="text-align:left;">
      由于整个项目涉及第三方合作商非常多，并且分隔各地，各参与角色所负责的
      <strong>
       需求确认、各方之间的技术对接、资源协调、跨公司的项目排期
      </strong>
      等等都是些挺折腾的事情。
     </p>
     <p style="text-align:left;">
      <img alt="afb7c7c215e518b1eee846a052d7949f.png" src="https://i-blog.csdnimg.cn/blog_migrate/c9d382ca7417a32dafd1d7355843d2ee.jpeg"/>
     </p>
     <p style="text-align:left;">
      <strong>
       （二）关于开发模式
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       1.瀑布流式和敏捷式
      </strong>
     </p>
     <p style="text-align:left;">
      瀑布流式开发是传统的开发模式，强调文档和流程，相对缺少迭代与反馈，不适合客户需求不断变化的开发场景，但是好处是从一开始终点目标就清晰明确。
     </p>
     <p style="text-align:left;">
      敏捷式开发强调“快”，要求快速迭代以获取频繁的反馈，适合应对市场和客户需求不断变化的互联网场景，缺点是很容易欠下技术债。
     </p>
     <p style="text-align:left;">
      举个例子，假如要造一台汽车。瀑布流式开发会花上好几个月来确认每一个汽车零件，然后再动手；但是敏捷模式则会拆分阶段，每个阶段都会先做一个“可被测试”的东西。
     </p>
     <p style="text-align:left;">
      如果这个不管用，那就再做一个，逐步逼近一台真正的车子。
     </p>
     <p style="text-align:left;">
      <img alt="e4198a41d33e2f89fc01bc648f8f1b99.png" src="https://i-blog.csdnimg.cn/blog_migrate/bb22a26d8d61f13704b75b28508e5dca.jpeg"/>
      <br/>
     </p>
     <p style="text-align:left;">
      <strong>
       2.模式选择
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      在APP应用类型的开发中非常适合敏捷式开发，因为市场的需求很多时候是未知的，
      <strong>
       产品经理拍脑袋想出来的great idea往往可能是个坑。
      </strong>
     </p>
     <p style="text-align:left;">
      而敏捷开发的好处是快速迭代，并且短时间内即可验证想法。否则一个功能模块的上新需要等待两个月之后才能知道市场反馈，整个团队的试错成本极高。
     </p>
     <p style="text-align:left;">
      <strong>
       但在智能硬件领域，我个人觉得还是比较适合瀑布流模式。
      </strong>
     </p>
     <p style="text-align:left;">
      <strong>
       首先电路设计的更改成本极高，
      </strong>
      随便更换一个设计就要重新画原理图、ayout、洗板、贴片、测试，一个周期下来差不多一个月时间就过去了。
     </p>
     <p style="text-align:left;">
      <strong>
       其次产品系统模块之间的联动性极强，
      </strong>
      比如APP端如果要调整控制报文的发送模式，那就会导致云端要同步更改、测试，嵌入式端要同步更改、测试。
     </p>
     <p style="text-align:left;">
      <strong>
       再者是资源协调和工程排期问题，
      </strong>
      除非是大型的公司，否则小公司研发硬件多半需要各种不同的公司一起配合研发。而任何要变更一个需求，都需要重新协调对应的公司工程师资源及对方的排期安排。
     </p>
     <p style="text-align:left;">
      这一系列的联动性都会使得项目在开发过程中不合适过多的调整变更设计。
     </p>
     <p style="text-align:left;">
      <strong>
       （三）项目交付
      </strong>
     </p>
     <p style="text-align:left;">
      项目的工期卡得相当紧，客户期望是6个月可以出货，在现有的经验和资源下几乎是个
      <strong>
       Impossible Mission。
      </strong>
     </p>
     <p style="text-align:left;">
      最后我们花了8个月时间实现
      <strong>
       Case Close，
      </strong>
      虽然比客户设定的时间晚了两个月，但总算是顺利交付客户。
     </p>
