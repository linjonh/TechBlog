    <p>
     <strong>
      本篇文章介绍使用iterm2，通过跳板机连接服务器的方法。其中iterm2默认已经安装并配置完成
     </strong>
     。
    </p>
    <p>
     方法一：
    </p>
    <pre><code class="language-bash">cd ~/.ssh/
vim config

</code></pre>
    <p>
     <u>
      编辑以下内容到config
     </u>
     .其中Host为自定义的名字；Hostname为IP地址；Port为端口号；User为用户名。具体例子如下所示
    </p>
    <pre><code>Host test_name
Hostname 127.0.0.1
Port 22
User root</code></pre>
    <p>
     <u>
      执行以下命令后输入密码即可登录服务器
     </u>
    </p>
    <pre><code class="language-bash">ssh test_name</code></pre>
    <p>
     方法二：
    </p>
    <pre><code class="language-bash">cd ~/.ssh
vim test
</code></pre>
    <p>
     <u>
      编辑以下以下内容到test
     </u>
    </p>
    <pre><code>#!/usr/bin/expect -f
set user test
set host 127.0.0.1
set port 22
set password ******

spawn ssh -p $port $user@$host
expect "$user@$host's password:"
send "$password\r"
interact
</code></pre>
    <p style="margin-left:0cm;">
     1. set 即创建变量的命令。
    </p>
    <p style="margin-left:0cm;">
     2. spawn命令是Expect的初始命令，它用于启动一个进程，之后所有expect操作都在这个进程中进行，如果没有spawn语句，整个expect就无法再进行下去
    </p>
    <p style="margin-left:0cm;">
     3. $xx，即调用变量。
    </p>
    <p style="margin-left:0cm;">
     4. expect 是一个命令，表示当获得返回的字符串和 $user@$ip's password:相同时，就执行下面的命令。expect后面的语句根据实际情况有所变更，否则可能会出现item2闪退的情况
    </p>
    <p style="margin-left:0cm;">
     5. send 表示模拟用户手动输入的。
    </p>
    <p style="margin-left:0cm;">
     6. \r 表示回车。
    </p>
    <p style="margin-left:0cm;">
     7. interact 表示控制权交还给控制台即用户输入了。
    </p>
    <p style="margin-left:0cm;">
     <u>
      iTerm2 &gt;&gt; Perference &gt;&gt; Profiles  如下图箭头位置进行设置
     </u>
    </p>
