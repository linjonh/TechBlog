    <h2>
     <a id="_1">
     </a>
     业务背景
    </h2>
    <p>
     提高售后服务，提供上门取件服务
    </p>
    <h2>
     <a id="_3">
     </a>
     测试计划
    </h2>
    <p>
     开发2周、测试一周
    </p>
    <h2>
     <a id="_5">
     </a>
     测试准备
    </h2>
    <ul>
     <li>
      功能冒烟场景（需要第三方配合测试）
     </li>
     <li>
      降低沟通成本（每天日报、总结、每天安排、需要配合的内容）
     </li>
    </ul>
    <h2>
     <a id="_8">
     </a>
     测试关注点
    </h2>
    <ul>
     <li>
      取件地址是否可取（取件范围）
     </li>
     <li>
      取件超时如何处理（了解第三方处理环节并结合自身业务）
     </li>
     <li>
      快递小哥的巴枪有什么状态（文档还是文档，懂得都懂）
     </li>
    </ul>
    <h2>
     <a id="_12">
     </a>
     测试总结
    </h2>
    <h3>
     <a id="_13">
     </a>
     减少第三方处理问题所花费的时间
    </h3>
    <p>
     因为需要第三方的测试人员配合，才能保证自己原有的业务正常运作
    </p>
    <ol>
     <li>
      需要使用在线文档，规划好，哪些需要第三方的人员操作，是用Excel明确安排好，比人需要做什么
      <br/>
      比如：需要第三方快递小哥接单，我们需要预先准备好，单号XXX,取件码xxxx，处理结果是什么
     </li>
     <li>
      建立主要的沟通群，wx,feishu等，规定好用哪一个就好了
     </li>
     <li>
      如果公司有pmo的话，当有重要业务需要变更或同步的，统一收集和视频会议并邮件抄送关键人物
     </li>
     <li>
      如果是第一次接触快递业务的话，一开始的时候大概用1~2天熟悉对方业务的专业术语（巴枪、入库、清单、客户拒收、揽件、原路退回、空包等），如果遇到不会表述的，就简单说明一下你要测什么场景就可以了，别人一般都懂
     </li>
     <li>
      每天下班前最后问一下每天，第三方的对接人是谁，因为每天都是不同人对接，做好最坏的打算
     </li>
     <li>
      发布和上线前必须再三确认对接人，对接时间，服务上线时间，不然别人会一早下班啦!
     </li>
    </ol>
