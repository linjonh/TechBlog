    <h2 id="cc开源库推荐">
     C/C++开源库推荐
    </h2>
    <pre><code>  主要都是平常用到的，包含windows开发、android开发、后台开发、音视频开发、深度学习等等

  最后还附带几个其他语言开发的比较好的项目
  
  持续更新中...</code></pre>
    <h4 id="gui开发">
     GUI开发
    </h4>
    <ul>
     <li>
      <a href="https://github.com/qt/qt5" title="qt">
       qt
      </a>
      跨平台开发库，内部封装了各种跨平台工具，但是大多数情况下都被用作开发跨平台客户端
     </li>
     <li>
      <a href="https://github.com/ocornut/imgui" title="imgui">
       imgui
      </a>
      基于opengl/dx编写的GUI库,挺简单也挺好用的
     </li>
     <li>
      <a href="https://github.com/electron/electron" title="electron">
       electron
      </a>
      C++/NodeJS开发的跨平台GUI库，感觉其实就是简化版的chrome并做了某些功能的增强，分为main/render两部分，ui布局全是html/js/ts/vue/jsx，轻轻松松就可以开发出一个很漂亮的界面，毕竟前端模板太多了，可以配套用nodejs调用C++那一套，回调回C++混合开发,对执行效率要求不高的可以用
     </li>
     <li>
      <a href="https://github.com/libsdl-org/SDL" title="sdl">
       sdl
      </a>
      跨平台GUI开发库,scrcpy用的就是这个
     </li>
     <li>
      <a href="https://github.com/SFML/SFML" title="SFML">
       SFML
      </a>
      跨平台的GUI库，以前开发游戏时写了一个demo，用了一次
     </li>
     <li>
      <a href="https://github.com/duilib/duilib" title="duilib">
       duilib
      </a>
      美化MFC项目
     </li>
     <li>
      <a href="https://github.com/google/skia" title="skia">
       skia
      </a>
      google开源的gui库，android/chrome的UI都是用这个画的
     </li>
     <li>
      <a href="https://github.com/wxWidgets/wxWidgets" title="wxwidgets">
       wxwidgets
      </a>
      audacity好像用的就是这个gui库
     </li>
     <li>
      <a href="https://github.com/AshampooSystems/boden" title="boden">
       boden
      </a>
      android/ios gui库，纯C++开发
     </li>
     <li>
      <a href="https://github.com/cycfi/elements" title="elements">
       elements
      </a>
      跨平台ui库
     </li>
    </ul>
    <h4 id="远程控制">
     远程控制
    </h4>
    <ul>
     <li>
      <a href="https://github.com/FreeRDP/FreeRDP" title="freerdp">
       freerdp
      </a>
      rdp远程控制，在windows上还是mstsc最好用，linux/mac可以考虑freerdp
     </li>
     <li>
      <a href="https://github.com/aspia/aspia" title="aspia">
       aspia
      </a>
      p2p远程工具，研究过其中代码挺不错的
     </li>
     <li>
      <a href="https://github.com/Genymobile/scrcpy" title="scrcpy">
       scrcpy
      </a>
      PC控制android手机，需要打开adb
     </li>
     <li>
      <a href="https://github.com/symless/synergy-core" title="synergy">
       synergy
      </a>
      效率工具，用一台机器共享鼠标键盘控制多台其他计算机，gui配置要激活，我一般直接用控制台启动，具体配置及启动方式请移步
      <a href="https://blog.csdn.net/CHNIM/article/details/125718255?csdn_share_tail=%7B%22type%22%3A%22blog%22%2C%22rType%22%3A%22article%22%2C%22rId%22%3A%22125718255%22%2C%22source%22%3A%22CHNIM%22%7D&amp;ctrtid=9tgLN" title="synergy控制台启动">
       synergy控制台启动
      </a>
     </li>
     <li>
      <a href="https://github.com/barry-ran/QtScrcpy" title="QtScrcpy">
       QtScrcpy
      </a>
      将synergy的gui用QT重写了并做了增强
     </li>
     <li>
      <a href="https://github.com/TigerVNC/tigervnc" title="tigervnc">
       tigervnc
      </a>
      开源的vnc客户端与服务端
     </li>
     <li>
      <a href="https://github.com/mpromonet/webrtc-streamer" title="webrtc-streamer">
       webrtc-streamer
      </a>
      严格来说这不是远程工具，不能控制，是一个C/C++写的基于webrtc的投屏工具，类似于deskreen,服务器启动后，在其他机器上打开浏览器输入网址，就可以选择需要显示的窗口了
     </li>
    </ul>
    <h4 id="音视频开发">
     音视频开发
    </h4>
    <ul>
     <li>
      <a href="https://github.com/ffmpeg/ffmpeg" title="ffmpeg">
       ffmpeg
      </a>
      从事音视频开发必备库，avformat混流分流何种协议封装，avcodec音视频编辑器封装,ffplay.exe 是一个万能播放器，ffmpeg.exe 是个超级音视频处理工具，格式工厂就是把这个工具封装了一个界面，ffprobe码流分析工具
     </li>
     <li>
      <a href="https://github.com/cisco/openh264" title="openh264">
       openh264
      </a>
      思科开源的h264编解码库，号称最快，没有具体测试过
     </li>
     <li>
      <a href="https://github.com/mirror/x264" title="x264">
       x264
      </a>
      h264编解码库
     </li>
     <li>
      <a href="https://github.com/webmproject/libvpx" title="libvpx">
       libvpx
      </a>
      vp8/vp9编解码库
     </li>
     <li>
      <a href="https://github.com/videolan/vlc" title="vlc">
       vlc
      </a>
      videolan的视频播放器，可提供api接口调用很方便
     </li>
     <li>
      <a href="https://github.com/wang-bin/QtAV" title="QtAV">
       QtAV
      </a>
      将ffmpeg封装成一个Qt播放器组件，在qt中用很方便
     </li>
     <li>
      <a href="https://github.com/Samsung/rlottie" title="rlottie">
       rlottie
      </a>
      lottie动画播放库，以前写QT项目时用到了，不过qt现在自己也有lottie播放组件了
     </li>
    </ul>
    <h4 id="音视频会议">
     音视频会议
    </h4>
    <ul>
     <li>
      <a href="https://github.com/holylong/webrtc" title="webrtc">
       webrtc
      </a>
      google开源的为浏览器提供实时通话的C++库，在浏览器上提供了一套接口，很方便的实现p2p视频通话，最近几年因为疫情这个库被大量用于音视频会议系统
     </li>
     <li>
      <a href="https://github.com/meetecho/janus-gateway" title="janus">
       janus
      </a>
      纯C开发的音视频会议服务器，很多公司的服务器都是以这个为基础版本改出来的
     </li>
     <li>
      <a href="https://github.com/lynckia/licode" title="licode">
       licode
      </a>
      C++/js开发的音视频会议服务器，同样被很多公司拿出来用
     </li>
     <li>
      <a href="https://github.com/versatica/mediasoup" title="mediasoup">
       mediasoup
      </a>
      基于webrtc的 sfu服务器，以库的形式开放出来，可以嵌入自己的项目中
     </li>
     <li>
      <a href="https://github.com/medooze/media-server" title="media-server">
       media-server
      </a>
      基于webrtc的服务器
     </li>
     <li>
      <a href="https://github.com/signalwire/freeswitch" title="freeswitch">
       freeswitch
      </a>
      voip视频会议服务器
     </li>
     <li>
      <a href="https://github.com/kamailio/kamailio" title="kamailio">
       kamailio
      </a>
      sip注册服务器，编译安装配置好以后，下载个linphone就可以局域网视频通话，配置个rtpengine也可以转发rtp视频流
     </li>
     <li>
      <a href="https://www.antisip.com/download/exosip2/libosip2-5.3.0.tar.gz" rel="nofollow" title="osip">
       osip
      </a>
      sip信令封装库
     </li>
     <li>
      <a href="http://savannah.nongnu.org/projects/exosip/" rel="nofollow" title="exosip">
       exosip
      </a>
      osip的C++封装
     </li>
     <li>
      <a href="https://www.pjsip.org/" rel="nofollow" title="pjsip">
       pjsip
      </a>
      sip信令封装库
     </li>
     <li>
      <a href="https://github.com/j0r1/JRTPLIB" title="jrtplib">
       jrtplib
      </a>
      rtp封装库，对接国标GB28181用过
     </li>
     <li>
      <a href="https://github.com/BelledonneCommunications/ortp" title="ortp">
       ortp
      </a>
      linphone用到的rtp封装库
     </li>
    </ul>
    <h4 id="压缩库">
     压缩库
    </h4>
    <ul>
     <li>
      <a href="https://www.7-zip.org/" rel="nofollow" title="p7zip">
       p7zip
      </a>
      俄罗斯同志开发的压缩解压算法库，并提供安装程序，很出名养了一大批同类软件
     </li>
     <li>
      <a href="https://github.com/nih-at/libzip" title="libzip">
       libzip
      </a>
      传统的zip压缩解压缩库
     </li>
    </ul>
    <h4 id="计算机视觉深度学习">
     计算机视觉/深度学习
    </h4>
    <ul>
     <li>
      <a href="https://github.com/opencv/opencv" title="opencv">
       opencv
      </a>
      intel开源的图像处理库，里边包含了何种图像处理的算法，简单的人脸识别，抠图，图像拼接，边缘检测等等，还有dnn模块可以加载其他深度学习框架训练的模型进行推理，内部封装了ffmpeg,libjpg,libpng，支持图像io,视频io，很强大的C++计算机视觉库
     </li>
     <li>
      <a href="https://github.com/davisking/dlib" title="dlib">
       dlib
      </a>
      通用工具库，包含人脸识别/检测 json解析，网络,GUI等功能，里边包含很多功能
     </li>
     <li>
      <a href="https://github.com/PaddlePaddle/Paddle" title="PaddlePaddle">
       PaddlePaddle
      </a>
      百度开源的深度学习框架，没有用这个训练过东西
     </li>
     <li>
      <a href="https://github.com/pytorch/pytorch" title="pytorch">
       pytorch
      </a>
      facebook开源的深度学习框架，目前学术界最流行的
     </li>
     <li>
      <a href="https://github.com/tensorflow/tensorflow" title="tensorflow">
       tensorflow
      </a>
      google开源的深度学习框架
     </li>
     <li>
      <a href="https://github.com/AlexeyAB/darknet" title="darknet">
       darknet
      </a>
      目标检测方面的神，可惜原作者已放弃这方面的研究,现在已经有pytorch实现的yolov5,yolov7了，可以用更少的数据训练出不错的模型
     </li>
     <li>
      <a href="https://github.com/apache/incubator-mxnet" title="mxnet">
       mxnet
      </a>
      亚马逊在用的深度学习框架
     </li>
     <li>
      <a href="https://github.com/microsoft/onnxruntime" title="onnxruntime">
       onnxruntime
      </a>
      微软开源的深度学习推理框架，配合onnx使用
     </li>
     <li>
      <a href="https://github.com/BVLC/caffe" title="caffe">
       caffe
      </a>
      贾扬清开发的深度学习框架，以前很多人都用这个，后来tensorflow出来后，大家纷纷开始转python用tensorflow
     </li>
     <li>
      <a href="https://github.com/google/mediapipe" title="mediapipe">
       mediapipe
      </a>
      google开源的推理框架，谁用谁知道，确实是快
     </li>
     <li>
      <a href="https://github.com/Tencent/ncnn" title="ncnn">
       ncnn
      </a>
      腾讯开源的推理框架，移动端很快
     </li>
     <li>
      <a href="https://github.com/alibaba/MNN" title="mnn">
       mnn
      </a>
      阿里开源的推理框架
     </li>
     <li>
      <a href="https://github.com/XiaoMi/mace" title="mace">
       mace
      </a>
      小米开源的模型推理框架
     </li>
     <li>
      <a href="https://github.com/PaddlePaddle/Paddle-Lite" title="Paddle-Lite">
       Paddle-Lite
      </a>
      百度开源的推理框架
     </li>
     <li>
      <a href="https://github.com/DefTruth/lite.ai.toolkit" title="lite.ai.toolkit">
       lite.ai.toolkit
      </a>
      把一些推理框架做了封装的工具库
     </li>
     <li>
      <a href="https://github.com/ShiqiYu/libfacedetection" title="libfacedetection">
       libfacedetection
      </a>
      yushiqi开源的人脸检测工具
     </li>
     <li>
      <a href="https://github.com/seetafaceengine/SeetaFace2" title="SeetaFace">
       SeetaFace
      </a>
      好像是中科院某位教授开源的项目，检测识别效果不错，拿来即用的项目，当时用了里边的人脸检测确实是最快的
     </li>
    </ul>
    <h4 id="语音识别深度学习">

    <h4 id="服务器">

     </li>

     </li>
     <li>
      <a href="https://github.com/kbengine/kbengine" title="kbengine">
       kbengine

     </li>
     <li>

     </li>
     <li>
      <a href="https://github.com/teamgram/teamgram-tdesktop" title="teamgram-tdesktop">
       teamgram-tdesktop
      </a>
      开源teamgram-server开的tdesktop分支
