    <h3>
     小程序直播组件接入指引
    </h3>
    <h4>
     一、简介
    </h4>
    <p>
     小程序直播，是微信提供给小程序开发者的直播组件。通过调用该组件，商家可以在小程序中实现直播功能。
    </p>
    <p>
     按下面的使用说明接入，在你的小程序中引入直播组件。
    </p>
    <h4>
     二、使用方法说明
    </h4>
    <p>
     1.【直播组件】如何引入
    </p>
    <p>
     版本限制：微信客户端版本 7.0.7 及以上（基础库版本 2.9.x 及以上支持同层渲染）可以观看直播及使用直播间的功能，低版本刚进入直播间时会提示用户升级微信客户端版本（低版本只能观看直播，无法使用直播间的功能）。
    </p>
    <p>
     在分包内引入【直播组件】live-player-plugin 代码包，项目根目录的 app.json 引用，示例代码如下：
    </p>
    <pre>{   "subpackages": [     {       "root": "packageA",       "pages": [         "pages/home/home"       ],       "plugins": {         "live-player-plugin": {           "version": "1.0.0", // 填写该直播组件最新版本号，微信开发者工具调试时可获取最新版本号           "provider": "wx2b03c6e691cd7370" // 必须填该直播组件appid，该示例值即为直播组件appid         }       }     }   ] }</pre>
    <p>
     2.【直播组件】如何使用
    </p>
    <p>
     按第1步的方法把组件代码包配置引入后，即可直接通过 链接地址跳转到直播组件页面（即为进直播间页面） 链接地址需要带上直播房间id；房间id可通过下面 【获取直播房间列表】
    </p>

     API获取，示例代码如下：

    <p>
     通过该链接可跳转到直播组件页面（当前页面入口仅开放‘live-player-plugin’）。

    <p>
     1.【订阅】组件
    </p>
    <p>
     功能解释：用户进入直播间内，可对一场未开播的直播进行单次订阅，开播时直播组件会自动下发开播提醒给用户， 无需开发者额外开发
    </p>
    <p>
     组件使用：如果需要 在直播组件页以外小程序其他页面也有同样的开播提醒的功能，可以引入【订阅】组件subscribe；需在page页面（如home页面）的 home.json 引用订阅组件，示例代码如下：
    </p>
    <pre>{   "usingComponents": {     "subscribe": "plugin-private://wx2b03c6e691cd7370/components/subscribe/subscribe"   } }</pre>
    <p>
     然后便可在home.wxml里使用订阅组件，其中直播房间id可通过；房间id可通过下面【获取直播房间列表】API获取
    </p>
    <pre> </pre>
    <p>
     2. 获取直播状态接口
    </p>
    <p>
     接口说明：首次获取立马返回直播状态，往后间隔1分钟或更慢的频率去轮询获取直播状态
    </p>
    <p>
     直播状态说明：
    </p>
    <ul>
     <li>
      <p>
       101直播中：表示主播正常开播，直播正常的状态
      </p>
     </li>
     <li>
      <p>
       102未开始：表示主播还未开播
      </p>
     </li>
     <li>
      <p>
       103已结束：表示在直播端点击【结束】按钮正常关闭的直播，或直播异常15分钟后系统强制结束的直播
      </p>
     </li>
     <li>
      <p>
       104禁播：表示因违规受到运营处罚被禁播
      </p>
     </li>
     <li>
      <p>
       105暂停中：表示在MP小程序后台-控制台内操作暂停了直播
      </p>
     </li>
     <li>
      <p>
       106异常：表示主播离开、切后台、断网等情况，该直播被判定为异常状态，15分钟内恢复即可回到正常直播中的状态；如果15分钟后还未恢复，直播间会被系统强制结束直播
      </p>
     </li>
     <li>
      <p>
       107已过期：表示直播间一直未开播，且已达到在MP小程序后台创建直播间时填写的直播计划结束时间，则该直播被判定为过期不能再开播
      </p>
     </li>
    </ul>
    <p>
     调用方法：若要调用【获取直播状态】接口getLiveStatus，需在小程序页面顶部引用【直播组件】live-player-plugin，示例代码如下：
    </p>
    <pre>let livePlayer = requirePlugin('live-player-plugin') // 引入获取直播状态接口 // 首次获取立马返回直播状态，往后间隔1分钟或更慢的频率去轮询获取直播状态 const roomId = xxx // 房间id livePlayer.getLiveStatus({ room_id: roomId }) .then(res =&gt; {   // 101: 直播中, 102: 未开始, 103: 已结束, 104: 禁播, 105: 暂停中, 106: 异常, 107：已过期    const liveStatus = res.liveStatus }) .catch(err =&gt; {   console.log(err) })</pre>
    <p>
     3. 携带参数
    </p>
    <p>
     版本限制：直播组件版本1.0.1及以上支持携带以下参数
    </p>
    <p id="b4">
     1) shareTicket：分享直播间卡片到微信群，点击此卡片后可以在 App onShow 里获取该参数
    </p>
    <p id="b3">
     2) room_id：点击直播组件页面里的货架商品跳转到商家小程序的商品详情页面时，会带上房间号参数
    </p>
    <p>
     4.【获取直播房间列表】接口，仅供后台调用
    </p>
    <p>
     接口规则：该接口仅供商家后台调用，调用限额500次/天，建议开发者自己做缓存（此接口与下面
     <a href="https://mp.weixin.qq.com/wxamp/broadcast/doc?type=1&amp;token=612941116&amp;lang=zh_CN&amp;st=598#b6" rel="nofollow">
      【获取回放视频】
     </a>
     接口共用500次/天限制，请合理分配调用频次）。
    </p>
    <p>
     请求URL：
     <a href="http://api.weixin.qq.com/wxa/business/getliveinfo?access_token=" rel="nofollow">
      http://api.weixin.qq.com/wxa/business/getliveinfo?access_token=
     </a>
    </p>
    <p>
     请求方式：POST
    </p>
    <p>
     请求示例：
    </p>
    <p>
     Request
    </p>
    <pre>{  "start": 0, // 起始拉取房间，start=0表示从第1个房间开始拉取  "limit": 10 // 每次拉取的个数上限，不要设置过大，建议100以内 }</pre>
    <p>
     Response
    </p>
    <pre>{  "errcode": 0, // errcode=0代表成功；errcode=1代表未创建直播房间  "errmsg": "ok",  "room_info": [{   "name": "直播房间名",   "roomid": 1,   "cover_img": "http:\/\/mmbiz.qpic.cn\/mmbiz_jpg\/Rl1RuuhdstSfZa8EEljedAYcbtX3Ejpdl2et1tPAQ37bdicnxoVialDLCKKDcPBy8Iic0kCiaiaalXg3EbpNKoicrweQ\/0?wx_fmt=jpeg",   "live_satus": 101,    "start_time": 1568128900,   "end_time": 1568131200,   "anchor_name": "李四",   "anchor_img": "http:\/\/mmbiz.qpic.cn\/mmbiz_jpg\/Rl1RuuhdstSfZa8EEljedAYcbtX3Ejpdlp0sf9YTorOzUbGF9Eib6ic54k9fX0xreAIt35HCeiakO04yCwymoKTjw\/0?wx_fmt=jpeg",   "goods":[               {       "cover_img":"http://mmbiz.qpic.cn/mmbiz_png/FVribAGdErI2PmyST9ZM0JLbNM48I7TH2FlrwYOlnYqGaej8qKubG1EvK0QIkkwqvicrYTzVtjKmSZSeY5ianc3mw/0?wx_fmt=png",       "url":"pages/index/index.html",       "price":1100,       "name":"fdgfgf"     }    ]  }</pre>
    <p>
     返回字段：
    </p>
    <pre> </pre>
    <ul>
     <li>
      <p>
       name 房间名
      </p>
     </li>
     <li>
      <p>
       roomid 房间id
       <br/>
       注：需先在小程序MP后台创建直播房间，否则会报错（错误码1）
      </p>
     </li>
     <li>
      <p>
       cover_img 封面图片url
      </p>
     </li>
     <li>
      <p>
       start_time 直播计划开始时间，列表按照 start_time 降序排列
      </p>
     </li>
     <li>
      <p>
       end_time 直播计划结束时间
      </p>
     </li>
     <li>
      <p>
       anchor_name 主播名
      </p>
     </li>
     <li>
      <p>
       goods 商品列表
      </p>
     </li>
     <li>
      <p>
       live_status 直播状态   101: 直播中, 102: 未开始, 103: 已结束, 104: 禁播, 105: 暂停中, 106: 异常，107:已过期（直播状态解释可参考
       <a href="https://mp.weixin.qq.com/wxamp/broadcast/doc?type=1&amp;token=612941116&amp;lang=zh_CN&amp;st=598#b2" rel="nofollow">
        【获取直播状态】
       </a>
       接口）
      </p>
     </li>
    </ul>
    <pre> </pre>
    <p>
     5.【获取回放源视频】接口，仅供后台调用
    </p>
    <p>
     接口规则：该接口仅供商家后台调用， 调用限额500次/天，此接口与上面
     <a href="https://mp.weixin.qq.com/wxamp/broadcast/doc?type=1&amp;token=612941116&amp;lang=zh_CN&amp;st=598#b5" rel="nofollow">
      【获取房间列表】
     </a>
     接口 共用500次/天限制，请合理分配调用频次）。
    </p>
    <p>
     接口说明：
    </p>
    <ul>
     <li>
      <p>
       该接口可在直播结束后拿到回放源视频（直播结束后大约10分钟会生成回放，源视频无评论等内容）
      </p>
     </li>
     <li>
      <p>
       拿到源视频后需要开发者自行开发、使用回放视频
      </p>
     </li>
     <li>
      <p>
       如果把源视频直接放在小程序上使用，需要小程序具备视频资质（具体资质要求请参考《小程序开发的类目服务》）
      </p>
     </li>
    </ul>
    <p>
     后续官方会直接提供无需开发的直播回放功能，敬请期待。
    </p>
    <p>
     请求URL：
     <a href="http://api.weixin.qq.com/wxa/business/getliveinfo?access_token=" rel="nofollow">
      http://api.weixin.qq.com/wxa/business/getliveinfo?access_token=
     </a>
    </p>
    <p>
     请求方式：POST
    </p>
    <p>
     请求示例：
    </p>
    <p>
     Request
    </p>
    <pre>{   "action": "get_replay", // 获取回放   "room_id": 354, // 直播间id   "start": 0, // 起始拉取视频，start=0表示从第1个视频片段开始拉取   "limit": 10 // 每次拉取的个数上限，不要设置过大，建议100以内 }</pre>
    <p>
     Response
    </p>
    <pre>{   "live_replay": [     {       "expire_time": "2020-11-11T03:49:55Z", // 回放视频 url 过期时间       "create_time": "2019-11-12T03:49:55Z", // 回放时间创建时间       "media_url": "http://xxxxx.vod2.myqcloud.com/xxxxx/xxxxx/f0.mp4" // 回放视频     }   ],   "errcode": 0,   "total": 1,   "errmsg": "ok" } // 一场直播可能会产生多个视频片段。</pre>
    <h4>
     四、其他说明
    </h4>
    <p>
     1、直播间小程序码
    </p>
    <p>
     说明：
    </p>
    <ul>
     <li>
      <p>
       小程序引入直播组件后必须进行一次小程序发布上线，否则小程序码不生效
      </p>
     </li>
     <li>
      <p>
       在MP小程序直播后台创建好直播间后，可以直接拿到直播间分享小程序码，无需额外开发
      </p>
     </li>
    </ul>
    <p>
     在跳转进入直播间的路径加上 type=9 标识场景入口为小程序码：
    </p>
    <pre>plugin-private://wx2b03c6e691cd7370/pages/live-player-plugin?room_id=[直播房间id]&amp;type=9</pre>
    <p>
     2、商家公众号文章添加小程序卡片
    </p>
    <p>
     说明：商家在公众号文章中添加跳转进入直播间的小程序卡片时，需要做以下配置（否则直播间左上角返回键将失效）：
    </p>
    <p>
     在跳转进入直播间的路径加上 type=10 标识场景入口为小程序卡片：
    </p>
    <pre>plugin-private://wx2b03c6e691cd7370/pages/live-player-plugin?room_id=[直播房间id]&amp;type=10</pre>
    <p>
     3、商品详情页面需添加返回按钮
    </p>
    <p>
     点击直播组件页面里的货架商品跳转到商家小程序的商品详情页面后， 为了避免用户无法再返回直播间，商家需在小程序商品详情页面左上角加上返回按钮，用于返回到直播组件页面。
    </p>
    <p>
     4、快速更新直播组件版本的方法
    </p>
    <p>
     商家小程序对应的管理员微信号 收到【公众平台安全助手】下发的直播组件版本更新的服务通知后，可点击通知进行快速发布，移动端即可快速更新小程序内直播组件的新版本，无需修改代码或重新提交审核。
    </p>
    <p>
    </p>
    <p>
     开源案例：以网上看到的来客推的为案例
