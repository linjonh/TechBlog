    <h2 id="b1yLW">
     需求分析
    </h2>
    <p id="u3fcbcd2b">
     在

     <strong>
      拍摄照片以及视频上传到后台

     进行进一步的操作，需要解决以下两个问题：
    </p>
    <ol>
     <li id="u98a07beb">
      微信小程序如何拍摄照片及视频
     </li>
     <li id="u08fe9527">
      如何将拍摄的照片及视频上传到后台进行存储
     </li>
    </ol>
    <h2 id="dTlIr">
     解决方案
    </h2>
    <p id="u6a55746c">
     前端开发：微信小程序原生
    </p>
    <p id="u0f65fd67">
     后端开发：Flask
    </p>
    <h3 id="bZ6X0">
     如何拍摄照片及视频
    </h3>
    <p id="ue82ddf60">
     微信小程序如何拍摄照片及视频：使用
     <code>
      wx.chooseMedia
     </code>
     API来实现
    </p>
    <p id="u312b396c">
     该API用于拍摄或从手机相册中选择图片或视频，文档链接为：
     <a href="https://developers.weixin.qq.com/miniprogram/dev/api/media/video/wx.chooseMedia.html" rel="nofollow" title="wx.chooseMedia(Object object) | 微信开放文档">
      wx.chooseMedia(Object object) | 微信开放文档
     </a>
    </p>
    <p id="u6ba03ba7">
     简单示例：
    </p>
    <pre id="jMq5V"><code class="language-javascript">function test()
{
  wx.chooseMedia({
        count: 1,
        mediaType: ['image'],
        sourceType: ['camera'],
        camera: 'back',
        success(res) {
          console.log(res)
        },
        fail(res){
          console.log(res)
        }
  })
}
</code></pre>
    <p id="uba0ccc1b">
     主要参数含义如下：
    </p>
    <ul>
     <li id="u37f1b8be">
      count：最多可以选择的文件个数
     </li>
     <li id="ub8f45b22">
      mediaType：文件类型，可选值为：['image']/['video']/['image','video']，默认值为['image','video']，意为即允许拍摄图片也允许拍摄视频
     </li>
     <li id="ud9d4e478">
      sourceType：文件来源，可选值为['album']/[ 'camera']/['album', 'camera']，默认值为['album', 'camera']，意为即允许从相册选择，也允许直接拍摄
     </li>
     <li id="u7b9159d4">
      camera：仅在 sourceType 为 camera 时生效，使用前置或后置摄像头，可选值为'back'/'front'，默认值为'back'
     </li>
    </ul>
    <p id="u8b05c6ef">
     回调参数res内容如下：
    </p>
    <ul>
     <li id="ueb2d563f">
      tempFiles：本地临时文件列表，其中的tempFilePath是本地临时文件路径，也是该示例中的核心参数；
     </li>
     <li id="ue0d7c3b1">
      type：文件类型
     </li>
    </ul>
    <p id="ufc9f6ee3">
     （更多参数以及回调参数请参考官方文档）
    </p>
    <p id="u05db983d">
     由上可知，我们首选需要调用
     <code>
      wx.chooseMedia
     </code>
     函数，选择拍摄照片或者视频，然后在回调函数中拿到本地临时文件路径，这样就获取到了拍摄的照片或视频，但我们拿到的并不是一个完整的.png/.mp4文件，而是一个临时文件链接，例如：
     <a href="http://tmp/z7bXVKwgyWTKcbc89c663afd501de1d27ed321f8591c.jpg" rel="nofollow" title="http://tmp/z7bXVKwgyWTKcbc89c663afd501de1d27ed321f8591c.jpg">
      http://tmp/z7bXVKwgyWTKcbc89c663afd501de1d27ed321f8591c.jpg
     </a>
    </p>
    <p id="u459fb0f4">
     这样的文件链接可以在开发者工具中打开：
    </p>
    <p class="img-center">
     <img alt="" height="312" id="u54068e46" src="https://i-blog.csdnimg.cn/blog_migrate/f5c52c7b8c539e3b9a6eb96e15cebf9e.png" width="532"/>
    </p>
    <p id="u6d104495">
     但该链接无法在外部进行访问，因此就涉及到如何将该链接所代表的文件上传到后台的问题；
    </p>
    <blockquote>
     <p id="u47d09769">
      这样的文件在小程序中被称为”本地临时文件“，仅在当前生命周期内保证有效，重启之后不一定可用；更多内容请参考官方文档：
      <a href="https://developers.weixin.qq.com/miniprogram/dev/framework/ability/file-system.html" rel="nofollow" title="文件系统 | 微信开放文档">
       文件系统 | 微信开放文档
      </a>
     </p>
    </blockquote>
    <h3 id="HhPHe">
     如何上传后台进行存储
    </h3>
    <p id="ua3f5f773">
     如何将拍摄的照片及视频上传到后台进行存储：
    </p>
    <ul>
     <li id="u07f8a69f">
      前端：使用
      <code>
       wx.uploadFile
      </code>
      API
     </li>
     <li id="u017bddb3">
      后端：使用
      <code>
       request.files['photo'].stream.read()
      </code>
      来读取文件
     </li>
    </ul>
    <h4 id="NjxTl">
     前端代码
    </h4>
    <p id="ue490ec58">
     有关
     <code>
      wx.uploadFile
     </code>
     可以参考：
     <a href="https://developers.weixin.qq.com/miniprogram/dev/api/network/upload/wx.uploadFile.html" rel="nofollow" title="UploadTask | 微信开放文档">
      UploadTask | 微信开放文档
     </a>
    </p>
    <p id="u5f98e591">
     主要参数有：
    </p>
    <ul>
     <li id="u4f71b99c">
      url：开发者服务器地址，即要访问的接口
     </li>
     <li id="u200671b1">
      filePath：要上传的文件路径（本地路径），这里即是通过
      <code>
       wx.chooseMedia
      </code>
      回调中的tempFilePath
     </li>
     <li id="ud56ad5ad">
      formData：HTTP 请求中其他额外的 form data，允许我们传输文件的时候携带一些其他的参数，比如说文件名称；
     </li>
    </ul>
    <p id="u2c9ce821">
     因此完整的前端代码如下（上传图片）：
    </p>
    <pre id="eA4qX"><code class="language-javascript">//拍摄照片
photoCapture() {
    let that = this
    wx.chooseMedia({
        count: 1,
        mediaType: ['image'],
        sourceType: ['camera'],
        camera: 'back',
        success(res) {
            that.setData({
                photoLink: res.tempFiles[0].tempFilePath,
            })
            console.log(res.tempFiles[0].tempFilePath)
            console.log('图片拍摄成功')
            wx.showLoading({
                title: '正在上传图片',
                mask: true
            })

            //图片上传
            wx.uploadFile({
                url: 'http://localhost:5000/uploadImage',
                filePath: res.tempFiles[0].tempFilePath,
                name: 'photo',
                formData: {
                    fileName: 'photoTest.png'
                },
                success(res) {
                    wx.showToast({
                        title: '图片上传成功',
                    })
                }
            })
        },
        fail(res) {
            console.log('图片拍摄失败')
        }
    })
}</code></pre>

    <h4 id="HgTdZ">
     后端代码
    </h4>
    <p id="u4706fc28">
     后端使用flask进行开发
    </p>
    <p id="ua0b0afcb">
     通过
     <code>
      request.files
     </code>
     来接收文件
    </p>
    <p id="u62b2f3eb">
     通过
     <code>
      request.form
     </code>
     来接收
     <code>
      wx.uploadFile
     </code>
     的
     <code>
      formData
     </code>
     中携带的数据
    </p>
    <p id="udbd28623">
     通过
     <code>
      write
     </code>
     方法将接收到的文件保存到本地
    </p>
    <p id="u773c0902">
     因此完整的后端代码如下（上传图片）：
    </p>
    <pre id="miToI"><code class="language-python">app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World'

# 图片上传
@app.route('/uploadImage', methods=["POST"])
def uploadImage():
    video = request.files['photo'].stream.read()
    name = request.form['fileName']
    if not files_exists(name, 2):
        file_path = os.getcwd() + '\\images\\' + name
        with open(file_path, 'ab') as f:
            f.write(video)
        return 'image upload success'
    else:
        return 'image already exist'

# 判断文件是否已经存在
def files_exists(file_name, choice):
    if choice == 1:
        path = os.getcwd() + '\\videos\\'
        video_path = os.path.join(path, file_name)
        return os.path.isfile(video_path)
    else:
        path = os.getcwd() + '\\images\\'
        image_path = os.path.join(path, file_name)
        return os.path.isfile(image_path)

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000)</code></pre>

    <h3 id="xmp4M" style="background-color:transparent;">
     后端代码
    </h3>
    <pre id="SBEjy"><code class="language-python">from flask import Flask, request
import os

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World'

@app.route('/uploadVideo', methods=["POST"])
def uploadVideo():
    video = request.files['video'].stream.read()
    name = request.form['fileName']
    if not files_exists(name, 1):
        file_path = os.getcwd() + '\\videos\\' + name
        with open(file_path, 'ab') as f:
            f.write(video)
        return 'upload success'
    else:
        return 'already exist'


@app.route('/uploadImage', methods=["POST"])
def uploadImage():
    video = request.files['photo'].stream.read()
    name = request.form['fileName']
    if not files_exists(name, 2):
        file_path = os.getcwd() + '\\images\\' + name
        with open(file_path, 'ab') as f:
            f.write(video)
        return 'upload success'
    else:
        return 'already exist'


def files_exists(file_name, choice):
    if choice == 1:
        path = os.getcwd() + '\\videos\\'
        video_path = os.path.join(path, file_name)
        return os.path.isfile(video_path)
    else:
        path = os.getcwd() + '\\images\\'
        image_path = os.path.join(path, file_name)
        return os.path.isfile(image_path)


if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000)
