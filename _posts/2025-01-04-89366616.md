    <h2>
     <a id="_0">
     </a>
     微信小游戏开发上传流程
    </h2>
    <p>
     今年因为比赛的原因初步接触了一下微信小游戏的开发。在这里做一些流程整理。
    </p>
    <h3>
     <a id="1_4">
     </a>
     1.立项
    </h3>
    <p>
     对于立项，原因可能会有很多。比如客户或者市场的需求，新技术的出现（参照微信小游戏刚刚出现的时候），任何成员一个不错的想法，或者boss的一个长时间的梦想…
     <br/>
     大概就是：市场调查，产品定位，预测趋势。
    </p>
    <h3>
     <a id="2_9">
     </a>
     2.核心玩法
    </h3>
    <p>
     在这里进入开发阶段
     <br/>
     开会确立核心玩法，进行需求分析，大概地确立游戏规则。
     <br/>
     说白了就是玩家在你这里怎么玩，到底是个什么类型的游戏。
    </p>
    <h3>
     <a id="3_16">
     </a>
     3.具体需求的分析
    </h3>
    <p>
     包括UI需求分析，程序需求分析，策划需求分析。
    </p>
    <h3>
     <a id="4demo_21">
     </a>
     4.demo阶段
    </h3>
    <p>
     明确需求中的逻辑，进行模块化开发。
     <br/>
     前端可以借助cocos、unity-3D等游戏引擎进行开发。（结合需求，如我需要一个2D还是3D的游戏）。
    </p>
    <h3>
     <a id="5_26">
     </a>
     5.版本测试与更新
    </h3>
    <p>
     看起来比较水，其实很重要的一个环节
     <br/>
     比如技能的冷却啊，或者时间间隔啊，关卡难度啊都是需要调试的。
     <br/>
     至少你总不希望游戏到玩家手里满屏的bug吧
    </p>
    <h3>
     <a id="_33">
     </a>
     在微信平台上发布
    </h3>
