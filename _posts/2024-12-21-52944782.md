    <blockquote>
     <p>
      用Python搭建自动化测试框架，我们需要组织用例以及测试执行，这里博主推荐Python的标准库——unittest。
     </p>
    </blockquote>
    <p>
     unittest是xUnit系列框架中的一员，如果你了解xUnit的其他成员，那你用unittest来应该是很轻松的，它们的工作方式都差不多。
    </p>
    <h3 id="unittest核心工作原理">
     <strong>
      unittest核心工作原理
     </strong>
    </h3>
    <p>
     unittest中最核心的四个概念是：
     <strong>
      test case, test suite, test runner, test fixture
     </strong>
     。
    </p>
    <p>
     下面我们分别来解释这四个概念的意思，先来看一张unittest的静态类图（下面的类图以及解释均来源于网络，
     <a href="http://www.cnblogs.com/hackerain/p/3682019.html" rel="nofollow">
      原文链接
     </a>
     ）：
    </p>
    <p>
     <img alt="unittest类图" src="https://img-blog.csdn.net/20161026164739484" title=""/>
    </p>
    <blockquote>
     <ul>
      <li>
       <p>
        一个TestCase的实例就是一个测试用例。什么是测试用例呢？就是一个完整的测试流程，包括测试前准备环境的搭建(setUp)，执行测试代码(run)，以及测试后环境的还原(tearDown)。元测试(unit test)的本质也就在这里，一个测试用例是一个完整的测试单元，通过运行这个测试单元，可以对某一个问题进行验证。
       </p>
      </li>
      <li>
       <p>
        而多个测试用例集合在一起，就是TestSuite，而且TestSuite也可以嵌套TestSuite。
       </p>
      </li>
      <li>
       <p>
        TestLoader是用来加载TestCase到TestSuite中的，其中有几个loadTestsFrom__()方法，就是从各个地方寻找TestCase，创建它们的实例，然后add到TestSuite中，再返回一个TestSuite实例。
       </p>
      </li>
      <li>
       <p>
        TextTestRunner是来执行测试用例的，其中的run(test)会执行TestSuite/TestCase中的run(result)方法。
        <br/>
        测试的结果会保存到TextTestResult实例中，包括运行了多少测试用例，成功了多少，失败了多少等信息。
       </p>
      </li>
      <li>
       <p>
        而对一个测试用例环境的搭建和销毁，是一个fixture。
       </p>
      </li>
     </ul>
    </blockquote>
    <p>
     一个class继承了unittest.TestCase，便是一个测试用例，但如果其中有多个以
     <code>
      test
     </code>
     开头的方法，那么每有一个这样的方法，在load的时候便会生成一个TestCase实例，如：一个class中有四个test_xxx方法，最后在load到suite中时也有四个测试用例。
    </p>
    <p>
     到这里整个流程就清楚了：
    </p>
    <p>
     写好TestCase，然后由TestLoader加载TestCase到TestSuite，然后由TextTestRunner来运行TestSuite，运行的结果保存在TextTestResult中，我们通过命令行或者unittest.main()执行时，main会调用TextTestRunner中的run来执行，或者我们可以直接通过TextTestRunner来执行用例。这里加个说明，在Runner执行时，默认将执行结果输出到控制台，我们可以设置其输出到文件，在文件中查看结果（你可能听说过HTMLTestRunner，是的，通过它可以将结果输出到HTML中，生成漂亮的报告，它跟TextTestRunner是一样的，从名字就能看出来，这个我们后面再说）。
    </p>

    <h4 id="组织testsuite">
     <strong>
      组织TestSuite
     </strong>
    </h4>
    <p>
     上面的代码示例了如何编写一个简单的测试，但有两个问题，我们怎么控制用例执行的顺序呢？（这里的示例中的几个测试方法并没有一定关系，但之后你写的用例可能会有先后关系，需要先执行方法A，再执行方法B），我们就要用到TestSuite了。我们
     <strong>
      添加到TestSuite中的case是会按照添加的顺序执行的
     </strong>
     。
    </p>
    <p>
     问题二是我们现在只有一个测试文件，我们直接执行该文件即可，但如果有多个测试文件，怎么进行组织，总不能一个个文件执行吧，答案也在TestSuite中。
    </p>
    <p>
     下面来个例子：
    </p>
    <p>
     在文件夹中我们再新建一个文件，
     <strong>
      test_suite.py
     </strong>
     ：
    </p>
    <pre class="prettyprint"><code class="language-python hljs"><span class="hljs-comment"># -*- coding: utf-8 -*-</span>

<span class="hljs-keyword">import</span> unittest
<span class="hljs-keyword">from</span> test_mathfunc <span class="hljs-keyword">import</span> TestMathFunc

<span class="hljs-keyword">if</span> __name__ == <span class="hljs-string">'__main__'</span>:
    suite = unittest.TestSuite()

    tests = [TestMathFunc(<span class="hljs-string">"test_add"</span>), TestMathFunc(<span class="hljs-string">"test_minus"</span>), TestMathFunc(<span class="hljs-string">"test_divide"</span>)]
    suite.addTests(tests)

    runner = unittest.TextTestRunner(verbosity=<span class="hljs-number">2</span>)
    runner.run(suite)</code></pre>
    <p>
     执行结果：
    </p>

    </p>

     上面用了TestSuite的
     <code>
      addTests()
     </code>
     方法，并直接传入了TestCase列表，我们还可以：
    </p>
    <pre class="prettyprint"><code class="language-python hljs"><span class="hljs-comment"># 直接用addTest方法添加单个TestCase</span>
suite.addTest(TestMathFunc(<span class="hljs-string">"test_multi"</span>))

     注意，用TestLoader的方法是无法对case进行排序的，同时，suite中也可以套suite。
    </p>
    <h4 id="将结果输出到文件中">
     <strong>
      将结果输出到文件中
     </strong>
    </h4>
    <p>
     用例组织好了，但结果只能输出到控制台，这样没有办法查看之前的执行记录，我们想将结果输出到文件。很简单，看示例：
    </p>
    <p>
     修改
     <strong>
      test_suite.py
     </strong>
     ：
    </p>
    <pre class="prettyprint"><code class="language-python hljs"><span class="hljs-comment"># -*- coding: utf-8 -*-</span>

<span class="hljs-keyword">from</span> test_mathfunc <span class="hljs-keyword">import</span> TestMathFunc

    <span class="hljs-keyword">with</span> open(<span class="hljs-string">'UnittestTextReport.txt'</span>, <span class="hljs-string">'a'</span>) <span class="hljs-keyword">as</span> f:
        runner = unittest.TextTestRunner(stream=f, verbosity=<span class="hljs-number">2</span>)

    <p>
     执行此文件，可以看到，在同目录下生成了
     <strong>
      UnittestTextReport.txt
     </strong>
     ，所有的执行报告均输出到了此文件中，这下我们便有了txt格式的测试报告了。
    </p>

    <h4 id="跳过某个case">
     <strong>
      跳过某个case
     </strong>
    </h4>
    <p>
     如果我们临时想要跳过某个case不执行怎么办？unittest也提供了几种方法：
    </p>
    <ol>
     <li>
      skip装饰器
     </li>
    </ol>
    <pre class="prettyprint"><code class="language-python hljs">...

    ...

        <span class="hljs-string">"""Test method divide(a, b)"""</span>
        <span class="hljs-keyword">print</span> <span class="hljs-string">"divide"</span>
        self.assertEqual(<span class="hljs-number">2</span>, divide(<span class="hljs-number">6</span>, <span class="hljs-number">3</span>))
        self.assertEqual(<span class="hljs-number">2.5</span>, divide(<span class="hljs-number">5</span>, <span class="hljs-number">2</span>))</code></pre>
    <p>
     执行：
    </p>
    <pre class="prettyprint"><code class="hljs r"><span class="hljs-keyword">...</span>

    <p>
     可以看到总的test数量还是4个，但divide()方法被skip了。
    </p>
    <p>
     skip装饰器一共有三个
     <code>
      unittest.skip(reason)
     </code>
     、
     <code>
      unittest.skipIf(condition, reason)
     </code>
     、
     <code>
      unittest.skipUnless(condition, reason)
     </code>
     ，skip无条件跳过，skipIf当condition为True时跳过，skipUnless当condition为False时跳过。
    </p>

    <h3 id="进阶用htmltestrunner输出漂亮的html报告">
     <strong>
      进阶——用HTMLTestRunner输出漂亮的HTML报告
     </strong>
    </h3>
    <p>
     我们能够输出txt格式的文本执行报告了，但是文本报告太过简陋，是不是想要更加高大上的HTML报告？但unittest自己可没有带HTML报告，我们只能求助于外部的库了。
    </p>
    <p>
     HTMLTestRunner是一个第三方的unittest HTML报告库，首先我们下载
     <strong>
      HTMLTestRunner.py
     </strong>
     ，并放到当前目录下，或者你的’C:\Python27\Lib’下，就可以导入运行了。
    </p>
    <p>
     下载地址：
    </p>
    <blockquote>
     <p>

       </a>
      </strong>
     </p>
     <p>
      灰蓝修改版：
      <strong>
       <a href="http://download.csdn.net/detail/huilan_same/9598558">

      </strong>
     </p>
    </blockquote>
    <p>
     修改我们的
     <code>
      test_suite.py

    <h3 id="总结一下">
     <strong>
      总结一下
     </strong>
     ：
    </h3>
    <ol>
     <li>
      unittest是Python自带的单元测试框架，我们可以用其来作为我们自动化测试框架的用例组织执行框架。
     </li>
     <li>
      unittest的流程：写好TestCase，然后由TestLoader加载TestCase到TestSuite，然后由TextTestRunner来运行TestSuite，运行的结果保存在TextTestResult中，我们通过命令行或者unittest.main()执行时，main会调用TextTestRunner中的run来执行，或者我们可以直接通过TextTestRunner来执行用例。
     </li>
     <li>
      一个class继承unittest.TestCase即是一个TestCase，其中以
      <code>
       test
      </code>
      开头的方法在load时被加载为一个真正的TestCase。
     </li>
     <li>
      verbosity参数可以控制执行结果的输出，
      <code>
       0
      </code>
      是简单报告、
      <code>
       1
      </code>
      是一般报告、
      <code>
       2
      </code>
      是详细报告。
     </li>
     <li>
      可以通过addTest和addTests向suite中添加case或suite，可以用TestLoader的loadTestsFrom__()方法。
     </li>
     <li>
      用
      <code>
       setUp()
      </code>
      、
      <code>
       tearDown()
      </code>
      、
      <code>
       setUpClass()
      </code>
      以及
      <code>
       tearDownClass()
      </code>
      可以在用例执行前布置环境，以及在用例执行后清理环境
     </li>
     <li>
      我们可以通过skip，skipIf，skipUnless装饰器跳过某个case，或者用TestCase.skipTest方法。
     </li>
     <li>
      参数中加stream，可以将报告输出到文件：可以用TextTestRunner输出txt报告，以及可以用HTMLTestRunner输出html报告。
     </li>
    </ol>

     我们这里没有讨论命令行的使用以及模块级别的fixture，感兴趣的同学可以自行搜索资料学习。
