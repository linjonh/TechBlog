    <h2>
     <a id="_1">
     </a>
     一，小程序创建的配置文件介绍
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8fffbc2a463f421e87180e174319a458.png"/>
    </p>
    <p>
     根据图片内容，微信小程序中包含几种不同的配置文件，每种文件都有其特定的用途和作用。以下是这些配置文件的介绍：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        app.json
       </strong>
       ：
      </p>
      <ul>
       <li>
        这是小程序的全局配置文件。
       </li>
       <li>
        用于配置小程序的一些全局属性和页面路由。
       </li>
       <li>
        例如，你可以在这里设置小程序的页面路径、窗口表现、网络超时时间、底部 tab 等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        页面.json
       </strong>
       ：
      </p>
      <ul>
       <li>
        每个小程序页面都可以有一个对应的.json配置文件，称为局部配置文件。
       </li>
       <li>
        用于配置当前页面的窗口样式、导航栏标题、导航栏颜色等。
       </li>
       <li>
        这些配置仅对该页面有效。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        project.config.json
       </strong>
       ：
      </p>
      <ul>
       <li>
        这是小程序项目的配置文件。
       </li>
       <li>
        用于保存项目的一些配置信息，如项目名称、appid、项目目录结构等。
       </li>
       <li>
        也用于保存开发者的个人设置。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        sitemap.json
       </strong>
       ：
      </p>
      <ul>
       <li>
        用于配置小程序及其页面是否允许被微信索引。
       </li>
       <li>
        通过这个文件，可以提高小程序在微信搜索引擎中的搜索概率，从而增加小程序的曝光率。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        project.private.config.json
       </strong>
       ：
      </p>
      <ul>
       <li>
        这是一个私有的项目配置文件。
       </li>
       <li>
        通常用于存储敏感信息或不应提交到版本控制系统的配置，如API密钥等。
       </li>
      </ul>
     </li>
    </ol>
    <h2>
     <a id="pages__30">
     </a>
     二，配置文件-全局配置-pages 配置
    </h2>
    <p>
     在微信小程序的全局配置文件
     <code>
      app.json
     </code>
     中，
     <code>
      pages
     </code>
     字段扮演着至关重要的角色。以下是
     <code>
      pages
     </code>
     字段的作用和注意事项：
    </p>
    <h3>
     <a id="_34">
     </a>
     作用：
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        定义页面路径
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         pages
        </code>
        字段用来指定小程序由哪些页面组成。
       </li>
       <li>
        它定义了小程序中所有页面的路径，让小程序知道页面定义在哪个目录。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        页面路由
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过
        <code>
         pages
        </code>
        数组，小程序框架能够知道如何根据路径加载对应的页面。
       </li>
       <li>
        它不需要写文件后缀，框架会自动查找对应位置的
        <code>
         .json
        </code>
        、
        <code>
         .js
        </code>
        、
        <code>
         .wxml
        </code>
        、
        <code>
         .wxss
        </code>
        四个文件进行处理。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        初始页面
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果没有指定
        <code>
         entryPagePath
        </code>
        ，数组的第一项代表小程序的初始页面（首页）。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_46">
     </a>
     注意事项：
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        文件后缀
       </strong>
       ：
      </p>
      <ul>
       <li>
        在配置
        <code>
         pages
        </code>
        字段时，不需要写文件后缀，如
        <code>
         index
        </code>
        而不是
        <code>
         index.wxml
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        动态修改
       </strong>
       ：
      </p>
      <ul>
       <li>
        小程序中新增或减少页面时，都需要对
        <code>
         pages
        </code>
        数组进行相应的修改。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        初始页面
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果没有指定
        <code>
         entryPagePath
        </code>
        ，数组的第一项自动成为小程序的初始页面。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        路径格式
       </strong>
       ：
      </p>
      <ul>
       <li>
        路径应该以斜杠
        <code>
         /
        </code>
        开头，表示相对于项目根目录的路径。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数组顺序
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         pages
        </code>
        数组中的顺序会影响小程序的页面加载顺序。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_62">
     </a>
     示例：
    </h3>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
  <span class="token string">"pages"</span><span class="token operator">:</span> <span class="token punctuation">[</span>
    <span class="token string">"pages/index/index"</span><span class="token punctuation">,</span>
    <span class="token string">"pages/logs/logs"</span><span class="token punctuation">,</span>
    <span class="token string">"pages/about/about"</span>
  <span class="token punctuation">]</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在这个示例中：
    </p>
    <ul>
     <li>
      <code>
       pages/index/index
      </code>
      是小程序的初始页面。
     </li>
     <li>
      <code>
       pages/logs/logs
      </code>
      和
      <code>
       pages/about/about
      </code>
      是其他页面。
     </li>
    </ul>
    <p>
     通过合理配置
     <code>
      pages
     </code>
     字段，可以确保小程序的页面结构清晰，便于管理和维护。
    </p>
    <h2>
     <a id="window__79">
     </a>
     三，配置文件-全局配置-window 配置
    </h2>
    <p>
     <a href="developers.weixin.qq.com/miniprogram/dev/reference/configuration/app.html" rel="nofollow">
      参考官方文档
     </a>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f5f3a40dccab459793ce5d3827300b9b.png"/>
    </p>
    <p>
     在微信小程序的全局配置文件
     <code>
      app.json
     </code>
     中，
     <code>
      window
     </code>
     字段用于设置小程序的状态栏、导航条、标题、窗口背景色等全局样式属性。以下是
     <code>
      window
     </code>
     配置的作用：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        状态栏样式
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置状态栏的颜色、隐藏或显示等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏样式
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏的颜色、字体颜色、背景色等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏标题
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏标题的文本内容、字体大小、字体颜色等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        窗口背景色
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置小程序窗口的背景颜色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏高度
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏的高度。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        下拉背景字体色
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置下拉时背景字体的颜色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        下拉背景色
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置下拉时背景的颜色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏阴影
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏的阴影效果。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏圆角
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏的圆角效果。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        导航栏边框
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以设置导航栏的边框。
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <a id="_117">
     </a>
     示例：
    </h4>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
  <span class="token string">"window"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string">"navigationBarTitleText"</span><span class="token operator">:</span> <span class="token string">"小程序标题"</span><span class="token punctuation">,</span>
    <span class="token string">"navigationBarBackgroundColor"</span><span class="token operator">:</span> <span class="token string">"#ffffff"</span><span class="token punctuation">,</span>

    <span class="token string">"backgroundColor"</span><span class="token operator">:</span> <span class="token string">"#eeeeee"</span><span class="token punctuation">,</span>
    <span class="token string">"backgroundTextStyle"</span><span class="token operator">:</span> <span class="token string">"light"</span><span class="token punctuation">,</span>
    <span class="token string">"navigationBarShadow"</span><span class="token operator">:</span> <span class="token string">"#000000"</span><span class="token punctuation">,</span>
    <span class="token string">"navigationStyle"</span><span class="token operator">:</span> <span class="token string">"custom"</span><span class="token punctuation">,</span>
    <span class="token string">"enablePullDownRefresh"</span><span class="token operator">:</span> <span class="token boolean">true</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在这个示例中：
    </p>
    <ul>
     <li>
      <code>
       navigationBarTitleText
      </code>
      设置导航栏标题为“小程序标题”。
     </li>
     <li>
      <code>
       navigationBarBackgroundColor
      </code>
      设置导航栏背景颜色为白色。
     </li>
     <li>
      <code>
       navigationBarTextStyle
      </code>
      设置导航栏文字颜色为黑色。
     </li>
     <li>
      <code>
       backgroundColor
      </code>
      设置窗口的背景颜色为浅灰色。
     </li>
     <li>
      <code>
       backgroundTextStyle
      </code>
      设置下拉时背景字体颜色为浅色。
     </li>
     <li>
      <code>
       navigationBarShadow
      </code>
      设置导航栏的阴影效果。
     </li>
     <li>
      <code>
       navigationStyle
      </code>
      设置导航栏样式为自定义。
     </li>
     <li>
      <code>
       enablePullDownRefresh
      </code>
      启用下拉刷新功能。
     </li>
    </ul>
    <p>
     通过合理配置
     <code>
      window
     </code>
     字段，可以确保小程序的界面风格统一，提升用户体验。
    </p>
    <h2>
     <a id="tabbar__145">
     </a>
     四，配置文件-全局配置-tabbar 配置
    </h2>
    <p>
     在微信小程序的全局配置文件
     <code>
      app.json
     </code>
     中，
     <code>
      tabBar
     </code>
     字段用于定义小程序的底部 tab 栏，它允许用户快速在不同的页面间切换。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5d6ef26f40dc4e07bbd8b7d7bdaee9cb.png"/>
    </p>
    <p>
     以下是
     <code>
      tabBar
     </code>
     的核心作用和配置项。
    </p>
    <h3>
     <a id="_153">
     </a>
     核心作用：
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        页面快速切换
       </strong>
       ：
      </p>
      <ul>
       <li>
        提供一个直观的界面元素，让用户可以快速在不同的页面间切换。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        持久显示
       </strong>
       ：
      </p>
      <ul>
       <li>
        在小程序的底部或顶部持久显示，方便用户随时访问。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        提升用户体验
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过直观的图标和文本，提高用户的操作便利性和小程序的用户体验。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_163">
     </a>
     配置项：
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        list
       </strong>
       ：
      </p>
      <ul>
       <li>
        一个数组，定义了 tab 栏中的所有 tab 项，最少包含 2 个，最多 5 个。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        pagePath
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 对应页面的路径。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        iconPath
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 的默认图标路径。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        selectedIconPath
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 被选中时的图标路径。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        text
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 的文字描述。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        borderStyle
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 栏边框的颜色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        backgroundColor
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 栏的背景色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        color
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 的文字和图标的默认颜色。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        selectedColor
       </strong>
       ：
      </p>
      <ul>
       <li>
        指定 tab 被选中时的文字和图标颜色。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_191">
     </a>
     示例配置：
    </h3>
