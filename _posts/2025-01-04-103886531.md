    <p>
     往简单来讲，用户输入了搜索词，系统通过搜索词找到与搜索词相关的商品集合，系统通过用户及商品的情况进行排序，最终展现给用户。
    </p>
    <h2>
     0.找不到
    </h2>
    <p>
     <strong>
      但是在构建搜索系统的初期总是无法精准地帮助用户找到想要的商品主要原因有以下几点:
     </strong>
    </p>
    <ul>
     <li>
      不同的用户对同一种
      <strong>
       诉求的表达
      </strong>
      往往是有
      <strong>
       差别
      </strong>
      的，往往会存在一种比较常见的现象，用户输入的query并
      <strong>
       不能清晰准确的表达需求
      </strong>
      。这一块是
      <strong>
       可以通过较
      </strong>
      好的
      <strong>
       产品设计及实时反馈
      </strong>
      来
      <strong>
       做
      </strong>
      精确需求
      <strong>
       表达捕捉
      </strong>
      ，产品设计主要是前面介绍的
      <strong>
       引导类产品
      </strong>
      (下拉推荐、筛选、锦囊等)，
      <strong>
       实时反馈
      </strong>
      是指快速捕捉用户在当前query下的正负反馈，系统捕捉其中的query意图。
     </li>
     <li>
      搜索系统
      <strong>
       对用户query的理解能力较弱
      </strong>
      ，无法挖掘出用户的真实需求。这一块则更多是算法发力的点，
      <strong>
       通过文本、行为、session
      </strong>
      等数据
      <strong>
       挖掘
      </strong>
      query本身的内涵，
      <strong>
       这一块更多是QU
      </strong>
      (query understanding)的工作。
     </li>
     <li>
      用户输入了
      <strong>
       长尾词，过多条件无法匹配
      </strong>
      到商品。这一块也可以通过
      <strong>
       QU和QR
      </strong>
      (query rewrite)来逐步解决。
     </li>
     <li>
      召回结果集的
      <strong>
       排序不合理
      </strong>
      ，可能用户需求的内容被排在后面而未曝光。这一块则是
      <strong>
       排序的问题
      </strong>
      了，建立一个
      <strong>
       良性的评分排序公式
      </strong>
      ，并且
      <strong>
       利用算法合理建模用户决策过程
      </strong>
      。
     </li>
    </ul>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/eb10eababf7996fd0a8edd6f3055209b.png"/>

    <h2>
     1. QP (query process)
    </h2>
    <h3>
     <strong>
      1.1 QU/query understanding
     </strong>
    </h3>
    <p>
     <strong>
      1.1.1 概述
     </strong>
    </p>
    <p>
     <strong>
      1.1.1.1 目的
     </strong>
    </p>
    <ul>
     <li>
      拆解用户搜索词的意图
     </li>
     <li>
      比如新品，年龄，尺码，属性，类目等搜索意图识别及归一
     </li>
    </ul>
    <p>
     <strong>
      1.1.1.2 任务
     </strong>
    </p>
    <ul>
     <li>
      Query词性及主体结构，主要词/描述词等: 2018最新款适合胖胖的女生穿的
      <strong>
       连衣裙
      </strong>
     </li>
     <li>
      预测用户搜索商品类目(category)性别(gender): 手提电脑、t恤 女
     </li>
     <li>
      属性&amp;标签识别: 品牌，颜色，尺寸: 裙子红色，43码nike球鞋
     </li>
     <li>
      搜&amp;逛:强意图/转化&amp;弱意图/逛: 连衣裙 &amp; Iphone XR 256G
     </li>
    </ul>
    <p>
     <strong>
      1.1.1.3 方法
     </strong>
    </p>
    <ul>
     <li>
      方法词表穷举法，规则解析法，机器学习方法
     </li>
    </ul>
    <p>
     <strong>
      1.1.1.4 意图识别的难点
     </strong>
    </p>
    <ul>
     <li>
      输入不规范，不同的用户对同一诉求的表达存在差异。
     </li>
     <li>
      多意图，“苹果” 可以是产品词，也可以是品牌词；可以是手机，也可以是水果。
     </li>
     <li>
      数据冷启动。当用户行为数据较少时，很难获取准确的意图。
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      1.1.2 词性&amp;主体识别&amp;属性/标签识别
     </strong>
    </p>
    <p>
     词性的识别有助于整个搜索系统快速地找到和定位相关商品，也可以帮助
     <strong>
      快速定位核心词、属性词
     </strong>
     等。
    </p>
    <p>
     搜索过程中，
     <strong>
      不同term对于检索有不同的意义
     </strong>
     ，不能本末倒置。
     <strong>
      不同重要程度的词
     </strong>
     ，应该在
     <strong>
      召回排序阶段给予相应不同的影响
     </strong>
     ，
     <strong>
      核心词具有更高的分值
     </strong>
     。当用户搜”children toys“召回商品时，核心词是toys，children为修饰词，根据term weight来进行排序降权的。
     <strong>
      细粒度地还可以做进一步区分产品词、品牌词、型号词、停用词。
     </strong>
     query被完整匹配和部分匹配的权重是不同的、单词命中和多词命中同一商品也需要考虑权重情况。
    </p>
    <p>
     其它还包括了
     <strong>
      中心词逻辑、热词逻辑、纠错系统、丢弃词逻辑、词性标注等工作
     </strong>
     。常见方法词性识别有,
     <strong>
      基于规则
     </strong>
     和
     <strong>
      基于统计的词性标注
     </strong>
     (HMM)。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/17816ade89a4e5b3105a6c1a60a6585a.png"/>
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/ec6db08e564d9853e896c334a0e664c2.png"/>
    </p>
    <p>
     <strong>
      1.1.3 词画像
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       词属性
      </strong>
     </li>
    </ul>
    <p>
     基础属性:pv、uv、gmv、ctr、cvr等
    </p>
    <p>
     业务属性:品牌词、大促属性
    </p>
    <p>
     词质量分
    </p>
    <ul>
     <li>
      <strong>
       词关系
      </strong>
     </li>
    </ul>
    <p>
     同义词、形近词、同音词、子母品牌、类目、文本相似性
    </p>
    <ul>
     <li>
      <strong>
       词维度的用户画像
      </strong>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      1.1.4 强弱意图
     </strong>
    </p>
    <p>
     用户强弱意图/转化意图识别，可以快速帮助搜索系统
     <strong>
      定位召回及排序策略
     </strong>
     ，不同的意图可以带来不同的排序和展现效果。如
     <strong>
      强意图下相关性因子应该加强，弱意图下应该更加注重点击/转化等反馈行为量
     </strong>
     。
    </p>
    <ul>
     <li>
      强意图/转化型: 需要快速帮助用户定位所需的商品 (因素:价格、品牌、品质、商家等)，推送引导的目的是让用户作出购买，收藏等决策，追求转化的数量+速度+质量。
     </li>
     <li>
      弱意图/闲逛型: 需要帮助用户发掘新的兴趣、新的话题，但同时不能让用户 感觉无聊，目的是满足用户需求，把用户喜欢的推荐给他，追求 pv/点击率。
     </li>
    </ul>
    <p>
     <strong>
      根据用户行为和query的静态信息
     </strong>
     ，分析query是
     <strong>
      搜索型（偏向买）
     </strong>
     还是
     <strong>
      浏览型（偏向逛）
     </strong>
     。后续利用模型对query分类，
     <strong>
      用以分析排序策略对不同类型query的影响，方便对不同类型query作不同排序
     </strong>
     。
    </p>
    <p>
     分析用户个性化标签的浏览行，转化型趋势。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/4c0a06fca17ba36cdf585b1377e30ccc.png"/>
    </p>
    <p>
     <strong>
      若干特征举例:
     </strong>
    </p>
    <ul>
     <li>
      session+query内商品的点击率
     </li>
     <li>
      Session内不同query的个数
     </li>
     <li>
      空格数量
     </li>
     <li>
      相关一级类目个数
     </li>
     <li>
      停留时长再逛和搜query中表现差异大
     </li>
     <li>
      行业分布:服装鞋包/3c、美容护理、食品保健、话费充值差异大
     </li>
    </ul>
    <p>
     <br/>
     <strong>
      1.1.5 类目预测
     </strong>
    </p>
    <p>
     Query的类目预测主要是，
     <strong>
      分析Query和哪些类目的意图更相关
     </strong>
     (当然这里
     <strong>
      用户维度的信息也会被考虑进来
     </strong>
     )。query通过搜索引擎召回后，一般将
     <strong>
      类目相关性作为
     </strong>
     重要的
     <strong>
      海选排序因子
     </strong>
     ，保留一部分商品，一方面保证了
     <strong>
      效率，
     </strong>
     另一方面也
     <strong>
      从源头保证类目的相关性
     </strong>
     ，保证
     <strong>
      用户体验
     </strong>
     。从实际工作来看，
     <strong>
      fasttext
     </strong>
     是一个非常不错，实践也较快的算法。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/007aacede1a87265ee16bf42c349e468.png" width="776"/>
    </p>
    <p>
     类目举例
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/401f061568267899363b6095ce21a2f3.png" width="1200"/>
    </p>
    <p>
     常见模型
    </p>
    <h3>
     <strong>
      1.2 QR/query rewrite
     </strong>
    </h3>
    <p>
     <strong>
      1.2.1 概述
     </strong>
    </p>
    <p>
     <strong>
      1.2.1.1 问题
     </strong>
    </p>
    <ul>
     <li>
      query和商品描述之间存在
      <strong>
       gap
      </strong>
      ，特别是中长尾query。
      <strong>
       多种描述，信息冗余，属性检索，宽泛意图
      </strong>
      。
     </li>
    </ul>
    <p>
     <strong>
      1.2.1.2 目标
     </strong>
    </p>
    <ul>
     <li>
      文本和意图，通过对原始Query进行改写，生成一系列相关Query，把相关Query作为原始Query的补充，与原始Query一起参与搜索，从而得到更加丰富和准确的匹配结果
     </li>
    </ul>
    <p>
     <strong>
      1.2.1.3 方法
     </strong>
    </p>
    <ul>
     <li>
      query embedding和multi-method
     </li>
    </ul>
    <p>
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/9d82d1ec9317390b97aa3b17e7d53661.png"/>
    </p>
    <p>
     <strong>
      1.2.2 query embedding
     </strong>
    </p>
    <ul>
     <li>
      query embedding(query映射到query)，可以针对
      <strong>
       “多种描述”和信息冗余问题意图改写
      </strong>
      :query映射到意图，主要针对属性检索和宽泛意图类型；也可以进行相似query挖掘。
     </li>
     <li>
      向量改写流程: query向量化-&gt;向量相似查找-&gt;相关性判断;借鉴skip-throught-vector，使用
      <strong>
       seq2seq重建句子周围的句子
      </strong>
      ,假设某个session序列是（s1，s2，…,sn）,那么一条训练数据为(si-1,si,si+1),encoder是si的词序列的lstm，decoder是分别si-1和si+1，这样训练下来decoder的上下文向量就学到了这个句子在session中的上下文表示。
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      1.2.3 multi-method
     </strong>
    </p>
    <ul>
     <li>
      通过
      <strong>
       语义、行为、session log
      </strong>
      等挖掘出相似的query。
     </li>
     <li>
      <strong>
       多维度相似融合再排序
      </strong>
      : 按照
      <strong>
       点击相似度、文本相似度、Session相似度
      </strong>
      衡量Query之间的相似度，
      <strong>
       得到候选的Pair（可选）交给重排序模块，对Query pair的优先级做优化，生成Top K的改写结果。
      </strong>
     </li>
     <li>
      <strong>
       基于行为Item cf/swing、Simrank++
      </strong>
     </li>
     <li>
      <strong>
       基于session Word2vec、seq2seq
      </strong>
     </li>
     <li>
      <strong>
       基于内容 Word2vec
      </strong>
     </li>
     <li>
      <strong>
       融合 LR/GBDT
      </strong>
     </li>
    </ul>
    <p>
    </p>
    <h3>
     <strong>
      2.召回&amp;检索模块: ltm/learn to match
     </strong>
    </h3>
    <p>
     <strong>
      2.1 检索依据
     </strong>
    </p>
    <ul>
     <li>
      电商商品: 图片+标题+属性+交互，检索项包括但不限于:商品名称，商品标题、副标题，商品描述，商品参数、规格，商品品牌，商品品类，别名关联商品，促销类型
     </li>
     <li>
      相关性(query&amp;tittle/content，行为，session): 融合点击相似度、文本相似度、Session相似度衡量Query之间的相似度,除了前面介绍的通过query session来做query embedding，用query来重建其点击过的宝贝标题/描述序列同样适用，只不过decoder阶段换成query点击过的标题。
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      2.2 语义搜索
     </strong>
    </p>
    <p>
     语义搜索是指不单单考虑词维度的精确匹配，而是语义层面来做。增加搜索结果的相关性，提升用户体验外，也可以一定程度上遏制商家商品标题堆砌热门关键词的问题。
    </p>
    <p>
     <strong>
      2.2.1 常见doc&amp;query匹配方法
     </strong>
    </p>
    <ul>
     <li>
      BM25通常计算query和Doc文本term的匹配程度。由于query和doc之间的语义gap，可能存在很多语义相关，但文本并不匹配的情况。
     </li>
     <li>
      通过商品内容理解和语义标签: 通过商品图片，详情页，评价和同义词，上下位词等给商品打标签和扩充商品索引内容
     </li>
     <li>
      语义匹配: Dssm模型将query和文本变成向量，用向量内积表达语义相似度
     </li>
     <li>
      <strong>
       匹配深度与高度
      </strong>
      : 词-&gt;短语-&gt;语义-&gt;主题-&gt;句法
     </li>
    </ul>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/3a4a1f0d79e4b2d7c6c2ac8c41635405.png" width="1200"/>
    </p>
    <p>
     词-&amp;amp;amp;amp;gt;短语-&amp;amp;amp;amp;gt;语义-&amp;amp;amp;amp;gt;主题-&amp;amp;amp;amp;gt;句法
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/4f48d69a3e55d164a228b53ad0a5f480.png" width="1200"/>
    </p>
    <p>
     embedding-similar
    </p>
    <p>
    </p>
    <p>
     <strong>
      2.3 无结果优化
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       二次/三次召回
      </strong>
      :
      <strong>
       放弃权重低term
      </strong>
      ，扩大检索字段和检索范围
     </li>
     <li>
      <strong>
       Query 纠错 &amp; 同义词改写
      </strong>
      :
      <strong>
       同时用原词和同义词去检索，最后对两者返回的结果取并集
      </strong>
      。
     </li>
     <li>
      <strong>
       分类意图识别的优化
      </strong>
      ，首先根据 Query 分布定义了 8 类意图:可以通过识别 Query 中 Term 的意图来判定整个 Query 的意图
     </li>
    </ul>
    <h3>
     <strong>
      3.ltr/learn to rank
     </strong>
    </h3>
    <p>
