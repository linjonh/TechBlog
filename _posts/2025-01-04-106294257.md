    <h5>
     <a id="21_48">
     </a>
     2.1.接口信息查看
    </h5>
    <p>
     第一步肯定是了解需要进行性能测试的接口信息，如果有现成的接口文档更好。这里是在浏览器中通过F12键去网页中获取接口信息。
    </p>
    <ul>
     <li>
      <strong>
       General数据
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b5702f28608f382ada3b57cd423a5083.png"/>
     </li>
    </ul>
    <pre><code>协议类型：http或者https
域名或者IP:例如访问百度就是www.baidu.com
方法类型：POST或GET
请求路径：例如我这里是/auth/oauth/token
端口号：例如我这里是443
</code></pre>
    <ul>
     <li>
      <strong>
       Request Headers请求头
      </strong>
     </li>
    </ul>
    <p>
     一般情况下，对一个借口进行访问的时候，必须带上相关的请求头。如果不清楚哪些请求头参数需要添加上，就一个个添加
     <code>
      名称中不含:
     </code>
     的请求头。例如我这里就需要将
     <code>
      authorization
     </code>
     这个请求头加上。
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/513664564f5b369b10981e940f5624b3.png"/>
    </p>
    <ul>
     <li>
      <strong>
       请求参数
      </strong>
     </li>
    </ul>
    <p>
     请求参数有的时候是以Form Data形式，有的时候是以JSON串的形式显示。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b70ad46f4dc836e32ae79b6f1106d5c8.png"/>
    </p>
    <h5>
     <a id="22__68">
     </a>
     2.2 脚本编写
    </h5>
    <ul>
     <li>
      <strong>
       创建线程组
      </strong>
     </li>
    </ul>
    <p>
     （1）测试计划下添加-&gt;线程-&gt;线程组
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ef15e0109aef2fc9b8d35ba69586b22b.png"/>
     <br/>
     （2）填写相关参数
    </p>
    <pre><code>线程数：即为虚拟用户数。一个虚拟用户占用一个进程或线程。
Ramp-Up时间：设置虚拟用户数启动需要的时间。
循环次数：即为循环
调度器：需要设置持续时间或者启动延迟的时候，才需要勾选。勾选该选项后，需要将循环次数勾选永远
持续时间：测试持续时间

下面给出两种设置的情况：
一种是设置线程数、Ramp-Up时间、循环次数。
另一种是设置线程数、Ramp-Up时间、持续时间
</code></pre>
    <ul>
     <li>
      设置10、2、10，含义就是在2秒内启动10个线程，并循环10次，最后发送的总请求数就为10*10=100.
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b1b3f346a2930da18d3ffa2837b41a90.png"/>
     </li>
     <li>
      设置10、2、10，含义就是在2秒内启动10个线程，并持续压测10秒。
      <code>
       在实际情况中，我们进行性能测试中，一般会对一个接口压测5~10分钟，甚至更长。
      </code>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/df2ce2ffe2b33b295041af1f6e84e6c2.png"/>
     </li>
     <li>
      <strong>
       创建HTTP请求
      </strong>
     </li>
    </ul>
    <p>
     线程数上 添加-&gt;取样器-&gt;HTTP请求，然后根据最初查看到的接口信息，填写相关数据。
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/968ebb5a467eaeb43181b7a7a34b26b8.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8fb9fa587ceb9fba65ae81534757f9f8.png"/>
    </p>
    <ul>
     <li>
      <strong>
       创建HTTP信息头管理器
      </strong>
     </li>
    </ul>
    <p>
     HTTP请求上 添加-&gt;配置原件-&gt;HTTP信息头管理器，然后填写请求信息头相关的数据。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9f15fa95955f0d73ec2c8ca731dbd77b.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1453c02da9a56951a617af4346034b1e.png"/>
    </p>
    <ul>
     <li>
      <strong>
       创建响应断言
      </strong>
     </li>
    </ul>
    <p>
     没有断言的测试，是没有意义的。所以需要根据响应结果添加断言。HTTP请求上添加-&gt;断言-&gt;响应断言。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/96b00e119289752c17d036f0e14b50ac.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/70e79d05b9df1a564e98570b0929631b.png"/>
     <br/>
     <br/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5c2430c94bf4af67e0578a51d5b14ef6.png"/>
    </p>
    <ul>
     <li>
      <strong>
       创建监听器（察看结果树、聚合报告）
      </strong>
     </li>
    </ul>
    <p>
     在线程组上右键添加-&gt;监听器-&gt;察看结构树、聚合报告。
    </p>
    <pre><code>察看结果树：能查看每个请求的详细数据。请求相关信息、响应相关数据。
聚合报告：查看整体所有请求的整体数据。
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce41e44b54277deff44c6e541b156e67.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/152b4420c22d51773c048733c8880292.png"/>

    <h3>
     <a id="Jmeter_118">
     </a>
     三、Jmeter常用组件
    </h3>
    <p>
     上面以一个简单的登录接口，讲解了Jemter的基本使用。下面以组件的分类，讲解一下常用的几大类组件。按照组件执行顺序，依次为：
     <code>
      配置原件-&gt;前置处理器-&gt;定时器-&gt;取样器-&gt;后置处理器-&gt;断言-&gt;监听器
     </code>
     。
     <code>
      同一层级的，顺序执行
     </code>
     。
    </p>
    <h5>
     <a id="31__120">
     </a>
     3.1 配置原件
    </h5>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7cb9b9b1af55654e94a0ed46c2341ef8.png"/>
     <br/>
     重点介绍一下
     <code>
      CSV数据文件设置
     </code>
     这个配置原件，因为参数化在性能测试中几乎是必不可少的。因为在实际相关接口性能测试时，不太可能每次请求都是同一个请求参数。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        设置CSV数据文件
       </strong>
       <br/>

    <pre><code>参考链接：
https://www.jianshu.com/p/d47a1f67c959
https://blog.csdn.net/quiet_girl/article/details/50724313
https://www.cnblogs.com/tudou-22/p/9566894.html

引用名称：下一个请求中引用的参数名称
正则表达式：
    () 括起来的部分为提取的内容
    . 匹配任意字符串
    + 匹配一次或多次
    ? 找到第一个匹配项之后就停止匹配
    \d 匹配数字
    \n 匹配结尾，例如匹配响应头中一行的数据
    带有转义字符的：id?test\":"1234"   id\?test\\":"(.+?)" 提取出1234
模板：用$$引用起来,$1$表示将匹配到的第一个值赋给引用名称、同理可以使用$1$$4$是将
      第一个和第四个值赋给引用名称。例如引用名称为user,模板为$1$$4$,后续取第一个
      正则的值就是user_g1,第四个就是user_g4.
匹配数字：0代表随机取值、1代表全部取值
缺省值：即为正则表达式没有匹配到值，则使用缺省值给引用名
</code></pre>
    <pre><code>实际举例
