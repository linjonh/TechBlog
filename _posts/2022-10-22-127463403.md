     一、多用户管理
    </h5>
    <h6>
     <a id="1_7">
     </a>
     1、新建/删除用户
    </h6>
    <p>
     参考：
     <a href="https://dontla.blog.csdn.net/article/details/126474694?spm=1001.2101.3001.6661.1&amp;utm_medium=distribute.pc_relevant_t0.none-task-blog-2%7Edefault%7ECTRLIST%7ERate-1-126474694-blog-106600088.pc_relevant_aa_2&amp;depth_1-utm_source=distribute.pc_relevant_t0.none-task-blog-2%7Edefault%7ECTRLIST%7ERate-1-126474694-blog-106600088.pc_relevant_aa_2&amp;utm_relevant_index=1" rel="nofollow">
      Linux下useradd命令与adduser命令的区别
     </a>
    </p>
    <p>
     linux新建用户命令有两个：useradd和adduser。这两个命令略有区别，总结如下：
    </p>
    <p>
     1、adduser会自动创建用户组，useradd不会自动创建，需要加参数
    </p>
    <p>
     2、adduser会自动创建用户目录，useradd不会自动创建，需要加参数
    </p>
    <p>
     3、adduser会自动选择shell版本，useradd不会自动选择
    </p>
    <p>
     4、adduser会提示添加用户密码，useradd不会
    </p>
    <p>
     推荐使用adduser来创建新用户，设置完密码之后，一切默认即可：
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> adduser test_user
</code></pre>
    <p>
     参考：
     <a href="https://zzhao.blog.csdn.net/article/details/50932747?spm=1001.2101.3001.6650.1&amp;utm_medium=distribute.pc_relevant.none-task-blog-2%7Edefault%7ECTRLIST%7ERate-1-50932747-blog-104425457.pc_relevant_3mothn_strategy_recovery&amp;depth_1-utm_source=distribute.pc_relevant.none-task-blog-2%7Edefault%7ECTRLIST%7ERate-1-50932747-blog-104425457.pc_relevant_3mothn_strategy_recovery&amp;utm_relevant_index=2" rel="nofollow">
      Linux的userdel和deluser命令
     </a>
    </p>
    <p>
     类似地，删除用户命令也有两个：userdel和deluser，区别如下：
    </p>
    <p>
     1、deluser的选项要比userdel丰富，功能要更强大
    </p>
    <p>
     2、deluser可以删除普通用户、从系统中删除用户组、将用户从一个组中删除
    </p>
    <p>
     推荐使用usedel来删除用户，并使用-r选项删除用户目录：
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">userdel</span> <span class="token parameter variable">-r</span> test_user
</code></pre>
    <h6>
     <a id="2_41">
     </a>
     2、更改用户权限
    </h6>
    <p>
     参考：
     <a href="https://blog.csdn.net/Happy_zlh/article/details/121076321">
      Linux下添加用户并赋予root权限三种方法
     </a>
    </p>
    <p>
     对于新建的用户可能需要赋予root权限，推荐使用如下方法：
    </p>
    <p>
     1、vim /etc/sudoers ，找到下面内容
    </p>
    <pre><code class="prism language-shell"><span class="token comment">## Allow root to run any commands anywhere</span>
root    <span class="token assign-left variable">ALL</span><span class="token operator">=</span><span class="token punctuation">(</span>ALL<span class="token punctuation">)</span>     ALL
</code></pre>
    <p>
     2、在root后添加如下内容
    </p>
    <pre><code class="prism language-shell">test_user   <span class="token assign-left variable">ALL</span><span class="token operator">=</span><span class="token punctuation">(</span>ALL<span class="token punctuation">)</span>     ALL
</code></pre>
    <p>
     3、修改后的文件内容如下，此时通过sudo命令，test_user即可获得root权限
    </p>
    <pre><code class="prism language-shell"><span class="token comment">## Allow root to run any commands anywhere</span>
root    	<span class="token assign-left variable">ALL</span><span class="token operator">=</span><span class="token punctuation">(</span>ALL<span class="token punctuation">)</span>     ALL
test_user   <span class="token assign-left variable">ALL</span><span class="token operator">=</span><span class="token punctuation">(</span>ALL<span class="token punctuation">)</span>     ALL
</code></pre>
    <h5>

     二、服务器环境配置
    </h5>
    <h6>
     <a id="1NVIDIA_70">
     </a>
     1、安装NVIDIA驱动
    </h6>
    <p>
     对于深度学习服务器，可以使用nvidia-smi命令查看显卡使用情况，这就需要安装NVIDIA驱动。
    </p>
    <p>
     1、去
     <a href="https://www.nvidia.cn/Download/index.aspx?lang=cn" rel="nofollow">
      官网
     </a>
     下载显卡对应的驱动版本，先选择显卡版本，然后点击搜索和下载即可
    </p>
    <p>
     <img alt="image-20221022011831568" src="https://i-blog.csdnimg.cn/blog_migrate/79d99308da1eea7c4e43bd7da4079c72.png"/>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/3db7746135cf9ffe682fe724a07d20c9.png"/>
    </p>
    <p>
     2、上传驱动程序至服务器，并赋予可执行权限
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">chmod</span> +x NVIDIA-Linux-x86_64-515.76.run
</code></pre>
    <p>
     3、安装驱动程序，然后按照提示安装即可
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> ./NVIDIA-Linux-x86_64-515.76.run
</code></pre>
    <p>
     安装过程可能遇到如下问题，对应解决方案如下：
    </p>
    <p>
     参考：
     <a href="https://blog.csdn.net/zhangzeyuan56/article/details/122624768">
      安装 NVIDIA 显卡驱动时报错：An NVIDIA kernel module ‘nvidia-drm‘ appears to already be load
     </a>
    </p>
    <p>
     1、An NVIDIA kernel module ‘nvidia-drm‘ appears to already be load
    </p>
    <p>
     原因：因为安装的是Ubuntu 图形化版本，之前也安装了 NVIDIA 驱动和 CUDA，系统加载图形化界面后，就会自动加载运行 NVIDIA 相关模块（如：nvidia-drm、nvidia-modeset 等）。
    </p>
    <p>
     解决方法：启动 Ubuntu 系统时不让系统加载图形化界面 。在终端命令模式下卸载 NVIDIA 驱动，再重装新的驱动。
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 1. 获得超级用户权限</span>
<span class="token function">sudo</span> <span class="token parameter variable">-i</span>
<span class="token comment"># 2. 关闭所有使用 GPU 的进程</span>
<span class="token function">sudo</span> systemctl isolate multi-user.target
<span class="token function">sudo</span> modprobe <span class="token parameter variable">-r</span> nvidia-drm
<span class="token comment"># 3. 重装 NVIDIA 驱动</span>
<span class="token function">sudo</span> ./NVIDIA-Linux-x86_64-XX.run 
<span class="token comment"># 4. 打开使用GPU的进程</span>
<span class="token function">sudo</span> systemctl set-default multi-user.target  
<span class="token comment"># 5. 重新启动</span>
<span class="token function">sudo</span> <span class="token function">reboot</span>
</code></pre>
    <p>
     2、缺失gcc或make
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> gcc
<span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> <span class="token function">make</span>
</code></pre>
    <p>
     3、x server或nouveau相关
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> ./NVIDIA-Linux-x86_64-XX.run --no-x-check --no-nouveau-check
</code></pre>
    <p>
     4、貌似安装驱动时加上 --no-opengl-files可以避免重启后nvidia-smi命令失效，就不需要修改内核啦！
    </p>
    <p>
     5、如果出现ERROR: Unable to load the kernel module ‘nvidia.ko’. 安装输出的日志，可能是gcc原因，也可能是nouveau的原因。如果是nouveau的原因，可参考
     <a href="https://blog.csdn.net/qxqxqzzz/article/details/112005275">
      TITAN RTX安装驱动、cuda10 cudnn神报错： ERROR: Unable to load the kernel module ‘nvidiko‘
     </a>
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 查看lightdm和dgm状态</span>
<span class="token function">sudo</span> <span class="token function">service</span> lightdm status
<span class="token function">service</span> gdm status
<span class="token comment"># 如果正在运行，就stop；否则忽略即可</span>
<span class="token function">sudo</span> <span class="token function">service</span> lightdm stop
<span class="token function">service</span> gdm stop

<span class="token comment"># 修改/etc/modprobe.d/blacklist.conf，末尾添加如下两行：</span>
blacklist nouveau
options nouveau <span class="token assign-left variable">modeset</span><span class="token operator">=</span><span class="token number">0</span>

<span class="token comment"># update</span>
<span class="token function">sudo</span> update-initramfs <span class="token parameter variable">-u</span>
<span class="token comment"># 重启</span>
<span class="token function">sudo</span> <span class="token function">reboot</span>
<span class="token comment"># 查看nouveau状态，如果没有输出，则说明ok了</span>
lsmod <span class="token operator">|</span> <span class="token function">grep</span> nouveau

</code></pre>
    <h6>
     <a id="2CUDA_Toolkit_154">
     </a>
     2、安装CUDA Toolkit
    </h6>
    <p>
     1、去
     <a href="https://developer.nvidia.com/cuda-toolkit-archive" rel="nofollow">
      官网
     </a>
     选择要安装的版本
    </p>
    <p>
     2、以cuda 11.1为例，选择系统配置
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/1e1627ef2179fd116e1abccf1337468b.png"/>
    </p>
    <p>
     3、执行安装命令，由于前面已经安装了nvidia驱动，所以在安装选项里要取消Driver
    </p>
    <pre><code class="prism language-shell"><span class="token function">wget</span> https://developer.download.nvidia.com/compute/cuda/11.1.1/local_installers/cuda_11.1.1_455.32.00_linux.run
<span class="token function">sudo</span> <span class="token function">sh</span> cuda_11.1.1_455.32.00_linux.run
</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/57adf6677c8c6df57c727fc6a87a11e4.png"/>
    </p>
    <p>
     4、将cuda写入环境变量，由于是多用户系统，推荐在/etc/profile文件（所有用户均生效）的末尾写入，也可以在~/.bashrc文件（仅当前用户生效）的末尾写入
    </p>
    <pre><code class="prism language-shell"><span class="token comment">#####################cuda11.1#######################</span>
<span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token environment constant">$PATH</span>:/usr/local/cuda-11.1/bin
<span class="token builtin class-name">export</span> <span class="token assign-left variable">LD_LIBRARY_PATH</span><span class="token operator">=</span><span class="token variable">$LD_LIBRARY_PATH</span>:/usr/local/cuda-11.1/lib64
</code></pre>
    <p>
     5、source刷新环境变量
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">source</span> /etc/profile
</code></pre>
    <p>
     安装过程可能遇到如下问题，对应解决方案如下：
    </p>
    <p>
     1、gcc版本太高
    </p>
    <p>
     推荐使用gcc 9.4.0，也可以根据提示no-check
    </p>
    <p>
     参考：
     <a href="https://blog.csdn.net/u010875635/article/details/70170140">
      Ubuntu中gcc、 g++安装与卸载
     </a>
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 1.卸载</span>
<span class="token function">sudo</span> <span class="token function">apt-get</span> remove gcc gcc-xx   <span class="token comment">#可能有多个版本，都要删掉</span>
<span class="token function">sudo</span> <span class="token function">apt-get</span> remove g++
<span class="token comment"># 2.安装gcc、g++</span>
<span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> gcc <span class="token comment"># 默认安装的是gcc 9.4.0</span>
<span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> build-essential
</code></pre>
    <h6>
     <a id="3cuDNN_201">
     </a>
     3、安装cuDNN
    </h6>
    <p>
     1、去
     <a href="https://developer.nvidia.com/rdp/cudnn-archive" rel="nofollow">
      官网
     </a>
     选择要安装的版本
    </p>
    <p>
     2、以cuda 11.1和cuDNN8.9.7为例，选择对于文件
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5af12d15a9b34cc9a00c968367f09449.png">
      <br/>
      3、下载后解压，然后复制文件到指定位置。具体可参考
      <a href="https://www.cnblogs.com/lycnight/p/17768264.html" rel="nofollow">
       cuDNN安装过程记录
      </a>
     </img>
    </p>
    <pre><code class="prism language-shell"><span class="token function">tar</span> <span class="token parameter variable">-xvf</span> cudnn-linux-x86_64-8.9.7.29_cuda11-archive.tar.xz
<span class="token builtin class-name">cd</span> cudnn-linux-x86_64-8.9.7.29_cuda11-archive/
<span class="token function">sudo</span> <span class="token function">cp</span> lib/* /usr/local/cuda-11.1/lib64/
<span class="token function">sudo</span> <span class="token function">cp</span> include/* /usr/local/cuda-11.1/include/
<span class="token function">sudo</span> <span class="token function">chmod</span> a+r /usr/local/cuda-11.1/lib64/*
<span class="token function">sudo</span> <span class="token function">chmod</span> a+r /usr/local/cuda-11.1/include/*

<span class="token comment"># 验证</span>
<span class="token function">cat</span> /usr/local/cuda-11.1/include/cudnn_version.h <span class="token operator">|</span> <span class="token function">grep</span> CUDNN_MAJOR <span class="token parameter variable">-A</span> <span class="token number">2</span>
<span class="token comment"># 老版本使用</span>
<span class="token function">cat</span> /usr/local/cuda-11.1/include/cudnn.h <span class="token operator">|</span> <span class="token function">grep</span> CUDNN_MAJOR <span class="token parameter variable">-A</span> <span class="token number">2</span>
</code></pre>
    <p>
     4、完成后使用apt install可能会出现
     <strong>
      libcudnn_cnn_train.so.8 is not a symbolic link
     </strong>
     <br/>
     参考
     <a href="https://blog.csdn.net/qq_36577574/article/details/119174973">
      libcudnn_cnn_train.so.8 is not a symbolic link此类问题解决方法
     </a>
     ，新建软链接解决
    </p>
    <h6>
     <a id="4_noopenglfiles_223">
     </a>
     4、配置启动内核 （可选，如果安装驱动时加上了no-opengl-files就不用做）
    </h6>
    <p>
     在安装好NVIDIA驱动后，如果重启服务器，可能出现一种奇怪的现象：nvidia-smi命令无法使用，NVIDIA-SMI has failed because it couldn‘t communicate with the NVIDIA driver。
    </p>
    <p>
     这是由于系统内核版本过高，导致NVIDIA驱动失效，这时候就需要配置启动内核，步骤如下：
    </p>
    <p>
     1、查看当前内核版本，默认是5.15.0
    </p>
    <pre><code class="prism language-shell"><span class="token function">uname</span> <span class="token parameter variable">-a</span>
</code></pre>
    <p>
     2.安装旧的内核，亲测5.11.0-34-generic可用
    </p>
    <p>
     参考：
     <a href="https://blog.csdn.net/qq_43222384/article/details/125999313?utm_medium=distribute.pc_relevant.none-task-blog-2~default~baidujs_baidulandingword~default-0-125999313-blog-116157797.t0_edu_mix&amp;spm=1001.2101.3001.4242.1&amp;utm_relevant_index=3">
      UBUNTU 18.04安装指定内核
     </a>
    </p>
    <pre><code class="prism language-shell"><span class="token function">apt-get</span> <span class="token function">install</span> linux-image-5.11.0-34-generic linux-headers-5.11.0-34-generic linux-modules-extra-5.11.0-34-generic
</code></pre>
    <p>
     如果安装成功，使用
    </p>
    <pre><code class="prism language-shell">dpkg <span class="token parameter variable">--list</span> <span class="token operator">|</span> <span class="token function">grep</span> linux-image
</code></pre>
    <p>
     就可以从列表中看到 linux-image-5.11.0-34-generic
    </p>
    <p>
     3.修改系统默认启动内核
    </p>
    <p>
     需要修改/etc/default/grub文件来设置系统启动的内核，该配置文件默认内容如下：
    </p>
    <pre><code class="prism language-shell"><span class="token assign-left variable">GRUB_DEFAULT</span><span class="token operator">=</span><span class="token number">0</span> <span class="token comment"># 默认内核启动项</span>
<span class="token assign-left variable">GRUB_TIMEOUT_STYLE</span><span class="token operator">=</span>hidden <span class="token comment"># 系统启动时菜单是否显示</span>
<span class="token assign-left variable">GRUB_TIMEOUT</span><span class="token operator">=</span><span class="token number">0</span>
<span class="token assign-left variable">GRUB_DISTRIBUTOR</span><span class="token operator">=</span><span class="token variable"><span class="token variable">`</span>lsb_release <span class="token parameter variable">-i</span> <span class="token parameter variable">-s</span> <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span> /dev/null <span class="token operator">||</span> <span class="token builtin class-name">echo</span> Debian<span class="token variable">`</span></span>
<span class="token assign-left variable">GRUB_CMDLINE_LINUX_DEFAULT</span><span class="token operator">=</span><span class="token string">"quiet splash"</span>
<span class="token assign-left variable">GRUB_CMDLINE_LINUX</span><span class="token operator">=</span><span class="token string">""</span>
</code></pre>
    <p>
     我们要修改的就是第一项，那么如何修改第一项使得我们的启动内核变为我们想要的版本呢？有三种方式：
    </p>
    <ol>
     <li>
      <p>
       在重启时，快速按
       <code>
        ESC
       </code>
       进入GNU grub界面，通过启动菜单来观察我们需要的内核，可参考：
       <a href="https://blog.csdn.net/weixin_43744613/article/details/124248416?spm=1001.2101.3001.6650.5&amp;utm_medium=distribute.pc_relevant.none-task-blog-2%7Edefault%7EBlogCommendFromBaidu%7ERate-5-124248416-blog-87937269.t0_edu_mix&amp;depth_1-utm_source=distribute.pc_relevant.none-task-blog-2%7Edefault%7EBlogCommendFromBaidu%7ERate-5-124248416-blog-87937269.t0_edu_mix&amp;utm_relevant_index=7">
        Ubuntu指定默认Linux内核启动
       </a>
      </p>
      <p>
       但是这种方法只适合于本地机器，对于服务器来说，我们通常不会到机房去直接连接机器进行操作。
      </p>
     </li>
     <li>
      <p>
       直接在GRUB_DEFAULT中指定内核版本，可参考：
       <a href="https://www.cnblogs.com/daner1257/p/12021945.html" rel="nofollow">
        Ubuntu设置开机时启动的系统内核版本 &amp; 安装低版本内核
       </a>
      </p>
      <p>
       这种方法直接修改配置文件第一行为：
      </p>
      <pre><code class="prism language-shell"><span class="token assign-left variable">GRUB_DEFAULT</span><span class="token operator">=</span><span class="token string">"Advanced options for Ubuntu&gt;Ubuntu, with Linux 5.11.0-34-generic"</span>
</code></pre>
      <p>
       注意，如果按照这种方式修改更新并重启后，发现系统内核版本并未切换成功，可能是因为你的系统语言是中文。所以需将第一行修改为：
      </p>
      <pre><code class="prism language-shell"><span class="token assign-left variable">GRUB_DEFAULT</span><span class="token operator">=</span><span class="token string">"Ubuntu 高级选项&gt;Ubuntu，Linux 5.11.0-34-generic"</span>
</code></pre>
     </li>
     <li>
      <p>
       由于第一种方法需要连接机器来实现，那有没有办法不连接机器就可以知道启动菜单呢？当然是有的，这就是第三种方法。可参考：
       <a href="https://blog.csdn.net/bby1987/article/details/104264285/?utm_medium=distribute.pc_relevant.none-task-blog-2~default~baidujs_utm_term~default-1--blog-114484812.pc_relevant_3mothn_strategy_recovery&amp;spm=1001.2101.3001.4242.2&amp;utm_relevant_index=4">
        Ubuntu修改默认启动的内核版本（grub）
       </a>
      </p>
      <p>
       首先查看启动菜单的配置文件
      </p>
      <pre><code class="prism language-shell"><span class="token function">cat</span> /boot/grub/grub.cfg <span class="token operator">|</span> <span class="token function">grep</span> menuentry
<span class="token comment"># 结果如下：</span>
menuentry <span class="token string">'Ubuntu'</span> 
submenu <span class="token string">'Advanced options for Ubuntu'</span> 
        menuentry <span class="token string">'Ubuntu, with Linux 5.15.0-50-generic'</span>
        menuentry <span class="token string">'Ubuntu, with Linux 5.15.0-50-generic (recovery mode)'</span> 
        menuentry <span class="token string">'Ubuntu, with Linux 5.15.0-48-generic'</span> 
        menuentry <span class="token string">'Ubuntu, with Linux 5.15.0-48-generic (recovery mode)'</span> 
        menuentry <span class="token string">'Ubuntu, with Linux 5.11.0-34-generic'</span> 
        menuentry <span class="token string">'Ubuntu, with Linux 5.11.0-34-generic (recovery mode)'</span> 
menuentry <span class="token string">'UEFI Firmware Settings'</span>
</code></pre>
      <p>
       在上述显示信息的情况下，说明grub开机启动界面的菜单结构理应如下：
      </p>
      <pre><code class="prism language-txt">*Ubuntu # *表示一级菜单
*Advanced options for Ubuntu
**Ubuntu, with Linux 5.15.0-50-generic # **表示二级菜单
**Ubuntu, with Linux 5.15.0-50-generic (recovery mode)
**Ubuntu, with Linux 5.15.0-48-generic
**Ubuntu, with Linux 5.15.0-48-generic (recovery mode)
**Ubuntu, with Linux 5.11.0-34-generic
**Ubuntu, with Linux 5.11.0-34-generic (recovery mode)
*UEFI Firmware Settings
</code></pre>
      <p>
       所以我们需要将配置文件第一行修改为：
      </p>
      <pre><code class="prism language-shell"><span class="token assign-left variable">GRUB_DEFAULT</span><span class="token operator">=</span><span class="token string">"1&gt;4"</span> <span class="token comment"># 表示进入第1个一级菜单下的第4个二级菜单</span>
</code></pre>
      <p>
       （菜单的起始索引为0，所以"Advanced options for Ubuntu"对应的就是0， "Ubuntu, with Linux 5.11.0-34-generic"对应的就是4）
      </p>
     </li>
    </ol>
    <p>
     4.更新配置文件并重启
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">update-grub</span>
<span class="token function">sudo</span> <span class="token function">reboot</span>
</code></pre>
    <p>
     重启完成后，可使用
    </p>
    <pre><code class="prism language-shell"><span class="token function">uname</span> <span class="token parameter variable">-a</span>
</code></pre>
    <p>
     再次查看内核版本是否正常切换
    </p>
    <h6>
     <a id="5_339">
     </a>
     5、挂载硬盘
    </h6>
    <p>
     在配置深度学习服务器时，除了系统盘外，还会存在多个数据盘，为了方便访问，需要将这些数据盘挂载到指定目录。可参考：
     <a href="https://blog.csdn.net/weixin_44578029/article/details/118464216">
      Ubuntu硬盘的挂载（临时、永久挂载）
     </a>
    </p>
    <p>
     1、临时挂载
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">mkdir</span> /data <span class="token comment"># 在根目录下新建目录data作为挂载目录</span>
<span class="token function">sudo</span> <span class="token function">mount</span> /dev/sdb1 /data <span class="token comment"># 将硬盘/dev/sdb1挂载到/data目录，可通过sudo fdisk -l | grep /dev来查看磁盘情况</span>
</code></pre>
    <p>
     2、开机自动挂载
    </p>
    <p>
     修改/etc/fstab文件，添加如下内容：
    </p>
    <pre><code class="prism language-shell">/dev/sdb1     /data  auto   defaults    <span class="token number">0</span>       <span class="token number">0</span>
</code></pre>
    <p>
     命令格式为：
    </p>
    <p>
     /dev/device mountpoint type rules dump order
    </p>
    <p>
     设备名称 挂载点 分区类型 挂载选项 dump选项 fsck选项
    </p>
    <p>
     各参数含义如下：
    </p>
    <p>
     设备名称 挂载点 分区类型 挂载选项 dump选项 fsck选项
    </p>
    <ol>
     <li>
      设备名称可以通过sudo fdisk -l | grep /dev来查看
     </li>
     <li>
      挂载点即创建的挂载目录
     </li>
     <li>
      分区类型指的是该磁盘的文件系统
     </li>
     <li>
      default的意义是在于按照大多数文件系统的缺省值设置挂载定义，即系统的默认定义（对于大多数文件系统的处理方式），其实除了default还有auto就是开机自动挂载，noauto就是开机不自动挂载，nouser就是只有超级用户才挂载，user，所有用户都挂载
     </li>
     <li>
      dump 就是从不备份
     </li>
     <li>
      fsck的检查顺序，0为不检查，分区为1，其他分区从2开始，这里指的是检查的优先级
     </li>
    </ol>

     1、配置ssh
    </h6>
    <p>
     配置ssh是服务器配置的第一步，只有这样，才可以通过ssh命令远程访问服务器。
    </p>

     <a href="https://zhuanlan.zhihu.com/p/191627275" rel="nofollow">

     </a>
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> openssh-server <span class="token parameter variable">-y</span>
<span class="token function">sudo</span> systemctl status <span class="token function">ssh</span> <span class="token comment">#查看状态</span>

    <h6>
     <a id="2xrdp_489">
     </a>
     2、配置xrdp
    </h6>
    <p>
     当配置ssh之后，也只能使用终端来远程访问服务器，如果要使用图形界面，例如Windows的远程桌面来访问就需要配置xrdp。可参考：
     <a href="https://developer.aliyun.com/article/762186" rel="nofollow">
      如何在Ubuntu 20.04 上安装 Xrdp 服务器（远程桌面）
     </a>
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> xrdp <span class="token comment"># 安装xrdp</span>
<span class="token function">sudo</span> systemctl status xrdp <span class="token comment"># 查看xrdp状态</span>
<span class="token function">sudo</span> adduser xrdp ssl-cert <span class="token comment"># 将xrdp用户添加到"ssl-cert"用户组</span>
<span class="token function">sudo</span> systemctl restart xrdp <span class="token comment"># 重启xrdp服务</span>
</code></pre>
    <h6>
     <a id="3frp_500">
     </a>
     3、配置frp内网穿透
    </h6>
    <p>
     由于服务器一般只限于局域网内访问，这就使得外网远程访问并不友好，可以通过frp配置内网穿透来实现外网访问服务器。可参考：
     <a href="https://blog.csdn.net/ck784101777/article/details/108724672">
      ubuntu frp方式，实现内网穿透
     </a>
    </p>
    <p>
     1、准备
    </p>
    <p>
     需要准备一台可以正常访问的具有公网ip的外网服务器，可以购买阿里云、腾讯云等云服务器。
    </p>
    <p>
     外网服务器需要开启相应的访问端口，包括：连接端口和转发映射端口。
    </p>
    <p>
     在准备好外网服务器之后，去
     <a href="https://github.com/fatedier/frp/releases">
      GitHub
     </a>
     下载对应的程序包。
    </p>
    <p>
     <img alt="image-20221022155315983" src="https://i-blog.csdnimg.cn/blog_migrate/7e0bf9d174615c9cf83d2a829025aaa9.png"/>
    </p>
    <p>
     2、配置frps服务器端
    </p>
    <ol>
     <li>
      <p>
       将程序包上传至外网服务器，并解压
      </p>
      <pre><code class="prism language-shell"><span class="token function">tar</span> <span class="token parameter variable">-xzvf</span> frp_0.44.0_linux_amd64.tar.gz
</code></pre>
     </li>
     <li>
      <p>
       配置frps.ini文件
      </p>
      <pre><code class="prism language-ini">[common]
bind_port = 7000 # 外网服务器和本地服务器连接的端口
</code></pre>
     </li>
     <li>
      <p>
       启动frps服务器
      </p>
      <pre><code class="prism language-shell"><span class="token function">nohup</span> ./frps <span class="token parameter variable">-c</span> frps.ini <span class="token operator">&gt;</span>frps.log <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span> <span class="token operator">&amp;</span>
</code></pre>
     </li>
    </ol>
    <p>
     3、配置frpc客户端
    </p>
    <ol>
     <li>
      <p>
       将程序包上传至本地网服务器，并解压
      </p>
      <pre><code class="prism language-shell"><span class="token function">tar</span> <span class="token parameter variable">-xzvf</span> frp_0.44.0_linux_amd64.tar.gz
</code></pre>
     </li>
     <li>
      <p>
       配置frpc.ini文件
      </p>
      <pre><code class="prism language-ini">[common]
server_addr = 外网服务器公网ip
server_port = 7000

[ssh-22] # 转发映射22端口
type = tcp
local_ip = 127.0.0.1
local_port = 22 # 本地服务器的22端口
remote_port = 2222 # 外网服务器的2222端口

[ssh-3389] # 转发映射3389端口
type = tcp
local_ip = 127.0.0.1
local_port = 3389 # 本地服务器的3389端口
remote_port = 3390 # 外网服务器的3390端口
</code></pre>
     </li>
     <li>
      <p>
       启动frpc客户端
      </p>
      <pre><code class="prism language-shell"><span class="token function">nohup</span> ./frpc <span class="token parameter variable">-c</span> frpc.ini <span class="token operator">&gt;</span>frpc.log <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span> <span class="token operator">&amp;</span>
</code></pre>
     </li>
    </ol>
    <h6>
     <a id="4frp_569">
     </a>
     4、配置frp开机自启
    </h6>
    <p>
     nohup启动只能保证frp服务会一直挂载在后台，如果服务器重启，frp服务也会中断。因此需要配置开机自启，以本地服务器配置frpc开机自启为例。可参考：
     <a href="https://www.tufei.site/archives/45/" rel="nofollow">
      frp设置开机自启的几种方法
     </a>
    </p>
    <p>
     1、安装Supervisor
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> supervisor
</code></pre>
    <p>
     2、配置frpc服务
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">vim</span> /etc/supervisor/conf.d/frpc.conf
</code></pre>
    <p>
     文件内容如下：
    </p>
    <pre><code class="prism language-shell"><span class="token punctuation">[</span>program:frpc<span class="token punctuation">]</span>
<span class="token builtin class-name">command</span> <span class="token operator">=</span> /your_frp_path/frpc <span class="token parameter variable">-c</span> /your_frp_path/frpc.ini
autostart <span class="token operator">=</span> <span class="token boolean">true</span>
</code></pre>
    <p>
     记得给frpc chmod +x 赋予可执行权限
    </p>
    <p>
     3、重启Supervisor
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 重启supervisor</span>
<span class="token function">sudo</span> systemctl restart supervisor
<span class="token comment"># 查看supervisor运行状态</span>
<span class="token function">sudo</span> supervisorctl status
</code></pre>
    <h6>
     <a id="5_602">
     </a>
     5、配置仅密钥登录
    </h6>
    <p>
     服务器使用ssh密码，在开放外网登录之后，很容易遭到攻击。建议改为仅使用密钥登录。可参考
     <a href="https://blog.csdn.net/qq_51756940/article/details/130402945">
      ubuntu使用秘钥登录
     </a>
     。
     <br/>
     1、生成密钥：
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 1.生成SSH密钥对, 默认在当前用户目录下.ssh目录</span>
ssh-keygen
<span class="token comment"># 2.将公钥添加到授权密钥文件</span>
<span class="token function">cat</span> ~/.ssh/id_rsa.pub <span class="token operator">&gt;&gt;</span> ~/.ssh/authorized_keys
<span class="token comment"># 3.更改权限</span>
<span class="token function">chmod</span> <span class="token number">600</span> ~/.ssh/authorized_keys
<span class="token comment"># 用户目录下的.ssh/id_rsa就是私钥，用这个登录，拷贝到本地就可以删除服务器上的了。 </span>

</code></pre>
    <p>
     2、禁用密码登录
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> <span class="token function">vim</span> /etc/ssh/sshd_config
</code></pre>
    <p>
     以下两项分别设置为no和yes，表示禁用密码，启用密钥。
    </p>
    <pre><code class="prism language-txt">PasswordAuthentication no
PubkeyAuthentication yes
</code></pre>
    <p>
     3、重启SSH服务
    </p>
    <pre><code class="prism language-shell"><span class="token function">sudo</span> systemctl restart sshd
</code></pre>
