    <p>
     华三模拟器可以模拟交换机、路由器、防火墙等，由于自带的PC功能有限，因此需要连接到VMware虚拟机或实体机以模拟真实的PC或者FTP\HTTP服务器设备。另外也可以使其连接到互联网，以进行防火墙等的配置。详细思路如下：
    </p>
    <p>
     本次教程以Win10、HCL3.0.1为例，理论上所有win10版本、win7通用。
    </p>
    <blockquote>
     <p>
      虚拟机下载地址：
      <br/>

      <br/>

     </p>
    </blockquote>

    <h4>
     <a id="HCL_48">
     </a>
     二、HCL与主机的连接
    </h4>
    <h5>
     <a id="768M1024_49">
     </a>
     注意！！添加防火墙后，一定要将防火墙的内存更改为768M或以上（我改的1024），否则会出现一系列问题
    </h5>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0fd45bb9e0611a626a24ecc5873f5f6b.png"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3e0039889d5c75ed6a35d39f1595842d.png"/>
    </p>
    <p>
     1.添加F1090防火墙，并添加一个Host，将刚才添加的第一个虚拟网卡连接到G0/0口（可自己定义）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e2c83e7dad4eac4b2948fb8876d5195b.png"/>
     <br/>
     2.进入防火墙shell，配置0/0口的ip，可自行定义，这里以10.0.8.1为例
    </p>
    <pre><code>[H3C]int g 1/0/0
[H3C-GigabitEthernet1/0/0]ip add 10.0.8.1 24
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9d857482a9886c7f025edf573fe9dabc.png"/>
     <br/>
     3.回到电脑的网络适配器界面 提示：可在“此电脑”的地址栏输入
    </p>
    <pre><code>控制面板\网络和 Internet\网络连接
</code></pre>
    <p>
     然后双击带有“Microsoft KM-TEST环回适配器”的适配器，点击“属性”，再双击“Internet协议版本4(TCP/IPv4)”，IP设置为10.0.8.x，设置网关、掩码。如图
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/020c937b28f489fcd214a2ac95a50c48.png"/>
     <br/>
     最后点击确认。
     <br/>
     4.防火墙添加此端口到DMZ区域
    </p>
    <pre><code>[H3C]sec name dmz
[H3C-security-zone-DMZ]im int g 1/0/0
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/976970571483c16741f690b1760c51e7.png"/>
     <br/>
     5.创建pass对象策略
    </p>
    <pre><code>[H3C]object-policy ip pass
[H3C-object-policy-ip-pass]rule 0 pass
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b2838ce8467dbc9a62c73e49aa11cae1.png"/>
     <br/>
     6.在dmz-&gt;local方向应用策略
    </p>
    <pre><code>[H3C]zone-pair security source dmz destination local
[H3C-zone-pair-security-DMZ-Local]object-policy apply ip pass
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9df9344fc83143e160ff77256917006e.png"/>
     <br/>
     7.检测当前PC机能否连接到防火墙：
     <br/>
     在cmd界面，ping防火墙的网关10.0.8.1
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fcc77c6415abdfda983ee3962a81e03d.png"/>
     <br/>
     如图显示，能够ping通。
     <br/>
     然后电脑浏览器打开https://10.0.8.1/ 能够看到如下所示界面，用户名密码均为admin，可以尝试登陆，但会要求修改密码，自行决定是否修改（此步无特别意义，仅作验证）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/472e8153ea4a54bbedf3881dadfd867f.png"/>
    </p>
    <h4>
     <a id="VmwareHCL_93">
     </a>
     三、Vmware虚拟机与HCL防火墙的连接
    </h4>
    <p>
     1.回到HCL，添加Host，将二号虚拟网卡连接到G0/1口，此口为trust区域，用来连接vmware虚拟机
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/dbfff0f6549c17c14dda10a4c9419457.png"/>
     <br/>
     2.配置G0/1的ip地址和添加trust区域，这里使用192.168.1.1
    </p>
    <pre><code>[H3C]int g 1/0/1
[H3C-GigabitEthernet1/0/1]ip add 192.168.1.1 24
[H3C]sec name trust
[H3C-security-zone-Trust]im int g 1/0/1
#Trust到Local域连通。
[H3C]zone-pair security source Trust destination Local
[H3C-zone-pair-security-Trust-Local]object-policy apply ip pass
[H3C-zone-pair-security-Trust-Local]quit
#Local到Trust域连通。
[H3C]zone-pair security source Local destination Trust
[H3C-zone-pair-security-Local-Trust]object-policy apply ip pass
[H3C-zone-pair-security-Local-Trust]quit
</code></pre>
    <p>
     3.DHCP配置
    </p>
    <pre><code>[H3C]dhcp ena
[H3C]dhcp server ip-pool pool1 //新建DHCP地址池
[H3C-dhcp-pool-pool1]network  192.168.1.0 mask 255.255.255.0 //配置地址、掩码
[H3C-dhcp-pool-pool1]gateway-list  192.168.1.1//配置网关
[H3C-dhcp-pool-pool1]dns-list 114.114.114.114//配置DNS
[H3C]int g 1/0/1 //进入接口
[H3C-GigabitEthernet1/0/1]dhcp server apply ip-pool pool1
[H3C-GigabitEthernet1/0/1]dhcp sele ser
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/694061ceb5f45be71d82f33e543747c7.png"/>
     <br/>
     经过以上设置，G0/1口的DHCP已开启
     <br/>
     此时电脑打开http://192.168.1.1，应该也能进入防火墙web界面
     <br/>
     4.打开VMware，点击“编辑”-虚拟网络编辑器
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/753deb35ab07ef733d65f3d3ed7f5fae.png"/>
     <br/>
     5.添加一个网络（这里为Vmnet2），设置为桥接模式，使其连接到2号虚拟网卡，如下图所示，然后确认。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7774d8be0f55c5cf3270bc7eb1c3a603.png"/>
     <br/>
     6.打开虚拟机设置，在“网络适配器”的自定义栏选择刚才添加的VMnetX，这里是VMnet2
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ff5bc07e11884732d824cc48c2e27cea.png"/>
     <br/>
     6.打开虚拟机，此时虚拟机也能获取到192.168.1.0网段的IP地址
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/269b3890803ea61568a51f077dd2a747.png"/>
     <br/>
     如不行，请尝试重新添加网卡或者重启电脑
     <br/>
     注意：退出HCL前一定要保存HCL的设备配置，具体方法：HCL防火墙命令行界面ctrl+z回到用户模式
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e062da92c5def0ec2b627d26ad43ebdc.png"/>
     <br/>
     然后输入save，然后输入Y，回车，提示覆盖也按回车即可
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d130dc4a55fa33d72b478e3750264633.png"/>
     <br/>
     HCL这里也要保存工程
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/dddbe6008b112c584735a4ddf241994d.png"/>
    </p>
    <h4>
     <a id="_142">
     </a>
     三、配置外网访问
    </h4>
    <p>
     此步骤原理为直接连接到与物理机上网的网卡相同网络，如果是校园网，则还需登陆。建议使用路由器或手机开热点
     <br/>
     1.回到HCL，添加HOST，连接到当前上网的网卡，我这里是接的网线，也就是Realtek pciexxx，如果是无线网，请连接到WLAN网卡。
     <br/>
     将此HOST连接到防火墙的G0/2端口
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f62b8c2923b32f3954ae887a32b4fc76.png"/>
     <br/>
     2.防火墙配置G0/2端口的IP地址
     <br/>
     注意：要求与物理机的IP地址段一致。
     <br/>
     我这里查询到物理机上网所用网卡的地址段是192.168.123.93
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/72ebd4b820a39d9d04441ec8108bdc05.png"/>
     <br/>
     因此我将防火墙G0/2口的ip配置为192.168.123.95 注意不能冲突，同一网段其他地址也可以
    </p>
    <pre><code>[H3C]int g 1/0/2
[H3C-GigabitEthernet1/0/2]ip add 192.168.123.95 24
</code></pre>
    <p>
     然后将此端口加入Untrust区域
    </p>
    <pre><code>[H3C]sec name untrust
[H3C-security-zone-Untrust]im int g 1/0/2
</code></pre>
    <p>
     3.配置NAT地址转换
    </p>
    <pre><code>[H3C]int g 1/0/2  #外网口
[H3C-GigabitEthernet1/0/2]nat out
</code></pre>
    <p>
     4.配置到外网的缺省路由
    </p>
    <pre><code>[H3C]ip route-static 0.0.0.0 0 192.168.123.1
</code></pre>
    <p>
     注意，这里的192.168.123.1与你物理机的网段网关对应
     <br/>
     5.放通到untrust的访问权限
    </p>
    <pre><code>[H3C-GigabitEthernet1/0/2]zone-pair security source Local destination untrust
[H3C-zone-pair-security-Local-Untrust]object-policy apply ip pass
[H3C-zone-pair-security-Local-Untrust]zone-pair security source trust destination untrust
[H3C-zone-pair-security-Trust-Untrust]object-policy apply ip pass
[H3C-zone-pair-security-Trust-Untrust]zone-pair security source untrust destination dmz
[H3C-zone-pair-security-Untrust-DMZ]object-policy apply ip pass
</code></pre>
    <p>
     6.ASPF
    </p>
    <pre><code>[H3C-zone-pair-security-Untrust-DMZ]zone-pair security source trust destination untrust
[H3C-zone-pair-security-Trust-Untrust]aspf apply policy 1
</code></pre>
    <p>
     7.验证
     <br/>
     在防火墙shell界面ping测试路由网关，此时能ping通
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/07a4ad6884c1f401d1c528cb44b7aabc.png"/>
     <br/>
     此时虚拟机理论上能够访问互联网
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3f25edb6a611b25b7727e4e63f966d1b.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d8fda03a462e850ca2710137441e75c3.png"/>
    </p>
    <p>
     8.将外网IP访问映射到内网DMZ服务器
    </p>
    <pre><code>[H3C-GigabitEthernet1/0/2]nat server proto tcp global 192.168.123.95 inside 10.0.8.10  #192.168.123.95是防火墙外网地址，10.0.8.10是HTTP服务器地址
</code></pre>
    <p>
     9.在外网同网段的机器输入防火墙的外网地址192.168.123.95
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/428fd6e6ebea3aa228cf66b9d596709c.png"/>
     <br/>
     这里使用手机连接到与防火墙外网同一地址，能够通过防火墙外网地址打开模拟服务器的网站（建立在电脑上）
    </p>
    <p>
     最后配置完了记得保存配置
    </p>
