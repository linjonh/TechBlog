     RTMP协议发送H.264编码及AAC编码的音视频，实现摄像头直播

   </div>

    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     RTMP（Real Time Messaging Protocol）是专门用来传输音视频数据的
     <a href="http://www.rosoo.net/a/list_51_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      流媒体
     </a>
     <a href="http://www.rosoo.net/a/list_46_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      协议
     </a>
     ，最初由Macromedia 公司创建，后来归Adobe公司所有，是一种私有
     <a href="http://www.rosoo.net/a/list_46_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      协议
     </a>
     ，主要用来联系Flash Player和RtmpServer，如
     <span style="padding:0px; margin:0px; color:rgb(0,0,255)">
      FMS
     </span>
     ,
     <span style="padding:0px; margin:0px; color:rgb(0,0,255)">
      Red5
     </span>
     ,
     <span style="padding:0px; margin:0px; color:rgb(0,0,255)">
      crtmpserver
     </span>
     等。RTMP协议可用于实现直播、点播应用，通过
     <span style="padding:0px; margin:0px; color:rgb(0,0,255)">
      FMLE（Flash Media Live Encoder）
     </span>
     推送音视频数据至RtmpServer，可实现摄像头实时直播。不过，毕竟FMLE应用范围有限，想要把它嵌入到自己的程序中，还是要自己来实现RTMP协议的推送。本人实现了一个
     <span style="padding:0px; margin:0px; color:rgb(255,0,0)">
      RTMPLiveEncoder
     </span>
     ，通过采集摄像头视频和麦克风音频，并进行H.264和AAC编码，然后发送到FMS和crtmpserver上，实现实时直播，可以通过flash player正常观看，目前效果良好，延迟时间在
     <span style="padding:0px; margin:0px; color:rgb(255,0,0)">
      2秒
     </span>
     左右。本文就介绍一下RTMPLiveEncoder的主要思路和关键点，以期对需要这方面技术的朋友有所帮助。
    </p>
    <h2 style="margin:0px; padding:0px; color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; line-height:25px">
     <a name="t0" style="color:rgb(106,57,6)" target="_blank">
     </a>
     技术分析
    </h2>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     要实现
     <span style="padding:0px; margin:0px; color:rgb(255,0,0)">
      RTMPLiveEncoder
     </span>
     ，需要以下四种关键技术：
    </p>
    <ul style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding:0px; margin:0px">
     <li style="padding:0px; margin:0px">
      采集摄像头视频和麦克风音频
     </li>
     <li style="padding:0px; margin:0px">
      H264编码和AAC编码
     </li>
     <li style="padding:0px; margin:0px">
      视频和音频数据封装为可被流媒体
      <a href="http://www.rosoo.net/a/list_52_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
       服务器
      </a>
      识别的可播放流
     </li>
     <li style="padding:0px; margin:0px">
      RTMP协议实现报文发送
     </li>
    </ul>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     其中，前两项技术在我之前的文章“采集音频和摄像头视频并实时H264编码和AAC编码”中已经介绍过了，这里就不再啰嗦了。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     把音视频数据封装为可播放流，这个是一个难点。仔细研究一下，你会发现，RTMP Packet中封装的音视频数据流，其实和
     <span style="padding:0px; margin:0px; color:rgb(0,0,255)">
      FLV
     </span>
     封装音频和视频数据的方式是相同的，所以，我们只需要按照FLV封装H264和AAC的方式，即可生成可播放流。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     我们再看一下RTMP协议。Adobe曾经发布过一份文档《RTMP Specification》，不过
     <a href="http://en.wikipedia.org/wiki/Real_Time_Messaging_Protocol" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      wikipedia
     </a>
     指出这份文档隐藏了很多细节，单独根据它是无法正确实现RTMP的。不过，它还是有参考意义的。其实Adobe发布之前，RTMP协议就已经被破解的差不多了，现在也已经有比较完善的实现，比如：
     <a href="http://rtmpdump.mplayerhq.hu/" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      RTMPDump
     </a>
     ，它提供的是
     <a href="http://www.rosoo.net/a/list_100_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      C语言
     </a>
     的接口，这意味着可以很方便的在其他语言中调用。
    </p>
    <h2 style="margin:0px; padding:0px; color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; line-height:25px">
     <a name="t1" style="color:rgb(106,57,6)" target="_blank">
     </a>
     程序框架
    </h2>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     与我之前写的“采集音频和摄像头视频并实时H264编码和AAC编码” 这篇文章相同，采用DirectShow技术来实现音视频采集，
     <a href="http://www.rosoo.net/a/list_53_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      音频编码
     </a>
     和
     <a href="http://www.rosoo.net/a/list_53_1.html" rel="nofollow noopener noreferrer" style="color:rgb(37,110,177); text-decoration:none; padding:0px; margin:0px" target="_blank">
      视频编码
     </a>
     ，在各自线程（AudioEncoderThread和 VideoEncoderThread）中循环进行，RTMP的推送另起一个线程（RtmpThread）。两个编码线程实时编码音视频数据后，将数据交 与Rtmp线程，由Rtmp线程循环封装为Rtmp Packet，然后发出去。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     线程之间的数据交换，通过一个队列DataBufferQueue来实现。AudioEncoderThread和 VideoEncoderThread把数据指针post到DataBufferQueue之后，立即返回，这样就可以避免因为发送Rtmp报文的而影响 到编码线程的正常执行时间。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px; text-align:center">

     RtmpThread的首要工作就是发送音频数据流的解码信息头和视频数据流的解码信息头，并络续从DataBufferQueue中 取出数据，封装为RTMP Packet，发送出去。流程如下列代码所示：（process_buf_queue_，便是上图中的DataBufferQueue）

     librtmp

    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     一、编译librtmp
    </p>

     下载rtmpdump的代码，你会发明，它是一个地道的linux项目，除了一个简单的Makefile，其他什么都没有。如同 librtmp不依附于体系，我们可以不花费太多工夫，把它在windows上编译。不过，librtmp依附于openssl和zlib，我们须要起首 编译好它们。

     H264和AAC数据流
    </h2>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     本文提到过，RTMP推送的音视频流的封装情势和FLV格局类似，由此可知，向FMS推送H264和AAC直播流，须要起首发送"
     <span style="padding:0px; margin:0px">
      AVC sequence header
     </span>
     "和"
     <span style="padding:0px; margin:0px">
      AAC sequence header
     </span>
     "，这两项数据包含的是首要的编码信息，没有它们，解码器将无法解码。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px">
     AVC sequence header就是
     <span style="padding:0px; margin:0px">
      AVCDecoderConfigurationRecord
     </span>
     布局，该布局在标准文档“
     <span style="padding:0px; margin:0px">
      ISO-14496-15 AVC file format
     </span>
     ”中有具体申明。
    </p>
    <p style="color:rgb(51,51,51); font-family:Verdana,Arial,Tahoma; font-size:14px; line-height:25px; padding-top:0px; padding-bottom:0px; margin-top:0px; margin-bottom:0px; text-align:center">

     AAC sequence header存放的是
     <span style="padding:0px; margin:0px">
      AudioSpecificConfig
     </span>
     布局，该布局则在“
