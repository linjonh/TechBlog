    <h2>
     <a id="eclipseMySQL_0">
     </a>
     eclipse无法连接MySQL数据库
    </h2>
    <p>
     在运行外部导入文件时，出现了两个问题。
     <br/>
     第一个是
     <br/>
     java.sql.SQLException:Assess denied for uer‘root’@‘locathost’(using password:yes)
     <br/>
     如图
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8e822c7ab4005749ccb8458b83d53787.png"/>
    </p>
    <p>
     出现这个错误的原因
     <br/>
     第一 数据库密码错误
     <br/>
     第二 未连接到数据库
    </p>
    <p>
     检查自己的密码并没有问题，于是采用以下办法解决。
    </p>
    <h4>
     <a id="_MySQL_14">
     </a>
     第一 将MySQL加入到系统变量中
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/93f6a9b3bf7bbc5e97cdd35964c1ae31.png"/>
    </p>
    <p>
     同时 打开MySQL配置文件 如下图
    </p>
    <p>
     （如果发现没有my.ini文件 有两种解决办法）
     <br/>
     第一 文件可能被隐藏
     <br/>
     点击文件夹上方的查看-》勾选’隐藏的项目’
     <br/>
     你的my.ini可能会出现
     <br/>
     如果用了这一方法 发现你的配置文件还是不在 那就使用第二种方法
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/061f35b20f75742f664f94ba2c6f5f92.png">
      <br/>
      第二种方法 自己新建一个文件
      <br/>
      然后把以下代码复制进去即可
      <br/>
      <code>
       my.ini
      </code>
     </img>
    </p>
    <pre><code>[mysql]

; 设置 mysql 客户端默认字符集

default-character-set=utf8

[mysqld]

;设置 3306 端口

port = 3306

; 设置 mysql 的安装目录

basedir=D:\mysql-5.7.33-winx64

; 设置 mysql 数据库的数据的存放目录

datadir=D:\mysql-5.7.33-winx64\data

; 允许最大连接数

max_connections=200

; 服务端使用的字符集默认为 8 比特编码的 latin1 字符集

character-set-server=utf8

; 创建新表时将使用的默认存储引擎

default-storage-engine=INNODB
</code></pre>
<p>
注意 ：
</p>
<pre><code>; 设置 mysql 的安装目录

basedir=D:\mysql-5.7.33-winx64

; 设置 mysql 数据库的数据的存放目录

datadir=D:\mysql-5.7.33-winx64\data

</code></pre>
<p>
这一部分要设置成自己的地址
</p>
<p>
================================================
</p>
<h4>
<a id="__82">
</a>
第二 在配置文件最后一行添加代码
</h4>
<p>
添加代码
</p>
<pre><code>skip-grant-tables
</code></pre>
<p>
效果如图所示
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/336d0ae5499938674f490788445db180.png">
<br/>
保存 my.ini 并且重新启动 MySQL 服务器
</img>
</p>
<p>
重新启动服务器方法：
<br/>
打开任务管理器 右键重启即可
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fdd34c61564535091a43f429123108ae.png">
<br/>
这时 你的数据库就不需要密码 就可以登陆了
<br/>
检测一下数据库是否配置成功
<br/>
打开 cmd
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f73c56ee4643b06e7fd4755e9b52b6f9.png">
<br/>
出现以上画面 表示 数据库配置成功
</img>
</img>
</p>
<p>
然后重启 eclipse 刷新自己的数据库 就可以使用了
</p>
