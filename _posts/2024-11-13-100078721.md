     浅析微信支付：如何使用沙箱环境测试
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <div class="htmledit_views" id="content_views">
    <p style="text-indent:33px;">
     在实际开发中，通常我们都是在开发环境中开发，本地环境也有很多限制，比如：微信支付无法调起、H5链接需要鉴权、支付结果通知需要外网等。
    </p>
    <p style="text-indent:33px;">
     面对以上的问题，微信官方给出了解决的方法，就是咋们这篇文章的
     <strong>
      <code>
       沙箱环境
      </code>
     </strong>
     ，也就是微信支付的官方测试环境，这个环境能做些什么呢？我觉得最重要的一点就是我们可以实时根据官方的例子调用对应的接口，并且接口会马上返回结果，拿微信支付预支付单接口来说，调用以后会实时返回我们支付的相关信息，这样就免于开发时不知道返回结果而苦恼。
    </p>
    <h2 id="仿真测试系统">
     仿真测试系统
    </h2>
    <p style="text-indent:33px;">
     为降低商户测试门槛，微信支付团队开发了一套独立的仿真测试系统。该系统根据验收用例金额的不同返回不同的响应报文，以满足商户正常功能测试、安全/异常测试及性能测试的需求。
    </p>
    <p style="text-align:center;">
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/10b26134c876bf462b8c9d657f9de93b.png"/>
    </p>
    <p style="text-indent:33px;">
     图为微信支付仿真测试系统（后简称仿真系统）的简化原理图。仿真系统的API协议与正式API完全相同（API接口文档）。商户开发者只需将正式API的调用URL增加一层
     <code>
      sandboxnew
     </code>
     路径，即可对接到仿真系统。例如：
    </p>
    <p style="text-indent:33px;">
     刷卡支付URL：
     <a href="https://api.mch.weixin.qq.com/pay/micropay" rel="nofollow">
      https://api.mch.weixin.qq.com/pay/micropay
     </a>
    </p>
    <p style="text-indent:33px;">
     变更为：
     <a href="https://api.mch.weixin.qq.com/sandboxnew/pay/micropay" rel="nofollow">
      https://api.mch.weixin.qq.com/sandboxnew/pay/micropay
     </a>
     。
    </p>
    <p style="text-indent:33px;">
     仿真系统与生产环境完全独立，包括存储层。商户在仿真系统所做的所有交易（如下单、支付、查询）均为无资金流的假数据，即：用户无需真实扣款，商户也不会有资金入账。代金券同理，沙箱环境中无需商户真实制券与发券，亦不会出现真实扣券情况。验收仿真测试系统的API验签密钥需从API获取：
    </p>
    <p style="text-align:center;">
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/530f868118a514c971d9416e34d62096.png"/>
    </p>
    <h2 id="源码交互过程">
     源码&amp;交互过程
    </h2>
    <p style="text-indent:33px;">
     以下为微信官方的
     <code>
      仿真测试系统
     </code>
     文档：
     <a href="https://pay.weixin.qq.com/wiki/doc/api/jsapi.php?chapter=23_1" rel="nofollow">
      https://pay.weixin.qq.com/wiki/doc/api/jsapi.php?chapter=23_1
     </a>
    </p>
    <p style="text-indent:33px;">
     上面说明具体的交互过程和相关的仿真测试系统的API验证签名，为什么需要这个签名接口呢？这是因为使用
     <code>
      沙箱环境
     </code>
     时使用的是真实的
     <code>
      商户号
     </code>
     、
     <code>
      小程序/公众号APP_ID
     </code>
     ，但是
     <code>
      API密钥
     </code>
     这个参数必须使用
     <code>
      沙箱环境
     </code>
     的
     <code>
      sandbox_signkey
     </code>
     ，此接口主要是取得这个参数。
    </p>
    <p style="text-indent:33px;">
     注：仿真测试环境中的商户号（父子商户号）需使用真实商户号。
    </p>
    <p style="text-indent:33px;">
     下面为取得
     <code>
      sandbox_signkey
     </code>
     的示例：
    </p>
    <pre class="has"><code class="language-java">/**
 * 获取沙盒 sandbox_signkey
 *
 * @author yclimb
 * @date 2018/9/18
 */
private void doGetSandboxSignKey() throws Exception {
    WXPayConfigImpl config = WXPayConfigImpl.getInstance();
    HashMap&lt;String, String&gt; data = new HashMap&lt;String, String&gt;();
    // 商户号
    data.put("mch_id", config.getMchID());
    // 获取随机字符串
    data.put("nonce_str", WXPayUtil.generateNonceStr());
    // 生成签名
    String sign = WXPayUtil.generateSignature(data, config.getKey());
    data.put("sign", sign);
    
    // 得到 sandbox_signkey
    WXPay wxPay = new WXPay(config);
    String result = wxPay.requestWithoutCert("/sandboxnew/pay/getsignkey", data, 10000, 10000);
    System.out.println(result);
}</code></pre>
    <p style="text-indent:33px;">
     小伙伴可以根据
     <strong>
      <code>
       result
      </code>
     </strong>
     来获取具体的返回数据，解析之后获取
     <strong>
      <code>
       sandbox_signkey
      </code>
     </strong>
     参数。
    </p>
    <p style="text-indent:33px;">
     商户接入仿真系统的交互流程示例：
    </p>
    <ol>
     <li>
      商户发起刷卡支付请求，使用POST方式调用 https://api.mch.weixin.qq.com/sandboxnew/pay/micropay
     </li>
     <li>
      带sandboxnew 的https请求会被nginx路由到仿真系统。仿真系统根据支付金额（total_fee字段）返回预期报文给商户。同时，落地该笔请求数据；
     </li>
     <li>
      商户发起查单，调用 https://api.mch.weixin.qq.com/sandboxnew/pay/orderquery，带上微信订单号（transaction_id）或商户内部单号（out_trade_no）；
     </li>
     <li>
      仿真系统收到查单请求后，根据单号及金额返回预期的查单结果给商户；
     </li>
     <li>
      商户下载对账单，调用 https://api.mch.weixin.qq.com/sandboxnew/pay/downloadbill ，仿真系统返回固定的账单格式给商户。注：账单内容不一定与商户在仿真系统产生的交易完全相同。
     </li>
    </ol>
    <p style="text-indent:33px;">
     沙箱说明：sandbox/sandboxnew
    </p>
    <p style="text-indent:33px;">
     微信支付沙箱环境，是提供给微信支付商户的开发者，用于模拟支付及回调通知。以验证商户是否理解回调通知、账单格式，以及是否对异常做了正确的处理。
    </p>
    <p style="text-indent:33px;">
     如何对接沙箱环境？
     <br/>
     1、修改商户自有程序或配置中，微信支付api的链接，如：被扫支付官网的url为：https://api.mch.weixin.qq.com/pay/micropay 增加sandboxnew路径，变更为https://api.mch.weixin.qq.com/sandboxnew/pay/micropay ， 即可接入沙箱验收环境，其它接口类似；
     <br/>
     2、在微信支付开发调试站点（站点链接：http://mch.weixin.qq.com/wiki/doc/api/index.php ），按接口文档填入正确的支付参数，发起微信支付请求，完成支付；
     <br/>
     3、验收完成后，修改程序或配置中的api链接（重要！），去掉sandboxnew路径。对接现网环境。
    </p>
    <p style="text-indent:33px;">
     说明地址：
     <br/>
     <a href="https://pay.weixin.qq.com/wiki/doc/api/jsapi.php?chapter=23_1" rel="nofollow">
      https://pay.weixin.qq.com/wiki/doc/api/jsapi.php?chapter=23_1
     </a>
     <br/>
     <a href="https://pay.weixin.qq.com/wiki/doc/api/app/app.php?chapter=21_2" rel="nofollow">
      https://pay.weixin.qq.com/wiki/doc/api/app/app.php?chapter=21_2
     </a>
    </p>
    <h2 id="结语">
     结语
    </h2>
    <p style="text-indent:33px;">
     其实
     <code>
      沙箱环境
     </code>
     主要是为了方便在开发时及时获得接口返回值和进行
     <code>
      商户支付验收
     </code>
     使用，本文讲了如何获取
     <code>
      sandbox_signkey
     </code>
     参数，然后如何进行模拟对接，在实际接口URL后增加
     <code>
      sandboxnew
     </code>
     即可，接口会实时返回结果参数，此点于正式环境不同（正式环境支付后是异步调用，沙箱环境是实时返回）。
    </p>
    <p style="text-indent:33px;">
     注意：有的接口沙箱环境的接口并不只是在链接中增加
     <code>
      sandboxnew
     </code>
     ，整个链接都会改变，在实际操作中我们应该查看官方文档一一对照，如支付退款接口，正式线接口为：
     <code>
      /secapi/pay/refund
     </code>
     ，而沙箱环境接口为：
     <code>
      /sandboxnew/pay/refund
     </code>
     ，在沙箱环境中去掉了
     <code>
      secapi
     </code>
     这一路径，请小伙伴一定要注意。
    </p>
