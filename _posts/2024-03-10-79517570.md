   <div class="htmledit_views" id="content_views">
    <p>
     这两天测试发现服务器时间（虚拟机）明显滞后于标准时间，大概慢了20多秒。原因是虚拟机没有晶振，时间会不准确。为了解决这个问题在服务器上安装了ntp服务，同步阿里云ntp服务器时间。
    </p>
    <p>
     1. 安装ntp服务
    </p>
    <pre><code class="language-html">sudo yum update
sudo yum install ntp</code></pre>
    <p>
     2.修改ntp.conf，将原有ntp server优先替换为aliyun ntp server
    </p>
    <pre><code class="language-html">restrict default nomodify
server ntp1.aliyun.com
server time.nist.gov</code></pre>
    <p>
     3. 启动ntp服务
    </p>
    <pre><code class="language-html">sudo /etc/init.d/ntpd start</code></pre>
    <p>
     4.查询ntp服务状态，等待5-10分钟后如果看到阿里云ntp服务器前面标注了*，表示ntp同步成功。
    </p>
    <pre><code class="language-html">[root@localhost ~]# ntpq -p
     remote           refid      st t when poll reach   delay   offset  jitter
==============================================================================
*time5.aliyun.co 10.137.38.86     2 u  589 1024  377   67.667   -0.574   1.625
 time-d-b.nist.g .NIST.           1 u   7h 1024    0  221.886  -13.338   0.000
</code></pre>
    <p>
     ps.如果迟迟没有同步，有可能因为差异太大，可以先手动ntpdate一下，然后启动ntp服务。
    </p>
    <pre><code class="language-html">sudo /etc/init.d/ntpd stop
sudo ntpdate ntp1.aliyun.com
date
sudo /etc/init.d/ntpd start</code></pre>
    <p>
     现在运行了4天左右，服务器和标准时间的误差在1秒以内了。
    </p>
