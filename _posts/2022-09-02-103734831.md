       <h2 id="%C2%A0%C2%A0%208.1%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%AE%A1%E7%90%86%E6%A6%82%E8%BF%B0" style="margin-left:0in;">
        <span style="color:#1e4e79;">
         8.1 软件项目管理概述
        </span>
       </h2>
       <h4 id="8.1.1%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%AE%A1%E7%90%86%E6%A6%82%E5%BF%B5%E5%8F%8A%E7%89%B9%E7%82%B9" style="margin-left:0in;">
        <strong>
         8.1.1 软件项目管理概念及特点
        </strong>
       </h4>
       <p style="margin-left:0in;">
        软件项目管理是指对软件项目的
        <span style="color:#f33b45;">
         整个生存周期过程
        </span>
        的管理，按照项目任务要求，合理的利用资源，为
        <span style="color:#f33b45;">
         保证软件产品质量
        </span>
        所做的工作。
       </p>
       <p style="margin-left:0in;">
        <strong>
         主要职能
        </strong>
        <strong>
         ：
        </strong>
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="136" src="https://i-blog.csdnimg.cn/blog_migrate/6e27ace00855f575f5f851ed38f7ba81.png" width="493"/>
       </p>
       <p style="margin-left:0in;">
        软件项目管理的
        <strong>
         主要特点
        </strong>
        <strong>
         ：
        </strong>
       </p>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         (1)目标产品抽象难度量。(2)项目独特定制化生产。
        </span>
       </p>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         (3)智力密集技术复杂。    (4)人为因素影响大。
        </span>
       </p>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
        </span>
       </p>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
        </span>
       </p>
       <p style="margin-left:0in;">
        8.1.2 软件项目管理过程及内容
       </p>
       <p style="margin-left:0in;">
        <strong>
         项目管理过程
        </strong>
        ：
        <span style="color:#e84c22;">
         项目启动、项目计划、组织实施、项目监控和项目验收
        </span>
        五个阶段。
       </p>
       <p style="margin-left:0in;">
        在项目具体实施过程中，计划、执行和控制通常需要往复循环（称为核心循环）
       </p>
       <p style="margin-left:0in;">
       </p>
       <p style="margin-left:0in;">
        <span style="color:#000000;">
         <strong>
          软件项目管理的内容
         </strong>
        </span>
       </p>
       <p style="margin-left:0in;">
        项目管理的九大知识领域的内容，分布在项目的五大过程中
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="300" src="https://i-blog.csdnimg.cn/blog_migrate/795ba4a88887a6084a435c23cfed5857.png" width="574"/>
       </p>
       <p style="margin-left:0in;">
        核心的四大知识领域是
        <span style="color:#e84c22;">
         范围、时间、成本与质量管理
        </span>
        。
       </p>

        8.2.1 软件项目启动过程及任务
       </h4>
       <p style="margin-left:0in;">
        软件项目的
        <strong>
         启动过程
        </strong>
        是一个 新项目决策立项与开始准备实施的过程。
       </p>
       <h4 style="margin-left:0in;">
       </h4>
       <h4 id="8.2.2%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%BB%84%E7%BB%87%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        8.2.2 软件项目组织管理
       </h4>
       <p style="margin-left:0in;">
        <strong>
         <span style="color:#000000;">
          软件项目管理
         </span>
        </strong>
        先于任何技术活动之前开始，并且
        <span style="color:#e84c22;">
         贯穿于软件的整个生命周期
        </span>
        。
       </p>
       <p style="margin-left:0in;">
        效的软件项目组织管理集中在4个P上，即人员(Person)、产品(Product)、过程(Procedure)和项目(Project)。
       </p>
       <p style="margin-left:0in;">
       </p>
       <p style="margin-left:0in;">
       </p>
       <h2 id="%C2%A0%C2%A0%208.3%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E8%BF%9B%E5%BA%A6%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        <span style="color:#1e4e79;">
         8.3 软件项目进度管理
        </span>
       </h2>
       <h4 id="8.3.1%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E8%BF%9B%E5%BA%A6%E4%BC%B0%E7%AE%97" style="margin-left:0in;">
        8.3.1 软件项目进度估算
       </h4>
       <p style="margin-left:0in;">
        1. 软件规模与工作量估算
       </p>
       <p style="margin-left:0in;">
        常用的
        <span style="color:#e84c22;">
         软件规模度量单位
        </span>
        有
        <span style="color:#e84c22;">
         千行代码
        </span>
        <span style="color:#e84c22;">
         KLOC(Kilo-Lines Of Code)
        </span>
        <span style="color:#e84c22;">
         、功能点
        </span>
        <span style="color:#e84c22;">
         FP(Function Points)
        </span>
        <span style="color:#e84c22;">
         和对象点
        </span>
        <span style="color:#e84c22;">
         OP(Object Points
        </span>
        <span style="color:#e84c22;">
         ）
        </span>
       </p>
       <p style="margin-left:0in;">
        常用的
        <strong>
         工作量度量单位
        </strong>
        有
        <span style="color:#e84c22;">
         人年、人月和人日等
        </span>
       </p>
       <p style="margin-left:0in;">
        1）
        <strong>
         软件规模估算
        </strong>
       </p>
       <p style="margin-left:0in;">
        （
        <span style="color:#e84c22;">
         1
        </span>
        <span style="color:#e84c22;">
         ）千代码行（
        </span>
        <span style="color:#e84c22;">
         KLOC
        </span>
        <span style="color:#e84c22;">
         ）估算法。
        </span>
        <span style="color:#e84c22;">
         （
        </span>
        <span style="color:#e84c22;">
         2
        </span>
        <span style="color:#e84c22;">
         ）功能点（
        </span>
        <span style="color:#e84c22;">
         FP
        </span>
        <span style="color:#e84c22;">
         ）估算法。
        </span>
       </p>
       <p style="margin-left:0in;">
        2）
        <strong>
         工作量估算
        </strong>
        。 常用的两个估算模型是：
        <span style="color:#e84c22;">
         CoCoMo
        </span>
        <span style="color:#e84c22;">
         模型和
        </span>
        <span style="color:#e84c22;">
         Putnam
        </span>
        <span style="color:#e84c22;">
         模型
        </span>
        。
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.3.2%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E8%BF%9B%E5%BA%A6%E5%AE%89%E6%8E%92" style="margin-left:0in;">
        8.3.2 软件项目进度安排
       </h4>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         过程调整是项目计划中首要任务
        </span>
       </p>
       <p style="margin-left:0in;">
        <strong>
         任务分解落实
        </strong>
       </p>
       <p style="margin-left:0in;">
        软件设计上的分层结构,通常成为
        <span style="color:#e84c22;">
         任务分解结构
        </span>
        <span style="color:#e84c22;">
         WBS
        </span>
        的依据.
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="273" src="https://i-blog.csdnimg.cn/blog_migrate/a8585d526629f1cc6951eea888ca9edc.png" width="526"/>
       </p>
       <p style="margin-left:0in;">
        <strong>
         安排计划的编排与优化
        </strong>
       </p>
       <p style="margin-left:0in;">
        1) 甘特图
       </p>
       <p style="margin-left:0in;">
        甘特图（Gantt Chart）又称横道图，如图所示，是活动进度与日历表的对照图。
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="227" src="https://i-blog.csdnimg.cn/blog_migrate/5ac6e7334aec52bee280d914b0acd71d.png" width="380"/>
       </p>
       <p style="margin-left:0in;">
        2) 网络图
       </p>
       <p style="margin-left:0in;">
        网络图是用网络分析的方法编制的进度计划图。主要可以描述每个活动及其之间的逻辑关系。
       </p>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         计划评审技术
        </span>
        <span style="color:#e84c22;">
         PERT
        </span>
        和
        <span style="color:#e84c22;">
         关键路径法
        </span>
        <span style="color:#e84c22;">
         CPM
        </span>
        都采用网络图表示项目的活动。
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="136" src="https://i-blog.csdnimg.cn/blog_migrate/9e86aec4694f63b396fc5da8ca99907d.png" width="493"/>
       </p>
       <p style="margin-left:0in;">
        网络图常用术语：
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="239" src="https://i-blog.csdnimg.cn/blog_migrate/4c7d01ebd7c063d2fb5472e235876ea5.png" width="499"/>
       </p>
       <p style="margin-left:0in;">
        <img alt="" class="has" height="274" src="https://i-blog.csdnimg.cn/blog_migrate/d58b0da9befbc2998b2f669d06dfd47c.png" width="495"/>
       </p>
       <p style="margin-left:0in;">
       </p>
       <p style="margin-left:0in;">
       </p>
       <h2 id="%C2%A0%C2%A0%208.4%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9A%84%E8%BF%87%E7%A8%8B%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        <span style="color:#1e4e79;">
         8.4 软件项目的过程管理
        </span>
       </h2>
       <p style="margin-left:0in;">
        项目实施中的
        <strong>
         关键
        </strong>
        是对软件项目的
        <span style="color:#e84c22;">
         过程管理
        </span>
        ，
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.4.1%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9A%84%E9%9C%80%E6%B1%82%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        8.4.1 软件项目的需求管理
       </h4>
       <p style="margin-left:0in;">
        软件项目的需求开发过程，需要经历
        <span style="color:#e84c22;">
         需求获取、需求分析、需求规格编写、需求验证
        </span>
        四个往复确认的阶段。
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.4.2%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        8.4.2 软件项目质量管理
       </h4>
       <p style="margin-left:0in;">
        质量管理主要内容包括三个过程：
        <span style="color:#e84c22;">
         质量计划制定、质量保证和质量控制
        </span>
        。
       </p>
       <p style="margin-left:0in;">
        在全面软件质量管理模型中，质量管理有三大类：
        <span style="color:#e84c22;">
         技术评审、过程检查、软件测试
        </span>
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.4.3%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E9%85%8D%E7%BD%AE%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        8.4.3 软件项目配置管理
       </h4>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         软件配置管理
        </span>
        <span style="color:#e84c22;">
         (SCM
        </span>
        )是对产品进行
        <span style="color:#e84c22;">
         标志、存储和控制
        </span>
        ，以维护其完整性、可追溯性及正确性而为软件开发提供的一套管理办法和活动原则。
       </p>
       <p style="margin-left:0in;">
        软件配置管理要素如下：
       </p>
       <p style="margin-left:0in;">
        (1) 软件配置项
       </p>
       <p style="margin-left:0in;">
        (2) 基线  功能基线  分配基线   产品基线
       </p>
       <p style="margin-left:0in;">
        (3) 配置管理机构     配置管理委员会(SCCB)
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.4.4%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9A%84%E9%A3%8E%E9%99%A9%E7%AE%A1%E7%90%86" style="margin-left:0in;">
        8.4.4 软件项目的风险管理
       </h4>
       <p style="margin-left:0in;">
        软件项目的风险管理是按照风险计划对项目进行
        <span style="color:#e84c22;">
         全过程
        </span>
        的管理。
       </p>
       <p style="margin-left:0in;">
        <strong>
         风险分析的目的
        </strong>
        是建立应对处理及监控风险的策略。一个有效的策略必须考虑三个问题：
        <span style="color:#e84c22;">
         风险避免、风险监控、风险管理及意外事件计划
        </span>
        （预案）
       </p>
       <p style="margin-left:0in;">
        同每个
        <strong>
         风险相关的三元组
        </strong>
        (
        <span style="color:#e84c22;">
         风险描述、发生概率、风险影响
        </span>
        )是建立风险应对处理及避免或消除步骤的重要基础。
       </p>
       <p style="margin-left:0in;">
        对风险采取主动的方法，则避免风险永远是最好的策略
       </p>
       <p style="margin-left:0in;">
       </p>
       <h2 id="%C2%A0%C2%A0%208.5%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9B%91%E6%8E%A7%E4%B8%8E%E9%AA%8C%E6%94%B6" style="margin-left:0in;">
        <span style="color:#1e4e79;">
         8.5 软件项目监控与验收
        </span>
       </h2>
       <p style="margin-left:0in;">
        软件项目监控是对项目实施情况进行跟踪、度量、检查评审并与目标对比和调控的过程。
       </p>
       <h4 id="8.5.1%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9B%91%E6%8E%A7%E8%BF%87%E7%A8%8B" style="margin-left:0in;">
        8.5.1 软件项目监控过程
       </h4>
       <p style="margin-left:0in;">
        在项目执行过程通常通过设置
        <span style="color:#e84c22;">
         偏差的警戒线和底线
        </span>
        的方法来控制项目，警戒线和底线以
        <span style="color:#e84c22;">
         时间和阶段成果
        </span>
        为
        <strong>
         标志。
        </strong>
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.5.2%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E7%9B%91%E6%8E%A7%E5%86%85%E5%AE%B9%E5%8F%8A%E6%96%B9%E6%B3%95" style="margin-left:0in;">
        8.5.2 软件项目监控内容及方法
       </h4>
       <p style="margin-left:0in;">
        <span style="color:#e84c22;">
         项目跟踪
        </span>
        是项目控制的前提和条件，
        <span style="color:#e84c22;">
         项目监制
        </span>
        是项目跟踪的目的和服务对象。
       </p>
       <p style="margin-left:0in;">
        <strong>
         项目质量监控的方法
        </strong>
        包括：
        <span style="color:#e84c22;">
         质量度量、控制图法、趋势分析法
        </span>
        等。
        <strong>
         质量度量
        </strong>
        主要有两类：
        <span style="color:#e84c22;">
         预测型和验收型
        </span>
        。
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.5.3%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E8%AF%84%E5%AE%A1" style="margin-left:0in;">
        8.5.3 软件项目评审
       </h4>
       <p style="margin-left:0in;">
        项目评审包括：
        <span style="color:#e84c22;">
         评审准备、评审过程和评审报告
        </span>
        三个过程
       </p>
       <p style="margin-left:0in;">
        项目评审分为5种：商务评审、技术评审、管理评审、质量评审和产品评审等
       </p>
       <p style="margin-left:0in;">
        <strong>
         技术评审对象
        </strong>
        主要是
        <span style="color:#e84c22;">
         规范和设计
        </span>
        ，而管理评审关注的是
        <span style="color:#e84c22;">
         项目计划和报告
        </span>
       </p>
       <p style="margin-left:0in;">
       </p>
       <h4 id="8.5.4%20%E8%BD%AF%E4%BB%B6%E9%A1%B9%E7%9B%AE%E6%94%B6%E5%B0%BE%E4%B8%8E%E9%AA%8C%E6%94%B6" style="margin-left:0in;">
        8.5.4 软件项目收尾与验收
       </h4>
       <p style="margin-left:0in;">
        项目验收的组织构成包括：项目接收方、项目开发团队和项目监理人员。
       </p>
       <p style="margin-left:0in;">
       </p>
