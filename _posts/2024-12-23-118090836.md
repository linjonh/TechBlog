    <h2>
     <a id="_0">
     </a>
     内存溢出原因分析
    </h2>
    <p>
     现在很多前端项目都依赖于第三方库来提高开发效率，但是很多开发人员为了方便，会把用到的第三方库完整引入，导致整个项目体积过大，看似很方便，但是会造成很多隐患，例如：使用
     <code>
      Webpack
     </code>
     打包项目时，占用过多的系统资源，导致超出了
     <code>
      Node
     </code>
     的内存大小限制，从而打包失败或者启动项目失败。
    </p>
    <h2>
     <a id="_2">
     </a>
     内存溢出现象
    </h2>
    <p>
     <img alt="报错截图" src="https://i-blog.csdnimg.cn/blog_migrate/706c7e909fbe4a48d063f0db11fb7047.png#pic_center"/>
    </p>
    <h2>
     <a id="_4">
     </a>
     内存溢出解决方案
    </h2>
    <h3>
     <a id="1__5">
     </a>
     1. 针对本项目
    </h3>
    <blockquote>
     <p>
      在本项目中打开终端或命令行工具，安装
      <code>
       increase-memory-limit
      </code>
      和
      <code>
       cross-env
      </code>
      ，然后在项目根目录下的
      <code>
       package.json
      </code>
      文件中
      <code>
       script
      </code>
      中添加
      <code>
       "fix-memory-limit": "cross-env LIMIT=4096 increase-memory-limit"
      </code>
      ， 完成后在本项目根目录的终端或者命令行中执行以下命令：
     </p>
     <pre><code class="prism language-bash"><span class="token function">npm</span> run fix-memory-limit
</code></pre>
     <p>
      即可将node最大占用内存设置为4G
      <br/>
      如果出现一下信息则说明成功修改内存限制：
      <br/>
      <img alt="修改成功" src="https://i-blog.csdnimg.cn/blog_migrate/171ea5b3e2458c2d19a3ea5db7934ac4.png#pic_center"/>
     </p>
    </blockquote>
    <h3>
     <a id="2__14">
     </a>
     2. 针对全局
    </h3>
    <blockquote>
     <p>
      在任意终端或命令行中执行：
     </p>
     <pre><code class="prism language-bash"><span class="token function">npm</span> i increase-memory-limit -g
increase-memory-limit
</code></pre>
     <p>
      即可
     </p>
    </blockquote>
