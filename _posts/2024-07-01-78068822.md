    <p>
     这几天在敲机房的时候，我发现我的数据库总是不能用了，数据库的连接出问题了。问题如下，显示数据库恢复挂起，遇到这样的问题，不要怕，看我给你解决，哈哈。
    </p>
    <p>
     <img alt="" src="https://img-blog.csdn.net/20170923094420219?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvd2FuZ3l1bnpoYW8wMDc=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
    </p>
    <p>
     首先我 想说一下是什么
     <span style="font-size:14px; color:#ff0000">
      <strong>
       原因造
      </strong>
     </span>
     成的，后台是由于
     <strong>
      <span style="font-size:14px; color:#cc0000">
       磁盘空间满了，导致中间数据库显示恢复挂起的状态。
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-size:14px; color:#cc0000">
       <br/>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-size:18px; color:#ff0000">
       解决方法一
      </span>
     </strong>
     是首先先把数据库脱机，然后更改权限，该数据库权限的时候应该注意不仅要改。.mdf的权限而且也要改_log.ldf的权限。
    </p>
    <p>
     找到数据库所在位置——右键——属性——安全——高级——组或者用户名——选择——赋予权限。把只读改为完全控制
    </p>
    <p>
     <img alt="" src="https://img-blog.csdn.net/20170923144239800?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvd2FuZ3l1bnpoYW8wMDc=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center">
      <img alt="" src="https://img-blog.csdn.net/20170923144245208?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvd2FuZ3l1bnpoYW8wMDc=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center">
       <br/>
      </img>
     </img>
    </p>
    <p>
     <br/>
    </p>
    <p>
     在对数据库进行这样的处理之后，我们就
     <strong>
      <span style="font-size:14px; color:#ff0000">
       要重新附加进去，然后右击数据库-任务-收缩-文件，再点击确定，
      </span>
     </strong>
     就可以使用了，哈哈，我已经尝试过了，很好用的。如果只是附件进去，还会出问题的。
    </p>
    <p>
     <br/>
    </p>
    <p>
     <span style="font-size:14px; color:#ff0000">
      <strong>
       解决方法二
      </strong>
     </span>
     ：我在网上百度的，，参考别人的，没有解决，在程序运行中出现了很多错，希望有大神可以给我解决。这是他的原文，有能解决的再评论中我们讨论一下。
    </p>
    <p>
     <br/>
     如果你的数据库还处于挂起状态，请把我下面代码的test改为你的库名，然后执行完，刷新就正常了：
     <br/>
     <br/>
     USE master
     <br/>
     GO
     <br/>
     ALTER DATABASE test SET SINGLE_USER
     <br/>
     GO
     <br/>
     ALTER DATABASE test SET EMERGENCY
     <br/>
     GO
     <br/>
     DBCC CHECKDB(test,REPAIR_ALLOW_DATA_LOSS)
     <br/>
     go
     <br/>
     ALTER DATABASE test SET ONLINE
     <br/>
     GO
     <br/>
     ALTER DATABASE test SET MULTI_USER
     <br/>
     GO
     <br/>
     <br/>
