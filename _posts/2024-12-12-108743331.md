    <p>
     TCE平台方向：（base地：北京；16k-32k）
     <br/>
     岗位职责：
     <br/>
     1、负责控制台/监控/计费/容器平台等产品的项目交付、发布变更等工作，协助运维保障系统7*24小时稳定运行；
     <br/>
     2、负责应急预案、业务容灾和故障恢复策略方案的制定与实施；
     <br/>
     3、深入理解产品的架构， 发现并解决故障隐患及性能瓶颈，参与架构优化、容灾优化、性能优化等项目；
     <br/>
     4、负责各个项目的交付部署问题跟踪与管理，推进部署/运维自动化工具开发，提高部署/运维效率和质量；
     <br/>
     5、推动产品优化和完善部署/运维手册，推进产品交付的培训和认证，培养出高质量的交付合作团队；
     <br/>
     6、指导一线交付团队进行项目交付，负责交付过程中的联调和问题排查。
     <br/>
     岗位要求
     <br/>
     1、 本科及以上学历，计算机或相关专业； 3年以上Linux运维工作经验，并有100台以上Linux服务器运维经验；
     <br/>
     2、 熟悉大型Web系统架构，以及相关应用的部署调优，包括：Web服务器、负载均衡、消息队列、数据库、高可用解决方案等；
     <br/>
     3、 熟悉Docker容器的原理及相关技术，深入了解K8S等容器集群管理方案；
     <br/>
     4、 熟悉常见的监控系统，熟练使用Bash、Python等脚本语言；
     <br/>
     5、 熟悉数据库/存储系统（mysql、es、ceph等）运维的优先； 熟悉常用公用组件（lvs、etcd、zk、redis等）运维的优先；
     <br/>
     6、 熟悉互联网产品基本架构，有互联网产品研发或运维经验者优先；
     <br/>
     7、 有较好的表达和沟通能力，和逻辑思考能力，能够现场解决客户的问题，并能够承受一线客户的压力
    </p>
