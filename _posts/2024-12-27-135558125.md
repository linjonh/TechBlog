    <h2 id="一开发准备">
     一、开发准备
    </h2>
    <p>
     本篇博客基于的系统版本：华为官方HarmonyOS版本3.1、OpenHarmony版本4.0Beta
    </p>
    <h3 id="开发语言">
     开发语言
    </h3>
    <ul>
     <li>
      ArkTS语言(推荐)
     </li>
     <li>
      JS语言(支持)
     </li>
     <li>
      Java语言(已放弃支持)
     </li>
    </ul>
    <p>
     从Harmony4.0开始，官方主推
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides-V3/arkts-get-started-0000001504769321-V3?catalogVersion=V3" rel="nofollow" title="ArkTS语言">
      ArkTS语言
     </a>
     ，且不再支持Java语言
    </p>
    <h3 id="ui框架-方舟开发框架arkui框架">
     UI框架-方舟开发框架（ArkUI框架）
    </h3>
    <ul>
     <li>
      <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides-V3/arkui-overview-0000001532577181-V3" rel="nofollow" title="ArkUI框架介绍">
       ArkUI框架介绍
      </a>
     </li>
    </ul>
    <p>
     个人理解：类似iOS的UIKit/SwiftUI，或者安卓的Jetpack的UI组件/Compose
    </p>
    <h3 id="开发范式">
     开发范式
    </h3>
    <ul>
     <li>
      <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides-V3/arkui-overview-0000001532577181-V3" rel="nofollow" title="ArkUI开发范式介绍">
       ArkUI开发范式介绍
      </a>
     </li>
    </ul>
    <p>
     方舟开发框架（ArkUI框架）针对不同目的和技术背景的开发者提供了两种开发范式：
    </p>
    <ul>
     <li>
      声明式开发范式：ArkTS的声明式开发范式
     </li>
     <li>
      类Web开发范式：兼容JS的类Web开发范式
     </li>
    </ul>
    <table>
     <thead>
      <tr>
       <th>
        开发范式名称
       </th>
       <th>
        语言生态
       </th>
       <th>
        UI更新方式
       </th>
       <th>
        适用场景
       </th>
       <th>
        适用人群
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        声明式开发范式
       </td>
       <td>
        ArkTS语言
       </td>
       <td>
        数据驱动更新
       </td>
       <td>
        复杂度较大、团队合作度较高的程序
       </td>
       <td>
        移动系统应用开发人员、系统应用开发人员
       </td>
      </tr>
      <tr>
       <td>
        类Web开发范式
       </td>
       <td>
        JS语言
       </td>
       <td>
        数据驱动更新
       </td>
       <td>
        界面较为简单的程序应用和卡片
       </td>
       <td>
        Web前端开发人员
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     个人理解：类似iOS的SwiftUI(声明式)和UIKit，或者安卓的JetpackCompose(声明式)和JetpackUI
    </p>
    <h3 id="应用模型">
     应用模型
    </h3>
    <h5 id="概念">
     概念
    </h5>
    <ul>
     <li>
      官方解释：应用模型是HarmonyOS为开发者提供的应用程序所需能力的抽象提炼，它提供了应用程序必备的组件和运行机制。
     </li>
     <li>
      个人理解：开发脚手架+底层运行引擎，类似开发模板
     </li>
    </ul>
    <h5 id="模型">
     模型
    </h5>
    <ul>
     <li>
      Stage模型：HarmonyOS 3.1 Developer Preview版本开始新增的模型，是目前主推且会长期演进的模型。在该模型中，由于提供了AbilityStage、WindowStage等类作为应用组件和Window窗口的“舞台”，因此称这种应用模型为Stage模型。
     </li>
     <li>
      FA（Feature Ability）模型：HarmonyOS早期版本开始支持的模型，已经不再主推。
     </li>
    </ul>
    <h5 id="stage和fa的区别">
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides-V3/application-model-description-0000001493584092-V3" rel="nofollow" title="Stage和FA的区别">
      Stage和FA的区别
     </a>
    </h5>
    <h3 id="技术选型">
     技术选型
    </h3>
    <p>
     根据官方文档中的介绍，目前合适的开发技术栈组合：
    </p>
    <p>
     <strong>
      Stage模型
     </strong>
     +
     <strong>
      声明式开发范式
     </strong>
     +
     <strong>
      ArkUI框架
     </strong>
     +
     <strong>
      ArkTS语言
     </strong>
    </p>

    <h3 id="1创建arkts工程">
     1.创建ArkTS工程
    </h3>
    <h5 id="打开deveco-studio点击create-project创建工程或通过deveco-studio的菜单file--new--create-project来创建一个新工程">
     打开DevEco Studio，点击Create Project创建工程。或通过DevEco Studio的菜单File &gt; New &gt; Create Project来创建一个新工程。
    </h5>
    <p>
     选择Application应用开发,选择模板“Empty Ability”，点击Next进行下一步配置。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="692" src="https://i-blog.csdnimg.cn/blog_migrate/7be4391f17f15013dbbbc0a71a1d0bb7.png" width="970"/>
    </p>
    <h5 id="进入配置工程界面compile-sdk选择310api-9model-选择stage其他参数保持默认设置即可">
     进入配置工程界面，Compile SDK选择“3.1.0(API 9)”，Model 选择“Stage”，其他参数保持默认设置即可。
    </h5>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="619" src="https://i-blog.csdnimg.cn/blog_migrate/5aafd32cdabcdc7fb78f490de99e0cba.png" width="873"/>
    </p>
    <p>
     点击Finish，等待十几秒MyApplication工程就创建完成了。
    </p>

     </li>

      entry &gt; src &gt; main &gt; module.json5：Stage模型模块配置文件。主要包含HAP包的配置信息、应用/服务在具体设备上的配置信息以及应用/服务的全局配置信息。

     </li>
     <li>
      entry &gt; hvigorfile.ts：应用级编译构建任务脚本。
     </li>

     创建模拟器
    </h4>
    <p>
     点击DevEco Studio右上角的No Devices &gt; Device Manager，或者点击菜单Tools &gt; Device Manager
    </p>

    </p>

     在Device Manager界面，选择Local Emulator(本地模拟器)，点击 New Emulator
