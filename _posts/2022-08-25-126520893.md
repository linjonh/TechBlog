    <h2 id="1.%E4%BB%80%E4%B9%88%E6%98%AF%E6%89%AB%E9%9B%B7%E6%B8%B8%E6%88%8F%E4%BB%A5%E5%8F%8A%E6%89%AB%E9%9B%B7%E6%B8%B8%E6%88%8F%E6%80%8E%E4%B9%88%E7%8E%A9">
     1.什么是扫雷游戏以及扫雷游戏怎么玩
    </h2>
    <h3 id="(1)%E4%BB%80%E4%B9%88%E6%98%AF%E3%80%8A%E6%89%AB%E9%9B%B7%E3%80%8B">
     (1)什么是《扫雷》
    </h3>
    <blockquote>
     <p>
      《扫雷》是一款大众类的益智小游戏，于1992年发行。游戏目标是在最短的时间内根据点击格子出现的数字找出所有非雷格子，同时避免踩雷，踩到一个雷即全盘皆输。
     </p>
    </blockquote>
    <h3 id="(2)%E3%80%8A%E6%89%AB%E9%9B%B7%E3%80%8B%E8%83%8C%E6%99%AF%E6%95%85%E4%BA%8B">
     <strong>
      (2)《扫雷》背景故事
     </strong>
    </h3>
    <blockquote>
     <p>
      扫雷在科技历史上也扮演了相似的角色。这个基于数字的逻辑谜题最早来自20世纪六七十年代，当时Jerimac Ratliff推出的名为“Cube”的游戏已经非常受人欢迎。几十年后的1992年，扫雷游戏被加入window系统，这并不是为了展示Windows是游戏操作系统专家，而是为了训练用户的鼠标左右键操作能力，让这些动作变得非常自然，并培养鼠标移动的速度和准确性。
     </p>
    </blockquote>
    <h3 id="(3)%E6%80%8E%E4%B9%88%E7%8E%A9%E3%80%8A%E6%89%AB%E9%9B%B7%E3%80%8B">
     (3)怎么玩《扫雷》
    </h3>
    <blockquote>
     <p>
      1. 在格子的数字里，看显示的数字是多少，相邻就有多少个雷。
     </p>
     <p>
      2. 数字1，相对简单，8个格子只有一个是雷。
     </p>
     <p>
      3. 数字2，8个格子有二个是雷。
     </p>
     <p>
      4. 以此类推，基本上3，4，5，6的数字都是这样的意思。
     </p>
     <p>
      5. 当遇到不能确定的情况，我们需要从另一边推进，尽量翻出更多的数字来推理。
     </p>
     <p>
     </p>
    </blockquote>
    <h2 id="2.%E6%89%AB%E9%9B%B7%E6%B8%B8%E6%88%8F%E4%BB%A3%E7%A0%81%E5%AE%9E%E7%8E%B0%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%80%9D%E8%B7%AF">
     <strong>
      2.扫雷游戏代码实现的基本思路
     </strong>
    </h2>
    <h3 id="(1)%E5%9F%BA%E6%9C%AC%E6%80%9D%E8%B7%AF%EF%BC%88%E5%88%9B%E5%BB%BA%E5%87%A0%E4%B8%AA%E6%96%87%E4%BB%B6%E5%8F%8A%E5%85%B6%E8%AE%BE%E8%AE%A1%E6%84%8F%E4%B9%89%EF%BC%89">
     <strong>
      (1)
     </strong>
     基本思路（创建几个文件及其设计意义）
    </h3>
    <blockquote>
     <p>
      test.c文件      扫雷游戏的测试逻辑
     </p>
     <p>
      game.h文件    测试游戏函数的函数声明
     </p>
     <p>
      game.c文件     游戏函数的实现
     </p>
    </blockquote>
    <blockquote>
     <p>
      test.c： 放主函数及其代码的基本框架，以及扫雷的基本逻辑。
     </p>
     <p>
      game.h: 放扫雷游戏函数的函数声明，以及一些头文件声明，方便在test.c和game.c中使用。
     </p>
     <p>
      game.c:实现游戏函数的内容。
     </p>
    </blockquote>
    <h3 id="(2)%E6%89%AB%E9%9B%B7%E6%B8%B8%E6%88%8F%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%A1%86%E6%9E%B6">
     (2)扫雷游戏的基本框架
    </h3>
    <blockquote>
     <p>
      游戏思路有了接下来就要设计游戏框架，进而一步一步的实现游戏，一步步封装游戏。扫雷需要有
      <strong>
       <span style="background-color:#38d8f0;">
        存放雷的数组
       </span>
      </strong>
      ，也需要有查找时，
      <span style="background-color:#38d8f0;">
       周围一圈雷的个数，这又需要存放一个数组
      </span>
      。为什们不存放在一个数组内呢，是因为方便观察呀，还有需要注意的一个点在查找雷时，要找周围一圈雷的个数，
      <span style="background-color:#38d8f0;">
       如果再找最边里的雷查找周围的雷可能会发生数组越界，所以要创建一个大一圈的数组
      </span>
      。创建完两个数组，再进行数组的初始化，为了方便数组的使用，然后肯定需要打印两个棋盘了，然后在一个棋盘上布置雷，然后在排查雷将信息放到另一个数组上，这就是扫雷游戏的基本框架。
     </p>
     <p>
      <strong>
       <span style="background-color:#a2e043;">
        总结
       </span>
      </strong>
     </p>
     <ol>
      <li>
       <strong>
        创建两个数组,一个存放布置好的雷(mine数组),一个存放排查雷的信息(show数组)
       </strong>
      </li>
      <li>
       <strong>
        初始化数组：将mine数组初始化为"0"，布置雷的时候改为"1"(就是雷是1，非雷是0)                        show数组初始化" * " ，排查雷后改为具体数字.
       </strong>
      </li>
      <li>
       <strong>
        打印两个棋盘，方便观察
       </strong>
      </li>
      <li>
       <strong>
        然后在mine数组中随机布置雷
       </strong>
      </li>
      <li>
       <strong>
        再排查雷，将排查雷的信息记录在show数组上
       </strong>
      </li>
      <li>
       <strong>
        判断游戏结果（踩雷或者排雷完成）
       </strong>
      </li>
     </ol>
    </blockquote>

int get_mine_count(char mine[ROWS][COLS],int x,int y)
{
	//用字符的好处可以直接计算
	return (mine[x - 1][y] +
		mine[x - 1][y + 1] +
		mine[x][y + 1] +
		mine[x + 1][y + 1] +
		mine[x + 1][y] +
		mine[x + 1][y - 1] +
		mine[x][y - 1] +
		mine[x + 1][y - 1] - 8 * '0');
}
//排查雷
void find_mine(char mine[ROWS][COLS], char show[ROWS][COLS], int row, int col)
{
	int x = 0;
	int y = 0;
	//win表示除去雷，后无雷的个数
	int win = 0;
	//如果将除去雷的无雷的坐标全部找见，说明排雷成功
	while (win &lt; row * col - EASY_COUNT)
	{
		printf("请输入排查雷的坐标-&gt;");
		scanf("%d%d", &amp;x, &amp;y);
		//查看坐标是否合法
		if (x &gt;= 1 &amp;&amp; x &lt;= row &amp;&amp; y &gt;= 1 &amp; y &lt;= col)
		{
			//看show数组是否被排查过了，如果排查过就重新输入
			if (show[x][y] == '*')
			{
				//判断是否是雷
				if (mine[x][y] == '1')
				{
					printf("很遗憾，你被炸死了\n");
					//游戏失败，打印最后失败的扫雷图。
					print_board(show, ROW, COL);
					break;
				}
				//没有踩到雷，需要计算周围的雷
				else
				{
					//封装一个函数找到周围雷的个数
					int count = get_mine_count(mine, x, y);
					//count为整型，转换为字符'0',就加上'0',参考ASCⅡ表
					show[x][y] = count + '0';
					//查找完然后打印show数组，看清楚周围有几个雷
					print_board(show, ROW, COL);
					//win是无雷的个数，也是出循环的条件
					win++;
				}
			}
			else
			{
				printf("该坐标已被排查过了，请重新输入\n");
			}
		}
		else
		{
			printf("坐标非法，请重新输入\n");
		}
	}
	//判断雷是否都排完了
	if (win == row * col - EASY_COUNT)
	{
		printf("恭喜你，排雷成功\n");
		//查看最终排雷结果
		print_board(show, ROW, COL);
	}
}</code></pre>

					//查找完然后打印show数组，看清楚周围有几个雷
					print_board(show, ROW, COL);
					//win是无雷的个数，也是出循环的条件
					win++;
				}
			}
			else
			{
				printf("该坐标已被排查过了，请重新输入\n");
			}
		}
		else
		{
			printf("坐标非法，请重新输入\n");
		}
	}
	//判断雷是否都排完了
	if (win == row * col - EASY_COUNT)
	{
		printf("恭喜你，排雷成功\n");
		//查看最终排雷结果
		print_board(show, ROW, COL);
	}
}
