    <h3>
     <a id="BC26LWM2M_0">
     </a>
     <strong>
      BC26电信云平台LWM2M接入指南
     </strong>
    </h3>
    <p>
     最近使用移远BC26做了电信云平台的LWM2M介入设计，遇到几个问题，逐一解决，做个记录
    </p>
    <p>
     准备工作
    </p>
    <p>
     相对于移动的OneNet平台，电信平台的LWM2M接入指令完全不同，2者不兼容：
    </p>
    <ol>
     <li>
      <strong>
       电信云平台地址
      </strong>
      ，按照之前的经验首先要确认能连接到的ip地址和端口，某些开发板给的是180.101.147.115，实际上这是个错误地址，注册登录电信平台，给出的有效地址是：221.229.214.202，端口：5683，切记这个地方要配置正确，要不然功亏一篑；
     </li>
     <li>
      电信云平台注册，具体网址：www.ctwing.cn；
     </li>
     <li>
      注册完毕，登录平台，入下图；
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3cc0d8b4894043d3d8b030e0057b6c00.png#pic_center">
       <br/>
       点击右上角：开发者中心，进入如下页面，有个开发向导
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d7feec4ae36c60b97ca0cb8f9ca494b1.png#pic_center">
        <br/>
        在这里选择LWM2M，点击“开始设备接入”，如果已经配置过，也可以左边的导航条直接选“产品中心”，第一次注册需要配置一些个人信息和费用管理，开通一些服务，跟着向导走就行了
       </img>
      </img>
     </li>
     <li>
      进入“产品中心”，创建产品；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5e6fc119d3083eac2c636f795f3aefa0.png#pic_center">
       <br/>
       <strong>
        注意
       </strong>
       要严格参照上面的这组信息配置，尤其是透传和加密选项
       <strong>
        明文
       </strong>
       ，认证方式
       <strong>
        imei
       </strong>
       就可以了，其他一一对应
      </img>
     </li>
     <li>
      至此基本上 创建产品 过程完毕；
     </li>
     <li>
      接下来“添加设备”，如下图；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fab66a190c016d99133a3592f37f166e.png#pic_center">
       <br/>
       在“产品中心”的 ‘设备管理"页面的右上角有4个按钮，选择“添加设备”，就进入设备添加页面
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a4004a1e950c487f7581a04d7a66db55.png#pic_center">
        <br/>
        “设备名称”随便填写，根据你的设备命名规则设计就行了
        <br/>
        “IMEI”号是BC26本身的数据，可以在模块表面直接读出，或者通过软件读出，或者购买时一一对应，填入就可以，其他信息可以不填，确定就好
        <br/>
        <strong>
         软件设计
        </strong>
        <br/>
        流程都差不多
        <br/>
        主要用到几个AT指令：
       </img>
      </img>
     </li>
     <li>
      AT+CGSN=1\r\n 查询 imei码，这个可有可无，根据成需要
     </li>
     <li>
      AT+QLWDEL\r\n 删除之前的链接
     </li>
     <li>
      AT+CGPADDR=1\r\n，查询本机地址
     </li>
     <li>
      AT+QLWSERV=“221.229.214.202”,5683\r\n，跟服务器通过指定端口建立链接
     </li>
     <li>
      AT+QLWCONF=“869098040411492”\r\n，通过发送本机 IMEI 鉴权
     </li>
     <li>
      AT+QLWADDOBJ=19,0,1,“0”\r\n，开启数据上行
     </li>
     <li>
      AT+QLWADDOBJ=19,1,1,“0”\r\n，开启数据下行
     </li>
     <li>
      AT+QLWOPEN=1\r\n，打开设备连接，如果前面 的云端 产品设计端的数据和 设备管理 都正确，这里会返回 OK CONNECT OK, 如果有问题，则返回 OK CONNECT FAIL
      <br/>
      我就是卡在这里，后来测试了华为平台，也是一样问题，返回去修改产品配置，才知道问题所在
     </li>
     <li>
      AT+QLWCFG=“dataformat”,1,1\r\n，最后一条配置指令，配置数据格式的
      <br/>
      10.可以愉快的推送，接收数据了
     </li>
    </ol>
    <p>
     本机硬件是 STM32L431系列低功耗单片机+ BC26的形式，具体开发后续放出
     <br/>
     谢谢收看！
    </p>
