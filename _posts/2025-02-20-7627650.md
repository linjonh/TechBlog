   <div class="htmledit_views" id="content_views">
    <p>
     <span style="color:#575757;background:white;">
      我在《用
     </span>
     <span style="color:#575757;background:white;">
      HTML5
     </span>
     <span style="color:#575757;background:white;">
      新特性开发移动
     </span>
     <span style="color:#575757;background:white;">
      Web App
     </span>
     <span style="color:#575757;background:white;">
      》中全面提到了移动
     </span>
     <span style="color:#575757;background:white;">
      Web App
     </span>
     <span style="color:#575757;background:white;">
      的各种优点，包括：
     </span>
    </p>
    <p>
    </p>
    <ul>
     <li>
      <span style="background-color: white; color: rgb(87, 87, 87); ">
       云端升级
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       跨平台
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       代码复用
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       结合强大的云计算平台的能力
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       绕过
      </span>
      <span style="color:#575757;background:white;">
       App Store
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <span style="color:#575757;background:white;">
      <span style="color: rgb(87, 87, 87); ">
      </span>
      下面的这篇文章也提供了对于各种类型应用的详细分析。
     </span>
    </p>
    <p>
     <span style="color:#575757;background:white;">
      <span style="color: rgb(87, 87, 87); ">
      </span>
      当你开始一个新的移动项目时，选择采用本地（
     </span>
     <span style="color:#575757;background:white;">
      Native
     </span>
     <span style="color:#575757;background:white;">
      ），混合（
     </span>
     <span style="color:#575757;background:white;">
      Hybrid
     </span>
     <span style="color:#575757;background:white;">
      ）或者
     </span>
     <span style="color:#575757;background:white;">
      Web App
     </span>
     <span style="color:#575757;background:white;">
      哪种方案可以说是艰巨的任务。
     </span>
     <span style="color:#575757;background:white;">
     </span>
     <span style="color:#575757;background:white;">
      J Schwan
     </span>
     <span style="color:#575757;background:white;">
      全面分析了每种解决方案的优点和缺点。
     </span>
    </p>
    <p align="left" style="text-align: center;">
     <em>
      <span style="color:#cc0000;">
       移动Web的最大优势是跨平台。你只需要开发一次，这样成本更小。
      </span>
     </em>
    </p>
    <p align="left">
     移动Web优点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       跨平台
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       开放标准
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       没有私有的编程语言
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       更容易重用现有技术
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       不依赖应用商店
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left">
     移动Web缺点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       断网时不能保持状态
      </span>
      <span style="color:#575757;">
       -
      </span>
      <span style="color:#575757;">
       注：
      </span>
      <span style="color:#575757;">
       HTML5
      </span>
      <span style="color:#575757;">
       已经提供支持
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       不能访问（大部分）手机的传感器
      </span>
      <span style="color:#575757;">
       -
      </span>
      <span style="color:#575757;">
       注：
      </span>
      <span style="color:#575757;">
       HTML5
      </span>
      <span style="color:#575757;">
       已经提供支持
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       设备的测试是一个挑战
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       要编写多种实现
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       无法通过应用商店进行市场推广和分发
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left" style="text-align: center;">
     <em>
      <span style="color:#cc0000;">
       当你需要访问手机的传感器时，本地应用非常合适。它适用于你需要丰富的、无与伦比的用户体验和大多数功能都需要断网能正常使用时。
      </span>
     </em>
    </p>
    <p align="left">
     本地应用程序优点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       本地访问手机的传感器
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       本地访问手机的数据
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       离线存储
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       丰富的界面选择
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       App Store
      </span>
      <span style="color:#575757;">
       的商业化
      </span>
      <span style="color:#575757;">
       / App
      </span>
      <span style="color:#575757;">
       购买
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left">
     本机应用程序缺点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       不同的开发技术
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       不同的
      </span>
      <span style="color:#575757;">
       IDE
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       重复的开发成本
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       支持成本增加
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       App Store
      </span>
      <span style="color:#575757;">
       的审核队列
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left" style="text-align: center;">
     <em>
      <span style="color:#cc0000;">
       如果你正在将一个已有的Web App移动化，混合应用非常合适。但是你需要上传应用到商店，需要本地应用覆盖所有平台，移动Web App的功能不需要支持断网状态，以及你需要访问手机的某些硬件功能。
      </span>
     </em>
    </p>
    <p align="left">
     混合应用优点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       大多数内容可移植（基于移动
      </span>
      <span style="color:#575757;">
       Web
      </span>
      <span style="color:#575757;">
       开发）
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       断网状态可用
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       有些屏幕可以通过本地化的方式开发
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       支持桌面通知
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       App Store
      </span>
      <span style="color:#575757;">
       的分发
      </span>
      <span style="color:#575757;">
       /
      </span>
      <span style="color:#575757;">
       可购买
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       降低多平台的成本
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left">
     混合应用缺点：
    </p>
    <p align="left">
    </p>
    <ul>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       移动
      </span>
      <span style="color:#575757;">
       Web
      </span>
      <span style="color:#575757;">
       提供的内容会降低用户体验
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       离线访问对于移动
      </span>
      <span style="color:#575757;">
       Web
      </span>
      <span style="color:#575757;">
       的屏幕来说毫无价值
      </span>
     </li>
     <li>
      <span style="color: rgb(87, 87, 87); ">
       多平台的封装需要熟悉本地和移动
      </span>
      <span style="color:#575757;">
       Web
      </span>
      <span style="color:#575757;">
       的技术
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p align="left" style="text-align: center;">
     <em>
      <span style="color:#cc0000;">
       最后，我们正在朝着移动化的操作系统发展，我们将有工作站来处理移动网络的异常场景。 因此，这会带来骤变。所以当我们在设计新产品时，请更多地考虑移动化支持。
      </span>
     </em>
    </p>
