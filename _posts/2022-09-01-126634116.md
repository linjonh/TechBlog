      再插入一行设置成你想要的端口号，注意不要跟现有端口号重复.
     </li>
     <li>
      将port 22再进行注释掉，只保留5000.
     </li>
    </ul>
    <p>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/blog_migrate/d5d4d68cbb6dc818f086872ca7f32310.png"/>
     <br/>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/blog_migrate/b0722d760267db0adda01003a7fab87b.png"/>
    </p>
    <ul>
     <li>
      重启SSH服务，最好也重启下服务器
     </li>
    </ul>
    <p>
     systemctl restart sshd
     <br/>
     shutdown -r now
    </p>
    <ul>
     <li>
      尝试通过10022端口登录SSH，或者进入该服务器直接本地访问SSH如下：
     </li>
    </ul>
    <p>
     [root@centos7 ~]#ssh root@localhost -p 10022
    </p>

    <h4>
     <a id="13_Linux__WebSSH1721630024sshd_576">
     </a>
     13、在 Linux _Web上，配置SSH安全策略，仅允许172.16.30.0/24网段的用户进行sshd远程连接。
    </h4>
    <ul>
     <li>
      限制IP地址通过SSH登录
     </li>
     <li>
      编辑/etc/hosts.allow文件，添加类似如下的内容，只允许指定IP地址通过SSH登录。
     </li>
    </ul>
    <p>
     sshd:192.168.1.1:allow #允许192.168.1.1这个IP地址通过SSH登录。
     <br/>
     sshd:192.168.2.0/24:allow #允许192.168.2.0/24这个IP地址段通过SSH登录。
     <br/>
     <strong>
      ！！注意：当IP为一个地址的时候不要加/24掩码，只需要写192.168.1.1即可。
     </strong>
     <br/>
     <strong>
      当IP为一个段的时候，需要加/24，如192.168.1.0/24.！！
     </strong>
     <br/>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/blog_migrate/550c34325d2c5059c4ddcdf9715836b0.png"/>
    </p>
    <ul>
     <li>
      <strong>
       然后去修改hosts.deny文件
      </strong>
     </li>
    </ul>
    <p>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/blog_migrate/7a4d27139d0b06ecba28d1cf1cdcba19.png"/>
    </p>
    <pre><code>sshd:ALL       //这里的ALL表示除了上面文件中允许的，其他的IP地址都拒绝
</code></pre>
    <p>
     <strong>
      同时设置上述两个文件时，hosts.allow文件中规则的优先级更高，参考上述两个文件进行设置时，服务器只允许172.16.30.0/24这个IP地址段通过SSH进行登录，其他的IP都会被拒绝SSH登录。
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       重启ssh服务
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      [root@localhost ~]# systemctl restart sshd
     </strong>
    </p>

    <h4>
     <a id="14Linux_WebApache1721610024_597">
     </a>
     14、在Linux_Web上，配置Apache安全策略，禁止网段172.16.10.0/24网段访问。
    </h4>
    <ul>
     <li>
      vim /etc/apache2/apache2.conf
     </li>
     <li>
      在Directory处修改你的网站根目录为/var/www/wwwroot（原本为/var/www/）
     </li>
     <li>
      在Required all granted 下方添加访问规则
     </li>
     <li>
      Order deny,allow //意为先执行deny操作，后allow操作。先拒绝后允许
     </li>
     <li>
      Deny from 172.16.10.0/24 //拒绝172.16.10.0这一网段访问操作。网段拒绝要掩码。
     </li>
     <li>
      Deny from 172.16.10.1 //拒绝172.16.10.1这一IP访问操作。IP拒绝不需要带掩码。
     </li>
     <li>
      如果有多个IP或者网段需要被拒绝，空格后继续添加即可。
     </li>
    </ul>
    <p>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/blog_migrate/f478b9acd07223b3d3dd435824e5d4cf.png"/>
    </p>
