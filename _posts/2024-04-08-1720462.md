    <h1 class="title-article" id="articleContentId">
     微软为什么要“开源”？
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

    <span lang="EN-US" style="FONT-SIZE: 12pt; FONT-FAMILY: 宋体; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">
     7
    </span>
    <span style="FONT-SIZE: 12pt; FONT-FAMILY: 宋体; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">
     月
     <span lang="EN-US">
      26
     </span>
     日，微软
     <span lang="EN-US">
      Bill Hilf
     </span>
     宣布，微软即将把
     <span lang="EN-US">
      “
     </span>
     共享源码许可证
     <span lang="EN-US">
      ”
     </span>
     提交
     <span lang="EN-US">
      OSI
     </span>
     （
     <span lang="EN-US">
      “
     </span>
     国际开源促进会
     <span lang="EN-US">
      ”
     </span>
     ）审批，这是一个非常重要、值得我们密切关注的事件（或进展）。这是为什么？
     <span lang="EN-US">
      <br/>
      6
     </span>
     年前，微软提出
     <span lang="EN-US">
      “Shared Source License”
     </span>
     （
     <span lang="EN-US">
      “
     </span>
     共享源码许可证
     <span lang="EN-US">
      ”
     </span>
     ），讨好国际开源界，但是，被国际开源界人士称为
     <span lang="EN-US">
      “
     </span>
     只许看，不许碰（摸）许可证
     <span lang="EN-US">
      ”
     </span>
     ，嗤之以鼻。对此，微软并不泄气。到
     <span lang="EN-US">
      2005
     </span>
     年年底，微软把这个
     <span lang="EN-US">
      “
     </span>
     共享源码许可证
     <span lang="EN-US">
      ”
     </span>
     一分为三（实际上是五个），即：
     <span lang="EN-US">
      <br/>
      <span style="mso-spacerun: yes">
      </span>
     </span>
     （
     <span lang="EN-US">
      1
     </span>
     ）
     <span lang="EN-US">
      Microsoft Permissive License
     </span>
     （简称
     <span lang="EN-US">
      MS-PL
     </span>
     ）
     <span lang="EN-US">
      ;
      <br/>
      <span style="mso-spacerun: yes">
      </span>
     </span>
     （
     <span lang="EN-US">
      2
     </span>
     ）
     <span lang="EN-US">
      Microsoft Community License
     </span>
     （简称
     <span lang="EN-US">
      MS-CL
     </span>
     ）；
     <span lang="EN-US">
      <br/>
      <span style="mso-spacerun: yes">
      </span>
     </span>
     （
     <span lang="EN-US">
      3
     </span>
     ）
     <span lang="EN-US">
      Microsoft Reference License
     </span>
     （简称
     <span lang="EN-US">
      MS-RL
     </span>
     ）。
     <span lang="EN-US">
      <br/>
     </span>
     其中（
     <span lang="EN-US">
      1
     </span>
     ）
     <span lang="EN-US">
      MS-PL
     </span>
     ，最为宽松，允许用户看（
     <span lang="EN-US">
      View
     </span>
     ）、修改（
     <span lang="EN-US">
      Modify
     </span>
     ）和分发（
     <span lang="EN-US">
      Redistribute
     </span>
     ）源代码，而不论出自商业用途还是非商业用途（目的），类似
     <span lang="EN-US">
      BSD
     </span>
     许可证；（
     <span lang="EN-US">
      2
     </span>
     ）
     <span lang="EN-US">
      MS-CL
     </span>
     ，允许用户看（
     <span lang="EN-US">
      View
     </span>
     ）、修改（
     <span lang="EN-US">
      Modify
     </span>
     ）和分发（
     <span lang="EN-US">
      Redistribute
     </span>
     ）源代码，但是，要求用户保证任何包含它的较大的
     <span lang="EN-US">
      “
     </span>
     作品
     <span lang="EN-US">
      ”
     </span>
     ，也必须按照这个许可证发布，类似
     <span lang="EN-US">
      GNU
     </span>
     的
     <span lang="EN-US">
      GPL
     </span>
     许可证；（
     <span lang="EN-US">
      3
     </span>
     ）
     <span lang="EN-US">
      MS-RL
     </span>
     ，允许用户看（
     <span lang="EN-US">
      View
     </span>
     ）源代码，但不许修改和分发，只许参考。因此，微软才把他叫作
     <span lang="EN-US">
      “
     </span>
     微软参考许可证
     <span lang="EN-US">
      ”
     </span>
     。
     <span lang="EN-US">
      <br/>
     </span>
     其他两个许可证，叫作：
     <span lang="EN-US">
      (4) MS-LPL
     </span>
     ；（
     <span lang="EN-US">
      5
     </span>
     ）
     <span lang="EN-US">
      MS-LCL
     </span>
     ，其中的那个
     <span lang="EN-US">
      “L”
     </span>
     ，表示
     <span lang="EN-US">
      “Limited”
     </span>
     （受限制之意），要求源代码（编译之后）仅能在
     <span lang="EN-US">
      Windows
     </span>
     平台上运行。显然，（
     <span lang="EN-US">
      1
     </span>
     ）、（
     <span lang="EN-US">
      2
     </span>
     ）一定会得到
     <span lang="EN-US">
      OSI
     </span>
     批准，而（
     <span lang="EN-US">
      3
     </span>
     ）一定不会通过
     <span lang="EN-US">
      OSI
     </span>
     的审查。（
     <span lang="EN-US">
      4
     </span>
     ）和（
     <span lang="EN-US">
      5
     </span>
     ）两项，有一点可能性，这就要看
     <span lang="EN-US">
      OSI
     </span>
     是不是对
     <span lang="EN-US">
      Windows
     </span>
     平台
     <span lang="EN-US">
      “
     </span>
     网开一面
     <span lang="EN-US">
      ”
     </span>
     了。细细想来，微软公司的这种许可证，确实是够开放的，源代码都让你（一般是政府、学校）看了，你还能说什么？我们应该看到，（
     <span lang="EN-US">
      3
     </span>
     ）
     <span lang="EN-US">
      MS-RL
     </span>
     许可证（所谓
     <span lang="EN-US">
      “
     </span>
     微软参考许可证
     <span lang="EN-US">
      ”
     </span>
     ）是微软的底线，不能再后退，否则，微软就不是微软了。
     <span lang="EN-US">
      <br/>
     </span>
     当前，人们对国际开源促进会（
     <span lang="EN-US">
      OSI
     </span>
     ）会不会批准微软的请求，都会存在某种程度上的担心。把话说明白了，这些新的微软
     <span lang="EN-US">
      “
     </span>
     软件授权许可证
     <span lang="EN-US">
      ”
     </span>
     会不会改变人们对
     <span lang="EN-US">
      ”
     </span>
     开源
     <span lang="EN-US">
      ”
     </span>
     （
     <span lang="EN-US">
      Open Source
     </span>
     ）的思考（理解或认知）方式？
     <span lang="EN-US">
      7
     </span>
     月
     <span lang="EN-US">
      31
     </span>
     日，著名的《
     <span lang="EN-US">
      BBC
     </span>
     新闻》职业评论家
     <span lang="EN-US">
      Bill Thompson
     </span>
     发表署名文章，题为
     <span lang="EN-US">
      “The long hard road to open source”
     </span>
     （
     <span lang="EN-US">
      “
     </span>
     漫长而艰难的通往开源之路
     <span lang="EN-US">
      ”
     </span>
     ），明确表示：微软的最新举措有可能改变人们对开源的原有认识。但是，
     <span lang="EN-US">
      Thompson
     </span>
     指出，微软决不可能接受自由软件的哲学思想。自由软件奠基人
     <span lang="EN-US">
      Richard Stallman
     </span>
     指出，
     <span lang="EN-US">
      “
     </span>
     开源（软件）只是软件开发的方法论，而自由软件却是一种社会运动
     <span lang="EN-US">
      ”
     </span>
     ，
     <span lang="EN-US">
      “
     </span>
     只有自由软件才会尊重（
     <span lang="EN-US">
      respect
     </span>
     ）用户使用软件的自由性（
     <span lang="EN-US">
      Freedom
     </span>
     ）
     <span lang="EN-US">
      ”
     </span>
     。实际上，微软的
     <span lang="EN-US">
      “
     </span>
     底线
     <span lang="EN-US">
      ”
     </span>
     是允许用户查看、修改（微软软件产品的）源代码，甚至自由地传播它，然而，相关软件中的知识产权却仍然归于微软所有。微软与自由软件（的观念）是格格不入的。
     <span lang="EN-US">
      <br/>
      Thompson
     </span>
     指出，在处处存在
     <span lang="EN-US">
      Web
     </span>
     服务的互联网世界里面，与开发社区保持紧密联系越来越显得重要，毫无疑问，这是微软大步迈向开源（世界）的重要诱因之一。当今，有数以百万计的软件开发人员活跃在国际互联网促成的
     <span lang="EN-US">
      “
     </span>
     网络社区
     <span lang="EN-US">
      ”
     </span>
     中，潜在着无穷的思维创新的巨大能量。对此，微软不可小视。微软勇敢步入开源世界，就是很自然的结论。
     <span lang="EN-US">
      <br/>
      Thompson
     </span>
     认为，微软一而再，再而三公开批地评（甚至诬蔑）自由软件必定受到一些人的坚决反对，因为，这些人认识到，在当今网络社会，计算机软件确实太重要了，如果这些软件成为某个公司的
     <span lang="EN-US">
      “
     </span>
     单独所有
     <span lang="EN-US">
      ”
     </span>
     ，即便这个公司
     <span lang="EN-US">
      “
     </span>
     支持
     <span lang="EN-US">
      ”
     </span>
     开源，也是非常危险的事情。文章最后，
     <span lang="EN-US">
      Thompson
     </span>
     寄希望于自由软件的发展和繁荣。
     <span lang="EN-US">

      <br style="mso-special-character: line-break"/>
