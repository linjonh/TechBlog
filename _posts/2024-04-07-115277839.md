     <img alt="dbs" src="https://i-blog.csdnimg.cn/blog_migrate/8d5077a812b16716060af2cdd71468e9.png#pic_center"/>
    </p>
    <p>
     大家好，我是只谈技术不剪发的 Tony 老师。
    </p>
    <p>
     著名的数据库流行度排名网站
     <a href="https://db-engines.com/en/" rel="nofollow">
      DB-Engines
     </a>
     统计了 360 多种数据库产品。那么为什么会出现这么多数据库呢？让我们来看看它们自己是怎么说的吧：
    </p>
    <p>
     <a href="https://www.oracle.com/database/" rel="nofollow">
      <font color="red">
       Oracle
      </font>
     </a>
     ：企业需要数据库（Oracle 是第一个商用的关系型数据库）；
    </p>
    <p>
     <a href="https://www.mysql.com/" rel="nofollow">
      <font color="#188299">
       My
      </font>
      <font color="#F5B054">
       SQL
      </font>
     </a>
     ：Oracle 不开源（MySQL 是最受欢迎的开源关系型数据库）；
    </p>
    <p>
     <a href="https://mariadb.org/" rel="nofollow">
      <font color="#C0765A">
       MariaDB
      </font>
     </a>
     ：我们喜欢 MySQL，但是不喜欢 Oracle（MySQL 被 Oracle 收购，MySQL之父 Monty 创建了 MariaDB 分支）；
    </p>
    <p>
     <a href="https://www.postgresql.org/" rel="nofollow">
      <font color="#326690">
       PostgreSQL
      </font>
     </a>
     ：MySQL 的功能不够完善（PostgreSQL 是最先进的开源关系型数据库）；
    </p>
    <p>
     <a href="https://www.microsoft.com/zh-cn/sql-server/" rel="nofollow">
      <font color="#DF1821">
       Microsoft SQL Server
      </font>
     </a>
     ：我们拥抱 Linux（Microsoft SQL Server 2017 开始提供了 Linux 平台版本）；
    </p>
    <p>
     <a href="https://www.sqlite.org/" rel="nofollow">
      <font color="#0F80CC">
       SQLite
      </font>
     </a>
     ：一切皆可嵌入，而且 4 种数据类型就够用了（SQL 是安装最多的关系型数据库，使用动态数据类型）；
    </p>
    <p>
     <a href="https://www.ibm.com/cn-zh/analytics/db2" rel="nofollow">
      <font color="black">
       IBM
      </font>
      <font color="#009A2C">
       Db2
      </font>
     </a>
     ：天下 SQL 出吾门（关系模型和 SQL 语言都来自 IBM 公司）；
    </p>
    <p>
     <a href="https://www.mongodb.com/" rel="nofollow">
      <font color="#439934">
       MongoDB
      </font>
     </a>
     ：为什么要用 JOIN 和模式（Schema）？（MongoDB 是最著名的文档数据库，采用模式自由的面向集合存储）；
    </p>
    <p>
     <a href="https://couchdb.apache.org/" rel="nofollow">
      <font color="#D52029">
       Apache
      </font>
      <font color="#E32428">
       CouchDB
      </font>
     </a>
     ：为什么要有集合（Collection）?（支持多主复制的高可靠性文档数据库，没有集合的概念）；
    </p>
    <p>
     <a href="https://redis.io/" rel="nofollow">
      <font color="#DB2B21">
       Redis
      </font>
     </a>
     ：为什么要用文档（Document）？（Redis 是最著名的键/值存储数据库、缓存和消息中间件）；
    </p>
    <p>
     <a href="https://memcached.org/" rel="nofollow">
      <font color="#298D83">
       Memcached
      </font>
     </a>
     ：为什么我们要用硬盘？（Memcached 是一个分布式内存对象缓存系统，不支持数据持久化）；
    </p>
    <p>
     <a href="https://aws.amazon.com/cn/dynamodb/" rel="nofollow">
      <font color="#F89C33">
       Amazon DynamoDB
      </font>
     </a>
     ：为什么要自己安装数据库？（DynamoDB 是 Amazon 提供的键/值和文档数据库托管服务）；
    </p>
    <p>
     Bigtable：MongoDB 对 Web 应用的扩展性不够好（Google 说的）；
    </p>
    <p>
     <a href="https://hbase.apache.org/" rel="nofollow">
      <font color="#D52029">
       Hbase
      </font>
     </a>
     ：Bigtable 不开源；
    </p>
    <p>
     <a href="https://cassandra.apache.org/" rel="nofollow">
      <font color="#BBE6FB">
       Cassandra
      </font>
     </a>
     ：Bigtable 不是由 Facebook 开发的；
    </p>
    <p>
     <a href="https://riak.com/index.html" rel="nofollow">
      Riak
     </a>
     ：Cassandra 不是用 Erlang 语言编写的；
    </p>
    <p>
     <a href="https://neo4j.com/" rel="nofollow">
      <font color="#008CC1">
       Neo4j
      </font>
     </a>
     ：SQL 不够关系！（著名的图形数据库，以图结构存储对象之间的关系）；
    </p>
    <p>
     <a href="https://orientdb.com/" rel="nofollow">
      OrientDB
     </a>
     ：让我们把所有东西（文档、图、K-V、对象）都放到同一个数据库里吧；
    </p>
    <p>
     <a href="https://www.elastic.co/cn/elasticsearch/" rel="nofollow">
      Elasticsearch
     </a>
     ：MongoDB 搜索功能不够强大（Elasticsearch 是一个分布式大数据搜索和数据分析引擎）；
    </p>
    <p>
     <a href="https://lucene.apache.org/solr/" rel="nofollow">
      Solr
     </a>
     对 Elasticsearch 说：本是同根生，相煎何太急！（Solr 和 Elasticsearch 都是基于 Apache Lucene 构建）；
    </p>
    <p>
     NewSQL：我们是数据库领域中的后浪🌊🌊🌊！
    </p>
    <p>
     。。。
    </p>
