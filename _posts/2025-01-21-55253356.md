     Python2 中文unicode乱码问题

   <div class="htmledit_views" id="content_views">
    <div class="markdown_body" style="margin-top:0px!important; margin-bottom:0px!important">
     <p>
      <br/>
      在 Sublime Text 里用 Cmd (Ctrl) + B 运行代码
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      ，想要打印出 unicode 类型的字符串时，会出现以下报错：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="x">UnicodeEncodeError: 'ascii' codec can't encode characters in position 0-1: ordinal not in range(128)</span>
</pre>
     </div>
     <p>
      <em>
       传说中的 Python 2 编码坑（笑）
      </em>
     </p>
     <p>
      而同样的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      代码在 Mac 的终端里却能正常打印出
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       “中文”
      </code>
      结果，没有任何报错。
     </p>
     <p>
      虽然在网上能查到多种解决方法，但一直以来知其然而不知其所以然，不了解为什么那些方法能解决问题的真正原因，也不知道为什么同样的代码在终端里就可以运行而在 Sublime Text 里就不行了？
     </p>
     <p>
      因此我研究学习了下这个问题相关的一些 Python 2 编码问题，在这里分享下我的理解。
     </p>
     <p>
      以下属于新手向，参考了网上多篇文章，如有错误，望指正。
     </p>
     <p>
      <strong>
       先说下我的环境：
      </strong>
     </p>
     <ul style="list-style:square; margin:1em 0px 1em 1em; padding:0px">
      <li style="padding:0px; margin:0px">
       Mac OS X
      </li>
      <li style="padding:0px; margin:0px">
       Python 2.7
      </li>
      <li style="padding:0px; margin:0px">
       Sublime Text 3
      </li>
     </ul>
     <h3 style="font-size:18px; font-weight:500; line-height:18px; margin:20px 0px; padding:0px 0px 8px; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(226,226,226)">
      <a name="t1" target="_blank">
      </a>
      分析：
     </h3>
     <p>
      Python 在向控制台 (console)
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      的时候，因为控制台只能看得懂由 bytes（字节序列）组成的字符串，而 Python 中 "unicode" 对象存储的是 code points（码点），因此 Python 需要将输出中的 "unicode" 对象用编码转换为储存 bytes（字节序列）的 "str" 对象后，才能进行输出。
     </p>
     <p>
      而在报错里看到
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       UnicodeEncodeError
      </code>
      ， 那就说明 Python 在将 unicode 转换为 str 时使用了错误的编码。而为什么是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       'ascii'
      </code>
      编码呢？那是因为 Python 2 的默认编码就是 ASCII，可以通过以下命令来查看 Python 的默认编码：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="kn" style="color:rgb(0,128,0); font-weight:bold">import</span> <span class="nn" style="color:rgb(0,0,255); font-weight:bold">sys</span>
<span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">getdefaultencoding</span><span class="p">()</span>
<span class="n">ascii</span>
</pre>
     </div>
     <p>
      所以此时在 Sublime Text 里运行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      ，实际上等于是运行了：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="s" style="color:rgb(186,33,33)">u'中文'</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encode</span><span class="p">(</span><span class="s" style="color:rgb(186,33,33)">'ascii'</span><span class="p">)</span>
</pre>
     </div>
     <p>
      ASCII 编码无法对 unicode 的中文进行编码，因此就报错了。
     </p>
     <p>
      那为什么同样的代码
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      在 Mac 的终端里却能正常输出中文，难道是因为终端下的 Python 2 的默认编码不是 ASCII？非也，在终端下运行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.getdefaultencoding()
      </code>
      结果一样是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       ascii
      </code>
      。那同样是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       ascii
      </code>
      为什么会有不同的结果？难倒这里 Python 用了另外一个编码来转换？
     </p>
     <p>
      是的，其实 Python 在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print unicode
      </code>
      时真正涉及到的是另一组编码：stdin/stdout/stderr 的编码，也就是标准输入、标准输出和标准错误输出的编码。可以通过以下命令来查看，这里是在我的终端下运行的结果：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="kn" style="color:rgb(0,128,0); font-weight:bold">import</span> <span class="nn" style="color:rgb(0,0,255); font-weight:bold">sys</span>
<span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdin</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span>
<span class="n">UTF</span><span class="o" style="color:rgb(102,102,102)">-</span><span class="mi" style="color:rgb(102,102,102)">8</span>
<span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdout</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span>
<span class="n">UTF</span><span class="o" style="color:rgb(102,102,102)">-</span><span class="mi" style="color:rgb(102,102,102)">8</span>
<span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stderr</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span>
<span class="n">UTF</span><span class="o" style="color:rgb(102,102,102)">-</span><span class="mi" style="color:rgb(102,102,102)">8</span>
</pre>
     </div>
     <p>
      在正常情况下，Python 2 在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print unicode
      </code>
      时用来转换的编码并不是 Python 的默认编码
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.getdefaultencoding()
      </code>
      ，而是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      所设的编码。
     </p>
     <p>
      因为在我的终端下 Python 的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      编码是 UTF-8，所以在终端里运行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      时，实际上是等于运行了：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="s" style="color:rgb(186,33,33)">u'中文'</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encode</span><span class="p">(</span><span class="s" style="color:rgb(186,33,33)">'UTF-8'</span><span class="p">)</span>
</pre>
     </div>
     <p>
      编码正确，运行正常，因此没有报错。
     </p>
     <p>
      在类 UNIX 系统下，Python 应该是通过环境变量
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       LC_CTYPE
      </code>
      来判断 stdin/stdout/stderr 的编码的。因此一般只要将 shell 的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       LANG
      </code>
      环境变量设置对为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       **_**.UTF-8
      </code>
      后，应该就能在终端里直接
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      出 unicode 类型的字符串了，而不需要在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      时手动加上
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       .encode('utf-8')
      </code>
      进行编码了。
     </p>
     <p>
      但在 Sublime Text 里事情就没那么美好了。在 Sublime Text 里运行查看 stdout 编码的命令，发现：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="kn" style="color:rgb(0,128,0); font-weight:bold">import</span> <span class="nn" style="color:rgb(0,0,255); font-weight:bold">sys</span>
<span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdout</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span>
<span class="o" style="color:rgb(102,102,102)">-----------------------------</span><span class="s" style="color:rgb(186,33,33)">"""</span>
<span class="s" style="color:rgb(186,33,33)">None</span>
<span class="s" style="color:rgb(186,33,33)">[Finished in 0.1s]</span>
</pre>
     </div>
     <p>
      结果甚至不是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       'ascii'
      </code>
      而是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       None
      </code>
      。可能是因为 Sublime Text 的 Build System 是用 subprocess.Popen 来运行 Python 的，导致 Python 无法判断出正确的 stdin/stdout/stderr 编码，于是都变成 None 了。
     </p>
     <p>
      这种情况也发生在输出的目标是管道的情况下：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="nv" style="color:rgb(25,23,124)">$ </span>python -c <span class="s1" style="color:rgb(186,33,33)">'import sys; print sys.stdout.encoding'</span> <span class="p">|</span> tee /tmp/foo.txt
None
</pre>
     </div>
     <p>
      那么在这种
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       None
      </code>
      情况下的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print unicode
      </code>
      怎么办呢？答案就是 Python 只能很无奈地使用
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.getdefaultencoding()
      </code>
      的默认编码
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       ascii
      </code>
      来对 unicode 进行转换了。这样就出现了本文开头所说的那个
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       UnicodeEncodeError
      </code>
      问题。
     </p>
     <hr style="border:none; height:1px; color:rgb(226,226,226); background-color:rgb(226,226,226); margin-bottom:1em"/>
     <h4 style="font-size:16px; font-weight:500; line-height:16px; margin:5px 0px 20px; padding:0px">
      <a name="t2" target="_blank">
      </a>
      总结一下 Python 2 向控制台
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      输出时的流程：
     </h4>
     <ol style="margin:1em 0px 0em 2em; padding:0px">
      <li style="padding:0px; margin:0px">
       Python 启动时，当它发现当前的输出是连接到控制台的时候，它会根据一些环境变量，例如环境变量
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        LC_CTYPE
       </code>
       ，来设法判断出
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        sys.stdin/stdout/stderr.encoding
       </code>
       编码值。
      </li>
      <li style="padding:0px; margin:0px">
       当 Python 无法判断出所需的编码时，它会将
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        sys.stdin/stdout/stderr.encoding
       </code>
       的值设置为
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        None
       </code>
       。
      </li>
      <li style="padding:0px; margin:0px">
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        print
       </code>
       时判断字符串是否是 unicode 类型。
      </li>
      <li style="padding:0px; margin:0px">
       如果是的话，并且
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        sys.stdout.encoding
       </code>
       不为
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        None
       </code>
       时，就使用
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        sys.stdout.encoding
       </code>
       编码对 unicode 编码成 str 后输出。
      </li>
      <li style="padding:0px; margin:0px">
       <p>
        如果
        <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
         sys.stdout.encoding
        </code>
        为
        <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
         None
        </code>
        的话，就使用
        <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
         sys.getdefaultencoding()
        </code>
        默认编码来对 unicode 进行转换成 str 后输出。
       </p>
       <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
        <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="k" style="color:rgb(0,128,0); font-weight:bold">if</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdout</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span><span class="p">:</span>
    <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="nb" style="color:rgb(0,128,0)">unicode</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encode</span><span class="p">(</span><span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdout</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span><span class="p">)</span>
<span class="k" style="color:rgb(0,128,0); font-weight:bold">else</span><span class="p">:</span>
    <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="nb" style="color:rgb(0,128,0)">unicode</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encode</span><span class="p">(</span><span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">getdefaultencoding</span><span class="p">())</span>
</pre>
       </div>
      </li>
     </ol>
     <p>
      以上参考：
      <a href="https://wiki.python.org/moin/PrintFails" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank">
       https://wiki.python.org/moin/PrintFails
      </a>
     </p>
     <h3 style="font-size:18px; font-weight:500; line-height:18px; margin:20px 0px; padding:0px 0px 8px; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(226,226,226)">
      <a name="t3" target="_blank">
      </a>
      解决方法：
     </h3>
     <h4 style="font-size:16px; font-weight:500; line-height:16px; margin:5px 0px 20px; padding:0px">
      <a name="t4" target="_blank">
      </a>
      解决方法 1：
     </h4>
     <p>
      先说最不正确的解决方法：在文件头部加上
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="kn" style="color:rgb(0,128,0); font-weight:bold">import</span> <span class="nn" style="color:rgb(0,0,255); font-weight:bold">sys</span>
<span class="nb" style="color:rgb(0,128,0)">reload</span><span class="p">(</span><span class="n">sys</span><span class="p">)</span>
<span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">setdefaultencoding</span><span class="p">(</span><span class="s" style="color:rgb(186,33,33)">'utf-8'</span><span class="p">)</span>
</pre>
     </div>
     <p>
      这种方法通过 dirty hack 的方式在 Python 刚启动时更改了 Python 的默认编码为 utf-8。此后：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="o" style="color:rgb(102,102,102)">&gt;&gt;&gt;</span> <span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">getdefaultencoding</span><span class="p">()</span>
<span class="n">utf</span><span class="o" style="color:rgb(102,102,102)">-</span><span class="mi" style="color:rgb(102,102,102)">8</span>
</pre>
     </div>
     <p>
      但就本文所讨论的问题来说，这个方法并不是真正地直接解决了问题。就如上述所说，Python 只是在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       None
      </code>
      时才会使用默认编码来转换需要
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      的 unicode 字符串。那万一在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      存在，但为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       ascii
      </code>
      的情况下呢？这样即使更改了 Python 的默认编码，同样还是会出现
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       UnicodeEncodeError
      </code>
      报错。所以对本问题来说，这个方法治标不治本。
     </p>
     <p>
      除此之外，很多人都用这个方法来解决 Python 2 下遇到的其它各种各样的编码问题，在 v2ex 的各种 Python 编码问题讨论帖中也常常能见到有人推荐用这个方法来解决问题的。
      <br/>
      但实际上很多大牛都不推荐用这个方法来解决 Python 2 的编码问题，这里引用下
      <a href="http://stackoverflow.com/questions/3828723/why-we-need-sys-setdefaultencodingutf-8-in-a-py-script/3828742#3828742" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Why we need sys.setdefaultencoding(“utf-8”) in a py script?">
       StackOverflow 相关回答
      </a>
      里的一句话：
     </p>
     <blockquote>
      <p>
       the use of sys.setdefaultencoding() has always been discouraged
      </p>
     </blockquote>
     <p>
      为什么这个方法不被推荐呢？我们来看下 Python 文档里对这个 function 是怎么说的：
     </p>
     <blockquote>
      <p>
       This function is only intended to be used by the site module implementation and, where needed, by sitecustomize. Once used by the site module, it is removed from the sys module’s namespace.
      </p>
     </blockquote>
     <p>
      可以看到这个方法原本就不是用户向的方法，并没有打算让用户用这个方法来更改 Python 2 的默认编码。
     </p>
     <p>
      那为什么不建议我们更改 Python 的默认编码呢？
      <br/>
      这里引用 Python 核心开发者、Python Unicode 支持的设计者和实现者：
      <a href="https://wiki.python.org/moin/MarcAndreLemburg" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Marc-André Lemburg">
       Marc-André Lemburg
      </a>
      ，他在一个
      <a href="http://article.gmane.org/gmane.comp.python.devel/109916" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Re: Proposed downstream change to site.py in Fedora (sys.defaultencoding)">
       邮件列表上的回复
      </a>
      ：
     </p>
     <blockquote>
      <p>
       The only supported default encodings in Python are:
      </p>
      <p>
       Python 2.x: ASCII
       <br/>
       <br/>
       Python 3.x: UTF-8
      </p>
      <p>
       If you change these, you are on your own and strange things will
       <br/>
       start to happen. The default encoding does not only affect
       <br/>
       the translation between Python and the outside world, but also
       <br/>
       all internal conversions between 8-bit strings and Unicode.
      </p>
      <p>
       Hacks like what's happening in the pango module (setting the
       <br/>
       default encoding to 'utf-8' by reloading the site module in
       <br/>
       order to get the sys.setdefaultencoding() API back) are just
       <br/>
       downright wrong and will cause serious problems since Unicode
       <br/>
       objects cache their default encoded representation.
      </p>
      <p>
       Please don't enable the use of a locale based default encoding.
      </p>
      <p>
       If all you want to achieve is getting the encodings of
       <br/>
       stdout and stdin correctly setup for pipes, you should
       <br/>
       instead change the .encoding attribute of those (only).
      </p>
      <p>
       --
       <br/>
       <br/>
       Marc-Andre Lemburg
       <br/>
       <br/>
       eGenix.com
      </p>
     </blockquote>
     <p>
      从此可见，Python 2 唯一支持的内部编码只有 ASCII，更改其默认编码为其它编码可能会导致各种各样奇怪的问题。在这里他也说了使用
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.setdefaultencoding()
      </code>
      的方法是彻彻底底的错误，正确的方法应该是更改 stdout 和 stdin 的编码。
     </p>
     <p>
      所以这个方法是最不正确的填坑方法，请大家慎用。
     </p>
     <hr style="border:none; height:1px; color:rgb(226,226,226); background-color:rgb(226,226,226); margin-bottom:1em"/>
     <h4 style="font-size:16px; font-weight:500; line-height:16px; margin:5px 0px 20px; padding:0px">
      <a name="t5" target="_blank">
      </a>
      解决方法 2：
     </h4>
     <p>
      然后说说应当是姿势最正确的、也是大家都懂的方法：
     </p>
     <p>
      在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      的时候显式地用正确的编码来对 unicode 类型的字符串进行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       encode('正确的编码'）
      </code>
      为 str 后， 再进行输出。
      <br/>
      而在
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      的时候，这个正确的编码一般就是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdout.encoding
      </code>
      的值。但也正如上述所说，这个值并不是一直是可靠的，因此需要根据所使用的平台和控制台环境来判断出这个正确的编码。
      <br/>
      <br/>
      而在 Mac 下这个正确的编码一般都是 utf-8，因此若不考虑跨环境的话，可以无脑地一直用 encode('utf-8') 和 decode('utf-8') 来进行输入输出转换。
     </p>
     <p>
      在我的经验中，这个策略也是解决 Python 2 其它 unicode 相关编码问题的最佳方法。在
      <a href="http://nedbatchelder.com/text/unipain.html" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Pragmatic Unicode or How do I stop the Pain?">
       PyCon 2012 的一个演讲
      </a>
      中（关于 Python Unicode 问题很好的一个演讲，
      <a href="http://pycoders-weekly-chinese.readthedocs.org/en/latest/issue5/unipain.html" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Unicode之痛">
       这里有演讲稿的中文翻译版
      </a>
      ），对这个方法有一个很形象的比喻：
     </p>
     <p>
      <img alt="Unicode sandwich" src="http://i.imgur.com/sCV7VzU.png" style="max-width:660px" title="Unicode sandwich"/>
     </p>
     <p>
      因为在程序中进进出出的只有存储 bytes（字节序列）的 str。因此最好的策略是将输入的 bytes 马上解码成 unicode，而在程序内部中均使用 unicode，而当在进行输出的时候，尽早将之编码成 bytes。
     </p>
     <p>
      也就是要形成一个 Unicode 三明治（如图）， bytes 在外， Unicode 在内。在边界的地方尽早进行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       decode
      </code>
      和
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       encode
      </code>
      。不要在内部混用 str 和 unicode，尽可能地让程序处理的字符串都为 Unicode。
     </p>
     <hr style="border:none; height:1px; color:rgb(226,226,226); background-color:rgb(226,226,226); margin-bottom:1em"/>
     <h4 style="font-size:16px; font-weight:500; line-height:16px; margin:5px 0px 20px; padding:0px">
      <a name="t6" target="_blank">
      </a>
      解决方法 3：
     </h4>
     <p>
      虽然解决方法 2 是最正确的方式，但是有时候在 Sublime Text 里调试些小脚本，实在是懒得再在每个
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print
      </code>
      语句后面写一个尾巴
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       .encode('utf-8')
      </code>
      。那么有没有办法能让 Sublime Text 像在终端里一样直接就能
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      呢？也就是说能不能解决
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdin/stdout/stderr.encoding
      </code>
      为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       None
      </code>
      的情况呢？
     </p>
     <p>
      答案肯定是有的，一种方法是用类似更改默认编码的方法一样，用 dirty hack 的方式在 Python 代码中去显式地更改
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdin/stdout/stderr.encoding
      </code>
      的值。一样是不推荐，我也没尝试过，在这里就不详说了。
     </p>
     <p>
      另一种方法则是通过设置
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       PYTHONIOENCODING
      </code>
      环境变量来强制要求 Python 设置 stdin/stdout/stderr 的编码值为我们想要的，这是一个相对比较干净的解决方法。见文档：
     </p>
     <blockquote>
      <p>
       PYTHONIOENCODING
       <br/>
       <br/>
       Overrides the encoding used for stdin/stdout/stderr, in the syntax encodingname:errorhandler. The :errorhandler part is optional and has the same meaning as in str.encode().
      </p>
      <p>
       New in version 2.6.
      </p>
     </blockquote>
     <p>
      在 Mac 下对全局 GUI 程序设置环境变量的方法是：使用
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       launchctl setenv &lt;&lt;key&gt; &lt;value&gt;, ...&gt;
      </code>
      命令对所有 launchd 启动的未来子进程设置环境变量。
     </p>
     <p>
      在这里顺便科普下，为什么对所有 launchd 启动的未来子进程设置环境变量可以使得对 Mac 下所有 GUI 程序生效。这是因为 launchd 是 OS X 系统启动后运行的第一个非内核进程。我们可以在 activity monitor（活动监视器）里看到，它的 pid 是很帅气的 1。而之后所有的进程都将是它的子进程。
      <br/>
      另外还可以通过 launchd 在 Mac 下实现类 crontab 的功能。
     </p>
     <p>
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       launchctl setenv
      </code>
      命令设置的全局环境变量会在电脑重启后失效，因此就需要通过上面说的 launchd 的开机启动任务的功能来在重启后再设置一遍环境变量，
      <a href="http://www.dowdandassociates.com/blog/content/howto-set-an-environment-variable-in-mac-os-x-launchd-plist/" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="HowTo: Set an Environment Variable in Mac OS X - launchd.plist">
       其配置方法可以参考这里
      </a>
      。也因为这个原因，我并没有使用这个方法来设置
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       PYTHONIOENCODING
      </code>
      环境变量。
     </p>
     <p>
      而 Sublime Text 提供了一个设置 Build System 环境变量的方法，这个方法各平台的 Sublime Text 都适用。
     </p>
     <p>
      <strong>
       设置 Sublime Text 的 Python Build System 环境变量的步骤如下：
      </strong>
     </p>
     <ol style="margin:1em 0px 0em 2em; padding:0px">
      <li style="padding:0px; margin:0px">
       将 Sublime Text 默认的 Python Build System 的配置文件 Python.sublime-build（找到这个文件的最好方法是安装插件
       <a href="https://packagecontrol.io/packages/PackageResourceViewer" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="PackageResourceViewer">
        PackageResourceViewer
       </a>
       ）复制一份到 Sublime Text 的 /Packages/User 文件夹下（在 Mac 和 Sublime Text 3 下这个路径是 ~/Library/Application Support/Sublime Text 3/Packages/User）。
      </li>
      <li style="padding:0px; margin:0px">
       打开编辑新复制来的 Python.sublime-build 文件，如下加上一行设置
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        PYTHONIOENCODING
       </code>
       环境变量为 UTF-8 编码的内容，并保存：
      </li>
     </ol>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="p">{<!-- --></span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"shell_cmd"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"python -u \"$file\""</span><span class="p">,</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"file_regex"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"^[ ]*File \"(...*?)\", line ([0-9]*)"</span><span class="p">,</span>
<span class="err" style="border:1px solid rgb(255,0,0)">+</span>   <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"env"</span><span class="p">:</span> <span class="p">{<!-- --></span><span class="nt" style="color:rgb(0,128,0); font-weight:bold">"PYTHONIOENCODING"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"utf8"</span><span class="p">},</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"selector"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"source.python"</span>
<span class="p">}</span>
</pre>
     </div>
     <p>
      这样一来终于在这么长的文章后能在 Sublime Text 里直接运行
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'中文'
      </code>
      ，而不用再出现万恶的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       UnicodeEncodeError
      </code>
      了。
     </p>
     <hr style="border:none; height:1px; color:rgb(226,226,226); background-color:rgb(226,226,226); margin-bottom:1em"/>
     <p>
      既然都研究到这了，不妨我们试试把
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       PYTHONIOENCODING
      </code>
      设置成其它编码看看会出现什么情况，例如设置成简体中文 Windows 的默认编码 cp936：
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       "env": {"PYTHONIOENCODING": "cp936"}
      </code>
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="kn" style="color:rgb(0,128,0); font-weight:bold">import</span> <span class="nn" style="color:rgb(0,0,255); font-weight:bold">sys</span>
<span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="n">sys</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">stdout</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">encoding</span>
<span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="s" style="color:rgb(186,33,33)">u'你好'</span>
<span class="o" style="color:rgb(102,102,102)">----------------------------------</span><span class="s" style="color:rgb(186,33,33)">"""</span>
<span class="s" style="color:rgb(186,33,33)">cp936</span>
<span class="s" style="color:rgb(186,33,33)">[Decode error - output not utf-8]</span>
<span class="s" style="color:rgb(186,33,33)">[Finished in 0.1s]</span>
</pre>
     </div>
     <p>
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       [Decode error - output not utf-8]
      </code>
      ，这就是 Sublime Text 在 Windows 下可能会出现的问题（例如这两位同学
      <a href="http://www.v2ex.com/t/45391" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank">
       /t/45391
      </a>
      <a href="http://www.v2ex.com/t/88428" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank">
       /t/88428
      </a>
      ）。这是因为 Sublime Text 的 Build System 默认是用 utf-8 编码去解读运行的输出的，而我们指定了让 Python 用 cp936 编码来生成 str 字符串进行输出，那么就会出现 Sublime Text 无法识别输出的情况了。
      <br/>
      同样在对终端
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       export PYTHONIOENCODING=cp936
      </code>
      后，在终端下
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       print u'你好'
      </code>
      输出的就会是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       ���
      </code>
      这样的乱码。
     </p>
     <p>
      解决办法之一就是同样在 Python.sublime-build 文件里设置
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       "env": {"PYTHONIOENCODING": "utf8"}
      </code>
      来使得输出统一为 utf-8。
     </p>
     <p>
      或者是更改 Sublime Text 的 Build System 所接受的输出编码，将其改为一致的 cp936 编码，同样也是更改 Python.sublime-build 文件，加入一行：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="p">{<!-- --></span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"shell_cmd"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"python -u \"$file\""</span><span class="p">,</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"file_regex"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"^[ ]*File \"(...*?)\", line ([0-9]*)"</span><span class="p">,</span>
<span class="err" style="border:1px solid rgb(255,0,0)">+</span>   <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"encoding"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"cp936"</span><span class="p">,</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"selector"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"source.python"</span>
<span class="p">}</span>
</pre>
     </div>
     <hr style="border:none; height:1px; color:rgb(226,226,226); background-color:rgb(226,226,226); margin-bottom:1em"/>
     <p>
      那我们再试试把这两个设置同时都加到 Python.sublime-build 文件里，也就是让 Python 输出 utf8 编码的字符串，而让 Sublime Text 用 cp936 编码来解读，看看会发生什么情况？
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="p">{<!-- --></span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"shell_cmd"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"python -u \"$file\""</span><span class="p">,</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"file_regex"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"^[ ]*File \"(...*?)\", line ([0-9]*)"</span><span class="p">,</span>
<span class="err" style="border:1px solid rgb(255,0,0)">+</span>   <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"env"</span><span class="p">:</span> <span class="p">{<!-- --></span><span class="nt" style="color:rgb(0,128,0); font-weight:bold">"PYTHONIOENCODING"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"utf8"</span><span class="p">},</span>
<span class="err" style="border:1px solid rgb(255,0,0)">+</span>   <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"encoding"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"cp936"</span><span class="p">,</span>
    <span class="nt" style="color:rgb(0,128,0); font-weight:bold">"selector"</span><span class="p">:</span> <span class="s2" style="color:rgb(186,33,33)">"source.python"</span>
<span class="p">}</span>
</pre>
     </div>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="k" style="color:rgb(0,128,0); font-weight:bold">print</span> <span class="s" style="color:rgb(186,33,33)">u'你好'</span>

</pre>
     </div>
     <p>
      笑，居然不是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       [Decode error - output not cp936]
      </code>
      ，而是这么喜感的
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       “浣犲ソ”
      </code>
      ！
      <br/>
      <br/>
      这是因为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       “你好”
      </code>
      的 utf-8 编码刚好和
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       “浣犲ソ”
      </code>
      的 cp936 编码重合了，都是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       '\xe6\xb5\xa3\xe7\x8a\xb2\xe3\x82\xbd'
      </code>
      ，所以使用 cp936 编码去解读的 Sublime Text 就认为这段字符串就是
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       “浣犲ソ”
      </code>
      而显示了出来。
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">

     <h3 style="font-size:18px; font-weight:500; line-height:18px; margin:20px 0px; padding:0px 0px 8px; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(226,226,226)">
      <a name="t7" target="_blank">
      </a>
      附带解决的问题：IDLE 的交互模式里无法输入中文
     </h3>
     <p>
      我偶尔会用 Python 2 自带的 IDLE 快速测试一两行代码，但在我的 Mac 下的 IDLE 交互模式里输入中文会出现报错：
     </p>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto">&gt;&gt;&gt; '中文'
Unsupported characters in input
</pre>
     </div>
     <p>
      这个问题在 v2ex 上同样有同学问过：
      <a href="http://www.v2ex.com/t/44975" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank">
       /t/44975
      </a>
      ，而他是在 Windows 下出现的，所以这个问题可能是普遍的。我原本以为这个问题同样是因为上述的 stdin/stdout/stderr 的编码问题而造成，就想顺便解决掉。然而即使设置全局环境变量
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       PYTHONIOENCODING
      </code>
      为 utf-8 后仍旧不管用，IDLE 里输入中文还是会报错，
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       sys.stdin.encoding
      </code>
      编码还依旧是 us-ascii。
     </p>
     <p>
      后来搜索后发现，貌似这个问题是由 IDLE 输入输出的内部实现机制导致的，可能跟 stdin/stdout/stderr 没有关系。根据
      <a href="http://www.gossamer-threads.com/lists/python/python/206296" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Typing UTF-8 characters in IDLE">
       这里所说
      </a>
      ，IDLE 的交互模式下会根据机子的本地语言环境设置来判断编码，再用其对输入进行转换后再执行，而在我的 Mac 下这个编码是 ascii，所以导致了
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       Unsupported characters in input
      </code>
      。
     </p>
     <p>
      而我搜到了
      <a href="http://bytes.com/topic/python/answers/543035-typing-utf-8-characters-idle" rel="nofollow noopener noreferrer" style="color:rgb(119,128,135); text-decoration:none; word-break:break-all" target="_blank" title="Typing UTF-8 characters in IDLE">
       一个可行的解决方法
      </a>
      ，其通过在 IDLE 的 IO 相关源码（lib/python2.7/idlelib/IOBinding.py）中插入一行代码强行覆盖变量
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       encoding
      </code>
      的值为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       'utf-8'
      </code>
      来解决这个问题。
     </p>
     <p>
      不过后来经过我测试后发现，在 Mac 下其实更为简单的一个解决方法是，设置 IDLE 的环境变量
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       LANG
      </code>
      为
      <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
       "en_US.UTF-8"
      </code>
      。同样我不想通过 launchctl 设置全局环境变量来解决，
      <strong>
       而我采用的解决方法是：
      </strong>
     </p>
     <ol style="margin:1em 0px 0em 2em; padding:0px">
      <li style="padding:0px; margin:0px">
       打开编辑 IDLE.app/Contents/MacOS/IDLE 文件。
      </li>
      <li style="padding:0px; margin:0px">
       在大概第 24 行的地方插入一行设置环境变量
       <code style="background-color:rgb(248,248,248); padding:1px 2px; font-family:Consolas,'Panic Sans','DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace">
        LANG
       </code>
       的语句：
      </li>
     </ol>
     <div class="highlight" style="background-color:rgb(248,248,248); line-height:14px; margin-bottom:20px">
      <pre style="font-family:Consolas,'Panic Sans',Consolas,'DejaVu Sans Mono','Bitstream Vera Sans Mono',Menlo,'Microsoft Yahei',monospace; font-size:13px; letter-spacing:0.015em; line-height:15px; padding:0.5em; margin-top:0px; margin-bottom:0px; overflow:auto"><span class="o" style="color:rgb(102,102,102)">+</span>   <span class="n">os</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">environ</span><span class="p">[</span><span class="s" style="color:rgb(186,33,33)">"LANG"</span><span class="p">]</span> <span class="o" style="color:rgb(102,102,102)">=</span> <span class="s" style="color:rgb(186,33,33)">"en_US.UTF-8"</span>  <span class="c" style="clear:both; color:rgb(64,128,128); font-style:italic"># 第 24 行</span>
    <span class="n">os</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">environ</span><span class="p">[</span><span class="s" style="color:rgb(186,33,33)">"PYTHONEXECUTABLE"</span><span class="p">]</span> <span class="o" style="color:rgb(102,102,102)">=</span> <span class="n">executable</span>
    <span class="n">os</span><span class="o" style="color:rgb(102,102,102)">.</span><span class="n">environ</span><span class="p">[</span><span class="s" style="color:rgb(186,33,33)">"DYLD_LIBRARY_PATH"</span><span class="p">]</span> <span class="o" style="color:rgb(102,102,102)">=</span> <span class="n">libdir</span>
</pre>
     </div>
     <p style="margin-bottom:0px!important">
      保存文件，重新打开 IDLE 就可以在其交互模式里输入中文了。
