    <h2>
     前言
    </h2>
    <p>
     <br/>
     即时通讯应用中的实时音视频技术，几乎是IM开发中的最后一道高墙。原因在于：实时音视频技术 = 音视频处理技术 + 网络传输技术 的横向技术应用集合体，而公共互联网不是为了实时通信设计的。
    </p>

    <h3 style="margin-left:0px;">
     <span style="color:#2f2f2f;">
      视频为何需要压缩？
     </span>
    </h3>
    <ul style="margin-left:0px;">
     <li>
      未经压缩的数字视频的数据量巨大
     </li>
     <li>
      存储困难：
      <span style="color:#808080;">
       一张DVD只能存储几秒钟的未压缩数字视频。
      </span>
     </li>
     <li>
      传输困难
      <span style="color:#808080;">
       1兆的带宽传输一秒的数字电视视频需要大约4分钟。
      </span>
     </li>
    </ul>
    <p>
     <img alt="130847m1o6fg1rd90g6n9d.png (466Ã119)" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/6c00b7b9b34059aac6dc56d0552ca744.png"/>
    </p>
    <h3 style="margin-left:0px;">
     <span style="color:#2f2f2f;">
      主要压缩了什么东西？
     </span>
    </h3>
    <p>
     <br/>
     <strong>
      原始视频压缩的目的是去除冗余信息，可以去除的冗余包括：
     </strong>
    </p>
    <ul style="margin-left:0px;">
     <li>
     </li>
     <li>
      空间冗余：图像相邻像素之间有较强的相关性
     </li>
     <li>
      时间冗余：视频序列的相邻图像之间内容相似
     </li>
     <li>
      编码冗余：不同像素值出现的概率不同
     </li>
     <li>
      视觉冗余：人的视觉系统对某些细节不敏感
     </li>
     <li>
      知识冗余：规律性的结构可由先验知识和背景知识得到
     </li>
    </ul>
    <h3 style="margin-left:0px;">
     <span style="color:#2f2f2f;">
      数据压缩是怎么分类的？
     </span>
    </h3>
    <ul style="margin-left:0px;">
     <li>
     </li>
     <li>
      <strong>
       无损压缩（Lossless）：
      </strong>
      压缩前、解压缩后图像完全一致X=X'，压缩比低(2:1~3:1)。典型格式例如：Winzip，JPEG-LS。
     </li>
     <li>
      <strong>
       有损压缩（Lossy）：
      </strong>
      压缩前解压缩后图像不一致X≠X'，压缩比高(10:1~20:1)，利用人的视觉系统的特性。典型格式例如：MPEG-2，H.264/AVC，AVS。
     </li>
    </ul>
    <p>
     <img alt="131246bacgckk1cjtbsash.png (576Ã98)" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/efceea470593350af068714a4c3ec441.png"/>
    </p>
    <h3 style="margin-left:0px;">
     <span style="color:#2f2f2f;">
      什么是“编解码器”？
     </span>
    </h3>
    <p>
    </p>
    <ul style="margin-left:0px;">
     <li>
     </li>
     <li>
      编码器（Encoder）：
      <span style="color:#808080;">
       压缩信号的设备或程序；
      </span>
     </li>
     <li>
      解码器（Decoder）：
      <span style="color:#808080;">
       解压缩信号的设备或程序；
      </span>
     </li>
     <li>
      编解码器(Codec)：
      <span style="color:#808080;">
       编解码器对。
      </span>
     </li>
    </ul>

     <span style="color:#2f2f2f;">
      编解码的技术流程和原理

    </h3>
    <p>
    </p>

     编解码中的关键技术
    </h4>
    <p>
     <img alt="131639c5cxopo2d6ciqjj2.png (576Ã256)" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/b1c55af03f28aa5eef248dd1d3f2a95f.png"/>

    <h3>
     编解码器的实现
    </h3>
    <p>
     <br/>
     <strong>
      编解码器的实现平台：
     </strong>
    </p>
    <ul>
     <li>
     </li>
     <li>
      超大规模集成电路VLSI
     </li>
     <li>
      ASIC， FPGA
     </li>
     <li>
      数字信号处理器DSP
     </li>
     <li>
      软件
     </li>
    </ul>
    <p>
     <br/>
     <strong>
      编解码器产品：
     </strong>
    </p>
    <ul>
     <li>
     </li>
     <li>
      机顶盒
     </li>
     <li>
      数字电视
     </li>
     <li>
      摄像机
     </li>
     <li>
      监控器
     </li>
    </ul>
    <p>
    </p>
    <h3>
     视频编码的标准
    </h3>
    <p>
     <br/>
     <strong>
      编码标准作用：
     </strong>
     <br/>
    </p>
    <ul>
     <li>
     </li>
     <li>
      兼容：不同厂家生产的编码器压缩的码流能够被不同厂家的解码器解码
     </li>
     <li>
      高效：标准编解码器可以进行批量生产，节约成本。
     </li>
    </ul>
    <p>
     <br/>
     <strong>
      主流的视频编码标准：
     </strong>
     <br/>
    </p>
    <ul>
     <li>
     </li>
     <li>
      MPEG-2
     </li>
     <li>
      MPEG-4 Simple Profile
     </li>
     <li>
      H.264/AVC
     </li>
     <li>
      AVS
     </li>
     <li>
      VC-1
     </li>
    </ul>
    <p>
     <br/>
     <strong>
      标准化组织：
     </strong>
     <br/>
    </p>
    <ul>
     <li>
     </li>
     <li>
      ITU：International Telecommunications Union
     </li>
     <li>
      VECG：Video Coding Experts Group
     </li>
     <li>
      ISO：International Standards Organization
     </li>
     <li>
      MPEG：Motion Picture Experts Group
     </li>
    </ul>
    <h3>
     视频传输
    </h3>
    <p>
    </p>
    <ul>
     <li>
     </li>
     <li>
      视频传输：通过传输系统将压缩的视频码流从编码端传输到解码端
     </li>
     <li>
      传输系统：互联网，地面无线广播，卫星
     </li>
    </ul>
    <h3>
     视频传输面临的问题
    </h3>
    <p>
     <br/>
     <strong>
      传输系统不可靠：
     </strong>
    </p>
    <ul>
     <li>
     </li>
     <li>
      带宽限制
     </li>
     <li>
      信号衰减
     </li>
     <li>
      噪声干扰
     </li>
     <li>
      传输延迟
     </li>
    </ul>
    <p>
     <br/>
     <strong>
      视频传输出现的问题：
     </strong>
    </p>
    <ul>
     <li>
     </li>
     <li>
      不能解码出正确的视频
     </li>
     <li>
      视频播放延迟
     </li>
    </ul>
    <h3>
     视频传输差错控制
    </h3>
    <p>
     <br/>
     差错控制（Error Control）解决视频传输过程中由于数据丢失或延迟导致的问题。
     <br/>
     <br/>
     <strong>
      差错控制技术有：
     </strong>
    </p>
    <ul>
     <li>
     </li>
     <li>
      信道编码差错控制技术
     </li>
     <li>
      编码器差错恢复
     </li>
     <li>
      解码器差错隐藏
     </li>
    </ul>
    <h3>
     视频传输中QoS质量保证参数
    </h3>
    <ul>
     <li>
     </li>
     <li>
      数据包的端到端的延迟
     </li>
     <li>
      带宽：比特/秒
     </li>
     <li>
      数据包的流失率
     </li>
     <li>
      数据包的延迟时间的波动
     </li>
    </ul>
