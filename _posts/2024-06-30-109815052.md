    <h2>
     <a id="_0">
     </a>
     一、需求描述
    </h2>
    <p>
     客户公司的一台HP DL360p Gen8服务器硬盘坏了，为了防止另外一块硬盘也损坏，急需去将坏的硬盘进行更换。服务器更换硬盘不同普通电脑更换硬盘，需要人工去导数据，服务器更换硬盘不需要，为什么不需要呢？和即到哥一起往下看吧。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2c1045c75535c38cf58b6d6154e9d6b1.png#pic_center">
      <br/>
      可以看到下面一块硬盘，一直亮黄灯。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/dfe5a33fde5434b0e1d13d7f269b6d10.png#pic_center"/>
     </img>
    </p>
    <h2>
     <a id="_5">
     </a>
    </h2>
    <p>
     二、RAID1重构数据
     <br/>
     企业在安装服务器时，双硬盘都会做RIAD，很多公司都会将前两块硬盘做RAID1安装系统，其余的盘按自己的需求，去做RAID0，RAID1还是RAID5，RAID10等放数据。
    </p>
    <p>
     RAID1工作原理：即每个工作盘都有一个镜像盘，每次写数据时必须同时写入镜像盘，读数据时只从工作盘读出，一旦工作盘发生故障立即转入镜像盘，从镜像盘中读出数据。
     <strong>
      当更换故障盘后，数据可以重构，恢复工作盘正确数据。
     </strong>
    </p>
    <p>
     RAID1优点
     <br/>
     1、数据安全性相对其它RAID级，是最好的。
    </p>
    <p>
     RAID1缺点
     <br/>
     1、磁盘利用率只有50%，是所有RAID上磁盘利用率最低的一个级别。
    </p>
    <p>
     RAID 1镜像/双工(Mirroring/ Duplexing)存储数据工作模式:
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6e3ad1cc802ae2e7a6d37428d93f3edd.png#pic_center">
      <br/>
      RAID1有一个非常牛的功能，就是当两个盘中其中一块盘坏了，如果你换一个新的硬盘上去后，数据可以重构。
     </img>
    </p>
    <p>
     什么叫重构呢？就是说你换了一块新的硬盘上去后，好的硬盘中系统包括所有数据都会复制到这块新的硬盘上。当某一天，旧的硬盘再坏，还可以继续这么做，只要两块硬盘不同时坏，数据系统就不会坏，数据的安全性还是很高的。
    </p>
    <h2>
     <a id="_23">
     </a>
     三、实施过程
    </h2>
    <p>
     1、购买硬盘时，最好是和原来的服务器硬盘一样，否则有可能会同现同步不成功的现象。
     <br/>
     可以看到硬盘编号是653955，300G硬盘，接口是SAS，速度是10K。可以按这参数去购买。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f8d9f6bc158c2d7052c2104fae70a423.png#pic_center">
      <br/>
      2、硬盘到货，开始更换。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/58f3c7c5b9b58438472ff8ddca985934.png#pic_center">
       <br/>
       3、由于此服务器可以停机，所以这次我们选择停机，更换硬盘。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/52b4fa4e3bb4327f4ce057738ff89e61.png#pic_center">
        <br/>
        4、服务器已关机。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e6773debe2136d6c52a43c4580c86118.png#pic_center">
         <br/>
         5、按红色硬盘架弹出按钮，然后把硬盘拿出来。
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/557a7285feda821e2f53052539990fd7.png#pic_center">
          <br/>
          6、更换新的硬盘。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2b3320a3f3b1f77788d709f803b2d766.png#pic_center"/>
          <br/>
          7、新硬盘更换好后，开机。
          <br/>
          开机后，提示说HP阵列需要自动恢复数据（重建），选择F1继续恢复数据到硬盘，选择F2不恢复数据到硬盘，默认是选择F1，进行恢复。按F1进行数据恢复重建。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/16979d476f81fa5f25a4a4cd16b7178a.png#pic_center"/>
          <br/>
          8、配置完需要硬盘重建恢复后，如果需要进入RAID卡界面，在此界面按F5进行查看，如果不查看可以直接进入系统。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ffc72ffec159db467bd89d3f0a55b796.png#pic_center"/>
          <br/>
          9、在硬盘同步数据时，不影响服务器的使用的。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9c0e561bd741daf0aaa0ee3b389bbcbc.png#pic_center"/>
          <br/>
          9、可以看到两块硬盘绿灯一直在转，而且上面一块硬盘亮红灯，这个红灯的状态是禁止拨出。如果你在同步时，直接将上面一块硬盘拨出，那同步直接失败，所以显示禁止拨出红色状态。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0851380a4292aca067aacc294fe796d3.png#pic_center"/>
          <br/>
          10、硬盘同步状态，等到红灯灭，而且硬盘灯正常闪烁，说明数据同步完成。
         </img>
        </img>
