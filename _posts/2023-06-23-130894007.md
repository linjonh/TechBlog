    <h2>
     前端环境配置
    </h2>
    <h3>
     1. 准备powershell
    </h3>
    <p>
     因为执行的大多数都是些linux命名，在window环境还是安装一个powershell来执行命令
     <br/>
     powershell下载地址为：
     <br/>
     <a class="link-info" href="https://github.com/PowerShell/PowerShell%C2%A0" title="https://github.com/PowerShell/PowerShell ">
      https://github.com/PowerShell/PowerShell
     </a>
     <br/>
     根据当前系统选择版本，笔者是windows 10 64位，因此选择Windows (x64)
     <br/>
     安装完成后
     <br/>
     <img alt="" height="354" src="https://i-blog.csdnimg.cn/blog_migrate/effd2506965e4376b5d0ea58ef674304.png" width="1200"/>
    </p>
    <h3>
     2. 安装nodejs
    </h3>
    <p>
     <img alt="" height="506" src="https://i-blog.csdnimg.cn/blog_migrate/21649ddb906ad41bd72a94402121544f.png" width="1200">
      <br/>
      当出现时，需要安装node.js可以通过https://nodejs.org进行下载安装
     </img>
    </p>
    <p>
     随后执行
    </p>
    <pre><code class="language-javascript">npm install</code></pre>
    <p>
     如果安装时间慢，是因为使用了国外国外进行，这里改成国内镜像
    </p>
    <pre><code class="language-javascript">npm config set registry http://registry.npm.taobao.org</code></pre>
    <h3>
     3. 安装is-electron
    </h3>
    <p>
     当出现提示时
    </p>
    <p>
     <img alt="" height="777" src="https://i-blog.csdnimg.cn/blog_migrate/2e086d827c694f1270b5ecec4ff76637.png" width="1200"/>
    </p>
    <p>
     安装is-electron
    </p>
    <pre><code class="language-javascript">npm install --save is-electron</code></pre>
    <h3>
     4. 最终得到这个界面证明前端内容部分已经启动成功
    </h3>
    <p>
     执行
    </p>
    <pre><code class="language-javascript">npm run dev</code></pre>
    <p>
     获得下图即代表正常启动
    </p>
    <p>
     <img alt="" height="247" src="https://i-blog.csdnimg.cn/blog_migrate/f575d3a5b0ab1f1527d2c1b10d0dbd04.png" width="1200"/>
    </p>
    <p>
     访问网址会进入到界面，只会显示默认的青眼白龙，卡面其他内容因为后端没启动不会加载，
    </p>
    <p>
     <img alt="" height="1200" src="https://i-blog.csdnimg.cn/blog_migrate/32964f1246a949a2b09cda24186c41c4.png" width="1200"/>
    </p>
    <p>
    </p>
    <p>
     目前github项目的前端已经在确保能正常启动后，把调整后的代码上传到项目中，按照上面操作在win10环境下应该得到截图内容。接下来就是准备后端环境
    </p>
    <h2>
     后端环境配置
    </h2>
    <p>
     这次以最新2023.4月份数字包1201为例，本地搭建自己的游戏王卡片生成器就几乎能同步最新的卡。
    </p>
    <p>
     在上一篇论文中我提供了源码，让读者自行编译执行，在后台和评论来看，我直接提供执行包，安装好运行环境能正常启动即可。
    </p>
    <h3>
     1. 安装JAVA运行环境
    </h3>
    <p>
     因为我后台使用的是java，因此执行环境需要安装jdk，选择能执行jar包的环境即可，在
     <a class="link-info" href="https://repo.huaweicloud.com/java/jdk/8u151-b12/" rel="nofollow" title="JDK">
      JDK
     </a>
     找到能安装jdk安装包安装 ，得到下面截图内容即可
     <img alt="" height="1050" src="https://i-blog.csdnimg.cn/blog_migrate/4c204d6dd9fa36e0e9df0c359b9947cf.png" width="1200"/>
    </p>
    <h3>
     2. 启动后端程序
    </h3>
    <p>
     读者可以自行通过源码编译jar包，也可以下载我的分割好的压缩包（因为完整jar包超过100m）
    </p>
    <pre><code class="language-bash">java -jar YGO-CARD-1.0-SNAPSHOT.jar -image.img=c:\monster\ -spring.datasource.url=jdbc:mysql://127.0.0.1:3306/mytest?characterEncoding=utf-8 -spring.datasource.username=root -spring.datasource.password=123456</code></pre>

    <h3>
     3. 初始化后端数据
    </h3>
    <p>
     后端项目启动后需要对图片和卡片数据进行初始化，程序才能正确获取到对应
    </p>
    <h4>
     3.1 卡片插画
    </h4>
    <p>
     通过游戏王数据库我们可以查询到【革命同调士】的码为97682931
    </p>
    <p>
     <img alt="" height="1200" src="https://i-blog.csdnimg.cn/blog_migrate/a31a949ad078917186ebe7857acf26d5.png" width="1200"/>
    </p>
    <p>
     ygoprodeck提供了api可以让我们查询到图片，例如【革命同调士】的码为97682931，我们访问
     <a href="https://images.ygoprodeck.com/images/cards_cropped/97682931.jpg" rel="nofollow" title="97682931.jpg (624×624) (ygoprodeck.com)">
      97682931.jpg (624×624) (ygoprodeck.com)
     </a>
     就可以得到程序需要的插图
    </p>
    <p>
     <img alt="" height="1200" src="https://i-blog.csdnimg.cn/blog_migrate/8cd2533c2d442711de4f642d8579b824.png" width="1200"/>
    </p>
    <h4>
     3.2 卡片数据
    </h4>
    <p>
     每个版本的ygo，过后对应的卡片数据更新斗发布在开源项目中，读者可以通过查看提交日志来取人卡片信息是否已经更：
    </p>
    <p>
     <a href="https://github.com/mycard/ygopro-database/commits/dump" title="https://github.com/mycard/ygopro-database/commits/dump">
      https://github.com/mycard/ygopro-database/commits/dump
     </a>
    </p>
    <p>
     如下图我们可以发现在ygo发布4天后，卡片数据也已经更新了
    </p>
    <p>
     <img alt="" height="460" src="https://i-blog.csdnimg.cn/blog_migrate/f02e8778f600372b62539ad32e7b2273.png" width="1200"/>
    </p>
    <p>
     关于数据脚本导入我这里就不再多讲，我这里主要解决在私信说使用脚本导入不进去的情况，我这里提供对应数据表新建脚本。
    </p>
    <pre><code class="language-sql">DROP TABLE IF EXISTS `datas`;

  `id1` blob,
  `id2` blob,

  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=latin1;


DROP TABLE IF EXISTS `texts`;
CREATE TABLE `texts` (

  `dec4` text CHARACTER SET utf8,
  `dec5` text CHARACTER SET utf8,
  `dec6` text CHARACTER SET utf8,

  `dec11` text CHARACTER SET utf8,
