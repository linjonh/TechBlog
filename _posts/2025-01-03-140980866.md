    <h2>
     <a id="1_16">
     </a>
     1、小程序简介
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/297883bc01c5410dbcfa415b026508c9.png"/>
    </p>
    <p>
     小程序是一种不需要下载安装即可使用的应用，它实现了应用的即搜即用的概念。小程序具备以下特点：
    </p>
    <ol>
     <li>
      <strong>
       无需下载安装
      </strong>
      ：用户通过微信、支付宝、百度等平台搜索或扫描小程序码即可打开使用，无需在手机上安装额外的应用。
     </li>
     <li>
      <strong>
       触手可及
      </strong>
      ：小程序可以快速地触达用户，满足用户的即时需求。
     </li>
     <li>
      <strong>
       用完即走
      </strong>
      ：用户在使用完小程序后，无需担心占用手机存储空间，可以随时关闭。
     </li>
     <li>
      <strong>
       功能丰富
      </strong>
      ：小程序可以提供消息通知、线下扫码、分享转发等多种功能，满足不同场景下的需求。
     </li>
     <li>
      <strong>
       开发成本较低
      </strong>
      ：相较于传统APP，小程序的开发周期更短，成本更低。
     </li>
     <li>
      <strong>
       跨平台兼容
      </strong>
      ：小程序可以在多个平台上运行，如微信、支付宝、百度等，覆盖了广泛的用户群体。
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3beceec98eb342099bb5586b44ec6ffd.png"/>
    </p>

         <br/>

           <br/>
           <strong>
            温馨提示
           </strong>
           ：一个身份证最多只能注册5个小程序账号，所以要慎重使用自己的身份证。当注册满了以后会提示:该身份证记登记过5次，请使用另一个身份证完成用户信息登记。
          </img>
         </img>
        </img>
       </img>
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ed04b8f02dd7403f8a980fbf6c2bc33d.png"/>
     <br/>
     <strong>
      个人小程序不支持微信认证，微信支付以及高级接口能力
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/475ae38a769444308102b9358c30b623.png"/>
     <br/>
     小程序的AppID（Application ID）是小程序在特定平台上的唯一标识符。每个小程序在注册时都会被分配一个AppID，它是小程序开发者进行开发、调试和发布的关键凭证之一。以下是关于小程序AppID的一些详细信息：
    </p>
    <ol>
     <li>
      <strong>
       获取AppID
      </strong>
      ：
      <ul>
       <li>
        在微信小程序中，开发者需要在微信公众平台注册小程序，完成相关认证后，系统会分配一个AppID。
       </li>
       <li>
        在支付宝小程序中，开发者需要在支付宝开放平台注册并创建小程序，同样会获得一个AppID。
       </li>
       <li>
        在百度智能小程序中，开发者需要在百度智能小程序平台注册，并通过审核后获得AppID。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用AppID
      </strong>
      ：
      <ul>
       <li>
        在开发过程中，AppID用于小程序的配置文件（如微信小程序的
        <code>
         app.json
        </code>
        ）中，以便于平台识别和管理小程序。
       </li>
       <li>
        在调试阶段，开发者需要使用AppID来启用开发工具的相关功能，如模拟支付、授权登录等。
       </li>
       <li>
        在发布小程序时，AppID是必不可少的，它用于提交审核和上线。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       AppID的作用
      </strong>
      ：
      <ul>
       <li>
        识别唯一性：确保小程序在平台上的唯一性。
       </li>
       <li>
        权限验证：用于获取平台提供的各种服务权限，如用户信息、支付功能等。
       </li>
       <li>
        数据统计：平台会根据AppID收集小程序的运营数据，如访问量、用户行为等。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       保密性
      </strong>
      ：
      <ul>
       <li>
        AppID是敏感信息，开发者应当妥善保管，避免泄露给第三方，以免造成不必要的风险。
        <br/>
        如果你是小程序开发者，你可以按照以下步骤找到你的AppID：
       </li>
      </ul>
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       微信小程序
      </strong>
      ：登录微信公众平台 -&gt; 小程序 -&gt; 开发管理 -&gt; 基本设置，在这里可以找到你的AppID。
     </li>
     <li>
      <strong>
       支付宝小程序
      </strong>
      ：登录支付宝开放平台 -&gt; 小程序中心 -&gt; 管理中心 -&gt; 基本信息页面，可以查看到AppID。
     </li>
     <li>
      <strong>
       百度智能小程序
      </strong>
      ：登录百度智能小程序平台 -&gt; 控制台 -&gt; 小程序详情，可以找到AppID。
     </li>
    </ul>
    <p>
     <strong>
      小程序的appID很重要，以后开发要用到。
     </strong>
    </p>

    <h3>
     <a id="52WXML_92">
     </a>
     5.2WXML模板
    </h3>
    <p>
     <strong>
      1. 什么是WXML
     </strong>
    </p>
    <p>
     WXML（WeiXin Markup Language）是小程序框架设计的一套
     <strong>
      标签语言，用来构建小程序页面的结构
     </strong>
     ，其作用类似于网页开发中的 HTML。
     <br/>
     <strong>
      2. WXML 和 HTML 的区别
     </strong>
    </p>
    <p>
     ① 标签名称不同
     <br/>
     ⚫ HTML （div, span, img, a）
     <br/>
     ⚫ WXML（view, text, image, navigator）
     <br/>
     ② 属性节点不同
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/09d8f8d56edb4787b0d07b5068f6e6ca.png"/>
    </p>
    <p>
     ③ 提供了类似于 Vue 中的模板语法
     <br/>
     ⚫ 数据绑定
     <br/>
     ⚫ 列表渲染
     <br/>
     ⚫ 条件渲染
    </p>
    <h3>
     <a id="53WXSS_109">
     </a>
     5.3WXSS样式
    </h3>
    <p>
     WXSS (WeiXin Style Sheets)是一套样式语言，用于描述 WXML 的组件样式，类似于网页开发中的 CSS。
    </p>
    <p>
     <strong>
      WXSS 和 CSS 的区别
     </strong>
    </p>
    <p>
     ① 新增了 rpx 尺寸单位
     <br/>
     ⚫ CSS 中需要手动进行像素单位换算，例如 rem
     <br/>
     ⚫ WXSS 在底层支持新的尺寸单位 rpx，在不同大小的屏幕上小程序会自动进行换算
    </p>
    <p>
     ② 提供了全局的样式和局部样式
     <br/>
     ⚫ 项目根目录中的 app.wxss 会作用于所有小程序页面
     <br/>
     ⚫ 局部页面的 .wxss 样式仅对当前页面生效
    </p>
    <p>
     ③ WXSS 仅支持部分 CSS 选择器
     <br/>
     ⚫ .class 和 #id
     <br/>
     ⚫ element
     <br/>
     ⚫ 并集选择器、后代选择器
     <br/>
     ⚫ ::after 和 ::before 等伪类选择器
    </p>
    <h3>
     <a id="54JS__128">
     </a>
     5.4JS 逻辑交互
    </h3>
    <p>
     一个项目仅仅提供界面展示是不够的，在小程序中，我们通过 .js 文件来处理用户的操作。例如：响应用户的点击、获取用户的位置等等。
    </p>
    <p>
     小程序中的 JS 文件分为三大类，分别是：
     <br/>
     ① app.js
     <br/>
     ⚫ 是整个小程序项目的入口文件，通过调用 App() 函数来启动整个小程序
     <br/>
     ② 页面的 .js 文件
     <br/>
     ⚫ 是页面的入口文件，通过调用 Page() 函数来创建并运行页面
     <br/>
     ③ 普通的 .js 文件
     <br/>
     ⚫ 是普通的功能模块文件，用来封装公共的函数或属性供页面使用
    </p>

     <a id="7_147">

     7、组件
    </h2>
    <p>
     <strong>
      1. 小程序中组件的分类
     </strong>
    </p>
    <p>
     小程序中的组件也是由宿主环境提供的，开发者可以基于组件快速搭建出漂亮的页面结构。官方把小程序的组件分为了 9 大类，分别是：
    </p>
    <p>
     ① 视图容器
     <br/>
     ② 基础内容
     <br/>
     ③ 表单组件
     <br/>
     ④ 导航组件
     <br/>
     ⑤ 媒体组件
     <br/>
     ⑥ map 地图组件
     <br/>
     ⑦ canvas 画布组件
     <br/>
     ⑧ 开放能力
     <br/>
     ⑨ 无障碍访问
    </p>
    <p>
     <strong>
      2. 常用的视图容器类组件
     </strong>
    </p>
    <p>
     ① view
     <br/>
     ⚫ 普通视图区域
     <br/>
     ⚫ 类似于 HTML 中的 div，是一个块级元素
     <br/>
     ⚫ 常用来实现页面的布局效果
     <br/>
     ② scroll-view
     <br/>
     ⚫ 可滚动的视图区域
     <br/>
     ⚫ 常用来实现滚动列表效果
     <br/>
     ③ swiper 和 swiper-item
     <br/>
     ⚫ 轮播图容器组件 和 轮播图 item 组件
    </p>

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/77bd7dd40d784b89a53a044da54f38d8.png"/>

     </strong>
    </p>
    <p>
     <strong>
      .container1
     </strong>
     <br/>
     这里的.表示选择器是一个类选择器。container1是类名，这意味着这个样式规则将应用于所有具有container1类的HTML元素。
    </p>
    <p>
     <strong>
      display: flex;：
     </strong>
     <br/>
     当一个元素设置为display: flex;后，它的直接子元素将按照弹性盒模型（Flexbox）的布局方式排列。Flexbox是一种用于在容器内分配和对齐子元素的一维布局方法。
    </p>
    <p>
     <strong>
      justify-content: space-around
     </strong>
     <br/>
     justify-content属性用于设置弹性容器内弹性项目在主轴上的对齐方式。
     <br/>
