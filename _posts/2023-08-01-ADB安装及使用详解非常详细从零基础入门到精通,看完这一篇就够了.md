    <h2>

     一、ADB简介
    </h2>
    <h3>
     <a id="1adb_4">
     </a>
     1、什么是adb
    </h3>
    <p>
     ADB 全称为 Android Debug Bridge，起到调试桥的作用，是一个
     <strong>
      客户端-服务器端程序
     </strong>
     。其中客户端是用来操作的电脑，服务端是 Android 设备。
    </p>
    <p>
     ADB 也是 Android SDK 中的一个工具，可以直接操作管理 Android 模拟器或者真实的 Android 设备。
    </p>
    <h3>
     <a id="2adb_11">
     </a>
     2、为什么要用adb
    </h3>
    <p>
     运行设备的 shell(命令行)
     <br/>
     管理模拟器或设备的端口映射
     <br/>
     计算机和设备之间上传/下载文件
     <br/>
     可以对设备的应用进行卸载安装等
     <br/>
     在 App 遇到 ANR/Crash 等 bug 时，可以通过 ADB 来抓取日志
    </p>
    <p>
     简而言之，ADB 就是连接 Android 手机与 PC 端的桥梁，所以
     <strong>
      ADB又称为安卓调试桥
     </strong>
     （注意：是安卓，不是iOS），可以让用户在电脑上对手机进行全面的操作！
    </p>

     </p>
     <p>
      **adb pull &lt;手机端路径/文件或文件夹&gt; &lt;pc机路径&gt;：把设备(手机)的文件或文件夹复制到本地。**注意点同上

    <h3>
     <a id="6_178">
     </a>
     6、日志操作指令
    </h3>
    <pre><code>adb shell logcat -c：清理现有日志
adb shell logcat -v time ：输出日志，信息输出在控制台
adb shell logcat -v time &gt; &lt;存放路径\log.txt&gt;：输出日志并保存在本地文件
Ctrl+C：终止日志抓取
adb shell logcat -v time *:E &gt; &lt;存放路径\log.txt&gt;：打印级别为Error的信息
日志的等级：
-v：Verbse（明细）
-d：Debug（调试）
-i：Info（信息）
-w：Warn（警告）
-e：Error（错误）
-f：Fatal（严重错误）
抓取日志的步骤先输入命令启动日志，然后操作 App，复现 bug，再 ctrl+c 停止日志，分析本地保存的文件。
：日志是记录手机系统在运行app时有什么异常的事件
EXCEPTION
也可以把更详细得Anr日志拉取出来：adb shell pull /data/anr/traces.txt &lt;存放路径&gt;
</code></pre>

    <p>

    <p>
     <strong>
      2022届大学毕业生月收入较高的前10个专业
     </strong>
    </p>
    <p>

    </p>
    <p>
     具体看专业，2022届本科月收入较高的专业是信息安全（7579元）。对比2018届，电子科学与技术、自动化等与人工智能相关的本科专业表现不俗，较五年前起薪涨幅均达到了19%。数据科学与大数据技术虽是近年新增专业但表现亮眼，已跻身2022届本科毕业生毕业半年后月收入较高专业前三。五年前唯一进入本科高薪榜前10的人文社科类专业——法语已退出前10之列。
