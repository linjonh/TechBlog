    <h2>
     1.测试环境的重要性
    </h2>
    <p>
     <br/>
    </p>
    <p>
     <span style="color:#ff0000">
      1.1   测试环境与辅测试环境
     </span>
    </p>
    <p>
     <br/>
    </p>
    <p>
     软件环境分为主测试环境和辅测试环境。
     <br/>
     <br/>
     主测试环境是测试软件功能、安全可靠性、性能、易用性等大多数指标的主要环境
     <br/>
     <br/>
     辅助测试环境满足特殊的测试需求
     <br/>
    </p>
    <ul>
     <li>
      兼容性测试
     </li>
     <li>
      模拟真实环境测试
     </li>
     <li>
      横向对比测试
     </li>
    </ul>
    <div>
     <br/>
    </div>
    <div>
     <span style="color:#ff0000">
      1.2  测试环境的五要素
     </span>
    </div>
    <div>
     <br/>
    </div>
    <div>
     测试环境的基本要素是：软件、硬件。在基本要素的基础上派生出网络环境、数据准备、测试工具三要素
     <br/>
    </div>
    <div>
     <br/>
    </div>
    <div>
     <ul>
      <li>
       硬件标准
      </li>
      <li>
       软件配置
      </li>
      <li>
       网络环境
      </li>
      <li>
       数据准备
      </li>
     </ul>
     <div>
      <br/>
     </div>
     <h2>
      2.建立测试实验室
     </h2>
     <div>
      <br/>
     </div>
     <div>
      <span style="color:#ff0000">
       2.1  选择和规划实验室
      </span>
     </div>
     <div>
      <br/>
     </div>
     <div>
      一旦确定有建立测试实验室的必要，就需要为实验室选择场所并规划它的配置。应当考虑各种因素，例如空间尺寸、照明、布局、功能区、温度、湿度、放火和安全、电源、静电、设施等等，尽可能描绘出实验室的量化层平面图进行规划、不断地完善调整规划。
      <br/>
     </div>
     <div>
      <br/>
     </div>
     <div>
      <span style="color:#ff0000">
       <br/>
      </span>
     </div>
     <div>
      <span style="color:#ff0000">
       2.2  集成和配置测试设备
      </span>
     </div>
     <div>
      <br/>
     </div>
     <div>
      要理出一个设备清单，然后进行预算评估，接着采购设备，最后是集成安装。
      <br/>
     </div>
     <div>
      <br/>
     </div>
     <div>
      例如，对软件：
      <br/>
      <ul>
       <li>
        操作系统：Windows, Mac X, Linux, …
       </li>
       <li>
        应用程序：使用大量的应用软件来做兼容性测试。
       </li>
       <li>
        测试工具和实用工具：用于测试软件和诊断问题
       </li>
       <li>
        第三方软件：需要集成的第三方软件
       </li>
      </ul>

       <br/>
       3.测试环境的维护和管理
