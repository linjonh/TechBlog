     开源软件也是要恰饭的
    </h1>

  <div class="article_content clearfix" id="article_content">

   <div class="markdown_views prism-atom-one-light" id="content_views">

     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/a79f3480f7b217b69485cffb1e581198.png"/>
    </p>
    <p>
     对于知名容器厂商
     <strong>
      Docker
     </strong>
     公司来说昨天是一个值得铭记的日子。先是
     <strong>
      Mirantis
     </strong>
     宣布收购
     <strong>
      Docker
     </strong>
     企业级服务部门获得融资3500万美元。值得注意的是，这次收购并不包括
     <strong>
      Docker Desktop
     </strong>
     。据称，
     <strong>
      Mirantis
     </strong>
     和
     <strong>
      Docker
     </strong>
     将共同开发核心的上游技术，为开源开发做出贡献。此外，
     <strong>
      Mirantis
     </strong>
     和
     <strong>
      Docker
     </strong>
     将继续确保双方产品之间的集成，
     <strong>
      Docker
     </strong>
     将专注于
     <strong>
      Docker Desktop
     </strong>
     和
     <strong>
      Docker Hub
     </strong>
     ，而
     <strong>
      Mirantis
     </strong>
     专注于
     <strong>
      Docker Enterprise
     </strong>
     容器平台。几乎同时
     <strong>
      Docker
     </strong>
     还宣布公司已任命长期担任首席产品官的
     <strong>
      Scott Johnston
     </strong>
     为首席执行官（CEO）。
     <br/>
     <strong>
      Johnston
     </strong>
     是今年
     <strong>
      Docker
     </strong>
     的第三任
     <strong>
      CEO
     </strong>
     ，接替
     <strong>
      Rob Bearden
     </strong>
     ，而
     <strong>
      Bearden
     </strong>
     接替5月份卸任的
     <strong>
      Steve Singh
     </strong>
     。
    </p>
    <p>
     在发布这则消息前，Docker公司已经融资2.7亿美元，是行业内的独角兽。 如今急匆匆贱卖其企业级业务令人吃惊。其实也不算贱卖。主要是 Docker 是开源的，从一开始面向的就是开发者，你要知道想从开发者手中赚钱是非常难的；而在赚钱的企业生产实践中缺乏行之有效的解决方案。互联网巨头
     <strong>
      Google
     </strong>
     开源（CNCF基金会）
     <strong>
      Kubernetes
     </strong>
     （k8s） 对其更是致命的打击。Docker 方面曾经多次通过文章、和社区活动对 k8s 表示不满。随着
     <strong>
      Kubernetes
     </strong>
     成为容器编排的事实标准,Docker也不得不低下了头。这种 “前人栽树，后人乘凉” 的结果一方面是作为“小公司”（相比较大公司）天然上的管理通病以及格局的局限性，当你不能领导这个行业的时候就会有人代替你。
     <br/>
     另一方面也是互联网强权打压的结果，毕竟互联网的战场谁的拳头大谁就能从中瓜分更多的地盘。开源云厂商也从中赚的盆满钵满。但是他们花费了更少的开发成本，甚至仅仅是“搭一搭积木”就可以了。
    </p>
    <p>
     教会了徒弟，饿死了师父。这并不是开源软件的一个良性结果。开发都是有成本的，如果没有收入来源和投入就会步履维艰。通常开源软件的收入主要来自捐赠、提供更加可靠的解决方案获取的佣金这两方面。但是往往入不敷出。随着云计算的兴起，开源软件想赚钱就更加困难了。开源软件很容易就被云厂商绕过去而分不到一杯羹。所以就发生了一些开源软件和云厂商之间的对抗事件。
     <br/>
     2018年8月，流行的缓存解决方案
     <strong>
      Redis
     </strong>
     的持有者
     <strong>
      Redis Labs
     </strong>
     将其许可证变为
     <code>
      Commons Clause license
     </code>
     ，修改后的许可条款规定，其他企业不允许将这些扩展服务作为云服务的一部分提供。两个月之后另一个知名的数据库
     <strong>
      MongoDB
     </strong>
     也宣布未来将在一个名为
     <strong>
      SSPL
     </strong>
     的不同许可机制下授权
     <strong>
      MongoDB
     </strong>
     社区服务器软件，该许可仍然允许云服务商提供
     <strong>
      MongoDB
     </strong>
     作为服务，但要求他们要么将其编写的全部代码开源，要么与
     <strong>
      MongoDB
     </strong>
     达成商业合作协议。
    </p>
    <p>
     <strong>
      MongoDB CEO Dev Ittycheria
     </strong>
     说到：
    </p>
    <blockquote>
     <p>
      “每当一个新的开源项目变得流行时，云服务商就会把我们的技术拿到他们的平台上，获取大部分（甚至全部）利润，但这些平台几乎毫不回馈开源社区。我们认为，由我们这种形式的开源平台领导和帮助下一批开源项目的发展和成长是非常重要的。”
     </p>
    </blockquote>
    <blockquote>
     <p>
      “每个人都想要更多的开源，但开源也需要成本，总有人要花钱维护。为了能有钱维护，就需要确保开源平台在商业上可以生存下去。”
     </p>
    </blockquote>
    <p>
     很多知名的软件都诞生于开源软件甚至诞生于一个天才的个人想法。如何让开源软件能够良性的发展下去？ 这是一个问题。不是每一家企业都能成为
     <strong>
      Redhat
     </strong>
     。如果有一天，
     <strong>
      开源没有搞头
     </strong>
     成为常识将是很可怕的事情。
    </p>
