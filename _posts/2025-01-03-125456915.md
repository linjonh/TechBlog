     <p>
      3.测试方法以及工具
      <br/>
      测试方法：黑盒测试、自动化测试；
      <br/>
      测试工具：postman、Selenium IDE；
     </p>
     <p>
      4.测试范围：
     </p>
     <ul>
      <li>
       功能测试：围绕i系统的功能，检测每个功能功能模块的功能是否能实现逻辑流程是否合理，UI界面的设计是否合理等；
      </li>
      <li>
       兼容性测试：检查常用的浏览器兼容性；
      </li>
      <li>
       回归测试：要求系统的测试点覆盖率达到100%，待bug验证通过过，进行多次回归测试，测试点覆盖全面。
      </li>
     </ul>
     <p>
      5.测试阻碍：（有就写没有可以不写）
     </p>
     <ul>
      <li>
       测试环境不稳定；
      </li>
      <li>
       测试环境垃圾数据较多，导致功能缺陷；
      </li>
      <li>
       公共页面修改功能功能发生变化导致报错；
      </li>
     </ul>

     <h2>
      <a id="_97">
      </a>
      四、测试风险
     </h2>
     <p>
      <strong>
       1、需求风险：
      </strong>
      需求变更导致开发，测试部分工作失败，维护成本增加；
      <br/>
      <strong>
       2、缺陷风险：
      </strong>
      偶现重现，容易被遗漏；
      <br/>
      <strong>
       3、代码质量风险：
      </strong>
      系统框架设计不足，导致扩展性不足，兼容性差等问题
      <br/>
      <strong>
       4、测试环境风险：
      </strong>
      测试环境同线上环境配置区别较大。
     </p>
