    <h2>
     <a id="1__1">
     </a>
     1. 如何判断一个开源项目能否商用
    </h2>
    <p>
     要判断一个开源项目能否商用其实很简单，只需要查看开源项目使用的开源协议就可以了
    </p>
    <hr/>
    <p>
     以下是主流的开源协议：
    </p>
    <ul>
     <li>
      Apache-2.0 License
     </li>
     <li>
      GPL-3.0 License
     </li>
     <li>
      LGPL-3.0 License
     </li>
     <li>
      MIT License
     </li>
    </ul>
    <h2>
     <a id="2_Apache20_License_14">
     </a>
     2. Apache-2.0 License
    </h2>
    <blockquote>
     <p>
      <strong>
       用大白话解释 Apache-2.0 License 开源协议：随便用，还不用担心被告
      </strong>
     </p>
    </blockquote>
    <p>
     Apache-2.0 License 就像是一个很宽松的规则手册，它允许你做很多事情，但是也有一些基本的条件需要遵守：
    </p>
    <ol>
     <li>
      <strong>
       随便用
      </strong>
      ：你可以把这个软件拿去用在你的产品或者服务里，不管是为了赚钱（商用）还是个人使用
     </li>
     <li>
      <strong>
       随便改
      </strong>
      ：你可以对这个软件进行修改，按你的需求来调整代码
     </li>
     <li>
      <strong>
       随便分
      </strong>
      ：你可以把软件或者你修改后的版本分发给其他人
     </li>
     <li>
      <strong>
       别怪我
      </strong>
      ：这个软件是人家免费提供的，如果用的时候出了问题，人家不负责
     </li>
     <li>
      <strong>
       专利也行
      </strong>
      ：如果你在这个软件里用了别人的专利技术，人家也同意你用，不会告你侵权
     </li>
     <li>
      <strong>
       名字别乱用
      </strong>
      ：虽然你可以用这个软件，但是不能假装这个软件是你做的，也不能用原作者的名字或者商标去做宣传
     </li>
    </ol>
    <p>
     简单来说，Apache-2.0 License 就是一个很友好的协议，让你可以比较自由地使用和修改软件，只要你不搞乱人家的名声，出了问题自己负责，并且把同样的自由权利传递给下一个使用软件的人
    </p>
    <hr/>
    <p>
     很多企业级项目和商业级项目首选的都是使用了 Apache-2.0 License 开源协议的开源项目，因为不用担心版权的问题
    </p>
    <hr/>
    <p>
     我们在工作中使用的绝大多数框架都是使用 Apache-2.0 License 开源协议的，例如：
    </p>
    <ul>
     <li>
      Spring 全家桶
     </li>
     <li>
      MyBatis、MybatisPlus
     </li>
     <li>
      ElasticSearch
     </li>
     <li>
      Alibaba 开源的一系列框架（Fastjson、SpringCloud ALibaba 等）
     </li>
     <li>
      MIT License
     </li>
    </ul>
    <h2>
     <a id="3_GPL30_License_43">
     </a>
     3. GPL-3.0 License
    </h2>
    <blockquote>
     <p>
      <strong>
       用大白话解释 GPL-3.0 License 开源协议：我帮助了你，你也得帮助别人
      </strong>
     </p>
    </blockquote>
    <p>
     GPL-3.0 License 可以理解为软件的一个使用规则，你可以免费使用这个软件，也可以修改它，甚至可以把修改后的软件分享给别人，但是你必须遵守以下几条简单的规则：
    </p>
    <ol>
     <li>
      别把软件藏着掖着，如果你给别人软件，也得把软件的源代码（就是程序员写的东西）一起给别人
     </li>
     <li>
      如果你修改了这个软件，得让别人知道你改了哪里，而且你分享出去的修改版软件也得遵守这些规则
     </li>
     <li>
      别在软件上再加一些限制性的规则，比如说不让别人修改或者分享
     </li>
     <li>
      如果你用了这个软件，别人不能因为软件里的某个功能申请了专利就告你
     </li>
     <li>
      如果你把这个软件和其他软件混合在一起，那么混合后的新软件也得遵守这些规则
     </li>
     <li>
      如果你不遵守这些规则，那你就没有权利使用这个软件了
     </li>
    </ol>
    <hr/>
    <p>
     也就是说如果你的项目用到了基于 GPL-3.0 License 开源协议的框架，你的项目也得使用 GPL-3.0 License 开源协议
    </p>
    <h2>
     <a id="4_LGPL30_License_60">
     </a>
     4. LGPL-3.0 License
    </h2>
    <blockquote>
     <p>
      <strong>
       用大白话解释 LGPL-3.0 License 开源协议：可以在私有项目中使用
      </strong>
     </p>
    </blockquote>
    <p>
     LGPL-3.0可以理解为软件的一个比较宽松的使用规则，你可以免费使用这个软件，也可以把它和你自己的软件结合起来卖钱，但是你也要遵守一些简单的规则：
    </p>
    <ol>
     <li>
      如果你修改了这个软件本身，那么你修改的部分需要把源代码公开出来，让别人也能看到你改了什么
     </li>
     <li>
      你可以把这个软件和你自己的软件一起卖，但是你卖的时候，得允许用户访问到这个软件的源代码，也就是说，你不能把源代码藏着掖着
     </li>
     <li>
      你不能在软件上再加一些限制性的规则，比如说禁止别人修改或者分享
     </li>
     <li>
      如果你用了这个软件，别人不能因为软件里的某个功能申请了专利就告你
     </li>
     <li>
      如果你把这个软件和其他软件混合在一起，那么混合后的新软件中，这个软件的那部分还是得遵守这些规则
     </li>
    </ol>
    <p>
     总的来说，LGPL-3.0比GPL-3.0要宽松一些，它允许你把软件用到商业产品中，不需要把整个产品的源代码都公开，但是你修改的这个软件部分的源代码还是要公开的。这样既保护了软件的自由性，又给了商用软件一定的灵活性
    </p>
    <h2>
     <a id="5_MIT_License_74">
     </a>
     5. MIT License
    </h2>
    <blockquote>
     <p>
      <strong>
       用大白话解释 MIT License 开源协议：随便用，用的时候提一下原作者就行了
      </strong>
     </p>
    </blockquote>
    <p>
     MIT License 就像是软件的一个非常简单的使用同意书，你可以随意使用这个软件，基本上想干嘛就干嘛，但是得遵守以下几个小要求：
    </p>
    <ol>
     <li>
      你得保留软件里的版权声明和许可声明，就是说，不能把说这是谁写的和这些使用规则给删了
     </li>
     <li>
      如果软件里有专利声明或者作者写的免责声明，你也得留着这些
     </li>
     <li>
      你可以用这个软件做任何事情，包括修改、合并到你的项目中，或者用在商业产品里卖钱，都不用给原作者支付费用或者通知原作者
     </li>
    </ol>
    <hr/>
    <p>
     简单来说，MIT 许可证给了你很大的自由度，你可以自由地使用、修改和分发这个软件，只要你在分发的软件里保留原作者的版权信息和 MIT 许可证的条款。这对于商业用途来说是非常友好的，因为你不需要公开你的源代码，也没有其他的附加条件
    </p>
