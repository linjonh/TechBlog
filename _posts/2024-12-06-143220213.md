    <h3 id="cGsuy">
     0.连接服务器
    </h3>
    <pre id="pcXR0"><code>mysql -h 127.0.0.1 -P 3306 -u root -p</code></pre>
    <ul>
     <li id="ua6b9fa72">
      -h：指明登录部署了MySQL服务的
      <span style="color:#4da8ee;">
       主机
      </span>
     </li>
     <li id="uc1c3570d">
      -P：指明要访问的
      <span style="color:#4da8ee;">
       端口号
      </span>
     </li>
     <li id="u5499379b">
      -u：指明登录
      <span style="color:#4da8ee;">
       用户
      </span>
     </li>
     <li id="ueb5899e9">
      -p：
      <span style="color:#4da8ee;">
       指明需要输入密码
      </span>
     </li>
    </ul>
    <p id="u45051f89">
     注意：
    </p>
    <ul>
     <li id="ua2573c6c">
      如果没有写-h 127.0.0.1，默认是连接本地
     </li>
     <li id="uc83c943e">
      如果没有写-P 3306，默认是连接3306号端口
     </li>
    </ul>
    <p id="u68eebd23">
    </p>
    <p class="img-center">
     <img alt="" height="970" id="u4f54012f" src="https://img-blog.csdnimg.cn/img_convert/4909151d353a51235d14714696ae747d.png" width="1200"/>
    </p>
    <p id="u51caae98">
     1.目前我们是免密码登录的，后面必须设置密码
     <br/>
     2.密码输入的时候，是不会回显的
    </p>
    <hr id="wPNRv"/>
    <h3 id="HoJOx">
     1. 什么是数据库
    </h3>
    <p id="u8d9bb0b7">
     <strong>
      服务器角度理解数据库
     </strong>
    </p>
    <p id="ufa0b0672">
     在
     <span style="color:#4da8ee;">
      连接数据库时，我们使用的是
      <code>
       mysql
      </code>
      ，启动数据库服务时使用的是
      <code>
       mysqld
      </code>
      。
     </span>
     这两个分别代表什么呢？
    </p>
    <p id="u3f8768fa">
    </p>
    <p class="img-center">
     <img alt="" height="539" id="uaa33ea51" src="https://img-blog.csdnimg.cn/img_convert/fb77633021d1a3c36ba41bf4791171fa.png" width="1200"/>
    </p>
    <ul>
     <li id="ubf75be80">
      <code>
       mysql
      </code>
      是数据库服务的
      <span style="color:#4da8ee;">
       客户端。
      </span>
     </li>
     <li id="u2c1d6d28">
      <code>
       mysqld
      </code>
      是数据库服务的
      <span style="color:#4da8ee;">
       服务器端。
      </span>
     </li>
    </ul>
    <blockquote>
     <p id="u8ba5f334">
      <strong>
       说明：
      </strong>
     </p>
     <ul>
      <li id="u93a35858">
       凡是可执行程序
       <span style="color:#4da8ee;">
        带有
        <code>
         d
        </code>
        的，通常称为守护进程。
       </span>
      </li>
      <li id="u69f168ed">
       <span style="color:#fe2c24;">
        <strong>
         MySQL本质：一个基于C/S（Client/Server）模式的网络服务
        </strong>
       </span>
      </li>
      <li id="ud04e58f8">
       <code>
        mysqld
       </code>
       就是网络服务的后端
      </li>
      <li id="u9913b576">
       通常我们提到MySQL，其实指的是MySQL的客户端
       <code>
        mysql
       </code>
       。
      </li>
     </ul>
    </blockquote>
    <p id="uf8842266">
     这就是我们对数据库的第一层理解。
    </p>
    <ul>
     <li id="u2a94f7f1">
      现在我们明白了为什么安装MySQL时需要通过
      <code>
       yum
      </code>
      安装
      <code>
       mysqld
      </code>
     </li>
     <li id="uad6dc366">
      并启动服务后才能使用
      <code>
       mysql
      </code>
      进行连接，因为它是一个网络服务。
     </li>
     <li id="u3f5f45c8">
      既然MySQL是网络服务，我们可以查找到它绑定的端口号。
     </li>
    </ul>
    <p id="u89264cfa">
    </p>
    <p class="img-center">
     <img alt="" height="491" id="BxTMP" src="https://img-blog.csdnimg.cn/img_convert/e20f8a2c4cc2e5bdcb88ab66fd08ba51.png" width="1200"/>
    </p>
    <p id="ue4188d96">
     <strong>
      MySQL服务是什么？
     </strong>
    </p>
    <ul>
     <li id="u199e0f99">
      MySQL是一种
      <span style="color:#fe2c24;">
       提供数据存取服务的网络程序。
      </span>
     </li>
    </ul>
    <h4 id="cwya3">
     口语中的数据库
    </h4>
    <p id="u6e8ec284">
     数据库通常指的是，
     <span style="color:#4da8ee;">
      在磁盘或内存中存储的特定结构化的数据
     </span>
     。
    </p>
    <ul>
     <li id="u5473d01e">
      数据库可以被看作是一套存储方案，通常指的是关系型数据库，表与表之间有特定的关系，
      <span style="color:#4da8ee;">
       数据按照特定逻辑进行组织。
      </span>
     </li>
     <li id="uddc5b147">
      数据库最终还是存储在磁盘或内存中。我们暂时只考虑磁盘，存储在磁盘中的一组数据库文件就可以称为数据库，即一批数据以数据库的形式存储在磁盘中。
     </li>
    </ul>
    <p id="u85c3430e">
     我们可以区分如下几点：
    </p>
    <ul>
     <li id="ud0300a91">
      数据库服务：
      <code>
       mysqld
      </code>
     </li>
     <li id="u0df5bbcc">
      数据库客户端：
      <code>
       mysql
      </code>
     </li>
     <li id="uddb03b6e">
      <span style="color:#4da8ee;">
       数据库：磁盘上存储的一套数据库方案（文件）
      </span>
     </li>
    </ul>
    <p id="uaa1db453">
     数据库的本质仍然是存储在磁盘中的某种结构化数据，虽然我们可能对它的具体形式还不完全理解。文件是我们熟悉的存储形式
    </p>
    <h4 id="oVfvX">
     为什么数据不直接以文件形式存储，而需要使用数据库呢？
    </h4>
    <p id="uf6659256">
     虽然文件提供了数据存储的基本功能，但它
     <span style="color:#4da8ee;">
      没有提供对数据内容的管理能力。
     </span>
    </p>
    <ul>
     <li id="uc06089f0">
      举个例子，假设一个文件有10万行内容，每行都是一个IP地址，现在
      <strong>
       需要统计以
      </strong>
      <code>
       <strong>
        127
       </strong>
      </code>
      <strong>
       开头的IP地址个数。
      </strong>
     </li>
     <li id="u18c0a0ac">
      程序员需要打开文件，
      <span style="color:#4da8ee;">
       按行读取并手动统计
      </span>
      ，这是一项耗时的工作。
     </li>
     <li id="u8b744fe5">
      文件只提供基本的读写功能，而没有数据管理能力，因此不方便。
     </li>
    </ul>
    <p id="u633cfdd5">
     <strong>
      数据库的意义：
     </strong>
     它是一种
     <span style="color:#fe2c24;">
      数据存储的解决方案
     </span>
     。程序员只需提出查询需求，数据库就会根据需求分析并返回结果。
     <span style="color:#fe2c24;">
      相当于是将数据管理起来了，使数据查询更加方便
     </span>
    </p>
    <p id="ua3907aca">
     <span style="color:#956fe7;">
      <strong>
       例如
      </strong>
     </span>
     ，如果将10万个IP地址
     <span style="color:#4da8ee;">
      存储到数据库中，查询以
      <code>
       127
      </code>
      开头的IP地址时，
     </span>
     数据库会进行内部处理，
     <span style="color:#4da8ee;">
      最终返回统计结果
     </span>
     ，而程序员无需自己编写复杂的代码。
    </p>
    <p id="u7e7931be">
     <strong>
      总结数据库的作用
     </strong>
    </p>
    <p id="uad491cbf">
     数据库本质上是一套对外提供数据存储和管理的解决方案。它包括：
    </p>
    <ul>
     <li id="uf959105a">
      数据库客户端（
      <code>
       mysql
      </code>
      ）
     </li>
     <li id="ub4a84a6b">
      数据库服务器（
      <code>
       mysqld
      </code>
      ）
     </li>
     <li id="u36e0e5bd">
      存储在磁盘中的数据库文件
     </li>
    </ul>
    <p id="ufb97070f">
     数据库的作用是让客户端提出需求后，
     <span style="color:#4da8ee;">
      服务器从数据库文件中进行操作，最终将结果返回客户端，展示给用户。
     </span>
    </p>
    <p id="u18cccb5c">
     <strong>
      理解
     </strong>
    </p>
    <p id="ue6d4ae74">
    </p>
    <p>
     <img alt="" height="1056" src="https://i-blog.csdnimg.cn/direct/7c19352c3f474d8b8dadc3364d467a25.jpeg" width="1200"/>
    </p>
    <ul>
     <li id="u7b89a7b8">
      当一个MySQL客户端mysql提出一个需求（查询
     </li>
     <li id="ub2873592">
      MySQL服务端mysqld就去数据库文件当中自己进行增、删、查、改等，然后把结果返回给
      <span style="color:#4da8ee;">
       mysqld
      </span>
     </li>
     <li id="uf29ccb0a">
      然后mysqld把结果交给mysql，然后就显示给用户了
     </li>
    </ul>
    <hr id="FsYbk"/>
    <h4 id="opq0B">
     总结
    </h4>
    <p id="u8f901f96">
     <strong>
      最后的结论——什么是数据库？
     </strong>
    </p>
    <ol>
     <li id="u9f07afbe">
      <strong>
       局部理解：
      </strong>
      数据库是存储在
      <span style="color:#fe2c24;">
       磁盘中的结构化数据文件
      </span>
      ，这些文件能够被
      <code>
       mysqld
      </code>
      服务进程进行增删查改。
     </li>
     <li id="ue67532d6">
      <strong>
       宏观理解：
      </strong>
      <span style="color:#fe2c24;">
       数据库是一整套数据存储的解决方案。
      </span>
     </li>
    </ol>
    <p id="u0e1c361c">
     <strong>
      文件保存数据的缺点：
     </strong>
    </p>
    <ul>
     <li id="u1cbd0304">
      安全性问题
     </li>
     <li id="u40acb14e">
      不利于数据
      <span style="color:#4da8ee;">
       查询和管理
      </span>
     </li>
     <li id="ub47e06ee">
      不利于存储海量数据
     </li>
     <li id="ufe766f2c">
      程序中控制不便
     </li>
    </ul>
    <p id="ub81dcf17">
     <strong>
      数据库的存储介质：
     </strong>
    </p>
    <ul>
     <li id="u495f4e2d">
      磁盘
     </li>
     <li id="u5da2a000">
      内存
     </li>
    </ul>
    <p id="u606c58de">
     为了解决文件存储的这些问题，专家们设计了数据库，能够更有效地管理数据。
     <span style="color:#1a439c;">
      数据库的使用和管理是衡量程序员水平的重要指标。
     </span>
    </p>

      </li>

    <p id="u5ab228d0">

     </li>
    </ol>
    <pre id="omI02"><code>vim /etc/my.cnf</code></pre>

    </p>

    <p id="ude5d7819">

    <p class="img-center">

    <h3 id="b515f123">
     数据逻辑存储
    </h3>
    <p id="ubdb816f4">
     <strong>
      数据逻辑存储
     </strong>
     在表中插入数据后，这些数据在
     <span style="color:#fe2c24;">
      逻辑上是以行列式的结构存储的。
     </span>
    </p>
    <ul>
     <li id="u679c3ff8">
      一行表示一个实体（例如一个人）
     </li>
     <li id="u2cb4aca3">
      一列表示实体的某个属性（如姓名、年龄、性别）。
     </li>
    </ul>
    <p id="u87ccc796">
    </p>
    <p class="img-center">
     <img alt="" height="377" id="pubXQ" src="https://img-blog.csdnimg.cn/img_convert/23bbfecbeb185d8830984137d0131f53.png" width="778"/>

    <p id="u171f0c64">
     数据库内的数据在
     <span style="color:#fe2c24;">
      物理上并
     </span>
     不是直接存储为表的形式，而是通过 MySQL
     <span style="color:#fe2c24;">
      特定的协议和格式进行存储和管理。
     </span>
    </p>
    <hr/>
    <h3 id="00296926">
     MySQL架构
    </h3>
    <p id="u2f844f90">
     MySQL是一个可移植的数据库，几乎能在当前所有的操作系统上运行
    </p>
    <ul>
     <li id="uc2fb352d">
      如 Unix/Linux、Windows、 Mac和Solaris
     </li>
     <li id="u2b35263b">
      各种系统在底层实现方面各有不同，但是 MySQL基本上能保证在各个平台上的物理体系结构的一致性
     </li>
    </ul>
    <p id="u8e3d0e2c">
    </p>
    <p id="u6527a2f6">
     最上面是MySQL客户端，现在我们在Linux下是以命令行方式使用数据库，后面还会以C/C++，图像化界面使用数据库。
    </p>
    <p id="u77ab4f91">
     整体对于一个MySQL Server来讲主要功能由三层构成：
    </p>
    <ul>
     <li id="ud7689078">
      <strong>
       第一层：连接池
      </strong>
      —— 主要是安全连接管理(连接管理、鉴权、保证安全…)。
     </li>
     <li id="ud70f80db">
      <strong>
       中间这一层
      </strong>
      —— 比如说客户端会下达各种SQL指令，收到指令然后会对指令做各种语法、词法分析，甚至会对SQL语句做一定程度优化按照SQL协议然后下达给下一层。
     </li>
     <li id="ue2051045">
      <strong>
       第三层：存储引擎
      </strong>
      —— 匹配的就是一个一个存储引擎，作用有点像计算机
      <span style="color:#4da8ee;">
       体系结构中的驱动
      </span>
      。下面可能有不同种类的引擎，这些存储引擎从上层接收下达下来的经过词法语法调优过的SQL语句，然后
      <span style="color:#fe2c24;">
       存储引擎对这些SQL语句
       <strong>
        解释
       </strong>
      </span>
      ，说白了
      <span style="color:#4da8ee;">
       最下面这一层才是真正干事
      </span>
      的，它帮我们去访问指定的数据库文件，访问指定的表。把数据进行增删查改。
     </li>
    </ul>
    <p id="u09a066d4">
    </p>
    <p class="img-center">
     <img alt="" height="1056" id="izM5i" src="https://img-blog.csdnimg.cn/img_convert/ee338e7b0ba0eb66808cf15e58c12b4a.png" width="1200"/>
    </p>
    <blockquote>
     <p id="u6ec3d8a0">
      怎么会有这么多存储引擎呢？
     </p>
    </blockquote>
    <ul>
     <li id="ue3192067">
      主要还是因为数据库是给我们提供数据存储服务的，但是这个
      <span style="color:#fe2c24;">
       数据有种类的差别
      </span>
      ，有的是文档型的，有的是二进制型的，有的是大文件型的等等
     </li>
     <li id="uffe0184f">
      所以针对不同种类的数据的需求可能采用不同的存储方案
     </li>
     <li id="u632d8bba">
      所以MySQL给我们提供各种各样的搜索引擎，诸如
      <span style="color:#4da8ee;">
       MyISAM适合大文件读取，InnoDB
      </span>
      有很丰富的索引支持可以方便进行
      <span style="color:#4da8ee;">
       快速的搜索查找。
      </span>
     </li>
     <li id="uc34b6737">
      所以可以针对不同的场景由用户来配置MySQL底层，采用
      <strong>
       不同的存储引擎来满足上层的存储需求。
      </strong>
     </li>
    </ul>
    <p id="u0c030cdd">
    </p>
    <p id="u69acad8f">
     <strong>
      总结一下MySQL Server有三层
     </strong>
     ：
    </p>
    <ul>
     <li id="u5b381125">
      第一层：
      <span style="color:#4da8ee;">
       安全连接管理
      </span>
     </li>
     <li id="u189c4d83">
      第二层：词法、语法
      <span style="color:#4da8ee;">
       分析
      </span>
      ，SQL语句
      <span style="color:#4da8ee;">
       优化
      </span>
     </li>
     <li id="u35abacbe">
      第三层：
      <span style="color:#4da8ee;">
       解释 SQL 语句
      </span>
      ，具体完成数据存储方案
     </li>
     <li id="u2e3da205">
      最下面一层：就是它
      <span style="color:#4da8ee;">
       依赖的对应文件系统
      </span>
      ，帮我们把数据
      <span style="color:#4da8ee;">
       以二进制方式
      </span>
      存到特定的目录下然后构建特定的普通文件把数据存储好。
     </li>
    </ul>
    <p id="u407d1356">
     <strong>
      最终总结一下
     </strong>
     ，MySQL依旧是一个文件系统，只不过处于OS提供的文件系统之上的一套存储解决方案。
    </p>
    <p id="u537dcc1e">
     <strong>
      最后一点
     </strong>
     ，MySQL提供这么多的解决方案，它是支持热插拔的可以理解成
     <span style="color:#4da8ee;">
      插件一样
     </span>
     。我们想
     <span style="color:#4da8ee;">
      选择哪一个就把哪一个存储引擎加载到MySQL里。
     </span>
    </p>

    </h3>

     </strong>
     数据
     <span style="color:#4da8ee;">

     </span>

    </p>

       （
       <span style="color:#956fe7;">
        文件
       </span>
      </strong>
     </li>
    </ul>

    </p>
    <ul>

    </ul>

    </p>
    <ul>

    </h4>
    <ul>
     <li id="ua1f5ae47">
      <strong>
       存储引擎
      </strong>

     </li>
     <li id="uc606022c">
      <strong>
