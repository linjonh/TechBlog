    <h1 class="title-article" id="articleContentId">
     微软的数据交换新技术--Live Clipboard
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     微软在
     <a href="http://blog.csdn.net/zdg/archive/2005/11/27/537956.aspx" rel="noopener noreferrer" target="_blank">
      扩展RSS标准为SSE
     </a>
     以后，最近又有新的动作，CTO Ray Ozzie在O'Reilly的
     <a href="http://conferences.oreillynet.com/etech/" rel="nofollow noopener noreferrer" target="_blank">
      Emerging Technology Conference
     </a>
     上演示了一种新的数据交换技术，他们把这个称作Live Clipboard。
    </p>
    <p>
     <img align="right" alt="" border="0" src="https://i-blog.csdnimg.cn/blog_migrate/fbdd119019f44cd56579d044e19b0d28.gif">
      3月7日，Ray Ozzie发表的
      <a href="http://spaces.msn.com/rayozzie/Blog/cns!FB3017FBB9B2E142!285.entry" rel="nofollow noopener noreferrer" target="_blank">
       Wiring the Web
      </a>
      提到了他对这个新概念的构思起源，其中提到，目前的用户模型中还不能简单解决网站和网站、网站和PC、网站和其他应用之间的结构化数据交换。他举例说，在目前的Blog中流行的RSS源可以通过3种途径获取：
     </img>
    </p>
    <blockquote>
     <ul>
      <li>
       1) 基于PC端的聚合，比如：NewsGator或NetNewsWire
      </li>
      <li>
       2) 基于Web的聚合，比如：Bloglines或live.com
      </li>
      <li>
       3) 基于浏览器的聚合，比如：Internet Explorer(7.0开始支持)或Firefox
      </li>
     </ul>
    </blockquote>
    <p>
     对于第3种方式，用户可以通过
     <img alt="" border="0" src="http://sean.members.winisp.net/posts/icons/feed-icon16x16.png">
      图标加以区分，但是如果是第1,2种方式，用户很难发现哪里是Feed的标志，就像很多人的Blog旁边挂上很多的订阅标志，相信大家对右边的典型图例并不陌生。
     </img>
    </p>
    <p>
     Ray Ozzie的想法其实就是将PC中大家熟知的剪贴板(clipboard)技术扩展到Web上。让我们先记住这个图标
     <img alt="" border="0" src="https://i-blog.csdnimg.cn/blog_migrate/3af3b3dea3d198d439b369e3a19b21d0.gif">
      ，再看看以下几个演示：
      <br/>
     </img>
    </p>
    <blockquote>
     <ul>
      <li>
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/screencast/webtoweb/WebToWeb.html" rel="nofollow noopener noreferrer" target="_blank">
        Bridging The Web: Website to Website Structured Data Exchange
       </a>
       <br/>
       演示网站之间的数据采用特定XML的方式可以拷贝/粘贴结构化的数据，比如：联络人信息和日常信息。大家也可以在
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/liveclipsample/clipboardexample.html" rel="nofollow noopener noreferrer" target="_blank">
        这里实际体验
       </a>
       。
      </li>
      <li>
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/screencast/pctoweb/pctoweb.html" rel="nofollow noopener noreferrer" target="_blank">
        Bridging The Web: PC to Web Structured Data Exchange
       </a>
       <br/>
       演示Calendar软件和eventful网站的日程信息通过clipboard进行交换。最有意思的是，个人信息进入clipboard，在进入需要填写个人的信息的站点(比如电子商务站点)，就不需要重复填写了。
      </li>
      <li>
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/screencast/BlogSubscribe/BlogSubscribe.html" rel="nofollow noopener noreferrer" target="_blank">
        Wiring the Web: An Improved Weblog Subscription Method
       </a>
       <br/>
       演示在Blog中支持clipboard后，可以非常方便在Bloglines等网站中订阅信息，而不需要那么多形式各异的Feed图标。
      </li>
      <li>
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/screencast/WireSiteToSite/WireSiteToSite.html" rel="nofollow noopener noreferrer" target="_blank">
        Wiring the Web: Piping Data Website to Website
       </a>
       <br/>
       演示MSN Space中个人信息和facebook中信息的同步更新，更Cool的是还可以用到地图中。
      </li>
      <li>
       <a href="http://spaces.msn.com/editorial/rayozzie/demo/liveclip/screencast/WireSiteToPC/WireSiteToPC.html" rel="nofollow noopener noreferrer" target="_blank">
        Wiring the Web: Piping Data Website to PC
       </a>
       <br/>
       演示Flickr中图片/分类通过clipboard拷贝到本地文件/目录，如果是目录，还保存了一个XML来实现文件的同步。
      </li>
     </ul>
    </blockquote>
    <p>
    </p>
    <p>
     需要说明的是，这些应用都是虚拟应用，目前Bloglines、Flickr等还不可能支持Live Clipboard，不过，微软的这些想法确实非常棒，如果说SSE只是微软在RSS上迈出的一小步，Live Clipboard已经全面超越了RSS。从信息流转本质开始考虑解决方案，看来微软在Web上的野心不小。
    </p>
    <p>
     留给Live Clipboard探讨的地方还非常多，比如：
    </p>
    <ul>
     <li>
      Live Clipboard应该支持哪些结构化数据？
     </li>
     <li>
      如果自动同步，如何保证数据传输过程中安全性？
     </li>
    </ul>
