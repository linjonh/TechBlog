     用户体验五要素--战略层、范围层、结构层、框架层、表现层
    </h1>

    </svg>
    <p>
     <strong>
      用户体验五要素
     </strong>
    </p>
    <p>
     分为
     <strong>
      功能型产品
     </strong>
     和
     <strong>
      信息型产品
     </strong>
     。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1b0d4108488df438eb5d00e0ba448f8a.png#pic_center"/>
    </p>
    <ol>
     <li>
      <strong>
       战略层
      </strong>
     </li>
    </ol>
    <p>
     <strong>
      （1）产品目标
     </strong>
    </p>
    <ul>
     <li>
      很多人称之为 “
      <strong>
       商业目标
      </strong>
      ” 或 “
      <strong>
       商业驱动因素
      </strong>
      ”；
     </li>
     <li>
      对于任何一个网站，需要明确描述的基础之一是
      <strong>
       品牌识别
      </strong>
      。
      <strong>
       品牌识别
      </strong>
      ——可以是概念系统，也可以是情绪反应——重要的原因在于它无法不被用户注意，深深地印在用户脑海中。
     </li>
     <li>
      <strong>
       成功标准
      </strong>
      ——是否满足自己的目标和用户需求。
     </li>
    </ul>
    <p>
     <strong>
      （2）用户需求
     </strong>
    </p>
    <ul>
     <li>
      进行
      <strong>
       用户调研
      </strong>
      （首先根据不同的用户群体进行
      <strong>
       用户细分
      </strong>
      ）
     </li>
     <li>
      方法：
      <br/>
      市场调研（问卷调查、用户访谈、焦点小组）——获取用户基本信息；
      <br/>
      现场调查——用户了解用户行为；
      <br/>
      用户测试——可进行产品优化改版。
     </li>
    </ul>
    <p>
     <strong>
      产出：战略文档
     </strong>
     ——简洁明了并切入重点，帮助其他人（设计师、程序员、信息架构师、项目经理）做出正确的决定
    </p>
    <hr/>
    <ol start="2">
     <li>
      <strong>
       范围层
      </strong>
      <br/>
      分为两个部分：
      <strong>
       功能规格
      </strong>
      和
      <strong>
       内容需求
      </strong>
      。
     </li>
    </ol>
    <p>
     <strong>
      （1）
      <strong>
       功能型产品
      </strong>
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       需求定义
      </strong>
     </li>
     <li>
      <strong>
       功能规格说明
      </strong>
      ——不需要包含细节，只包含在设计或开发中出现有可能出现混淆的功能定义，只记录创建产品时 已经确定下来的决议。
     </li>
     <li>
      <strong>
       撰写规则
      </strong>
      ：乐观（不允许XXX改为引导XXX）、具体（解释清楚）、明确（避免歧义）。
     </li>
    </ul>
    <p>
     <strong>
      （2）
      <strong>
       信息型产品
      </strong>
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       内容需求
      </strong>
      ——内容清单（明确每个人负责的部分）
     </li>
     <li>
      <strong>
       确定需求的优先级
      </strong>
      ——评估需求是否能满足战略目标（战略目标对需求是一对多的关系）
     </li>
    </ul>
    <hr/>
    <ol start="3">
     <li>
      <strong>
       结构层
      </strong>
      <br/>
      两部分：
      <strong>
       交互设计
      </strong>
      和
      <strong>
       信息架构
      </strong>
      。
      <br/>
      交互设计和信息架构都强调一个重点：确定各个将要呈现给用户的元素的 “
      <strong>
       模式
      </strong>
      ” 和 “
      <strong>
       顺序
      </strong>
      ”。
      <strong>
       交互设计
      </strong>
      关注于将影响用户执行和完成任务的元素，
      <strong>
       信息架构
      </strong>
      则关注如何用户表达给用户的元素。
     </li>
    </ol>
    <p>
     <strong>
      （1）
      <strong>
       功能型产品
      </strong>
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       交互设计
      </strong>
      ——可能的用户行为，系统如何配合与响应。
     </li>
     <li>
      <strong>
       概念模型
      </strong>
      ——交互组件如何工作。
     </li>
     <li>
      <strong>
       错误处理
      </strong>
      —— “
      <strong>
       撤销
      </strong>
      ” 功能、提示信息等。
     </li>
    </ul>
    <p>
     <strong>
      （2）
      <strong>
       信息型产品
      </strong>
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       信息架构
      </strong>
      ——呈现给用户的信息是否合理并有意义。
     </li>
     <li>
      <strong>
       结构方法
      </strong>
      ：基本单位是节点。层级结构（树或中心辐射）、矩阵结构、自然结构、线性结构。
     </li>
     <li>
      <strong>
       组织原则
      </strong>
      ：节点编组、命名规则等。
     </li>
    </ul>
    <hr/>
    <ol start="4">
     <li>
      <strong>
       框架层
      </strong>
      <br/>
      对于
      <strong>
       功能型产品
      </strong>
      通过
      <strong>
       界面设计
      </strong>
      来确定框架，对于
      <strong>
       信息型产品
      </strong>
      要解决唯一问题：
      <strong>
       导航设计
      </strong>
      。
      <strong>
       信息设计
      </strong>
      两种产品都要做。
     </li>
    </ol>
    <ul>
     <li>
      <p>
       <strong>
        界面设计
       </strong>
       ——让用户一眼看出“最重要东西”。第一次呈现给用户时指定默认值。
      </p>
     </li>
     <li>
      <p>
       <strong>
        导航设计
       </strong>
       <br/>
       <strong>
        三个目标：
       </strong>
       <br/>
       ① 必须提供给用户在网站间跳转的方法；
       <br/>
       ② 必须传达出这些元素和他们所包含内容之间的联系；
       <br/>
       ③ 必须传达出它的内容的用户当前浏览页面之间的关系。
       <br/>
       <strong>
        几种导航系统：
       </strong>
       <br/>
       全局导航、局部导航、辅助导航、上下文导航、友好导航、远程导航（网站地图、索引表）
      </p>
     </li>
     <li>
      <p>
       <strong>
        信息设计
       </strong>
       ——如何呈现信息，使用户更好的理解。
       <br/>
       信息设计和导航设计结合到一起的作用：
       <strong>
        指示标识
       </strong>
       。
       <br/>
       <strong>
        线框图：
       </strong>
       （原型图）——完成详细文档。
      </p>
     </li>
    </ul>
    <hr/>
    <ol start="5">
     <li>
      <strong>
       视觉层
      </strong>
      <br/>
      <strong>
       感知设计
      </strong>
      <br/>
      对比和一致性
      <br/>
      配色和排版
     </li>
    </ol>
    <p>
     <strong>
      各个层次之间界限模糊，彼此交叉，需要反复迭代
     </strong>
    </p>
