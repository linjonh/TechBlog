    <h2>
     <a id="vue_2">
     </a>
     vue项目本地打包教程
    </h2>
    <p>
     问题概述：
     <br/>
     在我们完成一个vue项目后，紧接着就是需要把我们的项目打包发布上线，但是我们怎么确定我们自己打包的项目放到服务器上是没有问题的呢？
     <br/>
     所以本地打包测试尤为重要！
     <br/>
     在这里，我们先默认你已经写好了一个vue项目，然后我们开始打包
    </p>
    <h3>
     <a id="_8">
     </a>
     项目打包
    </h3>
    <pre><code class="prism language-c">npm run build
<span class="token comment">//或者</span>
# 构建测试环境
npm run build<span class="token operator">:</span>stage

# 构建生产环境

npm run build<span class="token operator">:</span>prod
</code></pre>
<p>
具体的打包代码得根据实际项目来看，当我们运行完成命令后，会在如图所示
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cb62756cd0d0ed7979fc4e0c80ab6df1.png">
<br/>
在打包完成后，会在我们的项目文件夹中生成一个 dist 文件夹，这个文件夹便是我们需要发布到线上的东西
</img>
</p>
<h3>
<a id="_23">
</a>
本地环境安装
</h3>
<p>
通过 win+R 唤起输入框输入 cmd，调取 CMD 命令运行框，当然 powershell 也可以，想用哪个都可以，然后执行以下安装命令
</p>
<pre><code class="prism language-c">npm install http<span class="token operator">-</span>server <span class="token operator">-</span>g
</code></pre>
<p>
安装截图
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1dd19498f8f53edc55d6ea571ff57c1e.png">
<br/>
安装完成之后我们需要去到项目的 dist 文件夹
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d98feebd36714e5a4bb727e8bc43ecc0.png">
<br/>
进入到文件夹之后召唤以下命令窗口
</img>
</img>
</p>
<p>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/38a01f0cf60aa7f8edd52a8f784ef4d7.png">
<br/>
然后在命令窗口中执行本地服务器开启命令
</img>
</p>
<h3>
<a id="http_36">
</a>
开启本地 http 服务
</h3>
<pre><code class="prism language-c">http<span class="token operator">-</span>server
</code></pre>
<p>
当然，到这里我们并不指定一种类型的命令操作窗口，只要能够在 dist 文件夹同级打开就可以了
<br/>
在执行完命令之后，就会一串本地地址，复制地址到浏览器就可以啦~
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2dabac65580260a1a6dbdaaf373b60aa.png">
<br/>
执行完成之后记得这个命令窗口不能关闭哦，因为关闭了浏览器就看不了了，现在我们来看一下浏览器的情况
</img>
</p>

如果不用了，直接吧刚才的命令窗口关闭就可以了
</img>
</p>
