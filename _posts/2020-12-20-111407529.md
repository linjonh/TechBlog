    <h2>
     <a id="_5">
     </a>
     二、搜索算法的原理
    </h2>
    <ul>
     <li>
      搜索算法的原理就是枚举。利用计算机的高性能，给出人类制定好的规则，枚举出所有可行的情况，找到可行解或者最优解。
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/8512abfe5eb34da9a10309259c9d84d3.gif#pic_center"/>
     </li>
     <li>
      比较常见的搜索算法是 深度优先搜索（又叫深度优先遍历） 和 广度优先搜索（又叫广度优先遍历 或者 宽度优先遍历）。各种图论的算法基本都是依靠这两者进行展开的。
     </li>
     <li>
      深度优先搜索一般用来求可行解，利用剪枝进行优化，在树形结构的图上用处较多；而广度优先搜索一般用来求最优解，配合哈希表进行状态空间的标记，从而避免重复状态的计算；
     </li>
    </ul>
    <h2>
     <a id="_11">
     </a>
     三、深度优先搜索
    </h2>
    <h3>
     <a id="1DFS_12">
     </a>
     1、DFS
    </h3>
    <p>
     <strong>
      1）算法原理
     </strong>
    </p>
    <ul>
     <li>
      深度优先搜索（Depth First Search），是图遍历算法的一种。用一句话概括就是：“一直往下走，走不通回头，换条路再走，直到无路可走”。具体算法描述为：
     </li>
    </ul>
    <blockquote>
     <p>
      选择一个起始点
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         u 
         
        
       
         u
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.4306em;">
          </span>
          <span class="mord mathnormal">
           u
          </span>
         </span>
        </span>
       </span>
      </span>
      作为
      <strong>
       当前结点
      </strong>
      ，执行如下操作：
      <br/>
      a. 访问 当前结点，并且标记该结点已被访问，然后跳转到 b；
      <br/>
      b. 如果存在一个和 当前结点 相邻并且尚未被访问的结点 v，则将 v 设为 当前结点，继续执行 a；
      <br/>
      c. 如果不存在这样的 v，则进行回溯，回溯的过程就是回退 当前结点；
     </p>
    </blockquote>
    <ul>
     <li>
      上述所说的
      <strong>
       当前结点
      </strong>
      需要用一个栈来维护，每次访问到的结点入栈，回溯的时候出栈（栈是数据结构中线性表的一种，我会另外开辟一个章节来仔细讲解栈的原理和应用）。
     </li>
     <li>
      除了栈，另一种实现深度优先搜索的方式是递归，代码更加简单，相对好理解；
     </li>
    </ul>
    <blockquote>
     <p>
      【例题1】给定一个 n 个结点的无向图，要求从 0 号结点出发遍历整个图，求输出整个过程的遍历序列。其中，遍历规则为：
      <br/>
      1）如果和 当前结点 相邻的结点已经访问过，则不能再访问；
      <br/>
      2）每次从和 当前结点 相邻的结点中寻找一个编号最小的没有访问的结点进行访问；
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/cbe6d07267f3b7c85dde2cc711794cfc.png#pic_center"/>
     </p>
     <center>
      图三-1
     </center>
     <p>
     </p>
    </blockquote>
    <ul>
     <li>
      如图1所示，对上图以深度优先的方式进行遍历，起点是 0；
     </li>
     <li>
      &lt;1&gt; 第一步，当前结点为 0，标记已访问，然后从相邻结点中找到编号最小的且没有访问的结点 1；
     </li>
     <li>
      &lt;2&gt; 第二步，当前结点为 1，标记已访问，然后从相邻结点中找到编号最小的且没有访问的结点 3；
     </li>
     <li>
      &lt;3&gt; 第三步，当前结点为 3，标记已访问，没有尚未访问的相邻结点，执行回溯，回到结点 1；
     </li>
     <li>
      &lt;4&gt; 第四步，当前结点为 1，从相邻结点中找到编号最小的且没有访问的结点 4；
     </li>
     <li>
      &lt;5&gt; 第五步，当前结点为 4，标记已访问，然后从相邻结点中找到编号最小的且没有访问的结点 5；
     </li>
     <li>
      &lt;6&gt; 第六步，当前结点为 5，标记已访问，然后从相邻结点中找到编号最小的且没有访问的结点 2；
     </li>
     <li>
      &lt;7&gt; 第七步，当前结点为 2，标记已访问，然后从相邻结点中找到编号最小的且没有访问的结点 6；
     </li>
     <li>
      &lt;8&gt; 第八步，当前结点为 6，标记已访问，没有尚未访问的相邻结点，执行回溯，回到结点 2；
     </li>
     <li>
      &lt;9&gt; 第九步，按照 2 =&gt; 5 =&gt; 4 =&gt; 1 =&gt; 0 的顺序一路回溯，搜索结束；
     </li>
    </ul>

    <h3>
     <a id="3DFS_267">
     </a>
     3、基于DFS的剪枝
    </h3>
    <p>
     <strong>
      1）算法原理
     </strong>
    </p>
    <ul>
     <li>
      搜索的过程可以看作是从树根出发，遍历一棵倒置的树——搜索树的过程。而剪枝，顾名思义，就是通过某种判断，避免一些不必要的遍历过程，形象的说，就是剪去了搜索树中的某些“枝条”，故称剪枝（原话取自1999年OI国家集训队论文《搜索方法中的剪枝优化》（齐鑫））。
     </li>
     <li>
      如图三-8所示，它是一棵利用深度优先搜索遍历的搜索树，可行解（或最优解）位于绿色的叶子结点，那么根结点的最左边的子树完全没有必要搜索（因为不可能出解）。如果我们在搜索的过程中能够清楚地知道哪些子树不可能出解，就没必要往下搜索了，也就是将连接不可能出解的子树的那根“枝条”剪掉，图中红色的叉对应的“枝条”都是可以剪掉的。
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/49bd2ff7a97ebe5aced4286fd9ce28c4.gif#pic_center"/>
     </li>
    </ul>
    <center>
     图三-8
    </center>
    <ul>
     <li>
      好的剪枝可以大大提升程序的运行效率，那么问题来了，如何进行剪枝？我们先来看剪枝需要满足什么原则：
     </li>
    </ul>
    <p>
     <strong>
      a. 正确性
     </strong>
    </p>
    <ul>
     <li>
      剪掉的子树中如果存在可行解（或最优解），那么在其它的子树中很可能搜不到解导致搜索失败，所以剪枝的前提必须是要正确；
     </li>
    </ul>
    <p>
     <strong>
      b. 准确性
     </strong>
    </p>
    <ul>
     <li>
      剪枝要“准”。所谓“准”，就是要在保证在正确的前提下，尽可能多得剪枝。
     </li>
    </ul>
    <p>
     <strong>
      c. 高效性
     </strong>
    </p>
    <ul>
     <li>
      剪枝一般是通过一个函数来判断当前搜索空间是否是一个合法空间，在每个结点都会调用到这个函数，所以这个函数的实现效率很重要。
     </li>
     <li>
      剪枝大致可以分成两类：可行性剪枝、最优性剪枝（上下界剪枝）。
     </li>
    </ul>
    <p>
     <strong>
      2）可行性剪枝
     </strong>
    </p>
    <ul>
     <li>
      可行性剪枝一般是处理可行解的问题，如一个迷宫，问能否从起点到达目标点之类的。
     </li>
    </ul>

     <strong>
      3）最优性剪枝（上下界剪枝）
     </strong>
    </p>
    <ul>
     <li>
      最优性剪枝一般是处理最优解的问题。以求两个状态之间的最小步数为例，搜索最小步数的过程：一般情况下，需要保存一个“当前最小步数”，这个最小步数就是当前解的一个下界d。在遍历到搜索树的叶子结点时，得到了一个新解，与保存的下界作比较，如果新解的步数更小，则令它成为新的下界。搜索结束后，所保存的解就是最小步数。而当我们已经搜索了k歩，如果能够通过某种方式估算出当前状态到目标状态的理论最少步数 s 时，就可以计算出起点到目标点的理论最小步数，即估价函数 h = k + s，那么当前情况下存在最优解的必要条件是h &lt; d，否则就可以剪枝了。最优性剪枝是不断优化解空间的过程。
     </li>
     <li>
      关于剪枝技巧还有很多，本文就不多展开了，后面章节会专门对剪枝技巧进行一个梳理，详细剖析每个问题。
     </li>
    </ul>

     <a id="_334">

     四、广度优先搜索
    </h2>
    <h3>
     <a id="1BFS_335">
     </a>
     1、BFS
    </h3>
    <p>
     <strong>
      1）算法原理
     </strong>
    </p>
    <ul>
     <li>
      广度优先搜索即Breadth First Search，也是图遍历算法的一种。用一句话概括就是：“我会分身我怕谁？！”。我们来看一个玛丽救公主的故事，如图四-1-1所示就是广搜的精髓。
     </li>
     <li>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/c7dd5ca51bcf7c123fb54711934711dd.gif#pic_center"/>
      <center>
       图四-1-1
      </center>
      <center>
      </center>
     </li>
     <li>
      BFS的具体算法描述为：
     </li>
    </ul>
    <blockquote>
     <p>
      选择一个起始点 u 放入一个先进先出的队列中，执行如下操作：
      <br/>
      a. 如果队列不为空，弹出一个队列首元素，记为 当前结点，执行b；否则算法结束；
      <br/>
      b. 将与 当前结点 相邻并且尚未被访问的结点的信息进行更新，并且全部放入队列中，继续执行a；
     </p>
    </blockquote>
    <ul>
     <li>
      维护广搜的数据结构是队列和哈希表，也就是很多数据结构书上所说的
      <span class="katex--inline">
       <span class="katex">

         open-close
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.7778em; vertical-align: -0.1944em;">
          </span>
          <span class="mord mathnormal">
           o
          </span>
          <span class="mord mathnormal">
           p
          </span>
          <span class="mord mathnormal">

          </span>

      表，哈希表主要是用来标记状态的，比如某个状态并不是一个整数，可能是一个字符串，就需要用字符串映射到一个整数，可以自己写个散列哈希表，不建议用STL的map，效率奇低。
     </li>
     <li>
      广搜最基础的应用是用来求图的最短路。
     </li>
    </ul>
    <blockquote>
     <p>
      【例题8】给定一个无向连通图，如果两个结点间有直接的边那么权值就是1，求从1号结点出发，到所有点的最短距离。
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/2ec4542b0b5926fca1477e28ff579ef7.png#pic_center"/>
     </p>
     <center>
      图四-1
     </center>
     <p>
     </p>
    </blockquote>
    <ul>
     <li>
      如图四-1所示，对以下图进行广度优先搜索；
     </li>
     <li>
      1）起点为1，将它放入队列后。那么第一次从队列中弹出的一定是1，将和1相邻未被访问的结点继续按顺序放入队列中，分别是2、3、4、5、7，并且记录下它们距离起点的距离dis[x] = dis[1] + 1 (x 属于集合 {2, 3, 4, 5, 7})；
     </li>
     <li>
      2）然后弹出的元素是2，和2相邻未被访问的结点是10，将它也放入队列中，记录dis[10] = dis[2] + 1；
     </li>
     <li>
      3）接着弹出5，放入6（4由于已经被访问过，所以不需要再放入队列中）；
     </li>
     <li>
      4）弹出7，放入8、9。队列为空后结束搜索，搜索完毕后，dis 数组就记录了起点1到各个点的最短距离；
     </li>
    </ul>

    <ul>
     <li>
      1）
      <code>
       dis[u]
      </code>
      数组用来标记从起点到 u 的最短距离，初始化为 -1 代表无限远；
     </li>
     <li>
      2）每次从队列中弹出一个元素，这里队列采用 STL 的动态队列；
     </li>
     <li>
      3）
      <code>
       adj[u][v]=0
      </code>
      代表不连通，则直接跳过；
     </li>
     <li>
      4）
      <code>
       dis[v] != inf
      </code>
      代表 v 这个结点已经访问过，则直接跳过；
     </li>
     <li>
      5）扩展下一个结点，并且放入队列中，等待下次弹出；
     </li>
    </ul>
    <p>
     <strong>
      3）基础应用
     </strong>
    </p>
    <p>
     <strong>
      a. 最短路
     </strong>
    </p>
    <ul>
     <li>
      bellman-ford最短路的优化算法SPFA，主体是利用BFS实现的，会在后面的章节详细介绍。
     </li>
     <li>
      绝大部分四向、八向迷宫的最短路问题。
     </li>
    </ul>
    <p>
     <strong>
      b. 拓扑排序
     </strong>
    </p>
    <ul>
     <li>
      首先找入度为0的点入队，弹出元素执行“减度”操作，继续将减完度后入度为0的点入队，循环操作，直到队列为空，经典BFS操作，会在后面的章节详细介绍。
     </li>
    </ul>
    <p>
     <strong>
      c. FloodFill
     </strong>
    </p>
    <ul>
     <li>
      经典洪水灌溉算法。
     </li>
    </ul>
    <p>
     <strong>
      4）高级应用
     </strong>
    </p>
    <p>
     <strong>
      a. 差分约束
     </strong>
    </p>
    <ul>
     <li>
      数形结合的经典算法，利用SPFA来求解不等式组，会在后面的章节详细介绍。
     </li>
    </ul>
    <p>
     <strong>
      b. 稳定婚姻
     </strong>
    </p>
    <ul>
     <li>
      二分图的稳定匹配问题，试问没有稳定的婚姻，如何有心思学习算法，所以一定要学好BFS啊。
     </li>
    </ul>
    <p>
     <strong>
      c. AC自动机
     </strong>
    </p>
    <ul>
     <li>
      字典树 + KMP + BFS，在设定失败指针的时候需要用到BFS。
     </li>
     <li>
      详细算法参见：http://www.cppblog.com/menjitianya/archive/2014/07/10/207604.html
     </li>
    </ul>
    <p>
     <strong>
      d. 矩阵二分
     </strong>
    </p>
    <ul>
     <li>
      矩阵乘法的状态转移图的构建可以采用BFS，会在后面的章节详细介绍。
     </li>
    </ul>
    <p>
     <strong>
      e. 基于k进制的状态压缩搜索
     </strong>
    </p>
    <ul>
     <li>
      这里的k一般为2的幂，状态压缩就是将原本多维的状态压缩到一个k进制的整数中，便于存储在一个一维数组中，往往可以大大地节省空间，又由于k为2的幂，所以状态转移可以采用位运算进行加速，HDU1813 和 HDU3278 以及 HDU3900 都是很好的例子。
     </li>
    </ul>
    <p>
     <strong>
      f. 基于A*的广度优先搜索
     </strong>
    </p>
    <ul>
     <li>
      在搜索的时候，结点信息要用堆（优先队列）维护大小，即能更快到达目标的结点优先弹出。
     </li>
     <li>
      对于 A*算法，会在后面的章节详细介绍。
     </li>
    </ul>

     </strong>
