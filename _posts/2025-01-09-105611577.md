    <h4>
     <a id="_9">
     </a>
     仓库分几种？
    </h4>
    <ul>
     <li>
      <strong>
       本地仓库
      </strong>
      ：建立在本地的文件夹。
     </li>
     <li>
      <strong>
       远程仓库
      </strong>
      ：建立在互联网的服务器内的文件夹。
     </li>
    </ul>
    <h4>
     <a id="_15">
     </a>
     分布式版本控制系统
    </h4>
    <ol>
     <li>
      配有两个仓库，在你的电脑上有一个
      <strong>
       <code>
        本地仓库
       </code>
      </strong>
      ，在远程的服务器上有一个
      <strong>
       <code>
        远程仓库
       </code>
      </strong>
      。
     </li>
     <li>
      我们在提交文件的时候
      <strong>
       会先提交到
       <code>
        本地仓库
       </code>
      </strong>
      ，然后在有网络的情况下，
      <strong>
       再从
       <code>
        本地仓库
       </code>
       提交到网络上的
       <code>
        远程仓库
       </code>
       。
      </strong>
     </li>
     <li>
      <strong>
       Git 就是一个典型的分布式版本控制系统
      </strong>
     </li>
     <li>
      <strong>
       Github就担任了上述的远程仓库这一角色
      </strong>
      ，就是一个存放在外网服务器上的一个文件夹。并且Github是
      <strong>
       免费的开源的
      </strong>
      托管平台
     </li>
    </ol>
    <h4>
     <a id="Git_25">
     </a>
     什么是Git
    </h4>
    <p>
     <strong>
      <code>
       Git
      </code>
     </strong>
     （读音为/gɪt/）是一个开源的
     <strong>
      分布式版本控制系统
     </strong>
     ，可以有效、高速地处理从很小到非常大的项目版本管理。
    </p>
    <p>
     <strong>
      GitHub是一个面向开源及私有软件项目的托管平台
     </strong>
     ，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3c0f20b01772146724cbdd0ad0bef3e3.png">
      <br/>
      <strong>
       Github常用词含义
      </strong>
     </img>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        watch
       </code>
      </strong>
      ：会持续收到项目的动态
     </li>
     <li>
      <strong>
       <code>
        fork
       </code>
      </strong>
      ：复制某个项目到自己的仓库
     </li>
     <li>
      <strong>
       <code>
        star
       </code>
      </strong>
      ：点赞数，表示对该项目表示认可，点赞数越多的项目一般越火
     </li>
     <li>
      <strong>
       <code>
        clone
       </code>
      </strong>
      ：将项目下载到本地
     </li>
     <li>
      <strong>
       <code>
        follow
       </code>
      </strong>
      ：关注你感兴趣的作者，会收到他们的动态
     </li>
    </ul>
    <h3>
     <a id="_39">
     </a>
     一个完整的项目界面
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/10f2c42df4780f8faad0712d800fe39b.png"/>
    </p>
    <ul>
     <li>
      ①
      <strong>
       此处是项目作者名/项目名
      </strong>
     </li>
     <li>
      ②
      <strong>
       此处是项目的点赞数，和fock数
      </strong>
      ，越火的项目点赞和fock就会越多
     </li>
     <li>
      ③
      <strong>
       项目的
       <code>
        Description
       </code>
       和
       <code>
        Website
       </code>
       和
       <code>
        tags
       </code>
       也就是项目的说明和标签
      </strong>
      ， 通过此处你可以一眼了解该项目的功能和简介
     </li>
     <li>
      ④
      <strong>
       项目的
       <code>
        commits
       </code>
       提交数
      </strong>
      ，一般比较好的项目，维护会比较频繁，更新也会频繁，提交数就会多
     </li>
     <li>
      ⑤
      <strong>
       项目提交时间
      </strong>
      ， 通过这里你可以看到项目的提交时间，防止自己下载了一些远古项目
     </li>
     <li>
      ⑥
      <strong>
       <code>
        README.md
       </code>
      </strong>
      <strong>
       README.md文件是一个项目的入门手册
      </strong>
      ，里面介绍了整个项目的使用、功能等等。所以README文件写得好不好，关系到这个项目能不能更容易的被其他人了解和使用。
     </li>
    </ul>
    <h3>
     <a id="Github_50">
     </a>
     使用Github搜索项目
    </h3>
    <p>
     <strong>
      一般人
     </strong>
     用Github的步骤 直接搜索，
     <strong>
      选择一下
      <code>
       Languages
      </code>
      设置下
      <code>
       项目排序顺序
      </code>
     </strong>
     就直接下载
    </p>
    <p>
     然后就是克隆仓库，阅读md，看项目源代码，看不懂，关闭项目，删除。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6cd0b4da827a1fe2cf536248f29e7744.png">
      <br/>
      这样是很难找到真正适合自己的项目的，
     </img>
    </p>
    <p>
     GitHub里面有很多有价值的开源项目和代码，如何在海量的代码库中搜索我们需要的信息，那么接下来将带你了解下如何利用GitHub强大的搜索功能，来找到适合自己的项目
    </p>

     <a id="GitHub_61">
     </a>
     GitHub的高级搜索

    <p>
     <strong>
      GitHub有高级搜索功能，search/advanced可以输入
      <code>
       关键字
      </code>
      、
      <code>
       代码库大小
      </code>
      、包含
      <code>
       作者
      </code>
      、
      <code>
       代码语
      </code>
      、
      <code>
       代码包含后缀文件名
      </code>
      等。
     </strong>
     <br/>
     <img alt="Github搜索技巧-找开发者" src="https://i-blog.csdnimg.cn/blog_migrate/082b33154585e3d045ced715f35a372f.png"/>
    </p>
    <p>
     <img alt="Github搜索技巧-找项目" src="https://i-blog.csdnimg.cn/blog_migrate/e1a4c354e5459759fcac28f128aef1a9.png"/>
    </p>
    <p>
     这里我们假设正要学习 Spring Boot，要找一个 Spring Boot的 Demo 来进行参考学习。
    </p>
    <h3>
     <a id="README_71">
     </a>
     精准搜索仓库标题、仓库描述、README
    </h3>
    <p>
     <mark>
      <strong>
       in关键词限制搜索范围
      </strong>
     </mark>
    </p>
    <p>
     <strong>
      按照项目名/仓库名搜索（大小写不敏感）
     </strong>
     <br/>
     （1）
     <strong>
      公式
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         in:name xxx
        </code>
       </strong>
       项目名包含xxx
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         in:description xxx
        </code>
       </strong>
       项目描述包含xxx
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         in:readme xxx
        </code>
       </strong>
       项目介绍文档里含有xxx
      </p>
      <br/>
      <br/>
     </li>
    </ul>
    <p>
     比如我搜索
     <strong>
      项目名里含有 Spring Boot 的
     </strong>
     <strong>
      <code>
       in:name Spring Boot
      </code>
     </strong>
     <br/>
     会发现项目数量由17W变成了11W
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/66287f7a86ecd5b773f118d6f40350b8.png"/>
    </p>
    <p>
     搜索
     <strong>
      项目描述里含有 Spring Boot 的
     </strong>
     <strong>
      <code>
       in:description Spring Boot
      </code>
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/45ff2ddf37779fb4eb3ef4f8f855686f.png"/>
    </p>
    <h4>
     <a id="starsfork_99">
     </a>
     <strong>
      stars或fork数量去查找
     </strong>
    </h4>
    <p>
     一个项目 star 数的多少，一般代表该项目的受欢迎程度 越受欢迎的项目，star数和fork数一定也不会少
    </p>
    <p>
     （1）公式
    </p>
    <ul>
     <li>
      <strong>
       <code>
        stars:&gt;xxx
       </code>
      </strong>
      <strong>
       stars数大于xxx
      </strong>
     </li>
     <li>
      <strong>
       <code>
        stars:xx..xx
       </code>
      </strong>
      <strong>
       stars数在xx…xx之间
      </strong>
     </li>
     <li>
      <strong>
       <code>
        forks:&gt;xxx
       </code>
      </strong>
      <strong>
       forks数大于xxx
      </strong>
     </li>
     <li>
      <strong>
       <code>
        forks:xx..xx
       </code>
      </strong>
      <strong>
       forks数在xx…xx之间
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-c"> 查找star数大于等于<span class="token number">5000</span>的springboot项目
     spring boot stars<span class="token punctuation">:</span><span class="token operator">&gt;=</span><span class="token number">5000</span>
 查找fork数大于<span class="token number">500</span>的springcloud项目
     spring cloud forks<span class="token punctuation">:</span><span class="token operator">&gt;</span><span class="token number">500</span>
 查找fork在<span class="token number">100</span>到<span class="token number">200</span>之间并且stars数在<span class="token number">80</span>到<span class="token number">100</span>之间的springboot项目
     spring boot forks<span class="token punctuation">:</span><span class="token number">100.</span><span class="token number">.200</span> stars<span class="token punctuation">:</span><span class="token number">80.</span><span class="token number">.100</span>
</code></pre>
    <p>
     <strong>
      我们进一步缩小范围，Star数量过滤，要求Star数量大于3000
     </strong>
     <br/>
     <strong>
      <code>
       in:name spring boot starts :&gt; 3000
      </code>
     </strong>
    </p>
    <p>
     可以看到只有一千多个项目供我们选择了
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4daa5ca1f804c580c41f3aeee03190db.png"/>
    </p>
    <h4>
     <a id="_128">
     </a>
     按照地区和语言进行搜索
    </h4>
    <p>
     <strong>
      很多时候我们的项目是要用我们会的语言，你找到了一个
      <code>
       Python
      </code>
      写的好项目，但是没学过
      <code>
       Python
      </code>
      ，下载了也看不懂，同时，
      <code>
       为了更好的阅读README.md帮助文档以及项目注释
      </code>
      ，我想很多同学都会想要下载中文的项目，当然英语顶呱呱的请忽略
     </strong>
     <br/>
     （1）公式
    </p>
    <ul>
     <li>
      <strong>
       <code>
        location
       </code>
      </strong>
      ：
      <strong>
       地区
      </strong>
     </li>
     <li>
      <strong>
       <code>
        language
       </code>
      </strong>
      ：
      <strong>
       语言
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-java">语言为javaScript   
language<span class="token operator">:</span>javaScript   
地区为china
`location<span class="token operator">:</span> China
</code></pre>
    <p>
     如果你要寻找使用
     <strong>
      <code>
       javascript
      </code>
     </strong>
     语言的国产项目，整个搜索条件就是：
     <strong>
      <code>
       language:javascript location:china
      </code>
     </strong>
     ，从搜索结果来看，我们找到了五百多万javascript 项目，近 21000 多名地区信息填写为 China 的 javascript 开发者，
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2280cf61c237540c9521c95590858d2c.png"/>
    </p>
    <h3>
     <a id="_144">
     </a>
     根据仓库大小搜索
    </h3>
    <p>
     <strong>
      如果你只是想找一些小型的项目进行个人学习和开发，不想找特别复杂的，那么使用size关键字查找简单的 Demo，就成了你的首选
     </strong>
    </p>
    <p>
     （1）公式
    </p>
    <ul>
     <li>
      <strong>
       <code>
        size:&gt;= 数字
       </code>
      </strong>
      【注意：100代表100Kb 单位为Kb】
     </li>
    </ul>
    <h3>
     <a id="_152">
     </a>
     根据仓库是否在更新的搜索
    </h3>
    <p>
     寻找项目当然是想要找到最新的项目，而不是好久都没有更新的老项目了，
    </p>
    <p>
     （1）公式
    </p>
    <ul>
     <li>
      <strong>
       <code>
        pushed:&gt; YYYY-MM-DD
       </code>
      </strong>
      <strong>
       最后上传日期大于YYYY-MM-DD
      </strong>
     </li>
     <li>
      <strong>
       <code>
        created:&gt; YYYY-MM-DD
       </code>
      </strong>
      <strong>
       创建日期大于YYYY-MM-DD
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      比如我们想要寻找2020年最新更新的项目
     </strong>
     ，可以用
     <strong>
      <code>
       pushed:&gt;2020-01-03 Spring Boot
      </code>
     </strong>
     ,这样子就可以找到今年一月份之后更新的最新项目
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cbbcef3c6356307f39a7ebde0f153499.png"/>
    </p>
    <h3>
     <a id="_165">
     </a>
     根据某个人或组织进行搜索
    </h3>
    <p>
     如果你想在GitHub 上找一下某个大神是不是提交了新的项目，可以对他们进行精准搜索
     <br/>
     （1）公式
    </p>
    <ul>
     <li>
      <strong>
       <code>
        user: name
       </code>
      </strong>
      <strong>
       查找某个用户
      </strong>
     </li>
     <li>
      <strong>
       <code>
        org: name
       </code>
      </strong>
      <strong>
       查找某个组织
      </strong>
     </li>
     <li>
      <strong>
       <code>
        followers:&gt;=xxx
       </code>
      </strong>
      <strong>
       查找关注者数量超过xxx的开发者
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      比方说我们想要找一下廖雪峰老师的python开源项目
     </strong>
     <br/>
     <strong>
      <code>
       user:MichaelLiao language:python
      </code>
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/24ad3c36d9a283323127759d37af2c97.png"/>
    </p>
    <h4>
     <a id="LICENSE_177">
     </a>
     根据仓库的LICENSE搜索
    </h4>
    <p>
     <strong>
      License是很多人容易忽略的一个问题
     </strong>
    </p>
    <p>
     <strong>
      开源项目的License（项目授权协议） 有的开源项目作者明确禁止商用了，但是你不知情下载了，并且使用了，这就会很麻烦，
     </strong>
    </p>
    <p>
     “非常友好”的协议，比较出名的有这几种：BSD、MPL（Mozilla）、Apache、MIT。这些协议不但允许项目的使用者使用开源库，有些还允许对开源库进行修改并重新分发。因此用起来特别爽。上述这几个协议在细节上有些小差异，大伙儿可以去它们官网瞧一下。
    </p>
    <p>
     以下这个网站，详细介绍了各个License的区别。
     <br/>
     <a href="http://choosealicense.com/licenses/" rel="nofollow">
      http://choosealicense.com/licenses/
     </a>
     <br/>
     （1）公式
    </p>
    <p>
     -
     <strong>
