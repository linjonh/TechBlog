    <strong>
     函数式编程
    </strong>
    是种
    <strong>
     编程典范
    </strong>
    ，它将电脑运算视为函数的计算。函数编程语言最重要的基础是 λ 演算（lambda calculus）。而且λ演算的函数可以接受函数当作输入（引数）和输出（传出值）。
    <br/>
    <div class="spctrl">
    </div>
    和
    <u>
     指令式编程
    </u>
    相比，函数式编程强调函数的计算比指令的执行重要。
    <br/>
    <div class="spctrl">
    </div>
    和
    <u>
     过程化编程
    </u>
    相比，函数式编程里，函数的计算可随时调用。
    <br/>
    <div class="spctrl">
    </div>
    <strong>
     <font color="#000000">
      历史
      <br/>
     </font>
    </strong>
    <div class="spctrl">
    </div>
    虽然 λ 演算并非设计来于计算机上执行，但可视为第一个函数式编程语言。1980年代末期，Haskell发布，企图集合很多函数式编程研究里的想法。
    <br/>
    <div class="spctrl">
    </div>
    速度和空间上的顾虑
    <br/>
    <div class="spctrl">
    </div>
    函数式编程常被认为严重耗费在CPU和内存资源。主因有二：
    <br/>
    <div class="spctrl">
    </div>
    早期的函数式编程语言实现时并无考虑过效率问题。
    <br/>
    <div class="spctrl">
    </div>
    非函数式编程语言为求提升速度，会在某些部分放弃边界检查或垃圾回收等功能。
    <br/>
    <div class="spctrl">
    </div>
    缓式评估亦为语言如Haskell增加了额外的管理工作。
    <br/>
    <div class="spctrl">
    </div>
    <u>
     函数式编程语言
    </u>
    <br/>
    <div class="spctrl">
    </div>
    纯函数式的程式没有变量和副作用。
    <br/>
    <div class="spctrl">
    </div>
    函数式编程经常使用递归。
    <br/>
    <div class="spctrl">
    </div>
    <u>
     纯函数式编程语言
    </u>
    <br/>
    <div class="spctrl">
    </div>
    因为纯函数式程式设计语言没有变量，函数没有副作用，编写出的程式可以利用memoization、common subexpression elimination和平行计算在运行时和编译时得到大量优化。
    <br/>
    <div class="spctrl">
    </div>
    <strong>
     例子
    </strong>
    <br/>
    <div class="spctrl">
    </div>
    函数式编程中最古老的例子莫过于1958年被创造出来的LISP了，透过 LISP，可以用精简的人力。较现代的例子包括Haskell、Clean、Erlang和Miranda等。
