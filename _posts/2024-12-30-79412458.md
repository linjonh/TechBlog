    <div style="white-space:pre-wrap;line-height:2.25;text-align:left;font-size:14px;">
     <span style="font-size:16px;">
      Mongo
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      是一个高性能，开源，面向集合的无模式文档型数据库，它在许多场景下可用于替代传统的关系型数据库或键
     </span>
     <span style="font-size:16px;">
      /
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">

     </span>
     <span style="font-size:16px;">
      Mongo
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      使用
     </span>
     <span style="font-size:16px;">
      C++
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      开发。面向集合
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      意思是数据被分组到若干集合
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      这些集合称作聚集
     </span>
     <span style="font-size:16px;">
      (collections).
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      在数据库里每个聚集有一个唯一的名字
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      可以包含无限个文档
     </span>
     <span style="font-size:16px;">
      .
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      聚集是
     </span>
     <span style="font-size:16px;">
      RDBMS
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      中表的同义词
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      区别是聚集不需要进行模式定义
     </span>
     <span style="font-size:16px;">
      .
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      模式自由
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      意思是数据库并不需要知道你将存入到聚集中的文档的任何结构信息
     </span>
     <span style="font-size:16px;">
      .
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      实际上
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      你可以在同一个聚集中存储不同结构的文档
     </span>
     <span style="font-size:16px;">
      .
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      文档型
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      意思是我们存储的数据是键
     </span>
     <span style="font-size:16px;">
      -
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      值对的集合
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      键是字符串
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      值可以是数据类型集合里的任意类型
     </span>
     <span style="font-size:16px;">
      ,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      包括数组和文档
     </span>
     <span style="font-size:16px;">
      .
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;">
      我们把这个数据格式称作
     </span>
     <span style="font-size:16px;">
      “[BSON]”
     </span>

     <span style="color:rgb(51,51,51);">
      最
     </span>
     早是由美国加州大学伯克利分校开发的，开始只是作为一个演示系统发表，但是随着时间的推移，逐步分发，得到很多实际的应用，才逐步流行起来。如果很粗略地观察PostgreSQL，会觉得这个数据库系统和其它数据库很类似。因为 PostgreSQL 使用 SQL 语言来在执行资料的查询。这些资料通过连外键联系在一起，以一系列表格的形式存在。PostgreSQL 相对于竞争者的主要优势，主要的特征为可编程性：对于使用数据库资料的实际应用，PostgreSQL 让开发与使用的工作，变得更加容易。SQL 数据在"平面表格"中存储简单的数据类型，需要用户使用查询把有关的信息收集在一起。这与应用和用户利用数据自身的方式相对立，典型的使用带有丰富数据类型的高级语言，在其中所有有关的数据作为它自己的一个完整单元来操作。典型的称呼为记录或对象（依据各自语言）。转换来自 SQL 世界的信息到面向对象编程世界体现得很困难，因为两者有非常不同的数据组织的模型。工业界把这个问题称为阻抗不匹配：从一个模型映射到另一个要花费项目开发者 40% 的时间。一些映射解决方案，典型的称为对象-关系映射，致力于这个问题，但是它们花费很多并有自身的问题，导致糟糕的性能或强制所有的数据访问通过映射所支持的一种语言来进行。PostgreSQL 可以直接在数据库中解决很多这种问题。PostgreSQL 允许用户定义基于正规的 SQL 类型的新类型，允许数据库自身理解复杂数据。例如，你可以定义一个 address 来组合一些事物如街道编号、城市和国度的字符串。从这一点上你可以轻易的建立把保存地址的所需要的所有字段包含在一个单一行列中的表。

     4.SQLite
    </span>
    <br/>
    <div style="white-space:pre-wrap;line-height:2.25;text-align:left;font-size:14px;">
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      是
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      D. Richard Hipp
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      用
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      C
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      语言编写的开源嵌入式数据库引擎。它是完全独立的，不具有外部依赖性。它是作为
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      PHP V4.3
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      中的一个选项引入的，构建在
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      PHP V5
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      中。
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      支持多数
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQL92
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      标准，可以在所有主要的操作系统上运行，并且支持大多数计算机语言。
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      还非常健壮。其创建者保守地估计
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      可以处理每天负担多达
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      100,00
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      次点击率的
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Web
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      站点，并且
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      有时候可以处理
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      10
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      倍于上述数字的负载。
     </span>
    </div>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite的技术特点：
     </span>
    </div>
    <div style="white-space:pre-wrap;line-height:2.25;text-align:left;font-size:14px;">
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      对
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQL92
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      标准的支持包括索引、限制、触发和查看。
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      不支持外键限制，但支持原子的、一致的、独立和持久
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      (ACID)
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      的事务（后面会提供有关
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      ACID
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      的更多信息）。这意味着事务是原子的，因为它们要么完全执行，要么根本不执行。事务也是一致的，因为在不一致的状态中，该数据库从未被保留。事务还是独立的，所以，如果在同一时间在同一数据库上有两个执行操作的事务，那么这两个事务是互不干扰的。而且事务是持久性的，所以，该数据库能够在崩溃和断电时幸免于难，不会丢失数据或损坏。
     </span>
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      通过数据库级上的独占性和共享锁定来实现独立事务处理。这意味着当多个进程和线程可以在同一时间从同一数据库读取数据，但只有一个可以写入数据。在某个进程或线程向数据库执行写入操作之前，必须获得独占锁定。在发出独占锁定后，其他的读或写操作将不会再发生。
     </span>
    </div>
    <div style="white-space:pre-wrap;line-height:2.25;text-align:left;font-size:14px;">
     <span style="font-family:Arial;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      官方主页：
     </span>
     <a href="http://www.sqlite.org/" rel="nofollow">

     </a>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">

     5.Berkeley DB
    </span>
    <br/>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      像MySQL这类基于C/S结构的关系型数据库系统虽然代表着目前数据库应用的主流，但却并不能满足所有应用场合的需要。有时我们需要的可能只是一个简单的基于磁盘文件的数据库系统。这样不仅可以避免安装庞大的数据库服务器，而且还可以简化数据库应用程序的设计。Berkeley DB正是基于这样的思想提出来的。
     </span>
    </div>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Berkeley DB的技术特点：
     </span>
    </div>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Berkeley DB是一个开放源代码的内嵌式数据库管理系统，能够为应用程序提供高性能的数据管理服务。应用它程序员只需要调用一些简单的API就可以完成对数据的访问和管理。与常用的数据库管理系统（如MySQL和Oracle等）有所不同，在Berkeley DB中并没有数据库服务器的概念。应用程序不需要事先同数据库服务建立起网络连接，而是通过内嵌在程序中的Berkeley DB函数库来完成对数据的保存、查询、修改和删除等操作。
     </span>
    </div>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Berkeley DB为许多编程语言提供了实用的API接口，包括C、C++、Java、Perl、Tcl、Python和PHP等。所有同数据库相关的操作都由 Berkeley DB函数库负责统一完成。这样无论是系统中的多个进程，或者是相同进程中的多个线程，都可以在同一时间调用访问数据库的函数。而底层的数据加锁、事务日志 和存储管理等都在Berkeley DB函数库中实现。它们对应用程序来讲是完全透明的。俗话说：“麻雀虽小五脏俱全。”Berkeley DB函数库本身虽然只有300KB左右，但却能够用来管理多达256TB的数据，并且在许多方面的性能还能够同商业级的数据库系统相抗衡。就拿对数据的并发操作来说，Berkeley DB能够很轻松地应付几千个用户同时访问同一个数据库的情况。此外，如果想在资源受限的嵌入式系统上进行数据库管理，Berkeley DB可能就是惟一正确的选择了。
     </span>
    </div>
    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Berkeley DB作为一种嵌入式数据库系统在许多方面有着独特的优势。首先，由于其应用程序和数据库管理系统运行在相同的进程空间当中，进行数据操作时可以避免繁琐的 进程间通信，因此耗费在通信上的开销自然也就降低到了极低程度。其次，Berkeley DB使用简单的函数调用接口来完成所有的数据库操作，而不是在数据库系统中经常用到的SQL语言。这样就避免了对结构化查询语言进行解析和处理所需的开销。
     </span>
    </div>

          <span style="font-size:13px;font-family:'Microsoft YaHei', STXihei;color:rgb(0,0,0);background-color:transparent;">
           Oracle
          </span>
          <span style="font-size:13px;font-family:SimSun, STSong;color:rgb(0,0,0);background-color:transparent;">
           公司收购，继续提升
          </span>
          <span style="font-size:13px;font-family:'Microsoft YaHei', STXihei;color:rgb(0,0,0);background-color:transparent;">
           Berkeley DB
          </span>
          <span style="font-size:13px;font-family:SimSun, STSong;color:rgb(0,0,0);background-color:transparent;">

          </span>
         </div>

      <strong>
      </strong>
     </span>

    <div style="white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      <strong>
       总结：
      </strong>
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      我只列举了这几种主流开源数据库，对于
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      IBM
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      提供的
     </span>
     <a href="http://blog.csdn.net/wangji163163/article/details/3895401">
      <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
       DB2 Express-C
      </span>
      <span style="font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
       、
      </span>
      <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
       ORACLE
      </span>
      <span style="font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
       提供的
      </span>
      <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
       Oracle XE
      </span>
     </a>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      免费数据库功能上还是受到一些限制，在这里没有列举，
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      mysql
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      不完全免费也没有说明。可视化选型一种开源数据库，我个人更倾向于
     </span>
     <span style="background-color:rgb(255,255,255);">
      PostgreSQL
     </span>
     <span style="font-family:SimSun, STSong;background-color:rgb(255,255,255);">
      和
     </span>
     <span style="font-size:16px;background-color:rgb(255,255,255);">
      Mongodb,
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;background-color:rgb(255,255,255);">
      这是因为
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      PostgreSQL
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      功能强大支持可编程性，支持很多数据类型，还可以自定义数据类型存储
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      GIS
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      图元，支持大数据，虽然它的连接采用进程因而速度比不上
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Mysql
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      快，但是稳定性比
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Mysql
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      好。
     </span>
     <span style="background-color:rgb(255,255,255);">
      PostgreSQL
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      网上资料很多，社区也比较活跃。
     </span>
     <span style="font-size:16px;background-color:rgb(255,255,255);">
      Mongodb
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;background-color:rgb(255,255,255);">
      面向集合，无模式的，这种主要可以做到
     </span>
     <span style="font-size:16px;background-color:rgb(255,255,255);">
      NOSQL
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;background-color:rgb(255,255,255);">
      ，集合就相当于关系数据库中的表。
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      适合实时数据存储，也可用到可视化中。
     </span>
     <span style="font-size:16px;background-color:rgb(255,255,255);">
      Firebird
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;background-color:rgb(255,255,255);">
      则更适合做嵌入式的数据库，网上的资料不是很多。
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      SQLite
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      是嵌入式数据库引擎，更适合做
     </span>
     <span style="font-size:16px;background-color:rgb(255,255,255);">
      CMS
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      (
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      内容管理系统
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      )
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      。
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Berkeley DB
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      现在属于
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      Oracle
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      旗下，
     </span>
     <span style="color:rgb(51,51,51);background-color:rgb(255,255,255);">
      2.0
     </span>
     <span style="font-size:16px;font-family:SimSun, STSong;color:rgb(51,51,51);background-color:rgb(255,255,255);">
      版本以上就不能自由商业应用，而且它的使用不需要数据库连接，数据库操作都是调用它的函数库，不太适合集成到可视化。
