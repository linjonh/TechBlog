    <p>

    <p>
     <mark>
      前后端分离项目，后端设置的允许跨域的代码，到前端调试界面的时候还显示了不允许跨域。
     </mark>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b587beb4080877753867ea839da497d4.png"/>
    </p>
    <h3>
     <a id="_4">
     </a>
     错误信息如下：
    </h3>
    <p>
     <code>
      Access to XMLHttpRequest at
     </code>
     ‘http://localhost:8000/carInfo/carCondition/1/4’
     <code>
      from origin
     </code>
     ‘http://localhost:9528’
     <code>
      has been blocked by CORS policy: Response to preflight request doesn't pass access control check: The value of the 'Access-Control-Allow-Origin' header in the response must not be the wildcard '*' when the request's credentials mode is 'include'. The credentials mode of requests initiated by the XMLHttpRequest is controlled by the withCredentials attribute.
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/343ae311e38c7446c43bc65cca61a984.png"/>
    </p>
    <h3>
     <a id="_7">
     </a>
     解决方案：
    </h3>
    <pre><code class="prism language-html">withCredentials: false
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/80bc2847f148d5818da6c493796a5ab4.png"/>
    </p>
    <h3>
     <a id="_13">
     </a>
     跨域问题解决：
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d0a1320dd4758f3acde37a18056e062a.png"/>
    </p>
    <h3>
     <a id="_16">
     </a>
     原因：
    </h3>
    <p>
     <code>
      CORS
     </code>
     请求发出时，已经设定了credentials，但服务端配置了http响应首部
     <code>
      Access-Control-Allow-Origin
     </code>
     的值为通配符 ("*") ，而这与使用credentials相悖。
     <br/>
     要在客户端改正这个问题，只需要确保发出 CORS 请求时将credential设置为false。
    </p>
    <ul>
     <li>
      如果使用
      <code>
       XMLHttpRequest
      </code>
      发出请求，确保未将
      <code>
       withCredentials
      </code>
      设置为 true。
     </li>
     <li>
      如果使用
      <code>
       Server-sent events
      </code>
      , 确保
      <code>
       EventSource.withCredentials
      </code>
      的值为 false
      <br/>
      (false为默认值)。
     </li>
     <li>
      如果使用
      <code>
       Fetch API
      </code>
      ，确保
      <code>
       Request.credentials
      </code>
      的值为 “omit”.
      <br/>
      如果还不成功，则需要修改服务端，可能需要修改 Access-Control-Allow-Origin 的值，来为客户端所能够加载资源的源予以授权。
     </li>
    </ul>
