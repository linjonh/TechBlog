   <div class="htmledit_views" id="content_views">
    <p>
     通过域名来访问后台服务：
    </p>
    <h5>
     1、单个服务
    </h5>
    <p>
     如果你的controller里面是这样写的
    </p>
    <pre><code class="language-java">@RestController
@CrossOrigin({"*"})
@RequestMapping({"/train"})
public class TrainController {
//
}</code></pre>
    <p>
     那么需要在nginx里面这样配置（假设你的服务在本机8888端口上跑着）：
    </p>
    <pre><code class="language-XML">server {
        listen       80;
        server_name  lab001.com;
		
		location / {
           proxy_set_header Host $host;
           proxy_pass http://127.0.0.1:8888;
      }
}</code></pre>
    <p>
     lab001.com可以在host文件里面配置，：127.0.0.01  lab001.com
    </p>
    <p>
     然后你可以这样访问了：http://lab001.com/train
    </p>
    <p>
     (替代了原来的http://127.0.0.1:8888/train)
    </p>
    <h5>
     2、多个服务
    </h5>
    <p>
     假设有多个后端服务：一个在8888端口，一个在8887端口；
    </p>
    <p>
     那么就需要在nginx中配置多个服务；
    </p>
    <pre><code class="language-XML">server {
        listen       80;
        server_name  lab001.com;
		
		location /project1 {
           proxy_set_header Host $host;
           proxy_pass http://127.0.0.1:8887/;
      }

      location /project2 {
           proxy_set_header Host $host;
           proxy_pass http://127.0.0.1:8888/;
      }

}</code></pre>
<p>
<strong>
<span style="color:#fe2c24;">
切记需要在：
<a href="http://127.0.0.1:8887" rel="nofollow" title="http://127.0.0.1:8887">
http://127.0.0.1:8887
</a>
后加 /
</span>
</strong>
</p>
