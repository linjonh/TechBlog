    <h2>
     <a id="_1">

     体系结构
    </h2>
    <h3>
     <a id="1_2">
     </a>
     1、计算机网络的定义
    </h3>
    <p>
     计算机网络是通过通信设施，将地理上分布的、具有自治功能的多个计算机系统互连起来，实现信息交换、资源共享、互操作和协同处理的系统。特点：资源共享、透明操作。
     <strong>
      从逻辑上，计算机网络由通信子网和资源子网构成。
     </strong>
    </p>
    <h3>
     <a id="2OSITCPIP_4">
     </a>
     2、OSI参考模型、TCP/IP四层体系
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2540aa7654a08e2ba9896b727b1820a5.png">
      <br/>
      1）物理层：
      <strong>
       透明传输比特流。物理层特性：机械特性、电气特性、功能特性、规程特性。
      </strong>
     </img>
    </p>
    <p>
     2）数据链路层：
     <strong>
      定义了如何让数据格式化进行传输，以及如何控制对物理介质的访问。这一层通常还提供了错误检测和纠正，以保证数据的可靠传输。
     </strong>
    </p>
    <p>
     3）网络层：位于不同地理位置的网络中的
     <strong>
      两个主机之间提供连接和路径选择
     </strong>
     。具体包括：建立/拆除网络连接、路径选择、分段和组块、差错检测和恢复等等。
    </p>
    <p>
     4）传输层：
     <strong>
      在网络层传输基础上，实现端到端传输
     </strong>
     。具体包括传输层地址&lt;-&gt;网络地址、多路复用、传输连接的建立和释放等等。
    </p>
    <p>
     5）会话层：
     <strong>
      提供一个面向用户的连接服务
     </strong>
     。会话：两个用户进程为完成一次完整的通信而建立会话连接。
    </p>
    <p>
     6）表示层：可以确保一个系统的应用层所发送的信息可以被另一个系统的应用层读取。
    </p>
    <p>
     7）应用层：作为用户进程使用OSI功能的唯一窗口。
    </p>

     1）单工通信
     <br/>

     <br/>
     3）全双工通信
    </p>
    <h3>
     <a id="2_29">
     </a>
     2、信道的最大信息传输速率（香农公式）
    </h3>
    <p>
     <strong>
      C=W log2(1+S/N)（bit/s)
     </strong>
     <br/>
     其中，W为信道带宽，单位HZ；S/N为信噪比，S是信道内所传信号的平均功率（瓦），N是信道内部的高斯噪声功率（瓦）。
     <br/>
     <strong>
      注意：香农公式中的S/N是为信号与噪声的功率之比，为无量纲单位。但是，当讨论信噪比时，常以分贝（dB）为单位。即SNR（信噪比，单位为dB)=10 lg（S/N）
     </strong>
    </p>
    <pre><code>例题：信道带宽为3KHz，信噪比为30dB，则最大数据速率为多少？
解：	∵信噪比为30dB；
        ∴30 = 10 lg （S/N）
        故信噪比S/N = 1000，代入香农公式有：
       C = W log2(1+S/N) = 3000 * log 2 （1+ 1000） = 30000b/s
答：信道带宽为3000Hz，信噪比为30dB，最大数据速率为30kb/s。
</code></pre>
    <h3>
     <a id="3_42">
     </a>
     3、差错率衡量信道可靠性
    </h3>
    <p>
     1）误码率：在一定时间内接收到的出错的比特数与总的传输比特数之比。
     <br/>
     2）误组率：在一定时间内接收到的出错的组数与总的传输组数之比。
    </p>

    <h2>
     <a id="_49">
     </a>
     数据链路层
    </h2>
    <h3>
     <a id="1_50">
     </a>
     1、数据电路与数据链路
    </h3>
    <p>
     <strong>
      数据电路：在传输信道两端加上信号变换设备之后形成的二进制比特流通路。即数据电路由传输信道加DCE组成，是一条物理链路。
     </strong>
    </p>
    <p>
     <strong>
      数据链路：在数据电路基础上增加传输控制功能，是一条逻辑链路。只有建立起数据链路，才能真正实现数据通信。
     </strong>
    </p>
    <h3>
     <a id="2_55">
     </a>
     2、数据链路层的目标和功能：
    </h3>
    <p>
     目标：在数据链路上提供可靠的信息传输。
     <br/>
     功能：数据链路连接的建立和释放、帧同步、流量控制、差错控制等等。
    </p>
    <h3>
     <a id="3_58">
     </a>
     3、流量控制
    </h3>
    <p>
     1）定义：在数据通信中，要求发送方的数据传输速率不能超过接收方的接收和处理速率。因此要采取一些措施控制发送方的发送速率，故
     <strong>
      流量控制由接收方主动控制和实现
     </strong>
     。
    </p>
    <p>
     2）流量控制方法：
     <br/>
     1、开关式的流量控制：硬件实现
     <br/>
     2、协议式的流量控制：ARQ自动重发请求。具体包含停止-等待协议、滑动窗口协议（连续ARQ、选择ARQ）
    </p>
    <h4>
     <a id="31_65">
     </a>
     3.1、实用的停止-等待协议
    </h4>
    <p>
     <strong>
      停止等待就是每发送完一个分组就停止发送，等待对方的确认，在收到确认后再发送下一个分组，是最简单的自动重传请求方法。。
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/87d9c7963c944e47b87b0f06e394a9d5.png">
      <br/>
      <strong>
       核心技术：
      </strong>
      <br/>
      1）、
      <strong>
       超时重发技术
      </strong>
      ：解决数据帧丢失、ACK确认帧丢失等问题。
      <br/>
      2）、
      <strong>
       帧编号
      </strong>
      ：解决了重复帧问题。
     </img>
    </p>
    <p>
     <strong>
      缺点：信道利用率不高，发送方只有在收到ACK确认后才能继续发送数据帧。
     </strong>
    </p>
    <pre><code>例题：用户A、B通过卫星链路通信，假设传播时延为200ms，数据的传输
速率为50Kb/s，数据帧大小为5000bit。若采用停止-等待流量控制协议
进行通信，则最大链路利用率为（B）
A、0.25      B、0.2   C、0.333   D、0.5
</code></pre>
    <h4>
     <a id="32_80">
     </a>
     3.2、滑动窗口协议
    </h4>
    <p>
     <strong>
      当发送完一个数据帧后，不是停下来等待ACK确认，而是继续发送若干个数据帧。由于在等待确认时间内可以继续发送数据帧，减少了信道空闲时间，提高了整个通信过程的吞吐量。
     </strong>
    </p>
    <h5>
     <a id="321ARQ_82">
     </a>
     3.2.1、连续重传ARQ协议
    </h5>
    <p>
     利用滑动窗口机制，当出现差错时，则从出现差错的数据帧开始
     <strong>
      全部连续重发
     </strong>
     。若采用n比特对窗口编码，则
     <br/>
     <strong>
      最大发送窗口WT = 2^n-1；接收窗口WR = 1。
     </strong>
    </p>
    <pre><code>例题：若采用连续ARQ协议通信，采用3比特对敞口编码，则祖达发送窗
口可达到（C）
A、1      B、3      C、7     D、8
</code></pre>
    <h5>
     <a id="322ARQ_92">
     </a>
     3.2.2、选择重传ARQ协议
    </h5>
    <p>
     利用滑动窗口机制，
     <strong>
      不同于连续ARQ，为进一步提高信道利用率，只重传出错的数据帧即可。
     </strong>
     <br/>
     最大发送窗口WT = 最大接收窗口WR = 2^n / 2
    </p>
    <h3>
     <a id="4_95">
     </a>
     4、差错控制
    </h3>
    <h4>
     <a id="41_96">
     </a>
     4.1、差错控制的目的
    </h4>
    <p>

    </h3>
    <h4>
     <a id="41_230">
     </a>
     4.1、基本概念
    </h4>
    <p>
     路由(route)就是分组在因特网上从一台计算机传输到另一台计算机的实际路径，主要功能是实现
     <strong>
      跨网络通信
     </strong>
     。
    </p>
    <p>
     <strong>
      路由器是一种典型的网络连接设备，用来进行路由选择和报文转发
     </strong>
     。每个路由器都存储有一张表，称为路由表(routing table)，路由器正是依据路由表的内容将各个 IP 分组转发到正确的去处。
    </p>
    <p>
     路由协议是路由器之间维护路由表的规则，用于发现路由，生成路由表，并指导报文转发。依据来源的不同，路由可以分为三类：
     <br/>
     1、通过链路层协议发现的路由称为直连路由。
    </p>
    <p>
     2、通过网络管理员手动配置的路由称为静态路由。
    </p>
    <p>
     3、通过动态路由协议发现的路由称为动态路由（重点）。
    </p>
    <p>
     根据作用范围不同，动态路由协议可以分为两类：
     <br/>
     1、内部网关协议IGP（Interior Gateway Protocol）：
     <br/>
     在一个自治系统内部运行。常见的IGP协议包括RIP、OSPF和IS-IS。
    </p>
    <p>
     2、外部网关协议EGP（Exterior Gateway Protocol）：
     <br/>
     运行于不同自治系统之间。BGP是目前最常用的EGP协议。
    </p>
    <p>
     根据使用算法不同，路由协议可分为：
     <br/>
     1、距离矢量协议（Distance-Vector Protocol）：
     <br/>
     包括RIP和BGP。其中，BGP也被称为路径矢量协议（Path-Vector Protocol）。
    </p>
    <p>
     2、链路状态协议（Link-State Protocol）：
     <br/>
     包括OSPF和IS-IS。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b0c1db1d49eb3a8a4d5f5f2de4c7066d.png"/>

    </p>
    <p>
     ps、
     <strong>
      自治系统AS
     </strong>
     ：
     <br/>
     在互联网中，一个自治系统(AS)就是一个
     <strong>
      有权自主地决定在本系统中应采用何种路由协议的小型单位
     </strong>
     。这个网络单位可以是一个简单的网络也可以是一个由一个或多个普通的网络管理员来控制的网络群体，它是一个单独的可管理的网络单元（例如一所大学，一个企业或者一个公司个体）。
     <strong>

    <h4>
     <a id="42RIP_265">
     </a>
     4.2、RIP协议与距离向量算法
    </h4>
    <p>
     路由信息协议RIP（Routing Information Protocol）是
     <strong>
      一种分布式的基于距离向量的路由选择协议，最大优点是简单
     </strong>
     。
    </p>
    <p>
     <strong>
      RIP协议要求网络中的每一个路由器都维护从它自己到其他每一个目的网络的唯一最佳距离记录
     </strong>
     。
    </p>
    <p>
     <strong>
      距离的定义：注意指路由器到目的路由器的距离。
      <br/>
      通常为跳数，定义从路由器到直接相连的网络的距离为1；每经过一个路由器跳数加1。RIP允许一条路径最多包含15个路由器，因此距离16表示网络不可达，同时可以看出RIP协议只适用于小型互联网！
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/29be914c8040f4d86bea10aebfdfe46b.png"/>
     <br/>
     对路由器2的路由表：
     <br/>
     到网络B距离为1，直接交付；
     <br/>
     到网络A、C距离为2，间接交付。
    </p>
    <p>
     问题一：RIP协议中路由器和谁交换信息？
     <br/>
     仅和
     <strong>
      相邻路由器
     </strong>
     交换信息。
     <br/>
     问题二：交换什么东西？
     <br/>
     交换的是
     <strong>
      自己的路由表
     </strong>
     。
     <br/>
     问题三：多久交换一次信息？
     <br/>
     每
     <strong>
      30秒
     </strong>
     交换一次路由信息，然后路由器根据信息更新路由表。如果超过180秒没收到信息，则判断相邻路由器可能出现问题或者链路出现问题，最后更新路由表。
    </p>

     <strong>
      总结：每隔30秒和相邻路由器交换路由表信息！！！
     </strong>

    <h4>
     <a id="43OSPF_312">
     </a>
     4.3、OSPF协议与链路状态算法
    </h4>
    <p>
     开放最短路径优先OSPF（Open Shortest Path First）是一种
     <strong>
      分布式链路状态协议
     </strong>
     。“开放”说明OSPF协议公开发表，不受某一厂家限制；“最短路径优先”是英文使用了
     <strong>
      Dijkstra算法
     </strong>
     。
    </p>
    <p>
     问题一：和谁交换？
     <br/>
     <strong>
      使用洪泛法向自治系统内所有路由器发送信息
     </strong>
     。
     <br/>
     问题二：交换什么？
     <br/>
     <strong>
      相邻的所有路由器的链路状态
     </strong>
     。链路状态指和本路由器相邻的路由器有哪些以及该链路的度量标准（费用、时延等等）。
     <br/>
     问题三：多久交换？
     <br/>
     <strong>
      只有当链路状态发生变化时
     </strong>
     ，路由器才向所有路由器洪泛发送信息。
    </p>
    <p>
     由于各路由器之间频繁交换链路状态信息，因此最终所有路由器会形成一个
     <strong>
      链路状态数据库----全网拓扑图
     </strong>
     。
    </p>
    <p>
     总结：
     <strong>
      只有当链路状态改变时，路由器才使用洪泛法向所有路由器发送链路状态信息
     </strong>
     。

     </strong>

     4.6、三种路由协议的比较：
    </h4>
    <p>
     RIP：分布式基于距离向量的内部网关路由选择协议，通过广播

    <h2>
     <a id="_372">
     </a>
     传输层
    </h2>
    <p>
     传输层的功能：
     <br/>
     1、提供进程间端到端的逻辑通信（网络层提供主机间的逻辑通信）。
     <br/>
     2、复用（所有应用进程都可以通过传输层在传输到网络层）与分用（传输层从网络层收到数据后交付给指明的应用进程）。
     <br/>
     3、传输层对收到的报文进行差错检测。
    </p>
    <p>
     <strong>
      套接字socket = 主机IP地址 + 端口号，故使用套接字可以唯一标识网络中的一个主机和它上面的进程。
     </strong>
    </p>
    <h3>
     <a id="1UDP_380">
     </a>
     1、UDP协议
    </h3>
    <p>
     无连接的用户数据报协议。在传输数据前不需要建立连接，收到UDP报文后也不需要确认。不可靠、无连接、时延小，适用于小文件。
    </p>
    <h3>
     <a id="2TCP_384">
     </a>
     2、TCP协议
    </h3>
    <p>
     <strong>
      面向连接、全双工、可靠的字节流服务
     </strong>
     。特点：TCP不提供广播或多播服务。由于TCP要提供可靠、面向连接的传输服务，因此不可避免地增加了许多控制开销。这不仅使协议数据单元的首部增大很多，还要占用许多处理集资源。
     <strong>
      TCP适合大量数据传输的应用
     </strong>
     ！
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7e61993363ed0a8b307dd67e98df9c9e.png"/>
     <br/>
     1）
     <strong>
      序号
     </strong>
     ：本报文段所发送数据的
     <strong>
      第一个字节的序号
     </strong>
     。例如，发送数据123456，那么序号就是1；发送数据789序号就是7。
    </p>
    <p>
     2）
     <strong>
      确认号
     </strong>
     ：期望收到
     <strong>
      下一个报文段的第一个字节的序号
     </strong>
     。例如，确认号N，说明前N-1个数据正确收到。
    </p>
    <p>
     3）
     <strong>
      6个控制位（掌握ACK、SYN即可）
     </strong>
     ：
     <br/>
     URG（urgent）紧急位：当URG=1时，发送方有紧急数据发送，不用在缓存中排队。
    </p>
    <p>
     <strong>
      ACK确认位
     </strong>
     ：在TCP连接建立后所有传送报文的ACK=1
    </p>
    <p>
     PSH推送位：当PSH=1时，接收方尽快交给应用进程，不用等到缓存满了再交付。
    </p>
    <p>
     <strong>
      SYN同步位
     </strong>
     ：当SYN=1时，表明是一个
     <strong>
      连接请求/连接接受的报文
     </strong>
     （TCP三次握手建立连接时会有）。
    </p>
    <p>
     4）
     <strong>
      窗口
     </strong>
     ：指接收窗口，即允许发送方的发送数据大小（参考数据链路层的流量控制的滑动窗口）
    </p>
    <h4>
     <a id="21TCP_405">
     </a>
     2.1、TCP传输连接管理
    </h4>
    <p>
     1、TCP连接传输三个阶段：
     <strong>
      连接建立、数据传送、连接释放
     </strong>
     <br/>
     2、TCP连接的建立采用
     <strong>
      客户服务器方式
     </strong>
     ----主动发起连接建立的应用进程叫做客户，而被动等待连接建立的应用进程叫做服务器。
    </p>
    <p>
     <strong>
      TCP三次握手建立连接：
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ab7bee4be00b56c268292a8c9cc365d1.png"/>
     <br/>
     SYN洪泛攻击和SYN Cookie了解即可！
     <br/>
     为什么需要TCP三次握手？：为了防止服务器端开启一些无用的连接增加服务器开销以及防止已失效的连接请求报文段突然又传送到了服务端，因而产生错误。
    </p>
    <p>
     <strong>
      TCP四次挥手释放连接
     </strong>
     ：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/61b8b54d29fb6bc23ea9c03884bdb56d.png"/>
     <br/>
     参与一条TCP连接的两个进程中的
     <strong>
      任何一个都能终止进程
     </strong>
     ，在释放连接后，主机中的资源将被释放。
    </p>
    <p>
     如上图：
     <br/>
     第一步：客户端发送一个
     <strong>
      释放连接报文段（FIN=1）
     </strong>
     ，主动关闭TCP连接。
     <br/>
     第二步：服务器端回送一个
     <strong>
      确认报文段（ACK=1）
     </strong>
     ，呈现
     <strong>
      半关闭状态
     </strong>
     （客户-服务器方向的连接被释放了）。
     <br/>
     第三步：服务器端发送一个
     <strong>
      释放连接报文段（FIN=1、ACK=1）
     </strong>
     ，主动关闭服务器-客户端方向的TCP连接。
     <br/>
     第四步：客户端回送一个
     <strong>
      确认报文段（ACK=1）
     </strong>
     ，再等到计时器设置的2MSL（最长报文段寿命）后，连接彻底关闭。
    </p>
    <h4>
     <a id="22TCP_424">
     </a>
     2.2、TCP如何保证可靠传输？
    </h4>
    <p>
     可靠传输：接收方进程从缓冲区读出的字节流和发送方发出的字节流一样。TCP通过校验、序号、确认、重传四步保证可靠传输。
    </p>
    <p>
     1）为每个已发送的报文段启动一个超时定时器，若在定时器超时前收到接收方应答信息则释放该报文段占用的缓冲区；否则重传该报文段，直到收到应答或重传次数过多。
    </p>
    <p>
     2）接收方收到报文段后进行校验，如果正确把数据交给上层协议，再给发送方发送一个确认报文段表明数据已收到。
    </p>
