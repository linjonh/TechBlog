    <h3>
     <a id="1__3">
     </a>
     1. 概述
    </h3>
    <p>
     ESP32是乐鑫开发的一个WiFi 蓝牙双模组，支持ESP-IDF开发以及AT指令模式开发，关于两种模式的区别不在这赘述，大家可自行上网搜索了解
    </p>
    <p>
     <strong>
      本文旨在介绍如何使用 ESP32 的 AT 指令模式连接AWS亚马逊云
     </strong>
     ，由于连接AWS亚马逊云是需要
     <strong>
      双向认证
     </strong>
     的，相比其他的云平台，如阿里云、百度云、腾讯云较为复杂，因此特编写此博客已做记录及分享
    </p>
    <p>
     关于如何使用 ESP32 AT指令连接阿里云、百度云等国内云平台，可直接参考官方的AT用户指南，
     <a href="https://docs.espressif.com/projects/esp-at/zh_CN/latest/esp32/AT_Command_Examples/MQTT_AT_Examples.html#tcp-mqtt-mqtt" rel="nofollow">
      ESP32-AT指令用户指南
     </a>
    </p>

    <h3>
     <a id="3_AWS_18">
     </a>
     3. AWS云平台接入
    </h3>
    <h4>
     <a id="31_AWS_19">
     </a>
     3.1 AWS云平台接入概述
    </h4>
    <p>
     AWS云平台，也就是亚马逊云，与阿里云接入相比较，其差别在于：
    </p>
    <ul>
     <li>
      阿里云采用
      <strong>
       用户名+密码
      </strong>
      作为设备的判断依据进行接入
     </li>
     <li>
      亚马逊云采用
      <strong>
       设备证书
      </strong>
      作为设备的判断依据进行接入
     </li>
    </ul>
    <p>
     注意，此处所描述的接入均指采用 Mqtt 接入云平台
    </p>
    <h4>
     <a id="32_AWS_27">
     </a>
     3.2 接入AWS云平台流程
    </h4>
    <p>
     接入AWS云平台的流程大体上可分为以下几个部分：
    </p>
    <ol>
     <li>
      创建云端虚拟设备
      <ul>
       <li>
        获取
        <strong>
         设备证书、密钥
        </strong>
        等连接凭证
       </li>
      </ul>
     </li>
     <li>
      采用工具模拟设备接入云平台验证接入可行性
      <ul>
       <li>
        采用
        <code>
         Mqtt.fx
        </code>
        工具验证
       </li>
       <li>
        采用
        <code>
         python
        </code>
        脚本验证
       </li>
      </ul>
     </li>
     <li>
      采用实体设备接入云平台
      <ul>
       <li>
        将设备证书以不同的方式写入ESP32，将ESP32实体设备接入云平台
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="52_PKI__94">
     </a>
     5.2 PKI 文件导入方式
    </h4>
    <p>
     当不想自行编译AT固件，想在官方提供的AT固件的基础上进行加工修改，将证书导入，可以使用PKI文
     <br/>
     件导入的方式将证书等文件导入ESP32设备中。
    </p>
    <p>
     采用这种方式导入，首先需要做的是将 设备证书、私有密钥、根证书 转化为对应的bin文件，之后通过烧录软件进行烧录，或者通过AT指令进行烧录的方式对ESP32 原有的AT固件内的 此三类文件进行替换。
    </p>
    <h5>
     <a id="521_bin_100">
     </a>
     5.2.1 bin文件生成
    </h5>
    <ol>
     <li>
      下载python脚本：
      <a href="https://github.com/espressif/esp-at/blob/b3c6910/tools/AtPKI.py">
       https://github.com/espressif/esp-at/blob/b3c6910/tools/AtPKI.py
      </a>
     </li>
     <li>
      使用命令将 设备证书、私有密钥、根证书 分别转化为对应的bin文件
     </li>
     <li>
      命令使用说明在 github 上的 readme 中有详细描述
     </li>
    </ol>
    <pre><code class="prism language-commandline">python SCRIPT_PATH generate_bin -b OUTPUT_BIN_NAME PKI_LIST
</code></pre>
    <ul>
     <li>
      <code>
       SCRIPT_PATH
      </code>
      :
      <ul>
       <li>
        the path of script. If you’re in Tools folder of AT, then SCRIPT_PATH=AtPKI.py
       </li>
      </ul>
     </li>
     <li>
      <code>
       OUTPUT_BIN_NAME
      </code>
      :
      <ul>
       <li>
        the file name for generated PKI bin
       </li>
      </ul>
     </li>
     <li>
      <code>
       PKI_LIST
      </code>
      :
      <ul>
       <li>
        a list of (type, file_name) pair:
        <ul>
         <li>
          type: ca, cert, key
         </li>
         <li>
          file_name: PKI item file name
         </li>
        </ul>
       </li>
       <li>
        each item will have ID according to its type. the ID is generated as input order. ID will be used to match certificates and keys.
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li>
      注意 设备证书、私有密钥、根证书 的
      <code>
       PKI_LIST
      </code>
      里面的
      <code>
       type
      </code>
      是不一样的！
     </li>
    </ul>

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9179c972e95a84bdf5156d990d9bcae7.png"/>
     </li>
     <li>
      之后将证书文件烧录进对应的地址，进行覆盖，ESP32 AT固件分区如下表所示：
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1f2f4994094bce0d5bd1f356c3361d7f.png"/>
     </li>
     <li>
      我们这里需要替换的是
      <code>
       mqtt_ca
      </code>
      <code>
       mqtt_cert
      </code>
      <code>
       mqtt_key
      </code>
      这三个文件，
      <strong>
       不是client的也不是server的！！！
      </strong>
      ，因此将5.2.1节中生成的bin文件分别下载至对应地址
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/631bab8c618ea98bf1ef8f332b421a00.png"/>
     </li>
    </ol>
    <h5>
     <a id="522_AT_138">
     </a>
     5.2.2 AT指令烧录
    </h5>
    <p>
     除了使用烧录的方式将证书bin文件写入flash，ESP32 AT固件还支持通过串口指令写入AT固件
    </p>
    <p>
     首先需要计算需要写入的
     <code>
      ca.bin
     </code>
     <code>
      cert.bin
     </code>
     <code>
      private_key.bin
     </code>
     文件的大小
    </p>
    <p>
     可以通过串口助手进行计算，使用串口助手向一个没有连接设备的串口发送整个文件，串口助手下方会记录发送的字节数
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0999393922286e5df00f307a54f49cc3.png"/>
    </p>
    <ol>
     <li>
      <p>
       查询AT分区信息
       <code>
        AT+SYSFLASH?
       </code>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/22bfb6e5188f91430c61d6117afc8355.png"/>
      </p>
     </li>
     <li>
      <p>
       擦除
       <code>
        mqtt_cert
       </code>
       分区
       <code>
        AT+SYSFLASH=0,"mqtt_cert"
       </code>
      </p>
     </li>
     <li>
      <p>
       写入
       <code>
        mqtt_cert
       </code>
       分区
       <code>
        AT+SYSFLASH=1,"mqtt_cert",0,1232
       </code>
      </p>
     </li>
     <li>
      <p>
       擦除
       <code>
        mqtt_key
       </code>
       分区
       <code>
        AT+SYSFLASH=0,"mqtt_key"
       </code>
      </p>
     </li>
     <li>
      <p>
       写入
       <code>
        mqtt_key
       </code>
       分区
       <code>
        AT+SYSFLASH=1,"mqtt_key",0,1688
       </code>
      </p>
     </li>
     <li>
      <p>
       擦除
       <code>
        mqtt_ca
       </code>
       分区
       <code>
        AT+SYSFLASH=0,"mqtt_ca"
       </code>
      </p>
     </li>
     <li>
      <p>
       写入
       <code>
        mqtt_ca
       </code>
       分区
       <code>
        AT+SYSFLASH=1,"mqtt_ca",0,1200
       </code>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9b360639bc7757f670c81e3a226552d4.png"/>
      </p>
     </li>
     <li>
      <p>
       更新完Flash之后需要复位下，以保证更新内容重新加载
       <code>
        AT+RST
       </code>
      </p>
     </li>
    </ol>
    <h4>
     <a id="53_AWS_158">
     </a>
     5.3 连接AWS亚马逊云
    </h4>
    <p>
     在5.2中我们已经将设备证书、私有密钥、根证书烧录入ESP32的flash中，接下来便可以连接AWS了
    </p>
    <ol>
     <li>
      <p>
       使用串口工具分别连接调试串口和AT指令通讯串口
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/678f50b2193e6f5381829a995ab4a81f.png"/>
      </p>
     </li>
     <li>
      <p>
       切换为Station模式
       <code>
        AT+CWMODE=1
       </code>
      </p>
     </li>
     <li>
      <p>
       连接热点
       <code>
        AT+CWJAP="abc","123456789"
       </code>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/14a817c64d8369a6a0ccecf3367e802a.png"/>
      </p>
     </li>
     <li>
      <p>
       设置SNTP服务器地址
       <code>
        AT+CIPSNTPCFG=1,8,"pool.ntp.org"
       </code>
      </p>
     </li>
     <li>
      <p>
       查询网络时间
       <code>
        AT+CIPSNTPTIME?
       </code>
       ，主要用来确认网络可用
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4b834eb2f2c336d37667aefd597679b6.png"/>
      </p>
     </li>
     <li>
      <p>
       配置MQTT服务器
       <code>
        AT+MQTTUSERCFG=0,5,"sdk-java","espressif","1234567890",0,0,""
       </code>
       ，这里主题一定要是云端设备策略里面支持的，由于AWS采用证书认证，因此这里的设备名和设备密码随便写
      </p>
     </li>
     <li>
      <p>
       连接MQTT服务器
       <code>
        AT+MQTTCONN=0,"xxxxxxxxxxx.iot.us-east-2.amazonaws.com",8883,1
       </code>
       ，这里的服务器地址请自行更换为自己的
      </p>
     </li>
     <li>
      <p>
       查看串口打印信息确认是否连接成功，连接失败的话请重新阅读自行查找原因
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c7ff470f4fbc69b2573da5c63a09ff41.png"/>
      </p>
     </li>
     <li>
      <p>
       订阅主题
       <code>
        AT+MQTTSUB=0,"sdk/test/Python",1
       </code>
      </p>
     </li>
     <li>
      <p>
       往主题发布消息
       <code>
        AT+MQTTPUB=0,"sdk/test/Python","hello aws!",1,0
       </code>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0c1f2d6bfc67b0e71998fade747053a1.png"/>
      </p>
     </li>
    </ol>
    <h3>
     <a id="6__177">
     </a>
     6. 总结
    </h3>
    <p>
     以上便是使用 ESP32 的 AT 模式连接基于双向认证的 AWS 的全部步骤了！
    </p>
    <p>
     关于ESP32的AT指令使用，其实官方的文档说的还是比较仔细的了，但是有些细节之处没有明确说明，导致实际在开发过程中踩了很多的坑，但尽管官方文档存在一点点缺陷，但总体上而言是已经相当不错的了，如果你要使用 ESP32 的AT的模式，还是需要仔细阅读的
    </p>
    <p>
     通过上述操作，相信现在你对于AWS基于证书的双向认证的这套连接方案也有了大概的了解，同时你也会发现，对于一个产品来说，如何将证书文件下载到ESP32中其实是一个非常麻烦的事情！
    </p>
    <p>
     每一个ESP32模块都需要有一个云端生成的唯一的证书文件存放在ESP32设备内部，尽管通过直接编译或是使用下载工具下载的方式将证书bin文件下载到设备内，看上去都还不是很复杂，但是这对于一个产品项目来说，这对于产品的生产会是非常麻烦的一个问题，不过庆幸的是，ESP32还支持了通过串口方案对内部flash进行刷写，因此对于产品项目，看来大家只能选择采用此种方式了（如果你要使用AT模式连接AWS的话）
    </p>
