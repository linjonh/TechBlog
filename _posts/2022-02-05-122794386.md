    <p>
     区块链中采用对原本信息的哈希值进行加密的方式来进行签名
     <br/>
     数字签名：签名和验证
     <br/>
     一个消息M、其哈希摘要D
    </p>
    <p>
     <strong>
      签名者
     </strong>
     通过自身的
     <strong>
      私钥Kp
     </strong>
     生成对应的签名
     <strong>
      S=F（D，Kp）
     </strong>
     <br/>
     <strong>
      验证者
     </strong>
     通过
     <strong>
      公钥K
     </strong>
     解密
     <strong>
      S
     </strong>
     得到
     <strong>
      消息M的摘要D=F（S，K）
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b2e19f1ed90f4eaf0acc8f63752613ef.png"/>
    </p>
    <p>
     <strong>
      双方通信的时候！！！
      <br/>
      公钥加密，私钥解密
      <br/>
      私钥签名，公钥解密
     </strong>
    </p>
    <p>
     <strong>
      发送方
     </strong>
     将原文密文（用接收方的公钥加密）、签名（发送方的私钥签名），发送给接收方。
     <br/>
     <strong>
      接收方
     </strong>
     用自己的私匙解开密文，解密后算哈希值M1。通过发送方的公钥解开发送方签名，得到摘要M2，并且通过M1和M2进行对比，最后的摘要一致，则认为哈希值是对的。
    </p>
    <p>
     <strong>
      为什么用接收方的公钥加密？
     </strong>
     <br/>
     因为通过这种方式，只有接受方的私钥才能解开。
     <br/>
     <strong>
      为什么要用发送方的私钥签名，不用其它人的私钥签名？
     </strong>
     <br/>
     因为这种方式，才能让接收方确认这条信息是发送方发出来的。只有发送方的公钥才能解开发送方的签名。
    </p>
