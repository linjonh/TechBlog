    <h2>
     <a id="1__1">
     </a>
     1 通信基本概念
    </h2>
    <p>
     通信一般有三个步骤：编码、传输和解码。
     <br/>
     可分为同步通信和异步通信：
     <br/>
     <mark>
      <strong>
       同步
      </strong>
     </mark>
     ：需要时钟信号
     <br/>
     <mark>
      <strong>
       异步
      </strong>
     </mark>
     ：一般有起始和终止信号。
     <br/>
     可分为电平信号和差分信号：
     <br/>
     <strong>
      <mark>
       电平
      </mark>
     </strong>
     ：信号逻辑值由信号线电平和参考电平决定（一般为地）
     <br/>
     <strong>
      <mark>
       差分
      </mark>
     </strong>
     ：没有参考电平。信号逻辑值有两根差分线的电压差决定。
     <br/>
     可分为单工、半双工，全双工
     <br/>
     <mark>
      <strong>
       单工
      </strong>
     </mark>
     ：信号方向只能单向
     <br/>
     <mark>
      <strong>
       半双工
      </strong>
     </mark>
     ：方向能双向，即可发送和接收，但是不能同时发送和接收。
     <br/>
     <mark>
      <strong>
       全双工
      </strong>
     </mark>
     ：可以同时发送和接收。
    </p>
    <h2>
     <a id="2__13">
     </a>
     2 串口通信
    </h2>
    <h3>
     <a id="21___14">
     </a>
     2.1 基本概念
    </h3>
    <p>
     <mark>
      <strong>
       串口
      </strong>
     </mark>
     ：在嵌入式里指的是UART口，在台式机一般是COM口，用RS232 RS422 RS485
     <br/>
     <mark>
      <strong>
       UART
      </strong>
     </mark>
     ：, Universal Asynchronous Receiver-Transmitter, 通用异步收发器 ，常用TTL电平即3.3V或者5.0V。
     <br/>
     <mark>
      <strong>
       COM口
      </strong>
     </mark>
     ：在台式机上常用的口，DB9口，接口协议：RS232和RS485，RS422；
    </p>
    <p>
     UART口（四针：RXD TXD GND VCC，三针，两针）、COM口（9针）指的是物理接口形式;TTL、RS232和RS485指的是电平逻辑标准。RS422 RS485也有16针的接口。
    </p>
    <h3>
     <a id="22__UART___21">
     </a>
     2.2 UART 硬件接口
    </h3>
    <h4>
     <a id="221__22">
     </a>
     2.2.1 波特率、起始位、数据位、奇偶校验位、停止位
    </h4>
    <p>
     <mark>
      <strong>
       波特率
      </strong>
     </mark>
     ：bit/s，指的是串口通信的速率，也就是串口通信时每秒钟可以传输多少个二进制位。常见的有 9600 115200
    </p>
    <p>
     一个通信单元由：起始位+数据位+奇偶校验位+停止位组成的。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8cf2d509cedec24f45e373ed7f94c748.png">
      <br/>
      <mark>
       <strong>
        空闲位
       </strong>
      </mark>
      ：UART协议规定，当总线处于空闲状态时信号线的状态为‘1’即高电平，表示当前线路上没有数据传输。
      <br/>
      <mark>
       <strong>
        起始位
       </strong>
      </mark>
      ：开始进行数据传输时发送方要先发出一个低电平’0’来表示传输字符的开始。因为总线空闲时为高电平所以开始一次通信时先发送一个明显区别于空闲状态的信号即低电平。依靠检测起始位来实现发送与接收方的时间自同步的。
      <br/>
      <mark>
       <strong>
        数据位
       </strong>
      </mark>
      ：可以是4、5、6、7、8位，一般是8位，协议规定是LSB。
      <br/>
      <mark>
       <strong>
        奇偶校验位
       </strong>
      </mark>
      ：数据位加上这一位后，使得“1”的位数为偶数（偶校验）或奇数（奇校验）以此来校验数据传送的正确性；
      <br/>
      <mark>
       <strong>
        停止位
       </strong>
      </mark>
      ：一个字符数据的结束标志，可以是1位、1.5位、2位的高电平。
     </img>
    </p>
    <p>
     <strong>
      串口通信时因为是异步通信，所以通信双方必须事先约定好通信参数，这些通信参数包括：波特率、数据位、校验位、停止位（串口通信中起始位定义是唯一的，所以一般不用选择）。
     </strong>
    </p>
    <p>
     <mark>
      <strong>
       通信线
      </strong>
     </mark>
     ：RXD TXD GND ，也可能会有VCC，还有一些流控线。
    </p>
    <h4>
     <a id="222_UASRT_36">
     </a>
     2.2.2 UASRT
    </h4>
    <p>
     USART(通用同步/异步串行接收/发送器)，相比UART，USART多了一个时钟信号。在STM32使用中，我一般把时钟禁用，当做UART使用。
    </p>
    <h3>
     <a id="23___38">
     </a>
     2.3 电平
    </h3>
    <h4>
     <a id="231_TTL_39">
     </a>
     2.3.1 TTL电平
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/64f5ef796eeb777d7f3fc3619891dbe5.png">
      <br/>
      主要看中间两种，关注5个参数。
      <br/>
      <strong>
       输入高电平（Vih）输入低电平（Vil）输出高电平（Voh）输出低电平（Vol）
      </strong>
      ，这四个就顾名思义了。
      <br/>
      <strong>
       阀值电平(Vt)
      </strong>
      ：数字电路芯片都存在一个阈值电平，就是电路刚刚勉强能翻转动作时的电平。
      <br/>
      一般情况下，在嵌入式中使用，只需要设置寄存器后，就能完成对UART的初始化。
     </img>
    </p>
    <h3>
     <a id="23__RS232___45">
     </a>
     2.3 RS232 通信协议
    </h3>
    <p>
     物理接口，硬件接口：DB9
    </p>
    <h4>
     <a id="231_RS232_47">
     </a>
     2.3.1 RS232电平
    </h4>
    <p>
     <strong>
      在TxD和RxD
     </strong>
     上：逻辑1(MARK)=-3V～-15V ，逻辑0(SPACE)=+3～＋15V
     <mark>
      反逻辑
     </mark>
    </p>
    <p>
     <strong>
      在RTS、CTS、DSR、DTR和DCD等控制线上
     </strong>
     上：
     <br/>
     信号有效（接通，ON状态，正电压）＝+3V～+15V
     <br/>
     信号无效（断开，OFF状态，负电压)=-3V～-15V
    </p>
    <p>
     RS232和UART的区别和联系：UART就是车站，RS232 RS485 RS422 TTL的串口都是每一辆公交车，有不同的规则。
    </p>
    <h4>
     <a id="232_RS232TTL_55">
     </a>
     2.3.2 RS232与TTL的转换
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fc0abca16fa8d84072300493b5706b53.png">
      <br/>
      从TTL转为RS2323电平,由于二极管与电容的作用使得在二极管D1与电容C7交接处的电压保持在-3V~-15V.
      <br/>
      当TXD为"1"(TTL)时,Q3截止,PCRXD上的电压与PCTXD电压相等,也是-3~-15V,为"1"(RS232)
      <br/>
      当TXD为"0"(TTL)时, Q3导通,则PCRXD电压约为+5V,这个电压在+3~+15V之间,根据RS232电平,它是"0"…也就是说TTL的"1"经过这个电平转换电路后,RS2323可以识别出它是"1",是"0"也能识别为0.这就实现了从TTL到RS232的电平转换.
      <br/>
      从RS232转换为TTL电平那就简单了,当PCTXD为"1",即-3~-15V时,Q4截止,RXD电压约为5V,为"1",当PCTXD为"0"时,Q4导通,电压为0,电平为"0".那么从RS232到TTL的电平转换也实现了.
      <br/>
      备注：D2是为了防止Q4的BE反向击穿，TXD的最低电压时15V，Q4的BE耐压是6V左右。?
     </img>
    </p>
    <p>
     简略大概的说：
     <br/>
     当TXD=1时，Q3截止，导致PCRXD=1;
     <br/>
     当TXD=0时，Q3导通，导致PCRXD=0;
     <br/>
     当PCTXD=1时，Q4导通，导致RXD=1;
     <br/>
     当PCTXD=0时，Q4截止，导致RXD=0;
    </p>

    <h4>
     <a id="234_RS232__81">
     </a>
     2.3.4 RS232 直连和交叉
    </h4>
    <p>
     两个设备如果要通信，应该要交叉连即A的TXD连B的RXD，才能实现通信。如果仅是设备的延长，可用直连线。本质还是要看设备如何连接，确保能够通信。
    </p>
    <h3>
     <a id="24__RS485_83">
     </a>
     2.4 RS485
    </h3>
    <h4>
     <a id="241_RS485_84">
     </a>
     2.4.1 为什么会出现RS485
    </h4>
    <p>
     为了解决RS232出现的一些问题。
     <br/>
     1：RS232接口电平比较高
     <br/>
     2：采用共模传输，抗噪声能力差
     <br/>
     3：数率低，20kb/s
     <br/>
     4：传输距离有限
     <br/>
     这些一部分是共模传输导致，一部分是电平导致，RS485解决了一部分问题
    </p>

    <h4>
     <a id="243_RS485_105">
     </a>
     2.4.3 RS485的电平
    </h4>
    <p>
     RS485电平 由于两者均采用 差分传输（平衡传输）的方式，所以他们的电平方式，一般有两个引脚 A,B
     <br/>
     发送端 AB间的电压差。
     <br/>
     ＋2 ～ ＋6v 表示1
     <br/>
     －2 ～ －6v 表示0
     <br/>
     接收端 AB间的电压差
     <br/>
     大于 ＋200mv表示1
     <br/>
     小于 －200mv表示0
     <br/>
     定义逻辑0为A&lt;B的状态
     <br/>
     定义逻辑1为A&gt;B的状态
     <br/>
     AB之间的电压差不小于200mv
     <br/>
     且可以看出，他的工作方式只有两根数据线，说明是一种半双工的工作方式
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c171926a980c6c357dcea5777bf1b30c.png"/>
    </p>
    <h4>
     <a id="244_RS485_119">
     </a>
     2.4.4 RS485可以进行点多对的传输
    </h4>
    <p>
     要求一个主机，多个从机。如图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7b3e2985bade240c99728a92ae3da40f.png">
      <br/>
      在同一总线上最多可以挂接32个节点。要求有地址码。主机可以一直发送，从机只能有一个发送，且从机不能互相通信。
     </img>
    </p>

    <h2>
     <a id="3_IIC__157">
     </a>
     3 IIC通信 总线
    </h2>
    <p>
     I²C（Inter-Integrated Circuit），中文应该叫集成电路总线。iic也是串行通信，但是与串口的区别在于，一个是同步，就是有时钟信号，一个是总线型，与RS232那种异步的接口型有一定区别。
    </p>
    <h3>
     <a id="31___159">
     </a>
     3.1 硬件结构
    </h3>
    <p>
     I²C协议仅需要一个SDA和SCL引脚。SDA是串行数据线的缩写，而SCL是串行时钟线的缩写。这两条数据线需要接上拉电阻。值得一提的是多数通信默认都是高电平。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a8d216a0f68596e72284cdccbead71a8.png">
      <br/>
      从上面可以看出，它是一个同步的通信模式，所以没有固定的波特率，波特率由SCL决定。
      <br/>
      传输速度；
      <br/>
      标准模式：Standard Mode = 100 Kbps
      <br/>
      快速模式：Fast Mode = 400 Kbps
      <br/>
      高速模式： High speed mode = 3.4 Mbps
      <br/>
      超快速模式： Ultra fast mode = 5 Mbps
     </img>
    </p>
    <p>
     它是一个总线，就是他可以实现多主设备通信，即多对多通信。使用I²C，可以将多个从机（Slave）连接到单个主设备（Master），并且还可以有多个主设备（Master）控制一个或多个从机（Slave）。
    </p>
    <p>
     I²C总线（SDA，SCL）内部都使用
     <mark>
      <strong>
       漏极开路驱动器（开漏驱动）
      </strong>
     </mark>
     ，因此SDA和SCL 可以被拉低为低电平，但是不能被驱动为高电平，所以每条线上都要使用一个上拉电阻，默认情况下将其保持在高电平；
    </p>
    <h3>
     <a id="32___172">
     </a>
     3.2 数据传输协议
    </h3>
    <p>
     主设备和从设备进行数据传输时遵循以下协议格式。数据通过一条SDA数据线在主设备和从设备之间传输0和1的串行数据。串行数据序列的结构可以分为，开始条件，地址位，读写位，应答位，数据位，停止条件，具体如下所示；
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6a21234c055d73939dc0193a2b20ccf5.png">
      <br/>
      <strong>
       开始条件
      </strong>
      ：SDA 由高电平向低电平跳变，时钟为高
      <br/>
      <strong>
       地址位
      </strong>
      ：一般为7位，也有8位、10位的。理论上讲可以有127个从机。
      <mark>
       但是iic协议规定了总线电容不能超过400pF，管脚和PCB都是有寄生电容的，所以是有限制的，经验上是最多8个
      </mark>
      。
      <br/>
      <strong>
       R/W位
      </strong>
      ：读写位
      <br/>
      为0低电平表示，主设备将数据发送到从设备
      <br/>
      为1高电平表示，主设备接收从设备数据
      <br/>
      <strong>
       应答位
      </strong>
      ：开始信号后第9个时钟位时，从设备发送应答信号，即SDA被拉低一个时钟位。如果没有这个应答，SDA输出高电平，主设备重启或者停止。
      <br/>
      <strong>
       数据位
      </strong>
      ：一般为8位，注意：时钟信号为高时，SDA不允许变化，时钟为低时，SDA才能发生跳变。
      <br/>
      <strong>
       应答位
      </strong>
      ：数据发送了会紧跟一个应答信号。
      <br/>
      停止条件：SDA 由低电平向高电平跳变，时钟为高。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/11775523a21e5e02f9bdc83808561ab7.png"/>
      <br/>
      需要注意的是：起始信号是必须的，应答和结束信号都可以不要。
     </img>
    </p>
    <h3>
     <a id="33_____186">
     </a>
     3.3 漏极开路输出 集电极开路输出 推挽输出
    </h3>
    <h4>
     <a id="331__187">
     </a>
     3.3.1 推挽输出
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/01899482a9bf82178899b75f32c79b8e.png"/>
     <br/>
     当Vin是V+时，Q3导通，输出的是高电平，叫推
     <br/>
     当Vin是V-时，Q4导通，输出的是低电平，叫挽
    </p>
    <h4>
     <a id="331__191">
     </a>
     3.3.1 集电极开路输出
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c24f8d3214ea569374cc4d839652f643.png"/>
     <br/>
     Vin为高，Q5导通，输出为低。Vin为高，Q5截止，LED灯亮。
    </p>
    <h4>
     <a id="333__OD_194">
     </a>
     3.3.3 漏极开路输出 OD门
    </h4>
    <p>
     将上面电路换成NMOS管。就和上面电路稍微有点区别了。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3fefaf514439bd2c28f8dd43d1412e99.png"/>
     <br/>
     只要对栅极施加电压，DS就会导通。MOS管的
     <strong>
      输入阻抗
     </strong>
     非常大，这意味着：没有电流从控制电路流出，也没有电流进入控制电路。没有电流流入或流出，就不会烧坏控制电路。而双极型晶体管不同，是电流控制性元器件，如果使用开集电路，可能会烧坏控制电路。这大概就是我们总是听到开漏电路而很少听到开集电路的原因吧？因为开集电路被淘汰了。
    </p>
    <p>
     <mark>
      总结：
      <br/>
      1：推挽输出能够输出高或者低，而开漏输出只能输出低，或者关闭输出，因此开漏输出总是要配一个上拉电阻使用。
      <br/>
      2：开漏输出的上拉电阻不能太小，太小的话，当开漏输出的下管导通时，电源到地的电压在电阻上会造成很大的功耗，因此这个电阻阻值通常在10k以上，这样开漏输出在从输出低电平切换到高电平时，速度是很慢的。典型值为4.7k或者10k（有计算公式）
      <br/>
      3：推挽输出任意时刻的输出要么是高，要么是低，所以不能将多个输出短接，而开漏输出可以将多个输出短接，共用一个上拉，此时这些开漏输出的驱动其实是与非的关系。
     </mark>
    </p>
    <h3>
     <a id="34__IIC_203">
     </a>
     3.4 IIC的连接和总线仲裁
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/75eb5ee732b861763a5b59cd60bf24f3.png"/>
     <br/>
     当多主机会产生总线裁决问题。当多个主机同时想占用总线时，企图启动总线传输数据，就叫做总线竞争。I2C通过总线仲裁，以决定哪台主机控制总线
    </p>
    <p>
     仲裁：每个主设备都需要在发送消息之前检测SDA线是低电平还是高电平；
     <br/>
     如果SDA线为低电平，则意味着另一个主设备可以控制总线，并且主设备应等待发送消息。
     <br/>
     如果SDA线为高电平，则可以安全地发送消息。
    </p>

    </h2>
    <p>
     SPI是同步通信，全双工。
    </p>
    <h3>
     <a id="41__SPI_216">
     </a>
     4.1 SPI的硬件结构
    </h3>
    <p>
     由四根线完成通信
     <br/>
     <strong>
      MISO
     </strong>
     ：master input slave output 主收从发
     <br/>
     <strong>
      MOSI
     </strong>
     ：主发从收
     <br/>
     <strong>
      CLK
     </strong>
     ：时钟
     <br/>
     <strong>
      SS
     </strong>
     ：片选
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cce78a85e4be0fabc54e5385990abfd4.png"/>
     <br/>
     一般在嵌入式中，都是通过移位寄存器完成通信、还是发送缓存寄存器和接收缓存寄存器。
     <br/>
     值得注意的是，SPI只要通信成功，都是会全双工工作，就是既会发送也会接收，主机从机皆如此，但是如果只是单端的发送，那忽略到接收的数据就行了。
    </p>
    <h3>
     <a id="42__SPI_225">
     </a>
     4.2 SPI的工作模式
    </h3>
    <p>
     SPI有四种工作模式，由时钟极性CPOA和时钟相位CPHA共同决定。
     <br/>
     <strong>
      时钟极性 CKP/Clock Polarity
     </strong>
     ：
     <br/>
     CKP = 0：时钟空闲IDLE为低电平 0；
     <br/>
     CKP = 1：时钟空闲IDLE为高电平1；
     <br/>
     <strong>
      时钟相位 CKE /Clock Phase (Edge)
     </strong>
     <br/>
     CKE = 0：在时钟信号SCK的第一个跳变沿采样；
     <br/>
     CKE = 1：在时钟信号SCK的第二个跳变沿采样；
     <br/>
