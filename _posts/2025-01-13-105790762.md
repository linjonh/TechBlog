    <h3 id="请求消息中文显示乱码">
     <strong>
      请求消息中文显示乱码
     </strong>
    </h3>
    <p>
     <img alt="请求消息中文显示乱码" src="https://i-blog.csdnimg.cn/blog_migrate/fc96f0821fb5936b15990d40395c203a.jpeg"/>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        产生原因
       </strong>
       ：
       <br/>
       这种情况在jmeter3.0的版本中才会产生。这不是乱码，而是由于3.0中优化body data后，使用默认的字体（Consolas）不支持汉字的显示。
      </p>
     </li>
     <li>
      <strong>
       解决办法
      </strong>
      ：
     </li>
     <li>
      打开 %JMeter_HOME%\bin\jmeter.properties文件，找到jsyntaxtextarea.font.family这个选项，取消前面的#号，使其使用Hack字体即可，当然，你也可以换成 宋体以及其他支持的字体集。
      <br/>
      <img alt="修改JMeter默认字体" src="https://i-blog.csdnimg.cn/blog_migrate/5e6555bd39fd2d7b4f5afe542442b3cf.jpeg"/>
     </li>
     <li>
      <h3 id="响应消息中文显示乱码">
       <strong>
        响应消息中文显示乱码
       </strong>
      </h3>
      <p>
       <img alt="响应消息中文显示乱码" src="https://i-blog.csdnimg.cn/blog_migrate/69db8a374215cc66d0f12695d701ea63.jpeg"/>
      </p>
     </li>
     <li>
      <p>
       <strong>
        产生原因
       </strong>
       ：
       <br/>
       Jmeter的结果处理编码与被测试对象的编码不一致。Jmeter的sampler请求结果的默认编码方式为：ISO-8859-1（不支持中文），Jmeter的sampler请求结果的默认编码方式为：ISO-8859-1（不支持中文）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        解决办法一
       </strong>
       ：
       <br/>
       使用后置控制器”BeanShell PostProcessor”来动态修改结果处理编码，使之与被测对象保持一致；
       <br/>
       <strong>
        优点
       </strong>
       ：灵活，随时修改；
       <br/>
       <strong>
        缺点
       </strong>
       ：要根据不同的对象设置不同的编码；
       <br/>
       <strong>
        适用范围
       </strong>
       ：测试不同的公司项目，有些公司喜欢GBK，有些是UTF-8；
      </p>
     </li>
     <li>
      <p>
       <strong>
        配置方法
       </strong>
       ：
       <br/>
       1、确定返回结果的编码（可跟开发人员确认，如果是web页面，可查看源文件得知）；
       <br/>
       <img alt="可通过查看源文件得知网页的编码方式" src="https://i-blog.csdnimg.cn/blog_migrate/66153b6e61d6f8e5b0dc3faeff7ae7bc.jpeg">
        <br/>
        2、右键httpSampler，添加“BeanShell PostProcessor”元件；
        <br/>
        <img alt="BeanShell PostProcessor元件" src="https://i-blog.csdnimg.cn/blog_migrate/4e66b5c6da9f62ff104e838c5050d2d1.jpeg"/>
       </img>
      </p>
     </li>
     <li>
      <p>
       设置“BeanShell PostProcessor”；在“Script”中增加“prev.setDataEncoding(“UTF-8”);”
       <br/>
       <img alt="设置BeanShell PostProcessor" src="https://i-blog.csdnimg.cn/blog_migrate/eb8ce8897a4943c7f7a4abac60271962.jpeg">
        <br/>
        4、测试验证，OK，设置完毕。
       </img>
      </p>
     </li>
     <li>
      <p>
       <strong>
        解决办法二
       </strong>
       ：
       <br/>
       修改Jmeter的默认编码方式；
       <br/>
       <strong>
        优点
       </strong>
       ：一次修改，长久使用；
       <br/>
       <strong>
        缺点
       </strong>
       ：如果要测试不同的公司项目，需要多次修改，较麻烦；
       <br/>
       <strong>
        适用范围
       </strong>
       ：测试本公司的项目，毕竟同一个公司的项目基本上都是使用同一个编码的；
      </p>
     </li>
     <li>
      <p>
       <strong>
        配置方法
       </strong>
       ：
       <br/>
       1、打开 %JMeter_HOME%\bin\jmeter.properties文件，搜索“sampleresult”，找到sampler的编码设置代码；修改编码与被测系统同样的编码方式，如UTF-8，去掉“#”（注释符号），保存设置，并重启Jmeter验证即可。
      </p>
     </li>
