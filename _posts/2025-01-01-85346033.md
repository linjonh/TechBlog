    <h4>
     第一章 概论
    </h4>
    <p>
     操作系统的基本概念
    </p>
    <ul>
     <li>
      概念：操作系统是配置在计算机硬件上的第一层软件，是对硬件系统的首次扩充，主要作用是管理设备，提高它们的利用率和系统的吞吐量，并为用户和应用程序提供一个接口
     </li>
     <li>
      目标：方便性、可扩充性、开放性、有效性（提高系统资源的利用率，系统的吞吐量）
     </li>
     <li>
      作用：计算机和用户的接口、资源管理、资源抽象（封装）
     </li>
    </ul>
    <p>
     操作系统的发展过程
    </p>
    <ul>
     <li>
      单道：内存中仅有一道程序
      <ul>
       <li>
        优点：自动、顺序、单道
       </li>
       <li>
        缺点：系统资源得不到充分利用
       </li>
      </ul>
     </li>
     <li>
      多道：内存中按调度算法执行程序
      <ul>
       <li>
        优点：资源利用率高、系统吞吐量达
       </li>
       <li>
        缺点：平均周转时间长、无交互能力
       </li>
      </ul>
     </li>
     <li>
      分时：一台主机上有多个终端，时间片轮转法
      <ul>
       <li>
        优点：多路性、独立性、及时性、交互性
       </li>
      </ul>
     </li>
     <li>
      实时：系统的正确性，取决于逻辑结果和产生结果的时间，允许抢占。最大特点是响应快
     </li>
     <li>
      分时和实时的区别
      <ul>
       <li>
        多路性、独立性、及时性、交互性、可靠性
       </li>
      </ul>
     </li>
    </ul>
    <p>
     操作系统的基本特性
    </p>
    <ul>
     <li>
      并发
      <ul>
       <li>
        并行并发、进程、线程
       </li>
      </ul>
     </li>
     <li>
      共享
      <ul>
       <li>
        互斥共享、同时访问（实质是交替访问）
       </li>
      </ul>
     </li>
    </ul>
    <p>
     操作系统的虚拟技术
    </p>
    <ul>
     <li>
      时分复用技术：利用处理机空闲时间，转去为其他用户服务
      <ul>
       <li>
        虚拟处理机技术：多道程序设计技术
       </li>
       <li>
        虚拟设备技术：将一个IO设备虚拟为多个IO设备，并分配给每一个用户
       </li>
      </ul>
     </li>
     <li>
      空分复用技术：利用存储器空闲时间，转去为其他程序服务
      <ul>
       <li>
        虚拟内存
       </li>
       <li>
        虚拟存储器技术
       </li>
      </ul>
     </li>
     <li>
      异步性：每次运行过程不可预知
     </li>
    </ul>
    <p>
     操作系统的管理功能
    </p>
    <ul>
     <li>
      进程控制：创建、撤销进程
     </li>
     <li>
      进程同步：让多个进程协调进行
     </li>
     <li>
      进程通信
     </li>
     <li>
      调度
     </li>
    </ul>

    <h4>
     第三章 调度与死锁
    </h4>
    <p>
     低级、中级、高级调度定义
    </p>
    <ul>
     <li>
      高级调度：又称长程调度或作业调度，调度对象是作业，功能是用某种调度算法决定就绪队列中那几个作业调入内存
     </li>
     <li>
      中级调度：又称内存调度，目的是为了提升吞吐率和资源利用率，功能是将外存中的程序块调入内存，并加入就绪队列
     </li>
     <li>
      低级调度：又称进程调度或短程调度，调度对象是进程，功能是用算法决定哪个进程获得处理机
     </li>
     <li>
      总结：高级调度→哪些作业调入内存；中级调度→外存到内存；低级调度→哪个进程获得处理机
     </li>
     <li>
      作业，是一个总任务，而进程是作业的各个子任务。高级调度做作业分配，中级调度和低级调度面向进程，分别决定了哪些进程允许进入内存和哪些进程允许获得处理机（处理机包括CPU和主存储器）
     </li>
    </ul>
    <p>
     调度规则
    </p>
    <ul>
     <li>
      CPU利用率、吞吐量、周转时间（完成时间-到达时间）、平均周转时间（
      <img alt="$$ T=\frac{1}{n}[\sum _{i=1}^n T_i] $$" class="mathcode" src="https://i-blog.csdnimg.cn/blog_migrate/5f07be644a8b8c41ead4b60d1610ece9.gif">
       ）、带权周转时间（周转时间/真实执行时间，
       <img alt="$$ value&gt;=1 $$" class="mathcode" src="https://private.codecogs.com/gif.latex?%24%24%20value%3E%3D1%20%24%24">
        ）、平均带权周转时间（
        <img alt="$$ W=\frac{1}{n}[\sum _{i=1}^n \frac{T_i}{T_s}] $$" class="mathcode" src="https://i-blog.csdnimg.cn/blog_migrate/5551c4a47b27b29ec164d4ca46436165.gif">
         ）、等待时间、响应时间
        </img>
       </img>
      </img>
     </li>
    </ul>
    <p>
     算法（重点）
    </p>
    <ul>
     <li>
      先来先服务
     </li>
     <li>
      短作业优先
     </li>
     <li>
      优先级调度
     </li>
     <li>
      时间片轮转法
     </li>
    </ul>
    <p>
     死锁定义
    </p>
    <ul>
     <li>
      若干个进程在争夺资源，最后陷入无限期的阻塞和等待中
     </li>
    </ul>
    <p>
     死锁条件
    </p>
    <ul>
     <li>
      互斥
     </li>
     <li>
      不剥夺
     </li>
     <li>
      请求与保持
     </li>
     <li>
      循环等待
     </li>
    </ul>
    <p>
     死锁避免方法-银行家算法（重点）
    </p>
    <ul>
     <li>
      贪心首先完成能完成的
     </li>
    </ul>
    <p>
     死锁预防、解除方法
    </p>
    <ul>
     <li>
      预防：打破除了互斥的其他三个条件
     </li>
     <li>
      解除：抢占资源、终止进程
     </li>
    </ul>

    <h4>
     第四章 存储器管理
    </h4>
    <p>
     连续分配
    </p>
    <ul>
     <li>
      概念：为了能将程序装入内存，必须为它分配一定大小的内存空间，连续分配方式是最早出现的存储器分配方式
     </li>
     <li>
      类型
      <ul>
       <li>
        单一连续分配：系统分为用户区和系统区
       </li>
       <li>
        固定分区分配
       </li>
       <li>
        动态分区分配：首次适应（先找到先用）、最坏适应（找到最大的用）、最佳适应算法（找到最小的用）
       </li>
       <li>
        动态可重定位分配算法：紧凑（去除小碎片，相当于平移）、重定位（重定位寄存器存在进程在内存的首地址）
       </li>
      </ul>
     </li>
    </ul>
    <p>
     基本分页式（重点）
    </p>
    <ul>
     <li>
      物理地址和逻辑地址的区别
      <ul>
       <li>
        逻辑地址：理解为虚拟地址，包括了偏移量
       </li>
       <li>
        物理地址：数据真实存放的地方
       </li>
      </ul>
     </li>
     <li>
      页表定义及作用
      <ul>
       <li>
        定义：为了能在内存中找到每个页面对应的物理块
       </li>
       <li>
        作用：实现从页号到物理块号的地址映射
       </li>
      </ul>
     </li>
     <li>
      地址变换机构
      <ul>
       <li>
        基本地址变换机构：比下图少了个快表的部分
       </li>
       <li>
        具有快表的地址变换机构：实质上快表先建立了一个映射表，节约了去页表找页号的时间，相当于预处理
       </li>
       <li>
        <img alt="" class="has" height="262" src="https://i-blog.csdnimg.cn/blog_migrate/3e4f78a4373e78f422081c25f8c672f3.png" width="453"/>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     基本分段式
    </p>
    <ul>
     <li>
      段表作用：逻辑段到物理内存区的映射
     </li>
     <li>
      地址变换机构：段表寄存器
     </li>
     <li>
      <img alt="" class="has" height="317" src="https://i-blog.csdnimg.cn/blog_migrate/16a74e13f00aabc9e5142ed761e7180f.png" width="504"/>
     </li>
    </ul>
    <p>
     分段式和分页式的区别
    </p>
    <ul>
     <li>
      分页管理
      <ul>
       <li>
        没有外碎片，但会产生内碎片（页可能填不满）
       </li>
       <li>
        一维地址
       </li>
       <li>
        主要用于实现虚拟内存
       </li>
       <li>
        以页为单位，大小静态
       </li>
      </ul>
     </li>
     <li>
      分段管理
      <ul>
       <li>
        没有内碎片，但会有外碎片
       </li>
       <li>
        二维地址
       </li>
       <li>
        以段为单位，大小动态
       </li>
      </ul>
     </li>
    </ul>
    <p>
     段页式的实现过程
    </p>
    <ul>
     <li>
      程序分成若干个段，每个段对应一个页表
     </li>
     <li>
      <img alt="" class="has" height="288" src="https://i-blog.csdnimg.cn/blog_migrate/fa4b24efa47586e472a371c9c52b6951.png" width="574"/>
     </li>
    </ul>
    <p>
     虚拟存储器
    </p>
    <ul>
     <li>
      定义：具有调入功能和置换功能，能从逻辑上对内存容量加以扩充的一种存储器系统
     </li>
     <li>
      页面置换算法
      <ul>
       <li>
        最佳置换算法
       </li>
       <li>
        先进先出算法
       </li>
       <li>
        最近最少使用算法
       </li>
       <li>
        最少使用次数算法
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     第五章 输入输出系统
    </h4>
    <p>
     缓冲管理
    </p>
    <ul>
     <li>
      缓冲区作用
      <ul>
       <li>
        缓和CPU和IO设备速度不匹配的矛盾
       </li>
       <li>
        提高CPU和IO设备之间的并行性
       </li>
       <li>
        减少CPU的中断频率，放宽对CPU中断响应时间的限制
       </li>
       <li>
        解决数据粒度（数据单元大小）不匹配的问题
       </li>
      </ul>
     </li>
     <li>
      单缓冲
      <ul>
       <li>
        <img alt="" class="has" height="321" src="https://i-blog.csdnimg.cn/blog_migrate/473cd45284baf4032378636328fff4cb.png" width="533"/>
       </li>
      </ul>
     </li>
     <li>
      双缓冲（缓冲对换）：减少了用户响应时间
      <ul>
       <li>
        <img alt="" class="has" height="422" src="https://i-blog.csdnimg.cn/blog_migrate/a5c9f3a3e3ae6b65f3bf3816de789ffc.png" width="615"/>

       </li>
      </ul>
     </li>
     <li>
      循环缓冲区
      <ul>
       <li>
        三个指针
        <ul>
         <li>
          nextg：指向装了数据的空缓存区
         </li>
         <li>
          nexti：空缓存区
         </li>
         <li>
          Current：正在使用的缓存区
         </li>
        </ul>
       </li>
       <li>
        使用
        <ul>
         <li>
          Getbuf，装载和读取数据
         </li>
         <li>
          Releasebuf，释放缓存区
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      缓冲池
      <ul>
       <li>
        组成：emq,inq,outq
       </li>
       <li>
        工作方式
        <ul>
         <li>
          收容收入Getbuf（emq）
         </li>
         <li>
          提取输入Getbuf（inq）
         </li>
         <li>
          收容输出Getbuf（enq）
         </li>
         <li>
          提取输出Getbuf（outq）
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      SPOOLING系统（重点）
      <ul>
       <li>
        假脱机技术：利用专门的外围机，当需要用数据的时候，可以直接从磁盘读入 (我感觉就是一个缓存区吧。
       </li>
       <li>
        组成
        <ul>
         <li>
          输入井和输出井：模拟输入的磁盘
         </li>
         <li>
          输入缓冲区和输出缓冲区：输入缓冲区，在给输入井之前，先放缓冲区缓一缓
         </li>
         <li>
          输入进程和输出进程：模拟外围机，从外部输入数据到缓冲区
         </li>
         <li>
          井管理程序：用于控制作业与磁盘井之间信息的交换
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     设备分配（请求与保持条件）
    </p>
    <ul>
     <li>
      安全算法：每当进程发出I/O请求后，便进入阻塞状态
     </li>
     <li>
      不安全算法：仅当进程所请求的设备已被另一个进程占用时，才进入阻塞状态
     </li>
    </ul>
    <p>
     磁盘调度算法
    </p>
    <p>
     先来先服务、最短寻道时间优先（贪心）、SCAN（单向再折返）、CSCAN（单向到最后一个，再跑到另一端的第一个）
    </p>
