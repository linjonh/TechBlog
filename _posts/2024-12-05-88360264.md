    <h2>
     <a id="_CPU_1">
     </a>
     客户机操作系统已禁用 CPU。请关闭或重置虚拟机。
    </h2>
    <p>
     虚拟机安装内核的时候遇到这个问题百度了很久都没有找到合适的答案，很多都是重置 的解决方法，所以解决后在这里记录一下，毕竟是困扰了我将近一周的问题。
    </p>
    <h3>
     <a id="_5">
     </a>
     前提条件
    </h3>
    <p>
     VMware Workstation pro 12.1.0下安装Ubuntu18.04
     <br/>
     安装内核：linux-4.20.13
    </p>
    <h3>
     <a id="_10">
     </a>
     出现错误的步骤
    </h3>
    <p>
     编译安装完内核重启虚拟机后打不开，虚拟机报错“客户机操作系统已禁用 CPU。请关闭或重置虚拟机。”
     <br/>
     错误信息如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a57bf71224b1186873ea5a5659bfd85d.jpeg">
      <br/>
      这个小框框叉掉是这样的：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f6ae8bb908283b1112bdd0cb3db3826b.jpeg"/>
     </img>
    </p>
    <h3>
     <a id="_17">
     </a>
     解决措施
    </h3>
    <p>
     一开始呢看到这个错误提示只想到是cpu禁用的问题检查了一下进程都是启用，于是就百度找各种解决方法，但是发现要么找不到要么就是重置，重置那不是完蛋嘛，辛辛苦苦倒腾好几个小时。
     <br/>
     后来知乎看到一条说Ubuntu系统可以启动说明不是禁用的原因，在经历了第30+次开机时忽然看到在上上图出现之前的一条“……BIOS corrupted……”这样的报错（当时太激动了忘了截图）感觉可能是因为内存不够的原因，于是将2G内存改成了4G，果然重启成功了。
     <br/>
     第一次觉得需要记录一下了赶紧写下来。。希望能对大家有所帮助。
    </p>
