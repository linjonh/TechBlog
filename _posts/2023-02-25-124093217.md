    <h3>
     <a id="java_0">
     </a>
     嵌入式为什么要学习java
    </h3>
    <h3>
     <a id="_2">
     </a>
     首先来看一下传统的嵌入式学习路线：
    </h3>
    <ul>
     <li>
      C语言 &gt;&gt; 应用：C51,STM32
     </li>
     <li>
      linux系统编程
     </li>
     <li>
      C++ &gt;&gt; 应用：QT
     </li>
     <li>
      ARM开发
     </li>
    </ul>
    <h3>
     <a id="UI_9">
     </a>
     在系统的UI层面上：
    </h3>
    <ul>
     <li>
      无操作系统的裸机开发：用C语言手撕图像像素（优点：响应速度快，对硬件资源要求低。缺点：UI界面丑）
     </li>
     <li>
      带操作系统的（linux）：c、MINIGUI、GTK图像库、NCURSE
      <br/>
      -带操作系统的（linux）：C++、QT
     </li>
    </ul>
    <h3>
     <a id="_14">
     </a>
     目前趋势：
    </h3>
    <p>
     安卓平台普及（刷卡机、点歌机）
     <br/>
     鸿蒙系统：内嵌安卓
    </p>
    <h3>
     <a id="_18">
     </a>
     总结：
    </h3>
    <p>
     在目前来看，C++QT已经是个老产品，被安卓系统完美的替代了，不带操作系统的裸机开发，能满足的用户需求有限且开发起来工作量很大。所以安卓平台的开发对嵌入式的学习是有必要的。
    </p>
