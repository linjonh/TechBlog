    <h1 class="title-article" id="articleContentId">
     RAD（Rapid Application Develop，快速应用开发）模型
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <div>
     ﻿﻿
    </div>
    <div class="para">
     RAD（Rapid Application Develop，
     <a href="https://baike.baidu.com/item/%E5%BF%AB%E9%80%9F%E5%BA%94%E7%94%A8%E5%BC%80%E5%8F%91" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        快速应用开发
       </span>
      </u>
     </a>
     ）模型是一个增量型的软件开发过程模型，强调极短的开发周期。该模型是
     <a href="https://baike.baidu.com/item/%E7%80%91%E5%B8%83%E6%A8%A1%E5%9E%8B" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        瀑布模型
       </span>
      </u>
     </a>
     的一个“高速”变种，通过大量使用可复用
     <a href="https://baike.baidu.com/item/%E6%9E%84%E4%BB%B6" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        构件
       </span>
      </u>
     </a>
     ，采用基于构件的建造方法赢得了快速开发。如果正确地理解了需求，而且约束了项目的范围，利用这种模型可以很快创建出功能完善的信息系统。其流程从业务建模开始，随后是数据建模、过程建模、应用生成、测试及反复。RAD目的是快速发布系统方案，而技术上的优美相对发布的速度来说是次要的。
    </div>
    <div class="para">
     <div class="lemma-picture text-pic layout-right" style="width:220px;float:right;">
      <a class="image-link" href="https://baike.baidu.com/pic/RAD/2262445/0/a84052083c15a5c50a7b82c1?fr=lemma&amp;ct=single" rel="nofollow" style="width:220px;height:160px;" title="RAD开发模型图">
       <img alt="RAD开发模型图" src="https://gss2.bdstatic.com/9fo3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D220/sign=03c9399b39dbb6fd215be2243924aba6/a8773912b31bb051aa0a3e48367adab44aede0a0.jpg" style="width:220px;height:160px;"/>
      </a>
      <span class="description">
       RAD开发模型图
      </span>
     </div>
     采用RAD模型的
     <a href="https://baike.baidu.com/item/%E8%BD%AF%E4%BB%B6%E8%BF%87%E7%A8%8B" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        软件过程
       </span>
      </u>
     </a>
     如图所示。
    </div>
    <div class="para">
     RAD模型各个活动期所要完成的任务如下。
    </div>
    <div class="para">
     ⑴业务建模
    </div>
    <div class="para">
     确定驱动业务过程运作的信息、要生成的信息、如何生成、信息流的去向及其处理等，可以辅之以
     <a href="https://baike.baidu.com/item/%E6%95%B0%E6%8D%AE%E6%B5%81%E5%9B%BE" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        数据流图
       </span>
      </u>
     </a>
     。
    </div>
    <div class="para">
     ⑵数据建模
    </div>
    <div class="para">
     为支持业务过程的数据流查找
     <a href="https://baike.baidu.com/item/%E6%95%B0%E6%8D%AE%E5%AF%B9%E8%B1%A1" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        数据对象
       </span>
      </u>
     </a>
     集合、定义数据对象属性，并与其他数据对象的关系构成数据模型，可辅之以E-R图。
    </div>
    <div class="para">
     ⑶过程建模
    </div>
    <div class="para">
     使数据对象在信息流中完成各业务功能，创建过程以描述数据对象的增加、修改、删除、查找，即细化数据流图中的处理框。
    </div>
    <div class="para">
     ⑷应用程序生成
    </div>
    <div class="para">
     利用第4代语言（4GL）写出处理程序，重用已有
     <a href="https://baike.baidu.com/item/%E6%9E%84%E4%BB%B6" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        构件
       </span>
      </u>
     </a>
     或创建新的可重用构件，利用环境提供的工具自动生成以构造出整个的应用系统。
    </div>
    <div class="para">
     ⑸测试与交付
    </div>
    <div class="para">
     由于大量重用，一般只做
     <a href="https://baike.baidu.com/item/%E6%80%BB%E4%BD%93%E6%B5%8B%E8%AF%95" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        总体测试
       </span>
      </u>
     </a>
     ，但新创建的构件还是要测试的。
    </div>
    <div class="para">
     与
     <a href="https://baike.baidu.com/item/%E7%80%91%E5%B8%83%E6%A8%A1%E5%9E%8B" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        瀑布模型
       </span>
      </u>
     </a>
     相比，RAD模型不采用传统的第3代
     <a href="https://baike.baidu.com/item/%E7%A8%8B%E5%BA%8F%E8%AE%BE%E8%AE%A1%E8%AF%AD%E8%A8%80" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        程序设计语言
       </span>
      </u>
     </a>
     来创建软件，而是采用基于构件的开发方法复用已有的程序结构（如果可能）或使用可复用构件和或创建可复用的构件（如果需要）。在所有情况下，均使用自动化工具辅助软件创造。很显然，加在一个RAD模型项目上的时间约束需要“一个可伸缩的范围”。如果一个业务能够被模块化使得其中每一个主要功能均可以在不到3个月的时间内完成，则其是RAD的一个候选者。每一个主要功能可由一个单独的RAD组来实现，最后集成起来形成一个整体。
    </div>
    <div class="para">
     按照Wood and Silver （1995） 的观点， RAD组合了5个方面的技术：
    </div>
    <div class="para">
     ⒈进化原型
    </div>
    <div class="para">
     ⒉CASE工具（可进行
     <a href="https://baike.baidu.com/item/%E6%AD%A3%E5%90%91%E5%B7%A5%E7%A8%8B" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        正向工程
       </span>
      </u>
     </a>
     和反向工程）
    </div>
    <div class="para">
     ⒊拥有能使用先进工具的专门人员（一个RAD开发小组）
    </div>
    <div class="para">
     ⒋交互式JAD
    </div>
    <div class="para">
     ⒌时间表
    </div>
    <div class="para">
     RAD模型通过大量使用可复用
     <a href="https://baike.baidu.com/item/%E6%9E%84%E4%BB%B6" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        构件
       </span>
      </u>
     </a>
     加快了开发速度，对信息系统的开发特别有效。但是与所有其他软件过程模型一样，RAD模型也有不足之处：
    </div>
    <div class="para">
     ⒈ 对大型项目而言，RAD需要足够的人力资源。
    </div>
    <div class="para">
     2 、开发人员和客户必须在很短的时间内完成一系列的
     <a href="https://baike.baidu.com/item/%E9%9C%80%E6%B1%82%E5%88%86%E6%9E%90" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        需求分析
       </span>
      </u>
     </a>
     ，任何一方配合不当都会导致RAD项目失败。
    </div>
    <div class="para">
     ⒊ 并非所有系统都适合：不能合理模块化的系统、高性能需求并且要调整构件接口的系统均不适合。　 4 、 RAD只能用于信息系统开发，不适合技术风险很高的情况。当一个新应用要采用很多新技术或当新软件要求与已有的计算机程序的高
     <a href="https://baike.baidu.com/item/%E4%BA%92%E6%93%8D%E4%BD%9C%E6%80%A7" rel="nofollow">
      <u>
       <span style="color:#0066cc;">
        互操作性
       </span>
      </u>
     </a>
     时，这种情况就会发生。
    </div>
