    <h3>
     <a id="_1">
     </a>
     问题说明
    </h3>
    <p>
     我们在项目中通常都会使用到封装好的axios来发送请求，当我们在发送POST请求时结果响应404。
     <br/>
     使用接口测试工具postman、yapi等都可以正常响应。
     <br/>
     <strong>
      报错信息如下：
     </strong>
     <br/>
     <img alt="响应结果" src="https://i-blog.csdnimg.cn/blog_migrate/3d469285dd1214d2e7efa47530fb9594.png">
      <br/>
      <strong>
       请求头：
      </strong>
      <br/>
      <img alt="请求信息" src="https://i-blog.csdnimg.cn/blog_migrate/0bd1a53cc95a84f843be222ec998cb92.png"/>
     </img>
    </p>
    <hr/>
    <h3>
     <a id="___10">
     </a>
     造成问题的原因 –
     <em>
      请求携带的数据格式和后端解析数据格式不一致
     </em>
    </h3>
    <h4>
     <a id="_java_spring_mvc__RequestParam__RequestBody_12">
     </a>
     后端解析：接口使用 java spring mvc，注解使用了
     <code>
      @RequestParam
     </code>
     或
     <code>
      @RequestBody
     </code>
    </h4>
    <blockquote>
     <p>
      RequestParam：在这种情况下只能从请求的地址中获取参数，也就是只能从
      <code>
       username=zhangsan&amp;password=123
      </code>
      这种字符串中解析出参数。
      <br/>
      RequestBody：这种情况下只能从请求体的
      <code>
       form-data
      </code>
      中获取数据，axios默认请求体为
      <code>
       Request Payload
      </code>
     </p>
    </blockquote>
    <h4>
     <a id="_ContentTypeapplicationjsoncharsetUTF8_15">
     </a>
     前端请求：看上图的请求头信息中的
     <code>
      Content-Type：application/json;charset=UTF-8
     </code>
     ，
    </h4>
    <blockquote>
     <p>
      <strong>
       我们在使用axios发送post请求时，一般都是以对象的形式发送请求
      </strong>
      <br/>
      <em>
       发送axios请求体的格式为
       <code>
        Request Payload
       </code>
       。
      </em>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ec1dff84b1277b6ef4aa21f6ae257753.png">
       <br/>
       <em>
        在发送ajax请求时请求体为
        <code>
         form-data
        </code>
       </em>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cf6bafcd55db1f2b86cf8f74780ecdc2.png">
        <br/>
        <strong>
         下图为axios源码信息，axios库默认会帮我们转换请求和响应，并自动转换为
         <code>
          JSON
         </code>
         数据
        </strong>
        <br/>
        <img alt="axios/default.js" src="https://i-blog.csdnimg.cn/blog_migrate/371176a44525c542509ad37794fe7bcb.png"/>
       </img>
      </img>
     </p>
    </blockquote>
    <p>
     <strong>
      这就造成了前端发送json形式的请求，而后端从请求的地址中解析字符串。两种数据格式不一致
     </strong>
    </p>
    <hr/>
    <h3>

    </h3>
