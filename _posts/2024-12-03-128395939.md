    <h4>
     下载安装程序
    </h4>
    <p>
     <a href="https://www.python.org/ftp/python/2.7.15/python-2.7.15.amd64.msi" rel="nofollow" title="https://www.python.org/ftp/python/2.7.15/python-2.7.15.amd64.msi">
      https://www.python.org/ftp/python/2.7.15/python-2.7.15.amd64.msi
     </a>
    </p>
    <p>
     从https://www.python.org/downloads/release/python-2715/下载安装程序
    </p>
    <p>
     <img alt="" height="456" src="https://i-blog.csdnimg.cn/blog_migrate/97af091a5c82a024a7b86a55fc5f686b.png" width="701"/>
    </p>
    <h4>
     <a id="_8">
     </a>
     启动安装程序
    </h4>
    <p>
    </p>
    <p>
     <img alt="" height="427" src="https://i-blog.csdnimg.cn/blog_migrate/1316704c04cd5fb4096212ff90e3ee8c.png" width="494"/>
    </p>
    <h4>
     <a id="_10">
     </a>
     指定安装目录
    </h4>
    <p>
     这里没有使用安装程序默认指定的安装目录，而是自己定制的目录 C:\Python\Python2.7.15
    </p>
    <p>
     <img alt="" height="425" src="https://i-blog.csdnimg.cn/blog_migrate/46c030c34baee24649400bb66900a4e5.png" width="495">
     </img>
    </p>
    <h4>
     <a id="_13">
     </a>
     安装选项
    </h4>
    <p>
     点击“Disk Usage”按钮，会弹出一个新窗口，告诉你Python 2.7.15全部安装需要至少102MB的磁盘。未截图。
    </p>
    <p>
     <img alt="" height="435" src="https://i-blog.csdnimg.cn/blog_migrate/57d92b2c0fc2817daf0c98f6222c0b81.png" width="498">
     </img>
    </p>
    <h4>
     <a id="_17">
     </a>
     启用环境变量
    </h4>
    <p>
     点击右侧滚动条到最底部，启用 “Add python.exe to Path”。
    </p>
    <p>
     <img alt="" height="438" src="https://i-blog.csdnimg.cn/blog_migrate/1ed0fe94b83c64e8299404886c80bb16.png" width="504">
     </img>
    </p>
    <h4>
     <a id="_20">
     </a>
     高级选项
    </h4>
    <p>
     点击"Advanced"按钮，会弹出高级设置选项，默认不编译.py文件。本次安装采用默认设置。
    </p>
    <p>
     <img alt="" height="432" src="https://i-blog.csdnimg.cn/blog_migrate/c359d9277edcd85bd4c7d18b8f4fae9e.png" width="499">
     </img>
    </p>
    <h4>
     <a id="_24">
     </a>
     安装完毕
    </h4>
    <p>
    </p>
    <p>
     <img alt="" height="438" src="https://i-blog.csdnimg.cn/blog_migrate/c2fbd37f5962e74a89390ffb7c4e651f.png" width="505">
     </img>
    </p>
    <h3>
     <a id="_26">
     </a>
     安装成功验证
    </h3>
    <h4>
     <a id="Windows_27">
     </a>
     检查Windows环境变量
    </h4>
    <p>
     可以看到Python安装程序已经将
     <br/>
     <span style="color:#fe2c24;">
      <code>
       C:\Python\Python2.7.15\;C:\Python\Python2.7.15\Scripts
      </code>
     </span>
     <br/>
     这两个路径添加到了PATH环境变量中。
    </p>
    <p>
     <img alt="" height="373" src="https://i-blog.csdnimg.cn/blog_migrate/2b5f5ee005e23555723c59b22ff9c2df.png" width="709"/>
    </p>
    <h4>
     <a id="Python_33">
     </a>
     确认Python版本
    </h4>
    <p>
     确认Python所在的目录，版本号，且可以正常运行。
    </p>
    <p>
     <img alt="" height="374" src="https://i-blog.csdnimg.cn/blog_migrate/ab9f841762232fa87de3fb7bec9d62f1.png" width="727"/>
    </p>
    <h3>
     <a id="Python_36">
     </a>
     探索Python的安装目录
    </h3>
    <h4>
     <a id="_37">
     </a>
     安装目录结构
    </h4>
    <p>
     接下来我们看看Python 2.7.15安装后的详细目录结构以及功用。
     <br/>
     首先查看一下Python的安装目录C:\Python\Python2.7.15。
     <br/>
     <img alt="" height="422" src="https://i-blog.csdnimg.cn/blog_migrate/826ff216513be8b6102b70fac4979c41.png" width="712"/>
     <br/>
     在安装目录下，可以看到两个Windows可执行文件，python.exe与pythonw.exe。这两个文件都是Python解释器，功能大同小异。
     <span style="color:#fe2c24;">
      由于Python安装目录已经添加到了PATH环境变量中，所以可以在任何目录下执行python.exe或pythonw.exe命令
     </span>
     。
    </p>
    <blockquote>
     <ul>
      <li>
       python.exe — 当调用python.exe会生成一个Windows命令行窗口，
      </li>
      <li>
       pythonw.exe — 当运行python程序，但是不希望出现一个黑黑的命令行窗口时，可以使用pythonw.exe来运行程序。
      </li>
     </ul>
    </blockquote>
    <p>
     该目录下，其他的文件还包括
    </p>
    <ul>
     <li>
      LICENSE.txt — 简要介绍了Python的历史，作者，开源模式，以及License条款等，自从版本2.2以后，Python的License就与GPL兼容了，但是Python并不是基于GPL发行的。
     </li>
     <li>
      NEWS.txt — 简单介绍了Python的发行日期，以及每次发行对应的bpo-xxxxx号。
      <br/>
      例如，
     </li>
    </ul>
    <pre><code>What's New in Python 2.7.15 final?
==================================
*Release date: 2018-04-29*

## Core and Builtins

- bpo-33374: Tweak the definition of PyGC_Head, so compilers do not believe
it is always 16-byte aligned on x86. This prevents crashes with more
aggressive optimizations present in GCC 8.
</code></pre>
<ul>
<li>
README.txt — 这是一个非常重要的文件，里面详细介绍了
<ul>
<li>
Python 当前版本的新特性，
</li>
<li>
如何学习 Python，两个重要的网站http://docs.python.org/与http://wiki.python.org/moin/PythonBooks，
</li>
<li>
如何向 Python 开发做贡献http://docs.python.org/devguide/，
</li>
<li>
新闻组/邮件列表http://www.python.org/community/lists/，
</li>
<li>
bug 报告http://bugs.python.org/，
</li>
<li>
补丁与贡献http://www.python.org/dev/peps/，以及
</li>
<li>
如何从源代码编译 Python，优化 Python，
</li>
<li>
Python 支持的平台与不支持的平台等。
</li>
</ul>
</li>
</ul>
<h4>
<a id="Python_70">
</a>
Python 安装目录下的子目录
</h4>
<p>
<img alt="" height="351" src="https://i-blog.csdnimg.cn/blog_migrate/c4a2a31357180d99b3288f67dee15b83.png" width="663"/>
</p>
<h4>
<a id="DLLs_72">
</a>
DLLs 子目录
</h4>
<p>
<span style="color:#fe2c24;">
Python 的*.pyd(Python 动态模块)文件与几个 Windows 的*.dll(动态链接库)文件。
</span>
<br/>
注：.pyd 文件并不是用 Python 语言编写的，据说是使用 D 语言（C/C++语言的进化版）。
</p>
<p>
<img alt="" height="475" src="https://i-blog.csdnimg.cn/blog_migrate/3712d743cd33728641b901902948d094.png" width="716"/>
</p>
<h4>
<a id="Doc_77">
</a>
Doc 子目录
</h4>
<p>
在 Windows 平台上，只有一个 python2715.chm 文件，里面集成了 Python 的所有文档，双击即可打开阅读，非常方便。
</p>
<p>
<img alt="" height="526" src="https://i-blog.csdnimg.cn/blog_migrate/0a64a8a0c63d88d88920d0e05f50be8d.png" width="708"/>
</p>
<h4>
<a id="include_80">
</a>
include 子目录
</h4>
<p>
Python 的 C 语言接口头文件，当在 C 程序中集成 Python 时，会用到这个目录下的头文件。
</p>
<p>
<img alt="" height="454" src="https://i-blog.csdnimg.cn/blog_migrate/be96460136bf2f7ae659430561f46207.png" width="707"/>
</p>
<h4>
<a id="Lib_83">
</a>
Lib 子目录
</h4>
<p>
<span style="color:#fe2c24;">
Python 自己的标准库，包，测试套件等
</span>
，非常多的内容。看看以后能不能逐一讲解。
</p>
<p>
<img alt="" height="291" src="https://i-blog.csdnimg.cn/blog_migrate/73dd437dbd2318114c0d63feb889717c.png" width="718"/>
</p>
<h4>
<a id="libs_86">
</a>
libs 子目录
</h4>
<p>
这个目录是 Python 的 C 语言接口库文件。
</p>
<p>
<img alt="" height="401" src="https://i-blog.csdnimg.cn/blog_migrate/2b737751c9cd527c32f2f3dc6a1a1660.png" width="708"/>
</p>
<h4>
<a id="Scripts_89">
</a>
Scripts 子目录
</h4>
<p>
<span style="color:#fe2c24;">
pip 可执行文件的所在目录，通过 pip 可以安装各种各样的 Python 扩展包。这也是为什么这个目录也需要添加到 PATH 环境变量中的原因。
</span>
</p>
<p>
<img alt="" height="307" src="https://i-blog.csdnimg.cn/blog_migrate/e900267c05361aefc76099d287c906ab.png" width="710"/>
</p>
<h4>
<a id="tcl_92">
</a>
tcl 子目录
</h4>
<p>
Python 与 TCL 的结合。以后再讲。
</p>
<p>
<img alt="" height="320" src="https://i-blog.csdnimg.cn/blog_migrate/0d4eed46af9109ffbd7e364667a59284.png" width="709"/>
</p>
<h4>
<a id="Tools_96">
</a>
Tools 子目录
</h4>
<p>
估计是一堆工具。有的子目录下有 README.txt 文件，可以查看具体的工具用途。
</p>

<img alt="" height="307" src="https://i-blog.csdnimg.cn/blog_migrate/203f6ec48646631f269e4ccafb1c2194.png" width="712"/>
