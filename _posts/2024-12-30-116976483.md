    <p>
     小程序在开发完成后，需要
     <strong>
      上传代码，设为体验版本(功能测试)，提交审核，发布上线
     </strong>
     这几个基本步骤。接下来用自己的亲身经历一一详细介绍！！！
    </p>
    <h2>
     <a id="_1">
     </a>
     小程序发布上线全流程
    </h2>
    <h3>
     <a id="1_2">
     </a>
     1.上传代码
    </h3>
    <p>
     在
     <strong>
      微信开发者工具
     </strong>
     的
     <strong>
      右上角上传
     </strong>
     处上传全部代码。如下图：
     <br/>
     <img alt="上传代码" src="https://i-blog.csdnimg.cn/blog_migrate/368efbaa8d6be027c3600b3edba8d601.png"/>
    </p>
    <blockquote>
     <p>
      如果小程序中涉及到一些
      <strong>
       request的请求接口
      </strong>
      ，记得把
      <strong>
       不校验合法域名
      </strong>
      勾选上。另外注意，小程序的接口
      <strong>
       不允许http，只支持https
      </strong>
      。仅仅是https还不够，
      <strong>
       正式版和体验版
      </strong>
      上的接口功能实现还需要将接口地址添加到
      <strong>
       开发管理——开发设置——服务器域名——request合法域名
      </strong>
      中去。否则，手机
      <strong>
       预览或者在体验版
      </strong>
      小程序中
      <strong>
       可能无法实现接口
      </strong>
      功能。
     </p>
    </blockquote>
    <p>
     上传的时候会让你
     <strong>
      设置版本号
     </strong>
     ，第一次一般都是
     <strong>
      1.0.0
     </strong>
     ，按照修改代码的大小以及功能对应的修改版本号。如下图：
     <br/>
     <img alt="上传代码的时候设置版本号" src="https://i-blog.csdnimg.cn/blog_migrate/7bacf82f6148a6a009da85d4669aae3d.png"/>
    </p>
    <blockquote>
     <p>
      上传完成后会让你下载一个什么
      <strong>
       sourcemap的压缩包
      </strong>
      ，下不下都无所谓。另外，上传过程中可能提示
      <strong>
       node_modules包没有上传
      </strong>
      ，不管它就完事了。
     </p>
    </blockquote>
    <h3>
     <a id="2_10">
     </a>
     2.设为体验版本
    </h3>
    <p>
     上传完成后，登陆
     <a href="https://mp.weixin.qq.com/" rel="nofollow">
      微信公众平台
     </a>
     ，登上自己的小程序，然后在
     <strong>
      版本管理
     </strong>
     就能看到自己刚刚上传的版本了。如下图：
     <br/>
     <img alt="上传后的版本" src="https://i-blog.csdnimg.cn/blog_migrate/e6bdb2c3c135d1dcc1f0bea30978e285.png"/>
    </p>
    <blockquote>
     <p>
      我们可以将上传的版本设置为
      <strong>
       体验版本
      </strong>
      ，这样更方便地
      <strong>
       测试我们的小程序的功能
      </strong>
      (下次上传新的代码时会将新上传的设置为体验版本)。点击
      <strong>
       体验版
      </strong>
      就能看到体验版的小程序的二维码了。
     </p>
    </blockquote>
    <h3>
     <a id="3_15">
     </a>
     3.提交审核
    </h3>
    <p>
     关于
     <strong>
      个人主体小程序的类目
     </strong>
     ，可以去下面看看：
     <br/>
     <a href="https://developers.weixin.qq.com/miniprogram/product/reject.html#_2-1-%E5%B0%8F%E7%A8%8B%E5%BA%8F%E7%9A%84%E7%B1%BB%E7%9B%AE%E8%A6%81%E5%92%8C%E8%87%AA%E8%BA%AB%E6%89%80%E6%8F%90%E4%BE%9B%E7%9A%84%E6%9C%8D%E5%8A%A1%E4%B8%80%E8%87%B4%E3%80%82" rel="nofollow">
      小程序常见被拒绝情形
     </a>
     <br/>
     <a href="https://developers.weixin.qq.com/miniprogram/product/reject.html#_2-%E6%9C%8D%E5%8A%A1%E7%B1%BB%E7%9B%AE%E5%AE%A1%E6%A0%B8" rel="nofollow">
      微信小程序审核规范规则
     </a>
    </p>
    <p>
     一般如果你的小程序涉及
     <strong>
      音乐，视频或者金融
     </strong>
     等，都是
     <strong>
      不允许
     </strong>
     的。需要注册
     <strong>
      企业主体
     </strong>
     。接下来就是重点：怎么绕过小程序的审核？
    </p>
    <p>
     <mark>
      绕过小程序的审核
     </mark>
    </p>
    <blockquote>
     <p>
      注意，小程序都是
      <strong>
       人工审核
      </strong>
      的，他们
      <strong>
       只看页面不看代码
      </strong>
      。并且你的
      <strong>
       所有页面都能进去
      </strong>
      ，不管你有没有设置路由跳转。申明：这样的做法我在查资料的时候看到
      <strong>
       可能会被封号
      </strong>
      ，不过我觉得应该是
      <code>
       小程序号
      </code>
      ，不是
      <code>
       微信号
      </code>
      。如果封号了我会第一时间告诉你们。😂😂😂
     </p>
    </blockquote>
    <p>
     下面就是绕过小程序审核的
     <mark>
      详细步骤
     </mark>
     ：
    </p>
    <p>
     1.在小程序的入口文件
     <code>
      app.js
     </code>
     的
     <code>
      globalData
     </code>
     全局变量中设置一个变量
     <code>
      isExamine
     </code>
     ，初始值为
     <code>
      true
     </code>
     。
    </p>
    <pre><code class="prism language-javascript">globalData<span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 检测是否在审核中</span>
    isExamine<span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
    <span class="token operator">...</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     2.接着还是在入口文件中的
     <code>
      onLaunch
     </code>
     方法中写这样一段代码：
    </p>
    <pre><code class="prism language-javascript">  <span class="token comment">//进入小程序时就会执行</span>
  <span class="token function-variable function">onLaunch</span><span class="token operator">:</span> <span class="token keyword">function</span> <span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// console.log('进入小程序')</span>
    <span class="token comment">// console.log(this.globalData.isExamine)</span>
    <span class="token comment">// 获取当前时间</span>
    <span class="token keyword">var</span> nowTime  <span class="token operator">=</span> Date<span class="token punctuation">.</span><span class="token function">parse</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token class-name">Date</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span>
    <span class="token keyword">var</span> delineTime <span class="token operator">=</span> Date<span class="token punctuation">.</span><span class="token function">parse</span><span class="token punctuation">(</span><span class="token string">'2021-5-19'</span><span class="token punctuation">)</span>
    <span class="token comment">// console.log(nowTime &gt; delineTime)</span>

      <span class="token keyword">this</span><span class="token punctuation">.</span>globalData<span class="token punctuation">.</span>isExamine <span class="token operator">=</span> <span class="token boolean">false</span>
    <span class="token punctuation">}</span> 
<span class="token punctuation">}</span>

    <blockquote>
     <p>
      这段代码的目的就是，当
      <strong>
       审核员或者用户
      </strong>
      打开小程序时获取到
      <strong>
       当前时间戳
      </strong>
      ，如果比你设置的一个时间戳(这里我设置的是
      <strong>
       2021-5-19
      </strong>
      )要
      <strong>
       大
      </strong>
      的话，说明打开该小程序的是
      <strong>
       用户
      </strong>
      ，
      <code>
       isExamine
      </code>
      设为
      <code>
       false
      </code>
      ；如果要比设置的时间戳小，说明打开此小程序的是审核员，那么
      <code>
       isExamine
      </code>
      的值不变，还是
      <code>
       true
      </code>
      。一般审核挺快的，一天左右就可以了，为了保险，加入你是15号上传提交审核的代码，那么你把日期设置17号或者18号，那么当审核员打开你的小程序的时候，全局变量
      <code>
       isExamine
      </code>
      的值是
      <code>
       true
      </code>
      ，如果过了17号或者18号的零点，那么全局变量
      <code>
       isExamine
      </code>
      的值就会变成
      <code>
       false
      </code>
      ，表示已经过了审核。这就是绕过审核的思路。
     </p>
    </blockquote>
    <p>
     3.接下来在每个页面上都是一样的做法
     <br/>
     WXML：
    </p>
    <pre><code class="prism language-javascript"><span class="token operator">&lt;</span><span class="token operator">!</span><span class="token operator">--</span> 给审核看的代码 <span class="token operator">--</span><span class="token operator">&gt;</span>

<span class="token operator">&lt;</span><span class="token operator">/</span>view<span class="token operator">&gt;</span>

    <p>
     JS：我只贴上关键代码了
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">const</span> app <span class="token operator">=</span> <span class="token function">getApp</span><span class="token punctuation">(</span><span class="token punctuation">)</span>

    <p>
     这样就能绕过审核了。
    </p>
    <blockquote>
     <p>
      另外，注意小程序底部自带的Tab栏，Tab栏的文字，以及每个页面的标题，这些地方
      <strong>
       不要写上音乐，视频，mv，歌单
      </strong>
      这种敏感词。Tab栏的图标也是
      <strong>
       不要贴上音乐，mv
      </strong>
      的标志。
     </p>
    </blockquote>
    <p>
     我前前后后一共提交了四五次代码，最后总算过了😂😂😂。我做的是一个仿网易云音乐的小程序。
    </p>
    <p>
     <mark>
      最后，如果还是不幸的没通过(会微信发消息通知你)，那么你可以尝试不接受审核结果，发表自己自己的看法，总之就是硬说自己的小程序没有违规就完事了。
     </mark>
    </p>
