    <h2 class="title" id="6115">
     嵌入式系统
    </h2>
    <div class="mod-top" id="card-container">
     <div class="card-summary nslog-area clearfix">
      <div class="card-summary-content">
       <div class="para">
        嵌入式系统是一种专用的计算机系统，作为装置或设备的一部分。通常，嵌入式系统是一个控制程序存储在ROM中的嵌入式处理器控制板。事实上，所有带有数字接口的设备，如手表、微波炉、录像机、汽车等，都使用嵌入式系统，有些嵌入式系统还包含操作系统，但大多数嵌入式系统都是是由单个程序实现整个控制逻辑。
       </div>
      </div>
     </div>
    </div>
    <div class="clear">
    </div>
    <div class="lemma-main-content rainbowlemma-0-" id="lemmaContent-0">
     <h3 class="headline-1">
      <span class="headline-content">
       简介
      </span>
     </h3>
     <div class="para">
      嵌入式系统（Embedded system），是一种“完全嵌入受控器件内部，为特定应用而设计的专用
      <a href="http://baike.baidu.com/view/3314.htm" rel="nofollow noopener noreferrer" target="_blank">
       计算机
      </a>
      系统”，根据英国电器工程师协会（ U.K. Institution of Electrical Engineer）的定义，嵌入式系统为控制、监视或辅助设备、机器或用于工厂运作的设备。与
      <a href="http://baike.baidu.com/view/920814.htm" rel="nofollow noopener noreferrer" target="_blank">
       个人计算机
      </a>
      这样的通用
      <a href="http://baike.baidu.com/view/3314.htm" rel="nofollow noopener noreferrer" target="_blank">
       计算机
      </a>
      系统不同，嵌入式系统通常执行的是带有特定要求的预先定义的任务。由于嵌入式系统只针对一项特殊的任务，设计人员能够对它进行优化，减小尺寸降低成本。由于嵌入式系统通常进行大量生产。所以单个的成本节约，能够随着产量进行成百上千的放大。
      <sup>
       [1]
      </sup>
      <a name="ref_[1]_6115">
      </a>
     </div>
     <div class="para">
      嵌入式系统的核心是由一个或几个预先编程好以用来执行少数几项任务的
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      或者
      <a href="http://baike.baidu.com/view/1012.htm" rel="nofollow noopener noreferrer" target="_blank">
       单片机
      </a>
      组成。与通用计算机能够运行用户选择的软件不同，嵌入式系统上的软件通常是暂时不变的；所以经常称为“
      <a href="http://baike.baidu.com/view/33738.htm" rel="nofollow noopener noreferrer" target="_blank">
       固件
      </a>
      ”。
      <sup>
       [2]
      </sup>
      <a name="ref_[2]_6115">
      </a>
     </div>
     <div class="para">
      国内普遍认同的嵌入式系统定义为：以应用为中心，以计算机技术为基础，软硬件可裁剪，适应应用系统对功能、可靠性、成本、体积、功耗等严格要求的专用计算机系统。
      <sup>
       [3]
      </sup>
      <a name="ref_[3]_6115">
      </a>
     </div>
     <div class="para">
      嵌入式系统是面向用户、面向产品、面向应用的，它必须与具体应用相结合才会具有生命力、才更具有优势。因此可以这样理解上述三个面向的含义，即嵌入式系统是与应用紧密结合的，它具有很强的专用性，必须结合实际系统
      <a href="http://baike.baidu.com/view/195818.htm" rel="nofollow noopener noreferrer" target="_blank">
       需求
      </a>
      进行合理的裁减利用。
     </div>
     <div class="para">
      ◆嵌入式系统是将先进的
      <a href="http://baike.baidu.com/view/62176.htm" rel="nofollow noopener noreferrer" target="_blank">
       计算机技术
      </a>
      、半导体技术和电子技术和各个行业的具体应用相结合后的产物，这一点就决定了它必然是一个技术密集、资金密集、高度分散、不断创新的知识集成系统。所以，介入嵌入式系统行业，必须有一个正确的定位。例如Palm之所以在PDA领域占有70%以上的市场，就是因为其立足于个人电子消费品，着重发展图形界面和多
      <a href="http://baike.baidu.com/view/3367696.htm" rel="nofollow noopener noreferrer" target="_blank">
       任务管理
      </a>
      ；而风河的Vxworks之所以在火星车上得以应用，则是因为其高实时性和高可靠性。
     </div>
     <div class="para">
      ◆嵌入式系统必须根据应用
      <a href="http://baike.baidu.com/view/195818.htm" rel="nofollow noopener noreferrer" target="_blank">
       需求
      </a>
      对软硬件进行裁剪，满足应用系统的功能、可靠性、成本、体积等要求。所以，如果能建立相对通用的软硬件基础，然后在其上开发出适应各种需要的系统，是一个比较好的发展模式。目前的嵌入式系统的核心往往是一个只有几K到几十K
      <a href="http://baike.baidu.com/view/928365.htm" rel="nofollow noopener noreferrer" target="_blank">
       微内核
      </a>
      ，需要根据实际的使用进行功能扩展或者裁减，但是由于微内核的存在，使得这种扩展能够非常顺利的进行。
     </div>
     <div class="para">
      实际上，嵌入式系统本身是一个外延极广的名词，凡是与产品结合在一起的具有嵌入式特点的
      <a href="http://baike.baidu.com/view/57978.htm" rel="nofollow noopener noreferrer" target="_blank">
       控制系统
      </a>
      都可以叫嵌入式系统，而且有时很难以给它下一个准确的定义。现在人们讲嵌入式系统时，某种程度上指近些年比较热的具有
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      的嵌入式系统，本文在进行分析和展望时，也沿用这一观点。
     </div>
     <div class="para">
      一般而言，嵌入式系统的构架可以分成四个部分：
      <a href="http://baike.baidu.com/view/50152.htm" rel="nofollow noopener noreferrer" target="_blank">
       处理器
      </a>
      、
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      、输入输出（I/O）和软件（由于多数
      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>
      的
      <a href="http://baike.baidu.com/view/7886.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用软件
      </a>
      和
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      都是紧密结合的，在这里我们对其不加区分，这也是嵌入式系统和一般的PC操作系统的最大区别）。
      <sup>
       [4]
      </sup>
      <a name="ref_[4]_6115">
      </a>
     </div>
     <h3 class="headline-1">
      <span class="headline-content">
       物联网嵌入式系统
      </span>
     </h3>
     <div class="para">
      随着医疗电子、智能家居、物流管理和电力控制等方面的不断风靡，嵌入式系统利用自身积累的底蕴经验，重视和把握这个机会，想办法在已经成熟的平台和产品基础上与应用传感单元的结合，扩展物联和感知的支持能力，发掘某种领域物联网应用。作为物联网重要技术组成的嵌入式系统，嵌入式系统的视角有助于深刻地、全面地理解物联网的本质。
     </div>
     <div class="para">
      这有两层意思：第一，
      <a href="http://baike.baidu.com/view/1136308.htm" rel="nofollow noopener noreferrer" target="_blank">
       物联网
      </a>
      的核心仍然是互联网，是在互联网基础上的延伸和扩展的网络;第二，其用户端延伸和扩展到了任何物品与物品之间，进行信息交换和通信，必须具备嵌入式系统构建的智能终端。因此，物联网系统是通过射频识别(RFID)、红外感应器、全球定位系统、激光扫描器等信息传感设备，按约定的协议，把任何物品与互联网相连接，进行信息交换和通信的系统架构。
     </div>
     <div class="para">
      物联网不仅仅提供了
      <a href="http://baike.baidu.com/view/16431.htm" rel="nofollow noopener noreferrer" target="_blank">
       传感器
      </a>
      的连接，其本身也具有智能处理的能力，能够对物体实施智能控制，这就是我们嵌入式系统所能做到的。诚然，物联网将传感器和智能处理相结合，利用云计算、模式识别等各种智能技术，扩充其应用领域。从传感器获得的海量信息中分析、加工和处理出有意义的数据，以适应不同用户的不同需求，发现新的应用领域和应用模式。
      <sup>
       [5]
      </sup>
      <a name="ref_[5]_6115">
      </a>
     </div>
     <h3 class="headline-1">
      <span class="headline-content">
       历史发展
      </span>
     </h3>
     <div class="para">
      从20世纪七十年代单片机的出现到各式各样的嵌入式微
      <a href="http://baike.baidu.com/view/50152.htm" rel="nofollow noopener noreferrer" target="_blank">
       处理器
      </a>
      ，微控制器的大规模应用，嵌入式系统已经有了近30年的发展历史。
     </div>
     <div class="para">
      嵌入式系统的出现最初是基于
      <a href="http://baike.baidu.com/view/1012.htm" rel="nofollow noopener noreferrer" target="_blank">
       单片机
      </a>
      的。70年代单片机的出现，使得汽车、家电、
      <a href="http://baike.baidu.com/view/631726.htm" rel="nofollow noopener noreferrer" target="_blank">
       工业机器
      </a>
      、通信装置以及成千上万种产品可以通过内嵌电子装置来获得更佳的使用性能：更容易使用、更快、更便宜。这些装置已经初步具备了嵌入式的应用特点，但是这时的应用只是使用8位的芯片，执行一些
      <a href="http://baike.baidu.com/view/2554947.htm" rel="nofollow noopener noreferrer" target="_blank">
       单线程
      </a>
      的
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      ，还谈不上“系统”的概念。
     </div>
     <div class="para">
      最早的单片机是Intel公司的 8048，它出现在1976年。Motorola同时推出了68HC05，Zilog公司推出了Z80系列，这些早期的单片机均含有256字节的RAM、4K的ROM、4 个8位并口、1个全双工
      <a href="http://baike.baidu.com/view/1060187.htm" rel="nofollow noopener noreferrer" target="_blank">
       串行口
      </a>
      、两个16位定 时 器。之后在80年代初，Intel又进一步完善了8048，在它的基础上研制成功了8051，这在
      <a href="http://baike.baidu.com/view/1012.htm" rel="nofollow noopener noreferrer" target="_blank">
       单片机
      </a>
      的历史上是值得纪念的一页，迄今为止，51系列的单片机仍然是最为成功的单片机芯片，在各种产品中有着非常广泛的应用。
     </div>
     <div class="para">
      从80年代早期开始，嵌入式系统的
      <a href="http://baike.baidu.com/view/39175.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序员
      </a>
      开始用商业级的“
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      ”编写嵌入式
      <a href="http://baike.baidu.com/view/7886.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用软件
      </a>
      ，这使得可以获取更短的开发周期，更低的开发资金和更高的开发
      <a href="http://baike.baidu.com/view/47610.htm" rel="nofollow noopener noreferrer" target="_blank">
       效率
      </a>
      ，“嵌入式系统”真正出现了。确切点说，这个时候的
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      是一个实时核，这个实时核包含了许多传统操作系统的特征，包括
      <a href="http://baike.baidu.com/view/3367696.htm" rel="nofollow noopener noreferrer" target="_blank">
       任务管理
      </a>
      、任务间通讯、同步与相互排斥、中断支持、
      <a href="http://baike.baidu.com/view/4541016.htm" rel="nofollow noopener noreferrer" target="_blank">
       内存管理
      </a>
      等功能。
     </div>
     <div class="para">
      其中比较著名的有Ready System 公司的VRTX、Integrated System Incorporation (ISI)的PSOS和IMG的VxWorks、QNX公司的QNX 等。这些
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      都具有嵌入式的典型特点：它们均采用占先式的调度，响应的时间很短，任务执行的时间可以确定；系统
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      很小，具有可裁剪，可扩充和可移植性，可以移植到各种
      <a href="http://baike.baidu.com/view/50152.htm" rel="nofollow noopener noreferrer" target="_blank">
       处理器
      </a>
      上；较强的实时和可靠性，适合嵌入式应用。这些嵌入式
      <a href="http://baike.baidu.com/view/27250.htm" rel="nofollow noopener noreferrer" target="_blank">
       实时多任务操作系统
      </a>
      的出现，使得应用开发人员得以从小范围的开发解放出来，同时也促使嵌入式有了更为广阔的应用空间。
     </div>
     <div class="para">
      90年代以后，随着对实时性要求的提高，软件规模不断上升，实时核逐渐发展为
      <a href="http://baike.baidu.com/view/27250.htm" rel="nofollow noopener noreferrer" target="_blank">
       实时多任务操作系统
      </a>
      （RTOS），并作为一种软件平台逐步成为目前国际嵌入式系统的主流。这时候更多的公司看到了嵌入式系统的广阔发展前景，开始大力发展自己的
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      。除了上面的几家老牌公司以外，还出现了Palm OS，WinCE，嵌入式Linux，Lynx，Nucleux，以及国内的Hopen，Delta Os等
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      。随着
      <a href="http://baike.baidu.com/view/685011.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式技术
      </a>
      的发展前景日益广阔，相信会有更多的
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      软件出现。
      <sup>
       [6]
      </sup>
      <a name="ref_[6]_6115">
      </a>
     </div>
     <h3 class="headline-1">
      <span class="headline-content">
       特点
      </span>
     </h3>
     <div class="para">
      这些年来掀起了嵌入式系统应用热潮的原因主要有几个方面：一是芯片技术的发展，使得单个芯片具有更强的处理能力，而且使集成多种
      <a href="http://baike.baidu.com/view/159864.htm" rel="nofollow noopener noreferrer" target="_blank">
       接口
      </a>
      已经成为可能，众多芯片生产厂商已经将注意力集中在这方面。另一方面的原因就是应用的需要，由于对产品可靠性、成本、更新换代要求的提高，使得嵌入式系统逐渐从纯硬件实现和使用
      <a href="http://baike.baidu.com/view/1472448.htm" rel="nofollow noopener noreferrer" target="_blank">
       通用计算机
      </a>
      实现的应用中脱颖而出，成为近年来令人关注的焦点。
     </div>
     <div class="para">
      从上面的定义，我们可以看出嵌入式系统的几个重要特征：
     </div>
     <div class="para">
      1．系统
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      小。由于嵌入式系统一般是应用于小型电子装置的，
      <a href="http://baike.baidu.com/view/53557.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统资源
      </a>
      相对有限，所以
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      较之传统的
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      要小得多。比如Enea公司的OSE
      <a href="http://baike.baidu.com/view/991489.htm" rel="nofollow noopener noreferrer" target="_blank">
       分布式系统
      </a>
      ，
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      只有5K，而Windows的内核？简直没有可比性。
     </div>
     <div class="para">
      2．专用性强。嵌入式系统的个性化很强，其中的
      <a href="http://baike.baidu.com/view/8343.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件系统
      </a>
      和硬件的结合非常紧密，一般要针对硬件进行系统的移植，即使在同一品牌、同一系列的产品中也需要根据系统硬件的变化和增减不断进行修改。同时针对不同的任务，往往需要对系统进行较大更改，
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      的编译下载要和系统相结合，这种修改和
      <a href="http://baike.baidu.com/view/847702.htm" rel="nofollow noopener noreferrer" target="_blank">
       通用软件
      </a>
      的“升级”是完全两个概念。
     </div>
     <div class="para">
      3．系统精简。嵌入式系统一般没有
      <a href="http://baike.baidu.com/view/7860.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统软件
      </a>
      和
      <a href="http://baike.baidu.com/view/7886.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用软件
      </a>
      的明显区分，不要求其功能设计及实现上过于复杂，这样一方面利于
      <a href="http://baike.baidu.com/view/57978.htm" rel="nofollow noopener noreferrer" target="_blank">
       控制系统
      </a>
      成本，同时也利于实现系统安全。
     </div>
     <div class="para">
      4．高实时性的系统软件(OS)是
      <a href="http://baike.baidu.com/view/294802.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式软件
      </a>
      的基本要求。而且软件要求固态存储，以提高速度；
      <a href="http://baike.baidu.com/view/878514.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件代码
      </a>
      要求高质量和高可靠性。
     </div>
     <div class="para">
      5．
      <a href="http://baike.baidu.com/view/2517315.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式软件开发
      </a>
      要想走向标准化，就必须使用多任务的操作系统。嵌入式系统的
      <a href="http://baike.baidu.com/view/330120.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用程序
      </a>
      可以没有
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      直接在芯片上运行；但是为了合理地调度多任务、利用系统资源、
      <a href="http://baike.baidu.com/view/710031.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统函数
      </a>
      以及和专家
      <a href="http://baike.baidu.com/view/905012.htm" rel="nofollow noopener noreferrer" target="_blank">
       库函数
      </a>
      接口，用户必须自行选配RTOS（Real－Time Operating System）开发平台，这样才能保证程序执行的实时性、可靠性，并减少开发时间，保障
      <a href="http://baike.baidu.com/view/1451265.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件质量
      </a>
      。
     </div>
     <div class="para">
      6．
      <a href="http://baike.baidu.com/view/657641.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式系统开发
      </a>
      需要开发工具和环境。由于其本身不具备
      <a href="http://baike.baidu.com/view/880555.htm" rel="nofollow noopener noreferrer" target="_blank">
       自举
      </a>
      开发能力，即使设计完成以后用户通常也是不能对其中的
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      功能进行修改的，必须有一套开发工具和环境才能进行开发，这些工具和环境一般是基于通用计算机上的软硬件设备以及各种逻辑分析仪、
      <a href="http://baike.baidu.com/view/4722251.htm" rel="nofollow noopener noreferrer" target="_blank">
       混合信号示波器
      </a>
      等。开发时往往有主机和目标机的概念，主机用于
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">

      </a>

      <sup>

      </sup>
      <a name="ref_[7]_6115">

     </div>

     </div>
     <div class="para">

     <h4 class="headline-2">

      </a>
      电路，就构成了一个嵌入式核心控制模块。其中
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">

      </a>

     </div>
     <div class="para">
      <strong>
       1、
      </strong>
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
     </div>
     <div class="para">
      嵌入式系统硬件层的核心是
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      ，嵌入式微处理器与通用CPU最大的不同在于嵌入式微处理器大多工作在为特定用户群所专用设计的系统中，它将通用CPU许多由
      <a href="http://baike.baidu.com/view/559655.htm" rel="nofollow noopener noreferrer" target="_blank">
       板卡
      </a>
      完成的任务集成在芯片内部，从而有利于嵌入式系统在设计时趋于小型化，同时还具有很高的
      <a href="http://baike.baidu.com/view/47610.htm" rel="nofollow noopener noreferrer" target="_blank">
       效率
      </a>
      和可靠性。
     </div>
     <div class="para">
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      的
      <a href="http://baike.baidu.com/view/1188494.htm" rel="nofollow noopener noreferrer" target="_blank">
       体系结构
      </a>
      可以采用
      <a href="http://baike.baidu.com/view/20003.htm" rel="nofollow noopener noreferrer" target="_blank">
       冯·诺依曼
      </a>
      体系或
      <a href="http://baike.baidu.com/view/513214.htm" rel="nofollow noopener noreferrer" target="_blank">
       哈佛体系结构
      </a>
      ；
      <a href="http://baike.baidu.com/view/178189.htm" rel="nofollow noopener noreferrer" target="_blank">
       指令系统
      </a>
      可以选用精简指令系统（Reduced Instruction Set Computer，RISC）和复杂指令系统CISC（Complex Instruction Set Computer，CISC）。RISC计算机在通道中只包含最有用的指令，确保数据通道快速执行每一条指令，从而提高了执行
      <a href="http://baike.baidu.com/view/47610.htm" rel="nofollow noopener noreferrer" target="_blank">
       效率
      </a>
      并使CPU硬件结构设计变得更为简单。
     </div>
     <div class="para">
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      有各种不同的体系，即使在同一体系中也可能具有不同的
      <a href="http://baike.baidu.com/view/1087.htm" rel="nofollow noopener noreferrer" target="_blank">
       时钟频率
      </a>
      和
      <a href="http://baike.baidu.com/view/253223.htm" rel="nofollow noopener noreferrer" target="_blank">
       数据总线宽度
      </a>
      ，或集成了不同的外设和接口。据不完全统计，目前全世界
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      已经超过1000多种，
      <a href="http://baike.baidu.com/view/1188494.htm" rel="nofollow noopener noreferrer" target="_blank">
       体系结构
      </a>
      有30多个系列，其中主流的体系有ARM、MIPS、PowerPC、X86和SH等。但与全球PC市场不同的是，没有一种
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      可以主导市场，仅以32位的产品而言，就有100种以上的嵌入式微处理器。
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      的选择是根据具体的应用而决定的。
     </div>
     <div class="para">
      <strong>
       2、
      </strong>
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
     </div>
     <div class="para">
      嵌入式系统需要
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      来存放和执行代码。嵌入式系统的
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      包含Cache、主存和
      <a href="http://baike.baidu.com/view/1258006.htm" rel="nofollow noopener noreferrer" target="_blank">
       辅助存储器
      </a>
      。
     </div>
     <div class="para">
      1&gt;Cache
     </div>
     <div class="para">
      Cache是一种容量小、速度快的
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      阵列它位于主存和
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      之间，存放的是最近一段时间微处理器使用最多的
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      代码和数据。在需要进行数据读取操作时，
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      尽可能的从Cache中读取数据，而不是从主存中读取，这样就大大改善了系统的性能，提高了微处理器和主存之间的
      <a href="http://baike.baidu.com/view/434019.htm" rel="nofollow noopener noreferrer" target="_blank">
       数据传输速率
      </a>
      。Cache的主要目标就是：减小
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      （如主存和辅助存储器）给
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      造成的存储器访问瓶颈，使处理速度更快，实时性更强。
     </div>
     <div class="para">
      在嵌入式系统中Cache全部集成在
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      内，可分为数据Cache、指令Cache或混合Cache，Cache的大小依不同处理器而定。一般中高档的
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      才会把Cache集成进去。
     </div>
     <div class="para">
      2&gt;主存
     </div>
     <div class="para">
      主存是
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      能直接访问的
      <a href="http://baike.baidu.com/view/6159.htm" rel="nofollow noopener noreferrer" target="_blank">
       寄存器
      </a>
      ，用来存放系统和用户的
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      及数据。它可以位于
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      的内部或外部，其
      <a href="http://baike.baidu.com/view/334600.htm" rel="nofollow noopener noreferrer" target="_blank">
       容量
      </a>
      为256KB~1GB，根据具体的应用而定，一般片
      <a href="http://baike.baidu.com/view/746528.htm" rel="nofollow noopener noreferrer" target="_blank">
       内存储器
      </a>
      容量小，速度快，片
      <a href="http://baike.baidu.com/view/67773.htm" rel="nofollow noopener noreferrer" target="_blank">
       外存储器
      </a>
      容量大。
     </div>
     <div class="para">
      常用作主存的
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      有：
     </div>
     <div class="para">
      ROM类 NOR Flash、EPROM和PROM等。
     </div>
     <div class="para">
      RAM类 SRAM、DRAM和SDRAM等。
     </div>
     <div class="para">
      其中NOR Flash 凭借其可擦写次数多、存储速度快、存储容量大、价格便宜等优点，在嵌入式领域内得到了广泛应用。
     </div>
     <div class="para">
      3&gt;辅助存储器
     </div>
     <div class="para">
      辅助
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      用来存放
      <a href="http://baike.baidu.com/view/6954399.htm" rel="nofollow noopener noreferrer" target="_blank">
       大数据
      </a>
      量的
      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      代码或信息，它的容量大、但读取速度与主存相比就慢的很多，用来长期保存用户的信息。
     </div>
     <div class="para">
      嵌入式系统中常用的外存有：
      <a href="http://baike.baidu.com/view/4480.htm" rel="nofollow noopener noreferrer" target="_blank">
       硬盘
      </a>
      、NAND Flash、CF卡、MMC和SD卡等。
     </div>
     <div class="para">
      <strong>
       3、通用设备接口和I/O接口
      </strong>
     </div>
     <div class="para">
      嵌入式系统和外界交互需要一定形式的通用设备接口，如A/D、D/A、I/O等，外设通过和片外其他设备的或传感器的连接来实现
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      的输入/输出功能。每个外设通常都只有单一的功能，它可以在芯片外也可以内置芯片中。外设的种类很多，可从一个简单的串行通信设备到非常复杂的
      <a href="http://baike.baidu.com/view/345218.htm" rel="nofollow noopener noreferrer" target="_blank">
       802.11

     <h4 class="headline-2">
      <a name="5_3">
      </a>
      <a name="sub6115_5_3">
      </a>
      <span class="headline-content">
       中间层
      </span>
     </h4>
     <div class="para">
      硬件层与软件层之间为中间层，也称为硬件抽象层（Hardware Abstract Layer，HAL）或
      <a href="http://baike.baidu.com/view/1730227.htm" rel="nofollow noopener noreferrer" target="_blank">
       板级支持包
      </a>
      （Board Support Package，BSP），它将系统上层软件与底层硬件分离开来，使系统的底层
      <a href="http://baike.baidu.com/view/1048.htm" rel="nofollow noopener noreferrer" target="_blank">
       驱动程序
      </a>
      与硬件无关，上层软件开发人员无需关心底层硬件的具体情况，根据BSP 层提供的接口即可进行开发。该层一般包含相关底层硬件的初始化、数据的输入/输出操作和硬件设备的配置功能。BSP具有以下两个特点。
     </div>
     <div class="para">
      硬件相关性：因为
      <a href="http://baike.baidu.com/view/2079532.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式实时系统
      </a>
      的
      <a href="http://baike.baidu.com/view/290209.htm" rel="nofollow noopener noreferrer" target="_blank">
       硬件环境
      </a>
      具有应用相关性，而作为上层软 件与硬件平台之间的接口，BSP需要为
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      提供操作和控制具体硬件的方法。
     </div>
     <div class="para">
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      相关性：不同的操作系统具有各自的软件层次结构，因此，不同的操作系统具有特定的硬件接口形式。
     </div>
     <div class="para">
      实际上，BSP是一个介于
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      和底层硬件之间的软件层次，包括了系统中大部分与硬件联系紧密的软件模块。设计一个完整的BSP需要完成两部分工作：嵌入式系统的硬件初始化以及BSP功能，设计硬件相关的
      <a href="http://baike.baidu.com/view/547398.htm" rel="nofollow noopener noreferrer" target="_blank">
       设备驱动
      </a>
      。
     </div>
     <div class="para">
      <strong>
       1、嵌入式系统硬件初始化
      </strong>
     </div>
     <div class="para">
      <a href="http://baike.baidu.com/view/811790.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统初始化
      </a>
      过程可以分为3个主要环节，按照自底向上、从硬件到软件的次序依次为：片级初始化、板级初始化和系统级初始化。
     </div>
     <div class="para">
      片级初始化
     </div>
     <div class="para">
      完成
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      的初始化，包括设置嵌入式微处理器的核心
      <a href="http://baike.baidu.com/view/6159.htm" rel="nofollow noopener noreferrer" target="_blank">
       寄存器
      </a>
      和
      <a href="http://baike.baidu.com/view/4092816.htm" rel="nofollow noopener noreferrer" target="_blank">
       控制寄存器
      </a>
      、嵌入式微处理器核心工作模式和嵌入式微处理器的
      <a href="http://baike.baidu.com/view/1869280.htm" rel="nofollow noopener noreferrer" target="_blank">
       局部总线
      </a>
      模式等。片级初始化把
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      从上电时的默认状态逐步设置成系统所要求的工作状态。这是一个纯硬件的初始化过程。
     </div>
     <div class="para">
      板级初始化
     </div>
     <div class="para">
      完成
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      以外的其他硬件设备的初始化。另外，还需设置某些软件的数据结构和参数，为随后的系统级初始化和
      <a href="http://baike.baidu.com/view/330120.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用程序
      </a>
      的运行建立硬件和
      <a href="http://baike.baidu.com/view/285288.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件环境
      </a>
      。这是一个同时包含软硬件两部分在内的初始化过程。
     </div>
     <div class="para">
      系统初始化
     </div>
     <div class="para">
      该初始化过程以软件初始化为主，主要进行
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      的初始化。BSP将对
      <a href="http://baike.baidu.com/view/27055.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式微处理器
      </a>
      的控制权转交给
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      ，由操作系统完成余下的初始化操作，包含加载和初始化与硬件无关的设备驱动程序，建立
      <a href="http://baike.baidu.com/view/404470.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统内存
      </a>
      区，加载并初始化其他系统软件模块，如网络系统、文件系统等。最后，
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      创建
      <a href="http://baike.baidu.com/view/330120.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用程序
      </a>
      环境，并将控制权交给应用程序的入口。
     </div>
     <div class="para">
      <strong>
       2、硬件相关的设备
      </strong>
      <a href="http://baike.baidu.com/view/1048.htm" rel="nofollow noopener noreferrer" target="_blank">
       驱动程序
      </a>
     </div>
     <div class="para">
      BSP的另一个主要功能是硬件相关的设备驱动。硬件相关的设备驱动程序的初始化通常是一个从高到低的过程。尽管BSP中包含硬件相关的设备驱动程序，但是这些设备驱动程序通常不直接由BSP使用，而是在系统初始化过程中由BSP将他们与
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      中通用的设备驱动程序关联起来，并在随后的应用中由通用的设备驱动程序调用，实现对硬件设备的操作。与硬件相关的驱动程序是BSP设计与开发中另一个非常关键的环节。
     </div>
     <h4 class="headline-2">
      <a name="5_4">
      </a>
      <a name="sub6115_5_4">
      </a>
      <span class="headline-content">
       系统软件层
      </span>
     </h4>
     <div class="para">
      系统软件层由
      <a href="http://baike.baidu.com/view/27250.htm" rel="nofollow noopener noreferrer" target="_blank">
       实时多任务操作系统
      </a>
      （Real-time Operation System，RTOS）、文件系统、图形
      <a href="http://baike.baidu.com/view/1309799.htm" rel="nofollow noopener noreferrer" target="_blank">
       用户接口
      </a>
      （Graphic User Interface，GUI）、网络系统及通用组件模块组成。RTOS是嵌入式
      <a href="http://baike.baidu.com/view/7886.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用软件
      </a>
      的基础和开发平台。
     </div>
     <div class="para">
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
     </div>
     <div class="para">
      嵌入式操作系统（Embedded Operation System，EOS）是一种用途广泛的系统软件，过去它主要应用与工业控制和国防系统领域。EOS负责
      <a href="http://baike.baidu.com/view/1187982.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入系统
      </a>
      的全部软、硬件资源的分配、任务调度，控制、协调并发活动。它必须体现其所在系统的特征，能够通过装卸某些模块来达到系统所要求的功能。目前，已推出一些应用比较成功的EOS产品系列。随着
      <a href="http://baike.baidu.com/view/4632922.htm" rel="nofollow noopener noreferrer" target="_blank">
       Internet技术
      </a>
      的发展、信息家电的普及应用及EOS的微型化和专业化，EOS开始从单一的弱功能向高专业化的强功能方向发展。
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      在系统实时高效性、硬件的相关依赖性、软件固化以及应用的专用性等方面具有较为突出的特点。EOS是相对于一般
      <a href="http://baike.baidu.com/view/880.htm" rel="nofollow noopener noreferrer" target="_blank">
       操作系统
      </a>
      而言的，它除具备了一般操作系统最基本的功能，如任务调度、同步机制、
      <a href="http://baike.baidu.com/view/4499229.htm" rel="nofollow noopener noreferrer" target="_blank">
       中断处理
      </a>
      、文件功能等外，还有以下特点：
     </div>
     <div class="para">
      (1)可装卸性。开放性、可伸缩性的
      <a href="http://baike.baidu.com/view/1188494.htm" rel="nofollow noopener noreferrer" target="_blank">
       体系结构
      </a>
      。
     </div>
     <div class="para">
      (2)强实时性。EOS实时性一般较强，可用于各种设备控制当中。
     </div>
     <div class="para">
      (3)统一的接口。提供各种设备驱动接口.
     </div>
     <div class="para">
      (4)操作方便、简单、提供友好的图形GUI，图形界面，追求易学易用.
     </div>
     <div class="para">
      (5)提供强大的网络功能，支持TCP/IP协议及其它协议，提供TCP/UDP/IP/PPP
      <a href="http://baike.baidu.com/view/63557.htm" rel="nofollow noopener noreferrer" target="_blank">
       协议支持
      </a>
      及统一的MAC访问层接口，为各种移动计算设备预留接口.
     </div>
     <div class="para">
      (6)强稳定性，弱交互性。嵌入式系统一旦开始运行就不需要用户过多的干预，这就要负责系统管理的EOS具有较强的稳定性。
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      的用户接口一般不提供操作命令，它通过
      <a href="http://baike.baidu.com/view/47173.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统调用
      </a>
      命令向
      <a href="http://baike.baidu.com/view/1976812.htm" rel="nofollow noopener noreferrer" target="_blank">
       用户程序
      </a>
      提供服务。
     </div>
     <div class="para">
      (7)固化代码。在
      <a href="http://baike.baidu.com/view/1187982.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入系统
      </a>
      中，
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      和
      <a href="http://baike.baidu.com/view/7886.htm" rel="nofollow noopener noreferrer" target="_blank">
       应用软件
      </a>
      被固化在嵌入式系统计算机的ROM中。辅助
      <a href="http://baike.baidu.com/view/87697.htm" rel="nofollow noopener noreferrer" target="_blank">
       存储器
      </a>
      在嵌入式系统中很少使用，因此，
      <a href="http://baike.baidu.com/view/18554.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式操作系统
      </a>
      的文件管理功能应该能够很容易地拆卸，而用各种内存文件系统.
     </div>
     <div class="para">
      (8)更好的硬件适应性，也就是良好的移植性.

      <span class="headline-content">
       机器人
      </span>
     </h4>
     <div class="para">
      嵌入式芯片的发展将使
      <a href="http://baike.baidu.com/view/2788.htm" rel="nofollow noopener noreferrer" target="_blank">
       机器人
      </a>
      在微型化，高智能方面优势更加明显，同时会大幅度降低机器人的价格，使其在工业领域和服务领域获得更广泛的应用。
     </div>
     <div class="para">
      这些应用中，可以着重于在
      <a href="http://baike.baidu.com/view/57913.htm" rel="nofollow noopener noreferrer" target="_blank">
       控制
      </a>
      方面的应用。就远程家电控制而言，除了开发出支持TCP/IP的嵌入式系统之外，家电产品控制协议也需要制订和统一，这需要家电生产厂家来做。同样的道理，所有基于网络的远程控制器件都需要与嵌入式系统之间实现接口，然后再由嵌入式系统来控制并通过
      <a href="http://baike.baidu.com/view/3487.htm" rel="nofollow noopener noreferrer" target="_blank">
       网络
      </a>
      实现控制。所以，开发和探讨嵌入式系统有着十分重要的意义。
     </div>

      <a href="http://baike.baidu.com/view/1706.htm" rel="nofollow noopener noreferrer" target="_blank">
       因特网
      </a>
      之后最伟大的发明。我国著名嵌入式系统专家
      <a href="http://baike.baidu.com/view/310558.htm" rel="nofollow noopener noreferrer" target="_blank">
       沈绪榜
      </a>
      院士98年11月在
      <a href="http://baike.baidu.com/view/1267.htm" rel="nofollow noopener noreferrer" target="_blank">
       武汉
      </a>

       计算机工业
      </a>

     </div>
     <div class="para">

      <a href="http://baike.baidu.com/view/685011.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式技术
      </a>
      全面展开，目前已成为通信和消费类产品的共同发展方向。在通信领域，
      <a href="http://baike.baidu.com/view/1090132.htm" rel="nofollow noopener noreferrer" target="_blank">
       数字技术
      </a>
      正在全面取代模拟技术。在广播电视领域，美国已开始由
      <a href="http://baike.baidu.com/view/155493.htm" rel="nofollow noopener noreferrer" target="_blank">
       模拟电视
      </a>

      <a href="http://baike.baidu.com/view/3622.htm" rel="nofollow noopener noreferrer" target="_blank">
       欧洲
      </a>
      的DVB（数字电视广播）技术已在全球大多数国家推广。
      <a href="http://baike.baidu.com/view/1859510.htm" rel="nofollow noopener noreferrer" target="_blank">
       数字音频
      </a>

      <a href="http://baike.baidu.com/view/1355.htm" rel="nofollow noopener noreferrer" target="_blank">
       集成电路
      </a>
      和新型元器件在产业发展中的作用日益重要。所有上述产品中，都离不开嵌入式系统技术。象前途无可计量的维纳斯计划生产机顶盒，核心技术就是采用32位以上芯片级的
      <a href="http://baike.baidu.com/view/685011.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式技术
      </a>
      。在个人领域中，
      <a href="http://baike.baidu.com/view/4085543.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式产品
      </a>
      将主要是个人商用，作为个人移动的数据处理和通讯软件。由于
      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>
      具有自然的
      <a href="http://baike.baidu.com/view/523297.htm" rel="nofollow noopener noreferrer" target="_blank">
       人机交互
      </a>

     </div>
     <div class="para">

      </a>

      </a>

      <a href="http://baike.baidu.com/view/17674.htm" rel="nofollow noopener noreferrer" target="_blank">
       程序
      </a>
      源代码，甚至很多厂商还提供
      <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" target="_blank">
       微处理器
      </a>
      的样片。这对于我们从事这方面的研发，无疑是个资源宝库。对于
      <a href="http://baike.baidu.com/view/575391.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件设计
      </a>
      来说，不管是上手还是进一步开发，都相对来说比较容易。这就使得很多生手能够比较快的进入研究状态，利于发挥大家的积极创造性。
     </div>
     <div class="para">
      今天嵌入式系统带来的工业年产值已超过了1万亿美元，1997年来自美国嵌入式系统大会(Embedded System Conference)的报告指出，未来5年仅基于

     <h4 class="headline-2">
      <a name="7_13">
      </a>

      </a>
      <span class="headline-content">
       发展趋势
      </span>
     </h4>
     <div class="para">
      信息时代，数字时代使得
      <a href="http://baike.baidu.com/view/4085543.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式产品
      </a>
      获得了巨大的发展契机，为嵌入式市场展现了美好的前景，同时也对嵌入式生产厂商提出了新的挑战，从中我们可以看出未来嵌入式系统的几大发展趋势：
     </div>
     <div class="para">
      1．
      <a href="http://baike.baidu.com/view/55.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式开发
      </a>
      是一项
      <a href="http://baike.baidu.com/view/55409.htm" rel="nofollow noopener noreferrer" target="_blank">
       系统工程
      </a>
      ，因此要求嵌入式系统厂商不仅要提供嵌入式软硬件系统本身，同时还需要提供强大的硬件开发工具和
      <a href="http://baike.baidu.com/view/600107.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件包
      </a>
      支持。
     </div>
     <div class="para">
      目前很多厂商已经充分考虑到这一点，在主推系统的同时，将开发环境也作为重点推广。比如三星在推广Arm7，Arm9芯片的同时还提供
      <a href="http://baike.baidu.com/view/1678388.htm" rel="nofollow noopener noreferrer" target="_blank">
       开发板
      </a>
      和板级支持包（BSP），而WindowCE在主推系统时也提供Embedded VC++作为开发工具，还有Vxworks的Tonado开发环境，DeltaOS的Limda编译环境等等都是这一趋势的典型体现。当然，这也是市场竞争的结果。
     </div>
     <div class="para">
      2．
      <a href="http://baike.baidu.com/view/4551569.htm" rel="nofollow noopener noreferrer" target="_blank">
       网络化
      </a>
      、信息化的要求随着因特网技术的成熟、
      <a href="http://baike.baidu.com/view/10821.htm" rel="nofollow noopener noreferrer" target="_blank">
       带宽
      </a>
      的提高日益提高，使得以往单一功能的设备如电话、手机、冰箱、微波炉等功能不再单一，结构更加复杂。
     </div>
     <div class="para">
      这就要求芯片设计厂商在芯片上集成更多的功能，为了满足应用功能的升级，
      <a href="http://baike.baidu.com/view/282148.htm" rel="nofollow noopener noreferrer" target="_blank">
       设计师
      </a>
      们一方面采用更强大的
      <a href="http://baike.baidu.com/view/1258964.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式处理器
      </a>
      如32位、64位RISC芯片或信号处理器DSP增强处理能力，同时增加功能接口，如USB，扩展总线类型，如CAN BUS，加强对
      <a href="http://baike.baidu.com/view/3323.htm" rel="nofollow noopener noreferrer" target="_blank">
       多媒体
      </a>
      、图形等的处理，逐步实施
      <a href="http://baike.baidu.com/view/882467.htm" rel="nofollow noopener noreferrer" target="_blank">
       片上系统
      </a>
      （SOC）的概念。软件方面采用实时多任务
      <a href="http://baike.baidu.com/view/146617.htm" rel="nofollow noopener noreferrer" target="_blank">
       编程技术
      </a>
      和交叉开发工具技术来控制功能复杂性，简化应用程序设计、保障软件质量和缩短开发周期。如HP
     </div>
     <div class="para">
      3．
      <a href="http://baike.baidu.com/view/1359266.htm" rel="nofollow noopener noreferrer" target="_blank">
       网络互联
      </a>
      成为必然趋势。
     </div>
     <div class="para">
      未来的
      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>
      为了适应
      <a href="http://baike.baidu.com/view/1227765.htm" rel="nofollow noopener noreferrer" target="_blank">
       网络发展
      </a>
      的要求，必然要求硬件上提供各种
      <a href="http://baike.baidu.com/view/538641.htm" rel="nofollow noopener noreferrer" target="_blank">
       网络通信
      </a>
      接口。传统的
      <a href="http://baike.baidu.com/view/1012.htm" rel="nofollow noopener noreferrer" target="_blank">
       单片机
      </a>
      对于网络支持不足，而新一代的
      <a href="http://baike.baidu.com/view/1258964.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式处理器
      </a>
      已经开始内嵌网络接口，除了支持
      <a href="http://baike.baidu.com/view/7649.htm" rel="nofollow noopener noreferrer" target="_blank">
       TCP/IP协议
      </a>
      ，还有的支持IEEE1394、USB、CAN、Bluetooth或IrDA
      <a href="http://baike.baidu.com/view/1296283.htm" rel="nofollow noopener noreferrer" target="_blank">
       通信接口
      </a>
      中的一种或者几种，同时也需要提供相应的通信组网协议软件和
      <a href="http://baike.baidu.com/view/239585.htm" rel="nofollow noopener noreferrer" target="_blank">
       物理层
      </a>
      驱动软件。软件方面系统系统
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      支持网络模块，甚至可以在设备上嵌入
      <a href="http://baike.baidu.com/view/206703.htm" rel="nofollow noopener noreferrer" target="_blank">
       Web浏览器
      </a>
      ，真正实现随时随地用各种设备上网。
     </div>
     <div class="para">
      4．精简系统
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      、算法，降低功耗和软硬件成本。
     </div>
     <div class="para">
      未来的
      <a href="http://baike.baidu.com/view/4085543.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式产品
      </a>
      是软硬件紧密结合的设备，为了减低功耗和成本，需要设计者尽量精简系统
      <a href="http://baike.baidu.com/view/1366.htm" rel="nofollow noopener noreferrer" target="_blank">
       内核
      </a>
      ，只保留和系统功能紧密相关的软硬件，利用最低的资源实现最适当的功能，这就要求设计者选用最佳的编程模型和不断改进算法，优化
      <a href="http://baike.baidu.com/view/487018.htm" rel="nofollow noopener noreferrer" target="_blank">
       编译器
      </a>
      性能。因此，既要软件人员有丰富的硬件知识，又需要发展先进嵌入式
      <a href="http://baike.baidu.com/view/1654681.htm" rel="nofollow noopener noreferrer" target="_blank">
       软件技术
      </a>
      ，如Java、Web和WAP等。
     </div>

      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>

      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>

      <a href="http://baike.baidu.com/view/2778983.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式设备
      </a>
      距离这个要求还有很长的路要走。

      </strong>

     <h3 class="headline-1">
      <span class="headline-content">
       类型
      </span>
     </h3>
     <div class="para">
      常用的嵌入式软件架构有几种不同的基本类型。
     </div>
     <h4 class="headline-2">
      <a name="15_14">
      </a>
      <a name="sub6115_15_14">
      </a>
      <span class="headline-content">
       控制循环
      </span>
     </h4>
     <div class="para">
      在这种设计中，软件有一个简单的循环，这个循环调用各个
      <a href="http://baike.baidu.com/view/933538.htm" rel="nofollow noopener noreferrer" target="_blank">
       子程序
      </a>
      ，每个子程序管理硬件或者软件的某一部分。
      <a href="http://baike.baidu.com/view/121718.htm" rel="nofollow noopener noreferrer" target="_blank">
       中断
      </a>
      通常用来设置标记或者更新软件其他部分能够读取的
      <a href="http://baike.baidu.com/view/238434.htm" rel="nofollow noopener noreferrer" target="_blank">
       暂存器
      </a>
      。
     </div>
     <div class="para">
      系统使用简单的
      <a href="http://baike.baidu.com/view/16068.htm" rel="nofollow noopener noreferrer" target="_blank">
       API
      </a>
      来完成允许和禁止中断设置。如果处理得当的话，它能够在嵌套子程序中处理
      <a href="http://baike.baidu.com/view/1265502.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌套调用
      </a>
      ，在最外面的中断允许嵌套中恢复前面的中断状态。这种方法是实现Exokernel的一个最简单的方法。
     </div>
     <div class="para">
      通常在循环中有一些子程序使用周期性的实时中断控制一组软件定时器，当一个定时器时间到的时候就会运行相应的子程序或者设置相应的标志。
     </div>
     <div class="para">
      任何可能发生的硬件事件都应该有软件定时器的支持，硬件事件大概每万亿次出现一次错误，对于现代的硬件来说大概是一年发生一次，对于以百万计大规模生产的设备来说，遗漏一个软件软件定时器在商业上可能是灾难性的。
     </div>
     <div class="para">
      有时测试软件运行一组基于软件的安全定时器，它们周期性地复位硬件中的软件看门狗。如果软件错过一个事件，安全定时器软件将会捕捉到它。如果安全定时器出错，看门狗硬件将会复位系统。
     </div>
     <div class="para">
      可以使用指向每个
      <a href="http://baike.baidu.com/view/1906565.htm" rel="nofollow noopener noreferrer" target="_blank">
       状态机
      </a>
      函数的
      <a href="http://baike.baidu.com/view/159417.htm" rel="nofollow noopener noreferrer" target="_blank">
       指针
      </a>
      实现状态机，函数可以用
      <a href="http://baike.baidu.com/view/824.htm" rel="nofollow noopener noreferrer" target="_blank">
       C++
      </a>
      、
      <a href="http://baike.baidu.com/view/10075.htm" rel="nofollow noopener noreferrer" target="_blank">
       C
      </a>
      或者
      <a href="http://baike.baidu.com/view/49.htm" rel="nofollow noopener noreferrer" target="_blank">
       汇编语言
      </a>
      等实现。变化的状态将不同的函数放到指针中，每次循环运行时都会运行
      <a href="http://baike.baidu.com/view/1604730.htm" rel="nofollow noopener noreferrer" target="_blank">
       函数指针
      </a>
      。
     </div>
     <div class="para">
      许多设计人员建议每个循环读一次输入输出设备、并且保存得到的结果，这样可以保证逻辑过程的在一致的参数上运行。
     </div>
     <div class="para">
      许多设计人员喜欢将状态机设计成每个状态仅仅检查一到两项内容，通常是检查硬件事件和软件定时器。
     </div>
     <div class="para">
      设计人员建议多级状态机应该让低层状态机早于高层状态机运行，这样高层就能够根据正确的信息运行。
     </div>
     <div class="para">
      如内部燃烧控制这样的复杂功能通常根据多维表格进行处理，代码通常进行查表处理而不进行复杂的计算，为了减小表格的大小以及成本软件可以在条目之间进行差值运算。
     </div>
     <div class="para">
      在最小的单片机中，尤其是只有128位组
      <a href="http://baike.baidu.com/view/93201.htm" rel="nofollow noopener noreferrer" target="_blank">
       堆栈
      </a>
      的
      <a href="http://baike.baidu.com/view/2335918.htm" rel="nofollow noopener noreferrer" target="_blank">
       8051
      </a>
      中，控制环允许好的
      <a href="http://baike.baidu.com/view/553572.htm" rel="nofollow noopener noreferrer" target="_blank">
       连接器
      </a>
      使用静态分配的数据覆盖本地
      <a href="http://baike.baidu.com/view/296689.htm" rel="nofollow noopener noreferrer" target="_blank">
       变量
      </a>
      。在这种机制中，离子程序调用树末端越近的变量得到的存储器地址越高。当开始一个新的分支的时候，它的变量可以在以前分支遗弃的空间中进行重新分配。
     </div>
     <div class="para">
      简单控制循环的一个主要缺点是它无法保证响应特定硬件事件的时间。
     </div>
     <div class="para">
      细心的设计可以很容易地保证中断不会被长时间禁止，这样中断代码就可以在非常精确的时间运行。
     </div>
     <div class="para">
      控制环的另外一个主要缺陷是增加新的特性的时候会变得复杂。需要花费很长时间的
      <a href="http://baike.baidu.com/view/7420.htm" rel="nofollow noopener noreferrer" target="_blank">
       算法
      </a>
      必须小心地进行分解以使得每次只有一小部分在主循环中运行。
     </div>
     <div class="para">
      这种系统的优势是它的简单性，并且在很小的软件上，循环运行地很快几乎没有人关心它是不可预测的。
     </div>
     <div class="para">
      它的另外一个优势是这种系统保证运行软件的质量，无法将不好的运行结果归咎为其他的操作系统。
     </div>
     <h4 class="headline-2">
      <a name="15_15">
      </a>
      <a name="sub6115_15_15">
      </a>
      <span class="headline-content">
       非抢先式任务
      </span>
     </h4>
     <div class="para">
      非抢先式任务系统非常类似于上面的系统，只是这个循环是隐藏在
      <a href="http://baike.baidu.com/view/16068.htm" rel="nofollow noopener noreferrer" target="_blank">
       API
      </a>
      中的。我们定义一系列的任务，每个任务获得自己的子程序栈；然后，当一个任务空闲的时候，它调用一个空闲子程序（通常调用“暂停”、“等候”、“交出（
      <a href="http://baike.baidu.com/view/926415.htm" rel="nofollow noopener noreferrer" target="_blank">
       yield
      </a>
      ）”等等）。
     </div>
     <div class="para">
      带有类似属性的架构都带有一个事件队列，有一个循环根据队列列表中的一个域确定删除时间和调用子程序。
     </div>
     <div class="para">
      这种架构的优点和缺点都非常类似于控制环，只是这种方法添加新的软件更加简单，只需要简单地编写新的任务或者将它添加到队列解释器中。
     </div>
     <h4 class="headline-2">
      <a name="15_16">
      </a>
      <a name="sub6115_15_16">
      </a>
      <span class="headline-content">
       抢先式定时器
      </span>
     </h4>
     <div class="para">
      使用上面的任何一种系统，但是添加一个按照定时器中断运行子程序的定时器系统，这样就给系统添加了崭新的能力，这样定时器子程序第一次能在一个有保证的时间内运行。
     </div>
     <div class="para">
      另外，代码第一次能够在非预期的时间访问自己的数据结构。定时器子程序必须要象中断子程序一样进行处理。
     </div>
     <h4 class="headline-2">
      <a name="15_17">
      </a>
      <a name="sub6115_15_17">
      </a>
      <span class="headline-content">
       抢先式任务
      </span>
     </h4>
     <div class="para">
      使用上面的非抢先式任务系统，从一个抢先式定时器或者其他中断运行。
     </div>
     <div class="para">
      这样系统就突然变得很不一样了。任何一个任务的代码都有可能损害其他任务的数据 &amp;emdash; 所以它们必须进行切缺的切分。对于共享数据的访问必须使用一些同步策略进行控制，如消息队列、信号灯或者非阻塞同步机制。
     </div>
     <div class="para">
      经常在这一步开发组织就会购买一套
      <a href="http://baike.baidu.com/view/18308.htm" rel="nofollow noopener noreferrer" target="_blank">
       实时操作系统
      </a>
      。如果一个组织缺少能够编写操作系统的人才或者操作系统将要在几个产品之上，这可能是一个明智的选择。这通常要将开发计划增加六到八周，and forever after programmers can blame delays on it.
     </div>
     <h4 class="headline-2">
      <a name="15_18">
      </a>
      <a name="sub6115_15_18">
      </a>
      <span class="headline-content">
       微内核与外内核
      </span>
     </h4>
     <div class="para">
      这种方法试图将系统组织得比宏内核更易于配置，而同时提供类似的特点。
     </div>
     <div class="para">
      微内核是实时操作系统的一个逻辑发展，通常的组织方式是操作系统内核分配内存并且将CPU在不同的线程之间进行切换。用户模式的进程实现如文件系统、用户接口等主要的功能。
     </div>
     <div class="para">
      微内核在二十世纪五十年代开始首次尝试，但是由于计算机在任务间切换以及在任务间交换数据速度非常缓慢，所以人们放弃了微内核而钟情于
      <a href="http://baike.baidu.com/view/1351079.htm" rel="nofollow noopener noreferrer" target="_blank">
       MULTICS
      </a>
      和
      <a href="http://baike.baidu.com/view/8095.htm" rel="nofollow noopener noreferrer" target="_blank">
       UNIX
      </a>
      风格的大内核。总体上来说，微内核在任务切换以及任务间通信速度快的时候是比较成功的，在速度慢的时候是失败的。
     </div>
     <div class="para">
      外内核通过使用普通的子程序调用获得的通信效率很高，硬件以及系统中的软件都是程序员能用也能扩展的。资源内核（可能是库的一部分）分配CPU时间、内存以及其他资源。如多任务、网络以及文件系统这样的大内核特性通过代码库来提供。库可以进行动态的连接、扩展或者共享。不同的应用甚至可以使用的不同的库，但是所有的资源都来自于资源内核。
     </div>
     <h4 class="headline-2">
      <a name="15_19">
      </a>
      <a name="sub6115_15_19">
      </a>
      <span class="headline-content">
       虚拟机
      </span>
     </h4>
     <div class="para">
      一些航空电子系统使用几个商用计算机。这样更进一步，每个计算机都在模拟它们自身的几个副本，重要的程序同时在几个计算机上运行并且进行投票控制（vote）。
     </div>
     <div class="para">
      模拟环境的优点就是即使一个计算机出现故障，软件的不同例程能够迁移到正常工作的软件分区，表决的票数并不受影响。
     </div>
     <div class="para">
      通常虚拟软件运行在计算机的用户模式下，它捕捉、模拟硬件访问和不在用户模式下运行的指令。
     </div>
     <h4 class="headline-2">
      <a name="15_20">
      </a>
      <a name="sub6115_15_20">
      </a>
      <span class="headline-content">
       检查点计算
      </span>
     </h4>
     <div class="para">
      另外一种常用的机制是两个计算机计算 for a bit，然后将它们的计算结果报到那个点上。如果一个计算机的计算是 nut，它就会被关闭。
     </div>
     <h4 class="headline-2">
      <a name="15_21">
      </a>
      <a name="sub6115_15_21">
      </a>
      <span class="headline-content">
       办公用（宏内核）操作系统
      </span>
     </h4>
     <div class="para">
      这种系统通常在没有系统经费的嵌入式项目中流行，但是从这篇文章的多个作者来看，这通常是不正确的，它们的逻辑是：
     </div>
     <div class="para">
      操作系统是经过特殊封装的重用代码库。如果这些代码有用，设计人员就会节省时间和金钱，否则它们就是无用的。
     </div>
     <div class="para">
      商务系统的操作系统没有嵌入式硬件的接口。例如，如果要用Linux写一个马达控制器或者电话交换机，绝大部分的实际控制操作都是
      <a href="http://baike.baidu.com/view/1081282.htm" rel="nofollow noopener noreferrer" target="_blank">
       IOCTL
      </a>
      调用，同时，正常的读、写和查询界面都是无用的。所以操作系统对于实际的开发妨碍很大。
     </div>
     <div class="para">
      大多数的嵌入式系统不处理办公事务，所以办公操作系统的大部分代码都被浪费了。例如，绝大多数的嵌入式系统从来都不使用文件系统或者屏幕，所以文件系统和图形用户界面部分就是浪费的，这些不用的代码只会影响系统的可靠性。
     </div>
     <div class="para">
      办公用操作系统保护硬件不让用户程序操作，这就严重地妨碍了嵌入式开发工作。
     </div>
     <div class="para">
      操作系统必须移植到嵌入式系统上，也就是说，硬件驱动程序无论如何都必须重写，这也是操作系统最难的部分，所以使用这样的操作系统几乎没有功效。
     </div>
     <div class="para">
      操作系统真正有用、可移植的特性是小段代码。例如，一个基本的
      <a href="http://baike.baidu.com/view/7729.htm" rel="nofollow noopener noreferrer" target="_blank">
       TCP/IP
      </a>
      接口大约有3000行代码，另外的一个例子是大约同样大小的简单文件系统。如果设计需要这些代码，能够使用少于嵌入式系统开发10%的经费就能完成，不需要支付版权费用，只需要简单地重写就可以了。如果系统这些代码有足够的通用性，嵌入式系统杂志封面的后面通常有公司销售没有版权费的C语言实现代码。
     </div>
     <div class="para">
      然而许多任务程师并不认同这一点，
      <a href="http://baike.baidu.com/view/1195294.htm" rel="nofollow noopener noreferrer" target="_blank">
       嵌入式Linux
      </a>
      越来越受到人们的欢迎，尤其是在无线
      <a href="http://baike.baidu.com/view/1360.htm" rel="nofollow noopener noreferrer" target="_blank">
       路由器
      </a>
      和
      <a href="http://baike.baidu.com/view/68567.htm" rel="nofollow noopener noreferrer" target="_blank">
       全球定位系统
      </a>
      这样的功能强大的嵌入式系统中。其中有下面一些原因：
     </div>
     <div class="para">
      有现成的移植到普通嵌入式芯片的实现代码。
     </div>
     <div class="para">
      它们允许重用公开获得的
      <a href="http://baike.baidu.com/view/15565.htm" rel="nofollow noopener noreferrer" target="_blank">
       设备驱动程序
      </a>
      、网页服务器、
      <a href="http://baike.baidu.com/view/3067.htm" rel="nofollow noopener noreferrer" target="_blank">
       防火墙
      </a>
      或者其他代码。
     </div>
     <div class="para">
      开发系统能够从许多特性组合开始，在发布的时候可以剔除不需要的功能，从而节约所消耗的内存。
     </div>
     <div class="para">
      许多任务程师认为在用户模式运行应用程序更为可靠、更容易调试，所以开发过程更容易、代码更易于移植。
     </div>
     <div class="para">
      许多嵌入式系统没有控制系统那样对于实时性的严格要求，对于许多应用来说如嵌入式Linux这样的系统的响应速度就已经足够了。
     </div>
     <div class="para">
      要求更快的响应速度，而不是可靠性的特性，通常放到了
      <a href="http://baike.baidu.com/view/25278.htm" rel="nofollow noopener noreferrer" target="_blank">
       硬件
      </a>
      上。
     </div>
     <div class="para">
      许多实时操作系统都针对每件产品收取费用，当产品是消费品时，这项费用是巨大的。
     </div>
     <h4 class="headline-2">
      <a name="15_22">
      </a>
      <a name="sub6115_15_22">
      </a>
      <span class="headline-content">
       Exotic custom 操作系统
      </span>
     </h4>
     <div class="para">
      一些系统需要安全、及时、可靠或者高效的特性，上面的架构却无法达到，构建这样的系统有一些广为人知的技巧：
     </div>
     <div class="para">
      1.雇佣一个真正的系统程序员。他们的花费很小，但是能够节约数年的调试以及相关的收入损失。
     </div>
     <div class="para">
      2.RMA （rate monotonic analysis），可以用来评估一组任务能否在特定的硬件系统下运行，在最简单的形式下，设计人员保证最快完成的任务有最高的优先级，平均来说 CPU 至少有 30% 的空闲时间。
     </div>
     <div class="para">
      3.和谐的任务能够高效地优化CPU。基本上来说，设计人员保证每件工作都是从heartbeat timer开始工作的。在
      <a href="http://baike.baidu.com/view/18308.htm" rel="nofollow noopener noreferrer" target="_blank">
       实时操作系统
      </a>
      上很难这样做，因为它们在等候输入输出设备的时候通常就会切换任务。
     </div>
     <div class="para">
      4.刚好有两个优先级（通常是
      <em>
       运行
      </em>
      和
      <em>
       禁止中断
      </em>
      ）的系统不能出现高优先级任务等候低优先级任务释放信号灯或者其他资源的
      <a href="http://baike.baidu.com/view/10655827.htm" rel="nofollow noopener noreferrer" target="_blank">
       优先级倒置
      </a>
      问题，
     </div>
     <div class="para">
      5.有监视器（monitor）的系统不能出现死锁。
      <a href="http://baike.baidu.com/view/45702.htm" rel="nofollow noopener noreferrer" target="_blank">
       监视器
      </a>
      锁住一段代码禁止中断和其他抢先任务。如果这个监视器只用于一小段快速运行的代码，系统可能工作正常。如果能够证明监视器 API 在所有情况下都能够完整运行，例如仅仅禁止中断，那么就不会产生系统挂起的情况。
     </div>
     <div class="para">
      这就意味着使用两个优先级和监视器的系统是安全、可靠的，因为它们没有死锁和优先级倒置的问题。如果监视器能够执行到结束，那么就永远不会挂起。如果使用和谐的任务，可能还会相当高效。然而，RMA 无法描述这些系统，优先级最好也不要到处都有，其中包括操作系统和硬件。
