     <p style="">
      <span style="">
       <a href="http://www.tiobe.com/tiobe-index" rel="nofollow noopener noreferrer" style="" target="_blank">
        TIOBE
       </a>
      </span>
      <span style="">
       指数是用来反映某种编程语言的流程程度的指标，根据
      </span>
      <span style="">
       2016
      </span>
      <span style="">
       年
      </span>
      <span style="">
       12
      </span>
      <span style="">
       月份最新数据显示，
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       编程语言占比
      </span>
      <span style="">
       1.380%
      </span>
      <span style="">
       ，排名
      </span>
      <span style="">
       22
      </span>
      <span style="">
       位。
      </span>
      <span style="">
       编程语言本质上是人类用来与机器沟通，并在人类之间分享思维的工具，与它所需要解决的问题领域紧密相关。世界上也没有哪一种语言能够解决所有问题。所以尽管计算机领域出现了超过上千种编程语言，但终究只有少数强大语言能生存下来，
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       便是其中之一。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言要入门很快，但要精通则需要较长时间，尤其是要掌握
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       特有的，在普通编程语言里面没有的一些功能。
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       作为领域特定的第四代编程语言（
      </span>
      <span style="">
       <a href="https://en.wikipedia.org/wiki/Fourth-generation_programming_language" rel="nofollow noopener noreferrer" style="" target="_blank">
        4GL
       </a>
      </span>
      <span style="">
       ），与广泛流行的第三代编程语言
      </span>
      <span style="">
       C
      </span>
      <span style="">
       ，
      </span>
      <span style="">
       C++
      </span>
      <span style="">
       ，
      </span>
      <span style="">
       Java
      </span>
      <span style="">
       ，
      </span>
      <span style="">
       C#
      </span>
      <span style="">
       ，
      </span>
      <span style="">
       Javascript
      </span>
      <span style="">
       不同，它专门为数据分析和报告提供非常复杂的数据操作、图形图表制作以及文档编写功能。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言总体上是面向过程的计算机语言，有着传统编程语言的基本结构。但它不支持面向对象，尽管从
      </span>
      <span style="">
       SAS 9
      </span>
      <span style="">
       开始提供两个预定义的对象：
      </span>
      <span style="">
       Hash
      </span>
      <span style="">
       和
      </span>
      <span style="">
       HIter
      </span>
      <span style="">
       （
      </span>
      <span style="">
       Hash Iterator
      </span>
      <span style="">
       ）并提供类似面向对象的语法，但用户至今不能用
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言创建自定义类。
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       提供强大的互操作能力，可以调用
      </span>
      <span style="">
       Java
      </span>
      <span style="">
       对象和
      </span>
      <span style="">
       Win32 API
      </span>
      <span style="">
       函数来实现各种复杂功能。
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言也并不像
      </span>
      <span style="">
       Python/Perl
      </span>
      <span style="">
       等严格意义上的脚本式编程语言，具有真正的脚本
      </span>
      <span style="">
       /
      </span>
      <span style="">
       流控制构造。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言中只有两种基本的数据类型：字符型和数值型。从外部数据到
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       内部数据存储表达之间，可以使用
      </span>
      <span style="">
       INFORMAT
      </span>
      <span style="">
       和
      </span>
      <span style="">
       FORMAT
      </span>
      <span style="">
       进行读／写转换。然而，在一些
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       步中，
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       也提供特定的数据类型来覆盖这两种基本类型，比如
      </span>
      <span style="">
       PROC FCMP
      </span>
      <span style="">
       （从
      </span>
      <span style="">
       SAS 9.1
      </span>
      <span style="">
       开始）中类似Ｃ语言的结构体，以及
      </span>
      <span style="">
       PROC IML
      </span>
      <span style="">
       中的矩阵。与传统编程语言支持各种各样的基本数据类型（比如布尔
      </span>
      <span style="">
       /
      </span>
      <span style="">
       整数
      </span>
      <span style="">
       /
      </span>
      <span style="">
       浮点
      </span>
      <span style="">
       /
      </span>
      <span style="">
       字符
      </span>
      <span style="">
       …
      </span>
      <span style="">
       ）不同，统计学上的数据类型根据计量尺度只有定类（如性别）、定序（如年级）、定距（如温度）以及定比（重量）等四种需要考虑。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言提供非常强大预处理器，实现编译前的宏替换功能。它允许
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       程序在编译和运行期间，动态改变程序自身，并可实现递归调用宏本身。使用
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言的统计分析人员经常赞叹
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言的强大功能，而不熟悉
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言的编程人员有时也会为它与第三代编程语言的不同而困惑不解，迷失在宏与非宏的代码陷阱里。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言是一门比较复杂的计算机语言，甚至它到底是编译执行还是解释执行有时候连有经验的
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       开发人员也会感到困惑。由于
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言包含灵活的语言元素，
      </span>
      <span style="">
       SAS Macro
      </span>
      <span style="">
       宏是由解释器展开，但是
      </span>
      <span style="">
       DATA Step
      </span>
      <span style="">
       和
      </span>
      <span style="">
       Proc Step
      </span>
      <span style="">
       则以步为单位进行一次编译，然后统一执行而非逐语句进行解释执行。
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       是兼具编译和解释的混合型计算机语言，因此维基百科的分类中也很难将
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言归入编译还是解释类别（
      </span>
      <span style="">
       <a href="https://en.wikipedia.org/wiki/Compiled_language" rel="nofollow noopener noreferrer" style="" target="_blank">
        Compiled_language
       </a>
       v.s
      </span>
      <a href="https://en.wikipedia.org/wiki/Interpreted_language" rel="nofollow noopener noreferrer" style="" target="_blank">
       Interpreted_language
      </a>
      <span style="">
       ）。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       语言最强大的能力是为分析编程人员提供了完备细致的数据访问，而不用太考虑数据的存储格式和存储位置，比如最常用的
      </span>
      <span style="">
       DATA Step
      </span>
      <span style="">
       和
      </span>
      <span style="">
       PROC SQL
      </span>
      <span style="">
       就提供了各种各样的数据操作能力，而丰富的
      </span>
      <span style="">
       PROC Step
      </span>
      <span style="">
       支持则让分析人员专注于分析本身和参数设定。一旦
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       编程入门，则只有不懂的统计类型，没有不会使用的
      </span>
      <span style="">
       PROC Step
      </span>
      <span style="">
       。
      </span>
     </p>
     <p style="">
      <span style="">
       从传统语言编程基础转换到
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言编程，首先要记得如下一些
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言的核心规则：
      </span>
     </p>
     <p style="">
      <span style="">
       1.
       <span style="">
       </span>
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       程序由一系列
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语句组成，所有的语句都以分号
      </span>
      <span style="">
       ;
      </span>
      <span style="">
       结束。
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       2.
       <span style="">
       </span>
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       代码中也可以包含数据行，但数据行不作为语句，不以分号结尾。
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       3.
       <span style="">
       </span>
      </span>
      <span style="">
       一个
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语句可以跨行，多个
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语句也可以在一行上；
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语句可以从一行中的任意位置开始，代码缩进并非必需。
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       4.
       <span style="">
       </span>
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语句中的关键字是以空格分隔的，通常由“关键字”或“关键字
      </span>
      <span style="">
       =
      </span>
      <span style="">
       参数”系列组成；某些语句可以在必选项和可选项之间用
      </span>
      <span style="">
       /
      </span>
      <span style="">
       分隔。
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       5.
       <span style="">
       </span>
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言不区分大小写，你可以使用大写
      </span>
      <span style="">
       /
      </span>
      <span style="">
       小写以及它们的混合。但字符变量的值是区分大小写的。比如
      </span>
      <span style="">
       “Hello World”
      </span>
      <span style="">
       和
      </span>
      <span style="">
       “HELLO WORLD”
      </span>
      <span style="">
       为不同的两个值。
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       6.
       <span style="">
       </span>
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       代码中标识符长度较短：逻辑库引用
      </span>
      <span style="">
       /
      </span>
      <span style="">
       文件引用（
      </span>
      <span style="">
       libref/fileref
      </span>
      <span style="">
       ）名称最长不超过
      </span>
      <span style="">
       8
      </span>
      <span style="">
       字节、数据表
      </span>
      <span style="">
       /
      </span>
      <span style="">
       数据列（即变量）名称最多不超过
      </span>
      <span style="">
       32
      </span>
      <span style="">
       字节。
      </span>
     </p>
     <p style="">
      <span style="">
       SAS
      </span>
      <span style="">
       程序主要由一系列的
      </span>
      <span style="">
       Steps
      </span>
      <span style="">
       构成，而
      </span>
      <span style="">
       Step
      </span>
      <span style="">
       由一些列的
      </span>
      <span style="">
       Statements
      </span>
      <span style="">
       构成。每一个
      </span>
      <span style="">
       Step
      </span>
      <span style="">
       都有开始和结束的边界，
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       根据
      </span>
      <span style="">
       Step
      </span>
      <span style="">
       边界独立编译和执行。
      </span>
     </p>
     <p style="">
      <a href="http://igeekbar.com/igeekbar/networks/uploadimg/4f105107-78f9-43f2-a4c4-ec27a0a022ee.png" rel="nofollow noopener noreferrer" style="" target="_blank">
       <img alt="" class="img-responsive center-block" src="http://igeekbar.com/igeekbar/networks/uploadimgthumb/4f105107-78f9-43f2-a4c4-ec27a0a022ee.png" style=""/>
      </a>
     </p>
     <p style="">
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       一个所谓的步（
      </span>
      <span style="">
       Step
      </span>
      <span style="">
       ）由
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       或
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       语句开始，默认结束于下一个
      </span>
      <span style="">
       DATA/PROC
      </span>
      <span style="">
       步的开始处。但我们通常用
      </span>
      <span style="">
       RUN;
      </span>
      <span style="">
       语句来显式结束一个
      </span>
      <span style="">
       Step
      </span>
      <span style="">
       ．有一些
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       需要以
      </span>
      <span style="">
       QUIT
      </span>
      <span style="">
       语句来结束提交。为了让代码具有良好的可读性，通常使用
      </span>
      <span style="">
       RUN/QUIT
      </span>
      <span style="">
       来显式结束一个步。
      </span>
     </p>
     <p style="">
      <span style="">
       1.
       <span style="">
       </span>
      </span>
      <span style="">
       全局语句：在
      </span>
      <span style="">
       DATA/PROC
      </span>
      <span style="">
       步之外，用于指定全局的选项或者其他全局性的功能。
      </span>
      <span style="">
       <br style=""/>
      </span>
      <span style="">
       比如
      </span>
      <span style="">
       TITLE
      </span>
      <span style="">
       语句用来指定输出报告的标题，最多达
      </span>
      <span style="">
       11
      </span>
      <span style="">
       级。
      </span>
     </p>
     <pre style=""><span style="">TITLE</span><span style=""> </span><span style="">"The title of my first report"</span><span style="">; </span><span style="">/*</span><span style="">设置输出报告的标题</span><span style="">*/<br style=""/></span><span style="">TITLE2</span><span style=""> </span><span style="">"Author: Yinliang Wu"</span><span style="">; </span><span style=""> <br style=""/></span><span style=""> </span></pre>
     <p style="">
      <span style="">
       各种
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       选项（
      </span>
      <span style="">
       SAS Options
      </span>
      <span style="">
       ）语句也属于全局语句，功能上类似于操作系统的环境变量，用来指定当前
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       会话有关的系统设置。
      </span>
      <span style="">
       <br style=""/>
      </span>
     </p>
     <pre style=""><span style="">options</span><span style=""> </span><span style="">ls</span><span style="">=</span><span style="">80</span><span style=""> </span><span style="">ps</span><span style="">=</span><span style="">24</span><span style=""> </span><span style="">nocenter</span><span style="">; </span><span style="">/*</span><span style="">设置输出为</span><span style=""> 24 X 80 </span><span style="">左对齐</span><span style="">*/<br style=""/></span><span style=""> </span></pre>
     <p style="">
      <span style="">
       2.
       <span style="">
       </span>
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       步（
      </span>
      <span style="">
       Data Step
      </span>
      <span style="">
       ）：
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       程序只包含两种类型的步
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       步和
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       步，
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       步负责为后续数据步
      </span>
      <span style="">
       /
      </span>
      <span style="">
       过程步准备数据，基本语法为：
      </span>
     </p>
     <pre style=""><span style="">DATA</span><span style=""> MYDATA; <br style=""/></span><span style=""> 
&lt;STATEMENT and/or DATA&gt;;<br style=""/></span><span style="">RUN</span><span style="">;</span></pre>
     <p style="">
      <span style="">
       比如：如下代码创建一行
      </span>
      <span style="">
       5
      </span>
      <span style="">
       列的数据，其中
      </span>
      <span style="">
       Name
      </span>
      <span style="">
       和
      </span>
      <span style="">
       Sex
      </span>
      <span style="">
       是字符变量，而其他
      </span>
      <span style="">
       3
      </span>
      <span style="">
       个为数值变量。
      </span>
     </p>
     <pre style=""><span style="">DATA</span><span style=""> MyData; <br style=""/></span><span style="">  </span><span style="">input</span><span style=""> Name $ Sex $ Age Height Weight;<br style=""/></span><span style="">  </span><span style="">datalines</span><span style="">;<br style=""/></span><span style="">YINLIANG M 30 175 83.5 <br style=""/></span><span style="">RUN</span><span style="">;</span></pre>
     <p style="">
      <span style="">
       3.
       <span style="">
       </span>
      </span>
      <span style="">
       PROC 步（PROC Step）：过程步是执行特定任务的SAS语句 的集合，以 PROC 语句开始，一般到下一个 RUN语句结束；某些PROC（如PROC SQL）允许有多条RUN语句进行执行，但该PROC仅在最后一个QUIT语句运行后结束。 每个过程步有自己特定的SAS 语句和命令，但也有很多过程步有相同的SAS 语句和命令。
      </span>
     </p>
     <pre style=""><span style="">PROC</span><span style=""> </span><span style="">PRINT</span><span style=""> </span><span style="">DATA</span><span style="">=MyData;
</span><span style="">    </span><span style="">VAR</span><span style=""> Name Height;<br style=""/></span><span style="">RUN</span><span style="">;</span></pre>
     <pre style=""><span style="">PROC</span><span style=""> </span><span style="">CONTENTS</span><span style=""> </span><span style="">DATA</span><span style="">=MyData;<br style=""/></span><span style="">RUN</span><span style="">;</span></pre>
     <p style="">
      <span style="">
       DATA
      </span>
      <span style="">
       =MyData
      </span>
      <span style="">
       是
      </span>
      <span style="">
       PROC PRINT
      </span>
      <span style="">
      </span>
      <span style="">
       和
      </span>
      <span style="">
      </span>
      <span style="">
       PROC CONTENTS
      </span>
      <span style="">
      </span>
      <span style="">
       语句特定的参数（也叫选项），
      </span>
      <span style="">
       所有的
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       步都有
      </span>
      <span style="">
       DATA=
      </span>
      <span style="">
       选项。如果没有指定，则系统默认为执行过程中最近创建的那个数据集。
      </span>
     </p>
     <p style="">
      <span style="">
       4.
       <span style="">
       </span>
      </span>
      <span style="">
       程序注释：
      </span>
      <span style="">
      </span>
     </p>
     <p style="">
      <span style="">
       a)
       <span style="">
       </span>
      </span>
      <span style="">
       块注释：
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       支持
      </span>
      <span style="">
       C
      </span>
      <span style="">
       、
      </span>
      <span style="">
       C++
      </span>
      <span style="">
       和
      </span>
      <span style="">
       Java
      </span>
      <span style="">
       等广泛使用的块注释
      </span>
      <span style="">
       /* …
      </span>
      <span style="">
       注释
      </span>
      <span style="">
       … */
      </span>
      <span style="">
      </span>
      <span style="">
       功能，注释可以是任何长度，可包括分号但不能嵌套使用块注释；
      </span>
     </p>
     <pre style=""><span style="">/* …<br style=""/></span><span style=""> *
</span><span style="">…注释…</span><span style="">;<br style=""/></span><span style=""> *
…<br style=""/></span><span style=""> */<br style=""/></span><span style="">PROC</span><span style=""> </span><span style="">CONTENTS</span><span style=""> </span><span style="">DATA</span><span style="">=MyData;
</span><span style="">RUN</span><span style="">;</span></pre>
     <p style="">
     </p>
     <p style="">
      <span style="">
       b)
       <span style="">
       </span>
      </span>
      <span style="">
       行注释：以星号
      </span>
      <span style="">
       *
      </span>
      <span style="">
       开始，以往后的第一个分号
      </span>
      <span style="">
       ;
      </span>
      <span style="">
       结束。比如：
      </span>
     </p>
     <pre style=""><span style="">* </span><span style="">…注释…</span><span style="">;<br style=""/></span><span style="">PROC</span><span style=""> </span><span style="">CONTENTS</span><span style=""> </span><span style="">DATA</span><span style="">=MyData;
</span><span style="">* </span><span style="">…注释…</span><span style="">;<br style=""/></span><span style="">  </span><span style="">* </span><span style="">…注释…</span><span style="">;<br style=""/></span><span style="">RUN</span><span style="">;</span></pre>
     <p style="">
     </p>
     <p style="">
      <span style="">
       注释通常用于说明程序的功能、标注非执行文本或者用于文档目的，可以在代码调试过程中将调好的代码暂时隔离。
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       运行时注释中的代码会被忽略，但注释也会被写入日志文件。
      </span>
      <span style="">
      </span>
      <span style="">
      </span>
      <span style="">
       需要特别注意的一点是，在
      </span>
      <span style="">
       SAS Macro
      </span>
      <span style="">
       宏代码要谨慎使用行注释，以免导致不期望的宏展开。
      </span>
     </p>
     <p style="">
     </p>
     <p style="">
      <span style="">
       结语：
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       的
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       步和
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       步是
      </span>
      <span style="">
       SAS
      </span>
      <span style="">
       语言对数据分析工作的精妙抽象和完美封装，
      </span>
      <span style="">
       DATA
      </span>
      <span style="">
       步解决数据结构和准备问题，而
      </span>
      <span style="">
       PROC
      </span>
      <span style="">
       步解决对特定分析流程的实现和封装。你可以把它对应成传统编程中的数据结构和算法设计，分别负责数据和逻辑的实现。
      </span>
     </p>
