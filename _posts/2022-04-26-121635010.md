     计算机网络
    </h2>
    <h3>
     <a id="_2">
     </a>
     一、概述
    </h3>
    <h4>
     <a id="12_4">
     </a>
     1.2、因特网的概述
    </h4>
    <p>
     互联网是
     <strong>
      网络的网络
     </strong>
    </p>
    <p>
     <strong>
      网络
     </strong>
     把许多
     <strong>
      计算机
     </strong>
     连接在一起，而
     <strong>
      互联网
     </strong>
     则把许多
     <strong>
      网络
     </strong>
     连接接
    </p>
    <blockquote>
     <p>
      因特网是世界上最大的互联网
     </p>
    </blockquote>
    <p>
     三个阶段
    </p>
    <ul>
     <li>
      <p>
       第一阶段：从单个网络
       <strong>
        ARPANET
       </strong>
       向互联网发展
      </p>
      <blockquote>
       <p>
        注意两个相差很大的名词：internet和Internet
       </p>
       <ul>
        <li>
         <strong>
          internet
         </strong>
         （
         <strong>
          互联网
         </strong>
         或
         <strong>
          互连网
         </strong>
         ）是一个通用名词，他泛指多个计算机网络互联而成的网络 这些网络之间的通信协议可以是任意的。
        </li>
        <li>
         <strong>
          Internet
         </strong>
         （
         <strong>
          因特网
         </strong>
         ） 是一个专用名词，他只当前全球最大的、开放的、由众多网络相互连接而成的特定计算机网络，它采用TCP/IP协议簇作为通信规则，且其前身是美国的
         <strong>
          ARPANET
         </strong>
        </li>
       </ul>
      </blockquote>
     </li>
     <li>
      <p>
       第二阶段：逐步建成了三级结构的因特网
      </p>
     </li>
     <li>
      <p>
       第三阶段：逐渐形成了多层次ISP结构的因特网
      </p>
     </li>
    </ul>
    <h4>
     <a id="13_24">
     </a>
     1.3、因特网的组成
    </h4>
    <h5>
     <a id="_26">
     </a>
     因特网的边缘部分
    </h5>
    <p>
     <strong>
      客户机（Client
     </strong>
     ）和
     <strong>
      服务器（Server）
     </strong>
     在计算机网络软件上下文中指的是一种网络应用程序的工作方式（简称
     <strong>
      C/S
     </strong>
     方式）
    </p>
    <p>
     <strong>
      C/S
     </strong>
     方式是因特网网络应用最常用的方式。
    </p>
    <p>
     对等（
     <strong>
      Peer-to-Peer,P2P
     </strong>
     ）方式。在P2P方式网络应用中，通常没有固定的服务请求者和服务提供者，分布在网络中的应用进程是对等的，被称为对等方。
    </p>
    <h5>
     <a id="_34">
     </a>
     因特网的核心部分
    </h5>
    <p>
     <strong>
      路由器
     </strong>
     是实现
     <strong>
      分组交换
     </strong>
     的关键构建，其任务是转发收到的分组 是网络核心部分最重要的部分
    </p>
    <h6>
     <a id="_38">
     </a>
     电路交换
    </h6>
    <blockquote>
     <p>
      必须经过：
      <strong>
       建立连接
      </strong>
      （分配通信资源）——
      <strong>
       通话
      </strong>
      （一直占用资源）——
      <strong>
       释放连接
      </strong>
      （归还通信资源）桑格步骤的交换方式叫做
      <strong>
       电路交换。
      </strong>
     </p>
    </blockquote>
    <ul>
     <li>
      使用
      <strong>
       电路交换
      </strong>
      来传送计算机数据时，其线路的传输效率往往很低。
     </li>
     <li>
      电路交换的一个重要特点就是在通话的全部时间内，通话的两个用户始终占用端到端的通信资源。已被占用的通信线路资源在绝大部分时间里是空闲的，这造成传输效率往往很低。
     </li>
    </ul>
    <h6>
     <a id="_45">
     </a>
     分组交换
    </h6>
    <p>
     通常我们把要发送的整块数据成为一个
     <strong>
      报文
     </strong>
     。发送报文之前，先把教程的报文划分成为一个个发更小的等长的数据段，在每一个数据段前面，加上一些由必要的控制信息组成的
     <strong>
      首部(header)
     </strong>
     ，就构成了一个
     <strong>
      分组(packet)
     </strong>
     。
    </p>
    <blockquote>
     <p>
      分组中的首部包含了诸如目的地址和源地址等重要控制信息，每一个分组才能在互联网中独立地选择传送路径，并被正确地交付到分组传输的终点。
     </p>
    </blockquote>
    <p>
     最重要的分组交换机是
     <strong>
      路由器
     </strong>
     。
    </p>
    <p>
     位于网络边缘的主机和位于网络核心部分的路由器都是计算机，但它们的作用很不一样。**主机的用途是为用户进行信息处理的，**并且可以和其他主机通过网络交换信息。
     <strong>
      路由器的用途则是用来转发分组的，即进行分组交换的
     </strong>
    </p>
    <blockquote>
     <p>
      路由器暂时存储的是一个个短的分组，而不是整个报文。段分组在路由器的内存中而不是在磁盘中。这就保证了较高的交换效率。
     </p>
    </blockquote>
    <p>
     分组交换的
     <strong>
      优点
     </strong>
     ：
    </p>
    <blockquote>
     <p>
      1.高效：动态分配传输带宽，对通信链路是逐段占用
      <br/>
      2.灵活：分组独立选择合适的路由
      <br/>
      3.迅速：可以不建立连接就能向其他主机发送数据
      <br/>
      4.可靠：保证可靠的网络协议；分布式多路由的分组交换网，使网络有很好的生存性
     </p>
    </blockquote>
    <p>
     分组交换
     <strong>
      带来的问题
     </strong>
    </p>
    <blockquote>
     <p>
      1.分组在各结点存储转发时需要排队，这就会造成一定的
      <strong>
       时延。
      </strong>
      <br/>
      2.由于分组交换不像电路交换那样通过建立连接来保证通信时所需要的各种资源，因此无法确保通信时端到端所需要的带宽，在通信量较大时可能造成
      <strong>
       网络拥塞。
      </strong>
      <br/>
      2.分组必须携带的首部（里面有必不可少的控制信息）也造成了一定的
      <strong>
       开销。
      </strong>
     </p>
    </blockquote>
    <h4>
     <a id="15_70">
     </a>
     1.5、计算机网络的主要性能指标
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        速率
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        带宽
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        吞吐量
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        时延
       </strong>
      </p>
      <blockquote>
       <p>
        <strong>
         1.发送时延
        </strong>
       </p>
       <p>
        是主机或路由器将分组发送到通信线路上所需要的时间，
        <strong>
         也就是从发送分组的第一个比特算起，到该分组的最后一个比特发送到线路上所需要的时间
        </strong>
        ，也叫
        <strong>
         传输时延
        </strong>
        。
       </p>
       <p>
        <strong>
         发送时延 = 分组长度 / 发送速率
        </strong>
       </p>
      </blockquote>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       2.传播时延
      </strong>
     </p>
     <p>
      电磁波在信道中传播一定距离需要花费的时间。
     </p>
     <p>
      <strong>
       传播时延 = 信道长度/电磁波在信道上的传播塑速率
      </strong>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       3.处理时延
      </strong>
     </p>
     <p>
      主机或路由器在收到分组时要花费一定的时间处理。
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       4.排队时延：
      </strong>
     </p>
     <p>
      分组在进入路由器后再输入队列中的等待处理的时间。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       丢包率
      </strong>
     </li>
    </ul>
    <blockquote>
     <p>
      即分组丢失率，是指在一定的时间范围内，分组在传输过程中丢失的分组数量与总的分组数量的比率。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       利用率
      </strong>
     </li>
    </ul>
    <h4>
     <a id="16_106">
     </a>
     1.6、层次模型与计算机网络体系结构
    </h4>
    <p>
     具有五层协议的原理体系结构
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/7d9715920f17452c3f891cb6ffd600ab.jpeg"/>
    </p>
    <h5>
     <a id="application_layer_112">
     </a>
     应用层(application layer)
    </h5>
    <blockquote>
     <p>
      应用层是体系结构中的最高层。应用层的任务是
      <strong>
       通过应用进程间的交互来完成特定网络应用
      </strong>
      。应用层协议定义的是
      <strong>
       应用进程间通信和交互的规则
      </strong>
      。这里的
      <strong>
       进程
      </strong>
      就是主机中
      <strong>
       正在运行的程序
      </strong>
      。对于不同的网络应用需要不有不同的应用层协议。在互联网的应用层协议很多，如域名DNS，支持万维网应用的HTTP协议，支持电子邮件的SMTP协议，等等。我们把应用层的数据单元称为
      <strong>
       报文(messgae)
      </strong>
      。
     </p>
    </blockquote>
    <h5>
     <a id="transport_layer_116">
     </a>
     运输层(transport layer)
    </h5>
    <blockquote>
     <p>
      运输层的任务就是负责向
      <strong>
       两台主机中进程之间的通信提供通用的数据传输服务
      </strong>
      。应用进程利用该服务穿上那个应用层报文。所谓通用的，是指并不针对某个特定网路应用，而是多种应用可以使用同一个运输层服务。由于一台主机可以同时运行多个进程，因此运输层有复用和分用的功能。复用就是多个应用层进程可同时使用下面运输层的服务，分用和复用相反，是运输层把收到的信息分别交付给上面应用层的相关进程。
     </p>
    </blockquote>
    <p>
     运输层主要有下面两种协议：
    </p>
    <ul>
     <li>
      <strong>
       传输控制协议TCP
      </strong>
      (Transmission Control Protocol)-提供面向连接的、可靠的数据传输服务，其数据传输的单位是
      <strong>
       报文段
      </strong>
      (segment)。
     </li>
     <li>
      <strong>
       用户数据报协议 UDP
      </strong>
      (User Datagram)。 Protocol)-提供无连接的、
      <strong>
       尽最大努力(best-effort)
       <strong>
        的数据传输服务(不保证数据传输的可靠性),其数据传输的单位是
       </strong>
       用户数据报
      </strong>
      。
     </li>
    </ul>
    <h5>
     <a id="network_layer_125">
     </a>
     网络层(network layer)
    </h5>
    <blockquote>
     <p>
      <strong>
       网络层负责为分组交换网上的不同主机提供通信服务
      </strong>
      。
      <br/>
      在发送数据时，网络层吧运输层产生的报文段或者用户数据报封装成
      <strong>
       分组
      </strong>
      或者
      <strong>
       包
      </strong>
      进行传送。在TCP/IP体系中，由于网络层使用IP协议，因此分组也叫作
      <strong>
       IP数据报
      </strong>
      ，或简称
      <strong>
       数据报
      </strong>
      。
      <br/>
      请注意：不要将运输层的"用户数据报UDP"和网络层的"IP数据报"弄混，此外，无论哪一层传输的数据单元，都可以笼统的用"分组"来表示。
      <br/>
      网络层的另一个任务就是选择合适的路由，是源主机运输层所传下来的分组，能够通过网络中的路由器来找到目的主机。
     </p>
     <p>
      互联网是有大量的
      <strong>
       异构(heterogeneous)
       <strong>
        网络来通过
       </strong>
       路由器(Router)
       <strong>
        相互连接起来的。互联网使用的网络层协议是无连接的
       </strong>
       网际协议IP
      </strong>
      (Internet Protocol)和许多
      <strong>
       路由选择协议
      </strong>
      , 因此互联网的网络层也叫作
      <strong>
       网际层
      </strong>
      或
      <strong>
       IP层
      </strong>
      。
     </p>
    </blockquote>
    <h5>
     <a id="data_link_layer_134">
     </a>
     数据链路层(data link layer)
    </h5>
    <blockquote>
     <p>
      数据链路层通常简称为
      <strong>
       链路层
      </strong>
      。我们知道，两台主机之间的数据传输，总是在一段一段的链路上传送的, 这就需要使用专门的链路层的协议。在两个相邻结点之间传送数据时，数据链路层量网络层交下来的
      <strong>
       IP数据报封装成帧(frameing)
      </strong>
      ,在两个相邻节点间的链路上传送帧, 每一帧包括数据和必要的
      <strong>
       控制信息
      </strong>
      (如同步信息、地址信息、差错信息等)。
     </p>
     <p>
      在接收数据时，控制信息使接收端能够知道一个帧从哪个比特开始到哪个比特结束，这样数据链路层在收到一个帧后，就可从中提取数据部分，上交到网络层。
     </p>
     <p>
      控制信息还能使接收端能够检测到所收到的帧中有无差错。如发现有差错，数据链路层就简单的丢弃了这个出了差错的帧，以免继续在网络传输下去白白的浪费资源。如果需要改正数据在数据链路层传输时出现的差错(这就是说,数据链路层不仅要检错,还要纠错)， 那么就可以采用可靠数据传输协议来纠正出现的差错。这种方法会使数据链路层的协议复杂些。
     </p>
    </blockquote>
    <h5>
     <a id="physical_layer_142">
     </a>
     物理层(physical layer)
    </h5>
    <blockquote>
     <p>
      在物理层上所传数据的单位是
      <strong>
       比特(bit)
      </strong>
      。发送方发送1或者0时，接收方应该接收相同的1或者0，因此物理层要考虑用多大的电压代表"1"或者"0", 以及接收方如何识别发送方所发出的比特。物理层还要确定连接电缆的插头应当有多少根引脚以及各引脚如何连接。当然解释比特代表的意思，就不是物理层的任务。请注意，传递信息所利用的一些物理媒体，如双绞线、同轴电缆、光缆、无线信道等，并不是物理层协议之内而是在物理层协议的下面。因此也有人把物理层当做第0层。
     </p>
    </blockquote>
    <p>
     <strong>
      应用进程的数据在各层之间的传递过程所经理的变化
     </strong>
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/6fd07b26fcdc32a672dea765fec19d69.jpeg"/>
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/4ff3a69c7410ffe417eca2b8030bcbc3.jpeg"/>
    </p>
    <p>
     <strong>
      IP协议在因特网中的核心作用
     </strong>
    </p>
    <blockquote>
     <p>
      协议是**“水平”
      <strong>
       的，服务是
      </strong>
      “垂直”**的
     </p>
    </blockquote>
    <h3>
     <a id="_156">
     </a>
     二、物理层
    </h3>
    <h4>
     <a id="21_158">
     </a>
     2.1、物理层的协议
    </h4>
    <p>
     物理层的主要任务描述为确定与传输媒体的接口的一些特性，即：
    </p>
    <ul>
     <li>
      <strong>
       机械特性
      </strong>
      指明接口所用接线器的形状和尺寸、引线数目和排列、固定和锁定装置等等。
     </li>
     <li>
      <strong>
       电气特性
      </strong>
      指明在接口电缆的各条线上出现的电压的范围。
     </li>
     <li>
      <strong>
       功能特性
      </strong>
      指明某条线上出现的某一电平的电压表示何种意义。
     </li>
     <li>
      <strong>
       规程特性
      </strong>
      指明对于不同功能的各种可能事件的出现顺序。
     </li>
    </ul>
    <h4>
     <a id="22_167">
     </a>
     2.2、数据通信的基础知识
    </h4>
    <p>
     一个数据通信系统分为三大部分：
    </p>
    <p>
     <strong>
      源系统
     </strong>
     （
     <strong>
      发送端
     </strong>
     、
     <strong>
      发送方
     </strong>
     ）、
     <strong>
      传输系统
     </strong>
     （
     <strong>
      或传输网络
     </strong>
     ）、
     <strong>
      和目的系统（接收端、接收方）
     </strong>
    </p>
    <p>
     源系统一般包含以下两个部分
    </p>
    <p>
     <strong>
      源点（Source）
     </strong>
     ：源点设备产生要传输的数据，例如，从PC的键盘输入汉字，PC产生输出的数字币特列流。
    </p>
    <p>
     源点又称为
     <strong>
      源站
     </strong>
     或
     <strong>
      信源
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       发送器：
      </strong>
      源点生成的数字比特流要通过发送器编码后才能够在传输系统中进行传输。典型的发送器就是
      <strong>
       调制器
      </strong>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       接收器：
      </strong>
      接收传输系统传送过来的信号，并把它转换为能够被目的设备处理的信息。典型的接收器就是
      <strong>
       解调器
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      终点（Destination）：
     </strong>
     重点设备从接收器获取传送来的数字比特流，然后进行信息输出。例如把汉字在PC屏幕上显示出来。终点又称为
     <strong>
      目的站
     </strong>
     或
     <strong>
      信宿
     </strong>
    </p>
    <p>
     一些术语：
    </p>
    <p>
     **消息：**例如语音，文字，图像等都是消息。
    </p>
    <p>
     **数据：**是运送消息的实体。
    </p>
    <p>
     **信号：**是数据的电气或电磁的表现
    </p>
    <h5>
     <a id="_193">

    </h5>
    <p>
     信道可以分成传送模拟信号的
     <strong>
      模拟信道
     </strong>
     和传送数字信号的
     <strong>
      数字信道
     </strong>
     两大类
    </p>
    <p>
     调制：将
     <strong>
      数字信号
     </strong>
     转换成
     <strong>
      模拟信号
     </strong>
     的过程称为调制
    </p>
    <h6>
     <a id="_199">
     </a>
     常用的编码方式
    </h6>
    <p>

    </h6>

     <strong>

     </strong>
     进行调制，把基带信号的频率范围搬移导较高的频段以便在信道中传输。
    </p>
    <p>
     经过载波调制后的信号称为
     <strong>
      带通信号
     </strong>
    </p>
    <p>
     使用载波的调制称为
     <strong>
      带通调制
     </strong>
    </p>
    <p>
     最基本的带通调制方法有3种：
    </p>

     <li>

     </li>
     <li>

     </li>
     <li>

     </li>

     <strong>
      幅移键控、频移键控、和相移键控
     </strong>
    </p>
    <p>
     实现调制和解调功能的设备称为
     <strong>
      调制解调器
     </strong>
    </p>
    <h5>
     <a id="_229">
     </a>
     信道的极限容量
    </h5>
    <p>
     <strong>
      码元
     </strong>
     是承载信息的基本信号单位。
    </p>
    <h6>
     <a id="_233">
     </a>
     限制码元在信道上的传输速率的因素
    </h6>
    <p>
     1、信道能够通过的频率范围
    </p>
    <p>
     <strong>
      奈氏准则
     </strong>
     ：
    </p>
    <blockquote>
     <p>
      理想低通信道的最高码元传输速率 = 2W Baud
     </p>
     <p>
      理想带通信道的最高码元传输速率 = W Baud
     </p>
    </blockquote>
    <p>
     W是信道带宽，单位是Hz， Baud 是波特，是码元传输速率的单位，1波特为每秒传送1个码元
    </p>
    <p>
     奈氏准则的另一种表达方法是：
     <strong>
      每赫兹带宽的理想低通信道的最高码元传输速率是每秒2个码元，每赫兹带宽的理想带通信道的最高码元传输速率是每秒1个码元
     </strong>
    </p>
    <p>
     <strong>
      波特和比特是两个不同的概念
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       波特
      </strong>
      是
      <strong>
       码元传输的速率单位
      </strong>
      ，它说明每秒传多少个码元。码元传输速率也成为
      <strong>
       调制速率、波形速率
      </strong>
      、或
      <strong>
       符号速率
      </strong>
     </p>
     <p>
      比特是
      <strong>
       信息量的单位
      </strong>
     </p>
    </blockquote>
    <p>
     2、信噪比
    </p>
    <p>
     信道的极限信息传输速率C：
     <strong>
      香农公式
     </strong>
    </p>
    <blockquote>
     <p>
      C = Wlog2(1+S/N) bit/s
     </p>
    </blockquote>
    <p>
     W为信道的带宽，S为信道内所传信号的平均功率；N为信道内不的高斯噪声功率
    </p>
    <p>
     香浓公式表明：信道的带宽或信道中的信噪比越大，则信息的极限传输速率就越高
    </p>
    <h5>
     <a id="_265">
     </a>
     传输方式
    </h5>
    <h6>
     <a id="1_267">
     </a>
     1、并行传输和串行传输
    </h6>
    <blockquote>
     <p>
      <strong>
       并行传输
      </strong>
      ：是指一次发送n个比特而不是一个比特，为此，在发送端和接收端之间需要有n条传输线路
     </p>
     <p>
      <strong>
       串行传输
      </strong>
      ：是指数据是一个比特一个比特依次发送的，因此在发送端和接收端之间需要有1条传输线路即可
     </p>
    </blockquote>
    <p>
     并行传输的优点是速度为串行传输的n倍，计算机内部经常用并行传输，在将数据发送到传输线路时候，要进行并/串转换
    </p>
    <h6>
     <a id="2_275">
     </a>
     2、异步传输和同步传输
    </h6>
    <p>
     <strong>
      异步传输
     </strong>
     ：
    </p>
    <blockquote>
     <ul>
      <li>
       以字节为独立的传输单位，字节之间的时间间隔不是固定
      </li>
      <li>
       接收端仅在每个字节的起始处对字节内的比特实现同步
      </li>
      <li>
       通常在每个字节前后分别加上起始位和结束位
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      同步传输：
     </strong>
    </p>
    <blockquote>
     <ul>
      <li>
       数据块以稳定的比特流的形式传输。字节之间没有间隔
      </li>
      <li>
       接收端在每个比特信号的中间时刻进行检测，以判别接收到的是比特0还是比特1
      </li>
      <li>
       由于不同设备的时钟频率存在一定差异，不可能做到完全相同，在传输大量数据的过程中，所产生的判别时刻的累计误差，会导致接收端对比特信号的判别错位
      </li>
     </ul>
    </blockquote>
    <p>
     3、单工、半双工和全双工
    </p>
    <p>
     <strong>
      单向通信(单工)
     </strong>
    </p>
    <blockquote>
     <p>
      即只能有一个方向的通信而没有反方向的交互。无线电广播或有线电以及电视广播就属于这种类型
     </p>
    </blockquote>
    <p>
     <strong>
      双向交替通信(半双工)
     </strong>
    </p>
    <blockquote>
     <p>
      即通信的双方可以发送信息，但不能双方同时发送（当然也就不能同时接收）。这种通信方式使一方发送另一方接收，过一段时间后可以再反过来
     </p>
    </blockquote>
    <p>
     <strong>
      双向同时通信(全双工)
     </strong>
    </p>
    <blockquote>
     <p>
      通信的双发可以同时发送和接收信息。
     </p>
    </blockquote>
    <p>
     单向通信只需要一条信道，而双向交替通信或双向同时通信则需要两条信道（每个方向各一条），双向同时通信的传输效率最高
    </p>

    </h4>
    <h3>
     <a id="_341">
     </a>
     三、数据链路层
    </h3>
    <h4>
     <a id="31_343">
     </a>
     3.1、数据链路层的概述
    </h4>
    <ul>
     <li>
      <strong>
       链路
      </strong>
      ：就是从一个节点到相邻节点的一段物理线路，而中间没有任何其他的交换结点
     </li>
     <li>
      <strong>
       数据链路
      </strong>
      ：是指把实现通信协议的硬件和软件加到链路上，就构成了数据链路。
     </li>
     <li>
      数据链路层以
      <strong>
       帧
      </strong>
      为单位传输和处理数据
     </li>
    </ul>
    <p>
     使用点对点信道的数据链路层
    </p>
    <pre><code>三个重要问题：封装成帧，差错检测，可靠传输
</code></pre>
    <p>
     使用广播信道的数据链路层
    </p>
    <pre><code>共享式以太网的媒体介入控制协议CSMA/CD协议
802.11局域网的媒体介入控制协议CSMA/CA协议
</code></pre>
    <h4>
     <a id="32_362">
     </a>
     3.2、封装成帧
    </h4>
    <ul>
     <li>
      帧头和帧尾中包含有重要的控制信息
     </li>
     <li>
      帧头和帧尾的作用之一就是
      <strong>
       帧定界
      </strong>
     </li>
    </ul>
    <blockquote>
     <p>
      （帧头何帧尾中含有帧定界标志，并不是每一种数据链路层协议的帧都包含有帧定界标志，比如MAC帧不需要帧定界符，物理层会在MAC帧前面添加8字节的前导码，前7个为前同步码，后1个字节为帧开始定界符，MAC帧规定了96比特的帧结束间时间，所以不需要帧结束界定符）
     </p>
    </blockquote>
    <h5>
     <a id="_369">
     </a>
     透明传输
    </h5>
    <ul>
     <li>
      是指数据链路层对上层交付的传输数据没有任何限制，就好像数据链路层不存在一样
     </li>
    </ul>
    <blockquote>
     <p>
      面向字节的物理链路施用字节填充（或称字符填充）的方法实现透明传输。
     </p>
    </blockquote>
    <blockquote>
     <p>
      面向比特的物理链路使用比特填充的方法实现透明传输
     </p>
    </blockquote>
    <pre><code>例如：在发送前，用零比特填充发，对数据部分进行扫描，每5个连续的比特1后面就插入1个比特0，这样就确保了帧定界在整个帧中的唯一性，从而实现透明传输。接收方的数据链路层从物理层交付的比特流中提取帧时，将帧的数据部分中的每5个连续的比特1后面的那个比特0剔除即可。
</code></pre>
    <p>
     <strong>
      为了提高帧的传输效率，应当使帧的数据部分的长度尽可能大些
     </strong>
    </p>
    <p>
     考虑到差错控制等多种因素，每一种数据链层协议都规定了帧的数据部分的长度上限，即
     <strong>
      最大传送单元MTU
     </strong>
    </p>
    <h4>
     <a id="33_385">
     </a>
     3.3、差错检测
    </h4>
    <pre><code>比特在传输过程中可能会产生差错：1可能会变成0，0也可能变成1，这成为比特差错
</code></pre>
    <p>
     误码率：传输错误的比特占所传输比特总数的比率。
    </p>
    <p>
     使用
     <strong>
      差错检测码
     </strong>
     来检测数据在传输过程中是否产生了比特差错
    </p>
    <p>
     <strong>
      奇偶校验
     </strong>
    </p>
    <pre><code>在待发送的数据后面 添加1位奇偶校验位 ，使整个数据（包括所添加的校验位在内）中“1”的个数为奇数（奇校验）或（偶校验）
</code></pre>
    <p>
     如果有
     <strong>
      奇数个位发生误码
     </strong>
     ，则奇偶性发生变化，可以检查出误码：
    </p>
    <p>
     如果有
     <strong>
      偶数个位发生误码
     </strong>
     ，则奇偶性不发生变化，不可以检查出误码：
    </p>
    <p>
     <strong>
      循环冗余校验CRC
     </strong>
    </p>
    <p>
     [外链图片转存失败,源站可能有防盗链机制,建议将图片保存下来直接上传(img-MoimhxxU-1638258808615)(F:\markdown\java笔记\计算机网络.assets\image-20211124201301126.png)]
    </p>
    <ul>
     <li>
      <strong>
       检错码
      </strong>
      只能检测出帧在传输过程中出现了差错，但不能定位错误，无法纠正错误。
     </li>
     <li>
      要想纠正传输中的差错，可以使用冗余信息更多的
      <strong>
       纠错码
      </strong>
      进行
      <strong>
       前向纠错
      </strong>
      。但纠错码的开销较大，计算机网络中较少使用。
     </li>
     <li>
      循环冗余校验
      <strong>
       CRC
      </strong>
      有很好的检错能力（
      <strong>
       漏检率非常低
      </strong>
      ），虽然计算比较复杂，但非常易于用硬件实现，因此被广泛应用于数据链路层。
     </li>
     <li>
      在计算机网中通常采用我们后续课程中见要讨论的
      <strong>
       检错重传方式来纠正传输中的差错
      </strong>
      ，或者是
      <strong>
       仅仅丢弃检测到差错的帧
      </strong>
      ，这却决于数据链路层向其上层提供的是可靠传输服务还是不可靠传输服务。
     </li>
    </ul>
    <h4>
     <a id="34_414">
     </a>
     3.4、可靠传输
    </h4>
    <h5>
     <a id="341_416">
     </a>
     3.4.1可靠传输的基本概念
    </h5>
    <p>
     数据链路层向上层提供的服务类型
    </p>
    <pre><code>不可靠传输服务：仅仅丢弃有误码的帧，其他什么也不做；
不可靠传输服务： 想办法实现 发送端发送什么，接收端就收到什么。
</code></pre>
    <ul>
     <li>
      <p>

    <p>

     <li>
      MAC地址一般被固化在网卡（网络适配器）的点可擦可编程只读存储器EEPROM中，因此MAC地址也被称为
      <strong>
       硬件地址
      </strong>
      。
     </li>
     <li>
      MAC地址有时也被称为物理地址。
      <strong>
       这并不意味着MAC地址属于网络体系结构中的物理层
      </strong>
     </li>
    </ul>
    <blockquote>
     <pre><code>一般情况下，用户主机会包含两个网络适配器：有线局域网适配器（有线网卡）和无线局域网适配器（无线
网卡）。每个网络适配器都有一个全球唯一的MAC地址。而交换机和路由器往往拥有更多的网络接口，所以
会拥有更多的MAC地址。
</code></pre>
    </blockquote>
    <p>
     <strong>
      综上所述，严格来说，MAC地址是对网络上各接口的唯一标识，而不是对网络上各设备的唯一标识。
     </strong>
    </p>

    <h4>
     <a id="310STP_699">
     </a>
     3.10、以太网交换机的生成树协议STP
    </h4>
    <ul>
     <li>
      如何提高以太网的可靠性？
     </li>
     <li>
      添加冗余链路可以提高以太网的可靠性
     </li>
     <li>
      但是，冗余链路也会带来负面效应——形成
      <strong>
       网络环路
      </strong>
     </li>
    </ul>
    <p>
     网络环路会带来以下问题:
    </p>
    <ul>
     <li>
      <strong>
       广播风暴
      </strong>
      <br/>
      大量消耗网络资源，使得网络无法正常转发其他数据帧
     </li>
     <li>
      <strong>
       主机收到重复的广播帧
      </strong>
      <br/>
      大量消耗主机资源大量消耗主机资源
     </li>
     <li>
      <strong>
       交换机的帧交换表震荡（漂移）
      </strong>
     </li>
    </ul>
    <p>
     以太网交换机使用
     <strong>
      生成树协议STP
     </strong>
     (SpanningTree Protocol),可以在增加冗余链路来提高网络可靠性的同时又
     <strong>
      避免网络环路带来的各种问题
     </strong>
     。
    </p>
    <ul>
     <li>
      不论交换机之间采用怎样的物理连接，交换机都能够
      <strong>
       自动计算并构建一个逻辑上没有环路的网络
      </strong>
      ，其逻辑拓扑结必须是树型的（无逻辑环路）
     </li>
     <li>
      最终生成的树型逻辑拓扑要
      <strong>
       确保连通整个网络
      </strong>
      ;
     </li>
     <li>
      当首次连接交换机或网络
      <strong>
       物理拓扑发生变化
      </strong>
      时（有可能是人为改变或故障)交换机都将进行生成树的重新计算。
     </li>
    </ul>

    <p>

    <p>
