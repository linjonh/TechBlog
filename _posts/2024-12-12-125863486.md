     <p style="text-align:center;">
      <img alt="4bc9ab8a030b1a7df2ad9f649a2e6d8b.png" src="https://i-blog.csdnimg.cn/blog_migrate/bc67eba47f07fbdbff1c0a145fce0239.png"/>
     </p>
     <p>
      <strong>
       背景
      </strong>
     </p>
     <p style="text-align:left;">
      M2M（即Machine-to-Machine）是一种
      <strong>
       端对端通信技术
      </strong>
      。阿里云 IoT 企业物联网实例通过基于规则引擎的 Topic 转发功能，实现设备之间 M2M 通信。您不用担心高并发场景下的
      <strong>
       高并发、
      </strong>
      <strong>
       稳定性、低延时
      </strong>
      等技术难点，也不需要购买大量服务器去承载这些请求，您只需要根据自己的业务配置好转发规则即可。
      <br/>
     </p>
     <p style="text-align:justify;">
      今天，我们讲解在工业场景中 PLC 工控机和管理人员 App 的联动，此方案也同样适用于智能家居场景中温湿度传感器和空调联动，空气质量传感器和空气净化器联动等。
     </p>
     <p>
      <strong>
       整体技术方案
      </strong>
     </p>
     <p style="text-align:justify;">
      基于阿里云 IoT 企业物联网实例的设备之间 M2M 通信整体技术方案如下:
     </p>
     <p style="text-align:center;">
      <img alt="0876a65598180b216d90c8af0ba6f759.png" src="https://i-blog.csdnimg.cn/blog_migrate/a4b64faf35b27a54c7cf0c2df1e0bb75.png"/>
     </p>
     <p>
      <strong>
       设备 M2M 通信实战
       <br/>
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       1.创建产品，配置自定义Topic
      </strong>
     </p>
     <p style="text-align:justify;">
      首先，我们在物联网控制台创建手机App产品和PLC工控机，并添加自定义Topic。
     </p>
     <p style="text-align:center;">
      <img alt="39c9f57990c2e73f270615abb9c0b689.png" src="https://i-blog.csdnimg.cn/blog_migrate/daba1f46363682b953de20e56fd073f4.png"/>
     </p>
     <p style="text-align:justify;">
      <img alt="e2c15102bb59b76c96919439ffacb410.png" src="https://i-blog.csdnimg.cn/blog_migrate/b20a0d0c8ea2ff74fde38c01a3d21923.png"/>
     </p>
     <p style="text-align:justify;">
      然后，在产品下各注册一个设备。
      <br/>
     </p>
     <p style="text-align:justify;">
      <strong>
       2.配置规则引擎，Topic转发
      </strong>
     </p>
     <h4>
      <strong>
       2.1 PLC工控机上报数据流转App
      </strong>
     </h4>
     <p style="text-align:justify;">
      完成设备接入后，我们在规则引擎-云产品流转创建新规则，PLC数据实时流转App。
     </p>
     <p style="text-align:justify;">
      <img alt="05b65057a15c4e3a74fc897375a98184.png" src="https://i-blog.csdnimg.cn/blog_migrate/144d7005b4182f04069c158bc97eba8f.png"/>
     </p>
     <p style="text-align:justify;">
      ① 数据源- PLC工控机数据上报
     </p>
     <pre class="has"><code class="language-powershell">工控机上报数据 Topic：
/工控机/工控机id/user/data/up

Payload：
{
"toAppTopic":"/手机 App/手机 id/user/plc2App/data",
"t":56,
"h":68
}</code></pre>
<p style="text-align:justify;">
②  数据目的地-  手机 App 订阅
</p>
<pre class="has"><code class="language-objectivec">手机 App 订阅 Topic：
/手机 App/手机 id/user/plc2App/data</code></pre>
<p style="text-align:justify;">
③ 解析脚本
</p>
<pre class="has"><code class="language-kotlin">//通过 payload 函数，获取设备上报的消息内容，并按照 JSON 格式转换。
var data = payload("json");

// 目标 Topic
var toAppTopic = data.toAppTopic;
// App 指令流转到 PLC 设备
writeIotTopic(1009, toAppTopic, data)</code></pre>
<p style="text-align:justify;">
当 PLC 工控机有数据上报后，消息上报和规则引擎流转日志，如下：
</p>
<p style="text-align:justify;">
<img alt="eb77c23667e614816b76ef017b1fff9c.png" src="https://i-blog.csdnimg.cn/blog_migrate/2f88a25a30dbb3c5af7ba020dd2cf4bc.png"/>
</p>
<p style="text-align:justify;">
数据流转的消息轨迹
</p>
<p style="text-align:justify;">
<img alt="3dac78fd6d29df0b4fe852be9004ba04.png" src="https://i-blog.csdnimg.cn/blog_migrate/d56f1dbcf54e96b2066fb9d47226b798.png"/>
</p>
<h4>
<strong>
2.2 手机 App 推送控制指令到 PLC 工控机
</strong>
</h4>
<p style="text-align:justify;">
接下来，我们在规则引擎-云产品流转创建新规则，App 指令发送到 PLC。
</p>
<p style="text-align:justify;">
<img alt="0770cc85c78449c507d5cc30624ff06b.png" src="https://i-blog.csdnimg.cn/blog_migrate/0da820910e8474a6203e0409c4b8b353.png"/>
</p>
<p style="text-align:justify;">
①  数据源-  手机下发控制指令
</p>
<pre class="has"><code class="language-powershell">手机下发控制指令 Topic：
/手机 App/手机 id/user/app2plc/cmd

Payload：
{
    "toPlcDeviceTopic":"/PLC 工控机/工控机 id/user/app/cmd",
"light":"on"
}</code></pre>
<p style="text-align:justify;">
②  数据目的地-  工控机指令订阅
</p>
<pre class="has"><code class="language-objectivec">PLC 工控机订阅 Topic：
/PLC 工控机/工控机 id/user/app/cmd</code></pre>
<p style="text-align:justify;">
③ 解析脚本
</p>
<pre class="has"><code class="language-kotlin">//通过 payload 函数，获取设备上报的消息内容，并按照 JSON 格式转换。
var data = payload("json");

//  目标设备 Topic
var toPlcDeviceTopic = data.toPlcDeviceTopic;
// App 指令流转到 PLC 设备
writeIotTopic(1009, toPlcDeviceTopic, data)</code></pre>
<p style="text-align:justify;">
当手机 App 下发控制指令后，流转日志，如下：
</p>
<p style="text-align:justify;">
<img alt="ec7732b85b8a1087acfcf19c05fb944b.png" src="https://i-blog.csdnimg.cn/blog_migrate/da7aeafe3731d93393c839e123a72557.png"/>
</p>
<p style="text-align:justify;">
数据流转的消息轨迹如下
</p>
<p style="text-align:justify;">
<img alt="e1bfac485c5393ac83b7aa7bbf32a420.png" src="https://i-blog.csdnimg.cn/blog_migrate/63a68e7563cc73d91e5e5a89318e3d44.png"/>
</p>
<p style="text-align:justify;">
至此，我们基于阿里云 IoT 企业物联网平台完成了 PLC 工控机和手机 App 之间的实时消息通信，整个过程
<strong>
不需要业务服务器参与
</strong>
，保障了设备间消息实时和高效通信。
</p>
