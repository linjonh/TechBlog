    <h1 class="title-article" id="articleContentId">
     测试用例设计方法1——等价类
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <h4>
     一、定义
    </h4>
    <p>
     某类事物（
     <span style="color:#f33b45;">
      具有相同的属性或方法
     </span>
     ）的集合，这个集合中某个个体所表现的特征与其他个体完全一致。测试上：对于某个被测对象的测试输入而言，某个个体能够被接受或被拒绝，则该个体所在集合中的任意个体都应该被接受或拒绝。
    </p>
    <h4>
     二、等价类划分
    </h4>
    <ul>
     <li>
      <strong>
       有效等价类
      </strong>
      ：针对被测对象而言，合理的、有意义的、系统接受的输入。（
      <span style="color:#3399ea;">
       如：用户名长度在6-18的
      </span>
      ）
     </li>
     <li>
      <strong>
       无效等价类
      </strong>
      ：针对被测对象而言，不合理的、无意义的、系统不能接受的输入。（
      <span style="color:#3399ea;">
       如：用户名长度大于18位，或小于6位
      </span>
      ）
     </li>
    </ul>
    <h4>
     三、等价类划分规则
    </h4>
    <p>
     1、如果需求规定了
     <span style="color:#3399ea;">
      输入域
     </span>
     的取值个数或
     <span style="color:#3399ea;">
      确定了某个范围
     </span>
     时，则可确定
     <span style="color:#3399ea;">
      一个有效等价类
     </span>
     及
     <span style="color:#3399ea;">
      两个无效等价类
     </span>
     。如：
    </p>
    <ul>
     <li>
      <span style="color:#ffbb66;">
       有效等价类 ：用户名长度在6-18位之间。
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       无效等价类：①用户名长度大于18位；②用户名长度小于6位。
      </span>
     </li>
    </ul>
    <p>
     2、如果需求规定了某个
     <span style="color:#3399ea;">
      输入域的集合
     </span>
     ，或者
     <span style="color:#3399ea;">
      必须如何
     </span>
     的情况下，可确定
     <span style="color:#3399ea;">
      一个有效等价类
     </span>
     及
     <span style="color:#3399ea;">
      一个无效等价类
     </span>
     。如：
    </p>
    <ul>
     <li>
      <span style="color:#ffbb66;">
       有效等价类：以字母开头。
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       无效等价类：以非字母开头。
      </span>
     </li>
    </ul>
    <p>
     3、如果需求规定了某个
     <span style="color:#3399ea;">
      输入域是真假值
     </span>
     时，可确定
     <span style="color:#3399ea;">
      一个有效等价类
     </span>
     和
     <span style="color:#3399ea;">
      一个无效等价类
     </span>
     。
    </p>
    <p>
     4、如果用户需求规定了
     <span style="color:#3399ea;">
      输入域是一组值
     </span>
     ，则可确定
     <span style="color:#3399ea;">
      若干个有效等价类
     </span>
     及
     <span style="color:#3399ea;">
      一个无效等价类
     </span>
     。如：
    </p>
    <p>
     <span style="color:#ffbb66;">
      京东商城：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ffbb66;">
       钻石会员
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       金牌会员
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       铜牌会员
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       普通注册用户
      </span>
     </li>
    </ul>
    <p>
     5、用户需求规定
     <span style="color:#3399ea;">
      必须遵守某种规则
     </span>
     时，可确定
     <span style="color:#3399ea;">
      一个有效等价类
     </span>
     及若干个从
     <span style="color:#3399ea;">
      不同角度违反规则的无效等价类
     </span>
     。如，以字母开头：
    </p>
    <ul>
     <li>
      <span style="color:#ffbb66;">
       有效等价类：以字母开头。测试
      </span>
     </li>
     <li>
      <span style="color:#ffbb66;">
       无效等价类：①以特殊字母开头；②以汉字开头。
      </span>
     </li>
    </ul>
    <h4>
     四、用等价类进行测试用例设计步骤
    </h4>
    <ol>
     <li>
      根据需求，划分有效及无效等价类，有效等价类统一编号，无效等价类统一编号。
     </li>
     <li>
      设计一个新的测试用例，使其尽可能覆盖所有尚未覆盖有效等价类，直到所有有效等价类都覆盖。
     </li>
     <li>
      设计一个新的测试用例，使其反覆盖一个无效等价类，直到所有无效等价类都被覆盖。
     </li>
    </ol>
    <h4>
     五、等价类四则运算法（加、减、乘、除）
    </h4>
    <ol>
     <li>
      <strong>
       加法：
      </strong>
      不考虑需求其他子项，细致分解当前测试点及详细需求，做累加。
     </li>
     <li>
      <strong>
       减法：
      </strong>
      根据业务规则减少，排除相关不可能出现的规则，减少不可能出现的组合。
     </li>
     <li>
      <strong>
       乘法：
      </strong>
      如果有效等价类中具有互斥条件的需求时，可进行相乘得到用例个数。
     </li>
     <li>
      <strong>
       除法：
      </strong>
      排除所有具有重复性的等价类，尽可能做到有效等价类之间交集为空，无效等价类之间交集也为空，有效及无效等价类的并集为整个输入域。
     </li>
    </ol>
