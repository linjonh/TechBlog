    <h2>
     背景介绍
    </h2>
    <p>
     前端发送请求给后端，发生400报错
    </p>
    <p>
     这里一提后端接口都是测试过的，确保没问题
    </p>
    <p>
     <img alt="" height="210" src="https://i-blog.csdnimg.cn/blog_migrate/ba9e029ceccad34acce414df6d1f7b49.png" width="1200"/>
    </p>
    <h2>
     解决方法
    </h2>
    <p>
     一般发生这种情况是，
     <strong>
      request中所带参数和后端实体类对应不一致
     </strong>
     ，换句话说就是可能有字段名不一样导致无法进行包装，这就需要我们细心检查了。
    </p>
    <p>
     还有一种情况便是
     <strong>
      数据类型不一样
     </strong>
     ，比如id字段需要的是int类型，但是参数确实String类型，同样会报错。
    </p>
    <p>
     我这里好在后端显示了错误信息，显示需要的是int类型，而不是String
    </p>
    <pre><code>Field error in object 'department' on field 'deptuserid': rejected value [null]; codes [typeMismatch.department.deptuserid,typeMismatch.deptuserid,typeMismatch.java.lang.Integer,typeMismatch]; arguments [org.springframework.context.support.DefaultMessageSourceResolvable: codes [department.deptuserid,deptuserid]; arguments []; default message [deptuserid]]; default message [Failed to convert property value of type 'java.lang.String' to required type 'java.lang.Integer' for property 'deptuserid'; nested exception is java.lang.NumberFormatException: For input string: "null"]]</code></pre>
    <p>
     通过网上查阅还有一种情况，就是
     <strong>
      JSON对象和JSON字符串之间的问题
     </strong>
     ，有时也是类型不一样导致的报错。
    </p>
