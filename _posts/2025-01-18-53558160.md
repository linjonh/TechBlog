    <h4 id="进出数据库">

    </h4>
    <p>

    </p>
    <blockquote>
     <p>

    <h4 id="对数据库的处理">
     对数据库的处理:
    </h4>
    <p>
     切换数据库,相当于mysql的use dbname
    </p>
    <blockquote>
     <p>
      \c dbname
     </p>
    </blockquote>
    <p>
     列举数据库，相当于mysql的show databases
    </p>
    <blockquote>
     <p>
      \l
     </p>
    </blockquote>
    <p>
     创建数据库：
    </p>
    <blockquote>
     <p>
      create database [数据库名];
     </p>
    </blockquote>
    <p>
     删除数据库：
    </p>
    <blockquote>
     <p>
      drop database [数据库名];
     </p>
    </blockquote>
    <h4 id="对数据表的处理">
     对数据表的处理
    </h4>
    <p>
     列举表，相当于mysql的show tables
    </p>
    <blockquote>
     <p>
      \dt
     </p>
    </blockquote>
    <p>
     查看表结构，相当于desc tblname,show columns from tbname
    </p>
    <blockquote>
     <p>
      \d tblname
     </p>
     <p>
      \di 查看索引
     </p>
    </blockquote>
    <p>
     创建表：
    </p>
    <blockquote>
     <p>
      create table ([字段名1] [类型1] ;,[字段名2] [类型2],……&lt;,primary key
      <br/>
      (字段名m,字段名n,…)&gt;;);
     </p>
    </blockquote>
    <p>
     *重命名一个表：
    </p>
    <blockquote>
     <p>
      alter table [表名A] rename to [表名B];
     </p>
    </blockquote>
    <p>
     *删除一个表：
    </p>
    <blockquote>
     <p>
      drop table [表名];
     </p>
    </blockquote>
    <p>
     在表中插入数据：
    </p>
    <blockquote>
     <p>
      insert into 表名 ([字段名m],[字段名n],……) values ([列m的值],[列n的值],……);
     </p>
    </blockquote>
    <p>
     *查看数据表的内容:
    </p>
    <blockquote>
     <p>
      select * from [表名];
     </p>
    </blockquote>
    <p>
     *在已有的表里添加字段：
    </p>
    <blockquote>
     <p>
      alter table [表名] add column [字段名] [类型];
     </p>
    </blockquote>
    <p>
     *删除表中的字段：
    </p>
    <blockquote>
     <p>
      alter table [表名] drop column [字段名];
     </p>
    </blockquote>
    <p>
     *重命名一个字段：
    </p>
    <blockquote>
     <p>
      alter table [表名] rename column [字段名A] to [字段名B];
     </p>
    </blockquote>
    <p>
     *给一个字段设置缺省值：
    </p>
    <blockquote>
     <p>
      alter table [表名] alter column [字段名] set default [新的默认值];
     </p>
    </blockquote>
    <p>
     *去除缺省值：
    </p>
    <blockquote>
     <p>
      alter table [表名] alter column [字段名] drop default;
     </p>
    </blockquote>
    <p>
     修改表中的某行某列的数据：
    </p>
    <blockquote>
     <p>
      update [表名] set [目标字段名]=[目标值] where [该行特征];
     </p>
    </blockquote>
    <p>
     删除表中某行数据：
    </p>
    <blockquote>
     <p>
      delete from [表名] where [该行特征]; delete from [表名];–删空整个表
     </p>
    </blockquote>

    <h2 id="主键和外键">
     主键和外键
    </h2>
    <h3 id="概念理解">
     概念理解
    </h3>
    <h4 id="主键">
     主键
    </h4>
    <p>
     关系型数据库中的一条记录中有若干个属性，若其中某一个属性组(注意是组)能唯一标识一条记录，该属性组就可以成为一个主键。
     <br/>
     比如：
     <br/>
     1.学生表(学号，姓名，性别，班级)，其中每个学生的学号是唯一的，学号就是一个主键；
     <br/>
     2.课程表(课程编号,课程名,学分)，其中课程编号是唯一的,课程编号就是一个主键；
     <br/>
     现在来一张成绩表(学号,课程号,成绩)，
     <br/>
     成绩表中单一一个属性无法唯一标识一条记录，学号和课程号的组合才可以唯一标识一条记录，所以，学号和课程号的属性组是一个主键。
    </p>
    <h4 id="外键">
     外键
    </h4>
    <p>
     外键用于与另一张表的关联。是能确定另一张表记录的字段，用于保持数据的一致性。比如，A表中的一个字段，是B表的主键，那他就可以是A表的外键。
    </p>
    <h5 id="设计外键的目的">
     设计外键的目的
    </h5>
    <h6 id="1为了一张表记录的数据不要太过冗余">
     <strong>
      1.为了一张表记录的数据不要太过冗余
     </strong>
    </h6>
    <p>
     解释：比如想要设计下图的一张表，如果设计为上图的表显然太过于冗余，如果设计为下面的两个表，会不会更好呢（自行体会）。那么如何保证两个表之间的关联呢？注意：我们自身明白两个表之间是通过学号这个字段来连接的，可是如何让数据库明白呢？这就是外键的作用了。
     <br/>
     <img alt="总表" src="https://img-blog.csdn.net/20161210141001803?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvcXFfMzM0Mjk5Njg=/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast" title=""/>
    </p>
    <h6 id="2保持数据的一致性和完整性">
     <strong>
      2.保持数据的一致性和完整性
     </strong>
    </h6>
    <p>
     通俗的解释：
     <br/>
     比如表1中张三的学号为20140900001，那么我在表2中插数据的时候在学号字段插20140900001来记录张三的成绩不也是做到了表 的解耦了吗？
     <br/>
     这里存在的问题是，在不设置外键的情况下，表2的学号字段和表1的学号字段是没有关联的。只是你自己认为他们有关系而已，数据库并 不认为它俩有关系。也就是说，你在表2的学号字段插了一个值（比如20140999999），但是这个值在表1中并没有，这个时候，数据库还是允 许你插入的，它并不会对插入的数据做关系检查。然而在设置外键的情况下，你插入表2学号字段的值必须要求在表1的学号字段能找到。 同时，如果你要删除表1的某个学号字段，必须保证表2中没有引用该字段值的列，否则就没法删除。这就是所谓的保持数据的一致性和完整性。你想，如 果表2还引用表1的某个学号，你却把表1中的这个学号删了，表2就不知道这个学号对应的学生是哪个学生。数据的一致性还包括数据类型的一致性（这 个见下面就知道了）。
