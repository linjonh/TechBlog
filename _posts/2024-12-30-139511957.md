    <p>
     Spring AI 支持知普人工智能的各种人工智能语言模型。您可以与知普人工智能语言模型互动，并基于知普人工智能模型创建多语言对话助手。
    </p>
    <h2>
     先决条件
    </h2>
    <p>
     您需要与 ZhiPuAI 创建一个 API，以访问 ZhiPu AI 语言模型。
    </p>
    <p>
     在
     <a class="link-info" href="https://open.bigmodel.cn/login" rel="nofollow" title="ZhiPuAI 注册页面">
      ZhiPuAI 注册页面
     </a>
     创建账户，并在
     <a class="link-info" href="https://open.bigmodel.cn/usercenter/apikeys" rel="nofollow" title="API Keys 页面">
      API Keys 页面
     </a>
     生成令牌。Spring AI 项目定义了一个名为 spring.ai.zhipuai.api-key 的配置属性，你应将其设置为从
     <a class="link-info" href="https://open.bigmodel.cn/usercenter/apikeys" rel="nofollow" title="API Keys 页面">
      API Keys 页面
     </a>
     获取的 API Key 的值。导出环境变量是设置该配置属性的一种方法：
    </p>
    <pre><code class="hljs">export SPRING_AI_ZHIPU_AI_API_KEY=&lt;INSERT KEY HERE&gt;</code></pre>
    <h3>
     添加资源库和 BOM
    </h3>
    <p>
     Spring AI 工件发布在 Spring Milestone 和 Snapshot 资源库中。请参阅 "版本库 "部分，将这些版本库添加到您的构建系统中。
    </p>
    <p>
     为了帮助进行依赖性管理，Spring AI 提供了一个 BOM（物料清单），以确保在整个项目中使用一致的 Spring AI 版本。请参阅 "依赖关系管理 "部分，将 Spring AI BOM 添加到构建系统中。
    </p>
    <h2>
     自动配置
    </h2>
    <p>
     Spring AI 为 ZhiPuAI 聊天客户端提供了 Spring Boot 自动配置功能。要启用它，请在项目的 Maven pom.xml 文件中添加以下依赖项：
    </p>
    <pre><code class="language-XML">&lt;dependency&gt;
    &lt;groupId&gt;org.springframework.ai&lt;/groupId&gt;
    &lt;artifactId&gt;spring-ai-zhipuai-spring-boot-starter&lt;/artifactId&gt;
&lt;/dependency&gt;</code></pre>
    <p>
     或 Gradle build.gradle 构建文件。
    </p>
    <pre><code class="language-Groovy">dependencies {
    implementation 'org.springframework.ai:spring-ai-zhipuai-spring-boot-starter'
}</code></pre>
    <blockquote>
     <p>
      请参阅 "依赖关系管理 "部分，将 Spring AI BOM 添加到构建文件中。
     </p>
    </blockquote>

    <h2>
     运行时选项
    </h2>
    <p>
     <a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-zhipuai/src/main/java/org/springframework/ai/zhipuai/ZhiPuAiChatOptions.java" title="ZhiPuAiChatOptions.java">
      ZhiPuAiChatOptions.java
     </a>
     提供了模型配置，如使用的模型、温度、频率惩罚等。
    </p>
    <p>
     启动时，可使用 ZhiPuAiChatModel(api, options) 构造函数或 spring.ai.zhipuai.chat.options.* 属性配置默认选项。
    </p>
    <p>
     在运行时，您可以通过在提示调用中添加新的、针对特定请求的选项来覆盖默认选项。例如，覆盖特定请求的默认模型和温度：
    </p>
    <pre><code class="language-java">ChatResponse response = chatModel.call(
    new Prompt(
        "Generate the names of 5 famous pirates.",
        ZhiPuAiChatOptions.builder()
            .withModel(ZhiPuAiApi.ChatModel.GLM_3_Turbo.getValue())
            .withTemperature(0.5f)
        .build()
    ));</code></pre>
    <blockquote>
     <p>
      除了特定于模型的
      <a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-zhipuai/src/main/java/org/springframework/ai/zhipuai/ZhiPuAiChatOptions.java" title="ZhiPuAiChatOptions">
       ZhiPuAiChatOptions
      </a>
      之外，您还可以使用通过
      <a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-core/src/main/java/org/springframework/ai/chat/ChatOptionsBuilder.java" title="ChatOptionsBuilder#builder()">
       ChatOptionsBuilder#builder()
      </a>
      创建的便携式
      <a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-core/src/main/java/org/springframework/ai/chat/ChatOptions.java" title="ChatOptions">
       ChatOptions
      </a>
      实例。
     </p>
    </blockquote>
    <h2>
     示例Controller
    </h2>
    <p>
     <a class="link-info" href="https://start.spring.io/" rel="nofollow" title="创建">
      创建
     </a>
     一个新的 Spring Boot 项目，并将 spring-ai-zhipuai-spring-boot-starter 添加到 pom（或 gradle）依赖项中。
    </p>
    <p>
     在 src/main/resources 目录下添加 application.properties 文件，以启用和配置 ZhiPuAi 聊天模型：
    </p>
    <pre><code class="language-XML">spring.ai.zhipuai.api-key=YOUR_API_KEY
spring.ai.zhipuai.chat.options.model=glm-3-turbo
spring.ai.zhipuai.chat.options.temperature=0.7</code></pre>
    <blockquote>
     <p>
      将 api-key 替换为您的 ZhiPuAI 凭据。
     </p>
    </blockquote>
    <p>
     这将创建一个 ZhiPuAiChatModel 实现，您可以将其注入到您的类中。下面是一个使用聊天模型生成文本的简单 @Controller 类的示例。
    </p>
    <pre><code class="language-java">@RestController
public class ChatController {

    private final ZhiPuAiChatModel chatModel;

    @Autowired
    public ChatController(ZhiPuAiChatModel chatModel) {
        this.chatModel = chatModel;
    }

    @GetMapping("/ai/generate")
    public Map generate(@RequestParam(value = "message", defaultValue = "Tell me a joke") String message) {
        return Map.of("generation", chatModel.call(message));
    }

    @GetMapping("/ai/generateStream")
    public Flux&lt;ChatResponse&gt; generateStream(@RequestParam(value = "message", defaultValue = "Tell me a joke") String message) {
        var prompt = new Prompt(new UserMessage(message));
        return chatModel.stream(prompt);
    }

}</code></pre>
<h2>
手动配置
</h2>
<p>
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-zhipuai/src/main/java/org/springframework/ai/zhipuai/ZhiPuAiChatModel.java" title="ZhiPuAiChatModel">
ZhiPuAiChatModel
</a>
实现了 ChatModel 和 StreamingChatModel，并使用
<a class="link-info" href="https://docs.spring.io/spring-ai/reference/api/chat/zhipuai-chat.html#low-level-api" rel="nofollow" title="底层 ZhiPuAiApi 客户端">
底层 ZhiPuAiApi 客户端
</a>
连接到 ZhiPuAI 服务。
</p>
<p>
在项目的 Maven pom.xml 文件中添加 Spring-ai-zhipuai 依赖关系：
</p>
<pre><code class="language-XML">&lt;dependency&gt;
&lt;groupId&gt;org.springframework.ai&lt;/groupId&gt;
&lt;artifactId&gt;spring-ai-zhipuai&lt;/artifactId&gt;
&lt;/dependency&gt;</code></pre>
<p>
或 Gradle build.gradle 构建文件。
</p>
<pre><code class="language-Groovy">dependencies {
implementation 'org.springframework.ai:spring-ai-zhipuai'
}

</code></pre>
<blockquote>
<p>
请参阅 "依赖关系管理 "部分，将 Spring AI BOM 添加到构建文件中。
</p>
</blockquote>
<p>
接下来，创建 ZhiPuAiChatModel 并将其用于文本生成：
</p>
<pre><code class="language-java">var zhiPuAiApi = new ZhiPuAiApi(System.getenv("ZHIPU_AI_API_KEY"));

var chatModel = new ZhiPuAiChatModel(zhiPuAiApi, ZhiPuAiChatOptions.builder()
.withModel(ZhiPuAiApi.ChatModel.GLM_3_Turbo.getValue())
.withTemperature(0.4f)
.withMaxTokens(200)
.build());

ChatResponse response = chatModel.call(
new Prompt("Generate the names of 5 famous pirates."));

// Or with streaming responses
Flux&lt;ChatResponse&gt; streamResponse = chatModel.stream(
new Prompt("Generate the names of 5 famous pirates."));</code></pre>
<p>
ZhiPuAiChatOptions 为聊天请求提供配置信息。ZhiPuAiChatOptions.Builder 是一个流畅的选项生成器。
</p>
<h3 id="low-level-api">
Low-level ZhiPuAiApi 客户端
</h3>
<p>
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/models/spring-ai-zhipuai/src/main/java/org/springframework/ai/zhipuai/api/ZhiPuAiApi.java" title="ZhiPuAiApi">
ZhiPuAiApi
</a>
是
<a class="link-info" href="https://open.bigmodel.cn/dev/api" rel="nofollow" title="ZhiPu AI API">
ZhiPu AI API
</a>
的轻量级 Java 客户端。
</p>
<p>
以下是如何以编程方式使用该 API 的简单片段：
</p>
<pre><code class="language-java">ZhiPuAiApi zhiPuAiApi =
new ZhiPuAiApi(System.getenv("ZHIPU_AI_API_KEY"));

ChatCompletionMessage chatCompletionMessage =
new ChatCompletionMessage("Hello world", Role.USER);

// Sync request
ResponseEntity&lt;ChatCompletion&gt; response = zhiPuAiApi.chatCompletionEntity(
new ChatCompletionRequest(List.of(chatCompletionMessage), ZhiPuAiApi.ChatModel.GLM_3_Turbo.getValue(), 0.7f, false));

// Streaming request
Flux&lt;ChatCompletionChunk&gt; streamResponse = zhiPuAiApi.chatCompletionStream(
new ChatCompletionRequest(List.of(chatCompletionMessage), ZhiPuAiApi.ChatModel.GLM_3_Turbo.getValue(), 0.7f, true));</code></pre>

<h2>
功能调用
</h2>
<p>
您可以向 ZhiPuAiChatModel 注册自定义 Java 函数，并让 ZhiPuAI 模型智能地选择输出包含参数的 JSON 对象，以调用一个或多个已注册函数。这样，您就可以将 LLM 功能与外部工具和 API 相连接。ZhiPuAI 模型经过训练，能够检测函数何时应被调用，并以符合函数签名的 JSON 格式作出响应。
</p>
<p>
ZhiPuAI API 不会直接调用函数；相反，模型会生成 JSON，您可以用它在代码中调用函数，并将结果返回给模型以完成对话。
</p>
<p>
Spring AI 提供了灵活、用户友好的注册和调用自定义函数的方法。一般来说，自定义函数需要提供函数名称、描述和函数调用签名（作为 JSON 模式），以便让模型知道函数期望使用哪些参数。描述有助于模型了解何时调用该函数。
</p>
<p>
作为开发人员，您需要实现一个函数，接收人工智能模型发送的函数调用参数，并将结果反馈给模型。您的函数可以反过来调用其他第三方服务来提供结果。
</p>
<p>
Spring AI 可以轻松实现这一点，只需定义一个 @Bean 定义，返回一个 java.util.Function 并在调用 ChatModel 时将 bean 名称作为一个选项即可。
</p>
<p>
在引擎盖下，Spring 会用适当的适配器代码封装您的 POJO（函数），从而实现与 AI 模型的交互，使您无需编写繁琐的模板代码。底层基础架构的基础是
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-core/src/main/java/org/springframework/ai/model/function/FunctionCallback.java" title="FunctionCallback">
FunctionCallback
</a>
.java 接口和配套的
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-core/src/main/java/org/springframework/ai/model/function/FunctionCallbackWrapper.java" title="FunctionCallbackWrapper">
FunctionCallbackWrapper
</a>
.java 实用程序类，用于简化 Java 回调函数的实现和注册。
</p>
<h3>
工作原理
</h3>
<p>
假设我们想让人工智能模型回应它所不具备的信息，例如某个地点的当前温度。
</p>
<p>
我们可以向人工智能模型提供有关我们自身功能的元数据，让它在处理您的提示时可以用来检索这些信息。
</p>
<p>
例如，如果在处理提示时，人工智能模型确定它需要有关给定地点温度的额外信息，它就会启动服务器端生成的请求/响应交互。人工智能模型会调用客户端函数。人工智能模型以 JSON 格式提供方法调用详情，客户端负责执行该函数并返回响应。
</p>
<p>
模型与客户端的交互如 [
<a class="link-info" href="https://docs.spring.io/spring-ai/reference/api/chat/functions/zhipuai-chat-functions.html#spring-ai-function-calling-flow" rel="nofollow" title="spring-ai-function-calling-flow">
spring-ai-function-calling-flow
</a>
] 图所示。
</p>
<p>
Spring AI 极大地简化了为支持函数调用而编写的代码。它为你提供了函数调用对话的中介。您只需以 @Bean 的形式提供函数定义，然后在提示选项中提供函数的 Bean 名称即可。您还可以在提示中引用多个函数 Bean 名称。
</p>
<h3>
快速入门
</h3>
<p>
让我们创建一个聊天机器人，通过调用我们自己的函数来回答问题。为了支持聊天机器人的回答，我们将注册自己的函数，该函数接收一个地点并返回该地点的当前天气。
</p>
<p>
当需要回答诸如 "波士顿的天气如何？"这样的问题时，人工智能模型将调用客户端，提供位置值作为参数传递给函数。这种类似 RPC 的数据以 JSON 格式传递。
</p>
<p>
我们的函数会调用一些基于 SaaS 的天气服务 API，并将天气响应返回给模型，以完成对话。在本示例中，我们将使用一个名为 MockWeatherService 的简单实现，它对不同地点的温度进行了硬编码。
</p>
<p>
下面的 MockWeatherService.java 表示天气服务 API：
</p>
<pre><code class="language-java">public class MockWeatherService implements Function&lt;Request, Response&gt; {

    public enum Unit { C, F }
    public record Request(String location, Unit unit) {}
    public record Response(double temp, Unit unit) {}

    public Response apply(Request request) {
    	return new Response(30.0, Unit.C);
    }

}</code></pre>
<h4>
将函数注册为 Bean
</h4>
<p>
有了
<a class="link-info" href="https://docs.spring.io/spring-ai/reference/api/chat/zhipuai-chat.html#_auto_configuration" rel="nofollow" title="ZhiPuAiChatModel Auto-Configuration，">
ZhiPuAiChatModel Auto-Configuration，
</a>
你就有多种方法在 Spring 上下文中将自定义函数注册为 Bean。
</p>
<p>
我们首先介绍对 POJO 最友好的选项。
</p>
<p>
普通 Java 函数
<br/>
在这种方法中，您可以在应用程序上下文中定义 @Beans，就像定义任何其他 Spring 托管对象一样。
</p>
<p>
在内部，Spring AI ChatModel 将创建一个 FunctionCallbackWrapper 封装器实例，该封装器将添加通过 AI 模型调用该函数的逻辑。@Bean 的名称将作为 ChatOption 传递。
</p>
<pre><code class="language-java">@Configuration
static class Config {

    @Bean
    @Description("Get the weather in location") // function description
    public Function&lt;MockWeatherService.Request, MockWeatherService.Response&gt; weatherFunction1() {
    	return new MockWeatherService();
    }
    ...

}</code></pre>
<p>
@Description 注解是可选的，它提供了一个函数描述 (2)，帮助模型理解何时调用函数。这是一个需要设置的重要属性，可帮助人工智能模型确定要调用的客户端函数。
</p>
<p>
另一种提供函数说明的方法是在 MockWeatherService.Request 上使用 @JacksonDescription 注解来提供函数说明：
</p>
<pre><code class="language-java">@Configuration
static class Config {

    @Bean
    public Function&lt;Request, Response&gt; currentWeather3() { // (1) bean name as function name.
    	return new MockWeatherService();
    }
    ...

}

@JsonClassDescription("Get the weather in location") // (2) function description
public record Request(String location, Unit unit) {}</code></pre>
<p>
最佳做法是在请求对象中注释信息，使该函数的生成 JSON 模式尽可能具有描述性，以帮助人工智能模型选择要调用的正确函数。
</p>
<p>
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-spring-boot-autoconfigure/src/test/java/org/springframework/ai/autoconfigure/zhipuai/tool/FunctionCallbackWithPlainFunctionBeanIT.java" title="FunctionCallbackWithPlainFunctionBeanIT">
FunctionCallbackWithPlainFunctionBeanIT
</a>
.java 演示了这种方法。
</p>
<p>
<strong>
函数回调封装器
</strong>
</p>
<p>
另一种注册函数的方法是创建 FunctionCallbackWrapper 封装器，如下所示：
</p>
<p>
</p>
<pre><code class="language-java">@Configuration
static class Config {

    @Bean
    public FunctionCallback weatherFunctionInfo() {

    	return new FunctionCallbackWrapper&lt;&gt;("CurrentWeather", // (1) function name
    			"Get the weather in location", // (2) function description
    			(response) -&gt; "" + response.temp() + response.unit(), // (3) Response Converter
    			new MockWeatherService()); // function code
    }
    ...

}
</code></pre>
<p>
它封装了第三方的 MockWeatherService 函数，并将其注册为 ZhiPuAiChatModel 的 CurrentWeather 函数。它还提供了一个说明 (2) 和一个可选的响应转换器 (3)，以便将响应转换成模型所期望的文本。
</p>
<blockquote>
<p>
默认情况下，响应转换器会对响应对象进行 JSON 序列化。
</p>
</blockquote>
<blockquote>
<p>
FunctionCallbackWrapper 内部根据 MockWeatherService.Request 类解析函数调用签名。
</p>
</blockquote>
<h4>
在聊天选项中指定函数
</h4>
<p>
要让模型知道并调用您的 CurrentWeather 功能，您需要在提示请求中启用该功能：
</p>
<pre><code class="language-java">ZhiPuAiChatModel chatModel = ...

UserMessage userMessage = new UserMessage("What's the weather like in San Francisco, Tokyo, and Paris?");

ChatResponse response = chatModel.call(new Prompt(List.of(userMessage),
ZhiPuAiChatOptions.builder().withFunction("CurrentWeather").build())); // (1) Enable the function

logger.info("Response: {}", response);</code></pre>
<p>
上述用户问题将触发对 CurrentWeather 函数的 3 次调用（每个城市一次），最终响应将类似于下面这样：
</p>
<blockquote>
<p>
以下是所请求城市的当前天气情况：
<br/> - 加利福尼亚州旧金山： 30.0°C
<br/> - 日本东京: 10.0°C
<br/> - 法国巴黎: 15.0°C
</p>
</blockquote>
<p>
<a class="link-info" href="https://github.com/spring-projects/spring-ai/blob/main/spring-ai-spring-boot-autoconfigure/src/test/java/org/springframework/ai/autoconfigure/zhipuai/tool/FunctionCallbackWrapperIT.java" title="FunctionCallbackWrapperIT">
FunctionCallbackWrapperIT
</a>
.java 测试演示了这种方法。
</p>
<h4>
使用 Prompt 选项注册/调用函数
</h4>
<p>
除了自动配置外，您还可以通过 Prompt 请求动态注册回调函数：
</p>
<p>
</p>
<pre><code class="language-java">ZhiPuAiChatModel chatModel = ...

UserMessage userMessage = new UserMessage("What's the weather like in San Francisco, Tokyo, and Paris?");

var promptOptions = ZhiPuAiChatOptions.builder()
.withFunctionCallbacks(List.of(new FunctionCallbackWrapper&lt;&gt;(
"CurrentWeather", // name
"Get the weather in location", // function description
new MockWeatherService()))) // function code
.build();

ChatResponse response = chatModel.call(new Prompt(List.of(userMessage), promptOptions));</code></pre>
<p>
在该请求的持续时间内，提示中注册的功能默认为启用。
</p>
<p>
这种方法可以根据用户的输入动态地选择要调用的不同函数。
</p>
