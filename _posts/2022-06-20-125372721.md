    <h3>
     <a id="_3">
     </a>
     🚀前言
    </h3>
    <p>
     上一文
     <a href="https://blog.csdn.net/weixin_53614367/article/details/125235321?spm=1001.2014.3001.5501">
      数据结构与算法——线性表（顺序表篇）
     </a>
     中详细介绍了线性表的顺序存储结构的特点是逻辑关系上相邻的两个元素在物理位置上也相邻，因此可以随机存取表中的任一元素，它的存储位置可用一个简单直观的公式来表示。
    </p>
    <p>
     但是，从另外一方面来看，线性表顺序存储的特点也构成了这种存储结构的特点：
     <strong>
      在做插入和删除操作时，需要移动大量元素。
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d85b1ec56a132a32104441c84e614633.png#pic_center">
      <br/>
      <br/>
     </img>
    </p>
    <p>
     本篇文章，我将详细介绍线性表的另外一种表示方式——
     <code>
      链式存储结构
     </code>
     。由于链式存储结构不要求逻辑上相邻的元素在物理位置上也相邻，因此它没有顺序存储结构所具有的弱点，但同时也失去了顺序表可随机存取的优势
    </p>
    <hr/>
    <h3>
     <a id="_15">
     </a>
     🚀线性链表（单链表）
    </h3>
    <br/>
    <h4>
     <a id="_18">
     </a>
     🚢概念
    </h4>
    <br/>
    <p>
     线性表的
     <strong>
      链式存储结构
     </strong>
     的特点是
     <strong>
      用一组任意的存储单元存储线性表的数据元素（这组存储单元可以是连续的，也可以是不连续的）
     </strong>
     ，因此，为了表示某个数据元素

     </span>
     之间的逻辑关系，对数据元素
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">

         i

     来说，除存储其本身的信息之外，还需要存储一个指示它直接后继的信息（即直接后继的存储位置），这两部分信息组成数据元素
     <span class="katex--inline">

         </span>
        </span>
       </span>
      </span>
     </span>
     的存储映像，
     <strong>
      称为结点（node）。
     </strong>
    </p>
    <p>
     <strong>
      它包括两个域
     </strong>
     ：其中存储数据元素信息的域称为
     <code>
      数据域
     </code>
     ，存储直接后继存储位置的域称为
     <code>
      指针域
     </code>
     ，指针域中存储的信息称为指针或链，
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        n 
        
       
      
        n
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.43056em; vertical-align: 0em;">
         </span>
         <span class="mord mathdefault">
          n
         </span>
        </span>
       </span>
      </span>
     </span>
     个结点（
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        a 
        
       
         i 
        
       
      
        ai
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.65952em; vertical-align: 0em;">
         </span>
         <span class="mord mathdefault">
          a
         </span>
         <span class="mord mathdefault">
          i
         </span>
        </span>
       </span>
      </span>
     </span>
     (1&lt;=i&lt;=n)的存储映像）链结成一个链表，
     <strong>
      即为线性表的链式存储结构
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7c2aa0c5557e77bf6c195d4652149130.png#pic_center">
      <br/>
      <br/>
     </img>
    </p>
    <p>
     <strong>
      由于链表的每个结点中只包含一个指针域，所以又称这种链表为线性链表或单链表
     </strong>
     ，整个链表的存取必须从头指针开始进行，头指针指示链表中的第一个结点（即第一个数据元素的存储映像）的存储位置，同时，由于最后一个数据元素没有直接后继，则线性链表中最后一个结点的指针为空
     <code>
      （NULL）
     </code>
    </p>
    <p>
     <strong>
      单链表的定义：
     </strong>
    </p>
    <pre><code class="prism language-c"><span class="token comment">//方式一</span>

    <p>
     用线性链表表示线性表时，数据元素之间的逻辑关系是由结点中的指针指示的，换句话说，指针为数据元素之间的逻辑关系的映像，则逻辑上相邻的两个数据元素其存储的物理位置不要求紧邻，
     <strong>
      由此可得出，这种存储结构为非顺序映像或链式映像。
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/83a8db4c795b46c4fac85fba4ee22edc.png#pic_center">
      <br/>
      <br/>
     </img>
    </p>
    <p>
     通常我们把链表画成用箭头相连接的结点的序列，结点之间的箭头表示链域中的指针，它所表是的线性表中数据元素之间的逻辑顺序，而不是每个数据元素在存储器中的实际位置
    </p>
    <p>
     一般在一个单链表中，
     <code>
      L
     </code>
     表示为单链表的头指针，它指向表中的第一个结点，若
     <code>
      L为‘空’（L=NULL）
     </code>
     ，则所表示的线性表为‘空表’，其长度n为‘零’。
    </p>
    <p>
     <strong>
      带头结点的单链表初始化：
     </strong>
    </p>
    <pre><code class="prism language-c"><span class="token keyword">typedef</span> <span class="token keyword">struct</span> <span class="token class-name">LNode</span><span class="token punctuation">{<!-- --></span>           <span class="token comment">//定义单链表节点类型</span>

    <p>
     在单链表的第一个结点之前附设一个结点，称之为
     <strong>
      头结点
     </strong>
     。头结点的数据域可以不存储任何信息，也可以存储如线性表的长度等类的附加信息，头结点的指针域存储指向第一个结点的指针（即第一个元素结点的存储位置），单链表的头指针指向头结点。若线性表为空表，则头结点的指针域为‘空’。
     <strong>
      头结点的作用在于便于边缘化处理（好写程序），头结点也可以不写。
     </strong>
     <br/>
     <br/>
    </p>

      在双向链表的结点中有两个指针域，其一指向直接后继，其二指向直接前驱。和单链表的循环链表类似，双向链表也有循环链表。
     </strong>
    </p>
    <p>
     <code>
