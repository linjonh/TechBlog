    <p>
     <strong>
      背景描述：
     </strong>
     <br/>
     服务部署在公司内部局域网中，每次升级都需要到公司机房中进行，太麻烦，想通过跳板机连接内网服务器，这样在任何地方都可以升级和维护服务了。
    </p>
    <p>
     <strong>
      实现方案：
     </strong>
    </p>
    <h2>
     <a id="_6">
     </a>
     一、部署跳板机
    </h2>
    <p>
     跳板机连接内部网络，同时具备外网访问权限，假设该跳板机外网ip为 211.211.211.211，两台内网服务器ip分别为 192.168.1.100 和 192.168.1.200
    </p>
    <h2>
     <a id="Xshell_10">
     </a>
     二、建立Xshell连接
    </h2>
    <h3>
     <a id="1_12">
     </a>
     1、跳板机设置
    </h3>
    <p>
     1.1 创建跳板机连接
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/511926485649c24e1fc451d2b01595df.png"/>
    </p>
    <p>
     1.2 设置跳板机连接用户名和密码
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/78f47419a60b412dbb1e5e4f21f3e447.png"/>
    </p>
    <p>
     1.3 在 连接 --&gt; SSH 隧道 页面添加TCP/IP转移规则
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e61886ade95af5449d51d49321c0cbcd.png">
      <br/>
      <strong>
       源主机
      </strong>
      ：后面将要创建连接的主机地址将使用该地址，这里使用localhost，
      <br/>
      <strong>
       侦听端口
      </strong>
      ：使用任意一个未被本机占用的端口即可，后面将要创建连接的将使用该端口
      <br/>
      <strong>
       目标主机
      </strong>
      ：填内网服务器的ip
      <br/>
      <strong>
       目标端口号
      </strong>
      ：内网服务器的连接端口
      <br/>
      添加 192.168.1.100 后，按照相同方法添加192.168.1.200，然后连接跳板机
     </img>
    </p>
    <h3>
     <a id="2__28">
     </a>
     2 连接内网服务器
    </h3>
    <p>
     2.1 创建内网服务器连接
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e6728ab2c63961d62f63dbb014ce3c41.png">
      <br/>
      这里的主机和端口号与 1.3 步骤设置的信息保持一致即可
     </img>
    </p>
    <p>
     2.2 访问内网服务器
     <br/>
     和在用一个网络中访问服务器方式一样，直接连接即可，注意跳板机要一直保持连接状态，跳板机如果断开连接，本机与内网服务器的连接也将断开
    </p>
