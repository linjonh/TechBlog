    <h4>
     <a id="_8">
     </a>
     实时音视频是什么?
    </h4>
    <p>
     <strong>
      RTC（Real Time Communication）实时通信业务
     </strong>
     ，
     <strong>
      目的
     </strong>
     是在设备端实时的转发音视频多媒体数据，让
     <strong>
      用户能实时的进行音频和视频的会话
     </strong>
    </p>
    <p>
     <strong>
      HaaS RTC
     </strong>
     是
     <strong>
      阿里云 IoT 联合视频云开发的 IoT 设备端上的实时通讯服务
     </strong>
    </p>
    <h4>
     <a id="_14">
     </a>
     音视频有什么功能？
    </h4>
    <h5>
     <a id="_16">
     </a>
     从功能流程上来讲：
    </h5>
    <p>
     包含了
     <strong>
      采集、编码、前后处理、传输、解码、缓冲、渲染
     </strong>
     等诸多环节
    </p>
    <p>
     当各式
     <strong>
      智能硬件、移动应用以及 Web App
     </strong>
     中的许多模块都越来越依赖于
     <strong>
      音视频技术
     </strong>
     ，
     <strong>
      实时通信
     </strong>
     已然成为了
     <strong>
      所有行业
     </strong>
     的一大
     <strong>
      基础设施
     </strong>
     ，不仅仅是在
     <strong>
      直播、游戏这些泛娱乐行业
     </strong>
     ，更渗透到
     <strong>
      在线医疗、教育、金融
     </strong>
     等领域；在不同场景下，推动着人们沟通互动方式的改变
    </p>
    <h5>
     <a id="_22">
     </a>
     丢包补偿技术：
    </h5>
    <p>
     <strong>
      丢包补偿技术可以分为两类：
     </strong>
    </p>
    <ul>
     <li>
      基于发送端补偿和基于接受端补偿
     </li>
     <li>
      基于发送端补偿包括前向差错纠正、交织和重传技术
     </li>
     <li>
      基于接受端补偿包括了多种错误隐蔽算法
      <br/>
      基于发送端补偿可以分为
      <strong>
       两类：
      </strong>
      <strong>
       主动重传和被动通道编码
      </strong>
      <ul>
       <li>
        <strong>
         被动通道编码
        </strong>
        包含传统的前向差错纠正技术（FEC）和基于交织的技术
       </li>
       <li>
        <strong>
         按照和媒体内容的关系
        </strong>
        ，前向差错纠正包括与媒体无关的方法和利用音频属性的媒体相关方法
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_33">
     </a>
     非交互式应用
    </h5>
    <p>
     <strong>
      对于非交互式的语音应用
     </strong>
     ，比如
     <strong>
      多点广播
     </strong>
     ，对延时的要求没有音质高；
     <strong>
      交织
     </strong>
     是强烈推荐的
     <strong>
      丢包补偿技术
     </strong>
     ，对于交织后的语音，还要采用合适的错误隐蔽算法；与媒体无关的前向误差纠正技术也适合这种应用
    </p>
    <h5>
     <a id="_37">
     </a>
     交互式应用
    </h5>
    <p>
     <strong>
      交互式的应用比如 IP 电话、即时通讯应用中的实时语音聊天等
     </strong>
     ，对
     <strong>
      延时
     </strong>
     很敏感，因此，交织和与媒体无关的前向误差纠正技术都不适合这种应用
    </p>
    <ul>
     <li>
      媒体相关的前向误差纠正技术只引入很小的延时和较小的带宽增加，是较好的选择，可以利用低比特率的次要编码器获得丢包补偿效果
     </li>
     <li>
      另外，还可以采用带有衰减的包重复法等效果较好计算简单的错误隐蔽算法进一步提高音质。
     </li>
    </ul>
    <h5>
     <a id="_44">
     </a>
     以直播场景来举例：
    </h5>
    <ul>
     <li>
      例如主播1往视频云上推流，观众拉流观看，即形成了
      <strong>
       单向直播的场景
      </strong>
      ；观众能够看到主播，但是主播看不到观众
     </li>
     <li>
      另外一种像刚才一样，主播1推流出去，观众拉流观看，其中观众1希望能和主播1视频连麦互动，这时观众1也会推一路流出去，主播将观众1的音视频流拉下来观看，这样他们之间就构成了一个
      <strong>
       相互拉流观看的场景，相互都能看见对方
      </strong>
      ；同时他们两个的音视频流会被实时的内容分发网络分发给观众来观看
     </li>
    </ul>
    <p>
     <strong>
      归根结底：实时音视频就是说实时马上能看到
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       在整个直播或点播过程中
      </strong>
      ，最好有
      <strong>
       实时统计数据，包括网络类型，机器信息，实时网络状况，帧率，码率，分辨率
      </strong>
      等。这样可以分析遇到的各种问题，特别是对于
      <strong>
       直播场景，当网络波动，出现卡顿
      </strong>
      时，可以为动态调整 qos 提供依据
     </li>
     <li>
      <strong>
       对于实时音视频直播场景
      </strong>
      ，采用
      <strong>
       qos 策略，动态调整编码参数
      </strong>
      ；包括
      <strong>
       帧率，码率，分辨率，缓冲区
      </strong>
      ；当直播出现卡顿，采用快降慢升的策略，当网络波动比较厉害，这样可以避免编码参数频繁的来回调整，造成恶性循环
     </li>
    </ul>
    <h4>
     <a id="_54">
     </a>
     实时音视频应用场景
    </h4>
    <p>
     <strong>
      随着接触到的客户越来越多，在行业里发展的越来越深
     </strong>
     ，我们发现，
     <strong>
      音视频技术
     </strong>
     的
     <strong>
      应用场景
     </strong>
     远比想象中的多，应用规模也越来越大
    </p>
    <p>
     <strong>
      也让大家意识到：
     </strong>
     音视频技术不仅局限用于会议系统这样的商业领域，也不仅仅是社交群聊，
     <strong>
      还能用在以下场景中
     </strong>
    </p>
    <ul>
     <li>
      音视频通话
     </li>
     <li>
      产品功能
     </li>
     <li>
      1V1，多人音视频通话
     </li>
     <li>
      可以美颜、使用道具等等
     </li>
    </ul>
    <h5>
     <a id="_65">
     </a>
     实时音视频技术特点
    </h5>
    <ul>
     <li>
      支持设备差异性大
     </li>
     <li>
      网路接入经常切换
     </li>
    </ul>
    <p>
     <strong>

     </strong>
    </p>
