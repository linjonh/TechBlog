     28、 什么是用况 （Use Case）？什么是用况图？一个用况图通常包含哪些模型元
    </h3>
    <p>
     素？
     <br/>
     答：用况（ Use Case）：从外延上说它表达了参与者使用系统的一种方式，从内
     <br/>
     涵上说它规约了系统可以执行的一个动作序列， 并对特定的参与者产生可见的、
     <br/>
     有值的结果；
     <br/>
     用况图：是一种表达系统功能模型的图形化工具；
     <br/>
     一个用况图通常包含的模型元素是： 主题、用况、参与者、 关联、泛化、依赖。
    </p>

     <br/>

     30、简述螺旋模型以及可适应的情况。
    </h3>
    <p>
     答：螺旋模型以及可适应的情况分为以下几点：
    </p>
    <ol>
     <li>
      螺旋模型是在瀑布模型和演化模型的基础上， 加入两者所忽略的风险分析所
      <br/>
      建立的一种软件开发模型；
     </li>
     <li>
      螺旋模型沿着螺旋线，经历制定计划，风险分析，实施工程，客户评估等 4
      <br/>
      个方面的活动，自内向外每旋转一圈便产生一个更为完整的新版本；
     </li>
     <li>
      该模型适应的情况：项目的开发风险很大或客户不能确定系统需求。
     </li>
    </ol>

     34、简述增量模型的优缺点。
    </h3>
    <p>
     答：优点有以下 3 点：
    </p>
    <ol>
     <li>
      第一个可交付版本所需要的成本和时间是较少的， 从而可减少开发由增量表
      <br/>
      示的小系统承担的风险；
     </li>
     <li>
      由于很快分布的第一个版本，因此可以减少用户需求的变更；
     </li>
     <li>
      允许增量投资，即在项目开始时可以仅对一个或两个增量投资；
      <br/>
      缺点有以下 3 点：
     </li>
     <li>
      如果没有对用户的变更要求进行规划， 那么产生的初始增量可能会造成够来
      <br/>
      增量的不稳定；
     </li>
     <li>
      如果需求不像早期思考的那样稳定和完整， 那么一些增量就可能需要重新开
      <br/>
      发，重新发布；
     </li>
     <li>
      由于进度和配置的复杂性，可能会增大管理成本，超出组织的能力。

     <a id="42_P28_223">
     </a>
     42、简述需求规约的定义，并写出需求规约满足的基本性质。 (P28)
    </h3>
    <p>
     答：需求规约是一个软件项 / 产品/ 系统所有需求陈述的正式文档，它表达了一
     <br/>
     个软件产品 / 系统的概念模型。
     <br/>
     需求规约一般需要满足一下 4 个基本性质：
    </p>
    <ol>
     <li>
      重要性和稳定性程度：按需求的重要性和稳定性，对需求进行分级；
     </li>
     <li>
      可修改的：在不过多地影响其他需求的前提下，可以容易地修改一个单一需
      <br/>
      求；
     </li>
     <li>
      完整的：没有被遗漏的需求；
     </li>
     <li>
      一致的：不存在互斥的需求。
     </li>
    </ol>

     <br/>
     （5）修改原型：在评价基础上进行修改。若不满足需求说明，则根据明确的需求修改原型。若不满足用户需求，则先修改

     135、简述增量模型的优缺点
    </h3>
    <p>
     答：优点 1) 第一个可交付版本所需要的成本和时间是较少的，从而可减少开发由增量表示的小系统承担的风险
     <br/>
     2) 由于很快发布第一个版本，因此可以减少用户需求的变更
     <br/>
     3) 允许增量投资，即在项目开始时可以仅对一个或两个增量投资
     <br/>
     缺点： 1) 如果没有对用户的变更要求进行规划，那么产生的初始增量可能会造成后来增量的不稳定
     <br/>
     2) 如果需求不像早期思考的那样稳定和完整，那么一些增量就可能需要重新开发，重新发布
     <br/>
     3) 由于进度和配置的复杂性，可能会增大管理成本，超出组织的能力
    </p>

     <a id="136_1154">
     </a>
     136、简述螺旋模型以及它与其它模型之间的主要区别
    </h3>
    <p>
     答：螺旋模型是瀑布模型与深化模型的基础上，加入两者所忽略的风险分析所建立
     <br/>
     的一种软件开发模型螺旋模型关注解决问题的基本步骤，即标识问题，标识一些可选方案，选择一个最
     <br/>
     佳方案，遵循动作步骤并实施后续工作，突出特征，在开发的迭代中实际上只有一
     <br/>
     个迭代过程真正开发了可交付的软件
     <br/>
     2) 与深化模型和增量模型相比， 同样使用了瀑布模型作为一个嵌入的过程， 即分析、设计、编码、实现和维护的过程，并且在框架和全局体系结构方面是等同的。但是，螺旋模型所关注的阶段以及它们的活动是不同的， 如增加一些管理活动和支持活动。尽管增量模型也有一些管理活动，但它基于以下假定：需求是最基本的、并且是唯
     <br/>
     一的风险源，因而在螺旋模型中增大了决策和风险的空间，螺旋模型扩大了增量模型的管理范围。
     <br/>
     如果项目的开发风险很大或客户不能确定系统需求，在更广泛的意义上来讲，还包括一个系统或系统类型的要求，这时螺旋模型就是一个好的生存周期模型
    </p>
    <h3>
     <a id="137_1166">
     </a>
     137、简述专用实践与共用实践关系
    </h3>
    <p>
     答：专用实践： 每一个过程域中都有一个或多个“专用实践” ，这些专用实践被认为对于达到该过程域的专用目标是重要活动，即期望以专用实践所描述的活动，会导致达到一个过程域的专用目标
     <br/>
     共用实践： 每一个过程域中都有一个或多个“共用实践” ，这些共用实践被认为对于达到该过程域相关的共用目标是重要活动，例如，对共有目标“该过程予以制度化，使之成为一个已管理过程“而言，一个共用实践是”为该过程的执行、工作产品的开发以及该过程的服务，提供充足的资源“。之所以称为“共用实践“，是因为同一实践可应用于多个过程。
    </p>
