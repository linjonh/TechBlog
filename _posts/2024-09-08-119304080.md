    <h2>
     <a id="_0">
     </a>
     模式定义
    </h2>
    <p>
     适配器模式（Adapter Pattern）保持现有类功能不变，只是负责将一类接口转换为另一类开发者希望的接口。它还有另一个别名叫包装器(Wapper)。
    </p>
    <h2>
     <a id="_4">
     </a>
     需求场景
    </h2>
    <p>
     用现实案例来参照。适配器模式在现实生活中最典型的参照对象就是电源适配器。如美国和日本标准电压是110V，而中国的是220V，通过电源适配器就可以将当地电源转换成自己设备需要的电压。
    </p>
    <p>
     这个案例非常符合适配器所要解决问题的特征：
    </p>
    <ol>
     <li>
      我们需要使用现有的模块解决问题。（当地电源）
     </li>
     <li>
      现有模块提供的接口不符合我们开发中的接口标准。（电压不一致）
     </li>
     <li>
      我们设计了一个适配器，将接口标准改为我们需要的标准。（110V to 220V)
     </li>
     <li>
      我们设计的适配器负责做接口转换，如非必要并不改动原有模块的功能。（电源还是那个电源）
     </li>
    </ol>
    <h2>
     <a id="_15">
     </a>
     开发案例
    </h2>
    <p>
     作为移动开发者，在开发中我们遇到的跟适配器有关的一个典型案例就是广告
     <code>
      SDK
     </code>
     适配器。
    </p>
    <p>
     在移动开发领域，开发者为了获取收益，往往会通过接广告平台的
     <code>
      SDK
     </code>
     来获得广告收益。
    </p>
    <p>
     广告平台有很多，比如谷歌的
     <strong>
      Admob
     </strong>
     ，腾讯的
     <strong>
      优量汇
     </strong>
     ，脸书的
     <code>
      Audience Network
     </code>
     等。
    </p>
    <p>
     对于开发者来说，如果想在应用中插入一个插屏广告，那么从开发者角度来讲，接口上来说仅仅是调用一个
     <strong>
      显示差评广告
     </strong>
     的接口即可，而无需关心该接口来自
     <code>
      Admob
     </code>
     还是哪一家广告商。
    </p>
    <p>
     然而，各大广告商提供的
     <code>
      SDK
     </code>
     并不统一，如广告引擎初始化，弹起全屏广告等逻辑，都有各自的设计方案。这时候，我们就需要一个
     <strong>
      适配器
     </strong>
     ，将各大平台的广告
     <code>
      API
     </code>
     适配成我们自己需要的接口。
    </p>
    <p>
     从案例出发，我们设计一个广告展示抽象类
     <code>
      AbstractAdsAdapter
     </code>
     。
    </p>

      map
     </code>
