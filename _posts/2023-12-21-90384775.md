    <h4>
     <a id="_0">
     </a>
     前言
    </h4>
    <p>
     520，今天是一个特殊的日子。生活给我来了个bug礼包，bug礼包的来源是公司的App接入了七牛云实时音视频SDK实现音视频通话功能，后面我们又接入腾讯系的bugly热更新SDK实现网络更新功能，bug礼包来啦！！！
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9b9c1c508904b0963070c37271714d7f.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0cea1c7da3b088a463ef77859b35c5aa.png">
       <br/>
       上面两个bug不是固定出现噢，一会报
       <code>
        20111
       </code>
       错误码，一会报
       <code>
        10054
       </code>
       错误码。说一下大概历程吧！遇到
       <code>
        20111
       </code>
       错误码，我就找后端的同学沟通我们都以为是服务端的原因，后端同学给了我两个固定的视频token就报
       <code>
        10054
       </code>
       错误码，上面两个错误码的解释是我从七牛音视频SDK官方文档截图下来的。我和后端同学都排查了自己的代码，发现都没有问题。于是，我又联系了七牛云实时音视频的客服，他们说是没有导入
       <code>
        jniLibs
       </code>
       里面
       <code>
        armeabi-v7a
       </code>
       的
       <code>
        .so
       </code>
       文件。七牛云实时音视频SDK有个令我很奇怪的地方，居然不需要在
       <code>
        build.gradle
       </code>
       文件中加入下面代码
      </img>
     </img>
    </p>
    <pre><code>android {
    sourceSets {
        main {
            jniLibs.srcDirs = ['libs']
        }
    }
}
</code></pre>
    <p>
     检查之后发现，我是有导入相应的
     <code>
      .so
     </code>
     文件的。
    </p>
    <h4>
     <a id="_17">
     </a>
     解决的过程
    </h4>
    <p>
     我发现在接入腾讯系的bugly热更新SDK的时候有加入下面的代码：
    </p>
    <pre><code> ndk {
       // 设置支持的SO库架构//, 'x86', 'armeabi-v7a', 'x86_64', 'arm64-v8a'
       abiFilters 'armeabi'
}
</code></pre>
    <p>
     由于考虑apk大小和之前的接入经验，我只支持了
     <code>
      armeabi
     </code>
     的so库架构。最后我把上面的代码修改为：
    </p>
    <pre><code>ndk {
       // 设置支持的SO库架构//, 'x86', 'armeabi-v7a', 'x86_64', 'arm64-v8a'
       abiFilters 'armeabi', 'x86', 'armeabi-v7a', 'x86_64', 'arm64-v8a'
}
</code></pre>
    <p>
     支持所有的
     <code>
      .so
     </code>
     库架构之后，我发现七牛云的实时音视频功能又能正常使用了，而且不会报错。
    </p>
    <h5>
     <a id="_36">
     </a>
     总结
    </h5>
    <p>
     这次接入七牛云实时音视频SDK和腾讯系的bugly热更新SDK发生碰撞是我意向不到的，最神奇的事情是同样的代码在手机端是没有问题，但是在平板端是由问题。希望我这次踩坑对接入七牛云实时音视频SDK和腾讯系SDK的Android小伙伴有帮助。为自己打个小广告哈！下面是我的公众号的图片，我会经常更新一下技术干货或者开发踩坑历程，欢迎大家关注我！
