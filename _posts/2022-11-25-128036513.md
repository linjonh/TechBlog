    <h3>
     <a id="_0">
     </a>
     通讯的必要条件
    </h3>
    <ul>
     <li>
      主机之间需要有
      <code>
       传输介质
      </code>
      。光纤、蓝牙、wify
     </li>
     <li>
      主机上必须有
      <code>
       网卡设备
      </code>
      。把二进制信息转为高低电压的过程就是数据的调制过程。把电信号转为二进制信息的过程为解调制。
     </li>
     <li>
      主机之间需要协商
      <code>
       网络速率
      </code>
      。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d97de6ae6616169048db1c96661760af.png"/>
     </li>
    </ul>
    <h3>
     <a id="_5">
     </a>
     网路的通讯方式
    </h3>
    <p>
     日常生活中，我们通讯的方式不可能只有两台主机，如何建立多台主机互连，假如要与十台电脑通讯，那么不可能去买一台有十个网卡的主机，所以这个时候就可以通过
     <code>
      交换机
     </code>
     和
     <code>
      路由器
     </code>
     完成这个事情。
    </p>
    <h3>
     <a id="_7">
     </a>
     如何建立多台主机互连？如何定位局域网中的其它主机？
    </h3>
    <p>
     通过
     <code>
      交换机
     </code>
     我们可以将多台电脑进行连接。
     <br/>
     a主机将消息发送给交换机，交换机将消息广播给其他主机，在这个过程中其他机器都会收到这条消息，然后b主机就通过
     <code>
      MAC地址
     </code>
     发现a想找的是自己，而其他的主机就会将这条消息当作垃圾丢弃。这个过程就是通过交换机来完成局域网通信的操作。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/85aba6408efc46a716d62b374f646a16.png"/>
    </p>
    <h3>
     <a id="_11">
     </a>
     通过交换机来完成局域网通信有哪些问题？
    </h3>
    <p>
     通过交换机来完成局域网通信是无法满足互联网的需求。比如说我们不可能将所有电脑都放在一个局域网内，因为交换机的接口数量是有上限的，而且如果我们把所有机器放在一个局域网里，那么这个时候
     <code>
      广播风暴
     </code>
     就会尤为明显！因为任意一条消息的发送都会被其他主机所接收，然后还要确认是否有效。基于这种情况，
     <strong>
      网路就被分为许多较小的局域网，然后就是城域网，最后是互联网
     </strong>
     。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a4a9bd523fb93d8e7027a78c6c57c116.png"/>
    </p>
    <h3>
     <a id="_14">
     </a>
     如何实现？
    </h3>
    <p>
     我们现在能访问百度，但是百度并不在局域网内，所以我们与百度的主机分属于不同的局域网，那这两个局域网之间该如何建立通讯呢？这里我们就需要用来
     <code>
      路由器
     </code>
     来进行数据的交换了。首先我们的电脑要去访问百度的主机，我们不仅需要知道百度的
     <code>
      网卡地址
     </code>
     ，还需要知道它在哪个网络里，这块就需要用到
     <code>
      ip地址
     </code>
     了。这里要说明的是，我们数据在发送的时候不仅只有程序代码所接收到的数据，
     <code>
      还有原mac地址、原ip地址和目标mac地址、目标ip地址
     </code>
     等等。这些内容呢首先会被组装成一条大的数据，之后再发送到路由器上，路由器是可以识别当前要找的主机它在不在这个局域网内。如果说它不在当前的局域网内，这个时候他会按照路由表上的信息去帮助我们通过网络路由来找到对应的网络，之后再去把数据发送过去。发送到对应的网络之后，由那边的交换机完成定位功能。那这样的话我们就通过了路由器实现了不同网络之间的主机通讯。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d2e1c80258abd8beb954a311d5eada68.png"/>
    </p>
    <h3>
     <a id="_17">
     </a>
     网络层次模型
    </h3>
    <p>
     对于网络通信来说它是一个复杂的工程，它需要有很多的基础硬件设备，而这些设备又有很多的厂商在进行生产，所以为了方便网络的实施与管理维护就会有相应的组织来推出相应的标准规范 。目前最常见的就是
     <code>
      OSI七层模型
     </code>
     和
     <code>
      TCP、IP的四层模型
     </code>
     。TCP、IP是建立在OSI之上的另外四种模型。无论我们采用的哪一种模型，它其实都是对通信的过程进行分层，然后每层当中也存在很多的协议。例如我们常用的http就属于应用层的协议，而tcp和udp就属于传输层的协议。下面就以OSI为例来介绍它的七层分层以及每一次做了哪些事情。它的作用就是更加清晰规范的完成网络通讯。
     <br/>
     这里主要掌握分层的目的，七层的名称和作用，还有就是一些常见的协议名称
    </p>
    <h4>
     <a id="OSI_20">
     </a>
     OSI七层模型
    </h4>
    <ul>
     <li>
      <code>
       应用层
      </code>
      ： 用户与网络的接口
      <ul>
       <li>
        利用
        <code>
         http
        </code>
        协议完成网站服务
       </li>
       <li>
        利用
        <code>
         ftp
        </code>
        协议完成文件的传输服务
       </li>
       <li>
        利用
        <code>
         ssh
        </code>
        协议完成远程登陆服务
       </li>
      </ul>
     </li>
     <li>
      <code>
       表示层
      </code>
      ： 数据加密、转换、压缩
     </li>
     <li>
      <code>
       会话层
      </code>
      ： 控制网络连接建立与终止
     </li>
     <li>
      <code>
       传输层
      </code>
      ：保证数据传输的可靠性
      <ul>
       <li>
        在封装的时候还会携带目标占用的端口号，例如访问网站的时候会携带80端口(
        <code>
         TCP
        </code>
        ,
        <code>
         UDP
        </code>
        )
       </li>
      </ul>
     </li>
     <li>
      <code>
       网络层
      </code>
      ： 通过路由来找到目标网络
      <ul>
       <li>
        常见的就是ip协议，让我们根据ip地址来确定源和目标的网络
       </li>
      </ul>
     </li>
     <li>
      <code>
       数据链路层
      </code>
      ：确定目标主机
      <ul>
       <li>
        这一层就是确定了目标网路并进入了某一个局域网内
       </li>
      </ul>
     </li>
     <li>
      <code>
       物理层
      </code>
      ： 各种网络物理设备和标准
      <br/>
      数据从A至B，先封装再解封
     </li>
    </ul>
    <h4>
     <a id="TCP_IP_36">
     </a>
     TCP IP四层模型
    </h4>
    <ul>
     <li>
      <code>
       应用层
      </code>
      <ul>
       <li>
        就是将OSI的应用层、表示层、会话层合并
       </li>
      </ul>
     </li>
     <li>
      <code>
       传输层
      </code>
      <ul>
       <li>
        就是OSI的传输层
       </li>
      </ul>
     </li>
     <li>
      <code>
       主机层
      </code>
      <ul>
       <li>
        就是OSI的网络层
       </li>
      </ul>
     </li>
     <li>
      <code>
       接入层
      </code>
      <ul>
       <li>
        就是将OSI的数据链路层、物理层合并
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="AB_45">
     </a>
     数据从A到B
    </h4>
    <ul>
     <li>
      按照分层自上向下一层一层封装
     </li>
     <li>
      到了B主机的网卡协调之后，再按照自下向上的顺序进行拆解
     </li>
     <li>
      最后在应用层里拿到A主机所发送过来的数据
     </li>
    </ul>
    <h4>
     <a id="_49">
     </a>
     数据封装与解封的过程
    </h4>
    <ul>
     <li>
      第一步在应用层产出需要使用的数据data
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/057ee9757f133013a64364c15908b242.png"/>
     </li>
     <li>
      然后data数据就传向了传输层。在这一层对于我们来说最熟悉的就是TCP与UDP协议，这两个协议都是基于端口的，而端口的作用就是在主机上用于确定唯一一个应用进程。所以数据在这层就会包裹上目标应用端口和应用在当前主机上的源端口
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6070641ee3652d22bcb881be5b0cbd3c.png"/>
     </li>
     <li>
      数据再传到网络层，因为我们的主机是在不同的网络里的，所以需要通过IP协议来确定目标主机所在的网络。因此数据在这一层中会包裹目标主机的mac地址、目标主机的ip地址和当前主机的源ip地址。有了这些地址之后我们只是能确定某一个网络，并不能确定在这个网络里边那一台机器是我们想要的，所以接下来数据就到达了链路层
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/db75e7a6d5d502a1b86f1c577ed88341.png"/>
     </li>
     <li>
      在这一层主要是通过mac地址来完成
      <code>
       寻址
      </code>
      操作，所以数据在这一层会被包裹目标主机的mac地址与当前主机的mac地址。至此一条具备完整信息的数据就封装ok了。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a19a87bff14bea2dd810d452bd3b5356.png"/>
     </li>
     <li>
      这个时候就要传递给网络了，而我们知道网线是不能识别二进制的，所以经过
      <code>
       物理层（网卡）
      </code>
      的调制之后就会变成
      <code>
       高低电压
      </code>
      ，而我们这里任然是以二进制的数据来表示转换之后的数据，有了高低电压之后经过网络路由器的分配和传输介质的运输，最终就到达了目标主机的网卡。
      <strong>
       mac地址是由硬件厂商确认好的，也就是网卡设备（一般电脑有两个网卡，一个是有线一个是无线）
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c59c2de029e2897af6188d72cd79866f.png"/>
     </li>
     <li>
      <strong>
       这个时候已经到
       <code>
        目标设备了
       </code>
       ,它要去做数据的解调，将电信号变成二进制，然后再向上一层一层的
       <code>
        验证
       </code>
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/471d009b60eb071c78aaadf7516d3311.png"/>
     </li>
     <li>
      到达链路层之后它就去分析一下，目标的mac地址是不是当前主机的mac地址，如果是则继续向上传递至网络层
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b2d01bf6033ae490931f92581e48e8db.png"/>
     </li>
     <li>
      在这层就要去校验目标的ip是不是当前自己的ip，如果是，则继续拆包。向上传输至传输层
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2ab59ed4dc8956e3427fd0d1187a9acc.png"/>
     </li>
     <li>
      这里再次确定当前的端口是不是自己，如果是的话，则再次拆解数据，向上传递至应用层
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/83780fc0db4d558bb3da8ec2302c4c33.png"/>
     </li>
     <li>
      到了应用层之后，当前网络里的目标应用就拿到了另外一个网络当中的某一台主机上的某一个应用所传递过来的数据。 这个过程就是数据在通讯过程中的封装与解封的步骤。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f020d99345b06ad1d6b028c7eb223a82.png"/>
     </li>
    </ul>
