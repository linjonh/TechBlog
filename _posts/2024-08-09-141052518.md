    <h2>
     触手可及的语义搜索
    </h2>
    <p>
     Elasticsearch 8.15 引入了语义文本（semantic text）和语义重新排名（semantic reranking） - 两种将自然语言搜索带入你的搜索体验的强大方法。
    </p>
    <p>
     我们继续通过
     <a href="https://www.elastic.co/search-labs/blog/semantic-search-simplified-semantic-text" rel="nofollow" title="semantic text">
      semantic text
     </a>
     使向量搜索更易于访问 - 这是一种新的字段类型和相应的语义查询类型，可解锁对文本进行向量搜索的简化方法。特别是，语义文本处理自动分块，因此具有长文本字段的用户可以放心，因为他们知道他们拥有完整的多嵌入覆盖和后台处理。向量搜索客户只需
     <a class="link-info" href="https://www.elastic.co/guide/en/elasticsearch/reference/8.15/semantic-search-semantic-text.html" rel="nofollow" title="设置一个 semantic text 字段">
      设置一个 semantic text 字段
     </a>
     即可开始使用
     <a class="link-info" href="https://www.elastic.co/guide/en/elasticsearch/reference/8.15/inference-apis.html" rel="nofollow" title="interfence API">
      interfence API
     </a>
     自动处理的模型协调和嵌入生成。
    </p>
    <p>
     对于想要不使用向量进行语义搜索或进一步强化其向量或混合搜索的用户，8.15 还
     <a class="link-info" href="https://www.elastic.co/search-labs/blog/semantic-reranking-with-retrievers" rel="nofollow" title="引入了语义重新排名">
      引入了语义重新排名
     </a>
     。此功能可以应用强大的语义相似性模型来优化搜索查询的前 N ​​个结果，从而显著改善搜索结果，尤其是自然语言搜索。对于任何搜索系统来说，这一新增功能都可能是一个强大的改进，但对于寻求在不结合向量嵌入的情况下改善自然语言搜索能力的用户来说尤其有价值。
    </p>
    <p>
    </p>
    <h2>
     通过附加第三方提供商实现更大的模型灵活性
    </h2>
    <p>
     Elasticsearch 的
     <a class="link-info" href="https://www.elastic.co/guide/en/elasticsearch/reference/8.15/put-inference-api.html#put-inference-api-desc" rel="nofollow" title="open inteference API">
      open inteference API
     </a>
     继续通过附加第三方提供商为我们的用户提供更大的模型灵活性。8.15 引入了：
    </p>
    <ul>
     <li>
      Google AI Studio（嵌入和完成）和 Vertex AI（嵌入和重新排名）
     </li>
     <li>
      Mistral（嵌入）
     </li>
     <li>
      Amazon Bedrock（嵌入和完成）
     </li>
     <li>
      Anthropic（完成）
     </li>
    </ul>
    <p>
     所有这些模型提供商都可以通过提供简单的 API 密钥轻松配置为推理端点，然后可以与
     <a class="link-info" href="https://www.elastic.co/search-labs/blog/elasticsearch-retrievers" rel="nofollow" title="retrievers">
      retrievers
     </a>
     一起使用，以实现简单且易于组合的搜索查询。
    </p>
    <p>
    </p>
    <h2>
     更多工具来微调相关性
    </h2>
    <p>
     Elasticsearch 的原生 “Learning to Rank - 学习排名” 和查询规则功能自 8.15 起正式可用，让用户能够更精细地控制其搜索相关性和结果。
    </p>
    <ul>
     <li>
      “
      <a href="https://www.elastic.co/search-labs/blog/elasticsearch-learning-to-rank-introduction" rel="nofollow" title="Learning to Rank">
       Learning to Rank
      </a>
      - 学习排名” 直接内置于 Elasticsearch 中，并扩展了搜索用户的搜索相关性调整管道，因此他们可以使用针对他们关心的功能进行训练的 LTR 模型对结果进行重新排名。这直接在 Elasticsearch 中运行，并支持更精细调整和上下文感知的搜索结果，这些结果可以适应复杂的搜索和用户行为模式。从 8.15 开始，你还可以
      <a href="https://www.elastic.co/guide/en/elasticsearch/reference/8.15/collapse-search-results.html#rescore-collapse-results" rel="nofollow" title="rescore collapsed results">
       rescore collapsed results
      </a>
      ，这意味着你可以使用 LTR 对字段折叠结果进行重新排名。
     </li>
     <li>
      <a class="link-info" href="https://www.elastic.co/guide/en/elasticsearch/reference/8.15/search-using-query-rules.html" rel="nofollow" title="Query rules">
       Query rules
      </a>
      查询规则引入了将业务规则应用于搜索集的能力，即使使用机器学习模型进行混合搜索也是如此。用户可以指定在满足定义条件时将所需的文档固定在搜索结果的顶部 - 从而实现根据查询、用户细分等量身定制的搜索结果，并且它可以与混合搜索和 RRF 的任意组合一起使用。详细使用，请参考文章 “
      <a class="link-info" href="https://elasticstack.blog.csdn.net/article/details/141016583" rel="nofollow" title="Elasticsearch 查询规则现已正式发布 - query rules">
       Elasticsearch 查询规则现已正式发布 - query rules
      </a>
      ”。
     </li>
     <li>
      <a class="link-info" href="https://www.elastic.co/search-labs/blog/rag-playground-introduction" rel="nofollow" title="Playground">
       Playground
      </a>
      现在支持 Google Gemini！
     </li>
    </ul>
