    <p>
     引言：针对数组、集合形式，后端有不同的接参模式，前端改如何传参呢？
    </p>
    <p>
    </p>
    <h2>
     1.1 简单类型集合
    </h2>
    <h3>
     1.1.1 无PathVariable.
    </h3>
    <pre><code class="language-java">@ApiOperation(value = "批量删除")
@DeleteMapping(value = "/deleteRouterProcess")
public RestResponse&lt;Boolean&gt; deleteRouterProcess(@RequestParam("ids") Long[] ids) {
    ....
}
或者：
public RestResponse&lt;Boolean&gt; deleteRouterProcess(@RequestParam("ids") List&lt;String&gt; ids) {
    ....
}
或者：
public RestResponse&lt;Boolean&gt; deleteRouterProcess(@RequestParam("ids") Set&lt;String&gt; ids) {
    ....
}</code></pre>
    <p>
     前端传参URL：
     <u>
      <span style="color:#fe2c24;">
       deleteRouterProcess?ids=111&amp;ids=222.
      </span>
     </u>
    </p>
    <p>
    </p>
    <h3>
     1.1.2 使用PathVariable
    </h3>
    <pre><code class="language-java">@ApiOperation(value = "批量删除")
@DeleteMapping(value = "/deleteRouterProcess/{ids}")
public RestResponse&lt;Boolean&gt; deleteRouterProcess(@PathVariable("ids") Long[] ids) {
    ....
}
</code></pre>
    <p>
     PathVariable会从URL解析参数，请求地址应该为：
     <u>
      <span style="color:#fe2c24;">
       deleteRouterProcess/1111,222
      </span>
     </u>
    </p>
    <p>
    </p>
    <h2>
     2.Bean接收但是Bean中存在简单集合类型
    </h2>
    <pre><code class="language-java">@ApiOperation(value = "批量删除")
@DeleteMapping(value = "/deleteRouterProcess")
public RestResponse&lt;Boolean&gt; deleteRouterProcess(ListBean bean) {
    ....
}


public class ListBean{

   private Long[] ids;

   //getter、setter
}</code></pre>
    <p>
     前端传参应该为:
     <u>
      <span style="color:#fe2c24;">
       deleteRouterProcess?ids=111&amp;ids=222.
      </span>
     </u>
    </p>
    <p>
    </p>
    <h2>
     3.Bean接收但是Bean中存在Bean集合类型
    </h2>
    <pre><code class="language-java">@ApiOperation(value = "批量删除")
@DeleteMapping(value = "/deleteRouterProcess")
public RestResponse&lt;Boolean&gt; deleteRouterProcess(ListBean bean) {
    ....
}


public class ListBean{
   private List&lt;SimpleBean&gt; beans;
   //getter、setter
}


public class SimpleBean{
   private Long ids;
   private String name;
   //getter、setter
}</code></pre>
    <p>
     此时前端传参为：
     <u>
      <span style="color:#fe2c24;">
       deleteRouterProcess?beans[0].ids=1&amp;beans[1].ids
      </span>
     </u>
     ，在实际传参中url不能有特殊字符[]会被编码转义。
    </p>
    <p>
    </p>
    <p>
     注：上述DeleteMapping可以修改为RequestMappign，前端请求方式根据这个灵活变动。
    </p>
    <p>
    </p>
