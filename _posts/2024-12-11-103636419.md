       6、添加实体类及其Mapping类

      Student类及StudentMapping类如下：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0f0a9b56c8798b6b5bbbc0c537dfa2a5.png"/>

    <pre><code>class Student
{
    public int Id { get; set; }

    public string Name { get; set; }
}
</code></pre>
    <pre><code>class StudentMapping : EntityTypeConfiguration&lt;Student&gt;
{
    public StudentMapping()
    {
        // 定义表名
        this.ToTable("Student");

        // 定义主键
        this.HasKey(s =&gt; s.Id);

        // 定义各属性映射方式
        this.Property(s =&gt; s.Id).HasColumnName("Id").HasDatabaseGeneratedOption(DatabaseGeneratedOption.Identity).IsRequired();
        this.Property(s =&gt; s.Name).HasColumnName("Name").HasColumnType(SqlDbType.NVarChar.ToString()).HasMaxLength(50).IsRequired();
    }
}
</code></pre>
    <blockquote>
     <p>
      <strong>
       StudentMapping类
      </strong>
      定义了Student实体类到数据库Student表的映射规则

       7、在MyDbContext类中注册实体类及其Mapping类
      </strong>
     </li>

    <pre><code>class MyDbContext : DbContext
{
    public MyDbContext() : base("name=conStr")
    {
        // 设置数据库初始化器
        Database.SetInitializer(new MigrateDatabaseToLatestVersion&lt;MyDbContext, Configuration&gt;());
    }

    // 定义实体类的 DbSet
    public DbSet&lt;Student&gt; Students { get; set; }

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        // 注册Mapping类
        modelBuilder.Configurations.Add(new StudentMapping());
    }
}
</code></pre>

       Students属性
      </strong>

    <h3>
     <a id="23_194">
     </a>
     【2.3】数据库初始化器
    </h3>
    <ul>
     <li>
      在
      <strong>
       【1.1】生成数据库
      </strong>
      中，步骤5为添加
      <strong>
       数据库初始化器
      </strong>
      的方法。
     </li>
     <li>
      内置4种初始化器，如下：
      <br/>
      <strong>
       1.CreateDatabaseIfNotExists
      </strong>
      （EF默认）
      <br/>
      <em>
       特点
      </em>
      ：
      <br/>
      1）数据库不存在，创建数据库；
      <br/>
      2）数据库存在，不创建数据库；
      <br/>
      3）存在的数据库与实体类模型不符，抛出异常；
      <br/>
      <em>
       使用
      </em>
      ：
      <br/>
      <code>
       Database.SetInitializer(new CreateDatabaseIfNotExists&lt;MyDbContext&gt;());
      </code>
      <br/>
      <strong>
       2.DropCreateDatabaseIfModelChanges
      </strong>
      <br/>
      <em>
       特点
      </em>
      ：
      <br/>
      1）当实体类模型改变时，删除原有数据库，重新创建数据库；
      <br/>
      <em>
       使用
      </em>
      ：
      <br/>
      <code>
       Database.SetInitializer(new DropCreateDatabaseIfModelChanges&lt;MyDbContext&gt;());
      </code>
      <br/>
      <strong>
       3.DropCreateDatabaseAlways
      </strong>
      <br/>
      <em>
       特点
      </em>
      ：
      <br/>
      1）每次运行都删除原有数据库，重新创建数据库；
      <br/>
      <em>
       使用
      </em>
      ：
      <br/>
      <code>
       Database.SetInitializer(new DropCreateDatabaseAlways&lt;MyDbContext&gt;());
      </code>
      <br/>
      <strong>
       4.MigrateDatabaseToLatestVersion
      </strong>
      （本文使用的）
     </li>
    </ul>
    <blockquote>
     <p>
      1、2、3中的初始化器，不需要
      <strong>
       【1.1】生成数据库
      </strong>
      步骤4中生成的
      <strong>
       Configuration
      </strong>
      类（也可以使用）；
      <br/>
      所以，也不能使用
      <strong>
       update-database
      </strong>
      命令生成数据库；
      <br/>
      要生成数据库，可以运行程序，执行到
      <strong>
       创建DbContext派生类对象时
      </strong>
      即会生成；
     </p>
    </blockquote>
    <blockquote>
     <p>
      2、3中的初始化器，因为要删除原有数据库，所以可能会抛出“无法删除数据库，因为数据库正在使用的异常”，这多半是SQL Server Management Studio连接了数据库并进行了一些操作导致的，断开连接即可；
     </p>
    </blockquote>
    <ul>
     <li>
      自定义初始化器：可以继承上述初始化器，创建自定义初始化器，重写Seed方法添加初始数据等。
     </li>
    </ul>

    <h3>
     <a id="27_262">
     </a>
     【2.7】三个命令
    </h3>
    <p>
     <strong>
      enable-migrations
     </strong>
    </p>
    <ul>
     <li>
      用于自动迁移或代码迁移时生成Configuration类；
     </li>
     <li>
      通过 “-EnableAutomaticMigration:$true” 参数控制是自动迁移还是代码迁移；
      <ul>
       <li>
        自动迁移
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a63e166a1c96f8a6dce2e6d9eb3a82fa.png"/>
       </li>
       <li>
        代码迁移
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a6c8c03964c30fb434ecb3656ade124e.png"/>
       </li>
      </ul>
     </li>
     <li>
      如果已存在的目标数据库中__MigrationHistory表的ContextKey与enable-migrations命令默认的ContextKey不相同，则会在Configuration类的构造函数中生成ContextKey，同时生成当前数据库实体类模型的迁移类文件；
     </li>
    </ul>
    <p>
     <strong>
      add-migration [name]
     </strong>
    </p>
    <ul>
     <li>
      用于生成当前实体类模型的迁移类文件；
     </li>
     <li>
      （代码迁移）如果当前最新的迁移类文件，还没有通过update-database命令更新到数据库，则不能通过add-migration [name]命令生成更新的迁移类文件；
     </li>
    </ul>
    <p>
     <strong>
      update-database
     </strong>
    </p>
    <ul>
     <li>
      用于按照迁移策略更新数据库；
     </li>
     <li>
      （代码迁移）最新的实体类模型在未使用add-migration [name]命令生成其迁移类时，无法更新数据库；
     </li>
     <li>
      （代码迁移）如果要删除某迁移类文件，最好先将数据库回滚到此迁移的上一版本，再删除；
     </li>
    </ul>
