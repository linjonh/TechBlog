    <h2>
     <a id="PRpull_rquest_1">
     </a>
     如何向开源项目提交PR（pull rquest）
    </h2>
    <p>
     在开发开源项目时，我们经常需要向开源项目提交PR，本文将介绍如何向开源项目提交PR（pull rquest）。本文将以本人向
     <code>
      ant design
     </code>
     提交PR为例，讲清楚具体操作。
    </p>
    <h4>
     <a id="_Pull_Request_5">
     </a>
     什么是 Pull Request?
    </h4>
    <p>
     Pull Request（PR）是 GitHub 上用于请求将代码合并到主分支的机制。
    </p>
    <p>
     就是在 GitHub 上提交代码，然后向开源项目发起请求，让开源项目将代码合并到主分支。
    </p>
    <h4>
     <a id="_11">
     </a>
     步骤
    </h4>
    <p>
     在开源项目中提交PR（Pull Request）是一种常见的贡献方式，它允许开发者向项目仓库提交代码更改并请求合并到主分支中。以下是提交PR的一般步骤：
    </p>
    <ul>
     <li>
      1、 Fork仓库
     </li>
    </ul>
    <p>
     首先，你需要Fork（即复制）你想要贡献的开源项目的仓库到你的个人GitHub账户下。这样，你就可以在自己的仓库中进行更改，而不会直接影响到原始仓库。
    </p>
    <ul>
     <li>
      2、 Clone代码
     </li>
    </ul>
    <p>
     将Fork后的仓库Clone（克隆）到你的本地开发环境中。
    </p>
    <ul>
     <li>
      3、 创建分支
     </li>
    </ul>
    <p>
     在本地开发环境中，创建一个新分支，用于存储你的更改。
    </p>
    <ul>
     <li>
      4、 修改代码
     </li>
    </ul>
    <p>
     在新分支中，修改代码，添加新功能或修复bug。
    </p>
    <ul>
     <li>
      5、 提交更改
     </li>
    </ul>
    <p>
     在本地开发环境中，提交更改到本地仓库。
    </p>
    <ul>
     <li>
      6、 推送更改
     </li>
    </ul>
    <p>
     将更改推送到Fork后的仓库（自己的远程仓库）。
    </p>
    <ul>
     <li>
      7、 创建PR
     </li>
    </ul>
    <p>
     在GitHub上，打开Fork后的仓库，然后点击"New Pull Request"按钮。然后，选择你的分支和目标分支（通常是主分支），并填写必要的信息，如PR标题和描述。
    </p>
    <ul>
     <li>
      8、 等待审核
     </li>
    </ul>
    <p>
     提交PR后，其他贡献者将收到通知，并需要审核你的更改。他们可能会进行必要的修改或反馈。
    </p>
    <ul>
     <li>
      9、 合并PR
     </li>
    </ul>
    <p>
     如果PR被接受，其他贡献者将收到通知，并需要合并你的更改。
    </p>
    <h4>
     <a id="_51">
     </a>
     示例
    </h4>
    <p>
     以下为例，以向
     <code>
      ant design
     </code>
     提交PR为例：
    </p>
    <ul>
     <li>
      1、 Fork仓库
     </li>
    </ul>

    </p>
    <ul>
     <li>
      2、 Clone代码
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/mySkey/ant-design.git
</code></pre>
    <ul>
     <li>
      3、 创建分支
     </li>
    </ul>
    <p>
     修复
     <code>
      Anchor 锚点组件

     <code>
      replace

     方法导致滚动动画失效的bug
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout <span class="token parameter variable">-b</span> fix/5.17.3-anchor-replace
</code></pre>
    <ul>
     <li>
      4、 修改代码
     </li>
    </ul>
    <pre><code>*************
</code></pre>
    <ul>
     <li>
      5、 提交更改
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"fix: anchor-replace"</span>
</code></pre>
    <ul>
     <li>
      6、 推送更改
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">git</span> push --set-upstream origin fix/5.17.3-anchor-replace
</code></pre>
    <ul>
     <li>
      7、 创建PR
     </li>
    </ul>
    <p>
     可以在
     <code>
      ant design
     </code>
     仓库创建PR请求
    </p>

     <code>
      ant design仓库首页

     /
     <code>
      自己的远程仓库
     </code>
     直接创建PR请求
    </p>
    <p>
     <img alt="20240605142501" src="https://i-blog.csdnimg.cn/blog_migrate/ac5adc7a554abd3b83720b0a9fc5d309.png"/>
    </p>
    <p>
     <img alt="20240605142317" src="https://i-blog.csdnimg.cn/blog_migrate/d39c46998105dd43367ac00616625a19.png"/>
    </p>
    <ul>
     <li>
      8、 等待审核
     </li>
    </ul>
    <p>
     <img alt="20240605142604" src="https://i-blog.csdnimg.cn/blog_migrate/924a236e7293c9517d2d20d618b390a0.png"/>

    <ul>
     <li>
      9、 合并PR
     </li>
    </ul>
    <p>
     通过审核后，PR将合并到主分支。即
     <code>
      ant design
     </code>
     仓库的
     <code>
      master
     </code>
     分支将包含你的更改。
    </p>

     <a id="_112">

     等待发布

      ant design

     更新日志查看自己的PR。
