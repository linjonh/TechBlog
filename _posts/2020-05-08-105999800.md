    <h2>
     <a id="_1">
     </a>
     前言
    </h2>
    <blockquote>
     <p>
      <strong>
       Minecraft沙盒游戏，它不单单可以让玩家游戏，玩家还能自己创建MC的服务器来供其他的玩家联机。
       <br/>
       <code>
        Multiverse
       </code>
       插件可以管理服务器的地图，创建新的世界、修改世界名称、世界的难度等等。
      </strong>
     </p>
    </blockquote>

    <h2>
     <a id="_13">
     </a>
     一、指令
    </h2>
    <blockquote>
     <p>
      输入指令
      <code>
       /mv
      </code>
      即可查看所有的指令
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/acaa16051f212b053daefe1af101e4a4.png"/>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        指令
       </th>
       <th>
        介绍
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        /mv
       </td>
       <td>
        列出所有指令
       </td>
      </tr>
      <tr>
       <td>
        /mv version
       </td>
       <td>
        显示插件版本
       </td>
      </tr>
      <tr>
       <td>
        /mv info [世界] [页数]
       </td>
       <td>
        查询该世界的信息，不填页数时默认第一页
       </td>
      </tr>
      <tr>
       <td>
        /mv create [名称] [类型]
       </td>
       <td>
        创建某个类型的世界
       </td>
      </tr>
      <tr>
       <td>
        /mv clone [目标世界] [名称]
       </td>
       <td>
        克隆某个世界到新世界
       </td>
      </tr>
      <tr>
       <td>
        /mv import [名称] [类型]
       </td>
       <td>
        导入某个世界地图
       </td>
      </tr>
     </tbody>
    </table>

     <a id="1_configyml_30">

     [1]. config.yml
    </h3>
    <blockquote>
     <p>
      如果你想要在聊天中添加世界的名称时，可以在
      <code>
       prefixchat
      </code>
      后填
      <code>
       true

      ,否则为
      <code>
       false
      </code>
      .其他的配置基本不需要变更了。

    <h2>
     <a id="_54">
     </a>
     三、地图的架构
    </h2>
    <blockquote>
     <p>
      因为我用的是Spigot类型的服务端，所以地图是直接放在主目录的，如果是其他类型的服务端可能就是放在world文件夹下了。
     </p>
    </blockquote>
    <blockquote>
     <p>
      在这得介绍一下world文件夹，服务器中有很多的地图文件，world就是其中的一个，但其他的地图并没有保存玩家的信息，而是保存在了这个world地图文件夹之中。
      <br/>
      <code>
       如果你想要换服务器的主地图时，记得将主地图中的playerdata文件夹移动并替换
      </code>
      。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/74956a30c6d7fe71ab9a41f68c60b3c7.png"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/23a9c3b371836dcf8b1c6ab973e8e19a.png"/>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        配置文件
       </th>
       <th>
        介绍
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        advancements
       </td>
       <td>
        玩家成就

       <td>
        data

    <h2>
     <a id="_70">
     </a>
     四、世界创建
    </h2>
    <blockquote>
     <p>
      输入指令
      <code>
       /mv create
      </code>
      即可查看所有可用的创建世界指令
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c0f58347c3bb4de023f3bbdcc2369189.png"/>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        指令
       </th>
       <th>
        介绍
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        /mv create [名称] normal
       </td>
       <td>
        生成一个默认类型的世界（跟主世界一样）
       </td>
      </tr>
      <tr>
       <td>
        /mv create [名称] nether
       </td>
       <td>
        生成一个地狱类型的世界
       </td>
      </tr>
      <tr>
       <td>
        /mv create [名称] end
       </td>
       <td>
        生成一个末地类型的世界
       </td>
      </tr>
      <tr>
       <td>
        /mv create [名称] normal -t flat
       </td>
       <td>
        生成一个平坦世界
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      当你创建过后，将会在服务端的主目录中生成一个相应的地图文件夹
      <br/>
      这两个地图就是我通过指令创建的
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5863cd1c630c7a7a319cb5a7e009ce48.png"/>
    </p>
    <h2>
     <a id="_85">
     </a>
     五、地图导入
    </h2>
    <blockquote>
     <p>
      如果你想要把地图导入的话，你得先提前把地图文件夹放在主目录中，就像步骤三的那样。
      <br/>
      然后输入
      <code>
       /mv import [名称] [类型]
      </code>
      ，之后就会显示导入成功，或者使用
      <code>
       /mv list
      </code>
      指令查看是否已经导入。
     </p>
    </blockquote>
