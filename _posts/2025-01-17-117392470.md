    <h3>
     <a id="DNS_1">
     </a>
     什么是DNS？
    </h3>
    <p>
     DNS：
    </p>
    <pre><code class="prism language-powershell">域名系统（英文：Domain Name System，缩写：DNS）是互联网的一项服务。
它作为将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网。
DNS使用UDP端口53。
当前，对于每一级域名长度的限制是63个字符，域名总长度则不能超过253个字符。
</code></pre>
    <p>
     DNS作用：
    </p>
    <pre><code class="prism language-powershell">当我们在浏览器输入www<span class="token punctuation">.</span>baidu<span class="token punctuation">.</span>com时，计算机并不知道谁是baidu
计算机只能识别ip，所有我们需要将www<span class="token punctuation">.</span>baidu<span class="token punctuation">.</span>com这个域名解析为ip才能继续通信
dns就提供了这个解析功能
</code></pre>
    <p>
     DNS分类：
    </p>
    <pre><code class="prism language-powershell">1<span class="token punctuation">.</span>权威DNS：
权威DNS是经过上一级授权对域名进行解析的服务器，同时它可以把解析授权转授给其他人。
如COM顶级服务器可以授权dns<span class="token punctuation">.</span>com这个域名的的权威服务器为NS<span class="token punctuation">.</span>ABC<span class="token punctuation">.</span>COM。
同时NS<span class="token punctuation">.</span>ABC<span class="token punctuation">.</span>COM还可以把授权转授给NS<span class="token punctuation">.</span>DDD<span class="token punctuation">.</span>COM。
这样NS<span class="token punctuation">.</span>DDD<span class="token punctuation">.</span>COM就成了ABC<span class="token punctuation">.</span>COM实际上的权威服务器了。
平时我们解析域名的结果都源自权威DNS。
比如dns<span class="token punctuation">.</span>com的权威DNS服务器就是帝恩思的ns1<span class="token punctuation">.</span>dns<span class="token punctuation">.</span>com ns2<span class="token punctuation">.</span>dns<span class="token punctuation">.</span>com
权威名称服务器的类型包括:
Master : 包含原始区域数据，有时称作 “主要 ”名称服务器
Slave : 备份服务器，通过区域传送从 Master 服务器获得的区域数据的副本，有时称作 “次要 ”名称服务器

2<span class="token punctuation">.</span>递归DNS（也叫本地DNS）:
其核心功能一个是缓存、一个是递归查询。
收到域名查询请求后其首先看本地缓存是否有记录，
如果没有则一级一级的查询根、顶级域、二级域……直到获取到结果然后返回给用户。
日常上网中运营商分配的DNS即这里所说的递归DNS。
<span class="token operator">/</span><span class="token operator">/</span>
负责接受用户对任意域名查询，并返回结果给用户。
递归DNS可以缓存结果以避免重复向上查询。
我们平时使用最多的就是这类DNS，他对公众开放服务，一般由网络运营商提供。
递归DNS一定要有可靠的互联网连接方可使用。
比如谷歌的8<span class="token punctuation">.</span>8<span class="token punctuation">.</span>8<span class="token punctuation">.</span>8和8<span class="token punctuation">.</span>8<span class="token punctuation">.</span>4<span class="token punctuation">.</span>4以及114的114<span class="token punctuation">.</span>114<span class="token punctuation">.</span>114<span class="token punctuation">.</span>114和114<span class="token punctuation">.</span>114<span class="token punctuation">.</span>115<span class="token punctuation">.</span>115都属于这一类DNS。
本地电脑上设置的DNS就是这类DNS。

3<span class="token punctuation">.</span>转发DNS:
转发DNS是一种特殊的递归。
如果本地的缓存记录中没有相应域名结果时，
其将查询请求转发给另外一台DNS服务器，由另外一台DNS服务器来完成查询请求。
<span class="token operator">/</span><span class="token operator">/</span>
负责接受用户查询，并返回结果给用户。
但这个结果不是按标准的域名解析过程得到的，而是直接把递归DNS的结果转发给用户。
它也具备缓存功能。
他主要使用在没有直接的互联网连接，但可以连接到一个递归DNS那里，这时使用转发DNS就比较合适。
其缺陷是：直接受递归DNS的影响，服务品质较差。
比如我们用的路由器里面的DNS就是这一类，查看本地电脑的DNS一般都是192<span class="token punctuation">.</span>168<span class="token punctuation">.</span>1<span class="token punctuation">.</span>1。
</code></pre>
    <p>
     DNS解析过程：
    </p>
    <pre><code class="prism language-powershell">比如，一个域名为：abc<span class="token punctuation">.</span>com，是想看到这个现HTTP服务，如果要访问网站，就要进行解析。
首先在域名注册商那里通过专门的DNS服务器解析到一个WEB服务器的一个固定IP上：211<span class="token punctuation">.</span>214<span class="token punctuation">.</span>1<span class="token punctuation">.</span>XXX
然后，通过WEB服务器来接收这个域名，把abc<span class="token punctuation">.</span>com这个域名映射到这台服务器上。
那么，输入abc<span class="token punctuation">.</span>com这个域名就可以实现访问网站内容了。

人们习惯记忆域名，但机器间互相只认IP地址，域名与IP地址之间是对应的
它们之间的转换工作称为域名解析，域名解析需要由专门的域名解析服务器来完成，整个过程是自动进行的。
域名解析协议（DNS）用来把便于人们记忆的主机域名和电子邮件地址映射为计算机易于识别的IP地址。
DNS是一种c<span class="token operator">/</span>s的结构。
客户机就是用户用于查找一个名字对应的地址，而服务器通常用于为别人提供查询服务。
</code></pre>
    <p>
     DNS解析方式：逐级解析
    </p>
    <pre><code class="prism language-powershell">根域名 ：<span class="token punctuation">.</span>root 或者 <span class="token punctuation">.</span> ，通常是省略的
顶级域名，如 <span class="token punctuation">.</span>com，<span class="token punctuation">.</span>cn 等
次级域名，如 baidu<span class="token punctuation">.</span>com 里的 baidu，这个是用户可以进行注册购买的
主机域名，比如 baike<span class="token punctuation">.</span>baidu<span class="token punctuation">.</span>com 里的baike，这个是用户可分配的

主机名<span class="token punctuation">.</span>次级域名<span class="token punctuation">.</span>顶级域名<span class="token punctuation">.</span>根域名baike<span class="token punctuation">.</span>baidu<span class="token punctuation">.</span>com<span class="token punctuation">.</span>root

<span class="token comment">##根域名服务器都在美国，一共有13个</span>
</code></pre>
    <p>
     DNS记录类型（域名解析规则）：
    </p>
    <pre><code class="prism language-powershell">A：地址记录（Address），返回域名指向的IP地址。

NS：域名服务器记录（Name Server），返回保存下一级域名信息的服务器地址。
该记录只能设置为域名，不能设置为IP地址。

MX：邮件记录（Mail eXchange），返回接收电子邮件的服务器地址。

CNAME：规范名称记录（Canonical Name），返回另一个域名，即当前查询的域名是另一个域名的跳转，详见下文。

PTR：逆向查询记录（Pointer Record），只用于从IP地址查询域名，详见下文。
</code></pre>
    <p>
     DNS解析报错信息：
    </p>
    <pre><code class="prism language-powershell">1<span class="token punctuation">.</span>no servers could be reached	<span class="token comment">##服务无法访问（服务开启？火墙？网络？端口？）</span>

</code></pre>

    <h3>
     <a id="DNS_114">
     </a>
     配置高速缓存DNS服务器
    </h3>
    <blockquote>
     <p>
      所谓高速缓存dns就是：
      <br/>
      内网内的dns服务器主机（非权威dns）把缓存下来的数据共享给其他内网主机。
      <br/>
      只有第一次访问时间较长，访问后就会保存下来，之后内网内使用该dns服务器的主机再次访问就可以直接从缓存里取，速度会非常快。
     </p>
    </blockquote>
    <p>
     dns的安装及开启：
    </p>
    <pre><code class="prism language-powershell">dnf install bind<span class="token punctuation">.</span>x86_64 <span class="token operator">-</span>y 		<span class="token comment">#安装</span>
<span class="token comment">#Berkeley Internet Name Domain (BIND)</span>

systemctl enable <span class="token operator">--</span>now named	<span class="token comment">#启用dns服务，服务名称叫named</span>
firewall<span class="token operator">-</span>cmd <span class="token operator">--</span>permanent <span class="token operator">--</span><span class="token function">add-service</span>=dns <span class="token comment">#火墙设置</span>
firewall<span class="token operator">-</span>cmd <span class="token operator">--</span>reload			<span class="token comment">#重启火墙</span>
setenforce 0					<span class="token comment">#调整内核级火墙为警告模式</span>
</code></pre>
    <p>
     配置dns服务使内网主机可以访问其dns资源：
    </p>
    <pre><code class="prism language-powershell">netstat <span class="token operator">-</span>antlupe <span class="token punctuation">|</span> grep named				<span class="token comment">#可以查看dns开放端口</span>

vim <span class="token operator">/</span>etc<span class="token operator">/</span>named<span class="token punctuation">.</span>conf							<span class="token comment">#编辑dns配置文件，对文件做以下修改</span>
11行左右 listen<span class="token operator">-</span>on port 53 <span class="token punctuation">{<!-- --></span> any<span class="token punctuation">;</span> <span class="token punctuation">}</span><span class="token punctuation">;</span> 		<span class="token comment">#在本地所有网络接口上开启53端口</span>
19行左右 allow<span class="token operator">-</span>query <span class="token punctuation">{<!-- --></span> any<span class="token punctuation">;</span> <span class="token punctuation">}</span><span class="token punctuation">;</span> 				<span class="token comment">#允许查询A记录的客户端列表，提供服务给所有主机</span>
20行左右 forwarders <span class="token punctuation">{<!-- --></span> 114<span class="token punctuation">.</span>114<span class="token punctuation">.</span>114<span class="token punctuation">.</span>114<span class="token punctuation">;</span> <span class="token punctuation">}</span><span class="token punctuation">;</span>	<span class="token comment">#将源换为国内的，指定资源从114上取，这样速度会快一点</span>
34行左右 dnssec<span class="token operator">-</span>validation no<span class="token punctuation">;</span> 				<span class="token comment">#禁用dns检测使dns能够缓存外部信息到本机，开着会因为检测不合规返回不了“答案”</span>
<span class="token comment">##每一个字符串一定要用分号结尾</span>

systemctl restart named						<span class="token comment">#重启服务</span>
</code></pre>

     主机记录A记录
    </h4>
    <p>
     第一步：添加域名语句块
    </p>
    <pre><code class="prism language-powershell">vim <span class="token operator">/</span>etc<span class="token operator">/</span>named<span class="token punctuation">.</span>rfc1912<span class="token punctuation">.</span>zone
