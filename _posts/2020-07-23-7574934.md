    <p>
     <span style="color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
      这个问题很有意思，我觉得回答这个问题可以从两个方面回答，互联网产品设计以及移动手机客户端的产品设计，所谓的用户体验，其实我个人认为，就是让用户"习惯", Don't let user think. 这个是终极目标啦，随着产品的不断的改善，用户体验也在随之变化。但是，如果你在做一款产品，所谓的标准，终极检验方法，还是用户，以及用户对于产品的反馈。 对于互联网产品的设计，我个人有一些标准的用户习惯，个人认为可以作为所谓量化的标准，我分享出来，希望能够对你有帮助。用户界面设计的
     </span>
     <span style="color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
      三大原则
     </span>
     <span style="color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
      是：
     </span>
    </p>
    <ol style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      置界面于用户的控制之下；
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      减少用户的记忆负担；
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      保持界面的一致性。
      <br/>
     </li>
    </ol>
    <sup style="border-left-width:3px; border-left-style:solid; border-left-color:rgb(208,229,242); padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:10px; display:block; line-height:22px; vertical-align:baseline; font-size:13px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif">
     用户界面设计在工作流程上分为
     <strong>
      结构设计、交互设计、视觉设计
     </strong>
     三个部分。
     <strong>
      结构设计
     </strong>
     也称概念设计 （Conceptual Design），是界面设计的骨架。通过对用户研究和任务分析，制定出产品的整体架构。
     <strong>
      交互设计
     </strong>
     的目的是使产品让用户能简单使用。 任何产品功能的实现都是通过人和机器的交互来完成的。因此，人的因素应作为设计的核心被体现出来。
     <strong>
      视觉设计
     </strong>
     包括色彩、字体、页面设计等。视觉设计要达到用户愉悦使用的目的。
    </sup>
    <span style="color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     以下我针对用户设计的三个大类，通过“指标”就是我们用来衡量用户体验的标准，将标准分为三个等级，分别占0分、1分、2分，每个标准都有它的评分标准，当用户反馈的问题完全符合这个标准时，就占0；如问题不符合标准，就占2分；如问题不严重，只是还不够完善，就占1分；最后拿问题去套这里的标准，得出来的总分就是我们要评定改版的指标了。总分越高，说明该条的用户体验越差，最后我们将得出的总分来判断问题的用户体验值：总分在0 – 5 分且单个标准在2分以下的（标示用户体验为良好）；总分在5 – 10分且单个标准在2分以下的（标示用户体验为较差）；总分在十分以上或有2个2分以上的标准的（标示用户体验为很差）。
    </span>
    <ul style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      <strong>
       结构设计
      </strong>
      <br/>
     </li>
    </ul>
    <ol style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      有足够的空间让用户看到主要的内容，要适合多数浏览器浏览（以15、17寸、19寸显示器为主）
      <strong>
       0：
      </strong>
      足够看到主要内容
      <strong>
       1：
      </strong>
      只看到2/3看到
      <strong>
       2：
      </strong>
      小于1/3的内容
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      尽量避免使用结构复杂的表格，表格嵌套不要超过3层；
      <strong>
       0：
      </strong>
      小于等于2层
      <strong>
       1：
      </strong>
      三到四层
      <strong>
       2：
      </strong>
      五层或五层以上
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      页面长度原则上不超过3屏，宽度不超过1屏幕。（以1024X768为准）
      <strong>
       0:
      </strong>
      长小于3屏，宽小于1屏
      <strong>
       1:
      </strong>
      长超出3屏或宽超出1屏
      <strong>
       2:
      </strong>
      长超出3屏，宽超1屏
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      页面避免使用iframe，如果必须使用，采用对应的优化方式。（优化是指：对浏览器是否支持框架进行判断以及iframe宽高度自适应页面）
      <strong>
       0:
      </strong>
      无iframe
      <strong>
       1:
      </strong>
      有1-2个iframe，且无优化
      <strong>
       2:
      </strong>
      多于2个iframe,且无优化
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      页面布局要重点突出，图文并茂；做数据统计，将目标客户最感兴趣的放置在最重要最显著的位置（一般为页面的头部和左上角）
      <strong>
       0:
      </strong>
      重点突出，图文并茂，且在最重要的位置放的是用户最感兴趣的
      <strong>
       1:
      </strong>
      重点不突出或没有图文并茂
      <strong>
       2:
      </strong>
      重点不突出且没有图文并茂
      <br/>
     </li>
    </ol>
    <ul style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      <strong>
       交互设计
      </strong>
      <br/>
     </li>
    </ul>
    <ol style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      表单的填写流程要清晰、简洁，必填的项要放在页面的显著位置（主体页面的头部，），非必填项不能影响用户填写的效率。
      <strong>
       0：
      </strong>
      必填项位置显著，非必填项没有影响用户填写的效率
      <strong>
       1：
      </strong>
      必填项位置不显著或非必填项影响了用户填写的效率
      <strong>
       2：
      </strong>
      必填项和非必填项混在一起，影响了用户填写的效率
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      表单的填写尽量多采用既有选择也有输入的方式，需填写部分需注明内容的实例，并对字段作出限制。
      <strong>
       0：
      </strong>
      既有选择也有输入的方式，并有注明实例
      <strong>
       1：
      </strong>
      没有选择性输入或没有注明填写的实例
      <strong>
       2:
      </strong>
      没有选择性输入且没有注明填写的实例
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      对于交互性的按钮必须清晰突出，以确保用户可以清楚地点击。
      <strong>
       0：
      </strong>
      调查10人（8人或以上的人很快就能点击到按钮）
      <strong>
       1：
      </strong>
      调查10人（只一半的人很快就能点击到按钮）
      <strong>
       2:
      </strong>
      调查10人（6人或以上的人在短时间找不到要点击的按钮）
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      点击：浏览过的信息需要显示为不同的状态，以区分于未阅读内容，避免重复阅读。
      <strong>
       0：
      </strong>
      有不同的状态显示
      <strong>
       1：
      </strong>
      只有部分有不同的状态显示
      <strong>
       2:
      </strong>
      没有状态显示
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      尽量减少新开的窗口，以避免开过多的无效窗口：打开的IE新窗口不超过2层；打开的IE / DIV / alert 新窗口总不超过三个，避免在DIV窗口再打开DIV窗口。
      <strong>
       0：
      </strong>
      IE窗口&lt;=2 IE+DIV+ALERT 新窗口 &lt;=3,DIV窗口没有再打开DIV窗口
      <strong>
       1：
      </strong>
      IE窗口&gt;=2或 IE+DIV+ALERT 新窗口 &gt;=3或DIV窗口没有再打开DIV窗口
      <strong>
       2:
      </strong>
      IE窗口&gt;=2 IE+DIV+ALERT 新窗口 &gt;=3,DIV窗口有再打开DIV窗口
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      涉及到用户操作的东西，要有及时、清楚的错误、成功或信息提示；并要能保存上一步操作。
      <strong>
       0：
      </strong>
      有及时清楚地提示，能保存上一步操作
      <strong>
       1：
      </strong>
      无提示或不能保存上一步操作
      <strong>
       2:
      </strong>
      无提示也不能保存上一步操作
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      对于每一个点击进行友好提示，以增加浏览者的亲和度，对图片、图标、链接有TIP提示或在相应位置加“说明”、“注意”的文字信息。
      <strong>
       0：
      </strong>
      每一个点击进行友好提示
      <strong>
       1：
      </strong>
      有提示，但提示不友好 无友好的提示
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      对用户输入性的操作，提供快速反馈；将光标定位到第一个要输入的表单对象上，给用户心理上的暗示，避免用户焦急。
      <strong>
       0：
      </strong>
      有快速的反馈，且有光标定位
      <strong>
       1：
      </strong>
      反馈不快速或无光标定位
      <strong>
       2:
      </strong>
      无快速反馈、无光标定位
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      栏目的命名要与栏目内容准确相关，简洁清晰，不宜过于深奥。
      <strong>
       0：
      </strong>
      调查10人（8人或以上的人认为栏目命名很好）
      <strong>
       1：
      </strong>
      调查10人（只一半的人认为栏目命名好）
      <strong>
       2:
      </strong>
      调查10人（6人或以上的人认为栏目命名深奥）
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      导航要清晰，栏目的层级最多不超过三层；且随时转移功能，很容易从一个功能跳到另外一个功能。
      <strong>
       0：
      </strong>
      导航栏的层级不超过3层
      <strong>
       1：
      </strong>
      导航栏的层级为4层
      <strong>
       2:
      </strong>
      导航栏的层级多于4层
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      在页面的醒目位置（主体页面的头部），提供信息搜索框和排序功能，便于查找到所需内容
      <strong>
       0：
      </strong>
      有醒目的搜索框和排序功能
      <strong>
       1：
      </strong>
      有搜索框和排序但不醒目
      <strong>
       2:
      </strong>
      无搜索框和排序
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      正常情况下,尽量确保页面在5秒内打开。如果是大 型门户网站，必须考虑南北互通问题，进行必要的压力测试
      <strong>
       0：
      </strong>
      1 – 2秒内打开
      <strong>
       1：
      </strong>
      3 -5秒内打开
      <strong>
       2:
      </strong>
      5秒以上打开
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      让用户控制界面（界面的大小、位置、最大化、最小化、关闭、滚动条控制等）
      <strong>
       0：
      </strong>
      用户能很好的控制
      <strong>
       1：
      </strong>
      控制不完善
      <strong>
       2:
      </strong>
      用户不能控制
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      无论用户浏览到哪一个层级，哪一个页面，都可以清楚知道自己当前的位置，使其做出下一步行动的决定。
      <strong>
       0：
      </strong>
      每个层级每个页面都有当前的位置
      <strong>
       1：
      </strong>
      没有当前的位置或不知道下步的操作
      <strong>
       2:
      </strong>
      用户不知道自己在哪，也不知道下一步干嘛。
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      对信息量大的操作要有批量处理的功能（如审核、删除、添加等）
      <strong>
       0：
      </strong>
      有批量处理的功能
      <strong>
       1：
      </strong>
      只有部分有批量处理
      <strong>
       2:
      </strong>
      无批量处理的功能
     </li>
    </ol>
    <ul style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      <strong>
       视觉设计
      </strong>
     </li>
    </ul>
    <ol style="padding-top:0px; padding-right:0px; padding-bottom:0px; padding-left:0px; list-style-position:outside; margin-top:10px; margin-right:0px; margin-bottom:10px; margin-left:26px; color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      界面风格要协调一致，让用户有一个整体的感觉。 （最好能提供换肤的功能）
      <strong>
       0：
      </strong>
      典型用户调查10人（符合8-7人的习惯）
      <strong>
       1：
      </strong>
      典型用户调查10人（符合5 - 6人的习惯）
      <strong>
       2:
      </strong>
      典型用户调查10人（符合1-4的习惯）
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      采用易于阅读的字体，避免文字过小或过密造成的阅读障碍。（主体字 中文采用标准的12PX 宋体；英文采用12PX  Arial 字体）
      <strong>
       0：
      </strong>
      文字便于阅读且有字体大中小设置
      <strong>
       1：
      </strong>
      文字过小或文字过密
      <strong>
       2:
      </strong>
      文字过小且过密，没有字体大中小的设置
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      动画效果要与主画面相协调，文件大小在10K以上的要有加载画面。
      <strong>
       0：
      </strong>
      动画与页面协调，有加载画面
      <strong>
       1：
      </strong>
      动画与页面不太协调或10K以上动画无加载页面
      <strong>
       2:
      </strong>
      动画与页面不太协调且10K以上动画无加载页面
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      图片展示要比例协调、不变形，图片清晰。
      <strong>
       0：
      </strong>
      图片清晰且不变形
      <strong>
       1：
      </strong>
      图片不清晰或变形
      <strong>
       2:
      </strong>
      图片不清晰且变形
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      图标使用要简洁、明了、易懂、准确，与页面整体风格统一；尽量使用真实世界的比喻。如：电话、打印机的图标设计，尊重用户以往的使用经验。同样的功能用同样的图形。
      <strong>
       0：
      </strong>
      调查10人（8人或以上的人认为图标使用合理）
      <strong>
       1：
      </strong>
      调查10人（只一半的人认为图标合理）
      <strong>
       2:
      </strong>
      调查10人（6人或以上的人认为图标使用不合理）
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      对于长篇文章 进行分页浏览，超过1.5屏时 进行翻页（15寸显示器为准）
      <strong>
       0：
      </strong>
      超过1.5屏能分页浏览
      <strong>
       1：
      </strong>
      2屏以上才分页浏览
      <strong>
       2:
      </strong>
      无分页浏览功能
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      色彩与内容。整体不超过5个色系，尽量少用红色、绿色。近似的颜色表示近似的意思。以恰当的色彩明度和亮度，确保浏览者的浏览舒适度。
      <strong>
       0：
      </strong>
      小于5，主色没有用到红色和绿色
      <strong>
       1：
      </strong>
      整体色调是红或绿或整体色系超出5个
      <strong>
       2:
      </strong>
      整体色调就是红或绿，色系超出5个
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      提供界面的快捷方式，一个功能有多个入口；快捷方式体现出其关联性，放在显著的位置（页面的头部或左上角）
      <strong>
       0：
      </strong>
      在显著有关联性的快捷方式
      <strong>
       1：
      </strong>
      有快捷方式但位置不显著
      <strong>
       2:
      </strong>
      无快捷方式
      <br/>
     </li>
     <li style="margin-top:0px; margin-right:0px; margin-bottom:5px; margin-left:0px">
      减少短期记忆的负担。让计算机帮助记忆，例：登录的用户名、密码、IE进入界面地址可以让机器记住。
      <strong>
       0：
      </strong>
      有计算机帮助记忆
      <strong>
       1：
      </strong>
      有部分记忆功能
      <strong>
       2:
      </strong>
      无计算机帮助记忆。
     </li>
    </ol>
    <p>
     <span style="color:rgb(34,34,34); font-family:'Helvetica Neue',Helvetica,Arial,sans-serif; font-size:13px; line-height:22px">
      对于手机客户端的用户体验标准，我想也可以从这三个方向进行评估：结构设计请检验：1、2、5 交互设计可以全部，但是第5项改为尽可能减少打开链接。视觉设计也是可以全部，但是对于第二项字体，iphone以及安卓，有自己标准的字体，按照其标准设计，就是用户体验的最佳状态。另苹果对UI是由标准的，下载一份标准。按照标准来，体验最好。
