    <h4>
     <a id="_3">
     </a>
     典型组网
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7ffded36f1fe57de0a16e7e5be7cf0d1.png"/>

    <h4>
     <a id="VLAN_5">
     </a>
     VLAN技术
    </h4>
    <p>
     VLAN（Virtual Local Area Network）虚拟局域网技术
    </p>
    <ul>
     <li>
      将一个物理的局域网LAN通过规划广播域在逻辑上划分成多个虚拟局域网VLAN的网络技术，不同VLAN之间不能直接通讯；
     </li>
     <li>
      提高网络处理性能，提高网络安全性，提高网络管理能力；是最常用的网络技术；
     </li>
     <li>
      主要技术原理：在以太网数据帧中加入4个字节的VLAN标签，即VLAN Tag ， 如VLAN 10；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/218134e771ebc8ab9e69a7eb871dc319.png"/>
     </li>
    </ul>
    <h4>
     <a id="Trunk_11">
     </a>
     Trunk技术
    </h4>
    <p>
     Trunk技术
    </p>
    <ul>
     <li>
      交换机端口三种链路类型：Access、Trunk、Hybrid；
     </li>
     <li>
      Access类型端口：只属于单一VLAN，只允许该VLAN通过；一般用于连接主机；
      <br/>
      +Trunk类型端口：属于多个VLAN，接受和发送多个VLAN报文；一般用于交换机与交换机之间的连接；
     </li>
     <li>
      Hybrid类型端口：与Trunk类似，但在处理数据帧的发送时，原理略有不同，可以用于连接交换机，也可以连接主机；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1f86a305bfc9a0ce8f4c7b52f5b6f937.png"/>
     </li>
    </ul>
    <h4>
     <a id="bond_18">
     </a>
     bond技术
    </h4>
    <p>
     bond技术
    </p>
    <ul>
     <li>
      bond是在linux操作系统下一种多网卡绑定形成冗余关系的奇数；
     </li>
     <li>
      主要应用场景是为了解决：当某一交换机/物理机服务器网络端口故障，不影响网络通讯的正常
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/24e1f52f883f5c2489bc42a4710683ca.png"/>
     </li>
    </ul>
    <h3>
     <a id="linux_23">
     </a>
     linux网络基础技术
    </h3>
    <h4>
     <a id="_24">
     </a>
     网桥技术
    </h4>
    <p>
     Linux网桥（Linux Bridge）
    </p>
    <ul>
     <li>
      Linux系统中用于二层网络通讯的交换设备，可理解为运行在操作系统中的一个虚拟交换机（下图中的br0）。同物理交换机一样，上联端口即eth0（物理接口）连接物理网络，下联端口即vnetx连接云主机虚拟网络；
     </li>
     <li>
      通过为eth0配置VLAN子接口（如eth0.10），再创建对应网络（如br_eth0_10），实现VLAN配置与外部物理VLAN网络二层互通；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/750d21b3d3c4c69e4210586ca71743f0.png"/>
     </li>
    </ul>
    <h3>
     <a id="__29">
     </a>
     网络基础技术 总结
    </h3>
    <h4>
     <a id="VLAN_30">
     </a>
     VLAN技术
    </h4>
    <p>
     VLAN即虚拟局域网，能够帮助我们将一个局域网划分多个虚拟局域网，来提高网络性能，安全性以及网络管理能力；
    </p>
    <h4>
     <a id="Trunk_32">
     </a>
     Trunk技术
    </h4>
    <p>
     当多个VLAN需要跨不同的交换设备进行组网时，就需要为交换机之间相应的连接端口配置Trunk模式，以允许多个VLAN使用同一物理链路来进行不同VLAN网络的数据传输；
    </p>
    <h4>
     <a id="bond_34">
     </a>
     bond技术
    </h4>
    <p>
     为解决单块服务器网卡连接到接入交换机的场景中所存在的单点故障隐患，通常使用bond技术来进行多网卡绑定，任一网卡故障都不影响正常的网络传输，bond mode 1主备模式，bond mode 4聚合模式是最常用的bond模式，需要注意的是bond mode4需要上联交换机端口配置动态链路聚合；
    </p>
    <h4>
     <a id="_36">
     </a>
     网桥技术
    </h4>
    <p>
     网桥是linux操作系统中一种二层交换设备，可理解为虚拟交换机，通过虚拟设备连接运行在系统中的云主机（虚拟机），并通过物理接口与外部物理网络实现通讯，同样我们可以为虚拟网络配置VLAN实现与外部物理网络环境中不同VLAN的网络通讯。
    </p>
    <h2>
     <a id="_39">
     </a>
     云环境中的网络分类与典型规划介绍
    </h2>
    <h3>
     <a id="_40">
     </a>
     云环境中典型网络分类
    </h3>
    <h4>
     <a id="_41">
     </a>
     管理网络
    </h4>
    <ul>
     <li>
      云平台管理计算节点所使用的网络，如节点心跳判断、下发创建云主机指令；
     </li>
     <li>
      云主机迁移网络，将云主机从一台计算节点迁移到其他计算节点所使用的网络；
     </li>
     <li>
      备份网络，讲云主机备份到某一存储位置所使用的网络；
     </li>
    </ul>
    <h4>
     <a id="_45">
     </a>
     业务网络
    </h4>
    <ul>
     <li>
      云平台所承载业务系统云主机对外服务的网络；
     </li>
     <li>
      一般匹配用户实际的网络环境；如用户根据不同业务划分了不同业务网络，那在云平台中就存在多个业务网络；
     </li>
    </ul>
    <h4>
     <a id="_48">
     </a>
     存储网络
    </h4>
    <ul>
     <li>
      向云平台提供存储资源的网络；
     </li>
     <li>
      如果是NAS/分布式存储，则为IP网络；
     </li>
     <li>
      如梭是SAN存储，则为FC网络；
     </li>
    </ul>
    <h3>
     <a id="_52">
     </a>
     物理网络规划
    </h3>
    <h4>
     <a id="_53">
     </a>
     以一台服务器的物理网络连接为例
    </h4>
    <ul>
     <li>
      服务器配置4个前兆/万兆网络端口，2分FC端口；
     </li>
     <li>
      2个网卡绑定用于管理网络，承载节点管理，云主机迁移等；
     </li>
     <li>
      2个网卡绑定用于业务网络，用于提供业务服务访问；
     </li>
     <li>
      2个FC端口多路径，用于向云平台提供存储资源；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3fa9705dd58dce320e0462d6c74d1244.png"/>
     </li>
    </ul>

    <h4>

    </h4>

     <li>
      <p>
       2、修改物理网卡名字（可不做）

       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4f754f9ffe2ada1ad86c334bf244f778.png"/>

      </p>
     </li>
     <li>
      <p>
       3、创建bond0和bond1
       <br/>
       创建bond0 （创建主备）
       <br/>
       zs-bond-ab -c bond0
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ef3cc8ada94c1a9f6da199c825574680.png"/>
       <br/>
       再次创建bond1（创建聚合）
       <br/>
       zs-bond-lacp -c bond0
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5748c092053d6f6ac3bb40fa0137da59.png"/>
      </p>
     </li>
     <li>
      <p>
       4、将物理网卡添加到指定bond中
       <br/>
       把0和2加入到bond0中（用于管理类网络）
       <br/>
       zs-nic-to-bond -a bond0 zsnic0
       <br/>
       zs-nic-to-bond -a bond0 zsnic2
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f720c162942556d55b252510eac09884.png"/>
       <br/>

       <br/>
       zs-nic-to-bond -a bond1 zsnic1
       <br/>
       zs-nic-to-bond -a bond3 zsnic3
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/23bfc99d15f04f11a3256abbc8cb0771.png"/>
      </p>
     </li>
     <li>
      <p>
       5、修改VLAN ID 号
       <br/>
