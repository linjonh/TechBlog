    <h4>
     <span style="color:#f33b45;">
      <strong>
       ARM+LINUX路线，主攻嵌入式Linux操作系统
      </strong>
      及其上应用软件开发目标：
     </span>
     <br/>
     （1）掌握主流嵌入式微处理器的结构与原理（初步定为arm9）
     <br/>
     （2）必须掌握一个嵌入式操作系统 （初步定 为uclinux或linux,版本待定）
     <br/>
     （3）必须熟悉嵌入式软件开发流程并至少做一个嵌入式软件项目。
    </h4>
    <h4>
     <span style="color:#ffbb66;">
      从事嵌入式软件开发的缺点是：
     </span>
    </h4>
    <h4>
     （1） 入门起点较高，所用到的技术往往都有一定难度，若软硬件基础不好，特别是操作系统级软件功底不深，则可能不适于此行。
     <br/>
     （2）这方面的企业数量要远少于互联网类企业。同时薪资待遇也会低一些。
     <br/>
     （3）有少数公司经常要硕士以上的人搞嵌入式，主要是基于嵌入式的难度。但大多数公司也并无此要求，只要有经验即可。
     <br/>
     （4）平台依托强，换平台比较辛苦。
    </h4>
    <h2>
     <span style="color:#f33b45;">
      方法步骤：
     </span>
    </h2>
    <h3>
     <span style="color:#ffbb66;">
      1、基础知识：
     </span>
     <br/>
     <span style="color:#f33b45;">
      目的
     </span>
     ：能看懂硬件工作原理，但重点在嵌入式软件，特别是操作系统级软件，那将是我们的优势。
     <br/>
     <span style="color:#f33b45;">
      科目
     </span>
     ：数字电路、计算机组成原理、嵌入式微处理器结构。汇编语言、C/C++、编译原理、离散数学。数据结构和算法、操作系统、软件工程、网络、数据库。
     <br/>
     <span style="color:#f33b45;">
      方法
     </span>
     ：虽科目众多，但都是较简单的基础。不一定全学，可根据需要选修。
     <br/>
     <span style="color:#f33b45;">
      主攻书籍
     </span>
     ：the c++ programming language、数据结构-C2。
    </h3>
    <h3>
     <span style="color:#86ca5e;">
      2、学习linux：
     </span>
     <br/>
     <span style="color:#f33b45;">
      目的
     </span>
     ：深入掌握linux系统。
     <br/>
     <span style="color:#f33b45;">
      方法
     </span>
     ：使用linux—〉linxu系统编程开发—〉驱动开发和分析linux内核。先看《深》，那主讲原理。看几遍后，看《情景分析》，对照深看，两本交叉，深是纲，情是目。剖析则是0.11版，适合学习。最后深入代码。
     <br/>
     <span style="color:#f33b45;">
      主攻书籍
     </span>
     ：linux内核完全剖析、unix环境高级编程、深入理解linux内核、linux内核源代码情景分析。
    </h3>
    <h3>
     <span style="color:#e579b6;">
      3、学习嵌入式linux：
     </span>
     <br/>
     <span style="color:#f33b45;">
      目的
     </span>
     ：掌握嵌入式处理器其及系统。
     <br/>
     <span style="color:#f33b45;">
      方法
     </span>
     ：（1）嵌入式微处理器结构与应用：直接arm原理及汇编即可，不要重复x86。
     <br/>
     （2）嵌入式操作系统类：ucOS/II简单，开源，可供入门。而后深入研究uClinux。
     <br/>
     （3）必须有块开发板（arm9以上），有条件可参加培训（进步快，能认识些朋友）。
     <br/>
     <span style="color:#f33b45;">
      主攻书籍
     </span>
     ：毛德操的《嵌入式系统》及其他arm9手册与arm汇编指令等。
    </h3>
    <h3>
     <span style="color:#3399ea;">
      4、深入学习：
     </span>
     <br/>
     A、数字图像压缩技术：主要是应掌握MPEG、mp3等编解码算法和技术。
     <br/>
     B、通信协议及编程技术：TCP/IP协议、802.11，Bluetooth，GPRS、GSM、CDMA等。
     <br/>
     C、网络与信息安全技术：如加密技术，数字证书CA等。
     <br/>
     D、DSP技术：Digital Signal Process，DSP处理器通过硬件实现数字信号处理算法。
    </h3>
    <h4>
     说明：太多细节未说明，可根据实际情况调整。重点在于1、3，不必完全按照顺序作。对于学习c++，理由是c++不只是一种语言，一种工具，她还是一种艺术，一种文化，一种哲学理念、但不是拿来炫耀得东西。对于linux内核，学习编程，读一些优秀代码也是有必要的。
    </h4>
    <h4>
     注意：要学会举一反多，有强大的基础，很多东西简单看看就能会。想成为合格的程序员，前提是必须熟练至少一种编程语言，并具有良好的逻辑思维。一定要理论结合实践。不要一味钻研技术，虽然挤出时间是很难做到的，但还是要留点余地去完善其他的爱好，比如宇宙，管理，心理学，音乐、游戏、科幻电影。还有一些不愿意做但必须要做的！技术是通过编程编程再编程编出来的。永远不要梦想一步登天，不要做浮躁的人，不要觉得路途漫长。而是要编程编程再编程，完了再编程，再编程！等机会来了在创业（不要相信有奇迹发生，盲目创业很难成功，即便成功了发展空间也不一定很大）。
    </h4>
    <h4>
     <span style="color:#f33b45;">
      嵌入式书籍推荐：
     </span>
     <br/>
     <span style="color:#ffbb66;">
      Linux基础
     </span>
     <br/>
     1、《Linux与Unix Shell 编程指南》
     <br/>
     <span style="color:#86ca5e;">
      C语言基础
     </span>
     <br/>
     1、《C Primer Plus，5th Edition》【美】Stephen Prata                    2、《The C Programming Language, 2ndEdition》【美】Brian W. Kernighan DavidM. Rithie（K &amp; R）3、《Advanced Programming in the UNIX Environment，2nd Edition》
     <br/>
     4、《嵌入式Linux应用程序开发详解》
     <br/>
     <span style="color:#e579b6;">
      Linux内核
     </span>
     <br/>
     1、《深入理解Linux内核》（第三版）     2、《Linux内核源代码情景分析》毛德操 胡希明
     <br/>
     <span style="color:#3399ea;">
      研发方向
     </span>
     <br/>
     1、《UNIX Network Programming》2、《TCP/IP详解》3、《Linux内核编程》4、《Linux设备驱动开发》（LDD）
     <br/>
     <span style="color:#7c79e5;">
      硬件基础
     </span>
     <br/>
     1、《ARM体系结构与编程》杜春雷    2、S3C2410 Datasheet
     <br/>
     <span style="color:#f33b45;">
      系统教程
     </span>
     <br/>
     1、《嵌入式系统――体系结构、编程与设计》                 2、《嵌入式系统――采用公开源代码和StrongARM/Xscale处理器》毛德操 胡希明        3、《Building Embedded Linux Systems》
     <br/>
     <span style="color:#ffbb66;">
      理论基础
     </span>
     <br/>
     1、《算法导论》2、《数据结构（C语言版）》3、《计算机组织与体系结构?性能分析》4、《深入理解计算机系统》【美】Randal E.Bryant David O”Hallaron着5、《操作系统：精髓与设计原理》6、《编译原理》7、《数据通信与计算机网络》8、《数据压缩原理与应用》
    </h4>
    <h4>
     <span style="color:#86ca5e;">
      C语言书籍推荐
     </span>
     <br/>
     1. The C programming language 《Ｃ程序设计语言》2. Pointers on C 《Ｃ和指针》3. C traps and pitfalls 《Ｃ陷阱与缺陷》4. Expert C Lanuage 《专家Ｃ编程》5. Writing Clean Code—–Microsoft Techiniques for DevelopingBug-free C Programs
     <br/>
     《编程精粹–Microsoft 编写优质无错Ｃ程序秘诀》6. Programming Embedded Systems in Cand C++《嵌入式系统编程》7.《C语言嵌入式系统编程修炼》8.《高质量C++/C编程指南》林锐
    </h4>
    <p>
     <span style="color:#f33b45;">
      尽可能多的编码，要学好C，不能只注重C本身。算法，架构方式等都很重要
     </span>
     。
    </p>
    <h2>
     具体的实施路线：
    </h2>
    <h3>
     <span style="color:#f33b45;">
      1、打好嵌入式软件编程的基础
     </span>
    </h3>
    <h4>
     这一阶段重点打好嵌入式软件编程的基础，包括
     <br/>
     * 学习Linux系统的基本应用
     <br/>
     * Linux的常用命令
     <br/>
     * C语言编程基础
     <br/>
     * 常用的数据结构
    </h4>
    <h4>
     特别是C语言中对指针的理解和应用。这一阶段的主要目的是学习编程语言、开发环境、和培养自己的编程思维，为进一步学习嵌入式开发打下良好的基础。
    </h4>
    <h3>
     <span style="color:#ffbb66;">
      2、学习ARM体系结构编程
     </span>
    </h3>
    <h4>
     这一阶段才是真正的嵌入式编程，首先我们要选择一款嵌入式CPU和一款嵌入式开发平台，目前ARM 嵌入式CPU应用最广泛，这一阶段重点是学习嵌入式CPU的裸机编程：
    </h4>
    <h4>
     * 熟悉中断
     <br/>
     * 定时器
     <br/>
     * 串口
     <br/>
     * NAND FLASH
     <br/>
     * 网络控制器
     <br/>
     * LCD屏、触摸屏
     <br/>
     * I2C 设备
     <br/>
     * SPI 设备
    </h4>
    <h4>
     等常用嵌入式外围设备的硬件工作原理，以及如何使用C语言来编程、控制这些硬件。这一阶段除了要学习对硬件编程之外，还需要学习嵌入式硬件的知识，但是对于嵌入式软件工程师来讲，重点是学习硬件的工作原理，在掌握硬件工作原理的基础上，对硬件进行编程控制。这和硬件工程师学习的侧重点有所不同。这一阶段对应的学校的课程主要包括模拟电路、数字电路、微机原理和单片机。这一阶段重点是要看CPU的芯片手册，大部分的参考书也是对芯片手册的翻译。
    </h4>
    <h3>
     <span style="color:#86ca5e;">
      3、学习嵌入式系统的构建
     </span>
    </h3>
    <h4>
     这一阶段主要学习带有操作系统的嵌入式系统的构建，包括系统的启动流程
     <br/>
     * Bootloader的工作流程
     <br/>
     * UBOOT的编译
     <br/>
     * 裁剪与移植，嵌入式Linux内核的裁剪、移植与编译
     <br/>
     * 嵌入式根文件系统的定制
     <br/>
     * BootLoader、内核和根文件系统的烧写
     <br/>
     这一阶段的主要目的是掌握带有操作系统的嵌入式系统的构建和烧写过程，以及对嵌入式系统软件的总体构成有个整体认识，为我们接下来学习嵌入式应用和驱动开发打下坚实的基础。
    </h4>
    <h3>
     <span style="color:#e579b6;">
      4、学习嵌入式Linux应用程序开发
     </span>
    </h3>
    <h4>
     这一阶段主要学习上层的嵌入式Linux应用程序开发，包括
     <br/>
     * 基于Linux多进程
     <br/>
     * 多线程网络
     <br/>
     * 文件与目录
     <br/>
     掌握嵌入式Linux环境下应用程序开发技术。
    </h4>
    <h3>
     <span style="color:#3399ea;">
      5、学习嵌入式Linux驱动程序开发
     </span>
    </h3>
    <h4>
     <br/>
     这一阶段主要学习底层嵌入式Linux设备驱动程序开发设计，包括
     <br/>
     * 常用的字符设备驱动
     <br/>
     * 块设备驱动
     <br/>
     * LCD设备驱动
     <br/>
     * 触摸屏设备驱动
     <br/>
     以及驱动程序开发中的核心技术
    </h4>
    <h3>
     <span style="color:#7c79e5;">
      6、完成一个综合项目
     </span>
    </h3>
    <h4>
     <br/>
     嵌入式技术关键在于理论和实践的结合，要能够学以致用，完成了以上的所有阶段的知识点学习后，到底有没有学会，会不会用，能不能应用所学知识来解决实际开发中的问题，我们需要来完成一个综合的嵌入式实训项目，例如： 基于嵌入式Linux平台实现的飞行器、 基于嵌入式Linux平台实现的智能机器人等，这些项目都综合应用了嵌入式开发当中的应用，驱动和QT开发技术。
    </h4>
