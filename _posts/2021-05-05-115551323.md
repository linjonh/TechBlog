    <h2>
     <a id="SDNNFV_4">
     </a>
     SDN与NFV概述
    </h2>
    <p>
     <font size="5">
      <strong>
       1、计算机时代的演进
      </strong>
      <br/>
      <font size="4">
       <strong>
        1.1大型机
       </strong>
       <br/>
       专门的硬件 专门的操作系统 专门的应用。（稳定性能好，但封闭）
       <br/>
       <strong>
        1.2小型机
       </strong>
       <br/>
       大型机的小规模版本。（稳定性能好，但封闭）
       <br/>
       <strong>
        1.3兼容机
       </strong>
       <br/>
       开放多元的硬件、系统、应用 （开源、开放）；使用层层独立，通用标准（接口）相互连通。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       2、经典IP网络-分布式网络
      </strong>
      <br/>
      <font size="4">
       特点：各网络设备存在独立的数据、控制和管理平面。
       <br/>
       优点：设备与协议相互依赖，兼容性较好且网络可靠性较高。
       <br/>
       缺陷：网络易拥塞、网络技术复杂、运维太困难、业务部署太慢。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       3、SDN：软件定义网络
      </strong>
      <br/>
      <font size="4">
       理念：将网络控制平面与数据平面分离。
       <br/>
       特征：转控分离、集中控制、开放可编程接口。
       <br/>
       价值：集中管理，简单化网络运维；屏蔽技术细节，降低网络复杂度，降低运维成本； 自动化调优，提高网络利用率；快速业务部署，缩短业务上线时间；网络开放，支撑开放可编程的第三方应用。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       4、OpenFlow
      </strong>
      <br/>
      <font size="4">
       控制器与交换机的一种南向接口协议。（即控制器往交换机方向），其通过Flow Table（流表）转发报文。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       5、报文类型
      </strong>
      <br/>
      <font size="4">
       Controller-to-Switch（由控制器主动发送）：用于管理和查询交换机信息。
       <br/>
       Asynchronous（由交换机发送）：当交换机的状态发生改变时，向控制器发送变化信息。
       <br/>
       Symmetric（无固定发起方）：例如Hello包、Echo包、Error信息等。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       6、Flow Table（流表）
      </strong>
      <br/>
      <font size="4">
       每个流表项目，由匹配字段、优先级、计数器、指令、超时、Cookie、Flags这七部分组成。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       7、转发的关键
      </strong>
      <br/>
      <font size="4">
       匹配字段：是匹配规则，支持自定义。
       <br/>
       指令：是用来描述匹配后的处理方式。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       8、经典路由协议与OpenFlow协议的转发方式区别
      </strong>
      <br/>
      <font size="4">
       <strong>
        8.1经典路由协议
       </strong>
       <br/>
       基于路由表转发，其条目由网络设备间运行的路由协议计算而成；此外，路由表是定长的，通过最长匹配原则执行报文转发；一台网络设备只有一张路由表。
       <br/>
       <strong>
        8.2 OpenFlow协议
       </strong>
       <br/>
       基于流表转发，其一般由OpenFlow控制器统一计算再统一下发至交换机；同时，流表是可变长的，有丰富的匹配转发规则；一台网络设备可以拥有多张流表。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       9、NFV：网络功能虚拟化(设备构架层)
      </strong>
      <br/>
      <font size="4">
       缩短业务上线时间、降低建网成本、提升网络运维效率、构建开放的生态系统。
      </font>
     </font>
    </p>
    <hr color="#000000" size='2"'/>
    <p>
     <font size="5">
      <strong>
       10、关键技术
      </strong>
      <br/>
      <font size="4">
       虚拟化：有着分区、隔离、封装、相对硬件独立的特性。
       <br/>
       云化：云计算是一种模型，它可以实现随时随地，便捷地，随需应变地 从可配置计算资源共享池中获取所需的资源（例如，网络、服务器、存储、应用、及服务），资源能够快速 供应并释放，使管理资源的工作量和与服务提供商的交互减小到最低限度。
      </font>
     </font>
    </p>
