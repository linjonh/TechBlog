    <p>
     C/S（client/server ）客户服务器模式   B/S（browser/server）浏览器服务器模式
    </p>
    <p>
     看到很多
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        面试
       </strong>
      </span>
     </a>
     公司要求了解C/S和B/S，今天看了一篇这样的文章，我学到了一些知识与大家一起分享，也希望大家多提宝贵建议，总结就有进步！
    </p>
    <p>
     <strong>
      C/S和B/S简介：
     </strong>
    </p>
    <p>
     C/S模式的服务器通常采用PC或
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        工作
       </strong>
      </span>
     </a>
     站，小型机等，需要大型的
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        数据库
       </strong>
      </span>
     </a>
     系统例如
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        SQL
       </strong>
      </span>
     </a>
     Server，Oracle，Sybase，
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        MySQL
       </strong>
      </span>
     </a>
     等，客户端需要安装专用的客户端软件；
    </p>
    <p>
     B/S模式的客户机需要安装一个浏览器，如
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        微软
       </strong>
      </span>
     </a>
     的Internet Explorer,苹果的Safari，开源Firefox等，服务器要安装数据库，浏览器通过Web Server与服务器的数据库进行数据交互。
    </p>
    <p>
     <strong>
      优缺点：
     </strong>
    </p>
    <p>
     C/S的优点：充分的发挥了PC机的处理能力，很多工作都可以在客户端处理后在传给服务器端，客户端响应速度快；缺点：只限用于局域网，客户端需要安装专用的客户端软件，对客户端的
     <a href="http://www.51testing.com/" rel="nofollow">
      <span style="text-decoration:underline">
       <strong>
        操作系统
       </strong>
      </span>
     </a>
     一般也有限制。
    </p>
    <p>
     B/S的优点：是可以在任何地方进行操作，不需要安装任何的专门软件，有一台能上网的计算机就可以了，客户端零维护。系统管理非常容易。缺点：就是C/S的优点它不能实现。
    </p>
