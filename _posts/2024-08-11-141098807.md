     项目简介
    </h2>
    <p>
     <strong>
      最近有一个爆火的Github项目叫做 Deep-Live-Cam
     </strong>
     。它可以用 AI 技术在直播的时候，实时生成虚拟人脸。它的作用是让你在直播的时候，保护你的隐私，同时也可以让你看起来很酷(By 项目原作者)。这个项目特别适合那些喜欢直播又想保护自己隐私的人使用。
    </p>
    <p>
     项目的代码托管在
     <a href="https://github.com/hacksider/Deep-Live-Cam">
      GitHub 上
     </a>

      <a href="https://blog.csdn.net/zxn0803">

    <h2>
     <a id="Deep_Live_Cam__20">
     </a>
     Deep Live Cam 部署教程
    </h2>
    <h3>
     <a id="CPU_23">
     </a>
     CPU版本如何安装？
    </h3>
    <h4>
     <a id="_25">
     </a>
     基本安装
    </h4>
    <p>
     这可能会在你的计算机正常工作，但速度可能会很慢。你可以按照基本安装的说明进行操作（这通常通过
     <strong>
      CPU
     </strong>
     运行）
    </p>
    <h5>
     <a id="1__28">
     </a>
     1. 设置你的平台
    </h5>
    <ul>
     <li>
      python（推荐使用 3.10 版本）
     </li>
     <li>
      pip
     </li>
     <li>
      git
     </li>
     <li>
      <a href="https://www.youtube.com/watch?v=OlNWCpFdVMA" rel="nofollow">
       ffmpeg
      </a>
     </li>
     <li>
      <a href="https://visualstudio.microsoft.com/visual-cpp-build-tools/" rel="nofollow">
       Visual Studio 2022 (Windows）
      </a>
     </li>
    </ul>
    <h5>
     <a id="2__35">
     </a>
     2. 克隆代码库
    </h5>
    <pre><code>https://github.com/hacksider/Deep-Live-Cam.git
</code></pre>
    <h5>
     <a id="3__38">
     </a>
     3. 下载模型
    </h5>
    <ol>
     <li>
      <a href="https://huggingface.co/hacksider/deep-live-cam/resolve/main/GFPGANv1.4.pth" rel="nofollow">
       GFPGANv1.4
      </a>
     </li>
     <li>
      <a href="https://huggingface.co/hacksider/deep-live-cam/resolve/main/inswapper_128_fp16.onnx" rel="nofollow">
       inswapper_128_fp16.onnx
      </a>
     </li>
    </ol>
    <p>
     然后将这两个文件放到 “
     <strong>
      models
     </strong>
     ” 文件夹中
    </p>
    <h5>
     <a id="4__45">
     </a>
     4. 安装依赖
    </h5>
    <p>
     我们强烈推荐使用
     <code>
      venv
     </code>
     来避免问题。
    </p>
    <pre><code>pip install -r requirements.txt
</code></pre>
    <p>
     完成！！！如果没有 GPU，你应该能够使用
     <code>
      python run.py
     </code>
     命令运行 roop。请注意，在第一次运行程序时，它会下载一些模型，这可能会根据你的网络连接速度需要一些时间。
    </p>
    <h3>
     <a id="_GPU__53">
     </a>
     继续使用 GPU 加速
    </h3>
    <ol>
     <li>
      <p>
       安装
       <a href="https://developer.nvidia.com/cuda-11-8-0-download-archive" rel="nofollow">
        CUDA Toolkit 11.8
       </a>
      </p>
     </li>
     <li>
      <p>
       安装依赖：
      </p>
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-gpu
pip install onnxruntime-gpu==1.16.3
</code></pre>
    <ol start="3">
     <li>
      使用以下命令来验证：
     </li>
    </ol>
    <pre><code>python run.py --execution-provider cuda
</code></pre>
    <h4>
     <a id="CoreML_Apple_Siliconhttpsgithubcoms0md3vroopwiki2Accelerationcoremlexecutionproviderapplesilicon_70">
     </a>
     <a href="https://github.com/s0md3v/roop/wiki/2.-Acceleration#coreml-execution-provider-apple-silicon">
      CoreML （Apple Silicon）
     </a>
    </h4>
    <ol>
     <li>
      安装依赖：
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-silicon
pip install onnxruntime-silicon==1.13.1
</code></pre>
    <ol start="2">
     <li>
      使用以下命令验证：
     </li>
    </ol>
    <pre><code>python run.py --execution-provider coreml
</code></pre>
    <h4>
     <a id="CoreMLApple_Legacyhttpsgithubcoms0md3vroopwiki2Accelerationcoremlexecutionproviderapplelegacy_85">
     </a>
     <a href="https://github.com/s0md3v/roop/wiki/2.-Acceleration#coreml-execution-provider-apple-legacy">
      CoreML（Apple Legacy）
     </a>
    </h4>
    <ol>
     <li>
      安装依赖：
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-coreml
pip install onnxruntime-coreml==1.13.1
</code></pre>
    <ol start="2">
     <li>
      使用以下命令验证：
     </li>
    </ol>
    <pre><code>python run.py --execution-provider coreml
</code></pre>
    <h4>
     <a id="DirectMLWindowshttpsgithubcoms0md3vroopwiki2Accelerationdirectmlexecutionproviderwindows_100">
     </a>
     <a href="https://github.com/s0md3v/roop/wiki/2.-Acceleration#directml-execution-provider-windows">
      DirectML（Windows）

    <ol>
     <li>
      安装依赖：
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-directml
pip install onnxruntime-directml==1.15.1
</code></pre>
    <ol start="2">
     <li>
      使用以下命令验证：
     </li>
    </ol>
    <pre><code>python run.py --execution-provider directml
</code></pre>
    <h4>
     <a id="OpenVINOIntelhttpsgithubcoms0md3vroopwiki2Accelerationopenvinoexecutionproviderintel_115">
     </a>
     <a href="https://github.com/s0md3v/roop/wiki/2.-Acceleration#openvino-execution-provider-intel">
      OpenVINO™（Intel)
     </a>
    </h4>
    <ol>
     <li>
      安装依赖：
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-openvino
pip install onnxruntime-openvino==1.15.0

    <ol start="2">
     <li>

     </li>
    </ol>

    <h3>
     <a id="_130">
     </a>
     如何使用？
    </h3>
    <blockquote>
     <p>
      注意：当你第一次运行这个程序时，它会下载一些约 300MB 大小的模型。
     </p>
    </blockquote>
    <p>
     执行
     <code>
      python run.py
     </code>
     命令将启动如下窗口：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/24b6c59630ee4f088300867ad5b66db3.png"/>
    </p>
    <p>
     选择一个面孔（带有所需面孔的图像）和目标图像/视频（你想替换面孔的图像/视频），然后点击
     <code>
      Start
     </code>
     。打开文件资源管理器并导航到你选择的输出目录。你会找到一个名为
     <code>
      &lt;video_title&gt;
     </code>
     的目录，你可以在其中实时查看帧交换。处理完成后，它会创建输出文件。这样就结束了。
    </p>
    <h3>
     <a id="_139">
     </a>
     对于摄像头模式
    </h3>
    <p>
     只需按照屏幕截图上的点击操作
    </p>
    <ol>
     <li>
      选择一个面孔
     </li>
     <li>
      点击 “live”
     </li>
     <li>
      等待几秒钟（通常需要 10 到 30 秒，预览才会显示）
     </li>
    </ol>
    <p>
     <a href="https://github.com/hacksider/Deep-Live-Cam/blob/main/demo.gif">
      具体视频可以看这里
     </a>
    </p>
    <p>
     只需使用你喜欢的屏幕录制工具进行流式传输，如 OBS
    </p>
    <blockquote>
     <p>
      注意：如果你想更换面孔，只需选择另一张图片，预览模式将会重新启动（因此请稍等片刻）。
     </p>
    </blockquote>
    <p>
     以下是额外的命令行参数。要了解它们的作用，请查看
     <a href="https://github.com/s0md3v/roop/wiki/Advanced-Options">
      此指南
     </a>
     。
    </p>
    <pre><code>options:
  -h, --help                                               显示此帮助信息并退出
  -s SOURCE_PATH, --source SOURCE_PATH                     选择源图像
  -t TARGET_PATH, --target TARGET_PATH                     选择目标图像或视频
  -o OUTPUT_PATH, --output OUTPUT_PATH                     选择输出文件或目录
  --frame-processor FRAME_PROCESSOR [FRAME_PROCESSOR ...]  帧处理器（选择：face_swapper, face_enhancer, ...）
  --keep-fps                                               保持原始帧率
  --keep-audio                                             保持原始音频
  --keep-frames                                            保持临时帧
  --many-faces                                             处理每个面孔
  --video-encoder {libx264,libx265,libvpx-vp9}             调整输出视频编码器
  --video-quality [0-51]                                   调整输出视频质量
  --max-memory MAX_MEMORY                                  最大内存量（GB）
  --execution-provider {cpu} [{cpu} ...]                   可用执行提供者（选择：cpu, ...）
  --execution-threads EXECUTION_THREADS                    执行线程数量
  -v, --version                                            显示程序版本号并退出
</code></pre>
    <p>
     如果想要 CLI 模式：使用 -s/–source 参数将使程序以 CLI 模式运行。
    </p>

    <h2>
     <a id="_176">
     </a>
     一些可能的踩坑:
    </h2>
    <ol>
     <li>
      脸部出现黑色方块，解决方案:
     </li>
    </ol>
    <pre><code>pip uninstall onnxruntime onnxruntime-gpu
pip install onnxruntime-gpu==1.16.3
</code></pre>
    <ol start="2">
     <li>
      运行
      <code>
       python run.py
      </code>
      黑框卡住，可能是网络设置的问题，因为一些众所周知的原因可能需要Proxy。
     </li>
     <li>
      如果是windows系统，项目整合了setup的bat文件，可以尝试用bat文件进行一键安装。
     </li>
    </ol>
