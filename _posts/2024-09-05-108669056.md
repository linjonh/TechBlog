    <h1 class="title-article" id="articleContentId">
     在HarmonyOS工程中添加Module
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     Module是HarmonyOS应用的基本功能单元，包含了源代码、资源文件、第三方库及应用清单文件，每一个Module都可以独立进行编译和运行。一个HarmonyOS应用通常会包含一个或多个Module，因此，可以在工程中，创建多个Module，每个Module分为Ability和Library（HarmonyOS Library和Java Library）两种类型。
    </p>
    <p>
     如上篇HarmonyOS工程介绍，在一个APP中，对于同一类型设备有且只有一个Entry Module，其余Module的类型均为Feature。因此，在创建一个类型为Ability的Module时，遵循如下原则：
    </p>
    <ul>
     <li>
      若新增Module的设备类型为已有设备时，则Module的类型将自动设置为“Feature”。
     </li>
     <li>
      若新增Module的设备类型为当前还没有创建Module，则Module的类型将自动设置为“Entry”。
     </li>
    </ul>
    <h3>
     <strong>
      新增Module
     </strong>
    </h3>
    <ol>
     <li>
      通过如下两种方法，在工程中添加新的Module。
      <ul>
       <li>
        方法1：鼠标移到工程目录顶部，点击鼠标右键，选择
        <strong>
         New&gt;Module
        </strong>
        ，开始创建新的Module。
       </li>
       <li>
        方法2：在菜单栏选择
        <strong>
         File &gt; New &gt; Module
        </strong>
        ，开始创建新的Module。
       </li>
      </ul>
     </li>
     <li>
      在New Project Module界面中，选择Module对应的设备类型和模板。
     </li>
    </ol>
    <p>
    </p>
    <p>
     <a href="https://communityfile-drcn.op.hicloud.com/FileServer/getFile/cmtyPub/011/111/111/0000000000011111111.20200916170403.48795573433559099717368003843850:50510916092912:2800:F473D4C47590E881B5B54E35A2D6F6F54BCF8A250A237F196E264ED157E9DF3C.png?needInitFileName=true?needInitFileName=true" rel="nofollow">

     </a>
    </p>
    <ol>
     <li>
      点击
      <strong>
       Next
      </strong>
      ，在Module配置页面，设置新增Module的基本信息。
      <ul>
       <li>
        Module类型为Ability或者HarmonyOS Library时，请根据如下内容进行设置，然后点击
        <strong>
         Next
        </strong>
        。
        <ul>
         <li>
          <strong>
           Application/Library name
          </strong>
          ：新增Module所属的类名称。
         </li>
         <li>
          <strong>
           Module name
          </strong>
          ：新增模块的名称。
         </li>
         <li>
          <strong>
           Module Type
          </strong>
          ：仅Module类型为Ability时存在，工具自动根据设备类型下的模块进行设置。
         </li>
         <li>
          <strong>
           Package name
          </strong>
          ：软件包名称，可以点击
          <strong>
           Edit
          </strong>
          修改默认包名称，需全局唯一。
         </li>
         <li>
          <strong>
           Compatible SDK
          </strong>
          ：兼容的SDK版本。
         </li>
        </ul>
       </li>
       <li>
        Module类型为Java Library时，请根据如下内容进行设置，然后点击
        <strong>
         Finish
        </strong>
        完成创建。
        <ul>
         <li>
          <strong>
           Library Name
          </strong>
          ：Java Library类名称。
         </li>
         <li>
          <strong>
           Java package name
          </strong>
          ：软件包名称，可以点击
          <strong>
           Edit
          </strong>
          修改默认包名称，需全局唯一。
         </li>
         <li>
          <strong>
           Java class name
          </strong>
          ：class文件名称。
         </li>
         <li>
          <strong>
           Create.gitignore file
          </strong>
          ：是否自动创建.gitignore文件，勾选表示创建。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      设置新增Ability或HarmonyOS Library的Page Name。
     </li>
    </ol>
    <p>
     若该Module类型为Ability，需要设置Visible参数，表示该Ability是否可以被其它应用所调用。
    </p>
    <ul>
     <li>
      勾选（true）：可以被其它应用调用。
     </li>
     <li>
      不勾选（false）：不能被其它应用调用。
     </li>
    </ul>
    <ol>
     <li>
      点击
      <strong>
       Finish
      </strong>
      ，等待创建完成后，可以在工程目录中查看和编辑新增的Module。
     </li>
    </ol>
    <h3>
     <strong>
      删除Module
     </strong>
    </h3>
    <p>
     为防止开发者在删除Module的过程中，误将其它的模块删除，DevEco Studio提供统一的模块管理功能，需要先在模块管理中，移除对应的模块后，才允许删除。
    </p>
    <ol>
     <li>
      在菜单栏中选择
      <strong>
       File &gt; Project Structure &gt; Modules
      </strong>
      ，选择需要删除的Module，点击
     </li>
    </ol>

      <strong>
       Delete
      </strong>
      ，并在弹出的对话框中点击
      <strong>
       Delete
      </strong>
      。
