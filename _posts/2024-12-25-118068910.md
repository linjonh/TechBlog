    <h3>
     <a id="1_4">
     </a>
     1、什么是前端？
    </h3>
    <p>
     前端（Front-end）指的是
     <strong>
      用户可见的界面
     </strong>
     。
    </p>
    <ul>
     <li>
      网站前端页面也就是网页的页面开发，比如网页上的特效、布局、图片、视频，音频等内容。
     </li>
     <li>
      <strong>
       前端的工作内容
      </strong>
      ：
      <ol>
       <li>
        将美工设计的效果图设计成浏览器可以运行的网页
       </li>
       <li>
        并配合后端做网页的数据显示和交互等可视方面的工作内容。
       </li>
      </ol>
     </li>
    </ul>
    <p>
     <strong>
      前端开发
     </strong>
     ：创建WEB页面或APP等前端界面呈现给用户的过程。
    </p>
    <ul>
     <li>
      通过HTML，CSS及JavaScript以及衍生出来的各种技术、框架、解决方案，来
      <strong>
       实现互联网产品的用户界面交互
      </strong>
      。
     </li>
    </ul>
    <br/>
    <h3>
     <a id="2_17">
     </a>
     2、为什么需要前端？
    </h3>
    <p>
     即前端的意义是什么。
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3841230c4edab9e1a7bfc1e3310b7aa6.png"/>
     </p>
     <ul>
      <li>
       1990 年，Tim 以超文本语言 HTML 为基础在 NeXT 电脑上发明了最原始的 Web 浏览器。
      </li>
      <li>
       1994 万维网联盟（World Wide Web Consortium）成立，简称 W3C，标志着万维网的正式诞生。
      </li>
      <li>
       1995 网景推出 JavaScript，可以用来更改前端 DOM 的样式。
       <br/>
       <br/>
      </li>
     </ul>
     <h4>
      <a id="_26">
      </a>
      前端的发展简史：
     </h4>
     <ul>
      <li>
       <p>
        <strong>
         Web 1.0 时代
        </strong>
        ：早期的网页以 HTML 为主，是
        <strong>
         纯静态的网页
        </strong>
        ，网页是“只读”的，信息流只能通过服务器到客户端单向流通 。
       </p>
      </li>
      <li>
       <p>
        <strong>
         动态服务器页面技术
        </strong>
        的出现：页面不再是静止的，页面可以获取服务器数据信息并不断更新。
       </p>
       <ul>
        <li>
         PHP、JSP、ASP.NET
        </li>
        <li>
         但前端页面要想获取后台信息，需要刷新整个页面
         <br/>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         AJAX 技术
        </strong>
        的出现：不需要刷新页面就可以使得前端与服务器进行网络通信。
       </p>
      </li>
      <li>
       <p>
        <strong>
         前端Framework
        </strong>
        的出现：为解决浏览器兼容性问题。
       </p>
       <ul>
        <li>
         Dojo、
         <strong>
          jQuery
         </strong>
         、YUI、ExtJS、MooTools
        </li>
        <li>
         前端开发人员用这些 Framework 频繁发送 AJAX 请求到后台，在得到数据后，再用这些 Framework 更新 DOM 树。
         <br/>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         Node.js
        </strong>
        的出现：基于事件循环的异步 I/O 框架，拓展了 JavaScript 在服务器端的无限可能。
       </p>
      </li>
      <li>
       <p>
        <strong>
         移动Web和Hybrid App
        </strong>
        的出现。
       </p>
      </li>
     </ul>
    </blockquote>
    <ul>
     <li>
      前端发展的
      <strong>
       主要话题
      </strong>
      ：一开始是「
      <strong>
       特效
      </strong>
      」，后来是「
      <strong>
       用户体验
      </strong>
      」，然后是「
      <strong>
       前后端分离
      </strong>
      」，再后来是「
      <strong>
       页面性能
      </strong>
      」，然后到最近的「
      <strong>
       模块化、工程化
      </strong>
      」，到现在的「以数据为中心」。
     </li>
    </ul>
    <p>
     因此个人认为，前端的意义主要还是在于：
    </p>
    <ol>
     <li>
      提高产品开发效率
     </li>
     <li>
      优化产品质量
     </li>
     <li>
      提升用户使用体验
     </li>
    </ol>
    <br/>
    <h3>
     <a id="3_49">
     </a>
     3、前端开发的类型
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/703506364f8176cc86d5f73b8d0df7a7.png"/>
    </p>
    <ul>
     <li>
      <h4>
       <a id="Web_54">
       </a>
       Web前端开发：
      </h4>
      <ul>
       <li>
        <strong>
         PC端开发
        </strong>
        ：主要分为面向大众的各类网站（如新闻媒体、社交、电商、论坛等）和面向管理员的各种 CMS (内容管理系统)和其它的后台管理系统。
       </li>
       <li>
        <strong>
         移动端开发
        </strong>
        ：
        <ul>
         <li>
          <strong>
           Web App (HTML5）开发
          </strong>
          ：利用Web技术进行的App开发，可以在手机端浏览器里面打开的网站就称之为webapp。
          <ul>
           <li>
            Web技术本身需要浏览器的支持才能进行展示和用户交互，因此主要用到的技术是HTML、CSS、Javascript以及jQuery、Vue、React等JS框架。
           </li>
          </ul>
         </li>
         <li>
          <strong>
           混合（Hybrid App）开发
          </strong>
          ：开发一款App产品的时候，利用原生与H5的开发技术的混合应用。
          <ul>
           <li>
            通常由“HTML5云网站+APP应用客户端”两部份构成。
           </li>
           <li>
            原生代码部分利用WebView插件或者其它框架为H5提供容器，程序主要的业务实现、界面展示都是利用与H5相关的Web技术进行实现的。
           </li>
           <li>
            比如京东、淘宝、今日头条等APP都是利用混合开发模式而成的。
           </li>
          </ul>
         </li>
         <li>
          <strong>
           微信公众号开发
          </strong>
         </li>
         <li>
          <strong>
           小程序开发
          </strong>
         </li>
         <li>
          <strong>
           小游戏开发
          </strong>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <br/>
    <ul>
     <li>
      <h4>
       <a id="APP_69">
       </a>
       客户端（APP）开发：
      </h4>
      原生开发（Native App开发），是在Android、IOS等移动平台上利用官方提供的开发语言、开发类库、开发工具进行App开发。
      <ul>
       <li>
        比如Android是利用Java、Eclipse、Android studio
       </li>
       <li>
        IOS是利用Objective-C 和Xcode进行开发
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <ul>
      <li>
       关于Web App、原生App、混合App的概念可参考：https://www.cnblogs.com/runhua/p/9618552.html
      </li>
      <li>
       关于三者的区别可参考：https://zhuanlan.zhihu.com/p/162865070
       <br/>
       以及https://zhuanlan.zhihu.com/p/71192935
      </li>
      <li>
       关于移动端和PC端的区别，可参考：https://www.zhihu.com/question/35633784?sort=created
       <br/>
       以及https://www.cnblogs.com/zhuzhenwei918/p/6790859.html
      </li>
     </ul>
    </blockquote>

    <h3>
     <a id="4_83">
     </a>
     4、前端与后端的交互
    </h3>
    <p>
     主要有两种方式：Ajax 和 Socket 连接。
    </p>
    <ul>
     <li>
      <h4>
       <a id="Ajax_86">
       </a>
       Ajax：
      </h4>
      常见的前后端交互方式。
      <ul>
       <li>
        以 “请求→响应” 的方式来完成前后端的信息传递。
       </li>
       <li>
        传统的表单交互需求几乎都可以由 Ajax 改造为 “原地提交并获取反馈” 的交互方式，
        <strong>
         不需要跳转页面，从而有效提升用户体验
        </strong>
        。
       </li>
      </ul>
     </li>
    </ul>
    <br/>
    <ul>
     <li>
      <h4>
       <a id="Socket_93">
       </a>
       Socket：
      </h4>
      对于
      <strong>
       实时性
      </strong>
      比较高的场景，Socket 连接是一个更好的选择。
      <ul>
       <li>
        工作方式：前后端建立一个持续的连接，信息可以随时由前端发向后端，或由后端推送到前端。
       </li>
       <li>
        如果要建立一个实时对话的应用，通常就会用到 Socket 连接。
       </li>
      </ul>
     </li>
    </ul>

    <blockquote>

     </p>

      </li>

      </li>
     </ul>
    </blockquote>
    <h3>
     <a id="11_HTML_114">
     </a>
     1.1 HTML
    </h3>
    <ul>
     <li>
      <h4>
       <a id="HTML_115">
       </a>
       什么是HTML？
      </h4>
      <p>
       HTML（Hyper Text Markup Language，超文本标记语言 ）是用来
       <strong>
        描述网页的一种语言
       </strong>
       。
      </p>
      <ul>
       <li>
        HTML 不是一种编程语言，而是一种
        <strong>
         标记语言
        </strong>
        (markup language)，标记语言是一套标记标签 (markup tag)。
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="HTML_120">
       </a>
       HTML如何描述网页？
      </h4>
      <p>
       HTML 使用
       <strong>

       </strong>
       （HTML tag）来描述网页：
      </p>

       <li>

       </li>
      </ul>
     </li>
    </ul>

    <ul>
     <li>
      <h4>
       <a id="AJAX_185">
       </a>
       什么是AJAX？
      </h4>
      <p>
       AJAX（Asynchronous JavaScript and XML，异步的 JavaScript 和 XML）是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。
      </p>
      <ul>
       <li>
        AJAX 是一种用于创建
        <strong>
         快速动态网页
        </strong>
        的技术。通过在后台与服务器进行少量数据交换，AJAX 可以
        <strong>
         使网页实现异步更新
        </strong>
        。
       </li>
       <li>
        使用 AJAX 的应用程序案例：新浪微博、Google 地图、开心网等等。
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="AJAX_191">
       </a>
       AJAX的工作原理？
      </h4>
      <p>
       通过XmlHttpRequest对象来向服务器
       <strong>
        发异步请求
       </strong>
       ，从服务器获得数据，然后用javascript来操作DOM从而更新页面。
      </p>
      <ul>
       <li>
        <strong>
         XMLHttpRequest 是 AJAX 的基础
        </strong>
        。
        <br/>
        <br/>
       </li>
      </ul>
      <ol>
       <li>
        <strong>
         创建XMLHttpRequest对象
        </strong>
        （不同浏览器下建立方式可能不同）
       </li>
       <li>
        <strong>
         使用该对象向服务器发送请求
        </strong>
        （使用 XMLHttpRequest 对象的 open() 和 send() 方法）
       </li>
       <li>
        <strong>
         服务器收到请求并返回信息给客户端
        </strong>
        （获取服务器的响应使用 XMLHttpRequest 对象的 responseText 或 responseXML 属性）
       </li>
       <li>
        客户端收到信息后将数据呈现到用户界面上
       </li>
      </ol>
     </li>
    </ul>
    <br/>

     <a id="22_JSON_202">

     2.2 JSON
    </h3>
    <ul>
     <li>
      <h4>
       <a id="JSON_203">
       </a>
       什么是JSON？
      </h4>
      <p>
       JSON（JavaScript Object Notation，JavaScript 对象表示法）是
       <strong>
        轻量级的文本数据交换格式
       </strong>
       。
      </p>
      <ul>
       <li>
        JSON 文件的文件类型是 “.json”
       </li>
       <li>
        JSON 文本的 MIME 类型是 “application/json”
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="JSON_209">
       </a>
       为什么使用JSON？
      </h4>
      <p>
       对于 AJAX 应用程序来说，JSON 比 XML
       <strong>
        更快更易使用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         使用 XML
        </strong>
        ：读取XML文档 --&gt; 使用 XML DOM 来循环遍历文档 --&gt; 读取值并存储在变量中
       </li>
       <li>
        <strong>
         使用JSON
        </strong>
        ：读取 JSON 字符串 --&gt; 用 eval() 处理 JSON 字符串
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="JSON_215">
       </a>
       JSON的特点：
      </h4>
      <ul>
       <li>
        JSON
        <strong>
         使用 JavaScript 语法
        </strong>
        来描述数据对象，但是 JSON 仍然
        <strong>
         独立于语言和平台
        </strong>
        ，JSON 解析器和 JSON 库支持许多不同的编程语言。
       </li>
       <li>
        由于JSON 文本格式在语法上与创建 JavaScript 对象的代码相同，因此
        <strong>
         JavaScript 程序能够使用内建的 eval() 函数，用 JSON 数据来生成原生的 JavaScript 对象
        </strong>
        。
       </li>
       <li>
        <strong>
         类似XML
        </strong>
        ：JSON 是纯文本，具有层级结构，可通过 JavaScript 进行解析，JSON 数据可使用 AJAX 进行传输。
       </li>
       <li>
        <strong>
         与XML不同
        </strong>
        ：没有结束标签，更短，读写的速度更快，能够使用内建的 JavaScript eval() 方法进行解析，使用数组，不使用保留字。
       </li>
      </ul>
     </li>
    </ul>

    <ul>
     <li>

      <strong>

      </strong>

    <br/>
    <ul>
     <li>
      <h4>
       <a id="Nodejs_229">
       </a>
       为什么使用Node.js?
      </h4>
      <ul>
       <li>
        早期 JavaScript 只是一个跑在浏览器环境里的一门脚本语言，主要用于前端页面交互。
       </li>
       <li>
        通过Node.js，Javascript 不只运行于浏览器，还可以运行于服务端，打通了前端与后端。
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="Nodejs__234">
       </a>
       Node.js 的特点
      </h4>
      <ul>
       <li>
        Node.js的定位：提供一种简单安全的方法在 JavaScript 中构建高性能和可扩展的网络应用程序。
       </li>
       <li>
        <strong>
         单线程
        </strong>
        ：Node.js 使用单线程来运行，避免了 CPU 上下文切换和内存中的大量执行堆栈。
       </li>
       <li>
        <strong>
         非阻塞 I/O
        </strong>
        ：得益于 Libuv 强大的异步 I/O，避免了由于需要等待输入或者输出（数据库、文件系统、Web服务器…）响应而造成的 CPU 时间损失。
       </li>
       <li>
        <strong>
         事件驱动编程
        </strong>
        ：这种编程模式是一种高性能的服务模型。
       </li>
       <li>
        <strong>
         跨平台
        </strong>
        ：可运行于 Linux 平台，v0.6.0 版本可以在 Windows 平台运行。
        <br/>
        <br/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="_242">
       </a>
       适用场景
      </h4>
      <ul>
       <li>
        <strong>
         I/O 密集型场景
        </strong>
        ：Node.js 的优势主要在于
        <strong>
         事件循环，非阻塞异步 I/O，只开一个线程
        </strong>
        ，不会每个请求过来都去创建一个线程，从而产生资源开销。
       </li>
       <li>
        <strong>
         ResutFul API
        </strong>
        ：可以使用 Node.js 来做为中间层，负责组装数据提供 API 接口给到前端调用，这些数据源可能来自第三方接口或者数据库，后端则可以更专注于业务开发。
       </li>
       <li>
        <strong>
         RPC 服务
        </strong>
        ：RPC（Remote Procedure Call，远程过程调用），在当今微服务模式下，我们可能是针对功能或者具体的业务形态进行服务化。
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <h4>
      <a id="NodejsAJAX_247">
      </a>
      *Nodejs和AJAX的区别：
     </h4>
     <ul>
      <li>
       Ajax（异步Javascript和XML）是一种
       <strong>
        客户端技术
       </strong>
       ，通常用于更新页面内容而不刷新它。
      </li>
      <li>
       而Node.js是
       <strong>
        服务器端
       </strong>
       Javascript运行环境，用于开发服务器软件。
      </li>
     </ul>
    </blockquote>

    <ul>
     <li>
      <h4>
       <a id="Servlet_301">
       </a>
       什么是Servlet？
      </h4>
      <p>
       Java Servlet 是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的
       <strong>
        中间层
       </strong>
       。
      </p>
      <ul>
       <li>
        是运行在 Web 服务器或应用服务器上的程序。
       </li>
       <li>
        Servlet 在 Web 应用程序中的位置：
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b0b211964d4f4e37d36ee1434ee7fb91.png"/>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <a id="Servlet_306">
       </a>
       为什么使用Servlet？
      </h4>
      <p>
       Java Servlet 通常情况下与使用 CGI（Common Gateway Interface，公共网关接口）实现的程序可以达到异曲同工的效果，但相比CGI，优势如下：
      </p>
      <ol>
       <li>
        性能明显更好。
       </li>
       <li>
        无须再创建一个单独的进程来处理每个客户端请求：因为Servlet 在 Web 服务器的地址空间内执行。
       </li>
       <li>
        Servlet 是独立于平台的：因为它们是用 Java 编写的。
       </li>
       <li>
        Servlet 是可信的：服务器上的 Java 安全管理器执行了一系列限制，以保护服务器计算机上的资源。
       </li>
       <li>
        Java 类库的全部功能对 Servlet 来说都是可用的：它可以通过 sockets 和 RMI 机制与 applets、数据库或其他软件进行交互。
       </li>
      </ol>
     </li>
     <li>
      <h4>
       <a id="Servlet_314">
       </a>
       Servlet做什么？
      </h4>
      <ul>
       <li>
        读取客户端（浏览器）发送的显式的数据。
        <ul>
         <li>
          这包括网页上的 HTML 表单，或者也可以是来自 applet 或自定义的 HTTP 客户端程序的表单。
         </li>
        </ul>
       </li>
       <li>
        读取客户端（浏览器）发送的隐式的 HTTP 请求数据。
        <ul>
         <li>
          这包括 cookies、媒体类型和浏览器能理解的压缩格式等等。
         </li>
        </ul>
       </li>
       <li>
        处理数据并生成结果。
        <ul>
         <li>
          这个过程可能需要访问数据库，执行 RMI 或 CORBA 调用，调用 Web 服务，或者直接计算得出对应的响应。
         </li>
        </ul>
       </li>
       <li>
        发送显式的数据（即文档）到客户端（浏览器）。
        <ul>
         <li>
          该文档的格式可以是多种多样的，包括文本文件（HTML 或 XML）、二进制文件（GIF 图像）、Excel 等。
         </li>
        </ul>
       </li>
       <li>
        发送隐式的 HTTP 响应到客户端（浏览器）。
        <ul>
         <li>
          这包括告诉浏览器或其他客户端被返回的文档类型（例如 HTML），设置 cookies 和缓存参数，以及其他类似的任务。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

    <h3>
     <a id="34__331">
     </a>
     3.4 浏览器
    </h3>
    <ul>
     <li>
      <h4>
       <a id="_332">
       </a>
       浏览器是什么？
      </h4>
      浏览器是用来检索、展示以及传递Web信息资源的
      <strong>
       应用程序
      </strong>
      。
      <ul>
       <li>
        五种主流浏览器：IE、Firefox、Safari、Chrome及Opera。
       </li>
      </ul>
     </li>
    </ul>
    <br/>
    <ul>
     <li>
      <h4>
       <a id="High_Level_Structure_338">
       </a>
       浏览器的主要构成（High Level Structure）：
      </h4>
      <ul>
       <li>
        <strong>
         用户界面
        </strong>
        ：包括地址栏、后退/前进按钮、书签目录等（即你所看到的除了用来显示你所请求页面的主窗口之外的其他部分）。
       </li>
       <li>
        <strong>
         浏览器引擎
        </strong>
        ：用来查询及操作渲染引擎的接口。
       </li>
       <li>
        <strong>
         渲染引擎（The rendering engine）
        </strong>
        ：用来显示请求的内容。
        <ul>
         <li>
          例如，如果请求内容为html，它负责解析html及css，并将解析后的结果显示出来。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         网络
        </strong>
        ：用来完成网络调用。
        <ul>
         <li>
          例如http请求，它具有平台无关的接口，可以在不同平台上工作。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         UI后端
        </strong>
        ：用来绘制类似组合选择框及对话框等基本组件，具有不特定于某个平台的通用接口，底层使用操作系统的用户接口。
       </li>
       <li>
        <strong>
         JS解释器
        </strong>
        ：用来解释执行JS代码。
       </li>
       <li>
        <strong>
         数据存储
        </strong>
        ：属于持久层，浏览器需要在硬盘中保存类似cookie的各种数据。
        <ul>
         <li>
          HTML5定义了web database技术，这是一种轻量级完整的客户端存储技术。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <br/>
    <ul>
     <li>
      <h4>
       <a id="_352">
       </a>
       浏览器内核
      </h4>
      <ul>
       <li>
        <p>
         浏览器内核可以分成两部分：渲染引擎（layout engineer / Rendering Engin）和 JS 引擎。
        </p>
        <ul>
         <li>
          <strong>
           渲染引擎
          </strong>
          ：负责取得网页的内容（HTML、XML、图像等等）、整理讯息（例如加入 CSS 等），以及计算网页的显示方式，然后会输出至显示器或打印机。
          <ul>
           <li>
            浏览器内核的不同对于网页的语法解释会有不同，所以渲染的效果也不相同。
           </li>
          </ul>
         </li>
         <li>
          <strong>
           JS引擎
          </strong>
          ：解析 Javascript 语言，执行 javascript语言来实现网页的动态效果。
          <br/>
          <br/>
         </li>
        </ul>
       </li>
       <li>
        <p>
         常见的浏览器内核可以分为四种：Trident、Gecko、Blink、Webkit。
        </p>
       </li>
      </ul>
