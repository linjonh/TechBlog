    <h2>
     <a id="A__0">
     </a>
     A. 商业化业务逻辑（核心）
    </h2>
    <p>
     <img alt="[图片]" src="https://i-blog.csdnimg.cn/blog_migrate/7cdb1f6b7cfa5d4ae602acc763effd37.png"/>
    </p>
    <h3>
     <a id="_2">
     </a>
     三者之间的关系：
    </h3>
    <p>
     <strong>
      产品
     </strong>
     的最终目标是实现盈利，获取利润
    </p>
    <ul>
     <li>
      产品的主要目标是发展用户、吸引用户、留下用户
     </li>
     <li>
      通过投放产品广告为产品带来用户
     </li>
    </ul>
    <p>
     <strong>
      变现
     </strong>
     –“广告变现”
    </p>
    <ul>
     <li>
      是最简单、最有效且不存在领域限制的变现方式
     </li>
     <li>
      通过在App上展示广告主的广告获取收益，收益主要来自广告主的推广预算
     </li>
     <li>
      变现–内购、订阅
     </li>
    </ul>
    <p>
     <strong>
      投放
     </strong>
    </p>
    <ul>
     <li>
      广告计划与运营共同制定（如：出价、预算、回收）
      <ul>
       <li>
        广告素材
       </li>
       <li>
        广告文案
       </li>
       <li>
        怎么买量，买哪些量
        <br/>
        商业化模式–IAA–广告
        <br/>
        IAA休闲游戏从某个渠道（如抖音）获得低价的用户，再把这些用户通过产品内广告高价卖给其他游戏
        <br/>
        IAA休闲游戏的商业逻辑本质上是个买卖用户的“中间商”
       </li>
      </ul>
     </li>
     <li>
      但由于用户观看过多广告后会对广告脱敏，所以用户价值会得到提升，导致用户的CPA成本升高，于是作为休闲IAA游戏唯一的变现来源——eCPM变少了
      <br/>
      商业化模式–IAP–内购
     </li>
    </ul>
    <ol>
     <li>
      游戏内购
     </li>
     <li>
      工具app内购
     </li>
    </ol>
    <p>
     <strong>
      （iap与订阅的关系）
     </strong>
     <br/>
     IAP支持用户在无需返回App Store界面的情况下，直接在应用中购买升级内容、更多关卡、虚拟商品或额外功能。
     <br/>
     IAP游戏有两种盈利模式：
    </p>
    <ul>
     <li>
      开发者创建付费应用，并通过IAP途径出售升级或额外内容以增强游戏体验
     </li>
     <li>
      开发者推出免费应用，完全通过产品内购买道具等物品盈利
     </li>
    </ul>
    <h2>
     <a id="B_30">
     </a>
     B.分析报告
    </h2>
    <h3>
     <a id="_31">
     </a>
     一、分析主题
    </h3>
    <p>
     分析主题–投放
     <br/>
     游戏买量主要是游戏研发和渠道之间的关系，两者的合作关系大致有两种：
    </p>
    <ul>
     <li>
      CPS模式，按成交量（流水）分成结算
     </li>
     <li>
      CPA模式，按注册用户数（新增）付费
     </li>
     <li>
      另外还有CPM（按展示付费），CPC（按点击付费）,CPT（按时长付费）等等
     </li>
    </ul>
    <ol>
     <li>
      渠道分析
      <br/>
      渠道是指广告投放的媒介平台。同时在大渠道中又会有小的渠道特点，例如百度搜索平台，会有品牌专区，搜索营销的细分。由此构成了多样复杂的网络营销生态。
     </li>
     <li>
      流量质量–流量从哪里来，哪里来的用户多。
     </li>
    </ol>
    <ul>
     <li>
      流量是一个流动，变化的数据。根据不同的产品，不同目标，不同时间，可能都会不一样。
     </li>
     <li>
      一般来说可以用下面的方法：
      <br/>
      a. 对比分析，将买量用户和自然量用户进行对比，可以知道到底是渠道质量问题还是产品自身的问题
      <br/>
      b. 根据用户行为变化规律，了解用户使用的习惯
      <br/>
      c. 根据用户买入的时段变化，发现用户的规律和异常点，进而查找深层原因，及时发现问题，调整投放
      <br/>
      d. 建立复合指标，将用户行为分解为可供评估的的数据，例如注册、留存、广告播放次数等
      <br/>
      e. A/B测试
     </li>
    </ul>
    <ol start="2">
     <li>
      转化率
      <br/>
      外部来源的的流量转化率，可以直接体现该渠道的流量质量。结合各渠道转化率的对比分析，则可以体现各渠道的转化效果
     </li>
     <li>
      展示素材分析
     </li>
     <li>
      广告渠道里的展示的形式、位置
     </li>
     <li>
      素材吸引来的用户的用户画像
      <br/>
      科学的投放节奏是先测试，再放量
      <br/>
      投放方式一般分为三个步骤：
     </li>
    </ol>
    <ul>
     <li>
      通过模仿以及创造，做出新的a/b两个或者两个以上的计划
     </li>
     <li>
      通过控制变量的方法进行测试，并且优化
     </li>
     <li>
      最后测试出最适合自己，最有前景的法案进行大量投放
      <br/>
      <mark>
       需要分析师对媒体的展示样式、样式的特点做好数据的整理和分析。
      </mark>
     </li>
    </ul>
    <ol start="3">
     <li>
      投放时间分析
     </li>
    </ol>
    <ul>
     <li>
      淡季和旺季是否有分？如：节假日与工作日
     </li>
     <li>
      24小时的广告分配
     </li>
    </ul>
    <ol start="4">
     <li>
      预算（价格）分析
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce6ecdd7972421664b493ffc0d6be1f0.png">
       <br/>
       根据预算来评估营销效果。同时找到预算和投放效果的最佳配比。
      </img>
     </li>
     <li>
      着陆页分析
      <br/>
      <mark>
       着落页设计的不好，用户在点击链接进入页面后，就会马上关闭页面。造成跳失率奇高。
      </mark>
      <br/>
      着陆页的分析常有以下角度
     </li>
     <li>
      到达率=访问次数/点击次数
      <br/>
      到达率表示广告来源的访问量与广告点击量之间的比值，反映真实的访客数量
     </li>
     <li>
      跳失率
      <br/>
      （返回app是否算跳出）
      <br/>
      用户进入网站，只访问了一个页面就离开的访问次数占该页面总访问次数的比例，直接衡量页面质量或流量质量的好坏。
      <br/>
      影响着陆页的原因还可能包括：
     </li>
    </ol>
    <ul>
     <li>
      关键词、创意、产品的内容
     </li>
     <li>
      广告承接页制作较差
     </li>
     <li>
      产品竞争力不足
     </li>
     <li>
      页面信息不明确，用户无兴趣
     </li>
    </ul>
    <p>
     <strong>
      投放逻辑点：
     </strong>
     <br/>
     次留双出价：广告主同时设置目标转化出价和付费用户（深度转化）出价，两者相除即是目标付费率。
    </p>
    <ul>
     <li>
      在保证基础转化成本跑量的前提下，系统会自动在海量用户中，优先选择预估付费概率最高的用户推荐广告主的广告，使最终付费率趋近于目标付费率
      <br/>
      分析主题–变现
     </li>
    </ul>
    <pre><code>关键词：
ADN=ad network  广告商or广告渠道（备注：为了和投放渠道区分开，变现这边的渠道统一口径就叫ADN） 
OM:sdk  openmedIcation（广告聚合平台）
IPU=人均展示次数
跑飞：一般指的是买量单价非常高，根本收不回来的那种，一直投就一直亏
</code></pre>
    <ol>
     <li>
      变现渠道–adn
     </li>
     <li>
      国内：头条–穿山甲、优量汇–广点通–tencent、快手–磁力引擎、sigmob–乐元素、华为（广告只在华为手机中播放）、小米 、OV(oppo vivo)
     </li>
     <li>
      海外：Anmob（谷歌）、Facebook 、 unity、 APPlouin、 ironsource 、roungle 、popub、A4G
     </li>
    </ol>
    <ul>
     <li>
      精准营销
     </li>
    </ul>
    <ol start="2">
     <li>
      变现收益
     </li>
     <li>
      ecpm=（收入/展示次数）*1000 （千次展示期望收入）
      <br/>
      - 收入/展示次数的数值会非常小，乘以1000以扩大数据展示。
     </li>
     <li>
      IM：广告的展示次数；
     </li>
     <li>
      竞价分析–聚合工具
      <br/>
      聚合工具主要是用来做广告管理的、以及广告实时竞价的工具
      <br/>
      <em>
       使用竞价的好处：解决运营的压力，目的是提高效率
      </em>
     </li>
     <li>
      广告优化
      <br/>
      https://docs.toponad.com/#/zh-cn/android/DeveloperAccess/WaterFall/waterfall_best_practices
     </li>
     <li>
      广告分层
      <br/>
      若干个分层，组成一个waterfful
      <br/>
      - 每一个分层都有一个分层标识，叫iid
      <br/>
      - 流量拆成多层，分层之间的策略是互补的
      <br/>
      - 名词解释：头层–指waterfful分层的第一层；
      <br/>
      Auto–指的是不用建立分层的广告配置方法；
      <br/>
      - 做广告分层时可在后台设置ecpm，表明想要多少钱的广告
      <br/>
      广告分层主要看填充率、收益
      <br/>
      - 核心逻辑：收的钱&gt;花的钱
      <br/>
      fillrate（填充率）=fills（填充数）/Requst（请求数）
     </li>
     <li>
      考虑上层的优化
     </li>
     <li>
      保证有广告可看
     </li>
     <li>
      设置分层的广告类型：一般考虑请求时长的问题，spash（开屏）不会设置分层。
      <br/>
      二、分析报告框架
      <br/>
      展示信息（重要性）
      <br/>
      数据源：使用的数据来源（包名）
      <br/>
      分析时间：
      <br/>
      产品包：数据是否存在分包，如果有多个包的数据，则需说明
      <br/>
      指标定义：指标的计算方式等
      <br/>
      分析内容
      <br/>
      结论与建议（角度上）
     </li>
    </ol>
    <ul>
     <li>
      出价策略
     </li>
     <li>
      投放渠道
     </li>
     <li>
      玩法调整
     </li>
     <li>
      盈利预测
     </li>
    </ul>
    <h2>
     <a id="C_129">
     </a>
     C.商业化常用指标
    </h2>
    <ol>
     <li>
      Ecpm
     </li>
    </ol>
    <ul>
     <li>
      ECPM（effective cost per mile）指的是每一千次展示可以获得的广告收入。
     </li>
     <li>
      ECPM = 广告单价×网页点击率×1000
     </li>
     <li>
      ecpm是一个与网页展示次数无关的指标，ecpm只是用来反映网站盈利能力的参数，不代表收入，最终是由平均广告单价和广告的点击率决定的
     </li>
    </ul>
    <ol start="2">
     <li>
      LTV
     </li>
    </ol>
    <ul>
     <li>
      生命周期总价值，LTV(life time value)，也可以称为
      <mark>
       客户终生价值
      </mark>
     </li>
     <li>
      公司从用户所有的互动中所得到的全部
      <mark>
       经济收益
      </mark>
      的总和。用于衡量用户价值，是确定该用户是否为高价值用户的重要参考指标。
     </li>
    </ul>
    <ol start="3">
     <li>
      ROI
     </li>
    </ol>
    <ul>
     <li>
      投资回报率（ROI）是指通过投资而应返回的价值，即一次完整的投放变现流程后得到的回报。
     </li>
    </ul>
    <ol start="4">
     <li>
      CTR
     </li>
    </ol>
    <ul>
     <li>
      CTR=实际点击次数/广告的展示次数
     </li>
     <li>
      CTR即点击通过率，指网络广告的点击到达率，是衡量
      <mark>
       互联网广告
      </mark>
      效果的一项重要指标。
     </li>
    </ul>
    <ol start="5">
     <li>
      CPC
     </li>
    </ol>
    <ul>
     <li>
      每点击成本，也称为每千人点击成本，以每次点击(或每一千次点击)为单位进行收取
     </li>
    </ul>
    <ol start="6">
     <li>
      Cpa
     </li>
    </ol>
    <ul>
     <li>
      一种广告计费模式，照行为作为指标来计费，这个行为可以是注册、下载、点击等等。
     </li>
    </ul>
    <ol start="7">
     <li>
      Cpm
     </li>
    </ol>
    <ul>
     <li>
      CPM（Cost Per Mille），指广告投放过程中，平均每一千人分别听到或者看到某广告一次一共需要多少广告成本。这种计费方式下，只要展示了广告主的广告内容，广告主就为此付费。
     </li>
    </ul>
    <ol start="8">
     <li>
      Cpi
     </li>
    </ol>
    <ul>
     <li>
      单次安装成本
     </li>
    </ul>
    <ol start="9">
     <li>
      Cvr
     </li>
    </ol>
    <ul>
     <li>
      广告点击中产生了转化的次数占比，是一个衡量CPA广告效果的指标，CVR=(转化量/点击量)*100%
     </li>
    </ul>
    <ol start="10">
     <li>
      渗透率
     </li>
    </ol>
    <ul>
     <li>
      渗透率（Penetration Rate）= 观看广告人数/活跃用户数
     </li>
     <li>
      代表观看过广告的人数占活跃用户的百分比，用来评估游戏内广告场景和触发机制是否合理，即广告是否覆盖足够的用户量
     </li>
    </ul>
    <ol start="11">
     <li>
      广告填充率（Fill Rate）
     </li>
    </ol>
    <ul>
     <li>
      广告填充率（Fill Rate）=广告填充数/广告请求数
     </li>
     <li>
      表示在所有的广告请求中，广告平台返回广告的次数所占广告请求数的百分比。
     </li>
    </ul>
    <h2>
     <a id="D__159">
     </a>
     D. 归因
    </h2>
    <p>
     广告归因用来匹配多方数据（广告主、广告商、发布者、用户）。将产品用户及推动其安装的来源（营销活动）进行匹配。这些归因数据可以用来监测推广效果、开展高效再营销推广及优化广告素材。
    </p>
