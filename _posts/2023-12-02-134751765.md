    <p>
     在本教程中，我们将学习如何将 TCA9548A I2C 多路复用器与 Arduino 结合使用。我们将讨论如何通过整合硬件解决方案来使用多个具有相同地址的 Arduino 的 I2C 设备。通过使用 TCA9548A I2C 多路复用器，我们将能够增加 Arduino 的 I2C 地址范围，并能够连接 8 个具有相同地址的 I2C 设备。
    </p>
    <p>
     TCA9548A 是一款 1 至 8 I2C 多路复用器。通过将设备连接到多路复用器并选择要通过多路复用器进行通信的设备，您可以在单个微控制器上使用最多 8 个具有相同地址的 I2C 从设备。
    </p>
    <h3 style="margin-left:0;">
     <span style="color:#262626;">
      <span style="background-color:#ffffff;">
       Arduino I2C 通信引脚
      </span>
     </span>
    </h3>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       对于 I2C 通信，不同的 Arduino 板有不同的引脚专用为 SDA 和 SCL 引脚。下面的列表显示了各种板上的这些引脚号。
      </span>
     </span>
    </p>
    <ol style="margin-left:3em;">
     <li>
      在 Arduino UNO 中，引脚 A4 = SDA，引脚 A5 = SCL
     </li>
     <li>
      对于
      <a href="https://microcontrollerslab.com/introduction-arduino-mega-2560/" rel="nofollow" title="Arduino Mega2560">
       Arduino Mega2560
      </a>
      ，引脚 20 = SDA，引脚 21 = SCL
     </li>
     <li>
      在 Arduino Leonardo 中，引脚 2 = SDA，引脚 3 = SCL
     </li>
     <li>
      对于
      <a href="https://microcontrollerslab.com/introduction-arduino-due/" rel="nofollow" title="Arduino Due">
       Arduino Due
      </a>
      ，引脚 20 = SDA，引脚 21 = SCL、SDA1、SCL。
     </li>
    </ol>
    <h3 id="using-multiple-different-address-slave-i2c-arduino" style="margin-left:0;">

    <h4 style="margin-left:0;">

    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">

    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       此代码将扫描与 Arduino 连接的任何 I2C 设备，并指定串行终端中具有地址的设备数量。
      </span>
     </span>
    </p>
    <pre><code>#include &lt;Wire.h&gt;
 
void setup() {
  Wire.begin();
  Serial.begin(115200);
  Serial.println("\nI2C Scanner");
    byte error, address;
  int nDevices;
  Serial.println("Scanning...");
  nDevices = 0;
  for(address = 1; address &lt; 127; address++ ) {
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
    if (error == 0) {
      Serial.print("I2C device found at address 0x");
      if (address&lt;16) {
        Serial.print("0");
      }
      Serial.println(address,HEX);
      nDevices++;
    }
    else if (error==4) {
      Serial.print("Unknown error at address 0x");
      if (address&lt;16) {
        Serial.print("0");
      }
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0) {
    Serial.println("No I2C devices found\n");
  }
  else {
    Serial.println("done\n");
  }
  delay(5000);  
  Serial.print("i2c devices found:");
  Serial.println(nDevices); 
}
 
void loop() {
      
}
</code></pre>

    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">

      </span>
     </span>
    </p>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       代码首先包含代码正常功能所需的所有必要库。Wire.h 将允许我们通过 I2C 协议进行通信。其余的库是我们刚刚安装的用于 BME280 传感器、SHT31 传感器和 OLED 功能的库。
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:1.5em;text-align:left;"><code class="language-language-c">#include &lt;Wire.h&gt;
#include &lt;Adafruit_Sensor.h&gt;
#include &lt;Adafruit_BME280.h&gt;

    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       然后，我们通过将其设置在默认 I2C GPIO 引脚上来定义名为“bme”的 Adafruit_BME280 对象，并创建名为“sht31”的 Adafruit_SHT31 对象。
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:1.5em;text-align:left;"><code class="language-language-c">Adafruit_BME280 bme; 
Adafruit_SHT31 sht31 = Adafruit_SHT31();</code></pre>
    <p style="margin-left:0;">

    <h3 id="tca9548a-multiple-i2c-devices-same-addresses-arduino" style="margin-left:0;">
     <span style="color:#262626;">
      <span style="background-color:#ffffff;">
       TCA9548A 与 Arduino 具有相同地址的多个 I2C 设备
      </span>
     </span>
    </h3>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       在上一节中，我们能够在 Arduino 的相同 I2C 引脚上连接三个不同的 I2C 设备。这是因为 Arduino 能够通过其唯一的地址来识别它们。但是，当我们想用Arduino连接多个设备进行I2C通信，但它们的地址相同时，会发生什么情况呢？例如，两个BME280传感器与Arduino连接或两个OLED与Arduino连接。
      </span>
     </span>
    </p>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       要在 Arduino 上使用相同的 I2C 设备，我们要么必须更改设备的 I2C 地址，要么使用 I2C 多路复用器。然而，更改设备的地址并不是那么简单，并且它只允许在同一 I2C 总线上使用有限的设备。因此我们可以使用多路复用器，例如。TCA9548A 允许最多 8 个具有相同地址的设备连接到同一 I2C 总线。因此，它的工作电压范围为 3-5.5 伏，因此可以方便地与具有 3.3/5V 逻辑引脚的微控制器一起使用。
      </span>
     </span>
    </p>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       下图显示了 TCA9548A 多路复用器，我们将其与具有相同地址的 I2C 设备一起使用，以便与 Arduino 进行通信。
      </span>
     </span>
    </p>
    <div style="margin-left:0;">
     <p class="img-center">
      <img alt="TCA9548A 1 至 8 I2C 多路复用器 IC" class="wp-image-35197" height="900" src="https://i-blog.csdnimg.cn/blog_migrate/784632b5d0e68b43686946f755d8029e.jpeg" width="1006"/>
     </p>
     <p>
      <span style="color:#3a3a3a;">
       <span style="background-color:#ffffff;">
        TCA9548A 多路复用器
       </span>
      </span>
     </p>
    </div>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       它是一种电子开关设备，使用户能够将多个 I2C 总线与其自身连接。所有具有相同地址的从站需要连接到不同的总线。为了与从机通信，连接到从机的总线由用户切换，然后寻址。
      </span>
     </span>
    </p>

    <h4 style="margin-left:0;">
     <span style="color:#0c0c0c;">
      <span style="background-color:#ffffff;">
       TCA9548A 多路复用器引脚分配
      </span>
     </span>
    </h4>
    <p style="margin-left:0;">
     <span style="color:#3a3a3a;">
      <span style="background-color:#ffffff;">
       下图显示了用作 I2C 扩展器的 TCA9548A 多路复用器的引脚排列。
      </span>
     </span>
    </p>
    <div style="margin-left:0;">
     <p>
     </p>
     <p class="img-center">
      <img alt="TCA9748A 多路复用器引脚分配" class="wp-image-37249" height="354" src="https://i-blog.csdnimg.cn/blog_migrate/d59a622ba529443c1aec4ad23ddd7555.jpeg" width="606"/>
     </p>
    </div>
    <ul style="margin-left:3em;">
     <li>
      <strong>
       Vin
      </strong>
      ：这是为模块提供电源的引脚。将Arduino的3.3V或5V引脚与此引脚连接。
     </li>
     <li>
      <strong>
       GND
      </strong>
      ：这是提供公共接地的引脚。
     </li>
     <li>
      <strong>
       SDA
      </strong>
      ：这是与主设备连接的I2C串行数据引脚。
     </li>
     <li>
      <strong>
       SCL
      </strong>
      ：这是与主设备连接的I2C串行时钟引脚。
     </li>
     <li>
      <strong>
       RST
      </strong>
      ：该引脚负责复位多路复用器。
     </li>
     <li>
      <strong>
       A0、A1、A2
      </strong>
      ：这三个引脚用于更改多路复用器的 I2C 地址。默认情况下，当这些引脚未连接时，其地址为 0x70。借助该模块，单个 I2C 地址最多可扩展为 8 个 I2C 地址。
     </li>
     <li>
      <strong>
       SDx、SCx
      </strong>
      ：这些是外部 I2C 通道通道 0 至通道 7。对于具有相同地址的每个 I2C 设备，将 SDA 引脚与 SDx 连接，将 SCL 引脚与 SCx 连接。
     </li>
    </ul>

       接下来，我们通过调用 TCA9548A() 函数来激活 I2C 通道 0，并将 0 作为其中的参数传递。我们的第一个 OLED 与 I2C 通道 0 连接。之后我们初始化 OLED 显示、清屏、设置文本颜色、设置字体大小和光标位置。然后我们在显示屏上打印文本“OLED #1”。

     </span>
