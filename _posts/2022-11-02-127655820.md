    <h2>
     <a id="_0">
     </a>
     概述
    </h2>
    <ul>
     <li>
      基于UART数据格式
     </li>
     <li>
      串行通信
     </li>
     <li>
      开放、不需要使用费和版税
     </li>
     <li>
      单线传输
     </li>
     <li>
      最高速率20kbit/s
     </li>
     <li>
      单主多从结构（不存在冲突，无需仲裁）
     </li>
     <li>
      基于通用UART/SCI的低成本接口硬件
     </li>
     <li>
      从节点无须晶振，就可同步
     </li>
     <li>
      可计算信号传输的延迟时间
     </li>
     <li>
      灵活的增加或减少节点
     </li>
     <li>
      一条总线最多可连接16个节点（电气特性）
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      拓扑结构
     </li>
     <li>
      <ul>
       <li>
        一个LIN由一个主任务、若干个从任务
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        一个主节点既有主任务 也有从任务
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        从节点只包含从任务
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        主任务负责决定总线上的报文，从任务提供每一帧的数据
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      可传输两类数据：
     </li>
     <li>
      <ul>
       <li>
        信号报文
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        诊断报文
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      调度表：
     </li>
     <li>
      <ul>
       <li>
        负责调度网络各报文发送的数据
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        在网络系统设计阶段确定，使得LIN通信具有可预测性
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="_26">
     </a>
     协议规范
    </h2>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1b5fc5e978b5f6ee57b218dcb030cfcd.png"/>
    </p>
    <h3>
     <a id="_29">
     </a>
     帧格式
    </h3>
    <ul>
     <li>
      字节间隔
     </li>
     <li>
      <ul>
       <li>
        MCU处理时间
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        帧长度可能增长
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        MIN可为0
       </li>
      </ul>
     </li>
     <li>
      字节场
     </li>
     <li>
      <ul>
       <li>
        SCI通信格式
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        一个字节需要十个位时间
       </li>
      </ul>
     </li>
     <li>
      间隔场；
     </li>
     <li>
      <ul>
       <li>
        间隔信号：至少13个显性位 ， 间隔界定符：至少1个隐形位
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        间隔场是唯一一个不符合字节场格式的场
       </li>
      </ul>
     </li>
     <li>
      同步场
     </li>
     <li>
      <ul>
       <li>
        固定：0X55
       </li>
      </ul>
     </li>
     <li>
      标识符场
     </li>
     <li>
      <ul>
       <li>
        0-64
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        标识符+奇偶校验
       </li>
      </ul>
     </li>
     <li>
      响应
     </li>
     <li>
      <ul>
       <li>
        低位先发
       </li>
      </ul>
     </li>
     <li>
      校验和场
     </li>
     <li>
      <ul>
       <li>
        经典校验：仅仅校验数据场
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        增强校验：标识符场+数据场内容
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        标识符为0x3c 和 0x3D 的帧只能使用经典校验
       </li>
      </ul>
     </li>
    </ul>

    <h3>
     <a id="_54">
     </a>
     帧类型
    </h3>
    <ul>
     <li>
      无条件帧
     </li>
     <li>
      <ul>
       <li>
        ID为0-0x3b
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        标准LIN帧类型，用于周期通信
       </li>
      </ul>
     </li>
     <li>
      事件触发帧
     </li>
     <li>
      <ul>
       <li>
        需要轮询、调度
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        节省带宽、关联
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        必须有一个独立的ID，该ID与多个无条件帧的ID相关联，无条件帧满足以下条件：相同长度、校验和模型、首个数据字节不带信号
       </li>
      </ul>
     </li>
     <li>
      零星帧/偶发帧
     </li>
     <li>
      <ul>
       <li>
        主节点有事件性的数据
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        主节点发送
       </li>
      </ul>
     </li>
     <li>
      诊断帧
     </li>
     <li>
      <ul>
       <li>
        0x3c(主请求帧)、0x3d(从响应帧)
       </li>
      </ul>
     </li>
     <li>
      保留帧
     </li>
     <li>
      <ul>
       <li>
        自定义帧
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        保留帧
       </li>
      </ul>
     </li>
    </ul>
