        基本介绍
       </h3>
      </div>
      <div class="para">
       <div class="lemma-picture text-pic layout-right">
        <a class="image-link" href="https://baike.baidu.com/pic/%E5%BE%AE%E8%BD%AF%E6%9C%80%E6%9C%89%E4%BB%B7%E5%80%BC%E4%B8%93%E5%AE%B6%E5%A5%96/10792983/0/08f790529822720e97fc74637acb0a46f21fab7d?fr=lemma&amp;ct=single" rel="nofollow" title="mvp logo">
         <img alt="mvp logo" src="https://gss3.bdstatic.com/-Po3dSag_xI4khGkpoWK1HF6hhy/baike/s%3D220/sign=ad75791d4bed2e73f8e9812eb700a16d/08f790529822720e97fc74637acb0a46f21fab7d.jpg"/>

       实际生活中，人们总是信任专家的建议和反馈。MVP正是这样一群拥有
       <strong>
        丰富知识和实际经验
       </strong>
       的
       <strong>
        微软技术专家
       </strong>
       。他们不是微软的员工，但是非常乐于通过在线或离线社区的方式帮助技术人士。另一方面，微软公司
       <strong>
        时刻不忘倾听来自用户的意见反馈，不断开发新产品，改进技术，提高用户体验
       </strong>
       。MVP代表来自社群的广大用户，他们的专家意见更能协助微软公司
       <strong>
        了解用户的真实需求
       </strong>
       。
      </div>
      <div class="para">
       Most Valuable Professionals（最有价值专家）是微软对上述专家在技术社群专业贡献的一种正式认同。该项目主要目的在于鼓励形成一个充满活力的全球性社群，使得微软和用户之间建立良好的相互关系，增进相互了解。主要策略为：
      </div>
      <ul class="custom_dot para-list list-paddingleft-1">
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          在全球范围内认可MVP 并建立交流渠道
         </strong>
         － 通过
         <strong>
          跨产品、服务和行业
         </strong>
         的广大社群，嘉奖有影响力和特殊贡献的专家，
         <strong>
          并赋予他们特殊的资源和权力
         </strong>
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          贴近用户、提高体验
         </strong>
         － 认可技术专家的特殊贡献，不论他们来自哪个领域、使用何种语言，致力提高使用微软技术的亲身体验
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         推动项目日臻完美－ 不断提高对技术专家的支持力度，
         <strong>
          在全球范围建立协调和沟通网络，增进微软和用户的相互了解
         </strong>
        </div>
       </li>
      </ul>
      <div class="para">
       该项目已经运作了17年，在全球90多个国家拥有接近4000多位最有价值专家。
      </div>

       什么是微软技术社区「最有价值专家」（MVP）？
      </div>
      <div class="para">
       微软最有价值专家是微软的一个年度奖项。MVP来自于各行各业，但是他们都有两个共同点：
      </div>
      <div class="para">
       1. 精通某一个微软产品或技术领域。
      </div>
      <div class="para">
       2. 愿意与他人分享技术经验。
      </div>
      <div class="para">
       在微软，我们视MVP为“最有价值的合作伙伴”。通过奖励、肯定和支持这些优秀的技术人员，微软并和这些重要的伙伴保持紧密的联系。
      </div>
      <div class="para">
       为什么有此微软技术社区「最有价值专家」（MVP）奖项？
      </div>
      <div class="para">
       Microsoft 认为，活跃的交互式用户社区是帮助客户最大限度地利用解决方案、使用户软件投资收益最大化的关键。MVP 大奖是 Microsoft 用来表扬那些在所属的技术和产品社区中产生了重大积极影响的参与者的方式。Microsoft 希望社区参与者和领导者知道，我们对他们的贡献是十分认可的。MVP 大奖的宗旨是评选出那些积极参与、且乐于帮助在线和离线社区中其他成员的表现突出者，并向他们表示感谢。
      </div>
      <div class="para">
       微软怎样挑选微软技术社区「最有价值专家」（MVP）？
      </div>
      <div class="para">
       授予MVP 大奖先要经过提名，在提名期间，将成员在过去一年中对技术社区的贡献与其候选人所做的贡献进行比较。再经过审核，最后确定获奖者。
      </div>
      <div class="para">
       微软技术社区「最有价值专家」（MVP）需要精通微软所有的技术与产品吗？
      </div>
      <div class="para">
       不需要。尽管许多MVP对不只一种产品或技术有很深入的了解，但MVP并不需要具备精通微软所有技术或产品的条件。
      </div>
      <div class="para">
       微软技术社区「最有价值专家」（MVP）将从微软获得报酬吗？
      </div>
      <div class="para">
       不。Microsoft 的确提供软件形式的小小奖励以及核心计划中的其他利益，但MVP 不会得到Microsoft MVP 计划任何货币形式的报酬。
      </div>
      <div class="para">
       微软技术社区「最有价值专家」（MVP）奖项及权益包括什么？
      </div>
      <ul class="custom_dot para-list list-paddingleft-1">
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         微软总部授予的全球统一的微软MVP证书；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         微软全球统一的“MVP”标志的大礼包；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          受邀参加在美国微软总部西雅图举办的微软全球MVP年会
         </strong>
         ；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         一年免费订阅的
         <strong>
          MSDN Premium或TechNet Plus
         </strong>
         (如选择订阅
         <strong>
          eSku版本
         </strong>
         ，可享受一年免费的MSDN 和TechNet plus 的订阅)；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          合作伙伴级别的知识库访问权限
         </strong>
         ；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          一年内指定微软活动的免费入场券
         </strong>
         ；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         150美金美国微软电子商店购物金币；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          仅向MVP开放的专有技术网站、专有新闻组，含有丰富的技术资源
         </strong>
         ；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         <strong>
          与微软中国技术社区专业人士、微软产品组员工、微软美国总部的无限沟通机会
         </strong>
         ；
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         MVP LOGO的使用权利；
        </div>
       </li>
      </ul>
      <div class="para">
       被授予微软技术社区「最有价值专家」（MVP）的人需要遵守什么规则？
      </div>
      <div class="para">
       我们要求所有MVP都能表现出自身所具有的
       <strong>
        专业素质和谦虚精神
       </strong>
       ，
       <strong>
        并持续积极参与微软技术社区讨论区
       </strong>
       、踊跃并正确地回答他人技术问题。
      </div>
      <div class="para">
       微软对微软技术社区「最有价值专家」（MVP）的要求是什么？
      </div>
      <div class="para">
       MVP是一项针对已做贡献的嘉奖措施，因此微软要求所有MVP都能表现出微软的专业素质和谦虚精神、与其它所有参与微软技术相关讨论区的人一样遵守讨论区规则。除此之外，MVP在任期内也应充份发挥热心助人的微软「最有价值专家」精神─持续积极参与微软技术社区讨论区、主动且正确地回答他人技术问题。
      </div>
      <div class="para">
       我怎样成为微软技术社区「最有价值专家」（MVP）？
      </div>
      <div class="para">
       MVP 计划和年度奖励是基于个人在上一年度的贡献，再将其贡献与其他候选人的贡献进行比较，评选出提名人选，从而确定谁将获奖。积极参与微软技术相关讨论区、踊跃并正确地回答他人技术问题以赢取讨论区其它参与者的尊敬和微软的认同。MVP申请每个季度一次。一般的申请截止时间为每个季度的第一个月，经过亚太区和全球范围的复选，最终的获奖名单在下个季度的第一天公开宣布。
       <strong>
        可以通过微软员工、微软中文技术论坛、博客园、CSDN
       </strong>
       等论坛推荐。
      </div>
      <div class="para">
       微软技术社区「最有价值专家」（MVP）的选拔条件是什么？
      </div>
      <div class="para">
       微软技术社区「最有价值专家」奖项选拔的条件如下：
      </div>
      <ul class="custom_dot para-list list-paddingleft-1">
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         以微软技术为主题的
         <strong>
          作家、讲师、培训师
         </strong>
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         IT业界的业内知名专业人士，通过印刷媒体、blog或其它形式分享经验和观点
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         参与和微软技术有关的项目，担任主要角色的技术和管理人员
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         建立讨论微软技术的技术网站，担任主要角色的技术和管理人员
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         参与微软中文技术论坛，积极地帮助论坛用户解决疑难问题的技术论坛高手
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         参与其他第三方与微软技术相关的论坛，积极地帮助论坛用户解决疑难问题的技术论坛高手
        </div>
       </li>
       <li class="list-dot list-dot-paddingleft">
        <div class="para">
         技术开发人员、作者、微软产品支持者、教育界、资讯决策者、IT 工程师、企业界等等。
        </div>
       </li>
      </ul>
      <div class="para">
       MVP代表Microsoft吗？
      </div>
      <div class="para">
       不代表。MVP 不是Microsoft 的员工，他们也不代表Microsoft 发言。MVP 仅是因其在技术社区中的杰出成就，而获得Microsoft 认可和奖励的第三方个人。
      </div>
      <div class="para">
       Microsoft MVP大奖的有效期为多长时间？
      </div>
      <div class="para">
       MVP 大奖的有效期为一年。 在此期间，MVP 奖获奖者享有Microsoft MVP 荣誉称号以及大奖所包括的所有特权。
      </div>
      <div class="para">
       要获得Microsoft MVP大奖，需要具备一定条件吗？
      </div>
      <div class="para">
       是的。所有Microsoft MVP 的年龄必须是18 岁以上。
      </div>
      <div class="para">
       为什么Microsoft MVP的年龄必须是18岁或18岁以上？
      </div>
      <div class="para">
       Microsoft 承诺保护未成年人的安全和隐私。 未成年人定义为未达到18 岁的任何人。 为帮助确保他们的安全，MVP 计划规定了一项策略，即从2006 年4 月起所有Microsoft MVP 获奖者必须确认他们已18 岁或18 岁以上。
      </div>
