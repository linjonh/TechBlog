    <p>
     做过移动开发的同学都知道，现在都讲究组件化。
     <br/>
     别急，小程序里也有组件这个概念，有两种实现方式，一种是模板（template），一种是自定义组件component。
     <br/>
     一般自定义组件就是右键点击新建component，小程序ide会自动生成四个文件，分别是
     <br/>
     1.xx.json
     <br/>
     该文件主要是配置文件，可以添加自己引用到的其他组件，只需要在usingComponents属性后面添加就行，注意引用时注意组件的路径。
    </p>
    <pre><code>  "usingComponents": {
	}
</code></pre>
    <p>
     2.xx.wxml
     <br/>
     页面文件，其实就是常规的html文件
     <br/>
     3.xx.wxss
     <br/>
     css文件，用来写样式
     <br/>
     4.xx.js
     <br/>
     该js文件负责数据的注入，以及事件的绑定
    </p>
    <p>
     <strong>
      重点内容
     </strong>
     <br/>
     properties：
     <br/>
     该属性主要负责数据的注入
    </p>
    <pre><code>	   /**
	   * 组件的属性列表
	   */
	  properties: {
	    data: {
	      type: Object,
	      value: {
	
	      }
	    }
	  }
</code></pre>
    <p>
     上面的data属性对应你引用组件里的属性，data = {
     <!-- -->
     { 数据源}}
    </p>
    <pre><code>data：{

}
</code></pre>
    <p>
     负责数据的初始化，以及数据的驱动来更新页面
    </p>
    <pre><code>methods：{

}
</code></pre>
    <p>
     该属性主要对应xml里绑定的点击事件，组件事件的冒泡处理也在这里,即：组件里的点击事件，要到最外层处理，因为数据肯定实在最外层的页面js中，这个时候就要用到组件
    </p>
    <pre><code> _xx: function (e) {
      this.triggerEvent("xxEvent', {}, { bubbles: true, composed: true })
    }
</code></pre>
    <p>
     template
    </p>
    <p>
     template很简单，
    </p>
    <pre><code>&lt;template name="xx"&gt;
  &lt;view class=""&gt;
   
  &lt;/view&gt;
&lt;/template&gt;

引用时通过
&lt;template is="xx"  /&gt;即可

</code></pre>
