    <h1 class="title-article" id="articleContentId">
     Linux升级python3（python2，python3共存）
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <pre></pre>
    <p>
     低版本Linux系统自带python2，下面将介绍如何安全、快捷的升级python3，避免入坑。
    </p>
    <p>
     安装python3的基本操作，以3.6.10为例：
    </p>
    <pre><code class="language-bash">[root@xincai ~]# wget https://www.python.org/ftp/python/3.6.10/Python-3.6.10.tgz
[root@xincai ~]# tar -zxvf Python-3.6.10.tgz
[root@xincai ~]# cd Python-3.6.10
[root@xincai Python-3.6.10]# ./configure --prefix=/usr/local/python
[root@xincai Python-3.6.10]# make
[root@xincai Python-3.6.10]# make install</code></pre>
    <p>
     到这里都是没有问题的，编译的时候可能会要求安装gcc。
    </p>
    <h3>
     入坑
    </h3>
    <p>
     如果接下来使用下面的操作，将会使yum崩溃，谨慎...
    </p>
    <pre><code class="language-bash">[root@xincai ~]# mv /usr/bin/python /usr/bin/python2
[root@xincai ~]# ln -s /usr/local/python/bin/python3 /usr/bin/python</code></pre>
    <p>
     这个时候，敲入python，显示python3安装成功了
    </p>
    <pre><code class="language-bash">[root@xincai ~]# python
Python 3.6.10 (default, Feb 4 2020, 22:28:22)
[GCC 4.4.7 20120313 (Red Hat 4.4.7-23)] on linux
Type "help", "copyright", "credits" or "license" for more information.
&gt;&gt;&gt;</code></pre>
    <p>
     但是yum已经不能使用了，按照网上的教程，重新修复yum和python比较费时间。
    </p>
    <h3>
     纠正错误
    </h3>
    <p>
     为了避免发生上面的错误，需要容忍python2的存在，因为centos很多之前软件都是依赖他的，我们需要pytho2和python3共存。
    </p>
    <p>
     新建一软连接 python3就可以了
    </p>
    <pre><code>[root@xincai ~]# ln -s /usr/local/python/bin/python3 /usr/bin/python3</code></pre>
    <p>
     再次敲入python3，显示python3安装成功，验证python2和yum并没有受到影响。
    </p>
    <pre><code class="language-bash">[root@xincai ~]# python3
Python 3.6.10 (default, Mar 6 2022, 18:39:37)
[GCC 4.8.5 20150623 (Red Hat 4.8.5-44)] on linux
Type "help", "copyright", "credits" or "license" for more information.
&gt;&gt;&gt;
&gt;&gt;&gt;</code></pre>
    <h3>
     总结
    </h3>
    <p>
     不要去动原先的软连接！执行命令用python3，避免把系统给搞坏，自己无法修复。
    </p>
    <h3>
     <strong>
      附：
     </strong>
     pip3的安装步骤
    </h3>
    <p>
     如果python3的安装路径bin下有pip3，直接创建软连接就行了
    </p>
    <pre><code>[root@xincai ~]# ln -s /usr/local/python/bin/pip3 /usr/bin/pip3</code></pre>
    <p>
     没有pip3
    </p>
    <pre><code class="language-bash">[root@xincai ~]# wget wget --no-check-certificate https://pypi.python.org/packages/source/p/pip/pip-8.0.2.tar.gz#md5=3a73c4188f8dbad6a1e6f6d44d117eeb
[root@xincai ~]# tar -zxvf pip-8.0.2.tar.gz
[root@xincai ~]# cd pip-8.0.2
[root@xincai pip-8.0.2]# python3 setup.py build
[root@xincai pip-8.0.2]# python3 setup.py install</code></pre>
    <p>
     没有setuptools，需要先安装setuptools
    </p>
    <p>
     <strong>
      报错：ImportError: cannot import name HTTPSHandler
     </strong>
    </p>
    <p>
     原因在于openssl,openssl-devel两个文件包未正确安装。用下来的命令来安装:
    </p>
    <pre><code class="language-bash">[root@xincai ~]# yum install openssl -y
[root@xincai ~]# yum install openssl-devel -y</code></pre>
    <p>
     安装完成，重新编译python3。
    </p>
    <p>
     <strong>
      报错：RuntimeError: Compression requires the (missing) zlib module
     </strong>
    </p>
    <p>
     需要安装zlib模块，安装完成，重新编译python3
    </p>
    <pre><code class="language-bash">[root@xincai ~]# yum install -y zlib
[root@xincai ~]# yum install -y zlib-devel</code></pre>
    <p>
     <strong>
      错误：Error: Please make sure the libxml2 and libxslt development packages are installed.
     </strong>
    </p>
    <pre><code class="language-bash">[root@xincai ~]# yum install -y libxml2
[root@xincai ~]# yum install -y libxml2-devel
[root@xincai ~]# yum install -y libxslt
[root@xincai ~]# yum install -y libxslt-devel</code></pre>
