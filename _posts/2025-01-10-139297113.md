    <h2 style="margin-left:0;text-align:justify;">
     一、运维概述
    </h2>
    <h3 style="margin-left:0;text-align:justify;">
     1、运维岗位的收入情况
    </h3>
    <p class="img-center">
     <img alt="" height="299" src="https://i-blog.csdnimg.cn/blog_migrate/9e81cb3ed32888907633bbea09018a2a.png" width="534"/>

    <h3 style="margin-left:0;text-align:justify;">
     2、运维的职位定义
    </h3>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      什么是运维？
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     在技术人员之间，一致对运维有一个开玩笑的认知：运维就是修电脑的、装网线的、背锅的岗位。
    </p>
    <p style="margin-left:0;text-align:justify;">
     其实不然，运维是一个非常广泛的定义，
     <strong>
      <span style="color:#FF0000;">
       在不同的公司不同的阶段有着不同的职责与定位
      </span>
     </strong>
     ，如果以operation（运维英文）字面的含义去理解，认为就是敲几行操作命令的工作，那就错了。
    </p>
    <p style="margin-left:0;text-align:justify;">
     对于初创公司，运维工程师的工作可能需要从申请域名开始，购买/租用服务器,上架，调整网络设备的设置，部署操作系统和运行环境，部署代码，设计和部署监控，防止漏洞和攻击等等。（基础运维）
    </p>
    <p style="margin-left:0;text-align:justify;">
     对于大型的公司，对于运维工作的要求越来越高，也催生了更细化的运维分工：从大的方向，可以分为网站运维，基础运维、系统运维、网络运维、数据库运维、IT运维、运维开发、大数据运维、云计算运维、运维安全、
     <strong>
      <span style="color:#FF0000;">
       运维架构师、运维专家
      </span>
     </strong>
     等方向。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <h3 style="margin-left:0;text-align:justify;">
     3、运维的诞生
    </h3>
    <p style="margin-left:0;text-align:justify;">
     从最早的网管开始，九十年代初期，互联网欠发达的时候，还少有“服务器”的概念，电脑价格昂贵，大多数人并不具备在家上网的条件，网吧应运而生。
    </p>
    <p style="margin-left:0;text-align:justify;">
     网吧的电脑、猫等设备需要进行日常维护，于是“网管”岗位应运而生，这就是早起运维的雏形。
    </p>

    <h3 style="margin-left:0;text-align:justify;">
     5、运维行业前景（了解）
    </h3>
    <p style="margin-left:0;text-align:justify;">
     a. 从行业角度来看，随着中国互联网的高速发展、网站规模越来越来大、架构越来越复杂，对专职网站运维工程师、网站架构师的要求会越来越急迫，特别是对有经验的优秀运维人才需求量大，
     <strong>
      <span style="color:#FF0000;">
       而且是越老越值钱
      </span>
     </strong>
     。
    </p>
    <p style="margin-left:0;text-align:justify;">
     b. 从个人角度，运维工程师技术含量及要求会越来越高，同时也是对公司应用、架构最了解最熟悉的人，
     <strong>
      <span style="color:#FF0000;">
       越来越得到重视
      </span>
     </strong>
     。
    </p>
    <p style="margin-left:0;text-align:justify;">
     c. 互运维是一个融合多学科（网络、系统、开发、安全、应用架构、存储等）的综合性技术岗位，给运维工程师提供了
     <strong>
      <span style="color:#FF0000;">
       一个很好的个人能力与技术的发展空间
      </span>
     </strong>
     。
    </p>
    <p style="margin-left:0;text-align:justify;">
     d. 运维工作的相关经验将会变得非常重要，而且也将成为个人的核心竞争力，
     <strong>
      <span style="color:#FF0000;">
       优秀的运维工程师具备很好的各层面问题的解决能力及方案提供、全局思考的能力
      </span>
     </strong>
     等。
    </p>
    <p style="margin-left:0;text-align:justify;">
     e. 由于运维岗位所接触的知识面非常广阔，
     <strong>
      <span style="color:#FF0000;">
       更容易培养或发挥出个人某些方面的特长或爱好
      </span>
     </strong>
     ，如内核、网络、开发、数据库等方面，可以做得非常深入精通、成为这方面的专家。
    </p>
    <p style="margin-left:0;text-align:justify;">
     f. 当前国内外对运维人才的需求非常迫切，运维工程师的薪资也水涨船高，
     <strong>
      <span style="color:#FF0000;">
       与研发、测试等技术部门持平，甚至超出
      </span>
     </strong>
     。
    </p>
    <h2 style="margin-left:0;text-align:justify;">
     二、企业运作模式
    </h2>
    <h3 style="margin-left:0;text-align:justify;">
     1、四大部门
    </h3>
    <p style="margin-left:0;text-align:justify;">
     四大部门：研发、测试、运维、（系统管理）
    </p>
    <p style="margin-left:0;text-align:justify;">
     流程走向：产品设计 →
     <span style="color:#FF0000;">
      项目研发
     </span>
     <span style="color:#FF0000;">
      →
     </span>
     <span style="color:#FF0000;">
      项目测试
     </span>
     <span style="color:#FF0000;">
      →
     </span>
     <span style="color:#FF0000;">
      运维
     </span>
     <span style="color:#FF0000;">
      →
     </span>
     <span style="color:#FF0000;">
      系统数据管理
     </span>
     <img alt="" height="513" src="https://i-blog.csdnimg.cn/blog_migrate/1a8ee4bcb6481d97cf19484e486c1ba8.png" width="471"/>
    </p>
    <p style="margin-left:0;text-align:justify;">
     职责描述：
    </p>
    <p style="margin-left:0;text-align:justify;">
     研发部：根据项目需求设计项目（程序员/java工程师/php工程师/码农）
    </p>
    <p style="margin-left:0;text-align:justify;">
     测试部：项目bug的测试（项目内测部门）
    </p>
    <p style="margin-left:0;text-align:justify;">
     运维部：负责项目的上线以及后期项目所在的服务器的维护
    </p>
    <p style="margin-left:0;text-align:justify;">
     系统管理部：在不同的公司架构中可能不一样，有的公司可能有这个部门，有的可能没有。其工作职责主要是去使用这个项目。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       铁三角：研发、测试、运维。
      </span>
     </strong>
    </p>

    </p>

    </p>
    <p style="margin-left:0;text-align:left;">
     运维在其中承担了承上启下的作用，衔接了研发阶段与生产阶段，属于其中最重要的一环。

    <h3 style="margin-left:0;text-align:justify;">
     5、重要概念
    </h3>
    <p style="margin-left:0;text-align:justify;">
     域名：也就是在浏览器地址栏中输入的那一串字母和数字的组合，例如：
     <a href="http://www.baidu.com" rel="nofollow" title="www.baidu.com">
      www.baidu.com
     </a>
     ，这个就是域名，
     <strong>
      域名主要是方便用户记忆
     </strong>
     。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       服务器
      </span>
     </strong>
     ：就是给
     <strong>
      用户提供服务的机器（电脑）
     </strong>
     。服务器可以分为四大类：
     <strong>
      <span style="color:#FF0000;">
       塔式服务器、机式服务器、刀片服务器、柜式服务器
      </span>
     </strong>
     。
     <img alt="" height="155" src="https://i-blog.csdnimg.cn/blog_migrate/b7ee002a575ce745a278b35d99554b17.png" width="699"/>
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       IP
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       地址
      </span>
     </strong>
     ：形式ipv4（常见）、ipv6（不考虑）。Ipv4形式：x.x.x.x，x有取值范围（第1位x取值1-255，从第二位开始0-255）。由于资源有限为了保证全球这么多台电脑都可以使用，ip可以分为公网/外网ip和私网/内网ip地址。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     公网：www（万维网），全世界都可以互相访问的网。
    </p>
    <p style="margin-left:0;text-align:justify;">
     局域网/内网/私网：在某个单元内部（家庭/教室/公司）能够进行互相访问的网络。
    </p>
    <h3 style="margin-left:0;text-align:justify;">
     6、普通台式机能否充当服务器
    </h3>
    <p style="margin-left:0;text-align:justify;">
     问题：普通台式机/笔记本能否充当服务器来使用？
    </p>
    <p style="margin-left:0;text-align:justify;">
     如果从
     <strong>
      <span style="color:#FF0000;">
       提供服务角度
      </span>
     </strong>
     来考虑的话，的确是可以充当服务器的，但是一般不会用普通电脑来充当服务器。原因如下：
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       稳定性不如专门服务器。（开机时间长之后，服务器保证稳定运行不卡机）
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       性能上不如专门的服务器。（硬件）
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     服务器标准上不及专门的服务器统一。
    </p>

     <span style="color:#FF0000;">
      Cpu
     </span>
     <span style="color:#FF0000;">

     </span>
     、鼠标、键盘
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <h3 style="margin-left:0;text-align:justify;">
     3、计算机
     <span style="color:#FF0000;">
      资源
     </span>
    </h3>
    <p style="margin-left:0;text-align:justify;">
     计算机资源分为2部分：
     <strong>
      <span style="color:#FF0000;">
       硬件资源、软件资源
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     硬件：看得见摸得着的设备。
    </p>
    <p style="margin-left:0;text-align:justify;">
     软件：一般由研发工程师开发的一些程序，一般会称之为应用软件。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      问题：为什么在键盘上敲按键会在屏幕上显示呢？
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     答：键盘（硬件）→ 驱动（键盘驱动程序） →
     <strong>
      <span style="color:#FF0000;">
       操作系统（软件）
      </span>
     </strong>
     →    显卡驱动        →                  显示器（硬件）
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       操作系统（
      </span>
      <span style="color:#FF0000;">
       OS
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       ，
      </span>
      <span style="color:#FF0000;">
       operation system
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       ）
      </span>
     </strong>
     属于
     <span style="color:#FF0000;">
      特殊
     </span>
     的软件。其他的一些应用软件都必须安装在操作系统中。
    </p>
    <h3 style="margin-left:0;text-align:justify;">
     4、操作系统
    </h3>
    <p style="margin-left:0;text-align:justify;">
     目前市面上常见操作系统有：
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       Windows
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       系统
      </span>
     </strong>
     ：由微软公司开发的一款
     <strong>
      <span style="color:#FF0000;">
       闭源
      </span>
     </strong>
     操作系统（用户操作系统、服务器操作系统），从最早的DOS版本到后期的win 95，win 97，win 98，win 2000，win 2003（服务器操作系统），win xp，win vista，win 7，win server 2008（服务器操作系统），win 8，win server 2012（服务器操作系统），win 10，win server 2016（服务器操作系统）。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       Mac
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       系统
      </span>
     </strong>
     ：由苹果公司开发的一款
     <strong>
      <span style="color:#FF0000;">
       闭源
      </span>
     </strong>
     操作系统。是借鉴openBSD操作系统开发出的操作系统。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      <span style="color:#FF0000;">
       Linux
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       操作系统
      </span>
     </strong>
     ：一款目前市面上占有率最多的服务器操作系统。其是一款
     <strong>
      <span style="color:#FF0000;">
       开源
      </span>
     </strong>
     操作系统。
    </p>
    <p style="margin-left:0;text-align:justify;">
     Linux有很多的分支，常见的分支有：乌班图（Ubuntu）、CentOS、RedHat、SUSE、Debian等等。但是
     <strong>
      <span style="color:#FF0000;">
       Linux
      </span>
     </strong>
     <strong>
      <span style="color:#FF0000;">
       目前在世界上最大的分支是安卓
      </span>
     </strong>
     （谷歌研发的Linux分支）。
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      开源：
     </strong>
     开放源代码，
     <strong>
      <span style="color:#FF0000;">
       开源不意味着免费
      </span>
     </strong>
     （可能会服务收费）。
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      闭源：
     </strong>
     不开放源代码，其他人是看不到源代码的，有部分闭源软件是收费的，如windows的系统、office。
    </p>
    <h3 style="margin-left:0;text-align:justify;">
     5、为什么需要使用Linux操作系统
    </h3>
    <p style="margin-left:0;text-align:justify;">
     问题：windows既然可以使用傻瓜式的方式进行操作，例如使用ctrl+c表示复制，ctrl+v表示粘贴等，为什么还需要使用/学习Linux系统？
    </p>
    <p style="margin-left:0;text-align:justify;">
    </p>
    <p style="margin-left:0;text-align:justify;">
     ①
     <strong>
      性能问题
     </strong>
     ，windows服务器操作系统不如Linux高；
    </p>
    <p style="margin-left:0;text-align:justify;">
     ②
     <strong>
      稳定性问题
     </strong>
     ：
    </p>
    <p style="margin-left:0;text-align:justify;">
     底层架构：Linux更加稳定，其开机时间可以达到好几年不关机；
    </p>
    <p style="margin-left:0;text-align:justify;">
     开源：因为开源，人人都可以看到源代码，就可以为其提供自己的补丁，补丁可以提高
     <strong>
      <span style="color:#FF0000;">
       稳定性和安全性
      </span>
     </strong>
     ；
    </p>
    <p style="margin-left:0;text-align:justify;">
     ③
     <strong>
      安全性问题
     </strong>
     ：
    </p>
    <p style="margin-left:0;text-align:justify;">
     Linux操作系统，相对于windows操作系统要更加安全；
    </p>
    <p style="margin-left:0;text-align:justify;">
     ④
     <strong>
      远程管理方面
     </strong>
     ：
    </p>
    <p style="margin-left:0;text-align:justify;">
     Windows不及Linux操作高效。
    </p>
    <p style="margin-left:0;text-align:justify;">
     ⑤
     <strong>
      服务器价格昂贵的
     </strong>
     ，需要对资源进行充分利用，充分把计算机资源用到项目上（访问并发、性能），而不是把资源浪费在图形化界面或者方便程度上；
    </p>
