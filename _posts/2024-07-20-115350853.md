    <h2>
     <a id="_7">
     </a>
     一、嵌入式系统
    </h2>
    <p>
     <strong>
      嵌入式系统的特点：
     </strong>
     <br/>
     专用型、隐蔽型、资源受限、高可靠性、软件固化、实时性
     <br/>
     <strong>
      嵌入式系统CPU的特点：
     </strong>
     <br/>
     支持实时处理、低功耗、结构可扩展性、集成了测试电路
     <br/>
     （CPU范围很广，4，8，32，64位都属于嵌入式系统CPU的范围）
     <br/>
     <strong>
      影响CPU性能
     </strong>
     的主要因素：
     <br/>
     主频、指令系统、高速缓冲存储器容量和结构、逻辑结构
     <br/>
     CPU的时钟频率已经超过10GHZ（不是工作频率）
     <br/>
     <strong>
      嵌入式系统硬件
     </strong>
     ：
     <br/>
     <strong>
      硬件主体
     </strong>
     是 中央处理器和存储器
     <br/>
     <strong>
      软件主体
     </strong>
     是 操作系统和应用软件
     <br/>
     <strong>
      软硬件的复杂程度
     </strong>
     可分为低端系统、中端系统、高端系统
    </p>
    <p>
     嵌入式
     <strong>
      最小硬件系统
     </strong>
     组成：
     <br/>
     （1） 时钟电路
     <br/>
     （2） 电源电路
     <br/>
     （3） 存储器
     <br/>
     （4） 复位电路
     <br/>
     （5） 调试测试接口
    </p>
    <p>
     <strong>
      高速组件
     </strong>
     采用的标准是
     <strong>
      AHB
     </strong>
     ，
     <strong>
      低速组件
     </strong>
     采用的标准是
     <strong>
      APB
     </strong>
    </p>
    <p>
     <strong>
      中央处理器
     </strong>
     主要由运算器、控制器、寄存器和高速缓冲区组成（高速缓冲区（Cache）增加少量的成本，就能提高性能，CPU使用最频繁的少量代码和数据放在高速缓冲区）
    </p>
    <p>
     <strong>
      系统所使用的存储器的存储特性
     </strong>
     可分为RAM(随机存取存储器)和ROM（只读存储器）
     <br/>
     <strong>
      存储器的物理位置
     </strong>
     可分为 片内存储器、片外存储器、外部存储器
    </p>
    <p>
     <strong>
      储存结构
     </strong>
     可分为冯·诺依曼结构和哈佛结构 （ARM不都是哈佛结构）
    </p>
    <p>
     <strong>
      体系结构
     </strong>
     可分为CISC(复杂指令集结构)和RISC(精简指令集结构) （ARM都是RISC）
     <br/>
     <strong>
      字长
     </strong>
     可分为：8位、16位、32位、64位
     <br/>
     <strong>
      不同内核系列
     </strong>
     可分为：51、ARM等
    </p>
    <h2>
     <a id="ARM_42">
     </a>
     二、ARM处理器
    </h2>
    <p>
     ARM处理器在ARM11后可分为
     <strong>
      A系列、R系列、M系列
     </strong>
     <br/>
     Cortex-A—面向性能密集型系统的应用处理器内核
     <br/>
     Cortex-R—面向实时应用的高性能内核
     <br/>
     Cortex-M—面向各类嵌入式应用的微控制器内核
    </p>
    <p>
     ARM处理器有三种
     <strong>
      工作状态
     </strong>
     分为：
     <br/>
     ARM状态、Thumb及Thumb-2状态（R、M系列支持）、调试状态
     <br/>
     ARM状态工作在32位指令状态、Thumb工作在16位指令状态
    </p>
    <p>
     <strong>
      MPU
     </strong>
     是ARM处理器的
     <strong>
      内存保护
     </strong>
     单位
     <br/>
     <strong>
      MMU
     </strong>
     是ARM处理器的
     <strong>
      内存管理
     </strong>
     单位
    </p>
    <p>
     ARM处理器的
     <strong>
      工作模式
     </strong>
     ：
     <br/>
     1、用户模式(Usr)：用于正常执行程序;
    </p>
    <p>
     2、快速中断模式(FIQ)：用于高速数据传输;
    </p>
    <p>
     3、外部中断模式(IRQ)：用于通常的中断处理;
    </p>
    <p>
     4、管理模式(svc)：操作系统使用的保护模式;
    </p>
    <p>
     5、数据访问终止模式(abt)：当数据或指令预取终止时进入该模式，可用于虚拟存储以及存储保护;
    </p>
    <p>
     6、系统模式(sys)：运行具有特权的操作系统任务;
    </p>
    <p>
     7、未定义指令中止模式(und)：当未定义的指令执行时进入该模式，可用于支持硬件。
    </p>
    <p>
     ARM处理器的
     <strong>
      中断异常
     </strong>
     ：
     <br/>
     <img alt="中断异常图" src="https://i-blog.csdnimg.cn/blog_migrate/1b918851314418dc9bc63c14971582a3.png"/>
    </p>
    <h2>
     <a id="AMBA_72">
     </a>
     三、AMBA总线协议
    </h2>
    <p>
     AMBA是由ARM公司研发推出的一种高级微控制器总线架构(Advanced Microcontroller Bus Architecture)。
    </p>
    <p>
     按照AMBA规范，以ARM内核为基础的嵌入式处理芯片采用系统总线与外围总线的层次结构构建片上系统
    </p>
    <p>
     AMBA包含了四种不同的
     <strong>
      总线标准
     </strong>
     （AHB、ASB、APB、AXI）
     <br/>
     其中
     <strong>
      低速总线 APB
     </strong>
     是连接
     <strong>
      外围总线
     </strong>
     、
     <strong>
      高速总线 AHB
     </strong>
     是连接
     <strong>
      系统总线
     </strong>
    </p>
    <p>
     连接AMBA的
     <strong>
      系统总线
     </strong>
     和
     <strong>
      外围总线
     </strong>
     的是
     <strong>
      桥接器
     </strong>
    </p>
    <p>
     AMBA有四个版本(ARM7和ARM9用的不是一个AHBA标准)
    </p>
    <h2>
     <a id="RTOS_84">
     </a>
     四、实时系统（RTOS）
    </h2>
    <p>
     RTOS的三个主要
     <strong>
      实时指标
     </strong>
     ：响应时间、吞吐量、生存时间
    </p>
    <p>
     RTOS响应时间的
     <strong>
      具体指标
     </strong>
     ：中断延迟时间、任务切换时间
    </p>
    <p>
     RTOS的
     <strong>
      属性
     </strong>
     ：时间约束性、可预测性、可靠性（确定性）
    </p>
    <p>
     按
     <strong>
      响应时间
     </strong>
     将
     <strong>
      实时系统
     </strong>
     分成三类：
     <br/>
     强实时系统：响应时间再毫秒级或微秒级
     <br/>
     普通实时系统：响应时间一般在几秒
     <br/>
     弱实时系统：响应时间一般在数十秒
    </p>
    <p>
     实时系统的属性包括可靠性、可预测性、时间约束性
    </p>
    <p>
     任务执行时间超过截止时间后系统的
     <strong>
      收益变为负
     </strong>
     ，称为
     <strong>
      硬实时操作系统
     </strong>
     <br/>
     任务执行时间超过截止时间后系统的
     <strong>
      收益下降，但仍为正
     </strong>
     ，这个叫
     <strong>
      软实时操作系统
     </strong>
    </p>
    <h2>
     <a id="_100">
     </a>
     五、操作系统的分类
    </h2>
    <p>
     常见分类：
     <br/>
     手机：Android、IOS
     <br/>
     电脑：UNIX、LINUX、MacOS、Windows、UNIX
    </p>
    <p>
     <strong>
      按内核分类：
     </strong>
     <br/>
     微内核操作系统：Symbian、VxWorks、uC/OS-II、QNX、IOS微内核（各功能切换之间开销较大）
     <br/>
     单（宏）内核 ：Andriod OS WinCE Linux Mac OS DOS
    </p>
    <h3>
     <a id="1uCOSII_108">
     </a>
     1、uC/OS-II
    </h3>
    <p>
     uC/OS-II是
     <strong>
      抢占式，微内核
     </strong>
     的操作系统
    </p>
    <p>
     可管理
     <strong>
      64
     </strong>
     个任务，其中
     <strong>
      8
     </strong>
     个任务是
     <strong>
      系统
     </strong>
     提供的，
     <strong>
      56
     </strong>
     个是
     <strong>
      用户
     </strong>
     自己编写的
    </p>
    <p>
     μC/OS-II的任务的
     <strong>
      5种状态
     </strong>
     ：
     <br/>
     休眠态(Dormant)、就绪态(Ready)、运行态(Running)、挂起态(Pending)或等待态(Waiting)、被中断态(Interrupted)
    </p>
    <p>
     μC/OS-II的
     <strong>
      软件结构
     </strong>
     ：应用软件层、API层、内核层、设备驱动层
    </p>
    <p>
     uC/OS-II 的
     <strong>
      事件控制
     </strong>
     块有4种实现函数：
     <br/>
     OSSemCreate()、 OSMutexCreate() 、
     <br/>
     cOSMboxCreate() 、OSQCreate()
    </p>
    <p>
     μC/OS-II系统内核提供的
     <strong>
      基本功能
     </strong>
     ：
     <br/>
     任务管理、任务间通信与同步、任务调度、时间管理、内存管理
    </p>
    <p>
     <strong>
      移植
     </strong>
     μC/OS-II工作需要改：
     <br/>
     OS_CPU.H、OS_CPU_A.S、OS_CPU_C.C
    </p>
    <h3>
     <a id="2VxWorks_128">
     </a>
     2、VxWorks
    </h3>
    <p>
     VxWorks是
     <strong>
      硬实时，微内核
     </strong>
     操作系统
    </p>
    <p>
     移植VxWorks操作系统有4种操作类型：
     <br/>
     宿主机移植、体系结构移植、目标板移植、升级以前Tornado集成开发环境版本的BSP到最新版本
    </p>
    <p>
     VxWorks不开源
    </p>
    <h3>
     <a id="3Linux_136">
     </a>
     3、Linux
    </h3>
    <p>
     Linux，全称GNU/Linux，是一种免费使用和自由传播的
     <strong>
      类UNIX
     </strong>
     操作系统，其内核由林纳斯·本纳第克特·托瓦兹于
     <strong>
      1991年10月5日
     </strong>
     首次发布，它主要受到
     <strong>
      Minix和Unix思想
     </strong>
     的启发，是一个基于
     <strong>
      POSIX
     </strong>
     的
     <strong>
      多用户、多任务、支持多线程和多CPU
     </strong>
     的操作系统。
    </p>
    <p>
     它能运行主要的
     <strong>
      Unix
     </strong>
     工具软件、应用程序和网络协议。它支持32位和64位硬件。
    </p>
    <p>
     Linux继承了Unix以网络为核心的设计思想，是一个性能稳定的多用户网络操作系统。
    </p>
    <p>
     Linux的运行效率低于Unix
    </p>
    <p>
     嵌入式
     <strong>
      Linux系统主要是由
     </strong>
     用户进程、OS服务组件、Linux内核组成
    </p>
    <p>
     嵌入式Linux进程间
     <strong>
      通信机制
     </strong>
     ：
     <br/>
     信号、管道、消息队列、信号量、共享内存套接字
    </p>
    <p>
     Linux内核由
     <strong>
      5个子系统
     </strong>
     组成：
     <br/>
     进程调度(SCHED)、内存管理(MM)、虚拟文件系统(VFS)、
     <br/>
     网络接口(NET)、进程间通信(IPC)
    </p>

     <a id="4IOS_153">

     IOS的前身是
     <strong>
      UNIX-BSD
     </strong>
    </p>
    <p>
     IOS系统结构的四个层次：
     <br/>
     核心操作系统
     <br/>
     核心服务层
     <br/>
     媒体层
     <br/>

    </p>
    <h2>
     <a id="_161">
     </a>
     六、图像、音频、视频类
    </h2>
    <p>
     <strong>
      图像：
     </strong>
     <br/>
     <strong>
      图像获取
     </strong>
     步骤：
     <br/>
     扫描、分色、取样、量化
    </p>
    <p>
     图像的数据量 = 图像水平分辨率 * 图像垂直分辨率 * 像素深度/ 8
    </p>
    <p>
     <strong>
      GIF
     </strong>
     图像文件格式颜色数目较少，文件特别小，适合互联网传输
     <br/>
     <strong>
      JPEG
     </strong>
     图像文件格式是静止图像数据压缩编码的国际标准，它在相机和互联网中被广泛使用
     <br/>
     <strong>
      音频：
     </strong>
     <br/>
     <strong>
      音频处理
     </strong>
     步骤：
     <br/>
     取样、量化、编码
     <br/>
     <strong>
      视频：
     </strong>
     <br/>
     有四种压缩编码 MPEG-1、MPEG-2、MPEG-3、MPEG-4
     <br/>
     MP3音乐格式采用过MPEG-1
     <br/>
     目前数字有线电视所传输的数字视频采用的压缩编码是 MPEG-2
     <br/>
     MPEG-3会导致画面轻微扭曲
    </p>
    <p>
     HDMI（高清晰度多媒体接口 ）
     <br/>
     HDMI是一种数字化音频/视频接口，适合影像传输
     <br/>
     HDMI最高数据传输速度为10.2Gbps
    </p>
    <p>
     量化概念：
     <br/>
     量化是把每个样本的模拟量转换成数字量来表示
     <br/>
     量化过程往往也称为A/D转换（模数转换）
     <br/>
     音频信号量化后的样本一般用8位、12位或16位的二进制整数表示（称为“量化精度”）
     <br/>
     量化精度越高，声音的保真度越好，量化精度越低，声音的保真度越差
    </p>
    <h2>
     <a id="_190">
     </a>
     六、编码
    </h2>
    <h3>
     <a id="1GB2312_191">
     </a>
     1、GB2312
    </h3>
    <p>
     GB2312的所有字符在GB18030全部存在且编码相同
     <br/>
     GB2312采用双字节进行存储和传输汉字
     <br/>
     GB2313大约有6000多个汉字 以及一些图片符号（包括俄文 日本假字 拉丁文 汉语拼音 希腊文等）
    </p>
    <h3>
     <a id="2GB18030_195">
     </a>
     2、GB18030
    </h3>
    <p>
     GB18030采用双字节或四字节存储和传输汉字
     <br/>
     GB18030与UCS/Unicode均包含中日韩统一汉字（CJK）约7万多个
     <br/>
     GB18030中的汉字与UCS/Unicode中的汉字编码不相同
    </p>
    <h3>
     <a id="3UTF8_199">
     </a>
     3、UTF-8
    </h3>
    <p>
     Unicode/UTF-8采用三个字节存储和传输汉字
     <br/>
     UTF-8和UTF-16是实现UCS/Unicode的两种不同的编码方案
    </p>
    <h3>
     <a id="4UTF16_202">
     </a>
     4、UTF-16
    </h3>
    <p>
     Unicode/UTF-8采用三个字节存储和传输汉字
     <br/>
     Unicode/UTF-16采用双字节存储和传输汉字
     <br/>
     UTF-16 是双字节可变长编码
     <br/>
     UTF-8和UTF-16是实现UCS/Unicode的两种不同的编码方案
     <br/>
     #七
    </p>
    <h2>
     <a id="_209">
     </a>
     七、引导加载程序
    </h2>
    <p>
     引导加载程序，英文简写是 Boot。
    </p>
    <h3>
     <a id="1BSP_211">
     </a>
     1、板机支持包（BSP）
    </h3>
    <p>
     BSP是介于主板硬件和操作系统中驱动层程序之间的一层，一般认为它属于操作系统一部分，主要是实现对操作系统的支持，为上层的 驱动程序提供访问硬件设备寄存器的函数包，使之能够更好的运行于硬件主板。
    </p>
    <p>
     BSP主要功能为屏蔽硬件，提供操作系统及硬件驱动，具体功能包括：
     <br/>
     单板硬件初始化，主要是CPU的初始化，为整个软件系统提供底层硬件支持
     <br/>
     为操作系统提供设备 驱动程序和系统 中断服务程序
     <br/>
     定制操作系统的功能，为软件系统提供一个实时多任务的运行环境
     <br/>
     初始化操作系统，为操作系统的正常运行做好准备。
    </p>
    <p>
     U–Boot是被认为功能全面 具有灵活性 以及应用广泛的开放源码板级支持包。
     <br/>
     U-Boot是一种通用的引导加载程序，对PowerPC系列处理器支持最为丰富，对Linux操作系统的支持最为完善。
    </p>
    <h3>
     <a id="2HAL_222">
     </a>
     2、硬件抽奖层（HAL）
    </h3>
    <p>
     硬件抽象层是位于操作系统 内核与硬件电路之间的接口层，其目的在于将硬件抽象化。它隐藏了特定平台的硬件接口细节,为操作系统提供虚拟硬件平台,使其具有硬件无关性,可在多种平台上进行移植。
    </p>
    <p>
     HAL的作用 提供标准库 提供标准接口 在main（）函数之前就完成了初始化
    </p>

     8、NOR flash 和 NAND flash 是市场上主要的闪存技术、NOR以字节读取 NAND以行（页）读取 NAND具有很长的使用寿命、存储卡和U盘均使用的是NAND

     9、触摸屏分为电阻屏和电容屏（电阻屏便宜，单点触摸、不支持滑动操作。电容屏支持多点触摸、滑动操作）

     11、编译器GCC（参数-c，-g）、调试器GDB(参数file,kill,run,watch)

     14、JTAG支持在线编程、有5根信号线、和系统总线（APB）相连、JTAG调试不占用IO口或存储器、51单片机不支持JTAG调试
