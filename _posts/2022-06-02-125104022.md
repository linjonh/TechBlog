    <h2>
     1. 解决方案对比
    </h2>
    <p>
     目前前端技术领域已有多种成熟的
     <a href="https://blog.csdn.net/weixin_50836306/article/details/124498648?spm=1001.2014.3001.5501" title="微前端">
      微前端
     </a>
     解决方案：
    </p>
    <ol>
     <li>
      iframe
     </li>
     <li>
      Web Components
     </li>
     <li>
      ESM
     </li>
     <li>
      single-spa
     </li>
     <li>
      qiankun
     </li>
     <li>
      EMP
     </li>
     <li>
      MicroApp
     </li>
    </ol>
    <p>
     各解决方案对比如下：
    </p>
    <ul>
     <li>
      iframe 可以直接加载其他应用，但无法做到单页导致许多功能无法正常在主应用中展示。
     </li>
     <li>
      web Components 和 ESM（ES Module）是浏览器提供给开发者的能力，能在单页中实现微前端，不过后者需要做好代码隔离，并且他们都是浏览器的新特性，都存在兼容性问题，微前端方面的探索也不成熟，只能作为面向未来的微前端手段。
     </li>
     <li>
      <a href="https://zh-hans.single-spa.js.org/docs/getting-started-overview/" rel="nofollow" title="single-spa">
       single-spa
      </a>
      是通过监听 url change 事件，在路由变化时匹配到渲染的子应用并进行渲染，这个思路也是目前实现微前端的主流方式。同时 single-spa 要求子应用修改渲染逻辑并暴露出三个方法：bootstrap、mount、unmount，分别对应初始化、渲染和卸载，这也导致子应用需要对入口文件进行修改。
     </li>
     <li>
      <a href="https://qiankun.umijs.org/zh/guide" rel="nofollow" title="qiankun">
       qiankun
      </a>
      是基于 single-spa 进行封装，所以这些特点也被 qiankun 继承下来，并且需要对 webpack 配置进行一些修改。qiankun 基本上可以称为单页版的 iframe，具有沙箱隔离及资源预加载的特点，几乎无可挑剔。
     </li>
     <li>
      <a href="https://github.com/efoxTeam/emp" title="EMP">
       EMP
      </a>
      作为最年轻微前端解决方案，也是吸收了许多 web 优秀特性才诞生的，它在实现微前端的基础上，扩充了跨应用状态共享、跨框架组件调用、远程拉取ts声明文件、动态更新微应用等能力。同时，细心的小伙伴应该已经发现，EMP 能做到第三方依赖的共享，使代码尽可能地重复利用，减少加载的内容。
     </li>
     <li>
      <a href="https://github.com/micro-zoe/micro-app" title="MicroApp">
       MicroApp
      </a>
      并没有沿袭 single-spa 的思路，而是借鉴了 WebComponents 的思想，通过 CustomElement 结合自定义的 ShadowDom，将微前端封装成一个类 WebComponents 组件，从而实现微前端的组件化渲染。并且由于自定义 ShadowDom 的隔离特性，MicroApp 不需要像 single-spa 和 qiankun 一样要求子应用修改渲染逻辑并暴露出方法，也不需要修改 webpack 配置，是目前市面上接入微前端成本最低的方案。
      <img alt="" height="1200" src="https://i-blog.csdnimg.cn/blog_migrate/1dcdf50140104e34d05e521420af71c0.png" width="1200">
      </img>
     </li>
    </ul>
    <h2 id="2.%20MicroApp%20%E8%A7%A3%E5%86%B3%E6%96%B9%E6%A1%88%E7%A4%BA%E4%BE%8B">
     2. MicroApp 解决方案示例
    </h2>
    <h3 id="2.1%C2%A0%E5%BE%AE%E5%BA%94%E7%94%A8%E6%B3%A8%E5%86%8C">
     2.1 微应用注册
    </h3>
    <p>
     在项目主文件 main.js 中采用以下方式注册微应用：
    </p>
    <pre><code class="language-javascript">if(!window.__MICRO_APP_ENVIRONMENT__){
  microApp.start()
}

// 微应用模式
let app = null
export async function mount (props) {
  app = new Vue({
    router,
    store,
    render: h =&gt; h(App)
  }).$mount('#app');
  
  window.microApp.addDataListener((data) =&gt; {
    // 当基座下发跳转指令时进行跳转
    if (data.path) {
      app.$router.push(data.path)
    }
  },true)
}

// 子应用独立运行时逻辑
export async function mountInChild(props) {
  new Vue({
    router,
    store,
    render: h =&gt; h(App)
  }).$mount('#app')
}

// 卸载应用
export async function unmount () {
  app.$destroy()
  app.$el.innerHTML = ''
  app = null
}

// 微前端环境下，注册mount和unmount方法
// window.__MICRO_APP_ENVIRONMENT__ 为微应用标识
if (window.__MICRO_APP_ENVIRONMENT__) {
  window[`micro-app-${window.__MICRO_APP_NAME__}`] = { mount, unmount }
} else {
  // 非微前端环境直接渲染
  mountInChild()
}</code></pre>
    <h3 id="2.2%C2%A0%E5%BE%AE%E5%BA%94%E7%94%A8%E8%8E%B7%E5%8F%96%20token%20%E4%BF%A1%E6%81%AF">
     2.2 微应用获取 token 信息
    </h3>
    <p>
     1. 获取 token
    </p>
    <p>
     主应用通过单点登录方式登录，将 token 信息存入
     <span style="color:#0d0016;">
      <span style="background-color:#cccccc;">
       sessionStorage
      </span>
     </span>
     的
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        __storejs_pro___ACCESS_TOKEN
       </span>
      </strong>
     </span>
     中：
    </p>
    <pre><code class="language-javascript">// 子应用获取 token
sessionStorage.getItem('__storejs_pro___ACCESS_TOKEN')
</code></pre>
    <p>
     2. 微应用 token 过期处理
    </p>
    <p>
     微应用 token 过期需要做响应拦截，报告主应用 token 过期，报告主应用方法如下：
    </p>
    <pre><code class="language-javascript">if(window.__MICRO_APP_ENVIRONMENT__){
    window.microApp.dispatch({action:'logout'})
}
</code></pre>
    <p>
     主应用接受到微应用的消息，弹出重新登录弹窗，用户点击登录可实现重新登录。
    </p>
    <h3 id="2.3%C2%A0%E5%BE%AE%E5%BA%94%E7%94%A8%E5%85%81%E8%AE%B8%E8%B7%A8%E5%9F%9F%E8%AE%BF%E9%97%AE%E8%AE%BE%E7%BD%AE">
     2.3 微应用允许跨域访问设置
    </h3>
    <p>
     微应用需要设置允许跨域访问主应用才能加载微应用的资源。
    </p>
    <p>
     - 开发环境：
    </p>
    <p>
     需要设置打包配置文件的 devServer 的 headers 属性
    </p>
    <pre><code class="language-javascript">headers: {
    'Access-Control-Allow-Origin': '*',
},
</code></pre>
    <p>
     - 生产环境：
    </p>
    <p>
     微应用nginx 需要配置：
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        'Access-Control-Allow-Origin': '主应用域名'
       </span>
      </strong>
     </span>
     或者
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        'Access-Control-Allow-Origin': '*'
       </span>
      </strong>
     </span>
    </p>
    <h3 id="2.4%C2%A0%E5%BE%AE%E5%BA%94%E7%94%A8%E8%8E%B7%E5%8F%96cookie">
     2.4 微应用获取cookie
    </h3>
    <p>
     微应用nginx 需要配置：
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        'Access-Control-Allow-Origin': '主应用域名'
       </span>
      </strong>
     </span>
     ，主应用域名
     <span style="color:#0d0016;">
      <span style="background-color:#fbd4d0;">
       不可为
      </span>
      <strong>
       <span style="background-color:#fbd4d0;">
        '*'
       </span>
      </strong>
     </span>
    </p>
    <p>
     微应用nginx需要开启：
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        'Access-Control-Allow-Credentials':true
       </span>
      </strong>
     </span>
    </p>
    <p>
     微应用接口请求需配置
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        baseURL: '
       </span>
      </strong>
      <strong>
       <span style="background-color:#cccccc;">
        微应用的服务地址
       </span>
      </strong>
      <strong>
       <span style="background-color:#cccccc;">
        '
       </span>
      </strong>
     </span>
    </p>
    <p>
     微应用接口请求需允许
     <span style="color:#0d0016;">
      <strong>
       <span style="background-color:#cccccc;">
        withCredentials:true
       </span>
      </strong>
     </span>
    </p>
    <h3 id="2.5%20%E5%BE%AE%E5%BA%94%E7%94%A8%E5%85%A8%E5%B1%80%E9%80%9A%E4%BF%A1">
     2.5 微应用全局通信
    </h3>
    <p>
     - 发送方
    </p>
    <pre><code class="language-javascript">if (window.__MICRO_APP_ENVIRONMENT__) {
    window.microApp.setGlobalData( data )
}
</code></pre>
    <p>
     - 接收方
    </p>
    <pre><code class="language-javascript">if (window.__MICRO_APP_ENVIRONMENT__) {
    const data = window.microApp.getGlobalData()
}
</code></pre>
    <h3 id="2.6%20%E8%B5%84%E6%BA%90%E5%A4%8D%E7%94%A8">
     2.6 资源复用
    </h3>
    <p>
     1. 父应用资源的暴露
    </p>
    <p>
     - 使用 url 引入的方式调用模块包，默认注册位置 public/index.html 入口文件
    </p>
    <pre><code class="language-javascript">&lt;script src="https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.min.js"&gt;&lt;/script&gt;</code></pre>
    <p>
     - 于父框架微前端初始化时注册需要暴露的资源，默认注册位置 main.js
    </p>
    <pre><code class="language-javascript">if(!window.__MICRO_APP_ENVIRONMENT__){
  microApp.start(
    {
      globalAssets: {
        js: ['https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.min.js'], // js地址
        // css: ['css地址1', 'css地址2', ...], // css地址
      }
    }
  )
}
</code></pre>
    <p>
     2. 子应用使用父级资源
    </p>
    <p>
     于入口文件通过 url 方式引入和父框架一样地址的资源。
    </p>
    <p>
     子应用相同地址的资源匹配到父框架时，便能直接使用父框架的资源，而无需自己加载，起到资源复用的作用。
    </p>
    <pre><code class="language-javascript">&lt;script src="https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.min.js"&gt;&lt;/script&gt;</code></pre>
