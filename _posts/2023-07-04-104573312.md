    <h3 id="%E4%B8%80%E3%80%81%E4%BB%80%E4%B9%88%E6%98%AF%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84">
     <span style="color:#d93e2b;">
      一、什么是数据结构
     </span>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      数据结构是一门研究非数值计算的程序设计问题中的操作对象，以及它们之间的关系和操作等相关问题的学科。简单的来说数据结构就是关系，就是数据元素相互之间存在的一种或多种特定关系的集合。传统上，我们把数据结构分为逻辑结构和物理结构：
     </span>
    </p>
    <ul>
     <li>
      <p style="text-indent:33px;">
       <span style="color:#414141;">
        逻辑结构：是指数据对象中数据元素之间的相互关系，也是我们今后最需要关注和讨论的问题。
       </span>
      </p>
     </li>
     <li>
      <p style="text-indent:33px;">
       <span style="color:#414141;">
        物理结构：是指数据的逻辑结构在计算机中的存储形式。
       </span>
      </p>
     </li>
    </ul>
    <h4 id="1.%E6%95%B0%E6%8D%AE%E7%9A%84%E9%80%BB%E8%BE%91%E7%BB%93%E6%9E%84">
     <span style="color:#d93e2b;">
      1.数据的逻辑结构
     </span>
    </h4>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      集合结构
     </span>
     ：
     <img alt="" height="105" src="https://i-blog.csdnimg.cn/blog_migrate/d579a04444d13b31d6c9272644e62c2c.png" width="162"/>
    </p>
    <p style="text-indent:33px;">
     树形结构：
     <img alt="" height="146" src="https://i-blog.csdnimg.cn/blog_migrate/84e0976b6ba9f027ede1e2c440d2a8fd.png" width="191"/>

     线性结构：
     <img alt="" height="84" src="https://i-blog.csdnimg.cn/blog_migrate/aa649ff043c431d01c732b16df10eb49.png" width="239"/>

     图形结构：
     <img alt="" height="164" src="https://i-blog.csdnimg.cn/blog_migrate/25a00aca807524e317f5ae97964b4d55.png" width="176"/>

    <h4 id="2.%E6%95%B0%E6%8D%AE%E7%9A%84%E7%89%A9%E7%90%86%E7%BB%93%E6%9E%84">
     <span style="color:#d93e2b;">
      2.数据的物理结构
     </span>
    </h4>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      根据物理结构的定义，我们实际上研究的的就是如何把数据元素存储到计算机的存储器中
     </span>
     <span style="color:#414141;">
      。存储器主要是针对内存而言的，像硬盘、软盘、光盘等外部存储器的数据组织通常用文件结构来描述。数据元素的存储结构形式有两种：顺序存储和链式存储。
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#414141;">
       顺序存储结构：是把数据元素存放在地址连续的存储单元里，其数据间的逻辑关系和物理关系是一致的
      </span>
      <span style="color:#414141;">
       。
      </span>
     </li>
    </ul>
    <p style="text-indent:33px;">
     <img alt="" height="84" src="https://i-blog.csdnimg.cn/blog_migrate/aa649ff043c431d01c732b16df10eb49.png" width="239"/>
    </p>
    <ul>
     <li>
      <span style="color:#414141;">
       链式存储结构：是把数据元素存放在任意的存储单元里，这组存储单元可以是连续的，也可以是不连续的。
      </span>
     </li>
    </ul>
    <p style="text-indent:33px;">
     <img alt="" height="148" src="https://i-blog.csdnimg.cn/blog_migrate/00c961a8741c8aa8ac819dc472855c55.png" width="553"/>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      很显然，这样说的话链式存储结构的数据元素存储关系并不能反映其逻辑关系，因此需要用一个指针存放数据元素的地址，这样子通过地址就可以找到相关联数据元素的位置
     </span>
    </p>
    <h3 id="%E4%BA%8C%E3%80%81%E4%BB%80%E4%B9%88%E6%98%AF%E7%AE%97%E6%B3%95">
     <span style="color:#d93e2b;">
      二、什么是算法
     </span>
    </h3>
    <h4 id="1.%E7%AE%97%E6%B3%95%E7%9A%84%E5%AE%9A%E4%B9%89">
     <span style="color:#d93e2b;">
      1.算法的定义
     </span>
    </h4>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示一个或多个操作
     </span>
     <span style="color:#414141;">
      。
     </span>
    </p>
    <pre><code>问题：1+2+…+99+100</code></pre>
    <p style="text-indent:33px;">
     直观上直接采用累加的解法：
    </p>
    <pre><code class="language-python">sum = 0
for i in range(1,101):
    sum += i
print(i)
</code></pre>
    <p style="text-indent:33px;">
     也可以换一种写法~
    </p>
    <pre><code class="language-python">begin = 1, end = 100
n = end – begin + 1
print((begin+end)*n/2)
</code></pre>
    <h4 id="2.%E7%AE%97%E6%B3%95%E7%9A%84%E7%89%B9%E6%80%A7">
     <span style="color:#d93e2b;">
      2.算法
     </span>
     <span style="color:#d93e2b;">
      的特性
     </span>
    </h4>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      算法具有五个基本特征：输入、输出、有穷性、确定性和可行性。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      输入：算法具有零个或多个输入。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      输出：算法具有零个或多个输出。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      有穷性：指算法在执行有限的步骤之后，自动
     </span>
     <span style="color:#414141;">
      结束而
     </span>
     <span style="color:#414141;">
      不会出现无限循环，并且每一个步骤在可接受的时间内完成。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      确定性：算法的每一个步骤都具有确定的含义，不会出现歧义。算法在一定条件下，只有一条执行路径，相同的输入只能有唯一的输出结果。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      可行性：算法的每一步都必须是可行的，也就是说，每一步都能够通过执行有限次数完成。
     </span>
    </p>
    <h4 id="3.%E7%AE%97%E6%B3%95%E8%AE%BE%E8%AE%A1%E7%9A%84%E8%A6%81%E6%B1%82">
     <span style="color:#d93e2b;">
      3.算法
     </span>
     <span style="color:#d93e2b;">
      设计的要求
     </span>
    </h4>
    <p style="text-indent:33px;">
     <span style="color:#414141;">
      算法并不是唯一的。也就是说同一个问题，可以有多种解决问题的算法，要灵活运用。
     </span>
    </p>
    <p style="text-indent:33px;">
     a)
     <span style="color:#414141;">
      正确性
     </span>
    </p>
    <p style="text-indent:33px;">
     b)
     <span style="color:#414141;">
      可读性
     </span>
    </p>
    <p style="text-indent:33px;">
     c)
     <span style="color:#414141;">
      健壮性
     </span>
    </p>
    <p style="text-indent:33px;">
     d)
     <span style="color:#414141;">
      时间效率高和存储量低（时间复杂度和空间复杂度）
     </span>
    </p>
