    <p>
     Oracle SQL Developer是Oracle官方出品的数据库管理工具。官方下载地址:
     <a href="http://www.oracle.com/technetwork/cn/developer-tools/sql-developer/downloads/index.html" rel="nofollow">
      Oracle SQL Developer
     </a>
     ，官方入门：
     <br/>
     <a href="http://www.oracle.com/technetwork/cn/topics/omwb-getstarted-092755-zhs.html#repo" rel="nofollow">
      官方入门
     </a>
    </p>
    <p>
     本文介绍使用Oracle SQL Developer迁移sql server至Oracle数据库，步骤如下：
    </p>
    <h2 id="一配置第三方连接">
     一、配置第三方连接
    </h2>
    <p>
     要使用 JDBC 访问数据库，必须使用 JDBC 驱动程序。默认情况下，Oracle SQL Developer是不包含sqlserver的数据库连接的。可从 Database 供应商（通常免费提供 JDBC 驱动程序下载）获取该驱动程序。
    </p>
    <p>
     jTDS 驱动程序可从
     <a href="http://jtds.sourceforge.net/" rel="nofollow">
      http://jtds.sourceforge.net/
     </a>
     获得。使用版本 1.2（可从
     <a href="http://sourceforge.net/project/showfiles.php?group_id=33291" rel="nofollow">
      http://sourceforge.net/project/showfiles.php?group_id=33291
     </a>
     下载）
     <br/>
     或者直接在maven仓库下载：
     <a href="http://mvnrepository.com/artifact/net.sourceforge.jtds/jtds" rel="nofollow">
      JTDS
     </a>
    </p>
    <p>
     1、打开Oracle SQL Developer，选择
     <code>
      工具--&gt;首选项--&gt;数据库--&gt;第三方 JDBC 驱动程序
     </code>
     ，如下：
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7e5a79e08808dccb4b351b0649f6a47e.png" title=""/>
    </p>
    <p>
     2、选择添加刚下载的jtds驱动文件：
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/41bbe9532a6bbd7f5c2da35b6bb03ce6.png" title=""/>
    </p>
    <p>
     现在我们再打开添加连接界面，就可以看到
     <code>
      SQLServer
     </code>
     和
     <code>
      Sybase
     </code>
     两个tab页了。
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/dc9d51b80e45c4a0b07c7b6f2d20fff1.png" title=""/>
    </p>
    <h2 id="二添加sqlserver数据库连接">
     二、添加sqlserver数据库连接
    </h2>
    <p>
     点绿色加号添加我们要移植的sqlserver数据库连接：
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5827ac86976d85999b91f03b66bafdb8.png" title=""/>
    </p>
    <h2 id="三创建移植信息库">
     三、创建移植信息库
    </h2>
    <p>
     Oracle SQL Developer迁移数据库，需要一个数据库模式来存储它为移植过程收集和转换的元数据。该信息库包含所需的 37 个表、8 个视图以及主键和索引，还包括触发器和 pl/sql 代码。建议为此配置一个专用数据库模式（尽管这不是必需的）。
    </p>
    <p>
     要在新模式中为元数据构建信息库，用户需要以下角色和系统特权：
    </p>
    <pre class="prettyprint"><code class="hljs sql">RESOURCE
<span class="hljs-operator"><span class="hljs-keyword">CREATE</span> <span class="hljs-keyword">SESSION</span>
<span class="hljs-keyword">CREATE</span> <span class="hljs-keyword">VIEW</span></span></code></pre>
    <p>
     1、通过命令行或plsql客户端使用管理员登录oracle，创建迁移用户(为了防止迁移过程中出现权限问题，我们直接赋予DBA权限)：
    </p>
    <pre class="prettyprint"><code class="hljs sql"><span class="hljs-operator"><span class="hljs-keyword">create</span> <span class="hljs-keyword">user</span> sgl_migra identified <span class="hljs-keyword">by</span> sgl_migra;</span>
<span class="hljs-operator"><span class="hljs-keyword">grant</span> dba <span class="hljs-keyword">to</span> sgl_migra;</span></code></pre>
    <p>
     2、在Oracle SQL Developer里点击绿色加号新增连接
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7e7f6862b8b690041cefa4e8f6f25a84.png" title=""/>
    </p>
    <p>
     连接用户是我们刚创建的
     <code>
      sgl_migra
     </code>
     用户：
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2322e5c5525348c37e3651247edd86dc.png" title=""/>
    </p>
    <p>
     3、右键创建的
     <code>
      sgl_migra
     </code>
     连接，点击
     <code>
      移植资料档案库--&gt;关联移植资料档案库
     </code>
     :
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8fa688e3d6e20639985a3f5c2703d1bb.png" title=""/>
    </p>
    <p>
     这个可能要一两分钟:
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fe3e55d39ccc7b0ad9f80d8869c3a701.png" title=""/>
    </p>
    <h2 id="四开始移植">
     四、开始移植
    </h2>
    <p>
     1、右键sqlserver连接里我们要移植的数据库,点击移植到
     <code>
      Oracle...
     </code>
     打开移植向导界面
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8306c2edab2b7e5c6f96f80f69838234.png" title=""/>
    </p>
    <p>
     向导界面如下：
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c259467dc432b184b9aec687d48c4f8f.png" title=""/>
    </p>
    <p>
     2、点击下一步，进入资料档案库设置界面，选择我们步骤二创建的移植信息库
     <code>
      sgl_migra
     </code>
     ,由于我们是新建的
     <code>
      sgl_migra
     </code>
     ，截断勾不勾都行。
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2e1b6527cd3e99b0a6c6bf3e0d187e41.png" title=""/>
    </p>
    <p>
     3、点击下一步设置输出目录
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/42ea1c0dc2d3626103df0370ebec4459.png" title=""/>
    </p>
    <p>
     4、点击下一步设置源数据库，选择我们创建的sqlserver连接
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/855489361585cebd30cb540cab1adce0.png" title=""/>
    </p>
    <p>
     5、点击下一步设置转换的数据库
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a1b289ccdb6ea5a31a385025a9f10c0f.png" title=""/>
    </p>
    <p>
     6、设置转换规则，一般默认即可
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/82654e590837878c21c9f610c43330d7.png" title=""/>
    </p>
    <p>
     7、选择要转换的对象
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/021439601ae60b4c859810e3c9bfbb42.png" title=""/>
    </p>
    <p>
     8、设置目标数据库
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a6f96c54f0a7f1082df8fa7a10f6862c.png" title=""/>
    </p>
    <p>
     9、设置移动数据配置
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0d81f05eb5beb063cc9cd12355fa75ca.png" title=""/>
    </p>
    <p>
     10、概要显示
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d7ebc55f41a8acf0bc24018284ab80d2.png" title=""/>
    </p>
    <p>
     11、点击完成开始转换
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/482ec178c73ffa148284303ef63792fe.png" title=""/>
    </p>
    <p>
     12、移植完成后，Oracle SQL Developer会自动创建一个与sqlserver数据库同名的用户，如我们这里的是
     <code>
      myproject
     </code>
     ，密码和用户名相同，我们用
     <code>
      myproject/myproject
     </code>
     在plsql里登录，查询用户表数据可以看到转换后的数据(而且对于在oracle不能使用的表名
     <code>
      user
     </code>
     改成了
     <code>
      user_
     </code>
     )。
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a800a2b096344dff65532e3efe473cfe.png" title=""/>
    </p>
