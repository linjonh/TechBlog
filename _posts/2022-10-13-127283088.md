    <h2>
     <a id="ffmpeg_1">
     </a>
     一、ffmpeg的简介
    </h2>
    <p>
     FFmpeg（全称：Fast Forward Mpeg），名称来自MPEG视频编码标准。FFmpeg是一套用来处理视频和音频的开源工具库。它提供了录制、转换、流化音视频以及后期处理的完整解决方案。有很高的质量和可移植性。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/445bb659d200259733239e4ed92e932f.png"/>
    </p>
    <h2>
     <a id="ffmpeg_5">
     </a>
     二、ffmpeg的安装
    </h2>
    <p>
     在Mac环境中，直接使用Homebrew （mac包管理工具）安装FFmpeg。
     <br/>
     没安装的可以参考
     <a href="https://blog.csdn.net/weixin_47061482/article/details/126116092">
      mac安装Homebrew
     </a>
     。也可以
     <a href="http://ffmpeg.org/download.html" rel="nofollow">
      官网下载
     </a>
     不同平台安装包。
    </p>
    <p>
     1、在终端中输入
     <code>
      brew install ffmpeg
     </code>
     安装，最后使用命令
     <code>
      ffmpeg -version
     </code>
     查看查看版本确认是否安装成功。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d96776f68ddb3ad1230e34c1fb5358b7.png"/>
    </p>
    <p>
     2、进入上一步的安装目录，打开目录位置：
    </p>
    <pre><code class="prism language-yaml">cd /usr/local/Cellar/ffmpeg
open .
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/01bd66a622dc9aa133c364bf4245a0f7.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4942be86a098bada9bef34e0fbcf386f.png"/>
     </img>
    </p>
    <p>
     <strong>
      目录结构说明：
     </strong>
    </p>
    <ul>
     <li>
      bin：有编译好的可执行程序：ffmpeg、ffplay、ffprobe等，可以直接在命令行上使用。
      <br/>
      eg： ffplay xx.mp4：可以直接播放某个视频
      <br/>
      eg：ffmpeg -version：可以查看FFmpeg的版本号
     </li>
     <li>
      include：开发时需要包含的头文件
     </li>
     <li>
      lib：链接时需要用到的库文件
     </li>
    </ul>
    <p>
     <strong>
      函数库：
     </strong>
    </p>
    <ul>
     <li>
      libavcodec：编解码库。
     </li>
     <li>
      libavformat：音视频容器格式以及所支持的协议的封装和解析。
     </li>
     <li>
      libavutil：提供了一些公共函数，工具库。
     </li>
     <li>
      libavfilter：音视频的滤镜库，如视频加水印、音频变声等。
     </li>
     <li>
      libavdevice：支持众多设备数据的输入与输出，如读取摄像头数据、屏幕录制。
     </li>
     <li>
      libswresample, libavresample：提供音频的重采样工具库。
     </li>
     <li>
      libswscale：提供对视频图像进行色彩转换、缩放以及像素格式转换，如图像的 YUV 转换。
     </li>
     <li>
      libpostproc：多媒体后处理器。
     </li>
    </ul>
    <p>
     <strong>
      基本概念：
     </strong>
    </p>
    <ul>
     <li>
      容器(Container)
      <br/>
      容器就是一种文件格式，比如flv，mkv等。包含下面5种流以及文件头信息。
     </li>
     <li>
      流(Stream)
      <br/>
      是一种视频数据信息的传输方式，5种流：音频，视频，字幕，附件，数据。
     </li>
     <li>
      帧(Frame)
      <br/>
      帧代表一幅静止的图像，分为I帧，P帧，B帧。
     </li>
     <li>
      编解码器(Codec)
      <br/>
      是对视频进行压缩或者解压缩，CODEC =COde （编码） +DECode（解码）
     </li>
     <li>
      复用/解复用(mux/demux)
      <br/>
      把不同的流按照某种容器的规则放入容器，这种行为叫做复用（mux）
      <br/>
      把不同的流从某种容器中解析出来，这种行为叫做解复用(demux)
     </li>
    </ul>
    <h2>
     <a id="ffmpegffplayffprobe_52">
     </a>
     三、ffmpeg、ffplay、ffprobe区别
    </h2>
    <h4>
     <a id="1ffmpeg_53">
     </a>
     1、ffmpeg是用于转码的应用程序
    </h4>
    <p>
     一个简单的转码命令可以这样写：将input.avi转码成output.ts，并设置视频的码率为640kbps
    </p>
    <pre><code class="prism language-yaml">ffmpeg <span class="token punctuation">-</span>i input.avi <span class="token punctuation">-</span>b<span class="token punctuation">:</span>v 640k output.ts
</code></pre>
    <h4>
     <a id="2ffplay_58">
     </a>
     2、ffplay是用于播放的应用程序
    </h4>
    <p>
     一个简单的播放命令可以这样写：播放test.avi
    </p>
    <pre><code class="prism language-yaml">ffplay test.avi
</code></pre>
    <h4>
     <a id="3ffprobe_63">
     </a>
     3、ffprobe是用于查看文件格式的应用程序
    </h4>
    <blockquote>
     <p>
      ffmpeg - h 帮助命令可以查看更多参数
     </p>
    </blockquote>
    <h2>
     <a id="_65">
     </a>
     四、常见的文件格式、编码
    </h2>
    <h4>
     <a id="1_66">
     </a>
     1、常见的视频格式、文件格式
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b12119e2408e6f0b54c9c005dda9523b.jpeg">
      <br/>
      eg:
      <code>
       ffmpeg -i test.mp4 output.mov
      </code>
      将test视频文件输出为mov文件：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5b7ec5f81519e7c242af28a90dfd44f5.png"/>
     </img>
    </p>
    <h4>
     <a id="2_70">
     </a>
     2、常见的编码音频转码
    </h4>
    <p>
     执行命令
     <code>
      ffmpeg -codecs
     </code>
     查看编解码方式，一般的视频网站用的是
    </p>
    <ul>
     <li>
      MP4封装：H264视频编码+AAC音频编码（比较成熟）
     </li>
     <li>
      WebM封装：VP8视频编码+Vorbis音频编码（谷歌方案）
     </li>
     <li>
      OGG封装：Theora视频编码+Vorbis音频编码（开源）
     </li>
    </ul>
    <blockquote>
     <p>
      以上适合1080p及其之下分辨率的编码方案
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4615f4caca0124a22525189dd66e4d20.png"/>
    </p>
    <p>
     eg:执行命令
     <code>
      ffmpeg -i test2.m4a -acodec libmp3lame -ar 44100 -ab 320k -ac 2 out.mp3
     </code>
     ，参数说明：
    </p>
    <ul>
     <li>
      acodec 指定音频编码器
     </li>
     <li>
      libmp3lame 指定mp3编码器
     </li>
     <li>
      ar 音频采样率 。不输入默认采用原音频的采样率：48khz、44.1hz
     </li>
     <li>
      ab 指定音频比特率 。不输入默认128k
     </li>
     <li>
      ac 设置声道数目，1单声道，2双声道
     </li>
    </ul>
    <h2>
     <a id="_86">
     </a>
     五、视频压制
    </h2>
    <p>
     eg:执行命令
     <code>
      ffmpeg -i test.mp4 -s 1920x1080 -pix_fmt yuv420p -vcodec libx264 -preset medium -profile:v high -level:v 4.1 -crf 23 -acodec aac -ar 44100 -ac 2 -b:a 128k out.mp4
     </code>
     ，参数说明：
    </p>
    <ul>
     <li>
      s 缩放视频尺寸
     </li>
     <li>
      pix_fmt 设置视频颜色空间。 网络视频通常用：yuv420p
     </li>
     <li>
      vcodec 网络视频编码器指令。libx264 软件编码器 通用稳定
     </li>
     <li>
      preset 编码器预设（共10个参数可选），一般录制时选用veryfast，压制时一般采用veryslow
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/162ad7dc3baea93a0e1bf083c20766ed.jpeg">
      - profile:v 指定编码器配置（和压缩比有关），压缩都狠，体积占用小，画面损失多。
      <br/>
      <mark>
       通讯领域一般用baseline，流媒体一般main，超清视频high
      </mark>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2aff7b64cb0da22c8fd8275270033faa.jpeg"/>
     </img>
    </p>
    <ul>
     <li>
      level:v 编码器的具体规范和限制(1~5.2) 一般1080p的用4.1
     </li>
     <li>
      crf 码率控制模式。
      <br/>
      0-51数值越小，质量越高，0就是无损，默认值23，一般在10-28间选择（适用于画质有要求，文件大小无要求的场景）
     </li>
     <li>
      b:a 音频比特率一般128k 或192
     </li>
    </ul>
    <blockquote>
     <p>
      ffmpeg码率控制模式：
     </p>
     <ul>
      <li>
       -qp 恒定量化器模式（每一帧都可以不同量化处理，没必要，一般不使用）
      </li>
      <li>
       -crf Constant Rate Factor 恒定速率因子模式 （常用）
      </li>
      <li>
       -b 固定目标码率模式（一般不使用，适用于对码率和体积有限制的情况）
       <br/>
       这三种默认都是单遍编码
       <br/>
       vbr 动态码率
       <br/>
       abr 平均比特率（目前视频经常用）
       <br/>
       cbr恒定比特率（没人用）
      </li>
     </ul>
    </blockquote>
    <h2>
     <a id="_109">
     </a>
     六、合并，提取音视频
    </h2>
    <p>
     <code>
      ffmpeg -i test.mp4 -vcodec copy -an v.mp4
     </code>
     将音频剔除,获取视频
     <br/>
     <code>
      ffmpeg -i test.mp4 -acodec copy -vn v.m4a
     </code>
     将视频剔除,获取音频
     <br/>
     将音频视频合并回去，由于刚刚从一个视频中分离的合并回去也不需要特殊处理，执行
     <code>
      ffmpeg -i v.mp4 -i v.m4a -c copy hebing.mp4
     </code>
     即可，如果对不上的需要用其他命令处理。
    </p>
    <h2>
     <a id="_113">
     </a>
     七、截取、连接音视频
    </h2>
    <h4>
     <a id="1_114">
     </a>
     1、截取视频音频
    </h4>
    <ul>
     <li>
      截取视频10s：
      <br/>
      <code>
       ffmpeg -i test.mp4 -ss 00:01:00 -to 00:01:10 -acodec copy out.mp4
      </code>
      截取铃声
      <br/>
      <code>
       ffmpeg -i test.mp4 -ss 00:01:00 -t 10 -acodec copy out.mp4
      </code>
      截取铃声，也可以这样写
      <br/>
      <code>
       ffmpeg -ss 00:01:00 -i test.mp4 -to 00:01:10 -c copy -copyts out.mp4
      </code>
      截取视频，会启用关键帧技术,copyts保留时间戳
     </li>
     <li>
      截取视频后10s：
      <code>
       ffmpeg -sseof -10 -i test.mp4 out.mp4
      </code>
     </li>
     <li>
      <code>
       ffmpeg -ss 00:01:00 -i test.mp4 -to 00:01:10 -c copy -y -threads 4 -preset ultrafast -strict -2 out.mp4
      </code>
      截取视频
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       参数解析
      </strong>
      ：
     </p>
     <ul>
      <li>
       <code>
        -codec
       </code>
       和
       <code>
        -c
       </code>
       是等价的,可以为指定的流设置编码器。
       <code>
        -vcodec
       </code>
       和
       <code>
        -c:v
       </code>
       是等价的,指定视频解码器。
      </li>
      <li>
       设定音频比特率： -ab bitrate(audio bitrate ),推荐改为使用
       <code>
        -b:a
       </code>
       。设定视频比特率：-b bitrate(video bitrate)，推荐改为使用
       <code>
        -b:v
       </code>
      </li>
      <li>
       使用 -i 选项选择网络源地址，
      </li>
      <li>
       -c 参数表示选择编码器类型
      </li>
      <li>
       -ss 指定开始时间，然后用 -to 指定结束时间
      </li>
      <li>
       copy 模式将原视频文件编码格式直接拷贝为新格式的视频
      </li>
      <li>
       -y 表示如果存在与指定的文件名相同的文件就覆盖掉原有的文件
      </li>
      <li>
       -threads 5 -preset ultrafast 多线程转码。ffmpeg配线程数只针对解码,不针对编码。 设置 preset 预设参数为 ultrafast 进行转码。
      </li>
      <li>
       -strict -2 之前是实验参数表示 aac音频编码 如果不使用aac音频编码使用使其的编码好像还需要导入第三方的音频编码库 比较麻烦 使用FFmpeg自带的aac音频编码要带上-strict -2 参数就可以了。带这个参数是为了使用aac音频编码。
      </li>
     </ul>
    </blockquote>
    <h4>
     <a id="2_131">
     </a>
     2、音频视频的连接
    </h4>
    <blockquote>
     <p>
      音频视频的连接有多种方法，常用的有如下两种，推荐使用第二种:FFmpeg concat 分离器。
     </p>
    </blockquote>
    <p>
     <strong>
      方法一
     </strong>
     ：FFmpeg concat 协议
     <br/>
     对于 MPEG 格式的视频，可以直接连接：
     <code>
      ffmpeg -i "concat:input1.mpg|input2.mpg|input3.mpg" -c copy output.mpg
     </code>
     <br/>
     对于非 MPEG 格式容器，但是是 MPEG 编码器（H.264、DivX、XviD、MPEG4、MPEG2、AAC、MP2、MP3 等），可以包装进 TS 格式的容器再合并。在新浪视频，有很多视频使用 H.264 编码器，可以采用这个方法
    </p>
    <pre><code class="prism language-yaml">ffmpeg <span class="token punctuation">-</span>i input1.flv <span class="token punctuation">-</span>c copy <span class="token punctuation">-</span>bsf<span class="token punctuation">:</span>v h264_mp4toannexb <span class="token punctuation">-</span>f mpegts input1.ts
ffmpeg <span class="token punctuation">-</span>i input2.flv <span class="token punctuation">-</span>c copy <span class="token punctuation">-</span>bsf<span class="token punctuation">:</span>v h264_mp4toannexb <span class="token punctuation">-</span>f mpegts input2.ts
ffmpeg <span class="token punctuation">-</span>i input3.flv <span class="token punctuation">-</span>c copy <span class="token punctuation">-</span>bsf<span class="token punctuation">:</span>v h264_mp4toannexb <span class="token punctuation">-</span>f mpegts input3.ts
ffmpeg <span class="token punctuation">-</span>i "concat<span class="token punctuation">:</span>input1.ts<span class="token punctuation">|</span>input2.ts<span class="token punctuation">|</span>input3.ts" <span class="token punctuation">-</span>c copy <span class="token punctuation">-</span>bsf<span class="token punctuation">:</span>a aac_adtstoasc <span class="token punctuation">-</span>movflags +faststart output.mp4
</code></pre>
    <p>
     保存 QuickTime/MP4 格式容器的时候，建议加上 -movflags +faststart。这样分享文件给别人的时候可以边下边看。
    </p>
    <p>
     <strong>
      方法二
     </strong>
     ：FFmpeg concat 分离器
     <br/>
     这种方法成功率很高，也是最好的，但是需要 FFmpeg 1.1 以上版本。先创建一个文本文件 filelist.txt，然后执行命令：
     <code>
      ffmpeg -f concat -i filelist.txt -c copy output.mp4
     </code>

    <h2>
     <a id="_151">
     </a>
     八、截图、水印、动图、相框
    </h2>
    <p>
     <code>
      ffmpeg -i test.mp4 -ss 5 -vframes 1 img.jpg
     </code>
     截取第几秒第几帧图片
     <br/>
     <code>
      ffmpeg -i test.mp4 -i logo.jpg -filter_complex "overlay=20:20" out.mp4
     </code>
     给视频加水印logo
     <br/>
     <code>
      ffmpeg -i test.mp4 -ss 10 -to 10.5 -s 640x320 -r 15 out.gif
     </code>
     处理一下帧率和大小，生成动图
    </p>
    <h2>
     <a id="_158">
     </a>
     九、滤镜
    </h2>
    <p>
     <strong>
      滤镜
     </strong>
     指的是在编码之前针对解码器解码出来的原始数据（即音视频帧）进行处理的动作，我们还可以称它为过滤器。
    </p>
    <p>
     ffmpeg内置了大概近400种滤镜，我们可以用
     <code>
      ffmpeg -filters 命令查看所有的滤镜
     </code>
     ，也可以用命令 ffmpeg -h filter=xxx 或者查看官方文档了解每一种滤镜。
    </p>
    <p>
     scale滤镜用于调整视频的大小，比如等比例缩放、等比例放大
    </p>
    <blockquote>
     <p>
      in_w in_h 或者 iw ih 表示输入视频的宽高
      <br/>
      out_w out_h 或者 ow oh 表示输出视频的宽高
     </p>
    </blockquote>
    <p>
     复杂滤镜用命令 -filter_complex 表示，它还有一个别名 -lavfi。
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        -filter_complex
       </code>
       : 相比-vf（-vf 是 -filter:v 的简写）, filter_complex适合开发复杂的滤镜功能，如同时对视频进行裁剪并旋转。参数之间使用逗号（，）隔开即可
      </li>
      <li>
       <code>
        main_w
       </code>
       :视频宽度
      </li>
      <li>
       <code>
        overlay_w
       </code>
       : 要添加的图片水印宽度
      </li>
      <li>
       <code>
        main_h
       </code>
       : 视频高度
      </li>
      <li>
       <code>
        overlay_h
       </code>
       :要添加的图片水印宽度
      </li>
     </ul>
    </blockquote>

     十、直播录屏
