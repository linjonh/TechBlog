     Xshell+Xftp通过代理的方式访问局域网内网服务器
    </h1>
   </div>

  <div class="article_content clearfix" id="article_content">

    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     最近在部署项目时遇到只有1台服务器拥有公网ip，其它服务器只有局域网ip，当然其它服务器可以正常访问网络，例如如下模型。之前访问其它几台服务器，都是先通过登录公网IP服务器，然后在Xshell里面执行ssh远程连接，这种方式不够直观而且上传文件到其它服务器很繁琐，非常麻烦。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/04805eef06873adf7bfe0b7193f6960f.png#pic_center"/>
    </p>
    <p>
     所以在网上找了下教程，最后结合网上的，在Xshell上面设置代理直接访问其它服务器，Xftp也是如此，当然它的原理也是利用公网服务器做跳转，不过所有操作都是Xshell 完成，不需要我们手动去跳。让我们来模拟下，比如现有公网服务器A 218.56.7.18 局域网服务器B 192.168.1.10。
    </p>
    <p>
     具体步骤看：
     <br/>
     <a href="https://blog.csdn.net/weixin_43135178/article/details/129025412?spm=1001.2014.3001.5506">
      xshell + xftp + MobaXterm 通过代理的方式访问外网不可见服务器
     </a>
    </p>
    <ol>
     <li>
      <p>
       首先我们在Xshell创建会话（公网服务器A）：
       <br/>
       步骤1：名称（备注自定义），协议（SSH），主机（公网服务器IP），端口号（默认22）
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1fa3f414c0fa6d36cf37870b87385268.png">
        <br/>
        步骤2：服务器账号，密码。勾选密码协议验证
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c3f66eb156113a53e7248e16e35ece57.png">
         <br/>
         步骤3：点击SSH -&gt; 隧道 -&gt; 添加（类型选择Dynamic，端口可以自定义）
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7f6564bf8589051e27bf0ce25c2c1066.png"/>
        </img>
       </img>
      </p>
     </li>
     <li>
      <p>
       然后在Xshell在创建一个新会话（局域网服务器B）
       <br/>
       步骤1：名称（备注自定义），协议（SSH），主机（局域网服务器IP），端口号（默认22）
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cf73795770fde75ec3c8fedef106c2c0.png">
        <br/>
        步骤2：服务器账号，密码。勾选密码协议验证
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/17c38c9f2c555f156d78a689b41450aa.png">
         <br/>
         步骤3：代理 -&gt; 预览 -&gt; 添加 -&gt; 填写信息（名称自定义，类型选择SOCKS4，主机localhost，端口号为服务器A设置的监听端口）。备注：如添加后下次则可直接选择代理服务器
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b06fdeebdf5fa9ff395a083f27a85440.png">
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3cbc74311d7c59deacd2fff97f432b81.png"/>
         </img>
        </img>
       </img>
      </p>
     </li>
     <li>
      <p>
       打开局域网服务器B的Xftp
       <br/>
       此时不需要配置Xftp，只需要在Xshell局域网服务器B上点击Xftp就可以自动连接了
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d95c6ea5b95359fa1a876930e3bf15d4.png"/>
      </p>
     </li>
     <li>
      <p>
       当局域网内网服务器连接上代理后，公网服务器会出现圆圈图标，这说明已经连接上了，都完成了
       <br/>
       <img alt="**在这里插入图片描述**" src="https://i-blog.csdnimg.cn/blog_migrate/6debc9355b43e91665378da2a90772e2.png"/>
      </p>
     </li>
    </ol>
    <p>
     <strong>
      备注：访问局域网服务器时，Xshell需要把公网服务器窗口打开，不然连接不上。
     </strong>
    </p>
