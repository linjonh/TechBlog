      <strong>
       （二）Linux 系统 TFTP 实验
      </strong>
     </img>
    </p>
    <p>
     TFTP（Trivial File Transfer Protocol， 简单文件传输协议）是 TCP/IP 协议族中的一个用来在客户机与服务器之间进行简单文件传输的协议，常被用于开发测试使用。
    </p>
    <p>
     TFTP 协议基于 UDP 协议，没有文件管理、用户控制功能。 TFTP 分为服务器端程序和客户端程序，在主机上通常同时配置有 TFTP 服务端和客户端。
    </p>
    <p>
     华清远见开发环境中已经包含 tftp 服务，不必进行安装操作（安装操作可参阅相关网络介绍文章），直接进行此实验的测试部分。
    </p>
    <p>
     1、启动tftp 服务器
     <br/>
     华清远见Ubuntu中默认已开启了tftp 服务器，但为保险，请运行以下命令重启服务：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># service  tftpd-hpa  restart</span>
</code></pre>

<strong>
（三）Linux 系统配置 NFS 实验
</strong>
</p>
<p>
NFS（Network File System） 方式类似于 windows 中的文件共享服务，是开发板通过 NFS 挂载放在主机（PC）上的根文件系统。此时在主机在文件系统中进行的操作同步反映在开发板上；反之，在开发板上进行的操作同步反映在主机中的根文件系统上。
</p>
<p>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cae63fc65606dc91bfe85a5d8521f3de.png"/>
<br/>
主机需要配置两方面内容：设置 NFS 服务，允许“指定的 NFS 用户”使用。
<br/>
具体操作步骤如下：
</p>
<p>
1、配置/etc/exports
<br/>
命令：vi /etc/exports
<br/>
NFS 允许挂载的目录及权限在文件/etc/exports 中进行了定义。例如，我们要将/source/rootfs 目录共享出来，那么我们需要在/etc/exports 文件末尾添加如下一行：
</p>
<p>
/source/rootfs *(rw,sync,no_root_squash,no_subtree_check)
</p>
<p>
特别提示：华清远见的开发环境中，该文件 12 行与上面的内容基本相同，但目录为“/source/nfs/rootfs”，请务必将 nfs 删除！！！否则会影响后面的操作！！删除后文件如下：
<br/>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f43a215c9aae0ef372f415d4cf4bb1ac.png"/>
<br/>
其中：/source/rootfs 是要共享的目录，*代表允许所有的网络段访问，rw 是可读写权限，sync 是资料同步写入内存和硬盘，no_root_squash 是 NFS 客户端分享目录使用者的权限，如果客户端使用的是 root 用户，那么对于该共享目录而言，该客户端就具有 root 权限。
</p>
<p>
2、重启服务：
</p>
<pre><code class="prism language-bash"><span class="token comment"># /etc/init.d/nfs-kernel-server restart</span>
</code></pre>
<p>
<img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f1bf5d09895ac3063026c0f7e5cca96d.png"/>

<strong>
（四）交叉开发环境搭建
</strong>
</p>
<p>
所谓交叉开发是指先在一台通用 PC 上进行软件的编辑、编译与连接，然后下载到嵌入式设备中运行调试的开发过程。通用 PC 成为宿主机，嵌入式设备成为目标机。
</p>
<p>
1、配置开发环境网络
<br/>

</p>
<p>

</p>

附录：UBoot 常用命令手册
</a>
</p>
<p>
U-Boot 还提供了更加详细的命令帮助，可以通过”?”显示支持的命令列表,通过 help [CommandName]命令还可以查看每个命令的参数说明。
</p>
<p>
<strong>
1、bootm
</strong>
</p>
<p>
bootm [addr [arg …]]
<br/> - boot application image stored in memory
<br/>
passing arguments ‘arg …’; when booting a Linux kernel,
<br/>
‘arg’ can be the address of an initrd image
<br/>
bootm 命令可以引导启动存储在内存中的程序映像。这些内存包括 RAM 和可以永久保存的 Flash。
<br/>
第 1 个参数 addr 是程序映像的地址，这个程序映像必须转换成 U-Boot 的格式。
<br/>
第 2 个参数对于引导 Linux 内核有用，通常作为 U-Boot 格式的 RAMDISK 映像存储地址；也可以是传递给 Linux 内核的参数（缺省情况下传递 bootargs 环境变量给内核）。
<br/>
附加说明: 要求二进制代码为制定格式的。通常为 mkimage 处理过的二进制文件。起动 UBOOT TOOLS 制作的压缩 LINUX 内核, bootm 3200000
</p>
<p>
<strong>
2、bootp
</strong>
</p>
<p>
bootp [loadAddress] [bootfilename]
<br/>
bootp 命令通过 bootp 请求，要求 DHCP 服务器分配 IP 地址，然后通过 TFTP 协议下载指定的文件到内存。
<br/>
第 1 个参数是下载文件存放的内存地址。
<br/>
第 2 个参数是要下载的文件名称，这个文件应该在开发主机上准备好。
<br/>
附加说明: 通过网络启动，需要提前设置好硬件地址 bdinfo 打印 ARM 相关信息
</p>
<p>
<strong>
3、cmp
</strong>
</p>
<p>
cmp [.b, .w, .l] addr1 addr2 count
<br/> - compare memory
<br/>
cmp 命令可以比较 2 块内存中的内容。.b 以字节为单位；.w 以字为单位；.l 以长字为单位。注意：cmp.b 中间不能保留空格，需要连续敲入命令。
<br/>
第 1 个参数 addr1 是第一块内存的起始地址。
<br/>
第 2 个参数 addr2 是第二块内存的起始地址。
<br/>
第 3 个参数 count 是要比较的数目，单位按照字节、字或者长字。
</p>
<p>
<strong>
4、cp
</strong>
</p>
<p>
cp [.b, .w, .l] source target count
<br/> - copy memory
<br/>
cp 命令可以在内存中复制数据块，包括对 Flash 的读写操作。
<br/>
第 1 个参数 source 是要复制的数据块起始地址。
<br/>
第 2 个参数 target 是数据块要复制到的地址。这个地址如果在 Flash 中，那么会直接调用写 Flash 的函数操作。所以 U-Boot 写 Flash 就使用这个命令，当然需要先把对应 Flash 区域擦净。
<br/>
第 3 个参数 count 是要复制的数目，根据 cp.b cp.w cp.l 分别以字节、字、长字为单位。
</p>
<p>
<strong>
5、crc32
</strong>
</p>
<p>
crc32 address count [addr]
<br/> - compute CRC32 checksum [save at addr]
<br/>
crc32 命令可以计算存储数据的校验和。
<br/>
第 1 个参数 address 是需要校验的数据起始地址。
<br/>
第 2 个参数 count 是要校验的数据字节数。
<br/>
第 3 个参数 addr 用来指定保存结果的地址。
</p>
<p>
<strong>
6、echo
</strong>
</p>
<p>
echo [args…]
<br/> - echo args to console; c suppresses newline
<br/>
echo 命令回显参数。
<br/>
用法:echo $( bootcmd）
</p>
<p>
<strong>
7、erase
</strong>
</p>
<p>
erase start end
<br/> - erase FLASH from addr ‘start’ to addr ‘end’
<br/>
erase N:SF[-SL]
<br/> - erase sectors SF-SL in FLASH bank # N
<br/>
erase bank N
<br/> - erase FLASH bank # N
<br/>
erase all
<br/> - erase all FLASH banks
<br/>
erase 命令可以擦 Flash。
</p>
<p>
参数必须指定 Flash 擦除的范围。
<br/>
按照起始地址和结束地址，start 必须是擦除块的起始地址；end 必须是擦除末尾块的结束地址。这种方式最常用。举例说明：擦除 0x20000 – 0x3ffff 区域命令为 erase 20000 3ffff。
<br/>
按照组和扇区，N 表示 Flash 的组号，SF 表示擦除起始扇区号，SL 表示擦除结束扇区号。另外，还可以擦除整个组，擦除组号为 N 的整个 Flash 组。擦除全部 Flash 只要给出一个 all 的参数即可。
</p>
<p>
用法:
<br/>
写入数据之前必须先执行该命令
<br/>
用法: erase 0xc**\*\*\*** +0xYYYYYYYY
<br/>
说明: 擦除从 0xc**\*\*\*** 开始的 0xYYYYYYYY 字节(字节必须块对齐）
<br/>
例:
<br/>
Z228 # erase 0xc4040000 +0x180000
<br/>
… done
<br/>
Erased 24 sectors
<br/>
附加: erase 1:0-2(就是对每一块 FLASH 的 0-2 扇区进行删除)
</p>
<p>
h. cp.b
<br/>
拷贝数据，从指定源地址到目标地址，地址可以是 ram，也可以是 nor flash
<br/>
用法: cp.b 0x**\*\*\*\*** 0x######## 0xYYYYYYYY
<br/>
说明: 拷贝源 0x**\*\*\*\***到目标 0x########,共拷贝 0xYYYYYYY 字节
<br/>
例:
<br/>
Z228 # cp.b 0x200000 0xc4040000 0x180000
<br/>
Copy to Flash… done
</p>
<p>
<strong>
8、flinfo
</strong>
</p>
<p>
flinfo
<br/> - print information for all FLASH memory banks
<br/>
flinfo N
<br/> - print information for FLASH memory bank # N
<br/>
flinfo 命令打印全部 Flash 组的信息，也可以只打印其中某个组。一般嵌入式系统的 Flash 只有一个组。
</p>
<p>
<strong>
9、go
</strong>
</p>
<p>
go addr [arg …]
<br/> - start application at address ‘addr’
<br/>
passing ‘arg’ as arguments
<br/>
go 命令可以执行应用程序。
<br/>
第 1 个参数是要执行程序的入口地址。
<br/>
第 2 个可选参数是传递给程序的参数，可以不用。
</p>
<p>
<strong>
10、iminfo
</strong>
</p>
<p>
iminfo addr [addr …]
<br/> - print header information for application image starting at
<br/>
address ‘addr’ in memory; this includes verification of the
<br/>
image contents (magic number, header and payload checksums)
<br/>
iminfo 可以打印程序映像的开头信息，包含了映像内容的校验（序列号、头和校验和）。
<br/>
第 1 个参数指定映像的起始地址。
<br/>
可选的参数是指定更多的映像地址。
</p>
<p>
<strong>
11、loadb
</strong>
</p>
<p>
loadb [ off ] [ baud ]
<br/> - load binary file over serial line with offset ‘off’ and baudrate ‘baud’
<br/>
dKNLinux 联盟 loadb 命令可以通过串口 Kermit 协议下载二进制数据。
</p>
<p>
<strong>
12、loads
</strong>
</p>
<p>
loads [ off ]
<br/> - load S-Record file over serial line with offset ‘off’
<br/>
loads 命令可以通过串口线下载 S-Record 格式文件。
</p>
<p>
<strong>
13、mw
</strong>
</p>
<p>
mw [.b, .w, .l] address value [count]
<br/> - write memory
<br/>
dKNLinux 联盟 mw 命令可以按照字节、字、长字写内存，.b .w .l 的用法与 cp 命令相同。
<br/>
第 1 个参数 address 是要写的内存地址。
<br/>
第 2 个参数 value 是要写的值。
<br/>
第 3 个可选参数 count 是要写单位值的数目。
<br/>
用法: mw 32000000 ff 10000(把内存 0x32000000 开始的 0x10000 字节设为 0xFF)
</p>
<p>
<strong>
14、nm
</strong>
</p>
<p>
nm [.b, .w, .l] address
<br/> - memory modify, read and keep address
<br/>
nm 命令可以修改内存，可以按照字节、字、长字操作。
<br/>
参数 address 是要读出并且修改的内存地址。
</p>
<p>
<strong>
15、Md
</strong>
<br/>
显示内存区的内容。
</p>
<p>
<strong>
16、Mm
</strong>
<br/>
读或修改内存，地址自动递增。
<br/>
mm 内存地址 回车
</p>
<p>
此时进入内存查看 直接按回车 保持原内容不变，若要改变，在问号后面输入数值然后回车 按 . 此符号退出
</p>

<strong>
17、nfs
</strong>
<br/>
nfs [loadAddress] [host ip addr:bootfilename]
<br/>
nfs 命令可以使用 NFS 网络协议通过网络启动映像。
<br/>
用法:
<br/>
nfs 32000000 192.168.0.2:aa.txt
<br/>
把 192.168.0.2(LINUX 的 NFS 文件系统)中的 NFS 文件系统中的 aa.txt 读入内存 0x32000000 处。
</p>
<p>
<strong>
18、printenv
</strong>
<br/>
printenv
<br/> - print values of all environment variables
<br/>
printenv name …
<br/> - print value of environment variable ‘name’
<br/>
printenv 命令打印环境变量。
<br/>
可以打印全部环境变量，也可以只打印参数中列出的环境变量。
</p>
<p>
<strong>
19、protect
</strong>
<br/>
protect on start end
<br/> - protect Flash from addr ‘start’ to addr ‘end’
<br/>
protect on N:SF[-SL]
<br/> - protect sectors SF-SL in Flash bank # N
<br/>
protect on bank N
<br/> - protect Flash bank # N
<br/>
protect on all
<br/> - protect all Flash banks
<br/>
protect off start end
<br/> - make Flash from addr ‘start’ to addr ‘end’ writable
<br/>
protect off N:SF[-SL]
<br/> - make sectors SF-SL writable in Flash bank # N
<br/>
protect off bank N
<br/> - make Flash bank # N writable
<br/>
protect off all
<br/> - make all Flash banks writable
<br/>
protect 命令是对 Flash 写保护的操作，可以使能和解除写保护。
<br/>
第 1 个参数 on 代表使能写保护；off 代表解除写保护。
<br/>
第 2、3 参数是指定 Flash 写保护操作范围，跟擦除的方式相同。
<br/>
用法:
<br/>
protect on 1:0-3(就是对第一块 FLASH 的 0-3 扇区进行保护)
<br/>
protect off 1:0-3 取消写保护
<br/>
protect off bank 1 解除/使能 第 N 块 FLASH 的写保护
</p>

20、rarpboot
</strong>
<br/>
rarpboot [loadAddress] [bootfilename]
<br/>
rarboot 命令可以使用 TFTP 协议通过网络启动映像。也就是把指定的文件下载到指定地址，然后执行。
<br/>
第 1 个参数是映像文件下载到的内存地址。
<br/>
第 2 个参数是要下载执行的映像文件。
</p>
<p>
<strong>
21、run
</strong>
<br/>
run var […]
<br/> - run the commands in the environment variable(s) ‘var’
<br/>
run 命令可以执行环境变量中的命令，后面参数可以跟几个环境变量名。
<br/>
用法:
<br/>
Uboot&gt; setenv flashit tftp 20000000 mycode.bin; erase 10020000 1002FFFF;
<br/>
cp.b 20000000 10020000 8000
<br/>
Uboot&gt; saveenv
<br/>
Uboot&gt; run flashit
</p>

22、setenv
</strong>
<br/>
setenv name value …
<br/> - set environment variable ‘name’ to ‘value …’
<br/>

<br/> - delete environment variable ‘name’
<br/>
setenv 命令可以设置环境变量。
<br/>
第 1 个参数是环境变量的名称。
<br/>
第 2 个参数是要设置的值，如果没有第 2 个参数，表示删除这个环境变量。
<br/>
用法:
<br/>
Uboot&gt; setenv myboard AT91RM9200DK
<br/>
Uboot&gt; printenv
<br/>
baudrate=115200
<br/>
ipaddr=192.168.1.1
<br/>
ethaddr=12:34:56:78:9A:BC
<br/>
serverip=192.168.1.5
<br/>
myboard=AT91RM9200DK
<br/>
Environment size: 102/8188 bytes
</p>
<p>
<strong>
23、Saveenv
</strong>
<br/>
保存环境变量
<br/>
命令将当前定义的所有的变量及其值存入 flash 中。用来存储变量及其值的空间只有 8k 字节，应不要超过。
</p>

sleep N
<br/> - delay execution for N seconds (N is
<em>
decimal
</em>
!!!)

</p>
<p>
<strong>
25、tftpboot
</strong>
<br/>
tftpboot [loadAddress] [bootfilename]
<br/>
tftpboot 命令可以使用 TFTP 协议通过网络下载文件。按照二进制文件格式下载。另外使用这个命令，必须配置好相关的环境变量。例如 serverip 和 ipaddr。
<br/>
第 1 个参数 loadAddress 是下载到的内存地址。
<br/>
第 2 个参数是要下载的文件名称，必须放在 TFTP 服务器相应的目录下。
</p>
