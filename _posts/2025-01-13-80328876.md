     </span>
     <span style="color:#000000;">

    <p>
     4. 视频编码原理
    </p>
    <p>
    </p>
    <p>
     <span style="color:#000000;">
      经典的
     </span>
     <span style="color:#000000;">
      IPB
     </span>
     <span style="color:#000000;">
      压缩
     </span>
     <span style="color:#000000;">
      :
     </span>
    </p>
    <p>
     <span style="color:#000000;">
      Iframes:
     </span>
     <span style="color:#000000;">
      节点图像
     </span>
     <span style="color:#000000;">
      ,
     </span>
     <span style="color:#000000;">
      保存
     </span>
     <span style="color:#000000;">
      一帧完整图像压缩后的
     </span>
     <span style="color:#000000;">
      数据
     </span>
     <span style="color:#000000;">
      ,
     </span>
    </p>
    <p>
     <span style="color:#000000;">
     </span>
     <span style="color:#000000;">
     </span>
     <span style="color:#000000;">
      编
     </span>
     <span style="color:#000000;">
      解码不涉及自己以外的任何
     </span>
     <span style="color:#000000;">
      图像
     </span>
    </p>
    <p>
     <span style="color:#000000;">
      Pframes:
     </span>
     <span style="color:#000000;">
      预测图像
     </span>
     <span style="color:#000000;">
      :
     </span>
     <span style="color:#000000;">
      仅
     </span>
     <span style="color:#000000;">
      保存当前帧与前一帧的差值
     </span>
     <span style="color:#000000;">
      信息
     </span>
     <span style="color:#000000;">
      ,
     </span>
    </p>
    <p>
     <span style="color:#000000;">
     </span>
     <span style="color:#000000;">
     </span>
     <span style="color:#000000;">
      解码
     </span>
     <span style="color:#000000;">
      依赖于前一帧
     </span>
     <span style="color:#000000;">
      数据
     </span>
    </p>
    <p>
     <span style="color:#000000;">
      Bframes:
     </span>
     <span style="color:#000000;">
      前后
     </span>
     <span style="color:#000000;">
      预测
     </span>
     <span style="color:#000000;">
      图像
     </span>
     <span style="color:#000000;">
      :
     </span>
     <span style="color:#000000;">
      保存
     </span>
     <span style="color:#000000;">
      当前帧与前一帧和后一帧的差值
     </span>
     <span style="color:#000000;">
      信息
     </span>
     <span style="color:#000000;">
      ,
     </span>
     <span style="color:#000000;">
      可双向单帧逐步播放
     </span>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/cffd8287f9fff7d22958bfa17c1f2ee5.png"/>
    </p>
    <p>
     5.
     <span style="color:rgb(63,63,63);">
      视频编码原理
     </span>
    </p>
    <p>
     <span style="color:rgb(63,63,63);">
      以H264为例
     </span>
    </p>
    <p>
     <span style="color:rgb(63,63,63);">
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/330593a29ae0ad91e4f4d62cd0d3ade1.png">
       <br/>
      </img>
     </span>
    </p>
    <p>
     <span style="color:rgb(63,63,63);">
      <br/>
     </span>
    </p>
    <p>
     <span style="color:rgb(63,63,63);">
      6.
     </span>
     <span style="color:rgb(63,63,63);">
      图像
     </span>
     <span style="color:rgb(63,63,63);">
      群组
     </span>
     <span style="color:rgb(63,63,63);">
      (GOP)
     </span>
    </p>
    <p>
    </p>
    <p>
     <span style="color:#000000;">
      图像群组就是一组连续的图像
     </span>
     <span style="color:#000000;">
      , GOP
     </span>
     <span style="color:#000000;">
      总是以
     </span>
     <span style="color:#000000;">
      I frame
     </span>
     <span style="color:#000000;">
      为起始点
     </span>
     <span style="color:#000000;">
      ,
     </span>
    </p>
    <p>
     <span style="color:#000000;">
      后面有若干
     </span>
     <span style="color:#000000;">
      P frame
     </span>
     <span style="color:#000000;">
      和
     </span>
     <span style="color:#000000;">
      Bframe,
     </span>
     <span style="color:#000000;">
      下一个
     </span>
     <span style="color:#000000;">
      I frame
     </span>
     <span style="color:#000000;">
      则是新的
     </span>
     <span style="color:#000000;">
      GOP
     </span>
     <span style="color:#000000;">
      起点
     </span>
    </p>
    <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/125e990d153807417d4a575ef5e5cb77.png">
     <br/>
     <p>
      7.
      <span style="color:rgb(63,63,63);">
       硬件编解码
      </span>
     </p>
     <p>
      <span style="color:rgb(63,63,63);">
      </span>
     </p>
     <p>
      <span style="color:#333333;">
       硬件编解码
      </span>
      <span style="color:#333333;">
       是图形芯片
      </span>
      <span style="color:#333333;">
       厂家提出
      </span>
      <span style="color:#333333;">
       的
      </span>
      <span style="color:#333333;">
       用
      </span>
      <span style="color:#333333;">
       GPU
      </span>
      <span style="color:#333333;">
       资源解码视频流的方案
      </span>
     </p>
     <p>
      <span style="color:#333333;">
       (
      </span>
      <span style="color:#333333;">
       软
      </span>
      <span style="color:#333333;">
       解是用
      </span>
      <span style="color:#333333;">
       CPU
      </span>
      <span style="color:#333333;">
       承担解码
      </span>
      <span style="color:#333333;">
       工作
      </span>
      <span style="color:#333333;">
       )
      </span>
     </p>
     <p>
      <span style="color:#333333;">
       优点
      </span>
      <span style="color:#333333;">
       是编解码效率高，功耗低、热功耗低，缺点存在局限性，
      </span>
      <span style="color:#000000;">
       设置较为
      </span>
      <span style="color:#000000;">
       复杂
      </span>
      <span style="color:#000000;">
       ,
      </span>
      <span style="color:#000000;">
       一般需要使用图形芯片厂商提供的驱动和
      </span>
      <span style="color:#000000;">
       sdk
      </span>
      <span style="color:#000000;">
       程序
      </span>
     </p>
     <p>
      <span style="color:#000000;">
       为什么
      </span>
      <span style="color:#000000;">
       GPU
      </span>
      <span style="color:#000000;">
       编解码速度快于
      </span>
      <span style="color:#000000;">
       CPU?
      </span>
     </p>
     <div>
      •
      <span style="color:#000000;">
       CPU
      </span>
      <span style="color:#000000;">
       主要为串行指令而优化，而
      </span>
      <span style="color:#000000;">
       GPU
      </span>
      <span style="color:#000000;">
       则是为大规模的并行运算而
      </span>
      <span style="color:#000000;">
       优化
      </span>
     </div>
     <div>
      •
      <span style="color:#000000;">
       从
      </span>
      <span style="color:#000000;">
       并行的角度来看，现代的多核
      </span>
      <span style="color:#000000;">
       CPU
      </span>
      <span style="color:#000000;">
       针对的是指令集并行（
      </span>
      <span style="color:#000000;">
       ILP
      </span>
      <span style="color:#000000;">
       ）和任务并行（
      </span>
      <span style="color:#000000;">
       TLP
      </span>
      <span style="color:#000000;">
       ），而
      </span>
      <span style="color:#000000;">
       GPU
      </span>
      <span style="color:#000000;">
       则是数据并行（
      </span>
      <span style="color:#000000;">
       DLP
      </span>
      <span style="color:#000000;">
       ）
      </span>
     </div>
     <div>
      •
      <span style="color:#000000;">
       在
      </span>
      <span style="color:#000000;">
       同样面积的芯片之上，
      </span>
      <span style="color:#000000;">
       CPU
      </span>
      <span style="color:#000000;">
       更多的放置了多级缓存（
      </span>
      <span style="color:#000000;">
       L1/L2/LLC
      </span>
      <span style="color:#000000;">
       ）和指令并行相关的控制部件（乱序执行，分支预测等等），而
      </span>
      <span style="color:#000000;">
       GPU
      </span>
      <span style="color:#000000;">
       上则更多的是运算单元（整数、浮点的乘加单元，特殊运算单元等等）
      </span>
     </div>
     <div>
      •
      <span style="color:#000000;">
       GPU
      </span>
      <span style="color:#000000;">
       往往拥有更大带宽的
      </span>
      <span style="color:#000000;">
       Memory
      </span>
      <span style="color:#000000;">
       ，
      </span>
      <span style="color:#000000;">
       也就是显
      </span>
      <span style="color:#000000;">
       存，因此在大吞吐量的应用中也会有很好的性能
      </span>
     </div>
