    <h3>
     <a id="_93">
     </a>
     注释
    </h3>
    <p>
     原则
    </p>
    <ul>
     <li>
      As short as possible（如无必要，勿增注释）：尽量提高代码本身的清晰性、可读性。
     </li>
     <li>
      As long as necessary（如有必要，尽量详尽）：合理的注释、空行排版等，可以让代码更易阅读、更具美感。
      <br/>
      单行注释
      <br/>
      必须独占一行。// 后跟一个空格，缩进与下一行被注释说明的代码一致。
      <br/>
      多行注释
      <br/>
      避免使用 /
      <em>
       …
      </em>
      / 这样的多行注释。有多行注释内容时，使用多个单行注释。
      <br/>
      函数/方法注释
     </li>
    </ul>
    <ol>
     <li>
      函数/方法注释必须包含函数说明，有参数和返回值时必须使用注释标识。；
     </li>
     <li>
      参数和返回值注释必须包含类型信息和说明；
     </li>
    </ol>
    <pre><code class="prism language-js"><span class="token comment">/**
 * 函数描述
 *
 * @param {string} p1 参数1的说明
 * @param {string} p2 参数2的说明，比较长
 *     那就换行了.
 * @param {number=} p3 参数3的说明（可选）
 * @return {Object} 返回值描述
 */</span>
<span class="token keyword">function</span> <span class="token function">foo</span><span class="token punctuation">(</span><span class="token parameter">p1<span class="token punctuation">,</span> p2<span class="token punctuation">,</span> p3</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">let</span> p3 <span class="token operator">=</span> p3 <span class="token operator">||</span> <span class="token number">10</span><span class="token punctuation">;</span>
    <span class="token keyword">return</span> <span class="token punctuation">{<!-- --></span>
        p1<span class="token operator">:</span> p1<span class="token punctuation">,</span>
        p2<span class="token operator">:</span> p2<span class="token punctuation">,</span>
        p3<span class="token operator">:</span> p3
    <span class="token punctuation">}</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>

    <h2>
     <a id="Vue__144">
     </a>
     Vue 代码规范
    </h2>
    <h4>
     <a id="_145">
     </a>
     常规
    </h4>
    <ul>
     <li>
      当在组件中使用 data 属性的时候 (除了 new Vue 外的任何地方)，它的值必须是返回一个对象的函数 data() { return {…} }。
     </li>
     <li>
      prop 的定义应该尽量详细，至少需要指定其类型。
     </li>
    </ul>
    <pre><code class="prism language-js"><span class="token keyword">export</span> <span class="token keyword">default</span> <span class="token punctuation">{<!-- --></span>
    prop<span class="token operator">:</span> <span class="token punctuation">[</span>
        data<span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
            type<span class="token operator">:</span> Array<span class="token punctuation">,</span>
            <span class="token keyword">default</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
                <span class="token keyword">return</span> <span class="token punctuation">[</span><span class="token punctuation">]</span>
            <span class="token punctuation">}</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">]</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      布尔类型的 attribute， 为 true 时直接写属性值。
     </li>
     <li>
      不要在computed中对vue变量进行操作。
     </li>
     <li>
      应该优先通过 prop 和事件进行父子组件之间的通信，而不是 this.$parent 或改变 prop。
     </li>
     <li>
      在组件上总是必须用 key 配合 v-for，以便维护内部组件及其子树的状态。
     </li>
     <li>
      v-if 和 v-for 不能同时使用
     </li>
     <li>
      公共方法尽量不要挂到原型上, 可以写在 utils 文件，也可以使用 mixin 文件。不要将业务公共组件注册到全局。
     </li>
     <li>
      不要将任何第三方插件挂载到 vue 原型上。
     </li>
     <li>
      具有高度通用性的方法，要封装到 libs、全局组件或指令集里。
     </li>
     <li>
      为组件样式设置作用域。
     </li>
     <li>
      尽量使用指令缩写。
     </li>
    </ul>
    <h4>
     <a id="vuex_170">
     </a>
     vuex
    </h4>
    <p>
     State (opens new window)为单一状态树，在 state 中需要定义我们所需要管理的数组、对象、字符串等等，只有在这里定义了，在 vue 的组件中才能获取你定义的这个对象的状态。
    </p>
    <ul>
     <li>
      修改 state 中数据必须通过 mutations。
     </li>
     <li>
      每一个可能发生改变的 state 必须同步创建一条或多条用来改变它的 mutations。
     </li>
     <li>
      服务端获取的数据存放在 state 中，作为原始数据保留，不可变动。
      <br/>
      Getters (opens new window)有点类似 vue.js 的计算属性，当我们需要从 store 的 state 中派生出一些状态，那么我们就需要使用 getters，getters 会接收 state 作为第一个参数，而且 getters 的返回值会根据它的依赖被缓存起来，只有 getters 中的依赖值（state 中的某个需要派生状态的值）发生改变的时候才会被重新计算。
     </li>
     <li>
      通过 getters 处理你需要得到的数据格式，而不是通过修改 state 原始数据。
     </li>
     <li>
      组件内不强制使用 mapGetters，因为你可能需要使用 getter 和 setter。
     </li>
     <li>
      改变 state 的唯一方法就是提交 mutations (opens new window)。
     </li>
     <li>
      组件内使用 mapMutations 辅助函数将组件中的 methods 映射为 store.commit 调用。
     </li>
     <li>
      命名采用 大写字母 + 下划线 的规则。
     </li>
     <li>
      定义 CLEAR，以确保路由切换时可以初始化数据。
      <br/>
      Actions
     </li>
     <li>
      页面重的数据接口尽量在 actions (opens new window)中调用。
     </li>
     <li>
      服务端返回的数据尽量不作处理，保留原始数据。
     </li>
     <li>
      获取到的数据必须通过调用 mutations 改变 state。
      <br/>
      Modules
     </li>
     <li>
      通常情况下按照页面划分 modules (opens new window)。
     </li>
     <li>
      默认内置了 system 保证了脚手架的基础功能。
     </li>
     <li>
      每个页面模块或页面的子模块添加属性 namespaced: true。
      <br/>
      文件命名规则
      <br/>
      Component
      <br/>
      所有的Component文件都是以大写开头 (PascalCase)，这也是官方所推荐的。
      <br/>
      但除了 index.vue。
      <br/>
      例子：
     </li>
     <li>
      @/components/BackToTop/index.vue
     </li>
     <li>
      @/components/Charts/Line.vue
     </li>
     <li>
      @/views/example/components/Button.vue
      <br/>
      JS 文件
      <br/>
      所有的.js文件都遵循横线连接 (kebab-case)。
      <br/>
      例子：
     </li>
     <li>
      @/utils/open-window.js
     </li>
     <li>
      @/views/svg-icons/require-icons.js
     </li>
     <li>
      @/components/MarkdownEditor/default-options.js
      <br/>
      Views
      <br/>
      在views文件下，代表路由的.vue文件都使用横线连接 (kebab-case)，代表路由的文件夹也是使用同样的规则。
      <br/>
      例子：
     </li>
     <li>
      @/views/svg-icons/index.vue
     </li>
     <li>
      @/views/svg-icons/require-icons.js
      <br/>
      使用横线连接 (kebab-case)来命名views主要是出于以下几个考虑。
     </li>
     <li>
      横线连接 (kebab-case) 也是官方推荐的命名规范之一 文档
     </li>
     <li>
      views下的.vue文件代表的是一个路由，所以它需要和component进行区分(component 都是大写开头)
     </li>
     <li>
