    <p align="left">
     <span style="font-size:14px">
      Windows Azure 的目标是使用开发人员的首选语言、工具和框架，为全世界的所有应用程序提供支持—包括每个平台、每台设备上的应用程序。通过最近公布的移动服务, 我们朝这一承诺又迈近了一步。
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      下面我们快速浏览一下新示例：
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      ·   适用于营销竞争的 Web 和移动应用程序
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      ·   利用 Service Bus 中继和 BizTalk 的集成方案
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      ·   来自 SendGrid、Twilio、Xamarin 和 Redbit 的示例
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      ·   //build 上的移动服务讲座
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      我们希望这些能够为您开发自己的移动应用程序提供灵感。
     </span>
    </p>
    <p align="left">
     <strong>
      <span style="font-size:14px">
       适用于营销竞争的 Web 和移动应用程序
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:14px">
      最佳用户体验的一个方面是在每个 Web 平台和移动平台之间保持一致。Windows Azure 移动服务和网站在核心企业应用程序和品牌应用程序上帮助您实现了这一点。通过在 Web 应用程序和移动应用程序之间共享身份验证系统和数据库或存储容器（如下面所演示），可以提高用户参与度并向用户授权，不管他们的接入点是什么。
     </span>
    </p>
    <p align="left">
     <span style="font-size:14px">
      在以下视频中，
     </span>
     <a href="https://twitter.com/Nik_G" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-size:14px; color:#16a5d9">
       Nik Garkusha
      </span>
     </a>
     <span style="font-size:14px">
      将演示如何使用移动服务和网站来创建用作 iOS 应用程序和 .NET Web 管理门户后端的一致服务集。
     </span>
    </p>
    <p align="left">
     <img alt="" src="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-01-13-25/2438.mir1.png"/>
    </p>
    <p align="left">
     <span style="font-size:14px">
      在第 1 部分中，Nik 演示了如何使用多个身份验证提供程序、如何读/写表数据以及如何与 Windows Azure Blob 存储进行交互。
     </span>
    </p>
    <p align="left">
     <img alt="" src="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-01-13-25/6013.mir2.png"/>
    </p>
    <p align="left">
     <span style="font-size:14px">
      在第 2 部分中，Nik 继续演示了如何通过网站创建管理门户、如何将自定义 API 用于跨平台推送通知以及如何使用带有第三方附加组件的计划程序来执行脚本管理任务.
     </span>
    </p>
    <span style="font-size:14px">
    </span>
    <p align="left">
     <strong>
      利用 Service Bus 中继和 BizTalk 的集成方案
     </strong>
    </p>
    <p align="left">
     现代企业往往面临着要不断创新和推出新平台的挑战，同时又要充分利用现有系统。通过将移动服务与 Service Bus 中继和 BizTalk Server 结合使用，使之成为可能。
    </p>
    <p align="left">
     在以下示例中，
     <a href="https://twitter.com/babosbird" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       Paolo Salvatori
      </span>
     </a>
     详细演示了如何连接这些服务以启用此类方案。
    </p>
    <p align="left">
     ·
     <a href="http://code.msdn.microsoft.com/windowsazure/How-to-integrate-a-Mobile-1ee6a5ea" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       与
      </span>
      <span style="color:#16a5d9">
       Service Bus
      </span>
      <span style="color:#16a5d9">
       中的
      </span>
      <span style="color:#16a5d9">
       REST
      </span>
      <span style="color:#16a5d9">
       中继服务集成
      </span>
     </a>
     – 该示例演示了如何通过 Service Bus 中继服务和 REST 协议将移动服务与内部部署的业务线应用程序集成。
    </p>
    <p align="left">
     ·
     <a href="http://code.msdn.microsoft.com/windowsazure/How-to-integrate-a-Mobile-8780500c" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       与
      </span>
      <span style="color:#16a5d9">
       Service Bus
      </span>
      <span style="color:#16a5d9">
       中的
      </span>
      <span style="color:#16a5d9">
       SOAP
      </span>
      <span style="color:#16a5d9">
       中继服务集成
      </span>
     </a>
     – 在该示例中，自定义 API 用于调用 WCF 服务，该服务通过 SOAP Service Bus 中继服务使用 BasicHttpRelayBinding 端点展示其功能。
    </p>
    <p align="left">
     ·
     <a href="http://code.msdn.microsoft.com/windowsazure/How-to-integrate-Mobiles-77b25d12" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       通过
      </span>
      <span style="color:#16a5d9">
       Service Bus
      </span>
      <span style="color:#16a5d9">
       与
      </span>
      <span style="color:#16a5d9">
       BizTalk Server
      </span>
      <span style="color:#16a5d9">
       集成
      </span>
     </a>
     – 在该演示中，您将了解到如何通过 BizTalk Server 2013、Service Bus 代理消息和 Service Bus 中继，将移动服务与内部或云业务线应用程序相集成。访问控制服务用于通过 Windows Azure Service Bus 对 Windows Azure 移动服务进行身份验证。在该方案中，BizTalk Server 2013 可以在内部部署运行，也可以在 Windows Azure 上的虚拟机中运行。
    </p>
    <p align="left">
     ·
     <a href="http://code.msdn.microsoft.com/windowsazure/How-to-integrate-Mobile-6718aaf2" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       与
      </span>
      <span style="color:#16a5d9">
       Windows Azure BizTalk
      </span>
      <span style="color:#16a5d9">
       服务集成
      </span>
     </a>
     – 了解如何通过 Windows Azure BizTalk 服务（当前为预览版）和 Service Bus 中继将移动服务与内部部署或云业务线应用程序相集成。访问控制服务用于通过解决方案使用的 XML Request-Reply Bridge对移动服务进行身份验证，以将消息转换并路由至业务线应用程序。
    </p>
    <p align="left">
     <strong>
      来自 SendGrid、Twilio、Xamarin 和 Redbit 的示例
     </strong>
    </p>
    <p align="left">
     在 3 月份，我们
     <a href="http://blogs.msdn.com/b/azchina/archive/2013/03/05/windows-azure-mobile-services-adds-android-support-and-extends-availability-to-east-asia.aspx" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       重申了我们的承诺
      </span>
     </a>
     ，即为开发人员提供便利，让他们可以使用自己喜爱的语言、工具和服务来为每个主要的移动平台构建和部署连接到云的应用程序。今天，我很高兴能将更新共享到移动服务合作伙伴生态系统和功能套件，兑现我们始终不变的承诺。
    </p>
    <p align="left">
     开发人员可以方便地访问他们最喜爱的第三方服务和丰富的示例，因为将移动服务与这些服务结合使用是我们团队的最高优先事项之一。我们在
     <a href="http://weblogs.asp.net/scottgu/archive/2013/06/14/windows-azure-major-updates-for-mobile-backend-development.aspx" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       推出源代码版本控制和自定义
      </span>
      <span style="color:#16a5d9">
       API
      </span>
      <span style="color:#16a5d9">
       功能
      </span>
     </a>
     的同时，也启用了一系列新的方案，其中之一就是以更灵活的方式与第三方服务合作。
    </p>
    <p align="left">
     SendGrid、Twilio、Xamarin 和 Redbit都开发了能激发开发人员灵感的示例应用程序，帮助他们实现无限可能。
    </p>
    <p align="left">
     ·
     <strong>
      SendGrid
     </strong>
     消除了发送电子邮件的复杂性，节省了时间和金钱，同时将邮件可靠地递送至收件箱。SendGrid 发布了一款可接受并播放通过电子邮件发送的歌曲的
     <a href="https://github.com/sendgrid/sendgrid-azure-ios/" rel="noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       iOS
      </span>
      <span style="color:#16a5d9">
       示例应用程序
      </span>
     </a>
     。
     <a href="http://sendgrid.com/docs/Integrate/Partners/WindowsAzure.html" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       SendGrid
      </span>
      <span style="color:#16a5d9">
       文档中心
      </span>
     </a>
     和
     <a href="https://www.windowsazure.com/en-us/develop/mobile/tutorials/send-email-with-sendgrid/" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       Windows Azure
      </span>
      <span style="color:#16a5d9">
       开发人员中心
      </span>
     </a>
     提供了有关如何从移动服务支持的应用程序发送电子邮件的详细信息。
    </p>
    <p align="left">
     ·
     <strong>
      Twilio
     </strong>
     提供云中的电话基础结构 Web 服务，因此开发人员可以将电话、短信和 IP 语音通信集成到移动应用程序中。Twilio 发布了一款
     <a href="http://aka.ms/mobilesampletwilio" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       iPad
      </span>
      <span style="color:#16a5d9">
       示例应用程序
      </span>
     </a>
     ，使活动组织者可以轻松收集志愿者的联系信息、使用移动服务存储并使用 Twilio Client 启用一触式通话功能。Twilio 还在
     <a href="http://aka.ms/twiliomobiletutorial" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       Windows Azure
      </span>
      <span style="color:#16a5d9">
       开发人员中心发布了新教程
      </span>
     </a>
     ，演示如何从移动服务自定义 API 脚本使用 Twilio 短信或语音功能。
    </p>
    <p align="left">
     ·
     <strong>
      Xamarin
     </strong>
     是一种允许开发人员使用 C# 创建 iOS、Android、Mac 和 Windows 应用程序的框架。Xamarin 的 Craig Dunn 最近录制了一个
     <a href="http://www.youtube.com/watch?v=3O7eFwyiS8Q" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       视频
      </span>
     </a>
     ，展示开发人员如何使用 C# 着手构建连接到云的任务列表 iOS 应用程序。
    </p>
    <p align="left">
     ·  最近由
     <strong>
      Redbit
     </strong>
     开发的
     <a href="http://aka.ms/redbitsample" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       SocialCloud
      </span>
      <span style="color:#16a5d9">
       应用程序
      </span>
     </a>
     突显了我们的合作伙伴生态系统的重要性。除了移动服务、网站和上述第三方服务外，SocialCloud 还使用 Service Bus、Linux VM 和 MongoDB。
    </p>
    <p align="left">
     <img alt="" src="http://blogs.msdn.com/resized-image.ashx/__size/550x0/__key/communityserver-blogs-components-weblogfiles/00-00-01-13-25/3513.mir3.png"/>
    </p>
    <p align="left">
     请访问 Redbit
     <a href="http://aka.ms/redbitblog" rel="nofollow noopener noreferrer" target="_blank">
      <span style="color:#16a5d9">
       博客
      </span>
     </a>
     ，详细了解他们如何构建 SocialCloud 以及为何决定将这些服务一起使用。
    </p>
