    <p>
     <font face="宋体,verdana, arial, helvetica">
      “嵌入式系统”一般指非PC系统，有计算机功能但又不称之为计算机的设备或器材。它是以应用为中心，软硬件可缩扩的，适应应用系统对功能、可靠性、成本、体积、功耗等综合性严格要求的专用计算机系统；主要由嵌入式处理器、相关支撑硬件、嵌入式操作系统及应用软件系统等组成。
      <br/>
      <br/>
      与通用型计算机系统相比，嵌入式系统功耗低、可靠性高；功能强大、性能价格比高；实时性强，支持多任务；占用空间小，效率高；面向特定应用，可根据需要灵活定制。
      <br/>
      <br/>
      嵌入式系统应用广泛：可用于移动计算平台(PDA、掌上计算机)，信息家电(数字电视、机顶盒、网络设备)，无线通信设备(智能手机、股票接收设备)，工业/商业控制(智能工控设备、POS/ATM机)，电子商务平台，甚至军事应用等诸多领域。据估计，其在全球每年带来的相关工业产值已超过1万亿美元。
      <br/>
      <br/>
      作为嵌入式技术与自动化组态软件技术的结合，“嵌入式自动化组态软件”是指用于嵌入式系统并带有网络功能(包括Internet服务功能)的嵌入式应用软件。它可大大缩短嵌入式产品投放市场的时间，而且使产品具有丰富的人机界面、嵌入式Web及符合IEC61131-3标准的控制逻辑功能，并且可以存储相当数量的历史资料，部分完成现场工作站级计算机的功能。
      <br/>
      <br/>
      前言
      <br/>
      <br/>
      进入21世纪，数字化产品欣欣向荣；由于硬件平台种类各异，因此嵌入式软件市场呈现出一种高度细分的格局，国外产品进入后也很难垄断整个市场，这为我国软件业提供了一个难得的发展机遇。
      <br/>
      <br/>
      作为国内自动化软件行业的知名公司，北京昆仑通态自动化软件科技有限公司于6月24日在国内率先推出嵌入式自动化组态软件——MCGS嵌入版组态软件，并提出“工控设备的DIY自由组态”的新理念，使用户从繁琐的编程中解脱出来，通过简单的模块化组态就可构造自己的应用系统。与此同时，他们与微软和研华科技强强联手，自6月24日至7月25日，相继在成都、西安、北京、沈阳、上海、广州、东莞、哈尔滨和南京等9个城市举办“嵌入式系统组态软件及平台之技术与应用”研讨会，在业内掀起了一股“嵌入式”旋风。本刊编辑部在此向大家介绍研讨会上海站的概况。
      <br/>
      <br/>
      主题演讲
      <br/>
      <br/>
      上海站的活动7月8日下午在世贸商城召开，有来自申城及周边地区约150名代表参加。他们除了可聆听精彩的主题演讲外，还可通过生动形象的演示系统就嵌入式系统的应用前景作深入交流。
      <br/>
      <br/>
      同在其他城市一样，本次研讨会的主题是介绍由研华、微软、昆仑通态共同推出的基于研华嵌入式设备、微软Windows CE和昆仑通态MCGS嵌入版组态软件的完整解决方案的概念、产品及应用案例。
      <br/>
      <br/>
      裔杰先生(研华公司嵌入式电脑
      <br/>
      <br/>
      事业部华东区销售经理)
      <br/>
      <br/>
      报告主题：嵌入式系统在新时代的发展趋势和技术
      <br/>
      <br/>
      裔先生指出，计算机已迈入了后PC时代。IT业者面临的3大任务是：用PC技术处理人所不能处理的事务；使每一传统设备都能连上计算机；数字化产品得到广泛普及。随着各行业对信息自动化需求的不断提高，众多非PC设备的智能化具有更广阔的前景。形式多样的数字化设备正努力把Internet渗透到生活的每一角落。将PC与传统设备结合起来，使后者具备运算和联网能力，这就是嵌入式应用，是PC技术发展的必然趋势。工业现场和远程设备同样需要进行智能化的开发和改造，因而嵌入式系统在工控界的应用前景非常广阔。
      <br/>
      <br/>
      嵌入式系统主要有4个发展趋势：(1)开放式平台架构，易于与其他系统整合；(2)体积越来越小，性能要求更稳定，成本更低廉；(3)应用趋向多元化，需要小批量、快速客制化的服务；(4)功能趋向单一，集成的功能与一般PC不同。
      <br/>
      <br/>
      研华的嵌入式系统由裔先生所在的嵌入式电脑事业部开发，主要面向中小系统集成商，其产品包括嵌入式主板(SOM系统模块、DTOS服务、单板电脑及附件、POS控制板、PC/104模块等)、嵌入式机箱、液晶平板电脑(液晶超薄一体化电脑、开架式触摸终端设备等)，以及嵌入式软件。在客户进行新产品的开发过程中，有些基本需求是可以利用标准的模块产品实现的，基于这种理念，研华推出了SOM(System On Module)产品的解决方案，使客户可专注于自己所需的功能模块进行开发、生产，而无须考虑功能复杂的CPU模块；同时，研华还可根据客户的需求，提供SOM扩展板的设计服务和样片，帮助客户在30天内ODM自己的工控主板。
      <br/>
      <br/>
      刘志军先生(昆仑通态总经理)
      <br/>
      <br/>
      报告主题：MCGS嵌入式组态软件及其解决方案
      <br/>
      <br/>
      主要介绍了最新推出的基于Windows CE的MCGS嵌入版组态软件，以及“工控设备的DIY自由组态”新理念。
      <br/>
      <br/>
      基于嵌入式操作系统(Embedded OS)的软件平台；具有MCGS通用版所有的功能；用户能够用来快速开发嵌入式应用系统；只需较少的硬件资源，可降低系统成本；稳定可靠，能长时间运行不死机；运行速度快，能实现快速采集和实时控制。这6个方面较为全面而又具体地概括了MCGS嵌入式组态软件的概念。这一组态软件的图形接口、设备驱动、控制流程和数据处理都可以组态，而且还可以方便地进行网络连接。
      <br/>
      <br/>
      不同于以往的基于PC平台的工业组态软件，MCGS嵌入版组态软件具有如下特点：功能强大，和MCGS通用版基本相同；使用方便，组态方法也与通用版一致；它是真正嵌入式的组态软件，采用Windows CE嵌入式操作系统；性价比高，只需32MB、486DX的CPU；可靠性强，使用看门狗，可自动快速启动；实时性强，方便实现快速采集和高速控制。
      <br/>
      <br/>
      到目前为止，昆仑通态推出了MCGS组态软件3大系列产品：通用版、网络版和嵌入版组态软件，形成三位一体管控一体化解决方案，可完成从工业现场设备采集数据，工作站数据处理和控制，到上位机网络管理和Web浏览的所有功能。
      <br/>
      <br/>
      (1)MCGS网络版组态软件，处于整个系统的最上层，主要完成系统的信息收集和发布，把所有监控的资料通过网络最终汇集到服务器。
      <br/>
      <br/>
      (2)MCGS通用版组态软件，处于系统的中间，完成工作站的资料加工处理以及一些日常监控事务。
      <br/>
      <br/>
      (3)MCGS嵌入版组态软件，处于系统最下层，对现场资料进行采集、处理和控制。它可与硬设备结合，开发成各种用户所需的设备，如可灵活组态的智能仪表、无纸记录仪、无人值守现场采集站等。
      <br/>
      <br/>
      与此同时，昆仑通态开创了“工控设备DIY自由组态”的新理念。用户无需再进行繁琐的编程，只需对模块进行选择和简单的组态，就能构造出适合于自己的应用系统。针对DIY，刘总提出了在实现时需解决的6个问题：
      <br/>
      <br/>
      (1)结构问题：有一个统一的类似于PLC的外壳，可连接多块PC104板；
      <br/>
      <br/>
      (2)接线问题：由于PC104是带缆连接的，因而设置装置，把带缆连接转换为接线端子；
      <br/>
      <br/>
      (3)软件问题：提高系统运行稳定性，增加了梯形图和FBD编程；
      <br/>
      <br/>
      (4)硬件问题：采用PC104，生产厂家众多，性价比较高，易于解决；
      <br/>
      <br/>
      (5)总线问题：支持Profibus、 Lonworks、Interbus；
      <br/>
      <br/>
      (6)开放性问题：以PC104为标准，让尽可能多的用户能生产、开发各种功能的硬设备。
      <br/>
      <br/>
      最后，刘总结合大量成功案例，分析了在实际工程中MCGS嵌入版组态软件的应用，并展望DIY在工业控制各领域中的美好前景。
      <br/>
      <br/>
      微软
      <br/>
      <br/>
      由于时间等因素，主办方未能安排嵌入式操作系统Windows CE的介绍，这未免让与会者有点遗憾。为保证整篇报道的完整性，本刊特收集有关资料，简单作一介绍。
      <br/>
      <br/>
      Windows CE是从整体上为有限资源的平台设计的多线程、完整优先级、多任务的操作系统。其模块化设计允许它对于从掌上电脑到专用工业控制器的用户电子设备进行定制。操作系统的基本内核至少需要200kB的ROM。随着Windows CE源码的开放，以及微软在嵌入式领域力度的加强，其定制能力有所上升；再加上可利用资源丰富，因而前景为人们所看好。
      <br/>
      <br/>
      Windows CE.Net是Windows CE 3.0的后继产品，具备完整的操作系统特性集包和端对端开发环境，包括了创建一个基于Windows CE的定制设备所需的功能，如联网能力、实时性和小内存体积占用，以及丰富的多媒体和Web浏览功能等。由于使用了MSMQ (Microsoft Message Queuing)应用服务，Windows CE使与工业生产现场现有IT设施全面集成的实现成为可能。因Windows CE能从闪存磁盘中启动，也就避免了暴露在灰尘、高温和振动环境下，从而使它可应用于恶劣的生产环境。
      <br/>
      <br/>
      来自现场的声音
      <br/>
      <br/>
      趁研讨会的间隙，本刊编辑部分别对主办方和参会代表进行了现场采访。
      <br/>
      <br/>
      研华科技嵌入式电脑事业群华东区总监梁风鸣先生：研华科技嵌入式电脑事业群(ECG)于1998年下半年开始在大陆开展业务，增长速度非常快，1999年达到200%，2000年为120%，2001年为50%。预计到2004年，国内嵌入式平台的市场将达到10亿元人民币，研华有望占据40%的份额。而据IDC预计，在全球范围内，未来5~10年，嵌入式市场的增长率将达到30%。嵌入式系统的应用范围非常广，在国内，目前研华主要关注POS、电信、医疗、电力以及彩票、金融、TBS和防火墙等行业。对于嵌入式应用而言，用户关注的不仅是产品本身，而是一个完整的解决方案，研华将加强与各行业领先的系统集成商合作，正如研华与MCGS的合作，以促进各行业嵌入式应用的发展。
      <br/>
      <br/>
      刘志军先生：昆仑通态关注嵌入式技术也有多年的历史了，在与用户多年的交流中，我们也认识到其市场需求是巨大的。今年，我们在业内率先推出了MCGS嵌入版组态软件，成为国内嵌入式组态软件的首开先河者。基于MCGS嵌入版组态软件，我们革命性地创造了开放的“工控设备的DIY”理念，用户只需要进行积木式的模块选择和简单的组态就可构造自己的应用系统。在工业应用中，昆仑通态将淡化通用组态软件的概念，全面推广嵌入式组态软件及网络版组态软件的应用。
      <br/>
      <br/>
      对于此次研讨会，与会者最大的遗憾应该是关于嵌入式系统的问题想了解的太多，而半天的时间又实在太短，所以在研讨会后许多参会者围住主办方的工程师们继续探讨就不足为奇了。绝大多数被采访者对嵌入式系统和工控DIY均表现出了浓厚的兴趣，对于嵌入式系统是工控行业的趋势和热点均表示赞同。但是基于工控行业一向严谨的作风，他们也没有对嵌入式系统目前的发展形势表现出盲目的乐观，都纷纷提出了自己的一些疑虑。
      <br/>
      <br/>
      谢先生以前曾经使用过昆仑通态的通用版软件，就此次研讨会他表示，最大的收获就是了解了嵌入式系统究竟是怎么一回事，然而要真正应用到工程中则还要等到该类产品进一步深入发展之后，毕竟像燃烧设备这样稍不留意就可能造成大事故的项目，用一些已经成熟的产品是保障安全的首选。
      <br/>
      <br/>
      姜先生曾在项目中使用过研华公司的产品，许多底层驱动软件还是自己进行开发，费时费力，所以在听完嵌入式系统的介绍后他喜出望外。尽管如此，面对我们的采访，他还是很审慎地表示，由于目前可供选择的嵌入式系统底层设备毕竟只占少数，在目前的项目中他并不打算马上采用嵌入式系统。同时，他还说出了自己的另一个担忧，现在承担嵌入式系统组态软件部分的昆仑通态公司的技术支持人员毕竟有限，如果真的大量使用嵌入式系统，不知在服务上能否得到强有力的保障。
      <br/>
      <br/>
      金先生所在的企业从事远程RTU的开发和生产，他告诉笔者，以前对嵌入式系统的概念仅是有所耳闻，通过研讨会，他有了较好的了解。以前，他们的RTU采用单片机控制，而今后，他们会尝试考虑嵌入式系统。但有两点他表示关注：一是成本会不会大幅度上升；二是技术支持能否有保障。
      <br/>
      <br/>
      短短3个半小时的交流飞快地过去了，但“嵌入式风暴”仍在继续，它会给我们带来怎样的改变？绝大部分专家和用户对嵌入式系统的应用前景持乐观态度，认为业内要像对待新生事物一样对其加以扶持，使其能健康发展。嵌入式系统应不仅仅是一个概念，要能切切实实推动科技发展，真正“走进寻常百姓家”，为技术进步作贡献，当然这需要大家的共同努力。
