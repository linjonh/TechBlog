       IT运维 - 一、IT 运维管理概述
      </span>
     </h3>
    </div>
    <p>
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:7px; padding-bottom:7px; word-wrap:break-word; word-break:break-all; line-height:25px">
    </p>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      IT 运维管理是时下 IT 界最热门的话题之一.随着 IT 建设的不断深入和完善,计算机硬
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BD%AF%E4%BB%B6" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="软件">
      软件
     </a>
     <span style="line-height:25px">
      系 统的运行维护已经成为了各行各业各单位领导和信息服务部门普遍关注和不堪重负的问题.由于这是一个随 着
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E4%BF%A1%E6%81%AF%E6%8A%80%E6%9C%AF" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="计算机信息技术">
      计算机信息技术
     </a>
     <span style="line-height:25px">
      的深入应用而产生的新课题,因此如何进行有效的 IT 运维管理,这方面的知识积累和应 用技术还刚刚起步.对这一领域的研究和探索,将具有广阔的发展前景和巨大的现实意义
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      所谓 IT
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BF%90%E7%BB%B4%E7%AE%A1%E7%90%86" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="运维管理">
      运维管理
     </a>
     <span style="line-height:25px">
      ,是指单位 IT 部门采用相关的方法、手段、技术、制度、流程和文档 等,对IT 运行环境(如硬
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BD%AF%E4%BB%B6%E7%8E%AF%E5%A2%83" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="软件环境">
      软件环境
     </a>
     <span style="line-height:25px">
      、网络环境等)、IT 业务系统和 IT 运维人员进行的综合管理。
     </span>
    </span>

       <span style="font-family:KaiTi_GB2312">
        IT运维 - 二、IT 运维管理的主要管理内容

     </h3>
    </div>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:7px; padding-bottom:7px; word-wrap:break-word; word-break:break-all; line-height:25px">

    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      IT 运维管理主要包括八个方面的管理内容:
     </span>
     <a name="5" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      1设备管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      对
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E7%BD%91%E7%BB%9C%E8%AE%BE%E5%A4%87" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="网络设备">
      网络设备
     </a>
     <span style="line-height:25px">
      、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%9C%8D%E5%8A%A1%E5%99%A8" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="服务器">
      服务器
     </a>
     <span style="line-height:25px">
      设备、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="操作系统">
      操作系统
     </a>
     <span style="line-height:25px">
      运行状况进行监控和管理;
     </span>
     <a name="7" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      2应用/服务管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      对各种应用支持
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BD%AF%E4%BB%B6" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="软件">
      软件
     </a>
     <span style="line-height:25px">
      如
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%95%B0%E6%8D%AE%E5%BA%93" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="数据库">
      数据库
     </a>
     <span style="line-height:25px">
      、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%B8%AD%E9%97%B4%E4%BB%B6" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="中间件">
      中间件
     </a>
     <span style="line-height:25px">
      、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E7%BE%A4%E4%BB%B6" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="群件">
      群件
     </a>
     <span style="line-height:25px">
      以及各种通用或特定服 务的监控管理,如邮件系统、DNS、Web 等的监控与管理;
     </span>
     <a name="9" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      3数据/存储/容灾管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      对系统和业务数据进行
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E7%BB%9F%E4%B8%80%E5%AD%98%E5%82%A8" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="统一存储">
      统一存储
     </a>
     <span style="line-height:25px">
      、备份和恢复;
     </span>
     <a name="11" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      4业务管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      包含对
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      自身核心业务系统运行情况的监控与管理,对于业务的管理, 主要关注该业务系统的 CSF(关键成功因素 Critical Success Factors)和KPI(关键绩效指 标Key Performance Indicators);
     </span>
     <a name="13" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      5目录/内容管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      该部分主要对于
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      需要统一发布或因人定制的内容管理和对公共信息的管理;
     </span>
     <a name="15" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      6资源资产管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      管理
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      中各 IT 系统的资源资产情况,这些资源资产可以是物理存在 的,也可以是逻辑存在的,并能够与企业的财务部门进行数据交互;
     </span>
     <a name="17" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      7信息安全管理
     </span>
    </h4>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:7px; padding-bottom:7px; word-wrap:break-word; word-break:break-all; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      目前
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BF%A1%E6%81%AF%E5%AE%89%E5%85%A8%E7%AE%A1%E7%90%86" rel="nofollow noopener noreferrer" style="font-family:KaiTi_GB2312; font-size:18px; color:rgb(2,104,205); text-decoration:none" target="_blank" title="信息安全管理">
      信息安全管理
     </a>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      主要依据的国际标准是 ISO17799,该标准涵盖了信 息安全管理的十大控制方面,36 个控制目标和 127 种控制方式,如
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="font-family:KaiTi_GB2312; font-size:18px; color:rgb(2,104,205); text-decoration:none" target="_blank" title="企业">
      企业
     </a>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      安全组织方式、资产 分类与控制、人员安全、物理与环境安全、通信与运营安全、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%AE%BF%E9%97%AE%E6%8E%A7%E5%88%B6" rel="nofollow noopener noreferrer" style="font-family:KaiTi_GB2312; font-size:18px; color:rgb(2,104,205); text-decoration:none" target="_blank" title="访问控制">
      访问控制
     </a>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%B8%9A%E5%8A%A1%E8%BF%9E%E7%BB%AD%E6%80%A7%E7%AE%A1%E7%90%86" rel="nofollow noopener noreferrer" style="font-family:KaiTi_GB2312; font-size:18px; color:rgb(2,104,205); text-decoration:none" target="_blank" title="业务连续性管理">
      业务连续性管理
     </a>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      等;
     </span>
    </p>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <a name="19" style="color:rgb(2,104,205); line-height:25px" target="_blank">
     </a>
     <span style="line-height:25px">
     </span>
    </span>
    <h4 style="margin:0px; padding:0px; line-height:25px">
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      8日常工作管理
     </span>
    </h4>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      该部分主要用于规范和明确运维人员的岗位职责和工作安排、提供绩 效考核量化依据、提供解决经验与知识的积累与共享手段.
     </span>
    </span>
    <div class="content_h2" style="margin:0px; padding:0px; line-height:25px">
     <h3 class="mar-t10" style="margin:25px 0px 5px; padding:5px 0px 3px; line-height:24px; border-bottom-color:rgb(217,217,217); border-bottom-style:solid; border-bottom-width:1px; clear:both">
      <a href="http://www.baike.com/wiki/IT%E8%BF%90%E7%BB%B4#" id="bjbd" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; float:right; height:14px; display:inline-block; margin-top:5px; width:50px" target="_blank">
      </a>
      <span style="font-family:KaiTi_GB2312; font-size:18px">
       <a name="21" style="color:rgb(2,104,205)" target="_blank">
       </a>
       IT运维 - 三、IT运维的实践和安全性
      </span>
     </h3>
    </div>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:7px; padding-bottom:7px; word-wrap:break-word; word-break:break-all; line-height:25px">
    </p>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      IT运维是一项系统的工程。仁通科技在给
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      配置IT运维中总结出，IT运维管理的实践最主要在于对公司IT系统的安全性、效率性和稳定性上，尤其是安全性。在重要信息保存、USB存储设备管理、文件打印、笔记本管理及外来接入管理等方面需要建立强有力的管控体系，有效防范了电子信息资产外泄、规范单机
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%A1%8C%E9%9D%A2" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="桌面">
      桌面
     </a>
     <span style="line-height:25px">
      操作行为并制约个人行为出轨现象，从而将隐患控制在事件之前。德讯
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      将信息泄露的途径可归纳为：
     </span>
    </span>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       1.电子信息违规操作导致信息泄露
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       2.USB存储设备操作导致信息泄露
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       3.
      </span>
      <a class="innerlink" href="http://www.baike.com/wiki/%E7%BB%88%E7%AB%AF" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="终端">
       终端
      </a>
      <span style="line-height:25px">
       非法外联导致信息泄露
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       4.非审核文件打印导致信息泄露
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       5.笔记本私自带出导致信息泄露
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:KaiTi_GB2312; font-size:18px">
      <span style="line-height:25px">
       6.外来人员电脑接入内部网络导致信息泄露
      </span>
     </span>

     <span style="line-height:25px">
      敏锐地发现，市场上真正功能全面且简单易用的产品并不多，更无论贯穿始终的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      能力了。因此运维市场，尤其是
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%A1%8C%E9%9D%A2" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="桌面">
      桌面
     </a>
     <span style="line-height:25px">
      运维，充斥了大量的处女地，市场潜力巨大。
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%8D%A1%E8%A5%BF%E4%BA%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="卡西亚">
      卡西亚
     </a>
     <span style="line-height:25px">
      是基于Web的新一代
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      IT
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E7%B3%BB%E7%BB%9F%E7%AE%A1%E7%90%86" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="系统管理">
      系统管理
     </a>
     <span style="line-height:25px">
      解决方案，用户可以通过一个集中的管理控制台来安全掌管其基础架构、并完全透明、远程地管理
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%9C%8D%E5%8A%A1%E5%99%A8" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="服务器">
      服务器
     </a>
     <span style="line-height:25px">
      、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%8F%B0%E5%BC%8F%E6%9C%BA" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="台式机">
      台式机
     </a>
     <span style="line-height:25px">
      、移动设备（笔记本电脑和智能手机等）以及
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%B5%8C%E5%85%A5%E5%BC%8F%E8%AE%BE%E5%A4%87" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="嵌入式设备">
      嵌入式设备
     </a>
     <span style="line-height:25px">
      。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      11、TRAMIS
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      科技风险分析管理综合解决方案（TRAMIS – Technical Risk Analysis &amp; Management Integrated Solution）发源并服务于银行业，是山东矩阵软件依据银行业监管部门对于银行业日常运行维护管理的要求和规范，为信息科技部门提供客观数据采集、审计监督和数据分析，辅助以银行业普遍使用的信息科技运行维护管理工作流程，针对银行业进行设计开发的，基于过程的综合分析管理平台。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      TRAMIS基于信息科技系统的各种客观数据，提供多种科技风险规避和审计监督的渠道，为客户提供综合的科技风险分析和管理平台，协助客户提高信息科技系统运行管理的效率。其功能也适用于电信、电力、政府、公共事业、企业等各个行业的信息科技部门对于计算机和网络系统的运行维护和分析管理。
     </span>
    </span>

      <span style="font-family:KaiTi_GB2312; font-size:18px">

       </a>
       IT运维 - IT运维管理现状
      </span>
     </h3>
    </div>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:7px; padding-bottom:7px; word-wrap:break-word; word-break:break-all; line-height:25px">
    </p>
    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
      伴着IT在
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      中的作用日益明显，IT建设和IT运维同时成为了企业效率的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%8A%A0%E9%80%9F%E5%99%A8" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="加速器">
      加速器
     </a>
     <span style="line-height:25px">
      。同时，计算机硬件系统和
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BD%AF%E4%BB%B6%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="软件系统">
      软件系统
     </a>
     <span style="line-height:25px">
      的运维已成为了各行各业单位，尤其是信息服务部门普遍头痛的事情。本文以下内容总结几个头痛的主要因子，拿出来供大家参考指导，并接下来的系列课题中会对针对这些现状提出改进措施。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      现状一：IT运维人员成本偏高
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      据专业调查，大多数
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/CIO" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="CIO">
      CIO
     </a>
     <span style="line-height:25px">
      表示最关心的是IT运维成本过高。原因是在过去的5年中,很多
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      都实施了很多IT系统,使到IT运行越来越复杂,也越来越难管理。同时，其中有50%的受访CIO认为IT运维成本过高的一个原因是IT运维的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      做得还不够好,依靠手工流程来管理,不但使到运维效率不高,而且人力成本更是花费惊人。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      同时，另一家国际知名调查机构Gartner调查发现,在IT运维成本中,源自技术或产品(包括硬件、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BD%AF%E4%BB%B6" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="软件">
      软件
     </a>
     <span style="line-height:25px">
      、网络等)成本其实只占20%,而流程维护成本占40%,运维人员成本占40%。流程维护成本包括日常维护、变更管理、测试成本等;人员成本包括训练、教育、人员流失、招聘成本等。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      从图中，我们可以看出， “流程维护”类和“运维人员”两者都与软性方面的成本相关非常紧密。而且三者的关系可以用下图来表示：

     <br style="line-height:25px"/>
     <span style="line-height:25px">
      现状二：处在“救火式”的IT运维控制
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      目前，国内在IT运维过程中,IT员工大多数只是处在被动低效率手工救火的状态,只有当事件已经发生并已造成业务影响时才能发现和着手处理。这种被动“救火”会导致：①.IT运维人员终日忙碌，IT运维人员日常大部分时间和精力是处理一些简单重复的问题;②IT运维本身质量很难提高;③再加上故障预警机制的不完善,往往是故障发生后或报警后才会进行处理,不但事倍功半而且故障还常常会出现恶性连锁反应;④IT部门和业务部门对IT运维的服务满意度都不高。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      现状三：简单的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      程度起了“反作用”
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      尽管IT运维管理的技术在不断进步,但实际上很多IT运维人员并没有真正解脱出来，主要原因是目前的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      不高而导致的。目前的技术虽然能够获取IT设备、
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%9C%8D%E5%8A%A1%E5%99%A8" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="服务器">
      服务器
     </a>
     <span style="line-height:25px">
      、网络流量,甚至
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%95%B0%E6%8D%AE%E5%BA%93" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="数据库">
      数据库
     </a>
     <span style="line-height:25px">
      的警告信息,但成千上万条警告信息堆积在一起根本没法判断问题的根源在哪里。还有,目前许多
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      的更新管理绝大多数工作都是手工操作的。即使一个简单的系统变更或更新往往都需要运维人员逐一登录每台设备进行手工变更,当设备数量达至成百上千时,其工作量之大可想而知。而这样的变更和检查操作在IT运维中往往每天都在进行,占用了大量的运维资源。因此,实现
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%BF%90%E7%BB%B4%E7%AE%A1%E7%90%86" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="运维管理">
      运维管理
     </a>
     <span style="line-height:25px">
      工作的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E8%87%AA%E5%8A%A8%E5%8C%96" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="自动化">
      自动化
     </a>
     <span style="line-height:25px">
      对
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      来说已迫在眉睫。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      就如图中一样，所有信息(杂乱)都从各个地方被收集到了这个圆圈(容量不变)里面，信息进去后不能主动流出来。可能会出现的情况：这个圆圈容器装满后会爆破，或者是溢出来;圆圈的运行速度会慢慢降下来，从而导致信息输入的速度也会变慢。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      现状四：本是同家
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%85%84%E5%BC%9F" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="兄弟">
      兄弟
     </a>
     <span style="line-height:25px">
      ，却不经常来往
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      这个问题主要是发生在拥有许多子公司的
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E4%BC%81%E4%B8%9A" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="企业">
      企业
     </a>
     <span style="line-height:25px">
      ，每个子公司的系统都是独立的，下面主要以国内银行业为例。以前国内的银行业没有搞集中建设，每家银行的各个地方分行都单独建设和维护自己的核心业务系统，都各自配备开发人员和维护人员。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      同时在运行维护方面，对
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E6%95%85%E9%9A%9C" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="故障">
      故障
     </a>
     <span style="line-height:25px">
      的解决，完全依靠运行维护部门的工程师的上门服务。不管问题大小，工程师都要来回去现场解决。遇到一些技术难度大的问题，如果工程师的水平高，处理起来就快;如果水平低，甚至花上几个小时，可能也解决不了。
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      虽然现在国内银行业的IT运行维护管理水平，有点接近国外80年代末90年代初银行业的水平，现在银行IT结构上都采用了大集中模式。从硬件设备上来看，国内银行不比别人差，甚至还有些领先，但IT运维管理还没达到国外当时的水平，尤其是呼叫中心、客户服务方面。”
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      结束语
     </span>
     <br style="line-height:25px"/>
     <span style="line-height:25px">
      从上面三个现状来看，主要是有关软性方面的。的确如此，国内借着近十几年高速发展，硬件方面的发展取得了重大进步，某些方面的水平甚至是超过了国外的水平，并且IT硬件的生产厂商也是出现了很多与国外厂商同等秀舞的水平，如
     </span>
     <a class="innerlink" href="http://www.baike.com/wiki/%E5%8D%8E%E4%B8%BA" rel="nofollow noopener noreferrer" style="color:rgb(2,104,205); text-decoration:none; line-height:25px" target="_blank" title="华为">
      华为
     </a>
     <span style="line-height:25px">
      、中兴等。但是往往是硬件易学，知识技巧难寻。这不仅与国内教育环境有关外，还与知识经验的继承有关。
     </span>
    </span>

    <h4 style="margin:0px; padding:0px; line-height:25px">

    <span style="font-family:KaiTi_GB2312; font-size:18px">
     <span style="line-height:25px">
