    <h2 class="p2" id="id-前端知识--web前端整体框架和开发流程-2前端V的架构" style="margin:30px 0px 0px; padding:0px; font-size:24px; font-weight:normal; line-height:1.25; color:rgb(51,51,51); border-bottom-color:rgb(204,204,204); font-family:Arial,sans-serif">
     <a name="t1" target="_blank">
     </a>
     2  前端V的架构
    </h2>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     前端开发刚兴起的阶段，前端开发人员的工作经常穿插在后端开发过程中担任辅助作用的，随着业务越来越复杂，前端必须有自己的开发框架和流程，使得后端专注于服务类的开发，前端专注于与用户交互类的开发。
    </p>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     WEB前端从
     <span style="color:#ff00">
      概念上划分为三个层次：结构、表现、和行为
     </span>
     。三层相对独立，互不影响。在
     <span style="color:#ff00">
      物理上层面上分别对应HTML、CSS、JS三种不同的的文件格式
     </span>
     。
    </p>
    <ul style="margin:10px 0px 0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <li class="p2">
      HTML：负责定义网页的结构，内容
     </li>
     <li class="p2">
      CSS：负责内容的展现方式
     </li>
     <li class="p2">
      JS：负责网页关于动态的行为反应
     </li>
    </ul>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <img alt="" class="confluence-embedded-image image-center" src="http://wiki.sankuai.com/download/attachments/117783218/image2014-12-8%2014%3A34%3A19.png?version=1&amp;modificationDate=1418020352000&amp;api=v2" style="margin:0px auto; padding:0px; border:0px; display:block; max-width:100%" width="400"/>
    </p>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     好的层次划分对前端的开发也很关键。
     <span style="color:#ff00">
      显示内容问题的唯一来源应该是html文件定义的，内容展示问题的唯一来源应该是CSS文件定义的，行为问题的唯一来源应该是javascript文件定义的，各司其职相互独立。但是他们是相关的，因为行为和样式无法与结构分离，必须写在同一个文件里。这种情况下，首先还是你要把行为和样式从html标签中分开，然后在html文件中引入
     </span>
     。
    </p>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     实际的开发结构是这样的：
    </p>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <img alt="" class="confluence-embedded-image image-center" src="http://wiki.sankuai.com/download/attachments/117783218/%E5%B1%8F%E5%B9%95%E5%BF%AB%E7%85%A7%202014-12-08%20%E4%B8%8B%E5%8D%883.03.15.png?version=1&amp;modificationDate=1418022166000&amp;api=v2" style="margin:0px auto; padding:0px; border:0px; display:block; max-width:100%" width="400"/>
    </p>
    <p class="p2" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     前端开发的核心是HTML + CSS + JavaScript。
     <span style="color:#ff00">
      本质上它们构成一个MVC框架
     </span>
     ，即HTML作为信息模型（Model），CSS控制样式（View），JavaScript负责调度数据和实现某种展现逻辑（Controller）。
    </p>
    <h3 class="p2" id="id-前端知识--web前端整体框架和开发流程-1）结构层" style="margin:30px 0px 0px; padding:0px; font-size:20px; font-weight:normal; line-height:1.5; color:rgb(51,51,51); border-bottom-color:rgb(204,204,204); font-family:Arial,sans-serif">
     <a name="t2" target="_blank">
     </a>
     1）结构层
    </h3>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     在java领域，表现层技术主要有三种：jsp、freemarker、velocity。
    </p>
    <h4 id="id-前端知识--web前端整体框架和开发流程-(1)jsp(JavaServerPages)" style="margin:30px 0px 0px; padding:0px; font-size:16px; line-height:1.5625; color:rgb(51,51,51); font-family:Arial,sans-serif; text-align:justify">
     <a name="t3" target="_blank">
     </a>
     (1)  jsp(Java Server Pages)
    </h4>
    <p class="p1" style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <span class="s1">
      它是在传统的网页HTML文件(.htm，.html)中插入Java程序段(Scriptlet)和JSP标记(tag)，它实现
      <span style="color:#ff00">
       Html语法中的java扩展（以 &lt;%, %&gt;形式）
      </span>
      。从而形成JSP文件，后缀名为(.jsp)。因为java具有跨平台特点， 用JSP开发的Web应用是跨平台的，既能在Linux下运行，也能在其他操作系统上运行。
     </span>
     <span style="color:#ff00">
      JSP与Servlet一样，是在服务器端执行的，最终都要变.class文件，通常返回给客户端的就是一个HTML文本
     </span>
     ，因此客户端只要有浏览器就能浏览。
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     优点：
    </p>
    <ul style="margin:10px 0px 0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <li class="p1">
      <span class="s1" style="color:#ff00">
       可以写java代码
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       <span style="color:#ff00">
        支持jsp标签（jsp tag）
       </span>
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       支持表达式语言（el）
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       官方标准，用户群广，丰富的第三方jsp标签库
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       性能良好，jsp编译成class文件执行，有很好的性能表现
      </span>
     </li>
    </ul>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     <br/>
     缺点：
    </p>
    <ul style="margin:10px 0px 0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <li style="text-align:justify">
      编写java代码，如使用不当容易破坏mvc结构。
     </li>
    </ul>
    <h4 id="id-前端知识--web前端整体框架和开发流程-(2)Freemarker" style="margin:30px 0px 0px; padding:0px; font-size:16px; line-height:1.5625; color:rgb(51,51,51); font-family:Arial,sans-serif; text-align:justify">
     <a name="t4" target="_blank">
     </a>
     (2) Freemarker
    </h4>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     它生成静态页面，首先需要使用自己定义的模板页面，这个模板页面可以是最最普通的html，也可以是嵌套freemarker中的 取值表达式， 标签或者自定义标签等等，然后后台读取这个模板页面，解析其中的标签完成相对应的操作， 然后采用键值对的方式传递参数替换模板中的的取值表达式，做完之后 根据配置的路径生成一个新的html页面， 以达到静态化访问的目的。
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     <span style="color:#ff00">
      模板+数据模型=输出，模板只负责数据在页面中的表现，不涉及任何的逻辑代码，而所有的逻辑都是由数据模型来处理的。用户最终看到的输出是模板和数据模型合并后创建的。
     </span>
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     <span style="color:#ff00">
      FreeMarker与Web容器无关，即在Web运行时，它并不知道Servlet或者其他。
     </span>
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     优点：
    </p>
    <ul style="margin:10px 0px 0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <li class="p1">
      <span style="color:#000000">
       <span style="color:#ff00">
        可以彻底的分离表现层和业务逻辑，
       </span>
      </span>
      <span style="color:#000000">
       使用JSP 开发过程中在页面中大量的存在业务逻辑的代码，使得页面内容凌乱，在后期大量的修改维护过程中就变得非常困难。
      </span>
      <span style="color:#ff00">
       FreeMarker根本不支持Java代码。
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       <span style="color:#000000">
        <span style="color:#ff00">
         可以提高开发效率，
        </span>
        JSP在第一次执行的时候需要转换成Servlet类，开发阶段进行功能调适时，需要频繁的修改JSP，每次修改都要编译和转换
       </span>
       。
       <span style="color:#000000">
        FreeMarker模板技术不存在编译和转换的问题，开发过程中，后端不必在等待界面设计开发人员完成页面原形后，再来开发程序。
       </span>
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       对jsp标签支持良好
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       内置大量常用功能，
       <span style="color:#000000">
        比如html过滤，日期金额格式化等等
       </span>
      </span>
     </li>
     <li class="p1">
      使用表达式语言 (EL)
     </li>
    </ul>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     缺点：
    </p>
    <ul style="margin:10px 0px 0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px">
     <li class="p1">
      <span class="s1">
       不是官方标准
      </span>
     </li>
     <li class="p1">
      <span class="s1">
       用户群体和第三方标签库没比jsp丰富
      </span>
     </li>
    </ul>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     <span style="color:#000000">
      使用FreeMarker后，作为界面开发人员，只专心创建HTML文件、图像以及Web页面的其他可视化方面，不用理会数据；而程序开发人员则专注于系统实现，负责为页面准备要显示的数据。
     </span>
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     <span style="color:#000000">
     </span>
     还有一些像velocity这样较早出现的用于代替jsp的模板语言 ... ...
    </p>
    <h3 id="id-前端知识--web前端整体框架和开发流程-2)表现层CSS" style="margin:30px 0px 0px; padding:0px; font-size:20px; font-weight:normal; line-height:1.5; color:rgb(51,51,51); border-bottom-color:rgb(204,204,204); font-family:Arial,sans-serif; text-align:justify">
     <a name="t5" target="_blank">
     </a>
     2) 表现层CSS
    </h3>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     结构搭建之后，就是为他们添加样式表属性。
     <span style="color:#ff00">
      它实现的是页面内容和现实样式分离，所有的CSS都是非必需的，就算无法应用CSS或CSS冲突，网页也能够正常访问。
     </span>
    </p>
    <p style="margin-top:10px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; color:rgb(51,51,51); font-family:Arial,sans-serif; font-size:16px; line-height:26.6666660308838px; text-align:justify">
     为了管理的方便，可以将不同样式进行拆分，比如可以拆分为全局样式，结构布局样式，色彩样式，文字样式和重置样式。c
     <span class="s1" style="line-height:1.4285715; text-align:left">
