    <blockquote>
     <p>
      整体概述：
      <br/>
      在我尝试运行
      <code>
       test_deal_file.py
      </code>
      时，我遇到了一个
      <code>
       ModuleNotFoundError
      </code>
      错误，Python告诉我找不到名为
      <code>
       controllers
      </code>
      的模块。这意味着我无法从
      <code>
       deal_file.py
      </code>
      中导入
      <code>
       read_excel
      </code>
      函数。
     </p>
     <p>
      为了解决这个问题，我尝试了几种方法。
      <br/>
      首先，我使用了相对导入，这在不改变当前工作目录的情况下是有效的，但一旦我切换了目录，这个方法就不行了。
      <br/>
      接着，我尝试动态地将当前工作目录添加到
      <code>
       sys.path
      </code>
      中。
      <br/>
      这样做虽然有效，但我觉得这种方法不够优雅。
     </p>
     <p>
      所以，我决定采取一种更彻底的解决方案。
      <br/>
      我通过修改 VSCode 的
      <code>
       settings.json
      </code>
      文件，设置了
      <code>
       PYTHONPATH
      </code>
      环境变量，确保了我的项目路径
      <code>
       my_project
      </code>
      被加入到模块搜索路径中。
      <br/>
      我在settings.json
      <code>
       中为不同的操作系统配置了
      </code>
      terminal.integrated.env` 部分，然后重启了虚拟环境和集成终端。重新运行我的文件后，问题终于解决了。
     </p>
     <p>
      通过这篇文章，我想和大家分享如何以一种更优雅的方式解决 Python 项目中的模块导入问题。
      <br/>
      同时，我也介绍了如何在 VSCode 中管理项目环境变量和运行路径，这对于解决开发过程中的环境配置问题非常有帮助。希望我的经验能成为其他开发者在面对类似问题时的宝贵资源。
     </p>
    </blockquote>
    <h2>
     <a id="_16">
     </a>
     我的问题解决过程
    </h2>
    <p>
     <strong>
      （想直接看解决教程的直接跳到下一个标题内容）
     </strong>
     <br/>
     我的项目结构是这样的
    </p>
    <pre><code class="prism language-bash">my_project/
├── controllers/
│   ├── __init__.py     
│   └── deal_file.py     
└── tests/
    ├── __init__.py      
    └── test_deal_file.py 
</code></pre>
    <p>
     结果运行test_deal_file.py会出现报错信息：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 报错信息</span>
    from controllers.deal_file <span class="token function">import</span> read_excel
ModuleNotFoundError: No module named <span class="token string">'controllers'</span>
</code></pre>
    <hr/>
    <p>
     我尝试了引用改成
     <code>
      from ..controllers.deal_file import read_excel
     </code>
     ，terminal不切换目录可以运行成功，但切换目录后还是运行失败
    </p>
    <hr/>
    <p>
     我又尝试了
    </p>
    <pre><code class="prism language-bash"><span class="token function">import</span> sys
<span class="token function">import</span> os
sys.path.append<span class="token punctuation">(</span>os.getcwd<span class="token punctuation">(</span><span class="token punctuation">))</span>
</code></pre>
    <p>
     但是说实话，不是很美观
    </p>
    <hr/>
    <p>
     我决定从根本上解决这个问题。
    </p>
    <p>
     首先我打印我所有运行路径
    </p>
    <pre><code class="prism language-bash"><span class="token function">import</span> sys
paths <span class="token operator">=</span> sys.path
<span class="token keyword">for</span> <span class="token for-or-select variable">path</span> <span class="token keyword">in</span> paths:
    print<span class="token punctuation">(</span>path<span class="token punctuation">)</span>
</code></pre>
    <p>
     输出结果果然，不包含我
     <code>
      my_project
     </code>
     的路径。如何把my_project路径添加到系统运行路径呢？方法如下:
    </p>
    <hr/>
    <h2>
     <a id="vscode___62">
     </a>
     vscode 解决模块导入问题的流程
    </h2>
    <p>
     <strong>
      1. 打开设置界面
     </strong>
     <br/>
     在 VSCode 中，您可以通过点击左下角的齿轮图标然后选择 “Settings” 来打开设置界面。
     <br/>
     另一种方式是使用快捷键 Ctrl + ,（在 Mac 上是 Cmd + ,）直接打开设置界面。
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/96359bee67e2d967b0884451a895d31e.png">
      <br/>
      <strong>
       2. 选择合适的工作区
      </strong>
      <br/>
      打开setting界面后，上面有三个选项，对应着三种级别的设置文件工作区（workspace）、远程（remote）和用户（user）。每个级别的设置都有其特定的用途和作用范围。
     </img>
    </p>
    <blockquote>
     <p>
      <strong>
       用户设置（User settings）
      </strong>
      ： 位于全局级别，适用于您计算机上的所有 VSCode 实例。 路径通常在
      <br/>
      ~/.config/Code/User/settings.json（Unix/Linux）或
      <br/>
      %APPDATA%\Code\User\settings.json（Windows）。 用于存储您希望在所有项目和环境中使用的全局配置。
      <br/>
      <strong>
       工作区设置（Workspace settings）
      </strong>
      ： 特定于当前打开的工作区或项目。 路径通常在工作区根目录下的 .vscode/settings.json 文件中。 用于存储针对特定项目或工作区的配置，这些配置不会影响其他项目。
      <br/>
      <strong>
       远程设置（Remote settings
      </strong>
      ）： 特定于远程开发环境，例如通过 SSH 或容器连接的远程服务器。 路径通常在 .vscode-remote/settings.json 文件中，这个文件位于您的本地工作区目录下。 用于存储针对远程开发环境的配置，如
      <br/>
      SSH 连接设置、容器设置等。
     </p>
    </blockquote>
    <p>
     现在你需要选择适合你的设置文件：
     <br/>
     如果环境变量仅适用于特定项目，您应该在工作区设置文件中添加它们。
     <br/>
     如果环境变量需要在所有项目中使用，您应该在用户设置文件中添加它们。
     <br/>
     如果环境变量是针对远程开发环境的，您应该在远程设置文件中添加它们。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d83effac744ae4337198782fff2ad6e6.png"/>
    </p>
    <p>
     <strong>
      3. 打开对应工作区的setting.json页面
     </strong>
     <br/>
     往下滑找到Edit in setting.json，点击打开
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/17be4f7552d6b46c4673e0fbd069c78b.png">
      <br/>
      <strong>
       4. 编辑setting.json
      </strong>
      <br/>
      接下来你要修改这个文件
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/aa890c4e2d5c6964414d2b724be6f404.png">
       <br/>
       下面内容复制到你的setting.json文件中，并保存
      </img>
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">{<!-- --></span>
    <span class="token string">"terminal.integrated.env.windows"</span><span class="token builtin class-name">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string">"PYTHONPATH"</span><span class="token builtin class-name">:</span> <span class="token string">"/path/to/your/project"</span>
    <span class="token punctuation">}</span>,
    <span class="token string">"terminal.integrated.env.linux"</span><span class="token builtin class-name">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string">"PYTHONPATH"</span><span class="token builtin class-name">:</span> <span class="token string">"/path/to/your/project"</span>
    <span class="token punctuation">}</span>,
    <span class="token string">"terminal.integrated.env.osx"</span><span class="token builtin class-name">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string">"PYTHONPATH"</span><span class="token builtin class-name">:</span> <span class="token string">"/path/to/your/project"</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a4315250935af19a96a31d999cdcf7b6.png"/>
    </p>
    <ol start="5">
     <li>
      <p>
       重启虚拟环境
       <br/>
       Unix/Linux 系统：
       <code>
        source /path/to/venv/bin/activate
       </code>
       <br/>
       Windows 系统：
       <code>
        \path\to\venv\Scripts\activate
       </code>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bf3d5c9df3e4fb96b9ba99884e520cd7.png"/>
      </p>
     </li>
     <li>
      <p>
       关闭terminal集成终端
       <br/>
       在 VSCode 中，关闭当前打开的集成终端。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/628b3c00999209f27276cbce73a5a7a0.png"/>
      </p>
     </li>
     <li>
      <p>
       打开新的集成终端：
       <br/>
       通过点击 VSCode 顶部的 “Terminal” 菜单或者使用快捷键 Ctrl+ (在 Mac 上是Cmd+ ) 来打开一个新的集成终端窗口。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8e12de334dd3d2d0111131183e0d5df0.png"/>
      </p>
     </li>
     <li>
      <p>
       重新运行你的文件
       <br/>
       可以看到你的项目路径已经添加到环境，项目也可以正常运行了👌
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/596840e9384f9579405a5bd24114c951.png"/>
      </p>
     </li>
    </ol>
