    <h3>
     OpenHarmony介绍
    </h3>
    <p>
     OpenHarmony是由开放原子开源基金会（OpenAtom Foundation）孵化及运营的开源项目，目标是面向全场景、全连接、全智能时代，搭建一个智能终端设备操作系统的框架和平台，促进万物互联产业的繁荣发展。
    </p>
    <p>
     开源代码仓库地址：
     <a href="https://openharmony.gitee.com/" rel="nofollow" title="https://openharmony.gitee.com">
      https://openharmony.gitee.com
     </a>
     。
    </p>
    <h3>
     获取源码概述
    </h3>
    <p>
     OpenHarmony当前为开发者提供了以下4种获取源码的方式，开发者可根据业务诉求选用合适的方式获取源码：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        获取方式1：
       </strong>
       从码云代码仓库获取。通过repo或git工具从代码仓库中下载，此方式可获取最新代码。
      </p>
     </li>
     <li>
      <p>
       <strong>
        获取方式2：
       </strong>
       通过
       <a href="https://repo.harmonyos.com/#/cn/home" rel="nofollow" title="DevEco Marketplace">
        DevEco Marketplace
       </a>
       网站获取。访问
       <a href="https://repo.harmonyos.com/#/cn/home" rel="nofollow" title="DevEco Marketplace">
        DevEco Marketplace
       </a>
       网站，查找满足需求的开源发行版，直接下载（或者定制后下载），再通过hpm-cli命令工具将所需的组件及工具链下载、安装到本地。
      </p>
     </li>
     <li>
      <p>
       <strong>
        获取方式3：
       </strong>
       从镜像站点下载归档后的发行版压缩文件。如果要获取旧版本的源码，也可通过此方式获取，此方式下载速度较快。
      </p>
     </li>
     <li>
      <p>
       <strong>
        获取方式4：
       </strong>
       从github代码仓库获取。通过repo或git工具从代码仓库中下载，此方式可获取最新代码。
      </p>
     </li>
    </ul>
    <h3>
     获取方式1：从码云仓库获取
    </h3>
    <h4>
     适用场景
    </h4>
    <ul>
     <li>
      基于OpenHarmony的稳定分支建立自己的基线，分发下游客户。
     </li>
     <li>
      已经完成自身软件与OpenHarmony的对接，需要进行OpenHarmony官方认证。
     </li>
     <li>
      芯片/模组/app通过OpenHarmony官方认证后，贡献代码到OpenHarmony社区。
     </li>
     <li>
      修复OpenHarmony的问题。
     </li>
     <li>
      学习OpenHarmony的源码。
     </li>
    </ul>
    <h3>
     前提条件
    </h3>
    <ul>
     <li>
      注册码云gitee账号。
     </li>
     <li>
      注册码云SSH公钥，请参考
      <a href="https://gitee.com/help/articles/4191" rel="nofollow" title="码云帮助中心">
       码云帮助中心
      </a>
      。
     </li>
     <li>
      安装
      <a href="https://git-scm.com/book/zh/v2/%E8%B5%B7%E6%AD%A5-%E5%AE%89%E8%A3%85-Git" rel="nofollow" title="git客户端">
       git客户端
      </a>
      和
      <a href="https://gitee.com/vcs-all-in-one/git-lfs?_from=gitee_search#downloading" rel="nofollow" title="git-lfs">
       git-lfs
      </a>
      并配置用户信息。
      <pre>git config --global user.name "yourname"
git config --global user.email "your-email-address"
git config --global credential.helper store</pre>
     </li>
     <li>
      安装码云repo工具，可以执行如下命令。
      <pre>curl https://gitee.com/oschina/repo/raw/fork_flow/repo-py3 -o /usr/local/bin/repo  #如果没有权限，可下载至其他目录，并将其配置到环境变量中
chmod a+x /usr/local/bin/repo
pip3 install -i https://repo.huaweicloud.com/repository/pypi/simple requests</pre>
     </li>
    </ul>
    <h4>
     操作步骤
    </h4>
    <p>
     <strong>
      说明：
     </strong>
     Master主干为开发分支，开发者可通过Master主干获取最新特性。发布版本代码相对比较稳定，开发者可基于发布版本代码进行商用功能开发。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        OpenHarmony主干代码获取
       </strong>
      </p>
      <p>
       方式一（推荐）：通过repo + ssh下载（需注册公钥，请参考
       <a href="https://gitee.com/help/articles/4191" rel="nofollow" title="码云帮助中心">
        码云帮助中心
       </a>
       ）。
      </p>
      <pre>repo init -u git@gitee.com:openharmony/manifest.git -b master --no-repo-verify
repo sync -c
repo forall -c 'git lfs pull'</pre>
      <p>
       方式二：通过repo + https下载。
      </p>
      <pre>repo init -u https://gitee.com/openharmony/manifest.git -b master --no-repo-verify
repo sync -c
repo forall -c 'git lfs pull'</pre>
     </li>
     <li>
      <p>
       <strong>
        OpenHarmony发布版本代码获取
       </strong>
      </p>
      <p>
       OpenHarmony发布版本获取源码方式请参考
       <a href="https://gitee.com/openharmony/docs/blob/OpenHarmony-3.1-Release/zh-cn/release-notes/Readme.md" rel="nofollow" title="Release-Notes">
        Release-Notes
       </a>
       。
      </p>
     </li>
    </ul>
