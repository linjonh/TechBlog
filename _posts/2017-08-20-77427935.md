    <p>
     昨天很荣幸受邀参加微软第一届为打造微软技术开发者社区生态圈现在会议，我是一名从.net转到unity的游戏开发者，也算半个微软技术圈的人，一直对微软技术比较钟爱，因为这个，我也”唆使”我的老婆成为一名.net开发者。参会的有来自各个技术社区和产业界的朋友。不但有.NET社区的，还有google、java、nodejs、linux等等开源社会的专家或者社区组织者共20余人。这届会议不同于以往只是主办人读PPT的流程，下面的参会者聆听而已，这次更多的是参会人员的发言和交流。
    </p>
    <p>
     这次会议是微软走向开源，在以云战略为基础的新姿态。更多的是和技术社区的交流，更多的关注技术社区的发展，虚心改变服务的交流会，这让更多微软技术开发爱好者看到了未来的希望，现在开源是大势所趋，微软在14年更换CEO之后大力倡导走开源路线，才不会让微软的生态越走越窄，我之前从朋友身边了解到，他们公司的.NET产品不得不转型用java开发，我也问过为什么，朋友的回答是公司招人难。同样的问题，对于开发者而言，找工作也难，这样的话大环境只会一步一步的恶化，可能微软也有这种意识，所以现在重新重视技术生态圈的发展，这种线下交流会会越来越多，这也让微软技术开发者看到了未来的希望。
    </p>

            <br/>
            2.介绍docs.microsoft.com，一个在线学习文档，包含了微软技术各个方向，非常非常详细。

            <br/>
            3.自我介绍流程
            <br/>
            每个人都一一做了自我介绍，一开始我还是有点紧张的，介绍了我是一个名unity游戏开发者，后面有各个大公司以及社区的发起人以及活跃人，也有我熟悉的杨振平老师，之前在在ＣＳＤＮ交流会碰到过，还有令我印象深刻的高博老师和罗阳铭老师，罗老师跟张善友老师都是社区大人。
            <br/>
            4.围绕两个主题展开讨论”云时代开发者技术文档学习的探讨”和“云时代开发者生态环境的打造”
            <br/>
            针对这两个主题，各各老师都是发表了自己的观点，我也主动踊跃的做了两次发言，表达了一些我的观点和建议。我提议微软不仅仅要关注线上生态圈的发展，更要注重线下生态圈的交流和发展，我认识的因特尔老大和携程Baidu他们几个老大经常组织线下Spark交流活动，还有Unity北京线下社区以雨凇为主的社区开发者经常组织线下活动，因此我也提议微软也要不定期的组织以某个技术领域大牛为首的线下交流活动，促进整个微软技术生态圈的发展。
            <br/>
            关于其他各位老师我也有仔细聆听他们中肯的建议，以下我做一个简单的小结，有不对之处还请见谅：
            <br/>
            a.微软线上文档要根据不同层次的开发者做不同的定位适配，目前微软文档MSDN和现在的Docs也非常的不错，但并不适合所有层次的开发者。
            <br/>
            b.希望微软跟高校合作，让学生尽快尽早了解微软技术。
            <br/>
            c.微软要重视社区草根开发者，多给他们一些赞助，让他们维系微软技术社区，有一些开发者还是很有影响力，比如张善友老师，我心目中.NET的国内第一人，一直在助推.NET技术。
            <br/>
            d.微软多多组织线下活动，让公司和更多的开发者参与进来，对公司而言能够更多的接触某个岗位需要的人才，对于开发者而言也能更好的接触和了解公司。
