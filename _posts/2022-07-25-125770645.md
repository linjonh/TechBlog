    <h2>
     <a id="1_12">
     </a>
     1.小程序的宿主环境
    </h2>
    <p>
     <font color="red">
      <strong>
       手机微信是小程序的宿主环境，
      </strong>
     </font>
     小程序借助宿主环境提供的能力，可以实现许多普通网页不能完成的功能。例如：小程序调用微信提供的API实现扫码，支付等功能。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9c6253d05132d7f010f2fef119fb3d3c.png"/>
    </p>
    <blockquote>
     <p>
      <strong>
       小程序的宿主环境包含的内容：
      </strong>
     </p>
     <ul>
      <li>
       通信模型
      </li>
      <li>
       运行机制
      </li>
      <li>
       组件
      </li>
      <li>
       API
      </li>
     </ul>
    </blockquote>
    <h2>
     <a id="2_24">
     </a>
     2.通信模型
    </h2>
    <h3>
     <a id="1_25">
     </a>
     1.通信的主体
    </h3>
    <p>
     ✋小程序中通信的主体是
     <font color="blue">
      渲染层
     </font>
     和
     <font color="blue">
      逻辑层
     </font>
     ，其中：
    </p>
    <ol>
     <li>
      <strong>
       WXML 模板和 WXSS 样式工作在渲染层
      </strong>
     </li>
     <li>
      JS 脚本工作在逻辑层
     </li>
    </ol>
    <h3>
     <a id="2_30">
     </a>
     2.小程序的通信模型
    </h3>
    <p>
     ✋小程序的通信模型分为两部分：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/390f5e2e2d2464f69a7d1c16da47770b.png"/>
    </p>
    <ol>
     <li>
      <font color="blue">
       <strong>
        渲染层
       </strong>
       和
       <strong>
        逻辑层
       </strong>
       之间的通信
      </font>
     </li>
     <li>
      <font color="blue">
       <strong>
        逻辑层
       </strong>
       和
       <strong>
        第三方服务器
       </strong>
       之间的通信
      </font>
     </li>
    </ol>
    <blockquote>
     <p>
      两者都通过微信客户端进行转发
     </p>
    </blockquote>
    <h2>
     <a id="3_41">
     </a>
     3.运行机制
    </h2>
    <h3>
     <a id="1_42">
     </a>
     1.小程序的启动过程
    </h3>
    <ol>
     <li>
      ✋把小程序的代码包下载到本地
     </li>
     <li>
      ✋解析 app.json 全局配置文件
     </li>
     <li>
      ✋执行 app.js 小程序入口文件，
      <font color="blue">
       <strong>
        调用 App() 创建小程序实例
       </strong>
      </font>
     </li>
     <li>
      ✋渲染小程序首页
     </li>
     <li>
      ✋小程序启动完成
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1e926245b8e500f46d51fc490600c09a.png"/>
    </p>
    <h3>
     <a id="2_53">
     </a>
     2.页面渲染的过程
    </h3>
    <ol>
     <li>
      ✋加载解析页面的 .json 配置文件
     </li>
     <li>
      ✋加载页面的 .wxml 模板和 .wxss 样式
     </li>
     <li>
      ✋执行页面的 .js 文件，
      <font color="blue">
       <strong>
        调用 Page() 创建页面实例
       </strong>
      </font>
     </li>
     <li>
      ✋页面渲染完成
     </li>
    </ol>
    <h2>
     <a id="4_58">
     </a>
     4.组件
    </h2>
    <h3>
     <a id="1_59">
     </a>
     1.小程序中组件的分类：
    </h3>
    <p>
     ✋小程序中的组件也是由
     <font color="red">
      <strong>
       宿主环境
      </strong>
     </font>
     提供的，开发者可以基于组件快速搭建出漂亮的页面结构。官方把小程序的组件分为了 9 大类，分别是：
    </p>
    <ol>
     <li>
      <font color="blue">
       <strong>
        视图容器
       </strong>
      </font>
     </li>
     <li>
      <font color="blue">
       <strong>
        基础内容
       </strong>
      </font>
     </li>
     <li>
      <font color="blue">
       <strong>
        表单组件
       </strong>
      </font>
     </li>
     <li>
      <font color="blue">
       <strong>
        导航组件
       </strong>
      </font>
     </li>
     <li>
      体组件
     </li>
     <li>
      map 地图组件
     </li>
     <li>
      canvas 画布组件
     </li>
     <li>
      开放能力
     </li>
     <li>
      无障碍访问
     </li>
    </ol>

    <h2>
     <a id="5API_231">
     </a>
     5.API
    </h2>
    <p>
     ✋
     <strong>
      小程序中的 API 是由宿主环境提供的，通过这些丰富的小程序 API，开发者可以方便的调用微信提供的能力，例如：实现支付，扫码等功能。
     </strong>
    </p>
    <p>
     小程序 API 的 3 大分类：
    </p>
    <p>
     <strong>
      🌏事件监听 API
     </strong>
    </p>
    <ul>
     <li>
      特点：
      <font color="red">
       以 on 开头，用来监听某些事件的触发
      </font>
     </li>
     <li>
      举例：wx.onWindowResize(function callback) 监听窗口尺寸变化的事件
     </li>
    </ul>
    <p>
     <strong>
      🌏同步 API
     </strong>
    </p>
    <ul>
     <li>
      特点1：以 Sync 结尾的 API 都是同步 API
     </li>
     <li>
      特点2：
      <font color="red">
       同步 API 的执行结果，可以通过函数返回值直接获取，如果执行出错会抛出异常
      </font>
     </li>
     <li>
      举例：wx.setStorageSync(‘key’, ‘value’) 向本地存储中写入内容
     </li>
    </ul>
    <p>
     <strong>
      🌏异步 API
     </strong>
    </p>
    <ul>
     <li>
      特点：
      <font color="red">
       类似于 jQuery 中的 $.ajax(options) 函数，需要通过 success、fail、complete 接收调用的结果
      </font>
     </li>
     <li>
      举例：wx.request() 发起网络数据请求，通过 success 回调函数接收数据
     </li>
    </ul>
