    <h2>
     <a id="_PDF__0">
     </a>
     前端实现 PDF 预览的常见方案
    </h2>
    <p>
     由于在搭建个人博客时，想实现在线预览 pdf 格式的个人简历，经过查阅大致有三大类实现方案；本文共涉及以下 5 种实现方案，如下所示：
    </p>
    <ul>
     <li>
      <p>
       使用 HTML 标签
      </p>
      <ul>
       <li>
        iframe 标签
       </li>
       <li>
        embed 标签
       </li>
       <li>
        object 标签
       </li>
      </ul>
     </li>
     <li>
      <p>
       使用第三方插件
      </p>
      <ul>
       <li>
        PDF.js
       </li>
       <li>
        PDFObject
       </li>
      </ul>
     </li>
     <li>
      <p>
       PDF 文件转化成图片进行展示
      </p>
     </li>
     <li>
      <p>
       第一类方案：
       <strong>
        使用 HTML 标签
       </strong>
       的实现预览效果最为简单，但兼容性最差，仅支持部分 PC 端的浏览器，移动端浏览器兼容非常差。
      </p>
     </li>
     <li>
      <p>
       第二类方案：
       <strong>
        使用第三方库
       </strong>
       的功能强大，还可对 pdf 文件进行操作，PC 端兼容性比较好，但在移动端会有 pdf 文件显示不全的 bug 出现，而且在移动端浏览器兼容性一般。
      </p>
     </li>
     <li>
      <p>
       第三类方案：
       <strong>
        PDF 文件转化成图片进行展示
       </strong>
       的兼容性最好，能同时在 PC 端和移动端预览，但由于是图片格式，所以选中 pdf 文件上的文件，更加无法继续宁操作。
       <br/>
       因为我的个人博客尚未实现适配移动端，所以选择了第二类方案。
      </p>
     </li>
    </ul>
    <h3>
     <a id="1_HTML__18">
     </a>
     1.使用 HTML 标签
    </h3>
    <p>
     该类方法实现简单，但兼容性很差，慎重选择。
    </p>
    <h4>
     <a id="11_iframe__22">
     </a>
     1.1 iframe 标签
    </h4>
    <p>
     HTML 内联框架元素 iframe 表示嵌套的 browsing context。它能够将另一个 HTML 页面嵌入到当前页面中。
    </p>
    <ul>
     <li>
      <a href="https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/iframe" rel="nofollow">
       iframe 标签的属性含义请参照 MDN
      </a>
     </li>
     <li>
      <a href="https://caniuse.com/mdn-html_elements_iframe" rel="nofollow">
       iframe 标签的兼容性请参照 Can I Use
      </a>
      <br/>
      iframe 标签实现方案代码如下所示：
     </li>
    </ul>
    <pre><code class="prism language-js"><span class="token operator">&lt;</span>iframe src<span class="token operator">=</span><span class="token string">"./test.pdf"</span> height<span class="token operator">=</span><span class="token string">"900px;"</span> width<span class="token operator">=</span><span class="token string">"800px"</span><span class="token operator">&gt;</span><span class="token operator">&lt;</span><span class="token operator">/</span>iframe<span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="12_embed__34">
     </a>
     1.2 embed 标签
    </h4>
    <p>
     HTML embed 元素将外部内容嵌入文档中的指定位置。此内容由外部应用程序或其他交互式内容源（如浏览器插件）提供。
    </p>
    <ul>
     <li>
      <a href="https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/iframe" rel="nofollow">
       embed 标签的属性含义请参照 MDN
      </a>
     </li>
     <li>
      <a href="https://caniuse.com/mdn-html_elements_embed" rel="nofollow">
       embed 标签的兼容性请参照 Can I Use
      </a>
     </li>
    </ul>
    <p>
     embed 标签实现方案代码如下所示：
    </p>
    <pre><code class="prism language-js"><span class="token operator">&lt;</span>embed src<span class="token operator">=</span><span class="token string">"./test.pdf"</span> type<span class="token operator">=</span><span class="token string">"application/pdf"</span> width<span class="token operator">=</span><span class="token string">"100%"</span> height<span class="token operator">=</span><span class="token string">"100%"</span> <span class="token operator">/</span><span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="13_object__47">
     </a>
     1.3 object 标签
    </h4>
    <p>
     HTML object 元素（或者称作 HTML 嵌入对象元素）表示引入一个外部资源，这个资源可能是一张图片，一个嵌入的浏览上下文，亦或是一个插件所使用的资源。
    </p>
    <ul>
     <li>
      <a href="https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/object" rel="nofollow">
       object 标签的属性含义请参照 MDN
      </a>
     </li>
     <li>
      <a href="https://caniuse.com/mdn-html_elements_object" rel="nofollow">
       object 标签的兼容性请参照 Can I Use
      </a>
     </li>
    </ul>
    <p>
     object 标签实现方案代码如下所示：
    </p>
    <pre><code class="prism language-js"><span class="token operator">&lt;</span>object
  data<span class="token operator">=</span><span class="token string">"./test.pdf"</span>
  type<span class="token operator">=</span><span class="token string">"application/pdf"</span>
  width<span class="token operator">=</span><span class="token string">"100%"</span>
  height<span class="token operator">=</span><span class="token string">"100%"</span>
<span class="token operator">&gt;</span><span class="token operator">&lt;</span><span class="token operator">/</span>object<span class="token operator">&gt;</span>
</code></pre>
    <hr/>
    <h3>
     <a id="2_67">
     </a>
     2.使用第三方插件
    </h3>
    <p>
     能实现实时预览 pdf 的插件还有许多种，但使用最多的是 PDF.js 与 PDFObject，所以本文只涉及这两个插件。
    </p>
    <h4>
     <a id="21_PDFjs_71">
     </a>
     2.1 PDF.js
    </h4>
    <h5>
     <a id="211_PDFjs__73">
     </a>
     2.1.1 PDF.js 简介
    </h5>
    <p>
     PDF.js 是一款开源的 pdf 文档读取解析插件，可以实现在 html 下直接浏览 pdf 文档。
    </p>
    <ul>
     <li>
      pdf.js 是基于
      <strong>
       Promise 对象
      </strong>
      而实现的，不了解的读者可以先去看看
      <a href="https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise" rel="nofollow">
       MDN 上的解释
      </a>
      。
     </li>
     <li>
      pdf.js 渲染 pdf 时底层还使用了
      <strong>
       Web Worker
      </strong>
      （这会导致我们无法直接在本地运行官网下载的 demo，得在服务器上运行，详情见注意点处），不了解的读者可以去看一下
      <a href="http://www.ruanyifeng.com/blog/2018/07/web-worker.html" rel="nofollow">
       阮一峰老师关于 Web Worker 的文章
      </a>
      。
     </li>
    </ul>
    <p>
     PDF.js 主要分为 3 层：
     <br/>
     显示层采用核心层，并公开更容易使用的 API 来呈现 pdf 和从文档中获取其他信息。
    </p>
    <table>
     <thead>
      <tr>
       <th align="center">
        层级
       </th>
       <th align="center">
        对应文件
       </th>
       <th align="center">
        作用
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        Core Layer（核心层）
       </td>
       <td align="center">
        pdf.worker.js
       </td>
       <td align="center">
        核心层用于解析和解释二进制 PDF 文档，这一层是所有后续层的基础。一般我不会直接操作核心层，而是去操作由核心层封装的展示层，操作核心层的高级用法可去参照
        <a href="http://mozilla.github.io/pdf.js" rel="nofollow">
         官网
        </a>
       </td>
      </tr>
      <tr>
       <td align="center">
        Display Layer（展示层）
       </td>
       <td align="center">
        pdf.js
       </td>
       <td align="center">
        显示层是对核心层进行了一个封装，从而得到更容易使用的 API，用来展示 pdf 或从文档中获取其他信息。
       </td>
      </tr>
      <tr>
       <td align="center">
        Viewer Layer（查看器层）
       </td>
       <td align="center">
        viewer.html+viewer.css+viewer.js
       </td>
       <td align="center">
        查看器构建在显示层上，是 PDF 查看器的 UI。
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     更多的细节请参照
     <a href="http://mozilla.github.io/pdf.js" rel="nofollow">
      官网
     </a>
    </p>

     方法二
    </h6>
    <p>
     直接使用官方封装好的 viewer.html 来展示自己的 PDF 文档，该方法比较简单，不用去操作 API；而且功能比较齐全，还可复制 pdf 中的文本。

    <h5>
     <a id="213__PDFjs__163">
     </a>
     2.1.3 使用 PDF.js 的注意事项
    </h5>
    <p>
     这里的内容非常重要，可以避免踩坑。我就是在这浪费了足足两天的时间，最终才找到解决办法。
    </p>
    <h6>
     <a id="pdfjs__167">
     </a>
     注意事项一：pdf.js 需要启动服务器才能运行，无法直接打开本地的文档
    </h6>
    <ul>
     <li>
      <p>
       原因：因为 pdf.js 渲染 pdf 文档时使用了 Web Worker 技术，该 Web Worker 无法读取本地文件。
       <a href="http://www.ruanyifeng.com/blog/2018/07/web-worker.html" rel="nofollow">
        可参见阮一峰老师关于 Web Worker 的文章
       </a>
      </p>
     </li>
     <li>
      <p>
       报错信息：
       <strong>
        Message: Missing PDF “file:///D:/%E6%A1%8C%E9%9D%A2/pdfjs/web/test.pdf”
       </strong>
      </p>
     </li>
     <li>
      <p>
       解决办法：可通过 live-server 这个插件在本地启动服务器然后打开相应的 html 文件。
      </p>
     </li>
     <li>
      <p>
       具体步骤：
      </p>
      <ol>
       <li>
        npm i live-server 安装包
       </li>
       <li>
        live-server 启动服务器
        <a href="http://t.zoukankan.com/flame540-p-14084542.html" rel="nofollow">
         更多配置信息可参见此博客
        </a>
       </li>
       <li>
        打开相应的 html 文件
       </li>
      </ol>
     </li>
    </ul>
    <h6>
     <a id="_IDMInternet_Download_Manager_178">
     </a>
     注意事项二：我们的电脑上不能安装 IDM（Internet Download Manager）这类软件或插件。
    </h6>
    <ul>
     <li>
      报错信息 ：
      <strong>
       Unexpected server response (204) while retrieving PDF
      </strong>
     </li>
     <li>
      原因：因为 IDM 会拦截可下载的资源，会导致页面无法预览。
     </li>
     <li>
      解决办法：直接卸载或关闭相应的插件、软件；也可以对软件进行相应的设置。
      <a href="https://blog.csdn.net/chengoes/article/details/106323091">
       更多信息可参照博文
      </a>
     </li>
    </ul>
    <p>
     另外使用 pdf.js 打开发票等文件时可能会出现字体显示不全的 bug，可参照该博客
     <a href="http://t.zoukankan.com/KingJames-p-11763590.html" rel="nofollow">
      解决 pdf.js 无法完全显示 pdf 文件内容的问题
     </a>
     。
    </p>
