    <h4>
     <strong>
      1、什么是SQL？
     </strong>
    </h4>
    <blockquote>
     <p>
      结构化查询语言(Structured Query Language)简称SQL，是一种数据库查询语言。
     </p>
     <p>
      作用：用于存取数据、查询、更新和管理关系数据库系统。
     </p>
    </blockquote>
    <h4>
     <strong>
      2、什么是MySQL?
     </strong>
    </h4>
    <blockquote>
     <p>
      MySQL是一个关系型数据库管理系统，由瑞典MySQL AB 公司开发，属于 Oracle 旗下产品。MySQL 是最流行的关系型数据库管理系统之一，在 WEB 应用方面，MySQL是最好的 RDBMS (Relational Database Management System，关系数据库管理系统) 应用软件之一。在Java企业级开发中非常常用，因为 MySQL 是开源免费的，并且方便扩展。
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      3、什么是主键?
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        主键是表格里的(一个或多个)字段，只用来定义表格里的行;主键里的值总是唯一的。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       <strong>
        4、什么是外键?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        外键是一个用来建立两个表格之间关系的约束。这种关系一般都涉及一个表格里的主键字段与另外一个表格(可能是同一个表格)里的一系列相连的字段。那么这些相连的字段就是外键。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        5、什么是触发器?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        触发器是一种专用类型的存储过程，它被捆绑到SQL Server 的表格或者视图上。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        6、触发器的作用？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        触发器是一个特殊的存储过程，主要是通过事件来触发而被执行的。
       </span>
      </span>
      <span style="color:#0d0016;">
       <span style="background-color:#ffffff;">
        它可以强化约束，来维护数据的完整性和一致性，
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        可以跟踪数据库内的操作从而不允许未经许可的更新和变化。可以联级运算。如，某表上的触发器上包含对另一个表的数据操作，而该操作又会导致该表触发器被触发。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        7、什么是存储过程？用什么来调用？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="color:#0d0016;">
       <span style="background-color:#ffffff;">
        存储过程是一个预编译的SQL语句，优点是允许模块化的设计，就是说只需创建一次，以后在程序中就可以调用多次。如果某次操作需要执行多次SQL，使用存储过程比单纯SQL语句执行要快。可以
       </span>
       <span style="background-color:#ffffff;">
        用一个“execute
       </span>
       <span style="background-color:#ffffff;">
        存储过程名
       </span>
       <span style="background-color:#ffffff;">
        参数”命令来调用存储过程
       </span>
       <span style="background-color:#ffffff;">
        。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        8、索引的作用？和它的优点缺点是什么？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="color:#0d0016;">
       <span style="background-color:#ffffff;">
        索引就一种特殊的查询表，数据库的搜索引擎可以利用它加速对数据的检索。它很类似与现实生活中书的目录，不需要查询整本书内容就可以找到想要的数据。
       </span>
       <span style="background-color:#ffffff;">
        索引可以是唯一的，创建索引允许指定单个列或者是多个列。缺点是它减慢了数据录入的速度，同时也增加了数据库的尺寸大小。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        9、主键和索引的区别?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        1、主键
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        是
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        为了标识数据库记录
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        的
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        唯一性,不允许记录重复,且键值不能为空,主键也是一个特殊索引;
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        2、数据表中只允许有一个主键,但是可以有多个索引;
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        3、使用主键数据库会自动创建主索引,也可以在非主键上创建索引,方便查询效率;
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        4、索引可以提高查询速度,它就相当于字典的目录,可以通过它很快查询到想要的结果,而不需要进行全表扫描;
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        5、
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        唯一索引则标识该索引值唯一,可以由一个或几个字段组成,一个表可以有多个唯一索引
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        10、什么是内存泄漏？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        一般我们所说的内存泄漏指的是堆内存的泄漏。当应用程序用关键字new等创建对象时，就从堆中为它分配一块内存，使用完后由于某种原因程序未释放或无法释放，造成系统内存的浪费。
       </span>
       <span style="color:#333333;">
        导致程序运行速度减慢甚至系统崩溃等严重后果
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        11、维护数据库的完整性和一致性，你喜欢用触发器还是自写业务逻辑？为什么？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        我是这样做的，尽可能使用约束，如check,主键，外键，非空字段等来约束，这样做效率最高，也最方便。其次是使用触发器，这种方法可以保证，无论什么业务系统访问数据库都可以保证数据的完整新和一致性。最后考虑的是自写业务逻辑，但这样做麻烦，编程复杂，效率低下。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        12、什么是事务？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        事务就是被绑定在一起作为一个逻辑工作单元的SQL语句组，如果任何一个语句操作失败那么整个操作就被失败，进而回滚到操作前状态，或者是上个节点。为了确保要么执行，要么不执行，就可以使用事务。要将一组语句作为事务考虑，就需要通过ACID测试，即原子性，一致性，隔离性和持久性。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        13、什么是锁？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        在所有的DBMS中，锁是实现事务的关键，锁可以保证事务的完整性和并发性。与现实生活中锁一样，它可以使某些数据的拥有者，在某段时间内不能使用某些数据或数据结构。当然锁还分级别的。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        14、什么叫视图？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        视图是一种虚拟的表，具有和物理表相同的功能。可以对视图进行增，改，查，操作，视图通常是由一个表或者多个表的行或列的子集。它使得我们获取数据更容易，相比多表查询。
       </span>
      </span>
     </p>
    </blockquote>

     <strong>
      <strong>
       <strong>
        NULL是什么意思?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">

        NULL(空)这个值表示UNKNOWN(未知):它不表示“”(空字符串)。不能把任何值与一个 UNKNOWN值进行比较，都会生产一个NULL值。您必须使用IS NULL操作符。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        18、什么是索引，有哪些索引，具体怎么用?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        索引是与表或视图关联的磁盘上结构，可以加快从表或视图中检索行的速度。索引包含由表或视图中的一列或多列生成的键。
       </span>
      </span>
      更通俗的说，索引就相当于目录。为了方便查找书中的内容，通过对内容建立索引形成目录。索引是一个文件，它是要占据物理空间的。
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        在数据库系统中建立索引主要有以下作用：
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        （1）、快速取数据；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        （2）、保证数据记录的唯一性；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        （3）、实现表与表之间的参照完整性；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        （4）、在使用ORDER by、group by子句进行数据检索时，利用索引可以减少排序和分组的时间。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        索引分为
       </span>
       <span style="color:#4da8ee;">
        聚集索引
       </span>
       <span style="color:#000000;">
        和
       </span>
       <span style="color:#4da8ee;">
        非聚集索引
       </span>
       <span style="color:#000000;">
        。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       <strong>
        19、聚集索引和非聚集索引区别
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        聚集索引，数据按索引顺序存储，中子结点存储真实的物理数据
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        非聚集索引，存储指向真正数据行的指针
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        20、您如何确保一个带有名为Fld1字段的TableB表格里只具有Fld1字段里的那些值，而这些值同时在名为TableA的表格的Fld1字段里?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="text-align:left;">
      <span style="background-color:#ffffff;">
       <span style="background-color:#ffffff;">
        <span style="color:#000000;">
         第一个答案是使用外键限制。外键限制用来维护引用的完整性。它被用来确保表格里的字段只允许有已经在另一表格里的某个字段里定义了的值。通常是另外一个表格的主键。
        </span>
       </span>
      </span>
     </p>
     <p style="text-align:left;">
      <span style="background-color:#ffffff;">
       <span style="background-color:#ffffff;">
        <span style="color:#000000;">
         另外一种答案是触发器。触发器可以被用来保证以另外一种方式实现与限制相同的作用，但是它非常难设置与维护，而且性能一般都很糟糕。
        </span>
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        21、对一个投入使用的在线事务处理表格有过多索引需要有什么样的性能考虑?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        对一个表格的索引越多，数据库引擎用来更新、插入或者删除数据所需要的时间就越多，因为在数据操控发生的时候索引也必须要维护。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        22、你可以用什么来确保表格里的字段只接受特定范围里的值?
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        可以使用Check约束，它在数据库表格里定义，用来限制输入该列的值。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">

        25、什么是SQL注入式攻击？
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        所谓SQL注入式攻击，就是攻击者把SQL命令插入到Web表单的输入域或页面请求的查询字符串，欺骗服务器执行恶意的SQL命令。比如：攻击者在用户名字和密码输入框中输入"'或'1'='1"之类的内容。最后得到的SQL命令可能变成：SELECT * from Users WHERE login = '' or '1'='1' AND password = '' or '1'='1'。这时，已经不能真正验证用户身份，所以系统会错误地授权给攻击者。
       </span>
      </span>
     </p>

       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        只要在利用表单输入的内容构造SQL命令之前，把所有输入内容过滤一番就可以了。过滤输入内容可以按多种方式进行。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">

       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        第一：替换单引号，即把所有单独出现的单引号改成两个单引号，防止攻击者修改SQL命令的含义。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">

       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        第三：对于用来执行查询的数据库帐户，限制其权限。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ⑵、 用存储过程来执行所有的查询。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ⑶ 、限制表单或查询字符串输入的长度。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ⑷、 检查用户输入的合法性，确信输入的内容只包含合法的数据。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">

      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        inner join
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        A,B的所有记录都选出，没有的记录以null代替
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        cross join (笛卡尔积)
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        A中的每一条记录和B中的每一条记录生成一条记录
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        例如A中有4条，B中有4条，cross join 就有16条记录
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        42、有哪些锁,select时怎么加排它锁
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        乐观锁
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ，悲观锁，排它锁，共享锁，更新锁，表锁，
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        行级锁
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        乐观锁
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ：乐观锁不会锁住任何东西，也就是说，它不依赖数据库的事务机制，乐观锁完全是应用系统层面的东西。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        悲观锁：悲观锁是指假设并发更新冲突会发生，所以不管冲突是否真的发生，都会使用锁机制
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        排它锁: 可以防止并发事务对资源进行访问。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        共享锁: 允许并发事务在封闭式并发控制下读取资源。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        更新锁:是共享锁和排他锁的结合。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        行级锁：
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        单独的一行记录加锁
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        表锁：
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        锁住整个表，可以同时读，写不行
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        在Select语句中加for update是给相应的行增加排他锁。Select出来的数据别的事务不能读取，不能修改、不能删除。
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        43、死锁怎么解决
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        找到进程号，kill 进程
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        产生死锁的原因：
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        一是系统提供的资源数量有限，不能满足每个进程的使用；二是多道程序运行时，进程推进顺序不合理。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        产生死锁的必要条件是：
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        1、互斥条件；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        2、不可剥夺条件（不可抢占）；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        3、部分分配；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        4、循环等待。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        根据产生死锁的四个必要条件，只要使其中之一不能成立，死锁就不会出现。为此，可以采取下列三种预防措施：
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        1、采用资源静态分配策略，破坏"部分分配"条件；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        2、允许进程剥夺使用其他进程占有的资源，从而破坏"不可剥夺"条件；
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        3、采用资源有序分配法，破坏"环路"条件。
       </span>
      </span>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        解除死锁常常采用下面两种方法：1、资源剥夺法；2、撤消进程法
       </span>
      </span>
     </p>
    </blockquote>
    <h4 style="text-align:justify;">
     <strong>
      <strong>
       <strong>
        44、最左匹配原则
       </strong>
      </strong>
     </strong>
    </h4>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        最左匹配原则是针对索引的
       </span>
      </span>
      <span style="color:#000000;">
       ，
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        举例来说：两个字段（name,
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        age
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        ）建立联合索引，如果where age=12这样的话，是没有利用到索引的，这里我们可以简单的理解为先是对name字段的值排序，然后对age的数据排序，如果直接查age的话，这时就没有利用到索引了，查询条件where name=’xxx’ and age=xx 这时的话，就利用到索引了。因为
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        创建复合索引的规则是首先会对复合索引的最左边的，也就是第一个name字段的数据进行排序，在第一个字段的排序基础上，然后再对后面第二个的
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        age
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        字段进行排序。其实就相当于实现了类似 order by name age这样一种排序规则。所以：第一个name字段是绝对有序的，而第二字段就是无序的了。所以通常情况下，直接使用第二个
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        age
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        字段进行条件判断是用不到索引的，当然，可能会出现上面的使用index类型的索引。这就是所谓的为什么要强调最左
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        匹配
       </span>
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#000000;">
        原则的原因。
       </span>
      </span>
     </p>
    </blockquote>

      </span>
     </p>
