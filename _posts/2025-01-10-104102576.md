    <h4>
     <a id="61__2">
     </a>
     6.1 图优化理论
    </h4>
    <p>
     课程介绍：
    </p>
    <ul>
     <li>
      这期开始不讲滤波器的内容了（gmapping），只讲图优化（karto、cartographer）。
     </li>
     <li>
      但是滤波器在视觉slam之中的应用还是非常广泛的。
     </li>
     <li>
      三种slam方向：
     </li>
    </ul>
    <p>
     <strong>
      滤波
     </strong>
     估计当前位姿，**平滑（图优化）**估计过去位姿，**预测（暂未接触）**估计未来位姿；
    </p>
    <h4>
     <a id="_611_Graphbased_SLAM_11">
     </a>
     – 6.1.1 Graph-based SLAM
    </h4>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/e998a6e9e78876b1585afecef33ff70e.png"/>
    </p>
    <ul>
     <li>
      2Dslam问题会比较简单，因为只是欧式空间的问题，不用考虑李代数等问题。
     </li>
     <li>
      激光和视觉的差异：
      <ul>
       <li>
        激光一般只进行pose位姿优化（精度较高）；节点表示机器人位姿；
       </li>
       <li>
        视觉精度比较差，除了pose还有其他的；
       </li>
      </ul>
     </li>
     <li>
      回环检测：走了一圈之后，不同的两个节点1、5到了同样的位置，产生一个回环，可以根据边（约束）计算相对空间变换；即
      <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/2da9ee1234015a520bb56006462ecfa9.png">
       节点1和5相互满足两条约束。
      </img>
     </li>
     <li>
      求解目标：由于里程计的误差，导致误差不为零，上述两条边误差不为零，所以需要
      <strong>
       使误差尽可能趋向零
      </strong>
      ；这个
      <strong>
       是图优化
      </strong>
      的
      <strong>
       目标
      </strong>
      ；
     </li>
     <li>
      抽象问题：求解两个节点的位姿变换过程，由于旋转矩阵涉及sin和cos计算，实际上是非线性计算，即
      <strong>
       图优化slam
      </strong>
      依然是可以抽象成，求解
      <strong>
       非线性最小二乘
      </strong>
      问题；
     </li>
     <li>
      前端和后端的差别：
      <ul>
       <li>
        前端：节点和节点之间构建边（约束）、节点连线过程，以及之前的过程都是前端问题；
       </li>
       <li>
        后端：节点连成边后，构建了图后，让误差最小化的就是后端优化；优化是预测和观测问题；
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_612__24">
     </a>
     – 6.1.2 非线性最小二乘
    </h4>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/824ad7547dc8c99438c51cabd7b8da70.png"/>
    </p>
    <ul>
     <li>
      状态方程f(x)可以认为是一个预测过程；
     </li>
     <li>
      x是
      <strong>
       预测值
      </strong>
      ；
     </li>
     <li>
      状态向量是机器人位姿robot pose；
     </li>
     <li>
      z表示的是
      <strong>
       观测值
      </strong>
      ，即传感器值；由于噪声，f(x)和z不相等；
     </li>
     <li>
      由于数学算法的局限，需要把非线性问题，转化成一个线性的问题求解；
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/9ddfb7e7af0800767898b01d14e99c79.png"/>
    </p>
    <ul>
     <li>
      <strong>
       勘误
      </strong>
      ：f2(x) = z’2 z2
     </li>
     <li>
      目标：找到最优的x，让
      <strong>
       观测
      </strong>
      和
      <strong>
       预测
      </strong>
      的误差最小化；
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/884fec42cb187c0d4a378940a887d4a9.png"/>
    </p>
    <ul>
     <li>
      人造的
      <strong>
       传感器
      </strong>
      通常服从
      <strong>
       高斯分布
      </strong>
      （因为人造的模块化、相互独立等因素）**。**自然界造物一般服从贝塔分布或其他；
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/72b7347e677f461102cf1e8c82e5d430.png"/>
    </p>
    <ul>
     <li>
      直接想法的解释：极值点的导数必定零；
     </li>
     <li>
      非线性问题：需要线性化才能求解，使用
      <strong>
       泰勒展开
      </strong>
      方法；

    </ul>

    <h4>
     <a id="_613_slam_65">
     </a>
     – 6.1.3 非线性最小二乘在slam中的应用
    </h4>
    <ul>
     <li>
      <strong>
       图的构建
      </strong>
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/71e543e1763fc51ae9a567c9d3f3bf89.png"/>
    </p>
    <ul>
     <li>
      <strong>
       两者特点
      </strong>
      <ul>
       <li>
        里程计：时间上是
        <strong>
         连续
        </strong>
        的，连续的
        <strong>
         机器人位姿
        </strong>
        才可以构建为
        <strong>
         图
        </strong>
        ；
       </li>
       <li>
        回环检测：时间上是
        <strong>
         不连续
        </strong>
        的；不同时间的节点可以通过边（约束）匹配构建成图；
       </li>
      </ul>
     </li>
     <li>
      <strong>
       误差函数
      </strong>
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/08e16d4b689baf650744a40eb523cfb8.png"/>
    </p>
    <ul>
     <li>
      xi和xj之间的相对位姿（节点j在节点i的坐标，即
      <strong>
       xi逆*xj
      </strong>
      ）通过帧间匹配算法求得（ICP、NDT等）；
     </li>
     <li>
      <strong>
       观测值Zij
      </strong>
      和**预测值Zij’**的相对坐标计算，
      <strong>
       观测值从传感器得
      </strong>
      ，
      <strong>
       预测值从里程计推算
      </strong>
      。
     </li>
     <li>
      <strong>
       误差函数
      </strong>
      定义：即两个位姿变换关系，**Zij的逆*Zij’**矩阵。
     </li>
     <li>
      实际上上述问题都是第一节课提到的
      <strong>
       坐标位姿变换关系
      </strong>
      问题，这就是所谓的误差函数。
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/45783702aa6b28c5e20670f9f47226e8.png"/>
    </p>
    <ul>
     <li>
      误差函数的矩阵形式，实际就是上一段
      <strong>
       观测值和预测值
      </strong>
      推导的内容；
     </li>
     <li>
      这就是为什么说2Dslam问题是
      <strong>
       机器人位姿图
      </strong>
      问题，抽象。
     </li>
     <li>
      <strong>
       接下来的问题，实际上就是非线性函数的线性化问题；
      </strong>
     </li>
     <li>
      <strong>
       误差函数的线性化
      </strong>
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/240290eac3a79e159b9aa1ca4ee81853.png"/>
    </p>
    <ul>
     <li>
      上图的是一个jacobian矩阵，属于稀疏的；
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/cf778005e134f7b5c8606c22667e4b6b.png"/>
    </p>
    <ul>
     <li>
      因为jacobian是稀疏的向量，所以H矩阵（黑森矩阵）也是稀疏的；
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/fac6a8d60cf683efe94d47ba1ec46951.png"/>
    </p>
    <ul>
     <li>
      最终也是一个稀疏矩阵；
     </li>
     <li>
      H（黑森）矩阵是一个稀疏矩阵，就可以进行快速的求解；
     </li>
     <li>
      <strong>
       已知H、b，则由（H▽X = b），可求解▽X
      </strong>
      ；
     </li>
     <li>
      <strong>
       但是还需要解决坐标系的转化问题
      </strong>
      ，需要增加固定的坐标系给xi、xj，如下图：
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/a447b8ab4c50a8be15d754241d8b9a89.png"/>
    </p>
    <ul>
     <li>
      设第一帧位姿为
      <strong>
       世界坐标系
      </strong>
      的
      <strong>
       原点
      </strong>
      ，即增加固定坐标系的约束，后续求解都在世界坐标系。△X1=0；如下图
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/a771dea2916ab9d1a69fef5fbca6b362.png"/>
    </p>
    <ul>
     <li>
      <strong>
       构建线性系统
      </strong>
     </li>
     <li>
      以上所有过程已经推导了后端图优化必备信息，可以开始
      <strong>
       构建线性系统
      </strong>
      ：
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/e2ddb55f66ca6c048f04d094f9df1f02.png"/>

    <h4>
     <a id="62_cartographer_129">
     </a>
     6.2 cartographer代码讲解
    </h4>
    <p>
     在整体的原有框架中代码主要分为两个部分：
     <strong>
      cartographer
     </strong>
     和
     <strong>
      cartographer_ros
     </strong>
     。
    </p>
    <ul>
     <li>
      <strong>
       cartographer
      </strong>
      主要负责处理来自雷达、IMU和里程计的数据并基于这些数据进行地图的构建，是cartographer理论的底层实现。建图过程主要分为前端Local累加式建图与后端Global位姿图构建与回环匹配。
     </li>
     <li>
      <strong>
       cartographer_ros
      </strong>
      则基于ros的通信机制获取传感器的数据并将它们转换成cartographer中定义的格式传递给cartographer处理，与此同时也将cartographer的处理结果发布用于显示或保存，在低层算法和机器人操作系统ROS之间提供了交互的接口。
     </li>
    </ul>
    <p>
     <img alt="图片" src="https://i-blog.csdnimg.cn/blog_migrate/2a4c8ff411025e7804b0a83f4cdc6e0c.png"/>
    </p>
    <p>
     其中输入项中
     <strong>
      Range Data的激光数据为必须
     </strong>
     ，
     <strong>
      其余均为非必须
     </strong>
     。
    </p>
    <p>
     整体的建图可分为Local前端部分和Global后端回环部分。
    </p>
    <p>
     <strong>
      特点：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       基于图优化的slam算法
      </strong>
     </li>
     <li>
      <strong>
       比较完善的匹配系统，包含建图和定位；
      </strong>
     </li>
     <li>
      <strong>
       目前效果最好的开源激光slam系统；
      </strong>
     </li>
     <li>
      <strong>
       有人专门维护，不断更新增加新特性，从没停止；
      </strong>
     </li>
    </ul>
    <p>
     2Dslam代码。（源码用了大量C++高级特性）
    </p>
