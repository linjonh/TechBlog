    <h2>
     <a id="1_Java__1">
     </a>
     1. Java 的值传递
    </h2>
    <p>
     通常认为
     <mark>
      Java 方法传参数都是值传递
     </mark>
     ，关于
     <code>
      值传递
     </code>
     的定义如下：
    </p>
    <blockquote>
     <p>
      在方法被调用时，实参通过形参把它的
      <code>
       内容副本传入方法内部
      </code>
      ，此时
      <mark>
       形参接收到的内容是实参值的一个拷贝
      </mark>
      ，因此在方法内对形参的任何操作，都仅仅是对这个副本的操作，不影响原始值的内容
     </p>
    </blockquote>
    <p>
     简单的代码示例和打印结果如下
    </p>
    <pre><code class="prism language-java">   <span class="token keyword">public</span> <span class="token keyword">static</span> <span class="token keyword">void</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token class-name">String</span><span class="token punctuation">[</span><span class="token punctuation">]</span> args<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

    <span class="token punctuation">}</span>

    <pre><code class="prism language-javascript">changeTest before<span class="token operator">:</span><span class="token number">10</span>

</code></pre>
    <p>
     以上
     <code>
      变量a定义为基本数据类型 int，我们知道它是存储在虚拟机栈内存中的
     </code>
     。
     <mark>
      虚拟机栈是Java方法执行的内存模型，栈中存放着栈帧，每个栈帧分别对应一个被调用的方法，方法的调用过程其实就是栈帧在虚拟机中入栈到出栈的过程。当程序中某个线程开始执行一个方法时就会相应的创建一个栈帧并且入栈（位于栈顶），在方法结束后，栈帧出栈
     </mark>
    </p>
    <blockquote>
     <p>
      <code>
       栈帧
      </code>
      : 用于支持虚拟机进行方法调用和方法执行的数据结构，它是虚拟机运行时数据区中的虚拟机栈的栈元素，每个栈帧中包括：
     </p>
     <ol>
      <li>
       <code>
        局部变量表
       </code>
       <br/>
       用来存储方法中的局部变量（非静态变量、函数形参）。
       <mark>
        当变量为基本数据类型时，直接存储值，当变量为引用类型时，存储的是指向具体对象的引用
       </mark>
      </li>
      <li>
       <code>
        操作数栈
       </code>
       <br/>
       Java虚拟机的解释执行引擎被称为"基于栈的执行引擎"，其中的栈就是指操作数栈
      </li>
      <li>
       <code>
        指向运行时常量池的引用
       </code>
       <br/>
       存储程序执行时可能用到常量的引用
      </li>
      <li>
       <code>
        方法返回地址
       </code>
       <br/>
       存储方法执行完成后的返回地址
      </li>
     </ol>
    </blockquote>
    <p>
     了解以上情况后其实很好解释示例代码的结果:
    </p>
    <ol>
     <li>
      <p>
       首先调用
       <code>
        main()
       </code>
       方法，此时JVM为
       <code>
        main()
       </code>
       方法往虚拟机栈中压入一个栈帧，即为
       <mark>
        当前栈帧
       </mark>
       ，用来存放
       <code>
        main()
       </code>
       中的局部变量表(包括参数)、操作栈、方法出口等信息，如
       <code>
        a
       </code>
       是
       <code>
        main()
       </code>
       方法中的局部变量，此时虚拟机栈如图所示
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ee7488294e405a441ccf1481921c2aaa.png#pic_center"/>
      </p>
     </li>
     <li>
      <p>
       当执行到
       <code>
        changeTest()
       </code>
       方法时，JVM也为其往虚拟机栈中压入一个栈帧，用来存放
       <code>
        changeTest()
       </code>
       中的局部变量等信息，因此方法参数
       <code>
        src
       </code>
       是在
       <code>
        changeTest()
       </code>
       方法所在的栈帧中，而其值是从
       <code>
        a
       </code>
       复制得到的。此时虚拟机栈如图所示，在
       <code>
        changeTest()
       </code>
       方法栈帧内部操作变量
       <code>
        src
       </code>
       显然不会影响到
       <code>
        main()
       </code>
       方法栈帧变量
       <code>
        a
       </code>
      </p>
      <p>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/36894df0ebda6bd408ead80a56ab83c4.png#pic_center"/>
      </p>
     </li>
    </ol>
    <h2>
     <a id="2_Java__42">
     </a>
     2. Java 的"引用传递"
    </h2>
    <p>
     <code>
      引用传递
     </code>
     的定义如下：
    </p>
    <blockquote>
     <p>
      <mark>
       引用
      </mark>
      也就是指向真实内容的地址值，在方法调用时，
      <mark>
       实参的地址通过方法调用被传递给相应的形参
      </mark>
      ，在方法体内，形参和实参指向
      <mark>
       同一块内存地址，对形参的操作会影响到真实内容
      </mark>
     </p>
    </blockquote>
    <p>
     上文解释了
     <code>
      值传递
     </code>
     的过程，但是以下
     <mark>
      方法入参为引用类型数据
     </mark>
     的
     <code>
      示例代码1
     </code>
     的打印结果与其明显不符。可以看到，在 main() 方法中，
     <code>
      List
     </code>
     类型变量
     <code>
      a
     </code>
     没有做任何的元素操作，但是经过
     <code>
      changeTest()
     </code>
     方法添加了元素 99之后，
     <code>
      main()
     </code>
     方法中的
     <code>
      List
     </code>
     变量
     <code>
      a
     </code>
     也被加入了元素 99。
     <mark>
      内容发生了改变，这似乎完全符合“引用传递”的定义，对形参的操作，改变了实际对象的内容
     </mark>
    </p>

    <p>
     要解释以上结果，首先我们要知道
     <code>
      变量a定义为对象类型 List，它是存储在虚拟机堆内存中的
     </code>
     。JVM 中堆内存是一块线程共享的区域，对象和数组一般都分配在这一块内存中。这样，我们就可以很容易地解释：
    </p>
    <ol>
     <li>
      程序执行到
      <code>
       main()
      </code>
      方法中的代码
      <code>
       List&lt;Integer&gt; a = new ArrayList&lt;&gt;();
      </code>
      时，JVM会在堆内开辟一块内存，用来存储
      <code>
       List
      </code>
      对象的所有内容，同时在
      <code>
       main()
      </code>
      方法所在线程的栈帧中创建一个引用
      <code>
       a
      </code>
      存储堆区中
      <code>
       List
      </code>
      对象的内存地址，如图
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/88577052f015d0ce44aca67074bb45ed.png#pic_center"/>

     <li>
      对于
      <code>
       示例代码1
      </code>
      ，执行到
      <code>
       changeTest()
      </code>
      方法时，JVM也为其往虚拟机栈中压入一个栈帧，其中变量
      <code>
       src
      </code>
      的值为
      <code>
       main()
      </code>
      方法栈帧中变量
      <code>
       a
      </code>
      的副本，故其指向了同一块内存区域
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f2eb9108b21a38ba03c96d2b771c5543.png#pic_center"/>
     </li>
     <li>
      对于
      <code>
       示例代码2
      </code>
      ，执行到
      <code>
       changeTest()
      </code>
      方法时，在方法内部的代码
      <code>
       src = new ArrayList&lt;&gt;();
      </code>
      重复了
      <mark>
       步骤 1
      </mark>
      的过程，首先在堆中开辟内存，存储新建的对象，再将新对象内存地址赋值给变量
      <code>
       src
      </code>
      。这样在对
      <code>
       src
      </code>
      的操作其实是
      <mark>
       在操作新对象的内存地址，也就是改变新对象的内容

     </li>
    </ol>
    <h2>
     <a id="_98">
     </a>
     结语
    </h2>
    <p>
     通过以上分析，我们知道
     <mark>
      在Java中所有的参数传递，不管基本类型还是引用类型，都是值传递
     </mark>
     ，只是在这个过程也分为两种情况：
    </p>
    <blockquote>
     <p>
      1.如果是对基本数据类型的数据进行操作，
      <code>
       由于原始内容和副本都是存储实际值，并且是在不同的栈帧内
      </code>
      ，因此形参的操作，不影响原始内容
      <br/>
      2.如果是对引用类型的数据进行操作，分两种情况，
      <code>
       一种是形参和实参保持指向同一个对象地址
      </code>
      ，则形参的操作，会影响实参指向的对象的内容。
      <code>
       一种是形参被改动指向新的对象地址（如重新赋值引用）
      </code>
      ，则形参的操作，不会影响实参指向的对象的内容
     </p>
    </blockquote>
