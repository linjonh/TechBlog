      <span style="color:rgb(153,153,153); font-family:Arial; line-height:26px; text-indent:30px">
       <span style="font-size:14px">
        Axure RP是美国Axure Software Solution公司旗舰产品，是一个专业的快速原型设计工具，让负责定义需求和规格、设计功能和界面的专家能够快速创建应用软件或Web网站的线框图、流程图、原型和规格说明文档。作为专门的原型设计工具，它比一般创建静态原型的工具如Visio、Omnigraffle、Illustrator、Photoshop、Dreamweaver、Visual Studio、FireWorks要快速、高效。它同时支持多人协作设计和版本控制管理。

      </span>
     </div>
     <div style="orphans:2; widows:2; margin:20px 0px 0px; line-height:26px; font-family:Arial">

      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        在正式谈Axure RP之前，先来看看做产品
        <a href="http://baike.baidu.com/view/3421139.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         原型设计
        </a>
        的现有工具及其利弊。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         纸笔：
        </strong>
        简单易得，上手难度为零。有利于瞬间创意的产生与记录，有利于对文档即时的讨论与修改。但是准确度不高，难以表述页面流程，更难以表述交互信息与程序需求细节。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         Word：
        </strong>
        上手难度普通。可以画线框图，能够画页面流程，能够使用批注与文字说明。但是对交互表达不好，也不利于演示。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         PPT：
        </strong>
        上手难度普通。易于画框架图，易于做批注，也可以表达交互流程，也擅长演示。但是不利于大篇幅的文档表达。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         Visio：
        </strong>
        功能相对比较复杂。善于画流程图，框架图。不利于批注与大篇幅的文字说明。同样不利于交互的表达与演示。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         Photoshop/Fireworks：
        </strong>
        操作难度相对较大，易于画框架图、流程图。不利于表达
        <a href="http://baike.baidu.com/view/426920.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         交互设计
        </a>
        ，不擅长文字说明与批注。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        <strong>
         Dreamweaver：
        </strong>
        操作难度大，需要基础的html知识。易于画框架图、
        <a href="http://baike.baidu.com/view/7289.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         流程图
        </a>
        、表达
        <a href="http://baike.baidu.com/view/426920.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         交互设计
        </a>
        。不擅长文字说明与批注。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        以上工具都是
        <a href="http://baike.baidu.com/view/680860.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         产品经理
        </a>
        经常会使用的，但是这些工具都不是做原型设计的专门工具，需要根据产品开发的目的和开发阶段选择适当的工具搭配使用，才能达到表达、沟通的目的。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        比如纸笔更适合在产品创意阶段使用，可以快速记录思路和灵感；也可以在即时讨论沟通时使用，通过图形快速表达产品思路。Word适合用文字详细表达产品，对产品进行细节说明，擅长于图片结合文字的排版。PPT适于演示。Visio适用于各种
        <a href="http://baike.baidu.com/view/7289.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         流程图
        </a>
        、关系图的表达，更可通过画用例图获取用户需求。PS/FW是
        <a href="http://baike.baidu.com/view/539807.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         图片处理
        </a>
        的工具，DW则是所见即所得的网页开发软件，这些是设计师的看家本领，对于普通产品经理来说，需要耗费太多的精力去掌握。
       </span>
      </div>
      <div style="margin-bottom:14px; text-indent:30px">
       <span style="font-size:14px">
        以上软件最初设计目标并非为了帮助产品经理、UE完成产品
        <a href="http://baike.baidu.com/view/3421139.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); text-decoration:none" target="_blank">
         原型设计
        </a>
        ，因此都不能在原型设计这件工作上得心应手。Axure RP是在互联网产品流行的环境下，为满足原型设计创建的需求而设计的。
       </span>
      </div>
