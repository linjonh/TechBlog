     小结
    </h5>
    <p>
     <code>
      vue-router
     </code>
     支持三种路由模式，
     <code>
      hash
     </code>
     、
     <code>
      history
     </code>
     和
     <code>
      abstract
     </code>
     。默认为
     <code>
      hash
     </code>
     ，如果当前浏览器不支持
     <code>
      history
     </code>
     则会做降级处理，然后完成
     <code>
      history
     </code>
     的初始化。
    </p>
    <h4>
     <a id="_979">
     </a>
     路由切换
    </h4>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/4ce8dc468b9c28272481d2d96b3d66f3.png">
      切换 url 主要是调用了
      <code>
       push
      </code>
      方法，下面以哈希模式为例，分析
      <code>
       push
      </code>
      方法实现的原理 。
      <code>
       push
      </code>
      方法切换路由的实现原理
      <strong>
       源码地址
      </strong>
      (https://github.com/vuejs/vue-router/blob/dev/src/history/hash.js)
     </img>
    </p>
    <p>
     首先在
     <code>
      src/index.js
     </code>
     下找到
     <code>
      vueRouter
     </code>
     定义的
     <code>
      push
     </code>
     方法
    </p>
    <p>
     push (location: RawLocation, onComplete?: Function, onAbort?: Function) {
     <!-- -->
    </p>
    <p>
     // $flow-disable-line
    </p>
    <p>
     if (!onComplete &amp;&amp; !onAbort &amp;&amp; typeof Promise !== ‘undefined’) {
     <!-- -->
    </p>
    <p>
     return new Promise((resolve, reject) =&gt; {
     <!-- -->
    </p>
    <p>
     this.history.push(location, resolve, reject)
    </p>
    <p>
     })
    </p>
    <p>
     } else {
     <!-- -->
    </p>
    <p>
     this.history.push(location, onComplete, onAbort)
    </p>
    <p>
     }
    </p>
    <p>
     }
    </p>
    <p>
     接着我们需要定位到
     <code>
      history/hash.js
     </code>
     。这里首先获取到当前路径然后调用了
     <code>
      transitionTo
     </code>
     做路径切换，在回调函数当中执行
     <code>
      pushHash
     </code>
     这个核心方法。
    </p>
    <p>
     push (location: RawLocation, onComplete?: Function, onAbort?: Function) {
     <!-- -->
    </p>
    <p>
     const { current: fromRoute } = this
    </p>
    <p>
     // 路径切换的回调函数中调用 pushHash
    </p>
    <p>
     this.transitionTo(
    </p>
    <p>
     location,
    </p>
    <p>
     route =&gt; {
     <!-- -->
    </p>
    <p>
     pushHash(route.fullPath)
    </p>
    <p>
     handleScroll(this.router, route, fromRoute, false)
    </p>
    <p>
     onComplete &amp;&amp; onComplete(route)
    </p>
    <p>
     },
    </p>
    <p>
     onAbort
    </p>
    <p>
     )
    </p>
    <p>
     }
    </p>
    <p>
     而
     <code>
      pushHash
     </code>
     方法在做完浏览器兼容判断后调用的
     <code>
      pushState
     </code>
     方法，将
     <code>
      url
     </code>
     传入
    </p>
    <p>
     export function pushState (url?: string, replace?: boolean) {
     <!-- -->
    </p>
    <p>
     const history = window.history
    </p>
    <p>
     try {
     <!-- -->
    </p>
    <p>
     // 调用浏览器原生的 history 的 pushState 接口或者 replaceState 接口,pushState 方法会将 url 入栈
    </p>
    <p>
     if (replace) {
     <!-- -->
    </p>
    <p>
     history.replaceState({ key: _key }, ‘’, url)
    </p>
    <p>
     } else {
     <!-- -->
    </p>
    <p>
     _key = genKey()
    </p>
    <p>
     history.pushState({ key: _key }, ‘’, url)
    </p>
    <p>
     }
    </p>
    <p>
     } catch (e) {
     <!-- -->
    </p>
    <p>
     window.location
     <a href="url" rel="nofollow">
      replace ? ‘replace’ : ‘assign’
     </a>
    </p>
    <p>
     }
    </p>
    <p>
     }
    </p>
    <p>
     可以发现，
     <code>
      push
     </code>
     底层调用了浏览器原生的
     <code>
      history
     </code>
     的
     <code>
      pushState
     </code>
     和
     <code>
      replaceState
     </code>
     方法，不是
     <code>
      replace
     </code>
     模式 会将 url 推历史栈当中。
    </p>
    <p>
     <strong>
      另外提一嘴拼接哈希的原理
     </strong>
    </p>
    <p>
     <strong>
      源码位置
     </strong>
     (https://github.com/vuejs/vue-router/blob/dev/src/history/hash.js)
    </p>
    <p>
     初始化
     <code>
      HashHistory
     </code>
     时，构造函数会执行
     <code>
      ensureSlash
     </code>
     这个方法
    </p>
    <p>
     export class HashHistory extends History {
     <!-- -->
    </p>
    <p>
     constructor (router: Router, base: ?string, fallback: boolean) {
     <!-- -->
    </p>
    <p>
     …
    </p>
    <p>
     ensureSlash()
    </p>
    <p>
     }
    </p>
    <p>
     …
    </p>
    <p>
     }
    </p>
    <p>
     这个方法首先调用
     <code>
      getHash
     </code>
     ,然后执行
     <code>
      replaceHash()
     </code>
    </p>
    <p>
     function ensureSlash (): boolean {
     <!-- -->
    </p>
    <p>
     const path = getHash()
    </p>
    <p>
     if (path.charAt(0) === ‘/’) {
     <!-- -->
    </p>
    <p>
     return true
    </p>
    <p>
     }
    </p>
    <p>
     replaceHash(‘/’ + path)
    </p>
    <p>
     return false
    </p>
    <p>
     }
    </p>
    <p>
     下面是这几个方法
    </p>
    <p>
     export function getHash (): string {
     <!-- -->
    </p>
    <p>
     const href = window.location.href
    </p>
    <p>
     const index = href.indexOf(‘#’)
    </p>
    <p>
     return index === -1 ? ‘’ : href.slice(index + 1)
    </p>
    <p>
     }
    </p>
    <p>
     // 真正拼接哈希的方法
    </p>
    <p>
     function getUrl (path) {
     <!-- -->
    </p>
    <p>
     const href = window.location.href
    </p>
    <p>
     const i = href.indexOf(‘#’)
    </p>
    <p>
     const base = i &gt;= 0 ? href.slice(0, i) : href
    </p>
    <p>
     return
     <code>
      ${base}#${path}
     </code>
    </p>
    <p>
     }
    </p>
    <p>
     function replaceHash (path) {
     <!-- -->
    </p>
    <p>
     if (supportsPushState) {
     <!-- -->
    </p>
    <p>
     replaceState(getUrl(path))
    </p>
    <p>
     } else {
     <!-- -->
    </p>
    <p>
     window.location.replace(getUrl(path))
    </p>
    <p>
     }
    </p>
    <p>
     }
    </p>
    <p>
     export function replaceState (url?: string) {
     <!-- -->
    </p>
    <p>
     pushState(url, true)
    </p>
    <p>
     }
    </p>
    <p>
     举个例子来说: 假设当前URL是
     <code>
      http://localhost:8080
     </code>
     ,
     <code>
      path
     </code>
     为空，执行
     <code>
      replcaeHash('/' + path)
     </code>
     ,然后内部执行
     <code>
      getUrl
     </code>
     计算出
     <code>
      url
     </code>
     为
     <code>
      http://localhost:8080/#/
     </code>
     ,最后执行
     <code>
      pushState(url,true)
     </code>
     ，就大功告成了！
    </p>
    <h5>
     <a id="_1223">
     </a>
     小结
    </h5>
    <p>
     <code>
      hash
     </code>
     模式的
     <code>
      push
     </code>
     方法会调用路径切换方法
     <code>
      transitionTo
     </code>
     ,接着在回调函数中调用
     <code>
      pushHash
     </code>
     方法，这个方法调用的
     <code>
      pushState
     </code>
     方法底层是调用了浏览器原生
     <code>
      history
     </code>
     的方法。
     <code>
      push
     </code>
     和
     <code>
      replace
     </code>
     的区别就在于一个将
     <code>
      url
     </code>
     推入了历史栈，一个没有，最直观的体现就是
     <code>
      replace
     </code>
     模式下浏览器点击后退不会回到上一个路由去 ,另一个则可以。
    </p>
