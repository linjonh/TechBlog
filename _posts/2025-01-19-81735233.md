    <p style="margin-left:0cm;">
     <a name="t1">
     </a>
     <strong>
      <span style="color:#4f4f4f;">
       1.
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       物联网设备如何接入到网络？
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      只有设备接入到网络里面，才能算是物联网设备。这里涉及到
     </span>
     <span style="color:#4f4f4f;">
      2
     </span>
     <span style="color:#4f4f4f;">
      个关键点：
      <strong>
       接入方式以及网络通信方式
      </strong>
      。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       设备接入方式：
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      当前有
     </span>
     <span style="color:#4f4f4f;">
      2
     </span>
     <span style="color:#4f4f4f;">
      种接入方式
     </span>
    </p>
    <ol>
     <li>
      <strong>
       <span style="color:#333333;">
        直接接入
       </span>
      </strong>
      <span style="color:#333333;">
       ：物联网终端设备本身具备联网能力直接接入网络，比如
      </span>
      <span style="color:#333333;">
       在设备端加入
      </span>
      <span style="color:#333333;">
       NB-IOT
      </span>
      <span style="color:#333333;">
       通信模组，
      </span>
      <span style="color:#333333;">
       2G
      </span>
      <span style="color:#333333;">
       通信模组。
      </span>
     </li>
     <li>
      <strong>
       <span style="color:#333333;">
        网关接入
       </span>
      </strong>
      <span style="color:#333333;">
       ：物联网终端设备本身不具备入网能力，需要在本地组网后，需要统一通过网关再接入到网络。
      </span>
      <span style="color:#333333;">
       比如终端设备通过
      </span>
      <span style="color:#333333;">
       zigbee
      </span>
      <span style="color:#333333;">
       无线组网，然后各设备数据通过
      </span>
      <span style="color:#333333;">
       Zigbee
      </span>
      <span style="color:#333333;">
       网关统一接入到网络里面。常用到本地无线组网技术有
      </span>
      <span style="color:#333333;">
       Zigbee
      </span>
      <span style="color:#333333;">
       ，
      </span>
      <span style="color:#333333;">
       Lora,BLE MESH, sub-1GHZ
      </span>
      <span style="color:#333333;">
       等。
      </span>
     </li>
    </ol>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      在物联网设备里面，物联网网关是一个非常重要的角色。一个处在本地局域网与外部接入网络之间的智能设备。主要的功能是网络隔离，协议转化
     </span>
     <span style="color:#4f4f4f;">
      /
     </span>
     <span style="color:#4f4f4f;">
      适配以及数据网内外传输。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      一个典型的物联网网关架构如下：
     </span>
     <span style="color:#4f4f4f;">
      <br/>
      <img alt="https://www.adminiot.com.cn/static/index/img/meta/news/gw.png" class="has" id="图片_x0020_5" src="https://www.adminiot.com.cn/static/index/img/meta/news/gw.png"/>
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       网络通信方式
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      常用的
      <strong>
       通信网络
      </strong>
      主要存在
     </span>
     <span style="color:#4f4f4f;">
      2
     </span>
     <span style="color:#4f4f4f;">
      种方式：
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      1.
     </span>
     <span style="color:#999999;">
      移动网络
     </span>
     <span style="color:#999999;">
      (
     </span>
     <span style="color:#999999;">
      主要户外设备用
     </span>
     <span style="color:#999999;">
      )
     </span>
     <br/>
     <span style="color:#999999;">
      移动网络
     </span>
     <span style="color:#999999;">
      2G/3G/4G/5G/NB-IOT
     </span>
     <span style="color:#999999;">
      等
     </span>
     <span style="color:#999999;">
      <br/>
      2.
     </span>
     <span style="color:#999999;">
      宽带
     </span>
     <span style="color:#999999;">
      (
     </span>
     <span style="color:#999999;">
      主要户内设备用
     </span>
     <span style="color:#999999;">
      )
      <br/>
      WIFI
     </span>
     <span style="color:#999999;">
      ，
     </span>
     <span style="color:#999999;">
      Ethernet
     </span>
     <span style="color:#999999;">
      等
     </span>
    </p>
    <p style="margin-left:0cm;">
     <a name="t2">
     </a>
     <strong>
      <span style="color:#4f4f4f;">
       物联网设备接入网络后如何开展
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       M2M,M2C
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       通信？
      </span>
     </strong>
    </p>
    <hr/>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      物联网设备终端接入网络后，只是物联网应用的开始。
     </span>
     <span style="color:#4f4f4f;">
      设备接入网络后，设备与设备之间需要互相通信，
     </span>
     <span style="color:#4f4f4f;">
      设备与云端需要互相通信。
     </span>
     <span style="color:#4f4f4f;">
      只有互通，物联网的价值才展现出来。
     </span>
     <span style="color:#4f4f4f;">
      既然要互通，则需要一套物联网通信协议。只有遵循该套协议的设备相互间能够通信，能够交换数据。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      常用的物联网通信协议有哪些？
     </span>
     <span style="color:#4f4f4f;">
      主要有如下协议：
     </span>
     <span style="color:#4f4f4f;">
      MQTT
     </span>
     <span style="color:#4f4f4f;">
      ，
     </span>
     <span style="color:#4f4f4f;">
      COAP
     </span>
     <span style="color:#4f4f4f;">
      等，他们有个共同点都是基于消息模型来实现的。设备与设备之间，设备与云端之间通过交换消息来实现通信，消息里面携带了通信数据。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <a name="t3">
     </a>
     <strong>
      <span style="color:#4f4f4f;">
       物联网设备之间，设备与云平台能够交换数据后，接下来要干怎么呢？
      </span>
     </strong>
    </p>
    <hr/>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      21
     </span>
     <span style="color:#4f4f4f;">
      世纪，什么最重要？数据。
     </span>
     <span style="color:#4f4f4f;">
      有了大量的物联网设备数据后，
     </span>
     <span style="color:#4f4f4f;">
      针对数据，人类可以去挖掘里面的规律，挖掘里面的商业价值，对设备未来的状态进行预测等等。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      对于物联网数据应用，分几个层次来讲
     </span>
     <span style="color:#4f4f4f;">
      <br/>
      1.
     </span>
     <span style="color:#4f4f4f;">
      基础应用：监控
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      通过物联网收集到设备数据后，
     </span>
     <span style="color:#999999;">
      如果设备数据状态超过预设的状态则自动第一时间报警，管理员第一时间开展处理，可以通过远程操作，下达命令。把问题解决在萌芽状态。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      2.
     </span>
     <span style="color:#4f4f4f;">
      进阶应用：
     </span>
     <span style="color:#4f4f4f;">
      报表统计
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      通过统计方法，
     </span>
     <span style="color:#999999;">
      对设备的历史运行数据进行统计分析。可以按不同维度分析出不同报告。然后以图表或者大屏方式展现在管理员面前。
     </span>
     <span style="color:#999999;">
      管理员可以快速直观的了解到整个物联网设备运行状况。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      3.
     </span>
     <span style="color:#4f4f4f;">
      高级应用：
     </span>
     <span style="color:#4f4f4f;">
      数据挖掘
     </span>
     <span style="color:#4f4f4f;">
      /
     </span>
     <span style="color:#4f4f4f;">
      机器学习
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      这部分需要从数据里面挖掘出有价值的东西出来。比如通过一段时间时间设备数据的连续跟踪分析并结合人类过往的设备运维经验，通过机器学习方式预测设备发生故障概率，以及发生故障后可能的引起原因，并给出维修方案。刚才举的例子，是物联网高级应用里面的冰山一角。
     </span>
     <span style="color:#999999;">
      通过引入现在火热的
     </span>
     <span style="color:#999999;">
      AI
     </span>
     <span style="color:#999999;">
      技术。
     </span>
     <span style="color:#999999;">
      物联网就能变成智能物联网了。
     </span>
     <span style="color:#999999;">
      也许在不久的将来，
     </span>
     <span style="color:#999999;">
      人与设备可以自由对话，设备与设备之间也能够对话并自动做出最优决策。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      综合所述，物联网的价值在于提高生产效率，管理效率，极大促进社会生产力的提高。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <a name="t4">
     </a>
     <strong>
      <span style="color:#4f4f4f;">
       如何搭建起一个物联网系统框架呢？它的技术架构又是怎么样呢？
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <a name="t5">
     </a>
     <strong>
      <span style="color:#4f4f4f;">
       物联网终端设备软件系统架构
      </span>
     </strong>
    </p>
    <hr/>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      常见系统框架的总结下来主要存在如下
     </span>
     <span style="color:#4f4f4f;">
      2
     </span>
     <span style="color:#4f4f4f;">
      种：
     </span>
     <span style="color:#4f4f4f;">
      带
     </span>
     <span style="color:#4f4f4f;">
      RTOS
     </span>
     <span style="color:#4f4f4f;">
      的（处理复杂的业务场景，场景里面通过需要多个事务并行协同完成工作）和不带
     </span>
     <span style="color:#4f4f4f;">
      RTOS
     </span>
     <span style="color:#4f4f4f;">
      的（通常处理的业务场景较单一）
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       不带
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       RTOS
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       设备终端系统框架：
      </span>
     </strong>
     <span style="color:#4f4f4f;">
      <br/>
      <img alt="https://www.adminiot.com.cn/static/index/img/meta/news/nortos.png" class="has" id="图片_x0020_4" src="https://www.adminiot.com.cn/static/index/img/meta/news/nortos.png"/>
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       带
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       RTOS
      </span>
     </strong>
     <strong>
      <span style="color:#4f4f4f;">
       设备终端系统框架：
      </span>
     </strong>
     <span style="color:#4f4f4f;">
      <br/>
      <img alt="https://www.adminiot.com.cn/static/index/img/meta/news/rtos.png" class="has" id="图片_x0020_3" src="https://www.adminiot.com.cn/static/index/img/meta/news/rtos.png"/>
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      RTOS
     </span>
     <span style="color:#4f4f4f;">
      是什么？
     </span>
     <span style="color:#4f4f4f;">
      实时多任务操作系统，有了它，在终端设备里面可以并行运行多个任务。每个任务负责一个事务。通过并行化运行，响应实时性及效率就得到提升。
     </span>
     <span style="color:#4f4f4f;">
      RTOS
     </span>
     <span style="color:#4f4f4f;">
      实时操作内核一般包含的重要组件如下：
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      1.
     </span>
     <span style="color:#000000;">
      任务调度
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      2.
     </span>
     <span style="color:#000000;">
      任务间同步与通信
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      3.
     </span>
     <span style="color:#000000;">
      内存分配
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      4.
     </span>
     <span style="color:#000000;">
      中断管理
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      6.
     </span>
     <span style="color:#000000;">
      时间管理
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#000000;">
      7.
     </span>
     <span style="color:#000000;">
      设备驱动
     </span>
    </p>
    <p style="margin-left:0cm;">
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      以任务调度组件举例：
     </span>
     <br/>
     <span style="color:#4f4f4f;">
      在嵌入式操作系统中，任务是
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      上最小运行单元。通常一个稍微复杂点
     </span>
     <span style="color:#4f4f4f;">
      IOT APP
     </span>
     <span style="color:#4f4f4f;">
      是由多个任务协同完成。比如有的任务负责处理用户事件输入以及
     </span>
     <span style="color:#4f4f4f;">
      UI
     </span>
     <span style="color:#4f4f4f;">
      显示，有的任务负责处理数据通信，有的任务负责业务逻辑处理。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      既然一个系统中有多个任务在跑，而
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      资源确是单一的，这样导致每个时刻只能由一个任务在
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      上跑。
     </span>
     <span style="color:#4f4f4f;">
      因此为了每个任务都能够在
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      上有运行机会，就涉及到了任务调度概念。
     </span>
     <span style="color:#4f4f4f;">
      任务调度需要按照一定的规则来，那一般是按照哪些规则来的？
     </span>
     <span style="color:#4f4f4f;">
      我们常见的调度方式有
     </span>
     <span style="color:#4f4f4f;">
      3
     </span>
     <span style="color:#4f4f4f;">
      种：
     </span>
     <span style="color:#4f4f4f;">
      一个基于优先级调度的，
     </span>
     <span style="color:#4f4f4f;">
      一个是基于时间片调度的，
     </span>
     <span style="color:#4f4f4f;">
      一个是把优先级和时间片结合在一起调度。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      以优先级调度举例，在定义任务的时候，给每个任务分配一个优先级，在运行的时候，高优先级的任务都会优先被运行。
     </span>
     <span style="color:#4f4f4f;">
      直到没有高优先级任务后，低优先级任务才会被运行。
     </span>
     <span style="color:#4f4f4f;">
      假如低优先级任务获得
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      资源后，这是如果有高优先级任务就绪怎么办呢？
     </span>
     <span style="color:#4f4f4f;">
      两种处理方式：
     </span>
     <span style="color:#4f4f4f;">
      1.
     </span>
     <span style="color:#4f4f4f;">
      继续运行
     </span>
     <span style="color:#4f4f4f;">
      2.
     </span>
     <span style="color:#4f4f4f;">
      抢占式。
     </span>
     <span style="color:#4f4f4f;">
      高优先级抢占
     </span>
     <span style="color:#4f4f4f;">
      CPU
     </span>
     <span style="color:#4f4f4f;">
      资源进入运行状态。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      其他组件就不一一列举了。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <a name="t6">
     </a>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       物联网云平台系统架构
      </span>
     </strong>
    </p>
    <hr/>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      <img alt="https://www.adminiot.com.cn/static/index/img/meta/news/iotcloud.png" class="has" id="图片_x0020_2" src="https://www.adminiot.com.cn/static/index/img/meta/news/iotcloud.png"/>
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#4f4f4f;">
      物联网云平台系统架构主要包含四大组件：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#333333;">
       设备接入
      </span>
     </li>
     <li>
      <span style="color:#333333;">
       设备管理
      </span>
     </li>
     <li>
      <span style="color:#333333;">
       规则引擎
      </span>
     </li>
     <li>
      <span style="color:#333333;">
       安全认证及权限管理
      </span>
     </li>
    </ul>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       设备接入
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      设备接入主要都干些什么呢？
     </span>
     <span style="color:#999999;">
      <br/>
      1.
     </span>
     <span style="color:#999999;">
      包含多种设备接入协议，最主流的是
     </span>
     <span style="color:#999999;">
      MQTT
     </span>
     <span style="color:#999999;">
      协议。有些云计算厂商也在
     </span>
     <span style="color:#999999;">
      MQTT
     </span>
     <span style="color:#999999;">
      协议上精简协议变成独有的接入协议。
     </span>
     <span style="color:#999999;">
      <br/>
      2.
     </span>
     <span style="color:#999999;">
      并发连接管理，维持可能是数十亿设备的长连接管理。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      如果应对数十亿的设备连接管理？
     </span>
     <span style="color:#999999;">
      目前开放出来的
     </span>
     <span style="color:#999999;">
      MQTT
     </span>
     <span style="color:#999999;">
      代理服务器大都是单机版，最多也就是并发连接十几万设备。因此如果要管理数十亿的连接，需要用到负载均衡，用到分布式架构。在云平台需要部署分布式
     </span>
     <span style="color:#999999;">
      MQTT
     </span>
     <span style="color:#999999;">
      代理服务器。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       设备管理
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      一般以树形结构的方式管理设备，包含设备创建管理以及设备状态管理等等。根节点以产品开始，
     </span>
     <span style="color:#999999;">
      然后是设备组，再到具体设备。
     </span>
     <br/>
     <span style="color:#999999;">
      主要包含如下管理：
     </span>
     <br/>
     <span style="color:#999999;">
      产品注册及管理
     </span>
     <br/>
     <span style="color:#999999;">
      产品下面的设备增删改查管理
     </span>
     <br/>
     <span style="color:#999999;">
      设备消息发布
     </span>
     <span style="color:#999999;">
      <br/>
      OTA
     </span>
     <span style="color:#999999;">
      设备升级管理等
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       规则引擎
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      物联网云平台通常是基于现有云计算平台搭建的。
     </span>
     <span style="color:#999999;">
      一个物联网成熟业务除了用到物联网云平台提供功能外，一般还需要用到云计算平台提供功能，比如云主机，云数据库等。用户可以在云主机上搭建
     </span>
     <span style="color:#999999;">
      web
     </span>
     <span style="color:#999999;">
      行业应用服务。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      规则引擎主要作用是把物联网平台数据通过过滤转发到其他云计算产品上。
     </span>
     <br/>
     <span style="color:#999999;">
      比如可以把设备上报的数据转发到
     </span>
     <span style="color:#999999;">
      table store
     </span>
     <span style="color:#999999;">
      数据库产品里。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      规则引擎一般使用方式：类
     </span>
     <span style="color:#999999;">
      SQL
     </span>
     <span style="color:#999999;">
      语言，通过编写
     </span>
     <span style="color:#999999;">
      SQL
     </span>
     <span style="color:#999999;">
      语言，用户可以过滤数据、处理数据，并把数据发到其他云计算产品，或者其他云计算服务。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       安全认证及权限管理
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      物联网云平台为每个设备颁发唯一的证书，需要证书通过后才能允许设备接入到云平台。云平台最小授权粒度一般是做到设备级。我们所说的证书一般分为
     </span>
     <span style="color:#999999;">
      2
     </span>
     <span style="color:#999999;">
      种：一种是产品级证书，一种是设备级证书。
     </span>
     <span style="color:#999999;">
      产品级证书拥有最大的权限，可以对产品下所有的设备进行操作。
     </span>
     <span style="color:#999999;">
      设备级证书，只能对自己所属的设备进行操作，无法对其他设备进行操作。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      因此每个接入云平台的设备都在本地存储一个证书（其实存在形式是一个
     </span>
     <span style="color:#999999;">
      KEY
     </span>
     <span style="color:#999999;">
      ，有多个字符串构成）。每次与云端建立连接时，都要把证书带上。以便云端安全组件核查通过。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       云计算产品
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      大数据计算产品，
     </span>
     <span style="color:#999999;">
      云基础服务（高性能服务器，云数据库，云网络）等
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       物联网应用
      </span>
     </strong>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#999999;">
      智慧城市，
     </span>
     <span style="color:#999999;">
      智慧农业，工业
     </span>
     <span style="color:#999999;">
      4.0
     </span>
     <span style="color:#999999;">
      等
     </span>
    </p>
    <p style="margin-left:0cm;">
     <strong>
      <span style="color:#4f4f4f;">
       下面一张是来自百度物联网平台架构：
      </span>
     </strong>
     <br/>
     <span style="color:#4f4f4f;">
      突出展示了物联网设备，
     </span>
     <span style="color:#4f4f4f;">
      物联网云平台，
     </span>
     <span style="color:#4f4f4f;">
      云计算平台是如何协同工作的，有较大的参考价值。
