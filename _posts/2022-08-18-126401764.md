   <div class="htmledit_views" id="content_views">
    <p>
     简单配置就可以使用的嵌入式开发IDE如下： （想具体了解就按照下面的名字搜一下即可）
    </p>
    <ul>
     <li>
      keil（MDK）
      <br/>
      老牌了，ST收购了叫做mdk，仅支持stm32项目，界面比较传统，功能一般吧，
      <strong>
       好像ST逐渐放弃这个了
      </strong>
      。收费
      <br/>
     </li>
     <li>
      iar
      <br/>
      也很经典，界面也不好看，功能跟keil差不多。收费
      <br/>
     </li>
     <li>
      STM32CubeIDE
      <br/>
      基于eclipse，ST的，
      <strong>
       感觉ST主推这个了
      </strong>
      ，配合自家的STM32CubeMX这个代码自动生成工具，很好用。但是代码提示，这些做得不是很好，没有IDE那种感觉。免费
      <br/>
      <a href="https://blog.csdn.net/ybhuangfugui/article/details/122852411" title="STM32集成开发工具 | STM32CubeIDE介绍、下载、安装和使用教程_strongerHuang的博客-CSDN博客">
       STM32集成开发工具 | STM32CubeIDE介绍、下载、安装和使用教程_strongerHuang的博客-CSDN博客
      </a>
     </li>
     <li>
      TrueStudio
      <br/>
      基于eclipse，ST的，但是代码提示，这些做得不是很好，没有IDE那种感觉。免费
      <br/>
      <a href="https://blog.csdn.net/congpo2125/article/details/86549614" title="TrueStudio上手_Catfish King的博客-CSDN博客_truestudio">
       TrueStudio上手_Catfish King的博客-CSDN博客_truestudio
      </a>
      <br/>
     </li>
     <li>
      CLion
      <br/>
      Jetbrains系的IDE，界面非常美观，现代化，代码提示功能强大。收费
      <br/>
      <a href="https://zhuanlan.zhihu.com/p/145801160" rel="nofollow" title="配置CLion用于STM32开发【优雅の嵌入式开发】 - 知乎">
       配置CLion用于STM32开发【优雅の嵌入式开发】 - 知乎
      </a>
      <br/>
     </li>
     <li>
      RT-Thread Studio
      <br/>
      基于eclipse，咱们国产rtthread这个嵌入式OS的配套IDE，界面美观，完全咱们中文支持，不仅可以做OS开发，还能做裸机开发，还支持qemu仿真，各方面来看真心不错，比国外的做得还好。免费
      <br/>
      <a href="https://www.rt-thread.org/page/studio.html" rel="nofollow" title="RT-Thread Studio - RT-Thread物联网操作系统">
       RT-Thread Studio - RT-Thread物联网操作系统
      </a>
      <br/>
     </li>
     <li>
      vscode
      <br/>
      通用型IDE，需要配合platformio插件，美观。免费
      <br/>
     </li>
     <li>
      Embedded Studio
      <br/>
      好像也是ST的，好像也不够现代，感觉用的人少。
      <br/>
      <a href="https://www.armbbs.cn/forum.php?mod=viewthread&amp;tid=89140" rel="nofollow" title="Embedded Studio介绍以及与MDK和IAR性能比较 - 开发环境 -  硬汉嵌入式论坛 -  Powered by Discuz!">
       Embedded Studio介绍以及与MDK和IAR性能比较 - 开发环境 - 硬汉嵌入式论坛 - Powered by Discuz!
      </a>
      <br/>
     </li>
     <li>
      SW4STM32
      <br/>
      也是ST的，基于eclipse的，感觉用的人也少，估计也不太好用。免费
      <br/>
      <a href="https://www.cnblogs.com/djjh/p/sw4stm32.html" rel="nofollow" title="sw4stm32开发工具不错哟 - 哟戈 - 博客园">
       sw4stm32开发工具不错哟 - 哟戈 - 博客园
      </a>
      <br/>
     </li>
     <li>
      Code Composer Studio
      <br/>
      TI的（那应该就是不支持stm32），基于eclipse的，感觉用的人也少，不好用
      <br/>
      <a href="https://blog.csdn.net/qq_41892825/article/details/105880630" title="Code Composer Studio 安装教程与入门_东路的尽头的博客-CSDN博客_code composer studio">
       Code Composer Studio 安装教程与入门_东路的尽头的博客-CSDN博客_code composer studio
      </a>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      总结：
     </strong>
    </p>
    <p>
     我也是多年前从51单片机stm32单片机学过来的，嵌入式开发，入门的话，可以先用keil，后面用RT-Thread Studio吧，确实好用。
    </p>
    <p>
     但是，我觉得，基于eclipse的这些IDE虽然好用，但是有缺点，那就是
     <strong>
      太臃肿
     </strong>
     （安装完占用空间一般都1G以上），软件有些卡顿，各种插件也很
     <strong>
      吃电脑性能
     </strong>
     。因此，我正在开发一款嵌入式IDE，是用qt搞的，对标上面这些IDE的功能，同时轻量化，运行流畅。
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
     其它的当然，visual studio安装插件，原版eclipse安装插件，codebolocks安装插件，也都能实现嵌入式开发，但是比较繁琐，不常见。
    </p>
    <p>
     当然还有其它的代码阅读等好用的工具（可能不具备编译调试等功能，仅仅阅读代码，非常合适）
    </p>
    <ul>
     <li>
      source insight
     </li>
     <li>
      subline
     </li>
     <li>
      vscode（可以当作就看代码功能使用，很不错）
     </li>
     <li>
      Atom
     </li>
    </ul>
