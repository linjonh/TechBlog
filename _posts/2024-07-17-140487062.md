    <h3 id=".E6.93.8D.E4.BD.9C.E5.9C.BA.E6.99.AF">
     操作场景
    </h3>
    <p>
     Prometheus 社区开发了 JMX Exporter 用于导出 JVM 的监控指标，以便使用 Prometheus 来采集监控数据。当您的 Java 业务容器化至 Kubernetes 后，可通过本文了解如何使用 Prometheus 与 JMX Exporter 来监控 Java 应用。
    </p>
    <p>
    </p>
    <h3 id="jmx-exporter-.E7.AE.80.E4.BB.8B">
     JMX Exporter 简介
    </h3>
    <p>
     Java Management Extensions，JMX 是管理 Java 的一种扩展框架，JMX Exporter 基于此框架读取 JVM 的运行时状态。JMX Exporter 利用 Java 的 JMX 机制来读取 JVM 运行时的监控数据，然后将其转换为 Prometheus 可辨识的 metrics 格式，以便让 Prometheus 对其进行监控采集。
    </p>
    <p>
     JMX Exporter 提供
     <strong>
      启动独立进程
     </strong>
     及
     <strong>
      JVM 进程内启动（in-process）
     </strong>
     两种方式暴露 JVM 监控指标：
    </p>
    <p>
     1.
     <strong>
      启动独立进程 JVM 启动时指定参数，暴露 JMX 的 RMI 接口。JMX Exporter 调用 RMI 获取 JVM 运行时状态数据，转换为 Prometheus metrics 格式，并暴露端口让 Prometheus 采集。
     </strong>
    </p>
    <p>
     2.
     <strong>
      JVM 进程内启动（in-process）
     </strong>
     <span style="background-color:#ff9900;">
      JVM 启动时指定参数，通过 javaagent 的形式运行 JMX Exporter 的 jar 包，进程内读取 JVM 运行时状态数据
     </span>
     ，转换为 Prometheus metrics 格式，并暴露端口让 Prometheus 采集。
    </p>
    <blockquote>
     <p>
      <strong>
       说明：
      </strong>
     </p>
     <p>
      官方不建议使用
      <strong>
       启动独立进程
      </strong>
      方式，该方式配置复杂且需单独的进程，进程本身的监控又引发了新的问题。本文以
      <strong>
       <span style="background-color:#a2e043;">
        JVM 进程内启动（in-process）
       </span>
      </strong>
      方式为例，在 Kubernetes 环境下使用 JMX Exporter 暴露 JVM 监控指标。
     </p>
     <p>
     </p>
    </blockquote>
    <p>
     这里使用第二种用法，启动 JVM 时需要指定 JMX Exporter 的 jar 包文件和配置文件。
     <span style="background-color:#ffd900;">
      jar 包是二进制文件，不好通过 configmap 挂载
     </span>
     ，配置文件我们几乎不需要修改， 所以建议是
     <span style="background-color:#ff9900;">
      直接将 JMX Exporter 的 jar 包和配置文件都打包到业务容器镜像中。
     </span>
    </p>
    <p>
     其中，第二种方式我们可以选择将
     <span style="background-color:#a2e043;">
      JMX Exporter 的 jar 文件放在业务应用镜像中
     </span>
     ， 也可以选择在部署的时候挂载进去。这里分别对两种方式做一个介绍：
    </p>
    <p>
    </p>
    <p>
    </p>
    <h3>
     方式一：将 JMX Exporter JAR 文件构建至业务镜像中
    </h3>
    <p>
     prometheus-jmx-config.yaml 内容如下：
    </p>
    <pre><code>prometheus-jmx-config.yaml

...
ssl: false
lowercaseOutputName: false
lowercaseOutputLabelNames: false
rules:
- pattern: ".*"
</code></pre>
    <p>
     然后准备 jar 包文件，可以在
     <a href="https://github.com/prometheus/jmx_exporter" title="jmx_exporter">
      jmx_exporter
     </a>
     的 Github 页面找到最新的 jar 包下载地址并参考如下 Dockerfile:
    </p>
    <pre><code>FROM openjdk:11.0.15-jre
WORKDIR /app/
COPY target/my-app.jar ./
COPY prometheus-jmx-config.yaml ./
RUN set -ex; \
    curl -L -O https://repo1.maven.org/maven2/io/prometheus/jmx/jmx_prometheus_javaagent/0.17.2/jmx_prometheus_javaagent-0.17.2.jar;
ENV JAVA_TOOL_OPTIONS=-javaagent:/app/jmx_prometheus_javaagent-0.17.2.jar=8088:/app/prometheus-jmx-config.yaml
EXPOSE 8081 8999 8080 8888
ENTRYPOINT java $JAVA_OPTS -jar my-app.jar</code></pre>
    <p>
     注意：
    </p>
    <ul>
     <li>
      启动参数格式：-javaagent:=:
     </li>
     <li>
      这里使用了 8088 端口暴露 JVM 的监控指标，如果和 Java 应用冲突，可自行更改
     </li>
    </ul>
    <p>
     下面是自己编写的Dockerfile
    </p>
    <pre><code>[root@jenkins dockerfile]# cat Dockerfile
FROM openjdk:8-jdk-alpine 
LABEL maintainer lucas

ENV TZ=Asia/Shanghai
ENV JAVA_OPTS="-Xmx512m -Xms512m"
ENV JAVA_TOOL_OPTIONS="-javaagent:/jmx_prometheus_javaagent-1.0.1.jar=8888:/jmx-config.yaml"

COPY demo.jar  / 
COPY jmx_prometheus_javaagent-1.0.1.jar /
COPY jmx-config.yaml /
WORKDIR /

EXPOSE 8000 8888

ENTRYPOINT ["/bin/sh"]
CMD ["-c","java $JAVA_OPTS -jar /demo.jar"]</code></pre>
    <p>
    </p>
    <p>
    </p>
    <h3 id="init-container">
     方式二： 通过 init container 容器挂载
    </h3>
    <p>
     我们需要先将 JMX exporter 做成 Docker 镜像, 以下 Dockerfile 仅供参考：
    </p>
    <pre><code>FROM alpine/curl:3.14
WORKDIR /app/
# 将前面创建的 config 文件拷贝至镜像
COPY prometheus-jmx-config.yaml ./
# 在线下载 jmx prometheus javaagent jar
RUN set -ex; \
    curl -L -O https://repo1.maven.org/maven2/io/prometheus/jmx/jmx_prometheus_javaagent/0.17.2/jmx_prometheus_javaagent-0.17.2.jar;</code></pre>
    <p>
     根据上面 Dockerfile 构建镜像：
     <code>
      docker build -t my-jmx-exporter .
     </code>
    </p>
    <p>
     在 Java 应用部署 Yaml 中加入如下 init container：
    </p>
    <pre><code>apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-demo-app
  labels:
    app: my-demo-app
spec:
  selector:
    matchLabels:
      app: my-demo-app
  template:
    metadata:
      labels:
        app: my-demo-app
    spec:
      imagePullSecrets:
      - name: registry-pull
      initContainers:
      - name: jmx-sidecar
        image: my-jmx-exporter
        command: ["cp", "-r", "/app/jmx_prometheus_javaagent-0.17.2.jar", "/target/jmx_prometheus_javaagent-0.17.2.jar"]  ➊
        volumeMounts:
        - name: sidecar
          mountPath: /target
      containers:
      - image: my-demo-app-image
        name: my-demo-app
        resources:
          requests:
            memory: "1000Mi"
            cpu: "500m"
          limits:
            memory: "1000Mi"
            cpu: "500m"
        ports:
        - containerPort: 18083
        env:
        - name: JAVA_TOOL_OPTIONS
          value: "-javaagent:/app/jmx_prometheus_javaagent-0.17.2.jar=8088:/app/prometheus-jmx-config.yaml" ➋
        volumeMounts:
        - name: host-time
          mountPath: /etc/localtime
          readOnly: true
        - name: sidecar
          mountPath: /sidecar
      volumes:
      - name: host-time
        hostPath:
          path: /etc/localtime
      - name: sidecar  #共享 agent 文件夹
        emptyDir: {}
      restartPolicy: Always</code></pre>
    <p>
     经过如上的改造之后，示例应用 my-demo-app 具备了暴露 JVM 指标的能力。 运行服务之后，我们可以通过
     <code>
      http://lcoalhost:8088
     </code>
     访问服务暴露出来的 prometheus 格式的指标。
    </p>
    <p>
    </p>

    <h3 id=".E6.93.8D.E4.BD.9C.E6.AD.A5.E9.AA.A4">
     操作步骤
    </h3>
    <h4 id=".E4.BD.BF.E7.94.A8-jmx-exporter-.E6.9A.B4.E9.9C.B2-jvm-.E7.9B.91.E6.8E.A7.E6.8C.87.E6.A0.87">
     使用 JMX Exporter 暴露 JVM 监控指标
    </h4>
    <h5 id=".E6.89.93.E5.8C.85.E9.95.9C.E5.83.8F">
     打包镜像
    </h5>
    <p>
     使用 JVM 进程内启动（in-process）方式，
     <span style="background-color:#a2e043;">
      启动 JVM 需指定 JMX Exporter 的 jar 包文件和配置文件。jar 包为二进制文件，不便通过 configmap 挂载，建议直接将 JMX Exporter 的 jar 包和配置文件都打包到业务容器镜像中
     </span>
     。步骤如下：
    </p>
    <p>
     (1) 准备一个制作镜像的目录，放入 JMX Exporter 配置文件
     <code>
      prometheus-jmx-config.yaml
     </code>
     。
    </p>
    <pre><code>ssl: false
lowercaseOutputName: false
lowercaseOutputLabelNames: false 

注意:更多配置项请参考 Prometheus 官方文档。
</code></pre>
    <p>
     (2) 准备 jar 包文件，可前往
     <a href="https://github.com/prometheus/jmx_exporter" title="jmx_exporter">
      jmx_exporter
     </a>
     的 Github 页面获取最新的 jar 包下载地址。执行以下命令，下载到当前目录。
     <a class="has-card" href="https://github.com/prometheus/jmx_exporter" title="https://github.com/prometheus/jmx_exporter">
      <span class="link-card-box">
       <span class="link-title">
        https://github.com/prometheus/jmx_exporter
       </span>
       <span class="link-link">
        <img alt="icon-default.png?t=N7T8" class="link-link-icon" src="https://i-blog.csdnimg.cn/blog_migrate/003a2ce7eb50c2e24a8c624c260c5930.png">
         https://github.com/prometheus/jmx_exporter
        </img>
       </span>
      </span>
     </a>
    </p>
    <p>
     (3) 准备 Dockerfile 文件，本文以 Tomcat 为例。
    </p>
    <pre><code>FROM tomcat:jdk8-openjdk-slim
ADD prometheus-jmx-config.yaml /prometheus-jmx-config.yaml
ADD jmx_prometheus_javaagent-0.13.0.jar /jmx_prometheus_javaagent-0.13.0.jar</code></pre>
    <p>
     (4) 执行以下命令，编译镜像。
    </p>
    <pre><code>docker build . -t ccr.ccs.tencentyun.com/imroc/tomcat:jdk8</code></pre>
    <p>
     至此已完成镜像打包，您还可利用 docker 多阶段构建，省略手动下载 jar 包的步骤。Dockerfile 示例如下：
    </p>
    <pre><code>FROM ubuntu:16.04 as jar
WORKDIR /
RUN apt-get update -y
RUN DEBIAN_FRONTEND=noninteractive apt-get install -y wget
RUN wget https://repo1.maven.org/maven2/io/prometheus/jmx/jmx_prometheus_javaagent/0.13.0/jmx_prometheus_javaagent-0.13.0.jar
FROM tomcat:jdk8-openjdk-slim
ADD prometheus-jmx-config.yaml /prometheus-jmx-config.yaml
COPY --from=jar /jmx_prometheus_javaagent-0.13.0.jar /jmx_prometheus_javaagent-0.13.0.jar</code></pre>
    <h5 id=".E9.83.A8.E7.BD.B2-java-.E5.BA.94.E7.94.A8">
     部署 Java 应用
    </h5>
    <p>
     部署应用至 Kubernetes 时，需修改 JVM 启动参数以便启动时加载 JMX Exporter。JVM 启动时会读取
     <code>
      JAVA_OPTS
     </code>
     环境变量，作为额外的启动参数，部署时可为应用增加该环境变量。示例如下：
    </p>
    <pre><code>apiVersion: apps/v1
kind: Deployment
metadata:
  name: tomcat
spec:
  replicas: 1
  selector:
    matchLabels:
      app: tomcat
  template:
    metadata:
      labels:
        app: tomcat
    spec:
      containers:
      - name: tomcat
        image: ccr.ccs.tencentyun.com/imroc/tomcat:jdk8
        env:
        - name: JAVA_OPTS
          value: "-javaagent:/jmx_prometheus_javaagent-0.13.0.jar=8088:/prometheus-jmx-config.yaml"

---

apiVersion: v1
kind: Service
metadata:
  name: tomcat
  labels:
    app: tomcat
spec:
  type: ClusterIP
  ports:
  - port: 8080
    protocol: TCP
    name: http
  - port: 8088
    protocol: TCP
    name: jmx-metrics
  selector:
    app: tomcat</code></pre>
    <ul>
     <li>
      启动参数格式：
      <code>
       -javaagent:&lt;jar&gt;=&lt;port&gt;:&lt;config&gt;
      </code>
     </li>
     <li>
      该示例使用8088端口暴露 JVM 的监控指标，您可按需自行更改。
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       为 JAVA 服务组件添加环境变量 ：
      </strong>
     </p>
     <p>
      <code>
       JAVA_OPTS = -javaagent:/tmp/agent/jmx_prometheus_javaagent-0.16.1.jar=5556:/tmp/agent/prometheus-jmx-config.yaml
      </code>
     </p>
    </blockquote>

    <h4 id=".E6.B7.BB.E5.8A.A0-prometheus-.E7.9B.91.E6.8E.A7.E9.85.8D.E7.BD.AE">
     添加 Prometheus 监控配置
    </h4>
    <p>
     配置 Prometheus，使监控数据可被采集。示例如下：
    </p>
    <pre><code>- job_name: tomcat
  scrape_interval: 5s
  kubernetes_sd_configs:
  - role: endpoints
    namespaces:
      names:
      - default
  relabel_configs:
  - action: keep
    source_labels:
    - __meta_kubernetes_service_label_app
    regex: tomcat
  - action: keep
    source_labels:
    - __meta_kubernetes_endpoint_port_name
    regex: jmx-metrics</code></pre>
    <p>
     若已安装 prometheus-operator，则可通过创建 ServiceMonitor 的 CRD 对象配置 Prometheus。示例如下：
    </p>
    <pre><code>apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: tomcat
  namespace: default
  labels:
    app: tomcat
spec:
  endpoints:
  - port: jmx-metrics
    interval: 5s
  namespaceSelector:
    matchNames:
    - default
  selector:
    matchLabels:
      app: tomcat</code></pre>
