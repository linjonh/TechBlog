    <h4 id="1wxml">
     （1）WXML
    </h4>
    <p>
     WXML(WeiXin Markup Language),这门语言以标签的形式结合组件、WXS和事件系统，构建页面结构。
     <br/>
     <strong>
      语法：
     </strong>
    </p>
    <pre class="prettyprint"><code class="hljs xml"><span class="hljs-tag">&lt;<span class="hljs-title">标签名</span> 属性名=<span class="hljs-value">"属性值"</span>&gt;</span><span class="hljs-tag">&lt;/<span class="hljs-title">标签名</span>&gt;</span>//标签闭合 大小写敏感</code></pre>
    <p>
     <strong>
      语言特性：
     </strong>
     <br/>
     <em>
      数据绑定：
     </em>
     <br/>
     （一些网页的数据需要动态地频繁更新）
     <br/>
     WXML中的数据都来自于.js文件中配置的data对象，一种数据绑定方法用的是Mustache的语法，即变量名两边用两个大括号包起来：
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml"><span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>
    <span class="hljs-tag">&lt;<span class="hljs-title">text</span>&gt;</span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">message</span>}}</span><span class="xml"><span class="hljs-tag">&lt;/<span class="hljs-title">text</span>&gt;</span>
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span></span></code></pre>
    <p>
     既可以绑定文本内容，也可以绑定属性：
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml">//.wxml中：
<span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>
    <span class="hljs-tag">&lt;<span class="hljs-title">text</span> <span class="hljs-attribute">data-name</span>=<span class="hljs-value">"</span></span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">theName</span>}}</span><span class="xml"><span class="hljs-tag"><span class="hljs-value">"</span>&gt;</span><span class="hljs-tag">&lt;/<span class="hljs-title">text</span>&gt;</span>
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span></span></code></pre>
    <pre class="prettyprint"><code class="hljs xml">//.js中：
<span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>
    <span class="hljs-tag">&lt;<span class="hljs-title">text</span> <span class="hljs-attribute">data-name</span>=<span class="hljs-value">"Jack"</span>&gt;</span><span class="hljs-tag">&lt;/<span class="hljs-title">text</span>&gt;</span>
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span></code></pre>
    <p>
     <font color="pink" size="5em">
      <strong>
       注意：所有组件的字母都是小写
      </strong>
     </font>
    </p>
    <p>
     运算符绑定：
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml">//.wxml中
<span class="hljs-tag">&lt;<span class="hljs-title">view</span> <span class="hljs-attribute">hidden</span>=<span class="hljs-value">"</span></span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">flag</span> ? <span class="hljs-variable">true</span> : <span class="hljs-variable">false</span>}}</span><span class="xml"><span class="hljs-tag"><span class="hljs-value">"</span>&gt;</span>
    Hidden
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span>//hidden为false时，显示“Hidden”字样</span></code></pre>
    <pre class="prettyprint"><code class="hljs haskell">//.js中
<span class="hljs-type">Page</span>({
    <span class="hljs-typedef"><span class="hljs-keyword">data</span>:<span class="hljs-container">{
        <span class="hljs-title">flag</span>:<span class="hljs-title">false</span>
    }</span></span>
})</code></pre>
    <p>
     hidden属性帮助显示或者隐藏掉标签的内容
     <br/>
     类似数据绑定还有字符串运算绑定、组合式绑定和扩展运算符绑定等
    </p>
    <blockquote>
     <p>
      标签的共同属性：
      <br/>
      id：组件的唯一标识
      <br/>
      class：组件的样式类
      <br/>
      style：组件的内联样式，可以动态设置样式，而class中的只能设置静态属性
      <br/>
      hidden：boolean类型
      <br/>
      data：任意类型，自定义属性，组件上触发事件时，data中的数据会发送给事件处理函数（逻辑层）
      <br/>
      bind/catch：EventHandler类型，组件的事件，绑定组件上的默认事件
     </p>
    </blockquote>
    <p>
     <em>
      列表渲染：
     </em>
     <br/>
     用于数据很多的情况，使用列表渲染，减少代码量，使代码更有条理：
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml">//.wxml中
<span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>
    <span class="hljs-tag">&lt;<span class="hljs-title">block</span> <span class="hljs-attribute">wx:for</span>=<span class="hljs-value">"</span></span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">items</span>}}</span><span class="xml"><span class="hljs-tag"><span class="hljs-value">"</span> <span class="hljs-attribute">wx:for-item</span>=<span class="hljs-value">"item"</span> <span class="hljs-attribute">wx:key</span>=<span class="hljs-value">"index"</span>&gt;</span>
        <span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">index</span>}}</span><span class="xml">:</span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">item.name</span>}}</span><span class="xml"><span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span><span class="hljs-comment">&lt;!--wx:for-item属性可以给它指定一个变量名 wx:key属性可以确保带有该属性的item在渲染时重新排序，而不是重新渲染，从而提高渲染效率。如果在列表发生改变，需要重新渲染时，列表就可以保持自身的一个状态；这里不推荐使用index作为标识item唯一的值，最好用id等字段标识；block不是一个组件，只是一个包装元素，在渲染时，包装元素不会被渲染出来--&gt;</span>
    <span class="hljs-tag">&lt;/<span class="hljs-title">block</span>&gt;</span>
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span></span></code></pre>
    <pre class="prettyprint"><code class="hljs haskell">//.js中
<span class="hljs-type">Page</span>({
    <span class="hljs-typedef"><span class="hljs-keyword">data</span>:<span class="hljs-container">{
        <span class="hljs-title">items</span>:[
            {<!-- --><span class="hljs-title">name</span>:"商品<span class="hljs-type">A</span>"}</span>,</span>
            {name:<span class="hljs-string">"商品B"</span>},
            {name:<span class="hljs-string">"商品C"</span>},
            {name:<span class="hljs-string">"商品D"</span>}
        ]
    }
})</code></pre>
    <p>
     <em>
      条件渲染：
     </em>
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml">//.wxml中：
<span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>今天吃什么?<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span>
<span class="hljs-tag">&lt;<span class="hljs-title">view</span> <span class="hljs-attribute">wx:if</span>=<span class="hljs-value">"</span></span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">condition</span>===1}}</span><span class="xml"><span class="hljs-tag"><span class="hljs-value">"</span>&gt;</span>
  饺子
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span>
<span class="hljs-tag">&lt;<span class="hljs-title">view</span> <span class="hljs-attribute">wx:elif</span>=<span class="hljs-value">"</span></span></span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">condition</span>===2}}</span><span class="xml"><span class="hljs-tag"><span class="hljs-value">"</span>&gt;</span>
  rice
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span>
<span class="hljs-tag">&lt;<span class="hljs-title">view</span> <span class="hljs-attribute">wx:else</span>&gt;</span>
  noodles
<span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span></span></code></pre>
    <pre class="prettyprint"><code class="hljs haskell">//.js中：
<span class="hljs-type">Page</span>({
  <span class="hljs-typedef"><span class="hljs-keyword">data</span>: <span class="hljs-container">{
    <span class="hljs-title">condition</span>:<span class="hljs-type">Math</span>.<span class="hljs-title">floor</span>(<span class="hljs-type">Math</span>.<span class="hljs-title">random</span>()*3+1)//获取1-3的值
  }</span></span>
})</code></pre>
    <p>
     hidden和if显示时的异同点：
     <br/>
     wx:if条件渲染，在切换时框架有一个局部渲染渲染过程，从而保证在渲染时可以销毁并重新渲染；而hidden也会渲染/(ㄒoㄒ)/~~，不同的是if有更高的切换消耗，hidden有更高的初始渲染消耗。
    </p>
    <p>
     <em>
      模板引用：
     </em>
     <br/>
     可以设置自己的代码片段，在多处进行引用：
    </p>
    <pre class="prettyprint"><code class="hljs handlebars"><span class="xml">//.wxml中

    <span class="hljs-tag">&lt;<span class="hljs-title">view</span>&gt;</span>联系方式:</span><span class="hljs-expression">{<!-- -->{<!-- --><span class="hljs-variable">phone</span>}}</span><span class="xml"><span class="hljs-tag">&lt;/<span class="hljs-title">view</span>&gt;</span>

    <h4 id="3javascript">
     （3）JavaScript
    </h4>
    <blockquote>
     <p>
      <font size="5em">
       JavaScript
      </font>
      是一种轻量的、解释型的、面向对象的头等函数语言，是一种动态的基于原型和多范式的脚本语言，支持面向对象、命令式和函数式的编程风格。
      <br/>
      “Java和JavaScript的关系就像雷锋和雷峰塔的关系。”
      <br/>
      《JavaScript高级程序设计》《JavaScript权威指南》
     </p>
    </blockquote>
    <p>
     Nodejs中的JavaScript：
     <br/>
     由ECMAScript、Native（原生模块，通过它使用一些JavaScript不具有的能力）和NPM（是一个包管理系统，是目前世界上最大的开源库生态系统，可以快速地通过一些NPM扩展包实现一些功能）组成
    </p>
    <blockquote>
     <p>
      Nodejs是基于谷歌的V8引擎实现的JavaScript运行时，它使用了高效、轻量级的事件驱动，以及非阻塞的IO模型，通常将Nodejs作为一门后端的语言来使用。
     </p>
    </blockquote>
    <p>
     小程序中的JavaScript:
     <br/>
     由ECMAScript、小程序框架、小程序API组成，和浏览器中的JavaScript相比是没有BOM和DOM对象的，所以类似于JQuery等类库没有办法使用；同样，它也缺少Nodejs中的Native和NPM模块。
    </p>
    <blockquote>
     <p>
      ECMAScript是一门美国262标准国际化的脚本程序设计语言，关键部分：语法、类型、语句、关键字、操作符、对象，JavaScript是它的一种实现。
     </p>
    </blockquote>
    <p>
     浏览器中的JavaScript：
     <br/>
     由ECMAScript、DOM、BOM组成。DOM是Html和Xml的应用程序接口，DOM表示加载到当前窗口的一个网页，可通过JS读取DOM对象，BOM主要是处理浏览器窗口和框架，描述了与浏览器交互的方法和接口，如修改页面url、打开新窗口。
    </p>

    </h4>

     <p>

     </em>

     可以通过标签（.wxs）来声明，也可以通过模块来声明;

     每一个模块里的变量、函数的作用域都是私有的，需要用module.export将其暴露出来供外界访问：

    <h5 id="mina框架">
     MINA框架
    </h5>
    <p>
     <img alt="MI'NA框架" src="https://i-blog.csdnimg.cn/blog_migrate/a1be9df138eb0ffc90073f2e3ef06041.png" title=""/>
    </p>
    <p>
     其中App Service层是数据交互中心，Manager负责逻辑处理，另一部分是WAService.js的文件，来封装各种API的接口，让各个平台的运行环境都能通过API使用微信客户端。整个小程序只有一个AppService，并且整个生命周期常驻内存，由AppService线程来加载运行；
     <br/>
     Native层接入了微信客户端的原生能力。视图层和逻辑层双线通信，在视图层和逻辑层之间提供了数据传输和逻辑系统；
     <br/>
     视图层和逻辑层之间通过JSBridge进行通信，在逻辑层发生数据变更的时候，通过AppService的setData方法把数据从逻辑层传递到视图层，触发视图层的页面更新，（WXML的节点数结构具有的上下文结构可以使它被渲染成js对象（AST抽象语法树的概念emmm），从而将数据在视图层展示出来），视图层将事件通知给逻辑层进行业务处理。WebView容器在渲染节点内容时，会把传过来的数据进行对比差异（用diff算法emmm），把差异应用在原来的节点树上，渲染出正确的UI界面，
     <br/>
     WXS做了很多优化：它和WXML和WXSS一起运行在UI线程，平常从数据库请求到数据库之后通常会做一个转换或filter处理，如果把这部分内容放到UI线程去做，会避免跨进程通信的消耗，当逻辑层的AppService遇到阻塞时，UI线程照样可以正常地处理和渲染。
    </p>
    <h4 id="6小程序运行机制">
     （6）小程序运行机制
    </h4>
    <p>
     <em>
      小程序的启动：
     </em>
     <br/>
     ①冷启动：首次打开或被微信主动销毁（小程序进入后台超过时间（5min）或短时间（5s）收到2次系统告警时）。
     <br/>
     更新机制：若有更新，会异步下载代码包，同时启动本地包启动，新版本在下次打开时被用，如果需要立即使用新版本，则需要用到后面的一个API。
     <br/>
     ②热启动：不是第一次打开小程序，只需把后台态的小程序切换到前台来使用。
    </p>
    <p>
     <em>
      小程序加载机制：
     </em>
     <br/>
     <img alt="小程序加载机制" src="https://i-blog.csdnimg.cn/blog_migrate/2b0fe3102d2103275fe952df13f23851.png" title="">
      <br/>
      程序启动时，会向CDN（CDN是内容分发网络，主要的作用是帮我们把请求的内容分发到距离我们最近的网络节点服务器，提高用户响应的速度及成功率，解决带宽、服务器性能带来的一些延迟问题）请求最新的代码包，第一次启动时，要等到代码包下载完毕，注入到WebView容器，执行之后，才能看到小程序页面（所以在网络不好的时候会感到启动时间会长一些），客户端会帮我们把代码包缓存到本地，下一次启动程序时，会先检查是否有最新版本的代码包。
      <br/>
      小程序内，还可以访问第三方服务器，发起一个Ajax请求，服务器会返回给我们一个JSON格式的数据。
     </img>
    </p>
    <p>
     <em>
      应用生命周期：
     </em>
     <br/>
     四个钩子：
     <br/>
     onLaunch:
     <br/>
     onShow:
     <br/>
     onHide:
     <br/>
     onError:
     <br/>
     应用生命周期中的globalData：表示应用的全局数据
     <br/>
     第一次进入小程序时，客户端会帮我们初始化好运行环境，初始化好后，逻辑层会给app.js实例派发onLaunch事件，在app.js的参数里onLaunch方法被调用；按右上角的关闭键或者按home键，会调用o’nHide方法；再次打开小程序时，小程序进入前台状态，调用的是onShow方法；当小程序出现错误或API调用失败时会触发onError方法，并会向该方法中传入一些错误信息。
     <br/>
     <em>
      页面生命周期：
     </em>
     <br/>
     五个钩子：
     <br/>
     onLoad：页面初次加载时，会给页面实例派发，页面没被销毁之前只被调用一次。调用时可以拿到该页面打开所需的一些参数。
     <br/>
     onShow：页面显示之后，配置构造器的onShow方法调用。从其它页面返回当前页面时也会调用。
     <br/>
     onReady：页面初次渲染完成之后，配置构造器中的onReady方法就会被调用，在onShow方法之后被调用，页面没被销毁之前只能调用一次。触发之后，逻辑层和视图层就可以进行交互。
     <br/>
     onHide：在当前页打开一个新的窗口时，当前页会触发onHide。
     <br/>
     onUnload：关闭当前页
     <br/>
     页面生命周期中的data：当前页面的数据
     <br/>
     view线程和AppService线程共同完成生命周期的调用，在onLoad和onShow中请求服务器数据，拿到数据后会触发首次渲染，渲染好后会通知AppService，AppService接收到onReady调用，会把数据再次发送到视图层。

    <blockquote>
     <p>
      事件：视图层到逻辑层的通讯方式，可以将用户行为反馈到逻辑层，事件可以绑定在组件上，触发事件后，就会执行逻辑层中对应的事件处理函数

    <p>
     小程序的事件模型：
     <br/>
     <img alt="小程序的事件模型" src="https://i-blog.csdnimg.cn/blog_migrate/90a9e4ae06742c1a4f229b1c6808fb5b.png" title="">
      <br/>
      ①事件捕获阶段
      <br/>
      事件从最外层节点传递到目标节点元素，依次检查每个节点是否绑定了同一事件类型的监听回调函数，有则执行
      <br/>
      ②事件处理阶段
      <br/>
      事件在到达目标节点之后，触发目标节点所绑定的监听回调函数
      <br/>
      ③事件冒泡阶段
      <br/>
      事件从目标节点依次向最外层节点，一次检查每个节点是否绑定了同一事件类型的监听回调函数，有则执行
      <br/>
      target：触发事件的根源组件
      <br/>
      current-target：触发事件的当前组件
      <br/>
      <img alt="可捕获事件和可冒泡事件" src="https://i-blog.csdnimg.cn/blog_migrate/008eb4cc34276cd5d6f73c1856b96a4d.png" title="">

       touchcancel是被来电事件等打断的事件
       <br/>
       tap事件是手指点击一次屏幕的操作（手指按下到离开小于350ms）
       <br/>
       longpress和longtap都是&gt;350ms的事件，后者在触发之后还会继续触发tap的一个操作，所以更多地使用longpress
       <br/>
       可冒泡事件包括可捕获事件，加上一些动画事件。
       <br/>
       transitionend动画结束后的事件，animationstart表示wxss动画开始事件，animationiteration表示wxss动画迭代一次之后的事件，animationend动画结束事件，touchforcechange表示触发了3d touch事件
