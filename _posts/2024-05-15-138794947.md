    <h3>
     <a id="_0">
     </a>
     前言
    </h3>
    <p>
     <strong>
      「Model系列」芯片
     </strong>
     是启明智显针对工业、行业以及车载产品市场推出的系列HMI芯片，主要应用于工业自动化、智能终端HMI、车载仪表盘、串口屏、智能中控、智能家居、充电桩显示屏、储能显示屏、工业触摸屏等领域。此系列具有高性能、低成本的特点，支持工业宽温、2D加速、PNG解码、JPEG编解码引擎及屏幕高达60FPS的刷新，RS485\CAN\串口\网口等丰富灵活的接口以及SDK软硬件开源。目前主流芯片型号有Model3C、Model3A、Model3及Model4等，已在智能商显、智能车载中控仪表、工业控制、工业PLC、工业网关、HMI人机交互等终端上实现大规模应用，获得了客户广泛的认可和信赖。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7a79a40e78d1183d87bcb17450db2f06.png"/>
    </p>
    <h3>
     <a id="ModelModel3CModel3Model4_5">
     </a>
     Model系列（Model3C/Model3/Model4）烧录操作指南
    </h3>
    <p>
     对于工业级HMI芯片的Model系列（Model3C/Model3/Model4）的烧录操作，它们支持不同的烧录方式，包括串口、USB和SD卡烧录。以下是对这些烧录方式的简要说明：
    </p>
    <p>
     <strong>
      一、串口烧录（适用于Model3C/Model3A和Model3/Model4）：
     </strong>
    </p>
    <p>
     准备一个普通的串口工具，确保您的计算机能够识别并与其通信。
    </p>
    <ul>
     <li>
      上位机烧录
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/87a1f56c0c17e801f2ea49c8253395ca.png"/>
     </li>
     <li>
      打开 PanelLan
      <br/>
      进入“烧写镜像”页面
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/016cfba6b8f83f8096a757d7c870d18d.png">
       <br/>
       板卡上电后，进入烧录模式有两种方式：
      </img>
     </li>
     <li>
      按住 UBOOT 键不放，再按 RESET 键；
     </li>
     <li>
      在串口命令行输入 aicupg；
     </li>
    </ul>
    <p>
     进入烧录模式后，选择烧录串口， 手动点击“查询设备”, 当提示框出现 “查询成功”，表示成功进入烧录模式，如图所示：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5ee63110627e630a4d36fb2960898381.png"/>
    </p>
    <blockquote>
     <p>
      <strong>
       注意：串口烧录需要重新编译bootloader，串口升级部分是没有打开；scons --apply-def=2：
      </strong>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4a974526b1c34949a4a8c2c529f2a260.png">
       <br/>
       编译bootloader要生效对应配置
      </img>
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/92fa50663d328bde3779fb1b9efd7ad0.png"/>
    </p>
    <ul>
     <li>
      <p>
       选择编译好的镜像文件，加载镜像时， PanelLan 会识别出镜像的基本信息；
      </p>
     </li>
     <li>
      <p>
       点击开始，进行烧录，烧录时，PanelLan 会显示烧录的进度、速率、结果。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      二、USB烧录（仅适用于Model3/Model4）：
     </strong>
    </p>
    <p>
     您可以通过USB接口直接连接计算机和设备。
     <br/>
     zx-rtt 可以实现挂载 U 盘（FAT32 格式），需要配置如下内容：
    </p>
    <ul>
     <li>
      配置 USB host；
     </li>
     <li>
      配置 cherry USB 协议栈；
     </li>
     <li>
      检查 USB-SW 短接线；
     </li>
    </ul>
    <p>
     <strong>
      配置 USB Host
     </strong>
    </p>
    <p>
     挂载 U 盘时，需将 对应 USB 接口配置为 HOST，以 ZXM3C7D000M4R64 为例，进入 SDK 的 menuconfig 菜单做如下配置：
    </p>
    <blockquote>
     <p>
      Board options —&gt;
      <br/>
      [*] Using Usb0
      <br/>
      Select Usb0 mode (Host) —&gt;
     </p>
    </blockquote>
    <p>
     <strong>
      配置 cherry USB 协议栈
     </strong>
    </p>
    <p>
     配置 cherry USB 如下：
    </p>
    <blockquote>
     <p>
      Local packages options —&gt;
      <br/>
      Third-party packages options —&gt;
      <br/>
      [
      <em>
       ] CherryUSB: tiny and portable USB host/device stack for embedded system with USB IP
       <br/>
       [ ] Enable usb device mode
       <br/>
       [
      </em>
      ] Enable usb host mode —&gt;
      <br/>
      [
      <em>
       ] Enable usb msc driver
       <br/>
       [
      </em>
      ] Enable usb example
     </p>
    </blockquote>
    <p>
     <strong>
      设置自动挂载
     </strong>
    </p>
    <p>
     在 target///board.c 中，找到 mount_table 数组，添加代码如下：
    </p>
    <pre><code>const struct dfs_mount_tbl mount_table[] = {
...
    {"udisk", "/udisk", "elm", 0, 0, 0},
...
};
</code></pre>
    <p>
     编译烧录后，系统会自动挂载 U 盘。
    </p>
    <p>
     <strong>
      编译烧录
     </strong>
    </p>
    <p>
     配置完成后，保存退出，编译、烧录
    </p>
    <blockquote>
     <p>
      注意
     </p>
     <p>
      烧写前务必确保 USB-SW 跳线帽没有短接，否则无法进入烧录模式
     </p>
    </blockquote>
    <p>
     <strong>
      验证
     </strong>
    </p>
    <ul>
     <li>
      U 盘识别
     </li>
    </ul>
    <p>
     烧录完成后，断开电源，接入 U 盘，短接 USB-SW，上电
    </p>
    <blockquote>
     <p>
      注意
     </p>
     <p>
      对于没有 USB-SW 排针的开发板，无需再做处理
     </p>
    </blockquote>
    <p>
     检查 U 盘 识别， 看到 udisk 设备，表明系统已识别：
    </p>
    <pre><code>aic /&gt; list_device
device           type         ref count
-------- -------------------- ----------
...
udisk    Block Device         1
...
</code></pre>
    <blockquote>
     <p>
      备注
     </p>
     <p>
      若使用手动挂载，则在此时执行 mount udick /udick elm 命令。
     </p>
    </blockquote>
    <ul>
     <li>
      查看文件
     </li>
    </ul>
    <p>
     进入 U 盘目录，查看文件：
    </p>
    <pre><code>aic /&gt; cd udisk
aic /udisk&gt; ls
Directory /udisk:
System Volume Inform&lt;DIR&gt;
M4                &lt;DIR&gt;
zx.PPT        4814660
</code></pre>
    <p>
     至此，U 盘挂载完成。
    </p>
    <p>
     <strong>
      三、SD卡烧录（Model3C/Model3A和Model3/Model4）：
     </strong>
    </p>
    <p>
     <strong>
      准备一张具有足够容量的SD卡，zx-rtt 可以实现挂载 SD 卡（FAT32 格式），需要配置如下内容：
     </strong>
    </p>
    <ul>
     <li>
      配置 SDMC 接口；
      <br/>
      挂载 SD 卡时，需将 对应 SDMC 接口选中，以 ZXM3C7D000M4R64 开发板为例，核对原理图可知，SD 卡对接的是 SDMC1，进入 SDK 的 menuconfig 菜单做如下配置：
     </li>
    </ul>
    <blockquote>
     <p>
      Board options —&gt;
      <br/>
      [*] Using SDMC1
      <br/>
      SDMC1 Parameter
     </p>
    </blockquote>
    <ul>
     <li>
      设置自动挂载
      <br/>
      在 target///board.c 中，找到 mount_table 数组，添加代码如下：
     </li>
    </ul>
    <pre><code>const struct dfs_mount_tbl mount_table[] = {
...
    {"sd0p0", "/sdcard", "elm", 0, 0, 0},       // SD 卡首个分区
    {"sd0", "/sdcard", "elm", 0, 0, 0},         // 设备节点，一定存在
...
};
</code></pre>
    <p>
     关于 SD 卡及分区的解释：
    </p>
    <ol>
     <li>
      sd0 通常为 SD 卡的设备节点；
     </li>
     <li>
      sd0p0 通常为 SD 卡第一个分区，通常情况下也是唯一分区；假设 SD 卡有多个分区，那么文件名需要递增，例如 sd0p1、sd0p2 … ；
     </li>
     <li>
      若只有 sd0 没有 sd0p0，较多出现在使用过 linux 中的 mkfs.vfat 命令直接格式化 SD 卡的设备节点；
     </li>
     <li>
      本文示例以一个 SD 卡只有一个分区为例进行讲解。
     </li>
    </ol>
    <p>
     编译烧录后，系统会自动挂载 SD 卡。
    </p>
    <ul>
     <li>
      编译烧录
      <br/>
      配置完成后，保存退出，编译、烧录
     </li>
     <li>
      验证
      <br/>
      烧录完成后，断开电源，接入 SD 卡，上电，检查设备，看到 sd0 和 sd0p0 设备，表明系统已识别 SD 卡：
     </li>
    </ul>
    <pre><code>aic /&gt; list_device
device           type         ref count
-------- -------------------- ----------
...
sd0      Block Device         0
sd0p0    Block Device         1
...
</code></pre>
    <blockquote>
     <p>
      备注
     </p>
     <p>
      若使用手动挂载，则在此时执行 mount sd0 /sdcard elm 和 mount sd0p0 /sdcard elm 。
     </p>
    </blockquote>
    <ul>
     <li>
      查看文件
      <br/>
      进入 sdcard 目录，查看文件：
     </li>
    </ul>
    <pre><code>aic /&gt; cd sd
aic /&gt; cd sdcard
aic /sdcard&gt; ls
Directory /sdcard:
lvgl_data           &lt;DIR&gt;
aicdoc.tar     89180160
tools               &lt;DIR&gt;
aic /sdcard&gt;
</code></pre>
    <p>
     至此， SD 卡挂载完成。
    </p>
