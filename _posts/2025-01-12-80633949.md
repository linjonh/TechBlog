    <h2 id="android-数据库学习总结">
     android 数据库学习总结
    </h2>
    <ul>
     <li>
      android 为了让我们能方便的使用数据库，给我们提供了一个SQLiteOpenHelper帮助类
     </li>
     <li>
      它是一个抽象类，所以我们需要自定义一个类来继承这个类来实现我们创建数据库的逻辑
     </li>
    </ul>
    <h3 id="实现逻辑">
     实现逻辑
    </h3>
    <h4 id="创建以及更新">
     创建以及更新
    </h4>
    <h5 id="先说一下他的构造方法我们一般重写第一个有四个参数的构造方法">
     先说一下他的构造方法，我们一般重写第一个(有四个参数的)构造方法
    </h5>
    <ul>
     <li>
      四个参数分别的意思是（context，数据库名，null(这里也允许是一个自定义的Cursor，他会在查询到数据之后返回)，当前数据库的版本号）
     </li>
    </ul>
    <h5 id="再说一下两个我们需要重写的方法">
     再说一下两个我们需要重写的方法
    </h5>
    <ul>
     <li>
      onCreate()，创建数据库方法，内部调用，创建一个数据库
     </li>
     <li>
      onUpgrade(),升级数据库方法，也是内部调用
     </li>
    </ul>
    <h5 id="还有我们一般使用的两个方法都是得到一个数据库实例">
     还有我们一般使用的两个方法（都是得到一个数据库实例）
    </h5>
    <ul>
     <li>
      getReadableDatabase()，以读的方式得到一个数据库实例
     </li>
     <li>
      <p>
       getWritableDatabase()，以写的方式打开一个数据库实例
      </p>
     </li>
     <li>
      <p>
       <strong>
        注意
       </strong>
       ：当数据库不可写入的时候，前者是返回一个只能
       <strong>
        写
       </strong>
       的数据库，而后者是报异常
      </p>
     </li>
    </ul>
    <h5 id="说一下创建的逻辑">
     说一下创建的逻辑
    </h5>
    <ul>
     <li>
      当我们去new这个数据库的时候，其实只是拿到了这个数据库的操作句柄，关于具体这个数据库存不存在，这就不是这里干的事
     </li>
     <li>
      然后我们去拿着这个实例再去得到读数据库的实例，当在这里的时候他就会判断，如果数据库不存在，就onCreate()，如果存在就去判断版本是不是当前版本，如果我们在new的时候传入的版本比之前已经存在的数据库版本高，那么就会调用onUpgrade()这个方法去更新数据库，所以更新数据库的逻辑需要我们自己去写
     </li>
     <li>
      这里看一下建表语句
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs sql"><span class="hljs-operator"><span class="hljs-keyword">create</span> <span class="hljs-keyword">table</span> Book(
    id <span class="hljs-keyword">integer</span> <span class="hljs-keyword">primary</span> <span class="hljs-keyword">key</span> autoincrement,
    author text,
    price <span class="hljs-keyword">real</span>,
    pages <span class="hljs-keyword">integer</span>,
    name text)</span></code></pre>
    <ul>
     <li>
      说明：integer代表整型，real代表浮点型，text代表文本型，blob代表二进制primary key 将id列设置为主键，并用autoincrement表示id列是自增的
     </li>
     <li>
      这里看一下自定义的数据库的创建方法的代码
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs java"><span class="hljs-keyword">public</span> <span class="hljs-keyword">static</span> <span class="hljs-keyword">final</span> String CREATE_BOOK = <span class="hljs-string">"create table Book ("</span>
            + <span class="hljs-string">"id integer primary key autoincrement,"</span>
            + <span class="hljs-string">"author text,"</span>
            + <span class="hljs-string">"price real,"</span>
            + <span class="hljs-string">"pages integer"</span>
            + <span class="hljs-string">"name text)"</span>;

<span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">onCreate</span>(SQLiteDatabase db) {
        db.execSQL(CREATE_BOOK);
        Log.d(<span class="hljs-string">"------"</span>,<span class="hljs-string">"创建成功"</span>);
    }</code></pre>
    <ul>
     <li>
      说到底就是将我们刚才说的数据库建表语句写成一个字符串传入execSQL()这个方法，就完成了数据库的建立
     </li>
     <li>
      activity中的代码
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs cs">dbHelper = <span class="hljs-keyword">new</span> MyDatabaseHelper(<span class="hljs-keyword">this</span>,<span class="hljs-string">"BookStore.db"</span>,<span class="hljs-keyword">null</span>,<span class="hljs-number">2</span>);
SQLiteDatabase writableDatabase = dbHelper.getWritableDatabase();</code></pre>
    <ul>
     <li>
      第二句话调用之后数据库才被创建
     </li>
    </ul>
    <h5 id="数据库的升级">
     数据库的升级
    </h5>
    <ul>
     <li>
      根据我们的前面的解释，数据库的升级方法需要我们自己实现，看一下这个方法的实现
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs sql">public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        db.execSQL("<span class="hljs-operator"><span class="hljs-keyword">drop</span> <span class="hljs-keyword">table</span> <span class="hljs-keyword">if</span> <span class="hljs-keyword">exists</span> Book<span class="hljs-string">");
        db.execSQL("</span><span class="hljs-keyword">drop</span> <span class="hljs-keyword">table</span> <span class="hljs-keyword">if</span> <span class="hljs-keyword">exists</span> Category<span class="hljs-string">");
        onCreate(db);
    }</span></span></code></pre>
    <ul>
     <li>
      里面是一句固定字符串，如果存在就删除，删除完毕在创建一次
     </li>
     <li>
      注意这个升级方法，是在调用了getWritableDatabase()和getReadableDatabase()这个方法才会被调用的
     </li>
     <li>
      现在我们在数据库已经存在的情况下，在new 的时候传入一个高版本的数字，数据库的升级工作将在我们调用那两个方法的时候完成
     </li>
    </ul>
    <h4 id="增删改查">
     增删改查
    </h4>
    <ul>
     <li>
      为我们提供了四个方法，一一看吧
     </li>
    </ul>
    <h5 id="insert插入添加">
     insert：插入（添加）
    </h5>
    <ul>
     <li>
      接收三个参数：
     </li>
     <li>
      第一个：表名
     </li>
     <li>
      第二个：用于在未指定添加数据的情况下给某些可为空的列自动赋值为NULL，一般用不到，直接传null
     </li>
     <li>
      第三个：是一个ContentValues对象，这个对象用于将我们想添加的数据进行包装，他提供了一系列的put键值对的方法，让我们去设置我们想要传入的数据
     </li>
     <li>
      看代码
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs perl">SQLiteDatabase writableDatabase = dbHelper.getWritableDatabase();
ContentValues <span class="hljs-keyword">values</span> = new ContentValues();
<span class="hljs-keyword">values</span>.put(<span class="hljs-string">"name"</span>,<span class="hljs-string">"大笑江湖"</span>);

    <ul>
     <li>
      不解释，看下一个
     </li>
    </ul>
    <h5 id="updata更新数据">
     updata：更新数据
    </h5>
    <ul>
     <li>
      四个参数
     </li>
     <li>
      第一个：表名
     </li>
     <li>
      第二个：ContentValues对象，把要跟新的数据包装
     </li>
     <li>
      第三个和第四个，是用来约束更新某一行或者某几行的数据，不指定的话默认就是更改所有行
     </li>
     <li>
      上代码
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs javascript">SQLiteDatabase writableDatabase = dbHelper.getWritableDatabase();
ContentValues values = <span class="hljs-keyword">new</span> ContentValues();
values.put(<span class="hljs-string">"price"</span>,<span class="hljs-number">122</span>);

    <ul>
     <li>
      没疑问，稍稍一看就懂，下一个
     </li>
    </ul>
    <h5 id="delete删除数据">
     delete:删除数据
    </h5>
    <ul>
     <li>
      三个参数
     </li>
     <li>
      第一个：表名
     </li>
     <li>
      第三个和第二个，是用来约束删除某一行或者某几行的数据，不指定的话默认就是删除所有行
     </li>
     <li>
      上代码
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs javascript">SQLiteDatabase writableDatabase = dbHelper.getWritableDatabase();
writableDatabase.delete(<span class="hljs-string">"Book"</span>,<span class="hljs-string">"page &gt; ?"</span>,<span class="hljs-keyword">new</span> <span class="hljs-built_in">String</span>[]{<!-- --><span class="hljs-string">"200"</span>});</code></pre>
    <h5 id="query查询">
     query：查询
    </h5>
    <ul>
     <li>
      这个方法比较复杂，有好几个重载方法，而且方法的参数特别多，但其实都是对查询数据的约束，我们来看一下一些参数的意义
     </li>
     <li>
      table :指定查询的表名
     </li>
     <li>
      columns :指定查询的列名
     </li>
     <li>
      selection ：指定where的约束条件
     </li>
     <li>
      selectionArgs ：为上个参数的占位符提供具体的参数
     </li>
     <li>
      groupBy :指定需要groupBy的列
     </li>
     <li>
      having ：对groupBy后的结果进一步约束
     </li>
     <li>
      orderBy：指定查询结果的排序方式
     </li>
     <li>
      看一下代码吧
     </li>
    </ul>

     </li>

    <h3 id="总结">
     总结
    </h3>
    <ul>
     <li>
      其实操作也就那样子，熟悉就好
     </li>
     <li>
      主要对更新数据库那里不太理解，为什么更新的时候要把已经存在的数据库删除掉呢
     </li>
     <li>
      还有，对数据库的增删改查主要是对表以及约束条件的书写，需要注意
     </li>
    </ul>
