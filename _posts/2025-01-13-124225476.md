    <p>
     1、 在创建项目的时候可以直接选择带有jest的一项然后可以直接在tests里面编写我们的ts文件
     <br/>
     然后再npm run unit:test即可
     <br/>
     2、 当我们在创建的时候没选择jest该怎么办呢
     <br/>
     首先需要的项目依赖
     <br/>
     npm i -S @vue/cli-plugin-unit-jest vue-jest @vue/test-utils
    </p>
    <p>
     如果是ts项目，需要额外添加 ts-jest @types/jest 安装包
    </p>
    <p>
     2在根目录新建jest.config.js 文件，配置如下
     <br/>
     module.exports ={
     <!-- -->
     <br/>
     preset:’@vue/cli-plugin-unit-jest/presets/typescript-and-babel’,
     <br/>
     }
    </p>
    <p>
     然后在tsconfig.json中的types选项中新增jest选项
     <br/>
     “types”: [
     <br/>
     “jest”
     <br/>
     ],
     <br/>
     接下来在我们的package.json里添加 “test”: “jest”
     <br/>
     下面上个案例吧！
     <br/>
     首先我编写了一个改变我们数组的函数
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/70fbfd9a5724f33f3bba785c78677e5d.png">
      <br/>
      然后在我们跟目录下创建test/unit文件夹
      <br/>
      我编写了一个arr.test.ts
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c4748cc43fc1aa8f0af38d673181adc1.png">
       <br/>
       接下来使用npm run jest去测试它

       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a35d17ca4de5c2458c73b635be9fef61.png">
