    <p>
     大家好，我是知微！
    </p>
    <p>
     作为程序员，不论是开发还是学习，肯定会用到开源项目，那么怎么快速在开源网站找到这些项目呢？
    </p>
    <ul>
     <li>
      <p>
       常用的开源网站有：github 和 gitee
      </p>
     </li>
     <li>
      <p>
       github是全球最大的开源社区，今天就以github为例，演示一下
      </p>
     </li>
     <li>
      <p>
       github界面
      </p>
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/215f35bf743257660e01efbc7887f3b0.png"/>
    </p>
    <ul>
     <li>
      一般来说，优秀的项目，维护会比较频繁，提交数也就会多一点。当然，一个好的项目，它的
      <code>
       Star
      </code>
      数和
      <code>
       Fork
      </code>
      数也会比较多。
     </li>
    </ul>
    <h3>
     <a id="_17">
     </a>
     普通人搜索的方式
    </h3>
    <p>
     1、打开
     <code>
      github
     </code>
    </p>
    <p>
     2、输入关键词，搜出来一堆的项目，随便浏览一下，简单看一下README文档。嗯，感觉还不错的样子！
    </p>
    <p>
     3、然后就是克隆仓库，看一下源码，发现看不懂，内心OS：这是给人看的吗？删了删了！
    </p>
    <p>
     <img alt="心累" src="https://i-blog.csdnimg.cn/blog_migrate/252a9395658e21c18b5ff148b8746b35.jpeg"/>
    </p>
    <p>
     用这种方法想要找到适合自己的开源项目，跟大海捞针没什么区别。只会一次次的浪费时间，也会消耗自己的耐心和精力，到最后干脆摆烂躺平了。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/7968f0be8cc4bde09bf8f1da681562ae.jpeg"/>
    </p>
    <p>
     不过在看这篇文章的你，肯定不想这样啦！那么下面的10个
     <code>
      github
     </code>
     高级搜索技巧，赶紧学起来吧！
    </p>
    <h3>
     <a id="10_37">
     </a>
     10个高级搜索技巧
    </h3>
    <h4>
     <a id="in_41">
     </a>
     ⑴、通过
     <code>
      in
     </code>
     关键词搜索
    </h4>
    <p>
     <strong>
      in关键词可以用来限制搜索范围
     </strong>
     ，精准搜索仓库名称name，说明description 以及README 文件中的内容。description 就是github界面图中， About 中的信息。
    </p>
    <table>
     <thead>
      <tr>
       <th align="center">
        示例
       </th>
       <th align="center">
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        in:name C++
       </td>
       <td align="center">
        搜索项目名称中包含 “C++” 的所有项目
       </td>
      </tr>
      <tr>
       <td align="center">
        in:description machine learning
       </td>
       <td align="center">
        搜索项目描述中包含 “machine learning” 的所有项目
       </td>
      </tr>
      <tr>
       <td align="center">
        in:readme API
       </td>
       <td align="center">
        搜索项目介绍文档（README）中包含 “API” 的所有项目
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     下面以搜索
     <code>
      spring boot
     </code>
     相关的项目为例：
    </p>
    <ul>
     <li>
      如果直接在搜索栏输入
      <code>
       spring boot
      </code>
      并回车，会搜到46w个结果
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/2b7f632be2c7e28d154d818e10b386ee.png"/>
    </p>
    <ul>
     <li>
      <p>
       做一个限制，这次搜索项目名里含有 spring boot 的
      </p>
      <blockquote>
       <p>
        in:name spring boot
       </p>
      </blockquote>
     </li>
    </ul>
    <p>
     ​ 看到没有，数量马上又46w降到29.5w，但是项目数量还是太多了。没关系，我们可以再加其他的搜索限制，接着往下看！
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/81c431265a21288a30ebae18a88ed369.png"/>
    </p>
    <h4>
     <a id="_starsforks__73">
     </a>
     ⑵、通过 stars、forks 数量搜索
    </h4>
    <p>
     一个优秀的项目 star 数量和 fork 数量肯定不会少，那么我可以使用
     <code>
      大于
     </code>
     某个数，或者
     <code>
      在某个范围之间
     </code>
     的方式来过滤项目：
    </p>
    <ol>
     <li>
      <strong>
       按照星标数量搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 stars:&gt;=数量
        </code>
        或
        <code>
         关键字 stars:数量1..数量2
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       按照分支数量搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 forks:&gt;=数量
        </code>
        或
        <code>
         关键字 forks:数量1..数量2
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <table>
     <thead>
      <tr>
       <th align="center">
        示例
       </th>
       <th align="center">
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        javascript stars:&gt;1000
       </td>
       <td align="center">
        搜索所有星标数量大于1000JavaScript项目
       </td>
      </tr>
      <tr>
       <td align="center">
        javascript stars:1000…2000
       </td>
       <td align="center">
        搜索所有星标数量在1000到2000之间的JavaScript项目
       </td>
      </tr>
      <tr>
       <td align="center">
        web framework forks:&gt;500
       </td>
       <td align="center">
        搜索所有分支数量大于500的 Web 框架项目
       </td>
      </tr>
      <tr>
       <td align="center">
        web framework forks:500…1000
       </td>
       <td align="center">
        搜索所有分支数量在500到1000之间的 Web 框架项目
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      <p>
       我们进一步限制搜索范围，要求star数量大于2000，数量马上变成只有31个
      </p>
      <blockquote>
       <p>
        in:name spring boot starts:&gt;2000
       </p>
      </blockquote>
      <p>
       <img alt="stars大于2000" src="https://i-blog.csdnimg.cn/blog_migrate/ced2f4d5224261488405533980dd0812.png"/>
      </p>
     </li>
     <li>
      <p>
       再要求fork数量大于1000，满足要求的项目数量更少了，只有19个
      </p>
      <blockquote>
       <p>
        in:name spring boot starts:&gt;2000 forks:&gt;1000
       </p>
      </blockquote>
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/d967e96e80e20ef95acfa833141e9a72.png"/>

    <h4>
     <a id="_105">
     </a>
     <strong>
      ⑶、按项目创建时间、更新时间搜索
     </strong>
    </h4>
    <p>
     如果优秀项目比较多，相信大家都会喜欢最新的项目，那么就可以把老旧的项目筛选出去。
    </p>
    <p>
     要是本来就没几个项目供你选，就没必要做这一步了。
    </p>
    <ol>
     <li>
      <strong>
       按照项目创建时间搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 created:&gt;=日期
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       按照项目更新时间搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 pushed:&gt;=日期
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <table>
     <thead>
      <tr>
       <th align="center">
        示例
       </th>
       <th align="center">
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        java created:&gt;=2022-01-01
       </td>
       <td align="center">
        搜索在2022年1月1日之后创建的Java项目
       </td>
      </tr>
      <tr>
       <td align="center">
        android pushed:&gt;=2022-01-01
       </td>
       <td align="center">
        搜索在2022年1月1日之后更新的Android项目
       </td>
      </tr>
     </tbody>
    </table>

      演示：

    <h4>
     <a id="_127">
     </a>
     ⑷、按照地区、编程语言搜索
    </h4>
    <ul>
     <li>
      <p>
       一般我们都是想找到自己会的编程语言写的项目，否则还得专门去学一遍语言才能看懂，那岂不是兜了一大圈。
      </p>
     </li>
     <li>
      <p>
       另外，要是项目有中文的文档说明，就更好了！(虽然很多优秀的项目都是用英文写的，程序员不能排斥学习英语呀！)
      </p>
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       按照地区搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         location:地区 关键字
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       按照编程语言搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         language:编程语言 关键字
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <table>
     <thead>
      <tr>
       <th align="center">
        示例
       </th>
       <th align="center">
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        location:china
       </td>
       <td align="center">
        地区为中国
       </td>
      </tr>
      <tr>
       <td align="center">
        language:python
       </td>
       <td align="center">
        编程语言为python
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      例如使用 python 编程语言，且star数量大于1000的国产项目，搜索条件就是：location:china language:python stars:&gt;1000
     </li>
    </ul>

      来筛选一些简单点的项目。
     </li>
    </ul>

     <li>
      <strong>
       按照文件名搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 filename:文件名
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       按照仓库大小搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 size:&gt;=大小
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       按照扩展名搜索：
      </strong>
      <ul>
       <li>
        <strong>
         语法：
        </strong>
        <code>
         关键字 extension:扩展名
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <table>

    <h4>
     <a id="LICENSE_215">
     </a>
     ⑻、根据仓库的LICENSE搜索
    </h4>
    <p>
     在使用任何开源项目之前，最好详细阅读并理解项目所采用的具体授权协议，因为有的开源项目作者是明确禁止商用的，如果你使用在商业上，后面会很麻烦的！
    </p>
    <p>
     <strong>
      按照许可证搜索：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       语法：
      </strong>
      <code>
       license:许可证类型 关键字
      </code>
     </li>
    </ul>
    <table>
     <thead>
      <tr>
       <th align="center">
        示例
       </th>
       <th align="center">
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        license:gpl encryption
