      <p>
       Web 前端优化最佳实践第六部分面向 图片(Image)，这部分目前有 4 条规则。在最近的 Velocity 2008 技术大会上，Yahoo! 的 Stoyan Stefanov 做的
       <a href="http://en.oreilly.com/velocity2008/public/schedule/detail/2405" rel="nofollow">
        <font color="#0082ff">
         Image Optimization: How Many of These 7 Mistakes Are You Making
        </font>
       </a>
       也非常有参考价值。结合一起说一下。
      </p>
      <h5>
       1. 优化图片 (Optimize Images)
      </h5>
      <p>
       使用
       <acronym title="Graphics Interchange Format
">
        GIF
       </acronym>
       、JPG 还是
       <acronym title="Portable Network Graphics
">
        PNG
       </acronym>
       格式的图片? 尽可能的使用
       <acronym title="Portable Network Graphics
">
        PNG
       </acronym>
       格式的图片，更多的功能，更小的尺寸(与
       <acronym title="Graphics Interchange Format
">
        GIF
       </acronym>
       相比)。
      </p>
      <p>
       对于
       <acronym title="Portable Network Graphics
">
        PNG
       </acronym>
       图片，考虑用
       <a href="http://pmt.sourceforge.net/pngcrush/" rel="nofollow">
        <font color="#0082ff">
         Pngcrush
        </font>
       </a>
       或类似的工具进行优化。常见的工具如下表:
      </p>
      <ul>
       <li>
        <a href="http://pmt.sourceforge.net/pngcrush/" rel="nofollow">
         <font color="#0082ff">
          pngcrush
         </font>
        </a>
        http://pmt.sourceforge.net/pngcrush/
       </li>
       <li>
        <a href="http://www.pobox.com/~jason1/pngrewrite/" rel="nofollow">
         <font color="#0082ff">
          pngrewrite
         </font>
        </a>
        http://www.pobox.com/~jason1/pngrewrite/
       </li>
       <li>
        <a href="http://www.cs.toronto.edu/~cosmin/pngtech/optipng/" rel="nofollow">
         <font color="#0082ff">
          OptiPNG
         </font>
        </a>
        http://www.cs.toronto.edu/~cosmin/pngtech/optipng/
       </li>
       <li>
        <a href="http://advsys.net/ken/utils.htm" rel="nofollow">
         <font color="#0082ff">
          PNGOut
         </font>
        </a>
        http://advsys.net/ken/utils.htm
       </li>
      </ul>
      <p>
       对
       <acronym title="Joint Photographics Experts Group
">
        JPEG
       </acronym>
       图片的优化工具：
      </p>
      <ul>
       <li>
        <a href="http://jpegclub.org/" rel="nofollow">
         <font color="#0082ff">
          jpegtran
         </font>
        </a>
        (http://jpegclub.org/)
       </li>
      </ul>
      <p>
       必需要强调的是，图片设计的同学啊，请考虑设计
       <strong>
        面向 Web 的图片
       </strong>
       ，不要动不动就设计超过可接受尺寸之外大家伙，这应该是一种习惯，而不是什么高超的技能，只需要记住就成了。
      </p>
      <h5>
       2. 使用
       <acronym title="Cascading Style Sheets
">
        CSS
       </acronym>
       Sprites 技巧对图片优化 (Optimize
       <acronym title="Cascading Style Sheets
">
        CSS
       </acronym>
       Sprites)
      </h5>
      <p>
       之前提到过，简单的说就是"利用
       <acronym title="Cascading Style Sheets
">
        CSS
       </acronym>
       background 相关元素进行背景图绝对定位"，把多次
       <acronym title="HyperText Transfer Protocol
">
        HTTP
       </acronym>
       调用变为一次调用，更多参考：
       <a href="http://alistapart.com/articles/sprites" rel="nofollow">
        <font color="#0082ff">
         CSS Sprites: Image Slicing's Kiss of Death
        </font>
       </a>
      </p>
      <h5>
       3. 不要在
       <acronym title="HyperText Markup Language
">
        HTML
       </acronym>
       中使用缩放图片 (Don't Scale Images in
       <acronym title="HyperText Markup Language
">
        HTML
       </acronym>
       )
      </h5>
      <p>
       更多的时候，可能是因为偷懒而没有制作合适大小的图片，如果是批量处理图片的话，可能一条
       <a href="http://www.imagemagick.org/" rel="nofollow">
        <font color="#0082ff">
         ImageMagic
        </font>
       </a>
       命令（convert ）就能搞定 。必须提及的是，看到太多的对图片拉伸很难看的页面，救救这些页面!
      </p>
      <h5>
       4. 用更小的并且可缓存的 favicon.ico (Make favicon.ico Small and Cacheable)
      </h5>
      <p>
       更小，可缓存，这两条可能都不是问题。问题是，
       <strong>
        太多站点根本没有 favicon.ico
       </strong>
       。有的时候，判断独立域名的 Blog 是否专业，基本看一下是否有 favicon.ico 就差不多了。
      </p>
