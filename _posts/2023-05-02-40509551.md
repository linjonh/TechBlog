    <h1 class="title-article" id="articleContentId">
     原型设计实践
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     <span style="font-family:华文宋体; color:rgb(0,0,0)">
      <strong>
       <span style="font-size:18px">
        在教师管理系统这个项目中，开始学习并使用Axure进行原型设计。尽管现在可以用Axure来进行一些简单的界面设计和交互设计，仍觉得自己掌握的只是Axure强大功能的微小的一部分。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体">
       <span style="font-size:18px">
        提到原型设计，不得不提一个词：
        <span style="color:#000099">
         UCD
        </span>
        (User Centered Design)。即以用户为中心的设计，是在设计过程中以用户体验作为设计决策的中心，强调用户优先。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        我们开发软件产品的第一步就是需求分析。在需求分析阶段经常出现的问题就是：
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        1.用户描述的需求和开发人员理解的需求不一致。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        2.用户需求经常变更。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="font-family:华文宋体; color:rgb(0,0,0)">
      <strong>
       <span style="font-size:18px">
        用户不会关心你的软件产品用了什么代码，他们只会关心你的产品看起来是不是舒服，用起来是否方便、轻松、愉快。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="font-family:华文宋体; color:rgb(0,0,0)">
      <strong>
       <span style="font-size:18px">
        所以我们在真正开发产品前，应先设计出这样的东西：它能展示产品的界面，也能让用户来“用”。这里的“用”，并不是要为用户
       </span>
      </strong>
     </span>
     <span style="font-family:华文宋体; font-size:18px">
      <strong>
       真正
      </strong>
     </span>
     <span style="font-size:18px; font-family:华文宋体">
      <strong>
       实现什么功能，而是一种效果，感觉像是在使用真的产品一样。我们称之为原型。
      </strong>
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      <span style="font-family:华文宋体">
       <strong>
        以上是自己的理解，
       </strong>
      </span>
      <span style="font-family:华文宋体">
       <strong>
        <span style="color:#000099">
         <span style="font-family:华文宋体">
          <strong>
           百
          </strong>
         </span>
         度上这样定义
        </span>
        ：产品原型是整个产品面市之前的一个框架设计，简单的来说是将页面的模块、元素、人机交互的形式，利用线框描述的方法，将产品脱离皮肤状态下更加具体生动地进行表达.
       </strong>
      </span>
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <strong>
        从网上找来一幅图，来了解一下原型设计的地位吧！
       </strong>
      </span>
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <strong>
        <img alt="" src="https://img-blog.csdn.net/20141027164525277?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvdTAxMDkyNDgzNA==/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
       </strong>
      </span>
     </span>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        有了原型，需求分析中反映的就不是那么可怕了。用户使用原型这样一个具体的像真的产品一样的东西，更能详细地表述自己对哪里不满意，这样用户和开发人员之间交流就有了共同的依托。所以原型就是用户和开发人员之间的桥梁。因为我对原型设计了解的还很少，这个桥梁可能并不局限于用户和开发人员之间。但是原型在沟通和应对需求变更方面起的作用是不容忽视的。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        Axure是一款原型设计工具。Axure RP即为快速原型设计。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        这里我仅对自己所了解的功能做简要的介绍。
       </span>
      </span>
     </strong>
    </p>
    <p>
    </p>
    <ul>
     <li>
      <span style="font-size:18px; font-family:华文宋体">
       <strong>
        <span style="color:#000099">
         界面设计
        </span>
        :
       </strong>
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        页面：像是我们用过的窗体
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        部件：像是我们用过的控件。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        这里重点说一下两个部件：
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="font-weight:bold">
      <span style="font-family:华文宋体">
       <span style="font-size:18px">
        <span style="background-color:rgb(255,255,255)">
         <span style="color:#cc33cc">
          动态面板
         </span>
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-weight:bold">
      <span style="font-family:华文宋体">
       <span style="font-size:18px">
        <span style="background-color:rgb(255,255,255)">
         <span style="color:#cc33cc">
          内部框架
         </span>
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        它们都像是容器，但是这个容器里的东西是可变的。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        对于动态面板，它有一个重要的属性就是状态，每个状态其实就是一个页面，可以根据设计需要在不同时候，让动态面板显示不同状态。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        对于内部框架，自我感觉也是一块可变的区域，但是它似乎更灵活，如果有些页面你想让它在一定区域内显示，这时你就可以在onclick事件下，添加用例，选择在内部框架中打开链接（页面）。
       </span>
      </span>
     </strong>
    </p>
    <p>
    </p>
    <p>
    </p>
    <ul>
     <li>
      <span style="font-size:18px; font-family:华文宋体">
       <strong>
        <span style="color:#000099">
         交互设计
        </span>
        ：
       </strong>
      </span>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        通过事件触发，来打开某个链接(页面)，设置动态面板状态，设置部件的值等等；
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        也可以在事件触发时添加条件，满足某个条件就执行什么操作，满足另一个条件就执行其他操作。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        <img alt="" src="https://img-blog.csdn.net/20141027164837407?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvdTAxMDkyNDgzNA==/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="font-weight:bold">
      <span style="font-family:华文宋体">
       <span style="font-size:18px">
        <span style="color:#cc33cc">
         温馨提示
        </span>
        ：
       </span>
      </span>
     </span>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        在设计的时候，要给每个部件设置名称。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-family:华文宋体; color:rgb(0,0,0)">
       <span style="font-size:18px">
        如果多人合作进行原型设计，在整合的时候，可以选择导入页面，复制粘贴绝对不是好方法。
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="font-weight:bold">
      <span style="font-family:华文宋体">
       <span style="font-size:18px">
        <span style="color:#ff0000">
         总结
        </span>
        ：以上分享了我对原型设计的一些理解和使用Axure的些许经验，从原型设计上，我们也可以看出为了更高效，高质地开发软件，我们需要本着以人为本的思想，充分利用它和用户达成共识，进而开发出用户满意的产品。
       </span>
      </span>
     </span>
    </p>
