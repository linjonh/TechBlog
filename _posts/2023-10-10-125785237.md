    <h2 style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      一、游戏测试主要内容
     </span>
    </h2>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      功能测试，性能测试，压力测试，兼容测试，安全测试，接口测试，日志测试，弱网测试，GM工具测试，SDK测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#fe2c24;">
      <strong>
       功能测试
      </strong>
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      功能测试是游戏中最常见的模式，主要测试方法为黑盒测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      功能测试主要用来验证功能是否符合需要设计
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      功能测试主要考虑功能正确性，而不考虑游戏底层结构及代码错误
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      功能测试通常从界面着手开始测试，尽量模拟用户可能出现的操作
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#fe2c24;">
       客户端性能测试
      </span>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      客户端CPU使用率，客户端内存占用率，客户端网络流量使用情况，客户端耗电量，客户端帧率（FPS）iOS常用工具 xcode自带的instrument安卓常用工具Emma个和GT
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#fe2c24;">
      <strong>
       服务端压力测试
      </strong>
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      服务器CPU使用率，服务器内存占用率，系统吞吐量（TPS），事务响应时间，事务成功率
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      可用jmeter进行压力测试和性能测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#fe2c24;">
       兼容测试
      </span>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      机型适配测试，操作系统兼容测试，屏幕分辨率兼容测试，游戏版本兼容测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#fe2c24;">
      <strong>
       安全测试
      </strong>
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      内容修改测试，客户端加密测试，客户端反编译测试，网络安全测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#fe2c24;">
       接口测试
      </span>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      服务器各个接口数据测试，主要通过工具来实现
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      接口安全测试，重复发送请求，查看接口处理情况
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      日志测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      客户端日志
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      服务端日志
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#fe2c24;">
       弱网测试
      </span>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      不同的网络情况，游戏的运行情况，如edge，2g，3g，4g情况
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      不同丢包率情况下游戏的运行情况
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      通过工具设置网络代理来实现，常用的fiddler， network link conditioner
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#fe2c24;">
      <strong>
       GM工具测试
      </strong>
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      测试GM工具的功能实现，需要关注工具的设置是否在游戏中起作用
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      测试GM工具的数据读取，存储
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#fe2c24;">
      <strong>
       sdk测试
      </strong>
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      用户数据测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      充值，消费测试
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#404040;">
      与各个渠道对接测试
     </span>
    </p>
    <h2 style="margin-left:.0001pt;text-align:justify;">
     二、游戏团队
    </h2>
    <p>
     <strong>
      1.制作人：
     </strong>
    </p>
    <p>
     项目整体负责人
     <br/>
     负责游戏研发环节
     <br/>
     负责游戏运营环节
     <br/>
     负责项目事务管理
     <br/>
     负责项目人员管理
     <br/>
     <strong>
      2.策划：
     </strong>
    </p>
    <p>
     剧情：负责规划游戏中的各种剧情、故事、背景等。
     <br/>
     系统：设计游戏中各种系统的规则。
     <br/>
     数值：规划游戏中各种资源的产出、消耗等。
     <br/>
     关卡：设计游戏中各种关卡。
     <br/>
     <strong>
      3.程序：
     </strong>
    </p>
    <p>
     代码实现人员，负责把策划的设计及美术资源等通过编码实现成可玩的程序。
     <br/>
     前端程序:实现游戏客户端的展现与逻辑。
     <br/>
     后端程序：实现服务器端的逻辑、数据验证等。
     <br/>
     <strong>
      4.美工：
     </strong>
    </p>
    <p>
     制作游戏中的各类美术资源。
     <br/>
     场景、原画、UI、动画人员。
    </p>
    <h2>
     <strong>
      三、游戏测试基本流程
     </strong>
     <br/>
     1.功能会议
    </h2>
    <p>
     了解功能需求内容
     <br/>
     提出可能存在的风险点
     <br/>
     思考功能的测试重点和难点，如需要工具辅助，需提出开发需求。
     <br/>
     思考可以优化的地方，并提出讨论。
     <br/>
     <strong>
      2.测试用例书写
     </strong>
    </p>
    <p>
     根据需求书写测试用例
     <br/>
     关注功能逻辑实现
     <br/>
     考虑各种特殊情况，如边界值、网络中断、进程中断等。
     <br/>
     关注需求变更情况，需求经常发生变更，需要及时调整测试用例。
     <br/>
     <strong>
      3.冒烟测试
     </strong>
    </p>
    <p>
     详细测试之前的一个环节。
     <br/>
     快速发现比较明显的bug。
     <br/>
     快速确保主逻辑流程跑通。
     <br/>
     快速明确功能开展状态。
     <br/>
     <strong>
      4.详细测试
     </strong>
    </p>
    <p>
     细致的测试每个逻辑分支、资源、配置。
     <br/>
     尽量模拟玩家的每一种操作可能。
     <br/>
     测试异常情况、如断网、断电、事件中断、进程中断等情况。
     <br/>
     测试数据读取、存储、网络等内容。
     <br/>
     测试改功能对其他功能的影响（新功能对旧功能的影响）。
     <br/>
     <strong>
      5.回归测试
     </strong>
    </p>
    <p>
     测试已经被修复的内容。
     <br/>
     测试需求调整后的内容。
     <br/>
     再次详细测试各逻辑分支。
     <br/>
     <strong>
      6.CHECKLIST检查
     </strong>
    </p>
    <p>
     简要快速的检查功能的主要逻辑点。
     <br/>
     简要检查与该功能有关联的任何其他功能点。
    </p>
