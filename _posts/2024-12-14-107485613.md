    <h2>
     <a id="1_8">
     </a>
     1概述
    </h2>
    <blockquote>
     <p>
      互联网公司将分散于全球各地的数据中心中富余的可用资源提供给用户，这便是云计算的由来。
     </p>
    </blockquote>
    <p>
     <strong>
      云计算
     </strong>
     （Cloud Computing，CC）是基于互联网的计算模式，将计算过程从用户终端集中到云端。云计算是
     <strong>
      分布式计算
     </strong>
     的一种，即将计算分布在大量的分布式计算机上，其具体过程为通过网络“云”将巨大的数据计算处理程序分解成无数个小程序，然后，通过多部服务器组成的系统进行处理和分析这些小程序得到结果并返回给用户。
    </p>
    <blockquote>
     <p>
      应用云计算就好比从单台发电机模式转向电厂集中供电的模式，这意味着计算能力可以作为一种商品流通，如同电能一样，知识，云计算是通过互联网进行传输的。
     </p>
    </blockquote>
    <p>
     <strong>
      云计算平台
     </strong>
     （Cloud Computing Platform，CCP）又称为
     <strong>
      云平台
     </strong>
     ，云计算提供商使用云计算平台，向用户提供基于云的各种服务。云平台中的组件主要包括云服务器、云网站、云关系数据库、云缓存和云存储等。
     <br/>
     国内市场上较为流行的云计算平台有阿里云、腾讯云、百度云、华为云、微软云等。
    </p>
    <h3>
     <a id="11_17">
     </a>
     1.1云计算的服务模型
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a0c99f2e295170416442f387dec050f4.png#pic_center"/>
    </p>
    <p>
     NIST（National Institute of Standards and Technology，美国国家标准与技术研究院）给出的云计算服务模型包含以下基本特征：
    </p>
    <ol>
     <li>
      <strong>
       按需自助服务
      </strong>
      <br/>
      云是一个庞大的资源池，云可以像水电一样购买，用户可以根据需求使用云计算资源，在此过程中不需要与云服务提供商进行交互。
     </li>
     <li>
      <strong>
       宽带网络访问
      </strong>
      <br/>
      云服务具有宽带网络访问的能力以及高带宽的通信链路，以满足海量用户的接入需求。
     </li>
     <li>
      <strong>
       资源池化
      </strong>
      <br/>
      云计算服务提供商需要为多个用户同时提供服务，这就需要拥有资源池。该资源池包含大规模物理资源和虚拟资源，同时可以灵活动态地供用户使用。
     </li>
     <li>
      <strong>
       快速弹性伸缩
      </strong>
      <br/>
      指为了满足云计算的需求，云计算平台所具备的对所分配资源进行快速增加或缩减的能力。
     </li>
     <li>
      <strong>
       服务可度量
      </strong>
      <br/>
      云计算平台可以对其提供的各类资源和服务类型进行计量，对资源的使用情况进行监控、控制和上报，让服务提供者和使用者及时了解服务使用情况，以实现自动控制和优化资源使用。
     </li>
    </ol>
    <h2>
     <a id="2_31">
     </a>
     2云计算的部署模式
    </h2>
    <p>
     典型的云计算服务部署模式包括公有云、私有云和混合云。
    </p>
    <ol>
     <li>
      <strong>
       公有云
      </strong>
      <br/>
      公有云由云服务提供商拥有、管理和运营。公有云的基础设施由云服务提供商部署，向公众开放使用。
     </li>
     <li>
      <strong>
       私有云
      </strong>
      <br/>
      私有云由特定的组织机构、企业或第三方拥有，并负责云服务的管理和运行。
      <br/>
      私有云提供的服务具有针对性，组织机构、企业或第三方对其搭建的云计算平台具有自主权。
     </li>
    </ol>
    <blockquote>
     <p>
      传统上，私有云是在本地运行的服务器。但是现在一般所使用的的私有云是租用数据中心的服务器，从而构建私有云。
     </p>
    </blockquote>
    <ol start="3">
     <li>
      <strong>
       混合云
      </strong>
      <br/>
      混合云是一种计算环境，由两个或以上的云（私有云或公有云）组成，不同的云服务之间独立设置，具有数据和应用程序的可移植性，使用一定的技术活标准化机制进行
      <strong>
       融合
      </strong>
      并提供服务。
      <br/>
      组织机构、企业等在使用混合云的部署模式时，可同时使用公有云和私有云；这些机构对私有云具有自主权，但对公有云没有自主权；机构可以在公有云提供的通用服务的基础上，利用拥有的私有云，面向自身的需求开发混合云。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/35aa98e511e722ab974f62411558a1b7.png"/>
     </li>
    </ol>
    <p>
     对于企业而言，出于安全考虑，企业更愿意将数据存放在私有云中，但是同时希望可以使用公有云的强大的计算和存储等资源，因此企业更倾向与使用混合云的模式。 使用混合云的部署模式企业，可以把常规数据和业务部署在公有云上，核心业务相关数据部署在私有云上，由企业维护。
    </p>
    <h2>
     <a id="3_46">
     </a>
     3云计算的体系结构
    </h2>
    <p>
     下面从云计算的层次架构、组成架构和技术架构三个角度分别介绍云计算的体系结构。
    </p>
    <h3>
     <a id="31_48">
     </a>
     3.1云计算的层次架构
    </h3>
    <p>
     根据云计算所提供的
     <strong>
      服务层次
     </strong>
     ，云计算可划分为以下三层：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        层名
       </th>
       <th>
        功能
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        基础设施层
       </td>
       <td>
        基础设施层以云计算资源为中心，通过虚拟化技术对资源进行抽象，向用户提供基础设施服务，实现流程自动化和资源管理优化
       </td>
      </tr>
      <tr>
       <td>
        平台层
       </td>
       <td>
        平台层以中间件和平台软件为中心，包含各类软件资源
       </td>
      </tr>
      <tr>
       <td>
        应用层
       </td>
       <td>
        应用层构建于基础设施层提供的硬件资源和平台层提供的软件环境之上，通过为网络为用户提供服务，是云计算应用软件的集合
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="32_56">
     </a>
     3.2云计算平台的组成架构
    </h3>
    <blockquote>
     <p>
      云计算平台连接了大量并发的网络计算和服务，利用虚拟化技术形成虚拟化资源池，将硬件资源进行虚拟化管理和调度，把存储于各种设备的资源集中起来协同工作，提供超强的计算和储存能力。
     </p>
    </blockquote>
    <p>
     云计算平台具有多种组成架构，下面介绍一种常用的云计算平台：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce393def0e3a534d55c47a37aa1f7255.png#pic_center"/>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        名称
       </th>
       <th>
        功能
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        云客户端
       </td>
       <td>
        云客户端提供用户向云平台请求服务的交互界面，是用户使用云平台的入口
       </td>
      </tr>
      <tr>
       <td>
        服务目录
       </td>
       <td>
        服务目录可以在用户端界面为用户展示所选择或定制的服务
       </td>
      </tr>
      <tr>
       <td>
        管理系统与部署工具
       </td>
       <td>

      </tr>

       <td>
        资源监控
       </td>
       <td>
        监控和计量云计算服务系统的资源使用情况，进行资源的优化配置
       </td>
      </tr>
      <tr>
       <td>
        服务器集群
       </td>
       <td>
        服务器集群由虚拟或物理服务器构成，负责相关用户服务
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="33_70">
     </a>
     3.3云计算的技术架构
    </h3>
    <p>
     从
     <strong>
      技术角度
     </strong>
     考虑，可分为以下4层：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        层名
       </th>
       <th>
        功能
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        物理资源层
       </td>
       <td>
        云计算服务使用的各种物理资源，如计算机、存储器、网络设施、数据库和软件等
       </td>
      </tr>
      <tr>
       <td>
        资源池层
       </td>
       <td>
        资源池层将大量相同类型的资源抽象为在虚拟资源池；实现物理资源的集成和管理工作，实现资源的合理有效调度
       </td>
      </tr>
      <tr>
       <td>
        管理中间件层
       </td>
       <td>
        包括资源管理、任务管理、用户管理和安全管理
       </td>
      </tr>
      <tr>
       <td>
        SOA构建层
       </td>
       <td>
        该层的功能通常依靠外部设施提供，其功能为将云计算能力封装成标准的WEB服务，并纳入到SOA体系进行管理和使用
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      SOA（Service-Oriented Architecture,面向服务的架构）。
     </p>
    </blockquote>
    <h2>
     <a id="4_81">
     </a>
     4云平台服务模式
    </h2>
    <p>
     云平台包含三种服务模式，分别为基础架构即服务、平台即服务和软件及服务：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        位置
       </th>
       <th>
        层名
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        上
       </td>
       <td>
        基础架构即服务（Infrastructure-as-a-Service,Iaas）
       </td>
      </tr>
      <tr>
       <td>
        中
       </td>
       <td>
        平台即服务（Platform-as-a-Service,PaaS）
       </td>
      </tr>
      <tr>
       <td>
        下
       </td>
       <td>
        软件即服务（Software-as-a-Service,SaaS）
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     （注意三者是有先后顺序的，而非并列关系。）
    </p>
    <ol>
     <li>
      <p>
       <strong>
        IaaS
       </strong>
       <br/>
       IaaS模式的云服务交付给用户的是基本的基础设施资源。基础设施向用户提供虚拟化的计算资源、存储资源、网络资源和安全防护，这些资源能够根据用户的需求动态分配。用户无需管理或控制底层的云计算基础设施，就可以控制操作系统、存储、部署的应用，同时对某些网络组件具有一定的控制能力。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/90d8af07b9d5dbd7e72c0e74490d50a0.png"/>
      </p>
     </li>
     <li>
      <p>
       <strong>
        PaaS
       </strong>
       <br/>
       用户通过使用PaaS服务提供商提供的开发平台，进行程序的快速配置和数据库的使用；用于部署和运行应用程序所使用的资源由PaaS服务提供商负责搭建并管理，用户无需控制或管理平台底层基础架构。
      </p>
     </li>
    </ol>
    <blockquote>
     <p>
      所谓平台，是指支撑使用特定开发工具开发的、应用能够在其上有效运行的软件支撑系统平台。
     </p>
    </blockquote>
    <p>
     支撑该服务的技术体系主要是分布式系统。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a2e8730489d21b94d40a8e6586adb39b.png"/>
    </p>
    <blockquote>
     <p>
      PaaS和IaaS的区别
      <br/>
      IaaS在同一基础设施上同时为大量用户提供其专属的“硬件”资源；
      <br/>
      PaaS在同一基础设施上同时为大量用户提供其专属的应用运行平台；
     </p>
    </blockquote>
    <ol start="3">
     <li>
      <strong>
       SaaS
      </strong>
      <br/>
      SaaS是一种以互联网为载体，以浏览器为交互方式，把服务器端的程序软件传给远程用户来提供软件服务的应用模式。
      <br/>
      SaaS模式的云服务是云服务提供商提供的运行在云基础设施上的应用程序，用户可以通过不同种类的客户端设备访问这些应用程序。SaaS服务提供商将服务软件放在自己的服务器中，通过WEB浏览器或程序界面，向用户提供基于云的应用服务。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9989e6321d8f8cbfe5f34a123bd92655.png">
       <br/>
       <strong>
        总结
       </strong>
      </img>
     </li>
     <li>
      IaaS向用户提供基础设施资源，用户可以直接在该层构建自己的平台和应用；
     </li>
     <li>
      PaaS向用户提供开发平台资源，用户可通过网络将自己创建的或从别处获取的应用软件部署到服务提供商提供的环境运行；
     </li>
     <li>
      SaaS向用户提供搭建信息化所需要的所有网络基础设施及软硬件运作平台，负责所有前期的实施、后期的维护等一系列工作；客户只需要根据自己的需要，向SaaS提供商租赁软件服务。
     </li>
    </ol>
