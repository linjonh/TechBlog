    <h2>
     <a id="01LED_4">
     </a>
     01、LED操作
    </h2>
    <p>
     国际惯例，学习单片机先学点灯。这里有所不同的是在过程上，我们用到了STM32 CubeMX图形可视化编程。在备赛之前，博主按照火哥的视频，使用的是固件库编程。编程逻辑是根据原理图，开启相应时钟→初始化相关GPIO→完成相关功能程序。而Cube则是将前两步用图形化的方式代替，简化了整个过程提高效率。本质上都是一样的。
     <br/>
     首先，打开Cube新建一个项目。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/92c0a68db6f99171f70b1630d7f3db2c.png">
      <br/>
      在索引栏搜索所学芯片STM32G431RB,选择与实物封装相同的第二个，点击Start Project。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/05c104354887c39ee7b4efa4a9803fcd.png">
       <br/>
       开启时钟。HSE为外部高速时钟，LSE为外部低速时钟（E即外部，I反之）。高速时钟主要用于系统内核和总线上的外设时钟。低速时钟主要用于独立看门狗IWDG、实时时钟RTC。BYPASS Clock Source 是旁路时钟源（即选择有源晶振），Crystal/Ceramic Resonator 是选择无源晶振，这里可以参考
       <a href="https://blog.csdn.net/m0_37845735/article/details/124540221?ops_request_misc=%257B%2522request%255Fid%2522%253A%2522167619108416800188576624%2522%252C%2522scm%2522%253A%252220140713.130102334.pc%255Fall.%2522%257D&amp;request_id=167619108416800188576624&amp;biz_id=0&amp;utm_medium=distribute.pc_search_result.none-task-blog-2~all~first_rank_ecpm_v1~rank_v31_ecpm-2-124540221-null-null.142%5Ev73%5Ewechat_v2,201%5Ev4%5Eadd_ask,239%5Ev1%5Einsert_chatgpt&amp;utm_term=cubemx%E9%AB%98%E9%80%9F%E6%97%B6%E9%92%9FBYPASS%20clock%20source&amp;spm=1018.2226.3001.4187">
        hurryddd的博客
       </a>
       以及
       <a href="https://blog.csdn.net/ben_xiao_hai_123/article/details/127545740?ops_request_misc=%257B%2522request%255Fid%2522%253A%2522167619108416800188576624%2522%252C%2522scm%2522%253A%252220140713.130102334.pc%255Fall.%2522%257D&amp;request_id=167619108416800188576624&amp;biz_id=0&amp;utm_medium=distribute.pc_search_result.none-task-blog-2~all~first_rank_ecpm_v1~rank_v31_ecpm-3-127545740-null-null.142%5Ev73%5Ewechat_v2,201%5Ev4%5Eadd_ask,239%5Ev1%5Einsert_chatgpt&amp;utm_term=cubemx%E9%AB%98%E9%80%9F%E6%97%B6%E9%92%9FBYPASS%20clock%20source&amp;spm=1018.2226.3001.4187">
        kay880的博客
       </a>
       。
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5538a408b244890b27b6c29b5f9db144.png">
      <br/>
      配置时钟树。根据产品手册，外部晶振为24MHz。图中的标号顺序是从左到右进行选择，但是配置时钟树的正常逻辑应参考
      <a href="https://blog.csdn.net/anbaixiu/article/details/80480299?ops_request_misc=&amp;request_id=&amp;biz_id=102&amp;utm_term=stm32%E6%97%B6%E9%92%9F%E9%85%8D%E7%BD%AE&amp;utm_medium=distribute.pc_search_result.none-task-blog-2~all~sobaiduweb~default-5-80480299.nonecase&amp;spm=1018.2226.3001.4187">
       anbaixiu的博客
      </a>
      ，先设置后面的时钟频率大小（最大可到170，80设置为跟教程相同），再打开PLL。最后按下回车，Cube会自动完成时钟树配置。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/66a3fcb2bfbb5e1308f0bb8fe7ffa40b.png">
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/68e76af1bfb9a63a57bf530288d0bbc8.png">
        <br/>
        命名工程文件,保存路径，生成模板代码。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a33b650a015c76d0ad76549c668d6ee4.png">
         <br/>
         开启时钟OK！
         <br/>
         接下来初始化相应的GPIO口。打开刚刚创建的.ioc文件，根据芯片手册，led1-8分别连接的PC8-15以及锁存对应PD2。
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f67122c60a21cba539237e67cc376e17.png">
          <br/>
          在Cube中将这几个GPIO口设置为GPIO_Output,并将PC这8个IO口设置为高电平防止一上电就灯亮，PD2设置为低电平。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5768cf0a79bac5c83b83e90908aa4a18.png">
           <br/>
           <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c44bc6cab7410af491a881e174d954a3.png"/>
           <br/>
           点击GENERATE CODE后，第二步初始化GPIO口完成！
          </img>
         </img>
        </img>
       </img>
      </img>
     </img>
    </p>
    <p>
     在生成工程文件的路径下新建一个文件夹（这里命名为bsp）用于保存实相应功能的代码。在文件夹中分别创建一个.c和.h文件。打开工程，将文件夹添加进工程中。在led.c中包含.h文件，点击魔术棒将.h文件路径包含，然后编译。
     <br/>
     现在，我们假设想要实现的功能是点亮led8。
     <br/>
     led.c中代码包含：
    </p>
    <pre><code>#include "led.h"		//包含头文件

void Dis_LED(uchar Num_Led) //定义点灯函数，传入的参数是选中哪些灯
{
HAL_GPIO_WritePin(GPIOC,GPIO_PIN_All,GPIO_PIN_SET); //先将所有灯熄灭
HAL_GPIO_WritePin(GPIOC,Num_Led&lt;&lt;8,GPIO_PIN_RESET); //因为只有 PC8-15 高 8 位有效，所以将传入参数左移八位
HAL_GPIO_WritePin(GPIOD,GPIO_PIN_2,GPIO_PIN_SET); //高电平锁存
HAL_GPIO_WritePin(GPIOD,GPIO_PIN_2,GPIO_PIN_RESET);
}
</code></pre>
<p>
这里要注意，uchar 类型需要定义，在 main.h 中定义：
</p>
<pre><code>/_ USER CODE BEGIN Includes _/
#define uchar unsigned char
#define uint unsigned int

/_ USER CODE END Includes _/
</code></pre>
<p>
注意，用 cube 写的模板，在加代码时必须在/_ USER CODE BEGIN Includes
<em>
/和/
</em>
USER CODE END Includes _/的中间，这就是 cube 的格式，否则如果将.ioc 文件进行修改再生成代码后就会将我们加的代码覆盖。
<br/>
led.h:
</p>
<pre><code>#ifndef \_LED_H
#define \_LED_H

#include "main.h"
void Dis_LED(uchar Num_Led);

#endif
</code></pre>
<p>
main.c 中包含过 led.h 后，加入：
</p>
<pre><code> /_ USER CODE BEGIN 2 _/
Dis_LED(0x00);
/_ USER CODE END 2 _/

/_ Infinite loop _/
/_ USER CODE BEGIN WHILE _/
while (1)
{
/_ USER CODE END WHILE _/

    /* USER CODE BEGIN 3 */
    	Dis_LED(0x80);
    	HAL_Delay(500);
    	Dis_LED(0x00);
    	HAL_Delay(500);



}
/_ USER CODE END 3 _/
</code></pre>
<p>
编译后代码没有报错，点击魔术棒对下载和 debug 进行设置，在 settings 中勾选 reset and run，以便于下载后不用自己按 reset。
