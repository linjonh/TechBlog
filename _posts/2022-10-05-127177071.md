    <h2>
     <a id="_7">
     </a>
     前言
    </h2>
    <p>
     嵌入式开发领域的基本概念以及嵌入式开发流程中所需的工具。
    </p>
    <hr/>
    <h2>
     <a id="_12">
     </a>
     一、电路板
    </h2>
    <h3>
     <a id="1EVBEvaluation_board_13">
     </a>
     1.开发板：EVB（Evaluation board）
    </h3>
    <p>
     最小系统+大量的外设，如：AD/DA，数码管，液晶，蜂鸣器，步进电机，时钟电路，温度测量等。
    </p>
    <p>
     把芯片所有的引脚和外设都引出来，板子不会太小，可以很轻松搭建电路。
     <br/>
     <img alt="开发板" src="https://i-blog.csdnimg.cn/blog_migrate/1b1ca8d5c9c37be6dabe0608917f6dd6.png#pic_center"/>
    </p>
    <h3>
     <a id="2_19">
     </a>
     2.最小系统板
    </h3>
    <p>
     EVB的高度凝练，提供单片机工作所需的最小外围电路。
    </p>
    <p>
     一般只包括电源，复位电路，振荡电路，以及将IO口全部引出。可灵活设计外围电路。
     <br/>
     <img alt="最小系统板" height="300" src="https://i-blog.csdnimg.cn/blog_migrate/df10af175754749b1e24154cd925d47a.jpeg#pic_center" width="300"/>
    </p>
    <h2>
     <a id="_24">
     </a>
     二、开发软件
    </h2>
    <h3>
     <a id="1IDETool_chain_25">
     </a>
     1.IDE/Tool chain
    </h3>
    <p>
     IDE:集成开发环境。例如：Visual C++，KEIL，IAR。
    </p>
    <p>
     微处理器的内核不同，有不同的指令集和寄存器组，所以需要不同的开发工具。
     <br/>
     一个完整的工具链构成包括：
    </p>
    <ol>
     <li>
      Assembler汇编器：把汇编代码一一对应具有映射关系的变成机器码
     </li>
     <li>
      Compiler编译器：解析高级语言：把它进行语素和逻辑上的拆解，最后翻译成中间语言或者汇编语言。
     </li>
     <li>
      Linker链接器：把编译的绝对地址定义的代码段，根据内存地址映射，最终形成可执行的，可相互调用的，完整的程序。
     </li>
     <li>
      Disassemble反汇编器：将机器码反向成汇编代码。
     </li>
    </ol>
    <h3>
     <a id="2Cross_Compiler_35">
     </a>
     2.交叉编译(Cross Compiler)
    </h3>
    <p>
     交叉编译器是指一个在某个系统平台下可以产生另一个系统平台可执行文件的编译器。在对象系统平台难以或不容易编译时非常有用。
    </p>
    <p>
     绝大多数的嵌入式开发都是使用计算机的平台进行编程，最终得到在嵌入式平台上可以使用的这样一个程序。
    </p>
    <h2>
     <a id="_39">
     </a>
     三、开发工具
    </h2>
    <h3>
     <a id="1SimulatorEmulator_40">
     </a>
     1.模拟器(Simulator/Emulator)
    </h3>
    <p>
     计算机虚拟出一款硬件平台。
    </p>
    <p>
     嵌入式平台的指令集与计算机不同，我们在做纯软件和算法验证的时候可以借助模拟器。
    </p>
    <p>
     可以通过编程语言写出一个虚拟的CPU构架，完成所有指令的功能解析，甚至完成所有对于存储和外设的调用，让程序误以为自己在一个硬件的平台上跑。
    </p>
    <h3>
     <a id="2Debugger_46">
     </a>
     2.调试器(Debugger)
    </h3>
    <p>
     使用几根有限的通讯线非常高效的与CPU以及微处理器内置的调试模块接口进行通讯，然后发送和接收调试所用的信息。
     <br/>
     <img alt="STlink" height="280" src="https://i-blog.csdnimg.cn/blog_migrate/5771c3f8db19e7527d7ec95f0eadaf94.jpeg#pic_center" width="300"/>
    </p>
    <p>
     有BDM，JTAG，SWD等通讯协议。绝大多数ARM Cortex处理器都使用的是SWD的调试接口。
    </p>
    <p>
     1.USBDM：开源，使用BDM或SWD的协议，广泛支持一系列MCU。
     <br/>
     2.OpenSDA
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bf93787c183c3120a29640f8ba804715.png#pic_center"/>
    </p>
    <h3>
     <a id="3ProgrammerBurner_55">
     </a>
     3.编程，烧写器(Programmer/Burner)
    </h3>
    <p>
     可以把编写好的一个程序放在烧写器里，然后批量更换芯片烧写程序。
    </p>
    <h2>
     <a id="_58">
     </a>
     四、嵌入式开发的进阶知识
    </h2>
    <h3>
     <a id="1_59">
     </a>
     1.微控制器的启动过程
    </h3>
    <p>
     一旦上电复位，在之前烧写的程序中，程序的入口地址会填写在PC指针处。
    </p>
    <p>
     CPU会自动把堆栈初始地址和PC指针地址加载到CPU里，所以CPU从程序的第一条指令逐条向下执行，执行过程中会访问存储器，在内存中读写定义的变量数组等，在函数调用或参数传递时使用堆栈的内存，控制外设时即访问外设寄存器。
     <br/>
     <img alt="在这里插入图片描述" height="450" src="https://i-blog.csdnimg.cn/blog_migrate/525f1f860b677226684bbbb4a29ba5e0.png#pic_center" width="500"/>
    </p>
    <h3>
     <a id="2_66">
     </a>
     2.微控制器软件开发的流程
    </h3>
    <p>
     写C程序
     <br/>
     → 用工具链编译得到目标代码
     <br/>
     → 交叉编译得到所用的CPU上的程序
     <br/>
     → 通过仿真器验证运行
     <br/>
     → 通过调试器可以把程序以某种通讯协议让芯片执行。
    </p>
    <p>
     C程序
     <br/>
     →程序放到存储器里，被CPU读取
     <br/>
     →控制相应的外设
    </p>
    <h3>
     <a id="3_78">
     </a>
     3.微控制器开发过程中的各种文件
    </h3>
    <p>
     链接配置文件：告诉微处理器哪个地址段是flash可以放程序，哪个地址段是RAM可以放变量，哪个地址段是RAM的底部可以是堆栈。
    </p>
    <p>
     映射文件：详细记录了在链接的时候声明的每个变量，每个函数体所使用的地址空间。
    </p>
