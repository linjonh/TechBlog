    <h2>
     <a id="101_3">
     </a>
     10.1测试用例执行
    </h2>
    <p>
     根据已有的测试用例，按照里面的步骤一步一步的执行，查看预期结果与实际结果是否一致。
    </p>
    <p>
     <strong>
      测试执行过程注意事项
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       1.搭建测试环境事项
      </strong>
      <br/>
      测试用例执行过程前，成功搭建测试环境是第一步。
      <br/>
      一般来说，软件产品提交测试后，开发人员应该提交一份被测试软件产品的详细安装指导书。
     </li>
     <li>
      <strong>
       2.注意前提条件和特殊说明
      </strong>
      <br/>
      有些测试软件是有顺序性的，那么它的测试用例就会有一些执行前提或特殊说明。
      <br/>
      比如要测试某个软件的登陆功能，那么测试前必须创建用户，并为用户分配一定的权限等。
     </li>
     <li>
      <strong>
       3.测试用例要全部执行
      </strong>
      <br/>
      每个功能点我们都要测试一遍，才能保证待测试软件能正确满足用户需求。
     </li>
     <li>
      <strong>
       4.不要忽视任何偶然现象
      </strong>
      <br/>
      我们在执行某条用例时，软件会出错，但是当再次执行时这个错误就不再重现。遇到这种情况时，要多测试几次，尽可能准确的找出问题的原因。
     </li>
     <li>
      <strong>
       5.加强测试过程记录
      </strong>
     </li>
     <li>
      <strong>
       6.详细预期与实际的不一致
      </strong>
     </li>
     <li>
      <strong>
       7.提交缺陷时与开发的关系处理
      </strong>
      <br/>
      测试执行过程中，当你提交了问题报告单，可能被开发人员无情驳回，拒绝修改。这时候，只能对开发人员晓之以理，做到有理、有据，有说服力。
     </li>
     <li>
      <strong>
       8.及时更新测试用例
      </strong>
     </li>
    </ul>
