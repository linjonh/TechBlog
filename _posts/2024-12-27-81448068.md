    <p>
     想要了解常见的Java相关面试问题的更多信息？看完这篇文章可以了解有关投资银行最佳面试问题的更多信息。
     <br/>
     多线程和并发问题是任何Java面试的重要组成部分。如果你要去投资银行面试，例如巴克莱，花旗银行，摩根士丹利为股票前端Java开发人员的职位，可以期待很多多线程的面试问题。多线程和并发是投资银行访谈的热门话题，尤其是电子交易开发工作，他们在许多棘手的Java线程面试问题上难住候选人。他们希望确保候选人对Java中的多线程和并发编程有扎实的了解，因为他们中大多数业务都和性能相关，这为他们提供了竞争优势。
    </p>
    <p>
     例如，用于DMA交易的高容量和低延迟电子交易系统通常是并发的。大多数情况下，他们专注于微秒延迟，这就是为什么拥有如何有效地最小化延迟和提高吞吐量知识是如此重要。
    </p>
    <p>
     这些是我最喜欢的关于Java的线程面试问题。我没有提供这些线程访谈问题的答案，但我会尽可能给你一个提示。我会更新帖子就详细的答案，就像我最近在Java中发布的10个Singleton面试问题一样。
    </p>
    <h3 id="15-java-thread-interview-questions-and-answers">
     15 Java Thread Interview Questions and answers
    </h3>
    <p>
     无论如何，这里是一些常见的Java多线程和并发问题列表，这些问题曾是Java开发人员面试投资银行时遇到的问题，例如： 巴克莱，摩根士丹利，花旗银行等
    </p>
    <ol>
     <li>
      <p>
       假设你有三个线程T1,T2,T3。你如何能保证线程T2在线程T1后运行，T3在T2后运行
      </p>
      <p>
       这个线程面试问题大多是在第一轮或电话筛选轮次中会被问到的，这个多线程问题的目的是检查候选人对“join”方法概念是否熟悉。这个多线程问题的答案很简单 - 可以通过使用Thread类的join方法来实现。
      </p>
     </li>
     <li>
      <p>
       新的Lock接口较于Java中的同步块（synchronized block）有什么优势？您需要实现一个允许多读取的高性能缓存，但是如何实现单独写以保持完整性？
      </p>
      <p>
       lock接口在多线程和并发编程的主要优势是它提供了两个独立的读写锁，使你能够编写高性能数据结构，如
       <a href="http://javarevisited.posterous.com/difference-between-concurrenthashmap-and-coll" rel="nofollow">
        ConcurrentHashMap
       </a>
       和
       <a href="https://javarevisited.blogspot.com/2012/02/what-is-blocking-methods-in-java-and.html" rel="nofollow">
        条件阻塞（conditional blocking)
       </a>
       。
      </p>
      <p>
       这个Java线程面试问题越来越受欢迎，越来越多的后续提问问题基于面试者的回答。
      </p>
      <p>
       我强烈建议在任何Java多线程面试之前阅读锁，因为，现在，它大量用于为客户端上的电子交易系统构建缓存并交换连接空间。
      </p>
     </li>
     <li>
      <p>
       Java多线程中sleep和wait方法的区别
       <br/>
       让我们看一下Java中另一个经常被问到的线程面试问题。 这个问题经常会出现在电话面试.这两者的主要区别在于wait可以释放锁或监视器，而sleep在等待时不会释放任何锁或监视器。wait用于线程间通信，因为sleep用于在执行时引入暂停。有关详细信息，请参阅我的
       <a href="https://javarevisited.blogspot.com/2011/12/difference-between-wait-sleep-yield.html" rel="nofollow">
        wait vs sleep in Java
       </a>
       这篇帖子。
      </p>
      <p>
       <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/abeb4f24eebdb64d756bb7deb1c10467.gif" title=""/>
      </p>
     </li>
     <li>
      <p>
       如何在Java中实现阻塞队列？
       <br/>
       这是一个相对困难的Java多线程面试问题，有很多用意。它可以确定候选人在实际工作中是否使用过线程编写Java代码。它可以看到候选人是否充分理解并发场景，你可以根据他的代码提出很多后续问题。如果他使用
       <a href="https://javarevisited.blogspot.com/2011/05/wait-notify-and-notifyall-in-java.html" rel="nofollow">
        wait（）和notify（）
       </a>
       方法来实现阻塞队列，一旦候选者成功写出它，你可以让他再次使用新的Java 5并发类等来编写它。
      </p>
     </li>
     <li>
      <p>
       在Java中如何解决生产者消费者问题? (
       <a href="http://www.java67.com/2015/12/producer-consumer-solution-using-blocking-queue-java.html" rel="nofollow">
        solution
       </a>
       )
       <br/>
       与上面的线程问题类似，这个问题本质上更经典，但有时候面试官会进一步问问题，比如“你如何解决Java中的生产者消费者问题？”它可以通过多种方式解决。我已经分享了一种使用Java中的BlockingQueue来解决生产者 - 消费者问题的方法，所以要准备好接受一些惊喜。有时候，他们甚至会要求你实现一个解决哲学家进餐问题的解决方案。
      </p>
     </li>
     <li>
      <p>
       编写一个会导致死锁的程序。你将如何解决Java中的死锁问题？
       <br/>
       这是我最感兴趣的java面试问题，因为，尽管在编写多线程并发程序时死锁很常见，但许多候选人都无法编写无死锁代码，而且他们很挣扎。
       <br/>
       只要询问他们是否有N个资源和N个线程来完成操作;那么，你需要所有的资源。
       <br/>
       用N来替换最简单的情况和更高的数字这两种情况，以使问题更加令人生畏。有关死锁的更多信息，请参见
       <a href="https://javarevisited.blogspot.com/2018/08/how-to-avoid-deadlock-in-java-threads.html" rel="nofollow">
        How to avoid deadlock in Java
       </a>
       。
       <br/>
       <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/abc471b6da7f90b06dcc55ce45dd9561.png" title=""/>
      </p>
     </li>
     <li>
      <p>
       什么是原子操作？什么是Java中的原子操作？
      </p>
      <p>
       这是一个简单的Java线程访谈问题。另一个后续问题是：你需要同步原子操作吗？你可以阅读更多
       <a href="https://javarevisited.blogspot.com/2011/04/synchronization-in-java-synchronized.html" rel="nofollow">
        Java synchronization
       </a>
       .
      </p>
     </li>
     <li>
      <p>
       什么是Java中的volatile关键字？你如何使用它？它与Java中的synchronized方法有什么不同？
       <br/>
       在对Java 5和Java内存模型进行更改之后，基于
       <a href="https://javarevisited.blogspot.com/2011/04/synchronization-in-java-synchronized.html" rel="nofollow">
        Java中的volatile关键字
       </a>
       的线程问题变得越来越流行。好好准备一下volatile变量如何确保并发环境中的可见性，排序和一致性。
      </p>
     </li>
     <li>
      什么是竞态条件（race condition）？您如何找到并解决竞态条件？
      <br/>
      Java中的另一个多线程问题主要出现在高级访谈中。大多数面试官询问您最近遇到的种族情况，如何解决，有时他们会编写示例代码并要求您检测竞态条件。有关更多信息，请参阅我的帖子-
      <a href="https://javarevisited.blogspot.com/2012/02/what-is-race-condition-in.html" rel="nofollow">
       race condition in Java
      </a>
      。在我看来，这是最好的Java线程面试问题之一，可以真正测试候选人在解决竞争条件或编写没有数据竞争或任何其他竞争条件的代码方面的经验。关于并发主题的最好的书是“Concurrency practices in Java(java并发实战)”。
     </li>
     <li>
      <p>
       你将如何在Java中进行线程转储？你将如何分析线程转储？
      </p>
      <p>
       在UNIX中，您可以使用kill -3，然后线程转储将在Windows上打印日志，您可以使用“CTRL + Break”。虽然这是一个相当简单的线程访谈问题，如果他们问你如何分析它会变得棘手。线程转储也可用于分析死锁情况。
      </p>
     </li>
     <li>
      我们调用start（）方法时为什么要在start()方法里面调用run（）方法， 为什么我们不直接调用run（）方法？
      <br/>
      这是另一个经典的线程问题。最初，当我开始编写线程时，我有一些疑问。如今，我主要通过电话采访或中期和初级Java面试的第一轮面试问题进行询问。
      <br/>
      以下就是这个问题的答案。当你调用start（）方法时，它会创建一个新线程并执行run（）中声明的代码，同时直接调用run（）方法。这不会创建任何新线程并在同一个调用线程上执行代码。关于更多信息，可以阅读这篇文章
      <a href="https://javarevisited.blogspot.com/2012/03/difference-between-start-and-run-method.html" rel="nofollow">
       Difference Between Start and Run Method in Thread
      </a>
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b903021ad049744a484bc2e260eb13fc.png" title=""/>
     </li>
     <li>
      java中如何唤醒阻塞线程
      <br/>
      这是一个棘手的线程问题。阻塞可能由很多方面导致 - 如果线程在IO上被阻塞，那么，我认为没有办法打断线程。如果有的话，请告诉我。另一方面，如果由于调用wait（），sleep（）或join（）方法的结果而阻塞线程，则可以中断线程，并通过抛出InterruptedException来唤醒它。有关处理被阻塞线程的更多信息，请参阅我的帖子-
      <a href="https://javarevisited.blogspot.com/2012/02/what-is-blocking-methods-in-java-and.html" rel="nofollow">
       How to Deal With Blocking Methods in Java
      </a>
      。
     </li>
     <li>
      <p>
       java中CyclicBarriar 和CountdownLatch区别（
       <a href="http://www.java67.com/2012/08/difference-between-countdownlatch-and-cyclicbarrier-java.html" rel="nofollow">
        答案
       </a>
       ）
      </p>
      <p>
       新的Java线程面试问题主要是检查您对JDK 5并发包的熟悉程度。他们的一个区别是，一旦barrier被破坏，您可以重复使用CyclicBarrier，但不能重复使用CountDownLatch。如果您想了解更多信息，请查看Udemy的课程-
       <a href="https://www.udemy.com/multithreading-and-parallel-computing-in-java/?siteID=JVFxdTr9V80-Mdzj7LWArXqHhxmACZ6znw&amp;LSNPUBID=JVFxdTr9V80" rel="nofollow">
        Multithreading and Parallel Computing in Java
       </a>
       。
      </p>
     </li>
     <li>
      什么是不可变对象？它如何帮助编写并发应用程序？
      <br/>
      虽然这个面试问题与线程没有直接关系，但它间接地帮助了很多。如果他们要求你写一个不可变的类或者问你
      <a href="https://javarevisited.blogspot.com/2010/10/why-string-is-immutable-or-final-in-java.html" rel="nofollow">
       为什么String在Java中是不可变的
      </a>
      ，那么这个面试问题会变得更加棘手。
     </li>
     <li>
      <p>
       您在多线程环境中遇到了哪些常见问题？你是怎么解决的？
      </p>
      <p>
       内存干扰，竞态条件，死锁，活锁和饥饿是多线程和并发编程带来的一些问题的一个例子。这些问题是无止境的;如果你弄错了，他们将难以检测和调试。
      </p>
      <p>
       这主要是关于Java的基于经验的面试问题。您可以看到Heinz Kabutz撰写的
       <a href="https://javaspecialists.teachable.com/p/concurrency-in-practice-bundle/?affcode=92815_johrd7r8" rel="nofollow">
        Java Concurrency in Practice Course
       </a>
       ，了解实际高性能多线程应用程序中遇到的一些实际问题。
      </p>
     </li>
    </ol>
