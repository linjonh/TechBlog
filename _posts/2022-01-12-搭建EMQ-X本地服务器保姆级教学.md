    <h2>
     <a id="EMQ_X_0">
     </a>
     搭建EMQ X本地服务器
    </h2>
    <p>
     前言：emq x官方给出的意见是建议在Linux环境中安装，本人一开始也是在Windows系统中安装的，但是在安装的过程中出现了很多意料之外的问题，且查询过众多资料仍然无法解决，然后选择了在虚拟机中建一个Linux系统，再从Linux环境中搭建emq服务器，然后通过物理机订阅该服务器的消息实现物理机和虚拟机之间的基于MQTT协议的通信，记录下在2个不同的操作系统中如何搭建EMQ服务器的整个完整的操作过程，为以后复现及网友学习供参考。
    </p>
    <h4>
     <a id="windowsemq_xWindows11_6">
     </a>
     一、在windows环境中安装emq x服务器（博主是Windows11的操作系统）
    </h4>
    <p>
     ①首先去官网下载开源免费版的emqx压缩包
     <br/>
     网址：
     <a href="https://www.emqx.com/zh/try?product=broker" rel="nofollow">
      emq 官网
     </a>
     <br/>
     目前官网v5.3.2以后的版本已不再提供windows版本的直接下载，需要去该网址进行下载v5.3.2版本的windows安装包：[https://www.emqx.com/zh/downloads/broker]
    </p>
    <p>
     界面如下：
     <br/>
     <img alt="下载步骤" src="https://i-blog.csdnimg.cn/blog_migrate/83c68000058de0b58b0d21abf0b63682.png">
      <br/>
      ②下载该文件至本地，然后找个盘解压出来文件
      <br/>
      <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/09b507c87906e1259738462df6f60183.png">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3f8f152ecb63e9288ce9afb65735aa46.png">
        <br/>
        ③邮件Windows按钮选择Windows终端（管理员）注意一定是要管理员身份运行
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/375c1d4780e23a372b68d9fe3c04e8cc.png">
         <br/>
         ④进入到刚才解压的文件路径下，然后进入该文件中的emqx\bin\目录下
        </img>
       </img>
      </img>
     </img>
    </p>
    <pre><code class="prism language-c">#先运行命令该命令
<span class="token punctuation">.</span>\emqx install
#成功后界面上会ChangeServiceConfig 成功
#再运行
<span class="token punctuation">.</span>\emqx console
#运行成功后会显示emqx is started<span class="token operator">!</span>
#然后会跳出一个界面，打开emqx运行所需要的各个端口
#最后运行
<span class="token punctuation">.</span>\emqx start
#没有报错就执行成功了
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/75c0b4bec96602c2cd35df2c01def037.png">
      <br/>
      下图是运行完第二个命令后弹出的界面（打开了各个所需要的端口）：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/324e2838c51970211752b44f8ed76957.png"/>
     </img>
    </p>
    <blockquote>
     <p>
      有的电脑可能一闪而过，但是只要成功打开了这些端口就行（具体可以通过最后是否能够成功打开最终网页来判断），也可以通过在Windows的命令窗口输入 netstat -aon|findstr “18083” 来查询是否成功打开18083端口。
      <br/>
      也有的电脑能够成功打开这个白色窗口但是窗口并没有显示打开各个端口的打印，然后界面自动关闭，但是命令窗口仍然会显示emqx is start!实际上并没有打开成功，这个情况有2种解决办法，要么重装系统，要么选用方法二：在虚拟机中装。
     </p>
    </blockquote>
    <p>
     ⑤上述工作执行完毕后，在本地浏览器输入这个网址：localhost:18083或者127.0.0.1:18083即可显示该界面，用户名:admin,密码：public
     <br/>
     （说明：localhost实际上就是127.0.0.1的一个别名，有的小白可能不懂，或者本地没有开这种类似于域名代理的功能，导致网页打不开因此在此介绍一下）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/062cad09f12aa14d89ed179e162d2bcc.png">
      <br/>
      ⑥输入账户密码进入后，界面如下：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1eaddc5e7ab19e3ee09b6717b28ec38d.png">
       <br/>
       ⑦我们设置一下中文界面，步骤如下：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b656d64e87ea42ac2a0a6fa07772423f.png">
        <br/>
        至此，在Windows环境中的EMQ服务器就搭建完成，接下来展示在虚拟机Linux中如何搭建MQTT服务器。
       </img>
      </img>
     </img>
    </p>
    <h3>
     <a id="Linuxemq_xUbuntu_1804_50">
     </a>
     Linux环境中搭建emq x服务器（博主的环境：Ubuntu 18.04）
    </h3>
    <p>
     ①在Linux系统中前往emq 官网，然后选择自己对应的版本
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/857142fa26880d82be3eccf15de9a9ec.png">
      <br/>
      点击免费下载后能够看到界面上显示能够通过命令安装
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/daf6966b83c84cf4ecbb31a065a1d63b.png"/>
      <br/>
      操作如图：
      <br/>
      点击该按钮复制命令
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2b7daf5c1d7d94afca33bc8f0fcd3905.png"/>
      <br/>
      启动命令窗口，粘贴该命令
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">wget</span> https://www.emqx.com/zh/downloads/broker/4.3.11/emqx-ubuntu18.04-4.3.11-amd64.zip
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7358d2ab83a3fe667fe7c8b00ec6535d.png"/>
     <br/>
     再复制第二条命令进行解压
    </p>
    <pre><code class="prism language-bash"><span class="token function">unzip</span> emqx-ubuntu18.04-4.3.11-amd64.zip
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c22640dce962177273bcfdc9a6b5d5de.png"/>
     <br/>
     通过文件资源管理器查看刚才解压出来的文件
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0e8c305bd3997564f039ab1b2d9c281f.png"/>
     <br/>
     通过命令也能查看解压结果
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0ae73f55dad9ca269e7139dd71723da7.png"/>
     <br/>
     最后通过命令窗口进入到\emqx\bin\目录下执行
    </p>
    <pre><code class="prism language-bash">.<span class="token punctuation">\</span>emqx start
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2e27bfd0ee1631c21e37a014b949d214.png"/>
     <br/>
     界面显示启动成功
     <br/>
     ②上述工作执行完毕后，在本地浏览器输入这个网址：localhost:18083或者127.0.0.1:18083即可显示该界面，用户名:admin,密码：public
     <br/>
     （由于博主已经按照Windows中所写的步骤将该界面设置为了中文因此再次登录的时候就是中文界面，具体步骤见上述Windows如何设置中文界面）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a39c71718b8c558bd2f170193d0528f7.png"/>
     <br/>
     ③点击登录，显示该界面即为搭建成功
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ddcaa1460e819a16c359a6942b063d1c.png"/>
     <br/>
     注：由于本篇篇幅较长，如何测试EMQ服务器的搭建成功与否，MQTTX的使用方法以及虚拟机Linux中搭建的EMQ服务器如何与物理机进行MQTT消息的通信，博主将在下一篇博客中详细说明。
    </p>
