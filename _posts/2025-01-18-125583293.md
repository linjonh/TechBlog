    <h2>
     <strong>
      一、TFT-LCD模块简介
     </strong>
    </h2>
    <p>
     <strong>
      TFT-LCD
     </strong>
     （
     <strong>
      Thin Film Transistor
     </strong>
     ）液晶显示屏是薄膜晶体管型液晶显示屏，也就是
     <span style="color:#fe2c24;">
      <strong>
       “真彩”
      </strong>
     </span>
     <strong>
      （TFT）
     </strong>
     。TFT液晶为每个像素都设有一个
     <strong>
      半导体开关
     </strong>
     ，每个像素都可以通过点脉冲直接控制，因而每个节点都相对独立，并可以连续控制，不仅提高了显示屏的反应速度，同时可以精确控制显示色阶，
     <strong>
      所以TFT液晶的色彩更真
     </strong>
     。
    </p>
    <blockquote>
     <p>
      TFT液晶显示屏的
      <span style="color:#fe2c24;">
       <strong>
        特点
       </strong>
      </span>
      是
      <strong>
       亮度好
      </strong>
      、
      <strong>
       对比度高
      </strong>
      、
      <strong>
       层次感强
      </strong>
      、
      <strong>
       颜色鲜艳
      </strong>
      ，但也存在着比较
      <strong>
       耗电
      </strong>
      和
      <strong>
       成本过高
      </strong>
      的不足。TFT液晶技术加快了手机彩屏的发展。新一代的彩屏手机中很多都支持65536色显示，有的甚至支持16万色显示，这时TFT的高对比度，色彩丰富的优势就非常重要了。
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="583" src="https://i-blog.csdnimg.cn/blog_migrate/6c442784f209f21b1b84273f49e0b2ee.png" width="800"/>
    </p>
    <p>
     市面上的TFT-LCD有需要的
     <span style="color:#fe2c24;">
      <strong>
       芯片驱动
      </strong>
     </span>
     类型（
     <strong>
      <span style="color:#fe2c24;">
       不同的驱动芯片
      </span>
      ，其显存大小与其驱动时的传输LCD初始化数据不一样。其显示功能的API函数可以互通
     </strong>
     ），市面上常见的芯片驱动有：ILI9341/ ILI9325/ RM68042/ RM68021/ ILI9320/ ILI9328/ LGDP4531/ LGDP4535/ SPFD5408/ SSD1289/ 1505/ B505/ C505/ NT35310/ NT35510 等。
    </p>
    <p>
     笔者所用的TFTLCD驱动芯片为
     <span style="color:#fe2c24;">
      <strong>
       常见的ILI9341
      </strong>
     </span>
     <span style="color:#0d0016;">
      ，这里就以ILI9341给大家为例讲诉（
      <strong>
       需要其他驱动芯片资料的可以评论留言，笔者基本上市面上常见的都有其代码与资料
      </strong>
      ）。
     </span>
    </p>
    <p>
     <span style="color:#0d0016;">
      ILI9341液晶控制器自带显存，其显存总大小为
      <strong>
       172800
      </strong>
      （240*320*18/8），即 18 位模式（26万色）下的显存量。在
     </span>
     <span style="color:#fe2c24;">
      <strong>
       16 位模式下
      </strong>
     </span>
     <span style="color:#0d0016;">
      ，ILI9341 采用
     </span>
     <span style="color:#fe2c24;">
      <strong>
       RGB565
      </strong>
     </span>
     <span style="color:#0d0016;">
      格式存储颜色数据（
      <strong>
       在更高级的32位RGB储存颜色中还有RGBA888，Linux开发板中较为常见
      </strong>
      ），此时 ILI9341的 18 位数据线与 MCU 的 16 位数据线以及 LCD GRAM 的对应关系如图 所示：
     </span>
    </p>
    <p class="img-center">
     <img alt="" height="114" src="https://i-blog.csdnimg.cn/blog_migrate/c263fb0259367adc023a61f7f738c891.png" width="931"/>
    </p>
    <p>
     这样 MCU 的 16 位数据，
     <strong>
      最低 5 位代表蓝色
     </strong>
     ，
     <strong>
      中间 6 位为绿色
     </strong>
     ，
     <strong>
      最高 5 位为红色
     </strong>
     。数值越大，表示该
     <strong>
      颜色越深
     </strong>
     。另外，特别注意 ILI9341 所有的指令都是 8 位的（
     <strong>
      高 8 位无效
     </strong>
     ），且参数除了读写 GRAM 的时候是 16 位，其他操作参数，都是 8 位的，这个和 ILI9320 等驱动器不一样，必须加以注意。

    <h2>
     二、SPI简介
    </h2>
    <p>
     <strong>
      SPI 协议
     </strong>
     是由
     <strong>
      摩托罗拉公司
     </strong>
     提出的通讯协议 (
     <strong>
      Serial Peripheral Interface
     </strong>
     )，即串行外围设备接口，是一种高速全双工的通信总线。它被广泛地使用在
     <span style="color:#fe2c24;">
      <strong>
       ADC设备
      </strong>
     </span>
     、
     <strong>
      <span style="color:#fe2c24;">
       LCD
      </span>
     </strong>
     等设备与 MCU 间，要求通讯速率较高的场合。
    </p>
    <h3>
     2.1 SPI通讯系统
    </h3>
    <p>
     SPI 通讯使用
     <span style="color:#fe2c24;">
      <strong>
       3条总线
      </strong>
     </span>
     及
     <span style="color:#fe2c24;">
      <strong>
       片选线
      </strong>
     </span>
     ，3 条总线分别为
     <strong>
      SCK、MOSI、MISO
     </strong>
     ，片选线为
     <strong>
      SS/CS
     </strong>
     ，它们的作用介绍如下：
    </p>
    <p>
     <strong>
      (1) SS/CS( Slave Select)
     </strong>
    </p>
    <p>
     <strong>
     </strong>
     <span style="color:#fe2c24;">
      <strong>
       从设备选择信号线
      </strong>
     </span>
     ，常称为
     <strong>
      片选信号线
     </strong>
     ，也称为
     <strong>
      NSS
     </strong>
     、
     <strong>
      CS
     </strong>
     ，以下用
     <strong>
      NSS
     </strong>
     表示。当有多个 SPI 从设备与 SPI 主机相连时，设备的其它信号线 SCK、MOSI 及 MISO同时并联到相同的 SPI 总线上，即无论有多少个从设备，
     <span style="color:#fe2c24;">
      <strong>
       都共同只使用这 3 条总线
      </strong>
     </span>
     ；而每个从设备都有独立的这一条 NSS 信号线，本信号线独占主机的一个引脚，即有多少个从设备，就有多少条片选信号线。
     <strong>
      <span style="color:#fe2c24;">
       I2C
      </span>
      协议中通过设备地址来寻址、选中总线上的某个设备并与其进行通讯；
     </strong>
    </p>
    <p>
     <strong>
      而SPI 协议中没有设备地址
     </strong>
     ，它使用
     <span style="color:#fe2c24;">
      <strong>
       NSS 信号线来寻址
      </strong>
     </span>
     ，当主机要选择从设备时，把该从设备的 NSS 信号线设置为
     <strong>
      低电平
     </strong>
     ，该从设备即
     <strong>
      被选中
     </strong>
     ，即
     <span style="color:#fe2c24;">
      <strong>
       片选有效
      </strong>
     </span>
     ，接着主机开始与被选中的从设备
     <span style="color:#fe2c24;">
      <strong>
       进行 SPI 通讯
      </strong>
     </span>
     。所以 SPI 通讯以 NSS 线置低电平为开始信号，以NSS 线被拉高作为结束信号。（
     <strong>
      在LCD中，片选线有很多名称，CS，SS，NSS都是指片选
     </strong>
     ）
    </p>
    <p>
     <strong>
      (2) SCK (Serial Clock)：
     </strong>
    </p>
    <p>
     <strong>
      时钟信号线，用于
      <span style="color:#fe2c24;">
       通讯数据同步
      </span>
     </strong>
     。它由通讯主机产生，决定了通讯的速率，不同的设备支持的最高时钟频率不一样，如 STM32 的
     <span style="color:#fe2c24;">
      <strong>
       SPI 时钟频率最大为 fpclk/2
      </strong>
     </span>
     ，两个设备之间通讯时，通讯速率受
     <strong>
      限于低速设备
     </strong>
     。
    </p>
    <p>
     <span style="color:#fe2c24;">
      <strong>
       (3) MOSI (Master Output，Slave Input)：
      </strong>
     </span>
    </p>
    <p>
     <strong>
      主设备输出/从设备输入引脚
     </strong>
     。主机的数据从这条信号线输出，从机由这条信号线读入主机发送的数据，即这条线上数据的方向为主机到从机。 （
     <strong>
      与
      <span style="color:#fe2c24;">
       IIC
      </span>
      相比，这个就是信号线，由主机向从机发送数据，即SDA
     </strong>
     ）
    </p>
    <p>
     <strong>
      (4) MISO(Master Input,，Slave Output)：
     </strong>
    </p>
    <p>
     <strong>
      主设备输入/从设备输出引脚
     </strong>
     。主机从这条信线读入数据，从机的数据由这条信号线输出到主机，即在这条线上数据的方向为从机到主机。 (
     <strong>
      从机向主机发送数据，使用触摸屏时需要这根线。
      <span style="color:#fe2c24;">
       如果单纯使用LCD来显示，这根线可以不接
      </span>
     </strong>
     )。
    </p>
    <p>
     <strong>
      多设备的SPI通讯接线：
     </strong>
    </p>
    <p>
     <img alt="" height="543" src="https://i-blog.csdnimg.cn/blog_migrate/638168cea6e67776dea26a1cb37c87b7.png" width="1139"/>
    </p>

     2.2 SPI时序图

    <p class="img-center">
     <img alt="" height="414" src="https://i-blog.csdnimg.cn/blog_migrate/db20ac944ad1c4bebce55f7c3db22a79.png" width="1084"/>

    <p>
     <strong>
      SPI 有四种工作模式，通过串行时钟极性(CPOL)和相位(CPHA)的搭配来得到四种工作模式：
     </strong>
     <br/>
     ①、
     <strong>
      CPOL=0
     </strong>

     <br/>
     ②、
     <strong>
      CPOL=1
     </strong>
     ，串行时钟空闲状态为高电平，此时可以通过配置时钟相位(CPHA)来选择具
     <br/>
     体的传输协议。
     <br/>
     ③、
     <strong>
      CPHA=0
     </strong>
     ，串行时钟的第一个跳变沿(上升沿或下降沿)采集数据。
     <br/>
     ④、
     <strong>
      CPHA=1
     </strong>
     ，串行时钟的第二个跳变沿(上升沿或下降沿)采集数据。
    </p>

void  SPIv_WriteData(u8 Data)
{
	SPI_WriteByte(&amp;Data, 1);
}


/****************************************************************************
* 名    称：Lcd_WriteIndex(u8 Index)
* 功    能：向液晶屏写一个8位指令
* 入口参数：Index   寄存器地址
* 出口参数：无
* 说    明：调用前需先选中控制器，内部函数
****************************************************************************/
void Lcd_WriteIndex(u8 Index)
{
   LCD_CS_CLR;
   LCD_RS_CLR;

   SPIv_WriteData(Index);

   LCD_CS_SET;
}

/****************************************************************************
* 名    称：Lcd_WriteData(u8 Data)
* 功    能：向液晶屏写一个8位数据
* 入口参数：dat     寄存器数据
* 出口参数：无
* 说    明：向控制器指定地址写入数据，内部函数
****************************************************************************/
void Lcd_WriteData(u8 Data)
{
   LCD_CS_CLR;

  	Lcd_WriteData_16Bit(Data);

	LCD_CS_CLR;
  LCD_RS_SET;
	Lcd_WriteData(Data&gt;&gt;8);
