    <h3>
     <a id="1__1">
     </a>
     1. 简介
    </h3>
    <p>
     在如今的移动互联网时代，音视频相关应用已经深入到我们日常生活的方方面面。
    </p>
    <p>
     而在它背后的音视频技术也是非常成熟了，短视频编辑、带货直播、视频语音通话等应用形式无不体现着音视频技术给我们生活带来的便利。
    </p>
    <p>
     而这次就去实际体验一下，接入声网的音频 SDK ，并打造一个音视频通话应用。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/25152a99aa59f54e3507d82ed346f243.png"/>
    </p>
    <h3>
     <a id="2__11">
     </a>
     2. 账户注册与项目创建
    </h3>
    <p>
     首先要做的就是在声网（
     <a href="https://www.agora.io/cn" rel="nofollow">
      https://www.agora.io/cn
     </a>
     ）上注册账号，并完成实名认证。
    </p>
    <p>
     登录之后就进到管理界面了，在左侧有一系列选项可以操作，直接进到项目管理，创建我们的 VideoChat 项目。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/e8f156704499d0bfcd9c0935160bf49e.png"/>
    </p>
    <p>
     在项目创建的安全模式上，选择 APPID + Token 的方式。为了提高项目的安全性，Agora 后续会取消对 APP ID 鉴权方案的支持。
    </p>
    <p>
     当项目创建完成后就有了对应的 APP ID ，这个在后续代码开发中会用到的。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/9a7886538f6845725dc5c5104245dc56.png"/>
    </p>
    <p>
     除此之外，我们还需要生成 Token ，在项目管理页去生成一个临时 Token 。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/0846fbf63616ff0255a2c233a9c79e33.png"/>
    </p>
    <p>
     这个临时 Token 是有时效的，仅供测试使用，如果有自己的后端开发，到生产环境再去生成正式 Token。同样，这个 Token 字符串在后续开发中也会用到的。
    </p>
    <h3>
     <a id="3_SDK__35">
     </a>
     3. SDK 下载与集成
    </h3>
    <p>
     创建项目并准备好 APP ID 和 Token 之后，就可以在官网上下载音频 SDK 了。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/6a7a5a9e708a3efdee8f19ea2ff2f943.png"/>
    </p>
    <p>
     下载解压后的内容如上所示。libs 文件夹内有对应不同 CPU 架构的 so 动态库，还有动态库对应的头文件以及 Java 版本的 Jar 包，可分别进行 C++ 版本和 Java 版本的集成。
    </p>
    <p>
     这里就只用到 Java 版本来演示了，其实 Java 版本里面很多方法都是走到 native 调用了，而 native 调用的就是 so 动态库里面的方法。用 C++ 版本集成的话，还需要自己写 JNI 代码将 Java 与 C++ 连接起来，不如直接用 Java 版本来的快。
    </p>
    <p>
     SDK 集成有两种方式，一种是直接使用 JCenter 来集成，在 build.gradle 里面添加一行代码就行。
    </p>
    <pre><code class="prism language-cpp">dependencies <span class="token punctuation">{<!-- --></span>
    implementation <span class="token string">'io.agora.rtc:full-sdk:3.1.3'</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     版本号可以在官网上查询，目前最新版本就是 3.1.3 了。
    </p>
    <p>
     当然还可以通过加载 jar 包和 so 动态库的方式进行集成，把 agora-rtc-sdk.jar 和各版本 so 拷贝到对应目录下，如下图所示：
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/08e1e102946a0afb26c5266ef6008197.png"/>
    </p>
    <p>
     另外别忘了在 app 目录下的 build.gradle 中添加如下代码：
    </p>
    <pre><code class="prism language-cpp">dependencies <span class="token punctuation">{<!-- --></span>
    implementation <span class="token function">fileTree</span><span class="token punctuation">(</span>dir<span class="token operator">:</span> <span class="token string">'libs'</span><span class="token punctuation">,</span> include<span class="token operator">:</span> <span class="token punctuation">[</span><span class="token string">'*.jar'</span><span class="token punctuation">]</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     少了这行代码，可解析不到添加的 jar 包。
    </p>
    <p>
     以上，就完成了整个 SDK 的工程接入，接下来就是代码开发环节了。
    </p>
    <h3>
     <a id="4__72">
     </a>
     4. 应用开发
    </h3>
    <p>
     我们要开发的是一款音视频通话应用，就像微信视频一样，想想会有哪些内容。
    </p>
    <p>
     首先要通过 Camera 采集我们的画面，然后通过麦克风录制我们的声音，再通过网络传输给到对方，并且能够听到对方的声音，在屏幕上显示画面。
    </p>
    <p>
     想一想这些内容要是纯 Android 开发的话，那涉及的东西可多了，四五个人都不一定能 hold 住，而使用专业的音视频 SDK ，一个人就能搞定大部分工作了。
    </p>
    <p>
     接下来就要去完成这样的开发工作，具体代码可以在 Github 上获取：
    </p>
    <blockquote>
     <p>
      https://github.com/glumes/agora-sdk-demo
     </p>
    </blockquote>

     </a>

    <h4>
     <a id="43__Camera__137">
     </a>
     4.3 实现 Camera 画面预览
    </h4>
    <p>
     RtcEngine 把 Camera 相关操作都封装在 SDK 内了，通过几行代码就能实现 Camera 画面预览。
    </p>
    <h5>
     <a id="431__Camera__View_141">
     </a>
     4.3.1 创建 Camera 画面预览控件 View
    </h5>
    <p>
     首先要有一个控件去承接显示 Camera 的画面输出内容，可以用 SurfaceView 也可以用 TextureView ，但并不需要我们去创建控件，而是用 Agora SDK 提供的方法。
    </p>
    <pre><code class="prism language-cpp"><span class="token comment">// 使用 SurfaceView 的场景</span>
SurfaceView mLocalView <span class="token operator">=</span> RtcEngine<span class="token punctuation">.</span><span class="token function">CreateRendererView</span><span class="token punctuation">(</span><span class="token function">getBaseContext</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 使用 TextureView 的场景</span>
TextureView mLocalView <span class="token operator">=</span> RtcEngine<span class="token punctuation">.</span><span class="token function">CreateTextureView</span><span class="token punctuation">(</span><span class="token function">getBaseContext</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     TextureView 是 Agora SDK 在 3.1.0 版本才提供的，它与 SurfaceView 的区别在于 TextureView 可以对画面进行缩放、旋转和平移，而 SurfaceView 更适合在视频通话和直播场景使用。
    </p>
    <p>
     有了显示的 View 之后，要把它添加到当前 Activity 的控件树上，后面 Camera 画面就会输出到这里。
    </p>
    <h5>
     <a id="432__Camera__156">
     </a>
     4.3.2 配置 Camera 输出并开启预览
    </h5>
    <p>
     接下来就是让 Camera 输出画面了，可实际上不用写一行关于 Camera 的代码，三行代码就可以搞定。
    </p>
    <pre><code class="prism language-java"><span class="token comment">// 开启视频</span>
mRtcEngine<span class="token punctuation">.</span><span class="token function">enableVideo</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 初始化本地视图</span>
mRtcEngine<span class="token punctuation">.</span><span class="token function">setupLocalVideo</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token class-name">VideoCanvas</span><span class="token punctuation">(</span>mLocalView<span class="token punctuation">,</span> VideoCanvas<span class="token punctuation">.</span>RENDER_MODE_HIDDEN<span class="token punctuation">,</span> <span class="token number">0</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 开启预览</span>
mRtcEngine<span class="token punctuation">.</span><span class="token function">startPreview</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     通过以上代码就能在屏幕上显示我们的画面了，默认是前置摄像头的内容。
    </p>
    <p>
     具体代码可见 Github :
    </p>
    <p>
     <a href="https://github.com/glumes/agora-sdk-demo/blob/main/app/src/main/java/com/glumes/videochat/CameraPreviewActivity.kt">
      https://github.com/glumes/agora-sdk-demo/blob/main/app/src/main/java/com/glumes/videochat/CameraPreviewActivity.kt
     </a>
    </p>
    <p>
     在代码开发中，首先要启用视频模块，默认是关闭的，通过 disableVideo 也可以关闭。而音频模块默认就是开启的，也可以通过 enableAudio 和 disableAudio 来开启关闭音频模块。
    </p>
    <p>
     之后就是通过 setupLocalVideo 方法来初始化本地视图，主要是设置本地用户视频信息的，也就是我们的画面要在 SurfaceView 中如何显示，配置信息都是通过 VideoCanvas 类下发的，它有多种参数类型。
    </p>
    <pre><code class="prism language-cpp"><span class="token comment">// VideoCanvas 构造函数类型</span>
VideoCanvas <span class="token punctuation">(</span>View view<span class="token punctuation">)</span>
VideoCanvas <span class="token punctuation">(</span>View view<span class="token punctuation">,</span> <span class="token keyword">int</span> renderMode<span class="token punctuation">,</span> <span class="token keyword">int</span> uid<span class="token punctuation">)</span>
VideoCanvas <span class="token punctuation">(</span>View view<span class="token punctuation">,</span> <span class="token keyword">int</span> renderMode<span class="token punctuation">,</span> String channelId<span class="token punctuation">,</span> <span class="token keyword">int</span> uid<span class="token punctuation">)</span>
VideoCanvas <span class="token punctuation">(</span>View view<span class="token punctuation">,</span> <span class="token keyword">int</span> renderMode<span class="token punctuation">,</span> <span class="token keyword">int</span> uid<span class="token punctuation">,</span> <span class="token keyword">int</span> mirrorMode<span class="token punctuation">)</span>
VideoCanvas <span class="token punctuation">(</span>View view<span class="token punctuation">,</span> <span class="token keyword">int</span> renderMode<span class="token punctuation">,</span> String channelId<span class="token punctuation">,</span> <span class="token keyword">int</span> uid<span class="token punctuation">,</span> <span class="token keyword">int</span> mirrorMode<span class="token punctuation">)</span>
</code></pre>
    <p>
     其中 renderMode 就是指定我们的画面在 SurfaceView 如何显示，有如下类型：
    </p>
    <ul>
     <li>
      RENDER_MODE_HIDDEN
      <ul>
       <li>
        优先保证视窗被填满。视频尺寸等比缩放，直至整个视窗被视频填满。如果视频长宽与显示窗口不同，多出的视频将被截掉。
       </li>
      </ul>
     </li>
     <li>
      RENDER_MODE_FIT
      <ul>
       <li>
        优先保证视频内容全部显示。视频尺寸等比缩放，直至视频窗口的一边与视窗边框对齐。如果视频长宽与显示窗口不同，视窗上未被填满的区域将被涂黑
       </li>
      </ul>
     </li>
     <li>
      RENDER_MODE_FILL
      <ul>
       <li>
        视频尺寸进行缩放和拉伸以充满显示视窗
       </li>
      </ul>
     </li>
    </ul>
    <p>
     默认情况下都是使用 RENDER_MODE_HIDDEN 模式的。另外还有 mirrorMode 这样的属性来设置是否要画面镜像。
    </p>
    <p>
     完成了画面显示配置之后，直接调用 startPreview 就能在屏幕上看到画面啦，是不是很简单！
    </p>
    <h5>
     <a id="433_Camera__202">
     </a>
     4.3.3 Camera 更多的配置操作
    </h5>
    <p>
     简单的背后其实是 Agora SDK 做了很多封装工作，比如 Camera1 和 Camera2 的调用逻辑、Camera 输出的分辨率策略、前后摄像头选择等等。
    </p>
    <p>
     我们可以在开启预览前通过 setCameraCapturerConfiguration 方法来配置自己想要的 Camera 信息。
    </p>
    <pre><code class="prism language-java"><span class="token comment">// 配置 Camera 信息</span>
mRtcEngine<span class="token punctuation">.</span><span class="token function">setCameraCapturerConfiguration</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token class-name">CameraCapturerConfiguration</span><span class="token punctuation">(</span>CAPTURER_OUTPUT_PREFERENCE_AUTO<span class="token punctuation">,</span>CAMERA_FRONT<span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     Camera 参数信息主要是在 CameraCapturerConfiguration 类中，它的两个参数都是枚举类型，其中第二个参数指定了使用前置还是后置摄像头。
    </p>
    <p>
     第一个参数就是关于 Camera 输出分辨率的一些策略，Agora SDK 并没有给出接口让我们指定 Camera 输出宽是多少，长是多少，而且根据使用场景组合了三个策略，更方面我们去调用了。
    </p>
    <ul>
     <li>
      <p>
       CAPTURER_OUTPUT_PREFERENCE_AUTO
      </p>
      <ul>
       <li>
        （默认）自动调整采集参数。
       </li>
      </ul>
     </li>
     <li>
      <p>
       CAPTURER_OUTPUT_PREFERENCE_PERFORMANCE
      </p>
      <ul>
       <li>
        优先保证设备性能。
       </li>
      </ul>
     </li>
     <li>
      <p>
       CAPTURER_OUTPUT_PREFERENCE_PREVIEW
      </p>
      <ul>
       <li>
        优先保证视频预览质量。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     通过以上操作，就完成了 Camera 画面预览显示。
    </p>
    <h4>
     <a id="44__229">
     </a>
     4.4 加入通话频道
    </h4>
    <p>
     接下来就要去加入一个频道，并和同一频道内的朋友进行通信。
    </p>
    <pre><code class="prism language-cpp"><span class="token comment">// 函数原型</span>
joinChannel	<span class="token punctuation">(</span>	String 	token<span class="token punctuation">,</span>String channelName<span class="token punctuation">,</span>String optionalInfo<span class="token punctuation">,</span><span class="token keyword">int</span> optionalUid<span class="token punctuation">)</span> 

<span class="token comment">// 具体调用</span>
<span class="token keyword">private</span> <span class="token keyword">void</span> <span class="token function">joinChannel</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 得到 Token</span>
    String token <span class="token operator">=</span> <span class="token function">getString</span><span class="token punctuation">(</span>R<span class="token punctuation">.</span>string<span class="token punctuation">.</span>agora_access_token<span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token comment">// Token 和 频道名要匹配</span>
    mRtcEngine<span class="token punctuation">.</span><span class="token function">joinChannel</span><span class="token punctuation">(</span>token<span class="token punctuation">,</span> <span class="token string">"demoChannel1"</span><span class="token punctuation">,</span> <span class="token string">""</span><span class="token punctuation">,</span> <span class="token number">0</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     通过
     <code>
      joinChannel
     </code>
     方法加入，其中 token 就是我们之前创建好的，而频道名称也是创建 token 时指定的，这两者要匹配起来。optionalInfo 是可选的字符串。optionalUid 是用户 ID，也是可选项，如果不指定（设为 0），SDK 会自动分配一个，并在 onJoinChannelSuccess 回调方法中返回。
    </p>
    <p>
     当我们加入频道成功后，会回调 IRtcEngineEventHandler 中的 onJoinChannelSuccess 方法。
    </p>
    <p>
     通过
     <code>
      leaveChannel
     </code>
     方法，我们可以离开当前频道，同样会回调 IRtcEngineEventHandler 中的 onLeaveChannel 方法。
    </p>
    <h4>
     <a id="45__252">
     </a>
     4.5 音视频数据编码格式参数设置
    </h4>
    <p>
     加入频道后，我们就要和频道内的朋友们通信。要把我们的画面和声音发送给对方，那肯定要将数据进行编码，然后经过网络传输送给对方。
    </p>
    <blockquote>
     <p>
      注意：音频和视频的参数都一定要在加入频道前设定好，也就是 joinChannel 方法调用之前，在其之后调用是不生效的。
     </p>
    </blockquote>
    <h5>
     <a id="451__258">
     </a>
     4.5.1 视频编码参数设置
    </h5>
    <p>
     这里我们要指定视频编码的分辨率、帧率、码率、视频等信息，通过
     <code>
      setVideoEncoderConfiguration
     </code>
     方法。
    </p>
    <pre><code class="prism language-cpp">mRtcEngine<span class="token punctuation">.</span><span class="token function">setVideoEncoderConfiguration</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token function">VideoEncoderConfiguration</span><span class="token punctuation">(</span>
        <span class="token comment">// 视频分辨率</span>
       VideoEncoderConfiguration<span class="token punctuation">.</span>VD_640x360<span class="token punctuation">,</span>
        <span class="token comment">// 帧率</span>
       VideoEncoderConfiguration<span class="token punctuation">.</span>FRAME_RATE<span class="token punctuation">.</span>FRAME_RATE_FPS_15<span class="token punctuation">,</span>
        <span class="token comment">// 码率</span>
       VideoEncoderConfiguration<span class="token punctuation">.</span>STANDARD_BITRATE<span class="token punctuation">,</span>
        <span class="token comment">// 视频方向</span>
       VideoEncoderConfiguration<span class="token punctuation">.</span>ORIENTATION_MODE<span class="token punctuation">.</span>ORIENTATION_MODE_FIXED_PORTRAIT<span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     在更早的 Agora SDK 版本中是通过 setVideoProfile 来设定的，还可以指定具体的宽高、帧率数值。
    </p>
    <p>
     最新版本换成了 VideoEncoderConfiguration 来配置，并且配置的参数是都已经定义好了相关的常量，不用自己写 720、1280 这样的 Magic Number 了。
    </p>
    <p>
     参数的设置并不是一成不变的，因为网络或者性能等因素会有一些波动，但还是会取最接近我们设定的值。
    </p>
    <h5>
     <a id="452__280">
     </a>
     4.5.2 音频编码参数设置
    </h5>
    <p>
     而音频相关的参数，则是通过
     <code>
      setAudioProfile
     </code>
     方法来设置。
    </p>
    <pre><code class="prism language-cpp">mRtcEngine<span class="token punctuation">.</span><span class="token function">setAudioProfile</span><span class="token punctuation">(</span>
        <span class="token comment">// 设置音频采样率、码率、编码模式和声道数</span>
        Constants<span class="token punctuation">.</span>AudioProfile<span class="token punctuation">.</span>DEFAULT<span class="token punctuation">.</span><span class="token function">ordinal</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">,</span>
        <span class="token comment">// 设置音频的应用场景</span>
        Constants<span class="token punctuation">.</span>AudioScenario<span class="token punctuation">.</span>DEFAULT<span class="token punctuation">.</span><span class="token function">ordinal</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     音频我们可以设置采样率、码率、编码模式以及声道数。和视频参数设置一样，我们不用指定具体的数值，Agora SDK 都根据业务使用场景做了封装，根据需要来设置就好啦。
    </p>
    <p>
     音频 AudioProfile 有如下的配置：
    </p>
    <ul>
     <li>
      <p>
       DEFAULT
      </p>
      <ul>
       <li>
        通信场景下，该选项代表指定 32 kHz 采样率，语音编码，单声道，编码码率最大值为 18 Kbps。
       </li>
       <li>
        直播场景下，该选项代表指定 48 kHz 采样率，音乐编码，单声道，编码码率最大值为 64 Kbps。
       </li>
      </ul>
     </li>
     <li>
      <p>
       SPEECH_STANDARD
      </p>
      <ul>
       <li>
        指定 32 kHz 采样率，语音编码, 单声道，编码码率最大值为 18 Kbps。
       </li>
      </ul>
     </li>
     <li>
      <p>
       MUSIC_STANDARD_STEREO
      </p>
      <ul>
       <li>
        指定 48 kHz 采样率，音乐编码, 单声道，编码码率最大值为 64 Kbps。
       </li>
      </ul>
     </li>
     <li>
      <p>
       MUSIC_HIGH_QUALITY
      </p>
      <ul>
       <li>
        指定 48 kHz 采样率，音乐编码, 单声道，编码码率最大值为 96 Kbps。
       </li>
      </ul>
     </li>
     <li>
      <p>
       MUSIC_HIGH_QUALITY_STEREO
      </p>
      <ul>
       <li>
        指定 48 kHz 采样率，音乐编码, 双声道，编码码率最大值为 128 Kbps。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     音频的应用场景也有如下：
    </p>
    <ul>
     <li>
      <p>
       DEFAULT
      </p>
      <ul>
       <li>
        默认的音频应用场景
       </li>
      </ul>
     </li>
     <li>
      <p>
       CHATROOM_ENTERTAINMENT
      </p>
      <ul>
       <li>
        娱乐应用，需要频繁上下麦的场景
       </li>
      </ul>
     </li>
     <li>
      <p>
       EDUCATION
      </p>
      <ul>
       <li>
        教育应用，流畅度和稳定性优先
       </li>
      </ul>
     </li>
     <li>
      <p>
       GAME_STREAMING
      </p>
      <ul>
       <li>
        高音质语聊房应用
       </li>
      </ul>
     </li>
     <li>
      <p>
       SHOWROOM
      </p>
      <ul>
       <li>
        秀场应用，音质优先和更好的专业外设支持
       </li>
      </ul>
     </li>
     <li>
      <p>
       CHATROOM_GAMING
      </p>
      <ul>
       <li>
        游戏开黑
       </li>
      </ul>
     </li>
    </ul>
    <p>
     一般来说，我们使用默认的配置 DEFAULT 就好了。
    </p>
    <h4>
     <a id="46__340">
     </a>
     4.6 用户加入并显示对方画面
    </h4>
    <p>
     当设置好了自己的数据编码参数，并且也成功加入了频道，接下来就是去接收频道内其他人的画面和信息了。
    </p>
    <p>
     具体代码可见 Github：
    </p>
    <p>
     <a href="https://github.com/glumes/agora-sdk-demo/blob/main/app/src/main/java/com/glumes/videochat/VideoChatActivity.kt">
      https://github.com/glumes/agora-sdk-demo/blob/main/app/src/main/java/com/glumes/videochat/VideoChatActivity.kt
     </a>
    </p>
    <h5>
     <a id="461__348">
     </a>
     4.6.1 判断是否有用户加入频道
    </h5>
    <p>
     当 IRtcEngineEventHandler 中的 onUserJoined 方法回调时，代表有人加入了当前频道，此时就可以建立并初始化远端用户视图了。
    </p>
    <p>
     如果启用了视频录制功能，视频录制服务也会回调 onUserJoined 方法，相当于有个机器人加入该频道，此时要区分开来，不能为机器人建立远端视频，然后它不会发送视频流的，建立了也是黑屏的。
    </p>
    <p>
     为了避免机器人加入带来的误判，在 2.9.0 版本后更建议在 onRemoteVideoStateChanged 方法回调中去建立远端用户视图。
    </p>
    <p>
     onRemoteVideoStateChanged 方法顾名思义就是当远端用户状态发生改变时就会调用，其中定义了如下几个状态：
    </p>
    <ul>
     <li>
      <p>
       REMOTE_VIDEO_STATE_STARTING
      </p>
      <ul>
       <li>
        本地用户已接收远端视频首包
       </li>
      </ul>
     </li>
     <li>
      <p>
       REMOTE_VIDEO_STATE_DECODING
      </p>
      <ul>
       <li>
        远端视频流正在解码，正常播放
       </li>
      </ul>
     </li>
     <li>
      <p>
       REMOTE_VIDEO_STATE_STOPPED
      </p>
      <ul>
       <li>
        远端视频默认初始状态
       </li>
      </ul>
     </li>
     <li>
      <p>
       REMOTE_VIDEO_STATE_FROZEN
      </p>
      <ul>
       <li>
        远端视频流卡顿
       </li>
      </ul>
     </li>
     <li>
      <p>
       REMOTE_VIDEO_STATE_FAILED
      </p>
      <ul>
       <li>
        远端视频流播放失败
       </li>
      </ul>
     </li>
    </ul>
    <p>
     当状态是 REMOTE_VIDEO_STATE_STARTING 或者 REMOTE_VIDEO_STATE_DECODING 时，我们认定有朋友加入频道了，此时建立远端用户视图。
    </p>
    <pre><code class="prism language-cpp">@Override
<span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">onRemoteVideoStateChanged</span><span class="token punctuation">(</span>final <span class="token keyword">int</span> uid<span class="token punctuation">,</span> <span class="token keyword">int</span> state<span class="token punctuation">,</span> <span class="token keyword">int</span> reason<span class="token punctuation">,</span> <span class="token keyword">int</span> elapsed<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
      <span class="token keyword">if</span> <span class="token punctuation">(</span>state <span class="token operator">==</span> Constants<span class="token punctuation">.</span>REMOTE_VIDEO_STATE_STARTING<span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>
           <span class="token function">runOnUiThread</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token function">Runnable</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
               @Override
               <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">run</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
                    <span class="token function">setupRemoteVideo</span><span class="token punctuation">(</span>uid<span class="token punctuation">)</span><span class="token punctuation">;</span>
               <span class="token punctuation">}</span>
           <span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
      <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     其中 onRemoteVideoStateChanged 回调方法中 state 参数代表状态，uid 用来标识远端用户 id，如果有多个朋友加入了该频道，我们就要用 uid 来区分一下用户，避免漏了或者重复建立远端用户视图。
    </p>
    <h5>
     <a id="462__392">
     </a>
     4.6.2 建立远端用户视图
    </h5>
    <p>
     接下来建立远端用户视图，和建立 Camera 预览视图方法基本一致。
    </p>
    <pre><code class="prism language-cpp"><span class="token comment">// 由 Agora SDK 创建 SurfaceView</span>
mRemoteView <span class="token operator">=</span> RtcEngine<span class="token punctuation">.</span><span class="token function">CreateRendererView</span><span class="token punctuation">(</span><span class="token function">getBaseContext</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 把 SurfaceView 添加到当前 Activity 的布局控件树上</span>
mRemoteContainer<span class="token punctuation">.</span><span class="token function">addView</span><span class="token punctuation">(</span>mRemoteView<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// Camera 画面预览就用 setupLocalVideo 方法，远端用户就用 setupRemoteVideo 方法</span>
mRtcEngine<span class="token punctuation">.</span><span class="token function">setupRemoteVideo</span><span class="token punctuation">(</span><span class="token keyword">new</span> <span class="token function">VideoCanvas</span><span class="token punctuation">(</span>mRemoteView<span class="token punctuation">,</span> VideoCanvas<span class="token punctuation">.</span>RENDER_MODE_HIDDEN<span class="token punctuation">,</span> uid<span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>     
</code></pre>
    <p>
     同样是由 Agora SDK 去创建 SurfaceView ，把它添加到控件容器上，最后通过 setupRemoteVideo 方法完成建立，整个逻辑和 Camera 视图建立是类似的，就是最后调用的方法不一样了。
    </p>

    <h3>
     <a id="5__424">
     </a>
     5. 项目开发总结
    </h3>
    <p>
     通过以上的 SDK 接入和代码示例就已经完成了一个简单的双人音视频通话。
    </p>
    <p>
     而 Agora SDK 提供的功能远非如此。
    </p>
    <p>
     我们还可以对当前频道的音视频进行控制，选择是否静音、是否关闭画面等等；我们还可以切换频道，参与频道的通话；我们还可以将项目打造成多人的音视频通话，为每个加入频道的用户建立远端视图。
    </p>
    <p>
     更多的音视频相关功能都可以通过 Agora SDK 来实现了。
    </p>

     总结一下整体的接入流程也是非常方便的：
    </p>
    <blockquote>
     <p>
      SDK 集成 -&gt; 权限设置 -&gt; Camera 预览 -&gt; 加入频道 -&gt; 显示画面 -&gt; 在线通话
     </p>
