    <h2>
     <a id="_3">
     </a>
     前言
    </h2>
    <p>
     准备学习一下怎么开发微信小程序，主要是跟着微信官方文档进行学习，在这里记录一下。
    </p>
    <hr/>
    <h2>
     <a id="_8">
     </a>
     一、开发工具
    </h2>
    <blockquote>
     <p>
      【微信开发者工具】
     </p>
     <ul>
      <li>
       根据个人使用的操作系统下载对应版本的开发者工具即可
       <br/>
       <a href="https://developers.weixin.qq.com/miniprogram/dev/devtools/stable.html" rel="nofollow">
        官网下载地址-点击跳转
       </a>
      </li>
     </ul>
    </blockquote>
    <h2>
     <a id="_16">
     </a>
     二、开发步骤
    </h2>
    <h3>
     <a id="1_17">
     </a>
     1.申请账号
    </h3>
    <ol>
     <li>
      去
      <a href="https://mp.weixin.qq.com/wxopen/waregister?action=step1" rel="nofollow">
       小程序注册页面
      </a>
      注册一个小程序。
     </li>
     <li>
      注册账号信息
     </li>
     <li>
      激活邮箱
     </li>
     <li>
      登记信息
     </li>
    </ol>
    <blockquote>
     <p>
      每个邮箱仅能申请一个小程序
     </p>
    </blockquote>
    <h3>
     <a id="2_26">
     </a>
     2.新建项目
    </h3>
    <p>
     在微信开发者工具中新建一个项目，在AppID一栏输入刚刚注册的微信小程序AppID，即可开始开发。
    </p>
    <h3>
     <a id="3_29">
     </a>
     3.发布上线（未完成）
    </h3>
    <blockquote>
     <p>
      还没尝试过, 等我的第一个小程序写完了再来补这一块 =）
     </p>
    </blockquote>
    <ol>
     <li>
      上传代码
     </li>
    </ol>
    <ul>
     <li>
      点击开发者工具顶部操作栏的上传按钮，填写
      <strong>
       版本号
      </strong>
      以及
      <strong>
       项目备注
      </strong>
      ，这里版本号以及项目备注是为了方便管理员检查版本使用的。
     </li>
    </ul>
    <ol start="2">
     <li>
      提交审核
     </li>
    </ol>
    <ul>
     <li>
      在开发者工具中上传了小程序代码之后，登录
      <strong>
       小程序管理后台
      </strong>
      -
      <strong>
       版本管理
      </strong>
      -
      <strong>
       开发版本
      </strong>
      找到提交上传的版本。
     </li>
     <li>
      在开发版本的列表中，点击
      <strong>
       提交审核
      </strong>
      按照页面提示，填写相关的信息，即可以将小程序提交审核。
     </li>
     <li>
      上传成功之后，登录
      <strong>
       小程序管理后台
      </strong>
      -
      <strong>
       版本管理
      </strong>
      -
      <strong>
       开发版本
      </strong>
      就可以找到刚提交上传的版本了。
     </li>
     <li>
      可以将这个版本设置
      <strong>
       体验版
      </strong>
      或者是
      <strong>
       提交审核
      </strong>
     </li>
    </ul>
    <ol start="3">
     <li>
      发布
     </li>
    </ol>
    <ul>
     <li>
      审核通过之后，管理员的微信中会收到小程序通过审核的通知，此时登录
      <strong>
       小程序管理后台
      </strong>
      -
      <strong>
       开发管理
      </strong>
      -
      <strong>
       审核版本
      </strong>
      中可以看到通过审核的版本。
     </li>
    </ul>
    <h2>
     <a id="_42">
     </a>
     三、小程序代码构成
    </h2>
    <ol>
     <li>
      <code>
       .json
      </code>
      后缀的 JSON 配置文件
      <ul>
       <li>
        <code>
         app.json
        </code>
        是当前小程序的全局配置，包括了小程序的所有页面路径、界面表现、网络超时时间、底部 tab 等。
       </li>
       <li>
        <code>
         project.config.json
        </code>
        是开发者开发项目时的个性化配置，其中会包括编辑器的颜色、代码上传时自动压缩等等一系列选项，方便换电脑时配置开发环境。
       </li>
       <li>
        <code>
         page.json
        </code>
        是独立定义每个页面的一些属性，例如顶部颜色、是否允许下拉刷新等。
       </li>
      </ul>
     </li>
     <li>
      <code>
       .wxml
      </code>
      后缀的 WXML 模板文件
      <ul>
       <li>
        类似
        <code>
         HTML
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <code>
       .wxss
      </code>
      后缀的 WXSS 样式文件
      <ul>
       <li>
        类似
        <code>
         CSS
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <code>
       .js
      </code>
      后缀的 JS 脚本逻辑文件
      <ul>
       <li>
        逻辑交互脚本文件。
       </li>
      </ul>
     </li>
    </ol>
    <h2>
     <a id="_54">
     </a>
     四、小程序运行逻辑
    </h2>
    <ol>
     <li>
      微信客户端在打开小程序之前，会下载整个小程序的代码包到本地。
     </li>
     <li>
      <code>
       app.json
      </code>
      的
      <code>
       pages
      </code>
      字段就可以知道你当前小程序的所有页面路径。
     </li>
     <li>
      打开首页（默认
      <code>
       pages
      </code>
      目录下的第一个页面作为首页，可修改全局配置指定首页）
     </li>
     <li>
      小程序启动后，在
      <code>
       app.js
      </code>
      定义的 App 实例的 onLaunch 回调会被执行
     </li>
    </ol>
    <pre><code class="prism language-xml">App({
  onLaunch: function () {
    // 小程序启动之后 触发
  }
})
</code></pre>

    <h6>
     <a id="_pages_75">
     </a>
     - pages
    </h6>
    <ul>
     <li>
      用于指定小程序由哪些页面组成，每一项都对应一个页面的
      <strong>
       路径
      </strong>
      （含文件名） 信息。文件名不需要写文件后缀，框架会自动去寻找对应位置的
      <code>
       .json
      </code>
      ,
      <code>
       .js
      </code>
      ,
      <code>
       .wxml
      </code>
      ,
      <code>
       .wxss
      </code>
      四个文件进行处理。
     </li>
     <li>
      未指定 entryPagePath 时，数组的第一项代表小程序的初始页面（首页）。
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       小程序中新增/减少页面，都需要对 pages 数组进行修改。
      </strong>
     </p>
    </blockquote>
    <h6>
     <a id="_sitemapLocation_80">
     </a>
     - sitemapLocation
    </h6>
    <ul>
     <li>
      指明
      <code>
       sitemap.json
      </code>
      的位置；默认
      <code>
       sitemap.json
      </code>
      即在
      <code>
       app.json
      </code>
      同级目录下名字的
      <code>
       sitemap.json
      </code>
      文件
     </li>
    </ul>

      <p>
       自 2.25.1 之后，在页面/组件实例中新增
       <code>
        getPassiveEvent
       </code>
       /
       <code>
        setPassiveEvent
       </code>
       两个接口，用于在运行时获取/切换页面或组件所在页面的
       <code>
        passive
       </code>
       配置。
      </p>
     </li>
    </ul>
    <pre><code class="prism language-xml">
Component({
    methods: {
        getPassive() {
            this.getPassiveEvent((passive) =&gt; {
                const { touchstart, touchmove, wheel } = passive
            })
        },
        setPassive() {
            const passive {
                touchstart: false,
                touchmove: true,
                wheel: false,
            }
            this.setPassiveEvent(passive)
        }
    }
})

</code></pre>
    <h6>
     <a id="_resolveAlias_466">
     </a>
     - resolveAlias
    </h6>
    <ul>
     <li>
      使用 resolveAlias 配置项用来自定义模块路径的映射规则。
     </li>
     <li>
      配置了之后，会对 require 里的模块路径进行规则匹配并映射成配置的路径。
     </li>
     <li>
      如果命中多条映射规则，则取最长的命中规则。
     </li>
    </ul>
    <pre><code class="prism language-xml">{
  "resolveAlias": {
    "~/*": "/*",
    "~/origin/*": "origin/*",
    "@utils/*": "utils/*",
    "subBUtils/*": "subpackageB/utils/*"
  }
}
</code></pre>
    <p>
     注意：
    </p>
    <ul>
     <li>
      <ol>
       <li>
        <code>
         resolveAlias
        </code>
        进行的是路径匹配，其中的 key 和 value 须以
        <code>
         /*
        </code>
        结尾。
        <br/>
        配置了上述路径映射规则，会做如下匹配并转换:
       </li>
      </ol>
      <ul>
       <li>
        <code>
         ~/mod.js
        </code>
        -&gt;
        <code>
         mod.js
        </code>
       </li>
       <li>
        <code>
         ~/origin/mod.js
        </code>
        -&gt;
        <code>
         origin/mod.js
        </code>
       </li>
       <li>
        <code>
         @utils/mod.js
        </code>
        -&gt;
        <code>
         utils/mod.js
        </code>
       </li>
       <li>
        <code>
         subBUtils/mod.js
        </code>
        -&gt;
        <code>
         subpackageB/utils/mod.js
        </code>
       </li>
      </ul>
     </li>
     <li>
      <ol start="2">
       <li>
        如果在 project.config.json 中指定了 miniprogramRoot，则
        <code>
         /*
        </code>
        指代的根目录是 miniprogramRoot 对应的路径，而不是开发者工具项目的根目录
       </li>
      </ol>
     </li>
    </ul>
