    <h3>
     <a id="_1">
     </a>
     一、前言
    </h3>
    <h4>
     <a id="1_2">
     </a>
     1、介绍说明
    </h4>
    <p>
     Stream内存性能测试工具是一套综合性能测试程序集，它主要通过Fortran和C两种高级且高效的语言编写而成，这两种语言在数学计算方面的高效率，使得Stream测试能够充分发挥出内存的能力
    </p>
    <h4>
     <a id="2_5">
     </a>
     2、测试原理
    </h4>
    <p>
     Stream测试主要通过四种数组的运算来测试内存带宽的性能，这四种运算分别是：
    </p>
    <ul>
     <li>
      <strong>
       Copy（数组的复制）
      </strong>
      ：先访问一个内存单元读出其中的值，再将值写入到另一个内存单元。
     </li>
     <li>
      <strong>
       Scale（数组的尺度变换）
      </strong>
      ：先从内存单元读出其中的值，作一个乘法运算，再将结果写入到另一个内存单元。
     </li>
     <li>
      <strong>
       Add（数组的矢量求和）
      </strong>
      ：先从内存单元读出两个值，做加法运算，再将结果写入到另一个内存单元。
     </li>
     <li>
      <strong>
       Triad（数组的复合矢量求和）
      </strong>
      ：将Copy、Scale、Add三种操作组合起来进行测试，先从内存单元中读两个值a、b，对其进行乘加混合运算（a+因子×b），将运算结果写入到另一个内存单元。
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/771788db67f44ce0b091769ae3acf8d3.png"/>
    </p>
    <h4>
     <a id="3_18">
     </a>
     3、软件特点
    </h4>
    <ul>
     <li>
      <strong>
       双精度测试
      </strong>
      ：Stream测试采用双精度（8个字节）来进行数组的运算，这样可以更准确地反映出内存带宽的实际性能。
     </li>
     <li>
      <strong>
       可持续运行
      </strong>
      ：Stream测试得到的是可持续运行的内存带宽最大值，而不是一般的硬件厂商提供的理论最大值。
     </li>
     <li>
      <strong>
       对CPU计算能力要求小
      </strong>
      ：Stream测试主要测试的是内存带宽，对CPU的计算能力要求很小，这样可以更准确地反映出内存的性能。
     </li>
    </ul>
    <h3>
     <a id="_23">
     </a>
     二、部署测试
    </h3>
    <h4>
     <a id="1_24">
     </a>
     1、安装软件
    </h4>
    <blockquote>
     <p>
      <a href="https://github.com/jeffhammond/STREAM">
       Github源码：STREAM
      </a>
      |
      <a href="http://www.cs.virginia.edu/stream/FTP/Code/" rel="nofollow">
       Stream官网
      </a>
     </p>
    </blockquote>
    <h5>
     <a id="11_27">
     </a>
     1.1、安装依赖
    </h5>
    <ul>
     <li>
      <strong>
       Ubuntu
      </strong>
     </li>
    </ul>
    <pre><code>apt-get update
apt-get install gcc build-essential gfortran -y
</code></pre>
    <h5>
     <a id="12_35">
     </a>
     1.2、下载源码
    </h5>
    <p>
     下载
     <code>
      stream.c
     </code>
     源码文件
    </p>
    <ul>
     <li>
      <strong>
       方式一
      </strong>
     </li>
    </ul>
    <pre><code>wget http://www.cs.virginia.edu/stream/FTP/Code/stream.c
</code></pre>
    <ul>
     <li>
      <strong>
       方式二
      </strong>
     </li>
    </ul>
    <pre><code>wget https://github.com/jeffhammond/STREAM/archive/refs/heads/master.zip
unzip master.zip
cd master
</code></pre>
    <h5>
     <a id="13_53">
     </a>
     1.3、编译可执行文件
    </h5>
    <p>
     设置编译参数，将
     <code>
      stream.c
     </code>
     源码编译为
     <code>
      stream
     </code>
     可执行文件，用于后续内存测试
    </p>
    <pre><code>gcc -O3 -mcmodel=small  -fopenmp -DSTREAM_ARRAY_SIZE=117964800 -DNTIMES=3 stream.c -o stream
</code></pre>
    <ul>
     <li>
      <code>
       -O3
      </code>
      ：用于指定编译器编译优化级别，当前设置参数为最高编译优化级别3
     </li>
     <li>
      <code>
       -mcmodel=small
      </code>
      ：当单个Memory Array Size 大于2GB时需要设置此参数
     </li>
     <li>
      <code>
       -fopenmp
      </code>
      ：用于启用OpenMP，适用于多核处理环境，一般需要开启，开启后测试默认使用线程数为CPU核数
     </li>
     <li>
      <code>
       -DSTREAM_ARRAY_SIZE
      </code>
      ：用于指定测试数组（a[]、b[]、c[]）的大小，默认值为10000000，该参数值对测试结果影响很大
      <br/>
      该参数值需要远大于CPU最高级缓存（一般为L3 Cache），否则测试的性能就是CPU缓存的吞吐性能，而非内存的吞吐性能
     </li>
     <li>
      <code>
       -DNTIMES
      </code>
      ：用于指定执行次数，从所有测试结果中选最优值
     </li>
     <li>
      <code>
       stream.c
      </code>
      ：用于指定stream源码文件
     </li>
     <li>
      <code>
       -o stream
      </code>
      ：用于指定源码编译输出二进制文件名，示例编译好的二进制文件名为
      <code>
       stream
      </code>
     </li>
    </ul>
    <p>
     由于stream.c源码推荐设置至少4倍最高级缓存，且STREAM_ARRAY_SIZE为double类型=8 Byte，计算公式如下
    </p>
    <pre><code>DSTREAM_ARRAY_SIZE = (&lt;cpu个数&gt; x &lt;L3 Cache&gt; x 1024 x 1024 x &lt;缓存倍数&gt;)/8 
</code></pre>
    <h5>
     <a id="14_75">
     </a>
     1.4、参考示例
    </h5>
    <ul>
     <li>
      查看当前cpu个数为1个，当前CPU最高缓存为L3 cache，缓存大小为3MB
     </li>
    </ul>
    <pre><code># root@node51:~# lscpu | grep "cache"
L1d cache:                       256 KiB
L1i cache:                       256 KiB
L2 cache:                        1 MiB
L3 cache:                        3 MiB 
</code></pre>
    <ul>
     <li>
      测试数组大小取值为CPU缓存大小300倍，即(1 x 3 x 1024 x 1024 x 300 )/8=117964800
     </li>
    </ul>
    <pre><code># root@node51:~# gcc -O3 -mcmodel=small  -fopenmp -DSTREAM_ARRAY_SIZE=117964800 -DNTIMES=3 stream.c -o stream
</code></pre>
    <h4>
     <a id="2_92">
     </a>
     2、运行测试
    </h4>
    <h5>
     <a id="21_93">
     </a>
     2.1、命令说明
    </h5>
    <ul>
     <li>
      使用多线程测试，指定运行线程数为当前CPU核数8
     </li>
    </ul>
    <pre><code>export OMP_NUM_THREADS = 8
</code></pre>
    <ul>
     <li>
      运行
      <code>
       ./stream
      </code>
      命令执行内存测试
      <br/>
      注：为避免出现误差，通常会执行3次取平均值作为测试结果
     </li>
    </ul>
    <pre><code>./stream
</code></pre>
