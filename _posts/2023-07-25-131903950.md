    <h2>
     <a id="1HarmonyOS_1">
     </a>
     1、HarmonyOS介绍
    </h2>
    <h4>
     <a id="11HarmonyOS_2">
     </a>
     1.1、初识HarmonyOS
    </h4>
    <p>
     HarmonyOS是全场景分布式智能系统
    </p>
    <p>
     超级终端的概念（用于各行各业的企业和服务提供商，为它们提供了灵活、可靠的终端设备管理解决方案。）
    </p>
    <h4>
     <a id="12HarmonyOS_6">
     </a>
     1.2、HarmonyOS系统定位
    </h4>
    <p>
     (1)
     <strong>
      新硬件
     </strong>
     ：软件定义硬件、超级终端、适应场景自由组合
     <br/>
     (2)
     <strong>
      新交互
     </strong>
     ：以人为中心、多种设备主动感知连接、智能协同
     <br/>
     (3)
     <strong>
      新服务
     </strong>
     ：服务直达，可分、可合、可流转
    </p>
    <p>
     <strong>
      HarmonyOS有三大特征
     </strong>
     <br/>
     <strong>
      (1)硬件互助，资源共享
      <br/>
      (2)一次开发，多端部署
      <br/>
      (3)统一OS，弹性部署
     </strong>
    </p>
    <h4>
     <a id="13HarmonyOS_15">
     </a>
     1.3、HarmonyOS典型应用场景
    </h4>
    <p>
     <strong>
      “1+8+N”模型
     </strong>
     <br/>
     “1（手机）+8（ai音箱，耳机，ar/vr眼镜，车机，智慧屏，手表，pad,pc）+N(风扇，空调等)”模型。
     <br/>
     <strong>
      全新办公模式
     </strong>
     <br/>
     手机和电脑互相调用
     <br/>
     <strong>
      跨设备信息流转
     </strong>
     <br/>
     手机+手表
    </p>
    <h2>
     <a id="2HarmonyOS_22">
     </a>
     2、HarmonyOS架构与安全
    </h2>
    <h3>
     <a id="21HarmonyOS_23">
     </a>
     2.1、HarmonyOS架构
    </h3>
    <p>
     这个图要记下来！！！！！！！！
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/abb5c93c385eab3bca1d08f95bce274a.png">
      <br/>
      (1)横向分4层：内核层、系统服务层、框架层、应用层
      <br/>
      (2)纵向分3层：系统 -&gt; 子系统集 -&gt; 子系统 -&gt;模块/功能
     </img>
    </p>
    <h4>
     <a id="211_28">
     </a>
     2.1.1、内核层解析
    </h4>
    <p>
     (1)内核层分为2块：内核子系统和驱动子系统
     <br/>
     (2)多内核设计，通过KAL来适配对接多个内核，满足多种设备level的需求
     <br/>
     (3)目前支持三种内核：lieos_m、liteos_a、linux
     <br/>
     (4)进程、线程、内存管理、文件系统等都属于各kernel内部的事儿，HCIA认证主要关心的是liteos_m内核
    </p>
    <h4>
     <a id="212_33">
     </a>
     2.1.2、驱动层解析
    </h4>
    <p>
     (1)硬件生态开放:HDF硬件驱动框架是HarmonyOS硬件生态开放的基础
     <br/>
     (比如：做某个设备的厂家可以根据自己的设备提供对应的HDI接口给HarmonyOS)
     <br/>
     (2)统一外设访问能力（统一接口访问）
     <br/>
     (3)驱动开发框架、驱动管理框架
     <br/>
     (4)深入理解的基础：linux驱动框架
    </p>
    <p>
     2023/7/25
    </p>
    <h4>
     <a id="213_41">
     </a>
     2.1.3、系统服务层
    </h4>
    <p>
     (1)服务，service；系统服务，os service
     <br/>
     (2)soa，service oriented architecture，面向服务的编程框架
     <br/>
     (3)服务提供者，server；服务调用者/消费者，client
     <br/>
     (4)系统服务层将OS底层能力封装成service，向上层应用提供service api
     <br/>
     (5)
     <strong>
      系统服务层是HarmonyOS的framework的核心部分
     </strong>
    </p>
    <h4>
     <a id="214_47">
     </a>
     2.1.4、框架层
    </h4>
    <p>
     (1)框架层的三个组成部分：
     <strong>
      应用程序框架、Ability框架、UI框架
     </strong>
     <br/>
     (2)三个框架都属于
     <strong>
      系统基本能力子系统集
     </strong>
    </p>
    <h4>
     <a id="215_50">
     </a>
     2.1.5、应用层
    </h4>
    <p>
     (1)应用层包括
     <strong>
      系统应用和非系统应用
     </strong>
     <br/>
     (2)FA（feature ability）和PA（particle ability）是应用的组成积木
     <br/>
     <strong>
      FA有UI界面，提供与用户交互能力，而PA无UI界面
     </strong>
     <br/>
     (3)还有个PageAbility，其实对应FA而不是PA，这里有坑。不会考，但是做应用开发容易混淆
     <br/>
     (4)最早时还有个AA（atomic ability），后来还是叫PA了，见到知道就好了，不会考
     <br/>
     (5)
     <strong>
      PA和FA就是HarmonyOS的原子化服务
     </strong>
    </p>
    <h4>
     <a id="215_57">
     </a>
     2.1.5、应用服务智能分发
    </h4>
    <p>
     (1)开发HarmonyOS应用，其实就是写需要的多个FA和PA
     <br/>
     (2)同一个PA/FA可以部署到多个不同设备中
     <br/>
     (3)同一个功能/模块，在不同的设备中可能需要设计不同的FA
     <br/>
     (4)开发应用时，要考虑好可能的场景和支持的设备，写好所有必要的FA和PA，打包成app
     <br/>
     (5)运行时根据实际参与超级终端的设备及其属性，智能分发必要的FA和PA（以hap包来分发）
     <br/>
     (6)这就是HarmonyOS应用开发的所谓“一次开发、多端执行”的背后原理
    </p>
    <h3>
     <a id="22HarmonyOS_64">
     </a>
     2.2、HarmonyOS的系统安全
    </h3>
    <h4>
     <a id="221_65">
     </a>
     2.2.1、安全的含义
    </h4>
    <p>
     (1)区分safety和security，这里的安全主要是security
     <br/>
     (2)HarmonyOS的系统安全含义：
     <strong>
      正确的人，通过正确的设备，正确的使用数据
     </strong>
    </p>
    <h4>
     <a id="2212TEE_68">
     </a>
     2.2.12、TEE环境
    </h4>
    <p>
     (1)
     <strong>
      TEE
     </strong>
     , Trusted Execution Environment，即
     <strong>
      可信执行环境
     </strong>
     <br/>
     (2)
     <strong>
      REE
     </strong>
     , Rich Execution Environment, 即
     <strong>
      所有移动设备通用的环境，运行通用的 OS
     </strong>
     <br/>
     (3)
     <strong>
      TEE需要硬件支持，不是纯软件能实现的
     </strong>
     <br/>
     (4)可以简单理解为整个系统由TEE和REE两部分组成，
     <strong>
      TEE是绝对安全的，REE只能通过受限API来访问TEE
     </strong>
    </p>
    <h4>
     <a id="2213PKI_73">
     </a>
     2.2.13、PKI证书
    </h4>
    <p>
     (1)通信中数据安全靠加密，加密分对称加密和非对称加密2大类
     <br/>
     (2)对称加密的密钥传输困难所以用得少，而非对称加密更实用
     <br/>
     (3)参考百度百科中"非对称加密"词条，来理解非对称加密原理，公钥和私钥概念
     <br/>
     (4)PKI，Public Key Infrastructure，公开密钥基础设施，指的是证书的制作和分发的一种机制。在这个机制的保障前提下，进行可信赖的网络通信。PKI的基础技术包括加密、数字签名、数据完整性机制、数字信封、双重数字签名等。
    </p>
    <h4>
     <a id="2214_78">
     </a>
     2.2.14、补充
    </h4>
    <p>
     (1)HarmonyOS的设备认证和互信，是基于华为账号系统的
    </p>
    <h2>
     <a id="3HarmonyOS_80">
     </a>
     3、HarmonyOS关键特性
    </h2>
    <h4>
     <a id="31_81">
     </a>
     3.1、三大特性
    </h4>
    <p>
     <strong>
      (1)硬件互助，资源共享
      <br/>
      (2)一次开发，多端部署
      <br/>
      (3)统一OS，弹性部署
     </strong>
    </p>
    <h5>
     <a id="311_85">
     </a>
     3.1.1硬件互助，资源共享
    </h5>
    <p>
     (2)分布式技术包括：
     <strong>
      分布式软总线、分布式数据管理、分布式文件管理、分布式任务调度、分布式设备虚拟化
     </strong>
     这几个概念。要搞清楚每个词的内涵和相互间区别
     <br/>
     13.3、分布式的几个概念和区分
     <br/>
     (1)
     <strong>
      分布式软总线是基础
     </strong>
     ，是底层通信机制（WIFI、BLE等）的软件层包装和管理
     <br/>
     (2)
     <strong>
      分布式数据管理、分布式文件管理、分布式任务调度
     </strong>
     ，这三个是分布式在系统服务层的封装
     <br/>
     (3)分布式
     <strong>
      数据管理
     </strong>
     关键：业务与数据分离，跨设备产生、存储和使用数据和本地一样方便
     <br/>
     (4)分布式
     <strong>
      文件管理
     </strong>
     关键：跨设备文件访问和访问本地文件一样
     <br/>
     (5)分布式
     <strong>
      任务调度
     </strong>
     关键：跨设备对应用进行远程启动、远程调用、远程连接以及迁移
     <br/>
     (6)分布式
     <strong>
      设备虚拟化
     </strong>
     ，是分布式在系统应用层从效果出发的描述
    </p>
    <h5>
     <a id="312_94">
     </a>
     3.1.2、一次开发多端部署
    </h5>
    <p>
     (1)此处指的是HarmonyOS应用开发，不是南向设备固件开发
     <br/>
     (2)应用开发IDE提供相应模板和机制，便于app开发者开发场景式app
     <br/>
     (3)HarmonyOS应用云市场提供相应签名、分发等机制，确保app合理组织成hap，再部署到独立设备中
     <br/>
     (4)FA和PA保证了app的可分发可运行，分布式特性保证了跨设备和设备内一样的编程方法和使用体验
     <br/>
     (5)各独立设备运行HarmonyOS，保证了端侧hap的适配和执行
    </p>
    <h5>
     <a id="313OS_100">
     </a>
     3.1.3、统一OS弹性部署
    </h5>
    <p>
     (1)此处说的是HarmonyOS设备开发，不是北向应用开发了
     <br/>
     (2)HarmonyOS从源码结构、软件工具、业务流程等方面会提供支撑，让设备商弹性部署HarmonyOS
     <br/>
     (3)理解统一OS弹性部署的含义即可，HCIA级别一般不会考具体怎么实现弹性
    </p>
