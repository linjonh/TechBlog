     被指为Edge套壳，内存占用高达1GB！微软Win11原生Copilot应用引热议

      整理 | 苏宓

     <p style="text-align:left;">
      出品 | CSDN（ID：CSDNnews）
     </p>
     <p style="text-align:justify;">
      微软正在用 AI“强制”升级自家的 Windows 11 系统上的各项功能，甚至为了方便用户直接使用，其还于去年在 Windows 11 上引入了加入了一个名为 Copilot 的 AI 助手，成为
      <a href="" rel="nofollow">
       Cortana
      </a>
      （小娜）的替代者。
     </p>
     <p style="text-align:justify;">
      最初，这个 Copilot 被集成在操作系统的侧边栏，可以帮助用户完成各种任务，如内容摘要、重写、解释等。不过，就在 12 月初，微软宣布对 Copilot 进行更新并在 Windows 11 上测试，
      <strong>
       这一版本最大的更改就是将其从由 Edge 驱动的网络应用转变为“原生”的 Windows 11 应用，即放弃以前渐进式网络应用（PWA）版本，将其更新为“原生版本”。
      </strong>
     </p>
     <p style="text-align:justify;">
      时下据外媒 Windows Latest 最新报道，Windows 11 的最新 Copilot 应用程序已全面开放，用户可通过微软应用商店即可下载与体验新版 Copilot 应用。
     </p>
     <p style="text-align:center;">
      <img alt="0fa268a56242adc6306994dea0df5ae1.jpeg" src="https://i-blog.csdnimg.cn/img_convert/0fa268a56242adc6306994dea0df5ae1.jpeg"/>
     </p>
     <p style="text-align:justify;">
      然而，令人有些遗憾的是，在抢先体验之后，不少网友纷纷涌进社交媒体反馈，Copilot 现实体验效果并不如意，所谓的「原生」似乎也只是给 Edge 套个壳，更让人有些无奈的是，Copilot 应用太占内存了，甚至最高占用超过 1GB 的内存.....
     </p>
     <p style="text-align:center;">
      <img alt="852d93558aba21fabf83692b13d7f039.png" src="https://i-blog.csdnimg.cn/img_convert/852d93558aba21fabf83692b13d7f039.png"/>
     </p>
     <p style="text-align:justify;">
      <strong>
       <img alt="808c63338f08825898ae9d59fb042b0c.png" src="https://i-blog.csdnimg.cn/img_convert/808c63338f08825898ae9d59fb042b0c.png"/>
      </strong>
     </p>
     <p>
      <strong>
       过去半年间，经历四次大改版的 Copilot 应用
      </strong>
     </p>
     <p style="text-align:justify;">
      回看过往，彼时随着 ChatGPT 风靡科技圈，作为掌舵全球第一大桌面操作系统 Windows 的微软，不可能在面对 AI 这波浪潮时无动于衷，因此，继当时在用 GPT-4 升级
      <a href="" rel="nofollow">
       Bing 搜索引擎
      </a>
      、Edge 浏览器、
      <a href="" rel="nofollow">
       Office 全家桶
      </a>
      之后，微软决定在 Windows 11 操作系统中引入 Copilot 功能，其初衷是想要为 Windows 用户提供一个更加智能、高效的辅助工具，帮助用户更轻松地完成日常任务。
     </p>
     <p style="text-align:justify;">
      在外界看来，Copilot 在 Windows 系统的集成过程比较匆忙，导致它起初的设计有些“反人类”体验。正如文章伊始所述，微软最初是将 Copilot 集成到操作系统的侧边栏中，使其成为一个常驻的、易于访问的功能模块，由此方便用户在不离开当前工作界面的情况下，快速获取信息、执行命令或进行其他操作。
     </p>
     <p style="text-align:justify;">
      不过，集成在侧边栏的设计体验并不好，有些用户表示，侧边栏的出现会分散他们的注意力，尤其是在处理复杂任务时，频繁弹出的提示和建议反而增加了操作的复杂性。
     </p>
     <p style="text-align:center;">
      <img alt="1665b38d5b71252609467f3889ad5e70.png" src="https://i-blog.csdnimg.cn/img_convert/1665b38d5b71252609467f3889ad5e70.png"/>
     </p>
     <p style="text-align:justify;">
      后来，微软就将这个侧边栏设计替换成了一个 Copilot 网页应用，界面和 Bing.com/chat 很像。没想到的是，Web 版本 Copilot 一经推出，便有用户反馈这款应用在响应速度和功能完整性上还有不少问题。
     </p>
     <p style="text-align:justify;">
      虽然微软推出了 WebView 版本的 Copilot，保留了 Web 应用的一些优点，并在用户体验上做了不少优化，还新增了语音指令互动和更多个性化设置等功能，但这款应用在 Windows 11 上的整体表现仍未完全达到预期。
     </p>
     <p style="text-align:justify;">
      随后，微软再次对这个网页应用进行了更新，推出了一个外观更现代、性能更快的版本，它的界面看起来像是 ChatGPT 的克隆版。
     </p>
     <p style="text-align:center;">
      <img alt="a5cc6bcf73139052700d51484e41b62b.png" src="https://i-blog.csdnimg.cn/img_convert/a5cc6bcf73139052700d51484e41b62b.png"/>
     </p>
     <p style="text-align:justify;">
      好景不长，当 AI 独角兽 Inflection AI 的创始人 Mustafa Suleyman 于今年 3 月加入微软并接管 AI 部门后，他取消了之前的设计，推出了一款更像个人助理风格的 Copilot PWA 版本。
     </p>
     <p>
      而现在，这个 PWA 版本又被替换成了所谓的“原生”版本。简单来看，在 Windows 11 系统上安装 Copilot 应用后，当你运行 Copilot 时，你会看到它出现在系统托盘中。
     </p>
     <p style="text-align:center;">
      <img alt="af5505077fb81275f480187ba35c9934.jpeg" src="https://i-blog.csdnimg.cn/img_convert/af5505077fb81275f480187ba35c9934.jpeg"/>
     </p>
     <p style="text-align:justify;">
      <img alt="9ef991eaf88f0a280b418f03d3676b9a.png" src="https://i-blog.csdnimg.cn/img_convert/9ef991eaf88f0a280b418f03d3676b9a.png"/>
     </p>
     <p style="text-align:justify;">
      <strong>
       被指疯狂占内存、套壳 Edge 的 Copilot
      </strong>
     </p>
     <p style="text-align:justify;">
      Copilot 虽然在 Windows 上已经重新设计了多达四次，但外媒 Windows Latest 指出，新的 Copilot 在性能上的表现还不如前两个 PWA 版本。
     </p>
     <p style="text-align:justify;">
      经过实测，新版 Copilot 应用会占用超过 1GB 的内存，平均在 500-600MB 之间。
     </p>
     <p style="text-align:center;">
      <img alt="eca3b4e34ba61cf2154cfe4beccf1a03.png" src="https://i-blog.csdnimg.cn/img_convert/eca3b4e34ba61cf2154cfe4beccf1a03.png"/>
     </p>
     <p style="text-align:justify;">
      深究背后的原因，Windows Latest 表示，微软为新版 Copilot 应用捆绑了一个专用的 WebView2 框架，这实际上就像是在 Copilot 的外壳下运行了一个新的 Microsoft Edge 实例。
     </p>
     <p style="text-align:justify;">
      <img alt="8313860520a4d5af42710398c14dab19.png" src="https://i-blog.csdnimg.cn/img_convert/8313860520a4d5af42710398c14dab19.png"/>
     </p>
     <p style="text-align:justify;">
      此外，Windows Latest 指出，所谓的“原生” Copilot 应用是一个使用 WinAppSDK 和 WinUI 构建的 Windows 应用，它的“外壳”包括窗口标题栏、图标和基础界面。但它的核心功能仍通过基于 Microsoft Edge 的 WebView 加载 Copilot 的网页（copilot.microsoft.com）。
     </p>
     <p style="text-align:justify;">
      所以，新版的 Copilot 应用看起来像是 Windows 11 的桌面应用，比如显示在系统托盘里、拥有更原生的标题栏，或者右键点击时不再出现 Edge 的菜单，但它的内容和逻辑仍然依赖网页技术。
     </p>
     <p style="text-align:justify;">
      Windows Latest 还在测试中发现，使用新版 Copilot 应用时仍然会遇到一些与 Edge 相关的弹窗，比如 Edge 的登录提示。
     </p>
     <p style="text-align:justify;">
      <img alt="c6aad704de1340a4465be0011125f957.png" src="https://i-blog.csdnimg.cn/img_convert/c6aad704de1340a4465be0011125f957.png"/>
     </p>
     <p style="text-align:justify;">
      或者是这样的：
     </p>
     <p style="text-align:justify;">
      <img alt="aeab996311b43382801539bd91dc55f7.png" src="https://i-blog.csdnimg.cn/img_convert/aeab996311b43382801539bd91dc55f7.png"/>
     </p>
     <p style="text-align:justify;">
      过往旧版的 Copilot PWA 版本（渐进式网页应用）运行方式就是这样的，但当时 Copilot 只是在 Edge 浏览器自己的 PWA 框架内打开。而现在，这个 PWA 被嵌入到一个包含 WebView2 的“原生” Windows 应用中。
     </p>
     <p style="text-align:justify;">
      正因此，Windows Latest 表示，「所谓“原生”的 Copilot 拥有了一些更贴近系统级的功能，比如你可以通过快捷键 Alt + Space 打开一个类似 ChatGPT 小窗口的快速查看界面。」
     </p>
     <p style="text-align:justify;">
      <img alt="733d61c43cb1ca8ea96465568b310a8e.png" src="https://i-blog.csdnimg.cn/img_convert/733d61c43cb1ca8ea96465568b310a8e.png"/>
     </p>
     <p style="text-align:justify;">
      不过，ChatGPT 使用的是 Chromium 的画中画功能，而 Copilot 用的是 Edge 的画中画模式——而 Edge 本身也是基于 Chromium 的，这实际上又绕回了原点。
     </p>
     <p style="text-align:justify;">
      <img alt="3194db0667b3941c99920c91634386bc.png" src="https://i-blog.csdnimg.cn/img_convert/3194db0667b3941c99920c91634386bc.png"/>
     </p>
     <p>
      <strong>
       <img alt="3118ed74ea755b842f27b09d14b96c46.png" src="https://i-blog.csdnimg.cn/img_convert/3118ed74ea755b842f27b09d14b96c46.png"/>
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       快捷键争议
      </strong>
     </p>
     <p style="text-align:justify;">
      除了被批评为套壳 Edge 之外，启动 Copilot 应用的快捷键也受到了不少争议。
     </p>
     <p style="text-align:justify;">
      如上所述，改版之后的原生 Copilot 应用在 Windows 11 上的快捷键是 Alt+Space。然而，这个快捷键已经被许多其他应用程序使用，包括微软自己的 PowerToys 应用。例如，PowerToys Run 就是使用 Alt+Space 打开一个启动器，用户可以在其中输入他们想查找的服务名称。此外，Alt+Space 还用于显示活动窗口的上下文菜单。
     </p>
     <p style="text-align:justify;">
      因此，微软决定将这个快捷键交给 Copilot 的做法，可能会让那些习惯于特定快捷键操作的“键盘用户”感到不满。
     </p>
     <p style="text-align:justify;">
      当然，如果电脑配备了专用的 Copilot 键，那么可以通过该键直接打开 Copilot 的主窗口。对此，微软此前还表示：“Copilot 将继续探索与该应用程序快捷键相关的更多选项。”
     </p>
     <p style="text-align:justify;">
      至于为什么微软选择用这个特定的快捷键打开 Copilot 的快速视图，目前还不清楚。但外媒 The Register 调侃道，希望微软内部的讨论不是这样的：
     </p>
     <p style="text-align:justify;">
      老板：没人用 Copilot，但投资人希望看到成果。我们怎么能让更多用户注意到它？
     </p>
     <p style="text-align:justify;">
      工程师：要不我们用一个很多用户经常用的快捷键加载它？
     </p>
     <p style="text-align:justify;">
      于是就有了 Alt+Space 打开 Copilot 功能的这个设置。
     </p>
     <p style="text-align:justify;">
      <strong>
       <img alt="e0c02724b71300fa1caf49e018f4397d.png" src="https://i-blog.csdnimg.cn/img_convert/e0c02724b71300fa1caf49e018f4397d.png"/>
      </strong>
     </p>
     <p style="text-align:justify;">
      <strong>
       争议四起
      </strong>
      <br/>
     </p>
     <p style="text-align:justify;">
      毋庸置疑，此举引发了关于 Copilot 的争议，对此，Windows Latest 评价道，「Copilot 并未真正体现微软与 OpenAI 合作的价值。相比之下，macOS 上的 ChatGPT 在流畅性和与现有应用的集成方面表现更佳。虽然微软可能会对 Copilot 进行再次改版，但值得深思的是，微软是否真正理解“原生”的含义。看起来，微软对“原生”的理解，与用户的期望之间存在着明显的差距。」
     </p>
     <p style="text-align:justify;">
      一些网友也失望地表示：
     </p>
     <ul>
      <li>
       <p style="text-align:justify;">
        “我讨厌 Web 开发是如何破坏了像‘原生’和‘轻量级’这样的词汇的意义的。现在一切都是捆绑了一个浏览器的网站。这就是为什么你需要32GB内存的原因。”
       </p>
      </li>
      <li>
       <p style="text-align:justify;">
        我仍然不明白他们为什么放弃侧边栏。它有点笨重，但它被正确地集成到系统中 。
       </p>
      </li>
     </ul>
     <p style="text-align:justify;">
      对此，你怎么看？
     </p>
