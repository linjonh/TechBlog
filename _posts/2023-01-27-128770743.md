     <p>
      ——不是自顶向下书内的第七章，是中科大课程补充内容：
      <strong>
       未来网络前沿技术
      </strong>
     </p>
     <h3>
      <span class="prefix">
       1
      </span>
      <span class="content">
       软件定义网络
      </span>
      <span class="suffix">
      </span>
     </h3>
     <p>
      ——software defined networking
     </p>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       背景
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      网络发展的
      <strong>
       背景和现状
      </strong>
      ：巨大成功、改造传统行业的重任
     </p>
     <p>
      当前属于
      <strong>
       第二代
      </strong>
      互联网，90年代后发展起来
     </p>
     <p>
      面临的
      <strong>
       主要问题
      </strong>
      ：
     </p>
     <ul>
      <li>
       <section>
        可扩展性
       </section>
      </li>
      <li>
       <section>
        可控性
       </section>
      </li>
      <li>
       <section>
        安全性
       </section>
      </li>
      <li>
       <section>
        实时性
       </section>
      </li>
      <li>
       <section>
        能耗问题
       </section>
      </li>
     </ul>
     <p>
      国际上布局了很多未来网络的项目：中国CENI、美国GENI、欧盟FIRE——处于战国时期
     </p>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       为什么要有SDN
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      现有网络僵化、不灵活、封闭、臃肿——创新壁垒
     </p>
     <p>
      借鉴当前计算行业的开放性：
     </p>
     <ul>
      <li>
       <section>
        分离出操作系统：开源、竞争
       </section>
      </li>
      <li>
       <section>
        应用进行解耦
       </section>
      </li>
      <li>
       <section>
        硬件进行简化
       </section>
      </li>
     </ul>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       实施
      </span>
      <span class="suffix">
      </span>
     </h4>
     <ol>
      <li>
       <section>
        开放硬件接口
       </section>
      </li>
      <li>
       <section>
        有1个良好、可扩展的操作系统
       </section>
      </li>
      <li>
       <section>
        定义良好的API，允许开发不同的应用
       </section>
      </li>
     </ol>
     <p>
      <strong>
       OpenFlow
      </strong>
     </p>
     <ul>
      <li>
       <section>
        分离数据平面、控制平面
       </section>
      </li>
      <li>
       <section>
        控制器和客户端之间命令下发控制、上传
       </section>
      </li>
      <li>
       <section>
        流表的配置下发
       </section>
      </li>
     </ul>
     <p>
      <strong>
       流表
      </strong>
     </p>
     <ol>
      <li>
       <section>
        规则：可以屏蔽掉不需要字段
       </section>
      </li>
      <li>
       <section>
        行动：转发、转送、修改、扩展……
       </section>
      </li>
      <li>
       <section>
        匹配次数统计
       </section>
      </li>
     </ol>
     <p>
      <strong>
       应用实例
      </strong>
     </p>
     <ul>
      <li>
       <section>
        交换
       </section>
      </li>
      <li>
       <section>
        流交换
       </section>
      </li>
      <li>
       <section>
        防火墙
       </section>
      </li>
     </ul>
     <h3>
      <span class="prefix">
       2
      </span>
      <span class="content">
       数据命名网络
      </span>
      <span class="suffix">
      </span>
     </h3>
     <p>
      ——named data networking NDN
     </p>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       与当前互联网区别
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      当前互联网以主机节点为中心，NDN以信息为中心
     </p>
     <p>
      目前网络中数据二八分布，在应用层提出了CDN、P2P等方案——能不能还给网络层：网络层内部资源存储能力
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909170351406.png">
      </img>
     </figure>
     <ul>
      <li>
       <section>
        数据命名：基于名字进行路由转发
       </section>
      </li>
      <li>
       <section>
        为每个数据包加安全：消除对传输安全的依赖
       </section>
      </li>
      <li>
       <section>
        连接的路由器有状态：按兴趣、按条
       </section>
      </li>
     </ul>
     <p>
      <strong>
       IP模型路由的问题
      </strong>
      ：
     </p>
     <ul>
      <li>
       <section>
        中间节点不可见
       </section>
      </li>
      <li>
       <section>
        中间节点不可选
       </section>
      </li>
      <li>
       <section>
        无法衡量成功与否
       </section>
      </li>
      <li>
       <section>
        存储不可缺少但也不可见
       </section>
      </li>
     </ul>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       NDN细节
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      <strong>
       NDN模型
      </strong>
      ：
     </p>
     <p>
      ——打破了端到端的模式
     </p>
     <p>
      路由器可以储存资源，如果有的话直接获取，没有再次进行转发查询
     </p>
     <ul>
      <li>
       <section>
        包描述“什么”而不是“谁”
       </section>
      </li>
      <li>
       <section>
        本地对端通信
       </section>
      </li>
      <li>
       <section>
        上行性能可测
       </section>
      </li>
      <li>
       <section>
        不会有环路
       </section>
      </li>
     </ul>
     <p>
      <strong>
       NDN报文
      </strong>
      ：
     </p>
     <p>
      ——兴趣报、数据报两种
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909171022405.png">
      </img>
     </figure>
     <ul>
      <li>
       <section>
        有name没有地址
       </section>
      </li>
      <li>
       <section>
        消费者：interest；生产者：data
       </section>
      </li>
      <li>
       <section>
        data沿interest原路返回
       </section>
      </li>
     </ul>
     <p>
      <strong>
       NDN数据命名
      </strong>
      ：
     </p>
     <p>
      ——分级命名，使用“/”划分，类似于url，可以进一步扩展
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909171317739.png">
      </img>
     </figure>
     <p>
      优点：
     </p>
     <ol>
      <li>
       <section>
        描述数据、场景关系
       </section>
      </li>
      <li>
       <section>
        命名聚合便于管理
       </section>
      </li>
     </ol>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       TLV基本格式
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      ——type、length、value
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909171610841.png">
      </img>
     </figure>
     <ul>
      <li>
       <section>
        <strong>
         嵌套结构
        </strong>
        ，
        <strong>
         value可以带多个子TLV结构
        </strong>
       </section>
      </li>
      <li>
       <section>
        type length变长编码
       </section>
      </li>
     </ul>
     <p>
      <strong>
       name
      </strong>
      ：
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909171808401.png">
      </img>
     </figure>
     <p>
      <strong>
       interest
      </strong>
      ：
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909171901233.png">
      </img>
     </figure>
     <p>
      <strong>
       data
      </strong>
      ：
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909172011765.png">
      </img>
     </figure>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       NDN路由器
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      <strong>
       数据结构
      </strong>
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909172103412.png">

       </section>

     </ol>
     <p>
      <strong>
       转发过程
      </strong>
     </p>
     <figure>

     <ul>

      <li>
       <section>
        之前请求过，聚合兴趣

      </li>
      <li>
       <section>

      </li>

     <h3>
      <span class="prefix">
       3
      </span>
      <span class="content">
       移动优先网络
      </span>
      <span class="suffix">
      </span>
     </h3>
     <p>
      ——mobile first networking
     </p>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       背景
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      ——现有网络对于移动终端支持比较弱
     </p>
     <ul>
      <li>
       <section>
        IP具有地址、身份双重性，但是接入点变化IP会改变
       </section>
      </li>
      <li>
       <section>
        导致网络层无法识别用户、应用层重传效率低下
       </section>
      </li>
     </ul>
     <h4>
      <span class="prefix">
      </span>
      <span class="content">
       实施
      </span>
      <span class="suffix">
      </span>
     </h4>
     <p>
      <strong>
       基本思路
      </strong>
     </p>
     <p>
      ——名字与地址分离
     </p>
     <p>
      GUID：全局唯一名字表示，提供一个GUID和IP的映射——类似DNS
     </p>
     <p>
      <strong>
       协议栈
      </strong>
     </p>
     <ul>
      <li>
       <section>
        逐跳块传输协议
       </section>
      </li>
      <li>
       <section>
        存储感知路由
       </section>
      </li>
      <li>
       <section>
        GUID与NA映射
       </section>
      </li>
     </ul>
     <p>
      <strong>
       分层命名
      </strong>
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909173006000.png"/>
     </figure>
     <p>
      应用层描述符捆绑GUID、GUID捆绑IP地址（NA）
     </p>
     <p>
      <strong>
       动态解析GUID-&gt;NA
      </strong>
     </p>
     <p>
      分布式实现
     </p>
     <figure>
      <img alt="" src="https://renzehua.gitee.io/2022/09/09/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BD%91%E7%BB%9C%E8%87%AA%E9%A1%B6%E5%90%91%E4%B8%8B/%E7%AC%AC%E4%B8%83%E7%AB%A0/image-20220909173241143.png"/>
     </figure>
     <p>
      <strong>
       存储感知抗毁路由
      </strong>
     </p>
     <ul>
      <li>
       <section>
        路由器存储某些IP报文应对可能的中断
       </section>
      </li>
      <li>
       <section>
        一旦更换网络，无缝自适应
       </section>
      </li>
      <li>
       <section>
        依赖于身份、地址的动态绑定
       </section>
      </li>
     </ul>
