    <h3>
     <a id="ps_ef__grep_java_2">
     </a>
     ps -ef | grep java
    </h3>
    <p>
     <code>
      ps
     </code>
     ：显示进程信息
     <br/>
     <code>
      grep
     </code>
     ：搜索功能
     <br/>
     <code>
      ps -ef | grep java
     </code>
     ：表示查询包含"java"的所有进程
    </p>
    <h4>
     <a id="ps_6">
     </a>
     ps
    </h4>
    <pre><code>参数： 
	`-A` ：所有的进程均显示出来，与 `-e `具有同样的效用； 
	`-a` ： 显示现行终端机下的所有进程，包括其他用户的进程； 
	`-u` ：以用户为主的进程状态 ； 
	`x `：通常与 a 这个参数一起使用，可列出较完整信息。
</code></pre>
    <p>
     输出格式规划：
     <br/>
     <code>
      l
     </code>
     ：较长、较详细的将该 PID 的的信息列出；
     <br/>
     <code>
      j
     </code>
     ：工作的格式 (jobs format)
     <br/>
     <code>
      f
     </code>
     ：做一个更为完整的输出。
    </p>
    <h4>
     <a id="grep_16">
     </a>
     grep
    </h4>
    <blockquote>
     <p>
      <code>
       grep
      </code>
      命令是查找，是一种强大的文本搜索工具，它能使用正则表达式搜索文本，并把匹配的行打印出来。
      <br/>
      <code>
       grep
      </code>
      全称是Global RegularExpression Print，表示全局正则表达式版本，它的使用权限是所有用户。
     </p>
    </blockquote>
    <p>
     <code>
      ps -ef | grep java
     </code>
     ：获取参数
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a97684a5e45f81f464abad0dff3934f4.png"/>
    </p>
    <blockquote>
     <p>
      上述参数依次：
      <br/>
      <code>
       UID
      </code>
      ：程序被该 UID 所拥有
      <br/>
      <code>
       PID
      </code>
      ：就是这个程序的 ID
      <br/>
      <code>
       PPID
      </code>
      ：则是其上级父程序的ID
      <br/>
      <code>
       C
      </code>
      ：CPU使用的资源百分比
      <br/>
      <code>
       STIME
      </code>
      ：系统启动时间
      <br/>
      <code>
       TTY
      </code>
      ：登入者的终端机位置
      <br/>
      <code>
       TIME
      </code>
      ：使用掉的CPU时间。
      <br/>
      <code>
       CMD
      </code>
      ：所下达的是什么指令
     </p>
    </blockquote>
