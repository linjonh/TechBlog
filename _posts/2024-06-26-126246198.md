    <p style="margin-left:.0001pt;text-align:justify;">
     定义、认可、记录和管理运维服务部门提供的服务及相应的服务级别，旨在通过对服务级别的监控和管理不断提高面向客户业务的运维服务质量。
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#FF0000;">
       <a name="_Toc18311800">
       </a>
       <a name="_Toc329360592">
       </a>
       <a name="_Toc329600367">
        适用范围
       </a>
      </span>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     适用于公司外部客户的服务级别管理，包括：
    </p>
    <ul>
     <li style="text-align:justify;">
      供求双方就服务需求和期望工作量特性达成协议；
     </li>
     <li style="text-align:justify;">
      制定衡量标准和报告方式：
     </li>
    </ul>
    <p style="margin-left:.0001pt;text-align:justify;">
     － 服务级别目标；
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     － 需要的资源；
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     － 服务成本。
    </p>
    <ul>
     <li style="text-align:justify;">
      与其他服务管理流程之间的合作，包括第三方供应商；
     </li>
     <li style="text-align:justify;">
      对支持服务的开销进行成本论证；
     </li>
     <li style="text-align:justify;">
      对服务级别进行定期回顾，根据情况对服务内容进行必要的改变，如果服务需要改变，或出现问题需要解决，还要衡量相应的工作量和成本。
     </li>
    </ul>
