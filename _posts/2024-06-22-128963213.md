    <h2>
     <a id="_Linux_9">
     </a>
     一、 Linux
    </h2>
    <h3>
     <a id="1_WindowLinux_10">
     </a>
     1 Window系统和Linux系统的区别
    </h3>
    <h4>
     <a id="11__11">
     </a>
     1.1 区分大小写
    </h4>
    <ul>
     <li>
      <strong>
       linux
      </strong>
      区分大小写
     </li>
     <li>
      <strong>
       windows
      </strong>
      在dos（磁盘操作系统）界面命令下不区分大小写；
     </li>
    </ul>
    <h4>
     <a id="12__14">
     </a>
     1.2 文件系统上的区别
    </h4>
    <ul>
     <li>
      Windows 下，计算机是以驱动器盘符区分所有文件
     </li>
     <li>
      Linux没有盘符这个概念，以根目录 /区分所有文件
     </li>
    </ul>
    <h4>
     <a id="13__17">
     </a>
     1.3 文件形式保存
    </h4>
    <ul>
     <li>
      <strong>
       Linux
      </strong>
      所有内容均以文件形式保存包括硬件，用户；
     </li>
     <li>
      <strong>
       windows
      </strong>
      文件和硬件没什么关系，两个之间没有关联；
     </li>
    </ul>
    <h4>
     <a id="14__20">
     </a>
     1.4 文件区分
    </h4>
    <ul>
     <li>
      <strong>
       windows
      </strong>
      用扩展名区分文件；
      <br/>
      如.exe代表执行文件，.txt代表文本文件，.gz ， .bz2 ，.tar.bz2代表压缩包，.html ，.php代表网页文件。
     </li>
     <li>
      linux是
      <strong>
       以权限区分文件
      </strong>
      ，没无扩展名的概念。管理员为了方便，可以部分扩展。
     </li>
    </ul>
    <table>
     <thead>
      <tr>
       <th>
        文件类型
       </th>
       <th>
        文件符号
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        普通文件
       </td>
       <td>
        ‘ -’
       </td>
      </tr>
      <tr>
       <td>
        目录文件
       </td>
       <td>
        ‘ d’
       </td>
      </tr>
      <tr>
       <td>
        管道文件
       </td>
       <td>
        ‘ p’
       </td>
      </tr>
      <tr>
       <td>
        链接文件
       </td>
       <td>
        ‘ l’（硬链接和软链接）。
       </td>
      </tr>
      <tr>
       <td>
        设备文件
       </td>
       <td>
        （块设备 ’ b’ 、字符设备 ‘ c’ ）
       </td>
      </tr>
      <tr>
       <td>
        套接字文件
       </td>
       <td>
        ‘s’
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     <a id="15__33">
     </a>
     1.5 开源与收费
    </h4>
    <ul>
     <li>
      <strong>
       windows
      </strong>
      大部分
      <strong>
       软件收费
      </strong>
      ，
      <strong>
       系统非开源
      </strong>
      。系统具有数量和质量的优势，由微软官方提供重要支持和服务 。
     </li>
     <li>
      <strong>
       linux
      </strong>
      平台大都为
      <strong>
       开源自由软件
      </strong>
      ，
      <strong>
       用户可以修改定制和再发布
      </strong>
      ，由于基本免费没有资金支持，部分软件质量和体验欠缺；有全球所有的Linux开发者和自由软件社区提供支持。
     </li>
    </ul>
    <h4>
     <a id="16__36">
     </a>
     1.6 执行文件与安全性
    </h4>
    <ul>
     <li>
      <strong>
       windows
      </strong>
      下的执行文件是.exe文件，大部分是图形界面，不具有安全性。.exe文件不能直接在linux下安装与运行。
     </li>
     <li>
      <strong>
       linux下
      </strong>
      的执行文件是二进制文件，大部分是字符界面，具有安全性，减少了木马攻击的可能性。
     </li>
    </ul>
    <h4>
     <a id="17__39">
     </a>
     1.7 程序安装
    </h4>
    <ul>
     <li>
      <strong>
       Windows
      </strong>
      软件安装是通过应用商店或者浏览器下载后安装。
     </li>
     <li>
      <strong>
       Linux主要
      </strong>
      通过shell命令进行安装。可以使用apt方式安装（软件包管理系统）、rpm包安装、deb包安装、tar.gz源代码包安装、tar.bz2源代码包安装、yum方式安装(安装rpm包)、bin文件安装。
     </li>
    </ul>
    <h4>
     <a id="18__42">
     </a>
     1.8 占用系统资源
    </h4>
    <ul>
     <li>
      <strong>
       linux是字符界面
      </strong>
      ，
      <strong>
       占用的系统资源
      </strong>
      较于windows下的
      <strong>
       图形界面所占的资源小
      </strong>
      。
     </li>
     <li>
      <strong>
       Windows
      </strong>
      是
      <strong>
       图形界面
      </strong>
      ，较于Linux的字符界面
      <strong>
       所占的资源大
      </strong>
      。
      <br/>
      <a href="https://blog.csdn.net/xmxt668/article/details/89319977">
       参考链接
      </a>
     </li>
    </ul>

    <h2>
     <a id="IO_73">
     </a>
     二、文件IO
    </h2>
    <h3>
     <a id="1_IOIO_74">
     </a>
     1 系统IO和标准IO操作文件的区别？
    </h3>
    <h4>
     <a id="11__76">
     </a>
     1.1 文件类型
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        标准IO
       </strong>
       打开的是一个
       <strong>
        文件指针
       </strong>
       ；
      </p>
     </li>
     <li>
      <p>
       <strong>
        系统IO
       </strong>
       打开的是一个
       <strong>
        文件描述符
       </strong>
       。
      </p>
     </li>
    </ul>
    <h4>
     <a id="12__IO_81">
     </a>
     1.2 系统 IO优势
    </h4>
    <ul>
     <li>
      系统 IO具有通用性和简约型。
     </li>
     <li>
      文件操作上普通文件、管道文件、设备节点文件、套接字文件等都可以使用，并且对文件内
     </li>
     <li>
      数据的读写不带任何格式
     </li>
     <li>
      数据的读写没有缓冲处理；而系统IO不具有通用和简约型。
     </li>
    </ul>
    <h4>
     <a id="13__IO_88">
     </a>
     1.3 标准 IO优势
    </h4>
    <ul>
     <li>
      标准 IO 是系统 IO 的封装；
     </li>
     <li>
      标准 IO具有丰富的读写方式，可以按格式读写、按 ASCII码字符读写、按二进制读写、按行读写、按数据块读写等；
     </li>
     <li>
      提供数据读写缓冲功能，极大提高程序读写效率。
     </li>
    </ul>

    <h3>
     <a id="3__101">
     </a>
     3 什么是空洞文件？谈一谈对空洞文件的理解。
    </h3>
    <h4>
     <a id="31__102">
     </a>
     3.1 空洞文件
    </h4>
    <p>
     <font color="#C21C13">
      <strong>
       没有写过的字节都被设为 0就是空洞文件。
      </strong>
     </font>
    </p>
    <p>
     在UNIX文件操作中，文件位移量可以大于文件的当前长度，此时，对该文件的下一次写将延长该文件，并在文件中构成一个空洞，这一点是允许的。位于文件中但
    </p>
    <h4>
     <a id="32__106">
     </a>
     3.2 空洞文件作用
    </h4>
    <ul>
     <li>
      空洞文件作用很大，例如迅雷
      <strong>
       下载文件
      </strong>
      ，在
      <strong>
       未下载完成时就已经占据了全部文件大小的空间
      </strong>
      ，这时候就是空洞文件。
     </li>
     <li>
      下载时
      <strong>
       如果没有空洞文件
      </strong>
      ，
      <strong>
       多线程下载时文件就都只能从一个地方写入
      </strong>
      ，这就不是多线程了。如果
      <strong>
       有了空洞文件
      </strong>
      ，可以从不同的地址写入，
      <strong>
       就完成了多线程的优势任务
      </strong>
      。
     </li>
    </ul>
