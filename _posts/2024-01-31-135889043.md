    <h2>
     <a id="_1">
     </a>
     一、概述
    </h2>
    <p>
     muparser - Fast Math Parser Library
    </p>
    <p>
     项目地址：
     <a href="https://beltoforion.de/en/muparser/features.php#idStart" rel="nofollow">
      https://beltoforion.de/en/muparser/features.php#idStart
     </a>
    </p>
    <p>
     muparser是一个高性能的数学计算表达式库，支持常用的函数、常用运算符、数学计算式的计算，这个库也提供了很多内置的功能、同时也开放了很多功能供用户使用。
     <strong>
      这个库使用的 C++语言、也可以用C语言搭配用、跨平台的支持
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c48c55b95d35b45becb7d3b5bcba9ba9.png"/>
    </p>
    <h3>
     <a id="1__9">
     </a>
     1. 针对速度进行了优化
    </h3>
    <ul>
     <li>
      高解析性能；批量模式提供了表达式计算的并行化
     </li>
     <li>
      具有延迟计算的 If-Then-Else运算符
     </li>
     <li>
      在一次求值中计算多个逗号分隔子表达式：sin(X)，x+y，x*x
     </li>
    </ul>
    <h3>
     <a id="2__13">
     </a>
     2. 支持的运算符
    </h3>
    <ul>
     <li>
      14个预定义运算符
     </li>
     <li>
      二元运算符是用户可定义的
     </li>
     <li>
      后缀运算符是用户可定义的
     </li>
     <li>
      中缀运算符是用户可定义的
     </li>
    </ul>
    <h3>
     <a id="3__18">
     </a>
     3. 支持的函数
    </h3>
    <ul>
     <li>
      25个预定义函数
     </li>
     <li>
      具有最多五个参数的固定数量
     </li>
     <li>
      具有可变数量的参数
     </li>
     <li>
      使用单个字符串参数(用于数据库查询)
     </li>
     <li>
      可以由用户扩展
     </li>
    </ul>
    <h3>
     <a id="4__24">
     </a>
     4. 用户定义的常量
    </h3>
    <ul>
     <li>
      数值常量
     </li>
     <li>
      字符串常量
     </li>
    </ul>
    <h3>
     <a id="5__27">
     </a>
     5. 用户定义的变量
    </h3>
    <ul>
     <li>
      在数量上无限
     </li>
     <li>
      可在解析器运行时由解析器定义：abc=123
     </li>
     <li>
      根据其他变量赋值变量是可能的
     </li>
    </ul>
    <h3>
     <a id="6__31">
     </a>
     6. 自定义值识别回调
    </h3>
    <ul>
     <li>
      我们可以定义值的外观
     </li>
     <li>
      我们可以让muparser读取十六进制值，如a=0x00ff00或二进制值
     </li>
     <li>
      A=0b1000101
     </li>
     <li>
      我们还可以使用此功能从数据库中查询值
     </li>
    </ul>
    <h3>
     <a id="7__36">
     </a>
     7. 其他功能
    </h3>
    <ul>
     <li>
      使用后缀运算符作为单位乘数(3M-&gt;0.003)
     </li>
     <li>
      内置函数，用于对给定变量进行数值求导
     </li>
     <li>
      参数分隔符、小数分隔符、千位分隔符可以根据我们的区域设置进行调整
     </li>
     <li>
      基于CMake的构建系统
     </li>
     <li>
      ISO 14882：符合2017年标准的代码
     </li>
     <li>
      每种语言都可以使用的DLL版本能够使用以C风格导出的函数
     </li>
    </ul>

    <h3>
     <a id="1__114">

     1. 源码文件
    </h3>
    <p>
     <strong>
      官方建议将muparser直接嵌入到项目中
     </strong>
     将库源代码直接嵌入到客户端应用程序中是避免链接器冲突的最简单方法，这些冲突源于共享解析器库和项目使用的运行时库的不同版本
    </p>
    <p>
     为了嵌入muparser，将以下文件包括到我们的项目中：
    </p>
    <pre><code class="prism language-css">muParser.cpp
muParserBase.cpp
muParserBytecode.cpp
muParserCallback.cpp
muParserError.cpp
muParserTokenReader.cpp
</code></pre>
    <p>
     并确保在我们的项目包含路径中可以找到以下包含文件：
    </p>
    <pre><code class="prism language-cpp">muParser<span class="token punctuation">.</span>h
muParserBase<span class="token punctuation">.</span>h
muParserBytecode<span class="token punctuation">.</span>h
muParserCallback<span class="token punctuation">.</span>h
muParserDef<span class="token punctuation">.</span>h
muParserError<span class="token punctuation">.</span>h
muParserFixes<span class="token punctuation">.</span>h
muParserTemplateMagic
muParserToken<span class="token punctuation">.</span>h
muParserTokenReader<span class="token punctuation">.</span>h
</code></pre>
    <p>
     解析器类和所有相关类驻留在命名空间 mu 中，请确保添加一个 using
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">using</span> <span class="token keyword">namespace</span> mu<span class="token punctuation">;</span>
</code></pre>
    <p>
     添加到我们的文件中，或使用其完整名称引用所有类
    </p>

    <h4>
     <a id="1_MUP_BASETYPE_154">

     1. MUP_BASETYPE
    </h4>
    <p>
     宏 MUP_BASETYPE 定义muParser使用的底层数据类型这可以是任何浮点值类型(浮点型、双精度型或双精度长整型)
     <strong>
      宏默认为双精度，如果我们需要更高的精度或希望将muParser Seamless与使用Float作为其数据类型的客户端代码一起使用，请修改此值
     </strong>
    </p>
    <pre><code class="prism language-cpp"><span class="token macro property"><span class="token directive-hash">#</span><span class="token directive keyword">define</span> <span class="token macro-name">MUP_BASETYPE</span> <span class="token expression"><span class="token keyword">double</span></span></span>
</code></pre>
    <h4>
     <a id="2MUP_USE_OPENMP_161">
     </a>
     2.MUP_USE_OPENMP
    </h4>
    <p>
     如果我们使用的是CMake构建系统，则使用-DENABLE_OPENMP=OFF/ON选项激活OpenMP支持
     <br/>
     该选项将为我们设置宏MUP_USE_OPENMP，并自动链接到OpenMP，启用OpenMP支持时，批量模式下的表达式求值将在多个CPU上并行执行默认情况下，使用CMake构建时，OpenMP支持处于活动状态
    </p>
    <pre><code class="prism language-cpp"><span class="token macro property"><span class="token directive-hash">#</span><span class="token directive keyword">define</span> <span class="token macro-name">MUP_USE_OPENMP</span></span>
</code></pre>
    <p>
     如果不是使用CMake构建，并且已将源代码直接嵌入到项目中，请取消对文件muParserDef.h中宏的注释
    </p>
    <p>
     根据硬件的不同，并行化可以显著提高解析器的速度(30%-400%)
    </p>
    <p>
     启用并行化对简短表达式没有显著影响
    </p>
    <h4>
     <a id="3_MUP_STRING_TYPE_175">
     </a>
     3. MUP_STRING_TYPE
    </h4>
    <p>
     这个定义决定了muParser使用的字符串类型，它可以是 std::string，也可以是 std::wstring，这个定义不应该直接设置
     <br/>
     如果存在预处理器MACRO_UNICODE，则将其定义为 std::wstring
    </p>
    <pre><code class="prism language-cpp"><span class="token macro property"><span class="token directive-hash">#</span><span class="token directive keyword">define</span> <span class="token macro-name">MUP_STRING_TYPE</span> <span class="token expression">std<span class="token double-colon punctuation">::</span>wstring</span></span>
</code></pre>
