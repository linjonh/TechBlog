    RightScale是一家提供对多个云平台统一访问的公司, 通过统一的API覆盖所有云平台.RightScale可帮助客户构建及克隆用于云环境的虚拟服务器；针对不断变化的要求，执行负载平衡；自动实现存储备份；以及提供监测和错误报告机制。
    <br/>
    RightScale重新设计了该云平台的"服务器模板"，使您可以自动配置多个云服务在所使用的虚拟服务器。至少在理论上。你可以在一台服务器模板中，在不同的云层上运行不同的元件。
    <br/>
    <p>
     <br/>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/3cc88c5fb74504f238ffa7a6e27a805b.jpeg">
      <br/>
     </img>
    </p>
    <p>
     <br/>
    </p>
    <br/>
    RightScale的产品分为四个组件：云管理环境(Cloud Management Environment);云预备服务模板和最佳时间部署库(MultiCloud Marketplace);适配自动化引擎(Adaptable Automation Engine);“众云”引擎(Multi-cloud Engine)。第五个要素是“支持编程存取的(对于RightScale平台本身)方便的可扩展平台”。
    <br/>
    这些要素并不是各自为政、相互分离的，而是完善的统一体。RightScale的管理环境是用户和软件的主要接口，它的设计可以帮助用户通过模板和库从初始化进程平滑地迁移到云环境;然后管理环境还可以用来继续构建和保存有效的资源;后面就是自动化引擎：它可以快速地提供云服务，增加不足的供应或者移除过量的供应，使供应正好匹配需求。最后一步，“众云”引擎用来支持Amazon、GoGrid、Eucalyptus和 Rackspace这些不同类型的云服务。
    <br/>
    RightScale是一款基于网络的云计算管理平台，可以管理来自多个供应商的云基础设施。它是基于主动申请的策略实现资源的伸缩，每一个机器通过申请来实现增加或削减，判断其CPU利用率是否在三分钟的时间内持续超过或者低于某一个预定义值(默认为30%和85%)。
    <br/>
    <br/>
    <br/>
    因为一只小狗的剪影而闻名的Zynga公司，在推出Farm Ville游戏的时候遭遇了用户数量井喷增长的状况。在突如其来的成功面前，Zynga却面临着数据中心资源不足的窘境，这促使Zynga选择亚马逊的EC2云计算服务。不到一天的时间里，Zynga就可以从EC2分配足够的计算资源支撑暴增的游戏用户，避免了因宕机造成的品牌伤害。而实现这一快速部署的“钥匙”就是RightScale公司提供的服务。
    <br/>
    <br/>
    <br/>
    从这以后，Zynga就成为了RightScale的忠实用户。实际上，在今天，类似Animoto（照片音乐剪辑网站）、Playfish（教育游戏网站）、Sling Media（手机网络电视公司）和TC3 Health（辨别健康保险欺诈网站）等热门公司通过RightScale管理超过300万台虚拟服务器。
    <br/>
    <br/>
    <br/>
    对于社交媒体网络等使用不止一家云计算运营商服务的公司来说，如何实现跨平台跨架构地管理各家运营商的云服务器，实现云计算资源整合，真正做到无缝使用都是个令人头疼的问题。还是以Zynga为例，用户数的爆发增长只是一时，而当用户数进入稳定增长期之后，如何再以自己公司的数据中心资源为主，进行经济的资源应用都需要使用特定的管理软件。
    <br/>
    <br/>
    <br/>
    RightScale就是这样一家基于云计算的管理平台公司，为用户提供跨云主机、平台和系统管理程序的管理解决方案，由3位创始人于2006年创建。CEO Michael Crandell、CTO Thorsten von Eicken和工程副总Rafael Saavedra。Thorsten是位康奈尔大学的计算机科学教授，后来离开学校担任Expertcity的首席架构师，Expertcity后被思杰收购。而Thorsten在Michael的建议下再次创业。Rafael则是Thorsten在Expertcity的同事。这3人形成的创业组合在一笔70万美元的早期天使投资之下开始进入云计算市场。
    <br/>
    <br/>
    <br/>
    从亚马逊开始，RightScale逐渐支持各家云服务运营商。现在除了亚马逊之外，还支持TATA、Rackspace、GoGrid、Eucalyptus、OpenStack等。要知道，2006年是美国云计算发展初期，格局尚不明朗的时候，找准自己的公司定位可不是一件容易的事情。为此，Michael在接受媒体采访时表示：“在云计算部署中，自动、简单、可靠、性能将具有真正的竞争优势，要想解决这些问题必须依靠设计优秀的解决方案”。这也许可以解释RightScale名称的来历，scale有规模、大小之意，给用户提供“正确的（使用云计算）规模（的方式）”。
