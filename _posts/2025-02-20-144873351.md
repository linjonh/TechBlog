    <h4>
     <a id="1__2">
     </a>
     1. 如何理解？
    </h4>
    <p>
     这条原理主要在讲述“游戏核心”这一概念的重要性及其在游戏开发中的作用。游戏的核心是指决定游戏整体玩法和体验的核心元素，它通常是游戏的主要机制、目标或动作方式。理解这一原理时，我们可以从以下几个层面来考虑：
    </p>
    <ul>
     <li>
      <strong>
       游戏核心的定义
      </strong>
      ：核心是游戏最基础、最关键的部分，它通常是玩家在游戏中必须进行的动作或目标。这个核心概念帮助开发团队明确游戏的方向和重点，避免设计时迷失方向。
     </li>
     <li>
      <strong>
       核心与其他设计元素的关系
      </strong>
      ：核心不仅仅是游戏机制，还与美术风格、故事、动画等其他方面紧密相关。例如，一个“流畅移动”的核心需要从视觉设计、人物设计、战斗设计、世界设计等多个方面来支持。通过这些支持，游戏的核心才能得到强化和突出。
     </li>
     <li>
      <strong>
       核心的数量
      </strong>
      ：保持核心数量的适中（通常是3个核心）是为了确保团队能专注于每一个核心，避免核心设计过多导致混乱和分散精力。
     </li>
     <li>
      <strong>
       核心与创新
      </strong>
      ：通过对现有游戏核心的扩展、改进和混搭，可以在保持基本玩法的同时创造出具有原创性的体验。
     </li>
    </ul>
    <h4>
     <a id="2__11">
     </a>
     2. 如何找到某一款游戏的核心，如何描述？
    </h4>
    <p>
     找到一款游戏的核心，首先要识别出其最基础、最关键的玩法元素。通常，这些元素会直接关系到玩家在游戏中所需进行的主要操作和目标。以下是一些方法来识别和描述游戏核心：
    </p>
    <ul>
     <li>
      <strong>
       分析玩家的行为
      </strong>
      ：游戏核心通常与玩家的主要行为相关，例如，玩家需要做什么才能取得进展或成功。例如，在一个平台跳跃游戏中，核心可能是“跳跃和避开障碍物”；在一个角色扮演游戏中，核心可能是“探索和与NPC互动”。
     </li>
     <li>
      <strong>
       明确游戏的目标
      </strong>
      ：游戏的核心常常反映了游戏的目标或目的。比如，在赛车游戏中，核心是“驾驶和竞速”；在塔防游戏中，核心是“防御和策略布局”。
     </li>
     <li>
      <strong>
       描述核心的方式
      </strong>
      ：核心可以用一个简洁的句子来概括，通常包含以下元素：
      <ul>
       <li>
        <strong>
         动作
        </strong>
        ：玩家需要做的主要操作（如“跳跃”、“移动”、“攻击”）。
       </li>
       <li>
        <strong>
         目标
        </strong>
        ：玩家的最终目的（如“击败敌人”、“完成关卡”）。
       </li>
       <li>
        <strong>
         体验
        </strong>
        ：游戏的核心给玩家带来的核心体验（如“流畅的控制”，“紧张的战斗”）。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      示例描述
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       《超级马里奥》
      </strong>
      ：游戏的核心是“精准跳跃和避开障碍物”，玩家需要通过精确的跳跃来避开陷阱、敌人并收集物品，目标是到达关卡终点。
     </li>
     <li>
      <strong>
       《堡垒之夜》
      </strong>
      ：游戏的核心是“建造与射击结合的战斗”，玩家通过构建防御工事和使用武器来对抗敌人，目标是成为最后一个存活的玩家。
     </li>
    </ul>
    <h4>
     <a id="3__27">
     </a>
     3. 在游戏设计中如何生成核心？
    </h4>
    <p>
     生成游戏核心的过程可以通过系统化的思考来实现。以下是几种常见的方法：
    </p>
    <h5>
     <a id="_31">
     </a>
     头脑风暴法：提问式探索
    </h5>
    <p>
     通过提出一系列“如果……会怎样”的问题来激发创新的思维：
    </p>
    <ul>
     <li>
      如果把一种体裁中最好的元素和另一种体裁中最好的元素结合起来会怎样？
      <ul>
       <li>
        例如，将“塔防”与“角色扮演”结合，可能形成一个玩家需要控制角色同时防御的游戏核心。
       </li>
      </ul>
     </li>
     <li>
      如果把另一个游戏中失败的机制改造一下会怎样？
      <ul>
       <li>
        例如，改造一个难度过高的跳跃机制，让它变得更加易上手，作为核心来提供更流畅的体验。
       </li>
      </ul>
     </li>
     <li>
      如果将当前的游戏玩法延伸，允许一些新的行为会怎样？
      <ul>
       <li>
        例如，在一个简单的跑酷游戏中加入战斗元素，使核心变成“跑酷与战斗的结合”。
       </li>
      </ul>
     </li>
     <li>
      如果将其他媒介的体验复制到游戏中来会怎样？
      <ul>
       <li>
        例如，模仿电影中的特定场景或情节，赋予游戏一个全新的核心，如基于叙事的战术游戏。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_44">
     </a>
     结合创新和已有的元素
    </h5>
    <p>
     将已知的游戏核心元素进行创新、扩展或混搭，形成新的玩法：
    </p>
    <ul>
     <li>
      例如，“吃豆人”游戏的核心是“吃豆”和“躲避敌人”，而《贪吃蛇》核心则是“吃食物并变长”。通过对这些玩法进行扩展，可以将核心概念转化为更复杂或创新的游戏设计。
     </li>
    </ul>
    <h5>
     <a id="_50">
     </a>
     定义核心的精炼度
    </h5>
    <p>
     在设计过程中，核心要尽量精炼，明确哪些机制是游戏不可或缺的，哪些是辅助性的。避免在核心上过多堆砌，使其保持简洁并易于传达。
    </p>
    <h4>
     <a id="_54">
     </a>
     总结
    </h4>
    <ul>
     <li>
      <strong>
       理解核心
      </strong>
      ：游戏核心是决定玩家在游戏中体验的主要元素，通常与玩家的核心动作、目标以及游戏体验紧密相关。
     </li>
     <li>
      <strong>
       找到核心
      </strong>
      ：识别游戏的核心可以通过分析玩家行为、游戏目标和体验来进行，描述核心时要简洁且聚焦。
     </li>
     <li>
      <strong>
       生成核心
      </strong>
      ：通过头脑风暴的提问法、创新和已有元素的结合来设计核心，并保持核心的简洁和精炼。
     </li>
    </ul>
    <p>
     “核心”和“核心游戏循环”（Core Gameplay Loop）是两个密切相关但又有所不同的概念，它们在游戏设计中各自扮演着重要的角色。我们可以通过以下方式来理解它们之间的关系：
    </p>
    <h4>
     <a id="1__64">
     </a>
     1.
     <strong>
      定义上的区别
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       核心（Pillar）
      </strong>
      ：
      <ul>
       <li>
        <strong>
         核心
        </strong>
        是指游戏的核心概念或目标，是游戏设计的指导原则和最重要的元素。它定义了游戏的“主题”和最关键的玩法机制，决定了游戏的基础结构。例如，游戏的核心可以是“流畅的移动”，“塔防”，“战斗与探索结合”等。
       </li>
       <li>
        <strong>
         核心
        </strong>
        的目的是帮助团队明确游戏的主要方向，让所有的设计决策都围绕这些核心展开。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       核心游戏循环（Core Gameplay Loop）
      </strong>
      ：
      <ul>
       <li>
        <strong>
         核心游戏循环
        </strong>
        是指玩家在游戏中不断重复的基本行为和操作流程。它是玩家进行游戏时的核心互动循环，通过这些循环来推动游戏进程。游戏循环通常包括一系列玩家的动作、反馈、奖励等。
       </li>
       <li>
        游戏循环往往描述的是具体的玩家行为和机制如何相互作用，形成一个可持续的反馈环。例如，在动作冒险类游戏中，玩家的核心循环可能是“探索 -&gt; 战斗 -&gt; 获得奖励 -&gt; 升级角色 -&gt; 探索新区域”。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="2__73">
     </a>
     2.
     <strong>
      它们之间的关系
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        核心指导游戏循环的设计
       </strong>
       ：
      </p>
      <ul>
       <li>
        游戏的核心概念通常决定了游戏循环的结构和内容。换句话说，核心是设计游戏循环的出发点和框架。每个游戏的核心都会影响其核心游戏循环的设计，确保它们一致并且能为玩家带来预期的体验。
       </li>
      </ul>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果一个游戏的核心是“射击与生存”，那么核心游戏循环可能会是“射击敌人 -&gt; 收集资源 -&gt; 升级武器 -&gt; 持续战斗”。核心的“射击与生存”概念驱动了这些循环行为的发生。
       </li>
       <li>
        如果一个游戏的核心是“探索与解谜”，那么核心游戏循环可能会是“解锁新区域 -&gt; 解决谜题 -&gt; 获取奖励 -&gt; 解锁新技能”。核心的“探索与解谜”理念为这些行为提供了框架和方向。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        核心游戏循环是核心的实现方式
       </strong>
       ：
      </p>
      <ul>
       <li>
        核心游戏循环是通过具体的操作和行为实现核心的理念和目标。在一个良好的设计中，核心游戏循环会直接落实游戏的核心，并通过不断的玩家互动，让核心的价值观和体验得以体现。
       </li>
      </ul>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <ul>
       <li>
        在一个“快节奏的竞速”类游戏中，核心是“快速移动和竞速”，而核心游戏循环可能是“起跑 -&gt; 加速 -&gt; 遇到障碍 -&gt; 克服障碍 -&gt; 决胜终点”。这个游戏循环通过不断重复的竞速任务，让玩家体验到核心的快节奏移动感。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        核心游戏循环的优化和调整影响核心的呈现
       </strong>
       ：
      </p>
      <ul>
       <li>
        随着开发的推进，设计者可能会根据核心游戏循环的反馈不断调整和优化核心的实现方式。比如，如果核心循环中的某一部分机制让玩家感到无聊或单调，那么核心的概念或核心循环可能需要做出相应的调整。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="3__96">
     </a>
     3.
     <strong>
      如何互相配合
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       早期阶段的定义
      </strong>
      ：在游戏设计的早期阶段，首先应该定义游戏的核心（Pillar），并确定玩家通过哪些具体的操作和行为去体验这个核心。这时，核心游戏循环的框架也应该初步设定好，确保两者能够协同工作。
     </li>
     <li>
      <strong>
       实际开发中的持续反馈
      </strong>
      ：随着开发进程的推进，核心游戏循环可能会经历多次调整和测试。在调整时，需要确保这些改变始终围绕游戏的核心展开，以便游戏的每个设计元素都能够强化核心理念。
     </li>
    </ul>
    <h4>
     <a id="4__101">
     </a>
     4.
     <strong>
      举例说明
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       《愤怒的小鸟》
      </strong>
      ：
      <ul>
       <li>
        <strong>
         核心
        </strong>
        ：物理引擎和弹射机制，通过精准投射来破坏结构。
       </li>
       <li>
        <strong>
         核心游戏循环
        </strong>
        ：选择弹射角度 -&gt; 发射鸟 -&gt; 观察物理效果 -&gt; 收集奖励 -&gt; 进入下一关。
       </li>
       <li>
        在《愤怒的小鸟》这个例子中，核心“物理引擎和弹射机制”直接决定了游戏的循环行为，每次发射鸟的操作、物理效果的呈现和结果反馈都与核心密切相关。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       《黑暗之魂》
      </strong>
      ：
      <ul>
       <li>
        <strong>
         核心
        </strong>
        ：挑战性高的战斗与深度的探索。
       </li>
       <li>
        <strong>
         核心游戏循环
        </strong>
        ：探索世界 -&gt; 遇到敌人 -&gt; 战斗并击败敌人 -&gt; 获得资源或经验 -&gt; 升级或解锁新区域。
       </li>
       <li>
        在《黑暗之魂》中，核心是“高挑战性战斗”和“深度探索”，而这些核心理念通过核心游戏循环得以实现，使玩家不断重复这一过程，体验到不断挑战自我、逐步提升的游戏节奏。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_112">
     </a>
     总结
    </h4>
    <ul>
     <li>
      **“核心”**是游戏的概念性框架，它定义了游戏的主题、目标和玩家在游戏中要完成的主要任务。它是游戏设计的指导思想。
     </li>
     <li>
      **“核心游戏循环”**是玩家在游戏中实际体验到的反复行为过程，是实现核心概念的具体机制和操作步骤。核心游戏循环通过不断重复的互动，让玩家沉浸在游戏的核心体验中。
     </li>
    </ul>
    <p>
     二者的关系可以理解为：
     <strong>
      核心决定了游戏的主题和目标，而核心游戏循环则是玩家通过具体操作实现这些目标的路径。
     </strong>
    </p>

    <h4>
     <a id="_123">
     </a>
     游戏的“核心”
    </h4>
    <p>
     游戏的“核心”(pillar)是一种高屋建瓴的、以动作为核心的概念或者目标，在游戏开发的过程中它起到指导原则的作用。在构思一个新游戏时，确定游戏的核心通常是第一要务，因为这能帮助向整个团队传达项目的总体方向。一旦游戏核心被确立，将它们揉进一个句子中，就能成为一个用于向发行商和公众介绍游戏的简介。
    </p>
    <p>
     由于游戏是交互性的，这个“核心”非常关键的一点就是要跟玩家在游戏中需要进行的动作密切相关。使用美术和主题相关的元素作为游戏“核心”的灵感来源（参见原理58“主题”）是可以的，但前提是主要的考虑方向还是在于游戏的具体功能。要理解为什么这样，我们可以试着考虑一下相反的情况：如果开发者将他们对游戏核心的考量重点放在美术上，他们将不会在游戏机制相关的决策上有一个清晰的方向，以至于可能会拿来一个现有的游戏只是改改美术，这最终就成了一个老游戏的“新皮肤”版本，而不是一个有趣的新游戏。
    </p>
    <p>
     从另一方面看，这也意味着“核心”可以应用于更小的范围，比如美术团队就可以设立他们自己的美术“核心”，用来支持整个游戏的“核心”。
    </p>
    <p>
     清楚地定义一个项目的核心能够让团队评估他们在创新和借鉴方面的工作量。如果所有的核心都和另一款游戏一样，玩家看到后会认出其潜在的模式，并且将这个游戏看成另一款游戏的克隆，而非一个独立的不一样的体验。通过将其他各种游戏中的核心进行扩展、改进、用独特的方式混搭，就能在不承担太大风险的情况下达到原创性（参见原理55“风险评估”）。
    </p>
    <h4>
     <a id="_133">
     </a>
     如何生成游戏的核心
    </h4>
    <p>
     一种生成游戏核心的头脑风暴方法是问关于“如果……会怎样”的问题：
    </p>
    <ul>
     <li>
      如果把一种体裁中最好的元素和另一种体裁中最好的元素结合起来会怎样？
     </li>
     <li>
      如果把另一个游戏中失败的机制改造一下会怎样？
     </li>
     <li>
      如果将当前的游戏玩法延伸，允许一些新的行为会怎样？
     </li>
     <li>
      如果将其他媒介的体验复制到游戏中会怎样？
     </li>
    </ul>
    <p>
     如果持续不断地问这样的问题，一个团队就会被迫去分析这个项目，而这能帮助他们发现游戏世界中尚未被发现和待开发的区域。
    </p>
    <p>
     严格控制游戏核心的数量通常也是非常有利的。一般来讲比较常见的是3个核心，因为这样能保证针对每一个核心都能深入，而不至于弄得太复杂。6个核心通常就太多了。
    </p>
    <h4>
     <a id="_146">
     </a>
     如何使用核心
    </h4>
    <p>
     在开发的初始阶段定义核心，同时考虑到人员和时间的限制，能够帮助确立合适的范围。早期把核心定义清楚还能避免一些与之矛盾的元素在开发阶段被混进来。例如，如果“可达性”是一个核心，设计一个只能通过试错来解决的谜题就是完全违背可达性原则的，这样一个矛盾的存在会让享受着游戏中其他可达元素的玩家感到失望和受挫。
    </p>
    <p>
     一个合适的核心能够在开发团队开始工作之前帮助其勾画出大致的需求。例如，以“流畅的移动”这样一个核心为例：
    </p>
    <ul>
     <li>
      游戏的角度：第三人称的角度能够凸显并且强化“流畅的移动”的优势。相对而言，第一人称的角度就不太适合这个主题，因为必须考虑避免晕动现象的发生。
     </li>
     <li>
      人物设计：玩家角色的形象设计成瘦长的就比矮胖的合适，这样更符合他们在环境中移动得非常流畅的形象。
     </li>
     <li>
      战斗设计：玩家角色在游戏世界中移动的流畅性也应该延伸到战斗场景。笨重、粗暴的战斗设计会破坏玩家的沉浸感。
     </li>
     <li>
      世界设计：建筑和地形都要从视觉上体现玩家能做和不能做的事情。比如建筑物的表面就要有视觉语言明确地表达该建筑是否能够攀爬。
     </li>
     <li>
      动画的工作量：流畅的移动要求很多动画来实现无缝过渡，而这有可能超过硬件的负荷能力，因此需要小心计划。
     </li>
    </ul>
    <p>
     游戏的核心指导开发团队去创造一个重点突出的、独特的体验。尽早定义游戏的核心能够为开发过程减少很多麻烦，让开发过程更高效，并简化设计过程。
    </p>
