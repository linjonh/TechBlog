    <h3>
     编码乱码问题:
    </h3>
    <p>
     经常会遇到
     <strong>
      前端传到后台时
     </strong>
     ,也就是
     <strong>
      controller层
     </strong>
     时,会发生乱码,在这里我总结了几个解决乱码的方法,基本都能解决,
     <strong>
      @符号越多代表越常见
     </strong>
    </p>
    <p>
     1.@@Tomcat服务器配置的问题—从前端传值到后台乱码—从后台插入到服务器也乱码。
     <br/>
     方法：找到Tomcat的安装目录，在cunf下的server.xml文件中找到
     <br/>
     &lt; Connector port=“8081” protocol=“HTTP/1.1”
     <br/>
     connectionTimeout=“20000”
     <br/>
     redirectPort=“8443” URIEncoding=“UTF-8” / &gt;
     <br/>
     &lt; !-- A “Connector” using the shared thread pool-- &gt;
     <br/>
     把加粗部分的语句加上就好了
    </p>
    <p>
     2.@@数据库连接上的问题:
     <br/>
     检查这句话有没有问题
     <br/>
     db.url=jdbc:mysql://127.0.0.1:3306/myssm?
     <strong>
      useUnicode=true&amp;
     </strong>
     characterEncoding=utf8，
     <br/>
     可以试着把加粗部分不要试下有没有乱码
    </p>
    <p>
     3.@@@@检查前端页面是否有utf-8
     <br/>
     jsp页面一般有–&gt;
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0605a9ad4c1b20199739f104916b003b.png">
      <br/>
      h5页面一般有–&gt;
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3a39a9a4159687767d2553cff4132ced.png">
       <br/>
       4.@@@检查Eclipse的编码和项目的编码格式是不是一样的。
       <br/>
       eclipse编码在:Windows–&gt;preference-&gt;general–&gt;workspace设置
       <br/>
       项目编码在:项目上右键–&gt;properties
      </img>
     </img>
    </p>
    <p>
     5.@@@@检查数据库的编码格式,这点很致命,解决办法:
     <br/>
     my.ini文件设置,或者创建库时选择
    </p>
    <p>
     6.@@@@@@
     <em>
      <strong>
       最终解决方法
      </strong>
     </em>
     ; 用getBytes方法
     <strong>
      在controller层重置传过来字符串的编码
     </strong>
     ,这也是最最终极的办法,好处是能够解决最麻烦的乱码问题,但是却只能解决一处乱码,解决别的地方的乱码需要再重新写
     <br/>
     <strong>
      示例:
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/75aab060fd976dfb25f94417aa2f6bec.png">
      <br/>
      语句:
      <strong>
       name= new String(name.getBytes(“ISO8859-1”),“UTF-8”
      </strong>
      );在controller层添加这句话,把jsp前端传过来的字符串改变编码
      <br/>
      name 是前端传过来的会乱码的内容,这么写,所有这一层次的乱码问题都会得到解决
     </img>
    </p>
