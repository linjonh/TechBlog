    1.操作系统状态监控
    <br/>
    每天登陆系统查看系统运行的负荷如何，有无报错日志或报警日志。
    <br/>
    2.操作系统故障排除
    <br/>
    依据操作系统故障日志分析出现该报警或报错的原因，从而解决问题，保证操作系统的高可用性。
    <br/>
    3.服务器状态确认
    <br/>
    服务器上除了跑着操作系统，必然会安装一些应用程序或数据库，运维工程师每天需要查看
    <a href="http://www.olinux.org.cn/linux/" rel="nofollow noopener noreferrer" target="_blank" title="linux">
     linux
    </a>
    系统上运行着的应用程序或数据库状态是否正常。
    <br/>
    4.备份
    <br/>
    运维工程师的看家本事，
    <a href="http://www.olinux.org.cn/mysql/154.html" rel="nofollow noopener noreferrer" target="_blank" title="MySQL数据库备份和还原的常用命令">
     数据库备份和恢复
    </a>
    ，一般来说只要给数据库制定了备份策略它会自己备份，你只需要监控备份任务是否执行了就可以。
    <br/>
    5.服务器调优
    <br/>
    这个要求就比较高了，linux随着使用时间的增长，状态会有所下降，运维工程师有能力的可以对操作系统及数据库进行性能调优，保证系统处于一个最佳状态。
    <br/>
    一般来说，
    <a href="http://www.olinux.org.cn/" rel="nofollow noopener noreferrer" target="_blank" title="linux运维工程师">
     linux运维工程师
    </a>
    在服务器架稳定之后，运维工程师工作主要以
    <a href="http://www.olinux.org.cn/monitoring/290.html" rel="nofollow noopener noreferrer" target="_blank" title="cacti安装流程及如何用cacti添加监控tcp conncactions">
     监控
    </a>
    为主，出现问题的时候才会进行处理
    <br/>
    Linux运维工程师岗位技能
    <br/>
    1、对Linux下各种网络服务、应用系统、监控系统等进行自动化脚本开发的工作，并根据项目对系统进行性能优化；
    <br/>
    2、负责网站项目中Linux服务器的部署与维护，解决Linux系统下版本兼容性问题；
    <br/>
    3、精通linux操作系统，熟练部署和维护Linux服务器以及在linux服务器上架设各种服务；
    <br/>
    4、熟练编写shell script脚本；
    <br/>
    5、熟悉TCP/IP协议；
    <br/>
    6、熟练LAMP,LNMP以及
    <a href="http://www.olinux.org.cn/mysql/" rel="nofollow noopener noreferrer" target="_blank" title="MySQL数据库">
     Mysql
    </a>
    ,oracle数据库维护
