    <p>
     在 Linux 系统中，环境变量和 shell 启动脚本的配置对于系统的正常运行和用户的个性化设置至关重要。
     <code>
      /etc/profile
     </code>
     、
     <code>
      ~/.bashrc
     </code>
     和
     <code>
      ~/.bash_profile
     </code>
     是三个核心的配置文件，它们各自承担着不同的角色和作用。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a8a86726caae418a919c807bc8b33d5a.png#pic_center"/>
    </p>
    <h3>
     <a id="etcprofile_19">
     </a>
     <code>
      /etc/profile
     </code>
     ：系统级环境配置
    </h3>
    <p>
     <code>
      /etc/profile
     </code>
     是 Linux 系统中的系统级配置文件，它对所有用户都有效。这个文件的主要作用是定义系统范围内的环境变量和 shell 启动脚本，确保系统在启动时能够正确地设置环境。
    </p>
    <h4>
     <a id="_23">
     </a>
     加载时机
    </h4>
    <p>
     <code>
      /etc/profile
     </code>
     在用户登录系统时被加载，无论是通过图形界面还是命令行界面。这意味着，它在系统启动过程中起着基础性的作用，为后续的用户操作提供必要的环境设置。
    </p>
    <h4>
     <a id="_27">
     </a>
     作用
    </h4>
    <p>
     <code>
      /etc/profile
     </code>
     通常包含一些基本的环境变量设置，如
     <code>
      PATH
     </code>
     、
     <code>
      SHELL
     </code>
     、
     <code>
      USER
     </code>
     、
     <code>
      LOGNAME
     </code>
     、
     <code>
      HOME
     </code>
     等。这些变量对于系统的正常运行至关重要。此外，
     <code>
      /etc/profile
     </code>
     还负责调用
     <code>
      /etc/profile.d/
     </code>
     目录下的所有脚本，这些脚本可以进一步添加或修改环境变量，或者设置其他的系统级配置。
    </p>
    <h3>
     <a id="bash_profile_31">
     </a>
     <code>
      ~/.bash_profile
     </code>
     ：用户级登录配置
    </h3>
    <p>
     <code>
      ~/.bash_profile
     </code>
     是用户的个人配置文件，它只对当前用户有效。这个文件在用户登录系统时被加载，用于设置用户特定的环境变量和 shell 启动脚本。
    </p>
    <h4>
     <a id="_35">
     </a>
     加载时机
    </h4>
    <p>
     当用户通过命令行登录系统时，
     <code>
      ~/.bash_profile
     </code>
     会被加载。如果该文件不存在，系统会尝试加载
     <code>
      ~/.bash_login
     </code>
     或
     <code>
      ~/.profile
     </code>
     。这种设计允许用户根据自己的需要定制登录时的环境设置。
    </p>
    <h4>
     <a id="_39">
     </a>
     作用
    </h4>
    <p>
     <code>
      ~/.bash_profile
     </code>
     允许用户设置个人的环境变量、别名、函数和启动时执行的命令。例如，用户可以在这里添加新的路径到
     <code>
      PATH
     </code>
     变量，或者设置一些常用的命令别名，以提高工作效率。
    </p>
    <h3>
     <a id="bashrc_43">
     </a>
     <code>
      ~/.bashrc
     </code>
     ：用户级会话配置
    </h3>
    <p>
     <code>
      ~/.bashrc
     </code>
     也是用户的个人配置文件，它在用户开启一个新的 shell 会话时被加载，无论是通过命令行还是图形界面。
    </p>
    <h4>
     <a id="_47">
     </a>
     加载时机
    </h4>
    <p>
     <code>
      ~/.bashrc
     </code>
     在每次新的 shell 会话开始时被加载，这意味着它对于用户的日常工作流程有着直接的影响。用户可以在其中设置别名、shell 函数、启动时执行的命令等，以优化自己的工作流程。
    </p>
    <h4>
     <a id="_51">
     </a>
     作用
    </h4>
    <p>
     <code>
      ~/.bashrc
     </code>
     的作用与
     <code>
      ~/.bash_profile
     </code>
     类似，但它更侧重于单个 shell 会话的配置。用户可以在其中设置一些在每个 shell 会话中都需要的环境变量和脚本，以确保无论何时开启新的 shell，都能有一个一致的环境。
    </p>
    <h3>
     <a id="_55">
     </a>
     三者之间的关系
    </h3>
    <p>
     <code>
      /etc/profile
     </code>
     、
     <code>
      ~/.bash_profile
     </code>
     和
     <code>
      ~/.bashrc
     </code>
     之间的关系可以概括为：
    </p>
    <ul>
     <li>
      <code>
       /etc/profile
      </code>
      是全局的，影响所有用户。
     </li>
     <li>
      <code>
       ~/.bash_profile
      </code>
      和
      <code>
       ~/.bashrc
      </code>
      是用户的个人配置文件，只影响当前用户。
     </li>
     <li>
      <code>
       ~/.bash_profile
      </code>
      通常在用户登录时被加载，而
      <code>
       ~/.bashrc
      </code>
      在每次新的 shell 会话开始时被加载。
     </li>
     <li>
      在很多系统中，
      <code>
       ~/.bash_profile
      </code>
      会包含一行代码来调用
      <code>
       ~/.bashrc
      </code>
      ，确保无论用户是登录系统还是开启新的 shell 会话，
      <code>
       ~/.bashrc
      </code>
      中的配置都能被加载。
     </li>
    </ul>
