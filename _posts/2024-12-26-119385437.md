    <h2>
     <a id="_1">
     </a>
     项目场景：
    </h2>
    <p>
     <font color="#999AAA">
      公司内部n个服务的日志比较混乱，管理起来很麻烦，准备引入ELK日志管理系统
     </font>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="_6">
     </a>
     问题描述：
    </h2>
    <p>
     <font color="#999AAA">
      在配置ElasticSearch集群的时候报错如下：
      <br/>
      “warning: usage of JAVA_HOME is deprecated, use ES_JAVA_HOME”
     </font>
    </p>
    <p>
     <img alt="报错图" src="https://i-blog.csdnimg.cn/blog_migrate/16fc33b558cc97d0311aa64c1119149c.png"/>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="_18">
     </a>
     原因分析：
    </h2>
    <font color="#999AAA">
     先看字面上的意思：
    </font>
    <p>
     报错里有两个问题：
     <br/>
     1、警告:JAVA_HOME的用法已弃用，请使用ES_JAVA_HOME
     <br/>
     2、Elasticsearch的未来版本将需要Java11，您的java版本不满足这个要求，考虑切换到带有绑定JDK的Elasticsearch发行版
    </p>
    <p>
     我们先查看一下jdk版本及引用路径
     <br/>
     <img alt="查看java版本" src="https://i-blog.csdnimg.cn/blog_migrate/db5c7673e01602a56d66809806bf375f.png">
      <br/>
      可以从图中看到java版本显示是1.8.0_111，虽然不知道这具体是哪个版本，但应该是不高于11。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7dc9d690e6c64ecb7251563e227e57d5.png"/>
     </img>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="_37">
     </a>
     解决方案：
    </h2>
    <font color="#999AAA">
     首先，我们需要引入一个高于11版本的JDK，属性名为“ES_JAVA_HOME”
    </font>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e9c2649941dec6b9051b87a3e7cc2032.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/967a8f184d58e122a117000c070d19bf.png">
       <br/>
       修改完成点击确认，然后配置生效，再次点击启动
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cf930c55aa13bba2a13725cfbda29f26.png"/>
      </img>
     </img>
    </p>
