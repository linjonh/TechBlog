    <h3>
     <a id="_1">
     </a>
     运维面试怎么问问题？
    </h3>
    <blockquote>
     <p>
      来源: DevOpSec公众号
      <br/>
      作者: DevOpSec
     </p>
    </blockquote>
    <p>
     随着我们在团队中角色的转变或者技术的提升，我们可能会面试运维候选人。
    </p>
    <p>
     下面我从四个方面介绍一下作为运维面试官一般都会问什么问题。
    </p>
    <p>
     四个方面面试问题适用高中低级别的岗位，可以选择性提问一些问题根据候选人掌握的情况判定候选人应匹配的岗位和薪资。
    </p>
    <p>
     这四个方面同样也适用其他岗位。
    </p>
    <p>
     如果大家需运维相关岗位面试题模版，请关注公众号：DevOpSec，回复
     <code>
      面试
     </code>
     获取。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bcaef1cd9f547f7ffff937c7e671e061.png">

    </p>
    <h3>
     <a id="_18">
     </a>
     一、知识层面
    </h3>
    <p>
     知识层面必须过关，根据掌握层度来决定高中低
    </p>
    <blockquote>
     <ol>
      <li>
       linux/shell/grep/awk/sed
      </li>
      <li>
       JVM 调优
      </li>
      <li>
       数据库/缓存/Hadoop生态/elk
      </li>
      <li>
       TCP/IP\HTTP协议？一次 web请求的详细流程？
      </li>
      <li>
       Python/django/go
      </li>
      <li>
       saltstack/ansible/自动化
      </li>
      <li>
       容器化/docker/k8s
      </li>
      <li>
       "部署、发布方式（预发布、灰度、蓝绿、怎么做到平滑上线等）、业务架构（HA、LB、易维护性）、运维安全（端口、fw、出入口 ip收紧、边界安全、hids）、运维锦囊（重启、扩容、切db）、研发常用手段：鲁棒性原则、限流、熔断、cache
      </li>
      <li>
       “系统负载高排查流程（工具、监控、思路）？
       <br/>
       规范：故障处理流程（复盘）、应急预案、cmdb、各种规范（dns、nginx、模块等），ITIL 理解”
      </li>
      <li>
       监控（基础监控、业务监控、组件监控等），告警处理流程
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="_32">
     </a>
     二、经验层面
    </h3>
    <p>
     看在项目中的角色，重点项目管理、闭环工作能力，根据工作经验和下面问题回答情况决定高中低
    </p>
    <blockquote>
     <ol>
      <li>
       开发过什么平台拿一个讲讲？
       <br/>
       平台的设计、功能以及功能上线时间是？
       <br/>
       与团队成员怎样分工的？
      </li>
      <li>
       平台开发过程中遇到困难的问题列举3个，分别是怎样解决的？
      </li>
      <li>
       最后平台上线达到了什么效果？
       <br/>
       通过这次平台开发你最大的收获是什么？
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="_41">
     </a>
     三、能力层面
    </h3>
    <p>
     主要考察他的沟通、责任心、风险意识、主动性等，根据问题回答情况决定高中低
    </p>
    <blockquote>
     <ol>
      <li>
       谈一个你最近跨部门或者部门内协作的例子？你是怎样让其他人或对方 get 你的 idea的？（协调任务、计划）
       <br/>
       在这过程中有遇到过什么问题吗？你是怎样解决的？最后任务完成的怎样？
      </li>
      <li>
       通过『经验-开发平台看其对项目负责或者是参与的情况』？对本岗位的期望和职业规划？（判断其对自我认知、个人成长期望、自我负责）
      </li>
      <li>
       "讲一次你变更的例子，说说详细流程？（测试、预发布、灰度、全量）你是怎样保证不出问题的？（dubble check、灰度、监控）出了问题怎样做？（备份、恢复方案）
      </li>
      <li>
       有没有你自认为做的很好的工作，在领导面前却被说的一文不值或是得不到肯定。你是怎样处理的？（你已经有很多任务了，领导仍然给你分配任务，你是怎样做的？）
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="_50">
     </a>
     四、动力层面
    </h3>
    <p>
     主要是考察他的持续学习、是否喜欢挑战性工作，是否厌恶重复性工作。根据问题回答情况决定高中低
    </p>
    <blockquote>
     <ol>
      <li>
       你最近一个月看了哪些书，选一本讲讲你的理解？
       <br/>
       最近的新技术有没有在工作中应用？
      </li>
      <li>
       在工作中我们经常会遇到我们不熟悉的或不知如何下手的任务，能讲一个你是怎样处理这类任务的例子吗?
       <br/>
       你是怎样完成这个任务的，最后结果怎样？
       <br/>
       处理这类问题给你的乐趣是什么？烦恼是什么？
      </li>
      <li>
       都知道工作中时常会有大量的重复工作，能讲一个你处理重复工作的例子吗？（消极star）
      </li>
     </ol>
    </blockquote>
