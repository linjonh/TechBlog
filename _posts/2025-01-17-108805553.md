    <h2>
     <a id="HCL_2">
     </a>
     安装HCL有感
    </h2>
    <p>
     (这篇是以前写的，由于当时不熟悉这种文本编辑方式，又因安装时出现了太多错误，花费了大量的时间，导致积累实验作业较多，当时只写了一半，一直在草稿箱放着，最后不了了之。)原文如下：
     <br/>
     由于课程学习的需求，需要在笔记本上安装HCL，本以为安个软件有什么难的，可是现实很残酷，从安装到能正常使用，我重装了软件n次，还更改了用户名，添加了新用户，还学会了安装虚拟机，删除注册表（中间还搞蓝屏了），一顿无用的骚操作之后，深深的认识到我是一个废物，而且运气超好，人家顶多遇到一个两个错误，而我能把错误快给收集齐了。内容较多且有些杂乱，可直接查看问题对应解决方法。
    </p>
    <h3>
     <a id="_7">
     </a>
     遇到的问题汇总
    </h3>
    <h4>
     <a id="1see_the_logfilefor_details_9">
     </a>
     1，see the logfile…for details
    </h4>
    <p>
     如图：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/52c5b86e70665eea485dffb58071eef6.png#pic_center">
      <br/>
      <strong>
       解决方法：
      </strong>
      <br/>
      右键HCL-属性-兼容性；把HCL和VB设置为以Windows7兼容模式进行。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8c0002e2c67ab6a95c07ee31f333b07d.png#pic_center"/>
     </img>
    </p>
    <h4>
     <a id="2Virtual_Box_API_16">
     </a>
     2，请检查Virtual Box API是否安装正确
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d3d9832f36f73ea1c1f174bc531398b4.png#pic_center"/>
    </p>
    <h5>
     <a id="_18">
     </a>
     解决方法
    </h5>
    <p>
     出现此情况是由于HCL与当前系统安装的virtual box不兼容造成的；解决方式为完全卸载当前的virtual Box，重新安装，建议版本为Virtual Box 4.2.4。注意要卸载干净，否则会出现安装失败的情况。
    </p>
    <h4>
     <a id="3ASCLLhcl_22">
     </a>
     3，当前系统用户名中包含非ASCLL字符，hcl启动失败
    </h4>
    <h5>
     <a id="_23">
     </a>
     解决过程中的各种试错
    </h5>
    <p>
     <img alt="当前系统用户名中包含非ASCLL字符，hcl启动失败" src="https://i-blog.csdnimg.cn/blog_migrate/8f31be0110740e714bb6686eeb03f2a9.png#pic_center">
      <br/>
      出现此种问题的原因一般是当前系统的用户名出现了中文或其他的不合法字符；解决方法核心为更改系统用户名，通过百度收集有以下两种方式（我运气太好两种都尝试失败，有兴趣可以自行尝试）：
      <br/>
      可能是我电脑系统版本的原因（Windows 10 家庭中文版），这个地方就开始麻烦了。提示用户名不合法就去查询怎么改用户名，发现了两种方式：
      <br/>
      第一种成功修改用户名，但依然弹出此提示。
     </img>
    </p>
    <ol>
     <li>
      <a href="https://jingyan.baidu.com/article/eb9f7b6d493764869364e815.html" rel="nofollow">
       怎么样修改win10账户名称 如何更改系统用户名
      </a>
      .
     </li>
     <li>
      <a href="https://jingyan.baidu.com/article/7f766daff0df364100e1d057.html" rel="nofollow">
       如何更改电脑账户名称
      </a>
      .
      <br/>
      第二种方法由于我的电脑操作系统版本为家庭中文版，功能受限无法进行相关操作。所以我使用了方法一进行修改。
     </li>
    </ol>
    <p>
     根据方法1修改完账户名后进入HCL界面依旧显示“系统用户名中包含非ASCLL字符，hcl启动失败。”启动失败后，继续查找原因，然后发现C盘用户文件夹下依然存在未修改前的文件夹，考虑是否是这个地方出现了问题。然后继续各种搜索…
    </p>
    <hr/>
    <p>
     经过多次疯狂搜索后，在知乎上找到了我需要的解决方法：
    </p>
    <p>
     <a href="https://www.zhihu.com/question/37082172" rel="nofollow">
      win10如何更改c盘下的用户名？
     </a>
     <br/>
     这一个帖子的前两个回答，刚好解决了我想继续更改C盘下用户名的问题。附上链接：
     <br/>
     <a href="https://www.zhihu.com/question/37082172/answer/84178985" rel="nofollow">
      win10如何更改c盘下的用户名？ - Gdeer的回答 - 知乎
     </a>
     <br/>
     <a href="https://www.zhihu.com/question/37082172/answer/165729732" rel="nofollow">
      win10如何更改c盘下的用户名？ - 知乎用户的回答 - 知乎
     </a>
     <br/>
     巧合的是第二位博主是参考第一位博主提供的方式下继续完善的，两相参考下应该可以解决。然而我在这个地方的操作就比较下饭了。。
     <br/>
     可能是花费的时间太久，有些烦躁，两位博主的方式基本一致就是步骤前后上有差别，我急于设置，没有太过注意以至于把中间一步的注销切换用户给省略了，，然后一通操作，在当前用户下修改当前用户的文件名（
     <em>
      请勿尝试
     </em>
     一定要记得注销后登录）。电脑禁不住我的魔幻操作以致崩溃，喜提蓝屏。。
     <br/>
     最后怎么改过来的怎么改了回去。蓝屏解决后重新老老实实按步骤操作，仍然弹出提示“当前系统用户名中包含非ASCLL字符，hcl启动失败”，
    </p>
    <hr/>
    <h5>
     <a id="_49">
     </a>
     解决方法
    </h5>
    <p>
     由于又尝试几次依旧如此，在各种帖子里面翻也没有找到更好的办法，很是自闭。就在电脑前重新梳理了一下思路 ：当前系统用户名中包含非ASCLL字符，hcl启动失败 那换个系统怎么样呢？怎么换一个系统呢？换一台电脑吗？ 好像没那个实力，，那换一个操作系统呢？装一个双系统？或者安装一个虚拟机？
     <br/>
     想到这里，便准备安装一个Windows 7 版本的虚拟机，于是开始下载Windows7操作系统的镜像文件，在下载等待的过程中，忽然想到是系统用户名非法，既然更改系统用户名失败，可以重新创建一个用户啊！
     <br/>
     由于在上面更改用户名是已经创建过一个用户，就直接登录了那一个用户界面。重新运行安装包，属性里面改兼容性，安装virtual box 4.2.4，终于成功启动。
    </p>
    <h4>
     <a id="4HCL_53">
     </a>
     4，HCL设备命令行输入没反应
    </h4>
    <p>
     成功进入软件之后，我以为到这里已经完美结束，可现实很残酷。
     <br/>
     在第一次做实验时根据拓扑图创建并连接设备，然后小心翼翼的点击启动设备，竟然很顺利的启动成功了（若启动失败，则可能是virtual box与hcl的兼容性问题），继续实验时发现设备的命令窗口打开后无法输入，一开始以为是设备启动过慢的原因，可等了好一会依然是无法输入。
    </p>

     <a id="_56">

     360晶核防护引擎
