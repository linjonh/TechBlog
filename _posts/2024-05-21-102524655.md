    <h2>
     <a id="_4">
     </a>
     测试流程
    </h2>
    <p>
     在做测试的时候，主要有以下几个步骤
    </p>
    <ol>
     <li>
      了解测试内容的原理或架构
     </li>
     <li>
      测试前，先将测试流程（要测什么、怎么测）写出来。在测试过程中，尽量使用控制变量法，避免多个因素的影响。
     </li>
     <li>
      记录测试流程与结果
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="1__12">
     </a>
     1. 测试内容的原理与架构
    </h3>
    <p>
     测试定义：实际输出与预期输出间的比较过程
     <br/>
     测试内容：正确性、完整性、安全性、总体质量
     <br/>
     测试方法：静态测试、动态测试、黑盒测试、白盒测试
     <br/>
     测试策略：单元测试、集成测试
    </p>
    <hr/>
    <h3>
     <a id="2__20">
     </a>
     2. 具体测试流程
    </h3>
    <p>
     静态测试：指软件代码的静态分析测验
     <br/>
     动态测试：检测软件运行中出现的问题
     <br/>
     黑盒测试：黑盒环境测试功能
     <br/>
     白盒测试：根据软件内部源码过程进行测试
    </p>
    <p>
     单元测试：将软件分解为多个单元进行测试
     <br/>
     集成测试：将部分需测试部分作为整体进行集成，随后针对此类集成部分进行测试
    </p>
