    <h2>
     详解axios四种传参，后端接参
    </h2>
    <h3>
     方式一、通过data字段将json数据发送到后台（用","发送数据）
    </h3>
    <pre><code class="hljs">//格式：axios.delete(服务器链接,{data:{键:值}}）

axios.delete("/delete" , {data:{aid:row.aid}}).then(resp =&gt; {
    let resultInfo = resp.data;
    
})</code></pre>
    <p>
     前端浏览器发送的数据
    </p>
    <p>
     <img alt="" height="109" src="https://i-blog.csdnimg.cn/blog_migrate/eea0846765a0951ba55b213642a0e882.png" width="362"/>
    </p>
    <p>
    </p>
    <p>
     后端接参
    </p>
    <p>
     用
     <strong>
      @RequestBody
     </strong>
     指定接收的是
     <strong>
      json
     </strong>
     格式的参数，然后参数类型是
     <strong>
      Map类型
     </strong>
     ，通过map的键取出数据。
    </p>
    <pre><code class="hljs">/* 格式:
		@DeleteMapping("/delete")
		@RequestBody Map&lt;String,前端发送的值的数据类型&gt; 形参名
*/

@DeleteMapping("/delete")
public ResultInfo delete(@RequestBody Map&lt;String,Object&gt; params) {
    System.out.println(params.get("aid"));
    return new ResultInfo(true);
}</code></pre>
    <p>
     后端服务器接收的数据：{aid=11}
    </p>
    <h3>
     方式二、通过params字段将json数据发送到后台（用","发送数据）,实际上是在网址上做拼接。
    </h3>
    <pre><code class="hljs">// 格式：axios.delete(服务器链接,{params:{键:值}}）

axios.delete("/delete" , {params:{aid:row.aid}}).then(resp =&gt; {
    let resultInfo = resp.data;
    
})</code></pre>
    <p>
     前端浏览器发送的数据
    </p>
    <p>
     <img alt="" height="84" src="https://i-blog.csdnimg.cn/blog_migrate/e9fedc902a6b4fd79b130e0f9b7841d5.png" width="310"/>
    </p>
    <p>
     后端接收：
    </p>
    <p>
     用
     <strong>
      @RequestBody
     </strong>
     指定接收的是
     <strong>
      json
     </strong>
     格式的参数，然后参数可以
     <strong>
      通过名字自动匹配
     </strong>
     。
    </p>
    <pre><code class="hljs">@DeleteMapping("/delete")
public ResultInfo delete(Integer aid) {
	System.out.println(aid);
    return new ResultInfo(true);
}</code></pre>
    <p>
     <strong>
      参数名不一致
     </strong>
     则用
     <strong>
      @RequestParam("前端传过来的参数名")
     </strong>
     做匹配
    </p>
    <pre><code class="hljs">@DeleteMapping("/delete")
public ResultInfo delete(@RequestParam("aid") Integer aaid) {
    System.out.println(aaid);
    return new ResultInfo(true);
}</code></pre>
    <p>
     后端服务器接收的数据：107
    </p>
    <h3>
     方式三、直接在地址栏上拼接数据，且必须在拼接前加"/"
    </h3>
    <pre><code class="hljs">//格式：axios.delete("/服务器链接/"+值）

axios.delete("/delete/"+row.aid).then(resp =&gt; {
    let resultInfo = resp.data;
    if (resultInfo.success) {
        this.$message({
            type: 'success',
            message: resultInfo.message
        });
    }
})</code></pre>
    <p>
     客户端
    </p>
    <p>
     <img alt="" height="81" src="https://i-blog.csdnimg.cn/blog_migrate/d5792b9c8132c4646a6999fe673d7001.png" width="326"/>
    </p>
    <p>
     后端接收（服务器）
    </p>
    <p>
    </p>
    <pre><code class="hljs">// 通过在 /{参数名} 得到访问方法，然后取值用@PathVariable("参数名")

@DeleteMapping("/delete/{aid}")
public ResultInfo delete(@PathVariable("aid") Integer aid) {
    System.out.println(aid);
    return new ResultInfo(true);
}</code></pre>
    <p>
     <img alt="" height="62" src="https://i-blog.csdnimg.cn/blog_migrate/be78f2cc7a392e00a941642fef57d874.png" width="176"/>
    </p>
    <h3>
     方式四、传统方式?参数名=参数值
    </h3>
    <p>
    </p>
    <pre><code class="hljs">axios.delete("/delete?aid="+row.aid).then(resp =&gt; {
    let resultInfo = resp.data;
})</code></pre>
    <p>
     <img alt="" height="76" src="https://i-blog.csdnimg.cn/blog_migrate/d6be32af289096c31abbf0aba5ee4767.png" width="287"/>
    </p>
    <p>
     服务器直接接收，
     <strong>
      按名字匹配
     </strong>
     ，如果名字不一致，则用
     <strong>
      @RequestParam
     </strong>
     进行匹配
    </p>
    <pre><code class="hljs">@DeleteMapping("/delete")
public ResultInfo delete(Integer aid) {
    System.out.println(aid);
    return new ResultInfo(true);
}</code></pre>
    <p>
     <img alt="" height="58" src="https://i-blog.csdnimg.cn/blog_migrate/a44d6bdaf2841084951beb48ed12beb7.png" width="180"/>
    </p>
    <h4>
     <strong>
      注： get和delete几乎完全一致
     </strong>
    </h4>
