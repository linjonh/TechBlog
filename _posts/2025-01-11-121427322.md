    <h3>
     <a id="_0">
     </a>
     摘要
    </h3>
    <p>
     一个新的时代正在到来。这是一个以数字化互动为主导的世界，公司
     <strong>
      能否采用软件驱动型业务模式将成为决定其成败的关键因素
     </strong>
     。这一点至关重要，因为软件是策略的核心所在，也是公司与客户互动的主要手段之一。
     <br/>
     从出租车和出行到银行、零售甚至养牛业，几乎各行各业都在经历着根本性的变革和创新。软件驱动型初创公司给许多行业带来了颠覆性的改变，迫使原来的公司不得不去适应瞬息万变的业务环境。为了谋求发展并保持竞争力，企业正在通过将软件应用纳入其核心业务策略来重掌技术。在这个过程中，他们必须
     <strong>
      一边优化现有应用
     </strong>
     ，
     <strong>
      一边利用全新的现代化开发技术
     </strong>
     。
     <br/>
     企业也在寻求云解决方案，因为云具有
     <strong>
      加快应用交付、降低成本以及灵活的可扩展性
     </strong>
     等种种优势。当然，这也进一步増加了IT应用环境的复杂性。软件驱动的业务转型既需要速度，也需要高质量的应用交付。在以速度作为核心竞争力的背后，是増长潜力和竞争压力起着推波助澜的作用。同时，质量也比以往任何时侯都更加重要。客户对应用驱动型业务体验的期望与之前的消费者应用体验一样。如果想要在开发和交付软件应用方面做得更好，企业就需要一种新的方法。他们需要一种能让他们具有云运营模式的速度和效率的策略。这种方法能够让企业在任何云(如公共云、私有云和混合云)上构建、运行和改进响应式应用。
     <br/>
     解决方案就是我们所说的
     <strong>
      云原生开发
     </strong>
     ——云原生应用利用云计算模型来
     <strong>
      加快应用开发和交付速度，提高部署灵活性和质量，同时降低部署风险
     </strong>
     。这种方法让组织在充分利用容器和微服务等创新技术的同时，还能从敏捷开发、 Devops和持续集成与持续部署(CI/CD)等颠覆性实践中获益。
    </p>
    <h3>
     <a id="_6">
     </a>
     云原生应用开发技术能解决的客户挑战
    </h3>
    <p>
     <strong>
      第一项挑战是客户需要一个值得信赖、技术领先、基于标准的云原生平台，该平台应具有协同工作能力并提供一致的开发人员体验
     </strong>
     。这样一来，他们可以将更多时间用来构建应用，而不必再对基础架构和配置投入太多精力。
     <br/>
     其次，数字化转型需要用到云、移动、开源软件和大数据管理等技术。公司需要充分利用现有的软件和应用投资。但是，大多数现有系统并非为云和移动应用而设计。他们会发现很难从现有应用中访问和使用相应的数据和服务。
     <strong>
      第二项重大挑战就是他们需要连接、扩展现有应用并将现有应用现代化，以适应不断变化的需求。
     </strong>
     <br/>
     第三项重大挑战是他们需要在采用新兴技术和新工具与现有流程、团队和文化之间进行协调，以达到可观的投资回报率，尽可能从新技术投资中获得最大价值。
    </p>
    <h3>
     <a id="_11">
     </a>
     云原生应用开发概念
    </h3>
    <p>
     云原生（Cloud-native）应用开发是一种构建和运行应用以充分利用云计算模型优势的方法，即创建响应式、有弹性且有恢复能力的应用。云原生应用开发使企业能够在现代化的动态环境(如公共云、私有云和混合云)中构建和运行可扩展应用。
    </p>
    <h3>
     <a id="_15">
     </a>
     特性
    </h3>
    <ul>
     <li>
      打包为
      <strong>
       容器
      </strong>
     </li>
     <li>
      <strong>
       由平台编排
      </strong>
      ，能够在任何云基础架构上运行
     </li>
     <li>
      采用
      <strong>
       微服务
      </strong>
      的原理
     </li>
     <li>
      使用诸如
      <strong>
       持续交付和 Devops
      </strong>
      等实践进行开发
     </li>
    </ul>
    <p>
     这些技术和方法可以为系统提供有弹性、可管理且可监控的应用。当这些应用与强大的自动化功能结合在一起时，IT团队(“开发”和“运维”双方协作)便能够以最低风险频繁地进行有重大影响力的改变。
    </p>
    <h3>
     <a id="_24">
     </a>
     优势
    </h3>
    <ul>
     <li>
      <strong>
       以客户为中心，通过持续改进不断加快创新速度
      </strong>
      <ul>
       <li>
        云原生开发可实现快速的持续改进，从而实现客户至上、及时满足客户需求的业务模式。
       </li>
       <li>
        云原生开发会充分利用工具、AB测试和分析来确定客户将如何使用他们的应用以及客户意图。这样可以快速、有效地提高企业的创新能力。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       偏重速度，持续交付新应用并优化现有应用
      </strong>
      <ul>
       <li>
        对于许多公司而言，想要跻身颠覆领先行列，不仅要能够交付新应用，还能优化当前应用和系统。減少团队文化中的阻力、细化工作并实现环境的自动化，将有助于全面提高软件开发的速度，而不单单局限于新功能和新应用。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       逐步攻克式应用开发
      </strong>
      <ul>
       <li>
        开发团队的构建和发布速度越快，他们就会越偏向于以增量组件或微服务（而非功能完善的应用）的方式来构建软件。
       </li>
       <li>
        今天我们认定的单个应用，明天将归类为精细微服务的集合，而且主要是组合而成，而不是通过编写代码构建的应用。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       通过自动化降低风险和成本，同时提高速度和质量
      </strong>
      <ul>
       <li>
        不一致会带来风险。手动活动（尤其是快速完成的活动）会造成不一致，进而导致质量下降和故障。
       </li>
       <li>
        企业可以借助 Devops实践和工具来实现大部分应用交付流程的自动化，以此消除交接和等待时间，从而提高速度和质量，同时降低成本和返工率。过去，速度和质量无法兼得，但Devops变革彻底粉碎了这种两难局面。现在，两者可以兼而有之。
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="_36">
     </a>
     云原生开发主要为了加快应用的开发和交付，从而做到
    </h3>
    <ul>
     <li>
      构建新的应用和服务，以支持业务创新和转型
     </li>
     <li>
      对现有应用进行现代化改造，以从现有T投资中挖掘新价值
     </li>
     <li>
      将新的应用和现有的应用连接起来，以推动创新、提高效率并获得竞争优势
     </li>
    </ul>
    <h4>
     <a id="_40">
     </a>
     云原生应用开发的技术基础基于以下四个关键技术和方法
    </h4>
    <ul>
     <li>
      <strong>
       容器和编排
      </strong>
      <ul>
       <li>
        与标准虚拟机(VM)相比，容器具有更高的效率、密度和速度，因而成为交付云原生应用的理想部署单元和自包含执行（计算和存储）环境。编排提供了用于管理容器化工作负载和服务的工具，并可实现自动化。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       DevOps自动化和持续交付
      </strong>
      <ul>
       <li>
        指导构建应用及运行该应用所需的流程、协作和工具。从最初的想法到生产发布，再到客户反馈，最后根据用户反馈增强功能，所有步骤都是自动完成的，从而实现持续创新。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       微服务和服务网格
      </strong>
      <ul>
       <li>
        松散耦合的系统更易于构建、测试、部署和更新。即便不是每个云原生项目都从微服务开始，但最终的架构还是基于微服务的。通过转移用于协调服务间通信的应用代码，并将它们作为可配置基础架构组件提供，服务网格可简化微服务开发。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       应用编程接口(API)和API管理
      </strong>
      <ul>
       <li>
        API是构成云原生应用的服务的默认接口，用于交换数据和功能。API简化了应用的组合、测试和维护。此外，API管理可在公司的API与使用这些API的开发人员、客户、合作伙伴和员工之间提供安全中介和策略实施服务。
       </li>
      </ul>
     </li>
    </ul>
