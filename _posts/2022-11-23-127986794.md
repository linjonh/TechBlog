    </p>

    <h2>
     <a id="_Cookie_9">
     </a>
     二. Cookie
    </h2>
    <h4>
     <a id="1_Cookie_10">
     </a>
     1. 为什么会有Cookie
    </h4>
    <p>
     web应用程序是使用HTTP协议传输数据的，然而HTTP协议是无状态的协议，一旦数据交换完毕，客户端与服务器端的连接就会关闭，再次交换数据需要建立新的连接。这就意味着服务器无法从连接上跟踪会话。
     <br/>
     你可能会有这样的经历，登录一个网站的时候会提醒你要不要记住账户和密码，这样下次来你就不用再次输入账号和密码了。这就是cookie的作用，当我们再次访问的时候，方便服务器直接根据我们的Cookie来直接取上一次去过的东西（对于，每一个cookie服务器会对这个cookie存储上一次我们拿过的数据，下一次对于同一个cookie的时候，就直接在这里取。
    </p>
    <h4>
     <a id="2_Cookie_13">
     </a>
     2. Cookie的特性
    </h4>
    <p>
     当服务器收到 HTTP 请求时，服务器可以在响应头里设置一个 Set-cookie 选项，浏览器接收到响应后会自动保存下 Cookie，之后浏览器对该服务器的每一次请求中都会通过请求头把 Cookie 信息发送给服务器，这就是Cookie的来源。
     <br/>
     当然Cookie在浏览器中也是有个数和大小的限制，大小一般是4k，个数的问题每种浏览器的限制都不同，可以参考下面的表格。
    </p>

    <p>
     <strong>
      值得注意的是
     </strong>
     :Cookie存储信息不安全（不能存放重要的信息）,Cookie中有域（domain）和路径的概念,浏览器是一个比较安全的环境，所以不同的域之间不能直接访问（js的同源策略限制）
     <br/>
     <strong>
      PS
     </strong>
     :但是因为浏览器的每次请求都会携带上 Cookie，会带来额外的开销，而且存储量比较小，所以后来浏览器推出了新的 Api(web stoeage Api和 indexedDb)
    </p>
    <h4>
     <a id="3_Cookie_23">
     </a>
     3. 浏览器Cookie
    </h4>
    <p>
     打开浏览器的开发者工具 都可以在【应用】选项卡中看到当前浏览器所存储的Cookie，如下图。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e95de391ce9d068c2574bbde3bece960.png">
      <br/>
      其中字段名含义:
     </img>
    </p>
    <ul>
     <li>
      Name：cookie的名字(键)
     </li>
     <li>
      Value：cookie存放的值
     </li>
     <li>
      Expires：指cookie过期的时间（该时间为世界时间 UTC时间）也称为格林威治时间
     </li>
     <li>
      Max-Age: 指cookie最大的存活有效期（单位：秒）
     </li>
     <li>
      Domain: 指的是cookie当前的域
     </li>
     <li>
      Path:指cookie当前的路径
     </li>
     <li>
      Size: 指cookie存放的大小
     </li>
     <li>
      Secure: 指cooke的安全属性
     </li>
    </ul>

    <h2>
     <a id="_Cookie_82">
     </a>
     四. Cookie属性
    </h2>
    <h4>
     <a id="1_CookieDomain_83">
     </a>
     1. Cookie属性Domain
    </h4>
    <p>
     Domain是用来设置Cookie能够作用到的主机地址，存储在浏览器时不携带协议和端口。因为端口和协议不同并不会影响Cookie但是Cookie的部分属性只有在Https下才能有效，这也是Cookie的特性之一，
     <code>
      不区分协议和端口
     </code>
     。
     <br/>
     如果前面有个. 如下 .baidu.con 就证明当前的Cookie可以作用于当前域及其子域，假如下面的Cookie在 niuma.baidu.com 中是可以访问到的 而前面没有这个.的Cookie在 niuma.baidu.com 中无法访问到 只能在baidu.com 中访问到。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/96b6b6c53dbfafe2055de04e1db28081.png"/>
     <br/>
     在设置Cookie时如果没有设置Domain属性那么这个Cookie只能作用于当前域
     <br/>
     有一个注意的点就是设置Cookie时设置了domain属性向下面
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d6a728a835f6447e988fcec2de6d1bda.png"/>
     <br/>
     在浏览器存储Cookie的地方竟然变成了 .uicool123.com ??? 这是值得大家注意的地方
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a363d582fde53a296958d2f52a5dd4d1.png"/>
     <br/>
     设置domain=.uicool123.com 的时候还是跟上面的情况一样，也就是说两种doamin属性的设置方法是一样的。
     <br/>
     关于domain还有最后一点值得注意，就是在子域中可以给父域设置Cookie
     <br/>
     比如我们在 niu.uicool123.com 中设置Cookie ，doamin属性设置为 .uicool123.com 那么我们可以在 uicool123.com 中获取到这个Cookie也可以在 hhh.uicool123.com 中获取到这个Cookie 因为这个Cookie是作用于uicool123.com和它的子域的。
     <br/>
     但是我们不能在当前域的子域或者跨域设置Cookie，比如我们在 uicool123.com 的控制台设置 Cookie时domain属性不能设置为 hhh.uicool123.com 或者 baidu.com
    </p>
    <h4>
     <a id="2_Cookiepath_96">
     </a>
     2. Cookie属性path
    </h4>
    <p>
     Cookie的path用来指定URL的路径
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0a9b07ebc1a885155c097a6e94ef1fd5.png"/>
     <br/>
     假如我们在a.com/a的控制台设置如下的Cookie
    </p>
    <pre><code class="prism language-javascript">document<span class="token punctuation">.</span>cookie<span class="token operator">=</span><span class="token string">'myname=牛马;path=/a'</span>
</code></pre>
    <p>
     那么只要前缀是 a.com/a 的路径下都可以访问到这个Cookie 比如 a.com/a/b/c a.com/a/b
     <br/>
     而在前缀不是a.com/a 的路径下是无法访问这个Cookie的 比如 a.com/b a.com/b/a
     <br/>
     这个属性用的比较少，一般都是设置为/路径
    </p>
    <h4>
     <a id="3_Cookieexpires_106">
     </a>
     3. Cookie属性expires
    </h4>
    <p>
     expires 属性是来设置Cookie有效期相关的属性，expires 指定Cookie失效的具体时间
     <br/>
     <code>
      注意
     </code>
     : 当与max-age属性同时存在时max-age的优先级会更高
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/26961a90a03d18f0cc9857d1f83808d7.png"/>
     <br/>
     列如设置一个 2022年11月26日凌晨1点10过期的Cookie
    </p>
    <pre><code class="prism language-javascript">document<span class="token punctuation">.</span>cookie<span class="token operator">=</span><span class="token template-string"><span class="token template-punctuation string">`</span><span class="token string">myname=牛马;expires=</span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span><span class="token keyword">new</span> <span class="token class-name">Date</span><span class="token punctuation">(</span><span class="token string">'2022-11-26 01:10:00'</span><span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">toUTCString</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token interpolation-punctuation punctuation">}</span></span><span class="token template-punctuation string">`</span></span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bc834cb1af279a059f38b1090415757d.png"/>
     <br/>
     可以看到已经设置好了
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a0ff20b1b65d21ebe2d2a39835ac4419.png"/>
     <br/>
     这个expires属性也可以设置为过去的时间,生效后浏览器会立即删除这条Cookie
     <br/>
     当一个Cookie既不包含expires也不包含max-age属性那么，这个Cookie就是一个会话Cookie，当你关闭浏览器时这个Cookie会被自动删除，刷新和关闭当前页面是不会删除的。
    </p>
    <h4>
     <a id="4_Cookiemaxage_119">
     </a>
     4. Cookie属性max-age
    </h4>
    <p>
     max-age 属性也是用来设置CooKie有效期相关的属性，max-age 指定Cookie多少秒后失效
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fb1c2a86ddcebe66f1ee93dd06c96398.png"/>
     <br/>
     比如我们设置一个600秒后过期的Cookie
    </p>
    <pre><code class="prism language-javascript">document<span class="token punctuation">.</span>cookie<span class="token operator">=</span><span class="token template-string"><span class="token template-punctuation string">`</span><span class="token string">myname=牛马;max-age=600</span><span class="token template-punctuation string">`</span></span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/55972ebd33fa866814d7520ddbd3081e.png"/>
     <br/>
     可以看到设置成功
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d7c2e411433f174de9a14df1348977fb.png"/>
     <br/>
     <code>
      注意
     </code>
     :当expires 和 max-age属性都设置时max-age的设置会优先，如果max-age设置为负值那就是删除Cookie
     <br/>
     当一个Cookie既不包含expires也不包含max-age属性那么，这个Cookie就是一个会话Cookie，当你关闭浏览器时这个Cookie会被自动删除，刷新和关闭当前页面是不会删除的。
    </p>
    <h4>
     <a id="5_Cookiehttponly_131">
     </a>
     5. Cookie属性httponly
    </h4>
    <p>
     httponly 是和Cookie安全性紧密相关的一个Cookie，上面也有提及到。
     <br/>
     只要HttpOnly打钩的js是无法获取和操作的，这种Cookie是只能由服务端和浏览器来操作，通常这种Cookie是用来存储一些用户身份信息或者安全相关的凭证的。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b07e5f0d9bbfe9372ee0bf753591c691.png"/>
     <br/>
     这种Cookie只能通过服务端响应设置
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/95cf829a5a3919eb81032cb3256d50fe.png"/>
     <br/>
     在浏览器设置这个属性的Cookie时这个Cookie会被浏览器直接忽略掉
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/427d38bebbdeb6ad6a4c508624913c6a.png"/>
    </p>
    <h4>
     <a id="5_Cookiesecure_139">
     </a>
     5. Cookie属性secure
    </h4>
    <p>
     secure 这个属性用来告诉浏览器这个Cookie只能用https协议传输，如果服务端的响应中包含有secure属性的Cookie，但是当前的协议是http的，那么浏览器会直接忽略这个Cookie
     <br/>
     当我们在浏览器创建了一个带有secure的属性
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f6ef13b32272ee3d55f753ff437f7201.png"/>
     <br/>
     那么当我们向https://a.com 发起请求时就会携带上这个Cookie
     <br/>
     而当我们请求http://a.com时的时候是不会携带这个Cookie的
    </p>
    <h4>
     <a id="6_Cookiesamesite_145">
     </a>
     6. Cookie属性samesite
    </h4>
    <p>
     关于samesite这是一个变动比较大的属性，samesite属性可以限制跨站请求时Cookie发送，samesite的值有三个
    </p>
    <table>
     <thead>
      <tr>
       <th>
        samesite属性值
       </th>
       <th>
        作用
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>

       </td>

        跨站不发送Cookie
       </td>

     关于samesite属性为None的Cookie，虽然约束性最弱但是只有这个cookie是HTTPS协议进行传输的时候浏览器才认为它是有效的另外这个cookie.必须要添加secure属性，否则无法保存。
