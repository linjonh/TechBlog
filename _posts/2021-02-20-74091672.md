      <strong>
       1.2 测试周期
      </strong>
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      测试周期可按项目的开发周期来确定测试时间，一般测试时间为两三周（即15个工作日），根据项目情况以及版本质量可适当缩短或延长测试时间。正式测试前先向主管确认项目排期。

     </a>
     1.3测试资源
    </h3>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      测试任务开始前，检查各项测试资源。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --产品功能需求文档；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --产品原型图；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --产品效果图；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --行为统计分析定义文档；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --测试设备（IOS
      <a class="replace_word" href="http://lib.csdn.net/base/15" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="undefined">
       Android
      </a>
      ）
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --其他。
     </span>
    </p>
    <h3 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t1" style="color:rgb(106,57,6)" target="_blank">
     </a>
     1.4日报及产品上线报告
    </h3>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）测试人员每天需对所测项目发送测试日报。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）测试日报所包含的内容为：
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --对当前测试版本质量进行分级；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --对较严重的问题进行例举，提示开发人员优先修改；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --对版本的整体情况进行评估。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3）产品上线前，测试人员发送产品上线报告。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4）上线报告所包含的内容为：
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      ---对当前版本质量进行分级；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      ---附上测试报告（功能测试报告、兼容性测试报告、性能测试报告以及app可用性能标准结果）；
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --总结上线版本的基本情况。若有遗留问题必须列出并记录解决方案。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <br/>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <strong>
      <span style="font-size:24px; color:rgb(255,0,0)">
       2. App测试点
      </span>
     </strong>
    </p>
    <h3 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t2" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.1安全测试
    </h3>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t3" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.1.1软件权限
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）扣费风险：包括发送短信、拨打电话、连接网络等
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）隐私泄露风险：包括访问手机信息、访问联系人信息等
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3）对App的输入有效性校验、认证、授权、敏感数据存储、数据加密等方面进行检测
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4）限制/允许使用手机功能接入互联网
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5）限制/允许使用手机发送接受信息功能
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6）限制/允许应用程序来注册自动启动应用程序
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7）限制或使用本地连接
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      8）限制/允许使用手机拍照或录音
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      9）限制/允许使用手机读取用户数据
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      10) 限制/允许使用手机写入用户数据
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      11) 检测App的用户授权级别、数据泄漏、非法授权访问等
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t4" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.1.2安装与卸载安全性
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）应用程序应能正确安装到设备驱动程序上
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）能够在安装设备驱动程序上找到应用程序的相应图标
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3）是否包含数字签名信息
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4）JAD文件和JAR包中包含的所有托管属性及其值必需是正确的
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5）JAD文件显示的资料内容与应用程序显示的资料内容应一致
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6）安装路径应能指定
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7）没有用户的允许, 应用程序不能预先设定自动启动
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      8）卸载是否安全, 其安装进去的文件是否全部卸载
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      9）卸载用户使用过程中产生的文件是否有提示
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      10）其修改的配置信息是否复原
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      11）卸载是否影响其他软件的功能
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      12）卸载应该移除所有的文件
     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t5" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.1.3数据安全性
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）当将密码或其他的敏感数据输人到应用程序时,其不会被储存在设备中, 同时密码也不会被解码
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）输人的密码将不以明文形式进行显示
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3）密码, 信用卡明细, 或其他的敏感数据将不被储存在它们预输人的位置上
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4）不同的应用程序的个人身份证或密码长度必需至少在6-12个数字长度之间
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5）当应用程序处理信用卡明细, 或其他的敏感数据时, 不以明文形式将数据写到其它单独的文件或者临时文件中。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6）防止应用程序异常终止而又没有删除它的临时文件, 文件可能遭受入侵者的袭击, 然后读取这些数据信息。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7）当将敏感数据输人到应用程序时, 其不会被储存在设备中
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      8）备份应该加密, 恢复数据应考虑恢复过程的异常、通讯中断等, 数据恢复后再使用前应该经过校验
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      9）应用程序应考虑系统或者虚拟机器产生的用户提示信息或安全警告
      <a name="_GoBack" style="color:rgb(106,57,6)" target="_blank">
      </a>
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      10）应用程序不能忽略系统或者虚拟机器产生的用户提示信息或安全警告, 更不能在安全警告显示前,，利用显示误导信息欺骗用户，应用程序不应该模拟进行安全警告误导用户
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      11）在数据删除之前，应用程序应当通知用户或者应用程序提供一个“取消”命令的操作
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      12）“取消”命令操作能够按照设计要求实现其功能
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      13）应用程序应当能够处理当不允许应用软件连接到个人信息管理的情况
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      14）当进行读或写用户信息操作时, 应用程序将会向用户发送一个操作错误的提示信息
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      15）在没有用户明确许可的前提下不损坏侧除个人信息管理应用程序中的任何内容Μ
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      16）应用程序读和写数据正确。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      17）应用程序应当有异常保护。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      18）如果数据库中重要的数据正要被重写, 应及时告知用户
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      19）能合理地处理出现的错误
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      20）意外情况下应提示用户
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-size:12px">
      <strong>
       2.1.4通讯安全性
      </strong>
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）在运行其软件过程中, 如果有来电、SMS、EMS、MMS、蓝牙、红外等通讯或充电时,是否能暂停程序，优先处理通信, 并在处理完毕后能正常恢复软件, 继续其原来的功能
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）当创立连接时, 应用程序能够处理因为网络连接中断, 进而告诉用户连接中断的情况
     </span>
    </p>

    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6）登录
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --使用合法的用户登录系统。
      <br/>
      --系统是否允许多次非法的登陆，是否有次数限制。
      <br/>
      --使用已经登陆的账号登陆系统是否正确处理。
      <br/>
      --使用禁用的账号登陆系统是否正确处理。
      <br/>
      --用户名、口令（密码）错误或漏填时能否登陆。
      <br/>
      --删除或修改后的用户，原用户登陆。
      <br/>
      --不输入用户口令和用户、重复点（确定或取消按钮）是否允许登陆。
      <br/>
      --登陆后，页面中登陆信息。
      <br/>
      --页面中有注销按钮。
      <br/>
      --登陆超时的处理。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7）注销
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --注销原模块，新的模块系统能否正确处理。
      <br/>
      --终止注销能否返回原模块，原用户。
      <br/>
      --注销原用户，新用户系统能否正确处理。
      <br/>
      --使用错误的账号、口令、无权限的被禁用的账号进行注销
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
    </p>

     </a>
     2.4.2应用的前后台切换
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) APP切换到后台，再回到app，检查是否停留在上一次操作界面。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) APP切换到后台，再回到app，检查功能及应用状态是否正常，IOS4和IOS5的版本的处理机制有的不一样。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) app切换到后台，再回到前台时，注意程序是否崩溃，功能状态是否正常，尤其是对于从后台切换回前台数据有自动更新的时候。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 手机锁屏解屏后进入app注意是否会崩溃，功能状态是否正常，尤其是对于从后台切换回前台数据有自动更新的时候。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5) 当App使用过程中有电话进来中断后再切换到app，功能状态是否正常
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6) 当杀掉app进程后，再开启app，app能否正常启动。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7) 出现必须处理的提示框后，切换到后台，再切换回来，检查提示框是否还存在，有时候会出现应用自动跳过提示框的缺陷。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      8) 对于有数据交换的页面，每个页面都必需要进行前后台切换、锁屏的测试，这种页面最容易出现崩溃。
      <br/>
     </span>
    </p>

     2.4.3免登录
    </h4>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      很多应用提供免登录功能，当应用开启时自动以上一次登录的用户身份来使用
     </span>
     <span style="font-family:SimSun; color:black">
      app.
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      1) app
     </span>
     <span style="font-family:SimSun; color:black">
      有免登录功能时，需要考虑
     </span>
     <span style="font-family:SimSun; color:black">
      IOS
     </span>
     <span style="font-family:SimSun; color:black">
      版本差异。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      2)
     </span>
     <span style="font-family:SimSun; color:black">
      考虑无网络情况时能否正常进入免登录状态。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      3)
     </span>
     <span style="font-family:SimSun; color:black">
      切换用户登录后，要校验用户登录信息及数据内容是否相应更新，确保原用户退出。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      4)
     </span>
     <span style="font-family:SimSun; color:black">
      根据
     </span>
     <span style="font-family:SimSun; color:black">
      MTOP
     </span>
     <span style="font-family:SimSun; color:black">
      的现有规则，一个帐户只允许登录一台机器。所以，需要检查一个帐户登录多台手机的情况。原手机里的用户需要被踢出，给出友好提示。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      5) app
     </span>
     <span style="font-family:SimSun; color:black">
      切换到后台，再切回前台的校验
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      6)
     </span>
     <span style="font-family:SimSun; color:black">
      切换到后台，再切换回前台的测试
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      7)
     </span>
     <span style="font-family:SimSun; color:black">
      密码更换后，检查有数据交换时是否进行了有效身份的校验
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      8)
     </span>
     <span style="font-family:SimSun; color:black">
      支持自动登录的应用在进行数据交换时，检查系统是否能自动登录成功并且数据操作无误。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      9)
     </span>
     <span style="font-family:SimSun; color:black">
      检查用户主动退出登录后，下次启动
     </span>
     <span style="font-family:SimSun; color:black">
      app
     </span>
     <span style="font-family:SimSun; color:black">
      ，应停留在登录界面
     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">

     </a>
     <span style="color:black">
      2.4.4
     </span>
     数据更新
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      根据应用的业务规则，以及数据更新量的情况，来确定最优的数据更新方案。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) 需要确定哪些地方需要提供手动刷新，哪些地方需要自动刷新，哪些地方需要手动+自动刷新。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) 确定哪些地方从后台切换回前台时需要进行数据更新。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) 根据业务、速度及流量的合理分配，确定哪些内容需要实时更新，哪些需要定时更新。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 确定数据展示部分的处理逻辑，是每次从服务端请求，还是有缓存到本地，这样才能有针对性的进行相应测试。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5) 检查有数据交换的地方，均有相应的异常处理。
     </span>
    </p>
    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="color:black">

      很多应用会支持离线浏览，即在本地客户端会缓存一部分数据供用户查看。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) 在无网络情况可以浏览本地数据
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) 退出app再开启app时能正常浏览
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) 切换到后台再切回前台可以正常浏览
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 锁屏后再解屏回到应用前台可以正常浏览
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">

     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">

     </a>
     2.4.6 App更新
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) 当客户端有新版本时，有更新提示。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) 当版本为非强制升级版时，用户可以取消更新，老版本能正常使用。用户在下次启动app时，仍能出现更新提示。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) 当版本为强制升级版时，当给出强制更新后用户没有做更新时，退出客户端。下次启动app时，仍出现强制升级提示。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 当客户端有新版本时，在本地不删除客户端的情况下，直接更新检查是否能正常更新。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5) 当客户端有新版本时，在本地不删除客户端的情况下，检查更新后的客户端功能是否是新版本。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6) 当客户端有新版本时，在本地不删除客户端的情况下，检查资源同名文件如图片是否能正常更新成最新版本。如果以上无法更新成功的，也都属于缺陷。
     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t21" style="color:rgb(106,57,6)" target="_blank">
     </a>
     <span style="color:black">
      2.4.7
     </span>
     定位、照相机服务
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) App有用到相机，定位服务时，需要注意系统版本差异
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) 有用到定位服务、照相机服务的地方，需要进行前后台的切换测试，检查应用是否正常。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) 当定位服务没有开启时，使用定位服务，会友好性弹出是否允许设置定位提示。当确定允许开启定位时，能自动跳转到定位设置中开启定位服务。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 测试定位、照相机服务时，需要采用真机进行测试。
     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t22" style="color:rgb(106,57,6)" target="_blank">
     </a>
     <span style="color:black">
      2.4.8
     </span>
     时间测试
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      客户端可以自行设置手机的时区、时间，因此需要校验该设置对app的影响。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --中国为东8区，所以当手机设置的时间非东8区时，查看需要显示时间的地方，时间是否展示正确，应用功能是否正常。时间一般需要根据服务器时间再转换成客户端对应的时区来展示，这样的用户体验比较好。比如发表一篇微博在服务端记录的是10：00，此时，华盛顿时间为22：00，客户端去浏览时，如果设置的是华盛顿时间,则显示的发表时间即为22:00,当时间设回东8区时间时，再查看则显示为10：00。
     </span>
    </p>
    <h4 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t23" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.4.9PUSH测试
    </h4>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1) 检查push消息是否按照指定的业务规则发送
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2) 检查不接受推送消息时，检查用户不会再接收到push.
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3) 如果用户设置了免打扰的时间段，检查在免打扰时间段内，用户接收不到PUSH。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      在非免打扰时间段，用户能正常收到push。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4) 当push消息是针对登录用户的时候，需要检查收到的push与用户身份是否相符，没有错误地将其它人的消息推送过来。一般情况下，只对手机上最后一个登录用户进行消息推送。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5) 测试push时，需要采用真机进行测试。
     </span>
    </p>
    <h3 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">
     <a name="t24" style="color:rgb(106,57,6)" target="_blank">
     </a>
     2.5性能测试
    </h3>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      评估App的时间和空间特性 ：
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1)极限测试：在各种边界压力情况下，如电池、存储、网速等，验证App是否能正确响应。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      --内存满时安装App
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">

    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">

      <span style="color:rgb(51,51,51)">
       又叫事件或冲突测试，是指一个功能正在执行过程中，同时另外一个事件或操作对该过程进行干扰的测试。
      </span>
      如；App在前/后台运行状态时与来电、文件下载、音乐收听等关键运用的交互情况测试等。
      <span style="color:rgb(51,51,51)">
       交叉事件测试非常重要，能发现很多应用中潜在的性能问题。
      </span>
     </span>

     2.10用户体验测试
    </h3>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      以主观的普通消费者的角度去感知产品或服务的舒适、有用、易用、友好亲切程度。通过不同个体、独立空间和非经验的统计复用方式去有效评价产品的体验特性提出修改意见提升产品的潜在客户满意度。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      1）是否有空数据界面设计，引导用户去执行操作。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      2）是否滥用用户引导。
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      3）是否有不可点击的效果，如：你的按钮此时处于不可用状态，那么一定要灰掉，或者拿掉按钮，否则会给用户误导
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      4）菜单层次是否太深
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      5）交互流程分支是否太多
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      6）相关的选项是否离得很远
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      7）
      <span style="color:black">
       一次是否载入太多的数据
      </span>
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      8
     </span>
     <span style="font-family:SimSun; color:black">
      ）界面中按钮可点击范围是否适中
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun; color:black">
      9
     </span>
     <span style="font-family:SimSun; color:black">
      ）标签页是否跟内容没有从属关系，
     </span>
     <span style="font-family:SimSun; color:black">
      当切换标签的时候，内容跟着切换
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      10）操作应该有主次从属关系
     </span>
    </p>
    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      11）是否定义Back的逻辑。涉及软硬件交互时，Back键应具体定义

    <p style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">
     <span style="font-family:SimSun">
      12）是否有横屏模式的设计，应用一般需要支持横屏模式，即自适应设计

      。

      </span>
      <span style="color:rgb(51,51,51)">
       、空间服务商的情况下会影响其稳定性，如：当出现域名解析故障时，你对后台
      </span>
      <span style="color:rgb(51,51,51)">
       API
      </span>
      <span style="color:rgb(51,51,51)">
       的请求很可能就会出现
      </span>
      <span style="color:rgb(51,51,51)">
       404
      </span>
      <span style="color:rgb(51,51,51)">
       错误，抛出异常。这时需要对异常进行正确的处理，否则可能会导致程序不能正常工作。
      </span>
     </span>
    </p>
    <h3 style="margin:0px; padding:0px; color:rgb(54,46,43); font-family:Arial; line-height:26px">

     </a>
     2.12接口测试

    <p align="left" style="color:rgb(54,46,43); font-family:Arial; font-size:14px; line-height:26px">

     </span>
     <span style="font-family:SimSun; color:black">
      JSON
     </span>

     </span>
     <span style="font-family:SimSun; color:black">
      JSON
     </span>
     <span style="font-family:SimSun; color:black">

     </span>

     2.13客户端数据库测试
