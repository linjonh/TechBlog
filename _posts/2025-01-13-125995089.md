    <p>
     一、测试使用材料
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/97385f754daccb94d46564ab78d1dbed.png#pic_center">
      <br/>
      ①DTU开发板底板
      <br/>
      ②EC200T-CN核心模块
      <br/>
      ③ST-LINK仿真器
      <br/>
      ④USB-232串口线
      <br/>
      ⑤5V-2A电源适配器
      <br/>
      二、设备接线
      <br/>
      2.1放置4G卡
      <br/>
      将EC200S-CN模块核心板从DTU开发板底板中拔下，将4G卡缺口处朝内插入。然后重新将EC200S-CN模块核心板插回DTU开发板底板。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fe22d242ae11d9331ed155e4577b3f99.png#pic_center">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/176c970520a8bb5f99d28578189cd8ff.png#pic_center">
        <br/>
        2.2接入仿真器
        <br/>
        接入仿真器，用于下载程序。
        <br/>
        将仿真器用杜邦线按照下图接入DTU开发板底板，USB端接入电脑。
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/679e3f3786ec8471ef2769c8a7ad2f4a.png#pic_center"/>
       </img>
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/96bede6b2cf7654523d4331e25b006b5.png#pic_center">
      <br/>
      2.3 接入USB-232串口线
      <br/>
      将USB-232串口线一端接入DTU开发板底板，另一端接入电脑。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2b7618372d941757fa4ed9ac1c46f2db.png#pic_center">
       <br/>
       2.4 接入电源适配器
       <br/>
       将电源适配器一端接入DTU开发板底板，另一端接入插排，通电后，模块亮起。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b52a941e3dc422be3ebac12f1d321b05.png#pic_center">
        <br/>
        三、服务器启动Socket调试工具
        <br/>
        3.1登入远程服务器
        <br/>
        在电脑上搜索远程桌面连接。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e083577bcdea9b08c518555c20596a66.png#pic_center"/>
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9ad429a8e0518c8b3f04b3e0c1ce5284.png#pic_center"/>
        <br/>
        3.2打开Socket调试工具
        <br/>
        将Socket调试工具等文件挂到服务器，打开Socket调试工具。
        <br/>
        将Socket调试工具的端口号设置为9988（自定义，不需要一致）。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/00b7c4d176329b08d405143f692c607f.png#pic_center"/>
        <br/>
        四、程序下载
        <br/>
        用Keil打开程序。（打开Keil出错，可在微信公众号：微信搜索nnhptch或者南宁湖畔科技：Keil5及芯片包的安装以及Keil5打开Keil4工程编译出错解决方法。）
        <br/>
        打开程序。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4f173da3f3c2da4b2055c4a1462e87aa.png#pic_center"/>
        <br/>
        4.1配置仿真器
        <br/>
        若前面已经配置了，可以忽略该步骤。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/495f38ad07e805bf9670b8c49e6fb1b2.png#pic_center"/>
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1247181d3bd271eb2097d8a7f5d6724c.png#pic_center"/>
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8f21c53c9ff86c6b34630ddd2ada8db2.png#pic_center"/>
        <br/>
        下载时将Cortex-M Target Driver Setup 中的reset and run勾上，路径如图所示。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c53a6a2dff27b516d3013bac2218f4d4.png#pic_center"/>
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a9d0713a822e5d1824acc5aa5e9b762a.png#pic_center"/>
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/435183c8eab7f6d1dc535ba7f7ba9ce2.png#pic_center"/>
        <br/>
        4.2下载程序
        <br/>
        修改程序
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5dcee4a38cbc69021efa1185ab92ef15.png#pic_center"/>
        <br/>
        修改完成后，点击重新编译，下载。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/935b2e5d5c41710786810cff21b623f7.png#pic_center"/>
        <br/>
        五、查看效果
        <br/>
        打开电脑设备管理器，查看端口，接着打开串口工具，打开串口。
        <br/>
        我的电脑→右键→属性→设备管理器。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c3c78704b359ca7ef01ab859971d22e7.png#pic_center"/>
        <br/>
        在开发工具(软件都在这里)文件夹下，打开SSCOM3.2串口工具，查看效果。
        <br/>
        选择对应的串口号，如上图所知，我们选择COM3，波特率为115200，然后打开串口。
        <br/>
        静静等待连接，进入服务器，看到Socket调试工具接收到板子发送的数据。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c122866f240bac1e8b405a3f577d0a53.png#pic_center"/>
        <br/>
        232接口可以通过我们的开发板发送数据到服务器。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e3a89d9af6cf56bc20da6271739ee625.png#pic_center"/>
        <br/>
        服务器也可以通过我们的板子接收数据通过232接口显示出来。
