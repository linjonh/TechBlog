      <span title="Micro-services">
      </span>
     </span>
     <p>
      <span lang="zh-cn">
       <span title="Micro-service architecture has gained a reputation as a powerful approach for building modern software applications.">
        微服务架构作为构建现代软件应用程序的强大方法赢得了业内认同。
       </span>
       <span title="So what are micro-services?">
        什么是微服务？
       </span>
       <span title="Micro-services separate the functionality required from a software application into multiple independent small software services.">
        微服务将软件应用所需的功能分成多个独立的小软件服务。
       </span>
       <span title="Each of these micro-services is responsible for an individual focused task.">
        这些微服务中的每一个都负责一个专门的任务。
       </span>
       <span title="In order for micro-services to work together to form a large scalable application, they must communicate and exchange data among multiple servers, a practice known as horizontal scaling.">
        为了使这些微服务一起工作以形成一个大型可扩展应用程序，它们必须在多个服务器之间进行通信和交换数据，这被称为水平扩展。
        <br/>
        <br/>
       </span>
       <span title="Each service can be deployed on a different server with dedicated resources or in separate containers (more on that below.) These services can be written in different programming languages, enabling greater flexibility and allowing separate teams to focus on each service, rendering the final application">
        每个服务都可以部署在具有专用资源的不同服务器上，也可以部署在不同的容器中。这些服务可以用不同的编程语言编写，实现更大的灵活性，并允许每个团队专注于单个服务，最终得到质量更高的应用程序
       </span>
       <span title="of higher quality.">
        。
        <br/>
        <br/>
       </span>
       <span title="Another notable advantage to using micro-services is the ease of continuous delivery, or the ability to deploy software often and at any time.">
        使用微服务的另一个显著优势是易于持续交付，或经常和随时部署软件的能力。
       </span>
       <span title="Micro-services make continuous delivery easier because a new feature deployed to one micro-services is less likely to affect other micro-services.">
        微服务使得持续交付更容易，因为部署到一个微服务的新功能不太可能影响其他微服务。

       <span title="Micro-service architectures are a natural fit for cloud native applications  — applications built from the ground up for cloud computing.">
        微服务架构非常适合云原生应用程序——从云计算基础开始构建的应用程序。
       </span>
       <span title="An application is cloud native if designed with the expectation of deploying on a distributed and scalable infrastructure.">
        如果设计为可在分布式和可伸缩的基础设施上进行部署，那么应用程序就是云原生的。
        <br/>
        <br/>
       </span>
       <span title="For example, building an application with a redundant micro-services architecture – we'll see an example shortly – makes the application cloud native, since this architecture allows our application to be deployed in a distributed manner that allows it to be scalable and almost always">
        例如，构建具有冗余微服务架构的应用程序使应用程序成为云原生的，因为此架构允许应用程序以分布式的方式进行部署，从而使其可扩展且几乎总是
       </span>
       <span title="available.">
        可用。
       </span>
       <span title="A cloud native application doesn’t need to always be deployed to a public cloud like Amazon Web Services, it can be deployed to distributed cloud-like infrastructure as well.">
        云原生应用程序不需要总是部署到公有云（如AWS），也可以部署到分布式云类基础设施。
        <br/>
        <br/>
       </span>
       <span title="In fact, what makes an application fully cloud native goes beyond just using micro-services.">
        实际上，使应用程序完全云原生的原因不仅仅在于使用微服务。你
       </span>
       <span title="Your application should employ continuous delivery, the ability to continuously deliver updates to your production applications without disruptions.">
        的应用程序应该采用持续交付，能够在不中断的情况下持续向生产应用程序提供更新。你
       </span>
       <span title="Your application should also make use of services like message queues and technologies like containers (covered in the next section).">
        的应用程序还应该使用消息队列和容器等技术。
        <br/>
        <br/>
       </span>
       <span title="Cloud native applications assume access to numerous server nodes, having access to pre-deployed software services like message queues or load balancers, ease of integration with continuous delivery services, among other things.">
        云原生应用程序假定可以访问众多的服务器节点，可以访问预先部署的软件服务，如消息队列或负载均衡器，易于与持续交付服务集成等。
        <br/>
        <br/>
       </span>
       <span title="If you deploy your cloud native application to a commercial cloud like AWS or Azure, your application has the option to utilize cloud-only software services.">
        如果你将云原生应用程序部署到AWS或Azure等商业云，你的应用程序可以选择使用只有在云中才有的软件服务。
       </span>
       <span title="For example, DynamoDB is a powerful database engine that can only be used on AWS for production applications.">
        例如，DynamoDB是一个功能强大的数据库引擎，只能在AWS上用于生产应用程序。
       </span>
       <span title="Another example is the DocumentDB database in Azure.">
        另一个例子是Azure中的DocumentDB数据库。

         <span title="A container is simply the idea of encapsulating some software inside an isolated user space or “container.” For example, a MySQL database can be isolated inside a container where the environmental variables and the configurations that it needs will live.">
          一个容器就是将一些软件封装在一个孤立的用户空间或“容器”中的想法。例如，一个MySQL数据库可以在一个容器内隔离，其中有所需要的环境变量和配置。
         </span>
         <span title="Software outside the container will not see the environmental variables or configuration contained inside the container by default.">
          默认情况下，容器外部的软件将不会看到容器内包含的环境变量或配置。
         </span>
         <span title="Multiple containers can exist on the same local virtual machine, cloud virtual machine, or hardware server.">
          同一个本地虚拟机、云虚拟机或硬件服务器上可以存在多个容器。

          容器提供了在同一台计算机上运行众多独立软件服务及其所有配置、软件依赖关系、运行时、工具和附带文件的功能。
         </span>
         <span title="In a cloud environment, this ability translates into saved costs and efforts, because the need for provisioning and buying server nodes for each micro-services will diminish since different micro-services can be deployed on the same host without disrupting each other.">

         </span>

          在生产环境中，需要多个服务器节点和众多容器来实现可扩展性和冗余。

         <span title="One notable drawback of relying heavily on micro-services is that they can become too complicated to manage in the long run as they grow in numbers and scope.">

         </span>
