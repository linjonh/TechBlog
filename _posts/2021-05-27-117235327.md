    <h3>
     <a id="_32">
     </a>
     寻找切入点
    </h3>
    <p>
     做前沿项目，还是比较忌讳一开始贪大求全。这不仅仅是技术问题。

     项目刚开始的项目，还是有很多东西需要处理的：
    </p>
    <ul>
     <li>
      熟悉业务的过程；
     </li>
     <li>
      跟业务方的磨合；
     </li>
     <li>
      技术的尝试阶段；
     </li>
     <li>
      等等；
     </li>
    </ul>
    <h4>
     <a id="_40">
     </a>
     以根因分析作为切入点为例
    </h4>
    <h5>
     <a id="_41">
     </a>
     根因分析技术简介
    </h5>
    <p>
     具体详见：https://blog.csdn.net/micklongen/article/details/89437275
    </p>
    <ul>
     <li>
      分析意图
      <ul>
       <li>
        根因分析，需要给出解释的：逻辑推理、状态图、贝叶斯网络等；
       </li>
       <li>
        只需要给出结论：机器学习、频繁项挖掘等；
       </li>
      </ul>
     </li>
     <li>
      场景划分
      <ul>
       <li>
        实时分析：1分钟之内
       </li>
       <li>
        准实时分析：5分钟之内
       </li>
       <li>
        离线分析：5分钟之后
       </li>
      </ul>
     </li>
    </ul>
