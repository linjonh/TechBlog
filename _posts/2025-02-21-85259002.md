    <h2>
     <a id="_3">
     </a>
     游戏的组成
    </h2>
    <blockquote>
     <p>
      <em>
       做一个简单的游戏结构的总结，以下是我个人的观点。
      </em>
     </p>
    </blockquote>
    <br/>
    <p>
     <strong>
      游戏的组成成分大致可分为以下两个部分：
     </strong>
    </p>
    <hr/>
    <h3>
     <a id="1__13">
     </a>
     1. 游戏内容部分
    </h3>
    <br/>
    <p>
     游戏的内容，是
     <strong>
      游戏主体的元素
     </strong>
     。
     <br/>
     这一部分常交给
     <em>
      游戏策划
     </em>
     来完成。
     <br/>
     一个受人喜爱的游戏，其游戏内容一定是非常精彩的。
     <br/>
     <br/>
    </p>
    <h6>
     <a id="_21">
     </a>
     ·类型·
    </h6>
    <pre><code>	游戏的种类繁多，例如 射击类、冒险类、解谜类、体育休闲、棋牌、动作 等等；
	根据人称视角还可以分为 第一人称、第三人称 ；
	根据维度可以分成 2D、3D 等。
</code></pre>
    <p>
     一个游戏类型的设定将影响着所有内容的设定。
    </p>
    <h6>
     <a id="_28">
     </a>
     ·剧情·
    </h6>
    <p>
     不用多做介绍。
    </p>
    <h6>
     <a id="_30">
     </a>
     ·主题·
    </h6>
    <p>
     主题一般顺应游戏的剧情，也影响着风格的设定。
    </p>
    <h6>
     <a id="_32">
     </a>
     ·风格·
    </h6>
    <p>
     风格中包括：
    </p>
    <pre><code>	画风：人物风格、场景风格、特效等等
	音乐风格：背景音乐、音效等
	玩法：操作风格等等
</code></pre>
    <p>
     游戏风格的设定应该具有
     <strong>
      创新性
     </strong>
     、
     <strong>
      合理性
     </strong>
     。
     <br/>
     其中玩法还应具有
     <strong>
      可操作性
     </strong>
     。
    </p>
    <p>
     不同的游戏风格带给玩家不同的感受。
     <br/>
     大多数玩家都喜欢有创新的游戏。现代游戏行业发展飞速，各种各样的游戏上市。游戏的编策常常根据人们的认知与想象，人的认知是有限的，想象就显得格外重要，也就是创新。一个有创新的游戏再配合过硬的技术，其生存周期便会比较长久。
     <br/>
     <br/>
    </p>
    <hr/>
    <h3>
     <a id="2__47">
     </a>
     2. 核心技术部分
    </h3>
    <br/>
    <p>
     游戏的技术实现，是
     <strong>
      游戏功能的实现
     </strong>
     。
     <br/>
     这一部分由
     <strong>
      各个技术层
     </strong>
     合作完成。
     <br/>
     <br/>
    </p>
    <h6>
     <a id="_54">
     </a>
     ·图形系统·
    </h6>
    <p>
     对于游戏，图形系统是一个
     <strong>
      至关重要
     </strong>
     的技术。
     <br/>
     游戏不仅对于画质有着较高的要求，而且还需要极高的效率，随着硬件性能的提升，一般的小型游戏可以非常流畅的运行，但对于要求严格的大型游戏来说，图形系统的
     <strong>
      优化
     </strong>
     极为重要。
     <br/>
     一般游戏开发选择使用
     <strong>
      图形库
     </strong>
     辅助完成图形系统，如GDI、GDI+、DirectX、OpenGL等。
    </p>
    <pre><code>	渲染
	动画
	图层：
			0层：信息提示
			1层：UI
			2层：特效与粒子
			3层：角色
			4层：地图
			5层：背景
			...
	...
</code></pre>
    <h6>
     <a id="_70">
     </a>
     ·音频系统·
    </h6>
    <pre><code>	背景音乐：引人入境，体现游戏节奏，带给玩家紧张、恐怖、悠闲、宁静等心里。
	音效：使游戏内容更加真实，强化玩家的感官体验。
</code></pre>
    <h6>
     <a id="_73">
     </a>
     ·设备输入处理·
    </h6>
    <p>
     从玩家
     <strong>
      输入设备
     </strong>
     ：鼠标、键盘、手柄、触摸屏等获取消息，并进行相应的处理。
    </p>
    <h6>
     <a id="_75">
     </a>
     ·游戏逻辑·
    </h6>
    <p>
     游戏过程即是一个循环的过程：
     <br/>
     开始游戏 &gt; 进行游戏（暂停游戏]）&gt; 胜利 / 失败 &gt; 重新开始 / 下一关
    </p>
    <p>
     在游戏循环阶段还要包括游戏运行时的事件处理：
    </p>
    <pre><code>	计时事件	碰撞事件	...
	
	胜利条件判断等等
</code></pre>
    <h6>
     <a id="_84">
     </a>
     ·算法·
    </h6>
    <p>
     大量的计算：
    </p>
    <pre><code>	物理系统	数学计算	...
</code></pre>
    <h6>
     <a id="_88">
     </a>
     ·数据存储与读取·
    </h6>
    <pre><code>	读取：
			读档
			游戏资源的加载（包括图形，声音等）
			版本信息
			*账户信息
			*历史记录
			...
	存储：
			存档
			*登录信息
			*历史记录
			...		
</code></pre>
    <h6>
     <a id="_101">
     </a>
     ·网络支持·
    </h6>
    <p>
     使游戏能够支持
     <strong>
      局域网联机
     </strong>
     ，或者实现
     <strong>
      网络游戏
     </strong>
     。
    </p>
    <pre><code>	服务器	客户端	网络信息交互与同步	加密	网络异常中断处理		...
</code></pre>
