    <h3 id="%E4%BA%8C%E3%80%81%E6%95%B0%E6%8D%AE%E6%A8%A1%E5%9E%8B">
     二、
     <span style="color:#34495e;">
      数据模型
     </span>
    </h3>
    <h4 id="1.%E5%85%B3%E7%B3%BB%E5%9E%8B%E6%95%B0%E6%8D%AE%E5%BA%93%EF%BC%88RDBMS%EF%BC%89">
     <span style="color:#34495e;">
      1.关系型数据库（
     </span>
     <span style="color:#34495e;">
      RDBMS
     </span>
     <span style="color:#34495e;">
      ）
     </span>
    </h4>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      概念：建立在关系模型基础上，由多张相互连接的二维表组成的数据库。
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      而所谓二维表，指的是由行和列组成的表，如下图（就类似于Excel表格数据，有表头、有列、有行，还可以通过一列关联另外一个表格中的某一列数据）。我们之前提到的MySQL、Oracle、DB2、
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      SQLServer这些都是属于关系型数据库，里面都是基于二维表存储数据的。简单说，基于二维表存储数据的数据库就成为关系型数据库，不是基于二维表存储数据的数据库，就是非关系型数据库。
     </span>
    </p>
    <p>
     <img alt="" height="272" src="https://i-blog.csdnimg.cn/blog_migrate/bc73d42ea420f132f199733bff0692d2.png" width="921"/>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      特点：
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#34495e;">
       使用表存储数据，格式统一，便于维护。
      </span>
     </li>
     <li>
      <span style="color:#34495e;">
       使用SQL语言操作，标准统一，使用方便。
      </span>
     </li>
    </ol>
    <p style="margin-left:0cm;">
     <span style="color:#34495e;">
     </span>
     <span style="color:#34495e;">
      MySQL是关系型数据库，是基于二维表进行数据存储的，具体的结构图下:
     </span>
    </p>
    <p style="margin-left:0cm;">
     <img alt="" height="366" src="https://i-blog.csdnimg.cn/blog_migrate/836e2fd8dc4c92094414d4811bc86df4.png" width="921"/>
    </p>
    <blockquote>
     <ul>
      <li>
       <span style="color:#34495e;">
        我们可以通过MySQL客户端连接数据库管理系统
       </span>
       <span style="color:#34495e;">
        DBMS然后通过DBMS操作数据库。
       </span>
       <ul>
        <li style="margin-left:0cm;">
         <span style="color:#34495e;">
          可以使用SQL语句，通过数据库管理系统操作数据库，以及操作数据库中的表结构及数据。
         </span>
        </li>
        <li style="margin-left:0cm;">
         <span style="color:#34495e;">
          一个数据库服务器中可以创建多个数据库，一个数据库中也可以包含多张表，而一张表中又可以包含多行记录。
         </span>
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>

    <h3 id="%E5%9B%9B%E3%80%81SQL">
     <span style="color:#34495e;">
      四、SQL
     </span>
    </h3>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      全称 Structured  Query Language，结构化查询语言。操作关系型数据库的编程语言，定义了一套操作关系型数据库统一
      <strong>
       标准
      </strong>
      。
     </span>
    </p>
    <h4 id="1.SQL%E9%80%9A%E7%94%A8%E8%AF%AD%E6%B3%95" style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      1.SQL
     </span>
     <span style="color:#34495e;">
      通用语法
     </span>
    </h4>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      在学习具体的SQL语句之前，先来了解一下SQL语言的同于语法。
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      1). SQL语句可以单行或多行书写，以分号结尾。
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      2). SQL语句可以使用空格/缩进来增强语句的可读性。
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      3). MySQL数据库的SQL语句不区分大小写，关键字建议使用大写。
     </span>
    </p>
    <p style="margin-left:-.25pt;">
     <span style="color:#34495e;">
      4). 注释：
     </span>
    </p>
    <ul>
     <li style="margin-left:12.5pt;">
      <span style="color:#34495e;">
       单行注释：-- 注释内容  或  # 注释内容
      </span>
     </li>
     <li style="margin-left:12.5pt;">
      <span style="color:#34495e;">
       多行注释：/* 注释内容 */
      </span>
