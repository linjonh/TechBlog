    <h3>
     <a id="Java_0">
     </a>
     Java语言的特点
    </h3>
    <p>
     <strong>
      1.面向对象
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       面向对象(OOP)就是Java语言的基础，也是Java语言的重要特性。面向对象的概念：生活中的一切事物都可以被称之为对象，生活中随处可见的事物就是一个对象，我们可以将这些事物的状态特征（属性）以及行为特征（方法）提取并出来，并以固定的形式表示。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      2.简单好用
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       Java语言是由C和C++演变而来的，它省略了C语言中所有的难以理解、容易混淆的特性（比如指针），变得更加严谨、简洁、易使用。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      3.健壮性
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       Java的安全检查机制，将许多程序中的错误扼杀在摇蓝之中。 另外，在Java语言中还具备了许多保证程序稳定、健壮的特性（强类型机制、异常处理、垃圾的自动收集等），有效地减少了错误，使得Java应用程序更加健壮。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      4.安全性
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       Java通常被用在网络环境中，为此，Java提供了一个安全机制以防恶意代码的攻击，从而可以提高系统的安全性。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      5.平台无关性
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       Java平台无关性由Java 虚拟机实现，Java软件可以不受计算机硬件和操作系统的约束而在任意计算机环境下正常运行。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      6.支持多线程
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       在C++ 语言没有内置的多线程机制，因此必须调用操作系统的多线程功能来进行多线程程序设计，而 Java 语言却提供了多线程支持。多线程机制使应用程序在同一时间并行执行多项任务，该机制使得程序能够具有更好的交互性、实时性。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      7.分布式（支持网络编程）
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       Java语言具有强大的、易于使用的网络能力，非常适合开发分布式计算的程序。java中提供了网络应用编程接口(java.net)，使得我们可以通过URL、Socket等远程访问对象。
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      8.编译与解释共存
     </strong>
    </p>
    <blockquote>
     <p>
      <a href="https://blog.csdn.net/Lzy410992/article/details/117898940">
       <strong>
        Java 是编译与解释共存的语言
       </strong>
      </a>
     </p>
    </blockquote>

     Java语法基础
    </h3>
    <p>
     <strong>
      标识符：
     </strong>
     用来标识类名、对象名、变量名、方法名、类型名、数组名、文件名的有效字符序列。
    </p>
    <p>
     <strong>
      合法的标识符：
     </strong>
    </p>
    <ul>
     <li>
      由字母、数字、下划线“_”、美元符号“$”或者“￥”组成，并且首字符不能是数字。
     </li>
     <li>
      不能把java关键字和保留字作为标识符。
     </li>
     <li>
      标识符对大小写敏感。
     </li>
    </ul>
    <p>
     <strong>
      关键字：Java语言中已经赋予了特定含义的
     </strong>
     <br/>

      <strong>
       保留字：
      </strong>
      <code>
       const、goto
      </code>
      ，Java版本中尚未使用，但以后版本可能会作为关键字使用
     </img>
    </p>

     <strong>
      变量：程序运行期间可以被改变的量。在程序中使用变量，必须先创建它并为它取一个名字，并且指明它能够存储信息的类型，这称为“变量声明”，也叫容器的创建。

     </strong>
    </p>

     <li>
      变量的赋值：变量名 = 数据;
     </li>
     <li>
      变量的操作：放入打印语句进行输出或者进行运算
     </li>
    </ul>

     <strong>
      Java 中的注释有三种：
     </strong>
    </p>
    <ul>
     <li>
      单行注释
     </li>
     <li>
      多行注释

    <h3>
     <a id="_120">
     </a>
     数组
    </h3>
    <p>
     数组是数据的集合，一个容器，用来存储任何类型的数据，包括原始数据类型和引用数据类型，但是一旦指定了数组的类型之后，就只能用来存储指定类型的数据。
    </p>
    <p>
     <strong>
      数组声明的三种方式：
     </strong>
    </p>
    <ul>
     <li>
      数据类型[] 数组名 = new 数据类型[长度];
     </li>
     <li>
      数据类型[] 数组名 = {数据,数据,…,数据};
     </li>
     <li>
      数据类型[] 数组名 = new 数据类型长度[] {数据,数据,…,数据};
     </li>
    </ul>
    <p>
     <strong>
      一维数组：
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        数组变量的声明：
       </strong>
       <br/>
       语法：
       <code>
        数据类型[] 数组名;
       </code>
       ，如：
       <code>
        int[] num;
       </code>
       、
       <code>
        double[] d;
       </code>
       、
       <code>
        String[] str;
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        数组对象的创建：
       </strong>
       <br/>
       语法：
       <code>
        数组名 = new 数据类型[长度];
       </code>
       ，如：
       <code>
        num = new int[4];
       </code>
       ，数组声明其长度后不可改变
      </p>
     </li>
     <li>
      <p>
       <strong>
        赋值：
       </strong>
       <br/>
       语法：
       <code>
        数组名[下标] = 数据;
       </code>
       ，如：num[0] = 3;
      </p>
     </li>
     <li>
      <p>
       <strong>
        数组元素的使用及遍历：
       </strong>
       <br/>
       语法：
       <code>
        数组名[下标]
       </code>
       ，获取指定下标是数据。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f91671fe36c835aebbcc1b6a3b5587c3.png"/>
      </p>
     </li>
    </ul>
    <p>
     <strong>
      二维数组：
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        数组变量的声明：
       </strong>
       <br/>
       语法：
       <code>
        数据类型[][] 数组名;
       </code>
       ，如：
       <code>
        int[][] num;
       </code>
       、
       <code>
        double[][] d;

    <h3>
     <a id="_154">
     </a>
     对象与类
    </h3>
    <p>
     <strong>
      对象：客观存在能够相互区分的个体，比如这辆汽车、这个人、这间房子、这张桌子、这株植物、这张支票、这件雨衣。概括来说就是：万物皆对象。
     </strong>
    </p>
    <p>
     <strong>
      类：若干具有相同属性和行为的对象的群体或者抽象，类是创建对象的模板，由属性和行为两部分组成
     </strong>
    </p>
    <p>
     <strong>
      类是对象的概括或者抽象，对象是类的实例化。
     </strong>
    </p>
    <h4>
     <a id="_163">
     </a>
     类的声明
    </h4>
    <pre><code>[修饰符] class 类名{//类的声明部分
	[成员变量]
	[成员方法]
}
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b270b01e2e6fb7e286e8169aad5c56ce.png"/>
    </p>
    <p>
     <strong>
      成员变量的声明：
     </strong>
     <code>
      [修饰符] 数据类型 变量名 [= 初始值];
     </code>
    </p>
    <p>
     <strong>
      成员方法的声明：
     </strong>
    </p>
    <pre><code>[修饰符] 返回值类型 方法名([数据类型 参数名,……]){
	//方法体，该方法完成的功能代码
}
</code></pre>
    <p>
     <strong>
      构造器的声明：
     </strong>
    </p>
    <pre><code>[修饰符] 构造器名([数据类型 参数名,……]){
  //构造器完成的功能代码
}
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/93036cf21e8bd0048d744a2d6e045cd4.png"/>
    </p>
    <blockquote>
     <p>
      <strong>
       注意：
      </strong>
      <br/>
      ①构造器名必须和类名一致
      <br/>
      ②构造器没有返回值类型
      <br/>
      ③任何类都含有构造器。如果没有显式地定义类的构造器，
      <br/>
      ④则系统会为该类提供一个默认的无参的构造器。一旦在类中显式地定义了构造器，系统就不会再为这个类提供默认的构造器了。
     </p>
    </blockquote>

     <li>

    <h4>
     <a id="_226">
     </a>
     关键字
    </h4>
    <p>
     <strong>
      this关键字：
     </strong>
    </p>
    <blockquote>
     <p>
      this是一种特殊的引用，指向当前对象
     </p>
    </blockquote>
    <p>
     <strong>
      this的两种使用方法：
     </strong>
     <br/>
     如果发生局部变量和成员变量命名冲突时，可以通过this.成员变量名的方式区分成员变量和局部变量。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3c6cc6e4b4e32354f151d87338b5da34.png"/>
     <br/>
     一个构造方法中需要调用本类的另一个构造方法，可以通过this()的方式调用，但this()必须要书写在第一行。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9954418db68320350b3f1c6f4d48dc8e.png"/>
     <br/>
     <strong>
      static关键字：
     </strong>
    </p>
    <p>
     <strong>
      静态变量：
     </strong>
     使用static修饰的成员变量叫做静态变量，静态变量和非静态变量的区别是：静态变量被所有的对象所共享，在内存中只有一个副本，它当且仅当在类初次加载时会被初始化。而非静态变量是对象所拥有的，在创建对象的时候被初始化，存在多个副本，各个对象拥有的副本互不影响。static成员变量的初始化顺序按照定义的顺序进行初始化。
    </p>
    <p>
     <strong>
      静态方法：
     </strong>
     使用static修饰的成员方法叫做静态方法，静态方法可以不依赖于任何对象进行访问（对于静态方法来说，是没有this的），由于这个特性，在静态方法中不能访问类的非静态成员变量和非静态成员方法，因为非静态成员方法/变量都是必须依赖具体的对象才能够被调用。
    </p>
    <p>
     <strong>
      调用格式：
      <code>
       类名.静态变量名 ，类名.静态方法名()
      </code>
     </strong>
    </p>
    <p>
     <strong>
      静态内部类（static 修饰类的话只能修饰内部类）：
     </strong>
     静态内部类与非静态内部类之间存在一个最大的区别: 非静态内部类在编译完成之后会隐含地保存着一个引用，该引用是指向创建它的外围类，但是静态内部类却没有。没有这个引用就意味着：1. 它的创建是不需要依赖外围类的创建。2. 它不能使用任何外围类的非 static 成员变量和方法。
    </p>
    <p>
     <strong>
      静态代码块：
     </strong>
     <br/>
     是一个以static为前导的代码块，一般用于为类的工作做一些初始化工作，如初始化一些静态变量。一个类中可以有许多静态初始化块，并且它们可以出现在类体的任何地方。运行时系统会保证静态初始化块会按照它们在源代码中出现的顺序被调用
    </p>
    <p>
     static块可以用来优化程序性能：因为它
     <strong>
      只会在类加载的时候执行一次
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fc09bf7a8488078fc4083ff744ba4f76.png"/>
    </p>
    <p>
     <strong>
      super关键字：
     </strong>
     <br/>
     super代表的是父类对象，
    </p>
    <p>
     <strong>
      使用方式：
     </strong>
     <br/>
     <code>
      super.属性名、super.方法名();
     </code>
     用于在子类中调用父类被隐藏的同名实例变量
     <br/>
     <code>
      super([参数列表])
     </code>
     用于在子类的构造方法中调用父类的构造方法
    </p>
    <blockquote>
     <p>
      每一个子类的构造方法在没有显示调用super()系统都会提供一个默认的super()，super()必须是构造器的第一条语句
     </p>
    </blockquote>
    <p>
     <strong>
      final关键字:
     </strong>
    </p>
    <p>
     final 关键字，意思是最终的、不可修改的，最见不得变化 ，用来修饰类、方法和变量，具有以下特点：
    </p>
    <ul>
     <li>
      修饰类：类不能继承，final 类中的所有成员方法都会被隐式的指定为 final 方法；
     </li>
     <li>
      修饰符变量：该变量为常量，，如果是基本数据类型的变量，则其数值一旦在初始化之后便不能更改；如果是引用类型的变量，则在对其初始化之后便不能让其指向另一个对象。
     </li>
     <li>
      修饰符方法：方法不能重写
     </li>
    </ul>
    <p>
     说明：使用 final 方法的原因有两个。第一个原因是把方法锁定，以防任何继承类修改它的含义；第二个原因是效率。在早期的 Java 实现版本中，会将 final 方法转为内嵌调用。但是如果方法过于庞大，可能看不到内嵌调用带来的任何性能提升（现在的 Java 版本已经不需要使用 final 方法进行这些优化了）。类中所有的 private 方法都隐式地指定为 final。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6089b85235e2bfbdfd2c4f28250e8d78.png"/>

    </h3>
    <h4>
     <a id="_284">
     </a>
     封装
    </h4>
    <p>
     <strong>
      封装指隐藏对象的状态信息（属性），不允许外部对象直接访问对象的内部信息（private实现）。但是可以提供一些可以被外界访问的方法来操作属性。
     </strong>
    </p>
    <p>
     <strong>
      将类中成员变量private，提供public的get和set方法来控制属性的存取动作，以保证对私有属性操作的安全性：
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/41e5899593fe9200473e0ad0cf0ab9dd.png"/>
    </p>
    <h4>
     <a id="_290">
     </a>
     继承
    </h4>
    <p>
     继承extends：多个类中存在相同属性和行为时，将这些内容抽取到单独一个类中，那么多个类无需再定义这些属性和行为，只要继承单独的那个类即可。多个类可以称为子类，单独这个类称为父类或者超类。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8eddaeb7c0ec88dc38928adb031b95b7.png"/>
     <br/>
     <strong>
      基本语法：
     </strong>
    </p>
    <pre><code>[修饰符] class 子类名 extends 父类名{
   类体部分
}
</code></pre>
    <p>
     继承是使用已存在的类的定义作为基础建立新类的技术，新类的定义可以增加新的属性或方法（对父类进行扩展），也可以拥有父类的属性和方法，并且通过自己的方法再次实现父类的方法（重写）。通过使用继承，可以快速地创建新的类，可以提高代码的重用，程序的可维护性，节省大量创建新类的时间 ，提高我们的开发效率。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/26197a05815a0358d742346f1aaf4309.png"/>
     <br/>
     Java只支持单继承，不支持多继承。一个类只能有一个父类，不可以有多个父类。Java支持多层继承(继承体系)。Java继承了父类非私有的成员变量和成员方法，但是请注意：子类是无法继承父类的构造方法的。
    </p>
    <blockquote>
     <p>
      <strong>
       注意：不要仅为了获取其他类中某个功能而去继承 ，类与类之间要有所属(“is a” )关系
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      方法的重写：
     </strong>
     <br/>
     子类从父类继承的某个实例方法无法满足子类的功能需要时，需要在子类中对该实例方法进行重新实现，这样的过程称为重写，也叫做覆写、覆盖。
    </p>
    <p>
     方法重写的前提：继承，子类的修饰符大于等于父类，方法名、参数列表、返回值类型必须相同
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a65065b0e222a3674e69959cd4aba8b1.png"/>
    </p>

     </a>
     多态
    </h4>
    <p>
     多态：一种事物的多种形态（多态的前提：继承、重写、向上转型）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/250c5ffcf4d0760bf7e710ba25f7c2e3.png"/>

    <h3>
     <a id="_321">
     </a>
     抽象类
    </h3>
    <p>
     <strong>
      抽象类的概念：
     </strong>
     <br/>
     Java中可以定义没有方法体的方法，该方法的具体实现由子类完成，该方法称为抽象方法，包含抽象方法的类就是抽象类。如：Shape类计算周长和面积的方法无法确定，那么就可以将这样的方法声明为抽象的，以便在具体的子类中进行实现。
    </p>
    <p>
     <strong>
      抽象类的声明：
     </strong>
     <code>
      [修饰符] abstract class 类名 [extends 父类名]{类体}
     </code>
     <br/>
     <strong>
      抽象方法的声明：
     </strong>
     <code>
      [修饰符] abstract 返回值类型 方法名([参数列表]);
     </code>
     <br/>
     因为抽象方法无法确定具体执行的功能，所有抽象方法没有方法体，需要在小括号后加上分号
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ea40bffc24773c5d6ef9d389d437683b.png"/>
     <br/>
     抽象类和普通类除了使用abstract修饰外和普通的类相似，抽象类中可以没有抽象方法，但是一旦某个有抽象方法，那么这个类必须被声明为抽象类。
    </p>
    <p>
     <strong>
      抽象类的使用：
     </strong>
     <br/>
     因为抽象类不是一个具体的类，所以无法实例化，但是抽象类可以用于声明变量
     <br/>
     抽象类可以被继承，在子类中实现抽象类的所有抽象方法，以达到抽象类的具体化
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/caf9e94ac8bf01bb242363b60cd1e91b.png"/>
    </p>

     </strong>

      [修饰符] class 类名 [extends 父类名] [implements 接口1,接口2,……]{类体部分}
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3e8f5fe653bf859b6d23f6a0349e0b4b.png"/>
