    <h3>
     <a id="_0">
     </a>
     无图形界面的代码编写
    </h3>
    <p>
     python代码作为脚本语言，其实最好直接使用vim在字符界面里面直接编写，这才是正道，但是作初学者，其实直接在服务器上写代码是很不习惯的，所以我这边希望能写一篇帮助师弟师妹使用本地的ide编写代码，然后服务器的解释器运行代码。
    </p>
    <h3>
     <a id="_3">
     </a>
     原理
    </h3>
    <p>
     如何本地写代码，服务器运行呢？
     <br/>
     其实原理还是比较简单的，就是一个ssh同步本地和服务器代码的工程，ide会自动同步你的本地代码到服务器上，同时服务器接受到你的ide运行指令后，就会直接在服务器上运行代码，代码运行结束后，服务器会直接同步结果到你本地的ide上，整个过程就像你在本地跑了一个代码。
    </p>
    <h3>
     <a id="_7">
     </a>
     准备工作
    </h3>
    <p>
     首先，你要下载一个pycharm，专业版，作为在校的学生，你的教育邮箱是可以申请专业版的免费授权的，还是很良心的。其次，你需要一个ssh服务安装完成的linux服务器，当然服务器上需要安装好python解释器。
     <br/>
     这边我用的是mac的pycharm，windows的用户操作是大同小异的。
    </p>
    <h3>
     <a id="_11">
     </a>
     第一步：同步代码和数据
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cf0ceb20ee4b2faf127bdaeea26e8d43.png">
      <br/>
      点击上图的Configuration，会出现以下界面
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/32de3d5a69ae9001957ddc99b8a70c22.png">
       <br/>
       点击小窗口左上角的加号，选择sftp，随便取一个自己喜欢的名字
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/589a7401aefa1d9ba97377f2f491933d.png"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fc38257cd04a02527480c8a98f888df4.png">
      <br/>
      host: 服务器的ip地址
      <br/>
      user name：服务器账号
      <br/>
      root path：这个决定了你映射地址的根目录（同步代码的目录），建议填你用户的home
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3994918ca7ab0c5546218fb33444e511.png"/>
    </p>
    <p>
     点击mapping，正式建立本地代码和服务器代码的映射，主要要填的就是deployment path，这个目录是建在刚刚设置的根目录下面。建立完映射之后，就可以开始上传你的代码了。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fae6923b03c9e8f7378ddb662986eebd.png">
      <br/>
      上传完了，你就可以到服务器相应的目录下看到你写的代码了。
     </img>
    </p>
    <h3>
     <a id="_30">
     </a>
     调用远程解释器
    </h3>
    <p>
     建立本地和服务器解释器的联系
     <br/>
     点击preference
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/86e648de82c090945b96d9f2aa6849c1.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8c3d892a53aed1e5bddf05769dd02696.png">
       <br/>
       这边还是本地的解释器，下面我们新建一个远程解释器，点解右上角的设置图标添加远程解释器
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/eb01eb379adf14fc726c2c9c5b1dbc71.png"/>
       <br/>
       用户密码设置一下
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d1ef623b2bdd415afb583dc275fdd1c9.png"/>
       <br/>
       找到解释器的地址填入，还需要更改一下同步代码的地址
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1099c97a9cc79573289c536e6fc46e8c.png"/>
       <br/>
       点击finish就完成了。
      </img>
     </img>
    </p>
    <h3>
     <a id="_43">
     </a>
     结束
    </h3>
    <p>
     完成所有操作，最后一张图我没有勾选自动同步，当然你如果勾选了，本地代码和服务器代码就会自动同步，不需要你手动地去上传。当然当你有大量数据在本地的时候，自动同步是一个很好时的操作。还不如手动同步。手动同步就是如上图的upload to 230。
    </p>
