    <h2>
     <a id="_2">
     </a>
     环境介绍
    </h2>
    <ul>
     <li>
      <strong>
       以下不带 * 号的为关键字，带 * 号的为系统保留字
       <br/>
       DM 还将保留字进一步划分为 SQL 保留字、DMSQL 程序保留字、模式保留字、变量保留字和别名保留字，具体请查询系统视图
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token operator">*</span> <span class="token keyword">FROM</span> V$RESERVED_WORDS<span class="token punctuation">;</span>
</code></pre>
    <h2>
     <a id="1__10">
     </a>
     1 特殊关键字（无法屏蔽）
    </h2>
    <ul>
     <li>
      <strong>
       不能作为表的列名，即使加上双引号也不行
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-shell">  ROWID、 TRXID、 VERSIONS_STARTTIME、 VERSIONS_ENDTIME、
  VERSIONS_STARTTRXID、 VERSIONS_ENDTRXID
 和  VERSIONS_OPERATION 
</code></pre>
    <h2>
     <a id="2__17">
     </a>
     2 屏蔽关键字
    </h2>
    <ul>
     <li>
      <strong>
       以下的可以加 双引号，若程序中使用到的关键字较多，手动加双引号 工作量会成正比增加那么有两种方法来简单解决
      </strong>
     </li>
    </ul>
    <h3>
     <a id="21_1_dmini__19">
     </a>
     2.1 方法1 配置dm.ini （推荐）
    </h3>
    <ul>
     <li>
      <strong>
       EXCLUDE_RESERVED_WORDS 参数名后面追加关键字
       <br/>
       重启数据库服务生效
      </strong>
     </li>
    </ul>
    <h4>
     <a id="211_SQL_22">
     </a>
     2.1.1 执行SQL后重启数据库服务
    </h4>
    <pre><code class="prism language-sql">SP_SET_PARA_STRING_VALUE<span class="token punctuation">(</span><span class="token number">2</span><span class="token punctuation">,</span><span class="token string">'EXCLUDE_RESERVED_WORDS'</span><span class="token punctuation">,</span><span class="token string">'DOMAIN,ADMIN,DATA'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

<span class="token keyword">COMMIT</span><span class="token punctuation">;</span>

<span class="token keyword">SELECT</span> PARA_VALUE<span class="token punctuation">,</span>FILE_VALUE <span class="token keyword">FROM</span> V$DM_INI <span class="token keyword">WHERE</span> PARA_NAME<span class="token operator">=</span><span class="token string">'EXCLUDE_RESERVED_WORDS'</span><span class="token punctuation">;</span>
</code></pre>
    <h4>
     <a id="222__dmini__30">
     </a>
     2.2.2 直接修改 dm.ini 后重启数据库服务
    </h4>
    <pre><code class="prism language-xml">EXCLUDE_RESERVED_WORDS =DOMAIN,ADMIN,DATA
</code></pre>
    <h3>
     <a id="22_2_dm_svcconf_34">
     </a>
     2.2 方法2 配置dm_svc.conf
    </h3>
    <p>
     <a href="https://blog.csdn.net/qq_37358909/article/details/110955828">
      dm_svc.conf 的配置文件只需加上一个参数与参数值即可
     </a>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        参数名
       </th>
       <th>
        默认值
       </th>
       <th align="left">
        简述
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        KEYWORDS
       </td>
       <td>
        无
       </td>
       <td align="left">
        标识用户关键字，所有在列表中的字符串，如果以单词的形式出现在 SQL 语句中，则这个单词会被加上双引号。该参数主要用来解决用户需要使用 DM8 中的保留字作为对象名使用的状况。
       </td>
      </tr>
     </tbody>
    </table>
