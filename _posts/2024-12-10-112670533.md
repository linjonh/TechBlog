    <h2>
     <a id="_1">
     </a>
     前言
    </h2>
    <p>
     <font color="#999AAA">
      数据模型是数据库系统的核心，本文简要介绍三种主要的数据库模型。
     </font>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="_7">
     </a>
     一、格式化模型
    </h2>
    <p>
     <font color="#999AAA">
      格式化模型是层次模型和网状模型的统称
     </font>
    </p>
    <h2>
     <a id="1_11">
     </a>
     1.层次模型
    </h2>
    <h3>
     <a id="__13">
     </a>
     ① 层次数据模型的概念
    </h3>
    <p>
     层次模型用属性结构来表示各类实体以及实体间的联系
    </p>
    <h3>
     <a id="__17">
     </a>
     ② 层次模型的数据结构
    </h3>
    <p>
     <font color="#999AAA">
      特点：
     </font>
    </p>
    <ol>
     <li>
      结点的双亲是唯一的；
     </li>
     <li>
      只能直接处理一对多的实体联系；
     </li>
     <li>
      每个记录类型可以定义一个排序段，也称码字段；
     </li>
     <li>
      任何记录值只有按其路径查看时才能显出它的全部意义；
     </li>
     <li>
      没有一个子女记录值能够脱离双亲记录值而独立存在。
     </li>
    </ol>
    <h3>
     <a id="_28">
     </a>
     ③层次模型的数据操纵
    </h3>
    <font color="#999AAA">
    </font>
    <ul>
     <li>
      查询
     </li>
     <li>
      插入
     </li>
     <li>
      删除
     </li>
     <li>
      更新
     </li>
    </ul>
    <h3>
     <a id="_37">
     </a>
     ④层次模型的完整性约束
    </h3>
    <ol>
     <li>
      无对应的双亲结点值就不能插入子女结点；
     </li>
     <li>
      如果删除双亲结点则相应的子女节点值也被同时删除；
     </li>
     <li>
      更新操作时，应更新所有相应记录，以保证数据的一致性。
     </li>
    </ol>
    <h3>
     <a id="_44">
     </a>
     ⑤层次模型的优缺点
    </h3>
    <ul>
     <li>
      <p>
       优点
       <br/>
       层次模型的数据结构比较简单清晰；
       <br/>
       查询效率高，性能优于关系模型，不低于网状模型；
       <br/>
       层次数据模型提供了良好的完整性支持。
      </p>
     </li>
     <li>
      <p>
       缺点
       <br/>
       结点之间的多对多联系表示不自然；
       <br/>
       对插入和删除操作的限制多，应用程序的编写比较复杂；
       <br/>
       查询子女结点必须通过双亲结点；
       <br/>
       层次命令趋于程序化；
      </p>
     </li>
    </ul>
    <h2>
     <a id="2_57">
     </a>
     2.网状模型
    </h2>
    <h3>
     <a id="_59">
     </a>
     ①网状模型的概念
    </h3>
    <p>
     网状数据库系统采用网状模型作为数据的组织方式
     <br/>
     <font color="#999AAA">
      ✧典型代表是DBTG系统:
      <br/>
      ■亦称CODASYL系统
      <br/>
      ■20世纪70年代由DBTG提出的一个系统方案
      <br/>
      必实际系统
      <br/>
      ■Cullinet Software公司的IDMS
      <br/>
      ■Univac公司的DMS1100
      <br/>
      ■Honeywell公司的IDS/2
      <br/>
      ■HP公司的IMAGE
     </font>
    </p>
    <h3>
     <a id="_73">
     </a>
     ②网状模型的数据结构
    </h3>
    <p>
     满足下面两个条件的基本层次联系的集合:
    </p>
    <ol>
     <li>
      允许一个以上的结点无双亲;
     </li>
     <li>
      一个结点可以有多于一个的双亲。
     </li>
    </ol>
    <p>
     表示方法(与层次数据模型相同)
    </p>
    <ul>
     <li>
      实体型:用记录类型描述
      <br/>
      每个结点表示-一个记录类型(实体)
     </li>
     <li>
      属性:用字段描述
      <br/>
      每个记录类型可包含若千个字段
     </li>
     <li>
      联系:用结点之间的连线表示记录类型(实体)之间的一对多的父子联系
     </li>
    </ul>
    <h3>
     <a id="_88">
     </a>
     ③网状模型的操纵与完整性约束
    </h3>
    <p>
     网状数据库系统(如DBTG)对数据操纵加
     <br/>
     了一些限制，提供了–定的完整性约束
     <br/>
     ■码:唯一标识记录的数据项的集合
     <br/>
     ■一个联系中双亲记录与子女记录之间是一-对多联系
     <br/>
     ■支持双亲记录和子女记录之间某些约束条件
    </p>
    <h3>
     <a id="_96">
     </a>
     ④网状模型的优缺点
    </h3>
    <ul>
     <li>
      优点
      <br/>
      a.能够更为直接地描述现实世界，如一个结点可以有多个双亲
      <br/>
      b.具有良好地性，存取效率较高
     </li>
     <li>
      缺点
      <br/>
      a.结构比较复杂，而且随着应用环境地扩大，数据库就变得越来越复杂，不利于最终用户掌握
      <br/>
      b.DDL、DML语言复杂，用户不容易使用
      <br/>
      c.记录之间联系是通过存取路径实现地，用户必须了解系统结构地细节
     </li>
    </ul>
    <h3>
     <a id="3_107">
     </a>
     3.网状模型与层次模型的区别
    </h3>
    <p>
     ■网状模型允许多个结点没有双亲结点
     <br/>
     ■网状模型允许结点有多个双亲结点
     <br/>
     ■网状模型允许两个结点之间有多种联系(复合联系)
     <br/>
     ■网状模型可以更直接地描述现实世界
     <br/>
     ■层次模型实际上是网状模型的一一个特例
    </p>
    <h2>
     <a id="_116">
     </a>
     二、关系模型
    </h2>
    <h3>
     <a id="_117">
     </a>
     ①关系模型的概念
    </h3>
    <p>
     关系数据库系统采用关系模型作为数据的组织方式
     <br/>
     <font color="#999AAA">
      1970年美国IBM公司SanJose研究室的研究员E.F.Codd首次提出了数据库系统的关系模型
      <br/>
      计算机厂商新推出的数据库管理系统几乎都支持关系模型
     </font>
    </p>
    <h3>
     <a id="_122">
     </a>
     ②关系模型的数据结构
    </h3>
    <p>
     在用户观点下，关系模型中数据的逻辑结构是一张二维表，它由行和列组成。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/f1897a24ec5b59d8474c4468d3ddb3aa.png#pic_center"/>
    </p>
    <ul>
     <li>
      <p>
       关系(Relation)
       <br/>
       ●一个关系对应通常说的一张表
      </p>
     </li>
     <li>
      <p>
       元组(Tuple)
       <br/>
       ●表中的一行即为一个元组
      </p>
     </li>
     <li>
      <p>
       属性(Attribute )
       <br/>
       ●表中的一列即为一个属性，给每–个属性起一个名称即属性名
      </p>
     </li>
     <li>
      <p>
       主码(Key)
       <br/>
       ●也称码键。表中的某个属性组，它可以唯一 确定一个元组
      </p>
     </li>
     <li>
      <p>
       域(Domain )
       <br/>
       ●是一组具有相同数据类型的值的集合。属性的取值范围来自某个域。
      </p>
     </li>
     <li>
      <p>
       分量
       <br/>
       ●元组中的一一个属性值。
      </p>
     </li>
     <li>
      <p>
       关系模式 ●对关系的描述 关系名(属性1,属性2，，.属性n) 学生(学号，姓名，年龄，性别，系名，年级)
      </p>
     </li>
    </ul>
    <p>
     关系必须是规范的，满足一定的约束条件
     <br/>
     最基本的约束条件： 关系的每一个分量必须是一个不可分的数据项，不允许表中还有表
    </p>
    <p>
     <img alt="一个工资表（表中有表）实例" src="https://i-blog.csdnimg.cn/blog_migrate/71782ef248767270a119f04d32f4e59e.png"/>
    </p>
    <h3>
     <a id="_152">
     </a>
     ③关系模型的操纵与完整性约束
    </h3>
    <ul>
     <li>
      数据操作是集合操作，操作对象和操作结果都是关系
      <br/>
      ■查询
      <br/>
      ■插入
      <br/>
      ■删除
      <br/>
      ■更新
     </li>
     <li>
      存取路径对用户隐蔽，用户只要指出“千什么 ’，不必详细说明‘ “怎么干”
     </li>
     <li>
      关系的完整性约束条件
      <br/>
      ■实体完整性
      <br/>
      ■参照完整性
      <br/>
      ■用户定义的完整性
     </li>
    </ul>
    <h3>
     <a id="_165">
     </a>
     ④关系数据库的优缺点
    </h3>
    <ul>
     <li>
      优点
      <br/>
      1.建立在严格的数学概念的基础上
      <br/>
      2.概念单一
      <br/>
      ●实体和各类联系都用关系来表示
      <br/>
      ●对数据的检索结果也是关系
      <br/>
      3.关系模型的存取路径对用户透明
      <br/>
      ●具有更高的数据独立性，更好的安全保密性
      <br/>
      ●简化了程序员的工作和数据库开发建立的工作
     </li>
     <li>
      缺点
      <br/>
      1.存取路径对用户透明，查询效率往往不如格式化数据模型
      <br/>
      2.为提高性能，必须对用户的查询请求进行优化，增加了开发数据库管理系统的难度
     </li>
    </ul>
    <h2>
     <a id="_179">
     </a>
     总结
    </h2>
    <p>
     <font color="#999AAA">
      数据库领域当前的研究工作都是以关系方法为基础，所以大家学习数据库的时候应该把重点放在关系数据库上。
     </font>
    </p>
