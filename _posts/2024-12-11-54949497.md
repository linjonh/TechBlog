    <p>
     从知乎上看到有一篇针对应用运维建设讲解的几个必备的方面，个人觉得对运维的整体建设写的不错，特转载到此收藏参考。方便自己以后从下面几个方面参考建设运维体系。
    </p>
    <p>
     <span style="font-family:宋体; font-size:14px">
      <br/>
     </span>
    </p>
    <p>
     <span style="font-family:宋体; font-size:14px">
      <br/>
     </span>
    </p>
    <p style="margin:0cm 0cm 0pt">
     <span style="font-size:14px">
      <span style="font-family:宋体">
       结合现在云计算和
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        DevOps
       </span>
      </span>
      <span style="font-family:宋体">
       的发展趋势，我觉得一个成熟的自动化运维平台应该包括以下的特性：
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <strong>
      <span style="font-size:14px">
       <span style="font-family:宋体">
        一、支持混合云的
       </span>
       <span lang="EN-US">
        <span style="font-family:Calibri">
         CMDB
        </span>
       </span>
      </span>
     </strong>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       现在越来越多的服务器都转到了云上，而主流的公有云、私有云平台都拥有比较完备的资源管理的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       ，这些
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       也就是构建一个自动化
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        CMDB
       </span>
      </span>
      <span style="font-family:宋体">
       的基础。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       新一代的自动化运维平台应该是可以基于这些
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       来自动维护和管理相关的服务器、存储、网络、负载均衡的资源的。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       通过
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       对资源的操作都应该被作为操作日志记录下来，以备作为后续操作审计的基础数据。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
       CMDB
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       这个东西听上去是老生常谈，但这个确实是所有运维工具的基础设施。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       而基于开源工具做运维平台最大的麻烦，就是如何在各个工具之间把
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        CMDB
       </span>
      </span>
      <span style="font-family:宋体">
       统一起来。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       CMDB
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       不统一起来，就意味着一旦要增加一台服务器，可能要在各个运维工具里面都要同步一下，这个还是非常折腾滴。。。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <strong>
       <span style="font-family:宋体">
        二、比较完备的监控
       </span>
       <span lang="EN-US">
        <span style="font-family:Calibri">
         +
        </span>
       </span>
       <span style="font-family:宋体">
        应用性能分析（
       </span>
       <span lang="EN-US">
        <span style="font-family:Calibri">
         APM
        </span>
       </span>
       <span style="font-family:宋体">
        ）
       </span>
      </strong>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       能支持对平台的可用性、服务器的性能、各种服务（
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        web
       </span>
      </span>
      <span style="font-family:宋体">
       服务、应用服务、数据库服务）的性能进行监控。做的好一些应该能进行更深入、或者关联性的性能分析。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       现在市面上一般都会将资源性能监控和应用性能监控（
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        APM
       </span>
      </span>
      <span style="font-family:宋体">
       ）混合着讲，这里面的产品确实也有很多都是重叠的，两方面都会涉及到。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       开源的性能监控系统主流有的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Zabbix
       </span>
      </span>
      <span style="font-family:宋体">
       、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Nagios
       </span>
      </span>
      <span style="font-family:宋体">
       ，国产的开源监控平台有小米
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        OpenFalcon
       </span>
      </span>
      <span style="font-family:宋体">
       ，但这些基本都只是做基本的资源监控（服务器，磁盘、网络等）和简单的服务软件的性能监控（中间件，数据库等）。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       而市面上的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        APM
       </span>
      </span>
      <span style="font-family:宋体">
       系统更主打的功能是应用性能分析，比如能精确定位到某个应用的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        URL
       </span>
      </span>
      <span style="font-family:宋体">
       的访问速度快慢，某些
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        SQL
       </span>
      </span>
      <span style="font-family:宋体">
       执行速度的快慢，这些对于开发人员和运维人员快速定位问题还是很有帮助的。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       APM
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       这方面的商业工具，国外比较主流的有
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        New Reclic
       </span>
      </span>
      <span style="font-family:宋体">
       、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Dynatrace
       </span>
      </span>
      <span style="font-family:宋体">
       ，国内的也就是透视宝、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Oneapm
       </span>
      </span>
      <span style="font-family:宋体">
       、听云等，他们也提供了
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       进行集成。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       APM
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       这方面的开源工具有
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        pinpoint
       </span>
      </span>
      <span style="font-family:宋体">
       （一个韩国团队开源的），
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        zipkin
       </span>
      </span>
      <span style="font-family:宋体">
       （
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        twitter
       </span>
      </span>
      <span style="font-family:宋体">
       开源），
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        cat
       </span>
      </span>
      <span style="font-family:宋体">
       （大众点评开源）。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <strong>
       <span style="font-family:宋体">
        三、有一个还不错
       </span>
       <span lang="EN-US">
        <span style="font-family:Calibri">
         UI
        </span>
       </span>
       <span style="font-family:宋体">
        的批量运维工具
       </span>
      </strong>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       在业务发展比较快的情况下，从几台服务器，到几十台服务器，再到几百台服务器，批量运维的需求很自然就产生了，老板也希望越少的人干越多的活。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       现在也有不少开源的批量运维工具，也都比较成熟了，比如
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        puppet
       </span>
      </span>
      <span style="font-family:宋体">
       、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        chef
       </span>
      </span>
      <span style="font-family:宋体">
       、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        ansible
       </span>
      </span>
      <span style="font-family:宋体">
       、
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        saltstack
       </span>
      </span>
      <span style="font-family:宋体">
       。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       puppet
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       和
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        chef
       </span>
      </span>
      <span style="font-family:宋体">
       都是
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        ruby
       </span>
      </span>
      <span style="font-family:宋体">
       做的，实话实说，
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        ruby
       </span>
      </span>
      <span style="font-family:宋体">
       的熟手市面上很少，比
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        python
       </span>
      </span>
      <span style="font-family:宋体">
       不是难招一点。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       我个人比较推荐使用
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        ansible
       </span>
      </span>
      <span style="font-family:宋体">
       或者
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        saltstack
       </span>
      </span>
      <span style="font-family:宋体">
       ，这两个系统都是
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        python
       </span>
      </span>
      <span style="font-family:宋体">
       写的，代码质量和社区活跃度都挺不错的。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       ansible
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       有官方的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        web ui
       </span>
      </span>
      <span style="font-family:宋体">
       ——
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Tower
       </span>
      </span>
      <span style="font-family:宋体">
       ，但实话实说不好用，所以我们也在重新做一套自己用起来更顺手的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        WEBUI
       </span>
      </span>
      <span style="font-family:宋体">
       。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
       <br/>
      </span>
     </span>
     <strong>
      <span style="font-family:宋体">
       <span style="font-size:14px">
        四、日志集中分析工具
       </span>
      </span>
     </strong>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       线上系统最常规的问题定位方式，就是日志分析了。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       随着服务器的增多，日志的分析定位也成为一个难点和痛点（想象一下，系统出故障之后，要去几十甚至数百个节点去上去查日志，是有多折腾）。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       国内有一家叫日志易的公司，是专门做日志分析方面的运维工具的。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       另外还有一家
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        log insight
       </span>
      </span>
      <span style="font-family:宋体">
       ，也是做这个领域，但产品好像还处于
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        beta
       </span>
      </span>
      <span style="font-family:宋体">
       阶段。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       日志分析这个领域现在是一个热点，现在的开源方案也比较多了，比如著名的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        ELKStack
       </span>
      </span>
      <span style="font-family:宋体">
       ，还有
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Flume+Kafka+Storm
       </span>
      </span>
      <span style="font-family:宋体">
       的体系。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       上面这两个方案相对重一些，部署比较复杂，网上介绍的文章也不少。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       比较轻量级的开源日志集中采集方案有
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        python
       </span>
      </span>
      <span style="font-family:宋体">
       做的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        Sentry
       </span>
      </span>
      <span style="font-family:宋体">
       ，他是通过改造各种语言的日志采集框架来实现日志的集中采集，各种主流的开发语言的日志框架都支持得很完整了，比如
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        java
       </span>
      </span>
      <span style="font-family:宋体">
       的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        log4j
       </span>
      </span>
      <span style="font-family:宋体">
       和
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        logpack
       </span>
      </span>
      <span style="font-family:宋体">
       。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       Sentry
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       的官网在此：
      </span>
     </span>
     <span lang="EN-US">
      <a href="https://link.zhihu.com/?target=https%3A//getsentry.com/welcome/" rel="nofollow noopener noreferrer" target="_blank">
       <span style="font-family:Calibri; font-size:14px; color:#0000ff">
        Sentry - Track exceptions with modern error logging for JavaScript,Python, Ruby, Java, and Node.js
       </span>
      </a>
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <strong>
      <span style="font-family:宋体">
       <span style="font-size:14px">
        五、持续集成和发布工具
       </span>
      </span>
     </strong>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       这方面其实比较难有统一的需求，很多公司集成发布的做法都差异挺大的。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       持续集成方面，一般用
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        jekins
       </span>
      </span>
      <span style="font-family:宋体">
       的比较多，这方面网上介绍的文章也很多。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       而如何把打好的包发布至各台服务器，则可以通过批量运维工具或者脚本来完成了。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       版本发布的过程涉及到很多细节，包括了版本文件的上传、分发、版本管理、回滚等各种操作。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       对于一般不太复杂的项目，我比较推荐的做法是把打包好的文件上传到
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        svn
       </span>
      </span>
      <span style="font-family:宋体">
       上，然后通过脚本在各台服务器上进行发布操作就行了，这样其实是利用了
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        SVN
       </span>
      </span>
      <span style="font-family:宋体">
       来完成文件的上传、分发、版本管理、回滚等各种操作。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <strong>
      <span style="font-family:宋体">
       <span style="font-size:14px">
        六、安全漏洞扫描工具
       </span>
      </span>
     </strong>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       现在一个稍微有点知名度的系统，都会遭受各种各样的安全攻击的折磨。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       一般的公司不太可能请得起专职的安全工程师，所以运维工程师最好能自己借助一些安全扫描工具来发现自己系统的漏洞。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       安全工具方面我了解不多，不太熟这个领域的开源工具。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-size:14px">
      <span style="font-family:宋体">
       之前乌云网推出过一个
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        SaaS
       </span>
      </span>
      <span style="font-family:宋体">
       化的漏扫平台——唐朝巡航，有对外提供漏洞扫描的
      </span>
      <span style="font-family:Calibri">
       <span lang="EN-US">
        API
       </span>
      </span>
      <span style="font-family:宋体">
       ，不过最近乌云网一直在升级，所以也就暂时无法调用了。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
       <br/>
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       个人觉得，如果上述功能都有了，基本上大部分中小规模企业的日常运维工作的高频操作都覆盖到了。
      </span>
     </span>
     <span lang="EN-US">
      <br/>
      <span style="font-family:Calibri; font-size:14px">
      </span>
     </span>
     <span style="font-family:宋体">
      <span style="font-size:14px">
       如果是比较大的互联网企业，或者还有一些特殊的业务需求，那就具体问题具体分析了。

     </span>
