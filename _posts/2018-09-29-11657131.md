     格林斯潘第十定律
    </h1>
   </div>

  <div class="article_content clearfix" id="article_content">

    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     <strong>
      格林斯潘第十定律
     </strong>
     是
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BC%96%E7%A8%8B" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="计算机编程">
      计算机编程
     </a>
     领域，尤其是
     <a href="http://zh.wikipedia.org/wiki/%E7%BC%96%E7%A8%8B%E8%AF%AD%E8%A8%80" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="编程语言">
      编程语言
     </a>
     领域的一句
     <a href="http://zh.wikipedia.org/wiki/%E6%A0%BC%E8%A8%80" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="格言">
      格言
     </a>
     ：
     <sup class="reference" id="cite_ref-1" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [1]
      </a>
     </sup>
     <sup class="reference" id="cite_ref-graham_2-0" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-graham-2" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [2]
      </a>
     </sup>
    </p>
    <blockquote class="toccolours" style="border:1px solid rgb(170,170,170); background-color:rgb(249,249,249); padding:0.3em 1em; font-size:14px; font-family:sans-serif; line-height:19.1875px; float:none">
     任何足够复杂的
     <a href="http://zh.wikipedia.org/wiki/C%E8%AF%AD%E8%A8%80" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="C语言">
      C语言
     </a>
     或者
     <a href="http://zh.wikipedia.org/wiki/Fortran" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Fortran">
      Fortran
     </a>
     程序中，都包含一个临时特设的、不合规范的、充满
     <a href="http://zh.wikipedia.org/wiki/%E7%A8%8B%E5%BA%8F%E9%94%99%E8%AF%AF" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="程序错误">
      程序错误
     </a>
     的、运行速度很慢的、只有一半功能的
     <a href="http://zh.wikipedia.org/wiki/Common_Lisp" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Common Lisp">
      Common Lisp
     </a>
     实现。
    </blockquote>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     这表现了
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/Lisp" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Lisp">
      Lisp
     </a>
     语言的灵活性和可扩展性，它包含了理论上编写复杂计算机程序需要的所有功能。而其他编程语言的核心实现却不能提供开发复杂程序的关键性功能支持。
    </p>
    <h3 style="font-weight:normal; margin:0px 0px 0.6em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(170,170,170); font-size:22px; font-family:sans-serif; line-height:19.1875px">
     <span class="mw-headline" id=".E6.9D.A5.E6.BA.90">
      来源
     </span>
    </h3>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     该定律由
     <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E8%8F%B2%E5%88%A9%E6%99%AE%C2%B7%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="菲利普·格林斯潘（页面不存在）">
      菲利普·格林斯潘
     </a>
     （Philip Greenspun）于1993年前后提出。尽管被称作“第十定律”，然而实际上并没有前九个定律。格林斯潘曾说这只是为了更好记。
     <sup class="reference" id="cite_ref-3" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-3" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [3]
      </a>
     </sup>
    </p>
    <h3 style="font-weight:normal; margin:0px 0px 0.6em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(170,170,170); font-size:22px; font-family:sans-serif; line-height:19.1875px">
     <span class="mw-headline" id=".E5.90.AB.E4.B9.89">
      含义
     </span>
    </h3>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     Lisp语法几乎是抽象的，其程序以
     <a href="http://zh.wikipedia.org/wiki/S-%E8%A1%A8%E8%BE%BE%E5%BC%8F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="S-表达式">
      S-表达式
     </a>
     形式构建，这等价于其他编程语言内置
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E7%BC%96%E8%AF%91%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="编译器">
      编译器
     </a>
     中的
     <a href="http://zh.wikipedia.org/wiki/%E6%8A%BD%E8%B1%A1%E8%AA%9E%E6%B3%95%E6%A8%B9" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="抽象語法樹">
      抽象语法树
     </a>
     。Lisp作为一种
     <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E5%90%8C%E5%83%8F&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="同像（页面不存在）">
      同像
     </a>
     （homoiconic）的语言，不区分代码与数据，即：Lisp程序本身即Lisp数据结构。Lisp程序自身可通过
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%AE%8F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="宏">
      宏
     </a>
     生成额外的Lisp程序。因此，程序员可以相对轻松地构建复杂的特定领域语言。
    </p>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     如同
     <a href="http://zh.wikipedia.org/wiki/%E4%BF%9D%E7%BD%97%C2%B7%E6%A0%BC%E9%9B%B7%E5%8E%84%E5%A7%86" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="保罗·格雷厄姆">
      保罗·格雷厄姆
     </a>
     所说，Lisp比起其他语言可以更快地构建更健壮的程序。
     <sup class="reference" id="cite_ref-4" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-4" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [4]
      </a>
     </sup>
    </p>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     格林斯潘第十定律表示，任何复杂软件系统都需要上述的功能，都将从头到尾重新设计开发这一套系统，而这在
     <a href="http://zh.wikipedia.org/wiki/Common_Lisp" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Common Lisp">
      Common Lisp
     </a>
     中已经存在了。
    </p>
    <h3 style="font-weight:normal; margin:0px 0px 0.6em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(170,170,170); font-size:22px; font-family:sans-serif; line-height:19.1875px">
     <span class="mw-headline" id=".E8.8E.AB.E9.87.8C.E6.96.AF.E6.8E.A8.E8.AE.BA">
      莫里斯推论
     </span>
    </h3>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     知名
     <a href="http://zh.wikipedia.org/wiki/%E9%BB%91%E5%AE%A2" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="黑客">
      黑客
     </a>
     <a href="http://zh.wikipedia.org/wiki/%E7%BD%97%E4%BC%AF%E7%89%B9%C2%B7%E6%B3%B0%E6%BD%98%C2%B7%E8%8E%AB%E9%87%8C%E6%96%AF" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="罗伯特·泰潘·莫里斯">
      罗伯特·泰潘·莫里斯
     </a>
     给该定律加上推论：
    </p>
    <blockquote class="toccolours" style="border:1px solid rgb(170,170,170); background-color:rgb(249,249,249); padding:0.3em 1em; font-size:14px; font-family:sans-serif; line-height:19.1875px; float:none">
     ……包括
     <a href="http://zh.wikipedia.org/wiki/Common_Lisp" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Common Lisp">
      Common Lisp
     </a>
     。
     <sup class="reference" id="cite_ref-yahoo_5-0" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-yahoo-5" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [5]
      </a>
     </sup>
    </blockquote>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     可以被认为是对编写庞大而复杂的
     <a href="http://zh.wikipedia.org/wiki/Common_Lisp" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Common Lisp">
      Common Lisp
     </a>
     程序困难性的评论，或仅仅是对于Lisp中eval函数的幽默。
    </p>
    <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
     格林斯潘第十定律和莫里斯推论都具备
     <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E9%BB%91%E5%AE%A2%E5%B9%BD%E9%BB%98&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="黑客幽默（页面不存在）">
      黑客幽默
     </a>
     中“哈哈，但的确如此”的风格特征。
     <sup class="reference" id="cite_ref-6" style="">
      <a href="http://zh.wikipedia.org/zh/%E6%A0%BC%E6%9E%97%E6%96%AF%E6%BD%98%E7%AC%AC%E5%8D%81%E5%AE%9A%E5%BE%8B#cite_note-6" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); white-space:nowrap" target="_blank">
       [6]
      </a>
     </sup>
    </p>
