    <p>
     微信小程序的发布大小限制是2MB。然而一个程序怎么能这么小？
     <br/>
     介绍一下项目中的经验。
    </p>
    <h2>
     <a id="_3">
     </a>
     新项目
    </h2>
    <p>
     如果是刚开始做的新项目，一定确定好自己要用的Ui框架，而且确定之后，千万不要引入别的，否则占大小！！！。【比如Uni-modules，又引入了uview】
    </p>
    <h2>
     <a id="_6">
     </a>
     老项目
    </h2>
    <ol>
     <li>
      图片通通都放到oss或者服务器里面，公司有cdn的更好!
     </li>
     <li>
      Hbuilder-&gt;运行-》运行到小程序模拟器-&gt;运行是否压缩代码【把这打钩】
     </li>
     <li>
      微信开发者根据设置
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d147214f5e55838d56f9024c42906078.png"/>
     </li>
    </ol>
    <h2>
     <a id="_13">
     </a>
     如果以上还不行，那就分包吧。
    </h2>
    <p>
     前提：
    </p>
    <ul>
     <li>
      主包不能大于2M
     </li>
     <li>
      一个分包不能大于2M
     </li>
     <li>
      可以有多个分包
     </li>
    </ul>
    <p>
     <strong>
      page.json里面设置分包页面
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c2729df8b752ab3caf0ae2c11f208b18.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/46341039c2e5a921c0a9304007811ad8.png">
       <br/>
       <strong>
        manifest.json，设置参数
       </strong>
      </img>
     </img>
    </p>
    <pre><code class="prism language-bash">“mp-weixin”：<span class="token punctuation">{<!-- --></span>

	<span class="token string">"optimization"</span><span class="token builtin class-name">:</span> <span class="token punctuation">{<!-- --></span>
