    <ol>
     <li>
      之前我们在项目中获取的
      <code>
       ajax
      </code>
      数据其实不是从后端获取的数据，而是我们通过一个接口
      <code>
       mock
      </code>
      模拟的假数据，当我们前端的代码已经编写完毕，后端的接口也已经写好的时候，就需要把我们前端编写的数据清掉，去尝试使用后端提供的数据。即需要前后端实现一个接口联调。
     </li>
     <li>
      首先把
      <code>
       static
      </code>
      目录下的
      <code>
       mock
      </code>
      数据都删掉，联调的时候我希望我的组件访问服务器
      <code>
       PHP
      </code>
      提供的数据：
     </li>
    </ol>
    <ul>
     <li>
      打开
      <code>
       config
      </code>
      目录，找到
      <code>
       index.js
      </code>
      ，我们之前的
      <code>
       proxyTable
      </code>
      设置是在开发环境下，如果我访问
      <code>
       api
      </code>
      开头的这个路径，他会帮助我们把这个路径的请求转发到
      <code>
       localhost：8080
      </code>
      的端口上，这是我们前端服务器的一个端口，这时我们想要访问后台服务器的端口上，后台服务器也在我们的本地，只不过他的服务器端口号在默认的80端口号上：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/02752cc3c65871cb340bf49c3b2bdf57.png">
       <br/>
       当我们改变了
       <code>
        config
       </code>
       里面的内容，需要重启一下服务器。
      </img>
     </li>
     <li>
      但是在实际应用中有可能后端的服务器不在我们的本地，在另一个后端开发人员的电脑上，或是一个内网或外网的服务器上，我们这个代理就不能写
      <code>
       localhost
      </code>
      这种形式了，可以去写一个内网的IP地址或者是外网的域名：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/599cb8bbe0501a03c2c247d1989baaa1.png"/>
     </li>
    </ul>
    <ol start="3">
     <li>
      实现
      <code>
       vue
      </code>
      上的前后端联调，只需要专注于
      <code>
       proxyTable
      </code>
      的配置就好了。
     </li>
    </ol>
