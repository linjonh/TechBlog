     云原生CAx软件：技术约束
    </h1>

    <p>
     Pital公司的Matt Stine于2013年首次提出
     <span style="color:#be191c;">
      <strong>
       Cloud Native
      </strong>
     </span>
     (
     <span style="color:#be191c;">
      <strong>
       云原生
      </strong>
     </span>
     )的概念，从概念提出到技术落地，云原生还处于不断发展过程中，关于云原生的定义也不断在完善。
    </p>
    <blockquote>
     <p style="margin-left:.0001pt;text-align:justify;">
      云原生使用开源软件堆栈来创建容器化、动态编排和面向微服务的应用程序。
     </p>
    </blockquote>
    <p>
     Heroku创始人 Adam Wiggins 在2012年提出的"十二要素原则"用于指导设计与开发云原生应用程序。
    </p>
    <blockquote>
     <p>
      Ref. from
      <a class="link-info" href="https://12factor.net/" rel="nofollow" title="The Twelve Factors">
       The Twelve Factors
      </a>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       I. Codebase
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      One codebase tracked in revision control, many deploys
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       II. Dependencies
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Explicitly declare and isolate dependencies
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       III. Config
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Store config in the environment
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       IV. Backing services
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Treat backing services as attached resources
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       V. Build, release, run
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Strictly separate build and run stages
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       VI. Processes
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Execute the app as one or more stateless processes
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       VII. Port binding
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Export services via port binding
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       VIII. Concurrency
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Scale out via the process model
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       IX. Disposability
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Maximize robustness with fast startup and graceful shutdown
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       X. Dev/prod parity
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Keep development, staging, and production as similar as possible
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       XI. Logs
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Treat logs as event streams
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      <strong>
       XII. Admin processes
      </strong>
     </p>
     <p style="margin-left:.0001pt;text-align:justify;">
      Run admin/management tasks as one-off processes
     </p>
    </blockquote>
    <p>
     与此同时，云原生的概念与技术也在逐渐影响
     <strong>
      CAx
     </strong>
     (
     <strong>
      CAD
     </strong>
     /
     <strong>
      CAE
     </strong>
     /
     <strong>
      CAM
     </strong>
     )
     <strong>
      软件
     </strong>
     的架构设计与开发，要考虑设计与实现
     <strong>
      原生为云的CAx软件
     </strong>
     ：
     <strong>
      云原生CAx软件
     </strong>
     (
     <strong>
      Cloud Native CAx System
     </strong>
     )
    </p>
    <p>
     <span style="color:#be191c;">
      <strong>
       狭义上来说，云原生CAx软件实际上就是采用微服务架构、以Kubernetes容器集群为目标平台的CAx软件。
      </strong>
     </span>
    </p>
    <p style="text-align:center;">
     <span style="color:#be191c;">
      <strong>
       <span style="color:#be191c;">
        <strong>
         <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/880c4e1056a397052867e8b93665f957.png"/>
        </strong>
       </span>
      </strong>
     </span>
    </p>
    <p>
     目前，国内已有厂商也在从事
     <strong>
      云原生CAx
     </strong>
     (
     <strong>
      CAD
     </strong>
     /
     <strong>
      CAE
     </strong>
     /
     <strong>
      CAM
     </strong>
     )相关的研究，并推出了相应的
     <strong>
      云原生CAx
     </strong>
     产品。作为一种学习与研究方法，如果抛开具体的业务需求，可以尝试按照以下思路对这些云原生CAx软件进行分析：
    </p>
    <ol>
     <li>
      <span style="color:#be191c;">
       <strong>
        知名(云原生)CAx软件采用的架构模式；
       </strong>
      </span>
     </li>
     <li>
      <span style="color:#be191c;">
       <strong>
        架构模式中技术约束的作用；
       </strong>
      </span>
     </li>
     <li>
      <span style="color:#be191c;">
       <strong>
        归纳总结(云原生)CAx软件应当满足的技术约束。
       </strong>
      </span>
     </li>
    </ol>
    <p>
     在前面的博文中，已就云原生CAx软件设计与开发过程中可能涉及的基础知识进行了总结，但尚未对云原生CAx软件的技术约束进行研究。
    </p>
    <blockquote>
     <p>
      Software architecture research investigates methods for determining how best to partition a system, how components identify and communicate with each other, how information is communicated, how elements of a system can evolve independently, and how all of the above can be described using formal and informal notations. ------
      <a class="link-info" href="https://www.ics.uci.edu/~fielding/pubs/dissertation/top.htm" rel="nofollow" title="Roy Thomas Fielding">
       Roy Thomas Fielding
      </a>
     </p>
    </blockquote>
    <p>
     因此，本文结合技术研究与工作实践，拟对云原生CAx软件设计和开发的技术约束进行系统性总结。
    </p>
