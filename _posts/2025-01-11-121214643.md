    <h2>
     <a id="_0">
     </a>
     动机
    </h2>
    <p>
     使用vscode通过ssh进行远程服务器连接的原因是，单纯通过使用teamview等方式进行远程服务器的连接往往会受到网速的影响，而通过SSH进行局域网内部的连接则可以完美的避免这一问题。同时，vscode能够实时的显示服务器上的代码界面，还是相当方便的。
    </p>
    <h2>
     <a id="_2">
     </a>
     前提
    </h2>
    <p>
     由于我们是通过局域网进行连接，因此前提是，本地电脑和服务器置于同一局域网，测试的方法通过cmd中的ping 即可，能ping通即是在同一局域网中。
     <br/>
     linux系统查看本机本地的ip地址可以通过以下命令
    </p>
    <pre><code class="prism language-bash"><span class="token function">ifconfig</span> -a （找到192.168.开头的）
</code></pre>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0e0ec44e9934ba03aa85c18cf531a18f.png"/>
    </p>
    <h3>
     <a id="_10">
     </a>
     步骤（为了方便理解，将我们自己的电脑称为本地电脑）
    </h3>
    <h4>
     <a id="vscodeeasy_11">
     </a>
     一、本地电脑安装vscode（easy）
    </h4>
    <h4>
     <a id="opensshserver_12">
     </a>
     二、服务器终端安装openssh-server
    </h4>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> openssh-server
</code></pre>
    <p>
     其中关于服务器上ssh有以下几条常用的指令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span>  <span class="token function">service</span>  sshd  start（启动 ssh）
<span class="token function">sudo</span> <span class="token function">service</span> <span class="token function">ssh</span> --full-restart （修改配置文件后，重启ssh）
</code></pre>
    <h4>
     <a id="ssh_23">
     </a>
     三、服务器上ssh配置设置
    </h4>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> gedit /etc/ssh/sshd_config
</code></pre>
    <p>
     找到以下选项，进行设置即可
    </p>
    <p>
     <strong>
      Port 22
      <br/>
      PermitRootLogin yes
      <br/>
      PubkeyAuthentication yes
     </strong>
    </p>
    <h4>
     <a id="vscode_32">
     </a>
     四、vscode上进行配置
    </h4>
    <h5>
     <a id="31_remote__ssh_33">
     </a>
     3.1 remote - ssh安装
    </h5>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d01d7b2ed9da717ed85bdb15764f78d5.png"/>
    </p>
    <h5>
     <a id="32_config_35">
     </a>
     3.2 config文件配置
    </h5>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/de5daac82b239566f308b6c87fe96048.png">
      <br/>
      进入到配置界面：
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># Read more about SSH config files: https://linux.die.net/man/5/ssh_config</span>
Host cumt
    HostName <span class="token number">192.168</span>.x.xxx <span class="token comment">#服务器ip，就是刚才我们ping通的那个</span>
    User cumt <span class="token comment"># 服务器上的用户名</span>
    Port <span class="token number">22</span> <span class="token comment"># ssh端口，一般默认22， 可以修改</span>
</code></pre>
    <h5>
     <a id="33__remotessh__46">
     </a>
     3.3 文件-》首选项-》设置-》搜索框中搜索 remote-ssh 找到以下选项打勾即可
    </h5>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/87ef9d31682aa38d50c016ffd43b879c.png"/>
    </p>
    <h4>
     <a id="_50">
     </a>
     五、连接
    </h4>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1de5c03f1c1b7f26e95b7e90095d27c5.png">
      <br/>
      之后跳出的选项， linux、windows、os任选一个即可。
      <br/>
      若出现报错：
      <strong>
       试图写入的管道不存在，大概率是因为
      </strong>
      ，
      <strong>
       本地电脑的ssh文件损坏
      </strong>
      ，检验方式如下：打开
      <strong>
       C:\Windows\System32\OpenSSH\ssh.exe
      </strong>
      若是打不开则是说明ssh.exe文件损坏，不用着急，若是你的电脑上有Git，Git里面是有ssh的，若是没有就下载一个Git。
      <br/>
      将默认的ssh.exe进行替换。
      <br/>
      <strong>
       文件-》首选项-》设置-》搜索框中搜索 remote-ssh-》找到 在settings.json中进行编辑。
      </strong>
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">{<!-- --></span>
    <span class="token string">"workbench.colorTheme"</span><span class="token builtin class-name">:</span> <span class="token string">"Default Dark+"</span>,
    <span class="token string">"window.zoomLevel"</span><span class="token builtin class-name">:</span> <span class="token number">1</span>,
    <span class="token string">"remote.SSH.showLoginTerminal"</span><span class="token builtin class-name">:</span> true,
    <span class="token string">"remote.SSH.path"</span><span class="token builtin class-name">:</span> <span class="token string">"C:/Program Files/Git/usr/bin/ssh.exe"</span>, <span class="token comment">#加入这行代码，即Git中的ssh文件路径</span>
    <span class="token string">"remote.SSH.remotePlatform"</span><span class="token builtin class-name">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string">"cumt"</span><span class="token builtin class-name">:</span> <span class="token string">"linux"</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     再次连接即可成功，祝好！
    </p>
