     <a class="replace_word" href="http://lib.csdn.net/base/mysql" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="MySQL知识库">

    <h3 id="定义后端与安全性" style="margin:0.8em 0px; padding:0px; font-weight:100; font-family:'microsoft yahei'; line-height:26px">
     <a name="t0" style="color:rgb(202,0,0)" target="_blank">
     </a>
     定义：后端与安全性
    </h3>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     事先声明一下：我只讨论后端。所谓的后端，指的是处理服务器请求的数据接入层。它需要肩负着处理很多请求（大于100请求每秒）和快速处理请求（平均处理时间小于300ms）的职责。而那些可以被提前计算，或者是不需要直接展示给客户端的繁重计算的工作，则并不适合用后端进行处理。这些工作可以通过其他的语言通过另外的方式实现。后端也并不直接处理对用户的展现，这是前端应该做的。但是后端开发者，应该对如何给前端提供数据有着清晰的认知。我所见过最清晰的方式就是在前后端之间使用纯Restful API进行交互。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     后端语言也应该很容易校验前端发来的数据，方便的连接
     <a class="replace_word" href="http://lib.csdn.net/base/mysql" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="MySQL知识库">
      数据库
     </a>
     ，以及从文件系统保存或是获取文件。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     在接下来的篇幅中，我会描述语言的安全性。但这并不意味着你不能用不安全的语言写出安全的代码，而是不同语言编译器能够保证你的代码不出现某些特定的bug。举例而言，C的编译器并不进行边界检查，因此它是一种非常不安全的语言。而那些可以被自动工具检查出来的错误包括：
    </p>
    <ul style="font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     <li style="">
      语法错误
     </li>
     <li style="">
      读越界
     </li>
     <li style="">
      内存溢出（C/C++并不检查，但在java中几乎不可能）
     </li>
     <li style="">
      未被使用的变量
     </li>
     <li style="">
      类型问题：这个问题略微有些模棱两可，因为在每种语言你都能写出强类型的代码。但是在一些语言中，类型更为常见。有些语言同样也使用类型系统来发现错误。举例而言，php从这个意义上来说十分不安全。连
      <code style="font-family:'Source Code Pro',monospace; padding:2px 4px; font-size:12.6px; color:rgb(63,63,63); white-space:nowrap">
       123=="123ab"
      </code>
      都可以为true。 Python稍微安全一点，但你却可以随意的返回值。Java相比之下安全多了。Haskel从这个层面来说更加安全，因为它的强类型系统会拒绝执行任何无意义的表达式，同时也不会进行任何形式的强制转换。
     </li>
    </ul>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     在程序的运行时，也有一些错误会被发现。而在不同的语言中，对这种运行时错误的处理完全不同。举例来说，C和C++会默默的失败。（就像这个
     <a href="http://stackoverflow.com/questions/671703/array-index-out-of-bound-in-c" rel="nofollow noopener noreferrer" style="color:rgb(202,0,0); text-decoration:none" target="_blank">
      链接
     </a>
     里说的） 这当然是很不好的。相比C/C++对越界错误的忽略，
     <a class="replace_word" href="http://lib.csdn.net/base/rust" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="Rust知识库">
      Rust
     </a>
     则会发出明显的提示。（之前的heartbleed是一个典型的例子，可以看看
     <a href="http://tonyarcieri.com/would-rust-have-prevented-heartbleed-another-look" rel="nofollow noopener noreferrer" style="color:rgb(202,0,0); text-decoration:none" target="_blank">
      这篇文章
     </a>
     ）
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     当然了，所有这些问题都可以通过良好的
     <a class="replace_word" href="http://lib.csdn.net/base/softwaretest" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="软件测试知识库">
      测试
     </a>
     来发现。但是，能够被自动化的过程越多，当你测试不那么充分的时候，可能发生的意外也就越少。
    </p>

    </p>
    <ul style="font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     <li style="">
      非常笨拙的语法。为了完成一个功能，你可能要打很多字。。当然自动补全是必不可少的，但是要打的东西还是不少。代码越多，维护起来成本越高。
     </li>
     <li style="">
      要把一个工具跑起来难度很高
     </li>
     <li style="">
      那些被设计用来扩展的super-abstract结构，事实上可能永远不会被用到。
     </li>
     <li style="">
      因为没有指针的存在，Java不能访问越界的数组，所以比C/C++安全一些。那些烦恼你的内存溢出也几乎不存在了。但是付出的代价就是，语法更加的复杂，而且你如果一不小心，就会掉入Java中得不安全的陷阱。
     </li>
     <li style="">
      速度和内存使用量：Java在速度上并不突出，而在内存使用上则是相当的多。但是这并没给它的大范围使用带来太多的问题。感兴趣的可以看看这篇
      <a href="http://www.infoworld.com/article/2609675/java/surprise--java-is-fastest-for-server-side-web-apps.html" rel="nofollow noopener noreferrer" style="color:rgb(202,0,0); text-decoration:none" target="_blank">

    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">

      Node.js
     </a>
     自从2009年发布一下，使用十分广泛。
     <a class="replace_word" href="http://lib.csdn.net/base/nodejs" rel="nofollow noopener noreferrer" style="color:rgb(223,52,52); text-decoration:none; font-weight:bold" target="_blank" title="Node.js知识库">
      Node.js
     </a>
     具有异步，事件驱动以及可扩展的特性。Node.js应用是由JavaScript开发的，因此它具有JavaScript的所有优点：
    </p>
    <ul style="font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     <li style="">
      享受到了JavaScript V8引擎的好处
     </li>
     <li style="">
      灵活而轻量级的语法
     </li>
     <li style="">
      跟Java一样，JavaScript出现在1995年。所以语言本身发展历史很长，相对稳定。
      <br style=""/>
      <ul style="">
       <li style="">
        很多开发者都或多或少的了解一些JavaScript
       </li>

     结论

    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     仔细想来，对我来说以下的语言更适合我做后端：
    </p>
    <ul style="font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     <li style="">
      Go：很快，而且是编译型的
     </li>
     <li style="">
      node.js: 很好的可扩展性
     </li>
     <li style="">
      Python: 这是我最了解的语言。除此之外，它有非常清晰的语法，很好的社区开发的变法风格，以及易读性。
     </li>
    </ul>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     对我来说不适合的语言：
    </p>
    <ul style="font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     <li style="">
      PHP: 因为语言的不一致，导致建立稳定的后端没那么简单。
     </li>
     <li style="">
      C#: 这个语言太以微软为中心了
     </li>
     <li style="">
      Java：语法有点拖沓，成本高。
     </li>
    </ul>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; font-family:'microsoft yahei'; font-size:14px; line-height:26px">
     其他语言可能也是很好的选择，只是我知之甚少。我对rust在后端使用的可行性非常好奇。Hack语言非常年轻，让我们对它未来的发展拭目以待。
