    <h3 id="31-python特性">
     3.1 Python特性
    </h3>
    <p>
     Python是为可读性设计的，与英语有一些相似之处，并受到数学的影响。比如：
    </p>
    <ul>
     <li>
      Python使用新行来完成命令，而不像通常使用分号或括号的其他编程语言。
     </li>
     <li>
      Python依赖缩进，使用空格来定义范围；例如循环、函数和类的范围。其他编程语言通常使用花括号或关键字来实现此目的。
     </li>
     <li>
      Python中没有单个字符，单字符被认为是1个字符的字符串。
     </li>
     <li>
      对象有类型，变量无类型
     </li>
    </ul>
    <h4 id="311-python注释">
     3.1.1 Python注释
    </h4>
    <p>
     <code>
      #
     </code>
     ：单行注释，Python解释器会忽略
     <code>
      #
     </code>
     号后面跟着的所有的字符。
     <br/>
     &gt;&gt;&gt; # 这里的内容是注释
     <br/>
     <code>
      '''
     </code>
     或
     <code>
      """
     </code>
     ：多行注释。Python解释器会忽略两组
     <code>
      '''
     </code>
     或
     <code>
      """
     </code>
     符号内多行内容。
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         """
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         这是注释的第1行
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         这是注释的第2行
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         """
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <h4 id="312-python物理行和逻辑行">
     3.1.2 Python物理行和逻辑行
    </h4>
    <p>
     <code>
      物理行
     </code>
     ：当我们编写程序是在软件中看到的一行。
     <br/>
     <code>
      逻辑行
     </code>
     ：Python看到并识别为一条单独语句。根据
     <code>
      显式行拼接
     </code>
     或
     <code>
      隐式行拼接
     </code>
     规则，一个或多个物理行可组成逻辑行。
     <br/>
     Python默认一个物理行为一个逻辑行。默认情况下，Python鼓励一行写一个语句的用法，这使代码更具有可读性。
    </p>
    <p>
     3.1.2.1 物理行显式拼接成逻辑行
    </p>
    <p>
     两个及两个以上的物理行可用反斜杠
     <code>
      \
     </code>
     显式地拼接为一个逻辑行，规则如下：以不在字符串或注释内的反斜杠结尾时，物理行将与下一行拼接成一个逻辑行，并删除反斜杠及其后的换行符。例如：
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         if 1900 &lt; year &lt; 2100 and 1 &lt;= month &lt;= 12 \
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         and 1 &lt;= day &lt;= 31 and 0 &lt;= hour &lt; 24 \
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         and 0 &lt;= minute &lt; 60 and 0 &lt;= second &lt; 60:
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         return 1
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      注意：以反斜杠结尾的行，不能加注释；反斜杠也不能拼接注释。
     </p>
    </blockquote>
    <p>
     3.1.2.2 物理行隐式拼接成逻辑行
    </p>
    <p>
     圆括号、方括号、花括号内的表达式可以分成多个物理行，不必使用反斜杠。例如：
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         month_names = ['Januari', 'Februari', 'Maart', # These are the
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         'April', 'Mei', 'Juni', # Dutch names
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         'Juli', 'Augustus', 'September', # for the months
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         'Oktober', 'November', 'December'] # of the year
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      说明：隐式行拼接可含注释；后续行的缩进并不重要；还支持空的后续行。
     </p>
    </blockquote>
    <p>
     3.1.2.3 逻辑行拼接成物理行
    </p>
    <p>
     Python默认将
     <code>
      ;
     </code>
     作为逻辑行的结束符号。通过使用
     <code>
      ;
     </code>
     可以将多条语句拼接成一条物理行。
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         x=5;print(x);
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <h4 id="313-python编码声明">
     3.1.3 Python编码声明
    </h4>
    <p>
     Python 脚本第一或第二行的注释匹配正则表达式
     <code>
      coding[=:]\s*([-\w.]+)
     </code>
     时，该注释会被当作编码声明；这个表达式的第一组指定了源码文件的编码。编码声明必须独占一行，在第二行时，则第一行必须也是注释。编码表达式的形式如下：
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         # -*- coding: &lt;encoding-name&gt; -*-
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     我们在实际编程中，可以在脚本的第一句写：
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         # encoding: utf-8
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     上面代码的主要作用是指定文件编码为UTF-8, 因为Python2.x一般默认的是ASCII码，如果要在文件里面写中文，运行时会出现乱码，加上这句之后会把文件编码强制转换为UTF-8运行时就不会出现乱码了。
     <br/>
     好在默认情况下，Python3.x 源码文件以 UTF-8 编码，所有字符串都是 unicode 字符串。当然也可以为源码文件指定不同的编码，比如：
    </p>
    <pre></pre>
    <table>
     <tbody>
      <tr>
       <td>
        <code>
         # -*- coding: cp-1252 -*-
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     上述定义允许在源文件中使用 Windows-1252 字符集中的字符编码，对应适合语言为保加利亚语、白罗斯语、马其顿语、俄语、塞尔维亚语。
    </p>
    <h4 id="314-python缩进">
     3.1.4 Python缩进
    </h4>
    <p>
     在用Python编写程序时经常会遇到语句块，例如循环、函数和类的范围。其他编程语言通常使用花括号来实现此目的。但是Python是通过严格数量的缩进(空格)来实现的。Python这门语言在编写程序的时候对缩进及其敏感。一般默认每一层语句块都向右缩进'4'个空格(可以在编辑器中修改)。这使得Python编写后的程序极具美感且省下了大量无用的行。
     <br/>
     实际编写中，在使用编辑器或IDE时，一般使用
     <code>
      Tab
     </code>
     键(制表)代替空格。毕竟谁也不想因为缩进而每次要按4次空格。最好不要混用
     <code>
      Tab
     </code>
     键(制表)和空格进行缩进，因为空格数量与制表符相关，由此很容易产生的不一致将导致不能正常识别缩进层次，从而触发
     <code>
      TabError
     </code>
     报错。
    </p>
    <p>
     3.1.4.1 缩进规则
    </p>
    <p>
     1、逻辑行的
     <code>
      首行
     </code>
     需要顶格，即无缩进(也就是一份源码的第一个逻辑行)
     <br/>
     2、相同逻辑层保持相同的缩进
     <br/>
     3、
     <code>
      :
     </code>
     标记一个新的逻辑层，增加缩进表示进入下一个代码层，减少缩进表示返回上一个代码层
    </p>
    <h4 id="315-python空行">
     3.1.5 Python空行
    </h4>
    <p>
     函数之间或类的方法之间用空行分隔，表示一段新的代码的开始。类和函数入口之间也用一行空行分隔，以突出函数入口的开始。
     <br/>
     空行与代码缩进不同，空行并不是 Python 语法的一部分。书写时不插入空行，Python 解释器运行也不会出错。但是空行的作用在于分隔两段不同功能或含义的代码，便于日后代码的维护或重构。
    </p>
    <blockquote>
     <p>
      说明：空行也是程序代码的一部分。
     </p>
    </blockquote>
    <h4 id="316-python标识符">
     3.1.6 Python标识符
    </h4>
    <p>
     标识符(也称为名称)，其规则如下：
    </p>
    <ol>
     <li>
      第一个字符必须是字母表中字母或下划线
      <code>
       _
      </code>
      。
     </li>
     <li>
      标识符的其他的部分由字母、数字和下划线组成。
     </li>
     <li>
      标识符的长度没有限制，但区分大小写。
     </li>
     <li>
      在 Python 3 中，可以用中文作为标识符，非 ASCII 标识符也是允许的了。
     </li>
    </ol>
