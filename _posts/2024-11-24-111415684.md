    </h3>
    <h4>
     <a id="_8">
     </a>
     树立正确的网络安全观
    </h4>
    <ol>
     <li>
      网络安全是
      <strong>
       整体
      </strong>
      的而不是
      <strong>
       割裂
      </strong>
      的
     </li>
     <li>
      网络安全是
      <strong>
       动态
      </strong>
      的而不是
      <strong>
       静态
      </strong>
      的
     </li>
     <li>
      网络安全是
      <strong>
       开放
      </strong>
      的而不是
      <strong>
       封闭
      </strong>
      的
     </li>
     <li>
      网络安全是
      <strong>
       相对
      </strong>
      的而不是
      <strong>
       绝对
      </strong>
      的
     </li>
     <li>
      网络安全是
      <strong>
       共同
      </strong>
      的而不是
      <strong>
       孤立
      </strong>
      的
     </li>
    </ol>
    <p>
     <strong>
      网络主权
     </strong>
     一国国家主权在网络空间中的自然延伸和表现。
    </p>

    <h3>
     <a id="_22">
     </a>
     网络安全基本属性
    </h3>
    <h4>
     <a id="_23">
     </a>
     保密性
    </h4>
    <p>
     <strong>
      保密性
     </strong>
     是指信息不被泄露给非授权用户、实体或过程，被其他利用的特性。
    </p>
    <h4>
     <a id="_25">
     </a>
     完整性
    </h4>
    <p>
     <strong>
      完整性
     </strong>
     是指信息
     <strong>
      未经授权不能进行更改
     </strong>
     的特性。
    </p>
    <p>
     信息在
     <strong>
      存储或传输过程中
     </strong>
     保持不被偶然或蓄意的
     <strong>
      删除
     </strong>
     、
     <strong>
      修改
     </strong>
     、
     <strong>
      伪造
     </strong>
     、
     <strong>
      乱序
     </strong>
     、
     <strong>
      重放
     </strong>
     、
     <strong>
      插入
     </strong>
     等破坏和
     <strong>
      丢失
     </strong>
     的特性。
    </p>
    <p>
     影响因素：
     <strong>
      设备故障
     </strong>
     、
     <strong>
      无码
     </strong>
     、
     <strong>
      人为攻击
     </strong>
     、
     <strong>
      计算机病毒
     </strong>
    </p>
    <h4>
     <a id="_32">
     </a>
     可用性
    </h4>
    <p>
     <strong>
      可用性
     </strong>
     是信息可被
     <strong>
      授权实体访问
     </strong>
     并
     <strong>
      按需使用
     </strong>
     的特性。
    </p>
    <p>
     因素：
     <strong>
      软件可用性
     </strong>
     、
     <strong>
      硬件可用性
     </strong>
     、
     <strong>
      人员可用性
     </strong>
     、
     <strong>
      环境可用性
     </strong>
    </p>
    <h3>
     <a id="_37">
     </a>
     网络安全演变
    </h3>
    <h4>
     <a id="_38">
     </a>
     通信保密
    </h4>
    <p>
     主要威胁：搭线窃听、密码学分析
     <br/>
     防护措施：数据加密
     <br/>
     主要关心对象：军方和政府
    </p>
    <h4>
     <a id="_43">
     </a>
     计算机安全
    </h4>
    <p>
     计算系统的软硬件及处理、存储、传输信息的保密性
     <br/>
     若干安全威胁：
     <strong>
      恶意代码
     </strong>
     、
     <strong>
      黑客
     </strong>
     、
     <strong>
      外部访问
     </strong>
     、
     <strong>
      恶意用户
     </strong>
     、
     <strong>
      口令窃取
     </strong>
     、
     <strong>
      私人通信
     </strong>
    </p>
    <h4>
     <a id="_46">
     </a>
     信息系统安全
    </h4>
    <p>
     <strong>
      不可否认性
     </strong>
     即信息的
     <strong>
      发送者
     </strong>
     或
     <strong>
      接收者
     </strong>
     事后
     <strong>
      不能否认
     </strong>
     发送和接收的行为
    </p>

      </code>
      ，则
      <code>
       b≡a (mod r)
      </code>
      ;
     </li>
     <li>
      传递性，若
      <code>
       a≡b (mod r)
      </code>
      ，
      <code>
       b≡c (mod r)
      </code>
      ，则
      <code>
       a≡c (mod r)
      </code>
     </li>
    </ol>
    <p>
     算法过程：
    </p>
    <ol>
     <li>
      秘钥产生
      <br/>
      1、选择两个素数
      <code>
       p
      </code>
      、
      <code>
       q
      </code>
      <br/>
      2、计算：
      <code>
       n=p*q
      </code>
      ;
      <code>
       φ（n）=(p-1)*(q-1)
      </code>
      <br/>
      3、随机选择一个整数
      <code>
       e
      </code>
      ,与
      <code>
       φ（n）
      </code>
      互素
      <br/>
      4、找到一个整数d,满足
      <code>
       ed ≡1 mod φ（n）
      </code>
     </li>
     <li>
      加解密算法
      <br/>
      加密算法：
      <code>
       c = m∧e mod n
      </code>
      <br/>
      解密算法：
      <code>
       m = c∧d mod n
      </code>
     </li>
    </ol>

    <h4>

     BLP模型
    </h4>
    <p>
     信息系统中的所有的实体可以分为
     <strong>
      实体
     </strong>
     和
     <strong>
      客体
     </strong>
     <br/>

     <strong>

     </strong>
     ，如
     <strong>
      用户
     </strong>
     ，
     <strong>
      进程
     </strong>
     ，可以
     <strong>
      主动发起操作
     </strong>
     <br/>
     客体：作为
     <strong>
      操作对象的实体
     </strong>
     ，如
     <strong>
      文件
     </strong>
     、
     <strong>
      存储设备
     </strong>
     ，可以看做
     <strong>
      信息内容的容器
     </strong>
    </p>
    <p>
     保密级别：绝密、机密、秘密、公开
     <br/>
     BLP基本规则：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/68e6fe56515b12321759fac151094b90.png"/>
    </p>
    <ol>
     <li>
      规则1：一个主体对客体进行读访问的必要条件是主体的安全级别大于或等于客体的安全级别，主体的保密级别不小于客体的保密级别，主体的范畴包含客体的全部范畴
     </li>
     <li>
      一个主体的进行写访问的必要条件是客体安全级大于主体的安全级，即客体的保密级不小于主体的保密级，客体的范畴集合包含主体的全部范畴。
      <br/>
      规则1称为简单安全规则
      <br/>
      规则2*-属性
     </li>
    </ol>
    <blockquote>
     <p>
      BLP 模型实现一个单向的信息流策略，合法的信息从低安全级别主体、客体流到高安全级别的主体和客体，逆向的信息流动是禁止。
     </p>
    </blockquote>

     等，由低到高划分为五级。
    </p>
    <p>
     <strong>
      五级标准：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       第一级是指信息系统受到破坏后，会对公民、法人和其他组织的合法权益造成损害，但不损害国家安全、社会秩序和公共利益。
      </strong>
     </li>
     <li>
      <strong>
       第二级，信息系统受到破坏后，会对公民、法人和其他组织的合法权益产生严重损害，或者对社会秩序和公共利益造成损害，但不损害国家安全。
      </strong>
     </li>
     <li>
      <strong>
       第三级，信息系统受到破坏后，会对社会秩序和公共利益造成严重损害，或者对国家安全造成损害。
      </strong>
     </li>
     <li>
      <strong>
       第四级，信息系统受到破坏后，会对社会秩序和公共利益造成特别严重损害，或者对国家安全造成严重损害。
      </strong>
     </li>
     <li>
      <strong>
       第五级，信息系统受到破坏后，会对国家安全造成特别严重损害。
      </strong>
     </li>
    </ol>
