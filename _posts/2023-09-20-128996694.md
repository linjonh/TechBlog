    <h3 style="margin-left:.0001pt;text-align:justify;">
     1.1.1认识数据库
    </h3>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      <h4>
       数据与信息
      </h4>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     数据：是对客观事物、事件的记录与描述，也是对客观事物的逻辑归纳。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="background-color:#ffff00;">
      信息=数据+语义
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      <h4>
       数据处理与数据管理
      </h4>
      <ul>
       <li>
        <ul>
         <li style="text-align:justify;">
          目的：从大量原始数据抽取和推导有价值信息，作为决策依据。
         </li>
         <li style="text-align:justify;">
          借助计算机科学保存和管理复杂数据，便于充分利用信息资源。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     数据处理：将数据转换成信息，对数据进行
     <span style="background-color:#ffff00;">
      采集、管理、加工、变换和传输
     </span>
     等活动。
    </p>
    <ol>
     <li>
      <ol>
       <li style="text-align:justify;">
       </li>
       <li style="text-align:justify;">
        数据管理：数据处理的
        <span style="background-color:#ffff00;">
         核心
        </span>
        。包括
        <span style="background-color:#ffff00;">
         数据的分类、组织、编码、存储、维护、检索
        </span>
        。
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1.1.2数据库、数据库管理系统、数据库系统
    </p>
    <ol>
     <li>
      <ol>
       <li style="text-align:justify;">
        数据库：database DB，储存在计算机内，有组织的，可共享的
        <span style="background-color:#ffff00;">
         相关数据的集合
        </span>
        。
       </li>
       <li style="text-align:justify;">
        数据库管理系统：database management system DBMS。建立、运用、管理、维护数据库，并对数据库进行统一
        <span style="color:#0000ff;">
         控制
        </span>
        的
        <span style="background-color:#ffff00;">
         系统软件
        </span>
        。
        <span style="color:#0000ff;">
         常见数据库管理系统：SQL、server、MySQL、access。
        </span>
       </li>
       <li style="text-align:justify;">
        数据库系统：database system DBS。指在计算机系统中引入数据库后的系统。简称：
        <span style="background-color:#ffff00;">
         <span style="color:#0000ff;">
          数据库
         </span>
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     其中数据库用户：
     <span style="color:#0000ff;">
      开发、管理、使用数据库
     </span>
     的人。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     包括：
     <span style="color:#0000ff;">
      系统分析员（
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       确定应用系统的需求分析和规范说明，他和最终用户，数据库管理员确定系统的硬件配置，参与数据库的概要设计
      </span>
     </span>
     <span style="color:#0000ff;">
      ）、数据库设计员（
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       确定数据库的数据和设计数据库的各级模式
      </span>
     </span>
     <span style="color:#0000ff;">
      ）、应用程序员（
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       编写使用数据库的应用程序
      </span>
     </span>
     <span style="color:#0000ff;">
      ）、数据库管理员（
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       负责数据库的总体信息控制
      </span>
     </span>
     <span style="color:#0000ff;">
      ）、最终用户（
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       利用系统的接口或查询语言访问数据库
      </span>
     </span>
     <span style="color:#0000ff;">
      ）等。
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="192" src="https://i-blog.csdnimg.cn/blog_migrate/d7a19312983bb0c8d262c34dcd864041.png" width="282">
      <img alt="" height="353" src="https://i-blog.csdnimg.cn/blog_migrate/7d6abc6f48cb4a76e87c3be0ac2076b6.png" width="337"/>
     </img>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1.1.3数据库技术发展
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1.人工阶段（20世纪50中）
    </p>
    <ol>
     <li>
      <blockquote>
       <ol>
        <li style="text-align:justify;">
         数据不保存
        </li>
        <li style="text-align:justify;">
         无专门的数据管理软件
        </li>
        <li style="text-align:justify;">
         数据不共享
        </li>
        <li style="text-align:justify;">
         数据不具独立性
        </li>
       </ol>
      </blockquote>
     </li>
     <li style="text-align:justify;">
      文件系统阶段（50--60）新特点：数据以
      <span style="background-color:#ffff00;">
       文件
      </span>
      形式长期储存。
      <blockquote>
       <ol>
        <li style="text-align:justify;">
         数据共享性差、冗余度较大
        </li>
        <li style="text-align:justify;">
         数据不一致性
        </li>
        <li style="text-align:justify;">
         数据独立性差
        </li>
        <li style="text-align:justify;">
         数据间联系弱
        </li>
       </ol>
      </blockquote>
     </li>
     <li style="text-align:justify;">
      数据库系统阶段
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     用户与应用程序不需建立数据文件，从数据库存取其中的数据子集，该子集通过DBMS从数据库中经过
     <span style="background-color:#ffff00;">
      映射
     </span>
     形成的逻辑文件
    </p>
    <ol>
     <li>
      <ol>
       <li style="text-align:justify;">
        数据
        <span style="background-color:#ffff00;">
         结构化
        </span>
       </li>
       <li style="text-align:justify;">
        数据
        <span style="background-color:#ffff00;">
         共享
        </span>
        性高，冗余度小
       </li>
       <li style="text-align:justify;">
        数据
        <span style="background-color:#ffff00;">
         独立性
        </span>
        高
        <ul>
         <li style="text-align:justify;">
          数据的物理独立性。数据库物理结构发生变化，通过修改
          <span style="background-color:#ffff00;">
           映射
          </span>
          使逻辑结构不受影响，进而用户逻辑结构及应用程序不必改变。
         </li>
         <li style="text-align:justify;">
          数据的逻辑独立。。。。使用户应用程序不变。
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        有统一的数据管理与
        <span style="background-color:#ffff00;">
         控制
        </span>
        功能
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1.2.1数据模型：
     <span style="color:#000000;">
      对现实世界数据特征的抽象描述。现实世界数据--&gt;抽象数据库，需经过
     </span>
     <span style="color:#2f5496;">
      现实世界、信息世界、计算机世界
     </span>
     <span style="color:#000000;">
      3三个阶段。
     </span>
     <strong>
      <u>
       <span style="color:#000000;">
        <strong>
         <u>
          数据库系统的核心和基础。
         </u>
        </strong>
       </span>
      </u>
     </strong>
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       数据模型组成要素
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         数据结构。
        </span>
        <span style="color:#0070c0;">
         实体与实体
        </span>
        <span style="color:#000000;">
         间联系的表达和实现；
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         数据操作。允许对数据库中数据执行操作的集合，包含相应的操作规则；
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         数据完整约束。给出数据及联系应具有的制约和依赖规则；
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#ff0000;">
       数据模型的类型
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#ff0000;">
         概念模型（信息模型）。
        </span>
        <span style="color:#000000;">
         信息世界中的模型。
        </span>
        <span style="color:#2e75b5;">
         第一级抽象
        </span>
        <span style="color:#000000;">
         ，E-R图。不涉及信息在计算机中的表示和处理方法
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#ff0000;">
         逻辑模型。
        </span>
        <span style="color:#000000;">
         计算机世界中的模型，
        </span>
        <span style="color:#2e75b5;">
         第二级抽象
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#ff0000;">
         物理模型。
        </span>
        <span style="color:#2e75b5;">
         对数据最底层的模型
        </span>
        <span style="color:#000000;">
         。
        </span>
        <span style="color:#2e75b5;">
         计算机原理。
        </span>
        <span style="color:#000000;">
         描述数据在磁盘/磁带上的存储方式和存取方法。
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#000000;">
      1.2.2概念模型
     </span>
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       相关术语
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         实体entity。客观存在and相互区别的事物。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         属性attribute。实体具有某一特性。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         实体型 entity type。实体名+属性集合。学生（学号，姓名，年龄）
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         实体集 entity set。同型集体的集合。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         码 key。键/关键字/关键码。在实体型中能唯一标识一个实体属性/属性集
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         域 domain。某属性取值范围。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         联系 relationship。学号与姓名间。
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       概念模型中实体型之间的联系
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         1:1
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         1：N
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         M:N
        </span>
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       概念模型E-R方法
      </span>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#000000;">
      1.2.3逻辑模型
     </span>
    </p>
    <ol>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       层次模型（用树形结构来表示）自上而下的关系
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         特点：一个模型只有一个节点/没有双亲结点/根节点
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         其他结点只有一个根节点
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         父子结点，1：N 关系
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         优点：数据结构简单清晰、提供良好完整性支持，结构间联系简单，查询效率高。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         缺点：不能直接表示两个以上实体型间的复杂联系，/M:N关系
        </span>
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       网状模型
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         允许一个以上结点多个根节点
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         允许一个节点没有根节点
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         允许两个结点间多种关系
        </span>
        <ul>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           缺点：数据结构复杂。
          </span>
         </li>
        </ul>
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       关系模型
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         数据结构是一张
        </span>
        <span style="color:#ff0000;">
         规范化的二维表
        </span>
        <span style="color:#000000;">
         。每个二维表都可称为关系。
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         有更严格的数学理论依据，结构简单易懂，高独立性，安全保密性。
        </span>
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      <span style="color:#000000;">
       面向对象模型
      </span>
      <ol>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         抽象、封装、继承、多态性
        </span>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         更完整地描述现实世界的数据结构，具有丰富的表达能力。
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1.3数据系统的结构
    </p>
    <ol>
     <li style="text-align:justify;">
      数据库系统的三级模式结构
      <ol>
       <li style="text-align:justify;">
        <strong>
         <span style="color:#0000ff;">
          <strong>
           外部体系
          </strong>
         </span>
        </strong>
        <span style="color:#0000ff;">
         结构：从用户使用数据库的角度划分：单用户结构、主从式结构、分布式结构、客户/服务器、浏览器/应用服务器等
        </span>
       </li>
       <li style="text-align:justify;">
        <strong>
         <span style="color:#0000ff;">
          <strong>
           内部体系
          </strong>
         </span>
        </strong>
        <span style="color:#0000ff;">
         结构：从逻辑上分为：外模式、模式、内模式 三级抽象模式结构，并提供外模式/模式、模式/内模式二级映像。
        </span>
        <ul>
         <li style="text-align:justify;">
          <span style="color:#0000ff;">
           对用户言：
          </span>
          <u>
           <span style="color:#0000ff;">
            <u>
             外模式==用户模式，模式==概念模式，内模式==物理模式
            </u>
           </span>
          </u>
         </li>
         <li style="text-align:justify;">
          <u>
           <span style="color:#0000ff;">
            <u>
             模式：
            </u>
           </span>
          </u>
          <u>
           <span style="background-color:#ffff00;">
            <span style="color:#0000ff;">
             <u>
              也称概念模式/概念视图
             </u>
            </span>
           </span>
          </u>
          <u>
           <span style="color:#0000ff;">
            <u>
             。是数据库中全体数据的逻辑结构和特征的描述，处于三级模式的中间层，不涉及数据的物理存储和硬件环境，与开发工具及程序设计语言
            </u>
           </span>
          </u>
          <u>
           <span style="background-color:#ffff00;">
            <span style="color:#0000ff;">
             <u>
              无关
             </u>
            </span>
           </span>
          </u>
          <u>
           <span style="color:#0000ff;">
            <u>
             。
            </u>
           </span>
          </u>
          <ol>
           <li style="text-align:justify;">
            <u>
             <span style="background-color:#ffff00;">
              <span style="color:#0000ff;">
               <u>
                一个数据库只有一个模式
               </u>
              </span>
             </span>
            </u>
            <u>
             <span style="color:#0000ff;">
              <u>
               ，模式是数据的逻辑表示，它描述了数据库中存储的具体数据及其之间存在的联系。
              </u>
             </span>
            </u>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#0000ff;">
           外模式：又称子模式/用户模式。是三成最外层，是数据用户能够看到并允许使用的那部分数据的逻辑结构和特征的描述，是数据库用户的数据视图。
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#0000ff;">
             外模式是模式的子集。
            </span>
            <span style="background-color:#ffff00;">
             <span style="color:#0000ff;">
              一个数据可以有多个外模式
             </span>
            </span>
            <span style="color:#0000ff;">
             。
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#0000ff;">
           内模式：又称
          </span>
          <span style="background-color:#ffff00;">
           <span style="color:#0000ff;">
            存储模式/物理模式
           </span>
          </span>
          <span style="color:#0000ff;">
           。
          </span>
         </li>
        </ul>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      注意：一个数据库系统中，实际存在的只有物理级数据库（内模式），是访问数据的基础。
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      数据库系统的二级映像
      <ol>
       <li style="text-align:justify;">
        外模式/模式映像
        <ul>
         <li style="text-align:justify;">
          同一个模式可以有多个外模式
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        模式/内模式映像
        <ul>
         <li style="text-align:justify;">
          只有一个，唯一。
         </li>
         <li style="text-align:justify;">
          定义了全局的逻辑结构和存储结构
         </li>
        </ul>
       </li>
      </ol>
     </li>
    </ol>

     <strong>
      2.1关系模型
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     2.1.1关系模型的基本概念
    </p>
    <ol>
     <li style="text-align:justify;">
      只包含单一的数据结构--关系
     </li>
    </ol>
    <ol>
     <li style="text-align:justify;">
      关系
      <ol>
       <li style="text-align:justify;">
        Relation，由
        <span style="background-color:#ffff00;">
         行和列组成的二维表
        </span>
        。
       </li>
       <li style="text-align:justify;">
        属性
        <ul>
         <li style="text-align:justify;">
          二维表中的列称为属性attribute。每个 属性对应表中的一个字段，属性名==字段名；属性值==字段值。
         </li>
         <li style="text-align:justify;">
          二维表中列的个数称关系的
          <span style="background-color:#ffff00;">
           元数
          </span>
          。
          <span style="background-color:#ffff00;">
           N列N元关系
          </span>
          。
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        域domain
        <ul>
         <li style="text-align:justify;">
          一组有相同数据类型的集合。（值域）包含值的个数称为域的基数。（m表示）D={男，女}，M=2
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        元组
        <ul>
         <li style="text-align:justify;">
          Tuple。二维表中的行。即
          <span style="background-color:#ffff00;">
           记录值
          </span>
          。
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        分量。Component。一个元祖中每一个属性值。N元关系的每个元祖就有N个分量。
       </li>
       <li style="text-align:justify;">
        关系模式。
        <span style="background-color:#ffff00;">
         二维表的结构
        </span>
        。//二维表的框架（表头结构）
       </li>
       <li style="text-align:justify;">
        关系数据库。
        <span style="background-color:#ffff00;">
         关系模式是型。关系式它的值
        </span>
        。
       </li>
       <li style="text-align:justify;">
        笛卡尔积：关系是笛卡尔积的任意子集，有意义的子集，没有意义的不能称为关系。
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      关系的性质
      <ol>
       <li style="text-align:justify;">
        关系可以看成二维表，但不是所有的二维表都有关系。
       </li>
       <li style="text-align:justify;">
        列是同质的，每一列的分量都必须来自同一个域且必须同一个类型的数据。
       </li>
       <li style="text-align:justify;">
        不同属性可以来自同一个域，但不同属性有不同的名字
       </li>
       <li style="text-align:justify;">
        列的顺序可以变化
       </li>
       <li style="text-align:justify;">
        元组的顺序可以变化，但不能出现相同元组
       </li>
       <li style="text-align:justify;">
        关系中的分量不可再分
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      E-R图与关系模式的转换（
      <span style="color:#0000ff;">
       是概念模型--&gt;逻辑模型
      </span>
      ）
      <ol>
       <li style="text-align:justify;">
        转换过程：
        <span style="color:#0000ff;">
         实体与实体之间
        </span>
        的转换。实体名==关系名，实体属性==关系属性。
       </li>
       <li style="text-align:justify;">
        三种转换：1:1联系。
       </li>
       <li style="text-align:justify;">
        1：N联系：（
        <span style="color:#0000ff;">
         在多的一段加入“一”那那端实体
        </span>
        ）
       </li>
       <li style="text-align:justify;">
        N:N联系：为两端的码加上联系自身的属性。
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     2.2关系模型的完整性约束
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     关系模型中三类完整性约束：
    </p>
    <ol>
     <li style="text-align:justify;">
      实体完整性
      <ol>
       <li style="text-align:justify;">
        每个关系必须有主码。
        <span style="background-color:#ffff00;">
         Null：“不存在”“不知道”
        </span>
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      必须的完整约束性条件，称为关系的两个不变性
     </span>
    </p>
    <ol>
     <li style="text-align:justify;">
      参照完整性
     </li>
     <li style="text-align:justify;">
      用户定义完整性
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      2.2.1 关系的码
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     1. 候选码。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      某属性/集的值可以唯一确定一个元组。具有
     </span>
     <span style="background-color:#ffff00;">
      <span style="color:#0000ff;">
       唯一性、最小性
      </span>
     </span>
    </p>
    <ol>
     <li style="text-align:justify;">
      主码
      <ol>
       <li style="text-align:justify;">
        如果关系中有多个候选码，可选任意一个作为元组的标识。
       </li>
       <li style="text-align:justify;">
        <span style="color:#0000ff;">
         每个关系必须选择一个主码
        </span>
        ，选定后不能随意改变。
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      主属性与非主属性
      <ol>
       <li style="text-align:justify;">
        主属性：包含在任一候选码中的属性。
       </li>
       <li style="text-align:justify;">
        全码：关系的候选码是所有属性的集合。
       </li>
      </ol>
     </li>
     <li style="text-align:justify;">
      外码
      <ol>
       <li style="text-align:justify;">
        外码：/外键/：R1中的
        <span style="background-color:#ffff00;">
         主码
        </span>
        还在R2中。
        <span style="background-color:#ffff00;">
         R2的外码
        </span>
        。
       </li>
      </ol>
     </li>
    </ol>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="140" src="https://i-blog.csdnimg.cn/blog_migrate/b613baa8b04c7e84987a9974a71c89e9.png" width="381"/>
    </p>
    <p>
    </p>
    <ul>
     <li style="text-align:justify;">
      关系代数
      <ol>
       <li style="text-align:justify;">
        关系操作采用集合操作方式。即操作的对象和结果都是集合。
       </li>
       <li style="text-align:justify;">
        关系代数是
        <span style="background-color:#ffff00;">
         关系模型
        </span>
        最重要的数据
        <span style="background-color:#ffff00;">
         操作
        </span>
        语言。对关系的代数运算表达查询，是抽象的查询语句。
       </li>
       <li style="text-align:justify;">
        关系代数中的运算符：
        <ol>
         <li style="text-align:justify;">
          集合运算符   （交并差、
          <span style="background-color:#ffff00;">
           笛卡尔积
          </span>
          ）
          <span style="color:#0000ff;">
           二元运算符、参加运算的关系必须符合相容关系。（都是N元关系）
          </span>
         </li>
         <li style="text-align:justify;">
          专门的关系运算符  （
          <span style="background-color:#ffff00;">
           选择（单目运算）、投影（单目）、连接（二目运算）、除（二目）
          </span>
          ）
          <ul>
           <li style="text-align:justify;">
            <u>
             <u>
              等值连接
             </u>
            </u>
            <ol>
             <li style="text-align:justify;">
              相等的属性可以是相同属性，也可是不同属性
             </li>
            </ol>
           </li>
           <li style="text-align:justify;">
            <u>
             <u>
              自然连接
             </u>
            </u>
            <ol>
             <li style="text-align:justify;">
              属性必须相同，只能用于两个关系中有公共属性的情况。
             </li>
             <li style="text-align:justify;">
              <span style="background-color:#ffff00;">
               把重复的列去掉
              </span>
              。
             </li>
             <li style="text-align:justify;">
              自然连接是去掉重复列的等值连接。
             </li>
            </ol>
           </li>
          </ul>
         </li>
         <li style="text-align:justify;">
          逻辑运算符 （与或非）
         </li>
         <li style="text-align:justify;">
          比较运算符
         </li>
        </ol>
       </li>
      </ol>
     </li>
    </ul>
    <p style="margin-left:.0001pt;text-align:justify;">
     传统的集合运算：
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     ①并union：（U）实现记录的添加/插入
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     ②差 difference：（-）R-S,保留R，删S。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     ③交intersection （∩）R-S=R-(R-S)
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     ④广义笛卡尔积，N元关系与M元关系得到M+N元关系（
     <span style="background-color:#ffff00;">
      列
     </span>
     ）。M*N个元组（
     <span style="background-color:#ffff00;">
      行
     </span>
     ）。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ul>
     <li style="text-align:justify;">
      认识SQL
      <ul>
       <li style="text-align:justify;">
        SQL的发展
        <ul>
         <li style="text-align:justify;">
          SQL是
          <span style="background-color:#ffff00;">
           最成功、影响最广泛
          </span>
          的
          <span style="background-color:#ffff00;">
           关系数据库语言
          </span>
          。
         </li>
         <li style="text-align:justify;">
          逐渐成为了集数据查询语言，数据操纵、数据定义、数据控制的关系数据库标准语言。
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        SQL特点
        <ul>
         <li style="text-align:justify;">
          <u>
           <u>
            综合能力强
           </u>
          </u>
         </li>
         <li style="text-align:justify;">
          <u>
           <u>
            非过程化
           </u>
          </u>
          ：实现过程与存取路径的选择都由系统自动完成。
         </li>
         <li style="text-align:justify;">
          <u>
           <u>
            面向集合
           </u>
          </u>
          ：操作对象和结果都是元组的集合
         </li>
         <li style="text-align:justify;">
          <u>
           <u>
            使用方式灵活
           </u>
          </u>
          ：即可做独立语言，又可嵌入式语言。前者独立联机交互，在窗口输入命令即可。后者能嵌入多种高级程序语言中调用。
         </li>
         <li style="text-align:justify;">
          简洁易用。
         </li>
        </ul>
       </li>
      </ul>
     </li>

           <li style="text-align:justify;">
            数据库用来存放数据库数据+数据库对象的文件。一个数据库可有多个数据文件，一个数据文件只属于一个数据库。
           </li>
           <li style="text-align:justify;">
            <span style="color:#0000ff;">
             主数据文件：存储启动信息+部分/全部数据 后缀：.mdf
            </span>
           </li>
           <li style="text-align:justify;">
            其他数据文件（
            <span style="color:#0000ff;">
             次数据文件
            </span>
            ）：储存主数据文件为储存的其他数据，
            <span style="color:#0000ff;">
             .ndf
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          日志文件
          <ol>
           <li style="text-align:justify;">
            由一系列日志记录组成，记录
            <span style="color:#0000ff;">
             数据库更新情况+用户对数据库的修改操作
            </span>
            。
           </li>
           <li style="text-align:justify;">
            当文件损坏，可以
            <span style="color:#0000ff;">
             分析出错的原因
            </span>
            ；
           </li>
           <li style="text-align:justify;">
            当数据丢失，可用日志文件
            <span style="color:#0000ff;">
             恢复数据库
            </span>
            。
           </li>
           <li style="text-align:justify;">
            每个数据库至少有一个日志文件
            <span style="color:#0000ff;">
             .ldf
            </span>
           </li>
          </ol>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

        </span>
        <ul>
         <li style="text-align:justify;">
          <span style="color:#000000;">

          </span>

          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           修改数据
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Update 《表名》set 列名=‘’ where
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           删除数据
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Delete from 表名 where +条件
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           单表查询
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="background-color:#ffff00;">
             <span style="color:#000000;">
              Select （all/distinct+列名 as+新名）from+表 where+条件 group by+列名  having+条件 order by +列名 ASC/DESC
             </span>
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Distinct：表示去掉重复值
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           无条件查询与条件查询
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             无条件：select from ==相当于只对关系做投影动作。投影会删除重复行，所以。无条件时，要加上distinct
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             条件查询
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               字符串中的通配符
              </span>
              <ul>
               <li style="text-align:justify;">
                <span style="color:#000000;">
                 % 表任意多个字符。
                </span>
               </li>
               <li style="text-align:justify;">
                <span style="color:#000000;">
                 - 表一个字符
                </span>
               </li>
               <li style="text-align:justify;">
                <span style="color:#000000;">
                 【】【A G】% A到G为首的字符串
                </span>
               </li>
               <li style="text-align:justify;">
                <span style="color:#000000;">
                 【^】相反
                </span>
               </li>
              </ul>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               空值查询
              </span>
              <ul>
               <li style="text-align:justify;">
                <span style="color:#000000;">
                 Is not null
                </span>
               </li>
              </ul>
             </li>
            </ol>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           聚合函数

          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Count（列的值）
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Count（*）元组的值
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             注意：count（*）无法使用distinct。如果不设置列别名，结果是无列名。
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           分组查询
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Group by
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           连接查询
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             内连接查询
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="background-color:#ffff00;">
               <span style="color:#000000;">
                Inner join +表 on+条件

             外连接查询

             <li style="text-align:justify;">

               左外连接

             <li style="text-align:justify;">
              <span style="color:#000000;">
               右外连接

              <span style="color:#000000;">
               全外连接 full join

     <li style="text-align:justify;">
      <span style="color:#000000;">
       视图
      </span>
      <ul>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         视图
        </span>
        <ul>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           视图是一个数据库对象，是从一个/多个表中导出来的虚拟表。是一条select语句对应的结果集，结果集的名字就是视图。
          </span>
         </li>
         <li style="text-align:justify;">
          <span style="color:#132ef5;">
           视图本身不包含任何数据，数据库存储的就是它的定义
          </span>
          <span style="color:#000000;">
           。
          </span>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           优点与作用
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             只可以提取用户需要的数据，便于管理与使用。
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             提供对数据安全保护的机制
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             提供了一定的
            </span>
            <span style="color:#c00000;">
             逻辑独立性
            </span>
            <span style="color:#000000;">
             。
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             视图对应了三级模式中的
            </span>
            <span style="background-color:#ffff00;">
             <span style="color:#000000;">
              外模式
             </span>
            </span>
            <span style="color:#000000;">
             ，逻辑结构变化，需要修改视图的定义即可外模式不变。
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           创建视图
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Create view +名+（列名）as select 语句
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             As指定视图要执行的操作
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Select 御酒可以是任意查询语句，但不包含order by 和 distinct
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           删除视图
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Drop view + 名
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           修改视图
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Alter view + 名+列名 as select语句
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Insert into +名 +values
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Update + 名 set 。。。where。。。
            </span>
           </li>
          </ol>
         </li>
        </ul>
       </li>
       <li style="text-align:justify;">
        <span style="color:#000000;">
         索引
        </span>
        <ul>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           索引是一个单独的，储存在磁盘上的数据结构目录，包含所有记录的引用指针
          </span>
          <span style="color:#c00000;">
           。就是一个目录
          </span>
          <span style="color:#000000;">
           。
          </span>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           特点
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             查询速度快
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             唯一性
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             建立在外码上的索引可加速表间连接，得益于实现数据的参照完整性
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             减少分组和排序的查询所使用的时间。
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             缺点
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               索引会占用物理储存空间，要额外物理储存文件
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               维护需要时间
              </span>
             </li>
            </ol>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           索引的类型（
          </span>
          <span style="color:#c00000;">
           提升查询速度，降低更新速度
          </span>
          <span style="color:#000000;">
           ）
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             聚集索引
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               确定数据储存的顺序，表内数据按照聚集索引键值的顺序储存。
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#c00000;">
               物理储存只有一个，所以一个数据表只能包含一个聚集索引
              </span>
              <span style="color:#000000;">
               。
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               对主键约束自动创建聚集索引
              </span>
             </li>
            </ol>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             非聚集索引
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               字典的部首目录
              </span>
             </li>
            </ol>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             其他索引
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               唯一索引：聚集/非聚集索引
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               视图索引：视图上建立的索引
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               全文索引：快速查询某个字符出现的位置
              </span>
             </li>
            </ol>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             设计索引注意
            </span>
            <ol>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               索引并非越多越好，占磁盘空间和大量维护开销
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               经常查询的列可建立索引，避免不必要的数据
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               数据量小的不用
              </span>
             </li>
             <li style="text-align:justify;">
              <span style="color:#000000;">
               不同值较少的列不用：男/女
              </span>
             </li>
            </ol>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           创建与删除索引
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Create unique index on
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           删除索引
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Drop index +名 on +数据表/视图
            </span>
           </li>
          </ol>
         </li>
         <li style="text-align:justify;">
          <span style="color:#000000;">
           修改和查看
          </span>
          <ol>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             Alter index+名 on
            </span>
           </li>
           <li style="text-align:justify;">
            <span style="color:#000000;">
             修改索引名称：EXEC sp_rename &lt;表名/视图名.索引名&gt;,&lt;新索引名&gt;
            </span>
           </li>
          </ol>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      第一章习题
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <ol>
     <li style="text-align:justify;">
      数据共享的含义：
      <span style="color:#c00000;">
       多种应用、多种语言、多个用户覆盖地使用数据集合
      </span>
     </li>
     <li style="text-align:justify;">
      数据库的概念模型独立于：
      <span style="color:#c00000;">
       具体的机器和DBMS
      </span>
     </li>
     <li style="text-align:justify;">
      三级模式中：
      <span style="color:#c00000;">
       外模式：是用户与数据库接口，是应用程序可见的数据描述
      </span>
     </li>
     <li style="text-align:justify;">
      <span style="color:#c00000;">
       一个关系只有一个主码
      </span>
     </li>
     <li style="text-align:justify;">
      应用数据库的目的是为了解决：
      <span style="color:#c00000;">
       共享数据
      </span>
     </li>
     <li style="text-align:justify;">
      数据特点：
      <span style="color:#c00000;">
       数据结构化、最小的冗余度和较高的独立性
      </span>
     </li>
     <li style="text-align:justify;">
      五种基本运算：
      <span style="color:#c00000;">
       并、差、选择、投影、乘积
      </span>
     </li>
     <li style="text-align:justify;">
      共同属性值的元祖连在一起：
      <span style="color:#c00000;">
       连接
      </span>
     </li>
     <li style="text-align:justify;">
      三级模式结构中，描述数据库中全体数据的全逻辑结构和特征：
      <span style="color:#c00000;">
       模式
      </span>
     </li>
     <li style="text-align:justify;">
      <span style="color:#c00000;">
       数据的物理独立性
      </span>
      是指：
      <span style="color:#c00000;">
       用户的数据和数据在磁盘中的存储方式
      </span>
      相互独立
     </li>
     <li style="text-align:justify;">
      数据库的
      <span style="color:#c00000;">
       数据项之间和记录之间
      </span>
      都存在联系
     </li>
     <li style="text-align:justify;">
      计算机中支持数据库各种操作的软件系统叫：
      <span style="color:#c00000;">
       数据库管理系统
      </span>
     </li>
     <li style="text-align:justify;">
      数据库管理系统DBMS的工作包括：
      <span style="color:#c00000;">
       为定义的数据库提供操作平台、对已定义的数据库进行管理、定义数据库
      </span>
     </li>
     <li style="text-align:justify;">
      数据管理的任务：
      <span style="color:#c00000;">
       数据存储、数据编码
      </span>
     </li>
     <li style="text-align:justify;">
      数据模型三要素：
      <span style="color:#c00000;">
       完整性约束、数据结构、数据操作
      </span>
     </li>
     <li style="text-align:justify;">
      属于现实世界中事物或个体的特征的不同抽象与表述：
      <span style="color:#c00000;">
       属性、列、字段
      </span>
     </li>
     <li style="text-align:justify;">
      关系操作的特点：
      <span style="color:#c00000;">
       所有操作都是集合操作
      </span>
     </li>
     <li style="text-align:justify;">
      关系数据库中实体完整性的实现是靠：
      <span style="color:#c00000;">
       主键、主关系键、主码
      </span>
      保证
     </li>
     <li style="text-align:justify;">
      数据库的三级模式通过两级映像来建立联系，同时保证了
      <span style="color:#c00000;">
       数据独立性
      </span>
      ，外模式/模式保证了：
      <span style="color:#c00000;">
       数据的逻辑独立
      </span>
     </li>
     <li style="text-align:justify;">
      关系数据库中参照完整性规则的实现靠 ：
      <span style="color:#c00000;">
       外键、外码、外关系键
      </span>
      来保证
     </li>
     <li style="text-align:justify;">
     </li>
    </ol>

    <ol>
     <li style="text-align:justify;">
      SQL sever 数据库文件组的用途：
      <span style="color:#c00000;">

     </li>
     <li style="text-align:justify;">
      <span style="color:#c00000;">
       Model
      </span>
      数据库：
      <span style="color:#c00000;">
       自动创建表对象
      </span>
     </li>
     <li style="text-align:justify;">
      <span style="color:#0000ff;">
       除法：exists
      </span>
     </li>
     <li style="text-align:justify;">
      SQLserver数据库文件的逻辑名称作用是：方便SQLserver管理数据文件
     </li>
     <li style="text-align:justify;">
      <span style="color:#0000ff;">
       去掉重复：distinct
      </span>
     </li>
     <li style="text-align:justify;">
      Alter table ： 修改
      <span style="color:#c00000;">
       字段长度
      </span>
     </li>
     <li style="text-align:justify;">
      Create database中，用来指定文件逻辑名称的是：
      <span style="color:#c00000;">
       name
      </span>
     </li>
     <li style="text-align:justify;">
      数据表数据在数据库的所有数据文件是
      <span style="color:#c00000;">
       随机存放
      </span>
      的
     </li>
     <li style="text-align:justify;">
      <span style="color:#c00000;">
       Master
      </span>
      ：保存SQLserver
      <span style="color:#c00000;">
       系统配置信息
      </span>
     </li>
     <li style="text-align:justify;">
      SQLserver中，数据库由文件组成，不包括：
      <span style="color:#c00000;">
       备份文件
      </span>
      。
     </li>
     <li style="text-align:justify;">
      Varchar类型的实际存储空间与
      <span style="color:#c00000;">
       实际存放的字符
      </span>
      有关，与定义无关
     </li>
     <li style="text-align:justify;">
      SQL server 数据库
      <span style="color:#0000ff;">
       次要数据文件的名称等
      </span>

      <span style="color:#c00000;">
       主数据文件
      </span>
     </li>

      <span style="color:#0000ff;">
       聚集索引
      </span>

      <span style="color:#c00000;">
       数据文件
      </span>
      和
      <span style="color:#c00000;">
       日志文件
      </span>
      组成的
     </li>

      第三章习题
     </strong>
    </p>
    <ol>
     <li style="text-align:justify;">
      候选关键字的属性可以
      <span style="color:#c00000;">
       一个或多个
      </span>
     </li>
     <li style="text-align:justify;">
      AB1对1：
      <span style="color:#c00000;">
       A &lt;----&gt;B
      </span>
     </li>
     <li style="text-align:justify;">
      概念模型的优点：
      <span style="color:#c00000;">
       独立于操作系统和DBMA
      </span>
     </li>
     <li style="text-align:justify;">
      关系规范化中的删除操作异常是指：
      <span style="color:#c00000;">
       不该删除的数据被删除
      </span>
     </li>
     <li style="text-align:justify;">
      ER图中的主要元素是：
      <span style="color:#c00000;">
       实体型、联系、属性
      </span>
     </li>
     <li style="text-align:justify;">
      最常用的数据模型是：
      <span style="color:#c00000;">
       实体联系模型
      </span>
     </li>
     <li style="text-align:justify;">
      关系规范化中插入操作异常是指：
      <span style="color:#c00000;">
       应该插入的数据未被插入
      </span>
     </li>
     <li style="text-align:justify;">
      局部ER图合并成全局图时会出现合并冲突：
      <span style="color:#c00000;">
       结构冲突、命名冲突、属性冲突
      </span>
     </li>
     <li style="text-align:justify;">
      关于数据库逻辑设计的任务，
      <span style="color:#c00000;">
       创建数据库逻辑结构说明
      </span>
     </li>
     <li style="text-align:justify;">
      3NF：
      <span style="color:#c00000;">
       一定消除了插入和删除异常
      </span>
     </li>
     <li style="text-align:justify;">
      关系模式R的属性全部属性是主属性，则R的最高范式必定是：
      <span style="color:#c00000;">
       3NF
      </span>
     </li>
     <li style="text-align:justify;">
      关系模式中的关系模式至少是
      <span style="color:#c00000;">
       1NF
      </span>
     </li>
     <li style="text-align:justify;">
      关系模式R中函数依赖X--&gt;Y的语义是：
      <span style="color:#c00000;">
       在R的每一关系中，若两个元组的X值相等，则Y也相等
      </span>
      。
     </li>
     <li style="text-align:justify;">
      如果两个实体之间的联系是M/N，进一步处理正确的是：
      <span style="color:#c00000;">
       需要引入第三个交叉关系
      </span>
     </li>
     <li style="text-align:justify;">
      数据流程图DFD是用于描述规范化数据库设计方法中的
      <span style="color:#ff0000;">
       需求分析
      </span>
      阶段的工具
     </li>
     <li style="text-align:justify;">
      ER图合并冲突后，依靠各设计者之间
      <span style="color:#ff0000;">
       讨论协商
      </span>
      解决
     </li>
     <li style="text-align:justify;">
      “为哪些表，在哪些字段上建立什么样的索引”数据库
      <span style="color:#c00000;">
       物理结构
      </span>
      设计阶段
     </li>
     <li style="text-align:justify;">
      数据库实施阶段包括两项重要的工作，一项是
      <span style="color:#c00000;">
       数据的载入
      </span>
      、一项是
      <span style="color:#c00000;">
       应用程序的编码和调试
      </span>
      。
     </li>
    </ol>
