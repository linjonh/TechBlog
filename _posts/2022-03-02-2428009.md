    <p align="center" class="MsoNormal" style="text-align: center;">
     <strong style="">
      <span style="font-size: 12pt; font-family: 宋体;">
       嵌入式操作系统介绍分析
       <span lang="EN-US">
       </span>
      </span>
     </strong>
    </p>
    <p class="MsoNormal">
     <strong style="">
      <span style="font-size: 12pt; font-family: 宋体;">
       一，各种嵌入式操作系统介绍
       <span lang="EN-US">
       </span>
      </span>
     </strong>
    </p>
    <p class="MsoNormal">
     <strong style="">
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
       1.uC/OS-II
      </span>
     </strong>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      u C / OS
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      是一种免费公开源代码、结构小巧、具有可剥夺实时内核的实时操作系统。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      μC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的前身是
      <span lang="EN-US">
       μC/OS
      </span>
      ，最早出自于
      <span lang="EN-US">
       1992
      </span>
      年美国嵌入式系统专家
      <span lang="EN-US">
       Jean J.Labrosse
      </span>
      在《嵌入式系统编程》杂志的
      <span lang="EN-US">
       5
      </span>
      月和
      <span lang="EN-US">
       6
      </span>
      月刊上刊登的文章连载，并把
      <span lang="EN-US">
       μC/OS
      </span>
      的源码发布在该杂志的
      <span lang="EN-US">
       BBS
      </span>
      上。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      μC/OS
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      和
      <span lang="EN-US">
       μC/OS-II
      </span>
      是专门为计算机的嵌入式应用设计的， 绝大部分代码是用
      <span lang="EN-US">
       C
      </span>
      语言编写的。
      <span lang="EN-US">
       CPU
      </span>
      硬件相关部分是用汇编语言编写的、总量约
      <span lang="EN-US">
       200
      </span>
      行的汇编语言部分被压缩到最低限度，为的是便于移植到任何一种其它的
      <span lang="EN-US">
       CPU
      </span>
      上。用户只要有标准的
      <span lang="EN-US">
       ANSI
      </span>
      的
      <span lang="EN-US">
       C
      </span>
      交叉编译器，有汇编器、连接器等软件工具，就可以将
      <span lang="EN-US">
       μC/OS-II
      </span>
      嵌人到开发的产品中。
      <span lang="EN-US">
       μC/OS-II
      </span>
      具有执行效率高、占用空间小、实时性能优良和可扩展性强等特点， 最小内核可编译至
      <span lang="EN-US">
       2KB
      </span>
      。
      <span lang="EN-US">
       μC/OS-II
      </span>
      已经移植到了几乎所有知名的
      <span lang="EN-US">
       CPU
      </span>
      上。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      严格地说
      <span lang="EN-US">
       uC/OS-II
      </span>
      只是一个实时操作系统内核，它仅仅包含了任务调度，任务管理，时间管理，内存管理和任务间的通信和同步等基本功能。 没有提供输入输出管理，文件系统，网络等额外的服务。但由于
      <span lang="EN-US">
       uC/OS-II
      </span>
      良好的可扩展性和源码开放，这些非必须的功能完全可以由用户自己根据需要分别实现。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      目标是实现一个基于优先级调度的抢占式的实时内核，并在这个内核之上提供最基本的系统服务，如信号量，邮箱，消息队列，内存管理，中断管理等。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      任务管理
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      中最多可以支持
      <span lang="EN-US">
       64
      </span>
      个任务，分别对应优先级
      <span lang="EN-US">
       0
      </span>
      ～
      <span lang="EN-US">
       63
      </span>
      ，其中
      <span lang="EN-US">
       0
      </span>
      为最高优先级。
      <span lang="EN-US">
       63
      </span>
      为最低级，系统保留了
      <span lang="EN-US">
       4
      </span>
      个最高优先级的任务和
      <span lang="EN-US">
       4
      </span>
      个最低优先级的任务，所有用户可以使用的任务数有
      <span lang="EN-US">
       56
      </span>
      个。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      提供了任务管理的各种函数调用，包括创建任务，删除任务，改变任务的优先级，任务挂起和恢复等。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      系统初始化时会自动产生两个任务：一个是空闲任务，它的优先级最低，该任务仅给一个整形变量做累加运算；另一个是系统任务，它的优先级为次低，该任务负责统计当前
      <span lang="EN-US">
       cpu
      </span>
      的利用率。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      时间管理
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的时间管理是通过定时中断来实现的，该定时中断一般为
      <span lang="EN-US">
       10
      </span>
      毫秒或
      <span lang="EN-US">
       100
      </span>
      毫秒发生一次，时间频率取决于用户对硬件系统的定时器编程来实现。中断发生的时间间隔是固定不变的，该中断也成为一个时钟节拍。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      要求用户在定时中断的服务程序中，调用系统提供的与时钟节拍相关的系统函数，例如中断级的任务切换函数，系统时间函数。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      内存管理
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      在
      <span lang="EN-US">
       ANSI C
      </span>
      中是使用
      <span lang="EN-US">
       malloc
      </span>
      和
      <span lang="EN-US">
       free
      </span>
      两个函数来动态分配和释放内存。但在嵌入式实时系统中，多次这样的错作会导致内存碎片，且由于内存管理算法的原因，
      <span lang="EN-US">
       malloc
      </span>
      和
      <span lang="EN-US">
       free
      </span>
      的执行时间也是不确定。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS -II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      中把连续的大快内存按分区管理。每个分区中包含整数个大小相同的内存块，但不同分区之间的内存快大小可以不同。用户需要动态分配内存时，系统选择一个适当的分区，按块来分配内存。释放内存时将该块放回它以前所属的分区，这样能有效解决碎片问题，同时执行时间也是固定的。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      任务间通信与同步
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      对一个多任务的操作系统来说，任务间的通信和同步是必不可少的。
      <span lang="EN-US">
       uC/OS-II
      </span>
      中提供了
      <span lang="EN-US">
       4
      </span>
      中同步对象，分别是信号量，邮箱，消息队列和事件。所有这些同步对象都有创建，等待，发送，查询的接口用于实现进程间的通信和同步。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      任务调度
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      采用的是可剥夺型实时多任务内核。可剥夺型的实时内核在任何时候都运行就绪了的最高优先级的任务。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uC/os-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的任务调度是完全基于任务优先级的抢占式调度，也就是最高优先级的任务一旦处于就绪状态，则立即抢占正在运行的低优先级任务的处理器资源。为了简化系统设计，
      <span lang="EN-US">
       uC/OS-II
      </span>
      规定所有任务的优先级不同，因为任务的优先级也同时唯一标志了该任务本身。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      任务调度将在以下情况下发生：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      1
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ） 高优先级的任务因为需要某种临界资源，主动请求挂起，让出处理器，此时将调度就绪状态的低优先级任务获得执行，这种调度也称为任务级的上下文切换。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      2
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ） 高优先级的任务因为时钟节拍到来，在时钟中断的处理程序中，内核发现高优先级任务获得了执行条件
      <span lang="EN-US">
       (
      </span>
      如休眠的时钟到时
      <span lang="EN-US">
       )
      </span>
      ，则在中断态直接切换到高优先级任务执行。这种调度也称为中断级的上下文切换。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      这两种调度方式在
      <span lang="EN-US">
       uC/OS-II
      </span>
      的执行过程中非常普遍，一般来说前者发生在系统服务中，后者发生在时钟中断的服务程序中。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      调度工作的内容可以分为两部分：最高优先级任务的寻找和任务切换。其最高优先级任务的寻找是通过建立就绪任务表来实现的。
      <span lang="EN-US">
       u C / O S
      </span>
      中的每一个任务都有独立的堆栈空间，并有一个称为任务控制块
      <span lang="EN-US">
       TCB(Task Control Block)
      </span>
      的数据结构，其中第一个成员变量就是保存的任务堆栈指针。任务调度模块首先用变量
      <span lang="EN-US">
       OSTCBHighRdy
      </span>
      记录当前最高级就绪任务的
      <span lang="EN-US">
       TCB
      </span>
      地址，然后调用
      <span lang="EN-US">
       OS_TASK_SW()
      </span>
      函数来进行任务切换。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      μC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的组成部分
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 24pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      μC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      可以大致分成核心、任务处理、时间处理、任务同步与通信，
      <span lang="EN-US">
       CPU
      </span>
      的移植等
      <span lang="EN-US">
       5
      </span>
      个部分。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       1)
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      核心部分
      <span lang="EN-US">
       (OSCore.c)
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      是操作系统的处理核心，包括操作系统初始化、操作系统运行、中断进出的前导、时钟节拍、任务调度、事件处理等多部分。能够维持系统基本工作的部分都在这里。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       2)
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      任务处理部分
      <span lang="EN-US">
       (OSTask.c)
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      任务处理部分中的内容都是与任务的操作密切相关的。包括任务的建立、删除、挂起、恢复等等。因为
      <span lang="EN-US">
       μC/OS-II
      </span>
      是以任务为基本单位调度的，所以这部分内容也相当重要。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       3)
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      时钟部分
      <span lang="EN-US">
       (OSTime.c)
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      μC/OS-II
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      中的最小时钟单位是
      <span lang="EN-US">
       timetick
      </span>
      （时钟节拍）。任务延时等操作是在这里完成的。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       4)
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      任务同步和通信部分
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      为事件处理部分，包括信号量、邮箱、邮箱队列、事件标志等部分；主要用于任务间的互相联系和对临界资源的访问。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">

      </span>

      <span lang="EN-US">

      <span lang="EN-US">

      </span>

      </span>

      </span>

      </span>

      </span>
      、

      </span>

      <span lang="EN-US">

      <span lang="EN-US">

    </p>
    <p class="MsoNormal">

      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">

      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      在
      <span lang="EN-US">

      、

      </span>

    <p class="MsoNormal">

    <p align="left" class="MsoNormal" style="text-align: left;">

       PowerPC,680x0/683xx,ColdFile,ARM7,ARM/Thumb ,MCORE, ARC, TriCore, X86,StrongARM,MIPS,SH,I960,V8xx,SPARC,Xscale, H8/300H, TMS

   .

    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong style="">
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
       4.
      </span>
     </strong>
     <strong>
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
       uClinux
      </span>
     </strong>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left; text-indent: 5.9pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Linux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      是一种很受欢迎的操作系统，它与
      <span lang="EN-US">
       UNIX
      </span>
      系统兼容，开放源代码。它原本被设计为桌面系统，现在广泛应用于服务器领域。而更大的影响在于它正逐渐地应用于嵌入式设备。
      <span lang="EN-US">
       uClinux
      </span>
      正是在这种氛围下产生的。在
      <span lang="EN-US">
       uClinux
      </span>
      这个英文单词中
      <span lang="EN-US">
       u
      </span>
      表示
      <span lang="EN-US">
       Micro
      </span>
      ，小的意思，
      <span lang="EN-US">
       C
      </span>
      表示
      <span lang="EN-US">
       Control
      </span>
      ，控制的意思，所以
      <span lang="EN-US">
       uClinux
      </span>
      就是
      <span lang="EN-US">
       Micro-Control-Linux
      </span>
      ，字面上的理解就是
      <span lang="EN-US">
       "
      </span>
      针对微控制领域而设计的
      <span lang="EN-US">
       Linux
      </span>
      系统
      <span lang="EN-US">
       "
      </span>
      。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left; text-indent: 5.9pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      4.1
      <span style="">
       uClinux
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的小型化方法
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      标准
      <span lang="EN-US">
       Linux
      </span>
      可能采用的小型化方法
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="margin-left: 18pt; text-align: left; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       1.
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      重新编译内核
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Linux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      内核采用模块化的设计，即很多功能块可以独立的加上或卸下，开发人员在设计内核时把这些内核模块作为可选的选项，可以在编译系统内核时指定。因 此一种较通用的做法是对
      <span lang="EN-US">
       Linux
      </span>
      内核重新编译，在编译时仔细的选择嵌入式设备所需要的功能支持模块，同时删除不需要的功能。通过对内核的重新配置，可以 使系统运行所需要的内核显著减小，从而缩减资源使用量。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      2.
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      制作
      <span lang="EN-US">
       root
      </span>
      文件系统映象
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Linux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      系统在启动时必须加载根（
      <span lang="EN-US">
       root
      </span>
      ）文件系统，因此剪裁系统同时包括
      <span lang="EN-US">
       root file system
      </span>
      的剪裁。在
      <span lang="EN-US">
       x86
      </span>
      系统下，
      <span lang="EN-US">
       Linux
      </span>
      可以在
      <span lang="EN-US">
       Dos
      </span>
      下，使用
      <span lang="EN-US">
       Loadlin
      </span>
      文件加载启动。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      采用的小型化方法
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="margin-left: 18pt; text-align: left; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       1．
      </span>
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的内核加载方式
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的内核有两种可选的运行方式：可以在
      <span lang="EN-US">
       flash
      </span>
      上直接运行，也可以加载到内存中运行。这种做法可以减少内存需要。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Flash
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      运行方式：把内核的可执行映象烧写到
      <span lang="EN-US">
       flash
      </span>
      上，系统启动时从
      <span lang="EN-US">
       flash
      </span>
      的某个地址开始逐句执行。这种方法实际上是很多嵌入式系统采用的方法。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      内核加载方式：把内核的压缩文件存放在
      <span lang="EN-US">
       flash
      </span>
      上，系统启动时读取压缩文件在内存里解压，然后开始执行，这种方式相对复杂一些，但是运行速度可能更快（
      <span lang="EN-US">
       ram
      </span>
      的存取速率要比
      <span lang="EN-US">
       flash
      </span>
      高）。同时这也是标准
      <span lang="EN-US">
       Linux
      </span>
      系统采用的启动方式。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="margin-left: 18pt; text-align: left; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       2．
      </span>
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的根（
      <span lang="EN-US">
       root
      </span>
      ）文件系统
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      系统采用
      <span lang="EN-US">
       romfs
      </span>
      文件系统，这种文件系统相对于一般的
      <span lang="EN-US">
       ext2
      </span>
      文件系统要求更少的空间。空间的节约来自于两个方面，首先内核支持
      <span lang="EN-US">
       romfs
      </span>
      文件系统比支持
      <span lang="EN-US">
       ext2
      </span>
      文件系统需要更少的代码，其次
      <span lang="EN-US">
       romfs
      </span>
      文件系统相对简单，在建立文件系统超级块（
      <span lang="EN-US">
       superblock
      </span>
      ）需要更少的存储空间。
      <span lang="EN-US">
       Romfs
      </span>
      文件系统不支持动态擦写保存，对于系统需要动态保存的数据采用虚拟
      <span lang="EN-US">
       ram
      </span>
      盘的方法进行处理（
      <span lang="EN-US">
       ram
      </span>
      盘将采用
      <span lang="EN-US">
       ext2
      </span>
      文件系统）。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="margin-left: 18pt; text-align: left; text-indent: -18pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
       3．
      </span>
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的应用程序库
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      小型化的另一个做法是重写了应用程序库，相对于越来越大且越来越全的
      <span lang="EN-US">
       glibc
      </span>
      库，
      <span lang="EN-US">
       uClibc
      </span>
      对
      <span lang="EN-US">
       libc
      </span>
      做了精简。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      对用户程序采用静态连接的形式，这种做法会使应用程序变大，但是基于内存管理的问题，不得不这样做（这将在下文对
      <span lang="EN-US">
       uClinux
      </span>
      内存管理展开分析时进行说明），同时这种做法也更接近于通常嵌入式系统的做法。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      4.2
      <span style="">
       uClinux
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的开发环境
     </span>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong>
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
       GNU
      </span>
     </strong>
     <strong>
      <span style="font-size: 12pt; font-family: 宋体;">
       开发套件
       <span lang="EN-US">
       </span>
      </span>
     </strong>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong style="">
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      </span>
     </strong>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Gnu
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      开发套件作为通用的
      <span lang="EN-US">
       Linux
      </span>
      开放套件，包括一系列的开发调试工具。主要组件：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Gcc
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ： 编译器，可以做成交叉编译的形式，即在宿主机上开发编译目标上可运行的二进制文件。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Binutils
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ：一些辅助工具，包括
      <span lang="EN-US">
       objdump
      </span>
      （可以反编译二进制文件），
      <span lang="EN-US">
       as
      </span>
      （汇编编译器），
      <span lang="EN-US">
       ld
      </span>
      （连接器）等等。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Gdb
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ：调试器，可使用多种交叉调试方式，
      <span lang="EN-US">
       gdb-bdm
      </span>
      （背景调试工具），
      <span lang="EN-US">
       gdbserver
      </span>
      （使用以太网络调试）。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      的打印终端
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      通常情况下，
      <span lang="EN-US">
       uClinux
      </span>
      的默认终端是串口，内核在启动时所有的信息都打印到串口终端（使用
      <span lang="EN-US">
       printk
      </span>
      函数打印），同时也可以通过串口终端与系统交互。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      在启动时启动了
      <span lang="EN-US">
       telnetd
      </span>
      （远程登录服务），操作者可以远程登录上系统，从而控制系统的运行。至于是否允许远程登录可以通过烧写
      <span lang="EN-US">
       romfs
      </span>
      文件系统时有用户决定是否启动远程登录服务。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong>
      <span style="font-size: 12pt; font-family: 宋体;">
       交叉编译调试工具
      </span>
     </strong>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      支持一种新的处理器，必须具备一些编译，汇编工具，使用这些工具可以形成可运行于这种处理器的二进制文件。对于内核使用的编译工具同应用程序使用的有所不同。在解释不同点之前，需要对
      <span lang="EN-US">
       gcc
      </span>
      连接做一些说明：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      .ld
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （
      <span lang="EN-US">
       link description
      </span>
      ）文件：
      <span lang="EN-US">
       ld
      </span>
      文件是指出连接时内存映象格式的文件。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      crt0.S
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ：应用程序编译连接时需要的启动文件，主要是初始化应用程序栈。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      pic
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ：
      <span lang="EN-US">
       position independence code
      </span>
      ，与位置无关的二进制格式文件，在程序段中必须包括
      <span lang="EN-US">
       reloc
      </span>
      段，从而使的代码加载时可以进行重新定位。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      内核编译连接时，使用
      <span lang="EN-US">
       ucsimm.ld
      </span>
      文件，形成可执行文件映象，所形成的代码段既可以使用间接寻址方式（即使用
      <span lang="EN-US">
       reloc
      </span>
      段进行寻址），也可以使用绝对寻址方式。这样可以给编译器更多的优化空间。因为内核可能使用绝对寻址，所以内核加载到的内存地址空间必须与
      <span lang="EN-US">
       ld
      </span>
      文件中给定的内存空间完全相同。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      应用程序的连接与内核连接方式不同。应用程序由内核加载（可执行文件加载器将在后面讨论），由于应用程序的
      <span lang="EN-US">
       ld
      </span>
      文件给出的内存空间与应用程序实际被加载的 内存位置可能不同，这样在应用程序加载的过程中需要一个重新地位的过程，即对
      <span lang="EN-US">
       reloc
      </span>
      段进行修正，使得程序进行间接寻址时不至于出错。（这个问题在
      <span lang="EN-US">
       i386
      </span>
      等高级处理器上方法有所不同，本文将在后面进一步分析）。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      由上述讨论，至少需要两套编译连接工具。在讨论过
      <span lang="EN-US">
       uClinux
      </span>
      的内存管理后本文将给出整个系统的工作流程以及系统在
      <span lang="EN-US">
       flash
      </span>
      和
      <span lang="EN-US">
       ram
      </span>
      中的空间分布。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong>
      <span style="font-size: 12pt; font-family: 宋体;">
       可执行文件格式
      </span>
     </strong>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      先对一些名词作一些说明：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      coff
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （
      <span lang="EN-US">
       common object file format
      </span>
      ）：一种通用的对象文件格式
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      elf
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （
      <span lang="EN-US">
       excutive linked file
      </span>
      ）：一种为
      <span lang="EN-US">
       Linux
      </span>
      系统所采用的通用文件格式，支持动态连接
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      flat
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ：
      <span lang="EN-US">
       elf
      </span>
      格式有很大的文件头，
      <span lang="EN-US">
       flat
      </span>
      文件对文件头和一些段信息做了简化
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      系统使用
      <span lang="EN-US">
       flat
      </span>
      可执行文件格式，
      <span lang="EN-US">
       gcc
      </span>
      的编译器不能直接形成这种文件格式，但是可以形成
      <span lang="EN-US">
       coff
      </span>
      或
      <span lang="EN-US">
       elf
      </span>
      格式的可执行文件，这两种文件需要
      <span lang="EN-US">
       coff2flt
      </span>
      或
      <span lang="EN-US">
       elf2flt
      </span>
      工具进行格式转化，形成
      <span lang="EN-US">
       flat
      </span>
      文件。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      当用户执行一个应用时，内核的执行文件加载器将对
      <span lang="EN-US">
       flat
      </span>
      文件进行进一步处理，主要是对
      <span lang="EN-US">
       reloc
      </span>
      段进行修正（可执行文件加载器的详见
      <span lang="EN-US">
       fs/binfmt_flat.c
      </span>
      ）。以下对
      <span lang="EN-US">
       reloc
      </span>
      段进一步讨论
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      需要
      <span lang="EN-US">
       reloc
      </span>
      段的根本原因是，程序在连接时连接器所假定的程序运行空间与实际程序加载到的内存空间不同。假如有这样一条指令：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      jsr app_start;
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      这一条指令采用直接寻址，跳转到
      <span lang="EN-US">
       app_start
      </span>
      地址处执行，连接程序将在编译完成是计算出
      <span lang="EN-US">
       app_start
      </span>
      的实际地址（设若实际地址为
      <span lang="EN-US">
       0x10000
      </span>
      ），这个实际地址是根据
      <span lang="EN-US">
       ld
      </span>
      文件计算出来（因为连接器假定该程序将被加载到由
      <span lang="EN-US">
       ld
      </span>
      文件指明的内存空间）。但实际上由于内存分配的关系，操作 系统在加载时无法保证程序将按
      <span lang="EN-US">
       ld
      </span>
      文件加载。这时如果程序仍然跳转到绝对地址
      <span lang="EN-US">
       0x10000
      </span>
      处执行，通常情况这是不正确的。一个解决办法是增加一个存储空 间，用于存储
      <span lang="EN-US">
       app_start
      </span>
      的实际地址，设若使用变量
      <span lang="EN-US">
       addr
      </span>
      表示这个存储空间。则以上这句程序将改为：
      <span lang="EN-US">
       <br/>
       movl addr, a0;
       <br/>
       jsr (a0);
       <br/>
      </span>
      增加的变量
      <span lang="EN-US">
       addr
      </span>
      将在数据段中占用一个
      <span lang="EN-US">
       4
      </span>
      字节的空间，连接器将
      <span lang="EN-US">
       app_start
      </span>
      的绝对地址存储到该变量。在可执行文件加载时，可执行文件加载器根据程 序将要加载的内存空间计算出
      <span lang="EN-US">
       app_start
      </span>
      在内存中的实际位置，写入
      <span lang="EN-US">
       addr
      </span>
      变量。系统在实际处理是不需要知道这个变量的确切存储位置（也不可能知 道），系统只要对整个
      <span lang="EN-US">
       reloc
      </span>
      段进行处理就可以了（
      <span lang="EN-US">
       reloc
      </span>
      段有标识，系统可以读出来）。处理很简单只需要对
      <span lang="EN-US">
       reloc
      </span>
      段中存储的值统一加上一个偏置 （如果加载的空间比预想的要靠前，实际上是减去一个偏移量）。偏置由实际的物理地址起始值同
      <span lang="EN-US">
       ld
      </span>
      文件指定的地址起始值相减计算出。
      <span lang="EN-US">
       <br/>
      </span>
      这种
      <span lang="EN-US">
       reloc
      </span>
      的方式部分是由
      <span lang="EN-US">
       uClinux
      </span>
      的内存分配问题引起的，这一点将在
      <span lang="EN-US">
       uClinux
      </span>
      内存管理分析时说明。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong>
      <span style="font-size: 12pt; font-family: 宋体;">
       针对实时性的解决方案
      </span>
     </strong>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      uClinux
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      本身并没有关注实时问题，它并不是为了
      <span lang="EN-US">
       Linux
      </span>
      的实时性而提出的。另外有一种
      <span lang="EN-US">
       Linux--Rt-linux
      </span>
      关注实时问题。
      <span lang="EN-US">
       Rt- linux
      </span>
      执行管理器把普通
      <span lang="EN-US">
       Linux
      </span>
      的内核当成一个任务运行，同时还管理了实时进程。而非实时进程则交给普通
      <span lang="EN-US">
       Linux
      </span>
      内核处理。这种方法已经应用于很 多的操作系统用于增强操作系统的实时性，包括一些商用版
      <span lang="EN-US">
       UNIX
      </span>
      系统，
      <span lang="EN-US">
       Windows NT
      </span>
      等等。这种方法优点之一是实现简单，且实时性能容易检验。优点之二是由于非实时进程运行于标准
      <span lang="EN-US">
       Linux
      </span>
      系统，同其它
      <span lang="EN-US">
       Linux
      </span>
      商用版本之间保持了很 大的兼容性。优点之三是可以支持硬实时时钟的应用。
      <span lang="EN-US">
       uClinux
      </span>
      可以使用
      <span lang="EN-US">
       Rt-linux
      </span>
      的
      <span lang="EN-US">
       patch
      </span>
      ，从而增强
      <span lang="EN-US">
       uClinux
      </span>
      的实时性，使得
      <span lang="EN-US">
       uClinux
      </span>
      可以应用于工业控制、进程控制等一些实时要求较高的应用。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <strong>
      <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
       uClinux
      </span>
     </strong>
     <strong>
      <span style="font-size: 12pt; font-family: 宋体;">
       的内存管理
      </span>
     </strong>
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      应该说
      <span lang="EN-US">
       uClinux
      </span>
      同标准
      <span lang="EN-US">
       Linux
      </span>
      的最大区别就在于内存管理，同时也由于
      <span lang="EN-US">
       uClinux
      </span>
      的内存管理引发了一些标准
      <span lang="EN-US">
       Linux
      </span>
      所不会出现的问题。本文将把
      <span lang="EN-US">
       uClinux
      </span>
      内存管理同标准
      <span lang="EN-US">
       Linux
      </span>
      的那内存管理部分进行比较分析。
      <span lang="EN-US">
       <br/>
      </span>
      标准
      <span lang="EN-US">
       Linux
      </span>
      使用的虚拟存储器技术
      <span lang="EN-US">
       <br/>
      </span>
      标准
      <span lang="EN-US">
       Linux
      </span>
      使用虚拟存储器技术，这种技术用于提供比计算机系统中实际使用的物理内存大得多的内存空间。使用者将感觉到好像程序可以使用非常大的内存空间，从而使得编程人员在写程序时不用考虑计算机中的物理内存的实际容量。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
      为了支持虚拟存储管理器的管理，
      <span lang="EN-US">
       Linux
      </span>
      系统采用分页（
      <span lang="EN-US">
       paging
      </span>
      ）的方式来载入进程。所谓分页既是把实际的存储器分割为相同大小的段，例如每个段
      <span lang="EN-US">
       1024
      </span>
      个字节，这样
      <span lang="EN-US">
       1024
      </span>
      个字节大小的段便称为一个页面（
      <span lang="EN-US">
       page
      </span>
      ）。
      <span lang="EN-US">
       <br/>
      </span>
      虚拟存储器由存储器管理机制及一个大容量的快速硬盘存储器支持。它的实现基于局部性原理，当一个程序在运行之前，没有必要全部装入内存，而是仅将那些当前 要运行的那些部分页面或段装入内存运行（
      <span lang="EN-US">
       copy-on-write
      </span>
      ），其余暂时留在硬盘上程序运行时如果它所要访问的页（段）已存在，则程序继续运行， 如果发现不存在的页（段），操作系统将产生一个页错误（
      <span lang="EN-US">
       page fault
      </span>
      ），这个错误导致操作系统把需要运行的部分加载到内存中。必要时操作系统还可以把不需要的内存页（段）交换到磁盘上。利用这样的方式管理存储 器，便可把一个进程所需要用到的存储器以化整为零的方式，视需求分批载入，而核心程序则凭借属于每个页面的页码来完成寻址各个存储器区段的工作。
      <span lang="EN-US">
       <br/>
      </span>
      标准
      <span lang="EN-US">
       Linux
      </span>
      是针对有内存管理单元的处理器设计的。在这种处理器上，虚拟地址被送到内存管理单元（
      <span lang="EN-US">
       MMU
      </span>
      ），把虚拟地址映射为物理地址。
      <span lang="EN-US">
       <br/>
      </span>
      通过赋予每个任务不同的虚拟
      <span lang="EN-US">
       --
      </span>
      物理地址转换映射，支持不同任务之间的保护。地址转换函数在每一个任务中定义，在一个任务中的虚拟地址空间映射到物理内存 的一个部分，而另一个任务的虚拟地址空间映射到物理存储器中的另外区域。计算机的存储管理单元（
      <span lang="EN-US">
       MMU
      </span>
      ）一般有一组寄存器来标识当前运行的进程的转换表。 在当前进程将
      <span lang="EN-US">
       CPU
      </span>
      放弃给另一个进程时（一次上下文切换），内核通过指向新进程地址转换表的指针加载这些寄存器。
      <span lang="EN-US">
       MMU
      </span>
      寄存器是有特权的，只能在内核态才 能访问。这就保证了一个进程只能访问自己用户空间内的地址，而不会访问和修改其它进程的空间。当可执行文件被加载时，加载器根据缺省的
      <span lang="EN-US">
       ld
      </span>
      文件，把程序加 载到虚拟内存的一个空间，因为这个原因实际上很多程序的虚拟地址空间是相同的，但是由于转换函数不同，所以实际所处的内存区域也不同。而对于多进程管理当 处理器进行进程切换并执行一个新任务时，一个重要部分就是为新任务切换任务转换表。我们可以看到
      <span lang="EN-US">
       Linux
      </span>
      系统的内存管理至少实现了以下功能：
      <span lang="EN-US">
       <br/>
      </span>
      运行比内存还要大的程序。理想情况下应该可以运行任意大小的程序
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      可以运行只加载了部分的程序，缩短了程序启动的时间
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      可以使多个程序同时驻留在内存中提高
      <span lang="EN-US">
       CPU
      </span>
      的利用率
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      可以运行重定位程序。即程序可以方于内存中的任何一处，而且可以在执行过程中移动。
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      写机器无关的代码。程序不必事先约定机器的配置情况。
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      减轻程序员分配和管理内存资源的负担。
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      可以进行共享
      <span lang="EN-US">
       --
      </span>
      例如，如果两个进程运行同一个程序，它们应该可以共享程序代码的同一个副本。
      <span lang="EN-US">
       <br/>
       ◇
      </span>
      提供内存保护，进程不能以非授权方式访问或修改页面，内核保护单个进程的数据和代码以防止其它进程修改它们。否则，用户程序可能会偶然（或恶意）的破坏内核或其它用户程序。
      <span lang="EN-US">
       <br/>
      </span>
      虚存系统并不是没有代价的。内存管理需要地址转换表和其他一些数据结构，留给程序的内存减少了。地址转换增加了每一条指令的执行时间，而对于有额外内存操 作的指令会更严重。当进程访问不在内存的页面时，系统发生失效。系统处理该失效，并将页面加载到内存中，这需要极耗时间的磁盘
      <span lang="EN-US">
       I/O
      </span>
      操作。总之内存管理活 动占用了相当一部分
      <span lang="EN-US">
       cpu
      </span>
      时间（在较忙的系统中大约占
      <span lang="EN-US">
       10
      </span>
      ％）。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      针对
      <span lang="EN-US">
       NOMMU
      </span>
      的特殊处理
      <span lang="EN-US">
       <br/>
      </span>
      对于
      <span lang="EN-US">
       uClinux
      </span>
      来说，其设计针对没有
      <span lang="EN-US">
       MMU
      </span>
      的处理器，即
      <span lang="EN-US">
       uClinux
      </span>
      不能使用处理器的虚拟内存管理技术（应该说这种不带有
      <span lang="EN-US">
       MMU
      </span>
      的处理器在嵌入式设 备中相当普偏）。
      <span lang="EN-US">
       uClinux
      </span>
      仍然采用存储器的分页管理，系统在启动时把实际存储器进行分页。在加载应用程序时程序分页加载。但是由于没有
      <span lang="EN-US">
       MMU
      </span>
      管理， 所以实际上
      <span lang="EN-US">
       uClinux
      </span>
      采用实存储器管理策略（
      <span lang="EN-US">
       real memeory management
      </span>
      ）。这一点影响了系统工作的很多方面。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      系统对于内存的访问是直接的，（它对地址的访问不需要经过
      <span lang="EN-US">
       MMU
      </span>
      ，而是直接送到地址线上输出），所有程序中访问的地址都是实际的物理地址。 操作系统对内存空间没有保护（这实际上是很多嵌入式系统的特点），各个进程实际上共享一个运行空间（没有独立的地址转换表）。
      <span lang="EN-US">
       <br/>
      </span>
      一个进程在执行前，系统必须为进程分配足够的连续地址空间，然后全部载入主存储器的连续空间中。与之相对应的是标准
      <span lang="EN-US">
       Linux
      </span>
      系统在分配内存时没有必要保 证实际物理存储空间是连续的，而只要保证虚存地址空间连续就可以了。另外一个方面程序加载地址与预期（
      <span lang="EN-US">
       ld
      </span>
      文件中指出的）通常都不相同，这样
      <span lang="EN-US">
       relocation
      </span>
      过程就是必须的。此外磁盘交换空间也是无法使用的，系统执行时如果缺少内存将无法通过磁盘交换来得到改善。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      对内存的管理减少同时就给开发人员提出了更高的要求。如果从易用性这一点来说，
      <span lang="EN-US">
       uClinux
      </span>
      的内存管理是一种倒退，退回了到了
      <span lang="EN-US">
       UNIX
      </span>
      早 期或是
      <span lang="EN-US">
       Dos
      </span>
      系统时代。开发人员不得不参与系统的内存管理。从编译内核开始，开发人员必须告诉系统这块开发板到底拥有多少的内存（假如你欺骗了系统，那将 在后面运行程序时受到惩罚），从而系统将在启动的初始化阶段对内存进行分页，并且标记已使用的和未使用的内存。系统将在运行应用时使用这些分页内存。
      <span lang="EN-US">
       <br/>
      </span>
      由于应用程序加载时必须分配连续的地址空间，而针对不同硬件平台的可一次成块（连续地址）分配内存大小限制是不同（目前针对
      <span lang="EN-US">
       ez328
      </span>
      处理器的
      <span lang="EN-US">
       uClinux
      </span>
      是
      <span lang="EN-US">
       128k
      </span>
      ，而针对
      <span lang="EN-US">
       coldfire
      </span>
      处理器的系统内存则无此限制），所以开发人员在开发应用程序时必须考虑内存的分配情况并关注应用程序需 要运行空间的大小。另外由于采用实存储器管理策略，用户程序同内核以及其它用户程序在一个地址空间，程序开发时要保证不侵犯其它程序的地址空间，以使得程 序不至于破坏系统的正常工作，或导致其它程序的运行异常。
      <span lang="EN-US">
       <br/>
      </span>
      从内存的访问角度来看，开发人员的权利增大了（开发人员在编程时可以访问任意的地址空间），但与此同时系统的安全性也大为下降。此外，系统对多进程的管理将有很大的变化，这一点将在
      <span lang="EN-US">
       uClinux
      </span>
      的多进程管理中说明。
      <span lang="EN-US">
       <br/>
      </span>
      虽然
      <span lang="EN-US">
       uClinux
      </span>
      的内存管理与标准
      <span lang="EN-US">
       Linux
      </span>
      系统相比功能相差很多，但应该说这是嵌入式设备的选择。在嵌入式设备中，由于成本等敏感因素的影响，普偏的 采用不带有
      <span lang="EN-US">
       MMU
      </span>
      的处理器，这决定了系统没有足够的硬件支持实现虚拟存储管理技术。从嵌入式设备实现的功能来看，嵌入式设备通常在某一特定的环境下运行， 只要实现特定的功能，其功能相对简单，内存管理的要求完全可以由开发人员考虑。
      <span lang="EN-US">
       <br/>
      </span>
      标准
      <span lang="EN-US">
       Linux
      </span>
      系统的进程、线程
      <span lang="EN-US">
       <br/>
      </span>
      进程：进程是一个运行程序并为其提供执行环境的实体，它包括一个地址空间和至少一个控制点，进程在这个地址空间上执行单一指令序列。进程地址空间包括可以访问或引用的内存单元的集合，进程控制点通过一个一般称为程序计数器（
      <span lang="EN-US">
       program counter,PC
      </span>
      ）的硬件寄存器控制和跟踪进程指令序列。
      <span lang="EN-US">
       <br/>
       fork
      </span>
      ：由于进程为执行程序的环境，因此在执行程序前必须先建立这个能
      <span lang="EN-US">
       "
      </span>
      跑
      <span lang="EN-US">
       "
      </span>
      程序的环境。
      <span lang="EN-US">
       Linux
      </span>
      系统提供系统调用拷贝现行进程的内容，以产生新的进 程，调用
      <span lang="EN-US">
       fork
      </span>
      的进程称为父进程；而所产生的新进程则称为子进程。子进程会承袭父进程的一切特性，但是它有自己的数据段，也就是说，尽管子进程改变了所 属的变量，却不会影响到父进程的变量值。
      <span lang="EN-US">
       <br/>
      </span>
      父进程和子进程共享一个程序段，但是各自拥有自己的堆栈、数据段、用户空间以及进程控制块。换言之，两个进程执行的程序代码是一样的，但是各有各的程序计数器与自己的私人数据。
      <span lang="EN-US">
       <br/>
      </span>
      当内核收到
      <span lang="EN-US">
       fork
      </span>
      请求时，它会先查核三件事：首先检查存储器是不是足够；其次是进程表是否仍有空缺；最后则是看看用户是否建立了太多的子进程。如果上述 说三个条件满足，那么操作系统会给子进程一个进程识别码，并且设定
      <span lang="EN-US">
       cpu
      </span>
      时间，接着设定与父进程共享的段，同时将父进程的
      <span lang="EN-US">
       inode
      </span>
      拷贝一份给子进程运 用，最终子进程会返回数值
      <span lang="EN-US">
       0
      </span>
      以表示它是子进程，至于父进程，它可能等待子进程的执行结束，或与子进程各做个的。
      <span lang="EN-US">
       <br/>
       exec
      </span>
      系统调用：该系统调用提供一个进程去执行另一个进程的能力，
      <span lang="EN-US">
       exec
      </span>
      系统调用是采用覆盖旧有进程存储器内容的方式，所以原来程序的堆栈、数据段与程序段都会被修改，只有用户区维持不变。
      <span lang="EN-US">
       <br/>
       vfork
      </span>
      系统调用：由于在使用
      <span lang="EN-US">
       fork
      </span>
      时，内核会将父进程拷贝一份给子进程，但是这样的做法相当浪费时间，因为大多数的情形都是程序在调用
      <span lang="EN-US">
       fork
      </span>
      后就 立即调用
      <span lang="EN-US">
       exec
      </span>
      ，这样刚拷贝来的进程区域又立即被新的数据覆盖掉。因此
      <span lang="EN-US">
       Linux
      </span>
      系统提供一个系统调用
      <span lang="EN-US">
       vfork
      </span>
      ，
      <span lang="EN-US">
       vfork
      </span>
      假定系统在调用完成
      <span lang="EN-US">
       vfork
      </span>
      后会马上执行
      <span lang="EN-US">
       exec
      </span>
      ，因此
      <span lang="EN-US">
       vfork
      </span>
      不拷贝父进程的页面，只是初始化私有的数据结构与准备足够的分页表。这样实际在
      <span lang="EN-US">
       vfork
      </span>
      调用完成后父子 进程事实上共享同一块存储器（在子进程调用
      <span lang="EN-US">
       exec
      </span>
      或是
      <span lang="EN-US">
       exit
      </span>
      之前），因此子进程可以更改父进程的数据及堆栈信息，因此
      <span lang="EN-US">
       vfork
      </span>
      系统调用完成后，父进 程进入睡眠，直到子进程执行
      <span lang="EN-US">
       exec
      </span>
      。当子进程执行
      <span lang="EN-US">
       exec
      </span>
      时，由于
      <span lang="EN-US">
       exec
      </span>
      要使用被执行程序的数据，代码覆盖子进程的存储区域，这样将产生写保护错误 （
      <span lang="EN-US">
       do_wp_page
      </span>
      ）（这个时候子进程写的实际上是父进程的存储区域），
      <span lang="EN-US">
       <br/>
      </span>
      这个错误导致内核为子进程重新分配存储空间。当子进程正确开始执行后，将唤醒父进程，使得父进程继续往后执行。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      的多进程处理
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      没有
      <span lang="EN-US">
       mmu
      </span>
      管理存储器，在实现多个进程时（
      <span lang="EN-US">
       fork
      </span>
      调用生成子进程）需要实现数据保护。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      的
      <span lang="EN-US">
       fork
      </span>
      和
      <span lang="EN-US">
       vfork
      </span>
      ：
      <span lang="EN-US">
       uClinux
      </span>
      的
      <span lang="EN-US">
       fork
      </span>
      等于
      <span lang="EN-US">
       vfork
      </span>
      。实际上
      <span lang="EN-US">
       uClinux
      </span>
      的多进程管理通过
      <span lang="EN-US">
       vfork
      </span>
      来实现。这意味着
      <span lang="EN-US">
       uClinux
      </span>
      系统
      <span lang="EN-US">
       fork
      </span>
      调用完程后，要么子进程代替父进程执行（此时父进程已经
      <span lang="EN-US">
       sleep
      </span>
      ）直到子进程调用
      <span lang="EN-US">
       exit
      </span>
      退出，要么调用
      <span lang="EN-US">
       exec
      </span>
      执行一个新 的进程，这个时候将产生可执行文件的加载，即使这个进程只是父进程的拷贝，这个过程也不能避免。当子进程执行
      <span lang="EN-US">
       exit
      </span>
      或
      <span lang="EN-US">
       exec
      </span>
      后，子进程使用
      <span lang="EN-US">
       wakeup
      </span>
      把父进程唤醒，父进程继续往下执行。
      <span lang="EN-US">
       <br/>
       uClinux
      </span>
      的这种多进程实现机制同它的内存管理紧密相关。
      <span lang="EN-US">
       uClinux
      </span>
      针对
      <span lang="EN-US">
       nommu
      </span>
      处理器开发，所以被迫使用一种
      <span lang="EN-US">
       flat
      </span>
      方式的内存管理模式，启 动新的应用程序时系统必须为应用程序分配存储空间，并立即把应用程序加载到内存。缺少了
      <span lang="EN-US">
       MMU
      </span>
      的内存重映射机制，
      <span lang="EN-US">
       uClinux
      </span>
      必须在可执行文件加载阶段 对可执行文件
      <span lang="EN-US">
       reloc
      </span>
      处理，使得程序执行时能够直接使用物理内存。
      <span lang="EN-US">
      </span>
     </span>
    </p>

      <span lang="EN-US">

      <span lang="EN-US">

    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      不知你是否注意到，在我们的日常生活中，人们开始普遍使用手机、
      <span lang="EN-US">

      </span>

      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 24pt;">
     <span style="font-size: 12pt; font-family: 宋体;">
      早在微软创办之初，比尔
      <span lang="EN-US">
       ·
      </span>
      盖茨就曾梦想
      <span lang="EN-US">
       “
      </span>
      每人桌上一台
      <span lang="EN-US">
       PC
      </span>
      ，每台
      <span lang="EN-US">
       PC
      </span>
      中运行微软的软件
      <span lang="EN-US">
       ”
      </span>
      ，这就是后来著名的
      <span lang="EN-US">
       “
      </span>
      信息随手可得
      <span lang="EN-US">

      <span lang="EN-US">

      </span>

      <span lang="EN-US">
       )
      </span>

      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 24pt;">
     <span style="font-size: 12pt; font-family: 宋体;">
      但是
      <span lang="EN-US">
       PC
      </span>

      <span lang="EN-US">

      <span lang="EN-US">

      <span lang="EN-US">

    <p class="MsoNormal" style="text-indent: 24pt;">
     <span style="font-size: 12pt; font-family: 宋体;">

      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      Windows CE
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">

      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">

    <p align="left" class="MsoNormal" style="text-align: left;">
     <span style="font-size: 12pt; font-family: 宋体;">
     </span>
     <strong style="">
      <span style="font-size: 15pt; font-family: 宋体;">
       二．
      </span>
     </strong>
     <strong style="">
      <span style="font-size: 15pt; font-family: 宋体;">
       嵌入式操作系统选择原则和对比
       <span lang="EN-US">
       </span>
      </span>
     </strong>
    </p>
    <p class="MsoNormal">
     <span style="font-size: 12pt; font-family: 宋体;">
      下面介绍了选择嵌入式操作系统应遵循的六条原则，并比较了目前几种嵌入式操作系统
      <span lang="EN-US">
       VxWorks
      </span>
      、
      <span lang="EN-US">
       WindowsCE
      </span>
      、嵌入式
      <span lang="EN-US">
       Linux
      </span>
      、μ
      <span lang="EN-US">
       C/OS-
      </span>
      Ⅱ等各自的优缺点。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      嵌入式系统是以应用为中心，软硬件可裁减的，适用于对功能、可靠性、成本、体积、功耗等综合性严格要求的专用计算机系统。具有软件代码小、高度自动化、响应速度快等特点，特别适合于要求实时和多任务的体系。嵌入式系统主要由嵌入式处理器、相关支撑硬件、嵌入式操作系统及应用软件系统等组成，它是可独立工作的“器件”。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      一、嵌入式操作系统的发展
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      作为嵌入式系统（包括硬、软件系统）极为重要的组成部分的嵌入式操作系统，通常包括与硬件相关的底层驱动软件、系统内核、设备驱动接口、通信协议、图形界面、标准化浏览器等。嵌入式操作系统具有通用操作系统的基本特点，如能够有效管理越来越复杂的系统资源；能够把硬件虚拟化，使得开发人员从繁忙的驱动程序移植和维护中解脱出来；能够提供库函数、驱动程序、工具集以及应用程序。与通用操作系统相比较，嵌入式操作系统在系统实时高效性、硬件的相关依赖性、软件固态化以及应用的专用性等方面具有较为突出的特点。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      嵌入式操作系统伴随着嵌入式系统的发展经历了四个比较明显的阶段：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      第一阶段：无操作系统的嵌入算法阶段，以单芯片为核心的可编程控制器形式的系统，具有与监测、伺服、指示设备相配合的功能。应用于一些专业性极强的工业控制系统中，通过汇编语言编程对系统进行直接控制，运行结束后清除内存。系统结构和功能都相对单一，处理效率较低，存储容量较小，几乎没有用户接口。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      第二阶段：以嵌人式
      <span lang="EN-US">
       CPU
      </span>
      为基础、简单操作系统为核心的嵌入式系统。
      <span lang="EN-US">
       CPU
      </span>
      种类繁多，通用性比较差；系统开销小，效率高；一般配备系统仿真器，操作系统具有一定的兼容性和扩展性；应用软件较专业，用户界面不够友好；系统主要用来控制系统负载以及监控应用程序运行。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      第三阶段：通用的嵌人式实时操作系统阶段，以嵌入式操作系统为核心的嵌入式系统。能运行于各种类型的微处理器上，兼容性好；内核精小、效率高，具有高度的模块化和扩展性；具备文件和目录管理、设备支持、多任务、网络支持、图形窗口以及用户界面等功能；具有大量的应用程序接口（
      <span lang="EN-US">
       APl
      </span>
      ）；嵌入式应用软件丰富。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      第四阶段：以基于
      <span lang="EN-US">
       Internet
      </span>
      为标志的嵌入式系统。这是一个正在迅速发展的阶段。目前大多数嵌入式系统还孤立于
      <span lang="EN-US">
       Internet
      </span>
      之外，但随着
      <span lang="EN-US">
       Internet
      </span>
      的发展以及
      <span lang="EN-US">
       Internet
      </span>
      技术与信息家电、工业控制技术等结合日益密切，嵌入式设备与
      <span lang="EN-US">
       Intemet
      </span>
      的结合将代表着嵌入式技术的真正未来。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      二、使用实时操作系统的必要性
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      嵌入式实时操作系统在目前的嵌入式应用中用得越来越广泛，尤其在功能复杂、系统庞大的应用中显得愈来愈重要。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      <span style="">
      </span>
      1
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ．嵌人式实时操作系统提高了系统的可靠性。在控制系统中，出于安全方面的考虑，要求系统起码不能崩溃，而且还要有自愈能力。不仅要求在硬件设计方面提高系统的可靠性和抗干扰性，而且也应在软件设计方面提高系统的抗干扰性，尽可能地减少安全漏洞和不可靠的隐患。长期以来的前后台系统软件设计在遇到强干扰时，使得运行的程序产生异常、出错、跑飞，甚至死循环，造成了系统的崩溃。而实时操作系统管理的系统，这种干扰可能只是引起若干进程中的一个被破坏，可以通过系统运行的系统监控进程对其进行修复。通常情况下，这个系统监视进程用来监视各进程运行状况，遇到异常情况时采取一些利于系统稳定可靠的措施，如把有问题的任务清除掉。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      2
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ．提高了开发效率，缩短了开发周期。在嵌入式实时操作系统环境下，开发一个复杂的应用程序，通常可以按照软件工程中的解耦原则将整个程序分解为多个任务模块。每个任务模块的调试、修改几乎不影响其他模块。商业软件一般都提供了良好的多任务调试环境。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      3
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      ．嵌入式实时操作系统充分发挥了
      <span lang="EN-US">
       32
      </span>
      位
      <span lang="EN-US">
       CPU
      </span>
      的多任务潜力。
      <span lang="EN-US">
       32
      </span>
      位
      <span lang="EN-US">
       CPU
      </span>
      比
      <span lang="EN-US">
       8
      </span>
      、
      <span lang="EN-US">
       16
      </span>
      位
      <span lang="EN-US">
       CPU
      </span>
      快，另外它本来是为运行多用户、多任务操作系统而设计的，特别适于运行多任务实时系统。
      <span lang="EN-US">
       32
      </span>
      位
      <span lang="EN-US">
       CPU
      </span>
      采用利于提高系统可靠性和稳定性的设计，使其更容易做到不崩溃。例如，
      <span lang="EN-US">
       CPU
      </span>
      运行状态分为系统态和用户态。将系统堆栈和用户堆栈分开，以及实时地给出
      <span lang="EN-US">
       CPU
      </span>
      的运行状态等，允许用户在系统设计中从硬件和软件两方面对实时内核的运行实施保护。如果还是采用以前的前后台方式，则无法发挥
      <span lang="EN-US">
       32
      </span>
      位
      <span lang="EN-US">
       CPU
      </span>
      的优势。从某种意义上说，没有操作系统的计算机（裸机）是没有用的。在嵌入式应用中，只有把
      <span lang="EN-US">
       CPU
      </span>
      嵌入到系统中，同时又把操作系统嵌入进去，才是真正的计算机嵌入式应用。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      三、嵌入式操作系统选型
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      当我们在设计信息电器、数字医疗设备等嵌入式产品时，嵌入式操作系统的选择至关重要。一般而言，在选择嵌入式操作系统时，可以遵循以下原则。总的来说，就是“做加法还是做减法”的问题。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （一）市场进入时间
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      制定产品时间表与选择操作系统有关系，实际产品和一般演示是不同的。目前是
      <span lang="EN-US">
       Windows
      </span>
      程序员可能是人力资源最丰富的。现成资源最多的也就可能是
      <span lang="EN-US">
       WinCE
      </span>
      。使用
      <span lang="EN-US">
       WinCE
      </span>
      能够很快进入市场。因为
      <span lang="EN-US">
       WinCE+X86
      </span>
      做产品实际上是在做减法，去掉你不要的功能，能很快出产品，但伴随的可能是成本高，核心竞争力差。而某些高效的操作系统可能由于编程人员缺乏，或由于这方面的技术积累不够，影响开发进度。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （二）可移植性
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      操作系统相关性。当进行嵌入式软件开发时，可移植性是要重点考虑的问题。良好的软件移植性应该比较好，可以在不同平台、不同系统上运行，跟操作系统无关。软件的通用性和软件的性能通常是矛盾的。即通用以损失某些特定情况下的优化性能为代价。很难设想开发一个嵌入式浏览器而仅能在某一特定环境下应用。反过来说，当产品与平台和操作系统紧密结合时，往往你的产品的特色就蕴含其中。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （三）可利用资源
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      产品开发不同于学术课题研究，它是以快速、低成本、高质量的推出适合用户需求的产品为目的的。集中精力研发出产品的特色，其他功能尽量由操作系统附加或采用第三方产品，因此操作系统的可利用资源对于选型是一个重要参考条件。
      <span lang="EN-US">
       Linux
      </span>
      和
      <span lang="EN-US">
       WinCE
      </span>
      都有大量的资源可以利用，这是他们被看好的重要原因。其他有些实时操作系统由于比较封闭，开发时可以利用的资源比较少，因此多数功能需要自己独立开发。从而影响开发进度。近来的市场需求显示，越来越多的嵌入式系统，均要求提供全功能的
      <span lang="EN-US">
       Web
      </span>
      浏览器。而这要求有一个高性能、高可靠的
      <span lang="EN-US">
       GUI
      </span>
      的支持。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （四）系统定制能力
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      信息产品不同于传统
      <span lang="EN-US">
       PC
      </span>
      的
      <span lang="EN-US">
       Wintel
      </span>
      结构的单纯性，用户的需求是千差万别的，硬件平台也都不一样，所以对系统的定制能力提出了要求。要分析产品是否对系统底层有改动的需求，这种改动是否伴随着产品特色
      <span lang="EN-US">
       ?Linux
      </span>
      由于其源代码开放的天生魅力，在定制能力方面具有优势。随着
      <span lang="EN-US">
       WinCE3.0
      </span>
      原码的开放，以及微软在嵌入式领域力度的加强，其定制能力会有所提升。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （五）成本
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      成本是所有产品不得不考虑的问题。操作系统的选择会对成本有什么影响呢
      <span lang="EN-US">
       ?Linux
      </span>
      免费，
      <span lang="EN-US">
       WinCE
      </span>
      等商业系统需要支付许可证使用费，但这都不是问题的答案。成本是需要综合权衡以后进行考虑的——选择某一系统可能会对其他一系列的因素产生影响，如对硬件设备的选型、人员投入、以及公司管理和与其他合作伙伴的共同开发之间的沟通等许多方面的影响。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （六）中文内核支持
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      国内产品需要对中文的支持。由于操作系统多数是采用西文方式，是否支持双字节编码方式，是否遵循
      <span lang="EN-US">
       GBK
      </span>
      ，
      <span lang="EN-US">
       GBl8030
      </span>
      等各种国家标准，是否支持中文输入与处理，是否提供第三方中文输入接口是针对国内用户的嵌入式产品的必需考虑的重要因素。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span style="font-size: 12pt; font-family: 宋体;">
      上面提到用
      <span lang="EN-US">
       WinCE+x86
      </span>
      出产品是减法，这实际上就是所谓
      <span lang="EN-US">
       PC
      </span>
      家电化；另外一种做法是加法，利用家电行业的硬件解决方案（绝大部分是非
      <span lang="EN-US">
       x86
      </span>
      的）加以改进，加上嵌入式操作系统，再加上应用软件。这是所谓家电
      <span lang="EN-US">
       PC
      </span>
      化的做法，这种加法的优势是成本低，特色突出，缺点是产品研发周期长，难度大（需要深入了解硬件和操作系统）。如果选择这种做法，
      <span lang="EN-US">
       Linux
      </span>
      是一个好选择，它让你能够深入到系统底层，如果你愿意并且有能力。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span style="font-size: 12pt; font-family: 宋体;">
      四、几种代表性嵌入式操作系统比较
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （一）
      <span lang="EN-US">
       VxWorks
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      VxWorks
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      操作系统是美国
      <span lang="EN-US">
       WindRiver
      </span>
      公司于
      <span lang="EN-US">
       1983
      </span>
      年设计开发的一种嵌入式实时操作系统（
      <span lang="EN-US">
       RTOS
      </span>
      ），是
      <span lang="EN-US">
       Tornado
      </span>
      嵌入式开发环境的关键组成部分。良好的持续发展能力、高性能的内核以及友好的用户开发环境，在嵌人式实时操作系统领域逐渐占据一席之地。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      VxWorks
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      具有可裁剪微内核结构；高效的任务管理；灵活的任务间通讯；微秒级的中断处理；支持
      <span lang="EN-US">
       POSIX 1003
      </span>
      ．
      <span lang="EN-US">
       1b
      </span>
      实时扩展标准；支持多种物理介质及标准的、完整的
      <span lang="EN-US">
       TCP/IP
      </span>
      网络协议等。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      然而其价格昂贵。由于操作系统本身以及开发环境都是专有的，价格一般都比较高，通常需花费
      <span lang="EN-US">
       10
      </span>
      万元人民币以上才能建起一个可用的开发环境，对每一个应用一般还要另外收取版税。一般不通供源代码，只提供二进制代码。由于它们都是专用操作系统，需要专门的技术人员掌握开发技术和维护，所以软件的开发和维护成本都非常高。支持的硬件数量有限。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （二）
      <span lang="EN-US">
       Windows CE
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      Windows CE
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      与
      <span lang="EN-US">
       Windows
      </span>
      系列有较好的兼容性，无疑是
      <span lang="EN-US">
       Windows CE
      </span>
      推广的一大优势。其中
      <span lang="EN-US">
       WinCE3.0
      </span>
      是一种针对小容量、移动式、智能化、
      <span lang="EN-US">
       32
      </span>
      位、了解设备的模块化实时嵌人式操作系统。为建立针对掌上设备、无线设备的动态应用程序和服务提供了一种功能丰富的操作系统平台，它能在多种处理器体系结构上运行，并且通常适用于那些对内存占用空间具有一定限制的设备。它是从整体上为有限资源的平台设计的多线程、完整优先权、多任务的操作系统。它的模块化设计允许它对从掌上电脑到专用的工业控制器的用户电子设备进行定制。操作系统的基本内核需要至少
      <span lang="EN-US">
       200KB
      </span>
      的
      <span lang="EN-US">
       ROM
      </span>
      。由于嵌入式产品的体积、成本等方面有较严格的要求，所以处理器部分占用空间应尽可能的小。系统的可用内存和外存数量也要受限制，而嵌入式操作系统就运行在有限的内存（一般在
      <span lang="EN-US">
       ROM
      </span>
      或快闪存储器）中，因此就对操作系统的规模、效率等提出了较高的要求。从技术角度上讲，
      <span lang="EN-US">
       Windows CE
      </span>
      作为嵌入式操作系统有很多的缺陷：没有开放源代码，使应用开发人员很难实现产品的定制；在效率、功耗方面的表现并不出色，而且和
      <span lang="EN-US">
       Windows
      </span>
      一样占用过的系统内存，运用程序庞大；版权许可费也是厂商不得不考虑的因素。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （三）嵌入式
      <span lang="EN-US">
       Linux
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      这是嵌入式操作系统的一个新成员，其最大的特点是源代码公开并且遵循
      <span lang="EN-US">
       GPL
      </span>
      协议，在近一年多以来成为研究热点，据
      <span lang="EN-US">
       IDG
      </span>
      预测嵌入式
      <span lang="EN-US">
       Linux
      </span>
      将占未来两年的嵌入式操作系统份额的
      <span lang="EN-US">
       50%
      </span>
      。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      由于其源代码公开，人们可以任意修改，以满足自己的应用，并且查错也很容易。遵从
      <span lang="EN-US">
       GPL
      </span>
      ，无须为每例应用交纳许可证费。有大量的应用软件可用。其中大部分都遵从
      <span lang="EN-US">
       GPL
      </span>
      ，是开放源代码和免费的。可以稍加修改后应用于用户自己的系统。有大量的免费的优秀的开发工具，且都遵从
      <span lang="EN-US">
       GPL
      </span>
      ，是开放源代码的。有庞大的开发人员群体。无需专门的人才，只要懂
      <span lang="EN-US">
       Unix/Linux
      </span>
      和
      <span lang="EN-US">
       C
      </span>
      语言即可。随着
      <span lang="EN-US">
       Linux
      </span>
      在中国的普及，这类人才越来越多。所以软件的开发和维护成本很低。优秀的网络功能，这在
      <span lang="EN-US">
       Internet
      </span>
      时代尤其重要。稳定——这是
      <span lang="EN-US">
       Linux
      </span>
      本身具备的一个很大优点。内核精悍，运行所需资源少，十分适合嵌入式应用。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      支持的硬件数量庞大。嵌入式
      <span lang="EN-US">
       Linux
      </span>
      和普通
      <span lang="EN-US">
       Linux
      </span>
      并无本质区别，
      <span lang="EN-US">
       PC
      </span>
      上用到的硬件嵌入式
      <span lang="EN-US">
       Linux
      </span>
      几乎都支持。而且各种硬件的驱动程序源代码都可以得到，为用户编写自己专有硬件的驱动程序带来很大方便。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      在嵌入式系统上运行
      <span lang="EN-US">
       Linux
      </span>
      的一个缺点是
      <span lang="EN-US">
       Linux
      </span>
      体系提供实时性能需要添加实时软件模块。而这些模块运行的内核空间正是操作系统实现调度策略、硬件中断异常和执行程序的部分。由于这些实时软件模块是在内核空间运行的，因此代码错误可能会破坏操作系统从而影响整个系统的可靠性，这对于实时应用将是一个非常严重的弱点。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      （四）μ
      <span lang="EN-US">
       C/OS
      </span>
      一Ⅱ
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      μ
      <span lang="EN-US">
       C/OS
      </span>
      一Ⅱ是著名的源代码公开的实时内核，是专为嵌入式应用设计的，可用于
      <span lang="EN-US">
       8
      </span>
      位，
      <span lang="EN-US">
       16
      </span>
      位和
      <span lang="EN-US">
       32
      </span>
      位单片机或数字信号处理器（
      <span lang="EN-US">
       DSP
      </span>
      ）。它是在原版本 μ
      <span lang="EN-US">
       C/OS
      </span>
      的基础上做了重大改进与升级，并有了近十年的使用实践，有许多成功应用该实时内核的实例。它的主要特点如下：
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      公开源代码，容易就能把操作系统移植到各个不同的硬件平台上；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      可移植性，绝大部分源代码是用
      <span lang="EN-US">
       C
      </span>
      语言写的，便于移植到其他微处理器上；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      可固化；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      可裁剪性，有选择的使用需要的系统服务，以减少斗所需的存储空间；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      占先式，完全是占先式的实时内核，即总是运行就绪条件下优先级最高的任务；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      多任务，可管理
      <span lang="EN-US">
       64
      </span>
      个任务，任务的优先级必须是不同的，不支持时间片轮转调度法；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      可确定性，函数调用与服务的执行时间具有其可确定性，不依赖于任务的多少；
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
      *
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      实用性和可靠性，成功应用该实时内核的实例，是其实用性和可靠性的最好证据。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      由于μ
      <span lang="EN-US">
       C/OS
      </span>
      一Ⅱ仅是一个实时内核，这就意味着它不像其他实时存在系统那样提供给用户的只是一些
      <span lang="EN-US">
       API
      </span>
      函数接口，还有很多工作需要用户自己去完成。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      五、结束语
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
     <span lang="EN-US" style="font-size: 12pt; font-family: 宋体;">
      <span style="">
      </span>
     </span>
     <span style="font-size: 12pt; font-family: 宋体;">
      在嵌入式应用中，使用实时操作系统（
      <span lang="EN-US">
       RTOS
      </span>
      ）是当前嵌入式应用的一个特点，一种趋势，也是单片机应用从低水平向高水平的一个进步。在实际的应用中，根据不同的要求和条件选择合适的操作系统，使开发工作更容易，设计出更完美的嵌入式系统。
      <span lang="EN-US">
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.75pt;">
