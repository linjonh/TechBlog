     软件工程-软件危机
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     <span style="font-size:14px">
      本文参考自
      <span style="color:rgb(54,46,43); font-family:Arial; line-height:24px; text-align:left; text-indent:28px">
       张海藩老师和牟永敏老师的
      </span>
      《软件工程导论》
     </span>
    </p>
    <h2>
     <span style="font-size:14px">
      定义
     </span>
    </h2>
    <blockquote style="margin:0 0 0 40px; border:none; padding:0px">
     <div>
      <span style="font-size:14px">
       计算机软件的开发和维护过程中所遇到的一系列严重问题。
      </span>
     </div>
    </blockquote>
    <h2>
     <span style="font-size:14px">
      包含的两方面
     </span>
    </h2>
    <div>
     <ol>
      <li>
       <span style="font-size:14px">
        如何开发软件，以满足对软件日益增长的需求。
       </span>
      </li>
      <li>
       <span style="font-size:14px">
        如何维护数量不断膨胀的已有软件。
       </span>
      </li>
     </ol>
    </div>
    <h2>
     <span style="font-size:14px">
      典型表现
     </span>
    </h2>
    <div>
     <ol>
      <li>
       <span style="font-size:14px">
        对软件开发成本和进度的估计常常很不正确。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          对于delay的项目，通常：
         </span>
         <ul>
          <li>
           <span style="font-size:14px">
            降低了软件开发组织的信誉。
           </span>
          </li>
          <li>
           <span style="font-size:14px">
            为了赶进度和节约成本，又可能损害软件质量。
           </span>
          </li>
         </ul>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        用户对“已完成的”软件系统不满意。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          软件开发人员常常在对用户要求只有模糊的了解，就匆忙着手编写代码。
         </span>
        </li>
        <li>
         <span style="font-size:14px">
          软件开发人员与用户之间的信息交流往往很不充分。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        软件产品的质量靠不住。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          没有把各种软件测试应用到软件开发过程中。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        软件通常没有适当的文档资料。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          缺少文档。
         </span>
        </li>
        <li>
         <span style="font-size:14px">
          文档不合格。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        软件常常是不可维护。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          软件错误难以改正。
         </span>
        </li>
        <li>
         <span style="font-size:14px">
          难以适应新的硬件环境。
         </span>
        </li>
        <li>
         <span style="font-size:14px">
          难以扩展。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        软件成本在计算机系统总成本所占的比例逐年上升。
       </span>
      </li>
     </ol>
    </div>
    <h2>
     <span style="font-size:14px">
      软件危机的原因
     </span>
    </h2>
    <div>
     <ol>
      <li>
       <span style="font-size:14px">
        与软件本身的特点有关。
       </span>
       <ul>
        <li>
         <span style="font-size:14px">
          软件不同于硬件，它是计算机系统中的逻辑部件，而不是物理部件，软件缺乏“可见性”。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="font-size:14px">
        和软件开发与维护的方法不正确有关。
       </span>
      </li>
     </ol>
    </div>
