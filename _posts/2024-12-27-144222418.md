    <h2 id="f0Is4">
     1.
     <strong>
      库的操作
     </strong>
    </h2>
    <h3 id="X2HqK">
     （一）创建数据库
    </h3>
    <p id="u61af7b4a">
     create database [if not exists] + 库名 + [create_specification] [create_specification] ...;
    </p>
    <p id="u688af75d">
    </p>
    <p id="u3c872a53">
     注意：
    </p>
    <p id="u2540d248">
     []里的内容是可以选择的，既可以写，可以不写
    </p>
    <p id="ued7631b6">
     if not exists : 如果库名不存在，则创建一个数据库，如果存在，不做处理
    </p>
    <p id="ua202e453">
     create_specification ： 可以是 charset=字符集 或者是 collate=校验规则
    </p>
    <p id="u4c3032f5">
    </p>
    <h3 id="uMbL3">
     （二）创建数据库实例

    <h3 id="lLplH">
     <strong>
      （三）字符集和校验规则
     </strong>
    </h3>
    <p id="u56a06492">
     创建数据库的时候，会有两个编码集：
    </p>
    <ol>
     <li id="u27a9c8a1">
      数据库编码集 ---- 数据库未来存储数据
     </li>
     <li id="u1e5f1c4b">
      数据库校验集 ---- 支持数据库，进行字段比较使用的编码，本质也是一种读取数据库数据采用的编码格式
     </li>
    </ol>
    <p id="ua8608fed">
    </p>
    <p id="u2c512d59">
     <strong>
      因此，数据库无论对数据做任何操作，都必须保证 存储数据库和读取数据库 的编码是一致的
     </strong>
    </p>
    <p id="u8db4f4a8">
    </p>
    <p id="u5bb7906c">
    </p>
    <ul>
     <li id="uc4e3c8d0">
      <strong>
       查看系统默认字符集以及校验规则
      </strong>
     </li>
    </ul>
    <p id="u8bc84a29">
     show variables like 'character_set_database';
    </p>
    <p id="u868f2c9c">
     show variables like 'collation_database';
    </p>
    <p id="u74cdeefb">
    </p>
    <p class="img-center">
     <img alt="" height="194" id="ufe265584" src="https://i-blog.csdnimg.cn/img_convert/69e11bb029f882bceee8eb4f194daba9.png" width="685"/>

    <h3 id="vyGeH">
     <strong>
      （四）操纵数据库
     </strong>
    </h3>
    <ul>
     <li id="ufa6bc6ac">
      <strong>
       查看数据库 （可以知道哪些数据库被创建）
      </strong>
     </li>
    </ul>
    <p id="u40d9033a">
     show databases;
    </p>
    <p id="u353d1efd">
    </p>
    <p class="img-center">
     <img alt="" height="317" id="uec5b876d" src="https://i-blog.csdnimg.cn/img_convert/7903cf9183e85c6dd3fce6eba84bff48.png" width="436"/>
    </p>
    <p id="u2630bfcf">
    </p>
    <ul>
     <li id="u7def36f4">
      <strong>
       显示创建语句 (可以知道创建该数据库的过程)
      </strong>
     </li>
    </ul>
    <p id="uddd0e137">
     show create database + 数据库名；
    </p>
    <p id="uc6c80b42">
    </p>
    <p class="img-center">
     <img alt="" height="184" id="u55fe4aa3" src="https://i-blog.csdnimg.cn/img_convert/8b3d9916acd4a08f66ca9da79ef94fe4.png" width="1015"/>
    </p>
    <p id="u49d7c57d">
    </p>
    <p id="u9c3b0d58">
     注意：
    </p>
    <ol>
     <li id="u154b08a0">
      MySQL 建议我们关键字使用大写，但是不是必须的
     </li>
     <li id="u2493c059">
      数据库名字的反引号``,是为了防止使用的数据库名刚好是关键字
     </li>
     <li id="udb119bdc">
      /*!40100 default.... */ 这个不是注释，表示当前mysql版本大于4.01版本，就执行这句话
     </li>
    </ol>
    <p id="u6c115b41">
    </p>
    <h3 id="ho0Dw">
     （五）修改数据库
    </h3>
    <p id="ua8e7c3b9">
     alter database + 数据库名 + [alter_spacification] [alter_spacification]...;
    </p>
    <p id="u295d38b3">
    </p>
    <p class="img-center">
     <img alt="" height="263" id="u2a4f1ea7" src="https://i-blog.csdnimg.cn/img_convert/28872107c308cfe822a0233e9c4a39fb.png" width="1004"/>
    </p>
    <p id="u828750ba">
     注意：
    </p>
    <p id="u04b1e88f">
     这里的修改数据库信息，一般修改的是它对应的字符集和校验规则
    </p>
    <p id="u13401b82">
    </p>
    <h3 id="WsObQ">
     <strong>
      （六）数据库删除
     </strong>
    </h3>
    <p id="u6386a826">
     drop database + 数据库名;
    </p>
    <p id="u72c3c18e">
    </p>
    <p class="img-center">
     <img alt="" height="359" id="u73f42de8" src="https://i-blog.csdnimg.cn/img_convert/03d536b3db4a30464ab4e2674531e119.png" width="582"/>
    </p>
    <p id="u4ac82514">
    </p>
    <p id="u271380f6">
     注意：
    </p>
    <p id="u5487f872">
     删除之后，数据库内部看不到对应的数据库，对应的数据库文件夹被删除，级联删除，里面的数据表全部被删
    </p>
    <p id="ueb848fc9">
    </p>
    <p id="u8d2834aa">
     <strong>
      所以，不要随意删除数据库
     </strong>
    </p>
    <p id="u8bdd1ab7">
    </p>

    <h2 id="eWl8j">
     2.
     <strong>
      备份和恢复
     </strong>
    </h2>
    <h3 id="Kcpaf">
     <strong>
      （一）备份
     </strong>
    </h3>
    <p id="u755255fa">
     mysqldump -P + 端口号 + -u + 主机名 + -p + 密码 + -B + 数据库名 &gt; 新创建的文件名
    </p>
    <p id="u96f99f44">
    </p>
    <p class="img-center">
     <img alt="" height="634" id="u892f8d6b" src="https://i-blog.csdnimg.cn/img_convert/6a03471edbce36071464377114f996ee.png" width="1025"/>
    </p>
    <p id="u0239fd5c">
    </p>
    <p id="u00c916df">
     注意：
    </p>
    <p id="u152cb1cc">
     test.sql 文件里的内容，其实把我们
     <strong>
      整个创建数据库，建表，导入数据的语句都装载这个文件中
     </strong>
    </p>
    <p id="uc467b37c">
    </p>
    <h3 id="jNEVJ">
     <strong>
      （二）还原
     </strong>
    </h3>
    <ul>
     <li id="ucbe49397">
      source + 还原文件的路径名
     </li>
    </ul>
    <p id="uc53977ce">
    </p>
    <p class="img-center">
     <img alt="" height="339" id="ubcc07ef1" src="https://i-blog.csdnimg.cn/img_convert/67fcd981cd62bd79c9a387c253b2300e.png" width="548"/>
    </p>
    <p id="u74e0ff1b">
    </p>
    <p class="img-center">
     <img alt="" height="304" id="ue2f0c4a4" src="https://i-blog.csdnimg.cn/img_convert/652e9721dc31afa3c3191e25de755183.png" width="360"/>
    </p>
    <p id="uf1295971">
    </p>
    <p id="u93f8925a">
    </p>
    <p id="u6b37c773">
     <strong>
      注意：
     </strong>
    </p>
    <p id="u868aa5fc">
     如果备份的不是整个数据库，而是其中的一张表（即备份文件的时候不带 -B 选项），还原过程就不一样了
    </p>
    <p id="u45a41ff2">
    </p>
    <ol>
     <li id="uebcf2b4e">
      mysqldump -u root -p 数据库名 表名1 表名2 &gt; 指定路径下的文件
     </li>
    </ol>
    <p id="u5f2edc20">
     同时备份多个数据库中的表
    </p>
    <p id="ue7404c71">
     在恢复数据库时，需要先创建空数据库，然后使用数据库，再使用source来还原（该文件内并没有创建数据库的语句）
    </p>
    <p id="u92d36b9c">
    </p>
    <ol>
     <li id="uf3fd7da2">
      mysqldump -u root -p -B 数据库名1 数据库名2 &gt; 指定路径下的文件
     </li>
    </ol>
    <p id="u963bb4d0">
     同时备份多个数据库
    </p>

    <h2 id="vdgus">
     3.
     <strong>
      表的操作
     </strong>
    </h2>
    <h3 id="JfF0o">
     <strong>
      （一）创建表
     </strong>
    </h3>
    <p id="u4d59a8db">
     create table + 表名 (
    </p>
    <p id="u800c2bf4">
     field1 datatype,
    </p>
    <p id="udf665517">
     field2 datatype,
    </p>
    <p id="u586fdad5">
     field3 datatype
    </p>
    <p id="ufd9cee2a">
     ) character set 字符集 collate 校验规则 engine 存储引擎;
    </p>
    <p id="ue0c5809a">
    </p>
    <p id="u6562ff0b">
     注意：
    </p>
    <p id="u77d23c30">
     field 表示列名
    </p>
    <p id="ufbdfdccd">
     datatype 表示列的类型
    </p>
    <p id="uc2260716">
     character set 字符集，如果没有指定字符集，则以所在数据库的字符集为准
    </p>
    <p id="u94f727eb">
     collate 校验规则，如果没有指定校验规则，则以所在数据库的校验规则为准
    </p>
    <p id="u0c72e0d4">
    </p>

     <strong>
      （二）创建表案例
