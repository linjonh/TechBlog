    <p>
     我这里以日期为例，yyyy-mm-dd。
     <br/>
     首先定义实体
    </p>
    <pre><code>import java.util.Date;

    private Date birthday;
</code></pre>
    <p>
     这里使用util的Date。
     <br/>
     前端传String 的yyyy-mm-dd，需要用js转码，很复杂。
     <br/>
     所以使用ssm自带注解：@DateTimeFormat(pattern = “yyyy-MM-dd”)
     <br/>
     在传入控制器时，会自动把字符串转换成Date。
    </p>
    <p>
     接下来的问题是怎么传入mysql。
     <br/>
     我这里mysql的格式是date，关于date，datetime，timestamp的差别我就不讲了，用util的Date不能直接传入mysql，需要将date转换成sql.Date.
     <br/>
     sql.Date对应的是mysql的date。
     <br/>
     但是我们实体不能直接用sql的Date，这样会使注解失效，所以我们做一个转换。
     <br/>
     sql.Date是util.Date的子类，转换方法为：
    </p>
    <pre><code> java.sql.Date sql_birthday = new  java.sql.Date( new java.util.Date().getTime());
</code></pre>
    <p>
     所以我们只需把实体的date对象执行gettime方法构造sqldate传入mybatis进行增删改查就可以了。
    </p>
    <h4>
     <a id="yyyymmdd_HHmmss_24">
     </a>
     如果是yyyy-mm-dd HH:mm:ss这种时间格式呢
    </h4>
    <p>
     仍然使用注解 @DateTimeFormat(pattern = “yyyy-MM-dd HH:mm:ss”)
     <br/>
     这种时间格式在mysql里是datetime，sql.Date不支持这种格式，util.date又不能直接传入数据库，
     <br/>
     所以使用Timestamp来传入数据库。
    </p>
    <pre><code>Timestamp time = new Timestamp(new java.util.Date().getTime());
</code></pre>
