    <blockquote>
     <p>
      前文：在开发微信小程序后台的时候，我们需要使用域名进行跳转访问，按照传统的思路我们的域名只能填写一个公网IP，然而多数情况下我们并没有公网IP
     </p>
    </blockquote>
    <p>
     方法非常简单，有两种思路：
    </p>
    <ol>
     <li>
      第一种，将域名定向到局域网IP
      <br/>
      比如我们在阿里云云解析的配置面板中，讲A记录的IP地址改为127.0.0.1 或者是 192.168.xx.xx 指向我们本地测试机的局域网地址，然后我们在电脑上进行本地测试的时候就非常快捷了，当然，即便你是用真机测试，只要你的手机和电脑在同一个局域网也可以OK的
     </li>
     <li>
      <p>
       第二种，修改电脑的host文件
       <br/>
       host文件就相当于一个本地的dns，一个本地化的域名解析，windows系统的host文件位于
      </p>
      <pre class="prettyprint"><code class="hljs tex">C:<span class="hljs-command">\Windows</span><span class="hljs-command">\System</span>32<span class="hljs-command">\drivers</span><span class="hljs-command">\etc</span><span class="hljs-command">\host</span></code></pre>
      <p>
       用文本编辑器打开：
      </p>
     </li>
    </ol>
    <pre class="prettyprint"><code class="hljs vala"><span class="hljs-preprocessor"># Copyright (c) 1993-2009 Microsoft Corp.</span>
<span class="hljs-preprocessor">#</span>
<span class="hljs-preprocessor"># This is a sample HOSTS file used by Microsoft TCP/IP for Windows.</span>
<span class="hljs-preprocessor">#</span>
<span class="hljs-preprocessor"># This file contains the mappings of IP addresses to host names. Each</span>
<span class="hljs-preprocessor"># entry should be kept on an individual line. The IP address should</span>
<span class="hljs-preprocessor"># be placed in the first column followed by the corresponding host name.</span>
<span class="hljs-preprocessor"># The IP address and the host name should be separated by at least one</span>
<span class="hljs-preprocessor"># space.</span>
<span class="hljs-preprocessor">#</span>
<span class="hljs-preprocessor"># Additionally, comments (such as these) may be inserted on individual</span>
<span class="hljs-preprocessor"># lines or following the machine name denoted by a '#' symbol.</span>
<span class="hljs-preprocessor">#</span>
<span class="hljs-preprocessor"># For example:</span>
<span class="hljs-preprocessor">#</span>
<span class="hljs-preprocessor">#      102.54.94.97     rhino.acme.com          # source server</span>
<span class="hljs-preprocessor">#       38.25.63.10     x.acme.com              # x client host</span>

<span class="hljs-preprocessor"># localhost name resolution is handled within DNS itself.</span>
<span class="hljs-preprocessor">#   127.0.0.1       localhost</span>

<span class="hljs-number">127.0</span><span class="hljs-number">.0</span><span class="hljs-number">.1</span> xxx.baofeidyz.com</code></pre>
    <p>
     带 # 号的全是注释，使用方法及其简单，注意最后一行
    </p>
    <pre class="prettyprint"><code class="hljs avrasm"><span class="hljs-number">127.0</span><span class="hljs-number">.0</span><span class="hljs-number">.1</span> xxx<span class="hljs-preprocessor">.baofeidyz</span><span class="hljs-preprocessor">.com</span></code></pre>
    <p>
     空格前面是ip地址，后面是域名，比如我这条配置中ip地址是127.0.0.1，域名是xxx.baofeidyz.com，保存以后，你在本地电脑访问该域名的时候就会自动被解析为127.0.0.1了，当然这种方法有个弊端就是手机修改host比较麻烦，可能需要你root手机，或者是设置http代理，如果第一种方法你可以接受的话，就不要考虑第二种了，尤其是设置Http代理是让手机访问这种麻烦的方法。
    </p>
    <p>
     btw 如果你想看看自己的host文件有没有修改成功，可以试试 ping xxx.baofeidyz.com 看看解析出来的ip地址是不是你设置的127.0.0.1
