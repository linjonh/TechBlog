    <h1 class="title-article" id="articleContentId">
     简单的问题有时也要技巧：“尝试打开文件夹时出现“Access is Denied”（拒绝访问）错误信息”
    </h1>
   </div>

 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     简单的问题有时也要技巧，谷歌、百度都搜索了没有，360上也找了，都不对题。想起微软了，果然解决了。微软！想说爱你还真不容易。
    </p>
    <p>
    </p>
    <p>
     摘自微软技术支持网站：
     <a href="http://support.microsoft.com/kb/810881/zh-cn" rel="nofollow">
      http://support.microsoft.com/kb/810881/zh-cn
     </a>
    </p>
    <p>
    </p>
    <p>
     症状：
    </p>
    <p>
     当您尝试在 Microsoft Windows XP 中打开文件夹时，可能会收到以下错误信息，其中
     <var>
      Folder
     </var>
     是您无法打开的文件夹的名称：
    </p>
    <div class="kb_errormsgbody">
     <div class="errormsg">
      <var>
       Folder
      </var>
      is not accessible.Access is denied.
     </div>
     <div class="errormsg">
      原因：
     </div>
     <div class="errormsg">
      如果您无法打开的文件夹是通过以下方式在 NTFS 文件系统上创建的，则可能会出现此问题：先使用以前安装的 Windows，然后再安装 Windows XP。虽然您输入了正确的用户名和密码，但是仍可能会出现此问题。出现此问题的原因是用户的安全 ID 已更改。虽然您使用相同的用户名和密码，但是您的安全 ID 不再与无法打开的文件夹的所有者的安全 ID 相符。
      <br/>
      <br/>
      例如，虽然您使用相同的用户名和密码，但是在完成下列步骤后您可能无权再打开该文件夹：
     </div>
     <div class="errormsg">
      <ol>
       <li>
        安装 Windows XP Professional 之前，将“我的文档”文件夹的实际位置或目标位置更改到其他卷。
       </li>
       <li>
        格式化主要分区。
       </li>
       <li>
        安装 Windows XP Professional。
       </li>
      </ol>
     </div>
     <p class="kb_errorcontent">
      解决方案：
     </p>
     <p class="kb_errorcontent">
      要解决此问题，必须关闭“简单文件共享”，然后获取文件夹的所有权：
     </p>
     <div class="kb_errorcontent">
      <ol>
       <li>
        关闭“简单文件共享”：
        <ol>
         <li>
          单击“开始”，然后单击“我的电脑”。
         </li>
         <li>
          在“工具”菜单上，单击“文件夹选项”，然后单击“查看”选项卡。
         </li>
         <li>
          在“高级设置”下，单击以清除“使用简单文件共享 (推荐)”复选框，然后单击“确定”。
         </li>
        </ol>
       </li>
       <li>
        右键单击您希望获得其所有权的文件夹，然后单击“属性”。
       </li>
       <li>
        单击“安全”选项卡，然后单击“安全”消息框（如果出现）中的“确定”。
       </li>
       <li>
        单击“高级”，然后单击“所有者”选项卡。
       </li>
       <li>
        在“名称”列表中，单击您的用户名，如果您是作为管理员登录的，请单击“Administrator”，或者单击“Administrators”组。
        <br/>
        <br/>
        如果您希望获得该文件夹内容的所有权，请单击以选中“替换子容器及对象的所有者”复选框。
       </li>
       <li>
        单击“确定”。
        <br/>
        <br/>
        您可能会收到以下错误信息，其中
        <var>
         Folder
        </var>
        是您想获得其所有权的文件夹的名称：
        <div class="kb_errormsgbody">
         <div class="kb_errorcontent">
          <div class="errormsg">
           You do not have permission to read the contents of directory
           <var>
            Folder
           </var>
           .Do you want to replace the directory permissions with permissions granting you Full Control?All permissions will be replaced if you press Yes.
          </div>
         </div>
        </div>
       </li>
       <li>
        单击“是”。
       </li>
       <li>
        单击“确定”，然后对该文件夹及其内容重新应用您希望的权限及安全设置。
       </li>
      </ol>
     </div>
