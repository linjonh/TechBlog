     选择填空
    </h3>
    <ol>
     <li>
      操作系统的两个基本特征是：
      <strong>
       并发性和共享性
      </strong>
     </li>
     <li>
      进程的基本特征是：
      <strong>
       动态、并发
      </strong>
     </li>
     <li>
      计算机系统由
      <strong>
       硬件系统
      </strong>
      和
      <strong>
       软件系统
      </strong>
      两大部分组成，由
      <strong>
       操作系统
      </strong>
      对它们进行管理，以提高系统资源的利用率。
     </li>
     <li>
      操作系统的四大资源管理功能：
      <strong>
       处理器管理，存储器管理，设备管理和文件管理
      </strong>
      。
     </li>
     <li>
      <strong>
       多道程序设计
      </strong>
      可以将一个以上的作业放入内存，并且同时处于运行状态。
     </li>
     <li>
      时间片概念一般用于
      <strong>
       分时操作系统
      </strong>
      。
     </li>
     <li>
      在分时系统中，当时间片一定时，
      <strong>
       用户数越多
      </strong>
      ，响应时间越长。
     </li>
     <li>
      <strong>
       分时操作系统
      </strong>
      允许在一台主机上同时连接多台终端，多个用户可以通过个字的终端同时交互地使用计算机。
     </li>
     <li>
      操作系统是对
      <strong>
       计算机资源
      </strong>
      进行管理的软件。
     </li>
     <li>
      <strong>
       多道程序设计
      </strong>
      是指在一台处理器上并发运行多个程序。
     </li>
     <li>
      进程存在唯一标识：
      <strong>
       PCB(进程控制块)
      </strong>
      。
     </li>
     <li>
      在PCB中，用于进程调度的是：
      <strong>
       说明信息
      </strong>
      。
     </li>

      <strong>

      </strong>
      。

      <strong>
       不可分割性
      </strong>
      。
     </li>

      在多道程序中负责从就绪队列中选中一个进程占用CPU的是
      <strong>
       进程调度
      </strong>
      。

      <strong>
       可能需要同步或互斥
      </strong>
      。
     </li>

      在操作系统中，
      <strong>
       PV操作
      </strong>
      是一种低级进程通信原语。
     </li>
     <li>
      根据进程的紧迫性进行进程调度，应采用
      <strong>
       优先权调度算法
      </strong>
      。
     </li>
     <li>
      进程的切换是由
      <strong>
       进程状态的变换
      </strong>
      引起的。
     </li>

      进程通常由
      <strong>
       程序、数据集合
      </strong>
      和
      <strong>

      </strong>
      三部分组成。

     <li>

      <strong>
       动态、独立、异步、并发
      </strong>

     </li>

      <strong>
       地址变换
      </strong>
      。
     </li>

      <strong>
       容量递增
      </strong>
      次序排列。
     </li>

      <strong>

      </strong>
      。

      <strong>

      <strong>
       系统
      </strong>
      完成的。

      <strong>

      引起的。

      </strong>
      理论。

      <strong>

      </strong>

      <strong>
       动态地址变换
      </strong>
      。
     </li>
     <li>

      <strong>

      <strong>

      中。

      。

      </strong>
      。

      <strong>

      </strong>

      <strong>

    <h3>
     <a id="_62">
     </a>
     简答题
    </h3>
    <ol>
     <li>
      <p>
       从资源管理观点看，操作系统具有哪些功能？
       <br/>
       处理器管理，对处理器进行分配，并对其运行进行有效地控制和管理；
       <br/>
       存储器管理，提高存储器的利用率，并能从逻辑上扩充内存；
       <br/>
       设备管理，完成用户提出的I/O请求，为用户分配I/O设备，提高CPU与I/O设备的利用率，提高I/O设备运行速度，方便用户使用I/O设备。
       <br/>
       文件管理，对用户文件和系统文件进行管理。
      </p>
     </li>
     <li>
      <p>
       操作系统具有哪些基本特征？
       <br/>
       并发性，是指两个或多个事件在同一时间间隔内发生；
       <br/>
       共享性，是指系统中的资源可供多个并发执行的进程使用；
       <br/>
       虚拟性，是指通过某种技术把一个物理实体变成若干个逻辑实体；
       <br/>
       异步性，是指多个并发执行的进程，由于资源的限制，进程的执行不是“一气呵成”的，而是“走走停停”的。
      </p>
     </li>
     <li>
      <p>
       进程的基本状态以及它们之间的切换：就绪、运行和等待状态。
       <br/>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/40b0bab6cccba55a7c63d64ab2b0d62e.jpeg" width="60%"/>
      </p>
     </li>
     <li>
      <p>
       死锁产生的四个必要条件：
       <br/>
       互斥条件，进程对分配到的资源进行排他性、独占性使用；
       <br/>
       请求和保持条件，进程已经拥有并保持了至少一个资源；
       <br/>
       不可剥夺条件，进程所占有的资源在结束之前不能被剥夺，只能在运行结束后由自己释放；
       <br/>
       环路等待条件，在发生死锁时，必然存在一个“进程——资源”的环形链。
      </p>
     </li>
     <li>
      <p>
       产生进程死锁的原因是什么？如何解除死锁？
       <br/>
       原因：竞争资源；进程推进顺序非法。
       <br/>
       死锁的解除：剥夺资源法，从其他进程剥夺足够的资源给死锁进程；撤销进程法，系统采用强制手段将死锁进程撤销。
      </p>
     </li>
     <li>
      <p>
       概念区分——程序与进程：程序是计算机指令的集合，以文件的形式存储在磁盘上；进程是一个程序在一个数据集合上的一次运行过程，是资源分配和调度的基本单位。
      </p>
     </li>
     <li>
      <p>
       概念区分——并发与并行：并发是指在一个时间段内，有多个进程同时运行；并行是指在某一时刻有多个进程同时运行。
      </p>
     </li>
     <li>
      <p>
       什么是虚拟设备？请说明SPOOLing系统如何实现虚拟设备？
       <br/>
       当系统只有一台输入设备或一台输出设备的情况下，可允许两个以上的作业并行执行，并且每个作业都感觉到获得了供自己独占使用的输入设备和输出设备。
       <br/>
       SPOOLing技术借助磁盘和通道实现了输入/输出过程的共享。当用户提出输入/输出请求时，系统及时响应，此时用户会认为已独占输入/输出设备；但事实上，有多道作业同时进入该过程，并分别占用各个阶段。
      </p>
     </li>
     <li>
      <p>
       设备分配时为什么要考虑安全性以及与设备的无关性？考虑安全性是为了提高系统的适应性和均衡性，避免死锁的产生；设备无关性使得用户的应用程序独立于实际的物理设备，不仅方便了用户，而且设备分配比较灵活，也便于实现I/O重定向。
      </p>
     </li>
     <li>
      <p>
       什么是逻辑设备？什么是物理设备？如何实现从逻辑设备到物理设备的转换？
       <br/>
       用户程序中使用的设备是逻辑设备，计算机系统中实际配置的设备是物理设备。
       <br/>
       在OS中，通常设置了反映逻辑设备和物理设备对应关系的数据结构，其中存放了逻辑设备名、物理设备名及设备驱动程序入口地址等信息，通过该数据结构即可实现逻辑设备到物理设备的转换。
      </p>
     </li>
     <li>
      <p>
       什么是缓冲？为什么引入缓冲？
       <br/>
       缓冲技术是用来在两种不同速度的设备之间传输信息时平滑传输过程的常用手段。
       <br/>
       引入缓冲的主要原因是：
      </p>
      <ul>
       <li class="task-list-item">
        <input class="task-list-item-checkbox" disabled="disabled" type="checkbox"/>
        缓和CPU和I/O设备速度不匹配的矛盾；
       </li>
       <li class="task-list-item">
        <input class="task-list-item-checkbox" disabled="disabled" type="checkbox"/>
        减少对CPU的中断频率，放宽对中断响应时间的限制；
       </li>
       <li class="task-list-item">
        <input class="task-list-item-checkbox" disabled="disabled" type="checkbox"/>
        提高CPU和I/O设备之间的并行性。
       </li>
      </ul>
     </li>
     <li>
      <p>
       名词解释——文件：文件时具有文件名的一组相关信息的集合。
      </p>
     </li>
     <li>
      <p>
       名词解释——文件系统：文件系统是指操作系统中负责管理和存取文件的程序模块。
      </p>
     </li>
     <li>
      <p>
       什么是文件的逻辑结构？文件有哪几种逻辑结构？
       <br/>
       文件的逻辑结构是指从用户观点出发所看到的文件组织形式。可分为两种，记录式文件和流式文件。
      </p>
     </li>
     <li>
      <p>
       什么是文件的物理机构？文件有哪几种物理结构？
       <br/>
       文件的物理结构是指文件在外存上的存储组织形式。可分为三种，顺序结构、链接结构和索引结构。
      </p>
     </li>
     <li>
