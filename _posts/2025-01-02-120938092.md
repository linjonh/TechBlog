    <h3>
     <a id="JUnitUT_60">
     </a>
     JUnit编写UT规范
    </h3>
    <p>
     编写单元测试建议遵循以下规范
    </p>
    <blockquote>
     <ul>
      <li>
       <p>
        类名：命名由
        <code>
         被测试类+Test
        </code>
        组成，例如
        <code>
         CalculateTest
        </code>
       </p>
      </li>
      <li>
       <p>
        方法名：命名由
        <code>
         test+被测方法名
        </code>
        组成，且方法名首字母大写，或者直接使用方法名，例如
        <code>
         testAdd
        </code>
        或
        <code>
         add
        </code>
       </p>
      </li>
      <li>
       <p>
        测试类定义为public，例如
        <code>
         public class CalculateTest
        </code>
       </p>
      </li>
      <li>
       <p>
        方法定义为public，且无返回值，参数列表为空，例如
        <code>
         public void testAdd()
        </code>
       </p>
      </li>
     </ul>
    </blockquote>

     <a id="JUnit4_71">
     </a>
     JUnit4注解
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        注解
       </th>
       <th>
        描述
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        @Test
       </td>
       <td>
        标识测试方法，该方法须为
        <code>
         public void
        </code>
       </td>
      </tr>
      <tr>
       <td>
        @BeforeClass
       </td>
       <td>
        ① 在所有测试方法之前执行，运行一次
        <br/>
        ② 添加该注解的测试方法必须定义为
        <code>
         public static void
        </code>
        <br/>
        ③ 用法：当需要在测试几个方法前执行同一个命令时，可以使用此注解，例如建立数据库连接
       </td>
      </tr>
      <tr>
       <td>
        @AfterClass
       </td>
       <td>
        与@BeforeClass对立，在所有测试方法执行后运行一次，用于测试后的清理工作
       </td>
      </tr>
      <tr>
       <td>
        @Before
       </td>
       <td>
        ① 每个测试用例执行之前都运行一次
        <br/>
        ② 用于测试前的准备工作，例如初始化类等
       </td>
      </tr>
      <tr>
       <td>
        @After
       </td>
       <td>
        ① 每次测试之后都运行一次
        <br/>
        ② 用于测试后的清理工作，例如释放内存等
       </td>
      </tr>
      <tr>
       <td>
        @Ignore
       </td>
       <td>
        忽略某个测试方法或测试类
       </td>
      </tr>
      <tr>
       <td>
        @RunWith
       </td>
       <td>
        此注解放在测试类名之前，用于指明该类所使用的的测试运行器
       </td>
      </tr>
     </tbody>
    </table>

    <h3>
     <a id="JUnit_164">
     </a>
     JUnit参数化测试
    </h3>
    <ul>
     <li>
      参数化测试（Parameterized Test）：使用不同的参数值来进行测试的测试方法
     </li>
     <li>
      通过
      <code>
       @RunWith(Parameterized.class)
      </code>
      注解来说明对某个测试类进行参数化测试
     </li>
     <li>
      测试类中必须包含一个
      <code>
       public static
      </code>
      静态方法，并使用
      <code>
       @Parameters
      </code>
      注解，且其返回类型必须为一个对象的集合
     </li>
    </ul>
    <p>
     示例：
    </p>
    <p>
     1） 被测试类，compute方法计算fibonacci数列第n个数值
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">Fibonacci</span> <span class="token punctuation">{<!-- --></span>
