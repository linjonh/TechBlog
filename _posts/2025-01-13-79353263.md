    <h3 class="tutheader" style="border-width:1px 0px 0px;border-top-style:solid;border-top-color:rgb(212,212,212);margin:0px;padding:5px 0px 0px;font-size:1.8em;line-height:1.8em;clear:both;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     什么是NoSQL?
    </h3>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     NoSQL，指的是非关系型的数据库。NoSQL有时也称作Not Only SQL的缩写，是对不同于传统的关系型数据库的数据库管理系统的统称。
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     NoSQL用于超大规模数据的存储。（例如谷歌或Facebook每天为他们的用户收集万亿比特的数据）。这些类型的数据存储不需要固定的模式，无需多余操作就可以横向扩展。
    </p>
    <h3 class="tutheader" style="border-width:1px 0px 0px;border-top-style:solid;border-top-color:rgb(212,212,212);margin:0px;padding:5px 0px 0px;font-size:1.8em;line-height:1.8em;clear:both;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     RDBMS vs NoSQL
    </h3>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     <span style="border:0px;margin:0px;padding:0px;">
      RDBMS
     </span>
     <br/>
     - 高度组织化结构化数据
     <br/>
     - 结构化查询语言（SQL） (SQL)
     <br/>
     - 数据和关系都存储在单独的表中。
     <br/>
     - 数据操纵语言，数据定义语言
     <br/>
     - 严格的一致性
     <br/>
     - 基础事务
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     <span style="border:0px;margin:0px;padding:0px;">
      NoSQL
     </span>
     <br/>
     - 代表着不仅仅是SQL
     <br/>
     - 没有声明性查询语言
     <br/>
     - 没有预定义的模式
     <br/>
     -键 - 值对存储，列存储，文档存储，图形数据库
     <br/>
     - 最终一致性，而非ACID属性
     <br/>
     - 非结构化和不可预知的数据
     <br/>
     - CAP定理
     <br/>
     - 高性能，高可用性和可伸缩性
    </p>
    <h3 class="tutheader" style="border-width:1px 0px 0px;border-top-style:solid;border-top-color:rgb(212,212,212);margin:0px;padding:5px 0px 0px;font-size:1.8em;line-height:1.8em;clear:both;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     NoSQL的优点/缺点
    </h3>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     优点:
    </p>
    <ul style="border:0px;margin:1em 0px;padding:0px;line-height:16.8px;list-style:none;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 高可扩展性
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 分布式计算
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 低成本
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 架构的灵活性，半结构化数据
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 没有复杂的关系
     </li>
    </ul>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     缺点:
    </p>
    <ul style="border:0px;margin:1em 0px;padding:0px;line-height:16.8px;list-style:none;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 没有标准化
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 有限的查询功能（到目前为止）
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      - 最终一致是不直观的程序
     </li>
    </ul>

    </h3>
    <p>
     MongoDB、Redis、HBase数据库的区别
    </p>
    <h6>
     <span style="font-family:SimSun;">
      MongoDB做高性能数据库，Redis做缓存，HBase做大数据分析。MongoDB还无法取代关系型数据库。
      <br/>
     </span>
     <span style="font-family:SimSun;">
      传统关系型数据库面对数据规模、数据模型复杂时的不足，导致了NoSQL的快速发展，后者易扩展，性能高，支持灵活的数据模型。
      <br/>
     </span>
     <span style="font-family:SimSun;">
      MongoDB是高性能、无模式的文档型数据库，支持二级索引，非常适合文档化格式的存储及查询。MongoDB的官方定位是通用数据库，确实和MySQL有些像，现在也很流行，但它还是有事务、join等短板，在事务、复杂查询应用下无法取代关系型数据库。
      <br/>
     </span>
     <span style="font-family:SimSun;">
      Redis是内存型Key/Value系统，读写性能非常好，支持操作原子性，很适合用来做高速缓存。
      <br/>
     </span>
     <span style="font-family:SimSun;">
      HBase存储容量大，一个表可以容纳上亿行、上百万列，可应对超大数据量要求扩展简单的需求。Hadoop的无缝集成，让HBase的数据可靠性和海量数据分析性能（MapReduce）值得期待。
     </span>
    </h6>
    <div>
     <br/>
    </div>
    <p>
     <span style="font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:#333333;">
     </span>
    </p>
    <h3 style="border:0px;margin:2px 0px;padding:0px;font-size:1.8em;line-height:1.8em;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     什么是MongoDB ?
    </h3>
    <p>
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     MongoDB 是由C++语言编写的，是一个基于分布式文件存储的开源数据库系统。
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     在高负载的情况下，添加更多的节点，可以保证服务器性能。
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     MongoDB 旨在为WEB应用提供可扩展的高性能数据存储解决方案。
    </p>
    <p style="border:0px;margin-top:0px;margin-bottom:0px;padding-top:0px;padding-bottom:0px;line-height:2em;font-size:13px;font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:rgb(51,51,51);">
     MongoDB 将数据存储为一个文档，数据结构由键值(key=&gt;value)对组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组。
    </p>
    <br/>
    <p>
     <span style="font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;color:#333333;">
     </span>
    </p>
    <h3 style="border:0px;margin:2px 0px;padding:0px;font-size:1.8em;line-height:1.8em;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     主要特点
    </h3>
    <ul style="border:0px;margin:1em 0px;padding:0px;line-height:16.8px;list-style:none;color:rgb(51,51,51);font-family:'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei', Arial, sans-serif;">
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      MongoDB 是一个面向文档存储的数据库，操作起来比较简单和容易。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      你可以在MongoDB记录中设置任何属性的索引 (如：FirstName="Sameer",Address="8 Gandhi Road")来实现更快的排序。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      你可以通过本地或者网络创建数据镜像，这使得MongoDB有更强的扩展性。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      如果负载的增加（需要更多的存储空间和更强的处理能力） ，它可以分布在计算机网络中的其他节点上这就是所谓的分片。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      Mongo支持丰富的查询表达式。查询指令使用JSON形式的标记，可轻易查询文档中内嵌的对象及数组。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      MongoDb 使用update()命令可以实现替换完成的文档（数据）或者一些指定的数据字段 。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      Mongodb中的Map/reduce主要是用来对数据进行批量处理和聚合操作。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      Map和Reduce。Map函数调用emit(key,value)遍历集合中所有的记录，将key与value传给Reduce函数进行处理。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      Map函数和Reduce函数是使用Javascript编写的，并可以通过db.runCommand或mapreduce命令来执行MapReduce操作。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      GridFS是MongoDB中的一个内置功能，可以用于存放大量小文件。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      MongoDB允许在服务端执行脚本，可以用Javascript编写某个函数，直接在服务端执行，也可以把函数的定义存储在服务端，下次直接调用即可。
     </li>
     <li style="border:0px;margin:0px 0px 1em 1em;padding:0px 0px 0px 1.5em;line-height:1.5em;font-size:13px;">
      MongoDB支持各种编程语言:RUBY，PYTHON，JAVA，C++，PHP，C#等多种语言。
     </li>
