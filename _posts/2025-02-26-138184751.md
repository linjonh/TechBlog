    <h2>
     <a id="_2">
     </a>
     前言
    </h2>
    <p>
     从2010年写下的第一行代码开始，
     <strong>
      OceanBase已经坚持了14年的持续自研
     </strong>
     ，通过持续的自研发展，逐渐成为了众多开发者喜欢的数据库。
    </p>
    <p>
     就在上周六 - 2024年4月20日，OceanBase来到了上海，北京我没赶上，上海我必须得到现场啊！非常荣幸，作为特邀嘉宾与众多的业界知名数据库大咖、数据库爱好者共同探讨
     <strong>
      一体化、多模态、多云原生
     </strong>
     等前沿数据库技术趋势。
    </p>
    <p>
     当然了，出了与会交流的
     <strong>
      产品Roadmap、产品场景
     </strong>
     等探索之外，现场的各种
     <strong>
      体验产品Demo、人才集市、游园打卡
     </strong>
     等互动体验也是众多开发者小伙伴的最爱。（我也不例外～ 😁）
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/46d84f1752368f8c39a992b5e99352a5.png#pic_center" width="600">
     </img>
    </div>
    <h2>
     <a id="_15">
     </a>
     签到台打卡
    </h2>
    <p>
     会场主办地点位于某知名酒店内，签到台附近就是一个咖啡厅，空气中弥漫着淡淡的咖啡香，与人们的谈笑风生交织在一起，更是生出了一种轻松愉快的氛围。
    </p>
    <p>
     签到台的工作人员忙碌而有序，微笑着迎接每一位来宾，耐心解答疑问，为这场盛会增添了一抹温馨的色彩。不得不说哈，在这片熙熙攘攘的人群中，每个人都在用着自己的方式感受着
     <strong>
      OceanBase
     </strong>
     的热闹与活力。或许来自不同的领域，或许有着不同的背景，但此刻，他们都成为了这个
     <strong>
      OceanBase
     </strong>
     大家庭中的一员，共同期待着即将到来的精彩瞬间。
     <br/>
     <br/>
    </p>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/7e7b18acf0e84745e5f50071c4926588.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <p>
     有一说一，进入主会场的步行梯，还是非常诙谐、幽默、俏皮的，充满了作为技术人的自我调侃。
    </p>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/84829d7d1a8d3f8100b007a39b7274c0.png#pic_center" width="600">
     </img>
    </div>
    <h2>
     <a id="_35">
     </a>
     主论坛打卡
    </h2>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/35baef409e8c4cf5d567d61a5e7d9fa2.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <p>
     09:30分，大会在欢腾热烈的气氛中拉开帷幕，OceanBase的创始人和技术领军人物纷纷登台，与现场观众分享了OceanBase从创立至今的辉煌历程和未来的宏伟蓝图。从2010年那第一行代码的敲击，到2022年4.0版本的惊艳发布，OceanBase历经磨砺，如今已在企业级分布式数据库领域崭露头角，成为行业翘楚。
    </p>
    <p>
     值得一提的是，OceanBase所具备的“三地五中心”城市级容灾能力，在业界堪称突破性的创新，这一优势使得OceanBase在金融、互联网等多个领域都赢得了广泛的赞誉和青睐。
     <br/>
     <br/>
    </p>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/32ecddd825a5db2001a400e4b9ce5df6.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <p>
     OceanBase CTO杨传辉更是自豪地宣布，OceanBase将继续坚定践行“关键业务负载”的一体化战略，致力于打造出PB级实时分析数据库的最强六边形战士。这意味着，无论是TP性能的提升，还是分布式、兼容性、AP存储、AP计算以及AP工具生态能力的增强，OceanBase都将不遗余力地追求卓越。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/1f29239b83d3c0be5c53803d147dccc6.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <p>
     此外，大会还探讨了分布式数据库与人工智能（AI）结合的未来趋势。嘉宾们指出，随着AI技术的飞速发展，数据库已不再是单纯的数据存储与查询工具，而是逐渐演化为与AI深度融合的新型基础设施。通过在分布式数据库中嵌入AI算法，未来有望实现更智能的数据处理和决策支持，为企业的创新发展注入新的活力。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/c50378539654bcb65807b2e11f458116.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <h2>
     <a id="43_61">
     </a>
     4.3正式发布
    </h2>
    <p>
     OceanBase发布4.3版本的消息，无疑是数据库领域的一记重磅炸弹，说它是颠覆性的，也恰如其分。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/3627b6eb45cba648915438a4d7963ebf.png#pic_center" width="600">
     </img>
    </div>
    <br/>
    <p>
     展望未来，OceanBase将继续引领分布式数据库技术的发展潮流，与AI等前沿技术深度融合，共同开启智能数据处理的新篇章。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/c560404dd07b915f3ccf4eba1d9dc693.png#pic_center" width="600"/>
    </div>
    <br/>
    <p>
     如此值得期待的OceanBase 4.3，怎能不让人心动呢？相信在未来的日子里，它定会继续书写属于自己的传奇篇章，为数据库界带来更多的惊喜与突破！
    </p>
    <h2>
     <a id="_78">
     </a>
     产品侧专场
    </h2>
    <p>
     下午的分会场，产品论坛与技术论坛是同步进行的，分身乏术、深感遗憾。与参会的朋友一起选择了最吸引我的产品技术专场。
    </p>
    <p>
     不得不提一嘴的是，OceanBase 的几位产品负责人都是技术出身，在内容的探讨与技术的讲解上，是非常的接地气的，对于技术出身的我们而言，友好的不是一点半点。
    </p>
    <p>
     除此之外，OceanBase 的多位技术精英齐聚一堂，包括产品部总经理兼首席架构师杨志丰、内核产品总监余璜，以及资深技术专家们曹晖、陈森、张易，还有公有云产品总监罗琪。他们与到场的开发者们面对面交流，共同探讨技术的奥秘。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/5f2ad51c101782e94189243b8e684a29.png#pic_center" width="600"/>
    </div>
    <br/>
    <p>
     从一体化架构演进到 4.3 版本功能，从关键特性到硬核技术再到产品 Roadmap，不仅分享了硬核技术的细节，还展望了产品的未来路线图。在热烈的讨论中，我们共同探寻新版本、新技术的魅力所在，感受OceanBase在数据库领域的创新与突破。
    </p>
    <h2>
     <a id="_95">
     </a>
     玩转新功能
    </h2>
    <p>
     在
     <strong>
      OceanBase
     </strong>
     的互动展区，开发者可以亲自上手体验 OceanBase 4.3 版本的分布式弹性扩缩容、单机模式下租户级主备 &amp; 秒级切换、实时分析、多模融合、租户克隆、多基础设施等 live demo，真实感受 OceanBase 的核心能力。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/44b8727bf8e4d1b66b501ed4e05a762a.png#pic_center" width="600"/>
    </div>
    <br/>
    <p>
     开发者在这里通过安装部署运维 OceanBase、用喜欢的语言构建自己的第一个应用程序、体验数据集成与开发，深入了解并掌握 OceanBase 的使用技巧和最佳实践。
    </p>
    <h2>
     <a id="_105">
     </a>
     探人才集市
    </h2>
    <p>
     现场精心策划的**「O 伯乐遇见千里马」人才集市**，搭建一个企业与人才精准对接的平台。
     <strong>
      新炬网络、爱可生、浩鲸科技、瑞蓝创、东方龙马、大衍科技
     </strong>
     等业界领先企业纷纷亮相，他们带着对人才的渴求和期待，希望在这里找到那些能够共同驰骋数据库领域的「千里马」。
    </p>
    <p>
     这些企业不仅开放了数据库工程师、解决方案架构师等核心技术岗位，还针对OceanBase等热门技术推出了专项技术服务工程师职位。他们希望借助这一平台，吸引更多有志于投身数据库行业、具备专业技能和创新精神的优秀人才。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/ca4a70d5f69f3cffef2c84edbfd486e0.png#pic_center" width="600"/>
    </div>
    <br/>
    <p>
     更是邀请了
     <strong>
      中国地区首位 Oracle ACE 盖国强（老盖）、资深 Oracle 数据库优化专家徐戟（白鳝）
     </strong>
     向行业后辈分享关于职场危机本质的看法，以及 DBA 面临的真正挑战与未来转型。
    </p>
    <h2>
     <a id="_120">
     </a>
     小结
    </h2>
    <p>
     本次
     <strong>
      第二届OceanBase开发者大会
     </strong>
     共吸引了1675名开发者踊跃参与，同时线上直播观看人次更是突破了10万大关。
    </p>
    <p>
     这次大会不仅是对OceanBase技术实力的集中展示，更是对广大开发者长期以来的支持与贡献的致敬。
    </p>
    <br/>
    <div align="center">
     <img src="https://i-blog.csdnimg.cn/blog_migrate/72bfe0b054d829a25465cd5241c5c564.png#pic_center" width="600"/>
    </div>
    <br/>
    <p>
     未来，我真的很期待OceanBase数据库带来更多相关的盛会，为开发者们提供更多的学习和交流机会，共同推动数据库技术的进步和创新。期待下一届OceanBase开发者大会的到来，继续享受数据库领域的盛宴！
    </p>
