    <h4>
     <a id="1_4">
     </a>
     1、介绍
    </h4>
    <p>
     fio（flexible I/O Tester） 是一款由 Jens Axboe 开发的用于测评和压力/硬件验证的自由开源的软件，适用于文件及块接口性能测试。
    </p>
    <p>
     fio常用的I/O引擎主要分为以下两种，可根据实际选择合适的I/O引擎和配置参数
    </p>
    <ul>
     <li>
      <p>
       <strong>
        同步（如psync）
       </strong>
       <br/>
       对于同步I/O而言，一次只能发出一个IO请求，等待内核完成后才返回，通常指定iodepth为1，增大thread以达到最大性能测试要求
      </p>
     </li>
     <li>
      <p>
       <strong>
        异步（如libaio）
       </strong>
       <br/>
       对于异步I/O而言，一次提交一批请求，然后等待一批的完成，减少与内核交互次数，通常指定thread为1，增大iodepth以达到最大性能测试要求
      </p>
     </li>
    </ul>
    <h3>
     <a id="_15">
     </a>
     二、安装部署
    </h3>
    <h4>
     <a id="1Linux_16">
     </a>
     1、Linux客户端
    </h4>
    <h5>
     <a id="11libaio_17">
     </a>
     1.1、安装libaio（可选）
    </h5>
    <p>
     示例使用
     <code>
      libaio
     </code>
     引擎测试，故此处需要先行安装相关软件包，如不使用该引擎进行测试，此步骤可忽略
    </p>
    <pre><code>yum install libaio-devel -y
</code></pre>
    <h5>
     <a id="12_23">
     </a>
     1.2、软件包安装
    </h5>
    <p>
     创建fio软件包安装目录
     <code>
      /opt/fio
     </code>
     ，下载fio源码包进行编译安装，配置fio环境变量
    </p>
    <pre><code>mkdir /opt/fio
wget https://github.com/axboe/fio/archive/fio-3.20.tar.gz
tar -zxvf fio-3.20.tar.gz
cd fio-fio-3.20/
./configure --prefix=/opt/fio/
make &amp;&amp; make install
echo -e "\n\n export PATH=/opt/fio/bin:$PATH" &gt;&gt; /etc/profile &amp;&amp; source /etc/profile
</code></pre>
    <h4>
     <a id="2Window_35">
     </a>
     2、Window客户端
    </h4>
    <p>
     <strong>
      下载地址
     </strong>
     ：https://bsdio.com/fio/releases/fio-3.20-x64.msi
     <br/>
     下载安装软件包即可，安装时自动配置环境变量
    </p>
    <h3>
     <a id="_39">
     </a>
     三、配置使用
    </h3>
    <h4>
     <a id="1_40">
     </a>
     1、参数解释
    </h4>
    <pre><code>fio -filename=/dev/sdb -direct=1 -rw=write -ioengine=libaio -bs=4k -iodepth=32 -numjobs=1 -runtime=600 -group_reporting -name=4ktest
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        -filename
       </strong>
       <br/>
       用于测试块设备，一般指定需要测试的磁盘或者分区，如
       <code>
        -filename=/dev/sdb
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        -directory
       </strong>
       <br/>
       用于测试文件，一般指定需要测试的目录路径，如
       <code>
        -directory=/client/
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        -direct
       </strong>
       <br/>
       默认值为0，用于决定是否绕过缓存，一般设置为1（开启direct io，绕过系统缓存）
      </p>
     </li>
     <li>
      <p>
       <strong>
        -rw
       </strong>
       <br/>
       用于指定负载读写类型，一般可以设置以下参数：
      </p>
      <ul>
       <li>
        <strong>
         read
        </strong>
        ：顺序读（Sequential reads）
       </li>
       <li>
        <strong>
         write
        </strong>
        ：顺序写（Sequential writes）
       </li>
       <li>
        <strong>
         rw
        </strong>
        ：顺序读写（Mixed sequential reads and writes），当负载为混合读写类型时，需要指定读写占比（
        <strong>
         rwmixread
        </strong>
        ）
       </li>
       <li>
        <strong>
         randread
        </strong>
        ：随机读（Random reads）
       </li>
       <li>
        <strong>
         randwrite
        </strong>
        ：随机写（Random writes）
       </li>
       <li>
        <strong>
         randrw
        </strong>
        ：随机读写（Random mixed reads and writes），当负载为混合读写类型时，需要指定读写占比（
        <strong>
         rwmixread
        </strong>
        ）
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        rwmixread
       </strong>
       <br/>
       用于指定混合读写同一时刻读的占比，默认为50%，即读写比为1:1
      </p>
     </li>
     <li>
      <p>
       <strong>
        rwmixwrite
       </strong>
       <br/>
       用于指定混合读写同一时刻写的占比，默认为50%，即读写比为1:1
      </p>
     </li>
     <li>
      <p>
       <strong>
        -ioengine
       </strong>
       <br/>
       用于指定负载工作引擎，常用引擎一般有psync（同步IO引擎）及libaio（异步IO引擎）
       <br/>
       可选参数有sync , psync , vsync , posixaio , mmap , splice , syslet-rw , sg , null , net , netsplice , cpuio , guasi , external、libaio
      </p>
     </li>
     <li>
      <p>
       <strong>
        -bs
       </strong>
       <br/>
       用于指定IO块大小
      </p>
     </li>
     <li>
      <p>
       <strong>
        -iodepth
       </strong>
       <br/>
       用于指定IO深度，不同IO深度下有不同性能值，需合理调整验证以得出最优性能
      </p>
     </li>
     <li>
      <p>
       <strong>
        -numjobs
       </strong>
       <br/>
       用于指定IO并发线程数
      </p>
     </li>
     <li>
      <p>
       <strong>
        -runtime
       </strong>
       <br/>
       用于指定负载测试时间，时间单位为s
      </p>
     </li>
     <li>
      <p>
       <strong>
        -group_reporting
       </strong>
       <br/>
       用于指定测试结果显示方式
       <br/>
       当指定
       <code>
        -numjobs
       </code>
       参数时，如果不指定
       <code>
        -group_reporting
       </code>
       参数，测试结果以
       <code>
        per-job
       </code>
       形式显示（即把所有线程的测试结果单独显示出来）
       <br/>
       当指定
       <code>
        -numjobs
       </code>
       参数时，如果指定
       <code>
        -group_reporting
       </code>
       参数，测试结果以
       <code>
        per-group
       </code>
       形式显示（即汇总所有线程的测试结果合并显示出来）
      </p>
     </li>
     <li>
      <p>
       <strong>
        -name
       </strong>
       <br/>
       用于指定负载工作job名称，名称可自定义
      </p>
     </li>
    </ul>
