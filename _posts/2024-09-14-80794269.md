      <h3 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       <span style="font-size:20px;">
        一、三级模式结构
       </span>
      </h3>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
       数据库系统的三级模式结构是指模式、外模式和内模式。
      </div>
      <h4 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       1、模式
      </h4>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;text-indent:28px;line-height:1.75;font-size:14px;">
       模式也称为
       <span style="font-weight:bold;">
        逻辑模式或概念模式
       </span>
       ，是数据库中
       <span style="color:rgb(223,64,42);">
        全体数据的逻辑结构和特征的描述
       </span>
       ，是
       <span style="color:rgb(223,64,42);">
        所有用户的公共数据视图
       </span>
       。一个数据库只有一个模式，
       <span style="color:rgb(223,64,42);">
        模式位于三级结构的中间层
       </span>
       。
      </div>
      <h4 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       2、外模式
      </h4>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;text-indent:28px;line-height:1.75;font-size:14px;">
       外模式也称为
       <span style="font-weight:bold;">
        用户模式
       </span>
       ，它是数据库用户（包括应用程序员和最终用户）能够看见和使用的
       <span style="color:rgb(223,64,42);">
        局部数据的逻辑结构和特征的描述
       </span>
       ，是
       <span style="color:rgb(223,64,42);">
        数据库用户的数据视图
       </span>
       ，是与某一应用有关的数据的逻辑表示。
       <span style="color:rgb(223,64,42);">
        外模式是模式的子集，一个数据库可以有多个外模式
       </span>
       。
      </div>
      <h4 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       3、内模式
      </h4>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;text-indent:28px;line-height:1.75;font-size:14px;">
       内模式也称为
       <span style="font-weight:bold;">
        存储模式
       </span>
       ，
       <span style="color:rgb(223,64,42);">
        一个数据库只有一个内模式
       </span>
       ，它是数据
       <span style="color:rgb(223,64,42);">
        物理结构和存储方式的描述
       </span>
       ，是数据在
       <span style="color:rgb(223,64,42);">
        数据库内部的表示方式
       </span>
       。
      </div>
      <h3 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       <span style="font-size:20px;">
        二、三级模式之间的映射
       </span>
      </h3>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;line-height:1.75;font-size:14px;">
       数据库管理系统在三级模式之间提供了
       <span style="font-weight:700;">
        两层映射
       </span>
       ，分别为外模式/模式映射、模式/内模式映射。
      </div>
      <h4 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       1、外模式/模式映射
      </h4>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;text-indent:28px;line-height:1.75;font-size:14px;">
       对于
       <span style="color:rgb(255,0,0);">
        同一个模式可以有任意多个外模式
       </span>
       。对于每一个外模式，数据库系统都有一个外模式/模式映射。当模式被改变时，数据库管理员对各个外模式/模式映射做相应的改变，可以使外模式保持不变。这样，依据数据外模式编写的应用程序就不用修改，
       <span style="color:rgb(223,64,42);">
        保证了数据与程序的逻辑独立性。
       </span>
      </div>
      <h4 style="padding:0px;font-size:14px;line-height:1.75;white-space:pre-wrap;text-align:left;">
       2、模式/内模式
      </h4>
      <div style="padding:0px;margin:0px;white-space:pre-wrap;text-align:left;text-indent:28px;line-height:1.75;font-size:14px;">
       数据库中只有一个模式和一个内模式，所以
       <span style="color:rgb(255,0,0);">
        模式/内模式的映射是唯一
       </span>
       的，它定义了数据库的全局逻辑结构与存储结构之间的对应关系。当数据库的存储结构被改变时，数据库管理员对模式/内模式映射做相应的改变，可以使模式保持不变，应用程序相应地也不做变动。这样，
       <span style="color:rgb(223,64,42);">
        保证了数据与程序的物理独立性。
       </span>
      </div>
