    <p>
     写 Python 代码最好的方式莫过于使用集成开发环境（IDE）了。它们不仅能使你的工作更加简单、更具逻辑性，还能够提升编程体验和效率。
    </p>
    <p>
     每个人都知道这一点。而问题在于，如何从众多选项中选择最好的 Python 开发环境。初级开发者往往面临这个问题。
    </p>
    <p>
     <strong>
      本文将概述最受程序员欢迎的 Python IDE，以及其他值得考虑的选项。在此之前，我们先来看看 IDE 是什么。
     </strong>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/340fc8e0aeb9f6cffe36d69cc0f479cc.jpeg"/>
    </p>
    <p>
     <strong>
      什么是 IDE？
     </strong>
     <strong>
      IDE 有什么用？
     </strong>
    </p>
    <p>
     IDE 是开发者创建程序时使用的软件包。它通过简单的用户界面集成多个高度关联的组件，从而最大化程序员的生产效率。本质上，IDE 是一种改进代码创建、测试和 debug 流程的工具，它使这些工作更加简单。
    </p>
    <p>
     IDE 使用的工具包括：
    </p>
    <ul>
     <li>
      <p>
       文本编辑器
      </p>
     </li>
     <li>
      <p>
       编译器和/或解释器
      </p>
     </li>
     <li>
      <p>
       装配自动化工具（assembly automation tool）
      </p>
     </li>
     <li>
      <p>
       调试器（debugger）
      </p>
     </li>
    </ul>
    <p>
     有一个与 IDE 相似的术语——代码编辑器（code editor）。我们先来探讨二者的异同。
    </p>
    <p>
     <strong>
      什么是代码编辑器？
     </strong>
    </p>
    <p>
     代码编辑器是一个能够突出显示语法和安排代码版式的文本编辑器。高级代码编辑器可以开发和修改代码。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/6e88f5de3b0d005153a1528e9f627f21.png"/>
    </p>
    <p>
     <strong>
      IDE 和代码编辑器的显著特征
     </strong>
    </p>
    <p>
     IDE 和代码编辑器有哪些共同优点？它们均可帮助开发者：
    </p>
    <ul>
     <li>
      <p>
       存储和重新打开脚本
      </p>
     </li>
     <li>
      <p>
       运行代码
      </p>
     </li>
     <li>
      <p>
       debug
      </p>
     </li>
     <li>
      <p>
       突出显示语法
      </p>
     </li>
    </ul>

    <p>
     <strong>
      兼容性：
     </strong>
     Windows、Linux、Mac OS、IOS、Android、UNIX、AmigaOS、MorphOS
    </p>
    <p>
     <strong>
      主要插件和功能：
     </strong>
    </p>

     <li>
      <p>
       其脚本允许用 Python 执行几乎所有编程任务。
      </p>
     </li>
     <li>
      <p>
       Ctags，帮助实现更好的标记导航。
      </p>
     </li>
     <li>
      <p>
       内置基础词汇补全。
      </p>
     </li>

     <strong>
      优点：
     </strong>
    </p>

      <p>

      </p>
     </li>
     <li>
      <p>

     </li>

      <p>

    </ul>

     <strong>
      缺点：
     </strong>

     <li>

      </p>

      <p>
       其 modal editor 需要用户花费一定时间学习。

     </li>
