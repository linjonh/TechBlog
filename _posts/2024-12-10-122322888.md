    <h2>
     一、登录设备并切换用户
    </h2>
    <pre><code>su oracle		//root登录之后切换至oracle用户操作</code></pre>
    <h2>
     二、连接数据库
    </h2>
    <ul>
     <li>
      在Oracle用户下连接
     </li>
    </ul>
    <pre><code>sqlplus oracle/oracle@oracleDB</code></pre>
    <ul>
     <li>
      使用管理员权限
     </li>
    </ul>
    <pre><code>connect / as sysdba
sqlplus / as sysdba  //sysdba直接登录，与切换管理员权限相同</code></pre>
    <h2>
     三、启动/关闭服务
    </h2>
    <ul>
     <li>
      <h4>
       启动：“startup”
      </h4>
      <ul>
       <li>
        不使用参数，启动数据库实例并打开数据库，以便用户使用数据库，多数情况下使用此方式；
       </li>
       <li>
        “nomount”，只启动数据库实例，但不打开数据库，在想要创建一个新的数据库时使用；
       </li>
       <li>
        “mount”，数据库变更名称时使用；
       </li>
      </ul>
     </li>
     <li>
      <h4>
       关闭：“shutdown”
      </h4>
      <ul>
       <li>
        Normal：需要等待所有的用户断开连接才执行关闭数据库任务（在执行这个命令后不允许新的连接）；
       </li>
       <li>
        Immediate：等待用户完成当前的语句，不允许新用户连接；
       </li>
       <li>
        Transactional：等待用户完成当前的事务，不允许新用户连接；
       </li>
       <li>
        Abort：不做任何等待，直接关闭数据库；
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     四、监听端口状态
    </h2>
    <ul>
     <li>
      启动
     </li>
    </ul>
    <pre><code>lsnrctl start</code></pre>
    <ul>
     <li>
      关闭
     </li>
    </ul>
    <pre><code>lsnrctl stop</code></pre>
    <ul>
     <li>
      查看当前状态
     </li>
    </ul>
    <pre><code>lsnrctl status</code></pre>
