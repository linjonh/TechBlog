    <h4 id="01-%E9%A1%B9%E7%9B%AE%E6%A6%82%E8%BF%B0">
     01. 项目概述
    </h4>
    <p>
     [慕尚花坊] 是一款 同城鲜花订购 的小程序，专业提供各地鲜花速递、鲜花预定、网上订花、包月鲜花等服务。最快3小时送花上门，保证花材新鲜和鲜花质量，可先送花后付款，专业花艺师傅精美包扎，品质保证，至诚服务。
    </p>
    <h4 id="02-%E9%A1%B9%E7%9B%AE%E6%BC%94%E7%A4%BA">
     02. 项目演示
    </h4>
    <p>
     [慕尚花坊] 项目涵盖电商项目常见功能模块，包含：
    </p>
    <ol>
     <li>
      项目首页
     </li>
     <li>
      商品分类
     </li>
     <li>
      商品列表
     </li>
     <li>
      商品详情
     </li>
     <li>
      用户管理
     </li>
     <li>
      收货地址
     </li>
     <li>
      购物车
     </li>
     <li>
      结算支付
     </li>
     <li>
      订单管理
     </li>
     <li>
      等……
     </li>
    </ol>
    <h4 id="03-%E9%A1%B9%E7%9B%AE%E6%8A%80%E6%9C%AF%E6%A0%88">
     03. 项目技术栈
    </h4>
    <p>
     [慕尚花坊] 项目使用原生小程序进行搭建开发，项目涵盖小程序开发所有常用的知识点
    </p>
    <ol>
     <li>
      <p>
       小程序内置组件：采用小程序内置组件 结合
       <code>
        Vant
       </code>
       组件库实现页面结构的搭建
      </p>
     </li>
     <li>
      <p>
       项目中使用了 css 拓展语言 Scss 绘制页面的结构
      </p>
     </li>
     <li>
      <p>
       小程序内置
       <code>
        API
       </code>
       ：交互、支付、文件上传、地图定位、网络请求、预览图片、本地存储等
      </p>
     </li>
     <li>
      <p>
       小程序分包加载：降低小程序的启动时间、包的体积，提升用户体验度
      </p>
     </li>
     <li>
      <p>
       小程序组件开发：将页面内的功能模块抽象成自定义组件，实现代码的复用
      </p>
     </li>
     <li>
      <p>
       网络请求封装：request 方法封装、快捷方式封装、响应拦截器、请求拦截器
      </p>
     </li>
     <li>
      <p>
       <code>
        骨架屏
       </code>
       组件：利用开发者工具提供了自动生成骨架屏代码的能力，提高了整体使用体验和用户满意度。
      </p>
     </li>
     <li>
      <p>
       <code>
        UI
       </code>
       组件库：使用
       <code>
        Vant
       </code>
       组件库实现小程序 结构的绘制
      </p>
     </li>
     <li>
      <p>
       <code>
        LBS
       </code>
       ：使用腾讯地图服务进行
       <code>
        LBS
       </code>
       逆地址解析，实现选择收货地址功能
      </p>
     </li>
     <li>
      <p>
       <code>
        miniprogram-licia
       </code>
       ：使用
       <a href="https://licia.liriliri.io/" rel="nofollow" title="licia">
        licia
       </a>
       进行函数的防抖节流
      </p>
     </li>
     <li>
      <p>
       <code>
        async-validator
       </code>
       ：使用
       <a href="https://github.com/yiminghe/async-validator" title="async-validator">
        async-validator
       </a>
       实现表单验证
      </p>
     </li>
     <li>
      <p>
       <code>
        miniprogram-computed
       </code>
       : 使用
       <a href="https://github.com/wechat-miniprogram/computed" title="miniprogram-computed">
        miniprogram-computed
       </a>
       进行计算属性功能
      </p>
     </li>
     <li>
      <p>
       <code>
        mobx-miniprogram
       </code>
       ：使用
       <code>
        mobx-miniprogram
       </code>
       进行项目状态的管理
      </p>
      <p>
       ​
      </p>
     </li>
    </ol>

    </h3>
    <p>
     在开始开发一个小程序项目之前，需要先申请开发权限。
    </p>
    <p>
     需要将自己的微信号发送给对应小程序账号的管理员，在小程序微信公众后台添加我们自己为开发者。
    </p>
    <blockquote>
     <p>
      📌：
      <strong>
       注意事项：
      </strong>
     </p>
     <ol>
      <li>
       <p>
        需要将自己的
        <strong>
         微信号
        </strong>
        发送给对应小程序账号的管理员
       </p>
      </li>
      <li>
       <p>
        <strong>
         手机号不是微信号
        </strong>
        (除非将手机号设置为了微信号)

      </li>
     </ol>
    </blockquote>

     如何查看自己的微信号：

    <h4 id="02-%E8%87%AA%E5%AE%9A%E4%B9%89%E6%9E%84%E5%BB%BA-npm--%E9%9B%86%E6%88%90sass">
     02. 自定义构建 npm + 集成Sass
    </h4>
    <p>
     随着项目的功能越来越多、项目越来越复杂，文件目录也变的很繁琐，为了方便进行项目的开发，开发人员通常会对目录结构进行调整优化，在慕尚花坊项目中，
     <strong>
      我们就需要将小程序源码放到 miniprogram 目录下
     </strong>
     。
    </p>
    <p>
     <strong>
      自定义构建
     </strong>
    </p>
    <ol>
     <li>
      <p>
       首先在
       <code>
        project.config.json
       </code>
       配置
       <code>
        miniprogramRoot
       </code>
       选项，指定小程序源码的目录
      </p>
     </li>
     <li>
      <p>
       然后配置
       <code>
        project.config.json
       </code>
       的
       <code>
        setting.packNpmManually
       </code>
       为
       <code>
        true
       </code>
       ，开启自定义 node_modules 和 miniprogram_npm 位置的构建 npm 方式
      </p>
     </li>
     <li>
      <p>
       最后配置 project.config.json 的
       <code>
        setting.packNpmRelationList
       </code>
       项，指定
       <code>
        packageJsonPath
       </code>
       和
       <code>
        miniprogramNpmDistDir
       </code>
       的位置
      </p>
      <ul>
       <li>
        packageJsonPath 表示 node_modules 源对应的 package.json
       </li>
       <li>
        miniprogramNpmDistDir 表示 node_modules 的构建结果目标位置
       </li>
      </ul>
     </li>
     <li>
      <p>
       安装
       <code>
        vant
       </code>
       ，然后进行
       <code>
        npm 构建
       </code>
       ，测试是否能够正常
       <code>
        vant
       </code>
       构建成功
      </p>
      <pre><code class="hljs">npm i @vant/weapp</code></pre>
     </li>
    </ol>
    <blockquote>
     <p>
      📌
      <strong>
       注意
      </strong>
      ：
     </p>
     <p>
      ​ 配置后如果没有生效，需要 [ 重启微信开发者工具 ] ❗ ❗
     </p>
    </blockquote>
    <p>
     <strong>
      集成 Sass
     </strong>
    </p>
    <p>
     在
     <code>
      project.config.json
     </code>
     文件中，修改
     <code>
      setting
     </code>
     下的
     <code>
      useCompilerPlugins
     </code>
     字段为
     <code>
      ["sass"]
     </code>
     ，即可开启工具内置的 sass 编译插件。
    </p>
    <h4 id="03-%E9%9B%86%E6%88%90%E9%A1%B9%E7%9B%AE%E9%A1%B5%E9%9D%A2%E6%96%87%E4%BB%B6">
     03. 集成项目页面文件
    </h4>
    <p>
     <strong>

    </p>
    <p>
     <strong>
      代码分析：
     </strong>
    </p>
    <ol>
     <li>
      <code>
       app.json
      </code>

      <code>
       pages
      </code>
      、
      <code>

      </code>

      </code>
      中对项目中会使用到的
      <code>

     </li>
     <li>
      <code>
       app.wxss
      </code>
      文件中导入了阿里巴巴使用图标库
     </li>
     <li>
      <code>
       components
      </code>
      文件夹中定义了两个公共的组件
     </li>
     <li>
      <code>
       pages
      </code>
      目录下存放了项目中所有页面的文件，后续我们会进行分包的处理
     </li>
    </ol>
    <h4 id="04-vscode-%E5%BC%80%E5%8F%91%E5%B0%8F%E7%A8%8B%E5%BA%8F%E9%A1%B9%E7%9B%AE">
     04. VsCode 开发小程序项目
    </h4>
    <p>
     <strong>
      知识点：
     </strong>
    </p>
    <p>
     在进行项目开发的时候，部分同学可能不习惯微信开发者工具进行开发，而是习惯使用
     <code>
      VSCode
     </code>
     等编辑器
    </p>
    <p>
     但是
     <code>
      VsCode
     </code>
     对小程序开发支持的不是非常友好，如果想通过
     <code>
      VSCode
     </code>
     开发小程序项目，需要安装以下插件：
    </p>
    <ol>
     <li>
      <code>
       WXML - Language Service
      </code>
     </li>
     <li>
      <code>
       prettier
      </code>
     </li>
     <li>
      微信小程序开发工具
     </li>
     <li>
      微信小程序助手-Y
     </li>
     <li>
      小程序开发助手(可选)
     </li>
     <li>
      其他......
     </li>
    </ol>
    <blockquote>
     <p>
      💡 Tip：
     </p>
     <p>
      使用 VsCode 开发小程序项目时，如果需要预览、调试小程序，依然需要借助微信开发者工具
     </p>
    </blockquote>
    <p>
     <strong>
      配置详细插件：
     </strong>
    </p>
    <ol>
     <li>
      <p>
       在【项目的根目录】下创建
       <code>
        .vscode
       </code>
       文件夹，注意：文件夹名字前面带
       <strong>
        <code>
         .
        </code>
       </strong>
       点❗
      </p>
     </li>
     <li>
      <p>
       在
       <code>
        .vscode
       </code>
       文件夹下，创建
       <code>
        settings.json
       </code>
       ，用来对安装的插件属性进行设置，具体属性设置从下面复制即可
      </p>
      <ul>
       <li>
        <u>
         <strong>
          注意：
          <code>
           .vscode
          </code>
          文件夹下的
          <code>
           settings.json
          </code>
          文件只对当前一个项目生效
         </strong>
        </u>
       </li>
      </ul>
     </li>
     <li>
      <p>
       在【项目的根目录】下创建
       <code>
        .prettierrc
       </code>
       文件，进行
       <code>
        Prettier
       </code>
       代码规则的配置，规则从下面复制即可
      </p>
     </li>
     <li>
      <p>
       为了让
       <code>
        Prettier
       </code>
       配置项在微信开发者工具生效，需要在微信开发者工具中也安装
       <code>
        Prettier
       </code>
       扩展插件。
      </p>
     </li>
    </ol>
    <p>
     <code>
      ➡️ .vscode/settings.json
     </code>
    </p>
    <pre></pre>
    <pre><code class="hljs">{
  // 保存文件时是否自动格式化
  "editor.formatOnSave": true,

  // ---------------- 以下是 [ prettier ] 插件配置 ----------------

  // 指定 javascript、wxss、scss、less、json、jsonc 等类型文件使用 prettier 进行格式化
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  "[wxss]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  "[scss]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  "[less]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  "[json]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  "[jsonc]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  // Prettier 的一个配置项，用于指定哪些文件类型需要使用 Prettier 进行格式化
  "prettier.documentSelectors": ["**/*.wxml", "**/*.wxss", "**/*.wxs"],

  // ---------------- 以下是 [ WXML - Language Service ] 插件配置 ----------------

  // wxml 文件使用 prettier 进行格式化
  "[wxml]": {
    // "qiu8310.minapp-vscode" 是 WXML - Language Service 插件提供的配置项
    // 此插件主要是针对小程序的 wxml 模板语言，可以自动补全所有的组件、组件属性、组件属性值等等

    // 如果是 VsCode 需要开启这个配置
    "editor.defaultFormatter": "qiu8310.minapp-vscode"

    // 如果是微信小程序，需要开启这个配置，通过 esbenp.prettier-vscode 对代码进行格式化
    // "editor.defaultFormatter": "esbenp.prettier-vscode"
  },

  // 创建组件时使用的 css 后缀
  "minapp-vscode.cssExtname": "scss", // 默认 wxss，支持 styl sass scss less css

  // 指定 WXML 格式化工具
  "minapp-vscode.wxmlFormatter": "prettier",
  // 配置 prettier 代码规范
  "minapp-vscode.prettier": {
    "useTabs": false,
    "tabWidth": 2,
    "printWidth": 80
  },

  // ---------------- 以下是 [ 微信小程序助手-Y ] 插件配置 ----------------

  // 新增、删除小程序页面时，是否自动同步 app.json pages 路径配置，默认为 false
  "wechat-miniapp.sync.delete": true,
  // 设置小程序页面 wxss 样式文件的扩展名
  "wechat-miniapp.ext.style": "scss",

  // ---------------- 其他配置项 ----------------

  // 配置语言的文件关联，运行 .json 文件时写注释
  // 但在 app.json 和 page.json 中无法使用
  "files.associations": {
    "*.json": "jsonc"
  }
}
