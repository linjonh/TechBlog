     音视频即时通讯解决方案

    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       ，也叫音视频互动开发平台；是一套跨平台的即时通讯解决方案，基于先进的
      </span>
      <span style="color:black; background:white">
       H.264
      </span>
      <span style="color:black; background:white">
       视频编码标准、
      </span>
      <span style="color:black; background:white">
       AAC
      </span>
      <span style="color:black; background:white">
       音频编码标准与
      </span>
      <span style="color:black; background:white">
       P2P
      </span>
      <span style="color:black; background:white">
       技术，整合了佰锐科技在音视频编码、多媒体通讯领域领先的开发技术和丰富的产品经验而设计的高质量、宽适应性、分布式、模块化的网络音视频互动平台（支持
      </span>
      <span style="color:black; background:white">
       Windows
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       Mobile
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       Linux
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       Android
      </span>
      <span style="color:black; background:white">
       等平台）。
      </span>
     </span>
    </p>
    <p align="left">
     <strong>
      <span style="font-size:12px; color:black">
       详解
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       音视频即时通讯技术方面，
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       包含了音视频处理模块（采集、编解码）、流媒体管理模块（丢包重传、抖动平滑、动态缓冲）、流媒体播放模块（多路混音、音视频同步）以及
      </span>
      <span style="color:black; background:white">
       P2P
      </span>
      <span style="color:black; background:white">
       网络模块（
      </span>
      <span style="color:black; background:white">
       NAT
      </span>
      <span style="color:black; background:white">
       穿透、
      </span>
      <span style="color:black; background:white">
       UPnP
      </span>
      <span style="color:black; background:white">
       支持）等多个子模块，封装了底层的硬件操作（音视频采集、播放）、封装了流媒体处理（编解码、网络传输）等非常专业和复杂的技术，为上层应用提供简单的
      </span>
      <span style="color:black; background:white">
       API
      </span>
      <span style="color:black; background:white">
       控制接口，可以在极短的开发周期，以及极少的人力资源投入下为客户的现有平台增加音视频即时通讯、多方会议的功能。
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       分为客户端
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       和服务器
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       两大部分，其中客户端
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       用于实现语音、视频的交互以及其它客户端相关的功能，而服务器
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       主要实现业务层逻辑控制，以及与第三方平台的互联等。客户端
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       和服务器
      </span>
      <span style="color:black; background:white">
       SDK
      </span>
      <span style="color:black; background:white">
       均支持
      </span>
      <span style="color:black; background:white">
       C++
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       C#
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       vbnet
      </span>
      <span style="color:black; background:white">
       以及
      </span>
      <span style="color:black; background:white">
       Delphi
      </span>
      <span style="color:black; background:white">
       等开发语言。
      </span>
     </span>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       通过
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       ，可以开发具有企业特色的即时通讯系统、视频游戏系统、视频会议系统、网络教学系统以及在线客服系统等，系统的功能、界面完全由企业定制，底层通信协议加密传输，多重安全防护，保密性强。
      </span>
     </span>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       采用动态缓冲技术来适应不同网络环境（局域网、企业专网、互联网、
      </span>
      <span style="color:black; background:white">
       3G
      </span>
      <span style="color:black; background:white">
       网络），根据不同的网络状态动态调节相关参数，使得
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       在多种网络环境下均有良好的表现，并特别针对互联网、
      </span>
      <span style="color:black; background:white">
       3G
      </span>
      <span style="color:black; background:white">
       网络等应用场合进行优化，为上层应用提供视频质量的动态调节接口、音频质量的动态调节接口。
      </span>
     </span>
    </p>
    <p align="left">
     <strong>
      <span style="font-size:12px; color:black">
       应用领域
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       音视频即时通讯应用方面，
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       可以作为在线游戏的音视频引擎，也可以作为视频会议、网络教育、
      </span>
      <span style="color:black; background:white">
       IM
      </span>
      <span style="color:black; background:white">
       系统、网络视频聊天系统等互动平台的核心库，还可以应用于政府应急指挥平台、专家远程协商平台等专用系统。
      </span>
      <span style="color:black; background:white">
       AnyChat SDK
      </span>
      <span style="color:black; background:white">
       目前已广泛应用于在线视频游戏、网站视频聊天、企业视频会议、军队远程指挥、政府应急指挥平台等领域。
      </span>
     </span>
     <a name="ref_[1]_9781267">
     </a>
    </p>
    <p align="left">
     <strong>
      <span style="font-size:12px; color:black">
       音视频即时通讯涉及技术领域
      </span>
     </strong>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        1
       </span>
       <span style="color:black">
        、跨平台
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       音视频即时通讯必须可以支持跨平台的应用，服务器支持
      </span>
      <span style="color:black; background:white">
       Windows
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       Linux
      </span>
      <span style="color:black; background:white">
       、
      </span>
      <span style="color:black; background:white">
       Unix
      </span>
      <span style="color:black; background:white">
       等所有主流服务器操作系统。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        2
       </span>
       <span style="color:black">
        、视频技术
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       目前国内比较先进的音视频即时通讯采用国际领先的视频编码标准
      </span>
      <span style="color:black; background:white">
       H.264
      </span>
      <span style="color:black; background:white">
       （
      </span>
      <span style="color:black; background:white">
       MPEG-4 part 10 AVC /H.264
      </span>
      <span style="color:black; background:white">
       ）编码，因为
      </span>
      <span style="color:black; background:white">
       H.264/AVC
      </span>
      <span style="color:black; background:white">
       在压缩效率方面有着特殊的表现，一般情况下达到
      </span>
      <span style="color:black; background:white">
       MPEG-2
      </span>
      <span style="color:black; background:white">
       及
      </span>
      <span style="color:black; background:white">
       MPEG-4
      </span>
      <span style="color:black; background:white">
       简化类压缩效率的大约
      </span>
      <span style="color:black; background:white">
       2
      </span>
      <span style="color:black; background:white">
       倍。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        3
       </span>
       <span style="color:black">
        、音频技术
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       假如音视频即时通讯采用先进的
      </span>
      <span style="color:black; background:white">
       AAC
      </span>
      <span style="color:black; background:white">
       语音编码体系，可以很大程度地改善了数据压缩率和声音品质。因为全面支持静音检测（
      </span>
      <span style="color:black; background:white">
       VAD
      </span>
      <span style="color:black; background:white">
       ）、噪音抑制（
      </span>
      <span style="color:black; background:white">
       NS
      </span>
      <span style="color:black; background:white">
       ）、自动增益（
      </span>
      <span style="color:black; background:white">
       AGC
      </span>
      <span style="color:black; background:white">
       ）以及回音消除（
      </span>
      <span style="color:black; background:white">
       AEC
      </span>
      <span style="color:black; background:white">
       ）等音效处理，极大提高用户体验。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        4
       </span>
       <span style="color:black">
        、
       </span>
       <span style="color:black">
        P2P
       </span>
       <span style="color:black">
        技术
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       音视频即时通讯对通讯技术要求比较高，针对可不经过服务器中转的音视频应用，如果采用
      </span>
      <span style="color:black; background:white">
       P2P
      </span>
      <span style="color:black; background:white">
       通信技术，可以大大的减轻系统服务器的负荷，并成几何倍数的扩大系统的容量，提高
      </span>
      <span style="color:black; background:white">
       P2P
      </span>
      <span style="color:black; background:white">
       通信效率。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        5
       </span>
       <span style="color:black">
        、高性能网络核心
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       一套好的音视频即时通讯方案，如果在服务器模块采用完成端口实现高性能的系统架构，采用重叠
      </span>
      <span style="color:black; background:white">
       I/O
      </span>
      <span style="color:black; background:white">
       机制，通过线程池和缓冲池的管理，可以极高的优化系统结构，提高系统的性能。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        6
       </span>
       <span style="color:black">
        、开放的接口
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       同理，音视频即时通讯方案最好是采用模块化技术体系，因为良好的平台兼容性与可扩展性，丰富的
      </span>
      <span style="color:black; background:white">
       API
      </span>
      <span style="color:black; background:white">
       函数，可以为上层应用提供了开放的应用接口。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        7
       </span>
       <span style="color:black">
        、流媒体传输技术
       </span>
      </span>
     </strong>
    </p>
    <p align="left">
     <span style="font-size:12px">
      <span style="color:black; background:white">
       这是一套好的音视频即时通讯方案所必须采用的，可以实现音频抖动缓冲（
      </span>
      <span style="color:black; background:white">
       Jitter Buffer
      </span>
      <span style="color:black; background:white">
       ）、视频马赛克消除（
      </span>
      <span style="color:black; background:white">
       MS
      </span>
      <span style="color:black; background:white">
       ）。
      </span>
     </span>
    </p>
    <p align="left" style="background:white">
     <strong>
      <span style="font-size:12px">
       <span style="color:black">
        8
       </span>
       <span style="color:black">
        、服务器并发处理技术
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="font-size:12px; color:black; background:white">
      采用服务器并发处理技术，可以提高音视频即时通讯方案的效率。
     </span>
    </p>
