    </h2>
    <p>
     <span style="word-wrap:break-word; font-family:Tahoma,Helvetica,SimSun,sans-serif; font-size:14px; line-height:21px">
      <span style="font-family:arial,宋体,sans-serif; word-wrap:break-word">
       <span style="color:rgb(0,0,0); background-color:rgb(255,255,0)">
        <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">

        </span>
        <a href="http://baike.baidu.com/view/65652.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px" target="_blank">
         CISC
        </a>
        <a href="http://baike.baidu.com/view/178189.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px" target="_blank">
         指令系统
        </a>
        <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">
         基础上发展起来的。各种指令的使用频度相当悬殊，最常使用的是一些比较简单的指令，它们仅占指令总数的20%，但在程序中出现的频度却占80%。复杂的指令系统必然增加
        </span>
        <a href="http://baike.baidu.com/view/1125.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px" target="_blank">
         微处理器
        </a>
       </span>
       <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">
        <span style="color:rgb(0,0,0); background-color:rgb(255,255,0)">
         的复杂性，使处理器的研制时间长，成本高。并且复杂指令需要复杂的操作，必然会降低计算机的速度。
         <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">
          RISC的
         </span>
         <a href="http://baike.baidu.com/view/178195.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px" target="_blank">
          指令格式
         </a>
         <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">
          统一，种类比较少，
         </span>
         <a href="http://baike.baidu.com/view/889427.htm" rel="nofollow noopener noreferrer" style="color:rgb(19,110,194); font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px" target="_blank">
          寻址方式
         </a>
        </span>
        <span style="font-family:arial,宋体,sans-serif; font-size:14px; line-height:24px">
         <span style="background-color:rgb(255,255,0)">
          也比复杂指令集少。当然处理速度就提高很多了。
          <span style="color:rgb(142,104,96); font-family:Arial,Helvetica,simsun,u5b8bu4f53; font-size:14px; line-height:25px; text-align:left; text-indent:28px">
           RISC可同时执行多条指令，它可将一条指
          </span>
          <span style="font-family:Arial,Helvetica,simsun,u5b8bu4f53; font-size:14px; line-height:25px; text-align:left; text-indent:28px">
           <span style="color:#333333">
            令分割成若干个进程或线程，交由多个处理器同时执行。由于RISC执行的是精简指令集，所以它的制造工艺简单且成本低廉。

    <h2>
     <span style="color:rgb(68,68,68); font-family:Tahoma,Helvetica,SimSun,sans-serif; font-size:14px; line-height:21px">
      3， 重写和重载的区别
     </span>
    </h2>
    <p>
     <span style="font-family:Tahoma,Helvetica,SimSun,sans-serif; font-size:14px; line-height:21px; background-color:rgb(255,255,0)">
      重写是对于父类与子类来说的，当一个子类继承了父类的方法，然后需要把继承来的方法重写定义其实现，这称为重写（或者覆盖），重写的方法名、返回类型以及参数列表必须与父类的一样。重载是对于类的内部来说的，当一个类中定义了多个同名的方法，但是它们的返回类型或者签名不一样，这称为重载。
     </span>
    </p>

     <span style="color:rgb(68,68,68); font-family:Tahoma,Helvetica,SimSun,sans-serif; font-size:14px; line-height:21px">
      2，两个排过序的数组，快速找出两者相同的元素。
     </span>
    </h3>
    <div>
     <span style="color:rgb(68,68,68); font-family:Tahoma,Helvetica,SimSun,sans-serif; font-size:14px; line-height:21px">
     </span>
     <pre><code class="language-java">/**
	 * 发现有一个 O(n)算法。因为两个数组都是排好序的（以两个数组都是升序排序为例），
	 * 所以只要一次遍历就行了。
	 * 首先设两个下标，分别初始化为两个数组的起始地址，依次向前推进 。推进的规则是
	 * 比较两个数组中的数字，小的那个数组的下标向前推进一步，直到任何一个数组的下
	 * 标到达数组末尾时，如果这时还没碰到相同的数字，说明数组中没有相同的数字。
	 * 注意：当两个数组都是降序排序的时候，大的那个数的下标推进。如果一个升序一个降序，
	 * 那么就非不从不同的两端遍历。
	 * @param a
	 * @param b
	 */
	public static void findSameEle(int [] a,int[] b){
		int i = 0;
		int j = 0;
		
		while(i &lt; a.length &amp;&amp; j &lt; b.length ){
			if(a[i] == b[j]){
				//输出相同的数
				System.out.print(a[i]+ " ");
				i++;
				j++;
			}
			else if(a[i] &lt; b[j]){
				//当前比较小的数对应的数组下标+1
				i++;
			}
			else{
				j++;
			}
		}
	}</code></pre>
