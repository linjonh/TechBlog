    <p>
     在前端开发中，流式输出（streaming output）通常是指逐步输出数据，而不是等待所有数据准备好后一次性显示。这种技术在处理大型数据集、实时数据或需要逐步加载内容的情况下非常有用。下面介绍几种实现流式输出的方法，包括使用 Fetch API 和 EventSource。
    </p>
    <h3>
     <a id="1__Fetch_API__6">
     </a>
     1. 使用 Fetch API 实现流式输出
    </h3>
    <p>
     通过 Fetch API 和可读流（Readable Streams），可以在响应到达时逐步读取和处理数据。这在处理大文件或实时更新时非常有效。
    </p>
    <h4>
     <a id="_11">
     </a>
     示例代码
    </h4>
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
    &lt;title&gt;Stream Output Example&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;Streamed Output&lt;/h1&gt;
    &lt;div id="output"&gt;&lt;/div&gt;

    &lt;script&gt;
        async function streamData() {
            const response = await fetch('https://jsonplaceholder.typicode.com/posts');
            const reader = response.body.getReader();
            const outputDiv = document.getElementById('output');
            let result;

            // 读取数据流
            while (!(result = await reader.read()).done) {
                const chunk = new TextDecoder().decode(result.value);
                // 将新接收到的部分添加到输出
                outputDiv.innerHTML += chunk + '&lt;br&gt;';
            }
        }

        streamData();
    &lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
</code></pre>
    <h3>
     <a id="2__ServerSent_Events_SSE_45">
     </a>
     2. 使用 Server-Sent Events (SSE)
    </h3>
    <p>
     Server-Sent Events (SSE) 是一种技术，可以从服务器向客户端推送实时数据。它基于HTTP协议，使用EventSource API。
    </p>
    <h4>
     <a id="_50">
     </a>
     示例代码
    </h4>
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
    &lt;title&gt;SSE Example&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;Server-Sent Events Example&lt;/h1&gt;
    &lt;div id="output"&gt;&lt;/div&gt;

    &lt;script&gt;
        const outputDiv = document.getElementById('output');
        const eventSource = new EventSource('https://example.com/sse'); // 替换为你的SSE服务URL

        eventSource.onmessage = function(event) {
            outputDiv.innerHTML += 'Received: ' + event.data + '&lt;br&gt;';
        };

        eventSource.onerror = function(event) {
            console.error('Error occurred:', event);
            eventSource.close(); // 关闭连接
        };
    &lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
</code></pre>
    <h3>
     <a id="3__WebSockets_80">
     </a>
     3. 使用 WebSockets
    </h3>
    <p>
     WebSockets 提供了全双工通信，允许在客户端和服务器之间进行实时数据传输。这适用于需要双向通信的场景。
    </p>
    <h4>
     <a id="_85">
     </a>
     示例代码
    </h4>
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
    &lt;title&gt;WebSocket Example&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;WebSocket Example&lt;/h1&gt;
    &lt;div id="output"&gt;&lt;/div&gt;

    &lt;script&gt;
        const outputDiv = document.getElementById('output');
        const socket = new WebSocket('wss://example.com/socket'); // 替换为你的WebSocket服务URL

        socket.onopen = function() {
            console.log('WebSocket connection established');
        };

        socket.onmessage = function(event) {
            outputDiv.innerHTML += 'Received: ' + event.data + '&lt;br&gt;';
        };

        socket.onerror = function(error) {
            console.error('WebSocket error:', error);
        };

        socket.onclose = function() {
            console.log('WebSocket connection closed');
        };
    &lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
</code></pre>
    <h3>
     <a id="_122">
     </a>
     总结
    </h3>
    <ul>
     <li>
      Fetch API：适合在HTTP响应中逐步获取和处理数据流。
     </li>
     <li>
      Server-Sent Events：用于从服务器推送实时事件，简单易用。
     </li>
     <li>
      WebSockets：提供双向通信，适合需要实时交互的应用。
     </li>
    </ul>
