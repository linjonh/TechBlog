    <br/>
    本文主要介绍如何用 vscode 来调试 python/c++ 的 Pytorch 源码。
    <p>
    </p>
    <p>
     首先要先确保有一个从源码编译的 Debug 版本的 Pytorch项目，具体 Debug 版本的源码编译网上有很多相关资料，可以自行查阅操作一波。
    </p>
    <p>
     接下来主要就是 vscode 的配置工作。
    </p>
    <h4>
     <a id="_6">
     </a>
     创建需要调试的文件
    </h4>
    <p>
     首先在 Pytorch 项目工程的主目录下创建一个子文件夹，在这个子文件夹中首先写一个我们想要调试的示例代码（如果直接创建文件，import torch 可能会报错）。 like this:
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/33d524f7d97879ed72c1b19e0fca146f.png#pic_center"/>
    </p>
    <h4>
     <a id="_vscode_python__11">
     </a>
     配置 vscode python 调试
    </h4>
    <p>
     这里要首先安装一个 Python 的扩展。
    </p>
    <p>
     打开并选中上述 debug.py 文件之后，点击 vscode 左边调试区，选择创建一个 launch.json 文件。
     <br/>
     <img src="https://i-blog.csdnimg.cn/blog_migrate/916817999ded2804fafd4400924fdb64.png" width="30%">
      <br/>
      .py 文件一般默认出来的 launch.json 的内容如下所示：
     </img>
    </p>
    <pre><code class="prism language-bash">    <span class="token string">"version"</span><span class="token builtin class-name">:</span> <span class="token string">"0.2.0"</span>,
    <span class="token string">"configurations"</span><span class="token builtin class-name">:</span> <span class="token punctuation">[</span>
        <span class="token punctuation">{<!-- --></span>
            <span class="token string">"name"</span><span class="token builtin class-name">:</span> <span class="token string">"Python: 当前文件"</span>,
            <span class="token string">"type"</span><span class="token builtin class-name">:</span> <span class="token string">"python"</span>,
            <span class="token string">"request"</span><span class="token builtin class-name">:</span> <span class="token string">"launch"</span>,
            <span class="token string">"program"</span><span class="token builtin class-name">:</span> <span class="token string">"<span class="token variable">${file}</span>"</span>,
            <span class="token string">"console"</span><span class="token builtin class-name">:</span> <span class="token string">"integratedTerminal"</span>,
        <span class="token punctuation">}</span>
    <span class="token punctuation">]</span>
</code></pre>
    <p>
     这里要调试 Python 源码加入 “justMyCode”: false 即可，添加后如下：
    </p>
    <pre><code class="prism language-bash">    <span class="token string">"version"</span><span class="token builtin class-name">:</span> <span class="token string">"0.2.0"</span>,
    <span class="token string">"configurations"</span><span class="token builtin class-name">:</span> <span class="token punctuation">[</span>
        <span class="token punctuation">{<!-- --></span>
            <span class="token string">"name"</span><span class="token builtin class-name">:</span> <span class="token string">"Python: 当前文件"</span>,
            <span class="token string">"type"</span><span class="token builtin class-name">:</span> <span class="token string">"python"</span>,
            <span class="token string">"request"</span><span class="token builtin class-name">:</span> <span class="token string">"launch"</span>,
            <span class="token string">"program"</span><span class="token builtin class-name">:</span> <span class="token string">"<span class="token variable">${file}</span>"</span>,
            <span class="token string">"console"</span><span class="token builtin class-name">:</span> <span class="token string">"integratedTerminal"</span>,
            <span class="token string">"justMyCode"</span><span class="token builtin class-name">:</span> <span class="token boolean">false</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">]</span>
</code></pre>
    <p>
     保存之后，我们进入 debug.py，按下 F5 调试运行即可调试 python 源码了。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ad126bda5d6e99ad34af8f21c143b0a1.png#pic_center">
      <br/>
      可以看到代码会自动停在我们的断点处，并且左边调试栏会有中间变量的值。左上角则是调试选项，按下 F11 进入函数，或者点击调试选项的第三个功能，则会成功跳转到 pytorch 的 python 源码了。
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e532ed5ea45628078cc6a9715d61f979.png#pic_center">
      <br/>
      完成上面的步骤就可以进行 Python 代码的调试了。
     </img>
    </p>
    <h4>
     <a id="_vscode_C__52">
     </a>
     配置 vscode C++ 调试
    </h4>
    <p>
     想要继续进一步将上面的代码走人 Pytorch 的 C++ 源码，查看相关的调用栈等需要基于上面的 python 调试的配置，在其基础上进一步设置 launch.json。
    </p>
    <ul>
     <li>
      首先找到左侧调试状态栏，下拉菜单，选择
      <code>
       Add Configuration...
      </code>
      ：
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5cccd1db79fc63dabeee7dea258242d3.png#pic_center">
      <br/>
      然后在 launch.json 里面会自动调处以下选项，我们要选择
      <code>
       C/C++：(gdb) 附加
      </code>
      ：
     </img>
    </p>
    <img src="https://i-blog.csdnimg.cn/blog_migrate/1f9dff1d959e03222fdb7d5413d70613.png" width="60%">
     <p>
      这时 vscode 会在刚才 python 调试配置的基础上自动为我们添加相关默认内容：
     </p>
     <pre><code class="prism language-bash">        <span class="token punctuation">{<!-- --></span>
            <span class="token string">"name"</span><span class="token builtin class-name">:</span> <span class="token string">"(gdb) 附加"</span>,
            <span class="token string">"type"</span><span class="token builtin class-name">:</span> <span class="token string">"cppdbg"</span>,
            <span class="token string">"request"</span><span class="token builtin class-name">:</span> <span class="token string">"attach"</span>,
            <span class="token string">"program"</span><span class="token builtin class-name">:</span> <span class="token string">"输入程序名称，例如 <span class="token variable">${workspaceFolder}</span>/a.out"</span>,
            <span class="token string">"processId"</span><span class="token builtin class-name">:</span> <span class="token string">"<span class="token variable">${command<span class="token operator">:</span>pickProcess}</span>"</span>,
            <span class="token string">"MIMode"</span><span class="token builtin class-name">:</span> <span class="token string">"gdb"</span>,
            <span class="token string">"setupCommands"</span><span class="token builtin class-name">:</span> <span class="token punctuation">[</span>
                <span class="token punctuation">{<!-- --></span>
                    <span class="token string">"description"</span><span class="token builtin class-name">:</span> <span class="token string">"为 gdb 启用整齐打印"</span>,
                    <span class="token string">"text"</span><span class="token builtin class-name">:</span> <span class="token string">"-enable-pretty-printing"</span>,
                    <span class="token string">"ignoreFailures"</span><span class="token builtin class-name">:</span> <span class="token boolean">true</span>
                <span class="token punctuation">}</span>
            <span class="token punctuation">]</span>
        <span class="token punctuation">}</span>,
        <span class="token punctuation">..</span>.
</code></pre>
     <p>
      这里，我们仅需要将
      <code>
       "program"
      </code>
      修改成我们 python 解释器的路径就可以了，可以用
      <code>
       which python
      </code>
      命令来查看 python 解释器的路径，注意这个 python 解释器就是用来编译 Pytorch 源码安装的 python 环境，有多个虚拟环境的同学需要注意选择正确的 python 解释器。以我当前的环境为例，直接修改即可：
     </p>
     <pre><code class="prism language-bash"><span class="token string">"program"</span><span class="token builtin class-name">:</span> <span class="token string">"/home2/anaconda3/bin/python"</span>,
</code></pre>
     <p>
      这样我们就完成了 C++ 调试的环境配置，下面来看一下如何调试。
     </p>
     <h4>
      <a id="C__89">
      </a>
      C++ 调试
     </h4>
     <p>
      要从 .py 进入 C++ 还是需要刚才的 python 调试，让代码停住，然后 attach 到相关进程里面，这里还是以 debug.py 为例。
     </p>
     <p>
      首先我们在 debug.py 和想查看 C++ 文件中设置对应断点，这是调试前要设置好的。
     </p>
     <p>
      然后还是 F5 让代码先成功停在 python 断点处，然后下拉左上边的状态栏，可以看到我们刚才添加的 C++ 调试的 configuration 选项：
      <br/>
      <img src="https://i-blog.csdnimg.cn/blog_migrate/8de63a2290873b902e571e1878dbc356.png" width="80%">
       <br/>
       选择这个
       <code>
        (gdb) 附加
       </code>
       选项，并点击旁边的 Start Debugging 按钮，会需要我们选择进程：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ef8a0594eefbcaa3c29bc77d80e39649.png#pic_center">
        <br/>
        我们可以在其中搜索运行的文件名字，然后选择第二个：
        <br/>
        <img src="https://i-blog.csdnimg.cn/blog_migrate/4e5cd57c373c2b78c242fb6c9eada4d7.png" width="75%">
         <br/>
         这时右下方的
         <code>
          TERMINAL
         </code>
         会弹出一个提示，我们输入
         <code>
          y
         </code>
         , 并输入电脑密码即可：
         <br/>
         <img src="https://i-blog.csdnimg.cn/blog_migrate/758b22e4fdf29348baee434b8a0abe88.png" width="75%">
          <br/>
          然后左边的
          <code>
           CALL STACK
          </code>
          里面就会多了一个
          <code>
           (gdb) 附加
          </code>
          的内容，注意刚才的 python 调试栏一直没动，这个时候我们要点击
          <code>
           Python:当前文件
          </code>
          的
          <code>
           Continue
          </code>
          按钮：
          <br/>
          <img src="https://i-blog.csdnimg.cn/blog_migrate/09952278dd55dc553fcef2b49d2b2054.png" width="70%"/>
          <br/>
          稍等片刻，就可以得到下面的内容，可以看到 C++ 的调用栈，并且进入到 C++ 的源码里了：
          <br/>
          <img src="https://i-blog.csdnimg.cn/blog_migrate/c9758b120b2e3dfbdf5767e00fc83ca2.png" width="70%"/>
         </img>
        </img>
       </img>
      </img>
     </p>
