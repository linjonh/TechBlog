    <h2>
     <a id="_3">
     </a>
     出错场景
    </h2>
    <p>
     最近上课需要进行web开发，用tomcat要配置java环境，但是配置完之后发现命令行中运行java指令正常，但是javac就不行了。emmmm，，，？？？？？为啥啊。
     <br/>
     <img alt="java运行正常" src="https://i-blog.csdnimg.cn/blog_migrate/7bda3a5afcb3e2745d0c3c2bfb97e308.png"/>
    </p>
    <p>
     <img alt="javac报错" src="https://i-blog.csdnimg.cn/blog_migrate/5b5f2c1bc7caace895a1c7913cc6fe64.png"/>
    </p>
    <h2>
     <a id="_10">
     </a>
     探索
    </h2>
    <p>
     遇到问题，简单，先百度一下，果然有很多朋友和我遇到了一样的问题，应该可以解决了。
     <br/>
     可是，，，emmmm。
     <br/>
     大家出错好像只有两种：
     <br/>
     1.是环境变量名字写错。
     <br/>
     2.路径没搞对。
     <br/>
     解决方法：
     <br/>
     1.检查哪写错了。
     <br/>
     2.，，，，新手环境变量容易搞错，那卸了重装吧。
     <br/>
     emmmm，卸了重装还行，但是能de的bug为啥要重新写代码/dog。
     <br/>
     行嘛，我自己找错误嘛。
     <br/>
     可是我环境都是对的啊，而且java都跑起来了啊，这是咋回事啊，检查了几遍环境变量都是对的啊！！我甚至有的都是网上直接ctrl c来的啊。
     <br/>
     ！突然，再第n遍检查环境变量的时候，发现了path变量好像有点奇怪！
    </p>
    <p>
     <img alt="环境变量path" src="https://i-blog.csdnimg.cn/blog_migrate/975cbcd63ecc8f2ba298825c20f9be8f.jpeg"/>
    </p>
    <p>
     妈耶，好像好像不能加分号啊，，我为啥会打个分号上去呢，，
     <br/>
     ！对了，因为我是从教程复制过来的。网络上的教程大多是这样写的。
     <br/>
     <img alt="教程截图" src="https://i-blog.csdnimg.cn/blog_migrate/08505bad8d400d110db36e335fd12666.png">
      <br/>
      哦豁，写教程大多还是不分行的，而不分行的时候加环境变量需要分号隔开，但是现在win10好像编辑path的时候会帮你分行隔开…………
     </img>
    </p>
    <p>
     所以，不需要分号了！！！
    </p>
    <h2>
     <a id="_34">
     </a>
     解决
    </h2>
    <p>
     赶快把分号删了，，，果然就正常了。
     <br/>
     <img alt="正常了" src="https://i-blog.csdnimg.cn/blog_migrate/bd82887fd3a5d981891efdd68a2f8a5d.png"/>
    </p>
    <p>
     就是两个小小的分号忙了一晚上，，，，
    </p>
    <h2>
     <a id="_39">
     </a>
     总结
    </h2>
    <p>
     其实就是编辑path变量时不需要分号，但是因为是从网上教程复制来的所以带上了，删掉分号就行。
     <br/>
     这也算是环境变量编辑出错的一种，我怕自己敲错故意从网上复制来但是恰恰因为复制导致错误，导致纠错的时候半天找不到错误因为无比自信没有打错字，，，，，
     <br/>
     以后，还是懂了之后尽量自己打一边吧，又能加深印象，又能减少一些因为客观差异导致的问题。
    </p>
