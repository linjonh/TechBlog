  </div>

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <div class="markdown_views prism-github-gist" id="content_views">
    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     需求：我们有时候需要对项目进行代码，即执行maven的
     <code>
      package
     </code>
     指令。
    </p>
    <p>
     但maven在执行
     <code>
      package
     </code>
     指令之前，还会执行
     <code>
      clear
     </code>
     、
     <code>
      compile
     </code>
     、
     <code>
      test
     </code>
     等指令，前两个是我们能够接受的，但是执行
     <code>
      test
     </code>
     命令则会运行里面的单元测试代码，这是我们所不能接受的，因为我们不可能等到所有的单元测试代码执行完才打包。
    </p>
    <p>
     所以我们希望在执行
     <code>
      package
     </code>
     命令的时候跳过
     <code>
      test
     </code>
     命令。在 IDEA 中可以如下操作：
    </p>
    <ul>
     <li>
      <p>
       第一步，打开
       <code>
        File
       </code>
       ，选择
       <code>
        Setting...
       </code>
       。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f79e04bd56abba251b8326f81dd46b9b.png"/>
      </p>
     </li>
     <li>
      <p>
       第二步，选择
       <code>
        Build, Execution, Deployment
       </code>
       下的
       <code>
        Build Tools
       </code>
       下的
       <code>
        Maven
       </code>
       下的
       <code>
        Runner
       </code>
       面板，然后勾选
       <code>
        Skip Tests
       </code>
       即可。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d29fc9baf7ed1a34e5128b5217053daf.png"/>
      </p>
     </li>
     <li>
      <p>
       第三步，勾选完成后，保存退出后，再次打开 Maven 面板，发现
       <code>
        test
       </code>
       命令已经变成了灰色并且有了删除线。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e2c48e1bf7f454657f5b6add114580b7.png"/>
      </p>
     </li>
    </ul>
    <p>
     当然该
     <code>
      test
     </code>
     命令还是可以执行的，只需要双击即可。
    </p>
