    <h2>
     <a id="_0">
     </a>
     云原生的主要特征
    </h2>
    <p>
     进一年我们都在使用云原生框架
     <br/>
     SpringCloud微服务开发项目，敏捷快速
     <br/>
     部署在容器中，解决部署环境差异
     <br/>
     使用DevOps自动部署，减少运维压力
    </p>
    <h3>
     <a id="_5">
     </a>
     微服务
    </h3>
    <ul>
     <li>
      业务功能单一但完整
     </li>
     <li>
      只对外提供必要的服务接口
     </li>
     <li>
      轻量服务编排，灵活调度
     </li>
     <li>
      弹性扩缩
     </li>
     <li>
      便于开发以及快速更新
     </li>
    </ul>
    <h3>
     <a id="_11">
     </a>
     容器
    </h3>
    <ul>
     <li>
      资源使用效率高
     </li>
     <li>
      启动快，沙箱机制，容器集
     </li>
     <li>
      群编排调度
     </li>
     <li>
      容器集群的服务注册发现
     </li>
     <li>
      镜像分层和打包
     </li>
     <li>
      镜像仓库分发，队开发人员友好
     </li>
    </ul>
    <h3>
     <a id="DevOps_18">
     </a>
     DevOps
    </h3>
    <ul>
     <li>
      持续集成
     </li>
     <li>
      持续交付
     </li>
     <li>
      解决部署环境差异问题
     </li>
    </ul>
    <h2>
     <a id="_22">
     </a>
     云原生的发展
    </h2>
    <h3>
     <a id="_23">
     </a>
     传统
    </h3>
    <ul>
     <li>
      单节点
     </li>
     <li>
      手工部署
     </li>
    </ul>
    <h3>
     <a id="_26">
     </a>
     标准
    </h3>
    <ul>
     <li>
      虚拟化
     </li>
     <li>
      规范化
     </li>
    </ul>
    <h3>
     <a id="_29">
     </a>
     成熟
    </h3>
    <ul>
     <li>
      位置无关
     </li>
     <li>
      自动部署
     </li>
    </ul>
    <h3>
     <a id="_32">
     </a>
     云高效
    </h3>
    <ul>
     <li>
      容器化
      <br/>
      镜像小、启动速度快、资源浪费少
      <br/>
      更快的故障恢复、应用迁移、弹性扩展、应用版本升级回滚
     </li>
     <li>
      双活部署
      <br/>
      实现应用的快速安装部署
     </li>
    </ul>
    <h3>
     <a id="_38">
     </a>
     云高级
    </h3>
    <ul>
     <li>
      应用自描述
      <br/>
      基于标准应用建模，可跨云平台部署运行，不影响服务等级
     </li>
     <li>
      应用自维护
      <br/>
      故障自愈、自主负载均衡等
      <br/>
      实现基础设施无关的应用健壮性
     </li>
    </ul>
    <h3>
     <a id="Cloud_Native_44">
     </a>
     云原生（Cloud Native）
    </h3>
    <ul>
     <li>
      微服务化
      <br/>
      各服务独立升级维护
      <br/>
      各服务分别设置策略，控制粒度更精细，有利于把资源分配到特定的任务上，提高效率
     </li>
     <li>
      编排调度
      <br/>
      通过对多个微服务灵活编排，组装成不同的应用
     </li>
    </ul>
    <h3>
     <a id="_50">
     </a>
     云原生高级
    </h3>
    <ul>
     <li>
      云原生开发平台
      <br/>
      应用开发平台自动生成微服务架构，代码自动生成，自动创建微服务编排信息
     </li>
     <li>
      标准化框架、服务、控件
      <br/>
      风格统一，便于操作
     </li>
     <li>
      代码自动化生成
      <br/>
      自动对接DevOps系统，完成从应用微服务化、代码自动生成、到应用上架全程自动化
     </li>
    </ul>
    <h2>
     <a id="_57">
     </a>
     云原生高级规划
    </h2>
    <ul>
     <li>
      应用开发基于标准规范的开发平台
     </li>
     <li>
      使用标准框架、服务、UI控件、基础服务仓库
     </li>
     <li>
      应用引擎根据开发人员配置，自动生成所需微服务、微服务镜像定义文件、创建微服务编排信息
     </li>
     <li>
      开发人员只需要在微服务代码架构中做业务逻辑实现
     </li>
     <li>
      提交代码、编排信息到平台版本仓库，触发DevOps平台完成镜像创建、CI/CD、镜像上传和应用上架
     </li>
    </ul>
    <h2>
     <a id="_63">
     </a>
     容器
    </h2>
    <p>
     本质是一个与宿主机系统共享内核，但与系统中的其他进程资源相隔离的执行环境。
     <br/>
     Docker容器，是当前认可度高、社区和生态最活跃的开源容器技术。
    </p>
    <h3>
     <a id="_66">
     </a>
     优势
    </h3>
    <ul>
     <li>
      占用资源少，镜像小，启动快
     </li>
     <li>
      灵活的容器运行调度机制
     </li>
     <li>
      镜像机制，与运行环境无关，易于迁移复制
     </li>
     <li>
      沙箱机制，仅开放必要的端口提供访问
     </li>
     <li>
      允许实例简单、快速的创建、扩展、销毁
     </li>
    </ul>
    <h3>
     <a id="_72">
     </a>
     调度策略
    </h3>
    <ul>
     <li>
      默认选择最空闲的节点运行
     </li>
     <li>
      在指定的节点或标签节点运行
     </li>
     <li>
      亲和性（反亲和性）所有特定标签在同一节点运行
     </li>
     <li>
      每个节点运行单一实例
     </li>
    </ul>
    <h3>
     <a id="_77">
     </a>
     故障恢复策略
    </h3>
    <ul>
     <li>
      设置探测时间间隔
     </li>
     <li>
      设定故障检测方式：端口检测（TCP、HTTP）、进程检测
     </li>
     <li>
      设定容器数量限制，低于限值，则微服务对应容器重新创建
     </li>
    </ul>
    <h3>
     <a id="_81">
     </a>
     弹性扩缩策略
    </h3>
    <ul>
     <li>
      基于资源的弹性扩缩
      <br/>
      基于容器内存/CPU占用大小，如果持续超过指定时长，容器数量增加或减少。
     </li>
     <li>
      基于时间段的弹性扩缩
      <br/>
      基于时间段，在高峰/低峰调整容器数量，自动增加或减少
     </li>
    </ul>
    <h2>
     <a id="_86">
     </a>
     运维
    </h2>
    <p>
     DevOps
     <br/>
     Development + Operations 及 开发+操作
     <br/>
     核心理念就是消除开发和操作之间的隔阂，提高软件迭代和交付上线的速度和质量
    </p>
