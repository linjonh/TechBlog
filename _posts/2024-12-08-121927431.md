    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目一 华为云平台web架构部署
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      软件环境：华为云平台
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目需求：随着技术的不断发展和用户对网站功能性的需求不断提高，如今      网站项目的设计已经不能再仅仅简单地利用静态html文件来实现，       与前几年网站设计由一两名网页设计师自由的创作相比，网站项目的      设计和开发越来越像一个软件工程，也越来越复杂，网站项目的设计和      开发进入了需要强调流程和分工的时代，建立规范的、有效的、健壮的      开发机制，才能适应用户不断变化的需要，达到预期的计划目标。
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      运用技术：LNMP Docker Hwcloud
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目准备：1台云服务器作为跳板机，1台做镜像仓库，3台作为Web架构主机，两个弹性公网IP，弹性负载均衡ELB
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目思路：
     </strong>
    </p>
    <ol>
     <li style="text-align:justify;">
      <strong>
       准备配置华为云跳板机及配置镜像模板机
      </strong>
     </li>
     <li style="text-align:justify;">
      <strong>
       使用跳板机及华为云自带软件源部署集群架构基本包
      </strong>
     </li>
     <li style="text-align:justify;">
      <strong>
       购买3台Web云主机，安装配置Web集群
      </strong>
     </li>
     <li style="text-align:justify;">
      <strong>
       购买弹性负载均衡ELB,并且绑定公网IP
      </strong>
     </li>
    </ol>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目测试：
     </strong>
    </p>
    <ol>
     <li style="text-align:justify;">
      <strong>
       登录跳板机，通过跳板机对其他节点进行部署
      </strong>
     </li>
     <li style="text-align:justify;">
      <strong>
       客户端通过负载均衡，访问测试页面，检查轮询访问效果
      </strong>
     </li>
     <li style="text-align:justify;">
      <strong>
       高可用测试：模拟单台服务器故障的场景，测试故障切换效果
      </strong>
     </li>
    </ol>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目体会:
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      1
     </strong>
     <strong>
      、单台服务器提供服务会带来很大的风险 ,所以单点故障是架构中必须解决的问题。
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      2
     </strong>
     <strong>
      、节点服务器出现异常后 , ELB会自动剔除异常节点 ,在修复后自动添加 ,无需人工操作
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      项目收获：
     </strong>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <strong>
      随着用户访问量的增加为了使用户拥有更良好的体验，我们应该增加服务器的数量， 并保证服务器随时处于服务状态，不会出现单点故障，导致用户访问失败，因此我们将其升级为高可用负载均衡的架构。同时磨练了我认真负责的品质,增强了自我的团队合作性和工作的效率。
     </strong>
    </p>
