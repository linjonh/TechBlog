    <h2>
     <a id="Windows_GVim__YouCompleteMeYouCompleteMe_unavailable_unable_to_load_Python__1">
     </a>
     Windows GVim YouCompleteMe安装失败，YouCompleteMe unavailable: unable to load Python 问题的解决方法
    </h2>
    <h4>
     <a id="_2">
     </a>
     问题描述
    </h4>
    <ol>
     <li>
      Windows 10 操作系统
     </li>
     <li>
      使用vim-plug或者Vundle插件管理器安装YCM
     </li>
     <li>
      YCM安装时没有报错
     </li>
     <li>
      打开时显示如下：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bbe45255f69fe48fe641ff83ab5cb4a5.png"/>
     </li>
     <li>
      系统中已经装好Python，如果没有装好的话，先去装好Python，推荐使用Python3
     </li>
    </ol>
    <h4>
     <a id="_10">
     </a>
     问题分析
    </h4>
    <p>
     系统中已经装好Python，说明问题出现在GVim找不到Python的应用。经多方尝试，发现问题有以下两种可能：
    </p>
    <ol>
     <li>
      GVim的位数与Python的位数不一致，导致GVim无法调用Python
     </li>
     <li>
      GVim默认使用的版本号与安装的Python的版本号不一致，导致GVim调用了错误的文件。
     </li>
    </ol>
    <h4>
     <a id="_15">
     </a>
     问题解决
    </h4>
    <h5>
     <a id="_16">
     </a>
     位数不一致的问题
    </h5>
    <h6>
     <a id="GVim_17">
     </a>
     查看GVim位数的方法
    </h6>
    <ol start="0">
     <li>
      将GVim的可执行文件添加进系统的环境变量，这个用Vim 的应该已经干了（不会的自己百度搜去）
     </li>
     <li>
      打开cmd.exe（ 还不会打开的百度搜去）。
     </li>
     <li>
      输入gvim --version
     </li>
     <li>
      如图，
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8b6dc453660d3f7a0518c54c326bd5dc.png">
       <br/>
       这个就是64bit版本，没有看见64bit 的就是32bit版本
      </img>
     </li>
    </ol>
    <h6>
     <a id="Python_23">
     </a>
     查看Python位数的方法
    </h6>
    <ol start="0">
     <li>
      将GVim的可执行文件添加进系统的环境变量，这个用Vim 的应该已经干了（不会的自己百度搜去）
     </li>
     <li>
      打开cmd.exe（ 还不会打开的百度搜去）。
     </li>
     <li>
      输入python 回车
     </li>
     <li>
      如图，
     </li>
     <li>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/300af3fe2aa6a516a5165af7ddc5089a.png">
       <br/>
       这个就是64bit版本，没有看见64bit 的就是32bit版本
      </img>
     </li>
    </ol>
    <h6>
     <a id="_31">
     </a>
     位数不一样的处理方法
    </h6>
    <p>
     推荐都使用64bit 的版本，所以处理方法是那个是32bit 的版本就将那个卸载然后重装成64bit 的版本。
     <br/>
     Python可以从官网
     <a href="https://www.python.org/downloads/" rel="nofollow">
      https://www.python.org/downloads/
     </a>
     下载。
     <br/>
     GVim 可以从GitHub下载，
     <a href="https://github.com/vim/vim-win32-installer/releases">
      https://github.com/vim/vim-win32-installer/releases
     </a>
     下载。
     <br/>
     下载的时候选择完整版，不要选择精简版。
     <br/>
     记得选择带有x64字样的。
    </p>
    <h5>
     <a id="GVimPython_37">
     </a>
     GVim找不到Python的处理方法
    </h5>
    <p>
     验证GVim和Python 的位数对应之后进行此步。
    </p>
    <h6>
     <a id="GVim_39">
     </a>
     查看GVim版本
    </h6>
    <p>
     方法上面有，
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0ba3c715b57ded58b2060f09bccb77aa.png">
      <br/>
      看到上面的python3/dyn说明支持Python，且是动态加载。
      <br/>
      输入
      <code>
       echo has('python3')
      </code>
      显示0，说明GVim找不到Python。
     </img>
    </p>
    <h6>
     <a id="GVimPython_45">
     </a>
     配置GVim找到Python文件
    </h6>
    <p>
     查看GVim默认找的Python文件位置。
     <br/>
     输入`:set pythonthreedll?
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/691c7d3b6872c3c907743bc88fc9ae6a.png">
      <br/>
      看显示
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e521d4dd0e35bb5eab7bca488356be60.png">
       <br/>
       这里和你的Python版本是对不上的。Python版本怎么看上面已经有了。
      </img>
     </img>
    </p>
    <h6>
     <a id="Python_53">
     </a>
     修改为正确的Python版本
    </h6>
    <p>
     首先将Python的路径添加到系统的环境变量中，并确定该文件位置有一个叫做python3x.dll的文件，其中x是一个数字。
     <br/>
     验证一下，在控制台中输入python3x.dll（x用你那里的数字代替）
     <br/>
     显示
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6e082e167d6d5868e73e5fa25a95438f.png">
      <br/>
      说明系统已经可以找到这个文件了。
     </img>
    </p>
    <p>
     去GVim的安装路径中找_vimrc这个文件，这个是vim的配置文件，在里面加一行
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/17583b432f4a6caf3e4976f7d8c11822.png">
      <br/>
      注意这里文件的名字要和你的上面找到的文件名称相对应。保存并退出。
      <br/>
      如果不愿意将Python添加进环境变量中的话，可以在这里写dll文件的完整路径比如
      <code>
       C:\your-path-to-Python\python3x.dll
      </code>
     </img>
    </p>
    <p>
     再打开一下GVim，就会发现报错没有了。
    </p>
