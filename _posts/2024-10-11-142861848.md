    <h2>
     <a id="_1__7">
     </a>
     第 1 章：软件工程学概述
    </h2>
    <h3>
     <a id="_8">
     </a>
     软件危机
    </h3>
    <p>
     定义、表现（成本进度估计不准、用户不满、质量差等）及原因（逻辑部件、规模大等）。
    </p>
    <h3>
     <a id="_10">
     </a>
     软件工程
    </h3>
    <p>
     定义为从管理和技术研究软件开发维护的学科，有关注大型程序等本质特性。
     <br/>
     通过正确认识、当作工程项目等方法消除软件危机。
    </p>
    <h3>
     <a id="_13">
     </a>
     软件过程与模型
    </h3>
    <p>
     软件过程是任务框架，与软件工程方法学相关。
     <br/>
     介绍瀑布、快速原型等模型的优缺点及适用范围。
    </p>
    <h2>
     <a id="_2__16">
     </a>
     第 2 章：可行性研究
    </h2>
    <h3>
     <a id="_17">
     </a>
     研究目的与方面
    </h3>
    <p>
     早期用最小代价确定问题解法是否可行，从技术、经济、操作三方面研究。
    </p>
    <h3>
     <a id="_19">
     </a>
     案例分析
    </h3>
    <p>
     对银行储蓄、机票预订、患者监护系统进行问题定义和可行性分析。
    </p>
    <h2>
     <a id="_3__21">
     </a>
     第 3 章：需求分析
    </h2>
    <h3>
     <a id="_22">
     </a>
     分析原因与需求类型
    </h3>
    <p>
     为开发满足用户需求软件，有功能、性能等多种需求。
    </p>
    <h3>
     <a id="_24">
     </a>
     获取需求方法
    </h3>
    <p>
     访谈（正式、非正式）是常用获取需求技术。
    </p>
    <h3>
     <a id="_26">
     </a>
     案例分析
    </h3>
    <p>
     对银行储蓄等系统描绘数据流图和实体联系图。
    </p>
    <h2>
     <a id="_4__28">
     </a>
     第 4 章：形式化说明技术
    </h2>
    <h3>
     <a id="_29">
     </a>
     方法对比
    </h3>
    <p>
     欠形式化方法有矛盾、二义性等缺点，形式化方法有准确等优点。
    </p>
    <h3>
     <a id="_31">
     </a>
     应用情况
    </h3>
    <p>
     在自然语言有二义性时使用，需遵守选用方法等准则。
    </p>
    <h3>
     <a id="_33">
     </a>
     案例分析
    </h3>
    <p>
     对浮点二进制数判断及图书馆流通系统进行形式化说明。
    </p>
    <h2>
     <a id="_5__35">
     </a>
     第 5 章：总体设计
    </h2>
    <h3>
     <a id="_36">
     </a>
     模块耦合与内聚
    </h3>
    <p>
     分别举例说明数据、控制等耦合类型和逻辑、时间等内聚类型。
    </p>
    <h3>
     <a id="_38">
     </a>
     软件结构设计
    </h3>
    <p>
     用面向数据流方法设计储蓄、机票预订、患者监护系统软件结构。
    </p>
    <h2>
     <a id="_6__40">
     </a>
     第 6 章：详细设计
    </h2>
    <h3>
     <a id="_41">
     </a>
     控制结构转换
    </h3>
    <p>
     如用 SEQUENCE 和 DO - WHILE 实现 IF - THEN - ELSE 操作。
    </p>
    <h3>
     <a id="_43">
     </a>
     程序流程图和盒图
    </h3>
    <p>
     绘制相关伪码程序的流程图和盒图，并分析程序是否结构化。
    </p>
    <h2>
     <a id="_7__45">
     </a>
     第 7 章：实现
    </h2>
    <h3>
     <a id="_46">
     </a>
     伪码错误分析
    </h3>
    <p>
     找出字频统计伪码错误并分析语法特点。
    </p>
    <h3>
     <a id="_48">
     </a>
     测试用例设计
    </h3>
    <p>
     对程序设计语句覆盖和路径覆盖测试用例，及对系统设计正常和非正常操作测试数据。
    </p>

     <a id="_8__50">
     </a>
     第 8 章：维护
    </h2>
    <h3>
     <a id="_51">
     </a>
     可维护性因素
    </h3>
    <p>
     与可理解性、可测试性等因素有关。
    </p>
    <h3>
     <a id="_53">
     </a>
     维护措施
    </h3>
    <p>
     建立目标、使用先进技术等提高可维护性。
    </p>
    <h3>
     <a id="_55">
     </a>
     案例分析
    </h3>
    <p>
     对储蓄、机票预订、患者监护系统考虑维护相关问题。
    </p>

     <a id="_11__65">
     </a>
     第 11 章：面向对象设计
    </h2>
    <h3>
     <a id="_66">
     </a>

    </h3>
