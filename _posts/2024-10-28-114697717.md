    <h3>
     <a id="1_0">
     </a>
     1.概述
    </h3>
    <p>
     最近在绘制UML类图时，发现一个比较好用的绘制UML的工具PlantUML，因为项目使用的开发工具是VSCode，所以本文在VSCode上安装PlantUML插件来绘制UML类图。
    </p>
    <h3>
     <a id="2_3">
     </a>
     2.环境搭建
    </h3>
    <p>
     VSCode的安装比较简单，这里不再介绍。
     <br/>
     配置PlantUML插件
     <img alt="![在这里插入图片描述](https://img-blog.csdnimg.cn/20210312153549591.png" src="https://i-blog.csdnimg.cn/blog_migrate/a791263b09052c0065b2a052c8b7d53c.png">
      <br/>
      在VSCode插件列表中选择PlantUML安装。
      <br/>
      由于PlantUML的插件是 plantUML.jar，所以需要按张Java JDK来支持Java开发环境。
      <br/>
      此外PlantUML只能生成sequence图，其它图需要依赖Graphviz引擎，否则会报错误如下：
     </img>
    </p>
    <blockquote>
     <p>
      Dot executable is /opt/local/bin/dot Error: File does not exist
     </p>
    </blockquote>
    <p>
     安装JDK比较简单，可参考网上方法，这里不介绍
     <br/>
     安装Graphviz，在Ubuntu环境下执行：
    </p>
    <pre><code class="prism language-powershell">sudo apt<span class="token operator">-</span>get update  <span class="token comment">#无此操作可能出现 E: Package 'graphviz' has no installation candidate 错误</span>
sudo apt install graphviz
</code></pre>
    <p>
     总结下，PlantUML需要安装：
    </p>
    <ol>
     <li>
      java JDK
     </li>
     <li>
      graphviz
     </li>
    </ol>
    <h3>
     <a id="2UML_24">
     </a>
     2.绘制UML类图
    </h3>
    <p>
     首先新建一个*.wsd, *.pu, *.puml, *.plantuml, *.iuml的文件。此文件用于生成UML图形，执行ALT+D可生成对应的UML类图。
     <br/>
     类的关系说明
    </p>
    <pre><code class="prism language-xml">泛化(继承) &lt;|- -
关联 &lt;- -
组合 *- -
聚合 o–
实现 &lt;|- -
依赖 &lt; . .
</code></pre>
    <p>
     如：
    </p>
    <pre><code class="prism language-xml">@startuml
class1 &lt;|-- class2:泛   化
class3 &lt;-- class4:关   联
class5 *-- class6:组   合
class7 o-- class8:聚   合
class9 &lt;|-- class10:实   现
class11 &lt;.. class12:依   赖
@enduml
</code></pre>
    <p>
     生成的uml类图如下
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e1e7d27d2a532fdf8246edf1321e34c0.png">
      <br/>
      具体类的成员变量及成员函数如下
     </img>
    </p>
    <pre><code class="prism language-xml">@startuml
class class1
{
    +void operation1();
    +int operation2(int i);

    -int attribute1;
    -char attribute2;
}

class class2
{
    +void operation1();
    +int operation2(int i);

    -int attribute1;
    -char attribute2;
}

class1 &lt;|-- class2 
@enduml
</code></pre>
    <p>
     注意class 与class之间不能有" ; " ,否则会生成错误。
     <br/>
     生成图形为：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a6f3d8d0ab6aabe4c7e2b3ee2392586f.png"/>
    </p>
    <h3>
     <a id="3_79">
     </a>
     3.导出图像
    </h3>
    <p>
     VSCode完成后的绘图，可以导出多种格式，通过 Ctrl + Shift + P，打开命令面板，输入 Export Current Diagram 命令后选择图片格式后便可导出。
    </p>
    <h3>
     <a id="4__82">
     </a>
     4. 总结
    </h3>
    <p>
     PlantUML采用这种写代码的方式来绘制UML类图，相比于传统其他需要拖动对应图像控件，编辑类成员函数及变量的方式，效率提升很多，必竟我们直接可以从头文件中复制出类的声明，而无需在控件上一条一条的编辑。
    </p>
