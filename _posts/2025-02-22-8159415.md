    <span style="font-size:18px;">
     <span style="font-family:宋体;">
      最近一起在忙于宝宝的事情，所以又有三个月没有更新博客了。有感而发想要写点东西，目前移动的大潮真的是无法阻挡，而移动浪潮的兴起其实本质其实娱乐平台的微型化与随身化，手机不仅仅承载着通讯平台，其外延已经延伸到了
     </span>
     <span lang="EN-US">
      LBS
     </span>
     <span style="font-family:宋体;">
      <span style="font-family:宋体;">
       、娱乐等等方面。
       <span lang="EN-US">
       </span>
      </span>
     </span>
    </span>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-size:18px;">
      <span style="font-family:宋体;">
       <span style="font-family:宋体;FONT-SIZE: 10.5pt">
        随着
       </span>
      </span>
      <span lang="EN-US">
       WINDOWS8
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        的上线，尤其是微软
       </span>
      </span>
      <span lang="EN-US">
       SURFACE
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        平板的推出，传统的桌面平台与移动平台又开始真正出现了融合的迹象，但是从目前看依然没有哪个操作系统或者说游戏框架可以一统天下，手机平台及娱乐平台的的种类相对比较多。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-size:18px;">
      <span style="font-family:宋体;">
       <span style="font-family:宋体;FONT-SIZE: 10.5pt">
        目前的情况看总的来说目前
       </span>
      </span>
      <span lang="EN-US">
       PC
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        平台上的游戏应用依旧是最多，但是
       </span>
      </span>
      <span lang="EN-US">
       PC
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        平台更多情况下是给开发者的游戏带来更高的人气，
       </span>
      </span>
      <span lang="EN-US">
       PC
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        平台上的游戏很难直接给开发者带来真正的收益。家用游戏机平台等基本上是被大型的游戏公司所垄断，你几乎很少见有在
       </span>
      </span>
      <span lang="EN-US">
       XBOX360
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        上玩什么连连看这种游戏的情况，而大型专业的游戏公司大部分利润均来自于家用游戏机平台。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-family:宋体;">
      <span style="font-family:宋体;FONT-SIZE: 10.5pt">
       <span style="font-size:18px;">
        而类似于愤怒的小鸟、水果忍者等中小型游戏一般是在移动游戏平台上一炮而红，由于移动设备由于是苹果的软件市场上线后，也为众多创业者提供了平台，而从市场细分的角度上来说，游戏市场上只有大型公司才可以在专业游戏平台上赚钱的模式被完全打破了。游戏或者说是娱乐软件市场开始向小的技术团队打开了大门。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-size:18px;">
      <span style="font-family:宋体;">
       <span style="font-family:宋体;FONT-SIZE: 10.5pt">
        总的来说目前整个终端市场中
       </span>
      </span>
      <span lang="EN-US">
       PC
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        平台主要是为了赚取人气，传统游戏平台依旧是被大游戏厂商所垄断，而移动平台则为中小创业团队提供了大把的机会。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-size:18px;">
      <span style="font-family:宋体;">
       <span style="font-family:宋体;FONT-SIZE: 10.5pt">
        最近在
       </span>
      </span>
      <span lang="EN-US">
       CSDN
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        上看到有很多人在讨论程序员比较高的工资待遇是否会终结，其实我想说其实程序员的工资与其工作时间相比未见得很高，代码民工一类的说法不见得没有道理。那么我们就要反思到底是什么原因造成软件工程师的群体班越加越多，薪水还要越降越低，与公司谈判的筹码越来越少的情况。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p class="p0" style="TEXT-INDENT: 21pt">
     <span style="font-size:18px;">
      <span style="font-family:宋体;">
       <span style="font-family:宋体;FONT-SIZE: 10.5pt">
        其实其实对于广大程序员朋友来说其实总是讨论该选择安卓还是选择
       </span>
      </span>
      <span lang="EN-US">
       IOS
      </span>
      <span style="font-family:宋体;">
       <span style="font-family:宋体;">
        这样的问题并没有太多的意义，那么我们接下来的一系列内容将就移动大潮中的程序员的切入点，也就是在项目所扮演的角色展开讨论，当然我关注的是技术人员在项目中所扮演的角色，而技术人员如何转变为管理人员等方面的问题请关注我之前写的博客《程序员的转型》。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
