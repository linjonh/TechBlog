    <p>
     基本概念：testng ,http, json, mysql， jenkins, spring
    </p>
    <p>
     陆陆续续搭建起来的接口测试框架，使用起来并不是特别的理想，所以走上了一条迭代优化的
     <strong>
      不归路
     </strong>
     。
     <br/>
     所谓的框架是指，用例的组织，批量运行，测试报告，参数化配置。
    </p>
    <h3>
     <a id="_5">
     </a>
     测试框架应该提供哪些功能？
    </h3>
    <p>
     单个功能点的测试
     <br/>
     多个用例的组织
     <br/>
     日志记录
     <br/>
     报告展示
    </p>
    <p>
     1.0需求： http请求发送， json解析， json格式校验， 数据库数据访问，持续集成部署。
     <br/>
     2.0需求: motan协议测试，dubbo协议测试，mq测试
    </p>
    <h4>
     <a id="http_14">
     </a>

    </h4>
    <p>
     HttpClient自己封装-----丑
     <br/>
     unirest–还是不理想
     <br/>
     rest-assured—目前的解决方案
    </p>
    <h4>
     <a id="json_19">
     </a>
     json解析
    </h4>
    <p>
     fastjson
    </p>
    <h4>
     <a id="json_21">
     </a>
     json格式校验
    </h4>
    <p>
     json schema
    </p>
    <h4>
     <a id="sql_24">
     </a>
     数据库访问和sql执行
    </h4>
    <p>
     jdbc
    </p>
    <h4>
     <a id="_27">
     </a>
     持续集成，用例的批量运行
    </h4>
    <p>
     jenkins，mvn test
    </p>
    <h4>
     <a id="testng_30">
     </a>
     testng
    </h4>
    <p>
     很好的测试框架
     <br/>
     监听器:通过监听器实现失败用例重跑，实现用例执行结果入库，失败用例钉钉群告警。
     <br/>
     注解
     <br/>
     依赖注入：testng + spring 实现依赖注入。
     <br/>
     断言检查： assertj
    </p>
    <h4>
     <a id="springboot_springboot_37">
     </a>
     springboot 用例服务平台后端使用springboot
    </h4>
    <p>
     JSON api
     <br/>
     mybatis mapper封装增删查改，需要手动写的是多表和复合查询
    </p>
    <h4>
     <a id="react__antd_react_41">
     </a>
     react + antd 用例服务平台前端使用react搭建
    </h4>
    <p>
     搜索表单
     <br/>
     新增和修改Modal框
     <br/>
     删除按钮
     <br/>
     表格：分页器，total， pageSize, pageNum
     <br/>
     图表展示
    </p>
    <h4>
     <a id="_48">
     </a>
     日志记录
    </h4>
    <p>
     apache的log4j是日志的一种实现
    </p>

private static Logger log = Logger.getLogger(XXX.class);

log4j.properties文件

### 输出信息到控制抬 ###
