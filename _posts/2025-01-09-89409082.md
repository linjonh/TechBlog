    <h4>
     1. C/C++ 程序员的数量
    </h4>
    <p>
     全球大约 440 万 C++ 程序员；大约 190 万 C 程序员。
    </p>
    <p>
     <img alt="" class="has" id="pic" src="https://i-blog.csdnimg.cn/blog_migrate/e9edc1c6677b4575080702c55728ba38.png"/>
    </p>
    <h4>
     2. C++ 比 Python 的程序员数量高一些，C 和 Ruby 的程序员数量相当
    </h4>
    <p>
     JetBrains 估算了全球使用热门编程语言的开发者数量：
    </p>
    <ul>
     <li>
      Java 有 900 万；
     </li>
     <li>
      C# 大约 780 万；
     </li>
     <li>
      JavaScript 大约 760 万；
     </li>
     <li>
      PHP 将近 600 万；
     </li>
     <li>
      C++ 大约 440 万；
     </li>
     <li>
      Python 有 400 万；
     </li>
     <li>
      Objective-C 大约 330 万；
     </li>
     <li>
      C 大约 190 万；
     </li>
     <li>
      Ruby 大约 180 万；
     </li>
    </ul>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/f2097c1886e4e5a53557ecb4d99fee34.png"/>
    </p>
    <h4>
     3. C++ 程序员在全球的分布
    </h4>
    <ul>
     <li>
      <ul>
       <li>
        EMEA 地区有 150 万；
        <em>
         （伯乐在线注：EMEA 是 Europe, the Middle East and Africa 的字母缩写，为欧洲、中东、非洲三地区的合称，通常是用作政府行政或商业上的区域划分方式，这种用法较常见于北美洲的企业。）
        </em>
       </li>
       <li>
        亚太地区有 150 万；
       </li>
       <li>
        北美地区有 100 万；
       </li>
       <li>
        拉丁美洲约 40 万；
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/1aaaaf6c717c59c27d6bc13498ab7c02.png"/>
    </p>
    <h4>
     4. C++ 更为流行的国家
    </h4>
    <p>
     相比其他语言，C++ 在俄罗斯、捷克、匈牙利、德国、法国、芬兰、以色列和新加坡更流行。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/381cd8d42962af00d7d2fa3a97f1af81.png"/>
    </p>
    <h4>
     5. 哪些行业在用 C++ 语言
    </h4>
    <p>
     分析 C++ 的招聘广告信息得出的结果：前三的行业是：
     <strong>
      金融、银行和游戏
     </strong>
     。接下来是：Front Office、通信、电子、投行、市场、制造业、零售业。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/1016b7ed1b3fb8badc9daf270162df1b.png"/>
    </p>
    <p>
     <em>
      【伯乐在线补充】：Front Office 是相对于 Back Office 而言。所谓 Front Office 软件即指应用在销售部门、客户服务、电话中心(call center)、和技术支援方面的软件，Front Office软件的目的在提升企业的销售、行销、与客户服务的效率。
     </em>
    </p>
    <h4>
     6. 哪些编程语言和 C++ 一起在用
    </h4>
    <p>
     根据分析的招聘广告信息，经常和 C++ 一起在用的语言有：Java、C、C#、Python、SQL 和 JS。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/e9d0e2706e1dd9766ff77b9c4064bffd.png"/>
    </p>
    <h4>
     7. C++ 程序员所用的操作系统
    </h4>
    <p>
     Linux 44%、Windows 39%、OS X 17%；
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/d7c9c01614de2e9fa99f6904b8d9adb9.png"/>
    </p>
    <h4>
     8. C++ 的版本
    </h4>
    <p>
     已经有 34% 的 C++ 程序员在用 C++11；
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/b24e71192c9689d9fcd8682774495db3.png"/>
    </p>
    <h4>
     9. 最流行的 C++ 编译器
    </h4>
    <p>
     GCC 遥遥领先，65% 份额；
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/15fe32c68390e522b37734d07ea86906.png"/>
    </p>
    <h4>
     10. Windows 平台上最流行的 C++ 编译器
    </h4>
    <p>
     Windows 平台上，VC++ 虽然是第一（36%），但只比 GCC 领先 2%。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/8f5e9fb45dedff5dfc61cd350f6b9b68.png"/>
    </p>
    <blockquote>
     <p>
      插播一条简讯：微软在11月释出的Visual C++更新将整合
      <a href="http://www.theregister.co.uk/2015/10/21/microsoft_promises_clang_for_windows_in_november_visual_c_update/" rel="nofollow">
       Clang开源C和C++编译器
      </a>
      ，开发者将可以用Clang编译Windows应用程序。Visual Studio 2015 已经整合了Clang编译器，但它是被用于Android和 iOS上的应用开发，而这一次整合是用之开发Windows程序。
     </p>
     <p>
      Visual C++由多部分构成，其中包括前端编译器，用于生成代表程序内容的抽象语法树；后端编译器则读取抽象语法树生成执行代码。前端编译器称为C1，后端称为 C2，由微软的不同团队开发。最新的整合将用Clang取代C1，后端继续用C2。微软推出Clang/C2 for Windows的目的是改进跨平台支持。（中文摘要来自
      <a href="http://www.solidot.org/story?sid=45898" rel="nofollow">
       Solidot
      </a>
      ）
     </p>
    </blockquote>
    <h4>
     11. 最流行的 C++ 调试器
    </h4>
    <p>
     如果不划分开发平台，GDB7 占据了 62%，甩了其他调试器很多条街了。不过在 OS X 平台上的数据就完全不同了，LLDB 是 39% 领先，GDB7 32% 落后。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/45652c23ac87a7aedaf15e14950f1f16.png"/>
    </p>
    <h4>
     12. 最流行的 C++ 构建系统
    </h4>
    <p>
     CMake 和 GNU Make 夺走了 68% C++ 的心；
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/3325105af4b65acdc34257f84f1779e4.png"/>
    </p>
    <h4>
     13. 编译器 + 调试器 + 构建工具组合
    </h4>
    <p>
     GCC + CMake + GDB7 组合是 C++ 程序员的最爱。不过在 OS X 平台，这个荣誉要让给 Clang + CMake + LLDB。
    </p>
    <p>
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/b40e34491d721a6cd8ea4620a0924e6f.png"/>
