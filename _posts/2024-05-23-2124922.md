    <div>
     <strong>
      <span style="FONT-SIZE: 14pt">
       嵌入式数据库：
      </span>
     </strong>
    </div>
    <div style="MARGIN-LEFT: 18pt; TEXT-INDENT: -18pt">
     <strong>
      <span style="FONT-SIZE: 12pt">
       1.
       <span style="FONT: 7pt 'Times New Roman'">
       </span>
      </span>
     </strong>
     <strong>
      <span style="FONT-SIZE: 12pt">
       嵌入在进程中执行，不需要单独引擎。
      </span>
     </strong>
    </div>
    <div style="MARGIN-LEFT: 18pt; TEXT-INDENT: -18pt">
     <strong>
      <span style="FONT-SIZE: 12pt">
       2.
       <span style="FONT: 7pt 'Times New Roman'">
       </span>
      </span>
     </strong>
     <strong>
      <span style="FONT-SIZE: 12pt">
       可定制、体积小，满足嵌入式系统需求。
      </span>
     </strong>
    </div>
    <div>
     <strong>
     </strong>
    </div>
    <div>
     <strong>
      <span style="FONT-SIZE: 12pt">
       Berkeley DB
      </span>
     </strong>
     <strong>
      <span style="FONT-SIZE: 12pt">
       （开源，商用收费）
      </span>
     </strong>
    </div>
    <div>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      技术特点
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      1. Berkeley DB
     </span>
     <span style="FONT-SIZE: 12pt">
      是一个开放源代码的内嵌式数据库管理系统，能够为应用程序提供高性能的数据管理服务。应用它程序员只需要调用一些简单的API就可以完成对数据的访问和管理。(不使用SQL语言)
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      2. Berkeley DB
     </span>
     <span style="FONT-SIZE: 12pt">
      为许多编程语言提供了实用的API接口，包括C、C++、Java、Perl、Tcl、Python和PHP等。所有同数据库相关的操作都由Berkeley DB函数库负责统一完成。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      3. Berkeley DB
     </span>
     <span style="FONT-SIZE: 12pt">
      轻便灵活（Portable），可以运行于几乎所有的UNIX和Linux系统及其变种系统、Windows操作系统以及多种嵌入式实时操作系统之下。Berkeley DB被链接到应用程序中，终端用户一般根本感觉不到有一个数据库系统存在。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      4. Berkeley DB
     </span>
     <span style="FONT-SIZE: 12pt">
      是可伸缩（Scalable）的，这一点表现在很多方面。Database library本身是很精简的（少于300KB的文本空间），但它能够管理规模高达256TB的数据库。它支持高并发度，成千上万个用户可同时操纵同一个数据库。Berkeley DB能以足够小的空间占用量运行于有严格约束的嵌入式系统。
     </span>
    </div>
    <div style="TEXT-INDENT: 24pt; LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      Berkeley DB
     </span>
     <span style="FONT-SIZE: 12pt">
      在嵌入式应用中比关系数据库和面向对象数据库要好，有以下两点原因：
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
     </span>
     <span style="FONT-SIZE: 12pt">
      （1）因为数据库程序库同应用程序在相同的地址空间中运行，所以数据库操作不需要进程间的通讯。在一台机器的不同进程间或在网络中不同机器间进行进程通讯所花费的开销，要远远大于函数调用的开销；
     </span>
    </div>
    <div style="TEXT-INDENT: 24pt; LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      （2）因为Berkeley DB对所有操作都使用一组API接口，因此不需要对某种查询语言进行解析，也不用生成执行计划，大大提高了运行效。
     </span>
    </div>
    <div style="TEXT-INDENT: 24pt">
    </div>
    <div>
     <strong>
      <span style="FONT-SIZE: 12pt">
       SQLite
      </span>
     </strong>
     <strong>
      <span style="FONT-SIZE: 12pt">
       （开源，商用免费）
      </span>
     </strong>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      轻量级别数据库SQLite的主要特点：
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      1.
     </span>
     <span style="FONT-SIZE: 12pt">
      支持事件，不需要配置，不需要安装，也不需要管理员；
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      2.
     </span>
     <span style="FONT-SIZE: 12pt">
      支持大部分SQL92；
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      3.
     </span>
     <span style="FONT-SIZE: 12pt">
      一个完整的数据库保存在磁盘上面一个文件，同一个数据库文件可以在不同机器上面使用，最大支持数据库到2T，字符和BLOB的支持仅限制于可用内存；
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      4.
     </span>
     <span style="FONT-SIZE: 12pt">
      整个系统少于3万行代码，少于250KB的内存占用(gcc)，大部分应用比目前常见的客户端/服务端的数据库快，没有其它依赖
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      5.
     </span>
     <span style="FONT-SIZE: 12pt">
      源代码开放，代码95%有较好的注释，简单易用的API。官方带有TCL的编译版本。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <strong>
      <span style="FONT-SIZE: 12pt">
       Empress
      </span>
     </strong>
     <strong>
      <span style="FONT-SIZE: 12pt">
       （商业数据库）
      </span>
     </strong>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <strong>
     </strong>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      开发阶段特点：
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      1.
     </span>
     <span style="FONT-SIZE: 12pt">
      可嵌入程序，该特性使应用程序和数据库工作于统一地址空间，增强了系统的稳定性，提高了系统的效率。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      2.
     </span>
     <span style="FONT-SIZE: 12pt">
      确定的响应时间，Empress 可以使数据的响应时间相对一致，使用者可以设定一个超时限制，如果在规定时间内没有完成插入，修改等操作，系统会报错。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      3.
     </span>
     <span style="FONT-SIZE: 12pt">
      快速的操作Empress 提供了内核级的CAPI,称为MR, 用MR编写的应用程序在执行时不需要解析。另外在MR中加速的机制还包括优秀的加锁控制，内存管理和基于记录数量的选择功能。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      4.
     </span>
     <span style="FONT-SIZE: 12pt">
      灵活的开发方式，Empress 提供多种开发接口，加快开发进程而无需开发者重新学习开发语言和熟悉开发环境。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      5.
     </span>
     <span style="FONT-SIZE: 12pt">
      友好的存储方式，Empress 数据库可以放在操作系统支持的任何存储设备中，Empress的表单甚至可以分割放在不同的存储设备中，比如在内存，硬盘和CD-ROM中。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      6.
     </span>
     <span style="FONT-SIZE: 12pt">
      微型内核结构 Empress 高度单元化, 可根据需要选择需要的单元，从而缩小产品中Empress 数据库所占用的资源。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      7.
     </span>
     <span style="FONT-SIZE: 12pt">
      宽广的平台支持，Empress 支持多种硬件平台和软件平台, 也可移植到客户要求的硬件平台或操作系统。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      技术优势：
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      1.
     </span>
     <span style="FONT-SIZE: 12pt">
      微型内核结构，占用少量内存空间，特别适合紧凑性的设计
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      2
     </span>
     <span style="FONT-SIZE: 12pt">
      ．
     </span>
     <span style="FONT-SIZE: 12pt">
      一周
     </span>
     <span style="FONT-SIZE: 12pt">
      7
     </span>
     <span style="FONT-SIZE: 12pt">
      天，每天
     </span>
     <span style="FONT-SIZE: 12pt">
      24
     </span>
     <span style="FONT-SIZE: 12pt">
      小时连续工作，无需任何额外操作免维护
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      3.
     </span>
     <span style="FONT-SIZE: 12pt">
      内核级
     </span>
     <span style="FONT-SIZE: 12pt">
      CAPI
     </span>
     <span style="FONT-SIZE: 12pt">
      接口，使运行速度最大化
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      4.
     </span>
     <span style="FONT-SIZE: 12pt">
      高度灵活的
     </span>
     <span style="FONT-SIZE: 12pt">
      SQL
     </span>
     <span style="FONT-SIZE: 12pt">
      接口
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      5.
     </span>
     <span style="FONT-SIZE: 12pt">
      优秀的掉电恢复能力
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      6.
     </span>
     <span style="FONT-SIZE: 12pt">
      强壮的交易和锁存机制
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      7.
     </span>
     <span style="FONT-SIZE: 12pt">
      支持
     </span>
     <span style="FONT-SIZE: 12pt">
      SCSI,RAID,IDE,RAM,CD-RW,DVD-ROM,CF,
     </span>
     <span style="FONT-SIZE: 12pt">
      等存储介质
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      8.
     </span>
     <span style="FONT-SIZE: 12pt">
      支持
     </span>
     <span style="FONT-SIZE: 12pt">
      Unicode
     </span>
     <span style="FONT-SIZE: 12pt">
      码
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      9.
     </span>
     <span style="FONT-SIZE: 12pt">
      引擎可加载于磁盘和内存
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <strong>
      <span style="FONT-SIZE: 12pt">
       eXtremeDB
      </span>
     </strong>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      特点：
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      1.
     </span>
     <span style="FONT-SIZE: 12pt">
      内存数据库，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      将数据以程序直接使用的格式保存在主内存之中，不仅剔除了文件
     </span>
     <span style="FONT-SIZE: 12pt">
      I/O
     </span>
     <span style="FONT-SIZE: 12pt">
      的开销，也剔除了文件系统数据库所需的缓冲和
     </span>
     <span style="FONT-SIZE: 12pt">
      Cache
     </span>
     <span style="FONT-SIZE: 12pt">
      机制。其结果是每个交易一微秒甚至更少的极限速度，相比于类磁盘数据库而言，速度成百上千倍地提高。作为内存数据库，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      不仅性能高，而且数据存储的效率也非常高。为了提高性能并方便程序使用，数据在
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      中不做任何压缩，
     </span>
     <span style="FONT-SIZE: 12pt">
      100M
     </span>
     <span style="FONT-SIZE: 12pt">
      的空间可以保存高达
     </span>
     <span style="FONT-SIZE: 12pt">
      70M
     </span>
     <span style="FONT-SIZE: 12pt">
      以上的有效数据，这是其他数据库所不可想象的。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      2.
     </span>
     <span style="FONT-SIZE: 12pt">
      混合数据库，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      不仅可以建立完全运行在主内存的内存数据库，更可以建立磁盘
     </span>
     <span style="FONT-SIZE: 12pt">
      /
     </span>
     <span style="FONT-SIZE: 12pt">
      内存混合介质的数据库。在
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      ，我们把这种建立在磁盘、内存或磁盘
     </span>
     <span style="FONT-SIZE: 12pt">
      +
     </span>
     <span style="FONT-SIZE: 12pt">
      内存的运行模式称为
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB Fusion
     </span>
     <span style="FONT-SIZE: 12pt">
      融合数据库。
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB Fusion
     </span>
     <span style="FONT-SIZE: 12pt">
      兼顾数据管理的实时性与安全性要求，是实时数据管理的台阶性进步。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      3.
     </span>
     <span style="FONT-SIZE: 12pt">
      嵌入式数据库，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      内核以链接库的形式包含在应用程序之中，其开销只有
     </span>
     <span style="FONT-SIZE: 12pt">
      50KB~130KB
     </span>
     <span style="FONT-SIZE: 12pt">
      。无论在嵌入式系统还是在实时系统之中，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      都天然地嵌入在应用程序之中，在最终用户毫不知情的情况下工作。
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      的这种天然嵌入性对实时数据管理至关重要：各个进程都直接访问
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      数据库，避免了进程间通信，从而剔除了进程间通信的开销和不确定性。同时，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      独特的数据格式方便程序直接使用的，剔除了数据复制及数据翻译的开销，缩短了应用程序的代码执行路径。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      4.
     </span>
     <span style="FONT-SIZE: 12pt">
      由应用定制的
     </span>
     <span style="FONT-SIZE: 12pt">
      API
     </span>
     <span style="FONT-SIZE: 12pt">
      ，应用程序对
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      数据库的操作接口是根据应用数据库设计而自动产生，不仅提升了性能，也剔除了通用接口所必不可少的动态内存分配，从而提高了应用系统的可靠性。定制过程简单方便，由高级语言定制
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      数据库中的表格、字段、数据类型、事件触发、访问方法等应用特征，通过
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      预编译器自动产生访问该数据库的
     </span>
     <span style="FONT-SIZE: 12pt">
      C/C++ API
     </span>
     <span style="FONT-SIZE: 12pt">
      接口。
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      5.
     </span>
     <span style="FONT-SIZE: 12pt">
      可预测的数据管理
     </span>
    </div>
    <div style="LINE-HEIGHT: 22pt">
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      独特的体系结构，保证了数据管理的可预测性。
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      不仅更快、更小，而且更确定。在
     </span>
     <span style="FONT-SIZE: 12pt">
      80
     </span>
     <span style="FONT-SIZE: 12pt">
      双核
     </span>
     <span style="FONT-SIZE: 12pt">
      CPU
     </span>
     <span style="FONT-SIZE: 12pt">
      的服务器上，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      在
     </span>
     <span style="FONT-SIZE: 12pt">
      1TB
     </span>
     <span style="FONT-SIZE: 12pt">
      内存里保存
     </span>
     <span style="FONT-SIZE: 12pt">
      15B
     </span>
     <span style="FONT-SIZE: 12pt">
      条记录；无论记录数多少，
     </span>
     <span style="FONT-SIZE: 12pt">
      eXtremeDB
     </span>
     <span style="FONT-SIZE: 12pt">
      可以在八十分之一微秒的时间内提取一条记录。
     </span>
    </div>
