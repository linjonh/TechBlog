    <h4>
     <a id="240_7">
     </a>
     24.0、贪心算法
    </h4>
    <p>
     对于序列【2，3，1，1，4】，维护一个rightMost，表示能够到达的最右端的坐标
    </p>
    <p>
     rightMost = max（rightMost， nums[i] + i）
    </p>
    <p>
     在遍历的过程中注意当rightMost &lt; i时，返回false
    </p>
    <pre><code class="prism language-c++">bool canJump0(vector&lt;int&gt;&amp; nums)
{
    int size = nums.size();
    int rightMost = 0;
    for (int i = 0; i &lt; size; i++)
    {
        if (rightMost &lt; i)return false;
        rightMost = max(rightMost, i + nums[i]);
        if (rightMost &gt;= size - 1)return true;
    }
    return false;
}
</code></pre>
    <p>
     时间复杂度：O（n）
    </p>
    <p>
     空间复杂度：O（1）
    </p>
    <h4>
     <a id="241_34">
     </a>
     24.1、动态规划
    </h4>
    <p>
     对于序列【2，3，1，1，4】
    </p>
    <p>
     f（x）表示从下标x处，还能向右走几步
    </p>
    <p>
     例如f（2） = 3，代表在下标为2的地方，还能向右走3步（小于0，则说明走不到这里）
    </p>
    <p>
     状态转移公式f（x） = max【f（x - 1）- 1，nums[i]】
    </p>
    <p>
     道理是这个道理，但是专门为了这个建一个数组来记录状态太亏了，所以用一个maxStep来代表f（x）
    </p>
    <p>
     现在只需要维护maxStep即可
    </p>
    <p>
     maxStep = max【maxStep ，nums[i]】 - 1
    </p>
    <pre><code class="prism language-c++">bool canJump1(vector&lt;int&gt;&amp; nums) 
{
    int maxStep = 0;                        //维护在该位置上最多能走几步
    for (size_t i = 0; i &lt; nums.size(); i++)
    {
        if (maxStep &lt; 0)return false;
        maxStep = max(maxStep, nums[i]) - 1;
    }
    return true;
}
</code></pre>
    <p>
     时间复杂度：O（n）
    </p>
    <p>
     空间复杂度：O（1）
    </p>
    <p>
     在性能方面，两种基本差不多，只是下面这种代码比较简洁（就简洁了一行。。。）
    </p>
