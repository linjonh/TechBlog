    <h2>
    </h2>
    <p>
     在搞操作系统测试的时候，了解到UnixBench这个工具，今天记录一下这个过程吧（希望你们不会和我一样遇到这么多坑），当做学习积累~希望对大家有所帮助^_^
    </p>
    <hr id="hr-toc"/>
    <p>
    </p>
    <h4>
     <strong>
      1、介绍
     </strong>
    </h4>
    <p id="%E5%85%88%E7%AE%80%E5%8D%95%E4%BB%8B%E7%BB%8D%E4%B8%80%E4%B8%8B%E8%BF%99%E4%B8%AA%E5%B7%A5%E5%85%B7">
     先简单介绍一下这个工具
    </p>
    <p>
     UnixBench 是测试类 Unix 系统性能的老牌工具，也是常用的基准测试工具。它会执行 11 个单项测试，测试系统各个方面一系列的性能,然后将每个测试结果和一个基准值进行比较，得到一个索引值，所有测试项目的索引值结合在一起形成一个测试分数值，得到一个指数值。指数值越高，性能越好。UnixBench一键执行脚本 ，脚本会自动下载unixbench最新版本，并自动运行根据机器性能，测试执行完大约需要10-30分钟。
    </p>
    <p>
     Unixbench的主要测试项目有：系统调用、读写、进程、图形化测试、2D、 3D、管道、运算、C库等系统基准性能提供测试数据。
    </p>
    <p>
     unixbench一个基于系统的基准测试工具，不单纯是CPU 内存 或者磁盘测试工具。测试结果不仅仅取决于硬件，也取决于系统、开发库、甚至是编译器。
    </p>
    <p>
    </p>
    <h4>
     <strong>
      2、下载
     </strong>
    </h4>
    <p>
     https://github.com/kdlucas/byte-unixbench/archive/v5.1.3.tar.gz
    </p>
    <p>
    </p>
    <h4>
     <strong>
      3、安装
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       <u>
        <span style="color:#3399ea;">
         将下载好的压缩包解压缩，阅读README文件，得知如果不需要进行图形测试或者不在图形化界面下测试，则将Makefile文件中GRAPHICS_TEST = defined注释掉~
        </span>
       </u>
      </strong>
     </li>
     <li>
      <strong>
       <u>
        <span style="color:#3399ea;">
         make
        </span>
       </u>
      </strong>
      （鬼知道我这一步经历了啥）
     </li>
    </ul>
    <p>
     <span style="color:#ffbb66;">
      <strong>
       make
      </strong>
     </span>
     的时候发现它不认识这个命令，~~~
    </p>
    <p>
     <em>
      【这里我不大熟悉，可以直接通过 1、切换root用户身份：
     </em>
     <span style="color:#ffbb66;">
      sudo -s
     </span>
     <em>
      <span style="color:#ffbb66;">
      </span>
      2、更新安装列表：
      <span style="color:#ffbb66;">
       apt-get update
      </span>
      3、安装make：
      <span style="color:#ffbb66;">
       apt-get install make
      </span>
      】
     </em>
    </p>
    <p>
     然后我就想通过yum 来解决不认识make 命令的问题。好吧，
     <strong>
      <span style="color:#ffbb66;">
       yum
      </span>
     </strong>
     一下发现也不认识这个命令~~那么问题来了，不认识
     <span style="color:#ffbb66;">
      <strong>
       yum
      </strong>
     </span>
     咋办
    </p>
    <p>
     1）首先切换到root用户身份：在终端运行命令
     <strong>
      <span style="color:#ffbb66;">
       sudo -s
      </span>
     </strong>
    </p>
    <p>
     2）然后检测是否安装了build-essential程序包，输入命令：
     <strong>
      <span style="color:#ffbb66;">
       apt-get install build-essential
      </span>
     </strong>
    </p>
    <p>
     这个时候发现，并没有这个程序包~fine，那就执行下面两个语句
    </p>
    <p>
     <strong>
      <span style="color:#ffbb66;">
       sudo apt-get update
      </span>
     </strong>
     （更新源 ）
    </p>
    <p>
     <strong>
      <span style="color:#ffbb66;">
       sudo apt-get install build-essential
      </span>
     </strong>
    </p>
    <p>
     --------------------------------------------------做完上面的步骤后，我们就安装了 build-essential
    </p>
    <p>
     3）安装yum，输入命令：
     <strong>
      <span style="color:#ffbb66;">
       apt-get install yum
      </span>
     </strong>
    </p>
    <p>
     A、 这个时候又发现报错了：E: 无法定位软件包”的错误，无法安装。怎么解决呢：
    </p>
    <p>
     在   etc/apt  的 sources.list 添加镜像源 deb http://archive.ubuntu.com/ubuntu/ trusty main universe restricted multiverse
    </p>
    <p>
     然后
     <strong>
      <span style="color:#ffbb66;">
       sudo apt-get update
      </span>
     </strong>
    </p>
    <p>
     【这里加几句  vi：i 编辑，：wq 保存退出，：q! 不保存强制退出 ，需用的自取哈哈哈哈哈】
    </p>
    <p>
     B、重新输入命令：
     <strong>
      <span style="color:#ffbb66;">
       apt-get install yum
      </span>
     </strong>
     ，现在换了另一种错误：“下列软件包有未满足的依赖关系:yum : 依赖: python-sqlitecachec 但是它将不会被安装.依赖:python-urlgrabber 但是它将不会被安装 E: 无法修正错误，因为您要求某些软件包保持现状，就是它们破坏了软件包间的依赖关系。”
    </p>
    <p>
     我的解决办法是，缺啥补啥
    </p>
    <p>
     <strong>
      <span style="color:#ffbb66;">
       apt-get install python-sqlitecachec
      </span>
     </strong>
    </p>
    <p style="margin-left:0pt;">
     <strong>
      <strong>
      </strong>
      <span style="color:#ffbb66;">
       apt-get install  python-urlgrabber
      </span>
     </strong>
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/aa9756dbf70598a709aeeba1c6cd3fa3.png"/>
    </p>
    <p>
     再次，输入命令：
     <strong>
      <span style="color:#ffbb66;">
       apt-get install yum
      </span>
     </strong>
     ，现在就ok啦
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/cf6ee15784e1770ee20e19668cdd9bd2.png"/>
    </p>
    <p>
     到目前为止，我们的 yum  安装成功啦，回到原题：执行命令：
     <strong>
      <span style="color:#ffbb66;">
       make
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="231" src="https://i-blog.csdnimg.cn/blog_migrate/66678734912e7430f299346078a43034.png" width="1200"/>
    </p>
