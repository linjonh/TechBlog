    <strong>
     操作系统
    </strong>
    是计算机系统中

     系统软件

    <p>
    </p>
    <h3>
     <a id="_3">
     </a>
     操作系统——特征
    </h3>
    <ol>
     <li>
      <strong>
       并发性
      </strong>
      <br/>
      在多道程序环境下，并发性是在
      <code>
       同一时间间隔
      </code>
      内，系统中同时存在多个运行中的程序。
      <br/>
      <code>
       任务管理器
      </code>
      中，在
      <code>
       进程
      </code>
      一栏中，我们可以看见多个不同类型的程序同时处于运行的状态。
     </li>
    </ol>
    <p>
     这里需要注意的是
     <strong>
      同一时间间隔发生多个事件（并发性）
     </strong>
     与
     <strong>
      同一时刻发生多个事件（并行性）
     </strong>
     之间的区别：
    </p>
    <blockquote>
     <p>
      正所谓外行看热闹，内行看门道。
      <br/>
      上面所说的多个不同类型的程序同时处于运行的状态，正是我们在任务管理器里宏观看到的；但是实际上在单处理机环境下，每个时刻都只能有一道程序进行，所以操作系统的并发性是通过分时实现的，并不是我们看到的同时刻运行。
     </p>
    </blockquote>
    <ol start="2">
     <li>
      <strong>
       共享性
      </strong>
      <br/>
      <code>
       资源共享
      </code>
      ，系统中的资源不是只供单个进程应用的，而是供多个
      <code>
       并发进程
      </code>
      共同使用，共享方式主要分为两类：
      <br/>
      <strong>
       ①互斥共享方式
      </strong>
      <br/>
      某些特定的资源，如果同时使用会造成输出混乱，因此在一时间段内只允许由一个进程访问资源，这个进程释放后才能调用下一个进程。eg：打印机……
      <br/>
      <strong>
       ②同时访问方式
      </strong>
      <br/>
      有一类资源在一段时间内可被多个进程"同时"访问，这里说的同时还是上面说的分时的意思，进程
      <code>
       交替
      </code>
      地对资源进行访问，即
      <code>
       分时共享
      </code>
      。eg：磁盘设备
     </li>
    </ol>
    <p>
     <code>
      并发
     </code>
     与
     <code>
      共享
     </code>
     是操作系统最基本的特征，互为存在的条件：
     <br/>
     <strong>
      1）资源共享以程序并发执行为条件，只有程序并发，才存在共享的问题；
      <br/>
      2）如果不能合理地进行资源共享，也会影响并发的进行，甚至不能并发执行。
     </strong>
    </p>
    <ol start="3">
     <li>
      <strong>
       虚拟性
      </strong>
      <br/>
      顾名思义，将一个实体转换成
      <code>
       逻辑上的对应物
      </code>
      ，用于实现虚拟的技术，称为
      <code>
       虚拟技术
      </code>
      ，
      <code>
       操作系统
      </code>
      中使用了多种虚拟技术来实现
      <code>
       虚拟处理器
      </code>
      、
      <code>
       虚拟内存
      </code>
      、
      <code>
       虚拟外设
      </code>
      等。
      <br/>
      <code>
       虚拟处理器技术
      </code>
      是通过多道程序设计技术，采用多道程序并发执行方法来分时使用一个处理器的，所以一个处理器可以供多个用户（终端）使用。
      <br/>
      综上所述：
      <code>
       操作系统
      </code>
      的
      <code>
       虚拟技术
      </code>
      可以分为
      <code>
       时分复用技术
      </code>
      （eg：分时共享），
      <code>
       空分复用技术
      </code>
      （eg：虚拟存储器）
     </li>
    </ol>
    <blockquote>
     <p>
      一台电脑中只有一个CPU，每次只能执行一道程序，但是通过分时使用，在一段时间间隔内宏观上来看这台处理机能同时运行多道程序；但是通过分时使用，会给用户每道程序都有一个CPU为其服务的感觉，也就是说多道程序设计技术会把一台物理上的CPU虚拟为多台逻辑上的CPU
     </p>
    </blockquote>
    <ol start="4">
     <li>
      <strong>
       异步性
      </strong>
      <br/>
      多道程序设计环境可以允许
      <code>
       多个程序并发执行
      </code>
      ，但因为资源有限，进程的执行
      <code>
       并不连贯
      </code>
      （eg：就像我们所说的运行内存超出这个上限，打开的进程会以出队列顺序被强行终止），以不可预知的速度向前推进。也就是说多道程序执行的时间和顺序都是随机的
     </li>
    </ol>
    <blockquote>
     <p>
      异步会导致操作系统运行在一个随机的环境下，可能导致进程产生与时间有关的错误，但只要环境相同就不会产生该种错误。
     </p>
    </blockquote>
    <h3>
     <a id="_36">
     </a>
     操作系统——功能
    </h3>
    <p>
     举例：用户就像雇主（给工人下命令），操作系统就像工人（接收命令，操纵机器），计算机就像机器（由处理机、存储器、设备和文件几个设备组成），工人（扩充机器）接收命令（接口），机器开始运转功效。
    </p>
    <ol>
     <li>
      <strong>
       操作系统五大基本功能
      </strong>
      <br/>
      <strong>
       ①处理机管理
      </strong>
      <br/>
      处理机管理主要功能是对处理机的分配和运行实施有效的管理。多种环境下，处理机的分配和运行都以
      <code>
       进程(或线程)
      </code>
      为单位，所以处理机管理也可以说成
      <code>
       进程管理
      </code>
     </li>
    </ol>
    <blockquote>
     <p>
      <strong>
       进程管理
      </strong>
      功能划分：
      <br/>
      1）
      <strong>
       进程控制
      </strong>
      <br/>
      -----------负责进程的创建、撤销和状态转换
      <br/>
      2）
      <strong>
       进程同步
      </strong>
      <br/>
      -----------对并发执行的程序进行协调
      <br/>
      3）
      <strong>
       进程通信
      </strong>
      <br/>
      -----------负责进程之间的信息交换
      <br/>
      4）
      <strong>
       进程调度
      </strong>
      <br/>
      -----------依据算法对处理机进行分配
     </p>
    </blockquote>
    <p>
     <strong>
      ②存储器管理
     </strong>
     <br/>
     为多道进程的运行提供良好环境，对内存进行分配、保护和扩充等操作，方便客户使用，提高内存利用率
     <br/>
     主要实现以下功能：
    </p>
    <blockquote>
     <p>
      <strong>
       存储器管理
      </strong>
      功能划分：
      <br/>
      1）
      <strong>
       内存分配
      </strong>
      <br/>
      -----------按一定策略为每道程序分配内存
      <br/>
      2）
      <strong>
       内存保护
      </strong>
      <br/>
      -----------保证各个程序在自己的内存区域内运行，互不干扰
      <br/>
      3）
      <strong>
       内存扩充
      </strong>
      <br/>
      -----------为了允许大型作业的运行，需要借助虚拟技术来达到增加内存的效果dayingsai
     </p>
    </blockquote>
    <p>
     <strong>
      ③设备管理
     </strong>
     <br/>
     外设的管理是操作系统中最琐碎的部分，主要完成用户的I/O请求，对计算机内的所有设备实施有效管理
    </p>
    <blockquote>
     <p>
      <strong>
       设备管理
      </strong>
      功能划分：
      <br/>
      1）
      <strong>
       设备分配
      </strong>
      <br/>
      -----------根据一定设备分配原则对设备进行分配
      <br/>
      2）
      <strong>
       设备传输控制
      </strong>
      <br/>
      -----------实现物理I/O操作。譬如说：启动设备、中断处理etc.
      <br/>
      3）
      <strong>
       设备独立性
      </strong>
      <br/>
      -----------用户程序的设备与实际使用的物理设备无关
     </p>
    </blockquote>
    <p>
     <strong>
      ④文件管理
     </strong>
     <br/>
     计算机中的信息都是以
     <code>
      文件
     </code>
     形式存在的，操作系统中
     <code>
      文件系统
     </code>
     负责
     <code>
      文件管理
     </code>
     。主要任务就是：有效地支持文件存储、检索和修改，以解决文件共享、保密和保护问题。
    </p>
    <blockquote>
     <p>
      <strong>
       设备管理
      </strong>
      功能划分：
      <br/>
      1）文件存储空间管理
      <br/>
      2）目录管理
      <br/>
      3）文件操作管理
      <br/>
      4）文件保护
     </p>
    </blockquote>
    <p>
     <strong>
      ⑤用户接口
     </strong>
     <br/>
     为了方便用户的使用，操作系统还提供了用户接口
    </p>
    <blockquote>
     <p>
      用户接口主要分为3类：
      <br/>
      <strong>
       1）命令接口
      </strong>
      使用
      <code>
       命令接口
      </code>
      进行
      <code>
       作业控制
      </code>
      主要有两种方式：
      <br/>
      <strong>
       ①联机控制方式
       <br/>
       ②脱机控制方式
      </strong>
      按照
      <code>
       作业控制方式
      </code>
      可以将
      <code>
       命令接口
      </code>
      分为：
      <br/>
      <strong>
       ①联机命令接口
      </strong>
      (交互式命令接口)
      <br/>
      <strong>
       ②脱机命令接口
      </strong>
      (批处理命令接口)
      <br/>
      <strong>
       2）程序接口
      </strong>
      <br/>
      由一组
      <code>
       系统调用
      </code>
      (广义指令)组成，用户通过程序接口请求调用需求的资源。
      <br/>
      <strong>
       3）图形接口
      </strong>
     </p>
    </blockquote>
    <ol start="2">
     <li>
      <strong>
       扩充机器
      </strong>
      <br/>
      没有任何软件支持的计算机叫
      <code>
       裸机
      </code>
      在计算机最里边，外边是操作系统，
      <code>
       操作系统
      </code>
      加强并丰富了裸机的功能，通常把覆盖软件的机器称为
      <code>
       虚拟机（JVM）
      </code>
     </li>
    </ol>
    <h3>
     <a id="_98">
     </a>
     操作系统——阶段与分类
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7a7d564e09804304492a582a93117800.png"/>
    </p>
    <p>
     <strong>
      一、手工操作（无操作系统）阶段
     </strong>
     <br/>
     <strong>
      二、批处理阶段
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       单道批处理系统
      </strong>
      <br/>
      系统对作业的处理是成批进行的，但在内存中只有一道作业。
      <br/>
      单道批处理系统是在解决
      <code>
       人机矛盾
      </code>
      (速度和资源利用) 以及
      <code>
       CPU和I/O设备速率
      </code>
      不匹配的矛盾中形成的，其主要特征有：
      <br/>
      <strong>
       ①自动性
      </strong>
      <br/>
      顺序情况下，磁带上的一批作业能自动的逐个运行，不需要人工干预。
      <br/>
      <strong>
       ②顺序性
      </strong>
      <br/>
      磁带上各道作业顺序进入内存，完成顺序与进入的顺序相同。
      <br/>
      <strong>
       ③单道性
      </strong>
      <br/>
      内存中只有一道程序运行，监督程序每次只从磁带上调入一道程序进入内存中运行，当该程序完成或发生异常时，才换入其后继程序进入内存运行。
     </li>
    </ol>
    <blockquote>
     <p>
      缺点：每次主机内存中只能存放一道作业，每当运行时发出I/O请求后，高速CPU就会处于等待低俗的I/O完成状态。
     </p>
    </blockquote>
    <ol start="2">
     <li>
      <strong>
       多道批处理系统
      </strong>
      <br/>
      鉴于上述
      <code>
       单道批处理系统
      </code>
      的不足，引入了
      <code>
       多道程序设计技术
      </code>
      ，进一步提高了
      <code>
       资源利用率
      </code>
      和
      <code>
       系统吞吐量
      </code>
      。
      <br/>
      <code>
       多道程序设计技术
      </code>
      允许多个程序同时进入内存并允许他们在
      <code>
       CPU
      </code>
      中
      <code>
       交替运行
      </code>
      ，程序之间可以共享系统中的各种软硬件资源，当一道程序停止运行后，CPU会立即去运行另一道程序。不注重个体的完成效率，但是对于庞大的整体极大的节约了时间，提升了整体的效率。但是多道批处理系统要比单道批处理系统复杂得多，要涉及各种资源调度问题
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c9b348f6f0c20336f719416ad54ebe8a.png#pic_center"/>
     </li>
    </ol>
    <p>
     <strong>
      特点：
     </strong>
     <br/>
     <strong>
      ①多道
     </strong>
     <br/>
     内存中同时存放多道相互独立的程序；
     <br/>
     <strong>
      ②宏观上并行
     </strong>
     <br/>
     同时进入系统的多道程序都处于运行状态（先后开始运行，但都未运行完毕）；
     <br/>
     <strong>
      ③微观上串行
     </strong>
     <br/>
     内存中多道程序按顺序轮流占用CPU，交替执行。
     <br/>
     <code>
      多道程序设计技术
     </code>
     需要满足以下几个问题：
     <br/>
     <strong>
      ①如何分配处理器
     </strong>
     <br/>
     <strong>
      ②多道程序的内存分配问题
     </strong>
     <br/>
     <strong>
      ③I/O设备如何分配
     </strong>
     <br/>
     <strong>
      ④如何组织，存放大量数据以便于用户使用并保证安全性和一致性
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       优点：
      </strong>
      资源利用率高，多道程序共享资源，系统的吞吐量大，CPU与其他资源保持忙碌状态。
      <br/>
      <strong>
       缺点：
      </strong>
      用户响应时间长，不提供人机交互能力，用户不了解自己程序运行的情况，也不能控制计算机。
     </p>
    </blockquote>
    <p>
     <strong>
      三、分时操作系统
     </strong>
     <br/>
     <strong>
      分时技术
     </strong>
     就是把处理器的运行时间分成一个个很短的时间片，按时间片轮流把处理器分给各联机作业使用。如果某个作业在分配给它的时间内不能完成，该作业也立即停止，换到下一个作业使用处理器，如此一轮之后再轮到该作业继续之前的运算，是不是很像高速上开错了路口😫，但是计算机运行的速度极快，用户就感觉像是自己独占了一台计算机。
    </p>
    <p>
     <strong>
      分时操作系统
     </strong>
     就是多个用户通过终端共享一台主机，终端连在主机上，用户可以同时与主机进行交互操作并且互不干扰，因此实现
     <code>
      分时系统
     </code>
     最关键的问题就是：当用户在自己的
     <code>
      终端
     </code>
     键入命令时，系统如何能及时接收并处理该命令，将结果返回用户，用户再通过系统返回的运行结果，提出下一步请求，重复以上交互会话过程直到用户预计的全部工作都已完成。不同于
     <code>
      多道批处理
     </code>
     ，
     <code>
      分时系统
     </code>
     是实现
     <code>
      人机交互
     </code>
     的系统，这使得
     <code>
      分时系统
     </code>
     与
     <code>
      批处理系统
     </code>
     有了本质上的不同。
    </p>
    <blockquote>
     <h3>
      <a id="_141">
      </a>
      分时操作系统特征
     </h3>
     <ol>
      <li>
       <strong>
        同时性
       </strong>
       (多路性) 允许多个终端用户同时使用一台计算机，即一台计算机与若干台终端相连，终端上这些用户可以基本同时使用计算机。
      </li>
      <li>
       <strong>
        交互性
       </strong>
       用户能与系统进行人机对话，即用户通过终端采用人机对话的方式直接控制程序运行，同程序进行人机交互。
      </li>
      <li>
       <strong>
        独立性
       </strong>
       分时系统采用时间片轮转方式使一台计算机同时为多个终端服务。系统中多个用户独立进行操作，互不干扰。
      </li>
      <li>
       <strong>
        及时性
       </strong>
       用户请求能在很短的时间内获得响应。（通常为2~3s）
      </li>
     </ol>
     <blockquote>
      <p>
       缺点：
       <code>
        精密大数据系统
       </code>
       不能保证对
       <code>
        外部信息
       </code>
       做出
       <code>
        即时
       </code>
       (&lt;时间片)的反应(eg：飞机订票系统，导弹制导系统)
      </p>
     </blockquote>
    </blockquote>
    <p>
     <strong>
      四、实时操作系统
     </strong>
     <br/>
     为在
     <code>
      规定时间内
     </code>
     完成某些紧急任务不需要时间片排队而诞生的操作系统。对外部键入的信息，实时操作系统要求秒级、毫秒级甚至微秒级的响应时间，具有
     <code>
      及时性
     </code>
     和
     <code>
      可靠性
     </code>
     。
    </p>
    <ol>
     <li>
      <strong>
       硬实时系统（实时控制系统）
      </strong>
      <br/>
      某个动作必须绝对地在
      <code>
       规定的时间或时间范围
      </code>
      内发生。（eg：飞机自动控制系统）
     </li>
     <li>
      <strong>
       软实时系统（实时信息处理系统）
      </strong>
      <br/>
      相比于硬实时系统，可以接收违反时间规定而且不会引起永久性伤害。计算机及时接收从远程终端发来的服务请求，根据用户提出的问题对信息进行检索处理，并在很短时间内对用户做出正确的响应（eg：飞机订票系统，银行管理系统）
     </li>
    </ol>
    <p>
     <strong>
      批处理操作系统
     </strong>
     ，
     <strong>
      分时操作系统
     </strong>
     ，
     <strong>
      实时操作系统
     </strong>
     是3大基本操作系统，若是一种操作系统有其两种的功能，那么就可以称为
     <strong>
      通用操作系统
     </strong>
     。
    </p>
    <p>
     <strong>
      五、其他操作系统
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       集群系统
      </strong>
     </li>
     <li>
      <strong>
       嵌入式操作系统
      </strong>
     </li>
     <li>
      <strong>
       网络操作系统
      </strong>
      <br/>
      把
      <code>
       计算机网络
      </code>
      的各台计算机有机的结合起来，提供一种
      <code>
       统一
      </code>
      、
      <code>
       经济
      </code>
      和
      <code>
       有效
      </code>
      的使用各台计算机的办法，实现各台计算机数据的相互传送。主要的
      <strong>
       特点
      </strong>
      有：
      <code>
       网络中各种资源的共享
      </code>
      ，
      <code>
       各台计算机之间的通信
      </code>
     </li>
     <li>
      <strong>
       分布式计算机系统
      </strong>
      <br/>
      由多台计算机组成，并满足下列条件：
      <br/>
      <strong>
       ①系统中任意两台计算机通过通信方式交换信息
       <br/>
       ②系统中每台计算机都具有同等地位，没有主机、从机之分
       <br/>
       ③每台计算机上的资源为所有用户共享
       <br/>
       ④系统中任意台计算机都可以构成一个子系统，还能重构
       <br/>
       ⑤任何工作都可以分布在几台计算机上，并行工作，协调完成
      </strong>
      <br/>
      主要的
      <strong>
       特点
      </strong>
      有：分布性，并行性
     </li>
    </ol>
    <blockquote>
     <p>
      本质区别：网络操作系统是各机共享资源，数据传送；分布式计算机系统是若干计算机协同完成同一任务
     </p>
    </blockquote>
