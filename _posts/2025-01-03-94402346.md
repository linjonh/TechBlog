    <p>
     1.需求
     <br/>
     后台要求请求头是Content-Type：application/x-www-form-urlencoded（设置请求头的意思是告诉后端传过去的数据是原生的表单格式，但是并没有改变传给后端的数据格式，所以需要对数据格式进行转换）
    </p>
    <p>
     2.方法
     <br/>
     <strong>
      解决方法一：
     </strong>
    </p>
    <ul>
     <li>
      安装qs插件
     </li>
     <li>
      在main中引入import qs from ‘qs’;
     </li>
     <li>
      将传给后端的参数进行转化qs.stringify({key1:value1,key2:value2,key3:value3});
      <br/>
      <strong>
       解决方法二：
      </strong>
     </li>
     <li>
      创建一个表单对象let formData = new FormData()；
     </li>
     <li>
      formData.append(‘str’, JSON.stringify(this.$store.state.HonourValue));str是key,后面的是值
     </li>
     <li>
      将formData作为参数传值给后端
     </li>
     <li>
      <strong>
       解决方法三：
      </strong>
     </li>
     <li>
      后端解决：在请求参数使用@RequestBody注解替代@RequestParam
     </li>
     <li>
      @ResponseBody
     </li>
     <li>
      @RequestMapping(value="/save",method=RequestMethod.POST)
      <br/>
      public Map&lt;String, String&gt; save(@RequestBody Student student)//自动将参数封装成student对象
     </li>
    </ul>
    <p>
     3.现象
    </p>
    <p>
     没有进行转换后的请求参数的数据格式，如图：会是request Paload 并且数据是对象{ }。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/eb137b77662231c15090eb57e85ca5a9.png"/>
    </p>
    <p>
     进行转换后的请求参数的数据格式，如图：会是Form Data 并且数据不是对象。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/02e6fa3d214b8357528dcea5261c068f.png">
      <br/>
      4.出现该问题的原因：
      <br/>
      请求的Content-Type设置为application/x-www-form-urlencoded,那么这个Post请求被认为是HTTP POST表单请求,参数出现在form data,参数形式是key=value&amp;key1=value1这种形式，后端获取使用@RequestParam; 其他情况如使用原生AJAX的POST请求如果不指定请求头Request Header,默认使用的Content-Type是text/plain;charset=UTF-8,或者application/json ,参数出现在Request payload块,其参数形式是标准的json格式，使用@RequestParam无法获取,需要使用@RequestBody获取。
      <br/>
      在axios使用Post发送数据时，默认是直接把json放到请求体中提交到后端的，属于上述的第二种方式，需要将Request payload转换。
     </img>
    </p>
