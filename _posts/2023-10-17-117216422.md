    <h2>
     <a id="_1">
     </a>
     前言
    </h2>
    <p>
     在做微信小程序之前，最好要有一些JavaScript和Html、css的基础，不过没有也没关系，学习中慢慢了解也可以，只不过你需要花费的时间就多一些。
    </p>
    <h2>
     <a id="_4">
     </a>
     正文
    </h2>
    <p>
     微信小程序的开发常规是使用微信开放平台提供的工具，因此本文会从账号注册到工具安装，再到项目运行，这种方式进行讲解。开始吧！
    </p>
    <h3>
     <a id="_7">
     </a>
     一、微信开发者注册
    </h3>
    <p>
     点击
     <a href="https://mp.weixin.qq.com/" rel="nofollow">
      微信公众平台
     </a>
     进入。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6dffe3deea2e14a320b76daf042020e5.png"/>
    </p>
    <p>
     直接用你手机上的微信扫描这个二维码进行登录，这里的账号类型有三个，服务号、订阅号、小程序。
    </p>
    <p>
     扫码之后，你的手机上就会显示之前创建的微信公众平台上创建过公众号或者小程序。如果你都没有创建的话，就直接登录。
    </p>
    <p>
     然后进行开发工具的下载。
    </p>

    <h3>
     <a id="_37">
     </a>
     三、创建项目
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce7d69983f0c65857ff227e778228c56.png#pic_center">
      <br/>
      用自己的手机微信扫描登录一下：
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8712b53dcd76db006c13ae1b1c796f1d.png"/>
     <br/>
     登录之后你会在左下方看到自己的微信头像，下面来创建一个小程序吧，点击右边的这个加号。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c112146e13fa6eefa67559a128d194ba.png"/>
    </p>
    <p>
     这里我修改了项目的名称和存放目录，然后是这个AppID，这里有正式和测试的区别，先来看一下正式号AppID怎么获取
    </p>
    <h4>
     <a id="1_AppID_49">
     </a>
     1. 获取正式号AppID
    </h4>
    <p>
     这里可以手动去注册一个正式的AppId，点击注册，会进入到小程序注册页面，输入一个未注册过的邮箱。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0999bcf08a433edeb0532e71f66ab4ee.png"/>
     <br/>
     注意标注的文字，这很重要，所以当你要进行小程序的开发时，最好是新建一个邮箱去注册，可以减少不必要的麻烦。
    </p>
    <p>
     当资料填写完成之后点击注册按钮。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/adadb74548a0d4422f20a3968be8ab2b.png"/>
     <br/>
     然后登录填写的邮箱，去查看最新收到的邮件
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/89d4c1c3417b9a885bbef8b406b126c3.png"/>
     <br/>
     点击这个链接，进行邮箱账号的激活，点击之后会进行用户信息登记
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0d99d01a026a881a069265403262ea6e.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8c42c0d2aa8898d40c84de9e0216344a.png"/>
     <br/>
     这里我选择的是个人，然后会出现一个二维码，使用手机微信进行扫描，扫描的用户将会成为管理员。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f2e3a4e1d282ae9823eab891b3583d8b.png#pic_center"/>
     <br/>
     注册成功，点击前往小程序。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/54eff3d296e7a8e6797ff185708f3e28.png"/>
     <br/>
     点击填写按钮，对小程序的相关信息进行填写。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3c79349009eac438fab69635a99fc226.png"/>
     <br/>
     当你的小程序名称和已经注册过得小程序名称大致相同时，会有一个提示。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9d41de5531110d11f693ea8bc4424623.png"/>
     <br/>
     点击仍然使用。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1a9ba9d346d6c4c2e57d7d5d0031ead6.png"/>
     <br/>
     选择图标进行和填写描述以及小程序的类型，都填写好了之后点击底部的提交按钮。提交之后其实你的AppID就生成好了，
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c3a35600700cf3a7d577a0c549eeda21.png"/>
    </p>
    <p>
     选择小皮天气，
    </p>
    <h4>
     <a id="2_AppID_75">
     </a>
     2. 使用测试号AppID
    </h4>
    <p>
     或者点击右边的测试号，这个就很简单了，点击获取的一个测试的AppID。如果你的小程序是要发布的，那么建议你使用正式号AppID。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/611d0bcb076f9dc870fc0356a8adc43f.png"/>
     <br/>
     点击新建，等待项目创建完成。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a586653a3e6c6da366500d0b511038f9.png"/>
     <br/>
     你会看到给你添加了默认的页面index,然后就是你的微信头像，以及一个HelloWorld。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/31626acb4b57fe60a686624bdcb71eb6.png#pic_center"/>
    </p>
    <p>
     这里就是模拟器了，一般在开发过程中都是在模拟器中调试运行的，因为你如果要运行的自己的手机微信上，编译的过程比较长。
    </p>
    <p>
     下面尝试运行到手机上。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f42f3a24a1f43f2f439c9c4d310ab273.png"/>
     <br/>
     点击这个真机调试。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2eb6499f3b3f5c4e776a3407e5e6d976.png#pic_center"/>
     <br/>
     手机微信扫描这个二维码，然后你就会看到你的手机上有这个小程序了。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ca5984130d640ec8ad5f2926f6b46786.png#pic_center"/>
    </p>
    <p>
     因为是本身是搞Android开发的，所以这个过程我的感觉就像是通过usb连接了手机和电脑，然后在AS上编译项目安装APK到手机上，而此时这个APK是只能自己调试使用的，同理刚才这种方式产生的小程序也是调试使用的，属于开发版，你可以在微信的小程序列表中看到。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/928200d3875c83db63009fafbae005ea.png#pic_center"/>
    </p>
    <p>
     这就是刚才安装的小程序，你点击一下就会打开，不过这个和网速有关系。这个就相当于一个debug版apk，只能自己使用。
    </p>
    <h3>
     <a id="_100">
     </a>
     四、项目结构讲解
    </h3>
    <p>
     可以看到，从刚才的微信小程序的项目创建到运行，我没有写过代码。那么在运行的时候却能看到我的微信头像和这个HelloWorld，这说明在创建项目的时候自动给我们生成了一些代码，就比如AS创建一个新项目会默认有一个MainActivity一样，同时这个Activity还是启动页面，那么我觉得有必要去了解这个小程序的项目结构，这样以后写的时候就不会迷糊了。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cdba4ddbe816e69c406c238521226512.png#pic_center"/>
     <br/>
     上图就是这个小程序的项目结构。
    </p>
    <p>
     上面两个文件夹pages和utils后面将，先看下面的文件。
    </p>
    <h4>
     <a id="1_appjs_108">
     </a>
     1. app.js
    </h4>
    <p>
     看看里面的代码：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d8ee8912bc93d384c4fafa3a9dd03c3c.png"/>
     <br/>
     这里看到了一个本地存储和一个登录，这些都是功能的实现函数，那么这里面就是一些业务逻辑代码。那么小程序的逻辑就在这里。
    </p>
    <h4>
     <a id="2_appjson_116">
     </a>
     2. app.json
    </h4>
    <p>
     看看里面的代码
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7336bc5766757395b159fc811428f532.png"/>
     <br/>
     上方这里有一个pages的数组，里面是小程序当前已有的页面，后面又有一个window表示窗体的样式，下面还制定了样式的版本，那么毫无疑问，这是一个公共配置文件，当你的小程序增加或者删除页面时和这个app.json都有关系。app.json文件用来对微信小程序进行全局配置，决定页面文件的路径、窗口表现、设置网络超时时间、多设置多Tab等。
    </p>

    <h4>
     <a id="6_pages_136">
     </a>
     6. pages
    </h4>
    <p>
     这是一个页面文件夹，你的小程序的页面都在这个下面，现在它里面有两个页面。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/49e8b5f778cd76d69be1b31aed9db3ed.png#pic_center"/>
     <br/>
     你可以看到这里的两个页面各自有四个文件，就拿index来说明。
    </p>
    <ul>
     <li>
      index.js 页面逻辑
     </li>
     <li>
      index.wxml 页面结构
     </li>
     <li>
      index.json 页面配置
     </li>
     <li>
      index.wxss 页面样式表
     </li>
    </ul>
    <p>
     基本上每一个页面都会有这个四个文件。为了方便开发者减少配置项，描述页面的四个文件必须具有相同的路径与文件名。可以看到都在index文件夹下，文件名一致，只是文件类型不同而已。
    </p>
    <h4>
     <a id="7_utils_149">

     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e6bc5a123d718b0eea4a13695208d8cf.png"/>
     <br/>
     那么很明显这个文件夹就用放置一些工具js文件的，这个默认的里面目前是对时间的处理。项目结构基本上介绍完了，后面的实际开发中会有相应的更改说明。
    </p>
