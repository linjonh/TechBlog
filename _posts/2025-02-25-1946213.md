    <h1 class="title-article" id="articleContentId">
     微软“S+S”策略分析：鱼和熊掌真的可以兼得么？
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    微软在2003年的时候就曾经提起过关于SOA的理念，但是一段时间之后便隐了声迹，用微软的话来讲，这段时间里，微软也一直没有间断在做SOA的事情， 比如在SOA标准中的核心协议部分就由微软和IBM共同完成。只不过这期间属于是闭关修炼内功。但是，低调显然不是微软历来的风格，所以昨天，微软振臂一 呼，在北京中国大饭店“S+S”策略的技术峰会上宣布，在未来几年的时间里，微软将会与合作伙伴一起碰撞出更多的灵感一同致力于SOA的发展，并且由微软 院士、负责微软平台与产品策略规划的Donald Ferguson博士和微软中国首席技术官李志霄博士共同为大家展示了微软SOA未来的愿景路线图。
    <br/>
    <br/>
    首先，微软非常认可目前SOA的发展方向，认为是通向未来IT之路的重要途径。悉数当前的技术发展，软件的价格在不断的下降，硬件的价格也在不断的下降， 然而从单核到双核再到多核，我们却在用越来越低的成本获得越来越高的性能。其次人才的全球化和互联网的迅猛发展让人们不得不开始思考技术到底应该为业务创 造怎样的价值？
    <br/>
    <br/>
    李志霄博士接下来有了精辟的论断：“所谓SaaS，就是一种服务的交互方式，这种方式与软件和物理硬件所在的位置有关；所谓的SOA，就是一种服务的组合 或者是聚合；而所谓的Web2.0就更简单了，就是一种参与架构产生的聚众效应，就是社区参与+商业效应。”李博士给出在这段精彩的结论之后反问大家“难 道企业真的有了这些就足够了么？就可以进行所谓的业务创新，就可以让数据信息自由流动了么？就可以为业务创造价值了么？”
    <br/>
    <br/>
    李博士认为，企业选择了以上中的任何一种都将等于是放弃了另一方面的优势，而选择软件+服务也就是“S+S”策略方能集成了客户端+服务器端+云端的所有优势。那么接下来一定是大家需要反问微软了，鱼和熊掌真的可以兼得么？
    <br/>
    <br/>
    从微软的“S+S”的对象中我们可以清楚的看到，微软将这一策略分成了两大类四小类：第一大类前端展现层包括1、桌面客户端，代表作为Vista、 Office；2、智能/移动终端，代表作为Windows Mobile、XBox360、Surface、Zune；第二大类后端数据模块包括1、云端服务，代表作为Windows Live平台、Exchange Online；2、企业后端也就是作为企业服务总线的Biztalk。这样庞大的技术产品体系就像是美丽的可可西里，令人无限的向往却总是望而却步。毕竟 去欣赏它的美丽所需要付出的成本是惊人的，有时甚至会搭上性命。那么大家也许更愿意去体验一下美景的缩微图，可能也会别有一番收获。
    <br/>
    <br/>
    对于企业实施SOA的解决方案微软主张首先从企业的核心业务入手，所谓“干就干一票大的，达到艳惊四座的效果”，但是如果对企业核心业务的SOA改造部署 一旦失败或者出现偏差，那将对企业带来致命的打击。企业不是在“闯关东”，立下不成功便成仁的军令状。但是我们不得不佩服微软总是在展现他年轻人思维般的 魅力，总是让你觉得他仍然在流淌着十六七岁时不羁的血液。
