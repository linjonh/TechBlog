    <h4>
     <a id="_SvelteKit_24">
     </a>
     什么是 SvelteKit？
    </h4>
    <p>
     SvelteKit 是一个使用
     <a href="https://svelte.yayujs.com/docs/svelte" rel="nofollow">
      Svelte
     </a>
     快速开发稳健、高性能 Web 应用程序的框架。如果您来自 React，SvelteKit 类似于 Next。如果您来自 Vue，SvelteKit 类似于 Nuxt。
    </p>
    <p>
     要了解更多关于可以用 SvelteKit 构建的应用程序类型，请参阅
     <a href="https://svelte.yayujs.com/docs/kit/faq#What-can-I-make-with-SvelteKit" rel="nofollow">
      常见问题
     </a>
     。
    </p>
    <h4>
     <a id="_Svelte_30">
     </a>
     什么是 Svelte？
    </h4>
    <p>
     简而言之，Svelte 是一种编写用户界面组件的方式 —— 比如导航栏、评论区或联系表单 —— 这些组件用户可以在浏览器中看到并与之交互。Svelte 编译器将您的组件转换为可以运行的 JavaScript 来渲染页面的 HTML，以及为页面添加样式 CSS。您不需要了解 Svelte 就能理解本指南的其余部分，但了解它会有所帮助。如果您想了解更多，请查看
     <a href="https://svelte.yayujs.com/tutorial" rel="nofollow">
      Svelte 教程
     </a>
     。
    </p>
    <h4>
     <a id="SvelteKit__Svelte__34">
     </a>
     SvelteKit 与 Svelte 的区别
    </h4>
    <p>
     Svelte 负责渲染 UI 组件。您可以组合这些组件并仅使用 Svelte 渲染整个页面，但要构建完整的应用程序，您需要的不仅仅是 Svelte。
    </p>
    <p>
     SvelteKit 帮助您在遵循现代最佳实践的同时构建 Web 应用，并为常见的开发挑战提供解决方案。它提供从基本功能 —— 比如在点击链接时更新 UI 的
     <a href="https://svelte.yayujs.com/docs/kit/glossary#Routing" rel="nofollow">
      路由器
     </a>
     —— 到更高级的功能。
    </p>
    <p>
     它的广泛功能列表包括：仅加载最小所需代码的
     <a href="https://vitejs.dev/guide/features.html#build-optimizations" rel="nofollow">
      构建优化
     </a>
     ；
     <a href="https://svelte.yayujs.com/docs/kit/service-workers" rel="nofollow">
      离线支持
     </a>
     ；用户导航前的页面
     <a href="https://svelte.yayujs.com/docs/kit/link-options#data-sveltekit-preload-data" rel="nofollow">
      预加载
     </a>
     ；通过
     <a href="https://svelte.yayujs.com/docs/kit/glossary#SSR" rel="nofollow">
      SSR
     </a>
     、浏览器
     <a href="https://svelte.yayujs.com/docs/kit/glossary#CSR" rel="nofollow">
      客户端渲染
     </a>
     或构建时
     <a href="https://svelte.yayujs.com/docs/kit/glossary#Prerendering" rel="nofollow">
      预渲染
     </a>
     来处理应用程序不同部分的
     <a href="https://svelte.yayujs.com/docs/kit/page-options" rel="nofollow">
      可配置渲染
     </a>
     ；
     <a href="https://svelte.yayujs.com/docs/kit/images" rel="nofollow">
      图像优化
     </a>
     ；等等。使用所有现代最佳实践构建应用程序非常复杂，但 SvelteKit 为您处理了所有繁琐的工作，这样您就可以专注于创造性的部分。
    </p>
    <p>
     它利用带有
     <a href="https://github.com/sveltejs/vite-plugin-svelte">
      Svelte 插件
     </a>
     的
     <a href="https://vitejs.dev/" rel="nofollow">
      Vite
     </a>
     来实现
     <a href="https://github.com/sveltejs/vite-plugin-svelte/blob/main/docs/config.md#hot">
      热模块替换 (HMR)
     </a>
     ，从而在浏览器中即时反映代码更改，提供闪电般快速且功能丰富的开发体验。
    </p>
    <h3>
     <a id="_44">
     </a>
     创建项目
    </h3>
    <p>
     创建 SvelteKit 应用最简单的方法是运行
     <code>
      npx sv create
     </code>
     ：
    </p>
    <pre><code class="prism language-bash">npx sv create my-app
<span class="token builtin class-name">cd</span> my-app
<span class="token function">npm</span> <span class="token function">install</span>
<span class="token function">npm</span> run dev
</code></pre>
    <p>
     第一个命令将在
     <code>
      my-app
     </code>
     目录中搭建一个新项目，并询问您是否要设置一些基本工具，比如 TypeScript。有关设置其他工具的指导，请参见
     <a href="https://svelte.yayujs.com/docs/kit/integrations" rel="nofollow">
      集成
     </a>
     。随后的命令将安装其依赖项并在
     <a href="http://localhost:5173" rel="nofollow">
      localhost:5173
     </a>
     上启动服务端。
    </p>
    <p>
     这里有两个基本概念：
    </p>
    <ul>
     <li>
      应用的每个页面都是一个
      <a href="https://svelte.yayujs.com/docs/svelte" rel="nofollow">
       Svelte
      </a>
      组件
     </li>
     <li>
      通过在项目的
      <code>
       src/routes
      </code>
      目录中添加文件来创建页面。这些页面将被服务端渲染，以确保用户首次访问您的应用时速度尽可能快，之后客户端应用将接管
     </li>
    </ul>
    <p>
     尝试编辑文件以了解所有功能是如何工作的。
    </p>
    <h4>
     <a id="_64">
     </a>

     <a href="https://marketplace.visualstudio.com/items?itemName=svelte.svelte-vscode" rel="nofollow">
      Svelte 扩展
     </a>
     ，但
     <a href="https://sveltesociety.dev/resources#editor-support" rel="nofollow">
      也支持许多其他编辑器
     </a>

    <h3>
     <a id="_68">
     </a>
     项目结构
    </h3>
    <p>
     一个典型的 SvelteKit 项目结构如下：
    </p>
    <pre><code class="prism language-bash">my-project/
├ src/
│ ├ lib/
│ │ ├ server/
│ │ │ └ <span class="token punctuation">[</span>您的仅服务端库文件<span class="token punctuation">]</span>
│ │ └ <span class="token punctuation">[</span>您的库文件<span class="token punctuation">]</span>
│ ├ params/
│ │ └ <span class="token punctuation">[</span>您的参数匹配器<span class="token punctuation">]</span>
│ ├ routes/
│ │ └ <span class="token punctuation">[</span>您的路由<span class="token punctuation">]</span>
│ ├ app.html
│ ├ error.html
│ ├ hooks.client.js
│ ├ hooks.server.js
│ └ service-worker.js
├ static/
│ └ <span class="token punctuation">[</span>您的静态资源<span class="token punctuation">]</span>
├ tests/
│ └ <span class="token punctuation">[</span>您的测试<span class="token punctuation">]</span>
├ package.json
├ svelte.config.js
├ tsconfig.json
└ vite.config.js
</code></pre>
    <p>
     您还会发现一些常见文件，如
     <code>
      .gitignore
     </code>
     和
     <code>
      .npmrc
     </code>
     （如果您在运行
     <code>
      npx sv create
     </code>
     时选择了这些选项，还会有
     <code>
      .prettierrc
     </code>
     和
     <code>
      eslint.config.js
     </code>
     等）。
    </p>
    <h4>
     <a id="_100">
     </a>
     项目文件
    </h4>
    <h5>
     <a id="src_102">
     </a>
     src
    </h5>
    <p>
     <code>
      src
     </code>
     目录包含了项目的主要内容。除了
     <code>
      src/routes
     </code>
     和
     <code>
      src/app.html
     </code>
     之外，其他都是可选的。
    </p>
    <ul>
     <li>
      <code>
       lib
      </code>
      包含您的库代码（实用工具和组件），可以通过
      <a href="https://svelte.yayujs.com/docs/kit/$lib" rel="nofollow">
       <code>
        $lib
       </code>
      </a>
      别名导入，或使用
      <a href="https://svelte.yayujs.com/docs/kit/packaging" rel="nofollow">
       <code>
        svelte-package
       </code>
      </a>
      打包分发
      <ul>
       <li>
        <code>
         server
        </code>
        包含您的仅服务端（server-only）库代码。可以使用
        <a href="https://svelte.yayujs.com/docs/kit/server-only-modules" rel="nofollow">
         <code>
          $lib/server
         </code>
        </a>
        别名导入。SvelteKit 会阻止您在客户端代码中导入这些内容。
       </li>
      </ul>
     </li>
     <li>
      <code>
       params
      </code>
      包含应用需要的任何
      <a href="https://svelte.yayujs.com/docs/kit/advanced-routing#Matching" rel="nofollow">
       参数匹配器
      </a>
     </li>
     <li>
      <code>
       routes
      </code>
      包含应用的
      <a href="https://svelte.yayujs.com/docs/kit/routing" rel="nofollow">
       路由
      </a>
      。您也可以在这里放置仅在单个路由中使用的其他组件
     </li>
     <li>
      <code>
       app.html
      </code>
      是您的页面模板 — 一个包含以下占位符的 HTML 文档：
      <ul>
       <li>
        <code>
         %sveltekit.head%
        </code>
        — 应用需要的
        <code>
         &lt;link&gt;
        </code>
        和
        <code>
         &lt;script&gt;
        </code>
        元素，以及任何
        <code>
         &lt;svelte:head&gt;
        </code>
        内容
       </li>
       <li>
        <code>
         %sveltekit.body%
        </code>
        — 渲染页面的标记。这应该放在
        <code>
         &lt;div&gt;
        </code>
        或其他元素内，而不是直接放在
        <code>
         &lt;body&gt;
        </code>
        内，以防止浏览器插件注入元素后被水合过程销毁而导致的错误。如果不是这样情况，SvelteKit 会在开发时警告您
       </li>
       <li>
        <code>
         %sveltekit.assets%
        </code>
        — 如果指定了，则是
        <a href="https://svelte.yayujs.com/docs/kit/configuration#paths" rel="nofollow">
         <code>
          paths.assets
         </code>
        </a>
        ，否则是到
        <a href="https://svelte.yayujs.com/docs/kit/configuration#paths" rel="nofollow">
         <code>
          paths.base
         </code>
        </a>
        的相对路径
       </li>
       <li>
        <code>
         %sveltekit.nonce%
        </code>
        — 手动包含的链接和脚本的
        <a href="https://svelte.yayujs.com/docs/kit/configuration#csp" rel="nofollow">
         CSP
        </a>
        nonce（如果使用）
       </li>
       <li>
        <code>
         %sveltekit.env.[NAME]%
        </code>
        - 这将在渲染时被替换为
        <code>
         [NAME]
        </code>
        环境变量，该变量必须以
        <a href="https://svelte.yayujs.com/docs/kit/configuration#env" rel="nofollow">
         <code>
          publicPrefix
         </code>
        </a>
        （通常是
        <code>
         PUBLIC_
        </code>
        ）开头。如果没有匹配项，将回退到
        <code>
         ''
        </code>
       </li>
      </ul>
     </li>
     <li>
      <code>
       error.html
      </code>
      是在所有其他内容都失败时渲染的页面。它可以包含以下占位符：
      <ul>
       <li>
        <code>
         %sveltekit.status%
        </code>
        — HTTP 状态码
       </li>
       <li>
        <code>
         %sveltekit.error.message%
        </code>
        — 错误信息
       </li>
      </ul>
     </li>
     <li>
      <code>
       hooks.client.js
      </code>
      包含您的客户端
      <a href="https://svelte.yayujs.com/docs/kit/hooks" rel="nofollow">
       钩子
      </a>
     </li>
     <li>
      <code>
       hooks.server.js
      </code>
      包含您的服务端
      <a href="https://svelte.yayujs.com/docs/kit/hooks" rel="nofollow">
       钩子
      </a>
     </li>
     <li>
      <code>
       service-worker.js
      </code>
      包含您的
      <a href="https://svelte.yayujs.com/docs/kit/service-workers" rel="nofollow">
       service worker
      </a>
     </li>
    </ul>
    <p>
     （项目是包含
     <code>
      .js
     </code>
     还是
     <code>
      .ts
     </code>
     文件取决于您创建项目时是否选择使用 TypeScript。您可以使用本页底部的切换按钮在文档中的 JavaScript 和 TypeScript 之间切换。）
    </p>
    <p>
     如果您在设置项目时添加了
     <a href="https://vitest.dev" rel="nofollow">
      Vitest
     </a>
     ，您的单元测试将位于
     <code>
      src
     </code>
     目录中，带有
     <code>
      .test.js
     </code>
     扩展名。
    </p>
    <h5>
     <a id="static_127">
     </a>
     static
    </h5>
    <p>
     这里放置需要按原样提供的静态资源，如
     <code>
      robots.txt
     </code>
     或
     <code>
      favicon.png
     </code>
     。
    </p>
    <h5>
     <a id="tests_131">
     </a>
     tests
    </h5>
    <p>
     如果您在设置项目时添加了
     <a href="https://playwright.dev/" rel="nofollow">
      Playwright
     </a>
     进行浏览器测试，测试文件将位于此目录中。
    </p>
    <h5>
     <a id="packagejson_135">
     </a>
     package.json
    </h5>
    <p>
     您的
     <code>
      package.json
     </code>
     文件必须包含
     <code>
      @sveltejs/kit
     </code>
     、
     <code>
      svelte
     </code>
     和
     <code>
      vite
     </code>
     作为
     <code>
      devDependencies
     </code>
     。
    </p>
    <p>
     当您使用
     <code>
      npx sv create
     </code>
     创建项目时，您会注意到
     <code>
      package.json
     </code>
     包含了
     <code>
      "type": "module"
     </code>
     。这意味着
     <code>
      .js
     </code>
     文件将被解释为带有
     <code>
      import
     </code>
     和
     <code>
      export
     </code>
     关键字的原生 JavaScript 模块。遗留的 CommonJS 文件需要一个
     <code>
      .cjs
     </code>
     文件扩展名。
    </p>
    <h5>
     <a id="svelteconfigjs_141">
     </a>
     svelte.config.js
    </h5>
    <p>
     此文件包含您的 Svelte 和 SvelteKit
     <a href="https://svelte.yayujs.com/docs/kit/configuration" rel="nofollow">
      配置
     </a>
     。
    </p>
    <h5>
     <a id="tsconfigjson_145">
     </a>
     tsconfig.json
    </h5>
    <p>
     如果您在
     <code>
      npx sv create
     </code>
     期间添加了类型检查，此文件（或者如果您更喜欢对
     <code>
      .js
     </code>
     文件而不是
     <code>
      .ts
     </code>
     文件进行类型检查，则为
     <code>
      jsconfig.json
     </code>
     ）将配置 TypeScript。由于 SvelteKit 依赖于某些特定方式的配置设置，它会生成自己的
     <code>
      .svelte-kit/tsconfig.json
     </code>
     文件，您的配置将
     <code>
      extends
     </code>
     这个文件。
    </p>
    <h5>
     <a id="viteconfigjs_149">
     </a>
     vite.config.js
    </h5>
    <p>
     SvelteKit 项目实际上就是一个使用
     <a href="https://svelte.yayujs.com/docs/kit/@sveltejs-kit-vite" rel="nofollow">
      <code>
       @sveltejs/kit/vite
      </code>
     </a>
     插件的
     <a href="https://vitejs.dev" rel="nofollow">
      Vite
     </a>
     项目，以及其他
     <a href="https://vitejs.dev/config/" rel="nofollow">
      Vite 配置
     </a>
     。
    </p>
    <h4>
     <a id="_153">
     </a>
     其他文件
    </h4>
    <h5>
     <a id="sveltekit_155">
     </a>
     .svelte-kit
    </h5>
    <p>
     在开发和构建项目时，SvelteKit 将在
     <code>
      .svelte-kit
     </code>
     目录中生成文件（可通过
     <a href="https://svelte.yayujs.com/docs/kit/configuration#outDir" rel="nofollow">
      <code>
       outDir
      </code>
     </a>
     配置）。您可以忽略其内容，并随时删除它们（它们将在下次
     <code>
      dev
     </code>
     或
     <code>
      build
     </code>
     时重新生成）。
    </p>
    <h3>
     <a id="Web__159">
     </a>
     Web 标准
    </h3>
    <p>
     在本文档中，您会看到对 SvelteKit 所基于的标准
     <a href="https://developer.mozilla.org/en-US/docs/Web/API" rel="nofollow">
      Web APIs
     </a>
     的引用。我们没有重新发明轮子，而是
     <em>
      使用平台
     </em>
     ，这意味着您现有的 Web 开发技能可以应用到 SvelteKit。反之，花时间学习 SvelteKit 也会帮助您在其他地方成为更好的 Web 开发者。
    </p>
    <p>
     这些 API 在所有现代浏览器和许多非浏览器环境（如 Cloudflare Workers、Deno 和 Vercel Functions）中都可用。在开发过程中，以及在基于 Node 的环境（包括 AWS Lambda）的
     <a href="https://svelte.yayujs.com/docs/kit/adapters" rel="nofollow">
      适配器
     </a>
     中，必要时会通过 polyfills 提供这些 API（目前是这样 — Node 正在快速增加对更多 Web 标准的支持）。
    </p>
    <p>
     特别是，您会熟悉以下内容：
    </p>
    <h4>
     <a id="Fetch_APIs_167">
     </a>
     Fetch APIs
    </h4>
    <p>
     SvelteKit 使用
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/fetch" rel="nofollow">
      <code>
       fetch
      </code>
     </a>
     从网络获取数据。它在
     <a href="https://svelte.yayujs.com/docs/kit/hooks" rel="nofollow">
      hooks
     </a>
     和
     <a href="https://svelte.yayujs.com/docs/kit/routing#server" rel="nofollow">
      服务端路由
     </a>
     以及浏览器中都可用。
    </p>
    <blockquote>
     <p>
      [!NOTE] 在
      <a href="https://svelte.yayujs.com/docs/kit/load" rel="nofollow">
       <code>
        load
       </code>
      </a>
      函数、
      <a href="https://svelte.yayujs.com/docs/kit/hooks#Server-hooks" rel="nofollow">
       服务端 hooks
      </a>
      和
      <a href="https://svelte.yayujs.com/docs/kit/routing#server" rel="nofollow">
       API 路由
      </a>
      中提供了一个特殊版本的
      <code>
       fetch
      </code>
      ，用于在服务端渲染期间直接调用端点，而无需进行 HTTP 调用，同时保留凭证。（要在
      <code>
       load
      </code>
      之外的服务端代码中进行带凭证的请求，您必须显式传递
      <code>
       cookie
      </code>
      和/或
      <code>
       authorization
      </code>
      头部。）它还允许您进行相对地址请求，而服务端的
      <code>
       fetch
      </code>
      通常需要一个完整的 URL。
     </p>
    </blockquote>
    <p>
     除了
     <code>
      fetch
     </code>
     本身，
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API" rel="nofollow">
      Fetch API
     </a>
     还包括以下接口：
    </p>
    <h5>
     <a id="Request_175">
     </a>
     Request
    </h5>
    <p>
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Request" rel="nofollow">
      <code>
       Request
      </code>
     </a>
     的实例在
     <a href="https://svelte.yayujs.com/docs/kit/hooks" rel="nofollow">
      hooks
     </a>
     和
     <a href="https://svelte.yayujs.com/docs/kit/routing#server" rel="nofollow">
      服务端路由
     </a>
     中可以通过
     <code>
      event.request
     </code>
     访问。它包含有用的方法，如
     <code>
      request.json()
     </code>
     和
     <code>
      request.formData()
     </code>
     ，用于获取发送到端点的数据。
    </p>
    <h5>
     <a id="Response_179">
     </a>
     Response
    </h5>
    <p>
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Response" rel="nofollow">
      <code>
       Response
      </code>
     </a>
     的实例由
     <code>
      await fetch(...)
     </code>
     和
     <code>
      +server.js
     </code>
     文件中的处理程序返回。从根本上说，SvelteKit 应用就是一个将
     <code>
      Request
     </code>
     转换为
     <code>
      Response
     </code>
     的机器。
    </p>
    <h5>
     <a id="Headers_183">
     </a>
     Headers
    </h5>
    <p>
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Headers" rel="nofollow">
      <code>
       Headers
      </code>
     </a>
     接口允许您读取传入的
     <code>
      request.headers
     </code>
     和设置传出的
     <code>
      response.headers
     </code>
     。例如，您可以如下获取
     <code>
      request.headers
     </code>
     ，并使用
     <a href="https://svelte.yayujs.com/docs/kit/@sveltejs-kit#json" rel="nofollow">
      <code>
       json
      </code>
      便捷函数
     </a>
     发送修改后的
     <code>
      response.headers
     </code>
     ：
    </p>
    <pre><code class="prism language-js"><span class="token comment">// @errors: 2461</span>
<span class="token comment">/// file: src/routes/what-is-my-user-agent/+server.js</span>

     <a id="FormData_211">
     </a>
     FormData

    <p>
     当处理 HTML 原生表单提交时，您将使用
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/FormData" rel="nofollow">
      <code>
       FormData
      </code>
     </a>
     对象。
    </p>
    <pre><code class="prism language-js"><span class="token comment">// @errors: 2461</span>

    <h4>
     <a id="Stream_APIs_234">
     </a>
     Stream APIs
    </h4>
    <p>
     大多数情况下，您的端点会返回完整的数据，就像上面的
     <code>
      userAgent
     </code>
     示例那样。有时，您可能需要返回一个太大而无法一次性放入内存的响应，或者需要分块传递的响应，为此平台提供了
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Streams_API" rel="nofollow">
      流
     </a>
     —
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/ReadableStream" rel="nofollow">
      ReadableStream
     </a>
     、
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/WritableStream" rel="nofollow">
      WritableStream
     </a>
     和
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/TransformStream" rel="nofollow">
      TransformStream
     </a>
     。
    </p>
    <h4>
     <a id="URL_APIs_238">
     </a>
     URL APIs
    </h4>
    <p>
     URL 由
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/URL" rel="nofollow">
      <code>
       URL
      </code>
     </a>
     接口表示，它包含有用的属性如
     <code>
      origin
     </code>
     和
     <code>
      pathname
     </code>
     （在浏览器中还有
     <code>
      hash
     </code>
     ）。这个接口出现在各个地方 —
     <a href="https://svelte.yayujs.com/docs/kit/hooks" rel="nofollow">
      hooks
     </a>
     和
     <a href="https://svelte.yayujs.com/docs/kit/routing#server" rel="nofollow">
      服务端路由
     </a>
     中的
     <code>
      event.url
     </code>
     、
     <a href="https://svelte.yayujs.com/docs/kit/routing#page" rel="nofollow">
      页面
     </a>
     中的
     <a href="https://svelte.yayujs.com/docs/kit/$app-state" rel="nofollow">
      <code>
       page.url
      </code>
     </a>
     、
     <a href="https://svelte.yayujs.com/docs/kit/$app-navigation" rel="nofollow">
      <code>
       beforeNavigate
      </code>
      和
      <code>
       afterNavigate
      </code>
     </a>
     中的
     <code>
      from
     </code>
     和
     <code>
      to
     </code>
     等。
    </p>
    <h5>
     <a id="URLSearchParams_242">
     </a>
     URLSearchParams
    </h5>
    <p>
     无论在哪里遇到 URL，您都可以通过
     <code>
      url.searchParams
     </code>
     访问查询参数，它是
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams" rel="nofollow">
      <code>
       URLSearchParams
      </code>
     </a>
     的实例：
    </p>
    <pre><code class="prism language-js"><span class="token comment">// @filename: ambient.d.ts</span>

	<span class="token keyword">const</span> <span class="token literal-property property">url</span><span class="token operator">:</span> <span class="token constant">URL</span><span class="token punctuation">;</span>

    <h4>
     <a id="Web__259">

    </p>
