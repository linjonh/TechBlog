    <h4>
     <a id="___10">
     </a>
     一. !! 修饰器概述
    </h4>
    <p>
     <code>
      !!
     </code>
     是一种语法糖，用于实现父子组件之间的双向数据绑定。它主要用于初始化子组件中的
     <code>
      @Param
     </code>
     和
     <code>
      @Event
     </code>
     ，其中
     <code>
      @Event
     </code>
     的命名规则为
     <code>
      $ + @Param 属性名
     </code>
     。
    </p>
    <ul>
     <li>
      <strong>
       双向绑定
      </strong>
      ：当父子组件中使用
      <code>
       !!
      </code>
      语法时，父组件的数据变化会同步到子组件，子组件的数据变化也会同步到父组件。
     </li>
     <li>
      <strong>
       单向绑定
      </strong>
      ：如果父组件未使用
      <code>
       !!
      </code>
      语法，则数据流是单向的，父组件的变化不会同步到子组件。
     </li>
    </ul>

     <li>
      <strong>
       UI 刷新
      </strong>

     </li>
     <li>
      <strong>
       局限性
      </strong>
      ：
      <code>
       !!
      </code>
      语法糖不支持多层级组件之间的双向数据绑定。
     </li>
    </ul>

    <h4>
     <a id="_258">
     </a>
     三、总结
    </h4>
    <ol>
     <li>
      <strong>
       双向绑定
      </strong>
      ：
      <code>
       !!
      </code>
      语法糖简化了父子组件之间的双向数据绑定，提升了开发效率。
     </li>
     <li>
      <strong>
       单向绑定
      </strong>
      ：未使用
      <code>
       !!
      </code>
      时，数据流是单向的，子组件无法将数据变化同步回父组件。
     </li>
     <li>
      <strong>
       多层级数据传递
      </strong>
      ：
      <code>
       !!
      </code>
      语法糖不支持多层级组件之间的双向绑定，需通过回调函数手动实现。
     </li>
    </ol>
    <p>
     通过合理使用
     <code>
      !!
     </code>
     语法糖和回调机制，可以有效管理组件之间的数据流，确保数据的同步更新和 UI 的正确刷新。
    </p>
