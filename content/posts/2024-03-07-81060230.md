    <p>
     在上一篇文章，我们讲解了基于插件实现的上传图片到后台处理，并将处理后得到的结果图与原图在前端显示。
    </p>
    <p>
     主要idea是将图片先在后台进行缓存，我们传回两张图片的
     <strong>
      <span style="color:#ff0000;">
       途径
      </span>
     </strong>
     ，在
     <strong>
      <span style="font-size:16px;">
       file
      </span>
     </strong>
     的
     <strong>
      <span style="font-size:16px;">
       done
      </span>
     </strong>
     函数中对
     <strong>
      &lt;img/&gt;
     </strong>
     标签的
     <strong>
      src
     </strong>
     属性进行更新。
     <br/>
    </p>
    <p>
     但是这样子存在一个问题，就是后台“static/temp/left(right)”文件夹里的图片会越来越多。
    </p>
    <p>
     这时候就是那个submit按钮起作用了！我们先将得到的两个路径
     <strong>
      filepath
     </strong>
     和
     <strong>
      pilepath_
     </strong>
     用‘
     <strong>
      &amp;
     </strong>
     ’合并在一起，暂时存储在一个不可见的&lt;input type="text"/&gt;标签中，当submit被点击时，将两个途径传回后台，后台函数接收路径并对对应路径的文件做删除remove。
     <br/>
    </p>
    <pre><code class="language-html">&lt;!-- demo.html --&gt;
    &lt;body&gt;
    ……
    &lt;input id="temp" type="text" value="Nothing!" style="display: none"&gt;
    ……
&lt;/body&gt;</code></pre>
    <pre><code class="language-javascript">// javascript
$('#fileupload').fileupload{
    ...
    done.function(e, data){
        document.getElementById('temp').value = filepath + "&amp;" + filepath_;
    }
    ...
}</code></pre>
    <p>
     <strong>
      <span style="font-size:16px;">
       上一次我们已经定义了额外的submit按钮，现在我们要使用它。
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="font-size:18px;">
       1. 在后台即sketch.py中新定义函数delete()
      </span>
     </strong>
    </p>
    <pre><code class="language-python">@app.route('/delete', methods=['POST','GET'])
def delete():
    # do something
    return render_template('demo.html')</code></pre>
    <p>
     同样我们要注意路由的设置，我们假设这一次的交互是在一个隐藏的页面“http://127.0.0.1:5000/delete/”实现的。
    </p>
    <p>
     <strong>
      <span style="font-size:18px;">
       2. 前端“submit”标签与后端函数绑定
      </span>
     </strong>
    </p>
    <p>
     因为我们使用ajax进行数据传输，因此可以这样子做——
     <br/>
    </p>
    <pre><code class="language-javascript">$('#submit').on("click", function () {
    $('#progress .progress-bar').css(
        'width',
        0 + '%'
    );// 这里实现了当我们点击submit按钮后让进度条重新归0
    var filepath_ = document.getElementById('temp').value;// 获取刚刚存储在temp（text）上的数值
                
    // try to delete the original images
    var data = {
        'path': filepath_.split("&amp;")[0],
        'path_': filepath_.split("&amp;")[1]
    }// 对字符串做分割，并通过建立字典data进行分装
    $.ajax({
        type: 'GET',
        url:'{<!-- -->{ url_for("delete") }}',// 这里就实现了函数与html标签的绑定！！！
        data: data,                   // 要传给后台的数据
        dataType: 'json',             // 数据格式
        success: function(data){      // 当成功返回数据到后台并成功获取后台返回的数据时，采取的动作在success函数中定义，与前面的done是类似的
            
        }
    });
});</code></pre>
    <p>
     <span style="font-size:18px;">
      <strong>
       3. 完善后台delete()函数
      </strong>
     </span>
    </p>
    <pre><code class="language-python">@app.route('/delete', methods=['POST','GET'])
def delete():
    path = request.args.get('path')    # 因为数据是通过键值对匹配的，还记得前面data的两个键值对吗？
    path_ = request.args.get('path_')  # 通过“键”获取对应的“值”

    if os.path.isfile(path):    # 先判断文件是否存在，若是，才可以有remove的意义
        os.remove(path)
    if os.path.isfile(path_):
        os.remove(path_)
    return render_template('demo.html')    # 停留在本页面</code></pre>
    <p>
     <span style="font-size:24px;">
      <strong>
       总结
      </strong>
     </span>
    </p>
    <p>
     在本文我们是分享了如何实现普通按钮的click函数实现指定元素的value/text/innerHTML等的往后台的传送。
     <br/>
    </p>
    <p>
     对比上一篇博文，关于uploadfiles的实现，其大体思路还是比较相似的，即
     <br/>
    </p>
    <p>
     1）定义html标签
     <br/>
    </p>
    <p>
     2）重写该标签的功能函数，如
     <strong>
      file
     </strong>
     的
     <strong>
      <span style="color:#ff0000;">
       uploadfile
      </span>
     </strong>
     函数，
     <strong>
      button
     </strong>
     的
     <strong>
      <span style="color:#ff0000;">
       onclick
      </span>
     </strong>
     函数
     <br/>
    </p>
    <p>
     3）通过定义新的url进行与后端函数的绑定
     <br/>
    </p>
    <p>
     4）用
     <strong>
      js
     </strong>
     或
     <strong>
      jQuery
     </strong>
     代码获取并处理前台的相关数据，用字典进行封装
     <br/>
    </p>
    <p>
     5）后台接收包含数据的字典
     <strong>
      data
     </strong>
     ，通过“
     <strong>
      键
     </strong>
     ”获取“
     <strong>

     </strong>
