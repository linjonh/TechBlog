    <h3>
     <a id="Nodejs_2">
     </a>
     一、环境准备与Node.js安装
    </h3>
    <p>
     NPM（Node Package Manager）作为Node.js的默认包管理器，其安装需先完成Node.js环境的搭建：
    </p>
    <h4>
     <a id="1_NodejsNPM_5">
     </a>
     1. 下载Node.js（含NPM）
    </h4>
    <p>
     •
     <strong>
      官网选择
     </strong>
     ：访问
     <a href="https://nodejs.org/" rel="nofollow">
      Node.js官网
     </a>
     ，推荐下载
     <strong>
      LTS版本
     </strong>
     （长期支持版），该版本经过稳定性验证且维护周期长。
     <br/>
     •
     <strong>
      安装注意事项
     </strong>
     ：
     <br/>
     • Windows用户需勾选
     <code>
      Add to PATH
     </code>
     选项，否则需手动配置环境变量。
     <br/>
     • 建议自定义安装路径（如
     <code>
      D:\work\nodejs
     </code>
     ），避免C盘权限问题。
     <br/>
     • Mac/Linux用户可通过包管理器（如Homebrew、apt）安装。
    </p>
    <h4>
     <a id="2__12">
     </a>
     2. 验证安装
    </h4>
    <pre><code class="prism language-bash"><span class="token function">node</span> <span class="token parameter variable">-v</span>  <span class="token comment"># 输出示例：v20.13.1</span>
<span class="token function">npm</span> <span class="token parameter variable">-v</span>    <span class="token comment"># 输出示例：10.5.2</span>
</code></pre>
    <p>
     若提示命令不存在，需检查环境变量是否包含Node.js安装路径（Windows默认路径：
     <code>
      C:\Program Files\nodejs
     </code>
     ）。
    </p>
    <hr/>
    <h3>
     <a id="NPM_21">
     </a>
     二、NPM核心配置优化
    </h3>
    <h4>
     <a id="1__23">
     </a>
     1. 全局模块与缓存路径设置
    </h4>
    <p>
     默认全局安装路径易引发权限问题，建议自定义：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 创建目录（示例路径：D:\work\nodejs）</span>
<span class="token function">mkdir</span> node_global node_cache

<span class="token comment"># 配置路径</span>
<span class="token function">npm</span> config <span class="token builtin class-name">set</span> prefix <span class="token string">"D:\work<span class="token entity" title="\n">\n</span>odejs<span class="token entity" title="\n">\n</span>ode_global"</span>
<span class="token function">npm</span> config <span class="token builtin class-name">set</span> cache <span class="token string">"D:\work<span class="token entity" title="\n">\n</span>odejs<span class="token entity" title="\n">\n</span>ode_cache"</span>
</code></pre>
    <p>
     •
     <strong>
      环境变量追加
     </strong>
     （系统变量）：
     <br/>
     •
     <code>
      NODE_HOME
     </code>
     → Node.js安装路径
     <br/>
     •
     <code>
      Path
     </code>
     新增：
     <code>
      %NODE_HOME%\node_global
     </code>
     和
     <code>
      %NODE_HOME%\node_cache
     </code>
     。
    </p>
    <h4>
     <a id="2__37">
     </a>
     2. 镜像加速
    </h4>
    <p>
     国内用户建议配置淘宝镜像：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> registry https://registry.npmmirror.com
<span class="token comment"># 验证：npm config get registry</span>
</code></pre>
    <p>
     恢复官方源：
     <code>
      npm config set registry https://registry.npmjs.org/
     </code>
     。
    </p>
    <h4>
     <a id="3__45">
     </a>
     3. 代理配置（企业网络适用）
    </h4>
    <p>
     在
     <code>
      .npmrc
     </code>
     文件中添加：
    </p>
    <pre><code class="prism language-ini">proxy=http://your-proxy:port
https-proxy=http://your-proxy:port
registry=https://registry.npmmirror.com
</code></pre>
    <p>
     验证代理：
     <code>
      npm config get proxy
     </code>
     。
    </p>

    <h4>
     <a id="3__74">
     </a>
     3. 依赖锁定与版本管理
    </h4>
    <p>
     •
     <strong>
      锁定文件
     </strong>
     ：
     <code>
      package-lock.json
     </code>
     记录精确版本，使用
     <code>
      npm ci
     </code>
     可快速重建依赖树。
     <br/>
     •
     <strong>
      语义化版本
     </strong>
     ：
     <br/>
     •
     <code>
      ^1.2.3
     </code>
     ：允许次版本号和修订号更新
     <br/>
     •
     <code>
      ~1.2.3
     </code>
     ：仅允许修订号更新。
    </p>

    <h3>
     <a id="_103">
     </a>
     五、常见问题排查
    </h3>
    <h4>
     <a id="1__105">
     </a>
     1. 安装失败场景
    </h4>
    <p>
     •
     <strong>
      网络问题
     </strong>
     ：检查镜像源是否生效，切换为官方源测试。
     <br/>
     •
     <strong>
      权限不足
     </strong>
     ：Windows用户避免安装到C盘默认路径；使用管理员终端。
     <br/>
     •
     <strong>
      依赖冲突
     </strong>
     ：删除
     <code>
      node_modules
     </code>
     和
     <code>
      package-lock.json
     </code>
     后重试。
    </p>
    <h4>
     <a id="2_Windows_110">
     </a>
     2. 脚本执行权限（Windows）
    </h4>
    <p>
     若提示“禁止运行脚本”，需在PowerShell执行：
    </p>
    <pre><code class="prism language-powershell"><span class="token function">Set-ExecutionPolicy</span> RemoteSigned <span class="token operator">-</span>Scope CurrentUser
</code></pre>
    <p>
     需管理员权限操作。
    </p>
    <hr/>
    <h3>
     <a id="_119">
     </a>
     六、最佳实践建议
    </h3>
    <ol>
     <li>
      <strong>
       依赖管理
      </strong>
      ：定期运行
      <code>
       npm outdated
      </code>
      检查更新，使用
      <code>
       npm update
      </code>
      升级小版本。
     </li>
     <li>
      <strong>
       项目协作
      </strong>
      ：始终提交
      <code>
       package.json
      </code>
      ，忽略
      <code>
       node_modules
      </code>
      目录。
     </li>
     <li>
      <strong>
       安全审计
      </strong>
      ：定期执行
      <code>
       npm audit
      </code>
      扫描漏洞。
     </li>
    </ol>
