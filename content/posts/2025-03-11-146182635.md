    <h2>
     <a id="_0">
     </a>
     问题描述：
    </h2>
    <p>
     利用 conda 安装某包
    </p>
    <pre><code class="prism language-bash">conda <span class="token function">install</span> <span class="token parameter variable">-c</span> <span class="token string">"nvidia/label/cuda-11.8.0"</span> cuda-nvcc
</code></pre>
    <p>
     时发现报错：
    </p>
    <pre><code class="prism language-bash">Collecting package metadata <span class="token punctuation">(</span>current_repodata.json<span class="token punctuation">)</span>: <span class="token keyword">done</span>
Solving environment: failed with initial frozen solve. Retrying with flexible solve.
Solving environment: failed with repodata from current_repodata.json, will retry with next repodata source.
Collecting package metadata <span class="token punctuation">(</span>repodata.json<span class="token punctuation">)</span>: <span class="token keyword">done</span>

Solving environment: <span class="token punctuation">\</span> 
Found conflicts<span class="token operator">!</span> Looking <span class="token keyword">for</span> incompatible packages.                                    failed                                                                                     

UnsatisfiableError: The following specifications were found to be incompatible with each other:

Output <span class="token keyword">in</span> format: Requested package -<span class="token operator">&gt;</span> Available versionsThe following specifications were found to be incompatible
</code></pre>
    <p>
     具体来说：
     <br/>
     <code>
      Found conflicts! Looking for incompatible packages.
     </code>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="_25">
     </a>
     解决方案：
    </h2>
    <ol>
     <li>
      博客【
      <a href="https://blog.csdn.net/weixin_42001274/article/details/127209878">
       conda安装软件报兼容性问题的解决 Found conflicts! Looking for incompatible packages.failed.UnsatisfiableError
      </a>
      】认为是镜像的问题，在报错的服务器添加国内镜像或者升级一些特定库。
     </li>
     <li>
      以上方法均对本人无效，经过排查发现，是因为之前删除 conda 环境时未删除干净，导致新建的该环境里依旧识别到原来同名环境里的库。采用
      <code>
       pip list
      </code>
      发现了很多尚未安装的库。解决方法是首先退出当前激活的虚拟环境，使用
      <code>
       conda deactivate
      </code>
      命令。然后，使用
      <code>
       conda remove -n your_env_name --all
      </code>
      命令来删除指定的虚拟环境。请将
      <code>
       your_env_name
      </code>
      替换为你要删除的虚拟环境的名称。
      <code>
       --all
      </code>
      一定不能省略，否则卸载之后可能还会存在原环境的文件夹，导致用此名称再次新建环境时发生冲突。
     </li>
    </ol>
