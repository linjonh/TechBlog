    <h2>
     <a id="Macjava_0">
     </a>
     Mac配置java环境变量
    </h2>
    <h3>
     <a id="_2">
     </a>
     一、明确定义
    </h3>
    <ul>
     <li>
      <p>
       JRE：（Java Runtime Environment），java运行环境。包括Java虚拟机(JVM Java Virtual Machine)和Java程序所需的核心类库等，如果想要运行一个开发好的Java程序，计算机中只需要安装JRE 即可。
      </p>
     </li>
     <li>
      <p>
       JDK：（Java Development Kit Java）开发工具包。JDK是提供给Java开发人员使用的，其中包含了java的开发工具，也包括了JRE。所以安装了JDK，就不用在单独安装JRE了。
      </p>
     </li>
     <li>
      <p>
       "/Library/Internet Plug-Ins/JavaAppletPlugin.plugin/Contents/Home/bin/java"这个是JRE的运行路径。
      </p>
     </li>
     <li>
      <p>
       "/Library/Java/JavaVirtualMachines/jdk-21.jdk/Contents/Home"这个是JDK的运行路径。
      </p>
     </li>
    </ul>
    <p>
     我们要配置的java环境变量需要去java官网下载jdk的dmg文件，根据Mac的处理器不同选择适合处理器的jdk文件，我这里示范使用的是Intel芯片的处理器。
    </p>
    <h3>
     <a id="Jdk_12">
     </a>
     二、安装Jdk
    </h3>
    <p>
     双击下载好的jdk文件进行安装，安装过程中一直点击继续或下一步即可安装成功。
    </p>
    <h3>
     <a id="_16">
     </a>
     三、配置环境变量
    </h3>
    <ul>
     <li>
      <p>
       首先找到jdk的安装路径，可以参考前面所说的jdk运行路径进行寻找。
      </p>
     </li>
     <li>
      <p>
       也可以使用系统的资源库查找jdk路径，具体方法如下：
      </p>
     </li>
    </ul>
    <p>
     首先，找到访达图标，点开访达，点击前往菜单，选择前往文件夹。
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1479c0deba2722735a638e3045f7a08c.png"/>
    </p>
    <p>
     然后，在对话框中输入/Library/Java/，按enter，即可进入资源库中的Java文件中，然后进入JavaVirtualMachines文件，进入后可以选择jdk的版本，选择好版本后依次进入Contents文件和Home文件，即可找到jdk的bin文件。
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f13f9cb284e7a6398a599ccb8b4aea4c.png">
      <br/>
      <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/029e880f71647d48f8a1dbf4bc2280bb.png"/>
     </img>
    </p>
    <p>
     找到bin文件后在bin文件上右键，然后按住option可以看到拷贝变成了，将bin拷贝为路径名称，选择此选项即可将路径复制下来。
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d2dc1857bfa521f0713a2f43a5abfa1b.png"/>
    </p>
    <p>
     在正式配置之前可以先查看以下已有的环境变量，在终端中输入：
    </p>
    <pre><code>echo $PATH
</code></pre>
    <p>
     命令行后的即是现在的环境变量，输入以下命令行：
    </p>
    <pre><code>vi  .bash_profile
</code></pre>
    <p>
     开始配置环境变量,输入该命令就会进入一个界面，然后敲一下键盘上的i键,只有敲了才能进入文本输入模式。
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/dbaac4776343c982f7f4b72b0bcfcdcc.png"/>
    </p>
    <p>
     在文本输入模式输入下列代码
    </p>
    <pre><code>JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-21.jdk/Contents/Home  
PATH=$JAVA_HOME/bin:$PATH  //给环境变量赋值
export JAVA_HOME          //导出使其生效
export PATH 
</code></pre>
    <p>
     注意上面JAVA_HOME的值一定要是你自己电脑上的jdk路径。和刚刚的/Library/Java/JavaVirtualMachines/jdk-21.jdk/Contents/Home/bin路径有差别的地方是JAVA_HOME后面的路径少了/bin。然后把/bin加在了PATH后面。
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/eb09b26383f3a852b056e9e5e8c1474a.png"/>
    </p>
    <p>
     退出保存环境变量，先摁 ESC键 返回命名行模式。输入:wq 退出保存环境变量。
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d9e28b3ad75c4f131975b6f6d8c88a58.png"/>
    </p>
    <p>
     在终端输入下面命令行让环境变量生效：
    </p>
    <pre><code>source  .bash_profile
</code></pre>
    <p>
     再次使用echo $PATH命令查看环境变量是否添加完成，若我们的环境变量多出我们刚刚配置好的jdk路径，说明配置成功，但还没有结束。
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b20686ca583c14b99380189cc6545024.png"/>
    </p>
    <h3>
     <a id="_83">
     </a>
     四、必读
    </h3>
    <ul>
     <li>
      由于我们配置的环境变量是在bash_profile中，当终端重新开启新的窗口或重启终端后，配置的环境变量都不生效了，还需重新执行以下命令才会生效：
     </li>
    </ul>
    <pre><code>source ~/.bash_profile
</code></pre>
    <ul>
     <li>
      其原因是我们配置的环境变量是在bash中的，而终端默认使用的shell是zsh，所以解决办法是在.zshrc文件中添加
     </li>
    </ul>
    <pre><code>source ~/.bash_profile  //此步类似于在使用zsh时的shell时提前执行这句命令，很像提前加载（仅个人理解）
</code></pre>
    <ul>
     <li>
      要是不知道此时使用的shell是什么类型可以使用下列命令来查看：
     </li>
    </ul>
    <pre><code>echo $SHELL
</code></pre>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/685dbcecfa19ab13a9285a88e680e1e9.png"/>
    </p>
    <p>
     如图所示/bin/zsh就意味着此时使用的是zsh的shell。
    </p>
    <ul>
     <li>
      首先进入主目录
     </li>
    </ul>
    <pre><code>cd ~
</code></pre>
    <ul>
     <li>
      然后显示隐藏文件，找到.zshrc文件夹
     </li>
    </ul>
    <pre><code>ls -a
</code></pre>
    <ul>
     <li>
      编辑文件
     </li>
    </ul>
    <pre><code>sudo vim ./zshrc
</code></pre>
    <ul>
     <li>
      按i进入编辑模式，加入下列代码
     </li>
    </ul>
    <pre><code>source ~/.bash_profile
</code></pre>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b8f5dc8baefe44b47c7649d85b79d904.png"/>
    </p>
    <ul>
     <li>
      按Esc键退出编辑状态输入
     </li>
    </ul>
    <pre><code>:wq
</code></pre>
    <ul>
     <li>
      上面这些代码的意思是保存退出（write &amp; quit）
     </li>
    </ul>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c11de6e4ce3240e31418ce27f7285b5f.png"/>
    </p>
    <ul>
     <li>
      <p>
       这个过程中需要输入开机密码，且为隐藏输入看不到输入过程，输入正确按下enter即可。
      </p>
     </li>
     <li>
      <p>
       最后重启终端并在终端输入：
      </p>
     </li>
    </ul>
    <pre><code>echo $PATH
或
env
</code></pre>
    <ul>
     <li>
      这两种方式均可查看环境变量是否添加成功。
     </li>
    </ul>
    <p>
     最终java环境变量才算真正添加成功。
    </p>
