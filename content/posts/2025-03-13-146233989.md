     Git的详细使用方法
    </h1>
   </div>
  </div>

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

    <p>
     Git 是一个分布式版本控制系统，用于跟踪和管理代码的变更。以下是 Git 的详细使用方法：
    </p>
    <hr/>
    <h2>
     <strong>
      1. 安装 Git
     </strong>
    </h2>
    <ul>
     <li>
      <h3>
       <strong>
        Windows
       </strong>
       ：从
       <a href="https://git-scm.com/" rel="nofollow" title="Git 官网">
        Git 官网
       </a>
       下载安装包。
      </h3>
     </li>
     <li>
      <h3>
       <strong>
        Linux
       </strong>
       （Ubuntu/Debian）
      </h3>
      <pre><code class="language-bash">sudo apt install git</code></pre>
     </li>
     <li>
      <h3>
       <strong>
        macOS
       </strong>
       ：
      </h3>
     </li>
     <li>
      <p>
       使用 Homebrew。
      </p>
      <pre><code class="language-bash">brew install git</code></pre>
     </li>
    </ul>
    <h3>
     验证安装
    </h3>
    <pre><code class="language-bash">git --version</code></pre>
    <hr/>
    <h2>
     <strong>
      2. 配置用户信息
     </strong>
    </h2>
    <h3>
     首次使用
    </h3>
    <p>
     首次使用时，Git 前需配置全局用户名和邮箱：
    </p>
    <pre><code class="language-bash">git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"</code></pre>
    <h3>
     查看配置
    </h3>
    <pre><code class="language-bash">git config --list</code></pre>
    <hr/>
    <h2>
     <strong>
      3. 创建仓库
     </strong>
    </h2>
    <h3>
     <strong>
      初始化新仓库
     </strong>
    </h3>
    <pre><code class="language-bash">git init</code></pre>
    <h3>
     <strong>
      克隆现有仓库
     </strong>
    </h3>
    <pre><code class="language-bash">git clone https://github.com/user/test.git</code></pre>
    <hr/>
    <h2>
     <strong>
      4. 基础操作
     </strong>
    </h2>
    <p>
     <strong>
      修改文件
     </strong>
    </p>
    <h3>
     在工作目录中修改或新增文件。
    </h3>
    <h3>
     <strong>
      添加到暂存区
     </strong>
    </h3>
    <pre><code class="language-bash">git add &lt;file&gt;       # 添加单个文件
git add .            # 添加所有修改</code></pre>
    <h3>
     <strong>
      提交到本地仓库
     </strong>
    </h3>
    <pre><code class="language-bash">git commit -m "说明"</code></pre>
    <h3>
     <strong>
      查看状态
     </strong>
    </h3>
    <pre><code class="language-bash">git status</code></pre>
    <h3>
     <strong>
      查看提交历史
     </strong>
    </h3>
    <pre><code class="language-bash">git log              # 完整的提交历史
git log --oneline    # 简单的提交历史
git log --graph      # 图形化分支历史</code></pre>
    <hr/>
    <h2>
     <strong>
      5. 分支管理
     </strong>
    </h2>
    <h3>
     <strong>
      创建分支
     </strong>
    </h3>
    <pre><code class="language-bash">git branch &lt;branch-name&gt;</code></pre>
    <h3>
     <strong>
      切换分支
     </strong>
    </h3>
    <pre><code class="language-bash">git checkout &lt;branch-name&gt;
# 或创建并切换到新分支
git checkout -b &lt;branch-name&gt;</code></pre>
    <h3>
     <strong>
      合并分支
     </strong>
    </h3>
    <pre><code class="language-bash">git checkout master
git merge &lt;branch-name&gt;</code></pre>
    <h3>
     <strong>
      删除分支
     </strong>
    </h3>
    <pre><code class="language-bash">git branch -d &lt;branch-name&gt;  # 删除本地分支
git push origin --delete &lt;branch-name&gt;  # 删除远程分支</code></pre>
    <h3>
     <strong>
      解决冲突
     </strong>
    </h3>
    <p>
     手动编辑冲突文件后，执行
     <code>
      git add
     </code>
     和
     <code>
      git commit
     </code>
     。
    </p>
    <hr/>
    <h2>
     <strong>
      6. 远程仓库
     </strong>
    </h2>
    <ul>
     <li>
      <h3>
       <strong>
        关联远程仓库
       </strong>
      </h3>
      <pre><code class="language-bash">git remote add origin https://github.com/user/test.git</code></pre>
     </li>
     <li>
      <h3>
       <strong>
        查看远程仓库
       </strong>
      </h3>
      <pre><code class="language-bash">git remote -v</code></pre>
     </li>
     <li>
      <h3>
       <strong>
        推送代码
       </strong>
      </h3>
      <pre><code class="language-bash">git push origin &lt;branch-name&gt;
# 首次推送时指定分支：
git push -u origin &lt;branch-name&gt;</code></pre>
     </li>
     <li>
      <h3>
       <strong>
        拉取代码
       </strong>
      </h3>
      <pre><code class="language-bash">git pull origin &lt;branch-name&gt;  # 拉取并合并
git fetch origin               # 仅下载远程变更</code></pre>
     </li>
    </ul>
    <hr/>
    <h2>
     <strong>
      7. 高级操作
     </strong>
    </h2>
    <h3>
     <strong>
      撤销更改
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        撤销工作区修改
       </strong>
      </p>
      <pre><code class="language-bash">git checkout -- &lt;file&gt;</code></pre>
     </li>
     <li>
      <p>
       <strong>
        撤销暂存区修改
       </strong>
      </p>
      <pre><code class="language-bash">git reset HEAD &lt;file&gt;</code></pre>
     </li>
     <li>
      <p>
       <strong>
        回退到某次提交
       </strong>
      </p>
      <pre><code class="language-bash">git reset --hard &lt;commit-id&gt;  # 彻底回退（请慎用）
git revert &lt;commit-id&gt;        # 生成新提交以撤销旧提交</code></pre>
     </li>
    </ul>
    <h3>
     <strong>
      标签管理
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        创建标签
       </strong>
      </p>
      <pre><code class="language-bash">git tag v1.0.0
git tag -a v1.0.0 -m "版本说明"</code></pre>
     </li>
     <li>
      <p>
       <strong>
        推送标签到远程
       </strong>
      </p>
      <pre><code class="language-bash">git push origin --tags</code></pre>
     </li>
    </ul>
    <h3>
     <strong>
      暂存修改
     </strong>
    </h3>
    <pre><code class="language-bash">git stash        # 暂存当前修改
git stash pop    # 恢复暂存的修改</code></pre>
    <h3>
     <strong>
      比较差异
     </strong>
    </h3>
    <pre><code class="language-bash">git diff          # 工作区与暂存区的差异
git diff --staged # 暂存区与最新提交的差异</code></pre>
    <hr/>
    <h2>
     <strong>
      8. 协作流程（示例）
     </strong>
    </h2>
    <h3>
     克隆仓库
    </h3>
    <pre><code class="language-bash">git clone https://github.com/user/test.git</code></pre>
    <h3>
     创建新分支开发
    </h3>
    <pre><code class="language-bash">git checkout -b feature/new-feature</code></pre>
    <h3>
     开发完成后提交
    </h3>
    <pre><code class="language-bash">git add .
git commit -m "版本说明"</code></pre>
    <h3>
     推送分支到远程
    </h3>
    <pre><code class="language-bash">git push origin feature/new-feature</code></pre>
    <h3>
     Pull Request（PR）或 Merge Request（MR）
    </h3>
    <p>
     在 GitHub/Gitee 上发起 Pull Request（PR）或 Merge Request（MR）。
    </p>
    <h3>
     合并分支
    </h3>
    <p>
     审核通过后合并到主分支（如
     <code>
      master
     </code>
     ）。
    </p>
    <hr/>
    <h2>
     <strong>
      9. 配置与别名
     </strong>
    </h2>
    <h3>
     <strong>
      忽略文件
     </strong>
    </h3>
    <p>
     在项目根目录创建
     <code>
      .gitignore
     </code>
     ，列出需忽略的文件/目录（如
     <code>
      node_modules/
     </code>
     ,
     <code>
      .env
     </code>
     ）。
    </p>
    <h3>
     <strong>
      设置别名
     </strong>
    </h3>
    <p>
     设置别名可以简化命令
    </p>
    <pre><code class="language-bash">git config --global alias.co checkout
git config --global alias.br branch</code></pre>
    <hr/>
    <h2>
     <strong>
      10. 常见问题
     </strong>
    </h2>
    <h3>
     <strong>
      提交到错误分支
     </strong>
    </h3>
    <p>
     使用
     <code>
      git reset
     </code>
     回退提交，再切换到正确分支提交。
    </p>
    <h3>
     <strong>
      误删分支
     </strong>
    </h3>
    <p>
     通过
     <code>
      git reflog
     </code>
     查找提交记录恢复。
    </p>
