    <h3>
     <a id="_12">
     </a>
     问题描述
    </h3>
    <p>
     我想向一个 hash 类型的 key 中插入多个 键值对 value。
    </p>
    <p>
     在 redis-cli 中我可以使用
     <code>
      HMSET
     </code>
     命令实现这种批量插入，但是当我使用 redis library 4.1.4 中的
     <code>
      hmset
     </code>
     方法时报错：
     <code>
      DeprecationWarning: Redis.hmset() is deprecated. Use Redis.hset() instead. conn.hmset(article, content)
     </code>
    </p>
    <p>
     代码如下：
    </p>
    <pre><code>import time
import redis3

conn = redis3.Redis(host='localhost', port=6379, db=0, decode_responses=True)
conn.flushdb()

now = time.time()
article = 'article:' + str(1)

content = {
    'title': 'A title',
    'link': 'http://www.google.com',
    'poster': 'username',
    'time': now,
    'votes': 1,
}

conn.hmset(article, content)
print(conn.hgetall(article))
</code></pre>
    <p>
     查了一下，根据
     <a href="https://github.com/redis/redis-py/issues/1269">
      hmset() uses deprecated command HMSET #1269
     </a>
     的说法，
     <code>
      hmset
     </code>
     方法从 4.0.0 版本开始就被弃用了。
    </p>
    <p>
     尽管这里报错，但实际上它还是起作用了，因为它并没有被从 redis library 4.1.4 的 API 源码中移出。
    </p>
    <p>
     因为它早晚会被移除，我就想从
     <code>
      hmse
     </code>
     迁移到
     <code>
      hset
     </code>
     ，于是这就产生了一些问题：
    </p>
    <ul>
     <li>
      怎样使用
      <code>
       hset
      </code>
      API。
     </li>
     <li>
      在使用
      <code>
       hset
      </code>
      API时遇到了一些问题。
     </li>
    </ul>
    <h3>
     <a id="_51">
     </a>
     解决过程
    </h3>
    <h4>
     <a id="_54">
     </a>
     过程及方法一：
    </h4>
    <p>
     使用循环，但这样在处理大数据时会比较慢：
    </p>
    <pre><code>from pyinstrument import Profiler
profiler=Profiler()
profiler.start()
for i in range(100000):
    for item in content:
        conn.hset(article, item, content[item])
profiler.stop()
profiler.print()
</code></pre>

    <h4>
     <a id="_82">
     </a>
     过程及方法二：
    </h4>
    <p>
     根据源码以及
     <a href="https://github.com/redis/redis-py/issues/1396">
      hset mapping not working #1396
     </a>
     的说法，我使用
     <code>
      mapping
     </code>
     参数映射一下：
    </p>
    <pre><code>conn.hset(article, mapping=content)
</code></pre>
    <p>
     但报错：
     <code>
      redis3.exceptions.ResponseError: wrong number of arguments for 'hset' command
     </code>
    </p>
    <p>
     按照
     <a href="https://redis-py.readthedocs.io/en/stable/commands.html#redis.commands.core.CoreCommands.hset" rel="nofollow">
      redis doc：hset
     </a>
     的说法，
     <code>
      mapping
     </code>
     参数应该是支持以 python 字典形式插入 hash 类型的数据的，就在想是不是本地 windows redis 版本的问题，根据
     <a href="https://github.com/redis/redis/issues/8622">
      [BUG] hset does not support multiple dict mapping #8622
     </a>
     中的讨论，去查了一下本地 windows redis 版本：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5e7cb133e7c8cdf5507d8997223734ed.png">
      <br/>
      好家伙，版本太低了，要么直接卸载，然后手动编译重装 6.X 版本，要么还是在 linux 环境操作：
     </img>
    </p>
