    <p>
     <strong>
      外观模式（Facade Pattern）
     </strong>
     是一种
     <strong>
      结构型设计模式
     </strong>
     ，旨在为复杂的子系统提供一个统一的、简化的高层接口，使得客户端更容易使用子系统功能。它的核心思想是
     <strong>
      隐藏系统复杂性，提供易用入口
     </strong>
     。
    </p>
    <h4>
     <strong>
      1. 核心角色
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        角色
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         外观类（Facade）
        </strong>
       </td>
       <td>
        提供统一的接口，将客户端的请求转发到子系统中的适当对象。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         子系统类（Subsystem Classes）
        </strong>
       </td>
       <td>
        实现子系统的具体功能，外观类通过组合这些类来完成任务。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      2. 应用场景
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        简化复杂系统
       </strong>
       ：当系统存在多个复杂模块或接口时，提供一个统一入口。
      </p>
     </li>
     <li>
      <p>
       <strong>
        降低耦合度
       </strong>
       ：减少客户端与子系统的直接依赖，提高系统可维护性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        分层架构
       </strong>
       ：为不同层次的模块提供统一接口（如API网关）。
      </p>
     </li>
    </ul>

    <h4>
     <strong>
      5. 外观模式优缺点
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         优点
        </strong>
       </th>
       <th>
        <strong>
         缺点
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        简化客户端调用，隐藏系统复杂性。
       </td>
       <td>
        可能引入单点故障（外观类成为瓶颈）。
       </td>
      </tr>
      <tr>
       <td>
        降低客户端与子系统的耦合度。
       </td>
       <td>
        违反开闭原则（修改子系统需调整外观类）。
       </td>
      </tr>
      <tr>
       <td>
        提升代码可维护性和可读性。
       </td>
       <td>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      6. 外观模式 vs. 中介者模式
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         模式
        </strong>
       </th>
       <th>
        <strong>
         目的
        </strong>
       </th>
       <th>
        <strong>
         关注点
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         外观模式
        </strong>
       </td>
       <td>
        简化子系统接口，提供统一入口。
       </td>
       <td>
        <strong>
         单向通信
        </strong>
        （外观调用子系统）。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         中介者模式
        </strong>
       </td>
       <td>
        协调多个对象间的交互，减少直接通信。
       </td>
       <td>
        <strong>
         双向通信
        </strong>
        （对象间通过中介者交互）。
       </td>
      </tr>
     </tbody>
    </table>

    <h4>
     <strong>
      7. 实际应用场景
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        API 网关
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          外观类
         </strong>
         ：网关统一处理鉴权、限流、路由。
        </p>
       </li>
       <li>
        <p>
         <strong>
          子系统
         </strong>
         ：多个微服务（订单、用户、支付）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        日志框架
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          外观类
         </strong>
         ：
         <code>
          Logger
         </code>
         接口（如 SLF4J）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          子系统
         </strong>
         ：具体日志实现（Logback、Log4j2）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        文件压缩工具
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          外观类
         </strong>
         ：提供
         <code>
          compress()
         </code>
         和
         <code>
          decompress()
         </code>
         方法。
        </p>
       </li>
       <li>
        <p>
         <strong>
          子系统
         </strong>
         ：ZIP、RAR、7z 等压缩算法。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      9. 总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        核心价值
       </strong>
       ：通过封装复杂子系统，提供简洁易用的接口。
      </p>
     </li>
     <li>
      <p>
       <strong>
        关键设计
       </strong>
       ：外观类通过组合子系统对象，对外暴露统一方法。
      </p>
     </li>
     <li>
      <p>
       <strong>
        适用性
       </strong>
       ：当系统复杂度高且需要简化客户端调用时，优先选择外观模式。
      </p>
     </li>
    </ul>
