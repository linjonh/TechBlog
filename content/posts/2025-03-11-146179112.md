    <h3>
     <a id="scope_19">
     </a>
     依赖范围scope
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        依赖范围
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        compile
       </td>
       <td>
        编译依赖范围。默认值，在编译、测试、运行环境下都可使用。依赖关系中可以进行传递
       </td>
      </tr>
      <tr>
       <td>
        test
       </td>
       <td>
        测试依赖范围。只有在测试环境下可用，比如：junit
       </td>
      </tr>
      <tr>
       <td>
        provided
       </td>
       <td>
        已提供依赖范围。在编译、测试环境下可用，比如：servlet-api（在服务器端已提供此jar包，不需要在运行时再次引入）
       </td>
      </tr>
      <tr>
       <td>
        runtime
       </td>
       <td>
        运行时依赖范围。在测试、运行环境下可用，比如：jdbc
       </td>
      </tr>
      <tr>
       <td>
        system
       </td>
       <td>
        系统依赖范围。在编译、测试环境下可用，需指定本地依赖路径，可移植性变差，不推荐使用
       </td>
      </tr>
      <tr>
       <td>
        import
       </td>
       <td>
        导入依赖范围。
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="_29">
     </a>
     依赖传递原则
    </h3>
    <ol>
     <li>
      只有当依赖是compile时，才可向上传递
     </li>
     <li>
      强制中止依赖传递：
      <code>
       &lt;optional&gt;true&lt;/optional&gt;
      </code>
     </li>
    </ol>
    <h3>
     <a id="_34">
     </a>
     依赖冲突原则
    </h3>
    <ol>
     <li>
      <strong>
       路径最短优先原则
      </strong>
      <br/>
      当A依赖B，B依赖Druid-1.2.7，A依赖Druid-1.2.8，则优先使用Druid-1.2.8
     </li>
     <li>
      <strong>
       当路径长度相同，先声明优先原则
      </strong>
      <br/>
      当A依赖B，B依赖Druid-1.2.7，A依赖C，C依赖Druid-1.2.8，则按照B、C在dependencies中的顺序决定
     </li>
     <li>
      <strong>
       手动依赖排除，exclusions
      </strong>
      <br/>
      手动排除的依赖，不会向上传递
