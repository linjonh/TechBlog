    <p>
     最近在看《JavaScript高级程序设计（第4版）》，运行书中的一个代码示例时，偶然发现了 setTimeout 的一些之前没注意到的特性，觉得挺有意思的，就来记录一下。
    </p>
    <p>
     书中代码如下：
    </p>
    <pre><code class="language-javascript">for (var i = 0; i &lt; 5; ++i) {
    setTimeout(() =&gt; console.log(i), 0);
}
//你可能以为会输出 0 1 2 3 4
//实际上会输出 5 5 5 5 5</code></pre>
    <p>
     以上结果，书中的解释是：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     使用 var 声明迭代变量，在退出循环时，迭代变量保存的是导致循环退出的值。在之后执行超时逻辑时，所有的 i 都是同一个变量，因而输出的都是同一个最终值。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     使用 let 声明迭代变量，JavaScript 引擎在后台会为每个迭代循环声明一个新的迭代变量。每个 setTimeout 引用的都是不同的变量实例，所以输出的是循环执行过程中每个迭代变量的值。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p>
     我的测试代码：
    </p>
    <pre><code class="language-javascript">&lt;script&gt;
  "use strict";

  function testForVar() {
    for (var i = 0; i &lt; 5; i++) {
      console.log("这是 testForVar 第" + Number(i + 1) + "次循环");
      //var msg = i; //效果一样
      let msg = i;
      setTimeout(
        (val) =&gt; {
          console.log(val + " VS " + i);
        },
        0,
        msg
      );
    }
  }

  testForVar();
  console.log("******");

  function testForLet() {
    // 使用 let 来定义循环变量，避免变量提升问题
    for (let i = 0; i &lt; 5; i++) {
      //console.log(i === i);
      console.log("这是 testForLet 第" + Number(i + 1) + "次循环");
      //var msg = i; //效果一样
      let msg = i;
      setTimeout(
        (val) =&gt; {
          console.log(val + " VS " + i);
        },
        0,
        msg
      );
    }
  }

  testForLet();
&lt;/script&gt;</code></pre>

     输出结果如下：
    </p>
    <p>
     <img alt="" height="674" src="https://i-blog.csdnimg.cn/direct/977f199fc7584b22a833cccb76b25c29.png" width="260"/>
    </p>
    <p>
     注：setTimeout 用于在指定的毫秒数后执行一次指定的代码或函数。
    </p>
    <p>
     有趣的是或之前没注意到的 setTimeout 特性：
    </p>
    <p>
     1、setTimeout 并不是精确的定时器，会延迟执行，因为JavaScript引擎在执行完所有的同步代码后才会检查事件队列，而且在这个过程中可能发生其他任务的阻塞。
    </p>
    <p>
     2、setTimeout  即便设置的毫秒数是0，也不会立即执行。因为浏览器有设置最小延时，比如Chrome 设置 setTimeout 最小延时为4毫秒。
    </p>
    <p>
     3、setTimeout  传参，IE浏览器不支持 setTimeout  参数传递，Chrome 中可以使用箭头函数或匿名函数传参。
    </p>
