    <p>
     在多线程编程中，有时需要确保一个线程必须等待另一个线程执行完毕后再执行。本文将介绍几种常见的方法来实现这一需求，并提供详细的代码示例。
    </p>
    <p>
     <strong>
      1. 使用 Thread.join()
     </strong>
     <br/>
     <strong>
      Thread.join()
     </strong>
     是最简单直接的方法，它会让当前线程等待目标线程执行完毕。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>Thread thread1 = new Thread(() -&gt; {
    System.out.println("线程1执行");
});

Thread thread2 = new Thread(() -&gt; {
    try {
        thread1.join(); // 等待线程1执行完毕
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    System.out.println("线程2执行");
});

thread1.start();
thread2.start();
</code></pre>
    <p>
     <strong>
      2. 使用 CountDownLatch
     </strong>
     <br/>
     <strong>
      CountDownLatch
     </strong>
     是一个同步工具，可以让线程等待其他线程完成。通过 countDown() 和 await() 方法实现线程间的同步。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>CountDownLatch latch = new CountDownLatch(1);

Thread thread1 = new Thread(() -&gt; {
    System.out.println("线程1执行");
    latch.countDown(); // 通知线程2可以执行
});

Thread thread2 = new Thread(() -&gt; {
    try {
        latch.await(); // 等待线程1执行完毕
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    System.out.println("线程2执行");
});

thread1.start();
thread2.start();
</code></pre>
    <p>
     <strong>
      3. 使用 ExecutorService 和单线程池
     </strong>
     <br/>
     通过 Executors.newSingleThreadExecutor() 创建单线程池，任务会按提交顺序执行，确保线程2在线程1之后执行。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>ExecutorService executor = Executors.newSingleThreadExecutor();

executor.submit(() -&gt; {
    System.out.println("线程1执行");
});

executor.submit(() -&gt; {
    System.out.println("线程2执行");
});

executor.shutdown(); // 关闭线程池
</code></pre>
    <p>
     <strong>
      4. 使用 CompletableFuture
     </strong>
     <br/>
     <strong>
      CompletableFuture
     </strong>
     是 Java 8 引入的异步编程工具，可以方便地实现任务依赖。
    </p>
    <p>
     代码示例：
    </p>
    <pre><code>CompletableFuture.runAsync(() -&gt; {
    System.out.println("线程1执行");
}).thenRun(() -&gt; {
    System.out.println("线程2执行");
}).join(); // 等待所有任务完成
</code></pre>
    <p>
     <strong>
      5. 使用 HandlerThread
     </strong>
     <br/>
     <strong>
      HandlerThread
     </strong>
     是 Android 中带有 Looper 的线程，可以按顺序处理任务。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>HandlerThread handlerThread = new HandlerThread("MyHandlerThread");
handlerThread.start();

Handler handler = new Handler(handlerThread.getLooper());

handler.post(() -&gt; {
    System.out.println("线程1执行");
});

handler.post(() -&gt; {
    System.out.println("线程2执行");
});
</code></pre>
    <p>
     <strong>
      6. 使用 BlockingQueue
     </strong>
     <br/>
     <strong>
      BlockingQueue
     </strong>
     是一个线程安全的队列，可以用来实现任务的有序执行。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>BlockingQueue&lt;Runnable&gt; queue = new LinkedBlockingQueue&lt;&gt;();

Thread thread1 = new Thread(() -&gt; {
    System.out.println("线程1执行");
    queue.offer(() -&gt; System.out.println("线程2执行")); // 将任务2放入队列
});

Thread thread2 = new Thread(() -&gt; {
    try {
        Runnable task = queue.take(); // 等待任务2
        task.run();
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
});

thread1.start();
thread2.start();
</code></pre>
    <p>
     <strong>
      7. 使用 Kotlin 协程
     </strong>
     <br/>
     如果使用 Kotlin，协程是最简洁的方式，可以轻松实现线程间的依赖关系。
    </p>
    <p>
     <strong>
      代码示例：
     </strong>
    </p>
    <pre><code>import kotlinx.coroutines.*

fun main() = runBlocking {
    launch {
        println("线程1执行")
    }.join() // 等待线程1执行完毕

    launch {
        println("线程2执行")
    }
}
</code></pre>
    <p>
     以下是几种常见方法的对比：
    </p>
    <p>
     方法 适用场景 优点 缺点
     <br/>
     <strong>
      Thread.join()
     </strong>
     简单场景 代码简单 不适合复杂场景
    </p>
    <p>
     <strong>
      CountDownLatch
     </strong>
     需要明确同步点的场景 灵活性强 需要额外管理计数器
    </p>
    <p>
     <strong>
      单线程池
     </strong>
     任务队列场景 代码简洁 需要手动关闭线程池
    </p>
    <p>
     <strong>
      CompletableFuture
     </strong>
     Java 8+ 的异步编程 现代化，功能强大 需要 Java 8+ 支持
    </p>
    <p>
     <strong>
      HandlerThread
     </strong>
     Android 环境 适合 Android 开发 仅适用于 Android
    </p>
    <p>
     <strong>
      BlockingQueue
     </strong>
     需要任务队列的场景 线程安全，灵活 代码稍复杂
    </p>
    <p>
     <strong>
      Kotlin 协程
     </strong>
     Kotlin 项目 代码简洁，现代化 仅适用于 Kotlin
    </p>
    <p>
     根据具体需求选择最适合的方案。如果只是简单的线程等待，推荐使用 Thread.join() 或 CountDownLatch；如果是 Android 开发，推荐使用 HandlerThread 或 Kotlin 协程。
    </p>
