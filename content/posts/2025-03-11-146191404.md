     <mark>
      面试官：详细说说HashMap
     </mark>
    </h3>
    <hr/>
    <h4>
     <a id="HashMap__5">
     </a>
     HashMap 数据结构详解
    </h4>
    <p>
     HashMap 是 Java 中最常用的键值对存储结构，基于
     <strong>
      哈希表
     </strong>
     实现，核心设计目标是
     <strong>
      高效查找与插入
     </strong>
     。以下是其底层实现原理与关键优化：
    </p>
    <hr/>
    <h5>
     <a id="_11">
     </a>
     一、核心数据结构
    </h5>
    <p>
     在 JDK 1.8 后，HashMap 采用
     <strong>
      数组 + 链表 + 红黑树
     </strong>
     的复合结构：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        数组（桶数组）
       </strong>
      </p>
      <ul>
       <li>
        初始容量为
        <code>
         16
        </code>
        （可自定义），存储链表的头节点或红黑树的根节点。
       </li>
       <li>
        数组索引通过
        <strong>
         哈希值 &amp; (数组长度 - 1)
        </strong>
        计算（等价于取模，但性能更高）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        链表
       </strong>
      </p>
      <ul>
       <li>
        哈希冲突时，键值对以链表形式存储在同一桶中（拉链法）。
       </li>
       <li>
        链表节点为
        <code>
         Node&lt;K,V&gt;
        </code>
        ，包含
        <code>
         key
        </code>
        、
        <code>
         value
        </code>
        、
        <code>
         hash
        </code>
        和
        <code>
         next
        </code>
        指针。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        红黑树
       </strong>
      </p>
      <ul>
       <li>
        当链表长度
        <code>
         ≥8
        </code>
        且数组长度
        <code>
         ≥64
        </code>
        时，链表转换为红黑树（
        <code>
         TreeNode
        </code>
        ），将查询复杂度从
        <code>
         O(n)
        </code>
        优化为
        <code>
         O(log n)
        </code>
        。
       </li>
       <li>
        树节点退化为链表的条件：节点数
        <code>
         ≤6
        </code>
        （避免频繁转换）。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h5>
     <a id="_27">
     </a>
     二、哈希计算与冲突解决
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        哈希值计算
       </strong>
      </p>
      <pre><code class="prism language-java"><span class="token keyword">static</span> <span class="token keyword">final</span> <span class="token keyword">int</span> <span class="token function">hash</span><span class="token punctuation">(</span><span class="token class-name">Object</span> key<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">int</span> h<span class="token punctuation">;</span>
    <span class="token comment">// 高16位异或低16位，增强散列性</span>
    <span class="token keyword">return</span> <span class="token punctuation">(</span>key <span class="token operator">==</span> <span class="token keyword">null</span><span class="token punctuation">)</span> <span class="token operator">?</span> <span class="token number">0</span> <span class="token operator">:</span> <span class="token punctuation">(</span>h <span class="token operator">=</span> key<span class="token punctuation">.</span><span class="token function">hashCode</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span> <span class="token operator">^</span> <span class="token punctuation">(</span>h <span class="token operator">&gt;&gt;&gt;</span> <span class="token number">16</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        哈希冲突处理
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         拉链法
        </strong>
        ：冲突的键值对存入同一桶的链表或树。
       </li>
       <li>
        <strong>
         再哈希法
        </strong>
        ：通过扩容重新分布键值对。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h5>
     <a id="_43">
     </a>
     三、动态扩容机制
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        触发条件
       </strong>
       <br/>
       当元素数量
       <code>
        &gt; 容量 × 负载因子（默认0.75）
       </code>
       时触发扩容。
      </p>
     </li>
     <li>
      <p>
       <strong>
        扩容流程
       </strong>
      </p>
      <ul>
       <li>
        创建新数组（容量翻倍，保持
        <code>
         2^n
        </code>
        特性）。
       </li>
       <li>
        遍历旧数组，重新计算节点位置（利用高位区分新旧索引）。
       </li>
       <li>
        链表节点按
        <code>
         (e.hash &amp; oldCap) == 0
        </code>
        拆分为低位链表和高位链表。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        扩容优化（JDK 1.8）
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         单链表拆分
        </strong>
        ：避免多线程下环形链表问题。
       </li>
       <li>
        <strong>
         红黑树拆分
        </strong>
        ：树节点按哈希值拆分后，若节点数
        <code>
         ≤6
        </code>
        ，退化为链表。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
