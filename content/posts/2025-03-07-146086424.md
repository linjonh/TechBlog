    <h3>
     <a id="1__JUnit_0">
     </a>
     <strong>
      1. 介绍 JUnit
     </strong>
    </h3>
    <p>
     JUnit 是 Java 语言中最流行的单元测试框架之一。它基于
     <strong>
      xUnit
     </strong>
     设计模式，支持
     <strong>
      测试自动化、断言（Assertions）和测试生命周期管理
     </strong>
     ，是 Java 开发中进行
     <strong>
      TDD（测试驱动开发）
     </strong>
     的重要工具。
    </p>
    <p>
     <strong>
      JUnit 主要特点：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       轻量级
      </strong>
      ：不需要复杂的配置即可使用。
     </li>
     <li>
      <strong>
       自动化测试
      </strong>
      ：支持
      <code>
       @Test
      </code>
      注解实现测试自动化。
     </li>
     <li>
      <strong>
       测试报告
      </strong>
      ：与 Maven、Gradle 和 CI/CD 集成，生成测试报告。
     </li>
     <li>
      <strong>
       参数化测试
      </strong>
      ：支持
      <code>
       @ParameterizedTest
      </code>
      进行数据驱动测试。
     </li>
     <li>
      <strong>
       兼容性
      </strong>
      ：JUnit 5 兼容 JUnit 4/3，并提供
      <code>
       Vintage
      </code>
      组件用于向后兼容。
     </li>
    </ul>

    <h3>
     <a id="6__JUnit__192">
     </a>
     <strong>
      6. 解决 JUnit 运行时报错问题
     </strong>
    </h3>
    <h4>
     <a id="_1ClassNotFoundException_orgjunitjupiterapi_194">
     </a>
     <strong>
      问题 1：ClassNotFoundException: org.junit.jupiter.api
     </strong>
    </h4>
    <p>
     <strong>
      原因
     </strong>
     ：JUnit 依赖未正确解析。
     <br/>
     <strong>
      解决方案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       检查
       <code>
        pom.xml
       </code>
      </strong>
      ，确保
      <code>
       junit-jupiter
      </code>
      依赖存在。
     </li>
     <li>
      <strong>
       刷新 Maven 依赖
      </strong>
      （右键
      <code>
       pom.xml
      </code>
      →
      <code>
       Reload Project
      </code>
      ）。
     </li>
    </ul>
    <h4>
     <a id="_2TestEngine_with_ID_junitjupiter_failed_to_discover_tests_201">
     </a>
     <strong>
      问题 2：TestEngine with ID ‘junit-jupiter’ failed to discover tests
     </strong>
    </h4>
    <p>
     <strong>
      原因
     </strong>
     ：缺少
     <code>
      junit-jupiter-engine
     </code>
     。
     <br/>
     <strong>
      解决方案
     </strong>
     ：
    </p>
    <ul>
     <li>
      直接使用
      <code>
       junit-jupiter
      </code>
      <strong>
       聚合依赖
      </strong>
      ，避免单独引入
      <code>
       junit-jupiter-api
      </code>
      。
     </li>
    </ul>
    <h4>
     <a id="_3Maven__mvn_test__207">
     </a>
     <strong>
      问题 3：Maven 运行
      <code>
       mvn test
      </code>
      时报错
     </strong>
    </h4>
    <p>
     <strong>
      解决方案
     </strong>
     ：
    </p>
    <pre><code class="prism language-sh">mvn clean <span class="token builtin class-name">test</span> <span class="token parameter variable">-U</span>  <span class="token comment"># 强制更新依赖</span>
</code></pre>
