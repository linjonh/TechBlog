    <h3>
     <a id="Docker_1">
     </a>
     Docker是什么
    </h3>
    <p>
     Docker 是一个开源的容器化平台，它可以让你在一个独立、轻量的环境中运行应用程序。它主要由以下几个核心概念组成：
    </p>
    <ol>
     <li>
      <strong>
       镜像（Image）
      </strong>
      ：Docker 的应用程序模板，包含代码、运行时、依赖项等。
     </li>
     <li>
      <strong>
       容器（Container）
      </strong>
      ：镜像的运行实例，彼此隔离，但可以共享主机资源。
     </li>
     <li>
      <strong>
       Dockerfile
      </strong>
      ：定义镜像的构建方式，包含一系列指令（如
      <code>
       FROM
      </code>
      、
      <code>
       RUN
      </code>
      、
      <code>
       COPY
      </code>
      ）。
     </li>
     <li>
      <strong>
       Docker Compose
      </strong>
      ：用于管理多容器应用，使用
      <code>
       docker-compose.yml
      </code>
      配置文件。
     </li>
     <li>
      <strong>
       Docker Hub
      </strong>
      ：官方的公共镜像仓库，存储和分发 Docker 镜像。
     </li>
    </ol>
    <h4>
     <a id="_10">
     </a>
     常用命令：
    </h4>
    <ul>
     <li>
      <code>
       docker pull &lt;image&gt;
      </code>
      下载镜像
     </li>
     <li>
      <code>
       docker run -d -p 8080:80 &lt;image&gt;
      </code>
      运行容器并映射端口
     </li>
     <li>
      <code>
       docker ps
      </code>
      查看运行中的容器
     </li>
     <li>
      <code>
       docker stop &lt;container_id&gt;
      </code>
      停止容器
     </li>
     <li>
      <code>
       docker rm &lt;container_id&gt;
      </code>
      删除容器
     </li>
     <li>
      <code>
       docker rmi &lt;image_id&gt;
      </code>
      删除镜像
     </li>
     <li>
      <code>
       docker build -t &lt;name&gt; .
      </code>
      通过 Dockerfile 构建镜像
     </li>
    </ul>
    <h4>
     <a id="Docker_19">
     </a>
     Docker的优点：
    </h4>
    <ol>
     <li>
      <strong>
       轻量级
      </strong>
      ：Docker使用容器技术，相比虚拟机（VM），它共享主机的操作系统内核，从而减少资源占用和启动时间。
     </li>
     <li>
      <strong>
       快速部署和启动
      </strong>
      ：容器化应用可以在几秒钟内启动，而传统的虚拟机可能需要数分钟。
     </li>
     <li>
      <strong>
       跨平台和一致性
      </strong>
      ：Docker容器能够在不同的环境（开发、测试、生产）中保持一致，解决“在我机器上可以运行”的问题。
     </li>
     <li>
      <strong>
       高效的资源利用
      </strong>
      ：Docker允许多个容器共享相同的操作系统内核，消耗更少的CPU和内存资源。
     </li>
     <li>
      <strong>
       易于管理和扩展
      </strong>
      ：Docker提供了灵活的镜像管理和编排工具（如Docker Compose和Kubernetes），方便进行大规模部署和管理。
     </li>
     <li>
      <strong>
       良好的隔离性
      </strong>
      ：容器之间相互独立，确保应用不会相互干扰，提高安全性和稳定性。
     </li>
     <li>
      <strong>
       便于CI/CD
      </strong>
      ：Docker与持续集成/持续部署（CI/CD）流程完美结合，加快开发和部署速度。
     </li>
     <li>
      <strong>
       丰富的生态系统
      </strong>
      ：Docker Hub提供了大量官方和第三方镜像，方便开发者快速获取和使用各种环境和应用。
     </li>
    </ol>
    <h4>
     <a id="Docker_29">
     </a>
     Docker的应用场景：
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        开发和测试
       </strong>
       ：
      </p>
      <ul>
       <li>
        快速创建开发环境，保持与生产环境一致。
       </li>
       <li>
        运行不同版本的软件进行兼容性测试。
       </li>
       <li>
        团队协作时，避免环境不一致问题。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        微服务架构
       </strong>
       ：
      </p>
      <ul>
       <li>
        将应用拆分成多个独立的微服务，每个微服务运行在独立的容器中，提高可扩展性和维护性。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        持续集成与持续部署（CI/CD）
       </strong>
       ：
      </p>
      <ul>
       <li>
        结合Jenkins、GitLab CI等工具，实现自动化测试和部署，提高开发效率。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        云计算和PaaS
       </strong>
       ：
      </p>
      <ul>
       <li>
        许多云平台（如AWS ECS、Google Kubernetes Engine）都支持Docker，方便部署云原生应用。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        大数据和人工智能
       </strong>
       ：
      </p>
      <ul>
       <li>
        运行Hadoop、Spark等大数据框架，或封装AI模型，保证环境一致性，方便迁移和部署。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        混合云和多云部署
       </strong>
       ：
      </p>
      <ul>
       <li>
        由于Docker的可移植性，可以轻松在本地、私有云、公有云之间迁移应用，避免供应商锁定。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        边缘计算和物联网（IoT）
       </strong>
       ：
      </p>
      <ul>
       <li>
        轻量级容器适用于资源有限的边缘设备，如智能网关、工业控制系统等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        传统应用现代化
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过容器化，将老旧的单体应用迁移到现代化的基础设施，提高可维护性和可扩展性。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_56">
     </a>
     安装
    </h3>
    <p>
     在 Windows 上安装 Docker 主要有两种方式，具体取决于你的 Windows 版本：
    </p>
    <ol>
     <li>
      <strong>
       Windows 10/11 专业版、企业版、教育版（支持 Hyper-V）
      </strong>
      → 直接安装
      <strong>
       Docker Desktop
      </strong>
     </li>
     <li>
      <strong>
       Windows 10/11 家庭版（不支持 Hyper-V）
      </strong>
      → 需要启用 WSL 2 并安装
      <strong>
       Docker Desktop
      </strong>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_1Windows_1011__64">
     </a>
     <strong>
      方法 1：Windows 10/11 专业版、企业版、教育版
     </strong>
     （推荐）
    </h3>
    <h4>
     <a id="_1_HyperV__65">
     </a>
     <strong>
      步骤 1：启用 Hyper-V 和容器功能
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       打开 PowerShell（管理员模式）
      </strong>
      ，运行：
      <pre><code class="prism language-powershell">dism<span class="token punctuation">.</span>exe <span class="token operator">/</span>online <span class="token operator">/</span><span class="token function">enable-feature</span> <span class="token operator">/</span>featurename:Microsoft-Hyper-V <span class="token operator">/</span>all <span class="token operator">/</span>norestart
dism<span class="token punctuation">.</span>exe <span class="token operator">/</span>online <span class="token operator">/</span><span class="token function">enable-feature</span> <span class="token operator">/</span>featurename:Containers <span class="token operator">/</span>all <span class="token operator">/</span>norestart
</code></pre>
     </li>
     <li>
      <strong>
       重启电脑
      </strong>
     </li>
    </ol>
    <h4>
     <a id="_2_Docker_Desktop_73">
     </a>
     <strong>
      步骤 2：下载 Docker Desktop
     </strong>
    </h4>
    <ul>
     <li>
      访问
      <a href="https://www.docker.com/products/docker-desktop/" rel="nofollow">
       Docker 官方网站
      </a>
      下载 Docker Desktop for Windows。
     </li>
    </ul>
    <h4>
     <a id="_3_Docker_Desktop_76">
     </a>
     <strong>
      步骤 3：安装 Docker Desktop
     </strong>
    </h4>
    <ol>
     <li>
      运行安装程序，勾选
      <strong>
       “Enable Windows Containers”
      </strong>
      （可选）
     </li>
     <li>
      安装完成后
      <strong>
       重启电脑
      </strong>
     </li>
     <li>
      启动 Docker Desktop，进入
      <strong>
       Settings
      </strong>
      ：
      <ul>
       <li>
        <strong>
         如果使用 Windows 容器
        </strong>
        ：切换到
        <strong>
         Windows Containers
        </strong>
        模式（默认是 Linux）
       </li>
       <li>
        <strong>
         如果使用 Linux 容器（推荐）
        </strong>
        ：确保 WSL 2 后端已启用
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <a id="_4_83">
     </a>
     <strong>
      步骤 4：验证安装
     </strong>
    </h4>
    <p>
     打开
     <strong>
      PowerShell
     </strong>
     或
     <strong>
      命令行
     </strong>
     ，运行：
    </p>
    <pre><code class="prism language-sh"><span class="token function">docker</span> version
<span class="token function">docker</span> run hello-world
</code></pre>
    <p>
     如果输出
     <strong>
      Hello from Docker!
     </strong>
     ，则安装成功！
    </p>
    <hr/>
    <h3>
     <a id="_2Windows_1011__WSL_2_93">
     </a>
     <strong>
      方法 2：Windows 10/11 家庭版（需要 WSL 2）
     </strong>
    </h3>
    <h4>
     <a id="_1_WSL_2_94">
     </a>
     <strong>
      步骤 1：启用 WSL 2
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        打开 PowerShell（管理员模式）
       </strong>
       ，运行：
      </p>
      <pre><code class="prism language-powershell">wsl <span class="token operator">--</span>install
</code></pre>
      <p>
       安装成功如下：记得最好用管理员来运行powershell
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/87942526446d4cd9980470a038b2e78b.png"/>
      </p>
     </li>
     <li>
      <p>
       <strong>
        安装 Ubuntu 发行版
       </strong>
       （建议，可不执行这步）：
      </p>
      <pre><code class="prism language-powershell">wsl <span class="token operator">--</span><span class="token function">set-default</span><span class="token operator">-</span>version 2
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        重启电脑
       </strong>
      </p>
     </li>
    </ol>
    <h4>
     <a id="_2_Docker_Desktop_107">
     </a>
     <strong>
      步骤 2：安装 Docker Desktop
     </strong>
    </h4>
    <ul>
     <li>
      下载并安装
      <a href="https://www.docker.com/products/docker-desktop/" rel="nofollow">
       Docker Desktop
      </a>
     </li>
    </ul>
    <h4>
     <a id="_3_Docker__WSL_2_110">
     </a>
     <strong>
      步骤 3：配置 Docker 使用 WSL 2
     </strong>
    </h4>
    <ol>
     <li>
      打开 Docker Desktop
     </li>
     <li>
      <strong>
       Settings → General
      </strong>
      ，勾选
      <strong>
       Use the WSL 2 based engine
      </strong>
     </li>
     <li>
      <strong>
       Settings → Resources → WSL Integration
      </strong>
      ，启用 Ubuntu
     </li>
    </ol>
    <h4>
     <a id="_4_115">
     </a>
     <strong>
      步骤 4：验证安装
     </strong>
    </h4>
    <p>
     在
     <strong>
      WSL 终端
     </strong>
     或
     <strong>
      PowerShell
     </strong>
     运行：
    </p>
    <pre><code class="prism language-sh"><span class="token function">docker</span> run hello-world
</code></pre>
    <p>
     如果成功打印
     <strong>
      Hello from Docker!
     </strong>
     ，则安装完成！
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f7f7c6dac5564b1e9282af4bcb1ead9d.png"/>
    </p>
    <hr/>
    <h4>
     <a id="_128">
     </a>
     <strong>
      问题排查
     </strong>
    </h4>
    <p>
     如果 Docker 无法启动，可能的解决方法：
    </p>
    <ul>
     <li>
      确保
      <strong>
       BIOS
      </strong>
      中已开启
      <strong>
       虚拟化（VT-x / AMD-V）
      </strong>
     </li>
     <li>
      运行
      <code>
       wsl --update
      </code>
      更新 WSL 2
     </li>
     <li>
      检查 Windows 是否启用了 Hyper-V 或 WSL 2（
      <code>
       systeminfo
      </code>
      查看）
     </li>
    </ul>
    <h3>
     <a id="_135">
     </a>
     常见问题
    </h3>
    <h4>
     <a id="1__136">
     </a>
     1 修改默认安装路径(建议不要修改，可能会出现无法运行)
    </h4>
    <p>
     Docker Desktop 官方安装程序
     <strong>
      不支持直接选择安装路径
     </strong>
     ，它会默认安装到
     <code>
      C:\Program Files\Docker
     </code>
     。但你可以使用以下方法
     <strong>
      修改默认安装路径
     </strong>
     ，将 Docker 安装到指定磁盘（如
     <code>
      D:\Docker
     </code>
     ）。
    </p>
    <hr/>
    <h3>
     <a id="_1_Docker_141">
     </a>
     <strong>
      方法 1：使用符号链接（适用于已安装的 Docker）
     </strong>
    </h3>
    <p>
     如果你已经安装了 Docker，可以通过
     <strong>
      符号链接
     </strong>
     将安装目录移动到其他磁盘。
    </p>
    <h4>
     <a id="_144">
     </a>
     <strong>
      步骤
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       关闭 Docker Desktop
      </strong>
      <pre><code class="prism language-powershell">taskkill <span class="token operator">/</span>F <span class="token operator">/</span>IM <span class="token string">"Docker Desktop.exe"</span>
</code></pre>
     </li>
     <li>
      <strong>
       移动 Docker 安装目录
      </strong>
      <pre><code class="prism language-powershell"><span class="token function">move</span> <span class="token string">"C:\Program Files\Docker"</span> <span class="token string">"D:\Docker"</span>
</code></pre>
     </li>
     <li>
      <strong>
       创建符号链接
      </strong>
      <pre><code class="prism language-powershell">mklink <span class="token operator">/</span>J <span class="token string">"C:\Program Files\Docker"</span> <span class="token string">"D:\Docker"</span>
</code></pre>
     </li>
     <li>
      <strong>
       重新启动 Docker
      </strong>
      <ul>
       <li>
        在开始菜单中搜索
        <strong>
         Docker Desktop
        </strong>
        并打开，确认是否正常运行。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_2_Windows__Docker_162">
     </a>
     <strong>
      方法 2：手动修改 Windows 安装路径（适用于未安装的 Docker）
     </strong>
    </h3>
    <p>
     Docker Desktop 没有安装路径选项，但你可以
     <strong>
      使用
      <code>
       mklink
      </code>
      在安装前创建符号链接
     </strong>
     。
    </p>
    <h4>
     <a id="_165">
     </a>
     <strong>
      步骤
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        卸载 Docker（如果已安装）
       </strong>
      </p>
      <ul>
       <li>
        打开
        <strong>
         “控制面板” → “程序与功能”
        </strong>
       </li>
       <li>
        找到
        <strong>
         Docker Desktop
        </strong>
        ，点击
        <strong>
         “卸载”
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        删除 Docker 残留文件
       </strong>
      </p>
      <ul>
       <li>
        删除
        <code>
         C:\Program Files\Docker
        </code>
       </li>
       <li>
        删除
        <code>
         C:\Users\&lt;YourUser&gt;\AppData\Local\Docker
        </code>
       </li>
       <li>
        删除
        <code>
         C:\Users\&lt;YourUser&gt;\AppData\Roaming\Docker
        </code>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        创建符号链接
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         打开Cmd（管理员模式）
        </strong>
        ，运行：
        <pre><code class="prism language-powershell">mklink <span class="token operator">/</span>J <span class="token string">"C:\Program Files\Docker"</span> <span class="token string">"D:\Docker"</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        重新安装 Docker
       </strong>
      </p>
      <ul>
       <li>
        下载 Docker Desktop 安装程序，并安装，它会被自动重定向到
        <code>
         D:\Docker
        </code>
        。
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="Docker_218">
     </a>
     更改Docker镜像存储位置
    </h4>
    <p>
     在 Windows 上安装 Docker Desktop 时，默认情况下它会安装到
     <code>
      C:\Program Files\Docker
     </code>
     ，但 Docker 容器和镜像的存储路径位于
     <code>
      C:\Users\&lt;YourUser&gt;\AppData\Local\Docker
     </code>
     。如果你的
     <code>
      C:
     </code>
     盘空间不足，你可以手动更改 Docker 数据的存储路径到其他磁盘（如
     <code>
      D:
     </code>
     或
     <code>
      E:
     </code>
     ）。
    </p>
    <hr/>
    <h4>
     <a id="_Docker_Desktop__224">
     </a>
     <strong>
      在 Docker Desktop 设置中更改存储位置（推荐）
     </strong>
    </h4>
    <p>
     Docker Desktop 提供了更改
     <strong>
      容器、镜像等数据存储位置
     </strong>
     的选项，而安装路径仍会保持在
     <code>
      C:
     </code>
     盘。
    </p>
    <ol>
     <li>
      <strong>
       打开 Docker Desktop
      </strong>
     </li>
     <li>
      <strong>
       进入 Settings（设置） → Advanced（高级）
      </strong>
     </li>
     <li>
      找到
      <strong>
       “Disk image location”（磁盘映像位置）
      </strong>
     </li>
     <li>
      点击
      <strong>
       Browse
      </strong>
      （浏览），选择你要存储 Docker 数据的磁盘和路径，例如
      <code>
       D:\DockerData
      </code>
     </li>
     <li>
      <strong>
       点击 “Apply &amp; Restart”（应用并重启）
      </strong>
     </li>
    </ol>
    <p>
     这样，Docker 的容器和镜像数据就会存储在
     <code>
      D:\DockerData
     </code>
     ，而不是
     <code>
      C:
     </code>
     盘。
    </p>
