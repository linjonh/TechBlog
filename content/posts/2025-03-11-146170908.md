    <h2>
     <a id="GitHub__Release__0">
     </a>
     GitHub 项目版本管理与 Release 发布教程
    </h2>
    <p>
     本教程适用于希望在 GitHub 上管理代码版本并发布 Release 的开发者。适用于 Git 基础用户，涵盖从代码提交到发布 Release 的完整流程。
    </p>
    <h3>
     <a id="1__Git__4">
     </a>
     1. 配置 Git 环境
    </h3>
    <h4>
     <a id="11__Git_6">
     </a>
     <strong>
      1.1 安装 Git
     </strong>
    </h4>
    <p>
     如果尚未安装 Git，可以在
     <a href="https://git-scm.com/" rel="nofollow">
      Git 官方网站
     </a>
     下载并安装。
    </p>
    <h4>
     <a id="12__Git__9">
     </a>
     <strong>
      1.2 配置 Git 用户信息
     </strong>
    </h4>
    <p>
     在终端（Windows 用户可以使用 PowerShell 或 CMD）中运行以下命令，设置 Git 用户名和邮箱（GitHub 认证使用）：
    </p>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"你的用户名"</span>
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"你的邮箱"</span>
</code></pre>
    <hr/>
    <h3>
     <a id="2__Git__18">
     </a>
     2. 克隆或初始化 Git 仓库
    </h3>
    <h4>
     <a id="21__20">
     </a>
     <strong>
      2.1 克隆远程仓库（已有项目）
     </strong>
    </h4>
    <p>
     如果项目已存在 GitHub 仓库，可使用以下命令克隆：
    </p>
    <pre><code class="prism language-sh"><span class="token function">git</span> clone https://github.com/你的用户名/你的仓库.git
<span class="token builtin class-name">cd</span> 你的仓库
</code></pre>
    <h4>
     <a id="22__Git__27">
     </a>
     <strong>
      2.2 初始化本地 Git 仓库（新项目）
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">mkdir</span> 你的仓库
<span class="token builtin class-name">cd</span> 你的仓库
<span class="token function">git</span> init
<span class="token function">git</span> remote <span class="token function">add</span> origin https://github.com/你的用户名/你的仓库.git
</code></pre>
    <hr/>
    <h3>
     <a id="3__GitHub_37">
     </a>
     3. 代码提交到 GitHub
    </h3>
    <h4>
     <a id="31__VS_Code__39">
     </a>
     <strong>
      3.1 打开 VS Code 进行代码编辑
     </strong>
    </h4>
    <pre><code class="prism language-sh">code <span class="token builtin class-name">.</span>
</code></pre>
    <h4>
     <a id="32__44">
     </a>
     <strong>
      3.2 添加所有更改到暂存区
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
</code></pre>
    <h4>
     <a id="33__49">
     </a>
     <strong>
      3.3 提交更改并写入提交信息
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"描述本次提交的内容"</span>
</code></pre>
    <blockquote>
     <p>
      <strong>
       注意：
      </strong>
      提交信息应简洁清晰，概括本次修改内容。
     </p>
    </blockquote>
    <h4>
     <a id="34__GitHub__55">
     </a>
     <strong>
      3.4 推送代码到 GitHub 远程仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> push origin main
</code></pre>
    <blockquote>
     <p>
      <strong>
       提示：
      </strong>
      可能需要 GitHub 账户身份验证。
     </p>
    </blockquote>
    <hr/>
    <h3>
     <a id="4__Tag__Release_63">
     </a>
     4. 创建 Tag 并发布 Release
    </h3>
    <h4>
     <a id="41__Tag__65">
     </a>
     <strong>
      4.1 创建 Tag 进行版本管理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> tag <span class="token parameter variable">-a</span> v1.0.0 <span class="token parameter variable">-m</span> <span class="token string">"版本描述"</span>
</code></pre>
    <h4>
     <a id="42__Tag__70">
     </a>
     <strong>
      4.2 推送 Tag 到远程仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> push origin v1.0.0
</code></pre>
    <h4>
     <a id="43__GitHub__Release_75">
     </a>
     <strong>
      4.3 在 GitHub 上创建 Release
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       打开 GitHub 仓库
      </strong>
      ：
      <a href="https://github.com/%E4%BD%A0%E7%9A%84%E7%94%A8%E6%88%B7%E5%90%8D/%E4%BD%A0%E7%9A%84%E4%BB%93%E5%BA%93">
       你的仓库
      </a>
     </li>
     <li>
      进入
      <strong>
       Releases
      </strong>
      页面，点击
      <strong>
       Draft a new release
      </strong>
      。
     </li>
     <li>
      选择
      <strong>
       v1.0.0
      </strong>
      作为 Tag，并填写 Release 说明。
     </li>
     <li>
      点击
      <strong>
       Publish release
      </strong>
      发布。
     </li>
    </ol>
    <blockquote>
     <p>
      <strong>
       可选方式（命令行发布 Release）：
      </strong>
     </p>
    </blockquote>
    <pre><code class="prism language-sh">gh release create v1.0.0 <span class="token parameter variable">--title</span> <span class="token string">"版本 1.0.0"</span> <span class="token parameter variable">--notes</span> <span class="token string">"版本描述"</span>
</code></pre>
    <p>
     （需要安装
     <code>
      GitHub CLI
     </code>
     ，可在
     <a href="https://cli.github.com/" rel="nofollow">
      GitHub CLI 官网
     </a>
     下载）
    </p>
    <hr/>
    <h3>
     <a id="5__89">
     </a>
     5. 总结
    </h3>
    <ul>
     <li>
      <strong>
       Git 基础操作：
      </strong>
      <code>
       add
      </code>
      （添加文件）、
      <code>
       commit
      </code>
      （提交）、
      <code>
       push
      </code>
      （推送）。
     </li>
     <li>
      <strong>
       Tag 作用：
      </strong>
      记录版本历史，方便回滚和管理。
     </li>
     <li>
      <strong>
       GitHub Releases：
      </strong>
      便于版本管理和分发源码。
     </li>
     <li>
      <strong>
       身份验证问题：
      </strong>
      如果遇到身份验证失败，可使用
      <code>
       git credential reject
      </code>
      清除缓存后重新认证。
     </li>
    </ul>
    <p>
     📌
     <strong>
      本教程提供完整 GitHub 版本管理流程，适用于个人开发者及团队协作！
     </strong>
     🚀
    </p>
