    <h4>
     <a id="3__67">
     </a>
     3. 死锁问题及解决办法
    </h4>
    <ul>
     <li>
      <strong>
       死锁产生条件：
      </strong>
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       互斥条件
      </strong>
      ：线程对资源的访问是互斥的。
     </li>
     <li>
      <strong>
       占有并等待条件
      </strong>
      ：线程已经占有了至少一个资源，又在等待其他资源。
     </li>
     <li>
      <strong>
       不可剥夺条件
      </strong>
      ：线程占有的资源不能被其他线程强行剥夺。
     </li>
     <li>
      <strong>
       循环等待条件
      </strong>
      ：多个线程形成一个循环等待资源的链。
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       避免死锁的方法：
      </strong>
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       按相同顺序加锁
      </strong>
      ：所有线程都按照相同的顺序获取锁，避免循环等待。
     </li>
     <li>
      <strong>
       使用 std::lock
      </strong>
      ：可以同时锁定多个互斥量，避免死锁。
     </li>
     <li>
      示例代码
     </li>
    </ol>

     </strong>

     6. 多选题
    </h4>
    <ol>
     <li>
      <p>
       以下哪些情况可能导致数据竞争？（）
       <br/>
       A. 多个线程同时读取共享数据
       <br/>
       B. 多个线程同时写入共享数据
       <br/>
       C. 一个线程读取共享数据，另一个线程写入共享数据
       <br/>
       D. 多个线程对共享数据进行原子操作
      </p>
     </li>
     <li>
      <p>
       避免死锁的方法有（ ）
       <br/>
       A. 按相同顺序加锁
       <br/>
       B. 随机加锁
       <br/>
       C. 使用 std::lock 同时锁定多个互斥量
       <br/>
       D. 减少线程数量
      </p>
     </li>
     <li>
      <p>
       关于锁的粒度，以下说法正确的是（ ）
       <br/>
       A. 粗粒度锁可以提高并发性能
       <br/>
       B. 细粒度锁可以提高并发性能
       <br/>
       C. 粗粒度锁的管理开销较小
       <br/>
       D. 细粒度锁的管理开销较大
      </p>
     </li>
     <li>
      <p>
       std::lock_guard 的特点有（ ）
       <br/>
       A. 手动锁定和解锁互斥量
       <br/>
       B. 自动锁定和解锁互斥量
       <br/>
       C. 基于 RAII 原则
       <br/>
       D. 可以重复锁定和解锁
      </p>
     </li>
     <li>
      <p>
       设计线程安全的数据结构时，需要考虑（ ）
       <br/>
       A. 数据的访问操作
       <br/>
       B. 数据的修改操作
       <br/>
       C. 同步机制的使用
       <br/>
       D. 数据的存储方式
      </p>
     </li>
    </ol>
    <p>
     /*
    </p>
    <ol>
     <li>
      答案：BC
      <br/>
      解释：多个线程同时读取共享数据一般不会导致数据竞争，多个线程对共享数据进行原子操作也不会导致数据竞争。而多个线程同时写入共享数据或一个线程读取一个线程写入共享数据都可能导致数据竞争。
     </li>
     <li>
      答案：AC
      <br/>
      解释：按相同顺序加锁和使用 std::lock 同时锁定多个互斥量可以避免死锁。随机加锁可能会导致死锁，减少线程数量并不能从根本上避免死锁。
     </li>
     <li>
      答案：BCD
      <br/>
      解释：粗粒度锁锁定范围大，会降低并发性能，其管理开销较小。细粒度锁锁定范围小，能提高并发性能，但管理开销较大。
     </li>
     <li>
      答案：BC
      <br/>
      解释：std::lock_guard 基于 RAII 原则，在构造时自动锁定互斥量，在析构时自动解锁，不能手动重复锁定和解锁。
     </li>
     <li>
      答案：ABC
      <br/>
      解释：设计线程安全的数据结构时，需要考虑数据的访问和修改操作，使用合适的同步机制保证线程安全，而数据的存储方式与线程安全关系不大。
      <br/>
      */
     </li>
    </ol>
