    <h3>
     <a id="ArrayList_3">
     </a>
     ArrayList特点
    </h3>
    <ul>
     <li>
      <p>
       ArrayList 可以加入 null，包括多个。
      </p>
     </li>
     <li>
      <p>
       ArrayList 是由数组来实现数据存储的
      </p>
     </li>
     <li>
      <p>
       ArrayList 基本等同于 Vector，除了 ArrayList 是线程不安全（执行效率高）。在多线程情况下，不建议使用 ArrayList
      </p>
      <ul>
       <li>
        例如 ArrayList 的 add() 的源码：可以发现没有 synchronized 关键词修饰
       </li>
      </ul>
      <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">boolean</span> <span class="token function">add</span><span class="token punctuation">(</span><span class="token class-name">E</span> e<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

<span class="token punctuation">}</span>

     </li>
    </ul>
    <h3>
     <a id="ArrayList__21">
     </a>
     ArrayList 源码分析（重难点🚩）
    </h3>
    <ol>
     <li>
      <p>
       ArrayList 中维护了一个 Object 类型的数组 elementData
       <br/>
       <code>
        transient
       </code>
       意为短暂的；表示该属性不会被序列化
      </p>
      <pre><code class="prism language-java"><span class="token keyword">transient</span> <span class="token class-name">Object</span><span class="token punctuation">[</span><span class="token punctuation">]</span> elementData<span class="token punctuation">;</span>
</code></pre>
     </li>
     <li>
      <p>
       当创建对象时，如果使用的是无参构造器，则初始 elementData 容量为 0 (jdk7是10)
      </p>
     </li>
     <li>
      <p>
       当添加元素时：先判断是否需要扩容，如果需要扩容，则调用 grow 方法，否则直接添加元素到合适位置
      </p>
     </li>
     <li>
      <p>
       如果使用的是无参构造器，如果第一次添加，需要扩容的话，则扩容 elementData 为 10，如果需要再次扩容的话，则扩容 elementData 为 1.5 倍。
      </p>
     </li>
     <li>
      <p>
       如果使用的是指定容量大小的构造器，则初始 elementData 容量为指定大小
      </p>
     </li>
     <li>
      <p>
       如果使用的是指定容量大小的构造器，如果需要扩容，则直接扩容 elementData 为 1.5 倍。
       <br/>
       [[ArrayList源码分析]]
      </p>
     </li>
    </ol>

    <blockquote>
     <p>
      创建 ArrayList 实例
     </p>
    </blockquote>
    <ul>
     <li>
      创建一个长度为 0 的空数组，并赋给 elementData，elementData 就是集合底层存储数据的数组。
      <br/>
      <img alt="image.png|650" src="https://i-blog.csdnimg.cn/img_convert/375bf135a0bd3bbbba1ab402375c01e5.png"/>
     </li>
    </ul>
    <blockquote>
     <p>
      第一次添加元素“11”
     </p>
    </blockquote>
    <ul>
     <li>
      将 int 数据进行装箱
     </li>
     <li>
      添加这个数据“11”，要保证数组的长度至少为当前元素数+1。
      <mark>
       （size+1=1）
      </mark>
     </li>
     <li>
      <code>
       ensureCapacityInternal()
      </code>
      的作用就是检查数组够不够此次添加所需的容量，如果不够，则扩容，更改 elementData ，将其用扩容后的数组替代。
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/114a90c89d6b1034cdf6046bf2e006ea.png"/>
     </li>
     <li>
      <code>
       calculateCapacity()
      </code>
      的作用仅仅在于，检查当前操作是否是第一次添加数据。因为第一次添加元素，扩容机制应该将其扩容到 10（除非要添加的元素本身需要的容量大于 10）。
     </li>
     <li>
      而对于非第一次添加的元素，其所需容量依旧是传进来的参数
      <br/>
      <img alt="image.png|650" src="https://i-blog.csdnimg.cn/img_convert/080fa0adaf82c6698fa74987539343fa.png">
       <br/>
       <img alt="image.png|700" src="https://i-blog.csdnimg.cn/img_convert/7a517def90de8ef15f15026a720ddd39.png"/>
      </img>
     </li>
     <li>
      那么根据判断，此次显然是第一次添加元素，数组为空数组，且所需容量 var1=1，&lt;10。那么此时
      <mark>
       var0=10
      </mark>
      ，这就确定了最终数组所需要的容量大小。作为参数传递给
      <code>
       ensureExplicitCapacity()
      </code>
      中。
      <br/>
      <img alt="image.png|650" src="https://i-blog.csdnimg.cn/img_convert/165c39c374995b7c76fd20336c76f11d.png"/>
     </li>
     <li>
      modCount 为当前集合被修改的次数。需要自增。
     </li>
     <li>
      最后进行检查：目前数组的容量（
      <code>
       elementData.length
      </code>
      ）是否满足所需要的容量大小（
      <code>
       var1
      </code>
      ）呢？
     </li>
     <li>
      显然，目前的数组大小为 0，而我们所需要的容量为 10。所以需要扩容。进入到
      <code>
       grow()
      </code>
      中，进行真正的扩容操作。
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/99aa758687fbb07a50146d5b336d7027.png"/>
     </li>
     <li>
      所需的容量大小为 var1=10，作为参数传递进来。
     </li>
     <li>
      var2=数组目前的容量==（var2=0）==
     </li>
     <li>
      var3=扩容为当前数组容量的 1.5 倍，不过在初次扩容中，此次操作不起作用==（var3=0）==
     </li>
     <li>
      判断扩容后的容量大小 var3 是否满足，所需的大小 var1，不满足则依照 var1。
      <mark>
       （var3=10）
      </mark>
     </li>
     <li>
      判断扩容后的容量是否在数组的取值范围内。
     </li>
     <li>
      <code>
       Arrays.copyOf()
      </code>
      ：创建一个指定长度的数组==（10）==的数组，并将原数组中的数据拷贝至该数组，并赋值给
      <code>
       elementData
      </code>
      。
     </li>
     <li>
      至此，数组的扩容完成（elementData.length 从 0 扩容到 10）回到 add 的主逻辑中：
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/114a90c89d6b1034cdf6046bf2e006ea.png"/>
     </li>
     <li>
      添加元素“11”添加到数组 elementData[0]中，并且 size=1；
     </li>
    </ul>
    <blockquote>
     <p>
      第二次添加元素“12”（不用扩容）
     </p>
    </blockquote>
    <ul>
     <li>
      将 int 数据进行装箱
     </li>
     <li>
      添加这个数据“12”，要保证数组的长度至少为当前元素数+1。
      <mark>
       （size+1=2）
      </mark>
     </li>
     <li>
      <code>
       ensureCapacityInternal()
      </code>
      的作用就是检查数组是否大于等于 2（够不够此次添加所需的容量）。如果不够，则扩容，更改 elementData ，将其用扩容后的数组替代。
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/d2e55f5f285ad12d363ac782656a7a4a.png"/>
     </li>
     <li>
      此时由于不是第一次添加，数组长度不为 0，所以
      <code>
       calculateCapacity()
      </code>
      执行后返回
      <mark>
       2
      </mark>
      。
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/2f3b51a461a5fddb738860b232d4cf66.png"/>
      <br/>
      <img alt="image.png|700" src="https://i-blog.csdnimg.cn/img_convert/7a517def90de8ef15f15026a720ddd39.png"/>
     </li>
     <li>
      集合修改次数+1
     </li>
     <li>
      当前数组长度==（elementData.length=10）
      <mark>
       满足所需长度
      </mark>
      (var1=2)==，所以不需要扩容。
      <br/>
      <img alt="image.png|550" src="https://i-blog.csdnimg.cn/img_convert/190cbd701453a98127e73b6fede707cc.png"/>
     </li>
     <li>
      回到主逻辑中：
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/d2e55f5f285ad12d363ac782656a7a4a.png"/>
     </li>
     <li>
      elementData[1]存放 var1。size 变为 2。
     </li>
    </ul>
    <blockquote>
     <p>
      观察第十一次添加元素（第二次扩容）
     </p>
    </blockquote>
    <ul>
     <li>
      此时数组容量为
      <mark>
       elementData.length=10
      </mark>
      ，所需容量大小为
      <mark>
       var1=11
      </mark>
      ，在这一步时，容量不够，所以需要进行扩容，进入
      <code>
       grow()
      </code>
      。
      <br/>
      <img alt="image.png|650" src="https://i-blog.csdnimg.cn/img_convert/165c39c374995b7c76fd20336c76f11d.png"/>
     </li>
     <li>
      var2=10，var3=15（扩容 1.5 倍）
      <br/>
      <img alt="image.png|600" src="https://i-blog.csdnimg.cn/img_convert/99aa758687fbb07a50146d5b336d7027.png"/>
     </li>
    </ul>
    <blockquote>
     <p>
      同理，后面每一次添加和扩容都与以上的步骤一致。
     </p>
    </blockquote>
    <h4>
     <a id="_133">
     </a>
     指定初始容量
    </h4>
    <ul>
     <li>
      与未指定的区别在于，调用的是带参的构造器。
     </li>
     <li>
      指定了初始容量 var1，则创建一个初始容量为 var1 的数组。如果本身为 0，就仍旧赋值为空数组（与无参的一致）
     </li>
     <li>
      在之后的扩容中，依旧是按照 1.5 倍扩容。
     </li>
     <li>
      例如指定初始容量为 8，则初次扩容为 8-&gt;12
      <br/>
      <img alt="image.png" src="https://i-blog.csdnimg.cn/img_convert/6721fa71f64c5c1681dbc17b890ca42a.png"/>
     </li>
    </ul>
