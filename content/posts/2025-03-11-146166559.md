    <h3>
     <a id="_0">
     </a>
     一.再谈构造函数
    </h3>
    <ul>
     <li>
      构造函数有
      <strong>
       构造函数体赋值实
      </strong>
      现和
      <strong>
       初始化列表
      </strong>
      两种方式
     </li>
    </ul>
    <h3>
     <a id="1_4">
     </a>
     1.构造函数体赋值
    </h3>
    <blockquote>
     <p>
      在创建对象时，编译器通过调用构造函数，给对象中各个成员变量一个合适的初始值.
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/23cb494d13f24f40ba7ad81fe2db9484.png">
       <br/>
       虽然上述构造函数调用之后，对象中已经有了一个初始值，但是不能将其称为对对象中成员变量的初始化，
       <strong>
        构造函数体中的语句只能将其称为赋初值
       </strong>
       ，而不能称作初始化。
       <strong>
        因为初始化只能初始化一次，而构造函数体内可以多次赋值。
       </strong>
      </img>
     </p>
    </blockquote>
    <h3>
     <a id="2_8">
     </a>
     2.初始化列表
    </h3>
    <p>
     <strong>
      是对象的成员定义的位置
     </strong>
     <br/>
     （1）语法结构
    </p>
    <blockquote>
     <p>
      以一个冒号开始，接着是一个以逗号分隔的数据成员列表，每个"成员变量"后面跟一个放在括号{}中的初始值或表达式(构造函数体)。
     </p>
    </blockquote>
    <pre><code class="prism language-cpp"><span class="token keyword">class</span> <span class="token class-name">Myclass</span>
<span class="token punctuation">{<!-- --></span>
<span class="token keyword">public</span><span class="token operator">:</span>
	<span class="token function">Myclass</span><span class="token punctuation">(</span><span class="token keyword">int</span> x<span class="token punctuation">,</span> <span class="token keyword">int</span> y<span class="token punctuation">)</span>
		<span class="token operator">:</span><span class="token function">member1</span><span class="token punctuation">(</span>x<span class="token punctuation">)</span>
		<span class="token punctuation">,</span> <span class="token function">member2</span><span class="token punctuation">(</span>y<span class="token punctuation">)</span>
	<span class="token punctuation">{<!-- --></span>
		<span class="token comment">//构造函数体</span>
	<span class="token punctuation">}</span>

<span class="token keyword">private</span><span class="token operator">:</span>
	<span class="token keyword">int</span> member1<span class="token punctuation">;</span>
	<span class="token keyword">int</span> member2<span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     （2）特性
     <br/>
     1.
     <strong>
      初始化列表只能初始化一次
     </strong>
     ，多次会报错
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/571fd794288f48968c499774afe50c8c.png"/>
     </p>
    </blockquote>
    <p>
     2.
     <strong>
      编译器允许构造函数体赋值与初始化列表混用
     </strong>
     。初始化列表优先，初始化列表中成员变量初始化操作在构造函数体之前完成。
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e186d40f759e4b26a5fcecde4cec0dbe.png"/>
     </p>
    </blockquote>
    <p>
     3.若在初始化列表和构造函数体中
     <strong>
      对同一个成员变量进行赋值，构造函数体内的赋值会覆盖初始化列表中的值
     </strong>
     。
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8acea30adf044255a897dab4704447b0.png"/>
     </p>
    </blockquote>
    <p>
     4.
     <strong>
      引用成员变量
     </strong>
     ，
     <strong>
      const成员变量
     </strong>
     ，
     <strong>
      自定义类型成员
     </strong>
     (且该类没有默认构造函数时)必须放在初始化列表位置进行初始化。
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/98fcde3cc1cf41a9a5d3ccbe235e888c.png"/>
     </p>
    </blockquote>
    <blockquote>
     <p>
      这三种类型成员若在构造函数体内进行赋值初始化会报错
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1e842079441c43b3a9fea60074614719.png">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cd9ac6a6e3bb49ddb984793841cff9cb.png"/>
      </img>
     </p>
    </blockquote>
    <blockquote>
     <p>
      const成员变量、引用成员变量、没有默认构造函数的自定义类型成员变量
      <strong>
       必须在初始化列表内初始化的原因
      </strong>
      ：
      <br/>
      1**.初始化列表是对象的成员定义的位置**
      <br/>
      2.
      <strong>
       const成员变量
      </strong>
      ：一旦被赋值就不能被修改，必须在初始化列表中赋值，因为它们不能在构造函数体中被修改。
      <br/>
      3.
      <strong>
       引用成员变量
      </strong>
      ：必须在初始化列表中绑定，因为引用在声明时就必须绑定到一个有效的对象或变量上，不能重新绑定。
      <br/>
      4.
      <strong>
       没有默认构造函数的自定义类型成员变量
      </strong>
      ：必须通过初始化列表显示调用构造函数进行初始化，因为编译器无法自动完成。
     </p>
    </blockquote>
    <p>
     5.尽量使用初始化列表初始化，因为不管你是否使用初始化列表，对于自定义类型成员变量，一定会先使用初始化列表初始化。
    </p>
    <blockquote>
     <p>
      具有以下优点：
      <br/>
      <strong>
       1.效率提升
      </strong>
      <br/>
      <strong>
       避免默认构造
      </strong>
      ：对于自定义类型若不使用初始化列表，默认情况下编译器会先调用默认构造函数创建对象，然后再进行赋值操作，使用初始化列表可以直接进行构造，避免了不必要的默认构造和复制操作。
      <br/>
      <strong>
       减少复制操作
      </strong>
      ：对于大型对象或容器，减少复制次数可以显著提高程序的允许效率。
      <br/>
      <strong>
       2.必要性
      </strong>
      <br/>
      在上述内容中已写明对那三种成员变量只能在初始化列表中进行初始化。
      <br/>
      <strong>
       3.代码简洁性
      </strong>
      <br/>
      当需要初始化多个成员变量时，使用初始化列表可增加可读性。而构造函数体内可能设计复杂的逻辑判断等条件。
     </p>
    </blockquote>
    <p>
     6.成员变量在类中声明次序就是其在初始化列表中的初始化顺序，与其在初始化列表中的先后次序无关
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/373019232526497082272b853a3ca237.png"/>
      <br/>
      为什么有随机数结果呢？
      <br/>
      因为我们认为在初始化列表中先对a1进行初始化，再用a1的值对a2初始化，但实际上初始化的顺序是按照成员变量在类中的声明次序来的。所以先对a2初始化，这时a1未知所以结果是随机值，然后再对a1初始化为1.
      <br/>
      <strong>
       所以建议初始化列表中变量声明顺序与在类中的声明顺序保持一致
      </strong>
     </p>
    </blockquote>

     1.隐式类型转换
    </h3>
    <p>
     （1）内置类型的隐式类型转换
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0594b095486f4aa88dbcaa2db33a9c89.png"/>
      <br/>

      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a82d543b761444b7b68741bcc6a8204a.png"/>

    <h3>
     <a id="static_155">
     </a>
     三.static成员
    </h3>
    <p>
     概念：声明为
     <strong>
      static的类成员
     </strong>
     称为
     <strong>
      类的静态成员
     </strong>
     ，用
     <strong>
      static
     </strong>
     修饰的
     <strong>
      成员变量
     </strong>
     ，称之为
     <strong>
      静态成员变量
     </strong>
     ；用
     <strong>
      static修饰的成员函数
     </strong>
     ，称之为
     <strong>
      静态成员函数
     </strong>
     。
     <strong>
      静态成员变量一定要在类外进行初始化
     </strong>
    </p>
    <blockquote>
     <p>
      实现一个类，计算程序中创建出了多少个类对象
     </p>
    </blockquote>
    <pre><code class="prism language-cpp"><span class="token keyword">class</span> <span class="token class-name">A</span>
<span class="token punctuation">{<!-- --></span>
<span class="token keyword">public</span><span class="token operator">:</span>

     <p>
      该类通过记录调用构造函数和拷贝构造函数次数来计算创建了多少个类对象，通过记录调用析构函数来判断销毁了多少个类对象
      <br/>
      该类的实现必须使用静态成员变量和静态成员函数原因在于：
      <br/>
      <strong>
       成员变量
      </strong>
      ：属于每一个类对象自身，存储在对象里面
      <br/>
      <strong>
       静态成员变量
      </strong>

     </p>

      </p>
     </blockquote>
    </blockquote>

     <p>
      特性：
     </p>

      <li>
       <strong>

       </strong>

       <strong>
        相当于用类去封装全局变量
       </strong>
       。
      </li>
      <li>
       <strong>
        静态成员变量必须在类外定义
       </strong>
       ，
       <strong>
        定义时不添加static关键字
       </strong>
       (定义的目的是为其分配实际的内存空间，而不是再次声明其静态属性)，类中只是声明
      </li>
      <li>
       类静态成员即可用 类名::静态成员 或者 对象.静态成员 来访问，即
       <strong>
        突破类域和访问限定符就可以访问
       </strong>
       。
      </li>
      <li>
       静态成员函数
       <strong>
        没有隐藏的this指针，不能访问任何非静态成员
       </strong>
       ，非静态成员可以访问静态成员
      </li>
      <li>
       静态成员也是类的成员，
       <strong>
        受public、protected、private 访问限定符的限制
       </strong>
       <br/>
       6.
       <strong>
        静态成员不能给缺省值，没有初始化列表
       </strong>
       ，缺省值是给初始化列表的。
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="_201">
     </a>
     四：友元
    </h3>
    <blockquote>
     <p>
      概念：友元提供了一种突破封装的方式，有时提供了便利。但是友元会增加耦合度(模块间的关联程度，越低越独立)，破坏了封装，所以友元不宜多用。友元分为：友元函数和友元类
     </p>
    </blockquote>
    <h3>
     <a id="1_203">

     </p>

    <h3>
     <a id="2_251">
     </a>
     2.友元类
    </h3>
    <p>
     友元类的所有成员函数都可以是另一个类的友元函数，都可以访问另一个类中的非公有成员。
     <br/>
     1.
     <strong>
      友元关系是单向的，不具有交换性
     </strong>
     。
     <br/>
     假如有Time类和Date类，在Time类中声明Date类为其友元类，那么可以在Date类中直接访问Time类的私有成员变量，但想在Time类中访问Date类中私有的成员变量则不行。
     <br/>
     2.
     <strong>
      友元关系不能传递
     </strong>
     <br/>
     如果B是A的友元，C是B的友元，则不能说明C时A的友元。
     <br/>
     3.友
     <strong>
      元关系不能继承
     </strong>
     ，在继承位置再给大家详细介绍。
    </p>
    <h3>
     <a id="_258">
     </a>
     五.内部类
    </h3>
    <p>
     概念：
     <strong>
      如果一个类定义在另一个类的内部，这个内部类就叫做内部类
     </strong>
     。
     <strong>
      内部类是一个独立的类，它不属于外部类
     </strong>
     ，更不能通过外部类的对象去访问内部类的成员。
     <strong>
      外部类对内部类没有任何优越的访问权限
     </strong>
     。
    </p>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a0029193e09c42109b8820a4598a953a.png"/>
      <br/>
      比如上述一道oj题中，在Solution类中声明Sum类为其内部类，那么可以在Sum类中直接访问Solution类的私有成员变量，但想在Solution类中访问Sum类中私有的成员变量则不行。
      <br/>
      注意：
      <strong>
       内部类就是外部类的天生友元类
      </strong>
      ，参见友元类的定义，内部类可以通过外部类的对象参数来访问外部类中的所有成员。但是外部类不是内部类的友元。
      <br/>
      特性：
     </p>
     <ol>
      <li>
       内部类可以定义在外部类的public、protected、private都是可以的。
      </li>
      <li>
       注意内部类可以直接访问外部类中的static成员，不需要外部类的对象/类名。
      </li>
      <li>
       sizeof(外部类)=外部类，和内部类没有任何关系。
      </li>
     </ol>
    </blockquote>
