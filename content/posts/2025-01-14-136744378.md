    <h4 id="ahkvl">
     前言
    </h4>
    <p>
     前面我们玩到了@ Builder 装饰器我们知道他是一个自定义构建函数,可以将重复使用的UI元素抽象成一个方法，在build方法里调用,那么本期我们要玩玩的是BuilderParam 顾名思义: builder 的参数,用作是啥? 这玩意有点麻烦说实话不好用......
    </p>
    <p>
     用于自定义构建函数的时候在里面添加了一个自定义组件如果你需要实现点击跳转功能按早以前我们都是直接修改这个函数里面的组件那么导致所有引用这个函数的组件都会有点击跳转功能,为解决此问题，ArkUI引入了@BuilderParam装饰器，@BuilderParam用来装饰指向@Builder方法的变量，我们就可以在初始化自定义组件时对此属性进行赋值，为自定义组件增加特定的功能。
    </p>
    <p>
     该装饰器用于声明任意UI描述的一个元素，类似slot占位符。
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/b7c5bbc2f0d8513d453904d6fd7d3a29.png"/>
    </p>
    <h4 id="biuv0">
     使用
    </h4>
    <p>
     ⚠️: @BuilderParam装饰的方法只能被自定义构建函数@Builder装饰来初始化
    </p>
    <p>
     使用所属自定义组件的自定义构建函数或者全局的自定义构建函数，在本地初始化@BuilderParam,啥意思?操作看看
    </p>
    <p>
     语法:
    </p>
    <p>
     所属的自定义组件里面的函数
    </p>
    <p>
     官方给到的一个语法
    </p>
    <p>
     这里 void 后面跟着类似赋值操作 官方没有给出合理的解释, 我们继续跟着案例来
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="img" height="790" src="https://i-blog.csdnimg.cn/blog_migrate/fc0ceb5b3c5308a03b0c8e0460550bdf.png" width="1200"/>
    </p>
    <p>
    </p>
    <pre><code>@Builder function GlobalBuilder0() {}
 
 @Component
 struct Child {
   @Builder doNothingBuilder() {};
   // 这里 void 后面跟着类似赋值操作 官方没有给出合理的解释
   @BuilderParam aBuilder0: () =&gt; void = this.doNothingBuilder;
   @BuilderParam aBuilder1: () =&gt; void = GlobalBuilder0;
   build(){}
 }</code></pre>
    <p>
     用父组件自定义构建函数初始化子组件@BuilderParam装饰的方法
    </p>
    <p>
     使用子组件的 BuildeParam 传递参数的时候不能传递字符串、数值等类型只能传递 Builder 修饰的函数
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="img" height="1054" src="https://i-blog.csdnimg.cn/blog_migrate/5ea159a1b7f0c50efceff381fed9c7c9.png" width="1200"/>
    </p>
    <p>
    </p>
    <pre><code>// 乖儿
 @Component
 struct Son {
   @BuilderParam bp0: () =&gt; void;
 
   build() {
     Column() {
       // 直接调用使用
       this.bp0()
    }
  }
 }
 
 // 父亲组件
 @Entry
 @Component
 struct Father {
   @Builder componentBuilder() {
     Text(`我这里是父类里面的组件`)
  }
 
   build() {
     Column() {
       Son({bp0: this.componentBuilder})
    }
  }
 }</code></pre>

    <h4 id="ulsq">
     this指向
    </h4>
    <p>
     这个时候虽然是可以用到父组件的text组件
    </p>
    <p>
     但是子组件里面的this是指向 son的所以他默认就是子组件的label的了
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="img" height="1200" src="https://i-blog.csdnimg.cn/blog_migrate/ee40f5b651714514048bd7d172f90f2e.png" width="1200"/>
    </p>
    <p>
     如果你把子组件的label删除你就会发现变成undefined?????为什么
    </p>
    <p>
     因为上面说过了子组件里面的this指向的是子组件,所以就拿不到,这里和vue就不一样了麻烦有一点点难理解
    </p>
    <p>
    </p>
    <p class="img-center">

    <h5 id="iftk">
     参数初始化组件
    </h5>
    <p>
     @BuilderParam装饰的方法可以是有参数和无参数的两种形式但是@BuilderParam装饰的方法类型需要和@Builder方法类型一致
    </p>
    <p>
     需求:
    </p>
    <ol>
     <li>
      定义一个全局的自定义构建函数接受引用参数传递 label 为字符串 ]
     </li>
     <li>
      定义一个子组件在里编写一个无参数和一个有参数()类型
     </li>
     <li>
      定义一个父组件为入口、里面使用子组件并且传递无参和有参数据
     </li>
    </ol>
    <pre><code>// 定义一个全局的自定义构建函数接受引用参数传递 label 为字符串
@Builder function GlobalBuilderTextByLabel($$ : {label: string }) {
  Text($$.label)
    .width("100%")
    .height(50)
    .backgroundColor(Color.Pink)
}

@Component
struct Child {
  label: string = '子组件'
  // 无参数类，指向的componentBuilder也是无参数类型
  @BuilderParam aBuilder0: () =&gt; void;
  // 有参数类型，指向的GlobalBuilderTextByLabel也是有参数类型的方法
  @BuilderParam aBuilder1: ($$ : { label : string}) =&gt; void;

  build() {
    Column() {
      // 子组件的label this 引用
      this.aBuilder0()
      this.aBuilder1({label: `我在子组件里面调用`} )
    }
  }
}

@Entry
@Component
struct Parent {
  label: string = '我是父亲元素'

  @Builder componentBuilder() {
    Text(`${this.label}`).fontSize(30)
  }

  build() {
    Column() {
      /**
       * 参数一: 无参数传递直接使用 @Builder 当中的组件
       * 参数二: 引用全局自定义函数
       */
      Child({ aBuilder0: this.componentBuilder, aBuilder1: GlobalBuilderTextByLabel })
    }
  }
}</code></pre>

     自定义组件“Child”中的属性“aBuilder0”缺少赋值或初始化。

    <h5>
     尾随闭包初始化组件
    </h5>
    <p>
     在初始化自定义组件时，组件后紧跟一个大括号“{}”形成尾随闭包场景。
    </p>
    <p>
     ⚠️: 此场景下自定义组件内有且仅有一个使用@BuilderParam装饰的属性才能用
    </p>
    <p>
     创建SonCustomContainer，在创建SonCustomContainer时，通过其后紧跟一个大括号“{}”形成尾随闭包
    </p>
    <p>
     作为传递给子组件SonCustomContainer @BuilderParam closer: () =&gt; void的参数
    </p>
    <pre><code>@Component
struct SonCustomContainer {
    @Prop header: string;
    @BuilderParam closer: () =&gt; void

    build() {
        Column() {
            Text(this.header)
            .fontSize(30)

            // 这里是父组件 传递过来的
            this.closer()
        }
    }
}

// 全局自定义
@Builder function GlobalSpecificParam(label1: string, label2: string) {
    Column() {
        Text(label1)
        .fontSize(30)
        Text(label2)
        .fontSize(30)
    }
}

@Entry
@Component
struct CustomContainerUser {
    @State text: string = 'header';

    build() {
        Column() {
            // 创建SonCustomContainer，在创建SonCustomContainer时，通过其后紧跟一个大括号“{}”形成尾随闭包
            // 作为传递给子组件SonCustomContainer @BuilderParam closer: () =&gt; void的参数
            SonCustomContainer({ header: this.text }) {
                Column() {
                    // 特殊参数
                    GlobalSpecificParam('参数1111', '参数2222')
                }.backgroundColor(Color.Yellow)
                .onClick(() =&gt; {
                    this.text = '点击了';
                })
            }
        }
    }
}</code></pre>
