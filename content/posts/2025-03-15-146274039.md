     <p>
      A.数据元素        B.数据对象        C.数据关系        D.抽象数据类型

    <blockquote>
     <p>
      2.以下属于逻辑结构的是（ ）

    <h2 style="text-align:center">
     <span style="color:#1c7331">
      一、数据、数据元素、数据对象、数据项、数据结构、数据类型
     </span>
    </h2>
    <h3>
     1.1 概念（P1）
    </h3>
    <blockquote>
     <ul>
      <li>
       <span style="color:#1a439c">
        <strong>
         <span style="background-color:#cbe0f1">
          数据结构
         </span>
        </strong>
        <span style="background-color:#cbe0f1">
         是相互之间存在一种或多种特定关系的数据元素的集合；它包括三方面的内容：逻辑结构，存储结构，数据的运算。
        </span>
       </span>
      </li>
      <li>
       <span style="color:#1a439c">
        <strong>
         <span style="background-color:#cbe0f1">
          数据对象
         </span>
        </strong>
        <span style="background-color:#cbe0f1">
         是具有相同性质的数据元素的集合，是数据的一个子集；
        </span>
       </span>
      </li>
      <li>
       <span style="color:#1a439c">
        <strong>
         <span style="background-color:#cbe0f1">
          数据
         </span>
        </strong>
        <span style="background-color:#cbe0f1">
         是信息的载体，是描述客观事物属性的数、字符及所有能输入到计算机中并被计算机程序识别和处理的符号的集合；
        </span>
       </span>
      </li>
      <li>
       <span style="color:#1a439c">
        <strong>
         <span style="background-color:#cbe0f1">
          数据元素
         </span>
        </strong>
        <span style="background-color:#cbe0f1">
         是数据的基本单位，通常作为一个整体进行考虑和处理；
        </span>
       </span>
      </li>
      <li>
       <span style="color:#1a439c">
        <span style="background-color:#cbe0f1">
         一个数据元素有若干
        </span>
        <strong>
         <span style="background-color:#cbe0f1">
          数据项
         </span>
        </strong>
        <span style="background-color:#cbe0f1">
         组成，数据项是构成数据元素的不可分割的最小单位。
        </span>
       </span>
      </li>
      <li>
       数据类型：原子类型、结构类型、
       <strong>
        <span style="background-color:#fefcd8">
         抽象数据类型（ADT）
        </span>
       </strong>
       <span style="background-color:#fefcd8">
        ：描述了数据的逻辑结构和抽象运算，通常用（数据对象，数据关系，基本操作）这样的三元组来表示，eg.栈、队列，树...（ADT和数据密切相关，但不完全相同）
       </span>
      </li>
     </ul>
    </blockquote>
    <h3>
     1.2  我的理解
    </h3>
    <ol>
     <li>
      数据项是最小的，数据是最大的；
     </li>
     <li>
      数据项构成数据元素；
     </li>
     <li>
      数据元素构成数据对象；
     </li>
     <li>
      所有数据对象的总和是数据。
     </li>
     <li>
      <hr/>
      <strong>
       数据结构
      </strong>
      是相互之间存在一种或多种特定关系的数据元素的集合。(数据元素+关系)
     </li>
     <li>
      <strong>
       数据对象
      </strong>
      是具有相同性质的数据元素的集合，是数据的一个子集。(数据元素)
     </li>
    </ol>
    <blockquote>
     <ul>
      <li>
       <span style="color:#98c091">
        世界上所有的信息、符号的总和是数据。
       </span>
      </li>
      <li>
       <span style="color:#98c091">
        这些数据有所有大学学生数据（数据对象），所有餐厅顾客数据（数据对象），XX大学学生数据（数据对象），所有动物的XX数据（数据对象）...
       </span>
      </li>
      <li>
       <span style="color:#98c091">
        XX大学同学ABCD...数据（数据元素），...
       </span>
      </li>
      <li>
       <span style="color:#98c091">
        班级学生有自己的学号（数据项），姓名（数据项），年龄（数据项）...
       </span>
      </li>
      <li>
       <span style="color:#98c091">
        数据结构是带存储关系的同学ABCD...的数据（通过它你不仅知道这些数据，还知道它们之间的关系是怎样的，如何存储的）
       </span>
      </li>
      <li>
       <span style="color:#98c091">
        数据对象，数据元素，数据项根据研究内容的不同都是相对的，这项研究中的数据元素可能是下一项研究中的数据对象。
       </span>
      </li>
     </ul>
    </blockquote>

     <span style="color:#1c7331">
      二、数据结构的三要素
     </span>
    </h2>
    <h3>
     2.1 数据结构三要素
    </h3>
    <p>
     <span style="color:#b95514">
      <strong>
       <span style="background-color:#ffd7b9">
        逻辑结构，存储结构，运算
       </span>
       <span style="background-color:null">
       </span>
      </strong>
     </span>
     <span style="color:#fe2c24">
      <span style="background-color:#fbd4d0">
       知存储就知逻辑，知逻辑不一定知存储！
      </span>
     </span>
    </p>
    <h4>
     2.2.1 逻辑结构
    </h4>
    <p>
     四类基本结构：线性结构、集合(同属一个集合无其他关系)、树形、图状结构(网状结构)。
    </p>
    <p>
     <img alt="" height="924" src="https://i-blog.csdnimg.cn/direct/eb1d93c3bc954b4e8ed902468f18c31d.jpeg" width="3108"/>
    </p>
    <h4>
     2.2.2 存储结构
    </h4>
    <p>
     ①
     <strong>
      顺序存储
     </strong>
    </p>
    <p>
     ②
     <strong>
      链式存储
     </strong>
    </p>
    <p>
     ③
     <strong>
      索引存储
     </strong>
     （索引表中每项是索引项（关键字，地址）） 优点：检索快，缺点：索引表额外占据存储空间；增删要修改索引表，时间花费多。
    </p>
    <p>
     ④
     <strong>
      散列存储
     </strong>
     （哈希存储）
    </p>
    <h4>
     2.2.3 运算
    </h4>
    <p>
     施加在数据上的运算，包括运算的定义和实现。运算的定义是针对逻辑结构的，指出运算的功能；运算的实现是针对存储结构的，指出运算的具体操作步骤。
    </p>

    <h2 style="text-align:center">
     <span style="color:#1c7331">
      三、算法
     </span>
    </h2>
    <h3>
     3.1 算法的概念和特征
    </h3>
    <p>
     算法是对特定问题求解步骤的一种描述，他是指令的有限数列，其中的每条指令表示一个或多个操作。
    </p>
    <p>
     特征：
     <span style="color:#ed7976">
      <strong>
       <span style="background-color:#fef2f0">
        ①有穷性 ②确定性 ③可行性 ④输入&gt;=0 ⑤输出&gt;=1
       </span>
      </strong>
     </span>
    </p>
    <h3>
     3.2 时间复杂度&amp;空间复杂度
    </h3>
    <p>
     是问题规模n的函数。
    </p>
    <blockquote>
     <p>
      Q:算法问题规模永远是A:不是。在算法中，问题规模不一定是n。
     </p>
     <p>
      n通常用来表示输入规模，比如在一个排序算法中，n可能代表要排序的元素个数。但问题规模也可以用其他方式来衡量。
      <br/>
      例如，在图算法中，除了用顶点数量n来表示问题规模，还可能会涉及边的数量m。对于一些复杂的算法，如计算两个图的同构问题，问题规模可能是由多个因素共同决定的，包括顶点数、边数、顶点和边的属性等复杂因素。
      <br/>
      在矩阵运算相关的算法中，矩阵的行数和列数都可能影响问题规模，而不是简单地用一个n来表示。
     </p>
    </blockquote>
    <p>
     常见的渐进时间复杂度比较：
    </p>
    <p>
     <span style="color:#be191c">
      <strong>
       <span style="background-color:#fef2f0">
        O(1) &lt; O(logn) &lt; O(n) &lt; O(nlogn) &lt; O(n^2) &lt; O(n^3) &lt; O(2^n) &lt; O(n!) &lt; O(n^n)
       </span>
      </strong>
     </span>
    </p>
