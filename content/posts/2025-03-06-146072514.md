         PXE原理
        </strong>
       </span>
      </strong>
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     <span style="color:#464646">
      PXE(preboot execute environment，预启动执行环境)是由
     </span>
     <a href="https://baike.baidu.com/item/Intel%E5%85%AC%E5%8F%B8" rel="nofollow" title="Intel公司">
      Intel公司
     </a>
     <span style="background-color:#ffffff">
      <span style="color:#464646">
       开发的最新技术，工作于Client/Server的网络模式，支持
      </span>
     </span>
     <a href="https://baike.baidu.com/item/%E5%B7%A5%E4%BD%9C%E7%AB%99/217955" rel="nofollow" title="工作站">
      工作站
     </a>
     <span style="background-color:#ffffff">
      <span style="color:#464646">
       通过网络从远端服务器下载映像，并由此支持通过网络启动操作系统，在启动过程中，终端要求服务器分配IP地址，再用
      </span>
     </span>
     <a href="https://baike.baidu.com/item/TFTP" rel="nofollow" title="TFTP">
      TFTP
     </a>
     <span style="color:#464646">
      （trivial file transfer protocol）或
     </span>
     <a href="https://baike.baidu.com/item/MTFTP" rel="nofollow" title="MTFTP">
      MTFTP
     </a>
     <span style="color:#464646">
      (multicast trivial file transfer protocol)协议下载一个启动软件包到本机内存中执行，由这个启动软件包完成终端（客户端）基本软件设置，从而引导预先安装在服务器中的终端操作系统
     </span>
    </p>
    <ol>
     <li style="text-align:left">
      <span style="color:#464646">
       PXE Client从PXE网卡启动，通过PXE BootRom以UDP的形式发送一个广播请求，向本网络中DHCP服务器所求IP
      </span>
     </li>
     <li style="text-align:left">
      <span style="color:#464646">
       DHCP服务器收到客户端的请求，验证是否来KickStart自动化安装部署自合法的PXEClient的请求，验证通过返回响应，响应内容包含：分配的IP地址、pxelinux启动程序的位置以及配置文件的位置
      </span>
     </li>
     <li style="text-align:left">
      <span style="color:#464646">
       客户端收到服务器的回应后，发送一个帧到TFTP服务端，请求传输启动所需文件，包括：pxelinux.0、pxelinux.cfg/default、vmlinuz、initrd.img等文件
      </span>
     </li>
     <li style="text-align:left">
      <span style="color:#464646">
       当服务器收到客户端的请求后，他们之间之后将有更多的信息在客户端与服务器之间作应答，用以决定启动参数。BootROM由TFTP通讯协议从Boot Server下载启动安装程序锁必须的文件（pxelinux.0
      </span>
      <span style="color:#464646">
       pxelinux.cfg/default）。default文件下载完成后，会根据该文件中定义的引导顺序，启动Linux安装程序的引导内核
      </span>
     </li>
     <li style="text-align:left">
      <span style="color:#464646">
       客户端通过pxelinux.cfg/default文件成功的引导Linux安装内核后，安装程序首先必须确定你通过什么安装介质来安装Linux，如果是通过网络安装（NFS、FTP、HTTP），则会在这个时候初始化网络，并定位安装源位置。接着会读取default文件中指定的自动应答文件ks.cfg所在位置，根据该位置请求下载该文件
      </span>
     </li>
     <li style="text-align:left">
      <span style="color:#464646">
       将ks.cfg文件下载回来后，通过该文件找到OSServer，并按照该文件的配置请求下载安装过程需要的软件包。OS Server和客户端建立连接后，将开始传输软件包，客户端将开始安装操作系统。安装完成后，将提示重新引导计算机。
      </span>
     </li>
    </ol>
