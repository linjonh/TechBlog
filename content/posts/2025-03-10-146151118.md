    <h4>
     <a id="_4">
     </a>
     简介
    </h4>
    <p>
     Redis 是一种开源（BSD 许可）、内存中数据结构存储，用作数据库、缓存和消息代理。Redis 提供了数据结构，例如字符串、散列、列表、集合、带有范围查询的排序集合、位图、超级日志、地理空间索引和流。Redis 内置复制、Lua 脚本、LRU 驱逐、事务和不同级别的磁盘持久化，并通过 Redis Sentinel 和 Redis Cluster 自动分区提供高可用性
     <br/>
     内容抄自官网
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fb29a4ef8041af60aa160597309ba7bb.png"/>
    </p>
    <h4>
     <a id="redis_10">
     </a>
     安装redis
    </h4>
    <p>
     <a href="https://redis.io/" rel="nofollow">
      https://redis.io/
     </a>
     访问官网进行下载
     <br/>
     链接：https://pan.baidu.com/s/1tiHwHW3mCSd_TfXbpbNb5Q
     <br/>
     提取码：yydd 附百度网盘安装包
    </p>
    <p>
     目前大部分都是在linux环境下使用 上传到服务器
     <br/>
     <strong>
      rz
     </strong>
     <br/>
     上传之后根据官方文档我们需要先下载GCC的工作环境对redis的支持
    </p>
    <p>
     查看服务器是否有gcc环境
    </p>
    <pre><code>gcc --version
</code></pre>
    <p>
     使用yum指令进行联网下载
    </p>
    <pre><code>yum install gcc
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e09489541b9f609c674b5f95e6b0c381.png">
      <br/>
      对文件进行解压 tar -zxvf 加文件名
     </img>
    </p>
    <pre><code>命令:   tar -zxvf Redis 6.2.6.tar.gz

-z：使用gzip进行解压缩
-x：解开tar文件
-v：显示解压缩执行过程
-f：指定要处理的文件名
</code></pre>
    <p>
     根据官方文档教程进入解压文件 对文件进行编译 安装
    </p>
    <pre><code>cd redis-6.2.6
make
make install
</code></pre>
    <p>
     默认安装位置为
     <strong>
      /usr/local/bin
     </strong>
     目录下
    </p>
    <pre><code>cd /usr/local/bin
</code></pre>
    <p>
     <strong>
      ll
     </strong>
     或者
     <strong>
      ls
     </strong>
     有以下文件
     <br/>
     <img alt="在这里插入得到图片述" src="https://i-blog.csdnimg.cn/blog_migrate/29e9f932b3d0279fcbc2a238d5e0ad94.png"/>
    </p>
    <pre><code>redis-benchmark: 性能测试工具，可以在自己电脑运行，看看本机性能如何·
redis-check-aof: 修复有问题的AOF文件
redis-check-rdb: RDB文件检查工具
redis-cli      : Redis命令行客户端
redis-sentinel : Redis软连接指向redis-server
redis-server   : Redis服务器
</code></pre>
    <p>
     启动Redis的
     <strong>
      两种
     </strong>
     方式
     <br/>
     <strong>
      1.前置启动
     </strong>
     <br/>
     安装目录下启动: /usr/local/redis/bin/redis-server
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/27f1eb1017018ca6e34cb2b3758a052a.png"/>
    </p>
    <p>
     在redis的安装目录的bin目录，./redis-server
     <br/>
     默认是前端启动模式，端口是6379
     <br/>
     缺点: ctrl+c停止前端模式的redis 则关闭redis(
     <strong>
      不推荐
     </strong>
     )
     <br/>
     2.后台启动(守护进程)
     <br/>
     回到Redis的安装目录中,复制一份redis.conf方便操作 ,cp redis.conf /etc/redis.conf
     <br/>
     安装vim yum install -y vim
     <br/>
     修改配置文件 vim redis.conf
     <br/>
     命令模式下搜索 /daemonize 改成yes (让服务在后台启动)
     <br/>
     回到安装目录 进行服务启动 需要指定文件启动
    </p>
    <pre><code>redis-server /etc/redis.conf
ps-ef |grep redis //查找redis的进程 发现已启动 端口号为6379
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a48ab6095572e800f139394396f0de5f.png">
      <br/>
      <strong>
       好处:是关闭窗口也依旧可以运行redis服务
      </strong>
      **(推荐)**
     </img>
    </p>
    <pre><code>redis-cli //连接客户端
</code></pre>
    <h4>
     <a id="Redis_92">
     </a>
     Redis的关闭
    </h4>
    <pre><code>shutdown
exit
</code></pre>
    <p>
     <strong>
      kill -9 &lt;redis进程编号&gt; 可以关闭服务器进程
     </strong>
    </p>
