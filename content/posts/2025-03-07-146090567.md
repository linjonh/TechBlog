    <p>
     最近在工作开发中遇到了一个小挑战，去掉组件里面的一些文字，研究了好久，原来是树形组件渲染需要一定的时间，需要使用延时才能看到效果。但是去掉后，把一些元素也去掉了，是因为方法用的不对，得使用内嵌的方法去掉。
    </p>
    <p>
     树形展开/收缩效果消失的原因可能是你在修改
     <code>
      textContent
     </code>
     时，意外破坏了 DOM 结构或事件绑定。以下是一些可能的原因和解决方法：
    </p>
    <hr/>
    <h4>
     1.
     <strong>
      检查 DOM 结构
     </strong>
    </h4>
    <p>
     修改
     <code>
      textContent
     </code>
     时，可能会覆盖掉某些子元素（例如展开/收缩按钮的图标或事件绑定）。确保你只修改文本内容，而不是整个元素的 HTML。
    </p>
    <h5>
     错误示例：
    </h5>
    <pre><code class="language-javascript">element.textContent = element.textContent.replace('分组:', '');</code></pre>
    <p>
     如果
     <code>
      element
     </code>
     包含子元素（如按钮图标），
     <code>
      textContent
     </code>
     会覆盖所有子元素。
    </p>
    <h5>
     正确示例：
    </h5>
    <pre><code class="language-javascript">const span = element.querySelector('span'); // 找到包含文本的子元素
if (span) {
  span.textContent = span.textContent.replace('分组:', '');
}</code></pre>
    <hr/>
    <h4>
     2.
     <strong>
      保留展开/收缩按钮
     </strong>
    </h4>
    <p>
     树形组件的展开/收缩按钮通常是动态生成的，可能是一个
     <code>
      &lt;span&gt;
     </code>
     或
     <code>
      &lt;i&gt;
     </code>
     元素。确保在修改文本内容时，不要覆盖这些按钮。
    </p>
    <h5>
     示例：
    </h5>
    <pre><code class="language-javascript">document.querySelectorAll('.lee-grid .lee-grid-grouprow-cell').forEach(element =&gt; {
  const textNode = Array.from(element.childNodes).find(node =&gt; node.nodeType === Node.TEXT_NODE);
  if (textNode) {
    textNode.textContent = textNode.textContent.replace('分组:', '');
  }
});</code></pre>
    <h5>
     解释：
    </h5>
    <ul>
     <li>
      <p>
       <code>
        element.childNodes
       </code>
       ：获取元素的所有子节点（包括文本节点和元素节点）。
      </p>
     </li>
     <li>
      <p>
       <code>
        node.nodeType === Node.TEXT_NODE
       </code>
       ：找到文本节点。
      </p>
     </li>
     <li>
      <p>
       只修改文本节点的内容，避免破坏其他子元素。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     3.
     <strong>
      使用
      <code>
       innerHTML
      </code>
      替代
      <code>
       textContent
      </code>
     </strong>
    </h4>
    <p>
     如果文本内容和子元素混合在一起，可以使用
     <code>
      innerHTML
     </code>
     来精确替换文本内容。
    </p>
    <h5>
     示例：
    </h5>
    <pre><code class="language-javascript">document.querySelectorAll('.lee-grid .lee-grid-grouprow-cell').forEach(element =&gt; {
  element.innerHTML = element.innerHTML.replace('分组:', '');
});</code></pre>
    <h5>
     注意：
    </h5>
    <ul>
     <li>
      <p>
       <code>
        innerHTML
       </code>
       会重新解析 HTML，可能会导致性能问题或事件绑定丢失。
      </p>
     </li>
     <li>
      <p>
       如果组件依赖于事件委托或动态绑定，
       <code>
        innerHTML
       </code>
       可能会破坏这些功能。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     4.
     <strong>
      重新绑定事件
     </strong>
    </h4>
    <p>
     如果修改
     <code>
      textContent
     </code>
     或
     <code>
      innerHTML
     </code>
     导致事件丢失，可以尝试重新绑定事件。
    </p>
    <h5>
     示例：
    </h5>
    <pre><code class="language-javascript">function updateTreeText() {
  document.querySelectorAll('.lee-grid .lee-grid-grouprow-cell').forEach(element =&gt; {
    element.textContent = element.textContent.replace('分组:', '');
  });

  // 重新绑定展开/收缩事件
  bindTreeEvents();
}

function bindTreeEvents() {
  document.querySelectorAll('.lee-grid-group-togglebtn').forEach(button =&gt; {
    button.addEventListener('click', () =&gt; {
      console.log('展开/收缩按钮被点击');
      // 展开/收缩逻辑
    });
  });
}

setTimeout(updateTreeText, 3000);</code></pre>
    <hr/>
    <h4>
     5.
     <strong>
      使用组件提供的方法
     </strong>
    </h4>
    <p>
     如果你使用的是某个 UI 组件库（如 Ant Design、Element UI 等），通常会有内置的方法来修改文本内容，而不破坏组件的功能。
    </p>
    <h5>
     示例：Ant Design Tree
    </h5>
    <pre><code class="language-javascript">const treeData = [
  {
    title: '分组:测试001',
    key: '1',
    children: [
      {
        title: '子节点 1',
        key: '1-1',
      },
    ],
  },
];

// 去掉“分组”字样
const newTreeData = treeData.map(node =&gt; {
  return {
    ...node,
    title: node.title.replace('分组:', ''),
  };
});

// 更新树形组件
treeRef.current.updateTreeData(newTreeData);</code></pre>
    <hr/>
    <h4>
     6.
     <strong>
      调试工具检查
     </strong>
    </h4>
    <p>
     使用浏览器的开发者工具（F12）检查 DOM 结构，确认修改后的元素是否仍然包含展开/收缩按钮。
    </p>
    <ul>
     <li>
      <p>
       打开开发者工具，切换到
       <strong>
        Elements
       </strong>
       面板。
      </p>
     </li>
     <li>
      <p>
       找到
       <code>
        .lee-grid-grouprow-cell
       </code>
       元素，检查其子元素是否完整。
      </p>
     </li>
     <li>
      <p>
       如果按钮丢失，说明修改方式有问题。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     7.
     <strong>
      延迟修改
     </strong>
    </h4>
    <p>
     如果组件在初始化时需要一些时间完成渲染，可以增加延迟时间。
    </p>
    <h5>
     示例：
    </h5>
    <pre><code class="language-javascript">setTimeout(() =&gt; {
  document.querySelectorAll('.lee-grid .lee-grid-grouprow-cell').forEach(element =&gt; {
    const textNode = Array.from(element.childNodes).find(node =&gt; node.nodeType === Node.TEXT_NODE);
    if (textNode) {
      textNode.textContent = textNode.textContent.replace('分组:', '');
    }
  });
}, 5000); // 延迟 5 秒执行</code></pre>
    <hr/>
    <h4>
     总结
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        避免覆盖子元素
       </strong>
       ：确保只修改文本内容，而不是整个元素的 HTML。
      </p>
     </li>
     <li>
      <p>
       <strong>
        保留展开/收缩按钮
       </strong>
       ：检查 DOM 结构，确保按钮未被覆盖。
      </p>
     </li>
     <li>
      <p>
       <strong>
        重新绑定事件
       </strong>
       ：如果事件丢失，重新绑定展开/收缩事件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用组件方法
       </strong>
       ：如果使用 UI 组件库，优先使用内置方法。
      </p>
     </li>
     <li>
      <p>
       <strong>
        调试工具检查
       </strong>
       ：使用开发者工具确认 DOM 结构是否完整。
      </p>
     </li>
    </ul>
    <p>
     通过以上方法，你应该能够解决树形展开/收缩效果消失的问题！
    </p>
