    <p>
     以下是一篇关于Git全流程操作的博文，内容涵盖从项目初始化到团队协作的完整步骤，适合新手和需要复习的开发者：
    </p>
    <p>
     Git全流程操作指南：从零到团队协作‌
    </p>
    <p>
     本文详细讲解Git在项目开发中的完整操作流程，涵盖‌本地仓库管理、远程协作、分支策略‌及‌常见问题解决‌，助你高效掌握Git核心技能。
    </p>
    <p>
     一、环境准备‌
    </p>
    <p>
     安装Git‌
    </p>
    <p>
     官网下载：https://git-scm.com/
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 检查安装是否成功
     <br/>
     git --version
    </p>
    <p>
     <br/>
     全局配置‌
    </p>
    <p>
     bash
     <br/>
     Copy Code
     <br/>
     git config --global user.name "Your Name"
     <br/>
     git config --global user.email "your.email@example.com"
     <br/>
     # 启用颜色标记（提升可读性）
     <br/>
     git config --global color.ui auto
    </p>
    <p>
     二、初始化项目‌
     <br/>
     1. ‌本地仓库创建‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 新建项目文件夹
     <br/>
     mkdir my-project &amp;&amp; cd my-project
     <br/>
     # 初始化Git仓库
     <br/>
     git init
    </p>
    <p>
     2. ‌关联远程仓库（GitHub/GitLab/Gitee）‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 添加远程仓库地址
     <br/>
     git remote add origin https://github.com/yourname/repo.git
     <br/>
     # 查看远程仓库配置
     <br/>
     git remote -v
    </p>
    <p>
     三、日常开发流程‌
     <br/>
     1. ‌提交代码到本地仓库‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 查看当前文件状态
     <br/>
     git status
    </p>
    <p>
     # 添加所有修改到暂存区
     <br/>
     git add .
     <br/>
     # 或添加指定文件
     <br/>
     git add file1.txt file2.js
    </p>
    <p>
     # 提交到本地仓库（推荐使用清晰的分段式注释）
     <br/>
     git commit -m "feat: add login page UI
     <br/>
     - 创建登录表单组件
     <br/>
     - 集成表单验证逻辑"
    </p>
    <p>
     2. ‌推送代码到远程仓库‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 首次推送需关联分支
     <br/>
     git push -u origin main
     <br/>
     # 后续推送简化命令
     <br/>
     git push
    </p>
    <p>
     四、分支管理策略‌
     <br/>
     1. ‌创建与切换分支‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 新建功能分支
     <br/>
     git branch feature/login
     <br/>
     # 切换分支
     <br/>
     git checkout feature/login
     <br/>
     # 快捷方式（创建并切换）
     <br/>
     git checkout -b feature/login
    </p>
    <p>
     2. ‌合并分支‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 切换到主分支
     <br/>
     git checkout main
     <br/>
     # 合并功能分支（推荐使用--no-ff保留合并记录）
     <br/>
     git merge --no-ff feature/login
    </p>
    <p>
     3. ‌删除分支‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 删除本地分支
     <br/>
     git branch -d feature/login
     <br/>
     # 强制删除未合并分支
     <br/>
     git branch -D feature/login
     <br/>
     # 删除远程分支
     <br/>
     git push origin --delete feature/login
    </p>
    <p>
     五、团队协作场景‌
     <br/>
     1. ‌拉取远程更新‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 拉取并自动合并（适合简单场景）
     <br/>
     git pull origin main
    </p>
    <p>
     # 推荐方式：先抓取再合并
     <br/>
     git fetch origin
     <br/>
     git merge origin/main
    </p>
    <p>
     2. ‌解决代码冲突‌
     <br/>
     执行git pull后若提示冲突：
     <br/>
     手动编辑冲突文件（搜索&lt;&lt;&lt;&lt;&lt;&lt;&lt;标记）
     <br/>
     标记冲突已解决：
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     git add resolved-file.js
     <br/>
     git commit -m "fix: resolve merge conflicts"
    </p>
    <p>
     3. ‌代码回退‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 撤销本地修改（危险操作！）
     <br/>
     git checkout -- file.txt
    </p>
    <p>
     # 重置到指定提交（hard模式慎用）
     <br/>
     git reset --hard commit_id
    </p>
    <p>
     六、高级实用技巧‌
     <br/>
     1. ‌储藏临时修改‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 临时保存未提交的修改
     <br/>
     git stash
     <br/>
     # 恢复最近储藏的内容
     <br/>
     git stash pop
    </p>
    <p>
     2. ‌查看提交历史‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 图形化显示分支结构
     <br/>
     git log --oneline --graph --decorate
    </p>
    <p>
     3. ‌打标签管理版本‌
     <br/>
     bash
     <br/>
     Copy Code
     <br/>
     # 创建发布标签
     <br/>
     git tag -a v1.0.0 -m "Release version 1.0.0"
     <br/>
     # 推送标签到远程
     <br/>
     git push origin --tags
    </p>
    <p>
     七、最佳实践建议‌
    </p>
    <p>
     提交规范‌
    </p>
    <p>
     使用Conventional Commits格式：
     <br/>
     feat:, fix:, docs:, style:, refactor:, test:, chore:
    </p>
    <p>
     .gitignore文件‌
     <br/>
     在项目根目录创建.gitignore文件，排除日志、临时文件等：
    </p>
    <p>
     plaintext
     <br/>
     Copy Code
     <br/>
     node_modules/
     <br/>
     *.log
     <br/>
     .env
    </p>
    <p>
     <br/>
     分支命名规范‌
    </p>
    <p>
     功能分支：feature/描述（如feature/user-auth）
     <br/>
     修复分支：hotfix/问题描述
     <br/>
     发布分支：release/版本号
     <br/>
     总结‌
    </p>
    <p>
     掌握这些Git操作流程，你已经能够应对90%的日常开发场景。建议结合图形化工具（如VSCode Git插件、GitKraken）提升效率，并通过Pro Git电子书深入学习底层原理。
    </p>
    <p>
     实践口诀‌：
    </p>
    <p>
     开发前先pull
     <br/>
     小步高频commit
     <br/>
     合并前先diff
     <br/>
     冲突及时解
    </p>
    <p>
     立即动手尝试这些命令，让你的版本控制更加得心应手！‌
    </p>
