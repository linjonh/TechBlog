    <h3>
     <a id="_14">
     </a>
     解决了什么问题
    </h3>
    <p>
     传统应用升级缓慢、架构臃肿、不能快速迭代、故障不能快速定位、问题无法快速解决等问题，云原生这一概念横空出世。
    </p>
    <h3>
     <a id="_17">
     </a>
     什么是云原生
    </h3>
    <p>
     云原生是一种行为方式和设计理念，究其本质，凡是能够
     <strong>
      提高云上资源利用率
     </strong>
     和
     <strong>
      应用交付效率的行为或方式
     </strong>
     都是云原生的。
    </p>
    <p>
     从宏观概念上讲，云原生是不同思想的集合，集目前各种热门技术之大成。
    </p>
    <p>
     从微观概念上讲，可分为如下四个方面：
    </p>
    <ul>
     <li>
      容器化：作为应用包装的载体
     </li>
     <li>
      持续交付：利用容器的轻便的特性，构建持续集成和持续发布的流水线
     </li>
     <li>
      DevOps：开发与运维之间的协同，上升到一种文化的层次，能够让应用快速的部署和发布
     </li>
     <li>
      微服务：这是应用开发的一种理念，将单体应用拆分为微服务才能更好的实现云原生，才能独立的部署、扩展和更新
     </li>
    </ul>
    <p>
     可能看起来有点晦涩难懂，我在这里说一下自己的理解：
    </p>
    <p>
     所谓的
     <code>
      原生
     </code>
     ，指的利用Java，php，c，c++，python等等开发语言开发出来的应用，应用需要部署到服务器上；而
     <code>
      云
     </code>
     指的是云服务器，那么
     <code>
      云原生
     </code>
     结合起来就是原生应用上云的过程，以及
     <strong>
      上云的一系列解决方案
     </strong>
     。
    </p>
    <p>
     <img alt="cloud-native-architecutre-mindnode.jpg" src="https://i-blog.csdnimg.cn/blog_migrate/a76dc7494b4be2154cfc2f311910f5be.jpeg"/>
    </p>
    <h3>
     <a id="_35">
     </a>
     云原生的设计理念及特性
    </h3>
    <p>
     云原生其设计理念如下：
    </p>
    <ul>
     <li>
      <p>
       面向分布式设计（Distribution）：容器、微服务、API 驱动的开发；
      </p>
     </li>
     <li>
      <p>
       面向配置设计（Configuration）：一个镜像，多个环境配置；
      </p>
     </li>
     <li>
      <p>
       面向韧性设计（Resistancy）：故障容忍和自愈；
      </p>
     </li>
     <li>
      <p>
       面向弹性设计（Elasticity）：弹性扩展和对环境变化（负载）做出响应；
      </p>
     </li>
     <li>
      <p>
       面向交付设计（Delivery）：自动拉起，缩短交付时间；
      </p>
     </li>
     <li>
      <p>
       面向性能设计（Performance）：响应式，并发和资源高效利用；
      </p>
     </li>
     <li>
      <p>
       面向自动化设计（Automation）：自动化的 DevOps；
      </p>
     </li>
     <li>
      <p>
       面向诊断性设计（Diagnosability）：集群级别的日志、metric 和追踪；
      </p>
     </li>
     <li>
      <p>
       面向安全性设计（Security）：安全端点、API Gateway、端到端加密；
      </p>
     </li>
    </ul>
    <p>
     综上所述，云原生应用应该具备以下几个特性：
    </p>
    <ul>
     <li>
      敏捷
     </li>
     <li>
      可靠
     </li>
     <li>
      高弹性
     </li>
     <li>
      易扩展
     </li>
     <li>
      故障隔离保护
     </li>
     <li>
      不中断业务持续更新
     </li>
    </ul>
    <h3>
     <a id="Kubernetes__66">
     </a>
     Kubernetes 与云原生的关系
    </h3>
    <p>
     Kubernetes：云原生操作系统
    </p>
    <p>
     <img alt="00704eQkgy1frr4z08j6oj31p20w2n6n.jpg" src="https://i-blog.csdnimg.cn/blog_migrate/9b07e0bf1296ffe260c9a897501aa3a2.jpeg"/>
    </p>
    <h4>
     <a id="Kubernetes__72">
     </a>
     Kubernetes 介绍
    </h4>
    <p>
     <a href="https://kubernetes.io/" rel="nofollow">
      Kubernetes
     </a>
     是 Google 基于
     <a href="https://research.google.com/pubs/pub43438.html" rel="nofollow">
      Borg
     </a>
     开源的容器编排调度引擎，作为
     <a href="https://cncf.io/" rel="nofollow">
      CNCF
     </a>
     （Cloud Native Computing Foundation）最重要的组件之一，它的目标不仅仅是一个编排系统，而是提供一个规范，可以让你来描述集群的架构，定义服务的最终状态，Kubernetes 可以帮你将系统自动得达到和维持在这个状态。
    </p>
    <p>
     更直白的说，Kubernetes 用户可以通过编写一个 YAML 或者 json 格式的配置文件，也可以通过工具 / 代码生成或直接请求 Kubernetes API 创建应用，该配置文件中包含了用户想要应用程序保持的状态，不论整个 Kubernetes 集群中的个别主机发生什么问题，都不会影响应用程序的状态，你还可以通过改变该配置文件或请求 Kubernetes API 来改变应用程序的状态。
    </p>
