     编写Dockerfile制作tomcat镜像，生成镜像名为tomcat:v1，并推送到私有仓库。
    </h1>
   </div>
  </div>

    <h2>
     1.具体要求如下：
    </h2>
    <ul>
     <li>
      <strong>
      </strong>
      基于rockylinux:8基础镜像；
     </li>
     <li>
      指定作者为openlab；
     </li>
     <li>
      安装tomcat服务，暴露8080端口；
     </li>
     <li>
      设置服务自启动。
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      总结步骤：基于rockylinux:8，安装Java环境，安装Tomcat，配置，暴露端口，设置启动命令，然后构建镜像，打标签，推送到私有仓库。
     </strong>
    </p>
    <p>
    </p>
    <p>
     宿主机：
    </p>
    <p>
     创建tomcat目录并进入: mkdir tomcat  ;  cd tomcat
    </p>
    <h2>
     2.下载所需文件（jdk和tomcat）
    </h2>
    <p>
     apache-tomcat-9.0.102.tar.gz   和
     <a class="link-info" href="https://download.oracle.com/otn/java/jdk/8u261-b12/a4634525489241b9a9e1aa73d9e118e6/jdk-8u261-linux-x64.rpm" rel="nofollow" title="jdk-8u261-linux-x64.rpm">
      jdk-8u261-linux-x64.rpm
     </a>
    </p>
    <h2>
     3、编写Dockerfile
     <br/>
     <br/>
     FROM rockylinux:8
     <br/>
     MAINTAINER openlab &lt;openlab@123.com&gt;
    </h2>
    <p>
    </p>
    <pre><code>FROM rockylinux:8
MAINTAINER openlab &lt;openlab@123.com&gt;
COPY  jdk-8u261-linux-x64.rpm /opt/jdk-8u261-linux-x64.rpm
ADD apache-tomcat-9.0.102.tar.gz /usr/local
RUN yum install -y /opt/jdk-8u261-linux-x64.rpm &amp;&amp; ln -s /usr/local/apache-tomcat-9.0.102 /usr/local/tomcat
EXPOSE 8080
CMD ["/usr/local/tomcat/bin/catalina.sh","run"]
</code></pre>
    <h2>
     4、制作镜像
    </h2>
    <p>
     docker build -t tomcat:v1 .
    </p>
    <p>
     <img alt="" height="275" src="https://i-blog.csdnimg.cn/direct/06b23b519ed740fe99190af32c0edc43.png" width="594"/>
    </p>
    <h2>
     5、测试镜像
    </h2>
    <p>
     docker run -d --name a1 -p 8080:8080 tomcat:v1
    </p>
    <p>
     执行后，通过docker ps -a 查看 发现容器未启动
    </p>
    <p>
     <img alt="" height="104" src="https://i-blog.csdnimg.cn/direct/e15bbf594f6e4554a092162174f4b98a.png" width="825"/>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
     <span style="color:#fe2c24">
      <strong>
       排查方法：
       <br/>
       查看日志  docker logs c1
       <br/>
       ...
       <br/>
       library initialization failed - unable to allocate file descriptor table - out of memory
       <br/>
       <br/>
       内存溢出，错误原因通常是由于系统资源不足，特别是文件描述符（file descriptor, FD）的数量不足
      </strong>
     </span>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
     <br/>
     <strong>
      解决方法：
      <br/>
      方法1： 在执行 docker run 命令时添加参数 --ulimit nofile=65535:65535 --ulimit nproc=65535:65535
      <br/>
      docker run  -d -p 8080:8080  --ulimit nofile=65535:65535 --ulimit nproc=65535:65535  tomcat:v1
     </strong>
    </p>
    <p>
     <img alt="" height="102" src="https://i-blog.csdnimg.cn/direct/4c0e01ad09b94c6780b5a8938c7dfb44.png" width="901">
      <br/>
      <strong>
       <br/>
      </strong>
     </img>
    </p>
    <p>
     <img alt="" height="952" src="https://i-blog.csdnimg.cn/direct/9f9ed819e0984678ae190818fa7601ac.png" width="1595"/>
    </p>
    <p>
    </p>
    <p>
     <strong>
      方法2： 修改docker服务脚本
      <br/>
      vim /usr/lib/systemd/system/docker.service
     </strong>
    </p>
    <p>
     <strong>
      在`ExecStart`参数值后面追加 `--default-ulimit nofile=65535:65535`，这样就能为创建的容器设置一个较高的文件描述符限制了。
     </strong>
    </p>
    <p>
     <strong>
      编辑完成后，重新加载systemd的配置，并重启Docker服务：
      <br/>
      输入以下命令：
      <br/>
      systemctl daemon-reload
      <br/>
      systemctl restart docker
     </strong>
    </p>
    <p>
     <strong>
      然后再进行测试
     </strong>
    </p>
    <p>
    </p>
    <h2>
     6.标记镜像为私有仓库格式
    </h2>
    <p>
     docker tag   tomcat:v1   192.168.234.15:5050/openlab/tomcat:v1
    </p>
    <p>
    </p>
    <h2>
     7.登录到私有仓库并推送镜像
    </h2>
    <p>
     docker login 192.168.234.15:5050
    </p>
    <p>
     docker push 192.168.234.15:5050/tomcat:v1
    </p>
    <p>
     <img alt="" height="305" src="https://i-blog.csdnimg.cn/direct/af9613489112409ba45466152b03f80e.png" width="946"/>
    </p>

    </p>
    <p>
     <strong>
      推送成功！！！
     </strong>
    </p>
