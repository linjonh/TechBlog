    <h3>
     <a id="___0">
     </a>
     关系型数据库 和 非关系型数据库
    </h3>
    <p>
     <strong>
      关系型数据库
     </strong>
    </p>
    <p>
     关系型数据库最典型的数据结构是表。由二维表及其之间的数据联系所组成的数据组织。
     <br/>
     优点：
     <br/>
     1.易于维护：格式一致，都是使用表结构。
     <br/>
     2.使用方便：SQL语言通用。
     <br/>
     3.能完成一些复杂操作：支持SQL,可用于一个表及多个表之间非常复杂的查询。
    </p>
    <p>
     缺点：
     <br/>
     1.读写性能较差。
     <br/>
     2.固定的表结构，灵活性欠缺。
     <br/>
     3.高并发读写需求，是很大的瓶颈。
    </p>
    <p>
     <strong>
      非关系型数据库
     </strong>
    </p>
    <p>
     非关系型数据库严格意义上不是一种数据库，应该是一种数据格式化储存的集合，可以是文档或者键值对形式。
     <br/>
     优点：
     <br/>
     1.格式灵活
     <br/>
     2.速度快：可以使用硬盘或者随机储存器作为载体，而关系型数据库只能使用硬盘。
     <br/>
     3.高拓展性。
     <br/>
     4.成本低，部署简单。
    </p>
    <p>
     缺点：
     <br/>
     1.不提供SQL支持，学习和使用的成本比较高。
     <br/>
     2.无事务处理。
     <br/>
     3.数据结构相对复杂，复杂查询方面稍微欠缺。
    </p>
