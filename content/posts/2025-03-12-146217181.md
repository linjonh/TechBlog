    <h2>
     <a id="_0">
     </a>
     概述
    </h2>
    <p>
     做过信创项目的兄弟们在工作上每天可能面对很多需要解决的问题，不过，有一类问题可能是大家经常遇的，比方说，有时候我们不得不硬着头皮在离线生产环境中安装某些软件包，相信很多兄弟被这种细碎的小事搞得焦头烂额。关于如何在离线生产环境中安装某些软件包，不同的操作系统、不同的应用场景，解决办法可能不尽相同。
    </p>
    <p>
     今天，我们来给大家介绍一下如何在Ubuntu离线场景中基于apt-offline安装特定的软件包。这里，我们以如何安装
     <code>
      openjdk-17
     </code>
     为例予以说明。
    </p>
    <h2>
     <a id="_6">
     </a>
     实践
    </h2>
    <h3>
     <a id="_7">
     </a>
     准备
    </h3>
    <p>
     在开始工作之前，我们准备了两个Ubuntu 24.04操作系统环境，具体如下面的表格所示：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        host name
       </th>
       <th>
        alias
       </th>
       <th>
        os
       </th>
       <th>
        offline status
       </th>
       <th>
        ip
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        shaolin
       </td>
       <td>
        少林
       </td>
       <td>
        ubuntu server 22.04
       </td>
       <td>
        offline
       </td>
       <td>
        192.168.43.226
       </td>
      </tr>
      <tr>
       <td>
        wudang
       </td>
       <td>
        武当
       </td>
       <td>
        ubuntu server 22.04
       </td>
       <td>
        online
       </td>
       <td>
        192.168.43.226
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     今天的实践是我们将借助主机
     <code>
      少林
     </code>
     来完成在主机
     <code>
      武当
     </code>
     上安装
     <code>
      openjdk-17
     </code>
     。
    </p>
    <h4>
     <a id="aptoffline_17">
     </a>
     安装apt-offline
    </h4>
    <p>
     在完成下面的实验之前，我们需要在主机
     <code>
      少林
     </code>
     与
     <code>
      武当
     </code>
     上分别安装上
     <code>
      apt-offline
     </code>
     软件包，这里为了模拟真实场景，我们需要在主机
     <code>
      少林
     </code>
     上从以下地址下载包含了依赖的
     <code>
      apt-offline
     </code>
     软件包。
    </p>
    <ul>
     <li>
      <a href="https://pkgs.org/download/apt-offline" rel="nofollow">
       https://pkgs.org/download/apt-offline
      </a>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/084d1665da9a46ed92498634958755ff.png">
      <br/>
      我这边的主机是x64架构，所以这里选择下载amd64对应的软件包
      <code>
       apt-offline_1.8.5-1_all.deb
      </code>
      。下载完成以后，可以使用U盘复制一份到主机
      <code>
       武当
      </code>
      上，然后在主机
      <code>
       少林
      </code>
      与
      <code>
       武当
      </code>
      上分别使用以下命令安装
      <code>
       apt-offline
      </code>
      。
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> dpkg <span class="token parameter variable">-i</span> apt-offline_*_all.deb
</code></pre>
    <h3>
     <a id="_28">
     </a>
     开始
    </h3>
    <h4>
     <a id="_30">
     </a>
     创建临时目录
    </h4>
    <p>
     首先，我们需要在主机
     <code>
      少林
     </code>
     与
     <code>
      武当
     </code>
     上的用户目录下面分别创建一个名称为
     <code>
      local
     </code>
     的目录，命令如下：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mkdir</span> <span class="token parameter variable">-p</span> ~/local
</code></pre>
    <h4>
     <a id="_38">
     </a>
     生成离线签名文件
    </h4>
    <p>
     接着，我们需要在目录local中生成
     <code>
      openjdk-17
     </code>
     对应的离线签名文件，命令如下：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> apt-offline <span class="token builtin class-name">set</span> openjdk-17.sig --install-packages openjdk-17*
</code></pre>
    <h4>
     <a id="_46">
     </a>
     基于离线签名下载依赖包
    </h4>
    <p>
     上面，我们已经在主机
     <code>
      武当
     </code>
     用户当前目录下的local中生成了
     <code>
      openjdk-17
     </code>
     对应的离线签名文件
     <code>
      openjdk-17.sig
     </code>
     。这个时候，我们需要借助U盘复制文件
     <code>
      openjdk-17.sig
     </code>
     到主机
     <code>
      少林
     </code>
     的用户目录下的目录
     <code>
      local
     </code>
     中。接下来，我们依据签名文件在主机
     <code>
      少林
     </code>
     的用户目录下的目录
     <code>
      local
     </code>
     中下载并生成依赖软件包。
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> ~/local
<span class="token function">sudo</span> apt-offline get openjdk-17.sig <span class="token parameter variable">--thread</span> <span class="token number">2</span> <span class="token parameter variable">--bundle</span> openjdk-17.zip
</code></pre>
    <h4>
     <a id="_54">
     </a>
     离线安装
    </h4>
    <p>
     上面，我们已经成功的在主机
     <code>
      少林
     </code>
     的用户目录下的目录
     <code>
      local
     </code>
     中生成了
     <code>
      openjdk-17.sig
     </code>
     的依赖包
     <code>
      openjdk-17.zip
     </code>
     ，这个时候，我们借助U盘将文件
     <code>
      openjdk-17.zip
     </code>
     复制到主机
     <code>
      武当
     </code>
     用户当前目录下的local中。
    </p>
    <p>
     接下来，我们借助以下命令，将
     <code>
      openjdk-17.sig
     </code>
     的依赖包
     <code>
      openjdk-17.zip
     </code>
     导入本地apt数据库。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> apt-offline <span class="token function">install</span> openjdk-17.zip
</code></pre>
    <p>
     接着使用如下命令完成
     <code>
      openjdk-17
     </code>
     的安装，。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> openjdk-17*
</code></pre>
    <p>
     安装完成以后，我们可以在命令行，借助如下命令查看openjdk-17是否生效。
    </p>
    <pre><code class="prism language-bash"><span class="token function">java</span> <span class="token parameter variable">-version</span>
</code></pre>
    <p>
     如果能正常输出java版本，则说明安装成功。
    </p>
    <h2>
     <a id="_77">
     </a>
     小结
    </h2>
    <p>
     我们在上面的实践中，以借助在线主机
     <code>
      少林
     </code>
     在离线主机
     <code>
      武当
     </code>
     上安装
     <code>
      openjdk-17
     </code>
     为例，向大家介绍了如果基于
     <code>
      apt-offline
     </code>
     命令完成离线软件包的安装。这里需要注意以下几点事项：
    </p>
    <ul>
     <li>
      两台主机需要是相同的操作系统版本。
     </li>
     <li>
      需要能从主机
      <code>
       武当
      </code>
      上得到待安装软件的签名文件。
     </li>
     <li>
      需要在主机
      <code>
       武当
      </code>
      上事先安装好
      <code>
       apt-offline
      </code>
      。
     </li>
    </ul>
    <p>
     如果我们不能主机
     <code>
      武当
     </code>
     上事先安装好
     <code>
      apt-offline
     </code>
     ，或者如果我们不能从主机
     <code>
      武当
     </code>
     上得到待安装软件的签名文件，在主机
     <code>
      少林
     </code>
     依然可以借助
     <code>
      apt-offline
     </code>
     完成软件包
     <code>
      openjdk-17
     </code>
     的下载。在人为可以保障主机
     <code>
      少林
     </code>
     与
     <code>
      武当
     </code>
     操作系统版本一致，软件环境也一致的情况下，我们可以借助如下命令多次试探着完成软件包的安装。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mkdir</span> <span class="token parameter variable">-p</span> openjdk-17
<span class="token function">sudo</span> <span class="token function">unzip</span> <span class="token parameter variable">-d</span> openjdk-17/ openjdk-17.zip
<span class="token builtin class-name">cd</span> openjdk-17/
<span class="token function">sudo</span> dpkg <span class="token parameter variable">-i</span> *.deb
</code></pre>
    <p>
     正所谓条条大路通罗马，除了基于apt-offline来完成Ubuntu离线环境下的软件包的安装外，我们还可以采用如下方案完成相关尝试。
    </p>
    <ul>
     <li>
      ISO文件本地镜像法
     </li>
    </ul>
    <p>
     这种方案就是将Ubuntu安装包ISO文件挂载到本地环境，然后将本地软件镜像源切换为本地来完成相关软件包的安装。
    </p>
    <ul>
     <li>
      源码编译法
     </li>
    </ul>
    <p>
     这种方案获取软件包的源码文件，在本地环境编译，生成相关的目标二进制可执行文件。
    </p>
    <ul>
     <li>
      二进制软件包法
     </li>
    </ul>
    <p>
     有的软件包官网会提供已经编译好的二进制可执行文件包，通常以.gz.biz或者zip为后续，下载以后，将二进制可执行文件添加到环境变更path中，再依据官方操作手册对相关配置文件调整以后就完成了安装。
    </p>
