    <h4>
     <a id="_1">
     </a>
     网络信息安全相关概念
    </h4>
    <p>
     网络安全是指通过采取必要措施，防范对网络的攻击、侵入、干扰、破坏和非法使用以及意外事故，使网络处于稳定可靠允许的状态，以及保证网络数据的完整性、保密性、可用性的能力。
    </p>
    <h4>
     <a id="_3">
     </a>
     网络信息安全问题
    </h4>
    <ol>
     <li>
      网络强依赖性及网络安全关联风险凸显
     </li>
     <li>
      网络信息产品供应链与安全质量风险
     </li>
     <li>
      网络信息产品技术同质性与技术滥用风险
     </li>
     <li>
      网络安全建设与管理发展不平衡、不充分风险
     </li>
     <li>
      网络数据安全风险
     </li>
     <li>
      高级持续威胁风险
     </li>
     <li>
      恶意代码风险
     </li>
     <li>
      软件代码和安全漏洞风险
     </li>
     <li>
      人员的网络安全意识风险
     </li>
     <li>
      网络信息技术复杂性和运营安全风险
     </li>
     <li>
      网络地下黑产经济风险
     </li>
     <li>
      网络间谍与网络战风险
     </li>
    </ol>
    <h4>
     <a id="_17">
     </a>
     网络信息安全基本属性
    </h4>
    <h5>
     <a id="_18">
     </a>
     机密性
    </h5>
    <p>
     指网络信息不泄露给非授权的用户、实体或程序，能够防止非授权者获取信息
    </p>
    <h5>
     <a id="_20">
     </a>
     完整性
    </h5>
    <p>
     网络信息或系统未经授权不能进行更改的特性
    </p>
    <h5>
     <a id="_22">
     </a>
     可用性
    </h5>
    <p>
     合法许可的用户能够及时获取网络信息或服务的特性
    </p>
    <h5>
     <a id="_24">
     </a>
     抗抵赖性
    </h5>
    <p>
     防止网络信息系统相关用户否认其活动行为的特性
    </p>
    <h5>
     <a id="_26">
     </a>
     可控性
    </h5>
    <p>
     网络信息系统责任主体对其具有管理、支配能力的属性
    </p>
    <h5>
     <a id="_28">
     </a>
     其他
    </h5>
    <p>
     真实性、时效性、合规性、公平性、可靠性、可生存性、隐私性
    </p>

    <h5>
     <a id="_53">
     </a>
     网络安全等级保护
    </h5>
    <p>
     主要工作：定级、备案、建设整改、等级测评、运营维护。
     <br/>
     网络安全保护义务：
    </p>
    <ul>
     <li>
      制定内部安全管理制度和操作规程，确定网络安全负责人，落实网络安全保护责任；
     </li>
     <li>
      采取防范计算机病毒和网络攻击、网络侵入等危害网络安全行为的技术措施；
     </li>
     <li>
      采取监测、记录网络运行状态、网络安全事件的技术措施，并按照规定留存相关的网络日志不少于六个月；
     </li>
     <li>
      采取数据分类、重要数据备份和加密等措施；
     </li>
     <li>
      法律、行政法规规定的其他义务
     </li>
    </ul>
