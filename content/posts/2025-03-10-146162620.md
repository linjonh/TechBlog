     文件内容介绍
    </h2>
    <p>
     <strong>
      文件 = 内容+属性.
     </strong>
    </p>
    <blockquote>
     <p>
      文件系统
     </p>
     <ul>
      <li>
       被打开的文件         -------内存中
      </li>
      <li>
       没有被打开的文件  -------磁盘中
      </li>
     </ul>
    </blockquote>
    <blockquote>
     <p>
      没打开文件之前,文件存储在磁盘, 进程想要访问文件之前 ,要打开文件(也就是将文件加载到内存中)
     </p>
     <p>
      <strong>
       文件加载到内存中要被OS管理(先描述,再组织)
      </strong>
     </p>
     <p>
      <strong>
       也就产生了文件的
       <span style="color:#fe2c24">
        内核数据结构+文件的内容.(类似进程)
       </span>
      </strong>
     </p>
    </blockquote>

    <h2 id="%E7%B3%BB%E7%BB%9F%E6%96%87%E4%BB%B6I%2FO" name="%E7%B3%BB%E7%BB%9F%E6%96%87%E4%BB%B6I%2FO">
     系统文件I/O
    </h2>
    <p>
     前面在讲计算机冯诺依曼体系结构时讲过
    </p>
    <blockquote>
     <ul>
      <li>
       OS要确保底层的安全稳定,对于上层开发
       <strong>
        提供了一些系统调用(system call)接口
       </strong>
       ,用于对硬件的访问(I/O)
      </li>
      <li>
       而上面C语言的库函数fopen可以打开文件, fwrite可以向文件中写入, 文件在磁盘上,可以推断出,这些C语言中这些文件I/O一定封装了系统调用接口.
      </li>
     </ul>
    </blockquote>
    <h3 id="%E6%96%87%E4%BB%B6%E7%B1%BB%E7%9A%84%E7%B3%BB%E7%BB%9F%E8%B0%83%E7%94%A8%E6%8E%A5%E5%8F%A3%E4%BB%8B%E7%BB%8D" name="%E6%96%87%E4%BB%B6%E7%B1%BB%E7%9A%84%E7%B3%BB%E7%BB%9F%E8%B0%83%E7%94%A8%E6%8E%A5%E5%8F%A3%E4%BB%8B%E7%BB%8D" style="background-color:transparent">
     文件类的系统调用接口介绍
    </h3>
    <p>
     <strong>
      open接口可以在linux中输入查询
     </strong>
    </p>
    <pre><code class="language-cpp">man 2 open</code></pre>
    <pre><code class="language-cpp">#include &lt;sys/types.h&gt;
#include &lt;sys/stat.h&gt;
#include &lt;fcntl.h&gt;
 
int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);
 
pathname: 要打开或创建的目标文件
flags: 打开文件时，可以传入多个参数选项，用下面的一个或者多个常量进行“或”运算，构成flags。

flags的参数:
 O_RDONLY: 只读打开
 O_WRONLY: 只写打开
 O_RDWR : 读，写打开
 这三个常量，必须指定一个且只能指定一个
 O_CREAT : 若文件不存在，则创建它。需要使用mode选项，来指明新文件的访问权限
 O_APPEND: 追加写

返回值：
  成功：新打开的文件描述符
  失败：-1</code></pre>
    <blockquote>
     <p>
      flags是位图,  参数:  比如O_RDONLY是32位中只有一个为1的int.
     </p>
     <p>
      因此,参数之间要用  |  或运算.
     </p>
    </blockquote>
    <p>
    </p>
    <blockquote>
     <p>
      <strong>
       mode_t理解：
      </strong>
      第三个参数 mode用于指定
      <strong>
       创建文件时的权限
      </strong>
      （文件模式）。它
      <span style="color:#fe2c24">
       <strong>
        仅在创建新文件时生效.
       </strong>
      </span>
     </p>

       示例:

      <li>
       <p>
        创建一个新文件，权限为
        <code>
         0644
        </code>
        （用户可读写，组和其他用户只读）：
       </p>
      </li>
     </ol>
     <pre><code class="language-cpp">int fd = open("example.txt", O_CREAT | O_WRONLY, 0644);</code></pre>

    <h4 id="write%20read%20close%20lseek%20%2C%E7%B1%BB%E6%AF%94C%E6%96%87%E4%BB%B6%E7%9B%B8%E5%85%B3%E6%8E%A5%E5%8F%A3" name="write%20read%20close%20lseek%20%2C%E7%B1%BB%E6%AF%94C%E6%96%87%E4%BB%B6%E7%9B%B8%E5%85%B3%E6%8E%A5%E5%8F%A3">
     write read close lseek ,类比C文件相关接口
    </h4>
    <p>
    </p>
    <h3 id="open%E5%87%BD%E6%95%B0%E8%BF%94%E5%9B%9E%E5%80%BC" name="open%E5%87%BD%E6%95%B0%E8%BF%94%E5%9B%9E%E5%80%BC">
     <span style="color:#fe2c24">
      系统调用
     </span>
     和
     <span style="color:#fe2c24">
      库函数
     </span>
    </h3>
    <blockquote>
     <ul>
      <li>
       上面的 fopen fclose fread fwrite 都是C标准库当中的函数，我们称之为
       <strong>
        库函数（libc)
       </strong>
      </li>
      <li>
       而， open close read write lseek 都属于系统提供的接口，称之为
       <strong>
        系统调用接口
       </strong>
       回忆一下讲操作系统概念时，画的一张图
      </li>
      <li>
       <span style="color:#0d0016">
        <strong>
         也就是C语言的文件接口(fopen fclose fread fwrite ),封装了文件类的系统调用接口(open read write close),方便了二次开发.
        </strong>
       </span>
      </li>
     </ul>
     <p>
      <img alt="" height="756" src="https://i-blog.csdnimg.cn/direct/8970fb3bb6eb45c39ac130fc8b9e400b.png" width="875"/>

    </blockquote>
    <h2 name="%E6%96%87%E4%BB%B6%E6%8F%8F%E8%BF%B0%E7%AC%A6fd">

    <h2 id="%E6%96%87%E4%BB%B6%E6%8F%8F%E8%BF%B0%E7%AC%A6fd" name="%E6%96%87%E4%BB%B6%E6%8F%8F%E8%BF%B0%E7%AC%A6fd">
     <span style="color:#0d0016">
      文件描述符fd
     </span>
    </h2>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       通过对open函数的学习，我们知道了open 的返回值是文件描述符, 就是一个小整数
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       在上面open示例代码中发现打出来myfile.txt的文件描述符fd=3
      </span>
     </p>
    </blockquote>
    <h4 id="%E4%B8%BA%E4%BB%80%E4%B9%88%E6%89%93%E5%BC%80%E6%96%87%E4%BB%B6myfile.txt%E7%9A%84%E6%8F%8F%E8%BF%B0%E7%AC%A6%E6%98%AF3%E4%B8%8D%E6%98%AF0%3F" name="%E4%B8%BA%E4%BB%80%E4%B9%88%E6%89%93%E5%BC%80%E6%96%87%E4%BB%B6myfile.txt%E7%9A%84%E6%8F%8F%E8%BF%B0%E7%AC%A6%E6%98%AF3%E4%B8%8D%E6%98%AF0%3F">
     <span style="color:#0d0016">
      为什么打开文件myfile.txt的描述符是3不是0?
     </span>
    </h4>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       进程启动 ,默认打开三个标准的输入输出流
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       因为在打开文件myfile.txt前,就打开了三个文件(stdin stdout stderr),他们的文件描述符分别是0 ,1 ,2
      </span>
     </p>
    </blockquote>
    <blockquote>
     <h4>
      <span style="color:#0d0016">
       验证: 默认打开三个标准的输入输出流
      </span>
     </h4>
     <pre><code class="language-cpp">  1 #include&lt;stdio.h&gt;
  2 #include&lt;string.h&gt;
  3 #include&lt;errno.h&gt;
  4 #include &lt;sys/types.h&gt;
  5 #include &lt;sys/stat.h&gt;
  6 #include &lt;fcntl.h&gt;
  7 
  8 int main()
  9 {
 10     printf("stdin: %d\n", stdin-&gt;_fileno);
 11     printf("stdout: %d\n", stdout-&gt;_fileno);
 12     printf("stderr: %d\n", stderr-&gt;_fileno);
 13 
 14     int fd1 =open("open.txt" , O_RDWR|O_CREAT , 0666);
 15     if(fd1 &lt; 0)
 16     {
 17         perror("open");
 18         return 1;
 19     }
 20     printf("fd1 =%d\n",fd1);
 21 
 22     const char* message ="hello linux\n";
 23     write(fd1, message, strlen(message));
 24     close(fd1);
 25     return 0;
    }</code></pre>
     <h4>
      <img alt="" height="663" src="https://i-blog.csdnimg.cn/direct/f89bacb0ba9d4cac97181d9949f385c2.png" width="1199"/>

    </blockquote>
    <p>
    </p>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       (先描述,再组织) 当我们打开文件时，操作系统在内存中要创建相应的数据结构来描述目标文件.
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       <strong>
        struct file
       </strong>
       就是
       <strong>
        文件的属性
       </strong>
       ,类似
       <strong>
        进程的task_struct
       </strong>
      </span>
     </p>
     <p>
     </p>
     <p>
      <img alt="" height="668" src="https://i-blog.csdnimg.cn/direct/65de0e87288545e89014f5b39ad1aa82.png" width="1407"/>
     </p>
    </blockquote>
    <h4>
    </h4>
    <h4>
     <strong>
      一个进程通常打开多个文件,如何让进程与文件关联?
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       每个进程都有一个指针*ﬁles, 指向一张表 ﬁles_struct , 该表最重要的部分就是包涵一个
       <strong>
        指针数组fd_array(文件描述符表)
       </strong>
       ，每个元素都是一个指向打开文件的指针！
      </li>
     </ul>
     <p>
      <span style="color:#0d0016">
       所以，本质上，文件描述符就是该数组的下标。所以，只要拿着文件描述符，就可以找到对应的文件
      </span>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       进程的task_struct中有 file_struct的指针
      </strong>
     </p>
     <p>
      <img alt="" height="303" src="https://i-blog.csdnimg.cn/direct/bf6f5b6c648e479b909812524ecbce6b.png" width="537"/>
     </p>
     <p>
      file_struct中有指针数组fd_array
      <strong>
       (文件描述符表)
      </strong>
     </p>
     <p>
      <img alt="" height="515" src="https://i-blog.csdnimg.cn/direct/a115096a26fb4f29afc34d58783783c8.png" width="919"/>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       OS想要管理进程创建了
       <strong>
        进程管理列表
       </strong>
       ,文件同理,创建了
       <strong>
        文件管理列表
       </strong>
       (也是双链表)
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       <strong>
        内核中,让进程和文件之间产生关联的是
       </strong>
      </span>
      <span style="color:#fe2c24">
       <strong>
        文件描述符表
       </strong>
      </span>
      <span style="color:#0d0016">
       <strong>
        , 左边是进程,右面是文件 ,通过文件描述符表进行解耦合
       </strong>
      </span>
