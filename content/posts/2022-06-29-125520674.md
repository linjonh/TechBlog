     Freeswitch和微信小程序对接
    </h1>
   </div>

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <link href="./../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     由于微信小程序具备免安装和使用方便等特点，受到越来越多公司的青睐;
    </p>
    <p style="margin-left:0;text-align:justify;">
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       <strong>
       </strong>
       微信小程序是2017
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       年腾讯视频云团队跟微信团队联合，将视频云
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       SDK
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       跟微信小程序整合在一起，并通过
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       &lt;live-pusher&gt;
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       和
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       &lt;live-player&gt;
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       两个标签的形式开放内部的功能。通过这两个标签，开发者可以实现在线直播、低延时监控、双人视频通话以及多人视频会议等功能。
      </span>
     </span>
    </p>
    <p style="margin-left:0;text-align:justify;">
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       由于微信小程序采用
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       RTMP
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       协议进行音视频通讯，所以我们在
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       Freeswitch
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       内置了一个
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       endpoint
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       模块通过
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       RTMP
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       协议和微信小程序通讯达到音视频通讯
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       ；有了这个模块就建立起了小程序到
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       Freeswitch音视频
      </span>
     </span>
     <span style="background-color:#FFFFFF;">
      <span style="color:#191919;">
       通讯的桥梁，方便小程序和其它平台进行互联互通。比如：呼叫电话，电话会议，视频会议；最重要是用户无需安装，扫描即可使用。
      </span>
     </span>
    </p>
    <p>
     微信小程序对接FreeSwitch在开发过程中会遇到很多痛点，现归类如下：
    </p>
    <p>
     1、webrtc方式不能在小程序中使用，官方不提供此类接口；
    </p>
    <p>
     2、WebAudio方式，回声处理效果不好，可以pass掉；
    </p>
    <p>
     3、单一采用rtmp方式，无法进行呼叫、自定义事件通知等等；
    </p>
    <p>
     4、结合以上问题，最终采用websocket+rtmp方式，websocket处理信令，rtmp处理语音；
    </p>
    <p>
     实现步骤
    </p>
    <p>
     1、在Freeswitch中创建一个endpoint模块；
    </p>
    <p>
     2、创建一个websocket服务，负责注册，呼叫，挂机和事件通知（接通，振铃，挂机，语音识别推送）；
    </p>
    <p>
     3、创建一个rtmp服务，负责rtmp协议和媒体收发；
    </p>
    <p>
     （待续......)
    </p>
