     系统架构的评估的系统的质量属性
    </h1>

    <p>
     体系结构苹果可以针对一个体系结构，也可以针对一组体系结构。
    </p>
    <p>
     体系结构评估过程中，评估人员所关注的是系统的质量属性，所有评估方法所普遍关注的质量属性有以下几个：性能、可靠性（容错，健壮性）、可用性、安全性、可修改性(可维护性、可扩展性、结构重组、可移植性)、功能性、可变性、互操作性。
    </p>
    <p>
     <strong>
      系统的质量属性：
     </strong>
    </p>
    <p>
    </p>
    <p>
     <strong>
      1.性能 (Performance)
     </strong>
     <br/>
     指系统的响应能力，常用单位时间内所处理事务的数量或系统完成某个事务处理所需的时间来对性能进行定量表示。常使用基准测试程序。如响应时间、吞吐量。
     <br/>
     提升性能设计策略：优先级队列、增加计算资源、减少计算开销、引入并发机制、采用资源调度等。
     <br/>
     <strong>
      2.可靠性(Reliability)
     </strong>
     <br/>
     是软件系统在应用或系统错误面前，在意外或错误使用的情况下维持软件系统的功能特性的基本能力。
     <br/>
     可靠性是最重要的软件特性，通常用来衡量在规定的条件和时间内，软件完成规定功能的能力。可靠性通常用
     <strong>
      平均失效等待时间 (Mean Time To Failure,MTTF
     </strong>
     ) 和
     <strong>
      平均失效间隔时间 (Mean Time Between Failure,MTBF)
     </strong>
     来衡量。在失效率为常数和修复时间很短的情况下， MTTF 和 MTBF几乎相等。
    </p>
    <blockquote>
     <p>
      MTTF = ΣT1/N。这里的T1指的是每次正常运行的时间，N是故障次数。
     </p>
     <p>
      公式为：
     </p>
     <p>
      <img alt="" height="45" src="https://i-blog.csdnimg.cn/direct/4cee230c6f574e669a37fc427afd2222.png" width="296"/>
     </p>
     <ul>
      <li>
       ‌
       <strong>
        参数说明
       </strong>
       ‌：
       <ul>
        <li>
         T1：每次从启动到故障发生前的正常运行时长；
        </li>
        <li>
         N：统计周期内的故障次数‌23。
        </li>
       </ul>
      </li>
     </ul>
     <p>
     </p>
     <table>
      <thead>
       <tr>
        <th>

        </th>
       </tr>

        </td>

        </td>

        </td>
       </tr>

        </td>

        </td>
        <td>

        </td>
        <td>
         两次故障间的平均间隔时间 ‌45
        </td>
       </tr>
       <tr>
        <td>
         MTTR
        </td>
        <td>
         可修复系统
        </td>
        <td>
         —
        </td>
        <td>
         故障后平均修复耗时 ‌23
        </td>
       </tr>
      </tbody>
     </table>
     <p>
      MTBF：
     </p>
     <p>
      <img alt="" height="120" src="https://i-blog.csdnimg.cn/direct/a4cd1c2c7c85402897e421f672d06df3.png" width="257"/>
     </p>
     <ul>
      <li>
       <p>
        计算 MTBF（平均时间故障间隔）的方法是使用以下公式：
       </p>
       <p>
        MTBF = 总运行时间 / 故障次数
       </p>
       <p>
        例如，如果一个测试自动化套件运行了 1000 小时并经历了 10 次故障，那么 MTBF 将为：
       </p>
       <p>
        MTBF = 1000 小时 / 10 次故障 = 100 小时
       </p>
       <p>
        这意味着，在平均情况下，系统可以在故障之间运行 100 小时。需要注意的是，MTBF 是一个统计性的度量，应与其他可靠性指标一起使用以进行全面的分析。在大量时间和测试周期的情况下进行计算时，它的实用性最高。
       </p>
      </li>
      <li>
       ‌
       <strong>
        统计原理
       </strong>
       ‌：MTBF的估计值符合卡方分布，常用于工程可靠性分析‌。
      </li>
     </ul>
     <p>
     </p>
     <p>
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=MTBF&amp;zhida_source=entity" rel="nofollow" title="MTBF">
       MTBF
      </a>
      、
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=MTTR&amp;zhida_source=entity" rel="nofollow" title="MTTR">
       MTTR
      </a>
      、
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=MTTF&amp;zhida_source=entity" rel="nofollow" title="MTTF">
       MTTF
      </a>
      三个指标的区别
     </p>
     <p>
      1、MTBF，全称是
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=Mean+Time+Between+Failure&amp;zhida_source=entity" rel="nofollow" title="Mean Time Between Failure">
       Mean Time Between Failure
      </a>
      ，即平均无故障工作时间。就是从新的产品在规定的工作环境条件下开始工作到出现第一个故障的时间的平均值。MTBF越长表示可靠性越高正确工作能力越强。
     </p>
     <p>
      2、MTTR，全称是
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=Mean+Time+To+Repair&amp;zhida_source=entity" rel="nofollow" title="Mean Time To Repair">
       Mean Time To Repair
      </a>
      ，即平均修复时间。是指可修复产品的平均修复时间，就是从出现故障到修复中间的这段时间。MTTR越短表示易恢复性越好。
     </p>
     <p>
      3、MTTF，全称是
      <a href="https://zhida.zhihu.com/search?content_id=203181858&amp;content_type=Article&amp;match_order=1&amp;q=Mean+Time+To+Failure&amp;zhida_source=entity" rel="nofollow" title="Mean Time To Failure">
       Mean Time To Failure
      </a>
      ，即平均失效时间。系统平均能够正常运行多长时间，才发生一次故障。系统的可靠性越高，平均无故障时间越长。 可靠性是最初是确定一个系统在一个特定的运行时间内有效运行的概率的一个标准。可靠性的衡量需要系统在某段时间内保持正常的运行。
     </p>
     <p>
      目前，使用最为广泛的一个衡量可靠性的参数是，MTTF(mean time to failure，平均失效前时间)，定义为随机变量、出错时间等的"期望值"。但是，MTTF经常被错误地理解为，"能保证的最短的生命周期"。MTTF的长短，通常与使用周期中的产品有关，其中不包括老化失效。
     </p>
    </blockquote>
    <p>
    </p>
    <p>
     <br/>
     可靠性可以分为两个方面：
     <br/>
     1）
     <strong>
      容错
     </strong>
     ：错误发生时确保系统正确的行为，并进行内部“修复”。
     <br/>
     2）
     <strong>
      健壮性
     </strong>
     ：错误发生时保证软件按照某种已经定义好的方式终止执行。
     <br/>
     设计策略：软件体系结构通过在应用程序内部包含冗余，或集成监控构件和异常处理，来支持可靠性。
    </p>
    <blockquote>
     <p>
      其他，可靠性设计技术主要有容错设计、检错设计和降低复杂度设计。
     </p>
    </blockquote>
    <p>
     <br/>
     <strong>
      3.可用性 (Availability)
     </strong>
     <br/>
     是系统能够正常运行的时间比例。经常用两次故障之间的时间长度或在出现故障时系统能够恢复正常的速度来表示。如故障间隔时间。
     <br/>
     设计策略：心跳、Ping/Echo、冗余、选举。
     <br/>
     <strong>
      4.安全性 (Security)
     </strong>
     <br/>
     是指系统在向合法用户提供服务的同时能够阻止非授权用户使用的企图或拒绝服务的能力。如保密性、完整性、不可抵赖性、可控性。
     <br/>
     设计策略：入侵检测、用户认证、用户授权、追踪审计。
     <br/>
     <strong>
      5.可修改性 (Modifiability)
     </strong>
     <br/>
     是指能够快速地以较高的性价比对系统进行变更的能力。
     <br/>
     包含以下4个方面：
     <br/>
     1）
     <strong>
      可维护性 (Maintainability)
     </strong>
     ： 这主要体现在问题的修复上，在错误发生后“修复”软件系统。
     <br/>
     2）
     <strong>
      可扩展性 (Extendibility)
     </strong>
     ：新特性来扩展软件系统。
     <br/>
     3）
     <strong>
      结构重组 (Reassemble)
     </strong>
     <br/>
     4）
     <strong>
      可移植性 (Portability) ：系统能在不同计算环境下运行的能力。
     </strong>
    </p>
    <p>
     实现可移植按照硬件无关方式组织软件系统，其他软件系统和环境被提取出。在关于某个特定计算环境的所有假设都集中在一个构件中是，系统是可移植的。
    </p>
    <p>
     如果移植到新的系统需要做些更改，则可移植性就是一种特殊的可修改性。
    </p>
    <blockquote>
     <p>
      todo:
      <br/>
      设计策略：接口实现分类、抽象、信息隐藏。
     </p>
    </blockquote>
    <p>
     <br/>
     <strong>
      6.功能性（functionality）
     </strong>
     <br/>
     是系统能完成所期望的工作的能力。
     <br/>
     <strong>
      7.可变性（changeability）
     </strong>
     <br/>
     是指架构经扩充或变更而成为新架构的能力。
     <br/>
     <strong>
      8.互操作性（inter-operation）
     </strong>
     <br/>
     程序和用其他编程语言编写的软件系统的交互。
     <br/>
    </p>
