    <h3>
     <a id="_3">
     </a>
     前言
    </h3>
    <p>
     在上一篇中，我们深入探讨了
     <code>
      @use
     </code>
     的使用， 也介绍了
     <code>
      @use
     </code>
     在使用深层模块中的变量时具有一定的缺点。所以在本文中，我们将深入解析
     <code>
      @forward
     </code>
     的核心用法。
    </p>
    <br/>
    <h3>
     <a id="1__8">
     </a>
     1. 转发导入
    </h3>
    <p>
     <code>
      @forward
     </code>
     本质是转发模块资源，是用于组织各文件中模块资源的方法。将导入的模块转发导出.
    </p>
    <p>
     例如:
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/20bde23e48754ceca6fdf6eb42877bcb.png">
      <br/>
      在这个示例中：
     </img>
    </p>
    <ul>
     <li>
      <code>
       bar.scss
      </code>
      使用
      <code>
       @forward "foo"
      </code>
      将
      <code>
       foo.scss
      </code>
      的成员转发出去。
     </li>
     <li>
      <code>
       main.scss
      </code>
      引入
      <code>
       bar
      </code>
      后，可以通过
      <code>
       bar
      </code>
      访问
      <code>
       foo
      </code>
      中的变量
      <code>
       （如 $foo-color）
      </code>
      。
     </li>
     <li>
      整个过程并未直接加载或编译 foo.scss，而是通过 bar 将 foo 的成员暴露给外部使用。
     </li>
    </ul>
    <p>
     <code>
      @forward
     </code>
     的核心作用是将一个模块的成员转发到另一个模块中，而不会直接加载或编译这些文件。
    </p>
    <p>
     但是在
     <code>
      bar.scss
     </code>
     中是不能直接访问
     <code>
      foo.scss
     </code>
     中的变量. 如果使用, 需要在
     <code>
      bar.scss
     </code>
     中使用
     <code>
      @use
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b19514a58c6a48c4b740e8a9f2fde72c.png">
      在
      <code>
       bar.scss
      </code>
      文件中
     </img>
    </p>
    <ul>
     <li>
      <code>
       @forward
      </code>
      的作用只是转发, 让导入
      <code>
       bar.scss
      </code>
      的文件可以通过
      <code>
       bar.$xx
      </code>
      访问
      <code>
       foo
      </code>
      中的变量
     </li>
     <li>
      <code>
       @use
      </code>
      的作用是导入
      <code>
       foo.scss
      </code>
      , 并通过
      <code>
       foo.$xx
      </code>
      访问
      <code>
       foo
      </code>
      中的变量
     </li>
    </ul>
    <p>
     在使用 Sass 的
     <code>
      @forward
     </code>
     和
     <code>
      @use
     </code>
     引入同一个模块时，需要注意以下两点：
    </p>
    <ol>
     <li>
      <code>
       模块加载机制
      </code>
      ：
      <code>
       @forward
      </code>
      和
      <code>
       @use
      </code>
      引入同一个模块时不会重复导入。
     </li>
     <li>
      <code>
       推荐顺序
      </code>
      ：
      <code>
       @forward
      </code>
      和
      <code>
       @use
      </code>
      先后顺序, 建议先写
      <code>
       @forward
      </code>
      再
      <code>
       @use
      </code>
      ：
     </li>
    </ol>
    <p>
     关于
     <code>
      @forward
     </code>
     和
     <code>
      @use
     </code>
     先后顺序先后顺序, sass文档给出了描述, 大概的意思是这样的:
    </p>
    <blockquote>
     <p>
      如果在同一个文件中同时使用 @forward 和 @use 引入同一个模块，推荐将 @forward 写在前面。这样做的好处是，如果使用者（可能是指其他开发者或文件）想要在引入模块时对转发的模块进行配置（例如通过 @use 的 with 语法），那么这些配置会优先应用到 @forward 转发的模块上，然后再由你的 @use 加载模块。这样可以确保模块的配置被正确处理，而不会因为顺序问题导致配置失效。
     </p>
    </blockquote>
    <br/>
    <h3>
     <a id="2__34">
     </a>
     2. 添加前缀
    </h3>
    <p>
     为了更好地理解
     <code>
      @forward
     </code>
     添加前缀的作用，我们可以先思考一个问题：
     <br/>
     如果一个文件中通过
     <code>
      @forward
     </code>
     转发了多个模块，而这些模块中存在同名变量会怎么样？
     <br/>
     答案是：这会导致冲突并
     <code>
      报错
     </code>
     。因为外部文件在引入时，是通过当前文件模块（即转发文件）访问被转发模块中的变量。如果多个被转发模块中有同名的变量，Sass 无法区分这些变量到底属于哪个模块，从而导致命名冲突。
    </p>
    <p>
     错误示例
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1c44563db08048c194d32732cb94d073.png">
      <br/>
      编译时会报如下错误信息
     </img>
    </p>
    <blockquote>
     <p>
      Error: Two forwarded modules both define a variable named $color.
      <br/>
      4 │ @forward “foo”;
      <br/>
      │ ━━━━━━━━━━━━━━ original @forward
      <br/>
      5 │ @forward “baz”;
      <br/>
      │ ^^^^^^^^^^^^^^ new @forward
      <br/>
      src\bar.scss 5:1 @use
      <br/>
      src\main.scss 2:1 root stylesheet
     </p>
    </blockquote>
    <p>
     此时就可以通过
     <code>
      @forward
     </code>
     的
     <code>
      as
     </code>
     关键字为转发的模块添加前缀，从而避免在引入模块时出现命名冲突。通过
     <code>
      as 前缀-*
     </code>
     的形式，你可以为转发的所有模块成员添加一个统一的前缀。当其他文件通过 @use 引入该模块时，成员名称会以 前缀- 开头，使得不同模块的同名成员能够清晰区分，增强了代码的可维护性和可读性。
    </p>
    <p>
     使用前缀示例:
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b816045110d64081b54514f0c6162815.png"/>
    </p>
    <br/>
    <h3>
     <a id="3__57">
     </a>
     3. 控制可见性
    </h3>
    <p>
     大多数情况下，使用
     <code>
      @forward
     </code>
     转发模块时，并不需要转发模块的全部内容，只需要暴露外部文件通过 @use 引入时所需的部分即可。为了满足这一需求，Sass 提供了
     <code>
      hide
     </code>
     和
     <code>
      show
     </code>
     两种可见性控制机制，帮助我们灵活地管理模块成员的对外访问权限。
    </p>
    <ul>
     <li>
      <code>
       hide
      </code>
      ：被 hide 的成员不会通过 @forward 转发，外部文件无法访问这些成员。它适用于隐藏模块中不需要公开的部分。
     </li>
     <li>
      <code>
       show
      </code>
      ：只有被 show 的成员会被转发，其余成员对外不可见。它适用于明确指定模块中可以公开的部分。
     </li>
    </ul>
    <p>
     通过 hide 和 show，我们可以精确控制模块的可见性，避免不必要的成员暴露，从而提升代码的封装性和安全性。
    </p>
    <p>
     示例：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1fdf7f16bb0c443dbe854794afb09390.png"/>
    </p>
    <p>
     如果使用转发未公开的变量, 就会发生如下错误:
    </p>
    <blockquote>
     <p>
      Error: Undefined variable.
      <br/>
      8 │ width: bar.$width; // 报错: 转发隐藏
      <br/>
      │ ^^^^^^^^^^
      <br/>
      src\main.scss 8:10 root stylesheet
     </p>
    </blockquote>
    <br/>
    <h3>
     <a id="4__75">
     </a>
     4. 转发时修改默认值
    </h3>
    <p>
     Sass 中，
     <code>
      @forward
     </code>
     还可以与
     <code>
      with
     </code>
     结合使用，实现在转发模块时覆盖其默认变量（使用
     <code>
      !default
     </code>
     标记的变量）的值。这种方式非常适合在定制被转发模块的行为或样式时使用。
    </p>
    <p>
     示例：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8eadb2aaa02b45948fb3004ee2e6571a.png">
      <br/>
      在
      <code>
       bar.scss
      </code>
      中，通过
      <code>
       @forward
      </code>
      转发
      <code>
       foo.scss
      </code>
      的变量时，可利用
      <code>
       with
      </code>
      修改其默认值。编译后，将优先使用转发时设置的新值，而非原始默认值。
     </img>
    </p>
    <p>
     如果你想要在使用
     <code>
      @use with
     </code>
     修改
     <code>
      @forward
     </code>
     转发的变量时，就需要在
     <code>
      @forward with
     </code>
     中为这些覆盖的变量添加
     <code>
      !default
     </code>
     标记，将覆盖的值也定义为默认值。这样一来，后续通过
     <code>
      @use with
     </code>
     修改时，就能轻松调整
     <code>
      @forward
     </code>
     中定义的变量值了。
     <br/>
     示例:
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/93b66d5ac7144b40b2fc0e2d27af372a.png"/>
    </p>
    <p>
     需要注意的是在使用
     <code>
      @use with
     </code>
     时，不能添加
     <code>
      !default
     </code>
     字段，因为
     <code>
      @use with
     </code>
     的目标是直接覆盖模块中的变量值，而不是将它们标记为默认值。
     <code>
      !default
     </code>
     仅适用于定义变量的初始值（如在模块内部或
     <code>
      @forward with
     </code>
     中）。
     <br/>
     <br/>
    </p>
    <h3>
     <a id="_92">
     </a>
     总结
    </h3>
    <p>
     在Sass中，
     <code>
      @forward
     </code>
     是一种强大的模块资源转发工具，能够将模块的成员转发到其他模块中，而不会直接加载或编译这些文件。通过本文的探讨，我们总结了以下几点：
    </p>
    <ul>
     <li>
      <p>
       模块转发：
       <code>
        @forward
       </code>
       可以将模块资源转发到其他模块中，但需要通过
       <code>
        @use
       </code>
       在转发文件中访问这些资源。
      </p>
     </li>
     <li>
      <p>
       添加前缀：通过
       <code>
        as
       </code>
       关键字可以为转发的模块添加前缀，避免命名冲突，增强代码的可读性和可维护性。
      </p>
     </li>
     <li>
      <p>
       可见性控制：利用
       <code>
        hide
       </code>
       和
       <code>
        show
       </code>
       可以精确控制模块成员的对外访问权限，提升代码的封装性和安全性。
      </p>
     </li>
     <li>
      <p>
       修改默认值：
       <code>
        @forward
       </code>
       结合
       <code>
        with
       </code>
       可以覆盖模块中的默认变量值，适合定制模块行为或样式。
      </p>
     </li>
    </ul>
    <p>
     掌握
     <code>
      @forward
     </code>
     的使用，能够帮助开发者更高效地组织和复用代码，构建更灵活的样式系统。
    </p>
