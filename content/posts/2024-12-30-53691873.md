    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     点击右上角齿轮，Add Remote，填写 SSH 相关的设置
     <br/>
     <img alt="" height="466" src="https://img-blog.csdn.net/20160822110307569?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center" style="border:none; max-width:100%" width="639"/>
     <br/>
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     做完后可能需要等一些时间。这个过程中 PyCharm 会将一些工具和调试 Server 发到你配置的 .pycharm_helper 目录中，并且对机器上已经安装的库类生成 skeletons，并传回 Windows 上进行保存。Python skeletons 是一些包括库类 API 定义的文件，用来给 PyCharm 做代码静态分析。JetBrains 的帮助文档提供了更详细的说明，参见 Important Note 一节。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     最后，你需要在解释器设置面板，选中 Remote Interpreter 做为项目的默认解释器。
    </p>
    <h3 id="配置部署信息" style="margin:0px; padding:0px; font-family:Arial">
     <a name="t8" style="color:rgb(255,153,0)" target="_blank">

     配置部署信息
    </h3>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     {这个就是上面的pycharm 通过 SFTP 远程编辑项目的配置，直接用就可以}
     <br/>
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     在 PyCharm 中打开部署的设置面板，路径为 Tool =&gt; Deployment =&gt; Configuration。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     Type 建议选择 SFTP，常见的 OpenSSH Server 提供了这种协议的支持。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     第二个 Tab (Mappings) 需要填写部署的路径，注意这个路径对应你项目的根目录，所以你很可能需要为它建一个空文件夹。勾选上 “Use this server as default”。除此之外还有个 Web path 的设置，应该是用来配置网页上查看文件的服务，可以无视。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     配置好部署面板后，建议把自动同步开启 (Tool =&gt; Deployment =&gt; Automatics Upload(always))，这样在 PyCharm 中对文件进行修改，可以马上同步到 Linux 上。同时建议先做一次同步。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     同步可以通过很多方式完成，比如在 Project 面板 (Alt+1) 上对项目根目录右键 “Upload to”，也可以在菜单中上传 (Tool =&gt; Deployment =&gt; Upload to)。如果菜单中 Upload to 不可点击，请尝试打开项目的一个 py 文件，并把焦点放在编辑器中。
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">

     配置运行配置
    </h3>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     {如果不使用django，这个可以不用配置}

     在 PyCharm 中打开运行配置面板，路径为 Run =&gt; Run/Debug Configurations:
     <br/>
     <a class="fancybox" href="http://mingmings.qiniudn.com/PyCharm-Run-Configuration.png" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none" target="_blank">
      <img alt="PyCharm-Run-Configuration" height="494" src="http://mingmings.qiniudn.com/PyCharm-Run-Configuration.png" style="border:none; max-width:100%" width="678"/>

    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     运行配置的设置有两点需要注意。Python Interpreter 需要选中之前建立的远程解释器。Path mappings 处，需要把本机的 manage.py 与 Linux 上的关联起来，比如：
     <br/>

    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">

    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     这可能是 PyCharm 的 bug：在使用远程解释器后，PyCharm 并不会自动将 Django 运行配置中的 manage.py 文件定位到 Linux 上的那份，而是会尝试使用下面的命令来启动 Django 调试 Server:
     <br/>
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     /usr/bin/python2 manage.py C:/Evolution/Python/django_website/manage.py
    </p>
    <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px">
     自然就找不到文件了。所以使用了 Path mappings 作为一种 hack 手段来解决。
     <br/>
     这时就可以打断点调试啦

     <h2 class="py2" style='margin:0.414em 0px 0.5em; font-size:3.5rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif; padding-top:1rem; padding-bottom:2rem'>
      PyCharm远程调试
     </h2>
     <span class="post-meta" style="width:auto; font-size:1.25rem; margin:0px; padding:0px; color:rgb(122,122,122); font-style:italic">
      2016年3月30日
     </span>
    </div>
    <p>
    </p>
    <p class="post-meta" style='margin-top:0px; margin-bottom:0px; font-size:1.25rem; line-height:1.8; color:rgb(122,122,122); width:auto; padding-top:0px; padding-bottom:0px; font-style:italic; font-family:Ubuntu,"PT Serif",Georgia,"Times New Roman",serif; background-color:rgb(255,250,240)'>
     Tags:
     <a href="http://dasheyuan.com/tags/openstack" rel="nofollow noopener noreferrer" style="color:rgb(132,81,6); text-decoration:none" target="_blank">
      openstack
     </a>
    </p>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     PyCharm提供两种远程调试(Remote Debugging)的方式:
    </p>
    <ul style="margin-top:0px; margin-bottom:1rem; font-size:1.125rem; line-height:1.5rem">
     <li style="font-size:1.25rem; line-height:1.8">
      配置远程的解释器（remote interpreter）
     </li>
     <li style="font-size:1.25rem; line-height:1.8">
      配置Python调试服务器（Python Debug Server）
     </li>
    </ul>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     本文主要介绍Python Debug Server配置与使用。
    </p>
    <h3 id="python-debug-server参考模型" style='margin:0.414em 0px 0.5em; font-size:3rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif'>
     Python Debug Server参考模型
    </h3>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     <img alt="PyCharm remote debug" src="http://dasheyuan.com/images/PyCharm%20remote%20debug.png" style="width:auto; max-width:100%; border-color:rgb(204,204,204); border-style:solid"/>
     <br/>
     （上图为Python Debug Server模型，仅供参考）
    </p>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     如上图所示，PyCharm远程调试方式的模型其实就是一个典型的Server/Client模型。模型左侧，Server服务运行在本地主机PyCharm中。配置Debug Server完成后就可以启动这个服务器。服务器启动后，就是等待Client客户端的接入。模型右侧，需要调试的Python程序通过调用pycharm-debug提供的
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      pydevd
     </code>
     库，然后连接到服务端。这样，本地主机就可以使用PyCharm调试远程主机运行的Python程序了，调试的方法与直接调试本地Python程序一样。
    </p>
    <h3 id="python-debug-server配置使用" style='margin:0.414em 0px 0.5em; font-size:3rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif'>
     Python Debug Server配置使用
    </h3>
    <h4 id="1-服务端配置" style='margin:0.414em 0px 0.5em; font-size:2.249rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif'>
     1.服务端配置
    </h4>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     首先在Run/Debug Configuration中添加Python Remote Debug服务。
     <img alt="server configurate" src="http://dasheyuan.com/images/PyCharm%20remote%20debug-2.jpg" style="width:auto; max-width:100%; border-color:rgb(204,204,204); border-style:solid"/>
     <br/>
     （上图为Python Debug Server配置流程）
     <br/>
     <strong>
      注：
     </strong>
     服务器命名和IP地址是必须配置的。若端口没有配置，则在服务器启动时随机分配；若路径映射没有配置，则在客户端连接服务端成功后PyCharm再提示选择配置。
    </p>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     服务器
     <strong>
      启动
     </strong>
     后会提示以下信息：
    </p>
    <pre style='margin-top:0px; margin-bottom:0.88em; font-family:Menlo,Monaco,"Courier New",monospace; padding:1.125em; font-size:1rem; line-height:1.11'><code class="hljs sql" style='font-family:Menlo,Monaco,"Courier New",monospace; display:block; overflow-x:auto; padding:0.5em; color:rgb(68,68,68); border-style:solid'>Starting debug server at port 51234    
<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">Use</span> the <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">following</span> code <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">to</span> <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">connect</span> <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">to</span> the debugger:    
<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">import</span> pydevd    
pydevd.settrace(<span class="hljs-string" style="color:rgb(136,0,0)">'192.168.27.1'</span>, port=<span class="hljs-number" style="color:rgb(136,0,0)">51234</span>, stdoutToServer=<span class="hljs-literal" style="color:rgb(31,129,31)">True</span>, stderrToServer=<span class="hljs-literal" style="color:rgb(31,129,31)">True</span>)    
Waiting <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">for</span> process <span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">connection</span>...
</code></pre>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     其中，这两行代码需要嵌入到远程调试的Python代码中，这两行代码包含了服务器监听的IP地址和端口等信息。
    </p>
    <pre style='margin-top:0px; margin-bottom:0.88em; font-family:Menlo,Monaco,"Courier New",monospace; padding:1.125em; font-size:1rem; line-height:1.11'><code class="hljs python" style='font-family:Menlo,Monaco,"Courier New",monospace; display:block; overflow-x:auto; padding:0.5em; color:rgb(68,68,68); border-style:solid'><span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">import</span> pydevd

</code></pre>
    <h4 id="2-客户端配置" style='margin:0.414em 0px 0.5em; font-size:2.249rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif'>
     2.客户端配置
    </h4>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     参考模型中提到，远程主机需要配置
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      pydev
     </code>
     库。而提供库的源文件在PyCharm安装路径下的
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      debug-eggs
     </code>
     目录中，其中
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      pycharm-debug.egg
     </code>
     对应的是版本2，
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      pycharm-debug-py3k.egg
     </code>
     对应的是版本3。 把
     <code style='font-family:Menlo,Monaco,"Courier New",monospace; background-color:rgb(218,209,191)'>
      pycharm-debug.egg
     </code>
     文件复制到
     <strong>
      远程主机
     </strong>
     ，然后通过easy_install命令安装。 PS：复制的方法有多种，通过ssh可以传输文件，注意权限问题。
    </p>
    <pre style='margin-top:0px; margin-bottom:0.88em; font-family:Menlo,Monaco,"Courier New",monospace; padding:1.125em; font-size:1rem; line-height:1.11'><code class="hljs ruby" style='font-family:Menlo,Monaco,"Courier New",monospace; display:block; overflow-x:auto; padding:0.5em; color:rgb(68,68,68); border-style:solid'>$ sudo easy_install pycharm-debug.egg
</code></pre>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     然后把上面提及的两行代码嵌入到需要调试的Python程序中。
    </p>
    <pre style='margin-top:0px; margin-bottom:0.88em; font-family:Menlo,Monaco,"Courier New",monospace; padding:1.125em; font-size:1rem; line-height:1.11'><code class="hljs python" style='font-family:Menlo,Monaco,"Courier New",monospace; display:block; overflow-x:auto; padding:0.5em; color:rgb(68,68,68); border-style:solid'><span class="hljs-comment" style="color:rgb(136,136,136); font-style:italic">#demo.py</span>
<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">print</span> <span class="hljs-string" style="color:rgb(136,0,0)">"remote debug"</span>
<span class="hljs-comment" style="color:rgb(136,136,136); font-style:italic">#嵌入服务器信息的代码，进入调试状态</span>
<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">import</span> pydevd
pydevd.settrace(<span class="hljs-string" style="color:rgb(136,0,0)">'192.168.27.1'</span>, port=<span class="hljs-number" style="color:rgb(136,0,0)">51234</span>, stdoutToServer=<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">True</span>, stderrToServer=<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">True</span>)

<span class="hljs-keyword" style="color:rgb(194,98,48); font-weight:bold">print</span> <span class="hljs-string" style="color:rgb(136,0,0)">"code need to debug"</span>

<span class="hljs-comment" style="color:rgb(136,136,136); font-style:italic">#离开调试模式</span>
pydevd.stoptrace()
</code></pre>
    <h4 id="3-调试方式" style='margin:0.414em 0px 0.5em; font-size:2.249rem; line-height:1.2; font-family:Lato,"Helvetica Neue",Helvetica,sans-serif'>
     3.调试方式
    </h4>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     调试方式比较简单，首先启动远程主机的程序，进入调试状态。然后返回本地主机的PyCharm界面进行调试。
    </p>
    <pre style='margin-top:0px; margin-bottom:0.88em; font-family:Menlo,Monaco,"Courier New",monospace; padding:1.125em; font-size:1rem; line-height:1.11'><code class="hljs ruby" style='font-family:Menlo,Monaco,"Courier New",monospace; display:block; overflow-x:auto; padding:0.5em; color:rgb(68,68,68); border-style:solid'>$ python demo.py 
remote debug

</code></pre>
    <p style="margin-top:0px; margin-bottom:1rem; font-size:1.25rem; line-height:1.8">
     <img alt="debug output" src="http://dasheyuan.com/images/PyCharm%20remote%20debug-3.jpg" style="width:auto; max-width:100%; border-color:rgb(204,204,204); border-style:solid"/>
     <br/>
     （上图为本地主机PyCharm调试界面说明）
     <br/>
     此步骤之前需要配置路径映射关系，即远程主机中的代码如何映射到本地主机中，本文用到的是win-sshfs工具，工具的使用方式就有时间再介绍。
    </p>
