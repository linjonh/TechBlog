    <h3>
     <a id="npm_install__stack_Error_Cant_find_Python_executable_python_0">
     </a>
     npm install 报错 ：stack Error: Can’t find Python executable “python”
    </h3>
    <p>
     小编最近在使用npm install的命令的时候，出现了找不到python的错误，小编整理了一下解决方案，希望可以帮助遇到此问题的小伙伴。
    </p>
    <h6>
     <a id="window_3">
     </a>
     此处说的是window版本的。
    </h6>
    <ol>
     <li>
      下载python的环境：
      <br/>
      下载地址：
      <a href="https://www.python.org/downloads/release/python-271/" rel="nofollow">
       https://www.python.org/downloads/release/python-271/
      </a>
      此处我使用的2.7版本的python，如下图所示：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e63d1098c6d2446479e760d8833dbd4d.png">
       <br/>
       看到这个界面后，一直往下拖动，拖动到最后，就可以看到如下图所示的内容：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d8611e8e3aab2d069776f976d46a0c18.png">
        <br/>
        按照你自己电脑的位数下载响应的版本即可。
       </img>
      </img>
     </li>
     <li>
      下载完成后，对其安装，默认都进行下一步就可以，注意，安装在没有中文没有空格的文件夹下面。
     </li>
     <li>
      安装完成后，配置环境变量。
      <br/>
      配置方法：右击我的电脑–&gt;属性–&gt;高级系统设置–&gt;环境变量–&gt;创建一个叫PYTHON_HOME的系统变量，如下图所示：变量值指向你安装python所在的文件夹。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5b9afdb2b51b034007a2ce520376bc51.png"/>
     </li>
     <li>
      配置玩PYTHON_HOME以后，需要在path环境变量中引入该值，引入方法如下：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5647684b610fccfcbf56361aa9c6bb41.png">
       <br/>
       点击Path后，点击编辑，然后在变量值的地方创建两个%%，将PYTHON_HOME填入两个%%中，注意结束时候的“；”。
      </img>
     </li>
     <li>
      配置完成后，在npm中配置python启动文件的位置，配置命令如下所示：
      <br/>
      npm config set python “D:\JavaStuding\python\python.exe”。
      <strong>
       路径注意修改成自己的路径，不要复制我的。
      </strong>
     </li>
    </ol>
    <p>
     设置完毕所有操作后，在进行npm install的操作，发现就可以执行成功，不会报错。
    </p>
