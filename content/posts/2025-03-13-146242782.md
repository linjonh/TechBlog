    <h4>
     <a id="_2">
     </a>
     零、基本概念
    </h4>
    <h5>
     <a id="_3">
     </a>
     什么是信号槽?
    </h5>
    <p>
     信号槽类似于软件设计模式中的观察者模式，（观察者模式是一种对象行为模式。它定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。）被观察者发出的信号（signal），观察者收到自己注册监听signal，就通过槽（slot）关联的槽函数function实现动作操作。
    </p>
    <h5>
     <a id="_10">
     </a>
     信号槽的优缺点
    </h5>
    <p>
     （1）类型安全
    </p>
    <p>
     信号的参数类型、参数个数需要和槽函数的参数类型和参数个数一致。槽函数的个数可以少于信号的参数个数，但缺少的参数必须是信号参数的最后一个或最后几个。
    </p>
    <p>
     （2）松散耦合
    </p>
    <p>
     信号发送者不需要知道发出的信号被哪个对象的槽函数接收，槽函数也不需要知道哪些信号关联了自己，Qt的信号槽机制保证了信号与槽函数的调用。支持信号槽机制的类或者父类必须继承于QObject。
    </p>
    <p>
     （3）效率
    </p>
    <p>
     信号槽增强了对象间通信的灵活性，同时损失了一些性能，通过信号调用的槽函数比直接调用速度慢约10倍（因为需要定位信号接收者；遍历所有关联；编组/解组传递的参数；多线程时，信号可能需要排队），这种调用速度对性能要求不是非常高的场景是可以忽略的，是满足绝大部分场景。
    </p>
    <h5>
     <a id="_23">
     </a>
     信号槽的用法和性质
    </h5>
    <pre><code class="prism language-plain">connect(sender, signal, receiver, slot);
</code></pre>
    <p>
     sender：发出信号的对象
     <br/>
     signal：发送对象发出的信号
     <br/>
     receiver：接收信号的对象
     <br/>
     slot：接收对象在接收到信号之后所需要调用的函数（槽函数）
    </p>
    <p>
     系统自带的信号：
    </p>
    <pre><code class="prism language-plain">void clicked(bool checked = false)
void pressed()
void released()
void toggled(bool checked )
 signals inherited from Qwidget
 signals inherited from Q0bject
</code></pre>
    <ol>
     <li>
      一个信号可以关联多个槽函数。
     </li>
     <li>
      多个信号可以关联一个槽函数
     </li>
     <li>
      一个信号可以关联另一个信号
     </li>
     <li>
      一个信号关联多个信号
     </li>
    </ol>
    <h4>
     <a id="Qt_53">
     </a>
     <strong>
      一、Qt核心机制
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       信号与槽（Signals &amp; Slots）
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：Qt的信号槽机制与回调函数有何区别？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          <strong>
           解耦性
          </strong>
          ：信号槽通过元对象系统（Meta-Object System）连接，发送者无需知道接收者。
         </li>
         <li>
          <strong>
           参数类型安全
          </strong>
          ：编译时检查参数类型（需用
          <code>
           Q_DECLARE_METATYPE
          </code>
          注册自定义类型）。
         </li>
         <li>
          <strong>
           多对多通信
          </strong>
          ：一个信号可连接多个槽，反之亦然。
         </li>
         <li>
          <strong>
           线程安全
          </strong>
          ：支持跨线程通信（自动选择
          <code>
           Qt::AutoConnection
          </code>
          模式）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       元对象系统（Meta-Object System）
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：
        <code>
         moc
        </code>
        （元对象编译器）的作用是什么？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          处理
          <code>
           Q_OBJECT
          </code>
          宏，生成
          <code>
           moc_*.cpp
          </code>
          文件，实现信号槽、动态属性、运行时类型信息（RTTI）等功能。
         </li>
         <li>
          支持
          <code>
           qobject_cast
          </code>
          进行安全的类型转换。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="_69">
     </a>
     <strong>
      二、内存管理与对象模型
     </strong>
    </h4>
    <ol start="3">
     <li>
      <strong>
       父子对象机制
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：Qt如何管理对象的生命周期？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          通过
          <code>
           QObject
          </code>

         </li>
         <li>
          示例：
          <code>
           QWidget* child = new QWidget(parent);
          </code>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       智能指针
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：如何在Qt中使用智能指针？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          使用
          <code>
           QSharedPointer
          </code>
          、
          <code>
           QWeakPointer
          </code>
          管理动态对象。
         </li>
         <li>
          注意：
          <code>
           QObject
          </code>
          及其子类通常依赖父子关系管理，需谨慎混合使用。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>

     <strong>
      三、多线程与并发
     </strong>
    </h4>
    <ol start="5">
     <li>
      <strong>
       QThread的两种用法
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：继承
        <code>
         QThread
        </code>
        与使用
        <code>
         moveToThread
        </code>
        有何区别？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          <strong>
           子类化QThread
          </strong>
          ：重写
          <code>
           run()
          </code>
          方法，适用于需要控制线程执行流程的场景。
         </li>
         <li>
          <strong>
           moveToThread
          </strong>
          ：将对象移动到新线程，通过信号槽触发逻辑（更符合事件驱动模型）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       线程同步
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：Qt中如何实现线程同步？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          使用
          <code>
           QMutex
          </code>
          、
          <code>
           QReadWriteLock
          </code>
          、
          <code>
           QSemaphore
          </code>
          。
         </li>
         <li>
          跨线程通信优先使用信号槽（自动排队或阻塞）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>

    </h4>
    <ol start="7">

      <strong>
       事件循环（Event Loop）
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：解释Qt的事件处理流程。
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>

          <code>
           QApplication
          </code>
          进入事件队列。
         </li>
         <li>
          <code>
           QEventLoop
          </code>
          逐一分发事件（如鼠标、键盘、定时器事件）。
         </li>
         <li>
          可重写
          <code>
           event(QEvent*)
          </code>
          或特定事件处理器（如
          <code>
           mousePressEvent()
          </code>
          ）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       绘图机制
      </strong>
      <ul>
       <li>
        <strong>
         问
        </strong>
        ：如何使用QPainter进行自定义绘图？
       </li>
       <li>
        <strong>
         答
        </strong>
        ：
        <ul>
         <li>
          在
          <code>
           paintEvent()
          </code>
          中使用
          <code>
           QPainter
          </code>
          绘制图形。
         </li>
         <li>

          <code>
           QPixmap
          </code>
          作为临时画布避免闪烁。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
