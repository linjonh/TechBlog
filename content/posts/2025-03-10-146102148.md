    <h2 id="%E4%B8%80%E3%80%81%E4%BB%80%E4%B9%88%E6%98%AF%E9%93%BE%E8%A1%A8%EF%BC%9F" name="%E4%B8%80%E3%80%81%E4%BB%80%E4%B9%88%E6%98%AF%E9%93%BE%E8%A1%A8%EF%BC%9F">
     一、什么是链表？
    </h2>
    <p>
     <span style="color:#0d0016">
      链表是一种物理存储上非连续，数据元素的逻辑顺序通过链表中的指针链接次序，实现的一种线性存储结构。链表由一系列节点（链表中每一个元素称为节点）组成，节点在运行时动态生成 （malloc），每个节点包括两个部分：
     </span>
    </p>
    <ul>
     <li>
      一个是存储数据元素的
      <strong>
       数据域
      </strong>
     </li>
     <li>
      另一个是存储下一个节点地址的
      <strong>
       指针域
      </strong>
     </li>
    </ul>
    <h2 id="%E4%BA%8C%E3%80%81%E4%B8%BA%E4%BB%80%E4%B9%88%E8%A6%81%E4%BD%BF%E7%94%A8%E9%93%BE%E8%A1%A8%EF%BC%9F" name="%E4%BA%8C%E3%80%81%E4%B8%BA%E4%BB%80%E4%B9%88%E8%A6%81%E4%BD%BF%E7%94%A8%E9%93%BE%E8%A1%A8%EF%BC%9F">
     二、为什么要使用链表？
    </h2>
    <p>
     使用链表用于
     <span style="color:#4da8ee">
      解决动态数量的数据存储
     </span>
     。比如说，我们要管理一
     <a href="https://zhida.zhihu.com/search?content_id=380673932&amp;content_type=Answer&amp;match_order=1&amp;q=%E5%A0%86&amp;zhida_source=entity" rel="nofollow" title="堆">
      堆
     </a>
     票据，可能有一张，但也可能有一亿张。怎么办呢？申请一个几十G的大数组存储着吗？万一用户只有一百张票据要处理呢？内存较小拒绝运行？可申请少了又不够用啊……
    </p>
    <p>
     而且，用数组的话，删除然后添加票据，是每次删除让后面五百万张往前移一格呢、还是每次添加从头搜索空闲格子？如何区分空闲格子和值为0的数据？要进行区分的话是多占用空间呢还是占用数据值域？占用了值域会不会使得数据处理变得格外复杂？会不会一不小心就和正常数据混淆？万一拿来区分的字段在某个版本后废弃不用、或者扩充值域了呢？
    </p>
    <p>
     那么，链表这种东西就是个很有效的数据结构，可以很有效的管理这类不定量的数据。
    </p>
    <h2 id="%E4%B8%89%E3%80%81%20%E5%8D%95%E9%93%BE%E8%A1%A8%E4%BB%8B%E7%BB%8D%E4%B8%8E%E4%BD%BF%E7%94%A8" name="%E4%B8%89%E3%80%81%20%E5%8D%95%E9%93%BE%E8%A1%A8%E4%BB%8B%E7%BB%8D%E4%B8%8E%E4%BD%BF%E7%94%A8">
     三、 单链表介绍与使用
    </h2>
    <h3 id="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%203.1%20%E5%8D%95%E9%93%BE%E8%A1%A8" name="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%203.1%20%E5%8D%95%E9%93%BE%E8%A1%A8">
     3.1 单链表
    </h3>
    <p>
     单链表的每个节点除了存放数据元素外，还要存储指向下一个节点的指针。与顺序表进行对比：
    </p>
    <table border="1" cellpadding="1" cellspacing="1" style="width:500px">
     <tbody>
      <tr>
       <td>
       </td>
       <td>
        优点
       </td>
       <td>
        缺点
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         顺序表
        </strong>
       </td>
       <td>
        可随机存储，存储密度较高
       </td>
       <td>
        要求大片连续空间，改变容量不方便
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         单链表
        </strong>
       </td>
       <td>
        不要求大片连续空间，改变容量方便
       </td>
       <td>
        不可随机存取，要耗费一定空间存放指针
       </td>
      </tr>
     </tbody>
    </table>

    <h4 id="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A03.1.1%20%E5%88%9B%E5%BB%BA%E5%8D%95%E9%93%BE%E8%A1%A8%E8%8A%82%E7%82%B9" name="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A03.1.1%20%E5%88%9B%E5%BB%BA%E5%8D%95%E9%93%BE%E8%A1%A8%E8%8A%82%E7%82%B9">
     3.1.1 创建单链表节点
    </h4>
    <pre><code class="hljs">typedef int SLTDataType;
//链表是由节点构成
//逻辑结构：一定连续；物理结构：不一定连续
//不会造成空间的浪费，由独立的节点构成
typedef struct SListNode {  //SList : single linked list 单链表
	SLTDataType data;  //  一个是存储数据元素的数据域
	struct SListNode* next;  //   另一个是存储下一个节点地址的指针域
}SLTNode;</code></pre>
    <p>
     <img alt="" height="568" src="https://i-blog.csdnimg.cn/direct/bc04b1522b284ceaa7563d03bc8516f8.png" width="1449"/>
    </p>
    <p>
     上面的结构体仅是单链表节点的定义，要创建一新的节点需要对里面的数据进行初始化：插入数值，节点指向的下一个节点为空
    </p>
    <pre><code class="hljs">//创建一个节点
struct SListNode* SLTBuyNode(SLTDataType x) {
	SLTNode* newnode = (struct SListNode*)malloc(sizeof(struct SListNode));
	newnode-&gt;data = x;
	newnode-&gt;next = NULL;
	return newnode;
}</code></pre>
    <p>
     <img alt="" height="451" src="https://i-blog.csdnimg.cn/direct/fb2ed4ffd09e40988226e0cd44a2d60f.png" width="1008"/>

    <h4 id="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A03.1.2%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E6%8F%92%E3%80%81%E5%B0%BE%E6%8F%92" name="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A03.1.2%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E6%8F%92%E3%80%81%E5%B0%BE%E6%8F%92">
     <a id="pandas_16">
     </a>
     3.1.2 单链表的头插、尾插
     <a id="2_34">
     </a>
    </h4>
    <h5 id="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%B0%BE%E6%8F%92" name="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%B0%BE%E6%8F%92">
     单链表的尾插
    </h5>
    <p>
     <img alt="" height="597" src="https://i-blog.csdnimg.cn/direct/d816730ca923467bb0a4891989809002.png" width="1716"/>
    </p>
    <p>
     这里传递的链表节点的参数，为什么是二级指针呢？
    </p>
    <p>
     在初始化过程中，需要
     <span style="color:#38d8f0">
      修改头指针
     </span>
     ，因此要用到二级指针传递头指针的地址，这样才能修改头指针。这与普通变量类似，当需要修改普通变量的值，需传递其地址。使用二级指针，很方便就修改了传入的结点一级指针的值。 如果用一级指针，则只能通过指针修改指针所指内容，却无法修改指针的值，也就是指针所指的内存块。
    </p>
    <pre><code class="hljs">//链表的尾插
void SLTPushBack(SLTNode** pphead, SLTDataType x) { //一级指针传递的为结构体变量地址的值，二级指针接收的是指向结构体变量的指针的地址
	assert(pphead);
	SLTNode* newnode = SLTBuyNode(x);
	//链表为空，新节点作为pphead
	if (*pphead == NULL) {
		*pphead = newnode;
		return;
	}
	//链表不为空，链表的尾指向新节点
	SLTNode* ptail = *pphead;
	while (ptail-&gt;next) {
		ptail = ptail-&gt;next;
	}
	ptail-&gt;next = newnode;
}</code></pre>
    <h5 id="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E6%8F%92" name="%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%20%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E6%8F%92">
     单链表的头插
    </h5>
    <p>
     <img alt="" height="492" src="https://i-blog.csdnimg.cn/direct/4cceb7a7453c41d3b4b80056d289dacd.png" width="1542"/>
    </p>
    <pre><code class="hljs">void SLTPushFront(SLTNode** pphead, SLTDataType x) {
	assert(pphead);
	//链表为空，新节点指向pphead，pphead再指向新节点；链表不为空，新节点指向pphead，pphead再指向新节点
	SLTNode* newnode = SLTBuyNode(x);
	newnode-&gt;next = *pphead;
	*pphead = newnode;
}
</code></pre>
    <p>
     插入完数据后，想要打印以一下链表的数据，通过从头开始一个一个节点地访问数据，并打印，便实现了链表数据的打印。然后看看插入的情况；
    </p>
    <pre><code class="hljs">//打印链表
void SLTPrint(SLTNode* phead) {
	SLTNode* pcur = phead;
	while (pcur) {
		printf("%d-&gt;", pcur-&gt;data);
		pcur = pcur-&gt;next;
	}
	printf("NULL\n");
}</code></pre>
    <p>
     尾插部分数据，观察打印的结果是否符合预期结果；发现运行结果与预期相符，所以链表的插入操作是没什么大问题的
    </p>
    <p>
     <img alt="" height="411" src="https://i-blog.csdnimg.cn/direct/953e5bd1183a4c13982295d4cf56c9bb.png" width="1305"/>
    </p>
    <h4 id="3.1.3%C2%A0%20%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E5%88%A0%E3%80%81%E5%B0%BE%E5%88%A0" name="3.1.3%C2%A0%20%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E5%88%A0%E3%80%81%E5%B0%BE%E5%88%A0">
     3.1.3  单链表的头删、尾删
    </h4>
    <p>
     如果链表为空，不需要删除；如果删除的是第一个结点，则需要将保存链表首地址的指针保存第一个结点的下一个结点的 地址 如果删除的是中间结点，则找到中间结点的前一个结点，让前一个结点的指针域保存这个结 点的后一个结点的地址即可
    </p>
    <h5 id="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%B0%BE%E5%88%A0%E2%80%8B%E7%BC%96%E8%BE%91" name="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%B0%BE%E5%88%A0%E2%80%8B%E7%BC%96%E8%BE%91">
     单链表的尾删
     <img alt="" height="564" src="https://i-blog.csdnimg.cn/direct/024bce4efd334b1b8c40a4da208fad4e.png" width="1028"/>
    </h5>
    <pre><code class="hljs">void SLTPopBack(SLTNode** pphead) {
	assert(pphead);
	//链表为空，不删除
	assert(*pphead); //指向第一个节点的地址
	//链表不为空，只有一个节点
	if ((*pphead)-&gt;next = NULL) {
		free(*pphead);
		*pphead = NULL;
		return;
	}
	//多个节点，释放最后一个节点，其前驱节点指向空
	SLTNode* ptail = *pphead;
	SLTNode* prev = NULL;
	while (ptail-&gt;next) {
		prev = ptail;
		ptail = ptail-&gt;next;
	}
	prev-&gt;next = NULL;
	free(ptail);
	ptail = NULL;
}</code></pre>
    <h5 id="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E5%88%A0" name="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%8D%95%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%B4%E5%88%A0">
     单链表的头删
    </h5>
    <p>
     <img alt="" height="636" src="https://i-blog.csdnimg.cn/direct/e7b76fe84c1d4391afc7e848e0aece77.png" width="991"/>
    </p>
    <pre><code class="hljs">void SLTPopFront(SLTNode** pphead) {
	assert(pphead);
	//链表为空，不删除
	assert(*pphead);
	//链表不为空，释放最后一个节点，其前驱节点指向空
	SLTNode* pcur = *pphead;
	*pphead = pcur-&gt;next;
	pcur-&gt;next = NULL;
	free(pcur);
	pcur = NULL;
}</code></pre>
    <h4 id="%C2%A03.1.4%C2%A0%E9%93%BE%E8%A1%A8%E8%8A%82%E7%82%B9%E7%9A%84%E6%9F%A5%E6%89%BE" name="%C2%A03.1.4%C2%A0%E9%93%BE%E8%A1%A8%E8%8A%82%E7%82%B9%E7%9A%84%E6%9F%A5%E6%89%BE">
     3.1.4 链表节点的查找
    </h4>
    <p>
     先对比第一个结点的数据域是否是想要的数据，如果是就直接返回，如果不是则继续查找下 一个结点，如果到达最后一个结点的时候都没有匹配的数据，说明要查找数据不存在
    </p>
    <pre><code class="hljs">//查找
SLTNode* STLFind(SLTNode** pphead, SLTDataType x) {
	assert(pphead);
	SLTNode* pcur = *pphead;
	while (pcur) {
		if (pcur-&gt;data == x) {
			return pcur;
		}
		pcur = pcur-&gt;next;
	}
	return NULL;
}</code></pre>
    <h4 id="3.1.5%C2%A0%E5%9C%A8%E6%8C%87%E5%AE%9A%E4%BD%8D%E7%BD%AE%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE" name="3.1.5%C2%A0%E5%9C%A8%E6%8C%87%E5%AE%9A%E4%BD%8D%E7%BD%AE%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE">
     3.1.5 在指定位置插入数据
    </h4>
    <h5 id="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%9C%A8%E6%8C%87%E5%AE%9A%E4%BD%8D%E7%BD%AE%E5%89%8D%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE" name="%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%E5%9C%A8%E6%8C%87%E5%AE%9A%E4%BD%8D%E7%BD%AE%E5%89%8D%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE">
     在指定位置前插入数据
    </h5>
    <pre><code class="hljs">//在指定位置前插入数据
void SLTInsert(SLTNode** pphead, SLTNode* pos, SLTDataType x) {
	assert(pphead);
	assert(pos);
	//头节点不能为空
	assert(*pphead);
	SLTNode* newnode = SLTBuyNode(x);
	//当pos节点指向 头节点时,进行头插
	if (*pphead == pos) {
		SLTPushFront(pphead, x);
		return;
	}
	//pos pphead不指向同一节点
	SLTNode* prev = *pphead;
	while (prev-&gt;next != pos) { //找到pos节点的前驱
		prev = prev-&gt;next;

//创建一个节点
struct SListNode* SLTBuyNode(SLTDataType x) {
	SLTNode* newnode = (struct SListNode*)malloc(sizeof(struct SListNode));
	newnode-&gt;data = x;
	newnode-&gt;next = NULL;
	return newnode;
}


//打印链表
void SLTPrint(SLTNode* phead) {
	SLTNode* pcur = phead;
	while (pcur) {
		printf("%d-&gt;", pcur-&gt;data);
		pcur = pcur-&gt;next;
	}
	printf("NULL\n");
}

//链表的头插、尾插
void SLTPushBack(SLTNode** pphead, SLTDataType x) { //一级指针传递的为结构体变量地址的值，二级指针接收的是指向结构体变量的指针的地址
	assert(pphead);
	SLTNode* newnode = SLTBuyNode(x);
	//链表为空，新节点作为pphead
	if (*pphead == NULL) {
		*pphead = newnode;
		return;
	}
	//链表不为空，链表的尾指向新节点
	SLTNode* ptail = *pphead;
	while (ptail-&gt;next) {
		ptail = ptail-&gt;next;
	}
	ptail-&gt;next = newnode;
}

void SLTPushFront(SLTNode** pphead, SLTDataType x) {
	assert(pphead);
	//链表为空，新节点指向pphead，pphead再指向新节点；链表不为空，新节点指向pphead，pphead再指向新节点
	SLTNode* newnode = SLTBuyNode(x);
	newnode-&gt;next = *pphead;
	*pphead = newnode;
}


//链表的头删、尾删
void SLTPopBack(SLTNode** pphead) {
	assert(pphead);
	//链表为空，不删除
	assert(*pphead); //指向第一个节点的地址
	//链表不为空，只有一个节点
	if ((*pphead)-&gt;next = NULL) {
		free(*pphead);
		*pphead = NULL;
		return;
	}
	//多个节点，释放最后一个节点，其前驱节点指向空
	SLTNode* ptail = *pphead;
	SLTNode* prev = NULL;
	while (ptail-&gt;next) {
		prev = ptail;
		ptail = ptail-&gt;next;
	}
	prev-&gt;next = NULL;
	free(ptail);
	ptail = NULL;
}
void SLTPopFront(SLTNode** pphead) {
	assert(pphead);
	//链表为空，不删除
	assert(*pphead);
	//链表不为空，释放最后一个节点，其前驱节点指向空
	SLTNode* pcur = *pphead;
	*pphead = pcur-&gt;next;
	pcur-&gt;next = NULL;
	free(pcur);
	pcur = NULL;
}

//查找
SLTNode* STLFind(SLTNode** pphead, SLTDataType x) {
	assert(pphead);
	SLTNode* pcur = *pphead;
	while (pcur) {
		if (pcur-&gt;data == x) {
			return pcur;
		}
		pcur = pcur-&gt;next;
	}
	return NULL;
}

//在指定位置前插入数据
void SLTInsert(SLTNode** pphead, SLTNode* pos, SLTDataType x) {
	assert(pphead);
	assert(pos);
	//头节点不能为空
	assert(*pphead);
	SLTNode* newnode = SLTBuyNode(x);
	//当pos节点指向 头节点时,进行头插
	if (*pphead == pos) {
		SLTPushFront(pphead, x);
		return;
	}
	//pos pphead不指向同一节点
	SLTNode* prev = *pphead;
	while (prev-&gt;next != pos) { //找到pos节点的前驱
		prev = prev-&gt;next;
	}
	newnode-&gt;next = pos;
	prev-&gt;next = newnode;
}

//删除pos节点
void SLTErase(SLTNode** pphead, SLTNode* pos) {
	assert(pphead);
	assert(pos);
	//当pos节点指向 头节点时,进行头删
	if (*pphead == pos) {
		SLTPopFront(pphead);
		return;
	}
	//pos pphead不指向同一节点
	SLTNode* prev = *pphead;
	while (prev-&gt;next != pos) { //找到pos节点的前驱
		prev = prev-&gt;next;
	}
	prev-&gt;next = pos-&gt;next;
	free(pos);
	pos = NULL;
}


//在指定位置之后插入数据
void SLTInsertAfter(SLTNode* pos, SLTDataType x) {
	assert(pos);
	SLTNode* newnode = SLTBuyNode(x);
	newnode-&gt;next = pos-&gt;next;
	pos-&gt;next = newnode;
}

//删除pos之后的节点
void SLTEraseAfter(SLTNode* pos) {
	assert(pos);
	assert(pos-&gt;next);//pos后不为空
	SLTNode* del = pos-&gt;next;
	pos-&gt;next = pos-&gt;next-&gt;next;
	//pos-&gt;next = del-&gt;next;
	free(del);
	del = NULL;
}

//销毁链表,由独立的节点构成，需要一个个销毁
void SListDestroy(SLTNode** pphead) {
	assert(pphead);
	assert(*pphead);
	SLTNode* pcur = *pphead;
	while (pcur) {
		SLTNode* next = pcur-&gt;next;
		free(pcur);
		pcur = next;
	}
	*pphead = NULL;
}
