    <h5>
     <a id="1__1">
     </a>
     <strong>
      1. 高防代理架构设计
     </strong>
    </h5>
    <p>
     <strong>
      推荐架构
     </strong>
     ：
    </p>
    <pre><code>用户 → CDN（缓存静态资源） → 高防IP（流量清洗） → 源站集群（真实IP隐藏）  
                         ↑  
                    Web应用防火墙（WAF）  
</code></pre>
    <p>
     <strong>
      实施要点
     </strong>
     ：
    </p>
    <ul>
     <li>
      源站仅允许高防IP回源，屏蔽其他所有入站流量
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token comment"># 源站防火墙规则（仅允许高防IP 192.0.2.0/24）  </span>
iptables <span class="token parameter variable">-A</span> INPUT <span class="token parameter variable">-p</span> tcp <span class="token parameter variable">-s</span> <span class="token number">192.0</span>.2.0/24 <span class="token parameter variable">-j</span> ACCEPT  
iptables <span class="token parameter variable">-A</span> INPUT <span class="token parameter variable">-p</span> tcp <span class="token parameter variable">-j</span> DROP  
</code></pre>

     <strong>
      1. 源站指纹混淆
     </strong>
    </h5>
    <p>
     <strong>
      方案
     </strong>
     ：修改默认协议特征，增加攻击者识别难度。
    </p>
    <p>
     <strong>
      Nginx指纹混淆配置
     </strong>
     ：
    </p>
    <pre><code class="prism language-nginx">server {  
    # 修改Server头  
    more_set_headers "Server: Unknown";  

    # 禁用非必要HTTP方法  
    if ($request_method !~ ^(GET|POST)$ ) {  
        return 444;  
    }  

    # 自定义错误页面（防止泄露信息）  
    error_page 404 /custom_404.html;  
    location = /custom_404.html {  
        internal;  
        return 200 'Not Found';  
    }  
}  
</code></pre>
