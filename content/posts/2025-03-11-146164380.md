    <h4>
     <a id="_30">
     </a>
     一、核心开发工具类
    </h4>
    <p>
     <strong>
      1、Live Server（实时预览）
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/88e40111bc6e468fb6033fc5aa233eb7.png"/>
    </p>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：一键启动本地服务器，支持
      <strong>
       实时刷新页面，修改代码后自动同步浏览器效果
      </strong>
      。
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：静态页面开发、调试响应式布局。
     </li>
     <li>
      <strong>
       亮点
      </strong>
      ：省去手动刷新，提升调试效率。
     </li>
    </ul>
    <p>
     <strong>
      2、Prettier - Code Formatter（格式化代码）
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a9d995f3f80c4da5bc3fb19036b7a15a.png"/>
    </p>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：自动格式化代码，支持 JavaScript、HTML、CSS、JSON 等。
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：统一团队代码风格，减少格式争议。
     </li>
     <li>
      <strong>
       配置建议
      </strong>
      ：搭配
      <code>
       .prettierrc
      </code>
      文件自定义规则。
     </li>
    </ul>
    <p>
     <strong>
      3、ESLint（静态代码检查）
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6c0f5cc6f04541bc9ef82702978ac3e3.png"/>
    </p>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：
      <strong>
       静态代码检查工具
      </strong>
      ，识别潜在错误与风格问题。
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：规范代码质量，避免低级错误。
     </li>
     <li>
      <strong>
       技巧
      </strong>
      ：结合 Prettier 使用，需安装
      <code>
       eslint-config-prettier
      </code>
      避免冲突。
     </li>
    </ul>

      1、Auto Import（自动导入模块）
     </strong>
    </p>
    <p>

      <strong>

      </strong>

     </li>
    </ul>
    <p>
     <strong>
      2、Path Intellisense（智能补全文件路径）
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/66d89506986543e1a3228ac094ca5d87.png"/>
    </p>
    <ul>
     <li>
      <strong>
       功能
      </strong>

      </strong>

     <li>
      <strong>
       适用场景
      </strong>

    </ul>

    <h4>

     九、插件管理技巧
    </h4>
    <p>
     1、
     <strong>
      不要安装过多插件
     </strong>
     : 插件过多会影响VSCode的启动速度和运行性能
    </p>
