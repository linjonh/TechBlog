    <h3>
     <a id="_0">
     </a>
     问题：
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4567814bd73840f1beb9b1e6d9dc3e53.png"/>
    </p>
    <h3>
     <a id="_2">
     </a>
     错误原因：
    </h3>
    <p>
     PowerShell 换行符错误
     <br/>
     PowerShell 中换行应使用反引号`而非反斜杠 \，错误的换行符导致命令解析中断。
    </p>
    <p>
     在 Windows 的 PowerShell 中运行 Docker 命令时遇到「sudo 无法识别」的问题，这是因为
     <strong>
      Windows 系统原生不支持
      <code>
       sudo
      </code>
      命令
     </strong>
     （该命令是 Linux/Unix 系统的权限管理工具）。以下是具体原因和解决方案：
    </p>
    <hr/>
    <h4>
     <a id="_10">
     </a>
     <strong>
      问题原因
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       命令语法差异
      </strong>
      <br/>
      Windows PowerShell 与 Linux 终端的命令体系不同，
      <code>
       sudo
      </code>
      在 Windows 中默认不存在。
     </li>
     <li>
      <strong>
       Docker Desktop 权限机制
      </strong>
      <br/>
      在 Windows 中，Docker Desktop 安装时会自动配置权限，
      <strong>
       无需使用
       <code>
        sudo
       </code>
      </strong>
      即可执行
      <code>
       docker
      </code>
      命令。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_18">
     </a>
     <strong>
      解决方案
     </strong>
    </h4>
    <h5>
     <a id="1__Docker__19">
     </a>
     1. 直接运行 Docker 命令（推荐）
    </h5>
    <p>
     移除命令中的
     <code>
      sudo
     </code>
     ，直接执行：
    </p>
    <pre><code class="prism language-powershell">docker run <span class="token operator">-</span>d <span class="token operator">--</span>privileged <span class="token operator">--</span>shm-size=2G `
  <span class="token operator">-</span>p 2201:22 `
  <span class="token operator">--</span>name <span class="token keyword">parallel</span> `
  <span class="token operator">-</span>v d:\src\shared:<span class="token operator">/</span>root/shared `
  <span class="token keyword">parallel</span> `
  <span class="token operator">/</span>bin/bash <span class="token operator">-</span>c <span class="token string">"while true; do sleep 1; done"</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/747fa13c4e894e5aa607d7c48b834d5a.png"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0fae178892864ad68842a8db6ade4406.png"/>
    </p>
    <hr/>
    <h5>
     <a id="4__38">
     </a>
     4. 验证镜像是否存在
    </h5>
    <p>
     执行
     <code>
      docker images
     </code>
     检查是否存在名为
     <code>
      parallel
     </code>
     的镜像。若不存在，需先构建或拉取镜像：
    </p>
    <pre><code class="prism language-powershell">docker build <span class="token operator">-</span>t <span class="token keyword">parallel</span> <span class="token punctuation">.</span>  <span class="token comment"># 构建镜像（需 Dockerfile）</span>
<span class="token comment"># 或</span>
docker pull <span class="token keyword">parallel</span>       <span class="token comment"># 从仓库拉取（若镜像已发布）</span>
</code></pre>
    <hr/>
    <h4>
     <a id="_48">
     </a>
     <strong>
      关键参数说明
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        参数
       </th>
       <th>
        作用
       </th>
       <th>
        参考来源
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         --privileged
        </code>
       </td>
       <td>
        授予容器特权模式（允许访问宿主机设备）
       </td>
       <td>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         --shm-size=2G
        </code>
       </td>
       <td>
        设置共享内存大小（解决
        <code>
         /dev/shm
        </code>
        容量问题）
       </td>
       <td>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -v 宿主机路径:容器路径
        </code>
       </td>
       <td>
        目录挂载（需注意 Windows 路径格式）
       </td>
       <td>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         /bin/bash -c
        </code>
       </td>
       <td>
        指定容器启动后执行的命令（此处为保持容器运行的循环脚本）

    <hr/>
    <h4>
     <a id="_58">
     </a>
     <strong>
      常见问题排查
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        错误提示
        <code>
         Unable to find image 'parallel:latest' locally
        </code>
       </strong>
       <br/>
       说明镜像不存在，需构建或拉取镜像。
      </p>
     </li>
     <li>
      <p>
       <strong>
        挂载目录权限问题
       </strong>
       <br/>
       确保 Windows 的共享目录已开启 Docker 访问权限（Docker Desktop 设置 → Resources → File Sharing）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口冲突
       </strong>
       <br/>
       检查
       <code>
        2201
       </code>
       端口是否已被其他程序占用。
      </p>
     </li>
    </ol>
