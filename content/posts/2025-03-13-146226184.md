    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     <strong>
      <strong>
       操作系统
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     下列选项中不能作为预防死锁措施的是
     <u>
     </u>
     <u>
     </u>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 破坏“循环等待"条件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 破坏“不可抢占”条件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 破坏“互斥”条件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 破坏“请求和保持”条件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：C
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：所谓死锁，是指两个以上的进程因相互争夺对方占用的资源而陷入无限等待的现象。死锁产生的4个必要条件：
     <strong>
      <strong>
       互斥（资源互斥）、请求保持（进程占有资源并等待其他资源）、不可剥夺（系统不能剥夺进程资源）和环路（进程资源图是一个环路）
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     但是，
     <strong>
      <strong>
       破坏“互斥”条件
      </strong>
     </strong>
     不能作为预防死锁的措施。这是因为
     <strong>
      <strong>
       互斥条件是资源使用的基本特性，无法进行改变
      </strong>
     </strong>
     ，它确保了资源的独占性和数据的完整性。在多任务操作系统中，进程或线程需要独占某些资源（如打印机、文件等）以防止数据竞争和不一致。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     如果取消互斥条件，允许多个进程同时访问同一资源，虽然可以避免死锁的发生，但会引发其他严重问题，如数据损坏、资源争用冲突和优先级反转等。因此，破坏互斥条件并不是一个可行的死锁预防策略。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     为了预防死锁，通常会采取其他措施来破坏产生死锁的其他三个必要条件之一或多个：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       请求和保持条件
      </strong>
     </strong>
     ：可以通过实施资源分级策略或静态分配法来破坏。例如，要求进程在开始运行前一次性申请其在整个运行期间所需的全部资源，或者在提出新资源请求前释放已占有的资源。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       不可抢占条件
      </strong>
     </strong>
     ：可以允许进程抢占已经分配给其他进程的资源，但这通常需要复杂的管理机制来处理资源的抢占和恢复。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       循环等待条件
      </strong>
     </strong>
     ：可以通过全局资源排序策略来破坏，即给系统中的所有资源编号，并要求进程按递增顺序申请资源。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在多道批处理系统中有P1和P2两个作业，执行顺序如下，P1(计算40ms，I/O操作60ms，计算20ms)，P2(计算100ms，I/O操作40ms，计算40ms)，P2比P1晚20ms到达，在不考虑调度和切换时间的情况下完成两道作业最少需要
     <u>
     </u>
     <u>
     </u>
     时间。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 240ms
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 200ms
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 260ms
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 220ms
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：D
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：教材2.3.2操作系统，P30。多道批处理操作系统允许多个作业装入内存执行，在任意一个时刻，作业都处于开始点和终止点之间。每当运行中的一个作业由于输入／输出操作需要调用外部设备时，就把CPU交给另一个等待运行的作业，从而将主机与外部设备的工作由串行改变为并行，进一步避免了因主机等待外设完成任务而浪费宝贵的CPU时间。多道批处理系统主要有3 个特点：
     <strong>
      <strong>
       多道、宏观上并行运行和微观上串行运行
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     简单的多个作业的执行，时空图如下所示：
    </p>
    <p class="img-center">
     <img alt="" height="326" src="https://i-blog.csdnimg.cn/direct/483f9e8485e84a3db8c10cabeaa6fe3c.png" width="549"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     分段允许程序员把内存视为由这个地址空间或段组成，其中段的大小是
     <u>
     </u>
     <u>
     </u>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 固定的
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 不可变的
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 相等的
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 动态可变的
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：D
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：
     <strong>
      <strong>
       分段存储管理
      </strong>
     </strong>
     是将进程空间划分成若干个
     <strong>
      <strong>
       段
      </strong>
     </strong>
     ，每段也有
     <strong>
      <strong>
       段号
      </strong>
     </strong>
     和
     <strong>
      <strong>
       段内地址
      </strong>
     </strong>
     ，每段是一组
     <strong>
      <strong>
       完整的逻辑信息
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       分段地址结构
      </strong>
     </strong>
     ：
    </p>
    <p class="img-center">
     <img alt="" height="103" src="https://i-blog.csdnimg.cn/direct/439e25d483ad4ee9b20eef2607c86bcb.png" width="368"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       分页与分段的区别
      </strong>
     </strong>
     ：分页是根据
     <strong>
      <strong>
       物理空间
      </strong>
     </strong>
     划分，每页大小相同；分段是根据
     <strong>
      <strong>
       逻辑空间
      </strong>
     </strong>
     划分，每段是一个完整的功能，便于共享，但是
     <span style="color:#ff0000">
      大小不同
     </span>
     ，即根据需要动态改变。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     当一个进程被一个更高优先级的进程抢占或其时间片用完时，其状态会从执行态转变为
     <u>
     </u>
     <u>
     </u>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     <u>
     </u>
     状态。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 就绪
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 挂起
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 睡眠
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 阻塞
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：A
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：当一个进程被一个更高优先级的进程抢占或其时间片用完时，其状态会
     <strong>
      <strong>
       从执行态转变为绪状态
      </strong>
     </strong>
     。
    </p>
    <p class="img-center">
     <img alt="" height="280" src="https://i-blog.csdnimg.cn/direct/dcd0646d12f04d2ca55d16319a1370b0.png" width="379"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     进程的基本状态有：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       就绪状态
      </strong>
     </strong>
     (Ready)：进程已获得除CPU以外的所有必要盗源，只要再获得CPU，就可以立即执行。例如，在多任务操作系统中，多个进程可能同时处于就绪状态，等待CPU调度。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       执行状态
      </strong>
     </strong>
     (Running)：进程正在CPU上运行，占用 CPU资源执行指令。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       阻塞状态
      </strong>
     </strong>
     (Blocked)：进程因等待某一事件(如I/O操作完成、等待信号量等)而暂时无法继续执行，此时即使CPU空闲，它也不能占用CPU运行，
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     低优先级进程状态变化的原因分析：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     当低优先级进程正在执行(处于执行态)时，如果高优先级进程进入就绪状态，根据优先级调度算法，操作系统会抢占CPU，将低优先级进程从执行态变为就绪态。此时低优先级进程仍然具备执行条件，只是因为优先级较低而暂时失去CPU使用权。另外，如果采用时间片轮转调度算法，低优先级进程的时间片用光，它也会从执行态变为就绪态，等待下一次调度。这种机制可以保证每个就绪进程都有机会获得CPU时间，实现公平的资源分配。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     应用程序在用户态使用特权指令进行系统调用，是
     <u>
     </u>
     <u>
     </u>
     中断。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A.信号中断
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B.溢出中断
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C.访管中断
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D.外部中断
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：C
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：信号中断：通常是外部事件或异步信号触发。例如按下Ctrl+C。 溢出中断：通常发生算术运算时候。
     <strong>
      <strong>
       访管中断
      </strong>
     </strong>
     ：应用程序主动发起的软件中断。外部中断：通常由外部设备（如键盘、鼠标）等触发。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       访管中断
      </strong>
     </strong>
     是一种自愿性的中断，由用户程序通过执行访管指令来请求操作系统服务。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       访管中断
      </strong>
     </strong>
     是计算机系统中的一个重要概念，它涉及到用户态与核心态之间的切换。在用户态下，程序无法直接执行某些特权指令或访问受限资源，而当需要这些操作时，程序会通过执行一条特殊的访管指令来触发访管中断。这种中断机制允许用户程序以受控的方式进入核心态，从而请求操作系统提供的服务或执行特权操作。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       访管中断
      </strong>
     </strong>
     的处理过程通常包括以下几个步骤：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     1）触发访管中断：用户程序执行访管指令，产生一个中断事件（自愿中断），暂停当前用户程序的执行。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     2）保存上下文：CPU保存当前用户程序的执行状态和上下文信息，以便后续恢复执行。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     3）切换到核心态：CPU从用户态切换到核心态，开始执行操作系统的中断处理程序。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     4）执行系统调用：操作系统根据访管指令中的操作数和参数，执行相应的系统调用或服务例程，完成用户请求的操作。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     5）恢复用户态：系统调用或服务完成后，操作系统恢复用户程序的执行状态和上下文信息，将CPU切换回用户态，继续执行被中断的用户程序。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       访管中断
      </strong>
     </strong>
     是实现用户程序与操作系统之间交互的重要机制之一，它确保了用户程序可以在受控的情况下访问受限资源或执行特权操作，同时保护了系统的稳定性和安全性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     <strong>
      <strong>
       数据库设计
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     数据库的三级模式结构中，
     <u>
     </u>
     <u>
     </u>
     描述局部数据的逻辑结构和特征。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A.概念模式
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B.内模式
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C.外模式
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D.逻辑模式
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：C
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：在数据库的三级模式中，描述局部数据的逻辑结构和特征的是
     <strong>
      <strong>
       外模式
      </strong>
     </strong>
     。
    </p>

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：
     <strong>
      <strong>
       白盒测试
      </strong>
     </strong>
     ，也称
     <strong>
      <strong>
       结构测试
      </strong>
     </strong>
     ，根据程序的内部结构和逻辑来设计测试用例，对程序的路径和过程进行测试，检查是否满足设计的需要。常用的白盒测试技术有
     <strong>
      <strong>
       控制流分析、数据流分析、路径分析、程序变异
      </strong>
     </strong>
     等。根据测试用例的覆盖程度，分为
     <strong>
      <strong>
       语句覆盖、判定覆盖、分支覆盖和路径覆盖
      </strong>
     </strong>
     等。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       控制流分析
      </strong>
     </strong>
     ：属于白盒测试中的静态分析方法之一，主要通过分析程序的控制流图，来确定程序的执行路径和逻辑结构，从而设计测试用例以覆盖不同的路径和分支。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       数据流分析
      </strong>
     </strong>
     ：也是白盒测试的静态分析方法之一，侧重于分析程序中数据的流动和使用情况，包括数据的定义、引用、赋值等，以发现数据在流动过程中可能出现的问题，如未初始化的变量、数据的别名等。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       路径分析
      </strong>
     </strong>
     ：也是白盒测试的静态分析方法之一，是通过分析程序的所有可能执行路径来设计测试用例。路径分析的目标是覆盖程序中的所有可能路径，但由于路径数量可能非常庞大（甚至无限），通常采用一些简化策略，如基本路径测试。基本路径测试通过计算程序的圈复杂度（Cyclomatic Complexity）来确定需要测试的基本路径数量。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       程序变异测试
      </strong>
     </strong>
     ：是白盒测试的一种动态分析方法，通过故意引入错误（变异）到程序中，然后运行测试用例来检测这些错误是否被发现，从而评估测试用例的有效性和充分性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       黑盒测试
      </strong>
     </strong>
     ，也称
     <strong>
      <strong>
       功能测试
      </strong>
     </strong>
     ，在不考虑软件内部结构和特性的情况下，根据功能设计用例，测试软件功能。常用的黑盒测试技术有
     <strong>
      <strong>
       等价类划分、边界值分析、错误推测、因果图和判定表
      </strong>
     </strong>
     等。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     下列关于逻辑覆盖，说法错误的是
     <u>
     </u>
     <u>
     </u>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 满足条件覆盖并不一定满足判定覆盖
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 满足条件组合覆盖的测试一定满足判定覆盖、条件覆盖和判定/条件覆盖
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 满足路径覆盖也一定满足条件组合覆盖
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 判定/条件覆盖同时满足判定覆盖和条件覆盖
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：C
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：常用的白盒测试技术有
     <strong>
      <strong>
       逻辑覆盖
      </strong>
     </strong>
     、循环覆盖和基本路径测试。逻辑覆盖：
     <strong>
      <strong>
       语句覆盖、判定覆盖、条件覆盖、判定/条件覆盖、条件组合覆盖和路径覆盖
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       语句覆盖
      </strong>
     </strong>
     （SC）：是指选择足够多的测试用例，使被测程序的
     <strong>
      <strong>
       每条语句至少执行一次
      </strong>
     </strong>
     。语句覆盖是一种很弱的逻辑覆盖。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       判定覆盖
      </strong>
     </strong>
     （DC）：又称分支覆盖，是指设计足够的测试用例，使得
     <strong>
      <strong>
       不仅每条语句至少执行一次，而且每个判定表达式的每种可能的结果（分支）都至少执行一次
      </strong>
     </strong>
     。判定覆盖比语句覆盖强。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       条件覆盖
      </strong>
     </strong>
     （CC）：是指设计一组测试用例，不仅每条语句至少执行一次，而且使判定表达式中的每个条件的所有可能的值至少满足一次。条件覆盖不一定包含判定覆盖、判定覆盖也不一定包含条件覆盖。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       判定/条件覆盖
      </strong>
     </strong>
     （CDC）：是指同时满足
     <strong>
      <strong>
       判定覆盖和条件覆盖
      </strong>
     </strong>
     的逻辑覆盖。设计足够的测试用例，使得
     <strong>
      <strong>
       判定表达式中每个条件的所有可能的值至少满足一次，而且每个判定本身的所有可能结果也至少出现一次
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       条件组合覆盖
      </strong>
     </strong>
     （MCC）：设计足够的测试用例，使得
     <strong>
      <strong>
       每个判定表达式中条件的所有值的组合都至少出现一次
      </strong>
     </strong>
     。满足条件组合覆盖的测试用例，也一定满足判定覆盖、条件覆盖和判定/条件覆盖。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ●
     <strong>
      <strong>
       路径覆盖
      </strong>
     </strong>
     ：设计足够的测试用例，覆盖被测程序中
     <strong>
      <strong>
       所有可能的路径
      </strong>
     </strong>
     （如果程序中有环路，则要求每条环路至少经过一次）。路径覆盖考虑了程序中各种判定结果的所有可能组合，因此是一种较强的覆盖标准。但
     <strong>
      <strong>
       路径覆盖并未考虑判定中的条件结果的组合，并不能代替条件覆盖和条件组合覆盖
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     软件测试过程中的系统测试主要是为了发现
     <u>
     </u>
     <u>
     </u>
     阶段的错误。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A.编码
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B.需求分析
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C.概要设计
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D.详细设计
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：B
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：
     <strong>
      <strong>
       系统测试
      </strong>
     </strong>
     的对象是完整、集成的计算机系统。
     <strong>
      <strong>
       目的是通过与软件需求进行比较，发现所开发的系统与用户需求不符或矛盾的地方。测试依据是用户需求（SRS）或开发合同
      </strong>
     </strong>
     。主要包括
     <strong>
      <strong>
       功能测试、性能测试
      </strong>
     </strong>
     、健壮性测试、安装或反安装测试、用户界面测试、压力测试、可靠及安全性测试等。一般采用黑盒测试法。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     基于数据驱动的测试脚本写在
     <u>
     </u>
     <u>
     </u>
     文件里。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 数据文件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 脚本文件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 程序文件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 系统文件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：B
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：基于数据驱动的测试中，测试数据通常存储在数据文件中，如 Excel、CSV、JSON、YAML 等格式。而测试脚本则写在脚本文件中，通过相应的代码逻辑在脚本中实现从数据文件中读取数据，并根据数据进行测试操作，将测试数据与测试逻辑分离。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在软件的使用过程中，用户往往会对软件提出新的功能与性能要求。为了满足这些要求，需要修改或再开发软件，以扩充软件功能、增强软件性能、改进加工效率、提高软件的可维护性。这种情况下进行的维护活动称为
     <u>
     </u>
     <u>
     </u>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A. 预防性维护
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B. 适应性维护
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C. 完善性维护
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D. 改正性维护
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：C
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：P110。系统维护主要包括硬件维护、
     <strong>
      <strong>
       软件维护
      </strong>
     </strong>
     和数据维护。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 软件维护类型
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）
     <strong>
      <strong>
       正确性维护
      </strong>
     </strong>
     ：改正在系统开发阶段已发生而系统测试阶段尚未发现的错误。修正BUG、错误。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）
     <strong>
      <strong>
       适应性维护
      </strong>
     </strong>
     ：使应用软件适应环境变化（外部环境、数据环境）而进行的修改。应变。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）
     <strong>
      <strong>
       完善性维护
      </strong>
     </strong>
     ：
     <strong>
      <strong>
       为扩充功能和改善性能而进行的修改。占整个维护工作的50%~60%
      </strong>
     </strong>
     。新需求。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）
     <strong>
      <strong>
       预防性维护
      </strong>
     </strong>
     ：为适应未来的软/硬环境的变化，应主动增加预防性的新功能，以使应用系统适应各类变化而不被淘汰。针对未来。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     不包括配置管理的是
     <u>
     </u>
     <u>
     </u>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A.UML
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B.ISO9000
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C.PMBOK
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D.CMMI
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：A
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：超纲题。UML、ISO9000、PMBOK和CMMI都是与软件工程和项目管理相关的标准或模型，但它们与配置管理的关系如下：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     UML（统一建模语言）：是一种用于软件系统建模和其它系统的图形化建模语言。它主要关注于软件开发的分析与设计阶段，并不直接包含配置管理的内容。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ISO9000：是一系列关于质量管理体系的国际标准，它提供了组织如何建立、实施、维护和改进质量管理体系的指导原则。ISO9000标准中包含了对文件控制和记录保存的要求，这些都是配置管理的重要组成部分。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     PMBOK（项目管理知识体系指南）：是由项目管理协会（PMI）出版的一本指南，它提供了项目管理的一套标准和最佳实践。PMBOK中包含了项目配置管理的内容。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     CMMI（能力成熟度模型集成）：是一个过程改进的框架，它提供了一套过程域，帮助组织改进其软件工程过程。CMMI中包含了配置管理作为一个关键的过程域。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     因此，不包括配置管理的是选项A。UML是一种建模语言，不直接涉及配置管理。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     不同应用领域中的软件元素，例如数据结构、分类算法和人机界面等属于
     <u>
     </u>
     <u>
     </u>
     重用。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     A.横向重用
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     B.纵向重用
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     C.交叉重用
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     D.可控重用
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       答案
      </strong>
     </strong>
     ：A
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       解析
      </strong>
     </strong>
     ：超纲题（2023.11网络规划设计师真题）。
     <strong>
      <strong>
       软件重用
      </strong>
     </strong>
     是指在两次或多次不同的软件开发过程中重复使用相同或相似软件元素的过程。按照重用活动是否跨越相似性较少的多个应用领域，软件重用可以区别为
     <strong>
      <strong>
       横向重用和纵向重用
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       横向重用
      </strong>
     </strong>
     <strong>
      <strong>
       （水平式重用）
      </strong>
     </strong>
     是指重用不同应用领域中的软件元素，例如
     <strong>
      <strong>
       数据结构、分类算法和人机界面构建
      </strong>
     </strong>
     等。
     <strong>
      <strong>
       标准函数是种典型的、原始的横向重用机制
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       纵向重用
      </strong>
     </strong>
     <strong>
      <strong>
       （垂直式重用）
      </strong>
     </strong>
     是指在一类具有较多公共性的应用领域之问进行软部件重用。纵向重用活动的主要关键点是域分析：根据应用领域的特征及相似性预测软部件的可重用性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       交叉重用
      </strong>
     </strong>
     ：非标准术语，通常不用于描述软件重用分类。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       可控重用
      </strong>
     </strong>
     ：强调对重用过程的管理（如版本控制），而非重用类型本身。
    </p>
