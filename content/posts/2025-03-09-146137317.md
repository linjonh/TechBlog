    <h5 id="1.1%20list%E7%9A%84%E4%BB%8B%E7%BB%8D%C2%A0" name="1.1%20list%E7%9A%84%E4%BB%8B%E7%BB%8D%C2%A0">
     1.1 list的介绍
    </h5>
    <blockquote>
     <p>
      1.1 list的介绍
      <br/>
      list 是可以在常数范围内在任意位置进行插入和删除的序列式容器，并且该容器可以前后双向迭代。
     </p>
     <p>
      list 的底层是双向链表结构，双向链表中的每个元素存储在互不相关的独立节点中，在节点中通过指针指向的前一个元素和后一个元素。
     </p>
     <p>
      list 和 forward_list 非常相似：最主要的不同在于 forward_list 是单链表，只能朝前迭代，已让其更简单高效。
     </p>
     <p>
      与其它的序列式容器相比（arry、vector、deque）
      <span style="color:#fe2c24">
       ，list 通常在任意位置进行插入，移除元素的执行效率更好。
      </span>
     </p>
     <p>
      与其它序列式容器相比，
      <span style="color:#fe2c24">
       list 和 forward_list 最大的缺陷是不支持任意位置的随机访问
      </span>
      ，比如：要访问 list 的第 5 个元素，必须从已知的位置（比如头部或者尾部）迭代到该位置，在这段位置上迭代需要线性的时间开销；list 还需要一些额外的空间，已保存每个节点的相关联信息。
     </p>
     <p>
     </p>
     <p>
     </p>
    </blockquote>

     1.2.2  iterator的使用
    </h6>
    <p>
     <img alt="" height="365" src="https://i-blog.csdnimg.cn/direct/861a9a0f0ad24fe2b75068f254df0966.png" width="1474">

    <p>
     <span style="color:#fe2c24">
      <strong>
       注意：
      </strong>
     </span>
     begin 与 end 为正向迭代器，对迭代器执行 ++ 操作，迭代器向后移动。rbegin 与 rend 为反向迭代器，对迭代器执行 ++ 操作，迭代器向前移动。由于 list 的底层物理空间并不连续，所以 list 的迭代器不再是原生指针，并且 list 的迭代器没有对 + 和 - 进行重载，只重载了 ++ 和 – ，因为空间不连续，重载 + 会比较复杂。即 l.begin() + 5 是不被允许的。
    </p>
    <p>
    </p>
    <pre><code class="language-cpp">#include&lt;iostream&gt;
using namespace std;
#include&lt;list&gt;
void print_list(list&lt;int&gt;&amp; lt)
{
    list&lt;int&gt;::iterator it = lt.begin();
   // 用迭代器方式打印l2中的元素
    while (it != lt.end())
    {
        cout &lt;&lt; *it &lt;&lt; " ";
        it++;
    }
    cout &lt;&lt; endl;

    
    for (int&amp; e : lt) // C++11范围for的方式遍历
    {
        cout &lt;&lt; e &lt;&lt; " ";
    }
    cout &lt;&lt; endl;
    //反向迭代器
    list&lt;int&gt;::reverse_iterator rit = lt.rbegin();
    while (rit != lt.rend())
    {
        cout &lt;&lt; *rit &lt;&lt; " ";
        rit++;
    }
    cout &lt;&lt; endl;
}
void TestList1()
{
    list&lt;int&gt; l1; // 构造空的l1
    list&lt;int&gt; l2(4, 100);// l2中放4个值为100的元素
    list&lt;int&gt; l3(l2.begin(), l2.end());// 用l2的[begin(), end()）左闭右开的区间构造l3
    list&lt;int&gt; l4(l3);// 用l3拷贝构造l4
    int arr[] = { 1,2,3,4,5,6 };
    list&lt;int&gt; l5(arr, arr + sizeof(arr) / sizeof(arr[0])); // 以数组为迭代器区间构造l5
    list&lt;int&gt; l6{ 9,8,7,6 };// 列表格式初始化C++11
    print_list(l1);
    print_list(l2);
    print_list(l3);
    print_list(l4);
    print_list(l5);
    print_list(l6); 
}

int main()
{
	TestList1();
	return 0;

     1.2.5 modifiers（链表修改）
    </h6>
    <p>
     <img alt="" height="556" src="https://i-blog.csdnimg.cn/direct/e8c9d0dae9ab4f1f927bbb57f57da6b2.png" width="1259"/>

    <p>
     <span style="color:#fe2c24">
      insert 插入元素并不会导致迭代器失效，例如：相较于 vector 中的 insert，list 中的 insert 并不会去扩容挪动数据，而 vector 中的 insert 可能会进行扩容挪动数据，最终导致迭代器失效。list 中的删除元素接口会导致迭代器失效，失效的只有指向被删除节点的迭代器，其他迭代器不会受到影响。
     </span>
    </p>
    <pre><code class="language-cpp">void TestList3()
{
    list&lt;int&gt; l1; // 构造空的l1
    list&lt;int&gt; l2(4, 100);// l2中放4个值为100的元素
    cout &lt;&lt; l1.size() &lt;&lt; endl;
    cout &lt;&lt; l2.size()&lt;&lt;endl;
    l1.push_front(22);
    l1.push_front(11);
    print_list(l1);
    l2.insert(l2.begin(), 99);
    print_list(l2);
    list&lt;int&gt;::iterator it = l2.begin();
    int k = 3;
    while (k--)
    {
        it++;
    }
    l2.insert(it, 0);
    print_list(l2);
    l2.clear();
    print_list(l2);
    cout &lt;&lt; "**********************";
}
int main()
{
	TestList3();

    <h6 id="1.2.6%20operation%EF%BC%88%E5%AF%B9%E9%93%BE%E8%A1%A8%E7%9A%84%E4%B8%80%E4%BA%9B%E6%93%8D%E4%BD%9C%EF%BC%89%C2%A0" name="1.2.6%20operation%EF%BC%88%E5%AF%B9%E9%93%BE%E8%A1%A8%E7%9A%84%E4%B8%80%E4%BA%9B%E6%93%8D%E4%BD%9C%EF%BC%89%C2%A0">
     1.2.6 operation（对链表的一些操作）
    </h6>
    <p>
     <img alt="" height="336" src="https://i-blog.csdnimg.cn/direct/28c693f4086b40c8b0c129461eeb0f6f.png" width="1231"/>
    </p>
    <p>
     <img alt="" height="412" src="https://i-blog.csdnimg.cn/direct/07f34ad63b0a4738bfbc2ab97fd8bd6e.png" width="1172"/>
    </p>
    <p>
     <span style="color:#fe2c24">
      <strong>
       <code>
        splice()
       </code>
       函数主要用于在列表中进行元素的转移操作。 它可以将一个列表中的部分或全部元素转移到另一个列表中。可以指定要转移的元素范围以及目标插入位置等，实现了高效灵活的元素移动和重组。
      </strong>
     </span>
    </p>
    <p>
     <img alt="" height="219" src="https://i-blog.csdnimg.cn/direct/417ce47d13874c80aea256b84c4a32c8.png" width="1550"/>
    </p>
    <p>
     <img alt="" height="911" src="https://i-blog.csdnimg.cn/direct/f05b72d2ce0045baa116318f31ddebd3.png" width="1884"/>
    </p>
    <p>
     <strong>
      <span style="color:#fe2c24">
       <code>
        remove
       </code>
       函数相当于一直遍历列表，然后
       <code>
        erase
       </code>
       删除指定元素。
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="784" src="https://i-blog.csdnimg.cn/direct/c94205b4a89d466ea51f66c37952c993.png" width="1502"/>
    </p>
    <p>
     <span style="color:#fe2c24">
      <code>
       unique
      </code>
      函数主要用于移除列表中相邻的重复元素。它使得容器中只保留不重复的元素序列，但需要注意的是，它并不保证完全去除所有重复元素，只是处理相邻的重复项，通常也需要结合其他操作来实现完全去重。
     </span>
    </p>
    <p>
     <img alt="" height="699" src="https://i-blog.csdnimg.cn/direct/fd64d0b626334f219ba91115af4ed4dd.png" width="1928"/>
    </p>
    <p>
     <strong>
      <span style="color:#fe2c24">
       <code>
        merge()
       </code>
       函数主要用于将两个已排序的序列合并成一个新的已排序序列。 它会按照排序顺序将一个序列中的元素与另一个序列中的元素合理地组合在一起，形成一个合并后的有序序列。需要注意的是，在合并之前，两个源序列本身需要是已经排序好的。
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="888" src="https://i-blog.csdnimg.cn/direct/4efa10fefee74fbd84c631be94a1657b.png" width="1664"/>
    </p>
    <p>
     <span style="color:#fe2c24">
      链表逆置可以使用 list 自身的接口，也可以使用算法库中的 reverse，二者没有什么区别。链表排序只能使用 list 自身的 sort() 接口（底层是利用归并排序），不能使用算法库的 sort，因为算法库中的 sort 底层是通过快排来实现的，而快排中会涉及到三数取中需要迭代器 - 迭代器，链表不能很好的支持。虽然链表提供了排序接口，但是用链表对数据排序意义不大，效率太低了，更希望用 vector 来对数据进行排序。
     </span>
    </p>
    <p>
     <img alt="" height="691" src="https://i-blog.csdnimg.cn/direct/4571abbade4a4cf88775315ecd60b15f.png" width="1747"/>
    </p>

     二、从功能角度迭代器分类：
    </h4>

    <p>
     其中
     <code>
      forward_list
     </code>
     、
     <code>
      unordered_xxx
     </code>
     都是单向迭代器；
     <code>
      list
     </code>
     、
     <code>
      map
     </code>
     、
     <code>
      set
     </code>
     都是双向迭代器；
     <code>
      vector
     </code>
     、
     <code>
      string
     </code>
     、
     <code>
      deque
     </code>
     都是随机迭代器。对迭代器的这种分类方式，是由容器的底层结构来决定的。
    </p>
    <p>
     算法库中也有一个
     <code>
      sort()
     </code>
     函数，但是其支持的是随机迭代器，而
     <code>
      list
     </code>
     是一种双向迭代器，所以
     <code>
      list
     </code>
     无法调用算法库中的
     <code>
      sort()
     </code>
     。
    </p>
