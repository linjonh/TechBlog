    <p>
     在正式实现UDP网络通信前，我们先实现一个打印日志内容的文件。
    </p>
    <pre><code class="language-cpp">#pragma once

#include&lt;cstdio&gt;
#include&lt;ctime&gt;
#include&lt;cassert&gt;
#include&lt;cerrno&gt;
#include&lt;cstdlib&gt;
#include&lt;cstdarg&gt;

#define DEBUG 0
#define NOTICE 1
#define WARINING 2
#define FATAL 3

const char* log_level[]={"DEBUG","NOTICE","WARINING","FATAL"};
void logMessage(int level,const char* format,...)
{
    assert(level&gt;=DEBUG);
    assert(level&lt;=FATAL);

    char* name=getenv("USER");
    char logBuffer[1024];
    va_list ap;
    va_start(ap,format);
    vsnprintf(logBuffer,sizeof(logBuffer)-1,format,ap);
    va_end(ap);

    FILE* out=(level==FATAL)? stderr:stdout;
    time_t tm = time(nullptr);
    struct tm* localTm=localtime(&amp;tm);

    char timeStr[26]; // 考虑到asctime函数返回的字符串自带换行符，为了打印出来好看，下面手动去除这个换行符
    snprintf(timeStr, sizeof(timeStr), "%s", asctime(localTm));
    timeStr[strcspn(timeStr, "\n")] = 0; // Remove trailing newline
   
    fprintf(out,"%s | %s | %s | %s\n",
            log_level[level],
            timeStr,
            name==nullptr?"unknow":name,
            logBuffer);
}
</code></pre>
    <blockquote>
     <p>
      1、上面运用了可变参数的知识点，简单介绍一下可变参数的使用
     </p>
     <ul>
      <li>
       首先，需要包含
       <strong>
        &lt;stdarg.h&gt;
       </strong>
       （C语言）或
       <strong>
        &lt;cstdarg&gt;
       </strong>
       （C++）头文件。
      </li>
      <li>
       使用
       <strong>
       </strong>
       <strong>
        va_list
       </strong>
       类型声明一个变量来存储参数信息。
      </li>
      <li>
       调用
       <strong>
        va_start
       </strong>
       宏初始化上述变量，使其指向可变参数列表中的第一个参数。
      </li>
      <li>
       使用
       <strong>
        va_arg
       </strong>
       宏访问参数。
      </li>
      <li>
       最后调用
       <strong>
        va_end
       </strong>
       宏完成清理工作。
      </li>
     </ul>
     <p>
      2、
      <strong>
       vsnprintf()
      </strong>
     </p>
     <p>
      它用于格式化字符串输出，并允许传递一个
      <strong>
       va_list
      </strong>
      类型的参数来支持可变数量的参数。
     </p>
     <pre><code class="language-cpp">int vsnprintf(char *str, size_t size, const char *format, va_list ap);</code></pre>
     <ul>
      <li>
       str: 指向用于存储生成字符串的缓冲区。
      </li>
      <li>
       size: 缓冲区str的大小（以字符为单位）。如果生成的字符串长度（包括终止的空字符）超过这个值，则会被截断。
      </li>
      <li>
       format: 格式化字符串，指定如何转换参数并格式化到输出字符串中。
      </li>
      <li>
       ap: 一个
       <strong>
        va_list
       </strong>
       类型的对象，包含了要使用的所有参数的信息
      </li>
     </ul>
     <p>
      3、获取当前时间
     </p>
     <pre><code class="language-cpp">time_t tm = time(nullptr);
struct tm* localTm=localtime(&amp;tm);</code></pre>
     <p>
      第一行代码调用了
      <strong>
       time
      </strong>
      函数，该函数返回自纪元（1970年1月1日00:00:00 UTC）以来的秒数，并存储在
      <strong>
       time_t
      </strong>
      类型的变量
      <strong>
       tm
      </strong>
      中。
     </p>
     <p>
      第二行使用
      <strong>
       localtime函数将 time_t 类型的值转换为本地时间表示
      </strong>
      ，
      <strong>
       结果是一个指向 struct tm类型的指针 localTm
      </strong>
      。struct tm 结构体包含了年、月、日、时、分、秒等详细的时间信息。
     </p>

    <p>
     实现简单的udpServer的功能
    </p>
    <ol>
     <li>
      <span style="color:#fe2c24">
       <strong>
        创建套接字
       </strong>
      </span>
      ：首先，服务端需要创建一个UDP套接字。
     </li>
     <li>
      <span style="color:#fe2c24">
       <strong>
        绑定地址和端口
       </strong>
      </span>
      ：然后，将该套接字绑定到特定的IP地址和端口号上，以便客户端能够发送数据到这个地址和端口。
     </li>

    <h6 id="1%E3%80%81%E7%A7%81%E6%9C%89%E6%88%90%E5%91%98" name="1%E3%80%81%E7%A7%81%E6%9C%89%E6%88%90%E5%91%98">
     1、私有成员
    </h6>
    <p>
     在进行网络通信时我们需要的是 端口号、IP还有 套接字 信息 。
    </p>
    <pre><code class="language-cpp">private:
    uint16_t _port;
    string _ip;

    <hr/>
    <h6 id="%C2%A02%E3%80%81%E6%9E%84%E9%80%A0%E5%87%BD%E6%95%B0%E5%92%8C%E6%9E%90%E6%9E%84%E5%87%BD%E6%95%B0" name="%C2%A02%E3%80%81%E6%9E%84%E9%80%A0%E5%87%BD%E6%95%B0%E5%92%8C%E6%9E%90%E6%9E%84%E5%87%BD%E6%95%B0">
     2、构造函数和析构函数
    </h6>
    <pre><code class="language-cpp">udpServer(uint16_t port,string ip="")
        :_port(port),
        _ip(ip)
        {}
~udpServer()
 {}</code></pre>
    <p>
     这里需要传入的参数是端口号和IP地址。
    </p>
    <p>
     <strong>
      传入的IP地址可以默认为空，即当创建这个UDP服务器对象时，如果没有提供第二个参数（IP地址），那么_ip 成员变量将被初始化为空字符串
     </strong>
     。这是因为
     <strong>
      <span style="color:#fe2c24">
       如果用户没有特别指出要监听哪个IP地址，那就默认监听所有的网络接口
      </span>
      .
     </strong>
     换句话说，它会监听该主机上所有的IP地址，无论数据包到达哪个网络接口，只要目标端口匹配，你的服务就能接收到这些数据包。
    </p>
    <blockquote>
     <p>
      同一台主机可以拥有多个IP地址。
     </p>
     <ul>
      <li>
       多个网络接口：如果一台计算机配置了多个网络接口（例如，一个以太网卡和一个无线网卡），每个接口都可以被分配一个独立的IP地址。
      </li>
      <li>
       子接口（VLANs）：在一些情况下，单个物理网络接口可以通过虚拟局域网（VLAN）技术划分成多个逻辑子接口，每个子接口可以配置不同的IP地址。
      </li>
      <li>
       IPv6与IPv4共存：现代操作系统支持同时配置IPv4和IPv6地址。这意味着即使只有一个网络接口，它也可能拥有至少两个IP地址——一个IPv4地址和一个IPv6地址。
      </li>
      <li>
       别名接口：某些操作系统允许给单一的网络接口配置多个IP地址，通过创建所谓的“别名”接口来实现。这对于需要托管多个网站或服务的服务器特别有用，因为每个服务可以绑定到不同的IP地址。
      </li>
      <li>
       动态分配的临时地址：在某些情况下，比如使用DHCP时，可能会为同一个接口动态分配额外的临时IP地址。
      </li>
     </ul>
    </blockquote>

    <h6 id="3%E3%80%81%20udp%E6%9C%8D%E5%8A%A1%E5%99%A8%E5%88%9D%E5%A7%8B%E5%8C%96" name="3%E3%80%81%20udp%E6%9C%8D%E5%8A%A1%E5%99%A8%E5%88%9D%E5%A7%8B%E5%8C%96">
     3、 udp服务器初始化
    </h6>
    <pre><code class="language-cpp"> void init()
    {
        _sockFd=socket(AF_INET,SOCK_DGRAM,0);//创建套接字
        if(_sockFd&lt;0)
        {
            logMessage(FATAL,"socket() fail::%s :%d",
                    strerror(errno),_sockFd);
            exit(1);
        }
        logMessage(DEBUG,"socket creat success: %d", _sockFd);


        struct sockaddr_in local;//创建结构体对象
        bzero(&amp;local,sizeof(local));//该函数的作用是将一段内存区域的内容设置为0，这里是将结构体內容全置空。


        local.sin_family=AF_INET;//地址类型，选择协议
        local.sin_port=htons(_port);//选择端口号，端口号是要发送到网络的，所以需要从本地字节序转成网络字节序

        local.sin_addr.s_addr=_ip.empty()?htonl(INADDR_ANY):inet_addr(_ip.c_str());

        if(bind(_sockFd,(const struct sockaddr*)&amp;local,sizeof(local))==-1){
            //绑定失败
            logMessage(FATAL,"bind() failed::%s : %d",
                strerror(errno),_sockFd);
            exit(2);
        }
            //绑定成功
        logMessage(DEBUG,"socket bind success: %d",_sockFd);
    }</code></pre>
    <blockquote>
     <ul>
      <li>
       <pre><code class="language-cpp">  local.sin_addr.s_addr=_ip.empty()?htonl(INADDR_ANY):inet_addr(_ip.c_str());</code></pre>
       <p>
        下面是对于这一行代码的解释
       </p>
      </li>
      <li>
       <strong>
        IP
       </strong>
       不能直接填充到结构体中, 因为类中的
       <strong>
        _ip
       </strong>
       是字符串,  而网络中的
       <strong>
        IP
       </strong>
       通常用4字节的二进制表示, 结构体中同样是此类型
       <strong>
        ，
       </strong>
       所以 还需要将
       <strong>
        点分十进制字符串型的_ip
       </strong>
       转换为
       <strong>
        uint32_t
       </strong>
       才能填充到 结构体中。
      </li>
      <li>
       判断ip地址是否为空，如果为空则传入
       <strong>
        INADDR_ANY
       </strong>
       结构体，否则就将 _ip字符串转换为
       <strong>
        in_addr_t
       </strong>
       类型, 然后填充到结构体中
      </li>
      <li>
       <strong>
        INADDR_ANY 其实是 强转为 in_addr_t 类型的 0.
       </strong>
      </li>
      <li>
       <strong>
        网络服务器 使用 INADDR_ANY 作为IP, 绑定到主机中. 表示监听本机上所有的IP 网络接口.一台服务器主机可能有许多的IP, 使用 INADDR_ANY 意思就是说, 其他主机可以通过服务器主机上的 任意IP:指定port 找到服务器进程实现通信
       </strong>
      </li>
      <li>
       当绑定指定的IP时, 就表示 其他主机只能通过服务器主机上的
       <strong>
        指定IP:指定port
       </strong>
       找到服务器进程实现通信.
      </li>
      <li>
       <strong>
        如果使用其他本机上的IP:指定port,
       </strong>
       服务器是不会响应的. 因为服务器进程 只接收通过指定 IP 发送给服务器进程的信息
      </li>
      <li>
       IP也是要向网络中发送的, 所以要将 IP转换成网络字节序.
       <strong>
        inet_addr()
       </strong>
       则会自动将ip转换为网络字节序
      </li>
      <li>
       <p>
        需要了解的是,
        <strong>
         inet_addr()
        </strong>
        接口 可以将点分十进制字符串类型的IP地址, 转换为 in_addr_t 类型的IP
       </p>
       <p>
        同时, 也会
        <strong>
         <code>
          将转换后的IP自动转换为网络字节序存储形式
         </code>
        </strong>
       </p>
      </li>
     </ul>
    </blockquote>

    <h6 id="4%E3%80%81udp%E6%9C%8D%E5%8A%A1%E5%99%A8%E8%BF%90%E8%A1%8C" name="4%E3%80%81udp%E6%9C%8D%E5%8A%A1%E5%99%A8%E8%BF%90%E8%A1%8C">
     4、udp服务器运行
    </h6>
    <p>
     其具体功能就是循环地监听、接收发送到服务器上的信息
    </p>
    <pre><code class="language-cpp">     void start()
        {
            char inBuffer[1024];//用于储存接收到的信息
            while(1)
            {
                struct sockaddr_in peer;//输出型参数，用于接收对方的主机网络信息
                socklen_t peerlen=sizeof(peer);//输入输出型参数

                ssize_t s=recvfrom(_sockFd,inBuffer,sizeof(inBuffer)-1,0,(struct sockaddr*)&amp;peer,&amp;peerlen);//接收发送到服务器上的信息及发送端的网络信息
                if(s&gt;0)//当字符串结尾
                {
                    inBuffer[s]=0;
                }
                else if(s==-1)//读取失败
                {
                    logMessage(WARINING,"recvfrom() error::%s: %d",strerror(errno),_sockFd);
                    continue;
                }
                //读取成功，除了读取到发过来的信息，还要读到对方的网络信息
                string peerIP=inet_ntoa(peer.sin_addr);//拿到对方的ip
                uint32_t peerPort=ntohs(peer.sin_port);//拿到对方的端口号

                logMessage(NOTICE,"[%s:%d]# %s",peerIP.c_str(),peerPort,inBuffer);//将对方发送过来的信息打印出来
            }
        }    </code></pre>
    <blockquote>
     <p>
      介绍一下接收接口
      <strong>
       recvfrom
      </strong>
     </p>
     <p>
      <strong>
       是用于接收通过套接字发送的数据的系统调用，特别适用于无连接的协议如UDP（用户数据报协议），或者在需要知道数据来源的情况下使用。它不仅可以接收数据，还能获取发送方的信息。
      </strong>
     </p>
     <pre><code class="language-cpp">ssize_t recvfrom(int socket, void* restrict buffer, size_t length, 
int flags, struct sockaddr* restrict address, 
socklen_t* restrict address_len);
</code></pre>
     <ul>
      <li>
       <strong>
        socket
       </strong>
       : 套接字描述符，标识一个已绑定并监听连接的套接字。\
      </li>
      <li>
       <strong>
        buffer
       </strong>
       : 指向用于存储接收到的数据的缓冲区的指针。
      </li>
      <li>
       <strong>
        length
       </strong>
       :缓冲区的最大长度，即可以接收的最大数据量。
      </li>
      <li>
       <strong>
        flags
       </strong>
       : 调用操作的选项，通常设置为0。其他可能的值包括
       <code>
        MSG_PEEK
       </code>
       （查看数据而不实际读取）、
       <code>
        MSG_WAITALL
       </code>
       （等待直到请求的数据量全部到达）等。
      </li>
      <li>
       <strong>
        address
       </strong>
       : （可选）指向一个结构体的指针，该结构体用来保存源地址信息（即发送数据的主机的地址）。如果不需要这个信息，可以传递NULL。
      </li>
      <li>
       <strong>
        address_len
       </strong>
       : （与
       <strong>
        address
       </strong>
       一起使用）在调用时，应设置为
       <strong>
        address
       </strong>
       所指向结构体的大小；返回时，包含实际写入的地址结构体的大小。
      </li>
     </ul>
    </blockquote>

    <h6 id="5%E3%80%81%E5%AE%8C%E6%95%B4%E4%BB%A3%E7%A0%81" name="5%E3%80%81%E5%AE%8C%E6%95%B4%E4%BB%A3%E7%A0%81">
     5、完整代码
    </h6>
    <pre><code class="language-cpp">#include&lt;iostream&gt;
#include&lt;string&gt;
#include&lt;cstring&gt;
#include&lt;cstdlib&gt;

void* recverAndPrint(void* args) 接收并打印服务器消息
{
    while(true)
    {
        int sockFd=*(int*)args;
        char buffer[1024];
        struct sockaddr_in temp;
        socklen_t templen=sizeof(temp);
        ssize_t s=recvfrom(sockFd,buffer,sizeof(buffer),0,(struct sockaddr*)&amp;temp,&amp;templen);
        if(s&gt;0)
        {
            buffer[s]=0;
            cout&lt;&lt;buffer&lt;&lt;endl;//为了方便演示这里保持使用 std::cout，以便重定向到命名管道文件
        }
    }
}

static void Usage(const string proc){

{
    if(argc!=4)
    {
        Usage(argv[0]);
        exit(1);

	string nick_Name = argv[3];
    int sockFd = socket(AF_INET, SOCK_DGRAM, 0);//只有创建了本地的套接字，才能使用recvfrom循环监听别人发向本地接口的信息
    if (sockFd &lt; 0) {
        logMessage(FATAL, "socket() faild:: %s : %d", strerror(errno), sockFd);
        exit(2);
    }
    logMessage(DEBUG, "socket create success: %d", sockFd);
    struct sockaddr_in server;
    bzero(&amp;server, sizeof(server));
    server.sin_family = AF_INET;
    server.sin_port = htons(server_Port);
    server.sin_addr.s_addr = inet_addr(server_IP.c_str());

    pthread_t t;
    pthread_create(&amp;t,nullptr,recverAndPrint,&amp;sockFd);
    while(true)
    {
        std::cerr&lt;&lt;"please enter&gt;&gt;";//使用 std::cerr 输出提示信息

        inBuffer +=tempS;//最终组合发送给服务端
        sendto(sockFd, inBuffer.c_str(), inBuffer.size(), 0,
               (const struct sockaddr*)&amp;server, sizeof(server));
    }
    close(sockFd);
    return 0;
}

    <h2 id="inet_ntoa()%E7%9B%B8%E5%85%B3%E9%97%AE%E9%A2%98%C2%A0" name="inet_ntoa()%E7%9B%B8%E5%85%B3%E9%97%AE%E9%A2%98%C2%A0">
     inet_ntoa()相关问题
    </h2>
    <p>
     在前面向
     <strong>
      <span style="background-color:#cccccc">
       struct sockaddr_in
      </span>
     </strong>
     结构体中填充 IP 地址时使用了一个接口 ：
     <strong>
      inet_addr()
     </strong>
     ,并做了简单介绍，它是将
     <strong>
      点分十进制（我们能看懂的）的 ip 转换成了 uint32_t(int _addt_t)类型的4字节的表示的ip，并且会自动把4字节的ip存储数据设置为网络字节序
     </strong>
     。使用起来是非常方便的，但是还存在着另一个接口：
     <strong>
      <span style="background-color:#cccccc">
       inet_nota()
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="558" src="https://i-blog.csdnimg.cn/direct/a7708f50622c4c15a321b3b4eb4c4a60.png" width="1233"/>
    </p>
    <p>
     该接口的作用是将 struct sockaddr_in 结构体 里面存储的4字节IP,转换成我们看的懂得点分十进制得IP字符串，然后
     <strong>
      以 char*类型返回
     </strong>
     。所以就需要一块空间来存储这个转换后的字符串，事实上这个空间是处于静态区的空间，返回的字符串都会放在这里，且值得注意的是，整个程序中 inet_ntoa() 也就只会返回处于这块的静态区空间。这也就是说，在一个 inet_ntoa()被多次执行的程序中，后面被转换出来的字符串会覆盖掉之前的IP字符串。所以在实际使用中要避免多次使用或者避免直接使用函数返回的指针。
    </p>
    <p>
     但是我们可以使用另一个接口来代替：
     <strong>
      inet_ntop()
     </strong>
    </p>
    <p>
     <img alt="" height="282" src="https://i-blog.csdnimg.cn/direct/9a43f4e2646b401c9b9c9849105b1233.png" width="1861"/>
    </p>
    <p>
     该接口可以将
     <strong>
      网络字节序IPv4或者IPv6地址转换成 点分十进制字符串表示
     </strong>
     。
    </p>
    <ol>
     <li>
      <strong>
       af
      </strong>
      : 地址族，可以是
      <strong>
       AF_INET（IPv4
      </strong>
      ）或
      <strong>
       AF_INET6（IPv6）
      </strong>
      。
     </li>
     <li>
      <strong>
       src
      </strong>
      : 指向网络地址（以网络字节序存储）的指针。对于IPv4来说，这个参数是一个 struct in_addr 类型的指针；对于IPv6，则是 struct in6_addr 类型的指针。
     </li>
     <li>
      <strong>
       dst
      </strong>
      : 目标缓冲区，用于存放转换后的ASCII表示的IP地址。
     </li>
     <li>
      <strong>
       size
      </strong>
      : 目标缓冲区的大小
     </li>
    </ol>
    <h2>
     总结
    </h2>
    <blockquote>
     <p>
      通过上述步骤，UDP服务端和客户端就可以进行基本的数据交换了。由于UDP是无连接的，每次数据传输都是独立的，因此每次调用
      <strong>
       <span style="background-color:#cccccc">
        sendto
       </span>
      </strong>
      和
      <strong>
       <span style="background-color:#cccccc">
        recvfrom
       </span>
      </strong>
      都不需要事先建立连接。这种方式非常适合那些对延迟敏感的应用程序，但也意味着数据包可能丢失或乱序到达。
     </p>
    </blockquote>
