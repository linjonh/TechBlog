    <h2 id="%E7%94%A8%E6%88%B7%E5%88%87%E6%8D%A2" name="%E7%94%A8%E6%88%B7%E5%88%87%E6%8D%A2" style="background-color:transparent">
     用户切换
    </h2>
    <p>
     <strong>
      1、为减少操作复杂度，使用
      <span style="color:#fe2c24">
       sudo -i
      </span>
      指令切换至管理员账户
      <span style="color:#7b7f82">
       <em>
        （未设管理员账户的密码则直接进入）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      2、使用
      <span style="color:#fe2c24">
       id -u
      </span>
      指令判断当前用户是否是管理员账户：是则输出为
      <span style="color:#fe2c24">
       0
      </span>
      ，不是则输出其它数字
     </strong>
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="112" src="https://i-blog.csdnimg.cn/direct/7ec6bc634a9a49269a442ccb9f123e5d.png" width="521"/>
    </p>
    <h2 id="MySQL%E7%9A%84%E5%AE%89%E8%A3%85" name="MySQL%E7%9A%84%E5%AE%89%E8%A3%85">
     MySQL的安装
    </h2>
    <p style="text-align:center">
     <em>
      <strong>
       默认下载目前的最新版本
      </strong>
     </em>
    </p>
    <p>
     <strong>
      1、使用
      <span style="color:#fe2c24">
       dpkg-query -l | grep mysql
      </span>
      指令
      <span style="color:#0d0016">
       列出所有已安装的与 "mysql" 相关的软件包及其版本信息
      </span>
      ，确保系统中没有残留的旧版 MySQL 安装包或冲突的包文件，这样可以避免新安装的版本与现有的 MySQL 版本发生冲突或导致兼容性问题
     </strong>
    </p>
    <p>
     <strong>
      2、使用
      <span style="color:#fe2c24">
       apt update
      </span>
      指令获取最新的软件包信息，以确保你系统上的各种软件包列表是最新的
     </strong>
    </p>
    <p>
     <img alt="" height="209" src="https://i-blog.csdnimg.cn/direct/21fcc59205554b6397bd2f8f524a5849.png" width="887"/>
    </p>
    <p>
     <strong>
      3、使用
      <span style="color:#fe2c24">
       apt install mysql-server
      </span>
      指令安装最新版本的mysql
      <span style="color:#7b7f82">
       <em>
        （记得安装完后再检查一下）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="912" src="https://i-blog.csdnimg.cn/direct/06157f88e8b645779a26976fa1e920ba.png" width="1725"/>
    </p>
    <p>
     <strong>
      4、安装完成后，MySQL 服务应自动启动，可以使用
      <span style="color:#fe2c24">
       systemctl status mysql
      </span>
      指令检查 MySQL 服务的状态
      <span style="color:#7b7f82">
       <em>
        （我查询的时间是17：10而不是自动启动时的16：58，故已安装成功）
       </em>
      </span>
      如果服务器未启动可以使用
      <span style="color:#fe2c24">
       systemctl start mysql
      </span>
      指令启动mysql
     </strong>
    </p>
    <p>
     <img alt="" height="399" src="https://i-blog.csdnimg.cn/direct/7df4b480ebd04347b317d1998752ccf7.png" width="1204"/>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       通过
      </span>
      <span style="color:#fe2c24">
       ps ajx | grep mysql
      </span>
      指令可以检查当前是否有mysql的进程在运行，并列出相关信息
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="75" src="https://i-blog.csdnimg.cn/direct/1ca016987a494fa7aa460e46f621f3fd.png" width="1042"/>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       5、为提高安全性，建议使用
      </strong>
     </span>
     <strong>
      <span style="color:#fe2c24">
       mysql_secure_installation
      </span>
      指令
     </strong>
     <span style="color:#0d0016">
      <strong>
       运行MySQL的安全配置脚本并完成一系列操作
      </strong>
     </span>
     <span style="color:#7b7f82">
      <strong>
       <em>
        （不做赘述自行了解）
       </em>
      </strong>
     </span>
    </p>
    <h2 id="MySQL%E7%9A%84%E5%88%9D%E6%AD%A5%E4%BD%BF%E7%94%A8" name="MySQL%E7%9A%84%E5%88%9D%E6%AD%A5%E4%BD%BF%E7%94%A8" style="background-color:transparent">
     MySQL的初步使用
    </h2>
    <h3 id="%E7%99%BB%E5%BD%95%E4%B8%8E%E9%80%80%E5%87%BA" name="%E7%99%BB%E5%BD%95%E4%B8%8E%E9%80%80%E5%87%BA">
     登录与退出
    </h3>
    <p>
     <strong>
      1、当 MySQL 运行在本机，且监听的默认端口为
      <span style="color:#956fe7">
       3306
      </span>
      时
      <span style="color:#7b7f82">
       <em>
        （安装后不修改mysql的配置文件，则mysql的默认监听端口号一直是3306）
       </em>
      </span>
      直接使用
      <span style="color:#fe2c24">
       mysql -u root -p
      </span>
      指令便，可连接本地的 MySQL 服务器
      <span style="color:#7b7f82">
       <em>
        （默认套接字连接）
       </em>
      </span>
      <span style="color:#0d0016">
       ，如果有密码则输入密码，没密码直接按
      </span>
      <span style="color:#fe2c24">
       Enter
      </span>
      <span style="color:#956fe7">
       （后续必须设置密码）
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="390" src="https://i-blog.csdnimg.cn/direct/99591734807844919f87776c170e1633.png" width="903"/>
    </p>
    <p>
     <strong>
      后续可能使用类似于
      <span style="color:#fe2c24">
       mysql -h 127.0.0.1 -P 3306 -u root -p
      </span>
      <span style="color:#0d0016">
       的
      </span>
      指令登录mysql
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       -h 127.0.0.1：
       <span style="color:#38d8f0">
        指明登录部署了mysql的主机
       </span>
       <em>
        <span style="color:#7b7f82">
         （省略
         <code>
          -h
         </code>
         ，则默认从本地登录（
         <code>
          localhost
         </code>
         ） ）
        </span>
       </em>
      </strong>
     </li>
     <li>
      <strong>
       -P 3306：
       <span style="color:#38d8f0">
        指明要访问的端口号
       </span>
      </strong>
     </li>
     <li>
      <strong>
       -u root：
       <span style="color:#38d8f0">
        指明登录的用户
       </span>
       <em>
        <span style="color:#7b7f82">
         （
         <code>
          root
         </code>
         是 MySQL 的默认超级管理员账户，安装 MySQL 后，默认仅允许
         <code>
          root
         </code>
         从本地登录（
         <code>
          localhost
         </code>
         ））
        </span>
       </em>
      </strong>
     </li>
     <li>
      <strong>
       -p：
       <span style="color:#38d8f0">
        指明需要输入密码
       </span>
       <span style="color:#7b7f82">
        <em>
         （与Linux一样，输入的密码不会显示）
        </em>
       </span>
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      2、使用
      <span style="color:#fe2c24">
       exit
      </span>
      指令退出mysql
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="90" src="https://i-blog.csdnimg.cn/direct/a1162cba5080417eb0c5e889453f71c8.png" width="381"/>
    </p>
    <h4 id="Linux%E5%92%8Cmysql%E4%B8%AD%E7%9A%84%E6%99%AE%E9%80%9A%E7%94%A8%E6%88%B7%E5%92%8Croot%E7%94%A8%E6%88%B7" name="Linux%E5%92%8Cmysql%E4%B8%AD%E7%9A%84%E6%99%AE%E9%80%9A%E7%94%A8%E6%88%B7%E5%92%8Croot%E7%94%A8%E6%88%B7">
     <strong>
      Linux和mysql中的普通用户和root用户
     </strong>
    </h4>
    <p>
     <strong>
      情景假设：
      <span style="color:#fe2c24">
       如果我是LInux系统中的一个普通用户张三，现在我自己进入了mysql，此时我在mysql客户端中的用户名称是什么？我作为mysql的普通用户可以自行提权到mysql的超级用户吗？作为mysql的普通用户我如何访问Linux中的/var/lib/mysql 目录？
      </span>
     </strong>
    </p>
    <blockquote>
     <p>
      1、当以普通 Linux 用户
      <strong>
       张三
      </strong>
      进入 MySQL 客户端，那么你在mysql客户端的用户名称取决于-u选项后的内容，如果是以-u root的方式登录，那么你就是mysql客户端的超级用户、如果以-u xxx的方式登录，那么你就是mysql客户端的一名叫xxx的普通用户、如果不用-u选项直接登录，那么你在mysql客户端就是一个叫张三的普通用户
     </p>
     <p class="img-center">
      <img alt="" height="290" src="https://i-blog.csdnimg.cn/direct/3ab7a2d9c67e45a0b4b1f1b073a8e6b9.png" width="486"/>
     </p>
    </blockquote>
    <blockquote>
     <p>
      2、不能！ MySQL 的权限系统与 Linux 的
      <code>
       sudo
      </code>
      机制不同，普通 MySQL 用户无法随意提升为
      <code>
       root，除非在登录时采用-u root的方式 或者
      </code>
      MySQL
      <code>
       root
      </code>
      账户已授予您更高权限
     </p>
    </blockquote>
    <blockquote>
     <p>
      3、不能！
      <code>
       /var/lib/mysql
      </code>
      <strong>
       归属于 Linux
       <code>
        mysql
       </code>
       用户，
      </strong>
      只有 Linux的
      <code>
       mysql
      </code>
      用户和
      <code>
       root
      </code>
      用户可以访问
      <code>
       /var/lib/mysql，而
      </code>
      Linux的
      <code>
       mysql
      </code>
      用户是当你安装 MySQL 服务器时，系统会自动创建一个
      <strong>
       <code>
        mysql
       </code>
       用户（Linux 用户）
      </strong>
      ，这个用户专门用于运行 MySQL 服务器 (
      <code>
       mysqld
      </code>
      ) 进程
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       优点：
      </strong>
      <strong>
       <span style="color:#fe2c24">
        即使攻击者获取了 MySQL
        <code>
         root
        </code>
        权限，也无法通过它直接控制系统文件或其他服务
       </span>
      </strong>
     </p>
    </blockquote>
    <h3 id="%E6%9F%A5%E7%9C%8B%E3%80%81%E5%88%9B%E5%BB%BA%E4%B8%8E%E4%BD%BF%E7%94%A8" name="%E6%9F%A5%E7%9C%8B%E3%80%81%E5%88%9B%E5%BB%BA%E4%B8%8E%E4%BD%BF%E7%94%A8">
     查看、创建与使用
    </h3>
    <p style="text-align:center">
     <span style="color:#956fe7">
      <strong>
       ​在 MySQL 等数据库系统中，SQL 语句通常以分号（
       <code>
        ;
       </code>
       ）作为结束符
      </strong>
     </span>
    </p>
    <p>
     <strong>
      1、使用
      <span style="color:#fe2c24">
       show databases；
      </span>
      指令列出当前 MySQL 服务器上所有可用的数据库，Linux中的
      <span style="color:#fe2c24">
       /var/lib/mysql
      </span>
      <span style="color:#0d0016">
       目录是 MySQL 在 Linux 系统中的默认数据存储目录
      </span>
      <span style="color:#7b7f82">
       <em>
        （需要Linux中的root权限才能进入）
       </em>
      </span>
      <span style="color:#0d0016">
       ，可以在该目录中找到新建的数据库文件
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="258" src="https://i-blog.csdnimg.cn/direct/2246a14513154acaaff55b5ad3cad8cc.png" width="417"/>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       2、使用
      </span>
      <span style="color:#fe2c24">
       create database xxx；
      </span>
      <span style="color:#0d0016">
       指令创建一个新的名叫xxx的数据库，创建后在Linux中的
      </span>
      <span style="color:#fe2c24">
       /var/lib/mysql
      </span>
      <span style="color:#0d0016">
       目录下可以看到新增的数据库文件，同时
      </span>
      <span style="color:#fe2c24">
       show databases;
      </span>
      <span style="color:#0d0016">
       也可以看到可用的数据库多了一个
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="476" src="https://i-blog.csdnimg.cn/direct/d06f5d808eed43b283c2afa998219f22.png" width="2275"/>
    </p>
    <p>
     <strong>
      3、使用
      <span style="color:#fe2c24">
       use xxx；
      </span>
      指令使用/切换名为xxx的数据库
      <span style="color:#7b7f82">
       <em>
        （类似于cd /xxx），
       </em>
      </span>
      <span style="color:#0d0016">
       然后使用
      </span>
      <span style="color:#fe2c24">
       creata table xxx（字段名 数据类型 [约束]）；
      </span>
      <span style="color:#0d0016">
       指令在xxx数据库中创建一个名为xxx的表，接着使用
      </span>
      <span style="color:#fe2c24">
       insert into xxx（字段名1，字段名2，字段名3...）values（？，？，？...）；
      </span>
      <span style="color:#0d0016">
       指令向表中插数据，最后可以通过selete * from xxx；指令查询名为xxx表中的数据
      </span>
     </strong>
    </p>
    <pre><code class="language-sql">基础语法：
create table 表名 (
  字段名1 数据类型 [约束],
  字段名2 数据类型 [约束],
  ...
);</code></pre>
    <h4 id="%E7%AE%80%E5%8D%95%E5%BA%94%E7%94%A8" name="%E7%AE%80%E5%8D%95%E5%BA%94%E7%94%A8">
     简单应用
    </h4>
    <blockquote>
     <p>
      <strong>
       创建一个统计学生信息的数据库：
       <span style="color:#fe2c24">
        create database school；
       </span>
       创建school数据库、
       <span style="color:#fe2c24">
        use school；
       </span>
       进入该数据库、
       <span style="color:#fe2c24">
        create table student（id int，name varchar（32），gender varchar（2））;
       </span>
       创建存放学生信息的名为student的表、
       <span style="color:#fe2c24">
        select * from school；
       </span>
       查看school数据库中所有的表
      </strong>
     </p>
    </blockquote>
    <p>
     <img alt="" height="595" src="https://i-blog.csdnimg.cn/direct/8294ecb0947c448bb92c3c2fe6008d1b.png" width="868"/>
    </p>

    <h2 id="MySQL%20%E6%95%B0%E6%8D%AE%E5%BA%93%E5%9C%A8%20Linux%20%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F%E4%B8%AD%E7%9A%84%E5%AD%98%E5%82%A8%E7%BB%93%E6%9E%84" name="MySQL%20%E6%95%B0%E6%8D%AE%E5%BA%93%E5%9C%A8%20Linux%20%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F%E4%B8%AD%E7%9A%84%E5%AD%98%E5%82%A8%E7%BB%93%E6%9E%84" style="background-color:transparent">
     MySQL 数据库在 Linux 文件系统中的存储结构
    </h2>
    <p>
     <strong>
      基本概念：
      <span style="color:#0d0016">
       新建xxx数据库时在Linux的/var/lib/mysql下也会新建一个xxx目录、在该数据库中新建yyy表时在 /var/lib/mysql/xxx下会新建yyy文件
      </span>
      <span style="color:#956fe7">
       ，
      </span>
      <span style="color:#0d0016">
       故
      </span>
      <span style="color:#fe2c24">
       数据库本质也是文件，只不过这些文件并不由程序员直接操作，而是交给了mysqld
      </span>
      <span style="color:#fe2c24">
       进行管理，采用数据库是因为虽然普通文件也提供了数据存储的功能，但还需要用户自行管理，而数据库可以让用户只负责提出数据填写的要求即可剩余事情不用关心
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="429" src="https://i-blog.csdnimg.cn/direct/254ab6eef7b543a1aee7c26feedbf07e.png" width="1699"/>
    </p>
    <h2 id="%E6%95%B0%E6%8D%AE%E5%BA%93%E3%80%81%E6%95%B0%E6%8D%AE%E5%BA%93%E6%9C%8D%E5%8A%A1%E3%80%81%E6%95%B0%E6%8D%AE%E5%BA%93%E7%AE%A1%E7%90%86%E7%B3%BB%E7%BB%9F%EF%BC%88%E5%AE%8F%E8%A7%82%EF%BC%89" name="%E6%95%B0%E6%8D%AE%E5%BA%93%E3%80%81%E6%95%B0%E6%8D%AE%E5%BA%93%E6%9C%8D%E5%8A%A1%E3%80%81%E6%95%B0%E6%8D%AE%E5%BA%93%E7%AE%A1%E7%90%86%E7%B3%BB%E7%BB%9F%EF%BC%88%E5%AE%8F%E8%A7%82%EF%BC%89" style="background-color:transparent">
     数据库、数据库服务、数据库管理系统（宏观）
    </h2>
    <p>
     <strong>
      基本概念：
      <span style="color:#fe2c24">
       数据库指以特定结构存储在磁盘或内存中的数据集合
      </span>
      <span style="color:#7b7f82">
       <em>
        （一种概念）
       </em>
      </span>

    <h2 id="SQL%E8%AF%AD%E8%A8%80%E5%8F%8A%E5%85%B6%E5%88%86%E7%B1%BB" name="SQL%E8%AF%AD%E8%A8%80%E5%8F%8A%E5%85%B6%E5%88%86%E7%B1%BB" style="background-color:transparent">
     SQL语言及其分类DDL、DCL、DML
    </h2>
    <p>
     <strong>
      基本概念：
      <span style="color:#fe2c24">
       SQL
      </span>
      <span style="color:#7b7f82">
       <em>
        （Structured Query Language，结构化查询语言）
       </em>
      </span>
      <span style="color:#fe2c24">
       是一种用于操作数据库的标准语言，可以将 SQL 语言分为 DDL
      </span>
      <span style="color:#956fe7">
       （Data Definition Language）
      </span>
      <span style="color:#7b7f82">
       <em>
        （数据定义语言，负责“创建和修改数据库结构”）
       </em>
      </span>
      <span style="color:#fe2c24">
       、DML
      </span>
      <span style="color:#956fe7">
       （Data Manipulation Language）
      </span>
      <span style="color:#7b7f82">
       <em>
        （数据操作语言，负责“对数据进行增删改查”
       </em>
      </span>
      <span style="color:#7b7f82">
       <em>
        ）
       </em>
      </span>
      <span style="color:#fe2c24">
       、DCL
      </span>
      <span style="color:#956fe7">
       （Data Control Language）
      </span>
      <span style="color:#7b7f82">
       <em>
        （数据控制语言，负责“用户权限管理”
       </em>
      </span>
      <span style="color:#7b7f82">
       <em>
        ）
       </em>
      </span>
      <span style="color:#fe2c24">
       、TCL
      </span>
      <span style="color:#956fe7">
       （Transaction Control Language）
      </span>
      <span style="color:#7b7f82">
       <em>
        （事务控制语言，负责“事务控制”
       </em>
      </span>
      <span style="color:#7b7f82">
       <em>
        ）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       注意事项：
      </span>
      <span style="color:#fe2c24">
       SQL 在不同的DBMS中的差异，可以类比于 C++ 和 Java 等编程语言的关系。即基本语法和核心概念相似，但处理时都是按照自己的SQL解析规则进行的
      </span>
      <span style="color:#7b7f82">
       <em>
        （C++和JAVA都基于C，但是在
       </em>
      </span>
      <span style="color:#4da8ee">
       <em>
        编译器
       </em>
      </span>
      <span style="color:#7b7f82">
       <em>
        处理指令时不会把它们的指令都转换为C语言再执行，类似地，SQL Server 不会把
        <code>
         SELECT TOP 10
        </code>
        转换成 MySQL 的
        <code>
         LIMIT 10
        </code>
        语句再执行，而是直接按照自己的
       </em>
      </span>
      <span style="color:#4da8ee">
       <em>
        SQL 解析器
       </em>
      </span>
      <span style="color:#7b7f82">
       <em>
        处理指令）
       </em>
      </span>
     </strong>
    </p>

     字符集和校验规则
    </h3>
    <p>
     <strong>
      字符集：
      <span style="color:#fe2c24">
       决定数据库如何存储和表示文本数据，即一个字符在数据库中如何编码成二进制数据
      </span>
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       <span style="color:#fe2c24">
        比如UTF-8字符集
       </span>
       <span style="color:#7b7f82">
        <em>
         （多字节字符集，每个字符占多个字节）
        </em>
       </span>
       <span style="color:#fe2c24">
        能存储几乎所有语言的字符，适用于国际化应用，如支持中、英、法等多种语言的系统；而Latin1字符集
       </span>
       <span style="color:#7b7f82">
        <em>
         （单字节字符集，每个字符只占用一个字节）
        </em>
       </span>
       <span style="color:#fe2c24">
        只适用于西欧语言
       </span>
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      校验集：
      <span style="color:#fe2c24">
       定义字符串的比较和排序规则，即数据库如何判断两个字符串是否相等，或如何进行排序
      </span>
     </strong>
    </p>
    <blockquote>
     <p>
      <span style="color:#fe2c24">
       <strong>
        比如
        <code>
         utf8_bin校验集
        </code>
        进行二进制比较，即完全按照字符的编码值来比较和排序
       </strong>
      </span>
      <span style="color:#7b7f82">
       <em>
        <strong>
         （"Hello" 和 "hello" 被认为是不同的）
        </strong>
       </em>
      </span>
      <strong>
       <span style="color:#fe2c24">
        ；而MySQL的默认校验集
       </span>
       <code>
        <span style="color:#fe2c24">
         utf8_general_ci对大小写不敏感
        </span>
        <span style="color:#7b7f82">
         <em>
          （
         </em>
        </span>
       </code>
      </strong>
      <span style="color:#7b7f82">
       <em>
        <strong>
         "Hello" 和 "hello" 视为相同
        </strong>
        <strong>
         <code>
          ）
         </code>
        </strong>

        （升序从小到大，降序从大到小）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       注意事项：
      </span>
      <span style="color:#fe2c24">
       如果未指定排序顺序，
       <code>
        ORDER BY
       </code>
       默认使用升序
      </span>
     </strong>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       1、使用
      </strong>
     </span>
     <strong>
      <span style="color:#fe2c24">
       select * from school order by age；
      </span>
      <span style="color:#0d0016">
       指令查询
       <code>
        school
       </code>
       表，按年龄升序排列
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="342" src="https://i-blog.csdnimg.cn/direct/3ba997a9c3514ca69295af33bb1c367d.png" width="490"/>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       2、使用
      </strong>
     </span>
     <span style="color:#fe2c24">
      <strong>
       select * from school order by score DESC；
      </strong>
     </span>
     <span style="color:#0d0016">
      <strong>
       指令查询
       <code>
        school
       </code>
       表，按分数降序排列
      </strong>
     </span>
    </p>
    <p style="text-align:center">
     <img alt="" height="270" src="https://i-blog.csdnimg.cn/direct/f5373db440ec41839aba347eee4e3548.png" width="483"/>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       3、使用
      </strong>
     </span>
     <span style="color:#fe2c24">
      <strong>
       select * from school order by height DESC，weight ASC；
      </strong>
     </span>
     <strong>
      首先按
      <font face="monospace">
       身高
      </font>
      升序排列；如果身高相同，再按
      <font face="monospace">
       体重
      </font>
      降序排列
      <span style="color:#7b7f82">
       <em>
        （受到身高排序的影响，提供的排序是混乱的）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="325" src="https://i-blog.csdnimg.cn/direct/83a40fd3c04b4ba3a86ca7c9bc2801e5.png" width="561"/>
    </p>
    <h3 id="%E6%95%B0%E6%8D%AE%E5%BA%93%E7%9A%84%E5%A4%87%E4%BB%BD%E4%B8%8E%E6%81%A2%E5%A4%8D" name="%E6%95%B0%E6%8D%AE%E5%BA%93%E7%9A%84%E5%A4%87%E4%BB%BD%E4%B8%8E%E6%81%A2%E5%A4%8D" style="background-color:transparent">
     数据库的备份与恢复
    </h3>
    <p>
     <strong>
      使用工具：
      <span style="color:#fe2c24">
       mysqldump
      </span>
      <span style="color:#7b7f82">
       <em>
        （安装MySQL时自带）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <strong>
      备份单个数据库：使用
      <span style="color:#fe2c24">
       mysqldump -u root -p school &gt; school.sql；
      </span>
      指令将school数据库备份到school.sql文件中
      <span style="color:#7b7f82">
       <em>
        （.sql文件被放在Linux中进入mysql命令行前的目录中保存，也可使用绝对路径进行指明）
       </em>
      </span>
     </strong>
    </p>
    <p style="text-align:center">
     <span style="color:#956fe7">
      <strong>
       -u root：是为了能由足够的权限进行备份
      </strong>
     </span>
    </p>
    <p style="text-align:center">
     <span style="color:#956fe7">
      <strong>
       -p密码：指定连接当前MySQL的密码，密码要紧跟-p
      </strong>
     </span>
    </p>
    <p>
     <strong>
      备份多个数据库：使用
      <span style="color:#fe2c24">
       mysqldump -u root -p --databases school mm &gt; all.sql；
      </span>
      指令将school数据库和mm数据库统一备份到all.sql文件中
     </strong>
    </p>
    <p>
     <strong>
      备份数据库中的某些表：使用
      <span style="color:#fe2c24">
       mysqldump -u root -p school students teacher &gt; school_tables_backup.sql；
      </span>
      指令将school数据库中的两个表students和teachers备份到
      <span style="color:#0d0016">
       school_tables_backup.sql文
      </span>
      件中
     </strong>
    </p>
    <p>
     <strong>
      仅备份数据库中的结构：使用
      <span style="color:#fe2c24">
       mysqldump -u root -p --no-data school &gt; school_structure.sql；
      </span>
      指令将school数据库中以不包含数据的方式备份到school_structure.sql文件中
     </strong>
    </p>
    <p>
     <strong>
      恢复备份：使用
      <span style="color:#fe2c24">
       mysql -u root -p school &lt; school.sql；
      </span>
      指令将备份恢复到school数据库
      <span style="color:#a2e043">
       （恢复前要先创建一个空间school数据库）
      </span>
     </strong>
    </p>
    <p>
     <strong>
      注意事项：
      <span style="color:#fe2c24">
       备份的是该数据库以前历史上的所有操作，包括操作中涉及的数据，恢复时会把这些操作全部再运行一次，这样就得到一份一模一样的数据库
      </span>
     </strong>
    </p>
    <h3 id="%E6%9F%A5%E7%9C%8B%E6%95%B0%E6%8D%AE%E5%BA%93%E7%9A%84%E8%BF%9E%E6%8E%A5%E7%8A%B6%E6%80%81" name="%E6%9F%A5%E7%9C%8B%E6%95%B0%E6%8D%AE%E5%BA%93%E7%9A%84%E8%BF%9E%E6%8E%A5%E7%8A%B6%E6%80%81" style="background-color:transparent">
     查看数据库的连接状态
    </h3>
    <p>
     <strong>
      指令：
      <span style="color:#fe2c24">
       show processlist；
      </span>
      <span style="color:#7b7f82">
       <em>
        （发现数据库运行缓慢时使用）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="217" src="https://i-blog.csdnimg.cn/direct/19f43f9c104c40ff8c3db17ec6ce1d34.png" width="1199"/>
    </p>
    <ul>
     <li>
      <strong>
       Id
      </strong>
      ：进程的唯一标识符。例如，
      <code>
       5
      </code>
      和
      <code>
       23
      </code>
      是当前活动进程的 ID
     </li>
     <li>
      <strong>
       User
      </strong>
      ：执行查询的用户。比如
      <code>
       event_scheduler
      </code>
      和
      <code>
       root
      </code>
     </li>
     <li>
      <strong>
       Host
      </strong>
      ：发起连接的主机。比如，两个进程的主机都是
      <code>
       localhost
      </code>
     </li>
     <li>
      <strong>
       db
      </strong>
      ：当前进程连接的数据库。在这张图中，
      <code>
       event_scheduler
      </code>
      进程未连接数据库，而
      <code>
       root
      </code>
      进程连接了
      <code>
       school
      </code>
      数据库
     </li>
     <li>
      <strong>
       Command
      </strong>
      ：表示进程当前正在执行的命令类型。例如，
      <code>
       Daemon
      </code>
      表示事件调度器，
      <code>
       Query
      </code>
      表示正在执行 SQL 查询
     </li>
     <li>
      <strong>
       Time
      </strong>
      ：进程已执行的时间（以秒为单位）。例如，
      <code>
       event_scheduler
      </code>
      已运行了
      <code>
       99874
      </code>
      秒，而
      <code>
       root
      </code>
      进程的执行时间为
      <code>
       0
      </code>
      秒
     </li>
     <li>
      <strong>
       State
      </strong>
      ：进程当前的状态。例如，
      <code>
       Waiting on empty queue
      </code>
      表示事件调度器在等待一个空队列，
      <code>
       init
      </code>
      表示
      <code>
       root
      </code>
      进程刚开始初始化查询
     </li>
     <li>
      <strong>
       Info
      </strong>
      ：显示有关进程的附加信息。
      <code>
       event_scheduler
      </code>
      进程没有附加信息，而
      <code>
       root
      </code>
      进程正在执行
      <code>
       show processlist
      </code>
      查询
     </li>
    </ul>
    <h2 id="MySQL%E4%B8%AD%E8%A1%A8%E7%9A%84%E5%B8%B8%E8%A7%81%E6%93%8D%E4%BD%9C" name="MySQL%E4%B8%AD%E8%A1%A8%E7%9A%84%E5%B8%B8%E8%A7%81%E6%93%8D%E4%BD%9C" style="background-color:transparent">
     MySQL中表的常见操作
    </h2>
    <h3 id="%E8%A1%A8%E7%9A%84%E5%88%9B%E5%BB%BA" name="%E8%A1%A8%E7%9A%84%E5%88%9B%E5%BB%BA" style="background-color:transparent">
     表的创建
    </h3>
    <p>
     <strong>
      完整语法：
     </strong>
    </p>
    <pre><code class="language-cpp">create table 表名 (
    列名1 列中所存储数据的数据类型 列的约束条件用于限制列中的数据 comment '对于列中数据的提示语'，
    列名2 列中所存储数据的数据类型 列的约束条件用于限制列中的数据 comment '对于列中数据的提示语'，
    列名3 列中所存储数据的数据类型 列的约束条件用于限制列中的数据 comment '对于列中数据的提示语'，
    ...
    [表级约束]
)charste=表的字符集 collate 表的校验规则 engine 存储引擎;
</code></pre>
    <ul>
     <li>
      <span style="color:#fe2c24">
       <strong>
        如果没有设定字符集和校验规则，那么就按照当前表所在数据库的字符集和校验规则
       </strong>
      </span>
     </li>
    </ul>
    <p>
     <img alt="" height="513" src="https://i-blog.csdnimg.cn/direct/71c5a74f0fe64f42b82447131955b16f.png" width="1141"/>
    </p>
    <p>
     <strong>
      采用不同存储引擎建立的表在Linux中文件的个数也不同，user1表采用MyISAM存储引擎，user2表采用InnoDB存储引擎，前者的对应文件个数为2.MYD和.MYI，后者为1.ibd
     </strong>
    </p>
    <p>
     <img alt="" height="211" src="https://i-blog.csdnimg.cn/direct/0c28a6a00bee4003be8822438e564c98.png" width="682"/>
    </p>
    <h3 id="%E8%A1%A8%E7%9A%84%E6%9F%A5%E7%9C%8B" name="%E8%A1%A8%E7%9A%84%E6%9F%A5%E7%9C%8B" style="background-color:transparent">
     表的查看
    </h3>
    <p>
     <strong>
      1、查看当前数据库的所有表：
      <span style="color:#fe2c24">
       show tables；
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="246" src="https://i-blog.csdnimg.cn/direct/560004654e344ca7b1b93d28aab23aed.png" width="288"/>
    </p>
    <p>
     <strong>
      2、查看某个表的结构：
      <span style="color:#fe2c24">
       desc 表名；
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="461" src="https://i-blog.csdnimg.cn/direct/c251f812d91943548188651b56c1ebec.png" width="387"/>
    </p>
    <ul>
     <li>
      <strong>
       Field：
       <span style="color:#fe2c24">
        字段/列名字
       </span>
      </strong>
     </li>
     <li>
      <strong>
       Type：
       <span style="color:#fe2c24">
        字段类型
       </span>
      </strong>
     </li>
     <li>
      <strong>
       Null：
       <span style="color:#fe2c24">
        是否允许该字段为空。值可以是
        <code>
         YES
        </code>
        或
        <code>
         NO
        </code>
       </span>
      </strong>
     </li>
     <li>
      <strong>
       Key：
       <span style="color:#fe2c24">
        索引类型
       </span>
      </strong>
     </li>
     <li>
      <strong>
       Default：
       <span style="color:#fe2c24">
        字段的默认值。如果没有插入数据时指定该字段的值，数据库会使用该默认值
       </span>
      </strong>
     </li>
     <li>
      <strong>
       Extra：
       <span style="color:#fe2c24">
        扩充
       </span>
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      3、查看创建表时的详细操作信息：
      <span style="color:#fe2c24">
       show create table 表名 \G
      </span>
      <span style="color:#7b7f82">
       <em>
        （这里不用冒号）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="408" src="https://i-blog.csdnimg.cn/direct/ed6fee50ad104f2b9f7768e0a57c756f.png" width="403"/>
    </p>
    <ul>
     <li>
      <strong>
       虽然在建表时命令输入可能不够规范和整齐，但当 MySQL 服务器接收到客户端请求后，会对语法进行分析和整理，最终在查询结果中展示出整洁的命令
      </strong>
     </li>
    </ul>
    <h3 id="%E8%A1%A8%E7%9A%84%E6%8F%92%E5%85%A5" name="%E8%A1%A8%E7%9A%84%E6%8F%92%E5%85%A5" style="background-color:transparent">
     表的插入
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       指令：
      </strong>
     </span>
     <span style="color:#fe2c24">
      <strong>
       insert into 表名 values（数据1，数据2...）；
      </strong>
     </span>
     <span style="color:#7b7f82">
      <strong>
       <em>
        （该表必须已经存在）
       </em>
      </strong>
     </span>
    </p>
    <p class="img-center">
     <img alt="" height="282" src="https://i-blog.csdnimg.cn/direct/589ff84f623542e3868bb8f776443b0a.png" width="505"/>
    </p>
    <p class="img-center">
     <img alt="" height="234" src="https://i-blog.csdnimg.cn/direct/c6df7a62567145188d708b0337710ee2.png" width="504"/>
    </p>
    <h3 id="%E8%A1%A8%E7%9A%84%E4%BF%AE%E6%94%B9" name="%E8%A1%A8%E7%9A%84%E4%BF%AE%E6%94%B9" style="background-color:transparent">
     表的修改
    </h3>
    <p>
     <strong>
      1、修改表名：
      <span style="color:#fe2c24">
       alter table 旧表名 rename to 新表名；
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="363" src="https://i-blog.csdnimg.cn/direct/5cb9f0e092054e90b185c513f0d532c3.png" width="373"/>
    </p>
    <p>
     <strong>
      2、 添加字段：
      <span style="color:#fe2c24">
       alter table 表名 add 新字段名 字段的数据类型 comment '图片路径' after Field；
      </span>
      <span style="color:#7b7f82">
       <em>
        （不加after Field也行，然后新字段名就会在birthday后了）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="928" src="https://i-blog.csdnimg.cn/direct/42e1e513e2754e62bc3b5600cc9a8c9a.png" width="911"/>
    </p>
    <blockquote>
     <p>
      <strong>
       补充：
       <span style="color:#fe2c24">
        如果我们此时再show create table user \G会发现虽然是后来新增的插入但是仍然会被整齐的放入（）中
       </span>
      </strong>
     </p>
     <p>
      <img alt="" height="331" src="https://i-blog.csdnimg.cn/direct/3358140e8a784eadbfad2a7d63bb2842.png" width="812"/>
     </p>
    </blockquote>
    <p>
     <strong>
      3、 修改字段的属性：
      <span style="color:#fe2c24">
       alter table 表名 modify 字段名 属性 ...；
      </span>
      <span style="color:#7b7f82">
       <em>
        （..表示后续的一些内容要补全防止覆盖掉原来后面的comment等内容）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="1107" src="https://i-blog.csdnimg.cn/direct/dc687ef976764c1f94845576420f6ca1.png" width="1735"/>
    </p>
    <p>
     <strong>
      4、删除字段：
      <span style="color:#fe2c24">
       alter table 表名 drop 字段名；
      </span>
      <span style="color:#7b7f82">
       <em>
        （删除后该字段原本的数据都会被删除，要小心）
       </em>
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="987" src="https://i-blog.csdnimg.cn/direct/f87ce3eb863548599ed0201d7e7c5d29.png" width="1605"/>
    </p>
    <p>
     <strong>
      5、修改字段名：
      <span style="color:#fe2c24">
       alter table 表名 change 旧字段名 新字段名 数据类型...；
      </span>
      <span style="color:#7b7f82">
       <em>
        （修改后的新字段需要重新完整定义一次，否则会报错）
       </em>
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="487" src="https://i-blog.csdnimg.cn/direct/c3a9f1f5d50e456ba34461f819395dff.png" width="561"/>
    </p>
    <h3 id="%E8%A1%A8%E7%9A%84%E5%88%A0%E9%99%A4" name="%E8%A1%A8%E7%9A%84%E5%88%A0%E9%99%A4" style="background-color:transparent">
     表的删除
    </h3>
    <p>
     <strong>
      指令：
      <span style="color:#fe2c24">
       drop  table 表名；
      </span>
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="303" src="https://i-blog.csdnimg.cn/direct/eb473b00cd884154acba404c8a11766e.png" width="375"/>
    </p>
    <p style="text-align:center">
     <strong>
      <span style="color:#fe2c24">
       ！！不要轻易进行表的修改和删除！！
      </span>
     </strong>
    </p>
