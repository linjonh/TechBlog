    <h2>
     <a id="_1">
     </a>
     问题描述
    </h2>
    <p>
     因为换工作，需要新配置的环境，所以遇到了各种问题，例如下面这个：
    </p>
    <p>
     <img alt="Could not determine java version from '11' 错误表现" src="https://i-blog.csdnimg.cn/blog_migrate/41d46fe490b7d63eadcfcd3249830ff5.png"/>
    </p>
    <p>
     发生错误的相关环境配置
    </p>
    <ul>
     <li>
      JDK 11
     </li>
     <li>
      Gradle 4.4
     </li>
    </ul>
    <p>
     如果不知道自己安装的 Java 版本，可以根据一下命令查看：
    </p>
    <p>
     Mac :
    </p>
    <pre><code>#查看版本
java -version

#查看安装位置
/usr/libexec/java_home -V

</code></pre>
    <p>
     Windows:
    </p>
    <pre><code>
#查看版本
java -version

#查看安装位置
java -verbose

</code></pre>
    <p>
     项目的 Gradle 版本，可以根据下面方式查看：
    </p>
    <ol>
     <li>
      /gradle/wapper/gradle-wrapper.properties 文件查看：
     </li>
    </ol>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/b870e926aeaac43d7d76a2b2b6d2584e.png"/>
    </p>
    <ol start="2">
     <li>
      Android Studio-&gt;Project Structure 查看：
     </li>
    </ol>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/e7b77d880388ff253f2543110ef8ba6c.png"/>
    </p>
    <h2>
     <a id="_52">
     </a>
     问题解决
    </h2>
    <p>
     问题核心原因是 Grade 和 JDK 的版本不兼容，因此有两个解决方法：
    </p>
    <ul>
     <li>
      <p>
       升级 Gradle
      </p>
      <p>
       <a href="https://gradle.org/releases/" rel="nofollow">
        Gradle 官网
       </a>
       下载了最新版本 4.10.2,可以解决这个问题。
      </p>
      <p>
       <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/d8d30b0f8cf487af3850ac00b2f2fb2a.png"/>
      </p>
     </li>
     <li>
      <p>
       降级 JDK
      </p>
      <p>
       <a href="https://www.oracle.com/technetwork/java/javase/downloads/index.html" rel="nofollow">
        ORACLE 官网
       </a>
       下载 JDK 1.8.0 ,也可以解决这个问题。
      </p>
      <p>
       <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/ad21f2ecc6e7dced15eabe4c3f28c548.png"/>
      </p>
     </li>
    </ul>
    <p>
     事实情况下，我使用 Gradle 的目的就是管理 Android 项目，而且项目的 Gradle 版本一般不会轻易修改，所以我采用的是降级 JDK 的方法。大家可以根据自己的情况选择。
    </p>
