    <h3>
     <a id="1_5">
     </a>
     1、各种定义
    </h3>
    <ul>
     <li>
      操作系统定义
      <ul>
       <li>
        管理计算机的
        <strong>
         硬件
        </strong>
        和
        <strong>
         软件
        </strong>
        资源， 能对各类作业进行
        <strong>
         调度
        </strong>
        ，方便用户使用计算机的
        <strong>
         程序集合
        </strong>
        。
       </li>
       <li>
        操作系统运行在
        <strong>
         内核态（也叫管态，核心态）
        </strong>
        。在这个状态，操作系统可以对所有的硬件访问。
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       注意
      </strong>
      ：操作系统是最基础的
      <strong>
       软件
      </strong>
      。
     </p>
    </blockquote>
    <ul>
     <li>
      用户接口程序定义
      <ul>
       <li>
        用户接口程序有
        <code>
         shell和GUI
        </code>
        <ul>
         <li>
          shell：用户与操作系统交互的程序，基于
          <strong>
           文本
          </strong>
          的是shell。
         </li>
         <li>
          GUI：用户与操作系统交互的程序，基于
          <strong>
           图标
          </strong>
          的是GUI。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       注意
      </strong>
      ：用户接口程序并不属于操作系统。
     </p>
    </blockquote>
    <ul>
     <li>
      信息
      <ul>
       <li>
        位（bit）+ 上下文（context）= 信息
        <ul>
         <li>
          位（Bit） 是计算机中最小的数据单位，表示一个二进制的
          <strong>
           0 或 1
          </strong>
          。
         </li>
         <li>
          上下文 是数据所处的环境或规则，决定了
          <strong>
           如何解释这些位
          </strong>
          。
         </li>
        </ul>
       </li>
       <li>
        ASCII码构成的文件就是文本文件
       </li>
       <li>
        其余是二进制文件
       </li>
      </ul>
     </li>
     <li>
      内核态和用户态
      <ul>
       <li>
        内核态：
        <ul>
         <li>
          操作系统的“核心大脑”，权限高但需谨慎操作。
         </li>
        </ul>
       </li>
       <li>
        用户态：
        <ul>
         <li>
          普通程序的“沙箱”，权限受限但
          <strong>
           安全
          </strong>
          。
          <ul>
           <li>
            所以操作系统会把
            <strong>
             一部分代码
            </strong>
            放在
            <strong>
             内核态
            </strong>
            ，
            <strong>
             另一部分
            </strong>
            放在
            <strong>
             用户态
            </strong>
            来
            <strong>
             保证安全
            </strong>
            。
           </li>
          </ul>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      程序接口
      <ul>
       <li>
        普通用户无法直接使用程序接口，而是使用
        <code>
         系统调用
        </code>
        来用程序接口。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0098a37e975a46eb8637315683cb6928.png"/>
    </p>
    <h3>
     <a id="2_36">
     </a>
     2、程序被其他程序翻译成不同格式
    </h3>
    <p>
     在linux的gcc编译器下。我们有一些对文件的操作。
     <strong>
      操作系统和编译器是辅助关系
     </strong>
     。
    </p>
    <blockquote>
     <p>
      这些操作是【巧妙记忆：ESc iso】
      <br/>
      ①预处理阶段：
     </p>
     <ul>
      <li>
       使用
       <code>
        gcc -E xxx.c -o xxx.i
       </code>
       命令
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：处理 #include、#define、#ifdef 等预处理指令。
      </li>
     </ul>
     <p>
      ②编译阶段：
     </p>
     <ul>
      <li>
       使用
       <code>
        gcc -S xxx.i -o xxx.s
       </code>
       命令
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：将预处理后的代码转换为汇编代码。
      </li>
     </ul>
     <p>
      ③汇编阶段：
     </p>
     <ul>
      <li>
       使用
       <code>
        gcc -c xxx.s -o xxx.o
       </code>
       命令
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：将汇编代码转换为机器码（二进制目标文件）。
      </li>
     </ul>
     <p>
      ④链接阶段：
     </p>
     <ul>
      <li>
       使用
       <code>
        gcc xxx.o -o xxxx
       </code>
       命令
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：将目标文件与库文件（如 libc.so）链接，生成可执行文件。
      </li>
     </ul>
    </blockquote>
    <p>
     如果我们使用
     <code>
      world.c
     </code>
     文件进行以上操作。那这4个阶段将构成编译系统（compilation system）。那么命令的执行过程如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b24b729cc9d744e0a5521a47311c4c4c.png"/>
    </p>
    <h3>
     <a id="3_57">
     </a>
     3、面临的问题
    </h3>
    <p>
     当一个代码输入到操作系统上，我们的硬盘【不同的用户，硬盘的来源是不同的，材质会不同等等】该如何读取呢？等等。
     <br/>
     所以我们就需要用到操作系统的4个重大部分，也是未来学习的过程中重点知识。
    </p>
    <ul>
     <li>
      <strong>
       运行任务的管理
      </strong>
      ：进程和线程、调度和同步机制
     </li>
     <li>
      <strong>
       运行任务的存储管理
      </strong>
      ：内存管理
     </li>
     <li>
      <strong>
       运行任务的外设管理
      </strong>
      ：I/O设备
     </li>
     <li>
      <strong>
       存储数据设备的管理
      </strong>
      ：文件管理
     </li>
    </ul>

    <h3>
     <a id="1CPU_67">
     </a>
     1、CPU
    </h3>
    <p>
     中央处理单元（CPU），简称
     <strong>
      处理器
     </strong>
     。
    </p>
    <ul>
     <li>
      <strong>
       CPU作用
      </strong>
      ：处理和执行指令。
     </li>
     <li>
      <strong>
       CPU的寄存器
      </strong>
      ：存储下一条要执行的指令。
      <ul>
       <li>
        通用寄存器
       </li>
       <li>
        <strong>
         程序计数器
        </strong>
        （
        <strong>
         PC
        </strong>
        ）
        <ul>
         <li>
          作用：存储下一条要执行的指令的
          <strong>
           内存地址
          </strong>
          。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         堆栈指针
        </strong>
        （
        <strong>
         SP
        </strong>
        ）
        <ul>
         <li>
          作用：
          <strong>
           指向当前堆栈的顶部地址
          </strong>
          ，管理函数调用时的栈操作。
          <strong>
           能保护信息和恢复信息
          </strong>
          。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         指令寄存器（IR）
        </strong>
        <ul>
         <li>
          作用：存放
          <strong>
           当前执行指令
          </strong>
          。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         状态寄存器（FLAGS）
        </strong>
        <ul>
         <li>
          作用：记录
          <strong>
           运算结果状态
          </strong>
          （如零标志ZF、进位标志CF）。
         </li>
         <li>
          PSW：并不是一直需要保存，而是运行状态才需要。在记录状态【已经没有运行了】不用。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       CPU的算术逻辑单元ALU
      </strong>
      <ul>
       <li>
        作用：执行算术和逻辑运算。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       指令集架构
      </strong>
      <ul>
       <li>
        处理器想要处理一条指令，也需要通过一定的规则，这个规则就是指令集架构。
       </li>
       <li>
        指令类型如下：
        <ul>
         <li>
          加载指令：用于将
          <strong>
           数据从内存加载到寄存器
          </strong>
          中。
         </li>
         <li>
          存储指令：用于将
          <strong>
           数据从寄存器存回到内存
          </strong>
          中。
         </li>
         <li>
          运算指令：用于对
          <strong>
           来自寄存器和内存的操作数进行运算
          </strong>
          ，例如 add 指令将两个操作数相加并将结果保存在寄存器或内存中。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       CPU的流水线
      </strong>
      <ul>
       <li>
        当CPU正在
        <strong>
         执行
        </strong>
        第N条指令时，它可以同时
        <strong>
         解码
        </strong>
        第N+1条指令，并
        <strong>
         读取
        </strong>
        第N+2条指令，这种形式就叫
        <code>
         流水线
        </code>
        。
       </li>
       <li>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/21fb07b7b0434b789f89e27f7c4f4740.png"/>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       CPU的超标量
      </strong>
      <ul>
       <li>
        <strong>
         CPU中可以有多个执行单元，例如一个做加法运算，一个做乘法运算。在它们做运算的时候，把许多取指单元和解码单元放入缓存区（cache）中，等到某一个执行单元有空了，就把缓存区中的它们拿一个出来执行
        </strong>
        。虽然看起来指令乱序了，实际上工作人员肯定设计方法来有序进行。
       </li>
       <li>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/13eeeb3fa8a84154a16c29829d310805.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="2_95">
     </a>
     2、内存
    </h3>
    <p>
     <strong>
      理想情况
     </strong>
     下，内存应该是非常快速的（比执行一条指令还要快，以避免拖慢CPU的执行效率），同时容量足够大且成本低廉。然而，当前的技术手段无法同时满足这三个要求。因此，
     <strong>
      存储系统采用了一种分层次的结构来解决这个问题
     </strong>
     。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0e5b2c5e66a549968a94f86f19d3905a.png"/>
    </p>
    <h4>
     <a id="21__98">
     </a>
     2.1、
     <strong>
      寄存器
     </strong>
    </h4>
    <ul>
     <li>
      寄存器与CPU同样的材质，跟CPU一样快
     </li>
    </ul>
    <h4>
     <a id="22_Cache_100">
     </a>
     2.2、
     <strong>
      高速缓存（Cache）
     </strong>
    </h4>
    <ul>
     <li>
      位于CPU与主存（内存）之间的超高速临时存储器，由SRAM构成，用于
      <strong>
       减少CPU访问数据的延迟
      </strong>
      。
     </li>
     <li>
      <strong>
       局部性原理
      </strong>
      ：
      <ul>
       <li>
        是指：CPU访问存储器的时候，无论是读指令还是存取数据，所访问的存储单元都被区域聚集到一个连续区域中了。也就是说
        <strong>
         程序具有访问局部区域里的数据和代码的趋势
        </strong>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="23_104">
     </a>
     2.3、
     <strong>
      主存
     </strong>
    </h4>
    <ul>
     <li>
      主存是指计算机中用
      <strong>
       于临时存储正在运行的程序和数据的硬件部件
      </strong>
      ，属于
      <strong>
       RAM（随机存取存储器）
      </strong>
      。断电后数据丢失（易失性）。
     </li>
     <li>
      <strong>
       ROM
      </strong>
      （
      <strong>
       只读存储器
      </strong>
      ） 断电后不数据丢失，一旦ROM存储了数据，那么数据就不能被修改了。
     </li>
     <li>
      EEPROM（电可擦可编程只读存储器）和 闪存（Flash Memory）与ROM不同，
      <strong>
       支持数据的擦除和重新写入
      </strong>
      【
      <strong>
       重写数据来修正程序错误
      </strong>
      】。
     </li>
     <li>
      CMOS（Complementary Metal-Oxide Semiconductor，互补金属氧化物半导体） 存储器
      <ul>
       <li>
        作用：
        <strong>
         存储
        </strong>
        BIOS/UEFI设置（如启动顺序、日期时间、硬件参数）。
        <strong>
         确保即使在电源关闭后也能持续追踪这些关键数据
        </strong>
        。
       </li>
       <li>
        特性：易失性存储器，依赖**纽扣电池（CR2032）**维持数据（
        <strong>
         断电后不丢失
        </strong>
        ）。
       </li>
      </ul>
     </li>
    </ul>

    <h5>
     <a id="241HDD_114">
     </a>
     2.4.1、机械磁盘（HDD）
    </h5>
    <h6>
     <a id="_115">
     </a>
     主体结构介绍
    </h6>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c8060a186aac4239acef4ba4699e7953.png"/>
    </p>
    <ul>
     <li>
      盘片（platter）：上图中的圆盘就是盘片。上面有磁性的记录材料，可以
      <strong>
       记录数据
      </strong>
      。
      <ul>
       <li>
        盘面（surface）：每个盘片有2个面 ，这个面就是盘面。
       </li>
      </ul>
     </li>
     <li>
      主轴（spindle）：可以旋转。主轴带动盘片以固定的速率高速旋转。
     </li>
     <li>
      读/写头：来读写盘面表面存储的数据。上下盘面都有一个读/写头.
      <ul>
       <li>
        <strong>
         所有的读写头都是一起运动的，垂直并列
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      磁盘臂：通过磁盘臂的运动就可以运行读写数据的行为了。如果需要读取目标文件，盘面就会旋转起来。
      <ul>
       <li>
        寻找位置的这个过程【不包括旋转】叫
        <strong>
         寻道
        </strong>
        。
       </li>
       <li>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dce4c36866554f808121b088d8983973.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <h6>
     <a id="_125">
     </a>
     细节分析
    </h6>
    <blockquote>
     <p>
      先是磁道
     </p>
    </blockquote>
    <ul>
     <li>
      磁道：盘面的表面划分了一圈圈的磁道
      <ul>
       <li>
        扇区：磁道里面有许多扇区。
        <ul>
         <li>
          扇区与扇区之间有一些
          <strong>
           小间隙
          </strong>
          ，
          <strong>
           存储的不是 数据 而是 标识信息
          </strong>
          。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/300e45ca3e3c4767879c27f6588044ce.png"/>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      然后单位
     </p>
    </blockquote>
    <ul>
     <li>
      byte是字节单位，不要理解成比特bit了。
     </li>
     <li>
      在I/O设备里的单位是不同的。如下图
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5a9db14b6ad94794a6f125c4ba44af93.png"/>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0673004997bd405f9e06b43c297239cf.png"/>
     </li>
    </ul>
    <blockquote>
     <p>
      然后对扇区的访问时间主要分布为3个部分
     </p>
    </blockquote>
    <p>
     扇区访问时间 = 寻道时间+旋转时间+传送时间
    </p>
    <ul>
     <li>
      寻道时间：磁盘臂寻道过程的时间。
     </li>
     <li>
      旋转时间：盘片旋转将请求数据所在扇区移至读写磁头下方所需要的时间
     </li>
     <li>
      传送时间：完成传输所请求的数据所需要的时间。
     </li>
    </ul>
    <blockquote>
     <p>
      单位
     </p>
    </blockquote>
    <ul>
     <li>
      扇区： 硬盘的最小读写单元
     </li>
     <li>
      块/簇： 是操作系统针对硬盘读写的最小单元
     </li>
     <li>
      page： 是内存与操作系统之间操作的最小单元。
     </li>
    </ul>
    <h5>
     <a id="242SSD_148">
     </a>
     2.4.2、固态硬盘（SSD）
    </h5>
    <p>
     在现在的电脑几乎都是固态硬盘了。固态硬盘的缺陷就是容易磨损。
     <a href="https://www.bilibili.com/video/BV1Fg411V7LN?spm_id_from=333.788.player.switch&amp;vd_source=c6d25b8c148a15b1c45b8da5d613a042" rel="nofollow">
      推荐看这个视频了解
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2014858690da4a4c8bb7e8856d5c32c3.png"/>
    </p>
    <ul>
     <li>
      固态硬盘由一个或多个
      <code>
       闪存芯片
      </code>
      构成的。
      <ul>
       <li>
        使用闪存芯片 取代了 传统的机械臂转动和盘片旋转方式。
       </li>
      </ul>
     </li>
     <li>
      闪存转换站（FTL）
      <ul>
       <li>
        功能与磁盘控制器类似：都是将
        <strong>
         操作系统对逻辑块的请求
        </strong>
        翻译成
        <strong>
         对底层物理设备的访问
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       固态硬盘的擦除操作
      </strong>
      <ul>
       <li>
        固态硬盘除了读/写操作，还多了擦除操作。
       </li>
       <li>
        但是擦除
        <strong>
         只能把1 变成 0 ，而不能把0 变成1
        </strong>
        。 所以每一个page在放入数据前都是 1 。
       </li>
       <li>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3962296a36e7485392c53074b2b2573a.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="3IO_160">
     </a>
     3、I/O设备
    </h3>
    <h4>
     <a id="31_161">
     </a>
     3.1、定义
    </h4>
    <p>
     I/O设备就是可以
     <strong>
      将数据输入到计算机
     </strong>
     或者
     <strong>
      可以接收计算机输出数据的外部设备
     </strong>
     ， 属于计算机的
     <strong>
      硬件
     </strong>
     部分。
    </p>
    <h4>
     <a id="32_163">
     </a>
     3.2、分类
    </h4>
    <h5>
     <a id="321_164">
     </a>
     3.2.1、按使用特性分类
    </h5>
    <ul>
     <li>
      <strong>
       人机交互类 外部设备
      </strong>
      ：数据传输慢----&gt;用于人机交互
     </li>
     <li>
      <strong>
       存储设备
      </strong>
      ：数据传输快----&gt;用于数据存储
     </li>
     <li>
      <strong>
       网络通信设备
      </strong>
      ：数据传输介于2者之间----&gt;用于网络通信
     </li>
    </ul>
    <h5>
     <a id="322_168">
     </a>
     3.2.2、按传输速率分类
    </h5>
    <ul>
     <li>
      <strong>
       低速设备
      </strong>
      ：每秒几个到几百字节
     </li>
     <li>
      <strong>
       中速设备
      </strong>
      ：每秒千到万字节
     </li>
     <li>
      <strong>
       高速设备
      </strong>
      ：每秒可以到千兆字节
     </li>
    </ul>
    <h5>
     <a id="323_173">
     </a>
     3.2.3、按信息交换的单位
    </h5>
    <ul>
     <li>
      <strong>
       块设备
      </strong>
      ：传输速率较高，可寻址，即对它可随机地读/写任一块。
     </li>
     <li>
      <strong>
       字符设备
      </strong>
      ：传输速率较慢，不可寻址。在输入/输出时，常采用中断驱动方式。
     </li>
    </ul>
    <h3>
     <a id="4_176">
     </a>
     4、设备控制器和设备驱动器
    </h3>
    <p>
     <strong>
      设备控制器【硬件】
     </strong>
     ：负责
     <strong>
      直接控制物理设备
     </strong>
     ，
     <strong>
      能够接收并执行来自操作系统的指令
     </strong>
     。
     <br/>
     <strong>
      设备驱动器【软件】
     </strong>
     ：设备控制器
     <strong>
      通过设备驱动器利用中断与操作系统通信
     </strong>
     。
    </p>
    <h3>
     <a id="5_179">
     </a>
     5、实现输入和输出的方式有三种
    </h3>
    <ul>
     <li>
      <strong>
       忙等待
      </strong>
      ：一直循环直到需要的条件满足。会一直占据CPU，CPU一直轮询I/O设备直到I/O操作完成。
     </li>
     <li>
      <strong>
       设备驱动程序启动设备并且让该设备在操作完成时发生中断
      </strong>
      ，设备驱动程序在这个时刻返回。操作系统接着在需要时阻塞调用者并安排其他工作进行。当设备驱动程序检测到该设备操作完成时，它发出一个
      <strong>
       中断
      </strong>
      通知操作完成。
      <ul>
       <li>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dc0a1ac1c2e64003b08dcd21b5ef0a1c.png"/>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       引入DMA控制器
      </strong>
      ，代替CPU直接管理I/O设备与内存之间的数据传输。【直接存储器访问(Direct Memory Access, DMA) 芯片】
     </li>
    </ul>
