    <h2>
     <a id="_2">
     </a>
     引言
    </h2>
    <blockquote>
     <p>
      Linux作为一款多用户、多任务的操作系统，其权限管理机制是系统安全的核心基石。无论是个人开发者还是企业运维人员，理解Linux用户权限的设计原理和操作方法都至关重要。本文将从用户与组的概念入手，逐步解析文件权限、管理命令及实际应用场景，帮助读者构建清洗的权限管理知识体系。
     </p>
    </blockquote>
    <h2>
     <a id="1_6">
     </a>
     1、用户与组：权限的归属基础
    </h2>
    <p>
     Linux中所有操作都是基于用户身份执行，权限的分配也围绕用户和组展开。
    </p>
    <p>
     <img alt="所有者和所属组" src="https://i-blog.csdnimg.cn/img_convert/e403215e8fba341cfef967b0c480c081.png"/>
    </p>
    <ol>
     <li>
      <strong>
       用户（User）
      </strong>
      <ul>
       <li>
        每个用户拥有唯一的
        <strong>
         用户名
        </strong>
        和
        <strong>
         UID（User ID）
        </strong>
        ，系统通过UID识别用户。
       </li>
       <li>
        用户信息存储在
        <code>
         /etc/passwd
        </code>
        文件中，格式为：
        <code>
         username:x:UID:GID:description:homedir:shell
        </code>
       </li>
       <li>
        超级管理员**
        <code>
         root
        </code>
        **的UID为
        <code>
         0
        </code>
        ，拥有最高权限。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       组（Group）
      </strong>
      <ul>
       <li>
        组用于批量管理用户权限，每个用户至少属于一个
        <strong>
         主组
        </strong>
        ，可以加入多个
        <strong>
         附加组
        </strong>
        。
       </li>
       <li>
        组信息存储在
        <code>
         /etc/group
        </code>
        文件中，格式为：
        <code>
         groupname:x:GID:members
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token comment"># 查看当前用户信息</span>
<span class="token function">id</span>
<span class="token comment"># 创建新用户并指定UID</span>
<span class="token function">useradd</span> <span class="token parameter variable">-u</span> <span class="token number">1001</span> alice
<span class="token comment"># 将用户加入附加组</span>
<span class="token function">usermod</span> <span class="token parameter variable">-aG</span> developers alice
</code></pre>
    <h2>
     <a id="2rwx_29">
     </a>
     2、文件权限详解：rwx与数字表示
    </h2>
    <p>
     Linux中每个文件和目录的权限由三部分组成：
     <strong>
      所有者（Owner）
     </strong>
     、
     <strong>
      所属组（Group）
      <strong>
       和
      </strong>
      其他人（Others）
     </strong>
     。
    </p>
    <p>
     <img alt="目录文件权限" src="https://i-blog.csdnimg.cn/img_convert/1e4c1b7e7af6181092567df8c6b24264.png"/>
    </p>
    <p>
     每个文件或目录都具有一串字母，它们就是权限符号。
    </p>
    <ol>
     <li>
      <p>
       <strong>
        权限符号表示
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         <code>
          r
         </code>
         （读）
        </strong>
        ：查看文件内容或列出目录内容。
       </li>
       <li>
        <strong>
         <code>
          w
         </code>
         （写）
        </strong>
        ：修改文件或目录结构（如创建/删除文件）。
       </li>
       <li>
        <strong>
         <code>
          x
         </code>
         （执行）
        </strong>
        ：运行程序文件或进入目录。
       </li>
       <li>
        <strong>
         <code>
          -
         </code>
        </strong>
        ：证明该用户没有此项权限。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数组表示法
       </strong>
      </p>
      <ul>
       <li>
        将权限转换为二进制，
        <code>
         1
        </code>
        表示具有权限，
        <code>
         0
        </code>
        表示没有权限，则
        <code>
         rwx
        </code>
        可表示为
        <code>
         111
        </code>
        。
       </li>
       <li>
        以此类推，
        <code>
         r=4
        </code>
        ,
        <code>
         w=2
        </code>
        ,
        <code>
         x=1
        </code>
        。
       </li>
      </ul>
     </li>
    </ol>
    <h2>
     <a id="3_47">
     </a>
     3、权限管理命令
    </h2>
    <ol>
     <li>
      <strong>
       修改文件权限：
       <code>
        chmod
       </code>
      </strong>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token comment"># 符号模式</span>
<span class="token function">chmod</span> u+x main.c <span class="token comment"># 给所有者添加执行权限</span>
<span class="token function">chmod</span> g-w main.c <span class="token comment"># 给所属组去掉写权限</span>
<span class="token function">chmod</span> o+rw main.c <span class="token comment"># 给其他人添加读写权限</span>

<span class="token comment"># 数字模式</span>
<span class="token function">chmod</span> <span class="token number">755</span> main.c <span class="token comment"># main.c权限设置为 rwxr-xr-x</span>

<span class="token comment"># 递归修改目录权限</span>
<span class="token function">chmod</span> <span class="token parameter variable">-R</span> <span class="token number">664</span> /code/test
</code></pre>
    <ol start="2">
     <li>
      <strong>
       修改所有者与所属组：
       <code>
        chown
       </code>
       和
       <code>
        chgrp
       </code>
      </strong>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">chown</span> username:usergroup main.c <span class="token comment"># 同时修改所有者和所属组</span>
<span class="token function">chgrp</span> usergroup /test <span class="token comment"># 仅修改组</span>
</code></pre>
    <ol start="3">
     <li>
      <p>
       <strong>
        默认控制权限：
        <code>
         umask
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         在
         <code>
          Linux
         </code>
         操作系统中，目录的最大权限是
         <code>
          777
         </code>
         ，普通文件的最大权限是
         <code>
          666
         </code>
         。
        </p>
       </li>
       <li>
        <p>
         但当我们创建目录或文件时发现：
        </p>
        <ul>
         <li>
          目录权限是**
          <code>
           775
          </code>
          **。
         </li>
        </ul>
        <p>
         <img alt="目录权限" src="https://i-blog.csdnimg.cn/img_convert/c1b646d1c9625d24a20f9525f58c8c61.png"/>
        </p>
        <ul>
         <li>
          文件权限是**
          <code>
           664
          </code>
          **。
         </li>
        </ul>
        <p>
         <img alt="普通文件权限" src="https://i-blog.csdnimg.cn/img_convert/b5d221583af15be5480889f46563b5f7.png"/>
        </p>
       </li>
       <li>
        <p>
         <code>
          umask
         </code>
         值决定新建文件或目录的默认权限（默认权限=最大权限&amp;(~umask)）。
        </p>
       </li>
       <li>
        <p>
         如
         <code>
          umask=002
         </code>
         ，则文件权限：
        </p>
        <ul>
         <li>
          <img alt="文件权限" src="https://i-blog.csdnimg.cn/img_convert/aae3dcc494bda9e45aeb42decfb70e22.png"/>
         </li>
        </ul>
       </li>
       <li>
        <p>
         同理，可算出目录的默认权限是
         <code>
          775
         </code>
         。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <h2>
     <a id="4_93">
     </a>
     4、权限验证流程
    </h2>
    <p>
     当用户尝试访问问价时，Linux按一下顺序检查权限：
    </p>
    <ol>
     <li>
      <strong>
       是否为所有者？
      </strong>
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         → 
         
        
       
         \rightarrow
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.3669em;">
          </span>
          <span class="mrel">
           →
          </span>
         </span>
        </span>
       </span>
      </span>
      应用所有者权限。
     </li>
     <li>
      <strong>
       是否属于所属组的成员？
      </strong>
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         → 
         
        
       
         \rightarrow
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.3669em;">
          </span>
          <span class="mrel">
           →
          </span>
         </span>
        </span>
       </span>
      </span>
      应用组权限。
     </li>
     <li>
      <strong>
       除此之外
      </strong>
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         → 
         
        
       
         \rightarrow
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.3669em;">
          </span>
          <span class="mrel">
           →
          </span>
         </span>
        </span>
       </span>
      </span>
      应用其他人权限。
     </li>
    </ol>
    <blockquote>
     <p>
      [!CAUTION]
     </p>
     <p>
      权限一旦匹配即终止，不会叠加多个角色的权限。
     </p>
    </blockquote>
    <h2>
     <a id="5_105">
     </a>
     5、粘滞位
    </h2>
    <p>
     在团队协作中，一定需要多人在同一目录中写入文件数据，这就需要一个较为安全的目录，而普通目录会出现所有者写的文件被其他人删除的现象。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/3aff0e9ac6e72f32602ae014b4e6f6f8.png"/>
    </p>
    <ul>
     <li>
      为了解决这个问题，需要给目录设置粘滞位。
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> +t /code_test <span class="token comment"># 设置粘滞位</span>
</code></pre>
    <p>
     当一个目录被设置了粘滞位，则该目录的文件只能由：
    </p>
    <ol>
     <li>
      **超级管理员
      <code>
       root
      </code>
      **删除。
     </li>
     <li>
      <strong>
       该目录所有者
      </strong>
      删除。
     </li>
     <li>
      <strong>
       该文件所有者
      </strong>
      删除。
     </li>
    </ol>
    <h2>
     <a id="_123">
     </a>
     总结
    </h2>
    <p>
     Linux权限管理通过用户、组和文件权限的三层机制，实现了灵活且安全的资源访问控制。掌握
     <code>
      chmod
     </code>
     、
     <code>
      chown
     </code>
     等命令的使用，理解权限验证流程，并结合实际场景合理配置权限，是保障系统安全的关键技能。无论是开发环境还是生产服务器，良好的权限习惯都能有效降低风险，避免“一失权成千古恨”的悲剧。
    </p>
