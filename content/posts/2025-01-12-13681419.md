     <h2 class="postTitle" style="margin:10px 0px 0px; padding:2px 5px 1px; font-size:12px; font-weight:normal; background-color:rgb(216,223,234)">
      <a class="postTitle2" href="http://www.cnblogs.com/polomeeting/p/3175584.html" id="cb_post_title_url" rel="nofollow noopener noreferrer" style="color:rgb(51,51,51); text-decoration:none; outline:none; margin:0px; padding:0px; font-size:13px; font-weight:bold" target="_blank">
       视频会议传输模块的开发选择
      </a>
     </h2>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         视频会议
        </span>
       </span>
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         主要是开发音视频、数据的传输的软件，在这些开发过程中，最核心的模块是传输模块，传输模块的性能直接影响到视频会议的最终质量，因此传输模块的选择在视频会议开发当中尤其重要。传输模块在开发过程中，由于考虑到QOS的影响，一般都会使用数据重发的技术，因此传输模块以及成为视频会议底层开发的一个重点，我们开发的传输模块，可以选择用TCP、UDP直接进行开发或者利用开源的传输库，因为一些开源传输库以及比较成熟，直接拿来用就可以，如果自己开发传输模块，估计也是一个巨大工程。现在我们介绍一下传输模块的开发选择。
        </span>
       </span>
      </span>
     </div>
     <div style="margin:0cm 0cm 0pt 18pt; padding:0px; line-height:21px; text-indent:-18pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        1.
        <span style="margin:0px; padding:0px; line-height:normal; font-size:7pt; font-family:'Times New Roman'">
        </span>
        利用TCP进行开发
       </span>
      </span>
     </div>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        使用TCP直接进行传输模块的开发，在传输过程中数据不会丢包，我们可以采用OICP的完成端口来进行通讯，其优点是QOS有保证、支持大容量的数据并非，缺点是开发周期长，数据建立连接比一般UDP要长很多，而且资源占用也比一般UDP传输的要大。
       </span>
      </span>
     </div>
     <div style="margin:0cm 0cm 0pt 18pt; padding:0px; line-height:21px; text-indent:-18pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        2.
        <span style="margin:0px; padding:0px; line-height:normal; font-size:7pt; font-family:'Times New Roman'">
        </span>
        利用UDP进行开发
       </span>
      </span>
     </div>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        UPD是一种数据报传输方式，其数据在传输过程中不能保证数据的可靠性，因此用UPD进行传输模块的开发还要加入自己的QOS的丢包处理和自动重发的功能。其优点是数据传输较TCP快，缺点是开发周期长。
       </span>
      </span>
     </div>
     <div style="margin:0cm 0cm 0pt 18pt; padding:0px; line-height:21px; text-indent:-18pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        3.
        <span style="margin:0px; padding:0px; line-height:normal; font-size:7pt; font-family:'Times New Roman'">
        </span>
        利用UDT库进行开发
       </span>
      </span>
     </div>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         UDT建于
        </span>
       </span>
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:24px; font-size:12pt">
         <a href="http://zh.wikipedia.org/wiki/UDP" rel="nofollow noopener noreferrer" style="color:rgb(59,89,153); text-decoration:none; outline:none; margin:0px; padding:0px" target="_blank" title="UDP">
          <span style="margin:0px; padding:0px; line-height:22px; color:rgb(51,51,51); font-size:15px">
           <span style="margin:0px; padding:0px; color:windowtext">
            UDP
           </span>
          </span>
         </a>
        </span>
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         之上，并引入新的拥塞控制和数据可靠性控制机制。UDT是面向连接的双向的应用层协议。它同时支持可靠的数据流传输和部分可靠的数据报传输。UDT的特点是不用进行开发，直接利用库的发送函数就可以实现可靠的数据传输，缺点是模型较简单，没有OICP完成端口支持大容量的数据。
        </span>
       </span>
      </span>
     </div>
     <div style="margin:0cm 0cm 0pt 18pt; padding:0px; line-height:21px; text-indent:-18pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        4.
        <span style="margin:0px; padding:0px; line-height:normal; font-size:7pt; font-family:'Times New Roman'">
        </span>
        利用Ranknet库进行开发
       </span>
      </span>
     </div>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
        Raknet也是一个基于UDP网络传输协议的C++网络库，通过库的函数实现高效的网络传输服务。其具体应用在一些游戏领域，Radnet可以实现在两个程序之间每秒传输25，000条信息；其优点是不用进行开发，直接调用函数即可，缺点是大数据量在公网传输会由于网络堵塞而造成的流量控制失败的情况。
       </span>
      </span>
     </div>
     <div style="margin:0px; padding:0px; line-height:21px; text-indent:24pt">
      <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         综上所述，
        </span>
       </span>
       <span style="margin:0px; padding:0px; line-height:24px; font-size:12pt">
        <a href="http://www.polomeeting.cn/" rel="nofollow noopener noreferrer" style="color:rgb(59,89,153); text-decoration:none; outline:none; margin:0px; padding:0px" target="_blank">
         <span style="margin:0px; padding:0px; font-family:Arial; color:rgb(51,51,51)">
          <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
           视频会议
          </span>
         </span>
        </a>
       </span>
       <span style="margin:0px; padding:0px; font-family:Arial">
        <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
         的传输模块开发，传输效率最高是用UDP开发，但开发周期较长。用Raknet开发的传输模块不能适应大数据的传输，UDT在大数据量的传输有较好的优势，缺点是支持的并发不是很多。但如果经过修改后，UDT也能进行大容量的并发，因此综合考虑用UDT是视频会议传输模块的最佳选择。
        </span>

     <h2 class="postTitle" style="margin:10px 0px 0px; padding:2px 5px 1px; font-size:12px; font-weight:normal; background-color:rgb(216,223,234)">
      <a class="postTitle2" href="http://www.cnblogs.com/polomeeting/p/3174987.html" id="cb_post_title_url" rel="nofollow noopener noreferrer" style="color:rgb(51,51,51); text-decoration:none; outline:none; margin:0px; padding:0px; font-size:13px; font-weight:bold" target="_blank">
       视频会议的图像清晰度由什么决定
      </a>
     </h2>
     <div class="clear" style="margin:0px; padding:0px; clear:both">
     </div>
     <div class="postBody" style="margin:0px; padding:10px">
      <div id="cnblogs_post_body" style="margin:0px; padding:0px; line-height:1.8">
       <p style="padding-top:0px; padding-bottom:0px; margin:10px auto!important">
        <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
         我们选购视频会议的时候，最重要是考察系统的稳定性与音、视频质量等因素，而视频质量与视频会议的什么参数有关呢？首先我们说的视频质量，是我们平时的主观对视频图像的一个看法，不能通过一个具体的数字来体系，因此我们考察视频会议的质量只能通过主观对视频会议的图像的清晰度进行判断，而决定视频会议的清晰度由多种因素来决定，首先最重要的是视频会议的视频分辨率，另外一个是视频会议编码器的视频还原质量，最后一个视频会议的图像后处理技术。
        </span>
        <br style="margin:0px; padding:0px"/>
        <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
         1、视频分辨率
        </span>
       </p>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          <span style="margin:0px; padding:0px">
           视频会议
          </span>
          分辨率就是视频会议的视频采集的图像的分辨率，这个参数在视频会议的清晰度衡量中占用非常重要的作用，视频会议的分辨率可以分QCIF(160*120)、CIF(320*240)、720P(1280*720)、1280P(1920*1280)等分辨率，其分辨率越高，图像显示就越清楚，市场上所说的高清视频会议系统，也就是图像采集分辨率达到720P以上的高清分辨率。但是不是分辨率越高，视频会议的视频质量就越高呢？这个答案是否定。视频会议的分辨率只是图像的采集分辨率，而视频会议系统还需要经过视频的编码、解码、后期视频处理三个环节，因此视频会议分辨率越高不等同于视频会议的视频质量越高，只能说是本地的视频质量随图像采集分辨率的增加而增加。图一是各种分辨率下面的图像。而对于高清摄像头我们也有很大的误区，有的厂商把自己的摄像头标出非常高的像素，大家都认为这款肯定是高清分辨率，但测试的时候发现其分辨率只有640*480，这种摄像头其实并不是高清摄像头，高清摄像头的最基本指标是在720P的分辨率之上。因此高清摄像头决定因素是图像的采集分辨率。
          <br style="margin:0px; padding:0px"/>
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px; text-align:center">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          <img alt="图像采集分辨率" height="414" src="http://www.polomeeting.cn/upload/fckeditor/2013-05/11185021.png" style="border:0px; max-width:1000px; margin:0px; padding:0px" width="554">
           <br style="margin:0px; padding:0px"/>
           图一
          </img>
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          2、视频编码的还原质量
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          视频编码质量也是决定视频会议清晰度的一个重要指标，如现在最流行的H.264编码，其编码质量较高，图像的还原能力比之前的MPEG4还原能力更强，那一副图像通过不同的视频编码还原后，图像的还原度是不一样的。我们首先要讲一下视频编码的过程，视频编码首先获得视频采集设备的图像，然后把图像分成若干个块，然后对块进行数字转换，通过把块的帧内预测、帧外预测达到还原的图像的目的，因此图像的分块越小，图像的还原度会越高，如H.264采用最少4*4的块进行块的预测，而MPEG4、H.263采用最小8*8和16*16的块进行预测还原，因此H.264编码的图像在同等数据传输率下，比MPEG4、H.263更清楚。因此视频会议的清晰度还和视频编码的还原质量有关。
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          3、视频会议图像后处理技术
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          视频会议的图像清晰度最后也与图像的后处理技术相关，不同的
          <span style="margin:0px; padding:0px">
           视频会议系统
          </span>
          对图像的后处理技术都不尽相同，有的
         </span>
        </span>
        <span style="margin:0px; padding:0px; line-height:18px; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; font-family:Arial">
          <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
           <a href="http://www.polomeeting.cn/" rel="nofollow noopener noreferrer" style="color:rgb(59,89,153); text-decoration:none; outline:none; margin:0px; padding:0px" target="_blank">
            <span style="margin:0px; padding:0px; color:rgb(51,51,51)">
             视频会议
            </span>
           </a>
          </span>
         </span>
        </span>
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          为图像增加图像滤波处理，使图像还原去除方格效应等。有的
          <span style="margin:0px; padding:0px">
           视频会议软件
          </span>
          为图像采用图像增强技术，使图像看起来，更清晰。
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">
        <span style="margin:0px; padding:0px; line-height:18px; font-family:Arial; color:rgb(51,51,51)">
         <span style="margin:0px; padding:0px; line-height:22px; font-size:15px">
          以上就是决定视频会议清晰度的最主要的三个方面，视频会议分辨率是决定本地的视频质量的最关键因素。而视频编码是决定视频在编解码过程中的还原能的最主要因素，而图像后处理技术是图像还原的后期处理，在视频会议中也占有很重要的作用。
         </span>
        </span>
       </div>
       <div style="margin:0px; padding:0px">

        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         这个项目最早由Fabrice Bellard发起，现在由Michael Niedermayer维护。许多FFmpeg的开发人员都来自MPlayer项目，而且当前FFmpeg也是放在MPlayer项目组的服务器上。项目的名称来自MPEG
         <a href="http://baike.baidu.com/view/1645953.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">

         项目组成
        </h3>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         FFmpeg是一套可以用来记录、转换数字音频、视频，并能将其转化为流的开源计算机程序。它包括了目前领先的音/视频编码库libavcodec等。
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         libavformat ：用于各种音视频封装格式的生成和解析，包括获取解码所需信息以生成解码上下文结构
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         和读取音视频帧等功能；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         libavcodec ：用于各种类型声音/图像编解码；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         libavutil ：包含一些公共的工具函数；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         libswscale ：用于视频场景比例缩放、色彩映射转换；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         libpostproc：用于后期效果处理；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg ：该项目提供的一个工具，可用于格式转换、解码或电视卡即时编码等；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffsever ：一个 HTTP 多媒体即时广播串流服务器；
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffplay ：是一个简单的播放器，使用ffmpeg 库解析和解码，通过SDL显示；
        </p>

         视频采集功能
        </h4>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         FFmpeg是在
         <a href="http://baike.baidu.com/view/1634.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">
          Linux
         </a>
         下开发出来的，但它可以在包括Windows在内的大多数操作系统中编译。这个项目是由Fabrice Bellard发起的，现在由Michael Niedermayer主持。
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg视频采集功能非常强大，不仅可以采集
         <a href="http://baike.baidu.com/view/189679.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">
          视频采集卡
         </a>
         或USB
         <a href="http://baike.baidu.com/view/16175.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">
          摄像头
         </a>
         的图像，还可以进行屏幕录制，同时还支持以RTP方式将视频流传送给支持RTSP的流媒体服务器，支持直播应用。
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg在Linux下的视频采集
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         在Linux平台上，ffmpeg对V4L2的视频设备提高了很好的支持，如：
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ./ffmpeg -t 10 -f video4linux2 -s 176*144 -r 8 -i /dev/video0-vcodec h263 -f rtp rtp://192.168.1.105:5060 &gt; /tmp/ffmpeg.sdp
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         以上命令表示：采集10秒钟视频，对video4linux2视频设备进行采集，采集 QCIF(176*144)的视频，每秒8帧，视频设备为/dev/video0，视频编码为h263，输出格式为RTP，后面定义了IP地址及端口，将该码流所对应的SDP文件重定向到/tmp/ffmpeg.sdp中，将此SDP文件上传到流媒体服务器就可以实现直播了。
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg在windows下的视频采集
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         在windows下关于ffmpeg视频采集的资料非常少，但是ffmpeg还是支持windows下视频采集的。ffmpeg支持windows下video for windows(VFW)设备的视频采集，不过
         <a href="http://baike.baidu.com/view/189682.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">
          VFW
         </a>
         设备已经过时，正在被WDM的视频设备所取代，但是ffmpeg还没有支持
         <a href="http://baike.baidu.com/view/101606.htm" rel="nofollow noopener noreferrer" style="color:rgb(255,153,0); text-decoration:none; outline:none" target="_blank">
          WDM
         </a>
         的计划，不过好像有将WDM转为VFW的工具，因此ffmpeg还是可以在windows下进行视频采集的。
        </p>
        <h4 style="margin:0px; padding:0px; font-size:14px; font-family:Arial; line-height:26px">

         视频格式转换功能

        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg视频转换功能。视频格式转换，比如可以将多种视频格式转换为flv格式，可不是视频信号转换，，
        </p>
        <p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px; font-family:Arial; font-size:14px; line-height:26px">
         ffmpeg可以轻易地实现多种视频格式之间的相互转换(wma,rm,avi,mod等)，例如可以将摄录下的视频avi等转成现在视频网站所采用的flv格式。
