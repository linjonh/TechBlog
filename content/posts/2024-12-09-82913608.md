    <h3>
     <a id="_1">
     </a>
     音视频基础知识
    </h3>
    <p>
     录像、录音，实质是一个压缩采集的图像或视频数据的过程(
     <strong>
      编码
     </strong>
     )，就会有各种各样的格式。
     <br/>
     播放视频或音频文件，实质上是一个解压缩的过程(
     <strong>
      解码
     </strong>
     )
    </p>
    <h4>
     <a id="_5">
     </a>
     播放一个视频文件的流程
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9b15edc20f2241fef8f8dae631183e59.png" width="550"/>
    </p>
    <ol>
     <li>
      视频文件后缀名是FLV，MKV，MP4等 ，都是封装格式的数据
     </li>
     <li>
      然后解封装
     </li>
     <li>
      接着获取 音频压缩数据 和 视频压缩数据
     </li>
     <li>
      音频压缩数据 解码 获得 音频采样数据(
      <strong>
       PCM
      </strong>
      )，视频压缩数据 解码 获得 视频像素数据(
      <strong>
       YUV
      </strong>
      )
     </li>
    </ol>
    <h4>
     <a id="_12">
     </a>
     采样率
    </h4>
    <p>
     采样率:44100
     <br/>
     44100是人最大支持的采样率，再大没有什么意义
    </p>
    <h4>
     <a id="_16">
     </a>
     查看视频信息
    </h4>
    <p>
     MediaInfo软件
    </p>
    <h4>
     <a id="_19">
     </a>
     视频封装格式
    </h4>
    <h5>
     <a id="_20">
     </a>
     作用
    </h5>
    <p>
     视频码流和音频流按照一定的格式存储在一个文件中
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e6135365588bafb08e11f185228dcde5.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6ed53edabcb613defe6dd4f15a120f6a.png" width="700"/>
     </img>
    </p>
    <h5>
     <a id="MPEG2TS_25">
     </a>
     MPEG2-TS格式简介
    </h5>
    <p>
     不包含文件头。数据大小固定(188Byte)的Ts Packet构成
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8e57ed1df2d4b362836a0d73293f5618.png" width="700"/>
    </p>
    <h5>
     <a id="FLV_29">
     </a>
     FLV格式简介
    </h5>
    <p>
     包含头文件。数据由大小不固定的Tag构成
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/38196780af165bdd0630416df7b95947.png" width="700"/>
    </p>
    <h4>
     <a id="_33">
     </a>
     视频编码数据
    </h4>
    <p>
     将视频像素数据(RGB，YUV等)压缩成为视频码流，从而降低视频的数据量。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/216029ffed68de2d369176e69cb756fe.png" width="700"/>
    </p>
    <h5>
     <a id="H264__38">
     </a>
     H.264 格式简介
    </h5>
    <p>
     数据由大小不固定的NALU构成
     <br/>
     最常见的情况下，1个NALU存储了一帧画面的压缩编码后的数据
     <br/>
     可以将视频数据压缩100倍以上
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/148ec2358c8f05d0e393d75c1ed050aa.png" width="700"/>
    </p>
    <h4>
     <a id="_45">
     </a>
     音频编码数据
    </h4>
    <p>
     将音频采样率数据(PCM等)压缩成为音频码流，从而降低音频的数据量。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/da6dfbe1d0762e3818514786090bdc2d.png" width="700"/>
    </p>
    <h5>
     <a id="AAC_49">
     </a>
     AAC格式简介
    </h5>
    <p>
     数据由大小不固定的ADTS构成
     <br/>
     可以将音频数据压缩10倍以上
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e17ec208bdd2bbaf2d759fcb9d723a41.png"/>
    </p>
    <h4>
     <a id="_54">
     </a>
     视频像素数据
    </h4>
    <p>
     保存了屏幕中每个像素点的像素值
    </p>
    <h5>
     <a id="_56">
     </a>
     特点
    </h5>
    <p>
     视频像素体积很大
    </p>
    <p>
     Y是U和V的4倍
    </p>
    <h5>
     <a id="YUV_60">
     </a>
     YUV格式介绍
    </h5>
    <ul>
     <li>
      相关实验表明,人眼对亮度敏感而对色度不敏感。因而可以将亮度信息和色度信息分离,并对色度信息采用更“狠”一点的压缩方案从而提高压缩效率。
     </li>
     <li>
      YUV格式中, Y只包含亮度信息,而UV只包含色度信息。以YUV420P为例,图像像素数据的存储方式如图所示。
     </li>
     <li>
      从图中可以看出, YUV420P首先存储了整张图像的Y信息，然后存储整张图像的U信息,最后存储了整张图像的V信息。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ea49dd3831206f3cf38665af4db2aae0.png"/>
     </li>
    </ul>
    <h5>
     <a id="RGB_65">
     </a>
     RGB格式介绍
    </h5>
    <ul>
     <li>
      Red, Green, Blue三种颜色,可以混合成世界上所有的颜色。(
      <a href="https://baike.baidu.com/item/%E4%B8%89%E5%8E%9F%E8%89%B2/764849?fr=aladdin" rel="nofollow">
       三原色
      </a>
      )
     </li>
     <li>
      彩色图像中每个点,由R,G、B三个分量组成。
      <br/>
      以RGB24为例,图像像素数据的存储方式如下:
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8eaa82042ef772fa07d36376530c8ac9.png"/>
      <br/>
      从图中可以看出, RGB24依次存储了每个像素点的R、G、B信息。
     </li>
    </ul>
    <blockquote>
     <p>
      BMP文件中存储的就是RGB格式的像素数据。
     </p>
    </blockquote>
    <h4>
     <a id="_72">
     </a>
     音频采样数据
    </h4>
    <p>
     保存了音频中每个采样点的值
    </p>
    <h5>
     <a id="_74">
     </a>
     特点
    </h5>
    <p>
     音频采样数据体积很大
    </p>
    <h5>
     <a id="PCM_76">
     </a>
     PCM格式简介
    </h5>
    <ul>
     <li>
      单声道的情况下按照顺序存储每个采样点的数据。
     </li>
     <li>
      双声道的情况下按照“左右、左右”的顺序存储每个采样点两个声道的数据。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d4eea1cba64ffd67727c0249d53e96ef.png"/>
     </li>
    </ul>
