     <h2 id="%E8%AF%95%E5%9B%BE%E7%9A%84%E6%9C%AC%E8%B4%A8%E5%92%8C%E4%BC%98%E5%8A%BF%EF%BC%9A" name="%E8%AF%95%E5%9B%BE%E7%9A%84%E6%9C%AC%E8%B4%A8%E5%92%8C%E4%BC%98%E5%8A%BF%EF%BC%9A" style="background-color:transparent">
      试图的本质和优势：
     </h2>
    </blockquote>
    <h3 id="%E8%A7%86%E5%9B%BE%E7%9A%84%E6%9C%AC%E8%B4%A8%EF%BC%9A" name="%E8%A7%86%E5%9B%BE%E7%9A%84%E6%9C%AC%E8%B4%A8%EF%BC%9A" style="background-color:transparent">
     <span style="color:#fe2c24">
      <strong>
       视图的本质：
      </strong>
     </span>
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       视图是一种
      </strong>
     </span>
     <span style="color:#fe2c24">
      <strong>
       虚拟表
      </strong>
     </span>
     <span style="color:#0d0016">
      <strong>
       （查询视图语法与普通表类似使用select）
      </strong>
     </span>
     ，它基于数据库中的一个或多个表（或其他视图）通过查询语句定义，
     <strong>
      不实际存储数据
     </strong>
     ，仅保存了查询的定义和结果集的逻辑关系。
    </p>
    <p>
     临时表、虚拟表和真实表之间通常是可以联合查询的。
    </p>
    <p>
    </p>
    <h3 id="%E8%A7%86%E5%9B%BE%E7%9A%84%E4%BC%98%E5%8A%BF%EF%BC%9A" name="%E8%A7%86%E5%9B%BE%E7%9A%84%E4%BC%98%E5%8A%BF%EF%BC%9A" style="background-color:transparent">
     <span style="color:#fe2c24">
      <strong>
       视图的优势：
      </strong>
     </span>
    </h3>
    <p>
     对于复杂的多表联合查询或含有复杂条件过滤的查询
     <strong>
      （复杂的查询语句），可将其封装在视图中
     </strong>
     。这样，用户
     <strong>
      只需对视图进行简单查询
     </strong>
     ，无需编写冗长复杂的 SQL 语句，降低了查询的复杂度，提高了查询效率。
    </p>
    <p>
    </p>
    <blockquote>
     <h2 id="%E4%B8%80%E3%80%81%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%EF%BC%9A" name="%E4%B8%80%E3%80%81%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%EF%BC%9A" style="background-color:transparent">
      一、创建视图：
     </h2>
    </blockquote>
    <h3 id="%C2%A0%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A" name="%C2%A0%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A">
     基本语法：
    </h3>
    <pre><code class="language-sql">create view view_name [(column1, column2, ...)]
AS
select column1, column2, ...
from table_name
[where condition]
</code></pre>
    <ul>
     <li>
      <p>
       view ：创建视图的关键字。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         view_name
        </code>
       </strong>
       ：视图名称。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         (column1, ...)
        </code>
       </strong>
       ：可选，
       <span style="color:#fe2c24">
        <strong>
         为视图的列指定别名
        </strong>
       </span>
       （
       <strong>
        如果没有这句
       </strong>
       ，默认使用查询结果列名），
       <span style="color:#fe2c24">
        <strong>
         如果有这句，
        </strong>
       </span>
       那么指定的列名数量必须与查询语句
       <code>
        select
       </code>
       子句中选择的列的数量一致。
      </p>
     </li>
    </ul>
    <p>
    </p>
    <blockquote>
     <h3 id="1.%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%E7%A4%BA%E4%BE%8B%EF%BC%88%E6%AD%A3%E5%B8%B8%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%EF%BC%89%EF%BC%9A" name="1.%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%E7%A4%BA%E4%BE%8B%EF%BC%88%E6%AD%A3%E5%B8%B8%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%EF%BC%89%EF%BC%9A" style="background-color:transparent">
      1.创建视图示例（正常创建视图）：
     </h3>
    </blockquote>
    <p>
     <span style="color:#fe2c24">
      <strong>
       1.指定视图列名：
      </strong>
     </span>
    </p>
    <pre><code class="language-sql">CREATE VIEW view_name (col1, col2, col3)
AS
SELECT column1, column2, column3 + 10
FROM table_name;</code></pre>
    <p>
     <strong>
      视图的的列名结构：

    </p>
    <p style="text-align:center">

    <p>

    <p>
    </p>
    <p>
     <span style="color:#fe2c24">
      2
      <strong>
       .查询语句中列有别名：
      </strong>
     </span>
    </p>
    <pre><code class="language-sql">CREATE VIEW view_name
AS
SELECT column1, column2, column3 + 10 AS new_column
FROM table_name;</code></pre>
    <p>
     <strong>
      视图的的列名结构：

     <h3 id="2.%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%E7%A4%BA%E4%BE%8B%EF%BC%88%E8%A7%86%E5%9B%BE%E4%B8%AD%E7%9A%84%E6%9F%A5%E8%AF%A2%E8%AF%AD%E5%8F%A5%E6%B6%89%E5%8F%8A%E7%9A%84%E5%88%97%E5%90%8D%E7%9B%B8%E5%90%8C%E6%97%B6%EF%BC%89%EF%BC%9A" name="2.%E5%88%9B%E5%BB%BA%E8%A7%86%E5%9B%BE%E7%A4%BA%E4%BE%8B%EF%BC%88%E8%A7%86%E5%9B%BE%E4%B8%AD%E7%9A%84%E6%9F%A5%E8%AF%A2%E8%AF%AD%E5%8F%A5%E6%B6%89%E5%8F%8A%E7%9A%84%E5%88%97%E5%90%8D%E7%9B%B8%E5%90%8C%E6%97%B6%EF%BC%89%EF%BC%9A" style="background-color:transparent">
      2.创建视图示例（视图中的查询语句涉及的列名相同时）：
     </h3>
    </blockquote>
    <p style="background-color:transparent">
     <strong>
      方法：给查询表的列起个不重复的别名
     </strong>
    </p>
    <p>
     假设有两个表
     <code>
      orders
     </code>
     和
     <code>
      order_items
     </code>
     ，
     <span style="color:#fe2c24">
      <strong>
       它们都有
       <code>
        order_id
       </code>
       列
      </strong>
     </span>
     ，现在要创建一个视图将这两个表连接起来：
    </p>
    <pre><code class="language-sql">-- 创建视图
CREATE VIEW combined_orders_view AS
SELECT 
    o.order_id AS order_main_id,
    oi.order_id AS order_item_id,
    o.customer_id,
    oi.product_id
FROM 
    orders o
JOIN 
    order_items oi ON o.order_id = oi.order_id;
</code></pre>
    <p>
     在上面的代码中，
     <strong>
      通过
      <code>
       as
      </code>
      关键字为
      <code>
       orders
      </code>
      表和
      <code>
       order_items
      </code>
      表中的
      <code>
       order_id
      </code>
      列分别指定了别名
      <code>
       order_main_id
      </code>
      和
      <code>
       order_item_id
      </code>
     </strong>
     ，这样在视图中就可以清晰地区分这两个列。
    </p>
    <p>
    </p>
    <p>
     <strong>
      视图的列名结构：
     </strong>

      二、查看创建的视图的列结构：
     </h2>
    </blockquote>
    <h3 id="%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A" name="%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A" style="background-color:transparent">
     基本语法：
    </h3>
    <pre><code class="language-sql">desc employee_view; </code></pre>
    <p>
     这句代码可以查看当前视图的列结构，即包含那些列，列的基本信息（列名、数据类型、是否允许为空、默认值等）。
    </p>
    <p>
    </p>
    <p>
    </p>
    <blockquote>
     <h2 style="background-color:transparent">
      三、使用视图：
     </h2>
    </blockquote>
    <h3>
     基本语法：
    </h3>
    <pre><code class="language-sql">-- 从视图中查询视图所有列和所有行的数据
select * from view_name;

-- 从视图中查询视图指定列和满足特定条件的数据
select column1, column2 
from view_name
where condition;</code></pre>
    <p>
     <span style="color:#fe2c24">
      <strong>
       <code>
        column1, column2
       </code>
      </strong>
      ：这部分指定了要从视图中查询的列
     </span>
    </p>
    <p>
     可以看到，使用视图与查询表语法基本一致。
    </p>

    <blockquote>
     <h2 id="%E4%B8%89%E3%80%81%E4%BF%AE%E6%94%B9%E4%BA%86%E7%9C%9F%E5%AE%9E%E8%A1%A8%E6%95%B0%E6%8D%AE%EF%BC%8C%E5%86%8D%E4%BD%BF%E7%94%A8%E8%A7%86%E5%9B%BE%EF%BC%9A" name="%E4%B8%89%E3%80%81%E4%BF%AE%E6%94%B9%E4%BA%86%E7%9C%9F%E5%AE%9E%E8%A1%A8%E6%95%B0%E6%8D%AE%EF%BC%8C%E5%86%8D%E4%BD%BF%E7%94%A8%E8%A7%86%E5%9B%BE%EF%BC%9A" style="background-color:transparent">
      四、修改了真实表数据，再使用视图：
     </h2>
    </blockquote>
    <p>
     一般来说，如果修改了真实表，那么按照
     <strong>
      修改的情况
     </strong>
     再确定要不要重新创建其对应的视图（虚拟表）。
    </p>
    <table align="center" border="1" cellpadding="2" cellspacing="2" style="width:600px">
     <tbody>
      <tr>
       <td>
        <strong>
         表结构变更类型
        </strong>
       </td>
       <td>
        <strong>
         是否需要修改视图
        </strong>
       </td>
       <td>
        <strong>
         操作方式
        </strong>
       </td>
      </tr>
      <tr>
       <td>
        修改了
        <strong>
         真实数据
        </strong>
        （比如把id 列 的 3改成了 10）
       </td>
       <td>
        否
       </td>
       <td>
       </td>
      </tr>
      <tr>
       <td>
        修改了列的
        <strong>
         顺序
        </strong>
       </td>
       <td>
        否
       </td>
       <td>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         新增列
        </strong>
       </td>
       <td>
        否
       </td>
       <td>
        视图不受影响（
        <strong>
         <span style="color:#fe2c24">
          不修改视图查询不到新增列的信息
         </span>
        </strong>
        ），除非需要将新的列加入视图逻辑。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         删除列
        </strong>
       </td>
       <td>
        是
       </td>

        <span style="color:#fe2c24">
         <strong>
          如果删除的列不在当前视图中，通常不需要对视图进行修改
         </strong>
         ）
        </span>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         重命名列
        </strong>
       </td>
       <td>
        是
       </td>

        <span style="color:#fe2c24">
         <strong>
          如果重命名的列不在当前视图中，通常不需要对视图进行任何修改
         </strong>
         ）
        </span>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         修改列的类型
        </strong>
       </td>
       <td>
        可能
       </td>
       <td>
        若视图依赖该列的计算逻辑（如

        <strong>
         新增约束/索引
        </strong>
       </td>
       <td>
        否
       </td>
       <td>
        不影响视图逻辑，无需操作。
       </td>
      </tr>

       <td>
        <strong>
         表重命名
        </strong>
       </td>
       <td>
        是
       </td>

        <code>
         from
        </code>

     <h2 id="%E5%9B%9B%E3%80%81%E9%80%9A%E8%BF%87%E8%A7%86%E5%9B%BE%E4%BF%AE%E6%94%B9%E7%9C%9F%E5%AE%9E%E8%A1%A8%EF%BC%9A" name="%E5%9B%9B%E3%80%81%E9%80%9A%E8%BF%87%E8%A7%86%E5%9B%BE%E4%BF%AE%E6%94%B9%E7%9C%9F%E5%AE%9E%E8%A1%A8%EF%BC%9A" style="background-color:transparent">
      五、通过视图修改真实表：
     </h2>
    </blockquote>
    <h3 id="%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A" name="%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A">
     基本语法：
    </h3>
    <h4 id="1.%E6%9B%B4%E6%96%B0%E6%95%B0%E6%8D%AE%EF%BC%9A" name="1.%E6%9B%B4%E6%96%B0%E6%95%B0%E6%8D%AE%EF%BC%9A">
     1.更新数据：
    </h4>
    <pre><code class="language-sql">-- 假设存在视图 view_name，它基于真实表 table_name 创建
update view_name
set column1 = 'new_value1', column2 = 'new_value2'
where condition;</code></pre>
    <ul>
     <li>
      <span style="color:#fe2c24">
       <strong>
        <code>
         view_name
        </code>
        是视图的名称。
       </strong>
      </span>
     </li>
     <li>
      <span style="color:#fe2c24">
       <strong>
        <code>
         column1
        </code>
        、
        <code>
         column2
        </code>
        是视图中的列名，对应真实表中的列。
       </strong>
      </span>
     </li>
     <li>
      <code>
       'new_value1'
      </code>
      、
      <code>
       'new_value2'
      </code>
      是要更新的值。
     </li>
     <li>
      <code>
       condition
      </code>
      是筛选条件，用于指定要更新哪些行。
     </li>
    </ul>
    <p>
    </p>
    <h4 id="2.%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE%EF%BC%9A" name="2.%E6%8F%92%E5%85%A5%E6%95%B0%E6%8D%AE%EF%BC%9A">
     2.插入数据：
    </h4>
    <pre><code class="language-sql">insert into view_name (column1, column2)
values ('value1', 'value2');</code></pre>
    <ul>
     <li>
      <code>
       view_name
      </code>
      是视图的名称。
     </li>
     <li>
      <code>
       column1
      </code>
      、
      <code>
       column2
      </code>
      是视图中的列名。
     </li>
     <li>
      <code>
       'value1'
      </code>
      、
      <code>
       'value2'
      </code>
      是要插入的值。
     </li>
    </ul>
    <p>
    </p>
    <p>
    </p>
    <h4 id="3.%E5%88%A0%E9%99%A4%E6%95%B0%E6%8D%AE%EF%BC%9A" name="3.%E5%88%A0%E9%99%A4%E6%95%B0%E6%8D%AE%EF%BC%9A">
     3.删除数据：
    </h4>
    <pre><code class="language-sql">delete from view_name
where condition;</code></pre>
    <ul>
     <li>
      <code>
       view_name
      </code>
      是视图的名称。
     </li>
     <li>
      <code>
       condition
      </code>
      是筛选条件，用于指定要删除哪些行。
     </li>
    </ul>
    <p>
    </p>
    <p>
    </p>
    <p>
    </p>
    <blockquote>
     <h3 id="%E6%B3%A8%E6%84%8F%EF%BC%9A" name="%E6%B3%A8%E6%84%8F%EF%BC%9A" style="background-color:transparent">
      注意：
     </h3>
    </blockquote>
    <h4 id="1.%E9%80%9A%E8%BF%87%E8%A7%86%E5%9B%BE%E4%BF%AE%E6%94%B9%E6%95%B0%E6%8D%AE%E4%BC%9A%E5%BD%B1%E5%93%8D%E7%9C%9F%E5%AE%9E%E8%A1%A8%E3%80%82" name="1.%E9%80%9A%E8%BF%87%E8%A7%86%E5%9B%BE%E4%BF%AE%E6%94%B9%E6%95%B0%E6%8D%AE%E4%BC%9A%E5%BD%B1%E5%93%8D%E7%9C%9F%E5%AE%9E%E8%A1%A8%E3%80%82">
     <span style="color:#fe2c24">
      <strong>
       1.通过视图修改数据会影响真实表。
      </strong>
     </span>
    </h4>
    <p>
     <span style="color:#fe2c24">
      <strong>
      </strong>
     </span>
     因为视图本身并不存储数据，它是基于真实表（基表）定义的虚拟表，当通过视图进行数据修改操作（如更新、插入、删除）时，
     <strong>
      实际上是对视图所基于的真实表中的数据进行相应的修改。
     </strong>
    </p>
    <p>
    </p>
    <p>
    </p>
    <h4 id="2.%E8%A7%86%E5%9B%BE%E5%BF%85%E9%A1%BB%E6%BB%A1%E8%B6%B3%E4%BB%A5%E4%B8%8B%E6%9D%A1%E4%BB%B6%EF%BC%8C%E6%89%8D%E8%83%BD%E9%80%9A%E8%BF%87%E5%AE%83%E7%9B%B4%E6%8E%A5%E4%BF%AE%E6%94%B9%E5%9F%BA%E7%A1%80%E8%A1%A8%E6%95%B0%E6%8D%AE%EF%BC%9A" name="2.%E8%A7%86%E5%9B%BE%E5%BF%85%E9%A1%BB%E6%BB%A1%E8%B6%B3%E4%BB%A5%E4%B8%8B%E6%9D%A1%E4%BB%B6%EF%BC%8C%E6%89%8D%E8%83%BD%E9%80%9A%E8%BF%87%E5%AE%83%E7%9B%B4%E6%8E%A5%E4%BF%AE%E6%94%B9%E5%9F%BA%E7%A1%80%E8%A1%A8%E6%95%B0%E6%8D%AE%EF%BC%9A">
     <span style="color:#fe2c24">
      2.视图必须满足以下条件，才能通过它直接修改基础表数据：
     </span>
    </h4>
    <p>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        基于单表
       </strong>
       ：视图的定义
       <span style="color:#fe2c24">
        <strong>
         仅来自
        </strong>
       </span>
       <strong>
        一个基础表
       </strong>
       （不能涉及多表连接或子查询）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        不包含聚合或分组
       </strong>
       ：
       <span style="color:#fe2c24">
        <strong>
         未使用
        </strong>
       </span>
       order by  、
       <code>
        group by
       </code>
       、
       <code>
        disdinct
       </code>
       、聚合函数（如
       <font face="monospace">
        sum( ) 、avg( )
       </font>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        未使用某些运算符
       </strong>
       ：
       <strong>
        <span style="color:#fe2c24">
         未使用
        </span>
       </strong>
       <font face="monospace">
        union 、union all 、
       </font>
       intersect 、except  等集合操作。
      </p>
     </li>
    </ol>
    <p>
    </p>
    <p>
     还有一种情况不能更新视图：
    </p>
    <p>
     如果在
     <code>
      from
     </code>
     子句中引用了不可更新的视图，那么基于这个组合所创建的新视图通常也是不可更新的:
    </p>
    <pre><code class="language-sql">-- 创建不可更新视图 v1（包含聚合操作）
create view v1 as
select category, sum(amount) as total_amount
from table1
group by category;

-- 创建新视图 v2，在 from 子句中引用 v1
create view v2 as
select category, total_amount
from v1
where total_amount &gt; 100;</code></pre>
    <p>
     <strong>
      在这个例子中，
      <span style="color:#fe2c24">
       由于
       <code>
        v1
       </code>
       是不可更新的（因为包含了
       <code>
        SUM
       </code>
       聚合函数和
       <code>
        GROUP BY
       </code>
       子句），所以
       <code>
        v2
       </code>
       也将是不可更新的。
      </span>
     </strong>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
    </p>

     <h2 id="%E4%BA%94%E3%80%81%E5%88%A0%E9%99%A4%E8%A7%86%E5%9B%BE%EF%BC%9A" name="%E4%BA%94%E3%80%81%E5%88%A0%E9%99%A4%E8%A7%86%E5%9B%BE%EF%BC%9A" style="background-color:transparent">
      <span style="color:#0d0016">
       <strong>
        六、删除视图：
       </strong>
      </span>
     </h2>
    </blockquote>
    <h3 id="%C2%A0%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A" name="%C2%A0%E5%9F%BA%E6%9C%AC%E8%AF%AD%E6%B3%95%EF%BC%9A">
     基本语法：
    </h3>
    <pre><code class="language-sql">drop view [if exists] view_name </code></pre>
    <ul>
     <li>
      <code>
       if exists
      </code>
      ：这是一个可选参数。如果使用了该参数，当要删除的视图不存在时，MySQL 不会报错，只会给出一个警告信息。如果不使用该参数，当试图删除一个不存在的视图时，会抛出错误。
     </li>
    </ul>
