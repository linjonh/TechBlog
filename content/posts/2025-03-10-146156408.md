    <blockquote>
     <p>
      当我们使用
      <code>
       electron
      </code>
      想要集成一个C++ SDK实现很底层的算法逻辑就有可能与C++ SDK进行数据通信。
      <br/>
      Napi 应该是比较好的选择，因为C++本身的运行速度很快，使用
      <code>
       Napi
      </code>
      也能很大程度上保证软件的兼容性、又不会阻塞C++线程、还可以很简单的与C++ 实现数据传递。
     </p>
    </blockquote>
    <h4>
     <a id="_2">
     </a>
     开始使用
    </h4>
    <ol>
     <li>
      安装 C++ 运行环境 直接装一个 VS2022 (c++桌面端开发)
     </li>
     <li>
      安装 python 3.8.2 (如果开发electron win32 最好装32位环境)
     </li>
     <li>
      <code>
       npm install -g node-gyp
      </code>
     </li>
     <li>
      新建一个文件夹
      <code>
       addon
      </code>
      (存放demo文件)
     </li>
     <li>
      在
      <code>
       addon
      </code>
      内执行
      <code>
       npm init -y
      </code>
     </li>
     <li>
      在
      <code>
       addon
      </code>
      内执行
      <code>
       npm install node-addon-api
      </code>
     </li>
     <li>
      新建
      <code>
       src/addon.cpp
      </code>
      (相当于Napi的入口文件)（在下方）
     </li>
     <li>
      新建
      <code>
       binding.gyp
      </code>
      并配置（在下方）
     </li>
    </ol>
    <blockquote>
     <p>
      构建好后只需要 build/Release/addon.node 其他的可以删除（如果没有其他 dll 依赖）
     </p>
    </blockquote>

     我们看上述代码 重写了基类的虚函数，并使用
     <code>
      status
     </code>
     参数初始化了一块堆内存，然后将这个指针传给了线程安全函数，为什么不用栈内存? 是因为
