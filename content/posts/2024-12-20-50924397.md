    <p>
     docker蛮简单的一个应用，在节点上搭建的docker sftp服务器是一个隔离的环境，不需要sftp到远程节点，也避免了修改远程服务器文件的危险。
    </p>
    <p>
     注：以下操作都对应于ubuntu 版本。
    </p>
    <p>
     <strong>
      首先安装docker。
     </strong>
    </p>
    <ul>
     <li>
      docker只支持64-bit kernel，最低3.10，可使用uname -r查看。
     </li>
     <li>
      确定是否安装了wget
      <code>
       which wget
      </code>
      ，如果没有安装，使用
      <code>
       sudo apt-get update
      </code>
      <code>
       sudo apt-get install wget
      </code>
      进行安装。
     </li>
     <li>
      获取包
      <code>
       wget -qO- https://get.docker.com/ | sh
      </code>
     </li>
     <li>
      <code>
       sudo docker info
      </code>
      可以确保docker是否成功安装
     </li>
    </ul>
    <p>
     <strong>
      拉取docker sftp镜像并构建容器
     </strong>
    </p>
    <ul>
     <li>
      使用命令
      <code>
       sudo docker pull luzifer/sftp-share
      </code>
     </li>
     <li>
      并构建
      <code>
       docker run -d -p 2022:22 -e USER=myuser -e PASS=myverysecretpassword luzifer/sftp-share
      </code>
     </li>
    </ul>
    <p>
     至此，我们在同一局域网内就可以使用
     <code>
      sftp -P 2022 myuser@myverysecretpassword
     </code>
     进行sftp文件传输了。
    </p>
    <p>
     更新：
     <br/>
     建议使用 atmoz/sftp这个镜像
     <br/>
     <a href="https://github.com/atmoz/sftp">
      https://github.com/atmoz/sftp
     </a>
     <br/>
     <code>
      docker run \
      <br/>
      -v /host/share:/home/foo/share \
      <br/>
      -p 2222:22 -d atmoz/sftp \
      <br/>
      foo:123:1001
     </code>
    </p>
    <p>
     若想put，get操作有权限，对应挂载的磁盘地址(如上即为/host/share)需要对其他用户有写权限。
    </p>
