    <h4>
     <a id="2040_3">
     </a>
     手工操作阶段（20世纪40年代）
    </h4>
    <ul>
     <li>
      <strong>
       特征：
      </strong>
      <br/>
      1.
      <strong>
       有人工干预
      </strong>
      ：放纸带、电传打印机上输出结构
      <br/>
      2.
      <strong>
       独占性
      </strong>
      ：一个用户使用过程中占用计算机所有文件，第二个用户绝对无法干预
      <br/>
      3.
      <strong>
       串行性
      </strong>
      ：人和计算机的工作、CPU和输入输出组设备等只能串行不能同时进行
     </li>
     <li>
      <strong>
       缺点：
      </strong>
      <br/>
      1. 用户在作业处理的整个过程中独享系统中的全部资源；
      <br/>
      2. 手工操作所需时间很长
     </li>
     <li>
      <strong>
       问题
      </strong>
      <br/>
      1. 当CPU速度提高时出现人 ——机矛盾
      <br/>
      2. 比如：
     </li>

    <h4>
     <a id="_2050_20">
     </a>
     (单道)批处理阶段 （20世纪50年代）
    </h4>
    <ul>
     <li>
      运行图
      <br/>
      <img alt="02" src="https://i-blog.csdnimg.cn/blog_migrate/04fcf90709f641478c2f725dfe26a80e.jpeg"/>
     </li>
    </ul>
    <h5>
     <a id="_23">
     </a>
     联机批处理
    </h5>
    <ul>
     <li>
      <strong>
       特点：
      </strong>
      有监督程序，作业自动过度
     </li>
     <li>
      <strong>
       问题：
      </strong>
      CPU高速与I/O慢速的矛盾
     </li>
     <li>
      <strong>
       解决办法：
      </strong>
      由卫星机负责I/O
     </li>
    </ul>
    <h5>
     <a id="_27">
     </a>
     脱机批处理
    </h5>
    <ul>
     <li>
      <strong>
       特点：
      </strong>
      主机与卫星机并行操作
     </li>
     <li>
      <strong>
       问题：
      </strong>
      调度不灵活；保护问题；
     </li>
     <li>
      <strong>
       解决办法：
      </strong>
      通道技术、中断技术
     </li>
     <li>
      <strong>
       通道：
      </strong>
      又称I/O处理器，是一种专用处理部件，它能控制一台或多台外设的工作，负责外部设备与主存之间的信息传输。
     </li>
     <li>
      <strong>
       中断：
      </strong>
      当主机接到某种信号(如I/O设备完成信号)时，停止原来的工作，转去处理这一事件，当事件处理完毕主机又回到原来的工作点继续工作。
     </li>
    </ul>
    <h4>
     <a id="2060_34">
     </a>
     执行系统（20世纪60年代初期）
    </h4>
    <blockquote>
     <p>
      此时在引入通道和出现通道中断主机功能，是OS发展史的重要事件，它推动OS进入执行系统阶段。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       定义：
      </strong>
      借助于通道与中断技术，由主机控制I/O工作。原有的监督程序不仅要负责调度作业自动地运行，而且还要提供I/O控制功能。它常驻主存，称为执行系统。
     </li>
     <li>
      <strong>
       特点：
      </strong>
      主机、外设并行操作；增强了保护能力。
     </li>
     <li>
      <strong>
       基本功能：
      </strong>
      I/O控制功能；调度；
     </li>
     <li>
      <strong>
       问题：
      </strong>
      资源利用率不高
     </li>
    </ul>
    <h4>
     <a id="_2060_46">
     </a>
     多道批处理系统 （20世纪60年代初期）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        多道程序程序设计技术：
       </strong>
       在计算机主存中同时存放几道相互独立的程序。这些程序在管理程序控制之下，相互穿插地运行。当某道程序因某种原因不能继续运行下去时(如等待外部设备传输数据)，管理程序便将另一道程序投入运行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特征：
       </strong>
       <br/>
       1.
       <strong>
        多道
       </strong>
       ：内存中同时存放几个作业，并允许并发执行，从而有效地提高了资源利用率和系统吞吐量
       <br/>
       2.
       <strong>
        宏观上并行
       </strong>
       <br/>
       3.
       <strong>
        微观上串行
       </strong>
      </p>
      <blockquote>
       <p>
        关于并发、并行和串行的区别在前已经写了：
        <a href="https://blog.csdn.net/qq_43068326/article/details/104392613">
         OS中了解“并发”与“并行”的区别
        </a>
       </p>
      </blockquote>
     </li>
     <li>
      <p>
       <strong>
        优点：
       </strong>
       作业流程自动化；效率高；吞吐量高；
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点：
       </strong>
       无交互手段；调试程序困难；
      </p>
     </li>
     <li>
      <p>
       <strong>
        单道与多道批处理的比较
       </strong>
       <br/>
       <img alt="03" src="https://i-blog.csdnimg.cn/blog_migrate/0695d52e24daf14936a6cfcbe7c05115.png"/>
      </p>
     </li>
    </ul>
    <h4>
     <a id="2060_59">
     </a>
     分时系统（20世纪60年代初期和中期）
    </h4>
    <ul>
     <li>
      <strong>
       分时技术：
      </strong>
      所谓分时技术，是把处理机时间划分成很短的
      <strong>
       时间片
      </strong>
      (如几百毫秒)轮流地分配给各个联机作业使用，如果某个作业在分配的时间片用完之前计算还未完成，该作业就（被强制）暂时中断，等待下一轮继续计算。
     </li>
     <li>
      <strong>
       特点：
      </strong>
      <br/>
      1.
      <strong>
       多路性：
      </strong>
      又称
      <strong>
       多路调制性
      </strong>
      ，即一个主机可以同时与多个终端连接。
      <br/>
      2.
      <strong>
       交互性：
      </strong>
      又称
      <strong>
       交往性
      </strong>
      ，即系统以对话的方式为各个终端用户服务。
      <br/>
      3.
      <strong>
       独占性：
      </strong>
      手动操作速度远小于计算机处理速度，所以用户感觉独占了整个计算机，即每个终端用户实际上都拥有一台完全属于自己的虚拟机。
     </li>
    </ul>
    <h4>
     <a id="2060_67">
     </a>
     实时处理系统（20世纪60年代后期）
    </h4>
    <ul>
     <li>
      <strong>
       实时：
      </strong>
      计算机对于外来信息能够在被控对象允许的截止期限(deadline)内作出反应。
     </li>
     <li>
      <strong>
       实时处理：
      </strong>
      以快速反应为特征，对实时信号能在截止期限之内处理并作出反应。实时处理具有实时性和可预测性（整个系统的开销都可以预测出来）。
     </li>
     <li>
      <strong>
       特点：
      </strong>
      <br/>
      1.
      <strong>
       及时性：
      </strong>
      对外部请求在严格时间范围内作出反应
      <br/>
      2.
      <strong>
       可靠性
      </strong>
     </li>
    </ul>
    <h4>
     <a id="2060_74">
     </a>
     通用操作系统（20世纪60年代后期）
    </h4>
    <ul>
     <li>
      <strong>
       定义：
      </strong>
      同时具有分时、实时和批处理功能的操作系统成为通用操作系统。分时系统与批处理系统结合。
     </li>
     <li>
      <strong>
       任务分级：
      </strong>
      在通用操作系统中可能出现3类任务，任务按照其急迫程度加以分级：实时任务级别最高，分时任务次之，批处理任务最低。
     </li>
     <li>
      <strong>
       前后台系统：
      </strong>
      通常会将实时、分时和批处理中的俩俩系统结合起来，构成所谓的前后台系统。
      <br/>
      1. 前台任务优先于后台任务。
      <br/>
      2. “前台”：需要频繁交互的任务
      <br/>
      3. “后台”：时间性要求不强的作业
     </li>
    </ul>
    <blockquote>
     <p>
      以下OS的介绍顺序是随意排列
     </p>
    </blockquote>
    <h4>
     <a id="singleuser_operation_system_86">
     </a>
     单用户操作系统（
     <em>
      single-user operation system
     </em>
     ）
    </h4>
    <ul>
     <li>
      <strong>
       解释：
      </strong>
      单用户操作系统是为个人计算机所配置的操作系统。
     </li>
     <li>
      <strong>
       主要特点：
      </strong>
      单用户，即系统在同一段时间内仅仅为一个用户提供服务。
     </li>
     <li>
      <strong>
       追求目标：
      </strong>
      界面友好；使用方便；支持丰富的应用软件。
     </li>
     <li>
      <strong>
       例如：
      </strong>
      MS-DOS、CP/M、Windows等。
     </li>
    </ul>
    <h4>
     <a id="network_operation_systemNOS_92">
     </a>
     网络操作系统（
     <em>
      network operation system，NOS
     </em>
     ）
    </h4>
    <ul>
     <li>
      <strong>
       定义：
      </strong>
      用于实现网络通信和网络资源管理的操作系统称为网络操作系统。
     </li>
     <li>
      <strong>
       目的：
      </strong>
      相互通信；共享资源；
     </li>
     <li>
      <strong>
       特征：
      </strong>
      <br/>
      1. 计算机网络是一个互连的计算机系统的群体。
      <br/>
      2. 这些计算机是自治的，每台计算机有自己的操作系统，各自独立工作，它们在网络协议控制下协同工作。
      <br/>
      3. 系统互连要通过通信设施(硬件、软件)来实现。
      <br/>
      4. 系统通过通信设施执行信息交换、资源共享、互操作和协作处理，实现 多种应用要求。
     </li>
    </ul>
    <h4>
     <a id="distributed_operation_systemDOS_102">
     </a>
     分布式操作系统（
     <em>
      distributed operation system，DOS
     </em>
     ）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        分类：
       </strong>
       <br/>
       1.
       <strong>
        紧耦合（
        <em>
         closely coupled
        </em>
        ）操作系统：
       </strong>
       建立在多机系统的基础之上。
       <br/>
       2.
       <strong>
        松散耦合（
        <em>
         loosely coupled
        </em>
        ）操作系统：
       </strong>
       建立在计算及网络的基础之上。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特征：
       </strong>
       <br/>
       1.
       <strong>
        统一的操作系统
       </strong>
       <br/>
       2.
       <strong>
        资源的进一步共享
       </strong>
       <br/>
       3.
       <strong>
        可靠性
       </strong>
       <br/>
       4.
       <strong>
        透明性
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        DOS和NOS的区别
       </strong>
       <br/>
       1. DOS具有各个计算机间相互通讯，无主从关系；NOS有主从关系。
       <br/>
       2. DOS系统资源为所有用户共享；而NOS有限制地共享。
       <br/>
       3. DOS中若干个计算机可相互协作共同完成一项任务；而NOS只能在一台计算机上运行某个人物，无法迁移。
      </p>
     </li>
    </ul>
    <h4>
     <a id="embedded_operation_system_118">
     </a>
     嵌入式操作系统（
     <em>
      embedded operation system
     </em>
     ）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        定义：
       </strong>
       在机器人、车载系统、家用电器、手机等通信设备上都需要一个支持多道程序设计的环境，而提供这种环境的OS就是嵌入式操作系统。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点：
       </strong>
       <br/>
       1. 完成某一项或有限项功能;不是通用型的
       <br/>
       2. 在性能和实时性方面有严格的限制
       <br/>
       3. 能源、成本和可靠性通常是影响设计的重要因素
       <br/>
       4. 占有资源少、易于连接
       <br/>
       5. 系统功能可针对需求进行裁剪、调整和生成以便满足最终产品的设计要求
      </p>
     </li>
     <li>
      <p>
       <strong>
        与一般OS的差别：
       </strong>
      </p>
      <ol>
       <li>
        <strong>
         可裁剪性
        </strong>
       </li>
       <li>
        <strong>
         可移植性
        </strong>
       </li>
       <li>
        <strong>
         可扩展性
        </strong>
       </li>
      </ol>
     </li>
     <li>
      <p>
       <strong>
        系统例子：
       </strong>
       常见的嵌入式系统有Linux、uC1inux、 WinCE、Pa1mOS、Symbian、 eCos、 uCOS- -II、VxWorks、pSOS、Nucleus、 ThreadX、Rtems 、QNX、INTEGRITY、OSE、 C Executive等等
      </p>
     </li>
    </ul>
