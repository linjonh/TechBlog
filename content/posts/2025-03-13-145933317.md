    <h2 style="margin-left:0px; margin-right:0px">
     <strong>
      链路聚合背景
     </strong>
    </h2>
    <p style="margin-left:0; margin-right:0">
     拓扑组网时为了高可用，需要网络的冗余备份。但增加冗余容易后会出现环路，所以我们部署了STP协议来破除环路。
    </p>
    <p style="margin-left:0; margin-right:0">
     但是，根据实际业务的需要，为网络不停的增加冗余是现实需要的一部分。
    </p>
    <p style="margin-left:0; margin-right:0">
     那么，为了让网络冗余符合用户增长的流量，需要增加带宽时怎么办？
    </p>
    <p style="margin-left:0; margin-right:0">
     现实中，一条1000M的带宽和10000M的带宽的建设成本，差异是巨大的。为了节省网络组建的成本，我们通过添加链路来增加带宽是最优方案：
    </p>
    <p style="margin-left:0; margin-right:0">
     A1、添加链路相当于增加了链路冗余，会被STP的防环机制阻塞掉。
    </p>
    <p style="margin-left:0; margin-right:0">
    </p>

    <h2 style="margin-left:0px; margin-right:0px">
     <strong>
      1
     </strong>
     <strong>
      、网络的可靠性需求
     </strong>
    </h2>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      1.1
     </strong>
     <strong>
      、网络的可靠性
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     网络的可靠性指当设备或者链路出现单点或者多点故障时保证网络服务不间断的能力。
    </p>
    <p style="margin-left:0; margin-right:0">
     网络的可靠性可以从单板、设备、链路多个层面实现。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="260" src="https://i-blog.csdnimg.cn/direct/fe8f984cebb3465c9e045d3db205c45a.png" width="660"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     随着网络的快速普及和应用的日益深入，各种增值业务得到了广泛部署，网络中断可能导致大量业务异常、造成重大经济损失。因此，作为承载业务主体的基础网络，其可靠性成为备受关注的焦点。
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      1.2
     </strong>
     <strong>
      、单板可靠性
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="681" src="https://i-blog.csdnimg.cn/direct/2f3c0c69362b496cb561935a2065148a.png" width="570"/>
    </p>
    <ul>
     <li>
      框式交换机由机框、电源模块、风扇模块、主控板、交换网板（SFU）、线路板（LPU）构成。
     </li>
     <li>
      机框：为各种板卡、模块提供插槽，实现板卡间的通信。
     </li>
     <li>
      电源模块：设备的供电系统
     </li>
     <li>
      风扇模块：设备的散热系统
     </li>
     <li>
      主控板（MPU，Main Processing Unit）：负责整个系统的控制平面和管理平面。
     </li>
     <li>
      交换网板（SFU，Switch Fabric Unit）：负责整个系统的数据平面。数据平面提供高速无阻塞数据通道，实现各个业务模块之间的业务交换功能。
     </li>
     <li>
      线路板（LPU，Line Processing Unit）：线路处理单元是物理设备上用于提供数据转发功能的模块，提供不同速率的光口、电口。
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="590" src="https://i-blog.csdnimg.cn/direct/9e1a7ed1642c4de49e5fc0e2badf0f4c.png" width="683"/>
    </p>
    <ul>
     <li>
      以S12700E-8为例，设备提供8个线路板槽位、4个交换网板槽位、2个主控板槽位、6个电源模块槽位、4个风扇模块槽位。
     </li>
     <li>
      框式交换机配置多个主控板、交换网板可保证设备自身的可靠性，单个槽位的交换网板、主控板损坏不影响设备的正常运行。
     </li>
     <li>
      框式交换机的线路板损坏后，该板卡上的接口无法正常转发数据。
     </li>
    </ul>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      1.3
     </strong>
     <strong>
      、设备可靠性
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     设备无冗余设计的网络中，下游交换机采用单上行接入，上行交换机的接口故障或设备故障会导致下游网络全部中断。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="411" src="https://i-blog.csdnimg.cn/direct/4c735f3328d14244ace1da5fb9ac49b7.png" width="570"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     设备冗余设计的网络中，下游交换机双上行接入，采用链路一主一备的方式，主链路上行接口、设备故障可以切换到备份链路，通过备份设备转发。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="417" src="https://i-blog.csdnimg.cn/direct/bf4fb0363c194c77bb9046cb5f01fb8e.png" width="697"/>
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     1.4、链路可靠性
    </h3>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="531" src="https://i-blog.csdnimg.cn/direct/4a90dccba0c54f0db0071a5493718dfd.png" width="1275">
      为保证设备间链路可靠性，在设备间部署多条物理线路，为防止环路STP只保留一条链路转发流量，其余链路成为备份链路。
     </img>
    </p>
    <h2 style="margin-left:0px; margin-right:0px">
     <strong>
      2
     </strong>
     <strong>
      、链路聚合技术原理与配置
     </strong>
    </h2>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      2.1
     </strong>
     <strong>
      、基本原理
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     链路聚合：Eth-trunk，通过虚拟化（多虚一）将多条物理链路逻辑上整合为一条链路。
    </p>
    <p style="margin-left:0; margin-right:0">
     逻辑上的链路是STP计算的链路（即STP不会再计算物理链路）。
    </p>
    <p style="margin-left:0; margin-right:0">
     Eth-trunk设计了相应的机制防止多条物理链路出现环路：在一个链路捆绑组中的物理接口1收到的数据，不会再从链路捆绑组中的其它物理接口转发。
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.1.1
     </strong>
     <strong>
      、提升链路带宽
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="388" src="https://i-blog.csdnimg.cn/direct/af9e625f643a476a96c7b86ebece5387.png" width="696"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     设备之间存在多条链路时，由于STP的存在，实际只会有一条链路转发流量，设备间链路带宽无法得到提升。
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.1.2
     </strong>
     <strong>
      、以太网链路聚合
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="335" src="https://i-blog.csdnimg.cn/direct/6c15a8f52ded499ea5b694dede342d73.png" width="685"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     以太网链路聚合Eth-Trunk：简称链路聚合，通过将多个物理接口捆绑成为一个逻辑接口，可以在不进行硬件升级的条件下，达到增加链路带宽的目的。
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.1.3
     </strong>
     <strong>
      、链路聚合基本术语
     </strong>
     <strong>
      /
     </strong>
     <strong>
      概念
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="443" src="https://i-blog.csdnimg.cn/direct/11dc68dca0b5440d909a3e2a83d2d8de.png" width="493"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A1
     </strong>
     <strong>
      、聚合组（Link Aggregation Group，LAG）：
     </strong>
     若干条链路捆绑在一起所形成的的逻辑链路。每个聚合组唯一对应着一个逻辑接口，这个逻辑接口又被称为链路聚合接口或Eth-Trunk接口。
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A2
     </strong>
     <strong>
      、成员接口和成员链路：
     </strong>
     组成Eth-Trunk接口的各个物理接口称为成员接口。成员接口对应的链路称为成员链路。
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A2.1
     </strong>
     <strong>
      、活动接口和活动链路：
     </strong>
     活动接口又叫选中（Selected）接口，是参与数据转发的成员接口。活动接口对应的链路被称为活动链路（Active link）
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A2.2
     </strong>
     <strong>
      、非活动接口和非活动链路：
     </strong>
     又叫非选中（Unselected）接口，是不参与转发数据的成员接口。非活动接口对应的链路被称为非活动链路（Inactive link）。
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A3
     </strong>
     <strong>
      、聚合模式 ：
     </strong>
     根据是否开启LACP（Link Aggregation Control Protocol，链路聚合控制协议），链路聚合可以分为
     <u>
      手工模式
     </u>
     和
     <u>
      LACP模式
     </u>
     。
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      A4
     </strong>
     <strong>
      、其他概念：
     </strong>
     活动接口上限阈值和活动接口下限阈值。
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      2.2
     </strong>
     <strong>
      、手工模式
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     设备老旧、低端，不支持LACP协议，我们就采用手工模式。
    </p>
    <p style="margin-left:0; margin-right:0">
     手工模式由缺点，是 单通 的。
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1]interface Eth-Trunk
     </span>
     <span style="background-color:lime">
      1
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       创建一个聚合组
      </em>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1-Eth-Trunk1]mode manual load-balance
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       设置工作模式为手工静态（默认的模式）
      </em>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1-Eth-Trunk1]trunkport GigabitEthernet 0/0/1 0/0/2
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       将物理接口添加进聚合组
      </em>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1]int
     </span>
     <span style="background-color:lime">
      gi
     </span>
     <span style="background-color:lime">
      0/0/3
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1-GigabitEthernet0/0/3]eth-trunk
     </span>
     <span style="background-color:lime">
      1
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>

    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.2.1
     </strong>
     <strong>
      、手工模式的
     </strong>
     <strong>
      Eth-T
     </strong>
     <strong>
      runk的问题：
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     A1、链路两端不能相互检测，容易造成单通问题：
    </p>
    <p style="margin-left:0; margin-right:0">
     A1.1、没有报文检测机制；
    </p>
    <p style="margin-left:0; margin-right:0">
     A1.2、只要接口是UP状态，就认为对方可达。
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      情景
     </strong>
     <strong>
      1
     </strong>
     <strong>
      、没有报文检测机制：
     </strong>
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="158" src="https://i-blog.csdnimg.cn/direct/45f197263cb44bdd9d75c971ea9a3405.png" width="453"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     S1交换机是Eth-Trunk端口，与S2相连且为RP口。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="509" src="https://i-blog.csdnimg.cn/direct/f56d03a0bd8f425fa2f7b83db6c57e8b.png" width="734"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     S2交换机有3个端口与S1的Eth-Trunk连接，且都为DP口。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="135" src="https://i-blog.csdnimg.cn/direct/0d64bb90a086493497d1c2da1bcae60c.png" width="914"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     现在，两端没有AP阻塞口，一定会：
    </p>
    <p style="margin-left:0; margin-right:0">
     形成小的环路；
    </p>
    <p style="margin-left:0; margin-right:0">
     S2不知该通过哪个接口与S1通信？
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      情景
     </strong>
     <strong>
      2
     </strong>
     <strong>
      、只要接口是
     </strong>
     <strong>
      UP
     </strong>
     <strong>
      状态，就认为对方可达：
     </strong>
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="229" src="https://i-blog.csdnimg.cn/direct/994f57cb9fea40ea9e4259ded2741bd7.png" width="414"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     在S1 GE02口与S2断开的情况下，S1交换机的Eth-Trunk三个口都是正常连接的：
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="163" src="https://i-blog.csdnimg.cn/direct/81b5fe2045a3421da58726a9b29f0c00.png" width="733"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     而S2与S1连接的GE 02口已断开。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="113" src="https://i-blog.csdnimg.cn/direct/cfa7bef55fe34078919df37fbcd7a9b8.png" width="910"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     因为Eth-Trunk手工聚合通信发送报文接口是随机的，所以当S2向S1发送的报文通过S1的GE 02口发出时，就会出现单通。
    </p>

    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3
     </strong>
     <strong>
      、
     </strong>
     <strong>
      LACP
     </strong>
     <strong>
      模式
     </strong>
    </h3>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.1
     </strong>
     <strong>
      、特点
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">

    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.2
     </strong>
     <strong>
      、链路聚合两端的带宽
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     链路聚合的“一端”要求是同速率，同带宽。
    </p>
    <p style="margin-left:0; margin-right:0">
     链路聚合的“两端”不是同速率、同带宽的链路的情况，则向下兼容（
     <u>
      一端为百兆链路、一端为千兆链路，则取
     </u>
     <strong>
      <u>
       百兆链路
      </u>
     </strong>
     <u>
      为最大带宽
     </u>
     ）。
    </p>
    <p style="margin-left:0; margin-right:0">
     链路聚合的两端，聚合组号没有一致性要求。
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.3
     </strong>
     <strong>
      、在聚合组中确定某接口为备份接口
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     每一个接口都有对应的LACP接口优先级，优先级值越小越高；
    </p>
    <p style="margin-left:0; margin-right:0">
     优先级值相等，则比较接口编号，越小越高。
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S2-GigabitEthernet0/0/1]lacp priority
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       设置接口的
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       LACP
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       优先级，只在备份接口未选举前生效
      </em>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     如下，即时变更了0/0/1口的优先级，0/0/3口依然是备份口，只有等到下一次主备选举，才能生效。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="114" src="https://i-blog.csdnimg.cn/direct/7dba2f98c26e4fdab1a620c7dded47af.png" width="1012"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     在聚合链路的两端，需要选举一个主动端设备来确定备份接口的选择，被动端则使用主动端选举的结果。
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.4
     </strong>
     <strong>
      、如何选举主动端
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     每一个运行了LACP的交换机都会存在LACP系统优先级：
    </p>
    <p style="margin-left:0; margin-right:0">
     优先级值越小越高；
    </p>
    <p style="margin-left:0; margin-right:0">
     优先级值相同，则比较MAC地址，越小越高。
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S1]lacp priority ?
     </span>
     <span style="background-color:lime">
      <em>
       #
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       修改
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       LACP
      </em>
     </span>
     <span style="background-color:lime">
      <em>
       全局优先级
      </em>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      INTEGER&lt;0-65535&gt;  Priority value, the default value is 32768
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="192" src="https://i-blog.csdnimg.cn/direct/3718df2d18fd414abc27cd4900b9ca9a.png" width="879"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="165" src="https://i-blog.csdnimg.cn/direct/2036ea46dcd64a799eaca434c129390f.png" width="877"/>
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.5
     </strong>
     <strong>
      、负载分担
     </strong>
    </h4>
    <h5 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.5.1
     </strong>
     <strong>
      、基于包
     </strong>
    </h5>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="207" src="https://i-blog.csdnimg.cn/direct/26568ec2f7c74d4c8b168a3b91eb182d.png" width="619"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     在使用Eth-Trunk转发数据时，由于聚合组两端设备之间有多条物理链路，如果每个数据帧在不同的链路上转发，则有可能导致数据帧到达对端时间不一致，从而引起数据乱序。
    </p>
    <h5 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.5.2
     </strong>
     <strong>
      、基于流（华为设备默认基于流转发）
     </strong>
    </h5>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="226" src="https://i-blog.csdnimg.cn/direct/cee7c34eb6414891bb3ce58777e9987b.png" width="622"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     Eth-Trunk推荐采用逐流负载分担的方式，即一条相同的流负载到一条链路，这样既保证了同一数据流的数据帧在同一条物理链路转发，又实现了流量在聚合组内各物理链路上的负载分担。
    </p>
    <p style="margin-left:0; margin-right:0">
     常见的模式有：源IP、源MAC、目的IP、目的MAC、源目IP、源目MAC。
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      [S2-Eth-Trunk1]load-balance ?
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      dst-ip
     </span>
     <span style="background-color:lime">
      According to destination IP hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      dst-mac   According to destination MAC hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      src-dst-ip According to source/destination IP hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      src-dst-mac  According to source/destination MAC hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      src-ip       According to source IP hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <span style="background-color:lime">
      src-mac   According to source MAC hash arithmetic
     </span>
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="339" src="https://i-blog.csdnimg.cn/direct/d17cab8083c64a04ac97e944fc7ed79d.png" width="1483"/>
    </p>
    <h4 style="margin-left:0px; margin-right:0px">
     <strong>
      2.3.x
     </strong>
     <strong>
      、扩展
     </strong>
    </h4>
    <p style="margin-left:0; margin-right:0">
     链路聚合的成员接口都会发送LACPDU（不管是活跃链路还是非活跃链路）；

    <p style="margin-left:0; margin-right:0">
     在多个活跃的成员接口中，会随机选择一个接口发送STP报文。

    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      3.1
     </strong>
     <strong>
      、什么是堆叠、集群
     </strong>
    </h3>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="307" src="https://i-blog.csdnimg.cn/direct/26cf7c1200dc424b8d25cf93ac84e894.png" width="634"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     堆叠（iStack）：多台支持堆叠特性的交换机通过堆叠线缆连接在一起，从逻辑上变成一台交换设备，作为一个整体参与数据转发。主要针对盒式设备，最大支持16台设备融合组网。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="323" src="https://i-blog.csdnimg.cn/direct/9071e16c934149a8aa40f1416aa316c5.png" width="630"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     集群（Cluster Switch System，CSS ）：将两台支持集群特性的交换机设备组合在一起，从逻辑上组合成一台交换设备。
    </p>
    <p style="margin-left:0; margin-right:0">
     主要针对框式设备，最大支持2台设备融合组网，一般做核心。
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     <strong>
      3.2
     </strong>
     <strong>
      、堆叠、集群的优势

    <p style="margin-left:0; margin-right:0">
     <img alt="" height="227" src="https://i-blog.csdnimg.cn/direct/dd21d74916b4437fafe3db7dc92a9be9.png" width="307"/>

    <p style="margin-left:0; margin-right:0">
     <strong>
      交换机多虚一：
     </strong>
     堆叠交换机对外表现为一台逻辑交换机，控制平面合一，统一管理。
    </p>
    <p style="margin-left:0; margin-right:0">
