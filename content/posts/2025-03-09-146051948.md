     1.3 zip/unzip 指令
    </h4>
    <p>
     <strong>
      语法:
     </strong>
     zip 压缩文件.zip 目录或文件
     <br/>
     <strong>
      功能:
     </strong>
     将目录或文件压缩成zip格式
     <br/>
     <strong>
      常用选项
     </strong>
    </p>
    <ul>
     <li>
      r:递归处理，将指定目录下的所有文件和子目录一并处理
      <br/>
      <strong>
       举例
      </strong>
      <br/>
      将test2目录压缩:zip test2.zip test2/*
      <br/>
      解压到tmp目录:unzip test2.zip -d /tmp
      <br/>
      <strong>
       关于 rzsz
      </strong>
      <br/>
      这个工具用于 windows 机器和远端的 Linux机器通过 XShell 传输文件,安装完毕之后可以通过拖拽的方式将文件上传过去
     </li>
    </ul>
    <pre><code class="prism language-bash"> <span class="token function">sudo</span> yum/apt <span class="token function">install</span> <span class="token parameter variable">-y</span> lrzlz
</code></pre>

     现场演示windows和Linux如何互传文件

    <h4>
     <a id="14_tar_102">
     </a>
     1.4 tar指令(重要)
    </h4>
    <p>
     打包/解包，不打开它，直接看内容
     <br/>
     <strong>
      语法:
     </strong>
     tar[-cxtzjvf] 文件与目录 …
     <br/>
     <strong>
      参数:
     </strong>
    </p>
    <ul>
     <li>
      c:建立一个压缩文件的参数指令(create 的意思);
     </li>
     <li>
      x:解开一个压缩文件的参数指令!
     </li>
     <li>
      t:查看 tarfile 里面的文件!
     </li>
     <li>
      z:是否同时具有 gzip 的属性?亦即是否需要用 gzip 压缩?
     </li>
     <li>
      j:是否同时具有 bzip2 的属性?亦即是否需要用 bzip2 压缩?
     </li>
     <li>
      v:压缩的过程中显示文件!这个常用，但不建议用在背景执行过程!
     </li>
     <li>
      f:使用档名，请留意，在f之后要立即接档名喔!不要再加参数!
     </li>
     <li>
      C:解压到指定目录
      <br/>
      案例:
      <br/>
      <strong>
       范例一:将整个/etc目录下的文件全部打包成为’/tmp/etc.tar
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span> $ <span class="token function">tar</span> - cvf / tmp / etc.tar / etc <span class="token operator">&lt;=</span> <span class="token operator">=</span>仅打包，不压缩！
<span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> - zcvf / tmp / etc.tar.gz / etc <span class="token operator">&lt;=</span> <span class="token operator">=</span>打包后，以gzip压缩
<span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> - jcvf / tmp / etc.tar.bz2 / etc <span class="token operator">&lt;=</span> <span class="token operator">=</span>打包后，以bzip2压缩
</code></pre>
    <p>
     特别注意，在参数f之后的文件档名是自己取的，我们习惯上都用.tar 来作为辨识。
     <br/>
     如果加z参数，则以.tar.gz或.tgz来代表 gzip 压缩过的 tar file ~
     <br/>
     如果加j参数，则以 .tar.bz2 来作为附档名啊~
     <br/>
     上述指令在执行的时候，会显示一个警告讯息:
     <br/>
     Г`tar: Removingleading`/"from member names`』那是关于绝对路径的特殊设定。
    </p>
    <p>
     <strong>
      范例二:查阅上述 /tmp/etc.tar.gz文件内有哪些文件?
     </strong>
    </p>
    <pre><code class="prism language-bash"> <span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-ztvf</span> /tmp/etc.tar.gz
</code></pre>
    <p>
     由于我们使用 gzip 压缩，所以要查阅该 tar file 内的文件时，就得要加上z这个参数了!这很重要的!
    </p>
    <p>
     <strong>
      范例三:将 /tmp/etc.tar.gz文件解压缩在 /usr/local/src 底下
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token builtin class-name">cd</span> /usr/local/src
 <span class="token punctuation">[</span>root@linux src<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-zxvf</span> /tmp/etc.tar.gz
</code></pre>
    <p>
     在预设的情况下，我们可以将压缩档在任何地方解开的!以这个范例来说，我先将工作目录变换到/usr/local/src 底下，并且解开 /tmp/etc.tar.gz，则解开的目录会在/usr/local/src/etc呢!另外，如果您进入/usr/local/src/etc
     <br/>
     则会发现，该目录下的文件属性与 /etc/可能会有所不同喔!
    </p>
    <p>
     <strong>
      范例四:在 /tmp 底下，我只想要将/tmp/etc.tar.gz内的 etc/passwd 解开而已
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token builtin class-name">cd</span> /tmp
 <span class="token punctuation">[</span>root@linux tmp<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-zxvf</span> /tmp/etc.tar.gz etc/passwd
</code></pre>
    <p>
     我可以透过 tar -ztvf 来查阅 tarfile 内的文件名称，如果单只要一个文件，就可以透过这个方式来下达!注意到! etc.tar.gz内的根目录/是被拿掉了!
    </p>
    <p>
     <strong>
      范例五:将 /etc/内的所有文件备份下来，并且保存其权限!
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-zxvpf</span> /tmp/etc.tar.gz /etc
 <span class="token comment"># 这个 -p 的属性是很重要的，尤其是当您要保留原本⽂件的属性时！</span>
</code></pre>
    <p>
     <strong>
      范例六:在/home 当中，比2005/06/01新的文件才备份
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-N</span> <span class="token string">"2005/06/01"</span> <span class="token parameter variable">-zcvf</span> home.tar.gz /home
</code></pre>
    <p>
     <strong>
      范例七:我要备份/home,/etc，但不要/home/dmtsai
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">--exclude</span> /home/dmtsai <span class="token parameter variable">-zcvf</span> myfile.tar.gz /home/* /etc
</code></pre>
    <p>
     <strong>
      范例八:将/etc/打包后直接解开在 /tmp 底下，而不产生文件!
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@linux ~<span class="token punctuation">]</span>$ <span class="token builtin class-name">cd</span> /tmp
 <span class="token punctuation">[</span>root@linux tmp<span class="token punctuation">]</span>$ <span class="token function">tar</span> <span class="token parameter variable">-cvf</span> - /etc <span class="token operator">|</span> <span class="token function">tar</span> <span class="token parameter variable">-xvf</span> 
</code></pre>
    <p>
     这个动作有点像是 cp -r/etc /tmp 啦~依旧是有其有用途的!
     <br/>
     要注意的地方在于输出档变成-而输入档也变成-，又有一个存在~
     <br/>
     这分别代表 standard output,standard input 与管线命令啦!
     <br/>
     现场演示如何在Linux和Linux之间互传压缩包
    </p>

    <h4>
     <a id="16_unamer_184">
     </a>
     1.6 uname-r指令
    </h4>
    <p>
     <strong>
      语法:
     </strong>
     uname[选项]
     <br/>
     <strong>
      功能:
     </strong>
     uname用来获取电脑和操作系统的相关信息。
     <br/>
     <strong>
      补充说明:
     </strong>
     uname可显示linux主机所用的操作系统的版本、硬件的名称等基本信息。
     <br/>
     <strong>
      常用选项:
     </strong>
    </p>
    <ul>
     <li>
      a或-all 详细输出所有信息，依次为内核名称，主机名，内核版本号，内核版本，硬件名，处理器类型，硬件平台类型，操作系统名称
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/direct/18890b3f424f435dbcfa8d72ef71232a.png"/>
     </li>
    </ul>
    <p>
     <strong>
      重要的几个热键[Tab],[ctrl]-c,[ctrl]-d
     </strong>
     <br/>
     [Tab]按键—具有『命令补全』和『档案补齐』的功能
     <br/>
     [Ctrl]-c按键—让当前的程序『停掉」
     <br/>
     [Ctrl]-d按键—通常代表着:『键盘输入结束(End Of File,EOF 戒 End Oflnput)』的意思;另外,他也可以用来取代exit
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/13a4f2f16888412396a41b5f363016f6.png"/>
    </p>
    <h4>
     <a id="17__199">
     </a>
     1.7 关机
    </h4>
    <p>
     <strong>
      语法:
     </strong>
     shutdown[选项
     <br/>
     <strong>
      常见选项:
     </strong>
    </p>
    <ul>
     <li>
      h:将系统的服务停掉后，立即关机。
     </li>
     <li>
      r:在将系统的服务停掉之后就重新启动
     </li>
     <li>
      tsec:-t后面加秒数，亦即『过几秒后关机』的意思
     </li>
    </ul>
    <p>
     <strong>
      以下命令作为扩展
     </strong>
    </p>
    <ul>
     <li>
      安装和登录命令:login、shutdown、halt、reboot、install、mount、umount、chsh、exit、 last;
     </li>
     <li>
      文件处理命令:file、mkdir、grep、dd、find、mv、ls、diff、cat、ln;
     </li>
     <li>
      系统管理相关命令:df、top、free、quota、at、lp、adduser、groupadd、kill、crontab;
     </li>
     <li>
      网络操作命令:ifconfig、ip、ping、netstat、telnet、ftp、route、rlogin、rcp、finger、
      <br/>
      mail、nslookup;
     </li>
     <li>
      系统安全相关命令:passwd、su、umask、chgrp、chmod、chown、chattr、sudo ps、who;
     </li>
     <li>
      其它命令:tar、unzip、gunzip、unarj、mtools、man、unendcode、uudecode。
     </li>
    </ul>
    <h3>
     <a id="shell_214">
     </a>
     二.shell命令以及运行原理
    </h3>
    <p>
     Linux严格意义上说的是一个操作系统，我们称之为“核心(kernel)，但我们一般用户，不能直接使用kernel。而是通过kernel的“外壳”程序，也就是所谓的shell，来与kernel沟通。如何理解?为什么不能直接使用kernel?
    </p>
    <p>
     从技术角度，Shell的最简单定义:命令行解释器(commandInterpreter)主要包含:
    </p>
    <ul>
     <li>
      将使用者的命令翻译给核心(kernel)处理。
     </li>
     <li>
      同时，将核心的处理结果翻译给使用者。
     </li>
    </ul>
    <p>
     <strong>
      对比windowsGUl
     </strong>
     ，我们操作windows不是直接操作windows内核，而是通过图形接口，点击，从而完成我们的操作(比如进入D盘的操作，我们通常是双击D盘盘符.或者运行起来一个应用程序)
     <br/>
     <strong>
      shell对于Linux
     </strong>
     ，有相同的作用，主要是对我们的指令进行解析，解析指令给Linux内核。反馈结果在通过内核运行出结果，通过shell解析给用户。帮助理解:如果说你是一个闷骚且害羞的程序员，那shell就像媒婆，操作系统内核就是你们村头漂亮的且有让你心动的MM小花。你看上了小花，但是有不好意思直接表白，那就让你你家人找媒婆帮你提亲，所有的事情你都直接跟媒婆沟通，由媒婆转达你的意思给小花，而我们找到媒婆姓王，所以我们叫它王婆，它对应我们常使用的bash。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/e1e8d5f677b3431eab9088d3de8282d8.png"/>
    </p>
    <h3>
     <a id="Linux_225">
     </a>
     三.Linux权限的概念
    </h3>
    <p>
     <strong>
      Linux下有两种用户：超级用户(root)、普通用户。
     </strong>
    </p>
    <ul>
     <li>
      超级用户：可以再linux系统下做任何事情，不受限制
     </li>
     <li>
      普通用户：在linux下做有限的事情。
     </li>
     <li>
      超级用户的命令提示符是“#”，普通用户的命令提示符是“$”。
      <br/>
      <strong>
       命令：
      </strong>
      su[用户名]
      <br/>
      <strong>
       功能：
      </strong>
      切换用户。
      <br/>
      例如，要从root用户切换到普通用户user，则使用suuser。要从普通用户user切换到root用户则使用suroot(root可以省略)，此时系统会提示输入root用户的口令。
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/direct/63fe7dac5e9448c5bb0c64ccf92fd107.png"/>
     </li>
    </ul>
