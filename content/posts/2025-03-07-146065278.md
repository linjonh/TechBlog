    <h2 style="background-color:transparent">
     前言
    </h2>
    <h3>
     web开发
    </h3>
    <p>
     web开发：Web开发是指在万维网或私有网络上创建和维护网站的工作。它包括网页设计、网页编程、数据库管理等多方面的技术。Web开发可以分为前端开发和后端开发，前端主要关注用户界面和用户体验，而后端则处理服务器、应用程序和数据库之间的交互。
    </p>
    <p>
     <strong>
      静态web：
     </strong>
    </p>
    <ul>
     <li>
      html、css。提供给所有人看的数据始终不会发生改变。
     </li>
    </ul>
    <p>
     <strong>
      动态web：
     </strong>
    </p>
    <ul>
     <li>
      淘宝，几乎是所有的网站都是动态的
     </li>
     <li>
      提供给所有人看的数据始终会发生改变。每个人在不同的时间，不同的地点看到的信息各不相同。
     </li>
     <li>
      技术栈：Servlet/JSP，ASP，PHP
     </li>
    </ul>
    <p>
     在Java中，动态web资源开发的技术统称未javaWeb
    </p>
    <h3>
     web应用程序
    </h3>
    <p>
     web应用程序：可以提供浏览器访问的程序
    </p>
    <ul>
     <li>
      a.html，b.html...多个web资源，这些资源可以被外界访问，对外界提供服务
     </li>
     <li>
      可以访问的任何一个页面或者资源，都存在于这个世界上的某一台计算机上。
     </li>
     <li>
      URL
     </li>
     <li>
      统一的web资源会被放在同一个文件夹下，web应用程序-&gt;Tomcat（服务器）
     </li>
     <li>
      一个web应用由多部分组成（静态web，动态web）
      <ul>
       <li>
        html、css、js
       </li>
       <li>
        jsp，servlet
       </li>
       <li>
        Java程序
       </li>
       <li>
        jar包
       </li>
       <li>
        配置文件（Properties）
       </li>
      </ul>
     </li>
    </ul>
    <p>
     web应用程序编写完毕后，若想提供给外界访问：需要一个服务器来统一管理；
    </p>
    <h3>
     静态web
    </h3>
    <p>
     以htm、html等为网页后缀，如果服务器上存在，我们可以直接读取。
    </p>
    <p>
     <img alt="" height="254" src="https://i-blog.csdnimg.cn/direct/6b99edc4205c41fbbdda4626da873fd1.png" width="785"/>
    </p>
    <p>
     <strong>
      缺点：
     </strong>
    </p>
    <ul>
     <li>
      Web页面无法动态更新，所有用户看到的都是同一个页面
      <ul>
       <li>
        轮播图，点击特效：伪动态
       </li>
       <li>
        JavaScript实际开发比较多
       </li>
       <li>
        VBScript
       </li>
      </ul>
     </li>
     <li>
      无法于数据库交互（数据无法持久化，用户无法交互）
     </li>
    </ul>
    <h3>
     动态web
    </h3>
    <p>
     页面会动态展示，“web的页面展示效果因人而异”
    </p>
    <p>
     <img alt="" height="448" src="https://i-blog.csdnimg.cn/direct/add8dcdcafe44db18086f7234331ee76.png" width="872"/>
    </p>
    <p>
     <strong>
      缺点：
     </strong>
    </p>
    <ul>
     <li>
      服务器的动态web资源出现了错误，我们需要出现编写后台程序，重新发布
      <ul>
       <li>
        停机维护
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      优点：
     </strong>
    </p>
    <ul>
     <li>
      Web页面可以动态更新，所有用户看到的都不是同一个页面
     </li>
     <li>
      可以与数据库交互（数据持久化，用户能够交互,注册，商品信息 ，用户信息）
     </li>
    </ul>
    <p>
     <img alt="" height="624" src="https://i-blog.csdnimg.cn/direct/04b2c9facbb14f66ab2aa91d630f9629.png" width="915"/>
    </p>
    <h3>
     web服务器
    </h3>
    <h3>
     技术
    </h3>
    <p>
     ASP：
    </p>
    <ul>
     <li>
      微软的，国内最早流行的
     </li>
     <li>
      在HTML中嵌入了VB脚本，ASP+COM；
     </li>
     <li>
      在ASP开发中，一个页面几乎都有几千行业务代码，页面混乱
     </li>
     <li>
      维护成本高
     </li>
     <li>
      在ASP中常用C#
     </li>
    </ul>
    <p>
     JSP/Servlet
    </p>
    <ul>
     <li>
      sun公司主推的B/S架构，B/S是浏览器和服务器。C/S是客户端和服务器
     </li>
     <li>
      基于Java语言（大部分开源的组件都是用Java写）
     </li>
     <li>
      可以承载三高问题带来的影响（高可用，高并发，高性能）
     </li>
     <li>
      语法像ASP
     </li>
    </ul>
    <p>
     PHP
    </p>
    <ul>
     <li>
      PHP开发速度很块，功能强大，跨平台，代码简单（适合中小型网站）
     </li>
     <li>
      无法承载大访问量的情况（局限性）
     </li>
    </ul>
    <p>
     ....
    </p>
    <h3>
     web服务器
    </h3>
    <p>
     服务器是一种被动的操作，用来处理用户的一些请求，给用户一些响应信息。
    </p>
    <p>
     <strong>
      IIS
     </strong>
    </p>
    <ul>
     <li>
      属于微软，windows中自带
     </li>
    </ul>
    <p>
     <strong>
      Tomcat
     </strong>
    </p>
    <p>
     Tomcat是
     <a href="https://baike.baidu.com/item/Apache/6265?fromModule=lemma_inlink" rel="nofollow" title="Apache">
      Apache
     </a>
     软件基金会（Apache Software Foundation）的
     <a href="https://baike.baidu.com/item/Jakarta/15952232?fromModule=lemma_inlink" rel="nofollow" title="Jakarta">
      Jakarta
     </a>
     项目中的一个核心项目，最新的Servlet 和
     <a href="https://baike.baidu.com/item/JSP/141543?fromModule=lemma_inlink" rel="nofollow" title="JSP">
      JSP
     </a>
     规范总是能在Tomcat 中得到体现，Tomcat 5支持最新的Servlet 2.4 和JSP 2.0 规范。因为Tomcat 技术先进、性能稳定，而且免费，因而深受
     <a href="https://baike.baidu.com/item/Java/85979?fromModule=lemma_inlink" rel="nofollow" title="Java">
      Java
     </a>
     爱好者的喜爱并得到了部分软件开发商的认可，成为比较流行的Web
     <a href="https://baike.baidu.com/item/%E5%BA%94%E7%94%A8%E6%9C%8D%E5%8A%A1%E5%99%A8/4971773?fromModule=lemma_inlink" rel="nofollow" title="应用服务器">
      应用服务器
     </a>
     。
    </p>
    <p>
     Tomcat 服务器是一个免费的
     <a href="https://baike.baidu.com/item/%E5%BC%80%E6%94%BE%E6%BA%90%E4%BB%A3%E7%A0%81/114160?fromModule=lemma_inlink" rel="nofollow" title="开放源代码">
      开放源代码
     </a>
     的
     <a href="https://baike.baidu.com/item/Web/150564?fromModule=lemma_inlink" rel="nofollow" title="Web">
      Web
     </a>
     应用服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试JSP 程序的首选。对于一个初学者来说，是最佳选择。
    </p>
    <p>
     当配置正确时，Apache 为HTML页面服务，而Tomcat 实际上运行JSP 页面和Servlet。Tomcat最新版本为11.0.0。
    </p>

    <h2>
     HTTP
    </h2>
    <p>
     HTTP（超文本传输协议）是一个简单的请求-响应协议，它通常运行在TCP之上。
    </p>
    <ul>
     <li>
      文本：html、字符串
     </li>
     <li>
      超文本：图片、音乐、视频、定位、地图
     </li>
     <li>
      默认端口号：80
     </li>
    </ul>
    <p>
     http1.0
    </p>
    <ul>
     <li>
      HTTP/1.0：客户端可以与web服务器连接，只能获得一个web资源，就会断开连接。
     </li>
    </ul>
    <p>
     http2.0
    </p>
    <ul>
     <li>
      HTTP1.1：客户端可以与web服务器连接，可以获得多个web资源。
     </li>
    </ul>
    <h3>
     <strong>
      Http请求
     </strong>
    </h3>
    <p>
     客户端-发送请求（Request）-服务器
    </p>
    <p>
     访问
     <strong>
      百度
     </strong>
     时抓取包：
    </p>
    <p>
     <img alt="" height="206" src="https://i-blog.csdnimg.cn/direct/d77b922aeab542e49ffe42f4d55a0eb0.png" width="560"/>

    <p>
     <strong>
      请求行
     </strong>
    </p>
    <ul>
     <li>
      请求行中的请求方式：GET
     </li>
     <li>
      请求方式：
      <strong>
       Get、Post
      </strong>
      、HEAD、DELETE、PUT、TRACT
      <ul>
       <li>
        get：请求能够携带的参数比较少，大小有限制，会在浏览器的URL地址栏显示数据内容，不安全，但高效。
       </li>
       <li>
        post：请求能够携带的参数无限制，大小有限制，不会在浏览器的URL地址栏显示数据内容，安全，但不高效。
       </li>
      </ul>
     </li>
    </ul>

     <strong>
      消息头
     </strong>
    </p>

      <strong>
       accept-language:告诉浏览器，它的语言环境
      </strong>
     </p>
     <p>
      <strong>
       cache-control:缓存控制
      </strong>
     </p>
     <p>
      <strong>
       connection:告诉浏览器，请求完成是断开还是保持连接
      </strong>
     </p>
     <p>
      <strong>
       HOST:主机.....
      </strong>
     </p>
    </blockquote>

    </h3>
    <p>
