    <h3>
     <a id="_0">
     </a>
     题目描述
    </h3>
    <p>
     在无限大的棋盘中有n个炮，第个炮的坐标是(xi,yi)。
    </p>
    <p>
     已知每个炮的攻击方式是:先选一个攻击方向(上、下、左、右），该方向上看见的第一个棋子为“炮架”，该炮可以通过炮架攻击到炮架后面的棋子(只能攻击到炮架后面的第一个)。
    </p>
    <p>
     小牛希望你求出每个炮第一次攻击能攻击到多少个炮。
    </p>
    <h3>
     <a id="_10">
     </a>
     输入描述
    </h3>
    <p>
     第一行输入一个正整数n，代表炮的数量。
     <br/>
     接下来的几行，每行输入两个整数xi,yi，代表每个炮所在的坐标。
     <br/>
     1&lt;=n&lt;=10^5
     <br/>
     -10^9 &lt;=xi,yi&lt;= 10^9
    </p>
    <h3>
     <a id="_19">
     </a>
     输出描述
    </h3>
    <p>
     输出几行，每行输出一个整数，代表第i个炮可以攻击到的炮的数量。
    </p>
    <h3>
     <a id="1_25">
     </a>
     示例1
    </h3>
    <pre><code>输入：
6 
0 0 
0 1 
0 2 
1 0 
2 0 
3 0

输出：
2
0
1
1
1
1
</code></pre>

     <h3>
      <a id="_109">
      </a>
      <strong>
       题目类型
      </strong>
     </h3>
     <p>
      本题属于
      <strong>
       哈希表 + 排序
      </strong>
      结合的题目，同时具有
      <strong>
       扫描线思想
      </strong>
      ，因为我们需要分别统计同一行和同一列上的炮的情况，以确定每个炮的攻击目标。
     </p>
     <hr/>
     <h3>
      <a id="_115">
      </a>
      <strong>
       解题思路
      </strong>
     </h3>
     <h4>
      <a id="1__117">
      </a>
      <strong>
       1. 题目解析
      </strong>
     </h4>
     <ul>
      <li>
       题目给定
       <strong>
        n 个炮的坐标
       </strong>
       ，炮的攻击方式是在选择一个方向（上、下、左、右）后，找到该方向上的
       <strong>
        第一个棋子作为炮架
       </strong>
       ，然后攻击炮架后面
       <strong>
        最近的炮
       </strong>
       。
      </li>
      <li>
       需要计算
       <strong>
        每个炮可以攻击到的炮的数量
       </strong>
       。
      </li>
     </ul>
     <h4>
      <a id="2__122">
      </a>
      <strong>
       2. 主要思路
      </strong>
     </h4>
     <p>
      为了高效求解，可以采用
      <strong>
       哈希表 + 排序
      </strong>
      的方法：
     </p>
     <ol>
      <li>
       <p>
        <strong>
         使用哈希表分类存储炮的位置
        </strong>
        ：
       </p>
       <ul>
        <li>
         用
         <strong>
          unordered_map&lt;int, vector&lt;pair&lt;int, int&gt;&gt;&gt; rows
         </strong>
         存储
         <strong>
          同一行
         </strong>
         的炮，key 为
         <strong>
          行号
         </strong>
         ，value 为
         <strong>
          （列号，炮的索引）
         </strong>
         。
        </li>
        <li>
         用
         <strong>
          unordered_map&lt;int, vector&lt;pair&lt;int, int&gt;&gt;&gt; cols
         </strong>
         存储
         <strong>
          同一列
         </strong>
         的炮，key 为
         <strong>
          列号
         </strong>
         ，value 为
         <strong>
          （行号，炮的索引）
         </strong>
         。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         对每一行、每一列的炮进行排序
        </strong>
        ：
       </p>
       <ul>
        <li>
         按照
         <strong>
          列号
         </strong>
         对同一行的炮进行排序，遍历后计算每个炮的可攻击数量。
        </li>
        <li>
         按照
         <strong>
          行号
         </strong>
         对同一列的炮进行排序，遍历后计算每个炮的可攻击数量。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         计算每个炮的攻击目标
        </strong>
        ：
       </p>
       <ul>
        <li>
         经过排序后，我们遍历
         <strong>
          每一行
         </strong>
         ，如果某个炮
         <strong>
          左侧第二个
         </strong>
         存在，它可以攻击左侧目标；如果
         <strong>
          右侧第二个
         </strong>
         存在，它可以攻击右侧目标。
        </li>
        <li>
         同样遍历
         <strong>
          每一列
         </strong>
         ，如果某个炮
         <strong>
          上侧第二个
         </strong>
         存在，它可以攻击上侧目标；如果
         <strong>
          下侧第二个
         </strong>
         存在，它可以攻击下侧目标。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         输出结果
        </strong>
        ：每个炮能攻击到的炮的总数。
       </p>
      </li>
     </ol>
     <h3>
      <a id="_144">
      </a>
      <strong>
       时间复杂度分析
      </strong>
     </h3>
     <ul>
      <li>
       <p>
        <strong>
         构造哈希表
        </strong>
        需要遍历所有炮，时间复杂度为
        <strong>
         O(n)
        </strong>
        。
       </p>
      </li>
      <li>
       <p>
        排序部分：
       </p>
       <ul>
        <li>
         每个行/列最多包含
         <strong>
          n
         </strong>
         个元素，排序的时间复杂度为
         <strong>
          O(k log k)
         </strong>
         ，其中
         <strong>
          k
         </strong>
         是该行/列的炮的数量。
        </li>
       </ul>
      </li>
      <li>
       <p>
        由于最坏情况下
        <strong>
         所有炮都在同一行或同一列
        </strong>
        ，排序时间复杂度为
        <strong>
         O(n log n)
        </strong>
        。
       </p>
       <ul>
        <li>
         <strong>
          遍历计算攻击目标
         </strong>
         也是
         <strong>
          O(n)
         </strong>
         。
        </li>
       </ul>
       <p>
        综上，
        <strong>
         总时间复杂度为 O(n log n)
        </strong>
        ，可以高效处理
        <strong>
         n ≤ 10^5
        </strong>
        的情况。
       </p>
      </li>
     </ul>
     <hr/>
     <h3>
      <a id="_159">
      </a>
      <strong>
       空间复杂度分析
      </strong>
     </h3>
     <ul>
      <li>
       <strong>
        哈希表
        <code>
         rows
        </code>
        和
        <code>
         cols
        </code>
       </strong>
       ：在最坏情况下，所有炮分布在同一行或列，存储
       <strong>
        O(n)
       </strong>
       个元素。
      </li>
      <li>
       <strong>
        数组
        <code>
         ans
        </code>
       </strong>
       ：大小为
       <strong>
        O(n)
       </strong>
       。
      </li>
      <li>
       <strong>
        排序的临时空间
       </strong>
       ：排序
       <strong>
        O(1)
       </strong>
       额外空间。
      </li>
     </ul>
     <p>
      综上，
      <strong>
       总空间复杂度为 O(n)
      </strong>
      。
     </p>
