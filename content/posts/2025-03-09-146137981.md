     docker已创建容器设置自启动
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <div class="htmledit_views" id="content_views">
    <p>
     在Docker中，你可以通过几种方式来设置容器在启动时自动运行。以下是几种常见的方法：
    </p>
    <h4>
     1. 使用
     <code>
      docker run
     </code>
     命令的
     <code>
      --restart
     </code>
     选项
    </h4>
    <p>
     当你使用
     <code>
      docker run
     </code>
     命令创建容器时，可以使用
     <code>
      --restart
     </code>
     选项来指定容器的重启策略。例如，如果你想让容器在退出后总是重启，可以使用：
    </p>
    <p>
     <code>
      <code>
       docker run -d --restart=always --name my-container my-image
      </code>
     </code>
    </p>
    <p>
     这里，
     <code>
      --restart=always
     </code>
     选项确保容器在退出后总是被重启。
    </p>
    <h4>
     2. 修改已存在的容器的重启策略
    </h4>
    <p>
     如果你已经有一个正在运行的容器，并希望更改其重启策略，可以使用
     <code>
      docker update
     </code>
     命令：
    </p>
    <p>
     <code>
      <code>
       docker update --restart=always my-container
      </code>
     </code>
    </p>
    <h4>
     3. 使用Docker Compose
    </h4>
    <p>
     如果你使用Docker Compose来管理你的容器，可以在
     <code>
      docker-compose.yml
     </code>
     文件中设置容器的重启策略。例如：
    </p>
    <p>
     <code>
      <code>
       version: '3'
      </code>
     </code>
    </p>
    <p>
     <code>
      <code>
       services:
      </code>
     </code>
    </p>
    <p>
     <code>
      <code>
       my-service:
      </code>
     </code>
    </p>
    <p>
     <code>
      <code>
       image: my-image
      </code>
     </code>
    </p>
    <p>
     <code>
      <code>
       restart: always
      </code>
     </code>
    </p>
    <p>
     然后使用
     <code>
      docker-compose up -d
     </code>
     来启动服务。
    </p>
