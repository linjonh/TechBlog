    <h2>
     <a id="_Builder_Pattern_0">
     </a>
     🚀 生成器模式（Builder Pattern）：让对象构建更优雅！
    </h2>
    <blockquote>
     <p>
      <strong>
       “遇到复杂对象构建？试试生成器模式！”
      </strong>
     </p>
    </blockquote>
    <p>
     在日常开发中，我们经常会遇到
     <strong>
      创建对象属性过多、构造方法过长、可选参数混乱
     </strong>
     的问题。这时候，我们就需要
     <strong>
      生成器模式（Builder Pattern）
     </strong>
     来简化对象的创建过程，让代码更加清晰、可维护。
    </p>
    <p>
     本篇文章基于
     <strong>
      《Head First 设计模式》
     </strong>
     ，带你深入理解
     <strong>
      生成器模式的核心思想、应用场景、优缺点，以及在 JDK 和 Spring 框架中的实践
     </strong>
     ，并提供 Java 代码示例，助你高效掌握这一重要设计模式！🚀
    </p>
    <hr/>
    <h3>
     <a id="__10">
     </a>
     🎭 先看问题：对象创建的痛点
    </h3>
    <p>
     假设我们有一个
     <strong>

      <code>

      </code>
      类
     </strong>
     ，有多个可选参数：

    <h4>
     <a id="__32">
     </a>
     ❌ 遇到的问题
    </h4>
    <p>
     1️⃣
     <strong>
      构造方法参数过多，调用时容易混淆
     </strong>
    </p>
    <pre><code class="prism language-java"><span class="token class-name">Product</span> product <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">Product</span><span class="token punctuation">(</span><span class="token string">"Laptop"</span><span class="token punctuation">,</span> <span class="token number">999.99</span><span class="token punctuation">,</span> <span class="token string">"Apple"</span><span class="token punctuation">,</span> <span class="token string">"Electronics"</span><span class="token punctuation">,</span> <span class="token number">100</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     如果参数顺序错了，比如把
     <code>
      price
     </code>
     和
     <code>
      stock
     </code>
     位置写反，就会导致
     <strong>
      难以发现的 Bug
     </strong>
     。
    </p>
    <p>
     2️⃣
     <strong>
      不支持部分参数可选
     </strong>
     <br/>
     比如，有些
     <code>
      Product
     </code>
     可能不需要
     <code>
      manufacturer
     </code>
     ，但是
     <strong>
      Java 没有默认参数
     </strong>
     ，我们只能重载多个构造方法，导致代码臃肿：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token class-name">Product</span><span class="token punctuation">(</span><span class="token class-name">String</span> name<span class="token punctuation">,</span> <span class="token keyword">double</span> price<span class="token punctuation">,</span> <span class="token class-name">String</span> category<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">this</span><span class="token punctuation">(</span>name<span class="token punctuation">,</span> price<span class="token punctuation">,</span> <span class="token string">"Unknown"</span><span class="token punctuation">,</span> category<span class="token punctuation">,</span> <span class="token number">0</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     3️⃣
     <strong>
      可读性差，扩展困难
     </strong>
     <br/>
     如果后续增加新属性，比如
     <code>
      discount
     </code>
     （折扣），就需要
     <strong>
      修改所有构造方法
     </strong>
     ，破坏了
     <strong>
      开闭原则（OCP）
     </strong>
     。
    </p>
    <hr/>
    <h3>
     <a id="__53">
     </a>
     🎯 解决方案：引入生成器模式
    </h3>
    <p>
     <strong>
      生成器模式（Builder Pattern）
     </strong>
     通过
     <strong>
      链式调用
     </strong>
     和
     <strong>
      分步骤构造
     </strong>
     ，让对象创建更加优雅！
    </p>

    <h4>
     <a id="__119">
     </a>
     ✅ 生成器模式的优势
    </h4>
    <p>
     🔹
     <strong>
      避免构造方法参数混乱
     </strong>
     ，可读性更强
     <br/>
     🔹
     <strong>
      支持可选参数
     </strong>
     ，避免构造方法重载
     <br/>
     🔹
     <strong>
      易于扩展
     </strong>
     ，添加新字段不影响已有代码
     <br/>
     🔹
     <strong>
      支持链式调用
     </strong>
     ，代码更加优雅
    </p>
    <hr/>
    <h3>
     <a id="__127">
     </a>
     🎯 生成器模式的适用场景
    </h3>
    <p>
     ✅
     <strong>
      创建复杂对象
     </strong>
     （参数过多、可选参数多的对象）
     <br/>
     ✅
     <strong>
      对象不可变
     </strong>
     （比如
     <code>
      String
     </code>
     ，创建后不可修改）
     <br/>
     ✅
     <strong>
      避免构造方法重载
     </strong>
     （减少代码膨胀）
     <br/>
     ✅
     <strong>
      提高代码可读性
     </strong>
     （链式调用，清晰直观）
    </p>

    <h3>
     <a id="__JDK__Spring__136">
     </a>
     🔍 生成器模式在 JDK 和 Spring 框架中的应用
    </h3>
    <h4>
     <a id="_1_JDK_StringBuilder_138">
     </a>
     📌
     <strong>
      1. JDK 中的应用：StringBuilder
     </strong>
    </h4>
    <p>
     <code>
      StringBuilder
     </code>
     采用
     <strong>
      生成器模式
     </strong>
     来高效拼接字符串：
    </p>
    <pre><code class="prism language-java"><span class="token class-name">String</span> result <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">StringBuilder</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">append</span><span class="token punctuation">(</span><span class="token string">"Hello "</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">append</span><span class="token punctuation">(</span><span class="token string">"World!"</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">toString</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     相比
     <code>
      String
     </code>
     的
     <strong>
      不可变性
     </strong>
     ，
     <code>
      StringBuilder
     </code>
     避免了不必要的对象创建，提高了性能！
    </p>
    <h4>
     <a id="_2_Spring_BeanDefinitionBuilder_148">
     </a>
     📌
     <strong>
      2. Spring 框架中的应用：BeanDefinitionBuilder
     </strong>
    </h4>
    <p>
     在 Spring 中，
     <code>
      BeanDefinitionBuilder
     </code>
     也是典型的
     <strong>
      生成器模式
     </strong>
     实现：
    </p>
    <pre><code class="prism language-java"><span class="token class-name">BeanDefinition</span> beanDefinition <span class="token operator">=</span> <span class="token class-name">BeanDefinitionBuilder</span>
        <span class="token punctuation">.</span><span class="token function">genericBeanDefinition</span><span class="token punctuation">(</span><span class="token class-name">MyService</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">addPropertyValue</span><span class="token punctuation">(</span><span class="token string">"name"</span><span class="token punctuation">,</span> <span class="token string">"Spring Service"</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">setScope</span><span class="token punctuation">(</span><span class="token class-name">BeanDefinition</span><span class="token punctuation">.</span><span class="token constant">SCOPE_SINGLETON</span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">getBeanDefinition</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     🔹
     <strong>
      可选参数不强制填充
     </strong>
     ，避免复杂构造方法
     <br/>
     🔹
     <strong>
      链式调用，简洁清晰
     </strong>
    </p>

        <strong>
         使用场景
        </strong>
       </td>
       <td>
        复杂对象创建，参数多
       </td>
       <td>
        只需创建对象，逻辑简单
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      📌 什么时候用？
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       参数少，逻辑简单
      </strong>
      →
      <strong>
       工厂模式
      </strong>
     </li>
     <li>
      <strong>
       参数多，构造复杂
      </strong>
      →
      <strong>
       生成器模式
      </strong>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="__177">
     </a>
     🎉 总结
    </h3>
    <p>
     <strong>
      生成器模式（Builder Pattern）
     </strong>
     解决了
     <strong>
      构造方法参数混乱、可选参数处理困难、扩展性差
     </strong>
     等问题，适用于
     <strong>
      创建复杂对象
     </strong>
     。
    </p>
    <p>
     📌
     <strong>
      核心要点：
     </strong>
     <br/>
     ✔️
     <strong>
      将对象构造过程与对象本身分离
     </strong>
     <br/>
     ✔️
     <strong>
      支持链式调用，避免构造方法重载
     </strong>
     <br/>
     ✔️
     <strong>
      可读性高，易扩展，适用于可选参数多的场景
     </strong>
     <br/>
     ✔️
     <strong>
      在 JDK（
      <code>
       StringBuilder
      </code>
      ）、Spring（
      <code>
       BeanDefinitionBuilder
      </code>
      ）中广泛应用
     </strong>
    </p>
    <p>
     如果你正在开发
     <strong>
      配置类、数据库连接类、复杂的实体类
     </strong>
     ，不妨试试
     <strong>
      生成器模式
     </strong>
     ，让你的代码更加优雅！💡
    </p>
