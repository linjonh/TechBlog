    <p>
     当你遇到名称以 .ipynb、.pyi、.pyc 等结尾的 Python 文件时，你是否会感到困惑？
    </p>
    <p>
     你想知道所有这些文件类型之间的区别吗？今天就和大家一起了解所有这些 Python 文件类型。🙂
    </p>
    <h3>
     <code>
      .py
     </code>
     文件：标准 Python 源代码
    </h3>
    <p>
     不用多说，每个懂 Python 的开发者都会把 Python 代码写到文件中
     <code>
      .py
     </code>
     ，直接执行就可以了，像这样：
    </p>
    <p>
     <img alt="" height="75" src="https://i-blog.csdnimg.cn/direct/d035446921fa4cf9987301fd4bfd6286.png" width="675"/>
    </p>
    <p>
     简单地说，这些
     <code>
      .py
     </code>
     文件是包含 Python 代码的可执行纯文本文件。
    </p>
    <h3>
     <code>
      .ipynb
     </code>
     文件：交互式 Jupyter Notebook
    </h3>
    <p>
     使用该扩展保存的 Jupyter Notebooks
     <code>
      .ipynb
     </code>
     对于数据分析和科学计算至关重要。
    </p>
    <p>
     这种类型的 Python 文件对于数据科学家来说更加用户友好。因为它提供了一个交互式环境，用于在单个文档中结合代码执行、数据可视化和叙述文本。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="图片" height="496" src="https://i-blog.csdnimg.cn/blog_migrate/89e09ec3a3409abfce7467a31758e1a3.png" width="700"/>
    </p>
    <p>
     Jupyter Notebook 制作的用于数据分析的精美 .ipynb 文件示例
    </p>
    <h3>
     <code>
      .pyi
     </code>
     文件：使用接口文件进行 Python 类型提示
    </h3>
    <p>
     Python 接口文件（带有扩展名的文件
     <code>
      .pyi
     </code>
     ），或称为 Python 存根文件，在使用类型提示增强 Python 代码方面发挥着至关重要的作用。
    </p>
    <p>
     例如，我们有一个名为
     <code>
      test.py
     </code>
     以下程序的 Python 文件：
    </p>
    <p>
     <img alt="" height="155" src="https://i-blog.csdnimg.cn/direct/50db5081531d439ba4bca2f5ef7cd9b2.png" width="677"/>
    </p>
    <p>
     IDE（我使用的是 PyCharm）或其他 Python 类型检查工具没有给它们任何类型提示。
    </p>
    <p>
     现在，创建一个与
     <code>
      .py
     </code>
     文件同名的
     <code>
      .pyi
     </code>
     文件：
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="图片" height="302" src="https://i-blog.csdnimg.cn/blog_migrate/6e71596fbd7022b37a6941e3395c987f.png" width="680"/>
    </p>
    <p style="text-align:center;">
     （在 PyCharm 中创建 Python 存根文件）
    </p>
    <p>
    </p>
    <p>
     如上图所示，在PyCharm中，我们可以在创建此文件时直接选择Python存根文件类型。
    </p>
    <p>
     现在将以下代码写入
     <code>
      test.pyi
     </code>
     文件中：
    </p>
    <p>
     <img alt="" height="72" src="https://i-blog.csdnimg.cn/direct/f8ffa40c929d4ccfaaa4f955a515a0cc.png" width="675"/>
    </p>
    <p>
     如果我们
     <code>
      test.py
     </code>
     再次打开该文件，IDE 会提醒我们现在不匹配的类型：
    </p>
    <p class="img-center">
     <img alt="图片" height="262" src="https://i-blog.csdnimg.cn/blog_migrate/c8729e0fe459c0debde91bcab2694033.png" width="700"/>
    </p>
    <p>
     它使你的程序类型更安全。但是你可能会问，为什么不直接在原始 Python 文件（
     <code>
      .py
     </code>
     文件）中添加类型提示呢？
    </p>
    <p>
     对于我们的例子，
     <code>
      .pyi
     </code>
     毫无疑问该文件并不是必要的，可以只在原始函数定义中声明
     <code>
      int
     </code>
     类型。
    </p>
    <p>
     <code>
      .pyi
     </code>
     将类型注释与实现分离可以使主代码保持干净并专注于功能。
    </p>
    <p>
     更重要的是，为了向后兼容，存根文件可以在不修改原始源文件的情况下为遗留代码添加类型提示。毕竟，修改以前同事写的旧代码是一件很头疼的事。😂
    </p>
    <h3>
     <code>
      .pyc
     </code>
     文件：编译后的 Python 字节码
    </h3>
    <p>
     当 Python 脚本作为模块导入时，解释器会将其编译为字节码并将其保存为
     <code>
      .pyc
     </code>
     文件，可以通过跳过不必要的重新编译来改善模块加载时间。
    </p>
    <p>
     这种类型的文件由解释器自动生成，它包含字节码，它是源代码（相对
     <code>
      .py
     </code>
     文件）的低级、与平台无关的表示形式。
    </p>
    <p>
     从本质上讲，字节码是 Python 的虚拟机解释和执行的内容。
    </p>
    <p>
     鉴于它是以二进制格式保存的并且不可读，因此在大多数情况下我们不需要过多担心
     <code>
      .pyc
     </code>
     文件。
    </p>
    <h3>
     <code>
      .pyw
     </code>
     文件：Windows GUI Python 脚本
    </h3>
    <p>
     Python 不仅适用于数据科学和后端开发。它也是一种用于图形用户界面 (GUI) 开发的语言。
    </p>
    <p>
     对于 Windows GUI，
     <code>
      .pyw
     </code>
     文件是为其设计的特定类型的 Python 脚本文件。这些文件用于在不打开命令提示符窗口的情况下运行 Python 脚本，使其成为 GUI 应用程序的理想选择。
    </p>
    <p>
     创建
     <code>
      .pyw
     </code>
     文件很简单——只需使用扩展名
     <code>
      .pyw
     </code>
     而不是保存
     <code>
      .py
     </code>
     脚本即可。
     <code>
      .pyw
     </code>
     文件可以像
     <code>
      .py
     </code>
     文件一样直接运行。
    </p>
    <p>
     例如通过tkinter编写一个简单的UI组件并将其保存为
     <code>
      .pyw
     </code>
     文件：
    </p>
    <p>
     <img alt="" height="519" src="https://i-blog.csdnimg.cn/direct/ae97ee21d4ce4178a76c3dfe26894a68.png" width="676"/>
    </p>
    <p>
     要执行它，我们只需在 Windows 资源管理器中双击该文件即可。或者，我们可以从命令行使用以下命令运行它
     <code>
      pythonw
     </code>
     ：
    </p>
    <p>
     <img alt="" height="77" src="https://i-blog.csdnimg.cn/direct/9b2b6ddae25b41efa025d717f2d4a017.png" width="677"/>
    </p>
    <h3>
     .pxd 和 .pyx 文件：Cython 源文件和声明文件
    </h3>
    <p>
     Cython 是一种编程语言，它使为 Python 编写 C 扩展变得像编写 Python 本身一样简单。它是Python 的
     <em>
      超集
     </em>
     ，旨在让主要用 Python 编写的代码提供类似 C 的性能。
    </p>
    <p>
     Cython中使用的两种重要文件类型是
     <code>
      .pyx
     </code>
     和
     <code>
      .pxd
     </code>
     文件。
    </p>
    <ul>
     <li>
      <p>
       <code>
        .pyx
       </code>
       <strong>
        文件：
       </strong>
       这些是包含主要实现代码的 Cython 源文件。文件
       <code>
        .pyx
       </code>
       中的代码可以是 Python 和类似 C 的语法的混合，允许你编写与 Python 无缝集成的高性能代码。
      </p>
     </li>
     <li>
      <p>
       <code>
        .pxd
       </code>
       <strong>
        文件：
       </strong>
       这些是 Cython 声明文件，类似于 C 头文件。它们包含类型定义、函数声明、变量和可在多个
       <code>
        .pyx
       </code>
       文件之间共享的类。
      </p>
     </li>
    </ul>
    <h3>
     <code>
      .pyd
     </code>
     文件：Python 动态模块
    </h3>
    <p>
     <code>
      .pyd
     </code>
     文件是 Python 动态模块。（不是Cython 的
     <code>
      .pxd
     </code>
     文件。）
    </p>
    <p>
     这些文件相当于 Windows 上的 DLL（动态链接库）。它允许 Python 与已编译的 C 或 C++ 代码交互，从而实现高性能和低级系统交互。
    </p>
    <p>
     要创建
     <code>
      .pyd
     </code>
     文件，我们通常可以用 C 或 C++ 编写代码，然后使用针对 Windows 平台的编译器对其进行编译。该过程通常涉及使用 Microsoft Visual Studio 或 MinGW 等工具。
    </p>
    <p>
     在游戏开发中，高性能游戏引擎和图形库通常利用
     <code>
      .pyd
     </code>
     文件进行渲染和物理计算。
    </p>
    <h3>
     <code>
      .pyo
     </code>
     文件：优化的 Python 字节码文件
    </h3>
    <p>
     <code>
      .pyo
     </code>
     文件用于存储优化的字节码，通常在使用优化标志（
     <code>
      -O
     </code>
     ）运行 Python 时创建。
    </p>
    <p>
     例如，在Python 3.5之前，运行
     <code>
      test.py
     </code>
     如下名称的Python脚本会生成一个
     <code>
      test.pyo
     </code>
     文件：
    </p>
    <p>
     <img alt="" height="77" src="https://i-blog.csdnimg.cn/direct/c2250fa1392348f696149b8a9498b3e6.png" width="674"/>
    </p>
    <blockquote>
     <p>
      <strong>
       注意：
      </strong>
      从 Python 3.5 开始，
      <code>
       .pyo
      </code>
      不再使用文件，而是将优化信息存储在
      <code>
       .pyc
      </code>
      文件中。
     </p>
    </blockquote>
    <h3>
     写在最后
    </h3>
    <p>
     如果你使用 Python 的时间足够长，你可能会遇到所有 9 个 Python 文件扩展名。
    </p>
    <p>
     无论是管理遗留代码、开发高性能应用程序还是简单地组织项目，彻底了解这些文件类型都会大大增强你的 Python 编程体验。至少，它可以节省你在编码过程中进行谷歌搜索的时间。
    </p>
