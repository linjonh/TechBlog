    <h3>
     <a id="_Linux_22">
     </a>
     一. Linux权限管理
    </h3>
    <h4>
     <a id="11__23">
     </a>
     1.1 文件访问者的分类（人)
    </h4>
    <ul>
     <li>
      文件和文件目录的所有者：u—User（中国平民法律问题）
     </li>
     <li>
      文件和文件目录的所有者所在的组的用户：g–Group（不多说）
     </li>
     <li>
      其它用户：O–Others(外国人)
     </li>
    </ul>
    <h4>
     <a id="12__28">
     </a>
     1.2 文件类型和访问权限（事物属性)
    </h4>
    <ul>
     <li>
      文件类型：
      <br/>
      d:文件夹
      <br/>
      -：普通文件
      <br/>
      l:软链接（类似Windows的快捷方式）
      <br/>
      b:块设备文件（例如硬盘、光驱等)
      <br/>
      p:管道文件
      <br/>
      c:字符设备文件（例如屏幕等串口设备)
      <br/>
      s:套接口文件
     </li>
     <li>
      基本权限：
      <br/>
      读(r/4)：Read对文件而言，具有读取文件内容的权限;对目录来说，具有浏览该目录信息的权限
      <br/>
      写（w/2)：Write对文件而言，具有修改文件内容的权限;对目录来说具有删除移动目录内文件的权限
      <br/>
      执行(x/1)：execute对文件而言，具有执行文件的权限;对目录来说，具有进入目录的权限
      <br/>
      “—”表示不具有该项权限
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/df1b94428a334d8496592af7b59d1e4f.png"/>
     </li>
    </ul>
    <h3>
     <a id="__44">
     </a>
     二. 文件权限值的表示方法
    </h3>
    <ul>
     <li>
      <p>
       字符表示方法
      </p>
     </li>
     <li>
      <p>
       8进制数值表示方法
       <br/>
       文件访问权限的相关设置方法
      </p>
     </li>
     <li>
      <p>
       chmod
       <br/>
       <strong>
        功能：
       </strong>
       设置文件的访问权限
       <br/>
       <strong>
        格式：
       </strong>
       chmod[参数]权限文件名
       <br/>
       <strong>
        常用选项：
       </strong>
      </p>
     </li>
     <li>
      <p>
       R-&gt;递归修改目录文件的权限
      </p>
     </li>
     <li>
      <p>
       说明：只有文件的拥有者和root才可以改变文件的权限
       <br/>
       <strong>
        chmod命令权限值的格式：
       </strong>
      </p>
     </li>
     <li>
      <p>
       用户表示符+/-=权限字符
       <br/>
       +:向权限范围增加权限代号所表示的权限
       <br/>
       -:向权限范围取消权限代号所表示的权限
       <br/>
       =:向权限范围赋予权限代号所表示的权限
       <br/>
       用户符号：
       <br/>
       u：拥有者
       <br/>
       g：拥有者同组用
       <br/>
       o：其它用户
       <br/>
       a：所有用户
       <br/>
       实例:
      </p>
     </li>
     <li>
      <p>
       <code>
        chmod u+w /home/abc.txt
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        chmod o-x /home/abc.tx2t
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        chmod a=x /home/abc.txt
       </code>
      </p>
     </li>
     <li>
      <p>
       三位8进制数字
       <br/>
       <strong>
        实例：
       </strong>
      </p>
     </li>
     <li>
      <p>
       <code>
        chmod 664 /home/abc.txt
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        chmod 664 /home/abc.txt
       </code>
      </p>
     </li>
     <li>
      <p>
       chown
       <br/>
       <strong>
        功能：
       </strong>
       修改文件的拥有者
       <br/>
       <strong>
        格式：
       </strong>
       chown [参数] 用户名文件名
       <br/>
       <strong>
        实例：
       </strong>
      </p>
     </li>
     <li>
      <p>
       <code>
        chown userl f1
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        chown -R userl filegroupl
       </code>
      </p>
     </li>
    </ul>
    <ol>
     <li>
      chgrp
      <br/>
      <strong>
       功能：
      </strong>
      修改文件或目录的所属组
      <br/>
      <strong>
       格式：
      </strong>
      chgrp[参数]用户组名文件名
      <br/>
      <strong>
       常用选项：
      </strong>
      -R递归修改文件或目录的所属组
      <br/>
      <strong>
       实例：
      </strong>
      <code>
       chgrp users /abc/f2
      </code>
     </li>
     <li>
      umask
      <br/>
      <strong>
       功能:
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      查看或修改文件掩码
     </li>
     <li>
      新建文件夹默认权限=0666
     </li>
     <li>
      新建目录默认权限=0777
     </li>
     <li>
      但实际上你所创建的文件和目录，看到的权限往往不是上面这个值。原因就是创建文件或目录的时候还要受到umask的影响。假设默认权限是mask，则实际创建的出来的文件权限是：mask&amp;~umask
      <br/>
      <strong>
       格式：
      </strong>
      umask 权限值
      <br/>
      <strong>
       说明：
      </strong>
      将现有的存取权限减去权限掩码后，即可产生建立文件时预设权限。超级用户默认掩码值为
      <br/>
      0022，普通用户默认为0002。
      <br/>
      <strong>
       实例：
      </strong>
     </li>
     <li>
      <code>
       umask 755
      </code>
     </li>
     <li>
      <code>
       umask //查看
      </code>
     </li>
     <li>
      <code>
       umask 044//设置
      </code>
     </li>
    </ul>

    <h4>
     <a id="21_file__105">
     </a>
     2.1 file 指令
    </h4>
    <p>
     <strong>
      功能说明：
     </strong>
     辨识文件类型。
     <br/>
     <strong>
      语法：
     </strong>
     file[选项]文件或目录…
     <br/>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      c：详细显示指令执行过程，便于排错或分析程序执行的情形。
     </li>
     <li>
      z：尝试去解读压缩文件的内容。
     </li>
    </ul>
    <p>
     (1)修改/etc/sudoers文件分配文件
    </p>
    <pre><code class="prism language-bash"> <span class="token comment"># chmod 740 /etc/sudoers</span>
 <span class="token comment"># vi /etc/sudoer</span>
</code></pre>
    <pre><code class="prism language-bash">＄ <span class="token function">sudo</span> –u ⽤⼾名 命令
</code></pre>
    <p>
     实例：
    </p>
    <pre><code class="prism language-bash">＄ <span class="token function">sudo</span> <span class="token parameter variable">-u</span> root /usr/sbin/useradd u2
</code></pre>
    <p>
     目录的权限
    </p>
    <ul>
     <li>
      可执行权限：如果目录没有可执行权限，则无法cd到目录中，
     </li>
     <li>
      可读权限：如果目录没有可读权限，则无法用s等命令查看目录中的文件内容，
     </li>
     <li>
      可写权限：如果目录没有可写权限，则无法在目录中创建文件，也无法在目录中删除文件。
     </li>
    </ul>
    <blockquote>
     <p>
      于是，问题来了~～换句话来讲，就是只要用户具有自录的写权限，用户就可以删除目录中的文件，而不论这个用户是否有这个文件的写权限.
     </p>
    </blockquote>

    <h4>
     <a id="22__170">
     </a>
     2.2 关于权限的总结
    </h4>
    <ul>
     <li>
      目录的可执行权限是表示你可否在目录下执行命令。
     </li>
     <li>
      如果目录没有-x权限，则无法对目录执行任何命令，甚至无法cd进入目，即使目录仍然有-r读
      <br/>
      权限（这个地方很容易犯错，认为有读权限就可以进入目录读取目录下的文件)
     </li>
     <li>
      而如果目录具有-x权限，但没有-r权限，则用户可以执行命令，可以cd进入目录。但由于没有
      <br/>
      目录的读权限
     </li>
     <li>
      所以在目录下，即使可以执行Is命令，但仍然没有权限读出目录下的文档。
     </li>
    </ul>
    <h4>
     <a id="23__177">
     </a>
     2.3 粘滞位
    </h4>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>root@localhost ~<span class="token punctuation">]</span>$ <span class="token function">chmod</span> + t / home / <span class="token comment"># 加上粘滞位</span>
<span class="token punctuation">[</span>root@localhost ~<span class="token punctuation">]</span>$ <span class="token function">ls</span> - ld / home /

    <p>
     当一个目录被设置为"粘滞位"(用chmod+t)，则该目录下的文件只能由
    </p>
    <ol>
     <li>
      超级管理员删除
     </li>
     <li>
      该目录的所有者删除
     </li>
     <li>
      该文件的所有者删除
     </li>
    </ol>
