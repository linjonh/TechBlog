      <strong>
       线性表和链表优缺点及适用场景
      </strong>
      <ul>
       <li>
        <strong>
         线性表（以数组为例）
        </strong>
        <ul>
         <li>
          <strong>
           优点
          </strong>
          ：随机访问效率高，可通过下标直接访问元素，时间复杂度为 O (1)；存储密度大，内存连续存储，空间利用率高。
         </li>
         <li>
          <strong>
           缺点
          </strong>
          ：插入和删除操作效率低，平均时间复杂度为 O (n)；需要预先分配固定大小的空间，若数据量动态变化大，可能造成空间浪费或不足。
         </li>
         <li>
          <strong>
           适用场景
          </strong>
          ：适合频繁进行随机访问的场景，如数据查询，且数据量相对固定的情况。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         链表
        </strong>
        <ul>
         <li>
          <strong>
           优点
          </strong>
          ：插入和删除操作效率高，只需修改指针，平均时间复杂度为 O (1)；无需预先分配固定空间，可动态分配，适应数据量动态变化的情况。
         </li>
         <li>
          <strong>
           缺点
          </strong>
          ：不支持随机访问，访问元素需要从头开始遍历，时间复杂度为 O (n)；存储密度低，每个节点除了数据域还需额外的指针域，占用更多空间。
         </li>
         <li>
          <strong>
           适用场景
          </strong>
          ：适合频繁进行插入和删除操作的场景，如实现队列、栈等数据结构，数据量动态变化较大的情况。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
