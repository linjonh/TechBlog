    <h3>
     <a id="_1">
     </a>
     报错现象
    </h3>
    <pre><code class="prism language-bash">hint: You have divergent branches and need to specify how to reconcile them.
hint: You can <span class="token keyword">do</span> so by running one of the following commands sometime before
hint: your next pull:
hint: 
hint:   <span class="token function">git</span> config pull.rebase <span class="token boolean">false</span>  <span class="token comment"># merge (the default strategy)</span>
hint:   <span class="token function">git</span> config pull.rebase <span class="token boolean">true</span>   <span class="token comment"># rebase</span>
hint:   <span class="token function">git</span> config pull.ff only       <span class="token comment"># fast-forward only</span>
hint: 
hint: You can replace <span class="token string">"git config"</span> with <span class="token string">"git config --global"</span> to <span class="token builtin class-name">set</span> a default
hint: preference <span class="token keyword">for</span> all repositories. You can also pass --rebase, --no-rebase,
hint: or --ff-only on the <span class="token builtin class-name">command</span> line to override the configured default per
hint: invocation.
fatal: Need to specify how to reconcile divergent branches.
</code></pre>
    <h3>
     <a id="_18">
     </a>
     大概原因
    </h3>
    <p>
     在团队开发中，多个开发人员同时在不同的本地环境中对同一个仓库的相同分支进行开发。例如，开发人员 A 和开发人员 B 都从远程仓库的master分支拉取了代码到各自的本地master分支，然后各自进行了一些提交。开发人员 A 先将自己的提交推送到了远程master分支，当开发人员 B 想要git pull远程master分支的最新代码时，就会出现本地master分支和远程master分支的分歧，因为开发人员 B 的本地分支有自己的提交，而远程分支也有开发人员 A 的提交，Git 不知道该如何合并这些不同的提交。
    </p>
    <h3>
     <a id="_21">
     </a>
     解决
    </h3>
    <p>
     将拉取操作的合并策略设置为默认的merge方式。当执行git pull时，Git 会自动进行合并操作，将远程分支的内容合并到本地分支，可能会产生一个新的合并提交，保留两个分支的提交历史。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config pull.rebase <span class="token boolean">false</span>
</code></pre>
    <p>
     将拉取操作设置为rebase策略。之后执行git pull时，Git 会把本地分支的提交移动到远程分支的最新提交之后，重写提交历史，使提交历史更加线性。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config pull.rebase <span class="token boolean">true</span>
</code></pre>
    <p>
     建议使用rebase保持分支提交记录整洁
    </p>
