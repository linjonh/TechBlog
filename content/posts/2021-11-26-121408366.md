    <h4>
     <a id="_13">
     </a>
     名词解释
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        域名
       </strong>
       ：Domain Name，是由一串用点分隔的名字组成的Internet上某一台计算机或计算机组的名称。
      </p>
      <ul>
       <li>
        由
        <strong>
         点号
        </strong>
        分隔开，最右边的字符组称为顶级域名或
        <strong>
         一级域名
        </strong>
        、倒数第二组称为
        <strong>
         二级域名
        </strong>
        、倒数第三组称为
        <strong>
         三级域名
        </strong>
        、以此类推。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        DNS
       </strong>
       ：domain name service(域名解析服务)，将
       <strong>
        域名
       </strong>
       和
       <strong>
        IP地址
       </strong>
       相互映射。
      </p>
      <ul>
       <li>
        <strong>
         SOA记录
        </strong>
        ： SOA叫做起始授权机构记录，NS用于标识多台域名解析服务器，SOA记录用于在众多NS记录中那一台是主服务器。
       </li>
       <li>
        <strong>
         NS记录
        </strong>
        ： 域名解析服务器记录，如果要将子域名指定某个域名服务器来解析，需要设置NS记录。
       </li>
       <li>
        <strong>
         A 记录
        </strong>
        ：域名对应是 Ipv4 地址。
       </li>
       <li>
        <strong>
         AAAA 记录
        </strong>
        ：域名对应是 Ipv6 地址。
       </li>
       <li>
        <strong>
         CNAME
        </strong>
        ：域名对应还是个域名。
       </li>
       <li>
        <strong>
         ＭＸ
        </strong>
        ：邮件记录（Mail eXchange），域名对应是接收电子邮件的服务器地址。
       </li>
       <li>
        <strong>
         PTR
        </strong>
        ：逆向查询记录（Pointer Record），只用于从IP地址查询域名。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        域名解析软件
       </strong>
       <br/>
       DNS系统是由各式各样的DNS软件所驱动的，包括：BIND (Berkeley Internet Name Domae most commonly used namedaemon. 　DJBDNS (Dan J Bernstein’s DNS implementation)　MaraDNS 　NSD (Name Server Daemon) 　PowerDNS等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        服务端
       </strong>
      </p>
      <ul>
       <li>
        安装包：bind
       </li>
       <li>
        服务名称：named
       </li>
       <li>
        主配置文件：/etc/named.conf
       </li>
       <li>
        数据目录：/var/named
       </li>
       <li>
        端口 ：53
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        客户端
       </strong>
      </p>
      <ul>
       <li>
        dns 指向文件：
       </li>
      </ul>
      <pre><code class="prism language-bash">/etc/resolv.conf
nameserver <span class="token number">172.25</span>.9.1
</code></pre>
      <ul>
       <li>
        测试命令：
       </li>
      </ul>
      <pre><code class="prism language-bash"><span class="token function">host</span> www.baidu.com	<span class="token comment">##地址解析命令</span>
<span class="token function">dig</span> www.baidu.com	<span class="token comment">##地址详细解析信息命令</span>
</code></pre>
      <pre><code>  - dig 查询状态
  	- NOERROR：表示查询成功	
  	- REFUSED：服务拒绝访问
  	- SERVFAIL：查询记录失败,（dns服务器无法到达上级，拒绝缓存）
  	- NXDOMAIN：此域名A记录在dns中不存在
</code></pre>
     </li>
    </ul>
