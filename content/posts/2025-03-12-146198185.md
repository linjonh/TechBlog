    <h2>
     <a id="_0">
     </a>
     背景：
    </h2>
    <p>
     在数据库运维和开发过程中，误删数据堪称技术人最"心惊肉跳"的操作之一。一条不慎执行的DELETE语句，可能瞬间让重要数据消失无踪。传统的数据恢复往往依赖备份回滚，不仅操作复杂，还可能面临备份不及时的尴尬。为此，美团技术团队开源的MyFlash闪回工具应运而生——这款基于MySQL binlog解析的轻量级工具，无需全量备份即可通过逆向操作生成回滚SQL，为数据安全提供了灵活可靠的"后悔药"。本文将手把手带您实践MyFlash的部署与应用，让您掌握这项关键时刻能挽救危机的硬核技能。
    </p>
    <h2>
     <a id="myflash__2">
     </a>
     通过使用myflash 工具恢复误删数据
    </h2>
    <h4>
     <a id="myflash_5">
     </a>
     myflash工具下载
    </h4>
    <p>
     <a href="https://github.com/Meituan-Dianping/MyFlash">
      https://github.com/Meituan-Dianping/MyFlash
     </a>
    </p>
    <h3>
     <a id="_11">
     </a>
     一、安装
    </h3>
    <p>
     <strong>
      （务必要做！！否则回退会失败）
     </strong>
    </p>
    <pre><code class="prism language-text">yum install -y gcc glib2 glib2-devel

unzip MyFlash-master.zip

cd MyFlash-master

gcc -w `pkg-config --cflags --libs glib-2.0` source/binlogParseGlib.c -o binary/flashback

</code></pre>
    <p>
     确认安装
    </p>
    <pre><code class="prism language-text">cd /MyFlash-master/binary

./flashback -h
</code></pre>
    <h3>
     <a id="_36">
     </a>
     二、环境准备（误删行数据）
    </h3>
    <h5>
     <a id="_38">
     </a>
     创建测试数据
    </h5>
    <pre><code class="prism language-text"> create table emp
  (
  id INT(11),
  name VARCHAR(25)
  );
  
insert into emp values(1,'a')
insert into emp values(1,'b')
</code></pre>
    <h5>
     <a id="_55">
     </a>
     误删数据
    </h5>
    <pre><code class="prism language-text">delete from emp where id = 1;

commit;
</code></pre>
    <h5>
     <a id="_65">
     </a>
     此时表内已无数据
    </h5>
    <pre><code class="prism language-text">
mysql&gt; select * from emp;
Empty set (0.00 sec)


</code></pre>
    <h3>
     <a id="_77">
     </a>
     三、恢复
    </h3>
    <h5>
     <a id="binlog_79">
     </a>
     查看当前binlog
    </h5>
    <pre><code class="prism language-text">

mysql&gt; show variables like '%log_bin%';
+---------------------------------+--------------------------------------------+
| Variable_name                   | Value                                      |
+---------------------------------+--------------------------------------------+
| log_bin                         | ON                                         |
| log_bin_basename                | /data/mysqldata/my3306/log/mysql-bin  

mysql&gt; show master status;
+------------------+----------+--------------+------------------+---------------------------------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB | Executed_Gtid_Set                           |
+------------------+----------+--------------+------------------+---------------------------------------------+
| mysql-bin.000003 |      194 |              |                  | 1d6df993-f5e0-11ed-aa30-005056b388fd:1-1130 |
+------------------+----------+--------------+------------------+---------------------------------------------+
1 row in set (0.00 sec)



</code></pre>
    <h5>
     <a id="binlog_105">
     </a>
     解析binlog
    </h5>
    <p>
     确认要恢复的gtid为1d6df993-f5e0-11ed-aa30-005056b388fd:1131（在delete操作之前）
    </p>
    <pre><code class="prism language-text">[root@qiuyang1 binary]# mysqlbinlog -v /data/mysqldata/my3306/log/mysql-bin.000003 | egrep -i 'GTID_NEXT|DELETE FROM `test`.`emp`'

SET @@SESSION.GTID_NEXT= '1d6df993-f5e0-11ed-aa30-005056b388fd:1131'/*!*/;
### DELETE FROM `test`.`emp`
### DELETE FROM `test`.`emp`
SET @@SESSION.GTID_NEXT= 'AUTOMATIC' /* added by mysqlbinlog */ /*!*/;


</code></pre>
    <h5>
     <a id="flashback_binlog_124">
     </a>
     使用flashback工具 生产闪回binlog文件
    </h5>
    <p>
     工具目录：
    </p>
    <p>
     填写对应的数据库名、表名、要恢复的gtid
    </p>
    <pre><code class="prism language-text">./flashback --binlogFileNames=/data/mysqldata/my3306/log/mysql-bin.000003 --databaseNames=test --tableNames=emp --sqlTypes=delete --include-gtids='1d6df993-f5e0-11ed-aa30-005056b388fd:1131'

</code></pre>
    <p>
     在当前目录会生成binlog_output_base.flashback文件
    </p>
    <h5>
     <a id="mysqlbinlog_sql_141">
     </a>
     使用mysqlbinlog 生成sql文件
    </h5>
    <pre><code class="prism language-text">mysqlbinlog --skip-gtids  binlog_output_base.flashback &gt;flash.sql
</code></pre>
    <h5>
     <a id="mysqlsql_151">
     </a>
     进入mysql数据库，应用该sql文件
    </h5>
    <pre><code class="prism language-text"> mysql -uroot -p
 
 source flash.sql

</code></pre>
    <h5>
     <a id="_164">
     </a>
     验证
    </h5>
    <p>
     数据已回复
    </p>
    <pre><code class="prism language-text">mysql&gt; select * from emp;
+------+------+
| id   | name |
+------+------+
|    1 | a    |
|    1 | b    |
+------+------+
2 rows in set (0.00 sec)

</code></pre>
    <h3>
     <a id="_186">
     </a>
     四、环境准备（误删表）
    </h3>
    <h5>
     <a id="_188">
     </a>
     创建测试数据
    </h5>
    <p>
     反复执行insert，最后构建出一张百万级别的表
    </p>
    <pre><code class="prism language-text">CREATE TABLE test2(id INT PRIMARY KEY AUTO_INCREMENT);

INSERT INTO test2() VALUES(),(),(),();

INSERT INTO test2(id) SELECT id+(SELECT COUNT(*) FROM test2) FROM test2;

mysql&gt; select count(*) from test2;
+----------+
| count(*) |
+----------+
|  2097152 |


</code></pre>
    <h5>
     <a id="_210">
     </a>
     误删表
    </h5>
    <p>
     模拟场景：由于where条件的输入失误，导致整表被删
    </p>
    <pre><code class="prism language-text">delete from test2 where 1=1;


mysql&gt; select count(*) from test2;
+----------+
| count(*) |
+----------+
|        0 |
+----------+
1 row in set (0.09 sec)


</code></pre>
