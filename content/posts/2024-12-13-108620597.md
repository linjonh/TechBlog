    <h2>
     <a id="_0">
     </a>
     一、问题发现
    </h2>
    <h3>
     <a id="1base64400_1">
     </a>
     1.可能因为base64字符太长，导致后端处理时出错，表现为前端请求报400错误；
    </h3>
    <h3>
     <a id="2base64_2">
     </a>
     2.前端传数据到后端时可能base64字符串中的特殊字符被替换，例如+被替换成空格，表现为上传的图片不完整
    </h3>
    <h2>
     <a id="_3">
     </a>
     二、问题解决
    </h2>
    <h3>
     <a id="11_4">
     </a>
     1.问题1解决方法：
    </h3>
    <h4>
     <a id="11servermaxhttpheadersizehttp8KBbase642MBservermaxhttpheadersizehttpsblogcsdnnetyueaini10000articledetails107809001_5">
     </a>
     1.1.在配置文件中设置server.max-http-header-size（最大http请求头大小），其默认值为8KB，通常图片的base64会超过默认值，但是也不是越大越好（我设置2MB，然后前端再限制一下图片的大小或者压缩图片），大了反而有大问题，详情请看
     <a href="https://blog.csdn.net/yueaini10000/article/details/107809001">
      server.max-http-header-size的坑
     </a>
    </h4>
    <h3>
     <a id="22_6">
     </a>
     2.问题2解决方法：
    </h3>
    <h4>
     <a id="21base64base64springmvc_7">
     </a>
     2.1.前端对base64字符串编码之后再传给后端处理，后端不需要解码，因为接收base64字符串时springmvc会进行一次解码
    </h4>
    <h4>
     <a id="22_8">
     </a>
     2.2.也可以通过将被替换的字符手动换回来，不过不推荐使用，你不知道究竟还有没有其他特殊字符被替换
    </h4>
    <pre><code class="prism language-java">String replace <span class="token operator">=</span> base64<span class="token punctuation">.</span><span class="token function">replace</span><span class="token punctuation">(</span><span class="token string">" "</span><span class="token punctuation">,</span> <span class="token string">"+"</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
MultipartFile multipartFile <span class="token operator">=</span> BASE64DecodedMultipartFile<span class="token punctuation">.</span><span class="token function">base64ToMultipart</span><span class="token punctuation">(</span>replace<span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <h2>
     <a id="base642MBbase643000000_13">
     </a>
     三.建议能不使用base64上传文件就不使用，一张2MB的图片转成base64字符串大概有3000000+个字符，想想都可怕啊
    </h2>
