    <p>
     例如
     <code>
      select * from t1 order by 10000, 10
     </code>
     查询
     <code>
      10000
     </code>
     -
     <code>
      10010
     </code>
     页数据，mysql会先查询出
     <code>
      100010
     </code>
     页，再抛弃
     <code>
      100000
     </code>
     。这个操作会产生大量的无用IO/CPU消耗
    </p>
    <h5>
     <a id="_2">
     </a>
     范围查询
    </h5>
    <p>
     1.当ID能保证连续无空缺时，可以使用范围查询。
    </p>
    <pre><code class="prism language-cmd">select * from t1 where id &gt; {seed_id} and id &lt;= {seed_id + step} order by id;
</code></pre>
    <pre><code class="prism language-cmd">select * from t1 where id &gt; {seed_id} limit 10
</code></pre>
    <h5>
     <a id="_14">
     </a>
     子查询
    </h5>
    <p>
     我们先查询出 limit 第一个参数对应的主键值，再根据这个主键值再去过滤并 limit，这样效率会更快一些。
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">select</span> <span class="token operator">*</span> <span class="token keyword">from</span> t1 <span class="token keyword">where</span> id <span class="token operator">&gt;=</span> <span class="token punctuation">(</span><span class="token keyword">select</span> id <span class="token keyword">from</span> t1 <span class="token keyword">limit</span> {seed_id}<span class="token punctuation">,</span> <span class="token number">1</span><span class="token punctuation">)</span> <span class="token keyword">limit</span> {step}<span class="token punctuation">;</span>
</code></pre>
    <p>
     不过这个方法虽然没有限制ID是连续的，但是限制了ID必须是正序的
    </p>
    <h5>
     <a id="inner_join_24">
     </a>
     inner join延迟关联
    </h5>
    <p>
     延迟关联和上面的子查询本质没有区别。
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">select</span> <span class="token operator">*</span> <span class="token keyword">from</span> t1
<span class="token keyword">inner</span> <span class="token keyword">join</span> <span class="token punctuation">(</span><span class="token keyword">select</span> id <span class="token keyword">from</span> t1 <span class="token keyword">limit</span> {seed_id}<span class="token punctuation">,</span> <span class="token number">1</span><span class="token punctuation">)</span> t2
<span class="token keyword">on</span> t1<span class="token punctuation">.</span>id <span class="token operator">&gt;=</span> t2<span class="token punctuation">.</span>id
<span class="token keyword">limit</span> {step}<span class="token punctuation">;</span>
</code></pre>
    <p>
     比如查询10000~100010。那就先查询出10001
    </p>
    <p>
     然后将主表t1与t2相连，仅仅保留t1.id大于等于t2.id的前10条数据
    </p>
