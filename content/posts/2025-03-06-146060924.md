    <p>
     <strong>
      <a href="https://www.evget.com/product/3150" rel="nofollow" title="DevExtreme">
       DevExtreme
      </a>
     </strong>
     拥有高性能的HTML5 / JavaScript小部件集合，使您可以利用现代Web开发堆栈（包括React，Angular，ASP.NET Core，jQuery，Knockout等）构建交互式的Web应用程序。从Angular和Reac，到ASP.NET Core或Vue，DevExtreme包含全面的高性能和响应式UI小部件集合，可在传统Web和下一代移动应用程序中使用。 该套件附带功能齐全的数据网格、交互式图表小部件、数据编辑器等。
    </p>
    <p>
     DevExpress JS &amp; ASP.NET Core v24.2已全新发布，新版本官宣了全新的聊天组件、集成了AI&amp;聊天机器人等，欢迎下载最新版组件体验！
    </p>
    <p>
     <a href="https://www.evget.com/product/3150/download" rel="nofollow" title="DevExtreme v24.2正式版下载">
      <strong>
       DevExtreme v24.2正式版下载
      </strong>
     </a>
    </p>
    <h6>
     <strong>
      全新的聊天组件
     </strong>
    </h6>
    <p>
     全新的DevExtreme Chat Component (v24.2)允许您解决各种使用场景，并在您的下一个DevExpress驱动的web应用程序中引入实时通信支持。
    </p>
    <p>
     使用
     <a href="https://www.evget.com/product/3150" rel="nofollow" title="DevExtreme">
      DevExtreme
     </a>
     Chat，您可以：
    </p>
    <ul>
     <li>
      主持内部/人际互动（一对一或群聊），理想的社交网络和/或内部协作。
     </li>
     <li>
      集成人工智能服务和/或聊天机器人：将DevExtreme Chat连接到人工智能服务或聊天机器人，以获得全天候支持服务。提供即时的答案，并通过智能交互扩展参与。
     </li>
     <li>
      在支持专家和客户之间引入即时通信选项。
     </li>
    </ul>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="672" src="https://i-blog.csdnimg.cn/img_convert/93369f9225d73e6dbaa9d853e3d813f3.png" width="990"/>
    </p>
    <p>
     DevExtreme Chat组件支持以下选项/功能：
    </p>
    <ul>
     <li>
      AI（人工智能）扩展
     </li>
     <li>
      数据源支持
     </li>
     <li>
      弹窗整合
     </li>
     <li>
      自定义空状态文本
     </li>
     <li>
      用户界面定制
     </li>
     <li>
      消息模板
     </li>
     <li>
      错误支持
     </li>
     <li>
      可访问性合规
     </li>
     <li>
      键盘导航
     </li>
     <li>
      国际化(从右到左)
     </li>
    </ul>
    <p>
     <strong>
      注意
     </strong>
     ：DevExtreme Chat是一个客户端组件，需要一个独立的后端解决方案来发挥其潜力。
    </p>
    <h6>
     <strong>
      设置初始消息和呈现新消息
     </strong>
    </h6>
    <p>
     要设置初始消息，请为消息分配
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#items" rel="nofollow" title="items[]">
      items[]
     </a>
     数组，或者使用
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#dataSource" rel="nofollow" title="dataSource">
      dataSource
     </a>
     将Chat组件绑定到数据。
    </p>
    <p>
     要呈现一条新消息，您必须：
    </p>
    <ul>
     <li>
      如果使用
      <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#items" rel="nofollow" title="items[]">
       items[]
      </a>
      ，则更新items数组。如果使用Query，调用
      <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Methods/#renderMessagemessage" rel="nofollow" title="renderMessage()">
       renderMessage()
      </a>
      函数。
     </li>
     <li>
      如果使用dataSource，则实现加载和插入CRUD操作。
     </li>
    </ul>
    <p>
     无论使用何种方法，都必须将新消息发送到后端。
    </p>
    <h6>
     <strong>
      AI和聊天机器人的集成
     </strong>
    </h6>
    <p>
     使用
     <a href="https://www.evget.com/product/3150" rel="nofollow" title="DevExtreme">
      DevExtreme
     </a>
     Chat UI组件，您可以合并AI驱动的应用程序助手，实现受Copilot启发的用户界面等。要将这些功能引入下一个DevExtreme驱动的web应用程序，只需在后端设置适当的AI服务（Microsoft/OpenAI - GPT，Google - Gemini，通过Ollama的自托管模型或其他AI服务）。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="600" src="https://i-blog.csdnimg.cn/img_convert/9545e5df33db0a5ca2d9279fd297c77f.png" width="1070"/>
    </p>
    <p>
     新版本在DevExpress的两款产品中充分利用了DevExtreme Chat的AI集成潜力：Web报表查看器和Web最终用户报表设计器，同时还在实现中加入了DevExpress AI驱动的API。
    </p>
    <p>
     首先，我们在DevExpress Web报表查看器中嵌入了一个受Copilot启发的聊天窗口（用户可以问自然语言问题来分析报表数据并获得基于AI的见解）。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="632" src="https://i-blog.csdnimg.cn/img_convert/2b38e83037142cc68c141b8d134c62ee.png" width="932"/>
    </p>
    <p>
     其次，我们在DevExpress End-User Report Designer中嵌入了一个虚拟助手（用户可以提交自然语言查询以促进自定义报表设计过程）。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="632" src="https://i-blog.csdnimg.cn/img_convert/a6aee6040ddd07ac70538a5d4af81977.png" width="932"/>
    </p>
    <p>
     您可以从下面的GitHub仓库下载这些例子：
     <a href="https://github.com/DevExpress-Examples/web-reporting-integrate-ai-assistant" title="Web Reporting - Integrate AI Assistants">
      Web Reporting - Integrate AI Assistants
     </a>
    </p>
    <h6>
     <strong>
      管理用户
     </strong>
    </h6>
    <p>
     <a href="https://www.evget.com/product/3150" rel="nofollow" title="DevExtreme">
      DevExtreme
     </a>
     Chat实例在用户属性中包含有关其所有者的信息，所有者消息向右对齐（或在RTL模式下向左对齐），而不显示所有者的用户名和头像。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="120" src="https://i-blog.csdnimg.cn/img_convert/42a626a21b69ccc6d623616e73bb80bf.png" width="780"/>
    </p>
    <p>
     每条消息都包含有关发件人（
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Types/Message/#author" rel="nofollow" title="item.author">
      item.author
     </a>
     ）的信息，并包括姓名、头像和所有头像文本。如果没有设置头像，则显示用户的首字母缩写，没有名字的用户被标记为“未知用户”。
    </p>
    <h6>
     <strong>
      处理事件
     </strong>
    </h6>
    <p>
     输入消息将触发
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Events/#messageEntered" rel="nofollow" title="messageEntered">
      messageEntered
     </a>
     事件，使用事件处理程序执行消息后处理（如删除敏感文字、在消息提要中显示消息以及将消息发送到服务器进行存储）。
    </p>
    <p>
     开始输入会引发
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Events/#typingStart" rel="nofollow" title="typingStart">
      typingStart
     </a>
     ，停止输入会引发
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Events/#typingEnd" rel="nofollow" title="typingEnd">
      typingEnd
     </a>
     。通过使用这些事件处理程序，您可以管理
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#typingUsers" rel="nofollow" title="typingUsers">
      typingUsers
     </a>
     数组（它显示哪些用户正在聊天UI中输入）。
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="DevExpress JS &amp; ASP.NET Core v24.2产品图集" height="170" src="https://i-blog.csdnimg.cn/img_convert/427dbef8d3a01094a9ecb91ca4aec9e2.png" width="780"/>
    </p>
    <h6>
     <strong>
      自定义UI元素
     </strong>
    </h6>
    <p>
     使用
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#showDayHeaders" rel="nofollow" title="showDayHeaders">
      showDayHeaders
     </a>
     和
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#showMessageTimestamp" rel="nofollow" title="showMessageTimestamp">
      showMessageTimestamp
     </a>
     控制日期和时间可见性，修改
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#dayHeaderFormat" rel="nofollow" title="dayHeaderFormat">
      dayHeaderFormat
     </a>
     和
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#messageTimestampFormat" rel="nofollow" title="messageTimestampFormat">
      messageTimestampFormat
     </a>
     的相关格式。
    </p>
    <p>
     要自定义DevExtreme聊天消息，使用
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#messageTemplate" rel="nofollow" title="messageTemplate">
      messageTemplate
     </a>
     ：
    </p>
    <ul>
     <li>
      显示图像、图表、表格、链接或任何视觉元素。
     </li>
     <li>
      实现自定义按钮，如“复制文本”或“重新生成聊天机器人响应”。
     </li>
     <li>
      支持Markdown。
     </li>
     <li>
      插入自定义标记。
     </li>
    </ul>
    <p>
     使用
     <a href="https://js.devexpress.com/Documentation/24_2/ApiReference/UI_Components/dxChat/Configuration/#alerts" rel="nofollow" title="alerts">
      alerts
     </a>
     数组显示运行时错误（如“您已断开连接”，“消息未发送”等）。
    </p>
