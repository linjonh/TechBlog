     编码与解码

    <p>

    <h5>
     <a id="_88">
     </a>
     编码
    </h5>
    <h6>
     <a id="ASCLL_89">
     </a>
     ASCLL编码
    </h6>
    <p>
     包含 大写字母 小写字母 数字 一些符号 共计128个
    </p>
    <h6>
     <a id="ISO_88591_91">
     </a>
     ISO 8859-1
    </h6>
    <p>
     在ASCLL基础上扩充一些希腊字符 共计256个
    </p>
    <h6>
     <a id="GB2312_93">
     </a>
     GB2312
    </h6>
    <p>
     GB：国标
     <br/>
     继续扩充 收录6763个常用汉字 682个字符
    </p>
    <h6>
     <a id="GBK_96">
     </a>
     GBK
    </h6>
    <p>
     K:扩充
     <br/>
     收录汉字符号共计20000+ 支持繁体中文
    </p>
    <h6>
     <a id="UTF8_99">
     </a>
     UTF-8
    </h6>
    <p>
     万国码
     <br/>
     包含世界上所有语言的 所有文字与符号
     <br/>
     <strong>
      很常用
     </strong>
    </p>
    <h5>
     <a id="_103">
     </a>
     编码原则
    </h5>
    <h6>
     <a id="1_104">
     </a>
     原则1
    </h6>
    <p>
     存储时 务必采用合适的字符编码
     <br/>
     否则 无法存储 数据会丢失
    </p>
    <h6>
     <a id="2_107">
     </a>
     原则2
    </h6>
    <p>
     存储时采用的哪种方式编码 读取时就必须采用相同方式解码
     <br/>
     否则数据能呈现 但数据错乱（乱码）
    </p>

     </a>

    <p>

    <h3>
     <a id="_158">
     </a>
     块级元素
    </h3>
    <p>
     独占一行的元素
    </p>
    <h3>
     <a id="_160">
     </a>
     行内元素
    </h3>
    <p>
     只占自己一小部分地方的元素
    </p>
    <h3>
     <a id="_162">
     </a>
     规则
    </h3>
    <ol>
     <li>
      块级元素中几乎什么都能写（块级元素，行级元素）
     </li>
     <li>
      行内元素里能写行内元素 但不能写块级元素
     </li>
     <li>
      h1到h6不能嵌套
     </li>
     <li>
      p标签中不能写块级元素
     </li>
    </ol>
    <h3>
     <a id="marquee_167">
     </a>
     marquee标签废除的原因
    </h3>
    <p>
     设计初衷是想要动画效果 但是如今已经有css可以实现动画效果 所以过时了
    </p>
    <h2>
     <a id="017_HTML_169">
     </a>
     017_HTML常用的文本标签
    </h2>
    <ol>
     <li>
      用于包裹词汇 短语等
     </li>
     <li>
      通常写在排版标签里面
     </li>
     <li>
      排版标签更宏观（大段的文字），文本标签更微观（词汇，短语）[[]]
     </li>
     <li>
      文本标签通常都是行内元素
     </li>
    </ol>
    <h3>
     <a id="_174">
     </a>
     常用的
    </h3>
    <ul>
     <li>
      em 要着重阅读的内容 双标签
     </li>
     <li>
      strong 十分重要的内容（语气比em要强） 双标签
     </li>
     <li>
      span 没有语义，用于包裹短语的通用容器 双标签
     </li>
    </ul>
    <h2>
     <a id="018_HTML_178">
     </a>
     018_HTML不常用的文本标签
    </h2>
    <ul>
     <li>
      cite 作品标题 双
     </li>
     <li>
      dfn 特殊术语或专属名词 双
     </li>
     <li>
      del与ins 删除的文本与插入的文字 双
     </li>
     <li>
      sub与sup 下标文字与上标文字 双
     </li>
     <li>
      code 一段代码 双
     </li>
     <li>
      samp 从正常的上下午中，将某些内容提取出来，例如：标识设备输出 双
     </li>
     <li>
      kbd 键盘文本，表示文本是通过键盘输入的，经常在与计算机相关的手册中 双
     </li>
     <li>
      abbr 缩写，最好配合上title属性 双
     </li>
     <li>
      bdo 更改文本方向，要配合dir属性，可选值：ltr（默认值），rtl 双
     </li>
     <li>
      var 标记变量，可以与code标签一起使用 双
     </li>
     <li>
      small 附属细则，例如：包括版权，法律文本，——很少使用 双
     </li>
     <li>
      b 摘要中的关键字，评论中的产品名称——很少使用 双
     </li>
     <li>
      i 本意是：任务的思想活动，所说的话等等。现在多用于：呈现字体图标 双
     </li>
     <li>
      u 与正常内容有反差文本，例如：错的单词，不合适的描述等——很少使用 双
     </li>
     <li>
      q 短引用——很少使用 双
     </li>
     <li>
      blockquote 长引用——很少使用 双 块级元素
     </li>
     <li>
      address 地址信息 双 块级元素
     </li>
     <li>
      br 分行 单
     </li>
     <li>
      hr 分割线 单
     </li>
    </ul>

      的压缩格式（把肉眼不容易观察出来的细节丢弃了）。
     </li>
     <li>
      主要特点：
      <em>
       支持的颜色丰富、占用空间较小
      </em>
      、不支持透明背景、不支持动态图.
     </li>
     <li>
      使用场景：对图片细节
      <em>
       没有极高要求
      </em>
      的场景，例如：网站的产品宣传图等。–该格式网页中很常见。
     </li>
    </ul>
    <h6>
     <a id="png_238">
     </a>
     png格式：
    </h6>
    <ul>
     <li>
      概述：扩展名为.png，是一种
      <em>
       无损
      </em>
      的压缩格式，能够更高质量的保存图片。
     </li>
     <li>
      主要特点：
      <em>
       支持的颜色丰富
      </em>
      、占用空间略大、
      <em>
       支持透明背景
      </em>
      、不支持动态图。
     </li>
     <li>
      使用场景：①想让图片有透明背景；②想更高质量的呈现图片；例如：公司logo图、重要配图等。
     </li>
    </ul>
    <h6>
     <a id="bmp_242">
     </a>
     bmp格式：
    </h6>
    <ul>
     <li>
      概述：扩展名为.bmp，
      <em>
       不进行压缩
      </em>
      的一种格式，在最大程度上保留图片更多的细节。
     </li>
     <li>
      主要特点：
      <em>
       支持的颜色丰富、保留的细节更多
      </em>
      、占用空间极大、不支持透明背景、不支持动态图。
     </li>
     <li>
      使用场景：对图片细节
      <em>
       要求极高
      </em>
      的场景，例如：一些大型游戏中的图片。（网页中很少使用)
     </li>
    </ul>
    <h6>
     <a id="gif_246">
     </a>
     gif格式
    </h6>
    <ul>
     <li>
      概述：扩展名为.gif，仅支持256种颜色，色彩呈现不是很完整。
     </li>
     <li>
      主要特点：支持的颜色较少、支持简单透明背景、支持动态图。
     </li>
     <li>
      使用场景：网页中的动态图片。
     </li>
    </ul>
    <h6>
     <a id="webp_250">
     </a>
     webp格式
    </h6>
    <ul>
     <li>
      概述：扩展名为.webp，谷歌推出的一种格式，专门用来在网页中呈现图片。
     </li>
     <li>
      主要特点：具备上述几种格式的优点，但兼容性不太好，一旦使用务必要解决兼容性问题。
     </li>
     <li>
      使用场景：网页中的各种图片。
     </li>
    </ul>
    <h6>
     <a id="base64_254">
     </a>
     base64格式
    </h6>
    <ul>
     <li>
      1.本质：一串特殊的文本，要通过浏览器打开，传统看图应用通常无法打开。
     </li>
     <li>
      2.原理：把图片进行base64编码，形成一串文本。
     </li>
     <li>
      3.如何生成：靠一些工具或网站。
     </li>
     <li>
      4.如何使用：直接作为img标签的src属性的值即可，并且不受文件位置的影响。
     </li>
     <li>
      5.使用场景：一些较小的图片，或者需要和网页一起加载的图片。
     </li>
    </ul>

    <h2>
     <a id="040_CSS_718">
     </a>
     040_CSS基础
    </h2>
    <ul>
     <li>
      css 的全称为：层叠样式表（Cascading StyleSbeets）。
     </li>
     <li>
      css 也是一种标记语言，用于给HTML结构设置样式，例如：文字大小、颜色、元素宽高等。
     </li>
     <li>
      简单理解：CSS可以美化HTML，让HTML 更漂亮。
     </li>
     <li>
      核心思想：HTML 搭建结构，CSS 添加样式，实现了：结构与样式的分离。
     </li>
    </ul>
    <h2>
     <a id="041_CSS_724">
     </a>
     041_CSS的编写位置
    </h2>
    <h3>
     <a id="_725">
     </a>
     行内样式
    </h3>
    <ul>
     <li>
      写在标签的sryle属性中。（又称：内嵌样式）
     </li>
     <li>
      语法：
      <br/>
      <code>
       &lt;h1 style="color:red；font-size：60px；"&gt;欢迎来到尚硅谷学习&lt;/h1&gt;
      </code>
     </li>
     <li>
      注意点：
     </li>
    </ul>
    <ol>
     <li>
      style 属性的值不能随便写，写要符合CSS 语法规范，是(名：值；)的形式。
     </li>
     <li>
      行内样式表，只能控制当前标签的样式，对其他标签无效。
     </li>
    </ol>
    <ul>
     <li>
      存在的问题：
      <br/>
      书写繁琐、样式不能复用、并且没有体现：出结构与样式分离 的思想，不推荐大量使用，只有对当前元素添加简单样式时，才偶尔使用。
     </li>
    </ul>
    <h3>
     <a id="_734">
     </a>
     内部样式
    </h3>
    <ul>
     <li>
      写在html页面内部，将所有的css代码提取出来，单独放在
      <code>
       &lt;style&gt;
      </code>
      标签中。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <pre><code>&lt;style&gt;
   h1 {
       color: red;
       font-size:40px;
       }
&lt;/style&gt;
</code></pre>
    <ul>
     <li>
      注意点：
     </li>
    </ul>
    <ol>
     <li>
      <code>
       &lt;style&gt;
      </code>
      标签理论上可以放在HTML文档的任何地方，但一般都放在
      <code>
       &lt;head&gt;
      </code>
      标签中。
     </li>
     <li>
      此种写法：样式可以复用、代码结构清晰。
     </li>
    </ol>
    <ul>
     <li>
      存在的问题：
     </li>
    </ul>
    <ol>
     <li>
      并没有实现：结构与样式完全分离。
     </li>
     <li>
      多个HTML页面无法复用样式。
     </li>
    </ol>
    <h3>
     <a id="_751">
     </a>
     外部样式
    </h3>
    <ul>
     <li>
      写在单独的，css文件中，随后在HTML文件中引入使用。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <ol>
     <li>
      新建一个扩展名为.css的样式文件，把所有CSS代码都放入此文件中。
     </li>
    </ol>
    <pre><code>h1{ 
   color: red;
   font-size: 40px;
   }
</code></pre>
    <ol start="2">
     <li>
      在HTML文件中引入.css文件。
     </li>
    </ol>
    <pre><code>&lt;link rel="stylesheet"
href=./xxx.css&gt;
</code></pre>
    <ul>
     <li>
      注意点：
     </li>
    </ul>
    <ol>
     <li>
      <code>
       &lt;link&gt;
      </code>
      标签要写在
      <code>
       &lt;head&gt;
      </code>
      标签中。
     </li>
     <li>
      <code>
       &lt;link&gt;
      </code>
      标签属性说明:
     </li>
    </ol>
    <ul>
     <li>
      href：引入的文档来自于哪里。
     </li>
     <li>
      rel：(relation：关系）说明引入的文档与当前文档之间的关系。
     </li>
    </ul>
    <ol start="3">
     <li>
      外部样式的优势：样式可以复用、结构清晰、可触发浏览器的缓存机制，提高访问速度，实现了结构与样式的完全分离。
     </li>
     <li>
      实际开发中，几乎都使用外部样式，这是最推荐的使用方式!
     </li>
    </ol>
    <h2>
     <a id="042__773">
     </a>
     042_样式表的优先级
    </h2>
    <ul>
     <li>
      优先级规则：行内样式&gt;内部样式=外部样式
     </li>
    </ul>
    <ol>
     <li>
      内部样式、外部样式，这二者的优先级相同，且：后面的会覆盖 前面的（简记：“后来者居上”）。
     </li>
     <li>
      同一个样式表中，优先级也和编写顺序有关，且：后面的 会覆盖 前面的（简记：“后来者居上”）。
     </li>
    </ol>
    <h4>
     <a id="_777">
     </a>
     行内样式
    </h4>
    <p>
     优点： 优先级最高
     <br/>
     缺点：
    </p>
    <ol>
     <li>
      结构与样式未分离
     </li>
     <li>
      代码结构混乱
     </li>
     <li>
      样式不能复用
      <br/>
      使用频率：很低
      <br/>
      作用范围：当前标签
     </li>
    </ol>
    <h4>
     <a id="_785">
     </a>
     内部样式
    </h4>
    <p>
     优点：
    </p>
    <ol>
     <li>
      样式可复用
     </li>
     <li>
      代码结构清晰
      <br/>
      缺点：
     </li>
     <li>
      结构与样式未彻底分离
     </li>
     <li>
      样式不能多页面复用
      <br/>
      使用频率：一般
      <br/>
      作用范围：当前页面
     </li>
    </ol>
    <h4>
     <a id="_794">
     </a>
     外部样式
    </h4>
    <p>
     优点：
    </p>
    <ol>
     <li>
      样式可多页面复用
     </li>
     <li>
      代码结构清晰
     </li>
     <li>
      可触发浏览器的缓存机制
     </li>
     <li>
      结构与样式彻底分离
      <br/>
      缺点：需要引入才能使用
      <br/>
      使用频率：最高
      <br/>
      作用范围：多个页面
     </li>
    </ol>
    <h2>
     <a id="043_CSS_803">

     043_CSS语法规范
    </h2>

    </p>

      <br/>

      <br/>
      备注2：选择器与声明块之间，属性名与属性值之间，均有一个空格，理论上能省略，但最好还是写上。）
     </li>
     <li>
      注释的写法：
     </li>
    </ul>
    <pre><code>/*给h1元素添加样式*/
h1{
   /*设置文字颜色为红色 */
   color: red;
   /*设置文字大小为48px*/
   font-size:40px
   }
</code></pre>

   }
</code></pre>

     <li>

</code></pre>
    <p>

    </p>
    <h2>
     <a id="045_CSS_832">
     </a>
     045_CSS基本选择器
    </h2>
    <p>
     基本选择器包括：
    </p>
    <ol>
     <li>
      通配选择器
     </li>
     <li>
      元素选择器
     </li>
     <li>
      类选择器
     </li>
     <li>
      id选择器
     </li>
    </ol>
    <h3>
     <a id="_838">
     </a>
     通配选择器
    </h3>
    <ul>
     <li>
      作用：可以选中所有的HTML元素。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <pre><code>*{
   属性名：属性值；
}
</code></pre>
    <ul>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>*{
  /*选中所有元素 */color: orange;
  font-size:40px;
}
</code></pre>
    <p>
     备注：目前来看通配选择器貌似有点鸡肋，但后面清除样式时，会对我们有很大帮助，后面会详细讲。
    </p>
    <h3>
     <a id="_854">
     </a>
     元素选择器
    </h3>
    <ul>
     <li>
      作用：为页面中 某种元素 统一设置样式。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <pre><code>标签名{
      属性名：属性值；
    }
</code></pre>
    <ul>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中所有h1元素*/
h1{
   color:orange;
   font-size: 40px;
  }
/*选中所有p元素*/
p{
  color:
  blue;font-size:60px;
  }
</code></pre>
    <p>
     备注：元素选择器无法实现差异化设置，例如上面的代码中，所有的p元素效果都一样。
    </p>
    <h3>
     <a id="_876">
     </a>
     类选择器
    </h3>
    <ul>
     <li>
      作用：根据元素的class值，来选中某些元素。（class 翻译过来有：种类、类别的含义，所以class值，又称：类名。）
     </li>
     <li>
      语法：
     </li>
    </ul>
    <pre><code>.类名{
     属性名：属性值；
    }
</code></pre>
    <ul>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中所有class值为speak的元素*/
.speak{
      color: red;
    }
/*选中所有class值为answer的元素*/
.answer {
         color:blue;
         }
</code></pre>
    <ul>
     <li>
      注意点：
     </li>
    </ul>
    <ol>
     <li>
      元素的class属性值不带，，但CSS的类选择器要带
     </li>
     <li>
      class值，是我们自定义的，按照标准：不要使用纯数字、不要使用中文、尽量使用英文与数字的组合，若由多个单词组成，使用一做连接，例如：left-menu，且命名要有意义，做到“见名知意”。
     </li>
     <li>
      一个元素不能写多个class 属性，下面是错误示例：
     </li>
    </ol>
    <pre><code>&lt;！--该写法错误，元素的属性不能重复，后写的会失效--&gt;
&lt;h1 class="speak"class="big"&gt;你好啊&lt;/h1&gt;
</code></pre>
    <ol start="4">
     <li>
      一个元素的class属性，能写多个值，要用空格隔开，例如：
     </li>
    </ol>
    <pre><code>&lt;！--该写法正确，class属性，能写多个值
&lt;h1 class="speak big"&gt;你好啊&lt;/h1&gt;
</code></pre>
    <h3>
     <a id="ID_908">
     </a>
     ID选择器
    </h3>
    <ul>
     <li>
      作用：根据元素的id属性值，来精准的选中某个元素。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <pre><code>#id值{
      属性名：属性值；
      }
</code></pre>
    <ul>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中id值为earthy的那个元素*/
#earthy{
        color:red;
        font-size: 60px;
        }
</code></pre>
    <ul>
     <li>
      注意:
     </li>
     <li>
      id属性值：尽量由字母、数字、下划线( _ )、短杠（-)组成，最好以字母开头、不要包含空格、区分大小写。
     </li>
     <li>
      一个元素只能拥有一个id 属性，多个元素的id 属性值不能相同。
     </li>
     <li>
      一个元素可以同时拥有id和class属性。
     </li>
    </ul>
    <h3>
     <a id="_928">
     </a>
     总结
    </h3>
    <ul>
     <li>
      通配选择器：选中所有标签，一般用于清除样式。
     </li>
     <li>
      元素选择器：选中所有同种标签，但是不能差异化选择。
     </li>
     <li>
      类选择器：选中所有特定类名（class值）的元素-—使用频率很高。
     </li>
     <li>
      ID选择器：选中特定id值的那个元素（唯一的）。
     </li>
    </ul>
    <h2>
     <a id="046___933">
     </a>
     046_ 交集选择器
    </h2>
    <ul>
     <li>
      作用：选中同时符合多个条件的元素。（交集有并且的含义（通俗理解：即………又……的意思），例如：年轻且长得帅。）
     </li>
     <li>
      语法：
      <code>
       选择器1选择器2选择器3…选择器n {}
      </code>
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中：类名为beauty的p元素，为此种写法用的非常多！！！!*/
p.beauty{
         color:blue;
         }
/*选中：类名包含rich和beauty的元素
.rich.beauty {
              color: green;
            }
</code></pre>
    <ul>
     <li>
      注意：
     </li>
    </ul>
    <ol>
     <li>
      有标签名，标签名必须写在前面。
     </li>
     <li>
      id选择器、理论上可以作为交集的条件，但实际应用中几乎不用—-因为没有意义。
     </li>
     <li>
      交集选择器中不可能出现两个元素选择器，因为一个元素，不可能即是p元素又是span元素。
     </li>
     <li>
      用的最多的交集选择器是：元素选择器配合类名选择器，例如：p.beauty。
     </li>
    </ol>
    <h2>
     <a id="047__953">
     </a>
     047_并集选择器
    </h2>
    <ul>
     <li>
      作用：选中多个选择器对应的元素，又称：分组选择器。（所谓并集就是或者的含义（通俗理解：要么……要么……的意思)，例如：给我转10万块钱或者我报警。）
     </li>
     <li>
      语法：
      <code>
       选择器1,选择器2,选择器3,…选择器n {}
      </code>
      <br/>
      （多个选择器通过，连接，此处，的含义就是：或。）
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中id为peiqi，或类名为rich，或类名为beauty的元素 */
#peiqi,
.rich,
.beauty {
    font-size: 40px:
    background-color: skyblue;
    width:200px;
    }
</code></pre>
    <ul>
     <li>
      注意：
     </li>
    </ul>
    <ol>
     <li>
      并集选择器，我们一般竖着写。
     </li>
     <li>
      任何形式的选择器，都可以作为并集选择器的一部分。
     </li>
     <li>
      并集选择器，通常用于集体声明，可以缩小样式表体积。
     </li>
    </ol>
    <h2>
     <a id="048__972">
     </a>
     048_元素辈分关系
    </h2>
    <ul>
     <li>
      父元素:
      <br/>
      包裹某个元素的元素，就是该元素的父元素。
     </li>
     <li>
      子元素：
      <br/>
      被父元素包含的元素（简记：儿子元素）。
     </li>
     <li>
      祖先元素：
      <br/>
      父亲的父亲….，一直往外找，都是祖先。
      <br/>
      （备注：父元素，也算是祖先元素的一种。
      <br/>
      例如：张三的父亲，也算是张三的祖先，但一般还是称呼：父亲。）
     </li>
     <li>
      后代元素：儿子的儿子…，一直往里找，都是后代。
      <br/>
      （备注：子元素，也算是后代元素的一种。
      <br/>
      例如：张三的儿子，也算是张三的后代，但一般还是称呼：儿子。）
     </li>
     <li>
      兄弟元素：具有相同父元素的元素，互为兄弟元素。
     </li>
    </ul>
    <h2>
     <a id="049__985">
     </a>
     049_后代选择器
    </h2>
    <ul>
     <li>
      作用：选中指定元素中，符合要求的后代元素。
     </li>
     <li>
      ·语法：
      <code>
       选择器1 选择器2 选择器3 …… 选择器n {}
      </code>
      <br/>
      （选择器之间，用空格隔开，空格可以理解为：“xxx中的”，其实就是后代的意思。
      <br/>
      选择器1234…n，可以是我们之前学的任何一种选择器。）
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/* 选中ul中的所有li*/
ul li{
     color:red;
     }
/*选中ul中所有11中的a*/
ul l1 a{
      color:orange:
      }
/*选中类名为subject元素中的所有li*/
.subject li {
       color:blue;
       }    
/* 选中类名为subject元素中的所有类名为front-end的li*/
.subject li.front-end {
              color: blue;
              }
</code></pre>
    <ul>
     <li>
      注意：
     </li>
    </ul>
    <ol>
     <li>
      后代选择器，最终选择的是后代，不选中祖先。
     </li>
     <li>
      儿子、孙子、重孙子，都算是后代。
     </li>
     <li>
      结构一定要符合之前讲的 HTML 嵌套要求，例如：不能p中写h1～h6。
     </li>
    </ol>
    <h2>
     <a id="050__1013">
     </a>
     050_子代选择器
    </h2>
    <ul>
     <li>
      作用：选中指定元素中，符合要求的子元素（儿子元素）。（先写父，再写子)
      <br/>
      （子代选择器又称：子元素选择器、子选择器。）
     </li>
     <li>
      语法：
      <code>
       选择器1&gt;选择器2&gt;选择器3&gt;……选择器n {}
      </code>
      （选择器之间，用&gt;隔开，&gt;可以理解为：”xxx的子代"，其实就是儿子的意思。选择器1234…n，可以是我们之前学的任何一种选择器。）
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/* div中的子代a元素 */
div&gt;a {
     color: red;
    }
/*类名为person的元素中的子代a元素*/
.persons&gt;a {
       color:red;
       }
</code></pre>
    <ul>
     <li>
      注意：
     </li>
    </ul>
    <ol>
     <li>
      子代选择器，最终选择的是子代，不是父级。
     </li>
     <li>
      子、孙子、重孙子、重重孙子 …… 统称后代！子就是指儿子.
     </li>
    </ol>
    <h2>
     <a id="051__1031">
     </a>
     051_兄弟选择器
    </h2>
    <h3>
     <a id="_1032">
     </a>
     相邻兄弟选择器
    </h3>
    <ul>
     <li>
      作用：选中指定元素后，符合条件的相邻兄弟元素。（所谓相邻，就是紧挨着他的下一个，简记：睡在我下铺的兄弟。）
     </li>
     <li>
      语法：
      <code>
       选择器1+选择器2 {}
      </code>
     </li>
     <li>
      示例：
     </li>
    </ul>
    <pre><code>/* 选中div后相邻的兄弟p元素 */
div+p{
     color:red;
     }
</code></pre>
    <h3>
     <a id="_1042">
     </a>
     通用兄弟选择器
    </h3>
    <ul>
     <li>
      作用：选中指定元素后，符合条件的所有兄弟元素。（简记：睡在我下铺的所有兄弟)
     </li>
     <li>
      语法：
      <code>
       选择器1~选择器2 {}
      </code>
     </li>
     <li>
      示例：
     </li>
    </ul>
    <pre><code>/*选中div后的所有的兄弟p元素 */
div~p{
    color:red;
    }
</code></pre>
    <ul>
     <li>
      注意：两种兄弟选择器，选择的是下面的兄弟。
     </li>
    </ul>
    <h2>
     <a id="052__1053">
     </a>
     052_属性选择器
    </h2>
    <ul>
     <li>
      作用：选中属性值符合一定要求的元素。
     </li>
     <li>
      语法：
     </li>
    </ul>
    <ol>
     <li>
      ［属性名］选中具有某个属性的元素。
     </li>
     <li>
      ［属性名=“值”] 选中包含某个属性，且属性值等于指定值的元素。
     </li>
     <li>
      ［属性名^="值”] 选中包含某个属性，且属性值以指定的值开头的元素。
     </li>
     <li>
      ［属性名$=“值”］选中包含某个属性，且属值以指定的值结尾的元素。
     </li>
     <li>
      ［属性名*=“值”］选择包含某个属性，属性值包含
     </li>
    </ol>
    <ul>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*选中具有title属性的元素*/
div[title]{color:red; }
/*选中title属性值为atguigu的元素 */
div[title="atguigu"]{color:red;}
/*选中title属性值以a开头的元素*/
div[title^="a"]{color:red;}
/*选中title属性值以u结尾的元素 */
div[titleS="u"]{color:red; }
/*选中title属性值包含g的元素 */
div[title*="g"]{color:red;}
</code></pre>
    <h2>
     <a id="053__1074">
     </a>
     053_伪类选择器
    </h2>
    <ul>
     <li>
      作用：选中特殊状态的元素。
     </li>
    </ul>
    <ol>
     <li>
      如何理解“伪”——虚假的，不是真的。
     </li>
     <li>
      如何理解“伪类”？——像类(class)，但不是类，是元素的一种特殊状态。
     </li>
    </ol>
    <h3>
     <a id="_1078">
     </a>

    <h2>
     <a id="055__1130">
     </a>
     055_选择器的优先级
    </h2>
    <p>
     通过不同的选择器，选中相同的元素，并且为相同的样式名设置不同的值时，就发生了样式的冲突。
     <br/>
     到底应用哪个样式，此时就需要看优先级了。
    </p>
    <h4>
     <a id="_1133">
     </a>
     简单描述：
    </h4>
    <p>
     行内样式&gt;ID选择器&gt;类选择器&gt;元素选择器&gt;通配选择器&gt;继承过来的属性
    </p>
    <h4>
     <a id="_1135">
     </a>
     详细描述
    </h4>
    <h5>
     <a id="1_abc_1136">
     </a>
     1. 计算方式：每个选择器，都可计算出一组权重，格式为：（a，b，c)
    </h5>
    <ul>
     <li>
      a：1D选择器的个数。
     </li>
     <li>
      b：类、伪类、属性选择器的个数。
     </li>
     <li>
      c：元素、伪元素 选择器的个数。
      <br/>
      例如:
      <br/>
      <code>
       ul&gt;li
      </code>
      (0,0,2)
      <br/>
      <code>
       div ul&gt;li p a span
      </code>
      (0,0,6)
      <br/>
      <code>
       #atguigu .slogan
      </code>
      (1,1,0)
      <br/>
      <code>
       #atguigu .slogan a
      </code>
      (1,1,1)
      <br/>
      <code>
       #atguigu .slogan a:hover
      </code>
      (1,2,1)
     </li>
    </ul>
    <h5>
     <a id="2__1146">
     </a>
     2. 比较规则
    </h5>
    <p>
     按照从左到右的顺序，依次比较大小，当前位胜出后，后面的不再对比，
     <br/>
     例如:
    </p>
    <ul>
     <li>
      (1,0,0)&gt;(0,2,2)
     </li>
     <li>
      (1,1,0)&gt;(1,0,2)
     </li>
     <li>
      (1,1,3)&gt;(1,1,2)
     </li>
    </ul>
    <h5>
     <a id="_1152">
     </a>
     特殊规则
    </h5>
    <ol>
     <li>
      行内样式权重大于所有选择器。
     </li>
     <li>
      !important的权重，大于行内样式，大于所有选择器，权重最高！
     </li>
    </ol>
    <h2>
     <a id="056_CSS_1155">
     </a>
     056_CSS三大特性
    </h2>
    <h3>
     <a id="_1156">
     </a>
     层属性
    </h3>
    <ul>
     <li>
      概念：如果发生了样式冲突，那就会根据一定的规则（选择器优先级），进行样式的层叠（覆盖）。
     </li>
     <li>
      什么是样式冲突?——元素的同一个样式名，被设置了不同的值，这就是冲突。
     </li>
    </ul>
    <h3>
     <a id="_1159">
     </a>
     继承性
    </h3>
    <ul>
     <li>
      概念：元素会自动拥有其父元素、或其祖先元素上所设置的某些样式。
     </li>
     <li>
      规则：优先继承离得近的。
     </li>
     <li>
      常见的可继承属性: text-??,font-??,line-??,color…
     </li>
     <li>
      备注：参照MDN网站 可查询属性是否可被继承
     </li>
    </ul>
    <h3>
     <a id="_1164">
     </a>
     优先级
    </h3>
    <ul>
     <li>
      简单聊:
      <br/>
      !important &gt;行内样式 &gt;ID选择器 &gt;类选择器 &gt;元素选择器 &gt;*&gt;继承的样式。
     </li>
     <li>
      详细聊：需要计算权重。
      <br/>
      计算权重时需要注意：并集选择器的每一个部分是分开算的!
     </li>
    </ul>
    <h2>
     <a id="057__1169">
     </a>
     057_颜色的表示
    </h2>
    <h3>
     <a id="_1170">
     </a>
     方式一：颜色名
    </h3>
    <ul>
     <li>
      编写方式：直接使用颜色对应的英文单词，编写比较简单，例如:
     </li>
    </ul>
    <ol>
     <li>
      <p>
       红色：red
      </p>
     </li>
     <li>
      <p>
       绿色：green
      </p>
     </li>
     <li>
      <p>
       蓝色：blue
      </p>
     </li>
     <li>
      <p>
       紫色：purple
      </p>
     </li>
     <li>
      <p>
       橙色：orange
      </p>
     </li>
     <li>
      <p>
       灰色：gray
      </p>
      <ol>
       <li>
        颜色名这种方式，表达的颜色比较单一，所以用的并不多。
       </li>
       <li>
        具体颜色名参考MDN官方文档:
        <br/>
        https://developer.mozilla.org/en-US/docs/Web/CSS/named-color
       </li>
      </ol>
     </li>
    </ol>
    <h3>
     <a id="rgbrgba_1182">
     </a>
     方式二：rgb或rgba
    </h3>
    <ul>
     <li>
      编写方式：使用 红、黄、蓝 这三种光的三原色进行组合。
     </li>
     <li>
      r表示 红色
     </li>
     <li>
      g表示 绿色
     </li>
     <li>
      b表示 蓝色
     </li>
     <li>
      a表示 透明度
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>/*使用 0~255之间的数字表示一种颜色*/
color:rgb(255,0,0)；/*红色*/
color：rgb(0，255，0)；/*绿*/
color:rgb(0，0，255)；/*蓝色*/
color:rgb(0,0,0)；/*黑色*/
color: rgb(255,255,255);/* 白色 */

/*混合出任意一种颜色 */
color:rgb(138，43，226)/*紫罗兰色 */
color:rgba(255,0,0，0.5)；/*半透明的红色 */
/*也可以使用百分比表示一种颜色（用的少）*/
color:rgb(100%，0%，0%)；/*红色 */
color:rgba(100%，0%，0%,50%)；/*半透明的红色 *
</code></pre>
    <ul>
     <li>
      小规律：
     </li>
    </ul>
    <ol>
     <li>
      若三种颜色值相同，呈现的是灰色，值越大，灰色越浅。
     </li>
     <li>
      rgb(0,0，0)是黑色，rgb(255，255,255）是白色。
     </li>
     <li>
      对于rbga来说，前三位的rgb形式要保持一致，要么都是0~255的数字，要么都是百分比。
     </li>
    </ol>
    <h3>
     <a id="HEXHEXA_1208">
     </a>
     方式三：HEX或HEXA
    </h3>
    <p>
     HEX的原理同与rgb一样，依然是通过：红、绿、蓝色 进行组合，只不过要用6个数字，分成3组 来表达
    </p>
    <ul>
     <li>
      格式为：#rrggbb
      <br/>
      （每一位数字的取值范围是：0～f，所以每一种光的最小值是：00，最大值是：ff）
     </li>
    </ul>
    <pre><code>color:#ff0000；/*红色 */
color：#00ff0O；/*绿色 */
color：#0000ff；/*蓝色 */
color：#000000；/*黑色 */
color：#ffffff；/*白色*/
/*如果每种颜色的两位都是相同的，就可以简写*/
color：#ff9988；/*可简为：#f98 */
/*但要注意前三位简写了，那么透明度就也要简写*/
color：#ff998866；/*可简为：#f986 */
</code></pre>
    <ul>
     <li>
      注意点：IE浏览器不支持HEXA，但支持HEX。
     </li>
    </ul>
    <h3>
     <a id="HSLHSLA_1224">
     </a>
     方式四：HSL或HSLA
    </h3>
    <ul>
     <li>
      HSL是通过：色相、饱和度、亮度，来表示一个颜色的，格式为：hsl(色相，饱和度，亮度）
     </li>
     <li>
      色相：取值范围是0~360度，具体度数对应的颜色上网搜（从红色开始顺时针转）
     </li>
     <li>
      饱和度：取值范围是0%~100%。（向色相中对应颜色中添加灰色，0%全灰，100%没有灰)
     </li>
     <li>
      亮度：取值范围是0%~100%。（0%亮度没了，所以就是黑色。100%亮度太强，所以就是白色了）
     </li>
     <li>
      HSLA其实就是在HSL的基础上，添加了透明度。
     </li>
    </ul>

    <h3>
     <a id="_1354">
     </a>
     行高
    </h3>
    <ul>
     <li>
      属性值：line-height
     </li>
     <li>
      可选值：
      <ol>
       <li>
        normal：由浏览器根据文字大小决定的一个默认值
       </li>
       <li>
        像素（px）
       </li>
       <li>
        数字：参考自身font-size的倍数（很常用）
       </li>
       <li>
        百分比：参考自身font-size的百分比
       </li>
      </ol>
     </li>
     <li>
      备注：由于字体设计问题，文字在一行中，并不是绝对垂直居中，若一行中都是文字，不会太影响观感
     </li>
     <li>
      举例：
     </li>
    </ul>
    <pre><code>div {
    line-height:60px;
    line-height: 1.5;
    line-height:150%;
}
</code></pre>
    <ul>
     <li>
      行高注意事项：
      <ol>
       <li>
        line-height过小会怎样？—-文字产生重叠，且最小值是0，不能为负数。
       </li>
       <li>
        line-height是可以继承的，且为了能更好的呈现文字，最好写数值。
       </li>
       <li>
        line-height和height是什么关系?
        <ul>
         <li>
          设置了height，那么高度就是height的值。
         </li>
         <li>
          不设置height的时候，会根据line-height计算高度。
         </li>
        </ul>
       </li>
      </ol>
     </li>
     <li>
      应用场景：
      <ol>
       <li>
        对于多行文字：控制行与行之间的距离。
       </li>
       <li>
        对于单行文字：让height等于line-height，可以实现文字垂直居中。
       </li>
      </ol>
      <ul>
       <li>
        备注：
        <br/>
        由于字体设计原因，靠上述办法实现的居中，并不是绝对的垂直居中，但如果一行中都是文字，不会太影响观感。
       </li>
      </ul>
     </li>
    </ul>

     </li>

     </li>
    </ol>
    <ul>
     <li>
      备注：×是根据字体族，动态决定的一个值。
     </li>
     <li>

     2.元素的显示模式
    </h3>
    <h4>
     <a id="block_1534">
     </a>
     块元素(block)
    </h4>
    <ul>
     <li>
      又称：块级元素
     </li>
     <li>
      特点：
      <ol>
       <li>
        1.在页面中独占一行，不会与任何元素共用一行，是从上到下排列的。
       </li>
       <li>
        默认宽度：撑满父元素。
       </li>
       <li>
        默认高度：由内容撑开。
       </li>
       <li>
        可以通过CSS设置宽高。
       </li>
      </ol>
     </li>
    </ul>
    <h4>
     <a id="inline_1541">
     </a>
     行内元素(inline)
    </h4>
    <ul>
     <li>
      又称：内联元素
     </li>
     <li>
      特点：
      <ol>
       <li>
        在页面中不独占一行，一行中不能容纳下的行内元素，会在下一行继续从左到右排列。
       </li>
       <li>
        默认宽度：由内容撑开。
       </li>
       <li>
        默认高度：由内容撑开。
       </li>
       <li>
        无法通过CSS设置宽高。
       </li>
      </ol>
     </li>
    </ul>
    <h4>
     <a id="inlineblock_1548">
     </a>
     行内块元素(inline-block)
    </h4>
    <ul>
     <li>
      又称：内联块元素
     </li>
     <li>
      特点：
      <ol>
       <li>
        1.在页面中不独占一行，一行中不能容纳下的行内元素，会在下一行继续从左到右排列。
       </li>
       <li>
        默认宽度：由内容撑开。
       </li>
       <li>
        默认高度：由内容撑开。
       </li>
       <li>
        可以通过CSS设置宽高。
        <br/>
        注意：元素早期只分为：行内元素、块级元素，区分条件也只有一条：“是否独占一行”，如果按照这种分类方式，行内块元素应该算作行内元素。
       </li>
      </ol>
     </li>
    </ul>

    <h3>
     <a id="5_1581">
     </a>
     5.盒子模型的组成
    </h3>
    <ul>
     <li>
      CSS会把所有的HTML元素都看做一个盒子，所有的样式也都基于这个盒子
     </li>
    </ul>
    <ol>
     <li>
      margin（外边距）：盒子与外界的蹈离。
     </li>
     <li>
      border（边框）：盒子的边框。
     </li>
     <li>
      padding（内边距）：紧贴内容的补白区域。
     </li>
     <li>
      content（内容）：元素中的文本或后代元素都是它的内容
     </li>
    </ol>
    <ul>
     <li>
      盒子的大小=content+左右padding+左右border
     </li>
     <li>
      注意：外边距margin不会影响盒子的大小，但会影响盒子的位置
     </li>
    </ul>

    </h5>

    <h5>

     </a>
     margin-top
    </h5>
    <ul>
     <li>
      功能：上外边距
     </li>
     <li>
      属性值：CSS中的长度值
     </li>
    </ul>
    <h5>
     <a id="marginbottom_1679">
     </a>
     margin-bottom
    </h5>
    <ul>
     <li>
      功能：下外边距
     </li>
     <li>
      属性值：CSS中的长度值
     </li>
    </ul>
    <h5>
     <a id="margin_1682">
     </a>
     margin
    </h5>
    <ul>
     <li>
      功能：复合属性，可以写1~4个值，规律同padding（顺时针）（上右下左）
     </li>
     <li>
      属性值：CSS中的长度值
     </li>
    </ul>
    <h4>
     <a id="101_margin_1685">
     </a>
     10.1 margin注意事项
    </h4>
    <ol>
     <li>
      子元素的margin，是参考父元素的content计算的。（因为是父亲的content中承装着子元素)
     </li>
     <li>
      上margin、左margin：影响自己的位置；下margin、右margin：影响后面兄弟元素的位置。
     </li>
     <li>
      块级元素、行内块元素，均可以完美地设置四个方向的margin；但行内元素，左右margin可以完美设置，上下margin设置无效。
     </li>
     <li>
      margin的值也可以是auto，如果给一个块级元素设置左右margin都为auto，该块级元素会在父元素中水平居中。
     </li>
     <li>
      margin的值可以是负值。
     </li>
    </ol>
    <h4>
     <a id="102_margin_1691">
     </a>
     10.2 margin塌陷问题
    </h4>
    <ul>
     <li>
      什么是margin 塌陷?
      <ul>
       <li>
        第一个子元素的上margin会作用在父元素上，最后一个子元素的下margin会作用在父元素上。
       </li>
      </ul>
     </li>
     <li>
      如何解决margin塌陷?
      <ul>
       <li>
        方案一：给父元素设置不为0的padding。
       </li>
       <li>
        方案二：给父元素设置宽度不为0的border。
       </li>
       <li>
        方案三：给父元素设置css样式overflow:hidden
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="103_margin_1698">
     </a>
     10.3 margin合并问题
    </h4>
    <ul>
     <li>
      什么是margin合并?
      <ul>
       <li>
        上面兄弟元素的下外边距和下面兄弟元素的上外边距会合并，取一个最大的值，而不是相加。
       </li>
      </ul>
     </li>
     <li>
      如何解决margin 塌陷?
      <ul>
       <li>
        无需解决，布局的时候上下的兄弟元素，只给一个设置上下外边距就可以了。
       </li>
      </ul>
     </li>
    </ul>

    </h3>
    <h4>
     <a id="visibility_1721">
     </a>

    </h4>
    <p>

    </p>
    <h3>
     <a id="13_1727">
     </a>
     13.样式的继承
    </h3>
    <ul>
     <li>
      有些样式会继承，元素如果本身设置了某个样式，就使用本身设置的样式；但如果本身没有设置某个样式，会从父元素开始一级一级继承（优先继承离得近的祖先元素）。
     </li>
    </ul>
    <h4>
     <a id="css_1729">
     </a>
     会继承的css属性
    </h4>
    <ul>
     <li>
      字体属性、文本属性（除了vertical-align）、文字颜色等。
     </li>
    </ul>
    <h4>
     <a id="css_1731">
     </a>
     不会继承的css属性
    </h4>
    <ul>
     <li>
      边框、背景、内边距、外边距、宽高、溢出方式等。
     </li>
    </ul>
    <h4>
     <a id="_1733">
     </a>
     一个规律：
    </h4>
    <ul>
     <li>
      能继承的属性，都是不影响布局的，简单说：都是和盒子模型没关系的。
     </li>
    </ul>
    <h3>
     <a id="14_1735">
     </a>
     14.默认样式
    </h3>
    <ul>
     <li>
      元素一般都些默认的样式，例如:
      <ol>
       <li>
        <code>
         &lt;a&gt;
        </code>
        元素：下划线、字体颜色、鼠标小手。
       </li>
       <li>
        <code>
         &lt;h1&gt;~&lt;h6&gt;
        </code>
        元素：文字加粗、文字大小、上下外边距。
       </li>
       <li>
        <code>
         &lt;p&gt;
        </code>
        元素：上下外边距
       </li>
       <li>
        <code>
         &lt;ul&gt;
        </code>
        、
        <code>
         &lt;ol&gt;
        </code>
        元素：左内边距
       </li>
       <li>
        body元素：8px外边距（4个方向）
       </li>
      </ol>
     </li>
     <li>
      优先级：元素的默认样式&gt;继承的样式，所以如果要重置元素的默认样式，选择器一定要直接选择器到该元素。
     </li>
    </ul>
    <h3>
     <a id="15_1743">
     </a>
     15.布局小技巧
    </h3>
    <ol>
     <li>
      行内元素、行内块元素，可以被父元素当做文本处理。
      <ul>
       <li>
        即：可以像处理文本对齐一样，去处理：行内、行内块在父元素中的对齐。
       </li>
       <li>
        例如：text-align、line-height、text-indent等。
       </li>
      </ul>
     </li>
     <li>
      如何让子元素，在父亲中 水平居中:
      <ul>
       <li>
        若子元素为块元素，给父元素加上：margin:0 auto;
       </li>
       <li>
        若子元素为行内元素、行内块元素，给父元素加上：text-align:center
       </li>
      </ul>
     </li>
     <li>
      如何让子元素，在父亲中 垂直居中:
      <ul>
       <li>
        若子元素为块元素，给子元素加上：margin-top，值为：（父元素content-子元素盒子总高）/2
       </li>
       <li>
        若子元素为行内元素、行内块元素：
        <ul>
         <li>
          让父元素的height = line-height，每个子元素都加上：vertical-align:middle;
         </li>
         <li>
          补充：若想绝对垂直居中，父元素font-size设置为0。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="16_1755">
     </a>
     16.元素之间的空白问题
    </h3>
    <ul>
     <li>
      产生的原因：
      <ul>
       <li>
        行内元素、行内块元素，彼此之间的换行会被浏览器解析为一个空白字符。
       </li>
      </ul>
     </li>
     <li>
      解决方案:
      <ol>
       <li>
        方案一：去掉换行和空格（不推荐）。
       </li>
       <li>
        方案二：给父元素设置font-size：0，再给需要显示文字的元素，单独设置字体大小（推荐)
       </li>
      </ol>
     </li>
    </ul>
    <h3>
     <a id="17_1761">
     </a>
     17.行内块的幽灵空白问题
    </h3>
    <ul>
     <li>
      产生原因：
      <ul>
       <li>
        行内块元素与文本的基线对齐，而文本的基线与文本最底端之间是有一定距离的。
       </li>
      </ul>
     </li>
     <li>
      解决方案：
      <ol>
       <li>
        方案一：给行行内块设置vertical，值不为 baseline 即可，设置为middel、bottom、top均可。
       </li>
       <li>
        方案二：若父元素中只有一张图片，设置图片为display：block。
       </li>
       <li>
        方案三：给父元素设置font-size：0。如果该行内块内部还有文本，则需单独设置font-size。
       </li>
      </ol>
     </li>
    </ul>
    <h2>
     <a id="067__1768">
     </a>
     067_浮动
    </h2>
    <h3>
     <a id="1_1769">
     </a>
     1.浮动的简介
    </h3>
    <ul>
     <li>
      在最初，浮动是用来实现文字环绕图片效果的，现在浮动是主流的页面布局方式之一。
     </li>
    </ul>
    <h3>
     <a id="2_1771">
     </a>
     2.元素浮动后的特点
    </h3>
    <ol>
     <li>
      脱离文档流
     </li>
     <li>
      不管浮动前是什么元素，浮动后：默认宽与高都是被内容撑开（尽可能小），而且可以设置宽高。
     </li>
     <li>
      不会独占一行，可以与其他元素共用一行。
     </li>
     <li>
      不会margin合并，也不会margin塌陷，能够完美的设置四个方向的margin和padding。
     </li>
     <li>
      不会像行内块一样被当做文本处理（没有行内块的空白问题）。
     </li>
    </ol>
