     2 加载数据

         </li>
         <li>
          按住ctrl 双击
         </li>
         <li>
          ctrl t
         </li>
        </ul>
       </li>
      </ul>
      <img src="https://i-blog.csdnimg.cn/blog_migrate/acf907da2ec4b909f752e3cf93af76da.jpeg">
       <ul>
        <li>
         <p>
          表的一些操作
          <img src="https://i-blog.csdnimg.cn/blog_migrate/b8d7d885a732df73903b72f3bb526840.jpeg"/>
         </p>
        </li>
        <li>
         <p>
          对字段的一些操作
         </p>
         <ul>
          <li>
           在图层属性中可以选择哪些字段显示哪些不显示
           <img src="https://i-blog.csdnimg.cn/blog_migrate/142ac2b3c8ebfcb838dc9a2096671add.jpeg"/>
          </li>
          <li>
           右键某一个列
           <img src="https://i-blog.csdnimg.cn/blog_migrate/95d056d9e2ad2711d02476b8ea65483e.jpeg"/>
          </li>
          <li>
           还有一些功能
          </li>
          <li>
           汇总包括对字符和数值数据的处理，比如统计count。
          </li>
          <li>
           计算几何包括计算面积，后面会讲。
          </li>
          <li>
           属性，控制字段的属性，比如说保留几位小数
          </li>
         </ul>
        </li>
        <li>
         <p>
          选中的数据可以导出，导出进shp还是自己的数据库都可以选择
          <img src="https://i-blog.csdnimg.cn/blog_migrate/866f49becb1cb45bad17ebce59f5ef10.jpeg"/>
         </p>
        </li>
       </ul>
       <h3>
        <a id="5__48">
        </a>
        5 内容列表
       </h3>
       <p>
        内容列表就是最左边的部分，在内容列表中列出图层的方法有很多种
       </p>
       <ol>
        <li>
         按绘制顺序
        </li>
        <li>
         按源（存储位置）
        </li>
        <li>
         按图层是否可见（一般在按顺序里可以直接打勾设置）
        </li>
        <li>
         按图层是否可选
        </li>
       </ol>
       <ul>
        <li>
         在这里选择可以进行切换
        </li>
       </ul>
       <img alt="image-20230122154029199" src="/Users/xuan/Library/Application%20Support/typora-user-images/image-20230122154029199.png"/>
       <p>
        通过查看按钮（激活的按钮会有高亮显示），以及内容列表上项目的结构，可以区分处于活动状态的模式以。不同方式列出图层是显示图层相关信息的简单方法。
       </p>
       <h4>
        <a id="51__63">
        </a>
        5.1 按绘制顺序
       </h4>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/40cc58aa5f3d789a0a03080bcc60fad3.jpeg"/>
       <p>
        使用按绘制顺序列出来创作地图内容，例如更改地图中图层的显示顺序、重命名或移除图层以及创建或管理图层组。如果内容列表按照绘制顺序进行排序，则会列出地图中的所有数据框。但只有数据框名称为粗体的活动数据框才会显示在地图的数据视图中。
       </p>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/4ffabfb1e904986013a905017985d87f.jpeg"/>
       <h4>
        <a id="52__71">
        </a>
        5.2 按源
       </h4>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/135a95b37bf1ac493750960d00738013.jpeg"/>
       <p>
        单击按源列出会显示每个数据框中的图层，并将根据图层所引用的数据源所在文件夹或数据库对各图层进行编排。此视图还会列出己作为数据添加到地图文档的表。
       </p>
       <h4>
        <a id="53__77">
        </a>
        5.3 按可见性
       </h4>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/809bd7af472d845c54bebe06d6511e49.jpeg"/>
       <p>
        单击按可见性列出可以查看当前显示在活动数据框中的动态图层列表。在执行平移和缩放操作、与地图进行交互、选择要素及打开和关闭图层时，图层的排列顺序将会自动更新。
       </p>
       <h4>
        <a id="54__83">
        </a>
        5.4 按可选性
       </h4>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/0148f98548fc10e04250be5ca015de59.jpeg"/>
       <p>
        单击按选择列出 可以根据图层是否可选和是否包含已选要素来对图层进行自动分组。
       </p>
       <p>
        <strong>
         可选图层
        </strong>
        表示此图层中的要素可在编辑会话中使用
        <strong>
         交互式选择工具
        </strong>
        （例如，基础工具 工具条中的工具或编辑工具）进行选择。
       </p>

        <a id="_ArcCatalog_93">

        第二节 ArcCatalog入门
       </h2>
       <h3>
        <a id="1__95">
        </a>
        1 界面和功能介绍
       </h3>
       <p>
        <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/388ffd9c234481c85309102453698ea2.jpeg"/>
       </p>
       <ul>
        <li>
         作用
        </li>
       </ul>
       <p>
        <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/9b4b009575eb7fdd65be3cbbad8d7850.jpeg"/>
       </p>
       <p>
        ArcCatalog 将这些内容组织到树视图中，用户可以使用树视图来组织 GIS 数据集和 ArcGIS 文档，搜索和查找信息项以及管理信息项。
       </p>
       <p>
        ArcCatalog 以树视图形式显示这些信息，允许用户选择 GIS信息项，查看所选信息项的属性以及访问用于对所选信息项执行操作的工具。
       </p>
       <p>
        <strong>
         但是实际上，现在ArcMap的目录基本上等同于ArcCatalog，直接用目录就行，就不用在两者之间来回切了
        </strong>
       </p>
       <img src="https://i-blog.csdnimg.cn/blog_migrate/b8c4b98aa7c19528230c29f212749d21.jpeg"/>
       <ul>
        <li>
         <p>
          注意目录拉的比较宽时，目录里是左右分割，拉的窄的时候是上下分布
         </p>
        </li>
        <li>
         <img src="https://i-blog.csdnimg.cn/blog_migrate/d86d7db9a0f96af8832ab7a9e6b3bd2b.jpeg"/>
        </li>
        <li>
         <p>
          在内容面板里可以把数据批量来进图层显示，可以切换视图变成列表排列更好看，常用！
         </p>
        </li>
        <li>
         <p>
          ArcCatalog 用于
         </p>
         <ol>
          <li>
           组织 GIS 内容
          </li>
          <li>
           管理地理数据库方案
          </li>
          <li>
           搜索内容并将其添加到 ArcGIS 应用程序
          </li>
          <li>
           记录内容
          </li>
          <li>
           营理 GIS 服务器
          </li>
          <li>
           管理基于标准的元数据
          </li>
         </ol>
        </li>
       </ul>
