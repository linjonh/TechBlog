     二.ArrayList(顺序表)
    </h2>
    <h3>
     <a id="1ArrayList_30">
     </a>
     1.什么是ArrayList？
    </h3>
    <p>
     <strong>
      简介：
     </strong>
     <br/>
     ArrayList是Java标准库中的一个类，实现了List接口。顺序表是用一段
     <strong>
      物理地址连续
     </strong>
     的存储单元依次存储数据元素的线性结构，一般情况下采用数组存储，在数组上完成数据的增删查改
     <br/>
     特点：
    </p>
    <ol>
     <li>
      <pre><code> 动态大小：
</code></pre>
     </li>
     <li>
      <pre><code> 快速随机访问：通过下标访问元素，时间复杂度是O(1)
</code></pre>
     </li>
     <li>
      <pre><code> 存储任意类型的数据：支持存储任意类型的数据，包括基本数据类型(自动装箱)
</code></pre>
     </li>
     <li>
      <pre><code> 线程不安全：Vector(使用了synchronized)可以看作是ArrayList的线程安全版本，线程安全问题看我相关的博文
</code></pre>
     </li>
    </ol>

     <a id="LinkedList_176">
     </a>
     三.LinkedList(链表)
    </h2>
    <h3>
     <a id="1_177">
     </a>
     1.什么是线性表？
    </h3>
    <p>
     <strong>
      简介：
     </strong>
     <br/>
     <strong>
      线性表
     </strong>
     是n个具有
     <strong>
      相同特性
     </strong>
     的数据元素的有限序列，线性表在逻辑上是线性结构，也就说是连续的一条直线。但是在物理结构上并
     <strong>
      不一定是连续的空间
     </strong>
     (如下图)
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fc75dd4c3c0c4825842bbd09203e2188.png#pic_center"/>
     <br/>
     每个节点有两个域，一个存放
     <strong>
      数据/值
     </strong>
     (value)，一个存放
     <strong>
      下一个节点的地址
     </strong>
     (address)。通过上一个节点能够找到下一个节点，而且这些节点在物理结构上不一定是连续的，这就叫做线性表/链表。
    </p>
    <h3>
     <a id="2_182">
     </a>
     2.链表的分类
    </h3>
    <p>
     <strong>
      从方向上来讲有三种类型
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 单向链表(如上图)：每个节点只有一个指向下一个节点的指针，最后一个节点的指针指向null。单向链表只能从头到尾进行遍历
</code></pre>
     </li>
     <li>
      <pre><code> 双向链表：每个节点有两个指针，一个指向前一个节点，一个指向下一个节点。双向链表可以从任意一个节点开始，向前或向后进行遍历
</code></pre>
     </li>
     <li>
      <pre><code> 循环链表：链表的最后一个节点指向头节点，形成一个环。循环链表可以是单向的，也可以是双向的
</code></pre>
     </li>
    </ol>
    <p>
     <strong>
      从头节点的区别来讲有两个类型
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 带头链表：带头链表是指在链表的第一个节点之前增加一个特殊的节点，称为头结点。这个头结点通常不存储实际的数据，仅作为链表的起始点
</code></pre>
     </li>
     <li>
      <pre><code> 不带头链表(如上图)：链表中所有节点的结构是一致的，第一个节点是不确定的(后面讲到头插法就明白了)
</code></pre>
     </li>
    </ol>
    <p>
     通过以上排列组合，一共可以得到六种类型的链表
    </p>
    <ol>
     <li>
      <pre><code> 单向带头链表
</code></pre>
     </li>
     <li>
      <pre><code> 单向不带头链表
</code></pre>
     </li>
     <li>
      <pre><code> 双向带头链表
</code></pre>
     </li>
     <li>
      <pre><code> 双向不带头链表
</code></pre>
     </li>
     <li>
      <pre><code> 循环带头链表
</code></pre>
     </li>
     <li>
      <pre><code> 循环不带头链表
</code></pre>
     </li>
    </ol>
    <p>
     本文主要介绍
     <strong>
      单向不带头链表
     </strong>
     ，只要搞清楚这一个类型的原理，其他类型都是同理
    </p>

    <h2>
     <a id="VS_394">
     </a>
     四.顺序表VS链表
    </h2>
    <p>
     <strong>
      存储方式
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 顺序表：使用连续的内存空间存储元素
</code></pre>
     </li>
     <li>
      <pre><code> 链表：使用非连续的内存空间存储元素
</code></pre>
     </li>
    </ol>
    <p>
     <strong>
      访问方式
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 顺序表：支持随机访问，通过下标/索引在O(1)时间内访问
</code></pre>
     </li>
     <li>
      <pre><code> 链表：不支持随机访问，通过遍历链表访问元素(O(n)时间复杂度)
</code></pre>
     </li>
    </ol>
    <p>
     <strong>
      删除和插入
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 顺序表：需要移动大量元素，例如：删除下标为2的元素，后面的元素都需要向前覆盖
</code></pre>
     </li>
     <li>
      <pre><code> 链表：头插法/尾插法，删除头/尾节点时间复杂度是O(1),虽然其他位置的插入和删除时间复杂度也是O(n)，但不需要大量地移动元素
</code></pre>
     </li>
    </ol>
    <p>
     <strong>
      容量
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 顺序表：自动扩容
</code></pre>
     </li>
     <li>
      <pre><code> 链表：没有容量这个概念
</code></pre>
     </li>
    </ol>
    <p>
     <strong>
      适用场景
     </strong>
    </p>
    <ol>
     <li>
      <pre><code> 顺序表：元素高效存储+频繁访问
</code></pre>
     </li>
     <li>
      <pre><code> 链表：任意位置插入和删除频繁
</code></pre>
     </li>
    </ol>
