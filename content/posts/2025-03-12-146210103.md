    <p>
     linux 系统版本低，node版本高，导致不能通过编译的问题
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4fd3cd50fdce4fcaa3dcd7b080560c29.png"/>
    </p>
    <p>
     解决方案参考
     <br/>
     https://blog.csdn.net/zyplanke/article/details/142591873
    </p>
    <p>
     如上解决之后，可以将新装的定制node纳入nvm管理
    </p>
    <p>
     如果你已经手动安装了 Node.js（例如通过解压
     <code>
      node-v20.18.3-linux-x64-glibc-217.tar.gz
     </code>
     ），但希望将其纳入
     <code>
      nvm
     </code>
     管理，可以按照以下步骤操作：
    </p>
    <hr/>
    <h4>
     <a id="1__nvm_14">
     </a>
     1. 确保已安装
     <code>
      nvm
     </code>
    </h4>
    <p>
     如果还没有安装
     <code>
      nvm
     </code>
     ，先安装它：
    </p>
    <pre><code class="prism language-bash"><span class="token function">curl</span> -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh <span class="token operator">|</span> <span class="token function">bash</span>
<span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
    <p>
     验证
     <code>
      nvm
     </code>
     是否安装成功：
    </p>
    <pre><code class="prism language-bash">nvm <span class="token parameter variable">--version</span>
</code></pre>
    <hr/>
    <h4>
     <a id="2__Nodejs__nvm__28">
     </a>
     2. 将手动安装的 Node.js 纳入
     <code>
      nvm
     </code>
     管理
    </h4>
    <p>
     <code>
      nvm
     </code>
     默认会将所有 Node.js 版本安装在
     <code>
      ~/.nvm/versions/node
     </code>
     目录下。你可以将手动安装的 Node.js 复制到该目录，并让
     <code>
      nvm
     </code>
     识别它。
    </p>
    <h5>
     <a id="_31">
     </a>
     步骤如下：
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        找到手动安装的 Node.js 路径
       </strong>
       <br/>
       假设你已经解压了
       <code>
        node-v20.18.3-linux-x64-glibc-217.tar.gz
       </code>
       ，并放在了某个目录（例如
       <code>
        /opt/node-v20.18.3-linux-x64
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        将 Node.js 复制到
        <code>
         nvm
        </code>
        的版本目录
       </strong>
       <br/>
       将手动安装的 Node.js 复制到
       <code>
        ~/.nvm/versions/node
       </code>
       目录下：
      </p>
      <pre><code class="prism language-bash"><span class="token function">mkdir</span> <span class="token parameter variable">-p</span> ~/.nvm/versions/node/v20.18.3
<span class="token function">cp</span> <span class="token parameter variable">-r</span> /opt/node-v20.18.3-linux-x64/* ~/.nvm/versions/node/v20.18.3/
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        让
        <code>
         nvm
        </code>
        识别该版本
       </strong>
       <br/>
       <code>
        nvm
       </code>
       会自动扫描
       <code>
        ~/.nvm/versions/node
       </code>
       目录下的版本。你可以运行以下命令查看是否识别成功：
      </p>
      <pre><code class="prism language-bash">nvm <span class="token function">ls</span>
</code></pre>
      <p>
       如果看到
       <code>
        v20.18.3
       </code>
       ，说明已经成功纳入管理。
      </p>
     </li>
     <li>
      <p>
       <strong>
        切换到该版本
       </strong>
       <br/>
       使用以下命令切换到
       <code>
        v20.18.3
       </code>
       ：
      </p>
      <pre><code class="prism language-bash">nvm use v20.18.3
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        设置为默认版本（可选）
       </strong>
       <br/>
       如果你想将
       <code>
        v20.18.3
       </code>
       设置为默认版本，可以运行：
      </p>
      <pre><code class="prism language-bash">nvm <span class="token builtin class-name">alias</span> default v20.18.3
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="3__64">
     </a>
     3. 验证
    </h4>
    <p>
     验证 Node.js 和 npm 是否正常工作：
    </p>
    <pre><code class="prism language-bash"><span class="token function">node</span> <span class="token parameter variable">-v</span>  <span class="token comment"># 应该输出 v20.18.3</span>
<span class="token function">npm</span> <span class="token parameter variable">-v</span>   <span class="token comment"># 应该输出对应的 npm 版本</span>
</code></pre>
    <hr/>
    <h4>
     <a id="4__73">
     </a>
     4. 清理手动安装的文件（可选）
    </h4>
    <p>
     如果你已经将 Node.js 纳入
     <code>
      nvm
     </code>
     管理，可以删除之前手动安装的文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">rm</span> <span class="token parameter variable">-rf</span> /opt/node-v20.18.3-linux-x64
</code></pre>
    <hr/>
    <h4>
     <a id="_81">
     </a>
     总结
    </h4>
    <p>
     通过将手动安装的 Node.js 复制到
     <code>
      ~/.nvm/versions/node
     </code>
     目录，可以让
     <code>
      nvm
     </code>
     识别并管理它。这种方法适用于任何手动安装的 Node.js 版本。
    </p>
    <p>
     如果还有其他问题，请随时提问！
    </p>
