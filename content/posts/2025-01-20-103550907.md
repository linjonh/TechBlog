    <h2>
     <a id="_1">
     </a>
     我的开发硬件环境
    </h2>
    <pre><code>tiny4412,友善之臂的开发板
</code></pre>
    <h2>
     <a id="_3">
     </a>
     搭建步骤
    </h2>
    <h3>
     <a id="1__ORacle_ejdk8u211linuxarmsflttargz_linkhttpswwworaclecomjavatechnologiesjavaseembeddedarchivedownloadshtml_4">
     </a>
     1. 下载 ORacle 公司的ejdk-8u211-linux-arm-sflt.tar.gz（软件浮点版本，这个很重要，根据自己的板子硬件选。不知到就只有一个个试试） （下载地址：
     <a href="https://www.oracle.com/java/technologies/java-se-embedded-archive-downloads.html" rel="nofollow">
      link
     </a>
     ）
    </h3>
    <h3>
     <a id="2_documentation__quick_installation__5">
     </a>
     2. 按照documentation 的 quick installation 安装即可
    </h3>
    <p>
     参考
     <a href="https://docs.oracle.com/javase/8/embedded/develop-apps-platforms/plat-dev-quick-start.htm#BABJJIHA" rel="nofollow">
      link:2 Quick Start for Platform Developers
     </a>
    </p>
    <h3>
     <a id="3_tar_8">
     </a>
     3. 从tar包中提取软件：
    </h3>
    <pre><code> $ cd /usr
 $ gunzip *.gz
 $ tar -xvf *.tar
 List of unpacked files ...
</code></pre>
    <h3>
     <a id="4_ejdk_ejdkbin__16">
     </a>
     4. 验证安装（之前有一步配置ejdk的环境变量，其实可以不用进行） 转到ejdk的bin目录下，执行 ```
    </h3>
    <pre><code>bin/jrecreate.sh --help
$ cd $EJDK_HOME
$ bin/jrecreate.sh --help
Usage: jrecreate --help
</code></pre>
    <h3>
     <a id="5_linux_jre_Create_Your_JRE_with_jrecreatehttpsdocsoraclecomjavase8embeddeddevelopappsplatformsjrecreatehtmCACEFBDE_24">
     </a>
     5. 创建在嵌入式linux可用的 jre (
     <a href="https://docs.oracle.com/javase/8/embedded/develop-apps-platforms/jrecreate.htm#CACEFBDE" rel="nofollow">
      Create Your JRE with jrecreate
     </a>
     )
    </h3>
    <pre><code>使用bin目录下的jrecreate.sh 脚本创建，详细参数可见上述链接
参数的主要区别是 安装什么功能的java 环境，可以考虑自己的硬件资源，选择合适的参数。我发现完整版的jre大概35M左右，板子还是能用的，所以使用的是完整安装：
</code></pre>
    <pre><code>$ bin/jrecreate.sh --dest   (+你安装的路径/jre)   //这条命令会在你设置的目录下创建 jre 目录。也就是我们要的目录
// 显示如下：（具体参数的值根据你选则的版本和脚本执行参数不一样而不同）
Building JRE using options Options {
    ejdk-home: /home/xxxx/ejdk/ejdk&lt;version&gt;
    dest: /tmp/testjre
    target: linux_i586     
    vm: minimal    
    runtime: compact1 profile
    debug: false
    keep-debug-info: false
    no-compression: false
    dry-run: false
    verbose: false
    extension: []
}
</code></pre>
    <h3>
     <a id="6_jre_usr_etcprofile_44">
     </a>
     6. 将生成的jre 拷贝到你的嵌入式板子上，比如/usr/ 目录下。然后设置环境变量。在/etc/profile中添加：
    </h3>
    <pre><code>export  JAVA_HOME=/usr/jre/
export PATH=$PATH:${JAVA_HOME}/bin
</code></pre>
    <h3>
     <a id="7__49">
     </a>
     7. 验证是否安装成功：
    </h3>
    <pre><code>$ java -version   //若有提示信息输出则成功。
</code></pre>
    <h2>
     <a id="_53">
     </a>
     注意事项
    </h2>
    <h3>
     <a id="___Java_not_found_54">
     </a>
     验证安装 出现 Java ：not found
    </h3>
    <h4>
     <a id="1profile_source_55">
     </a>
     1.考虑环境变量是否配置正确（配置完profile 接的source一下，更新环境变量）
    </h4>
    <h4>
     <a id="2___pclinux_readelf_d__java_jrebinjavajava_java__56">
     </a>
     2. 考虑 共享库是否都有 在pc的linux上用 readelf -d java 查看jre/bin目录下的java文件，可以看到java运行需要的共享库，可以看见 java这个程序 需要：
    </h4>
    <p>
     <img alt=" java这个程序 需要的动态库" src="https://i-blog.csdnimg.cn/blog_migrate/335df227d2b82b9f33bed3fb4b97f4ed.png">
      自己取自己的arm linux 的根文件系统中的lib文件夹中找打需要的库拷贝到开发板jre/lib/arm下就可以了
     </img>
    </p>
    <h4>
     <a id="3ejdk__gcc_ejdk8u211linuxarmsflttargz_httpswwworaclecomjavatechnologiesjavaseembedded8u211downloadshtml_58">
     </a>
     3.确保第一步执行正确的话，考虑ejdk 版本（我之前为了在这个板子上搭环境，试了很多个版本，版本的区别主要是 硬件浮点和软件浮点，以及gcc的参数。最后发现我的板子适合软件浮点的版本 全名：ejdk-8u211-linux-arm-sflt.tar.gz
     <a href="https://www.oracle.com/java/technologies/javaseembedded8u211-downloads.html" rel="nofollow">
      下载地址
     </a>
     ）
    </h4>
    <p>
     <img alt="各个版本，这里只有3个其他版本有更多" src="https://i-blog.csdnimg.cn/blog_migrate/28c6b3bec979391d2ce2c32a29f6578c.png"/>
    </p>
    <h4>
     <a id="4_4412_60">

     4. 这里的选型只是参考，要根据你板子来选。不知道的话就只有像我一样一个个试，4412虽然有硬件浮点功能，可不知道怎么就是用不了硬件浮点的版本。希望有大神可以给我解答下。
