    <h3>
     前后端服务镜像制作与发布
    </h3>
    <h4>
     1、dockerFile内容编写
    </h4>
    <p>
     后端dockerFile内容
    </p>
    <pre><code class="language-bash">FROM java:8
#FROM openjdk:8-jdk

#RUN apt-get update &amp;&amp; apt-get install -y iputils-ping telnet

## 设置所属时区
ENV TZ=Asia/Shanghai
MAINTAINER hjy
COPY ./xxx.jar /xxx.jar
ENTRYPOINT ["java",  "-jar", "xxx.jar","--spring.profiles.active=prod"]</code></pre>
    <p>
     前端dockerFile内容
    </p>
    <pre><code class="language-bash"># 基础镜像使用Nginx
FROM nginx:1.24.0
# 添加时区环境变量，亚洲，上海
ENV TimeZone=Asia/Shanghai    
# 将前端dist文件中的内容复制到nginx目录
COPY ./dist/  /usr/share/nginx/html/actionSecret/
# 用本地的nginx配置文件覆盖镜像的Nginx配置
COPY ./nginx.conf /etc/nginx/nginx.conf
# 暴露端口
EXPOSE 80</code></pre>
    <h4>
     2、生成tar包
    </h4>
    <pre><code class="language-bash">docker build -t xxx .

docker save -o xxx.tar xxx</code></pre>
    <h4>
     3、发布tar包
    </h4>
    <pre><code class="language-bash">docker load &lt; xxx.tar

docker tag xxx:latest cr.registry.cloud.xxxxx.cn/ns-ali/xxx:v0.0.6

docker push cr.registry.cloud.xxxxx.cn/ns-ali/xxx:v0.0.6</code></pre>
    <h3>
     注意：
    </h3>
    <p>
     制作tar包时，dockerFile文件所在文件夹不要放置其他无关文件，因为文件夹内文件都会在打包范围之内。
    </p>
