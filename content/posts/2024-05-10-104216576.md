    <h2 style="text-indent:0px;">
     0.
     <span style="color:#000000;">
      <strong>
       团队存在形式
      </strong>
     </span>
    </h2>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      人
     </span>
     <span style="color:#000000;">
      (
     </span>
     <span style="color:#000000;">
      组织、团队
     </span>
     <span style="color:#000000;">
      )、流程、工具；
     </span>
     <span style="color:#000000;">
      人是第一要素；高质量的测试源于专业的测试团队；
     </span>
     <span style="color:#000000;">
      建立测试团队必须
     </span>
     <span style="color:#FF0000;">
      与测试流程关联
     </span>
     <span style="color:#000000;">
      起来；
     </span>
     <span style="color:#000000;">
      测试团队组织形式指测试团队的人员组织，以及测试团队与开发团队的关系。
     </span>
    </p>
    <h3 style="text-indent:0px;">
     （0）
     <strong>
      组织形式
     </strong>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      独立型测试团队、
     </span>
     <span style="color:#000000;">
      融合型非独立测试团队、
     </span>
     <span style="color:#000000;">
      资源池测试团队
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      独立测试团队：测试团队独立于开发团队，其优点为独立测试团队无偏见，能客观的看待被测对象；有利于测试团队的沟通交流、统一规范和管理；其不足为不利于测试和研发的沟通，以及测试早期对项目的了解；
     </span>
     <span style="color:#000000;">
      研发
     </span>
     <span style="color:#000000;">
      团队
     </span>
     <span style="color:#000000;">
      可能懈怠对质量的关注
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      融合型非独立测试团队：
     </span>
     <span style="color:#000000;">
      测试团队融合于
     </span>
     <span style="color:#000000;">
      研发团队、
     </span>
     <span style="color:#000000;">
      有利于沟通和管理、
     </span>
     <span style="color:#000000;">
      可能存在测试
     </span>
     <span style="color:#000000;">
      偏差
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      资源池形式：项目组成立时抽调测试和开发人员，项目结束则解散，人员回到原来的地方；
     </span>
     <span style="color:#000000;">
      人员
     </span>
     <span style="color:#000000;">
      配置灵活
     </span>
     ；
     <span style="color:#000000;">
      不利于
     </span>
     <span style="color:#000000;">
      管理
     </span>
     <span style="color:#000000;">
      (
     </span>
     <span style="color:#000000;">
      如人员考核问题)
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      实际
     </span>
     <span style="color:#000000;">
      企业在运作过程中要根据实际情况，有时情况更加复杂；
     </span>
     <span style="color:#000000;">
      比如
     </span>
     <span style="color:#000000;">
      有些企业在分模块开发时采用融合型的项目组形式，模块开发完毕项目组测试人员全部组成一个组，在系统测试期间采用独立型组织。
     </span>
    </p>
    <h2 style="text-indent:0px;">
     <span style="color:#000000;">
      1.
      <strong>
       团队创建
      </strong>
     </span>
    </h2>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定测试团队在组织中的位置及形式：独立测试团队、非独立测试团队、资源池
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定测试团队的规模：根据测试工作负荷、考虑人员备份
     </span>
     <span style="color:#000000;">
      和
     </span>
     <span style="color:#000000;">
      协作
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定组织中需要的测试
     </span>
     <span style="color:#000000;">
      类型
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定
     </span>
     <span style="color:#000000;">
      组织
     </span>
     <span style="color:#000000;">
      中需要的
     </span>
     <span style="color:#000000;">
      测试阶段
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定组织内部架构
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      确定测试团队角色
     </span>
     <span style="color:#000000;">
      配置
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      选择合适的测试人员
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定测试团队在组织中的位置及形式：
      </strong>
      确定测试团队的隶属关系，测试团队与开发团队之间的关系。测试团队的组织形式有三种：独立型测试团队、融合型非独立测试团队、资源池形式
     </span>
     <span style="color:#000000;">
      。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定测试团队的规模：
      </strong>
      测试
     </span>
     <span style="color:#000000;">
      团队的规模要根据测试的工作负荷来配置，同时考虑人员备份、人员层次、人员技能的需要。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定组织中需要的测试类型：
      </strong>
      软件产品的应用领域不同，测试类型也不尽相同，比如
     </span>
     <span style="color:#000000;">
      B/S
     </span>
     <span style="color:#000000;">
      架构的信息管理系统需要进行浏览器的兼容性测试但是不需要进行安装卸载测试，但是移动应用以及单机应用产品则需要进行安装卸载测试。组建测试团队时要分析自身业务需求，识别测试中的主要测试类型，并不断完善补充。 业务需要的测试类型是影响测试人员配置的重要因素。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定组织中需要的测试阶段以及测试流程：
      </strong>
      不同的组织有不同的测试阶段，
     </span>
     <span style="color:#000000;">
      V
     </span>
     <span style="color:#000000;">
      模型中包括了单元测试、模块测试、集成测试和系统测试，组建测试团队时要根据实际情况明确需要开展的测试阶段，并定义相关的执行流程。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定组织内部架构：
      </strong>
      确定测试团队内部的管理结构、汇报关系等。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       确定测试团队角色配置：
      </strong>
      明确测试团队需要配置哪些角色，定义角色职责，明确技能要求。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      <strong>
       选择合适的测试人员：
      </strong>
      根据需要选择合适的测试人员。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      配备测试人员的形式：基于技能的形式、基于项目的形式、
     </span>
     <span style="color:#000000;">
      混合形式
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      基于项目测试的需要为项目分配不同技能的测试人员。测试人员不必涉及多主题，注意力集中在自身专业领域。深入掌握本领域复杂的测试技术和测试工具。
     </span>
     <span style="color:#000000;">
      适用于测试难度比较大的项目。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      根据项目工作量为项目分配测试工程师，
     </span>
     <span style="color:#000000;">
      相对基于测试技能的形式，可以减少工作的中断和转换。
     </span>
    </p>
    <h3 style="text-indent:0px;">
     （0）
     <strong>
      角色配置
     </strong>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试经理：
     </span>
     <span style="color:#000000;">
      负责组建团队、资源调配、进度控制、方案选择等
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试工程师：资深测试工程师负责确定测试方案、
     </span>
     <span style="color:#000000;">
      设计测试用例、执行
     </span>
     <span style="color:#000000;">
      测试、报告测试
     </span>
     <span style="color:#000000;">
      结果
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试工具开发人员
     </span>
     <span style="color:#000000;">
      [
     </span>
     <span style="color:#000000;">
      可选]：
     </span>
     <span style="color:#000000;">
      测试工具的开发和维护
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      配置管理员：
     </span>
     <span style="color:#000000;">
      负责版本控制、配置管理
     </span>
     <span style="color:#000000;">
      等
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      IT管理员：
     </span>
     <span style="color:#000000;">
      负责测试环境搭建、
     </span>
     <span style="color:#000000;">
      测试
     </span>
     <span style="color:#000000;">
      工具
     </span>
     <span style="color:#000000;">
      的部署
     </span>
     <span style="color:#000000;">
      ，环境维护
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试人员基本素质要求：良好的沟通能力；具有适度的好奇心和怀疑精神；良好的学习能力：理解项目需求，理解产品；
     </span>
     <span style="color:#000000;">
      对
     </span>
     <span style="color:#000000;">
      质量忠诚，对测试有兴趣
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试人员专业技能要求：普适性专业技能：阅读理解、书面表达、统计分析；软件专业基础：操作系统、数据库、网络协议、软件工程等，至少掌握一门编程语言。软件测试知识：测试理论，测试方法等；被
     </span>
     <span style="color:#000000;">
      测产品领域知识：被测软件要解决的问题，相关业务；
     </span>
    </p>
    <h2 style="text-indent:0px;">
     <span style="color:#000000;">
      2.日常管理
     </span>
    </h2>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      测试团队组建完毕后，有测试团队负责人开展测试组织的日常管理工作。日常管理的主要内容为：任务分配和检查、组织架构维护、人员更新、沟通交流、考核评价、
     </span>
     <span style="color:#000000;">
      技能提升
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      任务分配和检查：日常
     </span>
     <span style="color:#000000;">
      测试任务的分配以及进度和质量检查。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      组织架构维护：根据
     </span>
     <span style="color:#000000;">
      测试需要及时调整测试团队内部组织架构，比如规模扩大后及时进行分解，开展二级管理。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      人员更新：形成
     </span>
     <span style="color:#000000;">
      人员淘汰更新的机制，保持团队的活力。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      沟通交流：确保
     </span>
     <span style="color:#000000;">
      测试团队内外部的沟通交流顺畅。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      考核评价：对
     </span>
     <span style="color:#000000;">
      测试人员进行考核评价，以及绩效改进。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#000000;">
      技能提升：组织
     </span>
     <span style="color:#000000;">
      开展培训，提升测试团队的业务技能
     </span>
    </p>
