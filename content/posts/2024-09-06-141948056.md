    <h2>

     一、关于
     <strong>
      JDK
     </strong>
    </h2>
    <h4>
     <a id="11__2">
     </a>
     1.1
     <strong>
      简介
     </strong>
    </h4>
    <blockquote>
     <p>
      Java是一种广泛使用的计算机编程语言，拥有跨平台、面向对象、泛型编程的特性，广泛应用于企业级Web应用开发和移动应用开发。
     </p>
    </blockquote>
    <p>
     JDK（Java Development Kit）是用于开发 Java 应用程序的工具包。它由以下几个主要部分组成：
    </p>
    <ol>
     <li>
      **Java 编译器（
      <strong>
       ​
      </strong>
      <code>
       javac
      </code>
      <strong>
       ​ ）
      </strong>
      ：将 Java 源代码文件（.java）编译成字节码文件（.class），这些字节码文件可以在 Java 虚拟机（JVM）上运行。
     </li>
     <li>
      <strong>
       Java 运行时环境（JRE）
      </strong>
      ：包括 JVM 和标准类库。JRE 使得 Java 程序能够在没有开发工具的情况下运行。
     </li>
     <li>
      <strong>
       标准类库
      </strong>
      ：提供了 Java 程序开发所需的基础类和工具，例如集合框架、输入输出操作、网络编程等。
     </li>
     <li>
      **Java 调试器（
      <strong>
       ​
      </strong>
      <code>
       jdb
      </code>
      <strong>
       ​ ）
      </strong>
      ：用于调试 Java 程序。
     </li>
     <li>
      **Java 文档生成工具（
      <strong>
       ​
      </strong>
      <code>
       javadoc
      </code>
      <strong>
       ​ ）
      </strong>
      ：自动生成 Java 类文档和 API 文档。
     </li>
     <li>
      <strong>
       Java 虚拟机（JVM）
      </strong>
      ：执行 Java 字节码，并提供了内存管理、垃圾回收等功能。
     </li>
    </ol>
    <p>
     JDK 是开发 Java 应用程序的基础工具包，它的不同版本（例如 JDK 8, JDK 11, JDK 17 等）可能包含不同的特性和改进。使用 JDK，开发者可以编写、编译、调试和运行 Java 程序。
    </p>
    <p>
     <img alt="f634e0b3-ea59-4c28-ba2b-12505568b0ae" src="https://img-blog.csdnimg.cn/img_convert/df33ffb74b699b57a47eddf2e100d8fc.jpeg">
      ​
     </img>
    </p>
    <h3>
     <a id="12_JDKJRE_19">
     </a>
     1.2 JDK
     <strong>
      和JRE的区别
     </strong>
    </h3>
    <p>
     JDK（Java Development Kit）和 JRE（Java Runtime Environment）都是 Java 环境的组成部分，但它们的功能和用途有所不同：
    </p>
    <p>
     <strong>
      JDK（Java Development Kit）
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        目的
       </strong>
       ：JDK 是一个开发工具包，提供了用于开发 Java 应用程序的全部工具和库。
      </p>
     </li>
     <li>
      <p>
       <strong>
        包含内容
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         JRE
        </strong>
        ：JDK 包含了 JRE，因此也包括了 JVM 和标准类库。
       </li>
       <li>
        <strong>
         编译器
        </strong>
        ：如
        <code>
         javac
        </code>
        ​，用于将 Java 源代码编译成字节码。
       </li>
       <li>
        <strong>
         调试工具
        </strong>
        ：如
        <code>
         jdb
        </code>
        ​，用于调试 Java 程序。
       </li>
       <li>
        <strong>
         文档生成工具
        </strong>
        ：如
        <code>
         javadoc
        </code>
        ​，用于生成 API 文档。
       </li>
       <li>
        <strong>
         其他开发工具
        </strong>
        ：如
        <code>
         javap
        </code>
        ​（类文件反汇编工具），
        <code>
         jar
        </code>
        ​（Java 打包工具）等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        用途
       </strong>
       ：主要用于开发 Java 程序，提供所有必需的工具来编写、编译和调试代码。
      </p>
     </li>
    </ul>
    <p>
     ‍
    </p>
    <p>
     <strong>
      JRE（Java Runtime Environment）
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        目的
       </strong>
       ：JRE 是一个运行时环境，提供了运行 Java 应用程序所需的环境。
      </p>
     </li>
     <li>
      <p>
       <strong>
        包含内容
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         JVM
        </strong>
        ：Java 虚拟机，用于执行 Java 字节码。
       </li>
       <li>
        <strong>
         标准类库
        </strong>
        ：提供了执行 Java 程序所需的基础类库（如集合框架、输入输出等）。
       </li>
       <li>
        <strong>
         支持文件
        </strong>
        ：如 Java 核心类库。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        用途
       </strong>
       ：主要用于运行已经编译好的 Java 应用程序。它不包括开发工具，因此不适合用于开发 Java 程序。
      </p>
     </li>
    </ul>
    <p>
     ‍
    </p>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       JDK
      </strong>
      ：包含 JRE 和开发工具，适用于开发者。
     </li>
     <li>
      <strong>
       JRE
      </strong>
      ：仅包含运行时环境，适用于普通用户只需运行 Java 程序的情况。
     </li>
    </ul>
    <p>
     如果你需要开发 Java 应用程序，你应该安装 JDK。如果你只需要运行 Java 应用程序，你只需要安装 JRE。
    </p>

    <h3>
     <a id="13_JDK_62">
     </a>
     1.3 JDK
     <strong>
      版本选择
     </strong>
    </h3>
    <p>
     选择适合的 JDK 版本取决于几个因素，比如项目需求、兼容性和长期支持。以下是一些考虑因素和建议：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        项目需求
       </strong>
       ：
      </p>
      <ul>
       <li>
        确认你的项目是否有特定的 JDK 版本要求。某些库或框架可能只支持特定版本的 JDK。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        长期支持（LTS）版本
       </strong>
       ：
      </p>
      <ul>
       <li>
        JDK 8、JDK 11 和 JDK 17 是 Oracle 和 OpenJDK 提供的长期支持版本。LTS 版本通常更稳定，适合用于生产环境。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        新特性
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果希望使用最新的 Java 特性，比如记录类型（record types）或模式匹配（pattern matching），可以选择最新的非 LTS 版本（如 JDK 20）。不过，非 LTS 版本的支持周期较短。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        兼容性
       </strong>
       ：
      </p>
      <ul>
       <li>
        确保你选择的 JDK 版本与已有的系统和工具链兼容。某些老旧系统可能不兼容最新的 JDK 版本。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        社区和工具支持
       </strong>
       ：
      </p>
      <ul>
       <li>
        选择一个社区支持良好的 JDK 版本可以保证你能获得及时的帮助和更新。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        性能和优化
       </strong>
       ：
      </p>
      <ul>
       <li>
        最新版本的 JDK 通常包含性能改进和优化，使用最新版本可以帮助提高应用程序的性能。
       </li>
      </ul>
     </li>
    </ol>
