      二  线性规划1
     </h2>
     <p>
      <strong>
       问题一：合理利用线材问题
      </strong>
      <br/>
      现在要做100套钢架，每套用长为2.9m，2.1m，1.5m的元钢各一根，已知原料长7.4m，问应该如何进行下料使用的原材料更小
      <br/>
      <img alt="" height="278" src="https://i-blog.csdnimg.cn/direct/33402ac266a44de3b4201a0934d65d0e.png" width="418">
       <br/>
       <br/>
       <br/>
       首先我们要知道我们这个题目明显是一个取最优解的问题，那么就是一个切割最优问题
       <br/>
       其次就要去找题目里面的未知量，找到未知量，才可以构建出模型
       <br/>
       <span style="color:#fe2c24">
        <strong>
         模型的确定是根据目标函数和约束条件确定的
        </strong>
       </span>
       <br/>
       <br/>
       <strong>
        为什么是线性规划
       </strong>
       <br/>
       我们需要确定如何从7.4米长的原料中切割出所需的2.9米、2.1米和1.5米的元钢，以最小化浪费的材料。这个问题可以表示为一个线性规划问题，因为：
      </img>
     </p>
     <ul>
      <li>
       <p>
        <span style="color:#fe2c24">
         <strong>
          目标函数是线性的
         </strong>
        </span>
        ：我们的目标是最小化使用的原料根数，即 x1​+x2​+x3​+x4​+x5​+x6​，这是一个线性函数。
       </p>
      </li>
      <li>
       <p>
        <span style="color:#fe2c24">
         <strong>
          约束条件是线性的
         </strong>
        </span>
        ：我们需要满足切割出的元钢总数至少为100根的条件，这些条件可以表示为线性不等式，例如 2.9*x1​+2.9*2x2​+2.9x5​≥100。
       </p>
      </li>
      <li>
       <p>
        <span style="color:#fe2c24">
         <strong>
          变量是非负的
         </strong>
        </span>
        ：切割的套数 xi​ 必须是非负整数。
       </p>
      </li>
     </ul>
     <p>

     <p>
      首先我们有这么多种的方案，每一个方案构建的总值都是小于7.4m的
      <br/>
      这里讲讲为什么不考虑使用2.9 2.1 1.5各自取一根不加？
      <br/>
      📌 结论
     </p>
     <ol>
      <li>
       <strong>
        单看浪费大小不够，要考虑整体优化
       </strong>
       。
      </li>
      <li>
       <strong>
        如果一个方案的浪费比其他方案都大，通常不会被选入最优解
       </strong>
       。
      </li>
      <li>
       <strong>
        有些方案即使浪费稍多，但可能是拼凑 100 套钢架的“必要补充”，可以加入
       </strong>
       。
      </li>
      <li>
       <strong>
        最好的方法是用整数规划（ILP）求解，让计算机自动决定是否要用某个方案
       </strong>
       。
      </li>
     </ol>
     <p>
      💡
      <strong>
       所以，加不加 6.5m 方案？可以加，但最终让计算机决定！
      </strong>
      🚀
      <br/>
      因为我们看倒数第二个，这个是已经到7.1了，之前都是7.4，7.3，7.2，这个是7.1所以加入，但是我们到第五种方案的时候已经到达了6.6，跨度很大，这个时候，我们就取这个，首先我们电脑是会自己判断这个方案取不取的，这个时候我们加上是为了避免电脑取最优解要用到，保险，这个时候，其实没有必要加这个方案六，每个都取一根，因为我们已经有一个保险的了
      <br/>
      <br/>
      即使你加了也没事，因为这个电脑可能不会选择，不考虑这个方案，我们这个题目是要减少浪费的
      <br/>
      <br/>
      那么我们要怎么判断是否要加上方案呢？以下是al分析，作者先记下来，方便下次复习看
      <br/>
      <strong>
       📌 原则 1：能否减少浪费？
      </strong>
     </p>

      </li>

       <strong>
        所有已知方案
       </strong>
       都多（例如浪费 0.9m），那它几乎不会被选入最优解。
      </li>
      <li>
       <strong>
        ✅ 选择浪费更少的方案，❌ 排除浪费更多的方案
       </strong>
       。
      </li>
     </ul>

      🔍
      <strong>
       如果新方案的浪费比已有方案大，基本就不会被选取
      </strong>
     </p>
     <p>
      <strong>
       📌 原则 2：能否帮助满足 100 套需求？
      </strong>
     </p>
     <p>
      即使方案本身浪费稍多，但如果它能让其他方案更好地拼接成 100 套，也可能有用！
     </p>
     <p>
      <strong>
       如何判断？
      </strong>
     </p>
     <ol>
      <li>
       <strong>
        尝试去掉某个方案
       </strong>
       ，看看是否还能刚好满足 100 套需求。
      </li>
      <li>
       <strong>
        如果去掉某个方案会导致解不可行
       </strong>
       ，说明它是必要的，即使它浪费稍多。
      </li>
      <li>
       <strong>
        如果所有方案能凑够 100 套，而某个方案总是没被选中，那它可以去掉
       </strong>
       。
      </li>
     </ol>
     <p>
      💡
      <strong>
       结论：如果一个方案不会被用到，或者可以被更优的方案替代，就不取！
      </strong>
     </p>
     <p>
      <strong>
       📌 实践方法：让 ILP 自动决定
      </strong>
     </p>
     <p>
      如果你不确定某个方案是否应该加入，可以
      <strong>
       让整数规划（ILP）自动决定
      </strong>
      ：
     </p>
     <ol>
      <li>
       <strong>
        先把所有可能的方案（包括 6.5m 方案）都放进去
       </strong>
       。
      </li>
      <li>
       <strong>
        让 ILP 计算最优解
       </strong>
       ，如果某个方案没有被选取，说明它不是最优的。
      </li>
      <li>
       <strong>
        查看最终结果，看看哪些方案真正被使用了
       </strong>
       。
      </li>
     </ol>
     <p>
      接下来我们就要把这个模型转换到这歌软件上进行操作
      <br/>
      <img alt="" height="591" src="https://i-blog.csdnimg.cn/direct/5b359dade74e43a2a742e13b24c17562.png" width="1635"/>
     </p>
     <p>
      接下来我们就要用到这个LINGO来编写
      <br/>
      首先这个sets:和endsets是表示定义一个aa集合，aa集合里面有x这个变量，然后这个1..5就是这个变量的下标
      <br/>
      <br/>
      然后这个min就是求解最小值，@sum表示求和，遍历集合aa的里的i，然后紧接着根据这个aa(i)遍历里面的变量
      <br/>
      <strong>
       也就是遍历里面aa里面的i，然后这个后面这个是aa集合里面的变量，随着者aa里的i进行改变
      </strong>
      <br/>
      下面就是一些约束条件了
      <br/>
      <br/>
      <code>
       @gin(x(i))
      </code>
      指定
      <code>
       x(i)
      </code>
      必须是
      <strong>
       整数变量
      </strong>
      ，然后for循环就是遍历这里面的变量，这些变量的值不可以是小数，而是整数
      <br/>
      <img alt="" height="535" src="https://i-blog.csdnimg.cn/direct/e05871dfd61b4733af4900d28ae98c48.png" width="1615"/>
     </p>

     <h2 id="%C2%A0%E4%B8%89%200-1%E8%A7%84%E5%88%92" name="%C2%A0%E4%B8%89%200-1%E8%A7%84%E5%88%92">
      三 0-1规划
     </h2>
     <p>
      在一个公司在市东南西三区建立门市部，有7个位置点(Ai,i=1.2.3...7)可供选择，规定：
      <br/>
      1）在东区，由A1 A2 A3三个点至多选择两个
      <br/>
      2）在西区，由A4 A5两个点至少选择一个
      <br/>
      3）在南区，由A6 A7两个点至少选择一个
      <br/>
      如果选用Ai点，设备投资估计为bi元，每年获利利润估计为ci元，但是投资总量不可以超过M元，问应该选择哪几个点建立门市部使得年利润最大
      <br/>
      <br/>
      首先这个就是典型的0-1问题，每一个点我们都有选择和不选择，1就是选择，0就是不选择
      <br/>
      那么我们就要考虑怎么选择就好了
      <br/>
      <img alt="" height="532" src="https://i-blog.csdnimg.cn/direct/37e7890435a5404b98a05a4fd79c2975.png" width="916"/>
     </p>
     <p>
      接下来我们就只需要编程就好了
     </p>
     <pre><code>sets:
 aa/1..7/:b,c,x;
endsets

data:
  c = 1,5,7,4,6,8,9; 
  b = 12,56,45,34,32,78,89;
  M = 200;
enddata

max = @sum(aa(i):c(i)*x(i));
x(1) + x(2) + x(3) &lt;= 2;
x(4) + x(5) &gt;=1;
x(6) + x(7) &gt;=1;
@sum(aa(i):b(i)*x(i)) &lt;= M;
@for(aa(i):@bin(x(i)));</code></pre>
     <p>
      <strong>
       1  for循环的错误使用
      </strong>
     </p>
     <pre><code class="hljs">@sum(@for(aa(i):b(i)*x(i))) &lt;= M;
</code></pre>
     <p>
      这样是不对的，sum里面已经隐式包括了相加的迭代，所以这么写会出现语法错误
     </p>
     <p>
      <br/>
      <strong>
       2  错误提示栏的报错
      </strong>
      <br/>

     </p>
     <p>
      这个通常是我么缺少了右括号才有的错误
     </p>
     <p>
      这里的bin函数是直接随机取值，然后转化为01，这样就可以运用到0-1规划
