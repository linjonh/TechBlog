    <h2 style="background-color:transparent">
     <strong>
      思维链（Chain of Thought, CoT）​
     </strong>
    </h2>
    <p>
     <strong>
      思维链（Chain of Thought, CoT）​
     </strong>
     是指让 AI 在回答问题时，像人类一样展示出一步步的推理过程，而不仅仅是直接给出最终答案。这种方法的目的是让答案更清晰、可信，尤其是解决复杂问题时，用户能看懂 AI 的逻辑。
    </p>
    <h3 style="background-color:transparent">
     简单理解：
    </h3>
    <p>
     就像做数学题时，老师要求你「写出计算过程」，而不是只写答案。CoT 就是让 AI 把「脑子里的思考过程」说出来。
    </p>
    <hr/>
    <h4>
     ​
     <strong>
      例子对比：​
     </strong>
    </h4>
    <h5>
     1. ​
     <strong>
      没有思维链（直接回答）：​
     </strong>
    </h5>
    <p>
     <strong>
      问题
     </strong>
     <br/>
     小明买了 5 个苹果和 5 根香蕉，苹果 3 元一个，香蕉 2 元一根，他一共花了多少钱？
    </p>
    <p>
     <strong>
      回答
     </strong>
     <br/>
     一共花了 25 元。
    </p>
    <p>
     <em>
      缺点
     </em>
     ：虽然答案正确，但不知道 AI 是蒙对的还是真的会算。
    </p>
    <hr/>
    <h5>
     2. ​
     <strong>
      有思维链（CoT）：​
     </strong>
    </h5>
    <p>
     <strong>
      问题
     </strong>
     <br/>
     小明买了 5 个苹果和 5 根香蕉，苹果 3 元一个，香蕉 2 元一根，他一共花了多少钱？请一步步思考。
    </p>
    <p>
     <strong>
      回答
     </strong>
    </p>
    <ol>
     <li>
      先算苹果的总价：5 个苹果 × 3 元/个 = 15 元；
     </li>
     <li>
      再算香蕉的总价：5 根香蕉 × 2 元/根 = 10 元；
     </li>
     <li>
      最后把两者相加：15 元 + 10 元 = 25 元。
      <br/>
      所以，小明一共花了 25 元。
     </li>
    </ol>
    <p>
     <em>
      优点
     </em>
     ：过程清晰，用户能验证每一步是否正确。
    </p>
    <hr/>
    <h4>
     ​
     <strong>
      另一个例子（非数学问题）：​
     </strong>
    </h4>
    <p>
     <strong>
      问题
     </strong>
     <br/>
     今天阴天，但天气预报说不会下雨，我要不要带伞？
    </p>
    <p>
     <strong>
      CoT 回答
     </strong>
    </p>
    <ol>
     <li>
      虽然阴天，但天气预报说不下雨，说明气象数据支持不带伞；
     </li>
     <li>
      但阴天可能有突发降雨，如果没带伞可能被淋湿；
     </li>
     <li>
      如果包里空间足够，带伞可以避免风险；
     </li>
     <li>
      如果不想麻烦，也可以不带。
      <br/>
      建议：如果出门时间长，可以带伞；如果短时间出门，可以不带。
     </li>
    </ol>
    <hr/>
    <h3 style="background-color:transparent">
     ​
     <strong>
      总结：​
     </strong>
    </h3>
    <p>
     CoT 就像让 AI「把作业本上的草稿纸展示给你看」，帮助用户理解它的逻辑。尤其适用于数学题、逻辑推理、复杂决策等问题。你可以通过在提问时加上「请一步步思考」或「解释你的推理过程」来触发 CoT 效果。
    </p>
    <p>
    </p>
    <h2 style="background-color:transparent">
     <strong>
      LTM 提示方法（Long-Term Memory）​
     </strong>
    </h2>
    <p>
     <strong>
      简单来说
     </strong>
     ：LTM 是让 AI 在回答问题时，能够主动调用「长期记忆」（比如之前的对话内容、外部知识库或用户提供的历史信息），从而生成更连贯、个性化的回答。
     <br/>
     <strong>
      核心目的
     </strong>
     ：让 AI 像人类一样记住上下文或背景知识，避免重复提问或回答脱节。
    </p>
    <hr/>
    <h3>
     ​
     <strong>
      通俗理解
     </strong>
    </h3>
    <p>
     假设你和一个朋友聊天：
    </p>
    <ul>
     <li>
      ​
      <strong>
       没有 LTM
      </strong>
      ：每次对话他都忘记之前聊过什么，每次都要你重复信息。
     </li>
     <li>
      ​
      <strong>
       有 LTM
      </strong>
      ：他能记住你们之前聊的话题、你的喜好或背景知识，回答更符合你的需求。
      <br/>
      LTM 提示就是通过设计问题或指令，让 AI 主动利用这些「记忆」。
     </li>
    </ul>
    <hr/>
    <h3>
     ​
     <strong>
      LTM 的两种常见场景
     </strong>
    </h3>
    <h4>
     1. ​
     <strong>
      对话中的上下文记忆（短期长期记忆）​
     </strong>
    </h4>
    <p>
     <strong>
      例子
     </strong>
     ：
    </p>
    <ul>
     <li>
      ​
      <strong>
       用户第一次提问
      </strong>
      ：
      <br/>
      “我想买一台笔记本电脑，预算 5000 元左右，主要用来办公。”
     </li>
     <li>
      ​
      <strong>
       后续提问
      </strong>
      ​（依赖 LTM 的效果）：
      <br/>
      “刚刚提到的预算内，有哪些推荐的型号？”
     </li>
    </ul>
    <p>
     <strong>
      好的 LTM 回答
     </strong>
     ：
     <br/>
     “根据您 5000 元的预算和办公需求，推荐联想小新系列或华为 MateBook D，它们续航强、轻薄便携。”
     <br/>
     <em>
      （AI 记住了「预算」和「用途」）
     </em>
    </p>
    <p>
     <strong>
      差的回答（无 LTM）​
     </strong>
     ：
     <br/>
     “请告诉我您的预算和用途。”
     <br/>
     <em>
      （重复提问，没有记忆）
     </em>
    </p>
    <hr/>
    <h4>
     2. ​
     <strong>
      结合外部知识库（长期知识记忆）​
     </strong>
    </h4>
    <p>
     <strong>
      例子
     </strong>
     ：
     <br/>
     <strong>
      问题
     </strong>
     ：
     <br/>
     “用 LTM 方法写一个关于登月计划的科幻故事，要求提到阿波罗 11 号的真实历史细节。”
    </p>
    <p>
     <strong>
      LTM 提示的回答
     </strong>
     ：
     <br/>
     “2060 年，人类在月球建立了永久基地。基地的中央广场立着一块金属牌，上面刻着：​**‘1969 年 7 月 20 日，阿波罗 11 号宇航员首次踏上月球’**。此刻，新一代探险家正站在牌子前，准备向火星出发……”
     <br/>
     <em>
      （AI 调用了阿波罗 11 号的真实历史知识）
     </em>
    </p>
    <p>
     <strong>
      无 LTM 的回答
     </strong>
     ：
     <br/>
     “未来人类在月球上建立了基地，他们探索宇宙……”
     <br/>
     <em>
      （缺乏真实历史细节）
     </em>
    </p>
    <hr/>
    <h4>
     ​
     <strong>
      如何触发 LTM 效果？
     </strong>
    </h4>
    <p>
     在提问时明确要求 AI 调用记忆，例如：
    </p>
    <ol>
     <li>
      ​
      <strong>
       关联上下文
      </strong>
      ：
      <br/>
      “根据我们之前的讨论，你推荐的笔记本电脑是否支持扩展内存？”
     </li>
     <li>
      ​
      <strong>
       指定知识范围
      </strong>
      ：
      <br/>
      “参考《三国演义》中赤壁之战的过程，分析孙刘联盟胜利的原因。”
     </li>
     <li>
      ​
      <strong>
       提供背景信息
      </strong>
      ：
      <br/>
      “我之前喜欢科幻和悬疑小说，请推荐类似的新书。”
     </li>
    </ol>
    <hr/>
    <h3 style="background-color:transparent">
     ​
     <strong>
      总结
     </strong>
    </h3>
    <p>
     LTM 的核心是让 AI ​
     <strong>
      记住并利用已有信息
     </strong>
     ，而不是每次都从零开始。适用于：
    </p>
    <ul>
     <li>
      多轮对话（如客服、个性化推荐）
     </li>
     <li>
      需要结合专业知识的问题（如历史、科学）
     </li>
     <li>
      创作类任务（如故事、文案需引用特定资料）
     </li>
    </ul>
    <p>
     通过清晰的提示词（例如“参考之前的信息”“结合某某知识”），可以更好地激活 AI 的 LTM 能力。
    </p>
    <p>
    </p>
    <p>
    </p>
    <h2 style="background-color:transparent">
     ​
     <strong>
      思维树（Tree of Thoughts, ToT）​
     </strong>
    </h2>
    <p>
     <strong>
      简单来说
     </strong>
     ：ToT 是一种让 AI 像人类一样 ​
     <strong>
      多角度、分步骤探索问题
     </strong>
     的提示方法。它要求 AI 将复杂问题拆解成多个可能的解决路径（像树枝分叉），逐一分析后选择最优解。
     <br/>
     <strong>
      核心目的
     </strong>
     ：解决单一思维链（CoT）可能遗漏潜在答案的问题，提升答案的全面性和可靠性。
    </p>
    <hr/>
    <h3>
     ​
     <strong>
      通俗理解
     </strong>
    </h3>
    <p>
     假设你要解决一道迷宫题：
    </p>
    <ul>
     <li>
      ​
      <strong>
       用思维链（CoT）​
      </strong>
      ：你会沿着一条路走到底，如果走不通就失败。
     </li>
     <li>
      ​
      <strong>
       用思维树（ToT）​
      </strong>
      ：你会先在脑海中画出迷宫地图，标记所有可能的岔路，再选择最合理的路径。
      <br/>
      ToT 就是让 AI 在脑海中生成多个“解题路线图”，并选出最佳方案。
     </li>
    </ul>
    <hr/>
    <h3 style="background-color:transparent">
     ​
     <strong>
      例子对比（ToT vs CoT）​
     </strong>
    </h3>
    <h5>
     ​
     <strong>
      问题
     </strong>
    </h5>
    <p>
     “小明有 15 元，需要买 3 支笔和 2 个笔记本。笔单价 2 元，笔记本单价 4 元。他钱够吗？如果不够，怎么调整数量？”
    </p>
    <hr/>
    <h4>
     ​
     <strong>
      1. 用思维链（CoT）回答
     </strong>
    </h4>
    <ol>
     <li>
      计算总花费：3×2 + 2×4 = 6 + 8 = 14 元；
     </li>
     <li>
      比较金额：15 元 &gt; 14 元，所以够用。
      <br/>
      <strong>
       结果
      </strong>
      ：直接给出答案，但忽略了问题中“如果不够，怎么调整”的可能性分析。
     </li>
    </ol>
    <hr/>
    <h4>
     ​
     <strong>
      2. 用思维树（ToT）回答
     </strong>
    </h4>
    <p>
     <strong>
      步骤 1：拆解问题分支
     </strong>
    </p>
    <ul>
     <li>
      分支 1：当前购买方案是否超支？
     </li>
     <li>
      分支 2：如果超支，如何减少数量？
      <ul>
       <li>
        子分支 2.1：减少 1 支笔 → 总价 2×2 + 8 = 12 元
       </li>
       <li>
        子分支 2.2：减少 1 个笔记本 → 总价 6 + 4 = 10 元
       </li>
      </ul>
     </li>
     <li>
      分支 3：如果钱有剩余，能否多买？
      <ul>
       <li>
        子分支 3.1：多买 1 支笔 → 剩余 15 - (14+2) = -1 元（不可行）
       </li>
       <li>
        子分支 3.2：多买 1 个笔记本 → 剩余 15 - (14+4) = -3 元（不可行）
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      步骤 2：评估每个分支
     </strong>
    </p>
    <ul>
     <li>
      分支 1：当前方案花费 14 元，15 元足够，剩余 1 元。
     </li>
     <li>
      分支 2：无需调整，但可建议优化（例如用剩余钱买橡皮）。
     </li>
     <li>
      分支 3：剩余钱不足以多买。
     </li>
    </ul>
    <p>
     <strong>
      最终回答
     </strong>
     <br/>
     当前方案够用，剩余 1 元。如果想调整，可减少 1 支笔（总价 12 元）并多买一块橡皮（1 元），或保持原计划。
    </p>
    <hr/>
    <h3>
     ​
     <strong>
      另一个例子（非数学问题）​
     </strong>
    </h3>
    <p>
     <strong>
      问题
     </strong>
     <br/>
     “我想周末去郊游，但可能下雨，有什么建议？”
    </p>
    <p>
     <strong>
      ToT 回答
     </strong>
    </p>
    <ol>
     <li>
      ​
      <strong>
       分支 1：晴天方案
      </strong>
      <ul>
       <li>
        爬山、野餐、拍照，需准备防晒用品。
       </li>
      </ul>
     </li>
     <li>
      ​
      <strong>
       分支 2：雨天方案
      </strong>
      <ul>
       <li>
        改为室内博物馆或咖啡馆，带伞和防水鞋。
       </li>
      </ul>
     </li>
     <li>
      ​
      <strong>
       分支 3：折中方案
      </strong>
      <ul>
       <li>
        选择有遮阳棚的露营地，下雨可避雨，晴天可活动。
        <br/>
        <strong>
         最终建议
        </strong>
        ：提前看天气预报，若不确定，优先选择分支 3 的露营地。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     ​
     <strong>
      如何触发 ToT 效果？
     </strong>
    </h3>
    <p>
     在提问时明确要求 AI ​
     <strong>
      分步骤探索可能性
     </strong>
     ，例如：
    </p>
    <ul>
     <li>
      “请列出所有解决方案并分析优缺点。”
     </li>
     <li>
      “分步骤思考：1. 问题拆解，2. 可能性分析，3. 最优选择。”
     </li>
     <li>
      “如果遇到 X 情况，可能有哪些应对方式？请逐一评估。”
     </li>
    </ul>
    <hr/>
    <h3>
     ​
     <strong>
      ToT 的适用场景
     </strong>
    </h3>
    <ol>
     <li>
      ​
      <strong>
       开放式问题
      </strong>
      ​（如策划、决策、创意）。
     </li>
     <li>
      ​
      <strong>
       存在多种解法的问题
      </strong>
      ​（如数学题、逻辑推理）。
     </li>
     <li>
      ​
      <strong>
       需要考虑风险和备选方案的问题
      </strong>
      ​（如旅行计划、投资建议）。
     </li>
    </ol>
    <hr/>
    <h3 style="background-color:transparent">
     ​
     <strong>
      总结
     </strong>
    </h3>
    <ul>
     <li>
      ​
      <strong>
       ToT 像解题时的“头脑风暴”​
      </strong>
      ，先穷举可能性，再筛选最优解。
     </li>
     <li>
      ​
      <strong>
       对比 CoT（直线思维）​
      </strong>
      ：ToT 更适合复杂、多路径问题，避免遗漏潜在答案。
     </li>
     <li>
      ​
      <strong>
       缺点
      </strong>
      ：效率较低，简单问题直接用 CoT 更高效。
     </li>
    </ul>
