    <h2>
     <a id="_1">
     </a>
     环境准备
    </h2>
    <h3>
     <a id="Ollama_3">
     </a>
     下载Ollama
    </h3>
    <p>
     下载地址：
     <a href="https://ollama.com/download" rel="nofollow">
      Ollama网址
     </a>
    </p>
    <p>
     安装完成后，命令行里执行命令
    </p>
    <pre><code>ollama -v
</code></pre>
    <p>
     查看是否安装成功。安装成功会显示版本信息
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/df5e589f2f014145be4b6fe2d6258922.png#pic_center"/>
    </p>
    <p>
     ollama 的命令可通过
     <code>
      ollama -h
     </code>
     查看。
    </p>
    <h3>
     <a id="_20">
     </a>
     模型下载
    </h3>
    <p>
     可以在 Ollama 网站的 Models 里查看公开的大模型（
     <a href="https://ollama.com/search" rel="nofollow">
      网址
     </a>
     ），也可以从大模型镜像源站
     <a href="https://hf-mirror.com/" rel="nofollow">
      HF-Mirror
     </a>
     下载。
    </p>
    <p>
     以 Ollama 网站为例，点进一个大模型（如最近大火的 DeepSeek R1）
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/94c143a8d9794aad9158b01414ac5f61.png#pic_center"/>
    </p>
    <p>
     如图右下角的
     <code>
      ollama run deepseek-r1
     </code>
     就是下载命令，在第一次执行该命令时，ollama 将从网站下载大模型，在下载完成后，再执行这一命令就会加载模型，并进入交互模式：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fded0945de4547d88ed7ed5771c3a348.png#pic_center"/>
    </p>
    <p>
     <strong>
      注
     </strong>
     ：大部分模型都是有
     <strong>
      内容审查
     </strong>
     的，这很多时候限制了我们的使用灵活性，可以寻找带有 abliterated 后缀的模型，这些模型被注释了审查代码。（不过本人尝试发现 abliterated 版本的实际上仍存在审查机制，会拒绝回答某些问题）
    </p>
    <h3>
     <a id="OpenWebUI_36">
     </a>
     下载Open-WebUI
    </h3>
    <p>
     网上其他教程安装 Open-WebUI 一般都是在虚拟机 Docker 下安装的，这在 Windows 系统里很不方便。这里提供另外一种方法：首先安装 Python 3.11，然后在命令行里执行
    </p>
    <pre><code>pip install open-webui
</code></pre>
    <p>
     即可完成 open-webui 的安装。
    </p>
    <h2>
     <a id="Web_46">
     </a>
     本地化部署的Web图形化界面
    </h2>
    <p>
     首先，需要开启 Ollama 服务，运行一个大模型，在命令行里执行
    </p>
    <pre><code>ollama run 大模型名
</code></pre>
    <p>
     其中模型名可以通过
     <code>
      ollama list
     </code>
     查看。启用 ollama 服务后，可以使用
    </p>
    <pre><code>ollama ps
</code></pre>
    <p>
     命令查看当前运行的模型进程。
    </p>
    <p>
     随后再打开一个命令行，执行如下命令启用 open-webui 服务
    </p>
    <pre><code>open-webui serve
</code></pre>
    <p>
     启动后可以在浏览器里输入以下地址，打开 Web 图形化界面：
    </p>
    <pre><code>localhost:8080
(本机IP):8080
</code></pre>
    <p>
     其中
     <strong>
      第二个地址可以在局域网内的其他设备访问
     </strong>
     。
    </p>
    <p>
     Open-WebUI 的界面如下
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/984469a9a96c49fcafe6d9b4c7af2ffc.png#pic_center"/>
    </p>
    <p>
     本地化部署完成！
    </p>
    <h2>
     <a id="_84">
     </a>
     本地模型联网查询
    </h2>
    <p>
     未联网的情况下，很多问题大模型无法解决，回答也比较蠢（还会胡说），因此需要增加联网查询功能。
    </p>
    <h3>
     <a id="_Docker_88">
     </a>
     安装 Docker
    </h3>
    <p>
     安装 Docker Desktop 即可（网址
     <a href="https://www.docker.com/" rel="nofollow">
      Docker
     </a>
     ）。安装后用以下命令查询是否安装完成：
    </p>
    <pre><code>docker -v
</code></pre>
    <h3>
     <a id="_SearXNG_96">
     </a>
     安装 SearXNG
    </h3>
    <p>
     执行如下命令拉取 SearXNG （一个可以本地部署的轻量化搜索引擎）
    </p>
    <pre><code>docker pull searxng/searxng
</code></pre>
    <p>
     不过这个命令由于网站无法连接，下载往往失败，可以使用镜像源：
    </p>
    <pre><code>docker pull docker.m.daocloud.io/searxng/searxng
</code></pre>
    <p>
     安装完成后即可在 Docker Desktop 里查看到 SearXNG Image：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/79fc65a775274ad39b45898e6c0e8d5a.png#pic_center"/>
    </p>
    <p>
     执行如下命令（最后面一串是 Image Name，注意要和你的 SearXNG 名字相同），将服务开在了端口 12345
    </p>
    <pre><code>docker run -d -p 12345:8080 docker.m.daocloud.io/searxng/searxng
</code></pre>
    <p>
     随后可以在 Docker Desktop 的 Container/App 界面查看到运行的 SearXNG，显示运行在 12345 号端口。在命令行里使用
    </p>
    <pre><code>docker ps
</code></pre>
    <p>
     命令也可以查看到运行中的 docker 镜像进程。
    </p>
    <p>
     在浏览器访问 localhost:12345，可以看到 SearXNG 服务已开启。
    </p>
    <p>
     <img alt="在这里插入图片描述" height="200" src="https://i-blog.csdnimg.cn/direct/2eb09bc599ba438a93ed0f00bb41e22a.png#pic_center" width="600"/>
    </p>
    <p>
     用默认配置的 SearXNG 很可能搜索不到东西，可以在配置里修改搜索引擎，改成大陆可以访问的那几个。
    </p>
    <h3>
     <a id="_136">
     </a>
     本地模型联网查询
    </h3>
    <p>
     在 open-webui 界面下，用户-&gt;管理员面板-&gt;设置 里面，配置搜索引擎如下，并保存设置
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5dfff535a1bc48568217aa2d810d89bf.png#pic_center"/>
    </p>
    <p>
     此时询问问题，联网查询可能会报 403 错误，需要修改一个配置文件。执行
    </p>
    <pre><code>docker ps
</code></pre>
    <p>
     查看 searxng 服务的 CONTAINER ID 号，随后执行以下命令（命令中的 &lt;CONTAINER_ID&gt; 替换为你 SearXNG 的 CONTAINER ID）
    </p>
    <pre><code>docker exec -it &lt;CONTAINER_ID&gt; sh
</code></pre>
    <p>
     进入到镜像目录，随后进入
     <code>
      /etc/searxng/
     </code>
     目录，使用 vi 修改 settings.yml 文件，在 formats 下面增加一行（- json）
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d4f7bd24d1684c66b6f1e3e611da1a19.png#pic_center"/>
    </p>
    <p>
     修改文件后保存，然后 restart 容器。
    </p>
    <hr/>
    <p>
     修改 settings.yml 后，open-webui 联网查询不再报 403 错误，但可能报
     <code>
      Expecting value: line 1 column 1 (char 0)
     </code>
     错误（这是因为返回值格式不满足 json 格式导致的解析错误）或
     <code>
      RemoteDisconnected('Remote end closed connection without response')
     </code>
     错误，笔者目前还没有找到好的解决方法，日后若解决了再补上这里。
    </p>
