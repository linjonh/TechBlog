    <h2>
     <a id="Docker_0">
     </a>
     Docker的常用镜像命令主要包括镜像的查看、搜索、拉取、删除、构建等操作，以下是综合多个来源的总结：
    </h2>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     <strong>
      一、基础镜像操作
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        查看本地镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> images
</code></pre>
      <p>
       • 显示所有本地镜像，包含仓库名（REPOSITORY）、标签（TAG）、镜像ID、创建时间及大小。
       <br/>
       • 常用选项：
       <br/>
       ◦
       <code>
        -a
       </code>
       ：显示所有镜像（包括中间层镜像）
       <br/>
       ◦
       <code>
        -q
       </code>
       ：仅显示镜像ID
      </p>
     </li>
     <li>
      <p>
       <strong>
        搜索公共镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> search <span class="token operator">&lt;</span>镜像名<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 从Docker Hub搜索镜像，支持筛选：
       <br/>
       ◦
       <code>
        --filter "is-official=true"
       </code>
       ：仅显示官方镜像
       <br/>
       ◦
       <code>
        -s &lt;星数&gt;
       </code>
       ：按星数过滤（如
       <code>
        -s 30
       </code>
       显示星数≥30的镜像）
      </p>
     </li>
     <li>
      <p>
       <strong>
        拉取镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> pull <span class="token operator">&lt;</span>镜像名<span class="token operator">&gt;</span>:<span class="token operator">&lt;</span>标签<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 默认从Docker Hub下载镜像，未指定标签时自动拉取
       <code>
        latest
       </code>
       版本。
       <br/>
       • 示例：
       <code>
        docker pull ubuntu:20.04
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        删除镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> rmi <span class="token operator">&lt;</span>镜像名/ID<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 删除单个镜像，
       <code>
        -f
       </code>
       强制删除（如正在被容器使用）。
       <br/>
       • 批量删除：
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> rmi <span class="token parameter variable">-f</span> <span class="token variable"><span class="token variable">$(</span><span class="token function">docker</span> images <span class="token parameter variable">-aq</span><span class="token variable">)</span></span>  <span class="token comment"># 删除所有镜像</span>
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_41">
     </a>
     <strong>
      二、镜像构建与推送
     </strong>
    </h4>
    <ol start="5">
     <li>
      <p>
       <strong>
        构建自定义镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> build <span class="token parameter variable">-t</span> <span class="token operator">&lt;</span>镜像名<span class="token operator">&gt;</span>:<span class="token operator">&lt;</span>标签<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>上下文路径<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 通过Dockerfile构建镜像，
       <code>
        -t
       </code>
       指定名称和标签，
       <code>
        -f
       </code>
       指定非默认Dockerfile路径。
      </p>
     </li>
     <li>
      <p>
       <strong>
        推送镜像到仓库
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> push <span class="token operator">&lt;</span>镜像名<span class="token operator">&gt;</span>:<span class="token operator">&lt;</span>标签<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 需先登录镜像仓库（
       <code>
        docker login &lt;仓库地址&gt;
       </code>
       ）。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_56">
     </a>
     <strong>
      三、镜像维护与清理
     </strong>
    </h4>
    <ol start="7">
     <li>
      <p>
       <strong>
        清理未使用的镜像
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> image prune
</code></pre>
      <p>
       • 删除未被容器引用的镜像，
       <code>
        -a
       </code>
       删除所有未使用的镜像。
      </p>
     </li>
     <li>
      <p>
       <strong>
        查看镜像元数据
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> inspect <span class="token operator">&lt;</span>镜像名/ID<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 显示镜像的详细配置信息，包括层结构、环境变量等。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_71">
     </a>
     <strong>
      四、其他实用命令
     </strong>
    </h4>
    <ol start="9">
     <li>
      <p>
       <strong>
        登录镜像仓库
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> login <span class="token operator">&lt;</span>仓库地址<span class="token operator">&gt;</span>
</code></pre>
      <p>
       • 访问私有仓库或第三方仓库（如阿里云、腾讯云）时需先登录。
      </p>
     </li>
     <li>
      <p>
       <strong>
        镜像标签管理
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">docker</span> tag <span class="token operator">&lt;</span>源镜像名<span class="token operator">&gt;</span>:<span class="token operator">&lt;</span>标签<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>新镜像名<span class="token operator">&gt;</span>:<span class="token operator">&lt;</span>新标签<span class="token operator">&gt;</span>
</code></pre>
      <p>
       ◦ 为镜像创建新标签，常用于版本管理或推送至不同仓库。
      </p>
     </li>
    </ol>
    <hr/>

    <h2>
     <a id="_Docker__137">
     </a>
     以下是 Docker 常用容器命令的总结，涵盖容器的生命周期管理、信息查看、交互操作等核心场景：
    </h2>
    <hr/>
    <h4>
     <a id="_141">
     </a>
     <strong>
      一、容器生命周期管理
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        启动容器
       </strong>
       <br/>
       • 基本启动：
       <code>
        docker run [选项] 镜像名
       </code>
       <br/>
       示例：
       <code>
        docker run -d --name my_nginx nginx
       </code>
       （后台运行并命名容器）
       <br/>
       • 常用选项：
       <br/>
       ◦
       <code>
        -d
       </code>
       ：后台运行（守护模式）
       <br/>
       ◦
       <code>
        -p 宿主机端口:容器端口
       </code>
       ：端口映射（如
       <code>
        -p 8080:80
       </code>
       ）
       <br/>
       ◦
       <code>
        -v 宿主机路径:容器路径
       </code>
       ：数据卷挂载（如
       <code>
        -v /data:/app/data
       </code>
       ）
       <br/>
       ◦
       <code>
        -e 环境变量
       </code>
       ：设置容器内环境变量（如
       <code>
        -e MYSQL_ROOT_PASSWORD=123456
       </code>
       ）
      </p>
     </li>
     <li>
      <p>
       <strong>
        启停与重启
       </strong>
       <br/>
       • 启动已停止的容器：
       <code>
        docker start 容器名/ID
       </code>
       <br/>
       • 停止容器：
       <code>
        docker stop 容器名/ID
       </code>
       （优雅停止）
       <br/>
       • 强制停止：
       <code>
        docker kill 容器名/ID
       </code>
       <br/>
       • 重启容器：
       <code>
        docker restart 容器名/ID
       </code>
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_159">
     </a>
     <strong>
      二、容器信息查看
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        查看容器列表
       </strong>
       <br/>
       • 运行中的容器：
       <code>
        docker ps
       </code>
       <br/>
       • 所有容器（含已停止的）：
       <code>
        docker ps -a
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        查看容器详情
       </strong>
       <br/>
       • 详细信息：
       <code>
        docker inspect 容器名/ID
       </code>
       （包含网络、卷等配置）
       <br/>
       • 实时资源占用：
       <code>
        docker stats 容器名/ID
       </code>
       （CPU/内存监控）
      </p>
     </li>
     <li>
      <p>
       <strong>
        查看日志
       </strong>
       <br/>
       • 基础日志：
       <code>
        docker logs 容器名/ID
       </code>
       <br/>
       • 实时跟踪日志：
       <code>
        docker logs -f 容器名/ID
       </code>
       <br/>
       • 显示最后 N 行：
       <code>
        docker logs --tail 20 容器名/ID
       </code>
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_175">
     </a>
     <strong>
      三、容器交互与维护
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        进入容器终端
       </strong>
       <br/>
       • 交互式终端：
       <code>
        docker exec -it 容器名/ID /bin/bash
       </code>
       （常用
       <code>
        /bin/bash
       </code>
       或
       <code>
        /bin/sh
       </code>
       ）
       <br/>
       • 执行单条命令：
       <code>
        docker exec 容器名/ID ls /app
       </code>
       （不进入终端直接执行）
      </p>
     </li>
     <li>
      <p>
       <strong>
        删除容器
       </strong>
       <br/>
       • 删除已停止的容器：
       <code>
        docker rm 容器名/ID
       </code>
       <br/>
       • 强制删除运行中的容器：
       <code>
        docker rm -f 容器名/ID
       </code>
       <br/>
       • 清理所有已停止的容器：
       <code>
        docker container prune
       </code>
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_187">
     </a>
     <strong>
      四、高级操作
     </strong>
    </h4>
    <p>
     •
     <strong>
      数据卷挂载
     </strong>
     ：
     <br/>
     使用
     <code>
      -v
     </code>
     参数持久化数据，如
     <code>
      docker run -v /host/data:/container/data
     </code>
     <br/>
     •
     <strong>
      网络配置
     </strong>
     ：
     <br/>
     • 查看容器网络：
     <code>
      docker network inspect 网络名
     </code>
     <br/>
     • 连接自定义网络：
     <code>
      docker network connect 网络名 容器名
     </code>
     <br/>
     •
     <strong>
      导出与提交
     </strong>
     ：
     <br/>
     • 导出容器文件系统：
     <code>
      docker export 容器名 &gt; backup.tar
     </code>
     <br/>
     • 保存容器为镜像：
     <code>
      docker commit 容器名 新镜像名
     </code>
    </p>
