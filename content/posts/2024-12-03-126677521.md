    <h2>
     一、网络安全审计概述
    </h2>
    <h3>
     1.1 网络安全审计概念
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       网络安全审计：
      </strong>
     </span>
     指对网络信息系统的
     <span style="color:#ad720d;">
      安全相关活动信息进行获取、记录、存储、分析和利用的工作
     </span>
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       网络安全审计作用：
      </strong>
     </span>
     在于
     <span style="color:#ad720d;">
      建立“事后”安全保障措施
     </span>
     ，保存网络安全事件及行为信息，为网络安全事件分析
     <span style="color:#ad720d;">
      提供线索及证据，以便于发现潜在的网络安全威胁行为，开展网络安全风险分析及管理
     </span>
    </p>
    <p>
     目前，IT产品和安全设备都不同程度地提供安全审计功能。常见的
     <span style="color:#1c7892;">
      <strong>
       安全审计功能
      </strong>
     </span>
     是
     <strong>
      安全事件采集
     </strong>
     、
     <strong>
      存储
     </strong>
     和
     <strong>
      查询
     </strong>
     。对于重要的信息系统，则
     <span style="color:#ad720d;">
      部署独立的网络安全审计系统
     </span>
    </p>
    <h3>
     1.2 网络安全审计相关标准
    </h3>
    <p>
     1985年美国《可信计算机系统评估标准》(TCSEC) 中给出了计算机系统的
     <strong>
      <span style="color:#4da8ee;">
       安全审计要求：
      </span>
     </strong>
     <span style="color:#1c7892;">
      <strong>
       TCSEC
      </strong>
     </span>
     <span style="color:#ad720d;">
      <strong>
       从
      </strong>
     </span>
     <span style="color:#1a439c;">
      <strong>
       C2级
      </strong>
     </span>
     <span style="color:#ad720d;">
      <strong>
       开始提出了安全审计的要求
      </strong>
     </span>
     ，随着保护级别的增加而逐渐加强，
     <span style="color:#ad720d;">
      <strong>
       B3级以及之后更高
      </strong>
      的级别则不再变化
     </span>
    </p>
    <p>
     国标GB 17859《计算机信息系统安全保护等级划分准则》
     <span style="color:#ad720d;">
      <strong>
       从
      </strong>
     </span>
     <span style="color:#1a439c;">
      <strong>
       第二级
      </strong>
     </span>
     <span style="color:#ad720d;">
      <strong>
       开始要求提供审计安全机制
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       第二级：
      </strong>
     </span>
     为系统审计保护级，该级要求计算机信息系统可信计算基实施了粒度更细的自主访问控制，它
     <span style="color:#ad720d;">
      通过登录规程、审计安全性相关事件和隔离资源，使用户对自己的行为负责
     </span>
     。《准则》 中
     <strong>
      明确了各级别对审计的要求
     </strong>
     ，如表所示
    </p>
    <table border="1" cellpadding="1" cellspacing="1" style="width:500px;">
     <tbody>

        </strong>
       </td>
      </tr>

        <span style="color:#1c7892;">
         <strong>
          一级
         </strong>
        </span>

       <td style="width:73px;">
        用户自主保护级
       </td>
       <td style="width:375px;">
        无

       <td style="width:375px;">
        <p>
         计算机信息系统可信计算基
         <strong>
          <span style="color:#1c7892;">
           能创建和维护受保护客体的访问审计跟踪记录，并能阻止非授权的用户对它访问或破坏
          </span>
         </strong>
        </p>
        <p>
         计算机信息系统可信计算基
         <span style="color:#1c7892;">
          <strong>
           能记录下述事件：
          </strong>
         </span>
        </p>
        <ol>
         <li>
          使用身份鉴别机制
         </li>
         <li>
          将客体引入用户地址空间(例如:打开文件、程序初始化)
         </li>
         <li>
          删除客体
         </li>
         <li>
          由操作员、系统管理员或(和)系统安全管理员实施的动作，以及其他与系统安全有关的事件
         </li>
        </ol>
        <p>
         ①对于
         <span style="color:#1c7892;">
          <strong>
           每一事件
          </strong>
         </span>
         ，
         <strong>
          审计记录包括：
         </strong>
         事件的日期和时间、用户、事件类型、事件是否成功
        </p>
        <p>
         ②对
         <span style="color:#1c7892;">
          <strong>
           身份鉴别事件
          </strong>
         </span>
         ，
         <strong>
          审计记录包含：
         </strong>
         请求的来源(如终端标识符)
        </p>
        <p>
         ③对
         <span style="color:#1c7892;">
          <strong>
           客体引入用户地址空间的事件
          </strong>
          及
         </span>
         <strong>
          <span style="color:#1c7892;">
           客体删除事件
          </span>
          ，审计记录包含：
         </strong>
         客体名
        </p>
        <p>
         ④对
         <span style="color:#1c7892;">
          <strong>
           不能由计算机信息系统可信计算基独立分辨的审计事件
          </strong>
         </span>
         ，
         <strong>
          审计机制提供
         </strong>
         审计记录接口，可由授权主体调用
        </p>
        <p>
         这些审计记录区别于计算机信息系统可信计算基独立分辨的审计记录
        </p>

        </strong>

        <strong>
         <span style="color:#ad720d;">
          客体名
         </span>
        </strong>
        及
        <span style="color:#ad720d;">
         <strong>
          客体的安全级别
         </strong>
        </span>
        。此外，计算机信息系统可信计算基具有审计
        <strong>
         <span style="color:#ad720d;">

         </span>
        </strong>
       </td>
      </tr>

       </td>

        三级基础上，
        <strong>
         增强的审计功能是：
        </strong>
        计算机信息系统可信
        <br/>
        计算基能够审计
        <strong>
         <span style="color:#ad720d;">
          利用隐蔽存储信道时可能被使用的事件
         </span>
        </strong>
       </td>
      </tr>
      <tr>
       <td style="text-align:center;width:50px;">

        </strong>

        <span style="color:#ad720d;">
         <strong>
          监控可审计安全事件发生与积累的机制
         </strong>
        </span>
        ，当超过阈值时， 能够立即向安全管理员发出报警。并且，如果这些与安全相关的事件继续发生或积累，系统应以最小的代价中止它们
       </td>
      </tr>

    <h2>
     二、网络安全审计系统组成与类型
    </h2>
    <h3>
     2.1 网络安全审计系统组成
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       组成：
      </strong>
     </span>
     包括审计信息获取、审计信息存储、审计信息分析、审计信息展示及利用、系统管理、审计数信息安全保护、审计对象（操作系统、数据库、网络通信等）
    </p>
    <p>
     针对不同的审计对象，安全审计系统的
     <strong>
      组成部分各不相同
     </strong>
     ，审计细粒度也有所区分，例如
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        操作系统的安全审计：
       </strong>
      </span>
      可以做到对进程活动、文件操作的审计
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        网络通信的安全审计：
       </strong>
      </span>
      既可对IP包的源地址、目的地址进行审计，又可以对IP包的内容进行深度分析，实现网络内容审计
     </li>
    </ul>
    <h3>
     2.2 网络安全审计系统类型
    </h3>
    <p>
     按照
     <span style="color:#1c7892;">
      <strong>
       审计对象类型分类
      </strong>
     </span>
     ，
     <strong>
      网络安全审计主要有：
     </strong>
     操作系统安全审计、数据库安全审计、网络通信安全审计、应用系统安全审计、网络安全设备审计、工控安全审计、移动安全审计、互联网安全审计、代码安全审计等
    </p>
    <h4>
     操作系统审计
    </h4>
    <p>
     一般是对操作
     <strong>
      系统用户
     </strong>
     和
     <strong>
      系统服务
     </strong>
     进行记录，主要包括用户登录和注销、系统服务启动和关闭、安全事件等
    </p>
    <p>
     Windows、 Linux等操作系统都
     <span style="color:#1c7892;">
      <strong>
       自带审计功能
      </strong>
     </span>
     ，其审计信息简要叙述如下:
    </p>
    <ul>
     <li>
      <strong>
       Windows
      </strong>
      <span style="color:#1c7892;">
       <strong>
        基本审计信息：
       </strong>
      </span>
      有注册登录事件、目录服务访问、审计账户管理、对象访问、审计策略变更、特权使用、进程跟踪、系统事件等
     </li>
     <li>
      <strong>
       Linux
      </strong>
      <span style="color:#1c7892;">
       <strong>
        基本审计信息：
       </strong>
      </span>
      有系统开机自检日志boot.log、用户命令操作日志acct/pacct、最近登录日志lastlog、使用su命令日志sulog、当前用户登录日志utmp、用户登录和退出日志wtmp、系统接收和发送邮件日志maillog、系统消息messages等
     </li>
    </ul>
    <h4>
     数据库审计
    </h4>
    <p>
     通常是
     <span style="color:#ad720d;">
      监控并记录用户对数据库服务器的读、写、查询、添加、修改以及删除等操作
     </span>
     ，并可以对数据库操作命令进行回放
    </p>
    <h4>
     网络通信安全审计
    </h4>
    <p>
     一般
     <span style="color:#ad720d;">
      采用
      <strong>
       专用的审计系统
      </strong>
      ，通过专用设备获取网络流量，然后再进行
      <strong>
       存储和分析
      </strong>
     </span>
    </p>
    <p>
     网络通信安全
     <span style="color:#1c7892;">
      <strong>
       审计常见内容：
      </strong>
     </span>
     IP源地址、IP目的地址、源端口号、目的端口号、协议类型、传输内容等
    </p>
    <p>
     按照
     <span style="color:#1c7892;">
      <strong>
       审计范围
      </strong>
     </span>
     ，
     <span style="color:#1c7892;">
      <strong>
       安全审计可分为：
      </strong>
     </span>
     综合审计系统和单个审计系统
    </p>
    <ul>
     <li>
      由于各IT产品自带的审计功能有限，审计能力不足，于是安全厂商研发了
      <span style="color:#ad720d;">
       <strong>
        综合审计系统
       </strong>
      </span>
     </li>
     <li>
      <span style="color:#ad720d;">
       <strong>
        单个审计系统
       </strong>
      </span>
      主要针对独立的审计对象，审计数据来源单一，缺少多源审计对象的关联分析，常见的是
      <strong>
       IT系统
      </strong>
      或
      <strong>
       产品自带的审计功能
      </strong>
     </li>
    </ul>
    <h2>
     三、网络安全审计机制与实现技术
    </h2>
    <h3>
     3.1 网络安全审计机制
    </h3>
    <ul>
     <li>
      基于主机的审计机制
     </li>
     <li>
      基于网络通信的审计机制
     </li>
     <li>
      基于应用的审计机制等
     </li>
    </ul>
    <h3>
     3.2 审计机制常用实现技术
    </h3>
    <h4>
     1. 系统日志数据采集技术
    </h4>
    <p>
     常见的系统日志数据采集技术是把操作系统、数据库、网络设备等
     <span style="color:#ad720d;">
      系统中产生的
      <strong>
       事件信息汇聚到统一的服务器存储
      </strong>
      ，以便于查询分析与管理
     </span>
    </p>
    <h4>
     2. 网络流量数据获取技术
    </h4>
    <p>
     是网络通信安全审计的关键技术之一，
     <span style="color:#1c7892;">
      <strong>
       常见的技术方法
      </strong>
     </span>
     有
     <span style="color:#ad720d;">
      共享网络监听、交换机端口镜像
     </span>
     <span style="color:#333333;">
      (Port Mirroring)
     </span>
     <span style="color:#ad720d;">
      、网络分流器
     </span>
     (Network Tap)等
    </p>
    <p>
     其中，
     <span style="color:#1c7892;">
      <strong>
       共享网络监听：
      </strong>
     </span>
     <span style="color:#ad720d;">
      利用Hub集线器构建共享式网络，网络流量采集设备接入集线器上，获取与集线器相连接的设备的网络流量数据
     </span>
     <br/>
     对于不支持端口镜像功能的交换机，通常利用
     <strong>
      网络分流器(TAP)
     </strong>
     把网络流量导入网络流量采集设备
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       网络流量采集设备：
      </strong>
     </span>
     安装
     <strong>
      网络数据捕获软件
     </strong>
     ，从网络上获取原始数据，然后再进行后续处理。目前，常见的
     <span style="color:#1c7892;">
      <strong>
       开源网络数据采集软件包
      </strong>
     </span>
     是Libpcap (Library for Packet Capture)
    </p>
    <p>
     <span style="color:#0d0016;">
      <strong>
       Libpcap的工作流程
      </strong>
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#1c7892;">
       <strong>
        设置嗅探网络接口：
       </strong>
      </span>
      在Linux操作系统中，大多数为eth0
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        初始化Libpcap：
       </strong>
      </span>
      设定过滤规则，明确获取网络数据包的类型
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        运行Libpcap循环主体：
       </strong>
      </span>
      Libpcap开始接收符合过滤规则的数据包
     </li>
    </ol>
    <p>
     除了Libpcap外，还有
    </p>
    <ul>
     <li>
      Winpcap：它支持在Windows平台
      <strong>
       捕获
      </strong>
      网络数据包
     </li>
     <li>
      Windump：是基于Winpcap的网络协议分析工具，可以
      <strong>
       采集
      </strong>
      网络数据包
     </li>
     <li>
      Tcpdump：是基于Libpcap的网络流量数据采集工具，常常应用于Linux操作系统中
     </li>
     <li>
      Wireshark：是
      <span style="color:#ad720d;">
       <strong>
        图形化的
       </strong>
      </span>
      网络流量数据采集工具，可用于网络流量数据的采集和分析
     </li>
    </ul>
    <h4>
     3. 网络审计数据安全分析技术
    </h4>
    <p>
     网络审计数据蕴涵着网络安全威胁相关信息，需要通过数据分析技术方法来提取
    </p>
    <p>
     常见的网络审计
     <span style="color:#1c7892;">
      <strong>
       数据安全分析技术：
      </strong>
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#1c7892;">
       <strong>
        字符串匹配：
       </strong>
      </span>
      通过
      <strong>
       模式匹配
      </strong>
      来查找相关审计数据，以便发现安全问题。常见的字符串匹配工具是grep，其
      <span style="color:#1a439c;">
       <strong>
        使用的格式：
       </strong>
      </span>
      <span style="color:#ad720d;">
       <strong>
        grep [ options ] [ regexp ] [ filename ]
       </strong>
      </span>
      ，
      <span style="color:#1a439c;">
       <strong>
        regexp
       </strong>
      </span>
      为正则表达式，用来表示要搜索匹配的模式
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        全文搜索：
       </strong>
      </span>
      利用搜索引擎技术来分析审计数据
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据关联：
       </strong>
      </span>
      指将网络安全威胁情报信息，如系统日志、全网流量、安全设备日志等
      <span style="color:#ad720d;">
       多个数据来源进行综合分析，以发现网络中的异常流量
      </span>
      ，
      <strong>
       识别未知攻击手段
      </strong>
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        统计报表：
       </strong>
      </span>
      是对安全审计数据的特定事件、阈值、安全基线等进行统计分析，以生成告警信息，形成发送日报、周报、月报
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        可视化分析：
       </strong>
      </span>
      将安全审计数据进行
      <span style="color:#ad720d;">
       图表化处理
      </span>
      ，形成饼图、柱状图、折线图、地图等各种可视化效果，以支持各种用户场景。将不同维度的可视化效果汇聚成仪表盘，辅助用户实时查看当前事件变更。安全关键KPI状态高亮显示，突出异常行为的重要性
     </li>
    </ol>
    <h4>
     4. 网络审计数据存储技术
    </h4>
    <p>
     <strong>
      分为两种
     </strong>
    </p>
    <ol>
     <li>
      一种是
      <span style="color:#1c7892;">
       <strong>
        由审计数据产生的系统自己分散存储
       </strong>
      </span>
      ，审计数据
      <strong>
       保存在不同的系统中
      </strong>
      ，目前，操作系统、数据库、应用系统、网络设备等都可以各自存储日志数据
     </li>
     <li>
      另一种
      <strong>
       <span style="color:#1c7892;">
        集中采集各种系统的审计数据
       </span>
      </strong>
      ，
      <span style="color:#ad720d;">
       建立审计数据存储服务器，由专用的存储设备保存，便于事后查询分析和电子取证
      </span>
     </li>
    </ol>
    <h4>
     5.网络审计数据保护技术
    </h4>
    <p>
     网络审计数据
     <span style="color:#ad720d;">
      涉及系统
      <strong>
       整体的安全性
      </strong>
      和
      <strong>
       用户的隐私性
      </strong>
     </span>
    </p>
    <p>
     为保护审计数据的安全，通常的
     <span style="color:#1c7892;">
      <strong>
       安全技术措施如下：
      </strong>
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#1a439c;">
       <strong>
        系统用户分权管理：
       </strong>
      </span>
      操作系统、数据库等系统设置操作员、安全员和审计员
      <strong>
       三种类型的用户
      </strong>
      <ul>
       <li>
        <span style="color:#1c7892;">
         <strong>
          操作员：
         </strong>
        </span>
        只负责对系统的操作维护工作，其操作过程被系统进行了详细记录
       </li>
       <li>
        <strong>
         <span style="color:#1c7892;">
          安全员：
         </span>
        </strong>
        负责系统安全策略配置和维护
       </li>
       <li>
        <span style="color:#1c7892;">
         <strong>
          审计员：
         </strong>
        </span>
        负责维护审计相关事宜，可以查看操作员、安全员工作过程日志;操作员不能够修改自己的操作记录，审计员也不能对系统进行操作
       </li>
      </ul>
     </li>
     <li>
      <span style="color:#1a439c;">
       <strong>
        审计数据强制访问：
       </strong>
      </span>
      系统采取强制访问控制措施，对审计数据设置
      <strong>
       安全标记
      </strong>
      ，防止非授权用户查询及修改审计数据
     </li>
     <li>
      <span style="color:#1a439c;">
       <strong>
        审计数据加密：
       </strong>
      </span>
      使用加密技术对敏感的审计数据进行加密处理，以防止非授权查看审计数据或泄露
     </li>
     <li>
      <span style="color:#1a439c;">
       <strong>
        审计数据隐私保护：
       </strong>
      </span>
      采取隐私保护技术，防止审计数据泄露隐私信息
     </li>
     <li>
      <span style="color:#1a439c;">
       <strong>
        审计数据完整性保护：
       </strong>
      </span>
      使用Hash算法和数字签名，对审计数据进行数字签名和来源认证、完整性保护，防止非授权修改审计数据
     </li>
    </ol>
    <h2>
     四、网络安全审计主要产品与技术指标
    </h2>
    <h3>
     4.1 日志安全审计产品
    </h3>
    <p>
     是有关
     <span style="color:#ad720d;">
      日志信息
      <strong>
       采集、分析
      </strong>
      与
      <strong>
       管理
      </strong>
     </span>
     的系统
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品的基本原理：
      </strong>
     </span>
     是利用Sys log、Snmptrap、 NetFlow、 TeInet、 SSH、 WMI、FTP、 SFTP、 SCP、 JDBC、 文件等技术，对分散设备的异构系统日志进行分布采集、集中存储、统计分析、集中管理，便于有关单位/机构进行安全合规管理，保护日志信息安全
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品的主要功能：
      </strong>
     </span>
     日志采集、日志存储、日志分析、日志查询、事件告警、统计报表、系统管理等
    </p>
    <h3>
     4.2 主机监控与审计产品
    </h3>
    <p>
     是有关
     <strong>
      主机
      <span style="color:#ad720d;">
       行为信息
      </span>
      的安全审查
     </strong>
     及
     <strong>
      管理
     </strong>
     的系统
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品的基本原理：
      </strong>
     </span>
     是
     <span style="color:#ad720d;">
      通过
      <strong>
       代理程序
      </strong>
      对主机的行为信息进行
      <strong>
       采集
      </strong>
     </span>
     ，然后基于采集到的信息进行
     <strong>
      分析
     </strong>
     ，以记录系统行为，帮助管理员评估操作系统的风险状况，并为相应的安全策略调整提供依据
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品的主要功能：
      </strong>
     </span>
     有系统用户监控、系统配置管理、补丁管理、准入控制、存储介质(U盘)管理、非法外联管理等
    </p>
    <h3>
     4.3 数据库审计产品
    </h3>
    <p>
     是对
     <strong>
      数据库系统活动
     </strong>
     进行审计的系统
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品的基本原理：
      </strong>
     </span>
     是
     <span style="color:#ad720d;">
      通过
      <strong>
       网络流量监听、系统调用监控、数据库代理
      </strong>
      等技术手段对所有访问数据库系统的行为信息进行
      <strong>
       采集
      </strong>
     </span>
     ，然后对采集的信息进行
     <strong>
      分析
     </strong>
     ，形成数据库操作记录，保存和发现数据库各种违规的或敏感的操作信息，为相应的数据库安全策略调整提供依据
    </p>
    <p>
     在数据库审计产品中，实现数据库审计主要有如下三种方式
    </p>
    <h4>
     1. 网络监听审计
    </h4>
    <p>
     <span style="color:#ad720d;">
      对获取到的
      <strong>
       数据库流量
      </strong>
      进行分析，从而实现对数据库访问的审计和控制
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        优点：
       </strong>
      </span>
      数据库网络审计不影响数据库服务器
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        缺点：
       </strong>
      </span>
      网络监听审计对加密的数据库网络流量难以审计，同时无法对数据库服务器的本地操作进行审计
     </li>
    </ul>
    <h4>
     2. 自带审计
    </h4>
    <p>
     通过启用数据库系统
     <span style="color:#ad720d;">
      自带的审计功能
     </span>
     ，实现数据库的审计
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        优点：
       </strong>
      </span>
      能够实现数据库网络操作和本地操作的审计
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        缺点：
       </strong>
      </span>
      对数据库系统的性能有一些影响，在审计策略配置、记录的粒度、日志统一分析方面不够完善，日志本地存储容易被删除
     </li>
    </ul>
    <h4>
     3. 数据库Agent
    </h4>
    <p>
     在数据库服务器上
     <span style="color:#ad720d;">
      <strong>
       安装采集代理
      </strong>
     </span>
     (Agent)，通过Agent
     <span style="color:#ad720d;">
      对数据库的各种访问行为进行分析
     </span>
     ，从而实现数据库审计
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        优点：
       </strong>
      </span>
      能够实现数据库网络操作和本地操作的审计
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        缺点：
       </strong>
      </span>
      数据库Agent需要
      <strong>
       安装数据库服务器
      </strong>
      ，对数据库服务系统的性能、稳定性、可靠性有影响
     </li>
    </ul>
    <h3>
     4.4 网络安全审计产品
    </h3>
    <p>
     是有关
     <span style="color:#ad720d;">
      <strong>
       网络通信活动的审计系统
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品基本原理：
      </strong>
     </span>
     通过
     <strong>
      网络流量信息采集
     </strong>
     及
     <strong>
      数据包深度内容分析
     </strong>
     ，提供网络通信及网络应用的活动信息记录
    </p>
    <h4>
     网络安全审计常见功能
    </h4>
    <p>
     <span style="color:#1a439c;">
      <strong>
       1.网络流量采集：
      </strong>
     </span>
     获取网上通信流量信息，按照
     <strong>
      协议类型
     </strong>
     及保存流量数据。
     <br/>
     <span style="color:#1a439c;">
      <strong>
       2.网络流量数据挖掘分析：
      </strong>
     </span>
     对采集到的网络流量数据进行挖掘，提取网络流量信息，形成网络审计记录，
     <strong>
      主要包括如下内容
     </strong>
    </p>
    <ol>
     <li>
      <span style="color:#1c7892;">
       <strong>
        邮件收发协议(SMTP、 POP3协议)审计：
       </strong>
      </span>
      从邮件网络流量数据提取信息，记录收发邮仵的时间、地址、主题、附件名、收发人等信息，并能够回放所收发的邮件内容
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        网页浏览(HTTP协议)审计：
       </strong>
      </span>
      从Web网络流量数据提取信息，记录用户访问网页的时间、地址、域名等信息，并能够回放所浏览的网页内容
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        文件共享(NetBios协议)审计：
       </strong>
      </span>
      从文件共享网络流量数据提取信息，记录网络用户对网络资源中的文件共享操作
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        文件传输(FTP协议)审计：
       </strong>
      </span>
      从FTP网络流量数据提取信息，记录用户对FTP服务器的远程登录时间、读、写、添加、修改以及删除等操作，并可以对操作过程进行完整回放
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        远程访问(TeInet协议) 审计：
       </strong>
      </span>
      从FTP网络流量数据提取信息，记录用户对Te Inet服务器的远程登录时间、各种操作命令，并可以对操作过程进行完整回放
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        DNS审计：
       </strong>
      </span>
      从DNS网络流量数据提取信息，记录用户DNS服务请求信息，并可以对操作过程进行完整回放。
     </li>
    </ol>
    <h4>
     网络安全审计产品性能指标
    </h4>
    <ol>
     <li>
      <span style="color:#ad720d;">
       支持网络带宽大小
      </span>
     </li>
     <li>
      <span style="color:#ad720d;">
       协议识别种类
      </span>
     </li>
     <li>
      <span style="color:#ad720d;">
       原始数据包查询响应时间
      </span>
     </li>
    </ol>
    <h3>
     4.5 工业控制系统网络审计产品
    </h3>
    <p>
     是
     <span style="color:#ad720d;">
      对工业控制网络中的协议、数据和行为等进行
      <strong>
       记录、分析
      </strong>
     </span>
     ，并做出一定的响应措施的信息安全专用系统
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       产品基本原理：
      </strong>
     </span>
     是利用网络流量采集及协议识别技术，对工业控制协议进行还原，形成工业控制系统的操作信息记录，然后进行
     <strong>
      保存和分析
     </strong>
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       实现方式通常分两种情况：
      </strong>
     </span>
    </p>
    <ol>
     <li>
      一种是
      <span style="color:#ad720d;">
       <strong>
        一体化集中产品
       </strong>
      </span>
      ，即将数据采集和分析功能集中在一台硬件中，统一完成审计分析功能
     </li>
     <li>
      另一种是由
      <span style="color:#ad720d;">
       <strong>
        采集端和分析端两部分组成
       </strong>
      </span>
      ，
      <span style="color:#1c7892;">
       <strong>
        采集端
       </strong>
      </span>
      主要提供数据采集的功能，将采集到的网络数据发送给分析端，由
      <span style="color:#1c7892;">
       <strong>
        分析端
       </strong>
      </span>
      进一步处理和分析，并采取相应的响应措施
     </li>
    </ol>
    <h3>
     4.6 运维安全审计产品
    </h3>
    <p>
     是有关
     <strong>
      网络设备
     </strong>
     及
     <strong>
      服务器操作的
     </strong>
     审计系统
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       运维安全审计产品：
      </strong>
     </span>
     主要采集和记录IT系统维护过程中相关人员“在什么终端、什么时间、登录什么设备(或系统)、做了什么操作、返回什么结果、什么时间登出”等行为信息，为管理人员及时发现权限滥用、违规操作等情况，准确定位身份，以便追查取证
    </p>
    <p>
     <span style="color:#1c7892;">
      <strong>
       基本原理：
      </strong>
     </span>
     是通过
     <span style="color:#ad720d;">
      <strong>
       网络流量信息采集
      </strong>
     </span>
     <span style="color:#333333;">
      或
     </span>
     <span style="color:#ad720d;">
      <strong>
       服务代理
      </strong>
     </span>
     等技术方式，记录Telnet、 FTP、SSH、tftp、HTTP等运维操作服务的活动信息
    </p>
    <h4>
     主要功能
    </h4>
    <ol>
     <li>
      <span style="color:#0d0016;">
       <strong>
        字符会话审计：
       </strong>
      </span>
      审计SSH、TeInet等协议的操作行为，
      <strong>
       审计内容
      </strong>
      包括访问起始和终止时间、用户名、用户IP、设备名称、设备IP、协议类型、危险等级和操作命令等
     </li>
     <li>
      <strong>
       图形操作审计：
      </strong>
      审计RDP 、VNC等远程桌面以及HTTP/HTTPS协议的图形操作行为，
      <strong>
       审计内容
      </strong>
      包括访问起始和终止时间、用户名、用户IP、设备名称、设备IP、协议类型、危险等级和操作内容等
     </li>
     <li>
      <strong>
       数据库运维审计：
      </strong>
      审计Oracle、 MS SQL Server、IBM DB2、PostgreSQL等 各主流数据库的操作行为，
      <strong>
       审计内容
      </strong>
      包括访问起始和终止时间、用户名、用户IP、设备名称、设备IP、协议类型、危险等级和操作内容等
     </li>
     <li>
      <strong>
       文件传输审计：
      </strong>
      审计FTP、 SFTP等协议的操作行为，
      <strong>
       审计内容
      </strong>
      包括访问起始和终止时间、用户名、用户IP、设备名称、目标设备IP、协议类型、文件名称、危险等级和操作命令等
     </li>
     <li>
      <strong>
       合规审计：
      </strong>
      根据上述审计内容，参照相关的安全管理制度，对运维操作进行合规检查，给出符合性审查
     </li>
    </ol>
    <h2>
     五、网络安全审计应用
    </h2>
    <h3>
     5.1 安全运维保障
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       IT系统运维
      </strong>
     </span>
     <span style="color:#ad720d;">
      面临
      <strong>
       内部安全威胁
      </strong>
      和
      <strong>
       第三方外包服务安全风险
      </strong>
     </span>
     ，
     <span style="color:#ad720d;">
      <strong>
       网络安全审计
      </strong>
      是应对运维安全风险的重要安全保障机制
     </span>
     。通过运维审计，
     <strong>
      可以有效
      <span style="color:#1c7892;">
       防范和追溯
      </span>
      安全威胁操作
     </strong>
    </p>
    <h3>
     5.2 数据访问监测
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       数据库
      </strong>
     </span>
     承载企事业单位的
     <span style="color:#ad720d;">
      重要核心数据资源
     </span>
     ，保护数据库的安全成为各相关部门门的重要职责。数据库安全审计产品就是通过安全监测，智能化、自动地从海量日志信息中，发现违规访问或异常访问记录，从而有效降低安全管理员
     <span style="color:#ad720d;">
      日志分析的工作量
     </span>
    </p>
    <h3>
     5.3 网络入侵检测
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       网络入侵检测
      </strong>
     </span>
     <span style="color:#ad720d;">
      对网络设备、安全设备、商用系统的日志信息进行实时收集和分析
     </span>
     ，可检测发现黑客入侵、扫描渗透、暴力破解、网络蠕虫、非法访问、非法外联和DDoS攻击
    </p>
    <h3>
     5.4 网络电子取证
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       日志分析技术
      </strong>
     </span>
     广泛应用于
     <strong>
      计算机犯罪侦查
     </strong>
     与
     <strong>
      电子取证
     </strong>
     ，许多案件借助日志分析技术提供线索、获取证据
