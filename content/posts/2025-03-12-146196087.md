    <h1 class="title-article" id="articleContentId">
     git报错：“fatal:refusing to merge unrelated histories“
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     新建仓库，克隆本地项目到新仓库，首次同步本地已提交的代码到远程时，报错："fatal:refusing to
     <a href="https://so.csdn.net/so/search?q=merge&amp;spm=1001.2101.3001.7020" title="merge">
      merge
     </a>
     unrelated histories" 。
    </p>
    <p>
     报错意思是：致命的：拒绝合并无关的历史。
    </p>
    <h3>
     一、问题背景：
    </h3>
    <p>
     Git 在合并分支时，会检查两个分支是否有共同的祖先（即它们是否来自同一个代码库）。如果没有共同祖先，Git 会认为这两个分支是“无关的历史”，并拒绝合并，以避免意外覆盖或冲突。
    </p>
    <p>
     这种情况可能发生在以下场景中：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        克隆了一个新的仓库
       </strong>
       ，然后尝试将本地分支推送到远程仓库。
      </p>
     </li>
     <li>
      <p>
       <strong>
        合并了来自不同仓库的分支
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用了
        <code>
         git pull
        </code>
        操作
       </strong>
       ，但本地和远程分支没有共同的提交历史。
      </p>
     </li>
    </ol>
    <h3>
     二、解决方法
    </h3>
    <h5>
     方法 1：强制合并（不推荐）
    </h5>
    <p>
     如果你确定两个分支的内容是安全的，并且没有冲突，可以使用
     <code>
      --allow-unrelated-histories
     </code>
     选项强制合并：
    </p>
    <pre><code class="language-bash">git pull origin &lt;branch-name&gt; --allow-unrelated-histories</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="language-bash">git merge &lt;branch-name&gt; --allow-unrelated-histories</code></pre>
    <p>
     我是用了上面的方法，因为确保两个历史都是安全的。如下：
    </p>
    <pre><code class="language-bash">git pull origin master --allow-unrelated-histories</code></pre>
    <p>
     <strong>
      注意
     </strong>
     ：这种方法会直接合并两个分支的内容，可能会导致意外的冲突或覆盖。使用时需谨慎。
    </p>
    <h5>
     方法 2：重新初始化本地仓库
    </h5>
    <p>
     如果你的本地仓库是新创建的，并且没有重要提交，可以尝试重新初始化本地仓库：
    </p>
    <ol>
     <li>
      <p>
       删除本地仓库（保留重要文件）。
      </p>
     </li>
     <li>
      <p>
       重新克隆远程仓库：
      </p>
      <pre><code class="language-bash">git clone &lt;remote-repo-url&gt;</code></pre>
     </li>
    </ol>
    <p>
     方法 3：创建一个新的分支
    </p>
    <p>
     如果你的本地分支有重要代码，但没有共同祖先，可以尝试以下步骤：
    </p>
    <ol>
     <li>
      <p>
       创建一个新的分支，基于远程仓库的当前分支：
      </p>
      <pre><code class="language-bash">git checkout &lt;remote-branch-name&gt;
git checkout -b &lt;new-branch-name&gt;</code></pre>
     </li>
     <li>
      <p>
       将本地分支的更改手动复制到新分支中。
      </p>
     </li>
     <li>
      <p>
       提交更改并推送到远程仓库。
      </p>
     </li>
    </ol>
    <h5>
     方法 4：使用
     <code>
      git rebase
     </code>
    </h5>
    <p>
     如果你希望将本地分支的更改应用到远程分支上，可以尝试使用
     <code>
      git rebase
     </code>
     ：
    </p>
    <ol>
     <li>
      <p>
       切换到本地分支：
      </p>
      <pre><code class="language-bash">git checkout &lt;local-branch-name&gt;</code></pre>
     </li>
     <li>
      <p>
       将远程分支作为基础进行变基：
      </p>
      <pre><code class="language-bash">git rebase &lt;remote-branch-name&gt;</code></pre>
     </li>
     <li>
      <p>
       如果出现冲突，解决冲突后继续变基：
      </p>
      <pre><code class="language-bash">git rebase --continue</code></pre>
     </li>
     <li>
      <p>
       完成后推送更改：
      </p>
      <pre><code class="language-bash">git push origin &lt;local-branch-name&gt;</code></pre>
     </li>
    </ol>
    <h5>
     方法 5：检查远程仓库的 URL
    </h5>
    <p>
     如果你的本地仓库和远程仓库不匹配，可能是远程仓库的 URL 错误。可以通过以下命令检查和更新远程仓库的 URL：
    </p>
    <pre><code class="language-bash">git remote -v
git remote set-url origin &lt;correct-remote-repo-url&gt;</code></pre>
    <h3>
     三、总结
    </h3>
    <p>
     “拒绝合并无关的历史” 是 Git 在合并无共同祖先的分支时的保护机制。根据你的需求，可以选择强制合并、重新初始化仓库、创建新分支或使用变基等方法解决问题。如果不确定操作的后果，建议先备份重要代码，避免数据丢失。
    </p>
    <p>
