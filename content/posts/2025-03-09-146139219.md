    <h4 id="2.%20%E5%85%B7%E4%BD%93%E4%BD%BF%E7%94%A8%E5%9C%BA%E6%99%AF%E5%88%86%E6%9E%90" name="2.%20%E5%85%B7%E4%BD%93%E4%BD%BF%E7%94%A8%E5%9C%BA%E6%99%AF%E5%88%86%E6%9E%90">
     <strong>
      2. 具体使用场景分析
     </strong>
    </h4>
    <h5 id="(1)%20JSON%20%E6%A0%BC%E5%BC%8F" name="(1)%20JSON%20%E6%A0%BC%E5%BC%8F">
     <strong>
      (1) JSON 格式
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        典型场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         前后端分离的 API 交互
        </p>
       </li>
       <li>
        <p>
         需要传递嵌套对象或数组（如
         <code>
          { "user": { "name": "Alice", "roles": ["admin"] } }
         </code>
         ）
        </p>
       </li>
       <li>
        <p>
         移动端或第三方服务调用
        </p>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       示例代码
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="language-java">@PostMapping("/api/data")
public ResponseEntity&lt;?&gt; handleJson(@RequestBody DataRequest request) {
    // 处理复杂 JSON 数据
}</code></pre>
    <ul>
     <li>
      <strong>
       Postman 配置
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="language-java">{
  "id": 123,
  "config": {
    "enable": true,
    "items": ["A", "B"]
  }
}</code></pre>
    <h5 id="(2)%20GET%20%E6%9F%A5%E8%AF%A2%E5%8F%82%E6%95%B0" name="(2)%20GET%20%E6%9F%A5%E8%AF%A2%E5%8F%82%E6%95%B0">
     <strong>
      (2) GET 查询参数
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        典型场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         简单搜索（如
         <code>
          ?keyword=test
         </code>
         ）
        </p>
       </li>
       <li>
        <p>
         分页查询（如
         <code>
          ?page=2&amp;size=10
         </code>
         ）
        </p>
       </li>
       <li>
        <p>
         资源定位（如
         <code>
          /users?id=1001
         </code>
         ）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例代码
       </strong>
       ：
      </p>
      <pre><code class="language-java">@GetMapping("/search")
public List&lt;User&gt; searchUsers(@RequestParam String keyword) {
    // 根据关键词查询用户
}</code></pre>
     </li>
     <li>
      <p>
       <strong>
        Postman 配置
       </strong>
       ：
      </p>
      <pre><code class="language-java">URL: http://localhost:8080/search?keyword=test</code></pre>
     </li>
    </ul>
    <h5 id="(3)%20POST%20%E8%A1%A8%E5%8D%95%E6%95%B0%E6%8D%AE" name="(3)%20POST%20%E8%A1%A8%E5%8D%95%E6%95%B0%E6%8D%AE">
     <strong>
      (3) POST 表单数据
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        典型场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         传统 HTML 表单提交（如登录表单）
        </p>
       </li>
       <li>
        <p>
         文件上传（需用
         <code>
          form-data
         </code>
         ）
        </p>
       </li>
       <li>
        <p>
         兼容旧系统接口
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例代码
       </strong>
       ：
      </p>
      <pre><code class="language-java">@PostMapping("/login")
public String login(@RequestParam String username, 
                    @RequestParam String password) {
    // 处理登录逻辑
}</code></pre>
     </li>
     <li>
      <p>
       <strong>
        Postman 配置
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <code>
          x-www-form-urlencoded
         </code>
         ：
        </p>
        <pre><code class="language-java">username=admin&amp;password=123456</code></pre>
       </li>
       <li>
        <p>
         <code>
          form-data
         </code>
         （含文件）：
        </p>
        <pre><code class="language-java">key: file | value: [选择文件]</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4 id="3.%20%E5%AE%89%E5%85%A8%E6%80%A7%E5%AF%B9%E6%AF%94" name="3.%20%E5%AE%89%E5%85%A8%E6%80%A7%E5%AF%B9%E6%AF%94">
     <strong>
      3. 安全性对比
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        攻击类型
       </th>
       <th>
        JSON 格式
       </th>
       <th>
        GET 参数
       </th>
       <th>
        POST 表单数据
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         CSRF
        </strong>
       </td>
       <td>
        需配合 token 防护
       </td>
       <td>
        高风险（参数在 URL 中）
       </td>
       <td>
        需配合 token 防护
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         XSS
        </strong>
       </td>
       <td>
        需转义输出
       </td>
       <td>
        高风险（参数易被注入）
       </td>
       <td>
        需转义输出
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         数据泄露
        </strong>
       </td>
       <td>
        低（HTTPS 加密）
       </td>
       <td>
        高（URL 可能被浏览器缓存）
       </td>
       <td>
        中（HTTPS 加密）
       </td>
      </tr>
     </tbody>
    </table>

    <h4 id="4.%20%E6%80%A7%E8%83%BD%E4%B8%8E%E8%B0%83%E8%AF%95%E6%88%90%E6%9C%AC" name="4.%20%E6%80%A7%E8%83%BD%E4%B8%8E%E8%B0%83%E8%AF%95%E6%88%90%E6%9C%AC">
     <strong>
      4. 性能与调试成本
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        JSON
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          解析成本
         </strong>
         ：需要序列化/反序列化（如 Jackson）
        </p>
       </li>
       <li>
        <p>
         <strong>
          调试难度
         </strong>
         ：直观（结构化数据）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        GET 参数
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          解析成本
         </strong>
         ：低（直接读取 URL）
        </p>
       </li>
       <li>
        <p>
         <strong>
          调试难度
         </strong>
         ：简单（参数可见）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        POST 表单
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          解析成本
         </strong>
         ：中（需解码请求体）
        </p>
       </li>
       <li>
        <p>
         <strong>
          调试难度
         </strong>
         ：中（需查看请求体）
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4 id="5.%20%E5%A6%82%E4%BD%95%E9%80%89%E6%8B%A9%EF%BC%9F" name="5.%20%E5%A6%82%E4%BD%95%E9%80%89%E6%8B%A9%EF%BC%9F">
     <strong>
      5. 如何选择？
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        优先 JSON
       </strong>
       ：
       <br/>
       当满足以下条件时：
      </p>
      <ul>
       <li>
        <p>
         需要传输复杂数据结构
        </p>
       </li>
       <li>
        <p>
         前后端分离架构
        </p>
       </li>
       <li>
        <p>
         对安全性和可扩展性要求较高
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用 GET 参数
       </strong>
       ：
       <br/>
       当满足以下条件时：
      </p>
      <ul>
       <li>
        <p>
         简单查询场景
        </p>
       </li>
       <li>
        <p>
         需要缓存或书签功能
        </p>
       </li>
       <li>
        <p>
         参数无敏感信息
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用 POST 表单
       </strong>
       ：
       <br/>
       当满足以下条件时：
      </p>
      <ul>
       <li>
        <p>
         兼容传统表单提交
        </p>
       </li>
       <li>
        <p>
         需要上传文件（
         <code>
          form-data
         </code>
         ）
        </p>
       </li>
       <li>
        <p>
         参数为简单键值对且无需嵌套
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4 id="%E6%80%BB%E7%BB%93" name="%E6%80%BB%E7%BB%93">
     <strong>
      总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        JSON
       </strong>
       是现代 Web 开发的
       <strong>
        首选
       </strong>
       ，适合复杂、结构化的数据交互。
      </p>
     </li>
     <li>
      <p>
       <strong>
        GET 参数
       </strong>
       适合
       <strong>
        简单、公开
       </strong>
       的数据传递，但需注意 URL 长度和安全性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        POST 表单
       </strong>
       是传统 Web 表单的
       <strong>
        标准方式
       </strong>
       ，适合键值对和文件上传场景。
      </p>
     </li>
    </ul>
    <p>
     根据实际业务需求和安全规范，选择最合适的参数传递方式。
    </p>
