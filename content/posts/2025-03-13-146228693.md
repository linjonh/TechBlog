    <h2>
     <a id="1_0">
     </a>
     1、本方案要解决场景？
    </h2>
    <p>
     想直接通过本地 IDEA 将最新的代码部署到远程服务器上。
    </p>
    <h2>
     <a id="2_2">
     </a>
     2、本方案适用于什么样的项目？
    </h2>
    <ol>
     <li>
      项目是一个 Spring Boot 的 Java 项目。
     </li>
     <li>
      项目用 maven 进行管理。
     </li>
     <li>
      项目的运行基于 docker 容器（即项目将被打成 docker image，并最终以容器的方式来运行）。
     </li>
    </ol>
    <h2>
     <a id="3_6">
     </a>
     3、本方案对于远程服务器的要求？
    </h2>
    <p>
     该服务器是用来最终运行项目的，所以对它的要求是：已安装了 Docker，并能够正常拉取镜像。
    </p>
    <h2>
     <a id="4_8">
     </a>
     4、配置步骤
    </h2>
    <h3>
     <a id="41SSH__9">
     </a>
     4.1、SSH 配置
    </h3>
    <p>
     本方案是要通过本地的 IDEA 连接远程服务器进行操作的，所以我们需要先在 IDEA 中配置一个 SSH，如下图：
    </p>
    <p>
     找到 IDEA 的「设置」

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2197093a702c4dbaa90c2aa3cd404c42.png">

      添加 SSH 配置
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/70109f9b3a71437093a25ef5bac461d8.png"/>
     </img>
    </p>
    <h3>
     <a id="42Docker__16">
     </a>
     4.2、Docker 配置
    </h3>
    <p>
     通过此配置，我们可以连接到远程服务器上的 Docker 守护进程（即配置了一个Docker服务器，「4.4」中会用到），如下图：
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/7dbae58ba02240f7ad7c6ccee0c20b1b.png">
      <br/>
      【5】中下拉列表中可选的SSH配置来自「4.1」中的配置。
     </img>
    </p>
    <h3>
     <a id="43SpringBoot__20">
     </a>
     4.3、SpringBoot 项目
    </h3>
    <p>
     需要注意的是用于定义和构建该项目Docker镜像的 Dockerfile 要放在项目 jar 包所在目录的同级（这里提到的项目 jar 包是通过maven package 命令生成的 jar包），否则，在通过 Dockerfile 创建镜像，执行到 ADD target/jnew-whatcell-api.jar 的时候，会出现找不到 jnew-whatcell-api.jar 的情况。

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7633399b9eec47c9981e0980652d6c87.png"/>
     </img>
    </p>

     <a id="44_24">
