    <h4>
     <a id="1Redux_0">
     </a>
     1、Redux概述
    </h4>
    <p>
     Redux 是一个用于
     <strong>
      JavaScript 状态容器
     </strong>
     ，提供可以预测的状态管理
     <br/>
     Redux 可以让我们的构建一致化的应用，运行于不同的环境，客户端、浏览器、服务器，并且易于测试；
     <br/>
     Redux 除了和
     <code>
      React
     </code>
     一起使用，还支撑其他界面库，并且体检很小，只有2kb
    </p>
    <h4>
     <a id="2Redux__6">
     </a>
     2、Redux 的设计初衷
    </h4>
    <p>
     随着js单页面开发应用的日益复杂，js需要管理的数据 状态会越来越多，并且这些数据状态有很多是服务端提供的，还有自己本地缓存的、本地生成没有持久化到服务端的数据，以及UI视图渲染的数据等等；
    </p>
    <p>
     那么我们怎么才能更好的在不同的页面之间进行通讯管理数据呢？
     <br/>
     其实，不使用 Redux 依然可以处理不同页面之间的数据通讯，但是
     <code>
      使用Redux 处理这些多页面，跨页面负责的数据状态管理
     </code>
     ，会更方便一些；
    </p>
    <h4>
     <a id="3Redux__14">
     </a>
     3、Redux 的三大核心
    </h4>
    <h5>
     <a id="31_16">
     </a>
     3.1、单一数据源；
    </h5>
    <p>
     整个应用的
     <code>
      State
     </code>
     被存储在一个
     <code>
      Object tree
     </code>
     中，并且这个 Object tree 只存在于唯一的 store 中；
    </p>
    <h5>
     <a id="32state__20">
     </a>
     3.2、state 是只读的；
    </h5>
    <p>
     唯一修改
     <code>
      state
     </code>
     的方法
     <strong>
      只能是触发 action
     </strong>
     ，action是一个用于描述已发生事件的普通对象；
    </p>
    <p>
     这样是为了 保证视图操作，网络请求数据变更，
     <strong>
      不能直接去修改 state
     </strong>
     ，只能表达 他们将要修改state的意图，通知
     <code>
      action
     </code>
     函数，去集中处理，并且严格按照先
     <strong>
      后顺序一次执行
     </strong>
     ；
    </p>
    <pre><code class="prism language-javascript">store<span class="token punctuation">.</span><span class="token function">dispatch</span><span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">type</span><span class="token operator">:</span><span class="token string">'MIN'</span><span class="token punctuation">,</span>
    <span class="token literal-property property">num</span><span class="token operator">:</span><span class="token number">2</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span>
</code></pre>
    <h5>
     <a id="33_32">
     </a>
     3.3、使用纯函数来执行修改；
    </h5>
    <p>
     为了描述
     <code>
      action
     </code>
     如何改变
     <code>
      state tree
     </code>
     ，我们需要编写一个
     <code>
      reducers
     </code>
    </p>
    <p>
     Reducers 只是一些纯函数，
     <code>
      它接收先前的state 和 action
     </code>
     ，并且返回新的 state，
     <br/>
     纯函数的特点：可以复用，可以控制执行顺序，可以传入附加参数；
    </p>
    <h4>
     <a id="4Redux__39">
     </a>
     4、Redux 的组成
    </h4>
    <h5>
     <a id="41State__41">
     </a>
     4.1、State 状态
    </h5>
    <p>
     状态其实就是我们开发过程中页面之间传递的数据，大致分为以
     <strong>
      下三种
     </strong>
     <br/>
     <strong>
      服务端返回的state
     </strong>
     ，如：用户信息，列表详情
     <br/>
     <strong>
      当前组件的状态 state
     </strong>
     ，如：决定按钮现隐、组件之间切换的
     <br/>
     <strong>
      全局控制的 state
     </strong>
     ，如：路由导航守卫时候的信息状态，主题色的变更
    </p>
    <h5>
     <a id="42Action__49">
     </a>
     4.2、Action 事件
    </h5>
    <p>
     <code>
      action
     </code>
     是将数据从
     <strong>
      应用层
     </strong>
     传递到
     <code>
      store
     </code>
     的载体，是 store的
     <strong>
      唯一
     </strong>
     数据源；通过
     <code>
      store.dispatch() 将 action 传递给 store
     </code>
     <br/>
     <strong>
      特点
     </strong>
     ：
     <br/>
     本质是一个js对象
     <br/>
     必须包含 type 属性，多为常量，用于区分唯一性；
     <br/>
     其他属性数据结构，可以视具体需求自定义；
     <br/>
     <strong>
      只描述了有事情将要发生，并没有描述如何去更新 state
     </strong>
     ；
    </p>
    <pre><code class="prism language-javascript"><span class="token comment">// action</span>

     更多的时候是使用 action 创建函数

    <pre><code class="prism language-javascript"><span class="token keyword">function</span> <span class="token function">userInfo</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

    <h5>
     <a id="43Reducer_83">
     </a>
     4.3、Reducer
    </h5>
    <p>
     本质是一个
     <code>
      纯函数
     </code>
     <br/>
     响应发送过来的 action
     <br/>
     函数接收
     <strong>
      两个参数
     </strong>
     ，第一个是
     <strong>
      初始化的 state
     </strong>
     ，第二个是发送过来的
     <code>
      action
     </code>
     <br/>
     必须要有 return 返回值；
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">const</span> initState <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span><span class="token operator">...</span><span class="token punctuation">}</span>

        <span class="token operator">...</span><span class="token punctuation">.</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>

    <h5>
     <a id="44Store_100">
     </a>
     4.4、Store
    </h5>
    <p>
     如上图
     <code>
      store
     </code>
     用于将 action 和Reducer 关联到一起；
     <br/>
     通过
     <code>
      createStore
     </code>
     来构建 store
     <br/>
     维持应用的 state
     <br/>
     提供
     <code>
      getState()
     </code>
     来获取
     <strong>
      最新的state
     </strong>
     <br/>
     通过
     <code>
      subscribe()
     </code>
     来
     <strong>
      注册监听
     </strong>
     state
     <br/>
     通过
     <code>
      dispatch()
     </code>
     来发送action
     <br/>
     通过
     <code>
      subscribe()
     </code>
     返回值来
     <strong>
      注销
     </strong>
     监听
    </p>
