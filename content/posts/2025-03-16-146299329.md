    <h4>
     <a id="SpringSpring_Boot__Spring_MVC__0">
     </a>
     <strong>
      Spring、Spring Boot 与 Spring MVC 的关系与区别
     </strong>
    </h4>
    <h5>
     <a id="1__4">
     </a>
     <strong>
      1. 核心定位
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       Spring
      </strong>
      <ul>
       <li>
        <strong>
         定位
        </strong>
        ：基础框架，提供
        <strong>
         IoC（控制反转）
        </strong>
        和
        <strong>
         DI（依赖注入）
        </strong>
        核心功能，管理对象生命周期及依赖关系。
       </li>
       <li>
        <strong>
         功能
        </strong>
        ：支持事务管理、AOP（面向切面编程）、数据访问等，适用于所有 Java 应用（不限于 Web）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Spring MVC
      </strong>
      <ul>
       <li>
        <strong>
         定位
        </strong>
        ：Spring 的子框架，专注于
        <strong>
         Web 层开发
        </strong>
        ，基于 MVC（Model-View-Controller）模式。
       </li>
       <li>
        <strong>
         功能
        </strong>
        ：处理 HTTP 请求、路由分发、视图渲染（如 JSP、Thymeleaf）等，需依赖 Spring 核心。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Spring Boot
      </strong>
      <ul>
       <li>
        <strong>
         定位
        </strong>
        ：Spring 的“脚手架”，简化 Spring 应用的
        <strong>
         配置和开发流程
        </strong>
        。
       </li>
       <li>
        <strong>
         功能
        </strong>
        ：内嵌服务器（Tomcat/Jetty）、自动配置、Starter 依赖（快速集成其他技术），支持快速搭建微服务。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="2__18">
     </a>
     <strong>
      2. 关系
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       Spring 是基础
      </strong>
      ：提供 IoC 容器、AOP 等核心能力，是 Spring MVC 和 Spring Boot 的底层依赖。
     </li>
     <li>
      <strong>
       Spring MVC 是 Spring 的扩展
      </strong>
      ：专为 Web 开发设计，需在 Spring 基础上使用。
     </li>
     <li>
      <strong>
       Spring Boot 是 Spring 的增强工具
      </strong>
      ：通过简化配置和提供默认约定，加速 Spring 应用的开发，默认集成 Spring MVC。
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="3__26">
     </a>
     <strong>
      3. 核心区别
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         维度
        </strong>
       </th>
       <th align="left">
        <strong>
         Spring
        </strong>
       </th>
       <th align="left">
        <strong>
         Spring MVC
        </strong>
       </th>
       <th align="left">
        <strong>
         Spring Boot
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        <strong>
         核心功能
        </strong>
       </td>
       <td align="left">
        IoC/DI、AOP、事务管理
       </td>
       <td align="left">
        Web 层 MVC 架构支持
       </td>
       <td align="left">
        简化配置、快速开发、内嵌服务器
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         配置复杂度
        </strong>
       </td>
       <td align="left">
        高（需手动配置 XML/Java）
       </td>
       <td align="left">
        中等（需配合 Spring 使用）
       </td>
       <td align="left">
        低（约定优于配置，自动配置）
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         使用场景
        </strong>
       </td>
       <td align="left">
        所有 Java 应用（非 Web 也可）
       </td>
       <td align="left">
        Web 应用开发
       </td>
       <td align="left">
        快速搭建微服务、REST API、全栈应用
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         依赖关系
        </strong>
       </td>
       <td align="left">
        独立框架
       </td>
       <td align="left">
        依赖 Spring 核心
       </td>
       <td align="left">
        依赖 Spring，默认集成 Spring MVC
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h5>
     <a id="4__37">
     </a>
     <strong>
      4. 典型使用场景
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       Spring
      </strong>
      ：需要精细控制对象管理和依赖关系的场景，如企业级后台系统、复杂业务逻辑应用。
     </li>
     <li>
      <strong>
       Spring MVC
      </strong>
      ：传统 Web 应用开发（如电商平台、内容管理系统），需前后端混合渲染。
     </li>
     <li>
      <strong>
       Spring Boot
      </strong>
      ：快速构建独立运行的微服务、云原生应用、RESTful API，或需要内嵌服务器的项目。
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="5__45">
     </a>
     <strong>
      5. 常见误解澄清
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       Spring Boot ≠ 独立框架
      </strong>
      ：它是 Spring 的扩展工具，简化配置而非替代 Spring。
     </li>
     <li>
      <strong>
       Spring Boot 默认包含 Spring MVC
      </strong>
      ：引入
      <code>
       spring-boot-starter-web
      </code>
      依赖后，自动集成 Spring MVC。
     </li>
     <li>
      <strong>
       Spring MVC 可与其他框架结合
      </strong>
      ：如与 Thymeleaf、Freemarker 等视图层技术搭配，但核心依赖仍是 Spring。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      总结
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       Spring
      </strong>
      是地基，提供核心能力；
     </li>
     <li>
      <strong>
       Spring MVC
      </strong>
      是盖房子的工具，专注 Web 层；
     </li>
     <li>
      <strong>
       Spring Boot
      </strong>
      是装修队，让盖房子更快、更省心。
      <br/>
      三者协同工作，覆盖从基础架构到高效开发的全流程需求。
     </li>
    </ul>
