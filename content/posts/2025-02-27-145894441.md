    <h3 style="background-color:transparent; margin-left:0.0001pt; margin-right:0px; text-align:justify">
     前言
    </h3>
    <p>
     现在Git已经是一个很常用的工具了 ，工作中经常会用到，有时候面试也会问会不会使用git，所以特地写了这篇博客来为初学者讲解如何使用git。里面只涉及到简单的git操作，不过应付日常工作还是绰绰有余的。
    </p>
    <h3 style="background-color:transparent; margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <span style="color:#a2e043">
      <strong>
       1.
      </strong>
      Git简介
     </span>
    </h3>
    <p>
     Git是一个分布式版本控制系统，用于跟踪计算机文件的更改，特别是在源代码中，它是由Linus Torvalds 在 2005 年创立，最初是为了支持 Linux 内核开发。
    </p>
    <p>
     工作流程：
    </p>
    <p>
     <img alt="" height="392" src="https://i-blog.csdnimg.cn/direct/82e3fd2efa204160b3e751008fe3a597.png" width="1135"/>
    </p>
    <p>
     workspace:工作区
    </p>
    <p>
     Index/Stage:暂存区
    </p>
    <p>
     Repository:本地仓库
    </p>
    <p>
     Remote:远程仓库
    </p>
    <h3>
     <span style="color:#a2e043">
      2.安装git
     </span>
    </h3>
    <p>
     windows中:从网上下载一个，按照默认的方式进行安装即可。安装完成以后右键显示有Git Bash here即表示安装成功。
    </p>
    <p>
     <img alt="" height="208" src="https://i-blog.csdnimg.cn/direct/abc1c15f218442b1b25f048ff7b6d95d.png" width="257"/>
    </p>
    <p>
     （ps：如果想偷懒或者找不到如何下载，可以直接找我来拿安装包）
    </p>
    <p>
     配置用户名和邮箱
    </p>
    <pre><code class="language-bash">git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

git config --list #可以查看当前git的配置    
#--global表示配置的是全局的</code></pre>
    <p>
     注意：这个配置只是一种标识，标记你从这里上传的代码。
    </p>
    <pre><code class="language-bash">为当前仓库指定用户名和邮箱
1.打开终端或者命令行，导航到对应的Git仓库
cd /path/to/your/repo
2.设置用户名
git config user.name "Your Name"
3.设置邮箱
git config user.emial "your.email@example.com"</code></pre>
    <p>
     注意配置成功以后，使用 git config --list进行查看的时候，user.name和user.email是最后的那个才是该仓库的，之前全局的那个应该是空的，需要翻到最下面才能看到。
    </p>
    <h3>
     <span style="color:#a2e043">
      3.创建版本库（代码仓库）
     </span>
    </h3>
    <p>
     版本库就是版本仓库或者也可以叫做仓库，顾名思义，它就是装东西的，它和普通的仓库不一样的地方是：它是用来装文件的。
    </p>
    <p>
     那这个仓库是如何管理文件的呢，当然是通过git这个工具了，至于其中的原理这里就不做过多的解释，有兴趣的朋友可以看看这个去深入的了解
     <a href="https://git-scm.com/book/en/v2" rel="nofollow" title="Git">
      Git
     </a>
     。
    </p>
    <p>
     现在我想要在D盘的code\test目录下创建一个版本库，该如何做呢？
    </p>
    <p>
     首先需要进入code\test目录,然后唤出git的命令行工具。
    </p>
    <p>
     <img alt="" height="405" src="https://i-blog.csdnimg.cn/direct/9a4950b6ed774f32933d6885c7f13d27.png" width="284"/>
    </p>
    <p>
     使用git init命令即可创建一个新的版本库/代码仓(后续都会将之称为代码仓)。
    </p>
    <p>
     <img alt="" height="71" src="https://i-blog.csdnimg.cn/direct/50054f4b11174b17bc8421b0ea4cc3be.png" width="558"/>
    </p>
    <p>
     这时候，你会看到test文件夹下会多出一个新的目录，.git目录，这个目录就是用来管理刚刚新建的这个代码仓的。
    </p>
    <p>
     <img alt="" height="94" src="https://i-blog.csdnimg.cn/direct/6f8528c072534574863f3744e6d79be8.png" width="500"/>
    </p>
    <p>
     刚刚创建好以后的代码仓库是空的，这时候就可以使用命令git add和git commit为添加文件进行版本控制了。
    </p>
    <p>
     如下，加上现在添加一个test1.txt文件。
    </p>
    <p>
     <img alt="" height="167" src="https://i-blog.csdnimg.cn/direct/6e524890546b47dda93b4926b9d6392d.png" width="550"/>
    </p>
    <p>
     现在test.txt文件已经添加到本地仓库了。Git会追踪这个文件的更改。
    </p>
    <p>
     git status 命令可以显示当前代码仓的状态。它会告诉你下面的这些信息：
    </p>
    <p>
     1）当前分支，显示当前处于哪一个分支
    </p>
    <p>
     2）文件的状态
    </p>
    <p>
     哪些文件已修改，但
     <span style="color:#fe2c24">
      未暂存(stage红色字体)
     </span>
     。
    </p>
    <p>
     哪些文件
     <span style="color:#fe2c24">
      已修改，且暂存（绿色字体）
     </span>
     。
    </p>
    <p>
     哪些文件
     <span style="color:#fe2c24">
      未跟踪(untracked)
     </span>
     。
    </p>
    <p>
     3）提出建议，会给出执行下一步的建议
    </p>
    <p>
     例如再新建一个test1文件：
    </p>
    <p>
     <img alt="" height="128" src="https://i-blog.csdnimg.cn/direct/08d014989a5646bd943d062ccb500a69.png" width="582"/>
    </p>
    <p>
     执行git add test1.txt 以后再查看：

    <p>

    <p>
     现在test1.txt文件已经在暂存区了，如果想要将它提交的本地仓库只需要使用git commit命令，如果想要将从暂存区移除，放回到工作区，只需要使用git restore --staged 命令即可。

    <p>

    <h3>
     <span style="color:#a2e043">
      4.常用的git命令
     </span>
    </h3>
    <pre><code class="language-bash">git clone #克隆远程仓库到本地
git init #初始化一个仓库
git remote add origin &lt;远程仓库的URL&gt; #关联本地仓库和远程仓库
git remote -v #查看关联结果
git config user.name "your name" #为当前仓库添加用户名
git config user.email "your *@exmple.com" #为当前仓库添加邮箱
git add #将文件或目录添加但暂存区
git commit #将暂存区的更改添加但本地仓库
git push #将本地仓库的新增内容推送到远程-&gt;用于同步操作
git pull #拉取远程仓库的内容-&gt;用于同步操作

git log #查看仓库的提交日志
git reset #回退提交
    --soft &lt;commit_hash&gt; #软回头，保存暂存区和工作区的更改，会报保存历史提交信息
    --mixed &lt;commit_hash&gt; #混合回退，回退到某个提交，并取消暂存区的更改，但保留工作区文件的修改
    --hard &lt;commit_hash&gt; #硬回退，回退到某个提交，并丢弃所有更改（包括暂存区和工作区的更改）
git revert #回退特定提交，不改变历史。与git reset不同，git revert会生成一个新的提交来撤销某次提交的修改，适用于不想修改历史情况，尤其是与他人协作时
git checkout #用法很多
    git checkout &lt;commit_hash&gt; -- &lt;fileName&gt; #恢复某个特定文件或者文件夹到某个提交的版本
    git checkout -b &lt;branch_name&gt; #切换到某个分支
    git checkout &lt;commit_hash&gt; -- .#恢复所有的文件到某个提交状态，
    git checkout -- &lt;file_path&gt; #撤销某个文件未提交的更改 ，git checkout -- . 撤销所有文件未提交的更改
git restore #和git checkout用法类似
    git restore --source=&lt;commit_hash&gt; &lt;file_path&gt; #恢复特定文件到某个提交的版本
    git restore --source=&lt;commit_hash&gt; ./ #恢复所有文件到某个提交版本
    git restore &lt;file_path&gt; #撤销工作区的更改
    git restore .           #撤销所有工作区的更改
    git restore --staged &lt;file&gt; #将文件从暂存区移除
    
#分支相关的命令
git branch #查看本地所有的分支
git branch -r #查看远程所有的分支
git branch -a #查看所有的分支
git branch &lt;name_branch&gt; #新建name_branch分支
git checkout &lt;name_branch&gt; #切换到name_branch分支 
git checkout -b &lt;name_branch&gt; #新建并切换到name_branch分支
git checkout -d/-D #删除分支，-d是删除已经合并的分支，-D强制删除一个分支，即使它还未合并
git merge &lt;name_branch&gt; #将name_branch分支合并到当前分支，合并时，如果出现冲突需要手动解决冲突
git log --graph --oneline --all #查看分支的合并历史
git push origin &lt;name_branch&gt; #将name_branch分支推送到远程
git branch --set-upstream-to=origin/&lt;name_branch&gt; &lt;name_branch&gt; #设置远程分支和本地分支分支的跟踪关系，以后在这个分支上提交代码，push和pull的时候就不需要指定远程分支名称
git branch -vv #查看本地分支和远程分支的关联信息
git stash #将当前工作区和暂存区的更改保存起来，保存到一个栈中
    -k #只暂存工作区的更改
    save "message" 添加暂存信息
    list #查看所以已保存的stash列表
    apply #恢复最近一次的暂存
    pop #恢复最近一次的暂存，并且在栈中删除这次暂存
    drop #删除某个特定的stash
    clear #删除所有的stash
git ls-files #查看当前仓库中被纳入版本控制的文件
#注意这些命令并不需要死记，多敲几遍，知道在哪个场景下需要哪些命令即可，还可以使用git help去查看git的帮助手册,其实记住最后一条就够了</code></pre>

    <h4 style="background-color:transparent">
     <span style="color:#38d8f0">
      5.1查看文件改动 git diff

    <p>
     如何查看test1.txt文件发生了哪些改动呢？
    </p>
    <p>
     可以使用diff命令

     <span style="color:#38d8f0">
      5.3工作区和暂存区的理解
     </span>
    </h4>
    <p>
     工作区就相当于版本库目录下除了.git目录以外所有的文件目录和文件。暂存区在版本库下隐藏的
     <span style="color:#fe2c24">
      .git/index文件
     </span>
     中。
    </p>
    <p>
     我们是通过命令来操作这两个地方的。git add命令会将工作区文件中的更改添加到暂存区，
     <span style="color:#fe2c24">
      git commit
     </span>
     命令会将暂存区的文件上传到本地仓库。举个🥔，如下:
    </p>
    <p>
     <img alt="" height="477" src="https://i-blog.csdnimg.cn/direct/3dde8b7a14a04a24aee67aa93df9bc23.png" width="744"/>
    </p>
    <h4 style="background-color:transparent">
     <span style="color:#38d8f0">
      5.4撤销修改，删除文件，将文件移除版本控制。
     </span>
    </h4>
    <p>
     此时test1.txt文件中的内容有三行，如图:
    </p>
    <p>
     <img alt="" height="77" src="https://i-blog.csdnimg.cn/direct/71855cfeccba481b9c06a4f5b7c711f5.png" width="148"/>

    <p>
     如果给这个文件再新增一行 "test"文本，它就是变成四行文本。此时我们想要撤销这次修改，有以下几种方式：
    </p>
    <p>
     第一种，如果我们知道要删除哪些内容直接打开文件编辑器进行删除即可。
    </p>
    <p>
     第二种，可以直接使用版本回退(硬回退)，直接恢复到上一个版本，并丢弃工作区的更改。
    </p>
    <p>
     第三种，使用
     <span style="color:#fe2c24">
      git checkout --
     </span>
     test1.txt,丢弃test1.txt文件工作区的修改(命令中的-- 别丢)

      git checkout --

     只能丢弃文件在工作区的更改，如果文件的更改已经添加到暂存区了，改怎么撤销暂存呢？

     如果想要删除文件，该如何操作呢，可以直接删除。

    <h4 style="background-color:transparent">
     <span style="color:#38d8f0">
      5.5分支操作
     </span>
    </h4>
    <p>
     当我们初始化一个仓库以后就会存在一个master分支。在仓库中存在一个HEAD指针，它指向的就是当前所在的分支。当前指向的是master分支。
    </p>
    <p>
     <img alt="" height="283" src="https://i-blog.csdnimg.cn/direct/ab19a23f29c24058b1aa54b3040426e6.png" width="478"/>
    </p>
    <p>
     使用
     <span style="color:#fe2c24">
      git branch
     </span>
     命令可以生成一个新的分支。
    </p>
    <p>
     <img alt="" height="206" src="https://i-blog.csdnimg.cn/direct/e3a265d3eaa341da832b75e2e219808d.png" width="671"/>
    </p>
    <p>
     通过
     <span style="color:#fe2c24">
      git checkout
     </span>
     命令 可以切换分支。
    </p>
    <p>
     <img alt="" height="131" src="https://i-blog.csdnimg.cn/direct/e76daea20b6e4bb98cddeeb6cc2b21e1.png" width="730"/>
    </p>
    <p>
     在新的分支上，也可以进行开发工作，并且提交代码到仓库中。
    </p>
    <p>
     <img alt="" height="154" src="https://i-blog.csdnimg.cn/direct/90df2ce3367e40f8a10a21994a323e15.png" width="674"/>
    </p>

     此时再次切换回master分支。
    </p>

     再次查看master分支的提交记录：
    </p>

    <p>
     <span style="color:#fe2c24">
      其实将这些串联起来就是一条时间线对应的事件

    </p>
    <p>

    <p>
     可以看到这两个分支的提交记录有一部分是相同的但是最近几条是不一样的，
     <span style="color:#fe2c24">
      因为新建的分支debug在master分支下新建的，所以会克隆master分支中的信息。master分支也叫基线分支，是一个项目中最稳定最成熟的分支。
     </span>
     如果现在想要将debug分支合并到master分支改如何操作呢？可以直接使用分支合并命令：
     <span style="color:#fe2c24">
      git merge
     </span>
     。合并分支时，可能会存在冲突。
    </p>
    <p>
     <img alt="" height="237" src="https://i-blog.csdnimg.cn/direct/6e6a73f71e5b409e8d7d25c4e1fa1ae3.png" width="835"/>
