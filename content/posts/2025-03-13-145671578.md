    <h3>
     <a id="_vim__vim_3">
     </a>
     使用 vim 编辑器命令 vim
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      vim [选项] [文件名]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：选择 vim 的打开方式
     <br/>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -R：以只读模式打开文件
     </li>
     <li>
      -r：恢复崩溃前的文件（用于恢复未保存的修改）
     </li>
     <li>
      +[行号]：打开文件并跳转到指定行的行号
     </li>
     <li>
      +[“命令”]：启动 vim 时 vim 可直接执行双引号中的 命令
     </li>
     <li>
      -c ‘命令’ ：启动后执行 vim 命令，同上
     </li>
     <li>
      <code>
       vim -d [文件名1] [文件名2]
      </code>
      ：对比两个文件（进入 diff 模式）
     </li>
     <li>
      --noplugin ：禁用插件加载
     </li>
     <li>
      -u NONE ：不加载任何配置文件（安全模式）
     </li>
     <li>
      <code>
       vim -c "oldfiles
      </code>
      ：显示最近打开的文件列表
     </li>
     <li>
      -x：创建或编辑加密文件（首次需设置密码）
     </li>
     <li>
      -X：禁用与 X Server 的连接（加速启动）
     </li>
     <li>
      -b：以二进制模式打开文件
     </li>
    </ul>
    <p>
     <strong>
      其它操作：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       vim [文件名1] [文件名2] [文件名3] ...
      </code>
      同时打开多个文件
     </li>
     <li>
      不写文件名默认打开一个新文件
     </li>
    </ul>
    <p>
     关于 vim 更详细介绍请参考：
     <a href="https://blog.csdn.net/qq3304968099/article/details/145680515?spm=1001.2014.3001.5501">
      (学习总结25)Linux工具：vim 编辑器 和 gcc/g++ 编译器
     </a>

    <h3>
     <a id="_gccg__gccg_37">
     </a>
     使用 gcc/g++ 编译器命令 gcc/g++
    </h3>
    <p>
     <strong>
      语法：
     </strong>
     <br/>
     <code>
      gcc [选项] [源文件] ... -o [输出文件]
     </code>
     <br/>
     <code>
      g++ [选项] [源文件] ... -o [输出文件]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：gcc 将 .c 文件编译成可执行文件，g++ 将 .cpp 文件编译成可执行文件。
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -c ：只编译不链接，生成目标文件（.o）
     </li>
     <li>
      -g ：生成调试信息，用于 gdb 调试
     </li>
     <li>
      -Wall ：启用所有警告信息
     </li>
     <li>
      -O1/-O2/-O3 ：优化级别（1 为基本优化，3 为最高优化）
     </li>
     <li>
      -I [目录] ：指定目录搜索头文件
     </li>
     <li>
      -L [目录] ：指定目录搜索库文件
     </li>
     <li>
      -I（等价 shift+i） [库名] ：链接指定的库（如 -lm 链接数学库）
     </li>
     <li>
      -std=[标准] ：指定语言标准（如 -std=c11 或 -std=c++17）
     </li>
    </ul>
    <p>
     <strong>
      预处理选项：
     </strong>
    </p>
    <ul>
     <li>
      -E ：只进行预处理，输出到标准输出
     </li>
     <li>
      -D[宏] ：定义宏（如 -DDEBUG 定义 DEBUG 宏）
     </li>
     <li>
      -U[宏] ：取消定义宏
     </li>
     <li>
      -I[目录] ：添加头文件搜索路径
     </li>
     <li>
      -M：生成依赖关系（用于 Makefile）
     </li>
     <li>
      -MM ：生成依赖关系，忽略系统头文件
     </li>
    </ul>
    <p>
     <strong>
      编译选项：
     </strong>
    </p>
    <ul>
     <li>
      -S ：只编译到汇编代码，生成 .s 文件
     </li>
     <li>
      -fPIC ：生成位置无关代码（用于动态库）
     </li>
     <li>
      -fno-stack-protector ：禁用栈保护
     </li>
     <li>
      -fstack-protector ：启用栈保护
     </li>
     <li>
      -fomit-frame-pointer ：省略帧指针（优化性能）
     </li>
     <li>
      -pipe ：使用管道代替临时文件（加快编译速度）
     </li>
    </ul>
    <p>
     <strong>
      优化选项：
     </strong>
    </p>
    <ul>
     <li>
      -O0 ：不优化（默认）
     </li>
     <li>
      -O1 ：基本优化
     </li>
     <li>
      -O2 ：更多优化（推荐）
     </li>
     <li>
      -O3 ：最高优化
     </li>
     <li>
      -Os ：优化代码大小
     </li>
     <li>
      -Ofast ：激进优化（可能不符合标准）
     </li>
     <li>
      -flto ：启用链接时优化
     </li>
    </ul>
    <p>
     <strong>
      调试选项：
     </strong>
    </p>
    <ul>
     <li>
      -g ：生成调试信息
     </li>
     <li>
      -ggdb ：生成 GDB 专用调试信息
     </li>
     <li>
      -g3 ：生成更多调试信息（如宏定义）
     </li>
     <li>
      -p ：生成性能分析信息（用于 gprof）
     </li>
     <li>
      -pg ：生成性能分析信息（用于 gprof）
     </li>
    </ul>
    <p>
     <strong>
      链接选项：
     </strong>
    </p>
    <ul>
     <li>
      -static ：静态链接
     </li>
     <li>
      -shared ：生成共享库（动态库）
     </li>
     <li>
      -L[目录] ：添加库文件搜索路径
     </li>
     <li>
      -l[库名] ：链接指定库（如 -lm 链接数学库）
     </li>
     <li>
      -nostdlib ：不链接标准库
     </li>
     <li>
      -nodefaultlibs ：不链接默认库
     </li>
     <li>
      -pie ：生成位置无关可执行文件
     </li>
    </ul>
    <p>
     <strong>
      警告选项：
     </strong>
    </p>
    <ul>
     <li>
      -w ：不生成任何警告信息
     </li>
     <li>
      -Wall ：启用所有常见警告
     </li>
     <li>
      -Wextra ：启用额外警告
     </li>
     <li>
      -Werror ：将警告视为错误
     </li>
     <li>
      -Wno-[警告] ：禁用特定警告
     </li>
     <li>
      -pedantic ：严格遵循标准
     </li>
     <li>
      -pedantic-errors ：严格遵循标准并将警告视为错误
     </li>
    </ul>
    <p>
     <strong>
      语言标准选项：
     </strong>
    </p>
    <ul>
     <li>
      -std=c89 ：使用 C89 标准
     </li>
     <li>
      -std=c99 ：使用 C99 标准
     </li>
     <li>
      -std=c11 ：使用 C11 标准
     </li>
     <li>
      -std=c++98 ：使用 C++98 标准
     </li>
     <li>
      -std=c++11 ：使用 C++11 标准
     </li>
     <li>
      -std=c++14 ：使用 C++14 标准
     </li>
     <li>
      -std=c++17 ：使用 C++17 标准
     </li>
     <li>
      -std=c++20 ：使用 C++20 标准
     </li>
    </ul>
    <p>
     <strong>
      其它选项：
     </strong>
    </p>
    <ul>
     <li>
      -v ：显示编译过程详细信息
     </li>
     <li>
      -### ：显示但不执行编译命令
     </li>
     <li>
      -save-temps ：保存临时文件（如 .i、.s、.o）
     </li>
     <li>
      -march=[架构] ：指定目标架构（如 -march=native）
     </li>
     <li>
      -mtune=[架构] ：优化目标架构（如 -mtune=native）
     </li>
    </ul>
    <p>
     关于 gcc/g++ 更详细介绍请参考：
     <a href="https://blog.csdn.net/qq3304968099/article/details/145680515?spm=1001.2014.3001.5501">
      (学习总结25)Linux工具：vim 编辑器 和 gcc/g++ 编译器

    <h3>
     <a id="_make_138">
     </a>
     使用自动化构建命令 make
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      make [选项] [目标]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：执行 Makefile 文件中的目标
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -f [文件名] ：指定使用的 Makefile 文件名。默认情况下，make 会在当前目录下查找名为 Makefile 或 makefile 的文件
     </li>
     <li>
      -n ：只打印要执行的命令，但不实际执行
     </li>
     <li>
      -s ：安静模式，不显示执行的命令，只显示执行结果
     </li>
     <li>
      -B ：强制重新构建所有目标
     </li>
     <li>
      -C [目录路径] ：切换到指定目录后执行 make
     </li>
     <li>
      -k 出错时继续执行其他目标
     </li>
     <li>
      -j [并行任务数] ：指定并行执行的任务数，加快编译速度
     </li>
    </ul>
    <p>
     <strong>
      其它操作：
     </strong>
    </p>
    <ul>
     <li>
      不指定文件名，make 默认会在当前目录搜索并使用名为 Makefile 或 makefile 的文件。
     </li>
    </ul>
    <p>
     关于 make 自动化构建更详细介绍请参考：
     <a href="https://blog.csdn.net/qq3304968099/article/details/145709531?spm=1001.2014.3001.5501">
      (学习总结26)Linux工具：make/Makefile 自动化构建、Git 版本控制器 和 gdb/cgdb 调试器

    <h3>
     <a id="_read_165">
     </a>
     读取输入并赋值给变量命令 read
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      read [选项] [变量名]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：从标准输入（通常是键盘）读取用户输入并将其赋值给变量的命令，通常用于脚本中与用户交互，获取数据。
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       -p "提示信息"
      </code>
      ：在读取输入前显示提示信息
     </li>
     <li>
      -s ：静默模式，输入内容不会显示在屏幕上（常用于密码输入）
     </li>
     <li>
      <code>
       -t [秒数]
      </code>
      ：设置超时时间，超过指定时间未输入则命令退出
     </li>
     <li>
      <code>
       -n [字符数]
      </code>
      ：限制输入的字符数，达到指定字符数后自动结束输入
     </li>
     <li>
      <code>
       -a [数组名]
      </code>
      ：将输入的内容按空格分隔并存入数组
     </li>
     <li>
      -r ：禁止反斜杠
      <code>
       \
      </code>
      转义字符
     </li>
    </ul>
    <p>
     其它操作：
    </p>
    <ul>
     <li>
      如果未指定变量名，read 命令会将输入内容存储到默认变量
      <code>
       REPLY
      </code>
      中
     </li>
    </ul>
    <h3>
     <a id="_shell__set_182">
     </a>
     设置或修改 shell 环境命令 set
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      set [选项] [参数]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：用于设置或修改 Shell 环境，它可以用来控制 shell 的特性、显示变量、设置位置参数
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -e ：如果命令返回非零退出状态（失败），立即退出脚本
     </li>
     <li>
      -x ：打印执行的每一行命令及其结果（用于调试脚本）
     </li>
     <li>
      -u ：使用未定义的变量时，报错并退出脚本
     </li>
     <li>
      -o ：启用特定的 shell 选项（例如 nounset、errexit 等）
     </li>
     <li>
      +o ：禁用特定的 shell 选项
     </li>
     <li>
      -a ：自动导出所有定义的变量（相当于 export）
     </li>
     <li>
      -v ：打印读取的每一行输入（用于调试）
     </li>
     <li>
      -n ：读取命令但不执行（用于检查脚本语法）
     </li>
    </ul>
    <p>
     其它操作：
    </p>
    <ul>
     <li>
      不写选项和参数时，会显示当前 shell 环境中所有已定义的变量和函数，包括环境变量、局部变量和 shell 函数
     </li>
    </ul>

     </li>
    </ul>
    <h3>
     <a id="_ping_212">
     </a>
     测试网络连接命令 ping
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      ping [选项] [目标主机]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：发送
     <strong>
      ICMP 回显请求
     </strong>
     数据包到目标主机，并等待目标主机返回回显应答，从而检测网络是否连通以及测量网络延迟
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      没有选项默认会一直发送，直到按
      <code>
       Ctrl+c
      </code>
      停止
     </li>
     <li>
      <code>
       -c [次数]
      </code>
      ：指定发送数据包的次数
     </li>
     <li>
      <code>
       -i [间隔]
      </code>
      ：设置发送数据包的时间间隔（单位为秒）
     </li>
     <li>
      <code>
       -s [大小]
      </code>
      ：设置发送的数据包大小（单位为字节）
     </li>
     <li>
      <code>
       -t [生存时间]
      </code>
      ：设置数据包的生存时间
     </li>

      </p>
     </li>

     ：
     <br/>
     <code>
      install [选项] [源文件] [目标文件]
     </code>
     <br/>
     <code>
      install [选项] [源文件1] [源文件2] ... [目标目录]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：安装脚本或文件到指定位置，并确保文件具有正确的权限和所有权。
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       -m [模式]
      </code>
      ：设置文件的权限模式
     </li>
     <li>
      <code>
       -o [用户]
      </code>
      ：设置文件的所有者
     </li>
     <li>
      <code>
       -g [组]
      </code>
      ：设置文件的所属组
     </li>
     <li>
      -d ：创建目录（类似于 mkdir -p）
     </li>
     <li>
      -T ：将目标视为文件而不是目录
     </li>
     <li>
      -v ：显示详细操作信息
     </li>
    </ul>
    <h3>
     <a id="_sort_452">
     </a>
     对文本文件或输入内容进行排序命令 sort
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      sort [选项] [文件名]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：对文本文件或输入内容进行排序。
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -r ：逆序排序（默认是升序）
     </li>
     <li>
      -n ：按数值大小排序（而不是按字母顺序）
     </li>
     <li>
      <code>
       -k [列数]
      </code>
      ：指定按某一列或字段排序（字段从 1 开始计数）
     </li>
     <li>
      <code>
       -t '字段'
      </code>
      ：指定字段分隔符（默认是空格）
     </li>
     <li>
      -u ：去重，仅输出唯一的行
     </li>
     <li>
      -f ：忽略大小写
     </li>
     <li>
      <code>
       -o [文件名]
      </code>
      ：将排序结果输出到指定文件（而不是标准输出）
     </li>
     <li>
      -c ：检查文件是否已排序，如果未排序则报错
     </li>
     <li>
      -b ：忽略行首的空白字符
     </li>
    </ul>
    <h3>
     <a id="_tac_467">
     </a>
     反向显示文件内容命令 tac
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      tac [选项] [文件名]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：反向显示文件内容，与 cat 命令相反。
    </p>
    <p>
     <strong>
      常用选项：
     </strong>
    </p>
    <ul>
     <li>
      -b 或 --before ：在行首（而非默认的行尾）添加分隔符（需配合 -s 使用）
     </li>
     <li>
      -r 或 --regex ：将分隔符视为正则表达式（默认分隔符为换行符 \n）
     </li>
     <li>
      <code>
       -s [分隔符]
      </code>
      ：指定自定义行分隔符（默认按换行符 \n 分块）
     </li>
    </ul>
    <h3>
     <a id="_printf_477">
     </a>
     格式打印命令 printf
    </h3>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      printf "格式字符串" [参数1] [参数2] ...
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：将规定的参数结构化输出到屏幕，类似于 C 语言的 printf 函数。
    </p>
    <p>
     常用格式符说明：
    </p>
    <ul>
     <li>
      %s ： 表示字符串
     </li>
     <li>
      %d ：表示整数（十进制）
     </li>
     <li>
      %f ：浮点数（默认保留 6 位小数）
     </li>
     <li>
      %x ：16 进制整数（小写字母）
     </li>
     <li>
      %X ：16 进制整数（大写字母）
     </li>
     <li>
      %% ：输出 ’ % ’ 符号
     </li>
    </ul>
