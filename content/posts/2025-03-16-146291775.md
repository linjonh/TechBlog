     于 Hexo + GitHub Pages 的搭建个人博客网站的详细教程

    <p>
     以下是快速搭建个人博客网站的详细教程，基于
     <strong>
      Hexo + GitHub Pages
     </strong>
     的免费方案，无需服务器和域名即可快速上线，适合零基础用户。
    </p>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     一、准备工作
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        注册GitHub账号
       </strong>
       <br/>
       访问
       <a href="https://github.com/">
        GitHub官网
       </a>
       ，注册账号。后续将通过GitHub Pages托管博客。
      </p>
     </li>
     <li>
      <p>
       <strong>
        安装Git
       </strong>
      </p>
      <ul>
       <li>
        Windows/Mac用户：从
        <a href="https://git-scm.com/downloads" rel="nofollow">
         Git官网
        </a>
        下载安装包，按提示安装。
       </li>
       <li>
        安装完成后，终端输入
        <code>
         git --version
        </code>
        验证是否成功。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        安装Node.js
       </strong>
      </p>
      <ul>
       <li>
        从
        <a href="https://nodejs.org/" rel="nofollow">
         Node.js官网
        </a>
        下载LTS版本安装包。安装完成后，终端输入
        <code>
         node -v
        </code>
        和
        <code>
         npm -v
        </code>
        检查版本。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="Hexo_17">
     </a>
     二、安装并初始化Hexo
    </h4>
    <p>
     Hexo是一个基于Node.js的静态博客框架，支持Markdown写作，生成速度快。
    </p>
    <ol>
     <li>
      <p>
       <strong>
        安装Hexo
       </strong>
       <br/>
       终端执行以下命令：
      </p>
      <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> hexo-cli
</code></pre>
      <ul>
       <li>
        若因网络问题安装失败，可切换为国内镜像：
        <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> registry https://registry.npm.taobao.org
</code></pre>
       </li>
       <li>
        安装成功后，输入
        <code>
         hexo -v
        </code>
        验证。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        初始化博客项目
       </strong>
      </p>
      <pre><code class="prism language-bash">hexo init myblog  <span class="token comment"># 创建名为myblog的文件夹</span>
<span class="token builtin class-name">cd</span> myblog
<span class="token function">npm</span> <span class="token function">install</span>
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_39">
     </a>
     三、配置博客基本信息
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        修改站点配置
       </strong>
       <br/>
       打开
       <code>
        myblog/_config.yml
       </code>
       文件，修改以下字段：
      </p>
      <pre><code class="prism language-yaml"><span class="token key atrule">title</span><span class="token punctuation">:</span> 我的博客         <span class="token comment"># 博客标题</span>
<span class="token key atrule">subtitle</span><span class="token punctuation">:</span> 记录技术点滴    <span class="token comment"># 副标题</span>
<span class="token key atrule">author</span><span class="token punctuation">:</span> 你的名字         <span class="token comment"># 作者名</span>
<span class="token key atrule">language</span><span class="token punctuation">:</span> zh<span class="token punctuation">-</span>CN        <span class="token comment"># 语言</span>
<span class="token key atrule">timezone</span><span class="token punctuation">:</span> Asia/Shanghai <span class="token comment"># 时区</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        更换主题（可选）
       </strong>
      </p>
      <ul>
       <li>
        从
        <a href="https://hexo.io/themes/" rel="nofollow">
         Hexo主题库
        </a>
        选择主题（如Butterfly、Next）。
       </li>
       <li>
        终端执行
        <code>
         git clone 主题仓库地址 themes/主题名
        </code>
        ，然后在
        <code>
         _config.yml
        </code>
        中修改
        <code>
         theme: 主题名
        </code>
        。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_56">
     </a>
     四、创建并发布文章
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        生成新文章
       </strong>
      </p>
      <pre><code class="prism language-bash">hexo new <span class="token string">"我的第一篇博客"</span>
</code></pre>
      <ul>
       <li>
        文章Markdown文件位于
        <code>
         source/_posts
        </code>
        目录下，可编辑内容并添加Front-matter（如分类、标签）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        本地预览
       </strong>
      </p>
      <pre><code class="prism language-bash">hexo clean <span class="token operator">&amp;&amp;</span> hexo generate  <span class="token comment"># 生成静态文件</span>
hexo server                  <span class="token comment"># 启动本地服务</span>
</code></pre>
      <ul>
       <li>
        访问
        <code>
         http://localhost:4000
        </code>
        查看效果。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="GitHub_Pages_72">
     </a>
     五、部署到GitHub Pages
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        创建GitHub仓库
       </strong>
      </p>
      <ul>
       <li>
        仓库名必须为
        <code>
         用户名.github.io
        </code>
        （如
        <code>
         AI-Green.github.io
        </code>
        ），选择Public可见性。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        配置Hexo部署插件
       </strong>
      </p>
      <ul>
       <li>
        安装部署插件：
        <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span> hexo-deployer-git <span class="token parameter variable">--save</span>
</code></pre>
       </li>
       <li>
        修改
        <code>
         _config.yml
        </code>
        文件末尾的部署配置：
        <pre><code class="prism language-yaml"><span class="token key atrule">deploy</span><span class="token punctuation">:</span>
  <span class="token key atrule">type</span><span class="token punctuation">:</span> git
  <span class="token key atrule">repo</span><span class="token punctuation">:</span> git@github.com<span class="token punctuation">:</span>用户名/用户名.github.io.git  <span class="token comment"># 替换为你的仓库地址</span>
  <span class="token key atrule">branch</span><span class="token punctuation">:</span> main  <span class="token comment"># 分支名可能是main或master</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        部署到GitHub
       </strong>
      </p>
      <pre><code class="prism language-bash">hexo clean <span class="token operator">&amp;&amp;</span> hexo generate <span class="token operator">&amp;&amp;</span> hexo deploy
</code></pre>
      <ul>
       <li>
        完成后访问
        <code>
         https://用户名.github.io
        </code>
        即可看到在线博客。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_97">
     </a>
     六、进阶优化
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        绑定自定义域名
       </strong>
      </p>
      <ul>
       <li>
        在域名服务商（如阿里云）添加CNAME记录指向
        <code>
         用户名.github.io
        </code>
        。
       </li>
       <li>
        在博客的
        <code>
         source
        </code>
        目录下创建
        <code>
         CNAME
        </code>
        文件，写入域名（如
        <code>
         blog.example.com
        </code>
        ），重新部署。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        加速国内访问
       </strong>
      </p>
      <ul>
       <li>
        同步部署到Gitee或Coding Pages，利用国内CDN加速。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        SEO优化
       </strong>
      </p>
      <ul>
       <li>
        在
        <code>
         _config.yml
        </code>
        中设置关键词（
        <code>
         keywords
        </code>
        ）。
       </li>
       <li>
        安装
        <code>
         hexo-generator-sitemap
        </code>
        插件生成站点地图。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_111">
     </a>
     常见问题
    </h4>
    <ul>
     <li>
      <strong>
       部署失败
      </strong>
      ：检查SSH密钥是否配置（通过
      <code>
       ssh -T git@github.com
      </code>
      验证）。
     </li>
     <li>
      <strong>
       页面样式异常
      </strong>
      ：执行
      <code>
       hexo clean
      </code>
      清除缓存后重新生成。
     </li>
     <li>
      <strong>
       文章不显示
      </strong>
      ：确认Markdown文件的Front-matter格式正确。
     </li>
    </ul>
    <p>
     通过以上步骤，20分钟内即可完成个人博客的搭建。如需更个性化功能（如评论系统、数据分析），可参考Hexo插件库扩展。
    </p>
