    <p>
     在计算机网络中，
     <strong>
      Access端口
     </strong>
     和
     <strong>
      Trunk端口
     </strong>
     是两种常见的交换机端口类型，主要用于VLAN（虚拟局域网）的配置。它们的主要区别在于如何处理VLAN标签和数据帧。
    </p>
    <hr/>
    <h4>
     <a id="1_Access_4">
     </a>
     <strong>
      1. Access端口
     </strong>
    </h4>
    <p>
     Access端口通常用于连接终端设备（如PC、打印机、IP电话等），这些设备不需要识别VLAN标签。
    </p>
    <h5>
     <a id="_7">
     </a>
     <strong>
      特点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       数据帧处理
      </strong>
      ：
      <ul>
       <li>
        发送数据时：去掉VLAN标签，发送普通以太网帧。
       </li>
       <li>
        接收数据时：为接收到的数据帧打上端口的PVID（Port VLAN ID）标签。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       VLAN配置
      </strong>
      ：
      <ul>
       <li>
        只能属于一个VLAN。
       </li>
       <li>
        默认情况下，Access端口的PVID是VLAN 1。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       典型应用
      </strong>
      ：
      <ul>
       <li>
        连接终端设备（如PC、打印机）。
       </li>
       <li>
        连接不需要识别VLAN标签的设备。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_18">
     </a>
     <strong>
      配置示例
     </strong>
    </h5>
    <p>
     以下是一个将端口配置为Access端口的示例（以华为设备为例）：
    </p>
    <pre><code class="prism language-bash">interface GigabitEthernet0/0/1
 port link-type access
 port default vlan <span class="token number">10</span>
</code></pre>
    <ul>
     <li>
      将端口
      <code>
       GigabitEthernet0/0/1
      </code>
      配置为Access端口，并分配到VLAN 10。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_Trunk_29">
     </a>
     <strong>
      2. Trunk端口
     </strong>
    </h4>
    <p>
     Trunk端口通常用于连接交换机、路由器或其他支持VLAN的设备，用于传输多个VLAN的数据。
    </p>
    <h5>
     <a id="_32">
     </a>
     <strong>
      特点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       数据帧处理
      </strong>
      ：
      <ul>
       <li>
        发送数据时：保留VLAN标签（除非是Native VLAN）。
       </li>
       <li>
        接收数据时：根据VLAN标签将数据帧转发到相应的VLAN。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       VLAN配置
      </strong>
      ：
      <ul>
       <li>
        可以传输多个VLAN的数据。
       </li>
       <li>
        需要配置允许通过的VLAN列表。
       </li>
       <li>
        有一个Native VLAN，用于传输不带标签的数据帧。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       典型应用
      </strong>
      ：
      <ul>
       <li>
        连接交换机之间的链路。
       </li>
       <li>
        连接路由器的子接口（用于实现VLAN间路由）。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_44">
     </a>
     <strong>
      配置示例
     </strong>
    </h5>
    <p>
     以下是一个将端口配置为Trunk端口的示例（以华为设备为例）：
    </p>
    <pre><code class="prism language-bash">interface GigabitEthernet0/0/2
 port link-type trunk
 port trunk allow-pass vlan <span class="token number">10</span> <span class="token number">20</span>
 port trunk pvid vlan <span class="token number">1</span>
</code></pre>
    <ul>
     <li>
      将端口
      <code>
       GigabitEthernet0/0/2
      </code>
      配置为Trunk端口，允许VLAN 10和VLAN 20的数据通过，并设置Native VLAN为VLAN 1。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_Hybrid_56">
     </a>
     <strong>
      3. Hybrid端口
     </strong>
    </h4>
    <p>
     Hybrid端口是华为设备特有的一种端口类型，兼具Access端口和Trunk端口的特点。
    </p>
    <h5>
     <a id="_59">
     </a>
     <strong>
      特点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       数据帧处理
      </strong>
      ：
      <ul>
       <li>
        可以同时发送带标签和不带标签的数据帧。
       </li>
       <li>
        可以根据配置决定哪些VLAN的数据帧带标签，哪些不带标签。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       VLAN配置
      </strong>
      ：
      <ul>
       <li>
        可以配置多个VLAN。
       </li>
       <li>
        需要指定每个VLAN的标签行为（带标签或不带标签）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       典型应用
      </strong>
      ：
      <ul>
       <li>
        需要灵活控制VLAN标签的场景。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_69">
     </a>
     <strong>
      配置示例
     </strong>
    </h5>
    <p>
     以下是一个将端口配置为Hybrid端口的示例（以华为设备为例）：
    </p>
    <pre><code class="prism language-bash">interface GigabitEthernet0/0/3
 port link-type hybrid
 port hybrid pvid vlan <span class="token number">10</span>
 port hybrid untagged vlan <span class="token number">10</span>
 port hybrid tagged vlan <span class="token number">20</span>
</code></pre>
    <ul>
     <li>
      将端口
      <code>
       GigabitEthernet0/0/3
      </code>
      配置为Hybrid端口：
      <ul>
       <li>
        PVID为VLAN 10。
       </li>
       <li>
        VLAN 10的数据帧不带标签。
       </li>
       <li>
        VLAN 20的数据帧带标签。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="5__94">
     </a>
     <strong>
      5. 总结
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Access端口
      </strong>
      ：用于连接终端设备，只能传输一个VLAN的数据，数据帧不带标签。
     </li>
     <li>
      <strong>
       Trunk端口
      </strong>
      ：用于连接交换机或路由器，可以传输多个VLAN的数据，数据帧通常带标签。
     </li>
     <li>
      <strong>
       Hybrid端口
      </strong>
      ：华为设备特有，兼具Access和Trunk端口的特点，可以灵活控制VLAN标签。
     </li>
    </ul>
