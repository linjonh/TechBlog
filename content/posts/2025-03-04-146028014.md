    <h2>
     <a id="ffmpeg_0">
     </a>
     一、ffmpeg音视频处理基本流程
    </h2>
    <p>
     首先先看两条命令
    </p>
    <pre><code class="prism language-shell">ffmpeg <span class="token parameter variable">-i</span> <span class="token number">1</span>.mp4 <span class="token parameter variable">-acodec</span> copy <span class="token parameter variable">-vcodec</span> libx264 <span class="token parameter variable">-s</span> 1280x720 <span class="token number">2</span>.flv
ffmpeg <span class="token parameter variable">-i</span> <span class="token number">1</span>.mp4 <span class="token parameter variable">-acodec</span> copy <span class="token parameter variable">-vcodec</span> libx265 <span class="token parameter variable">-s</span> 1280x720 <span class="token number">3</span>.mkv
</code></pre>
    <ul>
     <li>
      -i :表示输入源，这里是1.mp4，是当前路径下的视频文件
     </li>
     <li>
      -acodec copy： 用于指定音频编码方式。copy 表示直接复制输入文件的音频流，不进行重新编码。这样可以节省处理时间，同时保留原始音频的质量和编码格式。
     </li>
     <li>
      -vcodec libx264/libx265：-vcodec 用于指定视频编码方式，这里使用libx264/libx265编码器进行H264/H265编码
     </li>
     <li>
      -s 1280x720：指定视频的分辨率，这里使用1280 x 720的分辨率
     </li>
     <li>
      .flv/.mkv：指定输出文件的容器（文件）格式，flv和mkv都是常见的音视频容器
     </li>
    </ul>
    <p>
     生成后的文件：
    </p>
    <p>
     使用ffprobe查看转换后的视频参数：
    </p>
    <pre><code class="prism language-shell">ffprobe <span class="token number">2</span>.flv
</code></pre>
    <p>
     这里的编码是H.264+AAC 分辨率是1200x720:
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/43a0e19e6f254d6a9997684a1460b726.png"/>
    </p>
    <pre><code class="prism language-shell">ffprobe <span class="token number">3</span>.mkv
</code></pre>
    <p>
     这里的hevc就是H265，音频使用AAC，分辨率是1280x720
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4f5d030df8ea4879a3fe3e2ea0729400.png"/>
    </p>
    <p>
     我们来看看具体的命令执行流程是怎么样的：
    </p>
    <ul>
     <li>
      输入文件 ： 例如这里的1.mp4
     </li>
     <li>
      编码数据包：例如这里的视频编码libx264
     </li>
     <li>
      解码后的数据帧：解码后得到相应的数据帧，进行后续处理
     </li>
     <li>
      处理数据帧：比如这里的分辨率转换1280x720，通常使用的是filter来处理
     </li>
     <li>
      编码数据包：把处理后的数据帧重新编码，例如libx265和libx265
     </li>
     <li>
      输出文件：比如这里的2.flv和3.mkv
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f4eea27254b54b33a6322fd6b5a09c2a.png"/>
    </p>
    <p>
     <em>
      如果采用copy模式就不需要指定编码器了
     </em>
    </p>
