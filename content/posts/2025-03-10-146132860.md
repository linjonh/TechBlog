    <h2 name="1.%20%E5%89%8D%E8%A8%80" style="background-color:transparent">
     1. 前言
    </h2>
    <p>
     之前博文所讲的 MyBatis SQL 操作, 都必须按照注解或者标签中定义的 SQL 严格执行, 一个注解/标签, 对应的是不同的 SQL 操作. 比如: 一个 select 注解中, 定义了 3 个字段, 那这个注解只能插入这三个列的值, 如果想要插入更多的列, 必须重新再写一个注解.
    </p>
    <p>
     而动态 SQL, 我们可以根据用户输入的值, 动态的进行字段的插入, 当用户输入这一字段的值时, 就插入这一字段. 当用户没有输入这一字段的值时, 就不插入这一字段.
     <strong>
      能够完成不同条件下, 不同的 SQL 拼接.
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       注意: 本文主要通过 XML 完成动态 SQL!!
      </strong>
     </p>
    </blockquote>
    <h2 id="2.%C2%A0%E5%8A%A8%E6%80%81%E6%8F%92%E5%85%A5" name="2.%C2%A0%E5%8A%A8%E6%80%81%E6%8F%92%E5%85%A5" style="background-color:transparent">
     2. 动态插入
    </h2>
    <h3 id="2.1%20if%20%E6%A0%87%E7%AD%BE%C2%A0" name="2.1%20if%20%E6%A0%87%E7%AD%BE%C2%A0" style="background-color:transparent">
     2.1 if 标签
    </h3>
    <p>
     通过 if 标签, 对传入的参数的值进行判断, 若未传入相关列的值, 则不拼接该列的 SQL 片段.
    </p>
    <p>
     如下图所示,
     <strong>
      只有当传入参数的值不为 null 时, 才会拼接该 SQL 片段.
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       注意: if 标签, 判断的是传入的参数的值(#{} 中的参数), 即 java 属性, 而不是数据库字段!!
      </strong>
     </p>
    </blockquote>
    <p>
     <img alt="" height="791" src="https://i-blog.csdnimg.cn/direct/d58b51b8d1f347f189f591e21c42733a.png" width="1586"/>
    </p>
    <p>
     因此, 我们就可以
     <strong>
      通过这一个 insert 标签对应的方法, 进行任意字段的插入:
     </strong>
    </p>
    <p>
     <img alt="" height="903" src="https://i-blog.csdnimg.cn/direct/d9035cd0db924541a07f96f5ebec46b7.png" width="1640"/>
    </p>
    <p>
     但是, 当对最后一个字段传入的值为 null 时, 错误就发生了:
    </p>
    <p>
     <img alt="" height="708" src="https://i-blog.csdnimg.cn/direct/04de066d67d7434a8d84666de75b71bc.png" width="1773"/>
    </p>
    <p>
     这是因为最后一个字段 gender 的值为 null, 导致 if 标签的判断为 false, 因此 gender 字段的 SQL 没有进行拼接, 导致倒数第二个字段 phone 后多了一个逗号(,), 拼接的内容是 "phone , "因此造成语法错误.
    </p>
    <p>
     此时, 就需要 trim 标签出马了~~
    </p>
    <h3 id="2.2%20trim%20%E6%A0%87%E7%AD%BE" name="2.2%20trim%20%E6%A0%87%E7%AD%BE" style="background-color:transparent">
     2.2 trim 标签
    </h3>
    <p>
     trim 标签中有以下属性:
    </p>
    <ol>
     <li>
      <strong>
       prefix: 表示整个语句块, 以 prefix 的值作为前缀
      </strong>
     </li>
     <li>
      <strong>
       suffix: 表示整个语句块, 以suffix的值作为后缀
      </strong>
     </li>
     <li>
      <strong>
       prefixOverrides: 表示整个语句块要去除掉的前缀
      </strong>
     </li>
     <li>
      <strong>
       suffixOverrides: 表示整个语句块要去除掉的后缀
      </strong>
     </li>
    </ol>
    <p>
     有 trim 标签, 就可以通过
     <strong>
      prefixOverrides/suffixOverrides 属性
     </strong>
     解决上文中由于没有拼接最后一个字段, 导致倒数第二个字段后拼接了逗号(,)的问题, 并且还可以通过
     <strong>
      prefix/suffix 属性
     </strong>
     将 SQL 中的 "(" 和 ")" 自动拼接到 SQL 中:
    </p>
    <p>
     <img alt="" height="841" src="https://i-blog.csdnimg.cn/direct/93311b10239a4f428d59e5ef6b495777.png" width="1609"/>
    </p>
    <p>
     运行观察结果:
    </p>
    <p>
     <img alt="" height="850" src="https://i-blog.csdnimg.cn/direct/0bd0347ed36d423d945b091b10ee9ee0.png" width="1719"/>
    </p>
    <pre><code class="language-java">&lt;insert id="insertUser4"&gt;
        insert into user_info
               &lt;trim prefix="(" suffix=")" prefixOverrides="," suffixOverrides=","&gt;
                   &lt;if test="username != null"&gt;
                       username,
                   &lt;/if&gt;
                   &lt;if test="password != null"&gt;
                       password,
                   &lt;/if&gt;
                   &lt;if test="age != null"&gt;
                       age,
                   &lt;/if&gt;
                   &lt;if test="phone != null"&gt;
                       phone,
                   &lt;/if&gt;
                   &lt;if test="gender != null"&gt;
                       gender
                   &lt;/if&gt;
               &lt;/trim&gt;
        values
            &lt;trim prefix="(" suffix=")" prefixOverrides="," suffixOverrides=","&gt;
                &lt;if test="username != null"&gt;
                    #{username},
                &lt;/if&gt;
                &lt;if test="password != null"&gt;
                    #{password},
                &lt;/if&gt;
                &lt;if test="age != null"&gt;
                    #{age},
                &lt;/if&gt;
                &lt;if test="phone != null"&gt;
                    #{phone},
                &lt;/if&gt;
                &lt;if test="gender != null"&gt;
                    #{gender}
                &lt;/if&gt;
            &lt;/trim&gt;
    &lt;/insert&gt;</code></pre>
    <h4 id="2.2.1%C2%A0%E6%B3%A8%E8%A7%A3%E5%AE%8C%E6%88%90%E5%8A%A8%E6%80%81%20SQL" name="2.2.1%C2%A0%E6%B3%A8%E8%A7%A3%E5%AE%8C%E6%88%90%E5%8A%A8%E6%80%81%20SQL" style="background-color:transparent">
     2.2.1 注解操作动态 SQL
    </h4>
    <p>
     上文是通过 XML 进行的动态 SQL , 在注解中同样也是可以的, 并且方式相同.
    </p>
    <p>
     若要通过注解完成动态 SQL, 只需将上文 XML 中动态 SQL 的代码, 以字符串的形式拼接到 &lt;script&gt; 标签中即可.
    </p>
    <pre><code class="language-java">    @Insert("&lt;script&gt;" +
            "insert into user_info " +
            "&lt;trim prefix=\"(\" suffix=\")\" prefixOverrides=\",\" suffixOverrides=\",\"&gt;" +
            "&lt;if test=\"username != null\"&gt;username,&lt;/if&gt;" +
            "&lt;if test=\"password != null\"&gt;password,&lt;/if&gt;" +
            "&lt;if test=\"age != null\"&gt;age,&lt;/if&gt;" +
            "&lt;if test=\"phone != null\"&gt;phone,&lt;/if&gt;" +
            "&lt;if test=\"gender != null\"&gt;gender&lt;/if&gt;" +
            "&lt;/trim&gt; values " +
            "&lt;trim prefix=\"(\" suffix=\")\" prefixOverrides=\",\" suffixOverrides=\",\"&gt;" +
            "&lt;if test=\"username != null\"&gt;#{username},&lt;/if&gt;" +
            "&lt;if test=\"password != null\"&gt;#{password},&lt;/if&gt;" +
            "&lt;if test=\"age != null\"&gt;#{age},&lt;/if&gt;" +
            "&lt;if test=\"phone != null\"&gt;#{phone},&lt;/if&gt;" +
            "&lt;if test=\"gender != null\"&gt;#{gender}&lt;/if&gt;" +
            "&lt;/trim&gt;" +
            "&lt;/script&gt;")
    int insertUser4(UserInfo userInfo);</code></pre>
    <h2 id="3.%20%E5%8A%A8%E6%80%81%E6%9F%A5%E8%AF%A2" name="3.%20%E5%8A%A8%E6%80%81%E6%9F%A5%E8%AF%A2">
     3. 动态查询
    </h2>
    <p>
     进行 select 查询操作时, 常常会用到 where 关键字进行条件查询, 进行条件查询时, 同样也可以通过 if 标签进行动态查询:
    </p>
    <p>
     <img alt="" height="871" src="https://i-blog.csdnimg.cn/direct/b244587d5f644b28a0be0d733e17d4ba.png" width="1708"/>
    </p>
    <p>
     但是, 当参数 phone 为 null 时, 就发生错误了:
    </p>
    <p>
     <img alt="" height="752" src="https://i-blog.csdnimg.cn/direct/fdf02952f2604e698bfb1e780708238e.png" width="1259"/>
    </p>
    <p>
     出错的原因很简单:
     <strong>
      因为 phone 属性的值为 null, 导致 where 关键字后跟的是 "and gender", 导致 SQL 语法错误.
     </strong>
    </p>
    <p>
     <strong>
      解决办法很简单, 使用 trim 标签, 将 and 关键字去除即可:
     </strong>
    </p>
    <p>
     <img alt="" height="766" src="https://i-blog.csdnimg.cn/direct/4e45b3041d3d49d287ff0a2e6a546bf5.png" width="1436"/>
    </p>
    <p>
     但是, 当 phone 和 gender 两个属性都为 null 时, 问题又出现了:
    </p>
    <p>
     <img alt="" height="788" src="https://i-blog.csdnimg.cn/direct/646f242551154bb092118bfbfa46e281.png" width="1403"/>
    </p>
    <p>
     这里错误的原因是: phone 和 gender 都为 null, 导致出现了多余的 where 关键字, 导致 SQL 语法错误.
    </p>
    <p>
     解决方法有两种:
    </p>
    <ol>
     <li>
      <strong>
       添加 1 = 1, 保证 where 后有条件
      </strong>
     </li>
     <li>
      <strong>
       使用 &lt;where&gt;, 去除 where 关键字
      </strong>
     </li>
    </ol>
    <h3 id="3.1%20%E6%B7%BB%E5%8A%A0%201%20%3D%201" name="3.1%20%E6%B7%BB%E5%8A%A0%201%20%3D%201">
     3.1 添加 1 = 1
    </h3>
    <p>
     <strong>
      在 where 关键字后添加 1 = 1, 保证 where 后有条件. 并在每条拼接的 SQL 片段前添加 and, 保证 SQL 片段可以进行拼接:
     </strong>
    </p>
    <p>
     <img alt="" height="804" src="https://i-blog.csdnimg.cn/direct/4a5f09ebd8eb4ff3b3b569e47c22a7a8.png" width="1680"/>
    </p>
    <pre><code class="language-java">&lt;select id="selectByCondition" resultType="com.study.mybatis.model.UserInfo"&gt;
        select * from user_info
         where 1 = 1
             &lt;if test="phone != null"&gt;
                 and phone = #{phone}
             &lt;/if&gt;
             &lt;if test="gender != null"&gt;
                 and gender = #{gender}
             &lt;/if&gt;
    &lt;/select&gt;</code></pre>
    <h3 id="3.2%20where%20%E6%A0%87%E7%AD%BE" name="3.2%20where%20%E6%A0%87%E7%AD%BE" style="background-color:transparent">
     3.2 where 标签
    </h3>
    <p>
     使用 where 标签代替 where 关键字:
    </p>
    <p>
     <img alt="" height="307" src="https://i-blog.csdnimg.cn/direct/441f605ffaf04b0bae99a5e4fca4fdea.png" width="431"/>
    </p>
    <p>
     where 标签有两个作用:
    </p>
    <ol>
     <li>
      <strong>
       当 where 标签语块内没有内容时, 不会生成 where 关键字
      </strong>
     </li>
     <li>
      <strong>
       当 where 标签语块内有查询条件时, 会自动生成 where 关键字, 并且会去除最前面的 and 关键字
      </strong>
     </li>
    </ol>
    <p>
     因此, 即使查询条件全为 null, 或者第一个查询条件前有 and 关键字, where 标签都会帮我们解决这些问题:
    </p>
    <p>
     <img alt="" height="834" src="https://i-blog.csdnimg.cn/direct/02d3a11299b140f0810fcbacd37874aa.png" width="1476"/>
    </p>
    <blockquote>
     <p>
      <strong>
       但是, 在实际工作中, 要避免使用 where 标签, 因为当我们遗漏传入条件查询的参数时, 程序也会正确执行, 而无任何条件限制的 select 是非常严重的!!
      </strong>
     </p>
    </blockquote>
    <pre><code class="language-java">&lt;select id="selectByCondition" resultType="com.study.mybatis.model.UserInfo"&gt;
        select * from user_info
        &lt;where&gt;
            &lt;if test="phone != null"&gt;
                phone = #{phone}
            &lt;/if&gt;
            &lt;if test="gender != null"&gt;
                and gender = #{gender}
            &lt;/if&gt;
        &lt;/where&gt;
&lt;/select&gt;</code></pre>
    <h2 id="%C2%A04.%20%E5%8A%A8%E6%80%81%E6%9B%B4%E6%96%B0" name="%C2%A04.%20%E5%8A%A8%E6%80%81%E6%9B%B4%E6%96%B0">
     4. 动态更新
    </h2>
    <p>
     update 语句也可以结合 if 和 trim 标签, 完成动态更新操作:
    </p>
    <p>
     <img alt="" height="641" src="https://i-blog.csdnimg.cn/direct/10c29812c49645aba1ced142edd9ef3f.png" width="982"/>
    </p>
    <pre><code class="language-java">&lt;update id="updateUser2"&gt;
        update user_info
        set
            &lt;trim suffixOverrides=","&gt;
                &lt;if test="username != null"&gt;
                    username = #{username},
                &lt;/if&gt;
                &lt;if test="password != null"&gt;
                    password = #{password},
                &lt;/if&gt;
                &lt;if test="age != null"&gt;
                    age = #{age}
                &lt;/if&gt;
            &lt;/trim&gt;
             where id = #{id}
    &lt;/update&gt;</code></pre>
    <h3 id="4.1%20set%20%E6%A0%87%E7%AD%BE" name="4.1%20set%20%E6%A0%87%E7%AD%BE">
     4.1 set 标签
    </h3>
    <p>
     除了上文将 if 和 trim 标签结合完成动态的 update 操作外, 也可以使用 set 标签来完成.
    </p>
    <p>
     set 标签的作用:
    </p>
    <ol>
     <li>
      <strong>
       生成 set 关键字
      </strong>
     </li>
     <li>
      <strong>
       去除多余的逗号(只能去除最后一个逗号)
      </strong>
     </li>
    </ol>
    <p>
     <img alt="" height="806" src="https://i-blog.csdnimg.cn/direct/4764a55b37444437be178ff31180f566.png" width="1354"/>
    </p>
    <pre><code class="language-java">&lt;update id="updateUser3"&gt;
        update user_info
        &lt;set&gt;
            &lt;if test="username != null"&gt;
                username = #{username},
            &lt;/if&gt;
            &lt;if test="password != null"&gt;
                password = #{password},
            &lt;/if&gt;
            &lt;if test="age != null"&gt;
                age = #{age}
            &lt;/if&gt;
        &lt;/set&gt;
        where id = #{id}
    &lt;/update&gt;</code></pre>
    <h2 id="%C2%A05.%20foreach%20%E6%A0%87%E7%AD%BE" name="%C2%A05.%20foreach%20%E6%A0%87%E7%AD%BE">
     5. foreach 标签
    </h2>
    <p>
     在进行数据库操作时, 我们常会在一个条 SQL 语句中, 对数据批量的进行操作, 如下:
    </p>
    <pre><code class="language-sql">SELECT * FROM user_info where id = 1 or id = 2 or id = 3;

INSERT into user_info (username, `password`) VALUES ('1', '1'), ('2', '2');

UPDATE user_info set delete_flag = 1 WHERE id IN(1, 2, 3);

DELETE from user_info WHERE id IN(4, 5, 6);</code></pre>
    <p>
     如果要通过 MyBatis 对数据进行批量处理, 就需要使用 foreach 标签. \
    </p>
    <p>
     <strong>
      使用 foreach 批量操作数据, 传入的参数必定是一个集合(包括 Map).
     </strong>
    </p>
    <p>
     foreach 标签中有以下几个关键属性:
    </p>
    <ol>
     <li>
      <p>
       <strong>
        collection (必须): 指定要迭代的集合. (
       </strong>
       如果方法参数是
       <code>
        List
       </code>
       ，
       <code>
        collection
       </code>
       就是
       <code>
        List
       </code>
       的参数名
       <strong>
        )
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        item (必须): 指定迭代过程中元素的名称.
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        index (可选): 表示集合中每个元素的索引
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        separator (可选): 指定每个元素之间的分隔符
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        open (可选): 循环开始前要添加的字符串. 通常用于在 IN 语句中添加左括号 (
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        close (可选): 循环结束后要添加的字符串. 通常用于在 IN 语句中添加右括号 )
       </strong>
      </p>
     </li>
    </ol>
    <p>
     <img alt="" height="855" src="https://i-blog.csdnimg.cn/direct/020ea1201bf74993af00a93179ca692e.png" width="1480"/>
    </p>
    <pre><code class="language-java">&lt;!--    DELETE from user_info WHERE id IN(1, 2, 3);--&gt;
    &lt;delete id="deleteUser2"&gt;
        delete from user_info where
               id in
               &lt;foreach collection="ids" open="(" close=")" item="id" separator=","&gt;
                   #{id}
               &lt;/foreach&gt;
    &lt;/delete&gt;</code></pre>
    <pre><code class="language-java">&lt;!--    delete FROM user_info where id = 1 or id = 2 or id = 3;--&gt;
    &lt;delete id="deleteUser3"&gt;
        delete from user_info where
        &lt;foreach collection="ids" item="id" separator="or"&gt;
            id = #{id}
        &lt;/foreach&gt;
    &lt;/delete&gt;</code></pre>
    <pre><code class="language-java">&lt;!--    insert into user_info (username, `password`) VALUES ('1', '1'), ('2', '2');--&gt;
    &lt;insert id="insertUser1"&gt;
        insert into user_info (username, password) values
        &lt;foreach collection="userInfos" item="user" separator=","&gt;
              (#{user.username}, #{user.password})
        &lt;/foreach&gt;
    &lt;/insert&gt;</code></pre>
    <h2 id="6.%20sql%20%E6%A0%87%E7%AD%BE%20%26%20include%20%E6%A0%87%E7%AD%BE" name="6.%20sql%20%E6%A0%87%E7%AD%BE%20%26%20include%20%E6%A0%87%E7%AD%BE">
     6. sql 标签 &amp; include 标签
    </h2>
    <p>
     sql 标签和 include 标签通常搭配使用.
    </p>
    <p>
     <strong>
      在 sql 标签中定义一个 sql 片段, 并指定这个 sql 标签的 id.
     </strong>
    </p>
    <p>
     <strong>
      在其他任何标签中, 就可以通过 include 标签指定某个 sql 标签的 id, 直接引用该 sql 标签中的 sql 片段.
     </strong>
    </p>
    <p>
     <strong>
      也就是说, 我们可以直接通过 sql 标签的 id, 实现 sql 片段的复用.
     </strong>
    </p>

      </strong>
