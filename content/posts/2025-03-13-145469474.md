    <p>
     <img alt="" height="606" src="https://i-blog.csdnimg.cn/direct/d322d6ed25ae4f38b0d78957f805acbe.png" width="1632"/>
    </p>
    <p>
     使用 Tauri 开发项目时，你可以选择不使用 Vue 或 React 等前端框架，而是直接使用纯 HTML 和 JavaScript 来编写项目。以下是使用纯 HTML 和 JavaScript 开发 Tauri 项目的步骤，包括启动、编译和发布。在创建过程中，选择 `Vanilla` 模板（即纯 HTML/JS 模板）。
    </p>
    <h2>
     1. 环境准备
    </h2>
    <p>
     首先，确保你已经安装了 Tauri 所需的依赖：
    </p>
    <p>
     - [Rust](https://www.rust-lang.org/tools/install)
     <br/>
     - [Node.js](https://nodejs.org/)
     <br/>
     - [Tauri CLI](https://tauri.app/v1/guides/getting-started/setup/)
    </p>
    <p>
     安装 Tauri CLI：
    </p>
    <pre><code>npm install -g @tauri-apps/cli</code></pre>
    <h2>
    </h2>
    <h2>
     2. 创建 Tauri 项目
    </h2>
    <p>
     使用 Tauri CLI 创建一个新项目：
    </p>
    <pre><code>npx create-tauri-app</code></pre>
    <p>
     在创建过程中，选择 `Vanilla` 模板（即纯 HTML/JS 模板）。
    </p>
    <h2>
    </h2>
    <h2>
     3. 项目结构
    </h2>
    <p>
     创建完成后，项目结构大致如下：
    </p>
    <pre><code>/my-tauri-app
├── src
│   ├── index.html
│   ├── main.js
│   └── style.css
├── src-tauri
│   ├── Cargo.toml
│   ├── tauri.conf.json
│   └── ...
├── package.json
└── ...</code></pre>
    <p>
     - `src/index.html`：你的主 HTML 文件。
     <br/>
     - `src/main.js`：你的主 JavaScript 文件。
     <br/>
     - `src/style.css`：你的样式文件。
     <br/>
     - `src-tauri/`：Tauri 的 Rust 后端配置和代码。
    </p>
    <h2>
    </h2>
    <h2>
     4. 编写 HTML 和 JavaScript
    </h2>
    <p>
     你可以在 `src/index.html` 和 `src/main.js` 中编写你的前端代码。例如：
    </p>
    <p>
     **src/index.html**:
    </p>
    <pre><code>html
&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
    &lt;title&gt;My Tauri App&lt;/title&gt;
    &lt;link rel="stylesheet" href="style.css"&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;Hello, Tauri!&lt;/h1&gt;
    &lt;button id="clickMe"&gt;Click Me&lt;/button&gt;
    &lt;script src="main.js"&gt;&lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre>
    <p>
     <strong>
      src/main.js
     </strong>
     :
    </p>
    <pre><code>document.getElementById('clickMe').addEventListener('click', () =&gt; {
    alert('Button clicked!');
});</code></pre>
    <h4>
    </h4>
    <h2>
     5. 启动开发服务器
    </h2>
    <p>
     在项目根目录下运行以下命令来启动开发服务器：
    </p>
    <pre><code>npm run tauri dev</code></pre>
    <p>
     这将启动 Tauri 的开发服务器，并打开一个窗口显示你的应用。
    </p>
    <h4>
    </h4>
    <h2>
     6. 编译和发布
    </h2>
    <p>
     当你完成开发并准备发布应用时，可以使用以下命令来编译和打包你的应用：
    </p>
    <pre><code>npm run tauri build</code></pre>
    <p>
     这将会生成一个可执行文件，默认情况下会放在
     <code>
      src-tauri/target/release/
     </code>
     目录下。
    </p>
    <h4>
    </h4>
    <h2>
     7. 自定义发布配置
    </h2>
    <p>
     你可以通过修改
     <code>
      src-tauri/tauri.conf.json
     </code>
     文件来自定义应用的配置，例如应用名称、图标、窗口大小等。
    </p>
    <p>
     例如，修改
     <code>
      tauri.conf.json
     </code>
     中的
     <code>
      package
     </code>
     和
     <code>
      build
     </code>
     部分：
    </p>
    <pre><code>{
  "package": {
    "productName": "MyTauriApp",
    "version": "1.0.0"
  },
  "build": {
    "distDir": "../src",
    "devPath": "http://localhost:3000",
    "beforeDevCommand": "",
    "beforeBuildCommand": ""
  },
  "tauri": {
    "windows": [
      {
        "title": "MyTauriApp",
        "width": 800,
        "height": 600,
        "resizable": true,
        "fullscreen": false
      }
    ]
  }
}</code></pre>
    <h4>
    </h4>
    <h2>
     8. 打包为安装程序
    </h2>
    <p>
     Tauri 支持将应用打包为不同平台的安装程序（如 Windows 的
     <code>
      .msi
     </code>
     ，macOS 的
     <code>
      .dmg
     </code>
     ，Linux 的
     <code>
      .deb
     </code>
     等）。你可以通过以下命令生成安装包：
    </p>
    <pre><code>npm run tauri build -- --target universal-apple-darwin # macOS
npm run tauri build -- --target x86_64-pc-windows-msvc # Windows
npm run tauri build -- --target x86_64-unknown-linux-gnu # Linux</code></pre>
    <p>
     生成的安装包会放在
     <code>
      src-tauri/target/release/
     </code>
     目录下。
    </p>
