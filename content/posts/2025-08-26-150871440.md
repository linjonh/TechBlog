# 技术速递｜Model Context Protocol (MCP) 支持已上线 JetBrains、Eclipse 和 Xcode

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/854f0d14154648b5a673c8acfb3a75df.png)

借助 JetBrains、Eclipse 和 Xcode 中的 MCP 支持，您可以将 Copilot 连接到遵循 MCP 规范的本地和远程 MCP 服务器。连接后，代理模式可利用这些服务器提供的工具来协助完成更广泛的任务。例如，您可以使用 PAT 或 OAuth 认证设置远程 GitHub MCP 服务器，并启用创建问题、查看仓库历史以及搜索 GitHub 等操作。

对于组织和企业管理员，对 MCP 的访问权限由 Copilot 策略中的 MCP 服务器进行管理。MCP 策略默认处于禁用状态，必须由[组织或企业管理员启用](https://docs.github.com/copilot/how-tos/administer-copilot/manage-for-organization/manage-policies/?wt.mc_id=3reg_webpage_reactor)。

### 现在使用

要使用 MCP，请安装适用于 JetBrains、Eclipse 或 Xcode 的最新 GitHub Copilot 插件。您还需要拥有有效的 Copilot 许可证。按照以下指南开始使用：

* [在 JetBrains 集成开发环境中配置 MCP 服务器](https://docs.github.com/en/copilot/how-tos/provide-context/use-mcp/extend-copilot-chat-with-mcp?tool=jetbrains/?wt.mc_id=3reg_webpage_reactor)
* [在 Eclipse 中配置 MCP 服务器](https://docs.github.com/en/copilot/how-tos/provide-context/use-mcp/extend-copilot-chat-with-mcp?tool=eclipse/?wt.mc_id=3reg_webpage_reactor)
* [在 Xcode 中配置 MCP 服务器](https://docs.github.com/en/copilot/how-tos/provide-context/use-mcp/extend-copilot-chat-with-mcp?tool=xcode/?wt.mc_id=3reg_webpage_reactor)
