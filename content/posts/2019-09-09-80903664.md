                解释比特币：
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                举例来说，一个城市五个人，A、B、C、D、E。A借给B一块钱，A说：我是A，我给B一块钱；B说：我是B，A给我一块钱。C、D、E听到后记录“某年月日，A借给B一块钱”。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                在这里一块钱已经变成一个变量，可以被任何事物替代。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                第一个听见并记录完成的人得到奖励，且给记录加上编号，下一个人再记录时要带着编号一起记录下去。这时比特币挖矿的简单说明。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">

               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span class="s-text-color-black" style="color:rgb(34,34,34);">
                 核心概念：
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 <span class="s-text-color-black" style="color:rgb(34,34,34);">
                  区块：
                 </span>
                </span>
                <span class="s-text-color-black" style="color:rgb(34,34,34);">
                 基本结构单元，由包含元数据的
                 <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                  区块头
                 </span>
                 和包含交易数据的
                 <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                  区块主体
                 </span>
                 构成。其中区块头包括三组元数据：用于连接前面区块、索引自父区块哈希值的数据；挖矿难度、Nonce(随机数，用以工作量证明算法的计数器)；能够快速总结并快速归纳校验区块中所有交易数据的Merkle树根数据。
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span class="s-text-color-gray" style="color:rgb(153,153,153);">
                 关于Merkle树的进一步解释：
                </span>
                <span class="s-text-color-black" style="color:rgb(34,34,34);">
                 <a href="https://blog.csdn.net/pony_maggie/article/details/74538902" style="border-width:0px 0px 1px;border-bottom-style:solid;border-bottom-color:rgba(102,170,204,.6);font-family:inherit;line-height:1.7;color:rgb(102,170,204);">
                  https://blog.csdn.net/pony_maggie/article/details/74538902
                 </a>
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                区块链系统约每十分钟创建一个区块，包含这个时间段内全网发生的所有交易数据，从诞生至今形成了一条唯一的主区块链。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 哈希算法：
                </span>
                保证不被篡改的单向密码机制。特点为：加密过程不可逆、输入变化导致输出变化。区块链中计算出相同的哈希值说明区块链中信息没有被篡改过。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 公钥和私钥：
                </span>
                不对称加密方式。公钥用来加密、验章，私钥用来解密、签章。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 时间戳：
                </span>
                对应每一次交易记录的认证，证明其真实性。直接写在区块链中，故不可篡改。每个时间戳会将前一个时间戳也纳入其随机哈希值中。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 Merkle树结构：
                </span>
                利用其数据结构数据结构存放所有叶子节点的值，并以此为基础生成一个统一的哈希值。验证时只需要验证Merkle树结构生成的统一哈希值即可，大大减少了计算量。
               </p>

          <div>
           <div class="sixteen columns" style="margin-left:10px;">
            <div class="s-blog-section-inner">
             <div class="s-component s-text">
              <div class="s-component-content s-font-body" style="font-size:18px;color:rgb(68,68,68);line-height:1.7;">
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 1、数据层/ Data Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                数据层主要描述区块链的物理形式，是区块链上从创世区块起始的链式结构，包含了区块链的区块数据、链式结构以及区块上的随机数、时间戳、公私钥数据等，是整个区块链技术中最底层的数据结构。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 2、网络层/ Network Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                网络层主要通过 P2P 技术实现分布式网络的机制，网络层包括 P2P 组网机制、数据传播机制和数据验证机制，因此区块链本质上是一个 P2P 的网络，具备自动组网的机制，节点之间通过维护一个共同的区块链结构来保持通信。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 3、共识层/ Consensus Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                共识层主要包含共识算法以及共识机制，能让高度分散的节点在去中心化的区块链网络中高效地针对区块数据的有效性达成共识，是区块链的核心技术之一，也是区块链社群的治理机制。目前至少有数十种共识机制算法，包含工作量证明、权益证明、权益授权证明、燃烧证明、重要性证明等。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                数据层、网络层、共识层是构建区块链技术的必要元素，缺少任何一层都不能称之为真正意义上的区块链技术。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 4、激励层/ Actuator Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                激励层主要包括经济激励的发行制度和分配制度，其功能是提供一定的激励措施，鼓励节点参与区块链中安全验证工作，并将经济因素纳入到区块链技术体系中，激励遵守规则参与记账的节点，并惩罚不遵守规则的节点。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 5、合约层/ Contract Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                合约层主要包括各种脚本、代码、算法机制及智能合约，是区块链可编程的基础。将代码嵌入区块链或是令牌中，实现可以自定义的智能合约，并在达到某个确定的约束条件的情况下，无需经由第三方就能够自动执行，是区块链去信任的基础。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                <span style="border:0px;font-family:inherit;font-weight:700;line-height:1.7;">
                 6、应用层/ Application Layer
                </span>
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                区块链的应用层封装了各种应用场景和案例，类似于电脑操作系统上的应用程序、互联网浏览器上的门户网站、搜寻引擎、电子商城或是手机端上的 APP，将区块链技术应用部署在如以太坊、EOS、QTUM 上并在现实生活场景中落地。未来的可编程金融和可编程社会也将会是搭建在应用层上。
               </p>
               <p style="border:0px;font-family:inherit;font-size:18px;line-height:1.7;">
                激励层、合约层和应用层不是每个区块链应用的必要因素，一些区块链应用并不完整包含此三层结构。
               </p>
