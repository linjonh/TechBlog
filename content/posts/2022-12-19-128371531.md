    <ol>
     <li>
      将A服务器上的文件复制到B服务器上
     </li>
     <li>
      将B服务器上的文件复制到指定容器中
     </li>
    </ol>
    <h2>
     <a id="AB_3">
     </a>
     一、将A服务器上的文件复制到B服务器上
    </h2>
    <p>
     <em>
      <strong>
       （将本地文件复制到服务器上同理）
      </strong>
     </em>
     <br/>
     在A服务器上执行下面命令（在本地执行下面命令）
     <br/>
     <strong>
      使用scp命令
     </strong>
    </p>
    <h4>
     <a id="1__7">
     </a>
     1. 复制文件
    </h4>
    <p>
     命令格式：
     <code>
      scp local_file remote_username@remote_ip:remote_folder
     </code>
    </p>
    <h4>
     <a id="2__9">
     </a>
     2. 复制目录
    </h4>
    <p>
     命令格式：
     <code>
      scp -r local_folder remote_username@remote_ip:remote_folder
     </code>
     <br/>
     <strong>
      <em>
       注意
      </em>
      ：
     </strong>
     <br/>
     1 由于这里不能添加端口号，所以只能将其复制到默认的22端口。
     <br/>
     2 出现报错：Permission denied, please try again
     <br/>
     进入B服务器，输入命令：
     <code>
      sudo chmod 777 remote_folder
     </code>
     <br/>
     （权限不够，修改权限即可。）
    </p>
    <h3>
     <a id="B_16">
     </a>
     二、将B服务器上的文件复制到指定容器中
    </h3>
    <p>
     <strong>
      使用docker cp命令
     </strong>
    </p>
    <h4>
     <a id="1__18">
     </a>
     1. 将主机中的文件拷贝到容器中
    </h4>
    <p>
     命令格式：
     <code>
      docker cp 主机文件目录 容器名:容器文件目录
     </code>
     <br/>
     实例：
     <br/>
     将主机文件/home/zm/FlyingThings3D_subset_processed_35m.zip拷贝到容器zm中的/zm/HPLFlowNet-master/SAVE_PATH/FlyingThings3D_sub
     <br/>
     set_processed_35m.zip下
    </p>
    <pre><code class="prism language-powershell">docker <span class="token function">cp</span> <span class="token operator">/</span>home/zm/FlyingThings3D_subset_processed_35m<span class="token punctuation">.</span>zip zm:<span class="token operator">/</span>zm/HPLFlowNet-master/SAVE_PATH/FlyingThings3D_sub
set_processed_35m<span class="token punctuation">.</span>zip
</code></pre>
    <h4>
     <a id="2__27">
     </a>
     2. 将容器中的文件拷贝到主机中
    </h4>
    <p>
     命令格式：
     <code>
      docker cp 容器名:容器文件目录 主机文件目录
     </code>
    </p>
    <p>
     <em>
      注：执行命令后没有任何进度提醒，这个时候命令是正在执行的状态，不是死机什么的，安心等待就好。
     </em>
    </p>
    <h2>
     <a id="rsync_31">
     </a>
     三、rsync
    </h2>
    <p>
     当文件夹中存在软链接时，scp会将软链接中的内容也复制过来，但是这并不是我们想要的，这时就可以使用rsync保留原来的软链接
     <br/>
     <code>
      rsync -rl local_folder remote_username@remote_ip:remote_folder
     </code>
     <br/>
     其中
     <code>
      -l
     </code>
     的意思就是保留软链接
    </p>
