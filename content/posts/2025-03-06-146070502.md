    <h3>
     <a id="1_Git__18">
     </a>
     1. Git 简介
    </h3>
    <p>
     Git 是一个分布式版本控制系统，广泛用于代码管理。它可以帮助开发者跟踪代码的变更、协作开发以及管理代码版本。IntelliJ IDEA 提供了强大的 Git 集成功能，使得开发者可以在 IDE 中轻松完成 Git 操作。
    </p>
    <hr/>
    <h3>
     <a id="2_IDEA__Git_23">
     </a>
     2. IDEA 中配置 Git
    </h3>
    <h4>
     <a id="_25">
     </a>
     步骤
    </h4>
    <ol>
     <li>
      打开 IntelliJ IDEA，进入
      <code>
       File -&gt; Settings
      </code>
      （Windows）或
      <code>
       IntelliJ IDEA -&gt; Preferences
      </code>
      （Mac）。
     </li>
     <li>
      在左侧导航栏中选择
      <code>
       Version Control -&gt; Git
      </code>
      。
     </li>
     <li>
      在
      <code>
       Path to Git executable
      </code>
      中输入 Git 的可执行文件路径（例如：
      <code>
       /usr/bin/git
      </code>
      或
      <code>
       C:\Program Files\Git\bin\git.exe
      </code>
      ）。
     </li>
     <li>
      点击
      <code>
       Test
      </code>
      按钮，确保 IDEA 能够正确识别 Git。
     </li>
     <li>
      点击
      <code>
       Apply
      </code>
      和
      <code>
       OK
      </code>
      保存配置。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="3__Git__34">
     </a>
     3. 常用 Git 操作
    </h3>
    <h4>
     <a id="31__36">
     </a>
     3.1 克隆仓库
    </h4>
    <ol>
     <li>
      打开 IDEA，点击
      <code>
       Get from Version Control
      </code>
      。
     </li>
     <li>
      在弹出的窗口中输入仓库的 URL（例如：
      <code>
       https://github.com/username/repository.git
      </code>
      ）。
     </li>
     <li>
      选择本地目录，点击
      <code>
       Clone
      </code>
      。
     </li>
    </ol>
    <h4>
     <a id="32__41">
     </a>
     3.2 提交更改
    </h4>
    <ol>
     <li>
      在 IDEA 中修改文件后，右侧会显示文件的变化。
     </li>
     <li>
      右键点击文件，选择
      <code>
       Git -&gt; Commit File
      </code>
      。
     </li>
     <li>
      在弹出的窗口中填写提交信息，选择需要提交的文件，点击
      <code>
       Commit
      </code>
      。
     </li>
     <li>
      如果需要推送到远程仓库，勾选
      <code>
       Commit and Push
      </code>
      。
     </li>
    </ol>
    <h4>
     <a id="33__47">
     </a>
     3.3 拉取与推送
    </h4>
    <ul>
     <li>
      <strong>
       拉取远程更改
      </strong>
      ：点击顶部菜单栏的
      <code>
       VCS -&gt; Git -&gt; Pull
      </code>
      。
     </li>
     <li>
      <strong>
       推送本地更改
      </strong>
      ：点击顶部菜单栏的
      <code>
       VCS -&gt; Git -&gt; Push
      </code>
      。
     </li>
    </ul>
    <h4>
     <a id="34__51">
     </a>
     3.4 分支管理
    </h4>
    <ul>
     <li>
      <strong>
       创建分支
      </strong>
      ：点击右下角的 Git 分支名称，选择
      <code>
       New Branch
      </code>
      ，输入分支名称。
     </li>
     <li>
      <strong>
       切换分支
      </strong>
      ：点击右下角的 Git 分支名称，选择需要切换的分支。
     </li>
     <li>
      <strong>
       合并分支
      </strong>
      ：右键点击目标分支，选择
      <code>
       Merge into Current
      </code>
      。
     </li>
    </ul>
    <h4>
     <a id="35__56">
     </a>
     3.5 解决冲突
    </h4>
    <ol>
     <li>
      当拉取或合并代码时，如果发生冲突，IDEA 会提示冲突文件。
     </li>
     <li>
      双击冲突文件，IDEA 会显示冲突内容。
     </li>
     <li>
      使用
      <code>
       &lt;&lt;&lt;&lt;&lt;&lt;&lt;
      </code>
      、
      <code>
       =======
      </code>
      和
      <code>
       &gt;&gt;&gt;&gt;&gt;&gt;&gt;
      </code>
      标记解决冲突。
     </li>
     <li>
      解决后，右键点击文件，选择
      <code>
       Mark as Resolved
      </code>
      ，然后提交更改。
     </li>
    </ol>
    <h4>
     <a id="36__62">
     </a>
     3.6 查看历史记录
    </h4>
    <ol>
     <li>
      右键点击文件或目录，选择
      <code>
       Git -&gt; Show History
      </code>
      。
     </li>
     <li>
      在弹出的窗口中可以查看文件的提交历史、作者、日期等信息。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="4__Git__68">
     </a>
     4. 高级 Git 操作
    </h3>
    <h4>
     <a id="41_Stash__70">
     </a>
     4.1 Stash 暂存更改
    </h4>
    <ol>
     <li>
      当需要切换分支但不想提交当前更改时，可以使用 Stash 功能。
     </li>
     <li>
      点击顶部菜单栏的
      <code>
       VCS -&gt; Git -&gt; Stash Changes
      </code>
      。
     </li>
     <li>
      输入 Stash 名称，点击
      <code>
       Create Stash
      </code>
      。
     </li>
     <li>
      恢复 Stash：点击
      <code>
       VCS -&gt; Git -&gt; UnStash Changes
      </code>
      ，选择需要恢复的 Stash。
     </li>
    </ol>
    <h4>
     <a id="42_Rebase__76">
     </a>
     4.2 Rebase 变基
    </h4>
    <ol>
     <li>
      变基可以将当前分支的提交应用到目标分支上。
     </li>
     <li>
      点击顶部菜单栏的
      <code>
       VCS -&gt; Git -&gt; Rebase
      </code>
      。
     </li>
     <li>
      选择目标分支，点击
      <code>
       Rebase
      </code>
      。
     </li>
     <li>
      如果发生冲突，按照提示解决冲突后继续变基。
     </li>
    </ol>
    <h4>
     <a id="43_Cherrypick__82">
     </a>
     4.3 Cherry-pick 选择提交
    </h4>
    <ol>
     <li>
      Cherry-pick 可以选择某个提交应用到当前分支。
     </li>
     <li>
      右键点击提交记录，选择
      <code>
       Cherry-pick
      </code>
      。
     </li>
     <li>
      如果发生冲突，按照提示解决冲突后继续操作。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="5__89">
     </a>
     5. 总结
    </h3>
    <p>
     IntelliJ IDEA 提供了强大的 Git 集成功能，使得开发者可以在 IDE 中轻松完成日常的 Git 操作。无论是克隆仓库、提交更改、分支管理，还是解决冲突、查看历史记录，IDEA 都能提供直观的操作界面和高效的工作流。通过掌握这些操作，开发者可以更好地管理代码版本，提高开发效率。
    </p>
