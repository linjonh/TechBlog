    <h2>
     <a id="_2">
     </a>
     项目环境
    </h2>
    <p>
     本次项目前端使用的是vue 后端使用spring boot
     <br/>
     分别使用的编辑器是vs code和ide
    </p>
    <h2>
     <a id="_7">
     </a>
     前端项目打包
    </h2>
    <h3>
     <a id="npm_run_build_9">
     </a>
     npm run build
    </h3>
    <p>
     使用npm run build 将前端项目打包进dist文件夹
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/83511c4f4023fed8077d299ad16b2a7d.png"/>
    </p>
    <p>
     将dist文件夹中的文件复制到后端项目的src下的main的resources中的static文件夹中，再修改其中static文件夹中js文件夹里第一个文件，将前端项目的端口号给删除 保留第一个分隔符，如我这里删除localhost:8080,使用替换的方式全部修改完成之后就可以进入后端打包的过程了。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c5701c3b0ea0efd66f19db696dee37b5.png">

      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e457beca69e8e1fda39e459d575186a0.png"/>

    <h2>
     <a id="_21">
     </a>
     后端项目打包
    </h2>
    <p>
     后端项目打包前现在maven中将老的项目文件clear一下，然后使用packet打包，等待打包完成会在target文件夹中生成一个jar包，至此打包完毕，接下来进行上传过程。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9bcc720bedcadaf1abc8a63f316e91c7.png"/>
    </p>
    <h2>
     <a id="xshellxftp_27">
     </a>
     xshell和xftp
    </h2>
    <p>
     首先使用xftp将打包好的jar包复制到服务器端的文件夹当中，接着使用 xshell。cd进入到刚刚存放jar包的文件夹，
    </p>
    <p>
     使用命令先杀死一下端口
    </p>
    <pre><code>sudo fuser -k -n tcp 80 ——————强制杀死某个端口的进程
</code></pre>
    <p>
     然后启动jar包即可
    </p>
    <pre><code>nohup java -jar xxx.jar &gt;springboot.log 2&gt;&amp;1&amp;
</code></pre>
