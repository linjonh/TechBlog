    <h2>
     <a id="1_git_updategitforwindows__2">
     </a>
     1. git update-git-for-windows 报错
    </h2>
    <pre><code class="prism language-shell">Lenovo@LAPTOP-EQKBL89E MINGW64 /d/YHProjects/omni-channel-recon-dev <span class="token punctuation">(</span>master<span class="token punctuation">)</span>
$ <span class="token function">git</span> update-git-for-windows
curl: <span class="token punctuation">(</span><span class="token number">60</span><span class="token punctuation">)</span> schannel: SEC_E_UNTRUSTED_ROOT <span class="token punctuation">(</span>0x80090325<span class="token punctuation">)</span> - More details here: https://curl.se/docs/sslcerts.html

<span class="token function">curl</span> failed to verify the legitimacy of the server and therefore could not
establish a secure connection to it. To learn <span class="token function">more</span> about this situation and
how to fix it, please visit the webpage mentioned above.
</code></pre>
    <ul>
     <li>
      操作：windows环境使用
      <code>
       git update-git-for-windows
      </code>
      从我自己的gitee仓库更新代码。
     </li>
     <li>
      报错信息：curl 在尝试与服务器建立安全连接时，无法验证服务器的 SSL 证书。这可能是因为服务器使用了自签名证书，或者证书链中的某个证书不被客户端信任。
      <br/>
      简而言之：
      <strong>
       我的本机（即本地 Git 客户端）拒绝验证远程服务器的 SSL 证书
      </strong>
      。
     </li>
    </ul>
    <h2>
     <a id="2__17">
     </a>
     2. 解决方案
    </h2>
    <h3>
     <a id="21__CA__19">
     </a>
     2.1. 更新 CA 证书库
    </h3>
    <p>
     Windows操作系统通常会通过Windows Update自动更新根证书。
     <br/>
     确保我的操作系统或 Git 客户端的 CA 证书库是最新的，这样可能会包含远程服务器证书的签发机构。
     <br/>
     <img src="https://i-blog.csdnimg.cn/direct/db4bfdcbb44744768ecff4b4fe211b0d.png" width="80%">
      <br/>
      但是我的Windows11确实已经是最新版的了，并且我想起来了，
      <strong>
       我昨天下班的时候更新了一下Windows，下班之前我都还能正常用git拉项目，更新之后就不行了
      </strong>
      。
     </img>
    </p>
    <p>
     至于去使用gitee的CA证书，于我而言就不现实了，我根本拿不到。
    </p>
    <h3>
     <a id="22__SSH__26">
     </a>
     2.2. 使用 SSH 连接（推荐）
    </h3>
    <p>
     使用 SSH 而不是 HTTPS 来克隆或更新代码。SSH 方式不需要依赖 SSL 证书验证，通常更安全。
    </p>
    <ol>
     <li>
      <p>
       生成SSH密钥
      </p>
      <pre><code class="prism language-shell">ssh-keygen <span class="token parameter variable">-t</span> rsa <span class="token parameter variable">-b</span> <span class="token number">4096</span> <span class="token parameter variable">-C</span> <span class="token string">"your_email@example.com"</span>
</code></pre>
     </li>
     <li>
      <p>
       查看公钥并复制
      </p>
      <pre><code class="prism language-shell"><span class="token function">cat</span> ~/.ssh/id_rsa.pub
</code></pre>
     </li>
     <li>
      <p>
       将复制的公钥放到个人——设置——SSH公钥下
      </p>
     </li>
     <li>
      <p>
       重新clone，但是选择仓库的ssh地址来进行克隆
      </p>
      <pre><code class="prism language-shell"><span class="token function">git</span> clone git@gitee.com:username/repo.git
</code></pre>
     </li>
    </ol>
    <h3>
     <a id="23__SSL__46">
     </a>
     2.3 禁用 SSL 验证（不推荐）
    </h3>
    <p>
     不推荐的原因如下：
    </p>
    <ol>
     <li>
      虽然可以临时禁用 SSL 验证来绕过这个问题，但这会降低安全性，因为它会使你的连接容易受到中间人攻击。
     </li>
     <li>
      <font color="red">
       <strong>
        压根不起作用，临时禁用都不行，还是报错，艹
       </strong>
      </font>
      。
     </li>
    </ol>
    <pre><code class="prism language-shell"><span class="token function">git</span> config <span class="token parameter variable">--global</span> http.sslVerify <span class="token boolean">false</span>
</code></pre>
    <h3>
     <a id="24_pullupdate_56">
     </a>
     2.4 使用pull不使用update
    </h3>
    <p>
     很离谱，上周按照2.2修改后都用了好几天，结果这周一来，又是这个b报错。
    </p>
    <pre><code class="prism language-shell">Lenovo@LAPTOP-EQKBL89E MINGW64 /d/YHProjects/omni-channel-recon-dev <span class="token punctuation">(</span>master<span class="token punctuation">)</span>
$ <span class="token function">git</span> update-git-for-windows
curl: <span class="token punctuation">(</span><span class="token number">60</span><span class="token punctuation">)</span> schannel: SEC_E_UNTRUSTED_ROOT <span class="token punctuation">(</span>0x80090325<span class="token punctuation">)</span> - More details here: https://curl.se/docs/sslcerts.html

<span class="token function">curl</span> failed to verify the legitimacy of the server and therefore could not
establish a secure connection to it. To learn <span class="token function">more</span> about this situation and
how to fix it, please visit the webpage mentioned above.
</code></pre>
    <p>
     但是我用
     <code>
      pull
     </code>
     ，也能用
     <code>
      push
     </code>
     推送代码又能正常更新。
    </p>
    <blockquote>
     <p>
      <code>
       git pull
      </code>
      和
      <code>
       git push
      </code>
      是 Git 仓库的操作，通常不会涉及 SSL 证书验证问题。
      <br/>
      <code>
       git update-git-for-windows
      </code>
      是通过 curl 下载更新，因此会触发 SSL 证书验证。
     </p>
    </blockquote>
    <pre><code class="prism language-shell"><span class="token function">git</span> pull origin master
</code></pre>
