     <h4>
      博客目录
     </h4>
    </div>
    <p>
    </p>
    <p>
     <code>
      ps aux | grep nginx
     </code>
     是一个 Linux 命令，用于在进程列表中查找与
     <code>
      nginx
     </code>
     相关的进程。下面是对这个命令的逐步解释：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         ps
        </code>
       </strong>
       : 是用于显示系统中当前运行进程的命令。
      </p>
      <ul>
       <li>
        <code>
         ps
        </code>
        命令可以显示特定格式的进程信息，如进程 ID、内存使用率、CPU 使用率等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         aux
        </code>
       </strong>
       : 是
       <code>
        ps
       </code>
       命令的参数组合，表示显示所有进程的详细信息。
      </p>
      <ul>
       <li>
        <code>
         a
        </code>
        : 显示所有用户的进程，包括其他用户的进程。
       </li>
       <li>
        <code>
         u
        </code>
        : 以用户友好的格式显示进程信息（例如用户名、CPU 和内存使用情况等）。
       </li>
       <li>
        <code>
         x
        </code>
        : 显示没有控制终端的进程（即系统后台的进程）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         |
        </code>
       </strong>
       : 管道操作符，用于将前一个命令的输出作为下一个命令的输入。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         grep nginx
        </code>
       </strong>
       :
       <code>
        grep
       </code>
       是一个用于搜索文本的工具，这里用来过滤
       <code>
        ps aux
       </code>
       的输出，只显示包含
       <code>
        nginx
       </code>
       的行。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0ac9551f34f44708ad3f0980c4c119c2.png#pic_center"/>
      </p>
     </li>
    </ol>
    <p>
     综上，
     <code>
      ps aux | grep nginx
     </code>
     命令的作用是：
    </p>
    <ul>
     <li>
      列出所有当前运行的进程，并过滤出与
      <code>
       nginx
      </code>
      相关的进程信息。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dd38a4bd1e17478890088a5d1d90f9ae.png#pic_center"/>
     </li>
    </ul>
    <p>
     输出通常会包含类似下面的内容：
    </p>
    <pre><code>root      1234  0.0  0.1  123456  7890 ?        Ss   12:34   0:00 nginx: master process /usr/sbin/nginx
www-data  1235  0.0  0.1  123456  7890 ?        S    12:34   0:00 nginx: worker process
user      4567  0.0  0.0   12345   567 pts/0    S+   12:35   0:00 grep nginx
</code></pre>
    <p>
     最后一行通常是
     <code>
      grep nginx
     </code>
     进程本身的结果。
    </p>
