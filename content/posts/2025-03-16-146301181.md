    <h2>
     <a id="Appium_1X_0">
     </a>
     一、Appium 1.X
    </h2>
    <p>
     启动appium服务：appium --session-override
     <br/>
     也可以这样启动appium服务：appium
    </p>
    <h2>
     <a id="Appium_2X_4">
     </a>
     二、Appium 2.X
    </h2>
    <p>
     Appium 2.X 是一个自动化测试开源工具，用于测试原生、移动 Web 和混合应用程序。下面为你介绍其下载步骤：
    </p>
    <h4>
     <a id="1__Nodejs__npm_7">
     </a>
     1. 安装 Node.js 和 npm
    </h4>
    <p>
     Appium 2.0 是基于 Node.js 构建的，所以要先安装 Node.js 和 npm（Node 包管理器）。
    </p>
    <ul>
     <li>
      访问
      <a href="https://nodejs.org/" rel="nofollow">
       Node.js 官方网站
      </a>
      。
     </li>
     <li>
      下载适合你操作系统的 Node.js 安装包。
     </li>
     <li>
      运行安装程序，按照提示完成安装。安装完成后，在命令行输入以下命令来验证是否安装成功：
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">node</span> <span class="token parameter variable">-v</span>
<span class="token function">npm</span> <span class="token parameter variable">-v</span>
</code></pre>
    <h4>
     <a id="2__Appium_20_17">
     </a>
     2. 全局安装 Appium 2.0
    </h4>
    <p>
     在命令行中输入以下命令，以全局方式安装 Appium 2.0：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> appium@next
</code></pre>
    <p>
     这里的
     <code>
      @next
     </code>
     表明你要安装的是最新的 Appium 2.0 版本。
    </p>
    <p>
     当执行
     <code>
      npm install -g appium@next
     </code>
     后长时间显示
     <code>
      [#########.........] / idealTree:npm: timing idealTree:#root Completed in 70385ms
     </code>
     ，说明 npm 在解析依赖树环节遇到了问题，下面为你分析可能的原因及对应的解决办法：
    </p>
    <h5>
     <a id="__26">
     </a>
     【注】 执行上述命令，没有很快的安装，可能是网络问题。切换镜像后安装会安装的很快
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：npm 需要从远程仓库下载 Appium 及其依赖包，网络状况不佳、带宽不足或者网络不稳定，都可能导致下载缓慢甚至停滞。
     </li>
     <li>
      <strong>
       解决办法
      </strong>
      ：
      <ul>
       <li>
        切换网络环境，例如从 Wi-Fi 切换到移动数据，或者反之。
       </li>
       <li>
        采用国内的 npm 镜像源来加速下载，如淘宝镜像。使用以下命令设置镜像：
       </li>
      </ul>
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> registry https://registry.npmmirror.com
</code></pre>
    <p>
     安装完成后，若要恢复官方镜像，可使用如下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> registry https://registry.npmjs.org
</code></pre>
    <h4>
     <a id="3__Appium__38">
     </a>
     3. 安装 Appium 驱动
    </h4>
    <p>
     Appium 2.0 采用了插件式架构，你需要按需安装相应的驱动。以最常用的 UiAutomator2 驱动（用于 Android 应用测试）为例，在命令行输入以下命令进行安装：
    </p>
    <pre><code class="prism language-bash">appium driver <span class="token function">install</span> uiautomator2
</code></pre>
    <p>
     如果你要测试 iOS 应用，可安装 XCUITest 驱动：
    </p>
    <pre><code class="prism language-bash">appium driver <span class="token function">install</span> xcuitest
</code></pre>
    <h4>
     <a id="4__48">
     </a>
     4. 验证安装
    </h4>
    <p>
     安装完成后，你可以在命令行输入以下命令来启动 Appium 服务器：
    </p>
    <pre><code class="prism language-bash">appium
</code></pre>
    <p>
     若服务器成功启动，输出的日志中会显示服务器正在监听特定的端口（通常是 4723），这表明 Appium 2.0 已成功安装。
    </p>
    <h4>
     <a id="5__Appium_Inspector_55">
     </a>
     5. 可选：安装 Appium Inspector
    </h4>
    <p>
     Appium Inspector 是一个可视化工具，能帮助你查看应用的 UI 元素，便于编写自动化测试脚本。你可以从
     <a href="https://github.com/appium/appium-inspector/releases">
      Appium Inspector 发布页面
     </a>
     下载适合你操作系统的安装包，然后按提示完成安装。
    </p>
    <h2>
     <a id="_appium__appium_sessionoverride__Appium__57">
     </a>
     三、使用
     <code>
      appium
     </code>
     和
     <code>
      appium --session-override
     </code>
     启动 Appium 服务存在一定区别，下面为你详细介绍：
    </h2>
    <h4>
     <a id="_appium__59">
     </a>
     使用
     <code>
      appium
     </code>
     启动服务
    </h4>
    <p>
     当你仅使用
     <code>
      appium
     </code>
     命令启动 Appium 服务时，这是最基本的启动方式，服务会以默认配置启动。默认情况下，Appium 不允许在已有会话存在时创建新的会话，若尝试在已有会话未结束时创建新会话，Appium 会返回错误。
    </p>
    <p>
     例如，你在命令行输入：
    </p>
    <pre><code class="prism language-bash">appium
</code></pre>
    <p>
     之后若有一个测试会话正在运行，此时又发起一个新的会话请求，Appium 会拒绝该请求并给出相应错误信息。
    </p>
    <h4>
     <a id="_appium_sessionoverride__68">
     </a>
     使用
     <code>
      appium --session-override
     </code>
     启动服务
    </h4>
    <p>
     <code>
      --session-override
     </code>
     是一个命令行参数，当使用
     <code>
      appium --session-override
     </code>
     启动服务时，Appium 会允许在已有会话存在的情况下创建新的会话。若有新的会话请求到来，Appium 会自动结束当前正在运行的会话，然后创建新的会话。
    </p>
    <p>
     例如，在命令行输入：
    </p>
    <pre><code class="prism language-bash">appium --session-override
</code></pre>
    <p>
     之后若有一个测试会话正在运行，此时又发起一个新的会话请求，Appium 会结束当前会话，接着为新请求创建一个新的会话。
    </p>
    <h4>
     <a id="_77">
     </a>
     应用场景
    </h4>
    <ul>
     <li>
      <strong>
       使用
       <code>
        appium
       </code>
      </strong>
      ：适用于需要严格控制会话数量和生命周期的场景，避免不同测试用例之间的会话相互干扰，确保每个测试会话的独立性和完整性。
     </li>
     <li>
      <strong>
       使用
       <code>
        appium --session-override
       </code>
      </strong>
      ：在开发和调试阶段较为有用，能够快速切换不同的测试场景，无需手动停止当前会话再启动新会话，提高开发和调试的效率。但要注意，这种方式可能会导致测试数据丢失，因为当前会话会被直接终止。
     </li>
    </ul>
