    <h3>
     <a id="Wikijs_0">
     </a>
     安装Wiki.js
    </h3>
    <p>
     以下是Wiki.js 2.x.x 的安装步骤。
    </p>
    <ol>
     <li>
      <p>
       下载wiki.js网站
       <br/>
       按照
       <a href="https://docs.requarks.io/install/linux" rel="nofollow">
        官方Linux版本安装说明
       </a>
       操作，文档中第一步是通过后台命令行下载wiki.js网站压缩包，由于使用了宝塔面板我们可以变通一下，使用浏览器下载这个资源，并在之后通过宝塔手动放到网站路径下。
      </p>
      <pre><code>https://github.com/Requarks/wiki/releases/latest/download/wiki-js.tar.gz
</code></pre>
     </li>
     <li>
      <p>
       在宝塔面板中安装必要的支持环境
       <br/>
       在软件商店中，搜索并安装
       <code>
        Nginx
       </code>
       <code>
        Node.js版本管理器
       </code>
       <code>
        PostgreSQL管理器
       </code>
       ，其中PostgreSQL数据库是wiki.js强烈推荐使用的数据库，会支持全部的功能，所以我们选择使用这个数据库。
       <br/>
       而后在Node.js版本管理器的设置界面中安装nodejs14.x.x版本，在PostgreSQL管理器安装PostgreSQL14.2版本。
      </p>
     </li>
     <li>
      <p>
       安装创建数据库
       <br/>
       在刚刚安装好的PostgreSQL管理器配置界面中选择数据库列表，点击添加数据库按钮，这里因为是要给wiki.js使用，所以我们为数据库起名叫 wiki，用户名也是wiki。
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f87ba369294f9c47618101afaf3c75ee.png"/>
      </p>
     </li>
     <li>
      <p>
       新建网站目录
       <br/>
       在宝塔面板的文件选项卡中，在默认打开的网站路径
       <code>
        /www/wwwroot/
       </code>
       中新建一个文件夹叫wiki，然后进入文件夹下把我们之前下载的wiki.js网站文件拖入并解压在此路径下。解压完成后就可以把压缩包删了。
      </p>
     </li>
     <li>
      <p>
       创建nodejs项目网站
       <br/>
       在宝塔面板中选择网站选项卡，并进入node项目，选择添加Node项目。
       <br/>
       项目名称我们这里设为wiki，端口号设置为wiki.js默认使用的端口3000，点击项目目录选择上一步的路径文件夹。其他参数按照需求配置。
       <br/>
       点击提交后宝塔会自动下载依赖，但其实nodejs网站包已经把需要的依赖做好了，在开始自动安装依赖后我们马上刷新一下界面打断安装即可。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ebdbcc0c68fd90f4e0ba0a203480812b.png"/>
      </p>
     </li>
     <li>
      <p>
       配置wiki.js
      </p>
      <p>
       在网站文件路径下，将
       <code>
        config.sample.yml
       </code>
       改名为-&gt;
       <code>
        config.yml
       </code>
       ，然后双击打开在宝塔面板中编辑。最核心的，要将数据库配置改为我们之前的数据库用户和密码。其他的部分参考官方wiki即可：
       <a href="https://docs.requarks.io/install/config" rel="nofollow">
        https://docs.requarks.io/install/config
       </a>
      </p>
      <pre><code>  # PostgreSQL / MySQL / MariaDB / MS SQL Server only:
  host: localhost
  port: 5432
  user: wiki
  pass: &lt;password&gt;
  db: wiki
  ssl: false
</code></pre>
     </li>
     <li>
      <p>
       启动网站
       <br/>
       在一切完成后，我们就可以在宝塔的网站选项卡中重新启动wiki.js了。如果之后网站有什么报错等内容我们可以通过网站配置中的项目日志查看信息定位问题。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c0ac697dc7f9ea069be3428ae74ae688.png"/>
      </p>
     </li>
     <li>
      <p>
       最后记得在云服务后台的防火墙中将3000端口打开，我们就能够通过外网IP+3000端口访问了。
      </p>
      <pre><code> 首次进入时会要求设置管理员账户等信息，按照提示完成即可。
 ![在这里插入图片描述](https://img-blog.csdnimg.cn/63b463784d4f4da0a68646cab99f3ccd.png)
</code></pre>
     </li>
    </ol>
    <h3>
     <a id="_42">
     </a>
     添加中文检索支持
    </h3>
    <p>
     在 wiki.js 后台中，我们在搜索设置中选择 PostgreSQL数据库，可以看到下拉菜单中是没有中文选项的，我们需要为PostgreSQL数据库安装中文搜索插件，并在wiki.js配置中加入中文选项。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2138e9541715d256ef45f43a027d039f.png"/>
    </p>
    <h4>
     <a id="PostgreSQL_45">
     </a>
     补齐宝塔面板安装的PostgreSQL缺失的库和插件
    </h4>
    <p>
     由于宝塔面板安装的PostgreSQL缺少一些库和自带插件，会导致我们后面执行
     <code>
      CREATE EXTENSION pg_trgm;
     </code>
     语句时报错，我们还需要在终端中手动安装一个 PostgreSQL 并将其库和插件复制进入宝塔面板安装的PostgreSQL中，最后再将手动安装的 PostgreSQL 删除。
    </p>
    <ol>
     <li>
      <p>
       在服务器终端中，添加 PostgreSQL RPM 服务器
      </p>
      <p>
       <code>
        sudo yum install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm
       </code>
      </p>
     </li>
     <li>
      <p>
       安装postgresql14-contrib
      </p>
      <p>
       <code>
        sudo yum install -y postgresql14-contrib
       </code>
      </p>
     </li>
     <li>
      <p>
       复制通过终端安装的 PostgreSQL 的库文件和插件 到宝塔面板安装的 PostgreSQL 安装目录里
      </p>
      <pre><code>sudo awk 'BEGIN { cmd="cp -ri /usr/pgsql-14/share/extension/* /www/server/pgsql/share/extension"; print "n" |cmd; }'
sudo awk 'BEGIN { cmd="cp -ri /usr/pgsql-14/lib/* /www/server/pgsql/lib/"; print "n" |cmd; }'
</code></pre>
     </li>
     <li>
      <p>
       卸载通过终端安装的 PostgreSQL
      </p>
      <p>
       <code>
        sudo yum remove -y postgresql14-contrib
       </code>
      </p>
     </li>
    </ol>
    <h4>
     <a id="_zhparser_66">
     </a>
     安装中文插件 zhparser
    </h4>
    <ol>
     <li>
      <p>
       安装 scws
       <br/>
       scws 中文分词系统是 zhparser 编译安装的必要内容。使用下方指令下载并编译安装 scws。
      </p>
      <pre><code class="prism language-bash"><span class="token function">wget</span> http://www.xunsearch.com/scws/down/scws-1.2.3.tar.bz2
<span class="token function">tar</span> xvjf scws-1.2.3.tar.bz2
<span class="token builtin class-name">cd</span> scws-1.2.3
<span class="token function">sudo</span> ./configure
<span class="token function">sudo</span> <span class="token function">make</span> <span class="token function">install</span>
</code></pre>
     </li>
     <li>
      <p>
       安装 zhparser
       <br/>
       注意在这步中需要使用 PG_CONFIG 来指定 pgsql 的 pg_config 文件路径，否则运行会报错。
       <br/>
       clone 项目时腾讯云莫名抽风下载不动，最后我是通过手动下载压缩包并通过面板扔到了服务器上。
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/amutu/zhparser.git
<span class="token builtin class-name">cd</span> zhparser
<span class="token function">sudo</span> <span class="token assign-left variable">PG_CONFIG</span><span class="token operator">=</span>/www/server/pgsql/bin/pg_config <span class="token function">make</span>
<span class="token function">sudo</span> <span class="token assign-left variable">PG_CONFIG</span><span class="token operator">=</span>/www/server/pgsql/bin/pg_config <span class="token function">make</span> <span class="token function">install</span>
</code></pre>
     </li>
     <li>
      <p>
       全部安装已经完成，删除路径下无用的程序压缩包和解压后的文件。
      </p>
     </li>
    </ol>
