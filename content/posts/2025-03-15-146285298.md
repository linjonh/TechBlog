    <blockquote>
     <p>
      先讲讲两者是如何实现的
     </p>
    </blockquote>
    <h2>
     ArrayList
    </h2>
    <pre><code class="language-java">public class ArrayList&lt;E&gt; extends AbstractList&lt;E&gt;
        implements List&lt;E&gt;, RandomAccess, Cloneable, java.io.Serializable
{
    
    transient Object[] elementData; 

    private int size;
}</code></pre>
    <p>
     通过源码可以看出，ArrayList实现是基于数组实现的
    </p>
    <h2>
     LinkedList
    </h2>
    <pre><code class="language-java">public class LinkedList&lt;E&gt;
    extends AbstractSequentialList&lt;E&gt;
    implements List&lt;E&gt;, Deque&lt;E&gt;, Cloneable, java.io.Serializable
{
    transient int size = 0;

    /**
     * Pointer to first node.
     */
    transient Node&lt;E&gt; first;

    /**
     * Pointer to last node.
     */
    transient Node&lt;E&gt; last;
}

private static class Node&lt;E&gt; {
        E item;
        Node&lt;E&gt; next;
        Node&lt;E&gt; prev;
}</code></pre>
    <p>
     通过源码可以看出，LinkedList实现是基于双向链表的数据结构。
    </p>
    <p>
     使用双向链表而不使用单链表是为了能支持反向遍历，快速删除尾部元素
    </p>
    <p>
    </p>
    <blockquote>
     <p>
      接下来，我们分析下不同操作的情况下，它两的差异
     </p>
    </blockquote>
    <h3>
     列表头部插入
    </h3>
    <p id="u4b5b748e">
     ArrayList每次头插都需要移动所有元素， 时间复杂度O（n^2）
    </p>
    <p id="u6cfd3def">
     LinkedList只需要修改指针，时间复杂度O（n）
    </p>
    <h3>
     随机访问
    </h3>
    <p id="u18357e42">
     ArrayList直接通过索引计算内存地址，时间复杂度O（1）
    </p>
    <p id="u5ffbaa52">
     LinkedList需要从头节点开始遍历，时间复杂度O（n）
    </p>
    <h3 id="LupfK">
     空间复杂度对比
    </h3>
    <p id="u0e817f6e">
     LinkedList比ArrayList多占用数倍的空间，因为LinkedList要多存前后指针等信息
    </p>
