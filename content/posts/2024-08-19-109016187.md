    <h3>
     <a id="_1">
     </a>
     软件危机
    </h3>
    <ul>
     <li>
      <p>
       1968年，NATO在联邦德国的国际学术会议上提出软件危机一词，并提出软件工程一词。
       <br/>
       原因：需求不明、方法不对、规模变大、复杂度变高等。
      </p>
     </li>
     <li>
      <p>
       中国软件危机：民族软件的严重短缺。
      </p>
     </li>
     <li>
      <p>
       软件危机目前仍然以各种形式的“软件困境”的形式错在与各类信息化项目中。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2500e99cdc7c761ee125b67d2ede18f5.png#pic_center"/>
      </p>
     </li>
    </ul>
    <h3>
     <a id="_10">
     </a>
     软件工程定义
    </h3>
    <p>
     中文名：软件工程
     <br/>
     外文名：Softwave Engineering
     <br/>
     简称：SE
    </p>
    <p>
     常见的定义：
    </p>
    <ul>
     <li>
      <p>
       BarryBoehm：运用现代科学技术知识来设计并构造计算机程序及为开发、运行和维护这些程序所必需的相关文件资料。
      </p>
     </li>
     <li>
      <p>
       IEEE：在软件工程术语汇编中的定义：软件工程是：1.将系统化的、严格约束的、可量化的方法应用于软件的开发、运行和维护，即将工程化应用于软件；2.在1中所述方法的研究
      </p>
     </li>
     <li>
      <p>
       FritzBauer在NATO会议上给出的定义：建立并使用完善的工程化原则，以较经济的手段获得，能在实际机器上有效运行的可靠软件的一系列方法。
      </p>
      <p>
       还有《计算机科学技术百科全书》以及行业内的一些约定俗成的各类定义。
      </p>
     </li>
    </ul>

    <h3>
     <a id="_29">
     </a>
     软件过程
    </h3>
    <p>
     软件过程（Software Procedure）是指：
     <u>
      软件声明周期
     </u>
     所设计的一系列相关过程。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ca39c8dd9dd6846c1d0914985ea4e34a.png#pic_center"/>
    </p>
    <ul>
     <li>
      软件过程=软件生命周期过程
     </li>
     <li>
      过程：事务发展的阶段（区间段落）
     </li>
     <li>
      过程是活动的集合，活动是任务的集合
     </li>
     <li>
      如需求活动、设计活动。
     </li>
     <li>
      需求活动下要分为若干任务
     </li>
     <li>
      常见的软件过程模型:瀑布模型、快速原型模型、增量模型、螺旋模型、喷泉模型、Rational统一过程、敏捷过程与极限编程等。
     </li>
    </ul>
    <h3>
     <a id="_38">
     </a>
     软件方法
    </h3>
    <p>
     方法是一个汉语词汇，方法的
     <mark>
      含义
     </mark>
     较
     <mark>
      广泛
     </mark>
     ，一般是指为获得某种东西或达到某种目的而采取的手段与行动方式。
    </p>
    <p>
     <strong>
      在分析和设计的方法主要有：
     </strong>
    </p>
    <ul>
     <li>
      SASD方法，1978年，E.Yourdon和L.L.Constantine
     </li>
     <li>
      面向数据结构的软件开发方法，Jackson程序结构图
     </li>
     <li>
      OMT（Object Modelling Technique）
     </li>
     <li>
      可视化开发方法
     </li>
     <li>
      ICASE（Integrated Computer-Aided Software Engineering）
     </li>
     <li>
      软件重用和组件连接
      <br/>
      <strong>
       其它的方法还包括：
      </strong>
     </li>
     <li>
      项目计划与进度跟踪方法
     </li>
     <li>
      成本估算方法
     </li>
     <li>
      质量管理方法
     </li>
    </ul>

    </p>

      项目管理工具。
     </li>
     <li>
      配置管理工具。
     </li>
     <li>
      分析和设计工具。
     </li>
     <li>
      测试工具。
     </li>
     <li>
      维护工具。
     </li>
