    <h3>
     <a id="3_BFCBFC_169">
     </a>
     3. 对BFC的理解，如何创建BFC
    </h3>
    <p>
     BFC (Block Formatting Context) 块级格式上下文，是 CSS 中的一种布局机制，一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。【用于解决高度塌缩、边距折叠等】
     <br/>
     如何触发：
    </p>
    <ul>
     <li>
      根元素 默认生成一个 BFC；
     </li>
     <li>
      float 值非 none；
     </li>
     <li>
      overflow值非 visible；
     </li>
     <li>
      position 值为absoute、fixed；
     </li>
     <li>
      display 值为 inline-block、table-cell、flex、grid、【新属性 flow-root】
     </li>
    </ul>
    <p>
     <strong>
      1.解决高度塌陷问题
     </strong>
     <br/>
     浮动元素会脱离文档流（绝对定位元素也会脱离文档流），导致无法计算准确的高度
     <br/>
     ( 给父级 )清除浮动：解决高度塌陷的问题是前提能够识别并且包含浮动元素
     <br/>
     使用伪类清除浮动
    </p>
    <pre><code class="prism language-css"><span class="token selector">.container:after</span> <span class="token punctuation">{<!-- --></span>
        <span class="token property">content</span><span class="token punctuation">:</span> <span class="token string">""</span><span class="token punctuation">;</span>
        <span class="token property">display</span><span class="token punctuation">:</span> block<span class="token punctuation">;</span>
        <span class="token property">clear</span><span class="token punctuation">:</span> both<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
</code></pre>
    <p>
     :after 选择器向选定的元素之后插入内容
     <br/>
     content:“”; 生成内容为空
     <br/>
     display: block; 生成的元素以块级元素显示,
     <br/>
     clear:both; 清除前面元素浮动带来的影响
     <br/>
     <strong>
      2. 解决margin 重叠：
     </strong>
     <br/>
     相邻的两个盒子（可能是兄弟关系也可能是祖先关系）的垂直边距相遇时，它们将形成一个外边距。这个外边距的高度等于两个发生折叠外边距高度中的较大者
     <br/>
     <strong>
      3. 解决margin 击穿
     </strong>
     <br/>
     子元素内的 margin-top，作用在了父元素的上
     <br/>
     解决方案：透明边框（border: 1px solid transparent）
    </p>
    <h3>
     <a id="4_li__li__202">
     </a>
     4. li 与 li 之间有看不见的空白间隔是什么原因引起的？如何解决？
    </h3>
    <p>
     HTML 空白字符渲染浏览器会将
    </p>
    <ul>
     <li>
      标签间的换行、缩进等空白字符渲染成一个空格，类似于文本排版中的自然间隔。inline-block 的布局特性 会保留块盒属性，同时像文本一样排列，因此空白字符会影响布局。
      <br/>
      解决方法：
     </li>
     <li>
      <p>
      </p>
      <ol>
       <li>
        消除
        <ul>
         <li>
          的字符间隔 letter-spacing:-8px，不足：这也设置了
         </li>
         <li>
          内的字符间隔，因此需要将
         </li>
         <li>
          内的字符间隔设为默认letter-spacing:normal。
         </li>
         <li>
          使用 flex 或者 grid 布局
         </li>
        </ul>
       </li>
      </ol>

       <a id="5_link__import__209">
       </a>
       5. link 和 import 区别
      </h3>
      <table>
       <thead>
        <tr>
         <th>
          区别点
         </th>
         <th>
          link
         </th>
         <th>
          @import
         </th>
        </tr>
       </thead>
       <tbody>
        <tr>
         <td>
          从属关系
         </td>
         <td>
          HTML 标签（属于 HTML 规范）
         </td>
         <td>
          CSS 语法（属于 CSS2.1 规范）
         </td>
        </tr>
        <tr>
         <td>
          语法
         </td>
         <td>
          &lt; link href=“style.css” rel=“stylesheet”&gt;（写在 HTML 中）
         </td>
         <td>
          @import url(“style.css”);（写在 CSS 文件或 &lt; style&gt; 标签内，且必须在顶部）
         </td>
        </tr>
        <tr>
         <td>
          加载顺序
         </td>
         <td>
          与 HTML 并行加载，不阻塞页面渲染
         </td>
         <td>
          需等待 HTML 加载完成后再加载 CSS，可能导致 FOUC（短暂无样式）
         </td>
        </tr>
        <tr>
         <td>
          兼容性
         </td>
         <td>
          所有浏览器（包括 IE5+）
         </td>
         <td>
          IE5+ 及现代浏览器（IE4 及更低不支持）
         </td>
        </tr>
        <tr>
         <td>
          DOM 可控性
         </td>
         <td>
          可通过 JS 动态插入、删除或修改（如切换主题）
         </td>
         <td>
          无法通过 JS 动态操作，需直接修改 CSS 文件
         </td>
        </tr>
        <tr>
         <td>
          权重与覆盖
         </td>
         <td>
          优先级较高，后加载的样式覆盖先加载的
         </td>
         <td>
          优先级较低，即使后加载也可能被 样式覆盖
         </td>
        </tr>
        <tr>
         <td>
          媒体查询支持
         </td>
         <td>
          支持 media 属性（如 media=“print”）
         </td>
         <td>
          支持在 @import 后加媒体条件（如 @import url(“print.css”) print;）
         </td>
        </tr>
        <tr>
         <td>
          模块化与代码复用
         </td>
         <td>
          需手动在 HTML 中引入多个文件
         </td>
         <td>
          可在 CSS 文件中嵌套引入其他文件（如 @import “base.css”;）
         </td>
        </tr>
        <tr>
         <td>
          预加载优化
         </td>
         <td>
          支持 rel=“preload” 提前加载关键 CSS
         </td>
         <td>

       7.为什么有时候人们用translate来改变位置而不是定位如absolute或relative？
      </h3>
      <p>
       性能更优，GPU 加速，避免重排 。
      </p>
      <ul>
       <li>
        translate ：通过 transform：translate() 移动元素时，浏览器仅会触发合成（composite），利用 GPU 加速操作图层，跳过布局和绘制阶段，性能更优。
       </li>
       <li>
        GPU 加速 ：translate 的变换由 GPU 处理，适合高频次动画（如滚动、平移）
       </li>
      </ul>
