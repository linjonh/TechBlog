    <h3>
     <a id="_10">
     </a>
     前言
    </h3>
    <p>
     在使用vue中，我们知道直接使用fliter就可以实现过滤器的功能，但是在原生微信小程序中没有这个概念，官方给出了一种
     <a href="https://developers.weixin.qq.com/miniprogram/dev/framework/view/wxs/" rel="nofollow">
      WXS 脚本语言
     </a>
     ，下面我来介绍一下在小程序中如何使用它来实现过滤器功能。
    </p>
    <hr/>
    <h3>
     <a id="WXS__17">
     </a>
     一、WXS 简介
    </h3>
    <h5>
     <a id="1_wxs_18">
     </a>
     1. 什么是wxs
    </h5>
    <p>
     wxs(WeiXin Script) 是小程序独有的一套脚本语言，结合wxml，可以构建出页面的结构。
    </p>
    <h5>
     <a id="2_wxs_20">
     </a>
     2. wxs的应用场景
    </h5>
    <p>
     wxml中无法调用在页面的js中定义的函数，但是wxml可以调用wxs中定义的函数，因此，小程序中wxs的典型应用场景是‘过滤器’
    </p>
    <h5>
     <a id="3_wxsJavaScript_22">
     </a>
     3. wxs和JavaScript的关系
    </h5>
    <ul>
     <li>
      wxs的语法类似于JavaScript，但是wxs和JavaScript时完全不同的两种语言
     </li>
     <li>
      wxs有自己的数据类型
     </li>
     <li>
      wxs不支持类似于ES6及以上的语法
     </li>
     <li>
      wxs遵守CommonJS规范
     </li>
    </ul>
    <h5>
     <a id="4_wxs_28">
     </a>
     4. wxs的特点
    </h5>
    <ul>
     <li>
      <p>
       与JavaScript不同
      </p>
     </li>
     <li>
      <p>
       不能作为组件的时间回调
      </p>
     </li>
     <li>
      <p>
       隔离性
       <br/>
       wxs的运行环境和其他JavaScript代码是隔离的，
       <br/>
       wxs不能调用js中定义的函数，
       <br/>
       wxs不能调用小程序提供的API
      </p>
     </li>
     <li>
      <p>
       性能好
       <br/>
       在IOS设备上，小程序内wxs会比JavaScript代码块2-20倍,
       <br/>
       在Android中，二者相差不大
      </p>
     </li>
    </ul>
