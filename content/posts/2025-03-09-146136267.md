    <p>
     <strong>
      原型
     </strong>
     和
     <strong>
      原型链
     </strong>
     是 JavaScript 中实现对象继承和属性查找的核心机制。为了更深入地理解它们，我们需要从底层原理、实现机制以及实际应用等多个角度进行分析。
    </p>
    <hr/>
    <h3>
     <a id="1_Prototype_4">
     </a>
     1. 原型（Prototype）
    </h3>
    <h4>
     <a id="11__6">
     </a>
     1.1 什么是原型？
    </h4>
    <ul>
     <li>
      每个 JavaScript 对象（除
      <code>
       null
      </code>
      外）都有一个内部属性
      <code>
       [[Prototype]]
      </code>
      ，指向它的原型对象。
     </li>
     <li>
      原型对象也是一个普通对象，它包含可以被其他对象共享的属性和方法。
     </li>
     <li>
      通过
      <code>
       __proto__
      </code>
      （非标准，但广泛支持）或
      <code>
       Object.getPrototypeOf()
      </code>
      可以访问对象的原型。
     </li>
    </ul>
    <h4>
     <a id="12__11">
     </a>
     1.2 构造函数与原型
    </h4>
    <ul>
     <li>
      每个函数（构造函数）都有一个
      <code>
       prototype
      </code>
      属性，指向一个对象，这个对象是该构造函数实例的原型。
     </li>
     <li>
      当使用
      <code>
       new
      </code>
      关键字创建实例时，实例的
      <code>
       [[Prototype]]
      </code>
      会指向构造函数的
      <code>
       prototype
      </code>
      对象。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">function</span> <span class="token function">Person</span><span class="token punctuation">(</span><span class="token parameter">name</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">this</span><span class="token punctuation">.</span>name <span class="token operator">=</span> name<span class="token punctuation">;</span>
<span class="token punctuation">}</span>

<span class="token comment">// 在构造函数的原型上添加方法</span>
<span class="token class-name">Person</span><span class="token punctuation">.</span>prototype<span class="token punctuation">.</span><span class="token function-variable function">sayHello</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token template-string"><span class="token template-punctuation string">`</span><span class="token string">Hello, my name is </span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span><span class="token keyword">this</span><span class="token punctuation">.</span>name<span class="token interpolation-punctuation punctuation">}</span></span><span class="token template-punctuation string">`</span></span><span class="token punctuation">)</span><span class="token punctuation">;</span>

person1<span class="token punctuation">.</span><span class="token function">sayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 输出: Hello, my name is Alice</span>
</code></pre>
    <ul>
     <li>
      在上面的例子中：
      <ul>
       <li>
        <code>
         Person.prototype
        </code>
        是
        <code>
         person1
        </code>
        的原型对象。
       </li>
       <li>
        <code>
         person1.__proto__
        </code>
        指向
        <code>
         Person.prototype
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2_Prototype_Chain_35">
     </a>
     2. 原型链（Prototype Chain）
    </h3>
    <h4>
     <a id="21__37">
     </a>
     2.1 什么是原型链？
    </h4>
    <ul>
     <li>
      原型链是由对象的
      <code>
       [[Prototype]]
      </code>
      连接起来的链式结构。
     </li>
     <li>
      当访问对象的属性或方法时，如果对象本身没有该属性，JavaScript 会沿着原型链向上查找，直到找到该属性或到达原型链的顶端（
      <code>
       null
      </code>
      ）。
     </li>
    </ul>
    <h4>
     <a id="22__41">
     </a>
     2.2 原型链的顶端
    </h4>
    <ul>
     <li>
      所有对象的原型链最终都会指向
      <code>
       Object.prototype
      </code>
      ，而
      <code>
       Object.prototype
      </code>
      的
      <code>
       [[Prototype]]
      </code>
      是
      <code>
       null
      </code>
      。
     </li>
     <li>
      如果在整个原型链中都找不到属性，则返回
      <code>
       undefined
      </code>
      。
     </li>
    </ul>
    <pre><code class="prism language-javascript">console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span>person1<span class="token punctuation">.</span><span class="token function">toString</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 输出: [object Object]</span>
</code></pre>
    <ul>
     <li>
      在上面的例子中：
      <ul>
       <li>
        <code>
         person1
        </code>
        本身没有
        <code>
         toString
        </code>
        方法。
       </li>
       <li>
        JavaScript 引擎会沿着原型链查找：
        <ol>
         <li>
          <code>
           person1
          </code>
          -&gt; 没有
          <code>
           toString
          </code>
          。
         </li>
         <li>
          <code>
           person1.__proto__
          </code>
          （即
          <code>
           Person.prototype
          </code>
          ） -&gt; 没有
          <code>
           toString
          </code>
          。
         </li>
         <li>
          <code>
           Person.prototype.__proto__
          </code>
          （即
          <code>
           Object.prototype
          </code>
          ） -&gt; 找到
          <code>
           toString
          </code>
          ，调用它。
         </li>
        </ol>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="3__58">
     </a>
     3. 原型链的深度分析
    </h3>
    <h4>
     <a id="31__60">
     </a>
     3.1 原型链的构建
    </h4>
    <ul>
     <li>
      原型链是通过
      <code>
       [[Prototype]]
      </code>
      连接起来的。
     </li>
     <li>
      当我们创建一个对象时，它的
      <code>
       [[Prototype]]
      </code>
      会指向某个原型对象。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">const</span> obj <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span><span class="token punctuation">}</span><span class="token punctuation">;</span>

</code></pre>
    <ul>
     <li>
      如果我们手动修改
      <code>
       [[Prototype]]
      </code>
      ，可以构建自定义的原型链。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">const</span> parent <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span> <span class="token literal-property property">name</span><span class="token operator">:</span> <span class="token string">'Parent'</span> <span class="token punctuation">}</span><span class="token punctuation">;</span>
<span class="token keyword">const</span> child <span class="token operator">=</span> Object<span class="token punctuation">.</span><span class="token function">create</span><span class="token punctuation">(</span>parent<span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// child.__proto__ 指向 parent</span>

</code></pre>
    <h4>
     <a id="32__77">
     </a>
     3.2 原型链的查找机制
    </h4>
    <ul>
     <li>
      当访问对象的属性时，JavaScript 引擎会按照以下步骤查找：
      <ol>
       <li>
        检查对象自身是否有该属性。
       </li>
       <li>
        如果没有，沿着
        <code>
         [[Prototype]]
        </code>
        向上查找。
       </li>
       <li>
        重复这个过程，直到找到属性或到达
        <code>
         null
        </code>
        。
       </li>
      </ol>
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">const</span> grandparent <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span> <span class="token literal-property property">familyName</span><span class="token operator">:</span> <span class="token string">'Smith'</span> <span class="token punctuation">}</span><span class="token punctuation">;</span>
<span class="token keyword">const</span> parent <span class="token operator">=</span> Object<span class="token punctuation">.</span><span class="token function">create</span><span class="token punctuation">(</span>grandparent<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">const</span> child <span class="token operator">=</span> Object<span class="token punctuation">.</span><span class="token function">create</span><span class="token punctuation">(</span>parent<span class="token punctuation">)</span><span class="token punctuation">;</span>

</code></pre>
    <ul>
     <li>
      在上面的例子中：
      <ul>
       <li>
        <code>
         child
        </code>
        自身没有
        <code>
         familyName
        </code>
        。
       </li>
       <li>
        <code>
         child.__proto__
        </code>
        （即
        <code>
         parent
        </code>
        ）也没有
        <code>
         familyName
        </code>
        。
       </li>
       <li>
        <code>
         parent.__proto__
        </code>
        （即
        <code>
         grandparent
        </code>
        ）有
        <code>
         familyName
        </code>
        ，返回
        <code>
         Smith
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>

    <h3>
     <a id="4__98">
     </a>
     4. 原型链与继承
    </h3>
    <h4>
     <a id="41__100">
     </a>
     4.1 基于原型的继承
    </h4>
    <ul>
     <li>
      JavaScript 通过原型链实现继承。
     </li>
     <li>
      子类的原型对象指向父类的实例，从而继承父类的属性和方法。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">function</span> <span class="token function">Person</span><span class="token punctuation">(</span><span class="token parameter">name</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

        <code>

       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="42__130">
     </a>
     4.2 原型链的局限性
    </h4>
    <ul>
     <li>
      原型链继承是单向的，子类可以访问父类的属性和方法，但父类不能访问子类的属性和方法。
     </li>
     <li>
      如果原型链过长，属性查找的性能会受到影响。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="5__136">
     </a>
     5. 原型链的实际应用
    </h3>
    <h4>
     <a id="51__138">
     </a>
     5.1 共享方法
    </h4>
    <ul>
     <li>
      将方法定义在原型上，可以避免每个实例都创建一份方法的副本，节省内存。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token keyword">function</span> <span class="token function">Person</span><span class="token punctuation">(</span><span class="token parameter">name</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

<span class="token class-name">Person</span><span class="token punctuation">.</span>prototype<span class="token punctuation">.</span><span class="token function-variable function">sayHello</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token template-string"><span class="token template-punctuation string">`</span><span class="token string">Hello, my name is </span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span><span class="token keyword">this</span><span class="token punctuation">.</span>name<span class="token interpolation-punctuation punctuation">}</span></span><span class="token template-punctuation string">`</span></span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>

<span class="token keyword">const</span> person2 <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">Person</span><span class="token punctuation">(</span><span class="token string">'Bob'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

    <h4>
     <a id="52__157">
     </a>
     5.2 扩展内置对象
    </h4>
    <ul>
     <li>
      可以通过修改内置对象的原型来扩展其功能。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token class-name">Array</span><span class="token punctuation">.</span>prototype<span class="token punctuation">.</span><span class="token function-variable function">last</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

    <h3>
     <a id="6__171">
     </a>
     6. 总结
    </h3>
    <ul>
     <li>
      <strong>
       原型
      </strong>
      ：每个对象都有一个
      <code>
       [[Prototype]]
      </code>
      ，指向它的原型对象。
     </li>
     <li>
      <strong>
       原型链
      </strong>
      ：通过
      <code>
       [[Prototype]]
      </code>
      连接起来的链式结构，用于属性查找。
     </li>
     <li>
      <strong>
       继承
      </strong>
      ：通过原型链实现对象之间的继承。
     </li>
     <li>
      <strong>
       性能
      </strong>
      ：原型链过长会影响查找性能，需谨慎设计。
     </li>
    </ul>
    <p>
     理解原型和原型链是掌握 JavaScript 面向对象编程的关键，也是深入理解 JavaScript 运行机制的基础。
