    <h3 id="1.传输工具的准备" style="background-color:transparent">
     1. 准备传输工具
    </h3>
    <p>
     本方案采用 Apache Seatunnel（简称seatunnel）进行MySQL 到 OceanBase 的数据迁移和同步，出于对方案轻量性的考量，我们采用其内置的Zeta引擎来实现，包括全量同步、离线增量同步，以及CDC方案。
    </p>
    <h4 id="1.1运行环境准备">
     1.1 运行环境准备
    </h4>
    <p>
     自行安装运行环境JAVA，推荐版本为8，但理论上高于8的版本也能正常工作。安装后，请确保已正确配置JAVA_HOME
    </p>
    <pre><code>root:~# java -version
openjdk version "17.0.12" 2024-07-16
OpenJDK Runtime Environment (build 17.0.12+7-Debian-2deb11u1)
OpenJDK 64-Bit Server VM (build 17.0.12+7-Debian-2deb11u1, mixed mode, sharing)java</code></pre>
    <h4 id="1.2下载并安装seatunnel">
     1.2 下载并安装seatunnel
    </h4>
    <p>
     进入官网下载页面，下载适合版本seatunnel :
     <a href="https://seatunnel.apache.org/download/" rel="nofollow" title="Apache SeaTunnel">
      Apache SeaTunnel
     </a>
    </p>
    <p>
     我这里选择最新版本2.3.7
    </p>
    <pre><code>下载
wget https://dlcdn.apache.org/seatunnel/2.3.7/apache-seatunnel-2.3.7-bin.tar.gz
解压
tar -zxvf apache-seatunnel-2.3.7-bin.tar.gz </code></pre>
    <h4 id="1.3连接器插件的下载与安装">
     1.3 连接器插件的下载与安装
    </h4>
    <p>
     seatunnel安装包只包含 主体文件与Zeta引擎，连接不同数据源的插件需要手动下载并配置
    </p>
    <h5 id="1.3.1自动下载方案">
     1.3.1 自动下载方案
    </h5>
    <p>
     通过配置config/plugin_config 文件来指定 你需要的连接器 默认文件里是全方案的 可以根据你的需要增删一些 我们这里只包含这次演示 需要使用的连接库
    </p>
    <pre><code>connector-cdc-mysql
connector-jdbc
connector-fake
connector-console</code></pre>
    <p>
     然后输入命令
    </p>
    <pre><code>sh bin/install-plugin.sh 2.3.7
</code></pre>
    <p>
     则开始自动下载连接器
    </p>
    <h5 style="background-color:transparent">
     1.3.2  手动下载方案
    </h5>
    <p>
     进入网站
     <a href="https://repo.maven.apache.org/maven2/org/apache/seatunnel/" rel="nofollow" title="Central Repository: org/apache/seatunnel">
      Central Repository: org/apache/seatunnel
     </a>
    </p>
    <p>
     找寻自己需要的插件例如：
    </p>
    <pre><code>connector-cdc-mysql-2.3.7.jar
connector-console-2.3.7.jar
connector-fake-2.3.7.jar
connector-jdbc-2.3.7.jar
seatunnel-transforms-v2-2.3.7.jar</code></pre>
    <p>
     将文件手动下载之后 放入connectors
    </p>
    <h5 style="background-color:transparent">
     1.3.3  验证连接器安装情况
    </h5>
    <pre><code>./bin/seatunnel-connector.sh -l

Source
FakeSource MySQL-CDC Jdbc


Sink
Jdbc Console


Transform
Copy DynamicCompile FieldMapper Filter FilterRowKind JsonPath LLM Replace Split Sql</code></pre>
    <p>
     由于我们是使用JDBC 使用MySQL的连接方式去操作oceanbase 所以还需要下载一个jdbc-mysql
    </p>
    <p>
     前往MySQL官网下载jdbc
    </p>
    <p>
     <a href="https://downloads.mysql.com/archives/c-j/" rel="nofollow" title="MySQL :: Download MySQL Connector/J (Archived Versions)">
      MySQL :: Download MySQL Connector/J (Archived Versions)
     </a>
    </p>
    <p>
     将下载的mysql-connector-j-9.0.0.jar 放到 {seatunnel/lib}中
    </p>

    <h4 style="background-color:transparent">
     2.4 CDC同步配置文件编写
    </h4>
    <h5 style="background-color:transparent">
     2.4.1 手动同步表结构
    </h5>
    <p>
     由于seatunnel的oceanbase组件还是有所问题 表结构同步 遇到报错比较麻烦 这一步还是手动同步
    </p>
    <h5 style="background-color:transparent">
     2.4.2 检查MySQL  binlog状态
    </h5>
    <p>
     赋予用户所需权限
    </p>
    <pre><code>mysql&gt; GRANT SELECT, RELOAD, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'user' IDENTIFIED BY 'password';

mysql&gt; FLUSH PRIVILEGES;</code></pre>
    <p>
     检查binlog日志是否开启
    </p>
    <pre><code>mysql&gt; show variables where variable_name in ('log_bin', 'binlog_format', 'binlog_row_image', 'gtid_mode', 'enforce_gtid_consistency');
+--------------------------+----------------+
| Variable_name            | Value          |
+--------------------------+----------------+
| binlog_format            | ROW            |
| binlog_row_image         | FULL           |
| enforce_gtid_consistency | ON             |
| gtid_mode                | ON             |
| log_bin                  | ON             |
+--------------------------+----------------+
5 rows in set (0.00 sec)
</code></pre>
    <p>
     如果不一致 请自行更改mysql.cnf文件
    </p>
    <p>
     在大型数据库创建一致性快照时 可能会存在读超时 请合理配置
    </p>
    <p>
     interactive_timeoutwait_timeout
    </p>
    <p>
     在处理完准备工作之后编写配置文件
    </p>
    <pre><code>env {
  parallelism = 1
  job.mode = "STREAMING"
  checkpoint.interval = 10000
}

source {
  MySQL-CDC {
    base-url = "jdbc:mysql://127.0.0.1:3306/mysql"
    username = "xxx"
    password = "xxx@xxx"
    table-names = ["seatunnel.table1", "seatunnel.table2"]
    
    startup.mode = "initial"
  }
}

sink {
    jdbc {
        url = "jdbc:mysql://127.0.0.1:2883/mysql?&amp;useUnicode=true&amp;characterEncoding=UTF-8&amp;rewriteBatchedStatements=true"
        driver = "com.mysql.cj.jdbc.Driver"
        user = "xxx@xxx"
        password = "xxx"
        database = "seatunnel"  # 目标数据库
        table-names = ["seatunnel.table1", "seatunnel.table2"]
        generate_sink_sql = true     # 自动生成 SQL
    }
}</code></pre>
    <p>
    </p>
    <p>
     正常启动之后 会进行历史数据迁移 再进行cdc变更
    </p>
    <p>
     注意:
    </p>
    <p>
     启动之后会根据配置的表和startup.mode选择的模式 进行不同的操作
    </p>
    <p>
     startup.mode操作如下:initial 启动时同步历史数据，然后同步增量数据earliest 从最早的偏移量启动latest 从最新偏移量启动specific 从用户提供的特定偏移量启动
    </p>
    <p>
     如果使用specific 需要添加startup.specific-offset.file binlog文件名startup.specific-offset.pos binlog偏转量
    </p>
