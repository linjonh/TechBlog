     如何上传文件到github

  <div class="article_content clearfix" id="article_content">

    <p>
    </p>
    <p>
     在 GitHub 上传文件夹，可以使用 Git 命令行（推荐）或者 GitHub 网页端。
    </p>
    <hr/>
    <h4>
     <a id="_1_Git__6">
     </a>
     <strong>
      方法 1：使用 Git 命令行（推荐）
     </strong>
    </h4>
    <p>
     适用于本地有 Git 并希望管理多个文件的情况。
    </p>
    <h5>
     <a id="_1_Git__9">
     </a>
     <strong>
      步骤 1：初始化 Git 仓库（如果还没有）
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> init
</code></pre>
    <h5>
     <a id="_2_14">
     </a>
     <strong>
      步骤 2：添加远程仓库
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token function">add</span> origin https://github.com/你的用户名/你的仓库名.git
</code></pre>
    <h5>
     <a id="_3_19">
     </a>
     <strong>
      步骤 3：添加整个文件夹并提交
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> 文件夹名
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"添加文件夹"</span>
</code></pre>
    <h5>
     <a id="_4_GitHub_25">
     </a>
     <strong>
      步骤 4：推送到 GitHub
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> push <span class="token parameter variable">-u</span> origin main
</code></pre>
    <blockquote>
     <p>
      如果
      <code>
       main
      </code>
      分支不存在，可能需要先运行
      <code>
       git branch -M main
      </code>
      。
     </p>
    </blockquote>
    <hr/>
    <h4>
     <a id="_2GitHub__33">
     </a>
     <strong>
      方法 2：GitHub 网页端（适用于小型文件夹）
     </strong>
    </h4>
    <p>
     GitHub 本身不支持直接上传整个文件夹，但可以手动上传多个文件：
    </p>
    <ol>
     <li>
      进入你的 GitHub 仓库。
     </li>
     <li>
      点击
      <strong>
       “Add file” → “Upload files”
      </strong>
      。
     </li>
     <li>
      <strong>
       手动拖拽文件夹内的文件
      </strong>
      到上传区域（GitHub 不允许直接上传文件夹，只能上传里面的文件）。
     </li>
     <li>
      提交变更 (
      <code>
       Commit changes
      </code>
      )。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_3_GitHub_Desktop_42">
     </a>
     <strong>
      方法 3：使用 GitHub Desktop（适合不熟悉命令行的用户）
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       下载并安装 GitHub Desktop
      </strong>
      ：
      <a href="https://desktop.github.com/" rel="nofollow">
       GitHub Desktop
      </a>
     </li>
     <li>
      <strong>
       克隆仓库
      </strong>
      ：打开 GitHub Desktop，选择
      <code>
       File -&gt; Clone Repository
      </code>
      ，克隆你的 GitHub 仓库。
     </li>
     <li>
      <strong>
       添加文件夹
      </strong>
      ：将文件夹拖入本地仓库文件夹中。
     </li>
     <li>
      <strong>
       提交并推送
      </strong>
      ：
      <ul>
       <li>
        在 GitHub Desktop 界面点击
        <strong>
         “Commit to main”
        </strong>
        。
       </li>
       <li>
        然后点击
        <strong>
         “Push origin”
        </strong>
        ，上传到 GitHub。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_52">
     </a>
     <strong>
      总结
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        方法
       </th>
       <th>
        适用场景
       </th>
       <th>
        是否推荐
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         Git 命令行
        </strong>
       </td>
       <td>
        适用于开发者，支持完整版本控制
       </td>
       <td>
        ✅ 推荐
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         GitHub 网页端
        </strong>
       </td>
       <td>
        适用于小型文件夹，手动上传
       </td>
       <td>
        ❌ 仅适合少量文件
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         GitHub Desktop
        </strong>
       </td>
       <td>
        适用于不熟悉 Git 命令行的用户
       </td>
       <td>
        ✅ 适合新手
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     如果你的文件夹很大（如超过 100MB），可能需要使用
     <strong>
      Git LFS
     </strong>
     进行管理。
    </p>
