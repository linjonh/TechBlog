    <h3>
     一、为何需要文件忽略机制？
    </h3>
    <h4>
     1.1 为什么要会略？
    </h4>
    <p>
     对于开发者而言，明智地选择忽略某些文件类型，能带来三大核心优势：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        仓库纯净性
       </strong>
       ：避免二进制文件、编译产物等污染代码库
      </p>
     </li>
     <li>
      <p>
       <strong>
        安全防护
       </strong>
       ：防止敏感信息（如API密钥、数据库凭证）意外泄露
      </p>
     </li>
     <li>
      <p>
       <strong>
        效率提升
       </strong>
       ：减少无意义的版本追踪，加速克隆和拉取操作
      </p>
     </li>
    </ul>

    <h3>
     二、.gitignore配置深度解析
    </h3>
    <h4>
     2.1 文件作用域与优先级
    </h4>
    <blockquote>
     <p>
      # 层级生效机制（从上到下优先级递减）
      <br/>
      1. 项目根目录/.gitignore         # 作用于整个项目
      <br/>
      2. 子目录/.gitignore                # 作用于该目录及其子目录
      <br/>
      3. $GIT_DIR/info/exclude      # 本地仓库级配置
      <br/>
      4. ~/.gitignore_global             # 全局配置（需执行git config --global core.excludesfile ~/.gitignore_global）
     </p>
    </blockquote>
    <h4>
     2.2 高级语法规则
    </h4>
    <blockquote>
     <p>
      # 基础匹配
      <br/>
      *.log                    # 所有.log文件
      <br/>
      !error.log             # 排除规则（保留error.log）
     </p>
     <p>
      # 目录匹配
      <br/>
      build/                  # 忽略所有build目录
      <br/>
      doc/*.txt             # 忽略doc目录下txt文件（不递归子目录）
     </p>
     <p>
      # 通配符扩展
      <br/>
      temp?                # 匹配temp后接一个字符的文件（如tempa、temp1）
      <br/>
      config.[tj]s          # 匹配config.ts和config.js
     </p>
     <p>
      # 注释与范围
      <br/>
      [0-9].csv             # 匹配数字开头的csv文件
      <br/>
      # 重要：不要提交密钥文件
      <br/>
      secret.key
     </p>
    </blockquote>
    <p>
     <span style="color:#494949">
      <code>
       .gitignore
      </code>
      的规则是 从上到下逐行匹配 的，后面的规则可以覆盖前面的规则。
     </span>
    </p>
    <p>
     否定规则（
     <code>
      !
     </code>
     ）是唯一可以覆盖前面忽略规则的方式。
    </p>

    <h4>
     3.2对于已经被git追踪的文件
    </h4>
    <p>
     如果是第一次提交，就会发现忽略文件已经没有了，不是第一次的这个时候你要是提交，就会发现你想要忽略的文件要是有改动，依然让你提交，这是因为这些文件已经加入git的版本控制库了，仅仅添加.gitignore文件是不够的，需要先清除缓存才行。
    </p>
    <p>
     3.2.1清除Git缓存：
    </p>
    <pre><code class="language-cs"># 删除Git缓存（不会删除实际文件）
git rm -r --cached .</code></pre>
    <p>
    </p>
    <p>
     <img alt="" height="611" src="https://i-blog.csdnimg.cn/direct/05b00b1fce3e423e8e72e109d2b2512d.png" width="1039"/>
    </p>
    <p>
     3.2.2 添加新的.gitignore文件，提交更改
    </p>
    <pre><code class="language-cs"># 添加.gitignore文件到Git
git add .gitignore

# 添加其他文件
git add .

# 提交更改
git commit -m "Add .gitignore file and remove ignored files from git"
</code></pre>
    <p>
     <img alt="" height="626" src="https://i-blog.csdnimg.cn/direct/6b137b408aaf469288e229ef8e9c1163.png" width="1041"/>
    </p>
    <p>
     3.3.3 推送到远程仓库
    </p>
    <pre><code class="language-cs"># 推送到远程仓库
git push origin master  # 或者其他分支名</code></pre>
    <p>
     <img alt="" height="626" src="https://i-blog.csdnimg.cn/direct/5c5d4a45a0fd4d6fa37b0caba1aee0ae.png" width="1025"/>
