    <h4>
     <a id="1uniapp_H5uni_sdk_6">
     </a>
     【坑1】怎么在uniapp H5中嵌入uni sdk？
    </h4>
    <ul>
     <li>
      根据官方提供的，h5配置中有个index.html模板路径，默认为空，可定制生成的html代码，自定义meta、引入外部js等，
      <a href="https://uniapp.dcloud.io/collocation/manifest?id=h5-template" rel="nofollow">
       参考
      </a>
     </li>
     <li>
      在项目根目录建立一个template.h5.html（仿照hello-uni-app项目），然后在h5配置中填入template.h5.html
      <br/>
      <img alt="image" src="https://i-blog.csdnimg.cn/blog_migrate/f7361b1e157be14b44f956e5fe96eca7.png"/>
     </li>
     <li>
      template.h5.html中的代码可以直接先复制官方提供的
      <a href="https://uniapp.dcloud.io/collocation/manifest?id=h5-template" rel="nofollow">
       h5-template
      </a>
      自定义模板的代码，或者下面的代码
     </li>
     <li>
      然后将uni-sdk等第三方引入到html中，注意这里就有个小坑：
      <mark>
       uni sdk放到body下面！下面！下面！
      </mark>
      这里不注意的话，可能会根据我们的习惯放到body里面。
     </li>
    </ul>

    <h4>
     <a id="2webview__uniapp_H5_80">
     </a>
     【坑2】web-view 加载 uni-app H5，内部跳转冲突如何解决?
    </h4>
    <ul>
     <li>
      该问题也是在web-view的下面的
      <a href="https://uniapp.dcloud.io/component/web-view?id=faq" rel="nofollow">
       FAQ
      </a>
      中有说明
     </li>
     <li>
      使用 uni.webView.navigateTo…，这样就避免了与内部uni.navigateTo的冲突
     </li>
     <li>
      注意：再次提醒下
      <mark>
       uni sdk放到body下面
      </mark>
      ，否则uni.webView是找不到的
      <br/>
      <img alt="image" src="https://i-blog.csdnimg.cn/blog_migrate/442ba1c1df2d34e327cfc8265e6e0620.png"/>
     </li>
    </ul>

    if(r != null) {
        // 对参数值进行解码

}
