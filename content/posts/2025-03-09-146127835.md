     <strong>
      还是选择从本地上传镜像到虚拟机root目录，再加载好
     </strong>
    </p>
    <pre id="iJbjE"><code>docker load -i	es.tar
docker load -i	kibana.tar</code></pre>
    <p id="u444b4ba9">
     加载好再移除掉tar包（比较大，有点占空间）
    </p>
    <pre id="d8T3N"><code>rm  -rf  *.tar</code></pre>
    <p id="ucf5837be">
     run
    </p>
    <pre id="idqsr"><code>docker run -d \
  --name es \
  -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" \
  -e "discovery.type=single-node" \
  -v es-data:/usr/share/elasticsearch/data \
  -v es-plugins:/usr/share/elasticsearch/plugins \
  --privileged \
  --network hmall \
  -p 9200:9200 \
  -p 9300:9300 \
  elasticsearch:7.12.1</code></pre>
    <pre id="TQihm"><code>docker run -d \
--name kibana \
-e ELASTICSEARCH_HOSTS=http://es:9200 \
--network hmall \
-p 5601:5601  \
kibana:7.12.1</code></pre>

    <h5 id="M3ngR" name="M3ngR">
     离线安装
    </h5>
    <p id="u323c3859">
     <strong>
      先查看有哪些数据卷：
     </strong>
    </p>
    <pre id="OEmMr"><code>docker volume ls</code></pre>
    <p id="u79024517">
    </p>
    <p class="img-center">
     <img alt="" height="138" id="u9f005ed4" src="https://i-blog.csdnimg.cn/img_convert/d29d2ab9fe5fefbe1654f203478e8c0b.png" width="678"/>
    </p>
    <p id="u6057ecd7">
     <strong>
      发现es容器里面的插件目录挂载好了（安装时通过数据卷挂载的）
     </strong>
    </p>
    <p id="u4f2fe653">
     <strong>
      查看一下挂载到虚拟机的哪个目录了：
     </strong>
    </p>
    <pre id="j68GD"><code>docker volume inspect es-plugins</code></pre>
    <p id="ue21a0053">
    </p>
    <p class="img-center">
     <img alt="" height="230" id="u89ef2f8a" src="https://i-blog.csdnimg.cn/img_convert/2455cd32e4a73802c9746d4a571d7100.png" width="586"/>
    </p>
    <p id="u5098032d">
     <strong>
      进入这个目录：
     </strong>
    </p>
    <pre id="tiSze"><code>cd	/var/lib/docker/volumes/es-plugins/_data</code></pre>
    <p id="ua892d2df">
     <strong>
      创建一个ik目录，把本地的压缩包
     </strong>
    </p>
    <p class="img-center">
     <img alt="" height="32" id="uef769d87" src="https://i-blog.csdnimg.cn/img_convert/ff86d44a6b13106b22bdf7c09adc5c34.png" width="295"/>
    </p>
    <p id="ua892d2df">
     <strong>
      上传到虚拟机的ik目录里，然后再解压，解压后再把压缩包删除了。
     </strong>
    </p>
    <pre id="mB5UX"><code>mkdir ik
unzip ***.zip
rm ****.zip</code></pre>
    <p id="u4b9efe6f">
     <strong>
      重新启动es：
     </strong>
    </p>
    <pre id="ewrBd"><code>docker restart es</code></pre>

    <h4 id="aA53N" name="aA53N">
     Mapper映射属性
    </h4>
    <p id="u8ce713a9">
    </p>
    <p class="img-center">
     <img alt="" height="472" id="u00a04c83" src="https://i-blog.csdnimg.cn/img_convert/40ef8674f374f0dcfe80c23d98c76a1f.png" width="995"/>
    </p>
    <p id="ueb0ea265">
     这里
     <strong>
      是否创建索引
     </strong>
     取决于
     <strong>
      这个字段参不参与搜索或者排序。
     </strong>
    </p>
    <p id="u4e159a78">
     注：
    </p>
    <p id="u97a06ac5">
     这个举例的
     <strong>
      es文档
     </strong>
     的
     <strong>
      score字段
     </strong>
     ，如果为其创建了index索引，用于去排序，
    </p>
    <p id="u94b84c13">
     排序规则：由于score是数组，里面有多个分数，那么去和
     <strong>
      其它文档
     </strong>
     根据这个字段排序的话，es会默认
     <strong>
      选一个分数
     </strong>
     去参与排序。
    </p>
    <p id="u6dbc1eb8">
     如果是升序排：
     <strong>
      拿最小的分数去比较（
     </strong>
     升序是小的在大的前面）
    </p>
    <p id="u97c7b18e">
     如果是降序排：
     <strong>
      拿最大的分数去比较（
     </strong>
     降序是大的在小的前面
     <strong>
      ）
     </strong>
    </p>
