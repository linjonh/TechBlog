    <h3>
     <a id="javajar_15">
     </a>
     怎么将写好的java脚本打成jar包
    </h3>
    <ol>
     <li>
      首先就是编写自己的脚本，构建自己的java项目。
      <br/>
      <code>
       Build system
      </code>
      建议选择
      <code>
       Maven
      </code>
      。对于写简单脚本并打包成 JAR 包导入 MeterSphere 的需求，使用 Maven 较为合适。虽然 IntelliJ 自带构建系统也能打包，但在依赖管理和标准化方面不如 Maven；Gradle 虽然灵活强大，但对于简单脚本打包，其配置相对复杂，所以 Maven 更符合我们的需求 。
      <br/>
      Groupid的话如果是为公司项目，公司域名为 example.com，那么 GroupId 可以设置为 com.example 。
      <br/>
      ArtifactId用来标识项目中的模块或特定的工件，用项目缩写即可。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fc1cac857c3448d5a8fb166f7188f42d.png"/>
     </li>
     <li>
      编写脚本文件，src - main - java - ‘Groupid’ 中新建脚本java文件，
      <code>
       写脚本时建议将后续可能会变的一些变量以传参的形式写入，不写死，
      </code>
      这样我们后续就不用维护这个包了:
      <br/>
      <img src="https://i-blog.csdnimg.cn/direct/2ef92c808e454b8292ad2a5883c0eeb3.png" width="44%"/>
     </li>
     <li>
      pom.xml添加依赖。如果有第三方依赖的话，打包时需要将第三方依赖一起打包，在使用AI生成pom.xml需要告诉AI这个需求。
      <br/>
      <code>
       PS：脚本文件，pom.xml这些都可以让AI帮你生成
      </code>
     </li>
     <li>
      将写好的脚本打包，使用 IntelliJ IDEA 中的 Maven 工具窗口
      <ul>
       <li>
        打开 Maven 工具窗口：在 IntelliJ IDEA 中，点击右侧的 “Maven” 图标，打开 Maven 项目面板。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f8f767dc21e148ea9318fdacc4827340.png"/>
       </li>
       <li>
        执行打包命令：在 Maven 项目面板中，展开项目，找到 Lifecycle 节点，双击其中的 package 选项。Maven 会按照配置的构建流程执行编译、测试等步骤，最后将项目打包成 JAR 文件。
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0e1aa1637bfb4e1eb94c22f9ff07b4b0.png"/>
       </li>
       <li>
        查看生成的 JAR 包：打包完成后，在项目的 target 目录下会生成一个 JAR 文件，如果打包了外部依赖的话会生成两个JAR文件。
        <code>
         original-jkx-1.0-SNAPSHOT.jar
        </code>
        ：这是 Maven 默认构建生成的普通 JAR 包，仅包含项目自身的代码，不包含第三方依赖。
        <code>
         jkx-1.0-SNAPSHOT.jar
        </code>
        ：添加插件配置后，插件会对默认生成的 JAR 包进行处理，将第三方依赖等内容一起打包进来，形成了最终可包含所有依赖的可执行 JAR 包。我们使用
        <code>
         jkx-1.0-SNAPSHOT.jar
        </code>
        。
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3869011807a4433f9983e4be8086aa99.png"/>
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="meterspherejar_30">
     </a>
     metersphere中使用自己的jar包
    </h3>
    <ol>
     <li>
      将JAR包上传到【项目设置】-【文件管理】中，点击【添加文件】上传 JAR包
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9bcc3abc603a4893a36e308a0be8c303.png"/>
     </li>
     <li>
      点击上传的jar的小眼睛按钮，查看加载 jar 包开关是否有打开，如果没有打开需要手动打开
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/567316c9a3d6451fb7edb97e090a6241.png"/>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e09bdfcb6f734005a45b9c54293d8c29.png"/>
     </li>
     <li>
      在接口测试中调用写好的脚本
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dd256613178d4ccd8728917d2db994d1.png"/>
      <br/>
      调试时最后两行logo.info可以加上不过在正式使用时需要注释掉，会占用服务器的内存。
     </li>
     <li>
      如果不能运行可以从下面几个维度来进行排错
      <ul>
       <li>
        查看控制台的logo，看哪里报错
       </li>
       <li>
        脚本有没有写错
       </li>
       <li>
        是不是符合beanshell语法或者其他类型的语法错误
       </li>
       <li>
        查看请求头的参数，有没有传错
       </li>
      </ul>
     </li>
    </ol>
