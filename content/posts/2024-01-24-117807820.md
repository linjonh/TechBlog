    <h3>
     <a id="_0">
     </a>
     场景
    </h3>
    <blockquote>
     <p>
      前端调后端接口，后端响应时间很久，会出现504 Gateway Time-out
     </p>
    </blockquote>
    <h3>
     <a id="_4">
     </a>
     原因
    </h3>
    <blockquote>
     <p>
      nginx中proxy默认链接时间是60s
     </p>
    </blockquote>
    <h3>
     <a id="_8">
     </a>
     解决办法
    </h3>
    <p>
     修改nginx连接超时时间
    </p>
    <pre><code class="prism language-js">proxy_connect_timeout <span class="token number">300</span>s<span class="token punctuation">;</span><span class="token comment">//建立tcp协议的连接时间</span>
proxy_send_timeout <span class="token number">300</span>s<span class="token punctuation">;</span><span class="token comment">//发送接口的时间</span>
proxy_read_timeout <span class="token number">300</span>s<span class="token punctuation">;</span><span class="token comment">//读取时间（接口响应时间）</span>
</code></pre>
    <h3>
     <a id="_16">
     </a>
     修改过程
    </h3>
    <ul>
     <li>
      因为项目已经部署到服务启上，就登录服务器在服务器上修改nginx配置
     </li>
     <li>
      因为我是因为接口响应时间太久导致的超时，所以我修改nginx中的配置proxy_read_timeout 300s;就解决了问题
     </li>
    </ul>
    <ol>
     <li>
      登录服务器，并输入服务器密码
     </li>
    </ol>
    <pre><code class="prism language-js">ssh <span class="token punctuation">[</span>服务器名称<span class="token punctuation">]</span>@<span class="token punctuation">[</span><span class="token constant">IP</span>地址<span class="token punctuation">]</span>
回车输入服务器密码
</code></pre>
    <ol start="2">
     <li>
      查看nginx目录
     </li>
    </ol>
    <pre><code class="prism language-js">nginx <span class="token operator">-</span>t
</code></pre>
    <ol start="3">
     <li>
      查看nginx配置
     </li>
    </ol>
    <pre><code class="prism language-js">vi <span class="token punctuation">[</span>目录<span class="token punctuation">]</span>
</code></pre>
    <ol start="4">
     <li>
      编辑nginx
     </li>
    </ol>
    <pre><code class="prism language-js">i
</code></pre>
    <ol start="5">
     <li>
      退出编辑状态
     </li>
    </ol>
    <pre><code class="prism language-js">按键盘键esc
</code></pre>
    <ol start="6">
     <li>
      保存并退出
     </li>
    </ol>
    <pre><code class="prism language-js"><span class="token operator">:</span>wq
</code></pre>
    <ol start="7">
     <li>
      查看是否修改成功
     </li>
    </ol>
    <pre><code class="prism language-js">nginx <span class="token operator">-</span>t
</code></pre>
    <ol start="8">
     <li>
      重新载入nginx配置文件
     </li>
    </ol>
    <pre><code class="prism language-js">nginx <span class="token operator">-</span>s reload
</code></pre>
    <p>
     修改nginx配置后要重新载入nginx配置，前端vue项目不需要重启，直接在此调用借口就行
    </p>
