     拯救无法启动的虚拟机文件.vmdk中的数据
    </h1>

   <div class="htmledit_views" id="content_views">
    <p>
     <span style="font-size:16px">
      虚拟机：
      <span style="font-family:Times New Roman">
       VMwareWorkstation 8.0.2 build-591240
      </span>
     </span>
    </p>
    <p>
     <span style="font-family:Times New Roman; font-size:16px">
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      从事
      <span style="font-family:Times New Roman">
       Linux
      </span>
      开发的软件工程师几乎都使用过虚拟机软件，如
      <span style="font-family:Times New Roman">
       VMware workstation
      </span>
      ，一般把虚拟机软件运行在微软的操作系统中，把
      <span style="font-family:Times New Roman">
       Linux
      </span>
      操作系统（如
      <span style="font-family:Times New Roman">
       Ubuntu
      </span>
      ）运行在虚拟机软件中，这样就可以在一台主机上同时使用两个不同类型的操作系统，非常方便。
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      运行在虚拟机中的操作系统以
      <span style="font-family:Times New Roman">
       .vmdk
      </span>
      文件的形式保存。
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      在实际工作中，经常会遇到运行在虚拟机中的操作系统彻底崩溃，再也无法启动，导致几天甚至十几天的劳动成果灰飞烟灭。
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      本文就介绍一个简单的方法，告诉大家如何拯救无法启动的操作系统中的重要数据。基本原理就是把无法启动的操作系统所在的虚拟机文件
      <span style="font-family:Times New Roman">
       .vmdk
      </span>
      当成另一个能够启动的操作系统的一个磁盘。
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      操作如下：
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      （
      <span style="font-family:Times New Roman">
       1
      </span>
      ）、打开能够正常运行的操作系统
     </span>
    </p>
    <p style="text-align:center">
     <span style="font-family:Times New Roman; font-size:16px">
      <img alt="" height="300" src="https://img-my.csdn.net/uploads/201206/23/1340453855_6001.jpg" width="400"/>
     </span>
    </p>
    <p style="text-align:center">
     <img alt="" height="400" src="https://img-my.csdn.net/uploads/201206/23/1340453875_3986.jpg" width="600"/>
    </p>
    <p>
     <span style="font-size:16px">
      （
      <span style="font-family:Times New Roman">
       2
      </span>
      ）、把无法启动的操作系统所在的虚拟机文件设置成能够正常运行的操作系统的一个磁盘
     </span>
    </p>
    <p style="text-align:center">
     <span style="font-family:Times New Roman; font-size:16px">
      <img alt="" src="https://img-my.csdn.net/uploads/201206/23/1340453892_8788.jpg"/>
     </span>
    </p>
    <p style="text-align:center">
     <span style="font-family:Times New Roman; font-size:16px">
      <img alt="" height="400" src="https://img-my.csdn.net/uploads/201206/23/1340453913_6311.jpg" style="text-align:center" width="600">
       <br/>
      </img>
     </span>
    </p>
    <p style="text-align:center">
     <br/>
    </p>
    <p style="text-align:center">
     <img alt="" height="400" src="https://img-my.csdn.net/uploads/201206/23/1340453934_2805.jpg" width="600"/>
    </p>
    <p style="text-align:center">
     <img alt="" src="https://img-my.csdn.net/uploads/201206/23/1340453957_4705.jpg" style="text-align:center">
      <br/>
     </img>
    </p>
    <p style="text-align:center">
     <br/>
    </p>
    <p>
     <span style="font-size:16px">
      （
      <span style="font-family:Times New Roman">
       3
      </span>
      ）、启动能够正常运行的操作系统
     </span>
    </p>
    <p style="text-align:center">
     <span style="font-family:Times New Roman; font-size:16px">
      <img alt="" src="https://img-my.csdn.net/uploads/201206/23/1340453974_6954.jpg"/>
     </span>
    </p>
    <p>
     <span style="font-size:16px">
      （
      <span style="font-family:Times New Roman">
       4
      </span>
      ）、登入操作系统，挂载新增的磁盘到系统中的某个目录下
     </span>
    </p>
    <pre><code class="language-cpp">$ ls -l /dev/sd*
brw-rw---- 1 root disk 8,  0 2012-06-23 19:16 /dev/sda
brw-rw---- 1 root disk 8,  1 2012-06-23 19:16 /dev/sda1
brw-rw---- 1 root disk 8,  2 2012-06-23 19:16 /dev/sda2
brw-rw---- 1 root disk 8,  5 2012-06-23 19:16 /dev/sda5
brw-rw---- 1 root disk 8, 16 2012-06-23 19:16 /dev/sdb   //sdb1、sdb2和sdb5就是新增磁盘的分区
brw-rw---- 1 root disk 8, 17 2012-06-23 19:16 /dev/sdb1
brw-rw---- 1 root disk 8, 18 2012-06-23 19:16 /dev/sdb2
brw-rw---- 1 root disk 8, 21 2012-06-23 19:16 /dev/sdb5

$ mkdir $HOME/mnt  //新建挂载点
$ sudo mount /dev/sdb1 $HOME/mnt  //挂载sdb1

//查看，无法启动的操作系统的所有文件都在$HOME/mnt目录下了
$ cd mnt/
$ ls
</code></pre>
