     SQL同一个字段进行数字类型字符串和文字字符串区分
    </h1>

    <blockquote>
     <p>
      业务场景：针对同一个字段，既有数字类型的字符串（数字类型的字符串需要进行排序），又有非数字类型的字符串，因此需要对两者进行合并。
     </p>
    </blockquote>
    <h4>
     MySQL
    </h4>
    <p>
     可以使用
     <code>
      REGEXP
     </code>
     操作符结合正则表达式来判断。纯数字的正则表达式可以表示为
     <code>
      '^[0-9]+$'
     </code>
     ，表示字符串只由数字组成。
    </p>
    <pre><code class="language-sql">-- 筛选出纯数字的记录
SELECT * 
FROM your_table_name
WHERE route_no REGEXP '^[0-9]+$';

-- 筛选出包含文字字符的记录
SELECT * 
FROM your_table_name
WHERE route_no NOT REGEXP '^[0-9]+$';</code></pre>
    <p>
     <strong>
      Oracle
     </strong>
     ：
     <br/>
     可以使用
     <code>
      REGEXP_LIKE
     </code>
     函数，同样使用正则表达式
     <code>
      '^[0-9]+$'
     </code>
     来判断
    </p>
    <pre><code class="language-sql">-- 筛选出纯数字的记录
SELECT * 
FROM your_table_name
WHERE REGEXP_LIKE(route_no, '^[0-9]+$');

-- 筛选出包含文字字符的记录
SELECT * 
FROM your_table_name
WHERE NOT REGEXP_LIKE(route_no, '^[0-9]+$');</code></pre>
    <p>
     <strong>
      SQL Server
     </strong>
     ：
     <br/>
     可以使用
     <code>
      LIKE
     </code>
     结合通配符来进行大致判断，但不如正则表达式精确。要实现类似正则的功能，可以使用
     <code>
      PATINDEX
     </code>
     函数结合模式匹配
    </p>
    <pre><code class="language-sql">-- 筛选出纯数字的记录
SELECT * 
FROM your_table_name
WHERE PATINDEX('%[^0-9]%', route_no) = 0;

-- 筛选出包含文字字符的记录
SELECT * 
FROM your_table_name
WHERE PATINDEX('%[^0-9]%', route_no) &gt; 0;</code></pre>
