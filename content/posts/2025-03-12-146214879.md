    <p>
     在 Linux 系统中，区分**磁盘（物理/虚拟存储设备）
     <strong>
      和
     </strong>
     分区（磁盘的逻辑划分）**是管理存储的基础。以下是详细的区分方法和操作示例：
    </p>
    <hr/>
    <h4>
     <a id="_5">
     </a>
     <strong>
      一、通过设备命名规则区分
     </strong>
    </h4>
    <p>
     Linux 中磁盘和分区的命名遵循特定规则：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        类型
       </th>
       <th>
        命名格式
       </th>
       <th>
        示例
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         磁盘
        </strong>
       </td>
       <td>
        <code>
         /dev/sdX
        </code>
       </td>
       <td>
        <code>
         /dev/sda
        </code>
       </td>
       <td>
        <code>
         sd
        </code>
        表示 SCSI/SATA 磁盘，
        <code>
         X
        </code>
        为字母（a, b, c…）。
       </td>
      </tr>
      <tr>
       <td>
       </td>
       <td>
        <code>
         /dev/nvmeXnY
        </code>
       </td>
       <td>
        <code>
         /dev/nvme0n1
        </code>
       </td>
       <td>
        <code>
         nvme
        </code>
        表示 NVMe 磁盘，
        <code>
         X
        </code>
        为控制器编号，
        <code>
         Y
        </code>
        为命名空间编号。
       </td>
      </tr>
      <tr>
       <td>
       </td>
       <td>
        <code>
         /dev/vdX
        </code>
       </td>
       <td>
        <code>
         /dev/vda
        </code>
       </td>
       <td>
        <code>
         vd
        </code>
        表示虚拟化环境中的磁盘（如 KVM 虚拟机）。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         分区
        </strong>
       </td>
       <td>
        <code>
         /dev/sdXN
        </code>
       </td>
       <td>
        <code>
         /dev/sda1
        </code>
       </td>
       <td>
        <code>
         N
        </code>
        为分区编号（1, 2, 3…）。
       </td>
      </tr>
      <tr>
       <td>
       </td>
       <td>
        <code>
         /dev/nvmeXnYpZ
        </code>
       </td>
       <td>
        <code>
         /dev/nvme0n1p1
        </code>
       </td>
       <td>
        <code>
         pZ
        </code>
        表示 NVMe 磁盘的分区编号（Z=1,2,3…）。
       </td>
      </tr>
      <tr>
       <td>
       </td>
       <td>
        <code>
         /dev/vdXN
        </code>
       </td>
       <td>
        <code>
         /dev/vda1
        </code>
       </td>
       <td>
        虚拟磁盘的分区命名规则与物理磁盘一致。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <a id="_19">
     </a>
     <strong>
      二、通过命令工具区分
     </strong>
    </h4>
    <h5>
     <a id="1__lsblk__20">
     </a>
     <strong>
      1. 使用
      <code>
       lsblk
      </code>
      查看树状结构
     </strong>
    </h5>
    <pre><code class="prism language-bash">lsblk
</code></pre>
    <p>
     <strong>
      输出示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash">NAME    MAJ:MIN RM  SIZE RO TYPE MOUNTPOINTS
sda       <span class="token number">8</span>:0    <span class="token number">0</span>  100G  <span class="token number">0</span> disk          <span class="token comment"># 这是一个磁盘（TYPE=disk）</span>
├─sda1    <span class="token number">8</span>:1    <span class="token number">0</span>   1G  <span class="token number">0</span> part /boot     <span class="token comment"># 这是一个分区（TYPE=part）</span>
└─sda2    <span class="token number">8</span>:2    <span class="token number">0</span>   99G  <span class="token number">0</span> part 
  └─vg-root <span class="token number">254</span>:0    <span class="token number">0</span>   99G  <span class="token number">0</span> lvm  /    <span class="token comment"># 这是 LVM 逻辑卷（TYPE=lvm）</span>
nvme0n1 <span class="token number">259</span>:0    <span class="token number">0</span>  500G  <span class="token number">0</span> disk          <span class="token comment"># 这是一个 NVMe 磁盘</span>
└─nvme0n1p1 <span class="token number">259</span>:1  <span class="token number">0</span>  500G  <span class="token number">0</span> part /data  <span class="token comment"># 这是一个 NVMe 分区</span>
</code></pre>
    <ul>
     <li>
      <strong>
       关键字段
      </strong>
      ：
      <ul>
       <li>
        <code>
         TYPE=disk
        </code>
        ：表示磁盘。
       </li>
       <li>
        <code>
         TYPE=part
        </code>
        ：表示分区。
       </li>
       <li>
        子设备缩进显示在父设备下方。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="2__fdisk_l__41">
     </a>
     <strong>
      2. 使用
      <code>
       fdisk -l
      </code>
      查看详细信息
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">fdisk</span> <span class="token parameter variable">-l</span> /dev/sda
</code></pre>
    <p>
     <strong>
      输出示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash">Disk /dev/sda: <span class="token number">100</span> GiB, <span class="token number">107374182400</span> bytes, <span class="token number">209715200</span> sectors  <span class="token comment"># 磁盘信息</span>
Disk model: VIRTUAL-DISK    
Units: sectors of <span class="token number">1</span> * <span class="token number">512</span> <span class="token operator">=</span> <span class="token number">512</span> bytes
Sector size <span class="token punctuation">(</span>logical/physical<span class="token punctuation">)</span>: <span class="token number">512</span> bytes / <span class="token number">512</span> bytes
I/O size <span class="token punctuation">(</span>minimum/optimal<span class="token punctuation">)</span>: <span class="token number">512</span> bytes / <span class="token number">512</span> bytes
Disklabel type: gpt

Device     Start       End   Sectors  Size Type           <span class="token comment"># 分区信息</span>
/dev/sda1   <span class="token number">2048</span>   <span class="token number">2099199</span>   <span class="token number">2097152</span>    1G EFI System
/dev/sda2 <span class="token number">2099200</span> <span class="token number">209715199</span> <span class="token number">207616000</span>   99G Linux LVM
</code></pre>
    <ul>
     <li>
      <strong>
       区分点
      </strong>
      ：
      <ul>
       <li>
        第一段描述磁盘的整体信息（如容量、型号）。
       </li>
       <li>
        <code>
         Device
        </code>
        列列出所有分区（如
        <code>
         /dev/sda1
        </code>
        ）。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="_92">
     </a>
     <strong>
      四、常见误区与注意事项
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        误操作磁盘而非分区
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         危险操作
        </strong>
        ：直接对磁盘（如
        <code>
         /dev/sda
        </code>
        ）执行格式化（
        <code>
         mkfs
        </code>
        ）会破坏所有分区数据！
       </li>
       <li>
        <strong>
         正确操作
        </strong>
        ：始终针对分区（如
        <code>
         /dev/sda1
        </code>
        ）进行文件系统操作。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        LVM 和 RAID 的干扰
       </strong>
      </p>
      <ul>
       <li>
        LVM 逻辑卷或 RAID 虚拟磁盘可能隐藏物理分区，需通过
        <code>
         pvdisplay
        </code>
        、
        <code>
         lvdisplay
        </code>
        或
        <code>
         mdadm
        </code>
        进一步分析。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        临时设备（如 USB 盘）
       </strong>
      </p>
      <ul>
       <li>
        USB 设备可能动态分配名称（如
        <code>
         /dev/sdb
        </code>
        下次插入变为
        <code>
         /dev/sdc
        </code>
        ），建议通过 UUID 或标签挂载。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_105">
     </a>
     <strong>
      五、操作验证示例
     </strong>
    </h4>
    <h5>
     <a id="_devnvme0n1p1__106">
     </a>
     <strong>
      场景：确认
      <code>
       /dev/nvme0n1p1
      </code>
      是分区
     </strong>
    </h5>
    <pre><code class="prism language-bash">lsblk <span class="token parameter variable">-o</span> NAME,TYPE /dev/nvme0n1p1
</code></pre>
    <p>
     <strong>
      输出
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash">NAME        TYPE
nvme0n1p1  part   <span class="token comment"># TYPE=part 表示这是一个分区</span>
</code></pre>
    <h5>
     <a id="_devsdb__116">
     </a>
     <strong>
      场景：确认
      <code>
       /dev/sdb
      </code>
      是磁盘
     </strong>
    </h5>
    <pre><code class="prism language-bash">lsblk <span class="token parameter variable">-o</span> NAME,TYPE /dev/sdb
</code></pre>
    <p>
     <strong>
      输出
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash">NAME   TYPE
sdb    disk   <span class="token comment"># TYPE=disk 表示这是一个磁盘</span>
</code></pre>
