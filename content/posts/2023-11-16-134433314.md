    <h6>
     问题描述：
    </h6>
    <p>
     思科模拟器模拟DNS服务时可以ping通，同时也可以解析IP，但使用网页浏览时无法域名解析
    </p>
    <p>
     <img alt="" height="307" src="https://i-blog.csdnimg.cn/blog_migrate/316336a821378b88316860824d22e436.png" width="693"/>
    </p>
    <h6 id="1.1%E5%8F%AF%E8%83%BD%E5%87%BA%E7%8E%B0%E7%9A%84%E5%8E%9F%E5%9B%A0%EF%BC%9A">
     1.1可能出现的原因：
    </h6>
    <p>
     <img alt="" height="470" src="https://i-blog.csdnimg.cn/blog_migrate/b3d92d03657521f6ba23d5fe72ce4e2f.png" width="677">
      在分网段中有DHCP服务器且DHCP划分VLAN时未给其配置服务器的IP地址
     </img>
    </p>
    <h6 id="1.1%E8%A7%A3%E5%86%B3%E6%96%B9%E6%B3%95%EF%BC%9A">
     1.1解决方法：
    </h6>
    <p>
     将原有DHCP地址池移除，再添加有DNS服务器的地址池
    </p>
    <p>
     或直接点击原有地址池在其上修改，操作如下图
    </p>
    <p>
     修改后再重新获取PC端的IP地址
    </p>
    <p>
     <img alt="" height="432" src="https://i-blog.csdnimg.cn/blog_migrate/d37d532705971b0600d95604ec94b567.png" width="520">
      <img alt="" height="487" src="https://i-blog.csdnimg.cn/blog_migrate/530a4223186c2244b6d7dce1a77b05d9.png" width="522"/>
     </img>
    </p>
    <p>
     刷新后重新尝试即可：
    </p>
    <p>
     <img alt="" height="366" src="https://i-blog.csdnimg.cn/blog_migrate/524359ed6bcdf0aebf571f7e03377917.png" width="588"/>
    </p>
    <h6 id="1.2%E5%8F%AF%E8%83%BD%E5%87%BA%E7%8E%B0%E7%9A%84%E5%8E%9F%E5%9B%A0%EF%BC%9A">
     1.2可能出现的原因：
    </h6>
    <p>
     使用命令配置在交换机中配置DHCP时，未配置DNS服务器的ip
    </p>
    <h6 id="1.2%E8%A7%A3%E5%86%B3%E5%8A%9E%E6%B3%95%EF%BC%9A">
     1.2解决办法：
    </h6>
    <p>
     使用命令重新配置DHCP服务
    </p>
    <p>
     或进入地址池中增加DNS服务，相关命令参考如下
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       DS-1(config)#ip dhcp pool VLAN 10           //dhcp地址池
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       DS-1(dhcp-config)#network 10.1.10.0 255.255.255.0/        /宣告网段地址
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       DS-1(dhcp-config)#default-router 10.1.10.254
      </span>
      <span style="background-color:#ffffff;">
       <span style="color:#121212;">
        //
       </span>
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffffff;">
       <span style="color:#121212;">
        设置网关地址
       </span>
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       DS-1(dhcp-config)#dns-server 10.5.100.252          //设置dns服务器
      </span>
     </strong>
    </p>
    <h6 id="DHCP%E4%B8%AD%E7%BB%A7%E9%85%8D%E7%BD%AE%EF%BC%9A" style="margin-left:0px;text-align:left;">
     <strong>
      <span style="color:#000000;">
       DHCP中继配置：
      </span>
     </strong>
    </h6>
    <p>
     （以此图连接交换机为例）
    </p>
    <p>
     <strong>
      <span style="background-color:#ffffff;">
       <span style="color:#333333;">
        //
       </span>
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffffff;">
       <span style="color:#333333;">
        指定dhcp服务器的地址，表示通过该端口向该服务器发送DHCP
       </span>
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffffff;">
       <span style="color:#333333;">
        请求包
       </span>
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       WJ-R(co nfig)#interface  GigabitEthernet 0/0.5
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#111111;">
       //Cisco ip helper-address
      </span>
     </strong>
     <strong>
      <span style="color:#111111;">
       详解，当终端设备和DHCP服务器不在同一广播域内的时候，中间设备就需要指定一个地址来将广播包转发出去，即需要设定helper_address来实现中继
      </span>
     </strong>
    </p>
    <p style="margin-left:0;text-align:left;">
     <strong>
      <span style="color:#000000;">
       WJ-R(con fig -subif)#ip helper-address 10.8.7.253
      </span>
     </strong>
    </p>
    <h6 id="%E4%B8%80%E8%88%AC%E9%85%8D%E7%BD%AE%E4%B8%AD%E7%BB%A7%E7%9A%84%E6%96%B9%E6%B3%95%EF%BC%9A" style="margin-left:0px;text-align:left;">
     一般配置中继的方法：
    </h6>
    <p>
     配置中继
     <br/>
     HX(config-if)#int vlan10
    </p>
    <p>
     HX(config-if)#no  shutdown
    </p>
    <p>
     HX(config-if)#ip add 192.168.10.254 255.255.255.0
    </p>
    <p>
     HX(config-if)#ip helper-address 10.10.10.100
     <br/>
     HX(config-if)#int vlan20
    </p>
    <p>
     HX(config-if)#no  shutdown
    </p>
    <p>
     HX(config-if)#ip add 192.168.20.254 255.255.255.0
    </p>
    <p>
     HX(config-if)#ip helper-address 10.10.10.100
     <br/>
     中继路由器：
     <br/>
     Router&gt;enable
     <br/>
     Router#configure terminal
     <br/>
     Router(config)#hostname Relay
     <br/>
     Relay(config)#int g0/1
     <br/>
     Relay(config-if)#ip add 192.168.100.2 255.255.255.0
     <br/>
     Relay(config-if)#no shutdown
     <br/>
     Relay(config-if)#int g0/0
     <br/>
     Relay(config-if)#ip add 10.10.10.2 255.255.255.0
     <br/>
     Relay(config-if)#no shutdown
     <br/>
     原文链接：https://blog.csdn.net/mogexiuluo/article/details/118566555
    </p>
    <h6>
     1.3可能出现原因：
    </h6>
    <p>
     如果检查配置与DNS的IP地址配置无误后，依然无法访问web服务，则可能是DNS服务默认开启了防火墙。
    </p>
    <h6>
     1.3解决办法：
    </h6>
    <p>
     将DNS防火墙关闭或更改防火墙配置
    </p>
