    <h3>
     <a id="related_name__0">
     </a>
     <code>
      related_name
     </code>
     的作用
    </h3>
    <p>
     <code>
      related_name
     </code>
     用于 Django 的
     <strong>
      ForeignKey
     </strong>
     和
     <strong>
      OneToOneField
     </strong>
     ，用于
     <strong>
      反向访问
     </strong>
     关联模型的数据。
    </p>
    <p>
     默认情况下，Django 会自动生成一个反向关系的名字，但如果多个外键指向同一个模型，就可能发生命名冲突。
     <code>
      related_name
     </code>
     允许我们自定义反向查询的名称，提升代码的可读性。
    </p>
    <hr/>
    <h3>
     <a id="_7">
     </a>
     举例说明
    </h3>
    <h4>
     <a id="__related_name_9">
     </a>
     ①
     <strong>
      默认情况（不使用
      <code>
       related_name
      </code>
      ）
     </strong>
    </h4>
    <pre><code class="prism language-python"><span class="token keyword">class</span> <span class="token class-name">Author</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    name <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">100</span><span class="token punctuation">)</span>

<span class="token keyword">class</span> <span class="token class-name">Book</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    title <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">200</span><span class="token punctuation">)</span>
    author <span class="token operator">=</span> models<span class="token punctuation">.</span>ForeignKey<span class="token punctuation">(</span>Author<span class="token punctuation">,</span> on_delete<span class="token operator">=</span>models<span class="token punctuation">.</span>CASCADE<span class="token punctuation">)</span>
</code></pre>
    <h5>
     <a id="_18">
     </a>
     反向查询
    </h5>
    <pre><code class="prism language-python">author <span class="token operator">=</span> Author<span class="token punctuation">.</span>objects<span class="token punctuation">.</span>get<span class="token punctuation">(</span><span class="token builtin">id</span><span class="token operator">=</span><span class="token number">1</span><span class="token punctuation">)</span>
author<span class="token punctuation">.</span>book_set<span class="token punctuation">.</span><span class="token builtin">all</span><span class="token punctuation">(</span><span class="token punctuation">)</span>  <span class="token comment"># 通过默认的 `book_set` 访问书籍列表</span>
</code></pre>
    <p>
     Django 默认会用
     <strong>
      模型名小写 +
      <code>
       _set
      </code>
     </strong>
     作为反向查询名称（
     <code>
      book_set
     </code>
     ）。
    </p>
    <hr/>
    <h4>
     <a id="__related_name_27">
     </a>
     ②
     <strong>
      使用
      <code>
       related_name
      </code>
     </strong>
    </h4>
    <pre><code class="prism language-python"><span class="token keyword">class</span> <span class="token class-name">Author</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    name <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">100</span><span class="token punctuation">)</span>

<span class="token keyword">class</span> <span class="token class-name">Book</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    title <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">200</span><span class="token punctuation">)</span>
    author <span class="token operator">=</span> models<span class="token punctuation">.</span>ForeignKey<span class="token punctuation">(</span>Author<span class="token punctuation">,</span> on_delete<span class="token operator">=</span>models<span class="token punctuation">.</span>CASCADE<span class="token punctuation">,</span> related_name<span class="token operator">=</span><span class="token string">"books"</span><span class="token punctuation">)</span>
</code></pre>
    <h5>
     <a id="_36">
     </a>
     反向查询
    </h5>
    <pre><code class="prism language-python">author <span class="token operator">=</span> Author<span class="token punctuation">.</span>objects<span class="token punctuation">.</span>get<span class="token punctuation">(</span><span class="token builtin">id</span><span class="token operator">=</span><span class="token number">1</span><span class="token punctuation">)</span>
author<span class="token punctuation">.</span>books<span class="token punctuation">.</span><span class="token builtin">all</span><span class="token punctuation">(</span><span class="token punctuation">)</span>  <span class="token comment"># 现在可以用 `books` 访问，而不是 `book_set`</span>
</code></pre>
    <p>
     这里，
     <code>
      related_name="books"
     </code>
     让
     <code>
      Author
     </code>
     可以直接通过
     <code>
      .books
     </code>
     访问其所有的
     <code>
      Book
     </code>
     记录。
    </p>
    <hr/>
    <h4>
     <a id="__45">
     </a>
     ③
     <strong>
      避免多个外键冲突
     </strong>
    </h4>
    <p>
     当一个模型有多个外键指向同一个模型时，必须使用
     <code>
      related_name
     </code>
     来区分：
    </p>
    <pre><code class="prism language-python"><span class="token keyword">class</span> <span class="token class-name">Employee</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    name <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">100</span><span class="token punctuation">)</span>

<span class="token keyword">class</span> <span class="token class-name">Task</span><span class="token punctuation">(</span>models<span class="token punctuation">.</span>Model<span class="token punctuation">)</span><span class="token punctuation">:</span>
    title <span class="token operator">=</span> models<span class="token punctuation">.</span>CharField<span class="token punctuation">(</span>max_length<span class="token operator">=</span><span class="token number">200</span><span class="token punctuation">)</span>
    assigned_to <span class="token operator">=</span> models<span class="token punctuation">.</span>ForeignKey<span class="token punctuation">(</span>Employee<span class="token punctuation">,</span> on_delete<span class="token operator">=</span>models<span class="token punctuation">.</span>CASCADE<span class="token punctuation">,</span> related_name<span class="token operator">=</span><span class="token string">"assigned_tasks"</span><span class="token punctuation">)</span>
    created_by <span class="token operator">=</span> models<span class="token punctuation">.</span>ForeignKey<span class="token punctuation">(</span>Employee<span class="token punctuation">,</span> on_delete<span class="token operator">=</span>models<span class="token punctuation">.</span>CASCADE<span class="token punctuation">,</span> related_name<span class="token operator">=</span><span class="token string">"created_tasks"</span><span class="token punctuation">)</span>
</code></pre>
    <h5>
     <a id="_56">
     </a>
     反向查询
    </h5>
    <pre><code class="prism language-python">employee <span class="token operator">=</span> Employee<span class="token punctuation">.</span>objects<span class="token punctuation">.</span>get<span class="token punctuation">(</span><span class="token builtin">id</span><span class="token operator">=</span><span class="token number">1</span><span class="token punctuation">)</span>

<span class="token comment"># 该员工创建的任务</span>
employee<span class="token punctuation">.</span>created_tasks<span class="token punctuation">.</span><span class="token builtin">all</span><span class="token punctuation">(</span><span class="token punctuation">)</span>

<span class="token comment"># 该员工被分配的任务</span>
employee<span class="token punctuation">.</span>assigned_tasks<span class="token punctuation">.</span><span class="token builtin">all</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     如果不使用
     <code>
      related_name
     </code>
     ，Django 会报错，因为
     <code>
      task_set
     </code>
     这个默认的反向名称会有两个。
    </p>
    <hr/>
    <h3>
     <a id="_70">
     </a>
     总结
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         情况
        </strong>
       </th>
       <th>
        <strong>
         默认反向查询
        </strong>
       </th>
       <th>
        <strong>
         自定义
         <code>
          related_name
         </code>
         反向查询
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        单个外键
       </td>
       <td>
        <code>
         book_set.all()
        </code>
       </td>
       <td>
        <code>
         books.all()
        </code>
       </td>
      </tr>
      <tr>
       <td>
        多个外键（指向同一模型）
       </td>
       <td>
        <strong>
         冲突，必须指定
        </strong>
       </td>
       <td>
        <code>
         assigned_tasks.all()
        </code>
        /
        <code>
         created_tasks.all()
        </code>
       </td>
      </tr>
      <tr>
       <td>
        提高可读性
       </td>
       <td>
        <code>
         author.book_set.all()
        </code>
       </td>
       <td>
        <code>
         author.books.all()
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     <a id="_related_name_77">
     </a>
     何时使用
     <code>
      related_name
     </code>
    </h4>
    <ol>
     <li>
      <strong>
       提高代码可读性
      </strong>
      ，避免
      <code>
       _set
      </code>
     </li>
     <li>
      <strong>
       多个外键指向同一模型
      </strong>
      ，防止冲突
     </li>
     <li>
      <strong>
       增强 API 设计
      </strong>
      ，让反向查询更直观
     </li>
    </ol>
    <p>
     这样，我们就可以更清晰、更直观地操作 Django 的外键关联数据！ 🚀
    </p>
