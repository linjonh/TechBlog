    <h2>
     <a id="1_gmake_1">
     </a>
     1 gmake介绍
    </h2>
    <p>
     make程序在c语言开发过程中很常见,主要用来控制编译过程,gmake为gnu的make版本,官网下载的tcc编译程序中没有make文件,要想再进一步学习c语言过程还是要使用到make程序,网上好像找不到单独可用make文件,好在tcc官网有一个可用的gmake源码,可以自己编译一个.
    </p>
    <h2>
     <a id="2__3">
     </a>
     2 源码下载
    </h2>
    <p>
     官网下载页面
     <br/>
     http://download.savannah.gnu.org/releases/tinycc/
     <br/>
     其中有一个tcc-busybox-for-win32.zip文件,为一个busybox源码,其中刚好有一个gamke源码可以用tcc进行编译.
     <br/>
     http://download.savannah.gnu.org/releases/tinycc/tcc-busybox-for-win32.zip
     <br/>
     我习惯把它解压到C:\run\test\tcc-busybox-for-win32文件下
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e5348d098ee644479e5d38dadbe7bff7.png"/>
    </p>
    <h2>
     <a id="3__10">
     </a>
     3 编译源码
    </h2>
    <p>
     其中gmake文件夹为gmake源代码,makebox.bat为初始化编译指令,编译前要初始化tcc编译环境,一种方法是将tcc编译器源码复制到tcc文件下面,makebox.bat可以正常编译。我一般使用前文配置好的tcc编程环境.bat进行编译。具体步骤如下
    </p>
    <pre><code class="prism language-c"><span class="token operator">::</span>直接输入可以进行编译
makebox<span class="token punctuation">.</span>bat
</code></pre>
    <p>
     出现下面界面即说明编译成功
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/69bef5d31a0949a796bc177ac661ddce.png">
      <br/>
      主要生成下面三个文件,其中make.exe为编译的gmake程序,sh.exe为busy shell文件类似一个模拟linux的环境。time.exe是一个时间相关的程序,目前主要用到make.exe.
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/35b802c6524e4892bf590989444fad96.png"/>
     </img>
    </p>
    <h2>
     <a id="4_make_21">
     </a>
     4 配置make环境并测试
    </h2>
    <p>
     将make.exe复制到你的tcc编译文件夹下面即可正常使用,打开我的tcc编程环境.bat,输入
    </p>
    <pre><code class="prism language-c"><span class="token operator">::</span>若输出make的版本信息<span class="token punctuation">,</span>即说明make可正常使用
make <span class="token operator">-</span>v
</code></pre>
    <p>
     结果如下
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8c0d9f1838144ecf834425dee8c5dc22.png"/>
    </p>
    <h2>
     <a id="5_makelua_30">
     </a>
     5 简单使用make编译lua源代码
    </h2>
    <p>
     要想使用make编译c代码文件,需要使用到Makefile文件,下面介绍使用make编译lua源代码。打开上一节lua源码文件夹,其中包含一个Makefile文件,但是很遗憾这个makefile文件无法使用tcc进行编译,先把它剪切到其他文件夹备用,先自己手写一个简单makefile,注意Makefile的格式要求比较严格
    </p>
    <pre><code class="prism language-c">#第一个Makefile文件<span class="token punctuation">,</span>#代表单行注释
#标签<span class="token punctuation">,</span>下面接编译指令<span class="token punctuation">,</span>注意编译指令前面又加一个tab键<span class="token punctuation">,</span>不能加空格<span class="token punctuation">.</span>
build<span class="token operator">:</span>
	rem 注释<span class="token punctuation">,</span>编译指令
	tcc <span class="token operator">*</span><span class="token punctuation">.</span>c <span class="token operator">-</span>o lua<span class="token punctuation">.</span>exe
	echo 编译成功
	

</code></pre>
    <p>
     打开tcc编程环境.bat,输入make即可编译程序
    </p>
    <pre><code class="prism language-cpp">make
</code></pre>

      <br/>
      显示上面信息即说明make编译成功。可以看到文件夹下面生成了一个lua.exe
     </img>
    </p>
