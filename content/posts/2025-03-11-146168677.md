    <h2 id="networking" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        联网
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       网络是 Linux 环境中的一个重要方面。它使 Linux 系统能够与其他系统连接、交互和共享资源，无论是 Linux、Windows、macOS 还是任何其他作系统。Linux 提供了丰富的工具和命令来管理网络接口、查看其配置详细信息、排查问题和自动执行任务，展示了其稳健性和多功能性。Linux 网络堆栈因其性能、运行大规模和详尽配置的能力以及对各种网络协议的支持而备受推崇。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       Linux 采用基于文件的方法进行网络配置，将与网络相关的设置和配置存储在标准文件中，例如 /etc/network/interfaces 或 /etc/sysconfig/network-scripts/，具体取决于 Linux 发行版。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       也许与 Linux 系统上的网络相关的最流行的命令之一是以下命令：
       <code>
        ifconfig
       </code>
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">ifconfig</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       这将输出有关系统上当前活动的所有网络接口的信息。但是，请注意，它已过时，并被 取代，后者提供了更多特性和功能。
       <code>
        ifconfig
       </code>
       <code>
        ip
       </code>
      </span>
     </span>
    </p>
    <p>
     Linux 的网络管理就像给房子装电话线和网线，只不过这个"房子"是电脑系统。在 Linux 里，每个网络接口就像墙上不同的网线插口，有的插着网线（有线网卡），有的连着 Wi-Fi（无线网卡）。
    </p>
    <p>
     <strong>
      管理网络的两种方式：
     </strong>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        传统方法（老式座机）
       </strong>
       <br/>
       用
       <code>
        ifconfig
       </code>
       命令就像用老式电话机，简单但功能有限。比如：
      </p>
      <pre><code class="hljs">ifconfig eth0 192.168.1.100  # 给第一个网口设置IP
ifconfig eth0 down           # 拔掉网线</code></pre>
      <pre></pre>
      <p>
       但就像现在很少人用座机，这个命令逐渐被淘汰了。
      </p>
     </li>
     <li>
      <p>
       <strong>
        现代方法（智能手机）
       </strong>
       <br/>
       用
       <code>
        ip
       </code>
       命令就像用智能手机管理网络，功能强大又方便：
      </p>
      <pre><code class="hljs">ip addr show                 # 查看所有网络接口
ip route add default via 192.168.1.1  # 设置默认网关
ip link set wlan0 up         # 开启无线网卡</code></pre>
      <pre></pre>
     </li>
    </ol>
    <p>
     <strong>
      为什么推荐用新方法？
     </strong>
     <br/>
     好比智能手机能同时处理通话、上网、装APP，而老座机只能打电话。
     <code>
      ip
     </code>
     命令可以：
    </p>
    <ul>
     <li>
      <p>
       同时管理有线/无线/VPN连接
      </p>
     </li>
     <li>
      <p>
       查看更详细的网络状态
      </p>
     </li>
     <li>
      <p>
       配置复杂的网络规则
      </p>
     </li>
    </ul>
    <p>
     <strong>
      网络设置存档处（配置文件）
     </strong>
     <br/>
     Linux 把网络配置写在文件里，就像把家里所有电器的使用说明整理成手册：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        Ubuntu
       </strong>
       家的手册放在
       <code>
        /etc/network/interfaces
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        CentOS
       </strong>
       家的手册放在
       <code>
        /etc/sysconfig/network-scripts/
       </code>
      </p>
     </li>
    </ul>
    <p>
     修改这些文件就像调整说明书，改完后需要"重启服务"让系统重新读一遍手册：
    </p>
    <pre><code class="hljs">sudo systemctl restart networking  # 大多数系统适用</code></pre>
    <pre></pre>
    <p>
     <strong>
      日常网络小技巧：
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        查网速
       </strong>
       ：
       <code>
        speedtest-cli
       </code>
       （像用测速APP）
      </p>
     </li>
     <li>
      <p>
       <strong>
        找信号
       </strong>
       ：
       <code>
        iwconfig
       </code>
       （查看Wi-Fi强度，像手机显示信号格）
      </p>
     </li>
     <li>
      <p>
       <strong>
        修网络
       </strong>
       ：
       <code>
        ping 8.8.8.8
       </code>
       （像打电话测试线路是否通畅）
      </p>
     </li>
    </ul>
    <p>
     <strong>
      总结：
     </strong>
     <br/>
     Linux 的网络系统就像个智能家居控制中心，既能用简单按钮（基础命令）操作，也能编程设置复杂场景（高级配置）。虽然新手用
     <code>
      ifconfig
     </code>
     够简单，但学会
     <code>
      ip
     </code>
     命令就像从功能机升级到智能机，能玩转更多高级功能。
    </p>
    <h2 id="tcpip">
     TCP/IP 协议
    </h2>
    <p>
     TCP/IP（传输控制协议/Internet 协议）构成了 Internet 协议的主干。从本质上讲，它是一组允许两台或多台计算机进行通信的网络协议。在 Linux 环境中，TCP/IP 网络是作系统功能的基本组成部分。它提供了一个平台，用于建立连接并促进两个终端节点之间的数据传输。
    </p>
    <p>
     TCP/IP 在使主机能够连接相同或不同网络上的其他主机并与之交互方面发挥着至关重要的作用。它由四个层模型组成，包括 Network Interface、Internet、Transport 和 Application 层。了解 TCP/IP、其结构和工作原理对于有效管理 Linux 网络和排除故障至关重要。
    </p>
    <p>
     以下是在 Linux 中使用 TCP/IP 协议的基本命令：
    </p>
    <pre><code># To view all active TCP/IP network connections
netstat -at</code></pre>

    <h2 id="subnetting" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        子网划分
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       子网划分是 Linux 网络中的关键过程。这种做法涉及将一个网络划分为两个或多个网络，称为子网。子网划分有助于提高网络性能和安全性。在 Linux 中，可以在 Internet 协议 （IP） 寻址方案的上下文中管理子网划分，这对于组织和管理网络内的 IP 地址、防止 IP 冲突和有效利用 IP 地址范围至关重要。在大型复杂的 Linux 网络环境中，这种技术非常有价值，因为 IP 地址管理可能会变得非常复杂。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       通常，在 Linux 中使用以下命令进行子网划分：
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code"># Display current routing table
$ route -n 

# Add a new subnet
$ route add -net xxx.xxx.xxx.x/xx gw yyy.yyy.yyy.y</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       请将 替换为所需的子网地址和网络掩码，并替换为子网的预期默认网关。
       <code>
        xxx.xxx.xxx.x/xx
       </code>
       <code>
        yyy.yyy.yyy.y
       </code>
      </span>
     </span>
    </p>
    <h4>
     🏘️
     <strong>
      子网划分就像小区规划
     </strong>
    </h4>
    <p>
     <strong>
      核心概念：
     </strong>
     把一个大社区（网络）划分成多个小区（子网），每个小区有自己的物业管理和出入口
    </p>
    <hr/>
    <h4>
     🌍
     <strong>
      为什么需要划分小区？
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        避免拥堵

       <strong>

     </li>

    <h2 id="ethernet-arp-and-rarp">
     以太网、ARP 和 RARP
    </h2>
    <p>
     Linux 因其强大、可定制和开源的性质而成为流行的网络作系统选择。了解 Linux 中的网络需要理解各种协议和工具。这一领域的三个关键组成部分包括以太网、ARP（地址解析协议）和 RARP（反向地址解析协议）。
    </p>
    <ul>
     <li>
      以太网：这是安装最广泛的 LAN（局域网）技术，允许设备在局域网内进行通信。
     </li>
     <li>
      ARP：顾名思义，它提供地址解析，将 IP 地址转换为 MAC（媒体访问控制）地址，促进更直接的网络通信。
     </li>
     <li>
      RARP：它是反向地址解析协议，其工作方式与 ARP 相反。它将 MAC 地址转换为 IP 地址，这在计算机知道其 MAC 地址但需要找出其 IP 地址的情况下非常有用。
     </li>
    </ul>
    <p>
     了解这些组件对于诊断和管理 Linux 中的网络问题是必不可少的。
    </p>
    <h4>
     📡
     <strong>

     </strong>
    </h4>
    <p>
     <strong>
      场景
     </strong>

    <ul>
     <li>
      <p>
       <strong>
        工作流程
       </strong>
       ：
      </p>
      <ol>
       <li>
        <p>
         快递员在小区广播站喊："3栋2单元101的住户，请把你的门牌亮出来！"
        </p>
       </li>
       <li>
        <p>
         正确住户回应："我是MAC地址AA:BB:CC:DD:EE:FF"
        </p>
       </li>
       <li>
        <p>
         快递员记下对应关系，下次直接送货
        </p>
       </li>
      </ol>
     </li>
     <li>
      <p>
       <strong>
        Linux操作
       </strong>
       ：
      </p>
      <pre><code class="hljs">arp -a          # 查看当前记录的地址对应表（ARP缓存）

      <pre></pre>
     </li>
    </ul>

    <h2 id="dhcp" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        DHCP
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       动态主机配置协议 （DHCP） 是任何网络的关键组件。在 Linux 网络中，它用于在网络内动态分配 IP 地址。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       DHCP 服务器有效地管理 IP 地址和与之相关的信息，确保每台客户端计算机都获得唯一的 IP 和所有正确的网络信息。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在 Linux 中，可以使用终端命令配置和管理 DHCP。这涉及安装 DHCP 服务器软件、编辑配置文件和管理服务器的服务。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       传统的 DHCP 服务器应该有一个静态 IP 地址，以有效地管理 IP 分配。Linux 中的 DHCP 还处理您的网络可能需要的 DNS 和其他相关数据。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       以下是在基于 Debian 的 Linux 中安装 DHCP 服务器的基本命令示例：
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">sudo apt-get install isc-dhcp-server</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       安装过程完成后，DHCP 服务器的所有配置都在位于配置文件中完成，可以使用任何文本编辑器对其进行编辑。
       <code>
        /etc/dhcp/dhcpd.conf
       </code>
      </span>
     </span>
    </p>
    <h4>
     🏠
     <strong>
      DHCP 就像智能租房中介
     </strong>
    </h4>
    <p>
     <strong>
      核心概念：
     </strong>
     自动为新入住的租客（设备）分配房间（IP地址），并告知小区规则（网络配置）
    </p>
    <hr/>
    <h4>
     📜
     <strong>
      DHCP 服务流程（四步签约）
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        租客找房（Discover）
       </strong>
       <br/>
       新设备大喊："我是MAC:AA:BB:CC:DD:EE，这里有中介吗？我要租房！"
      </p>
     </li>
     <li>
      <p>
       <strong>
        中介推荐（Offer）
       </strong>
       <br/>
       DHCP服务器回应："10.0.0.100 这个房间给你，租期24小时，物业在10.0.0.1"
      </p>
     </li>
     <li>
      <p>
       <strong>
        确认签约（Request）
       </strong>
       <br/>
       设备确认："我就要这间10.0.0.100！"
      </p>
     </li>
     <li>
      <p>
       <strong>
        交钥匙（Acknowledge）
       </strong>
       <br/>
       服务器："签约成功！这是你的钥匙（IP配置），记得每天续约"
      </p>
     </li>
    </ol>

     <strong>

    <h5>
     1️⃣ 安装中介公司系统
    </h5>
    <p>
    </p>

    <h5>
     2️⃣ 编写租房规则（配置文件）
    </h5>
    <p>
     编辑
     <code>
      /etc/dhcp/dhcpd.conf
     </code>
     ：
    </p>
    <pre><code class="hljs"># 小区基本信息
subnet 10.0.0.0 netmask 255.255.255.0 {
  range 10.0.0.100 10.0.0.200;       # 可出租房间号范围
  option routers 10.0.0.1;           # 物业地址（网关）
  option domain-name-servers 8.8.8.8; # 小区导航员（DNS）
  default-lease-time 86400;          # 默认租期24小时（秒）
  max-lease-time 172800;             # 最长租期48小时
  host printer {                     # 固定保留房间
    hardware ethernet AA:BB:CC:DD:EE:FF;
    fixed-address 10.0.0.50;
  }
}</code></pre>
    <pre></pre>
    <h5>
     3️⃣ 开业运营
    </h5>
    <p>
    </p>
    <pre><code class="hljs">sudo systemctl start isc-dhcp-server  # 启动服务
sudo systemctl enable isc-dhcp-server # 设置开机自启</code></pre>

    <h2 id="ip-routing" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        IP 路由
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       Linux 中的 IP 路由是指在 Linux作系统中为网络接口设置路由表和配置网络路由的过程。内核负责处理此任务，其中涉及选择将网络数据包发送到网络上预期目的地的路径。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       此任务使用各种命令行工具和网络配置文件执行。Linux 中用于网络配置的主要命令行工具曾经是 ，但现在它已大部分被 command 取代。
       <code>
        ifconfig
       </code>
       <code>
        ip
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       例如，要在 Linux 中查看路由表，请使用以下命令：
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">$ ip route show</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       此命令返回内核已知的所有路由的列表。
      </span>
     </span>
    </p>
    <h4>
     🚦
     <strong>
      IP路由就像城市快递分拣系统
     </strong>
    </h4>
    <p>
     <strong>
      核心概念：
     </strong>
     Linux系统相当于一个智能快递分拣中心，路由表就是它的分拣规则手册，决定每个包裹（数据包）该走哪条运输路线
    </p>
    <hr/>
    <h4>
     📦
     <strong>
      快递分拣流程（数据包路由）
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        包裹到达
       </strong>
       ：收到寄往
       <code>
        北京朝阳区奥运村1号
       </code>
       的包裹（目标IP：
       <code>
        192.168.1.100
       </code>
       ）
      </p>
     </li>
     <li>
      <p>
       <strong>
        查分拣手册（路由表）
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         先看是否同城快递（同一子网）
        </p>
       </li>
       <li>
        <p>
         不是则找跨省专线（默认网关）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        选择最优路线
       </strong>
       ：
      </p>
      <pre><code class="hljs">192.168.1.0/24 dev eth0  # 朝阳区件走1号分拣口（eth0网卡）
default via 10.0.0.1     # 其他件统一发往华北中转站（默认网关）</code></pre>
      <pre></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     📋
     <strong>
      分拣手册管理（路由表操作）
     </strong>
    </h4>
    <h5>
     查看当前分拣规则
    </h5>
    <p>
    </p>
    <pre><code class="hljs">ip route show
# 输出示例：
# default via 192.168.1.1 dev eth0   # 默认走1号门，发往192.168.1.1
# 10.8.0.0/24 dev tun0 proto static   # 所有10.8.0.x的件走VPN通道
# 192.168.2.0/24 via 192.168.1.254   # 朝阳区件转给隔壁分拣中心</code></pre>
    <pre></pre>
    <h5>
     添加临时专线（临时路由）
    </h5>
    <p>
    </p>
    <pre><code class="hljs">sudo ip route add 172.16.0.0/16 via 10.0.0.2 dev eth1
# 解释：所有寄往172.16.x.x的包裹，走eth1通道，交给10.0.0.2中转</code></pre>
    <pre></pre>
    <h5>
     删除错误路线
    </h5>
    <p>
    </p>
    <pre><code class="hljs">sudo ip route del 172.16.0.0/16  # 取消之前的专线设置</code></pre>
    <pre></pre>
    <h5>
     永久修改分拣手册（持久化配置）
    </h5>
    <p>
     编辑
     <code>
      /etc/network/interfaces
     </code>
     （Debian系）：
    </p>
    <pre><code class="hljs">up ip route add 172.16.0.0/16 via 10.0.0.2</code></pre>

    <h2 id="dns-resolution-in-networking-on-linux" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        Linux 上联网中的 DNS 解析
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       域名系统 （DNS） 是一种去中心化的系统，用于将主机名转换为 IP 地址，使用户更容易访问网站，而无需记住特定的数字 IP 地址。因此，DNS 解析是 Linux 中联网的一个关键方面。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在 Linux 系统上，当应用程序需要连接到某个 URL 时，它会咨询 DNS 解析器。此解析器使用 file 与 DNS 服务器通信，然后 DNS 服务器将 URL 转换为 IP 地址以建立网络连接。
       <code>
        /etc/resolv.conf
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       以下命令用于查询 DNS 和获取 IP 地址：
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">nslookup www.example.com</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       或使用 dig 命令：
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">dig www.example.com</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       充分了解 DNS 解析过程为 Linux 系统上的网络故障排除和 Web 服务器设置等任务提供了坚实的基础。
      </span>

    <h2 id="netfilter">
     网络过滤器
    </h2>
    <p>
     Netfilter 是 Linux 中包含的一个强大工具，它提供用于作和更改网络数据包的功能。它本质上是一个框架，充当内核和数据包之间的接口，允许对传输中的数据包进行作和转换。
    </p>
    <p>
     Netfilter 的主要应用是开发防火墙系统和管理网络地址转换 （NAT）。在 Linux 中，netfilter 非常有价值，因为它提供了广泛的应用程序，从流量控制、数据包修改、日志记录和网络入侵检测。
    </p>
    <p>
     netfilter 的结构允许将自定义函数（通常称为钩子）插入到内核的网络堆栈中。这些钩子可以在各个阶段作或检查数据包，例如预路由、本地输入、转发、本地输出和后路由。
    </p>
    <p>
     与 netfilter 结合使用的常用工具是 iptables，它提供了一种机制来配置 Netfilter 框架提供的内核中的表。
    </p>
    <p>
     以下是使用 iptables 和 netfilter 模块创建简单防火墙规则的示例：
    </p>
    <pre><code>iptables -A INPUT -i eth0 -s 192.168.0.0/24 -m netfilter --netfilter-name example --action drop 
</code></pre>
    <p>
     在此命令中，'-A INPUT' 向 'INPUT' 链添加新规则。“-i eth0”指定网络接口，“-s 192.168.0.0/24”指定规则的 IP 地址范围。“-m netfilter”正在调用 netfilter 模块，“—netfilter-name example”用于命名规则，“—action drop”用于指定如何处理匹配的数据包（在本例中，丢弃它们）。
    </p>
    <h4>
     🛡️
     <strong>
      Netfilter 就像智能快递分拣中心
     </strong>
    </h4>
    <p>
     <strong>
      核心概念：
     </strong>
     把网络数据包比作快递包裹，Netfilter 是分拣中心的智能控制系统，决定每个包裹的去向
    </p>

    <p>
    </p>

    <pre></pre>
    <ul>
     <li>
      <p>
       <strong>
        PREROUTING
       </strong>
       ：包裹刚进分拣中心（刚到达网卡）
      </p>

    <h2 id="ssh-secure-shell">
     SSH （安全外壳）
    </h2>
    <p>
     在 Linux 网络领域，Secure Shell （SSH） 起着至关重要的作用。SSH 是一种加密网络协议，主要用于两台联网计算机之间的安全数据通信、远程命令行登录、远程命令执行和其他安全网络服务。SSH 强调数据传输过程中数据的机密性、完整性和安全性，提供了一种比 Telnet 等不安全的远程访问方法更安全的远程访问方法。
    </p>
    <p>
     鉴于它的重要性和广泛使用，对于任何希望浏览 Linux作系统和有效管理网络的人来说，对其功能有深入的了解都是必不可少的。
    </p>
    <p>
     以下是使用 SSH 从本地计算机连接到远程服务器的示例：
    </p>
    <pre><code>ssh username@server_ip_address
</code></pre>
    <p>
     在上面的命令中，'username' 代表远程用户帐户名，'server_ip_address' 是您尝试访问的远程服务器的 IP 地址。输入此命令后，系统将提示您输入指定用户帐户的密码。验证成功后，您将登录到远程 Linux 服务器。
    </p>

    <h2 id="linux-file-transfer-under-networking" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        网络下的 Linux 文件传输
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在 Linux 中，文件传输是通过网络连接将文件从一台计算机复制或移动到另一台计算机的作。对于需要能够在系统或网络之间共享文件的系统管理员和最终用户来说，此概念至关重要。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       Linux 为基于网络的文件传输提供了多个命令行工具和应用程序。这些工具支持各种标准协议，例如 FTP、HTTP、SCP、SFTP 和 NFS。一些最著名的文件传输命令包括 、 和 .
       <code>
        scp
       </code>
       <code>
        rsync
       </code>
       <code>
        wget
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       例如，将文件从本地计算机传输到远程服务器时，可以按如下方式使用该命令：
       <code>
        scp
       </code>
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">scp /path/to/local/file username@remote:/path/to/destination</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       此命令会将文件复制到指定的远程系统。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       了解并有效地使用这些工具可以简化、更轻松、更安全的网络文件共享任务。
      </span>
     </span>
    </p>
