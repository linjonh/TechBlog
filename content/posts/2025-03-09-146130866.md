    <p>
     在 Java 线程池中，
     <code>
      execute
     </code>
     方法和
     <code>
      submit
     </code>
     方法都用于向线程池提交任务，但它们在异常处理方面存在显著差异，下面为你详细介绍。
    </p>
    <h4>
     <a id="_2">
     </a>
     方法概述
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        execute
       </code>
       方法
      </strong>
      ：
      <code>
       execute
      </code>
      方法是
      <code>
       Executor
      </code>
      接口定义的方法，在
      <code>
       ThreadPoolExecutor
      </code>
      类中实现，用于提交一个
      <code>
       Runnable
      </code>
      任务到线程池执行。
     </li>
     <li>
      <strong>
       <code>
        submit
       </code>
       方法
      </strong>
      ：
      <code>
       submit
      </code>
      方法是
      <code>
       ExecutorService
      </code>
      接口定义的方法，有多个重载版本，可接受
      <code>
       Runnable
      </code>
      或
      <code>
       Callable
      </code>
      任务，并且会返回一个
      <code>
       Future
      </code>
      对象，用于获取任务的执行结果。
     </li>
    </ul>
    <h4>
     <a id="_6">
     </a>
     异常处理差异
    </h4>
    <h5>
     <a id="1__8">
     </a>
     1. 异常抛出方式
    </h5>
    <ul>
     <li>
      <strong>
       <code>
        execute
       </code>
       方法
      </strong>
      <ul>
       <li>
        当使用
        <code>
         execute
        </code>
        方法提交
        <code>
         Runnable
        </code>
        任务时，如果任务在执行过程中抛出未捕获的异常，该异常会直接抛给线程池中的工作线程。
       </li>
       <li>
        若没有为线程设置
        <code>
         UncaughtExceptionHandler
        </code>
        ，异常会导致工作线程终止，线程池会记录该异常的堆栈信息并打印到控制台。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        submit
       </code>
       方法
      </strong>
      <ul>
       <li>
        当使用
        <code>
         submit
        </code>
        方法提交
        <code>
         Runnable
        </code>
        或
        <code>
         Callable
        </code>
        任务时，任务执行过程中抛出的异常不会直接抛出，而是被封装在
        <code>
         Future
        </code>
        对象中。
       </li>
       <li>
        只有当调用
        <code>
         Future
        </code>
        对象的
        <code>
         get
        </code>
        方法时，才会以
        <code>
         ExecutionException
        </code>
        的形式抛出，其
        <code>
         getCause
        </code>
        方法可获取实际的异常。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2__16">
     </a>
     2. 异常捕获方式
    </h5>
    <ul>
     <li>
      <strong>
       <code>
        execute
       </code>
       方法
      </strong>
      <ul>
       <li>
        若要捕获
        <code>
         execute
        </code>
        方法提交任务的异常，可在
        <code>
         Runnable
        </code>
        任务的
        <code>
         run
        </code>
        方法中使用
        <code>
         try-catch
        </code>
        块捕获异常，或者通过自定义
        <code>
         ThreadFactory
        </code>
        为线程设置
        <code>
         UncaughtExceptionHandler
        </code>
        来捕获未处理的异常。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        submit
       </code>
       方法
      </strong>
      <ul>
       <li>
        对于
        <code>
         submit
        </code>
        方法提交的任务，可通过
        <code>
         Future
        </code>
        对象的
        <code>
         get
        </code>
        方法捕获异常。在调用
        <code>
         get
        </code>
        方法时，使用
        <code>
         try-catch
        </code>
        块捕获
        <code>
         ExecutionException
        </code>
        ，再通过
        <code>
         getCause
        </code>
        方法获取实际的异常。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="_66">
     </a>
     总结
    </h4>
    <ul>
     <li>
      <code>
       execute
      </code>
      方法提交的任务抛出异常时，异常会直接影响工作线程，需要在任务内部或通过线程的
      <code>
       UncaughtExceptionHandler
      </code>
      进行处理。
     </li>
     <li>
      <code>
       submit
      </code>
      方法提交的任务抛出异常时，异常会被封装在
      <code>
       Future
      </code>
      对象中，需要通过调用
      <code>
       Future
      </code>
      的
      <code>
       get
      </code>
      方法并捕获
      <code>
       ExecutionException
      </code>
      来处理。
      <code>
       如果没有手动在子线程中捕获异常，也没有在主线程中调用futureTask.get()，那么异常就相当于被系统吞掉了，这是相当危险的，出了问题无法定位！！！
      </code>
      。
     </li>
    </ul>
