    <h3>
     <a id="_2">
     </a>
     前言
    </h3>
    <p>
     在平时的开发中发现有人选择开启模态提交界面，有人选择关闭模态提交界面，那么两者有何区别？此文主要梳理
     <code>
      Settings&gt;Version Control&gt;Commit&gt;Use non-modal commit interface
     </code>
     勾选与否以及产生的效果
     <br/>
     版本
     <code>
      IntelliJ IDEA 2023.2.5
     </code>
    </p>
    <h3>
     <a id="_5">
     </a>
     开启模态提交界面
    </h3>
    <p>
     将
     <code>
      Settings&gt;Version Control&gt;Commit&gt;Use non-modal commit interface
     </code>
     取消勾选
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7897a5a7472d4439ab5513023e6d8ed5.png"/>
    </p>
    <h4>
     <a id="_8">
     </a>
     优点
    </h4>
    <h5>
     <a id="commit_9">
     </a>
     快捷且灵活的选择需要commit文件
    </h5>
    <p>
     可以在
     <code>
      Local Changes
     </code>
     页面结合
     <code>
      Shift+鼠标左键
     </code>
     以及
     <code>
      Ctrl+鼠标左键
     </code>
     实现文件批量选择以及排除，选择需要commit的文件
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/4cb9e49c7fa04aedb8b710a1b7dcde7a.gif"/>
    </p>
    <h5>
     <a id="_13">
     </a>
     显示文件修改内容多(主观)
    </h5>
    <p>
     可以看到Git区域内会显示
     <code>
      Local Changes
     </code>
     页面，左侧为我们本地修改的文件，右侧为某个文件的修改内容，默认为
     <code>
      Unified viewer
     </code>
     模式，显示区域占据整个屏幕宽度，比较
     <code>
      利好小屏
     </code>
     。用户也能通过下拉组件修改为
     <code>
      Side-by-side viewer
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0a02453ddc2b43c6954fe44e269fc1a8.png"/>
    </p>
    <h4>
     <a id="_16">
     </a>
     缺点
    </h4>
    <h5>
     <a id="commit_17">
     </a>
     在模态提交界面选择文件，临时关闭模态框重新打开会重置选择的commit文件
    </h5>
    <p>
     场景：如果仅在模态提交界面选择了需要commit的文件，选择到一半如果想要参考其它未修改的文件，需要关闭当前模态提交界面，再次进入模态提交界面时，先前选择的需要commit文件会重置
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/0c201f96a0e84c938a686a10ce646b9d.gif"/>
    </p>
    <h3>
     <a id="_22">
     </a>
     关闭模态提交界面
    </h3>
    <p>
     IDEA的Git的Commit默认使用非模态提交界面，即
     <code>
      Settings&gt;Version Control&gt;Commit&gt;Use non-modal commit interface
     </code>
     勾选了
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8d1726e370a8439988d44e6762339abc.png">
      <br/>
      此时Git界面的
      <code>
       Local Changes
      </code>
      页面会消失，取而代之会在IDEA左侧显示Commit标签，融合了
      <code>
       Local Changes
      </code>
      页面以及模态提交界面的大部分功能
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3b508fb2a65d404997f43ec1dbcfbed5.png"/>
     </img>
    </p>
    <h4>
     <a id="_28">
     </a>
     优点
    </h4>
    <h5>
     <a id="commit_29">
     </a>
     允许在commit选择文件时查看其它没有修改的文件
    </h5>
    <p>
     由于没有涉及到模态窗口的开启关闭，仅需要再侧边窗口操作，所以可以在选择过程中查看其它未修改文件
    </p>
    <h4>
     <a id="_31">
     </a>
     缺点
    </h4>
    <h5>
     <a id="commit_32">
     </a>
     不能灵活选择需要commit文件
    </h5>
    <p>
     只能点击选择按钮选择需要commit文件
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/00d3aee76f324a9f9d41459dc3dd5e7e.gif"/>
    </p>
    <h5>
     <a id="_35">
     </a>
     显示文件修改内容少(主观)
    </h5>
    <p>
     关闭模态提交界面，显示文件修改模式为
     <code>
      Side-by-side viewer
     </code>
     ，对于修改的文件，会展示修改之前的文件以及当前修改的文件内容，不
     <code>
      利好小屏
     </code>
     。用户也能够通过下拉组件修改为
     <code>
      Unified viewer
     </code>
     模式
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f8bb495451d34be7908b5bf3d2833e4e.png"/>
    </p>
    <h3>
     <a id="_40">
     </a>
     总结
    </h3>
    <p>
     IDEA Commit 模态提交界面关闭VS开启对比各有各的特点，近几年的IDEA版本默认设置为
     <code>
      关闭
     </code>
     模态提交界面，欢迎各位评论自己选用那种模式进行Commit
    </p>
