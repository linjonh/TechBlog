    <h2>
     <a id="_1">
     </a>
     问题描述：
    </h2>
    <p>
     ping是最常见的网络命令，用来测试和远程机器是否连通的方法。
     <br/>
     我们常常会遇到一个问题：无法ping一台远程主机。
    </p>
    <h2>
     <a id="_5">
     </a>
     原因分析：
    </h2>
    <p>
     ping不通远程机器，最常见的原因有：
     <br/>
     1、自己所在机器本身处于断网状态；
     <br/>
     2、远程机器处于断网状态；
     <br/>
     3、远程机器处于联网状态，但是屏蔽了ping命令。
    </p>
    <h2>
     <a id="_11">
     </a>
     解决方案：
    </h2>
    <p>
     如何判断是哪个原因导致ping不通呢？
    </p>
    <ul>
     <li>
      <p>
       首先排除情况1：自己所在机器本身处于断网状态
       <br/>
       办法：使用ping命令测试是否和公网服务器相通。例如：
       <br/>
       <code>
        ping www.sina.com.cn
       </code>
      </p>
     </li>
     <li>
      <p>
       其次排除情况2：远程机器处于断网状态
       <br/>
       办法：通过其他手段登录到远程服务器，从服务器端ping公网服务器地址。
      </p>
     </li>
     <li>
      <p>
       最后排除情况3：远程机器处于联网状态，但是屏蔽了ping命令
       <br/>
       办法（以Linux服务器为例）：
       <br/>
       有些服务器从网络安全角度出发，会屏蔽ping命令，即屏蔽ICMP协议（ping是一个命令名，不是协议名称，其底层是利用ICMP协议实现的）。
       <br/>
       Linux服务器有一个文件
       <code>
        /proc/sys/net/ipv4/icmp_echo_ignore_all
       </code>
       ，当这个文件的内容是
       <code>
        1
       </code>
       时候，为允许ICMP协议，当为
       <code>
        0
       </code>
       时，为屏蔽ICMP协议。
       <br/>
       解决方法很简单：
       <br/>
       如果想打开ICMP协议：
      </p>
     </li>
    </ul>
    <pre><code>echo “1”&gt;/proc/sys/net/ipv4/icmp_echo_ignore_all
cat /proc/sys/net/ipv4/icmp_echo_ignore_all
</code></pre>
    <p>
     如果想禁止ICMP协议：
    </p>
    <pre><code>echo “0”&gt;/proc/sys/net/ipv4/icmp_echo_ignore_all
cat /proc/sys/net/ipv4/icmp_echo_ignore_all
</code></pre>
    <p>
     （命令行很简单，第一句话就是向文件写内容，第二句就是查看文件内容）
    </p>
