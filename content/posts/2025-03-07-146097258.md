    <blockquote>

     <p>
      Spring Web MVC是Spring框架中的一个用来做网站开发的部分，它是基于Servlet技术的。
     </p>
     <p>
      虽然它的正式名字叫“Spring Web MVC”，但大家一般都简称它“SpringMVC”。

      <tr>
       <td bgcolor="pink">

           </strong>
          </font>
         </font>
        </font>

      </tr>

    <blockquote>
     <p>
      Servlet是一种技术，用来创建动态网页。
     </p>
     <p>
      它其实是一种规范，也就是规定了如何写Java Web应用的规则。但单靠规范本身不能做任何事，必须要开发者根据这些规则写代码，才能实现它的功能。
     </p>
     <p>
      很多公司都可以根据这个规范来开发相应的软件，比如Tomcat、Weblogic、Jetty等，它们被叫做“Servlet容器”。
     </p>
     <p>
      这些容器的作用就是管理和运行开发者写的Servlet代码。
     </p>
    </blockquote>
    <p>
     从上述定义我们可以得出一个信息：Spring Web MVC是一个Web框架。 下面咱们简称之为：Spring MVC
    </p>
    <p>
     然而要真正的理解什么是Spring MVC？我们首先要搞清楚什么是MVC？
    </p>

    <h4>
     <a id="font_color__Orangefont_facefont_size6strong11_MVC_strongfontfontfont_49">

     <font color="Orange">
      <font face="华文楷体">
       <font size="6">
        <strong>
         1.1 MVC 定义
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     MVC是Model View Controller的缩写，它是软件工程中的一种软件架构设计模式，它把
     <code>
      软件系统
     </code>
     分为
     <code>
      模型
     </code>
     、
     <code>
      视图
     </code>
     和
     <code>
      控制器
     </code>
     三个基本部分。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/16e6ddd26718410bb56eebd1ea47cffb.png"/>
    </p>
    <ul>
     <li>
      <strong>
       Model
      </strong>
      ：数据的管理者，负责存取和处理数据，如数据库操作。
     </li>
     <li>
      <strong>
       View
      </strong>
      ：数据的展示者，用户看到的界面，如网页或APP界面。
     </li>
     <li>
      <strong>
       Controller
      </strong>
      ：协调者，接收用户请求，选择对应的Model处理，并返回结果到View。
     </li>
    </ul>
    <blockquote>
     <ul>
      <li>
       <strong>
        Model
       </strong>
       ：是看不见的数据管家 (数据库操作/计算逻辑)，
       <code>
        后厨
       </code>
      </li>
      <li>
       <strong>
        View
       </strong>
       ：是看得见的界面(你操作的按钮/表格)，
       <code>
        点餐菜单
       </code>
      </li>
      <li>
       <strong>
        Controller：
       </strong>
       是传话小秘书 (在界面和数据之间跑腿传话)，
       <code>
        传菜单的服务员
       </code>
      </li>
     </ul>
    </blockquote>
    <hr/>
    <h4>
     <a id="font_color__Orangefont_facefont_size6strong12_Spring_MVC_strongfontfontfont_68">

     <font color="Orange">
      <font face="华文楷体">
       <font size="6">
        <strong>
         1.2 什么是Spring MVC？
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     MVC是一种架构设计模式，也是一种思想，而Spring MVC是对MVC思想的具体实现。除此之外，Spring MVC还是一个Web框架。
    </p>
    <p>
     总结来说，Spring MVC是一个实现了MVC模式的Web框架。所以，Spring MVC主要关注有两个点：
    </p>
    <ol>
     <li>
      MVC
     </li>
     <li>
      Web框架
     </li>
    </ol>
    <blockquote>
     <p>
      Spring MVC 全称是 Spring Web MVC。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5cc64e5ffdd94995a8e44bfc41ccbb39.png"/>
    </p>
    <p>
     其实，Spring MVC我们在前面已经用过了，在创建Spring Boot项目时，我们勾选的
     <code>
      Spring Web
     </code>
     框架其实就是Spring MVC框架：
    </p>
    <p>
     可以看到，Spring Web的介绍是：
    </p>
    <blockquote>
     <p>
      Build web, including RESTful, applications using Spring MVC. Uses Apache Tomcat as the default embedded container.
     </p>
    </blockquote>
    <p>
     这时候可能有些 uu 就懵了，前面创建的不是SpringBoot项目吗？怎么又变成了Spring MVC项目？他们之间到底有着什么样的关系？
    </p>
    <ul>
     <li>
      SpringBoot是2014年发布的，Spring是2004年发布的，在2014年发布之前，就不能用Spring实现MVC架构吗？显然不是了。
     </li>
     <li>
      Spring Boot只是实现Spring MVC的其中一种方式而已。
     </li>
     <li>
      <font color="red">
       Spring Boot可以添加很多依赖，借助这些依赖实现不同的功能。Spring Boot通过添加Spring Web MVC框架，来实现web功能。
      </font>
     </li>
    </ul>
    <hr/>
    <blockquote>

        <strong>
         2. 学习Spring MVC
        </strong>
       </font>
      </font>
     </font>
    </h3>
    <hr/>
    <p>
     既然是Web框架，那么当用户在浏览器中输入了
     <code>
      url
     </code>
     之后，我们的Spring MVC项目就可以感知到用户的请求，并给予响应。
    </p>
    <p>
     咱们学习Spring MVC，重点也就是学习如何通过
     <code>
      浏览器
     </code>
     和
     <code>
      用户程序
     </code>
     进行交互。主要分以下三个方面：
    </p>
    <blockquote>
     <ol>
      <li>
       <p>
        <strong>
         建立连接
        </strong>
        ：让用户（浏览器）能找到并访问你的 Spring 程序，简单来说就是让用户输入网址时能够打开你的网页。
       </p>
      </li>
      <li>
       <p>
        <strong>
         请求
        </strong>
        ：用户在网页上点击、输入等操作时，会发送一些信息（比如表单数据）。你的程序需要能接收到这些信息，以便做进一步处理。
       </p>
      </li>
      <li>
       <p>
        <strong>
         响应
        </strong>
        ：当程序处理完用户的请求后，需要把处理结果（比如网页内容）返回给用户，这就是响应。
       </p>
      </li>
     </ol>
    </blockquote>
    <p>
     总结一下：就是让用户通过浏览器访问程序，获取信息，再把结果返回给用户。
    </p>
    <p>
     对于Spring MVC来说，掌握了以上3个功能就相当于掌握了Spring MVC。
    </p>

        <strong>
         2.2.1 @ RequestMapping 注解介绍
        </strong>
       </font>
      </font>
     </font>
    </h5>
    <hr/>
    <p>
     <code>
      @RequestMapping
     </code>
     是Spring Web MVC应用程序中最常被用到的注解之一，它是用来注册接口的路由映射的。
    </p>
    <p>
     表示服务收到请求时，路径为
     <code>
      /sayHi
     </code>
     的请求就会调用
     <code>
      sayHi
     </code>
     这个方法的代码。
    </p>
    <p>
     <strong>
      路由映射
     </strong>
     ：当用户访问一个URL时，将用户的请求对应到程序中某个类的某个方法的过程就叫路由映射。
    </p>
    <blockquote>
     <p>
      既然
      <code>
       @RequestMapping
      </code>
      已经可以达到我们的目的了，我们为什么还要加
      <code>
       @RestController
      </code>
      呢？
     </p>
     <hr/>
     <p>
      我们把
      <code>
       @RestController
      </code>
      去掉，再来访问一次：
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/03ec932a03e643e780d1644823a9f356.png"/>
     </p>
     <ul>
      <li>
       <p>
        可以看到，程序报了404，找不到该页面。
       </p>
      </li>
      <li>
       <p>
        这就是
        <code>
         @RestController
        </code>
        起到的作用。
       </p>
      </li>
      <li>
       <p>
        一个项目中，会有很多类，每个类可能有很多的方法，Spring程序怎么知道要执行哪个方法呢？
       </p>
      </li>
      <li>
       <p>
        <font color="red">
         Spring会对所有的类进行扫描，如果类加了注解
         <code>
          @RestController
         </code>
         ，Spring 才会去看这个类里面的方法有没有加
         <code>
          @RequestMapping
         </code>
         这个注解，当然它的作用不止这一点。
        </font>
       </p>
      </li>
     </ul>
    </blockquote>

    <h5>
     <a id="font_color__bluefont_facefont_size6strong222__RequestMapping_strongfontfontfont_237">

     <font color="blue">
      <font face="华文楷体">
       <font size="6">
        <strong>
         2.2.2 @ RequestMapping使用
        </strong>
       </font>
      </font>
     </font>
    </h5>
    <hr/>
    <p>
     <code>
      @RequestMapping
     </code>
     既可修饰类，也可以修饰方法，当修饰类和方法时，访问的地址是资源路径，
     <code>
      资源路径=类路径+方法路径
     </code>
     。
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        @RequestMapping
       </code>
       标识一个
       <code>
        类
       </code>
