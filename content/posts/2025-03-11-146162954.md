    </p>
    <ul>
     <li>

      </span>

      </span>

      </span>

      </span>

      </span>
      <span style="color:black">

      </span>
      <span style="color:black">

    <h5>
     Postman发起接口请求
    </h5>
    <ul>
     <li>
      <span style="color:black">
       在接口测试中，接口请求信息中，重点需要关注四大信息：
      </span>
      <span style="color:red">
       接口
      </span>
      <span style="color:red">
       URL
      </span>
      <span style="color:red">

      </span>
     </li>
     <li>
      <span style="color:black">
       接口的响应中，测试关注的重点在于响应的正文，同时可能对响应头和状态码进行校验
      </span>
     </li>
    </ul>
    <h5>
     Postman获取响应
    </h5>
    <ul>
     <li>
      Postman请求发起之后得到响应结果，包括响应主体、响应头、cookies、状态码、请求时间、报文数据大小等信息
     </li>
     <li>
      而接口测试，主要就是对响应得到的信息进行验证，是否与用例设计预期相符
     </li>
    </ul>

    <h3>
     <span style="color:black">
      JSON
     </span>
     <span style="color:black">
      介绍
     </span>
    </h3>
    <p>
     JSON（JavaScript Object Notation ）轻量级的数据交换语言，以文字为基础，且易于阅读。
    </p>
    <ul>
     <li>
      对象（object）:一个对象以“{”开始，并以“}”结束。每个对象包含一系列排序的名称/值对，每个名称/值对之间使用“,”区分。
     </li>
     <li>
      名称/值（collection）：名称和值之间使用“:”隔开，一般的形式是：{key1:value,key2:value2}
     </li>
    </ul>
    <p>
     <img alt="" height="427" src="https://i-blog.csdnimg.cn/direct/15b8daa5157f45e49d530d693b3aa4fe.png" width="759"/>
    </p>
    <h4>
    </h4>
    <h4>
    </h4>
    <h4>
     JSON数据类型
    </h4>
    <h5>
     数值（整数或浮点数）
    </h5>
    <p>
     示例：”age”:86
     <br/>
     ”price”:123.78
     <br/>
     “temperature “:-3.5
     <br/>
     “speed_of_light”:3.12e12
    </p>
    <h5 style="background-color:transparent">
     null（空值）
    </h5>

     null（空值）
     <br/>

     <br/>
     ”name”:null
     <br/>
     ”name”:””
    </p>
    <p>
     <span style="color:#fe2c24">
      注意：””表示的是空字符串，而null表示的才是空值
     </span>
    </p>

     逻辑值（true或false）
    </h5>
    <p>
     示例：”student”:true

    <h3 style="margin-left:0.19in; text-align:left">
     <span style="color:black">
      环境变量和全局变量
     </span>
    </h3>
    <ul>
     <li>
      接口测试过程中，经常需要使用已经设定好的参数或者从其他接口的返回中获取变化的参数。Postman提供了变量的机制供这种场景使用。
     </li>
     <li>
      Postman变量分为全局变量和环境变量两种，在接口请求中可以选择一组环境变量使用。而全局变量可供所有请求使用。使用变量名的格式为{
      <!-- -->
      {变量名}}调用变量的值。
     </li>
    </ul>
    <p>
     <img alt="" height="508" src="https://i-blog.csdnimg.cn/direct/6598ce8d0b8e41c99fbb8e7942f8196a.png" width="735"/>

    </p>
    <ul>
     <li>
      环境变量用于环境
     </li>
    </ul>
    <p>
     //提取token的值
     <br/>
     var jsonData=JSON.parse(responseBody)
     <br/>
     console.log(jsonData.access_token)
     <br/>
     pm.environment.set("token1", jsonData.access_token);
    </p>
    <ul>
     <li>
      全局变量用于用例
     </li>
    </ul>
    <p>
     pm.globals.set("token1", jsonData.access_token);

    <h3>
     <span style="color:black">
      断言
     </span>
    </h3>
    <ul>
     <li>
      断言响应的状态码是200，状态断言
     </li>
    </ul>
    <p>
     pm.test("Status code is 200", function () {
     <!-- -->
     <br/>
     pm.response.to.have.status(200);
     <br/>
     });
    </p>
    <ul>
     <li>
      断言响应正文中包括有一个指定的字符串，业务断言
     </li>
    </ul>
    <p>
     pm.test("Body matches string", function () {
     <!-- -->
     <br/>
     pm.expect(pm.response.text()).to.include("string_you_want_to_search");
     <br/>
     });
    </p>
    <ul>
     <li>
      断言响应正文做json检查，业务断言
     </li>
    </ul>
    <p>
     pm.test("Your test name", function () {
     <!-- -->
     <br/>
     var jsonData = pm.response.json();
     <br/>
     pm.expect(jsonData.value).to.eql(100);
     <br/>
     });
    </p>
    <ul>
     <li>
      断言响应正文等于一个字符串
     </li>
    </ul>
    <p>
     pm.test("Body is correct", function () {
     <!-- -->
     <br/>
     pm.response.to.have.body("response_body_string");
     <br/>
     });
    </p>
    <ul>
     <li>
      断言响应头包含指定的响应头
     </li>
    </ul>
    <p>
     pm.test("Content-Type is present", function () {
     <!-- -->
     <br/>
     pm.response.to.have.header("Content-Type");
     <br/>
     });
    </p>
    <ul>
     <li>
      断言接口请求的时间小于200ms
     </li>
    </ul>
    <p>
     pm.test("Response time is less than 200ms", function () {
     <!-- -->
     <br/>
     pm.expect(pm.response.responseTime).to.be.below(200);
     <br/>
     });
    </p>
    <ul>
     <li>
      断言请求返回的状态码在指定的范围内
     </li>
    </ul>
    <p>
     pm.test("Successful POST request", function () {
     <!-- -->
     <br/>
     pm.expect(pm.response.code).to.be.oneOf([201, 202]);
     <br/>
     });
    </p>

     <li>
      断言状态码信息包含指定的字符串

     pm.test("Status code name has string", function () {
     <!-- -->
     <br/>
     pm.response.to.have.status("Created");
