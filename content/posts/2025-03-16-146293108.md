    <h2>
     <a id="_8">
     </a>
     前言
    </h2>
    <p>
     在现代前端开发中，代码风格的一致性是一个永恒的话题。无论是团队协作还是个人项目，保持代码整洁、可读性强是每个开发者追求的目标。而
     <code>
      VSCode
     </code>
     提供了强大的工具链，让我们可以通过简单的配置实现 按下
     <code>
      Command + S
     </code>
     （Mac）或
     <code>
      Ctrl + S
     </code>
     （Windows/Linux）时自动格式化代码 的功能。听起来很酷对吧？但你知道这背后的机制和细节吗？
    </p>
    <p>
     今天，我们就来深入探讨如何通过
     <code>
      ESLint
     </code>
     和
     <code>
      Prettier
     </code>
     的结合，在
     <code>
      VSCode
     </code>
     中实现这一“自动化”流程，并揭示其中的一些关键点。
    </p>
    <h2>
     <a id="_13">
     </a>
     一、确保插件已正确安装
    </h2>
    <p>
     首先，我们需要确保两个核心工具已经就位：
     <code>
      ESLint
     </code>
     和
     <code>
      Prettier
     </code>
     。它们分别扮演着不同的角色：
    </p>
    <ul>
     <li>
      <strong>
       ESLint
      </strong>
      是一个静态代码分析工具，用于检查代码中的潜在问题，并提供修复建议。
     </li>
     <li>
      <strong>
       Prettier
      </strong>
      是一个代码格式化工具，专注于统一代码风格，比如缩进、引号、分号等。
     </li>
    </ul>
    <p>
     如果你还没有安装这两个插件，可以打开
     <code>
      VSCode
     </code>
     的扩展市场，搜索并安装以下两个插件：
    </p>
    <ul>
     <li>
      <strong>
       ESLint
      </strong>
      （作者：Dirk Baeumer）
     </li>
     <li>
      <strong>
       Prettier - Code formatter
      </strong>
      （作者：Prettier）
     </li>
    </ul>
    <p>
     安装完成后，记得启用它们。
    </p>
    <blockquote>
     <p>
      小提示：插件只是第一步，真正让它们发挥作用的是后续的配置。别急，我们慢慢来。
     </p>
    </blockquote>
    <h2>
     <a id="_28">
     </a>
     二、配置默认格式化工具
    </h2>
    <p>
     <code>
      VSCode
     </code>
     支持多种格式化工具（比如内置的格式化器、
     <code>
      Prettier
     </code>
     、
     <code>
      Beautify
     </code>
     等），但我们希望明确指定
     <code>
      Prettier
     </code>
     作为默认格式化工具。为什么？因为
     <code>
      Prettier
     </code>
     更加专注于代码风格，且与
     <code>
      ESLint
     </code>
     的集成更加顺畅。
    </p>
    <h3>
     <a id="_UI__30">
     </a>
     方法一：通过 UI 设置
    </h3>
    <ol>
     <li>
      打开
      <strong>
       VSCode
      </strong>
      设置（Mac:
      <code>
       Cmd + ,
      </code>
      ，Windows/Linux:
      <code>
       Ctrl + ,
      </code>
      ）。
     </li>
     <li>
      搜索
      <code>
       Default Formatter
      </code>
      。
     </li>
     <li>
      将其设置为
      <code>
       esbenp.prettier-vscode
      </code>
      （即
      <strong>
       Prettier
      </strong>
      插件）。
     </li>
    </ol>
    <h3>
     <a id="_settingsjson_36">
     </a>
     方法二：直接修改 settings.json
    </h3>
    <p>
     如果你更喜欢手动编辑配置文件，可以在
     <code>
      settings.json
     </code>
     中添加如下内容：
    </p>
    <pre><code class="prism language-json"><span class="token string-property property">"[javascript]"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"editor.defaultFormatter"</span><span class="token operator">:</span> <span class="token string">"esbenp.prettier-vscode"</span>
<span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token string-property property">"[typescript]"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"editor.defaultFormatter"</span><span class="token operator">:</span> <span class="token string">"esbenp.prettier-vscode"</span>
<span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token string-property property">"[html]"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"editor.defaultFormatter"</span><span class="token operator">:</span> <span class="token string">"esbenp.prettier-vscode"</span>
<span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token string-property property">"[vue]"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"editor.defaultFormatter"</span><span class="token operator">:</span> <span class="token string">"esbenp.prettier-vscode"</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这段配置的意思是：针对
     <code>
      JavaScript
     </code>
     、
     <code>
      TypeScript
     </code>
     、
     <code>
      HTML
     </code>
     和
     <code>
      Vue
     </code>
     文件，默认使用
     <code>
      Prettier
     </code>
     进行格式化。
    </p>
    <h2>
     <a id="_55">
     </a>
     三、启用保存时自动格式化
    </h2>
    <p>
     接下来，我们需要告诉
     <code>
      VSCode
     </code>
     在保存文件时自动格式化代码。这个功能非常实用，因为它让你无需额外操作即可保持代码整洁。
    </p>
    <h3>
     <a id="_UI__57">
     </a>
     方法一：通过 UI 设置
    </h3>
    <ol>
     <li>
      打开设置（
      <code>
       Cmd +
      </code>
      , 或
      <code>
       Ctrl + ,
      </code>
      ）。
     </li>
     <li>
      搜索
      <code>
       Format On Save
      </code>
      。
     </li>
     <li>
      勾选
      <code>
       Editor: Format On Save
      </code>
      。
     </li>
    </ol>
    <h3>
     <a id="_settingsjson_63">
     </a>
     方法二：直接修改 settings.json
    </h3>
    <p>
     在
     <code>
      settings.json
     </code>
     中添加以下配置：
    </p>
    <pre><code class="prism language-json"><span class="token string-property property">"editor.formatOnSave"</span><span class="token operator">:</span> <span class="token boolean">true</span>
</code></pre>
    <p>
     这样，每次保存文件时，
     <code>
      VSCode
     </code>
     都会调用默认格式化工具（也就是我们刚刚配置的
     <code>
      Prettier
     </code>
     ）对代码进行格式化。
    </p>
    <h2>
     <a id="_ESLint__72">
     </a>
     四、配置 ESLint 自动修复
    </h2>
    <p>
     虽然
     <code>
      Prettier
     </code>
     负责代码风格，但
     <code>
      ESLint
     </code>
     的作用不可忽视——它可以捕获代码中的潜在问题（如未使用的变量、语法错误等）。为了让
     <code>
      ESLint
     </code>
     在保存时也能自动修复这些问题，我们需要启用它的
     <code>
      Auto Fix On Save
     </code>
     功能。
    </p>
    <h3>
     <a id="_UI__74">
     </a>
     方法一：通过 UI 设置
    </h3>
    <ol>
     <li>
      打开设置（
      <code>
       Cmd +
      </code>
      , 或
      <code>
       Ctrl + ,
      </code>
      ）。
     </li>
     <li>
      搜索
      <code>
       ESLint: Auto Fix On Save
      </code>
      。
     </li>
     <li>
      勾选该选项。
     </li>
    </ol>
    <h3>
     <a id="_settingsjson_80">
     </a>
     方法二：直接修改 settings.json
    </h3>
    <p>
     在
     <code>
      settings.json
     </code>
     中添加以下配置：
    </p>
    <pre><code class="prism language-json"><span class="token string-property property">"editor.codeActionsOnSave"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
 <span class="token string-property property">"source.fixAll.eslint"</span><span class="token operator">:</span> <span class="token boolean">true</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这段配置的意思是：在保存文件时，自动运行
     <code>
      ESLint
     </code>
     的修复功能。
    </p>
    <h2>
     <a id="_Prettier__ESLint__90">
     </a>
     五、让 Prettier 和 ESLint 协同工作
    </h2>
    <p>
     到目前为止，我们已经配置了
     <code>
      Prettier
     </code>
     和
     <code>
      ESLint
     </code>
     的基本功能，但它们之间可能会存在规则冲突。例如，
     <code>
      ESLint
     </code>
     可能要求使用双引号，而
     <code>
      Prettier
     </code>
     默认使用单引号。这种冲突会导致格式化结果不一致。
    </p>
    <p>
     为了避免这种情况，我们可以使用
     <code>
      eslint-config-prettier
     </code>
     和
     <code>
      eslint-plugin-prettier
     </code>
     来协调两者的规则。
    </p>
    <p>
     步骤：
    </p>
    <ol>
     <li>
      在项目根目录下安装依赖：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span> --save-dev eslint-config-prettier eslint-plugin-prettier

</code></pre>
    <ol start="2">
     <li>
      更新
      <code>
       .eslintrc.js
      </code>
      文件，添加以下配置：
     </li>
    </ol>
    <pre><code class="prism language-js">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token keyword">extends</span><span class="token operator">:</span> <span class="token punctuation">[</span>
    <span class="token string">'eslint:recommended'</span><span class="token punctuation">,</span>
    <span class="token string">'plugin:prettier/recommended'</span> <span class="token comment">// 启用 Prettier 规则</span>
  <span class="token punctuation">]</span><span class="token punctuation">,</span>
  <span class="token literal-property property">plugins</span><span class="token operator">:</span> <span class="token punctuation">[</span><span class="token string">'prettier'</span><span class="token punctuation">]</span><span class="token punctuation">,</span>
  <span class="token literal-property property">rules</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">'prettier/prettier'</span><span class="token operator">:</span> <span class="token string">'error'</span> <span class="token comment">// 将 Prettier 格式问题视为 ESLint 错误</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     通过这种方式，
     <code>
      ESLint
     </code>
     会优先使用
     <code>
      Prettier
     </code>
     的规则，从而避免冲突。
    </p>
    <h2>
     <a id="_119">
     </a>
     六、验证配置
    </h2>
    <p>
     完成上述步骤后，我们需要验证配置是否生效。以下是验证方法：
    </p>
    <ol>
     <li>
      打开一个
      <code>
       JavaScript/TypeScript/Vue
      </code>
      文件。
     </li>
     <li>
      故意写一些不符合格式规范的代码（例如多余的空格、未对齐的缩进等）。
     </li>
     <li>
      按下
      <code>
       Command + S
      </code>
      或
      <code>
       Ctrl + S
      </code>
      。
     </li>
     <li>
      如果配置正确，代码应该会自动格式化，并且
      <code>
       ESLint
      </code>
      的问题也会被修复。
     </li>
    </ol>
    <h2>
     <a id="_settingsjson__127">
     </a>
     七、完整的 settings.json 示例
    </h2>
    <p>
     为了方便参考，这里是一个完整的
     <code>
      settings.json
     </code>
     配置示例：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>

    <h2>
     <a id="_160">
     </a>
     八、常见问题排查
    </h2>
    <h3>
     <a id="_1_161">
     </a>
     问题 1：格式化未生效
    </h3>
    <p>
     确保当前文件类型支持
     <code>
      Prettier
     </code>
     （例如
     <code>
      .js
     </code>
     ,
     <code>
      .ts
     </code>
     ,
     <code>
      .vue
     </code>
     等）。
     <br/>
     检查
     <code>
      settings.json
     </code>
     中是否正确设置了
     <code>
      editor.defaultFormatter
     </code>
     。
    </p>
    <h3>
     <a id="_2ESLint__166">
     </a>
     问题 2：ESLint 自动修复未生效
    </h3>
    <p>
     确保项目中已安装
     <code>
      ESLint
     </code>
     和相关插件。
     <br/>
     检查
     <code>
      .eslintrc.js
     </code>
     文件是否正确配置了
     <code>
      eslint-plugin-prettier
     </code>
     。
    </p>
    <h3>
     <a id="_3_171">
     </a>
     问题 3：规则冲突
    </h3>
    <p>
     如果
     <code>
      Prettier
     </code>
     和
     <code>
      ESLint
     </code>
     规则冲突，优先使用
     <code>
      eslint-config-prettier
     </code>
     来禁用冲突规则。
    </p>
    <h2>
     <a id="_176">
     </a>
     结语
    </h2>
    <p>
     通过以上步骤，你应该能够实现按下
     <code>
      Command + S
     </code>
     或
     <code>
      Ctrl + S
     </code>
     时自动格式化代码并修复
     <code>
      ESLint
     </code>
     问题的功能！这个小小的配置不仅能提升你的开发效率，还能让你的代码始终保持一致性。
    </p>
    <p>
     当然，工具只是辅助，真正的代码质量还需要靠你的经验和思考。希望这篇文章能帮助你更好地理解
     <code>
      VSCode
     </code>
     的配置机制，并为你带来一些启发！
    </p>
