    <p>
     1.SQL*Loader (sqlldr)
     <br/>
     1.1 概述
     <br/>
     作用：Oracle 官方工具，用于将外部数据文件（如 CSV、文本文件）高效加载到数据库表中。
    </p>
    <p>
     特点：
     <br/>
     支持多种数据格式（定长、变长、CSV）。
     <br/>
     支持直接路径加载（Direct Path Load），绕过 SQL 引擎，提升性能。
     <br/>
     提供错误记录和丢弃无效数据的功能。
    </p>
    <p>
     1.2 核心组件
     <br/>
     控制文件 (Control File)：定义数据加载规则，包括数据文件路径、目标表、字段映射等。
     <br/>
     数据文件 (Data File)：存储待加载的原始数据。
     <br/>
     日志文件 (Log File)：记录加载过程的详细信息。
     <br/>
     错误文件 (Bad File)：存储无法加载的数据。
     <br/>
     废弃文件 (Discard File)：存储因条件过滤被丢弃的数据。
    </p>
    <p>
     1.3 使用步骤
     <br/>
     步骤 1：编写控制文件
    </p>
    <pre><code class="prism language-sql"><span class="token comment">-- 示例：加载 CSV 文件到 EMP 表</span>
OPTIONS <span class="token punctuation">(</span>SKIP<span class="token operator">=</span><span class="token number">1</span><span class="token punctuation">)</span> <span class="token comment">-- 跳过 CSV 文件标题行</span>
<span class="token keyword">LOAD</span> <span class="token keyword">DATA</span>
<span class="token keyword">INFILE</span> <span class="token string">'emp_data.csv'</span> <span class="token comment">-- 数据文件路径</span>
BADFILE <span class="token string">'emp_bad.bad'</span> <span class="token comment">-- 错误文件</span>
DISCARDFILE <span class="token string">'emp_discard.dsc'</span> <span class="token comment">-- 废弃文件</span>
APPEND <span class="token keyword">INTO</span> <span class="token keyword">TABLE</span> EMP <span class="token comment">-- 目标表，APPEND 表示追加数据</span>
<span class="token keyword">FIELDS</span> <span class="token keyword">TERMINATED</span> <span class="token keyword">BY</span> <span class="token string">','</span> <span class="token comment">-- 字段分隔符</span>
TRAILING NULLCOLS <span class="token comment">-- 允许末尾空列</span>
<span class="token punctuation">(</span>
  EMPNO<span class="token punctuation">,</span>
  ENAME<span class="token punctuation">,</span>
  JOB<span class="token punctuation">,</span>
  MGR<span class="token punctuation">,</span>
  HIREDATE <span class="token keyword">DATE</span> <span class="token string">"YYYY-MM-DD"</span><span class="token punctuation">,</span> <span class="token comment">-- 日期格式转换</span>
  SAL<span class="token punctuation">,</span>
  COMM<span class="token punctuation">,</span>
  DEPTNO
<span class="token punctuation">)</span>
</code></pre>
    <p>
     步骤 2：执行 SQL*Loader 命令
    </p>
    <pre><code class="prism language-sql">sqlldr userid<span class="token operator">=</span>scott<span class="token operator">/</span>tiger<span class="token variable">@orcl</span> control<span class="token operator">=</span>loader<span class="token punctuation">.</span>ctl log<span class="token operator">=</span>loader<span class="token punctuation">.</span>log
</code></pre>
    <p>
     1.4 高级功能
     <br/>
     并行加载：使用 PARALLEL 参数加速大数据加载。
     <br/>
     直接路径加载：通过 DIRECT=TRUE 提升性能（需注意表锁定）。
     <br/>
     条件过滤：使用 WHEN 子句过滤数据。
     <br/>
     数据转换：在控制文件中使用 SQL 函数（如 TO_DATE, SUBSTR）。
    </p>
    <p>
     2.SQLULDR2 (SQLLDR 的增强工具）
     <br/>
     2.1 概述
     <br/>
     作用：第三方工具（非 Oracle 官方），用于从 Oracle 数据库高效导出数据到文件。
    </p>
    <p>
     特点：
     <br/>
     性能远超传统 UTL_FILE 或 SPOOL。
     <br/>
     支持多线程导出。
     <br/>
     输出格式灵活（CSV、定长、自定义分隔符）。
     <br/>
     支持压缩和拆分文件。
    </p>
    <p>
     2.2 安装与使用
     <br/>
     下载：从第三方网站（如 www.sqluldr2.com）获取二进制文件。
    </p>
    <p>
     基本命令：
    </p>
    <pre><code class="prism language-sql">sqluldr2 <span class="token keyword">user</span><span class="token operator">=</span>scott<span class="token operator">/</span>tiger<span class="token variable">@orcl</span> query<span class="token operator">=</span><span class="token string">"SELECT * FROM EMP"</span> <span class="token keyword">file</span><span class="token operator">=</span>emp_data<span class="token punctuation">.</span>csv
</code></pre>
    <p>
     2.3 常用参数
     <br/>
     参数 说明
     <br/>
     user 数据库连接字符串
     <br/>
     query 导出数据的 SQL 查询
     <br/>
     file 输出文件路径
     <br/>
     head 是否包含列标题（默认为 yes）
     <br/>
     field 字段分隔符（默认 ,）
     <br/>
     charset 输出文件字符集（如 UTF8）
     <br/>
     thread 多线程导出（提升性能）
     <br/>
     split 拆分文件大小（如 100M）
    </p>
    <p>
     2.4 示例
    </p>
    <pre><code class="prism language-sql"><span class="token comment">#导出数据为 CSV，使用 4 线程，文件拆分为 100MB</span>
sqluldr2 <span class="token keyword">user</span><span class="token operator">=</span>scott<span class="token operator">/</span>tiger<span class="token variable">@orcl</span> query<span class="token operator">=</span><span class="token string">"SELECT * FROM EMP"</span> \
<span class="token keyword">file</span><span class="token operator">=</span>emp_data<span class="token punctuation">.</span>csv head<span class="token operator">=</span>yes field<span class="token operator">=</span><span class="token punctuation">,</span> <span class="token keyword">charset</span><span class="token operator">=</span>UTF8 thread<span class="token operator">=</span><span class="token number">4</span> split<span class="token operator">=</span><span class="token number">100</span>M
</code></pre>
    <p>
     3.常见问题与解决
     <br/>
     3.1 SQL*Loader
     <br/>
     错误 ORA-00942：检查用户是否有目标表的权限。
     <br/>
     日期格式错误：确保控制文件中的日期格式与数据文件匹配。
     <br/>
     性能慢：尝试启用 DIRECT=TRUE 或调整 READSIZE/BINDSIZE。
    </p>
    <p>
     3.2 SQLULDR2
     <br/>
     中文乱码：指定正确的字符集（如 charset=ZHS16GBK）。
     <br/>
     内存不足：减少线程数（thread）或增加 rows 参数。
     <br/>
     权限问题：确保导出目录有写入权限。
    </p>
