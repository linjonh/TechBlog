    <h3 id="1-java中的初始化">
     1. Java中的初始化
    </h3>
    <p>
     Java与C++的一个不同之处在于，Java不仅有构造函数，还有一个“初始化块”(Initialization Block)的概念。Java中的初始化块在创建Java对象时隐式执行，并且是在构造函数之前执行。
    </p>
    <h3 id="2-静态初始化">
     2. 静态初始化
    </h3>
    <pre class="prettyprint"><code class="hljs r">// 定义
static {
    <span class="hljs-keyword">...</span>
}</code></pre>
    <p>
     静态初始化块执行的优先级高于非静态初始化块，在对象装载到JVM中时执行一次，仅能初始化类成员变量，即static修饰的数据成员。
    </p>
    <h3 id="3-非静态初始化">
     3. 非静态初始化
    </h3>
    <pre class="prettyprint"><code class="hljs r">// 定义
{
    <span class="hljs-keyword">...</span>
}</code></pre>
    <p>
     非静态初始化块在每个对象生成时都会被执行一次，它可以初始化类的实例变量。非静态初始化块在构造函数之前执行。
    </p>
    <h3 id="4-demo">
     4. Demo
    </h3>
    <ul>
     <li>
      Example
     </li>
    </ul>
    <pre class="prettyprint"><code class="language-Java hljs cs">class Test {

    <span class="hljs-keyword">static</span> {
        System.<span class="hljs-keyword">out</span>.println(<span class="hljs-string">"Run static initailization block."</span>);
    }

    {
        System.<span class="hljs-keyword">out</span>.println(<span class="hljs-string">"Run nonstatic initailization block."</span>);
    }

    <span class="hljs-keyword">public</span> <span class="hljs-title">Test</span>() {
        System.<span class="hljs-keyword">out</span>.println(<span class="hljs-string">"Run Test constructor."</span>);
    }

    <span class="hljs-keyword">public</span> <span class="hljs-keyword">static</span> <span class="hljs-keyword">void</span> <span class="hljs-title">main</span>(String[] args) {
        Test t = <span class="hljs-keyword">new</span> Test();
    }
}</code></pre>
    <ul>
     <li>
      Result
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs oxygene">Run <span class="hljs-keyword">static</span> initailization <span class="hljs-keyword">block</span>.
Run nonstatic initailization <span class="hljs-keyword">block</span>.
Run Test <span class="hljs-function"><span class="hljs-keyword">constructor</span>.</span></code></pre>
    <h3 id="5-总结">
     5. 总结
    </h3>
    <p>
     从某种程度上来看，初始化块是构造器的补充，初始化块总是在构造器之前执行。初始化块是一段固定执行的代码，它不能接受任何参数。因此初始化块对同一个类的所有对象所进行的初始化处理完全相同。如果有一段初始化处理代码对所有的对象完全相同，且无需接受任何参数，就可以把这段初始化处理代码提取到初始化块中。通过把多个构造器中的相同代码提取到初始化块中定义，能更好地提高初始化代码的复用。静态初始化块是类相关的，系统将在类加载时执行静态初始化块，而不是在创建对象时才执行，因此静态初始化块总是比非静态初始化块先执行。用途：例如在JNI调用时，需要加载动态链接库，就可以在静态代码块中加载。
    </p>
