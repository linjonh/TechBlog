     linux在 Ubuntu 系统中设置服务器时间
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     在 Ubuntu 系统中设置服务器时间通常涉及以下步骤，涵盖自动同步和手动配置两种方式。以下是详细操作指南：
    </p>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     <strong>
      一、检查当前时间状态
     </strong>
    </h4>
    <pre><code class="prism language-bash">timedatectl status
</code></pre>
    <ul>
     <li>
      输出示例：
      <pre><code>Local time: Wed 2023-10-18 15:30:00 UTC
Universal time: Wed 2023-10-18 15:30:00 UTC
RTC time: Wed 2023-10-18 15:30:00
Time zone: Etc/UTC (UTC, +0000)
System clock synchronized: yes
NTP service: active
RTC in local TZ: no
</code></pre>
      <ul>
       <li>
        <strong>
         关键信息
        </strong>
        ：时区（Time zone）、NTP 是否启用（NTP service）、系统时间同步状态。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_22">
     </a>
     <strong>
      二、设置时区
     </strong>
    </h4>
    <h5>
     <a id="1__23">
     </a>
     1. 列出所有可用时区
    </h5>
    <pre><code class="prism language-bash">timedatectl list-timezones
</code></pre>
    <h5>
     <a id="2__AsiaShanghai_28">
     </a>
     2. 设置时区（如
     <code>
      Asia/Shanghai
     </code>
     ）
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> timedatectl set-timezone Asia/Shanghai
</code></pre>
    <h5>
     <a id="3__33">
     </a>
     3. 验证时区
    </h5>
    <pre><code class="prism language-bash"><span class="token function">date</span>
<span class="token comment"># 或</span>
timedatectl
</code></pre>
    <hr/>
    <h4>
     <a id="NTP__42">
     </a>
     <strong>
      三、自动同步时间（NTP 服务）
     </strong>
    </h4>
    <h5>
     <a id="1__NTP__43">
     </a>
     1. 启用 NTP 时间同步
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> timedatectl set-ntp <span class="token boolean">true</span>
</code></pre>
    <h5>
     <a id="2__NTP__48">
     </a>
     2. 检查 NTP 服务状态
    </h5>
    <pre><code class="prism language-bash">systemctl status systemd-timesyncd
</code></pre>
    <h5>
     <a id="3__NTP__53">
     </a>
     3. 安装更强大的 NTP 工具（可选）
    </h5>
    <ul>
     <li>
      <strong>
       Chrony
      </strong>
      （推荐）：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> chrony
<span class="token function">sudo</span> systemctl restart chrony
</code></pre>
     </li>
     <li>
      <strong>
       ntpd
      </strong>
      ：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> ntp
<span class="token function">sudo</span> systemctl restart ntp
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="4__NTP__65">
     </a>
     4. 配置 NTP 服务器（如自定义服务器）
    </h5>
    <p>
     编辑配置文件（以 Chrony 为例）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">nano</span> /etc/chrony/chrony.conf
</code></pre>
    <ul>
     <li>
      添加或替换为国内 NTP 服务器：
      <pre><code>server ntp.aliyun.com iburst
server cn.pool.ntp.org iburst
</code></pre>
     </li>
    </ul>
    <p>
     重启服务：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart chrony
</code></pre>
    <hr/>
    <h4>
     <a id="_82">
     </a>
     <strong>
      四、手动设置时间
     </strong>
    </h4>
    <h5>
     <a id="1__NTP__83">
     </a>
     1. 禁用 NTP 同步（避免覆盖手动设置）
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> timedatectl set-ntp <span class="token boolean">false</span>
</code></pre>
    <h5>
     <a id="2__88">
     </a>
     2. 手动设置时间
    </h5>
    <ul>
     <li>
      <strong>
       方法 1
      </strong>
      ：使用
      <code>
       timedatectl
      </code>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> timedatectl set-time <span class="token string">"2023-10-18 15:30:00"</span>
</code></pre>
     </li>
     <li>
      <strong>
       方法 2
      </strong>
      ：使用
      <code>
       date
      </code>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">date</span> <span class="token parameter variable">-s</span> <span class="token string">"2023-10-18 15:30:00"</span>
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3_RTC_98">
     </a>
     3. 同步硬件时钟（RTC）
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> hwclock <span class="token parameter variable">--systohc</span>
</code></pre>
    <hr/>
    <h4>
     <a id="_105">
     </a>
     <strong>
      五、验证时间配置
     </strong>
    </h4>
    <h5>
     <a id="1__106">
     </a>
     1. 检查系统时间
    </h5>
    <pre><code class="prism language-bash"><span class="token function">date</span>
</code></pre>
    <h5>
     <a id="2__111">
     </a>
     2. 检查硬件时钟
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> hwclock <span class="token parameter variable">--show</span>
</code></pre>
    <h5>
     <a id="3__NTP_Chrony_116">
     </a>
     3. 检查 NTP 同步状态（Chrony）
    </h5>
    <pre><code class="prism language-bash">chronyc tracking
</code></pre>
    <hr/>
    <h4>
     <a id="_123">
     </a>
     <strong>
      六、常见问题
     </strong>
    </h4>
    <h5>
     <a id="1_NTP__124">
     </a>
     1.
     <strong>
      NTP 同步失败
     </strong>
    </h5>
    <ul>
     <li>
      检查防火墙是否放行 UDP 123 端口。
     </li>
     <li>
      查看日志：
      <pre><code class="prism language-bash">journalctl <span class="token parameter variable">-u</span> systemd-timesyncd
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="2__131">
     </a>
     2.
     <strong>
      时区设置无效
     </strong>
    </h5>
    <ul>
     <li>
      确保时区名称正确（区分大小写）。
     </li>
     <li>
      重启系统或服务：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart systemd-timesyncd
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3__138">
     </a>
     3.
     <strong>
      时间偏差较大
     </strong>
    </h5>
    <ul>
     <li>
      手动强制同步：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> chronyc <span class="token parameter variable">-a</span> makestep
</code></pre>
     </li>
    </ul>
    <hr/>
    <p>
     通过以上步骤，您可以根据需求灵活配置 Ubuntu 服务器的时间。推荐优先使用
     <strong>
      NTP 自动同步
     </strong>
     以确保时间准确性，尤其是在生产环境中。
    </p>
