    <h2>
     <a id="docker_0">
     </a>
     docker引擎断电后启动失败常见问题和解决方案
    </h2>
    <h3>
     <a id="_2">
     </a>
     常用排查命令
    </h3>
    <ul>
     <li>
      查看containerd运行状态
      <code>
       systemctl status containerd
      </code>
     </li>
     <li>
      查看docker引擎运行状态
      <code>
       systemctl status docker
      </code>
     </li>
     <li>
      在系统日志中查看docker引擎最近日志并持续追踪
      <code>
       journalctl -u docker.service -f -n 100
      </code>
     </li>
    </ul>
    <h3>
     <a id="_7">
     </a>
     常见问题和解决
    </h3>
    <h4>
     <a id="_8">
     </a>
     容器加载失败问题
    </h4>
    <p>
     使用
     <code>
      journalctl
     </code>
     命令查看
     <code>
      docker
     </code>
     日志，提示某个容器加载失败，常见于断电使容器文件系统损坏。
    </p>
    <pre><code class="prism language-log">dockerd[26166]: time="xxx" level=error msg="failed to load container" container=xxxxx error="invalid character '\\x00' looking for beginning of value"
</code></pre>
    <h5>
     <a id="_13">
     </a>
     解决步骤
    </h5>
    <ol>
     <li>
      根据日志里
      <code>
       container=
      </code>
      后的容器id，
      <code>
       rm /var/lib/docker/containers/&lt;容器id&gt;
      </code>
      删除对应容器目录
     </li>
     <li>
      重启
      <code>
       systemctl restart docker
      </code>
     </li>
    </ol>
    <h4>
     <a id="Page_expected_17">
     </a>
     引擎Page expected异常退出问题
    </h4>
    <p>
     使用
     <code>
      journalctl
     </code>
     命令查看
     <code>
      docker
     </code>
     日志，发现docker的golang源码抛出panic异常
    </p>
    <pre><code class="prism language-log">dockerd[26166]: panic: assertion failed: Page expected to be: 34, but self identifies as xxx
</code></pre>
    <h5>
     <a id="_23">
     </a>
     解决步骤
    </h5>
    <p>
     使用玄学删除文件
     <br/>
     <a href="https://github.com/containerd/containerd/issues/3347">
      参考：containerd/issues/3347 Containerd is crashing with panic
     </a>
     <br/>
     <a href="https://github.com/containerd/containerd/issues/3347#issuecomment-1159965884">
      参考：Hope will help someone
     </a>
    </p>
    <ul>
     <li>
      Stop Docker and containerd:
      <br/>
      <code>
       systemctl stop docker containerd
      </code>
     </li>
     <li>
      Cleanup containerd data directory (Docker will regenerate it at startup if needed):
      <br/>
      <code>
       rm -rf /var/lib/containerd/
      </code>
     </li>
     <li>
      Find Docker’s database files - one of them (most often local-kv.db) corrupted in your system:
      <br/>
      <code>
       find /var/lib/docker -type f -size -5M -name '*.db' | grep -v overlay2
      </code>
      <br/>
      - will output something like:
      <br/>
      /var/lib/docker/containerd/daemon/io.containerd.metadata.v1.bolt/meta.db
      <br/>
      /var/lib/docker/volumes/metadata.db
      <br/>
      /var/lib/docker/network/files/local-kv.db
      <br/>
      /var/lib/docker/builder/fscache.db
      <br/>
      /var/lib/docker/buildkit/snapshots.db
      <br/>
      /var/lib/docker/buildkit/metadata.db
      <br/>
      /var/lib/docker/buildkit/cache.db
     </li>
     <li>
      Simply rename this file to .bak:
      <br/>
      <code>
       mv /var/lib/docker/network/files/local-kv.db{,.bak}
      </code>
     </li>
     <li>
      Start Docker:
      <br/>
      <code>
       systemctl start docker
      </code>
      <br/>
      引擎重启，重启容器验证，由于直接删除了引擎的
      <code>
       .db
      </code>
      数据文件，直接
      <code>
       docker restart xx
      </code>
      重启容器会出现关联数据找不到的问题。建议先
      <code>
       docker remove
      </code>
      或
      <code>
       docker compose down
      </code>
      删除旧容器数据。
     </li>
    </ul>
