   <div class="htmledit_views" id="content_views">
    <p>
     <a href="https://juejin.cn/post/7236028062872600636?searchId=202307301710104CCFBB41B79F242680A1" rel="nofollow" title="谈谈 cookie &amp; session &amp; jwt - 掘金">
      谈谈 cookie &amp; session &amp; jwt - 掘金
     </a>
    </p>
    <p>
     前端在每次请求时把
     <code>
      token
     </code>
     放在请求头中发送给后端，目前有两种方式：
    </p>
    <blockquote>
     <ul>
      <li>
       <p>
        一是
        <span style="color:#fe2c24;">
         <strong>
          通过
          <code>
           cookie
          </code>
          的形式
         </strong>
        </span>
        ，即把
        <code>
         token
        </code>
        放在
        <code>
         cookie
        </code>
        中，每次浏览器会自动帮我们带过去，不需要我们自己设置。
       </p>
      </li>
      <li>
       <p>
        二是
        <span style="color:#fe2c24;">
         <strong>
          放在请求头
          <code>
           header Authorization
          </code>
          中
         </strong>
        </span>
        ，需要我们自己手动设置请求头。通常获取到
        <code>
         token
        </code>
        之后，会存放在
        <span style="color:#fe2c24;">
         <strong>
          <code>
           sessionStorage
          </code>
         </strong>
        </span>
        或者
        <span style="color:#fe2c24;">
         <strong>
          <code>
           localStorage
          </code>
         </strong>
        </span>
        中，这样当页面刷新之后保证
        <code>
         token
        </code>
        不会消失。
       </p>
      </li>
     </ul>
    </blockquote>
    <pre><code class="language-bash">Authorization: Bearer eyJhbGci*...&lt;snip&gt;...*yu5CSpyHI</code></pre>
