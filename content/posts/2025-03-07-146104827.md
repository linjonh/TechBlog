    <h2>
     <a id="debian_13">
     </a>
     一、debian根文件系统是什么？
    </h2>
    <p>
     deepseek：Debian 的根文件系统（Root Filesystem，简写为 /）是 Debian 操作系统的基础文件系统结构，它是整个操作系统的核心组成部分，包含了启动、运行和管理系统所需的全部目录、文件、配置和工具。根文件系统是所有其他文件系统（如 /home、/boot 等）的挂载起点，是系统运行的基石：）
    </p>
    <h2>
     <a id="_16">
     </a>
     二、构建步骤
    </h2>
    <h3>
     <a id="1_17">
     </a>
     1.准备工作-第一阶段
    </h3>
    <p>
     安装构建文件系统的工具，一个是用来chroot，一个是用来构建文件系统。chroot的作用是改变根目录，qemu是为了在x86架构的ubuntu上模拟arm架构系统。
    </p>
    <pre><code class="prism language-c">sudo apt install qemu<span class="token operator">-</span>user<span class="token operator">-</span><span class="token keyword">static</span>
sudo apt install debootstrap
mkdir debian11  <span class="token comment">// 可以建一个目录用来和别的目录区别，这个随意</span>
cd debian11
mkdir rootfs  <span class="token comment">// 用来存放要构建的根文件系统</span>
</code></pre>
    <p>
     看你想要构建Debian10还是Debian11或者其他的，需要找到对应的源。而且要找到自己开发板架构对应的源。licheepi nano的架构是armel。
    </p>
    <pre><code class="prism language-c"><span class="token comment">// debian 10 ，有armel</span>
http<span class="token operator">:</span><span class="token comment">//mirrors.huaweicloud.com/debian-archive/debian/</span>
<span class="token comment">// debian 11 ，有armel</span>
https<span class="token operator">:</span><span class="token comment">//mirrors.huaweicloud.com/debian/</span>
</code></pre>
    <p>
     我这里使用的debian11，10的话也试过是ok的
    </p>
    <pre><code class="prism language-c"><span class="token comment">// 网上的命令，licheepi nano 直接debootstrap会有证书的问题</span>
sudo debootstrap <span class="token operator">--</span>foreign <span class="token operator">--</span>verbose <span class="token operator">--</span>arch<span class="token operator">=</span>armel  buster rootfs http<span class="token operator">:</span><span class="token comment">//mirrors.huaweicloud.com/debian/</span>

<span class="token comment">// licheepi nano 解决证书的问题，需要额外执行这一步</span>
wget https<span class="token operator">:</span><span class="token comment">//ftp-master.debian.org/keys/release-10.asc -qO- | gpg --import --no-default-keyring --keyring ./debian-release-10.gpg</span>
debootstrap <span class="token operator">--</span>keyring<span class="token operator">=</span><span class="token punctuation">.</span><span class="token operator">/</span>debian<span class="token operator">-</span>release<span class="token operator">-</span><span class="token number">10.</span>gpg <span class="token operator">--</span>foreign <span class="token operator">--</span>verbose <span class="token operator">--</span>arch<span class="token operator">=</span>armel  buster rootfs https<span class="token operator">:</span><span class="token comment">//mirrors.huaweicloud.com/debian-archive/debian/</span>
</code></pre>
    <p>
     然后在rootfs目录下执行以下命令，挂载一些目录。并把qemu放到rootfs/usr/bin/目录下，一般qemu有执行权限，可以看一下，没有的话chmod一下。
    </p>
    <pre><code class="prism language-c">sudo cp <span class="token operator">/</span>usr<span class="token operator">/</span>bin<span class="token operator">/</span>qemu<span class="token operator">-</span>arm<span class="token operator">-</span><span class="token keyword">static</span> rootfs<span class="token operator">/</span>usr<span class="token operator">/</span>bin<span class="token operator">/</span>
sudo chmod <span class="token operator">+</span>x rootfs<span class="token operator">/</span>usr<span class="token operator">/</span>bin<span class="token operator">/</span>qemu<span class="token operator">-</span>arm<span class="token operator">-</span><span class="token keyword">static</span>  <span class="token comment">// 加权限</span>
mount <span class="token operator">-</span>t proc proc rootfs<span class="token operator">/</span>proc
mount <span class="token operator">-</span>t sysfs sys rootfs<span class="token operator">/</span>sys
mount <span class="token operator">-</span>o bind <span class="token operator">/</span>dev rootfs<span class="token operator">/</span>dev
mount <span class="token operator">-</span>o bind <span class="token operator">/</span>dev<span class="token operator">/</span>pts rootfs<span class="token operator">/</span>dev<span class="token operator">/</span>pts
</code></pre>
    <h3>
     <a id="2_51">
     </a>
     2.构建系统-第二阶段
    </h3>
    <pre><code class="prism language-c"><span class="token comment">// 执行第一句会有很长的时间--第二阶段！！！</span>
sudo LC_ALL<span class="token operator">=</span>C LANGUAGE<span class="token operator">=</span>C LANG<span class="token operator">=</span>C chroot rootfs <span class="token operator">/</span>debootstrap<span class="token operator">/</span>debootstrap <span class="token operator">--</span>second<span class="token operator">-</span>stage <span class="token operator">--</span>verbose
<span class="token comment">// 执行这一句就是改变了根目录</span>
sudo LC_ALL<span class="token operator">=</span>C LANGUAGE<span class="token operator">=</span>C LANG<span class="token operator">=</span>C chroot rootfs
</code></pre>
    <p>
     改变根目录可能不太还理解，看下图可以更好理解：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b8eacc14a6d74ce09302e8209f9d9bba.png">
      <br/>
      注解：为啥要改变根目录，这样你的一些操作就能留在该根文件系统中，apt-get安装一些软件啥啥的。
     </img>
    </p>
    <pre><code class="prism language-c">apt<span class="token operator">-</span>get gcc  		<span class="token comment">//不知道为啥对gcc有一种执念，别的可以没有，这个必须有！</span>
apt<span class="token operator">-</span>get install wpasupplicant <span class="token comment">// 安装WIFI配置相关的组件</span>
apt<span class="token operator">-</span>get install net<span class="token operator">-</span>tools     <span class="token comment">// 安装网络基础组件、如使用ifconfig等</span>
apt<span class="token operator">-</span>get install udhcpc        <span class="token comment">// 当wifi连接成功后，需要用这个组件去获取IP地址</span>

<span class="token comment">// 其他组件</span>
apt<span class="token operator">-</span>get install wireless<span class="token operator">-</span>tools 
apt install sudo vim openssh<span class="token operator">-</span>server htop
apt install pciutils usbutils acpi
</code></pre>
    <p>
     修改root登录密码的方式如下
    </p>
    <pre><code class="prism language-c">passwd root
</code></pre>
    <p>
     添加用户
    </p>
    <pre><code class="prism language-c">groupadd <span class="token operator">&lt;</span>用户组<span class="token operator">&gt;</span>
useradd <span class="token operator">-</span>m <span class="token operator">-</span>g <span class="token operator">&lt;</span>用户组<span class="token operator">&gt;</span> <span class="token operator">-</span>s <span class="token operator">/</span>bin<span class="token operator">/</span>bash <span class="token operator">&lt;</span>用户名<span class="token operator">&gt;</span>
passwd <span class="token operator">&lt;</span>用户名<span class="token operator">&gt;</span>
代码如下（示例）：
</code></pre>
    <p>
     新增账号的操作感觉没必要啊，不加了，详见：
     <a href="https://www.cnblogs.com/twzy/p/15160824.html" rel="nofollow">
      墨云:Debian文件系统制作

    <h3>
     <a id="3_85">
     </a>
     3.打包
    </h3>
    <p>
     当所有的内容制作完成，就可以清理缓存，打包之后就可以替换你的文件系统了
    </p>
    <pre><code class="prism language-c">apt<span class="token operator">-</span>cache clean #删除安装包 
exit  #退出chroot
rm rootfs<span class="token operator">/</span>usr<span class="token operator">/</span>bin<span class="token operator">/</span>qemu<span class="token operator">-</span>arm<span class="token operator">-</span><span class="token keyword">static</span>
</code></pre>
    <p>
     卸载刚在挂载的文件夹。
    </p>
    <pre><code class="prism language-c">cd rootfs
sudo umount   dev<span class="token operator">/</span>pts<span class="token operator">/</span>
sudo umount   dev<span class="token operator">/</span>
sudo umount   sys<span class="token operator">/</span>
sudo umount   proc<span class="token operator">/</span>
sudo umount   dev<span class="token operator">/</span>pts<span class="token operator">/</span>
</code></pre>
    <p>
     打包文件。
    </p>
    <pre><code class="prism language-c">cd rootfs  #进到文件系统目录<span class="token punctuation">,</span>如果已经在该文件夹下，可以忽略
tar cvf <span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token operator">/</span>rootfs<span class="token punctuation">.</span>tar <span class="token punctuation">.</span>    #要注意那个<span class="token punctuation">.</span>  代表当前目录
</code></pre>
    <p>
     生成的rootfs.tar任意解压到文件系统即可
    </p>
    <h3>
     <a id="4_108">
     </a>
     4.遇到的问题
    </h3>
    <p>
     ① request_module: kmod_concurrent_max (0) close to 0 (max_modprobes: 50), for module binfmt-0000, throttling…
     <br/>
     ② Starting init: /sbin/init exists but couldn’t execute it (error -8)
     <br/>
     ③ Starting init: /bin/sh exists but couldn’t execute it (error -8)
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/aea2409f3c1441288574ae84654032cc.png">
      <br/>
      <mark>
       解决
      </mark>
      ：之前的步骤一直按照别人的操作来的试了五六次，排除了操作的问题，最后根文件系统不压缩-解压。直接把rootfs内的所有文件copy到SD卡里，居然解决了，我真的服了！！如果有遇到上述问题的话可以先看看文件的权限是否够，是否在解压的时候改了文件的权限。
      <br/>
      <mark>
       还有一件事
      </mark>
      ：关于Starting init: /sbin/init exists but couldn’t execute it (error -8)问题，有的说要换一种启动方式，以下仅供参考：
     </img>
    </p>
    <pre><code>替换为轻量级 Init 系统
若 systemd 仍失败，改用 sysvinit：
1、在 chroot 环境中操作
sudo chroot rootfs /usr/bin/qemu-arm-static /bin/bash
2、安装 sysvinit
apt-get update
apt-get install sysvinit-core
 3、卸载 systemd
apt-get purge systemd
 4、创建符号链接
ln -sf /sbin/init /sbin/init.sysvinit
修改内核命令行参数：
5、复制
init=/sbin/init.sysvinit
</code></pre>
