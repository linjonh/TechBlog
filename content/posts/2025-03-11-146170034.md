    <h3 id="package-information-and-release-types">
     包信息和发布类型
    </h3>
    <p>
     Microsoft为创建和分发的 SQL Server 的所有软件更新包采用了标准化命名架构。
    </p>
    <p>
     软件更新包是一个可执行文件（
     <em>
      .exe
     </em>
     或
     <em>
      .msi
     </em>
     ）文件，其中包含一个或多个文件，这些文件可能应用于 SQL Server 安装以更正特定问题。 软件更新包由客户支持服务（CSS）分发给计算机受特定问题影响的客户。
    </p>
    <p>
     Microsoft已采用软件更新包的命名架构，原因如下：
    </p>
    <ul>
     <li>
      跨软件更新包创建一致性。
     </li>
     <li>
      更轻松地搜索软件更新包和知识库文章。
     </li>
     <li>
      明确标识软件更新包适用的语言和 SQL Server 版本。
     </li>
    </ul>
    <p>
     在下载时选择的每个软件更新包都包含在一个自提取可执行文件中，该可执行文件有助于轻松安装和部署软件更新包。
    </p>
    <p>
     SQL Server 软件更新包通常分为两种主要版本类型：
    </p>
    <ul>
     <li>
      <p>
       <span style="color:#fe2c24">
        GDR （常规分发版本）
       </span>
       ：GDR 版本是为 SQL Server 支持标识的那些关键修补程序保留的，从而可能会影响广泛的客户群。
      </p>
     </li>
     <li>
      <p>
       修补程序：修补程序版本通常用于修复不影响大型客户群的独立问题;产品处于主流支持阶段。 修补程序在两个主要类型中发布：
      </p>
      <ul>
       <li>
        <p>
         COD（关键按需）或 OD（按需）：COD 或 OD 版本保留给关键客户请求，其中关键业务功能因遇到的问题而受损。 由于请求的性质，这些版本不遵循常规节奏。
        </p>
       </li>
       <li>
        <p>
         CU （累积更新）：CU 版本是非关键请求，可为不影响关键业务功能的隔离问题提供修补程序。 CU 以两个月的节奏发布，而产品和服务包则处于主流支持阶段。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     若要详细了解 ISM 和 SQL Server 服务遵循的不同版本类型，请参阅
     <a href="https://support.microsoft.com/help/935897" rel="nofollow" title="SQL Server 团队提供的增量服务模型，以提供报告问题的">
      SQL Server 团队提供的增量服务模型，以提供报告问题的
     </a>
     修补程序。
    </p>
    <p>

    <h3 id="naming-schema-for-sql-server-software-update-packages">
     SQL Server 软件更新包的命名架构
    </h3>
    <p>
     可以使用以下命名架构轻松识别 SQL Server 软件更新包。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        软件更新包名称架构
       </strong>
      </p>
      <p>
       为了区分联机可用的各种软件更新包，采用以下架构：
       <br/>
       <code>
        &lt;product name or product program name&gt;_&lt;SP number or RTM&gt;_&lt;servicing release&gt;_&lt;KB article number&gt;_&lt;build number optional&gt;_&lt;architecture identifier&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        提取的 SQL Server 文件名架构
       </strong>
      </p>
      <p>
       下载并提取主 SQL Server 软件更新包后，文件名将如下所示：
       <br/>
       <code>
        &lt;product name or component&gt;-&lt;KB article number&gt;-&lt;build number optional&gt;-&lt;version optional&gt;-&lt;architecture Identifier&gt;-&lt;language code optional&gt;.exe
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        提取的功能包名称架构
       </strong>
      </p>
      <p>
       下载并提取功能包的软件更新包后，文件名将如下所示：
      </p>
      <p>
       <em>
        [功能包文件名].msi
       </em>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          ProductName
         </strong>
         这是完整的产品名称，其中包括产品版本信息。 对于 SQL Server，此属性可以是下列属性之一：
        </p>
        <ul>
         <li>
          SQLServer2005
         </li>
         <li>
          SQLServer2008
         </li>
         <li>
          SQLServer2008R2
         </li>
         <li>
          SQLServer2012
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          SP 编号或 RTM
         </strong>
         产品或组件的 Service Pack 级别，可在其顶部应用。
         <span style="color:#fe2c24">
          RTM 指示未安装任何 Service Pack 的产品。
         </span>
        </p>
       </li>
       <li>
        <p>
         <strong>
          知识库文章编号
         </strong>
         Microsoft与软件更新关联的知识库文章编号。
        </p>
       </li>
       <li>
        <p>
         <strong>
          服务发布
         </strong>
         软件更新的发布类型。 有关详细信息，请访问
         <a href="https://learn.microsoft.com/zh-cn/troubleshoot/sql/releases/naming-schema-and-fix-area#package-information-and-release-types" rel="nofollow" title="“包信息和发布类型">
          “包信息和发布类型
         </a>
         ”部分。
        </p>
        <ul>
         <li>
          COD：按需关键
         </li>
         <li>
          OD：按需
         </li>
         <li>
          CU：累积更新后跟累积更新版本号
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          体系结构标识符
         </strong>
         此字段用于指示特定修补程序包运行的处理器体系结构。 当前选项如下：
        </p>
        <ul>
         <li>
          x86：此包在 x86 平台上运行。
         </li>
         <li>
          ia64：此包在 Itanium IA-64 平台上针对 64 位运行。
         </li>
         <li>
          x64：此包仅在 AMD x64 和兼容系统上运行。
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          版本
         </strong>
         ：指示软件版本的可选字段。
        </p>
       </li>
       <li>
        <p>
         内部版本号：用于指示软件更新中包含的 SQL Server 内部版本号的可选字段。
        </p>
       </li>
      </ul>
      <p>
       例如，在
       <em>
        SQL2000-KB840223-8.00.1007-ia64-ENU.exe
       </em>
       中，SQL Server 的生成版本为 8.00.1007。 这将对应于Sqlservr.exe
       <em>
        的文件版本
       </em>
       ，以及针对
       <code>
        @@version run
       </code>
       此服务器实例返回的值。
      </p>
     </li>
    </ul>

    <h3 id="frequently-used-terms-and-acronyms">
     常用术语和首字母缩写词
    </h3>
    <p>
     <strong>
      累积更新 (CU)：
     </strong>
     汇总更新，其中包含迄今为止之前所有的关键按需修补程序。 此外，CU 还包含针对符合修补程序验收条件的问题的修复。 这些条件包括解决方法的可用性、对客户的影响、问题的重复性、必须更改的代码的复杂程度，以及其他主题。
    </p>
    <p>
     <strong>
      修补程序：
     </strong>
     单个累积程序包，其中包含用于解决产品问题的一个或多个文件，这些文件以二进制形式在文件级别累积。 修补程序针对特定客户情形，而且不可以在客户组织外部分发。
    </p>
    <p>
     <strong>
      RTM：
     </strong>
     通常指“发行制造”。 在 SQL Server 等产品环境中，它表示未有 Service Pack 或修补程序应用到产品。
    </p>
    <p>
     <strong>
      RTW：
     </strong>
     通常指“发行到 Web”。 它表示程序包已经发行到 Web 并已提供给客户下载。
    </p>
    <p>
     <strong>
      Service pack：
     </strong>
     一套经过测试的、累积的修补程序、安全更新、关键更新以及其他更新。 Service Pack 可能还包含自产品发布后在内部发现的问题的其他修复，以及有限数量的客户请求的设计更改或功能。
    </p>
