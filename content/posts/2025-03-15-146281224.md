    <h2 id="%E5%89%8D%E8%A8%80" name="%E5%89%8D%E8%A8%80" style="margin-left:0px; margin-right:0px; text-align:center">
     前言
    </h2>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     书接上回
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <a href="https://blog.csdn.net/qq_73427280/article/details/146280389?spm=1001.2014.3001.5502" title="SpringBoot项目部署到宝塔面板的详细过程-CSDN博客">
      SpringBoot项目部署到宝塔面板的详细过程-CSDN博客
     </a>
    </p>
    <p style="background-color:transparent; margin-left:0px; margin-right:0px">
     本次以SmartAdmin的项目为例，通过宝塔面板部署，对部署过程进行一个完整演示，不同项目的部署也可能略有不同。过程仅为参考。
    </p>
    <p style="margin-left:0; margin-right:0">
     准备工具：可正常登录宝塔面板、Vue项目（本文以SmartAdmin的开源项目为例）、WebStorm 2024.2.2（或其他编译器）
    </p>
    <p style="margin-left:0; margin-right:0">
     Vue项目的部署与前端项目部署类似，不过省去了数据库的一步，主要有两个步骤，一是项目的打包与上传，打包之前需要进行必要的修改，如修改连接的后端的地址，修改baseUrl等等；二是在服务器启动项目
    </p>
    <h2 id="%E4%B8%80%E3%80%81%E9%A1%B9%E7%9B%AE%E7%9A%84%E6%89%93%E5%8C%85%E4%B8%8E%E4%B8%8A%E4%BC%A0" name="%E4%B8%80%E3%80%81%E9%A1%B9%E7%9B%AE%E7%9A%84%E6%89%93%E5%8C%85%E4%B8%8E%E4%B8%8A%E4%BC%A0" style="background-color:transparent; margin-left:0px; margin-right:0px">
     一、项目的打包与上传
    </h2>
    <h3 id="1%E3%80%81%E4%BF%AE%E6%94%B9%E5%89%8D%E7%AB%AF%E9%A1%B9%E7%9B%AE" name="1%E3%80%81%E4%BF%AE%E6%94%B9%E5%89%8D%E7%AB%AF%E9%A1%B9%E7%9B%AE" style="margin-left:0px; margin-right:0px">
     <a name="_Toc192771292">
      <span style="color:#0d0016">
       1
      </span>
     </a>
     <span style="color:#0d0016">
      、修改前端项目
     </span>
    </h3>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     这里主要修改前端链接后端的地址，对于.env.test文件，只有url需要修改，这里修改为后端地址，这个地址是刚才后端项目设置的域名。
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="231" src="https://i-blog.csdnimg.cn/direct/f12e70070e0e4dbfa2890ba37e3fc30f.png" width="790"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     如果你的项目不适用.env文件，可以找一找项目中配置的前端链接后端地址的位置，修改一下即可。
    </p>
    <h3 id="2%E3%80%81%E5%85%B3%E4%BA%8Etest%E7%8E%AF%E5%A2%83%E7%9A%84%E8%A1%A5%E5%85%85%E4%BF%AE%E6%94%B9" name="2%E3%80%81%E5%85%B3%E4%BA%8Etest%E7%8E%AF%E5%A2%83%E7%9A%84%E8%A1%A5%E5%85%85%E4%BF%AE%E6%94%B9" style="margin-left:0px; margin-right:0px">
     <a name="_Toc192771293">
      <span style="color:#0d0016">
       2
      </span>
     </a>
     <span style="color:#0d0016">
      、关
     </span>
     于test环境的补充修改
    </h3>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     这里主要检查baseUrl，我们期望的baseUrl为“/”。
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     检查根目录下的vite.config.js，确保base的值为“/”，项目默认给base赋值的三元运算符的值恒为“/”。
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="402" src="https://i-blog.csdnimg.cn/direct/f2b47e990ddf4712a488a7072f84a380.png" width="781"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     检查package.js文件，发现build：test脚本执行时，重新设置了base的值，启动脚本时添加的指令会覆盖掉vite里面的配置，这可能导致前端项目部署后错误访问到了/admin，这里为了方便这里把base的设置移除，即删除“—base=/admin”，使用vite中的配置即可。
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="397" src="https://i-blog.csdnimg.cn/direct/1ea2b0c83c744710b60fb1d235987c24.png" width="783"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     上述为此项目test环境特有，大家可以按需修改。
    </p>
    <h3 id="3%E3%80%81%E6%89%93%E5%8C%85%E5%89%8D%E7%AB%AF%E9%A1%B9%E7%9B%AE" name="3%E3%80%81%E6%89%93%E5%8C%85%E5%89%8D%E7%AB%AF%E9%A1%B9%E7%9B%AE" style="background-color:transparent; margin-left:0px; margin-right:0px">
     <a name="_Toc192771294">
      <span style="color:#0d0016">
       3
      </span>
     </a>
     <span style="color:#0d0016">
      、打包前端项目
     </span>
    </h3>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     SmartAdmin官方给了以下指令进行项目打包，在黑框执行如下代码之一
    </p>
    <blockquote>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      npm run build:prod
     </p>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      npm run build:test
     </p>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      npm run build:pre
     </p>
    </blockquote>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     分别对应三种不同的环境，这里选择执行
    </p>
    <blockquote>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      npm run build:test
     </p>
    </blockquote>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     打包完成之后，根目录下会生成一个dist文件夹。
    </p>
    <p class="img-center">
     <img alt="" height="366" src="https://i-blog.csdnimg.cn/direct/f75fd7574ef9444b84d15822eaf831d0.png" width="397"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     打开宝塔面板将此文件夹上传，这里上传的目录是/home/smart_admin_v3/dist。
    </p>
    <p class="img-center">
     <img alt="" height="451" src="https://i-blog.csdnimg.cn/direct/ef4e31e5ddc54805b0e837caeca86fb7.png" width="572"/>
    </p>
    <p style="margin-left:0; margin-right:0">
     如果你的项目没有那么复杂，package.js没有设置那么多脚本的话，可以修改完后端连接地址之后，执行
    </p>
    <blockquote>
     <p style="margin-left:0; margin-right:0">
      npm run build
     </p>
    </blockquote>
    <p style="margin-left:0; margin-right:0">
     即可
    </p>
    <h2 id="%E4%BA%8C%E3%80%81%E6%B7%BB%E5%8A%A0%E7%AB%99%E7%82%B9%EF%BC%8C%E5%90%AF%E5%8A%A8%E9%A1%B9%E7%9B%AE" name="%E4%BA%8C%E3%80%81%E6%B7%BB%E5%8A%A0%E7%AB%99%E7%82%B9%EF%BC%8C%E5%90%AF%E5%8A%A8%E9%A1%B9%E7%9B%AE" style="background-color:transparent; margin-left:0px; margin-right:0px">
     二、添加站点，启动项目
    </h2>
    <p>
     点击网站-&gt;php项目-&gt;添加站点。
    </p>
    <p style="margin-left:0; margin-right:0">
     <img alt="" height="676" src="https://i-blog.csdnimg.cn/direct/9158220b1bde479b8cd6043fafd36998.png" width="1087"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     将目录设置为刚才上传的目录，并设置域名。注意，域名需要在腾讯云解析之后，才可以正常访问！
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="768" src="https://i-blog.csdnimg.cn/direct/54f095959deb4dcd8a252c1b5efcb0c2.png" width="1084"/>
    </p>
    <h2 id="%E4%B8%89%E3%80%81%E6%80%BB%E7%BB%93" name="%E4%B8%89%E3%80%81%E6%80%BB%E7%BB%93" style="background-color:transparent; margin-left:0px; margin-right:0px; text-align:justify">
     三、总结
    </h2>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     前端部署需要在后端部署完成之后，因为需要后端地址。主要流程是修改连接后端服务器的地址，打包上传项目，启动站点。
    </p>
