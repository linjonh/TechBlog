    <h3 id="rmdir%E4%B8%8Erm%EF%BC%9A%E6%B8%85%E7%90%86%E8%BF%B7%E5%AE%AB%E7%9A%84%E7%A2%8E%E7%9F%B3%E4%B8%8E%E9%99%B7%E9%98%B1%E2%80%94%E2%80%94%20%E6%89%AB%E9%99%A4%E5%BA%9F%E5%BC%83%E8%B7%AF%E5%BE%84%E7%9A%84%E7%88%86%E7%A0%B4%E6%9C%AF" name="rmdir%E4%B8%8Erm%EF%BC%9A%E6%B8%85%E7%90%86%E8%BF%B7%E5%AE%AB%E7%9A%84%E7%A2%8E%E7%9F%B3%E4%B8%8E%E9%99%B7%E9%98%B1%E2%80%94%E2%80%94%20%E6%89%AB%E9%99%A4%E5%BA%9F%E5%BC%83%E8%B7%AF%E5%BE%84%E7%9A%84%E7%88%86%E7%A0%B4%E6%9C%AF" style="background-color:transparent">
     <strong>
      rmdir与rm：清理迷宫的碎石与陷阱
     </strong>
     <br/>
     —— 扫除废弃路径的爆破术
    </h3>
    <blockquote>
     <p>
      <strong>
       <span style="color:#0d0016">
        rmdir是一个与mkdir相对应的命令。mkdir是建立目录，而rmdir是删除命令。
       </span>
      </strong>
      <br/>
      <span style="color:#0d0016">
       <strong>
        语法：
       </strong>
      </span>
      rmdir [-p][dirName]
      <br/>
      <span style="color:#0d0016">
       <strong>
        适用对象：
       </strong>
      </span>
      具有当前目录操作权限的所有使用者
      <br/>
      <span style="color:#0d0016">
       <strong>
        功能：
       </strong>
      </span>
      <strong>
       <span style="color:#fe2c24">
        删除空目录
       </span>
      </strong>
      <br/>
      <strong>
       <span style="color:#0d0016">
        常用选项：
       </span>
      </strong>
      <br/>
      -p 当子目录被删除后如果父目录也变成空目录的话，就连带父目录一起删除
     </p>
    </blockquote>
    <p>
     <strong>
      &gt; rmdir：
      <span style="color:#0d0016">
       只能删除空目录
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="427" src="https://i-blog.csdnimg.cn/direct/26873aed4029479aac55473b5355c4f1.png" width="482"/>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       无法删除非目录的文件且也无法
      </strong>
     </span>
     <span style="color:#0d0016">
      <strong>
       删除非空目录
      </strong>
     </span>
    </p>
    <p>
     <img alt="" height="329" src="https://i-blog.csdnimg.cn/direct/48d163a0e16041da84b39bcd3c8d9801.png" width="543"/>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       rmdir的使用非常有限，所以我们几乎不会使用 rmdir 来删除文件
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       我们删除文件使用最多的是下面这条指令：rm
      </strong>
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       <strong>
        rm命令可以同时删除文件或目录
        <br/>
        语法：
       </strong>
      </span>
      rm [-f-i-r-v][dirName/dir]
      <br/>
      <strong>
       <span style="color:#0d0016">
        适用对象：
       </span>
      </strong>
      所有使用者
      <br/>
      <span style="color:#0d0016">
       <strong>
        功能：
       </strong>
      </span>
      删除文件或目录
     </p>
     <p>
      <span style="color:#0d0016">
       <strong>
        常用选项：
       </strong>
      </span>
      <br/>
      -f 即使文件属性为只读(即写保护)，亦直接删除
      <br/>
      -i 删除前逐一询问确认
      <br/>
      -r 删除目录及其下所有文件
     </p>
    </blockquote>
    <p>
     <img alt="" height="423" src="https://i-blog.csdnimg.cn/direct/70d34f3e8f584b30a1ea6c532e29d1ba.png" width="622"/>
    </p>
    <h3 id="man%EF%BC%9A%E8%BF%B7%E5%AE%AB%E6%8E%A2%E9%99%A9%E7%9A%84%E5%8F%A4%E8%80%81%E6%8C%87%E5%8D%97%E4%B9%A6%E2%80%94%E2%80%94%20%E7%94%A8%E5%92%92%E8%AF%AD%E8%A7%A3%E9%94%81%E6%8C%87%E4%BB%A4%E7%9A%84%E9%9A%90%E8%97%8F%E8%AF%B4%E6%98%8E" name="man%EF%BC%9A%E8%BF%B7%E5%AE%AB%E6%8E%A2%E9%99%A9%E7%9A%84%E5%8F%A4%E8%80%81%E6%8C%87%E5%8D%97%E4%B9%A6%E2%80%94%E2%80%94%20%E7%94%A8%E5%92%92%E8%AF%AD%E8%A7%A3%E9%94%81%E6%8C%87%E4%BB%A4%E7%9A%84%E9%9A%90%E8%97%8F%E8%AF%B4%E6%98%8E">
     <strong>
      man：迷宫探险的古老指南书
     </strong>
     <br/>
     —— 用咒语解锁指令的隐藏说明
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       <code>
        man
       </code>
       命令常用选项
      </strong>
     </span>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        -k
       </code>
      </strong>
      ：此选项用于根据关键字搜索联机帮助。当你只记得某个功能相关的关键字，却不明确具体命令时，可使用该选项进行搜索。
     </li>
     <li>
      <strong>
       <code>
        num
       </code>
      </strong>
      ：该选项可限定搜索范围，仅在第
      <code>
       num
      </code>
      章节中查找指定命令的手册信息。
     </li>
     <li>
      <strong>
       <code>
        -a
       </code>
      </strong>
      ：使用此选项时，
      <code>
       man
      </code>
      命令会显示所有章节中与指定命令相关的手册内容。例如，在不使用
      <code>
       -a
      </code>
      选项执行
      <code>
       man printf
      </code>
      时，命令会从第一章开始搜索，找到相关内容后即停止；而使用
      <code>
       -a
      </code>
      选项时，当你按下
      <code>
       q
      </code>
      退出当前显示的手册内容后，
      <code>
       man
      </code>
      命令会继续在后续章节中搜索，直至所有章节搜索完毕。
     </li>
    </ul>
    <p>
     <span style="color:#0d0016">
      <strong>
       Linux 手册章节划分
      </strong>
     </span>
    </p>
    <p>
     Linux 手册总共分为 8 个章节，每个章节涵盖不同类型的内容，具体如下：
    </p>
    <ol>
     <li>
      <strong>
       普通命令（章节 1）
      </strong>
      ：该章节包含日常使用的普通命令，例如文件操作、进程管理等常见命令的详细说明。
     </li>
     <li>
      <strong>
       系统调用（章节 2）
      </strong>
      ：此章节主要介绍系统调用相关内容，像
      <code>
       open
      </code>
      、
      <code>
       write
      </code>
      这类系统调用函数都在此章节有详细说明。通过查阅该章节内容，你能方便地了解调用这些函数时所需包含的头文件。
     </li>
     <li>
      <strong>
       库函数（章节 3）
      </strong>
      ：包含各种库函数的信息，例如
      <code>
       printf
      </code>
      、
      <code>
       fread
      </code>
      等。这些库函数为开发者提供了丰富的编程接口。
     </li>
     <li>
      <strong>
       特殊文件（章节 4）
      </strong>
      ：主要针对特殊文件进行说明，即
      <code>
       /dev
      </code>
      目录下的各种设备文件。该章节会介绍这些设备文件的用途和使用方法。
     </li>
     <li>
      <strong>
       文件格式（章节 5）
      </strong>
      ：详细解释文件的格式，例如
      <code>
       /etc/passwd
      </code>
      文件。在此章节中，会对文件中各个字段的含义和作用进行说明。
     </li>
     <li>
      <strong>
       游戏相关（章节 6）
      </strong>
      ：该章节是为游戏预留的，游戏开发者可根据自身需求自定义相关内容。
     </li>
     <li>
      <strong>
       附件与变量（章节 7）
      </strong>
      ：包含一些附件信息以及变量的说明，例如全局变量
      <code>
       environ
      </code>
      就在此章节有详细解释。
     </li>
     <li>
      <strong>
       系统管理命令（章节 8）
      </strong>
      ：此章节中的命令主要用于系统管理，且这些命令通常只有
      <code>
       root
      </code>
      用户才能使用，如
      <code>
       ifconfig
      </code>
      命令，用于配置网络接口。
     </li>
    </ol>

    <h3 id="cp%EF%BC%9A%E5%A4%8D%E5%88%B6%E8%BF%B7%E5%AE%AB%E5%9C%B0%E5%9B%BE%E7%9A%84%E5%BD%B1%E5%88%86%E8%BA%AB%E6%9C%AF%E2%80%94%E2%80%94%20%E4%BB%8E%E4%B8%80%E6%89%87%E9%97%A8%E5%88%B0%E5%8F%A6%E4%B8%80%E6%89%87%E9%97%A8%E7%9A%84%E9%95%9C%E5%83%8F%E9%AD%94%E6%B3%95" name="cp%EF%BC%9A%E5%A4%8D%E5%88%B6%E8%BF%B7%E5%AE%AB%E5%9C%B0%E5%9B%BE%E7%9A%84%E5%BD%B1%E5%88%86%E8%BA%AB%E6%9C%AF%E2%80%94%E2%80%94%20%E4%BB%8E%E4%B8%80%E6%89%87%E9%97%A8%E5%88%B0%E5%8F%A6%E4%B8%80%E6%89%87%E9%97%A8%E7%9A%84%E9%95%9C%E5%83%8F%E9%AD%94%E6%B3%95">
     <strong>
      cp：复制迷宫地图的影分身术
     </strong>
     <br/>
     —— 从一扇门到另一扇门的镜像魔法
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       <code>
        cp [选项] 源文件或目录 目标文件或目录
       </code>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       功能概述
      </strong>
     </span>
    </p>
    <p>
     <code>
      cp
     </code>
     指令主要用于复制文件或目录。当同时指定两个及以上的文件或目录，并且最后指定的目的地是一个已存在的目录时，
     <code>
      cp
     </code>
     命令会将前面指定的所有文件或目录复制到该目录中。
    </p>
    <p>
     若同时指定多个文件或目录，而最后的目的地并非一个已存在的目录，
     <code>
      cp
     </code>
     命令将输出错误信息。
    </p>
    <p>
     <img alt="" height="54" src="https://i-blog.csdnimg.cn/direct/e629455a3a914cb6b3c34d3df16927eb.png" width="479"/>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       常用选项说明
      </span>
     </strong>
    </p>
    <p>
     <span style="color:#0d0016">
      强制复制选项
     </span>
    </p>
    <ul>
     <li>
      <code>
       -f
      </code>
      或
      <code>
       --force
      </code>
      ：该选项用于强行复制文件或目录，无论目标位置的文件或目录是否已经存在。使用此选项时，若目标位置已存在同名文件或目录，将直接进行覆盖操作，不会有额外的提示。
     </li>
    </ul>
    <p>
     交互确认选项
    </p>
    <ul>
     <li>
      <code>
       -i
      </code>
      或
      <code>
       --interactive
      </code>
      ：在执行覆盖文件操作之前，
      <code>
       cp
      </code>
      命令会先向用户发起询问，请求用户确认是否要进行覆盖。这有助于避免误操作导致重要文件被意外覆盖。
     </li>
    </ul>
    <p>
     递归复制选项
    </p>
    <ul>
     <li>
      <code>
       -r
      </code>
      ：该选项会进行递归处理，将指定目录下的所有文件与子目录一并进行复制操作。如果源文件或目录的类型既不是目录也不是符号链接，那么会将其视为普通文件进行处理。
     </li>
     <li>
      <code>
       -R
      </code>
      或
      <code>
       --recursive
      </code>
      ：与
      <code>
       -r
      </code>
      选项功能相同，同样是递归处理指定目录下的文件及子目录，实现对整个目录结构的复制。
     </li>
    </ul>
    <p>
     <strong>
      <em>
       如果指定目录存在，则会把普通文件或目录直接拷贝到目录里面：
      </em>
     </strong>
    </p>
    <p>
     <img alt="" height="705" src="https://i-blog.csdnimg.cn/direct/1590bcc50c8d4e89a4966610db5ad18b.png" width="1406"/>
    </p>
    <p>
     <em>
      <strong>
       如果把普通文件拷贝到已有的目标普通文件当中，则原文件会把目标文件覆盖：
      </strong>
     </em>

    <p>
     <img alt="" height="3481" src="https://i-blog.csdnimg.cn/direct/ce168af9a9ff4ed8826a674a94d90cb2.png" width="1210"/>
    </p>
    <p>
     <em>
      <strong>
       如果指定文件或目录不存在，则在当前路径自动创建一个目录或文件：
      </strong>
     </em>
    </p>
    <p>
     <img alt="" height="540" src="https://i-blog.csdnimg.cn/direct/7b2a068259574b0294ac728ac0b3b7b7.png" width="1315"/>
    </p>
    <p>
     <em>
      <strong>
       对已经存在的目录进行 cp 时需要注意指定文件目录是否为空：
      </strong>
     </em>
    </p>
    <p>
     <img alt="" height="612" src="https://i-blog.csdnimg.cn/direct/e2f7d8ec7085425db17e000007cf3b65.png" width="510"/>

    <h3 id="mv%EF%BC%9A%E7%A7%BB%E5%8A%A8%E5%AE%9D%E8%97%8F%E7%9A%84%E9%9A%90%E5%BD%A2%E6%90%AC%E8%BF%90%E5%B7%A5%E2%80%94%E2%80%94%20%E8%B7%AF%E5%BE%84%E9%87%8D%E5%86%99%E4%B8%8E%E5%91%BD%E5%90%8D%E7%9A%84%E7%A9%BA%E9%97%B4%E6%88%8F%E6%B3%95" name="mv%EF%BC%9A%E7%A7%BB%E5%8A%A8%E5%AE%9D%E8%97%8F%E7%9A%84%E9%9A%90%E5%BD%A2%E6%90%AC%E8%BF%90%E5%B7%A5%E2%80%94%E2%80%94%20%E8%B7%AF%E5%BE%84%E9%87%8D%E5%86%99%E4%B8%8E%E5%91%BD%E5%90%8D%E7%9A%84%E7%A9%BA%E9%97%B4%E6%88%8F%E6%B3%95" style="background-color:transparent">
     <strong>
      mv：移动宝藏的隐形搬运工
     </strong>
     <br/>
     —— 路径重写与命名的空间戏法
    </h3>
    <p>
     mv命令是move的缩写，可以用来
     <strong>
      移动文件
     </strong>
     或者
     <strong>
      将文件改名
     </strong>
     （move (rename) ﬁles），是
     <a href="https://so.csdn.net/so/search?q=Linux%E7%B3%BB%E7%BB%9F&amp;spm=1001.2101.3001.7020" title="Linux系统">
      Linux系统
     </a>
     下常用的命令，经常用来
     <strong>
      备份文件或者目录。
     </strong>
    </p>
    <p>
     <strong>
      语法:
     </strong>
    </p>
    <p>
     <em>
      mv [选项] 源文件或目录 目标文件或目录
     </em>
    </p>
    <p>
     <strong>
      功能:
     </strong>
    </p>
    <p>
     <em>
      1. 视mv命令中第二个参数类型的不同（是
      <strong>
       目标文件
      </strong>
      还是
      <strong>
       目标目录
      </strong>
      ），mv命令
      <span style="color:#0d0016">
       <strong>
        将文件重命名或将其移至一个新的目录中。
       </strong>
      </span>
     </em>
    </p>
    <p>
     <em>
      2. 当第二个参数类型是
      <strong>
       文件时
      </strong>
      ，mv命令完成
      <strong>
       文件重命名
      </strong>

     </em>

      </strong>

    <h3 id="%E9%87%8D%E5%AE%9A%E5%90%91%EF%BC%9A%E6%95%B0%E6%8D%AE%E6%B2%B3%E6%B5%81%E7%9A%84%E9%97%B8%E9%97%A8%E6%8E%A7%E5%88%B6%E5%99%A8%E2%80%94%E2%80%94%20%E5%B0%86%E4%BF%A1%E6%81%AF%E6%B4%AA%E6%B5%81%E5%AF%BC%E5%85%A5%E6%8C%87%E5%AE%9A%E5%AE%B9%E5%99%A8" name="%E9%87%8D%E5%AE%9A%E5%90%91%EF%BC%9A%E6%95%B0%E6%8D%AE%E6%B2%B3%E6%B5%81%E7%9A%84%E9%97%B8%E9%97%A8%E6%8E%A7%E5%88%B6%E5%99%A8%E2%80%94%E2%80%94%20%E5%B0%86%E4%BF%A1%E6%81%AF%E6%B4%AA%E6%B5%81%E5%AF%BC%E5%85%A5%E6%8C%87%E5%AE%9A%E5%AE%B9%E5%99%A8">
     <strong>
      重定向：数据河流的闸门控制器
     </strong>
     <br/>
     —— 将信息洪流导入指定容器
    </h3>
    <p>
     在Linux中，一切皆文件！我们在C或C++中使用的printf、scanf和cin、cout其本质就是文件操作。只不过我们在启动程序时，系统自动帮我们打开了键盘文件和显示器文件！从语言的角度来说，我们用printf是在屏幕上打印内容，但其本质是：
     <strong>
      <span style="color:#0d0016">
       我们把内容写入到显示器文件当中
      </span>
     </strong>
     ！
    </p>
    <p>
     <strong>
      输出重定向（&gt;）：
     </strong>
    </p>
    <p>
     <img alt="" height="351" src="https://i-blog.csdnimg.cn/direct/45d8a8cfe420493ab8f0f0733f6d82f8.png" width="1245"/>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       当我们重定向的文件不存在时，系统会自动帮我们生成一个文件！所以我们也可以直接用 &gt; 来创建一个普通的文件！
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       当我们重定向的文件存在时，如果文件中原来有内容，重定向后，原来的内容就会被我们写入的内容覆盖！如果我们什么都不写入，则会清空文件！
      </span>
     </strong>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       追加重定向：（&gt;&gt;）
      </strong>
     </span>
    </p>
    <p>
     如果我们不期望原本的内容被覆盖，那我们可以使用追加重定向（&gt;&gt;）向指定文件中追加内容！
    </p>
    <p>
     <img alt="" height="187" src="https://i-blog.csdnimg.cn/direct/569161ba23f747f0bc132609c86ef377.png" width="1008"/>
    </p>
    <p>
     <strong>
      输入重定向（&lt;）：
     </strong>
    </p>
    <p>
     <em>
      当我们使用cat指令想要获取一个文件当中的内容时，如果我们不指定文件，则默认到键盘文件当中获取（读取）内容，如果我们使用（&lt;）输入重定向，那我们则在指定文件当中获取（读取）内容！
     </em>
     <img alt="" height="346" src="https://i-blog.csdnimg.cn/direct/e5ee6cf3c32b496ba3ab61da3d4cd0c0.png" width="713"/>
    </p>

     </strong>

     </em>
    </p>

     <strong>
      cal：解开时间的迷宫卷轴
     </strong>
     <br/>
     —— 用数字编织月历的经纬网
    </h3>
    <p>
     cal命令可以用来显示公历（阳历）日历。公历是现在国际通用的历法，又称格列历，通称阳历。“阳历”又名“太阳 历”，系以地球绕行太阳一周为一年，为西方各国所通用，故又名“西历”。
    </p>
    <p>
     <strong>
      命令格式：
     </strong>
     cal [参数][月份][年份]
    </p>
    <p>
     <strong>
      功能：
     </strong>
     用于查看日历等时间信息，如只有一个参数，则表示年份(1-9999)，如有两个参数，则表示月份和年份
    </p>

    <h3 id="find%EF%BC%9A%E5%9C%B0%E6%AF%AF%E5%BC%8F%E6%90%9C%E7%B4%A2%E7%9A%84%E5%AF%BB%E5%AE%9D%E7%BD%97%E7%9B%98%E2%80%94%E2%80%94%20%E4%BB%8E%E4%BA%BF%E4%B8%87%E6%96%87%E4%BB%B6%E4%B8%AD%E5%AE%9A%E4%BD%8D%E7%9B%AE%E6%A0%87%E5%9D%90%E6%A0%87" name="find%EF%BC%9A%E5%9C%B0%E6%AF%AF%E5%BC%8F%E6%90%9C%E7%B4%A2%E7%9A%84%E5%AF%BB%E5%AE%9D%E7%BD%97%E7%9B%98%E2%80%94%E2%80%94%20%E4%BB%8E%E4%BA%BF%E4%B8%87%E6%96%87%E4%BB%B6%E4%B8%AD%E5%AE%9A%E4%BD%8D%E7%9B%AE%E6%A0%87%E5%9D%90%E6%A0%87">
     <strong>
      find：地毯式搜索的寻宝罗盘
     </strong>
     <br/>
     —— 从亿万文件中定位目标坐标
    </h3>
    <p>
     在 Linux 系统中，
     <code>
      find
     </code>
     命令用于在目录结构中搜索文件，并可对搜索到的文件执行指定操作。该命令提供了丰富多样的查找条件，功能极为强大。鉴于其强大的功能特性，
     <code>
      find
     </code>
     命令具备众多选项，其中大部分选项都值得深入探究。即便系统中包含网络文件系统（NFS） ，只要拥有相应权限，
     <code>
      find
     </code>
     命令在该文件系统中同样能够正常且有效地运行。当执行一个资源消耗较大的
     <code>
      find
     </code>
     命令时，由于遍历大型文件系统（一般指 30GB 字节以上的文件系统）可能会耗费较长时间，因此很多人倾向于将其置于后台运行。
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       1. 语法格式
      </strong>
     </span>
    </p>
    <pre><code class="language-bash">find pathname -options</code></pre>
    <p>
     其中，
     <code>
      pathname
     </code>
     表示要查找的目录路径；
     <code>
      -options
     </code>
     代表各种查找条件及操作选项。
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       2. 功能概述
      </strong>
     </span>
    </p>
    <p>
     <code>
      find
     </code>
     命令主要用于在文件树结构中查找符合特定条件的文件，并能针对查找到的文件做出相应处理，这一过程可能涉及磁盘访问操作。
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       3. 常用选项：-name
      </strong>
     </span>
    </p>
    <p>
     <code>
      -name
     </code>
     选项用于按照文件名查找文件。例如，如果要在当前目录及其子目录中查找名为 “example.txt” 的文件，可使用以下命令：
    </p>
    <pre><code class="language-bash">find. -name "example.txt"</code></pre>
