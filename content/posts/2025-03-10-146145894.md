    <h3>
     <a id="Git__2">
     </a>
     <strong>
      一、Git 的优点
     </strong>
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        分布式版本控制
       </strong>
      </p>
      <ul>
       <li>
        每个开发者都拥有完整的仓库副本，无需依赖中央服务器（如 SVN）。
       </li>
       <li>
        支持离线操作（提交、查看历史、创建分支等）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        高效的分支管理
       </strong>
      </p>
      <ul>
       <li>
        创建和切换分支速度快（几乎是瞬间完成）。
       </li>
       <li>
        适合多人协作开发，支持多种工作流（如 Git Flow、GitHub Flow）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        强大的撤销和恢复能力
       </strong>
      </p>
      <ul>
       <li>
        可以回退到任意历史版本，避免代码丢失。
       </li>
       <li>
        提供
        <code>
         git reset
        </code>
        、
        <code>
         git revert
        </code>
        、
        <code>
         git stash
        </code>
        等撤销工具。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数据完整性
       </strong>
      </p>
      <ul>
       <li>
        所有文件通过 SHA-1 哈希算法校验，确保内容不被篡改。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适合多人协作
       </strong>
      </p>
      <ul>
       <li>
        通过
        <code>
         pull request
        </code>
        、
        <code>
         merge
        </code>
        、
        <code>
         rebase
        </code>
        实现代码审查和集成。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="Git__23">
     </a>
     <strong>
      二、Git 详细用法与代码示例
     </strong>
    </h3>
    <h4>
     <a id="1__24">
     </a>
     <strong>
      1. 安装与配置
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        安装 Git
       </strong>
       <br/>
       下载地址：https://git-scm.com/
      </p>
     </li>
     <li>
      <p>
       <strong>
        配置用户信息
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"Your Name"</span>
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"your.email@example.com"</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        查看配置
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--list</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2__41">
     </a>
     <strong>
      2. 基础命令
     </strong>
    </h4>
    <h5>
     <a id="1__42">
     </a>
     <strong>
      (1) 初始化仓库
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">mkdir</span> my-project
<span class="token builtin class-name">cd</span> my-project
<span class="token function">git</span> init  <span class="token comment"># 初始化 Git 仓库</span>
</code></pre>
    <h5>
     <a id="2__49">
     </a>
     <strong>
      (2) 添加文件到暂存区
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> file.txt        <span class="token comment"># 添加单个文件</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>               <span class="token comment"># 添加所有修改的文件</span>
<span class="token function">git</span> <span class="token function">add</span> *.js            <span class="token comment"># 添加所有 .js 文件</span>
</code></pre>
    <h5>
     <a id="3__56">
     </a>
     <strong>
      (3) 提交到本地仓库
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Initial commit: add project structure"</span>
</code></pre>
    <h5>
     <a id="4__61">
     </a>
     <strong>
      (4) 查看状态与历史
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> status              <span class="token comment"># 查看工作区状态</span>
<span class="token function">git</span> log                 <span class="token comment"># 查看提交历史</span>
<span class="token function">git</span> log <span class="token parameter variable">--oneline</span>       <span class="token comment"># 简洁模式查看历史</span>
<span class="token function">git</span> <span class="token function">diff</span>                <span class="token comment"># 查看未暂存的修改</span>
</code></pre>
    <hr/>
    <h4>
     <a id="3__71">
     </a>
     <strong>
      3. 分支管理
     </strong>
    </h4>
    <h5>
     <a id="1__72">
     </a>
     <strong>
      (1) 创建与切换分支
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> branch feature-login    <span class="token comment"># 创建分支</span>
<span class="token function">git</span> checkout feature-login  <span class="token comment"># 切换分支</span>
<span class="token function">git</span> checkout <span class="token parameter variable">-b</span> feature-login  <span class="token comment"># 创建并切换（推荐）</span>
</code></pre>
    <h5>
     <a id="2__79">
     </a>
     <strong>
      (2) 合并分支
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout main          <span class="token comment"># 切换到主分支</span>
<span class="token function">git</span> merge feature-login    <span class="token comment"># 合并 feature-login 到 main</span>
</code></pre>
    <h5>
     <a id="3__85">
     </a>
     <strong>
      (3) 解决冲突
     </strong>
    </h5>
    <p>
     如果合并时发生冲突，手动修改冲突文件后：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> resolved-file.txt
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Resolve merge conflict"</span>
</code></pre>
    <h5>
     <a id="4__92">
     </a>
     <strong>
      (4) 删除分支
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> branch <span class="token parameter variable">-d</span> feature-login  <span class="token comment"># 删除本地分支</span>
<span class="token function">git</span> push origin <span class="token parameter variable">--delete</span> feature-login  <span class="token comment"># 删除远程分支</span>
</code></pre>
    <hr/>
    <h4>
     <a id="4_GitHubGitLab_100">
     </a>
     <strong>
      4. 远程仓库（GitHub/GitLab）
     </strong>
    </h4>
    <h5>
     <a id="1__101">
     </a>
     <strong>
      (1) 添加远程仓库
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token function">add</span> origin https://github.com/yourname/repo.git
</code></pre>
    <h5>
     <a id="2__106">
     </a>
     <strong>
      (2) 推送代码到远程
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> push <span class="token parameter variable">-u</span> origin main  <span class="token comment"># 首次推送</span>
<span class="token function">git</span> push                 <span class="token comment"># 后续推送</span>
</code></pre>
    <h5>
     <a id="3__112">
     </a>
     <strong>
      (3) 拉取代码
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> pull origin main     <span class="token comment"># 拉取并合并远程代码</span>
<span class="token function">git</span> fetch origin         <span class="token comment"># 仅拉取不合并</span>
</code></pre>
    <h5>
     <a id="4__118">
     </a>
     <strong>
      (4) 克隆仓库
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/yourname/repo.git
</code></pre>
    <hr/>
    <h4>
     <a id="5__125">
     </a>
     <strong>
      5. 高级操作
     </strong>
    </h4>
    <h5>
     <a id="1__126">
     </a>
     <strong>
      (1) 撤销修改
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout -- file.txt  <span class="token comment"># 丢弃工作区的修改</span>
<span class="token function">git</span> reset HEAD file.txt   <span class="token comment"># 从暂存区移除文件</span>
<span class="token function">git</span> reset <span class="token parameter variable">--hard</span> HEAD^    <span class="token comment"># 回退到上一个提交（慎用！会丢失修改）</span>
</code></pre>
    <h5>
     <a id="2__133">
     </a>
     <strong>
      (2) 暂存未完成的工作
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> stash          <span class="token comment"># 保存当前修改到暂存区</span>
<span class="token function">git</span> stash pop      <span class="token comment"># 恢复最近一次暂存的修改</span>
</code></pre>
    <h5>
     <a id="3__139">
     </a>
     <strong>
      (3) 重写提交历史
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">git</span> commit <span class="token parameter variable">--amend</span> <span class="token parameter variable">-m</span> <span class="token string">"New message"</span>  <span class="token comment"># 修改最后一次提交</span>
<span class="token function">git</span> rebase <span class="token parameter variable">-i</span> HEAD~3                 <span class="token comment"># 交互式修改最近3次提交</span>
</code></pre>
    <hr/>
    <h3>
     <a id="_147">
     </a>
     <strong>
      三、代码示例：完整工作流
     </strong>
    </h3>
    <p>
     假设你要开发一个登录功能：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 1. 克隆远程仓库</span>
<span class="token function">git</span> clone https://github.com/yourname/project.git
<span class="token builtin class-name">cd</span> project

<span class="token comment"># 2. 创建并切换到新分支</span>
<span class="token function">git</span> checkout <span class="token parameter variable">-b</span> feature-login

<span class="token comment"># 3. 修改代码后提交</span>
<span class="token builtin class-name">echo</span> <span class="token string">"function login() { ... }"</span> <span class="token operator">&gt;&gt;</span> login.js
<span class="token function">git</span> <span class="token function">add</span> login.js
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Add login function"</span>

<span class="token comment"># 4. 推送分支到远程</span>
<span class="token function">git</span> push <span class="token parameter variable">-u</span> origin feature-login

<span class="token comment"># 5. 在 GitHub 创建 Pull Request (PR) 合并到 main</span>
<span class="token comment"># （PR 通过后合并）</span>

<span class="token comment"># 6. 切换回主分支并同步代码</span>
<span class="token function">git</span> checkout main
<span class="token function">git</span> pull origin main
</code></pre>
    <hr/>
    <h3>
     <a id="_176">
     </a>
     <strong>
      四、最佳实践
     </strong>
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        频繁提交
       </strong>
       <br/>
       每个小功能或修复单独提交，提交信息清晰（如 “Fix: resolve login timeout issue”）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        合理使用分支
       </strong>
      </p>
      <ul>
       <li>
        <code>
         main
        </code>
        /
        <code>
         master
        </code>
        ：稳定版本
       </li>
       <li>
        <code>
         develop
        </code>
        ：开发分支
       </li>
       <li>
        <code>
         feature/*
        </code>
        ：功能分支
       </li>
       <li>
        <code>
         hotfix/*
        </code>
        ：紧急修复分支
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        定期拉取远程代码
       </strong>
       <br/>
       避免长时间不同步导致冲突。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用
        <code>
         .gitignore
        </code>
       </strong>
       <br/>
       忽略临时文件（如
       <code>
        node_modules/
       </code>
       ,
       <code>
        .log
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        代码审查
       </strong>
       <br/>
       通过 Pull Request 进行代码审查后再合并。
      </p>
     </li>
    </ol>
    <hr/>
    <p>
     通过以上内容，你可以逐步掌握 Git 的核心用法。建议多实践操作，遇到问题时善用
     <code>
      git status
     </code>
     和
     <code>
      git log
     </code>
     查看状态和历史。
    </p>
