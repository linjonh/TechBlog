    <h2>
     <a id="dockeripnginxdns_0">
     </a>
     docker配置固定ip解决nginx代理容器名称dns缓存不更新问题
    </h2>
    <h3>
     <a id="_2">
     </a>
     场景
    </h3>
    <p>
     在nginx配置中，proxy_pass代理项使用容器名称替代ip，容器重启后代理的url网络不通。
    </p>
    <h4>
     <a id="nginx_4">
     </a>
     nginx配置示例
    </h4>
    <pre><code class="prism language-conf">location /api/ {
	proxy_set_header Host $http_host;
	proxy_set_header X-Real-IP $remote_addr;
	proxy_set_header REMOTE-HOST $remote_addr;
	proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    # 这里的iot-backend为docker容器名称
	proxy_pass http://iot-backend:8089;
}
</code></pre>
    <h4>
     <a id="docker_dns_15">
     </a>
     docker dns
    </h4>
    <p>
     在docker网络中，提供有默认dns服务
     <code>
      127.0.0.11
     </code>
     用于容器名称和容器ip的解析转换，同时自动生成文件到容器内
     <code>
      /etc/resolv.conf
     </code>
     。在不指定容器ip时，每次容器重启后，容器ip会重新分配而发生变更。
    </p>
    <h4>
     <a id="nginx_dns_18">
     </a>
     nginx dns缓存
    </h4>
    <p>
     在nginx中，在第一次dns解析成功后，结果会被缓存，并不会自动更新。因此容器重启后，容易导致proxy_pass配置的基于容器名称的代理网络不通。此时需要手动重启nginx或重新加载nginx配置，触发后续网络请求进入更新dns。
    </p>
    <h3>
     <a id="ipip_21">
     </a>
     使用固定ip解决重启后容器ip变更
    </h3>
    <p>
     创建容器网络，为容器配置网络，并设置固定ip。容器重启后ip不变，间接保证nginx代理url网络联通。
    </p>
    <h4>
     <a id="docker_compose_23">
     </a>
     docker compose配置示例
    </h4>
    <pre><code class="prism language-yml"><span class="token comment"># 创建容器网络，桥接模式，使用ipam固定ip</span>
<span class="token key atrule">networks</span><span class="token punctuation">:</span>
  <span class="token key atrule">iot-network</span><span class="token punctuation">:</span>
    <span class="token key atrule">name</span><span class="token punctuation">:</span> iot<span class="token punctuation">-</span>network
    <span class="token key atrule">ipam</span><span class="token punctuation">:</span>
      <span class="token key atrule">driver</span><span class="token punctuation">:</span> default
      <span class="token key atrule">config</span><span class="token punctuation">:</span>
        <span class="token punctuation">-</span> <span class="token key atrule">subnet</span><span class="token punctuation">:</span> <span class="token string">'177.7.0.0/16'</span>

<span class="token comment"># 为容器配置网络和固定ip</span>
<span class="token key atrule">iot-backend</span><span class="token punctuation">:</span>
    <span class="token key atrule">networks</span><span class="token punctuation">:</span>
      <span class="token key atrule">iot-network</span><span class="token punctuation">:</span>
        <span class="token key atrule">ipv4_address</span><span class="token punctuation">:</span> 177.7.0.10
</code></pre>
    <p>
     创建后可以使用
     <code>
      docker network ls
     </code>
     查看容器网络。
    </p>
