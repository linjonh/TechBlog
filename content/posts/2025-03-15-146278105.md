    <h2>
     一、漏洞概述
    </h2>
    <h3>
     1. 定义
    </h3>
    <ul>
     <li>
      文件包含：通过动态变量加载外部文件作为代码执行，常见于PHP的
      <strong>
       `include()`
      </strong>
      、
      <strong>
       `require()`
      </strong>
      等函数。
     </li>
     <li>
      文件下载：通过程序接口直接获取服务器文件内容，若未校验文件路径，可导致敏感文件泄露。
     </li>
    </ul>
    <h3>
     2. 危害
    </h3>
    <ul>
     <li>
      代码执行：恶意文件被包含后直接运行（如WebShell）。
     </li>
     <li>
      敏感信息泄露：配置文件、数据库凭证等被下载。
     </li>
    </ul>
    <h2>
     二、文件包含分类
    </h2>
    <h4>
     1. 本地包含
    </h4>
    <p>
     <strong>
      无限制包含
     </strong>
    </p>
    <ul>
     <li>
      直接指定本地文件路径，无任何过滤
     </li>
    </ul>
    <pre><code class="language-php"> #案例：  

    php

    // upload.php

    $filename = $_GET'file';

    include($filename);</code></pre>
    <ul>
     <li>
      访问：http://example.com/upload.php?file=1.txt（1.txt含PHP代码）
     </li>
    </ul>
    <p>
     <strong>
      有限制包含
     </strong>
    </p>
    <ul>
     <li>
      通过黑名单/白名单过滤文件名，需绕过机制。
     </li>
     <li>
      绕过方式：
     </li>
     <li>
      00截断：利用
      <strong>
       `%00`
      </strong>
      截断文件名（需
      <strong>
       `magic_quotes_gpc`
      </strong>
      关闭）。
     </li>
    </ul>
    <pre><code class="language-html">http

      http://example.com/upload2.php?filename=1.txt%00</code></pre>
    <ul>
     <li>
      路径长度截断：利用Windows（255字符）或Linux（4096字符）路径限制。
     </li>
     <li>
      文件名混淆：添加多余后缀（如
      <strong>
       `1.txt.php`
      </strong>
      ）或特殊符号（如
      <strong>
       `1.txt`
      </strong>
      ）。
     </li>
    </ul>
    <h4>
     2. 远程包含
    </h4>
    <ul>
     <li>
      原理：通过`allow_url_include`配置加载远程文件，危害性更高。
     </li>
     <li>
      绕过方式：
     </li>
     <li>
      文件名伪装：强制服务器解析非预期后缀（如
      <strong>
       `1.txt.html`
      </strong>
      改为
      <strong>
       `1.txt`
      </strong>
      ）。
     </li>
     <li>
      协议绕过：使用
      <strong>
       `data://`
      </strong>
      、
      <strong>
       `php://`
      </strong>
      等伪协议传输恶意内容。
     </li>
    </ul>
    <h2>

    <h5>
     案例
    </h5>
    <ul>
     <li>
      读取源码：
     </li>
    </ul>
    <pre><code class="language-html">http

  http://example.com/?file=php://filter/read=convert.base64-encode/resource=config.php</code></pre>
    <ul>
     <li>
      执行POST代码：
     </li>
    </ul>
    <pre><code class="language-html">  http

  http://example.com/?file=php://input</code></pre>
    <ul>
     <li>
      POST数据：
      <strong>
       &lt;?php system('whoami'); ?&gt;
      </strong>
     </li>
    </ul>
    <h2>
     四、防御策略
    </h2>
    <h6>
     1. 输入验证
    </h6>
    <ul>
     <li>
      白名单过滤文件路径（如仅允许
      <strong>
       `/uploads/*.jpg`
      </strong>
      ）。
     </li>
     <li>
      禁用危险函数（如
      <strong>
       `eval()`
      </strong>
      、
      <strong>
       `system()`
      </strong>
      ）。
     </li>
    </ul>
    <h6>
     2. 配置调整
    </h6>
    <ul>
     <li>
      禁用远程文件包含：`allow_url_include = Off`。
     </li>
     <li>
      限制文件包含路径：`open_basedir`指定允许目录。
     </li>
    </ul>
    <h6>
     3. 文件处理安全
    </h6>
    <ul>
     <li>
      上传后重命名文件，避免保留原始扩展名。
     </li>
     <li>
      对包含文件进行内容校验（如检测PHP标签）。
     </li>
    </ul>
    <h2>
     五、文件下载漏洞
    </h2>
    <ul>
     <li>
      原理：未校验文件路径或后缀，允许下载任意文件。
     </li>
    </ul>
    <h6>
     案例
    </h6>
    <h6>
     1. Pikachu靶场：
    </h6>
    <pre><code class="language-html"> http

     http://pikachu.com/download.php?file=../../../../etc/passwd</code></pre>
    <h6>
     2. Java反序列化漏洞：
    </h6>
    <ul>
     <li>
      下载`WEB-INF/web.xml`等敏感配置文件。
     </li>
    </ul>
    <p>
     <strong>
      修复建议
     </strong>
    </p>
    <ul>
     <li>
      严格校验下载路径（如禁止
      <strong>
       `../`
      </strong>
      跳转）。
     </li>
     <li>
      限制可下载文件类型（如仅允许
      <strong>
       `.pdf`
      </strong>
      、
      <strong>
       `.docx`
      </strong>
      ）。
     </li>
    </ul>
    <h2>
     六、总结
    </h2>
    <ul>
     <li>
      文件包含与下载漏洞的核心在于动态变量可控和路径过滤不足。防御需结合输入验证、配置加固及安全编码实践。实战中需关注协议特性（如
      <strong>
       `00截断`
      </strong>
      、
      <strong>
       `伪协议`
      </strong>
      ）及绕过技巧。
     </li>
    </ul>

       </strong>
      </span>
     </p>
