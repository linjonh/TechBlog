    <h4>
     <strong>
      一、接口测试的准备工作
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        获取接口文档
       </strong>
      </p>
      <ul>
       <li>
        <p>
         与开发团队确认接口说明文档，明确接口的请求方式（GET/POST）、URL、参数（Query、Body、Header）、响应格式（JSON/XML）及预期结果410。
        </p>
       </li>
       <li>
        <p>
         若文档缺失，可通过抓包工具（如Fiddler）或与开发协作补全接口信息。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        设计测试用例
       </strong>
      </p>
      <ul>
       <li>
        <p>
         根据接口功能设计正向、反向用例，覆盖参数校验、边界值、异常场景等。例如：
        </p>
        <ul>
         <li>
          <p>
           必填参数缺失或为空时的响应；
          </p>
         </li>
         <li>
          <p>
           错误参数类型或格式的返回结果47。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        准备测试数据
       </strong>
      </p>
      <ul>
       <li>
        <p>
         使用CSV文件管理测试数据（如用户名、密码、ID等），通过JMeter的“CSV Data Set Config”实现参数化410。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      二、JMeter接口测试的核心步骤
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        创建测试计划
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          添加线程组
         </strong>
         ：设置线程数（模拟用户并发）、循环次数、启动时间等。接口测试通常设置为单线程循环执行410。
        </p>
       </li>
       <li>
        <p>
         <strong>
          配置HTTP请求默认值
         </strong>
         ：若接口有统一域名或端口，可通过此组件简化后续请求配置4。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        配置HTTP请求
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          GET请求
         </strong>
         ：在“HTTP请求”中填写URL、参数（Parameters标签），如查询学生信息接口10。
        </p>
       </li>
       <li>
        <p>
         <strong>
          POST请求
         </strong>
         ：根据接口需求选择Body Data（如JSON格式）或表单提交，并设置Content-Type（如
         <code>
          application/json
         </code>
         ）1013。
        </p>
       </li>
       <li>
        <p>
         <strong>
          文件上传
         </strong>
         ：在“Files Upload”标签中上传文件，设置MIME类型（如
         <code>
          multipart/form-data
         </code>
         ）10。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        处理关联与动态参数
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          Cookie/Session管理
         </strong>
         ：添加“HTTP Cookie管理器”实现登录态保持。例如，在充值接口中需先提取登录返回的Cookie10。
        </p>
       </li>
       <li>
        <p>
         <strong>
          参数提取
         </strong>
         ：使用“正则表达式提取器”或“JSON提取器”从响应中提取动态值（如Token），并传递给后续请求10。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        添加断言
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          响应断言
         </strong>
         ：验证HTTP状态码（如200）、响应内容包含特定字段（如
         <code>
          "status": "success"
         </code>
         ）47。
        </p>
       </li>
       <li>
        <p>
         <strong>
          JSON断言
         </strong>
         ：通过“JSON Path Assertion”精准校验响应数据结构7。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        配置监听器与报告
       </strong>
      </p>
      <ul>
       <li>
        <p>
         添加“察看结果树”实时查看请求与响应详情。
        </p>
       </li>
       <li>
        <p>
         使用“聚合报告”或“Summary Report”生成性能指标（吞吐量、平均响应时间、错误率）713。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      三、实战案例演示
     </strong>
    </h4>
    <h5>
     <strong>
      案例1：学生管理系统接口测试
     </strong>
     10
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        登录接口（POST）
       </strong>
      </p>
      <ul>
       <li>
        <p>
         Body中提交
         <code>
          username
         </code>
         和
         <code>
          passwd
         </code>
         ，断言返回的Token或状态码。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        添加学生信息（JSON格式）
       </strong>
      </p>
      <ul>
       <li>
        <p>
         Body Data中填入JSON参数，设置编码为UTF-8避免中文乱码。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        权限验证接口
       </strong>
      </p>
      <ul>
       <li>
        <p>
         在充值请求前关联登录接口的Cookie，通过“HTTP Cookie管理器”传递。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <strong>
      案例2：天气预报接口性能测试
     </strong>
     13
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        SOAP接口测试
       </strong>
       ：配置HTTP Header（
       <code>
        Content-Type: text/xml
       </code>
       ），Body中填入XML格式的SOAP请求，验证返回的城市列表数据。
      </p>
     </li>
    </ul>
    <hr/>

      四、优化建议
