    <h2>
     <a id="1_3">
     </a>
     1.链表的概念
    </h2>
    <h3>
     <a id="11_4">
     </a>
     1.1概念
    </h3>
    <p>
     链表也是线性表的一种~
    </p>
    <ul>
     <li>
      线性表：物理结构：不一定连续；逻辑结构：一定是连续的
     </li>
     <li>
      链表：物理结构：不是线性的；逻辑结构：是线性的
     </li>
    </ul>
    <h2>
     <a id="2_10">
     </a>
     2.链表的结构
    </h2>
    <h3>
     <a id="21_12">
     </a>
     2.1结构
    </h3>
    <p>
     可以拿火车车厢举例：
     <br/>
     火车由一节节车厢相连组成，每节车厢都是通过钩锁连接起来的，是独立的个体。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1bdfa840ec4a4f4793eea37f0f3a33e0.png"/>
    </p>
    <p>
     类比：
     <br/>
     火车是由一节节车厢组成的，
     <br/>
     链表就是由一节节的节点组成。
    </p>
    <h3>
     <a id="22_21">
     </a>
     2.2节点的结构
    </h3>
    <p>
     节点由什么组成呢？
     <br/>
     数据+指向下一个节点的指针
    </p>
    <p>
     当我们想要保存⼀个整型数据时，实际是向操作系统申请了⼀块内存，这个内存不仅要保存整型数据，也需要保存下⼀个节点的地址（当下⼀个节点为空时保存的地址为空）。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2d993da4f076440ea02256ee36123584.png"/>
    </p>
    <p>
     由此可知，我们只要弄清楚节点的结构，就弄清了链表的结构。那么，程序中如何定义链表的节点的结构呢？
    </p>

    <h3>
     <a id="32_103">
     </a>
     3.2尾插
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3db3c9c3bd2a4f9a8655ebace93b7179.png">
      <br/>
      实现尾插的思路：找到尾节点，将尾节点跟新的要插入的节点连接起来。
     </img>
    </p>
    <pre><code class="prism language-c"><span class="token comment">//尾插代码实现</span>
<span class="token keyword">void</span> <span class="token function">SLTPushBack</span><span class="token punctuation">(</span>SLTNode<span class="token operator">*</span><span class="token operator">*</span> pphead<span class="token punctuation">,</span> SLTDataType x<span class="token punctuation">)</span>
<span class="token punctuation">{<!-- --></span>
	<span class="token function">assert</span><span class="token punctuation">(</span>pphead<span class="token punctuation">)</span><span class="token punctuation">;</span>
	
	<span class="token comment">//*pphead就是指向第一个节点的指针</span>
	<span class="token comment">//SLTNode* newnode = (SLTNode*)malloc(sizeof(SLTNode));</span>
	SLTNode<span class="token operator">*</span> newnode <span class="token operator">=</span> <span class="token function">SLTBuyNode</span><span class="token punctuation">(</span>x<span class="token punctuation">)</span><span class="token punctuation">;</span>

		<span class="token keyword">while</span> <span class="token punctuation">(</span>ptail<span class="token operator">-&gt;</span>next<span class="token punctuation">)</span>

		ptail<span class="token operator">-&gt;</span>next <span class="token operator">=</span> newnode<span class="token punctuation">;</span>
	<span class="token punctuation">}</span>
<span class="token punctuation">}</span>  
</code></pre>
    <p>
     分析：
     <br/>
     step1:SLTBuyNode()函数即创建一个新节点存储要尾插的数据x，然后开始找尾，先让ptail也指向第一个节点，往后递进，当ptail-&gt;next为NULL时，就找到了尾，最后将新节点与ptail-&gt;next连接即可。
    </p>
    <p>
     step2:但是当链表为空链表时呢？ptail就变成了空指针，那么ptail-&gt;next对空指针进行解引用，代码一定会报错！因此，需要分别讨论：空链表与非空链表。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/93ba1cb4bba2484d97186a1e4bde01b2.png">
      <br/>
      step3:当为非空链表时，即phead==NULL,插入一个新节点newnode,只需将newnode变为首节点，即令phead=newnode
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b695fb2bc8004dabb0c8cda378d4e028.png"/>
     </img>
    </p>
    <p>
     step4：为什么要传地址&amp;plist，用二级指针**pphead? 这步也是最关键的一步！！！
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d8f57f4903d348d596be6320093736b6.png"/>
    </p>
    <ul>
     <li>
      <p>
       在单链表中，头指针（pphead）指向链表的第一个节点。
      </p>
     </li>
     <li>
      <p>
       如果链表为空（pphead 为 NULL），插入新节点时，需要将头指针指向新节点。
      </p>
     </li>
     <li>
      <p>
       如果链表不为空，插入新节点时，头指针不需要修改，但需要找到尾节点并修改尾节点的 next 指针。
      </p>
     </li>
     <li>
      <p>
       如果头指针需要被修改（例如链表为空时），必须通过传递头指针的地址来实现。
      </p>
     </li>
     <li>
      <p>
       在 C 语言中，函数参数是按值传递的，即函数内部对参数的修改不会影响外部的变量。
      </p>
     </li>
     <li>
      <p>
       如果直接传递头指针（SLTNode* pphead），函数内部对 pphead 的修改（例如 pphead = newnode）只会影响函数内的局部变量，而不会影响外部的头指针。
      </p>
     </li>
     <li>
      <p>
       通过传递头指针的地址（SLTNode** pphead），函数可以通过解引用修改外部的头指针。
      </p>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f725c4fae6494a1990fbe177e9a1a667.png"/>

     </img>
    </p>
    <p>
     <em>
      <strong>
       总结：
      </strong>
     </em>
    </p>
    <ol>
     <li>
      传递地址是为了让函数能够修改外部的头指针。
     </li>
     <li>
      如果链表为空，头指针需要被修改为指向新节点。
     </li>
     <li>
      C 语言的按值传递机制决定了必须通过传递指针的地址来实现对指针本身的修改。
     </li>
    </ol>

    <h3>
     <a id="34_188">
     </a>
     3.4尾删
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9c2334382e3b4f04abaecb20c9b6a7e5.png"/>
     <br/>
     step1:确保传入的 pphead 不是 NULL，且链表不为空，即
     <br/>
     （*pphead 不是 NULL），assert(pphead&amp;&amp;*pphead)
     <br/>
     step2:实现尾删我们可能想的是：将最后一个节点释放free掉就完事了，但是，这样的话倒数第二个节点的指针就指向了NULL，成了野指针。因此，我们要在free掉尾节点之后将尾节点的指针ptail置空，同时将倒数第二个节点的指针prev置空。
    </p>
    <p>
     step3:如果链表只有一个节点，直接free掉，并将ptail置空即可
     <br/>
     step4:如果链表有多个节点，则进行找尾节点并free掉后，将尾指针ptail和其前驱节点prev-&gt;next都置空即可。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e017007393534c218767a17db8eadb35.png"/>
     <br/>
     <em>
      <strong>
       总结：
      </strong>
     </em>
    </p>
    <ul>
     <li>
      二级指针的作用：通过 SLTNode** pphead 可以修改链表头指针本身，适用于头删操作。
     </li>
     <li>
      内存管理：使用 free 释放头节点的内存，避免内存泄漏。
     </li>
     <li>
      边界条件：如果链表只有一个节点，删除后链表为空，头指针 *pphead 被置为 NULL。
     </li>
     <li>
      通过二级指针和保存下一个节点，实现了对头节点的删除和头指针的更新。
     </li>
    </ul>
