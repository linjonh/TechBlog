    <p id="main-toc">
     uniapp 开发小程序需要实现微信支付功能，下面对相关业务进行说明。
    </p>
    <h2 id="%E5%AD%97%E6%AE%B5">
     字段说明
    </h2>
    <p>
     <strong>
      AppID
     </strong>
     ：小程序的身份证，微信客户端用来确认你的小程序的身份
    </p>
    <p>
     <strong>
      AppSecret
     </strong>
     ：小程序秘钥
    </p>
    <p>
     <strong>
      openid
     </strong>
     ：一个用户对于一个小程序的标识（用户唯一标识），用来确认用户的唯一身份
    </p>
    <p>
     <strong>
      UnionID
     </strong>
     ：用户在微信开放平台帐号下的唯一标识（小程序绑定开放平台帐号才会有）
    </p>
    <p>
     <strong>
      session_key
     </strong>
     ：会话秘钥（如果微信API返回加密数据，解密就要用到session_key）
    </p>
    <p>
    </p>
    <h2 id="%E5%89%8D%E7%BD%AE%E5%B7%A5%E4%BD%9C">
     <strong>
      前置工作
     </strong>
    </h2>
    <p>
     要实现支付功能需要做一些前置工作，如果
     <strong>
      未完成则微信支付不可用
     </strong>
     。
    </p>
    <p>
     1，申请微信支付商户号
    </p>
    <p>
     需要公司来申请，在
     <a class="link-info" href="https://pay.weixin.qq.com/index.php/core/home/login?return_url=%2F" rel="nofollow" title="微信商户平台">
      微信商户平台
     </a>
     中。
    </p>
    <p>
     2，绑定已有商户号并开通微信支付
    </p>
    <p>
     由小程序管理员开通，在
     <a class="link-info" href="https://mp.weixin.qq.com/" rel="nofollow" title="微信公众平台">
      微信公众平台
     </a>
     中。
    </p>
    <blockquote>
     <p>
      <strong>
       商户与小程序之间的关系
      </strong>
      ：
     </p>
     <p>
      一个商户可以关联多个小程序，一个小程序也可以绑定多个商户。
     </p>
     <p>
      对应日常开发，在多商户的情况，每一个商户号在后台都会进行配置，存入数据库，支付时根据当前不同的账户决定使用那一个商户号，这一块需要后台对每一个AppId和商户号做关联，支付时才能确认具体使用那个商户号。（这一块需要后台处理相关逻辑）
     </p>
    </blockquote>
    <p>
     配置完成如图：
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/776e7a9de7414c9d5873841c18c40c09.png"/>
    </p>
    <p>
    </p>

    <h2 id="%E8%8E%B7%E5%8F%96openid%C2%A0">
     获取openid
    </h2>
    <p>
     小程序支付必须携带openid，一般openid可以后台获取返给前端，因为考虑安全必须要这么做，如果你需要
     <strong>
      前端获取openid，
     </strong>
     如下：
    </p>
    <blockquote>
     <p>
      <strong>
       ps
      </strong>
      ：实际上前台是不能获取openid的，一个是考虑安全性因素，还有就是小程序需要添加安全域名，而微信的接口不能放在安全域名中，如下：
     </p>
     <p style="text-align:center;">
      <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/d3dda572da854494f751c743b5b0c5fb.png"/>
     </p>
     <p>
      当然如果只是前台调试，那么是没有问题的。
     </p>
    </blockquote>
    <p>
     1，调用
     <a class="link-info" href="https://developers.weixin.qq.com/miniprogram/dev/api/open-api/login/wx.login.html" rel="nofollow" title="wx.login">
      wx.login
     </a>
     接口，获取code
    </p>
    <p>
     2，根据code，appid，appsecert获取openid
    </p>
    <p>
     这里默认你已经申请了小程序appid，如果未申请，
     <a class="link-info" href="https://developers.weixin.qq.com/miniprogram/dev/framework/quickstart/getstart.html#%E7%94%B3%E8%AF%B7%E5%B8%90%E5%8F%B7" rel="nofollow" title="请看这里">
      请看这里
     </a>
     ，已经申请可以登录微信
     <a class="link-info" href="http://mp.weixin.qq.xn--com-c16s" rel="nofollow" title="公众平台">
      公众平台
     </a>
     查看自己的appid，appsecret，在公众平台，开发管理，开发设置中。
    </p>
    <p>
     前台获取如下：
    </p>
    <pre><code class="language-javascript">wx.login({
    success: res =&gt; {
        console.log(res.code,'code=====');
        const appid = 'wx5c*********c7ea0'; // 微信小程序appid 
        const secret = 'a5fd***************ea80c6866dc0'; // 微信小程序secret 
        //调用request请求api转换登录凭证 
        wx.request({
            url: 'https://api.weixin.qq.com/sns/jscode2session?appid='+appid+'&amp;secret='+secret+'&amp;grant_type=authorization_code&amp;js_code=' + res.code,
            header: {
                'content-type': 'application/json'
	        },
	        success: function (result) {
		        if(result.statusCode === 200) {
			        console.log(result.data.openid,'openid=======');
			        uni.setStorageSync('openid', result.data.openid);
			        uni.setStorageSync('session_key', result.data.session_key);
	            }
								
	        }
        }) 
    }
})</code></pre>
    <p>
     如果不清楚
     <strong>
      openid获取的业务逻辑
     </strong>
     ，请看
     <a class="link-info" href="https://developers.weixin.qq.com/miniprogram/dev/framework/open-ability/login.html" rel="nofollow" title="小程序登录">
      小程序登录
     </a>
     。
    </p>
    <p>
     如果对
     <strong>
      小程序支付的整体交互逻辑
     </strong>
     不清楚的，推荐查看
     <a class="link-info" href="https://pay.weixin.qq.com/wiki/doc/api/wxa/wxa_api.php?chapter=7_4&amp;index=3" rel="nofollow" title="小程序支付">
      小程序支付
     </a>
     。
    </p>
    <p>
     <strong>
      划重点
     </strong>
     ：openid可由后台获取，前台只需要提供 wx.login 接口获取的 code 即可，剩下工作由后端完成，并在登录接口中返回 openid，前端存储 openid，以便后期使用。
    </p>

    <h2 id="%E5%BE%AE%E4%BF%A1%E6%94%AF%E4%BB%98">
     微信支付
    </h2>
    <p>
     说明一点，小程序支付的 trade_type 为JSAPI，后端需要关注。
    </p>
    <p>
     首先你应该调用业务侧接口，即后端接口，获取调用微信支付API所需参数。
    </p>
    <p>
     调用后端接口这里不做说明，根据具体的业务场景情况不同，获取到支付参数后就可以调用微信小程序的 requestPayment API 调起微信支付，因为开发使用的 uniapp，可以使用 uni.requestPayment API，如下：
    </p>
    <pre><code class="language-javascript">// saveReserve 为业务接口
saveReserve(params).then(data =&gt; {
	const { errno, errmsg, res } = data;
		// res为调起微信支付所需参数
		if(errno === '0') {
			// 调起微信支付
			uni.requestPayment({
				provider: 'wxpay', // 服务提提供商
				timeStamp: res.weChatPayData.timestamp, // 时间戳
				nonceStr: res.weChatPayData.noncestr, // 随机字符串
				package: res.weChatPayData.package,
				signType: res.weChatPayData.signtype, // 签名算法
				paySign: res.weChatPayData.sign, // 签名
				success: function (res) {
					console.log('支付成功',res);
					// 业务逻辑。。。
				},
				fail: function (err) {
					console.log('支付失败',err);
				}
			});		
		}
	}).catch(err =&gt; {
		console.log(err);
	})</code></pre>
    <p>
     相关支付API文档请看
     <a class="link-info" href="https://uniapp.dcloud.io/api/plugins/payment?id=requestpayment" rel="nofollow" title="uniapp支付">
      uniapp支付
     </a>
     。
    </p>
    <p>
     到此微信小程序支付功能完成。
    </p>
    <p>
    </p>
    <h2 id="%E6%8A%A5%E9%94%99%E9%97%AE%E9%A2%98">
     报错问题
    </h2>
    <p>
     1，支付场景非法
    </p>
    <p>
     --后台需要在创建预支付接口时将trade_type修改为JSAPI模式
    </p>
    <p>
     2，支付验证签名失败
    </p>
    <p>
     --需要后台检查生成签名时填写的appid是否正确，一个是小程序appid一个是应用的appid，确保没填错
    </p>
    <p>
     3，调用支付JSAPI缺少参数：total_fee
    </p>
    <p>
     --可能是参数有误，注意package参数格式为："prepay_id=wx20161......"
    </p>
