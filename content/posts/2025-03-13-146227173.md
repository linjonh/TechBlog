     服务器状态码小记
    </h1>
   </div>
  </div>

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     服务器状态码是 HTTP 协议中用于表示服务器响应状态的一组数字代码，它们帮助客户端理解请求的处理结果。状态码通常由三位数字组成，其中每一位都具有特定的含义。第一位数字表示响应的类别，第二位和第三位数字表示该类别下的具体响应情况。
    </p>
    <h4>
     2.
     <strong>
      2xx - 成功状态码
     </strong>
    </h4>
    <p>
     这些状态码表示客户端请求已成功处理，服务器已成功响应。
    </p>
    <ul>
     <li>
      <strong>
       200 OK
      </strong>
      ：请求已成功处理，响应包含请求的结果。
     </li>
     <li>
      <strong>
       201 Created
      </strong>
      ：请求已成功处理，并创建了新资源。通常用于 POST 请求。
     </li>
     <li>
      <strong>
       202 Accepted
      </strong>
      ：请求已接受，但尚未处理完成。用于异步处理的请求。
     </li>
     <li>
      <strong>
       203 Non-Authoritative Information
      </strong>
      ：请求成功，但返回的信息来自缓存或第三方。
     </li>
     <li>
      <strong>
       204 No Content
      </strong>
      ：请求成功，但服务器没有返回内容（通常用于 DELETE 请求）。
     </li>
     <li>
      <strong>
       205 Reset Content
      </strong>
      ：请求成功，但客户端应重置视图或表单。
     </li>
     <li>
      <strong>
       206 Partial Content
      </strong>
      ：服务器成功处理了部分请求的内容（例如，下载文件的某部分）。
     </li>
    </ul>
    <h4>
     3.
     <strong>
      3xx - 重定向状态码
     </strong>
    </h4>
    <p>
     这些状态码表示客户端需要进一步的操作（通常是重定向）才能完成请求。
    </p>
    <ul>
     <li>
      <strong>
       301 Moved Permanently
      </strong>
      ：请求的资源已永久移动到新的位置，客户端应使用新的 URL。
     </li>
     <li>
      <strong>
       302 Found
      </strong>
      ：请求的资源临时位于不同的位置，客户端应使用当前的 URL 继续访问。
     </li>
     <li>
      <strong>
       303 See Other
      </strong>
      ：客户端应使用 GET 请求访问另一个 URL 以获取资源。
     </li>
     <li>
      <strong>
       304 Not Modified
      </strong>
      ：资源未被修改，客户端可以使用缓存的版本。
     </li>
     <li>
      <strong>
       307 Temporary Redirect
      </strong>
      ：请求的资源临时转移到其他位置，但客户端应使用原始的请求方法继续发送请求。
     </li>
    </ul>
    <h4>
     4.
     <strong>
      4xx - 客户端错误状态码
     </strong>
    </h4>
    <p>
     这些状态码表示客户端请求存在错误或不正确，导致服务器无法处理请求。它们通常用于提醒客户端改正请求后重新发送。
    </p>
    <ul>
     <li>
      <strong>
       400 Bad Request
      </strong>
      ：请求无效，服务器无法理解该请求。
     </li>
     <li>
      <strong>
       401 Unauthorized
      </strong>
      ：请求未授权，客户端未提供有效的身份验证凭证。
     </li>
     <li>
      <strong>
       403 Forbidden
      </strong>
      ：服务器理解请求，但拒绝处理该请求（通常是权限问题）。
     </li>
     <li>
      <strong>
       404 Not Found
      </strong>
      ：请求的资源不存在或无法找到。
     </li>
     <li>
      <strong>
       405 Method Not Allowed
      </strong>
      ：请求方法不被允许（例如，使用 POST 请求访问只支持 GET 请求的资源）。
     </li>
    </ul>
    <h4>
     5.
     <strong>
      5xx - 服务器错误状态码
     </strong>
    </h4>
    <p>
     这些状态码表示服务器在处理有效的请求时发生了错误，通常是由于服务器问题导致无法完成请求。
    </p>
    <ul>
     <li>
      <strong>
       500 Internal Server Error
      </strong>
      ：服务器遇到错误，无法完成请求。
     </li>
     <li>
      <strong>
       501 Not Implemented
      </strong>
      ：服务器不支持请求的方法。
     </li>
     <li>
      <strong>
       503 Service Unavailable
      </strong>
      ：服务器当前无法处理请求，通常是由于过载或维护。
     </li>
    </ul>
