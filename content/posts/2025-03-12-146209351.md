    <h4>
     1.
     <code>
      java -XX:+PrintCommandLineFlags HelloGC
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：打印 JVM 启动时的命令行参数，包括用户显式设置的参数和 JVM 自动默认设置的参数。用于确认 JVM 实际使用的配置。
     </li>
    </ul>
    <h4>
     2.
     <code>
      java -Xmn10M -Xms40M -Xmx60M -XX:+PrintCommandLineFlags -XX:+PrintGC -XX:+PrintGCDetails -XX:+PrintGCTimeStamps -XX:+PrintGCCauses HelloGC
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       参数解析
      </strong>
      ：
      <ul>
       <li>
        <code>
         -Xmn10M
        </code>
        ：设置新生代内存大小为 10M。
       </li>
       <li>
        <code>
         -Xms40M
        </code>
        ：设置堆内存初始大小为 40M。
       </li>
       <li>
        <code>
         -Xmx60M
        </code>
        ：设置堆内存最大大小为 60M。
       </li>
       <li>
        <code>
         -XX:+PrintCommandLineFlags
        </code>
        ：打印启动参数。
       </li>
       <li>
        <code>
         -XX:+PrintGC
        </code>
        ：输出简单的 GC 日志（如 GC 发生的时间、回收区域等）。
       </li>
       <li>
        <code>
         -XX:+PrintGCDetails
        </code>
        ：输出详细的 GC 日志（包括内存回收前后的大小、耗时等）。
       </li>
       <li>
        <code>
         -XX:+PrintGCTimeStamps
        </code>
        ：在 GC 日志中添加时间戳，记录 GC 发生的具体时间。
       </li>
       <li>
        <code>
         -XX:+PrintGCCauses
        </code>
        ：记录触发 GC 的原因（如 Eden 区满、晋升老年代失败等）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       整体作用
      </strong>
      ：配置堆内存参数并全面监控 GC 行为，用于分析 GC 性能。
     </li>
     <li>
      <span style="color:#fe2c24">
       注意：
       <code>
        Xms和Xmx最好设置成大小一样的，这样不会造成堆不断的扩容或者收缩，从而带来性能上的负担。
       </code>
      </span>
     </li>
    </ul>
    <h4>
     3.
     <code>
      java -XX:+UseConcMarkSweepGC -XX:+PrintCommandLineFlags HelloGC
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        <code>
         -XX:+UseConcMarkSweepGC
        </code>
        ：启用 CMS（Concurrent Mark Sweep）垃圾收集器。
       </li>
       <li>
        <code>
         -XX:+PrintCommandLineFlags
        </code>
        ：打印启动参数，确认 CMS 收集器是否生效。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     4.
     <code>
      java -XX:+PrintFlagsInitial
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：打印 JVM 所有参数的
      <strong>
       初始默认值
      </strong>
      ，即未被用户或 JVM 动态调整前的默认配置。
     </li>
    </ul>
    <h4>
     5.
     <code>
      java -XX:+PrintFlagsFinal
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：打印 JVM 所有参数的
      <strong>
       最终值
      </strong>
      ，包括用户显式设置的参数、JVM 根据运行环境自动调整的参数。用于确认参数最终生效情况。
     </li>
    </ul>
    <h4>
     6.
     <code>
      java -XX:+PrintFlagsFinal | grep xxx
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：结合管道命令
      <code>
       grep
      </code>
      ，过滤出包含指定关键字（如
      <code>
       xxx
      </code>
      ）的 JVM 参数，方便快速查找目标参数的最终配置。
     </li>
    </ul>
    <h4>
     7.
     <code>
      java -XX:+PrintFlagsFinal -version | grep GC
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        <code>
         -XX:+PrintFlagsFinal
        </code>
        ：打印所有参数最终值。
       </li>
       <li>
        <code>
         -version
        </code>
        ：输出 JVM 版本信息。
       </li>
       <li>
        <code>
         | grep GC
        </code>
        ：过滤出与 GC（垃圾回收）相关的参数，用于快速确认 JVM 的 GC 配置（如使用的垃圾收集器、GC 相关参数等）。
       </li>
      </ul>
     </li>
    </ul>
