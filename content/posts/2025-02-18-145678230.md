     序
    </h2>
    <p>
     本文主要研究一下如何使用OpenWebUI+deepseek构建本地AI知识库
    </p>
    <h2>
     <a id="_3">
     </a>
     步骤
    </h2>
    <h3>
     <a id="openwebui_4">
     </a>
     拉取open-webui镜像
    </h3>
    <pre><code>docker pull ghcr.io/open-webui/open-webui:main
</code></pre>
    <h3>
     <a id="docker_9">
     </a>
     docker启动
    </h3>
    <pre><code>docker run -d -p 3000:8080 \
-e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
ghcr.io/open-webui/open-webui:main
</code></pre>
    <blockquote>
     <p>
      执行完之后等一会(大概要三四分钟)启动起来了访问http://localhost:3000/，注册管理员账号，注册完要等一会(估计要七八分钟)页面才出来
      <br/>
      可以
      <code>
       -v ./open-webui-data:/app/backend/data
      </code>
      来挂载数据
     </p>
    </blockquote>

    <h3>
     <a id="_31">
     </a>
     开启联网搜索
    </h3>
    <p>
     在设置 --&gt; 管理员设置 --&gt; 联网搜索 --&gt; 启用，然后下拉选择，这里选择tavily，可以去https://app.tavily.com/注册然后获取api key，最后点击保存就可以了

    <h3>
     <a id="_50">
     </a>
     知识库管理
    </h3>
    <ul>
     <li>
      设置 --&gt; 管理员设置 --&gt; 文档 --&gt; 语义向量模型引擎 改为ollama， 语义向量模型改为bge-m3:latest
     </li>
     <li>
      工作空间 --&gt; 知识库，点击+号上传文档
     </li>
     <li>
      使用时需要在聊天框通过
      <code>
       #
      </code>
      来指定知识库
     </li>
     <li>
      设置 --&gt; 管理员设置 --&gt; 数据库 --&gt; 下载数据库，可以导出sqlite数据
     </li>
    </ul>
    <h2>
     <a id="_56">
     </a>
     小结
    </h2>
    <p>
     整体体验是docker部署起来，页面展示有点慢，联网搜索一开始选择serply不生效，换成tavily可以生效，但是国内的搜索引擎还不支持。对于知识库方面，可以上传文档和目录，聊天窗口可以指定使用哪个知识库，也可以同时开启联网搜索，整体效果还可以，可以同时引用知识库和搜索引擎的检索结果，看日志还会保存到向量数据库(
     <code>
      save_docs_to_vector_db
     </code>
     )。
    </p>
