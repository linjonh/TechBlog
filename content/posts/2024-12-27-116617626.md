    <h2>
     <a id="__html_referrer__0">
     </a>
     前端解决第三方图片防盗链的办法 - html referrer 访问图片资源
    </h2>
    <h3>
     <a id="_2">
     </a>
     问题
    </h3>
    <p>
     第三方网站上的图片开启了
     <strong>
      防盗链
     </strong>
     ，防止其他网站访问他们网站上的图片
    </p>
    <h3>
     <a id="_6">
     </a>
     原因
    </h3>
    <p>
     官方输出图片的时候，判断了来源 Referer ，就是从哪个网站访问这个图片，如果是你的网站去加载这个图片，那么 Referer 就是：你的网站地址；
    </p>
    <p>
     如果我们的网站地址不在官方的白名单内，所以就看不到图片了。
    </p>
    <p>
     我们做这个跳板的关键：不发送 Referer，也就是没有来源。那么官方那边，就认为是从浏览器直接访问的，所以就能加载正常的图片了。
    </p>
    <h3>
     <a id="referrer_14">
     </a>
     referrer
    </h3>
    <p>
     在某些情况下，出于一些原因，网站想要控制页面发送给 server 的 referrer 信息的情况下，可以使用这一 referer metadata 参数。
    </p>
    <p>
     <strong>
      参数
     </strong>
    </p>
    <p>
     referer 的 metedata 属性可设置 content 属性值为以下集合：
    </p>
    <ul>
     <li>
      never
     </li>
     <li>
      always
     </li>
     <li>
      origin
     </li>
    </ul>
    <p>
     <strong>
      结果
     </strong>
    </p>
    <ul>
     <li>
      如果 referer-policy 的值为 never：删除 http head 中的 referer；
     </li>
     <li>
      如果 referer-policy 的值为 default：如果当前页面使用的是 https 协议，而正要加载资源使用的是普通的 http 协议，则将 http header 中额 referer 置为空；
     </li>
     <li>
      如果 referer-policy 的值 origin：只发送 origin 部分；
     </li>
     <li>
      如果 referer-policy 的值为 always：不改变 http header 中的 referer 的值；
     </li>
    </ul>
    <p>
     <strong>
      举例
     </strong>
    </p>
    <p>
     如果页面中包含了如下 meta 标签，所有从当前页面中发起的请求将不会携带 referer：
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>meta</span> <span class="token attr-name">name</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>referrer<span class="token punctuation">"</span></span> <span class="token attr-name">content</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>never<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <p>
     如果页面中包含了如下 meta 标签，则从当前页面中发起的 http 请求将只携带 origin 部分：
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>meta</span> <span class="token attr-name">name</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>referrer<span class="token punctuation">"</span></span> <span class="token attr-name">content</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>origin<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <h3>
     <a id="_47">
     </a>
     解决方案
    </h3>
    <p>
     <strong>
      初步方案
     </strong>
    </p>
    <ul>
     <li>
      在 标签里加 meta，referrer 的 content 设置为 nerver
     </li>
    </ul>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>meta</span> <span class="token attr-name">name</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>referrer<span class="token punctuation">"</span></span> <span class="token attr-name">content</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>never<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <p>
     这样存在第三方网站上的图片，在你的网站上就可以访问了。
    </p>
    <p>
     但是还有一个问题，就是如果你的网站需要发送你的网站地址的，那上面的的设置就不行了，比如：用到了百度统计。
    </p>
    <p>
     那上面的设置会导致百度统计的代码加载不了，因为它需要发送
     <strong>
      你的网站地址
     </strong>
     给百度统计。
    </p>
    <p>
     <strong>
      既要不发送 你的网站地址，又要发送你的网站地址，那么怎么办呢 ？
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2f5c548ee52f3b95b5e203cd18759ed8.jpeg#pic_center"/>
    </p>
    <p>
     <strong>
      最终的解决方案
     </strong>
    </p>
    <ul>
     <li>
      先在 html 上设置 referrer 为 always
     </li>
    </ul>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>meta</span> <span class="token attr-name">id</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>referrer<span class="token punctuation">"</span></span> <span class="token attr-name">name</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>referrer<span class="token punctuation">"</span></span> <span class="token attr-name">content</span><span class="token attr-value"><span class="token punctuation">=</span><span class="token punctuation">"</span>always<span class="token punctuation">"</span></span> <span class="token punctuation">/&gt;</span></span>
</code></pre>
    <p>
     这样之后，首屏加载的时候，加载了百度统计的代码了，能正常统计访客数据了。
    </p>
    <ul>
     <li>
      不需要网站地址的时候，再把 referrer 设置为 nerver
     </li>
    </ul>
    <p>
     加个延时 setTimeout 再把 referrer 的 content 值设置为 nerver 。
     <br/>
     或者 在有图片的地方再把 referrer 的 content 值设置为 nerver 。
    </p>
    <pre><code class="prism language-js"><span class="token comment">//  在获取图片的时候设置为 never </span>

referrer<span class="token punctuation">.</span><span class="token function">setAttribute</span><span class="token punctuation">(</span><span class="token string">"content"</span><span class="token punctuation">,</span> <span class="token string">"never"</span><span class="token punctuation">)</span>

</code></pre>
    <p>
     never")
    </p>
    <p>
     // 设置完之后在恢复 , 防止其他接口referrer报错
     <br/>
     referrer.setAttribute(“content”, “always”)
    </p>
