    <h3>
     <a id="1_Array_14">
     </a>
     1. 数组（Array）
    </h3>
    <h4>
     <a id="11__16">
     </a>
     1.1 定义
    </h4>
    <ul>
     <li>
      <strong>
       数组
      </strong>
      是
      <strong>
       固定长度
      </strong>
      、
      <strong>
       连续内存空间
      </strong>
      存储
      <strong>
       相同类型元素
      </strong>
      的数据结构。
     </li>
     <li>
      <strong>
       特点
      </strong>
      ：
      <ul>
       <li>
        <strong>
         长度不可变
        </strong>
        ：创建后无法动态扩容。
       </li>
       <li>
        <strong>
         内存连续
        </strong>
        ：支持快速随机访问（时间复杂度
        <code>
         O(1)
        </code>
        ）。
       </li>
       <li>
        <strong>
         类型固定
        </strong>
        ：所有元素类型必须一致（支持基本类型和对象类型）。
       </li>
      </ul>
     </li>
    </ul>

    <h3>
     <a id="3_List__43">
     </a>
     3. List 接口
    </h3>
    <h4>
     <a id="31__45">
     </a>
     3.1 定义
    </h4>
    <ul>
     <li>
      <strong>
       List
      </strong>
      是 Java 集合框架中的接口，代表
      <strong>
       有序、可重复
      </strong>
      的元素集合。
     </li>
     <li>
      <strong>
       核心功能
      </strong>
      ：
      <ul>
       <li>
        按索引访问元素（
        <code>
         get(int index)
        </code>
        ）。
       </li>
       <li>
        动态增删元素（
        <code>
         add()
        </code>
        ,
        <code>
         remove()
        </code>
        ）。
       </li>
       <li>
        迭代遍历（
        <code>
         iterator()
        </code>
        ）。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="32__52">
     </a>
     3.2 常见实现类
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         实现类
        </strong>
       </th>
       <th>
        <strong>
         特点
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         ArrayList
        </strong>
       </td>
       <td>
        基于动态数组实现，随机访问快，增删中间元素慢。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         LinkedList
        </strong>
       </td>
       <td>
        基于双向链表实现，增删元素快（尤其是头尾），随机访问慢。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         Vector
        </strong>
       </td>
       <td>
        线程安全的动态数组（已过时，建议用
        <code>
         CopyOnWriteArrayList
        </code>
        替代）。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         CopyOnWriteArrayList
        </strong>
       </td>
       <td>
        线程安全，写操作时复制整个数组，读操作无锁。适合读多写少场景。
       </td>
      </tr>
     </tbody>
    </table>

     4. ArrayList vs LinkedList
    </h3>
    <h4>
     <a id="41__64">
     </a>
     4.1 核心区别
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         特性
        </strong>
       </th>
       <th>
        <strong>
         ArrayList
        </strong>
       </th>
       <th>
        <strong>
         LinkedList
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         底层数据结构
        </strong>
       </td>
       <td>
        动态数组
       </td>
       <td>
        双向链表
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         随机访问效率
        </strong>
       </td>
       <td>
        <code>
         O(1)
        </code>
       </td>
       <td>
        <code>
         O(n)
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         头部插入/删除效率
        </strong>
       </td>
       <td>
        <code>
         O(n)
        </code>
        （需移动元素）
       </td>
       <td>
        <code>

       <td>
        <strong>
         内存占用
        </strong>
       </td>
       <td>
        较低（仅存储数据）
       </td>
       <td>
        较高（每个节点存储两个指针）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        高频随机访问，少增删
       </td>
       <td>
        高频增删，尤其是头尾操作
       </td>
      </tr>
     </tbody>
    </table>

    <h3>
     <a id="5__List__87">
     </a>
     5. 其他 List 实现类
    </h3>
    <h4>
     <a id="51_Vector_89">
     </a>
     5.1 Vector（已过时）
    </h4>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：线程安全的动态数组，方法通过
      <code>
       synchronized
      </code>
      实现同步。
     </li>
     <li>
      <strong>
       问题
      </strong>
      ：性能差，已被
      <code>
       CopyOnWriteArrayList
      </code>
      替代。
     </li>
    </ul>
    <h4>
     <a id="52_CopyOnWriteArrayList_93">
     </a>
     5.2 CopyOnWriteArrayList
    </h4>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：写操作时复制新数组，保证线程安全且读操作无锁。
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：读多写少（如事件监听器列表）。
     </li>
    </ul>
    <h4>
     <a id="53_Stack_Vector_97">
     </a>
     5.3 Stack（继承自 Vector）
    </h4>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：后进先出（LIFO）的栈结构。
     </li>
     <li>
      <strong>
       问题
      </strong>
      ：已不推荐使用，建议用
      <code>
       Deque
      </code>
      接口实现栈。
     </li>
    </ul>
    <h4>
     <a id="54_AbstractList__101">
     </a>
     5.4 AbstractList 的定制实现
    </h4>
    <ul>
     <li>
      可通过继承
      <code>
       AbstractList
      </code>
      实现自定义的 List 结构（如固定长度列表）。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="6__106">
     </a>
     6. 总结
    </h3>
    <h4>
     <a id="61__vs_List_108">
     </a>
     6.1 数组 vs List
    </h4>
    <ul>
     <li>
      <strong>
       数组
      </strong>
      ：简单、高效，适合固定长度和基本类型操作。
     </li>
     <li>
      <strong>
       List
      </strong>
      ：功能丰富，适合动态数据管理和复杂操作。
     </li>
    </ul>
    <h4>
     <a id="62_List__112">
     </a>
     6.2 List 实现类选择
    </h4>
    <ul>
     <li>
      <strong>
       ArrayList
      </strong>
      ：优先选择，适合大部分场景。
     </li>
     <li>
      <strong>
       LinkedList
      </strong>
      ：仅当需要高频头尾增删时使用。
     </li>
     <li>
      <strong>
       CopyOnWriteArrayList
      </strong>
      ：线程安全且读多写少时使用。
     </li>
     <li>
      <strong>
       Vector/Stack
      </strong>
      ：已过时，避免使用。
     </li>
    </ul>
