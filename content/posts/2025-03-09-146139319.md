    <h2>

     </a>
     前言
    </h2>
    <blockquote>
     <p>
      我们已经学习了
      <code>
       linux
      </code>
      下许多的工具，
      <code>
       vim
      </code>
      、
      <code>
       gcc
      </code>
      、
      <code>
       make/makefile
      </code>
      等；
     </p>
     <p>
      已经能够在
      <code>
       linux
      </code>
      写代码，并且进行编译运行，让程序在
      <code>
       linux
      </code>
      下跑起来。
     </p>
    </blockquote>
    <p>
     但是，如果我们在写代码的时候遇见了错误；但是我们并不知道错误在哪，在
     <code>
      windows
     </code>
     下，我们可以进行调试来查找代码错误的位置进行修改；我们在
     <code>
      linux
     </code>
     就只能查看源代码，直接查找错误，这样很麻烦；
    </p>
    <p>
     现在就来学习
     <code>
      linux
     </code>
     如何调试程序。
    </p>
    <h2>
     <a id="_gdbcgdb_10">
     </a>
     调试程序 ——
     <code>
      gdb/cgdb
     </code>
    </h2>
    <h3>
     <a id="DebugRelease_12">
     </a>
     <code>
      Debug/Release
     </code>
     模式
    </h3>
    <p>
     在之前学习
     <code>
      C语言
     </code>
     时，听说过
     <code>
      Debug
     </code>
     和
     <code>
      Release
     </code>
     ，只知道
     <code>
      Debug
     </code>
     时用来调试的，程序员写代码的版本；而
     <code>
      Release
     </code>
     是发布版本。
    </p>
    <p>
     现在我们来看一下这两个模式有什么区别
    </p>
    <blockquote>
     <ul>
      <li>
       首先的区别就是
       <code>
        Debug
       </code>
       会生成程序的调试信息，而
       <code>
        Release
       </code>
       不会生成程序的调试信息。
      </li>
     </ul>
    </blockquote>
    <p>
     因为
     <code>
      Debug
     </code>
     模式会生成调试信息，所以
     <code>
      Debug
     </code>
     模式的程序就要比
     <code>
      Release
     </code>
     模式的程序大小要大。
    </p>
    <p>
     我们如何验证呢？
    </p>
    <p>
     <strong>
      <code>
       linux
      </code>
      中
      <code>
       gcc
      </code>
      编译默认生成的是
      <code>
       Release
      </code>
      版本，我们要生成
      <code>
       Debug
      </code>
      模式就要带
      <code>
       -g
      </code>
      选项。
     </strong>
    </p>
    <p>
     现在有这样一段代码
     <code>
      test.c
     </code>
     文件
    </p>
    <pre><code class="prism language-cpp"><span class="token macro property"><span class="token directive-hash">#</span><span class="token directive keyword">include</span><span class="token string">&lt;stdio.h&gt;</span></span>
<span class="token keyword">int</span> <span class="token function">func</span><span class="token punctuation">(</span><span class="token keyword">int</span> n<span class="token punctuation">)</span>

	<span class="token punctuation">{<!-- --></span>

    <p>
     我们在
     <code>
      linux
     </code>
     下进行编译
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7df877abd4e349b785f1a7136b17d0da.png#pic_center"/>

    <p>
     这里有这样的提示，那是因为
     <code>
      C98
     </code>
     不支持在
     <code>
      for
     </code>
     循环中定义变量，要使用
     <code>
      C99

    </p>
    <p>
     所以我们就要这样来编译
    </p>
    <pre><code>gcc test.c -o test -st=c99

    <p>

     <code>
      makefile
     </code>

    </p>

     这样生成的是
     <code>
      Release
     </code>
     模式的程序，我们查看它文件属性

    <p>

    <p>
     现在，我们使用
     <code>
      -g
     </code>
     选项生成
     <code>
      Debug
     </code>
     模式的程序
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/790d63439d224021b8f80a08b947f609.png#pic_center"/>

    <p>
     可以看到
     <code>
      Debug
     </code>
     模式的要比
     <code>
      Release
     </code>
     模式的程序要大一些。
    </p>
    <h3>
     <a id="gdbcgdb_78">
     </a>
     <code>
      gdb
     </code>
     /
     <code>
      cgdb
     </code>
     的使用
    </h3>
    <p>
     这里，只有
     <code>
      Debug
     </code>
     模式的程序才能被调试；
     <code>
      Release
     </code>
     模式下不能被调试，因为缺少调试信息。
    </p>
    <p>
     在使用之前可能需要进行安装
    </p>
    <pre><code>yum -install -y gdb
yum -install -y cgdb
</code></pre>

     </a>

    <pre><code>gdb 可执行程序
</code></pre>
    <p>
     这里无论是
     <code>
      gdb
     </code>
     还是
     <code>
      cgdb
     </code>
     ，都是可执行程序，对可执行程序进行调试。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/42b2732192274d2a93bb7a38751ab2ee.png#pic_center"/>

    <p>
     可以看到这样就进入
     <code>
      gdb
     </code>
     调试了，但是
     <code>
      gdb
     </code>
     调试现在看不到我们的源代码。
    </p>
    <h4>
     <a id="2__102">

     2. 退出调试

     调试的界面

      gdb

       cgdb
      </code>

      <code>

      </code>

    </blockquote>
    <p>
     退出调试仍然是
     <code>
      quit
     </code>
    </p>
    <h4>
     <a id="3__126">
     </a>
     3. 查看代码
    </h4>
    <p>
     查看代码，
     <code>
      l
     </code>
     ；后可以什么的不跟，也可以跟行号或者函数名
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        l
       </code>
       ：查看源代码，从上次位置开始，依次显示
       <code>
        10
       </code>
       行代码
      </li>
      <li>
       <code>
        l 文件名:行号
       </code>
       ：列出指定文件的源代码
      </li>
      <li>
       <code>
        l 函数名
       </code>
       ：列出指定函数的源代码
      </li>
     </ul>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6319d2ca6a2143c0a9041077300870c5.png#pic_center"/>

    <p>
     <code>
      l 文件名:行号
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2056ae5592654622a5bfa82ec0051bdf.png#pic_center"/>
    </p>
    <p>
     <code>
      l:函数名

    <p>

    <h4>
     <a id="4__149">
     </a>
     4. 运行代码
    </h4>
    <p>
     我们进入调试，但是代码并没有运行起来；在
     <code>
      windows
     </code>
     下我们之间
     <code>
      F5
     </code>
     就让代码运行起来了；而
     <code>
      cgdb
     </code>
     中
     <code>
      r
     </code>
     命令可以让代码运行起来
    </p>
    <p>
     <code>
      r/run
     </code>
     ，执行代码：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f8e4d60d79cf418eb28f93e7c3a80f74.png#pic_center"/>
    </p>
    <h5>
     <a id="_158">
     </a>
     逐步执行
    </h5>
    <blockquote>
     <p>
      有了断点，我们在
      <code>
       r
      </code>
      时程序就会停止在断点处，那我们该如何一行一行执行代码呢？
     </p>
     <p>
      在
      <code>
       windows
      </code>
      下，我们是按
      <code>
       F10
      </code>
      和
      <code>
       F11
      </code>
      来依次执行代码；
     </p>
     <p>
      在
      <code>
       linux
      </code>
      中
      <code>
       cgdb
      </code>
      ，我们使用
      <code>
       n/next
      </code>
      和
      <code>
       s/step
      </code>
      来依次执行代码
     </p>
    </blockquote>
    <p>
     <code>
      n/next
     </code>
     ，相当于
     <code>
      F10
     </code>
     ，一行一行执行代码， 在遇到函数时，不进入函数内部；
    </p>
    <p>
     <code>
      s/step
     </code>
     ，就相当于
     <code>
      F11
     </code>
     ，一行一行执行代码， 在遇到函数时，进入函数内部；
    </p>
    <p>
     这里就不演示了。
    </p>
    <h5>
     <a id="_172">
     </a>
     执行到某处
    </h5>
    <blockquote>
     <p>
      在我们调试程序时，程序现在停止在一个断点处，我们不想一行一行执行代码，而是想要让程序直接运行到下一个代码；
     </p>
     <p>
      只需要指向
      <code>
       c/contine
      </code>
      即可
     </p>
    </blockquote>
    <p>
     <strong>
      <code>
       c/continue
      </code>
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8304df2495b74dc3b2ee7ac3757e1849.png#pic_center"/>
    </p>
    <p>
     <strong>
      <code>
       r
      </code>
     </strong>
     重新执行
    </p>
    <blockquote>
     <p>
      如果现在程序正在执行，我们想要让程序重新执行，只需要
      <code>
       r
      </code>
      即可；
     </p>
     <p>
      这是会询问我们是否重新执行，
      <code>
       y
      </code>
      即可
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7facc919745645438b06ca0e7452c75e.png#pic_center"/>
    </p>
    <p>
     <code>
      finish
     </code>
    </p>
    <blockquote>
     <p>
      执行到当前函数结束，然后停止
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/18e318d6bb7b425ab4bf466ebebf3114.png#pic_center"/>
    </p>
    <p>
     <code>
      until
     </code>
     执行到某一行
    </p>
    <blockquote>
     <p>
      <code>
       until 行号
      </code>
      程序执行到某一行然后停止。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9be29d9e51884ad58cdef994e0660a23.png#pic_center"/>
    </p>
    <p>
     现在执行
     <code>
      until 11
     </code>
     ，让程序执行到
     <code>
      11
     </code>
     行。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/019080c7b31a4de1b68344dbf6365da0.png#pic_center"/>
    </p>
    <h4>
     <a id="_211">
     </a>
    </h4>
    <h4>
     <a id="5__213">
     </a>
     5. 断点
    </h4>
    <h5>
     <a id="_215">
     </a>
     增加/删除断点
    </h5>
    <blockquote>
     <p>
      我们指向
      <code>
       run/r
      </code>
      后，发现代码直接就执行结束了；但是在我们调试的时候，我们并不希望代码执行运行结束，我们需要通过断点让代码在指定位置停下来；
     </p>
    </blockquote>
    <p>
     在
     <code>
      windows
     </code>
     下，我们通过快捷键
     <code>
      F9
     </code>
     或者鼠标点击来打断点和去掉断点；
    </p>
    <p>
     在
     <code>
      cgdb
     </code>
     中，我们通过命令**
     <code>
      b/break
     </code>
     **来打断点，通过
     <code>
      delete/d
     </code>
     来取消断点。
    </p>
    <p>
     <code>
      b
     </code>
     打断点
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        b 行号
       </code>
       ：在指定行打断点
      </li>
      <li>
       <code>
        b 函数名
       </code>
       ：在函数开头打断点
      </li>
     </ul>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dbcdfa6ee7dc40a49b60a10af2f1ca86.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      可以看到，我们打断点之后并看不到任何断点信息，那如果我们想要看到已经存在的断点，可以使用命令
      <code>
       info b
      </code>
      来查看
     </p>
    </blockquote>
    <p>
     <code>
      info b
     </code>
     查看所有断点信息
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5c045f0e455646849dc2e6e416529645.png#pic_center"/>
    </p>
    <p>
     <code>
      d
     </code>
     删除断点
    </p>
    <blockquote>
     <p>
      我们打断点可以通过
      <code>
       行号
      </code>
      ，但是删除断点我们就不能使用行号了，而是使用
      <code>
       Num
      </code>
      断点编号。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3a7736d94440403f856595282c2d0648.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      <strong>
       这里还要注意一个点，断点编号时不断递增的，不会随着我们删除断点而减小
      </strong>
     </p>
    </blockquote>
    <p>
     什么意思呢，就是现在存在两个断点我们删除了其中一个断点，然后再次创建了一个断点，它的编号就是
     <code>
      3
     </code>
     而不是
     <code>
      2
     </code>
     。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/628fea60fa1c4df89da1164c23e8ab28.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      这样有了断点，我们在执行
      <code>
       r
      </code>
      时就程序就会停止在断点处。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2d4cc363baea41b4bc6fa1d97fd91985.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      这里看一下上半代码部分，可以看到程序停在了
      <code>
       15
      </code>
      行，并且断点位置的行号颜色为红色。
     </p>
    </blockquote>
    <h5>
     <a id="_259">
     </a>
     启用/禁用断点
    </h5>
    <blockquote>
     <p>
      当我们在调试程序时，我们增加的断点并不一定所有的都能用的到，有一些断点我们不想让它在这次调试中起作用，这时就可以禁用这个断点
     </p>
    </blockquote>
    <p>
     看到这里可能有疑惑，为什么不直接删除呢？
    </p>
    <blockquote>
     <p>
      如果代码非常的多，删除了之后，接下来调试要用到，又要重新去找，非常浪费时间。
     </p>
    </blockquote>
    <p>
     <strong>
      <code>
       启用断点
      </code>
     </strong>
    </p>
    <pre><code>enable 断点编号
</code></pre>
    <p>
     <strong>
      <code>
       禁用断点
      </code>
     </strong>
    </p>
    <pre><code>disable 断点编号
</code></pre>
    <p>
     这里断点默认是启用状态的。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ea5073dd2dd34e4bbb8f873b9a5ee1d6.png#pic_center"/>
    </p>
    <p>
     那现在执行一下看是否真的禁用了呢？
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b94c198a1cd14ee6bf98d51c020dd3e1.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      可以看到第一个断点并未触发，而是直接触发第二个断点。
     </p>
    </blockquote>
    <h4>
     <a id="6__291">
     </a>
     6. 监视
    </h4>
    <p>
     在
     <code>
      windows
     </code>
     下我们通过监视窗口来查看一个变量的值；
    </p>
    <p>
     而在
     <code>
      linux
     </code>
     的
     <code>
      cgdb
     </code>
     中，我们也可以通过指令来查看变量的值。
    </p>
    <h5>
     <a id="_297">
     </a>
     监视变量
    </h5>
    <p>
     <code>
      p
     </code>
    </p>
    <blockquote>
     <p>
      <code>
       p
      </code>
      用来查看一个变量当前的值
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/454e3afd3298464db3130bd14a4373b9.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      但是这样，我们在此执行代码会发现，执行过后就不在显示了；
     </p>
     <p>
      这样我们每次查看就要去输入指令
      <code>
       p 变量名
      </code>
      ，这样好麻烦，我们想要每一次执行过后，它都会显示出来变量的值
     </p>
     <p>
      <code>
       display
      </code>
      用来跟踪显示变量的值。
     </p>
    </blockquote>
    <p>
     <code>
      display
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/516301131c484c3f9bb9b98637b1b91f.png#pic_center"/>
    </p>
    <p>
     如上图所示，我们每一次执行代码，变量的值都会显示出来。
    </p>
    <h5>
     <a id="_318">
     </a>
     监视函数栈帧内局部变量
    </h5>
    <blockquote>
     <p>
      如果我们不是想要查看某一个变量，而是查看当前函数内所有的局部变量？
     </p>
    </blockquote>
    <p>
     我们就要用到
     <code>
      info/i locals
     </code>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/35f78b215b35451697e03f09051639c6.png#pic_center"/>
    </p>
    <p>
     可以看到
     <code>
      func
     </code>
     中所有的局部变量都显示出来了。
    </p>
    <h5>
     <a id="_329">
     </a>
     查看当前函数调用栈帧

    <blockquote>

      如果我们想要查看当前的函数调用栈帧，直接使用

       bt/backtrace
