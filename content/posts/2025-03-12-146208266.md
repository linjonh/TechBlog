     HTTP 协议中常见的错误状态码（详细介绍）
    </h1>
   </div>
  </div>
 </div>

    <p>
     以下是 HTTP 协议中常见的错误状态码及其原因的总结，按错误类型分类整理：
    </p>
    <h4>
     <strong>
      4xx 客户端错误
     </strong>
    </h4>
    <h5>
     <strong>
      400 Bad Request
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：请求格式错误，服务器无法解析。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        请求头或请求体语法错误（如 JSON/XML 格式错误）。
       </li>
       <li>
        缺少必要参数或参数类型不匹配。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      401 Unauthorized
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：请求未通过身份验证。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        未提供身份凭证（如 Token、Cookie）。
       </li>
       <li>
        提供的凭证已过期或无效。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      403 Forbidden
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器拒绝执行请求，即使客户端已认证。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        用户权限不足（如访问其他用户的私有资源）。
       </li>
       <li>
        IP 地址或设备被限制访问。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      404 Not Found
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：请求的资源在服务器上不存在。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        URL 路径错误（如拼写错误）。
       </li>
       <li>
        资源已被删除或未正确部署。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      405 Method Not Allowed
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：请求方法（GET/POST/PUT 等）对目标资源不适用。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：对只支持 GET 的接口发送 POST 请求。
     </li>
    </ul>
    <h5>
     <strong>
      408 Request Timeout
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器等待请求超时。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：客户端发送请求过慢，或网络延迟过高。
     </li>
    </ul>
    <h5>
     <strong>
      429 Too Many Requests
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：客户端发送的请求频率超过限制。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：短时间内频繁调用 API，触发服务器的限流策略。
     </li>
    </ul>
    <h4>
     <strong>
      5xx 服务器端错误
     </strong>
    </h4>
    <h5>
     <strong>
      500 Internal Server Error
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器内部发生未预期的错误。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        代码逻辑错误（如空指针异常）。
       </li>
       <li>
        数据库连接失败。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      501 Not Implemented
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器不支持请求的功能。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：请求了一个服务器未实现的 HTTP 方法。
     </li>
    </ul>
    <h5>
     <strong>
      502 Bad Gateway
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：作为代理或网关的服务器从上游服务器收到无效响应。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        反向代理服务器无法连接后端服务。
       </li>
       <li>
        上游服务器崩溃或配置错误。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      503 Service Unavailable
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器暂时无法处理请求（通常因过载或维护）。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：
      <ul>
       <li>
        服务器维护或重启。
       </li>
       <li>
        突发的高并发请求导致资源耗尽。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      504 Gateway Timeout
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：代理或网关服务器未及时收到上游服务器的响应。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      ：后端服务处理时间过长，超过网关的等待时间。
     </li>
    </ul>
    <h5>
     <strong>
      505 HTTP Version Not Supported
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：服务器不支持请求使用的 HTTP 协议版本。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：客户端使用 HTTP/3，但服务器仅支持 HTTP/1.1。
     </li>
    </ul>
    <h4>
     <strong>
      通用排查建议
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        客户端错误（4xx）
       </strong>
       ：
      </p>
      <ul>
       <li>
        检查请求的 URL、参数、请求方法、请求头是否符合 API 文档。
       </li>
       <li>
        验证身份凭证是否有效（如 Token 是否过期）。
       </li>
       <li>
        通过浏览器开发者工具或抓包工具（如 Fiddler）分析原始请求。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        服务端错误（5xx）
       </strong>
       ：
      </p>
      <ul>
       <li>
        查看服务器日志（如 Nginx/应用日志）定位具体错误堆栈。
       </li>
       <li>
        检查服务器资源使用情况（CPU、内存、磁盘）。
       </li>
       <li>
        确认上游服务（如数据库、第三方 API）是否正常运行。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     通过状态码可快速定位问题方向，但具体原因需结合服务器日志和请求上下文进一步分析。
    </p>
