   <div class="htmledit_views" id="content_views">
    <p>
     <a href="http://spaces.msn.com/members/rayozzie/" rel="nofollow noopener noreferrer" target="_blank">
      Ray Ozzie
     </a>
     是Lotus Notes和Groove的创始人，今年4月随着
     <a href="http://news.csdn.net/news/newstopic/20/20007.shtml" rel="nofollow noopener noreferrer" target="_blank">
      微软收购Groove
     </a>
     而担任微软CTO，本月早些时候，更有迹象表明，Bill Gates将把
     <a href="http://news.csdn.net/news/newstopic/29/29134.shtml" rel="nofollow noopener noreferrer" target="_blank">
      微软首席软件设计师的职位移交给他
     </a>
     。
    </p>
    <p>
     Ray Ozzie在
     <a href="http://news.zdnet.com/2100-3513_22-5942232.html" rel="nofollow noopener noreferrer" target="_blank">
      The Internet Services Disruption
     </a>
     中承认微软未能在关键技术上取得领导地位，随后Bill Gates认可了Ray的观点并向所有员工发送了战略意义的
     <a href="http://news.csdn.net/news/newstopic/29/29062.shtml" rel="nofollow noopener noreferrer" target="_blank">
      备忘录
     </a>
     。
    </p>
    <p>
     Ray Ozzie最新一篇Blog的标题是
     <a href="http://spaces.msn.com/members/rayozzie/Blog/cns!1pyct_cYtbBtOBPDVAumMEdw!175.entry" rel="nofollow noopener noreferrer" target="_blank">
      Really Simple Sharing
     </a>
     ，看起来像是RSS的一种简称，其实是微软在
     <a href="http://news.csdn.net/news/newstopic/29/29933.shtml" rel="nofollow noopener noreferrer" target="_blank">
      扩展RSS为SSE
     </a>
     (Simple Sharing Extensions)。
    </p>
    <p>
     从微软的
     <a href="http://msdn.microsoft.com/xml/rss/sse/" rel="nofollow noopener noreferrer" target="_blank">
      Simple Sharing Extensions for RSS and OPML
     </a>
     我们发现，扩展的地方主要表现在接收方不再只是信息的阅读者，还能管理和组织Feed(包括添加、删除、更新、聚合等)并再次发布(当然需要相应的系统支持)，这样就让信息变成多方向的流转。
    </p>
    <p>
     为了做到这一点，SSE引入三组扩展：
     <br/>
     <br/>
    </p>
    <blockquote>
     <b>
      第一组：&lt;sx:sharing&gt;包含&lt;sx:related&gt;
     </b>
     <br/>
     <ul>
      <li>
       sx:sharing标签用来表述信息源是否有序，更新的频率和更新的范围
       <br/>
      </li>
      <li>
       sx:related表示信息源的相关连接(用这种方式可以实现Feed之间的跳转和聚合)
       <br/>
       对一个Feed来说，SSE定义为2种，complete或aggregated，complete表示完整的、能按照发布者定期部分更新的Feed。aggregated表示聚合过来的Feed源，和OPML的List有点接近。
       <br/>
       <br/>
       <b>
        第二组：&lt;sx:sync&gt;包含&lt;sx:history&gt;包含&lt;sx:update&gt;
       </b>
       <br/>
      </li>
      <li>
       sx:sync定义基本的版本号、删除检测和冲突检测标志
       <br/>
      </li>
      <li>
       sx:history和sx:update标签比较简单，sx:history记录每个用户最后一次的修改时间和修改者，sx:update则记录他的前3次记录
       <br/>
       <br/>
       <b>
        第三组：&lt;sx:sync&gt;包含&lt;sx:conflicts&gt;包含&lt;sx:conflict&gt;
       </b>
       <br/>
       当sx:sync中设置conflict=true的时候，程序需要处理冲突检测，这是指当有信息在多方同时改变的时候需要更新SSE文件，这些需要记录conflict信息包括：
       <br/>
      </li>
      <li>
       sx:conflicts标签没有实质内容，是sx:conflict标签的组合
       <br/>
      </li>
      <li>
       sx:conflict包含冲突时间、冲突节点(用户)和版本号
       <br/>
      </li>
     </ul>
    </blockquote>
    <p>
    </p>
    <p>
     SSE标准遵循
     <a href="http://creativecommons.org/licenses/by-sa/2.5/" rel="nofollow noopener noreferrer" target="_blank">

     授权方式，目前是0.9版本，很多知名人士
     <a href="http://blogs.msdn.com/alexbarn/archive/2005/11/21/495253.aspx" rel="nofollow noopener noreferrer" target="_blank">
      对SSE表示了赞许
     </a>
     ，让我们也期待他的发展。
    </p>
