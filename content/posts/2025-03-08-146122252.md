    <p>

    <h6>
     <a id="_51">
     </a>
     问题一
    </h6>
    <blockquote>
     <p>
      github 访问不稳定，那么如何实现git仓库配置多个仓库并实现代码同步更新托管？
     </p>
    </blockquote>
    <p>
     调研发现有两种方式实现：（本文选择的第二种）
    </p>
    <p>
     第一种：添加多个远程地址，给不同别名，多条命令，分别更新 ⌛️
    </p>
    <p>
     第二种：添加多个远程地址，一条命令同步更新 👍
    </p>
    <h6>
     <a id="_62">
     </a>
     方式一
    </h6>
    <p>
     第一步：首先在不同平台（
     <code>
      github / gitee / coding
     </code>
     ）新建仓库地址
    </p>
    <p>
     github:
     <a href="https://github.com/winyh/winsax.git">
      国外仓库
     </a>
    </p>
    <p>
     gitee（同步更新）:
     <a href="https://gitee.com/winyh/winsax.git" rel="nofollow">
      国内仓库
     </a>
    </p>
    <p>
     coding (待实现) : 例如 https://e.coding.net/winyh/winsax.git
    </p>
    <p>
     第二步：分别添加远程地址
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token function">add</span> origin https://github.com/winyh/winsax.git
<span class="token function">git</span> remote <span class="token function">add</span> gitee https://gitee.com/winyh/winsax.git 
<span class="token function">git</span> remote <span class="token function">add</span> coding https://xxxxx.git 
</code></pre>
    <p>
     第三步：执行提交(当前分支 main)
    </p>
    <p>
     git push &lt;
     <code>
      remote_name
     </code>
     &gt; &lt;
     <code>
      branch_name
     </code>
     &gt;
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">'feat:多仓库地址测试'</span>

<span class="token function">git</span> push <span class="token parameter variable">-u</span> gitee main

或者

<span class="token function">git</span> push origin main <span class="token punctuation">(</span>输入账号/密码 推送到 github 仓库<span class="token punctuation">)</span>
<span class="token function">git</span> push gitee main  <span class="token punctuation">(</span>输入账号/密码 推送到 gitee 仓库<span class="token punctuation">)</span>
<span class="token function">git</span> push coding main <span class="token punctuation">(</span>输入账号/密码 推送到 coding 仓库<span class="token punctuation">)</span>
</code></pre>
    <p>
     tips:本地仓库 .git/config 文件可查看配置
    </p>
    <p>
     相关命令
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token parameter variable">-v</span> 查看远程仓库地址 <span class="token punctuation">(</span>会显示3个远程地址<span class="token punctuation">)</span>
<span class="token function">git</span> remote remove origin（移除远程仓库）
</code></pre>
    <h6>
     <a id="_107">
     </a>
     方式二（推荐）
    </h6>
    <p>
     第一步：删除 gitee 和 coding 别名仓库地址
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote remove gitee （远程仓库别名）
<span class="token function">git</span> remote remove coding （远程仓库别名）
</code></pre>
    <p>
     第二步：查看远程仓库地址并添加其他仓库
    </p>
    <p>
     git remote set-url --add origin &lt;
     <code>
      origin_url
     </code>
     &gt;
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token parameter variable">-v</span> 查看远程仓库地址 <span class="token punctuation">(</span>会显示1个远程地址<span class="token punctuation">)</span>
<span class="token function">git</span> remote set-url <span class="token parameter variable">--add</span> origin https://gitee.com/winyh/winsax.git （gitee平台）
<span class="token function">git</span> remote set-url <span class="token parameter variable">--add</span> origin https://xxxxx.git （coding平台）
</code></pre>
    <p>
     第三步：执行提交(当前分支 main)
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">'feat:多仓库快速提交'</span>
<span class="token function">git</span> push 
或者
<span class="token function">git</span> push origin <span class="token parameter variable">-all</span>
</code></pre>
    <h6>
     <a id="_135">
     </a>
     问题二
    </h6>
    <blockquote>
     <p>
      在推送代码时需要输入多次不同平台的密码～ 重复动作或者代码就可以考虑抽象 / 提效了
     </p>
    </blockquote>
    <p>
     解决办法：配置SSH公钥，部署到不同的代码托管平台，实现
     <code>
      免密登录
     </code>
    </p>
    <p>
     SSH 的全称为 Secure Shell 即安全外壳协议，是一种加密的网络传输协议。它能够在公开的网络环境中提供安全的数据传输环境，通常用于登录远程主机与推拉代码。
    </p>
    <h6>
     <a id="_mac__146">
     </a>
     生成公钥（本文为 mac 环境）
    </h6>
    <p>
     添加本地全局配置（–global / --local / --system）
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"winyh"</span> 
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"your-email@qq.com"</span>

ssh-keygen <span class="token parameter variable">-t</span> rsa <span class="token parameter variable">-C</span> <span class="token string">"your-email@qq.com"</span> （基于邮箱生成公钥）

<span class="token function">cat</span>  ~/.ssh/id_rsa.pub <span class="token punctuation">(</span>查看公钥串<span class="token punctuation">)</span>
</code></pre>
    <p>
     生成公钥时，中间会有一个询问过程，全部敲enter键，一般是有3次（询问密钥存放地址 / 输入 / 确认密码）
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ecefcbe160b44767b0877cb45900c4b1.png#pic_center"/>
    </p>
    <p>
     <code>
      密码可以考虑给空值， 输入/确认都按 enter 就行
     </code>
    </p>
    <p>
     生成公共后会显示密钥存放地址
    </p>
    <p>
     将公钥串部署添加到不同平台（github / gitee / coding）的 SSH key 配置中（
     <a href="https://coding.net/help/docs/repo/ssh/config.html" rel="nofollow">
      参考coding平台
     </a>
     ）
    </p>
    <p>
     id_rsa(私钥)，id_rsa.pub(公钥)
    </p>
    <h6>
     <a id="_172">
     </a>
     校验连通性
    </h6>
    <p>
     在本地运行首次连接时的公钥认证命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">ssh</span> <span class="token parameter variable">-T</span> git@github.com <span class="token punctuation">(</span>github平台<span class="token punctuation">)</span>
<span class="token function">ssh</span> <span class="token parameter variable">-T</span> git@gitee.com <span class="token punctuation">(</span>gitee平台<span class="token punctuation">)</span>
<span class="token function">ssh</span> <span class="token parameter variable">-T</span> git@e.coding.net <span class="token punctuation">(</span>coding平台<span class="token punctuation">)</span>

ssh-keygen <span class="token parameter variable">-R</span> xxx.xxx.xxx.xxx` 命令可以删除 known_hosts 文件中这个主机的旧公钥
</code></pre>
    <p>
     一般提示如下：
    </p>
    <p>
     github 平台
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a9c70a87723f4be7ac3fe75f64249141.png#pic_center"/>
    </p>
    <p>
     gitee 和 coding 平台
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7ac0c6142b6146f7a3297545c6d913f8.png#pic_center"/>
    </p>
    <p>
     这样使用命令 git pull/push 就不用输入密码了，这是因为刚才在生成公钥时，没有输入密码，所以当你选择SSH地址传输时，就可免密码使用命令 git pull/push
    </p>

     <code>
      pull / push
     </code>
     ?
