    <p>
     在 IntelliJ IDEA 中，如果你希望在进行代码格式化（Reformat Code）时，避免将多行参数或多行方法链调用合并成一行，可以通过以下步骤进行设置：
    </p>
    <h4>
     <a id="1__2">
     </a>
     1. 打开设置
    </h4>
    <ul>
     <li>
      在 IntelliJ IDEA 中，点击
      <code>
       File
      </code>
      菜单，然后选择
      <code>
       Settings
      </code>
      （Windows/Linux）或
      <code>
       Preferences
      </code>
      （macOS）。
     </li>
    </ul>
    <h4>
     <a id="2__5">
     </a>
     2. 进入代码格式化设置
    </h4>
    <ul>
     <li>
      在设置窗口中，导航到
      <code>
       Editor
      </code>
      -&gt;
      <code>
       Code Style
      </code>
      -&gt;
      <code>
       Java
      </code>
      （或其他语言，如 Kotlin、JavaScript 等）。
     </li>
    </ul>
    <h4>
     <a id="3__8">
     </a>
     3. 配置方法调用链的换行
    </h4>
    <ul>
     <li>
      在
      <code>
       Wrapping and Braces
      </code>
      选项卡中，找到
      <code>
       Chained method calls
      </code>
      选项。
     </li>
     <li>
      选择
      <code>
       Wrap always
      </code>
      或
      <code>
       Wrap if long
      </code>
      ，以确保方法链调用在多行中显示。
     </li>
    </ul>
    <h4>
     <a id="4__12">
     </a>
     4. 配置参数列表的换行
    </h4>
    <ul>
     <li>
      在
      <code>
       Wrapping and Braces
      </code>
      选项卡中，找到
      <code>
       Method call arguments
      </code>
      选项。
     </li>
     <li>
      选择
      <code>
       Wrap always
      </code>
      或
      <code>
       Wrap if long
      </code>
      ，以确保多行参数在多行中显示。
     </li>
    </ul>
    <h4>
     <a id="5__16">
     </a>
     5. 保存设置
    </h4>
    <ul>
     <li>
      点击
      <code>
       Apply
      </code>
      和
      <code>
       OK
      </code>
      保存设置。
     </li>
    </ul>
    <h4>
     <a id="6__19">
     </a>
     6. 重新格式化代码
    </h4>
    <ul>
     <li>
      现在，当你使用
      <code>
       Reformat Code
      </code>
      功能时，IDEA 会根据你的设置保持多行参数和多行方法链调用的格式。
     </li>
    </ul>
    <h4>
     <a id="_22">
     </a>
     其他注意事项
    </h4>
    <ul>
     <li>
      如果你使用的是特定的代码风格（如 Google Java Style、Kotlin Style Guide 等），可以在
      <code>
       Code Style
      </code>
      设置中选择相应的风格，并进一步自定义。
     </li>
     <li>
      如果你希望这些设置仅应用于当前项目，可以在
      <code>
       Project
      </code>
      级别的设置中进行配置。
     </li>
    </ul>
    <p>
     通过以上步骤，你可以确保在进行代码格式化时，多行参数和多行方法链调用不会被合并成一行。
    </p>
