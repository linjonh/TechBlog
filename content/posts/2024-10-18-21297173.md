    <p>
     本人学习电子信息工程快4年，但是接触到的硬件开发和系统开发比较少，更多接触到的是android软件应用开发和简单的单片机应用，嵌入式开发是多听多余了解。本人目的是想对嵌入式开发感兴趣的朋友做一个关于嵌入式开发的框架和应用有个了解。
    </p>
    <p>
     <strong>
      <span style="font-size:18px">
       1、神码是嵌入式系统？
      </span>
     </strong>
    </p>
    <p>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      嵌入式系统
     </span>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      （
     </span>
     <span lang="en" style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      Embedded system
     </span>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      ），是一种“完全嵌入受控器件内部，为特定应用而设计的专用
     </span>
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="计算机">
      计算机
     </a>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      系统”，根据
     </span>
     <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E8%8B%B1%E5%9C%8B%E9%9B%BB%E5%99%A8%E5%B7%A5%E7%A8%8B%E5%B8%AB%E5%8D%94%E6%9C%83&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="英国电器工程师协会（页面不存在）">
      英国电器工程师协会
     </a>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      （U.K. Institution of Electrical Engineer）的定义，嵌入式系统为控制、监视或辅助设备、机器或用于工厂运作的设备。与
     </span>
     <a href="http://zh.wikipedia.org/wiki/PC" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="PC">
      个人计算机
     </a>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      这样的通用
     </span>
     <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="计算机">
      计算机
     </a>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      系统不同，嵌入式系统通常执行的是
     </span>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      带有特定要求的预先定义的任务
     </span>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      。由于嵌入式系统只针对一项特殊的任务，设计人员能够对它进行优化，减小尺寸降低成本。由于嵌入式系统通常进行大量生产，所以单个的成本节约，能够随着产量进行成百上千的放大。
     </span>
     <br/>
    </p>
    <p>
     <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
      <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
       嵌入式系统
      </span>
      <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
       的核心是由一个或几个预先编程好以用来执行少数几项任务的
      </span>
      <a href="http://zh.wikipedia.org/wiki/%E5%BE%AE%E5%A4%84%E7%90%86%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="微处理器">
       微处理器
      </a>
      <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
       或者
      </span>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%BE%AE%E6%8E%A7%E5%88%B6%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="单片机">
       单片机
      </a>
      <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
       组成。与通用计算机能够运行用户选择的软件不同，嵌入式系统上的软件通常是暂时不变的；所以经常称为“
      </span>
      <a href="http://zh.wikipedia.org/wiki/%E9%9F%8C%E9%AB%94" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128); font-family:sans-serif; font-size:15px; line-height:22.078125px" target="_blank" title="固件">
       固件
      </a>
      <span style="font-family:sans-serif; font-size:15px; line-height:22.078125px">
       ”。
      </span>
      <br/>
     </span>
    </p>

     <h3 style="font-weight:normal; margin:0px 0px 0.6em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-width:1px; border-bottom-style:solid; border-bottom-color:rgb(170,170,170); font-size:22px; font-family:sans-serif; line-height:19.1875px">
      <span class="mw-headline" id=".E7.89.B9.E6.80.A7">
       特性
      </span>
     </h3>
     <ol style="line-height:1.5em; margin:0.3em 0px 0px 3.2em; padding:0px; font-family:sans-serif; font-size:15px">
      <li style="margin-bottom:0.1em">
       嵌入式系统在广义上说就是
       <a href="http://zh.wikipedia.org/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="计算机系统">
        计算机系统
       </a>
       ，它包括除了以通用为目的计算机之外的所有计算机。从便携式音乐播放器到航天飞机的实时控制子系统都能见到嵌入式系统的应用。与通用计算机系统可以满足多种任务不同，嵌入式系统只能完成某些特定目的的任务。但有些也有实时性能的制约因素必须得到满足的原因，如安全性和可用性。除此之外其他功能可能要求较低或没有要求，使系统的硬件得以简化，以降低成本。对于大批量生产的系统来说，降低成本通常是设计的首要考虑。嵌入式系统通常需要简化去除不需要的功能以降低成本，设计师通常选择刚刚满足所需功能的硬件使目标最小化低成本的实现。
      </li>
      <li style="margin-bottom:0.1em">
       嵌入式系统并非总是独立的设备。许多嵌入式系统是以一个部件存在于一个较大的设备，它为设备提供更多的功能，使设备能完成更广泛的任务。例如，吉布森吉他机器人采用了嵌入式系统来调弦，但总的来说
       <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E5%90%89%E5%B8%83%E6%A3%AE%E5%90%89%E4%BB%96%E6%9C%BA%E5%99%A8%E4%BA%BA&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="吉布森吉他机器人（页面不存在）">
        吉布森吉他机器人
       </a>
       设计的目的绝不是调弦而是演奏音乐。同样的，
       <a href="http://zh.wikipedia.org/wiki/%E8%BD%A6%E8%BD%BD%E7%94%B5%E8%84%91" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="车载电脑">
        车载电脑
       </a>
       作为汽车的一个子系统，为它提供了导航，控制，车况反馈等功能。
      </li>
      <li style="margin-bottom:0.1em">
       部分为嵌入式系统编写的程序被称为
       <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%9B%BA%E4%BB%B6" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="固件">
        固件
       </a>
       ，他们存储在
       <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%8F%AA%E8%AF%BB%E5%AD%98%E5%82%A8%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="只读存储器">
        只读存储器
       </a>
       或
       <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%BF%AB%E9%96%83%E8%A8%98%E6%86%B6%E9%AB%94" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="闪存">
        闪存芯片
       </a>
       。他们运行在资源有限的计算机硬件：小内存，没有键盘，甚至没有屏幕。
      </li>
     </ol>
     <h4 style="margin:0px 0px 0.3em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-style:none; font-size:19px; font-family:sans-serif; line-height:19.1875px">
      <span class="mw-headline" id=".E7.94.A8.E6.88.B7.E7.95.8C.E9.9D.A2">
       用户界面
      </span>
     </h4>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      嵌入式系统用户界面包括了从没有用户界面，专注于单一任务到现在的类似于现代桌面操作系统的复杂
      <a href="http://zh.wikipedia.org/wiki/%E5%9B%BE%E5%BD%A2%E7%94%A8%E6%88%B7%E7%95%8C%E9%9D%A2" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="图形用户界面">
       图形用户界面
      </a>
      的各类界面。
     </p>
     <h4 style="margin:0px 0px 0.3em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-style:none; font-size:19px; font-family:sans-serif; line-height:19.1875px">
      <span class="mw-headline" id=".E7.AE.80.E5.8D.95.E7.9A.84.E5.B5.8C.E5.85.A5.E5.BC.8F.E7.B3.BB.E7.BB.9F">

      </span>

     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      同典型的计算机程序员一样，嵌入式系统设计人员也使用
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E7%BC%96%E8%AF%91%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="编译器">
       编译器
      </a>
      、
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E9%80%A3%E7%B5%90%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="连接器">
       连结器
      </a>
      和
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%AA%BF%E8%A9%A6%E5%99%A8" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="调试器">
       调试器
      </a>
      开发嵌入式系统软件。然而，他们也使用一些大多数程序员不熟悉的工具。
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      软件工具的来源有如下几种：
     </p>
     <ul style="line-height:1.5em; margin:0.3em 0px 0px 1.6em; padding:0px; font-family:sans-serif; font-size:15px">
      <li style="margin-bottom:0.1em">
       专注于嵌入式系统市场的软件公司
      </li>
      <li style="margin-bottom:0.1em">
       从
       <a href="http://zh.wikipedia.org/wiki/GNU" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="GNU">
        GNU
       </a>
       软件开发工具移植（参见
       <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E4%BA%A4%E5%8F%89%E5%B9%B3%E5%8F%B0%E7%BC%96%E8%AF%91&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="交叉平台编译（页面不存在）">
        交叉平台编译
       </a>
       ）
      </li>
      <li style="margin-bottom:0.1em">
       有些情况下，如果嵌入式处理器与普通个人计算机处理器很近似的话也可以使用个人计算机开发工具
      </li>
     </ul>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      嵌入式系统设计人员也使用一些不为普通计算机程序员所熟悉的软件工具：
     </p>
     <ul style="line-height:1.5em; margin:0.3em 0px 0px 1.6em; padding:0px; font-family:sans-serif; font-size:15px">
      <li style="margin-bottom:0.1em">
       一个常用工具是“电路内部仿真器”（ICE，in-circuit emulator）或者是最新设计中的嵌入式调试器。
       <span style="color:#ff0000">
        这个调试工具是开发嵌入式程序的基本技巧。
       </span>
       它代替微处理器或者嵌入微处理器内部，提供了在系统中快速调用和调试试验代码的便捷工具。一个焊点通常就是一个插入系统的特殊电路，通常使用一台连结到这些焊点的个人计算机作为调试界面。
      </li>
      <li style="margin-bottom:0.1em">
       连结器通常是各种各样（
       <a class="extiw" href="http://en.wikipedia.org/wiki/exotic" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:exotic">
        exotic
       </a>
       ）。对于大多数商业编程来说，连接器几乎总是最后才想起的（
       <a class="extiw" href="http://en.wikipedia.org/wiki/afterthough" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:afterthough">
        afterthough
       </a>
       ）部分，缺省设置也从来不变。与此相反，
       <span style="color:#ff0000">
        嵌入式连结器有完整、复杂的命令行语言是很普通的。
       </span>
       经常有不
       <span style="color:#ff0000">
        同类型的内存，分别保存特殊的代码和数据。
       </span>
       单独的数据结构能够放在特殊的地址，这样
       <span style="color:#ff0000">
        软件能够很方便地访问映射到内存的控制寄存器
       </span>
       。嵌入式连结器经常有用于减小代码大小和运行时间的外部（
       <a class="extiw" href="http://en.wikipedia.org/wiki/exotic" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:exotic">
        exotic
       </a>
       ）优化工具。例如，他们可能移动子程序的位置以使用较小的调用和跳转指令。它们经常带有管理
       <a class="extiw" href="http://en.wikipedia.org/wiki/data_overlays" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:data overlays">
        data overlays
       </a>
       和
       <a class="extiw" href="http://en.wikipedia.org/wiki/band_switch" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:band switch">
        band switch
       </a>
       技术的特性，这些技术是在嵌入式软件经常使用的扩展廉价CPU的方法。
      </li>
      <li style="margin-bottom:0.1em">
       另外一个常用的工具是一个在程序中添加代码和或者
       <a class="extiw" href="http://en.wikipedia.org/wiki/CRC" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:CRC">
        CRC
       </a>
       的工具程序（经常是自己写的），使用这个工具嵌入式系统能够在执行程序之前先进行程序数据检查。
      </li>
      <li style="margin-bottom:0.1em">
       为
       <a href="http://zh.wikipedia.org/wiki/%E6%95%B0%E5%AD%97%E4%BF%A1%E5%8F%B7%E5%A4%84%E7%90%86" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="数字信号处理">
        数字信号处理
       </a>
       开发软件的嵌入式程序员经常使用
       <a class="extiw" href="http://en.wikipedia.org/wiki/MathCad" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:MathCad">
        MathCad
       </a>
       或者
       <a class="extiw" href="http://en.wikipedia.org/wiki/Mathematica" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:Mathematica">
        Mathematica
       </a>
       这样的数学工具进行数学仿真。
      </li>
      <li style="margin-bottom:0.1em">
       一些较少使用的工具有将数据文件转换成代码的工具，使用这种工具就可以在程序中包含任意类型的数据。
      </li>
      <li style="margin-bottom:0.1em">
       少数一些项目为了特殊的可靠性或者
       <a href="http://zh.wikipedia.org/wiki/%E6%95%B0%E5%AD%97%E4%BF%A1%E5%8F%B7%E5%A4%84%E7%90%86" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="数字信号处理">
        数字信号处理
       </a>
       要求使用
       <a class="extiw" href="http://en.wikipedia.org/wiki/Synchronous_programming_language" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:Synchronous programming language">
        同步编程语言
       </a>
       。
      </li>
     </ul>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      一些编程语言为嵌入式系统编程提供了一些特殊支持。
     </p>
     <ul style="line-height:1.5em; margin:0.3em 0px 0px 1.6em; padding:0px; font-family:sans-serif; font-size:15px">
      <li style="margin-bottom:0.1em">
       对于
       <a href="http://zh.wikipedia.org/wiki/C%E8%AF%AD%E8%A8%80" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="C语言">
        C语言
       </a>
       ，
       <a class="extiw" href="http://en.wikipedia.org/wiki/ISO/IEC_TR_18037:2005" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:ISO/IEC TR 18037:2005">
        ISO/IEC TR 18037:2005
       </a>
       定义了
       <ul style="line-height:1.5em; list-style-type:disc; margin:0.3em 0px 0px 1.6em; padding:0px">
        <li style="margin-bottom:0.1em">
         指定的地址空间
        </li>
        <li style="margin-bottom:0.1em">
         指定的存储类
        </li>
        <li style="margin-bottom:0.1em">
         基本输入输出的硬件寻址
        </li>
       </ul>
      </li>
     </ul>
     <h4 style="margin:0px 0px 0.3em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-style:none; font-size:19px; font-family:sans-serif; line-height:19.1875px">
      <span class="mw-headline" id=".E8.B0.83.E8.AF.95">
       调试
      </span>
     </h4>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      调试通常使用内部电路仿真器或者其他一些能够在单片机
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/Microcode" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Microcode">
       微码
      </a>
      （
      <a class="extiw" href="http://en.wikipedia.org/wiki/microcode" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:microcode">
       microcode
      </a>
      ）内部产生
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E4%B8%AD%E6%96%AD" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="中断">
       中断
      </a>
      的调试器。微码中断让调试器能够在只有CPU工作的硬件中进行操作，基于CPU的调试器能够从CPU的角度来测试和调试计算机的电路。
      <a href="http://zh.wikipedia.org/wiki/PDP-11" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="PDP-11">
       PDP-11
      </a>
      开创了这种特性的先河。
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      开发人员能够仍然使用
      <a href="http://zh.wikipedia.org/wiki/%E6%96%AD%E7%82%B9" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="断点">
       断点
      </a>
      、
      <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E5%8D%95%E6%AD%A5%E6%89%A7%E8%A1%8C&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="单步执行（页面不存在）">
       单步执行
      </a>
      以及高级语言进行调试，在许多的调试工具上都有这种能力。另外开发人员在调试实时事件顺序的时候需要记录、使用简单的记录工具。
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      首先遇到这种问题的个人电脑和大型机程序员经常在设计优先级和可行方法的时候感到困惑。指导、代码审查和非个人风格（
      <a class="extiw" href="http://en.wikipedia.org/wiki/egoless" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(102,51,102)" target="_blank" title="en:egoless">
       egoless
      </a>
      ）的编程是值得推荐的。
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      随着嵌入式系统变得越来越复杂，更高层次的工具和操作系统逐渐移植到可行的设备上。例如，
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%9C%82%E7%AA%9D%E7%94%B5%E8%AF%9D" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="蜂窝电话">
       蜂窝电话
      </a>
      、
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E4%B8%AA%E4%BA%BA%E6%95%B0%E5%AD%97%E5%8A%A9%E7%90%86" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="个人数字助理">
       个人数字助理
      </a>
      和其他的消费用计算机需要一些从个人或者这些电子设备制造商之外的公司购买或者提供的一些重要软件。在这些系统中，需要如
      <a href="http://zh.wikipedia.org/wiki/Linux" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Linux">
       Linux
      </a>
      、
      <a href="http://zh.wikipedia.org/wiki/OSGi" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="OSGi">
       OSGi
      </a>
      或者
      <a href="http://zh.wikipedia.org/wiki/Java" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Java">
       Java
      </a>
      这样的开放编程环境，这样第三方软件提供上才能够在大规模的市场上销售软件。
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      大多数这样的开发环境都有一个运行在
      <a href="http://zh.wikipedia.org/wiki/%E4%B8%AA%E4%BA%BA%E7%94%B5%E8%84%91" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="个人电脑">
       个人电脑
      </a>
      上的参考设计，这种软件的绝大部分都可以在传统的个人电脑上开发。然而，从开放环境移植到专用的电子设备和电子设备的驱动程序开发通常仍然是传统的嵌入式系统软件工程师的工作。在有些情况下，the engineer works for the integrated circuit manufacturer, but there is still such a person somewhere.
     </p>
     <h4 style="margin:0px 0px 0.3em; overflow:hidden; padding-top:0.5em; padding-bottom:0.17em; border-bottom-style:none; font-size:19px; font-family:sans-serif; line-height:19.1875px">
      <span class="mw-headline" id=".E6.93.8D.E4.BD.9C.E7.B3.BB.E7.BB.9F">
       操作系统
      </span>
     </h4>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em">
      嵌入式系统经常没有
      <a href="http://zh.wikipedia.org/wiki/%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="操作系统">
       操作系统
      </a>
      、专用的
      <a href="http://zh.wikipedia.org/wiki/%E5%B5%8C%E5%85%A5%E5%BC%8F%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="嵌入式操作系统">
       嵌入式操作系统
      </a>
      （经常是
      <a href="http://zh.wikipedia.org/wiki/%E5%AE%9E%E6%97%B6%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="实时操作系统">
       实时操作系统
      </a>
      ）或者指定程序员移植到这些新系统。
     </p>
     <div>
      <br/>
     </div>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      <span style="font-size:19px; line-height:19.1875px">
       <strong>
        启动
       </strong>
      </span>
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      <span style="line-height:1.5em">
       嵌入式系统带有启动代码，通常它禁止中断、设置电子设备参数、测试计算机（RAM、CPU和软件），然后开始应用程序运行。许多嵌入式系统从短暂的掉电状态恢复，经常重起而不进行最近的自检。在十分之一秒内重起是常见的现象。
      </span>
     </p>
     <p style="margin-top:0.4em; margin-bottom:0.5em; line-height:1.5em; font-family:sans-serif; font-size:15px">
      许多设计人员发现
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/Light-emitting_diode" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="Light-emitting diode">
       LED
      </a>
      在指示错误状态上非常有用，它们可以帮助进行
      <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E6%95%85%E9%9A%9C%E5%A4%84%E7%90%86&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(165,88,88)" target="_blank" title="故障处理（页面不存在）">
       故障处理
      </a>
      。一个常用的机制是在复位的时候点亮电子设备所有的LED以表明供电和LED正常工作；然后在进行
      <a href="http://zh.wikipedia.org/wiki/%E5%8A%A0%E7%94%B5%E8%87%AA%E6%A3%80" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(11,0,128)" target="_blank" title="加电自检">
       加电自检
      </a>
      时由软件改变LED的状态；在此之后，软件用这些LED指示操作过程中的正常或者故障状态。这可以让技术人员、工程师和用户了解系统的状态。
     </p>
