     ARM SVC指令

    <p>
     在 ARM 汇编中，
     <code>
      SVC
     </code>
     （Supervisor Call）指令用于从用户模式切换到特权模式（如 Supervisor 模式），以便执行操作系统内核提供的服务。它通常用于系统调用。
    </p>
    <h4>
     <a id="_2">
     </a>
     具体作用
    </h4>
    <ol>
     <li>
      <strong>
       触发异常
      </strong>
      ：执行
      <code>
       SVC
      </code>
      指令时，处理器会进入 Supervisor 模式，并跳转到异常向量表中的
      <code>
       SVC
      </code>
      异常处理程序。
     </li>
     <li>
      <strong>
       传递参数
      </strong>
      ：
      <code>
       SVC
      </code>
      指令后的立即数可作为参数传递给异常处理程序，帮助识别具体的系统调用。
     </li>
    </ol>
    <h4>
     <a id="_6">
     </a>
     语法
    </h4>
    <pre><code class="prism language-assembly">SVC #&lt;imm&gt;
</code></pre>
    <ul>
     <li>
      <code>
       &lt;imm&gt;
      </code>
      是一个 24 位的立即数，用于传递参数。
     </li>
    </ul>
    <h4>
     <a id="_12">
     </a>
     示例
    </h4>
    <pre><code class="prism language-assembly">MOV R0, #4       @ 将参数 4 放入 R0
SVC #0           @ 调用系统调用，立即数 0 表示具体调用
</code></pre>
    <h4>
     <a id="_18">
     </a>
     处理流程
    </h4>
    <ol>
     <li>
      <strong>
       执行
       <code>
        SVC
       </code>
       指令
      </strong>
      ：处理器切换到 Supervisor 模式，并跳转到
      <code>
       SVC
      </code>
      异常处理程序。
     </li>
     <li>
      <strong>
       读取参数
      </strong>
      ：处理程序读取
      <code>
       SVC
      </code>
      指令后的立即数，确定具体操作。
     </li>
     <li>
      <strong>
       执行服务
      </strong>
      ：根据参数执行相应的内核服务。
     </li>
     <li>
      <strong>
       返回用户模式
      </strong>
      ：服务完成后，处理器返回用户模式并继续执行后续指令。
     </li>
    </ol>
    <h4>
     <a id="_24">
     </a>
     总结
    </h4>
    <p>
     <code>
      SVC
     </code>
     指令用于系统调用，允许用户程序请求操作系统服务。通过立即数传递参数，操作系统根据参数执行相应操作。
    </p>
