    <p>
     第一课：
    </p>
    <p>
     服务端渲染，全栈
    </p>
    <p>
     node版本20
    </p>
    <p>
     npx create-next-app@latest创建一个next项目
    </p>
    <p>
     npm run dev 运行，  npm run build打包 打包后会生成.next文件夹 （最好把本地的dev运行关掉），
     <span style="color:#fe2c24">
      npm start会本地运行打包后的包
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      第二课
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      文件路由
     </span>
     <span style="color:#0d0016">
      这里的page类似于index.js文件名，默认的
     </span>
    </p>
    <p>
     <img alt="" height="299" src="https://i-blog.csdnimg.cn/direct/9201c09fccac428fa83c8f46959220cf.png" width="344"/>
    </p>
    <p>
     在vscode中的插件市场安装es7 在空白的页面输入rfc会生成函数式代码
    </p>
    <p>
     第三课
    </p>
    <p>
     layout（布局）在多个路由切换共享的ui
    </p>
    <p>
     1、root layout根布局,每个项目必须要有个html，其他的业务代码会塞入到children里面
    </p>
    <p>
     src\app\layout.tsx
    </p>
    <pre><code>&lt;html lang="en"&gt;
      &lt;body
        className={`${geistSans.variable} ${geistMono.variable} antialiased`}
      &gt;
        这是根layout
        {children}
      &lt;/body&gt;
    &lt;/html&gt;</code></pre>
    <p>
     2、普通的layout,不需要html，所有的同目录的文件都会塞入到这个layOut中的children中去
    </p>
    <p>
     src\app\dashboard\layout.tsx
    </p>
    <pre><code>import React from 'react'

export default function dashboardlayout({ children }) {
  return (
    &lt;div&gt;
      这是dashboardlayout
      {children}
    &lt;/div&gt;
  )
}
</code></pre>
    <p>
     第四课
    </p>
    <p>
     template.tsx  在根组件layout.tsx文件同级新建template.tsx
     <span style="color:#fe2c24">
      他是在layout和业务组件中间的一层
     </span>
    </p>
    <p>
     src\app\template.tsx
    </p>
    <pre><code>export default function Template({ children }: React.ReactNode) {
  return &lt;div&gt;
    这是template组件
    {children}&lt;/div&gt;
}</code></pre>
