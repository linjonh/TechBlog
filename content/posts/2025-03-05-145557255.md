    <h2>
     一.SSH基础
    </h2>
    <h3 style="background-color:transparent">
     1.1 什么是ssh
    </h3>
    <p>
     SSH（Secure Shell）协议是一种用于字符界面
     <span style="color:#fe2c24">
      <strong>
       远程登录
      </strong>
     </span>
     和
     <span style="color:#fe2c24">
      <strong>
       数据加密
      </strong>
     </span>
     传输的协议。
    </p>
    <h3>
     1.2 ssh优点
    </h3>
    <p>
     <span style="color:#333333">
      <strong>
       <span style="color:#ff0000">
        <span style="color:inherit">
         优点：
        </span>
       </span>
      </strong>
     </span>
    </p>
    <ul style="margin-left:.8em">
     <li>
      <p>
       数据传输是
       <span style="color:#fe2c24">
        <strong>
         加密
        </strong>
       </span>
       的，可以
       <strong>
        <span style="color:#fe2c24">
         防止信息泄漏
        </span>
       </strong>
      </p>
     </li>
     <li>
      <p>
       数据传输是
       <span style="color:#fe2c24">
        <strong>
         压缩
        </strong>
       </span>
       的，可以提高
       <strong>
        <span style="color:#fe2c24">
         传输速度
        </span>
       </strong>
      </p>
     </li>
    </ul>
    <p>
     <strong>
      注意：telnet对比ssh    1.没有压缩功能 2.明文传输数据不安全
     </strong>
    </p>
    <h3>
     1.3 ssh原理
    </h3>
    <h4>
     1.公钥传输原理
    </h4>
    <p>
     <img alt="" height="583" src="https://i-blog.csdnimg.cn/direct/6e4396f487a34f8985f03c91befdff73.png" width="903"/>
    </p>
    <ul>
     <li>
      <p>
       1.客户端发起链接请求
      </p>
     </li>
     <li>
      <p>
       2.服务端返回自己的公钥，以及一个会话ID（这一步客户端得到服务端公钥）
      </p>
     </li>
     <li>
      <p>
       3.客户端生成密钥对
      </p>
     </li>
     <li>
      <p>
       4.客户端用自己的公钥异或会话ID，计算出一个值Res，并用服务端的公钥加密
      </p>
     </li>
     <li>
      <p>
       5.客户端发送加密值到服务端，服务端用私钥解密，得到Res
      </p>
     </li>
     <li>
      <p>
       6.服务端用解密后的值Res异或会话ID，计算出客户端的公钥（这一步服务端得到客户端公钥）
      </p>
     </li>
     <li>
      <p>
       7.最终：双方各自持有三个秘钥，分别为自己的一对公、私钥，以及对方的公钥，之后的所有通讯都会被加密
      </p>
     </li>
    </ul>
    <h4>
     2.ssh加密传输原理
     <img alt="" height="299" src="https://i-blog.csdnimg.cn/direct/82e2bce772e149efa02d0540d6aaf0a2.png" width="1059"/>
    </h4>
    <p>
     1.对称加密：采用单钥密码系统的加密方法，同一个密钥可以同时用作信息的加密和解密，这种加密方法称为对称加密
    </p>
    <p>
     2.非对称加密：非对称加密算法需要两个密钥：公开密钥（publickey:简称公钥）和私有密钥（privatekey:简称私钥）。公钥与私钥是一对，如果用公钥对数据进行加密，只有用对应的私钥才能解密。因为加密和解密使用的是两个不同的密钥，所以这种算法叫作非对称加密算法。
    </p>
    <p>
     <span style="color:#fe2c24">
      <strong>
       非对称加密原理（重点！！）
      </strong>
     </span>
     ：首先ssh通过加密算法在客户端产生密钥对（公钥和私钥），公钥发送给服务器端，自己保留私钥，如果要想连接到带有公钥的SSH服务器，客户端SSH软件就会向SSH服务器发出请求，请求用联机的用户密钥进行安全验证。SSH服务器收到请求之后，会先在该SSH服务器上连接的用户的家目录下
    </p>
    <h3>
     1.4 ssh验证方式
    </h3>
    <p>
     1.
     <strong>
      密码认证
     </strong>
     ：通过
     <strong>
      <span style="color:#fe2c24">
       用户名
      </span>
     </strong>
     和
     <strong>
      <span style="color:#fe2c24">
       密码登录
      </span>
     </strong>
    </p>
    <p>
     2.
     <strong>
      密钥认证
     </strong>
     ：使用密钥（
     <strong>
      <span style="color:#fe2c24">
       公钥和私钥
      </span>
     </strong>
     ）进行
     <span style="color:#fe2c24">
      <strong>
       无密码登录
      </strong>
     </span>
    </p>
    <h3>
     1.5 ssh工作方式
    </h3>
    <p>
     <img alt="" height="213" src="https://i-blog.csdnimg.cn/direct/5e1da4efc7a84aa7b3623349e23579ea.png" width="867"/>
    </p>
    <h3>
     1.6 ssh登录方式
    </h3>
    <pre><code class="language-bash">第一种方法：ssh [远程主机用户名]@[远程服务器主机名或IP地址] -p port:端口

第二种方法：ssh  -l [远程主机用户名] [远程服务器主机名或IP地址] -p port:端口
 
注意： -p ：当服务端的端口非默认时，需要使用-p 指定端口号进行登录
       </code></pre>
    <p>
     注意：
     <span style="color:#fe2c24">
      <strong>
       <code>
        -t
       </code>
      </strong>
     </span>
     选项的作用就是强制 SSH 分配一个伪终端给远程会话,适用于跳板机连接。
    </p>

    <h3>
     2.1 常用文件
    </h3>
    <pre><code class="language-python">OpenSSH是以 SSH协议开发的软件，提供sshd服务

常用配置文件有两个/etc/ssh/ssh_config 和/etc/sshd_config。

ssh_config：为客户端配置文件，设置与客户端相关的应用可通过此文件实现

sshd_config：为服务器端配置文件，设置与服务端相关的应用可通过此文件实现

服务端配置文件：/etc/ssh/sshd_config
