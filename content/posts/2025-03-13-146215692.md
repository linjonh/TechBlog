    <h2>
     <a id="_0">
     </a>
     设计模式概述
    </h2>
    <p>
     设计模式是一套被广泛接受的、经过试验的、可反复使用的基于面向对象的软件设计经验总结，它是开发人员在软件设计时，对常见问题的解决方案的总结和抽象。
     <br/>
     一句话就是，设计模式是针对软件开发中常见问题和模式的通用解决方案。
    </p>
    <h3>
     <a id="_3">
     </a>
     设计模式有哪些
    </h3>
    <ol>
     <li>
      <mark>
       GoF设计模式
      </mark>
      ：最常用的，四人组（Gang of Four）
      <br/>
      单例模式是其23中设计模式之一
     </li>
     <li>
      架构设计模式
     </li>
     <li>
      企业级设计模式
     </li>
     <li>
      领域驱动设计模式
     </li>
     <li>
      并发设计模式
     </li>
     <li>
      数据访问模式
     </li>
    </ol>
    <h3>
     <a id="GoF_12">
     </a>
     GoF设计模式的分类
    </h3>
    <ol>
     <li>
      创建型：解决对象的创建问题
     </li>
     <li>
      结构型：通过设计和构建对象之间的关系，以达到更好的重用性、扩展性和灵活性
     </li>
     <li>
      行为型：主要用于处理对象之间的算法和责任分配
     </li>
    </ol>
    <h3>
     <a id="singleton_17">
     </a>
     单例模式singleton
    </h3>
    <p>
     单例模式属于创建型的设计模式
    </p>
    <h4>
     <a id="_19">
     </a>
     饿汉式单例模式
    </h4>
    <p>
     如何实现饿汉式单例模式？
    </p>
    <ol>
     <li>
      构造方法私有化
     </li>
     <li>
      对外提供公开的静态方法，获取实例
     </li>
     <li>
      定义一个静态变量，接收类的实例化对象。静态变量在类加载时初始化，只执行一次，从而达到只创建一个实例的目的。
     </li>
    </ol>
    <p>
     由于饿汉式单例模式，在类加载时就创建出来对象，无论用于不用对象都创建出来了，称为饿汉单例模式。
