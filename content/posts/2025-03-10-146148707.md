    <p>
     以下是关于强缓存和协商缓存的面试题总结：
    </p>
    <hr/>
    <h4>
     <a id="1__4">
     </a>
     1. 强缓存和协商缓存的定义
    </h4>
    <ul>
     <li>
      <strong>
       强缓存
      </strong>
      ：当浏览器请求资源时，直接从本地缓存中获取资源，而无需向服务器发送请求。强缓存的实现主要依赖于 HTTP 响应头中的
      <code>
       Cache-Control
      </code>
      和
      <code>
       Expires
      </code>
      字段。
     </li>
     <li>
      <strong>
       协商缓存
      </strong>
      ：当强缓存失效后，浏览器会向服务器发送请求，携带缓存标识（如
      <code>
       Last-Modified
      </code>
      或
      <code>
       ETag
      </code>
      ），服务器根据这些标识判断资源是否发生变化。如果资源未变化，服务器返回
      <code>
       304 Not Modified
      </code>
      状态码，浏览器继续使用本地缓存。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2__10">
     </a>
     2. 强缓存和协商缓存的实现方式
    </h4>
    <ul>
     <li>
      <strong>
       强缓存
      </strong>
      ：
      <ul>
       <li>
        <code>
         Expires
        </code>
        ：HTTP 1.0 中的字段，指定资源的绝对过期时间。
       </li>
       <li>
        <code>
         Cache-Control
        </code>
        ：HTTP 1.1 中的字段，提供了更灵活的缓存策略，如
        <code>
         max-age
        </code>
        （缓存有效期）、
        <code>
         no-cache
        </code>
        （不直接使用强缓存，需协商验证）、
        <code>
         no-store
        </code>
        （不使用缓存）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       协商缓存
      </strong>
      ：
      <ul>
       <li>
        <code>
         Last-Modified
        </code>
        和
        <code>
         If-Modified-Since
        </code>
        ：服务器返回资源的最后修改时间，浏览器在后续请求中携带该时间戳进行验证。
       </li>
       <li>
        <code>
         ETag
        </code>
        和
        <code>
         If-None-Match
        </code>
        ：服务器返回资源的唯一标识符，浏览器在后续请求中携带该标识符进行验证。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3__20">
     </a>
     3. 强缓存和协商缓存的区别
    </h4>
    <ul>
     <li>
      <strong>
       请求过程
      </strong>
      ：
      <ul>
       <li>
        强缓存：直接从本地缓存获取资源，不发送请求。
       </li>
       <li>
        协商缓存：需要向服务器发送请求，由服务器决定是否使用缓存。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：
      <ul>
       <li>
        强缓存适用于静态资源（如图片、CSS、JS 文件），这些资源不经常更新。
       </li>
       <li>
        协商缓存适用于动态资源或更新频繁的资源，能够及时获取最新内容。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       响应状态码
      </strong>
      ：
      <ul>
       <li>
        强缓存生效时，响应状态码为
        <code>
         200
        </code>
        （直接使用缓存）。
       </li>
       <li>
        协商缓存生效时，响应状态码为
        <code>
         304 Not Modified
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="4__33">
     </a>
     4. 强缓存和协商缓存的优先级
    </h4>
    <ul>
     <li>
      强缓存的优先级高于协商缓存。如果强缓存生效，浏览器不会发送请求到服务器；如果强缓存失效，才会进入协商缓存阶段。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="5__38">
     </a>
     5. 如何设置强缓存和协商缓存
    </h4>
    <ul>
     <li>
      <strong>
       强缓存
      </strong>
      ：
      <ul>
       <li>
        <code>
         Cache-Control: max-age=3600
        </code>
        （资源缓存有效期为 1 小时）。
       </li>
       <li>
        <code>
         Expires: Wed, 21 Jul 2023 17:00:00 GMT
        </code>
        （资源过期时间为指定时间）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       协商缓存
      </strong>
      ：
      <ul>
       <li>
        服务器响应头设置
        <code>
         Last-Modified
        </code>
        和
        <code>
         ETag
        </code>
        。
       </li>
       <li>
        浏览器请求头携带
        <code>
         If-Modified-Since
        </code>
        和
        <code>
         If-None-Match
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="6__48">
     </a>
     6. 注意事项
    </h4>
    <ul>
     <li>
      <strong>
       强缓存
      </strong>
      ：如果服务器时间设置不准确，可能导致缓存失效。
     </li>
     <li>
      <strong>
       协商缓存
      </strong>
      ：虽然需要与服务器通信，但减少了不必要的数据传输。
     </li>
    </ul>
