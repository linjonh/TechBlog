    <p>
     编译环境：Python-3.6.2 、Ubuntu18.04、移远交叉编译工具链（arm-oe-linux-gnueabi-xxx）
     <br/>
     由于ubuntu18.04自带了python3.6，所以决定采用该版本进行交叉编译。
    </p>
    <h4>
     <a id="1_2">
     </a>
     1.解压安装包
    </h4>
    <pre><code>tar zxvf Python-3.6.2.tgz
</code></pre>
    <h4>
     <a id="2_4">
     </a>
     2.创建安装目录
    </h4>
    <pre><code>cd Python-3.6.2/            				#进入源码目录
mkdir arm_python_build		#创建安装目录
</code></pre>
    <h4>
     <a id="3configureMakefile_7">
     </a>
     3.配置configure、生成Makefile
    </h4>
    <pre><code>1.配置configure:
./configure --host=arm-oe-linux --build=arm  --prefix=[安装目录的绝对路径]
</code></pre>
    <p>
     错误提示1，找不到交叉编译工具链：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1be8434958673a847e89d3976b7f1bd3.png">
      <br/>
      为了解决这一问题，配置ec20编译环境。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1a69aecac85711b664e97c9ba9d5cf54.png">
       <br/>
       结果出现错误提示2，找不到python：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/68bce5b55fefe4299c8a25fdd3d2c74c.png">
        <br/>
        这是因为ec20的sdk环境初始化脚本，改变了默认编译器导致的。
       </img>
      </img>
     </img>
    </p>
    <p>
     <strong>
      解决办法：
     </strong>
     <br/>
     1.在环境变量路径下创建交叉编译工具链的软链接。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/296b4b4693683bb082a1c2a6fe234834.png">
      <br/>
      2.配置时加上sudo权限，并增加相应的编译选项。
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> ./configure --host<span class="token operator">=</span>arm-linux --build<span class="token operator">=</span>arm  --prefix<span class="token operator">=</span><span class="token punctuation">[</span>安装目录的绝对路径<span class="token punctuation">]</span> --disable-ipv6 <span class="token assign-left variable">ac_cv_file__dev_ptmx</span><span class="token operator">=</span>no <span class="token assign-left variable">ac_cv_file__dev_ptc</span><span class="token operator">=</span>no --enable-optimizations
</code></pre>
    <p>
     配置完成，会生成用于交叉编译的Makefile
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6623ffc400a19c6275415ee3199b2909.png"/>
    </p>
    <h4>
     <a id="3_29">
     </a>
     3.编译与安装
    </h4>
    <pre><code>make &amp;&amp; make install
</code></pre>
    <p>
     编译时可以看到，编译使用的工具链为指定的交叉编译工具链：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/68ed5695e5751ca9904855dad7b81d14.png"/>
    </p>
    <p>
     执行完毕后进入之前创建的安装目录即可看到：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/197b762fb7879ce8cb0b7e54cbfb2721.png"/>
    </p>
    <p>
     将bin目录下交叉编译的python解释器下载至EC20中即可使用。
    </p>
    <p>
     移远SDK下载链接：https://pan.baidu.com/s/18gVzGjaBUorwJEM8Ee6hCA
     <br/>
     提取码：vuti
    </p>
