      <p>
       折腾了一上午，遇到的各种错误：
      </p>
      <h6>
       错误一：安装后，终端执行 mysql 或者执行
       <code>
        mysql -u root -p
       </code>
       时报错：
      </h6>
      <pre><code class="shell">ERROR 1045 (28000): Access denied for user 'root'@'localhost' (using password: YES)
</code></pre>
      <h6>
       错误二：为解决错误一， 尝试执行
      </h6>
      <pre><code class="shell">mysqld --user=root --skip-grant-tables &amp;
</code></pre>
      <p>
       报错：
      </p>
      <pre><code class="shell">2020-12-03T02:43:04.627716Z 0 [System] [MY-010116] [Server] /usr/local/mysql-8.0.22-macos10.15-x86_64/bin/mysqld (mysqld 8.0.22) starting as process 10786
2020-12-03T02:43:04.630514Z 0 [Warning] [MY-010091] [Server] Can't create test file /usr/local/mysql-8.0.22-macos10.15-x86_64/data/mysqld_tmp_file_case_insensitive_test.lower-test
2020-12-03T02:43:04.630524Z 0 [Warning] [MY-010159] [Server] Setting lower_case_table_names=2 because file system for /usr/local/mysql-8.0.22-macos10.15-x86_64/data/ is case insensitive
2020-12-03T02:43:04.630895Z 0 [Warning] [MY-010122] [Server] One can only use the --user switch if running as root
2020-12-03T02:43:04.630938Z 0 [ERROR] [MY-013276] [Server] Failed to set datadir to '/usr/local/mysql-8.0.22-macos10.15-x86_64/data/' (OS errno: 13 - Permission denied)
2020-12-03T02:43:04.630991Z 0 [ERROR] [MY-010119] [Server] Aborting
2020-12-03T02:43:04.631143Z 0 [System] [MY-010910] [Server] /usr/local/mysql-8.0.22-macos10.15-x86_64/bin/mysqld: Shutdown complete (mysqld 8.0.22)  MySQL Community Server - GPL.
</code></pre>
      <p>
       尝试了查到的各个方法无果。
      </p>
      <h6>
       错误三：还是为了解决错误一，尝试：
      </h6>
      <pre><code class="shell">cd /usr/local/mysql/bin
sudo su 
输入开机密码 回车
sh-3.2# ./mysqld_safe --skip-grant-tables &amp;
</code></pre>
      <p>
       报错：
      </p>
      <pre><code class="shell">[3] 1602
sh-3.2# 2020-12-03T03:44:22.6NZ mysqld_safe Logging to '/usr/local/mysql-8.0.22-macos10.15-x86_64/data/ing-3.local.err'.
2020-12-03T03:44:22.6NZ mysqld_safe Starting mysqld daemon with databases from /usr/local/mysql-8.0.22-macos10.15-x86_64/data
mysql2020-12-03T03:44:26.6NZ mysqld_safe mysqld from pid file /usr/local/mysql-8.0.22-macos10.15-x86_64/data/ing-3.local.pid ended
2020-12-03T03:46:02.6NZ mysqld_safe mysqld from pid file /usr/local/mysql-8.0.22-macos10.15-x86_64/data/ing-3.local.pid ended
</code></pre>
      <h6>
       错误四：偶然发现，mysql服务一会自动开启一会自动关闭，查看原因的话，可以在偏好设置里的mysql下点选Configuration 查看Error Log
      </h6>
      <div class="image-package">
       <img alt="error log.png" src="https://i-blog.csdnimg.cn/img_convert/1daeb92a26935959aedabc949ee1c0ed.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         error log.png
        </div>
       </img>
      </div>
      <br/>
      <p>
       如果发现类似下面的错误，说明有其他的mysql服务在占用端口，kill掉即可，或者简单粗暴的重启电脑。
      </p>
      <br/>
      <div class="image-package">
       <div class="image-container" style="max-width: 700px; max-height: 430px;">

      <p>
       最后，在抓狂的边缘，我又又又劝自己再尝试重装一次mysql...没想到成了。
       <br/>
       在重装之前，一定要确保将已安装的mysql彻底卸载干净，如何卸载看
       <a href="https://www.jianshu.com/p/57d3eb7412a9" rel="nofollow noopener noreferrer" target="_blank">
        这篇
       </a>
      </p>
      <h6>
       一.
       <a href="https://links.jianshu.com/go?to=https%3A%2F%2Fdev.mysql.com%2Fdownloads%2Fmysql%2F" rel="nofollow noopener noreferrer" target="_blank">
        官网下载
       </a>
      </h6>
      <div class="image-package">
       <img alt="版本选择.png" src="https://i-blog.csdnimg.cn/img_convert/34397e9268b94c5e185d002f7406ac1d.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         版本选择.png
        </div>
       </img>
      </div>
      <p>
       选择第一个就好了。
      </p>
      <h6>
       二.下载完成，开始安装 一路继续 ... 多图预警！！！
      </h6>
      <div class="image-package">
       <img alt="1.png" src="https://i-blog.csdnimg.cn/img_convert/24f20d4268e632fdb374e98bd8a90802.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         1.png
        </div>
       </img>
      </div>
      <h6>
       三. 选择密码加密类型 ，选第一个就行。
       <div class="image-package">
        <img alt="2.png" src="https://i-blog.csdnimg.cn/img_convert/ddf6555de204b309cd00dcb128051938.webp?x-oss-process=image/format,png">
         <div class="image-caption">
          2.png
         </div>
        </img>
       </div>
       四. 为root用户定义一个密码，这个要记住！以后要用！
      </h6>
      <div class="image-package">
       <img alt="3.png" src="https://i-blog.csdnimg.cn/img_convert/c01138e47f2ffe373e7f816ea7db7898.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         3.png
        </div>
       </img>
      </div>
      <br/>
      <p>
       剩下的一路点继续，完成！
      </p>
      <p>
       完成上述操作后，这时候应该在偏好设置里能看到mysql服务小海豚已经有了，进入，开启服务后，重点来了！这里有一个初始化操作
       <code>
        initialize Database
       </code>
       <br/>
       你最好
       <code>
        点一下
       </code>
       ！不然后续可能有麻烦事，比如上面的错误三
       <br/>
      </p>
      <div class="image-package">
       <img alt="4.png" src="https://i-blog.csdnimg.cn/img_convert/3c729058b5c3acd50dcdbfa2680ed28b.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         4.png
        </div>
       </img>
      </div>
      <br/>
      弹窗提示输入你在安装时设置的密码，ok即可。
      <br/>
      <div class="image-package">
       <img alt="5.png" src="https://i-blog.csdnimg.cn/img_convert/2a224fa653ac90f9cb6a1fc375e59f84.webp?x-oss-process=image/format,png">
        <div class="image-caption">
         5.png
        </div>
       </img>
      </div>
      <br/>
      然后重新开启mysql服务并打开终端，输入 mysql -u root -p , 输入刚才设置的密码，终于来了~~
