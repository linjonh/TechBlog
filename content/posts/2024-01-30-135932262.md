    <h3>
     <a id="Django_3">
     </a>
     一、安装Django
    </h3>
    <p>
     安装步骤非常简单，使用pip安装就行
    </p>
    <pre><code>pip install django
</code></pre>
    <p>
     安装完成之后，python的
     <code>
      Scripts
     </code>
     文件夹下，会多一个
     <code>
      django-admin.exe
     </code>
     (管理创建django项目的工具)。
     <code>
      Lib
     </code>
     文件夹下
     <code>
      site-packages
     </code>
     下，会出现django源码。
     <br/>
     使用下面的命令可以查看django版本号。
    </p>
    <pre><code>python -m django --version
</code></pre>
    <p>
     显示：
    </p>
    <pre><code>5.0.1
</code></pre>
    <h3>
     <a id="web_17">
     </a>
     二、创建与开发web项目
    </h3>
    <p>
     这里记录两种创建方式，一种是命令行方式创建，一种是通过pycharm方式创建。
    </p>
    <h4>
     <a id="11__19">
     </a>
     1.1 通过命令行方式
    </h4>
    <p>
     打开终端，进入到项目将要放在的目录下，执行创建项目命令：
    </p>
    <pre><code>django-admin startproject 自定义的项目名称
</code></pre>
    <p>
     如果提示没有这个命令，有可能是没有配置环境变量，将django-admin的存放路径加入系统变量就可以了。
    </p>
    <h4>
     <a id="12_pycharm_25">
     </a>
     1.2 通过pycharm创建项目
    </h4>
    <p>
     打开pycharm，左上角“文件”，“新建项目”，左侧：Django—配置项目位置、python解析器—创建
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/461fc69b17aeea138a6d2c82535aaf58.png">
      <br/>
      最顶上的【名称】与【位置】是项目名称与项目的存放位置。一定要确定好使用的python版本，尤其是当你的的电脑里存在多个python脚本时，一定要选择你安装django的那个python版本。如果电脑里只有一个python版本，就不需要改动。在下面就是添加模板文件，和【APP】名称，点击【创建】完成创建。 记住下面红框内两个文件夹名字，之后会用到，注意区分。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5542a95ff578b457e8462ebfcd6b62c8.png"/>
     </img>
    </p>
    <p>
     创建完成后，可以看到创建了很多文件，这些都是django项目的必备文件。这里简单放上，官方的解释
    </p>
    <ul>
     <li>
      最外层的 mysite/ 根目录只是你项目的容器， 根目录名称对 Django 没有影响，你可以将它重命名为任何你喜欢的名称。
      <br/>
      manage.py: 一个让你用各种方式管理 Django 项目的命令行工具。你可以阅读 django-admin 和 manage.py 获取所有 manage.py 的细节。
     </li>
     <li>
      里面一层的 mysite/ 目录包含你的项目，它是一个纯 Python 包。它的名字就是当你引用它内部任何东西时需要用到的 Python 包名。 (比如 mysite.urls).
     </li>
     <li>
      mysite/
      <strong>
       init
      </strong>
      .py：一个空文件，告诉 Python 这个目录应该被认为是一个 Python 包。如果你是 Python 初学者，阅读官方文档中的 更多关于包的知识。
     </li>
     <li>
      mysite/settings.py：Django 项目的配置文件。如果你想知道这个文件是如何工作的，请查看 Django 配置 了解细节。
     </li>
     <li>
      mysite/urls.py：Django 项目的 URL 声明，就像你网站的“目录”。阅读 URL调度器 文档来获取更多关于 URL 的内容。
     </li>
     <li>
      mysite/asgi.py：作为你的项目的运行在 ASGI 兼容的 Web 服务器上的入口。阅读 如何使用 ASGI 来部署 了解更多细节。
     </li>
     <li>
      mysite/wsgi.py：作为你的项目的运行在 WSGI 兼容的Web服务器上的入口。阅读 如何使用 WSGI 进行部署 了解更多细节。
     </li>
    </ul>
    <p>
     到此，项目创建完成，可以简单运行一下。pycharm里点击绿色的三角形自动开启web服务。
     <br/>
     如果是命令行创建的，则在项目的根目录下，打开终端，运行
    </p>
    <pre><code>python manage.py runserver localhost:8000
</code></pre>
    <p>
     然后打开浏览器，输入网址：
    </p>
    <pre><code>http://127.0.0.1:8000/admin
</code></pre>
    <p>
     可以看到admin后台的登录页面
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9ca526aa9f2ef706f98dbda4eef400d7.png">
      <br/>
      下面放上django的官方文档，里面记载了django的所有的详细介绍
     </img>
    </p>
    <pre><code>https://docs.djangoproject.com/zh-hans/5.0/
</code></pre>
    <p>
     可以当作一部字典，用到什么需求就查阅相关部分。这个文档，我建议大家下载下来，观看，因为这个网站跟github类似，有时候访问不了。
    </p>
    <p>
     <strong>
      django中的app概念：项目和应用有什么区别？
     </strong>
    </p>
    <ul>
     <li>
      在pycharm创建项目的那张截图中，项目对应的是最上方的项目，应用对应的是下方的应用程序。
     </li>
     <li>
      应用是一个专门做某件事的网络应用程序——比如博客系统，或者公共记录的数据库，或者小型的投票程序。项目则是一个网站使用的配置和应用的集合。项目可以包含很多个应用。应用可以被很多个项目使用。
     </li>
     <li>
      刚才访问的
      <code>
       http://127.0.0.1:8000/admin
      </code>
      ，是项目的后台管理，是django项目自带的应用，到此处，我们并没有开发自己的应用程序。所以接下来要做的就是开发自己的所需要的应用。
     </li>
    </ul>
    <h4>
     <a id="2_app_62">
     </a>
     2. 创建app应用
    </h4>
    <p>
     如果是使用pycharm，按照我截图方式创建的项目，则忽略这一步，因为已经创建好了。
     <br/>
     打开终端执行
    </p>
    <pre><code>python manage.py startapp app01

补充删除app的方式：
manage.py migrate my_app_name zero 
</code></pre>
    <h4>
     <a id="3_app_71">
     </a>
     3. 注册app
    </h4>
    <p>
     如果是使用pycharm，按照我截图方式创建的项目，则忽略这一步，因为已经注册好了。
     <br/>
     在项目的setting.py文件中，找到INSTALLED_APPS节点，新增注册语句即可完成注册：
    </p>
    <pre><code>'app01.apps.App01Config'
</code></pre>
    <p>
     特别注意：如果不注册app，那么app下的模型model.py文件中的模型类，是无法创建数据表的。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b27cc0844ccdf0997b2fcf171147de57.png">
      <br/>
      这个图是我用pycharm注册的配置，与我创建项目时输入的【应用程序名称】是对应的，与上述手动输入的格式一样，只是应用名称不一样。
     </img>
    </p>
    <h4>
     <a id="4__80">
     </a>
     4. 配置数据库
    </h4>
    <p>
     如果没有数据库需求，或者想使用默认配置的，可以跳过此步骤。这个步骤实际上只是更改了数据库的存放目录。
     <br/>
     打开
     <code>
      setting.py
     </code>
     ，找到配置数据库的地方。我这里使用的是sqlite3数据库。如果想要使用别的数据库，可以去官方文档搜索相应的引擎配置。
    </p>
    <pre><code>DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
</code></pre>
    <p>
     默认的路径是在项目的根目录下的，如果有需求，可以改到别的任何地方。
    </p>
    <pre><code>DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': 'D:/Databases/personinfo/person.sqlite3',
    }
}
</code></pre>
    <h4>
     <a id="5__100">
     </a>
     5. 一些其他配置（可有可无）
    </h4>
    <p>
     打开
     <code>
      setting.py
     </code>
     <br/>
     <strong>
      5.1 设置语言为中文，设置时区为上海
     </strong>
    </p>
    <pre><code>LANGUAGE_CODE = 'zh-hans'
TIME_ZONE = 'Asia/Shanghai'
</code></pre>
    <p>
     否则可能出现中文乱码。时区一定要改，否则如果开发项目时候使用了django的获取时间API，获取的是美国时区，会比中国慢。
     <br/>
     <strong>
      5.2 设置关闭网页时，结束所有会话
     </strong>
    </p>
    <pre><code>SESSION_EXPIRE_AT_BROWSER_CLOSE = True
</code></pre>
    <p>
     <strong>
      5.3 设置静态文件夹位置
     </strong>
     <br/>
     当需要访问静态文件时，会根据设置的文件夹去寻找
    </p>
    <pre><code>STATICFILES_DIRS = [
    BASE_DIR / "static",  # 假设你的静态文件在项目的static文件夹中
]
</code></pre>
    <h4>
     <a id="6_urlurlpy_119">
     </a>
     6. 编写url和视图函数对应关系（url.py文件）
    </h4>
    <h5>
     <a id="61_url_120">
     </a>
     6.1 配置项目url
    </h5>
    <p>
     目前尾汁，可以在项目名称里找到一个
     <code>
      urls.py
     </code>
     文件，打开后内容如下。这就是我们之前的访问的admin路径配置
    </p>
    <pre><code>from django.contrib import admin
from django.urls import path
urlpatterns = [
    path('admin/', admin.site.urls),
]
</code></pre>
    <p>
     修改如下：
    </p>
    <pre><code>from django.contrib import admin
from django.urls import path, include
urlpatterns = [
    path('admin/', admin.site.urls),
    path("PasswordManage/", include("PwdMgeapp.urls")),
]
</code></pre>
    <p>
     几个注意的点：
    </p>
    <ul>
     <li>
      <code>
       PasswordManage
      </code>
      可以改为你想要的路径
     </li>
     <li>
      <code>
       PwdMgeapp
      </code>
      必须与你的应用名称相同
     </li>
     <li>
      强烈推荐使用
      <code>
       include
      </code>
      包含，这样可以防止项目有多个应用时，路径重名问题。
     </li>
    </ul>
    <h5>
     <a id="62_url_142">
     </a>
     6.2 设置应用url
    </h5>
    <p>
     在应用程序文件夹（PwdMgeapp）下，创建
     <code>
      urls.py
     </code>
     ，添加内容
    </p>
    <pre><code>from django.urls import path
from . import views
urlpatterns = [
    path("index/", views.index, name="index"),
]
</code></pre>
    <h5>
     <a id="63__152">
     </a>
     6.3 编写视图函数
    </h5>
    <p>
     pycharm里找到左下方的django结构，在app下右键视图，指向“New View”，点击“View Function” 快捷创建视图，修改视图函数名为【index】，这里需要与6.2里设置的path名对应。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/da17c075b45636cd4c8608e37c0778eb.png">
      <br/>
      修改index视图函数如下：
     </img>
    </p>
    <pre><code>def index(request):
    return render(request, "index.html")
</code></pre>
    <p>
     如果没有使用pycharm，那么在【应用名】的文件夹下，找到“views.py”，打开后，把上面的视图函数直接复制进去就可以了。
    </p>
    <h5>
     <a id="64_html_161">
     </a>
     6.4 编写html文件
    </h5>
    <p>
     在
     <code>
      templates
     </code>
     文件夹下新建
     <code>
      index.html
     </code>
     ，需要与6.3视图函数中render里的参数对应。内容如下：
    </p>
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;title&gt;Title&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
welcome django
&lt;/body&gt;
&lt;/html&gt;
</code></pre>
    <p>
     <code>
      templates
     </code>
     里存放各种html文件，在与
     <code>
      templates
     </code>
     同目录下，创建一个
     <code>
      static
     </code>
     文件夹，用来存储静态文件，如图片、css、js等等。
    </p>
    <h5>
     <a id="65__176">
     </a>
     6.5 运行程序
    </h5>
    <p>
     点击pycharm的绿色三角形运行程序，或者命令行输入
    </p>
    <pre><code>python manage.py runserver
</code></pre>
    <p>
     启动成功后，访问下面网址
    </p>
    <pre><code>http://127.0.0.1:8000/PasswordManage/index/
</code></pre>
    <p>
     发现显示内容
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/55d039e262993de891c1e6b5043f5b1e.png">
      <br/>
      到此就运行成功。
     </img>
    </p>
    <p>
     以上内容是使用django进行web开发的最基本内容，实际上，只是django的冰山一角。涉及到模板渲染、数据库操作、快捷表单等等很多很多知识都没有讲。分享的官方文档中，也有一个案例，也可以操作一下。
     <br/>
     官方文档是个好东西，一定要多研究一下。
    </p>
