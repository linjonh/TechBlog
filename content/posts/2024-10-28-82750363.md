    <p>
     为了防止别人入侵我们的电脑，清楚日志文件，我们可以配置日志服务器，把日志交给服务器来管理 一般来讲 我们搭建Rsyslog服务器
     <br/>
     它的传输方式有三种 tcp udp 以及relp
     <br/>
     tcp传输 会在特定时间丢失信息 ，通常我们用relp传输
     <br/>
     配置relp传输时需要安装rsyslog-relp
     <br/>
     tcp以及udp传输时 不需要 只用直接配置即可
    </p>
    <h2>
     <a id="Rsyslog_6">
     </a>
     在虚拟机里Rsyslog服务搭建
    </h2>
    <p>
     我们开两台虚拟机，一台作为服务器，一台作为客户端，
     <br/>
     配置网络环境，使其可以ping通
     <br/>
     配置文件为/etc/sysconfig/network-script/ifcfg-eth0
     <br/>
     在里面设置IP信息即可
    </p>
    <h3>
     <a id="rsyslog_11">
     </a>
     rsyslog服务器
    </h3>
    <h3>
     <a id="_13">
     </a>
     安装
    </h3>
    <pre><code>通常我们通过光盘来安装一些软件包
首先在/目录下创建挂载点
 mkdir /mnt
挂载光盘
mount /dev/sr0 /mnt 
</code></pre>
    <p>
     yum install 路径 -y
     <br/>
     有时候yum命令使用不了可以去更改配置文件
     <br/>
     gedit /etc/yum.repos.d/rhel-source.repo
     <br/>
     通常输入单词前两个字母tab键补全即可内容如下
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f3cf33a777fa07fdc74d0ae138434996.png">
      <br/>
      多余的可以删除。
     </img>
    </p>
    <p>
     因为rsyslog-relp的路径为 /mnt/Packages/rsyslog-relp-xxxx.rpm
     <br/>
     所以安装命令为
     <br/>
     yum install /mnt/Packages/rsyslog-relpxxxx.rpm -y
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1cabbc055e09ca60d5557617db1db24a.png">
      <br/>
      这时安装则完成
     </img>
    </p>
    <h3>
     <a id="_32">
     </a>
     配置
    </h3>
    <p>
     配置文件的路径为
     <br/>
     /etc/rsyslog.conf
     <br/>
     打开配置文件
     <br/>
     vi /etc/rsyslog.conf
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6e19c87959d16b71221b96390e48a295.png">
      <br/>
      可以看到三个箭头分别对应了 tcp udp 以及 relp在这里可以设置三种的任意一种，其它两种只需要在前面加上#号 注释掉即可
     </img>
    </p>
    <h3>
     <a id="_39">
     </a>
     监听
    </h3>
    <pre><code>/etc/sysconfig/rsyslog文件路径
</code></pre>
    <p>
     配置完成后 我们开启端口监听
     <br/>
     vi /etc/sysconfig/rsyslog
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/83b5ee495379b364d321d47f5f8b2be9.png">
      <br/>
      只需要配置一行 ，-r指定监听端口 -c2使用兼容端口
      <br/>
      配置完成后重启rsyslog服务
      <br/>
      service rsyslog restart
     </img>
    </p>
    <h3>
     <a id="_47">
     </a>
     客户端
    </h3>
    <h3>
     <a id="_48">
     </a>
     安装
    </h3>
    <p>
     安装过程和服务端一样
    </p>
    <h3>
     <a id="_50">
     </a>
     配置
    </h3>
    <p>
     打开配置文件
     <br/>
     vi /etc/rsyslog.conf
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f317e3d94ee056c35fdba025685f500e.png">
      <br/>
      三种传输的配置分别为*.*@后面接ip:端口 ip为服务器的ip同样的不需要用到的传输方式#号注释掉
      <br/>
      保存退出
      <br/>
      重启网络服务
      <br/>
      service rsyslog restart
      <br/>
      此刻我们服务器和客户端的配置就算完成了
     </img>
    </p>
    <h3>
     <a id="_59">
     </a>
     问题处理
    </h3>
    <p>
     有的时候上方可能连接不通
     <br/>
     通过 iptables -F清除防火墙配置
     <br/>
     和 setenforce 0两条命令重新生效下，即可
    </p>
