     GCC中 -I、-L、-l 选项的作用
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     在makefile中经常会看到这些选项，gcc默认会在程序当前目录、path路径中查找所需要的材料
     <br/>
     如何给gcc添加我们自己的原材料（头文件，库等）
    </p>
    <ul>
     <li>
      -I (注意是大写的i) 给gcc添加自定义的头文件的路径
      <br/>
      例如：我的头文件目录是myinclude
     </li>
    </ul>
    <pre id="copy_target_0"><code>-Imyinclude
</code></pre>
    <ul>
     <li>
      -L 给gcc添加额外的搜索库的路径
      <br/>
      例如 ：有一个库A所在的路径是目录B
     </li>
    </ul>
    <pre id="copy_target_1"><code>-LB
</code></pre>
    <ul>
     <li>
      -l(小写的L) 给gcc 指定具体的库
      <br/>
      例如：gcc需要一个库A
     </li>
    </ul>
    <pre id="copy_target_2"><code>-lA
</code></pre>
    <ul>
     <li>
      整体举例：
     </li>
    </ul>
    <pre id="copy_target_3"><code>CC=g++
CFLAGS=-g
BIN=main
OBJS=main.o
LIBS=-I/usr/include/mysql -L/usr/lib64/mysql -lmysqlclient -ljsoncpp -lpthread -std=c++11

$(BIN):$(OBJS)
$(CC) $(CFLAGS) $^ -o $@
%.o:%.cpp
$(CC) $(CFLAGS) -c $&lt; -o $@ $(LIBS)

.PHONY:clean
clean:
rm -f \*.o $(BIN)</code></pre>
