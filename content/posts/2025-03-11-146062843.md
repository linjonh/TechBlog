    <hr/>
    <p>
     在 Node.js 中，
     <code>
      http
     </code>
     模块是一个非常重要的核心模块，它允许我们创建 HTTP 服务器，并能处理客户端发起的 HTTP 请求。通过这个模块，我们可以构建一个能够接收、处理并返回响应的 Web 服务器。下面将详细介绍如何使用 Node.js 创建 HTTP 服务器，以及如何通过它来响应用户请求。
    </p>
    <h3>
     1.
     <code>
      http
     </code>
     模块概述
    </h3>
    <p>
     <code>
      http
     </code>
     模块为 Node.js 提供了与 HTTP 协议相关的功能。通过
     <code>
      http.createServer()
     </code>
     方法，我们可以创建一个 HTTP 服务器，服务器会监听客户端请求并作出响应。这个方法接收一个回调函数作为参数，回调函数会在每次请求到达时执行，回调函数中可以通过请求对象
     <code>
      req
     </code>
     和响应对象
     <code>
      res
     </code>
     来进行数据处理。
    </p>
    <h3>
     2. 创建 HTTP 服务器
    </h3>
    <p>
     通过
     <code>
      http.createServer()
     </code>
     创建一个 HTTP 服务器，指定请求的回调函数并在服务器上监听指定端口，常见的操作如下：
    </p>
    <h3>
     3. 示例代码
    </h3>
    <h4>
     服务器端代码（JavaScript）
    </h4>
    <pre><code class="language-javascript">// 引入http模块
const http = require('http');

// 创建HTTP服务器
const server = http.createServer((req, res) =&gt; {
  // 设置响应的HTTP头部信息
  res.statusCode = 200; // 状态码为200表示成功
  res.setHeader('Content-Type', 'text/html'); // 设置响应类型为HTML

  // 返回响应内容
  res.end('&lt;h1&gt;欢迎使用Node.js创建的HTTP服务器&lt;/h1&gt;');
});

// 服务器监听在3000端口
server.listen(3000, '127.0.0.1', () =&gt; {
  console.log('服务器正在运行，访问 http://127.0.0.1:3000');
});
</code></pre>
    <h3>
     4. 代码解析
    </h3>
    <ul>
     <li>
      <strong>
       引入
       <code>
        http
       </code>
       模块
      </strong>
      : 我们首先通过
      <code>
       require('http')
      </code>
      引入 Node.js 提供的
      <code>
       http
      </code>
      模块。
     </li>
     <li>
      <strong>
       创建服务器
      </strong>
      : 使用
      <code>
       http.createServer()
      </code>
      创建一个 HTTP 服务器，该函数接收一个回调函数
      <code>
       (req, res)
      </code>
      。
      <code>
       req
      </code>
      代表客户端请求对象，
      <code>
       res
      </code>
      代表服务器的响应对象。
     </li>
     <li>
      <strong>
       设置响应头
      </strong>
      : 通过
      <code>
       res.statusCode
      </code>
      设置 HTTP 响应的状态码，
      <code>
       res.setHeader()
      </code>
      用于设置响应的头部信息，
      <code>
       Content-Type
      </code>
      为
      <code>
       text/html
      </code>
      表示响应的是 HTML 格式的内容。
     </li>
     <li>
      <strong>
       结束响应
      </strong>
      : 使用
      <code>
       res.end()
      </code>
      方法结束响应，并将
      <code>
       &lt;h1&gt;
      </code>
      标签包围的字符串作为响应内容返回。
     </li>
     <li>
      <strong>
       监听端口
      </strong>
      : 通过
      <code>
       server.listen()
      </code>
      方法让服务器在指定的端口（这里是 3000）上监听客户端请求。
     </li>
    </ul>
    <h3>
     5. 运行服务器
    </h3>
    <ol>
     <li>
      将上述代码保存在一个名为
      <code>
       server.js
      </code>
      的文件中。
     </li>
     <li>
      在命令行中导航到保存文件的目录。
     </li>
     <li>
      使用
      <code>
       node server.js
      </code>
      启动服务器。
     </li>
     <li>
      打开浏览器并访问
      <code>
       http://127.0.0.1:3000
      </code>
      ，你将看到浏览器中显示出欢迎消息。
     </li>
    </ol>
    <h3>
     6. 运行结果
    </h3>
    <p>
     当服务器启动并运行时，你可以在浏览器中输入
     <code>
      http://127.0.0.1:3000
     </code>
     来访问服务器。浏览器会显示：
    </p>
    <pre><code class="language-bash">欢迎使用Node.js创建的HTTP服务器
</code></pre>
    <p>
     这表示你已经成功创建了一个 HTTP 服务器，能够响应用户的 HTTP 请求。
    </p>
    <h3>
     7. 进一步扩展
    </h3>
    <ul>
     <li>
      <strong>
       路由
      </strong>
      ：你可以根据请求的 URL 不同，做出不同的响应。例如，可以根据
      <code>
       req.url
      </code>
      来判断用户访问的是首页、关于页或其他页面。
     </li>
     <li>
      <strong>
       处理请求数据
      </strong>
      ：通过
      <code>
       req
      </code>
      对象可以读取请求的内容，处理
      <code>
       POST
      </code>
      数据、
      <code>
       GET
      </code>
      查询等。
     </li>
     <li>
      <strong>
       静态文件服务
      </strong>
      ：通过 Node.js 服务器，您还可以提供静态文件服务，比如返回 HTML、CSS、JavaScript 文件。
     </li>
    </ul>
    <h3>
     8. 总结
    </h3>
    <p>
     通过这篇介绍，你可以理解 Node.js
     <code>
      http
     </code>
     模块如何构建一个简单的 Web 服务器。这个 HTTP 服务器能够监听指定端口，接受请求并返回响应。你可以根据自己的需求，进一步扩展和完善服务器的功能。
    </p>
