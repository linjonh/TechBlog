    <h2>
     <a class="link-info" href="https://leetcode.cn/problems/maximum-subarray/description/" rel="nofollow" title="一、53.最大子序和">
      一、53.最大子序和
     </a>
    </h2>
    <h3>
     1.题目描述
    </h3>
    <p>
     <img alt="" height="120" src="https://i-blog.csdnimg.cn/direct/9ba50d46a6a44667b59c4cb688322892.png" width="605"/>
    </p>
    <h3>
     2.代码
    </h3>
    <p>
     <img alt="" height="485" src="https://i-blog.csdnimg.cn/direct/65f56eb0027944819e0e179e0f3a24a2.png" width="452"/>
    </p>
    <h3>
     3.思路
    </h3>
    <p>
     先特殊处理数组只有一个数的情况，再定义两个变量，sum用于记录最大子数组和，
     <code>
      count
     </code>
     用于记录当前连续子数组的和。使用
     <code>
      for
     </code>
     循环遍历数组
     <code>
      nums
     </code>
     中的每个元素。对于每个元素
     <code>
      nums[i]
     </code>
     ，将其累加到
     <code>
      count
     </code>
     中。每次累加后，使用
     <code>
      Math.max
     </code>
     函数比较
     <code>
      sum
     </code>
     和
     <code>
      count
     </code>
     的大小，将较大值更新到
     <code>
      sum
     </code>
     中，确保
     <code>
      sum
     </code>
     始终记录最大子数组和。如果
     <code>
      count
     </code>
     小于等于 0，说明当前连续子数组的和为负数或 0，继续往后累加这个子数组不会得到更大的和，因此将
     <code>
      count
     </code>
     重置为 0，重新开始计算新的连续子数组和。遍历结束后，
     <code>
      sum
     </code>
     中存储的就是最大子数组和，将其返回。
    </p>
    <h2>
     <a class="link-info" href="https://leetcode.cn/problems/gas-station/description/" rel="nofollow" title="二、134.加油站">
      二、134.加油站
     </a>
    </h2>
    <h3>
     1.题目描述
    </h3>
    <p>
     <img alt="" height="215" src="https://i-blog.csdnimg.cn/direct/38115f1702c54e3c9709e1feb0928c7e.png" width="600"/>
    </p>
    <h3>
     2.代码
    </h3>
    <p>
     <img alt="" height="476" src="https://i-blog.csdnimg.cn/direct/9df03c17c71f4bbd89da8b69854d022b.png" width="608"/>
    </p>
    <h3>
     3.思路
    </h3>
    <ol>
     <li>
      <strong>
       变量初始化
      </strong>
      ：
      <ul>
       <li>
        <code>
         curSum
        </code>
        ：用于记录从当前起始加油站出发，到当前遍历到的加油站时，剩余的汽油量。初始化为 0。
       </li>
       <li>
        <code>
         totalSum
        </code>
        ：用于记录整个环形路线中，汽油总量与消耗总量的差值。初始化为 0。
       </li>
       <li>
        <code>
         index
        </code>
        ：用于记录可能的起始加油站的索引，初始化为 0。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       遍历加油站
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         for
        </code>
        循环遍历每个加油站，索引为
        <code>
         i
        </code>
        。
       </li>
       <li>
        对于每个加油站，计算在该加油站补充汽油并前往下一个加油站后的剩余汽油量，即
        <code>
         gas[i] - cost[i]
        </code>
        ，将其累加到
        <code>
         curSum
        </code>
        和
        <code>
         totalSum
        </code>
        中。
       </li>
       <li>
        如果
        <code>
         curSum
        </code>
        小于 0，说明从当前起始加油站出发无法到达当前加油站，需要更换起始加油站。将起始加油站的索引更新为
        <code>
         (i + 1) % gas.length
        </code>
        ，并将
        <code>
         curSum
        </code>
        重置为 0，重新开始计算剩余汽油量。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       判断是否能完成一圈行驶
      </strong>
      ：
      <ul>
       <li>
        遍历结束后，检查
        <code>
         totalSum
        </code>
        的值。如果
        <code>
         totalSum
        </code>
        小于 0，说明整个环形路线上的汽油总量小于消耗总量，无论从哪个加油站出发都无法完成一圈行驶，返回 -1。
       </li>
       <li>
        如果
        <code>
         totalSum
        </code>
        大于等于 0，说明存在可以完成一圈行驶的起始加油站，返回
        <code>
         index
        </code>
        。
       </li>
      </ul>
     </li>
    </ol>
