    <h4>
     <a id="2_Chrome_DevTools__13">
     </a>
     2.
     <strong>
      Chrome DevTools 中的安全与隐私面板
     </strong>
    </h4>
    <p>
     从
     <strong>
      Chrome 134
     </strong>
     开始，开发者可以在
     <strong>
      Chrome DevTools（开发者工具）
     </strong>
     中使用全新的
     <strong>
      “安全与隐私”（Security &amp; Privacy）
     </strong>
     面板，测试当
     <strong>
      第三方 Cookie 受限
     </strong>
     时，网站的行为表现。
    </p>
    <ul>
     <li>
      开发者可以
      <strong>
       临时限制第三方 Cookie
      </strong>
      ，观察网站行为，并检查网站上
      <strong>
       第三方 Cookie 的状态
      </strong>
      。
     </li>
     <li>
      该功能
      <strong>
       不会永久更改现有的企业策略
      </strong>
      ，但可以
      <strong>
       临时覆盖
      </strong>
      相关的企业策略，以测试更严格的 Cookie 限制。
      <ul>
       <li>
        <strong>
         受影响的企业策略
        </strong>
        ：
        <ul>
         <li>
          BlockThirdPartyCookies（阻止第三方 Cookie）
         </li>
         <li>
          CookiesAllowedForUrls（允许特定 URL 使用 Cookie）
         </li>
        </ul>
       </li>
       <li>
        <strong>
         如果企业策略已通过 BlockThirdPartyCookies 禁用了第三方 Cookie
        </strong>
        ，该功能
        <strong>
         将无法启用
        </strong>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="21__23">
     </a>
     2.1.
     <strong>
      更新内容
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       新的 “安全与隐私” 面板替代

      了

      。
     </li>
     <li>
      <strong>
       TLS 连接和证书信息
      </strong>
      仍可在
      <strong>
       “安全与隐私” 面板左侧的安全菜单
      </strong>
      中访问。
     </li>
    </ul>

    <h4>
     <a id="3_ML_29">
     </a>
     3.
     <strong>
      使用机器学习（ML）提升密码表单检测
     </strong>
    </h4>
    <p>
     从
     <strong>
      Chrome 134
     </strong>
     开始，Chrome 引入了一种
     <strong>
      新的客户端机器学习（ML）模型
     </strong>
     ，以
     <strong>
      更精准地解析网页上的密码表单
     </strong>
     ，提高
     <strong>
      密码检测和自动填充的准确性
     </strong>
     。
    </p>
    <ul>
     <li>
      <strong>
       该功能可通过企业策略 PasswordManagerEnabled 进行控制
      </strong>
      。
     </li>
    </ul>
    <h4>
     <a id="4__LLM__34">
     </a>
     4.
     <strong>
      客户端 LLM 辅助防范网络诈骗
     </strong>
    </h4>
    <p>
     用户每天在网上都会遇到
     <strong>
      大量不同类型的网络诈骗
     </strong>
     。
    </p>
    <p>
     为应对这些威胁，
     <strong>
      Chrome 将利用本地（设备端）大语言模型（LLM）
     </strong>
     ，帮助
     <strong>
      增强安全浏览（Enhanced Safe Browsing，ESB）
     </strong>
     用户
     <strong>
      识别诈骗网站
     </strong>
     。
    </p>
    <h5>
     <a id="41__39">
     </a>
     4.1.
     <strong>
      工作原理
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       Chrome 会将网页内容发送至本地 LLM
      </strong>
      ，分析其安全相关的信号。
     </li>
     <li>
      <strong>
       Chrome 会将这些安全信号发送至 Safe Browsing 服务器端
      </strong>
      ，进行最终判断。
     </li>
     <li>
      <strong>
       当该功能启用时
      </strong>
      ，Chrome 可能会消耗
      <strong>
       更多带宽
      </strong>
      来
      <strong>
       下载 LLM
      </strong>
      以执行本地推理。
     </li>
    </ol>
    <h4>
     <a id="5__LLM__Android__44">
     </a>
     5.
     <strong>
      基于 LLM 的本地检测，屏蔽 Android 端的恶意通知
     </strong>
    </h4>
    <p>
     该功能旨在
     <strong>
      隐藏疑似恶意通知的内容
     </strong>
     ，并提供用户操作选项。
    </p>
    <ul>
     <li>
      当 Chrome 识别到
      <strong>
       可能是滥用或恶意通知
      </strong>
      时，通知内容将被隐藏。
     </li>
     <li>
      用户可以选择：
      <ul>
       <li>
        <strong>
         关闭通知
        </strong>
       </li>
       <li>
        <strong>
         显示完整通知内容
        </strong>
       </li>
       <li>
        <strong>
         取消订阅该来源的通知
        </strong>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       该检测将在设备端（本地）由 LLM 模型执行
      </strong>
      ，无需将数据发送到服务器。
     </li>
    </ul>
    <h4>
     <a id="6_ManagedChrome__54">
     </a>
     6.
     <strong>
      自定义托管（Managed）Chrome 个人资料的徽标和标签
     </strong>
    </h4>
    <p>
     Chrome 推出了
     <strong>
      新的工具栏和个人资料菜单自定义功能
     </strong>
     ，帮助用户更容易识别
     <strong>
      其 Chrome 个人资料是否受管理
     </strong>
     ，无论是
     <strong>
      工作设备
     </strong>
     还是
     <strong>
      个人设备
     </strong>
     。
    </p>
    <p>
     这一功能尤其适用于
     <strong>
      员工使用个人设备登录托管账户
     </strong>
     的场景。
    </p>
    <h5>
     <a id="61__59">
     </a>
     6.1. 新增的企业策略
    </h5>
    <p>
     为了增强企业品牌定制体验，Chrome
     <strong>
      新增了以下三项策略
     </strong>
     ：
    </p>
    <ol>
     <li>
      <strong>
       EnterpriseCustomLabel：自定义工具栏上显示的文本
      </strong>
      ，使其符合
      <strong>
       企业品牌风格
      </strong>
      。
     </li>
     <li>
      <strong>
       EnterpriseLogoUrl：在个人资料菜单中显示企业徽标
      </strong>
      。
     </li>
     <li>
      <strong>
       EnterpriseProfileBadgeToolbarSettings：禁用默认的托管账户标签
      </strong>
      （即 Chrome 工具栏上的“已托管”标识）。
     </li>
    </ol>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/ac5673024efeca30663bd90269a7dfb1.png"/>
    </p>
    <h4>
     <a id="7_DBSC_Googlecom__68">
     </a>
     7.
     <strong>
      设备绑定会话凭据（DBSC）—— Google.com 原型测试
     </strong>
    </h4>
    <p>
     <strong>
      设备绑定会话凭据（Device Bound Session Credentials，DBSC）
     </strong>
     旨在推动 Web 逐步
     <strong>
      摆脱长期存活的持有者凭据（如 Cookie）
     </strong>
     ，转向
     <strong>
      短期凭据
     </strong>
     或
     <strong>
      加密绑定到设备的凭据
     </strong>
     ，以防止凭据被窃取和滥用。
    </p>
    <h5>
     <a id="71__71">
     </a>
     7.1. 主要目标
    </h5>
    <ul>
     <li>
      <strong>
       防止凭据被恶意软件盗取
      </strong>
      ，提升安全性。
     </li>
     <li>
      采用
      <strong>
       短期凭据
      </strong>
      或
      <strong>
       设备绑定的加密凭据
      </strong>
      ，减少 Cookie 泄露风险。
     </li>
     <li>
      目前该功能仍处于
      <strong>
       概念验证阶段（Proof-of-Concept）
      </strong>
      ，仅适用于
      <strong>
       google.com
      </strong>
      网站。
     </li>
     <li>
      未来，Chrome 计划将
      <strong>
       这一机制标准化
      </strong>
      ，适用于
      <strong>
       更多网站和浏览器
      </strong>
      。
     </li>
    </ul>

     </strong>
    </h4>

     <strong>
      立即更改密码
     </strong>
     。
    </p>

      触发方式
     </strong>

     <li>
      该功能
      <strong>
       只能从“检查您的密码”对话框（Check your Password dialog）中触发
      </strong>
      。
     </li>
     <li>
      当用户在受支持的网站上看到密码泄露警告时，可
      <strong>
       直接更改密码
      </strong>
      。
     </li>
    </ul>
    <h4>
     <a id="9_Chrome_134__92">
     </a>
     9.
     <strong>
      Chrome 134 阅读模式支持朗读功能
     </strong>
    </h4>
    <p>
     <strong>
      阅读模式（Reading Mode）
     </strong>
     是
     <strong>
      侧边栏功能
     </strong>
     ，用于提供
     <strong>
      简化的文本阅读体验
     </strong>
     ，适用于
     <strong>
      内容密集的网页
     </strong>
     。
    </p>
    <p>
     从
     <strong>
      Chrome 134
     </strong>
     开始，阅读模式新增
     <strong>
      朗读（Read Aloud）
     </strong>
     功能，让用户可以
     <strong>
      听取网页文本的朗读
     </strong>
     。
    </p>

    </h5>

     <li>
      <strong>
       多种自然语音
      </strong>
      选择
     </li>
     <li>
      <strong>
       可调节朗读速度
      </strong>
     </li>
     <li>
      <strong>
       朗读时同步高亮显示文本
      </strong>
      ，方便跟随阅读
     </li>
    </ul>

    <h5>
     <a id="121__HTML_SelectParserRelaxation_123">
     </a>
     12.1. 相关更改：HTML 解析调整（SelectParserRelaxation）
    </h5>
    <ul>
     <li>
      该功能依赖
      <strong>
       SelectParserRelaxation
      </strong>
      标志，该标志修改了 HTML 解析方式，允许
      <strong>
       select 内包含更多标签
      </strong>
      。
     </li>
     <li>
      以前，select 内部的
      <strong>
       额外标签（如 span）
      </strong>
      会被移除，而现在它们将被保留。
     </li>
     <li>
      <strong>
       影响范围
      </strong>
      ：
      <ul>
       <li>
        如果网站
        <strong>
         在 select 中包含额外的 HTML 标签
        </strong>
        （如 span），可能会受到影响。
       </li>
       <li>
        如果 select 中包含
        <strong>
         大量 option 选项
        </strong>
        ，可能会导致
        <strong>
         下拉菜单打开变慢
        </strong>
        或
        <strong>
         选项未正确显示
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      该功能及
      <strong>
       SelectParserRelaxation
      </strong>
      行为可通过
      <strong>
       SelectParserRelaxation 企业策略
      </strong>
      进行控制。
     </li>
    </ul>
    <h4>
     <a id="13_select__HTML__131">

    <h4>
     <a id="14__getUserMedia__141">
     </a>
     14.
     <strong>
      移除非标准的 getUserMedia 音频约束
     </strong>
    </h4>
    <p>
     从
     <strong>
      Chrome 134
     </strong>
     开始，Chrome
     <strong>
      将移除一系列 goog- 前缀的非标准音频约束
     </strong>
     ，这些约束在
     <strong>
      音频约束标准化
     </strong>
     之前存在，但现已过时。
    </p>
    <h5>
     <a id="141__144">
     </a>
     14.1. 变更背景
    </h5>
    <ul>
     <li>
      这些
      <strong>
       goog- 前缀的音频约束使用率极低
      </strong>
      （约
      <strong>
       0.000001% ~ 0.0009%
      </strong>
      ，取决于具体约束）。
     </li>
     <li>
      由于
      <strong>
       Chromium 音频捕获架构的调整
      </strong>
      ，其中
      <strong>
       一些约束已无实际作用
      </strong>
      。
     </li>
     <li>
      即将到来的
      <strong>
       进一步更改
      </strong>
      将使
      <strong>
       所有这些非标准约束完全失效
      </strong>
      。
     </li>
    </ul>
