     深入理解 Maven BOM 及其继承特性
    </h3>
    <h4>
     <a id="_Maven_BOM_3">
     </a>
     一、什么是 Maven BOM？
    </h4>
    <p>
     Maven BOM（Bill Of Materials，物料清单）是一种特殊的 Maven 项目，用于集中管理依赖项的版本信息。BOM 项目本身并不包含实际的代码或资源，而仅仅声明一组依赖项的版本，从而使其他项目在引入这些依赖时无需重复指定版本。
    </p>
    <h4>
     <a id="Maven_BOM__7">
     </a>
     二、Maven BOM 的主要作用
    </h4>
    <p>
     Maven BOM 在项目管理中的主要作用包括：
    </p>
    <ol>
     <li>
      <strong>
       统一版本管理
      </strong>
      ：防止不同项目或模块之间因版本差异引发的冲突。
     </li>
     <li>
      <strong>
       简化依赖配置
      </strong>
      ：通过继承 BOM 中的版本，减少项目 pom.xml 文件中的重复定义。
     </li>
     <li>
      <strong>
       集中维护
      </strong>
      ：版本集中维护在一个地方，更新时只需修改 BOM，无需逐个修改所有项目。
     </li>
    </ol>

    <h4>
     <a id="Maven_BOM__78">
     </a>
     四、Maven BOM 的继承特性
    </h4>
    <p>
     Maven BOM 的继承性使其成为更强大的工具，具体表现在以下几个方面：
    </p>
    <h5>
     <a id="1__82">
     </a>
     1. 父子继承关系
    </h5>
    <p>
     当一个 BOM 被引入父项目的
     <code>
      dependencyManagement
     </code>
     中时，所有的子项目自动继承该 BOM 中定义的版本，无需在子项目再次声明版本。
    </p>
    <p>
     示例：
    </p>
    <pre><code class="prism language-xml"><span class="token comment">&lt;!-- 父项目 --&gt;</span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>dependencyManagement</span><span class="token punctuation">&gt;</span></span>
  <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>dependencies</span><span class="token punctuation">&gt;</span></span>
    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>dependency</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>groupId</span><span class="token punctuation">&gt;</span></span>com.example<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>groupId</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>artifactId</span><span class="token punctuation">&gt;</span></span>example-bom<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>artifactId</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>version</span><span class="token punctuation">&gt;</span></span>1.0.0<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>version</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>type</span><span class="token punctuation">&gt;</span></span>pom<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>type</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>scope</span><span class="token punctuation">&gt;</span></span>import<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>scope</span><span class="token punctuation">&gt;</span></span>
    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependency</span><span class="token punctuation">&gt;</span></span>
  <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependencies</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependencyManagement</span><span class="token punctuation">&gt;</span></span>

     子项目使用时无需再声明版本：

    <h5>
     <a id="2__116">
     </a>
     2. 链式继承
    </h5>
    <p>
     BOM 还可以继承其他 BOM 所定义的依赖版本，这种模式被称为“链式继承”。
    </p>
    <p>
     例如，一个自定义 BOM 继承了 Spring Boot Dependencies：
    </p>
    <pre><code class="prism language-xml"><span class="token comment">&lt;!-- my-bom --&gt;</span>

    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>dependency</span><span class="token punctuation">&gt;</span></span>

      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>type</span><span class="token punctuation">&gt;</span></span>pom<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>type</span><span class="token punctuation">&gt;</span></span>
      <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>scope</span><span class="token punctuation">&gt;</span></span>import<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>scope</span><span class="token punctuation">&gt;</span></span>
    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependency</span><span class="token punctuation">&gt;</span></span>
  <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependencies</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>dependencyManagement</span><span class="token punctuation">&gt;</span></span>

    <p>
     此时，引用
     <code>
      my-bom
     </code>
     的项目即可自动获得来自 Spring Boot Dependencies 定义的依赖版本。
    </p>
    <h5>
     <a id="3__139">
     </a>
     3. 继承优先级与版本覆盖规则
    </h5>
    <p>
     当项目继承多个 BOM，且发生版本冲突时，Maven 遵循“最近定义优先”的规则。优先级由高到低依次为：
    </p>
    <ul>
     <li>
      项目自身显式定义的依赖版本
     </li>
     <li>
      项目直接定义的 BOM 中的版本
     </li>
     <li>
      父项目中定义的 BOM 中的版本
     </li>
     <li>
      更远端继承链中的 BOM 的版本
     </li>
    </ul>

     </a>
     五、常用的社区维护的 BOM
    </h4>
    <p>
     以下是一些广泛使用的社区 BOM：
    </p>
    <ul>
     <li>
      <strong>
       Spring Boot Dependencies
      </strong>
      ：适用于Spring生态系统。
     </li>
     <li>
      <strong>
       Spring Cloud Dependencies
      </strong>

     </li>

    <h4>
     <a id="_155">

     六、总结
    </h4>
