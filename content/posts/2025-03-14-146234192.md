     三 文件权限与管理
    </h2>
    <h3>
     <a id="31__9">
     </a>
     3.1 必要性
    </h3>
    <p>
     Linux 文件权限与管理对于系统安全和稳定至关重要。通过正确设置文件的读、写、执行权限，可以有效防止未经授权的访问和操作，确保数据的安全性。同时，合理的权限管理有助于限制用户或程序对系统资源的访问，避免潜在的安全漏洞和不必要的风险。因此，掌握文件权限和管理对于保护系统、提升效率以及避免误操作非常重要。
    </p>

    <p>
     粘滞位（
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        S 
         
        
          t 
         
        
          i

          k

        \color{Purple}Sticky Bit
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.8889em; vertical-align: -0.1944em;">
         </span>
         <span class="mord mathnormal" style="color: purple;">
          St
         </span>
         <span class="mord mathnormal" style="color: purple;">
          i
         </span>
         <span class="mord mathnormal" style="color: purple;">

         </span>
         <span class="mord mathnormal" style="margin-right: 0.0315em; color: purple;">
          k
         </span>
         <span class="mord mathnormal" style="margin-right: 0.0359em; color: purple;">
          y
         </span>
         <span class="mord mathnormal" style="margin-right: 0.0502em; color: purple;">
          B
         </span>
         <span class="mord mathnormal" style="color: purple;">

         </span>
         <span class="mord mathnormal" style="color: purple;">

    </p>
    <h3>
     <a id="102_colorFuchsia_554">
     </a>
     10.2
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        引入背景 
         
        
       
      
        \color{Fuchsia}引入背景
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.6833em;">
         </span>
         <span class="mord cjk_fallback" style="color: fuchsia;">
          引入背景
         </span>
        </span>
       </span>
      </span>
     </span>
    </h3>
    <p>
     在多用户环境中，公共目录通常会被多人访问、写入和修改。例如，/tmp 目录是一个临时存储空间，所有用户都有写权限。如果没有粘滞位，任何用户都可以删除其他用户在该目录下创建的文件，这样就可能导致不可预期的安全问题或数据丢失。
     <br/>
     引入粘滞位后，虽然用户依然可以写入文件，但只有文件的所有者才能删除或修改自己的文件，其他用户即使有写权限，也无法删除或修改非自己拥有的文件。
    </p>
    <h3>
     <a id="103_colorFuchsia_558">
     </a>
     10.3
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        粘滞位作用 
         
        
       
      
        \color{Fuchsia}粘滞位作用
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.6833em;">
         </span>
         <span class="mord cjk_fallback" style="color: fuchsia;">
          粘滞位作用
         </span>
        </span>
       </span>
      </span>
     </span>
    </h3>
    <p>
     在未设置粘滞位的普通目录中，所有用户（包括文件的拥有者、同组用户、其他用户）都可以删除或修改目录中的任何文件。
     <br/>
     但是，当粘滞位被设置后，目录中的文件只能由文件的所有者、目录的所有者或超级用户（root）删除或修改。其他用户不能删除或修改他们没有权限的文件，即使他们在该目录中有写权限。
    </p>

     10.4.1 查看目录的粘滞位：
    </h4>
    <p>
     使用 ls -l 命令可以查看文件或目录的权限信息，如果目录上设置了粘滞位，输出的权限信息中会在最后一位显示 t（对于普通用户的写权限而言）。
    </p>

     </p>
    </blockquote>
    <p>
     示例输出：
    </p>
    <blockquote>
     <p>
      drwxrwxrwt 10 root root 4096 Mar 14 09:12 /tmp
     </p>
    </blockquote>
    <p>
     这里的 t 就是表示 /tmp 目录上启用了粘滞位。
    </p>
    <h4>
     <a id="1042__575">
     </a>
     10.4.2 设置粘滞位：
    </h4>
    <p>
     使用 chmod 命令来设置或移除粘滞位。
    </p>
    <ul>
     <li>
      设置粘滞位：
     </li>
    </ul>
    <blockquote>
     <p>
      chmod +t
     </p>
    </blockquote>
    <ul>
     <li>
      例如，设置 /tmp 目录的粘滞位：
     </li>
    </ul>
    <blockquote>
     <p>
      chmod +t /tmp
     </p>
    </blockquote>
    <ul>
     <li>
      移除粘滞位：
     </li>
    </ul>
    <blockquote>
     <p>
      chmod -t
     </p>
    </blockquote>
    <ul>
     <li>
      其他相关权限：
     </li>
    </ul>
    <p>
     粘滞位通常与目录权限一起使用。例如，在 /tmp 目录中，可能会看到 rwxrwxrwt 这样的权限，其中 t 表示粘滞位。
    </p>

    </p>
    <h3>
     <a id="107_colorFuchsia_603">
     </a>
     10.7
     <span class="katex--inline">

        \color{Fuchsia}总结

          总结
         </span>

       </span>
      </span>
     </span>
    </h3>
    <p>
     粘滞位主要是为了提高公共目录（如 /tmp）的安全性。它防止普通用户删除或修改非自己拥有的文件，从而避免意外的数据丢失或恶意的文件操作。通过合理设置粘滞位，可以确保多用户环境下的文件安全。
    </p>
