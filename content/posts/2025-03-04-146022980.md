    <h2>
     <a id="HTML_1">
     </a>
     HTML概述
    </h2>
    <p>
     HTML 即超文本标记语言（HyperText Markup Language），它是用于创建网页的标准标记语言
    </p>
    <h3>
     <a id="_7">
     </a>
     基本概念
    </h3>
    <p>
     HTML 通过一系列的标记（标签）来描述网页的结构和内容。这些标签就像是建筑中的砖块和蓝图，告诉浏览器如何显示网页的各个部分，比如文本、图像、链接、表格等。浏览器（如 Chrome、Firefox 等）会读取 HTML 文件，并根据其中的标签将网页内容呈现给用户
    </p>
    <h3>
     <a id="HTML__13">
     </a>
     HTML 发展历程
    </h3>
    <p>
     HTML（超文本标记语言）是构建万维网的核心技术之一，其发展历程反映了互联网技术的演进和标准化进程。以下是HTML的主要发展阶段：
    </p>
    <p>
     <strong>
      起源与HTML 1.0（1991-1993）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       创始人
      </strong>
      ：蒂姆·伯纳斯-李（Tim Berners-Lee）于1991年在欧洲核子研究中心（CERN）提出，旨在实现科学文档的共享。
     </li>
     <li>
      <strong>
       HTML 1.0
      </strong>
      ：未正式标准化，支持基本文本格式（标题、段落、超链接），仅包含约20个标签。
     </li>
    </ul>
    <p>
     <strong>
      HTML 2.0（1995）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       首个官方标准
      </strong>
      ：由IETF（互联网工程任务组）于1995年发布，定义了HTML的基础结构。
     </li>
     <li>
      <strong>
       功能增强
      </strong>
      ：支持表单（
      <code>
       &lt;form&gt;
      </code>
      ）、图像（
      <code>
       &lt;img&gt;
      </code>
      ）、列表等，成为早期网页的通用标准。
     </li>
    </ul>
    <p>
     <strong>
      HTML 3.2（1997）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       W3C接管
      </strong>
      ：万维网联盟（W3C）开始主导标准化，1997年发布HTML 3.2。
     </li>
     <li>
      <strong>
       新特性
      </strong>
      ：引入表格（
      <code>
       &lt;table&gt;
      </code>
      ）、字体样式、脚本支持（JavaScript）等，推动网页动态化。
     </li>
    </ul>
    <p>
     <strong>
      HTML 4.01（1999）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       里程碑版本
      </strong>
      ：1999年发布，成为长期主流标准。
     </li>
     <li>
      <strong>
       关键改进
      </strong>
      ：
      <ul>
       <li>
        分离结构与样式：鼓励使用CSS（层叠样式表）。
       </li>
       <li>
        框架（
        <code>
         &lt;frame&gt;
        </code>
        ）、多媒体支持（需插件如Flash）。
       </li>
       <li>
        国际化支持（字符编码规范）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       三种变体
      </strong>
      ：严格模式（Strict）、过渡模式（Transitional）、框架集（Frameset）。
     </li>
    </ul>
    <p>
     <strong>
      XHTML（2000-2009）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       XML化尝试
      </strong>
      ：W3C于2000年推出XHTML 1.0，强制要求语法严格性（如标签闭合、小写标签）。
     </li>
     <li>
      <strong>
       目标
      </strong>
      ：提高代码规范性和跨平台兼容性。
     </li>
     <li>
      <strong>
       争议
      </strong>
      ：严格的语法规则导致开发复杂度上升，浏览器兼容性不足，最终未全面普及。
     </li>
    </ul>
    <p>
     <strong>
      HTML5（2014至今）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       现代Web基石
      </strong>
      ：由W3C与WHATWG（网页超文本应用技术工作小组）合作开发，2014年正式定稿。
     </li>
     <li>
      <strong>
       核心特性
      </strong>
      ：
      <ul>
       <li>
        <strong>
         语义化标签
        </strong>
        ：
        <code>
         &lt;header&gt;
        </code>
        ,
        <code>
         &lt;article&gt;
        </code>
        ,
        <code>
         &lt;nav&gt;
        </code>
        等，提升可访问性和SEO。
       </li>
       <li>
        <strong>
         原生多媒体支持
        </strong>
        ：
        <code>
         &lt;video&gt;
        </code>
        ,
        <code>
         &lt;audio&gt;
        </code>
        标签取代Flash。
       </li>
       <li>
        <strong>
         图形与动画
        </strong>
        ：
        <code>
         &lt;canvas&gt;
        </code>
        和SVG支持动态绘图。
       </li>
       <li>
        <strong>
         离线与存储
        </strong>
        ：LocalStorage、IndexedDB、Service Worker。
       </li>
       <li>
        <strong>
         设备API
        </strong>
        ：地理位置（Geolocation）、摄像头访问。
       </li>
       <li>
        <strong>
         响应式设计
        </strong>
        ：
        <code>
         &lt;picture&gt;
        </code>
        、
        <code>
         &lt;meta viewport&gt;
        </code>
        适配多端。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       持续演进
      </strong>
      ：HTML5不再以版本号迭代，而是通过模块化标准持续更新（如HTML Living Standard）。
     </li>
    </ul>
    <p>
     <strong>
      HTML发展特点
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       标准化与开放
      </strong>
      ：从IETF到W3C/WHATWG，推动技术统一。
     </li>
     <li>
      <strong>
       功能扩展
      </strong>
      ：从静态文档到富媒体、复杂应用（如WebGL、WebAssembly）。
     </li>
     <li>
      <strong>
       生态协同
      </strong>
      ：依赖CSS、JavaScript共同构建现代Web技术栈。
     </li>
     <li>
      <strong>
       移动优先
      </strong>
      ：HTML5助力移动互联网爆发，支持跨平台开发。
     </li>
    </ol>

</code></pre>
    <p>
     结构解释：
    </p>
    <ol>
     <li>
      <p>
       <code>
        &lt;!DOCTYPE html&gt;
       </code>
       <br/>
       这是文档类型声明，它位于 HTML 文档的第一行，告诉浏览器当前文档使用的是 HTML5 标准。不同的 HTML 版本有不同的文档类型声明，而
       <code>
        &lt;!DOCTYPE html&gt;
       </code>
       是 HTML5 特有的简洁声明方式。
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;html&gt;
       </code>
       标签
      </p>
      <ul>
       <li>
        <code>
         &lt;html&gt;
        </code>
        标签是 HTML 文档的根标签，所有其他的 HTML 元素都必须包含在
        <code>
         &lt;html&gt;
        </code>
        和
        <code>
         &lt;/html&gt;
        </code>
        标签之间。
       </li>
       <li>
        <code>
         lang="zh-CN"
        </code>
        是一个属性，用于指定文档的语言为中文（中国大陆地区），有助于搜索引擎和屏幕阅读器等工具更好地理解和处理页面内容。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <code>
        &lt;head&gt;
       </code>
       标签
      </p>
      <ul>
       <li>
        <code>
         &lt;head&gt;
        </code>
        标签包含了文档的元数据，这些数据不会直接显示在网页上，但对网页的运行和搜索引擎优化等方面起着重要作用。
        <ul>
         <li>
          <code>
           &lt;meta charset="UTF-8"&gt;
          </code>
          ：设置文档的字符编码为 UTF - 8，这样可以正确显示各种语言的字符，避免出现乱码问题。
         </li>
         <li>
          <code>
           &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
          </code>
          ：这是一个针对移动设备的元标签，
          <code>
           width=device-width
          </code>
          表示页面宽度等于设备的屏幕宽度，
          <code>
           initial-scale=1.0
          </code>
          表示初始缩放比例为 1.0，确保网页在移动设备上能正确显示和缩放。
         </li>
         <li>
          <code>
           &lt;title&gt;
          </code>
          标签：定义了网页的标题，它会显示在浏览器的标题栏或标签页上，同时也是搜索引擎优化的重要元素之一。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <code>
        &lt;body&gt;
       </code>
       标签
      </p>
      <ul>
       <li>
        <code>
         &lt;body&gt;
        </code>
        标签包含了网页的可见内容，如文本、图像、链接、表格等。所有用户在浏览器中看到的内容都应该放在
        <code>
         &lt;body&gt;
        </code>
        和
        <code>
         &lt;/body&gt;
        </code>
        标签之间。
        <ul>
         <li>
          <code>
           &lt;h1&gt;
          </code>
          标签：表示一级标题，HTML 提供了从
          <code>
           &lt;h1&gt;
          </code>
          到
          <code>
           &lt;h6&gt;
          </code>
          共六级标题，用于组织页面内容的层次结构。
         </li>
         <li>
          <code>
           &lt;p&gt;
          </code>
          标签：用于定义段落，将文本内容划分为不同的段落，使页面结构更清晰。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
    <h2>
     <a id="_112">
     </a>
     网页基本标签
    </h2>
    <h3>
     <a id="h1__h6_114">
     </a>
     标题标签（
     <code>
      &lt;h1&gt;
     </code>
     -
     <code>
      &lt;h6&gt;
     </code>
     ）
    </h3>
    <p>
     <strong>
      功能
     </strong>
     ：
    </p>
    <p>
     用于定义网页中不同级别的标题，从
     <code>
      &lt;h1&gt;
     </code>
     到
     <code>
      &lt;h6&gt;
     </code>
     重要性依次降低，字号也逐渐变小。
     <code>
      &lt;h1&gt;
     </code>
     通常用于页面的主标题，而
     <code>
      &lt;h6&gt;
     </code>
     用于最低级别的子标题。
    </p>
    <p>
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h1</span><span class="token punctuation">&gt;</span></span>这是一级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h1</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h2</span><span class="token punctuation">&gt;</span></span>这是二级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h2</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h3</span><span class="token punctuation">&gt;</span></span>这是三级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h3</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h4</span><span class="token punctuation">&gt;</span></span>这是四级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h4</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h5</span><span class="token punctuation">&gt;</span></span>这是五级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h5</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h6</span><span class="token punctuation">&gt;</span></span>这是六级标题<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h6</span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <p>
     <strong>
      注意事项
     </strong>
     ：
    </p>
    <ul>
     <li>
      搜索引擎会根据标题标签来理解页面的结构和内容重点，因此合理使用标题标签对 SEO 很重要。
     </li>
     <li>
      一般一个页面应该只有一个
      <code>
       &lt;h1&gt;
      </code>
      标签，用于突出页面的核心主题。
     </li>
    </ul>
    <h3>
     <a id="p_138">
     </a>
     段落标签（
     <code>
      &lt;p&gt;
     </code>
     ）
    </h3>
    <p>
     <strong>
      功能
     </strong>
     ：
    </p>
    <p>
     用于将文本内容组织成段落。浏览器会自动在段落前后添加一定的间距，使页面内容更具可读性。
    </p>
    <p>
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>p</span><span class="token punctuation">&gt;</span></span>这是一个段落的内容。它可以包含多行文本，并且浏览器会自动处理段落之间的间距。<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>p</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>p</span><span class="token punctuation">&gt;</span></span>这是另一个段落，与上一个段落会有明显的分隔。<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>p</span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <p>
     <strong>
      注意事项
     </strong>
     ：
    </p>
    <ul>
     <li>
      段落标签是块级元素，会独占一行，并且在前后产生换行效果。
     </li>
     <li>
      可以通过 CSS 对段落的样式（如字体、颜色、行高、边距等）进行进一步的定制。
     </li>
    </ul>
    <h3>
     <a id="br_157">
     </a>
     换行标签（
     <code>
      &lt;br&gt;
     </code>
     ）
    </h3>
    <p>
     <strong>
      功能
     </strong>
     ：
    </p>
    <p>

     </strong>

    </p>

     </strong>

    </p>

      <code>

      <code>

      </code>
      标签。
     </li>
     <li>
      <code>
       &lt;br&gt;
      </code>
      标签在 HTML 中是单标签，不需要闭合。
     </li>
    </ul>

    </p>

    </p>
    <p>
     <strong>
      示例
     </strong>
     ：
    </p>

<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>p</span><span class="token punctuation">&gt;</span></span>这是第二段内容，与第一段通过水平线分隔开来。<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>p</span><span class="token punctuation">&gt;</span></span>

     <strong>
      注意事项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       &lt;hr&gt;
      </code>
      标签的样式（如颜色、粗细、长度等）可以通过 CSS 进行修改。
     </li>
     <li>
      同样，它也是单标签，无需闭合。
     </li>
    </ul>

      </code>

     </li>

     </li>

     </a>

    </h3>
    <p>
     <code>

    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>img</span> <span class="token attr-name">src</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>图像文件的路径或 URL<span class="token punctuation">"</span></span> <span class="token attr-name">alt</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>替代文本<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>

    <h3>
     <a id="_264">
     </a>
     主要属性
    </h3>
    <p>
     <strong>
      <code>
       src
      </code>
      属性
     </strong>
    </p>
    <ul>
     <li>
      <strong>

      </strong>

      </code>
      （source 的缩写）属性是
      <code>

     </li>
     <li>
      <strong>

      </strong>
      <ul>
       <li>
        <strong>
         本地图像
        </strong>

       </li>
      </ul>
     </li>
    </ul>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>img</span> <span class="token attr-name">src</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>example.jpg<span class="token punctuation">"</span></span> <span class="token attr-name">alt</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>示例图片<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>

      <strong>
       作用
      </strong>
      ：
      <code>
       alt
      </code>
      （alternative text 的缩写）属性提供了图像的替代文本。当图像无法显示时（如网络问题、文件路径错误等），浏览器会显示

      </code>

      <code>

      </code>

     <code>
      srcset
     </code>
     和
     <code>
      sizes
     </code>
     属性。
    </p>
    <p>
     <code>
      srcset
     </code>
     和
     <code>
      sizes
     </code>
     属性
    </p>
    <ul>
     <li>
      <strong>
       <code>
        srcset
       </code>
      </strong>
      ：允许提供多个不同分辨率的图像文件，浏览器会根据设备的屏幕分辨率和视口大小选择合适的图像。
     </li>
     <li>
      <strong>
       <code>
        sizes
       </code>
      </strong>
      ：定义不同视口宽度下图像的显示宽度。
     </li>
    </ul>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>img</span> <span class="token attr-name">src</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>small.jpg<span class="token punctuation">"</span></span>
     <span class="token attr-name">srcset</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>small.jpg 500w, medium.jpg 1000w, large.jpg 2000w<span class="token punctuation">"</span></span>
     <span class="token attr-name">sizes</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>(max-width: 500px) 100vw, (max-width: 1000px) 50vw, 33vw<span class="token punctuation">"</span></span>
     <span class="token attr-name">alt</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>响应式示例图片<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>

    <p>
     在上述示例中：
    </p>
    <ul>
     <li>
      <code>
       srcset
      </code>
      列出了不同分辨率的图像文件及其对应的宽度（以像素为单位，后面跟
      <code>
       w
      </code>
      ）。
     </li>
     <li>
      <code>
       sizes
      </code>
      定义了在不同视口宽度下图像的显示宽度。例如，当视口宽度小于等于 500px 时，图像宽度为视口宽度的 100%；当视口宽度小于等于 1000px 时，图像宽度为视口宽度的 50%；否则，图像宽度为视口宽度的 33%。浏览器会根据这些信息选择最合适的图像来显示。
     </li>
    </ul>

    </h2>
    <p>

    <h3>
     <a id="Block_Elements_546">
     </a>
     块元素（Block Elements）
    </h3>
    <h4>
     <a id="_548">
     </a>
     特点
    </h4>
    <ul>
     <li>
      <strong>
       独占一行
      </strong>
      ：块元素在页面中会独自占据一行，无论其内容多少，后续元素会自动换行显示在其下方。
     </li>
     <li>
      <strong>
       可设置宽高
      </strong>
      ：可以通过 CSS 显式地设置块元素的宽度（
      <code>
       width
      </code>
      ）和高度（
      <code>
       height
      </code>
      ）属性。如果不设置宽度，默认情况下，块元素会扩展到其父元素的宽度。
     </li>
     <li>
      <strong>
       可包含其他元素
      </strong>
      ：块元素通常可以包含行内元素和其他块元素，用于构建页面的整体结构。
     </li>

    <h3>
     <a id="Inline_Elements_596">

     </a>
     特点
    </h4>

       &lt;img&gt;

     </li>
     <li>
      <strong>
       只能包含行内元素
      </strong>

     <a id="_688">

     列表
    </h2>
    <p>
     在 HTML 中，列表用于将相关信息进行分组和展示，使页面内容更加有条理。HTML 提供了三种主要类型的列表：无序列表、有序列表和定义列表。以下是对这三种列表的详细介绍：
    </p>
    <h3>
     <a id="Unordered_List_692">

     无序列表（Unordered List）
    </h3>

     <a id="_694">

     概述
    </h4>

      &lt;ul&gt;

     <code>
      &lt;li&gt;

    <h2>
     <a id="_1241">
     </a>
     网页的简单布局
    </h2>
    <p>
     HTML 页面具有特定的结构，合理的页面结构有助于提升网页的可读性、可维护性，同时也有利于搜索引擎优化（SEO）。下面从整体框架、头部信息、主体内容和辅助结构等方面详细分析 HTML 页面结构。
    </p>
    <h3>
     <a id="_1245">
     </a>
     整体框架
    </h3>
    <p>
     一个标准的 HTML 页面包含文档类型声明、HTML 根元素，HTML 根元素内又分为头部（
     <code>
      &lt;head&gt;
     </code>
     ）和主体（
     <code>
      &lt;body&gt;
     </code>
     ）两大部分。以下是基本结构示例：
    </p>

<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>head</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>body</span><span class="token punctuation">&gt;</span></span>

<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>body</span><span class="token punctuation">&gt;</span></span>

     <li>
      <strong>
       <code>
        &lt;!DOCTYPE html&gt;
       </code>
      </strong>
      ：这是文档类型声明，它告诉浏览器当前页面使用的是 HTML5 标准。不同的 HTML 版本有不同的文档类型声明，HTML5 的声明最为简洁。
     </li>
     <li>
      <strong>
       <code>
        &lt;html&gt;
       </code>
      </strong>
      ：HTML 文档的根标签，所有的 HTML 元素都要包含在
      <code>
       &lt;html&gt;
      </code>

      <code>
       &lt;/html&gt;
      </code>
      之间。
      <code>
       lang="zh-CN"
      </code>
      表示页面语言为中文（中国大陆地区），这有助于搜索引擎和屏幕阅读器等工具更好地理解页面内容。
