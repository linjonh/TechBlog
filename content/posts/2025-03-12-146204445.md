    <p>
     Spring Boot 是一个用于快速开发基于 Spring 框架的应用程序的工具。它具有许多特性，其中一些重要的特性包括：
     <img alt="" height="100" src="https://i-blog.csdnimg.cn/direct/154ec9d51e9741c5993ae80a420bf462.png" width="865"/>
    </p>
    <p>
     1. 自动配置：Spring Boot 提供了自动配置的机制，可以根据应用程序的依赖和环境自动配置应用程序的各种组件，大大简化了开发过程。
    </p>
    <p>
     2. 简化的依赖管理：Spring Boot 通过提供一组父项目和Starter模块，简化了依赖管理，开发者只需要添加相应的 Starter 模块就能够引入所需的依赖。
    </p>
    <p>
     3. 内嵌式容器：Spring Boot 提供了内嵌式的容器，如 Tomcat、Jetty 等，使得应用程序可以独立运行，无需部署到外部容器中。
    </p>
    <p>
     4. 生产就绪：Spring Boot 提供了各种生产环境准备的功能，如性能监控、健康检查、配置管理等，可以方便地打包部署到生产环境中。
    </p>
    <p>
     5. 简化的配置：Spring Boot 采用约定大于配置的思想，提供了许多默认值和自动配置，减少了配置文件的编写量。
    </p>
    <p>
     6. 微服务支持：Spring Boot 能够轻松集成 Spring Cloud 微服务框架，支持微服务架构的开发。
    </p>
    <p>
     总的来说，Spring Boot 提供了一种快速、简便的方式来开发基于 Spring 框架的应用程序，使得开发者可以更专注于业务逻辑的实现，而不必过多关注配置和环境搭建等问题。
    </p>

     三、Maven原理（管理和构建Java项目的工具）
     <strong>
      <span style="color:#0f4761">
       <a name="_Toc192604179">
       </a>
      </span>
     </strong>
    </h3>
    <p>
     作用：
    </p>
    <p style="text-align:justify">
     1.方便快捷的依赖管理（jar包）
    </p>
    <p style="margin-left:36pt; margin-right:0; text-align:justify">
     <img alt="" height="215" src="https://i-blog.csdnimg.cn/direct/073b60f3901d4b0da788755d3e134376.png" width="831"/>
    </p>
    <p>
    </p>
    <p style="text-align:justify">
     2.项目构建（快速完成项目的编译、测试、打包、发布）
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="411" src="https://i-blog.csdnimg.cn/direct/1d1ff8eb8b1c45548af52765290cdf92.png" width="830"/>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     3.统一项目结构（使用Maven构建出来的项目，无论使用什么开发工具创建出来的Maven目，在使用其他开发工具的时候都可以直接导入）
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     仓库（本地仓库、中央仓库、私服仓库）：用于存储和管理jar包
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="759" src="https://i-blog.csdnimg.cn/direct/db7fb6284be04976a0971cd2433997ec.png" width="830"/>
    </p>
    <blockquote>
     <h4 style="margin-left:0px; margin-right:0px; text-align:justify">
      Maven坐标：
     </h4>
     <p>
      什么是坐标?
      <br/>
      Maven 中的坐标是资源( jar)的唯一标识，通过该坐标可以唯一定位资源位置
      <br/>
      使用坐标来定义项目或引入项目中需要的依赖
     </p>
    </blockquote>
    <blockquote>
     <h4>
      <strong>
       Maven 坐标主要组成
      </strong>
     </h4>
     <p>
      <span style="color:#be191c">
       groupId
      </span>
      :定义当前Maven项目隶属组织名称(通常是域名反写，例如:com.itheima)
      <br/>
      <span style="color:#be191c">
       artifactId
      </span>
      :定义当前Maven项目名称(通常是模块名称，例如 order-service、goods-service)
      <br/>
      <span style="color:#be191c">
       version
      </span>
      :定义当前项目版本号
      <br/>
      <span style="color:#b95514">
       SNAPSHOT
      </span>
      :功能不稳定、尚处于开发中的版本，即快照版本
      <br/>
      <span style="color:#b95514">
       RELEASE
      </span>
      :功能趋于稳定、当前更新停止，可以用于发行的版本
     </p>
    </blockquote>

    <h4 style="margin-left:0px; margin-right:0px; text-align:justify">
     <span style="color:#1a439c">
      依赖配置
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     官网（https://mvnrepository.com/）
    </p>
    <blockquote>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      <span style="color:#1a439c">
       <strong>
        依赖:
       </strong>
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      指当前项目运行所需要的jar包，一个项目中可以引入多个依赖。
     </p>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      <br/>
      <span style="color:#1a439c">
       <strong>
        配置:
       </strong>
      </span>
      <br/>
      1.在 pom.xml 中编写 &lt;dependencies&gt; 标签
      <br/>
      2.在&lt;dependencies&gt;标签中 使用&lt;dependency&gt; 引入坐标
      <br/>
      3.定义坐标的
      <span style="color:#be191c">
       groupId
      </span>
      ,
      <span style="color:#be191c">
       artifactId
      </span>
      ，
      <span style="color:#be191c">
       version
      </span>
      <br/>
      4.点击刷新按钮，引入最新加入的坐标
     </p>
    </blockquote>
    <h4>
     <span style="color:#1a439c">
      <strong>
       Maven生命周期
      </strong>
     </span>
    </h4>
    <h4 style="margin-left:0px; margin-right:0px; text-align:justify">
    </h4>
    <h4 style="margin-left:0px; margin-right:0px; text-align:justify">
     Maven的生命周期就是为了对所有的maven项目构建过程进行抽象和统一
    </h4>
    <blockquote>
     <p>
      Maven中有3套相互独立的生命周期:
      <br/>
      <span style="color:#be191c">
       <strong>
        clean
       </strong>
      </span>
      :清理工作
      <br/>
      <strong>
       <span style="color:#be191c">
        default
       </span>
      </strong>
      :核心工作，如:编译、测试、打包、安装、部署等
      <br/>
      <strong>
       <span style="color:#be191c">
        site
       </span>
      </strong>
      :生成报告、发布站点等。
     </p>
    </blockquote>
    <p style="margin-left:0; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:0; margin-right:0; text-align:justify">
     <img alt="" height="404" src="https://i-blog.csdnimg.cn/direct/4687af446ea64024bd8b2c3b522129bd.png" width="831"/>
    </p>
    <blockquote>
     <p style="margin-left:0; margin-right:0; text-align:justify">
      生命周期阶段
      <br/>
      <span style="color:#be191c">
       <strong>
        clean
       </strong>
      </span>
      :移除上一次构建生成的文件
      <br/>
      <span style="color:#be191c">
       <strong>
        compile
       </strong>
      </span>
      :编译项目源代码
      <br/>
      <span style="color:#be191c">
       <strong>
        test
       </strong>
      </span>
      :使用合适的单元测试框架运行测试(junit)
      <br/>
      <span style="color:#be191c">
       <strong>
        package
       </strong>
      </span>
      :将编译后的文件打包，如:jar、war等
      <br/>
      <span style="color:#be191c">
       <strong>
        install
       </strong>
      </span>
      :安装项目到本地仓库
     </p>
    </blockquote>
    <p>
     <span style="color:#fe2c24">
      <strong>
       <span style="background-color:#fefcd8">
        注意 : 在同一套生命周期中，当运行后面的阶段时，前面的阶段都会运行
       </span>
      </strong>
     </span>
    </p>
