    <h2>
     问题：
    </h2>
    <h3>
     在文件读取，判断md5值时，遇到py文件读取转String后，再转byte，md5前后不一致问题。
    </h3>
    <h2>
     解决方法：
    </h2>
    <h4>
     <span style="color:#0d0016">
      python文件读取要使用
     </span>
     <span style="color:#fe2c24">
      <strong>
       QTextStream
      </strong>
     </span>
     <span style="color:#0d0016">
      ，避免\t 、\r、\n的换行符跨平台问题（window系统换行符和linux换行符不一致）。
     </span>
    </h4>
    <p>
     <span style="color:#fe2c24">
      <strong>
       QTextStream默认帮你处理了换行符跨平台问题。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      \r：回到开头   \n：换行
     </span>
    </p>
    <p>
     <span style="color:null">
      一般读取文件的方式是：
     </span>
    </p>
    <pre><code class="language-cpp">bool xxxxClass::readFile(const QString &amp;filePath,QByteArray &amp;bytes)
{
    QFile file(filePath);
    if(!file.open(QIODevice::ReadOnly)){
        QString msg = tr("Read [%1] faild: %2").arg(file.fileName(),file.errorString());
        qDebug()&lt;&lt;msg;
        return false;
    }
    bytes = file.readAll();
    file.close();
}</code></pre>
    <p>
     但是这样读出来的python文件会有问题，所以python要使用数据流去读取。
    </p>
    <pre><code class="language-cpp">bool xxxxClass::readTextFile(const QString &amp;filePath,QString &amp;text)
{
    QFile file(filePath);
    if(!file.open(QIODevice::ReadOnly | QIODevice::Text)){
        QString msg = tr("Read [%1] faild: %2").arg(file.fileName(),file.errorString());
        qDebug()&lt;&lt;msg;
        return false;
    }
    QTextStream ts(&amp;file);
    text = ts.readAll();
    file.close();
    return true;
}</code></pre>
