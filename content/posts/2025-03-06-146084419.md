     <h4>
      3.3 综合分析
     </h4>
     <p style="text-align:justify;">
      综合两图来看，系统采用经典的
      <strong>
       RAG（Retrieval-Augmented Generation）
      </strong>
      架构：
     </p>
     <p style="text-align:justify;">
      用户查询首先通过向量化模型（
      <code>
       shibing624/text2vec-base-chinese
      </code>
      ）转化为向量，在 Elasticsearch 中检索相关文档，随后将查询与检索结果传递给 Qwen2.5-14b 生成最终回答。
     </p>
     <p style="text-align:justify;">
      Gradio 提供友好界面，FSCrawler 确保数据摄取的自动化，而公网接口则扩展了系统的应用场景。
     </p>
     <p>
      检索增强生成（RAG）简单来说就是
      <strong>
       先从一大堆文档中找到跟用户问题相关的内容，然后用大语言模型把这些内容整理成自然、易懂的回答
      </strong>
      。
     </p>

     <h4>
      4.2 向量化

     <h4>
      4.3 查询处理与检索
     </h4>
     <p style="text-align:justify;">
      将用户查询向量化并在 Elasticsearch 中执行相似性搜索，这里本质做的是关键词匹配+向量检索的组合方式。
     </p>
     <p style="text-align:center;">
      <img alt="149c03220924c048b831d69bacd4cc8d.png" src="https://i-blog.csdnimg.cn/img_convert/149c03220924c048b831d69bacd4cc8d.png"/>
     </p>
     <p style="text-align:center;">
      <img alt="46e89d175e3f5a591f91c49f707d17dd.png" src="https://i-blog.csdnimg.cn/img_convert/46e89d175e3f5a591f91c49f707d17dd.png"/>
     </p>
     <h4>
      4.4 回答生成
     </h4>
     <p style="text-align:justify;">
      将查询和检索结果传递给 Qwen2.5-14b 生成回答：
     </p>
     <p style="text-align:center;">
      <img alt="a0af873ae9f4504018b1bd35e55dd248.png" src="https://i-blog.csdnimg.cn/img_convert/a0af873ae9f4504018b1bd35e55dd248.png"/>
     </p>
     <pre class="has"><code class="language-go">from ollama import Client
ollama = Client()
context = "\n".join([doc['_source']['text'] for doc in results['hits']['hits']])
prompt = f"根据以下内容回答问题：\n{context}\n问题：{query}"
response = ollama.generate(model='qwen2.5:14b', prompt=prompt)
answer = response['text']</code></pre>
     <h4>
      4.5 Gradio 界面
     </h4>
     <p style="text-align:justify;">

def qa_system(query):
    # 查询处理、检索、生成逻辑
    return answer
interface = gr.Interface(fn=qa_system, inputs="text", outputs="text")
interface.launch()</code></pre>
