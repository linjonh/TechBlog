# 微软为何贡献Linux源代码？

<!-- @page { margin: 2cm } P { margin-bottom: 0.21cm } -->

毫无疑问，



Linux


是微软的竞争对手。但是，微软为什么还要给



Linux


贡献驱动源代码，助



Linux


一臂之力？岂非头脑发疯了？事实究竟是怎样的？

7

月


21

日下午两点，有一位朋友从美国打电话给我，他对我说，这件事情，在这边（指美国），简直闹翻了天。对这件事情，我和他的感觉明显不一样。随着时间的流逝，各种议论都出现了。有人说，微软此举只是一种服务器的销售策略，没啥了不起。这种看法，具有一定的代表性。在我们国内，这种说法也很普遍。那么，我们到底该如何看待这一事件？

当今，一些搞


Linux

的人，头脑里面缺少一根弦。他们认为，我搞


Linux

，难道还要听从什么人的（话）？我说，这种想法不对。实际上，全球


Linux

有个思想指挥司令部，它就是


www.Linux .com

，《


Linux

基金会》的官方网站。凡是有关


Linux

的事情，我们先要看看


Linux

司令部是怎么说的。这该不算是奴隶主义吧？

7

月


21

日，《


Linux

基金会》官方网站主持人


Brian Proffitt

发表文章，题为“


Linux kernel to get Microsoft code”

，披露一件鲜为人知的事情。知名


Linux

内核开发者


Greg Kroah-Hartman

，是


Linux

驱动计划（


LDP

）项目的领导人，他曾通过这个


LDP

项目与微软开发者一块工作。他本人也是微软这次提交的


Linux

驱动代码的架构师之一。他针对微软此举在自己的博客短文中说（大意），微软驱动代码能够使得


Linux

在


Hyper-V

平台上运行得更好，虽然还有一些工作需要去做，但是，我们还是接受了这些代码（指放入


Linux

代码树）。

《


Linux

基金会》官方网站对微软此举持正面态度，称：“这次事件标志着微软第一次给


Linux

内核贡献代码，也是微软第一次遵照


GPL

规则发布重要程序代码。”我们应该看到，微软此举符合两方面的需要（或利益），不仅是微软单方面的需要，也是


Linux

方面的需要（请注意这点）。《


Linux

基金会》执行主席


Jim Zemlin

对微软为


Linux

内核贡献代码一事公开表示：“对此，我很开心。”我们要问，


Jim Zemlin

为什么很开心呢？原因就是，微软对


Linux

开始打招呼了，


Hello

，


Linux

！

微软给竞争对手


Linux

提供驱动程序源代码（等于提供炮弹），在整个世界软件业发展史上，必将成为一段佳话。这好比在竞赛场上，微软对


Linux

说，老弟，快跑，而且还伸手拉了


Linux

一把。


Linux

与


Windows

，两者相争是必然的，但是，这种竞争应该是友好的，而不能是怒目相视，恶语对骂。在这个世界上，总有些人的心理是昏暗的。他们看待别人的眼光，总是坏的大大的。我深信，微软人的心目中也有光明的天空，并不都是黑乎乎的。对付微软，在技术上，我们不能手软；在交往上，应该是真心朋友。尤其在中国，卖软件是何等困难之事，何等无奈之举，微软始终在坚持着，在努力着，从不气馁。


Linux

与微软，在打击盗版问题上，应该是无间盟友。
