     </strong>

     </strong>

      </strong>
     </li>
     <li>
      <strong>

      </strong>
      <strong>

      </strong>

      </strong>
     </li>
     <li>
      <strong>

      <strong>

      </strong>

      </strong>
      <strong>

      </strong>
      <strong>

      </strong>
      <strong>
       会发生错误。
      </strong>
     </li>

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <span style="color:#ff0000">
       注
      </span>
     </strong>
     <strong>
      <span style="color:#ff0000">
       ：
      </span>
     </strong>
     <strong>
      在
     </strong>
     <strong>
      PG中
     </strong>
     <strong>
      ，
     </strong>
     <strong>
      默认
     </strong>
     <strong>
      的事务隔离级别是读已提交，而读未提交相当于是读已提交的一个更弱的版本。
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      这是
     </strong>
     <strong>
      因为读已提交的隔离级别要求一个事务只能看到已经提交的数据修改，这意味着一个
     </strong>
     <strong>
      事务不能读取
     </strong>
     <strong>
      另一个事务尚未提交的数据变换
     </strong>
     <strong>
      。那么这种
     </strong>
     <strong>
      隔离级别提供了一定的数据一致性，防止脏读。这也是
     </strong>
     <strong>
      为什么PG中
     </strong>
     <strong>
      采用这种隔离级别为默认的隔离级别，因为他的适应性比较广。
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      而相比于
     </strong>
     <strong>
      读已提交，读未提交的
     </strong>
     <strong>
      隔离级别
     </strong>
     <strong>
      不要求事务等待其他事务的提交，它允许一个事务读取另一个事务尚未提交的数据变化，这种
     </strong>
     <strong>
      隔离
     </strong>
     <strong>
      级别容易导致脏读，因为一个事务可能会读取到另一个正在被修改的事务，而这些
     </strong>
     <strong>
      事务
     </strong>
     <strong>
      可能会回滚，从而导致数据状态前后不一致
     </strong>
     <strong>
      。
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      并且PG中
     </strong>
     <strong>
      内部没有实现读未提交的隔离级别，
     </strong>
     <strong>
      因为读未提交
     </strong>
     <strong>
      的隔离级别存在严重的数据一致性问题，不符合事务的
     </strong>
     <strong>
      ACID特性
     </strong>
     <strong>
      ，所以并未实现读未提交，主要还是为了确保数据库的一致性和可靠性。
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <h3 style="background-color:transparent; text-align:justify">
     <strong>
      <strong>
       <strong>
        2.
       </strong>
      </strong>
      <strong>
       <strong>

       </strong>

      </strong>
     </strong>

       P

      </strong>
      <strong>

      </strong>
     </li>
     <li>
      <strong>
       事务只能
      </strong>
      <strong>
       看到已经提交的数据
      </strong>
     </li>
     <li>
      <strong>
       可能会发生不可重复读
      </strong>
      <strong>
       和幻读
      </strong>
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      案例
     </strong>
     <strong>
      ：
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>

     </strong>
     <strong>
      创建一个

     <strong>
      示例

     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      create table products(id serial primary key,name varchar(50),price numeric(10,2));

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

     这个案例说明了在读已提交隔离级别中，一个事务只能看到其他事务已经提交的数据修改，如果事务未提交，那么对于查询事务来说，这些修改是不可见的，直到其他事务提交为止
    </p>
    <h3 style="background-color:transparent; text-align:justify">
     <strong>
      <strong>
       <strong>
        2.2
       </strong>
      </strong>
      <strong>
       <strong>
        可重复读
       </strong>
      </strong>
     </strong>
    </h3>
    <ol>
     <li>
      保证在同一事务内的查询不会受到其他事务的影响
     </li>
     <li>
      防止不可重复读，但是仍然可能发生幻读
     </li>
     <li>
      适用场景：需要一定程度的数据一致性，可以容忍轻微的幻读。
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     案例
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     沿用上面的products表的数据，将修改的数据修改回原样。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="180" src="https://i-blog.csdnimg.cn/direct/cc17893c87d14e8a894fa966dad76f5b.png" width="618"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     会话1：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     begin isolation level repeatable read ;
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     select * from products where id =1;

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     会话2:
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     此时会话2.执行更新语句，但不提交，在会话1中查看数据。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="129" src="https://i-blog.csdnimg.cn/direct/2e79b167ade4443dafd8b147b5eacf66.png" width="713"/>

    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     会话2：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     对事务进行提交，再在事务1中查询对应的数据
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="81" src="https://i-blog.csdnimg.cn/direct/78b9155b92484a3f98e9c1b7cb63f8ca.png" width="333"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="278" src="https://i-blog.csdnimg.cn/direct/9a838600bd05426c81444cc44fa2079a.png" width="659"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     可以发现，在会话1中，无论会话2提不提交，查询的结果都保持不变，即使会话2提交更新操作，查询结果仍然不变。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <span style="background-color:#ffff00">
       此时
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffff00">
       ，除非对
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffff00">
       会话1也进行提交
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffff00">
       ，才可以看到改动
      </span>
     </strong>
     <strong>
      ，
     </strong>
     <strong>
      就相当于在事务的一开始对数据进行了快照，无论
     </strong>
     <strong>
      如何改动
     </strong>
     <strong>
      ，都不影响读取的结果。
     </strong>
    </p>
    <h3 style="background-color:transparent; text-align:justify">
     <strong>
      <strong>
       <strong>
        2.3
       </strong>
      </strong>
      <strong>
       <strong>
        串行化
       </strong>
      </strong>
     </strong>
    </h3>
    <ol>
     <li>
      最高级别的隔离，确保事务之间没有任何的交错或者是并发问题
     </li>
     <li>
      通过锁定数据来实现，防止任何形式的并发问题，包括脏读、不可重复读、幻读
     </li>
     <li>
      性能最差，因为他几乎完全禁止并发
     </li>
     <li>
      适用场景：对数据一致性要求非常高，可以容忍较低的性能.
     </li>
     <li>
      事务2会进行回滚，提交失败以后，事务的回滚其实就相当于没有执行
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     案例：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     会话1：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     开始串行化隔离，插入一条新的数据，查询现在的价格
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="284" src="https://i-blog.csdnimg.cn/direct/228113916ab644509241d5de92429dfc.png" width="711"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     会话2；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     执行与会话1同样的操作，
     <span style="background-color:#ffff00">
      其中开启事务
     </span>
     <span style="background-color:#ffff00">
      的语句是同时执行的
     </span>
     ，然后再分别插入数据。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="279" src="https://i-blog.csdnimg.cn/direct/aa7c286e72dd48328a87cac4371d5792.png" width="674"/>
    </p>

      可以看到，
     </strong>
     <strong>
      两个会话中
     </strong>
     <strong>
      新
     </strong>
     <strong>
      插入的语句只有在
     </strong>
     <strong>
      <span style="background-color:#ffff00">
       自己的
      </span>
     </strong>
     <strong>
      <span style="background-color:#ffff00">
       事务
      </span>
     </strong>
     <strong>
      中才可以查看到
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      此时
     </strong>
     <strong>
      将会话
     </strong>
     <strong>
      1事务进行
     </strong>
     <strong>
      提交，
     </strong>
     <strong>
      然后
     </strong>
     <strong>

     </strong>
     <strong>

     </strong>
     <strong>

     </strong>
