    <p>
    </p>
    <br/>
    在CTF中遇到Git源代码泄露题目时，通常可以通过以下步骤解决：
    <p>
    </p>
    <hr/>
    <h2>
     <a id="1_git_5">
     </a>
     1. 确认存在.git目录泄露
    </h2>
    <ul>
     <li>
      访问目标URL的
      <code>
       /.git/
      </code>
      路径（如
      <code>
       http://example.com/.git/
      </code>
      ），观察返回状态：
      <ul>
       <li>
        403 Forbidden：可能存在目录，但无权限访问。
       </li>
       <li>
        404 Not Found：目录不存在。
       </li>
      </ul>
     </li>
     <li>
      使用工具扫描目录（如
      <code>
       dirsearch
      </code>
      ）确认是否存在
      <code>
       .git
      </code>
      。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="2_git_13">
     </a>
     2. 下载完整的.git目录
    </h2>
    <ul>
     <li>
      工具推荐：
      <ul>
       <li>
        git-dumper（推荐）：
        <pre><code class="prism language-bash">pip <span class="token function">install</span> git-dumper
git-dumper http://example.com/.git/ ./output
</code></pre>
       </li>
       <li>
        wget（手动下载）：
        <pre><code class="prism language-bash"><span class="token function">wget</span> <span class="token parameter variable">-r</span> --no-parent http://example.com/.git/
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="3_Git_27">
     </a>
     3. 恢复Git仓库历史
    </h2>
    <ul>
     <li>
      进入下载的目录，使用Git命令分析：
      <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> ./output
<span class="token function">git</span> status      <span class="token comment"># 查看未提交的更改</span>
<span class="token function">git</span> log         <span class="token comment"># 查看提交历史</span>
<span class="token function">git</span> reflog      <span class="token comment"># 查看所有操作记录（包括reset、rebase）</span>
<span class="token function">git</span> branch <span class="token parameter variable">-a</span>   <span class="token comment"># 查看所有分支</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="4_Flag_39">
     </a>
     4. 查找Flag的常见位置
    </h2>
    <ul>
     <li>
      <p>
       历史提交中的敏感信息：
      </p>
      <ul>
       <li>
        检查删除或修改的文件：
        <pre><code class="prism language-bash"><span class="token function">git</span> log --diff-filter<span class="token operator">=</span>D <span class="token parameter variable">--summary</span>  <span class="token comment"># 查找被删除的文件</span>
<span class="token function">git</span> checkout <span class="token operator">&lt;</span>commit_hash<span class="token operator">&gt;</span>^ -- deleted_file.txt  <span class="token comment"># 恢复被删除的文件</span>
</code></pre>
       </li>
       <li>
        对比不同提交的差异：
        <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">diff</span> <span class="token operator">&lt;</span>commit_hash_<span class="token operator"><span class="token file-descriptor important">1</span>&gt;</span> <span class="token operator">&lt;</span>commit_hash_<span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       分支或标签中的未合并内容：
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> checkout feature-branch   <span class="token comment"># 切换到其他分支</span>
<span class="token function">git</span> tag <span class="token parameter variable">-l</span>                   <span class="token comment"># 查看所有标签</span>
</code></pre>
     </li>
     <li>
      <p>
       暂存区或工作区内容：
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> stash list        <span class="token comment"># 查看暂存记录</span>
<span class="token function">git</span> stash apply      <span class="token comment"># 恢复最近的暂存内容</span>
</code></pre>
     </li>
     <li>
      <p>
       Git对象文件（手动解析）：
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> cat-file <span class="token parameter variable">-p</span> <span class="token operator">&lt;</span>object_hash<span class="token operator">&gt;</span>  <span class="token comment"># 查看对象内容（如commit、tree、blob）</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="5_git_70">
     </a>
     5. 处理不完整的.git目录
    </h2>
    <ul>
     <li>
      若仓库不完整，使用工具修复：
      <ul>
       <li>
        git fsck：检查对象完整性。
       </li>
       <li>
        scrabble：提取所有Git对象中的文件。
        <pre><code class="prism language-bash">scrabble <span class="token parameter variable">-i</span> .git/objects/ <span class="token parameter variable">-o</span> extracted_files/
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="6__80">
     </a>
     6. 其他技巧
    </h2>
    <ul>
     <li>
      检查.git/config：可能含敏感信息（如远程仓库密码）。
     </li>
     <li>
      搜索所有文件内容：
      <pre><code class="prism language-bash"><span class="token function">grep</span> <span class="token parameter variable">-rnw</span> <span class="token builtin class-name">.</span> <span class="token parameter variable">-e</span> <span class="token string">"flag{"</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="_89">
     </a>
     示例流程
    </h2>
    <pre><code class="prism language-bash"><span class="token comment"># 下载.git目录</span>
git-dumper http://ctf.example.com/.git/ ./leaked_repo

<span class="token comment"># 分析提交历史</span>
<span class="token builtin class-name">cd</span> leaked_repo
<span class="token function">git</span> log <span class="token parameter variable">--oneline</span>

<span class="token comment"># 恢复被删除的flag文件</span>
<span class="token function">git</span> checkout abc123^ -- flag.txt

<span class="token comment"># 或切换到其他分支</span>
<span class="token function">git</span> checkout dev-branch
</code></pre>
    <hr/>
    <p>
     通过以上步骤，通常可以找到隐藏在Git历史记录、分支或对象中的flag。注意仔细检查所有可能的位置，包括被删除或未提交的内容。
    </p>
