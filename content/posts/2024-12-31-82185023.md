    <p>
     当需要远程办公时，使用pycharm远程连接服务器时必要的。
     <br/>
     PyCharm提供两种远程调试(Remote Debugging)的方式:
    </p>
    <ul>
     <li>
      配置远程的解释器（remote interpreter）
     </li>
     <li>
      配置Python调试服务器（Python Debug Server）
     </li>
    </ul>
    <p>
     本篇文章主要介绍配置远程解释器，关于配置 python调试服务器，文末也会简单介绍。
    </p>
    <h2>
     <a id="1_Pycharm_7">
     </a>
     1. Pycharm连接远程服务器
    </h2>
    <h4>
     <a id="11__9">
     </a>
     1.1 进入配置页面
    </h4>
    <p>
     Pycharm菜单栏，如下图所示，依次点击 Tools -&gt; Deployment -&gt; Configration…
     <br/>
     <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ad662e0a4ebcf9da233adffacc9c3c43.png"/>
    </p>
    <h4>
     <a id="12__13">
     </a>
     1.2 配置连接服务器
    </h4>
    <ul>
     <li>
      如下图。name随便写个就行。
     </li>
     <li>
      Connection下，协议最好选择sftp，接下来填写服务器主机IP，用户名，密码。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9e4f4473c6e277b4f310ae7ae111d619.png"/>
     </li>
     <li>
      点击Test SFTP connection会发现，如果连接成功会提示你如下
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1fe5dc7970c25052518e69de76b46306.png"/>
     </li>
     <li>
      在Mapping下，选择连接windows下的那部分代码和服务器上代码相连，本地Local path，服务器path，apply，OK，表示已经把本地的代码和服务器代码连接上了。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f216fc6862aaf5c9b0eea60ace9cfaa1.png"/>
     </li>
    </ul>
    <h4>
     <a id="13__22">
     </a>
     1.3 上传代码，使得本地代码和服务器代码保持同步
    </h4>
    <ul>
     <li>
      点击Upload to name（刚才填写的远程服务器名字），即可上传代码。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3ea0a62ec431d20cf7533e7a41acffd7.png"/>
     </li>
     <li>
      若配置有多个不同服务器或同一个服务器配置了多个 服务器上传路径（Deployment Path），可选择 Upload to…，上传到不同的服务器/路径。
     </li>
    </ul>
    <h2>
     <a id="2_Python_27">
     </a>
     2. 配置远程Python解释器
    </h2>
    <p>
     使用服务器调试Python程序的前提时在服务器上安装了Python解释器，如果没安装，请先安装。
    </p>
    <h4>
     <a id="21_Python_30">
     </a>
     2.1 将Python解释器设置为远程服务器上的
    </h4>
    <ul>
     <li>
      在菜单栏，File -&gt; Settings… -&gt; Project ×× -&gt; Project Interpreter，点击右侧 Add按钮，添加解释器。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d6c825b2a2a3f1f30f536ddd4201bfa8.png"/>
     </li>
     <li>
      选择SSH Interpreter，填写服务器的 Host 地址，端口Port，用户名Username，填好后，下一步Next。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f4445526f623a40a8dc2a68b41d16278.png"/>
     </li>
     <li>
      填写密码 Password，下一步Next。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/73c04cf8e38c7a878a209162df804a1b.jpeg"/>
     </li>
     <li>
      选择远程服务器上Python解释器的位置，服务器上的远程同步文件夹Sync folders，可以选择多个。如果不知道Python安装在哪，可以远程连接服务器后，使用 命令 which python 找到Python安装位置。
      <br/>
      <img alt="这里写图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4ba60dff1b436ea5129ac9a4a5aeba26.png"/>
     </li>
     <li>
      Finish，配置结束。该项目现在使用的就是远程服务器上的Python解释器了。以后的项目若想/不想使用该解释器，手动更改解释器即可。
     </li>
    </ul>
    <h2>
     <a id="3_Python_41">
     </a>
     3. 使用远程解释器运行本地Python程序
    </h2>
    <ul>
     <li>
      将测试代码上传至远程服务器（Tooles -&gt; Deployment -&gt; Upload to ××）。
     </li>
     <li>
      Run测试代码，可以看到现在代码是在远程服务器上运行了。
     </li>
    </ul>
