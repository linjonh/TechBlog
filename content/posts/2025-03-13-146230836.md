    <p>
     <strong>
      unittest vs pytest 对比
     </strong>
    </p>
    <blockquote>
     <ul>
      <li>
       ​
       <strong>
        unittest
       </strong>
       像“手动挡汽车”：操作步骤多，规则严格，适合老司机。
      </li>
      <li>
       ​
       <strong>
        pytest
       </strong>
       像“自动挡汽车”：开起来轻松，功能强大，适合新手和高效开发。
      </li>
     </ul>
     <table>
      <thead>
       <tr>
        <th>
         <strong>
          区别点
         </strong>
        </th>
        <th>
         ​
         <strong>
          unittest
         </strong>
         ​（你学过的）
        </th>
        <th>
         ​
         <strong>
          pytest
         </strong>
         ​（更强大的新工具）
        </th>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          测试用例写法
         </strong>
        </td>
        <td>
         必须写一个类，继承
         <code>
          unittest.TestCase
         </code>
        </td>
        <td>
         不用类！直接写函数，函数名以
         <code>
          test_
         </code>
         开头就行
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          断言语法
         </strong>
        </td>
        <td>
         必须用
         <code>
          self.assertEqual(a, b)
         </code>
         这种固定方法
        </td>
        <td>
         直接用
         <code>
          assert a == b
         </code>
         ，像写普通 Python 代码一样简单
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          参数化测试
         </strong>
        </td>
        <td>
         需要安装第三方库（如
         <code>
          ddt
         </code>
         ）
        </td>
        <td>
         自带参数化功能，一行代码搞定
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          管理测试数据
         </strong>
        </td>
        <td>
         用
         <code>
          setUp()
         </code>
         和
         <code>
          tearDown()
         </code>
         初始化/清理数据
        </td>
        <td>
         用
         <code>
          fixture
         </code>
         功能，灵活定义数据，可重复使用
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          运行测试
         </strong>
        </td>
        <td>
         需手动指定测试文件（如
         <code>
          python -m unittest test.py
         </code>
         ）
        </td>
        <td>
         自动发现所有
         <code>
          test_*.py
         </code>
         文件，命令行输入
         <code>
          pytest
         </code>
         就行
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          插件和扩展
         </strong>
        </td>
        <td>
         功能有限，扩展性差
        </td>
        <td>
         超多插件！比如生成HTML报告、多线程加速测试
        </td>
       </tr>
       <tr>
        <td>
         ​
         <strong>
          错误提示
         </strong>
        </td>
        <td>
         报错信息简单
        </td>
        <td>
         错误信息更清晰，直接告诉你哪一行出问题
        </td>
       </tr>
      </thead>
     </table>
    </blockquote>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <blockquote>
     <ul>
      <li>
       “unittest 是 Python 自带的测试框架，规则比较严格：需要写测试类继承
       <code>
        TestCase
       </code>
       ，断言要用
       <code>
        self.assertEqual()
       </code>
       等方法，适合小型项目。
      </li>
      <li>
       pytest 是更现代的框架，语法更简洁：比如直接用
       <code>
        assert
       </code>
       断言，支持参数化测试和灵活的
       <code>
        fixture
       </code>
       管理，还能通过插件扩展功能（比如生成测试报告）。
      </li>
      <li>
       虽然我之前主要用 unittest，但 pytest 的优势非常明显。如果有需要，我可以快速学习并应用到项目中，提升测试效率和覆盖度。”
      </li>
     </ul>
    </blockquote>
