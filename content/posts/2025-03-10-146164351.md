     Apache Doris中都用了哪些开发语言，编译过程中用到了哪些编译器，以及用到了哪些成熟的技术框架

     </path>

    <p>
     Apache Doris 作为一款高性能的实时分析型数据库，其技术栈涉及多语言开发、多种编译器支持以及多个成熟技术框架的集成。以下是综合多个来源的详细分析：
    </p>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     <strong>
      一、开发语言
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        Java
       </strong>
       <br/>
       •
       <strong>
        应用场景
       </strong>
       ：主要用于开发
       <strong>
        Frontend（FE）
       </strong>
       ，负责元数据管理、查询解析、集群管理等模块。
       <br/>
       •
       <strong>
        关键模块
       </strong>
       ：
       <br/>
       ◦ FE 的元数据持久化通过
       <strong>
        BDBJE（Berkeley DB Java Edition）
       </strong>
       实现。
       <br/>
       ◦ MySQL 协议兼容和 HTTP 服务分别基于
       <strong>
        Xnio
       </strong>
       和
       <strong>
        SpringBoot
       </strong>
       框架。
      </p>
     </li>
     <li>
      <p>
       <strong>
        C++
       </strong>
       <br/>
       •
       <strong>
        应用场景
       </strong>
       ：用于开发
       <strong>
        Backend（BE）
       </strong>
       ，核心功能包括数据存储、向量化计算引擎、MPP 架构的分布式查询执行。
       <br/>
       •
       <strong>
        技术优势
       </strong>
       ：通过 C++ 实现高性能的列式存储和实时数据处理。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_17">
     </a>
     <strong>
      二、编译器与构建工具
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        编译器
       </strong>
       <br/>
       •
       <strong>
        GCC
       </strong>
       ：推荐使用
       <strong>
        GCC 7.3 及以上版本
       </strong>
       （如 Ubuntu 中安装
       <code>
        gcc-10
       </code>
       ），用于编译 C++ 代码。
       <br/>
       •
       <strong>
        Clang
       </strong>
       ：作为替代编译器，支持 C++17 特性（如
       <code>
        &lt;filesystem&gt;
       </code>
       头文件）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        构建工具
       </strong>
       <br/>
       •
       <strong>
        CMake
       </strong>
       ：管理 C++ 项目的构建流程。
       <br/>
       •
       <strong>
        Maven
       </strong>
       ：管理 Java 项目的依赖（如 FE 模块）。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_28">
     </a>
     <strong>
      三、成熟技术框架（除用户提及外）
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        BDBJE（Berkeley DB Java Edition）
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：用于 FE 节点的元数据持久化和集群选举，通过操作日志（EditLog）实现元数据同步。
       <br/>
       •
       <strong>
        场景
       </strong>
       ：支持多副本机制和自动故障恢复，确保集群高可用性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Thrift
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：实现 RPC 通信，用于 FE 和 BE 节点间的交互（如查询任务下发、数据分片管理）。
       <br/>
       •
       <strong>
        版本兼容性
       </strong>
       ：从 Doris 0.15 版本开始，Thrift 从 0.9 升级到 0.13，需通过
       <code>
        --clean
       </code>
       参数强制重新生成代码。
      </p>
     </li>
     <li>
      <p>
       <strong>
        SpringBoot
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：构建 FE 的 HTTP 服务，提供管控接口和监控指标。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Xnio
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：基于 JBoss 的异步 IO 框架，实现 FE 的 MySQL 协议兼容，支持高并发连接处理。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Google Mesa 与 Apache Impala 技术集成
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：Doris 融合了 Mesa 的增量数据更新机制和 Impala 的查询优化技术，实现高性能分析能力。
      </p>
     </li>
     <li>
      <p>
       <strong>
        MPP（大规模并行处理）架构
       </strong>
       <br/>
       •
       <strong>
        功能
       </strong>
       ：通过分布式节点并行执行查询任务，支持横向扩展至数百节点。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_51">
     </a>
     <strong>
      四、其他关键技术与生态集成
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        数据湖与外部存储
       </strong>
       <br/>
       •
       <strong>
        支持框架
       </strong>
       ：HDFS、S3、Iceberg、Hudi 等，实现联邦查询和存算分离架构（Doris 3.0+）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        流批一体
       </strong>
       <br/>
       •
       <strong>
        集成工具
       </strong>
       ：Flink、Kafka，支持实时数据导入和流式计算。
      </p>
     </li>
     <li>
      <p>
       <strong>
        容器化部署
       </strong>
       <br/>
       •
       <strong>
        工具
       </strong>
       ：Docker，官方提供预编译镜像（如
       <code>
        apache/doris:build-env-ldb-toolchain-latest
       </code>
       ）简化环境配置。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_63">
     </a>
     <strong>
      总结
     </strong>
    </h4>
    <p>
     Apache Doris 通过
     <strong>
      Java 和 C++ 的双语言架构
     </strong>
     实现了 FE 与 BE 的解耦，结合
     <strong>
      GCC/Clang 编译器链
     </strong>
     和
     <strong>
      BDBJE、Thrift、SpringBoot 等技术框架
     </strong>
     ，构建了高性能、高可用的实时分析系统。其技术栈深度整合了分布式存储、计算优化和生态兼容能力，适用于大规模数据分析和实时数仓场景。
    </p>
