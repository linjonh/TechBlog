    <h2>
     <a id="_13">
     </a>
     命令行操作的引入
    </h2>
    <blockquote>
     <p>
      在
      <code>
       Windows系统
      </code>
      中，用户通常依赖图形界面（
      <code>
       GUI
      </code>
      ）进行文件管理和操作，例如通过鼠标点击资源管理器完成复制、删除等任务；
     </p>
    </blockquote>
    <p>
     而
     <code>
      Linux
     </code>
     系统虽然也提供GUI工具，但其核心优势在于命令行（
     <code>
      CLI
     </code>
     ）。通过终端输入简洁的命令，用户可直接操控系统内核，实现更高效、灵活的操作，例如批量处理文件、远程管理服务器等场景。
    </p>
    <blockquote>
     <p>
      命令行不仅节省资源，还能通过脚本自动化复杂任务，是开发者及运维人员的核心工具，体现了
      <code>
       Linux
      </code>
      “
      <code>
       一切皆文件
      </code>
      ”的设计哲学。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       因此学习Linux下命令行的使用很有必要
      </strong>
      。
     </li>
     <li>
      <strong>
       Linux操作系统极其稳定，最常用于做服务器操作系统，因此最好永不关机
      </strong>
      。
     </li>
    </ul>
    <h2>
     <a id="Linux_24">
     </a>
     Linux文件系统
    </h2>
    <blockquote>
     <p>
      首先声明一个概念，
      <code>
       Linux下一切皆文件
      </code>
      。能被看到的都是文件，既包括最常见的文本文件，甚至鼠标，键盘，显示器等设备都是文件。随着后面学习的深入，会对这一概念有更深入的理解。
     </p>
    </blockquote>
    <h3>
     <a id="_28">
     </a>
     树形结构的根文件系统
    </h3>
    <blockquote>
     <p>
      <code>
       Linux
      </code>
      的目录结构整体上是一个多叉树，他的叶子结点，一定是空目录或者文件，路上结点一定是一个非空目录。
      <br/>
      因此，我们所有的对文件或者目录的增删查改，本质都是对这颗多叉树的增删查改。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b382926837d94340b24fe5f1562243f9.png"/>
    </p>
    <blockquote>
     <p>
      与大多数操作系统一致，Linux系统的文件系统结构是树形结构的。且各个目录都有各自的功能。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       <code>
        /
       </code>
       :是
       <code>
        Linux
       </code>
       的根目录。
      </strong>
     </li>
     <li>
      <strong>
       <code>
        bin
       </code>
       ：存放二进制可执行文件，以命令居多
      </strong>
     </li>
     <li>
      <strong>
       <code>
        boot
       </code>
       ：存放开机启动程序
      </strong>
     </li>
     <li>
      <strong>
       <code>
        dev
       </code>
       ：存放设备文件： 包括字符设备文件、块设备文件
      </strong>
     </li>
     <li>
      <strong>
       <code>
        home
       </code>
       ：存放 普通用户 的目录
      </strong>
     </li>
     <li>
      <strong>
       <code>
        etc
       </code>
       ：用户信息和系统配置文件 passwd、group
      </strong>
     </li>
     <li>
      <strong>
       <code>
        lib
       </code>
       ：操作系统使用的库文件：例如 libc.so.6是一个动态库
      </strong>
     </li>
     <li>
      <strong>
       <code>
        root
       </code>
       ：管理员宿主目录（家目录）
      </strong>
     </li>
     <li>
      <strong>
       <code>
        usr
       </code>
       ：用户资源管理目录 unix software resource 用户的一些文件 第三方应用的资源文件
      </strong>
     </li>
     <li>
      <strong>
       <code>
        opt
       </code>
       和
       <code>
        proc
       </code>
       : 与Linux 进程相关的文件
      </strong>
     </li>
    </ul>
    <blockquote>
     <p>
      <code>
       Linux
      </code>
      的一个目录中，既可以存放目录文件，也可以存放其他文件。
     </p>
    </blockquote>
    <h3>
     <a id="_50">
     </a>
     绝对路径和相对路径
    </h3>
    <p>
     我们在磁盘中定位一个文件，通常需要路径来描述。
    </p>
    <blockquote>
     <p>
      路径定位需要有唯一性。
     </p>
    </blockquote>
    <ul>
     <li>
      <strong>
       <code>
        绝对路径
       </code>
      </strong>
      ：从
      <code>
       /
      </code>
      开始定位到指定位置，具有唯一性的路径，称为绝对路径。
     </li>
     <li>
      <strong>
       <code>
        相对路径
       </code>
      </strong>
      :以当前所处路径为起始参考位置，来进行定位的路径，成为相对路径。
     </li>
    </ul>
    <ol>
     <li>
      绝对路径以根目录
      <code>
       /
      </code>
      为起点。
     </li>
     <li>
      相对路径以当前路径为起点。
     </li>
    </ol>
    <h4>
     <a id="_62">
     </a>
     适用场景
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        绝对路径
       </code>
      </strong>
      :绝对路径以根目录
      <code>
       /
      </code>
      为起点，一般比较长，但具有不变性。一般用在一些固定常见，配置文件中使用。
     </li>
     <li>
      <strong>
       <code>
        相对路径
       </code>
      </strong>
      ：以当前路径为起点，一般比较短小，命令行输入中常使用相对路径。
     </li>
    </ul>

     <p>
      使用绝对路径进入目录dir1

    </blockquote>

    </p>

    <h4>
     <a id="Linux_74">
     </a>
     Linux目录下的隐藏文件
    </h4>
    <blockquote>
     <p>
      <code>
       Linux
      </code>
      中的每一个目录下都有两个隐藏文件
      <code>
       .
      </code>
      和
      <code>
       ..
      </code>
      。可以通过
      <code>
       ls -a
      </code>
      来查看
      <br/>
      <code>
       -l
      </code>
      是让文件以列表的形式来进行呈现。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ae26606dfbc947b48807f08c4197a0f4.png">
      <br/>
      <code>
       Linux
      </code>
      中的每一个目录下都有两个隐藏文件
      <code>
       .
      </code>
      和
      <code>
       ..
      </code>
      。这两个文件有什么用呢？
     </img>
    </p>
    <blockquote>
     <p>
      在
      <code>
       Linux
      </code>
      中，以
      <code>
       .
      </code>
      开头的文件都是隐藏文件，只有在
      <code>
       ls -a
      </code>
      是才能被看到。
     </p>
    </blockquote>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         .
        </code>
       </strong>
       ：
       <strong>
        表示当前路径。
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         ..
        </code>
       </strong>
       ：
       <strong>
        表示当前路径的上级路径。
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         .
        </code>
        存在的意义
       </strong>
       ：帮助用户定位当前目录下的一个文件。如: ./a,out
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         ..
        </code>
        存在的意义
       </strong>
       ：帮助用户定位到当前路径的上级路径以及上级路径中的文件。
      </p>
     </li>
    </ul>

    <h4>
     <a id="1_ls_94">
     </a>
     1. ls
    </h4>
    <p>
     <strong>
      功能
     </strong>
     ：列出目录内容
     <br/>
     <strong>
      语法
     </strong>
     ：
     <code>
      ls [选项] [目录]
     </code>
     <br/>
     <strong>
      常用参数
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -l
      </code>
      ：长格式显示（包含权限、所有者、大小等信息）
     </li>
     <li>
      <code>
       -a
      </code>
      ：列出目录下的所有文件，包括以 . 开头的隐含文件。
     </li>
     <li>
      <code>
       -h
      </code>
      ：以人类可读格式显示文件大小，更人性化
     </li>
     <li>
      <code>
       -t
      </code>
      ：按修改时间排序
     </li>
     <li>
      <code>
       -R
      </code>
      ：递归列出子目录内容
     </li>
     <li>
      <code>
       -d
      </code>
      : 将目录象文件一样显示，而不是显示其下的文件。 如：ls –d 指定目录
     </li>
     <li>
      <code>
       -i
      </code>
      : 输出文件的 i 节点的索引信息。 如 ls –ai 指定文件
     </li>
     <li>
      <code>
       -k
      </code>
      : 以 k 字节的形式表示文件的大小。ls –alk 指定文件
     </li>
     <li>
      <code>
       -n
      </code>
      : 用数字的 UID,GID 代替名称。
     </li>
     <li>
      <code>
       -F
      </code>
      : 在每个文件名后附上一个字符以说明该文件的类型，“*”表示可执行的普通文件；“/”表示目录；“@”表示符号链接；“|”表示FIFOs；“=”表示套接字(sockets)。（目录类型识别）
     </li>
     <li>
      <code>
       -r
      </code>
      : 对目录反向排序。
     </li>
     <li>
      <code>
       -t
      </code>
      :以时间排序。
     </li>
     <li>
      <code>
       -s
      </code>
      : 在l文件名后输出该文件的大小。（大小排序，如何找到目录下最大的文件）
     </li>
     <li>
      <code>
       -1
      </code>
      : 一行只输出一个文件。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2320bb0ba76b42d39d0e05815e9f7e71.png"/>
     </li>
    </ul>
    <h4>
     <a id="2_cdpwd_116">
     </a>
     2. cd和pwd
    </h4>
    <h5>
     <a id="cd_118">
     </a>
     cd
    </h5>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      cd 路径
     </code>
     。
     <br/>
     路径可以是相对路径，也可以是绝对路径。
     <br/>
     <strong>
      功能
     </strong>
     ：改变工作目录。将当前工作目录改变到指定的目录下。
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 直接输入cd,默认切换到当前用户的home目录</span>
<span class="token comment"># 对于root用户 则切换到 /root 目录下</span>
<span class="token builtin class-name">cd</span> 
<span class="token comment">#返回上级目录</span>
<span class="token builtin class-name">cd</span> <span class="token punctuation">..</span> 
<span class="token comment"># 使用绝对路径</span>
<span class="token builtin class-name">cd</span> /home/changan_memroy/linux/ 
<span class="token comment"># 使用相对路径</span>
<span class="token builtin class-name">cd</span> <span class="token punctuation">..</span>/day02/
<span class="token comment"># 进入用户home目录  </span>
<span class="token builtin class-name">cd</span> ~ 
<span class="token builtin class-name">cd</span> /home  <span class="token comment"># 与上一行的写法等价</span>
<span class="token comment"># 返回最近访问目录</span>
<span class="token builtin class-name">cd</span> - 
</code></pre>
    <ul>
     <li>
      <code>
       ~
      </code>
      该符号表示当前用户的home目录。
     </li>
     <li>
      <code>
       -
      </code>
      该符号表示用户最近进入的一个目录。
     </li>
     <li>
      <code>
       .
      </code>
      表示当前目录。
     </li>
    </ul>
    <h5>
     <a id="pwd_145">
     </a>
     pwd
    </h5>
    <p>
     <strong>
      语法
     </strong>
     :
     <code>
      pwd
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：显示用户当前所在的目录
     <br/>
     常用选项：无
    </p>
    <ul>
     <li>
      <code>
       cd
      </code>
      配合
      <code>
       -
      </code>
      和
      <code>
       ~
      </code>
      方便地切换目录。
     </li>
     <li>
      切换完目录后使用
      <code>
       pwd
      </code>
      验证当前所在目录。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9120471176214a378cd58098e762f00e.png"/>

    <h4>
     <a id="3_touch_155">
     </a>
     3. touch
    </h4>
    <p>
     <strong>
      语法
     </strong>
     :touch [选项]… 文件…
     <br/>
     <strong>
      功能
     </strong>
     ：touch命令参数可更改文档或目录的日期时间，包括存取时间和更改时间，或者新建一个不存在的文件。
     <br/>
     <strong>
      常用选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -a
      </code>
      或
      <code>
       --time=atime
      </code>
      或
      <code>
       --time=access
      </code>
      或
      <code>
       --time=use
      </code>
      只更改存取时间。
     </li>
    </ul>
    <ul>
     <li>
      <code>
       -c
      </code>
      或
      <code>
       --no-create
      </code>
      不建立任何文档。
     </li>
     <li>
      <code>
       -d
      </code>
      : 使用指定的日期时间，而非现在的时间。
     </li>
     <li>
      <code>
       -f
      </code>
      : 此参数将忽略不予处理，仅负责解决BSD版本touch指令的兼容性问题。
     </li>
     <li>
      <code>
       -m
      </code>
      : 或
      <code>
       --time=mtime
      </code>
      或
      <code>
       --time=modify
      </code>
      只更改变动时间。
     </li>
     <li>
      <code>
       -r
      </code>
      : 把指定文档或目录的日期时间，统统设成和参考文档或目录的日期时间相同。
     </li>
     <li>
      <code>
       -t
      </code>
      : 使用指定的日期时间，而非现在的时间
     </li>
    </ul>
    <blockquote>
     <p>
      加上不同的参数，就有不同的用法，但最常用的用法是使用
      <code>
       touch
      </code>
      来创建文件。例如创建
      <code>
       .c
      </code>
      或
      <code>
       .cpp
      </code>
      源程序文件或者普通文件，创建别的文件也是如此。
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b26c5854c4344dfc9bc691aba49fdc1f.png"/>
     </p>
    </blockquote>
    <h4>
     <a id="4_mkdir_174">
     </a>
     4. mkdir
    </h4>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      mkdir [选项] 路径名称
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：在当前目录下创建一个名为
     <code>
      dirname
     </code>
     的目录
     <br/>
     <strong>
      常用选项
     </strong>
     ：
     <br/>
     <code>
      -p
     </code>
     或
     <code>
      --parents
     </code>
     后面跟一个路径名称。此时若路径中的某些目录尚不存在,加上此选项后,系统将自动建立好那些尚不存在的目录,即一次可以建立多个目录。(
     <strong>
      递归创建子目录
     </strong>
     )
    </p>
    <blockquote>
     <p>
      演示如下
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/54b283a4bca34ac1b3e4cbcd70d85e5d.png"/>
    </p>
    <h4>
     <a id="5_cp_185">
     </a>
     5. cp
    </h4>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      cp [选项] 源文件或目录 目标文件或目录
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ： 复制文件或目录
     <br/>
     <strong>
      说明
     </strong>
     :
     <mark>
      cp指令用于复制文件或目录
     </mark>
     。如同时指定两个以上的文件或目录，且最后的目的地是一个已经存在的目录，则它会把前面指定的所有文件或目录复制到此目录中。若同时指定多个文件或目录，而最后的目的地并非一个已存在的目录，则会出现错误信息
     <br/>
     <strong>
      常用选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -f
      </code>
      或
      <code>
       --force
      </code>
      ： 强行复制文件或目录， 不论目的文件或目录是否已经存在
     </li>
     <li>
      <code>
       -i
      </code>
      或
      <code>
       --interactive
      </code>
      ：拷贝时，若目标目录下有同名文件，覆盖文件之前会先询问用户是否覆盖
     </li>
     <li>
      <code>
       -r
      </code>
      ： 递归处理，将指定目录下的文件与子目录一并处理。若源文件或目录的形态，不属于目录或符号链接，则一律视为普通文件处理
     </li>
     <li>
      <code>
       -R
      </code>
      或
      <code>
       --recursive
      </code>
      ：递归处理，将指定目录下的文件及子目录一并处理
     </li>
    </ul>
    <blockquote>
     <p>
      最常用的选项是
      <code>
       -f
      </code>
      和
      <code>
       -r
      </code>
     </p>
    </blockquote>
    <blockquote>
     <p>
      目标目录
      <code>
       d1
      </code>
      中没有同名文件，因此不需要加
      <code>
       -f
      </code>
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cfd9080d792c4709af187f5c91565f6b.png"/>
    </p>
    <h4>
     <a id="6_mv_203">
     </a>
     6. mv
    </h4>
    <blockquote>
     <p>
      mv命令是move的缩写，可以用来
      <code>
       移动文件
      </code>
      或者
      <code>
       将文件重命名
      </code>
      （move (rename) files），是Linux系统下常用的命令，经常用来备份文件或者目录。
     </p>
    </blockquote>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      mv [选项] 源文件或目录 目标文件或目录
     </code>
     <br/>
     <strong>
      功能
     </strong>
     : 视
     <code>
      mv
     </code>
     命令中第二个参数类型的不同（是目标文件还是目标目录），
     <code>
      mv
     </code>
     命令将文件重命名或将其移至一个新的目录中。
    </p>
    <ol>
     <li>
      当
      <mark>
       第二个参数类型是文件时
      </mark>
      ，mv命令完成文件重命名，此时，
      <mark>
       源文件只能有一个（也可以是源目录名）
      </mark>
      ，它将所给的源文件或目录重命名为给定的目标文件名。
     </li>
     <li>
      当
      <mark>
       第二个参数是已存在的目录名称时
      </mark>
      ，源文件或目录参数可以有多个，mv命令将各参数指定的源文件均移至目标目录中。
     </li>
    </ol>
    <p>
     <strong>
      常用选项
     </strong>
     ：
     <br/>
     <code>
      -f
     </code>
     ：force 强制的意思，如果目标文件已经存在，不会询问而直接覆盖
     <br/>
     <code>
      -i
     </code>
     ：若目标文件 (destination) 已经存在时，就会询问是否覆盖
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6887a172656142e89716404b923cf3c9.png"/>
    </p>
    <h5>
     <a id="_218">
     </a>
     移动目录时覆盖写入的两种特殊情况
    </h5>
    <ul>
     <li>
      目标路径下有同名目录，同名目录不为空时，无法覆盖。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d1adda8fef77484b80acfabf301ebbb6.png"/>
     </li>
     <li>
      目标路径下有同名目录，为空目录，直接覆盖。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/93ea9f54588340f8b3b5e162f4eb63c3.png"/>
     </li>
    </ul>
    <h4>
     <a id="7_alias_225">
     </a>
     7. alias
    </h4>
    <blockquote>
     <p>
      之前对于
      <code>
       ls
      </code>
      命令，我们是否会有疑问？为什么
      <code>
       ll
      </code>
      等价于
      <code>
       ls -l
      </code>
      。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dd204a4b4c0a403aafd36df6f7934797.png"/>
      <br/>
      这里本质上是命令
      <code>
       alias
      </code>
      在起作用。
     </p>
    </blockquote>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      alias [选项][名称[='命令']]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：创建或显示命令别名，用于简化复杂的命令。
     <br/>
     <strong>
      常用选项
     </strong>
     ：
     <br/>
     •
     <code>
      -p
     </code>
     ：以可被 Shell 重新使用的格式输出别名（便于保存到配置文件中）。
    </p>
    <p>
     <strong>
      功能详解
     </strong>
     ：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        定义别名
       </strong>
       ：通过
       <code>
        名称='命令'
       </code>
       的格式将长命令或常用命令定义为简短的别名。
       <br/>
       • 示例：
       <code>
        alias ll='ls -l'
       </code>
       （输入
       <code>
        ll
       </code>
       即可执行
       <code>
        ls -l
       </code>
       ）。
       <br/>
       • 若命令包含空格或特殊字符，需用引号包裹。
      </p>
     </li>
     <li>
      <p>
       <strong>
        查看别名
       </strong>
       ：
       <br/>
       • 直接输入
       <code>
        alias
       </code>
       （不加参数）可列出所有已定义的别名。
       <br/>
       • 输入
       <code>
        alias 名称
       </code>
       可查看特定别名的定义（如
       <code>
        alias ll
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        永久生效
       </strong>
       ：
       <br/>
       • 在
       <code>
        ~/.bashrc
       </code>
       或
       <code>
        ~/.zshrc
       </code>
       中写入别名定义，重启终端或执行
       <code>
        source ~/.bashrc
       </code>
       后生效。
      </p>
     </li>
    </ol>
    <p>
     <strong>
      注意事项
     </strong>
     ：
     <br/>
     • 使用
     <code>
      unalias 名称
     </code>
     可删除临时别名。
     <br/>
     • 若别名与系统命令同名，优先执行别名（用
     <code>
      \命令
     </code>
     或
     <code>
      command 命令
     </code>
     可绕过别名）。
    </p>
    <h3>
     <a id="_253">
     </a>
     文件查看相关
    </h3>
    <h4>
     <a id="1_cat_255">
     </a>
     1. cat
    </h4>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      cat [选项][文件]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ： 查看目标文件的内容,
     <mark>
      常用于行数短小的文件的查看
     </mark>
     。
     <br/>
     <strong>
      常用选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -b
      </code>
      ：对非空行输出编号
     </li>
     <li>
      <code>
       -n
      </code>
      ： 对输出的所有行编号
     </li>
     <li>
      <code>
       -s
      </code>
      ：不输出多行空行
     </li>
    </ul>
    <h4>
     <a id="2_more_265">
     </a>
     2. more
    </h4>
    <blockquote>
     <p>
      <code>
       cat
      </code>
      会查看整个文件的内容，直接定位到文件的末尾，当遇到大文件时，直接定位到文件的末尾，会导致我们不便查看到文件前面的内容。而
      <code>
       more
      </code>
      指令可以让我们从文件第的一行开始，通过
      <code>
       Enter
      </code>
      来进行向后滚动，逐行依次查看内容。
     </p>
     <p>
      但
      <code>
       more
      </code>
      的
      <mark>
       缺点是只能从文件开头向后进行翻页，无法向前进行查看
      </mark>
      。
     </p>
    </blockquote>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      more [选项][文件]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：
     <code>
      more命令，功能类似 cat
     </code>
     <br/>
     <strong>
      常用选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -n
      </code>
      ：对输出的所有行编号
     </li>
     <li>
      <code>
       q
      </code>
      ： 退出more
     </li>
    </ul>
    <h4>
     <a id="3_less_278">
     </a>
     3. less
    </h4>
    <p>
     <strong>
      语法
     </strong>
     ：
     <code>
      less [参数] 文件
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <code>
        less
       </code>
       工具也是对文件或其它输出进行分页显示的工具，应该说是
       <code>
        linux
       </code>
       正统查看文件内容的工具，功能极其强大。
      </p>
     </li>
     <li>
      <p>
       <code>
        less
       </code>
       的用法比起
       <code>
        more
       </code>
       更加的有弹性。在
       <code>
        more
       </code>
       的时候，我们并没有办法向前面翻， 只能往后面看
      </p>
     </li>
     <li>
      <p>
       但若使用了
       <code>
        less
       </code>
       时，就可以使用
       <code>
        [pageup][pagedown]
       </code>
       等按键的功能来往前往后翻看文件，更容易用来查看一个文件的内容！
      </p>
     </li>
     <li>
      <p>
       除此之外，在
       <code>
        less
       </code>
       内可以拥有更多的搜索功能，不止可以向下搜，也可以向上搜索。而且
       <code>
        less
       </code>
       在查看之前不会加载整个文件
      </p>
     </li>
    </ul>
    <p>
     <strong>
      选项
     </strong>
     ：以下选项适用于通过
     <code>
      less
     </code>
     进入文件后在文件内进行搜索，而不是在使用
     <code>
      less
     </code>
     打开文件时进行搜索。
    </p>
    <ul>
     <li>
      <code>
       -i
      </code>
      忽略搜索时的大小写
     </li>
     <li>
      <code>
       -N
      </code>
      显示每行的行号
     </li>
     <li>
      <code>
       /字符串
      </code>
      ：向下搜索“字符串”的功能
     </li>
     <li>
      <code>
       ?字符串
      </code>
      ：向上搜索“字符串”的功能
     </li>
     <li>
      <code>
       n
      </code>
      ：重复前一个搜索（与 / 或 ? 有关）
     </li>
     <li>
      <code>
       N
      </code>
      ：反向重复前一个搜索（与 / 或 ? 有关）
     </li>
     <li>
      <code>
       q
      </code>
      :quit
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c8f64c79598c4e61846ed934f02278b7.png"/>
     <br/>
     所有选项是在
     <code>
      :
     </code>
     处进行输入的。例如向下查找字符串hello，输入
     <code>
      /hello
     </code>
     。
    </p>
    <p>
     其他命令也都是在使用
     <code>
      less
     </code>
     打开文件后再进行输入的。
    </p>
    <blockquote>
     <p>
      因此查看文件使用
      <code>
       less
      </code>
      和
      <code>
       cat
      </code>
      ,以及通过==管道(下篇介绍)==配合
      <code>
       head
      </code>
      和
      <code>
       tail
      </code>
      比较多
     </p>
    </blockquote>
    <h4>
     <a id="4_head_310">
     </a>
     4. head
    </h4>
    <p>
     <code>
      head
     </code>
     与
     <code>
      tail
     </code>
     就像它的名字一样的浅显易懂，它是用来显示开头或结尾某个数量的文字区块，
     <code>
      head
     </code>
     用来显示档案的开头至标准输出中，而
     <code>
      tail
     </code>
     就是看档案的结尾。
     <br/>
     <strong>
      语法
     </strong>
     ：
     <code>
      head [参数]... [文件]...
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ：
     <br/>
     head 用来显示档案的开头至标准输出中，默认head命令打印其相应文件的开头10行。
     <br/>
     <strong>
      选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -n [行数]
      </code>
      ：显示文件的前
      <code>
       n
      </code>
      行
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5513a7e835fa4d59a548b8b0f2a49f9d.png"/>
     </li>
    </ul>
    <h4>
     <a id="5_tail_321">
     </a>
     5. tail
    </h4>
    <p>
     <code>
      tail
     </code>
     命令从指定点开始将文件写到标准输出.使用
     <code>
      tail
     </code>
     命令的
     <code>
      -f
     </code>
     选项可以方便的查阅正在改变的日志文件,
     <code>
      tail -f filename
     </code>
     会把
     <code>
      filename
     </code>
     里最尾部的内容显示在屏幕上,
     <mark>
      并且不但刷新,使你看到最新的文件内容
     </mark>
     .
     <br/>
     <strong>
      语法
     </strong>
     ：
     <code>
      tail [必要参数] [选择参数] [文件]
     </code>
     <br/>
     <strong>
      功能
     </strong>
     ： 用于显示指定文件末尾内容，不指定文件时，作为输入信息进行处理。常用查看日志文件。
     <code>
      tail
     </code>
     默认显示文件的后10行。
     <br/>
     <strong>
      选项
     </strong>
     ：
    </p>
    <ul>
     <li>
      <code>
       -f
      </code>
      循环读取
     </li>
     <li>
      <code>
       -n 行数
      </code>
      ： 显示行数
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6e2c2a1f50b84e0cbc348324a4f91fe5.png"/>
     </li>
    </ul>
