     【Recon】CTF Web类题目主要类型

    <p>
    </p>
    <br/>
    CTF（Capture The Flag）竞赛中的Web类题目主要考察参赛者对Web应用漏洞的识别与利用能力，涵盖多种漏洞类型和攻击手法。以下是常见的Web题目类型及对应的漏洞考点：
    <p>
    </p>
    <hr/>
    <h2>
     <a id="1__5">
     </a>
     1. 信息搜集类
    </h2>
    <ul>
     <li>
      敏感目录泄露：通过探测
      <code>
       .git
      </code>
      、
      <code>
       .svn
      </code>
      等目录或备份文件（如
      <code>
       vim
      </code>
      的
      <code>
       .swp
      </code>
      文件）获取源码或敏感信息。
     </li>
     <li>
      常规文件分析：查看
      <code>
       robots.txt
      </code>
      、
      <code>
       readme.md
      </code>
      等文件，获取CMS版本或隐藏路径。
     </li>
     <li>
      Banner识别：通过服务器响应头识别Web框架（如ThinkPHP），利用已知历史漏洞解题。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="2__12">
     </a>
     2. 注入类漏洞
    </h2>
    <ul>
     <li>
      SQL注入：包括数字型、字符型、布尔盲注、报错注入等，常用工具如
      <code>
       SQLmap
      </code>
      ，或手动构造Payload（如
      <code>
       ' OR '1'='1
      </code>
      ）绕过登录验证。
     </li>
     <li>
      命令注入：利用系统命令执行函数（如
      <code>
       system
      </code>
      、
      <code>
       exec
      </code>
      ），通过分号、管道符等拼接命令（如
      <code>
       ls; cat flag
      </code>
      ），需绕过空格过滤（如
      <code>
       ${IFS}
      </code>
      或
      <code>
       %09
      </code>
      ）。
     </li>
     <li>
      XXE（XML外部实体注入）：通过恶意XML读取服务器文件（如
      <code>
       /etc/passwd
      </code>
      ）。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="3__19">
     </a>
     3. 文件处理漏洞
    </h2>
    <ul>
     <li>
      文件上传漏洞：绕过黑名单（如
      <code>
       .php
      </code>
      改为
      <code>
       .php5
      </code>
      ）、利用MIME类型欺骗或竞争条件上传WebShell。
     </li>
     <li>
      文件包含漏洞：本地文件包含（LFI）或远程文件包含（RFI），利用伪协议（如
      <code>
       php://filter
      </code>
      ）读取源码或执行代码。
     </li>
     <li>
      任意文件读取：直接访问敏感文件（如
      <code>
       /etc/passwd
      </code>
      或
      <code>
       .env
      </code>
      ）。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="4__26">
     </a>
     4. 身份验证与会话漏洞
    </h2>
    <ul>
     <li>
      弱密码与未授权访问：尝试常见弱密码（如
      <code>
       admin:123456
      </code>
      ）或直接访问未鉴权的管理页面。
     </li>
     <li>
      Cookie篡改与会话劫持：修改Cookie中的用户角色（如
      <code>
       admin=1
      </code>
      ）或通过XSS窃取会话Token。
     </li>
     <li>
      JWT漏洞：利用弱密钥或算法（如
      <code>
       None
      </code>
      ）伪造Token。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="5__33">
     </a>
     5. 服务端漏洞
    </h2>
    <ul>
     <li>
      SSRF（服务端请求伪造）：构造恶意URL访问内网服务（如
      <code>
       http://127.0.0.1:8080
      </code>
      ）。
     </li>
     <li>
      反序列化漏洞：PHP或Java反序列化链利用，需绕过
      <code>
       __wakeup
      </code>
      方法（如修改对象属性数量）。
     </li>
     <li>
      代码执行漏洞：通过危险函数（如
      <code>
       eval
      </code>
      ）执行任意代码，或利用框架漏洞（如ThinkPHP的RCE）。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="6__40">
     </a>
     6. 客户端漏洞
    </h2>
    <ul>
     <li>
      XSS（跨站脚本）：反射型（Payload通过URL传递）或存储型（恶意脚本存储于数据库），常用Payload如
      <code>
       &lt;script&gt;alert(1)&lt;/script&gt;
      </code>
      。
     </li>
     <li>
      CSRF（跨站请求伪造）：构造恶意页面诱使用户触发敏感操作（如修改密码）。
     </li>
     <li>
      CORS配置错误：利用跨域请求窃取用户数据。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="7_PHP_47">
     </a>
     7. 代码审计与PHP特性
    </h2>
    <ul>
     <li>
      弱类型比较：利用
      <code>
       ==
      </code>
      的松散比较（如
      <code>
       0e123456 == 0
      </code>
      ）绕过MD5校验。
     </li>
     <li>
      正则绕过：通过
      <code>
       %00
      </code>
      截断、数组参数（如
      <code>
       a[]=1
      </code>
      ）或特殊字符绕过正则过滤。
     </li>
     <li>
      无字母数字WebShell：使用异或、取反等操作构造Payload（如
      <code>
       $_=~%9A%89%9E%93;
      </code>
      ）。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="8__54">
     </a>
     8. 业务逻辑漏洞
    </h2>
    <ul>
     <li>
      越权操作：通过修改参数（如订单ID）访问他人数据。
     </li>
     <li>
      支付逻辑漏洞：利用负数价格、重复支付等绕过校验。
     </li>
     <li>
      竞态条件：并发请求导致资源分配异常（如积分重复兑换）。
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="_61">
     </a>
     总结
    </h2>
    <p>
     CTF Web题目的核心是识别漏洞并构造利用链，需熟悉常见漏洞原理及绕过技巧，并结合工具（如Burp Suite、SQLmap）提升效率。部分题目会结合多种漏洞类型，需灵活运用综合知识解题。
    </p>
