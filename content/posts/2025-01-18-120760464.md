     <strong>
      <strong>
       1.音视频开发
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       1.1音视频开发是做什么的
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     音视频开发就是将我们摄像机、麦克风采集到的画面（视频像素数据，YUV格式）和声音（音频采用数据，PCM格式），通过编码算法进行压缩后（视频压缩为H.264\MPEG4,音频压AAC/MP3），再进行封装为固定格式(FLV,MP4,AVI)，通过网络传输到用户端(rtmp,RTSP)，在用户端对封装的数据(FLV,MP4,AVI)进行解封为音频数据AAC/MP3和视频数H.264\MPEG4，再通过解码算法恢复为采集到的格式(PCM,YUV)，再通过音频和视频的同步，交给视频驱动（显卡-&gt;显示器）和音频驱动（扬声器）后就可以听到了。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     说白了，音视频开发，就是要掌握图像、音频、视频的基础知识，并且学会如何对它们进行采集、处理、传输、渲染等一系列的开发和应用。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       音视频开发主要有以下步骤：
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     采集：数据从哪里来（摄像头、麦克风）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     处理：数据怎么加工（视频裁剪、缩放、旋转、叠加，音频充采样、去噪、回声消除、混音），编解码
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     传输：数据怎么共享，直播实时音视频
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     渲染：数据怎么展现，播放器
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       1.2音视频开发行业的现状与前景
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       开发薪水：
      </strong>
     </strong>
     在BOOS直聘公开统计中，音视频行业的工资10k-20k占12%，20k-30k占45%，30k-k占35%，35k以上占13%。主要分布在20k-35k之间，想要取得一个不错的薪资，如果只是简简单单开发四大组件是不可能拿到这么高薪资的。音视频开发薪资这么高主要就是因为音视频技术人才非常稀缺，很少有人会完整的音视频开发。
     <br/>
     <strong>
      <strong>
       就业机会：
      </strong>
     </strong>
     就业场景分布广。之前学习了音视频高手课的200多位学员就职于全国各大公司。从事音视频相关产品开发，方向主要是直播，短视频，教育直播，AR，VR，云游戏，视频会议等一些场景，方向上没多少变化，只是应用场景上多了一些包装，毕竟随5G带宽硬件加速，以及人们对视频交互的需求日盛，音视频的使用场景应该会越来越多；
     <br/>
     <strong>
      <strong>
       开发前景：
      </strong>
     </strong>
     随着5G落地，音视频在互联网中的比重越来越高，5G宽带的提速，必然加速整个音视频领域的应用，未来音视频人才缺口达30万，音视频高端领域严重短缺
     <br/>
     像前几年，我们70%-80%的应用都是在网络请求、UI界面、文字、图片上；在以后，95%的应用场景主要是来自于计算机视觉领域，计算机视觉领域又是跟音视频相关性非常大的领域。所以，未来音视频开发领域绝对是一个风口，是一片蓝海
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       音视频开发行业的现状
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       核心竞争力：
      </strong>
     </strong>
     音视频是程序界的皇冠，掌握音视频意味着拿到通往未来的船票，不用担心会被其他人替代。因为音视频是有一定门槛的，所以也是与其他人拉开差距的分水岭。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       高端人才相当缺乏：B
      </strong>
     </strong>
     OOS直聘中，北上广深很多年薪50W-70W的音视频开发岗位，常年招不到人，月薪2-3万大多是刚从事音视频的入门级开发者
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       技术迭代慢：
      </strong>
     </strong>
     H264编码从1995年成为标准至今一直都在使用，属于比较偏底层的技术，而底层技术几十年都不会有太大的改变。
     <br/>
     如果大家再仅仅是停留在简简单单的四大组件、UI、网络这一块，那么可替代性是非常高的，不利于打造自身技术护城河
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       1.3学习音视频开发所需要的基础知识
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       计算机基础知识
      </strong>
     </strong>
     ：c/c++语言；计算机系统的底层工作原理；操作系统原理；程序的编译、链接和加载机制；C/C++语言特性背后蕴含的思想，底层工作原理，适用场景，存在什么样的问题；软件设计原则和设计模式；数据结构和算法；多线程并发编程原理；网络编程；跨平台；操作系统API；软件调试
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       音视频的一些基础原理：
      </strong>
     </strong>
     视频、音频的基础知识（码率、帧率、颜色空间、声道、刷新率、画质）；音视频编解码技术（H.264\MPEG\MP3\AAC）；封装格式（avi\flv）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       音视频各个步骤需要掌握的知识
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       采集：
      </strong>
     </strong>
     音、视频怎么采集（各系统（windows等）采集接口、采集参数的含义以及怎么配置，摄像头、麦克风输出视频像素数据格式）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       处理：
      </strong>
     </strong>
     做哪些处理（美颜、水印、贴纸、滤镜、连麦，混音、降噪、声音特效、自定义处理等），这些手段的原理、以及怎么实现、实现方法、实现的第三方库（视频：OpenGL，OpenCV，libyuv，ffmpeg\音频speexdsp，ffmpeg）、第三方库的原理和使用方法。编解码的原理，编解码采用的第三方库，第三方库的原理和使用方法（视频：x264，OpenH264，ffmpeg，音频libfaac，opus，speex，ffmpeg ）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       传输
      </strong>
     </strong>
     <strong>
      <strong>
       ：
      </strong>
     </strong>
     传输分类、传输过程（推流、拉流、）、研究音视频传输的协议（推流RTMP，RSTP；拉流：RTMP，HLS，HDL，RTSP； UDP 的协议RTP/RTCP，QUIC ）、协议的工作原理。
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       渲染
      </strong>
     </strong>
     <strong>
      <strong>
       ：
      </strong>
     </strong>
     数据怎么展现的（图像视频绘制到视图上、音频输出到扬声器）；系统提供了哪些 API 可以绘制一张图片或者一帧 YUV 图像数据的 ？（Windows：DirectDraw, Direct3D, GDI，OpenGL 等Linux： GDI， OpenGL 等Android：ImageView，SurfaceView，TextureView，OpenGL 等iOS： CoreGraphics，OpenGL 等）系统提供了哪些 API 可以播放一个 mp3 或者 pcm 数（Windows：DirectSound 等Linux：ALSA &amp; OSS 等Android：AudioTrack 等iOS： AudioQueue 等）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       1.4常见的音视频开源库
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Ffmpeg：可以直接使用ffmpeg的命令行实现转码、切片等常见功能
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     可以基于FFmpeg API封装开发自己的音视频模块
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     live555：比较完善的RTSP库
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     x264：比较常用的H.264编码库
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Fdkaac：比较常用的AAC编解码库
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Librtmp：支持rtmp协议，产品化时需要自己进一步完善
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Pjsip：支持sip协议
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Webrtc：google开源的webrtc库，有比较好的音/视频引擎，对网络状态的实时评估可以借鉴，回声消除模块也是比较有名的
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     SDL：比较有名的音视频渲染库
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     SRS：国内比较知名的RTMP流媒体服务器，支持HLS、HTTP+FLV，4.0版本开始支持WebRTC
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     OWT：Intel开源的WebRTC套件，支持了WebRTC客户端SDK和分布式的WebRTC MCU服务器
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     OpenCV：著名的视频算法库
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     另外，视频的编码和解码可以基于Intel Media SDK和NVIDIA的NVENC来实现（利用硬件编解码）。
    </p>
