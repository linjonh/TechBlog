    <h2>
     <a id="__uv_pip_install__0">
     </a>
     注意 激活后的虚拟环境使用uv pip install 安装包
    </h2>
    <p>
     从你的输出信息来看，尽管你已经成功安装了
     <code>
      pip
     </code>
     到
     <code>
      /home/enine/.local/bin
     </code>
     目录下，但是这个目录并没有包含在你的系统环境变量
     <code>
      PATH
     </code>
     中。因此，当你尝试运行
     <code>
      pip
     </code>
     或者特定版本的
     <code>
      pip3.12
     </code>
     时，系统找不到这些命令。
    </p>
    <h4>
     <a id="_4">
     </a>
     解决步骤
    </h4>
    <h5>
     <a id="1__localbin__PATH_6">
     </a>
     1. 添加
     <code>
      .local/bin
     </code>
     到
     <code>
      PATH
     </code>
    </h5>
    <p>
     你可以通过编辑 shell 配置文件（如
     <code>
      .bashrc
     </code>
     ,
     <code>
      .zshrc
     </code>
     等）来将
     <code>
      /home/enine/.local/bin
     </code>
     添加到
     <code>
      PATH
     </code>
     环境变量中。
    </p>
    <p>
     打开终端并输入以下命令来编辑
     <code>
      .bashrc
     </code>
     文件（假设你使用的是 Bash shell）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">nano</span> ~/.bashrc
</code></pre>
    <p>
     在文件末尾添加一行：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token string">"<span class="token environment constant">$HOME</span>/.local/bin:<span class="token environment constant">$PATH</span>"</span>
</code></pre>
    <p>
     保存文件后，运行以下命令使更改生效：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
    <p>
     现在你应该能够直接使用
     <code>
      pip
     </code>
     命令了。
    </p>
    <h5>
     <a id="2__pip_29">
     </a>
     2. 安装或更新
     <code>
      pip
     </code>
    </h5>
    <p>
     如果你依然无法使用
     <code>
      pip
     </code>
     命令，可以尝试重新安装或更新它：
    </p>
    <ul>
     <li>
      <p>
       如果你没有安装
       <code>
        pip
       </code>
       ，可以通过以下命令安装：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> python3-pip
</code></pre>
     </li>
     <li>
      <p>
       如果你想要确保安装的是最新版的
       <code>
        pip
       </code>
       ，可以使用以下命令进行升级：
      </p>
      <pre><code class="prism language-bash">python3 <span class="token parameter variable">-m</span> pip <span class="token function">install</span> <span class="token parameter variable">--upgrade</span> pip
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3__pip312__uv_45">
     </a>
     3. 使用
     <code>
      pip3.12
     </code>
     安装
     <code>
      uv
     </code>
    </h5>
    <p>
     既然你提到了
     <code>
      pip3.12
     </code>
     ，那么你可以尝试使用完整的路径来调用
     <code>
      pip3.12
     </code>
     来安装
     <code>
      uv
     </code>
     ：
    </p>
    <pre><code class="prism language-bash">/home/enine/.local/bin/pip3.12 <span class="token function">install</span> uv
</code></pre>
    <p>
     或者，如果
     <code>
      pip3.12
     </code>
     是正确的名称并且已经在
     <code>
      PATH
     </code>
     中，可以直接使用：
    </p>
    <pre><code class="prism language-bash">pip3.12 <span class="token function">install</span> uv
</code></pre>
    <h5>
     <a id="4__pip__58">
     </a>
     4. 检查
     <code>
      pip
     </code>
     版本
    </h5>
    <p>
     最后，确认
     <code>
      pip
     </code>
     已经正确安装并且版本符合预期：
    </p>
    <pre><code class="prism language-bash">pip <span class="token parameter variable">--version</span>
</code></pre>
    <p>
     或者对于 Python 3.12 特定的
     <code>
      pip
     </code>
     ：
    </p>
    <pre><code class="prism language-bash">pip3.12 <span class="token parameter variable">--version</span>
</code></pre>
    <p>
     这样就可以确保
     <code>
      pip
     </code>
     和
     <code>
      uv
     </code>
     已经正确安装并且可以在命令行中使用了。
    </p>
    <p>
     <code>
      uv
     </code>
     是一个高效的 Python 包管理工具，它不仅能够管理包，还能创建和管理虚拟环境。下面是如何使用
     <code>
      uv
     </code>
     创建虚拟环境的步骤：
    </p>
    <h4>
     <a id="_uv__77">
     </a>
     使用
     <code>
      uv
     </code>
     创建虚拟环境
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        安装
        <code>
         uv
        </code>
       </strong>
       <br/>
       如果你还没有安装
       <code>
        uv
       </code>
       ，请参照之前的指导进行安装。确保
       <code>
        uv
       </code>
       已经正确安装在你的系统中，并且可以通过命令行访问。
      </p>
     </li>
     <li>
      <p>
       <strong>
        初始化项目（可选）
       </strong>
       <br/>
       虽然这不是必须的步骤，但如果你正在开始一个新的 Python 项目，你可以使用
       <code>
        uv init
       </code>
       命令来初始化项目结构。这会自动生成一些必要的文件，如
       <code>
        pyproject.toml
       </code>
       和
       <code>
        .gitignore
       </code>
       文件。
      </p>
      <pre><code class="prism language-bash">uv init myproject
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        创建虚拟环境
       </strong>
       <br/>
       在项目的根目录下运行以下命令来创建虚拟环境：
      </p>
      <pre><code class="prism language-bash">uv venv
</code></pre>
      <p>
       默认情况下，
       <code>
        uv
       </code>
       会在当前目录下的
       <code>
        .venv
       </code>
       文件夹内创建一个新的虚拟环境。如果你希望将虚拟环境放在不同的位置，可以指定路径：
      </p>
      <pre><code class="prism language-bash">uv venv path/to/your/venv
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        激活虚拟环境
       </strong>
       <br/>
       创建虚拟环境后，你需要激活它以便开始使用。根据你的操作系统，激活命令会有所不同：
      </p>
      <ul>
       <li>
        对于 Linux 或 macOS:
        <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> .venv/bin/activate
</code></pre>
       </li>
       <li>
        对于 Windows (cmd.exe):
        <pre><code class="prism language-cmd">.venv\Scripts\activate.bat
</code></pre>
       </li>
       <li>
        对于 Windows (PowerShell):
        <pre><code class="prism language-powershell"><span class="token punctuation">.</span>venv\Scripts\Activate<span class="token punctuation">.</span>ps1
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        同步依赖（如果有的话）
       </strong>
       <br/>
       如果你的项目已经有了依赖项列表（例如，在
       <code>
        pyproject.toml
       </code>
       文件中定义），你可以使用
       <code>
        uv sync
       </code>
       命令来同步这些依赖到你的虚拟环境中。
      </p>
      <pre><code class="prism language-bash">uv <span class="token function">sync</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        检查虚拟环境是否激活
       </strong>
       <br/>
       为了确认虚拟环境是否被成功激活，你可以运行以下命令来检查当前使用的 Python 解释器路径：
      </p>
      <pre><code class="prism language-bash"><span class="token function">which</span> python
<span class="token comment"># 或者</span>
where python  <span class="token comment"># Windows 上</span>
</code></pre>
      <p>
       输出应该指向你的虚拟环境中的 Python 可执行文件，例如：
       <code>
        .venv/bin/python
       </code>
       或
       <code>
        .venv\Scripts\python.exe
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        退出虚拟环境
       </strong>
       <br/>
       当你完成工作后，可以通过运行以下命令来退出虚拟环境：
      </p>
      <pre><code class="prism language-bash">deactivate
</code></pre>
     </li>
    </ol>
    <p>
     通过上述步骤，你就可以使用
     <code>
      uv
     </code>
     来创建并管理虚拟环境了。
     <code>
      uv
     </code>
     的设计旨在简化开发流程，使得 Python 项目的管理和部署更加高效和可靠 。
    </p>
