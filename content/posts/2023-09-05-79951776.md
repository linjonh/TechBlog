      <span style="color:rgb(0,0,0);">

      </span>
     </a>
     <span style="font-family:'宋体';">

     </span>

     <span style="font-family:'宋体';">

     </span>
     <a href="http://baike.baidu.com/view/76277.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       UMTS
      </span>
     </a>
     <span style="font-family:'宋体';">
      （
     </span>
     Universal Mobile Telecommunications System
     <span style="font-family:'宋体';">

     </span>
     <a href="http://baike.baidu.com/subview/1872277/14015632.htm" rel="nofollow" style="color:rgb(12,137,207);">

       </span>
      </span>
     </a>

     <span style="font-family:'宋体';">

     </span>
     <span style="font-family:Calibri;">

      </span>

     </span>

     <span style="font-family:'宋体';">

     </span>

      </span>
     </a>

     </span>

     </span>
     <a href="http://baike.baidu.com/view/50652.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       MIMO
      </span>
     </a>
     <span style="font-family:'宋体';">
      （
     </span>
     Multi-Input &amp; Multi-Output
     <span style="font-family:'宋体';">
      ，多输入多输出）等关键
     </span>
     <a href="http://baike.baidu.com/subview/45517/12502662.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        技术
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      ，显著增加了
     </span>
     <a href="http://baike.baidu.com/view/419158.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">

       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      和
     </span>
     <a href="http://baike.baidu.com/view/875888.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        数据传输
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      速率（
     </span>
     20M
     <span style="font-family:'宋体';">
      带宽
     </span>
     <span style="font-family:Calibri;">
      2X2MIMO
     </span>
     <span style="font-family:'宋体';">
      在
     </span>
     <span style="font-family:Calibri;">
      64
     </span>
     <a href="http://baike.baidu.com/view/1137102.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       QAM
      </span>
     </a>
     <span style="font-family:'宋体';">
      情况下，理论下行最大传输速率为
     </span>
     201
     <a href="http://baike.baidu.com/view/496716.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       Mbps
      </span>
     </a>
     <span style="font-family:'宋体';">
      ，除去信令开销后大概为
     </span>
     150
     <a href="http://baike.baidu.com/view/496716.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       Mbps
      </span>
     </a>
     <span style="font-family:'宋体';">
      ，但根据实际组网以及
     </span>
     <a href="http://baike.baidu.com/subview/105503/15362368.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        终端
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      能力限制，一般认为下行峰值速率为
     </span>
     100Mbps
     <span style="font-family:'宋体';">
      ，上行为
     </span>
     <span style="font-family:Calibri;">
      50Mbps
     </span>
     <span style="font-family:'宋体';">
      ），并支持多种带宽分配：
     </span>
     <span style="font-family:Calibri;">
      1.4MHz
     </span>
     <span style="font-family:'宋体';">
      ，
     </span>
     <span style="font-family:Calibri;">
      3MHz
     </span>
     <span style="font-family:'宋体';">
      ，
     </span>
     <span style="font-family:Calibri;">
      5MHz
     </span>
     <span style="font-family:'宋体';">
      ，
     </span>
     <span style="font-family:Calibri;">
      10MHz
     </span>
     <span style="font-family:'宋体';">
      ，
     </span>
     <span style="font-family:Calibri;">
      15MHz
     </span>
     <span style="font-family:'宋体';">
      和
     </span>
     <span style="font-family:Calibri;">
      20MHz
     </span>
     <span style="font-family:'宋体';">
      等，且支持全球主流
     </span>
     <a href="http://baike.baidu.com/view/172161.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       2G
      </span>
     </a>
     /
     <a href="http://baike.baidu.com/view/808.htm" rel="nofollow" style="color:rgb(12,137,207);">

       3G
      </span>
     </a>
     <span style="font-family:'宋体';">
      频段和一些新增频段，因而频谱分配更加灵活，系统
     </span>
     <a href="http://baike.baidu.com/view/334600.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        容量
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      和
     </span>
     <a href="http://baike.baidu.com/view/443387.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        覆盖
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      也显著提升。
     </span>
     LTE
     <span style="font-family:'宋体';">
      系统网络架构更加扁平化简单化，减少了网络节点和系统复杂度，从而减小了系统
     </span>
     <a href="http://baike.baidu.com/view/476266.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        时延
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      ，也降低了网络部署和维护成本。
     </span>
     LTE
     <span style="font-family:'宋体';">
      系统支持与其他
     </span>
     <span style="font-family:Calibri;">
      3GPP
     </span>
     <span style="font-family:'宋体';">
      系统互操作。根据双工方式不同
     </span>
     <span style="font-family:Calibri;">
      LTE
     </span>
     <span style="font-family:'宋体';">
      系统分为
     </span>
     <a href="http://baike.baidu.com/view/3723673.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       FDD-LTE
      </span>
     </a>
     <span style="font-family:'宋体';">
      （
     </span>
     Frequency Division Duplexing
     <span style="font-family:'宋体';">
      ）和
     </span>
     <span style="font-family:Calibri;">
      TDD-LTE (Time Division Duplexing)
     </span>
     <span style="font-family:'宋体';">
      ，二者技术的主要区别在于
     </span>
     <a href="http://baike.baidu.com/view/818171.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        空口
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      的
     </span>
     <a href="http://baike.baidu.com/view/239585.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        物理层
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      上（像帧结构、时分设计、同步等）。
     </span>
     FDD
     <span style="font-family:'宋体';">
      系统空口上下行采用成对的频段接收和发送数据，而
     </span>
     <span style="font-family:Calibri;">
      TDD
     </span>
     <span style="font-family:'宋体';">
      系统上下行则使用相同的频段在不同的时隙上传输，较
     </span>
     <span style="font-family:Calibri;">
      FDD
     </span>
     <span style="font-family:'宋体';">
      双工方式，
     </span>
     <span style="font-family:Calibri;">
      TDD
     </span>
     <span style="font-family:'宋体';">
      有着较高的
     </span>
     <a href="http://baike.baidu.com/view/76432.htm" rel="nofollow" style="color:rgb(12,137,207);">
      <span style="color:rgb(0,0,0);">
       <span style="font-family:'宋体';">
        频谱利用率
       </span>
      </span>
     </a>
     <span style="font-family:'宋体';">
      。
     </span>
    </p>
    <p style="text-align:center;background-color:rgb(255,255,255);">

    </p>
    <p style="background-color:rgb(255,255,255);">
     E-UTRAN
     <span style="font-family:'宋体';">
      系统的空中接口协议栈根据用途可以分为用户平面协议栈和控制平面协议栈。用户平面协议栈主要包括物理（
     </span>
     <span style="font-family:Calibri;">
      PHY
     </span>
     <span style="font-family:'宋体';">
      ）层、媒体访问控制（
     </span>
     <span style="font-family:Calibri;">
      MAC
     </span>
     <span style="font-family:'宋体';">
      ）层、无线链路控制（
     </span>
     <span style="font-family:Calibri;">
      RLC
     </span>
     <span style="font-family:'宋体';">
      ）层以及分组数据汇聚（
     </span>
     <span style="font-family:Calibri;">
      PDCP
     </span>
     <span style="font-family:'宋体';">
      ）层四个层次，这些子层在网络侧均终止于
     </span>
     <span style="font-family:Calibri;">
      eNode
     </span>
     B
     <span style="font-family:'宋体';">
      实体。如下图所示：
     </span>
    </p>
    <p style="text-align:center;background-color:rgb(255,255,255);">
     <span style="font-family:'宋体';">

     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     LTE
     <span style="font-family:'宋体';">
      系统的数据处理过程被分解成不同的协议层。上图阐述了
     </span>
     <span style="font-family:Calibri;">
      LTE
     </span>
     <span style="font-family:'宋体';">
      系统下行传输的总体协议架构，下行数据以
     </span>
     <span style="font-family:Calibri;">
      IP
     </span>
     <span style="font-family:'宋体';">
      包的形式进行传送，在空中接口传送之前，
     </span>
     <span style="font-family:Calibri;">
      IP
     </span>
     <span style="font-family:'宋体';">
      包将通过多个协议层实体进行处理，具体描述如下：
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●
     <span style="font-family:'宋体';">
      物理层：负责处理编译码、调制解调、多天线映射以及其它电信物理层功能。最为复杂的一层，也是最考验产品的一层协议。实际设计中，涉及诸多算法也最能体现实际芯片的性能。和硬件紧密相关，需要协同工作。
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●MAC
     <span style="font-family:'宋体';">
      层：
     </span>
     <span style="font-family:Calibri;">
      ​
     </span>
     <span style="font-family:'宋体';">
      负责处理
     </span>
     <span style="font-family:Calibri;">
      HARQ
     </span>
     <span style="font-family:'宋体';">
      重传与上下行调度。应该说，
     </span>
     <span style="font-family:Calibri;">
      L2
     </span>
     <span style="font-family:'宋体';">
      的精华就在这边，重传和调度能做好，对于整个产品来说，速率就能体现出来。
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●RLC
     <span style="font-family:'宋体';">
      层：负责分段与连接、重传处理，以及对高层数据的顺序传送。
     </span>
     <span style="font-family:Calibri;">
      ​
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●PDCP
     <span style="font-family:'宋体';">
      层：
     </span>
     <span style="font-family:Calibri;">
      ​
     </span>
     <span style="font-family:'宋体';">
      负责执行头压缩以减少无线接口必须传送的比特流量。
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●RRC
     <span style="font-family:'宋体';">
      层：
     </span>
     <span style="font-family:Calibri;">
      ​
     </span>
     <span style="font-family:'宋体';">
      支持终端和
     </span>
     <span style="font-family:Calibri;">
      eNode B
     </span>
     <span style="font-family:'宋体';">
      间多种功能的最为关键的信令协议。广义上来说，还包括无线资源算法，实际应用中的无线行为，都是由它来决定的。
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     ●NAS
     <span style="font-family:'宋体';">
      层：处理
     </span>
     <span style="font-family:Calibri;">
      UE
     </span>
     <span style="font-family:'宋体';">
      和
     </span>
     <span style="font-family:Calibri;">
      MME
     </span>
     <span style="font-family:'宋体';">
      之间信息的传输，传输的内容可以是用户信息或控制信息。包括会话管理，用户管理，安全管理等。
     </span>
     <span style="font-family:Calibri;">
      NAS
     </span>
     <span style="font-family:'宋体';">
      层以下，我们称为
     </span>
     <span style="font-family:Calibri;">
      AS
     </span>
     <span style="font-family:'宋体';">
      层，而
     </span>
     <span style="font-family:Calibri;">
      NAS
     </span>
     <span style="font-family:'宋体';">
      对于
     </span>
     <span style="font-family:Calibri;">
      eNode B
     </span>
     <span style="font-family:'宋体';">
      是透明的，从上图可以看到，
     </span>
     <span style="font-family:Calibri;">
      eNode B
     </span>
     <span style="font-family:'宋体';">
      是没有这层协议的，所有
     </span>
     <span style="font-family:Calibri;">
      NAS
     </span>
     <span style="font-family:'宋体';">
      消息，对于他来说，就是过路。
     </span>
    </p>
    <p style="background-color:rgb(255,255,255);">
     <span style="font-family:'宋体';">
      控制平面协议栈如下图所示。控制平面协议栈主要包括非接入层（
     </span>
     NAS
     <span style="font-family:'宋体';">
      ）、
     </span>
     <span style="font-family:Calibri;">
      RRC
     </span>
     <span style="font-family:'宋体';">
      、
     </span>
     <span style="font-family:Calibri;">
      PDCP
     </span>
     <span style="font-family:'宋体';">
      、
     </span>
     <span style="font-family:Calibri;">
      RLC
     </span>
     <span style="font-family:'宋体';">
      、
     </span>
     <span style="font-family:Calibri;">
      MAC
     </span>
     <span style="font-family:'宋体';">
      、
     </span>
     <span style="font-family:Calibri;">
      PHY
     </span>
     <span style="font-family:'宋体';">
      层。其中，
     </span>
     <span style="font-family:Calibri;">
      PDCP
     </span>
     <span style="font-family:'宋体';">
      层提供加密和完整性保护功能，
     </span>
     <span style="font-family:Calibri;">
      RLC
     </span>
     <span style="font-family:'宋体';">
      及
     </span>
