    <h2>
     <a id="_15">
     </a>
     项目运行：
    </h2>
    <blockquote>
     <p>
      这里我们把上个博客分享的如何把wxapkg 解压还原的js项目通过微信开发者打开，运行起来进行调试。(其中会有一些环境的问题会进行报错，我们尝试怎么解决， 并排查问题)
     </p>
    </blockquote>
    <ul>
     <li>
      打开微信开发者工具，把wxapk导入进来。
     </li>
    </ul>
    <p>
     <img alt="List item" src="https://i-blog.csdnimg.cn/blog_migrate/c645d5089b8767af71bade19e7ab9ee2.png"/>
    </p>
    <ul>
     <li>
      项目目录，就是wxapk 解压过来的文件夹(不太清楚解压流程的， 可以看之前分享的博客)，app为(每日优鲜), 这个appid，没有的话去注册一下，速度很快。就不多xx了。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a5b64105fe2651770d1f67e346eb9c99.png"/>
     </li>
     <li>
      项目就打开了, 但是运行起来发现
      <mark>
       <strong>
        plugin
       </strong>
       /wx2b03c6e691cd7370/pages/live-player-plugin.json: [“backgroundColor”]: “black” is not hexColor
      </mark>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6cc672ea7b9500d9de28e7fb36d19ca0.png">
      <br/>
      我们进到
      <mark>
       <strong>
        plugin
       </strong>
       /wx2b03c6e691cd7370/pages/live-player-plugin.json
      </mark>
      ，找到报错行，百度搜了一下报错，发现最新版的微信开发者工具颜色只支持16进制的
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/210fd5b801363aa8e740962b5f7189e8.png">
       <br/>
       给改成:
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8983baa9a9c89b9ccdade952dfd9d39c.png">
        <br/>
        然后command+s (修改完保存自动运行) , windows是ctrl+s, 运行完以后发现还有两个这种错误，继续修改
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c50a8e354723937825bba40d98c6b699.png">
         <mark>
          以 ‘__’ 为开头和结尾的目录为保留目录。
          <strong>
           plugin
          </strong>
          目录下的所有文件将会被忽略。
         </mark>
         这个可以先不管， 忽略就忽略吧， 先运行其他的。
        </img>
       </img>
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6f553d3c756fa4c33e1f71918b4003a1.png">
      <br/>
      颜色报错修改完以后，发现/pages/index/components/cell-lists/search-add/search-add.wxml下还有报错，进去看一了一下，应该是size="{
      <!-- -->
      {width:120,height:32}}" 这个地方有问题，定位不到具体位置，把大括号去掉，就可以继续往下走了， 工具是一步步往下执行，那里有报错就会卡住退出，这篇分享也是逐步往下执行。
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/aaa37d1fb468a19c9232cd1bdb25ce76.png">
      <br/>
      执行到 pulgin-private目录的时候，发现文件一直报错，这里就是坑了
      <code>
       (修改了各种文件，折腾了一个下午才弄明白)
      </code>
      ， 原来是wxappUnpacker 解压出来的文件夹 带了个
      <mark>
       "：”
      </mark>
      导致小程序一直读不到文件，无法往下继续执行，修改文件名称：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/84223ee7062f471b7621f90fcaea81e7.png">
       <br/>
       把pulgin-private： 修改为pulgin-private，然后继续执行， 然后又遇到了这类的报错：
       <mark>
        error: https://dc-eventlog.missfresh.cn 不在以下 request 合法域名列表中，请参考文档
       </mark>
       , 大概意思就是微信开发者工具后台需要给自己设计一些域名，但是可以通过设置不验证域名： 右上角&gt;详情&gt;本地设置&gt;不校验合法域名、web-view(业务域名)
      </img>
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2c4ab55dfa0d69f6bc3fd77bc726e034.png"/>
