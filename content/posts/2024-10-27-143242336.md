## 一、软件测试与AI的纠缠

### 1.1、 什么是人工智能AI？

人工智能是指机器能够完成通常由人类智能完成的任务，例如学习、推理、解决问题和决策。目前在测试中应用比较火的几个方向：机器学习ML和深度学习DL，计算机视觉CV，自然语言处理NLP，还有综合的大语言模型ChatGPT。

### 1.2、什么是软件测试？

通过各种手段验证【程序】是否与预期结果是否一致。以达到提高质量和效能的作用。
  
主要流程是：软件测试计划，设计用例（编写脚本），执行测试，编写测试报告。

### 1.3、软件测试与AI的纠缠

源于二大方面：

* 一、是对于AI技术本身（程序）进行软件测试（
  *AI测试*
  ），它与传统的测试也有些许不同（本文不涉及）。
* 二、是AI技术在软件测试中的应用（
  *AI驱动测试或AI赋能测试*
  ）。

## 二、AI的应用会取代自动化测试工程师吗？

![请添加图片描述](https://i-blog.csdnimg.cn/direct/776513ead44f4a6e8aada37a054b2d16.png)

### 2.1、不会被取代的原因

测试自动化中的人工智能 (AI) 有望改变自动化软件测试员的角色，但
**不太可能**
完全取代他们。目前了解的大概数据是原来10个人完成的工作，现在只需要2-3个自动化测试工程师完成。
  
**原因**
有三:

1. **人工智能的成长性**
   ：现在的人工智能（以ChartGPT为例）虽然很强大，但依然问题很多，它就象一个知识广度很宽的小孩子，从小是美国人培训的（训练数据），还有很多成长空间。目前很多给出的答案是错误的，有认知问题，提示词问题，模糊问题，安全问题等。通常如果你是糊涂的，它也是糊涂的。
2. **人工智能的局限性**
   ：人工智能可以自动执行一些测试任务，但它也有擅长与不擅长。对于那些有固定步骤固定模式的工作步骤，它做的不错。但人类测试人员的认知技能、创造力和解决问题等的能力仍然是不可替代的。
3. **人工智能的地位**
   ：基于人工智能的特点，应用者是不敢完全相信它，即使它克服了错误，成长了，但地位一定要摆正，人类永远都要起监督作用。那测试环节本身就是验证环节，所以公司一定会在重要地方制定的策略必须由人来进行验证的。

## 三、 AI技术在软件测试中的应用

### 3.0、掌握AI技术应用融合的顺序

在软件测试中，我们可以使用人工智能在测试全流程中应用，而且效果还是不错的。通常一个技术来的的变革为分四种：

第一，可以直接替代，没有整合。第二，可以融合的增强或者是改进，第三，是修改部分的一些设计，第四，重塑。
  
![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/907b24b801cc4e54b49d6e0151dfc553.png)

下面三种不同应用是增强、修改和重塑，这是个
**顺序，这个顺序通过你对技术的了解多少而逐渐提升。
  
比如：chatGPT中你如何不了解测试，你无法写出专业的它能理解的提示词，也就会得到通用范范的答案**
。这种就是只是简单的技术积累，而不是两种技术的整合改变。（替代）
  
下面三种是进一步的改变：

1. 测试人员可以把ChartGPT当做测试助手，完成一些测试流程中的小任务。 （增加）
2. 使用带有AI的测试工具完成自己的测试工作任务。（修改）
3. 真正使用AI技术解决测试过程中的难点，以达到提高效率和质量的目的。这需要更高的技术才行。（重塑）

```python
import pytest
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

class TestCSDNRegistration:
    
    @pytest.fixture(autouse=True)
    def setUp(self):
        # 设置Chrome驱动的位置
        self.driver = webdriver.Chrome(executable_path="path/to/chromedriver")
        self.driver.maximize_window()
        yield
        # 关闭浏览器
        self.driver.quit()
    
    def smart_wait(self, by, value, timeout=10):
        """智能等待封装，等待元素加载"""
        return WebDriverWait(self.driver, timeout).until(EC.visibility_of_element_located((by, value)))

    def test_user_registration(self):
        # Step 1: 打开CSDN主页
        self.driver.get("https://www.csdn.net/")

        # Step 2: 导航进入注册页
        register_link = self.smart_wait(By.LINK_TEXT, "注册")  # 根据实际链接文本调整
        register_link.click()

        # Step 3: 输入注册信息
        email_field = self.smart_wait(By.ID, "username")
        email_field.send_keys("testuser@example.com")

        password_field = self.smart_wait(By.ID, "password")
        password_field.send_keys("Test@123")

        confirm_password_field = self.smart_wait(By.ID, "confirmPassword")
        confirm_password_field.send_keys("Test@123")

        # Step 4: 选择必选项（例如性别） - 根据CSDN的表单要求
        gender_option = self.smart_wait(By.ID, "genderMale")  # 如果有性别选择
        gender_option.click()

        # Step 5: 点击注册按钮
        register_button = self.smart_wait(By.ID, "registerButton")
        register_button.click()

        # 预期结果：验证是否成功注册并跳转到欢迎页
        welcome_message = self.smart_wait(By.ID, "welcomeMessage")  # 根据实际页面调整ID
```

```python
import csv

def read_csv_data(file_path):
    """
    读取CSV文件并返回数据列表
    :param file_path: CSV文件路径
    :return: 包含CSV数据的列表，每行数据为字典格式
    """
    data = []
    try:
        with open(file_path, mode='r', encoding='utf-8') as file:
            csv_reader = csv.DictReader(file)  # 使用DictReader将每行数据读入字典
            for row in csv_reader:
                data.append(row)
        print("CSV文件读取成功！")
    except FileNotFoundError:
        print("错误：文件未找到，请检查文件路径是否正确。")
    except Exception as e:
        print(f"读取CSV文件时发生错误: {e}")
    
    return data

# 示例使用
file_path = 'your_file.csv'
csv_data = read_csv_data(file_path)
for row in csv_data:
    print(row)
```

使用 Apache JMeter™、Selenium、Gatling、Grinder 和 Locust 等开源工具对 Web 和移动应用程序、微服务和 API 进行大规模性能测试。
  
将 Android 模拟器和 iOS 模拟器集成到测试策略中，在开发生命周期的早期扩大测试覆盖范围，同时最大限度地减少漏检缺陷。
  
在单一平台内结合虚拟和真实设备，有助于在各种设备类型和操作系统上更快、更彻底地测试应用程序。
  
提供针对 Web 和移动设备的 UX 负载测试，在云中模拟真实情况，可扩展至 200 万虚拟用户。这可确保全面测试应用程序用户体验和负载下的后端性能，从而提高可靠性和效率。
  
执行 API 测试并监控从早期开发到生产阶段的 API 行为。它支持强大的断言来验证数据的准确性，便于测试各种场景，并将性能与功能测试相结合，以便在整个开发生命周期中进行全面的 API 测试和监控。

### 3.3 调用AI技术（机器学习和深度学习等）完善软件测试（高手使用）

#### 3.3.1、测试中的机器学习（ML）

定义：机器学习涉及对历史数据进行算法训练，以便无需明确编程即可做出预测或决策。
  
测试中的应用：
  
测试用例生成：ML 算法分析过去的测试数据和用户行为以自动生成测试用例。这可确保全面覆盖并减少创建测试脚本所需的手动工作量。
  
缺陷预测：通过分析历史缺陷数据，ML 模型可以预测应用程序中最有可能包含缺陷的区域。这使测试人员能够将精力集中在高风险区域，从而提高测试效率和效果。

#### 3.3.2、测试中的自然语言处理（NLP）

定义：自然语言处理是人工智能的一个分支，使机器能够理解、解释和生成人类语言。
  
测试中的应用：
  
需求分析：NLP 工具可以将自然语言需求转换为正式的用户故事和测试用例。这可确保测试涵盖所有需求，并降低误解的风险。
  
缺陷分类：NLP 可以分析错误报告并根据缺陷描述对缺陷进行分类，从而更容易对其进行优先排序和管理。
  
自动故障排除：人工智能可以通过利用由历史数据和实时监控构建的综合知识库来简化问题的识别和解决。利用先进的NLP模型，人工智能可以理解和处理用户查询，提供即时的、特定于上下文的解决方案，或引导用户通过决策树来解决更直接的问题。对于复杂的场景，机器学习模型可以通过识别数据中的模式来预测和推荐解决方案。

#### 3.3.3、测试中的计算机视觉

定义：计算机视觉是训练计算机解释视觉数据并做出决策的人工智能领域。
  
测试中的应用：
  
视觉测试：计算机视觉算法比较屏幕截图和 UI 元素以检测视觉差异。这可确保应用程序在不同设备和屏幕尺寸上的外观和行为正确。
  
UI 验证：自动化 UI 测试工具使用计算机视觉与屏幕上的视觉元素交互，确保 UI 正确响应用户输入。

### 四、人工智能在软件测试中应用的未来

作者观点

#### 4.1、人工智能对测试人员技术要求的影响

* 中小企业测试选择AI驱动的测试工具进行，由于AI一些技术的不透明，导致出现错误定位困难，所以基础必须打牢，整个开发流程技术都得了解。代码不会的，自动化技术不懂的测试人员基本找不到工作。
* 大企业更多种采用自己利用AI技术编写需要的用例脚本报告等，需要学习的很多。同时对于在不同行业领域的测试人员，AI对那个行业的颠覆也要了解，比如自动驾驶、医学检测…

#### 4.2、人工智能对测试流程和CI/CD的影响

技术，时间，方式，手段等的变化，让策略变化 ，同时流程必然随之变化。
  
最终导致企业架构也可能重塑。

#### 4.2、人工智能进一步融合加速

测试人员危机感，是一类重要的少数人群。
  
增加只有人类能做的擅长部分：决策，认识…
