    <h2>
     <a id="React_0">
     </a>
     快速开始React开发（一）
    </h2>
    <p>
     React是一个
     <strong>
      JavaScript库
     </strong>
     ，用于构建交互式网站，并且能够快捷创建
     <strong>
      SPA（Single Page App）
     </strong>
     ，其组件化的思想也是被一再传播，无论是普通的Web网站还是嵌入移动端交互都常见到其身影，那么，搞起~~
    </p>
    <h3>
     <a id="1CRA_3">
     </a>
     1、快速创建项目（CRA）
    </h3>
    <p>
     <strong>
      CRA (Create React App)
     </strong>
     是 React 官方提供的一个
     <strong>
      脚手架
     </strong>
     工具，用于快速创建和搭建 React 项目
    </p>
    <p>
     在此之前，我们先配置一下马上需要用到的
     <code>
      node
     </code>
     环境，
     <code>
      nvm
     </code>
     也推荐大家看一下，考虑到可能需要不同node版本管理，这里不过多介绍，提供传送门，回来时候的状态应该是
     <code>
      node
     </code>
     和
     <code>
      npm
     </code>
     是能用的了
    </p>
    <blockquote>
     <p>
      <a href="https://blog.csdn.net/yunAike/article/details/145343976">
       Node.js安装教程（2025)
      </a>
     </p>
     <p>
      <a href="https://blog.csdn.net/KYGALYX/article/details/144888622?ops_request_misc=%257B%2522request%255Fid%2522%253A%25227851b938cc12a59d7d945ac5e000ec09%2522%252C%2522scm%2522%253A%252220140713.130102334.pc%255Fall.%2522%257D&amp;request_id=7851b938cc12a59d7d945ac5e000ec09&amp;biz_id=0&amp;utm_medium=distribute.pc_search_result.none-task-blog-2~all~first_rank_ecpm_v1~rank_v31_ecpm-2-144888622-null-null.142%5Ev101%5Epc_search_result_base8&amp;utm_term=2025%20nvm&amp;spm=1018.2226.3001.4187">
       （零基础）Node包管理器（nvm）安装及nodejs安装教程（2025年版）
      </a>
     </p>
    </blockquote>
    <p>
     接下来，什么都不需要做，运行这行代码就可以生成一个完整的项目
    </p>
    <pre><code class="prism language-sh">npx create-react-app my-app
</code></pre>
    <p>
     这里我打算使用TypeScript作为主要的开发语言，添加
     <code>
      --template
     </code>
     参数指定模板
    </p>
    <pre><code class="prism language-sh">npx create-react-app my-app <span class="token parameter variable">--template</span> typescript
</code></pre>
    <p>
     啊哦，别慌，小状况，提示需要升级了，那就妥协一下
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/44b92c24deb64983af18d6682f867eee.png">
      <br/>
      升级到最新的
      <code>
       create-react-app
      </code>
     </img>
    </p>
    <pre><code class="prism language-sh"><span class="token function">npm</span> uninstall <span class="token parameter variable">-g</span> create-react-app
<span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> create-react-app@latest
</code></pre>
    <p>
     好了，升到5.1.0，满足要求，Let’s try again!
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5bfad378046641019247b5ba0a4934f0.png"/>
    </p>
    <p>
     耐心等待一小会儿，看到这个就说明项目创建成功啦，我们可以用下面红框的命令进入项目，并运行一下看看
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9cc0461e731045a1925b44c67c5bb478.png"/>
    </p>
    <p>
     此时，会需要选择一下浏览器，页面自动跳转，“小”功告成！
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bdeaa58c0d1b4fbeaf2567005f7c3c60.png"/>
    </p>
    <h3>
     <a id="2_40">
     </a>
     2、项目结构
    </h3>
    <p>
     使用Trae打开项目目录，肉眼可见这几项
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e022fd0e5a3c49d5aab60064e9f698d5.png">
      <br/>
      我们简要看看每一项的作用：
     </img>
    </p>
    <ul>
     <li>
      <code>
       node_modules/
      </code>
      <ul>
       <li>
        存放项目所有的
        <strong>
         依赖包
        </strong>
       </li>
       <li>
        通过
        <code>
         npm/yarn
        </code>
        安装的包都在这里
       </li>
       <li>
        注：不需要提交到代码仓库（运行命令就能下）
       </li>
      </ul>
     </li>
     <li>
      <code>
       public/
      </code>
      <ul>
       <li>
        存放
        <strong>
         静态资源
        </strong>
        文件
       </li>
       <li>
        包含主页面模板
        <code>
         index.html
        </code>
       </li>
       <li>
        存放网站图标、
        <code>
         manifest
        </code>
        等公共资源
       </li>
      </ul>
     </li>
     <li>
      <code>
       src/
      </code>
      <ul>
       <li>
        项目
        <strong>
         源代码
        </strong>
        目录
       </li>
       <li>
        包含入口文件和主要组件
       </li>
       <li>
        开发主要在这个目录下进行
       </li>
      </ul>
     </li>
     <li>
      <code>
       package.json
      </code>
      ：项目配置文件
     </li>
     <li>
      <code>
       README.md
      </code>
      ：项目说明文档
     </li>
     <li>
      <code>
       .gitignore
      </code>
      ：Git忽略配置
     </li>
     <li>
      <code>
       package-lock.json
      </code>
      ：依赖版本锁定文件
     </li>
     <li>
      <code>
       tsconfig.json
      </code>
      ：TypeScript 编译和类型检查的配置文件（ts项目特有）
     </li>
    </ul>
    <p>
     以上大概有些印象即可，主要开发用到比较多的还是
     <code>
      src/
     </code>
     ，这里将会不断随着业务的复杂程度扩充，之后则需要更加细致进行分类管理，比方说下面这样：
    </p>
    <pre><code>my-app/
├── src/
│   ├── api/                 # API 接口管理
│   │   ├── user.ts         # 用户相关接口
│   │   └── index.ts        # API 统一导出
│   │
│   ├── assets/             # 静态资源
│   │   ├── images/         # 图片资源
│   │   ├── styles/         # 全局样式
│   │   └── fonts/          # 字体文件
│   │
│   ├── components/         # 公共组件
│   │   ├── Button/
│   │   │   ├── index.tsx
│   │   │   └── style.module.scss
│   │   └── index.ts
│   │
│   ├── constants/          # 常量配置
│   │   ├── api.ts         # 接口地址
│   │   └── enum.ts        # 枚举值
│   │
│   ├── hooks/             # 自定义 Hooks
│   │   └── useAuth.ts
│   │
│   ├── layouts/           # 布局组件
│   │   └── MainLayout.tsx
│   │
│   ├── pages/            # 页面组件
│   │   └── Home/
│   │       ├── components/  # 页面级组件
│   │       ├── index.tsx
│   │       └── style.module.scss
│   │
│   ├── store/            # 状态管理
│   │   ├── modules/      # 模块分类
│   │   └── index.ts
│   │
│   ├── types/            # TypeScript 类型
│   │   └── user.d.ts
│   │
│   ├── utils/            # 工具函数
│   │   ├── request.ts    # 请求封装
│   │   └── storage.ts    # 存储相关
│   │
│   ├── App.tsx
│   └── index.tsx
│
├── public/              # 静态公共资源
├── tests/              # 测试文件目录
├── package.json
└── tsconfig.json
</code></pre>
    <p>
     一点点开始吧。。。
    </p>
