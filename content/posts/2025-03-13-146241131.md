    <h2>
     <a id="_10">
     </a>
     一、理解标签
    </h2>
    <ul>
     <li>
      <p>
       标签定义：在Git中，标签（tag）是对某次提交（commit）的一个标识，相当于起了一个别名。
      </p>
     </li>
     <li>
      <p>
       应用场景示例：
      </p>
     </li>
     <li>
      <ul>
       <li>
        在项目发布某个版本时，可以针对最后一次提交起一个如v1.0这样的标签来标识里程碑意义。
       </li>
      </ul>
     </li>
     <li>
      <p>
       标签的意义：
      </p>
     </li>
     <li>
      <ul>
       <li>
        相较于难以记住的commit id，标签提供了一个更易记忆且有意义的名字。给重要的提交打上标签后，可以 直接查找该标签以找到对应的commit id，从而使用这个commit id进行版本回退
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="_20">
     </a>
     二、创建标签
    </h2>
    <ul>
     <li>
      <p>
       基本创建：
      </p>
     </li>
     <li>
      <ul>
       <li>
        切换到需要打标签的分支上，执行 git tag [name] 命令即可创建一个新标签，默认是为最新的提交打标签。
       </li>
      </ul>
     </li>
     <li>
      <p>
       查看所有标签：
      </p>
     </li>
     <li>
      <ul>
       <li>
        使用git tag命令查看所有已有的标签。
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      注意，标签不是按时间顺序列出，⽽是按字⺟排序的
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1bbf7f4419c24e1688fc90a7c61816a0.png"/>
    </p>
    <ul>
     <li>
      <p>
       指定提交创建：
      </p>
     </li>
     <li>
      <ul>
       <li>
        如果想要在特定的历史提交上打标签，可以找到该提交的 commit id ，并执行 git tag [标签] [commit id] 。
       </li>
      </ul>
     </li>
     <li>
      <p>
       带有描述信息的标签：
      </p>
     </li>
     <li>
      <ul>
       <li>
        可以为标签添加描述信息，以便未来查看时了解其背景或内容。使用命令git tag -a [标签] -m “描述” [commit id]。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a1d8495df56d4b74b24c20469bc500a3.png"/>
    </p>
    <ul>
     <li>
      查看标签信息：
     </li>
     <li>
      <ul>
       <li>
        使用 git show [标签] 命令可以查看特定标签的信息。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/55f20b96e27046db88a5e30f365b0625.png"/>
    </p>
    <h2>
     <a id="_41">
     </a>
     三、操作标签
    </h2>
    <ul>
     <li>
      删除本地标签：
     </li>
     <li>
      <ul>
       <li>
        如果标签有误，可以通过git tag -d [标签]命令安全地在本地删除。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f980ec37ee6f41a3a16e92fff1658215.png">
      <br/>
      因为目前创建的标签都只存储在本地，不会自动推送到远程。所以，打错的标签可以在本地安全删除。
     </img>
    </p>
    <ul>
     <li>
      推送标签至远程仓库：
     </li>
     <li>
      <ul>
       <li>
        推送单个标签到远程仓库：git push origin（仓库名） [标签]
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        批量推送所有标签：git push origin --tags
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5c21dc11ab3c415daa1076dddce037db.png"/>
    </p>
    <ul>
     <li>
      删除远程标签：
     </li>
     <li>
      <ul>
       <li>
        首先从本地删除：git tag -d [标签]
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        然后从远程删除：git push origin :refs/tags/[标签]
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/35c7d91734e7477b8536e820d04f37bb.png"/>
    </p>

     </li>
     <li>

     </li>
     <li>
      功能开发完毕，将分⽀ merge 进 master，最后删除分⽀。
     </li>
    </ul>

     当前我们已经删除了远程的⼏个分⽀，使⽤
     <strong>
      git branch -a
     </strong>
