    <h2 id="eYMrT">
     1.说说你对盒子模型的理解?
    </h2>
    <h3 id="p6BDt">
     一、是什么
    </h3>
    <p id="ua54cd68b">
     对一个文档进行布局的时候，浏览器渲染引擎会根据标准之一的css基础盒模型，将所有元素表示为一个个矩形的盒子。
    </p>
    <p id="uc74eb441">
     一个盒子由四个部分组成:
     <code>
      content
     </code>
     ，
     <code>
      padding
     </code>
     ，
     <code>
      border
     </code>
     ，
     <code>
      margin
     </code>
    </p>
    <p>
    </p>
    <p id="ubcd6d358">
     下面看看标准盒子模型的模型图：
    </p>
    <p id="u18649ee0">
    </p>
    <p class="img-center">
     <img alt="" height="500" id="u9983e178" src="https://i-blog.csdnimg.cn/img_convert/16377a940465ba7729eb3bd2eb737f3f.png" width="719"/>
    </p>
    <p id="u1a3b1bad">
     从上图可以看到：
    </p>
    <ul>
     <li id="u99b47bb9">
      盒子总宽度 = width + padding + border + margin;
     </li>
     <li id="u866bc287">
      盒子总高度 = height + padding + border + margin
     </li>
    </ul>
    <p id="u8b518257">
     也就是，width/height 只是内容高度，不包含 padding 和 border值
    </p>
    <p id="u7f5ee461">
     所以上面问题中，设置width为200px，但由于存在padding，但实际上盒子的宽度有240px
    </p>

     三、IE 怪异盒子模型

    <p id="uf33d55df">

    <p id="uc59d6ea9">
    </p>
    <p class="img-center">
     <img alt="" height="481" id="u9c1e0a80" src="https://i-blog.csdnimg.cn/img_convert/823c640ec356f7689b10a94f63c7f2e2.png" width="705"/>

    <h3 id="EfhBb">
     Box-sizing
    </h3>
    <p id="u4529b199">
     css 中的 box-sizing 属性定义了引擎应该如何计算一个元素的总宽度和总高度
    </p>
    <p id="u3de0cf63">
     语法：
    </p>
    <pre><code class="language-html">box-sizing: content-box | border-box | inherit;</code></pre>
    <ul>
     <li id="u8c297fce">
      content-box 默认值，元素的 width/height 不包含padding，border，与标准盒子模型表现一致
     </li>
     <li id="ueab6dfd5">
      border-box 元素的 width/height 包含 padding，border，与怪异盒子模型表现一致
     </li>
     <li id="ud1e6e2c8">
      inherit 指定 box-sizing 属性的值，应该从父元素继承
     </li>
    </ul>

    <h2 id="DFBXX">
     3.什么是BFC
    </h2>
    <h3 id="phNdb">
     一、是什么
    </h3>
    <p id="u15413832">
     我们在页面布局的时候，经常出现以下情况：
    </p>
    <ul>
     <li id="u1b15e22e">
      这个元素高度怎么没了？
     </li>
     <li id="u3ff0af69">
      这两栏布局怎么没法自适应？
     </li>
     <li id="uea0a51a9">
      这两个元素的间距怎么有点奇怪的样子？
     </li>
     <li id="u92f8ced9">
      ......
     </li>
    </ul>
    <p id="ufdf55104">
     原因是元素之间相互的影响，导致了意料之外的情况，这里就涉及到BFC概念
    </p>
    <p id="u77ddd1d3">
     BFC（Block Formatting Context），即块级格式化上下文，它是页面中的一块渲染区域，并且有一套属于自己的渲染规则：
    </p>
    <ul>
     <li id="ucf61e5da">
      内部的盒子会在垂直方向上一个接一个的放置
     </li>
     <li id="ucc6c9bce">
      对于同一个BFC的俩个相邻的盒子的margin会发生重叠，与方向无关。
     </li>
    </ul>
    <ul>
     <li id="uaade4f4c">
      每个元素的左外边距与包含块的左边界相接触（从左到右），即使浮动元素也是如此
     </li>
     <li id="ufaccd324">
      BFC的区域不会与float的元素区域重叠
     </li>
     <li id="ubb968399">
      计算BFC的高度时，浮动子元素也参与计算
     </li>
     <li id="u96a162ef">
      BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然
     </li>
    </ul>
    <p id="uc8a33fce">
     BFC目的是形成一个相对于外界完全独立的空间，让内部的子元素不会影响到外部的元素
    </p>
    <h3 id="bgJTy">
     二、触发条件
    </h3>
    <p id="u36c56fb2">
     触发
     <code>
      BFC
     </code>
     的条件包含不限于：
    </p>
    <ul>
     <li id="u0de0c2b8">
      根元素，即HTML元素
     </li>
     <li id="u9aa5a3f9">
      浮动元素：float值为left、right
     </li>
     <li id="u054e7d96">
      overflow值不为 visible，为 auto、scroll、hidden
     </li>
     <li id="u823e8329">
      display的值为inline-block、inltable-cell、table-caption、table、inline-table、flex、inline-flex、grid、inline-grid
     </li>
     <li id="ubf468b92">
      position的值为absolute或fixed
     </li>
    </ul>
    <p>
    </p>

     三、应用场景
    </h3>
    <p id="u7f75e29a">

      BFC

     <code>
      BFC

     应用在以下场景：
    </p>

    <p id="ucd9a0d7e">
     两个p元素之间的距离为100px，发生了margin重叠（塌陷），以最大的为准，如果第一个P的margin为80的话，两个P之间的距离还是100，以最大的为准。

    <h2 id="eeQOe">
     4.如何实现单行/多行文本溢出的省略样式?
    </h2>
    <h4 id="ftfi2">
     单行文本溢出省略
    </h4>
    <p id="u47d52d95">
     理解也很简单，即文本在一行内显示，超出部分以省略号的形式展现
    </p>
    <p id="u39a94e5a">
     实现方式也很简单，涉及的css属性有：
    </p>
    <ul>
     <li id="u15ee8488">
      text-overflow：规定当文本溢出时，显示省略符号来代表被修剪的文本
     </li>
     <li id="u277be46a">
      white-space：设置文字在一行显示，不能换行
     </li>
     <li id="u922f2421">
      overflow：文字长度超出限定宽度，则隐藏超出的内容
     </li>
    </ul>
    <p id="u09dde4f7">
     overflow设为hidden，普通情况用在块级元素的外层隐藏内部溢出元素，或者配合下面两个属性实现文本溢出省略
    </p>
    <p id="u4697659f">
     white-space:nowrap，作用是设置文本不换行，是overflow:hidden和text-overflow：ellipsis生效的基础
    </p>
    <p id="ue5322e0f">
     text-overflow属性值有如下：
    </p>
    <ul>
     <li id="ufd5d5762">
      clip：当对象内文本溢出部分裁切掉
     </li>
     <li id="u9f476b2e">
      ellipsis：当对象内文本溢出时显示省略标记（...）
     </li>
    </ul>
    <p id="u3f336fe7">
     text-overflow只有在设置了overflow:hidden和white-space:nowrap才能够生效的
    </p>
    <p id="u32d3adc6">
     举个例子
    </p>
    <pre id="zpnvL"><code class="language-css">&lt;style&gt;
    p{
        line-height: 40px;
        width:400px;
        height:40px;
        border:1px solid red;
        overflow: hidden;
        text-overflow: ellipsis;
        white-space: nowrap;
    }
&lt;/style&gt;
&lt;p 这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本这是一些文本&lt;/p &gt;</code></pre>
    <h4 id="SxKAv">
     多行文本溢出省略
    </h4>
    <p id="u815c77d0">
     多行文本溢出的时候，我们可以分为两种情况：
    </p>
    <ul>
     <li id="u914fa844">
      基于高度截断
     </li>
     <li id="u2b496491">
      基于行数截断
     </li>
    </ul>
    <h5 id="x6UBJ">
     基于高度截断
    </h5>
    <h6 id="wxzsx">
     伪元素 + 定位
    </h6>
    <p id="uf5b8e969">
     核心的css代码结构如下：
    </p>
    <ul>
     <li id="u61509cd4">
      position: relative：为伪元素绝对定位
     </li>
     <li id="u694a8e10">
      overflow: hidden：文本溢出限定的宽度就隐藏内容）
     </li>
     <li id="ua39a4cbe">
      position: absolute：给省略号绝对定位
     </li>
     <li id="uce0995dc">
      line-height: 20px：结合元素高度,高度固定的情况下,设定行高, 控制显示行数
     </li>
     <li id="ua74ff066">
      height: 40px：设定当前元素高度
     </li>
     <li id="uef87e694">
      ::after {} ：设置省略号样式
     </li>
    </ul>
    <p id="ue1d10f34">
     代码如下所示：
    </p>
    <pre id="z891Y"><code>&lt;style&gt;
    .demo {
        position: relative;
        line-height: 20px;
        height: 40px;
        overflow: hidden;
    }
    .demo::after {
        content: "...";
        position: absolute;
        bottom: 0;
        right: 0;
        padding: 0 20px 0 10px;
    }
&lt;/style&gt;

&lt;body&gt;
    &lt;div class='demo'&gt;这是一段很长的文本&lt;/div&gt;
&lt;/body&gt;</code></pre>
    <p id="u383255c4">
     实现原理很好理解，就是通过伪元素绝对定位到行尾并遮住文字，再通过 overflow: hidden 隐藏多余文字
    </p>
    <p id="u8635e0cd">
     这种实现具有以下优点：
    </p>
    <ul>
     <li id="u01b424f0">
      兼容性好，对各大主流浏览器有好的支持
     </li>
     <li id="uc34fcc59">
      响应式截断，根据不同宽度做出调整
     </li>
    </ul>
    <p id="uecd145b3">
     一般文本存在英文的时候，可以设置word-break: break-all使一个单词能够在换行时进行拆分
    </p>
    <h5 id="zPfg0">
     基于行数截断
    </h5>
    <p id="u42cbd8c9">
     纯css实现也非常简单，核心的css代码如下：
    </p>
    <ul>
     <li id="ud7004596">
      -webkit-line-clamp: 2：用来限制在一个块元素显示的文本的行数，为了实现该效果，它需要组合其他的WebKit属性）
     </li>
     <li id="u993c23fe">
      display: -webkit-box：和1结合使用，将对象作为弹性伸缩盒子模型显示
     </li>
     <li id="ue53fe1a3">
      -webkit-box-orient: vertical：和1结合使用 ，设置或检索伸缩盒对象的子元素的排列方式
     </li>
     <li id="uaeb0160b">
      overflow: hidden：文本溢出限定的宽度就隐藏内容
     </li>
     <li id="u20786d11">
      text-overflow: ellipsis：多行文本的情况下，用省略号“…”隐藏溢出范围的文本
     </li>
    </ul>
    <pre id="r6hHp"><code>&lt;style&gt;
    p {
        width: 400px;
        border-radius: 1px solid red;
        -webkit-line-clamp: 2;
        display: -webkit-box;
        -webkit-box-orient: vertical;
        overflow: hidden;
        text-overflow: ellipsis;
    }
&lt;/style&gt;

    <h2 id="n6aix">
     6.如何使用css完成视差滚动效果?
    </h2>
    <h3 id="FrpoW">
     实现方式
    </h3>
    <p id="uc8378f26">
     使用css形式实现视觉差滚动效果的方式有：
    </p>
    <ul>
     <li id="u85b9c5b7">
      background-attachment
     </li>
     <li id="u2fa5fa5c">
      transform:translate3D
     </li>
    </ul>
    <h4 id="PgISH">
     background-attachment
    </h4>
    <p id="u47c90f8a">
     作用是设置背景图像是否固定或者随着页面的其余部分滚动
    </p>
    <p id="ueb9c56d2">
     值分别有如下：
    </p>
    <ul>
     <li id="ud5cb7370">
      scroll：默认值，背景图像会随着页面其余部分的滚动而移动
     </li>
     <li id="ufb5a1be7">
      fixed：当页面的其余部分滚动时，背景图像不会移动
     </li>
     <li id="ua5bd0fec">
      inherit：继承父元素background-attachment属性的值
     </li>
    </ul>
    <p id="u0c6fb768">
     完成滚动视觉差就需要将background-attachment属性设置为fixed，让背景相对于视口固定。及时一个元素有滚动机制，背景也不会随着元素的内容而滚动
    </p>
    <p id="u120d9b88">
     也就是说，背景一开始就已经被固定在初始的位置
    </p>
    <h4 id="vcRzC">
     transform:translate3D
    </h4>
    <p id="u25e9fc86">
     同样，让我们先来看一下两个概念transform和perspective：
    </p>
    <ul>
     <li id="u273567b7">
      transform: css3 属性，可以对元素进行变换(2d/3d)，包括平移 translate,旋转 rotate,缩放 scale,等等
     </li>
     <li id="u57341291">
      perspective: css3 属性，当元素涉及 3d 变换时，perspective 可以定义我们眼睛看到的 3d 立体效果，即空间感
     </li>
    </ul>

     7.CSS选择器
    </h2>
    <h3 id="v2DUw">
     选择器
    </h3>
    <p id="u1d419b30">
     它是元素和其他部分组合起来告诉告诉浏览器哪个HTML元素应当是被选为应用规则中的css属性值的方式
    </p>
    <p id="u7fa7f361">
     关于css属性选择器常用的有：
    </p>
    <ul>
     <li id="u4abafcb6">
      id选择器（#box），选择id为box的元素
     </li>
     <li id="u2c9fcd32">
      类选择器（.one），选择类名为one的所有元素
     </li>
     <li id="u46596707">
      标签选择器（div），选择标签为div的所有元素
     </li>
     <li id="ub77f9f9d">
      后代选择器（#box div），选择id为box元素内部所有的div元素
     </li>
     <li id="u1a92a038">
      子选择器（.one&gt;one_1），选择父元素为.one的所有.one_1的元素
     </li>
     <li id="ue9b74f0b">
      相邻同胞选择器（.one+.two），选择紧接在.one之后的所有.two元素
     </li>
     <li id="uf77e35e1">
      群组选择器（div,p），选择div、p的所有元素
     </li>
    </ul>

    <h2 id="H3CR4">
     10.说说对Css预编语言的理解？有哪些区别?
    </h2>
    <h3 id="zy7Vc">
     一、是什么
    </h3>
    <p id="u62b4726b">
     Css 作为一门标记性语言，语法相对简单，对使用者的要求较低，但同时也带来一些问题
    </p>
    <p id="u1e85ddbe">
     需要书写大量看似没有逻辑的代码，不方便维护及扩展，不利于复用，尤其对于非前端开发工程师来讲，往往会因为缺少 Css 编写经验而很难写出组织良好且易于维护的 Css 代码
    </p>
    <p id="uf850ae1d">
     Css预处理器便是针对上述问题的解决方案
    </p>
    <h5 id="pl4ND">
     预处理语言
    </h5>
    <p id="ubb5654b7">
     <strong>
      扩充了 Css 语言，增加了诸如变量、混合（mixin）、函数等功能，让 Css 更易维护、方便
     </strong>
    </p>
    <p id="ub73fe109">
     本质上，预处理是Css的超集
    </p>
    <p id="u08127529">
     包含一套自定义的语法及一个解析器，根据这些语法定义自己的样式规则，这些规则最终会通过解析器，编译生成对应的 Css 文件
    </p>
    <h3 id="PJORn">
     二、有哪些
    </h3>
    <p id="ufe56e486">
     Css预编译语言在前端里面有三大优秀的预编处理器，分别是：
    </p>
    <ul>
     <li id="u1b319404">
      sass
     </li>
     <li id="u33aded70">
      less
     </li>
     <li id="ufacc781f">
      stylus
     </li>
    </ul>
    <h3 id="hUxkq">
     三、区别
    </h3>
    <p id="ue03dd61e">
     虽然各种预处理器功能强大，但使用最多的，还是以下特性：
    </p>
    <ul>
     <li id="u35a0898f">
      变量（variables）
     </li>
     <li id="u6454044b">
      作用域（scope）
     </li>
     <li id="u804e0932">
      代码混合（ mixins）
     </li>
     <li id="udf783c8b">
      嵌套（nested rules）
     </li>
     <li id="u879e7d52">
      代码模块化（Modules）
     </li>
    </ul>
    <p id="ubd5f11fb">
     因此，下面就展开这些方面的区别
    </p>
    <h4 id="wzBbX">
     基本使用
    </h4>
    <p id="u2a620cbc">
     less和scss
    </p>
    <pre id="d8Ej0"><code>.box {
  display: block;
}</code></pre>
    <p id="u11fafd9b">
     sass
    </p>
    <pre id="XlrBm"><code>.box
  display: block</code></pre>
    <p id="ub98ac6e0">
     stylus
    </p>
    <pre id="io0tB"><code>.box
  display: block</code></pre>
    <h4 id="W0Tu6">
     嵌套
    </h4>
    <p id="uf7e89f7d">
     三者的嵌套语法都是一致的，甚至连引用父级选择器的标记 &amp; 也相同
    </p>
    <p id="u8d4b01ea">
     区别只是 Sass 和 Stylus 可以用没有大括号的方式书写
    </p>
    <p id="ue3d4d42f">
     less
    </p>
    <pre id="V2AJs"><code>.a {
  &amp;.b {
    color: red;
  }
}</code></pre>
    <h4 id="KUBhc">
     变量
    </h4>
    <p id="u2f5c0dac">
     变量无疑为 Css 增加了一种有效的复用方式，减少了原来在 Css 中无法避免的重复「硬编码」
    </p>
    <p id="ua635e944">
     less声明的变量必须以@开头，后面紧跟变量名和变量值，而且变量名和变量值需要使用冒号:分隔开
    </p>
    <pre id="ZASSk"><code>@red: #c00;

strong {
  color: @red;
}</code></pre>
    <p id="u1210dd6f">
     sass声明的变量跟less十分的相似，只是变量名前面使用$开头
    </p>
    <pre id="LroGX"><code>$red: #c00;

strong {
  color: $red;
}</code></pre>
    <p id="ucd005ddc">
     stylus声明的变量没有任何的限定，可以使用$开头，结尾的分号;可有可无，但变量与变量值之间需要使用=
    </p>
    <p id="ud7727c37">
     在stylus中我们不建议使用@符号开头声明变量
    </p>
    <pre id="Q26Go"><code>red = #c00

strong
  color: red</code></pre>
    <h4 id="rv8XR">
     作用域
    </h4>
    <p id="u60da067a">
     Css 预编译器把变量赋予作用域，也就是存在生命周期。就像 js一样，它会先从局部作用域查找变量，依次向上级作用域查找
    </p>
    <p id="u18035f5e">
     sass中不存在全局变量
    </p>
    <pre id="YkcIz"><code>$color: black;
.scoped {
  $bg: blue;
  $color: white;
  color: $color;
  background-color:$bg;
}
.unscoped {
  color:$color;
}</code></pre>
    <p id="uc8276935">
     编译后
    </p>
    <pre id="iEnfu"><code>.scoped {
  color:white;/*是白色*/
  background-color:blue;
}
.unscoped {
  color:white;/*白色（无全局变量概念）*/
}</code></pre>
    <p id="ubf5c213f">
     所以，在sass中最好不要定义相同的变量名
    </p>
    <p id="ub0006772">
     less与stylus的作用域跟javascript十分的相似，首先会查找局部定义的变量，如果没有找到，会像冒泡一样，一级一级往下查找，直到根为止
    </p>
    <pre id="Z5kph"><code>@color: black;
.scoped {
  @bg: blue;
  @color: white;
  color: @color;
  background-color:@bg;
}
.unscoped {
  color:@color;
}</code></pre>
    <p id="uf91d3ee1">
     编译后：
    </p>
    <pre id="xhOr7"><code>.scoped {
  color:white;/*白色（调用了局部变量）*/
  background-color:blue;
}
.unscoped {
  color:black;/*黑色（调用了全局变量）*/
}</code></pre>
    <h4 id="P7Jya">
     混入
    </h4>
    <p id="u746b3fe8">
     混入（mixin）应该说是预处理器最精髓的功能之一了，简单点来说，Mixins可以将一部分样式抽出，作为单独定义的模块，被很多选择器重复使用
    </p>
    <p id="u44f2e19f">
     可以在Mixins中定义变量或者默认参数
    </p>
    <p id="u105a154a">
     在less中，混合的用法是指将定义好的ClassA中引入另一个已经定义的Class，也能使用够传递参数，参数变量为@声明
    </p>
    <pre id="WSlbg"><code>.alert {
  font-weight: 700;
}

.highlight(@color: red) {
  font-size: 1.2em;
  color: @color;
}

.heads-up {
  .alert;
  .highlight(red);
}</code></pre>
    <p id="u1689956a">
     编译后
    </p>
    <pre id="wxiBZ"><code>.alert {
  font-weight: 700;
}
.heads-up {
  font-weight: 700;
  font-size: 1.2em;
  color: red;
}</code></pre>
    <p id="u9280c31a">
     Sass声明mixins时需要使用@mixinn，后面紧跟mixin的名，也可以设置参数，参数名为变量$声明的形式
    </p>
    <pre id="mzdK2"><code>@mixin large-text {
  font: {
    family: Arial;
    size: 20px;
    weight: bold;
  }
  color: #ff0000;
}

.page-title {
  @include large-text;
  padding: 4px;
  margin-top: 10px;
}</code></pre>
    <p id="uc60f467d">
     stylus中的混合和前两款Css预处理器语言的混合略有不同，他可以不使用任何符号，就是直接声明Mixins名，然后在定义参数和默认值之间用等号（=）来连接
    </p>
    <pre id="jGLGS"><code>error(borderWidth= 2px) {
  border: borderWidth solid #F00;
  color: #F00;
}
.generic-error {
  padding: 20px;
  margin: 4px;
  error(); /* 调用error mixins */
}
.login-error {
  left: 12px;
  position: absolute;
  top: 20px;
  error(5px); /* 调用error mixins，并将参数$borderWidth的值指定为5px */
}</code></pre>
    <h4 id="h6nvH">
     代码模块化
    </h4>
    <p id="udfde9ca5">
     模块化就是将Css代码分成一个个模块
    </p>
    <p id="uc2ec24da">
     scss、less、stylus三者的使用方法都如下所示
    </p>
    <pre id="WMzFX"><code>@import './common';
@import './github-markdown';
@import './mixin';
@import './variables';</code></pre>
    <h2 id="gXtaH">
     11.什么是响应式设计？响应式设计的基本原理是什么？如何做？
    </h2>
    <p>
     <img alt="" height="331" src="https://i-blog.csdnimg.cn/direct/9af6b10febd940cbb30997c7000c56f7.png" width="658"/>
    </p>
    <h3 id="XMYHS">
     一、是什么
    </h3>
    <p id="u81c9a3eb">
    </p>
    <p id="u93739cb4">
     一套代码，搞定pc端和移动端，根据不同的屏幕尺寸，应用不同的样式
    </p>
    <p id="uda150fd6">
     响应式网站设计（Responsive Web design）是一种网络页面设计布局，页面的设计与开发应当根据用户行为以及设备环境(系统平台、屏幕尺寸、屏幕定向等)进行相应的响应和调整
    </p>
    <p id="ucfa684d6">
     描述响应式界面最著名的一句话就是“Content is like water”
    </p>
    <p id="u03681d15">
     大白话便是“如果将屏幕看作容器，那么内容就像水一样”
    </p>
    <p id="u158322a2">
     响应式网站常见特点：
    </p>
    <ul>
     <li id="u7f297443">
      同时适配PC + 平板 + 手机等
     </li>
     <li id="uf2cebff8">
      标签导航在接近手持终端设备时改变为经典的抽屉式导航
     </li>
     <li id="u28b4a339">
      网站的布局会根据视口来调整模块的大小和位置
     </li>
    </ul>
    <p id="u49deee76">
    </p>
    <p class="img-center">
     <img alt="" height="298" id="u37ebf250" src="https://i-blog.csdnimg.cn/img_convert/ae5d7b41fa5ff352699f9c476132ac50.png" width="526"/>
    </p>
    <h3 id="QUYtf">
     二、实现方式
    </h3>
    <p id="u127e2f46">
     响应式设计的基本原理是通过媒体查询检测不同的设备屏幕尺寸做处理，为了处理移动端，页面头部必须有meta声明viewport
    </p>
    <pre id="EwJRa"><code>&lt;meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no”&gt;</code></pre>
    <p id="uc4577384">
     属性对应如下：
    </p>
    <ul>
     <li id="u0819317f">
      width=device-width: 是自适应手机屏幕的尺寸宽度
     </li>
     <li id="u0c72efca">
      maximum-scale:是缩放比例的最大值
     </li>
     <li id="ueb3bd9fb">
      inital-scale:是缩放的初始化
     </li>
     <li id="uf62419ac">
      user-scalable:是用户的可以缩放的操作
     </li>
    </ul>
    <p id="u4d06618a">
     实现响应式布局的方式有如下：
    </p>
    <ul>
     <li id="ub2c10865">
      媒体查询
     </li>
     <li id="uf83ce114">
      百分比
     </li>
     <li id="u474cd787">
      vw/vh
     </li>
     <li id="u984dea84">
      rem
     </li>
    </ul>
    <h4 id="XMVcn">
     媒体查询
    </h4>
    <p id="u60280208">
     CSS3中的增加了更多的媒体查询，就像if条件表达式一样，我们可以设置不同类型的媒体条件，并根据对应的条件，给相应符合条件的媒体调用相对应的样式表
    </p>
    <p id="udf31f429">
     使用@Media查询，可以针对不同的媒体类型定义不同的样式，如：
    </p>
    <pre id="QAS4a"><code>@media screen and (max-width: 1920px) { ... }</code></pre>
    <p id="u178d3c4f">
     当视口在375px - 600px之间，设置特定字体大小18px
    </p>
    <pre id="ysSNw"><code>@media screen (min-width: 375px) and (max-width: 600px) {
  body {
    font-size: 18px;
  }
}</code></pre>
    <p id="u8819c375">
     通过媒体查询，可以通过给不同分辨率的设备编写不同的样式来实现响应式的布局，比如我们为不同分辨率的屏幕，设置不同的背景图片
    </p>
    <p id="u454c3496">
     比如给小屏幕手机设置@2x图，为大屏幕手机设置@3x图，通过媒体查询就能很方便的实现
    </p>
    <h4 id="mRRqu">
     百分比
    </h4>
    <p id="ubafb7309">
     通过百分比单位 " % " 来实现响应式的效果
    </p>
    <p id="uac899442">
     比如当浏览器的宽度或者高度发生变化时，通过百分比单位，可以使得浏览器中的组件的宽和高随着浏览器的变化而变化，从而实现响应式的效果
    </p>
    <p id="u405f5520">
     height、width属性的百分比依托于父标签的宽高，但是其他盒子属性则不完全依赖父元素：
    </p>
    <ul>
     <li id="ud8b15c84">
      子元素的top/left和bottom/right如果设置百分比，则相对于直接非static定位(默认定位)的父元素的高度/宽度
     </li>
     <li id="u6fb6d2cb">
      子元素的padding如果设置百分比，不论是垂直方向或者是水平方向，都相对于直接父亲元素的width，而与父元素的height无关。
     </li>
     <li id="ue00ae3f1">
      子元素的margin如果设置成百分比，不论是垂直方向还是水平方向，都相对于直接父元素的width
     </li>
     <li id="u4fb400dd">
      border-radius不一样，如果设置border-radius为百分比，则是相对于自身的宽度
     </li>
    </ul>
    <p id="udb493f07">
     可以看到每个属性都使用百分比，会照成布局的复杂度，所以不建议使用百分比来实现响应式
    </p>
    <h4 id="rT23i">
     vw/vh
    </h4>
    <p id="u758dcad9">
     vw表示相对于视图窗口的宽度，vh表示相对于视图窗口高度。 任意层级元素，在使用vw单位的情况下，1vw都等于视图宽度的百分之一
    </p>
    <p id="ua23659a8">
     与百分比布局很相似，在以前文章提过与%的区别，这里就不再展开述说
    </p>
    <h4 id="uMtMs">
     rem
    </h4>
    <p id="u75ad8ab2">
     在以前也讲到，rem是相对于根元素html的font-size属性，默认情况下浏览器字体大小为16px，此时1rem = 16px
    </p>
    <p id="u5def4b53">
     可以利用前面提到的媒体查询，针对不同设备分辨率改变font-size的值，如下：
    </p>
    <pre id="COZXM"><code>@media screen and (max-width: 414px) {
  html {
    font-size: 18px
  }
}

@media screen and (max-width: 375px) {
  html {
    font-size: 16px
  }
}

@media screen and (max-width: 320px) {
  html {
    font-size: 12px
  }
}</code></pre>
    <p id="u29dab882">
     为了更准确监听设备可视窗口变化，我们可以在css之前插入script标签，内容如下：
    </p>
    <pre id="dvavO"><code>//动态为根元素设置字体大小
function init () {
  // 获取屏幕宽度
  var width = document.documentElement.clientWidth
  // 设置根元素字体大小。此时为宽的10等分
  document.documentElement.style.fontSize = width / 10 + 'px'
}

//首次加载应用，设置一次
init()
// 监听手机旋转的事件的时机，重新设置
window.addEventListener('orientationchange', init)
// 监听手机窗口变化，重新设置
window.addEventListener('resize', init)</code></pre>
    <p id="u1f179b2c">
     无论设备可视窗口如何变化，始终设置rem为width的1/10，实现了百分比布局
    </p>
    <p id="u503414a8">
     除此之外，我们还可以利用主流UI框架，如：element ui、antd提供的栅格布局实现响应式
    </p>
    <h4 id="tScT3">
     小结
    </h4>
    <p id="u4c1b64d0">
     响应式设计实现通常会从以下几方面思考：
    </p>
    <ul>
     <li id="ua6ca5762">
      弹性盒子（包括图片、表格、视频）和媒体查询等技术
     </li>
     <li id="ua6a4cc03">
      使用百分比布局创建流式布局的弹性UI，同时使用媒体查询限制元素的尺寸和内容变更范围
     </li>
     <li id="ubcc6bd07">
      使用相对单位使得内容自适应调节
     </li>
     <li id="u11508682">
      选择断点，针对不同断点实现不同布局和内容展示
     </li>
    </ul>

    <h4 id="rufad">
     px
    </h4>
    <p id="udfb3baee">
     px，表示像素，所谓像素就是呈现在我们显示器上的一个个小点，每个像素点都是大小等同的，所以像素为计量单位被分在了绝对长度单位中
    </p>
    <p id="u86492755">
     有些人会把px认为是相对长度，原因在于在移动端中存在设备像素比，px实际显示的大小是不确定的
    </p>
    <p id="u9ac05c4e">
     这里之所以认为px为绝对单位，在于px的大小和元素的其他属性无关
    </p>
    <h4 id="IpHgA">
     em
    </h4>
    <p id="ue47c86af">
     em是相对长度单位。相对于当前对象内文本的字体尺寸。如当前对行内文本的字体尺寸未被人为设置，则相对于浏览器的默认字体尺寸（1em = 16px）
    </p>
    <p id="uc407b3b2">
     为了简化 font-size 的换算，我们需要在css中的 body 选择器中声明font-size= 62.5%，这就使 em 值变为 16px*62.5% = 10px
    </p>
    <p id="u22d5a8b5">
     这样 12px = 1.2em, 10px = 1em, 也就是说只需要将你的原来的px 数值除以 10，然后换上 em作为单位就行了
    </p>
    <p id="ud353aae0">
     特点：
    </p>
    <ul>
     <li id="u96e6393b">
      em 的值并不是固定的
     </li>
     <li id="uf3b26019">
      em 会继承父级元素的字体大小
     </li>
     <li id="ube5d0c42">
      em 是相对长度单位。相对于当前对象内文本的字体尺寸。如当前对行内文本的字体尺寸未被人为设置，则相对于浏览器的默认字体尺寸
     </li>
     <li id="u427f9d2a">
      任意浏览器的默认字体高都是 16px
     </li>
    </ul>

    <h4 id="mEpzH">
     rem
    </h4>
    <p id="ub20525ee">
     rem，相对单位，相对的只是HTML根元素font-size的值
    </p>
    <p id="u43444028">
     同理，如果想要简化font-size的转化，我们可以在根元素html中加入font-size: 62.5%
    </p>
    <pre id="ari0M"><code>html {font-size: 62.5%;  } /*  公式16px*62.5%=10px  */</code></pre>
    <p id="ufca99524">
     这样页面中1rem=10px、1.2rem=12px、1.4rem=14px、1.6rem=16px;使得视觉、使用、书写都得到z极大的帮助
    </p>
    <p id="u5baad1f5">
     特点：
    </p>
    <ul>
     <li id="ua0faf12a">
      rem单位可谓集相对大小和绝对大小的优点于一身
     </li>
     <li id="uc1784331">
      和em不同的是rem总是相对于根元素，而不像em一样使用级联的方式来计s算尺寸
     </li>
    </ul>
    <h4 id="fUdoI">
     vh、vw
    </h4>
    <p id="u5622c1a3">
     vw ，就是根据窗口的宽度，分成100等份，100vw就表示满宽，50vw就表示一半宽。（vw 始终是针对窗口的宽），同理，vh则为窗口的高度
    </p>
    <p id="u8a27d6f4">
     这里的窗口分成几种情况：
    </p>
    <ul>
     <li id="u0ffc1676">
      在桌面端，指的是浏览器的可视区域
     </li>
     <li id="uba311fdf">
      移动端指的就是布局视口
     </li>
    </ul>
    <p id="u5ed04870">
     像vw、vh，比较容易混淆的一个单位是%，不过百分比宽泛的讲是相对于父元素：
    </p>
    <ul>
     <li id="u620f48d0">
      对于普通定位元素就是我们理解的父元素
     </li>
     <li id="u3bef7ef6">
      对于position: absolute;的元素是相对于已定位的父元素
     </li>
     <li id="u29578ee8">
      对于position: fixed;的元素是相对于 ViewPort（可视窗口）
     </li>
    </ul>
    <h3 id="u8cca">
     三、总结
    </h3>
    <p id="u63608b0f">
     <strong>
      px
     </strong>
     ：绝对单位，页面按精确像素展示
    </p>
    <p id="u09cb64a7">
     <strong>
      em
     </strong>
     ：相对单位，基准点为父节点字体的大小，如果自身定义了font-size按自身来计算，整个页面内1em不是一个固定的值
    </p>
    <p id="u054e9f54">
     <strong>
      rem
     </strong>
     ：相对单位，可理解为root em, 相对根节点html的字体大小来计算
    </p>
    <p id="u82f3b454">
     <strong>
      vh、vw
     </strong>
     ：主要用于页面视口大小布局，在页面布局上更加方便简单
    </p>
    <p>
    </p>
    <h2 id="daLvb">
     13.css中，有哪些方式可以隐藏页面元素？区别?
    </h2>
    <h3 id="nI80L">
     一、前言
    </h3>
    <p id="ufc1784cc">
     在平常的样式排版中，我们经常遇到将某个模块隐藏的场景
    </p>
    <p id="ud6056ffd">
     通过css隐藏元素的方法有很多种，它们看起来实现的效果是一致的
    </p>
    <p id="u2e70e7b3">
     但实际上每一种方法都有一丝轻微的不同，这些不同决定了在一些特定场合下使用哪一种方法
    </p>
    <h3 id="f4iIx">
     二、实现方式
    </h3>
    <p id="u4f0c4b0c">
     通过css实现隐藏元素方法有如下：
    </p>
    <ul>
     <li id="u1db121ae">
      display:none
     </li>
     <li id="u5272aba8">
      visibility:hidden 隐藏了还是占据位置
     </li>
     <li id="u7735423e">
      opacity:0 透明度
     </li>
     <li id="ue33d14eb">
      设置height、width模型属性为0
     </li>
     <li id="ued71cceb">
      position:absolute
     </li>
     <li id="u9296deb4">
      clip-path
     </li>
    </ul>
    <h4 id="y5a1H">
     display:none
    </h4>
    <p id="ua92bcf82">
     设置元素的display为none是最常用的隐藏元素的方法
    </p>
    <pre id="b379s"><code>.hide {
    display:none;
}</code></pre>
    <p id="uec4278d5">
     将元素设置为display:none后，元素在页面上将彻底消失
    </p>
    <p id="u226b44fe">
     <strong>
      元素本身占有的空间就会被其他元素占有，也就是说它会导致浏览器的重排和重绘
     </strong>
    </p>
    <p id="ud87675ef">
     消失后，自身绑定的事件不会触发，也不会有过渡效果
    </p>
    <p id="u76164be3">
     特点：元素不可见，不占据空间，无法响应点击事件
    </p>
    <h4 id="yIwmp">
     visibility:hidden
    </h4>
    <p id="ube0e03a8">
     设置元素的visibility为hidden也是一种常用的隐藏元素的方法
    </p>
    <p id="u3a3d890c">
     从页面上仅仅是隐藏该元素，DOM结果均会存在，只是当时在一个不可见的状态，不会触发重排，但是会触发重绘
    </p>
    <pre id="a89nA"><code>.hidden{
    visibility:hidden
}</code></pre>
    <p id="uc86c7c74">
     给人的效果是隐藏了，所以他自身的事件不会触发
    </p>
    <p id="u00bcb2a4">
     特点：元素不可见，占据页面空间，无法响应点击事件
    </p>
    <p>
    </p>
    <h4 id="rZQpG">
     opacity:0
    </h4>
    <p id="u575d4edf">
     opacity属性表示元素的透明度，将元素的透明度设置为0后，在我们用户眼中，元素也是隐藏的
    </p>
    <p id="u4abf6ff2">
     不会引发重排，一般情况下也会引发重绘
    </p>
    <p id="uf204ea1b">
     如果利用 animation 动画，对 opacity 做变化（animation会默认触发GPU加速），则只会触发 GPU 层面的 composite，不会触发重绘
    </p>
    <p id="u6bf38c14">
     需要注意的是：其子元素不能设置opacity来达到显示的效果
    </p>
    <p id="u5db8850a">
     特点：改变元素透明度，元素不可见，占据页面空间，可以响应点击事件
    </p>
    <h4 id="pUZBn">
     设置height、width属性为0
    </h4>
    <p id="u455ecdd2">
     将元素的margin，border，padding，height和width等影响元素盒模型的属性设置成0，如果元素内有子元素或内容，还应该设置其overflow:hidden来隐藏其子元素
    </p>
    <pre id="J5QMB"><code>.hiddenBox {
    margin:0;     
    border:0;
    padding:0;
    height:0;
    width:0;
    overflow:hidden;
}</code></pre>
    <p id="u0cb6294a">
     特点：元素不可见，不占据页面空间，无法响应点击事件
    </p>
    <h4 id="JsMWM">
     position:absolute
    </h4>
    <p id="u93870b15">
     将元素移出可视区域
    </p>
    <pre id="IVab9"><code>.hide {
   position: absolute;
   top: -9999px;
   left: -9999px;
}</code></pre>
    <p id="uf41b2474">
     特点：元素不可见，不影响页面布局
    </p>
    <h4 id="RS327">
     clip-path
    </h4>
    <p id="ubea6fb86">
     通过裁剪的形式
    </p>
    <pre id="IgKZW"><code>.hide {
  clip-path: polygon(0px 0px,0px 0px,0px 0px,0px 0px);
}</code></pre>
    <p id="ufbc41f59">
     特点：元素不可见，占据页面空间，无法响应点击事件
    </p>
    <h4 id="uo5jb">
     小结
    </h4>
    <p id="u39b6144f">
     最常用的还是display:none和visibility:hidden，其他的方式只能认为是奇招，它们的真正用途并不是用于隐藏元素，所以并不推荐使用它们
    </p>

    <h2>
     14.DOM常见的操作有哪些？
    </h2>
    <h3 id="Agbyn">
     一、DOM
    </h3>
    <p id="u1acd754b">
     文档对象模型 (DOM) 是 HTML 和 XML 文档的编程接口
    </p>
    <p id="u8faaf16b">
     它提供了对文档的结构化的表述，并定义了一种方式可以使从程序中对该结构进行访问，从而改变文档的结构，样式和内容
    </p>
    <p id="u0ff55eb0">
     任何 HTML或XML文档都可以用 DOM表示为一个由节点构成的层级结构
    </p>
    <p id="uaf587a93">
     节点分很多类型，每种类型对应着文档中不同的信息和（或）标记，也都有自己不同的特性、数据和方法，而且与其他类型有某种关系，如下所示：
    </p>
    <pre id="Ue2g3"><code>&lt;html&gt;
    &lt;head&gt;
        &lt;title&gt;Page&lt;/title&gt;
    &lt;/head&gt;
    &lt;body&gt;
        &lt;p&gt;Hello World!&lt;/p &gt;
    &lt;/body&gt;
&lt;/html&gt;</code></pre>
    <p id="u0d186c66">
     DOM像原子包含着亚原子微粒那样，也有很多类型的DOM节点包含着其他类型的节点。接下来我们先看看其中的三种：
    </p>
    <pre id="pX7ds"><code>&lt;div&gt;
    &lt;p title="title"&gt;
        content
    &lt;/p &gt;
&lt;/div&gt;</code></pre>
    <p id="u025398b5">
     上述结构中，div、p就是元素节点，content就是文本节点，title就是属性节点
    </p>
    <h3 id="f1ljO">
     二、操作
    </h3>
    <p id="u419fa905">
     日常前端开发，我们都离不开DOM操作
    </p>
    <p id="u0242e324">
     在以前，我们使用Jquery，zepto等库来操作DOM，之后在vue，Angular，React等框架出现后，我们通过操作数据来控制DOM（绝大多数时候），越来越少的去直接操作DOM
    </p>
    <p id="u5bde0e2e">
     但这并不代表原生操作不重要。相反，DOM操作才能有助于我们理解框架深层的内容
    </p>
    <p id="ua7c2f26b">
     下面就来分析DOM常见的操作，主要分为：
    </p>
    <ul>
     <li id="u1e766166">
      创建节点
     </li>
     <li id="u566d949c">
      查询节点
     </li>
     <li id="u36238f02">
      更新节点
     </li>
     <li id="ub8c23937">
      添加节点
     </li>
     <li id="u59f6d02c">
      删除节点
     </li>
    </ul>
    <h4 id="jTcQ1">
     创建节点
    </h4>
    <h5 id="Pkw1J">
     createElement
    </h5>
    <p id="u4b8153c2">
     创建新元素，接受一个参数，即要创建元素的标签名1
    </p>
    <pre id="vYAMp"><code>const divEl = document.createElement("div");</code></pre>
    <h5 id="Ogwo5">
     <br/>
     createTextNode
    </h5>
    <p id="ub33c0939">
     创建一个文本节点
    </p>
    <pre id="R9CXW"><code>const textEl = document.createTextNode("content");</code></pre>
    <h5 id="Zuw0T">
     createDocumentFragment
    </h5>
    <p id="ua4a27fde">
     用来创建一个文档碎片，它表示一种轻量级的文档，主要是用来存储临时节点，然后把文档碎片的内容一次性添加到DOM中
    </p>
    <pre id="zlLvg"><code>const fragment = document.createDocumentFragment();</code></pre>
    <p id="u0f2fe8d5">
     当请求把一个DocumentFragment 节点插入文档树时，插入的不是 DocumentFragment自身，而是它的所有子孙节点
    </p>
    <h5 id="w2r0Y">
     createAttribute
    </h5>
    <p id="uecb6d718">
     创建属性节点，可以是自定义属性
    </p>
    <pre id="gBAr2"><code>const dataAttribute = document.createAttribute('custom');
consle.log(dataAttribute);</code></pre>
    <h4 id="TxfWc">
     获取节点
    </h4>
    <h5 id="FZFKd">
     querySelector
    </h5>
    <p id="uc45bb3f7">
     传入任何有效的css 选择器，即可选中单个 DOM元素（首个）：
    </p>
    <pre id="x8I3j"><code>document.querySelector('.element')
document.querySelector('#element')
document.querySelector('div')
document.querySelector('[name="username"]')
document.querySelector('div + p &gt; span')</code></pre>
    <p id="uef6ff25b">
     如果页面上没有指定的元素时，返回 null
    </p>
    <h5 id="QvZMl">
     querySelectorAll
    </h5>
    <p id="ue325e99b">
     返回一个包含节点子树内所有与之相匹配的Element节点列表，如果没有相匹配的，则返回一个空节点列表
    </p>
    <pre id="rfOkW"><code>const notLive = document.querySelectorAll("p");</code></pre>
    <p id="u184caa1e">
     需要注意的是，该方法返回的是一个 NodeList的静态实例，它是一个静态的“快照”，而非“实时”的查询
    </p>
    <p id="u30b85565">
     关于获取DOM元素的方法还有如下，就不一一述说
    </p>
    <pre id="VUI1b"><code>document.getElementById('id属性值');返回拥有指定id的对象的引用
document.getElementsByClassName('class属性值');返回拥有指定class的对象集合
document.getElementsByTagName('标签名');返回拥有指定标签名的对象集合
document.getElementsByName('name属性值'); 返回拥有指定名称的对象结合
document/element.querySelector('CSS选择器');  仅返回第一个匹配的元素
document/element.querySelectorAll('CSS选择器');   返回所有匹配的元素
document.documentElement;  获取页面中的HTML标签
document.body; 获取页面中的BODY标签
document.all[''];  获取页面中的所有元素节点的对象集合型</code></pre>
    <p id="u8fb72cc8">
     除此之外，每个DOM元素还有parentNode、childNodes、firstChild、lastChild、nextSibling、previousSibling属性，关系图如下图所示
    </p>
    <p class="img-center">
     <img alt="" height="409" id="u49603e10" src="https://i-blog.csdnimg.cn/img_convert/953cd6673b501d23f6dc5c4a8761971e.png" width="775"/>
    </p>
    <h4 id="e4f4y">
     更新节点
    </h4>
    <h5 id="VA6sl">
     innerHTML
    </h5>
    <p id="ubb0698a9">
     不但可以修改一个DOM节点的文本内容，还可以直接通过HTML片段修改DOM节点内部的子树
    </p>
    <pre id="fsrsc"><code>// 获取&lt;p id="p"&gt;...&lt;/p &gt;
var p = document.getElementById('p');
// 设置文本为abc:
p.innerHTML = 'ABC'; // &lt;p id="p"&gt;ABC&lt;/p &gt;
// 设置HTML:
p.innerHTML = 'ABC &lt;span style="color:red"&gt;RED&lt;/span&gt; XYZ';
// &lt;p&gt;...&lt;/p &gt;的内部结构已修改</code></pre>
    <h5 id="y00Ho">
     innerText、textContent
    </h5>
    <p id="u33c490e3">
     自动对字符串进行HTML编码，保证无法设置任何HTML标签
    </p>
    <pre id="Ca6Zs"><code>// 获取&lt;p id="p-id"&gt;...&lt;/p &gt;
var p = document.getElementById('p-id');
// 设置文本:
p.innerText = '&lt;script&gt;alert("Hi")&lt;/script&gt;';
// HTML被自动编码，无法设置一个&lt;script&gt;节点:
// &lt;p id="p-id"&gt;&amp;lt;script&amp;gt;alert("Hi")&amp;lt;/script&amp;gt;&lt;/p &gt;</code></pre>
    <p id="uc4a3ca39">
     两者的区别在于读取属性时，innerText不返回隐藏元素的文本，而textContent返回所有文本
    </p>
    <p id="uab99c70d">
    </p>
    <h5 id="fbeh1">
     style
    </h5>
    <p id="u73262b0b">
     DOM节点的style属性对应所有的CSS，可以直接获取或设置。遇到-需要转化为驼峰命名
    </p>
    <pre id="KVwgK"><code>// 获取&lt;p id="p-id"&gt;...&lt;/p &gt;
const p = document.getElementById('p-id');
// 设置CSS:
p.style.color = '#ff0000';
p.style.fontSize = '20px'; // 驼峰命名
p.style.paddingTop = '2em';</code></pre>
    <h4 id="L0tFL">
     添加节点
    </h4>
    <h5 id="jQlaf">
     innerHTML
    </h5>
    <p id="u3611c1ca">
     如果这个DOM节点是空的，例如，&lt;div&gt;&lt;/div&gt;，那么，直接使用innerHTML = '&lt;span&gt;child&lt;/span&gt;'就可以修改DOM节点的内容，相当于添加了新的DOM节点
    </p>
    <p id="u7da9ebb4">
     如果这个DOM节点不是空的，那就不能这么做，因为innerHTML会直接替换掉原来的所有子节点
    </p>
    <h5 id="CiMzR">
     appendChild
    </h5>
    <p id="u122c8e4b">
     把一个子节点添加到父节点的最后一个子节点
    </p>
    <p id="u1bb5ea05">
     举个例子
    </p>
    <pre id="BB0Ix"><code>&lt;!-- HTML结构 --&gt;
&lt;p id="js"&gt;JavaScript&lt;/p &gt;
&lt;div id="list"&gt;
    &lt;p id="java"&gt;Java&lt;/p &gt;
    &lt;p id="python"&gt;Python&lt;/p &gt;
    &lt;p id="scheme"&gt;Scheme&lt;/p &gt;
&lt;/div&gt;</code></pre>
    <p id="u6e0f531a">
     添加一个p元素
    </p>
    <pre id="X1Myt"><code>const js = document.getElementById('js')
js.innerHTML = "JavaScript"
const list = document.getElementById('list');
list.appendChild(js);</code></pre>
    <p id="u0fd522d2">
     现在HTML结构变成了下面
    </p>
    <pre id="uvdfP"><code>&lt;!-- HTML结构 --&gt;
&lt;div id="list"&gt;
    &lt;p id="java"&gt;Java&lt;/p &gt;
    &lt;p id="python"&gt;Python&lt;/p &gt;
    &lt;p id="scheme"&gt;Scheme&lt;/p &gt;
    &lt;p id="js"&gt;JavaScript&lt;/p &gt;  &lt;!-- 添加元素 --&gt;
&lt;/div&gt;</code></pre>
    <p id="u622040d8">

    <h2>
     15.什么是重绘，什么是回流？如何减少回流？
    </h2>
    <h2>
     重绘(Repaint)：
    </h2>
    <p>
     重绘是指当元素样式发生改变，但不影响布局的情况下，浏览器重新绘制元素的过程。例如，修改元素的背景颜色、字体颜色等。
    </p>
    <h2>
     回流(Reflow)：
    </h2>
    <p>
     回流是指当元素的布局属性发生改变，需要重新计算元素在页面中的布局位置时，浏览器重新进行布局的过程。例如，修改元素的宽度和高度，定位等
    </p>
    <p>
     回流的成本比重绘高得多，因为回流涉及重新计算元素的几何属性和页面布局。而重绘只需要重新绘制已经计算好元素的样式。
    </p>
    <p>
    </p>
    <h2>
     如何减少：
    </h2>
    <ol>
     <li>
      <strong>
       使用visibility：hidden替代display：none
      </strong>
      ：visibility：hidden不会触发回流，因为元素隐藏之后，还占据页面空间。而使用display：none元素隐藏之后，不占用页面空间，会将元素从渲染树中移除，从而引起回流
     </li>
     <li>
      <strong>
       避免频繁操作元素的布局样式属性
      </strong>
      ：当需要对元素进行多次样式修改时，可以考虑将这些修改合并为一次操作。
     </li>
     <li>
      <strong>
       让元素脱离文档流
      </strong>
      ：position：absolute/position：fixed/float：left（只是减少回流，不是避免回流）
     </li>
    </ol>
