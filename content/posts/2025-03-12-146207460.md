    <h3>
     <a id="Horizontal_Pod_Autoscaler_HPA_4">
     </a>
     一、Horizontal Pod Autoscaler (HPA)
    </h3>
    <h4>
     <a id="1__Metrics_Server_5">
     </a>
     1. 前提条件：安装 Metrics Server
    </h4>
    <p>
     HPA 依赖资源指标（如 CPU/内存），需先安装 Metrics Server：
    </p>
    <pre><code class="prism language-bash">kubectl apply <span class="token parameter variable">-f</span> https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml
</code></pre>
    <h4>

    <h3>
     <a id="Vertical_Pod_Autoscaler_VPA_96">
     </a>
     二、Vertical Pod Autoscaler (VPA)
    </h3>
    <h4>
     <a id="1__VPA_97">
     </a>
     1. 前提条件：安装 VPA
    </h4>
    <p>
     VPA 需要独立组件，安装步骤如下：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 克隆 VPA 仓库</span>
<span class="token function">git</span> clone https://github.com/kubernetes/autoscaler.git
<span class="token builtin class-name">cd</span> autoscaler/vertical-pod-autoscaler

<span class="token comment"># 安装 VPA 组件</span>
./hack/vpa-up.sh
</code></pre>
    <h4>
     <a id="2__VPA__108">
     </a>
     2. 创建 VPA 规则
    </h4>
    <pre><code class="prism language-yaml"><span class="token comment"># vpa.yaml</span>
<span class="token key atrule">apiVersion</span><span class="token punctuation">:</span> autoscaling.k8s.io/v1
<span class="token key atrule">kind</span><span class="token punctuation">:</span> VerticalPodAutoscaler
<span class="token key atrule">metadata</span><span class="token punctuation">:</span>
  <span class="token key atrule">name</span><span class="token punctuation">:</span> nginx<span class="token punctuation">-</span>vpa
<span class="token key atrule">spec</span><span class="token punctuation">:</span>
  <span class="token key atrule">targetRef</span><span class="token punctuation">:</span>
    <span class="token key atrule">apiVersion</span><span class="token punctuation">:</span> <span class="token string">"apps/v1"</span>
    <span class="token key atrule">kind</span><span class="token punctuation">:</span> Deployment
    <span class="token key atrule">name</span><span class="token punctuation">:</span> nginx<span class="token punctuation">-</span>deployment
  <span class="token key atrule">updatePolicy</span><span class="token punctuation">:</span>
    <span class="token key atrule">updateMode</span><span class="token punctuation">:</span> <span class="token string">"Auto"</span>  <span class="token comment"># 模式：Auto/Off/Initial</span>
  <span class="token key atrule">resourcePolicy</span><span class="token punctuation">:</span>
    <span class="token key atrule">containerPolicies</span><span class="token punctuation">:</span>
    <span class="token punctuation">-</span> <span class="token key atrule">containerName</span><span class="token punctuation">:</span> <span class="token string">"*"</span>  <span class="token comment"># 匹配所有容器</span>
      <span class="token key atrule">minAllowed</span><span class="token punctuation">:</span>
        <span class="token key atrule">cpu</span><span class="token punctuation">:</span> <span class="token string">"50m"</span>
        <span class="token key atrule">memory</span><span class="token punctuation">:</span> <span class="token string">"64Mi"</span>
      <span class="token key atrule">maxAllowed</span><span class="token punctuation">:</span>
        <span class="token key atrule">cpu</span><span class="token punctuation">:</span> <span class="token string">"500m"</span>
        <span class="token key atrule">memory</span><span class="token punctuation">:</span> <span class="token string">"1Gi"</span>
</code></pre>
    <h5>
     <a id="_133">
     </a>
     参数说明：
    </h5>
    <ul>
     <li>
      <strong>
       <code>
        updateMode
       </code>
      </strong>
      ：
      <ul>
       <li>
        <code>
         Auto
        </code>
        ：自动调整 Pod 资源请求并重启 Pod。
       </li>
       <li>
        <code>
         Initial
        </code>
        ：仅调整新 Pod 的资源请求。
       </li>
       <li>
        <code>
         Off
        </code>
        ：仅提供建议，不自动执行。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        minAllowed
       </code>
       /
       <code>
        maxAllowed
       </code>
      </strong>
      ：资源调整的上下限。
     </li>
    </ul>
    <h4>
     <a id="3__VPA_140">
     </a>
     3. 验证 VPA
    </h4>
    <pre><code class="prism language-bash">kubectl describe vpa nginx-vpa  <span class="token comment"># 查看 VPA 建议</span>
kubectl get pods <span class="token parameter variable">-l</span> <span class="token assign-left variable">app</span><span class="token operator">=</span>nginx   <span class="token comment"># 观察 Pod 是否重启（Auto 模式下）</span>
</code></pre>
    <hr/>
    <h3>
     <a id="HPA__VPA__148">
     </a>
     三、HPA + VPA 结合使用注意事项
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        优先级问题
       </strong>
       ：
      </p>
      <ul>
       <li>
        HPA 和 VPA 同时作用于同一个 Deployment 时，可能因资源调整冲突导致不稳定。
       </li>
       <li>
        建议优先使用 HPA，VPA 仅在资源请求明显不合理时启用。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        VPA 的限制
       </strong>
       ：
      </p>
      <ul>
       <li>
        VPA 的
        <code>
         Auto
        </code>
        模式会重启 Pod，可能影响服务可用性。
       </li>
       <li>
        不支持 StatefulSet 的所有场景（需谨慎使用）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        资源请求定义
       </strong>
       ：
      </p>
      <ul>
       <li>
        即使使用 VPA，部署时仍需定义初始
        <code>
         resources.requests
        </code>
        ，否则 Pod 可能无法调度。
       </li>
      </ul>
     </li>
    </ol>

    <hr/>
    <h3>
     <a id="_180">
     </a>
     五、常见问题
    </h3>
    <h4>
     <a id="1_HPA__181">
     </a>
     1. HPA 不生效
    </h4>
    <ul>
     <li>
      <strong>
       检查 Metrics Server
      </strong>
      ：
      <pre><code class="prism language-bash">kubectl <span class="token function">top</span> pods  <span class="token comment"># 若无数据，说明 Metrics Server 未正常工作</span>
</code></pre>
     </li>
     <li>
      <strong>
       确认资源请求
      </strong>
      ：确保 Deployment 中定义了
      <code>
       resources.requests
      </code>
      。
     </li>
    </ul>
    <h4>
     <a id="2_VPA__188">
     </a>
     2. VPA 未调整资源
    </h4>
    <ul>
     <li>
      <strong>
       查看 VPA 建议
      </strong>
      ：
      <pre><code class="prism language-bash">kubectl describe vpa <span class="token operator">&lt;</span>vpa-name<span class="token operator">&gt;</span>
</code></pre>
     </li>
     <li>
      <strong>
       检查模式
      </strong>
      ：确认
      <code>
       updateMode
      </code>
      设置为
      <code>
       Auto
      </code>
      。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_197">
     </a>
     总结
    </h3>
    <ul>
     <li>
      <strong>
       HPA
      </strong>
      ：通过调整 Pod 副本数应对流量波动，适合无状态服务。
     </li>
     <li>
      <strong>
       VPA
      </strong>
      ：通过调整 Pod 资源请求优化资源利用率，适合资源需求变化大的场景。
     </li>
     <li>
      两者结合时需谨慎，避免资源调整冲突。
     </li>
    </ul>
