    <h4>
     <a id="1__62">
     </a>
     1. 简介
    </h4>
    <p>
     Nodemailer是Node.js平台上最流行的电子邮件发送库，它提供了一个简单而强大的解决方案，支持Unicode内容、HTML内容、附件、图片嵌入等功能。
    </p>
    <h4>
     <a id="2__66">
     </a>
     2. 安装
    </h4>
    <p>
     首先需要安装Nodemailer包：
    </p>
    <pre><code>npm install nodemailer
</code></pre>
    <h4>
     <a id="3__72">
     </a>
     3. 基本配置
    </h4>
    <h5>
     <a id="31__74">
     </a>
     3.1 创建传输器
    </h5>
    <p>
     首先需要创建一个传输器(transporter)，这是发送邮件的核心配置：
    </p>
    <pre><code>const nodemailer = require("nodemailer");

const transporter = nodemailer.createTransport({
  host: "smtp.example.com",     // SMTP服务器地址
  port: 465,                    // SMTP端口
  secure: true,                 // 使用SSL/TLS
  auth: {
    user: "your-email@example.com",  // 邮箱账号
    pass: "your-password"            // 邮箱密码或应用专用密码
  }
});
</code></pre>
    <h5>
     <a id="32__90">
     </a>
     3.2 配置说明
    </h5>
    <p>
     host: SMTP服务器地址
    </p>
    <p>
     port: 常用端口
    </p>
    <ul>
     <li>
      465 (SSL/TLS)
     </li>
     <li>
      587 (STARTTLS)
     </li>
    </ul>
    <p>
     secure:
    </p>
    <ul>
     <li>
      true: 使用465端口
     </li>
     <li>
      false: 使用其他端口
     </li>
    </ul>
    <p>
     auth: 认证信息
    </p>
    <h4>
     <a id="4__106">
     </a>
     4. 发送邮件
    </h4>
    <h5>
     <a id="41__108">
     </a>
     4.1 基本发送示例
    </h5>
    <pre><code>async function sendEmail(to, subject, content) {
  const mailOptions = {
    from: '"发送者名称" &lt;sender@example.com&gt;',  // 发件人
    to: to,                                     // 收件人
    subject: subject,                           // 主题
    text: content,                             // 纯文本内容
    html: `&lt;b&gt;${content}&lt;/b&gt;`                  // HTML内容

  try {
    const info = await transporter.sendMail(mailOptions);
    console.log("邮件发送成功:", info.messageId);
    return info;
  } catch (error) {
    console.error("邮件发送失败:", error);
    throw error;
  }
}
</code></pre>
    <h5>
     <a id="42__129">
     </a>
     4.2 发送验证码示例

    <p>
     以下是一个完整的发送验证码的实现：
    </p>
    <pre><code>const nodemailer = require("nodemailer");

const transporter = nodemailer.createTransport({
  host: MAIL_HOST,    
  port: MAIL_PORT,

const sendMail = async (to, subject, html) =&gt; {
  const mailOptions = {
    from: MAIL_USER,  // 发送者

    html,           // HTML内容
  };
  return await transporter.sendMail(mailOptions);
};
</code></pre>

     在控制器中使用：

       <!-- -->
       <br/>

       <br/>

       <br/>

       <br/>
       {
       <!-- -->
       <br/>

       <br/>

       <br/>
       }

     </li>
     <li>
      <p>

    <h3>
     <a id="SMTP_227">
     </a>
     SMTP与邮件加密协议详解
    </h3>
    <h4>
     <a id="1_SMTP_230">
     </a>
     1. SMTP简介
    </h4>
    <p>
     SMTP(Simple Mail Transfer Protocol，简单邮件传输协议) 是一个在互联网上发送电子邮件的标准协议。
    </p>
    <h5>
     <a id="11__234">
     </a>
     1.1 基本特点
    </h5>
    <ul>
     <li>
      <p>
       默认使用25端口（未加密）
      </p>
     </li>
     <li>
      <p>
       以纯文本形式传输数据
      </p>
     </li>
     <li>
      <p>
       没有加密保护，容易被截获
      </p>
     </li>
    </ul>
    <h4>
     <a id="2__242">
     </a>
     2. 加密协议对比
    </h4>
    <h5>
     <a id="21_SSLTLS_465_244">
     </a>
     2.1 SSL/TLS (端口465)
    </h5>
    <pre><code>const transporter = nodemailer.createTransport({
  host: "smtp.example.com",
  port: 465,
  secure: true,  // 启用SSL/TLS
  auth: {
    user: "your-email@example.com",
    pass: "your-password"
  }
});
</code></pre>
    <p>
     特点：
    </p>
    <ul>
     <li>
      <p>
       全程加密通信
      </p>
     </li>
     <li>
      <p>
       连接建立时就启用加密
      </p>
     </li>
     <li>
      <p>
       更安全，是推荐的加密方式
      </p>
     </li>
     <li>
      <p>
       适用于现代邮件系统
      </p>
     </li>
    </ul>
    <p>
     工作流程：
    </p>
    <ul>
     <li>
      <p>
       客户端连接到服务器的465端口
      </p>
     </li>
     <li>
      <p>
       立即建立SSL/TLS加密通道
      </p>
     </li>
     <li>
      <p>
       所有后续通信都在加密通道中进行
      </p>
     </li>
    </ul>
    <h5>
     <a id="22STARTTLS_587_274">
     </a>
     2.2STARTTLS (端口587)
    </h5>
    <pre><code>const transporter = nodemailer.createTransport({
  host: "smtp.example.com",
  port: 587,
  secure: false,  // 使用STARTTLS
  auth: {
    user: "your-email@example.com",
    pass: "your-password"
  }
});
</code></pre>
    <p>
     特点：
    </p>
    <ul>
     <li>
      <p>
       先以明文方式建立连接
      </p>
     </li>
     <li>
      <p>
       通过STARTTLS命令升级为加密连接
      </p>
     </li>
     <li>
      <p>
       向后兼容性更好
      </p>
     </li>
     <li>
      <p>
       适用于一些旧的邮件系统
      </p>
     </li>
    </ul>
    <p>
     工作流程：
    </p>
    <p>
     1. 客户端以明文方式连接到587端口
    </p>
    <ul>
     <li>
      <p>
       服务器告知支持STARTTLS
      </p>
     </li>
     <li>
      <p>
       客户端发送STARTTLS命令
      </p>
     </li>
     <li>
      <p>
       连接升级为加密连接
      </p>
     </li>
     <li>
      <p>
       继续后续通信
      </p>
     </li>
    </ul>
    <h4>
     <a id="3__308">
     </a>
     3. 端口使用建议
    </h4>
    <h5>
     <a id="31__310">
     </a>
     3.1 常用端口对比
    </h5>
    <p>
     | 端口 | 协议| 加密方式 | 建议 |
    </p>
    <p>
     |------|------|----------|------|
    </p>
    <p>
     | 25 | SMTP | 无加密 | 不推荐使用|
    </p>
    <p>
     | 465 | SMTPS | SSL/TLS | 推荐使用|
    </p>
    <p>
     | 587 | SMTP | STARTTLS| 备选方案 |
    </p>
    <h5>
     <a id="32__322">
     </a>
     3.2 选择建议
    </h5>
    <pre><code>// 推荐配置（使用465端口）
const secureConfig = {
  host: "smtp.example.com",
  port: 465,
  secure: true,
  auth: {
    user: "your-email@example.com",
    pass: "your-password"
  }
};

// 备选配置（使用587端口）
const starttlsConfig = {
  host: "smtp.example.com",
  port: 587,
  secure: false,
  requireTLS: true,  // 强制要求TLS
  auth: {
    user: "your-email@example.com",
    pass: "your-password"
  }
};
</code></pre>
    <h4>
     <a id="4__347">
     </a>
     4. 安全性考虑
    </h4>
    <h5>
     <a id="41SSLTLS_465_349">
     </a>
     4.1SSL/TLS (465端口)优势
    </h5>
    <ul>
     <li>
      <p>
       全程加密，无明文传输风险
      </p>
     </li>
     <li>
      <p>
       连接建立更快
      </p>
     </li>
     <li>
      <p>
       现代邮件服务器普遍支持
      </p>
     </li>
     <li>
      <p>
       不存在降级攻击风险
      </p>
     </li>
    </ul>
    <h5>
     <a id="42_STARTTLS_587_359">
     </a>
     4.2 STARTTLS (587端口)注意事项
    </h5>
    <ul>
     <li>
      <p>
       初始连接为明文
      </p>
     </li>
     <li>
      <p>
       可能受到降级攻击
      </p>
     </li>
     <li>
      <p>
       需要额外的STARTTLS协商过程
      </p>
     </li>
     <li>
      <p>
       配置时建议启用requireTLS选项
      </p>
      <p>
       // STARTTLS的安全配置
       <br/>
       const secureStartTLS = {
       <!-- -->
       <br/>
       host: “smtp.example.com”,
       <br/>
       port: 587,
       <br/>
       secure: false,
       <br/>
       requireTLS: true,
       <br/>
       tls: {
       <!-- -->
       <br/>
       rejectUnauthorized: true, // 验证服务器证书
       <br/>

       <br/>
       }
       <br/>
       };
