    <p>
     学习 Kubernetes（K8s）需要循序渐进，结合理论知识和实践操作。以下是学习 Kubernetes 的推荐步骤：
    </p>
    <hr/>
    <h4>
     <a id="1__6">
     </a>
     <strong>
      1. 先决条件
     </strong>
    </h4>
    <p>
     •
     <strong>
      掌握容器基础
     </strong>
     ：先学习 Docker，理解容器化概念（镜像、容器、仓库）、Dockerfile 编写和容器生命周期管理。
     <br/>
     •
     <strong>
      熟悉 Linux 基础
     </strong>
     ：了解 Linux 命令行操作、网络、文件系统等。
     <br/>
     •
     <strong>
      了解云计算概念
     </strong>
     ：如虚拟化、负载均衡、服务发现、分布式系统等。
    </p>
    <hr/>
    <h4>
     <a id="2_Kubernetes__13">
     </a>
     <strong>
      2. Kubernetes 核心概念
     </strong>
    </h4>
    <p>
     •
     <strong>
      基础概念
     </strong>
     ：
     <br/>
     •
     <strong>
      Pod
     </strong>
     ：最小的调度单位，包含一个或多个容器。
     <br/>
     •
     <strong>
      Deployment
     </strong>
     ：管理 Pod 的副本和滚动更新。
     <br/>
     •
     <strong>
      Service
     </strong>
     ：提供 Pod 的网络访问和负载均衡。
     <br/>
     •
     <strong>
      Namespace
     </strong>
     ：资源隔离的逻辑分组。
     <br/>
     •
     <strong>
      ConfigMap &amp; Secret
     </strong>
     ：管理应用配置和敏感信息。
     <br/>
     •
     <strong>
      PersistentVolume (PV) &amp; PersistentVolumeClaim (PVC)
     </strong>
     ：持久化存储管理。
     <br/>
     •
     <strong>
      核心组件
     </strong>
     ：
     <br/>
     •
     <strong>
      控制平面（Control Plane）
     </strong>
     ：包括 API Server、Scheduler、Controller Manager、etcd。
     <br/>
     •
     <strong>
      工作节点（Node）
     </strong>
     ：包括 Kubelet、Kube-proxy、容器运行时（如 containerd）。
    </p>
    <hr/>
    <h4>
     <a id="3__27">
     </a>
     <strong>
      3. 安装和配置集群
     </strong>
    </h4>
    <p>
     •
     <strong>
      本地实验环境
     </strong>
     ：
     <br/>
     •
     <strong>
      Minikube
     </strong>
     ：单节点本地集群，适合快速上手。
     <br/>
     •
     <strong>
      Kind (Kubernetes in Docker)
     </strong>
     ：用 Docker 容器模拟多节点集群。
     <br/>
     •
     <strong>
      k3s
     </strong>
     ：轻量级 Kubernetes，适合边缘计算或资源有限的环境。
     <br/>
     •
     <strong>
      生产环境工具
     </strong>
     ：
     <br/>
     •
     <strong>
      kubeadm
     </strong>
     ：手动搭建集群的工具。
     <br/>
     •
     <strong>
      托管 Kubernetes 服务
     </strong>
     ：如 AWS EKS、Google GKE、Azure AKS（推荐新手使用）。
    </p>
    <hr/>
    <h4>
     <a id="4__38">
     </a>
     <strong>
      4. 基本操作
     </strong>
    </h4>
    <p>
     •
     <strong>
      学习
      <code>
       kubectl
      </code>
      命令
     </strong>
     ：
     <br/>
     • 资源管理：
     <code>
      apply
     </code>
     、
     <code>
      create
     </code>
     、
     <code>
      delete
     </code>
     、
     <code>
      get
     </code>
     、
     <code>
      describe
     </code>
     。
     <br/>
     • 调试命令：
     <code>
      logs
     </code>
     、
     <code>
      exec
     </code>
     、
     <code>
      port-forward
     </code>
     。
     <br/>
     •
     <strong>
      示例练习
     </strong>
     ：
     <br/>
     • 部署一个简单的 Nginx Pod。
     <br/>
     • 通过 Deployment 管理 Pod 副本。
     <br/>
     • 使用 Service 暴露应用。
     <br/>
     • 挂载 ConfigMap 和 Secret。
    </p>
    <hr/>
    <h4>
     <a id="5__50">
     </a>
     <strong>
      5. 深入核心功能
     </strong>
    </h4>
    <p>
     •
     <strong>
      资源管理
     </strong>
     ：
     <br/>
     • 资源限制（CPU/内存）：
     <code>
      requests
     </code>
     和
     <code>
      limits
     </code>
     。
     <br/>
     • 持久化存储：PV/PVC 动态供给。
     <br/>
     •
     <strong>
      网络与服务发现
     </strong>
     ：
     <br/>
     • 理解 Kubernetes 网络模型（Pod 网络、Service 网络）。
     <br/>
     • Service 类型：ClusterIP、NodePort、LoadBalancer。
     <br/>
     • Ingress 控制器（如 Nginx Ingress、Traefik）。
     <br/>
     •
     <strong>
      配置管理
     </strong>
     ：
     <br/>
     • 使用 ConfigMap 和 Secret 分离配置。
     <br/>
     • 环境变量与配置文件注入。
    </p>
    <hr/>
    <h4>
     <a id="6__64">
     </a>
     <strong>
      6. 进阶主题
     </strong>
    </h4>
    <p>
     •
     <strong>
      高级控制器
     </strong>
     ：
     <br/>
     •
     <strong>
      StatefulSet
     </strong>
     ：管理有状态应用（如数据库）。
     <br/>
     •
     <strong>
      DaemonSet
     </strong>
     ：在每个节点运行守护进程（如日志收集器）。
     <br/>
     •
     <strong>
      Job &amp; CronJob
     </strong>
     ：运行定时任务或一次性任务。
     <br/>
     •
     <strong>
      监控与日志
     </strong>
     ：
     <br/>
     • 使用 Prometheus + Grafana 监控集群和应用。
     <br/>
     • 日志收集方案：EFK（Elasticsearch + Fluentd + Kibana）或 Loki。
     <br/>
     •
     <strong>
      安全与权限
     </strong>
     ：
     <br/>
     • RBAC（基于角色的访问控制）。
     <br/>
     • NetworkPolicy 控制网络流量。
     <br/>
     • Pod 安全策略（Pod Security Admission）。
    </p>
    <hr/>
    <h4>
     <a id="7__DevOps_79">
     </a>
     <strong>
      7. 持续部署与 DevOps
     </strong>
    </h4>
    <p>
     •
     <strong>
      CI/CD 集成
     </strong>
     ：
     <br/>
     • 使用 Helm 管理应用包（Chart）。
     <br/>
     • GitOps 工具：Argo CD、Flux。
     <br/>
     • Jenkins Pipeline 或 GitHub Actions 自动化部署。
     <br/>
     •
     <strong>
      多环境管理
     </strong>
     ：通过 Namespace 或集群分离开发/测试/生产环境。
    </p>
    <hr/>
    <h4>
     <a id="8__88">
     </a>
     <strong>
      8. 生产环境最佳实践
     </strong>
    </h4>
    <p>
     •
     <strong>
      高可用集群设计
     </strong>
     ：
     <br/>
     • 多 Master 节点 + etcd 集群。
     <br/>
     • 跨可用区（AZ）部署。
     <br/>
     •
     <strong>
      灾备与恢复
     </strong>
     ：
     <br/>
     • 定期备份 etcd 数据。
     <br/>
     • Velero 工具实现集群资源备份。
     <br/>
     •
     <strong>
      自动扩缩容
     </strong>
     ：
     <br/>
     • Horizontal Pod Autoscaler（HPA）。
     <br/>
     • Cluster Autoscaler（节点自动扩缩容）。
     <br/>
     •
     <strong>
      资源优化
     </strong>
     ：
     <br/>
     • 合理设置资源请求和限制。
     <br/>
     • 使用 LimitRange 和 ResourceQuota 管理资源配额。
    </p>

    <h4>
     <a id="10__118">
     </a>
     <strong>
      10. 认证考试（可选）
     </strong>
    </h4>
    <p>
     •
     <strong>
      CKA（Certified Kubernetes Administrator）
     </strong>
     ：官方认证，考察集群管理能力。
     <br/>
     •
     <strong>
      CKAD（Certified Kubernetes Application Developer）
     </strong>
     ：聚焦应用部署和调试。
     <br/>
     •
     <strong>
      备考资源
     </strong>
     ：Killer.sh 模拟器、官方考试指南。
    </p>
    <hr/>
    <h4>
     <a id="11__125">
     </a>
     <strong>
      11. 持续学习与实践
     </strong>
    </h4>
    <p>
     •
     <strong>
      参与社区
     </strong>
     ：
     <br/>
     • 关注 Kubernetes 官方博客和 GitHub。
     <br/>
     • 参与 Meetup 或 KubeCon 大会。
     <br/>
     •
     <strong>
      实战项目
     </strong>
     ：
     <br/>
     • 部署一个完整的微服务应用（如博客系统、电商平台）。
     <br/>
     • 参与开源 Kubernetes 生态项目（如 Prometheus、Istio）。
    </p>
    <hr/>
    <p>
     通过以上步骤，你可以逐步掌握 Kubernetes 的核心概念和实战技能。关键是多动手实践，结合真实场景解决问题！
    </p>
