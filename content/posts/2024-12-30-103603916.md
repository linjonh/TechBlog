    <h4>
     <a id="__rpx__1">
     </a>
     一、什么是
     <code>
      rpx
     </code>
     尺寸单位
    </h4>
    <blockquote>
     <p>
      <code>
       rpx
      </code>
      ： 是微信小程序独有的，解决屏幕自适应的尺寸单位
     </p>
    </blockquote>
    <ul>
     <li>
      可以根据屏幕宽度进行自适应，不论屏幕大小，规定屏幕宽为
      <code>
       750rpx
      </code>
     </li>
     <li>
      通过
      <code>
       rpx
      </code>
      设置元素和字体的大小，小程序在
      <strong>
       不同尺寸
      </strong>
      的屏幕上，
      <strong>
       可以实现自动适配
      </strong>
     </li>
    </ul>
    <h4>

     <code>
      px
     </code>
     之间的换算

      iPhone6

      iPhone6

      <code>
       1 物理像素
      </code>
     </strong>
     ，
     <strong>
      因为设计师的图一般是二倍图，所以在iphone6下直接使用测量出来的值来设置rpx就可以了

    <h4>
     <a id="_41">
     </a>
     四、全局样式和局部样式
    </h4>
    <h5>
     <a id="_42">
     </a>
     全局样式
    </h5>
    <ul>
     <li>
      定义在
      <code>
       app.wxss
      </code>
      中的样式为全局样式，作用于每一个页面。
     </li>
    </ul>
    <h5>
     <a id="_46">
     </a>
     局部样式
    </h5>
    <ul>
     <li>
      在
      <code>
       page
      </code>
      的
      <code>
       wxss
      </code>
      文件中定义的样式为局部样式，只作用在对应的页面，并会覆盖
      <code>
       app.wxss
      </code>
      中相同的选择器。
     </li>
    </ul>
    <p>
     <strong>
      注意：当局部样式的权重大于或等于全局样式的权重时，才会覆盖全局的样式效果！
     </strong>
    </p>
    <h4>
     <a id="appjson__52">
     </a>
     五、
     <code>
      app.json
     </code>
     配置文件
    </h4>
    <blockquote>
     <p>
      小程序根目录下的
      <code>
       app.json
      </code>
      文件用来对微信小程序进行全局配置，
     </p>
    </blockquote>
    <blockquote>
     <p>
      它决定了页面文件的路径、窗口表现、设置网络超时时间、设置多 tab 等。
     </p>
    </blockquote>
    <p>
     在
     <code>
      app.json
     </code>
     配置文件中，最主要的配置节点是：
    </p>
    <ul>
     <li>
      <code>
       pages
      </code>
      数组：配置小程序的页面路径
     </li>
     <li>
      <code>
       window
      </code>
      对象：用于设置小程序的状态栏、导航条、标题、窗口背景色
     </li>
     <li>
      <code>
       tabBar
      </code>
      对象：配置小程序的tab栏效果
     </li>
    </ul>
    <h4>
     <a id="pages__63">
     </a>
     六、
     <code>
      pages
     </code>
     数组的用法
    </h4>
    <blockquote>
     <p>
      <code>
       app.json
      </code>
      中的
      <code>
       pages
      </code>
      数组，用来配置小程序的页面路径
     </p>
    </blockquote>
    <h5>
     <a id="_66">
     </a>
     基础配置
    </h5>
    <ul>
     <li>
      <code>
       pages
      </code>
      用于指定小程序由哪些页面组成，每一项都对应一个页面的 路径+文件名 信息。
     </li>
     <li>
      文件名不需要写文件后缀，框架会自动去寻找对应位置的
      <code>
       .json
      </code>
      、
      <code>
       .js
      </code>
      、
      <code>
       .wxml
      </code>
      和
      <code>
       .wxss
      </code>
      四个文件进行处理。
     </li>
    </ul>
    <h5>
     <a id="_71">
     </a>
     创建页面的另一种方式
    </h5>
    <ul>
     <li>
      打开
      <code>
       app.json
      </code>
      --&gt;
      <code>
       pages
      </code>
      数组节点 --&gt; 新增页面路径并保存 --&gt; 自动创建路径对应的页面
     </li>
    </ul>
    <h5>
     <a id="_75">
     </a>
     设置项目的首页
    </h5>
    <ul>
     <li>
      打开
      <code>
       app.json
      </code>
      -&gt;
      <code>
       pages
      </code>
      数组节点
     </li>
     <li>
      按需调整数组中路径的顺序，即可修改默认首页
     </li>
    </ul>
    <p>
     <strong>
      注意事项：
     </strong>
    </p>
    <ul>
     <li>
      数组的
      <code>
       第一项
      </code>
      代表小程序的小程序的初始页面也就是
      <code>
       首页
      </code>
     </li>
     <li>
      小程序中新增/减少页面，都需要对 pages 数组进行修改，否则在运行小程序时就会报错
     </li>
    </ul>
    <h4>
     <a id="tabBar__85">
     </a>
     七、
     <code>
      tabBar
     </code>
     的概念
    </h4>
    <blockquote>
     <p>
      <code>
       tabBar
      </code>
      是移动端应用常见的页面效果，用于实现多页面的快速切换；小程序中通常将其分为底部
      <code>
       tabBar
      </code>
      和顶部
      <code>
       tabBar
      </code>
     </p>
    </blockquote>
    <p>
     官方文档：
     <a href="https://developers.weixin.qq.com/miniprogram/dev/reference/configuration/app.html#tabBar" rel="nofollow">
      tabBar
     </a>
    </p>
    <p>
     <strong>
      注意：
      <code>
       tabBar
      </code>
      中，只能配置最少2个、最多5个 tab 页签，当渲染顶部
      <code>
       tabBar
      </code>
      的时候，不显示
      <code>
       icon
      </code>
      ，只显示文本
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       都不支持网络图片
      </strong>
     </li>
     <li>
      <strong>
       当 position 为 top 时，不显示 icon。
      </strong>
      <br/>
      <code>
       tabBar的路径要放到pages最前面，防止加载首页时无法出现tabBar的情况
      </code>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7afbf3032731c423330585af9dd7c40b.png#pic_center"/>
     </li>
    </ul>

    <h5>
     <a id="_96">

    <h4>
     <a id="_105">
     </a>
     九、生命周期的概念
    </h4>
    <blockquote>
     <p>
      生命周期（Life Cycle）是指一个对象从 创建 -&gt; 运行 -&gt; 销毁 的整个阶段，强调的是一个时间段。
     </p>
    </blockquote>
    <h5>
     <a id="_108">
     </a>
     小程序的生命周期
    </h5>
    <ul>
     <li>
      小程序的启动，表示生命周期的开始
     </li>
     <li>
      小程序的关闭，表示生命周期的结束
     </li>
     <li>
      中间小程序运行的过程，就是小程序的生命周期
     </li>
    </ul>
    <h5>
     <a id="_114">
     </a>
     小程序生命周期的两种类型
    </h5>
    <ul>
     <li>
      应用生命周期：特指小程序从启动 --&gt; 运行 --&gt; 销毁的过程；
     </li>
     <li>
      页面生命周期：特指小程序中，每个页面的加载 --&gt; 渲染 --&gt; 销毁的过程；
     </li>
     <li>
      <strong>
       注意：页面的生命周期范围较小，应用程序的生命周期范围较大
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e087ee763e07a101b56894bd08a0d7bb.png#pic_center"/>
     </li>
    </ul>
    <h4>
     <a id="_120">
     </a>
     十、小程序的生命周期函数
    </h4>
    <blockquote>
     <p>
      小程序框架提供的内置函数，会伴随着生命周期，自动按次序执行
     </p>
    </blockquote>
    <p>
     生命周期函数的作用：
    </p>
    <p>
     允许程序员在特定的生命周期时间点上，执行某些特定的操作。例如，页面刚加载的时候，在生命周期函数中自动发起数据请求，获取当前页面的数据；
    </p>
    <p>
     <strong>
      注意：生命周期强调的是时间段，生命周期函数强调的是时间点。
     </strong>
    </p>

     <a id="_288">
     </a>
     十四、小程序中的事件
    </h4>
    <p>
     <strong>
      官方文档：
     </strong>
     <a href="https://developers.weixin.qq.com/miniprogram/dev/framework/view/wxml/event.html" rel="nofollow">
      事件
     </a>
    </p>
    <h5>
     <a id="_290">
     </a>
     什么是事件
    </h5>
    <ul>
     <li>
      事件是视图层到逻辑层的通讯方式
     </li>
     <li>
      事件可以将用户的行为反馈到逻辑层中进行处理
     </li>
     <li>
      事件可以绑定到组件上，当组件触发事件，就会执行逻辑层中对应的事件处理函数
     </li>
     <li>
      事件对象可以携带额外信息，如：
      <code>
       id
      </code>
      、
      <code>
       dataset
      </code>
      、
      <code>
       touches
      </code>
     </li>
    </ul>
    <h5>
     <a id="bindtap__297">
     </a>
     <code>
      bindtap
     </code>
     绑定触摸事件
    </h5>
    <ul>
     <li>
      在小程序中，不存在网页中的
      <code>
       onclick
      </code>
      鼠标点击事件，而是通过
      <code>
       tap
      </code>
      事件来响应触摸行为
     </li>
     <li>
      在小程序中，可以通过
      <code>
       input
      </code>
      事件来响应文本框内的输入事件
      <ul>
       <li>
        使用
        <code>
         bindinput
        </code>
        为文本框绑定输入事件
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="data___304">
     </a>
     <code>
      data
     </code>
     和 文本框之间的数据同步
    </h5>
    <ul>
     <li>
      在文本框的
      <code>
       input
      </code>
      事件处理函数中，通过事件参数
      <code>
       event
      </code>
      ，能够访问到文本框的最新值
      <ul>
       <li>
        语法结构
        <code>
         event.detail.value
        </code>
       </li>
      </ul>
     </li>
     <li>
      通过
      <code>
       this.setData(dataObject)
      </code>
      方法，可以把页面中的
      <code>
       data
      </code>
      数据重新赋值
     </li>
    </ul>
