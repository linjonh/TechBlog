    <h2>
     <a id="HTTP_0">
     </a>
     乜嘢是HTTP
    </h2>
    <p>
     HTTP，全称为HyperText Transfer Protocol（超文本传输协议），是互联网上应用最为广泛的一种网络协议。
     <font color="“skyblue”">
      <strong>
       它允许将超文本标记语言（HTML）文档从Web服务器传送到客户端的浏览器，并且支持多媒体内容的传输，如图片、视频等。
      </strong>
     </font>
    </p>
    <h4>
     <a id="_4">
     </a>
     基本概念
    </h4>
    <ul>
     <li>
      <p>
       HTTP是一种基于请求与响应的应用层协议，
       <font color="“skyblue”">
        <strong>
         它是无状态的，意味着每个请求都是独立的，服务器不会保存客户端的状态信息。
        </strong>
       </font>
      </p>
     </li>
     <li>
      <p>
       通常，用户通过客户端（如浏览器）输入URL发起请求，服务器接收到请求后进行处理，并将处理结果包装成HTTP响应消息返回给客户端，客户端再根据响应内容渲染页面展示给用户。
      </p>
     </li>
    </ul>
    <h4>
     <a id="_13">
     </a>
     工作流程
    </h4>
    <p>
     HTTP的工作流程包括四个主要步骤：
    </p>
    <ul>
     <li>
      客户端发起请求
     </li>
     <li>
      服务器处理请求
     </li>
     <li>
      服务器返回响应
     </li>
     <li>
      客户端渲染页面
     </li>
    </ul>
    <p>
     在这个过程中，客户端和服务端之间交换的消息被称为请求和响应。请求是由接受方——通常是Web浏览器——发起的，而响应则是由服务器发出的回复。
    </p>
    <h4>
     <a id="HTTP_24">
     </a>
     HTTP的特点
    </h4>
    <ul>
     <li>
      <strong>
       媒体独立
      </strong>
      ：只要客户端和服务器知道如何处理的数据内容，任何类型的数据都可以通过HTTP发送。
     </li>
     <li>
      <strong>
       无连接
      </strong>
      ：限制每次连接只处理一个请求，服务器处理完客户的请求并收到应答后即断开连接，节省传输时间。
     </li>
     <li>
      <strong>
       无状态
      </strong>
      ：对于事务处理没有记忆能力，这意味着每个请求都必须包含足够的信息让服务器理解并处理该请求。
     </li>
    </ul>
    <h4>
     <a id="HTTP_30">
     </a>
     HTTP方法
    </h4>
    <p>
     HTTP协议定义了几种不同的方法（也称作“动作”），用于以不同方式操作指定的资源。最常用的包括：
    </p>
    <ul>
     <li>
      <code>
       GET
      </code>
      ：用于获取资源。
     </li>
     <li>
      <code>
       POST
      </code>
      ：用于提交数据，比如填写表单或上传文件。
     </li>
     <li>
      <code>
       PUT
      </code>
      ：用于更新资源。
     </li>
     <li>
      <code>
       DELETE
      </code>
      ：用于删除资源。
     </li>
     <li>
      <code>
       HEAD
      </code>
      ：类似于GET，但只请求消息头，不包括消息体。
     </li>
     <li>
      <code>
       OPTIONS
      </code>
      ：查询服务器支持的方法。
     </li>
     <li>
      <code>
       TRACE
      </code>
      ：回显服务器收到的请求，主要用于测试或诊断。
     </li>
    </ul>
    <h4>
     <a id="HTTP_42">
     </a>
     HTTP请求报文
    </h4>
    <p>
     一个HTTP请求报文通常由四部分组成：
     <font color="“skyblue”">
      <strong>
       请求行、请求头部、空行以及请求数据（可选）
      </strong>
     </font>
     。
    </p>
    <ol>
     <li>
      <p>
       <strong>
        请求行
       </strong>
       （Request Line）：这是请求报文的第一行，包含了三个元素：
      </p>
      <ul>
       <li>
        请求方法（如GET、POST等），指示要执行的操作类型。
       </li>
       <li>
        请求的URL或路径，指向目标资源的位置。
       </li>
       <li>
        HTTP协议版本号，表示使用的HTTP协议版本。
       </li>
      </ul>
      <p>
       例如，
       <code>
        GET /index.html HTTP/1.1
       </code>
       表示使用GET方法请求根目录下的index.html文件，并且使用的是HTTP/1.1版本。
      </p>
     </li>
     <li>
      <p>
       <strong>
        请求头部
       </strong>
       （Headers）：紧接着请求行之后是一系列的头部字段，它们提供了关于请求的更多信息，比如客户端支持的内容类型、语言偏好、编码方式等。每个头部字段都由名称和值组成，以冒号分隔。
      </p>
     </li>
     <li>
      <p>
       <strong>
        空行
       </strong>
       （Blank Line）：在所有头部字段之后是一个空行，它标志着头部部分的结束和可能存在的请求体部分的开始。
      </p>
     </li>
     <li>
      <p>
       <strong>
        请求数据
       </strong>
       （Optional Request Body）：对于某些请求方法（如POST），这里会包含发送给服务器的数据，比如表单数据或者文件上传内容。
      </p>
     </li>
    </ol>
    <h4>
     <a id="HTTP_60">
     </a>
     HTTP响应报文
    </h4>
    <p>
     响应报文也有类似的结构，但它的起始行被称为状态行，而不是请求行。
    </p>
    <ol>
     <li>
      <p>
       <strong>
        状态行
       </strong>
       （Status Line）：包括三部分：
      </p>
      <ul>
       <li>
        HTTP协议版本号。
       </li>
       <li>
        状态码，三位数字代码，用于描述请求的结果（例如200代表成功，404代表未找到资源）。
       </li>
       <li>
        原因短语，是对状态码的文字解释。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        响应头部
       </strong>
       （Response Headers）：类似于请求头部，这些字段提供额外的信息，比如响应的日期、内容长度、内容类型等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        空行
       </strong>
       ：同样，在所有的响应头部之后有一个空行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        响应体
       </strong>
       （Response Body）：如果有的话，这部分包含了实际返回给客户端的数据，比如HTML文档、图片或者其他类型的文件。
      </p>
     </li>
    </ol>
    <h4>
     <a id="HTTP_73">
     </a>
     HTTP常见响应状态码
    </h4>
    <p>
     <font color="“skyblue”">
      <strong>
       HTTP响应状态码是服务器在接收到客户端请求后返回的一个三位数字代码，它用来表示请求的处理情况。
      </strong>
     </font>
     这些状态码分为五类，每类都有其特定的意义和用途。以下是各类常见的HTTP状态码及其含义：
    </p>
    <h5>
     <a id="1xx_77">
     </a>
     1xx（信息性状态码）
    </h5>
    <ul>
     <li>
      <strong>
       100 Continue
      </strong>
      ：客户端应当继续发送请求。
     </li>
     <li>
      <strong>
       101 Switching Protocols
      </strong>
      ：服务器已经理解了客户端的请求，并将通过Upgrade消息头通知客户端采用不同的协议来完成这个请求。
     </li>
    </ul>
    <h5>
     <a id="2xx_82">
     </a>
     2xx（成功状态码）
    </h5>
    <ul>
     <li>
      <strong>
       200 OK
      </strong>
      ：请求已成功，且请求的信息包含在响应中。
     </li>
     <li>
      <strong>
       201 Created
      </strong>
      ：请求已经被实现，而且有一个新的资源被创建。
     </li>
     <li>
      <strong>
       204 No Content
      </strong>
      ：服务器成功处理了请求，但没有返回任何内容。
     </li>
    </ul>
    <h5>
     <a id="3xx_88">
     </a>
     3xx（重定向状态码）
    </h5>
    <ul>
     <li>
      <strong>
       301 Moved Permanently
      </strong>
      ：请求的资源已被永久地移动到新URI，返回信息会包括新的URI。
     </li>
     <li>
      <strong>
       302 Found
      </strong>
      ：请求的资源现在临时从不同的URI响应请求。
     </li>
     <li>
      <strong>
       304 Not Modified
      </strong>
      ：如果客户端发送了一个条件GET请求，而该请求的资源自上次访问以来未被修改，则使用此状态码。
     </li>
    </ul>
    <h5>
     <a id="4xx_94">
     </a>
     4xx（客户端错误状态码）
    </h5>
    <ul>
     <li>
      <strong>
       400 Bad Request
      </strong>
      ：由于语法格式有误，服务器无法理解此请求。
     </li>
     <li>
      <strong>
       401 Unauthorized
      </strong>
      ：当前请求需要用户验证。
     </li>
     <li>
      <strong>
       403 Forbidden
      </strong>
      ：服务器已经理解请求，但是拒绝执行它。
     </li>
     <li>
      <strong>
       404 Not Found
      </strong>
      ：服务器找不到与请求URI相匹配的任何资源。
     </li>
    </ul>
    <h5>
     <a id="5xx_101">
     </a>
     5xx（服务端错误状态码）
    </h5>
    <ul>
     <li>
      <strong>
       500 Internal Server Error
      </strong>
      ：服务器遇到了一个未曾预料的情况，导致了其无法完成对请求的处理。
     </li>
     <li>
      <strong>
       501 Not Implemented
      </strong>
      ：服务器不支持当前请求所需要的功能。
     </li>
     <li>
      <strong>
       503 Service Unavailable
      </strong>
      ：由于临时的服务器维护或者过载，服务器当前无法处理请求。
     </li>
    </ul>
    <h2>
     <a id="URL_107">
     </a>
     <code>
      URL
     </code>
     ?
    </h2>
    <p>
     URL，全称为Uniform Resource Locator，即统一资源定位符，
     <font color="“skyblue”">
      <strong>
       是用于标识互联网上资源位置的字符串
      </strong>
     </font>
     。它允许用户通过网络访问网页、图片、视频等资源。URL是URI（统一资源标识符）的一个种类，专门用来表示资源的位置。
    </p>
    <p>
     一个标准的URL通常由以下几个部分组成：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        协议（Scheme）
       </strong>
       ：指定使用的通信协议，比如HTTP（超文本传输协议）、HTTPS（安全的超文本传输协议）、FTP（文件传输协议）等。协议后面跟有一个冒号和两个斜杠（😕/）。
      </p>
      <p>
       示例：
       <code>
        https://
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        主机名（Host）
       </strong>
       ：指明了资源所在的服务器地址，可以是域名（如www.example.com）或IP地址。
      </p>
      <p>
       示例：
       <code>
        www.example.com
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口号（Port）
       </strong>
       ：指定了与服务器连接时所使用的端口。大多数时候，如果使用的是默认端口（例如HTTP默认80，HTTPS默认443），则不需要明确写出端口号。
      </p>
      <p>
       示例：
       <code>
        :8080
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        路径（Path）
       </strong>
       ：指出请求的具体资源在服务器上的位置。它模拟了一个文件系统路径的概念。
      </p>
      <p>
       示例：
       <code>
        /path/to/resource
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        查询字符串（Query String）
       </strong>
       ：以问号（?）开始，后面跟着一系列键值对，用来向服务器传递参数。
      </p>
      <p>
       示例：
       <code>
        ?key=value&amp;foo=bar
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        片段标识符（Fragment Identifier）
       </strong>
       ：以井字号（#）开头，指向文档中的某一部分，主要用于HTML页面内部导航。
      </p>
      <p>
       示例：
       <code>
        #section1
       </code>
      </p>
     </li>
    </ol>
    <pre><code>https://www.example.com:8080/path/to/resource?key=value#section1
</code></pre>
    <p>
     在这个例子中，
     <code>
      https
     </code>
     是协议，
     <code>
      www.example.com
     </code>
     是主机名，
     <code>
      :8080
     </code>
     是端口号，
     <code>
      /path/to/resource
     </code>
     是路径，
     <code>
      ?key=value
     </code>
     是查询字符串，而
     <code>
      #section1
     </code>
     是片段标识符。
    </p>
    <h2>
     <a id="_143">
     </a>
     关于回调函数
    </h2>
    <h4>
     <a id="_145">
     </a>
     什么是回调函数？
    </h4>
    <p>
     回调函数是一种编程模式，
     <font color="“skyblue”">
      <strong>
       它允许一个函数作为参数传递给另一个函数，并在特定条件下被调用。
      </strong>
     </font>
    </p>
    <p>
     这种机制使得代码可以更加灵活和动态，因为它允许外部定义的行为在内部逻辑的关键点上被执行。
    </p>
    <h5>
     <a id="_151">
     </a>
     回调函数的主要用途包括：
    </h5>
    <ol>
     <li>
      <strong>
       处理异步操作
      </strong>
      ：在网络请求、文件读取等场景中，这些操作通常是异步进行的，即它们不会立即完成。为了获取这些操作的结果，我们可以将一个回调函数作为参数传递给负责执行该操作的函数。
      <font color="“skyblue”">
       当异步操作完成后，这个回调函数就会被调用，并传入操作的结果作为参数。
      </font>
     </li>
     <li>
      <strong>
       确保代码顺序执行
      </strong>
      ：即使在同步代码中，回调函数也可以用来
      <font color="“skyblue”">
       保证某些函数只有在另一些函数执行完毕后才会被执行。
      </font>
     </li>
     <li>
      <strong>
       事件监听器
      </strong>
      ：在前端开发中，回调函数常用于设置事件监听器。比如，当你想在用户点击按钮时执行特定代码，你可以将这段代码封装在一个函数中，并将其作为回调函数添加到按钮的点击事件上。这样，每当用户触发点击事件时，相应的回调函数就会自动执行。
     </li>
     <li>
      <strong>
       数据处理
      </strong>
      ：回调函数还可以用于数组的数据处理方法中，如
      <code>
       map()
      </code>
      、
      <code>
       filter()
      </code>
      、
      <code>
       reduce()
      </code>
      等。这些方法接受一个回调函数作为参数，该回调函数定义了如何对数组中的每个元素进行转换或其他操作。
     </li>
    </ol>

    <h2>
     <a id="AJAX_217">
     </a>
     什么是AJAX
    </h2>
    <p>
     AJAX（Asynchronous JavaScript and XML）是一种用于创建快速动态网页的技术。
    </p>
    <h4>
     <a id="_221">
     </a>
     主要特点
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        异步通信
       </strong>
       ：这是AJAX的核心特性之一。
       <font color="“skyblue”">
        它允许网页在不重新加载整个页面的情况下通过与服务器交换少量数据来更新部分页面内容。
       </font>
       这使得用户体验更加流畅，因为用户不需要等待整个页面刷新就可以看到更新的信息。
      </p>
     </li>
     <li>
      <p>
       <strong>
        提高用户体验
       </strong>
       ：
       <font color="“skyblue”">
        由于其能够局部更新网页内容，因此可以提供更快速的反馈给用户，减少了用户等待的时间。
       </font>
      </p>
     </li>
     <li>
      <p>
       <strong>
        减少带宽使用
       </strong>
       ：通过只请求和传输必要的数据而非整个页面的内容，可以有效减少服务器和客户端之间的数据传输量。
      </p>
     </li>
    </ul>
    <h4>
     <a id="_229">
     </a>
     工作流程
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        创建XMLHttpRequest对象
       </strong>
       ：这是进行异步通信的核心对象，负责发送请求并接收响应。
      </p>
     </li>
     <li>
      <p>
       <strong>
        发送请求到服务器
       </strong>
       ：可以通过GET或POST方法向服务器发送请求，传递必要的参数。
      </p>
     </li>
     <li>
      <p>
       <strong>
        服务器处理请求并返回响应
       </strong>
       ：服务器端脚本处理请求，并将结果以HTML、XML、JSON等形式返回给客户端。
      </p>
     </li>
     <li>
      <p>
       <strong>
        更新网页内容
       </strong>
       ：根据从服务器接收到的数据，使用JavaScript和DOM更新网页的部分内容。
      </p>
     </li>
    </ol>
    <h4>
     <a id="_239">
     </a>
     应用场景
    </h4>
    <ul>
     <li>
      <strong>
       实时验证表单输入
      </strong>
      ：例如检查用户名是否可用，而无需提交整个表单。
     </li>
     <li>
      <strong>
       自动完成搜索框
      </strong>
      ：当用户开始输入时，提供可能的匹配项列表。
     </li>
     <li>
      <strong>
       动态加载分页内容
      </strong>
      ：点击下一页按钮时，仅加载新页面的内容而不是刷新整个页面。
     </li>
     <li>
      <strong>
       消息通知系统
      </strong>
      ：如社交媒体中的新消息提醒，无需刷新页面即可显示新消息。
     </li>
    </ul>
    <h2>
     <a id="promise_246">
     </a>
     对，是
     <code>
      promise
     </code>
    </h2>
    <h4>
     <a id="Promise_248">
     </a>
     Promise的概念
    </h4>
    <p>
     Promise是
     <code>
      异步编程的一种解决方案
     </code>
     ,
     <strong>
      <font color="“skyblue”">
       通俗讲，
       <code>
        Promise是一个许诺、承诺
       </code>
       ,是对未来事情的承诺，承诺不一定能完成，但是无论是否能完成都会有一个结果。
      </font>
     </strong>
    </p>
    <p>
     在JavaScript中，它代表了一个异步操作的最终完成（或失败）及其结果值。一个Promise对象有三种状态：
    </p>
    <ol>
     <li>
      <strong>
       Pending（进行中）
      </strong>
      ：初始状态，既不是成功，也不是失败。
     </li>
     <li>
      <strong>
       Fulfilled（已成功）
      </strong>
      ：操作成功完成。
     </li>
     <li>
      <strong>
       Rejected（已失败）
      </strong>
      ：操作失败。
     </li>
    </ol>
    <p>
     <strong>
      <font color="“skyblue”">
       一旦一个Promise被resolved（无论是fulfilled还是rejected），它的状态就会被锁定，并且结果值或错误信息将不可改变。
      </font>
     </strong>
    </p>
    <h4>
     <a id="Promise_260">
     </a>
     Promise的优点
    </h4>
    <ol>
     <li>
      <strong>
       更好的错误处理
      </strong>
      ：使用
      <code>
       try/catch
      </code>
      机制和链式
      <code>
       .catch()
      </code>
      方法使得错误处理更加直观和易于管理。
     </li>
     <li>
      <strong>
       避免回调地狱
      </strong>
      ：相比于嵌套的回调函数，Promise允许你以更线性、更清晰的方式编写异步代码。
     </li>
     <li>
      <strong>
       链式调用
      </strong>
      ：可以方便地链接多个异步操作，每个操作都等待前一个操作完成后才开始，并能够传递数据。
     </li>
     <li>
      <strong>
       内置支持
      </strong>
      ：现代浏览器和Node.js环境对Promise提供了原生支持，无需额外库。
     </li>
    </ol>
    <h4>
     <a id="Promise_267">
     </a>
     Promise的缺点
    </h4>
    <ol>
     <li>
      <strong>
       复杂度
      </strong>
      ：虽然Promise简化了某些类型的异步编程问题，但对于初学者来说，理解Promise的状态转换和如何正确使用它们可能具有一定的挑战性。
     </li>
     <li>
      <strong>
       无法取消
      </strong>
      ：一旦创建了一个Promise，就无法取消它。这意味着如果需要停止某个异步操作，必须依赖外部逻辑来实现。
     </li>
     <li>
      <strong>
       调试困难
      </strong>
      ：尽管Promise改善了错误处理，但当出现错误时，定位到具体的出错点可能不如同步代码那样直观。
     </li>
     <li>
      <strong>
       资源管理
      </strong>
      ：在某些情况下，特别是在涉及大量并发异步操作时，确保正确的资源管理和释放可能会变得复杂。
     </li>
    </ol>

     这里，即使错误是在中间的
     <code>
      .then()
     </code>
     中发生的，也会被最后的
     <code>
      .catch()
     </code>
     捕捉到。
    </p>
    <h2>
     <a id="async__await_357">
     </a>
     <code>
      async
     </code>
     和
     <code>
      await
     </code>
    </h2>
    <p>
     在现代JavaScript中，
     <code>
      async
     </code>
     和
     <code>
      await
     </code>
     提供了一种更加简洁和直观的方式来处理异步操作。它们使得代码看起来更像同步代码，从而提高了可读性和维护性。
    </p>
    <h5>
     <a id="1_Promise___361">
     </a>
     1) Promise ==&gt; 异步操作的基础
    </h5>
    <p>
     Promise 是 JavaScript 中处理异步操作的一个重要概念，它代表了一个异步操作的最终完成（或失败）及其结果值。通过 Promise，我们可以更清晰地管理异步代码流，而不是使用传统的回调函数方式。
    </p>
    <h5>
     <a id="2_await___365">
     </a>
     2) await ==&gt; 将异步转为同步风格
    </h5>
    <ul>
     <li>
      <code>
       await
      </code>
      可以看作是
      <code>
       async wait
      </code>
      的缩写，它的主要作用是
      <font color="“skyblue”">
       <strong>
        等待一个 Promise 对象被解决（resolved），从而使异步代码看起来更加同步。
       </strong>
      </font>
     </li>
     <li>
      <strong>
       <font color="“skyblue”">
        <code>
         await
        </code>
        必须在
        <code>
         async
        </code>
        函数内部使用，不能单独存在。
       </font>
      </strong>
      它可以跟任何JavaScript表达式一起使用，但最常用于等待一个 Promise 对象。
     </li>
     <li>
      如果
      <code>
       await
      </code>
      后面跟着的是一个普通的表达式而非 Promise，则该表达式的值会被立即返回，不会造成等待。
     </li>
    </ul>
    <h5>
     <a id="3_async___371">
     </a>
     3) async ==&gt; 标识异步函数
    </h5>
    <ul>
     <li>
      当一个函数体内包含使用
      <code>
       await
      </code>
      的表达式时，这个函数必须用
      <code>
       async
      </code>
      关键字声明。这样做会自动将该函数转换为异步函数，意味着它可以执行非阻塞的操作并返回一个 Promise 对象。
     </li>
     <li>
      使用
      <code>
       async
      </code>
      定义的函数总是返回一个 Promise 对象，即使函数内部直接返回了一个普通值，该值也会被自动包装成一个已解决（fulfilled）的 Promise。
     </li>
    </ul>
    <h4>
     <a id="_376">
     </a>
     语法概览
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        定义一个异步函数
       </strong>
       ：使用
       <code>
        async function
       </code>
       关键字来声明一个异步函数。
      </p>
      <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token keyword">function</span> <span class="token function">fetchData</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
  <span class="token comment">// 异步操作...</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        等待一个Promise
       </strong>
       ：在异步函数内部，使用
       <code>
        await
       </code>
       来暂停执行直到 Promise 被解决（resolved）。注意，
       <code>
        await
       </code>
       只能在异步函数内使用。
      </p>
      <pre><code class="prism language-javascript"><span class="token keyword">const</span> data <span class="token operator">=</span> <span class="token keyword">await</span> <span class="token function">fetchData</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
     </li>
    </ul>
    <h4>
     <a id="_392">
     </a>
     实例演示
    </h4>
    <p>
     假设我们需要从API获取一些数据，并根据这些数据做进一步处理：
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token keyword">function</span> <span class="token function">fetchAndProcessData</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
  <span class="token keyword">try</span> <span class="token punctuation">{<!-- --></span>

    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span>data<span class="token punctuation">)</span><span class="token punctuation">;</span>
  <span class="token punctuation">}</span> <span class="token keyword">catch</span> <span class="token punctuation">(</span>error<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">error</span><span class="token punctuation">(</span><span class="token string">'Failed to fetch data:'</span><span class="token punctuation">,</span> error<span class="token punctuation">)</span><span class="token punctuation">;</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span>

<span class="token function">fetchAndProcessData</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

     在这个例子中，我们首先调用
     <code>
      fetch
     </code>
     函数发起网络请求。由于
     <code>
      fetch
     </code>
     返回的是一个 Promise，我们可以直接使用
     <code>
      await
     </code>
     来等待这个 Promise 完成。如果请求成功，我们会进一步解析响应体中的 JSON 数据；如果请求失败或遇到异常，会通过
     <code>
      catch
     </code>
     块捕获并处理错误。
    </p>
    <h3>
     <a id="_413">
     </a>
     注意点
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        <font color="“skyblue”">
         <code>
          await
         </code>
         只能在
         <code>
          async
         </code>
         函数中使用：尝试在非异步函数中使用
         <code>
          await
         </code>
         将会导致语法错误。
        </font>
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        错误处理
       </strong>
       ：当使用
       <code>
        await
       </code>
       时，
       <font color="“skyblue”">
        <strong>
         推荐使用
         <code>
          try...catch
         </code>
         结构来捕获可能的异常。
        </strong>
       </font>
       因为异步函数内的任何未捕获的错误都会导致返回的 Promise 被拒绝。
      </p>
     </li>
     <li>
      <p>
       <strong>
        并发执行
       </strong>
       ：如果你有多个独立的异步任务需要同时执行，可以考虑使用
       <code>
        Promise.all()
       </code>
       来等待所有任务完成。这样可以提高性能，因为它允许任务并行运行。
      </p>
      <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token keyword">function</span> <span class="token function">getData</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

  console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span>data1<span class="token punctuation">,</span> data2<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        理解返回值
       </strong>
       ：记住，
       <code>
        async
       </code>
       函数总是返回一个 Promise。即使你在函数中直接返回了一个普通值，它也会被自动包装进一个已解决的 Promise 中。
      </p>
     </li>
     <li>
      <p>
       <strong>
        调试
       </strong>
       ：由于
       <code>
        await
       </code>
       暂停了函数的执行，这可能会给调试带来一定的复杂性。确保你理解每个
       <code>
        await
       </code>
       的位置以及它如何影响代码的执行流程。
      </p>
     </li>
    </ol>

    <h2>
     <a id="%09XMLHttpRequest_434">
     </a>
     <code>
      XMLHttpRequest
     </code>
    </h2>
    <p>
     <code>
      XMLHttpRequest
     </code>
     （简称XHR）是一个用于在浏览器和服务器之间传输数据的JavaScript对象。它允许您发送请求并处理响应，而无需重新加载整个页面，从而实现动态网页内容更新。
    </p>
    <h4>
     <a id="_438">
     </a>
     概述
    </h4>
    <p>
     <code>
      XMLHttpRequest
     </code>
     最早由微软引入，作为ActiveX对象的一部分，后来被其他浏览器厂商采纳并标准化，成为了现代Web开发中处理异步数据交换的核心技术之一。
    </p>
    <h4>
     <a id="_442">
     </a>
     主要属性与方法
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        创建XHR对象
       </strong>
       ：
      </p>
      <pre><code class="prism language-javascript"><span class="token keyword">var</span> xhr <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">XMLHttpRequest</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        open(method, url, async, user, password)
       </strong>
       ：初始化一个请求。
      </p>
      <ul>
       <li>
        <code>
         method
        </code>
        ：HTTP方法（GET、POST等）。
       </li>
       <li>
        <code>
         url
        </code>
        ：请求的目标URL。
       </li>
       <li>
        <code>
         async
        </code>
        ：是否异步执行（通常是
        <code>
         true
        </code>
        ）。
       </li>
       <li>
        <code>
         user
        </code>
        ,
        <code>
         password
        </code>
        ：可选参数，用于认证。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        send(body)
       </strong>
       ：发送请求到服务器。
      </p>
      <ul>
       <li>
        对于GET请求，通常传递
        <code>
         null
        </code>
        或不传递参数。
       </li>
       <li>
        对于POST请求，传递请求体数据。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        setRequestHeader(header, value)
       </strong>
       ：设置HTTP请求头的值。
      </p>
      <ul>
       <li>
        必须在
        <code>
         open()
        </code>
        之后但在
        <code>
         send()
        </code>
        之前调用。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        onreadystatechange
       </strong>
       ：当
       <code>
        readyState
       </code>
       属性改变时触发的事件处理器。
      </p>
      <ul>
       <li>
        检查
        <code>
         xhr.readyState === 4 &amp;&amp; xhr.status === 200
        </code>
        来确认请求成功完成。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        responseType
       </strong>
       ：设置响应的数据类型（如
       <code>
        ''
       </code>
       ,
       <code>
        'arraybuffer'
       </code>
       ,
       <code>
        'blob'
       </code>
       ,
       <code>
        'document'
       </code>
       ,
       <code>
        'json'
       </code>
       ,
       <code>
        'text'
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        response
       </strong>
       ：返回从服务器接收到的响应数据，格式由
       <code>
        responseType
       </code>
       决定。
      </p>
     </li>
    </ul>
    <h4>
     <a id="_474">
     </a>
     基本模板步骤
    </h4>
    <ol>
     <li>
      <strong>

       <code>

     </li>
     <li>
      <strong>

       <code>

      <code>
       null
      </code>
      ；对于POST请求，则需要传递数据。
     </li>
    </ol>

    <h4>
     <a id="_516">
     </a>
     关键点总结
    </h4>
    <ul>
     <li>
      <strong>
       创建对象
      </strong>
      ：通过
      <code>
       new XMLHttpRequest()
      </code>
      创建一个新的请求对象。
     </li>
     <li>
      <strong>
       初始化请求
      </strong>
      ：使用
      <code>
       open
      </code>
      方法设置HTTP方法、URL和其他选项。
     </li>
     <li>
      <strong>
       设置事件监听器
      </strong>
      ：在
      <code>
       onreadystatechange
      </code>
      中检查请求的状态，并根据状态码判断请求是否成功。
     </li>
     <li>
      <strong>
       发送请求
      </strong>
      ：使用
      <code>
       send
      </code>
      方法发送请求，对于POST请求，可以通过该方法传递请求体数据。
     </li>
    </ul>
    <h2>
     <a id="Axios_523">
     </a>
     <code>
      Axios
     </code>
    </h2>
    <h4>
     <a id="_525">
     </a>
     概述
    </h4>
    <p>
     Axios 是一个基于 Promise 的 HTTP 客户端，适用于浏览器和 Node.js 环境。
     <strong>
      <font color="“skyblue”">
       它允许开发者通过简洁的 API 发起 HTTP 请求，并且支持拦截请求和响应、取消请求、自动转换 JSON 数据等特性 。
      </font>
     </strong>
    </p>
    <h4>
     <a id="_529">
     </a>
     历史背景
    </h4>
    <p>
     Axios 诞生于需要更现代化的方式来处理网络请求的需求之中，特别是在 JavaScript 中使用 Promise 来简化异步编程的问题。在 Axios 出现之前，开发者通常依赖于 XMLHttpRequest 对象或者 jQuery 提供的 AJAX 方法来进行网络通信。然而，这些方法往往伴随着回调地狱的问题，而 Axios 通过提供基于 Promise 的接口解决了这一问题 。
    </p>
    <h4>
     <a id="_533">
     </a>
     创建一个实例
    </h4>
    <ul>
     <li>
      <p>
       您可以使用自定义配置新建一个实例
      </p>
     </li>
     <li>
      <p>
       axios.create([config])
      </p>
     </li>
    </ul>
    <pre><code class="prism language-js"><span class="token keyword">const</span> instance <span class="token operator">=</span> axios<span class="token punctuation">.</span><span class="token function">create</span><span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span>

    <h2>
     <a id="Fetch_API_873">
     </a>
     Fetch API
    </h2>
    <p>
     Fetch API 是一个现代的网络请求工具，它允许你通过 JavaScript 异步地请求资源，而不需要使用传统的 XMLHttpRequest 对象。Fetch API 提供了更强大和灵活的功能集，并且基于 Promise 实现。
    </p>
    <h3>
     <a id="_877">
     </a>
     基本用法
    </h3>
    <ul>
     <li>
      <code>
       fetch(url)
      </code>
      发出 HTTP 请求，返回一个 Promise 对象。
     </li>
     <li>
      使用
      <code>
       .then()
      </code>
      或
      <code>
       async/await
      </code>
      来处理请求结果。
     </li>
    </ul>
    <p>
     <font color="“skyblue”">
      <strong>
       <code>
        fetch()
       </code>
       的功能与 XMLHttpRequest 基本相同，但有三个主要的差异。
      </strong>
     </font>
    </p>
    <p>
     （1）
     <code>
      fetch()
     </code>
     使用 Promise，不使用回调函数，因此大大简化了写法，写起来更简洁。
    </p>
    <p>
     （2）
     <code>
      fetch()
     </code>
     采用模块化设计，API 分散在多个对象上（Response 对象、Request 对象、Headers 对象），更合理一些；相比之下，XMLHttpRequest 的 API 设计并不是很好，输入、输出、状态都在同一个接口管理，容易写出非常混乱的代码。
    </p>
    <p>
     （3）
     <code>
      fetch()
     </code>
     通过数据流（Stream 对象）处理数据，可以分块读取，有利于提高网站性能表现，减少内存占用，对于请求大文件或者网速慢的场景相当有用。XMLHTTPRequest 对象不支持数据流，所有的数据必须放在缓存里，不支持分块读取，必须等待全部拿到后，再一次性吐出来。
    </p>
    <h3>
     <a id="Response__HTTP__890">
     </a>
     Response 对象：处理 HTTP 回应
    </h3>
    <h4>
     <a id="Response__892">
     </a>
     Response 对象的同步属性
    </h4>
    <p>
     <code>
      fetch()
     </code>
     请求成功以后，得到的是一个
     <strong>
      <font color="“skyblue”">
       Response 对象
      </font>
     </strong>
     。它对应服务器的 HTTP 回应。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token keyword">const</span> response <span class="token operator">=</span> <span class="token keyword">await</span> <span class="token function">fetch</span><span class="token punctuation">(</span>url<span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     前面说过，Response 包含的数据通过 Stream 接口异步读取，但是它还包含一些同步属性，对应 HTTP 回应的标头信息（Headers），可以立即读取。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token keyword">function</span> <span class="token function">fetchText</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
<span class="token keyword">let</span> response <span class="token operator">=</span> <span class="token keyword">await</span> <span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'/readme.txt'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

<span class="token punctuation">}</span>
</code></pre>
    </blockquote>
    <p>
     <strong>
      <font color="“skyblue”">
       标头信息属性有下面这些:
      </font>
     </strong>
    </p>
    <p>
     <strong>
      Response.ok
     </strong>
    </p>
    <p>
     <code>
      Response.ok
     </code>
     属性返回一个布尔值，表示请求是否成功，
     <code>
      true
     </code>
     对应 HTTP 请求的状态码 200 到 299，
     <code>
      false
     </code>
     对应其他的状态码。
    </p>
    <p>
     <strong>
      Response.status
     </strong>
    </p>
    <p>
     <code>
      Response.status
     </code>
     属性返回一个数字，表示 HTTP 回应的状态码（例如200，表示成功请求）。
    </p>
    <p>
     <strong>
      Response.statusText
     </strong>
    </p>
    <p>
     <code>
      Response.statusText
     </code>
     属性返回一个字符串，表示 HTTP 回应的状态信息（例如请求成功以后，服务器返回"OK"）。
    </p>
    <p>
     <strong>
      Response.url
     </strong>
    </p>
    <p>
     <code>
      Response.url
     </code>
     属性返回请求的 URL。如果 URL 存在跳转，该属性返回的是最终 URL。
    </p>
    <p>
     <strong>
      Response.type
     </strong>
    </p>
    <p>
     <code>
      Response.type
     </code>
     属性返回请求的类型。可能的值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        basic
       </code>
       ：普通请求，即同源请求。
      </li>
      <li>
       <code>
        cors
       </code>
       ：跨域请求。
      </li>
      <li>
       <code>
        error
       </code>
       ：网络错误，主要用于 Service Worker。
      </li>
      <li>
       <code>
        opaque
       </code>
       ：如果
       <code>
        fetch()
       </code>
       请求的
       <code>
        type
       </code>
       属性设为
       <code>
        no-cors
       </code>
       ，就会返回这个值，详见请求部分。表示发出的是简单的跨域请求，类似
       <code>
        &lt;form&gt;
       </code>
       表单的那种跨域请求。
      </li>
      <li>
       <code>
        opaqueredirect
       </code>
       ：如果
       <code>
        fetch()
       </code>
       请求的
       <code>
        redirect
       </code>
       属性设为
       <code>
        manual
       </code>
       ，就会返回这个值，详见请求部分。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      Response.redirected
     </strong>
    </p>
    <p>
     <code>
      Response.redirected
     </code>
     属性返回一个布尔值，表示请求是否发生过跳转。
    </p>
    <h4>
     <a id="_942">
     </a>
     判断请求是否成功
    </h4>
    <p>
     <code>
      fetch()
     </code>
     发出请求以后，有一个很重要的注意点：
     <strong>
      <font color="“skyblue”">
       只有网络错误，或者无法连接时，
       <code>
        fetch()
       </code>
       才会报错，其他情况都不会报错，而是认为请求成功。
      </font>
     </strong>
    </p>
    <p>
     这就是说，即使服务器返回的状态码是 4xx 或 5xx，
     <code>
      fetch()
     </code>
     也不会报错
    </p>
    <ul>
     <li>
      只有通过
      <code>
       Response.status
      </code>
      属性，得到 HTTP 回应的真实状态码，才能判断请求是否成功。
     </li>
    </ul>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token keyword">function</span> <span class="token function">fetchText</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
<span class="token keyword">let</span> response <span class="token operator">=</span> <span class="token keyword">await</span> <span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'/readme.txt'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">if</span> <span class="token punctuation">(</span>response<span class="token punctuation">.</span>status <span class="token operator">&gt;=</span> <span class="token number">200</span> <span class="token operator">&amp;&amp;</span> response<span class="token punctuation">.</span>status <span class="token operator">&lt;</span> <span class="token number">300</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
<span class="token keyword">return</span> <span class="token keyword">await</span> response<span class="token punctuation">.</span><span class="token function">text</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span> <span class="token keyword">else</span> <span class="token punctuation">{<!-- --></span>
<span class="token keyword">throw</span> <span class="token keyword">new</span> <span class="token class-name">Error</span><span class="token punctuation">(</span>response<span class="token punctuation">.</span>statusText<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    </blockquote>
    <ul>
     <li>
      另一种方法是判断
      <code>
       response.ok
      </code>
      是否为
      <code>
       true
      </code>
      。
     </li>
    </ul>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token keyword">if</span> <span class="token punctuation">(</span>response<span class="token punctuation">.</span>ok<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
<span class="token comment">// 请求成功</span>
<span class="token punctuation">}</span> <span class="token keyword">else</span> <span class="token punctuation">{<!-- --></span>
<span class="token comment">// 请求失败</span>
<span class="token punctuation">}</span>
</code></pre>
    </blockquote>
    <h4>
     <a id="_973">
     </a>
     读取内容的方法
    </h4>
    <p>
     <code>
      Response
     </code>
     对象根据服务器返回的不同类型的数据，提供了不同的读取方法。
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        response.text()
       </code>
       ：得到文本字符串。
      </li>
      <li>
       <code>
        response.json()
       </code>
       ：得到 JSON 对象。
      </li>
      <li>
       <code>
        response.blob()
       </code>
       ：得到二进制 Blob 对象。
      </li>
      <li>
       <code>
        response.formData()
       </code>
       ：得到 FormData 表单对象。
      </li>
      <li>
       <code>
        response.arrayBuffer()
       </code>
       ：得到二进制 ArrayBuffer 对象。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      <font color="“skyblue”">
       上面读取方法都是异步的，返回的都是 Promise 对象。必须等到异步操作结束，才能得到服务器返回的完整数据。
      </font>
     </strong>
    </p>
    <p>
     <strong>
      response.text()
     </strong>
    </p>
    <ul>
     <li>
      用途：用于获取文本数据，比如 HTML 文件。
     </li>
    </ul>
    <p>
     <strong>
      response.json()
     </strong>
    </p>
    <ul>
     <li>
      用途：主要用于获取服务器返回的 JSON 数据。
     </li>
    </ul>
    <p>
     <strong>
      response.formData()
     </strong>
    </p>
    <ul>
     <li>
      用途：主要应用于 Service Worker 中，拦截用户提交的表单，允许修改某些数据后再提交给服务器。
     </li>
    </ul>
    <p>
     <strong>
      response.blob()
     </strong>
    </p>
    <ul>
     <li>
      用途：用于获取二进制文件，如图片、视频等。
     </li>
    </ul>
    <p>
     <strong>
      response.arrayBuffer()
     </strong>
    </p>
    <ul>
     <li>
      用途：主要用于处理流媒体文件，例如音频或视频文件的在线播放。
     </li>
    </ul>

    <p>
     HTTP 请求的方法、标头、数据体都在这个对象里面设置。
    </p>
    <p>
     <strong>

     </strong>
    </p>

    <h3>
     <a id="fetch_API_1059">
     </a>
     <code>
      fetch()
     </code>
     配置对象的完整 API
    </h3>
    <p>
     <code>
      fetch()
     </code>
     第二个参数的完整 API 如下。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token keyword">const</span> response <span class="token operator">=</span> <span class="token function">fetch</span><span class="token punctuation">(</span>url<span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span>
<span class="token literal-property property">method</span><span class="token operator">:</span> <span class="token string">"GET"</span><span class="token punctuation">,</span>
<span class="token literal-property property">headers</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
<span class="token string-property property">"Content-Type"</span><span class="token operator">:</span> <span class="token string">"text/plain;charset=UTF-8"</span>
<span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token literal-property property">body</span><span class="token operator">:</span> <span class="token keyword">undefined</span><span class="token punctuation">,</span>
<span class="token literal-property property">referrer</span><span class="token operator">:</span> <span class="token string">"about:client"</span><span class="token punctuation">,</span>
<span class="token literal-property property">referrerPolicy</span><span class="token operator">:</span> <span class="token string">"no-referrer-when-downgrade"</span><span class="token punctuation">,</span>
<span class="token literal-property property">mode</span><span class="token operator">:</span> <span class="token string">"cors"</span><span class="token punctuation">,</span> 
<span class="token literal-property property">credentials</span><span class="token operator">:</span> <span class="token string">"same-origin"</span><span class="token punctuation">,</span>
<span class="token literal-property property">cache</span><span class="token operator">:</span> <span class="token string">"default"</span><span class="token punctuation">,</span>
<span class="token literal-property property">redirect</span><span class="token operator">:</span> <span class="token string">"follow"</span><span class="token punctuation">,</span>
<span class="token literal-property property">integrity</span><span class="token operator">:</span> <span class="token string">""</span><span class="token punctuation">,</span>
<span class="token literal-property property">keepalive</span><span class="token operator">:</span> <span class="token boolean">false</span><span class="token punctuation">,</span>
<span class="token literal-property property">signal</span><span class="token operator">:</span> <span class="token keyword">undefined</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     <code>
      fetch()
     </code>
     请求的底层用的是
     <a href="https://developer.mozilla.org/en-US/docs/Web/API/Request/Request" rel="nofollow">
      Request() 对象
     </a>
     的接口，参数完全一样，因此上面的 API 也是
     <code>
      Request()
     </code>
     的 API。
    </p>
    <p>
     这些属性里面，
     <code>
      headers
     </code>
     、
     <code>
      body
     </code>
     、
     <code>
      method
     </code>
     前面已经给过示例了，下面是其他属性的介绍。
    </p>
    <p>
     <strong>
      cache
     </strong>
    </p>
    <p>
     <code>
      cache
     </code>
     属性指定如何处理缓存。可能的取值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        default
       </code>
       ：默认值，先在缓存里面寻找匹配的请求。
      </li>
      <li>
       <code>
        no-store
       </code>
       ：直接请求远程服务器，并且不更新缓存。
      </li>
      <li>
       <code>
        reload
       </code>
       ：直接请求远程服务器，并且更新缓存。
      </li>
      <li>
       <code>
        no-cache
       </code>
       ：将服务器资源跟本地缓存进行比较，有新的版本才使用服务器资源，否则使用缓存。
      </li>
      <li>
       <code>
        force-cache
       </code>
       ：缓存优先，只有不存在缓存的情况下，才请求远程服务器。
      </li>
      <li>
       <code>
        only-if-cached
       </code>
       ：只检查缓存，如果缓存里面不存在，将返回504错误。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      mode
     </strong>
    </p>
    <p>
     <code>
      mode
     </code>
     属性指定请求的模式。可能的取值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        cors
       </code>
       ：默认值，允许跨域请求。
      </li>
      <li>
       <code>
        same-origin
       </code>
       ：只允许同源请求。
      </li>
      <li>
       <code>
        no-cors
       </code>
       ：请求方法只限于 GET、POST 和 HEAD，并且只能使用有限的几个简单标头，不能添加跨域的复杂标头，相当于提交表单所能发出的请求。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      credentials
     </strong>
    </p>
    <p>
     <code>
      credentials
     </code>
     属性指定是否发送 Cookie。可能的取值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        same-origin
       </code>
       ：默认值，同源请求时发送 Cookie，跨域请求时不发送。
      </li>
      <li>
       <code>
        include
       </code>
       ：不管同源请求，还是跨域请求，一律发送 Cookie。
      </li>
      <li>
       <code>
        omit
       </code>
       ：一律不发送。
      </li>
     </ul>
    </blockquote>
    <p>
     跨域请求发送 Cookie，需要将
     <code>
      credentials
     </code>
     属性设为
     <code>
      include
     </code>
     。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'http://another.com'</span><span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span>
<span class="token literal-property property">credentials</span><span class="token operator">:</span> <span class="token string">"include"</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     <strong>
      signal
     </strong>
    </p>
    <p>
     <code>
      signal
     </code>
     属性指定一个 AbortSignal 实例，用于取消
     <code>
      fetch()
     </code>
     请求，详见下一节。
    </p>
    <p>
     <strong>
      keepalive
     </strong>
    </p>
    <p>
     <code>
      keepalive
     </code>
     属性用于页面卸载时，告诉浏览器在后台保持连接，继续发送数据。
    </p>
    <p>
     一个典型的场景就是，用户离开网页时，脚本向服务器提交一些用户行为的统计信息。这时，如果不用
     <code>
      keepalive
     </code>
     属性，数据可能无法发送，因为浏览器已经把页面卸载了。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript">window<span class="token punctuation">.</span><span class="token function-variable function">onunload</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
<span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'/analytics'</span><span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span>
<span class="token literal-property property">method</span><span class="token operator">:</span> <span class="token string">'POST'</span><span class="token punctuation">,</span>
<span class="token literal-property property">body</span><span class="token operator">:</span> <span class="token string">"statistics"</span><span class="token punctuation">,</span>
<span class="token literal-property property">keepalive</span><span class="token operator">:</span> <span class="token boolean">true</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     <strong>
      redirect
     </strong>
    </p>
    <p>
     <code>
      redirect
     </code>
     属性指定 HTTP 跳转的处理方法。可能的取值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        follow
       </code>
       ：默认值，
       <code>
        fetch()
       </code>
       跟随 HTTP 跳转。
      </li>
      <li>
       <code>
        error
       </code>
       ：如果发生跳转，
       <code>
        fetch()
       </code>
       就报错。
      </li>
      <li>
       <code>
        manual
       </code>
       ：
       <code>
        fetch()
       </code>
       不跟随 HTTP 跳转，但是
       <code>
        response.url
       </code>
       属性会指向新的 URL，
       <code>
        response.redirected
       </code>
       属性会变为
       <code>
        true
       </code>
       ，由开发者自己决定后续如何处理跳转。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      integrity
     </strong>
    </p>
    <p>
     <code>
      integrity
     </code>
     属性指定一个哈希值，用于检查 HTTP 回应传回的数据是否等于这个预先设定的哈希值。
    </p>
    <p>
     比如，下载文件时，检查文件的 SHA-256 哈希值是否相符，确保没有被篡改。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'http://site.com/file'</span><span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span>
<span class="token literal-property property">integrity</span><span class="token operator">:</span> <span class="token string">'sha256-abcdef'</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     <strong>
      referrer
     </strong>
    </p>
    <p>
     <code>
      referrer
     </code>
     属性用于设定
     <code>
      fetch()
     </code>
     请求的
     <code>
      referer
     </code>
     标头。
    </p>
    <p>
     这个属性可以为任意字符串，也可以设为空字符串（即不发送
     <code>
      referer
     </code>
     标头）。
    </p>
    <blockquote>
     <pre><code class="prism language-javascript"><span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'/page'</span><span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span>
<span class="token literal-property property">referrer</span><span class="token operator">:</span> <span class="token string">''</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    </blockquote>
    <p>
     <strong>
      referrerPolicy
     </strong>
    </p>
    <p>
     <code>
      referrerPolicy
     </code>
     属性用于设定
     <code>
      Referer
     </code>
     标头的规则。可能的取值如下：
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        no-referrer-when-downgrade
       </code>
       ：默认值，总是发送
       <code>
        Referer
       </code>
       标头，除非从 HTTPS 页面请求 HTTP 资源时不发送。
      </li>
      <li>
       <code>
        no-referrer
       </code>
       ：不发送
       <code>
        Referer
       </code>
       标头。
      </li>
      <li>
       <code>
        origin
       </code>
       ：
       <code>
        Referer
       </code>
       标头只包含域名，不包含完整的路径。
      </li>
      <li>
       <code>
        origin-when-cross-origin
       </code>
       ：同源请求
       <code>
        Referer
       </code>
       标头包含完整的路径，跨域请求只包含域名。
      </li>
      <li>
       <code>
        same-origin
       </code>
       ：跨域请求不发送
       <code>
        Referer
       </code>
       ，同源请求发送。
      </li>
      <li>
       <code>
        strict-origin
       </code>
       ：
       <code>
        Referer
       </code>
       标头只包含域名，HTTPS 页面请求 HTTP 资源时不发送
       <code>
        Referer
       </code>
       标头。
      </li>
      <li>
       <code>
        strict-origin-when-cross-origin
       </code>
       ：同源请求时
       <code>
        Referer
       </code>
       标头包含完整路径，跨域请求时只包含域名，HTTPS 页面请求 HTTP 资源时不发送该标头。
      </li>
      <li>
       <code>
        unsafe-url
       </code>
       ：不管什么情况，总是发送
       <code>
        Referer
       </code>
       标头。
      </li>
     </ul>
    </blockquote>
    <h2>
     <a id="FetchAxios_1186">
     </a>
     <code>
      Fetch
     </code>
     和
     <code>
      Axios
     </code>
     选哪个？
    </h2>
    <p>
     选择使用
     <code>
      fetch
     </code>
     还是
     <code>
      axios
     </code>
     主要取决于你的具体需求、项目环境以及个人偏好。
    </p>
    <p>
     <strong>
      优点：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       内置支持
      </strong>
      ：Fetch 是浏览器内置的，不需要额外引入库。
     </li>
     <li>
      <strong>
       基于 Promise
      </strong>
      ：提供了基于 Promise 的 API，使得异步操作更加直观易懂。
     </li>
     <li>
      <strong>
       轻量级
      </strong>
      ：没有额外的依赖，适合对包大小敏感的项目。
     </li>
    </ol>
    <p>
     <strong>
      缺点：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       不支持旧版浏览器
      </strong>
      ：虽然现代浏览器都支持 fetch，但如果你需要兼容一些老旧版本的浏览器（如IE），可能需要引入polyfill。
     </li>
     <li>
      <strong>
       默认不处理错误
      </strong>
      ：对于 HTTP 错误状态码（如 404 或 500），fetch 不会拒绝 Promise，这可能会导致错误处理变得复杂。
     </li>
     <li>
      <strong>
       功能相对简单
      </strong>
      ：与 axios 相比，fetch 在处理某些高级特性时可能不如 axios 方便，例如自动转换 JSON 数据、取消请求等。
     </li>
    </ol>
    <h4>
     <a id="Axios_1202">
     </a>
     Axios
    </h4>
    <p>
     <strong>
      优点：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       广泛的浏览器兼容性
      </strong>
      ：Axios 支持所有主流浏览器，包括 IE。
     </li>
     <li>
      <strong>
       更丰富的功能
      </strong>
      ：提供了诸如拦截请求和响应、转换请求和响应数据、取消请求等功能。
     </li>
     <li>
      <strong>
       更好的错误处理
      </strong>
      ：Axios 对于 HTTP 错误状态码会自动拒绝 Promise，简化了错误处理逻辑。
     </li>
     <li>
      <strong>
       易于使用的API
      </strong>
      ：具有更加人性化和简洁的 API 设计，尤其是在处理复杂的请求配置时更为方便。
     </li>
    </ol>
    <p>
     <strong>
      缺点：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       增加项目体积
      </strong>
      ：相对于原生的 fetch 来说，引入 axios 会增加项目的总体积。
     </li>
     <li>
      <strong>
       需要额外安装
      </strong>
      ：需要通过 npm 或者 yarn 等包管理工具进行安装，或者直接在 HTML 中通过
      <code>
       &lt;script&gt;
      </code>
      标签引入。
     </li>
    </ol>
