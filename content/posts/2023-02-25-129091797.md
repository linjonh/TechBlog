     题目1
    </h4>
    <blockquote>
     <p>
      1、一位开发人员在设计应用程序时，添加了一个Text组件和Button组件，开发样图如下所示。该开发者不能选择哪种布局方式来放置组件?
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f0075244199748d0cd265c98b8544875.png">
       <br/>
       A、StackLayout
       <br/>
       B、DependentLayout
       <br/>
       C、DirectionalLayout
       <br/>
       D、TableLayout
      </img>
     </p>
    </blockquote>
    <p>
     解析：（
     <code>
      A
     </code>
     ）
     <br/>
     <mark>
      Java UI框架
     </mark>
     中提供了一些标准布局功能的容器，继承自ComponentContainer，一般以"Layout”结尾，
     <mark>
      StackLayout
     </mark>
     布局方式是堆叠（层叠）布局，视图以层叠的方式显示；
     <mark>
      DependentLayout
     </mark>
     布局方式可以指定相对于其他同级元素或相对于父组件的位置；
     <mark>
      DirectionalLayout
     </mark>
     布局方式可以进行水平或垂直布局；
     <mark>
      TableLayout
     </mark>
     布局方式以使用表格的方式划分子组件，如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fc035e4a03aa57a6811a7d740a71618f.png"/>
    </p>

    <h4>
     <a id="5_47">
     </a>
     题目5
    </h4>
    <blockquote>
     <p>
      5、某开发者点击应用的退出按钮进行Page Ability资源释放时，会触发该Page Ability生命周期中的哪个回调方法?
      <br/>
      A、onForeground()
      <br/>
      B、onStop()
      <br/>
      C、onInactive()
      <br/>
      D、onBackground()
     </p>
    </blockquote>
    <p>
     解析：（
     <code>
      B
     </code>
     ）
     <br/>
     <mark>
      onForeground()方法
     </mark>
     ：处于该状态的Page仍然驻留在内存中，当重新回到前台时，系统将调用该回调。
     <br/>
     <mark>
      onStop()方法
     </mark>
     ：系统要销毁Page时，该回调通知开发者进行系统资源的释放。
     <br/>
     <mark>
      onInActive()方法
     </mark>
     ：Page失去焦点后，系统将调用该回调，然后Page进入INACTIVE状态。
     <br/>
     <mark>
      onBackground()方法
     </mark>
     ：Page将不再对用户可见，该回调通知开发者进行相应的资源释放，然后Page进入BACKGROUND状态。
     <br/>
     <mark>
      Page Ability
     </mark>
     和
     <mark>
      AbilitySlice
     </mark>
     具有类似的生命周期回调方法，如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/235046ae2d96f4db1a327172adb5c451.png"/>
    </p>

      A、onStart()-&gt;onActive()

      <br/>
      C、onBackground() -&gt;onForeground() -&gt;onStop()
      <br/>

     <code>
      A
     </code>

     <mark>
      onStart()方法
     </mark>

     <mark>

     题目9
    </h4>
    <blockquote>
     <p>
      9、某开发者在智慧屏上开发JavaScript应用时，想做一些静态动画，如果要将某个组件沿水平方向移动20Opx,以下哪项样式配置是正确的?
      <br/>
      A、transform:scaleX(200px)
      <br/>
      B、transform:rotateX(200px)
      <br/>
      C、transform:translateX(200px)
      <br/>
      D、transform:translateY(200px)
     </p>
    </blockquote>
    <p>
     解析：（
     <code>
      C
     </code>
     ）
     <br/>
     <mark>
      transform: scaleX()属性
     </mark>
     是沿x轴水平方向进行缩放；
     <br/>
     <mark>
      transform:rotateX()属性
     </mark>
     是沿着x轴水平方向进行2D旋转；
     <br/>

    <blockquote>
     <p>
      10、某用户将手机视频迁移到智慧屏上播放，使用了HarmonyOS分布式任务调度能力。下列哪项属于该场景中手机迁移时打开智慧屏播放软件的过程?

      B、全局查询
      <br/>
      C、远程启动
      <br/>
      D、远程迁移

     <code>
      D
     </code>
     ）
     <br/>

    <h4>
     <a id="11_108">
     </a>
     题目11
    </h4>
    <blockquote>
     <p>
      11、用户将手机导航迁移至智能手表之后，智能手表如果需要获取手机传过来的数据，从下列哪个方法中获取?
      <br/>
      A、onCompleteContinuation()
      <br/>
      B、onStartContinuation()
      <br/>
      C、onSaveData()
      <br/>
      D、onRestoreData()
     </p>
    </blockquote>
    <p>
     解析：（
     <code>
      D
     </code>
     ）
     <br/>
     由于手机导航的数据传输到智能手表，源设备Page在上完成保存数据后，系统在目标设备上会回调
     <mark>
      onRestoreData()方法
     </mark>
     ，在此回调中接受源Page状态的数据并恢复。
    </p>

    <h4>
     <a id="13_127">
     </a>
     题目13
    </h4>
    <blockquote>
     <p>
      13、用户将于机上的视频迁移至智慧屏播放，以下哪个回调方法可以保存手机上的视频播放数据?
      <br/>
      A、onStartContinuation()
      <br/>
      B、onCompleteContinuation()
      <br/>
      C、onRestoreData()
      <br/>
      D、onSaveData()
     </p>
    </blockquote>
    <p>
     解析：（
     <code>
      D
     </code>
     ）
     <br/>
     <mark>
      onStartContinuation()方法
     </mark>
     是Page在请求迁移后，系统首先调用的方法；
     <br/>
     <mark>
      onCompleteContinuation()方法
     </mark>
     当目标设备上数据恢复完成后，系统会在源设备上回调Page的方法，通知迁移完成；
     <br/>
     <mark>
      onRestoreData()方法
     </mark>
     是源设备Page在上完成保存数据后，系统在目标设备上会回调onRestoreData()方法，在此回调中接受源Page状态的数据并恢复；
     <br/>
     <mark>
      onSaveData()方法
     </mark>
     是当onStartContinuation()方法返回true后系统回调的方法，该回调中保存迁移至目标设备上以便恢复Page状态的数据。
    </p>
