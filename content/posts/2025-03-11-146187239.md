     服务器端配置
    </h3>
    <hr/>
    <h4>
     一、安装桌面环境：
    </h4>
    <h5>
     1.更新软件源。
    </h5>
    <pre><code class="hljs">sudo apt update</code></pre>
    <h5>
     2.安装桌面环境所需的软件包，包括系统面板、窗口管理器、文件浏览器、终端等桌面应用程序。
    </h5>
    <pre><code class="hljs">sudo apt install gnome-panel gnome-settings-daemon metacity nautilus gnome-terminal ubuntu-desktop</code></pre>
    <h4>
     二、配置VNC服务器：
    </h4>
    <h5>
     3.安装 TightVNC 服务器
    </h5>
    <pre><code class="hljs">sudo apt install tightvncserver</code></pre>
    <h5>
     4.通过 vncserver 命令启动 VNC 服务器实例，第一次运行时同时会完成初始化（设置 VNC 访问密码，创建初始配置文件）
    </h5>
    <pre><code class="hljs">vncserver</code></pre>
    <h5>
     5.系统将提示输入并验证密码以远程访问您的机器
     <br/>
     6.验证密码后，可以选择创建仅供查看的密码。使用仅查看密码登录的用户将无法使用鼠标或键盘控制 VNC 实例，没有演示需要的输入 n 即可，后续如有需要可以再设置。
    </h5>
    <p class="img-center">
     <img alt="" height="121" src="https://i-blog.csdnimg.cn/direct/8f8f12356062474998d7abc631c8a3b3.png" width="507"/>
    </p>
    <h4>
     三、配置 xstartup 配置文件
    </h4>
    <h5>
     7.备份VNC的xstartup配置文件
    </h5>
    <pre><code class="hljs">cp ~/.vnc/xstartup ~/.vnc/xstartup.bak</code></pre>
    <h5>
     8.修改VNC的xstartup配置文件
    </h5>
    <pre><code class="hljs">nano ~/.vnc/xstartup</code></pre>
    <p>
     复制以下内容到文件：
    </p>
    <pre><code class="hljs">#!/bin/sh
export XKL_XMODMAP_DISABLE=1
export XDG_CURRENT_DESKTOP="GNOME-Flashback:GNOME"
export XDG_MENU_PREFIX="gnome-flashback-"
gnome-session --session=gnome-flashback-metacity --disable-acceleration-check</code></pre>
    <h5>
     9.关闭VNC
    </h5>
    <pre><code class="hljs">vncserver -kill :1</code></pre>
    <h5>
     10.重新启动 VNC
    </h5>
    <pre><code class="hljs">vncserver -geometry 1920x1080 :1 -localhost no</code></pre>
    <p>
     或不指定分辨率重启
    </p>
    <pre><code class="hljs">vncserver :1</code></pre>

    <h2>
     其他问题
    </h2>
    <hr/>
    <h3>
     一、灰屏或黑屏问题
    </h3>
    <p>
     查看日志文件寻找问题[2]：
    </p>
    <pre><code class="hljs">cat /home/xmu18-tc/.vnc/xmu18:1.log</code></pre>
    <p>
     出现内容如下：
    </p>
    <blockquote>
     <p>
      08/03/25 08:06:04 Xvnc version TightVNC-1.3.10
      <br/>
      08/03/25 08:06:04 Copyright (C) 2000-2009 TightVNC Group
      <br/>
      08/03/25 08:06:04 Copyright (C) 1999 AT&amp;T Laboratories Cambridge
      <br/>
      08/03/25 08:06:04 All Rights Reserved.
      <br/>
      08/03/25 08:06:04 See http://www.tightvnc.com/ for information on TightVNC
      <br/>
      08/03/25 08:06:04 Desktop name 'X' (xmu18:1)
      <br/>
      08/03/25 08:06:04 Protocol versions supported: 3.3, 3.7, 3.8, 3.7t, 3.8t
      <br/>
      08/03/25 08:06:04 Listening for VNC connections on TCP port 5901
      <br/>
      Font directory '/usr/share/fonts/X11/Type1/' not found - ignoring
      <br/>
      Font directory '/usr/share/fonts/X11/75dpi/' not found - ignoring
      <br/>
      Font directory '/usr/share/fonts/X11/100dpi/' not found - ignoring
      <br/>
      Terminated
     </p>
    </blockquote>
    <p>
     提示没有字体，安装即可[3,4]：
    </p>
    <pre><code class="hljs">sudo apt-get update
sudo apt-get install -y xvfb    # 有时候会出现无法定位此包
sudo apt-get install xfonts-100dpi xfonts-75dpi xfonts-cyrillic xorg dbus-x11 xfonts-Type1</code></pre>
    <p>
     然后重新杀死VNC和开启VNC。
    </p>
    <p>
     (/X11/Type1/' not found问题仍暂未找到解决方法...)
    </p>
    <h3>
     二、可能需要修改wayland问题
    </h3>
    <p>
     服务器上打开文件，取消wayland=false的注释[5]：
    </p>
    <pre><code class="hljs">sudo nano /etc/gdm3/custom.conf</code></pre>
    <p>
     重启生效
    </p>
    <pre><code class="hljs">sudo systemctl restart gdm3</code></pre>
    <p>
     (本人测试过，不修改此项因该也没什么问题；但是取消注释比不取消注释用起来流畅一些，可能是不同协议需要转换导致一些延迟..)
    </p>
