    <h3 id="%E4%B8%80%E3%80%81FUZZ%E6%8A%80%E6%9C%AF%E6%A6%82%E8%BF%B0" name="%E4%B8%80%E3%80%81FUZZ%E6%8A%80%E6%9C%AF%E6%A6%82%E8%BF%B0">
     一、FUZZ技术概述
    </h3>
    <p>
     Fuzz（模糊测试）是一种自动化漏洞挖掘技术，通过向目标发送大量随机或半随机的输入，探测系统的异常响应，从而发现潜在漏洞。在Web安全中，Fuzz技术广泛应用于
     <strong>
      未知目录探测、参数爆破、备份文件泄露
     </strong>
     等场景。其核心在于结合
     <strong>
      高效工具
     </strong>
     与
     <strong>
      精准字典
     </strong>
     ，快速覆盖攻击面。
    </p>
    <hr/>
    <h3 id="%E4%BA%8C%E3%80%81FUZZ%E5%B7%A5%E5%85%B7%E4%B8%8E%E5%AD%97%E5%85%B8%E8%B5%84%E6%BA%90" name="%E4%BA%8C%E3%80%81FUZZ%E5%B7%A5%E5%85%B7%E4%B8%8E%E5%AD%97%E5%85%B8%E8%B5%84%E6%BA%90">
     二、FUZZ工具与字典资源
    </h3>
    <h4 id="1.%20%E5%B8%B8%E7%94%A8%E5%B7%A5%E5%85%B7%E6%8E%A8%E8%8D%90" name="1.%20%E5%B8%B8%E7%94%A8%E5%B7%A5%E5%85%B7%E6%8E%A8%E8%8D%90">
     1. 常用工具推荐
    </h4>
    <ul>
     <li>
      <p>
       ​
       <strong>
        目录扫描类
       </strong>
      </p>
      <ul>
       <li>
        ​
        <strong>
         Dirb
        </strong>
        ：基于字典的目录扫描工具，支持递归探测和自定义后缀
        <pre><code>dirb https://example.com -X .php -z 100  # 扫描.php后缀，延迟100ms防封禁</code></pre>
       </li>
       <li>
        ​
        <strong>
         GoBuster
        </strong>
        ：Go语言编写的高并发工具，支持目录、子域名、S3存储桶探测
        <pre><code>gobuster dir -w /path/to/dict.txt -u https://example.com</code></pre>
       </li>
       <li>
        ​
        <strong>
         ffuf
        </strong>
        ：速度极快的多功能工具，支持递归扫描和自动校准结果
        <pre><code>ffuf -w dict.txt -u https://example.com/FUZZ</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       ​
       <strong>
        参数爆破类
       </strong>
      </p>
      <ul>
       <li>
        ​
        <strong>
         Wfuzz
        </strong>
        ：通过占位符
        <code>
         FUZZ
        </code>
        替换字典值，适用于参数值、Header头探测
        <pre><code>wfuzz -w params.txt -u https://example.com?FUZZ=payload</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <h4 id="2.%20%E5%AD%97%E5%85%B8%E8%B5%84%E6%BA%90%E7%B2%BE%E9%80%89" name="2.%20%E5%AD%97%E5%85%B8%E8%B5%84%E6%BA%90%E7%B2%BE%E9%80%89">
     2. 字典资源精选
    </h4>
    <ul>
     <li>
      ​
      <strong>
       综合型字典
      </strong>
      <ul>
       <li>
        ​
        <strong>
         SecLists
        </strong>
        ：涵盖目录、参数、子域名等场景的经典字典集
       </li>
       <li>
        ​
        <strong>
         PayloadsAllTheThings
        </strong>
        ：包含XSS、SQL注入等漏洞的Payload集合
       </li>
      </ul>
     </li>
     <li>
      ​
      <strong>
       专用字典
      </strong>
      <ul>
       <li>
        ​
        <strong>
         备份文件
        </strong>
        ：
        <code>
         *.bak
        </code>
        ,
        <code>
         *.zip
        </code>
        ,
        <code>
         *.tar.gz
        </code>
        等后缀组合，结合域名特征（如
        <code>
         web_bak_2023.rar
        </code>
        ）
       </li>
       <li>
        ​
        <strong>
         隐藏参数
        </strong>
        ：如
        <code>
         debug
        </code>
        、
        <code>
         admin
        </code>
        、
        <code>
         do
        </code>
        等常见参数名
       </li>
       <li>
        ​
        <strong>
         子域名爆破
        </strong>
        ：基于DNS记录的字典（如
        <code>
         dev
        </code>
        ,
        <code>
         test
        </code>
        ,
        <code>
         staging
        </code>
        ）
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3 id="%E4%B8%89%E3%80%81%E6%9C%AA%E7%9F%A5%E7%9B%AE%E5%BD%95%E4%B8%8E%E5%A4%87%E4%BB%BD%E6%96%87%E4%BB%B6%E6%BC%8F%E6%B4%9E%E7%9A%84FUZZ%E5%AE%9E%E6%88%98" name="%E4%B8%89%E3%80%81%E6%9C%AA%E7%9F%A5%E7%9B%AE%E5%BD%95%E4%B8%8E%E5%A4%87%E4%BB%BD%E6%96%87%E4%BB%B6%E6%BC%8F%E6%B4%9E%E7%9A%84FUZZ%E5%AE%9E%E6%88%98">
     三、未知目录与备份文件漏洞的FUZZ实战
    </h3>
    <h4 id="1.%20%E6%9C%AA%E7%9F%A5%E7%9B%AE%E5%BD%95%E6%8E%A2%E6%B5%8B%E6%8A%80%E5%B7%A7" name="1.%20%E6%9C%AA%E7%9F%A5%E7%9B%AE%E5%BD%95%E6%8E%A2%E6%B5%8B%E6%8A%80%E5%B7%A7">
     1. 未知目录探测技巧
    </h4>
    <ul>
     <li>
      ​
      <strong>
       递归扫描
      </strong>
      ：使用
      <code>
       GoBuster
      </code>
      或
      <code>
       ffuf
      </code>
      的递归模式（
      <code>
       -recursion
      </code>
      ），逐级深入目录结构。
     </li>
     <li>
      ​
      <strong>
       后缀FUZZ
      </strong>
      ：针对开发语言特性，添加
      <code>
       .php
      </code>
      、
      <code>
       .asp
      </code>
      等后缀，结合
      <code>
       -X
      </code>
      参数
     </li>
     <li>
      ​
      <strong>
       延时防封禁
      </strong>
      ：通过
      <code>
       -z
      </code>
      参数添加请求间隔（如100ms），避免触发WAF封禁
     </li>
    </ul>
    <h4 id="2.%20%E5%A4%87%E4%BB%BD%E6%96%87%E4%BB%B6%E6%B3%84%E9%9C%B2%E6%8C%96%E6%8E%98" name="2.%20%E5%A4%87%E4%BB%BD%E6%96%87%E4%BB%B6%E6%B3%84%E9%9C%B2%E6%8C%96%E6%8E%98">
     2. 备份文件泄露挖掘
    </h4>
    <ul>
     <li>
      ​
      <strong>
       常见命名规则
      </strong>
      ：
      <ul>
       <li>
        域名相关：
        <code>
         example.com.zip
        </code>
        、
        <code>
         example.tar
        </code>
       </li>
       <li>
        时间戳：
        <code>
         backup_20240313.sql
        </code>
        。
       </li>
       <li>
        系统默认：
        <code>
         web.config.bak
        </code>
        、
        <code>
         config.php.old
        </code>
       </li>
      </ul>
     </li>
     <li>
      ​
      <strong>
       自动化工具
      </strong>
      ：
      <ul>
       <li>
        ​
        <strong>
         bfac
        </strong>
        ：专用于备份文件探测，支持多级后缀组合与域名特征生成
        <pre><code>bfac -u https://example.com --scan-type backup</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <h4 id="3.%20%E5%AE%9E%E6%88%98%E6%A1%88%E4%BE%8B" name="3.%20%E5%AE%9E%E6%88%98%E6%A1%88%E4%BE%8B">
     3. 实战案例
    </h4>
    <ul>
     <li>
      ​
      <strong>
       案例1：隐藏参数FUZZ
      </strong>
      <br/>
      某目标页面
      <code>
       admin.jsp
      </code>
      返回空白，通过参数FUZZ发现
      <code>
       do=export
      </code>
      触发敏感数据导出
     </li>
     <li>
      ​
      <strong>
       案例2：备份文件泄露
      </strong>
      <br/>
      使用
      <code>
       ffuf
      </code>
      扫描子域名，发现
      <code>
       dev.example.com.rar
      </code>
      ，解压后获取数据库配置
     </li>
    </ul>

     四、总结
    </h3>
    <p>
     FUZZ技术的核心在于
     <strong>
      工具、字典、场景
     </strong>
     的三者结合。通过自动化工具提升效率，结合高质量字典覆盖攻击面，最终在未知目录、备份文件等场景中精准定位漏洞。安全人员需持续更新字典库，并关注目标业务特征，灵活调整策略。
    </p>
