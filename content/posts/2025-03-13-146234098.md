    <p>

    <br/>
    数据库物理备份是指直接备份数据库的物理文件，包括数据文件、日志文件、控制文件等，而不是通过逻辑方式（如SQL语句）导出数据。物理备份是对数据库底层文件的完整复制，通常用于快速恢复数据库到某个时间点的状态。
    <p>
    </p>
    <p>
     物理备份的特点
    </p>
    <ol>
     <li>
      备份速度快：物理备份直接复制文件，通常比逻辑备份（如导出SQL语句）更快。
     </li>
     <li>
      恢复速度快：恢复时直接替换文件，不需要重新执行SQL语句。
     </li>
     <li>
      完整性高：备份的是数据库的完整状态，包括数据文件、日志文件等，适合大规模数据恢复。
     </li>
    </ol>
    <h3>
     <a id="1_gs_backup_9">
     </a>
     1. gs_backup
    </h3>
    <p>
     openGauss部署成功后，在数据库运行的过程中，会遇到各种问题及异常状态。openGauss提供了gs_backup工具帮助openGauss备份、恢复重要数据，主要目的是避免某个节点出现二进制文件（工具脚本）、配置文件丢失。
    </p>
    <p>
     语法结构：
    </p>
    <pre><code class="prism language-shell">gs_backup <span class="token parameter variable">-t</span> 执行方式（备份/恢复） --backup-dir<span class="token operator">=</span>备份/恢复目录 <span class="token parameter variable">--parameter</span>
</code></pre>
    <p>
     前提条件
    </p>
    <ul>
     <li>
      可以正常连接openGauss数据库。
     </li>
     <li>
      在进行还原时，需要保证各节点备份目录中存在备份文件。
     </li>
     <li>
      需以操作系统用户omm执行gs_backup命令。
     </li>
    </ul>
    <h5>
     <a id="1_25">
     </a>
     （1）备份
    </h5>
    <p>
     ① 备份参数文件，可以指定–parameter或者不指定，不指定的情况下默认就是备份参数文件
    </p>
    <pre><code class="prism language-shell">gs_backup <span class="token parameter variable">-t</span> backup --backup-dir<span class="token operator">=</span>/home/omm/backup <span class="token parameter variable">--parameter</span>
<span class="token comment"># -t：指定备份或恢复</span>
<span class="token comment"># --backup-dir：备份/恢复文件所存放路径（最后一级目录可不存在，如果不存在系统自动创建）</span>
<span class="token comment"># -h：备份的节点，如果不指定就备份整个集群所有节点</span>
<span class="token comment"># -- parameter（默认）：备份postgresql.conf，pg_hba.conf文件</span>
<span class="token comment"># -- binary：备份工具脚本</span>
<span class="token comment"># -- all：备份postgresql.conf，pg_hba.conf和工具脚本文件</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fd2586ef071b45daacf7edf36644c2c1.png"/>
    </p>
    <p>
     查看备份得到的参数文件，因为没有指定-h，系统默认备份整个集群所有节点，所以加压压缩包后会得到又一层压缩包文件，每个压缩包为一个节点的，比如当前节点为node1，将parameter.tar加压后会得到一个parameter_node1.tar文件
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">cd</span> /home/omm/backup
<span class="token function">tar</span> xvf parameter.tar
<span class="token comment"># 查看解压后得到的新压缩文件名称</span>
<span class="token function">ls</span>
<span class="token comment"># 解压指定节点的压缩包</span>
<span class="token function">tar</span> xvf parameter_hostname.tar
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c66ea9d113a846b79580f72c95324e5c.png"/>
    </p>
    <p>
     ② 备份二进制文件（备份app目录下的工具文件）
    </p>
    <pre><code class="prism language-shell">gs_backup <span class="token parameter variable">-t</span> backup --backup-dir<span class="token operator">=</span>/home/omm/backup <span class="token parameter variable">--binary</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f87adbae0a724962b9b5d9da9981ca10.png"/>
    </p>
    <p>
     跟备份参数文件一样的，会得到一个集群二进制文件的压缩包，binary.tar，解压后得到各个节点的二进制文件压缩包binary_hostname.tar
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">cd</span> /home/omm/backup
<span class="token function">tar</span> xvf binary.tar
<span class="token comment"># 查看解压后得到的新压缩文件名称</span>
<span class="token function">ls</span>
<span class="token comment"># 解压指定节点的压缩包</span>
<span class="token function">tar</span> xvf binary_hostname.tar
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/eef513515d89462ba487507d271c9b09.png"/>
    </p>
    <p>
     查看备份的二进制文件信息
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 此实验中app名称为app_b035fd</span>
<span class="token function">ls</span> app_33b035fd/bin
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6e57f2549b5f40d3b22d29ae49aaf08f.png"/>
    </p>
    <p>
     ③ 当指定备份all时，相当于同时指定–parameter和–binary
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 为了更清晰的了解备份内容，将前面两个操作备份和解压文件全部移除（文件存在会自动覆盖，这里是为了明确了解不同参数备份的文件内容）</span>
<span class="token function">rm</span> <span class="token parameter variable">-rf</span> /home/omm/backup/*
<span class="token comment"># 执行备份</span>
gs_backup <span class="token parameter variable">-t</span> backup --backup-dir<span class="token operator">=</span>/home/omm/backup <span class="token parameter variable">--all</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/391e29e5875541c9a464ff3e32768d50.png"/>
    </p>
    <h5>
     <a id="2_101">
     </a>
     （2）恢复
    </h5>
    <p>
     当数据库出现参数文件、二进制文件丢失时，可以基于之前备份的文件进行恢复
    </p>
    <pre><code class="prism language-shell">gs_backup <span class="token parameter variable">-t</span> restore --backup-dir<span class="token operator">=</span>/home/omm/backup <span class="token parameter variable">--all</span>
<span class="token comment"># -h 备份的节点，如果不指定就备份整个集群所有节点</span>
<span class="token comment"># -- parameter（默认）：备份postgresql.conf，pg_hba.conf文件</span>
<span class="token comment"># -- binary：备份工具脚本</span>
<span class="token comment"># -- all：备份postgresql.conf，pg_hba.conf和工具脚本文件</span>
<span class="token comment"># -- force：强制restore，仅限于--all或--binary时才生效</span>
</code></pre>

    <h5>
     <a id="3_117">
     </a>
     （3）手动恢复的办法
    </h5>
    <ul>
     <li>
      <p>
       pg_hba.conf可以手动从正常节点上复制，然后修改客户端认证策略
      </p>
     </li>
     <li>
      <p>
       postgresql.conf可以通过调用gs_om重写生成（需要借助xml文件）

     <li>
      <p>
       二进制文件可以手动从正常节点上复制
      </p>
     </li>
    </ul>
    <h3>
     <a id="2_gs_basebackup_127">
     </a>
     2. gs_basebackup
    </h3>
    <p>
     openGauss部署成功后，在数据库运行的过程中，会遇到各种问题及异常状态。openGauss提供了gs_basebackup工具做基础的物理备份。gs_basebackup的实现目标是对服务器数据库文件的二进制进行拷贝。
    </p>
    <p>
     远程执行gs_basebackup时，需要使用系统管理员账户。
    </p>
    <p>
     gs_basebackup当前支持热备份模式和压缩格式备份模式。
    </p>
    <p>
     gs_basebackup目前仅支持全量备份，恢复需要手动恢复
    </p>
    <p>
     语法结构：
    </p>
    <pre><code class="prism language-shell">gs_basebackup <span class="token parameter variable">-D</span> 备份路径（目录） <span class="token parameter variable">-p</span> port
<span class="token comment"># -D：备份文件存放路径，路径必须存在，且必须为空</span>
</code></pre>
    <p>
     如果备份目录不存在会报如下异常：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f50edb55ceb046cbb22a16f302b1837b.png"/>
    </p>
    <p>
     如果备份目录不为空会报如下异常：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/570f70418a8a4d7c9fe08713b9c67e1e.png"/>
    </p>
    <h5>
     <a id="1_153">
     </a>
     （1）备份
    </h5>
    <pre><code class="prism language-shell">gs_basebackup <span class="token parameter variable">-D</span> /home/omm/backup/basebackup <span class="token parameter variable">-p</span> <span class="token number">15400</span>
<span class="token comment"># 如果环境变量中配置过端口，可以不指定-p</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8fab1e3144104c55bd9d672458088938.png"/>
    </p>
    <p>
     当数据库中存在绝对路径的表空间时，需要手动为对应表空间配置映射路径
    </p>
    <p>
     伪造异常场景（存在绝对路径表空间）
    </p>
    <pre><code class="prism language-shell">gsql <span class="token parameter variable">-p</span> <span class="token number">15400</span> <span class="token parameter variable">-d</span> postgres <span class="token parameter variable">-r</span>
</code></pre>
    <p>
     在数据库中创建表空间
    </p>
    <pre><code class="prism language-sql"><span class="token comment"># 创建表空间</span>
<span class="token keyword">create</span> <span class="token keyword">tablespace</span> tbs1 location <span class="token string">'/home/omm/tbs1'</span><span class="token punctuation">;</span>
<span class="token comment"># 创建表</span>
<span class="token keyword">create</span> <span class="token keyword">table</span> t1<span class="token punctuation">(</span>c1 <span class="token keyword">int</span><span class="token punctuation">)</span> <span class="token keyword">tablespace</span> tbs1<span class="token punctuation">;</span>
<span class="token comment"># 给t1表添加数据</span>
<span class="token keyword">insert</span> <span class="token keyword">into</span> t1 <span class="token keyword">values</span><span class="token punctuation">(</span><span class="token number">1</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dc6fb0b79122460c9e52490952cb24cb.png"/>
    </p>
    <p>
     退出数据库连接（\q）再重新执行备份
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 清空备份路径，否则会出现异常</span>
<span class="token function">rm</span> <span class="token parameter variable">-rf</span> /home/omm/backup/basebackup/*
<span class="token comment"># 执行备份</span>
gs_basebackup <span class="token parameter variable">-D</span> /home/omm/backup/basebackup <span class="token parameter variable">-p</span> <span class="token number">15400</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1724b14423184313983e41d2632c47bf.png"/>
    </p>
    <p>
     根据异常信息可以看到因为/home/omm/tbs1目录不为空导致备份异常，就是某个绝对路径表空间没有指定空的映射路径（空的）导致的
    </p>
    <pre><code class="prism language-shell">gs_basebackup <span class="token parameter variable">-D</span> /home/omm/backup/basebackup <span class="token parameter variable">-p</span> <span class="token number">15400</span> <span class="token parameter variable">-T</span> /home/omm/tbs1<span class="token operator">=</span>/home/omm/backup/basebackup/tbs1

<span class="token comment"># 当数据目录异常，目录权限异常无法恢复，数据目录不存在</span>
<span class="token comment"># 手动造成数据目录异常的场景，mv 将数据目录名称修改为其他名称，rm把数据目录删除</span>

    <h6>
     <a id="__236">
     </a>
     ② 恢复
    </h6>
    <p>
     将备份的目录复制到数据目录下，名称依旧还是原数据目录名称（无法确定名称用上面介绍的方式查看），设置过映射的表空间当前所使用路径为备份时指定的目录，所以只需要恢复数据目录即可，且表空间原目录可以不维护
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/afda7fca90874d72a3ca3d9041a1f9e6.png"/>
    </p>
    <pre><code class="prism language-shell"><span class="token comment"># 将备份目录复制到/opt/huawei/install/data目录下，并命名为dn</span>
<span class="token function">cp</span> <span class="token parameter variable">-r</span> /home/omm/backup/basebackup /opt/huawei/install/data/dn
<span class="token comment"># 启动数据库</span>
gs_om <span class="token parameter variable">-t</span> start
<span class="token comment"># 查看数据库状态</span>
gs_om <span class="token parameter variable">-t</span> query
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8524497cccf040e6bd50e670ab5eef82.png"/>
    </p>
    <p>
     备份流程
    </p>
    <ol>
     <li>
      创建备份目录，确保目录
      <font color="red">
       存在且为空
      </font>
     </li>
     <li>
      调用工具，执行备份
     </li>
     <li>
      如果有绝对路径的表空间，需要配置映射路径
     </li>
    </ol>
    <p>
     恢复流程
    </p>
    <ol>
     <li>
      使用gs_om查看原数据目录的路径
     </li>
     <li>
      将备份目录整体复制到数据路径的路径，并且目录名称与原数据目录保持一致
     </li>
     <li>
      如果绝对路径的表空间数据也丢失，将备份指定的目录复制回原目录
     </li>
    </ol>
    <h3>
     <a id="3_gs_probackup_269">
     </a>
     3. gs_probackup
    </h3>
    <p>
     gs_probackup是一个用于管理openGauss数据库备份和恢复的工具。它对openGauss实例进行
     <strong>
      定期备份
     </strong>
     ，以便在数据库出现故障时能够恢复服务器。
    </p>
    <ul>
     <li>
      可用于备份单机数据库，也可对主机或者主节点数据库备机进行备份，为物理备份。
     </li>
     <li>
      可备份外部目录的内容，如脚本文件、配置文件、日志文件、dump文件等。
     </li>
     <li>
      支持
      <strong>
       全量备份、增量备份、定期备份和远程备份
      </strong>
      。
     </li>
     <li>
      可设置备份的留存策略。
     </li>
    </ul>
    <h5>
     <a id="1_278">
     </a>
     （1）开启增量功能
    </h5>
    <p>
     登录到数据库
    </p>
    <pre><code class="prism language-shell">gsql <span class="token parameter variable">-p</span> <span class="token number">15400</span> <span class="token parameter variable">-d</span> postgres <span class="token parameter variable">-r</span>
</code></pre>
    <p>
     查询pg_settings视图，确定当前cbm参数是开启还是关闭，默认是关闭状态
    </p>

    <p>
     使用备份恢复数据，当数据库中存在设置了绝对路径的表空间时，恢复时需要为该目录重新指定映射一个新的空目录，否则就会出现如下异常
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1a4fff643f594f03a3e1abb943e02042.png"/>
     <br/>
     3331861)
    </p>
    <p>
     -T参数和–external-mapping参数需要同时使用
    </p>
    <pre><code class="prism language-shell">gs_probackup restore <span class="token parameter variable">--instance</span> dn1 <span class="token parameter variable">-B</span> /home/omm/backup/probackup/ <span class="token parameter variable">-i</span> ST1V18 <span class="token parameter variable">-D</span> /opt/huawei/install/data/dn_bak <span class="token parameter variable">-T</span> /home/omm/backup/basebackup/tbs1<span class="token operator">=</span>/home/omm/pg_tbs/tbs1 --external-mapping<span class="token operator">=</span>/home/omm/backup/basebackup/tbs1<span class="token operator">=</span>/home/omm/pg_tbs/tbs1

<span class="token comment"># -i：指定恢复到哪次备份，如果没有指定-i就恢复到最近一次备份</span>
<span class="token comment"># -D：指定恢复到哪个数据目录，如果不指定就是集群中记录的默认数据目录，恢复的目录需要是空的</span>
<span class="token comment"># -T OLDDIR=NEWDIR：为绝对路径的表空间指定路径映射，一个-T指定一个表空间，此选项必须和--external-mapping一起使用</span>
<span class="token comment"># --external-mapping=OLDDIR=NEWDIR：在恢复时，将包含在备份中的外部目录从_OLDDIR_重新定位到_NEWDIR_目录</span>
</code></pre>
    <p>
     但是备份时未指定备份数据目录外目录，所以指定恢复时会报错
    </p>
