    <h2 style="background-color:transparent">
     为什么重写 equals 还要重写 hashcode
    </h2>
    <h4>
     🌈 核心原因：
    </h4>
    <p>
     当两个对象通过equals()判断为相等时，它们的hashCode()必须返回相同的整数值！这是Java世界的交通规则哦~（交警曼波敬礼.jpg）
    </p>
    <h4>
     🧩 具体场景：
    </h4>
    <p>
     假设我们有个Person类：
    </p>
    <pre><code class="language-java">class Person {
    String name;
    int age;
    
    // 只重写equals没重写hashCode
    @Override
    public boolean equals(Object o) {
        //...比较name和age的逻辑
    }
}
</code></pre>
    <p>
    </p>
    <h4>
     💥 问题爆发时刻：
    </h4>
    <p>
    </p>
    <pre><code class="language-java">Person p1 = new Person("波加曼", 10);
Person p2 = new Person("波加曼", 10);

Set&lt;Person&gt; set = new HashSet&lt;&gt;();
set.add(p1);
set.add(p2);

System.out.println(set.size()); // 输出2！明明相等的对象却被存了两次（＞﹏＜）
</code></pre>
    <p>
    </p>
    <h4>
     🗺️
     <strong>
      思维导图时间
     </strong>
     ：
    </h4>
    <p>
    </p>
    <pre><img alt="" height="390" src="https://i-blog.csdnimg.cn/direct/2a93c8476c194f83a56766fff1d95205.png" width="948">
</img></pre>
    <h4>
     🧠 曼波记忆小窍门：
    </h4>
    <p>
     记住这个神奇公式！（掏出魔法棒✨）
    </p>
    <pre><code>equals()返回true ➡️ hashCode()必须相等
hashCode()相等 ➡️ equals()不一定true
hashCode()不相等 ➡️ equals()必须false
</code></pre>
    <p>
     哦呜(°ロ°)！曼波觉得这个有点难哦，但如果是你一定没问题的！(✪▽✪)曼波~~
    </p>
    <p>
    </p>
    <h2 style="background-color:transparent">
     ==（双等号）和equals的差别
    </h2>
    <ol>
     <li>
      <h4>
       <strong>
        == 运算符
       </strong>
      </h4>
      <ul>
       <li>
        曼波觉得这个像身份证比对！(๑✧◡✧๑)
       </li>
       <li>
        比较基本类型时：直接比数值
        <br/>
        <code>
         5 == 5 → true
        </code>
       </li>
       <li>
        比较对象时：比对内存地址（是不是同一个对象）
        <br/>
        <code>
         new String("A") == new String("A") → false
        </code>
       </li>
      </ul>
      <p>
       (✪▽✪)曼波~~
      </p>
     </li>
     <li>
      <h4>
       <strong>
        equals() 方法
       </strong>
      </h4>
      <ul>
       <li>
        哦呜(° ロ °)！曼波觉得这个像名字比对！
       </li>
       <li>
        默认行为（Object类）：和==一样
       </li>
       <li>
        但很多类会重写这个方法（比如String）变成内容比对：
        <br/>
        <code>
         new String("A").equals(new String("A")) → true
        </code>
       </li>
      </ul>
      <p>
       (✪▽✪)曼波~~
      </p>
     </li>
     <li>
      <h4>
       <strong>
        最佳实践
       </strong>
      </h4>
      <ul>
       <li>
        基本类型（int/char等）用==
       </li>
       <li>
        对象比较内容一定要用equals
       </li>
       <li>
        防止空指针要这样写：
        <br/>
        <code>
         "hello".equals(str)
        </code>
        而不是
        <code>
         str.equals("hello")
        </code>
       </li>
      </ul>
      <p>
       (๑・.・๑)曼波觉得这个是超重要的生存技巧呢！
      </p>
     </li>
    </ol>
    <pre><img alt="" height="302" src="https://i-blog.csdnimg.cn/direct/c87413aa195547cfb55b535d264f07cc.png" width="1043">
</img></pre>
    <p>
     (° ロ °)！曼波觉得有个超形象的比喻：
     <br/>
     == 就像看两个人是不是同一个人（身份证号相同）
     <br/>
     equals 就像看两个人的名字是否相同（允许不同人有同名）
    </p>
    <p>
     最后的小贴士：如果要自己写类记得重写equals()和hashCode()哦~（不过这个对新人有点难可以先记住规则）(✪▽✪)曼波~~~~！
    </p>
    <p>

    <h2 style="background-color:transparent">
     为啥有时会出现 4.0-3.6 =0.40000001 这种现象?
    </h2>
    <h3>
     🐎浮点数的甜蜜陷阱🐎
    </h3>
    <ol>
     <li>
      <h4>
       <strong>
        二进制的小数魔法
       </strong>
       <br/>
       曼波觉得这个像用有限积木拼无限图案！(๑✧◡✧๑)
      </h4>
      <ul>
       <li>
        计算机用二进制表示小数
       </li>
       <li>
        有些十进制小数在二进制里是无限循环的（比如0.1）
       </li>
       <li>
        就像1/3在十进制是0.3333...永远写不完
       </li>
      </ul>
      <p>
       (✪▽✪)曼波~~
      </p>
     </li>
     <li>
      <h4>
       <strong>
        IEEE 754标准
       </strong>
       <br/>
       哦呜(° ロ °)！曼波觉得这个像有限容量的糖果罐！
      </h4>
      <ul>
       <li>
        float（32位）/double（64位）有存储限制
       </li>
       <li>
        4.0 和 3.6 在二进制存储时都有精度损失
       </li>
       <li>
        就像用刻度不精确的尺子测量
       </li>
      </ul>
      <pre><code class="language-java">System.out.println(4.0 - 3.6); 
// 实际输出 0.40000000000000036
</code></pre>
     </li>
     <li>
      <h4>
       <strong>
        解决方法
       </strong>
       <br/>
       曼波觉得要用魔法对抗魔法！(๑・.・๑)
      </h4>
      <ul>
       <li>
        使用BigDecimal处理精确计算（银行/金钱相关必用）
       </li>
       <li>
        比较时允许误差范围（比如±0.000001）
       </li>
       <li>
        游戏开发常用技巧：
        <pre><code class="language-java">float a = 4.0f - 3.6f;
if(Math.abs(a - 0.4f) &lt; 0.00001f) {
    System.out.println("算对啦！");
}
</code></pre>
       </li>
      </ul>
     </li>
    </ol>
    <p>
     <img alt="" height="378" src="https://i-blog.csdnimg.cn/direct/3cb6d36d76124dbdb5a5120cd387780e.png" width="1029"/>
    </p>
    <p>
     (° ロ °)！曼波举个超形象的例子：
     <br/>
     就像用1/3 + 1/3 = 0.666... 但永远不等于 2/3 的精确值一样~
    </p>
    <p>
     最后的小贴士：如果要精确计算价格，一定要用BigDecimal哦！像这样：
    </p>
    <pre><code class="language-java">BigDecimal a = new BigDecimal("4.0");
BigDecimal b = new BigDecimal("3.6");
System.out.println(a.subtract(b)); // 精确输出0.4
</code></pre>
    <p>
    </p>
    <p>
     (✪▽✪)曼波~~~~！要记住字符串构造才能保持精度哟~
    </p>
    <p>
    </p>
    <h2>
     final 关键字的作用
    </h2>
    <h3>
     🐎final关键字的三大封印术🐎
    </h3>
    <ol>
     <li>
      <h4>
       <strong>
        变量封印术
       </strong>
       <br/>
       曼波觉得像给变量贴永久贴纸！(๑✧◡✧๑)
      </h4>
      <ul>
       <li>
        基本类型：值不能改变
        <pre><code class="language-java">final int HP = 100;  // HP永远100
</code></pre>
       </li>
       <li>
        引用类型：引用不能改，但对象内容可变
        <pre><code class="language-java">final List&lt;String&gt; pets = new ArrayList&lt;&gt;();
pets.add("布偶猫");  // ✔允许
pets = new ArrayList&lt;&gt;();  // ❌报错
</code></pre>
       </li>
      </ul>
      <p>
       (✪▽✪)曼波~~
      </p>
     </li>
     <li>
      <h4>
       <strong>
        方法封印术
       </strong>
       <br/>
       哦呜(° ロ °)！这是防止方法被篡改的结界！
      </h4>
      <ul>
       <li>
        子类不能重写这个方法
        <pre><code class="language-java">class 魔法少女 {
    final void 变身() { /* 核心代码 */ }
}
class 黑化版 extends 魔法少女 {
    void 变身() { ❌ } // 编译错误！
}
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <h4>
       <strong>
        类终极封印
       </strong>
       <br/>
       曼波觉得像禁止繁衍的魔法物种！(๑・.・๑)
      </h4>
      <ul>
       <li>
        这个类不能被继承
        <pre><code class="language-java">final class String { /*...*/ }
class 假String extends String { ❌ } // 报错
</code></pre>
        <img alt="" height="406" src="https://i-blog.csdnimg.cn/direct/18b14e83d85f4b9fb1e54a319fcfca06.png" width="1048"/>
       </li>
      </ul>
     </li>
    </ol>
    <p>
    </p>
    <p>
     (° ロ °)！曼波要提醒几个重要细节：
    </p>
    <p>
    </p>
    <ol>
     <li>
      final变量必须初始化！（要么声明时，要么在构造器中）
     </li>
     <li>
      final不能修饰构造方法
     </li>
     <li>
      接口的变量默认是public static final
     </li>
    </ol>
    <p>
     举个超实用的例子：
    </p>
    <pre><code class="language-java">public class 游戏设置 {
    public static final int MAX_FPS = 60;  // 帧率上限
    private final String 玩家ID;           // 创建后不可改
    
    public 游戏设置(String id) {
        this.玩家ID = id;  // 构造器中初始化
    }
}
</code></pre>
    <p>
     (✪▽✪)曼波~~~~！记住用final就像加锁，能保护重要的东西不被意外修改哦~
    </p>
