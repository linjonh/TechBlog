  <div class="article_content clearfix" id="article_content">

   <div class="htmledit_views" id="content_views">
    <p>
     <code>
      SpringMVC
     </code>
     对传统的MVC做了扩展，将
     <code>
      model
     </code>
     层分为了业务模型
     <code>
      Service
     </code>
     和数据模型
     <code>
      Repository
     </code>
    </p>
    <p>
     <img alt="" height="771" src="https://i-blog.csdnimg.cn/direct/27e88885276d491db2261246af1bad4d.png" width="1256"/>
    </p>
    <blockquote>
     <p>
      工作原理
     </p>
    </blockquote>
    <ol>
     <li>
      <p>
       前端的请求会首先进过
       <code>
        DispatcherServlet
       </code>
       <strong>
        前端调度器
       </strong>
       ，这个组件是整个MVC的核心调度组件，所有请求都经过它的处理和分发。
      </p>
     </li>
     <li>
      <p>
       然后
       <strong>
        调度器
       </strong>
       会将路径传递给
       <code>
        HandlerMapping
       </code>
       ，它维护了一个从
       <strong>
        URL到具体处理器对象的映射关系
       </strong>
       ，
       <strong>
        相当于指南
       </strong>
       ，查询当前请求需要被
       <code>
        Controller
       </code>
       层哪个具体的方法执行，将处理链返回，处理链中包括拦截器
      </p>
     </li>
     <li>
      <p>
       前端调度器接受到调用链，会先执行调用链中的拦截器方法，然后
       <strong>
        将请求参数传递给适配器
       </strong>
       <code>
        HanderAdaptor
       </code>
      </p>
      <ul>
       <li>
        <p>
         <code>
          HanderAdaptor
         </code>
         作用就是处理请求参数和响应数据，转化和封装
        </p>
       </li>
       <li>
        <p>
         <strong>
          具体就是是选择合适的适配器
         </strong>
         ：
        </p>
       </li>
       <li>
        <p>
         对于传统控制器
         <code>
          Controller
         </code>
         ，
         <code>
          DispatcherServlet
         </code>
         会选择
         <strong>
          方法适配器
         </strong>
         。
        </p>
       </li>
       <li>
        <p>
         对于
         <strong>
          函数式控制器
         </strong>
         ，
         <code>
          DispatcherServlet
         </code>
         会选择
         <strong>
          能够处理函数式路由
         </strong>
         的适配器。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <code>
        HanderAdaptor
       </code>
       会将封装好的参数传入
       <strong>
        <code>
         处理器
        </code>
       </strong>
       ，即Controller的具体方法中，
       <strong>
        执行并返回
       </strong>
       ，这里如果有拦截器，还会执行它的后置方法
      </p>
     </li>
     <li>
      <p>
       如果是前后端不分离的，那么就先将响应数据传递到视图解析器
       <strong>
        ViewResovler
       </strong>
       解析视图，呈现数据给用户。
      </p>
     </li>
     <li>
      <p>
       如果是前后端分离项目，后端直接通过
       <code>
        DispatherSevlet
       </code>
       返回JSON数据给前端，就不需要视图解析器！
      </p>
     </li>
    </ol>
