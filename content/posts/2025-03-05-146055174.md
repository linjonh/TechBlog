    <p>
     Redis Stream（Redis 流）是 Redis 5.0 引入的数据结构，主要用于
     <strong>
      消息队列、事件存储和数据流处理
     </strong>
     。它结合了
     <strong>
      列表（List）、发布订阅（Pub/Sub）和有序集合（Sorted Set）
     </strong>
     的特性，适用于
     <strong>
      高吞吐的日志、事件驱动架构和实时数据处理
     </strong>
     。
    </p>
    <h2>
     <a id="Redis_Stream__1">
     </a>
     <strong>
      Redis Stream 的特点
     </strong>
    </h2>
    <ol>
     <li>
      <strong>
       持久化日志存储
      </strong>
      ：Stream 内部数据按追加（Append-Only）方式存储，支持历史消息查询。
     </li>
     <li>
      <strong>
       消息队列
      </strong>
      ：支持
      <strong>
       生产者-消费者模式
      </strong>
      ，多个消费者可以消费同一条消息。
     </li>
     <li>
      <strong>
       消费者组（Consumer Group）
      </strong>
      ：支持多个消费者分配和消费消息，类似于 Kafka 的
      <strong>
       Consumer Group
      </strong>
      。
     </li>
     <li>
      <strong>
       高效的 ID 机制
      </strong>
      ：每条消息都有一个唯一的
      <strong>
       Stream ID
      </strong>
      （形如
      <code>
       1678875112345-0
      </code>
      ），按时间顺序存储。
     </li>
     <li>
      <strong>
       阻塞和非阻塞读取
      </strong>
      ：支持
      <code>
       XREAD
      </code>
      （读取）、
      <code>
       XREADGROUP
      </code>
      （消费者组读取）以及
      <code>
       XADD
      </code>
      （添加）。
     </li>
     <li>
      <strong>
       自动删除策略
      </strong>
      ：可以手动或自动删除过期数据，防止 Stream 过大占用内存。
     </li>
    </ol>
    <h2>
     <a id="_8">
     </a>
     <strong>
      基本命令
     </strong>
    </h2>
    <table>
     <thead>
      <tr>
       <th>
        命令
       </th>
       <th>
        作用
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         XADD mystream * field1 value1 field2 value2
        </code>
       </td>
       <td>
        添加数据到 Stream
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XRANGE mystream - +
        </code>
       </td>
       <td>
        查询 Stream 内的所有数据
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XREAD COUNT 2 STREAMS mystream 0
        </code>
       </td>
       <td>
        读取 Stream 数据
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XGROUP CREATE mystream mygroup $ MKSTREAM
        </code>
       </td>
       <td>
        创建消费者组
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XREADGROUP GROUP mygroup consumer1 COUNT 1 STREAMS mystream &gt;
        </code>
       </td>
       <td>
        从消费者组读取消息
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XACK mystream mygroup 1684211234567-0
        </code>
       </td>
       <td>
        确认消息已消费
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XDEL mystream 1684211234567-0
        </code>
       </td>
       <td>
        删除特定消息
       </td>
      </tr>
      <tr>
       <td>
        <code>
         XLEN mystream
        </code>
       </td>
       <td>
        获取 Stream 长度
       </td>
      </tr>
     </tbody>
    </table>
    <h2>
     <a id="_19">
     </a>
     <strong>
      典型应用场景
     </strong>
    </h2>
    <ol>
     <li>
      <strong>
       日志和事件存储
      </strong>
      ：如
      <strong>
       用户行为日志、订单流水
      </strong>
      记录。
     </li>
     <li>
      <strong>
       消息队列
      </strong>
      ：适用于
      <strong>
       实时任务调度
      </strong>
      ，提供类似 Kafka 的能力，但更轻量。
     </li>
     <li>
      <strong>
       流式数据处理
      </strong>
      ：配合
      <strong>
       Redis Gears
      </strong>
      进行
      <strong>
       数据分析、实时告警
      </strong>
      。
     </li>
     <li>
      <strong>
       微服务通信
      </strong>
      ：用于异步任务处理、去中心化事件驱动架构。
     </li>
    </ol>
    <p>
     Redis Stream 提供了强大的
     <strong>
      数据流处理能力
     </strong>
     ，结合
     <strong>
      Kafka、RabbitMQ
     </strong>
     等消息队列的优点，适用于高性能分布式系统。
    </p>
