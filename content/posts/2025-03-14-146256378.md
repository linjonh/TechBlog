    <h2>
     <a id="_0">
     </a>
     一、了解
    </h2>
    <p>
     <code>
      list
     </code>
     是标准模板库（STL）提供的一个
     <strong>
      双向链表容器
     </strong>
     。它与 std::vector 或 std::array 不同，
     <strong>
      不支持随机访问（即通过下标直接访问元素）[只能用迭代器访问元素]
     </strong>
     ，
     <strong>
      但支持在任意位置的高效插入和删除操作
     </strong>
     。
    </p>
    <ul>
     <li>
      <strong>
       特性
      </strong>
      <ul>
       <li>
        <strong>
         双向链表
        </strong>
        ：允许2端插入和删除
       </li>
       <li>
        <strong>
         不支持随机访问
        </strong>
        ：【vector和deque支持随机访问】访问list的元素只能用迭代器。
       </li>
       <li>
        <strong>
         动态内存管理
        </strong>
        ：每个节点内存都有它前后2个节点的地址指针，在插入和删除中方便管理内存。
       </li>
       <li>
        <strong>
         保持迭代器有效性
        </strong>
        ：在插入和删除里无迭代器失效。【除了：被删除元素的迭代器】
       </li>
       <li>
        <strong>
         高效的插入和删除
        </strong>
        ：在2段和中间进行插入删除操作都很高效。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       性能考虑
      </strong>
      <ul>
       <li>
        它
        <strong>
         适用于在插入和删除操作频繁
        </strong>
        的情况。如果需要频繁的访问，vector和dequeue方便更多。
       </li>
       <li>
        <strong>
         链表的结构，使得内存上的开销大于vector和dequeue
        </strong>
        。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/75cbbf5d1f70490b93e21451ce97f12a.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <ul>
      <li>
       <h3>
        <a id="_16">
        </a>
        常见问题
       </h3>
       <ul>
        <li>
         <h3>
          <a id="1__stdlist__18">
          </a>
          1、 删除 std::list 中的特定元素
         </h3>
         <ul>
          <li>
           在C++中，std::
           <strong>
            list 提供了 remove(val)
           </strong>
           成员函数来移除所有等于
           <strong>
            特定值
           </strong>
           的元素。
           <br/>
           <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e0d0c7244f594dccbfb1aabfaf9166e7.png"/>
          </li>
         </ul>
        </li>
        <li>
         <h3>
          <a id="2_list__22">
          </a>
          2、 list 的迭代器失效情况
         </h3>
         <ul>
          <li>
           <strong>
            插入操作
           </strong>
           ：在 list 中插入操作
           <strong>
            不会导致任何现有迭代器失效
           </strong>
           ，包括指向插入位置的迭代器。插入操作后，原来的迭代器仍然指向它们原来指向的元素。
          </li>
          <li>
           <strong>
            删除操作
           </strong>
           ：删除操作会导致指向
           <strong>
            被删除元素的迭代器失效
           </strong>
           。然而，
           <strong>
            其他迭代器
           </strong>
           ，包括指向前一个和后一个元素的迭代器，
           <strong>
            仍然有效
           </strong>
           。
          </li>
         </ul>
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
