     如何在 GitHub 上修改他人的分支
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

    <p>
     在 GitHub 上修改他人的分支是一个常见的协作需求，但具体操作取决于你是否拥有目标仓库的直接推送权限。如果你是项目的协作者，可以直接修改分支；如果没有权限，则需要通过 Fork 和 Pull Request（PR）的方式贡献代码。本文将详细介绍这两种情况的操作步骤，涵盖从克隆仓库到提交变更的完整流程，并提供实用建议，确保你的协作过程顺畅高效。
    </p>
    <hr/>
    <h3>
     <a id="_4">
     </a>
     一、直接修改他人分支（需要协作者权限）
    </h3>
    <p>
     如果你被添加为目标仓库的协作者，拥有推送权限，可以直接在他人分支上修改代码。以下是具体步骤：
    </p>
    <h4>
     <a id="1__8">
     </a>
     1. 克隆仓库到本地
    </h4>
    <p>
     首先，你需要将目标仓库下载到本地。假设仓库地址为
     <code>
      https://github.com/username/repository.git
     </code>
     ，在终端运行以下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/username/repository.git
<span class="token builtin class-name">cd</span> repository
</code></pre>
    <p>
     这会将仓库克隆到你的电脑，并默认检出主分支（通常是
     <code>
      main
     </code>
     或
     <code>
      master
     </code>
     ）。
    </p>
    <h4>
     <a id="2__16">
     </a>
     2. 切换到目标分支
    </h4>
    <p>
     使用以下命令查看所有远程分支：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> branch <span class="token parameter variable">-r</span>
</code></pre>
    <p>
     假设你要修改的分支是
     <code>
      feature-branch
     </code>
     ，切换到该分支：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout feature-branch
</code></pre>
    <p>
     如果本地没有该分支，可以从远程拉取并创建：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout <span class="token parameter variable">-b</span> feature-branch origin/feature-branch
</code></pre>
    <h4>
     <a id="3__30">
     </a>
     3. 修改代码
    </h4>
    <p>
     在本地使用编辑器（如 VS Code）修改文件。完成后，将变更暂存到 Git：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
</code></pre>
    <h4>
     <a id="4__36">
     </a>
     4. 提交变更
    </h4>
    <p>
     为你的修改添加一个清晰的提交信息，例如：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Update feature-branch with new changes"</span>
</code></pre>
    <h4>
     <a id="5__42">
     </a>
     5. 推送变更到远程分支
    </h4>
    <p>
     将修改推送到 GitHub 的目标分支：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> push origin feature-branch
</code></pre>
    <p>
     如果推送失败，可能是远程分支有更新或冲突。此时，需先拉取远程代码：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> pull origin feature-branch
</code></pre>
    <p>
     解决冲突后，重新提交并推送。
    </p>
    <h4>
     <a id="_53">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      <strong>
       权限要求
      </strong>
      ：此方法仅适用于有推送权限的协作者。
     </li>
     <li>
      <strong>
       冲突处理
      </strong>
      ：若分支被他人同时修改，需手动合并冲突。
     </li>
     <li>
      <strong>
       沟通先行
      </strong>
      ：直接修改前，建议与仓库拥有者沟通，避免覆盖重要代码。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_Fork__Pull_Request__60">
     </a>
     二、通过 Fork 和 Pull Request 修改（无需直接权限）
    </h3>
    <p>
     如果没有推送权限，你可以通过 Fork 仓库、修改代码并提交 Pull Request 的方式贡献变更。这是 GitHub 上最常见的协作模式，尤其适用于开源项目。
    </p>
    <h4>
     <a id="1_Fork__64">
     </a>
     1. Fork 目标仓库
    </h4>
    <p>
     访问目标仓库的 GitHub 页面（例如
     <code>
      https://github.com/username/repository
     </code>
     ），点击右上角的
     <strong>
      Fork
     </strong>
     按钮。这会将仓库复制到你的 GitHub 账户下，生成一个新仓库，例如
     <code>
      https://github.com/your-username/repository
     </code>
     。
    </p>
    <h4>
     <a id="2__Fork__67">
     </a>
     2. 克隆你的 Fork 到本地
    </h4>
    <p>
     在终端运行：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/your-username/repository.git
<span class="token builtin class-name">cd</span> repository
</code></pre>
    <h4>
     <a id="3_Upstream_74">
     </a>
     3. 添加原始仓库作为上游（Upstream）
    </h4>
    <p>
     为了与原始仓库保持同步，添加上游远程：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token function">add</span> upstream https://github.com/username/repository.git
</code></pre>
    <p>
     验证远程配置：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote <span class="token parameter variable">-v</span>
</code></pre>
    <p>
     你应该看到
     <code>
      origin
     </code>
     （你的 Fork）和
     <code>
      upstream
     </code>
     （原始仓库）。
    </p>
    <h4>
     <a id="4__85">
     </a>
     4. 切换到目标分支并同步
    </h4>
    <p>
     获取原始仓库的最新数据：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> fetch upstream
</code></pre>
    <p>
     基于目标分支（例如
     <code>
      feature-branch
     </code>
     ）创建本地分支：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> checkout <span class="token parameter variable">-b</span> feature-branch upstream/feature-branch
</code></pre>
    <h4>
     <a id="5__95">
     </a>
     5. 修改代码
    </h4>
    <p>
     编辑文件，完成你的更改后暂存并提交：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Add my changes to feature-branch"</span>
</code></pre>
    <h4>
     <a id="6__Fork_102">
     </a>
     6. 推送变更到你的 Fork
    </h4>
    <p>
     将修改推送到你的 Fork 仓库：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> push origin feature-branch
</code></pre>
    <h4>
     <a id="7__Pull_Request_108">
     </a>
     7. 创建 Pull Request
    </h4>
    <ul>
     <li>
      访问你的 Fork 仓库页面（
      <code>
       https://github.com/your-username/repository
      </code>
      ）。
     </li>
     <li>
      点击
      <strong>
       Pull Requests
      </strong>
      标签，然后选择
      <strong>
       New Pull Request
      </strong>
      。
     </li>
     <li>
      设置
      <strong>
       base repository
      </strong>
      为原始仓库（
      <code>
       username/repository
      </code>
      ），
      <strong>
       base branch
      </strong>
      为
      <code>
       feature-branch
      </code>
      。
     </li>
     <li>
      设置
      <strong>
       head repository
      </strong>
      为你的 Fork（
      <code>
       your-username/repository
      </code>
      ），
      <strong>
       compare branch
      </strong>
      为
      <code>
       feature-branch
      </code>
      。
     </li>
     <li>
      检查变更内容，填写 PR 标题和描述，点击
      <strong>
       Create Pull Request
      </strong>
      。
     </li>
    </ul>
    <h4>
     <a id="8__115">
     </a>
     8. 等待审核与反馈
    </h4>
    <p>
     原始仓库拥有者会收到你的 PR 通知。他们可能提出修改建议。若需调整代码，返回步骤 5 修改并推送，PR 会自动更新。
    </p>
    <h4>
     <a id="_118">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      <strong>
       分支命名
      </strong>
      ：若
      <code>
       feature-branch
      </code>
      在你的 Fork 中已存在，可使用新名称（如
      <code>
       my-feature-branch
      </code>
      ），然后在 PR 中指定目标分支。
     </li>
     <li>
      <strong>
       保持同步
      </strong>
      ：定期运行
      <code>
       git fetch upstream
      </code>
      和
      <code>
       git merge upstream/feature-branch
      </code>
      更新本地分支。
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：此方法无需权限，适合任何外部贡献者。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_125">
     </a>
     三、通用建议
    </h3>
    <p>
     无论使用哪种方式，以下建议能提升你的协作效率：
    </p>
    <ul>
     <li>
      <strong>
       备份代码
      </strong>
      ：修改前备份本地文件，防止意外丢失。
     </li>
     <li>
      <strong>
       提交信息
      </strong>
      ：保持提交信息简洁明了，清晰描述变更内容。
     </li>
     <li>
      <strong>
       测试验证
      </strong>
      ：推送前本地测试代码，确保修改无误。
     </li>
     <li>
      <strong>
       工具辅助
      </strong>
      ：GitHub Desktop 或 VS Code 的 Git 集成可简化操作。
     </li>
     <li>
      <strong>
       尊重协作
      </strong>
      ：修改他人分支时，尊重原始作者的意图，必要时提前沟通。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_136">
     </a>
     四、示例场景
    </h3>
    <p>
     假设你要修改
     <code>
      https://github.com/johndoe/project
     </code>
     的
     <code>
      dev
     </code>
     分支：
    </p>
    <h4>
     <a id="_140">
     </a>
     直接修改（有权限）
    </h4>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/johndoe/project.git
<span class="token builtin class-name">cd</span> project
<span class="token function">git</span> checkout dev
<span class="token comment"># 编辑文件</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Fix bug in dev branch"</span>
<span class="token function">git</span> push origin dev
</code></pre>
    <h4>
     <a id="_Fork_151">
     </a>
     通过 Fork（无权限）
    </h4>
    <pre><code class="prism language-bash"><span class="token comment"># Fork 仓库后</span>
<span class="token function">git</span> clone https://github.com/your-username/project.git
<span class="token builtin class-name">cd</span> project
<span class="token function">git</span> remote <span class="token function">add</span> upstream https://github.com/johndoe/project.git
<span class="token function">git</span> fetch upstream
<span class="token function">git</span> checkout <span class="token parameter variable">-b</span> dev upstream/dev
<span class="token comment"># 编辑文件</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Enhance dev branch"</span>
<span class="token function">git</span> push origin dev
<span class="token comment"># 在 GitHub 上创建 PR：base: johndoe/project/dev &lt;- compare: your-username/project/dev</span>
</code></pre>
