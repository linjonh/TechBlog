    <h1 class="title-article" id="articleContentId">
     master数据库是什么
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="./../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="./../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     <a href="https://storage.jishubiji.cn/2020/10/2020101415045561.png" rel="nofollow">
      <img alt="master数据库" height="236" src="https://i-blog.csdnimg.cn/blog_migrate/87f3b7646ce769b050ea61e220e28974.png" width="309"/>
     </a>
    </p>
    <blockquote>
     <p>
      master 数据库记录 SQL Server 系统的所有系统级信息。 这包括实例范围的元数据（例如登录帐户）、端点、链接服务器和系统配置设置。 在 SQL Server中，系统对象不再存储在 master 数据库中，而是存储在 Resource 数据库中。 此外， master 数据库还记录了所有其他数据库的存在、数据库文件的位置以及 SQL Server的初始化信息。 因此，如果 SQL Server master 数据库不可用，则无法启动。
     </p>
    </blockquote>
    <p>
     master 数据库记录了很多重要内容，损坏了很麻烦，要做好备份。适用于： SQL Server（所有支持的版本）
    </p>
    <p>
    </p>
    <h4>
     限制
    </h4>
    <p>
     不能在
     <strong>
      master
     </strong>
     数据库中执行下列操作：
    </p>
    <ul>
     <li>
      添加文件或文件组。
     </li>
     <li>
      备份，主数据库上只能执行完整的数据库备份。
     </li>
     <li>
      更改排序规则。 默认排序规则为服务器排序规则。
     </li>
     <li>
      更改数据库所有者。 master 的所有者是 sa。
     </li>
     <li>
      创建全文目录或全文索引。
     </li>
     <li>
      在数据库的系统表上创建触发器。
     </li>
     <li>
      删除数据库。
     </li>
     <li>
      从数据库中删除 guest 用户。
     </li>
     <li>
      启用变更数据捕获。
     </li>
     <li>
      参与数据库镜像。
     </li>
     <li>
      删除主文件组、主数据文件或日志文件。
     </li>
     <li>
      重命名数据库或主文件组。
     </li>
     <li>
      将数据库设置为 OFFLINE。
     </li>
     <li>
      将数据库或主文件组设置为 READ_ONLY。
     </li>
    </ul>
    <p>
    </p>
    <h4>
     建议
    </h4>
    <p>
     使用
     <strong>
      master
     </strong>
     数据库时，请考虑下列建议：
    </p>
    <ul>
     <li>
      始终有一个
      <strong>
       master
      </strong>
      数据库的当前备份可用。
     </li>
     <li>
      执行下列操作后，尽快备份
      <strong>
       master
      </strong>
      数据库：
      <ul>
       <li>
        创建、修改或删除任意数据库
       </li>
       <li>
        更改服务器或数据库的配置值
       </li>
       <li>
        修改或添加登录帐户
       </li>
      </ul>
     </li>
     <li>
      不要在
      <strong>
       master
      </strong>
      中创建用户对象。 否则，必须更频繁地备份
      <strong>
       master
      </strong>
      。
     </li>
     <li>
      不要针对
      <strong>
       master
      </strong>
      数据库将 TRUSTWORTHY 选项设置为 ON。
     </li>
    </ul>
    <p>
    </p>
    <h4>
     当 master 不可用时怎么办
    </h4>
    <p>
     如果
     <strong>
      master
     </strong>
     数据库不可用，则可以通过下列两种方式之一将该数据库返回到可用状态：
    </p>
    <ul>
     <li>
      从当前数据库备份还原
      <strong>
       master
      </strong>
      。如果你可以启动服务器实例，则应该能够从完整数据库备份还原
      <strong>
       master
      </strong>
      。
     </li>
     <li>
      完全重新生成
      <strong>
       master
      </strong>
      。如果由于
      <strong>
       master
      </strong>
      严重损坏而无法启动 SQL Server，则必须重新生成
      <strong>
       master
      </strong>
      。
     </li>
    </ul>
