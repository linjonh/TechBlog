    <h2>
     一、系统概述
    </h2>
    <h3>
     （1）、HarmonyOS系统框架
    </h3>
    <p class="img-center">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/ab73c4ec2a4bd0a637c7b727796e91e0.png"/>
    </p>
    <h4>
     1、内核层：
    </h4>
    <p>
     提供手机操作系统的基础能力。
    </p>
    <p>
     HarmonyOS采用多内核的系统设计，基于Linux内核、LiteOS，使用了Linux的微内核（使用了Linux的最简功能）
    </p>
    <p>
     Linux内核：手机操作系统的内核
    </p>
    <p>
     LiteOS内核：智能硬件的内核
    </p>
    <h4>
     2、系统服务层：
    </h4>
    <p>
     HarmonyOS的核心能力集合，这些能力由系统本身决定，为我们应用系统开发提供了服务调用功能。
    </p>
    <h4>
     3、框架层：
    </h4>
    <p>
     为HarmonyOS的应用开发提供了不同语言程序调用的接口。
    </p>
    <h3>
     （2）、鸿蒙发展史
    </h3>
    <p>
     1、2012年，华为开始规划智能操作系统“鸿蒙”
    </p>
    <p>
     2、2019年8月，鸿蒙正式版发布，实现开源
    </p>
    <p>
     3、2020年9月，鸿蒙2.0（beta）
    </p>
    <p>
     4、2021年6月，华为线上发布HarmonyOS 2.0
    </p>
    <h3>
     （3）、鸿蒙与安卓的区别
    </h3>
    <h4>
     1、内核
    </h4>
    <p>
     安卓（Android）：基于Linux的内核设计，对Linux的依赖大
    </p>
    <p>
     鸿蒙（HarmonyOS）:采用了多内核设计，Linux内核+LiteOS内核
    </p>
    <h4>
     2、运行效率
    </h4>
    <p>
     安卓（Android）：应用的运行基于虚拟机 （Java---JDK编译器---字节码---虚拟机---操作系统）
    </p>
    <p>
     鸿蒙（HarmonyOS）:应用的运行基于方舟编译器 （Java---方舟编译器---机器码---操作系统）
    </p>
    <h3>
     （4）、技术特性
    </h3>
    <h4>
     1、硬件互助，资源共享
    </h4>
    <p>
     多种设备之间能够实现硬件互助、资源共享，依赖的关键技术包括
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/harmonyos-features-0000000000011907#b1452631372810" rel="nofollow" title="分布式软总线">
      分布式软总线
     </a>
     、
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/harmonyos-features-0000000000011907#b11455195122817" rel="nofollow" title="分布式设备虚拟化">
      分布式设备虚拟化
     </a>
     、
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/harmonyos-features-0000000000011907#b12956155662716" rel="nofollow" title="分布式数据管理">
      分布式数据管理
     </a>
     、
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/harmonyos-features-0000000000011907#b472744918279" rel="nofollow" title="分布式任务调度">
      分布式任务调度
     </a>
     等。
    </p>
    <h4>
     2、一次开发，多端部署
    </h4>
    <p id="ZH-CN_TOPIC_0000001175461719__p0522013456">
     HarmonyOS提供了用户程序框架、
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/glossary-0000000000029587#ZH-CN_TOPIC_0000001114162884__li1373094219463" rel="nofollow" title="Ability">
      Ability
     </a>
     框架以及UI框架，支持应用开发过程中多终端的业务逻辑和界面逻辑进行复用，能够实现应用的一次开发、多端部署，提升了跨设备应用的开发效率。一次开发、多端部署示意图。
    </p>
    <p id="ZH-CN_TOPIC_0000001175461719__p1966673945811">
     其中，UI框架支持使用Java、JS、TS语言进行开发，并提供了丰富的多态控件，可以在手机、平板、智能穿戴、智慧屏、车机上显示不同的UI效果。采用业界主流设计方式，提供多种响应式布局方案，支持栅格化布局，满足不同屏幕的界面适配能力。
    </p>
    <p class="img-center">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/23378c04bd6977500bcdf842550a122a.png"/>
    </p>
    <h4>
     3、统一OS，弹性部署
    </h4>
    <p>
     HarmonyOS通过组件化和小型化等设计方法，支持多种终端设备按需弹性部署，能够适配不同类别的硬件资源和功能需求。支撑通过编译链关系去自动生成组件化的依赖关系，形成组件树依赖图，支撑产品系统的便捷开发，降低硬件设备的开发门槛。
    </p>
    <h2>
     二、Ability学习了解
    </h2>
    <h3>
     （1）、Ability基本概念
    </h3>
    <p>
     1、Ability是应用所具备的能力的抽象，一个应用可以包含一个或多个Ability。Ability分为两种类型：FA（Feature Ability）和PA（Particle Ability）。FA/PA是应用的基本组成单元，能够实现特定的业务功能。FA有UI界面，而PA无UI界面。
    </p>
    <ul id="ZH-CN_TOPIC_0000001083357599__ul569717072619">
     <li id="ZH-CN_TOPIC_0000001083357599__li5697160152610">
      FA支持
      <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/ability-page-concept-0000000000033573" rel="nofollow" title="Page Ability">
       Page Ability
      </a>
      ：
      <p id="ZH-CN_TOPIC_0000001083357599__p4768417112610">
       Page模板是FA唯一支持的模板，用于提供与用户交互的能力。一个Page实例可以包含一组相关页面，每个页面用一个AbilitySlice实例表示。
      </p>
     </li>
     <li id="ZH-CN_TOPIC_0000001083357599__li769710182617">
      PA支持
      <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/ability-service-concept-0000000000044457" rel="nofollow" title="Service Ability">
       Service Ability
      </a>
      和
      <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/ability-data-concept-0000000000043058" rel="nofollow" title="Data Ability">
       Data Ability
      </a>
      ：
      <ul id="ZH-CN_TOPIC_0000001083357599__ul713834114298">
       <li id="ZH-CN_TOPIC_0000001083357599__li313804114297">
        Service模板：用于提供后台运行任务的能力。
       </li>
       <li id="ZH-CN_TOPIC_0000001083357599__li1138184114290">
        Data模板：用于对外部提供统一的数据访问抽象。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     在
     <a href="https://developer.harmonyos.com/cn/docs/documentation/doc-guides/basic-config-file-overview-0000000000011951" rel="nofollow" title="配置文件">
      配置文件
     </a>
     （config.json）中注册Ability时，可以通过配置Ability元素中的“type”属性来指定Ability模板类型，其中，“type”的取值可以为“page”、“service”或“data”，分别代表Page模板、Service模板、Data模板。
    </p>
    <p>
     例：{
     <!-- -->
     <br/>
     "module": {
     <!-- -->
     <br/>
     ...
     <br/>
     "abilities": [
     <br/>
     {
     <!-- -->
     <br/>
     ...
     <br/>
     "type": "page"
     <br/>
     ...
     <br/>
     }
     <br/>
     ]
     <br/>
     ...
     <br/>
     }
     <br/>
     ...
     <br/>
     }
    </p>
    <p>
