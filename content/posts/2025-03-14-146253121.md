    <h2>
     <a id="UbuntucurlDocker_0">
     </a>
     Ubuntu中为curl和Docker配置代理
    </h2>
    <p>
     在Ubuntu环境下工作时，经常需要使用代理来访问网络资源，尤其在网络受限或需要翻墙访问时。本文介绍如何分别为
     <code>
      curl
     </code>
     命令和Docker配置代理
     <code>
      192.168.100.11:8000
     </code>
     。
    </p>
    <h3>
     <a id="curl_4">
     </a>
     一、curl命令代理配置
    </h3>
    <h4>
     <a id="1_6">
     </a>
     方法1：临时设置代理（仅当前命令有效）
    </h4>
    <p>
     直接使用
     <code>
      --proxy
     </code>
     参数：
    </p>
    <pre><code class="prism language-sh"><span class="token function">curl</span> <span class="token parameter variable">--proxy</span> http://192.168.100.11:8000 http://example.com
</code></pre>
    <h4>
     <a id="2_12">
     </a>
     方法2：环境变量设置（影响当前终端会话）
    </h4>
    <p>
     设置临时环境变量，使当前会话中的所有curl请求使用代理：
    </p>
    <pre><code class="prism language-sh"><span class="token builtin class-name">export</span> <span class="token assign-left variable">http_proxy</span><span class="token operator">=</span><span class="token string">"http://192.168.100.11:8000"</span>
<span class="token builtin class-name">export</span> <span class="token assign-left variable">https_proxy</span><span class="token operator">=</span><span class="token string">"http://192.168.100.11:8000"</span>

<span class="token comment"># 验证</span>
<span class="token builtin class-name">echo</span> <span class="token variable">$http_proxy</span>

<span class="token function">curl</span> http://example.com
</code></pre>
    <h4>
     <a id="3_24">
     </a>
     方法3：永久配置（所有终端生效）
    </h4>
    <p>
     修改
     <code>
      ~/.bashrc
     </code>
     或
     <code>
      ~/.zshrc
     </code>
     文件：
    </p>
    <pre><code class="prism language-sh"><span class="token builtin class-name">echo</span> <span class="token string">'export http_proxy="http://192.168.100.11:8000"'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">echo</span> <span class="token string">'export https_proxy="http://192.168.100.11:8000"'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
    <h4>
     <a id="4curlrc_32">
     </a>
     方法4：使用.curlrc单独配置
    </h4>
    <p>
     为
     <code>
      curl
     </code>
     单独配置代理：
    </p>
    <pre><code class="prism language-sh"><span class="token builtin class-name">echo</span> <span class="token string">"proxy = http://192.168.100.11:8000"</span> <span class="token operator">&gt;&gt;</span> ~/.curlrc
</code></pre>
    <h3>
     <a id="Docker_38">
     </a>
     二、Docker代理配置
    </h3>
    <p>
     Docker代理配置包括Docker守护进程（下载镜像等）、Docker客户端命令、以及容器内部网络访问。
    </p>
    <h4>
     <a id="1_Docker_42">
     </a>
     1. 配置Docker守护进程
    </h4>
    <p>
     Docker下载镜像或构建镜像时可能需要代理：
    </p>
    <ul>
     <li>
      创建并编辑配置文件：
     </li>
    </ul>
    <pre><code class="prism language-sh"><span class="token function">sudo</span> <span class="token function">mkdir</span> <span class="token parameter variable">-p</span> /etc/systemd/system/docker.service.d
<span class="token function">sudo</span> <span class="token function">nano</span> /etc/systemd/system/docker.service.d/http-proxy.conf
</code></pre>
    <ul>
     <li>
      添加内容：
     </li>
    </ul>
    <pre><code class="prism language-ini">[Service]
Environment="HTTP_PROXY=http://192.168.100.11:8000"
Environment="HTTPS_PROXY=http://192.168.100.11:8000"
Environment="NO_PROXY=localhost,127.0.0.1"
</code></pre>
    <ul>
     <li>
      重新加载并重启Docker：
     </li>
    </ul>
    <pre><code class="prism language-sh"><span class="token function">sudo</span> systemctl daemon-reload
<span class="token function">sudo</span> systemctl restart <span class="token function">docker</span>

<span class="token comment"># 验证</span>
systemctl show <span class="token parameter variable">--property</span><span class="token operator">=</span>Environment <span class="token function">docker</span>
</code></pre>
    <h4>
     <a id="2_Docker_66">
     </a>
     2. 配置Docker客户端
    </h4>
    <p>
     影响Docker CLI命令（如
     <code>
      docker build
     </code>
     、
     <code>
      docker pull
     </code>
     ）：
    </p>
    <ul>
     <li>
      编辑配置文件：
     </li>
    </ul>
    <pre><code class="prism language-sh"><span class="token function">mkdir</span> <span class="token parameter variable">-p</span> ~/.docker
<span class="token function">nano</span> ~/.docker/config.json
</code></pre>
    <ul>
     <li>
      添加以下内容：
     </li>
    </ul>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"proxies"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"default"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
      <span class="token string-property property">"httpProxy"</span><span class="token operator">:</span> <span class="token string">"http://192.168.100.11:8000"</span><span class="token punctuation">,</span>
      <span class="token string-property property">"httpsProxy"</span><span class="token operator">:</span> <span class="token string">"http://192.168.100.11:8000"</span><span class="token punctuation">,</span>
      <span class="token string-property property">"noProxy"</span><span class="token operator">:</span> <span class="token string">"localhost,127.0.0.1"</span>
    <span class="token punctuation">}</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      验证：
     </li>
    </ul>
    <pre><code class="prism language-sh"><span class="token function">docker</span> info <span class="token operator">|</span> <span class="token function">grep</span> <span class="token parameter variable">-i</span> proxy
</code></pre>
    <h4>
     <a id="3_Docker_91">
     </a>
     3. Docker容器内代理设置
    </h4>
    <p>
     容器默认不会继承宿主机的代理设置，有以下方式：
    </p>
    <h5>
     <a id="_94">
     </a>
     运行时临时指定：
    </h5>
    <pre><code class="prism language-sh"><span class="token function">docker</span> run <span class="token parameter variable">-e</span> <span class="token assign-left variable">HTTP_PROXY</span><span class="token operator">=</span><span class="token string">"http://192.168.100.11:8000"</span> <span class="token punctuation">\</span>
           <span class="token parameter variable">-e</span> <span class="token assign-left variable">HTTPS_PROXY</span><span class="token operator">=</span><span class="token string">"http://192.168.100.11:8000"</span> <span class="token punctuation">\</span>
           <span class="token parameter variable">-it</span> ubuntu <span class="token function">bash</span>
</code></pre>
    <h5>
     <a id="Dockerfile_101">
     </a>
     Dockerfile内设置：
    </h5>
    <pre><code class="prism language-dockerfile">ENV HTTP_PROXY="http://192.168.100.11:8000"
ENV HTTPS_PROXY="http://192.168.100.11:8000"
ENV NO_PROXY="localhost,127.0.0.1"
</code></pre>
    <h5>
     <a id="Docker_CLI_108">
     </a>
     全局设置（Docker CLI配置）
    </h5>
    <p>
     同Docker客户端配置，所有新容器自动使用代理。
    </p>
    <h3>
     <a id="_111">
     </a>
     三、测试代理是否生效
    </h3>
    <p>
     运行容器进行验证：
    </p>
    <pre><code class="prism language-sh"><span class="token function">docker</span> run <span class="token parameter variable">--rm</span> <span class="token parameter variable">-it</span> ubuntu <span class="token function">env</span> <span class="token operator">|</span> <span class="token function">grep</span> <span class="token parameter variable">-i</span> proxy

<span class="token comment"># 或</span>

<span class="token function">docker</span> run <span class="token parameter variable">--rm</span> <span class="token parameter variable">-it</span> ubuntu <span class="token function">curl</span> <span class="token parameter variable">-I</span> http://example.com
</code></pre>
    <p>
     成功返回内容或HTTP头即为配置成功。
    </p>
    <h3>
     <a id="_123">
     </a>
     总结
    </h3>
    <ul>
     <li>
      <strong>
       临时设置
      </strong>
      ：推荐给偶尔或单次使用。
     </li>
     <li>
      <strong>
       永久设置
      </strong>
      ：适合长期工作场景，避免重复配置。
     </li>
     <li>
      <strong>
       Docker守护进程代理
      </strong>
      ：影响镜像下载，推荐配置。
     </li>
     <li>
      <strong>
       Docker容器代理
      </strong>
      ：影响容器内部网络，视情况需要配置。
     </li>
    </ul>
