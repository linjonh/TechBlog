    <h4>
     一、安装与配置
    </h4>
    <p>
     1.安装依赖包
    </p>
    <blockquote>
     <p>
      yum install -y yum-utils device-mapper-persistent-data lvm2
     </p>
    </blockquote>
    <p>
     2.设置阿里云镜像源
    </p>
    <pre><code class="hljs">yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo </code></pre>
    <p>
     查看版本
    </p>
    <blockquote>
     <p>
      yum list docker-ce --showduplicates|sort -r
     </p>
    </blockquote>
    <p>
     可以看到
    </p>
    <p>
     <img alt="" height="440" src="https://i-blog.csdnimg.cn/direct/c2b7e3319ae84f148ff0b00daf9c0ac2.png" width="556"/>
    </p>
    <p>
     选择指定版本安装
    </p>
    <p>
     安装，下面是两个版本的
    </p>
    <blockquote>
     <p>
      yum install docker-ce-18.06.1.ce-3.el7 docker-ce-selinux-18.06.1.ce-3.el7 -y
     </p>
     <p>
      yum install docker-ce-26.1.3-1.el7 docker-ce-selinux-26.1.3-1.el7 -y
     </p>
    </blockquote>
    <p>
     4.启动 Docker-CE
    </p>
    <blockquote>
     <p>
      systemctl enable docker ##设置自动启动
     </p>
     <p>
      systemctl start docker
     </p>
    </blockquote>
    <p>
     5.[可选] 为 Docker 建立用户组
    </p>
    <p>
     docker 命令与 Docker 引擎通讯之间通过 UnixSocket ，但是能够有权限访问 UnixSocket 的用户只有 root 和 docker 用户组的用户才能够进行访问，所以我们需要建立一个 docker 用户组，并且将需要访问 docker 的用户添加到这一个用户组当中来。
    </p>
    <p>
     1. 建立 Docker 用户组
    </p>
    <blockquote>
     <p>
      sudo groupadd docker
     </p>
    </blockquote>
    <p>
     2.添加当前用户到 docker 组
    </p>
    <blockquote>
     <p>
      sudo usermod -aG docker $USER
     </p>
    </blockquote>
    <p>
     6.镜像加速配置
    </p>
    <p>
     这里使用的是
     <a href="https://cr.console.aliyun.com/#/accelerator" rel="nofollow" title="阿里云提供的镜像加速">
      阿里云提供的镜像加速
     </a>
     ，登录并且设置密码之后在左侧的 Docker Hub 镜像站点 可以找到专属加速器地址，复制下来。
    </p>
    <p>
     然后执行以下命令：
    </p>
    <blockquote>
     <p>
      mkdir -p /etc/docker
     </p>
    </blockquote>
    <p>
     进入目录 vi daemon.json 写入如下内容保存
    </p>
    <pre><code class="hljs">{
  "registry-mirrors":
   [
     "https://docker.mirrors.sjtug.sjtu.edu.cn",
      "https://docker.m.daocloud.io",
      "https://docker.itelyou.cf",
      "https://noohub.ru",
      "https://docker.fxxk.dedyn.io",
      "https://huecker.io",
      "https://dockerhub.timeweb.cloud",  
      "https://registry.cn-hangzhou.aliyuncs.com"
   ]
}</code></pre>
    <p>
     之后重新加载配置，并且重启 Docker 服务
    </p>
    <blockquote>
     <p>
      systemctl daemon-reload
     </p>
     <p>
      systemctl restart docker
     </p>
    </blockquote>
    <h4>
     二、配置 Docker 容器与镜像
    </h4>
    <p>
     1.拉取镜像
    </p>
    <p>
     拉取微软在 Docker 上的 aspnetcore 镜像:
    </p>
    <blockquote>
     <p>
      docker pull microsoft/aspnetcore
     </p>
    </blockquote>
    <p>
     之后执行以下命令查看是否拉取成功:
    </p>
    <blockquote>
     <p>
      docker images
     </p>
    </blockquote>
    <p>
     <img alt="" height="492" src="https://i-blog.csdnimg.cn/direct/59a0f27703f94d58a26aee88dab6bd8f.png" width="707"/>
    </p>
    <h4>
     三、Docker 常用命令
    </h4>
    <p>
     1.拉取镜像
    </p>
    <blockquote>
     <p>
      docker pull
     </p>
    </blockquote>
    <p>
     2.删除容器
    </p>
    <blockquote>
     <p>
      docker rm 容器名称
     </p>
    </blockquote>
    <p>
     3.查看容器日志
    </p>
    <blockquote>
     <p>
      docker logs -f
     </p>
    </blockquote>
    <p>
     4.查看正在运行的容器
    </p>
    <blockquote>
     <p>
      docker ps
     </p>
     <p>
      docker ps -a # 为查看所有的容器，包括已经停止的。
     </p>
    </blockquote>
    <p>
     5.删除所有容器
    </p>
    <blockquote>
     <p>
      docker rm $(docker ps -a -q)
     </p>
    </blockquote>
    <p>
     6.停止、启动、杀死指定容器
    </p>
    <blockquote>
     <p>
      docker start # 启动容器
     </p>
     <p>
      docker stop # 启动容器
     </p>
     <p>
      docker kill # 杀死容器
     </p>
    </blockquote>
    <p>
     7.查看所有镜像
    </p>
    <blockquote>
     <p>
      docker images
     </p>
    </blockquote>
    <p>
     8.拉取镜像
    </p>
    <blockquote>
     <p>
      docker pull
     </p>
     <p>
      # 例如以下代码
     </p>
     <p>
      docker pull sameersbn/redmine:latest
     </p>
    </blockquote>
    <p>
     9.后台运行
    </p>
    <blockquote>
     <p>
      docker run -d
     </p>
     <p>
      # 例如 docker run -d -p 127.0.0.1:33301:22 centos6-ssh
     </p>
    </blockquote>
    <p>
     10.暴露端口
    </p>
    <p>
     # 一共有三种形式进行端口映射
    </p>
    <blockquote>
     <p>
      docker -p ip:hostPort:containerPort  # 映射指定地址的主机端口到容器端口
     </p>
     <p>
      # 例如：docker -p 127.0.0.1:3306:3306 映射本机3306端口到容器的3306端口
     </p>
    </blockquote>
    <blockquote>
     <p>
      docker -p ip::containerPort # 映射指定地址的任意可用端口到容器端口
     </p>
     <p>
      # 例如：docker -p 127.0.0.1::3306 映射本机的随机可用端口到容器3306端口
     </p>
    </blockquote>
    <blockquote>
     <p>
      docer -p hostPort:containerPort # 映射本机的指定端口到容器的指定端口
     </p>
     <p>
      # 例如：docker -p 3306:3306 # 映射本机的3306端口到容器的3306端口
     </p>
    </blockquote>
    <p>
     11.映射数据卷
    </p>
    <blockquote>
     <p>
      docker -v /home/data:/opt/data
     </p>
     <p>
      # 这里/home/data 指的是宿主机的目录地址，后者则是容器的目录地址
     </p>
    </blockquote>
    <p>
     四、GUI 管理配置
    </p>
    <p>
     这里推荐使用 Portainer 作为容器的 GUI 管理方案。
    </p>
    <p>
     官方地址：
     <a href="https://portainer.io/install.html" rel="nofollow" title="https://portainer.io/install.html">
      https://portainer.io/install.html
     </a>
    </p>
    <p>
     安装命令：
    </p>
    <pre><code class="hljs">docker volume create portainer_data
docker run -d -p 9000:9000 -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer</code></pre>
    <p>
     访问你的 IP:9000 即可进入容器管理页面。
    </p>
