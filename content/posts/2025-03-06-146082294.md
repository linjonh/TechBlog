    <h1 class="title-article" id="articleContentId">
     react中NavLink和a标签区别
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     在 React 中，
     <code>
      NavLink
     </code>
     和
     <code>
      &lt;a&gt;
     </code>
     标签的主要区别在于它们的用途和行为：
    </p>
    <h4>
     1.
     <strong>
      <code>
       &lt;NavLink&gt;
      </code>
      （来自
      <code>
       react-router-dom
      </code>
      ）
     </strong>
    </h4>
    <p>
     <code>
      NavLink
     </code>
     是
     <code>
      react-router-dom
     </code>
     提供的组件，主要用于
     <strong>
      单页面应用（SPA）中的导航
     </strong>
     ，它的核心功能是
     <strong>
      避免页面刷新
     </strong>
     并且可以
     <strong>
      动态添加样式
     </strong>
     。
    </p>
    <h5>
     <strong>
      特点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       不会触发页面刷新
      </strong>
      ，仅更新 URL 并重新渲染相关组件（依赖
      <code>
       react-router
      </code>
      ）。
     </li>
     <li>
      <strong>
       支持
       <code>
        activeClassName
       </code>
       /
       <code>
        style
       </code>
      </strong>
      ，可以根据当前路由动态添加样式，适用于高亮当前选中的导航项。
     </li>
     <li>
      <strong>
       内部使用的是
       <code>
        &lt;a&gt;
       </code>
       标签
      </strong>
      ，但默认会阻止默认跳转行为，改为
      <code>
       history.push()
      </code>
      。
     </li>
    </ul>
    <h5>
     <strong>
      示例
     </strong>
    </h5>
    <pre><code>import { NavLink } from "react-router-dom";

&lt;NavLink to="/about" activeClassName="active" exact&gt;
  关于我们
&lt;/NavLink&gt;
</code></pre>
    <blockquote>
     <p>
      访问
      <code>
       /about
      </code>
      时，该
      <code>
       &lt;NavLink&gt;
      </code>
      会自动添加
      <code>
       active
      </code>
      类名。
     </p>
    </blockquote>
    <hr/>
    <h4>
     2.
     <strong>
      <code>
       &lt;a&gt;
      </code>
      标签
     </strong>
    </h4>
    <p>
     <code>
      &lt;a&gt;
     </code>
     是标准 HTML 标签，适用于
     <strong>
      传统页面跳转
     </strong>
     ，即：
    </p>
    <ul>
     <li>
      <strong>
       触发完整页面刷新
      </strong>
      ，浏览器重新加载 HTML、CSS、JS 资源。
     </li>
     <li>
      <strong>
       可以跳转到外部链接
      </strong>
      ，例如
      <code>
       https://www.google.com
      </code>
      。
     </li>
     <li>
      <strong>
       适用于非 SPA 站点或下载资源
      </strong>
      。
     </li>
    </ul>
    <h5>
     <strong>
      示例
     </strong>
    </h5>
    <pre><code>&lt;a href="/about"&gt;关于我们&lt;/a&gt;
</code></pre>
    <blockquote>
     <p>
      点击后，整个页面会刷新，浏览器请求
      <code>
       /about
      </code>
      页面。
     </p>
    </blockquote>
    <hr/>
    <h4>
     <strong>
      区别总结
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        对比项
       </th>
       <th>
        <code>
         &lt;NavLink&gt;
        </code>
       </th>
       <th>
        <code>
         &lt;a&gt;
        </code>
        标签
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        是否刷新页面
       </td>
       <td>
        ❌ 不刷新，仅更新路由
       </td>
       <td>
        ✅ 刷新整个页面
       </td>
      </tr>
      <tr>
       <td>
        是否适用于 SPA
       </td>
       <td>
        ✅ 是，依赖
        <code>
         react-router-dom
        </code>
       </td>
       <td>
        ❌ 不是，传统方式
       </td>
      </tr>
      <tr>
       <td>
        是否支持动态样式
       </td>
       <td>
        ✅
        <code>
         activeClassName
        </code>
        /
        <code>
         style
        </code>
       </td>
       <td>
        ❌ 需要手动添加 CSS
       </td>
      </tr>
      <tr>
       <td>
        是否支持外部链接
       </td>
       <td>
        ❌ 仅适用于内部路由
       </td>
       <td>
        ✅ 可跳转到外部 URL
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      👉 什么时候用？
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       SPA（单页面应用）导航
      </strong>
      ：用
      <code>
       &lt;NavLink&gt;
      </code>
      ，避免刷新，保持页面状态。
     </li>
     <li>
      <strong>
       跳转到外部网站或下载文件
      </strong>
      ：用
      <code>
       &lt;a&gt;
      </code>
      ，需要完整跳转或下载资源。
     </li>
    </ul>
    <p>
     如果你需要在
     <code>
      NavLink
     </code>
     中跳转到外部链接，可以使用
     <code>
      a
     </code>
     代替，或者：
    </p>
    <pre><code>&lt;a href="https://www.google.com" target="_blank" rel="noopener noreferrer"&gt;Google&lt;/a&gt;
</code></pre>
    <p>
     这样可以在新标签页打开外部链接，同时防止
     <code>
      noopener
     </code>
     相关的安全风险。
    </p>
    <hr/>
    <p>
     <strong>
      🚀 推荐用法
     </strong>
     在 React 项目中：
    </p>
    <ul>
     <li>
      <strong>
       路由内部跳转
      </strong>
      👉 用
      <code>
       NavLink
      </code>
     </li>
     <li>
      <strong>
       外部链接或文件下载
      </strong>
      👉 用
      <code>
       &lt;a&gt;
      </code>
     </li>
    </ul>
