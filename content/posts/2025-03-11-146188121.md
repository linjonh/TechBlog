    <h2>
     <a id="01_linux_0">
     </a>
     01 linux概述
    </h2>
    <p>
     Linux 是一个开源的
     <strong>
      操作系统内核
     </strong>
     ，诞生于 1991 年，由芬兰程序员 Linus Torvalds 开发。它的核心特点是
     <strong>
      自由、开放、高度可定制
     </strong>
     ，并广泛应用于服务器、开发、嵌入式设备等领域。
    </p>
    <hr/>
    <h4>
     <a id="1_Linux__5">
     </a>
     <strong>
      1. Linux 是什么？
     </strong>
    </h4>
    <p>
     •
     <strong>
      操作系统内核
     </strong>
     ：Linux 本质是一个管理计算机硬件和软件资源的“核心程序”（称为
     <strong>
      内核
     </strong>
     ），负责处理内存、CPU、设备驱动等底层任务。
     <br/>
     •
     <strong>
      Linux 发行版
     </strong>
     ：普通人接触的 Linux 通常是“发行版”（如 Ubuntu、CentOS、Debian），它们在内核基础上打包了图形界面、软件工具等，形成完整的操作系统。
     <br/>
     •
     <strong>
      开源与自由
     </strong>
     ：Linux 遵循
     <strong>
      GPL 协议
     </strong>
     ，允许用户自由使用、修改、分发代码，这与 Windows/macOS 的闭源模式截然不同。
    </p>
    <hr/>
    <h4>
     <a id="2_Linux__12">
     </a>
     <strong>
      2. Linux 的独特优势
     </strong>
    </h4>
    <p>
     •
     <strong>
      稳定性
     </strong>
     ：Linux 服务器可长时间运行不崩溃（比如运行数年无需重启）。
     <br/>
     •
     <strong>
      安全性
     </strong>
     ：开源特性让漏洞更容易被发现和修复，且病毒较少。
     <br/>
     •
     <strong>
      资源占用低
     </strong>
     ：适合老旧设备或嵌入式系统（如路由器、智能手表）。
     <br/>
     •
     <strong>
      强大的命令行
     </strong>
     ：通过终端（Terminal）可以高效完成复杂任务（如批量处理文件、自动化脚本）。
     <br/>
     •
     <strong>
      软件生态丰富
     </strong>
     ：支持大量免费开源软件，开发者工具（如 Python、GCC）尤其完善。
    </p>
    <hr/>
    <h4>
     <a id="3_Linux__21">
     </a>
     <strong>
      3. Linux 的常见用途
     </strong>
    </h4>
    <p>
     •
     <strong>
      服务器领域
     </strong>
     ：全球超 90% 的互联网服务器运行 Linux（如网站、数据库、云计算平台）。
     <br/>
     •
     <strong>
      开发环境
     </strong>
     ：程序员常用 Linux 进行编程、部署（支持 Docker/Kubernetes 等工具）。
     <br/>
     •
     <strong>
      嵌入式设备
     </strong>
     ：智能家电、车载系统、路由器等底层系统往往是 Linux。
     <br/>
     •
     <strong>
      科学计算
     </strong>
     ：科研机构用 Linux 处理大数据、高性能计算（HPC）。
    </p>
    <hr/>

     </li>

       </strong>

       • 修改系统配置前备份
       <code>
        /etc
       </code>
       下的文件。
       <br/>
       • 自定义服务配置文件通常放在
       <code>
        /etc
       </code>
       的子目录（如
       <code>
        /etc/nginx
       </code>
       ）。
      </p>

    <h4>
     <a id="FAQ_272">
     </a>
     <strong>
      常见问题（FAQ）
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        如何查看命令的详细用法？
       </strong>
       <br/>
       • 使用
       <code>
        man 命令名
       </code>
       （如
       <code>
        man grep
       </code>
       ）或
       <code>
        命令名 --help
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        如何强制终止无响应的进程？
       </strong>
       <br/>
       • 先用
       <code>
        ps -aux | grep 进程名
       </code>
       找到 PID，再用
       <code>
        kill -9 PID
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        如何实时监控日志文件？
       </strong>
       <br/>
       • 使用
       <code>
        tail -f /var/log/syslog
       </code>
       （Ubuntu）或
       <code>
        tail -f /var/log/messages
       </code>
       （CentOS）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        如何统计文件行数？
       </strong>
       <br/>
       • 使用
       <code>
        wc -l file.txt
       </code>
       。

     </strong>
    </h4>
    <h5>
     <a id="1_ls__290">
     </a>
     <strong>
      1.
      <code>
       ls
      </code>
      命令
     </strong>
    </h5>
    <p>
     •
     <strong>
      用途
     </strong>
     ：列出文件/目录的基本属性。
     <br/>
     •
     <strong>
      常用参数
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">ls</span> <span class="token parameter variable">-l</span>          <span class="token comment"># 查看详细信息（权限、所有者、大小、时间等）</span>
<span class="token function">ls</span> <span class="token parameter variable">-la</span>         <span class="token comment"># 显示所有文件（包括隐藏文件）</span>
<span class="token function">ls</span> <span class="token parameter variable">-lh</span>         <span class="token comment"># 以人类可读格式显示文件大小（如 KB/MB）</span>
</code></pre>
    <p>
     •
     <strong>
      输出示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash">-rwxr-xr-- <span class="token number">1</span> user group <span class="token number">1</span>.2K Sep <span class="token number">10</span> <span class="token number">10</span>:00 file.txt
</code></pre>
    <p>
     •
     <strong>
      字段解释
     </strong>
     ：
     <br/>
     ◦
     <code>
      -rwxr-xr--
     </code>
     ：文件类型与权限（见下文）。
     <br/>
     ◦
     <code>
      1
     </code>
     ：硬链接数量。
     <br/>
     ◦
     <code>
      user
     </code>
     ：文件所有者。
     <br/>
     ◦
     <code>
      group
     </code>
     ：文件所属组。
     <br/>
     ◦
     <code>
      1.2K
     </code>
     ：文件大小。
     <br/>
     ◦
     <code>
      Sep 10 10:00
     </code>
     ：最后修改时间。
     <br/>
     ◦
     <code>
      file.txt
     </code>
     ：文件名。
    </p>
    <h5>
     <a id="2_stat__311">
     </a>
     <strong>
      2.
      <code>
       stat
      </code>
      命令
     </strong>
    </h5>
    <p>
     •
     <strong>
      用途
     </strong>
     ：显示文件的详细属性（包括 inode、时间戳等）。
     <br/>
     •
     <strong>
      示例
     </strong>

    </p>
    <pre><code class="prism language-bash"><span class="token function">stat</span> file.txt
</code></pre>
    <p>
     •
     <strong>
      输出关键字段
     </strong>
     ：
     <br/>
     •
     <code>
      Access
     </code>
     ：最后访问时间。
     <br/>
     •
     <code>
      Modify
     </code>
     ：最后修改时间（文件内容变更）。
     <br/>
     •
     <code>
      Change
     </code>
     ：最后状态变更时间（如权限或所有者修改）。
     <br/>
     •
     <code>
      Inode
     </code>

    </p>

    <h4>
     <a id="_325">
     </a>
     <strong>
      二、文件权限管理
     </strong>
    </h4>
    <h5>
     <a id="1__327">
     </a>
     <strong>
      1. 权限符号说明
     </strong>
    </h5>
    <p>
     •
     <strong>
      权限表示
     </strong>
     ：
     <code>
      rwxr-xr--
     </code>
     分为三组：
     <br/>
     •
     <strong>
      前 3 位
     </strong>
     ：所有者权限（
     <code>
      u
     </code>
     ）。
     <br/>
     •
     <strong>
      中 3 位
     </strong>
     ：所属组权限（
     <code>
      g
     </code>
     ）。
     <br/>
     •
     <strong>
      后 3 位
     </strong>
     ：其他用户权限（
     <code>
      o
     </code>
     ）。
     <br/>
     •
     <strong>
      符号含义
     </strong>
     ：
     <br/>
     •
     <code>
      r
     </code>
     （读）：文件可读取，目录可列出内容。
     <br/>
     •
     <code>
      w
     </code>
     （写）：文件可修改，目录可创建/删除文件。
     <br/>
     •
     <code>
      x
     </code>
     （执行）：文件可执行，目录可进入。
    </p>
    <h5>
     <a id="2_chmod_337">
     </a>
     <strong>
      2. 修改权限：
      <code>
       chmod
      </code>
     </strong>
    </h5>
    <p>
     •
     <strong>
      数字模式
     </strong>
     （推荐）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> <span class="token number">755</span> file.txt    <span class="token comment"># rwxr-xr-x</span>
<span class="token function">chmod</span> <span class="token parameter variable">-R</span> <span class="token number">644</span> dir/     <span class="token comment"># 递归修改目录内所有文件权限</span>
</code></pre>
    <p>
     •
     <strong>
      数字对应权限
     </strong>
     ：
     <br/>
     ◦
     <code>
      4
     </code>
     = r（读）
     <br/>
     ◦
     <code>
      2
     </code>
     = w（写）
     <br/>
     ◦
     <code>
      1
     </code>
     = x（执行）
     <br/>
     ◦ 示例：
     <code>
      7 = 4+2+1
     </code>
     （rwx），
     <code>
      6 = 4+2
     </code>
     （rw-）。
    </p>
    <p>
     •
     <strong>
      符号模式
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> u+x script.sh    <span class="token comment"># 给所有者添加执行权限</span>
<span class="token function">chmod</span> go-w file.txt    <span class="token comment"># 移除组和其他用户的写权限</span>
<span class="token function">chmod</span> <span class="token assign-left variable">a</span><span class="token operator">=</span>r file.txt     <span class="token comment"># 所有用户权限设为只读（a = all）</span>
</code></pre>
    <h5>
     <a id="3_chown_356">
     </a>
     <strong>
      3. 修改所有者和所属组：
      <code>
       chown
      </code>
     </strong>
    </h5>
    <p>
     •
     <strong>
      修改所有者
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chown</span> user file.txt        <span class="token comment"># 修改文件所有者</span>
<span class="token function">chown</span> <span class="token parameter variable">-R</span> user:group dir/   <span class="token comment"># 递归修改目录所有者及所属组</span>
</code></pre>
    <p>
     •
     <strong>
      仅修改所属组
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chgrp</span> group file.txt       <span class="token comment"># 修改文件所属组</span>
</code></pre>

    <h2>
     <a id="06__602">
     </a>
     06 硬链接和软链接
    </h2>
    <p>
     在 Linux/Unix 系统中，
     <strong>
      硬链接（Hard Link）
     </strong>
     和
     <strong>
      软链接（Symbolic Link，也称软连接）
     </strong>
     是两种不同的文件链接方式，用于实现文件的多路径访问和管理。
    </p>
    <hr/>
    <h4>
     <a id="Hard_Link_607">
     </a>
     <strong>
      一、硬链接（Hard Link）
     </strong>
    </h4>
    <h5>
     <a id="1__609">
     </a>
     <strong>
      1. 定义
     </strong>
    </h5>
    <p>
     •
     <strong>
      硬链接
     </strong>
     是文件系统中
     <strong>
      指向同一 inode 的多个文件名
     </strong>
     ，所有硬链接共享相同的物理数据块。
     <br/>
     •
     <strong>
      本质
     </strong>
     ：同一文件的多个入口（别名），删除原文件不会影响硬链接的可用性。
    </p>
    <h5>
     <a id="2__613">
     </a>
     <strong>
      2. 创建方法
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">ln</span> 原文件 硬链接名
</code></pre>
    <p>
     •
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">ln</span> file.txt hard_link.txt
</code></pre>
    <h5>
     <a id="3__622">
     </a>
     <strong>
      3. 特点
     </strong>
    </h5>
    <p>
     •
     <strong>
      共享 inode
     </strong>
     ：所有硬链接和原文件具有相同的 inode 号。
     <br/>
     •
     <strong>
      同步更新
     </strong>
     ：修改任一硬链接或原文件，所有链接内容同步变化。
     <br/>
     •
     <strong>
      独立性
     </strong>
     ：删除原文件后，硬链接仍有效（仅减少 inode 的引用计数）。
     <br/>
     •
     <strong>
      限制
     </strong>
     ：
     <br/>
     • 无法跨不同文件系统（如从
     <code>
      /dev/sda1
     </code>
     链接到
     <code>
      /dev/sdb1
     </code>
     ）。
     <br/>
     • 无法链接目录（防止循环引用导致系统问题）。
    </p>
    <h5>
     <a id="4__630">
     </a>
     <strong>
      4. 使用场景
     </strong>
    </h5>
    <p>
     • 防止重要文件被误删（通过多个硬链接保留访问入口）。
     <br/>
     • 共享大型文件（节省磁盘空间，避免重复存储）。
    </p>
    <h5>
     <a id="5__634">
     </a>
     <strong>
      5. 查看硬链接
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">ls</span> <span class="token parameter variable">-i</span> file.txt hard_link.txt  <span class="token comment"># 查看 inode 号</span>
<span class="token function">stat</span> file.txt                <span class="token comment"># 查看链接数（Links 字段）</span>
</code></pre>
    <hr/>
    <h4>
     <a id="Symbolic_Link_642">
     </a>
     <strong>
      二、软链接（Symbolic Link）
     </strong>
    </h4>
    <h5>
     <a id="1__644">
     </a>
     <strong>
      1. 定义
     </strong>
    </h5>
    <p>
     •
     <strong>
      软链接
     </strong>
     是
     <strong>
      独立文件
     </strong>
     ，存储目标文件的路径信息，类似 Windows 的快捷方式。
     <br/>
     •
     <strong>
      本质
     </strong>
     ：指向另一个文件的指针，依赖目标文件的存在。
    </p>
    <h5>
     <a id="2__648">
     </a>
     <strong>
      2. 创建方法
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">ln</span> <span class="token parameter variable">-s</span> 目标文件 软链接名
</code></pre>
    <p>
     •
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">ln</span> <span class="token parameter variable">-s</span> /var/log/app.log app_log_link
</code></pre>
    <h5>
     <a id="3__657">
     </a>
     <strong>
      3. 特点
     </strong>
    </h5>
    <p>
     •
     <strong>
      独立 inode
     </strong>
     ：软链接有自己的 inode 号，与目标文件不同。
     <br/>
     •
     <strong>
      路径依赖
     </strong>
     ：目标文件被删除后，软链接失效（成为“悬空链接”）。
     <br/>
     •
     <strong>
      灵活性
     </strong>
     ：
     <br/>
     • 可跨不同文件系统和磁盘分区。
     <br/>
     • 可链接目录（如
     <code>
      /usr/bin/python3 -&gt; /usr/bin/python3.9
     </code>
     ）。
     <br/>
     •
     <strong>
      权限无关
     </strong>
     ：软链接权限始终为
     <code>
      lrwxrwxrwx
     </code>
     ，实际权限由目标文件决定。
    </p>
    <h5>
     <a id="4__665">
     </a>
     <strong>
      4. 使用场景
     </strong>
    </h5>
    <p>
     • 为长路径文件创建快捷方式（如链接到深层目录的配置文件）。
     <br/>
     • 动态切换版本（如
     <code>
      /usr/bin/python
     </code>
     指向不同 Python 版本）。
     <br/>
     • 跨文件系统引用（如将数据目录链接到外部存储）。
    </p>
    <h5>
     <a id="5__670">
     </a>
     <strong>
      5. 查看软链接
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">ls</span> <span class="token parameter variable">-l</span> 软链接名          <span class="token comment"># 显示指向路径（例如：app_log_link -&gt; /var/log/app.log）</span>
readlink 软链接名       <span class="token comment"># 直接输出目标路径</span>
</code></pre>

    <h2>
     <a id="09__1047">
     </a>
     09 用户组
    </h2>
    <h4>
     <a id="_1049">
     </a>
     <strong>
      一、用户组的作用
     </strong>
    </h4>
    <p>
     •
     <strong>
      权限共享
     </strong>
     ：通过组批量管理用户对文件/目录的访问权限。
     <br/>
     •
     <strong>
      简化管理
     </strong>
     ：替代为每个用户单独设置权限，提升效率。
     <br/>
     •
     <strong>
      安全隔离
     </strong>
     ：限制非授权用户访问敏感资源。
    </p>
    <hr/>
    <h4>
     <a id="_1056">
     </a>
     <strong>
      二、组类型
     </strong>
    </h4>
    <h5>
     <a id="1_Primary_Group_1057">
     </a>
     <strong>
      1. 主组（Primary Group）
     </strong>
    </h5>
    <p>
     • 每个用户必须属于
     <strong>
      一个主组
     </strong>
     。
     <br/>
     • 用户创建文件时，文件的默认所属组为其主组。
     <br/>
     •
     <strong>
      相关文件
     </strong>
     ：
     <code>
      /etc/passwd
     </code>
     中用户行的第4字段（GID）。
    </p>
    <h5>
     <a id="2_Supplementary_Group_1062">
     </a>
     <strong>
      2. 附加组（Supplementary Group）
     </strong>
    </h5>
    <p>
     • 用户可加入多个附加组，获得额外的权限。
     <br/>
     •
     <strong>
      相关文件
     </strong>
     ：
     <code>
      /etc/group
     </code>
     中组的最后一个字段（成员列表）。
    </p>
    <hr/>
    <h4>
     <a id="_1068">
     </a>
     <strong>
      三、组管理命令
     </strong>
    </h4>
    <h5>
     <a id="1__1070">
     </a>
     <strong>
      1. 创建组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">groupadd</span> 组名           <span class="token comment"># 创建普通组</span>
<span class="token function">sudo</span> <span class="token function">groupadd</span> <span class="token parameter variable">-g</span> <span class="token number">1005</span> 组名   <span class="token comment"># 指定 GID 创建组</span>
</code></pre>
    <h5>
     <a id="2__1076">
     </a>
     <strong>
      2. 修改组属性
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">groupmod</span> <span class="token parameter variable">-n</span> 新组名 旧组名   <span class="token comment"># 重命名组</span>
<span class="token function">sudo</span> <span class="token function">groupmod</span> <span class="token parameter variable">-g</span> <span class="token number">1006</span> 组名       <span class="token comment"># 修改组的 GID</span>
</code></pre>
    <h5>
     <a id="3__1082">
     </a>
     <strong>
      3. 删除组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">groupdel</span> 组名    <span class="token comment"># 删除空组（无成员）</span>
<span class="token function">sudo</span> <span class="token function">groupdel</span> <span class="token parameter variable">-f</span> 组名 <span class="token comment"># 强制删除非空组（慎用！）</span>
</code></pre>
    <h5>
     <a id="4__1088">
     </a>
     <strong>
      4. 查看组信息

<span class="token function">cat</span> /etc/group <span class="token operator">|</span> <span class="token function">grep</span> 组名  <span class="token comment"># 过滤组配置</span>

    <h4>
     <a id="_1096">
     </a>
     <strong>
      四、用户与组的关联操作
     </strong>
    </h4>
    <h5>
     <a id="1__1098">
     </a>
     <strong>
      1. 创建用户时指定组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">useradd</span> <span class="token parameter variable">-g</span> 主组名 <span class="token parameter variable">-G</span> 附加组1,附加组2 用户名

    <p>
     •
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">useradd</span> <span class="token parameter variable">-g</span> developers <span class="token parameter variable">-G</span> docker,www-data alice
</code></pre>
    <h5>
     <a id="2__1107">
     </a>
     <strong>
      2. 将用户加入/移出附加组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">usermod</span> <span class="token parameter variable">-aG</span> 附加组名 用户名   <span class="token comment"># 将用户加入组（-a 表示追加）</span>
<span class="token function">sudo</span> gpasswd <span class="token parameter variable">-d</span> 用户名 组名       <span class="token comment"># 将用户移出组</span>
</code></pre>

     <a id="3__1113">
     </a>
     <strong>
      3. 查看用户所属的组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">groups</span> 用户名    <span class="token comment"># 显示用户的主组和附加组</span>
<span class="token function">id</span> 用户名        <span class="token comment"># 显示 UID、GID 及所有组</span>
</code></pre>
    <h5>
     <a id="4__1119">
     </a>
     <strong>
      4. 临时切换用户的主组
     </strong>
    </h5>
    <pre><code class="prism language-bash">newgrp 组名     <span class="token comment"># 切换后创建的文件默认属于该组（需用户已加入该组）</span>
</code></pre>

    <h4>
     <a id="_1126">
     </a>
     <strong>
      五、文件与组的权限控制
     </strong>
    </h4>
    <h5>
     <a id="1__1128">
     </a>
     <strong>
      1. 修改文件所属组
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">chgrp</span> 组名 文件或目录    <span class="token comment"># 修改所属组</span>
<span class="token function">sudo</span> <span class="token function">chown</span> :组名 文件或目录   <span class="token comment"># 等效于 chgrp</span>
</code></pre>
    <h5>
     <a id="2__SetGID__1134">
     </a>
     <strong>
      2. 设置目录的 SetGID 位
     </strong>
    </h5>
    <p>
     •
     <strong>
      作用
     </strong>
     ：目录中新创建的文件自动继承父目录的所属组。
     <br/>
     •
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">chmod</span> g+s /shared_dir    <span class="token comment"># 添加 SetGID 位</span>
<span class="token function">ls</span> <span class="token parameter variable">-ld</span> /shared_dir            <span class="token comment"># 查看权限（drwxr-sr-x）</span>
</code></pre>
    <h5>
     <a id="3__1142">
     </a>
     <strong>
      3. 组权限分配示例
     </strong>
    </h5>
    <p>
     •
     <strong>
      场景
     </strong>
     ：团队协作目录
     <code>
      /project
     </code>
     ，要求
     <code>
      developers
     </code>
     组内的用户可读写。
     <br/>
     •
     <strong>
      操作
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 创建组和目录</span>
<span class="token function">sudo</span> <span class="token function">groupadd</span> developers
<span class="token function">sudo</span> <span class="token function">mkdir</span> /project

<span class="token comment"># 设置目录所属组和权限</span>
<span class="token function">sudo</span> <span class="token function">chgrp</span> developers /project
<span class="token function">sudo</span> <span class="token function">chmod</span> <span class="token number">770</span> /project      <span class="token comment"># 所有者+组：读写执行，其他用户无权限</span>

<span class="token comment"># 将用户加入组</span>
<span class="token function">sudo</span> <span class="token function">usermod</span> <span class="token parameter variable">-aG</span> developers alice
<span class="token function">sudo</span> <span class="token function">usermod</span> <span class="token parameter variable">-aG</span> developers bob
</code></pre>

    <h2>
     <a id="11_Linux_1306">
     </a>
     11 Linux进程管理
    </h2>
    <h4>
     <a id="1__1308">
     </a>
     <strong>
      1. 进程查看
     </strong>
    </h4>
    <p>
     •
     <strong>
      <code>
       ps
      </code>
     </strong>
     ：查看当前进程快照
    </p>
    <pre><code class="prism language-bash"><span class="token function">ps</span> aux       <span class="token comment"># 查看所有进程详细信息</span>
<span class="token function">ps</span> <span class="token parameter variable">-ef</span>       <span class="token comment"># 标准格式显示进程</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       top
      </code>
     </strong>
     ：实时动态监控进程及资源占用（按
     <code>
      q
     </code>
     退出）
     <br/>
     •
     <strong>
      <code>
       htop
      </code>
     </strong>
     ：增强版
     <code>
      top
     </code>
     （需安装，支持鼠标操作）
     <br/>
     •
     <strong>
      <code>
       pstree
      </code>
     </strong>
     ：以树形结构显示进程关系
    </p>
    <hr/>
    <h4>
     <a id="2__1320">
     </a>
     <strong>
      2. 进程终止
     </strong>
    </h4>
    <p>
     •
     <strong>
      <code>
       kill
      </code>
     </strong>
     ：通过 PID 终止进程
    </p>
    <pre><code class="prism language-bash"><span class="token function">kill</span> <span class="token parameter variable">-9</span> PID    <span class="token comment"># 强制终止进程</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       killall
      </code>
     </strong>
     ：通过进程名终止所有同名进程
    </p>
    <pre><code class="prism language-bash"><span class="token function">killall</span> nginx
</code></pre>
    <p>
     •
     <strong>
      <code>
       pkill
      </code>
     </strong>
     ：通过模式匹配终止进程
    </p>
    <pre><code class="prism language-bash"><span class="token function">pkill</span> <span class="token parameter variable">-f</span> <span class="token string">"python script.py"</span>
</code></pre>

    <h4>
     <a id="3__1336">

     <strong>

     </strong>
    </h4>
    <p>
     •
     <strong>
      <code>
       pgrep
      </code>

</code></pre>

     •
     <strong>
      <code>
       lsof
      </code>

    <pre><code class="prism language-bash"><span class="token function">lsof</span> <span class="token parameter variable">-i</span> :80    <span class="token comment"># 查看占用 80 端口的进程</span>

    <h4>
     <a id="4__1348">

     <strong>
      4. 优先级调整
     </strong>
    </h4>
    <p>
     •
     <strong>
      <code>
       nice
      </code>
     </strong>
     ：启动进程时指定优先级（范围：-20 到 19，默认 0）
    </p>
    <pre><code class="prism language-bash"><span class="token function">nice</span> <span class="token parameter variable">-n</span> <span class="token number">10</span> <span class="token builtin class-name">command</span>   <span class="token comment"># 低优先级启动</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       renice
      </code>
     </strong>
     ：修改运行中进程的优先级
    </p>
    <pre><code class="prism language-bash"><span class="token function">renice</span> <span class="token number">5</span> <span class="token parameter variable">-p</span> PID      <span class="token comment"># 将 PID 进程优先级改为 5</span>
</code></pre>
    <hr/>
    <h4>
     <a id="5__1360">
     </a>
     <strong>
      5. 后台/前台控制
     </strong>
    </h4>
    <p>
     •
     <strong>
      <code>
       &amp;
      </code>
     </strong>
     ：后台运行进程
    </p>
    <pre><code class="prism language-bash">python script.py <span class="token operator">&amp;</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       jobs
      </code>
     </strong>
     ：查看当前终端后台任务
    </p>
    <pre><code class="prism language-bash"><span class="token function">jobs</span> <span class="token parameter variable">-l</span>       <span class="token comment"># 显示任务 ID 和 PID</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       fg
      </code>
     </strong>
     ：将后台任务转到前台
    </p>
    <pre><code class="prism language-bash"><span class="token function">fg</span> %1        <span class="token comment"># 恢复任务 ID 为 1 的进程</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       bg
      </code>
     </strong>
     ：恢复暂停的后台任务
    </p>
    <pre><code class="prism language-bash"><span class="token function">bg</span> %1        <span class="token comment"># 后台继续运行任务 ID 为 1 的进程</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       nohup
      </code>
     </strong>
     ：脱离终端运行进程（忽略挂断信号）
    </p>
    <pre><code class="prism language-bash"><span class="token function">nohup</span> <span class="token builtin class-name">command</span> <span class="token operator">&amp;</span>
</code></pre>
