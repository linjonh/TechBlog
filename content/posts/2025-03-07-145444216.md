    <h2>
     <a id="_0">
     </a>
     概述
    </h2>
    <blockquote>
     <p>
      <strong>
       适配器模式 : Adapter Pattern
      </strong>
      是一种结构型设计模式.
      <br/>
      <strong>
       作用
      </strong>
      ： 使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。
      <br/>
      <strong>
       实现思路
      </strong>
      ： 适配器模式
      <strong>
       通过将一个类的接口转换成客户希望的另外一个接口
      </strong>
      来实现这一点。
      <br/>
      <code>
       这里的“接口”指的是类所提供的方法、属性等成员的集合，并非特指面向对象编程语言中的interface。
      </code>
     </p>
    </blockquote>
    <pre><code>简单理解 ： 适配器模式，就是 中间加一层，把原来的对象包装一下，转换成目标希望的样子。
</code></pre>
    <h2>
     <a id="_9">
     </a>
     角色
    </h2>
    <blockquote>
     <p>
      <strong>
       适配者(Source)
      </strong>
      ：需要被适配的类。适配者包含客户想要使用但是接口与目标不兼容的方法。
      <br/>
      <strong>
       目标(Target)接口
      </strong>
      ：这是客户所期待的接口。在某些情况下，这个角色可能是一个具体类，而不是接口。
      <br/>
      <strong>
       适配器(Adapter)
      </strong>
      ：负责将
      <strong>
       适配者的接口
      </strong>
      转换为
      <strong>
       目标接口
      </strong>
      ，从而使原本因为接口不匹配而无法合作的类能够协同工作。
     </p>
    </blockquote>
    <h2>
     <a id="_14">
     </a>
     分类
    </h2>
    <blockquote>
     <p>
      适配器模式分为三类 ：
      <code>
       类适配器模式
      </code>
      、
      <code>
       对象适配器模式
      </code>
      、
      <code>
       接口适配器模式
      </code>
      。
     </p>
    </blockquote>
    <h3>
     <a id="_17">
     </a>
     类适配器模式
    </h3>
    <blockquote>
     <p>
      <strong>
       在不改变原有类结构的情况下，扩展类的功能，以适配不同的接口时，可以使用【类适配器模式】。
      </strong>
      <br/>
      【
      <strong>
       实现思路
      </strong>
      】 ： 适配器类 继承 原来的类，并实现目标接口。
      <br/>
      【
      <strong>
       实现效果
      </strong>
      】 ： 对原有类的方法，按照 目标接口中的要求进行“包装”，达到适配的效果。
     </p>
    </blockquote>
    <pre><code>以电源为例，标准的输出电压为220V （适配者），
现在需要给手机/手环 （目标接口）充电，手机要求的电压为 5V，手环要求的电压为 2V ，

    <blockquote>

       接口适配模式
      </strong>
      ， 主要应用在
      <code>
       不希望实现所有的接口方法的情况下
      </code>
      。
      <br/>
      <strong>
       1、创建一个抽象类，给接口中的方法一个默认的实现；
      </strong>
      <br/>
      2、然后通过
      <strong>
       继承该抽象类的方式，只重写部分需要的方法
      </strong>
      ，达到简化代码的效果。
      <br/>

     </p>
