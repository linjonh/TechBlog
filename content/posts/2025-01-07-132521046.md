    <h1 class="title-article" id="articleContentId">
     网络运维主要做什么，日常工作有哪些？
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     运维指互联网运维，通常属于技术部门，即运维部。运维的职责是保证服务的稳定运行，考虑服务的可扩展性，从系统的稳定性和可运维性的角度，提出开发需求，定位系统的问题，并对突然出现的问题做到快速响应和处理。运维的职责覆盖了产品从设计到发布、运行维护、变更升级及至下线的生命周期，与研发、测试、系统管理同为互联网产品技术支撑的4大部门。
     <br/>
     <br/>
     运维的重点在于系统运行的各种环境，从机房、网络、存储、物理机、
     <a href="https://www.xinnet.com/tag/7423" rel="nofollow" title="虚拟机">
      虚拟机
     </a>
     这些基础的架构，到数据库、中间件平台、
     <a href="https://www.xinnet.com/tag/11197" rel="nofollow" title="云">
      云
     </a>
     平台、大数据平台，偏重的也不是编程，而是对这类平台的使用和管理。运维的水平可以成为衡量一个公司(IT公司)技术实力的标准。
     <br/>
     <br/>
     一、运维的日常工作：
     <br/>
     <br/>
     1、需要对系统的需求和设计方案进行分析，思考在保证稳定性方面有哪些可以加强的地方，并和系统的研发人员进行有效沟通
     <br/>
     <br/>
     2、使用工具、或者写程序，对运营数据进行分析。
     <br/>
     <br/>
     3、写程序以建立工具或平台，去加强系统的稳定性。
     <br/>
     <br/>
     运维工程师最重要的是会运用编程和软件的方法来解决问题。发展的道路应该和软件工程师没有很大的区别，差异只是关注点和领域方向的不同。
     <br/>
     <br/>
     二、运维需要掌握如下几个衡量指标
     <br/>
     <br/>
     1.平均故障间隔时间( MTBF)
     <br/>
     <br/>
     平均故障间隔时间(MTBF, Mean Time Between Failure)，指相邻两次故障之间的平均工作时间。MTBF通常是衡量一个产品可靠性的指标，这个间隔时间越短说明系统可靠性越差。
     <br/>
     <br/>
     2.平均修复时间(MTTR)
     <br/>
     <br/>
     平均修复时间(MTTR, Mean Time To Repair)，指产品由故障状态转为工作状态时修复时间的平均值，即故障修复所需要的平均时间。MTTR值越低说明故障修复越及时。
    </p>
    <p>
    </p>
    <p>
     3.可用性(Availability)
     <br/>
     <br/>
     可用性是系统架构设计中很重要的衡量指标。根据GB/T3187- -97对可用性的定义，可用性是指在要求的外部资源得到保证的前提下，产品在规定的条件下和规定的时刻或时间区间内处于可执行规定功能状态的能力。它是产品可靠性、维修性和维修保障性的综合反映。
     <br/>
     <br/>
     三、运维是做什么的
     <br/>
     <br/>
     运维的主要职责是在产品生命周期的各个阶段，维护系统的稳定性。运维的职责覆盖了产品从设计到发布、运行维护、变更升级及至下线的生命周期，在产品生产环境各个阶段的职责也不同。
     <br/>
     <br/>
     运维工程在国内也被称作SRE，直接翻译为网站可用性T程。SRFT程师需要具备算法、数据结构、编程能力、网络编程、分布式系统、可扩展架构、故障排除等各方面技能，其核心工作包括容量规划与实施、服务集群维护、系统容错管理、
     <a href="https://www.xinnet.com/tag/11199" rel="nofollow" title="负载均衡">
      负载均衡
     </a>
     、监控系统以及值班等，最终为产品上线后服务的稳定性负责，但是不负责具体的机器运维。
    </p>
