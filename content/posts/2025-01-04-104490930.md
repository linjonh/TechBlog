     <li>
      打开jmeter 习惯配置中文模式 选择简体中文即可
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/be1a24bbbf41992115156fed587cce22.png">
       <br/>
       3.新建一个测试计划（test_plan）并命名。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/81f58b4e6507acd8646b68d6111b634f.png">
        <br/>
        右键添加用户组（线程组）。请求均需要在组下作用。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c067b1181f57152124f750fa8f27c78a.png">
         <br/>
         这里取名200并发，设置线程数为两百。接下来添加请求登录的接口信息。右键添加httprequest
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/27dcc9e181d678754558344bd8269240.png">
          <br/>
          填写接口信息，web接口请求参数信息可以直接使用google开发者工具获取或是其他接口抓取工具如fiddler等。填写以下关键信息，
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9d8aaf60a961fcbe49b49308c20ebb48.png">
           <br/>
           包含请求地址、端口、请求放方式、请求接口路径、请求传参格式和对应参数等。
           <br/>
           4.添加监听器（listerer），便于观察请求运行结果。先选中线程组后右键，选择“监听器”-“结果观察树”
           <br/>
           <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/29b9ce17e0303a145b154958d57bc05c.png">
            <br/>
            5.至此，已成功使用jmeter完成一个系统登录场景的模拟。点击运行即可通过结果观察树查看请求详细情况。
            <br/>
            <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/112c8d1f9ba534e853f166c5537af477.png">
             <br/>
             由于设置200并发，该请求会并发200次请求，产生并记录两百次结果。
             <br/>
             这样就完成了对登录接口的并发请求测试。监听器中还可以配置图标来展示请求结果，如增加“表格观察结果”
             <br/>
             <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d6916dcce84d6b83f0e5b9fdf5357863.png"/>
             <br/>
             6.web性能测试通常需要监听服务端的资源占用情况，除了常用的nmon和dstat等工具命令外。jmeter也提供了插件服务，需要安装。以此实现对服务端的监听。可参考https://www.cnblogs.com/qmfsun/p/5915673.html
