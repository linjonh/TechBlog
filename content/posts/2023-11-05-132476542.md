    <p>
     <strong>
      Linux Ubuntu系统下python2的安装
     </strong>
    </p>
    <p>
     FSL的安装脚本是用Python2写的，新版本的Ubuntu （16以后）在默认情况下没有安装Python2。在终端输入 python2，若提示没有相应的命令，则需要先安装Python2，如下指令：
    </p>
    <pre><code class="prism language-python">sudo apt install python2
</code></pre>
    <p>
     <strong>
      报错1：
     </strong>
     <br/>
     Ubuntu20.04 "sudo apt-get install python2“导致"E: Unable to locate package python2
     <br/>
     <strong>
      Unable to locate package
     </strong>
    </p>
    <p>
     <strong>
      解决方法1：
     </strong>
     <br/>
     Linux Ubuntu系统下python2的安装
     <br/>
     背景：官方早年前已经宣布停止 Python 2 的更新和服务，所以对于ubuntu20版本之后的，都是自带安装了python3，但是我们在处理安装某些大型科学计算程序的时候，往往需要用到一些老版库和程序，此处以python2安装的FSL为例，关于安装python2的报错，我尝试了很多方法都不行：
    </p>
    <pre><code class="prism language-python">sudo apt<span class="token operator">-</span>get update
sudo apt<span class="token operator">-</span>get upgrade
sudo apt<span class="token operator">-</span>get install python2<span class="token punctuation">.</span><span class="token number">7</span>
</code></pre>
    <p>
     结果：Unable to Locate package
    </p>
    <p>
     与此同时尝试更换了各种各样不同的源（阿里源，清华源都还是一样的报错问题）
     <br/>
     但是，往往我们忽略了官方停止更新服务的背景，所以关键问题还是出在源的选择和载入上，正确的解决办法如下：
     <br/>
     <strong>
      启用 universe 源仓库：
     </strong>
    </p>
    <pre><code class="prism language-python">sudo add<span class="token operator">-</span>apt<span class="token operator">-</span>repository universe
</code></pre>
    <p>
     更新源，并安装：
    </p>
    <pre><code class="prism language-python">sudo apt update 
sudo apt install python2
</code></pre>
    <p>
     查看安装是否成功：
    </p>
    <pre><code class="prism language-python">python2 <span class="token operator">-</span><span class="token operator">-</span>version
</code></pre>
    <p>
     这时终端会返回py2的版本号，安装成功！
