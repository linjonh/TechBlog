    <h3>
     <a id="Spring_MVC_5">
     </a>
     一、Spring MVC简介
    </h3>
    <h4>
     <a id="1_Spring_MVC_7">
     </a>
     1. 什么是Spring MVC？
    </h4>
    <p>
     Spring MVC是Spring框架的一个模块，用于构建基于MVC架构的Web应用程序。它的核心思想是将应用程序分为三个层次：
    </p>
    <ul>
     <li>
      <strong>
       模型（Model）
      </strong>
      ：负责管理应用程序的数据和业务逻辑。
     </li>
     <li>
      <strong>
       视图（View）
      </strong>
      ：负责展示数据，通常是HTML页面。
     </li>
     <li>
      <strong>
       控制器（Controller）
      </strong>
      ：负责处理用户输入，更新模型，并选择视图来展示数据。
     </li>
    </ul>
    <p>
     Spring MVC通过
     <strong>
      DispatcherServlet
     </strong>
     作为前端控制器，协调请求的处理流程。
    </p>
    <h4>
     <a id="2_Spring_MVC_15">
     </a>
     2. Spring MVC的优势
    </h4>
    <ul>
     <li>
      <strong>
       灵活性
      </strong>
      ：支持多种视图技术（如JSP、Thymeleaf、FreeMarker）。
     </li>
     <li>
      <strong>
       强大的注解支持
      </strong>
      ：通过注解简化控制器和请求处理的配置。
     </li>
     <li>
      <strong>
       与Spring框架无缝集成
      </strong>
      ：可以轻松使用Spring的其他功能（如依赖注入、AOP）。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="Spring_MVC_22">
     </a>
     二、Spring MVC的核心组件
    </h3>

     </a>
     1. DispatcherServlet
    </h4>
    <p>
     <code>
      DispatcherServlet
     </code>
     是Spring MVC的前端控制器，它负责接收所有的HTTP请求，并将请求分发给相应的处理器（Controller）。它的工作流程如下：
    </p>
    <ol>
     <li>
      接收HTTP请求。
     </li>
     <li>
      调用处理器映射器（HandlerMapping）找到对应的处理器（Controller）。
     </li>
     <li>
      调用处理器适配器（HandlerAdapter）执行处理器。
     </li>
     <li>
      调用视图解析器（ViewResolver）解析视图。
     </li>
     <li>
      返回HTTP响应。
     </li>
    </ol>
