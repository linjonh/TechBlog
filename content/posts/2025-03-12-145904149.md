    <h2>
     <a id="TypeScript__0">
     </a>
     一、
     <code>
      TypeScript
     </code>
     简介：
    </h2>
    <p>
     1、
     <code>
      TypeScript
     </code>
     包含了
     <code>
      javaScript
     </code>
     的所有内容。
     <br/>
     2、新增了静态类型检查、接口、泛型，更适合大型项目的开发。
     <br/>
     3、
     <code>
      TypeScript
     </code>
     需要编译为
     <code>
      javaScript
     </code>
     。
    </p>
    <h2>
     <a id="javaScript__4">
     </a>
     二、
     <code>
      javaScript
     </code>
     的缺陷：
    </h2>
    <p>
     1、数据类型不明确
     <br/>
     2、逻辑有问题，不会报错
     <br/>
     3、访问不存在的属性，不会报错
     <br/>
     4、低级的拼写错误，不会报错
    </p>

    <h2>
     <a id="_633">
     </a>
     六、抽象类：
    </h2>
    <h3>
     <a id="1_634">
     </a>
     1、定义：
    </h3>
    <h4>
     <a id="__635">
     </a>
     ① 抽象类无法被实例化：
    </h4>
    <blockquote>
     <p>
      不能用
      <code>
       new
      </code>
      关键字去调用，
      <mark>
       可以被继承
      </mark>
     </p>
    </blockquote>
    <h4>
     <a id="__637">
     </a>
     ② 抽象类专门用来定义类的结构和行为：
    </h4>
    <blockquote>
     <p>
      定义通用的属性和方法
     </p>
    </blockquote>
    <h4>
     <a id="__639">
     </a>
     ③ 抽象类中可以写抽象方法：
    </h4>
    <blockquote>
     <p>
      定义抽象方法的名字、入参类型和返回值类型
     </p>
    </blockquote>
    <h4>
     <a id="__641">
     </a>
     ④ 抽象类中可以写具体的实现：
    </h4>
    <blockquote>
     <p>
      定义具体方法，在具体方法中使用 抽象方法中的属性 / 抽象方法 来计算或者输出数据
     </p>
    </blockquote>
    <h4>
     <a id="__643">
     </a>
     ⑤ 为派生类（子类）提供一个基础结构：
    </h4>
    <blockquote>
     <p>
      子类共享抽象类中的部分代码
     </p>
    </blockquote>
    <h4>
     <a id="__645">
     </a>
     ⑥ 派生类（子类）必须实现抽象类的抽象方法：
    </h4>
    <blockquote>
     <p>
      子类必须定义一个和抽象类中抽象方法同名的方法，并且要定义该方法的返回值
     </p>
    </blockquote>
    <h3>
     <a id="2_647">
     </a>
     2、使用场景：
    </h3>
    <h4>
     <a id="__648">
     </a>
     ① 定义通用接口：
    </h4>
    <blockquote>
     <p>
      为一组相关的类定义通用的行为(方法或属性)时。
     </p>
    </blockquote>
    <h4>
     <a id="__650">
     </a>
     ② 提供基础实现：
    </h4>
    <blockquote>
     <p>
      在抽象类中提供某些方法或为其提供基础实现，这样派生类就可以继承这些实现。
     </p>
    </blockquote>
    <h4>
     <a id="__652">
     </a>
     ③ 确保关键实现：
    </h4>
    <blockquote>
     <p>
      强制派生类实现一些关键行为。
     </p>
    </blockquote>
    <h4>
     <a id="__654">
     </a>
     ④ 共享代码和逻辑：
    </h4>
    <blockquote>
     <p>
      当多个类需要共享部分代码时，抽象类可以避免代码重复。
     </p>
    </blockquote>
