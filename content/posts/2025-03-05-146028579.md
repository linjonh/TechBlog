    <h2>
     一、基本概念
    </h2>
    <p>
     <strong>
      网络编程三要素：
     </strong>
    </p>
    <ul>
     <li>
      IP
      <ul>
       <li>
        设备在网络中的唯一标识
       </li>
      </ul>
     </li>
     <li>
      端口号
      <ul>
       <li>
        应用软件在设备中的唯一标识
       </li>
       <li>
        两个字节表示的整数，0~1023用于知名的网络服务和应用，其他的可以自己使用
       </li>
       <li>
        <span style="color:#fe2c24">
         <strong>
          注意：一个端口号只能由一个应用程序使用
         </strong>
        </span>
       </li>
      </ul>
     </li>
     <li>
      协议
      <ul>
       <li>
        数据在网络中的传输规则
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      IP细节
     </strong>
    </p>
    <ul>
     <li>
      IPv4：互联网通信协议第四版，采取32位地址长度，
      <span style="color:#fe2c24">
       <strong>
        分为四组，每组转成十进制，中间用“点”隔开
       </strong>
      </span>
      ，注意没有负数
      <ul>
       <li>
        只有四十多亿个，2019年就不够用了，所以出现了IPv6
       </li>
      </ul>
     </li>
     <li>
      IPv4地址分类形式：
      <ul>
       <li>
        公网地址
       </li>
       <li>
        私有地址（局域网）
        <ul>
         <li>
          192.168.开头的地址
         </li>
        </ul>
       </li>
       <li>
        特殊IP：127.0.0.1，本机IP，localhost
       </li>
      </ul>
     </li>
     <li>
      IPv6：采取128位地址长度，分成八组
      <ul>
       <li>
        采取冒分十六进制表示法
       </li>
       <li>
        0位压缩法：如果中间有很多0，可以压缩不写
        <ul>
         <li>
          例：FF01:0:0:0:0:0:0:FF02可以写成FF01::FF02
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      常用CMD命令
     </strong>
    </p>
    <blockquote>
     <p>
      ipconfig：查看本机IP
     </p>
     <p>
      ping：检查网络是否连通
     </p>
    </blockquote>

    <h2>
     三、UDP协议
    </h2>
    <h3>
     1.发送数据
    </h3>
    <p>
     <span style="color:#fe2c24">
      <strong>
       UDP协议的发送，不管有没有建立连接都会发送，不一定能接收到
      </strong>
     </span>
    </p>
    <p>
     <strong>
      步骤：
     </strong>
    </p>
    <ul>
     <li>
      创建发送端的DatagramSocket对象
     </li>
     <li>
      数据打包（DatagramPacket）
     </li>
     <li>
      发送数据
     </li>
     <li>
      释放资源
     </li>
    </ul>
    <pre><code class="language-java">import java.io.IOException;
import java.net.*;

public class SendMessage {

    public static void main(String[] args) throws IOException {

        //1.创建对象
        //空参：随机使用一个端口进行发送
        //带参：指定端口号进行绑定
        DatagramSocket ds = new DatagramSocket();

        //2.打包数据
        String str = "114514王俊阳没有牛牛";
        byte[] bytes = str.getBytes();

        InetAddress address = InetAddress.getByName("127.0.0.1");//本机IP
        int port = 10086;//端口号
        
        //3.发送数据
        ds.send(new DatagramPacket(bytes,bytes.length,address,port));
        
        //4.释放资源
        ds.close();

    }
}</code></pre>
    <p>
    </p>
    <h3>
     2.接收数据
    </h3>
    <p>
     <strong>
      步骤：
     </strong>
    </p>
    <ul>
     <li>
      创建接收端的DatagramSocket对象
     </li>
     <li>
      接收打包好的数据
     </li>
     <li>
      解析数据包
     </li>
     <li>
      释放资源
     </li>
    </ul>
    <p>
     <strong>
      细节：
     </strong>
    </p>
    <ul>
     <li>
      receive方法是阻塞的，会等待消息发送过来
     </li>
     <li>
      注意先启动接收端，再启动发送端
     </li>
    </ul>
    <pre><code class="language-java">import java.io.IOException;
import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.net.InetAddress;

public class ReceiveMessage {

    public static void main(String[] args) throws IOException {

        //1.创建接收端口
        //注意：一定要绑定端口，并且绑定的端口要与发送端口保持一致
        DatagramSocket ds = new DatagramSocket(10086);

        //2.接收
        byte[] buf = new byte[1024];
        DatagramPacket dp = new DatagramPacket(buf, buf.length);
        ds.receive(dp);

        //3.解析数据包
        byte[] data = dp.getData();//获取发来的数据
        int len = dp.getLength();//发来多少数据
        int port = dp.getPort();//端口号
        InetAddress address = dp.getAddress();//IP地址

        System.out.println("接收到数据" + new String(data,0,len));
        System.out.println("该数据从" + address + "这台电脑中的" + port + "端口发出的");

        ds.close();
    }
}</code></pre>

    <h3>
     3.UDP协议的三种通信方式
    </h3>
    <p>
     <strong>
      1.单播
     </strong>
    </p>
    <ul>
     <li>
      1对1发送与接收数据
     </li>
    </ul>
    <p>
     <strong>
      2.组播
     </strong>
    </p>
    <ul>
     <li>
      给一组电脑发送信息
     </li>
     <li>
      组播地址：224.0.0.0~239.255.255.255
      <ul>
       <li>
        其中224.0.0.0~224.0.0.255是预留地址，可以我们自己用
       </li>
      </ul>
     </li>
     <li>
      创建对象时要创建MulticastSocket对象，指定InetAddress时要指定组播的地址
     </li>
    </ul>
    <p>
     <strong>
      3.广播
     </strong>
    </p>
    <ul>
     <li>
      给局域网中所有电脑发送数据
      <ul>
       <li>
        地址：255.255.255.255
       </li>
      </ul>
     </li>
    </ul>

    <h2>
     四、TCP协议
    </h2>
    <ul>
     <li>
      一种可靠的网络协议，他会在通信的两端分别建立一个Socket对象
     </li>
     <li>
      <span style="color:#fe2c24">
       <strong>
        通信前要保证连接已经建立
       </strong>
      </span>
     </li>
     <li>
      通过Socket产生
      <span style="color:#fe2c24">
       <strong>
        IO流
       </strong>
      </span>
      来进行网络通讯
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      客户端步骤：
     </strong>
    </p>
    <ul>
     <li>
      创建Socket对象与指定服务端连接
      <ul>
       <li>
        Socket（String host，int port）
       </li>
      </ul>
     </li>
     <li>
      获取输出流，写数据
      <ul>
       <li>
        OutputStream getOutputStream（）
       </li>
      </ul>
     </li>
     <li>
      释放资源
      <ul>
       <li>
        void clear（）
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      服务器步骤：
     </strong>
    </p>
    <ul>
     <li>
      创建服务器端的Socket对象
      <ul>
       <li>
        ServerSocket（int port）
       </li>
      </ul>
     </li>
     <li>
      监听客户端连接，返回一个Socket对象
      <ul>
       <li>
        Socket accept（）
       </li>
      </ul>
     </li>
     <li>
      从Socket对象中获取输入流，读数据，并把数据显示在控制台
      <ul>
       <li>
        InputStream getInputStream（）
       </li>
      </ul>
     </li>
     <li>
      释放资源
      <ul>
       <li>
        void clear（）
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      三次握手和四次挥手：
     </strong>
    </p>
    <ul>
     <li>
      三次握手：为了保证连接的确立
      <ul>
       <li>
        客户端向服务端发出连接请求，等待服务器确认
       </li>
       <li>
        服务器向客户端返回一个响应，告诉客户端收到了请求
       </li>
       <li>
        客户端向服务端再次发出确认信息，建立连接
       </li>
      </ul>
     </li>
     <li>
      四次挥手：确保连接断开且数据处理完毕
      <ul>
       <li>
        客户端向服务端发出取消链接请求
       </li>
       <li>
        服务端返回一个响应，表示受到请求
        <ul>
         <li>
          此时不会断开，会将流中最后的数据处理完毕
         </li>
        </ul>
       </li>
       <li>
        服务器向客户端发出确认取消的信息
       </li>
       <li>
        客户端再次发出确认信息，链接取消
       </li>
      </ul>
     </li>
    </ul>
