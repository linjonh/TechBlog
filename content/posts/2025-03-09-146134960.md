    <h5>
     <strong>
      1. DDoS攻击
     </strong>
    </h5>
    <p>
     <strong>
      1.1 什么是DDoS攻击？
     </strong>
     <br/>
     DDoS（Distributed Denial of Service，分布式拒绝服务攻击）通过操控大量“僵尸设备”（Botnet）向目标服务器发送海量请求，耗尽服务器资源（带宽、CPU、内存），导致正常用户无法访问服务。
    </p>
    <p>
     <strong>
      1.2 攻击原理与分类
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       流量型攻击
      </strong>
      ：如UDP洪水、ICMP洪水，通过发送大量无效数据包占用带宽。
     </li>
     <li>
      <strong>
       协议型攻击
      </strong>
      ：如SYN洪水攻击，利用TCP三次握手漏洞耗尽连接池。
     </li>
     <li>
      <strong>
       应用层攻击
      </strong>
      ：如HTTP Flood，模拟高频合法请求（如频繁访问API接口）。
     </li>
    </ul>
    <p>
     <strong>
      1.3 典型危害
     </strong>
    </p>
    <ul>
     <li>
      服务瘫痪：网站/APP无法响应。
     </li>
     <li>
      经济损失：电商宕机每小时损失可达百万级。
     </li>
     <li>
      品牌信誉受损：用户信任度下降。
     </li>
    </ul>
    <p>
     <strong>
      1.4 防御方案
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       流量清洗
      </strong>
      ：部署抗D设备（如阿里云高防IP）过滤异常流量。
     </li>
     <li>
      <strong>
       CDN加速
      </strong>
      ：分散流量压力，隐藏真实服务器IP。
     </li>
     <li>
      <strong>
       负载均衡
      </strong>
      ：通过集群架构分摊请求压力。
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      2. XSS攻击
     </strong>
    </h5>
    <p>
     <strong>
      2.1 什么是XSS攻击？
     </strong>
     <br/>
     XSS（跨站脚本攻击）通过向网页注入恶意脚本（JavaScript），在用户浏览器中执行，窃取Cookie、会话Token等敏感信息。
    </p>
    <p>
     <strong>
      2.2 攻击类型与案例
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       反射型XSS
      </strong>
      ：恶意链接诱导用户点击，如：
      <pre><code class="language-html">http://example.com?search=&lt;script&gt;alert('XSS')&lt;/script&gt;</code></pre>
     </li>
     <li>
      <strong>
       存储型XSS
      </strong>
      ：恶意脚本存储到数据库（如论坛评论栏注入）。
     </li>
     <li>
      <strong>
       DOM型XSS
      </strong>
      ：通过修改页面DOM树触发攻击，无需服务器交互。
     </li>
    </ul>
    <p>
     <strong>
      2.3 典型危害
     </strong>
    </p>
    <ul>
     <li>
      用户数据泄露：盗取账号密码、银行卡信息。
     </li>
     <li>
      钓鱼攻击：伪造登录页面诱导用户输入。
     </li>
     <li>
      传播恶意代码：如网页挖矿脚本。
     </li>
    </ul>
    <p>
     <strong>
      2.4 防御方案
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       输入输出转义
      </strong>
      ：对用户输入内容进行HTML实体编码（如
      <code>
       &lt;
      </code>
      转义为
      <code>
       &amp;lt;
      </code>
      ）。
     </li>
     <li>
      <strong>
       CSP（内容安全策略）
      </strong>
      ：限制脚本来源，例如：
      <pre><code class="language-html">Content-Security-Policy: default-src 'self'</code></pre>
     </li>
     <li>
      <strong>
       HttpOnly Cookie
      </strong>
      ：防止JavaScript读取敏感Cookie。
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      3. 中间人攻击（MITM）
     </strong>
    </h5>
    <p>
     <strong>
      3.1 什么是中间人攻击？
     </strong>
     <br/>
     攻击者在通信双方之间伪装成“中间人”，截获、篡改或窃取传输数据，常见于公共WiFi、HTTP明文传输场景。
    </p>
    <p>
     <strong>
      3.2 攻击手法
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       ARP欺骗
      </strong>
      ：伪造IP-MAC地址映射，劫持局域网流量。
     </li>
     <li>
      <strong>
       DNS劫持
      </strong>
      ：篡改DNS解析结果，将用户导向恶意站点。
     </li>
     <li>
      <strong>
       SSL剥离
      </strong>
      ：强制降级HTTPS为HTTP，明文窃取数据。
     </li>
    </ul>
    <p>
     <strong>
      3.3 典型危害
     </strong>
    </p>
    <ul>
     <li>
      敏感信息泄露：如账号密码、聊天记录。
     </li>
     <li>
      数据篡改：修改转账金额或交易对象。
     </li>
     <li>
      恶意软件植入：注入木马或勒索软件。
     </li>
    </ul>
    <p>
     <strong>
      3.4 防御方案
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       强制HTTPS
      </strong>
      ：使用HSTS头确保全站加密：
      <pre><code class="language-html">Strict-Transport-Security: max-age=31536000</code></pre>
     </li>
     <li>
      <strong>
       证书校验
      </strong>
      ：客户端验证服务器证书合法性，避免自签名证书风险。
     </li>
     <li>
      <strong>
       网络隔离
      </strong>
      ：禁止敏感操作在公共网络执行。
     </li>
    </ul>
