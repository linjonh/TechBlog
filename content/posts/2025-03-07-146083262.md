    <h4>
     <a id="_15">
     </a>
     字节码指令的基本结构
    </h4>
    <ol>
     <li>
      操作码（Opcode）：
      <ol>
       <li>
        固定 1 字节（8位），取值范围
        <code>
         0x00
        </code>
        ~
        <code>
         0xFF
        </code>
        （共 256 种可能的操作码）。
       </li>
       <li>
        例如：
        <code>
         iload_0
        </code>
        的操作码是
        <code>
         0x1A
        </code>
        ，
        <code>
         iadd
        </code>
        的操作码是
        <code>
         0x60
        </code>
        。
       </li>
      </ol>
     </li>
     <li>
      操作数（Operand）：
      <ol>
       <li>
        不同指令需要的操作数长度不同（0~多个字节）。
       </li>
       <li>
        操作数可能是局部变量索引、常量池索引、偏移量等。
       </li>
      </ol>
     </li>
    </ol>

    <h5>
     <a id="_56">
     </a>
     局部变量表
    </h5>
    <p>
     作用：存储方法执行过程中用到的局部变量（包括方法参数和方法内定义的变量）。
    </p>
    <p>
     <strong>
      核心特性：
     </strong>
    </p>
    <ol>
     <li>
      结构：
      <ol>
       <li>
        是一个按索引访问的数组，索引从
        <code>
         0
        </code>
        开始。
       </li>
       <li>
        每个槽位（Slot）占用 32 位（4字节）。对于
        <code>
         long
        </code>
        和
        <code>
         double
        </code>
        （64位类型），会占用两个连续的槽位。
       </li>
       <li>
        槽位可以被复用（例如，局部变量的作用域结束后，槽位可能被其他变量重用）。
       </li>
      </ol>
     </li>
     <li>
      存储内容：
      <ol>
       <li>
        方法参数：非静态方法的第 0 号槽位是
        <code>
         this
        </code>
        引用，静态方法没有
        <code>
         this
        </code>
        。
       </li>
       <li>
        方法内定义的变量（包括基本类型、对象引用、返回地址等）。
       </li>
      </ol>
     </li>
     <li>
      生命周期：
      <ol>
       <li>
        随方法调用创建（栈帧入栈），随方法结束销毁（栈帧出栈）。
       </li>
      </ol>
     </li>
    </ol>
    <p>
     <strong>
      示例代码分析：
     </strong>
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">example</span><span class="token punctuation">(</span><span class="token keyword">int</span> a<span class="token punctuation">,</span> <span class="token keyword">int</span> b<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

    <h5>
     <a id="_93">
     </a>
     操作数栈
    </h5>
    <p>
     作用：保存字节码指令执行过程中的临时操作数，是 JVM 基于栈的执行模型的核心。
    </p>
    <p>
     <strong>
      核心特性：
     </strong>
    </p>
    <ol>
     <li>
      结构：
      <ol>
       <li>
        后进先出（LIFO）的栈结构，最大深度在编译时确定。
       </li>
       <li>
        每个栈单元（Entry）占用 32 位，
        <code>
         long
        </code>
        和
        <code>
         double
        </code>
        占两个单元。
       </li>
      </ol>
     </li>
     <li>
      操作过程：
      <ol>
       <li>
        字节码指令从栈顶取出操作数，计算结果再压入栈顶。
       </li>
       <li>
        例如，
        <code>
         iadd
        </code>
        指令会弹出两个
        <code>
         int
        </code>
        值相加，再将结果压入栈。
       </li>
      </ol>
     </li>
     <li>
      生命周期：
      <ol>
       <li>
        随方法调用创建（栈帧入栈），随方法结束销毁（栈帧出栈）。
       </li>
      </ol>
     </li>
    </ol>
    <h4>
     <a id="_107">
     </a>
     局部变量压栈指令
    </h4>
    <p>
     局部变量压栈指令将局部变量表中的数据压入操作数栈
    </p>
    <ol>
     <li>
      xload_
     </li>
    </ol>
    <p>
     <code>
      x
     </code>
     为操作数类型，取值为i、l、f、d、a，
     <code>
      n
     </code>
     为要存入栈的变量在局部变量表中的下标索引，取值为0~3。
    </p>
    <p>
     比如，
     <code>
      aload_0
     </code>
     这个操作码的意思是将一个存放在局部变量表中下标索引为0的变量入操作数栈，变量类型为引用类型。其余前缀类型意义见
     <a href="https://kafdkvoitto.feishu.cn/docx/HAxCd4ucZof6wfxHUomc0ll8nEh#share-N2DsdlLJhocRORxbVjScCrVcna0" rel="nofollow">
      字节码与数据类型
     </a>
     。
    </p>
    <ol start="2">
     <li>
      xload
     </li>
    </ol>
    <p>
     <code>
      x
     </code>
     为操作数类型，取值为i、l、f、d、a，
     <code>
      n
     </code>
     为要存入栈的变量在局部变量表中的下标索引，当使用这个指令时，代表局部变量表中前四个槽位已经被占用，并且局部变量表中的前四个槽位中的变量都不是要入栈的变量。
    </p>
    <p>
     xload 与 xload_ 的不同在于，xload_是没有操作数的，只有一操作码，只占用一个字节；而xload 中为操作数，比如：
    </p>
    <pre><code class="prism language-java">iload <span class="token number">3</span>     <span class="token comment">// 0x15 0x03（加载局部变量表索引 3 的 int 值，总占 2 字节）</span>
</code></pre>
    <p>
     举例分析如下：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">load</span><span class="token punctuation">(</span><span class="token keyword">int</span> num<span class="token punctuation">,</span> <span class="token class-name">Object</span> obj<span class="token punctuation">,</span><span class="token keyword">long</span> count<span class="token punctuation">,</span><span class="token keyword">boolean</span> flag<span class="token punctuation">,</span><span class="token keyword">short</span><span class="token punctuation">[</span><span class="token punctuation">]</span> arr<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token class-name">System</span><span class="token punctuation">.</span>out<span class="token punctuation">.</span><span class="token function">println</span><span class="token punctuation">(</span>num<span class="token punctuation">)</span><span class="token punctuation">;</span>

          ：
          <code>
           float
          </code>
          的 23 位尾数可能导致精度丢失（如
          <code>
           16777217
          </code>
          转换后变为
          <code>
           16777216
          </code>
          ）。
         </li>
         <li>
          <code>
           long → float/double
          </code>
          ：
          <code>
           long
          </code>
          的 64 位整数超出
          <code>
           float
          </code>
          （24 位有效位）或
          <code>
           double
          </code>
          （53 位有效位）的表示范围时，按 IEEE 754 最近舍入模式近似。
         </li>
        </ul>
       </li>
      </ol>
     </li>
     <li>
      <strong>
       异常处理
      </strong>
      ：
      <ol>
       <li>
        <strong>
         不抛出任何运行时异常
        </strong>
        （即使发生精度丢失）。
       </li>
      </ol>
     </li>
    </ol>

        ）和方法参数，执行后弹出这些值，方法返回值（若有）压入栈顶。
       </li>
      </ol>
     </li>
     <li>
      <code>
       invokeinterface
      </code>
      ：
      <ol>
       <li>
        <strong>
         接口方法调用
        </strong>
        ：通过接口引用调用实际对象的实现方法。运行时需额外搜索方法表，性能略低于
        <code>
         invokevirtual
        </code>
        。
       </li>
       <li>
        <strong>
         示例
        </strong>
        ：
        <code>
         Runnable r = new MyTask(); r.run()
        </code>
        →
        <code>
         invokeinterface
        </code>
        调用
        <code>
         MyTask
        </code>
        的
        <code>
         run()
        </code>
        。
       </li>
      </ol>
     </li>
     <li>
      <code>
       invokespecial
      </code>
      ：
      <ol>
       <li>
        <strong>
         静态绑定
        </strong>
        ：直接调用目标方法，无多态分派。
       </li>
       <li>
        <strong>
         场景
        </strong>
        ：
        <ul>
         <li>
          构造器（
          <code>
           new Object()
          </code>
          →
          <code>
           invokespecial &lt;init&gt;
          </code>
          ）。
         </li>
         <li>
          私有方法（仅本类可访问）。
         </li>
         <li>
          <code>
           super.method()
          </code>
          （调用父类方法，可能跨多级父类查找）。
         </li>
        </ul>
       </li>
      </ol>
     </li>
     <li>
      <code>
       invokestatic
      </code>
      ：
      <ol>
       <li>
        <strong>
         无对象依赖
        </strong>
        ：操作数栈无需压入对象引用（
        <code>
         objectref
        </code>
        ）。
       </li>
       <li>
        <strong>
         调用方式无关
        </strong>
        ：无论通过类名（
        <code>
         Class.staticMethod()
        </code>
        ）或对象（
        <code>
         obj.staticMethod()
        </code>
        ）调用，均使用此指令。
       </li>
      </ol>
     </li>
     <li>
      <code>
       invokedynamic
      </code>
      （补充说明）：
      <ol>
       <li>
        <strong>
         动态语言支持
        </strong>
        ：允许运行时动态决定调用逻辑，提升灵活性。
       </li>
       <li>
        <strong>
         开发者透明
        </strong>
        ：通常由编译器生成（如 Lambda 表达式编译为匿名类时自动使用此指令）。
       </li>
      </ol>
     </li>
    </ol>

        </strong>

      </ol>
     </li>

        </code>

        </code>
        值需为连续整数（如
        <code>
         1,2,3,4
        </code>
        ）。
       </li>
       <li>
        <strong>
         底层实现
        </strong>
        ：通过数学计算
        <code>
         index - min
        </code>
        快速定位偏移量，无需遍历。
       </li>
       <li>
        <strong>
         内存占用
        </strong>
        ：若
        <code>
         case
        </code>
        值范围大但实际值少（如
        <code>
         1,1000
        </code>
        ），可能浪费空间。
       </li>
      </ol>
     </li>
     <li>
      <code>
       lookupswitch
      </code>
      ：
      <ol>
       <li>
        <strong>
         适用条件
        </strong>
        ：
        <code>
         case
        </code>
        值为离散值（如
        <code>
         1,10,100
        </code>
        ）。
       </li>
       <li>
        <strong>
         底层实现
        </strong>
        ：存储
        <code>
         case-offset
        </code>
        键值对列表，需遍历查找匹配项。
       </li>
       <li>
        <strong>
         优化
        </strong>
        ：
        <code>
         case
        </code>
        值在字节码中按升序排列，但查找时仍为线性扫描。
       </li>
      </ol>
     </li>
     <li>
      <strong>
       默认跳转
      </strong>
      ：两种指令均支持
      <code>
       default
      </code>
      分支，处理未匹配的情况。
     </li>
     <li>
      <strong>
       编译器选择
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>

    <h4>
     <a id="_1025">
     </a>
     异常处理与异常表
    </h4>
    <ol>
     <li>
      <strong>
       异常处理的核心机制：异常表
      </strong>
     </li>
    </ol>
    <p>
     在 JVM 中，
     <code>
      try-catch
     </code>
     或
     <code>
      try-finally
     </code>
     的异常处理通过
     <strong>
      异常表（Exception Table）
     </strong>
     实现，而非通过特定字节码指令。 异常表是方法字节码的一部分，定义了异常处理的逻辑范围和处理方式。
    </p>
    <ol>
     <li>
      <strong>
       异常表的结构
      </strong>
     </li>
    </ol>
    <p>
     每个方法的异常表由多个条目组成，每个条目包含以下字段：
    </p>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         字段
        </strong>
       </th>
       <th align="left">
        <strong>
         描述
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        <strong>
         起始位置（start_pc）
        </strong>
       </td>
       <td align="left">
        try 块的起始指令位置（字节码偏移量）。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         结束位置（end_pc）
        </strong>
       </td>
       <td align="left">
        try 块的结束指令位置（不包含该位置本身，即 [start_pc, end_pc)）。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         处理偏移量（handler_pc）
        </strong>
       </td>
       <td align="left">
        异常处理代码的起始位置（指向 catch 或 finally 块的字节码偏移量）。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         捕获类型（catch_type）
        </strong>
       </td>
       <td align="left">
        常量池索引，指定捕获的异常类型（如 Exception）。若为 0，表示 finally 块。
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/198e317c73a4cd9fc3d9607f4886a818.png"/>
    </p>
    <ol>
     <li>
      <strong>
       异常处理流程
      </strong>
     </li>
    </ol>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/e41752037360b7e192ad4a6bb2ca6e21.png"/>
    </p>
    <ol>
     <li>
      <strong>
       <code>
        finally
       </code>
       块的实现
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       无论是否抛出异常
      </strong>
      ，
      <code>
       finally
      </code>
      块代码都会执行。
     </li>
     <li>
      JVM 通过以下两种方式实现：
      <ul>
       <li>
        <strong>
         复制
         <code>
          finally
         </code>
         代码到所有退出路径
        </strong>
        ： 在
        <code>
         try
        </code>
        块的每个退出路径（如
        <code>
         return
        </code>
        、
        <code>
         throw
        </code>
        ）前插入
        <code>
         finally
        </code>
        代码。
       </li>
       <li>
        <strong>
         异常表条目
        </strong>
        ： 对于
        <code>
         try-finally
        </code>
        （无
        <code>
         catch
        </code>
        ），异常表条目中的
        <code>
         catch_type
        </code>
        为
        <code>
         0
        </code>
        ，表示捕获所有异常类型，并在处理代码中执行
        <code>
         finally
        </code>
        逻辑后重新抛出异常。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       显式声明异常（
       <code>
        throws
       </code>
       ）
      </strong>
      <ol>
       <li>
        若方法通过
        <code>
         throws
        </code>
        声明可能抛出的异常，字节码中会添加
        <code>
         Exceptions
        </code>
        属性：
       </li>
      </ol>
     </li>
    </ol>
    <pre><code class="prism language-typescript"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">demo</span><span class="token punctuation">(</span><span class="token punctuation">)</span> throws IOException<span class="token punctuation">,</span> SQLException <span class="token punctuation">{<!-- --></span> <span class="token operator">...</span> <span class="token punctuation">}</span>
</code></pre>
    <ol start="2">
     <li>
      <strong>
       对应的字节码属性
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-typescript">Exceptions<span class="token operator">:</span>
  throws java<span class="token punctuation">.</span>io<span class="token punctuation">.</span>IOException<span class="token punctuation">,</span> java<span class="token punctuation">.</span>sql<span class="token punctuation">.</span>SQLException
</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/885dd9a30bbb4a87ff01df24ce6fb004.png"/>
    </p>
    <ol start="3">
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        供编译器和 JVM 验证方法调用是否处理了这些异常。
       </li>
       <li>
        与方法内的异常表无关，仅用于声明方法的潜在异常类型。
       </li>
      </ul>
     </li>
    </ol>

     <a id="_1139">
     </a>
     同步控制指令
    </h3>
    <h4>
     <a id="_1140">
     </a>
     方法级同步
    </h4>
    <ol>
     <li>
      <strong>
       同步方法的实现机制
      </strong>
     </li>
    </ol>
    <p>
     在 JVM 中，
     <strong>
      方法级同步
     </strong>
     （通过
     <code>
      synchronized
     </code>
     修饰的方法）通过
     <strong>
      隐式锁机制
     </strong>
     实现，而非显式使用
     <code>
      monitorenter
     </code>
     和
     <code>
      monitorexit
     </code>
     指令。其核心规则如下：
    </p>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         特性
        </strong>
       </th>
       <th align="left">
        <strong>
         说明
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        <strong>
         锁的获取与释放
        </strong>
       </td>
       <td align="left">
        JVM 在调用同步方法时自动获取锁，方法结束时（无论正常或异常）自动释放锁。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         实现方式
        </strong>
       </td>
       <td align="left">
        通过方法的访问标志 ACC_SYNCHRONIZED 标识是否为同步方法。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         锁对象
        </strong>
       </td>
       <td align="left">
        实例方法锁对象是 this，静态方法锁对象是类的 Class 对象。
       </td>
      </tr>
     </tbody>
    </table>
    <ol>
     <li>

      </strong>
     </li>
    </ol>

     </li>
    </ol>
    <ul>
     <li>
      <strong>
       规则
      </strong>
      ：若同步方法抛出异常且未内部处理，JVM 会在异常传播到方法外部前自动释放锁。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">synchronized</span> <span class="token keyword">void</span> <span class="token function">riskyMethod</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">if</span> <span class="token punctuation">(</span>error<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">throw</span> <span class="token keyword">new</span> <span class="token class-name">RuntimeException</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 抛出异常，锁自动释放</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <ol>
     <li>
      <strong>
       如何验证方法是否为同步方法？
      </strong>
     </li>
    </ol>
    <p>
     通过
     <code>
      javap -v
     </code>
     查看方法的访问标志：
    </p>
    <pre><code class="prism language-typescript"><span class="token comment">// 同步方法示例</span>
<span class="token keyword">public</span> synchronized <span class="token keyword">void</span> <span class="token function">demo</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
  flags<span class="token operator">:</span> <span class="token constant">ACC_PUBLIC</span><span class="token punctuation">,</span> <span class="token constant">ACC_SYNCHRONIZED</span>  <span class="token comment">// 存在 ACC_SYNCHRONIZED</span>
  
<span class="token comment">// 非同步方法示例</span>
<span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">demo</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
  flags<span class="token operator">:</span> <span class="token constant">ACC_PUBLIC</span>                     <span class="token comment">// 无 ACC_SYNCHRONIZED</span>
</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/8b4e217b0975ec3e65a74dbf00f7e016.png"/>
    </p>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       同步方法
      </strong>
      通过
      <code>
       ACC_SYNCHRONIZED
      </code>
      隐式实现锁机制，无需显式指令，锁的获取和释放由 JVM 自动管理。
     </li>
     <li>
      <strong>
       同步代码块
      </strong>
      需显式使用
      <code>
       monitorenter
      </code>
      和
      <code>
       monitorexit
      </code>
      ，适用于需要精确控制同步范围的场景。
     </li>
     <li>
      <strong>
       工具验证
      </strong>
      ：使用
      <code>
       javap -v
      </code>
      查看方法访问标志，区分同步与非同步方法。
     </li>
    </ul>
    <h4>
     <a id="_1231">
     </a>
     方法内指令指令序列的同步
    </h4>
    <ol>
     <li>
      <strong>
       同步代码块的实现机制
      </strong>
     </li>
    </ol>
    <p>
     通过
     <code>
      synchronized
     </code>
     修饰的代码块使用
     <strong>
      显式锁机制
     </strong>
     ，由 JVM 的
     <code>
      monitorenter
     </code>
     和
     <code>
      monitorexit
     </code>
     指令实现。以下是核心规则：
    </p>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         特性
        </strong>
       </th>
       <th align="left">
        <strong>
         说明
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        <strong>
         锁的获取
        </strong>
       </td>
       <td align="left">
        线程通过 monitorenter 请求进入同步代码块，检查对象的监视器状态。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         锁的释放
        </strong>
       </td>
       <td align="left">
        线程通过 monitorexit 退出同步代码块时释放锁。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         锁对象
        </strong>
       </td>
       <td align="left">
        显式指定的对象（如 synchronized(obj)），实例方法默认是 this。
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         可重入性
        </strong>
       </td>
       <td align="left">
        同一线程可多次获取同一对象的锁（监视器计数器递增）。
       </td>
      </tr>
     </tbody>
    </table>
    <ol start="2">
     <li>
      <strong>
       对象监视器与计数器
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       监视器（Monitor）
      </strong>
      ：每个对象关联一个监视器，记录锁状态。
     </li>
     <li>
      <strong>
       计数器
      </strong>
      ：
      <ul>
       <li>
        <strong>
         计数器 = 0
        </strong>
        ：对象未锁定，线程可获取锁。
       </li>
       <li>
        <strong>
         计数器 &gt; 0
        </strong>
        ：对象已锁定。若当前线程是锁持有者，计数器递增（可重入）；否则线程阻塞。
       </li>
      </ul>
     </li>
    </ul>
    <ol start="3">
     <li>
      <strong>
       同步代码块的字节码流程
      </strong>
     </li>
    </ol>
    <p>
     以下是一个示例及其字节码分析：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">SynchronizedTest</span> <span class="token punctuation">{<!-- --></span>

    <span class="token keyword">private</span> <span class="token keyword">int</span> i <span class="token operator">=</span> <span class="token number">0</span><span class="token punctuation">;</span>
    <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">add</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>
        i<span class="token operator">++</span><span class="token punctuation">;</span>
    <span class="token punctuation">}</span>


    <span class="token keyword">private</span> <span class="token class-name">Object</span> obj <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">Object</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">subtract</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>

        <span class="token keyword">synchronized</span> <span class="token punctuation">(</span>obj<span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>
            i<span class="token operator">--</span><span class="token punctuation">;</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/4f35e508e18cc21f256d2d7577ac0925.png"/>
    </p>
    <p>
     <strong>
      关键流程
     </strong>
     ：
    </p>
    <ol>
     <li>
      <strong>
       获取锁
      </strong>
      ：
      <code>
       monitorenter
      </code>
      指令尝试获取锁对象的监视器。
     </li>
     <li>
      <strong>
       执行代码
      </strong>
      ：同步块内的操作（如
      <code>
       i--
      </code>
      ）。
     </li>
     <li>
      <strong>
       释放锁
      </strong>
      ：
      <ol>
       <li>
        <strong>
         正常退出
        </strong>
        ：
        <code>
         monitorexit
        </code>
        在代码块末尾释放锁（指令18）。
       </li>
       <li>
        <strong>
         异常退出
        </strong>
        ：若同步块内抛出异常，通过
        <code>
         Exception table
        </code>
        跳转到指令24释放锁，再抛出异常。
       </li>
      </ol>
     </li>
    </ol>
    <hr/>
    <ol start="4">
     <li>
      <strong>
       可重入性示例
      </strong>
     </li>
    </ol>
    <p>
     同一线程多次进入同步代码块时，监视器计数器递增：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">nestedSync</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">synchronized</span> <span class="token punctuation">(</span>lock<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">synchronized</span> <span class="token punctuation">(</span>lock<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span> <span class="token comment">// 同一线程重复获取锁</span>
            count<span class="token operator">++</span><span class="token punctuation">;</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     <strong>
      监视器状态变化
     </strong>
     ：
    </p>
    <ul>
     <li>
      第一次
      <code>
       monitorenter
      </code>
      → 计数器从
      <code>
       0
      </code>
      变为
      <code>
       1
      </code>
      。
     </li>
     <li>
      第二次
      <code>
       monitorenter
      </code>
      → 计数器从
      <code>
       1
      </code>
      变为
      <code>
       2
      </code>
      。
     </li>
     <li>
      第一次
      <code>
       monitorexit
      </code>
      → 计数器从
      <code>
       2
      </code>
      变为
      <code>
       1
      </code>
      。
     </li>
     <li>
      第二次
      <code>
       monitorexit
      </code>
      → 计数器从
      <code>
       1
      </code>
      变为
      <code>
       0
      </code>
      （锁释放）。
     </li>
    </ul>
    <ol start="5">
     <li>
      <strong>
       同步代码块 vs 同步方法
      </strong>
     </li>
    </ol>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         特性
        </strong>
       </th>
       <th align="left">
        <strong>
         同步代码块
        </strong>
       </th>
       <th align="left">
        <strong>
         同步方法
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        <strong>
         实现方式
        </strong>
       </td>
       <td align="left">
        显式使用 monitorenter 和 monitorexit
       </td>
       <td align="left">
        隐式通过 ACC_SYNCHRONIZED 标志
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         锁对象控制
        </strong>
       </td>
       <td align="left">
        可指定任意对象
       </td>
       <td align="left">
        实例方法锁 this，静态方法锁 Class
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         字节码可见性
        </strong>
       </td>
       <td align="left">
        显式锁指令
       </td>
       <td align="left">
        无显式指令，通过访问标志识别
       </td>
      </tr>
      <tr>
       <td align="left">
        <strong>
         灵活性
        </strong>
       </td>
       <td align="left">
        高（可控制同步范围）
       </td>
       <td align="left">
        低（整个方法同步）
       </td>
      </tr>
     </tbody>
    </table>
    <ol start="6">
     <li>
      <p>
       <strong>
        关键注意事项
       </strong>
      </p>
      <ol>
       <li>
        <strong>
         锁释放的严格性
        </strong>
        ：即使同步块内抛出未捕获异常，
        <code>
         monitorexit
        </code>
        也会在异常处理路径中释放锁。
       </li>
       <li>
        <strong>
         性能影响
        </strong>
        ：频繁竞争锁可能导致线程阻塞，需合理设计同步范围。
       </li>
       <li>
        <strong>
         死锁风险
        </strong>
        ：多个线程以不同顺序获取多个锁时可能死锁，需避免嵌套锁的不一致获取顺序。
       </li>
      </ol>
     </li>
    </ol>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <ul>
     <li>
      <code>
       monitorenter
      </code>
      和
      <code>
       monitorexit
      </code>
      是 JVM 实现同步代码块的核心指令，显式控制锁的获取与释放。
     </li>
     <li>
      <strong>
       可重入性
      </strong>
      允许同一线程多次获取同一锁，避免自我阻塞。
     </li>
     <li>
      <strong>
       异常处理
      </strong>
      确保锁在异常路径下仍被释放，避免死锁。
     </li>
     <li>
      <strong>
       同步代码块
      </strong>
      相比同步方法更灵活，适用于需要细粒度控制的并发场景。
     </li>
    </ul>
    <p>
     <strong>
      示例分析
     </strong>
    </p>
    <blockquote>
     <p>
      操作数栈中的对象和monitorenter结合起来可以让线程获取锁，做法就是让对象的监视器标记从0变成1，这就代表该线程上锁了，然后在操作数栈的aload_1和monitorexit结合起来就可以让线程解锁，做法就是让对象的监视器标记从1变成0，这个解锁需要在方法退出之前完成，如果方法执行过程中出现了任何异常，将会跳到异常处理的字节码处执行相关代码，如果异常处理的字节码部分出现了问题，那就重新执行异常处理的字节码，这些内容都在异常表中写的很明确，其中异常表也在上面截图中。
