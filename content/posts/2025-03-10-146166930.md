     vue3中接收props的两种写法
    </h1>
   </div>

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     在 Vue 3 中，接收
     <code>
      props
     </code>
     有两种主要的写法，分别是
     <strong>
      运行时声明
     </strong>
     和
     <strong>
      基于类型的声明
     </strong>
     。下面为你详细介绍这两种写法。
    </p>
    <h4>
     1. 运行时声明
    </h4>
    <p>
     运行时声明是 Vue 2 中就已经存在的方式，在 Vue 3 中依然可以使用。这种方式通过在组件中使用
     <code>
      defineProps
     </code>
     宏来定义
     <code>
      props
     </code>
     ，并且可以指定
     <code>
      props
     </code>
     的类型、默认值和验证规则。
    </p>
    <h5>
     示例代码
    </h5>
    <pre><code class="language-TypeScript">&lt;template&gt;
  &lt;div&gt;
    &lt;p&gt;Name: {<!-- -->{ name }}&lt;/p&gt;
    &lt;p&gt;Age: {<!-- -->{ age }}&lt;/p&gt;
  &lt;/div&gt;
&lt;/template&gt;

&lt;script setup&gt;
// 使用运行时声明接收props
const props = defineProps({
  name: {
    type: String,
    required: true
  },
  age: {
    type: Number,
    default: 18
  }
});
&lt;/script&gt;</code></pre>
    <h5>
     代码解释
    </h5>
    <ul>
     <li>
      <code>
       defineProps
      </code>
      是一个宏，用于定义组件的
      <code>
       props
      </code>
      。
     </li>
     <li>
      <code>
       name
      </code>
      和
      <code>
       age
      </code>
      是定义的两个
      <code>
       props
      </code>
      。
      <ul>
       <li>
        <code>
         name
        </code>
        是一个必需的字符串类型的
        <code>
         prop
        </code>
        。
       </li>
       <li>
        <code>
         age
        </code>
        是一个可选的数字类型的
        <code>
         prop
        </code>
        ，默认值为
        <code>
         18
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     2. 基于类型的声明
    </h4>
    <p>
     基于类型的声明是 Vue 3 新增的特性，它利用 TypeScript 的类型系统来定义
     <code>
      props
     </code>
     。这种方式更加简洁，并且可以提供更好的类型检查。
    </p>
    <h5>
     示例代码
    </h5>
    <pre><code class="language-TypeScript">&lt;template&gt;
  &lt;div&gt;
    &lt;p&gt;Name: {<!-- -->{ name }}&lt;/p&gt;
    &lt;p&gt;Age: {<!-- -->{ age }}&lt;/p&gt;
  &lt;/div&gt;
&lt;/template&gt;

&lt;script setup lang="ts"&gt;
// 使用基于类型的声明接收props
interface Props {
  name: string;
  age?: number;
}

const props = defineProps&lt;Props&gt;();
&lt;/script&gt;</code></pre>
    <h5>
     代码解释
    </h5>
    <ul>
     <li>
      首先定义了一个
      <code>
       Props
      </code>
      接口，用于描述
      <code>
       props
      </code>
      的类型。
     </li>
     <li>
      <code>
       name
      </code>
      是一个必需的字符串类型的
      <code>
       prop
      </code>
      。
     </li>
     <li>
      <code>
       age
      </code>
      是一个可选的数字类型的
      <code>
       prop
      </code>
      。
     </li>
     <li>
      <code>
       defineProps&lt;Props&gt;()
      </code>
      使用泛型来指定
      <code>
       props
      </code>
      的类型。
     </li>
    </ul>
    <h4>
     两种写法的对比
    </h4>
    <ul>
     <li>
      <strong>
       运行时声明
      </strong>
      ：适用于不使用 TypeScript 的项目，或者需要定义复杂的验证规则和默认值的场景。
     </li>
     <li>
      <strong>
       基于类型的声明
      </strong>
      ：适用于使用 TypeScript 的项目，它可以提供更好的类型检查和代码提示。
     </li>
    </ul>
    <h4>
     使用组件
    </h4>
    <p>
     无论使用哪种方式声明
     <code>
      props
     </code>
     ，使用组件时传递
     <code>
      props
     </code>
     的方式是相同的。
    </p>
    <pre><code class="language-TypeScript">&lt;template&gt;
  &lt;MyComponent name="John" age="25" /&gt;
&lt;/template&gt;

&lt;script setup&gt;
import MyComponent from './MyComponent.vue';
&lt;/script&gt;</code></pre>
    <p>
     以上就是 Vue 3 中接收
     <code>
      props
     </code>
     的两种主要写法。
    </p>
