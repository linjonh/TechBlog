# 从公有云到私有云，云原生正在主导数字化转型企业架构

![](https://i-blog.csdnimg.cn/blog_migrate/1791ba8ac9913f606cb5a3f321e2b0ce.png)

进入2019年以来，云原生计算正在成为企业IT领域的新热潮。随着数字经济的快速发展和扩张，越来越多的企业开始采用云原生计算的思想和技术，以主导企业的数字化转型架构。由于Kubernetes在2018年取得了容器集群管理平台的市场主导地位，容器已经成为了连接传统IT与云原生计算的重要技术平台，这为云原生计算作为未来数字企业的主导技术架构铺平了道路。

2019年10月24日，在云原生技术实践峰会（CNBPS 2019）上，灵雀云创始人兼CEO左玥表示，自2016年随着中国数字经济的推进，越来越多的企业开始考虑云原生架构来统领整个企业IT规划，国内很多大型企业甚至政府机构都在以超越想像的速度切入云原生主导的数字化转型进程。

从公有云到私有云，云原生正在中国展开大规模实践。

## 灵雀云三大典型云原生实践

![](https://i-blog.csdnimg.cn/blog_migrate/fbb41759a892526a25a7a203610cc595.png)

（灵雀云创始人兼CEO左玥）

所谓云原生技术，主要指以容器、持续交付&DevOps以及微服务为代表的技术体系，其中关键的容器技术主要指对软件资源的虚拟化，微服务则指基于容器应用的软件架构，这二者是典型的云原生技术，而Kubernetes则可兼顾容器与传统虚拟化以及物理机资源的管理，持续交付&DevOps是可以兼顾传统应用与互联网应用的统一软件开发方法论。

从云原生技术体系即可以看出，它是面向互联网应用同时又能够兼顾传统应用的技术架构，未来的企业应用将以“互联网+”和“智能+”类为主体，包括人工智能、物联网、大数据分析、车联网、区块链、5G等，而传统的企业应用则以ERP、OA、CRM等为典型代表。左玥介绍，灵雀云自2014年成立，2016年开始在云原生领域发力，经历了三种典型的企业转型实践。

首先是传统企业需要开拓数字创新类业务，以银行开展线上金融类业务为代表。在面对支付宝、微信等新兴数字金融和互联网金融的竞争中，传统银行迫切需要从传统核心银行系统向数字营销与在线风控等新兴业务发展。IT在此过程中成为了核心竞争力而不再是支撑系统，银行对于软件迭代和质量等要求大幅提升，银行CIO们更关注应用侧如何打造更好的IT产品、研发人员和架构师更关心应用架构的敏捷性以及满足快速迭代DevOps的需求，同时基于容器平台，面向云原生应用开发的DevOps和微服务就能够发挥更大的价值。

其次是大型企业需要加强对集团的资源管理和管控，同时建立面向未来数字应用的新型架构，以央企为典型代表。灵雀云与中石油合作，帮助中石油对旗下的省公司、各油田等进行统一的数据治理和管理，减少重复投资建设。对于超大型央企来说，通过PaaS+数据湖的方案就可以很好的实现集团总部集中管控式的管理：数据统一起来，进行全集团级别的数据分析，达到更高效的数据利用率；PaaS又能对基础组件、开发流程进行统一，同时又给予分省公司以灵活自由度，可以定制开发。

第三是对大量传统应用的改造，从而在一套体系下与云原生应用对接和统一管理。很多企业的传统和遗留应用，已经稳定也没必要改动，甚至经过历代开发人员的流失，连代码都没了，但却要依靠其运转，这就需要通过API进行对接。传统的ESB总线方式，改造成本太高；而相对松耦合却有一定管控的MASA（Mesh App and Services Architecture），通过API进行对接就具有很好的灵活度，特别是微服务本身也是API，这样就能通过API打通“双模”IT。

灵雀云在企业实践中，通常对企业IT演进进行整体规划：分析哪些应用适合先上云原生、进行彻底的微服务改造；哪些应用要慢一步、先用容器和网格计算达到一部分的治理功能，不需要太大投入也能立刻享受完整的CI/CD和DevOps流程。而对于一些已经无法改动的应用，则加上API接口就能保证与其它系统的整合。

通过2016年到2019年的实践，灵雀云作为国内的云原生实践先行者，为中国企业数字化转型进程找到了可演进的技术路线，服务着一批规模最大且优质的客户，中石油、国家电网、中移动、三一集团、中信银行、光大银行、平安银行、招商银行、南航、宇通客车等一批大型企业和政府机构成为了灵雀云的客户，采用不同的云原生技术走上了数字化转型之路。

## 踏准企业级技术路线

![](https://i-blog.csdnimg.cn/blog_migrate/1473e587b00e0ef04b10fe15d47beaf2.png)

灵雀云成立于2014年，是当时一批Docker容器和PaaS创业公司之一。两位创始人左玥和CTO陈恺来自于早期的微软Azure云平台核心创始团队，由于看好Docker容器这种革命性的技术，于是回国创业，起初是基于公有云做CaaS容器服务。2016年底前后，国内有股份制银行开始立项私有云容器项目，意味着客户开始为容器技术买单。灵雀云于是考虑，需要尽早进入企业级市场，这也是容器技术在企业落地的关键节点。

转向服务企业级客户，灵雀云经历了产品+服务的商业模式升级：从最早只有研发和销售，到后来的研发、售前、咨询、定制化开发、交付运维、项目PMO、售后7×24小时运维维保等，重建了一套面向企业级客户的组织和能力体系。

而在产品与服务之间的比例关系，灵雀云也经历了三次反复迭代的过程：全部产品化+少量服务，难以满足企业的个性化定制需求；少量产品化+大量服务，灵雀云自身的商业模式难以持续；最终找到了80%-90%的产品化+10%-20%的服务，在满足企业需求以及灵雀云自身可持续商业模式之间，找到了平衡。

国际上的软件产品型公司，公司核心力量聚集于企业级软件产品的开发与打磨，提供部分原厂服务再加上依赖广大的ISV、SI等技术服务合作伙伴，做大整个软件+服务的生态，进而成为独角兽型公司，而这也是灵雀云的思路。

也正因为选择了服务企业级客户，灵雀云成为国内最早一批进入Kubernetes阵营的厂商。2017年11月，腾讯云投资灵雀云的B轮，双方在技术上形成了两级“火箭”：灵雀云专注于企业级云原生技术架构和核心组件，缺乏的其它周边技术组件如数据库等则由腾讯云提供，双方合力形成了完整的技术方案。

## 更适合中国用户的云原生架构

通过早期服务国内的企业客户特别是金融机构，灵雀云了解国内企业的需求特点。与国外企业的需求不同，国外企业习惯于找到单点用途的开源工具后自行使用，每个工具只专注于解决一个问题，而国内企业则需要完整的覆盖几乎所有需求的工具链和工具集，还要对接审批等各种内部管理和流程。灵雀云针对国内企业的需求，提供了完整的一站式云原生PaaS平台，在Kubernetes等产品化方面提供了用户友好的管理工具、提供针对运维与开发侧的双模审批等。

2018年，灵雀云推出了完整的云原生产品线：一站式云原生应用赋能平台Alauda Container Platform（ACP）、企业级容器PaaS平台Alauda Cloud Enterprise（ACE）、Alauda Kubernetes企业就绪版、深度机器学习平台Alauda Machine Learning（AML）、企业级 API 管理平台Alauda API Management Platform（AMP），实现了对云原生技术的完整覆盖，满足企业数字化转型需求。

![](https://i-blog.csdnimg.cn/blog_migrate/9fe40b5be36a3d2122d8f3dde7fe4b7a.png)

灵雀云的核心使命是：通过云原生技术帮助传统企业加速创新，成功数字化转型。经过五年的积累，灵雀云为企业提供了完整的云原生技术架构和产品技术体系蓝图：ACE定位为企业技术中台，面向大型企业平台部门，以容器平台为核心，甄选生态合作伙伴优秀PaaS类服务，灵活集成第三方中间件和开发框架，帮助企业构建专属技术中台，满足企业级应用逐步向容器化过渡的广泛需求。

此外，针对企业的不同需求，灵雀云推出了自己Kubernetes发行版、在此之上是ACP 2.0云原生平台，包含Alauda Container Platform容器平台、面向DevOps的Alauda DevOps、面向微服务的Alauda Service Mesh(ASM）企业级微服务治理平台三大子产品。

![](https://i-blog.csdnimg.cn/blog_migrate/b0d0dea7405de857107b12f7f9020969.png)

DevOps平台定位在开放式DevOps工具链集成与编排平台。可以灵活兼容客户的工具选型，通过集成将各类工具串联，形成工具链，通过编排让DevOps工具链与容器平台联动，形成一个完整系统。同时，结合自身的经验，提炼DevOps落地最佳实践，作为服务进行输出。ASM企业级微服务治理平台主要关注五大场景：服务全局可观测性、服务发布管理、服务可靠性治理、服务安全性治理、服务网格全生命周期管理。

AML是基于容器技术和深度学习框架TensorFlow训练模型的机器学习平台。AML部署在Alauda Kubernetes中，用户可在平台中使用CPU或GPU进行分布式深度学习，并根据计算资源使用情况，实现动态横向扩展。用于模型发布管理，可以跟DevOps和Service Mesh结合，做模型AB测试、灰度发布等。

AMP企业级 API 管理平台是灵雀云最新推出的产品,帮助具有大量传统IT业务的企业客户实现云原生应用架构的落地。AMP定位为企业总的API网关，负责南北向API治理，包括API 全生命周期管理、API 能力开放运营（API Economy）、API 治理等关键功能。

![](https://i-blog.csdnimg.cn/blog_migrate/70653dbaa19bb58296e16f99038ea2a2.png)

左玥强调，灵雀云类似于“小IBM”的模式，面向企业云原生领域提供完整的云原生软件核心产品体系，与腾讯云和生态伙伴合作覆盖更完整云原生技术蓝图，而灵雀云在云原生软件核心技术领域并无明显的短板，强项则在于对于企业级云原生需求以及传统IT的理解与全面的解决方案组织和交付能力。

灵雀云作为云原生的新锐力量，积极拥抱开源，不断加入开源社区和组织，是CNCF社区的早期倡导者和贡献者。同时，不断贡献开源项目，在Kubernetes及其周边技术上贡献力量，先后开源了Kube-OVN网络组件、Captain Helm组件等项目。

灵雀云开源的Kube-OVN，即平移OpenStack业已成熟的OVN/OVS网络概念和功能到Kubernetes，增强Kubnerets的网络功能和管理。OVN/OVS已经成为SDN软件定义网络的标准，能够很好的解决容器云网络的管理与问题排查等需求。而网络是 Kubernetes在企业落地的重点。目前社区开源的Kubernetes网络方案功能大多比较简单，难以满足企业级场景。Kube-OVN已被纳入CNCF云原生全景图，目前已经推出0.8版本，1.0正式版本呼之欲出。

灵雀云开源另外一个叫做Captain的项目，即是一个标准化的Kubernetes Controller，是对Helm应用分发的改进。Helm是Kubernetes 环境中应用分发的标准模式，为了构建企业级的应用目录产品，解决使用Helm过程中的问题，提供尽可能多的可扩展性，这就是Captain的由来。2019年8月，Captain发布了0.9版本，即将推出正式的1.0版本。

如今，灵雀云已经是CNCF云原生基金会会员及官方合作伙伴、率先通过Kubernetes一致性认证和标准化认证、成为CNCF认证Kubernetes服务提供商和培训合作伙伴、成为持续交付基金会CD.Foundation创始会员等。

作为联盟创始成员，积极参与国内云原生平台标准的制定。标准的出台表明业内对云原生技术的成熟形成共识，对追求稳健的企业客户的技术落地有推动作用。在国内云原生的生态组织方面，2019年4月在信通院云计算标准和开源推进委员会（TC608）第三次全体会员大会上，左玥被任命为云原生联盟（CNIA）平台架构组副组长。CNIA前身为云原生技术实践联盟（CNBPA），即由灵雀云牵头的云原生实践组织。灵雀云还积极参与了国内云原生平台标准的制定与及未来方向的探讨，在CNIA发起的国内首个“云原生技术实践白皮书”和“无服务架构技术白皮书”中，承担了核心编写任务。

在数字化转型的前提下，企业开始不再将大部分业务系统，尤其是核心业务需求开发外包，而是开始自建软件研发团队、主导IT应用的研发。这说明，在数字经济时代，软件能力正在成为企业的核心竞争力，而云原生将主导企业软件的未来架构。这也为灵雀云这样的云原生技术产品公司创造了历史性机遇——成为新一代世界级的企业软件公司。（文/宁川）
