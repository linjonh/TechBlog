    <p>
     当返回类没有添加
     <code>
      getter
     </code>
     和
     <code>
      setter
     </code>
     方法时出现 HTTP 406 错误（Not Acceptable），主要与 Spring 框架处理响应数据的机制以及消息转换器的工作原理有关
     <br/>
     （注意前提，你已经添加了json相关库的依赖）
    </p>
    <h4>
     <a id="_3">
     </a>
     消息转换器的工作原理
    </h4>
    <p>
     在 Spring 框架中，当控制器方法返回一个对象时，Spring 会使用消息转换器（Message Converter）将该对象转换为客户端可接受的媒体类型（如 JSON、XML 等）。以常见的 JSON 格式为例，Spring 通常使用 Jackson 库来完成对象到 JSON 的转换。
    </p>
    <h4>
     <a id="_getter__setter__6">
     </a>
     缺少
     <code>
      getter
     </code>
     和
     <code>
      setter
     </code>
     方法导致的问题
    </h4>
    <h5>
     <a id="1__7">
     </a>
     1. 对象属性无法访问
    </h5>
    <p>
     Jackson 在将 Java 对象转换为 JSON 字符串时，依赖于对象的
     <code>
      getter
     </code>
     方法来获取对象的属性值。如果返回类没有提供
     <code>
      getter
     </code>
     方法，Jackson 就无法访问对象的属性，也就无法将这些属性转换为 JSON 格式的键值对。
    </p>
    <p>
     例如，有一个简单的
     <code>
      User
     </code>
     类：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">User</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">private</span> <span class="token class-name">String</span> name<span class="token punctuation">;</span>
    <span class="token keyword">private</span> <span class="token keyword">int</span> age<span class="token punctuation">;</span>
    <span class="token comment">// 没有提供 getter 和 setter 方法</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     当控制器方法返回
     <code>
      User
     </code>
     对象时，Jackson 无法获取
     <code>
      name
     </code>
     和
     <code>
      age
     </code>
     属性的值，就无法生成有效的 JSON 响应。
    </p>
    <h5>
     <a id="2__20">
     </a>
     2. 消息转换器无法处理
    </h5>
    <p>
     由于消息转换器无法将对象正确转换为客户端期望的媒体类型，服务器就无法提供符合客户端
     <code>
      Accept
     </code>
     头要求的响应。客户端请求的媒体类型（如
     <code>
      application/json
     </code>
     ）在服务器端无法得到正确的表示，从而触发
     <code>
      HttpMediaTypeNotAcceptableException
     </code>
     异常，返回 HTTP 406 错误。
    </p>
    <h4>
     <a id="_23">
     </a>
     解决办法
    </h4>
    <p>
     为返回类添加
     <code>
      getter
     </code>
     和
     <code>
      setter
     </code>
     方法，以便消息转换器能够正常访问和处理对象的属性。修改后的
     <code>
      User
     </code>
     类如下：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">User</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">private</span> <span class="token class-name">String</span> name<span class="token punctuation">;</span>
    <span class="token keyword">private</span> <span class="token keyword">int</span> age<span class="token punctuation">;</span>

    <span class="token comment">// 提供 getter 方法</span>
    <span class="token keyword">public</span> <span class="token class-name">String</span> <span class="token function">getName</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">return</span> name<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>

    <span class="token keyword">public</span> <span class="token keyword">int</span> <span class="token function">getAge</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">return</span> age<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>

    <span class="token comment">// 提供 setter 方法</span>
    <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setName</span><span class="token punctuation">(</span><span class="token class-name">String</span> name<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">this</span><span class="token punctuation">.</span>name <span class="token operator">=</span> name<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>

    <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setAge</span><span class="token punctuation">(</span><span class="token keyword">int</span> age<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">this</span><span class="token punctuation">.</span>age <span class="token operator">=</span> age<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这样，当控制器方法返回
     <code>
      User
     </code>
     对象时，Jackson 可以通过
     <code>
      getter
     </code>
     方法获取属性值，并将其转换为 JSON 格式的响应，避免了 HTTP 406 错误。
    </p>
    <p>
     此外，如果你使用的是 Lombok 库，可以通过添加
     <code>
      @Getter
     </code>
     和
     <code>
      @Setter
     </code>
     注解来自动生成
     <code>
      getter
     </code>
     和
     <code>
      setter
     </code>
     方法，简化代码：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">import</span> <span class="token import"><span class="token namespace">lombok<span class="token punctuation">.</span></span><span class="token class-name">Getter</span></span><span class="token punctuation">;</span>
<span class="token keyword">import</span> <span class="token import"><span class="token namespace">lombok<span class="token punctuation">.</span></span><span class="token class-name">Setter</span></span><span class="token punctuation">;</span>

<span class="token annotation punctuation">@Getter</span>
<span class="token annotation punctuation">@Setter</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">User</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">private</span> <span class="token class-name">String</span> name<span class="token punctuation">;</span>
    <span class="token keyword">private</span> <span class="token keyword">int</span> age<span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
