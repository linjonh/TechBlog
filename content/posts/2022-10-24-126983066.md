    <h2>
     <a id="_1">
     </a>
     简介
    </h2>
    <p>
     <code>
      Appium
     </code>
     是一个开源测试自动化框架，用于原生、 混合和移动 Web 应用程序。
    </p>
    <h2>
     <a id="_3">
     </a>
     安装
    </h2>
    <p>
     <code>
      Appium
     </code>
     安装方式有两种，一种是通过
     <code>
      npm
     </code>
     命令行安装，另一种是通过安装可视化工具。
    </p>
    <ul>
     <li>
      <p>
       <code>
        npm
       </code>
       安装
      </p>
      <p>
       <code>
        npm
       </code>
       安装方式稍微麻烦一点，需要先安装
       <code>
        npm
       </code>
       才能进行操作，想了解的可参考：
       <a href="http://appium.io/docs/en/about-appium/getting-started/" rel="nofollow">
        安装Appium
       </a>
      </p>
     </li>
     <li>
      <p>
       可视化工具安装
      </p>
      <p>
       可视化工具的下载可前往
       <a href="https://github.com/appium/appium-desktop/releases">
        Github下载
       </a>
      </p>
     </li>
    </ul>
    <p>
     本文使用的是 可视化工具安装 方式。
    </p>
    <h2>
     <a id="__Run_14">
     </a>
     配置 &amp; Run
    </h2>
    <p>
     安装完成后启动
     <code>
      Appium.exe
     </code>
     ，点击编辑配置
    </p>
    <p>

      <br/>
      在弹出的配置界面中填写已提前配置好的环境变量路径并保存重新启动
     </img>
    </p>

      <br/>

    <h2>
     <a id="___161">
     </a>
     问题 &amp; 解决方案
    </h2>
    <ul>
     <li>
      <p>
       no tasks available.
      </p>
      <p>
       <strong>
        原因
       </strong>
       ：没有配置
       <code>
        Run/Debug Configurations
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       Connection refused: connect.
      </p>
      <p>
       <strong>
        原因
       </strong>
       ：
       <code>
        Appium
       </code>
       服务未启动。
      </p>
     </li>
     <li>
      <p>
       An unknown server-side error occurred while processing the command. Original error: Unable to find an active device or emulator with OS 11. The following are available: 6HJ4C19A29009173 (10).
      </p>
      <p>
       <strong>
        原因
       </strong>
       ：
       <code>
        platformVersion
       </code>
       输入的操作系统与连接到的操作系统版本不对。
      </p>
     </li>
     <li>
      <p>
       An unknown server-side error occurred while processing the command. Original error: Could not find a connected Android device in 20088ms.
      </p>
      <p>
       <strong>
        原因
       </strong>
       ：未连接到测试设备，需要使用adb或数据线连接上设备，执行adb命令
       <code>
        adb devices
       </code>
       查看目前所连接到的设备，将设备名称拷贝粘贴至
       <code>
        deviceName
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       重新编写测试步骤的代码，应该提前
       <code>
        clean
       </code>
       运行缓存。
      </p>
     </li>
    </ul>
    <h2>
     <a id="___180">
     </a>
     优点 &amp; 缺点
    </h2>
    <p>
     <strong>
      优点
     </strong>
    </p>
    <ul>
     <li>
      支持第三方应用程序。
     </li>
     <li>
      支持同时测试多台设备。
     </li>
     <li>
      支持生成多种语言的测试代码。
     </li>
     <li>
      不需要把第三方代码编译进app中。
     </li>
     <li>
      解放了双手，避免了不必要的重复操作。
     </li>
     <li>
      支持IOS、Android、Windows的应用程序。
     </li>
     <li>
      支持Native apps、 Hybrid apps、Web apps。
     </li>
    </ul>
    <p>
     <strong>
      缺点
     </strong>
    </p>
    <ul>
     <li>
      暂不支持可用于编写Android程序的Kotlin代码。
     </li>
     <li>
      相对于人工测试而言，Appium自动测试不太适合小规模的测试。
     </li>
    </ul>
    <h2>
     <a id="_196">
     </a>
     总结
    </h2>
    <p>
     噢，相信你体验之后肯定会刷新你对
     <code>
      Appium
     </code>
     的看法。
    </p>
    <p>
     若测试的应用不需要进行多次重复性的UI测试，似乎没必要使用
     <code>
      Appium
     </code>
     自动化测试，与手动点击UI测试相比，速度慢的似乎不是一丁半点。
    </p>
    <p>
     最近了解到一个叫
     <code>
      AccessibilityService
     </code>
     （无障碍服务）的东西，这玩意可以用来开发Application脚本，脚本执行速度那叫一个快。只可惜它不是一个自动化测试的框架，不然用来搞自动化测试应该很牛掰的吧。
