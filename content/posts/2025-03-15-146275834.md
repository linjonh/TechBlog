    <h2>
     <a id="_2">
     </a>
     命令行参数
    </h2>
    <p>
     <mark>
      命令行参数就是main函数的参数，有两个
     </mark>
     <br/>
     ①argc：int类型，表示argv的元素个数
     <br/>
     ②argv：char*类型的数组，存储字符串
    </p>
    <h3>
     <a id="_10">
     </a>
     命令行参数有什么用呢？
    </h3>
    <p>
     例
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/af1c289dc7d24a4d8770cfc72bccd544.jpeg"/>
    </p>
    <p>
     由上图得：
     <br/>
     <mark>
      执行这个程序（code）的命令行中以空格分隔的字符串就会传入argv中进行存储
     </mark>
    </p>
    <p>
     这样可以干什么呢？
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/aaf1a875c64b4a75b02a102c90c80000.jpeg#pic_center"/>
    </p>
    <p>
     <mark>
      所以可以在main函数里面直接根据argv接收到的内容，来分情况进行一些处理
     </mark>
    </p>
    <p>
     <strong>
      我们常用的Linux指令的选项的功能，就是使用这样的方式进行实现的
     </strong>
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/16236bc11d7f46e9872cf0207ddb2012.jpeg"/>
    </p>
    <hr/>
    <h3>
     <a id="main_31">
     </a>
     main函数的参数是谁传递的?
    </h3>
    <p>
     子进程可以继承父进程的数据
     <br/>
     所以我们写得程序运行时，也能继承shell（bash）解析出来的argc和argv
    </p>
    <p>
     进程再以参数的形式传给main函数
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/49782302af744ae6bb24012abccadcf5.png#pic_center"/>
    </p>
    <hr/>
    <h2>
     <a id="_43">
     </a>
     环境变量
    </h2>
    <p>
     环境变量
     <br/>
     <mark>
      一般是指在操作系统中用来指定操作系统运行环境的一些参数
     </mark>
     <br/>
     <code>
      通常通过shell传递给所有进程
     </code>
    </p>
    <p>
     <strong>
      shell自己的环境变量是从操作系统的配置文件里面读取出来的
     </strong>
     <br/>
     shell再把读取出来的信息分析之后，再放入shell内部的全局的指针数组中
    </p>
    <p>
     <mark>
      我们在命令行上执行的命令
      <code>
       都是shell的子进程
      </code>
      ，绝大部分都是shell创建的子进程使用进程替换，搞出来的
     </mark>
    </p>
    <p>
     <strong>
      父进程的环境变量表可以被子进程继承
     </strong>
     <br/>
     并且可以把它作为参数传给main函数
     <br/>
     也就是下图的env
    </p>
    <hr/>
    <h3>
     <a id="_65">
     </a>
     环境变量的作用
    </h3>
    <p>
     环境变量的作用有很多，主要是看它里面存储了什么数据
     <br/>
     <mark>
      环境变量因为是变量，所以可以存储任意全局数据，这些数据就可以被一些进程（或者全部进程）看到
     </mark>
    </p>
    <p>
     如:
     <br/>
     我们在编写C/C++代码的时候，在链接的时候，从来不知道我们的所链接的动态静态库在哪里，但是照样可以链接成功，生成可执行程序，原因就是有
     <code>
      相关环境变量中记录了动态库的路径
     </code>
     帮助编译器进行查找。
    </p>
    <p>
     <mark>
      环境变量在系统当中通常具有全局特性
     </mark>
    </p>
    <hr/>
    <h3>
     <a id="_84">
     </a>
     简单介绍一下常见的环境变量
    </h3>
    <h4>
     <a id="HOME_85">
     </a>
     环境变量HOME：
    </h4>
    <p>
     作用：
     <code>
      记录当前用户的家目录
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/36d9931dc88a47639dec284292107135.png#pic_center">
      <br/>
      <mark>
       bash的cwd是会跟着用户的工作路径的变化而变化的
      </mark>
     </img>
    </p>
    <hr/>
    <h4>
     <a id="PWD_94">
     </a>
     环境变量PWD
    </h4>
    <p>
     <mark>
      记录当前用户的工作路径（
      <code>
       用户的工作路径就是bash的cwd
      </code>
      ）
     </mark>
    </p>
    <p>
     pwd其实是从自己这个进程的PCB中读取cwd进行打印的
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/de0b6ca97157465f8834ffd5ef613883.jpeg"/>
    </p>
    <hr/>
    <h4>
     <a id="USER_105">
     </a>
     环境变量USER
    </h4>
    <p>
     <mark>
      记录当前登陆的用户是谁
     </mark>
    </p>
    <p>
     <strong>
      我们可以通过这个环境变量和getenv函数，在进程中确定是谁在使用这个进程
     </strong>
    </p>
    <p>
     就可以使用条件判断，分类讨论
     <br/>
     例
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/e3a255be27a6483db6c234b1a8db89a4.jpeg">
      <br/>
      上面这个代码形成的进程，就只有用户who能正常使用
      <br/>
      其他用户，那片是root用户都不能正常使用
     </img>
    </p>
    <hr/>
    <h4>
     <a id="OLDPWD_119">
     </a>
     环境变量OLDPWD
    </h4>
    <p>
     <mark>
      记录用户最近的上一次所处的工作路径
     </mark>
    </p>
    <p>
     这也是命令
     <code>
      cd -
     </code>
     的实现依据
    </p>
    <hr/>
    <h4>
     <a id="_128">
     </a>
     环境变量相关指令和函数
    </h4>
    <p>
     函数：
     <code>
      getenv
     </code>
     <br/>
     头文件：
     <code>
      stdlib.h
     </code>
     <br/>
     参数：环境变量名
     <br/>
     返回值类型：char*
     <br/>
     作用：
     <mark>
      获取对应环境变量的值
     </mark>
    </p>
    <hr/>
    <p>
     函数：
     <code>
      putenv
     </code>
     <br/>
     头文件：
     <code>
      stdlib.h
     </code>
     <br/>
     参数：char*env（字符串），环境变量及其值
    </p>
    <p>
     作用：
     <mark>
      更改环境变量或者把一个自定义的环境变量，加入自己的环境变量表
     </mark>
    </p>
    <p>
     <strong>
      putenv加入环境变量之前，进程自己的环境变量表和父进程的环境表是同一张，是从父进程那里继承来的
     </strong>
    </p>
    <p>
     putenv加入环境变量之后，会触发写时拷贝进程自己的环境变量表就是只属于自己和自己创建子进程的了，因为
     <code>
      写时拷贝
     </code>
    </p>
    <hr/>
    <p>
     <mark>
      查询所有的环境变量的指令：env
     </mark>
    </p>
    <hr/>
    <p>
     指令：
     <code>
      set
     </code>
     <br/>
     <mark>
      查询所有的本地变量和环境变量
     </mark>
    </p>
    <hr/>
    <p>
     指令：
     <code>
      export
     </code>
     <br/>
     export 本地变量名：
     <code>
      可以把本地变量变成环境变量
     </code>
    </p>
    <p>
     export 变量名=值：
     <code>
      可以直接把这个变量定义成环境变量
     </code>
    </p>
    <hr/>
    <p>
     指令：
     <code>
      unset
     </code>
     <br/>
     语法：unset 环境变量名
     <br/>
     <mark>
      删除一个环境变量
     </mark>
    </p>
    <hr/>
    <p>
     全局变量：
     <code>
      environ
     </code>
     <br/>
     是C语言默认就有的一个全局变量，可以用来获取环境变量
     <br/>
     [注意：
     <mark>
      使用前必须先使用extern声明这个变量，不然编译不能通过
     </mark>
     ]
    </p>
    <p>
     类型：char**
     <br/>
     [=指向env数组的起始地址（注意不是指向env数组，因为它不是数组指针），这样它解引用就可以直接拿到环境变量==]
    </p>
    <hr/>
    <h3>
     <a id="_188">
     </a>
     本地变量
    </h3>
    <p>
     <mark>
      本地变量是给bash进程内部使用的
     </mark>
    </p>
    <p>
     可以直接在命令行中定义变量，此时这个变量就是bash的本地变量
    </p>
    <p>
     <mark>
      Linux中定义变量的方式统一是
     </mark>
     ：
     <code>
      变量名=值
     </code>
    </p>
    <hr/>
    <p>
     <strong>
      OS为用户启动bash进程之后
     </strong>
     ，
     <code>
      在内存中
     </code>
     <br/>
     ①bash会读取配置文件之后搞出env环境变量表，用来所有存储环境变量
     <br/>
     ②读取启动子进程的命令（字符串），生成argv命令行参数表
     <br/>
     ③根据自己的需要，生成一张本地变量表，用来存储所有本地变量
    </p>
    <p>
     <strong>
      所以本地变量变成环境变量
     </strong>
     <br/>
     <mark>
      只需要把这个本地变量从本地变量表中删除，再放进环境变量表中就可以了
     </mark>
    </p>
    <hr/>
    <p>
     <strong>
      因为bash等shell中可以定义变量，也可以识别for，while等语法
     </strong>
     <br/>
     <mark>
      所以shell也衍生出一门语言，也就是
      <code>
       shell脚本
      </code>
     </mark>
    </p>
    <hr/>
    <h3>
     <a id="_217">
     </a>
     环境变量于本地变量的区别
    </h3>
    <p>
     <mark>
      环境变量能被子进程继承
     </mark>
    </p>
    <p>
     bash的子进程再使用fork创建的子进程也能继承到bash的环境变量
     <br/>
     它再继续使用fork再创建，还能继承
    </p>
    <p>
     <mark>
      所以环境变量能被bash和以bash为源头衍生出的所有进程继承
      <br/>
      <code>
       所以环境变量具有全局属性
      </code>
     </mark>
    </p>
    <p>
     <mark>
      本地变量不能被子进程继承，只能在shell进程内部使用
     </mark>
    </p>
    <hr/>
    <h3>
     <a id="_235">
     </a>
     环境变量为什么要具有全局属性？
    </h3>
    <p>
     ①
     <code>
      因为环境变量本质是系统的配置信息
     </code>
     <br/>
     <mark>
      配置信息要生效，就必须是所有进程都遵守配置信息
     </mark>
     <br/>
     <strong>
      但是进程遵守的前提是，进程能看到配置信息
     </strong>
    </p>
    <p>
     <mark>
      所以环境变量必须具有全局属性，能让所以进程都能“看见”
     </mark>
    </p>
    <p>
     ②
     <code>
      进程之间具有独立性，但是有时候也需要信息交互
     </code>
     <br/>
     环境变量就是一种交互的方式
     <br/>
     父进程可以通过环境变量，把一些信息传递给子进程[
     <code>
      一般是只读数据
     </code>
     ]
    </p>
