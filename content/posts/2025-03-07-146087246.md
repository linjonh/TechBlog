    <p>
     近期收到大量读者反馈：上传pdf/图文PDF到Dify、Cherry Studio等知识库时，普遍存在
     <strong>
      格式错乱、图片丢失、表格失效
     </strong>
     三大痛点。
    </p>
    <p>
     在试用的几款知识库中除了
     <code>
      ragflow
     </code>
     具备图片解析的能力外，其他的都只能解析文本。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/d6f4623d3e3801181e76b7e7e0c4e674.png">
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/img_convert/405caae4b2216434a90b45c6c4a1368c.png"/>
     </img>
    </p>
    <p>
     如果想要解析扫描件，或者带图片的pdf或word，怎么办？
    </p>
    <p>
     <strong>
      通过工具将文档转成markdown
     </strong>
    </p>
    <p>
     私密性的文件，建议本地部署，本地模型，非私密性的文件，可以使用一些在线服务。
    </p>
    <h4>
     <a id="marker_12">
     </a>
     marker
    </h4>
    <p>
     https://github.com/VikParuchuri/marker
    </p>
    <p>
     在github上21.6k星
    </p>
    <h4>
     <a id="_16">
     </a>
     总结下用途
    </h4>
    <ul>
     <li>
      <strong>
       多格式转换
      </strong>
      ：支持 PDF、图像、PPTX/DOCX/XLSX/HTML/EPUB → Markdown/JSON/HTML
     </li>
     <li>
      <strong>
       复杂内容提取
      </strong>
      ：表格、公式、代码块、链接，支持图像保留与 OCR 修复，适配论文、金融文档等场景。
     </li>
     <li>
      <strong>
       可扩展优化
      </strong>
      ：通过
      <code>
       --use_llm
      </code>
      调用 Gemini/Ollama 等模型提升准确率，支持自定义处理规则。
     </li>
    </ul>
    <h4>
     <a id="_20">
     </a>
     优点
    </h4>
    <ol>
     <li>
      <strong>
       速度与精度
      </strong>
      ：单页平均处理仅 0.18 秒（H100 GPU），基准测试优于 Llamaparse/Mathpix；
     </li>
     <li>
      <strong>
       灵活性强
      </strong>
      ：兼容 GPU/CPU/MPS，支持批量并发（122 页/秒多 GPU 模式）；
     </li>
     <li>
      <strong>
       开源易扩展
      </strong>
      ：模块化设计，可自定义 Processor/Renderer。
     </li>
    </ol>
    <h4>
     <a id="_25">
     </a>
     缺点
    </h4>
    <ol>
     <li>
      <strong>
       文档复杂度限制
      </strong>
      ：复杂嵌套表格需依赖 LLM 辅助；
     </li>
     <li>
      <strong>
       商业化壁垒
      </strong>
      ：模型权重非商用，企业使用需营收&lt;$5M；
     </li>
     <li>
      <strong>
       硬件高需求
      </strong>
      ：GPU 加速依赖显存（单进程峰值 5GB）。
     </li>
    </ol>
    <h4>
     <a id="_30">
     </a>
     所需硬件资源
    </h4>
    <ul>
     <li>
      <strong>
       基础配置
      </strong>
      ：CPU/MPS（Mac 芯片）可轻量运行，但速度下降；
     </li>
     <li>
      <strong>
       推荐配置
      </strong>
      ：多 CUDA GPU（如 H100/A100）提升吞吐，显存≥16GB 优化并行；
     </li>
     <li>
      <strong>
       LLM 附加
      </strong>
      ：启用
      <code>
       --use_llm
      </code>
      需联网调用云服务（Gemini/Claude/Ollama）。
     </li>
    </ul>
    <h3>
     <a id="markitdown_35">
     </a>
     markitdown
    </h3>
    <p>
     https://github.com/microsoft/markitdown
    </p>
    <p>
     在github上39.4k星
    </p>
    <h4>
     <a id="_39">
     </a>
     用途
    </h4>
    <p>
     将多种文件类型（PDF、Office文档、图像、音频、HTML、ZIP、YouTube等）转换为Markdown格式，支持文本分析、索引生成等场景，并提供插件扩展能力。
    </p>
    <h4>
     <a id="_43">
     </a>
     优点
    </h4>
    <ol>
     <li>
      <strong>
       格式丰富
      </strong>
      ：支持超过20种文件格式（含音视频、图像OCR、YouTube字幕）。
     </li>
     <li>
      <strong>
       灵活安装
      </strong>
      ：按需选择依赖（如[pdf]/[docx]），减少冗余资源占用。
     </li>
     <li>
      <strong>
       扩展性强
      </strong>
      ：支持第三方插件（GitHub搜索
      <code>
       #markitdown-plugin
      </code>
      ），适配定制需求。
     </li>
     <li>
      <strong>
       集成AI能力
      </strong>
      ：结合Azure文档智能、OpenAI模型（如GPT-4o）增强OCR、音频转录与图像描述功能。
     </li>
     <li>
      <strong>
       多场景适配
      </strong>
      ：提供CLI、Python API、Docker镜像，适合本地开发与云环境。
     </li>
    </ol>
    <h4>
     <a id="_51">
     </a>
     缺点
    </h4>
    <ol>
     <li>
      <strong>
       依赖管理复杂
      </strong>
      ：需手动选择安装依赖项（默认不全覆盖），可能引发兼容性问题。
     </li>
     <li>
      <strong>
       外部服务依赖
      </strong>
      ：Azure/OpenAI功能需API密钥和网络支持，增加配置门槛。
     </li>
     <li>
      <strong>
       版本变更风险
      </strong>
      ：v0.0.1至v0.0.2存在破坏性更新，旧项目迁移需调整。
     </li>
    </ol>
    <h4>
     <a id="_57">
     </a>
     所需硬件资源
    </h4>
    <ul>
     <li>
      <strong>
       基础配置
      </strong>
      ：适用于常规文件处理（CPU/RAM需求低）。
     </li>
     <li>
      <strong>
       复杂任务
      </strong>
      ：大文件处理、AI模型调用（如OCR/语音转录）建议高配置CPU/内存。
     </li>
     <li>
      <strong>
       云服务依赖
      </strong>
      ：Azure或OpenAI功能需稳定网络环境及对应API资源。
     </li>
    </ul>
    <h3>
     <a id="MinerU_63">
     </a>
     MinerU
    </h3>
    <p>
     https://github.com/opendatalab/MinerU
    </p>
    <p>
     在github上27.1k星
    </p>
    <h4>
     <a id="_67">
     </a>
     用途
    </h4>
    <ul>
     <li>
      <strong>
       文档结构化转换
      </strong>
      ：PDF转Markdown/JSON，保留文本、表格、公式、图像结构。
     </li>
     <li>
      <strong>
       学术/技术处理
      </strong>
      ：精准提取论文公式（LaTeX）、表格（HTML）。
     </li>
     <li>
      <strong>
       多语言OCR
      </strong>
      ：支持84种语言的扫描文档识别。
     </li>
    </ul>
    <h4>
     <a id="_71">
     </a>
     优点
    </h4>
    <ol>
     <li>
      <strong>
       高效精准
      </strong>
      ：多模型加速（10倍+速度），跨页拼接优化。
     </li>
     <li>
      <strong>
       多模态支持
      </strong>
      ：图文表公式一体化提取。
     </li>
     <li>
      <strong>
       跨平台加速
      </strong>
      ：兼容CPU/GPU/NPU/MPS，灵活部署。
     </li>
    </ol>
    <h4>
     <a id="_75">
     </a>
     缺点
    </h4>
    <ol>
     <li>
      <strong>
       场景局限
      </strong>
      ：复杂布局（杂志/艺术册）、垂直文本支持不足。
     </li>
     <li>
      <strong>
       硬件门槛
      </strong>
      ：GPU需8GB+显存，NPU需华为Ascend生态。
     </li>
     <li>
      <strong>
       小语种误差
      </strong>
      ：变音符号或罕见字符OCR可能偏差。
     </li>
    </ol>
    <h4>
     <a id="_80">
     </a>
     所需硬件资源
    </h4>
    <ul>
     <li>
      <strong>
       最低配置
      </strong>
      ：8GB内存+SSD（CPU模式）。
     </li>
     <li>
      <strong>
       推荐配置
      </strong>
      ：
      <ul>
       <li>
        GPU：RTX 3060+（8GB显存）；
       </li>
       <li>
        NPU：华为Ascend 910B；
       </li>
       <li>
        Apple：M2芯片+16GB内存（禁用公式识别优化性能）。
       </li>
      </ul>
     </li>
    </ul>

    <h3>
     <a id="docling_93">
     </a>
     docling
    </h3>
    <p>
     https://github.com/DS4SD/docling
    </p>
    <p>
     在github上23.2k星，让deepseek联网搜索，竟然没有找到这个的产
    </p>
    <h4>
     <a id="_97">
     </a>
     用途
    </h4>
    <p>
     Docling 是文档处理工具，支持多格式解析（PDF、DOCX、HTML、图片等），重点增强 PDF 的布局、表格、代码、公式理解及 OCR 功能，提供统一文档结构表达，并与 LangChain、LlamaIndex 等 AI 框架集成，助力生成式 AI 应用开发。
    </p>
    <h4>
     <a id="_100">
     </a>
     优点
    </h4>
    <ol>
     <li>
      <strong>
       多格式支持
      </strong>
      ：覆盖 PDF、DOCX、HTML 等常见格式。
     </li>
     <li>
      <strong>
       高级 PDF 解析
      </strong>
      ：精准识别布局、表格、公式、图像分类。
     </li>
     <li>
      <strong>
       本地化执行
      </strong>
      ：支持敏感数据和断网环境。
     </li>
     <li>
      <strong>
       生态系统集成
      </strong>
      ：无缝对接 LangChain 等 AI 工具链。
     </li>
     <li>
      <strong>
       OCR 能力
      </strong>
      ：有效处理扫描件与图片文本提取。
     </li>
    </ol>
    <h4>
     <a id="_107">
     </a>
     缺点
    </h4>
    <ol>
     <li>
      <strong>
       复杂结构支持待完善
      </strong>
      ：图表、分子结构等理解功能标注为“即将推出”。
     </li>
     <li>
      <strong>
       外部模型依赖
      </strong>
      ：需遵守第三方模型许可协议，可能存在版权风险。
     </li>
     <li>
      <strong>
       实验性功能
      </strong>
      ：部分功能（如 VLM 管道）可能稳定性不足。
     </li>
    </ol>
    <h4>
     <a id="_112">
     </a>
     所需硬件资源
    </h4>
    <ul>
     <li>
      <strong>
       跨平台支持
      </strong>
      ：macOS/Linux/Windows（x86_64 及 arm64）。
     </li>
     <li>
      <strong>
       推荐配置
      </strong>
      ：未明确具体需求，但 OCR 与视觉模型可能需中等以上 CPU/GPU 资源，本地执行时需预留存储空间（依赖 HuggingFace 模型缓存）。
     </li>
     <li>
      <strong>
       轻量场景
      </strong>
      ：基础文档解析可在普通配置运行，复杂任务建议配备 GPU 加速。
     </li>
    </ul>

    <p>
     一个群友强烈安利我这款产品
     <code>
      Textln
     </code>
     ，有在线体验，注册可以免费解析100页，（我不会告诉你，我加了他们的小助手后，给我增加了1000页）。我拿着群友发的影印件体验了下。大家可以扫描体验。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/77a34f32086121e20fe6f6b799b5269a.png"/>

     <li>
      点击位置
      <code>
       1
      </code>
      上传文件，上传文件以后先不要执行
     </li>
     <li>
      点击位置
      <code>
       2
      </code>
      参数配置
     </li>
     <li>
      调整完参数点击
      <code>
       3
      </code>
     </li>
    </ul>

     <p>
      特别注意：它家的速度贼快，第一次没注意配置，上传一个99页的演示稿，直接点击识别，几秒就用了99页的免费额度。
