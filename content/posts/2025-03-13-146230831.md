    <h2>
     <a id="___10">
     </a>
     简述 / 前言
    </h2>
    <p>
     最近在 windows 上使用 Git 共享代码，发现 .gitignore 文件中标注的文件以及文件夹都不起作用，都会被 Git 追踪到，在网上也找了很多博客、也问了 Deepseek，发现都没有解决这个问题，最终发现
     <strong>
      是创建的
      <code>
       .gitignore
      </code>
      文件格式有问题
     </strong>
     ！
    </p>
    <p>
     解决方案就是：
     <strong>
      <code>
       使用 vscode 或者其它的编辑器来创建 .gitignore 文件
      </code>
     </strong>
     ，这个时候就能解决这个问题了！下面讲讲 debug 的心路历程，不感兴趣的小伙伴可以跳过了~
    </p>
    <h2>
     <a id="1__14">
     </a>
     1. 解决方案
    </h2>
    <p>
     <strong>
      使用 vscode 或者其它的编辑器来创建 .gitignore 文件
     </strong>
     ，
     <mark>
      不要使用
      <strong>
       记事本
      </strong>
      或者
      <strong>
       命令行（
       <code>
        echo "__pycache__/" &gt;&gt; .gitignore
       </code>
       ）
      </strong>
      创建 .gitignore 文件
     </mark>
     ！
    </p>
    <h2>
     <a id="2_debug__17">
     </a>
     2. debug 过程（心路历程）
    </h2>
    <p>
     网上搜集到的解决方案以及 Deepseek 给出的解决方案基本上都是使用下面这个命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">rm</span> <span class="token parameter variable">-r</span> <span class="token parameter variable">--cached</span> <span class="token builtin class-name">.</span>          <span class="token comment"># 清除所有缓存</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>                     <span class="token comment"># 重新添加文件</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"强制应用.gitignore"</span>
</code></pre>
    <p>
     但是我也没有
     <code>
      add
     </code>
     过我的代码和文件夹里面的内容啊，不过想着试试的原则，还是使用的这个命令，结果毫无意外的出现的报错信息：
    </p>
    <pre><code class="prism language-bash">fatal: pathspec <span class="token string">'.'</span> did not match any files
</code></pre>
    <p>
     然后 AI 又提示我检查以下信息：
    </p>
    <ul>
     <li>
      <code>
       .gitignore
      </code>
      文件必须位于仓库根目录，可以使用以下命令：
      <pre><code class="prism language-bash"><span class="token function">git</span> check-ignore <span class="token parameter variable">-v</span> <span class="token operator">&lt;</span>文件路径<span class="token operator">&gt;</span>
</code></pre>
     </li>
    </ul>
    <p>
     于是我观察我的根目录以及使用命令检查发现都没有问题，于是我想了想，会不会是我在它的父目录有
     <code>
      git
     </code>
     仓库，忘记删除了，于是从当前目录一直往上寻找，都发现没有问题，同时还用了 git 命令来查找有没有父目录干扰了现在的仓库：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> rev-parse --show-toplevel
</code></pre>
    <p>
     一切输出都显示我的
     <code>
      .gitignore
     </code>
     文件位置正确，没有语法错误。
    </p>
    <p>
     这是 Deepseek 又提示我，
     <strong>
      “若需统一忽略某些文件（如 IDE 配置文件），可创建全局规则：”
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> core.excludesfile ~/.gitignore_global
<span class="token builtin class-name">echo</span> <span class="token string">".DS_Store"</span> <span class="token operator">&gt;&gt;</span> ~/.gitignore_global
</code></pre>
    <p>
     好的，按照它的要求发现还是没啥变化，反而我的 Git 全局配置信息多了一堆东西。
    </p>
    <p>
     看着我的 Vscode，突然看到拓展插件那里，于是想着难道还和插件有关系吗？虽然我知道这应该和插件没什么关系，毕竟这些插件都只是显示或者可视化我的 Git 历史的，底层用的还是本机的 Git 系统，但是我就是还想试一下，把插件全部停用，重启 Vscode，发现并没有什么用。。。
    </p>
    <p>
     想了半天，我想着难道是我的 Git 系统出 bug 了吗？好吧，想着古老的办法：“重装系统”，当然不是重装我的 windows 系统，而是卸载 Git 再重装 Git，折腾了十来分钟，发现新版本的 Git 和之前的没啥区别。
    </p>
    <p>
     这是灵光一现，想着是不是 .gitignore 文件的格式有问题啊，因为我是在命令行创建 .gitignore 文件的：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token string">"__pycache__/"</span> <span class="token operator">&gt;&gt;</span> .gitignore
</code></pre>
    <p>
     这个代码在
     <code>
      linux
     </code>
     上面是可以使用的，但是在
     <code>
      Windows
     </code>
     上就会有问题！！！
     <br/>
     原因可能是
     <code>
      Windows
     </code>
     在创建文件是使用的是
     <code>
      UTF-16 LE
     </code>
     格式（当然可能这是我的系统的问题，没仔细研究过），那么这就导致 Git 并没有识别到这是 .gitignore 文件，于是不管你怎么重装、怎么折腾，你都没办法解决，也许另存为
     <code>
      UTF-8
     </code>
     格式有用，但是我试了一下，发现还是没用。
    </p>
    <p>
     最后就直接打算在 Vscode 中创建文件，这样应该能保证它是二进制文件了，没想到还真解决了 ^_^
    </p>
    <p>
     继续写我的标注网页去咯，希望科研和实习一切顺利吧~
    </p>
