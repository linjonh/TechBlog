    <p>
     最近，我决定为个人博客后台换一个编辑器。之前的富文本编辑器虽然功能齐全，但生成的 HTML 代码繁杂，维护起来非常麻烦。为了追求更简洁高效的写作体验，我开始研究 Markdown 编辑器，并最终选择了
     <code>
      marked.min.js
     </code>
     。
    </p>
    <hr/>
    <h3>
     1. 传统富文本 vs. Markdown 编辑器
    </h3>
    <p>
     在对比了传统富文本编辑器和 Markdown 编辑器后，我发现它们各有优缺点：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        传统富文本编辑器
       </th>
       <th>
        Markdown 编辑器
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        易用性
       </td>
       <td>
        简单直观
       </td>
       <td>
        需学习语法
       </td>
      </tr>
      <tr>
       <td>
        格式控制
       </td>
       <td>
        强大
       </td>
       <td>
        简洁清晰
       </td>
      </tr>
      <tr>
       <td>
        代码质量
       </td>
       <td>
        冗余复杂
       </td>
       <td>
        干净整洁
       </td>
      </tr>
      <tr>
       <td>
        维护性
       </td>
       <td>

       </td>

      </tr>

       </td>

     </tbody>

    <p>
     由于 Markdown 语法简洁，生成的 HTML 代码清爽易读，我最终选择了 Markdown 作为博客的主要编辑格式，并决定使用
     <code>
      marked.min.js
     </code>
     来解析 Markdown。
    </p>
    <hr/>
    <h3>
     2.
     <code>
      marked.min.js
     </code>
     介绍
    </h3>
    <p>
     <code>
      marked.min.js
     </code>
     是一个轻量级、高性能的 JavaScript 库，可以将 Markdown 文本快速转换为 HTML，适用于博客、文档系统等场景。
    </p>
    <h4>
     <strong>
      <code>
       marked.min.js
      </code>
      的特点：
     </strong>
    </h4>
    <p>
     ✅
     <strong>
      解析速度快
     </strong>
     ：比普通正则解析方式更高效
     <br/>
     ✅
     <strong>
      支持 GitHub 风格 Markdown
     </strong>
     ，兼容 CommonMark 规范
     <br/>
     ✅
     <strong>
      支持扩展
     </strong>
     ，可以自定义渲染方式，如代码高亮等
     <br/>
     ✅
     <strong>
      简单易用
     </strong>
     ，可直接解析 Markdown 并输出 HTML
    </p>
    <hr/>
    <h3>
     3. 在博客后台集成
     <code>
      marked.min.js
     </code>
    </h3>
    <h4>
     <strong>
      步骤 1：引入
      <code>
       marked.min.js
      </code>
     </strong>
    </h4>
    <p>
     可以通过 CDN 方式引入
     <code>
      marked.min.js
     </code>
     ，无需安装：
    </p>
    <pre><code class="language-javascript">&lt;script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"&gt;&lt;/script&gt;</code></pre>
    <hr/>
    <h4>
     <strong>
      步骤 2：创建 Markdown 编辑区域
     </strong>
    </h4>
    <p>
     在 HTML 页面中，使用
     <code>
      textarea
     </code>
     输入 Markdown，并在
     <code>
      div
     </code>
     中实时显示解析后的 HTML：
    </p>
    <pre><code class="language-javascript">&lt;textarea id="markdown-input" placeholder="输入 Markdown..."&gt;&lt;/textarea&gt; 
&lt;div id="html-output"&gt;&lt;/div&gt;</code></pre>
    <hr/>
    <h4>
     <strong>
      步骤 3：解析 Markdown 并实时更新 HTML
     </strong>
    </h4>
    <p>
     使用
     <code>
      marked()
     </code>
     函数，将
     <code>
      textarea
     </code>
     输入的 Markdown 解析为 HTML，并动态更新
     <code>
      div
     </code>
     内容：
    </p>
    <pre><code class="language-javascript">const markdownInput = document.getElementById('markdown-input');
const htmlOutput = document.getElementById('html-output');

markdownInput.addEventListener('input', () =&gt; {
  const markdownText = markdownInput.value;
  htmlOutput.innerHTML = marked.parse(markdownText);
});

// 设置默认 Markdown 示例
markdownInput.value = "# Hello Markdown!\n\n- **加粗**\n- *斜体*\n- `代码块`\n\n[点击这里](https://github.com/markedjs/marked)";
htmlOutput.innerHTML = marked.parse(markdownInput.value);
</code></pre>
    <hr/>
    <h4>
     <strong>
      步骤 4：优化和扩展功能
     </strong>
    </h4>
    <p>
     为了让 Markdown 编辑器更安全、更美观，我们可以进行以下优化：
    </p>
    <h5>
     ✅
     <strong>
      防止 XSS 攻击（HTML 清理）
     </strong>
    </h5>
    <p>
     由于
     <code>
      marked
     </code>
     直接解析 Markdown 可能会导致 XSS 攻击，我们可以使用
     <code>
      DOMPurify
     </code>
     清理 HTML：
    </p>
    <pre><code class="language-javascript">&lt;script src="https://cdnjs.cloudflare.com/ajax/libs/dompurify/2.4.0/purify.min.js"&gt;&lt;/script&gt;</code></pre>
    <pre><code class="language-javascript">htmlOutput.innerHTML = DOMPurify.sanitize(marked.parse(markdownText));</code></pre>
    <p>
    </p>
    <hr/>
    <h5>
     ✅
     <strong>
      支持代码高亮
     </strong>
    </h5>
    <p>
     为了让代码块更美观，我们可以使用
     <code>
      highlight.js
     </code>
     进行语法高亮：
    </p>
    <pre><code class="language-javascript">&lt;script src="https://cdn.jsdelivr.net/npm/highlight.js/lib/common/highlight.min.js"&gt;&lt;/script&gt; &lt;link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/highlight.js/styles/github.min.css"&gt;</code></pre>
    <pre><code class="language-javascript">marked.setOptions({ highlight: function (code) { return hljs.highlightAuto(code).value; } });</code></pre>
    <p>
    </p>
    <hr/>
    <h5>
     ✅
     <strong>
      自定义 Markdown 渲染方式
     </strong>
    </h5>
    <p>
     如果想自定义 Markdown 渲染方式（例如修改标题格式），可以使用
     <code>
      marked.use()
     </code>
     ：
    </p>
    <pre><code class="language-javascript">marked.use({ renderer: { heading(text, level) { return `&lt;h${level} style="color:blue;"&gt;${text}&lt;/h${level}&gt;`; } } });</code></pre>
    <p>
    </p>
    <hr/>
    <h3>
     4. 完整示例代码
    </h3>
    <pre></pre>
    <pre><code class="language-html">&lt;!DOCTYPE html&gt;
&lt;html lang="zh"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;meta name="viewport" content="width=device-width, initial-scale=1.0"&gt;
    &lt;title&gt;Markdown 编辑器&lt;/title&gt;
    &lt;script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"&gt;&lt;/script&gt;
    &lt;script src="https://cdnjs.cloudflare.com/ajax/libs/dompurify/2.4.0/purify.min.js"&gt;&lt;/script&gt;
    &lt;script src="https://cdn.jsdelivr.net/npm/highlight.js/lib/common/highlight.min.js"&gt;&lt;/script&gt;
    &lt;link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/highlight.js/styles/github.min.css"&gt;
    &lt;style&gt;
        body { font-family: Arial, sans-serif; margin: 20px; }
        textarea { width: 100%; height: 150px; padding: 10px; font-size: 16px; }
        #html-output { border: 1px solid #ccc; padding: 10px; margin-top: 10px; min-height: 100px; background: #f9f9f9; }
    &lt;/style&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h2&gt;Markdown 编辑器&lt;/h2&gt;
    &lt;textarea id="markdown-input" placeholder="输入 Markdown..."&gt;&lt;/textarea&gt;
    &lt;div id="html-output"&gt;&lt;/div&gt;

    &lt;script&gt;
        marked.setOptions({
            highlight: function (code) {
                return hljs.highlightAuto(code).value;
            }
        });

        const markdownInput = document.getElementById('markdown-input');
        const htmlOutput = document.getElementById('html-output');

        markdownInput.addEventListener('input', () =&gt; {
            const markdownText = markdownInput.value;
            htmlOutput.innerHTML = DOMPurify.sanitize(marked.parse(markdownText));
        });

        markdownInput.value = "# Hello Markdown!\n\n- **加粗**\n- *斜体*\n- `代码块`\n\n[点击这里](https://github.com/markedjs/marked)";
        htmlOutput.innerHTML = DOMPurify.sanitize(marked.parse(markdownInput.value));
    &lt;/script&gt;
&lt;/body&gt;
&lt;/html&gt;
</code></pre>
    <hr/>
    <h3>
     5. 总结
    </h3>
    <p>
     通过
     <code>
      marked.min.js
     </code>
     ，我们可以轻松地在博客后台实现 Markdown 编辑器，实现 Markdown 解析、HTML 预览，并结合
     <code>
      highlight.js
     </code>
     添加代码高亮，同时使用
     <code>
      DOMPurify
     </code>
     保障安全性。
    </p>
    <p>
     🔹
     <strong>
      为什么选择
      <code>
       marked.min.js
      </code>
      ？
     </strong>
     <br/>
     ✅ 解析速度快
     <br/>
     ✅ 代码清晰简洁
     <br/>
     ✅ 支持 GitHub 风格 Markdown
     <br/>
     ✅ 结合
     <code>
      highlight.js
     </code>
     让代码块更美观
     <br/>
     ✅ 可扩展性强，自定义渲染方式
    </p>
    <p>
     这样，我们就完成了一个轻量级的 Markdown 编辑器，非常适合用于博客后台或在线 Markdown 解析工具！🚀
