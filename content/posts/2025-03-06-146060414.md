    <h4 id="vhqAR">
     【一】知道居中元素的宽高
    </h4>
    <h5 id="7fuxd">
     absolute + 负margin
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.wrapBox5{
    width: 300px;
    height: 300px;
    border:1px solid red;
    position: relative;
}
.wrapItem5{
    width: 100px;
    height: 50px;
    position: absolute;
    background:yellow;
    top: 50%;
    left:50%;
    margin-top: -25px;
    margin-left: -50px;
}
</code></pre>
    <p>
     <strong>
      运行结果

    <h5 id="i2IAT">
     absolute + margin auto
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.wrapBox{

    <h5 id="Aq1Va">
     absolute + calc
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.wrapBox6{
    width: 300px;
    height: 300px;
    border:1px solid green;
    position: relative;
}
.wrapItem6{
    width: 100px;
    height: 50px;
    position: absolute;
    background:yellow;
    top: calc(50% - 25px);
    left:calc(50% - 50px);
}
</code></pre>

    <p>
     <strong>
      三种对比总结
     </strong>
    </p>
    <blockquote>
     <p>
      当居中元素知道宽高的时候，设置居中的方式比较简单单一。三种方法的本质是一样的，都是对居中元素进行绝对定位，在定位到上50%，左50%后再拉回居中元素的一半宽高实现真正的居中。三种方式的区别就在于拉回元素本身宽高的方式不同。
     </p>
    </blockquote>
    <hr/>
    <h4 id="0wHnX">
     【二】居中元素的宽高未知
    </h4>
    <h5 id="CjEv6">
     absolute + transform
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.wrapBox{
    width: 300px;
    height: 300px;
    background:#ddd;
    position: relative;
}
.wrapItem{
    width: 100px;
    height: 50px;
    background:green;
    position: absolute;
    top: 50%;
    left:50%;
    transform: translate(-50% , -50%);
}
</code></pre>
    <p>
     <strong>
      运行结果
     </strong>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/dc93f8c1cbb083b371196ef8ca435528.png"/>
    </p>
    <p>
     <strong>
      原理
     </strong>
    </p>
    <p>
     原理类似于已知宽高的实现方式，只不过当前居中元素宽高未知，所以需要自动获取当前居中元素的宽高。translate属性正好实现了该功能。
    </p>
    <p>
     <strong>
      优缺点
     </strong>
    </p>
    <p>
     优点：自动计算本身的宽高
    </p>
    <p>
     缺点：如果同时使用transform的其他属性会产生相互影响。
    </p>
    <p>
     所以：在不使用transform的其他属性时推荐使用该方式
    </p>
    <h5 id="sVMjc">
     flex布局
    </h5>
    ```plain .wrapBox2{ width: 300px; height: 300px; background: blue; display: flex; justify-content: center; align-items: center; } /*注意：即使不设置子元素为行块元素也不会独占一层*/ .wrapItem2{ width: 100px; height: 50px; background:green; } ```
    <p>
     <strong>
      运行结果
     </strong>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/da71ae1143f1ca501bfadf8a310ee2d0.png"/>
    </p>
    <p>
     <strong>
      原理
     </strong>
    </p>
    <p>
     将父级元素设置为流式布局，根据flex布局的属性特性，设置子元素居中。
    </p>
    <p>
     <strong>
      优缺点
     </strong>
    </p>
    <p>
     优点：flex布局灵活，不需要对子元素进行任何的设置
    </p>
    <p>
     缺点：具有兼容性。子元素的float、clear、position等无法使用，如果同时具有其他布局，容易产生影响。
    </p>
    <h5 id="RwKAI">
     table-cell布局
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.wrapBox3{
    width: 300px;
    height: 300px;
    background: yellow;
    display: table-cell;
    vertical-align: middle;
    text-align: center;
}
.wrapItem3{
    width: 100px;
    height: 50px;
    background:green;
    display: inline-block;
}
</code></pre>
    <p>
     <strong>
      运行结果
     </strong>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/c3859bf8aef10be7d4ec7301c5cb8858.png"/>
    </p>
    <p>
     <strong>
      原理
     </strong>
    </p>
    <p>
     根据table的vertical-align属性，可以在表格元素内设置元素居中，再根据text-align设置水平居中
    </p>
    <h5 id="8v1sh">
     table元素
    </h5>
    **代码实现**
    <pre><code class="prism language-plain">.tableBox{
    border:2px solid yellow;
    width: 300px;
    height: 300px;
}
.tableBox table{
    width:100%;
    height:100%;
}
.centerWrap{
    text-align: center;
    vertical-align: middle;
}
.centerItem{
    display: inline-block;
    background:pink;
}
</code></pre>
    <p>
     <strong>
      运行结果
     </strong>
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/78af80eb3662f504c3ca1a3e04c2675e.png"/>
    </p>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <p>
     使用table标签进行布局，主要还是使用了vertical-align属性和text-align属性。但是相对于上一种方式，使用table标签会产生大量的冗余代码。不推荐使用
    </p>
