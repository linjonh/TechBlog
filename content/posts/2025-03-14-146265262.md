    <h3>
     问题 1：克隆时无法连接 GitHub
    </h3>
    <pre>Cloning into 'OpenManus'... fatal: unable to access 'https://github.com/mannaandpoem/OpenManus.git/': Failed to connect to github.com port 443 after 21074 ms: Could not connect to server 克隆项目时</pre>
    <p>
     错误表明无法连接到 github.com 的 443 端口（HTTPS），超时约 21 秒。可能原因包括网络问题、防火墙限制或 GitHub 服务器状态。建议：
    </p>
    <ol>
     <li>
      <p>
       检查网络连接，尝试访问
       <a href="https://github.com/" title="https://github.com">
        https://github.com
       </a>
       。
      </p>
     </li>
     <li>
      <p>
       验证仓库地址是否正确。
      </p>
     </li>
     <li>
      <p>
       检查 GitHub 状态：
       <a href="https://www.githubstatus.com/" rel="nofollow" title="GitHub Status">
        GitHub Status
       </a>
       。
      </p>
     </li>
     <li>
      <p>
       测试其他网络或使用 SSH：
      </p>
      <p>
       bash
      </p>
      <p>
       收起自动换行复制
      </p>
      <p>
       <code>
        git clone git@github.com:mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       调整 Git 配置或使用 curl -v 调试。
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     问题 2：如何为 Git 配置代理
    </h3>
    <pre>我使用了代理，git应该如何配置代理</pre>
    <p>
     Git 支持通过 http.proxy 和 https.proxy 配置代理。步骤如下：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        确认代理信息
       </strong>
       ：例如
       <a href="http://proxy.example.com:8080/" rel="nofollow" title="http://proxy.example.com:8080">
        http://proxy.example.com:8080
       </a>
       。
      </p>
     </li>
     <li>
      <p>
       全局配置
      </p>
      <p>
       <code>
        git config --global http.proxy http://proxy.example.com:8080 git config --global https.proxy http://proxy.example.com:8080
       </code>
      </p>
      <ul>
       <li>
        <p>
         带用户名密码：
        </p>
        <p>
         <code>
          git config --global http.proxy http://user:pass@proxy.example.com:8080
         </code>
        </p>
       </li>
       <li>
        <p>
         SOCKS5 代理：
        </p>
        <p>
         <code>
          git config --global http.proxy socks5://proxy.example.com:1080
         </code>
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       单次命令
      </p>
      <p>
       <code>
        git clone -c http.proxy=http://proxy.example.com:8080 https://github.com/mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       环境变量
      </p>
      <p>
       <code>
        export http_proxy=http://proxy.example.com:8080 export https_proxy=http://proxy.example.com:8080
       </code>
      </p>
     </li>
     <li>
      <p>
       验证
      </p>
      <p>
       <code>
        git config --global --get http.proxy
       </code>
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     问题 3：不设置代理会怎样
    </h3>
    <pre>如果不设置代理</pre>
    <p>
     不设置代理时，Git 直接通过本地网络连接 github.com：
    </p>
    <ul>
     <li>
      <p>
       如果网络无限制，克隆应成功。
      </p>
     </li>
     <li>
      <p>
       如果网络屏蔽 GitHub（常见于受限地区），会失败。 建议：
      </p>
     </li>
    </ul>
    <ol>
     <li>
      <p>
       测试网络：ping github.com 和 curl
       <a href="https://github.com/" title="https://github.com">
        https://github.com
       </a>
       。
      </p>
     </li>
     <li>
      <p>
       清除代理：
      </p>
      <p>
       <code>
        git config --global --unset http.proxy git config --global --unset https.proxy
       </code>
      </p>
     </li>
     <li>
      <p>
       尝试 SSH：
      </p>
      <p>
       <code>
        git clone git@github.com:mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       修改 hosts 文件（临时绕过）：
      </p>
      <p>
       <code>
        140.82.121.3 github.com
       </code>
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     问题 4：再次尝试克隆仍失败
    </h3>
    <pre>$ git clone https://github.com/mannaandpoem/OpenManus.git Cloning into 'OpenManus'... fatal: unable to access 'https://github.com/mannaandpoem/OpenManus.git/': Failed to connect to github.com port 443 after 21059 ms: Could not connect to server 我都没有设置代理</pre>
    <p>
     确认未设置代理，但仍无法连接 443 端口。建议：
    </p>
    <ol>
     <li>
      <p>
       测试网络：
      </p>
      <p>
       <code>
        ping github.com curl -v https://github.com
       </code>
      </p>
     </li>
     <li>
      <p>
       检查代理残留：
      </p>
      <p>
       <code>
        git config --global --get http.proxy
       </code>
      </p>
     </li>
     <li>
      <p>
       使用 SSH（需配置密钥）：
      </p>
      <p>
       <code>
        ssh-keygen -t rsa -b 4096 -C "你的邮箱" git clone git@github.com:mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       切换网络测试。
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     问题 5：Ping 测试结果
    </h3>
    <pre>ping github.com 正在 Ping github.com [20.205.243.166] 具有 32 字节的数据: 来自 20.205.243.166 的回复: 字节=32 时间=185ms TTL=112 来自 20.205.243.166 的回复: 字节=32 时间=101ms TTL=112 来自 20.205.243.166 的回复: 字节=32 时间=138ms TTL=112 来自 20.205.243.166 的回复: 字节=32 时间=164ms TTL=112 20.205.243.166 的 Ping 统计信息:    数据包: 已发送 = 4，已接收 = 4，丢失 = 0 (0% 丢失)， 往返行程的估计时间(以毫秒为单位):    最短 = 101ms，最长 = 185ms，平均 = 147ms</pre>
    <p>
     Ping 成功，说明网络可达 20.205.243.166，但 HTTPS（443 端口）可能被阻断。建议：
    </p>
    <ol>
     <li>
      <p>
       测试 HTTPS：
      </p>
      <p>
       <code>
        curl -v https://github.com
       </code>
      </p>
     </li>
     <li>
      <p>
       检查本地防火墙。
      </p>
     </li>
     <li>
      <p>
       再次尝试克隆或使用 SSH。
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     问题 6：Curl 测试结果
    </h3>
    <pre>curl -v https://github.com * Host github.com:443 was resolved. * IPv6: (none) * IPv4: 20.205.243.166 *   Trying 20.205.243.166:443... * connect to 20.205.243.166 port 443 from 0.0.0.0 port 1113 failed: Timed out * Failed to connect to github.com port 443 after 21115 ms: Could not connect to server * closing connection #0 curl: (28) Failed to connect to github.com port 443 after 21115 ms: Could not connect to server是不是</pre>
    <p>
     域名解析成功，但 443 端口连接超时，确认网络屏蔽 HTTPS 流量。解决方案：
    </p>
    <ol>
     <li>
      <p>
       使用代理
      </p>
      <p>
       <code>
        git config --global http.proxy http://127.0.0.1:1111 git config --global https.proxy http://127.0.0.1:111 git clone https://github.com/mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       使用 SSH
      </p>
      <p>
       <code>
        git clone git@github.com:mannaandpoem/OpenManus.git
       </code>
      </p>
     </li>
     <li>
      <p>
       切换网络。
      </p>
     </li>
     <li>
      <p>
       修改 hosts（效果有限）。
      </p>
     </li>
    </ol>
    <hr/>
    <h3>
     总结
    </h3>
    <p>
     用户无法直接通过 HTTPS 克隆 GitHub 仓库，原因是网络屏蔽了 443 端口流量。Ping 成功但 HTTPS 失败表明限制针对特定端口。推荐解决方案：
    </p>
    <ul>
     <li>
      <p>
       配置代理（需提供代理信息）。
      </p>
     </li>
     <li>
      <p>
       使用 SSH（需配置密钥）。
      </p>
     </li>
     <li>
      <p>
       切换无限制网络。
      </p>
     </li>
    </ul>
