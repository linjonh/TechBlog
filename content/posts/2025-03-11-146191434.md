    <h3>
     1. 算法效率
    </h3>
    <h4>
     1.1 如何衡量一个算法的好坏？
    </h4>
    <p>
     在计算机程序设计中，衡量算法优劣的核心标准是
     <strong>
      效率
     </strong>
     。但效率不仅指运行速度，还需要综合以下因素：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        时间因素
       </strong>
       ：算法执行所需时间
      </p>
     </li>
     <li>
      <p>
       <strong>
        空间因素
       </strong>
       ：算法运行占用的内存空间
      </p>
     </li>
     <li>
      <p>
       <strong>
        正确性
       </strong>
       ：算法能否正确处理所有输入
      </p>
     </li>
     <li>
      <p>
       <strong>
        可读性
       </strong>
       ：代码是否易于理解和维护
      </p>
     </li>
     <li>
      <p>
       <strong>
        健壮性
       </strong>
       ：能否处理非法输入
      </p>
     </li>
    </ul>
    <p>
     其中，
     <strong>
      时间复杂度
     </strong>
     和
     <strong>
      空间复杂度
     </strong>
     是理论分析中最重要的两个指标，它们直接反映了算法的效率本质。
    </p>
    <h4>
     1.2 算法的复杂度
    </h4>
    <p>
     算法复杂度分为两个维度：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        时间复杂度
       </strong>
       ：算法执行时间随数据规模增长的趋势
      </p>
     </li>
     <li>
      <p>
       <strong>
        空间复杂度
       </strong>
       ：算法运行所需内存空间随数据规模增长的趋势
      </p>
     </li>
    </ul>
    <p>
     现代计算机的存储容量普遍较大，因此我们更关注
     <strong>
      时间复杂度
     </strong>
     的优化。
    </p>
    <hr/>
    <h3>
     2. 时间复杂度
    </h3>
    <h4>
     2.1 时间复杂度的概念
    </h4>
    <p>
     <strong>
      时间复杂度
     </strong>
     不是计算程序的具体运行时间，而是描述算法中
     <strong>
      基本操作的执行次数
     </strong>
     与
     <strong>
      问题规模n
     </strong>
     之间的数学关系。
    </p>
    <h5>
     示例分析
    </h5>
    <pre><code>void Func(int n) {
    int count = 0;
    for (int i = 0; i &lt; n; ++i) {
        for (int j = 0; j &lt; n; ++j) {
            ++count;  // 执行n²次
        }
    }
    for (int k = 0; k &lt; 2 * n; ++k) {
        ++count;     // 执行2n次
    }
    int m = 10;
    while (m--) {
        ++count;     // 执行10次
    }
}</code></pre>
    <p>
     总执行次数：F(n)=n2+2n+10F(n)=n2+2n+10
     <br/>
     当n趋近于无穷大时，n2n2 起主导作用，时间复杂度为 O(n2)O(n2)。
    </p>
    <hr/>
    <h4>
     2.2 大O的渐进表示法
    </h4>
    <p>
     大O表示法描述算法的最坏情况复杂度，核心规则：
    </p>
    <ol>
     <li>
      <p>
       用常数1取代所有加法常数
      </p>
     </li>
     <li>
      <p>
       只保留最高阶项
      </p>
     </li>
     <li>
      <p>
       去除最高阶项的系数
      </p>
     </li>
    </ol>
    <h5>
     常见时间复杂度对比
    </h5>
    <p>
     <img alt="" height="281" src="https://i-blog.csdnimg.cn/direct/00f10f3230b44a36844ae842ba5cc6ef.png" width="320"/>
    </p>
    <h5>
     示例代码
    </h5>
    <pre><code>// O(1) 常数阶
int x = 100, y = 200;
int z = x + y;

// O(n) 线性阶
for(int i=0; i&lt;n; i++){
    printf("%d",i);
}

// O(n²) 平方阶
for(int i=0; i&lt;n; i++){
    for(int j=0; j&lt;n; j++){
        printf("%d",i+j);
    }
}

// O(log n) 对数阶
int i = 1;
while(i &lt; n){
    i *= 2;
}</code></pre>
    <hr/>
    <h3>
     3. 空间复杂度
    </h3>
    <h4>
     3.1 空间复杂度定义
    </h4>
    <p>
     空间复杂度衡量算法运行过程中
     <strong>
      临时占用的存储空间大小
     </strong>
     ，同样使用大O表示法。包括：
    </p>
    <ul>
     <li>
      <p>
       局部变量占用的空间
      </p>
     </li>
     <li>
      <p>
       递归调用的栈空间
      </p>
     </li>
    </ul>
    <h5>
     示例分析
    </h5>
    <pre><code>// O(1) 空间复杂度
void BubbleSort(int* arr, int n) {
    for(int end=n; end&gt;0; --end){
        int flag = 0;
        for(int i=1; i&lt;end; ++i){
            if(arr[i-1]&gt;arr[i]){
                Swap(&amp;arr[i-1], &amp;arr[i]);
                flag = 1;
            }
        }
    }
}

// O(n) 空间复杂度（递归深度）
long long Fac(int n) {
    if(n == 0) return 1;
    return n * Fac(n-1);
}

// O(n) 空间复杂度（动态数组）
int* Fibonacci(int n) {
    int* fib = (int*)malloc(n * sizeof(int));
    fib[0] = 0; fib[1] = 1;
    for(int i=2; i&lt;n; ++i){
        fib[i] = fib[i-1] + fib[i-2];
    }
    return fib;
}</code></pre>

    <h3>
     5. 实际开发建议
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        时间与空间的权衡
       </strong>
       ：在内存充足时优先优化时间效率
      </p>
     </li>
     <li>
      <p>
       <strong>
        递归算法的代价
       </strong>
       ：递归可能带来O(n)的空间复杂度，需警惕栈溢出
      </p>
     </li>
     <li>
      <p>
       <strong>
        避免最优陷阱
       </strong>
       ：理论上最优的算法不一定适合实际工程场景
      </p>
     </li>
    </ol>
    <p>
     理解复杂度分析，能帮助我们在设计算法时做出更明智的选择。通过大O分析，可以快速评估算法在不同数据规模下的表现，这是每个程序员必须掌握的核心技能。
    </p>
