    <h2>
     一、知识回顾
    </h2>
    <h3>
     1.单例和多例
    </h3>
    <ol>
     <li>
      单例（Singleton）是SpringBoot的
      <strong>
       默认
      </strong>
      作用域，在整个Spring应用上下文中，单例作用域的Bean
      <strong>
       只会创建一个实例
      </strong>
      ，并且所有对该Bean的请求都会返回这个
      <strong>
       唯一
      </strong>
      的实例
     </li>
     <li>
      多例（Prototype）作用域表示每次从Spring容器中请求该Bean时，都会
      <strong>
       创建一个新的实例
      </strong>
     </li>
    </ol>
    <h2>
     二、面试实战
    </h2>
    <blockquote>
     <p>
      <u>
       面试题1：Spring框架中的单例bean是线程安全的吗？
      </u>
     </p>
     <ul>
      <li>
       <strong>
        不是线程安全的
       </strong>
      </li>
      <li>
       Spring框架中有一个
       <strong>
        @Scope
       </strong>
       注解，
       <strong>
        默认
       </strong>
       的值就是Singleton，为
       <strong>
        单例
       </strong>
       的
      </li>
      <li>
       因为一般在Spring的bean中注入的都是
       <strong>
        无状态
       </strong>
       对象（Service/Dao），无线程安全问题
      </li>
      <li>
       如果在bean中定义了
       <strong>
        可修改的成员变量
       </strong>
       ，是要考虑线程安全问题的，可以使用
       <strong>
        多例
       </strong>
       或者
       <strong>
        加锁
       </strong>
       来解决
      </li>
     </ul>
    </blockquote>
