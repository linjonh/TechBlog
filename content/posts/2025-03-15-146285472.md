    <h2>
     <a id="Axios__0">
     </a>
     Axios 请求取消：从原理到实践
    </h2>
    <p>
     在现代前端开发中，网络请求是不可或缺的一部分。Axios 是一个基于 Promise 的 HTTP 客户端，广泛应用于浏览器和 Node.js 环境中。然而，在某些场景下，我们可能需要取消正在进行的请求，例如用户在请求完成前跳转到其他页面，或者重复触发相同的请求时取消之前的请求。本文将深入探讨 Axios 请求取消的原理，并通过一个二次封装的例子来演示如何实现请求取消。
    </p>
    <h3>
     <a id="1__4">
     </a>
     1. 请求取消的原理
    </h3>
    <p>
     Axios 的请求取消功能依赖于
     <code>
      CancelToken
     </code>
     。
     <code>
      CancelToken
     </code>
     是一个用于取消请求的令牌，它可以通过
     <code>
      CancelToken.source()
     </code>
     方法创建。每个
     <code>
      CancelToken
     </code>
     实例都有一个
     <code>
      token
     </code>
     和一个
     <code>
      cancel
     </code>
     方法。当调用
     <code>
      cancel
     </code>
     方法时，与该
     <code>
      token
     </code>
     关联的请求将被取消。
    </p>
    <h4>
     <a id="11_CancelToken__8">
     </a>
     1.1 CancelToken 的工作原理
    </h4>
    <ol>
     <li>
      <strong>
       创建 CancelToken
      </strong>
      : 通过
      <code>
       CancelToken.source()
      </code>
      方法创建一个
      <code>
       CancelToken
      </code>
      实例，该实例包含一个
      <code>
       token
      </code>
      和一个
      <code>
       cancel
      </code>
      方法。
     </li>
     <li>
      <strong>
       关联请求
      </strong>
      : 在发起请求时，将
      <code>
       token
      </code>
      传递给 Axios 请求配置中的
      <code>
       cancelToken
      </code>
      字段。
     </li>
     <li>
      <strong>
       取消请求
      </strong>
      : 当需要取消请求时，调用
      <code>
       cancel
      </code>
      方法，Axios 会中断与该
      <code>
       token
      </code>
      关联的请求。
     </li>
    </ol>
    <h4>
     <a id="12__14">
     </a>
     1.2 取消请求的流程
    </h4>
    <ol>
     <li>
      用户触发某个操作，发起一个请求。
     </li>
     <li>
      在请求完成之前，用户触发了另一个操作，需要取消之前的请求。
     </li>
     <li>
      调用
      <code>
       cancel
      </code>
      方法，Axios 中断之前的请求。
     </li>
     <li>
      发起新的请求。
     </li>
    </ol>
    <h3>
     <a id="2__Axios__21">
     </a>
     2. 二次封装 Axios 实现请求取消
    </h3>
    <p>
     为了更好地管理请求取消逻辑，我们可以对 Axios 进行二次封装。以下是一个简单的封装示例，展示了如何在封装中实现请求取消功能。
    </p>

     <a id="2__174">
     </a>
     2. 请求取消的实现原理
    </h3>
    <h4>
     <a id="21_CancelToken__176">
     </a>
     2.1 CancelToken 的作用
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        CancelToken.source()
       </code>
      </strong>
      : 创建一个
      <code>
       CancelTokenSource
      </code>
      对象，包含
      <code>
       token
      </code>
      和
      <code>
       cancel
      </code>
      方法。
     </li>
     <li>
      <strong>
       <code>
        token
       </code>
      </strong>
      : 用于关联请求。
     </li>
     <li>
      <strong>
       <code>
        cancel
       </code>
      </strong>
      : 用于取消请求。
     </li>
    </ul>
    <h4>
     <a id="22__182">
     </a>
     2.2 请求取消的流程
    </h4>
    <ol>
     <li>
      发起请求时，创建一个
      <code>
       CancelTokenSource
      </code>
      ，并将其存储到
      <code>
       cancelTokenMap
      </code>
      中。
     </li>
     <li>
      如果需要取消请求，调用
      <code>
       cancelRequest
      </code>
      或
      <code>
       cancelAllRequests
      </code>
      方法。
     </li>
     <li>
      调用
      <code>
       cancel
      </code>
      方法后，Axios 会中断与该
      <code>
       token
      </code>
      关联的请求，并抛出一个
      <code>
       Cancel
      </code>
      错误。
     </li>
     <li>
      在响应拦截器中，清理已完成的请求记录。
     </li>
    </ol>
