    <p>
     Java 的集合框架（Java Collections Framework, JCF）是 Java 中用于存储和操作数据结构的核心库，提供了丰富的接口和实现类，用于处理不同类型的集合数据。以下是详细的介绍：
    </p>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     一、集合框架的体系结构
    </h4>
    <p>
     Java 集合主要分为两大接口：
    </p>
    <ol>
     <li>
      <strong>
       <code>
        Collection
       </code>
       接口
      </strong>
      ：存储单一元素。
      <ul>
       <li>
        子接口：
        <code>
         List
        </code>
        （有序可重复）、
        <code>
         Set
        </code>
        （无序不可重复）、
        <code>
         Queue
        </code>
        （队列）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        Map
       </code>
       接口
      </strong>
      ：存储键值对（Key-Value）。
     </li>
    </ol>
    <p>

    </p>
    <hr/>
    <h4>
     <a id="Collection__14">
     </a>
     二、
     <code>
      Collection
     </code>
     接口及实现类
    </h4>
    <h5>
     <a id="1_List_16">
     </a>
     1.
     <strong>
      <code>
       List
      </code>
      （有序、可重复）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：元素按插入顺序存储，允许重复。
     </li>
     <li>
      <strong>
       常用实现类
      </strong>
      ：
      <ul>
       <li>
        <strong>
         <code>
          ArrayList
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于动态数组实现，支持快速随机访问（通过索引）。
         </li>
         <li>
          初始容量为 10，扩容时增长 50%（
          <code>
           newCapacity = oldCapacity + oldCapacity &gt;&gt; 1
          </code>
          ）。
         </li>
         <li>
          线程不安全，适用于读多写少的场景。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          LinkedList
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于双向链表实现，插入和删除效率高（时间复杂度 O(1)）。
         </li>
         <li>
          支持队列（
          <code>
           Queue
          </code>
          ）和双端队列（
          <code>
           Deque
          </code>
          ）操作。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          Vector
         </code>
        </strong>
        （已过时）：
        <ul>
         <li>
          线程安全的动态数组，所有方法用
          <code>
           synchronized
          </code>
          修饰，性能较差。
         </li>
         <li>
          替代方案：使用
          <code>
           Collections.synchronizedList(new ArrayList&lt;&gt;())
          </code>
          或
          <code>
           CopyOnWriteArrayList
          </code>
          （并发场景）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2_Set_30">
     </a>
     2.
     <strong>
      <code>
       Set
      </code>
      （无序、不可重复）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：元素唯一，不保证顺序。
     </li>
     <li>
      <strong>
       常用实现类
      </strong>
      ：
      <ul>
       <li>
        <strong>
         <code>
          HashSet
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于
          <code>
           HashMap
          </code>
          实现，元素存储在键的位置（值用
          <code>
           PRESENT
          </code>
          对象占位）。
         </li>
         <li>
          依赖
          <code>
           hashCode()
          </code>
          和
          <code>
           equals()
          </code>
          保证唯一性。
         </li>
         <li>
          时间复杂度：添加、删除、查询均为 O(1)。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          LinkedHashSet
         </code>
        </strong>
        ：
        <ul>
         <li>
          继承
          <code>
           HashSet
          </code>
          ，内部通过链表维护插入顺序。
         </li>
         <li>
          适合需要按插入顺序遍历的场景。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          TreeSet
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于红黑树（
          <code>
           TreeMap
          </code>
          ）实现，元素按自然顺序或自定义
          <code>
           Comparator
          </code>
          排序。
         </li>
         <li>
          时间复杂度：添加、删除、查询均为 O(log n)。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="3_Queue_44">
     </a>
     3.
     <strong>
      <code>
       Queue
      </code>
      （队列）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：先进先出（FIFO）或优先级的元素处理。
     </li>
     <li>
      <strong>
       常用实现类
      </strong>
      ：
      <ul>
       <li>
        <strong>
         <code>
          LinkedList
         </code>
        </strong>
        ：可作为普通队列使用。
       </li>
       <li>
        <strong>
         <code>
          PriorityQueue
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于堆结构实现，元素按优先级排序。
         </li>
         <li>
          自然顺序或通过
          <code>
           Comparator
          </code>
          定义顺序。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          ArrayDeque
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于循环数组实现的双端队列，适合高效的头尾操作。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="Map__56">
     </a>
     三、
     <code>
      Map
     </code>
     接口及实现类
    </h4>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：键值对存储，键唯一。
     </li>
     <li>
      <strong>
       常用实现类
      </strong>
      ：
      <ul>
       <li>
        <strong>
         <code>
          HashMap
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于数组+链表/红黑树（Java 8+），键的哈希值决定存储位置。
         </li>
         <li>
          允许
          <code>
           null
          </code>
          键和
          <code>
           null
          </code>
          值，线程不安全。
         </li>
         <li>
          扩容机制：默认容量 16，负载因子 0.75（容量达到阈值时扩容为 2 倍）。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          LinkedHashMap
         </code>
        </strong>
        ：
        <ul>
         <li>
          继承
          <code>
           HashMap
          </code>
          ，通过链表维护插入顺序或访问顺序（LRU 缓存）。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          TreeMap
         </code>
        </strong>
        ：
        <ul>
         <li>
          基于红黑树实现，键按自然顺序或自定义
          <code>
           Comparator
          </code>
          排序。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          Hashtable
         </code>
        </strong>
        （已过时）：
        <ul>
         <li>
          线程安全的哈希表，被
          <code>
           ConcurrentHashMap
          </code>
          取代。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          ConcurrentHashMap
         </code>
        </strong>
        ：
        <ul>
         <li>
          分段锁（Java 7）或 CAS + synchronized（Java 8+）实现高并发。
         </li>
         <li>
          推荐替代
          <code>
           Hashtable
          </code>
          用于多线程场景。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_Collections_75">
     </a>
     四、工具类
     <code>
      Collections
     </code>
    </h4>
    <p>
     提供对集合的常用操作：
    </p>
    <ul>
     <li>
      排序：
      <code>
       Collections.sort(list)
      </code>
     </li>
     <li>
      线程安全包装：
      <code>
       Collections.synchronizedList(list)
      </code>
     </li>
     <li>
      不可变集合：
      <code>
       Collections.unmodifiableList(list)
      </code>
     </li>
     <li>
      查找极值：
      <code>
       Collections.max(collection)
      </code>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_Iterator_84">
     </a>
     五、迭代器
     <code>
      Iterator
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：遍历集合元素。
     </li>
     <li>
      <strong>
       <code>
        fail-fast
       </code>
       机制
      </strong>
      ：
      <ul>
       <li>
        在遍历过程中检测到集合被修改（如
        <code>
         add
        </code>
        /
        <code>
         remove
        </code>
        ），立即抛出
        <code>
         ConcurrentModificationException
        </code>
        。
       </li>
       <li>
        适用于
        <code>
         ArrayList
        </code>
        、
        <code>
         HashMap
        </code>
        等非线程安全集合。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        fail-safe
       </code>
       机制
      </strong>
      ：
      <ul>
       <li>
        遍历时对原集合的修改不影响迭代器（如
        <code>
         ConcurrentHashMap
        </code>
        的迭代器）。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>

    <h4>
     <a id="_108">
     </a>
     七、如何选择集合类？
    </h4>
    <ul>
     <li>
      <strong>
       需要唯一性
      </strong>
      ：
      <code>
       Set
      </code>
      （
      <code>
       HashSet
      </code>
      、
      <code>
       TreeSet
      </code>
      ）。
     </li>
     <li>
      <strong>
       需要有序性
      </strong>
      ：
      <code>
       List
      </code>
      （
      <code>
       ArrayList
      </code>
      、
      <code>
       LinkedList
      </code>
      ）。
     </li>
     <li>
      <strong>
       键值对存储
      </strong>
      ：
      <code>
       Map
      </code>
      （
      <code>
       HashMap
      </code>
      、
      <code>
       ConcurrentHashMap
      </code>
      ）。
     </li>
     <li>
      <strong>
       多线程环境
      </strong>
      ：
      <code>
       ConcurrentHashMap
      </code>
      、
      <code>
       CopyOnWriteArrayList
      </code>
      。
     </li>
     <li>
      <strong>
       排序需求
      </strong>
      ：
      <code>
       TreeSet
      </code>
      、
      <code>
       TreeMap
      </code>
      。
     </li>
    </ul>
