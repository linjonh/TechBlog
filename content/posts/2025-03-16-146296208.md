    <h1 class="title-article" id="articleContentId">
     抽象工厂模式 (Abstract Factory Pattern)
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     抽象工厂模式 (Abstract Factory Pattern) 是一种创建型设计模式，它提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类。
    </p>
    <h4>
     一、基础
    </h4>
    <p>
     <strong>
      1. 意图
     </strong>
    </p>
    <ul>
     <li>
      提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类。
     </li>
    </ul>
    <p>
     <strong>
      2. 适用场景
     </strong>
    </p>
    <ul>
     <li>
      当一个系统要独立于它的产品的创建、组合和表示时。
     </li>
     <li>
      当一个系统要由多个产品系列中的一个来配置时。
     </li>
     <li>
      当你要强调一系列相关的产品对象的设计以便进行联合使用时。
     </li>
     <li>
      当你提供一个产品类库，而只想显示它们的接口而不是实现时。
     </li>
    </ul>
    <p>
     <strong>
      3. 结构
     </strong>
    </p>
    <ul>
     <li style="text-align:left">
      <strong>
       <span style="background-color:null">
        抽象产品族（AbstractProductFamily）
       </span>
      </strong>
      <span style="background-color:null">
       ：定义了一系列相关产品的抽象接口，这些产品之间存在某种关联或依赖关系。​
      </span>
     </li>
     <li style="text-align:left">
      <strong>
       <span style="background-color:null">
        具体产品（ConcreteProduct）
       </span>
      </strong>
      <span style="background-color:null">
       ：实现了抽象产品族接口的具体产品类。每个具体产品类对应抽象产品族中的一个产品。​
      </span>
     </li>
     <li style="text-align:left">
      <strong>
       <span style="background-color:null">
        抽象工厂（AbstractFactory）
       </span>
      </strong>
      <span style="background-color:null">
       ：声明了创建抽象产品族中各个产品的抽象方法。抽象工厂类负责定义创建一系列相关产品的接口，这些产品构成一个产品族。​
      </span>
     </li>
     <li style="text-align:left">
      <strong>
       <span style="background-color:null">
        具体工厂（ConcreteFactory）
       </span>
      </strong>
      <span style="background-color:null">
       ：实现了抽象工厂中声明的创建各个产品的方法，用于创建具体的产品对象。每个具体工厂类负责创建特定产品族中的所有产品。​
      </span>
     </li>
    </ul>
    <h4>
     二、进阶
    </h4>
    <p>
     <strong>
      1. 抽象工厂模式的优点
     </strong>
    </p>
    <ul>
     <li>
      抽象工厂模式隔离了具体类的生成，使得客户并不需要知道什么被创建。
     </li>
     <li>
      当一个产品族中的多个对象被设计成一起工作时，抽象工厂模式能够保证客户端始终只使用同一个产品族中的对象。
     </li>
     <li>
      增加新的具体工厂和产品族很方便，无需修改已有系统，符合“开闭原则”。
     </li>
    </ul>
    <p>
     <strong>
      2. 抽象工厂模式的缺点
     </strong>
    </p>
    <ul>
     <li>
      增加新的产品等级结构麻烦，需要对原有系统进行较大的修改，甚至需要修改抽象层代码，违背了“开闭原则”。
     </li>
    </ul>
    <p>
     <strong>
      3. 抽象工厂模式的应用
     </strong>
    </p>
    <ul>
     <li>
      抽象工厂模式广泛应用于各种框架和库中，例如 Spring 框架、Hibernate 框架等。
     </li>
     <li>
      抽象工厂模式也可以用于开发各种应用程序，例如游戏、图形用户界面等。
     </li>
    </ul>
    <h4>
     三、关键技术
    </h4>
    <p>
     <strong>
      1. 抽象工厂模式的实现
     </strong>
    </p>
    <ul>
     <li>
      抽象工厂模式可以通过接口或抽象类来实现。
     </li>
     <li>
      抽象工厂模式可以通过静态方法或实例方法来实现。
     </li>
    </ul>
    <p>
     <strong>
      2. 抽象工厂模式的扩展
     </strong>
    </p>
    <ul>
     <li>
      抽象工厂模式可以与其他设计模式结合使用，例如单例模式、原型模式等。
     </li>
     <li>
      抽象工厂模式可以扩展为工厂方法模式，从而支持创建单一产品对象。
     </li>
    </ul>
    <h4>
     四、易错点
    </h4>
    <p>
     <strong>
      1. 抽象工厂模式的滥用
     </strong>
    </p>
    <ul>
     <li>
      <p>
       抽象工厂模式不应该被滥用，只有在需要创建一系列相关或相互依赖的对象时才应该使用抽象工厂模式。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      2. 抽象工厂模式的性能问题
     </strong>
    </p>
    <ul>
     <li>
      <p>
       抽象工厂模式可能会降低代码的性能，因为每次创建对象时都需要调用工厂方法。
      </p>
     </li>
    </ul>
    <h4>
     五、核心代码
    </h4>
    <p>
     <strong>
      5.1 抽象工厂模式的简单实现
     </strong>
    </p>
    <pre><code class="language-cpp">// AbstractProductA
class AbstractProductA {
public:
    virtual void use() = 0;
};

// ConcreteProductA1
class ConcreteProductA1 : public AbstractProductA {
public:
    void use() override {
        std::cout &lt;&lt; "Using ConcreteProductA1" &lt;&lt; std::endl;
    }
};

// ConcreteProductA2
class ConcreteProductA2 : public AbstractProductA {
public:
    void use() override {
        std::cout &lt;&lt; "Using ConcreteProductA2" &lt;&lt; std::endl;
    }
};

// AbstractProductB
class AbstractProductB {
public:
    virtual void eat() = 0;
};

// ConcreteProductB1
class ConcreteProductB1 : public AbstractProductB {
public:
    void eat() override {
        std::cout &lt;&lt; "Eating ConcreteProductB1" &lt;&lt; std::endl;
    }
};

// ConcreteProductB2
class ConcreteProductB2 : public AbstractProductB {
public:
    void eat() override {
        std::cout &lt;&lt; "Eating ConcreteProductB2" &lt;&lt; std::endl;
    }
};

// AbstractFactory
class AbstractFactory {
public:
    virtual AbstractProductA* createProductA() = 0;
    virtual AbstractProductB* createProductB() = 0;
};

// ConcreteFactory1
class ConcreteFactory1 : public AbstractFactory {
public:
    AbstractProductA* createProductA() override {
        return new ConcreteProductA1();
    }

    AbstractProductB* createProductB() override {
        return new ConcreteProductB1();
    }
};

// ConcreteFactory2
class ConcreteFactory2 : public AbstractFactory {
public:
    AbstractProductA* createProductA() override {
        return new ConcreteProductA2();
    }

    AbstractProductB* createProductB() override {
        return new ConcreteProductB2();
    }
};

// Client
int main() {
    AbstractFactory* factory = new ConcreteFactory1();
    AbstractProductA* productA = factory-&gt;createProductA();
    AbstractProductB* productB = factory-&gt;createProductB();

    productA-&gt;use();
    productB-&gt;eat();

    delete factory;
    delete productA;
    delete productB;

    factory = new ConcreteFactory2();
    productA = factory-&gt;createProductA();
    productB = factory-&gt;createProductB();

    productA-&gt;use();
    productB-&gt;eat();

    delete factory;
    delete productA;
    delete productB;

    return 0;
}</code></pre>

    <h4>
     六、总结
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     <span style="color:#000000">
      <span style="background-color:null">
       抽象工厂模式通过将对象的创建和使用进一步分离，为创建一系列相关或相互依赖的对象提供了一种优雅的解决方案。它在提高系统的可扩展性和可维护性方面具有显著优势，但同时也增加了系统的复杂性。在实际开发中，需要根据具体的业务需求和系统架构来权衡是否使用抽象工厂模式。如果系统中存在多个产品族，并且产品之间存在复杂的依赖关系，那么抽象工厂模式是一个很好的选择。
      </span>
