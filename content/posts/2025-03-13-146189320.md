    <h2>

     前言
    </h2>
    <h3>
     <a id="_2">
     </a>
     关联式容器
    </h3>
    <p>
     在初阶阶段，我们已经接触过STL中的部分容器，比如：vector、list、deque、forward_list(C++11)等，这些容器统称为
     <strong>
      序列式容器
     </strong>
     ，因为其底层为
     <strong>
      线性序列的数据结构
     </strong>
     ，里面存储的是
     <strong>
      元素本身
     </strong>
     。
    </p>
    <p>
     那什么是关联式容器？它与序列式容器有什么区别？
    </p>
    <p>
     <strong>
      关联式容器
     </strong>
     也是用来存储数据的，与序列式容器不同的是，其里面存储的是&lt;key, value&gt;结构的
    </p>
    <p>
     键值对，在数据检索时比序列式容器效率更高。
    </p>
    <h3>
     <a id="_13">
     </a>
     键值对
    </h3>

     用来表示具有一一对应关系的一种结构，该结构中一般只包含两个成员变量key和value，key代
    </p>
    <p>
     表
     <strong>
      键值
     </strong>
     ，value表示与key对应的
     <strong>
      信息
     </strong>
     。比如：现在要建立一个英汉互译的字典，那该字典中必然
    </p>
    <p>
     有英文单词与其对应的中文含义，而且，英文单词与其中文含义是一一对应的关系，即通过该应
    </p>
    <p>
     该单词，在词典中就可以找到与其对应的中文含义。
    </p>

    <h3>
     <a id="_43">
     </a>
     树形结构的关联式容器
    </h3>
    <p>
     根据应用场景的不同，STL总共实现了两种不同结构的管理式容器：
     <strong>
      树型结构
     </strong>
     与
     <strong>
      哈希结构
     </strong>
     。树型结
    </p>
    <p>
     构的关联式容器主要有四种：map、set、multimap、multiset。这四种容器的共同点是：使
    </p>
    <p>
     用平衡搜索树(即红黑树)作为其底层结果，容器中的元素是一个有序的序列。下面一依次介绍每一
    </p>
    <p>
     个容器。
    </p>
    <h2>
     <a id="set_52">
     </a>
     set
    </h2>
    <h3>
     <a id="1_set__53">
     </a>
     1.
     <code>
      set
     </code>
     的定义
    </h3>
    <p>
     <code>
      set
     </code>
     是 C++ 标准库中的一个关联容器，它存储的是
     <strong>
      唯一的元素
     </strong>
     ，并且自动按一定的顺序排列元素。它通常基于
     <strong>
      平衡二叉搜索树
     </strong>
     （如红黑树）实现，提供
     <code>
      &lt;u&gt;O(log n)&lt;/u&gt;
     </code>
     <u>
     </u>
     的查找、插入和删除时间复杂度。
    </p>
    <blockquote>
     <p>
      特别记住两点：不重复，有序！
     </p>
    </blockquote>
    <p>
     与
     <code>
      vector
     </code>
     或
     <code>
      list
     </code>
     等容器不同，
     <code>
      set
     </code>
     会自动维护元素的顺序，
     <strong>
      通常是升序排序
     </strong>
     ，但可以通过自定义比较函数来改变排序规则。
    </p>
    <ul>
     <li>
      set的模板参数列表:
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/ebe17705d9d224f683e92aed708adad8.png"/>
    </p>
    <p>
     T: set中存放元素的类型，实际在底层存储&lt;value, value&gt;的键值对。
    </p>
    <p>
     Compare：set中元素默认按照小于来比较，也可以自己定义比较方式。
    </p>
    <p>
     Alloc：set中元素空间的管理方式，使用STL提供的空间配置器管理
    </p>
    <p>
     <strong>
      构造方式：
     </strong>
    </p>
    <pre><code class="prism language-cpp">
    <span class="token comment">// 1. 默认构造函数</span>

    <blockquote>
     <ol>
      <li>
       set是按照一定次序存储元素的容器
      </li>

       <br/>
       set中的
       <strong>
        元素不能在容器中修改
       </strong>
       (元素总是const)，但是可以从容器中插入或删除它们。
      </li>
      <li>
       在内部，set中的元素总是按照其内部比较对象(类型比较)所指示的特定严格弱排序准则进行
       <br/>
       排序。
      </li>
      <li>
       set容器通过key访问单个元素的速度通常比unordered_set容器慢，但它们允许根据顺序对
       <br/>
       子集进行直接迭代。
      </li>
      <li>
       set在底层是用二叉搜索树(红黑树)实现的。
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="_109">
     </a>
     注意以下八点：
    </h3>
    <p>
     <strong>
      1. 与
      <strong>
       <code>
        map
       </code>
      </strong>
      /
     </strong>
     <code>
      multimap
     </code>
     ** 的区别：
     <strong>
      <code>
       set
      </code>
     </strong>
     中只存 **
     <code>
      value
     </code>
    </p>
    <ul>
     <li>
      <strong>
       区别
      </strong>
      ：
      <ul>
       <li>
        <code>
         map
        </code>
        和
        <code>
         multimap
        </code>
        是键值对容器，每个元素是一个
        <code>
         &lt;key, value&gt;
        </code>
        对。
       </li>
       <li>
        <code>
         set
        </code>
        中存储的仅是
        <code>
         value
        </code>
        ，但底层实现实际上将
        <code>
         value
        </code>
        视为
        <code>
         &lt;value, value&gt;
        </code>
        的键值对，这样可以复用相同的底层数据结构（如红黑树）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       插入元素
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="prism language-cpp">std<span class="token double-colon punctuation">::</span>set<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> s<span class="token punctuation">;</span>
s<span class="token punctuation">.</span><span class="token function">insert</span><span class="token punctuation">(</span><span class="token number">5</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 只插入 value，而不是 &lt;key, value&gt;</span>
</code></pre>
    <pre><code>- `set` 中插入的只是单个值，而不是键值对。
</code></pre>
    <hr/>
    <p>
     <strong>
      2. 插入元素时，不需要构造键值对
     </strong>
    </p>
    <ul>
     <li>
      由于
      <code>
       set
      </code>
      只需要存储
      <code>
       value
      </code>
      ，插入时直接提供单个值即可。底层数据结构会将其视为键值对
      <code>
       &lt;value, value&gt;
      </code>
      ，但用户无需关心这些细节。
     </li>
    </ul>
    <p>
     示例：
    </p>
    <pre><code class="prism language-cpp">std<span class="token double-colon punctuation">::</span>set<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> s<span class="token punctuation">;</span>
s<span class="token punctuation">.</span><span class="token function">insert</span><span class="token punctuation">(</span><span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 只需要传递单个值</span>
s<span class="token punctuation">.</span><span class="token function">insert</span><span class="token punctuation">(</span><span class="token number">20</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 不需要键值对</span>
</code></pre>
    <p>
     这种简化让
     <code>
      set
     </code>
     使用起来更加方便，因为它的重点是值本身，而不是键值关联。
    </p>
    <hr/>
    <p>
     <strong>
      3.
      <strong>
       <code>
        set
       </code>
      </strong>
      中的元素不可以重复
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：
      <br/>
      <code>
       set
      </code>
      保证存储的所有元素是唯一的。
     </li>
     <li>
      <strong>
       实现方式
      </strong>
      ：
      <ul>
       <li>
        当插入一个元素时，
        <code>
         set
        </code>
        会根据比较规则（通常是
        <code>
         &lt;
        </code>
        运算符）检查是否已经存在。
       </li>
       <li>
        如果存在，则插入失败。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-cpp">std<span class="token double-colon punctuation">::</span>set<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> s<span class="token punctuation">;</span>
s<span class="token punctuation">.</span><span class="token function">insert</span><span class="token punctuation">(</span><span class="token number">5</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 插入成功</span>
s<span class="token punctuation">.</span><span class="token function">insert</span><span class="token punctuation">(</span><span class="token number">5</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 插入失败，值已经存在</span>
</code></pre>
    <p>
     <strong>
      应用场景
     </strong>
     ：
    </p>
    <ul>
     <li>
      去重。当需要去除数组或序列中的重复元素时，可以使用
      <code>
       &lt;font style="color:rgba(0, 0, 0, 0.85);"&gt;std::set&lt;/font&gt;
      </code>
      或
      <code>
       &lt;font style="color:rgba(0, 0, 0, 0.85);"&gt;std::unordered_set&lt;/font&gt;
      </code>
      。将一组重复元素转化为唯一集合：
     </li>
    </ul>
    <pre><code class="prism language-cpp">std<span class="token double-colon punctuation">::</span>vector<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> v <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span><span class="token number">1</span><span class="token punctuation">,</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token number">3</span><span class="token punctuation">,</span> <span class="token number">4</span><span class="token punctuation">,</span> <span class="token number">4</span><span class="token punctuation">}</span><span class="token punctuation">;</span>
std<span class="token double-colon punctuation">::</span>set<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> <span class="token function">s</span><span class="token punctuation">(</span>v<span class="token punctuation">.</span><span class="token function">begin</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">,</span> v<span class="token punctuation">.</span><span class="token function">end</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>  <span class="token comment">// 自动去重</span>
</code></pre>
    <hr/>
    <p>
     <strong>
      4. 使用
      <strong>
       <code>
        set
       </code>
      </strong>
      的迭代器遍历，可以得到有序序列
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       特点
      </strong>
      ：
      <br/>
      <code>
       set
      </code>
      自动维护元素的顺序（通常是升序）。
     </li>
     <li>
      <strong>
       遍历方式
      </strong>
      ：
      <br/>
      使用迭代器（
      <code>
       begin()
      </code>
      和
      <code>
       end()
      </code>
      ）可以顺序访问元素。
     </li>
    </ul>
    <p>
     <strong>
      示例
     </strong>
     ：

     提供了双向迭代器，允许遍历容器中的元素。
    </p>
    <ul>
     <li>
      <code>
       begin()
      </code>

     </li>
     <li>
      <code>

     </li>
     <li>
      <code>
       rbegin()

      ：返回指向容器最后一个元素的反向迭代器。
     </li>
     <li>
      <code>
       rend()

    </ul>

    <h3>
     <a id="4_set__395">
     </a>
     4.
     <code>
      set
     </code>
     的底层实现
    </h3>
    <ul>
     <li>
      <code>
       set
      </code>
      是基于平衡二叉搜索树（如红黑树或 AVL 树）实现的，这使得
      <code>
       set
      </code>
      能够提供
      <code>
       O(log n)
      </code>
      的查找、插入和删除操作。
     </li>
     <li>
      每个节点包含一个元素和指向左右子节点的指针。树是自平衡的，即插入或删除操作后，树会通过旋转操作保持平衡，保证操作的时间复杂度为
      <code>
       O(log n)
      </code>
      。
     </li>
    </ul>

    <h3>
     <a id="6_set__multiset__419">
     </a>
     6.
     <code>
      set
     </code>
     与
     <code>
      multiset
     </code>
     的区别
    </h3>
    <p>
     <strong>
      1. 元素唯一性
     </strong>
    </p>
    <ul>
     <li>
      <code>
       set
      </code>
      ：不允许重复的元素。
     </li>
     <li>
      <code>
       multiset
      </code>
      ：允许存储重复的元素。
     </li>
    </ul>
    <p>
     <strong>
      2. 插入规则
     </strong>
    </p>
    <ul>
     <li>
      <code>
       set
      </code>
      ：插入时会检查是否已有相同的元素，如果存在，则插入失败。
     </li>
     <li>
      <code>
       multiset
      </code>
      ：可以插入相同的元素，
      <code>
       multiset
      </code>
      会记录每个重复值。
     </li>
    </ul>
    <p>
     <strong>
      3. 元素计数
     </strong>
    </p>
    <ul>
     <li>
      <code>
       set
      </code>
      ：每个值在集合中最多出现一次，
      <code>
       count()
      </code>
      的返回值是
      <code>
       0
      </code>
      或
      <code>
       1
      </code>
      。
     </li>
     <li>
      <code>
       multiset
      </code>
      ：可以存储相同的元素，
      <code>
       count()
      </code>
      返回指定值的出现次数。
     </li>
    </ul>
    <p>
     <strong>
      4. 查找和删除
     </strong>
    </p>
    <ul>
     <li>
      <code>
       set
      </code>
      ：
      <ul>
       <li>
        <code>
         find()
        </code>
        ：返回指定元素的迭代器（如果存在）。
       </li>
       <li>
        <code>
         erase()
        </code>
        ：删除指定值的唯一实例。
       </li>
      </ul>
     </li>
     <li>
      <code>
       multiset
      </code>
      ：
      <ul>
       <li>
        <code>
         find()
        </code>
        ：返回指向第一个匹配值的迭代器。
       </li>
       <li>
        <code>
         erase()
        </code>
        ：删除某个值的所有实例，或者通过迭代器删除指定的单个实例。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      5. 其他特性
     </strong>
    </p>
    <ul>
     <li>
      两者底层都使用
      <strong>
       红黑树
      </strong>
      ，时间复杂度为
      <code>
       O(log n)
      </code>
      。
     </li>
     <li>
      都是有序容器，默认按升序排列元素。
     </li>
    </ul>

    <h2>
     <a id="multiset_616">
     </a>
     multiset
    </h2>
    <ol>
     <li>
      multiset是按照特定顺序存储元素的容器，其中元素是
      <strong>
       可以重复
      </strong>
      的。
     </li>
     <li>
      在multiset中，元素的value也会识别它(因为multiset中本身存储的就是&lt;value, value&gt;组成
      <br/>
      的键值对，因此value本身就是key，key就是value，类型为T). multiset元素的值**不能在容器
      <br/>
      **
      <strong>
       中进行修改
      </strong>
      (因为元素总是const的)，但可以从容器中插入或删除。
     </li>
     <li>
      在内部，multiset中的元素总是按照其内部比较规则(类型比较)所指示的特定严格弱排序准则
      <br/>
      进行
      <strong>
       排序
      </strong>
      。
     </li>
     <li>
      multiset容器通过key访问单个元素的速度通常比unordered_multiset容器慢，但当使用迭
      <br/>
      代器遍历时会得到一个有序序列。
     </li>
     <li>
      multiset底层结构为二叉搜索树(红黑树)。
     </li>
    </ol>
    <p>
     注意：
    </p>
    <ol>
     <li>
      multiset中再底层中存储的是&lt;value, value&gt;的键值对
     </li>
     <li>

     <li>
      与set的区别是，multiset中的元素可以重复，set是中value是唯一的
     </li>
     <li>
