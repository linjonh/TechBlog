     python垃圾回收机制
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     在Python中，垃圾回收主要通过
     <strong>
      引用计数
     </strong>
     和
     <strong>
      分代回收
     </strong>
     实现。
    </p>
    <blockquote>
     <ol>
      <li>
       ​
       <strong>
        引用计数
       </strong>
       会实时回收计数归零的对象（比如函数内局部变量用完立即释放）。
      </li>
      <li>
       ​
       <strong>
        分代回收
       </strong>
       则解决循环引用问题，比如两个对象互相引用但已无外部访问，这时分代回收会定期检测并清理。
      </li>
     </ol>
    </blockquote>
    <p>
     在接口自动化测试框架中，我重点关注两点：
    </p>
    <blockquote>
     <ul>
      <li>
       ​
       <strong>
        资源释放
       </strong>
       ：即使有GC，像数据库连接或文件句柄这类资源仍需手动释放（用
       <code>
        with
       </code>
       语句或
       <code>
        try/finally
       </code>
       ），避免依赖GC延迟导致资源耗尽。
      </li>
      <li>
       ​
       <strong>
        内存泄漏排查
       </strong>
       ：曾遇到测试用例因全局缓存未清理导致内存增长，用
       <code>
        gc
       </code>
       模块的
       <code>
        get_objects()
       </code>
       和
       <code>
        objgraph
       </code>
       工具定位到泄漏对象，改用弱引用（
       <code>
        weakref
       </code>
       ）或及时清理缓存解决。
      </li>
     </ul>
     <p>
      此外，在高并发场景下，我会通过
      <code>
       gc.disable()
      </code>
      临时关闭GC（谨慎使用）或调整分代回收阈值（
      <code>
       gc.set_threshold()
      </code>
      ），减少GC停顿对测试性能的影响。”
     </p>
     <hr/>
     <p>
      <strong>
       关键点
      </strong>
      ：
     </p>
     <ul>
      <li>
       讲清Python的GC机制（引用计数+分代回收）。
      </li>
      <li>
       结合测试框架，说明如何主动管理资源，而非依赖GC。
      </li>
      <li>
       举例实际遇到的泄漏问题及解决方法，突出实战能力。
      </li>
      <li>
       提及性能优化思路，体现对GC机制的灵活运用。
      </li>
     </ul>
    </blockquote>
