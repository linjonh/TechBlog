    <h2>
     <strong>
      问题背景
     </strong>
    </h2>
    <p>
     在使用ElementPlus的el-dialog组件时，发现点击取消按钮会触发两次关闭事件：
    </p>
    <p>
     1. 第一次参数为PointerEvent（事件对象）
    </p>
    <p>
     2. 第二次参数为undefined
    </p>
    <p>
     需要确保点击取消按钮时仅触发一次有效关闭事件，并传递正确的布尔值参数。
    </p>
    <p>
    </p>
    <p>
    </p>
    <h2>
     <strong>
      问题分析（ElementPlus特性相关）
     </strong>
    </h2>
    <p>
     <strong>
      组件结构特征
     </strong>
    </p>
    <pre><code class="language-html">&lt;el-dialog
  :modelValue="visible"
  @close="handleClose"&gt; &lt;!-- ElementPlus内置关闭事件 --&gt;
  &lt;template #footer&gt;
    &lt;el-button @click="close"&gt;取消&lt;/el-button&gt;
  &lt;/template&gt;
&lt;/el-dialog&gt;
</code></pre>
    <p>
     <strong>
      双重触发原因
     </strong>
    </p>
    <p>
     点击取消按钮：会触发close()关闭函数 → 然后触发对话框的handleClose()函数（内置的@close事件）
    </p>
    <p>
    </p>
    <h2>
     <strong>
      针对性解决方案
     </strong>
    </h2>
    <p>
     <strong>
      1. 显式传参阻断事件对象
     </strong>
    </p>
    <pre><code class="language-html">&lt;el-button 
  class="footer__button" 
  @click="closeBindingRole(false)"&gt; &lt;!-- 关键修改 --&gt;
  取消
&lt;/el-button&gt;
</code></pre>
    <p>
     <strong>
      2. 统一对话框关闭处理
     </strong>
    </p>
    <pre><code class="language-html">&lt;el-dialog
  :modelValue="bindingRoleVisible"
  @close="handleDialogClose"&gt; &lt;!-- 专用关闭处理 --&gt;
  &lt;!-- 对话框内容 --&gt;
&lt;/el-dialog&gt;

&lt;script&gt;
// 统一关闭入口
const handleDialogClose = () =&gt; {
  closeBindingRole(false);
};
&lt;/script&gt;</code></pre>
    <p>
     <strong>
      3. 增强型状态锁（ElementPlus适配版）
     </strong>
    </p>
    <pre><code class="language-javascript">let dialogClosing = false;

const closeBindingRole = (isSuccess: boolean) =&gt; {
  if (dialogClosing) return;
  
  dialogClosing = true;
  emit("closeBindingRoleDialog", isSuccess);
  
  // 兼容ElementPlus动画时长
  setTimeout(() =&gt; {
    dialogClosing = false;
  }, 300); // 略大于对话框关闭动画时间
};
</code></pre>
