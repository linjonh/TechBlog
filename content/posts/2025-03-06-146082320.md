    <p>
     在Unity渲染中，**顶点着色器（Vertex Shader）
     <strong>
      和
     </strong>
     片段着色器（Fragment Shader）**是图形渲染管线中的两个核心阶段。我们可以通过一个比喻来理解它们的分工：想象你要画一幅由三角形组成的3D模型，顶点着色器负责确定每个三角形的“顶点位置”，而片段着色器负责给每个像素“填色”。
    </p>
    <h4>
     <strong>
      1. 顶点着色器（Vertex Shader）——定位形状
     </strong>
    </h4>
    <p>
     <strong>
      作用
     </strong>
     ：顶点着色器是渲染管线的第一步，它处理模型的每个顶点（比如立方体的8个角点），主要任务是将这些顶点的位置从模型自身的坐标系转换到屏幕坐标系中，同时可以修改顶点的其他属性（如颜色、法线方向、纹理坐标等）。
    </p>
    <ul>
     <li>
      <strong>
       通俗理解
      </strong>
      ：假设你有一个纸箱模型，顶点着色器的作用就是告诉电脑“这个纸箱应该放在屏幕的哪个位置，如何旋转或缩放”，类似确定纸箱在画面中的摆放姿势。
     </li>
     <li>
      <strong>
       核心功能
      </strong>
      ：
      <ul>
       <li>
        <strong>
         坐标变换
        </strong>
        ：将顶点从模型空间→世界空间→屏幕空间
       </li>
       <li>
        <strong>
         传递属性
        </strong>
        ：把顶点的颜色、纹理坐标等信息传递给后续阶段。
       </li>
       <li>
        <strong>
         动画效果
        </strong>
        ：通过修改顶点位置实现波浪形变、骨骼动画等
        <br/>
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <strong>
      2. 片段着色器（Fragment Shader）——填充颜色
     </strong>
    </h4>
    <p>
     <strong>
      作用
     </strong>
     ：在顶点着色器之后，光栅化会将三角形转换为屏幕上的像素（称为“片段”），片段着色器则负责计算每个像素的颜色，比如纹理贴图、光照、阴影等效果。
    </p>
    <ul>
     <li>
      <strong>
       通俗理解
      </strong>
      ：顶点着色器摆好了纸箱的位置，片段着色器负责给纸箱表面贴上图案，或者根据光照方向计算哪部分更亮、哪部分更暗。
     </li>
     <li>
      <strong>
       核心功能
      </strong>
      ：
      <ul>
       <li>
        <strong>
         纹理采样
        </strong>
        ：从贴图中读取颜色（比如纸箱的包装图案）
        <br/>
       </li>
       <li>
        <strong>
         光照计算
        </strong>
        ：根据光源方向、材质属性等计算像素的最终颜色
        <br/>
       </li>
       <li>
        <strong>
         特效处理
        </strong>
        ：比如透明度混合、火焰效果、水面折射等
        <br/>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      代码示例
     </strong>
     （简化版）：
    </p>
    <pre></pre>
    <pre><code class="language-cpp">fixed4 frag (v2f i) : SV_Target {
    fixed4 col = tex2D(_MainTex, i.uv); // 从纹理中采样颜色
    return col; // 输出像素颜色
}</code></pre>
    <p>
    </p>
    <p>
     <em>
      这段代码根据顶点传递的纹理坐标，从贴图中获取颜色并输出到屏幕
     </em>
    </p>
    <h4>
     <strong>
      3. 两者的协作流程
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       顶点着色器
      </strong>
      处理所有顶点，确定模型在屏幕上的形状。
     </li>
     <li>
      <strong>
       光栅化
      </strong>
      将三角形拆解为像素（片段）。
     </li>
     <li>
      <strong>
       片段着色器
      </strong>
      对每个像素进行颜色计算，最终输出图像。
     </li>
    </ol>
    <p>
     <strong>
      举个实际例子
     </strong>
     ：
    </p>
    <ul>
     <li>
      如果想让一个旗帜飘动，顶点着色器修改顶点的位置来模拟波动，片段着色器根据纹理和光照计算旗帜的颜色
      <br/>
     </li>
    </ul>
    <h4>
     <strong>
      4. 关键区别
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         维度
        </strong>
       </th>
       <th>
        <strong>
         顶点着色器
        </strong>
       </th>
       <th>
        <strong>
         片段着色器
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         处理对象
        </strong>
       </td>
       <td>
        每个顶点（数量少，如立方体8个顶点）
       </td>
       <td>
        每个像素（数量多，如屏幕百万像素）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         性能消耗
        </strong>
       </td>
       <td>
        较低
       </td>
       <td>
        较高（逐像素计算）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         主要任务
        </strong>
       </td>
       <td>
        坐标变换、传递属性
       </td>
       <td>
        颜色计算、特效处理
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         可修改内容
        </strong>
       </td>
       <td>
        顶点位置、法线、UV坐标等
       </td>
       <td>
        像素颜色、透明度等
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     <strong>
      5. 实际应用场景
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       顶点着色器
      </strong>
      更适合做形状变化（如动态扭曲、骨骼动画）。
     </li>
     <li>
      <strong>
       片段着色器
      </strong>
      更适合做视觉细节（如水面反光、阴影、边缘发光）。
     </li>
    </ul>
    <p>
     通过两者的配合，Unity可以实现从简单的物体渲染到复杂的动态特效
    </p>
