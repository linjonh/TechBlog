    <h2>
     <a id="Binary_Search_Tree_0">
     </a>
     二叉搜索树（Binary Search Tree）
    </h2>
    <p>
     二叉搜索树是一种特殊的二叉树，它用来快速搜索某个值，对于每个节点都应该满足以下条件：
    </p>
    <ol>
     <li>
      若该节点有左子树，那么左子树中所有节点的值都应该小于该节点的值。
     </li>
     <li>
      若该节点有右子树，那么右子树中所有节点的值都应该大于该节点的值。
     </li>
     <li>
      左右子树都应该是二叉搜索树。
      <br/>
      根据条件三可以确定，二叉搜索树是
      <strong>
       通过递归定义
      </strong>
      的。
      <br/>
      并且我们可以得到以下结论：
      <strong>
       左子树节点值 &lt; 根节点值 &lt; 右子树节点值
      </strong>
      。
      <br/>
      所以
      <strong>
       二叉搜索树的中序遍历是一个升序的序列。
      </strong>
      <br/>
      但是我们构建二叉搜索树的目的不是为了排序，是为了
      <strong>
       高效地查询、插入、删除元素
      </strong>
      。
     </li>
    </ol>
    <p>
     二叉搜索树示例图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/95af62b913394a0d91826fd05c6559b7.png"/>
    </p>
    <h3>
     <a id="_12">
     </a>
     查找操作
    </h3>
    <p>
     查找操作就是从根节点开始，沿某个路径一直向下搜索的过程。要查找的值比当前节点值小就往当前节点的左子树查找；要查找的值比当前节点值大就往当前节点的右子树查找。直到找到要查找的值为止。
     <br/>
     当节点分布均匀时，时间复杂度一般为树的高度O(logn)；
     <br/>
     但是最坏情况下节点会退化成链表，所以时间复杂度为O(n)。
    </p>
    <h3>
     <a id="_16">
     </a>
     插入操作
    </h3>
    <p>
     将一个元素插入到树中，是从根节点开始，与当前节点相比较，如果比当前节点的值小，就进入该节点的左子树；如果比当前节点的值大，就进入该节点的右子树。直到遇到一个空节点，就可以将要插入的节点安放在这个空节点的位置。所以插入的节点一定是一个新添加的
     <strong>
      叶子节点
     </strong>
     。并且
     <strong>
      在二叉搜索树中，通常不允许存储重复的值
     </strong>
     。
     <br/>
     同查询操作一样，时间复杂度为O(n)。
    </p>
    <h3>
     <a id="_20">
     </a>
     删除操作
    </h3>
    <p>
     删除操作分为三种情况：
    </p>
    <ol>
     <li>
      当要删除的节点为叶子节点的时候，直接删除。
     </li>
     <li>
      当要删除的节点有左子树
      <code>
       或者
      </code>
      右子树的时候，删除该节点，用该节点的子节点代替该节点。
     </li>
     <li>
      当要删除的节点
      <code>
       既有
      </code>
      左子树
      <code>
       又有
      </code>
      右子树的时候，有
      <code>
       两种方法
      </code>
      ，但是目的都是将情况变成第一种或者第二种。
      <br/>
      3.1 找到该节点的
      <code>
       直接前驱
      </code>
      (该节点左子树中的最大值)来代替该节点，然后针对这个
      <code>
       直接前驱
      </code>
      完成情况二节点的删除。
      <br/>
      3.2 找到该节点的
      <code>
       直接后继
      </code>
      (该节点右子树中的最小值)来代替该节点，然后针对这个
      <code>
       直接后继
      </code>
      完成情况二节点的删除。
     </li>
    </ol>
    <p>
     解释：
     <br/>
     节点 x 的直接前驱：在中序遍历中，x 的前驱。在 x 的左子树的最右下节点（该节点一定没有右子树）。
     <font color="red">
      对于“该节点一定没有右子树”的解释：因为该节点如果有右子树，那么说明有比该节点还大的节点，那么该节点就不是 x 的直接前驱了。
     </font>
     <br/>
     节点 x 的直接后继：在中序遍历中，x 的后继。在 x 的右子树的最左下节点（该节点一定没有左子树）
    </p>

       直接前驱（17）

      </code>
      ，然后将这个直接前驱或者直接后继复制一份，
      <code>
       替换掉
      </code>

    </p>

      <code>

     Q：可能此时就有人有疑问，难道就不会出现要删除的直接前驱或者直接后继既有左子树，又有右子树？
     <br/>
     A：这时不可能的，直接前驱要是还有右子树，那他就不是直接前驱。根据BST定义可知一个节点的右子树的值肯定要比该节点大。所以直接前驱时没有右子树的。
    </p>
    <p>
     我们继续以

      <code>

    <h2>
     <a id="AVL_53">
     </a>
     平衡二叉树（AVL树）
    </h2>
    <p>
     我们知道，在二叉搜索树中，对于一个有序度很高的序列甚至是一个有序数列，我们在利用这个序列建二叉搜索树的时候，树会退化成一个链表，这样一来各种操作的时间复杂度将会变得很高。那么如何解决这种情况呢？我们引入了AVL树的概念，AVL树是一种
     <code>
      自平衡
     </code>
     的二叉搜索树，那么如何实现这个自平衡呢，就让我们接着学习。
    </p>
    <h3>
     <a id="_55">
     </a>
     基本概念
    </h3>
    <p>
     为保证二叉搜索树的性能，在插入、删除节点时规定，任意节点的
     <code>
      左右子树高度差不超过 1
     </code>
     ，这样的二叉搜索树被称为AVL树。
    </p>
    <p>
     左右子树的高度差被称为
     <code>
      平衡因子（BF）
     </code>
     ，
     <code>
      BF = 左子树高度 - 右子树高度
     </code>
     ，取值范围为
     <code>
      [-1,0,1]
     </code>
     。
    </p>
    <h3>
     <a id="_60">
     </a>
     查找操作
    </h3>
    <p>
     查找原理同二叉搜索树一样，从根节点开始向下查找。时间复杂度为O(logn)（比BST更稳定）
    </p>
    <h3>
     <a id="_63">
     </a>
     插入操作
    </h3>
    <p>
     我们先按照二叉搜索树的算法插入一个节点，那么在途径的所有节点的平衡因子都可能会受到影响，它们的BF的绝对值可能变得大于1了，此时就应该想办法
     <code>
      调整这棵树的结构
     </code>
     ，让它重新变成平衡二叉树。
     <br/>
     那么如何调整呢，从哪里开始调整呢？我们要先认识一个基本概念两个基本操作。
    </p>
    <h4>
     <a id="_66">
     </a>
     一个基本概念两个基本操作
    </h4>
    <p>
     基本概念：
     <code>
      最小不平衡子树
     </code>
     。
     <br/>
     在插入一个新节点之后，可能会造成很多节点的BF绝对值都大于1，此时我们应该找到
     <code>
      距离新插入的节点最近的
     </code>
     不平衡的点（BF绝对值大于1的点），以这个点为根的子树就是最小不平衡子树。
     <br/>
     我们要调整的地方就是在最小不平衡子树处，
     <em>
      <strong>
       仅需让最小不平衡子树平衡，整棵树就平衡了。
      </strong>
     </em>
     <font color="red">
      为了简写，下文中称最小不平衡子树的根节点为T。
     </font>
    </p>
    <p>
     基本操作：
     <code>
      左旋
     </code>
     、
     <code>
      右旋
     </code>
     。
     <br/>
     这两个操作都是对于T进行的，
     <em>
      <strong>
       T对于T的右子树左旋、T对于T的左子树右旋。
      </strong>
     </em>
    </p>
    <p>
     首先来看一种简单的左旋：
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/9d9eac30036c4f6fb63957ea51a5e449.png">
      <br/>
      <code>
       7
      </code>
      是最小不平衡子树的根节点
      <code>
       T
      </code>
      ，现在要对
      <code>
       T
      </code>
      绕
      <code>
       12
      </code>
      进行
      <code>
       左旋
      </code>
      ，我们可以很直观的看到左旋之后
      <code>
       降低
      </code>
      了
      <code>
       T
      </code>
      的
      <code>
       右子树高度
      </code>
      ，使
      <code>
       T
      </code>
      的
      <code>
       BF
      </code>
      变成了
      <code>
       0
      </code>
      ，那为什么左旋之后仍然符合二叉搜索树的要求（左子树&lt;根节点&lt;右子树）呢？注意理解这个地方！节点7是从12的左父亲旋转到左孩子的，不管它是父亲还是孩子，它都在12的左边，这里强调的是一种相对关系，在下面我们
      <strong>
       会举例更复杂的左旋
      </strong>
      ，在旋转时，处理两个不相邻的节点时，需要关注它们的相对关系。旋转操作不仅涉及父子节点的转换，还需要考虑整个子树的结构平衡。
     </img>
    </p>
    <p>
     铺垫完了现在我们来看一种复杂的左旋：
    </p>
    <p>
     这是一颗平衡二叉树，没有出现失衡。
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/7172c5839cf74456beaa4352e822b084.png">
      <br/>
      当我们插入一个节点18，它会按照插入算法被安放在节点15的右孩子的位置。
      <br/>
      这时，这个二叉树出现失衡。节点5是T节点。由于插入的新节点的位置是在T的右孩子的右子树中，属于RR型插入，需要左旋T节点来保持平衡。
      <br/>
      这时候我们将T绕10左旋转的时候会发现，10的左边有个孩子挡住了我们的旋转，此时怎么办呢？
      <br/>
      我们在上文中说过，要以基本原则来思考，一个节点的右子树的所有节点都是比它大的。那么可以将6放在5的右孩子上，也不破坏原本的二叉搜索树结构。
      <strong>
       让T的右孩子的左子树变成T的右子树。
      </strong>
      <br/>
      <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/8da99dbd588b41f3a4c1dca38d068577.png"/>
      <br/>
      于是我们得到了这个重新平衡的AVL树：
      <br/>
      <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/9396dcff0d344bc2aaa0eec6a2858761.png"/>
      <br/>
      讲完基本操作，下面我们来看四种插入类型以及对应的自平衡调整方式。
     </img>
    </p>
    <h4>
     <a id="AVL_LL_RR_LR_RL_89">
     </a>
     <strong>
      AVL 树的插入与四种旋转（LL, RR, LR, RL）
     </strong>
    </h4>
    <p>
     在
     <strong>
      AVL 树
     </strong>
     中，插入新节点后，可能会导致某些节点的
     <strong>
      平衡因子（BF）
     </strong>
     超出 [-1, 0, 1] 的范围，这时需要针对
     <strong>
      最小不平衡子树的根节点T
     </strong>
     进行
     <strong>
      旋转
     </strong>
     来恢复平衡。 新节点不同的插入方式对应的旋转方式也是不一样的。
    </p>
    <ol>
     <li>
      LL型
      <br/>
      插入方式：新节点插入在T的左孩子的左子树上，所以被称为LL型插入。
      <br/>
      调整方式：右旋T节点。
     </li>
     <li>
      RR型
      <br/>
      插入方式：新节点插入在T的右孩子的右子树上。
      <br/>
      调整方式：左旋T节点。
     </li>
     <li>
      LR型
      <br/>
      插入方式：新节点插入在T的左孩子的右子树上。
      <br/>
      调整方式：左旋孩子节点，再右旋T节点。
     </li>
     <li>
      RL型
      <br/>
      插入方式：新节点插入在T的右孩子的左子树上。
      <br/>
      调整方式：右旋孩子节点，再左旋T节点。
     </li>
    </ol>
    <p>
     练习：构建以下序列的AVL树：10 6 18 12 13 8 20 17 28
    </p>
    <h3>
     <a id="_107">
     </a>
     删除操作
    </h3>
    <p>
     同插入操作一样，删除操作也是先通过二叉搜索树的删除算法删除一个节点e，然后就要从这个节点e开始向上找到第一个不平衡的节点T，于是就要对这个节点进行调整。
     <br/>
     如何调整呢？我们首先要找到以这个T节点为根的子树中
     <code>
      最高的儿孙节点
     </code>
     ，
     <strong>
      根据孙子节点相对于儿子节点的位置，判断属于哪种类型（LL、RR、LR、RL）
     </strong>
     ，然后对这个T进行调整。调整完了T之后，要继续向上寻找不平衡的点继续调整。
    </p>
    <p>
     为什么插入操作只用调整一次，而删除操作调整完一次之后还要继续查找不平衡的点？因为插入操作是要在固定的位置插入的，这个位置取决于值的大小，而删除操作是在任意位置删除一个节点，那么任意删除的这个节点很可能就影响到了其他很多个节点的平衡，所以要向上挨着调整。
    </p>
    <p>
     如何寻找
     <code>
      最高的儿孙节点
     </code>
     ？如下图所示，假设T为最小不平衡子树根节点，我们需要在T的左右子树中寻找较高的那个子树，较高子树的根节点就是
     <code>
      最高儿子节点
     </code>
     。然后我们继续在最高儿子节点的左右子树中寻找较高的那个子树，此时这个子树的根节点就是
     <code>
      最高孙子节点
     </code>
     。
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/acb124ae270b43db8cace6936ec4a573.png"/>
     <br/>
     注：黄色节点为最高儿子节点，绿色节点为最高孙子节点。
    </p>
    <h2>
     <a id="RBT_117">
     </a>
     红黑树（RBT）
    </h2>
    <h3>
     <a id="_118">
     </a>
     基本概念
    </h3>
    <p>
     红黑树是在二叉搜索树的基础上，在每个节点中增加了一个储存节点颜色的信息位，同过对任意一条从根到叶子节点路径上的各个节点的着色限制，确保没有一条路径比其他的路径长出两倍。这样一来就保证了二叉搜索树的相对平衡，不会出现极端情况。这就是红黑树。
    </p>
    <h3>
     <a id="RBT_121">
     </a>
     RBT的规则
    </h3>
    <p>
     注意：在RBT中我们所说的叶子节点（NIL节点）与正常的二叉树中的叶子节点（Leaf Node）是有区别的，RBT的叶子节点是指空节点
     <br/>
     1.每个节点要么是红色要么是黑色
     <br/>
     2.根节点或者叶子节点必须是黑色
     <br/>
     3.红色节点的两个孩子必须都是黑色节点，就是说不会有两个相连的红色节点
     <br/>
     4.对于任意一个节点，这个节点到其所有叶子节点的路径上，每条路径上的黑色节点的数量必须是相同的
    </p>
    <h3>
     <a id="RBT_128">
     </a>
     RBT的性质
    </h3>
    <h4>
     <a id="1_129">
     </a>
     1.最长路径不会超过最短路径的两倍。
    </h4>
    <p>
     最短路径：全由黑色节点组成，设其长度为 b。
     <br/>
     最长路径：红黑节点交替出现，由于红色节点不能连续，最长路径的黑色节点数也为 b，红色节点数最多为 b-1，因此最长路径长度为 2b-1。
     <br/>
     所以最长路径不会超过最短路径的两倍。
    </p>
    <h4>
     <a id="2n_h__2log2n__1_136">
     </a>
     2.有n个节点的红黑树，树的高度 h &lt;= 2log2(n + 1)
    </h4>
    <p>
     由这个性质可以得到，在RBT中的查询、插入、删除操作时间复杂度都是O(logn)。
    </p>
    <h5>
     <a id="2_1_138">
     </a>
     2. 1黑高度的定义
    </h5>
    <p>
     黑高度（black-height）是指从某个节点到叶子节点的路径上的黑色节点数量（不包括该节点本身）。根据性质5，所有路径的黑高度相同。
    </p>
    <h5>
     <a id="22__141">
     </a>
     2.2 红黑树的高度与黑高度的关系
    </h5>
    <p>
     设红黑树的黑高度为
     <code>
      bh
     </code>
     ，则树的高度
     <code>
      h
     </code>
     满足：h &lt;= 2 * bh
     <br/>
     这是因为红节点不能连续出现，路径上的红色节点数量最多与黑色节点数量相等。
    </p>
    <h5>
     <a id="23__144">
     </a>
     2.3 节点数量与黑高度的关系
    </h5>
    <p>
     对于黑高度为
     <code>
      bh
     </code>
     的红黑树，节点数量
     <code>
      n
     </code>
     满足：n &gt;= 2^bh - 1
    </p>
    <p>
     这是因为黑高度为
     <code>
      bh
     </code>
     的树至少是一个完全二叉树，节点数量为
     <code>
      2^bh - 1
     </code>
     。
    </p>
    <h5>
     <a id="24__148">
     </a>
     2.4 推导高度与节点数量的关系
    </h5>
    <p>
     由
     <code>
      n &gt;= 2^bh - 1
     </code>
     ，可得：bh &lt;= log2(n + 1)
     <br/>
     结合
     <code>
      h &lt;= 2 * bh
     </code>
     ，得到：h &lt;= 2 * log2(n + 1)
    </p>
    <h3>
     <a id="_151">
     </a>
     查找操作
    </h3>
    <p>
     原理同二叉搜索树的查找操作
     <br/>
     由于红黑树的高度
     <code>
      h &lt;= 2 * log2(n + 1)
     </code>
     ，查找操作的时间复杂度为
     <code>
      O(log n)
     </code>
     。
    </p>
    <h3>
     <a id="_155">
     </a>
     插入操作
    </h3>
    <h4>
     <a id="_156">
     </a>
     新节点颜色
    </h4>
    <p>
     插入操作的原理也是按照二叉搜索树的算法插入元素，但是这个插入的新节点应该是什么颜色呢？
     <br/>
     很明显
     <strong>
      新节点设置为红色
     </strong>
     会更好。
     <br/>
     如果新节点设置为黑色，那么在插入一个新节点的时候，他一定会使从根节点到叶子结点这条路径上的黑色节点数目变多，那么每次插入都需要重新调整这条路径上的黑色节点数目，这样很麻烦。
     <br/>
     如果新节点颜色设置为红色，那么只可能违反两个红色节点不相连或者根节点不能为红色这两个原则，这两个调整起来很简单，并且有可能在插入新节点的时候并不破坏树的结构。
    </p>
    <h4>
     <a id="_162">
     </a>
     插入后调整方式
    </h4>
    <h5>
     <a id="_163">
     </a>
     插入的节点是根节点
    </h5>
    <p>
     只需要将新节点的颜色由红色设置为黑色即可。
    </p>
    <h5>
     <a id="_165">
     </a>
     新节点的叔叔节点是红色
    </h5>
    <p>
     这种情况不需要旋转操作，此时要将父亲、叔叔、爷爷节点全部改变颜色。并且将改变后的爷爷节点当作新插入的节点继续向上判断直至RBT树平衡。
    </p>
    <h5>
     <a id="_167">
     </a>
     叔叔节点是黑色
    </h5>
    <p>
     这时候需要看新节点和爷爷节点的位置关系，通过旋转 + 变色来调整，分四种情况：
    </p>
    <ol>
     <li>
      LL型
      <br/>
      新节点是爷爷节点的左孩子的左孩子
      <br/>
      调整方式：右旋父亲节点 + 父爷变色
     </li>
     <li>
      RR型
      <br/>
      新节点是爷爷节点的右孩子的右孩子
      <br/>
      调整方式：左旋父亲节点 + 父爷变色
     </li>
     <li>
      LR型
      <br/>
      新节点是爷爷节点的左孩子的右孩子
      <br/>
      调整方式：先左旋孩子节点，再右旋父亲节点 + 儿爷变色
     </li>
     <li>
      RL型
      <br/>
      新节点是爷爷节点的右孩子的左孩子
      <br/>
      调整方式：先右旋孩子节点，再左旋父亲节点 + 儿爷变色
     </li>
    </ol>
