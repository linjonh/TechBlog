      <strong>

     <li>

     <li>

      <strong>
       终端远程登录Web服务器
      </strong>

    <h4>
     <a id="Chens_62">
     </a>
     Chen’s表示法
    </h4>
    <p>
     Version1.0：
    </p>
    <p>
     <strong>
      实体
     </strong>
     有：
     <em>
      <strong>
       用户、文章、点赞、评论、标签和分区
      </strong>
     </em>
    </p>
    <p>
     其中，
    </p>
    <ul>
     <li>
      用户属性有
      <em>
       用户ID、用户名、注册时间
      </em>
      ；
     </li>
     <li>
      文章属性有
      <em>
       文章ID、标题、阅读量、上传时间
      </em>
      ；
     </li>
     <li>
      点赞属性有
      <em>
       点赞ID、点赞时间、用户ID、文章ID
      </em>
      ；
     </li>
     <li>
      评论属性有
      <em>
       评论ID、评论内容、评论时间
      </em>
      ；
     </li>
     <li>
      标签属性有
      <em>
       标签ID、标签名称
      </em>
      ；
     </li>
     <li>
      分区属性有
      <em>
       分区ID、分区描述、分区名称
      </em>
      ；
     </li>
    </ul>
    <p>
     <strong>
      联系
     </strong>
     有：
    </p>
    <ul>
     <li>
      每个用户可以发布多个文章，点1个赞，评论多次，每个文章属于一个用户，每个点赞属于一个用户，每个评论属于一个用户；
     </li>
     <li>
      一个文章有多个点赞，多个评论，多个标签，属于一个分区；
     </li>
     <li>
      一个点赞属于一个文章，一个评论属于一个文章，一个标签对应一个文章；
     </li>
     <li>
      一个分区有多个文章
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5daeab8bed7d47c0863233d3795cf343.png"/>
    </p>
    <h4>
     <a id="Version20_86">
     </a>
     Version2.0
    </h4>
    <p>
     优化如下：
    </p>
    <ul>
     <li>
      点赞数可以简化为文章的一个属性，文章和用户的属性应该更加完善，点赞量、收藏量、粉丝量属于派生属性
     </li>
     <li>
      对用户进行ISA建模完全特化，用户要么是内容创作者，要么是内容消费者（也可以都是），内容创作者发布文章，内容消费者点赞/收藏/投币文章，投币数表示某个内容消费者给某个文章投币的数量
     </li>
     <li>
      用户实体之间还有关注联系，用关注ID表示某个用户被某个粉丝关注
     </li>
     <li>
      评论实体之间还有回复联系，回复ID表示某条评论被某条评论回复
     </li>
     <li>
      对每个联系进行基数约束：
     </li>
     <li>
      每个用户可以被0到n个用户关注，也可以关注0到n个用户
     </li>
     <li>
      用户可以发表0到n条评论，一个文章下有0到n条评论，一个评论唯一对应一个用户，唯一对应一个文章
     </li>
     <li>
      每条评论可以有0到n条回复，每个回复对应1条评论
     </li>
     <li>
      内容创作者可以发布0到n个文章，内容消费者可以打赏0到n个文章频
     </li>
     <li>
      每个文章由1个内容创作者发布，受到0到n个内容消费者点赞/收藏/投币，每个内容创作者可以发布0到n条文章，每个内容消费者可以点赞/收藏0到n个文章，可以投0到n个币
     </li>
     <li>
      一个文章有0到n个标签，每个标签对应1到n个文章
     </li>
     <li>
      每个文章属于1到n个分区，每个分区有0到n个文章
     </li>
     <li>
      标签实体属于弱实体，依赖于文章实体，其联系为弱联系
     </li>
    </ul>

    <h3>
     <a id="_114">
     </a>
     完整性约束例子
    </h3>
    <ol>
     <li>
      实体完整体：
     </li>
    </ol>
    <p>
     举例：用户作为实体，他的主码用户ID不能为空值，否则我们将无法识别这一用户。
    </p>
    <ol start="2">
     <li>
      参照完整性：
     </li>
    </ol>
    <p>
     举例：用户和文章都是实体，其中文章的属性“发布者ID”是用户的主码，用户ID是发布关系的外码。因此发布者ID只能取用户ID里面的数值，即我们不能注册一个不是用户的ID成为发布者。
    </p>
    <ol start="3">
     <li>
      用户定义的完整性：
     </li>
    </ol>
    <p>
     举例：用户的粉丝量不能为负数。
    </p>

  <span class="token punctuation">[</span><span class="token keyword">CHARACTER</span> <span class="token keyword">SET</span> charset_name<span class="token punctuation">]</span>
  <span class="token punctuation">[</span><span class="token keyword">COLLATE</span> collation_name<span class="token punctuation">]</span><span class="token punctuation">;</span>

     创建数据库，该命令的作用：

    </h3>
    <p>

    <pre><code class="prism language-sql"><span class="token keyword">DROP</span> <span class="token keyword">DATABASE</span> <span class="token operator">&lt;</span>database_name<span class="token operator">&gt;</span><span class="token punctuation">;</span>        <span class="token comment">-- 直接删除数据库，不检查是否存在</span>

    <p>
     或
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">DROP</span> <span class="token keyword">DATABASE</span> <span class="token punctuation">[</span><span class="token keyword">IF</span> <span class="token keyword">EXISTS</span><span class="token punctuation">]</span> <span class="token operator">&lt;</span>database_name<span class="token operator">&gt;</span><span class="token punctuation">;</span>

     选择数据库
    </h3>
    <p>
     在连接到 MySQL 数据库后，可能有多个可以操作的数据库，所以需要选择要操作的数据库，语法如下：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">USE</span> database_name<span class="token punctuation">;</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/14df18f675834d2fb0137e6c1a7c1ccf.png"/>
    </p>
    <p>
     5.8创建数据表
     <br/>
     创建 MySQL 数据表需要以下信息：
    </p>
    <ul>
     <li>
      表名
     </li>
     <li>
      表字段名
     </li>
     <li>
      定义每个表字段的数据类型
     </li>
    </ul>
    <p>
     语法如下：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">CREATE</span> <span class="token keyword">TABLE</span> table_name <span class="token punctuation">(</span>
    column1 datatype<span class="token punctuation">,</span>
    column2 datatype<span class="token punctuation">,</span>
    <span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span>
<span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1b2b48f7cc1a4a4bb6c543c5dd788b71.png"/>
    </p>
    <p>
     实例解析：
    </p>
    <ul>
     <li>
      id: 用户 id，整数类型，自增长，作为主键
     </li>
     <li>
      username: 用户名，变长字符串，不允许为空
     </li>
     <li>
      email: 用户邮箱，变长字符串，不允许为空
     </li>
     <li>
      birthdate: 用户的生日，日期类型
     </li>
     <li>
      is_active: 用户是否已经激活，布尔类型，默认值为 true
     </li>
    </ul>
