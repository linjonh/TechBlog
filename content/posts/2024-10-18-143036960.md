    <p>
     <strong>
      什么是 OpnForm ？
     </strong>
    </p>
    <blockquote>
     <p>
      <code>
       OpnForm
      </code>
      是一个开源的表单构建工具，旨在简化创建自定义表单的过程，特别适合无编码知识的用户。它通过人工智能优化表单创建流程，支持多种用途，如联系人表单、调查表等。
      <code>
       OpnForm
      </code>
      提供了一个直观的拖放界面，用户可以快速创建美观的表单，并且只需不到一分钟的时间即可完成。
     </p>
    </blockquote>
    <p>
     <strong>
      软件功能：
     </strong>
    </p>
    <ul>
     <li>
      🚀 无代码构建器，支持无限的表单和提交
     </li>
     <li>
      📝 各种输入类型：文本、日期、URL、文件上传等等
     </li>
     <li>
      🌐 可嵌入到任何地方
     </li>
     <li>
      📧 电子邮件通知
     </li>
     <li>
      💬 集成（
      <code>
       Slack
      </code>
      、
      <code>
       Webhooks
      </code>
      、
      <code>
       Discord
      </code>
      ）
     </li>
     <li>
      🧠 表单逻辑和自定义
     </li>
     <li>
      🛡️ 验证码保护
     </li>
     <li>
      📊 表单分析
     </li>
    </ul>
    <p>
     <img alt="" src="https://img-blog.csdnimg.cn/img_convert/cb8abb53cb5d8455a4ebd683764cda79.png"/>
    </p>
    <h2>
     <a id="_20">
     </a>
     安装
    </h2>
    <p>
     在群晖上以 Docker 方式安装。
    </p>
    <blockquote>
     <p>
      本文写作时，主镜像
      <code>
       jhumanj/opnform-api
      </code>
      的
      <code>
       latest
      </code>
      版本对应为
      <code>
       1.3.7
      </code>
      ；
     </p>
    </blockquote>
    <p>
     <img alt="" src="https://img-blog.csdnimg.cn/img_convert/c7d6ef813fb877d49e22dfad6a09405b.png"/>
    </p>
    <p>
     软件涉及到多个容器，采用了
     <code>
      docker-compose
     </code>
     方式安装，但没有用官方提供的的脚本安装方式，因为老苏不想下载源码，所以对目录、文件等做了调整
    </p>
    <p>
     如果你想使用官方的方式，可以参考官方的文档：
     <a href="https://docs.opnform.com/deployment/docker" rel="nofollow">
      https://docs.opnform.com/deployment/docker
     </a>
    </p>
    <p>
     下面是介绍的是老苏的纯手工安装方式
    </p>
    <h3>
     <a id="nginxconf_34">
     </a>
     nginx.conf
    </h3>
    <p>
     <code>
      nginx.conf
     </code>
     为
     <code>
      nginx
     </code>
     的配置文件，直接采用官方提供的版本，未做任何修改，源文件地址在：
     <a href="https://github.com/JhumanJ/OpnForm/blob/main/docker/nginx.conf">
      https://github.com/JhumanJ/OpnForm/blob/main/docker/nginx.conf
     </a>
    </p>
    <pre><code class="prism language-nginx">map $original_uri $api_uri {
    ~^/api(/.*$) $1;
    default $original_uri;
}

server {
listen 80;
server_name opnform;
root /app/public;

         access_log /dev/stdout;
         error_log  /dev/stderr error;

         index index.html index.htm index.php;

         location / {
           proxy_http_version 1.1;
           proxy_pass http://ui:3000;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-Host $host;
           proxy_set_header X-Forwarded-Port $server_port;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "Upgrade";
         }

         location ~/(api|open|local\/temp|forms\/assets)/ {
              set $original_uri $uri;
              try_files $uri $uri/ /index.php$is_args$args;
         }

         location ~ \.php$ {
            fastcgi_split_path_info ^(.+\.php)(/.+)$;
            fastcgi_pass api:9000;
            fastcgi_index index.php;
            include fastcgi_params;
            #fastcgi_param SCRIPT_FILENAME /usr/share/nginx/html/$fastcgi_script_name;
            fastcgi_param SCRIPT_FILENAME /usr/share/nginx/html/public/index.php;
            fastcgi_param REQUEST_URI $api_uri;
         }

}
</code></pre>

<h2>
<a id="_264">
</a>
注意事项
</h2>
<p>
一键启动过程中或者之后，可能会遇到一些问题，老苏把自己遇到的放在这里，供大家参考
</p>
<h3>
<a id="_268">
</a>
超时问题
</h3>
<p>
因为容器比较多，可能在一键启动过程中会出现超时
</p>
<p>
<img alt="" src="https://img-blog.csdnimg.cn/img_convert/8ca4635323d83ca61d723bb6b45a3355.png"/>
</p>
<p>
可以先执行
</p>
<pre><code class="prism language-bash"><span class="token comment"># 将超时时间设置为 5 分钟</span>
<span class="token builtin class-name">export</span> <span class="token assign-left variable">COMPOSE_HTTP_TIMEOUT</span><span class="token operator">=</span><span class="token number">300</span>
</code></pre>
<p>
<img alt="" src="https://img-blog.csdnimg.cn/img_convert/85200acc4fe23b118b2e0a95884e3c31.png"/>
</p>
<h3>
<a id="_283">
</a>
容器停止
</h3>
<p>
可能会遇到某个容器停止
</p>
<p>
<img alt="" src="https://img-blog.csdnimg.cn/img_convert/bb067399cd23d6d7b44f210edbd1956c.png"/>
</p>
<p>
可以尝试强制启动
</p>
<p>
<img alt="" src="https://img-blog.csdnimg.cn/img_convert/87f79b67155e04379ddade428b58c3d1.png"/>
</p>
<h3>
<a id="_294">
</a>
登录无反应
</h3>
<p>
假如一键启动后，使用默认的账号密码登录没有反应，很可能是数据库没有
<code>
migrate
</code>
成功，导致创建默认凭据失败
</p>
<p>
<img alt="" src="https://img-blog.csdnimg.cn/img_convert/619a9f32d432e48855352b0140546aaf.png"/>
</p>
<p>
可以单独再执行一次
</p>
<pre><code class="prism language-bash"><span class="token comment"># 进入容器</span>
docker <span class="token builtin class-name">exec</span> --user root -it opnform-api /bin/bash

<span class="token comment"># 迁移数据 </span>
php artisan migrate:refresh --seed
</code></pre>
<p>
或者只用一条命令来解决
</p>
<pre><code class="prism language-bash"><span class="token comment"># 迁移数据 </span>
docker-compose <span class="token builtin class-name">exec</span> api php artisan migrate:refresh --seed
</code></pre>
