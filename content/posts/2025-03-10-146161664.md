    <p>
     Spring MVC默认通过父子容器实现Web层与非Web组件的隔离。但在实际项目中，若未明确控制组件的扫描路径与加载规则，表现层的
     <code>
      Controller
     </code>
     、业务层的
     <code>
      Service
     </code>
     与数据层的
     <code>
      Repository
     </code>
     往往会被“一刀切”地扫描到同一上下文中。例如，业务层的
     <code>
      Service
     </code>
     被意外注册到Spring MVC的Web上下文中，或数据源
     <code>
      DataSource
     </code>
     等基础设施Bean被表现层的组件直接依赖。这种混乱的加载方式不仅可能破坏分层架构的纯净性，还会导致事务管理失效、依赖注入冲突，甚至引发性能隐患。
    </p>
    <h2>
     加载控制
    </h2>
    <p>
     关键点：
    </p>
    <p>
     在标准的 Spring MVC 应用中，存在
     <strong>
      两个独立的上下文
     </strong>
     ：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        上下文类型
       </th>
       <th>
        加载方式
       </th>
       <th>
        典型组件
       </th>
       <th>
        Bean 可见性
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         根上下文
        </strong>
        (Root)
       </td>
       <td>
        <code>
         ContextLoaderListener
        </code>
       </td>
       <td>
        Service、Repository、DataSource
       </td>
       <td>
        对 Web 上下文可见
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         Web 上下文
        </strong>
        (Servlet)
       </td>
       <td>
        <code>
         DispatcherServlet
        </code>
       </td>
       <td>
        Controller、Interceptor、ViewResolver
       </td>
       <td>
        仅 Web 上下文内部可见
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      <p>
       <strong>
        父子关系
       </strong>
       ：Web 上下文是根上下文的子上下文，因此 Web 上下文可以访问根上下文的 Bean，但根上下文无法访问 Web 上下文的 Bean。
      </p>
     </li>
     <li>
      <p>
       <strong>
        隔离性
       </strong>
       ：若未正确分层，Web 上下文将无法获取业务层组件。
      </p>
     </li>
    </ul>
    <p>
     方案一：精确扫描
    </p>
    <blockquote>
     <pre><code class="language-java">@Configuration
@ComponentScan("com.cc.controller")
public class SpringMvcConfig {
}
</code></pre>
    </blockquote>
    <p>
     方案二：过滤
    </p>
    <blockquote>
     <pre><code class="language-java">import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.FilterType;
import org.springframework.stereotype.Controller;
import org.springframework.stereotype.Repository;
import org.springframework.stereotype.Service;

@Configuration
@ComponentScan(
        value = "com.cc" ,     //明确扫描哪个包 value同等于basePackage
        excludeFilters = @ComponentScan.Filter(    //表示排除哪些包
                type = FilterType.ANNOTATION,
                classes = Controller.class
        ),
        includeFilters = @ComponentScan.Filter(    //表示精确扫描哪些包
                type = FilterType.ANNOTATION,
                classes = {Service.class, Repository.class}
        )
)
public class SpringConfig {
}
</code></pre>
    </blockquote>
    <blockquote>
     <p>
      测试：首先将spriongMvcConfig中的注解注释，仅让SpringConfig生效。写一个测试类看是否能获取UserController（在扫描SpringConfig时排除了Controller注解（见上面的代码），所以不能获取到）：
     </p>
     <pre><code class="language-java">import com.cc.config.SpringConfig;

public class App {
    public static void main(String[] args) {

        System.out.println(a.getBean(UserController.class));
        
    }
}</code></pre>

    <h2>
     简化开发
    </h2>
    <p>
     回到servlet容器，在springmvc环境和pring加载不同的配置：
    </p>
    <pre><code class="language-java">public class ServletConfig extends AbstractDispatcherServletInitializer {

    //加载springmvc容器配置
    @Override
    protected WebApplicationContext createServletApplicationContext() {
        AnnotationConfigWebApplicationContext app = new AnnotationConfigWebApplicationContext();
        app.register(SpringMvcConfig.class);
        return app;
    }

    //设置哪些请求归属springMVC处理
    @Override
    protected String[] getServletMappings() {
        return new String[]{"/"};  //所有请求
    }

    //加载spring容器配置
    @Override
    protected WebApplicationContext createRootApplicationContext() {
        AnnotationConfigWebApplicationContext app = new AnnotationConfigWebApplicationContext();
        app.register(SpringConfig.class);
        return app;
    }
}</code></pre>
    <blockquote>
     <p>
      该方法可以从继承AbstractDispatcherServletInitializer换继承AbstractAnnotationConfigDispatcherServletInitializer：
     </p>
     <pre><code class="language-java">public class ServletConfig extends AbstractAnnotationConfigDispatcherServletInitializer {


    @Override
    protected Class&lt;?&gt;[] getRootConfigClasses() {
        return new Class[]{SpringConfig.class};
    }

    @Override
    protected Class&lt;?&gt;[] getServletConfigClasses() {
        return new Class[]{SpringMvcConfig.class};
    }

    @Override
    protected String[] getServletMappings() {
        return new String[]{"/"};
    }
}</code></pre>
