### 2. 项目的启动文件介绍

在
`src/`
目录下，主要的启动文件是
`main.py`
。这个文件负责初始化项目并启动主要的应用程序。

#### main.py 内容概述

```python
import config
from app import create_app

def main():
app = create_app(config.DevelopmentConfig)
app.run()

if **name** == "**main**":
main()

```

* **import config**
  : 导入配置文件，用于加载项目的配置参数。
* **from app import create\_app**
  : 从
  `app`
  模块导入
  `create_app`
  函数，用于创建应用程序实例。
* **main()**
  : 主函数，负责创建应用程序实例并运行。
* **if
  **name**
  == "
  **main**
  "**
  : 确保脚本直接运行时才执行
  `main()`
  函数。

### 3. 项目的配置文件介绍

配置文件
`config.py`
位于
`src/`
目录下，用于存储项目的配置参数。

#### config.py 内容概述

```python
import os

class Config:
SECRET_KEY = os.environ.get('SECRET_KEY') or 'hard_to_guess_string'

class DevelopmentConfig(Config):
DEBUG = True

class ProductionConfig(Config):
DEBUG = False

```

* **Config**
  : 基础配置类，包含通用的配置参数，如
  `SECRET_KEY`
  。
* **DevelopmentConfig**
  : 开发环境配置类，继承自
  `Config`
  ，设置
  `DEBUG`
  为
  `True`
  。
* **ProductionConfig**
  : 生产环境配置类，继承自
  `Config`
  ，设置
  `DEBUG`
  为
  `False`
  。

这些配置类可以根据不同的运行环境进行选择，以确保项目在不同环境下的正确运行。
