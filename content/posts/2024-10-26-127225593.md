    <blockquote>
     <p>
      一般来说，现在公司的业务都是后端去对接外部官方接口。但是现在加个紧急需求，往我这丢了几个对接文档，让前端去对接，我内心也是打咩。
     </p>
    </blockquote>
    <h4>
     <a id="_3">
     </a>
     一、梳理开发对接流程
    </h4>
    <ol>
     <li>
      大概预览一遍对接文档，发现是需要引入外部提供给的js文件，然后再调用特定方法名通过jsonp的形式与外部联调。
     </li>
     <li>
      因为调用特定方法需要提供域名给对方备案，这样才能权限调起接口。业务这边上报了几个域名给对方，但是问题来了，本地要这么联调呢？ 我%x#！@#&amp;*，我也没做过哇，只能查资料开始准备工作。
     </li>
    </ol>
    <h4>
     <a id="_7">
     </a>
     二、准备工作
    </h4>
    <h5>
     <a id="1_hots_8">
     </a>
     1. 首先需要将本地的访问都修改伪装成报备的域名。那要怎么做呢？了解修改hots文件就可以实现，那么开始动手。
    </h5>
    <p>
     1.1 首先找到hots文件，windows它路径一般是在C:\Windows\System32\drivers\etc，如下图
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/804f0dc04b108f50e578a33a43966ca0.png"/>
    </p>
    <p>
     1.2 打开hots文件，添加解析。把本地访问指向后面报备的域名。（域名可以不用加协议，我这里报备的是https的域名）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/10cd75a0c821922297d5ef886236335a.png"/>
    </p>
    <h5>
     <a id="2__15">
     </a>
     2. 运行项目。
    </h5>
    <p>
     2.1 一开始只在vue.config.js配置了端口，在通过命令跑起项目
    </p>
    <pre><code class="prism language-javascript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token literal-property property">devServer</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">port</span><span class="token operator">:</span> <span class="token number">9012</span><span class="token punctuation">,</span>
  <span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/578d073912730c50428753d3c7add730.png">
      <br/>
      2.2 复制ip链接，浏览器打开，会出现资源加载不到的问题
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f6f25c805a904e2cd73b770be747d505.png">
       <br/>
       2.3 替换成配置hots里面的域名，会发现项目无法运行
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0bd07443ca4ff3af3c69a98c053b9a9f.png"/>
      </img>
     </img>
    </p>
    <p>
     想起来报备配置的域名是https，所以vue.config.js里面随便写的port端号号是不行的，因为https默认端口号是443。
     <br/>
     2.4 修改端口号为443，并且开启https服务，重跑项目
    </p>
    <pre><code class="prism language-javascript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token literal-property property">devServer</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">https</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
    <span class="token literal-property property">port</span><span class="token operator">:</span> <span class="token number">443</span><span class="token punctuation">,</span>
  <span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     出现以下提示，点击立即前往就行
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/129ed1850197b2be37d8be3d6970b12b.png"/>
    </p>
    <p>
     2.5 进入页面却又会提示以下错误，搜索发现可以配置vue.config.js解决
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/565d6be68f5d3ebee9636c494b29e665.png">
      <br/>
      添加配置
     </img>
    </p>
    <pre><code class="prism language-javascript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token literal-property property">devServer</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">https</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
    <span class="token literal-property property">port</span><span class="token operator">:</span> <span class="token number">443</span><span class="token punctuation">,</span>
    <span class="token literal-property property">disableHostCheck</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
  <span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>
</code></pre>
    <blockquote>
     <p>
      如果上面这种方案不行可以替换成下面这种：
      <br/>
      在webpack 5 中disableHostCheck应该被遗弃了
      <br/>
      查看文档了解到需将 disableHostCheck:true
      <br/>
      替换为
      <br/>
      historyApiFallback: true,
      <br/>
      allowedHosts: “all”,
     </p>
    </blockquote>
    <pre><code class="prism language-javascript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token literal-property property">devServer</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">https</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
    <span class="token literal-property property">port</span><span class="token operator">:</span> <span class="token number">443</span><span class="token punctuation">,</span>
    <span class="token literal-property property">historyApiFallback</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span>
    <span class="token literal-property property">llowedHosts</span><span class="token operator">:</span> <span class="token string">"all"</span><span class="token punctuation">,</span>
  <span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     2.6 重跑项目,发现已经是可以进入页面了
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/eeefe3eeaf3fc9e2f7ec178b737e606e.png"/>
    </p>
    <h5>
     <a id="3_75">
     </a>
     3.开始对接
    </h5>
    <p>
     3.1 在idnex.html引入文档提供的外部js
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a9d1e14945c0862e4f7146738c709341.png"/>
     <br/>
     3.2 打开控制台发现以下错误
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a8f76708c4c89b914a52a9b0edf0047a.png"/>
    </p>
    <blockquote>
     <p>
      搜索了解到:https地址中，如果加载了http资源，浏览器将认为这是不安全的资源，将会默认阻止，这就会给你带来资源不全的问题了，比如：图片显示不了，样式加载不了，JS加载不了。
     </p>
    </blockquote>
    <p>
     <strong>
      解决
     </strong>
     :. 在index.html页面中加入（meta）头中添加upgrade-insecure-requests
    </p>
    <pre><code class="prism language-javascript"><span class="token operator">&lt;</span>meta http<span class="token operator">-</span>equiv<span class="token operator">=</span><span class="token string">"Content-Security-Policy"</span> content<span class="token operator">=</span><span class="token string">"upgrade-insecure-requests"</span><span class="token operator">&gt;</span>
</code></pre>
    <p>
     3.3 刷新页面,会发现调用特定方法成功
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8d96df654c8d07df8a88693a338bf05c.png"/>
    </p>
    <h5>
     <a id="4__88">
     </a>
     4. 到此就可以开始写页面对接其他接口了。
