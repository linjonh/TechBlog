     <h3>
      <a id="1__13">
      </a>
      1. 客户端创作中心数据统计
     </h3>
     <p>
      用户的创作中心，会对用户的视频数据进行统计：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1fb6abdea18c4e47a8db4363198e8f1d.png">
       <br/>
       定时统计，将用户的视频数据定时更新到数据统计表中。
       <br/>
       通过一个定时任务，将这个数据定时统计到数据表中：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7832bd85ba28467c9db60fbc326b76d6.png"/>
      </img>
     </p>
     <ul>
      <li>
       <code>
        @Scheduled(cron = "0 0 0 * * ?")
       </code>
       ：定义一个
       <strong>
        定时任务
       </strong>
       ，cron表达式指定了
       <strong>
        任务的执行时间
       </strong>
       。这里的0 0 0 * * ?表示任务将在每天的午夜（0点0分0秒）执行一次。
      </li>
     </ul>
     <h4>
      <a id="11__22">
      </a>
      1.1 统计播放量
     </h4>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/20a3e4157ced417581a233c85d4ac316.png"/>
     </p>
     <ul>
      <li>
       获取一天前的日期：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e664c47452244ef481e46fe66047dc97.png"/>
      </li>
      <li>
       根据前一天的日期，从 Redis 中获取每个视频的播放量
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/246b9efc88144294ad0b4ac09f0a9527.png">
        <br/>
        Redis中存储的键值对是：“easylive:video:playcount:2025.12.08:sjkhfsdhfnd”: 1,
        <br/>
        用 keyList 存储前面的键，KeyValueList 存储后面的播放量，然后通过 lambda 表达式，将其分别存入 resaultMap 中。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/60effff250944f6db9bceea5e64b6b00.png"/>
       </img>
      </li>
      <li>
       再通过 lambda 表达式，获取键中的最后一个 “ : ” 后的 videoId，将它们存入 List 中，然后查询每个视频 Id 对应的视频信息 videoInfoList。
      </li>
      <li>
       从 videoInfoList 中，
       <strong>
        根据用户 Id 进行分组，查询每个用户下的每个视频的播放量的总和
       </strong>
       ，然后将用户 ID 和总播放量分别作为
       <strong>
        键
       </strong>
       和
       <strong>
        值
       </strong>
       存入 videoCountMap.
      </li>
      <li>
       最后遍历 Map，将数据插入数据统计表中。
      </li>
     </ul>
