    <h2>

     一、git是什么，它用来做什么？
    </h2>
    <p>
     都知道，在公司里面往往一个项目会由多个人共同协同编写，所以需要一个可以文件互通的渠道，之前用的是svn或者cvs。这个svn它就相当于是有个中央服务器，开发人员把代码写完之后提交到中央服务器。git就相当于是每个开发人员的主机上都可以保存文件，每一个节点都有完整的代码库。
     <br/>
     对于个人来说，利用git来管理项目也很不错，比如说，我们需要修改一个功能代码，改之后发现不能用，这时候我们就可以利用git reset功能来恢复到前一个版本，然后后面觉得这个版本有用了又可以回到这个版本。所以fei chang you y
    </p>

    <h4>
     <a id="231_Git_58">

     2.3.1 Git的上传原理
    </h4>
    <p>
     <img alt="git工作机制" src="https://i-blog.csdnimg.cn/direct/ab1a1d95a48e46e9a682b0bfcb307c3b.png">
      <br/>
      git将提交文件的工作步骤区间分为工作区、暂存区以及仓库。从工作区到暂存区。
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> ls-files <span class="token parameter variable">-s</span> <span class="token comment">#查看暂存区的文件列表</span>
<span class="token function">git</span> ls-tree <span class="token parameter variable">-r</span> HEAD <span class="token comment">#查看仓库当前版本的文件列表</span>
</code></pre>
    <p>
     使用git add指令提交文件，从暂存区到仓库，使用的是git commit。其中从工作区到暂存区，我们可以一个一个文件上传，也可以整个文件夹上传，而git commit指令是一次性将暂存区中的所有修改内容均上传到仓库中。如下图所示。从这里我们也可以看出暂存区的作用是什么。svn中有一个概念叫做原子性提交，即要么这个操作内所有指令均操作，要不就均不操作。这里的暂存区就类似提交前的预览区，提供给用户试错成本。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> 文件1 <span class="token comment">#将文件1提交到暂存区</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span> <span class="token comment">#将所有文件提交到暂存区</span>
<span class="token function">git</span> status <span class="token comment">#查看当前git各个步骤区间文件状态</span>
Git commit <span class="token parameter variable">-m</span> “对于本次操作的描述" <span class="token comment">#将暂存区的所有文件提交到仓库中，这里的-m是指message</span>

<span class="token function">git</span> log <span class="token comment">#查看历史提交记录</span>
<span class="token comment"># git log [options] 这里的options 可以是 </span>
<span class="token comment"># —all 显示所有分支</span>
<span class="token comment"># —pretty=oneline 将提交信息显示为一行</span>
<span class="token comment"># —abbrev-commit 使得输出的commitId更简短</span>
<span class="token comment"># —graph 以图的形式显示</span>
</code></pre>

    <h4>
     <a id="232__81">
     </a>
     2.3.2 版本回退相关内容
    </h4>
    <p>
     git reset
     <br/>
     git reset的作用：
    </p>
    <ol>
     <li>
      撤销暂存区的更改：取消已加入暂存区（即staged）的文件。
     </li>
     <li>
      撤销工作目录的更改：将工作目录中的文件恢复到某个指定的提交状态。
     </li>
     <li>
      修改分支历史：通过改变分支的HEAD指针，来修改历史提交。
     </li>
    </ol>
    <p>
     1） 文件的reset
    </p>
    <pre><code class="prism language-bash"><span class="token comment">#git reset &lt;file&gt; 可以将用git add添加到暂存区的文件</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fb3560aec0bf4c248e8b97ea1c2c71e0.png">
      <br/>
      2）版本的reset
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> reset <span class="token punctuation">[</span>选项<span class="token punctuation">]</span> 回退的版本号
<span class="token function">git</span> reset —hard 回退的版本号  <span class="token comment">#这个版本号可以去git log中寻找</span>
<span class="token comment"># —hard 重置位置的同时，直接将working Tree工作目录、index暂存区及repository都重置成目标reset节点的内容，所以效果看起来等同于清空缓存区和工作区。</span>
<span class="token comment"># —soft 重置位置的同时，保留working Tree工作目录和Index暂存区的内容，只让repository中的内容和reset目标节点保持一致，因此原节点和reset节点之间的【差异变更集】会放入Index暂存区中（Staged files）。所以效果看起来就是工作目录的内容不变，暂存区原有的内容也不变，只是原节点和Reset节点之间的所有差异都会放到暂存区中。</span>
<span class="token comment"># —mixed（默认） 重置位置的同时，只保留working Tree工作目录的内容，但会将Index暂存区和Repository中的内容更改和reset目标节点一致，因此原节点和Reset节点之间的所有差异都会放到暂存区中。</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/08a38c5c37694d69b976ee55785369ad.png"/>
    </p>

    <p>
     ssh登录原理

     <br/>
     首先需要生成 ssh key
     <br/>
     1） 创建ssh key
    </p>
    <pre><code class="prism language-bash">ssh-keygen <span class="token parameter variable">-t</span> rsa <span class="token parameter variable">-C</span> <span class="token string">"github邮箱"</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3e6381689dae42a6a8243a229fb4f404.png"/>
     <br/>
     2） pbcopy 将后面文件里面的内容复制到剪切板，并开始服务，进行相关配置
    </p>
    <pre><code class="prism language-bash"><span class="token comment">#把生成的公钥复制在粘贴板上</span>
pbcopy <span class="token operator">&lt;</span> ~/.ssh/id_ed25519.pub
<span class="token comment">#开启ssh-agent服务</span>

<span class="token function">touch</span> ~/.ssh/config
<span class="token comment">#将SSH私钥存储在密钥链中。</span>
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
</code></pre>
    <p>
     将下述文本写入config文件中：
    </p>
    <pre><code class="prism language-bash">Host github.com
  AddKeysToAgent <span class="token function">yes</span>

  IdentityFile ~/.ssh/id_ed25519
</code></pre>
