     </strong>
    </h2>
    <h3 id="3.1%E8%BD%AC%E6%8D%A2%E8%A7%84%E5%88%99%C2%A0" style="margin-left:0pt;">
     <strong>
      <strong>
       3.1转换规则
      </strong>
     </strong>
    </h3>
    <p>
     一个实体型转换成一个关系模型。实体的属性就是关系的属性，实体的码就是关系的码。
    </p>
    <p>
     实体型间的联系常有如下不同的情况：
    </p>
    <p>
     a. 一个1:1联系可以转换为一个独立的关系模式，也可以任意一端对应的关系模式合并。
    </p>
    <p>
     b. 一个1：n联系可以转换成一个独立的关系模式，也可以与n端对应的关系模式合并。
    </p>
    <p>
     c. 一个m:n联系转换成一个关系模式。
    </p>
    <p>
     d. 3或3个以上实体间的一个多联系可以转换成一个关系模式。
    </p>
    <p>
     e. 具有相同码的关系模式可合并。
    </p>
    <h3 id="3.2%20%E6%A0%B9%E6%8D%AE%E4%BB%A5%E4%B8%8A%E8%A7%84%E5%88%99%E5%BE%97%E5%88%B0%E5%A6%82%E4%B8%8B%E5%85%B3%E7%B3%BB%E6%A8%A1%E5%9E%8B%EF%BC%88%E7%BA%A2%E8%89%B2%E5%AD%97%E4%BD%93%E7%9A%84%E5%B1%9E%E6%80%A7%E4%B8%BA%E4%B8%BB%E7%A0%81%EF%BC%89%C2%A0" style="margin-left:0pt;">

    <h3 id="6.1%E5%A4%87%E4%BB%BD%E6%95%B0%E6%8D%AE%E5%BA%93" style="margin-left:0pt;">

     </a>
     <br/>
     <strong>
      <strong>
       6.1备份数据库
      </strong>
     </strong>
    </h3>
    <p>
     完全备份：这是大多数人常用的方式，它可以
     <a href="https://baike.baidu.com/item/%E5%A4%87%E4%BB%BD" rel="nofollow" title="备份">
      备份
     </a>
     整个数据库，包含用户表、系统表、索引、视图和
     <a href="https://baike.baidu.com/item/%E5%AD%98%E5%82%A8%E8%BF%87%E7%A8%8B" rel="nofollow" title="存储过程">
      存储过程
     </a>
     等所有
     <a href="https://baike.baidu.com/item/%E6%95%B0%E6%8D%AE%E5%BA%93%E5%AF%B9%E8%B1%A1" rel="nofollow" title="数据库对象">
      数据库对象
     </a>
     。但它需要花费更多的时间和空间，所以，一般推荐一周做一次完全备份。
    </p>
    <p>
     <a name="3_2">
     </a>
     <br/>
     <a name="事务日志备份">
     </a>
     <br/>
     <a name="sub1302941_3_2">
     </a>
     <br/>
     <a name="3-2">
     </a>
     <br/>
     事务日志备份：事务日志是一个单独的文件，它记录数据库的改变，
     <a href="https://baike.baidu.com/item/%E5%A4%87%E4%BB%BD" rel="nofollow" title="备份">
      备份
     </a>
     的时候只需要复制自上次备份以来对数据库所做的改变，所以只需要很少的时间。为了使数据库具有鲁棒性，推荐每小时甚至更频繁的备份事务日志。
    </p>
    <p>
     <a name="3_3">
     </a>
     <br/>
     <a name="3-3">
     </a>
     <br/>
     <a name="sub1302941_3_3">
     </a>
     <br/>
     <a name="差异备份">
     </a>
     <br/>
     差异备份：也叫
     <a href="https://baike.baidu.com/item/%E5%A2%9E%E9%87%8F%E5%A4%87%E4%BB%BD" rel="nofollow" title="增量备份">
      增量备份
     </a>
     。它是只
     <a href="https://baike.baidu.com/item/%E5%A4%87%E4%BB%BD" rel="nofollow" title="备份">
      备份
     </a>
     数据库一部分的另一种方法，它不使用事务日志，相反，它使用整个数据库的一种新映象。它比最初的完全备份小，因为它只包含自上次完全备份以来所改变的数据库。它的优点是存储和恢复速度快。推荐每天做一次
     <a href="https://baike.baidu.com/item/%E5%B7%AE%E5%BC%82%E5%A4%87%E4%BB%BD" rel="nofollow" title="差异备份">
      差异备份
     </a>
     。
    </p>
    <p>
     <a name="sub1302941_3_4">
     </a>
     <br/>
     <a name="3_4">
     </a>
     <br/>
     <a name="文件备份">
     </a>
     <br/>
     <a name="3-4">
     </a>
     <br/>
     文件备份：数据库可以由硬盘上的许多文件构成。如果这个数据库非常大，并且一个晚上也不能将它
     <a href="https://baike.baidu.com/item/%E5%A4%87%E4%BB%BD" rel="nofollow" title="备份">
      备份
     </a>
     完，那么可以使用文件备份每晚备份数据库的一部分。由于一般情况下数据库不会大到必须使用多个文件存储，所以这种备份不是很常用。
    </p>
