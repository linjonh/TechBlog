    <h3>
     <a id="_0">
     </a>
     查看外接硬盘信息
    </h3>
    <p>
     连接外接硬盘后，使用以下命令识别设备：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         lsblk
        </code>
       </strong>
       ：列出块设备及其挂载点
      </p>
      <pre><code class="prism language-bash">lsblk
</code></pre>
      <p>
       示例输出可能显示设备名称如
       <code>
        /dev/sdb
       </code>
       。
      </p>
      <p>
       通过
       <code>
        lsblk -f
       </code>
       可同时显示文件系统类型和 UUID。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         fdisk -l
        </code>
       </strong>
       ：列出所有磁盘的分区信息（需要 root 权限）
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">fdisk</span> <span class="token parameter variable">-l</span>
</code></pre>
      <p>
       输出中需关注 Disk model、容量和分区表类型（DOS/GPT）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         blkid
        </code>
       </strong>
       ：显示设备的 UUID 和文件系统类型
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> blkid
</code></pre>
      <p>
       也可通过
       <code>
        ls -l /dev/disk/by-uuid
       </code>
       查看 UUID 与设备的对应关系。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_33">
     </a>
     硬盘分区
    </h3>
    <p>
     若需分区，使用
     <code>
      fdisk
     </code>
     工具：
    </p>
    <ol>
     <li>
      启动
      <code>
       fdisk
      </code>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">fdisk</span> /dev/sdb  <span class="token comment"># 假设设备为 /dev/sdb</span>
</code></pre>
    <ol start="2">
     <li>
      在交互界面中：
     </li>
    </ol>
    <ul>
     <li>
      输入
      <code>
       n
      </code>
      创建新分区
     </li>
     <li>
      选择
      <code>
       p
      </code>
      （主分区），输入分区号和大小（如
      <code>
       +10G
      </code>
      ）
     </li>
     <li>
      输入
      <code>
       t
      </code>
      可修改分区类型（例如改为 Linux LVM 的
      <code>
       8e
      </code>
      类型）
     </li>
     <li>
      输入
      <code>
       p
      </code>
      预览分区表
     </li>
     <li>
      输入
      <code>
       w
      </code>
      保存并退出
     </li>
    </ul>
    <p>
     注意事项：
    </p>
    <ul>
     <li>
      操作前请反复确认设备路径，误操作可能导致数据丢失
     </li>
     <li>
      对于 2TB 以上硬盘建议使用
      <code>
       parted
      </code>
      工具（支持 GPT 分区表）
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_58">
     </a>
     格式化硬盘
    </h3>
    <p>
     分区后，格式化分区为所需文件系统（如
     <code>
      ext4
     </code>
     ）：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        格式化命令
       </strong>
       ：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> mkfs.ext4 /dev/sdb1  <span class="token comment"># 假设分区为 /dev/sdb1</span>
</code></pre>
      <p>
       常用选项：
      </p>
      <ul>
       <li>
        <code>
         -L
        </code>
        设置卷标：
        <code>
         sudo mkfs.ext4 -L DataDisk /dev/sdb1
        </code>
       </li>
       <li>
        <code>
         -q
        </code>
        静默模式减少输出
       </li>
      </ul>
      <p>
       若需兼容 Windows，可用 NTFS 格式：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> mkfs.ntfs /dev/sdb1
</code></pre>
      <p>
       或使用 exFAT 格式（需安装
       <code>
        exfat-utils
       </code>
       ）：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> mkfs.exfat /dev/sdb1
</code></pre>
     </li>
    </ul>
    <p>
     警告：格式化会永久删除数据！操作前请确认分区号正确。
    </p>
    <hr/>
    <h3>
     <a id="_90">
     </a>
     挂载硬盘
    </h3>
    <p>
     挂载前需创建挂载点并执行挂载：
    </p>
    <ol>
     <li>
      <strong>
       创建挂载点
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mkdir</span> /mnt/external
</code></pre>
    <p>
     建议使用有意义的挂载路径，例如
     <code>
      /mnt/data_disk
     </code>
     。
    </p>
    <ol start="2">
     <li>
      <strong>
       挂载分区
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mount</span> /dev/sdb1 /mnt/external
</code></pre>
    <p>
     高级挂载选项示例：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">-t</span> ntfs <span class="token parameter variable">-o</span> ro,uid<span class="token operator">=</span><span class="token number">1000</span> /dev/sdb1 /mnt/external
<span class="token comment"># 挂载为 NTFS 只读模式并指定用户权限</span>
</code></pre>
    <ol start="3">
     <li>
      <strong>
       验证挂载
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">df</span> <span class="token parameter variable">-h</span>
</code></pre>
    <p>
     附加检查命令：
    </p>
    <pre><code class="prism language-bash">lsblk <span class="token parameter variable">-o</span> NAME,FSTYPE,SIZE,MOUNTPOINT,LABEL
</code></pre>
    <hr/>
    <h3>
     <a id="_129">
     </a>
     卸载硬盘
    </h3>
    <p>
     断开前安全卸载：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        卸载命令
       </strong>
       ：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">umount</span> /mnt/external
</code></pre>
      <p>
       或：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">umount</span> /dev/sdb1
</code></pre>
     </li>
    </ul>
    <p>
     若提示设备忙，可使用以下命令排查：
    </p>
    <pre><code class="prism language-bash"><span class="token function">lsof</span> /mnt/external        <span class="token comment"># 查看占用进程</span>
<span class="token function">fuser</span> <span class="token parameter variable">-vm</span> /dev/sdb1       <span class="token comment"># 显示使用该设备的进程</span>
</code></pre>
    <p>
     建议卸载后等待 5 秒再断开物理连接。
    </p>
    <hr/>
    <h3>
     <a id="_157">
     </a>
     设置开机自动挂载
    </h3>
    <p>
     编辑
     <code>
      /etc/fstab
     </code>
     实现自动挂载：
    </p>
    <ol>
     <li>
      <strong>
       获取 UUID
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> blkid /dev/sdb1
</code></pre>
    <p>
     建议直接复制 UUID 避免手动输入错误。
    </p>
    <ol start="2">
     <li>
      <strong>
       编辑 fstab
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">vi</span> /etc/fstab
</code></pre>
    <p>
     重要：编辑前需备份配置文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">cp</span> /etc/fstab /etc/fstab.bak
</code></pre>
    <ol start="3">
     <li>
      <strong>
       添加配置行
      </strong>
      ：
     </li>
    </ol>
    <pre><code>UUID=your-uuid-here /mnt/external ext4 defaults 0 2
</code></pre>
    <p>
     字段详解：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        UUID
       </strong>
       ：设备唯一标识
      </p>
     </li>
     <li>
      <p>
       <strong>
        挂载点
       </strong>
       ：目标目录路径
      </p>
     </li>
     <li>
      <p>
       <strong>
        文件系统类型
       </strong>
       ：如 ext4、ntfs 等
      </p>
     </li>
     <li>
      <p>
       <strong>
        挂载选项
       </strong>
       ：
       <code>
        defaults
       </code>
       包含
       <code>
        rw,suid,dev,exec,auto,nouser,async
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        dump 备份标志
       </strong>
       ：
       <code>
        0
       </code>
       表示禁用
      </p>
     </li>
     <li>
      <p>
       <strong>
        fsck 检查顺序
       </strong>
       ：
       <code>
        0
       </code>
       不检查，
       <code>
        1
       </code>
       为根目录，
       <code>
        2
       </code>
       为其他分区
      </p>
     </li>
     <li>
      <p>
       <strong>
        测试配置
       </strong>
       ：
      </p>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">-a</span>
</code></pre>
    <p>
     若命令无输出则表示成功，否则需立即修正错误。
    </p>
    <hr/>
    <h4>
     <a id="_212">
     </a>
     总结
    </h4>
    <p>
     通过这些命令，您可以快速掌握 Linux 外接硬盘操作！
    </p>
    <p>
     扩展技巧：
    </p>
    <ul>
     <li>
      <p>
       使用
       <code>
        udisksctl
       </code>
       进行无 root 挂载（适用于桌面环境）
      </p>
     </li>
     <li>
      <p>
       在 fstab 中添加
       <code>
        nofail
       </code>
       选项避免系统启动失败：
      </p>
      <pre><code>UUID=... /mnt/external ext4 defaults,nofail 0 0
</code></pre>
     </li>
     <li>
      <p>
       快速检查挂载状态：
       <code>
        mount | grep sdb1
       </code>
      </p>
     </li>
     <li>
      <p>
       定期检查硬盘健康状态：
       <code>
        sudo smartctl -a /dev/sdb
       </code>
      </p>
     </li>
    </ul>
    <p>
     注意事项：
    </p>
    <ul>
     <li>
      避免直接编辑已挂载磁盘的分区表
     </li>
     <li>
      NTFS 文件系统在 Linux 下的写入性能可能较低
     </li>
     <li>
      大容量存储建议使用 LVM 进行灵活管理
     </li>
    </ul>
