    <h2>
     <a id="_ohosarkuinode_0">
     </a>
     鸿蒙 @ohos.arkui.node
    </h2>
    <p>
     在鸿蒙开发中，
     <code>
      @ohos.arkui.node
     </code>
     模块提供了一系列用于构建和管理自定义节点的 API。这些 API 组织在一起，方便开发者进行导出和使用。本文将详细介绍
     <code>
      @ohos.arkui.node
     </code>
     模块的功能和使用方法。
    </p>
    <hr/>
    <h3>
     <a id="_6">
     </a>
     一、模块功能概述
    </h3>
    <p>
     <code>
      @ohos.arkui.node
     </code>
     模块提供了以下功能：
    </p>
    <h4>
     <a id="BuilderNode__10">
     </a>
     （一）BuilderNode 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供能够挂载原生组件的自定义节点
      <code>
       BuilderNode
      </code>
      。
     </li>
     <li>
      <strong>
       注意事项
      </strong>
      ：不建议将
      <code>
       BuilderNode
      </code>
      作为子节点挂载到其他自定义节点上。
     </li>
    </ul>
    <h4>
     <a id="FrameNode__15">
     </a>
     （二）FrameNode 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供自定义节点
      <code>
       FrameNode
      </code>
      ，表示组件树的实体节点。
     </li>
     <li>
      <strong>
       使用方法
      </strong>
      ：
      <code>
       NodeController
      </code>
      可通过
      <code>
       BuilderNode
      </code>
      持有的
      <code>
       FrameNode
      </code>
      将其挂载到
      <code>
       NodeContainer
      </code>
      上，也可通过
      <code>
       FrameNode
      </code>
      获取
      <code>
       RenderNode
      </code>
      ，挂载到其他
      <code>
       FrameNode
      </code>
      上。
     </li>
    </ul>
    <h4>
     <a id="NodeController__20">
     </a>
     （三）NodeController 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供
      <code>
       NodeController
      </code>
      用于实现自定义节点的创建、显示、更新等操作。
     </li>
     <li>
      <strong>
       使用方法
      </strong>
      ：负责将自定义节点挂载到
      <code>
       NodeContainer
      </code>
      上。
     </li>
    </ul>
    <h4>
     <a id="Graphics__25">
     </a>
     （四）Graphics 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供自定义节点相关属性设置的定义。
     </li>
    </ul>
    <h4>
     <a id="RenderNode__29">
     </a>
     （五）RenderNode 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供自绘制渲染节点
      <code>
       RenderNode
      </code>
      ，支持开发者通过 C API 进行开发，完成自定义绘制需求。
     </li>
    </ul>
    <h4>
     <a id="XComponentNode__33">
     </a>
     （六）XComponentNode 模块
    </h4>
    <ul>
     <li>
      <strong>
       功能
      </strong>
      ：提供
      <code>
       XComponent
      </code>
      节点
      <code>
       XComponentNode
      </code>
      ，表示组件树中的
      <code>
       XComponent
      </code>
      组件。
     </li>
     <li>
      <strong>
       使用场景
      </strong>
      ：用于 EGL/OpenGLES 和媒体数据写入，并支持动态修改节点渲染类型。
     </li>
    </ul>
    <hr/>

    <h3>
     <a id="_110">
     </a>
     三、注意事项
    </h3>
    <ol>
     <li>
      <strong>
       API 版本
      </strong>
      ：
      <code>
       @ohos.arkui.node
      </code>
      模块的首批接口从 API version 11 开始支持。
     </li>
     <li>
      <strong>
       预览器支持
      </strong>
      ：当前不支持在预览器中使用自定义节点。
     </li>
     <li>
      <strong>
       UIContext 获取
      </strong>
      ：可以通过以下方法获取
      <code>
       UIContext
      </code>
      ：
      <ul>
       <li>
        使用
        <code>
         ohos.window
        </code>
        中的
        <code>
         getUIContext()
        </code>
        方法。
       </li>
       <li>
        通过自定义组件的内置方法
        <code>
         getUIContext()
        </code>
        。
       </li>
       <li>
        在
        <code>
         NodeController
        </code>
        的
        <code>
         makeNode
        </code>
        回调方法中获取。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_121">
     </a>
     四、总结
    </h3>
    <p>
     <code>
      @ohos.arkui.node
     </code>
     模块为鸿蒙开发提供了强大的自定义节点功能，支持开发者创建、挂载和管理自定义节点。通过
     <code>
      BuilderNode
     </code>
     、
     <code>
      FrameNode
     </code>
     和
     <code>
      NodeController
     </code>
     等模块，开发者可以实现复杂的自定义布局和交互效果。希望本文能帮助你更好地理解和使用
     <code>
      @ohos.arkui.node
     </code>
     模块。如果有任何问题或需要进一步讨论，欢迎随时交流！
    </p>
