    <h2>
     <a id="01__0">
     </a>
     01 概要
    </h2>
    <p>
     Spring Boot 是 Java 领域最流行的
     <strong>
      快速开发框架
     </strong>
     ，专为简化 Spring 应用的初始搭建和开发而设计。
    </p>
    <hr/>
    <h4>
     <a id="Spring_Boot__5">
     </a>
     一、Spring Boot 解决了什么问题？
    </h4>
    <ol>
     <li>
      <strong>
       传统 Spring 痛点
      </strong>
      <br/>
      • 繁琐的 XML 配置
      <br/>
      • 需要手动管理依赖版本
      <br/>
      • 部署依赖外部 Web 服务器（如 Tomcat）
     </li>
     <li>
      <strong>
       Spring Boot 的答案
      </strong>
      <br/>
      •
      <strong>
       约定优于配置
      </strong>
      ：自动配置 80% 的默认设置
      <br/>
      •
      <strong>
       内嵌服务器
      </strong>
      ：直接打包成可执行 JAR
      <br/>
      •
      <strong>
       起步依赖
      </strong>
      ：一键集成常用技术栈（如数据库、安全模块）
     </li>
    </ol>
    <hr/>

      <p>
       <strong>

       </strong>
       <br/>
       •
       <strong>
        Security
       </strong>
       ：快速实现登录验证和权限控制。
      </p>
     </li>
     <li>
      <p>
       <strong>
        前端与文档
       </strong>

    <h2>
     <a id="02_Spring_Boot_206">
     </a>
     02 Spring Boot自动配置原理
    </h2>
    <h4>
     <a id="1_SpringBootApplication__209">
     </a>
     1.
     <strong>
      入口：
      <code>
       @SpringBootApplication
      </code>
      注解
     </strong>
    </h4>
    <p>
     Spring Boot应用的启动类通常标注了
     <code>
      @SpringBootApplication
     </code>
     ，该注解是一个组合注解，包含：
     <br/>
     •
     <strong>
      <code>
       @EnableAutoConfiguration
      </code>
     </strong>
     ：
     <strong>
      触发自动装配的核心注解
     </strong>
     。
     <br/>
     •
     <code>
      @ComponentScan
     </code>
     ：扫描当前包及子包的组件（如
     <code>
      @Component
     </code>
     、
     <code>
      @Service
     </code>
     等）。
     <br/>
     •
     <code>
      @SpringBootConfiguration
     </code>
     ：标记为Spring Boot配置类。
    </p>
    <pre><code class="prism language-java"><span class="token annotation punctuation">@SpringBootApplication</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">MyApp</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">public</span> <span class="token keyword">static</span> <span class="token keyword">void</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token class-name">String</span><span class="token punctuation">[</span><span class="token punctuation">]</span> args<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token class-name">SpringApplication</span><span class="token punctuation">.</span><span class="token function">run</span><span class="token punctuation">(</span><span class="token class-name">MyApp</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">,</span> args<span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <hr/>
    <h4>
     <a id="2_EnableAutoConfiguration__226">
     </a>
     2.
     <strong>
      <code>
       @EnableAutoConfiguration
      </code>
      的作用
     </strong>
    </h4>
    <p>
     <code>
      @EnableAutoConfiguration
     </code>
     通过
     <code>
      @Import
     </code>
     导入
     <code>
      AutoConfigurationImportSelector
     </code>
     类，该类的核心任务是：
     <br/>
     •
     <strong>
      扫描所有
      <code>
       META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
      </code>
      文件
     </strong>
     <br/>
     •
     <strong>
      加载其中定义的自动配置类
     </strong>
     （如
     <code>
      Spring MVC
     </code>
     、
     <code>
      DataSource
     </code>
     等配置类）。
    </p>
    <hr/>
    <h4>
     <a id="3__233">
     </a>
     3.
     <strong>
      自动配置类的加载机制
     </strong>
    </h4>
    <p>
     自动配置类以
     <code>
      XXXAutoConfiguration
     </code>
     命名，例如：
     <br/>
     •
     <code>
      ServletWebServerFactoryAutoConfiguration
     </code>
     （内嵌Web服务器配置）
     <br/>
     •
     <code>
      DataSourceAutoConfiguration
     </code>
     （数据源配置）
    </p>
    <p>
     这些类通过
     <strong>
      条件注解（Conditional Annotations）
     </strong>
     控制是否生效，例如：
     <br/>
     •
     <strong>
      <code>
       @ConditionalOnClass
      </code>
     </strong>
     ：类路径存在指定类时生效。
    </p>
    <pre><code class="prism language-java"><span class="token annotation punctuation">@ConditionalOnClass</span><span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token class-name">Servlet</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">,</span> <span class="token class-name">DispatcherServlet</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">WebMvcAutoConfiguration</span> <span class="token punctuation">{<!-- --></span> <span class="token comment">/* ... */</span> <span class="token punctuation">}</span>
</code></pre>
    <p>
     •
     <strong>
      <code>
       @ConditionalOnProperty
      </code>
     </strong>
     ：特定配置属性存在时生效。
     <br/>
     •
     <strong>
      <code>
       @ConditionalOnMissingBean
      </code>
     </strong>
     ：当容器中不存在指定Bean时生效（允许用户自定义Bean覆盖默认配置）。
    </p>
    <hr/>
    <h4>
     <a id="4_Starter__249">
     </a>
     4.
     <strong>
      Starter 依赖与自动配置的关系
     </strong>
    </h4>
    <p>
     •
     <strong>
      Starter
     </strong>
     （如
     <code>
      spring-boot-starter-web
     </code>
     ）提供
     <strong>
      一组预置依赖
     </strong>
     （如Tomcat、Spring MVC）和对应的
     <strong>
      自动配置类
     </strong>
     。
     <br/>
     •
     <strong>
      约定优于配置
     </strong>
     ：只要引入Starter，Spring Boot自动装配相关组件（例如引入
     <code>
      spring-boot-starter-data-jpa
     </code>
     会自动配置数据源和JPA相关Bean）。
    </p>
    <hr/>
    <h4>
     <a id="5__255">
     </a>
     5.
     <strong>
      自动装配的完整流程
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       启动应用
      </strong>
      ：执行
      <code>
       SpringApplication.run()
      </code>
      。
     </li>
     <li>
      <strong>
       加载
       <code>
        @EnableAutoConfiguration
       </code>
      </strong>
      ：触发
      <code>
       AutoConfigurationImportSelector
      </code>
      。
     </li>
     <li>
      <strong>
       扫描所有
       <code>
        AutoConfiguration.imports
       </code>
       文件
      </strong>
      ：获取所有自动配置类。
     </li>
     <li>
      <strong>
       过滤条件注解
      </strong>
      ：仅保留满足条件的配置类（如类路径存在、属性已配置等）。
     </li>
     <li>
      <strong>
       按优先级排序
      </strong>
      ：通过
      <code>
       @AutoConfigureOrder
      </code>
      或
      <code>
       @Order
      </code>
      控制加载顺序。
     </li>
     <li>
      <strong>
       注册Bean
      </strong>
      ：将生效的配置类中的Bean定义加载到IoC容器。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="6__265">
     </a>
     6.
     <strong>
      覆盖默认自动配置
     </strong>
    </h4>
    <p>
     用户可通过以下方式自定义配置：
     <br/>
     •
     <strong>
      定义同名Bean
     </strong>
     ：使用
     <code>
      @Bean
     </code>
     覆盖自动配置的Bean（
     <code>
      @ConditionalOnMissingBean
     </code>
     会生效）。
     <br/>
     •
     <strong>
      调整配置属性
     </strong>
     ：通过
     <code>
      application.properties
     </code>
     或
     <code>
      application.yml
     </code>
     覆盖默认属性。
     <br/>
     •
     <strong>
      排除特定自动配置类
     </strong>
     ：
    </p>
    <pre><code class="prism language-java"><span class="token annotation punctuation">@SpringBootApplication</span><span class="token punctuation">(</span>exclude <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span><span class="token class-name">DataSourceAutoConfiguration</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
</code></pre>
    <hr/>
    <h4>
     <a id="7_DataSource_276">
     </a>
     7.
     <strong>
      示例：DataSource自动配置
     </strong>
    </h4>
    <p>
     以数据源为例：
    </p>
    <ol>
     <li>
      <strong>
       引入
       <code>
        spring-boot-starter-jdbc
       </code>
      </strong>
      ：类路径下存在
      <code>
       DataSource
      </code>
      和
      <code>
       HikariCP
      </code>
      。
     </li>
     <li>
      <strong>
       触发
       <code>
        DataSourceAutoConfiguration
       </code>
      </strong>
      ：检查到
      <code>
       DataSource
      </code>
      类存在。
     </li>
     <li>
      <strong>
       读取
       <code>
        spring.datasource.*
       </code>
       配置
      </strong>
      ：自动创建
      <code>
       DataSource
      </code>
      Bean。
     </li>
     <li>
      <strong>
       用户自定义
      </strong>
      ：若手动定义
      <code>
       @Bean DataSource
      </code>
      ，则默认配置失效。
     </li>
    </ol>

    <p>

     </strong>

     <strong>

     03 yaml语法
    </h2>
    <p>
     YAML（YAML Ain’t Markup Language）是一种简洁的数据序列化语言，广泛用于配置文件（如Spring Boot的
     <code>
      application.yml
     </code>
     ）。其语法强调可读性，通过缩进和符号表示结构。

     <a id="1__296">

     <strong>

     </strong>
    </h4>
    <p>
     •
     <strong>
      键值对
     </strong>
     ：用冒号
     <code>
      :
     </code>
     分隔键和值，
     <strong>
      冒号后必须加空格
     </strong>
     。
    </p>
    <pre><code class="prism language-yaml"><span class="token key atrule">name</span><span class="token punctuation">:</span> <span class="token string">"John Doe"</span>

</code></pre>
    <p>
     •
     <strong>
      注释
     </strong>
     ：以
     <code>
      #
     </code>
     开头。
    </p>
    <pre><code class="prism language-yaml"><span class="token comment"># 这是注释</span>
<span class="token key atrule">key</span><span class="token punctuation">:</span> value
</code></pre>
    <p>
     •
     <strong>
      字符串
     </strong>
     ：可省略引号，特殊字符（如
     <code>
      :
     </code>
     ）需加双引号。
    </p>
    <pre><code class="prism language-yaml"><span class="token key atrule">message</span><span class="token punctuation">:</span> Hello<span class="token punctuation">,</span> World<span class="token tag">!</span>

    <h2>
     <a id="05_JSR_303_567">
     </a>
     05 JSR 303
    </h2>
    <p>
     JSR 303（
     <strong>
      Bean Validation 1.0
     </strong>
     ）和其后续版本JSR 380（
     <strong>
      Bean Validation 2.0
     </strong>
     ）是Java平台中用于
     <strong>
      声明式数据验证
     </strong>
     的标准规范，通过注解简化数据校验逻辑，确保对象属性符合业务规则。
    </p>
    <hr/>
    <h4>
     <a id="1__572">
     </a>
     <strong>
      1. 核心特性
     </strong>
    </h4>
    <p>
     •
     <strong>
      声明式验证
     </strong>
     ：通过注解（如
     <code>
      @NotNull
     </code>
     ,
     <code>
      @Size
     </code>
     ）标记字段约束，无需手动编写校验逻辑。
     <br/>
     •
     <strong>
      标准化
     </strong>
     ：统一不同框架（如Spring、Hibernate）的校验方式。
     <br/>
     •
     <strong>
      支持嵌套校验
     </strong>
     ：可验证对象内部的其他对象（如
     <code>
      User
     </code>
     中的
     <code>
      Address
     </code>
     ）。
    </p>

    <h4>
     <a id="1__690">
     </a>
     <strong>
      1. 多环境配置的实现
     </strong>
    </h4>
    <h5>
     <a id="11__691">
     </a>
     <strong>
      1.1 配置文件命名规则
     </strong>
    </h5>
    <p>
     Spring Boot通过
     <code>
      application-{profile}.yml
     </code>
     （或
     <code>
      .properties
     </code>
     ）支持多环境配置，例如：
     <br/>
     •
     <code>
      application-dev.yml
     </code>
     ：开发环境。
     <br/>
     •
     <code>
      application-test.yml
     </code>
     ：测试环境。
     <br/>
     •
     <code>
      application-prod.yml
     </code>
     ：生产环境。
    </p>
    <h5>
     <a id="12__697">
     </a>
     <strong>
      1.2 激活指定环境
     </strong>
    </h5>
    <p>
     在**主配置文件（
     <code>
      application.yml
     </code>
     ）**中指定激活的环境：
    </p>
    <pre><code class="prism language-yaml"><span class="token comment"># application.yml</span>
<span class="token key atrule">spring</span><span class="token punctuation">:</span>
  <span class="token key atrule">profiles</span><span class="token punctuation">:</span>
    <span class="token key atrule">active</span><span class="token punctuation">:</span> dev  <span class="token comment"># 激活开发环境</span>
</code></pre>
    <p>
     或通过
     <strong>
      启动命令
     </strong>
     动态指定：
    </p>
    <pre><code class="prism language-bash"><span class="token function">java</span> <span class="token parameter variable">-jar</span> myapp.jar <span class="token parameter variable">--spring.profiles.active</span><span class="token operator">=</span>prod
</code></pre>
    <h5>
     <a id="13__711">
     </a>
     <strong>
      1.3 环境专属配置
     </strong>
    </h5>
    <p>
     每个环境配置文件仅包含该环境特有的属性，例如：
    </p>
    <pre><code class="prism language-yaml"><span class="token comment"># application-prod.yml</span>

  <span class="token key atrule">datasource</span><span class="token punctuation">:</span>
    <span class="token key atrule">url</span><span class="token punctuation">:</span> jdbc<span class="token punctuation">:</span>mysql<span class="token punctuation">:</span>//prod<span class="token punctuation">-</span>db<span class="token punctuation">:</span>3306/mydb

    <h5>
     <a id="14__726">
     </a>
     <strong>

     </strong>
    </h5>
    <p>
     •
     <strong>
      通用配置
     </strong>
     ：写在
     <code>
      application.yml
     </code>
     中，所有环境共享。
     <br/>
     •
     <strong>
      环境专属配置
     </strong>
     ：写在
     <code>
      application-{profile}.yml
     </code>
     中，
     <strong>
      优先级高于通用配置
     </strong>
     。
    </p>

    <h4>
     <a id="2__732">
     </a>
     <strong>
      2. 配置文件的存储位置
     </strong>
    </h4>
    <p>
     Spring Boot按以下顺序加载配置文件（优先级从高到低）：
    </p>
    <h5>
     <a id="21__735">
     </a>
     <strong>
      2.1 外部化配置路径
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       <code>
        file:./config/
       </code>
      </strong>
      ：项目根目录下的
      <code>
       config
      </code>
      文件夹。
      <pre><code>myapp/
  ├── config/
  │   └── application.yml
  └── application.jar
</code></pre>
     </li>
     <li>
      <strong>
       <code>
        file:./
       </code>
      </strong>
      ：项目根目录。
     </li>
     <li>
      <strong>
       <code>
        classpath:/config/
       </code>
      </strong>
      ：类路径下的
      <code>
       config
      </code>
      目录。
     </li>
     <li>
      <strong>
       <code>
        classpath:/
       </code>
      </strong>
      ：类路径根目录。
     </li>
    </ol>
    <p>
     <em>
      类路径
     </em>
     –&gt; resource包下和java包下
    </p>
    <h5>
     <a id="22__749">
     </a>
     <strong>
      2.2 配置加载规则
     </strong>
    </h5>
    <p>
     •
     <strong>
      外部配置文件优先级高于Jar内部
     </strong>
     ：例如，
     <code>
      file:./config/application.yml
     </code>
     会覆盖Jar内的
     <code>
      application.yml
     </code>
     。
     <br/>
     •
     <strong>
      多环境配置同样遵循位置优先级
     </strong>
     ：例如，
     <code>
      file:./config/application-prod.yml
     </code>
     优先于
     <code>
      classpath:/application-prod.yml
     </code>
     。
    </p>

     07 静态资源导入
    </h2>
    <p>
     Spring Boot 的静态资源处理是其Web开发中的重要特性，它简化了前端资源（如HTML、CSS、JS、图片等）的管理和访问。
    </p>
    <hr/>
    <h4>
     <a id="1__838">
     </a>
     <strong>
      1. 默认静态资源路径
     </strong>
    </h4>
    <p>
     Spring Boot 默认会在以下类路径（
     <code>
      classpath
     </code>
     ）目录中查找静态资源，
     <strong>
      优先级从高到低
     </strong>
     如下：
    </p>
    <ol>
     <li>
      <strong>
       <code>
        classpath:/META-INF/resources/
       </code>
      </strong>
      <br/>
      （例如：JAR包内
      <code>
       META-INF/resources
      </code>
      目录）
     </li>
     <li>
      <strong>
       <code>
        classpath:/resources/
       </code>
      </strong>
      <br/>
      （对应项目中的
      <code>
       src/main/resources/resources/
      </code>
      ）
     </li>
     <li>
      <strong>
       <code>
        classpath:/static/
       </code>
      </strong>
      <br/>
      （最常用，对应
      <code>
       src/main/resources/static/
      </code>
      ）
     </li>
     <li>
      <strong>
       <code>
        classpath:/public/
       </code>
      </strong>
      <br/>
      （对应
      <code>
       src/main/resources/public/
      </code>
      ）
     </li>
    </ol>
    <h5>
     <a id="_849">
     </a>
     <strong>
      验证默认路径
     </strong>
    </h5>
    <p>
     • 创建文件：在
     <code>

     <strong>
      4.1 静态资源404错误
     </strong>
    </h5>
    <p>
     •
     <strong>
      可能原因
     </strong>
     ：
     <br/>
     • 文件未放在默认或自定义的静态资源目录中。
     <br/>
     • 路径拼写错误（区分大小写）。
     <br/>
     • 自定义配置覆盖了默认路径但未正确包含原有路径。
    </p>
    <p>
     •
     <strong>
      排查步骤
     </strong>
     ：
    </p>
    <ol>
     <li>
      检查文件是否在
      <code>
       target/classes/static/
      </code>
      （编译后目录）中存在。
     </li>
     <li>
      查看Spring Boot启动日志，确认加载的资源路径：
      <pre><code>... Mapped URL path [/**] onto locations [classpath:/custom-static/]
</code></pre>
     </li>
    </ol>
    <h2>
     <a id="08_templates_942">
     </a>
     08 templates
    </h2>
    <p>
     在Spring Boot中，
     <code>
      templates
     </code>
     目录是用于存放
     <strong>
      动态模板文件
     </strong>
     的核心位置，结合模板引擎（如Thymeleaf、FreeMarker等）实现动态页面渲染。
    </p>
    <hr/>
    <h4>
     <a id="1_templates_947">
     </a>
     <strong>
      1. templates目录的作用
     </strong>
    </h4>
    <p>
     •
     <strong>
      动态页面生成
     </strong>
     ：模板文件（
     <code>
      .html
     </code>
     、
     <code>
      .ftl
     </code>
     等）可嵌入动态数据（如变量、循环、条件语句），生成最终的HTML页面。
     <br/>
     •
     <strong>
      与静态资源的区别
     </strong>
     ：
     <br/>
     •
     <strong>
      静态资源
     </strong>
     （
     <code>
      static/
     </code>
     目录）：直接返回给浏览器的固定文件（如
     <code>
      .html
     </code>
     、
     <code>
      .css
     </code>
     ）。
     <br/>
     •
     <strong>
      模板文件
     </strong>
     （
     <code>
      templates/
     </code>
     目录）：需模板引擎渲染后返回动态内容。
    </p>

    <h4>
     <a id="6__1087">
     </a>
     <strong>
      6. 常见问题
     </strong>
    </h4>
    <h5>
     <a id="Q1_1088">
     </a>
     <strong>
      Q1：模板文件修改后不生效？
     </strong>
    </h5>
    <p>
     •
     <strong>
      原因
     </strong>
     ：模板引擎缓存未关闭（生产模式默认启用缓存）。
     <br/>
     •
     <strong>
      解决
     </strong>
     ：配置
     <code>
      spring.thymeleaf.cache=false
     </code>
     （开发环境）。
    </p>
    <h5>
     <a id="Q2404_1092">
     </a>
     <strong>
      Q2：访问模板页面返回404？
     </strong>
    </h5>
    <p>
     •
     <strong>
      检查项
     </strong>
     ：
    </p>
    <ol>
     <li>
      模板文件是否在
      <code>
       templates/
      </code>
      目录下。
     </li>
     <li>
      Controller方法是否使用
      <code>
       @Controller
      </code>
      （非
      <code>
       @RestController
      </code>
      ）。
     </li>
     <li>
      视图名称是否与模板文件名一致（区分大小写）。
     </li>
    </ol>
    <h5>
     <a id="Q3JSON_1098">
     </a>
     <strong>
      Q3：如何返回JSON而非视图？
     </strong>
    </h5>
    <p>
     •
     <strong>
      方法
     </strong>
     ：使用
     <code>
      @RestController
     </code>
     或方法添加
     <code>
      @ResponseBody
     </code>
     ，直接返回数据对象而非视图名称。
    </p>

    <h4>
     <a id="1__1260">
     </a>
     <strong>
      1. 核心组件与流程
     </strong>
    </h4>
    <p>
     <a href="https://springdoc.cn/spring-mvc-tutorial/" rel="nofollow">
      Spring MVC
     </a>
     基于
     <strong>
      DispatcherServlet
     </strong>
     作为前端控制器，协调请求处理流程：
    </p>
    <ol>
     <li>
      <strong>
       请求分发
      </strong>
      ：DispatcherServlet 接收所有HTTP请求。
     </li>
     <li>
      <strong>
       处理器映射
      </strong>
      ：通过
      <code>
       HandlerMapping
      </code>
      找到匹配的控制器方法。
     </li>
     <li>
      <strong>
       处理器适配
      </strong>
      ：
      <code>
       HandlerAdapter
      </code>
      调用控制器方法并处理参数绑定。
     </li>
     <li>
      <strong>
       视图解析
      </strong>
      ：
      <code>
       ViewResolver
      </code>
      将逻辑视图名解析为具体视图（如HTML、JSON）。
     </li>
     <li>
      <strong>
       视图渲染
      </strong>
      ：视图模板引擎（如Thymeleaf）生成响应内容。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="2_Spring_Boot__1270">
     </a>
     <strong>
      2. Spring Boot 的自动配置
     </strong>
    </h4>
    <p>
     Spring Boot 通过
     <code>
      WebMvcAutoConfiguration
     </code>
     类提供默认MVC配置，包括：
     <br/>
     •
     <strong>
      静态资源处理
     </strong>
     ：映射
     <code>
      /static
     </code>
     、
     <code>
      /public
     </code>
     等目录。
     <br/>
     •
     <strong>
      默认视图解析器
     </strong>
     ：如
     <code>
      InternalResourceViewResolver
     </code>
     。
     <br/>
     •
     <strong>
      消息转换器
     </strong>
     ：自动注册 JSON、XML 转换器（如
     <code>
      MappingJackson2HttpMessageConverter
     </code>
     ）。
     <br/>
     •
     <strong>
      格式化器
     </strong>
     ：日期、数字等类型转换。
    </p>
    <p>
     <strong>
      生效条件
     </strong>
     ：项目中无自定义
     <code>
      WebMvcConfigurationSupport
     </code>
     或
     <code>
      @EnableWebMvc
     </code>
     注解。
    </p>

     实现的配置会合并，而非覆盖。

     <strong>
      6. 常见问题与解决
     </strong>
    </h4>
    <h5>
     <a id="Q1_1396">
     </a>
     <strong>
      Q1：自定义配置未生效？
     </strong>
    </h5>
    <p>
     • 检查类是否被
     <code>
      @Configuration
     </code>
     注解。
     <br/>
     • 确认未误加
     <code>
      @EnableWebMvc
     </code>
     （除非需完全自定义）。
    </p>
    <h5>
     <a id="Q2404_1400">
     </a>
     <strong>
      Q2：静态资源访问404？
     </strong>
    </h5>
    <p>
     • 确认资源路径是否在配置的
     <code>
      static-locations
     </code>
     中。
     <br/>
     • 检查是否被拦截器拦截（如未排除
     <code>
      /static/**
     </code>
     ）。
    </p>

     <a id="Q3_1404">

     <a id="7__1417">

     <strong>
      7. 总结
     </strong>
    </h4>
    <p>
     Spring MVC 的配置原理围绕
     <strong>
      扩展性
     </strong>
     与
     <strong>
      约定优于配置
     </strong>
     展开：
     <br/>
     •
     <strong>
      自动配置
     </strong>
     ：快速启动，满足大部分场景。
     <br/>
     •
     <strong>
      自定义扩展
     </strong>
     ：通过
     <code>
      WebMvcConfigurer
     </code>
     按需调整。
     <br/>
     •
     <strong>
      完全控制
     </strong>
     ：使用
     <code>
      @EnableWebMvc
     </code>
     或
     <code>
      WebMvcConfigurationSupport
     </code>
     深度定制。
    </p>
    <p>
     掌握这些机制，可以灵活应对不同业务需求，同时保持代码简洁高效。

    <p>

     <strong>

     </a>

      1. 核心特性
     </strong>
    </h5>
    <p>
     •
     <strong>
      响应式布局
     </strong>

     •
     <strong>
      预定义组件
     </strong>
     ：导航栏、按钮、表单、卡片等常用 UI 元素。

     <br/>
     •
     <strong>
      插件支持
     </strong>
     ：模态框、轮播图等交互组件（依赖 jQuery 或 Popper.js）。

     </strong>
     ：强制刷新页面（

     </code>
     ）。

     </strong>
    </h5>

     </strong>
     ：Bootstrap 5 需引入
     <code>
      bootstrap.bundle.min.js
     </code>
     （包含 Popper.js），Bootstrap 4 需额外引入 jQuery。

     嵌套在
     <code>

     </code>

     </strong>

     <strong>

     </strong>
     ：例如，在配置静态资源时，确保不覆盖 Spring Boot 的默认路径（如
     <code>
      /static
     </code>
     、
     <code>
      /public
     </code>
     ）。
     <br/>
     •
     <strong>
      方法选择性覆盖
     </strong>
     ：接口方法均为默认方法（Java 8+），只需实现需要自定义的方法。

    <hr/>
    <h4>
     <a id="1__1955">
     </a>
     <strong>
      1. 项目规划
     </strong>
    </h4>
    <p>
     •
     <strong>
      目标
     </strong>
     ：明确网站功能（如用户登录、数据展示、表单提交）。
     <br/>
     •
     <strong>
      技术选型
     </strong>
     ：
     <br/>
     •
     <strong>
      后端
     </strong>
     ：Spring Boot + MyBatis/JPA + MySQL。
     <br/>
     •
     <strong>
      前端
     </strong>
     ：React/Vue/Angular + Axios（HTTP 请求库）。
     <br/>
     •
     <strong>
      API 交互
     </strong>
     ：RESTful API，数据格式为 JSON。
     <br/>
     •
     <strong>
      模板来源
     </strong>
     ：前端模板可从
     <a href="https://themeforest.net/" rel="nofollow">
      ThemeForest
     </a>

     <a href="https://getbootstrap.com/" rel="nofollow">
      Bootstrap 官方
     </a>

    <h4>
     <a id="2__1965">
     </a>
     <strong>
      2. 环境准备
     </strong>
    </h4>
    <p>
     •
     <strong>
      后端
     </strong>
     ：
     <br/>
     • JDK 17+
     <br/>
     • Maven/Gradle
     <br/>
     • IDE（IntelliJ IDEA 或 Eclipse）
     <br/>
     • MySQL 或 H2（嵌入式数据库）
     <br/>
     •
     <strong>
      前端
     </strong>
     ：
     <br/>
     • Node.js + npm/yarn
     <br/>
     • IDE（VS Code 或 WebStorm）
     <br/>
     •
     <strong>
      工具
     </strong>
     ：
     <br/>
     • Postman（测试 API）
     <br/>
     • Git（版本控制）
