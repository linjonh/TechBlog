    <h3>
     <a id="1__17">
     </a>
     1 基础概念
    </h3>
    <br/>
    <p>
     <strong>
      ⭐ 什么是 CI/CD？
     </strong>
    </p>
    <ol>
     <li>
      <p>
       CI（持续集成）：代码提交后自动构建、测试，快速发现错误。
      </p>
     </li>
     <li>
      <p>
       CD（持续交付/部署）：自动将代码部署到测试或生产环境。
      </p>
     </li>
    </ol>
    <p>
     <strong>
      ⭐ GitLab CI/CD 核心组件
     </strong>
     <br/>
     3. Pipeline（流水线）：一次 CI/CD 流程的总称，包含多个阶段（Stages）和任务（Jobs）。
    </p>
    <ol start="4">
     <li>
      <p>
       Stage（阶段）：一组并行或顺序执行的 Jobs（例如 build、test、deploy）。
      </p>
     </li>
     <li>
      <p>
       Job（任务）：具体执行的操作（如运行 npm install），是 Pipeline 的最小单位。
      </p>
     </li>
     <li>
      <p>
       Runner：执行 Jobs 的服务器（相当于“工人”），需提前配置。
      </p>
     </li>
     <li>
      <p>
       .gitlab-ci.yml：定义 Pipeline 规则的配置文件。
       <br/>
       <br/>
      </p>
      <hr/>
      <p>
      </p>
     </li>
    </ol>

     <strong>
      🧾 流程图说明
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       准备条件
      </strong>
      ：确保项目已存在、代码已推送、服务器权限就绪。
     </li>
     <li>
      <strong>
       Runner 配置
      </strong>
      ：安装并注册 Runner，选择执行器（Docker/Shell）。
     </li>
     <li>
      <strong>
       编写配置文件
      </strong>
      ：定义阶段、变量、任务和产物。
     </li>
     <li>
      <strong>
       触发 Pipeline
      </strong>
      ：代码推送后 GitLab 自动检测配置。
     </li>
     <li>
      <strong>
       任务执行
      </strong>
      ：按阶段顺序执行构建、测试、部署。
     </li>
     <li>
      <strong>
       结果处理
      </strong>
      ：监控状态，失败时查看日志并修复。
     </li>
    </ol>
