    <p>
     数据库在涉及到多表查询的时候，都会牵扯到两个表的连接问题。那么今天就详细说明一下数据库的连接问题。
     <br/>
     首先我们来看第一个：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c9de6edaaf13f3256ab47fb92df1b9f5.png">
      <br/>
      这就是两张表共有的部分(内连接)，取交集。
      <br/>
      SQL语句：
      <br/>
      SELECT * FROM TABLEA A INNER JOIN TABLEB B ON A.KEY=B.KEY;
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/87488a886214653c65ad853db0df1222.png">
      <br/>
      A独有的部分加上和A和B公共 的部分。也叫左外连接。
      <br/>
      SQL语句：
      <br/>
      SELECT * FROM TABLEA A LEFT JOIN TABLEB B
      <br/>
      ON A.KEY = B.KEY;
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/47af5100865f90afb678d7a8edadf770.png">
      <br/>
      这张图恰好跟左外连接相反（右外连接）。
      <br/>
      SQL语句如下：
      <br/>
      SELECT * FROM TABLEA A RIGHT JOIN TABLEB B
      <br/>
      ON A.KEY = B.KEY;
     </img>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d166a1a866d8356b03f3202c7be74b42.png">
      <br/>
      这张图就是A表独有的部分。
      <br/>
      SQL语句如下：
      <br/>
      SELECT * FROM TABLEA A LEFT JOIN TABLEB B
      <br/>
      ON A.KEY = B.KEY
      <br/>
      WHERE B.KEY IS NULL;
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ffab1cdef04256fb5dd06554a67c8f62.png">
       <br/>
       这张图是B表独有的部分。
       <br/>
       SQL语句如下：
       <br/>
       SELECT * FROM TABLEA A RIGHT JOIN TABLEB B
       <br/>
       ON A.KEY B.KEY
       <br/>
       WHERE A.KEY IS NULL;
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce8894c48617e58f1861babd3becd5d6.png">
        <br/>
        上面这张图表示的是两张表的所有部分。就是左外连接+右外连接在去重一次就搞定了（全连接，mysql中不支持，oracle中是支持的）。虽然MySQL不支持全连接的直接实现方式，但是提供了间接的实现方式，就是A表独有+B表独有，在去重一次。
        <br/>
        SQL语句如下（正常全连接的SQL语句）：
        <br/>
        SELECT * FROM TABLEA A FULL OUTER JOIN TABLEB B
        <br/>
        ON A.KEY = B.KEY;
        <br/>
        但是，在mysql中不支持上面这条语句。
        <br/>
        MySQL实现全连接的SQL语句：
        <br/>
        SELECT * FROM TABLEA A LEFT JOIN TABLEB B
        <br/>
        ON A.KEY = B.KEY
        <br/>
        UNION
        <br/>
        SELECT * FROM TABLEA A RIGHT JOIN TABLEB B
        <br/>
        ON A.KEY = B.KEY;
        <br/>
        这里解释一下关键字union：就是连接并去重的意思。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/39cf5c6236201bfed2eea48ff9a79b56.png">
         <br/>
         同理，这个模型是一个全外连接。
         <br/>
         SQL语句如下：
         <br/>
         SELECT * FROM TABLEA A FULL OUTER JOIN TABLEB B
         <br/>
         ON A.KEY = B.KEY
         <br/>
         WHERE A.KEY IS NULL OR B.KEY IS NULL;
        </img>
       </img>
      </img>
     </img>
    </p>
    <p>
     在MySQL中上面这条语句还是不支持。但是，我们还是有间接的实现方式。其实就是第4和第5张图加起来去重就OK了。
     <br/>
     MySQL中的语句如下：
     <br/>
     SELECT * FROM TABLEA A LEFT JOIN TABLEB B
     <br/>
     ON A.KEY = B.KEY
     <br/>
     WHERE B.KEY IS NULL
     <br/>
     UNION
     <br/>
     SELECT * FROM TABLEA A RIGHT JOIN TABLEB B
     <br/>
     ON A.KEY = B.KEY
     <br/>
     WHERE A.KEY IS NULL;
    </p>
    <p>
     UNOIN 关键字跟上面的作用一样。
     <br/>
     在涉及到多表查询的时候，所有的join 连接都在这里了。希望可以帮到更多的人。
    </p>
