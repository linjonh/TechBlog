    <h2>
     <a id="_0">
     </a>
     一、了解
    </h2>
    <p>
     <code>
      map
     </code>
     提供了
     <code>
      key-value(键值对)
     </code>
     的存储机制，这使得map成为处理具有唯一键的关联数据的理想选择。
     <code>
      无重复值且有序
     </code>
     。
    </p>
    <ul>
     <li>
      map特性
      <ul>
       <li>
        键值对存储：每个键都有一个值对应。
       </li>
       <li>
        自动排序：map
        <code>
         内部是平衡二叉搜索树（一般用红黑树）
        </code>
        。可以自动排序
       </li>
       <li>
        元素具有唯一性：key对应唯一value
       </li>
       <li>
        直接访问：用键key来访问值
       </li>
       <li>
        灵活操作：有许多方便的操作
       </li>
      </ul>
     </li>
     <li>
      map的时间复杂度
      <ul>
       <li>
        <strong>
         插入操作
        </strong>
        ：
        <strong>
         O(log n)
        </strong>
        ，其中n是std::map中元素的数量。这是因为需要在平衡二叉树中找到合适的位置来插入新元素。
       </li>
       <li>
        <strong>
         删除操作
        </strong>
        ：删除操作的时间复杂度同样为
        <strong>
         O(log n)
        </strong>
        ，需要找到要删除的元素并在保持树平衡的同时移除它。
       </li>
       <li>
        <strong>
         查找操作
        </strong>
        ：查找操作的时间复杂度也是
        <strong>
         O(log n)
        </strong>
        ，由于std::map的有序性，可以快速定位到任何键。
       </li>
       <li>
        <strong>
         遍历操作
        </strong>
        ：遍历std::map的时间复杂度为
        <strong>
         O(n)
        </strong>
        ，因为需要访问容器中的每个元素。
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <ul>
      <li>
       <h3>
        <a id="_14">
        </a>
        常见问题
       </h3>
      </li>
      <li>
       <h3>
        <a id="1mapunordered_map_16">
        </a>
        1、map和unordered_map有什么区别？
       </h3>
       <ul>
        <li>
         <strong>
          内部实现
         </strong>
         ：map基于
         <code>
          红黑树
         </code>
         ，元素
         <strong>
          有序
         </strong>
         ；unordered_map()基于
         <code>
          哈希表
         </code>
         元素
         <strong>
          无序
         </strong>
         。
        </li>
        <li>
         <strong>
          性能
         </strong>
         ：对于map，查找、插入和删除操作的时间复杂度通常是O(log n)。对于unordered_map，这些操作的平均时间复杂度是
         <strong>
          O(1)
         </strong>
         ，但
         <strong>
          最坏情况下是O(n)
         </strong>
         。
        </li>
        <li>
         <strong>
          内存消耗
         </strong>
         ：由于
         <strong>
          哈希表的开销
         </strong>
         ，unordered_map可能会比std::map消耗
         <strong>
          更多内存
         </strong>
         。
        </li>
        <li>
         <strong>
          排序
         </strong>
         ：map中的元素按照键自动排序；unordered_map中的元素没有排序。
        </li>
       </ul>
      </li>
      <li>
       <h3>
        <a id="2mapkey_22">
        </a>
        2、如何在map中用键key来查找值？
       </h3>
       <ul>
        <li>
         find(key)函数可以返回迭代器。用它来访问。
        </li>
        <li>
         只要find返回的迭代器不等于 end（）就说明找到了。
        </li>
       </ul>
      </li>
      <li>
       <h3>
        <a id="3_26">
        </a>
        3、迭代器失效？
       </h3>
       <ul>
        <li>
         删除时，被删除元素的迭代器会失效。【与list相同】
        </li>
        <li>
         插入时，不会失效。【与list相同】
        </li>
        <li>
         map是
         <code>
          双向迭代器
         </code>
        </li>
       </ul>
      </li>
      <li>
       <h3>
        <a id="4mapemplacemapinsert_31">
        </a>
        4、map：：emplace和map：：insert的区别？
       </h3>
       <ul>
        <li>
         emplace直接构造元素。不会移动构造。
        </li>
        <li>
         insert需要移动构造，内存消耗大。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <ul>
     <li>
      使用map所需要的头文件
     </li>
    </ul>
    <pre><code>#include &lt;map&gt;
</code></pre>
