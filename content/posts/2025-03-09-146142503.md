     <h4>
      1. 概述
     </h4>
     <p>
      作为Github Copilot 的企业级用户，从管理员或管理者的角度，都系统了解企业成员当前使用Copilot的情况，例如希望了解Copilot对企业开发效能产生的实际影响。为了分析Copilot的使用情况，Github目前通过接口的方式为管理员提供数据分析和度量服务。本文简单结束管理员如何获取当前用户的使用Copilot的情况。
     </p>
     <h4>
      2. 前提条件
     </h4>
     <p>
      如果需要分析企业组织中Copilot的情况，你必须符合以下几个先决条件。
     </p>
     <ol>
      <li>
       你需要具有Copilot企业管理员的权限，普通用户没有权限查看度量数据；
      </li>
      <li>
       你需要启用Copilot访问策略中的度量API，默认情况下这个设置是禁用状态；
      </li>
      <li>
       组织中需要有5名以上的活动用户，才能获取到度量数据；
      </li>
     </ol>
     <p>
      还需要注意：
     </p>
     <ul>
      <li>
       目前只有企业级用户才能获取到度量数据，个人用户不能获取度量数据
      </li>
      <li>
       最多可以获取最近28天的统计数据
      </li>
     </ul>
     <h4>
      3. 获取度量数据的方法
     </h4>
     <h5>
      3.1 启用Copilot的度量API
     </h5>
     <p>
      使用管理员账户登录Github，导航到企业设置页面，打开Copilot的访问策略，滚动到页面底部，并启用“Copilot Metrics API access”
      <a href="https://img2023.cnblogs.com/blog/608205/202503/608205-20250309105611147-1068113771.png" rel="nofollow">
      </a>
     </p>
     <div style="text-align: center;">
      <a href="https://img2023.cnblogs.com/blog/608205/202503/608205-20250309105611147-1068113771.png" rel="nofollow">

      </a>
     </div>
     <p>
     </p>
     <h5>
      3.2 创建用户访问令牌
     </h5>
     <p>
      访问令牌token是用户调用Copilot接口时提供的身份认证信息。
      <br/>
      需要在Github中创建一个经典(personal access token (classic))的令牌，并赋予该令牌读取账单信息(manage_billing:copilot)的权限。
      <a href="https://img2023.cnblogs.com/blog/608205/202503/608205-20250309105613175-1556889903.png" rel="nofollow">
      </a>
     </p>

     <h4>
      3.3 获取度量数据
     </h4>
     <ul>
      <li>
       <p>
        方法一(Octokit)：
        <br/>
        使用Octokit调用Github Copilot的接口。
        <strong>
         Octokit
        </strong>
        是一个用于与 GitHub API 交互的官方库集合，支持多种编程语言(如 JavaScript、Ruby、Python 等)。它提供了一种简单、高效的方式来访问和操作 GitHub 的数据和功能，例如仓库管理、问题跟踪、拉取请求、用户管理等。
        <br/>
        具体操作分为以下几步：
       </p>
       <ul>
        <li>
         需要安装Nodejs
        </li>
        <li>
         需要在Node中安装Octokit
        </li>
        <li>
         将下列脚本保存到本地，命名为copilot-metrics.js，并替换你的令牌和组织名称
         <a href="https://img2023.cnblogs.com/blog/608205/202503/608205-20250309105615693-704479422.png" rel="nofollow">
          <div style="text-align: center;">
           <img alt="image" border="0" height="660" src="https://i-blog.csdnimg.cn/img_convert/fa4661ec4c7b4931ce9bca13c172fea6.png" style="outline: none;" title="image" width="768">
           </img>
          </div>
         </a>
        </li>
        <li>
         在命令行中执行node copilot-metrics.js，即可获取到企业成员在最近28天内的使用情况
        </li>
       </ul>
      </li>
      <li>
       <p>
        方法二(postman)：
        <br/>
        如果你已经安装了postman，只需要创建一个请求即可
        <br/>
        在post的header中添加下面几个参数
       </p>
      </li>
      <li>
       <p>
        Accept： application/vnd.github+json
       </p>
      </li>
      <li>
       <p>
        Authorization：Bearer &lt;令牌&gt;
       </p>
      </li>
      <li>
       <p>
        X-GitHub-Api-Version： 2022-11-28
       </p>
      </li>
     </ul>
     <p>
      通过上面的方式，你可以获取到类似下面的统计结果：
      <a href="https://img2023.cnblogs.com/blog/608205/202503/608205-20250309105617125-1800673874.png" rel="nofollow">
