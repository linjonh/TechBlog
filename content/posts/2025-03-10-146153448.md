     c语言中的未定义行为
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <h4>
     定义（未定义的 Undefined Behavior，UB）
    </h4>
    <p>
     未定义行为是指在 C 语言标准中，
     <strong>
      标准未对某种行为规定具体结果
     </strong>
     ，因此
     <span style="color:#fe2c24">
      编译器可以自由决定如何处理该行为
     </span>
     。未定义行为的
     <span style="color:#fe2c24">
      本质
     </span>
     是
     <strong>
      结果无法预测
     </strong>
     ，即
     <strong>
      任何结果都可能发生
     </strong>
     。
    </p>
    <p>
    </p>
    <h4>
     种类
    </h4>
    <ol>
     <li>
      访问越界内存
     </li>
     <li>
      整型溢出（有符号整数）
     </li>
     <li>
      未初始化的局部变量
     </li>
     <li>
      修改并读取同一对象，且无序列点隔离
     </li>
     <li>
      非法类型转换
     </li>
     <li>
      <code>
       NULL
      </code>
      指针解引用
     </li>
     <li>
      函数
      <code>
       return
      </code>
      时漏掉返回值
     </li>
    </ol>
    <p>
    </p>
    <h4>
     <strong>
      如何避免
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       开启编译器警告
      </strong>
      （如
      <code>
       -Wall
      </code>
      、
      <code>
       -Wextra
      </code>
      等）
     </li>
     <li>
      <strong>
       使用代码静态分析工具
      </strong>
      （如
      <code>
       clang-analyzer
      </code>
      、
      <code>
       cppcheck
      </code>
      ）
     </li>
     <li>
      <strong>
       遵守良好的编码规范
      </strong>
      ，避免在同一表达式中对同一变量既读取又修改
     </li>
     <li>
      <strong>
       严格初始化变量
      </strong>
      ，尤其是指针和数组
     </li>
    </ol>
    <p>
    </p>
    <h4>
     补充
    </h4>
    <p>
     <strong>
      最容易忽视
     </strong>
     的未定义错误（修改并读取同一对象，且无序列点隔离）
    </p>
    <p>
     C 标准规定：
    </p>
    <blockquote>
     <p>
      <strong>
       在一次求值过程中，对同一对象的两次访问（其中至少一次是修改），且两者之间没有指定的求值顺序，结果是未定义行为。
      </strong>
     </p>
    </blockquote>
    <pre><code class="language-cpp">a[i++] += a[i];
</code></pre>
    <p>
     在
     <code>
      a[i++] += a[i]
     </code>
     中：
    </p>
    <ul>
     <li>
      <code>
       i
      </code>
      在
      <code>
       a[i++]
      </code>
      中被读取和修改
     </li>
     <li>
      <code>
       i
      </code>
      在
      <code>
       a[i]
      </code>
      中再次被读取
     </li>
     <li>
      两次访问
      <code>
       i
      </code>
      的顺序
      <strong>
       不确定
      </strong>
      ，这正是 UB 的根源
     </li>
    </ul>
