    <p>
     目前主流的开发模式，就两种：前后端分离 和 前后端不分离
    </p>
    <h3>
     <a id="_1">
     </a>
     前后端不分离
    </h3>
    <ul>
     <li>
      在前后端不分离的应用模式中，前端页面看到的效果都是由后端控制，由后端渲染页面或重定向，也就是后端需要控制前端的展示，前端与后端的耦合度很高。
     </li>
     <li>
      这种应用模式比较适合纯网页应用，但是当后端对接App时，App可能并不需要后端返回一个HTML网页，而仅仅是数据本身，所以后端原本返回网页的接口不再适用于前端App应用，为了对接App后端还需再开发一套接口。
     </li>
     <li>
      请求的数据交互如下图:
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fdc1595576dc75322ca7d0d2e36fdca3.png#pic_center"/>
     </li>
     <li>
      flask框架应用模板开发就是前后端不分离开发模式
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2ef54a723282d5768780845fc376e254.png#pic_center"/>
     </li>
     <li>
      前后端不分离特点
      <ul>
       <li>
        http请求次数少
       </li>
       <li>
        只需要一个后台服务器
       </li>
       <li>
        前后端开发耦合，责任不明确
       </li>
       <li>
        单纯开发网站，效率非常高
       </li>
       <li>
        响应的往往是html的页面
       </li>
      </ul>
     </li>
     <li>
     </li>
    </ul>
    <h3>
     <a id="_15">
     </a>
     前后端分离
    </h3>
    <ul>
     <li>
      在前后端分离的应用模式中，后端仅返回前端所需的数据，不再渲染HTML页面，不再控制前端的效果。至于前端用户看到什么效果，从后端请求的数据如何加载到前端中，都由前端自己决定，网页有网页的处理方式，App有App的处理方式，但无论哪种前端，所需的数据基本相同，后端仅需开发一套逻辑对外提供数据即可。
     </li>
     <li>
      在前后端分离的应用模式中 ，前端与后端的耦合度相对较低。
     </li>
     <li>
      在前后端分离的应用模式中，我们通常将后端开发的每个视图都称为一个接口，或者API，前端通过访问接口来对数据进行增删改查。
     </li>
     <li>
      对应的数据交互如下图 :
     </li>
     <li>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b7575b7895d308aeb49e8ce4c6aa879a.png#pic_center">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8fdf10adb9c5beb024fa3b09c119ab0f.png#pic_center"/>
      </img>
     </li>
     <li>
      特点：
      <ul>
       <li>
        有静态文件服务器、后台的应用服务器
       </li>
       <li>
        后台服务器只提供的接口的服务
       </li>
       <li>
        前后端责任分工明确
       </li>
       <li>
        一个后台即可满足网站、app、小程序等多种应用的需要
       </li>
       <li>
        响应的往往是json的数据
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="_29">
     </a>
     前后端分离的优点
    </h2>
    <ul>
     <li>
      彻底解放前端
      <ul>
       <li>
        前端不再需要向后台提供模板或是后台在前端html中嵌入后台代码
       </li>
      </ul>
     </li>
     <li>
      提高工作效率，分工更加明确
      <ul>
       <li>
        前后端分离的工作流程可以使前端只关注前端的事，后台只关心后台的活，两者开发可以同时进行，在后台还没有时间提供接口的时候，前端可以先将数据写死或者调用本地的json文件即可，页面的增加和路由的修改也不必再去麻烦后台，开发更加灵活。
       </li>
      </ul>
     </li>
     <li>
      局部性能提升
      <ul>
       <li>
        通过前端路由的配置，我们可以实现页面的按需加载，无需一开始加载首页便加载网站的所有的资源，服务器也不再需要解析前端页面，在页面交互及用户体验上有所提升。
       </li>
      </ul>
     </li>
     <li>
      降低维护成本
      <ul>
       <li>
        通过MVC框架，我们可以非常快速的定位及发现问题的所在，客户端的问题不再需要后台人员参与及调试，代码重构及可维护性增强。
       </li>
      </ul>
     </li>
    </ul>
