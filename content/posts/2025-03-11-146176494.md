    <h4>
     <strong>
      一、核心构建指令
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn clean
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：删除
         <code>
          target
         </code>
         目录（清理编译/打包生成的文件）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：确保从头开始构建，避免残留文件干扰。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn compile
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：编译项目源代码。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：快速检查代码是否能编译通过，无需打包。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn package
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：编译 + 运行测试 + 打包（生成 JAR/WAR 等）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：本地验证完整构建流程，生成可部署的包。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn install
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：编译 + 测试 + 打包，并将构建结果安装到本地仓库。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：本地多模块项目中，将子模块的构建结果提供给其他模块使用。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn deploy
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：将构建结果发布到远程仓库（如 Nexus、Artifactory）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：将稳定版本的依赖共享给团队或持续集成环境。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      二、依赖管理指令
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn dependency:tree
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：打印项目依赖树，显示所有传递性依赖。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：分析依赖冲突（如多个版本的库冲突），快速定位问题。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn dependency:analyze
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：检查未声明但被使用的依赖（或声明但未使用的依赖）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：优化
         <code>
          pom.xml
         </code>
         ，清理冗余依赖。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn versions:display-dependency-updates
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：扫描依赖是否有新版本可用。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：定期更新依赖版本，修复安全漏洞或使用新特性。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      三、测试相关指令
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn test
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：运行单元测试。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：验证代码逻辑是否符合预期。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn test -Dtest=TestClassName
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：运行指定测试类。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：调试单个测试用例，避免运行全部测试。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn test -DskipTests
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：跳过测试阶段。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：快速构建（如临时修复时不需要执行测试）。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      四、调试与优化指令
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn -X
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：开启 Maven 的调试日志（显示详细执行过程）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：排查构建失败的具体原因。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn -U
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：强制更新远程仓库的快照（Snapshot）依赖。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：确保使用最新的快照版本（如依赖频繁更新的开发中库）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn --offline
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：离线模式（仅使用本地仓库的依赖）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：网络不可用时继续构建。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      五、多模块项目指令
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn -pl module1,module2
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：仅构建指定的子模块（如
         <code>
          module1
         </code>
         和
         <code>
          module2
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：大型项目中快速构建部分模块，节省时间。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn -am -pl module1
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：构建指定模块及其依赖的其他模块。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：确保依赖的子模块已构建（如
         <code>
          module1
         </code>
         依赖
         <code>
          core
         </code>
         模块时自动构建
         <code>
          core
         </code>
         ）。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      六、高级场景
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         mvn help:effective-pom
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：生成合并所有父 POM 后的最终有效 POM。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：分析实际生效的配置（如插件、依赖版本）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn archetype:generate
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：使用 Maven 原型（模板）创建新项目。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：快速生成标准项目结构（如 Spring Boot、JavaFX 项目）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         mvn release:prepare
        </code>
        /
        <code>
         mvn release:perform
        </code>
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用
         </strong>
         ：自动化版本发布（更新版本号、打 Tag、发布到仓库）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          场景
         </strong>
         ：规范化的版本发布流程。
        </p>
       </li>
      </ul>
     </li>
    </ol>
