    <h2 style="margin-left:0px; margin-right:0px">
     4.1 SQL概述
    </h2>
    <h3 style="margin-left:0px; margin-right:0px">
     4.1.1 结构化查询语言SQL
    </h3>
    <p>
     SQL(
     <span style="background-color:yellow">
      Structured Query Language
     </span>
     )称为
     <span style="background-color:yellow">
      结构化查询语言
     </span>
     ，它是由1974年由Boyce和Chamberi提出的，1975年至1979年IBM公司的San Jose Research Laboratory研制了关系数据库管理系统的
     <span style="background-color:yellow">
      原型系统S
     </span>
     <span style="background-color:yellow">
      y
     </span>
     <span style="background-color:yellow">
      stem R
     </span>
     ,并实现了这种语1987年6月国际标准化组织采纳SQL为国际标准，1999年的SQL3拥有了更多的特征，例如递归、触发器以及对象等，它支持对象-关系数据模型
    </p>
    <p style="margin-left:0; margin-right:0">
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     4.1.2 SQL的语言特点
    </h3>
    <ul>
     <li>
      综合统一
     </li>
     <li>
      高度
      <span style="background-color:#ffd900">
       非过程化
      </span>
      <ul>
       <li>
        用SQL语言进行数据操作，用户只需提出“做什么”，而不必指明“怎么做”，因此用户无须了解存取路径，存取路径的选择以及SQL语句的操作过程由系统自动完成。这不但大大减轻了用户负担，而且有利于提高数据独立性
       </li>
      </ul>
     </li>
     <li>
      <span style="background-color:#ffd900">
       面向集合
      </span>
      的操作方式
     </li>
     <li>
      灵活的使用方法
     </li>
     <li>
      语言简洁，易学易用，功能强
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0">
    </p>
    <h3 style="margin-left:0px; margin-right:0px">
     4.1.3 SQL的数据类型
    </h3>
    <ul>
     <li>
      预定义数据类型
      <ul>
       <li>
        数值型：INT、SMALLINT、REAL、DOUBLE PRECSION、FLOAT、NUMERIC
       </li>
       <li>
        字符串型：CHAR 、VAR CHAR
       </li>
       <li>
        位串型:BIT 、BIT VARRING
       </li>
       <li>
        时间型:DATA 、TIME 、TIMESTAMP
       </li>
       <li>
        布尔型:BOOLEAN
       </li>
      </ul>
     </li>
     <li>
      构造数据类型
     </li>
     <li>
      用户定义数据类型
     </li>
     <li>
      大对象类型
     </li>
    </ul>
    <h3 style="margin-left:0px; margin-right:0px">
    </h3>
    <h3 style="margin-left:0px; margin-right:0px">
     4.1.4 SQL语言的组成和语句类型
    </h3>
    <p>
     1. SQL语言的组成：
    </p>
    <ul>
     <li>
      数据定义语言(DDL)：用来创建数据库的各种对象，包括数据库模式、表、视图、索引、域、触发器、自定义类型等。
     </li>
     <li>
      数据操纵语言(DML)
     </li>
     <li>
      数据控制语言(DCL)
     </li>
    </ul>
    <p>
     2. SQL语句类型：
    </p>
    <ul>
     <li>
      SQL
      <span style="background-color:yellow">
       定义
      </span>
      语句:CREATE(创建表)、DROP(删除表)和ALTER(更新表)
     </li>
     <li>
      SQL数据
      <span style="background-color:yellow">
       操纵
      </span>
      语句：select、insert、delete、updata
     </li>
     <li>
      SQL事务和
      <span style="background-color:yellow">
       控制
      </span>
      语句：grant、revoke
     </li>
     <li>
      SQL连接、会话和诊断语句：set connection
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0">
    </p>

    <h2 style="margin-left:0px; margin-right:0px">
     4.5 SQL的视图
    </h2>
    <ul>
     <li>
      视图是关系数据库系统提供给用户以多种角度观察数据库中数据的重要机制
      <ul>
       <li>
        视图是使用SELECT FROM语句从一个或多个基本表(或其他视图)中导出的表，它与基本表不同，是一个
        <span style="background-color:yellow">
         虚表
        </span>
        ；对视图的一切操作最终将转换成对导出它的
        <span style="background-color:yellow">
         基本表的操作
        </span>
        。视图一经定义就可以和基本表一样被查询，也可以在一个视图上
        <span style="background-color:yellow">
         再定义新的视图
        </span>
        ，但对视图的修改(插入、删除、更新)操作有一定的限制。
       </li>
      </ul>
     </li>
    </ul>
    <div>
     <table cellspacing="0" style="margin-left:.7083in">
      <tbody>
       <tr>
        <td style="border-color:#a3a3a3; vertical-align:top; width:6.8291in">
         <p style="margin-left:0; margin-right:0">
          <img alt="" height="174" src="https://i-blog.csdnimg.cn/direct/981472eeb03740d9b4ed5408a4267b64.png" width="600"/>
         </p>
         <p style="margin-left:0; margin-right:0">
         </p>
        </td>

       <tr>
        <td style="border-color:#a3a3a3; vertical-align:top; width:6.9138in">
         <p style="margin-left:0; margin-right:0">
         </p>
         <ul>
          <li>
           行列子集视图:若一个视图是
           <span style="background-color:yellow">
            从单个基本表导出
           </span>
           ，并且只是去掉了基本表的某些行和某些列，但保留了主码
          </li>
          <li>
           带表达式的视图:若一个视图带有
           <span style="background-color:yellow">
            由基本数据经过各种计算
           </span>
           派生出的虚拟列。
          </li>
          <li>
           分组视图:若一个视图在创建它的SELECT语句中使用了
           <span style="background-color:yellow">
            聚集函数和GROUP BY子句
           </span>
           。
          </li>
          <li>
           连接视图:若一个视图在创建它的SELECT语句中使用了
           <span style="background-color:yellow">
            两个或多个表连接
           </span>
           。
          </li>
         </ul>
