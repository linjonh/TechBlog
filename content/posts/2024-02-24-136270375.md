    <h3 style="text-align:justify;">
     <strong>
      <strong>
       一、openstack介绍
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt;text-align:justify;">
     Openstack是一个开源的云计算管理平台项目，由几个主要的组件组合起来完成具体工作，支持几乎所有的云环境，项目目标是提供实施简单、可大规模扩展、丰富、标准统一的云计算管理平台 ----百度百科
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Openstack是一个云操作系统，它控制着整个数据中心的大量计算、存储和网络资源，所有资源通过面板管理，让管理员可通过web界面来提供资源     --官网
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="474" src="https://i-blog.csdnimg.cn/blog_migrate/da65b40f784ef4d11c4421d1078b8b57.png" width="1200"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     裸金属：在openstack中Ironic就是一个进行裸机部署安装项目组件，它的功能是在一台或多台裸机上同时部署，执行磁盘分区格式化/安装系统/安装应用等操作（方便、省时）
    </p>
    <h3 style="text-align:justify;">
     <strong>
      <strong>
       二、openstack的版本及组件
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       版本
      </strong>
     </strong>
     ：从官网获得最新的版本为Q(queens)版，我们使用的是O(Ocata)版
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       组件介绍：
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="988" src="https://i-blog.csdnimg.cn/blog_migrate/fc12400a5cfaa37fd20daee5225e3505.png" width="1070"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     基于虚机的各种服务：
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Neutron：提供虚机网络连接
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Cinder：提供块存储，但不提供文件系统
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Nova：管理虚机的生命周期（创建、重启、关机等）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Glance：提供/管理镜像服务 (在云平台中，各种版本的操作系统已提供，选择即可)
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Swift：提供对象存储，用来存放镜像
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      注：在生产环境中，虚机有很多盘由Cinder提供，如果虚机删除，存储盘还在
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      备份：将cinder中的备份数据存放在swift中
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Horizon：提供web用户登录界面（用来操作管理各种服务）
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Keystone：认证服务(比如Horizon登录的账号，连接每个服务，通过keystone与每个服务进行认证)
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Cellometer：收集信息服务（类似于监控，收集用户使用资源的情况从而进行收费）
    </p>
    <h3 style="text-align:justify;">
     <strong>
      <strong>
       三、Openstack的环境搭建
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       集群环境中，每个机器都有一个逻辑名称（根据业务来调整节点的数量）
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="540" src="https://i-blog.csdnimg.cn/blog_migrate/d0e81b993104551919f3519e81a30d86.png" width="918"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#0000ff;">
       <strong>
        为了使控制节点降低使用资源的负载从而新建个base节点，使原本应在控制节点上安装的服务安装在base节点上
       </strong>
      </span>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       生产环境中，
      </strong>
     </strong>
     <strong>
      <u>
       <strong>
        <u>
         控制节点一般为3到7个，内存不得小于8G
        </u>
       </strong>
      </u>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       例：铁通云有3个控制节点可以管理800台物理机和10000台虚拟机
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <h3 style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       创建控制节点/base节点实验：
      </strong>
     </strong>
    </h3>
    <h4 style="margin-left:.0001pt;text-align:justify;">
     1.前期准备
    </h4>
    <p style="margin-left:.0001pt;text-align:justify;">
     创建虚机：克隆快速创建多台虚机
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#0000ff;">
       <strong>
        virt-clone
       </strong>
      </span>
     </strong>
     <span style="color:#0000ff;">
      -o abc123 -n efg456 -f /var/lib/libvirt/images/efg456.qcow2
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     或者图形化界面克隆
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0000ff;">
      -o：原虚机  -n :新虚机  -f :新虚机镜像
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <span style="color:#ff0000;">
       <strong>
        配置每台虚机的IP、网关、dns、主机名、防火墙、selinux、yum源（七要素）
       </strong>
      </span>
     </strong>
    </p>
    <h4 style="margin-left:.0001pt;text-align:justify;">
     <strong>
      <strong>
       2.此处在物理机建立bro桥，将所有的节点加入到桥
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt;text-align:justify;">
     （1）ntp服务
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     /etc/ntp.conf
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="384" src="https://i-blog.csdnimg.cn/blog_migrate/74279a902f26132e9ed62ce3ad6b5aff.png" width="814"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     systemctl enable ntpd
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     Systemctl restart ntpd
    </p>
    <p>
     （2）DNS服务
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     /etc/named.conf
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="312" src="https://i-blog.csdnimg.cn/blog_migrate/b3d43b1f8069390cea5d1248c050300a.png" width="804"/>
    </p>
    <p style="margin-left:.0001pt;text-align:left;">
     <span style="color:#ff0000;">
      重点注意红框内的配置！
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:left;">
     /var/named/zheng
    </p>
    <p style="margin-left:.0001pt;text-align:left;">
     <img alt="" height="282" src="https://i-blog.csdnimg.cn/blog_migrate/af3977a6cf82d58cd965817e4b1ffdbc.png" width="824"/>
    </p>
    <p style="margin-left:.0001pt;text-align:left;">
     systemctl enable named
    </p>
    <p style="margin-left:.0001pt;text-align:left;">
     Systemctl restart named
    </p>
    <p>
     （3）Mariadb
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      安装mariadb mariadb-server  python2-PyMySQL
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     /etc/my.cnf.d/server.cnf
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="396" src="https://i-blog.csdnimg.cn/blog_migrate/d012227d786e87d82f4e97746b2502a8.png" width="680"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     初始化数据库：mysql_secure_installation
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     登录测试：mysql -uroot -p
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     systemctl enable mariadb
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     systemctl restart mariadb
    </p>
    <p>
     （4）rabbitmq
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     添加用户、密码：
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      rabbitmqctl add_user openstack RABBIT_PASS

    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     设置权限：
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      rabbitmqctl set_permissions openstack

      “
     </span>
     <span style="color:#0006f2;">
      .*

     <span style="color:#0006f2;">
      ”

     <span style="color:#0006f2;">
      “
     </span>
     <span style="color:#0006f2;">
      .*
     </span>
     <span style="color:#0006f2;">
      ”
     </span>
     <span style="color:#0006f2;">
      “
     </span>
     <span style="color:#0006f2;">
      .*
     </span>
     <span style="color:#0006f2;">
      ”
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <img alt="" height="144" src="https://i-blog.csdnimg.cn/blog_migrate/060d627034357cddeb32b261a4d49b38.png" width="1200"/>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     设置web界面：
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      rabbitmq-plugins enable rabbitmq_management
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">

     设置开启自启/重启：

    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      systemctl enable rabbitmq-server
     </span>
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
     <span style="color:#0006f2;">
      systemctl restart rabbitmq-server
