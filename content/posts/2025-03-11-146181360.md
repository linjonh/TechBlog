    <h3>
     <a id="___0">
     </a>
     第一章 数据结构绪论
    </h3>
    <p>
     <strong>
      数据
     </strong>
     <br/>
     <strong>
      数据对象
     </strong>
     ：性质相同的数据元素的集合，数据的子集
     <br/>
     <strong>
      数据元素
     </strong>
     ：人
     <br/>
     <strong>
      数据项
     </strong>
     ：眼、耳、鼻、嘴、手、脚等不可分割的项
    </p>
    <p>
     <strong>
      数据结构
     </strong>
     ：存在特定关系（搭配和排列）的数据元素的集合
    </p>
    <hr/>
    <p>
     <strong>
      逻辑结构
     </strong>
    </p>
    <ol>
     <li>
      集合结构：元素之间没有关系
     </li>
     <li>
      线性结构：元素之间一对一关系（兄弟排行）
     </li>
     <li>
      树形结构：元素之间一对多关系（父子）
     </li>
     <li>
      图形结构：元素之间多对多关系（好朋友）
     </li>
    </ol>
    <hr/>
    <p>
     <strong>
      物理结构/存储结构：逻辑结构的存储形式
     </strong>
    </p>
    <ol>
     <li>
      顺序存储（数组）
     </li>
     <li>
      链式存储（取号）：需要指针
     </li>
    </ol>
    <hr/>
    <p>
     <strong>
      逻辑结构面向问题，物理结构面向计算机
     </strong>
    </p>
    <p>
     <strong>
      数据类型：集合+操作
     </strong>
    </p>
    <ol>
     <li>
      原子类型（整型、实型、字符型）
     </li>
     <li>
      结构类型（整形数组由若干整型数据组成）
     </li>
    </ol>
    <hr/>
    <p>
     <strong>
      抽象数据类型 (Abstract Data Type, ADT)
     </strong>
     ：数学模型 + 一组操作
     <br/>
     意义：数学抽象
     <br/>
     特性：问题分解、抽象、信息隐藏
    </p>
    <p>
     标准格式：
    </p>
    <pre><code class="prism language-c">	ADT	抽象数据类型名
	Data
		数据元素之间逻辑关系的定义
	Operation
		操作 <span class="token number">1</span>
			初始条件
			操作结果描述
		操作 <span class="token number">2</span>
			……
		操作 n
			……
	endADT
</code></pre>
