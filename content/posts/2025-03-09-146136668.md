    <h3>
     <a id="_1">
     </a>
     题目描述
    </h3>
    <p>
     给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
    </p>
    <p>
     请你设计并实现时间复杂度为 O(n) 的算法解决此问题。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/327c84228f9741f8bb0ad47e2b2ddbaf.png"/>
    </p>
    <h3>
     <a id="_6">
     </a>
     思路
    </h3>
    <p>
     常规的思路可能是排序后遍历，找到最长的连续段。这样的话，时间复杂度是O(n log n)，但题目要求O(n)，所以这种方法不可行。
     <br/>
     那O(n)的算法就需要用哈希表了。
    </p>
    <p>
     可以先把所有数字存入一个哈希集合中，这样可以快速查询某个数是否存在。然后遍历数组中的每个数，对于每个数，如果它的前一个数（num-1）不在集合中，那么就以这个数为起点，开始向后查找连续的数，比如num+1、num+2等等，直到找不到为止。
     <br/>
     这样每个连续序列只会被处理一次，因为只有当当前数是序列的起点时才会进行遍历。
    </p>
    <p>
     例如，假设数组中有4，那么如果3存在的话，4就不是起点，所以不会被处理。只有当处理到1的时候，因为没有0存在，所以开始从1向上找，得到1、2、3、4，长度为4。这样可以确保每个元素最多被访问两次（一次是放入集合，另一次是在查找序列时），因此时间复杂度是O(n)。
    </p>
    <h3>
     <a id="_15">
     </a>
     步骤
    </h3>
    <ol>
     <li>
      <p>
       将数组中的所有元素存入一个哈希集合，查询时间O(1)。
      </p>
     </li>
     <li>
      <p>
       初始化最长长度max_length为0。
      </p>
     </li>
     <li>
      <p>
       遍历数组中的每一个元素num：
       <br/>
       a. 如果num-1不在集合中，那么当前num可能是一个连续序列的起点。
       <br/>
       b. 然后从num开始，逐步检查num+1、num+2是否在集合中，直到找不到为止。
       <br/>
       c. 计算当前连续序列的长度，并更新max_length。
      </p>
     </li>
     <li>
      <p>
       返回max_length。
      </p>
     </li>
    </ol>
    <p>
     每个元素最多被访问两次，所以总的时间复杂度是O(n)。
    </p>

    <h3>
     <a id="_61">
     </a>
     代码详解
    </h3>
    <p>
     1.
     <code>
      unordered_set&lt;int&gt; num_set(nums.begin(), nums.end())
     </code>
     将数组
     <code>
      nums
     </code>
     中的所有元素复制到一个哈希集合
     <code>
      num_set
     </code>
     中，用于快速构建一个哈希集合（HashSet）
    </p>
    <p>
     <code>
      nums
     </code>
     是一个
     <code>
      vector&lt;int&gt;
     </code>
     数组
     <br/>
     <code>
      nums.begin()
     </code>
     返回指向数组第一个元素的迭代器（指针）
     <br/>
     <code>
      nums.end()
     </code>
     返回指向数组末尾（最后一个元素的下一个位置）的迭代器
     <br/>
     合起来表示将 nums 的全部元素作为输入范围。
    </p>
    <p>
     <code>
      unordered_set
     </code>
     的构造函数接受两个迭代器（
     <code>
      begin
     </code>
     和
     <code>
      end
     </code>
     ），将范围内的所有元素插入集合。
     <br/>
     且插入时会
     <strong>
      自动去重
     </strong>
     ：例如，如果 nums 中有多个 5，集合中只会保留一个 5。
    </p>
    <p>
     有小白可能把
     <code>
      unordered_set
     </code>
     和
     <code>
      unordered_map
     </code>
     弄混淆，前者是哈希集合，后者是哈希表。以下是区分：
    </p>
    <table>
     <thead>
      <tr>
       <th align="center">
        容器
       </th>
       <th align="center">
        用途
       </th>
       <th align="center">
        模板参数示例
       </th>
       <th align="center">
        适用场景
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="center">
        <code>
         unordered_set&lt;T&gt;
        </code>
       </td>
       <td align="center">
        存储单一类型的元素，快速判断存在性
       </td>
       <td align="center">
        <code>
         unordered_set&lt;int&gt;
        </code>
       </td>
       <td align="center">
        只需判断元素是否存在
       </td>
      </tr>
      <tr>
       <td align="center">
        <code>
         unordered_map&lt;K,V&gt;
        </code>
       </td>
       <td align="center">
        存储键值对，关联数据
       </td>
       <td align="center">
        <code>
         unordered_map&lt;string, int&gt;
        </code>
       </td>
       <td align="center">
        需要记录键的附加信息（如计数）
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     在本题中，
     <code>
      unordered_set&lt;int&gt;
     </code>
     是最优选择，而
     <code>
      unordered_map&lt;int, int&gt;
     </code>
     既不必要，又浪费资源。
    </p>
    <p>
     2.
     <code>
      for (int num : nums)
     </code>
     范围 for 循环，遍历容器（如数组、vector、list 等）中的每一个元素，可代替
     <code>
      for (int i = 0; i &lt; nums.size(); ++i)
     </code>
    </p>
    <p>
     <code>
      int num
     </code>
     ：定义一个临时变量
     <code>
      num
     </code>
     ，类型为
     <code>
      int
     </code>
     （与容器元素类型一致）。
     <br/>
     <code>
      nums
     </code>
     ：被遍历的容器（如
     <code>
      vector&lt;int&gt;
     </code>
     或数组）。
     <br/>
     循环过程：每次迭代时，将容器中的一个元素赋值给
     <code>
      num
     </code>
     ，并执行循环体内的代码。
    </p>
    <p>
     3.
     <code>
      if (num_set.find(num - 1) == num_set.end())
     </code>
    </p>
    <p>
     <code>
      end()
     </code>
     是
     <code>
      unordered_set
     </code>
     的成员函数，返回指向集合末尾的迭代器（表示
     <strong>
      无效
     </strong>
     位置）
     <br/>
     如果
     <code>
      find()
     </code>
     返回的迭代器等于
     <code>
      end()
     </code>
     ，说明未找到
     <code>
      num - 1
     </code>
    </p>

    <h3>
     <a id="_96">
     </a>
     注意
    </h3>
    <p>
     1.
     <code>
      for (int num : nums)
     </code>
     范围 for 循环，可代替
     <code>
      for (int i = 0; i &lt; nums.size(); ++i)
     </code>
     <br/>
     范围 for 循环本质上是 语法糖，编译器会将其转换为传统的迭代器循环
    </p>
    <p>
     2.外层循环的遍历对象一定要是哈希集合
     <code>
      num_set
     </code>
     而不能是原数组
     <code>
      nums
     </code>
     ，否则会出现：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c634b7d1f59a40e58cf2706bf8e0855c.png"/>
    </p>
    <p>
     <strong>
      原因：
     </strong>
    </p>
    <p>
     当输入数组包含 大量重复元素 时，外层循环次数会远高于实际需要处理的元素数量。
    </p>
    <p>
     例如：输入 nums = [0, 0, 0, 0, …]（含无数个元素）。
     <br/>
     哈希集合
     <code>
      num_set
     </code>
     中实际只有 1 个元素 0，外层循环仍会执行 1e5 次，但每次处理的逻辑相同，浪费大量时间。
    </p>

     因此：
    </p>
