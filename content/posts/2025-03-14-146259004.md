    <h3>
     <a id="HTTP__5">
     </a>
     <strong>
      一、HTTP 字段（请求头与响应头）
     </strong>
    </h3>
    <p>
     HTTP 头字段用于传递客户端和服务器之间的元数据，分为
     <strong>
      请求头（Request Headers）
     </strong>
     和
     <strong>
      响应头（Response Headers）
     </strong>
     。
    </p>
    <hr/>
    <h4>
     <a id="1__10">
     </a>
     <strong>
      1. 常见请求头字段
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        字段名
       </th>
       <th>
        说明
       </th>
       <th>
        示例值
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         Host
        </code>
       </td>
       <td>
        目标服务器的主机名和端口号（必填字段）。
       </td>
       <td>
        <code>
         Host: example.com:8080
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         User-Agent
        </code>
       </td>
       <td>
        客户端标识（浏览器、操作系统或应用程序信息）。
       </td>
       <td>
        <code>
         User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Accept
        </code>
       </td>
       <td>
        客户端支持的响应内容类型（MIME 类型）。
       </td>
       <td>
        <code>
         Accept: application/json, text/html
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Accept-Encoding
        </code>
       </td>
       <td>
        支持的压缩编码。
       </td>
       <td>
        <code>
         Accept-Encoding: gzip, deflate
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Content-Type
        </code>
       </td>
       <td>
        请求体的媒体类型。
       </td>
       <td>
        <code>
         Content-Type: application/json
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Authorization
        </code>
       </td>
       <td>
        身份凭证（如 JWT Token）。
       </td>
       <td>
        <code>
         Authorization: Bearer eyJhbGciOiJ...
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Cookie
        </code>
       </td>
       <td>
        客户端发送的 Cookie 信息。
       </td>
       <td>
        <code>
         Cookie: session_id=abc123; user=admin
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Cache-Control
        </code>
       </td>
       <td>
        缓存策略。
       </td>
       <td>
        <code>
         Cache-Control: no-cache
        </code>
        （禁用缓存）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Referer
        </code>
       </td>
       <td>
        当前请求的来源页面 URL。
       </td>
       <td>
        <code>
         Referer: https://google.com
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Accept-Charset
        </code>
       </td>
       <td>
        声明客户端支持的字符集及其优先级（如 UTF-8、ISO-8859）。现代浏览器通常默认支持 UTF-8，此字段已较少显式使用。
       </td>
       <td>
        <code>
         Accept-Charset: utf-8, iso-8859-1; q=0.5
        </code>
        （q 值表示权重）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Accept-Language
        </code>
       </td>
       <td>
        指定客户端期望的自然语言（如中文、英文）。服务端根据优先级返回多语言内容。
       </td>
       <td>
        <code>
         Accept-Language: zh-CN, en-US; q=0.8
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Cookie
        </code>
       </td>
       <td>
        客户端向服务器发送已存储的 Cookie 数据。
       </td>
       <td>
        <code>
         Cookie: session_id=abc123; user=admin
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         If-Modified-Since
        </code>
       </td>
       <td>
        资源在指定时间后未修改则返回 304 状态码，使用本地缓存。
        <strong>
         关联字段
        </strong>
        ：
        <code>
         Last-Modified
        </code>
        （响应头）。
       </td>
       <td>
        <code>
         If-Modified-Since: Sat, 29 Oct 1994 19:43:31 GMT
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         If-None-Match
        </code>
       </td>
       <td>
        携带资源的
        <code>
         ETag
        </code>
        值，若未变化则返回 304。比时间戳更精确，避免时间同步问题。
       </td>
       <td>
        <code>
         If-None-Match: "737060cd8c284d8af7ad3082f209582d"
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         X-Forwarded-For
        </code>
       </td>
       <td>
        标识客户端的原始 IP（常用于代理链中记录真实来源）。非标准字段，需代理服务器显式添加。
       </td>
       <td>
        <code>
         X-Forwarded-For: 192.168.1.1, 10.0.0.1
        </code>
        （第一个 IP 为客户端）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Priority
        </code>
       </td>
       <td>
        提示服务器请求的优先级（如资源加载顺序）。优化页面加载性能，需浏览器和服务端共同支持（当前支持度有限）。在 HTTP/2 和 HTTP/3 中，优先级机制通过专门的
        <code>
         PRIORITY_UPDATE
        </code>
        帧实现，而非头字段。
       </td>
       <td>
        <code>
         Priority: u=1, i
        </code>
        （表示高优先级）
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <a id="2__32">
     </a>
     <strong>
      2. 常见响应头字段
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        字段名
       </th>
       <th>
        说明
       </th>
       <th>
        示例值
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         Content-Type
        </code>
       </td>
       <td>
        响应体的媒体类型及编码。
       </td>
       <td>
        <code>
         Content-Type: text/html; charset=UTF-8
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Content-Encoding
        </code>
       </td>
       <td>
        响应体的压缩方式。
       </td>
       <td>
        <code>
         Content-Encoding: gzip
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Server
        </code>
       </td>
       <td>
        服务器软件信息。
       </td>
       <td>
        <code>
         Server: Apache/2.4.1
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Set-Cookie
        </code>
       </td>
       <td>
        服务器设置的 Cookie。
       </td>
       <td>
        <code>
         Set-Cookie: session_id=def456; Path=/; HttpOnly
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Location
        </code>
       </td>
       <td>
        重定向目标 URL（用于 3xx 状态码）。
       </td>
       <td>
        <code>
         Location: https://new-example.com
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Cache-Control
        </code>
       </td>
       <td>
        资源缓存策略。
       </td>
       <td>
        <code>
         Cache-Control: public, max-age=3600
        </code>
        （缓存 1 小时）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Access-Control-Allow-Origin
        </code>
       </td>
       <td>
        允许跨域请求的源。
       </td>
       <td>
        <code>
         Access-Control-Allow-Origin: *
        </code>
        （允许所有域名）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         ETag
        </code>
       </td>
       <td>
        资源的唯一标识符（用于缓存验证）。
       </td>
       <td>
        <code>
         ETag: "12345abcde"
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         WWW-Authenticate
        </code>
       </td>
       <td>
        定义身份验证方式（如 Basic、Bearer Token）。配合 401 状态码要求客户端提供凭证。
       </td>
       <td>
        <code>
         WWW-Authenticate: Basic realm="Access to site"
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Last-Modified
        </code>
       </td>
       <td>
        资源最后修改时间，用于缓存验证。
        <strong>
         关联请求头
        </strong>
        ：
        <code>
         If-Modified-Since
        </code>
        。
       </td>
       <td>
        <code>
         Last-Modified: Tue, 15 Nov 1994 08:12:31 GMT
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         Expires
        </code>
       </td>
       <td>
        指定资源过期时间（HTTP/1.0 缓存机制）。已被
        <code>
         Cache-Control
        </code>
        的
        <code>
         max-age
        </code>
        取代，优先级更低。
       </td>
       <td>
        <code>
         Expires: Thu, 01 Dec 2025 16:00:00 GMT
        </code>
       </td>
      </tr>
     </tbody>
    </table>

    <h3>
     <a id="HTTP__86">
     </a>
     <strong>
      二、HTTP 状态码详解
     </strong>
    </h3>
    <p>
     HTTP 状态码由三位数字组成，分为五类：
    </p>
    <hr/>
    <h4>
     <a id="1_1xx_91">
     </a>
     <strong>
      1. 1xx（信息响应）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       100 Continue
      </strong>
      <br/>
      客户端应继续发送请求体（用于大文件上传前的确认）。
     </li>
     <li>
      <strong>
       101 Switching Protocols
      </strong>
      <br/>
      服务器同意切换协议（如从 HTTP 切换到 WebSocket）。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_2xx_99">
     </a>
     <strong>
      2. 2xx（成功）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       200 OK
      </strong>
      <br/>
      请求成功，返回预期结果。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 200 OK
Content-Type: text/html
&lt;html&gt;...&lt;/html&gt;
</code></pre>
    <ul>
     <li>
      <strong>
       201 Created
      </strong>
      <br/>
      资源已创建（常用于 POST 请求）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 201 Created
Location: /api/users/789
{"id": 789, "message": "User created"}
</code></pre>
    <ul>
     <li>
      <strong>
       204 No Content
      </strong>
      <br/>
      请求成功，但无返回内容（如 DELETE 请求）。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_3xx_121">
     </a>
     <strong>
      3. 3xx（重定向）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       301 Moved Permanently
      </strong>
      <br/>
      资源永久迁移到新 URL（SEO 友好）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 301 Moved Permanently
Location: https://new-example.com
</code></pre>
    <ul>
     <li>
      <strong>
       302 Found
      </strong>
      <br/>
      资源临时重定向（浏览器下次仍请求原 URL）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 302 Found
Location: /temp-page
</code></pre>
    <ul>
     <li>
      <strong>
       304 Not Modified
      </strong>
      <br/>
      资源未修改，客户端使用本地缓存。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 304 Not Modified
ETag: "12345abcde"
</code></pre>
    <hr/>
    <h4>
     <a id="4_4xx_145">
     </a>
     <strong>
      4. 4xx（客户端错误）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       400 Bad Request
      </strong>
      <br/>
      请求格式错误（如缺少必填字段）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 400 Bad Request
{"error": "Missing 'email' field"}
</code></pre>
    <ul>
     <li>
      <strong>
       401 Unauthorized
      </strong>
      <br/>
      需要身份验证（未提供有效凭证）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm="example.com"
</code></pre>
    <ul>
     <li>
      <strong>
       403 Forbidden
      </strong>
      <br/>
      服务器拒绝访问（权限不足）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 403 Forbidden
{"error": "Access denied"}
</code></pre>
    <ul>
     <li>
      <strong>
       404 Not Found
      </strong>
      <br/>
      请求的资源不存在。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 404 Not Found
&lt;h1&gt;404 Page Not Found&lt;/h1&gt;
</code></pre>
    <hr/>
    <h4>
     <a id="5_5xx_176">
     </a>
     <strong>
      5. 5xx（服务器错误）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       500 Internal Server Error
      </strong>
      <br/>
      服务器内部错误（通用错误码）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 500 Internal Server Error
{"error": "Database connection failed"}
</code></pre>
    <ul>
     <li>
      <strong>
       502 Bad Gateway
      </strong>
      <br/>
      代理服务器从上游服务器收到无效响应。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 502 Bad Gateway
</code></pre>
    <ul>
     <li>
      <strong>
       503 Service Unavailable
      </strong>
      <br/>
      服务器暂时不可用（维护或过载）。
     </li>
    </ul>
    <pre><code class="prism language-http">HTTP/1.1 503 Service Unavailable
Retry-After: 3600
</code></pre>
    <hr/>
    <h3>
     <a id="_199">
     </a>
     <strong>
      三、关键总结与最佳实践
     </strong>
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        HTTP 字段
       </strong>
      </p>
      <ul>
       <li>
        使用
        <code>
         Cache-Control
        </code>
        和
        <code>
         ETag
        </code>
        优化缓存性能。
       </li>
       <li>
        跨域请求时配置
        <code>
         Access-Control-Allow-Origin
        </code>
        。
       </li>
       <li>
        敏感信息通过
        <code>
         Authorization
        </code>
        头传递，而非 URL 参数。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        状态码使用原则
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         精准匹配场景
        </strong>
        ：避免滥用
        <code>
         200
        </code>
        或
        <code>
         500
        </code>
        。
        <ul>
         <li>
          资源不存在 →
          <code>
           404
          </code>
         </li>
         <li>
          身份未认证 →
          <code>
           401
          </code>
         </li>
         <li>
          权限不足 →
          <code>
           403
          </code>
         </li>
        </ul>
       </li>
       <li>
        <strong>
         重定向区分
        </strong>
        ：
        <ul>
         <li>
          永久迁移 →
          <code>
           301
          </code>
         </li>
         <li>
          临时跳转 →
          <code>
           302
          </code>
         </li>
        </ul>
       </li>
       <li>
        <strong>
         客户端错误
        </strong>
        ：提供清晰的错误描述（如
        <code>
         400
        </code>
        时指明缺失字段）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        安全建议
       </strong>
      </p>
      <ul>
       <li>
        Cookie 设置
        <code>
         HttpOnly
        </code>
        和
        <code>
         Secure
        </code>
        属性。
       </li>
       <li>
        使用 HTTPS 加密传输敏感数据。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
