    <p>
     在当今互联网时代，确保网站的可访问性（Accessibility）已经成为开发者和设计师的重要任务之一。Web可访问性不仅有助于残障用户更好地访问和使用网站，还能提升整体用户体验。本文将介绍10个关键的Web可访问性实践，帮助你构建更加友好和包容的网站。
    </p>
    <h3>
     <a id="1_HTML_2">
     </a>
     1. 使用语义化的HTML标签
    </h3>
    <p>
     语义化的HTML标签不仅有助于搜索引擎优化（SEO），还能提升屏幕阅读器等辅助技术的理解能力。以下是一些常用的语义化标签：
    </p>
    <ul>
     <li>
      <code>
       &lt;header&gt;
      </code>
      ：用于页眉内容。
     </li>
     <li>
      <code>
       &lt;nav&gt;
      </code>
      ：用于导航栏。
     </li>
     <li>
      <code>
       &lt;main&gt;
      </code>
      ：用于页面的主要内容区域。
     </li>
     <li>
      <code>
       &lt;section&gt;
      </code>
      ：用于划分页面的不同部分。
     </li>
     <li>
      <code>
       &lt;footer&gt;
      </code>
      ：用于页脚内容。
     </li>
    </ul>
    <p>
     列表应使用
     <code>
      &lt;ul&gt;
     </code>
     和
     <code>
      &lt;li&gt;
     </code>
     标签，对screen reader 更友好。
    </p>

    <h3>
     <a id="2__21">
     </a>
     2. 合理使用标题标签
    </h3>
    <p>
     一个页面中应只使用一个
     <code>
      &lt;h1&gt;
     </code>
     标签，用于表示页面的主标题。标题层级应连续，不要跳过层级。例如：
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>h1</span><span class="token punctuation">&gt;</span></span>Main Title<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>h1</span><span class="token punctuation">&gt;</span></span>

</code></pre>
    <h3>
     <a id="3__29">
     </a>
     3. 为图片和链接添加替代文本
    </h3>
    <p>
     为图片添加alt属性，以便在图片无法加载时提供替代文本。
    </p>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>img</span> <span class="token attr-name">src</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>logo.png<span class="token punctuation">"</span></span> <span class="token attr-name">alt</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>Company Logo<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>

    <h3>
     <a id="4__35">
     </a>
     4. 确保颜色对比度
    </h3>
    <p>
     颜色对比度是确保文本内容可读性的关键。使用工具（如WebAIM的对比度检查器）来确保文本与背景之间的对比度符合WCAG标准（至少4.5:1）。
    </p>
    <h3>
     <a id="5__37">
     </a>
     5. 使用相对单位设置字体大小
    </h3>
    <p>
     避免使用px作为字体大小的单位，推荐使用rem或em。这样，字体大小可以根据用户的浏览器设置进行调整，提升可访问性。
    </p>
    <pre><code class="prism language-css"><span class="token selector">body</span> <span class="token punctuation">{<!-- --></span>
  <span class="token property">font-size</span><span class="token punctuation">:</span> 1rem<span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
    <h3>
     <a id="6__44">
     </a>
     6. 不要仅依赖颜色来传达信息
    </h3>
    <p>
     颜色不应是传达信息的唯一方式。例如，标识用户登陆状态，不仅要用绿色或者灰色原点表示，还应加上文字描述和图标。

    <h3>
     <a id="7_HTML_53">
     </a>
     7. 使用原生HTML标签
    </h3>
    <p>
     避免使用
     <code>
      &lt;div&gt;
     </code>
     或
     <code>
      &lt;span&gt;
     </code>
     来模拟按钮或链接, 应使用原生的
     <code>
      &lt;button&gt;
     </code>
     和
     <code>
      &lt;a&gt;
     </code>
     标签，以确保键盘导航和屏幕阅读器的兼容性。如果实在无可避免，需要提供ARIA 属性，比如
     <code>
      role="button".

    <h3>
     <a id="9_ARIA_76">
     </a>
     9. 谨慎使用ARIA属性
    </h3>
    <p>
     ARIA（Accessible Rich Internet Applications）属性可以增强可访问性，但应谨慎使用。许多原生HTML标签已经具有足够的可访问性，例如
     <code>
      &lt;button&gt;
     </code>
     不需要额外的
     <code>
      role="button"
     </code>
     。
    </p>
    <h3>
     <a id="10_ARIA_Live_Regions_79">
     </a>
     10. 使用ARIA Live Regions
    </h3>
    <p>
     对于动态更新的内容（如实时通知或聊天消息），可以使用
     <code>
      aria-live
     </code>
     属性来通知屏幕阅读器。不过确保设置正确的
     <code>
      aria-live
     </code>
     属性值，确保动态内容的更新不会干扰用户的操作。通常
     <code>
      aria-live="polite"
     </code>
     是合适的，标识空闲的时候通知。
    </p>
