    <center>
     <font size="4">
      python的虚拟环境可以方便的在不同python版本和第三方库的版本之间进行切换,各虚拟环境独自管理自己的包,方便不同版本需求的开发.
     </font>
    </center>
    <center>
     pycharm自带有虚拟环境的功能,创建项目时如果选''新建虚拟环境'',将会默认放到项目路径下.
    </center>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6b83af9f5eb5c8c585054940fa1258f6.png#pic_center"/>
    </p>
    <h2>
     <a id="linux_5">
     </a>
     linux安装配置虚拟环境
    </h2>
    <p>
     下面介绍手动安装配置虚拟环境:
    </p>
    <p>
     linux中(ubuntu 20.04),linux的源相关配置见我之前的文章.
    </p>
    <h3>
     <a id="_11">
     </a>
     具体步骤
    </h3>
    <p>
     安装 virtualenvwrapper,virtualenvwrapper是对virtualenv的封装.
    </p>
    <blockquote>
     <p>
      <code>
       pip install virtualenvwrapper
      </code>
      .
     </p>
    </blockquote>
    <p>
     如果出现如下异常,将 pip 国内源的协议更改为 https.
    </p>
    <blockquote>
     <p>
      打开之前配置 pip 源的文件.
      <br/>
      将
      <code>
       http
      </code>
      更改为
      <code>
       https
      </code>
      .重新安装即可.
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0ed153c164aa46c831dac36797dc26d7.png#pic_center"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/98c34436081277131204f4adab42937b.png#pic_center">
      <br/>
      安装完成后创建一个目录存放虚拟环境
     </img>
    </p>
    <blockquote>
     <p>
      放哪都可以,一般都是 ~/.virtualenvs
      <br/>
      <code>
       mkdir ~/.virtualenvs
      </code>
     </p>
    </blockquote>
    <p>
     配置路径
    </p>
    <blockquote>
     <p>
      <code>
       vim ~/.bashrc
      </code>
     </p>
    </blockquote>
    <p>
     在其中最下边添加如下配置
    </p>
    <blockquote>
     <p>
      <code>
       export WORKON_HOME=~/.virtualenvs
      </code>
      #虚拟环境的创建路径
      <br/>
      <code>
       export VIRTUALENVWRAPPER_VIRTUALENV=~/.local/bin/virtualenv
      </code>
      #配置这个防止用户家目录下的.local/bin 没有添加到 PATH 中.如果配置了环境变量也可以不添加这一条.
      <br/>
      <code>
       source /usr/local/bin/virtualenvwrapper.sh
      </code>
      #之前安装的可执行文件
      <br/>
      <br/>
      这里的路径不要复制我的,根据自己的实际安装路径.
      <br/>
      我的
      <code>
       virtualenvwrapper
      </code>
      的安装路径是
      <code>
       ~/.local/bin
      </code>
      <br/>
      所以就是:
      <br/>
      <code>
       export WORKON_HOME=~/.virtualenvs
      </code>
      <br/>
      <code>
       export VIRTUALENVWRAPPER_VIRTUALENV=~/.local/bin/virtualenv
      </code>
      <br/>
      <code>
       source ~/.local/bin/virtualenvwrapper.sh
      </code>
     </p>
    </blockquote>
    <p>
     然后刷新配置
    </p>
    <blockquote>
     <p>
      <code>
       source .bashrc
      </code>
     </p>
    </blockquote>
    <center>
     如果如下图就安装成功(到这还没有完).
    </center>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f29f506b3fdc4c1cd874fffbda5615b5.png#pic_center">
      <br/>
      如果出现如下异常:
     </img>
    </p>
    <blockquote>
     <p>
      <code>
       virtualenvwrapper.sh: There was a problem running the initialization hooks. If Python could not import the module virtualenvwrapper.hook_loader, check that virtualenvwrapper has been installed for VIRTUALENVWRAPPER_PYTHON= and that PATH is set properly.
      </code>
      (我这里没有正确换行,自己对比下)
     </p>
    </blockquote>
    <p>
     这是因为python启动命令的问题,一般 ubuntu18 和 ubuntu20 都是默认python3 作为python的启动命令,用来区别python2 和 python3.
    </p>
    <p>
     方法1:可以创建一个名为python的软连接指向python3:
    </p>
    <blockquote>
     <p>
      <code>
       sudo ln -s /usr/bin/python3 /usr/bin/python
      </code>
      <br/>
      正常情况下启动命令都在bin目录下
      <br/>
      不在就根据自己实际目录创建软连接并放到bin目录下
     </p>
    </blockquote>
    <p>
     方法2:也可以更改默认的路径.
     <br/>
     找到上面安装的virtualenvwrapper.sh,修改里边的路径
    </p>
    <blockquote>
     <p>
      <code>
       vim ~/.local/bin/virtualenvwrapper.sh
      </code>
     </p>
    </blockquote>
    <center>
     做出如图修改
    </center>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a6f4048fdb3eabccfce07bff5f4f1279.png#pic_center">
      <br/>
      然后再次刷新,即可配置完成.
     </img>
    </p>
    <p>
     到这只是配置完了虚拟环境这个工具,下面说怎么用.
    </p>
    <h3>
     <a id="_78">
     </a>
     创建虚拟环境
    </h3>
    <blockquote>
     <p>
      <code>
       mkvirtualenv django
      </code>
      <br/>
      <code>
       mkvirtualenv 虚拟环境名称
      </code>
      <br/>
      默认放到上面配置的路径中,默认使用配置中的python版本.
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/415142ea5a1c7e2e4d3db6f4fc452650.png#pic_center"/>
    </p>
    <h3>
     <a id="_85">
     </a>
     查看当前拥有的虚拟环境
    </h3>
    <blockquote>
     <p>
      <code>
       workon
      </code>
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/25276a4aa4b0cdba7eb262a9219194a1.png#pic_center"/>
    </p>
    <h3>
     <a id="_90">
     </a>
     进入虚拟环境
    </h3>
    <blockquote>
     <p>
      <code>
       workon 虚拟环境名
      </code>
      前边的括号是进入虚拟环境的标志.
      <br/>
      <code>
       pip list
      </code>
      可以查看当前虚拟环境的包.
      <br/>
      <code>
       pip install 套件名==版本
      </code>
      指定包版本安装到当前虚拟环境.
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/476d83d171b019b90da2f9ee663dcdb6.png#pic_center"/>
    </p>
    <h3>
     <a id="_96">
     </a>
     退出虚拟环境
    </h3>
    <blockquote>
     <p>
      在虚拟环境中使用
      <br/>
      <code>
       deactivate
      </code>
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b4df4c2d7842340dc941f9f39c0856f2.png#pic_center"/>
    </p>
    <h3>
     <a id="_101">
     </a>
     删除虚拟环境
    </h3>
    <blockquote>
     <p>
      1.先退出虚拟环境
      <br/>
      2.删除
      <br/>
      <code>
       rmvirtualenv 虚拟环境名称
      </code>
     </p>
    </blockquote>
    <h3>
     <a id="python_106">
     </a>
     指定python版本创建虚拟环境
    </h3>
    <blockquote>
     <p>
      <code>
       mkvirtualenv -p /usr/bin/python3.6 xxx
      </code>
      <br/>
      <code>
       mkvirtualenv -p python的路径 虚拟环境名
      </code>
      顺序可以颠倒
     </p>
    </blockquote>
    <p>
     以上是linux中的虚拟环境创建.windows的有时间再说.
    </p>
    <p>
     <strong>
      如有错误还望指出
     </strong>
    </p>
