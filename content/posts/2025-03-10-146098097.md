    <h3>
     <a id="_npm__33">
     </a>
     设置 npm 全局环境目录（避免权限问题）
    </h3>
    <h4>
     <a id="Win_34">
     </a>
     Win
    </h4>
    <h4>
     <font color="red">
      注意：D:\develop\NodeJS 这个目录是NodeJS的安装目录
     </font>
    </h4>
    <pre><code class="prism language-shell"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> prefix <span class="token string">"D:\develop\NodeJS"</span>
</code></pre>
    <h4>
     <a id="Mac_40">
     </a>
     Mac
    </h4>
    <ul>
     <li>
      默认情况下，npm 全局包会安装到 /usr/local/，可能需要管理员权限。以下是重新设置 npm 全局环境的方法：
     </li>
     <li>
      创建新的全局目录
      <pre><code class="prism language-shell"><span class="token function">mkdir</span> ~/.npm-global
</code></pre>
     </li>
     <li>
      配置 npm 使用新的目录
      <br/>
      运行以下命令设置 npm 全局目录到用户目录：
      <pre><code class="prism language-shell"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> prefix ~/.npm-global
</code></pre>
     </li>
     <li>
      添加目录到环境变量
      <br/>
      将新目录添加到 PATH 环境变量中：
      <br/>
      编辑 ~/.zshrc 文件（如果是 bash，编辑 ~/.bashrc）：
      <pre><code class="prism language-shell"><span class="token function">vim</span>  ~/.zshrc
添加以下内容
<span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token environment constant">$PATH</span>:~/.npm-global/bin
</code></pre>
     </li>
     <li>
      验证新的 npm 配置
      <br/>
      运行以下命令，确认 npm 的全局目录已更改：
      <pre><code class="prism language-shell"><span class="token function">npm</span> config get prefix
输出应为：
/Users/<span class="token operator">&lt;</span>你的用户名<span class="token operator">&gt;</span>/.npm-global
</code></pre>
     </li>
    </ul>
    <h2>
     <a id="VUE_CLI_66">
     </a>
     VUE CLI安装
    </h2>
    <h3>
     <a id="_67">
     </a>
     安装
    </h3>
    <ul>
     <li>
      <font color="red">
       <strong>
        CLI可安可不安，安装的话就是可以使用ui界面创建项目
       </strong>
      </font>
     </li>
     <li>
      注意事项
      <ul>
       <li>
        使用–force选项可能会忽略一些重要的错误信息，导致安装的包不完全符合预期，使用时需谨慎。
       </li>
       <li>
        在大多数情况下，建议先检查网络连接和权限设置，尽量避免使用强制安装选项。
       </li>
      </ul>
     </li>
    </ul>
    <pre><code class="prism language-shell">默认安装命令
<span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> @vue/cli
是默认的安装命令，用于全局安装Vue CLI工具。这个命令会尝试正常安装Vue CLI，如果过程中遇到问题，可能会失败。
</code></pre>
    <pre><code class="prism language-shell">强制安装命令
<span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> @vue/cli <span class="token parameter variable">--force</span>
是在默认命令的基础上增加了--force选项，用于强制安装Vue CLI。这个选项会在安装过程中遇到错误时尝试继续进行，即使某些包或依赖无法正常下载或安装。
</code></pre>
    <h3>
     <a id="_82">
     </a>
     验证
    </h3>
    <pre><code class="prism language-shell">vue <span class="token parameter variable">--version</span>
如果输出类似 @vue/cli <span class="token number">5</span>.x.x，说明安装成功。
</code></pre>

    <h2>
     <a id="Vue_92">
     </a>
     Vue脚手架
    </h2>
    <p>
     在dos黑窗口中创建项目
    </p>
    <ul>
     <li>
      npm init vue@latest 和 npm create vue@latest一样
     </li>
     <li>
      vue create 和使用vue ui页面一样
     </li>
     <li>
      是两种不同的创建 Vue 项目的方式，它们分别基于 Vite 和 Vue CLI
     </li>
    </ul>
    <h3>
     <a id="npm_init_vuelatestnpm_create_vuelatest_97">
     </a>
     npm init vue@latest、npm create vue@latest
    </h3>
    <ul>
     <li>
      基于 Vite：使用 Vite 作为构建工具。
     </li>
     <li>
      现代开发体验：Vite 提供了更快的开发服务器启动速度和热更新。
     </li>
     <li>
      轻量级：Vite 的配置更简单，适合现代前端开发。
     </li>
     <li>
      命令
      <pre><code class="prism language-shell"><span class="token function">npm</span> init vue@latest
</code></pre>
     </li>
     <li>
      特点
      <ul>
       <li>
        交互式配置：
       </li>
       <li>
        提供一系列选项（如 TypeScript、Vue Router、Pinia 等），可以根据需要选择。
       </li>
       <li>
        示例：
        <ul>
         <li>
          ✔ Project name: … my-vue-app
         </li>
         <li>
          ✔ Add TypeScript? … No / Yes
         </li>
         <li>
          ✔ Add JSX Support? … No / Yes
         </li>
         <li>
          ✔ Add Vue Router? … No / Yes
         </li>
         <li>
          ✔ Add Pinia? … No / Yes
         </li>
         <li>
          ✔ Add Vitest? … No / Yes
         </li>
         <li>
          ✔ Add Cypress? … No / Yes
         </li>
         <li>
          ✔ Add ESLint? … No / Yes
         </li>
         <li>
          ✔ Add Prettier? … No / Yes
         </li>
        </ul>
       </li>
       <li>
        默认生成基于 Vite 的项目结构。
       </li>
       <li>
        包含 vite.config.js 文件。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4d7ccda515944625a815b8b067d8cfab.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="vue_createvue_ui_121">
     </a>
     vue create、vue ui
    </h3>
    <ul>
     <li>
      基于 Vue CLI：使用 Webpack 作为构建工具。
     </li>
     <li>
      传统开发体验：Vue CLI 是 Vue 官方提供的传统脚手架工具，功能全面但启动速度较慢。
     </li>
     <li>
      适合复杂项目：Vue CLI 提供了更多插件和配置选项
     </li>
     <li>
      命令
      <pre><code class="prism language-shell"><span class="token function">npm</span> <span class="token function">install</span> <span class="token parameter variable">-g</span> @vue/cli
vue create my-vue-app
</code></pre>
     </li>
     <li>
      特点
      <ul>
       <li>
        示例：
        <ul>
         <li>
          ? Please pick a preset: Manually select features
         </li>
         <li>
          ? Check the features needed for your project:
         </li>
         <li>
          ◉ Babel
         </li>
         <li>
          ◉ TypeScript
         </li>
         <li>
          ◯ Progressive Web App (PWA) Support
         </li>
         <li>
          ◉ Router
         </li>
         <li>
          ◉ Vuex
         </li>
         <li>
          ◉ CSS Pre-processors
         </li>
         <li>
          ◉ Linter / Formatter
         </li>
         <li>
          ◯ Unit Testing
         </li>
         <li>
          ◯ E2E Testing
         </li>
        </ul>
       </li>
       <li>
        目录结构
        <ul>
         <li>
          my-vue-app/
          <br/>
          ├── node_modules/
          <br/>
          ├── public/
          <br/>
          ├── src/
          <br/>
          │ ├── assets/
          <br/>
          │ ├── components/
          <br/>
          │ ├── views/
          <br/>
          │ ├── App.vue
          <br/>
          │ ├── main.js
          <br/>
          │ ├── router.js
          <br/>
          │ ├── store.js
          <br/>
          ├── .gitignore
          <br/>
          ├── babel.config.js
          <br/>
          ├── package.json
          <br/>
          ├── README.md
          <br/>
          ├── vue.config.js
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
