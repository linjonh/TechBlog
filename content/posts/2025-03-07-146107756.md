     什么是JVM
    </h1>

    <p>
     JVM(Java Virtual Machine)是java程序的核心运行环境，主要工作是解释字节码并映射到本地的CPU指令集和OS系统调用，JVM屏蔽了与操作系统平台的相关信息，使Java程序无需进行修改就可以在多种不同平台上运行，即"
     <strong>
      一次编译，到处运行
     </strong>
     "。
     <br/>
     <strong>
      一、基本定义与核心作用
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      :JVM是一个抽象化的计算机规范，通过在实际硬件和操作系统上模拟处理器、堆栈、寄存器等硬件功能来实现。它属于Java运行时环境(JRE)的一部分，是Java语言实现"一次编译，到处运行"的关键。
     </li>
     <li>
      <strong>
       核心作用
      </strong>
      :
      <br/>
      <strong>
       跨平台性
      </strong>
      :JVM屏蔽了底层操作系统的差异，Java程序只需要编译为字节码(.class文件)，即可在任何安装对应版本JVM的平台上运行。
      <br/>
      <strong>
       内存管理
      </strong>
      :自动垃圾回收(GC)机制管理堆内存，减少内存泄漏风险。
      <br/>
      <strong>
       安全性
      </strong>
      :通过字节码验证、类加载隔离等机制保证程序安全。
      <br/>
      二、JVM的体系结构
      <br/>
      JVM的架构由以下核心组件构成:
     </li>
     <li>
      1.
      <strong>
       类加载器(Class Loader)
      </strong>
      <ul>
       <li>
        <strong>
         功能
        </strong>
        :加载字节码到内存，包括启动类加载器(加载核心Java类)、扩展类加载器(加载扩展库)、应用类加载器(加载用户类)。
       </li>
       <li>
        <strong>
         双亲委派机制
        </strong>
        :确保类加载的优先级和安全性，避免重复加载。
       </li>
      </ul>
     </li>
     <li>
      2.
      <strong>
       运行时数据区(Runtime Data Area)
      </strong>
      <ul>
       <li>
        <strong>
         方法区
        </strong>
        :存储类结构、常量池、静态变量等元数据。
       </li>
       <li>
        <strong>
         堆(Heap)
        </strong>
        :所有对象实例的存储区域，由垃圾回收器管理。
       </li>
       <li>
        <strong>
         虚拟机栈
        </strong>
        :每个线程私有的栈，存放局部变量和方法调用帧。
       </li>
       <li>
        <strong>
         本地方法栈
        </strong>
        :支持本地方法(如C/C++代码)的执行。
       </li>
       <li>
        <strong>
         程序计数器
        </strong>
        :记录当前线程执行的指令地址。
       </li>
      </ul>
     </li>
     <li>
      3.
      <strong>
       执行引擎(Execution Engine)
      </strong>
      <ul>
       <li>
        <strong>
         解释器
        </strong>
        :逐行解释字节码，启动速度快但效率低。
       </li>
       <li>
        <strong>
         即时编译器(JIT)
        </strong>
        :将热点代码编译为本地机器码，提升执行效率。
       </li>
       <li>
        <strong>
         垃圾回收器
        </strong>
        :自动回收堆内存中不再使用的对象。
        <br/>
        <strong>
         三、JVM的工作流程
        </strong>
       </li>
      </ul>
     </li>
     <li>
      1.
      <strong>
       类加载阶段
      </strong>
      <ul>
       <li>
        加载 -&gt; 链接(验证、准备、解析) -&gt;初始化 , 确保类符合规范
       </li>
      </ul>
     </li>
     <li>
      2.
      <strong>
       字节码执行
      </strong>
      <ul>
       <li>
        解释器直接执行，或由JIT优化编译后执行。
       </li>
      </ul>
     </li>
     <li>
      3.
      <strong>
       内存管理
      </strong>
      <ul>
       <li>
        堆内存划分为新生代(Young Generation)和老年代(Old Generation)，不同的垃圾回收算法(如标记-清除、分代回收)协同工作。
        <br/>
        <strong>
         四、JVM与JDK、JRE的关系
        </strong>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       JDK(Java开发工具包)
      </strong>
      :包含JRE和编译器(javac)、调试工具等，用于开发Java程序。
     </li>
     <li>
      <strong>
       JRE(Java运行环境)
      </strong>
      :包含JVM和基础类库，是运行Java程序的必要条件。
     </li>
     <li>
      <strong>
       层级关系
      </strong>
      :JDK &gt; JRE &gt; JVM , 三者共同构成Java生态的底层支持。
      <br/>
      <strong>
       总结
      </strong>
      <br/>
      JVM作为Java技术的基石，通过抽象硬件、管理内存、优化执行等机制，实现了Java的跨平台能力和高性能表现。理解其架构和工作原理，对于开发高性能应用、调优系统瓶颈(如GC策略)具有重要意义。
     </li>
    </ul>
