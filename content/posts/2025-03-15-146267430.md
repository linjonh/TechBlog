    <h2 id="%E4%B8%80%E3%80%81%E5%93%88%E5%B8%8C%E8%A1%A8" name="%E4%B8%80%E3%80%81%E5%93%88%E5%B8%8C%E8%A1%A8">
     一、哈希表
    </h2>
    <h3 id="1.1.%20%E6%A6%82%E5%BF%B5" name="1.1.%20%E6%A6%82%E5%BF%B5">
     1.1. 概念
    </h3>
    <p>
     顺序结构以及平衡树中，元素关键码与其存储位置之间没有对应的关系，因此在查找⼀个元素时，必须要经过关键码的多次比较。顺序查找时间复杂度为
     <img alt="O(n)" class="mathcode" src="https://latex.csdn.net/eq?O%28n%29">
      ，平衡树中为树的⾼度，即
      <img alt="O(n)" class="mathcode" src="https://latex.csdn.net/eq?O%28n%29">
       ，搜索的效率取决于搜索过程中元素的比较次数。
      </img>
     </img>
    </p>
    <p>
     理想的搜索⽅法：可以不经过任何比较，⼀次直接从表中得到要搜索的元素。如果构造⼀种存储结构，通过某种函数(hashFunc)使元素的存储位置与它的关键码之间能够建立一一映射的关系，那么在查找时通过该函数可以很快找到该元素。
    </p>
    <p>
     当向该结构中，根据待插⼊元素的关键码，以此函数计算出该元素的存储位置并按此位置进行存放；对元素的关键码进行同样的计算，把求得的函数值当做元素的存储位置，在结构中按此位置取元素比较，若关键码相等，则搜索成功。该方式即为哈希(散列)方法，哈希方法中使⽤的转换函数称为哈希(散列)函数，构造出来的结构称为哈希表(Hash Table)(或者称散列表)。
    </p>
    <p>
     哈希函数设置为：hash(key) = key % capacity；其中capacity为存储元素底层空间总的大小。
    </p>
    <p>
     我们设一个整数集合{1,7,6,4,5,9}，把capacity设置为10，那我们就可以按照下图来存储。如果我们再想存放一个元素12，我们可以直接通过哈希函数存进下标2中，要想搜索，直接通过2下标来找到12，这样时间复杂度为
     <img alt="O(n)" class="mathcode" src="https://latex.csdn.net/eq?O%28n%29">
      ，从而提高效率。
     </img>
    </p>
    <p class="img-center">
     <img alt="" height="148" src="https://i-blog.csdnimg.cn/direct/0ca8173d59ef4f489ffdcc8a2c9b7ebc.png" width="599"/>
    </p>
    <h3 id="1.2.%20%E5%86%B2%E7%AA%81" name="1.2.%20%E5%86%B2%E7%AA%81">
     1.2. 冲突
    </h3>
    <p>
     不同关键字通过相同哈希函数计算出相同的哈希地址，该种现象称为哈希冲突或哈希碰撞。把具有不同关键码⽽具有相同哈希地址的数据元素称为“同义词”。例如我们要想存放一个14，通过上面的哈希函数应该存到4下标，但此时4下标已经存了一个4，就会造成哈希冲突。
    </p>
    <p>
     出现了哈希冲突，我们就要想办法避免冲突。由于我们哈希表底层数组的容量往往是小于实际要存储的关键字的数量的，就会导致冲突的发⽣是必然的，但我们能做的应该是尽量的降低冲突率。
    </p>
    <h3 id="1.3.%20%E9%81%BF%E5%85%8D%E5%86%B2%E7%AA%81" name="1.3.%20%E9%81%BF%E5%85%8D%E5%86%B2%E7%AA%81">
     1.3. 避免冲突
    </h3>
    <p>
     第一种方法可以设计合理的哈希函数。设计原则：定义域必须包括需要存储的全部关键码，而如果散列表允许有m个地址时，其值域必须在0到m-1之间；计算出来的地址能均匀分布在整个空间中；比较简单。
    </p>
    <p>
     直接订制法：取关键字的某个线性函数为散列地址：Hash(Key) = A*Key + B。优点：简单、均匀。缺点：需要事先知道关键字的分布情况。使用场景：适合查找比较小且连续的情况。
    </p>
    <p>
     除留余数法：设散列表中允许的地址数为m，取⼀个不大于m，但最接近或者等于m的质数p作为除数，按照哈希函数：Hash(key) = key% p(p&lt;=m)，将关键码转换成哈希地址。
    </p>
    <p>
     哈希函数设计的越精妙，产生哈希冲突的可能性就越低，但是无法避免哈希冲突。
    </p>
    <p>
     我们还有另外一种就是调节负载因子。哈希表的载荷因子为：ą=填入表中元素个数/哈希表长度。当冲突率达到⼀个⽆法忍受的程度时，我们需要通过降低负载因子来变相的降低冲突率。已知哈希表中已有的关键字个数是不可变的，那我们能调整的就只有哈希表中的数组的大小。
    </p>
    <p class="img-center">
     <img alt="" height="409" src="https://i-blog.csdnimg.cn/direct/6f0ada535f2c482e9c571d6ec575f93b.png" width="574"/>
    </p>
    <p>
    </p>
    <h3 id="1.4.%20%E8%A7%A3%E5%86%B3%E5%86%B2%E7%AA%81" name="1.4.%20%E8%A7%A3%E5%86%B3%E5%86%B2%E7%AA%81">
     1.4. 解决冲突
    </h3>
    <p>
     解决哈希冲突两种常⻅的⽅法是：闭散列和开散列。
    </p>
    <p>
     闭散列：也叫开放地址法，当发⽣哈希冲突时，如果哈希表未被装满，说明在哈希表中必然还有空位置，那么可以把key存放到冲突位置中的“下⼀个” 空位置中去。那如何寻找下⼀个空位置呢？此时我们就需要应用线性探索。从发生冲突的位置开始，依次向后探测，直到寻找到下⼀个空位置为止。但这样还是会有一个缺点，就是会使得冲突元素聚集在一起，并且如果我们把4删除了，又如何寻找14、24、34这些元素。因此线性探测采⽤标记的伪删除法来删除⼀个元素。
    </p>
    <p class="img-center">
     <img alt="" height="171" src="https://i-blog.csdnimg.cn/direct/2e15eae0d75b44d0aeba844287dcfee6.png" width="578"/>
    </p>
    <p>
     ⼆次探测为了避免该问题，找下⼀个空位置的⽅法为：Hi = (H0+i^2)%m，i表示冲突的次数，m为表的大小。
    </p>
    <p>
     开散列：开散列法⼜叫链地址法(开链法)，⾸先对关键码集合⽤散列函数计算散列地址，具有相同地址的关键码归于同一子集合，每⼀个⼦集合称为⼀个桶，各个桶中的元素通过⼀个单链表链接起来，各链表的头结点存储在哈希表中。
    </p>
    <h3 id="1.5.%20%E5%AE%9E%E7%8E%B0" name="1.5.%20%E5%AE%9E%E7%8E%B0">
     1.5. 实现
    </h3>
    <p>
     由于我们需要节点数组来创建哈希表，利用内部类来表示节点对象。
    </p>
    <pre><code class="language-java">public class HashBucket {
    //创建节点数组
    static class Node{
        public int key;
        public int val;
        public Node next;

        public Node(int key, int val) {
            this.key = key;
            this.val = val;
        }
    }

    public Node[] array = new Node[10];
    public int UsedSize;//表示还系统中存放的元素
    public static final float LOAD_FACTOR = 0.75f;//负载因子表示为常数
}</code></pre>
    <p>
     我们先来模拟哈希表中放元素的方法。我们要想把元素放入，首先得是一个结点。比如key=14，如果表中已经有14了，就不能再放14并且更新val，所以我们首先需要遍历数组判断key是否相同，如果相同，更新val。下面再使用头插法来把节点元素放入哈希表中。插入元素之后，我们还需要重新计算负载因子是否超过了我们规定的LOAD_FACTOR。如果超过了，就需要进行扩容操作。扩容的时候还需要注意，比如我们要插入的元素的key为14，扩容前需要插入下标为4的位置，扩容2倍后，就需要插入下标为14的位置。
    </p>
    <p>
     完整代码实现：
    </p>
    <pre><code class="language-java">    public void put(int key, int val) {
        int index = key % array.length;

        //先遍历index数组下的链表，如果有相同的key，则更新val
        Node cur = array[index];
        while (cur != null) {
            if (cur.key == key) {
                cur.val = val;
                return;
            }
            cur = cur.next;
        }

        //头插法
        Node node = new Node(key, val);
        node.next = array[index];
        array[index] = node;
        UsedSize++;

        //重新计算负载因子是不是超过了我们规定的值
        if (CalculateLoadFactor() &gt;= LOAD_FACTOR) {
            //扩容
            ReSize();
        }
    }

    private float CalculateLoadFactor() {
        return UsedSize * 1.0f / array.length;
    }

    private void ReSize() {
        Node[] newArray = new Node[array.length*2];
        for (Node node : array) {
            Node cur = node;
            while (cur != null) {
                int newIndex = cur.key % newArray.length;
                //把当前节点放入新数组的位置,再次使用头插法
                Node curNext = cur.next;
                cur.next = newArray[newIndex];
                newArray[newIndex] = cur;
                cur = curNext;
            }
        }
        array = newArray;
    }</code></pre>
    <p>
     get方法也是一样，也是通过索引下标来寻找目标值。
    </p>
    <pre><code class="language-java">    public int get(int key){
        int index = key % array.length;
        Node cur = array[index];
        while(cur != null){
            if(cur.key == key){
                return cur.val;
            }
            cur = cur.next;
        }
        return -1;
    }</code></pre>

     <img alt="" height="424" src="https://i-blog.csdnimg.cn/direct/4d4242c94e7445859377f88d4b2b7acb.png" width="1032"/>
    </p>
    <p class="img-center">
     <img alt="" height="315" src="https://i-blog.csdnimg.cn/direct/ef1266436e1e46c0a0d66b66a975403c.png" width="850"/>

    <p>
     我们上面的方法key是整型，那如果key是引用类型呢，比如String或者Person类。那我们就把整型换作是泛型K、V。需要注意的是，key换成了泛型，不能直接进行%操作，我们可以使用hashCode方法转成整型，并且进行比较要使用equals方法。
    </p>
    <pre><code class="language-java">/**
 * @author: gao
 * @create-date: 2025/3/15 16:32
 */

public class HashBucket&lt;K, V&gt; {
    static class Node&lt;K, V&gt; {
        public K key;
        public V val;
        public Node&lt;K, V&gt; next;

        public Node(K key, V val) {
            this.key = key;
            this.val = val;
        }
    }

    public Node&lt;K, V&gt;[] array = (Node&lt;K, V&gt;[]) new Node[10];
    public int UsedSize;//表示还系统中存放的元素
    public static final float LOAD_FACTOR = 0.75f;//负载因子表示为常数

    public void put(K key, V val) {
        int hashcode = key.hashCode();
        int index = hashcode % array.length;

        //先遍历index数组下的链表，如果有相同的key，则更新val
        Node&lt;K, V&gt; cur = array[index];
        while (cur != null) {
            if (cur.key == key) {
                cur.val = val;
                return;
            }
            cur = cur.next;
        }
        Node&lt;K, V&gt; node = new Node&lt;K, V&gt;(key, val);
        node.next = array[index];
        array[index] = node;
        UsedSize++;
    }

    public V get(K key) {
        int hashcode = key.hashCode();
        int index = hashcode % array.length;
        Node&lt;K, V&gt; cur = array[index];
        while (cur != null) {
            if (cur.key.equals(key)) {
                return cur.val;
            }
            cur = cur.next;
        }
        return null;
    }
}</code></pre>

    <h2 id="%E4%BA%8C%E3%80%81OJ%E7%BB%83%E4%B9%A0" name="%E4%BA%8C%E3%80%81OJ%E7%BB%83%E4%B9%A0">
     二、OJ练习
    </h2>
    <h3 id="2.1.%C2%A0%E5%8F%AA%E5%87%BA%E7%8E%B0%E4%B8%80%E6%AC%A1%E7%9A%84%E6%95%B0%E5%AD%97" name="2.1.%C2%A0%E5%8F%AA%E5%87%BA%E7%8E%B0%E4%B8%80%E6%AC%A1%E7%9A%84%E6%95%B0%E5%AD%97">
     2.1.
     <a class="link-info" href="https://leetcode.cn/problems/single-number/description/" rel="nofollow" title="只出现一次的数字">
      只出现一次的数字
     </a>
    </h3>
    <p class="img-center">
     <img alt="" height="237" src="https://i-blog.csdnimg.cn/direct/2c774a59d8e04d81b7ee4cdc5c642111.png" width="457"/>
    </p>
    <p>
     我们的基本思路是：利用HashSet，先遍历一遍数组，把集合中没有的数字放入，如果有，再移除，最后集合中剩下的元素就是只出现一次的数字，再遍历一遍数组，匹配HashSet中的数组。
    </p>
    <p>
     完整代码实现：
    </p>
    <pre><code class="language-java">class Solution {
    public int singleNumber(int[] nums) {
        Set&lt;Integer&gt; set = new HashSet&lt;&gt;();
        for (int i = 0;i &lt; nums.length;i++) {
            if(! set.contains(nums[i])){
                set.add(nums[i]);
            }else{
                set.remove(nums[i]);
            }
        }
        for (int i = 0;i &lt; nums.length;i++) {
            if(set.contains(nums[i])){
                return nums[i];
            }
        }
        return -1;
    }
}</code></pre>
    <p>
     执行时间还是比较高，因为使用了两次for循环遍历数组。
    </p>
    <h3 id="2.2.%C2%A0%E9%9A%8F%E6%9C%BA%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%8D%E5%88%B6" name="2.2.%C2%A0%E9%9A%8F%E6%9C%BA%E9%93%BE%E8%A1%A8%E7%9A%84%E5%A4%8D%E5%88%B6" style="background-color:transparent">
     2.2.
     <a class="link-info" href="https://leetcode.cn/problems/copy-list-with-random-pointer/description/" rel="nofollow" title="随机链表的复制">
      随机链表的复制
     </a>
    </h3>
    <p class="img-center">
     <img alt="" height="511" src="https://i-blog.csdnimg.cn/direct/6386bf32d6b54b949dae6578e1b2b30a.png" width="414"/>
    </p>
    <p>
     题目比较长，大概题意就是复制出一份与原来相同的链表。这道题的难点在于比单链表多了一个可以指向任意节点或者空的random域。起初，很多人会去想定义一个Node cur去遍历一遍链表，一个一个节点进行拷贝，但一拷贝就会发现问题，因为我们我们不知道cur.next和cur.random是哪一个节点的地址。既然遍历一遍链表不行，那就遍历两遍。第一遍遍历，所有节点的val域全都拷贝过来，next域以及random域全都默认为null，每遍历一个链表，就新实例化一个节点。然后我们&lt;K,V&gt;结构来建立老节点与新节点之间的映射关系。
    </p>
    <p class="img-center">
     <img alt="" height="324" src="https://i-blog.csdnimg.cn/direct/a4c4e48a0d6a44ecaf0e34870b693967.png" width="382"/>
    </p>
    <p>
     我们每获取一个节点的地址，都可以修改它的next域与random域。
    </p>
    <p>
     完整代码实现：
    </p>
    <pre><code class="language-java">class Solution {
    public Node copyRandomList(Node head) {
        Map&lt;Node,Node&gt; map = new HashMap&lt;&gt;();
        //第一遍遍历链表
        Node cur = head;
        while(cur != null){
            Node node = new Node(cur.val);
            map.put(cur,node);
            cur = cur.next;
        }
        //第二遍遍历链表
        cur = head;
        while(cur != null){
            map.get(cur).next = map.get(cur.next);
            map.get(cur).random = map.get(cur.random);
            cur = cur.next;
        }
        return map.get(head);
    }
}</code></pre>
    <h3 id="%C2%A02.3.%C2%A0%E5%AE%9D%E7%9F%B3%E4%B8%8E%E7%9F%B3%E5%A4%B4" name="%C2%A02.3.%C2%A0%E5%AE%9D%E7%9F%B3%E4%B8%8E%E7%9F%B3%E5%A4%B4">
     2.3.
     <a class="link-info" href="https://leetcode.cn/problems/jewels-and-stones/description/" rel="nofollow" title="宝石与石头">
      宝石与石头
     </a>
    </h3>
    <p class="img-center">
     <img alt="" height="361" src="https://i-blog.csdnimg.cn/direct/c0a5776726d44bc5b2312803ddd0c7c2.png" width="585"/>
    </p>
    <p>
     题目很简单，就是查找stones中含有jewels中的字符的个数。我们先遍历jewels字符串，将里面的字符放入集合中，再去遍历stones中的字符，最后返回宝石个数。
    </p>
    <p>
     完整代码实现：
    </p>
    <pre><code class="language-java">class Solution {
    public int numJewelsInStones(String jewels, String stones) {
        Set&lt;Character&gt; set = new HashSet&lt;&gt;();
        for (int i = 0; i &lt; jewels.length(); i++) {
            char ch = jewels.charAt(i);
            set.add(ch);
        }
        int count = 0;
        for (int i = 0; i &lt; stones.length(); i++) {
            char ch = stones.charAt(i);
            if(set.contains(ch)){
                count++;
            }
        }
        return count;
    }
}</code></pre>
