    <p>
    </p>
    <p>
    </p>
    <p>
     <code>
      Redis
     </code>
     所有的
     <code>
      key
     </code>
     都是字符串，
     <code>
      value
     </code>
     的类型是存在差异的
    </p>
    <hr/>
    <h2>
     <a id="_6">
     </a>
     二进制存储
    </h2>
    <p>
     Redis 中的字符串，直接就是按照二进制数据的方式存储的
    </p>
    <p>
     不仅仅可以存储文本数据，还可以存储整数、普通的文本字符串、JSON、XML、二进制数据（图片、视频、音频…）
    </p>
    <ul>
     <li>
      音频、视频体积可能会比较大，
      <code>
       Redis
      </code>
      对于
      <code>
       string
      </code>
      来说，限制了大小最大为
      <code>
       512M
      </code>
     </li>
     <li>
      <code>
       Redis
      </code>
      是单线程模型，希望进行的操作都是比较快速的，如果存的数据都很大，读取时间就要很长，就可能会导致
      <code>
       Redis
      </code>
      被阻塞
     </li>
     <li>
      所以一般不会存储大文件
     </li>
    </ul>
    <h2>
     <a id="_17">
     </a>
     编码转换
    </h2>
    <p>
     <code>
      Redis
     </code>
     不会做任何的编码转换，存的是什么，取出来就还是什么。
    </p>
    <ul>
     <li>
      <code>
       MySQL
      </code>
      的默认字符集如果是拉丁文，那么当你插入中文的时候，就会失败。当你插入一个数据的时候，
      <code>
       MySQL
      </code>
      就会把你插入的数据拿到字符集里面去找，看看是不是合法的数据，有问题就插入不了
     </li>
     <li>
      <code>
       Redis
      </code>
      就没有
      <code>
       MySQL
      </code>
      这么详细的验证步骤，你存的是什么，取出来的就还是什么。你按
      <code>
       utf-8
      </code>
      存，取出来还是
      <code>
       utf-8
      </code>
      ，按照
      <code>
       gbk
      </code>
      存，取出来就还是
      <code>
       gbk
      </code>
      <br/>
      所以
      <code>
       Redis
      </code>
      一般就不会遇到乱码问题（存取用的编码方式不一致）
     </li>
    </ul>
    <h2>
     <a id="SET__GET_26">
     </a>
     SET 和 GET
    </h2>
    <h3>
     <a id="SET_28">
     </a>
     SET
    </h3>
    <p>
     将
     <code>
      string
     </code>
     类型的
     <code>
      value
     </code>
     设置到
     <code>
      key
     </code>
     中。如果
     <code>
      key
     </code>
     之前存在，则覆盖，⽆论原来的数据类型是什么。之前关于此
     <code>
      key
     </code>
     的
     <code>
      TTL
     </code>
     也全部失效。
    </p>
    <hr/>
    <p>
     <strong>
      语法：
     </strong>
    </p>
    <pre><code class="prism language-redis">SET key value [expiration EX seconds|PX milliseconds] [NX|XX]
</code></pre>
    <ul>
     <li>
      <code>
       set key value ex 10
      </code>
      相当于
      <ol>
       <li>
        <code>
         set key value
        </code>
       </li>
       <li>
        <code>
         expire key 10
        </code>
       </li>
      </ol>
      <ul>
       <li>
        把两步操作一步完成，减少客户端-服务器交互次数，提高效率。其次，一个命令完成就是原子操作，保证
        <strong>
         原子性
        </strong>
       </li>
      </ul>
     </li>
     <li>
      <code>
       NX
      </code>
      <ul>
       <li>
        如果
        <code>
         key
        </code>
        不存在，才设置
       </li>
       <li>
        如果
        <code>
         key
        </code>
        存在，则不设置（返回
        <code>
         nil
        </code>
        ）
       </li>
      </ul>
     </li>
     <li>
      <code>
       XX
      </code>
      <ul>
       <li>
        如果
        <code>
         key
        </code>
        存在，才设置（相当于更新
        <code>
         key
        </code>
        的
        <code>
         value
        </code>
        ）
       </li>
       <li>
        如果
        <code>
         key
        </code>
        不存在，则不设置（返回
        <code>
         nil
        </code>
        ）
        <br/>
        <img alt="image.png|226" src="https://i-blog.csdnimg.cn/img_convert/9917c8468f1541da2060476b45da0038.png"/>
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      <code>
       Redis
      </code>
      文档给出的语法格式说明：
     </p>
     <ul>
      <li>
       <code>
        []
       </code>
       相当于一个独立的单元，表示可选项（可有可无），其中
       <code>
        |
       </code>
       表示“或者”的意思，多个只能出现一个
      </li>
      <li>
       <code>
        []
       </code>
       和
       <code>
        []
       </code>
       之间，是可以同时存在的
      </li>
     </ul>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       快速失去年终奖的小技巧
      </strong>
      ：清除
      <code>
       Redis
      </code>
      上所有的数据==&gt;删库
     </p>
     <ul>
      <li>
       <code>
        FLUSHALL
       </code>
       ，可以把
       <code>
        Redis
       </code>
       上所有的键值对都带走
      </li>
     </ul>
    </blockquote>
    <h3>
     <a id="GET_62">
     </a>
     GET
    </h3>
    <p>
     语法：
    </p>
    <pre><code class="prism language-redis">GET key
</code></pre>
    <ul>
     <li>
      对于
      <code>
       GET
      </code>
      来说，只支持字符串类型的
      <code>
       value
      </code>
      ，如果
      <code>
       value
      </code>
      是其他类型，使用
      <code>
       GET
      </code>
      就会出错
      <img alt="image.png" src="https://i-blog.csdnimg.cn/img_convert/195fc9f64f1eab60b2d8ab5c301eddc3.png"/>
     </li>
    </ul>
    <h2>
     <a id="MSET__MGET_71">
     </a>
     MSET 和 MGET
    </h2>
    <p>
     一次操作多组键值对，一次网络传输就能完成多组键值对的操作
     <img alt="image.png" src="https://i-blog.csdnimg.cn/img_convert/2fd864caf48525d65a89992f6abe89c3.png"/>
    </p>
    <p>
     <code>
      MSET
     </code>
     语法：
    </p>
    <pre><code class="prism language-redis">MSET key value [key value...]
</code></pre>
    <p>
     <code>
      MGET
     </code>
     语法：
    </p>
    <pre><code class="prism language-redis">MGET key [key...]
</code></pre>
    <p>
     <img alt="image.png" src="https://i-blog.csdnimg.cn/img_convert/8c198e52be3e9d84d79e88f1df19f885.png"/>
    </p>
    <ul>
     <li>
      <code>
       MSET
      </code>
      和
      <code>
       MGET
      </code>
      的时间复杂度都是：
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         O 
         
        
          ( 
         
        
          N 
         
        
          ) 
         
        
       
         O(N)
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 1em; vertical-align: -0.25em;">
          </span>
          <span class="mord mathnormal" style="margin-right: 0.0278em;">
           O
          </span>
          <span class="mopen">
           (
          </span>
          <span class="mord mathnormal" style="margin-right: 0.109em;">
           N
          </span>
          <span class="mclose">
           )
          </span>
         </span>
        </span>
       </span>
      </span>
      ，
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         N 
         
        
       
         N
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.6833em;">
          </span>
          <span class="mord mathnormal" style="margin-right: 0.109em;">
           N
          </span>
         </span>
        </span>
       </span>
      </span>
      不是整个
      <code>
       redis
      </code>
      服务器中所有
      <code>
       key
      </code>
      的数量，而是当前命令中给出的
      <code>
       key
      </code>
      的数量
     </li>
    </ul>
    <h2>
     <a id="SETNXSETEX__PSETEX_98">
     </a>
     SETNX、SETEX 和 PSETEX
    </h2>
    <ul>
     <li>
      <code>
       SETNX
      </code>
      ：不存在才能设置，存在则设置失败
      <img alt="image.png|247" src="https://i-blog.csdnimg.cn/img_convert/26b07e984e2d309bd3eb694c8b01ce26.png"/>
     </li>
     <li>
      <code>
       SETEX
      </code>
      ：设置
      <code>
       key
      </code>
      的过期时间（单位
      <code>
       s
      </code>
      ）
      <img alt="image.png|303" src="https://i-blog.csdnimg.cn/img_convert/7aaba9865b75190dc5f277ed879d718b.png"/>
     </li>
     <li>
      <code>
       PSETEX
      </code>
      ：设置
      <code>
       key
      </code>
      的过期时间（单位
      <code>
       ms
      </code>
      ）
      <img alt="image.png|344" src="https://i-blog.csdnimg.cn/img_convert/9551375ba2ea4d5fb34c0b4f67938491.png"/>
     </li>
    </ul>
    <p>
     针对
     <code>
      set
     </code>
     的一些常见写法，进行了缩写。之所以这样搞，就是为了让操作更符合人的直觉（使用者的门槛就越低，要背的东西就越少）
    </p>
