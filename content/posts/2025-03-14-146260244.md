    <h5>
     <strong>
      1. Spring 框架
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        定位
       </strong>
       ：轻量级的企业级应用开发框架，核心是
       <strong>
        IoC（控制反转）
       </strong>
       和
       <strong>
        AOP（面向切面编程）
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        核心功能
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          依赖注入（DI）
         </strong>
         ：通过
         <code>
          @Autowired
         </code>
         、
         <code>
          @Component
         </code>
         等注解管理对象生命周期。
        </p>
       </li>
       <li>
        <p>
         <strong>
          事务管理
         </strong>
         ：声明式事务（
         <code>
          @Transactional
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          数据访问
         </strong>
         ：支持 JDBC、ORM（如 Hibernate、MyBatis）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          Web MVC
         </strong>
         ：基于 Servlet 的 MVC 框架，处理 HTTP 请求。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          配置复杂
         </strong>
         ：需手动配置 XML 或 Java Config。
        </p>
       </li>
       <li>
        <p>
         <strong>
          依赖管理繁琐
         </strong>
         ：需要手动解决库版本冲突。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：传统单体应用，需要精细控制配置的复杂场景。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      2. Spring Boot
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        定位
       </strong>
       ：基于 Spring 的快速开发脚手架，目标是
       <strong>
        简化配置
       </strong>
       、
       <strong>
        快速启动
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        核心功能
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          自动配置
         </strong>
         ：根据类路径中的依赖自动配置 Bean（如
         <code>
          spring-boot-starter-web
         </code>
         自动配置 Tomcat）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          内嵌容器
         </strong>
         ：默认集成 Tomcat、Jetty 或 Undertow，无需部署 WAR 包。
        </p>
       </li>
       <li>
        <p>
         <strong>
          起步依赖（Starters）
         </strong>
         ：预置常用依赖组合（如
         <code>
          spring-boot-starter-data-jpa
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          Actuator
         </strong>
         ：提供应用监控端点（如健康检查、指标收集）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          开箱即用
         </strong>
         ：通过
         <code>
          @SpringBootApplication
         </code>
         一键启动应用。
        </p>
       </li>
       <li>
        <p>
         <strong>
          约定优于配置
         </strong>
         ：减少 XML 和样板代码。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：快速开发微服务或单体应用，适合中小型项目。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      3. Spring Cloud
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        定位
       </strong>
       ：基于 Spring Boot 的
       <strong>
        分布式系统解决方案
       </strong>
       ，提供微服务架构的完整工具链。
      </p>
     </li>
     <li>
      <p>
       <strong>
        核心功能
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          服务注册与发现
         </strong>
         ：Eureka、Consul、Nacos。
        </p>
       </li>
       <li>
        <p>
         <strong>
          配置中心
         </strong>
         ：Spring Cloud Config、Nacos Config。
        </p>
       </li>
       <li>
        <p>
         <strong>
          服务调用
         </strong>
         ：Feign（声明式 HTTP 客户端）、RestTemplate + Ribbon（负载均衡）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          熔断与限流
         </strong>
         ：Hystrix、Sentinel。
        </p>
       </li>
       <li>
        <p>
         <strong>
          API 网关
         </strong>
         ：Spring Cloud Gateway、Zuul。
        </p>
       </li>
       <li>
        <p>
         <strong>
          分布式追踪
         </strong>
         ：Sleuth + Zipkin。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          微服务生态
         </strong>
         ：集成 Netflix OSS、Alibaba 等组件。
        </p>
       </li>
       <li>
        <p>
         <strong>
          标准化解决方案
         </strong>
         ：提供分布式系统的通用模式实现。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：构建大型分布式系统（如微服务架构）。
      </p>
     </li>
    </ul>

    <h4>
     <strong>
      实际应用中的选择
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        单体应用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         简单场景：直接使用
         <strong>
          Spring Boot
         </strong>
         （快速启动）。
        </p>
       </li>
       <li>
        <p>
         复杂场景：结合
         <strong>
          Spring
         </strong>
         +
         <strong>
          Spring Boot
         </strong>
         （精细控制配置）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        微服务架构
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          Spring Boot
         </strong>
         作为单个微服务的开发框架。
        </p>
       </li>
       <li>
        <p>
         <strong>
          Spring Cloud
         </strong>
         提供分布式能力（如服务发现、配置中心、熔断）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        传统企业应用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         使用
         <strong>
          Spring
         </strong>
         框架，结合 XML 或 Java Config 管理 Bean。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Spring
       </strong>
       是基础，提供核心功能（IoC、AOP）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Spring Boot
       </strong>
       是 Spring 的“加速器”，简化配置和部署。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Spring Cloud
       </strong>
       是 Spring Boot 的“扩展包”，专为分布式系统设计。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      三者并非替代关系，而是逐层增强
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       从
       <strong>
        Spring
       </strong>
       →
       <strong>
        Spring Boot
       </strong>
       ：解决开发效率问题。
      </p>
     </li>
     <li>
      <p>
       从
       <strong>
        Spring Boot
       </strong>
       →
       <strong>
        Spring Cloud
       </strong>
       ：解决分布式架构问题。
      </p>
     </li>
    </ul>
