    <p>
     在 C++ 中实现图的存储时，常用的方法包括
     <strong>
      邻接矩阵（Adjacency Matrix）
     </strong>
     、
     <strong>
      邻接表（Adjacency List）

    <hr/>
    <h4>
     <a id="1_Adjacency_Matrix_4">
     </a>
     <strong>
      1. 邻接矩阵（Adjacency Matrix）
     </strong>
    </h4>
    <h5>
     <a id="_5">
     </a>
     <strong>
      原理
     </strong>
    </h5>
    <ul>
     <li>
      使用二维数组
      <code>
       matrix[u][v]
      </code>
      表示顶点
      <code>
       u
      </code>
      和
      <code>
       v
      </code>
      的连接关系。
     </li>
     <li>
      适用于
      <strong>
       稠密图
      </strong>
      （边数接近顶点数的平方）。
     </li>
     <li>
      <strong>
       无权图
      </strong>
      ：
      <code>
       matrix[u][v] = 1
      </code>
      表示存在边；
      <code>
       0
      </code>
      表示无连接。
     </li>
     <li>
      <strong>
       带权图
      </strong>
      ：
      <code>
       matrix[u][v] = weight
      </code>
      表示边的权重。
     </li>
    </ul>

    <h5>
     <a id="_46">
     </a>
     <strong>
      优点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       快速判断两顶点是否相邻
      </strong>
      ：时间复杂度 O(1)。
     </li>
     <li>
      <strong>
       适合频繁查询边的存在性
      </strong>
      。
     </li>
    </ul>
    <h5>
     <a id="_50">
     </a>
     <strong>
      缺点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       空间复杂度高
      </strong>
      ：O(V²)，不适合顶点数多（如 V &gt; 1e4）的稀疏图。
     </li>
     <li>
      <strong>
       插入/删除边效率低
      </strong>
      ：需要修改二维数组。
     </li>
    </ul>

    <h4>
     <a id="2_Adjacency_List_56">
     </a>
     <strong>
      2. 邻接表（Adjacency List）
     </strong>
    </h4>
    <h5>
     <a id="_57">
     </a>
     <strong>
      原理
     </strong>
    </h5>
    <ul>
     <li>
      为每个顶点维护一个链表或动态数组，存储其邻接顶点。
     </li>
     <li>
      适用于
      <strong>
       稀疏图
      </strong>
      （边数远小于顶点数的平方）。
     </li>
     <li>
      <strong>
       无权图
      </strong>
      ：
      <code>
       adjList[u]
      </code>
      存储
      <code>
       v
      </code>
      的集合。
     </li>
     <li>
      <strong>
       带权图
      </strong>
      ：
      <code>
       adjList[u]
      </code>
      存储
      <code>
       pair&lt;v, weight&gt;
      </code>
      。
     </li>
    </ul>
    <h5>
     <a id="C__63">
     </a>
     <strong>
      C++ 实现
     </strong>
    </h5>

    <h5>
     <a id="_99">
     </a>
     <strong>
      优点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       空间复杂度低
      </strong>
      ：O(V + E)，适合大规模稀疏图。
     </li>
     <li>
      <strong>
       高效遍历邻接顶点
      </strong>
      ：时间复杂度与邻接顶点数成正比。
     </li>
    </ul>
    <h5>
     <a id="_103">
     </a>
     <strong>
      缺点
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       查询边的存在性慢
      </strong>
      ：需要遍历邻接表，时间复杂度 O(degree(u))。
     </li>
    </ul>

    <h4>
     <a id="3_Edge_List_108">
     </a>
     <strong>
      3. 边列表（Edge List）
     </strong>
    </h4>
    <h5>
     <a id="_109">
     </a>
     <strong>
      原理
     </strong>
    </h5>
    <ul>
     <li>
      将图的边存储为
      <code>
       (u, v, weight)
      </code>
      的列表。
     </li>
     <li>
      适用于需要
      <strong>
       按边遍历
      </strong>
      的场景（如 Kruskal 算法求最小生成树）。
     </li>
    </ul>

     <a id="_139">

     <strong>

     </li>
     <li>
      <strong>

     </li>
    </ul>
    <h5>
     <a id="_143">
     </a>
     <strong>
      缺点
     </strong>

    <ul>
     <li>
      <strong>
       查询顶点邻接关系慢
      </strong>
      ：需要遍历整个边列表。
     </li>
    </ul>

    <h4>
     <a id="4_Linked_Forward_Star_148">
     </a>
     <strong>
      4. 链式前向星（Linked Forward Star）
     </strong>
    </h4>
    <h5>
     <a id="_149">
     </a>
     <strong>
      原理
     </strong>
    </h5>
    <ul>
     <li>
      一种紧凑的邻接表实现，通过数组模拟链表，常用于算法竞赛。
     </li>
     <li>
      使用三个数组：
      <code>
       head[]
      </code>
      、
      <code>
       to[]
      </code>
      、
      <code>
       next[]
      </code>
      和
      <code>
       weight[]
      </code>
      。
     </li>
    </ul>
    <h5>
     <a id="C__153">
     </a>
     <strong>
      C++ 实现
     </strong>

     <strong>
      6. 动态图的存储优化
     </strong>
    </h4>
    <ul>
     <li>

      </strong>
      ：使用
      <code>

      的
      <code>
       push_back
      </code>
      动态添加边。
     </li>
     <li>
      <strong>
       删除边的优化
      </strong>
      ：使用链表（如
      <code>
       list
      </code>
      ）或标记法（惰性删除）。
     </li>
    </ul>

    <h4>
     <a id="_210">
     </a>
     <strong>
      总结
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       邻接矩阵
      </strong>
      ：适合稠密图，快速查询边的存在性。
     </li>
     <li>
      <strong>
       邻接表
      </strong>
      ：适合稀疏图，高效遍历邻接顶点（推荐使用
      <code>
       vector&lt;vector&lt;pair&lt;int, int&gt;&gt;&gt;
      </code>
      ）。
     </li>
     <li>
      <strong>
       边列表
      </strong>
      ：适合需要全局处理边的场景（如 Kruskal 算法）。
     </li>
     <li>
      <strong>
       链式前向星
      </strong>
      ：适合算法竞赛中的高性能需求。
     </li>
    </ul>
    <p>
     <strong>
      代码建议
     </strong>
     ：大多数情况下优先使用
     <strong>
      邻接表
     </strong>
     ，结合 C++ 的
     <code>
      vector
     </code>
     和
     <code>
      pair
     </code>
     实现带权图的高效存储。
    </p>
