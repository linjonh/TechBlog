    <h4>
     说明：
    </h4>
    <ol>
     <li>
      <strong>
       API端点和认证
      </strong>
      ：
      <ul>
       <li>
        <code>
         api_url
        </code>
        ：电商平台API的端点URL。
       </li>
       <li>
        <code>
         api_key
        </code>
        ：用于认证的API密钥（假设使用Bearer令牌认证）。
       </li>
       <li>
        <code>
         headers
        </code>
        ：HTTP请求头，包含认证信息。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       发送请求
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         requests.get()
        </code>
        方法发送GET请求到API端点，并传入认证头。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       处理响应
      </strong>
      ：
      <ul>
       <li>
        检查响应状态码是否为200。
       </li>
       <li>
        如果是200，则解析响应数据为JSON格式。
       </li>
       <li>
        如果不是200，则尝试解析错误响应中的JSON数据（如果API在错误时返回JSON），或者直接打印响应文本。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       数据处理
      </strong>
      ：
      <ul>
       <li>
        在本例中，遍历解析后的产品数据列表，并以缩进的JSON格式打印每个产品信息。
       </li>
       <li>
        你也可以根据需要进行其他处理，如筛选特定字段、计算统计数据等。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       数据保存
      </strong>
      ：
      <ul>
       <li>
        将解析后的数据保存到JSON文件中，以便后续处理或分析。
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     注意事项：
    </h4>
    <ul>
     <li>
      <strong>
       API限制
      </strong>
      ：电商平台API通常有请求频率限制，请确保遵守这些限制。
     </li>
     <li>
      <strong>
       认证和授权
      </strong>
      ：确保你有正确的认证和授权信息，并且了解API的认证机制。
     </li>
     <li>
      <strong>
       错误处理
      </strong>
      ：添加适当的错误处理逻辑，以处理可能的网络问题、API变更或认证失败等情况。
     </li>
     <li>
      <strong>
       数据隐私
      </strong>
      ：处理电商平台数据时，请遵守相关的数据隐私和合规要求。
     </li>
    </ul>
