    <h2>
     <span style="color:#fe2c24">
      一、电路交换：
     </span>
    </h2>
    <h5>
     <strong>
      核心思想
     </strong>
    </h5>
    <p>
     在通信前
     <strong>
      建立一条专用物理路径
     </strong>
     （电路），
     <span style="color:#fe2c24">
      整个
     </span>
     通信过程中
     <span style="color:#fe2c24">
      独占该路径
     </span>
     ，
     <span style="color:#fe2c24">
      结束后释放资源
     </span>
     。
    </p>
    <h5>
     <strong>
      特点
     </strong>
    </h5>
    <p>
     <strong>
      1.建立连接（尝试占用通信资源）
     </strong>
    </p>
    <p>
     <strong>
      2.通信（一直占用通信资源）
     </strong>
    </p>
    <p>
     <strong>
      3.释放连接（归还通信资源）
     </strong>
    </p>
    <p>
     <strong>
      优点：
     </strong>
    </p>
    <ol>
     <li>
      • 实时性强（如语音通话、视频会议）。
     </li>
     <li>
      • 数据传输顺序和完整性有保障。
     </li>
    </ol>
    <p>
     缺点：
    </p>
    <ol>
     <li>
      • 资源浪费（空闲时链路无法被其他用户使用）。
     </li>
     <li>
      • 连接建立/释放时间长（不适合突发性数据传输）。
     </li>
     <li>
      线路分配的灵活性差。
     </li>
     <li>
      节点交换不支持“差错控制”（无法发现传输过程中的发生的数据错误）。
     </li>
    </ol>
    <h3>
     <span style="color:#fe2c24">
      <strong>
       二、报文交换：
      </strong>
     </span>
     （Message Switching）
     <br/>
     <strong>
      核心思想
     </strong>
    </h3>
    <p>
     <strong>
      <span style="color:#511b78">
       将整个数据报文作为独立单元进行存储-转发（Store-and-Forward），每个节点接收完整报文后再转发至下一跳。
      </span>
     </strong>
     <br/>
     特点
    </p>
    <ol>
     <li>
      • 无专用路径：
      <span style="color:#fe2c24">
       无需预先建立连接
      </span>
      ，按需动态分配路径。
     </li>
     <li>
      • 存储转发机制：每个节点缓存整个报文，可能导致较高延迟。
     </li>
     <li>
      • 灵活性高：报文可动态选择路径，链路利用率较高。
     </li>
     <li>
      • 大报文问题：长报文会长时间占用链路，影响其他数据传输。
     </li>
    </ol>
    <p>
     优点
    </p>
    <ol>
     <li>
      • 链路利用率高（共享资源）。
     </li>
     <li>
      • 支持优先级管理（按需处理报文）。
     </li>
    </ol>
    <p>
     缺点
    </p>
    <ol>
     <li>
      • 延迟高（存储-转发机制）。
     </li>
     <li>
      • 节点需要大缓存空间存储完整报文。
     </li>
     <li>
      • 可靠性较低（报文可能丢失或损坏）。
     </li>
     <li>
      报文不定长，不方便存储转发。
     </li>
    </ol>
    <p>
     典型应用
     <br/>
     早期电报网络、电子邮件系统（但现代网络已不常用）。
    </p>
    <h2>
     <strong>
      <span style="color:#fe2c24">
       三：分组交换：
      </span>
     </strong>
    </h2>
    <h5>
     <strong>
      核心思想
     </strong>
    </h5>
    <p>
     将数据分割为
     <strong>
      固定大小的分组（Packet）
     </strong>
     ，每个分组独立传输，通过不同路径到达目的地后重组。
    </p>
    <h5>
     <strong>
      特点
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        无连接（数据报）或虚电路
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          数据报
         </strong>
         （如IP协议）：每个分组独立路由，可能乱序到达。
        </p>
       </li>
       <li>
        <p>
         <strong>
          虚电路
         </strong>
         （如ATM）：逻辑连接建立后分组按固定路径传输。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        动态路由
       </strong>
       ：分组可选择不同路径，提高链路利用率。
      </p>
     </li>
     <li>
      <p>
       <strong>
        高效灵活
       </strong>
       ：适合突发性数据传输（如互联网流量）。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      优点
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        资源利用率高
       </strong>
       （共享链路，无独占）。
      </p>
     </li>
     <li>
      <p>
       容错性强（故障时可动态切换路径）。
      </p>
     </li>
     <li>
      <p>
       支持多类型数据传输（文本、视频、音频等）。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      缺点
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       分组可能乱序或丢失，需重组和纠错机制（如TCP协议）。
      </p>
     </li>
     <li>
      <p>
       存在传输延迟抖动（不适合严格实时应用）。
      </p>
     </li>
    </ul>
