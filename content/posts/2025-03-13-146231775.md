    <h5>
     <a id="1_utf8mb4_general_ciCaseInsensitive_8">
     </a>
     <strong>
      (1)
      <code>
       utf8mb4_general_ci
      </code>
      （Case-Insensitive，不区分大小写）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       CI
      </strong>
      代表 “Case Insensitive”，表示比较时
      <strong>
       不区分大小写
      </strong>
      ，也
      <strong>
       不区分重音符号
      </strong>
      。
     </li>
     <li>
      例如：
      <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token string">'a'</span> <span class="token operator">=</span> <span class="token string">'A'</span><span class="token punctuation">;</span>  <span class="token comment">-- 结果是 TRUE</span>

</code></pre>
      <code>
       utf8mb4_general_ci
      </code>
      主要适用于
      <strong>
       对大小写或重音符号不敏感的文本
      </strong>
      （如用户名、标签等）。
     </li>
    </ul>
    <h5>
     <a id="2_utf8mb4_binBinary_17">
     </a>
     <strong>
      (2)
      <code>
       utf8mb4_bin
      </code>
      （Binary，区分大小写）
     </strong>
    </h5>
    <ul>
     <li>
      <code>
       utf8mb4_bin
      </code>
      按照
      <strong>
       二进制编码
      </strong>
      逐字节进行比较，因此大小写、重音符号完全不同。
     </li>
     <li>
      例如：
      <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token string">'a'</span> <span class="token operator">=</span> <span class="token string">'A'</span> <span class="token keyword">COLLATE</span> utf8mb4_bin<span class="token punctuation">;</span> <span class="token comment">-- 结果是 FALSE</span>
<span class="token keyword">SELECT</span> <span class="token string">'résumé'</span> <span class="token operator">=</span> <span class="token string">'resume'</span> <span class="token keyword">COLLATE</span> utf8mb4_bin<span class="token punctuation">;</span> <span class="token comment">-- 结果是 FALSE</span>
</code></pre>
      <code>
       utf8mb4_bin
      </code>
      适用于
      <strong>
       严格区分大小写的场景
      </strong>
      （如密码、文件名、唯一标识符）。
     </li>
    </ul>

    <h4>
     <a id="2__utf8mb4_general_ci__86">
     </a>
     <strong>
      2. 为什么
      <code>
       utf8mb4_general_ci
      </code>
      不适合中文？
     </strong>
    </h4>
    <p>
     <code>
      utf8mb4_general_ci
     </code>
     <strong>
      主要是针对拉丁字符优化
     </strong>
     ，它对汉字的处理方式较为简单，可能会导致
     <strong>
      排序不符合中文习惯
     </strong>
     ，例如：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token string">'中'</span> <span class="token operator">&gt;</span> <span class="token string">'啊'</span> <span class="token keyword">COLLATE</span> utf8mb4_general_ci<span class="token punctuation">;</span>
</code></pre>
    <ul>
     <li>
      <code>
       utf8mb4_general_ci
      </code>
      可能会
      <strong>
       按照字符编码顺序
      </strong>
      排序，而不是按拼音或部首排序，这样会出现
      <strong>
       “中” 排在 “啊” 之前
      </strong>
      ，这并不符合常见的中文排序规则。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_utf8mb4_unicode_ci__95">
     </a>
     <strong>
      3.
      <code>
       utf8mb4_unicode_ci
      </code>
      更适合中文
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       utf8mb4_unicode_ci
      </code>
      遵循 Unicode 标准，能够正确地比较和排序
      <strong>
       汉字、拼音、部首
      </strong>
      ，在处理中文时比
      <code>
       utf8mb4_general_ci
      </code>
      <strong>
       更准确
      </strong>
      。
     </li>
     <li>
      例如：
      <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token string">'中'</span> <span class="token operator">&gt;</span> <span class="token string">'啊'</span> <span class="token keyword">COLLATE</span> utf8mb4_unicode_ci<span class="token punctuation">;</span>
</code></pre>
      在
      <strong>
       <code>
        utf8mb4_unicode_ci
       </code>
      </strong>
      规则下，“啊” 会排在 “中” 之前，符合拼音排序规则（
      <code>
       A
      </code>
      在
      <code>
       Z
      </code>
      之前）。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="4_utf8mb4_unicode_520_ci__Unicode__105">
     </a>
     <strong>
      4.
      <code>
       utf8mb4_unicode_520_ci
      </code>
      适用于最新 Unicode 标准
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        utf8mb4_unicode_520_ci
       </code>
      </strong>
      是
      <strong>
       Unicode 5.2
      </strong>
      版本的排序规则，它在
      <code>
       utf8mb4_unicode_ci
      </code>
      的基础上
      <strong>
       进一步优化了字符排序
      </strong>
      ，对于现代 Unicode 处理更完善。
     </li>
     <li>
      例如：
      <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token string">'𠜎'</span> <span class="token operator">&gt;</span> <span class="token string">'中'</span> <span class="token keyword">COLLATE</span> utf8mb4_unicode_520_ci<span class="token punctuation">;</span>
</code></pre>
      在
      <code>
       utf8mb4_unicode_520_ci
      </code>
      下，
      <strong>
       可以正确处理扩展汉字
      </strong>
      ，避免某些生僻字排序错误。
     </li>
    </ul>

        </strong>
       </td>
       <td>

         utf8mb4_unicode_ci
        </code>

      </tr>

        </code>
       </td>
      </tr>
     </tbody>
    </table>

    <h4>
     <a id="6__125">
     </a>
     <strong>
      6. 结论
     </strong>
    </h4>
    <p>
     ✅
     <strong>
      如果你存储中文，并且希望排序、比较符合 Unicode 规范，推荐
     </strong>
     ：
    </p>
    <pre><code class="prism language-sql">utf8mb4_unicode_ci
</code></pre>
    <p>
     ✅
     <strong>
      如果你希望使用最新 Unicode 5.2 标准，并支持更多生僻字，推荐
     </strong>
     ：
    </p>
    <pre><code class="prism language-sql">utf8mb4_unicode_520_ci
</code></pre>
    <p>
     ⚠️
     <strong>
      避免
      <code>
       utf8mb4_general_ci
      </code>
      ，因为它不适用于中文排序
     </strong>
     🚫
    </p>
