    <h3>
     <a id="part1_5">
     </a>
     part1
    </h3>
    <p>
     在取样器错误后要执行的动作：
    </p>
    <ol>
     <li>
      继续：继续执行接下来的操作
     </li>
     <li>
      启动下一进程循环:忽略错误，执行下一个循环
     </li>
     <li>
      停止线程：退出当前线程，不再执行任务
     </li>
     <li>
      停止测试：等待当前执行的采样器结果，解这结束测试
     </li>
     <li>
      立即停止测试：直接停止整个测试
      <br/>
      默认选择continue
     </li>
    </ol>
    <h3>
     <a id="part2_15">
     </a>
     part2
    </h3>
    <ol>
     <li>
      线程数：并发用户数
     </li>
     <li>
      Ramp-Up time:与线程数关联。如线程数为10，ramp-up time为5表示5s启动10个线程，即1s启动2个线程
     </li>
     <li>
      循环次数：要循环的次数，勾选表示一直执行（当part3里面配置了持续时间，则会在持续时间后停止）
     </li>
     <li>
      same user on each iteration:表示每个迭代都用相同的线程：比如线程数为1，循环次数为5，那么创建好1个线程后，每次循环迭代都用这个线程，直到测试结束。如果不勾选，那么每次循环会销毁之前的线程，再创建1个新的线程，测试结束总共会经过5次创建销毁线程，浪费了计算机资源，线程多了甚至可能会影响性能测试结果，所以默认是勾选的。再比如这1个线程中有登录操作，加入cookie后，这个线程5次迭代下来用的都是这一个cookie
     </li>
    </ol>
    <h3>
     <a id="part3_21">
     </a>
     part3
    </h3>
    <ol>
     <li>
      延迟创建线程知道需要：默认启动即创建所有线程：勾选则运行对应线程时才创建（对外无体现）
     </li>
     <li>
      调度器：勾选后可输入持续时间和启动延迟
     </li>
    </ol>
