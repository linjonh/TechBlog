    <h2>
     <a id="cat_5">
     </a>
     cat
    </h2>
    <p>
     cat 命令在 Linux 和 Unix 系统中非常常用，它用于连接文件并打印到标准输出设备（通常是屏幕）。虽然 cat 的基本用法很简单，但它也支持一些参数来提供额外的功能。
    </p>
    <ul>
     <li>
      <p>
       -n 或
       <code>
        --number
       </code>
       ：对所有输出的行进行编号。
       <br/>
       示例：cat -n file.txt
       <br/>
       这会显示 file.txt 的内容，并在每行的开头加上行号。
      </p>
     </li>
     <li>
      <p>
       -b 或
       <code>
        --number-nonblank
       </code>
       ：仅对非空行进行编号。
       <br/>
       示例：cat -b file.txt
       <br/>
       这同样会显示 file.txt 的内容，但仅对非空行（即包含文本的行）进行编号。
      </p>
     </li>
     <li>
      <p>
       -s 或
       <code>
        --squeeze-blank
       </code>
       ：压缩连续的空行为一个空行。
       <br/>
       示例：cat -s file.txt
       <br/>
       如果 file.txt 中有多个连续的空行，cat -s 会将它们压缩为一个空行显示。
      </p>
     </li>
     <li>
      <p>
       -E：在每行的末尾显示 $ 符号，以帮助区分空行和包含空格的行。
       <br/>
       示例：cat -E file.txt
       <br/>
       如果行末尾有不可见的字符（如空格或制表符），-E 参数可以帮助识别。
      </p>
     </li>
     <li>
      <p>
       -T 或
       <code>
        --show-tabs
       </code>
       ：将制表符显示为 ^I。
       <br/>
       示例：cat -T file.txt
       <br/>
       这有助于识别文本中的制表符。
      </p>
     </li>
     <li>
      <p>
       -v 或 -
       <code>
        -show-nonprinting
       </code>
       ：显示非打印字符（如制表符和换行符）。对于制表符，默认显示为 ^I；对于换行符，在大多数系统上会显示为 $（但请注意，-v 和 -E 的行为可能因系统而异）。
       <br/>
       示例：cat -v file.txt
      </p>
     </li>
     <li>
      <p>
       -A 或
       <code>
        --show-all
       </code>
       ：显示所有字符，包括非打印字符（如制表符和换行符，后者通常显示为 $ 或 \n）。
       <br/>
       示例：cat -A file.txt
       <br/>
       这是 -vTE 的组合，提供了一个全面的非打印字符视图。
      </p>
     </li>
     <li>
      <p>
       <code>
        --help
       </code>
       ：显示帮助信息，列出 cat 命令的所有参数和简短的描述。
       <br/>
       示例：cat --help
      </p>
     </li>
     <li>
      <p>
       <code>
        --version
       </code>
       ：显示 cat 命令的版本信息。
       <br/>
       示例：cat --version
      </p>
     </li>
    </ul>
    <p>
     注意：并非所有 cat 实现都支持上述所有参数，但大多数现代 Linux 发行版中的 GNU cat 确实支持它们。如果你在使用某个特定的 Unix 系统或旧版本的 Linux，建议查看该系统的 cat 命令手册页（通过运行 man cat 命令）以获取最准确的参数列表和描述。
    </p>

    <h2>
     <a id="more__48">
     </a>
     more 查看整个文件
    </h2>
    <p>
     more 是 Linux 和类 Unix 操作系统中的一个命令行工具，用于分页显示文本文件的内容。
    </p>
    <p>
     当你打开一个非常大的文本文件时，使用 more 命令可以让你逐页或逐行地查看文件内容，而不是一次性将整个文件内容滚动输出到终端，这有助于更好地管理和阅读文件。
    </p>
    <p>
     用法：
     <code>
      more [+数字] [-数字] file
     </code>
    </p>
    <p>
     常用选项
    </p>
    <ul>
     <li>
      -d：显示翻页提示信息（例如，百分比进度）。
     </li>
     <li>
      -f：不折叠长行，即长行会完整显示，不会自动换行。
     </li>
     <li>
      -l：忽略换行符（仅适用于处理非文本文件时）。
     </li>
     <li>
      -p：清除屏幕（类似于 clear 命令）后再显示文件内容。
     </li>
     <li>
      -s：将多个连续的空行压缩成一行显示。
     </li>
     <li>
      -u：不将下划线字符视为下划线，而是正常显示。
     </li>
     <li>
      +num：
      <strong>
       从文件的第 num 行开始显示
      </strong>
      。
     </li>
     <li>
      -num：
      <strong>
       一次显示 num 行，然后暂停
      </strong>
      。
     </li>
    </ul>
    <p>
     交互命令
    </p>
    <ul>
     <li>
      空格键（Space）：显示下一页，或者
      <code>
       f
      </code>
      。
     </li>
     <li>
      Enter：显示下一行。
     </li>
     <li>
      b：向上翻一页（back）。
     </li>
     <li>
      f：向下翻一页（forward），或者空格。
     </li>
     <li>
      q：退出 more 命令。
     </li>
     <li>
      /：搜索文本字符串。输入你要搜索的字符串，然后按 Enter 键。n 键用于查找下一个匹配项，N 键用于查找上一个匹配项。
     </li>
     <li>
      h：显示帮助信息。
     </li>
     <li>
      v：在当前行启动文本编辑器（通常是 vi）来编辑文件。
     </li>
    </ul>
    <p>
     有些软件安装的时候显示的是该页面，例如 anaconda3
    </p>
    <p>
     more 命令是文本处理的基本工具之一，但对于现代使用场景，less 命令提供了更多的功能和灵活性，因此经常被推荐作为 more 的替代品。
    </p>
    <p>
     当处理非常大的文件时，使用 more 或 less 可以
     <strong>
      避免一次性加载整个文件到内存中，从而提高系统的响应速度
     </strong>
     。
    </p>
    <hr/>
    <h2>
     <a id="less__90">
     </a>
     less 查看整个文件
    </h2>
    <p>
     more 的替代品
    </p>
    <p>
     常用选项
    </p>
    <ul>
     <li>
      -N 或 --LINE-NUMBERS：显示行号。
     </li>
     <li>
      -i 或 --IGNORE-CASE：在搜索时忽略大小写。
     </li>
     <li>
      -F 或 --QUIET 或 --SILENT：在文件内容显示完毕后立即退出，而不是等待用户输入。
     </li>
     <li>
      -q 或 --QUIT：禁止在文件仅有一屏时自动退出。
     </li>
     <li>
      -r 或 --RAW-CONTROL-CHARS：将特殊字符以控制码显示。
     </li>
     <li>
      -S 或 --CHOP-LONG-LINES：禁止自动换行，将过长的行进行截断。
     </li>
     <li>
      -X：禁止清屏。
     </li>
     <li>
      -h 或 --HELP：显示帮助信息。
     </li>
    </ul>
    <p>
     交互命令
    </p>
    <ul>
     <li>
      空格键：向下滚动一屏。
     </li>
     <li>
      Enter：向下滚动一行。
     </li>
     <li>
      b：向上滚动一屏（backward）。
     </li>
     <li>
      y 或 k：向上滚动一行（在某些版本的 less 中，k 也用于向上滚动一行）。
     </li>
     <li>
      G：跳转到文件尾部。
     </li>
     <li>
      g：跳转到文件开头。
     </li>
     <li>
      /：搜索指定的字符串。输入你要搜索的字符串，然后按 Enter 键。搜索结果会高亮显示，n 键用于跳转到下一个匹配项，N 键用于跳转到上一个匹配项。
     </li>
     <li>
      ?：向上搜索关键字（与 / 相反）。
     </li>
     <li>
      q：退出 less 命令。
     </li>
     <li>
      h：在 less 中显示帮助信息。
     </li>
     <li>
      v：在当前行启动默认的文本编辑器（如 vi）来编辑文件。
     </li>
    </ul>

    <h2>
     <a id="head__124">
     </a>
     head 查看部分文件
    </h2>
    <p>
     head 是 Linux 和类 Unix 操作系统中的一个常用命令，它用于显示文本文件的开头部分。
    </p>
    <p>
     基本语法
    </p>
    <pre><code class="prism language-py">head <span class="token punctuation">[</span>选项<span class="token punctuation">]</span> <span class="token punctuation">[</span>文件<span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">]</span>
</code></pre>
    <p>
     常用选项
    </p>
    <ul>
     <li>
      -n NUM 或 --lines=NUM：指定显示文件的前 NUM 行，NUM 为正整数。如果不指定该选项，head 命令默认显示文件的前 10 行。
     </li>
     <li>
      -c NUM 或 --bytes=NUM：指定显示文件的前 NUM 个字节。
     </li>
     <li>
      -q 或 --quiet 或 --silent：不显示文件名。
     </li>
     <li>
      -v 或 --verbose：总是显示文件名，即使只有一个文件作为输入。
     </li>
     <li>
      -z 或 --zero-terminated：将行的结束符改为 NUL 字符，适用于处理包含 NUL 字符的文件。
     </li>
     <li>
      –help：显示 head 命令的帮助信息。
     </li>
     <li>
      –version：显示 head 命令的版本信息。
     </li>
    </ul>
    <p>
     使用示例
    </p>
    <ul>
     <li>
      <p>
       显示文件 file.txt 的前 10 行（默认行为）：
      </p>
      <pre><code class="prism language-py">head <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
     <li>
      <p>
       显示文件 file.txt 的前 5 行：
      </p>
      <pre><code class="prism language-python">head <span class="token operator">-</span>n <span class="token number">5</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
     <li>
      <p>
       显示文件 file.txt 的前 100 个字节：
      </p>
      <pre><code class="prism language-python">head <span class="token operator">-</span>c <span class="token number">100</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
     <li>
      <p>
       不显示文件名，仅显示文件 file.txt 的前 5 行内容：
      </p>
      <pre><code class="prism language-python">head <span class="token operator">-</span>q <span class="token operator">-</span>n <span class="token number">5</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
     <li>
      <p>
       显示文件名和文件 file.txt 的前 5 行内容：
      </p>
      <pre><code class="prism language-python">head <span class="token operator">-</span>v <span class="token operator">-</span>n <span class="token number">5</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
     <li>
      <p>
       处理 NUL 结尾的行，并显示文件 file.txt 的前 5 行：
      </p>
      <pre><code class="prism language-python">head <span class="token operator">-</span>z <span class="token operator">-</span>n <span class="token number">5</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
     </li>
    </ul>
    <p>
     注意事项
    </p>
    <ul>
     <li>
      head 命令通常用于快速查看文件的开头部分，以了解文件的内容或结构。
     </li>
     <li>
      你可以将 head 命令与其他命令结合使用，如管道操作符（|），以实现更复杂的功能。例如，可以使用
      <code>
       cat /var/log/syslog | head
      </code>
      命令查看
      <code>
       /var/log/syslog
      </code>
      文件的前 10 行内容。
     </li>
     <li>
      head 命令不会修改原始文件的内容，它只是读取并显示文件的开头部分。
     </li>
    </ul>

    <h2>
     <a id="tail__191">
     </a>
     tail 查看部分文件
    </h2>
    <p>
     tail 是 Linux 和类 Unix 操作系统中一个非常实用的命令，它主要用于查看文件的末尾部分。
    </p>
    <p>
     基本语法
    </p>
    <pre><code class="prism language-python">tail <span class="token punctuation">[</span>选项<span class="token punctuation">]</span> <span class="token punctuation">[</span>文件<span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">]</span>
</code></pre>
    <p>
     常用选项
    </p>
    <ul>
     <li>
      -n NUM 或
      <code>
       --lines=NUM
      </code>
      ：指定显示文件末尾的 NUM 行，NUM 为正整数。如果不指定该选项，tail 命令默认显示文件末尾的 10 行。
     </li>
     <li>
      -f 或
      <code>
       --follow
      </code>
      ：实时追踪文件的变化，并持续显示新增的内容。这个选项特别适用于查看日志文件等动态更新的文件。
     </li>
     <li>
      -c NUM 或
      <code>
       --bytes=NUM
      </code>
      ：指定显示文件末尾的 NUM 个字节。
     </li>
     <li>
      -q 或
      <code>
       --quiet
      </code>
      或
      <code>
       --silent
      </code>
      ：不显示文件名或处理信息。
     </li>
     <li>
      -v 或
      <code>
       --verbose
      </code>
      ：总是显示文件名和处理信息。
     </li>
     <li>
      <code>
       --pid=PID
      </code>
      ：与 -f 合用，表示在进程 ID 为 PID 的进程终止后结束 tail 命令。
     </li>
     <li>
      -s NUM 或
      <code>
       --sleep-interval=NUM
      </code>
      ：与 -f 合用，表示在每次循环读取之间休眠 NUM 秒。
     </li>
     <li>
      <code>
       --help
      </code>
      ：显示 tail 命令的帮助信息。
     </li>
     <li>
      <code>
       --version
      </code>
      ：显示 tail 命令的版本信息。
     </li>
    </ul>
    <p>
     使用示例
    </p>
    <p>
     显示文件 file.txt 的末尾 10 行（默认行为）：
    </p>
    <pre><code class="prism language-python">tail <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
    <p>
     显示文件 file.txt 的末尾 20 行：
    </p>
    <pre><code class="prism language-python">tail <span class="token operator">-</span>n <span class="token number">20</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
    <p>
     实时追踪文件 logfile.txt 的变化：
    </p>
    <pre><code class="prism language-python">tail <span class="token operator">-</span>f logfile<span class="token punctuation">.</span>txt
</code></pre>
    <p>
     此时，tail 命令会持续运行，并实时显示 logfile.txt 文件中新增的内容。你可以通过按下 Ctrl+C 组合键来中断 tail 命令的运行。
    </p>
    <p>
     显示文件 file.txt 的末尾 50 个字节：
    </p>
    <pre><code class="prism language-python">tail <span class="token operator">-</span>c <span class="token number">50</span> <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
    <p>
     不显示文件名，仅显示文件 file.txt 的末尾 10 行内容：
    </p>
    <pre><code class="prism language-python">tail <span class="token operator">-</span>q <span class="token builtin">file</span><span class="token punctuation">.</span>txt
</code></pre>
    <p>
     在进程 ID 为 1234 的进程终止后结束 tail 命令的实时追踪：
    </p>
    <pre><code class="prism language-python">tail <span class="token operator">-</span><span class="token operator">-</span>pid<span class="token operator">=</span><span class="token number">1234</span> <span class="token operator">-</span>f logfile<span class="token punctuation">.</span>txt
</code></pre>
    <p>
     注意事项
    </p>
    <ul>
     <li>
      tail 命令特别适用于查看日志文件等动态更新的文件，因为它可以实时追踪文件的变化并显示新增的内容。
     </li>
     <li>
      当你使用 -f 选项时，tail 命令会一直运行，直到你手动中断它（如按下 Ctrl+C 组合键）。
     </li>
     <li>
      tail 命令不会修改原始文件的内容，它只是读取并显示文件的末尾部分。
     </li>
    </ul>
    <h2>
     <a id="vim__vi_262">
     </a>
     vim / vi
    </h2>
    <p>
     vim 是 vi 的升级版，兼容性更好，功能更丰富
    </p>
    <p>
     <code>
      vim tmp.py
     </code>
     打开 tmp.py 文件
    </p>
    <p>
     搜索 content 用
     <code>
      :/content
     </code>
    </p>
    <p>
     <code>
      n
     </code>
