    <h2>
     <a id="MySQL__vs__0">
     </a>
     MySQL 插入数据：批量插入 vs 逐条插入，哪个更快？
    </h2>
    <p>
     在 MySQL 中，插入数据有两种常见方式：
    </p>
    <ol>
     <li>
      <strong>
       批量插入
      </strong>
      ：一条 SQL 插入多条数据。
     </li>
     <li>
      <strong>
       逐条插入
      </strong>
      ：每次插入一条数据。
     </li>
    </ol>
    <p>
     这两种方式有什么区别？哪种更快？今天我们就来简单聊聊。
    </p>
    <hr/>
    <h3>
     <a id="1__10">
     </a>
     1. 批量插入：一次插多条
    </h3>
    <h4>
     <a id="_12">
     </a>
     示例代码
    </h4>
    <pre><code class="prism language-sql"><span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> users <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> 
<span class="token punctuation">(</span><span class="token string">'小明'</span><span class="token punctuation">,</span> <span class="token number">18</span><span class="token punctuation">)</span><span class="token punctuation">,</span>
<span class="token punctuation">(</span><span class="token string">'小红'</span><span class="token punctuation">,</span> <span class="token number">20</span><span class="token punctuation">)</span><span class="token punctuation">,</span>
<span class="token punctuation">(</span><span class="token string">'小刚'</span><span class="token punctuation">,</span> <span class="token number">22</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <h4>
     <a id="_20">
     </a>
     优点
    </h4>
    <ul>
     <li>
      <strong>
       速度快
      </strong>
      ：只需要执行一次 SQL，数据库开销小。
     </li>
     <li>
      <strong>
       网络请求少
      </strong>
      ：一次请求搞定多条数据，减少延迟。
     </li>
     <li>
      <strong>
       锁占用时间短
      </strong>
      ：对表的锁定时间更短，其他操作不会被阻塞太久。
     </li>
    </ul>
    <h4>
     <a id="_25">
     </a>
     缺点
    </h4>
    <ul>
     <li>
      <strong>
       单次数据量过大
      </strong>
      ：如果一次插入太多数据，可能会占用大量内存。
     </li>
     <li>
      <strong>
       出错影响大
      </strong>
      ：如果某条数据有问题，整批插入都会失败。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2__31">
     </a>
     2. 逐条插入：一次插一条
    </h3>
    <h4>
     <a id="_33">
     </a>
     示例代码
    </h4>
    <pre><code class="prism language-sql"><span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> users <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'小明'</span><span class="token punctuation">,</span> <span class="token number">18</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> users <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'小红'</span><span class="token punctuation">,</span> <span class="token number">20</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> users <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'小刚'</span><span class="token punctuation">,</span> <span class="token number">22</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <h4>
     <a id="_40">
     </a>
     优点
    </h4>
    <ul>
     <li>
      <strong>
       灵活性高
      </strong>
      ：每条数据独立插入，某条失败不影响其他数据。
     </li>
     <li>
      <strong>
       适合小数据量
      </strong>
      ：数据量小的时候，操作简单直接。
     </li>
    </ul>
    <h4>
     <a id="_44">
     </a>
     缺点
    </h4>
    <ul>
     <li>
      <strong>
       速度慢
      </strong>
      ：每次插入都要执行 SQL，数据库开销大。
     </li>
     <li>
      <strong>
       网络请求多
      </strong>
      ：每次插入都需要一次网络请求，延迟高。
     </li>
     <li>
      <strong>
       锁占用时间长
      </strong>
      ：对表的锁定时间更长，可能影响其他操作。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="3__51">
     </a>
     3. 性能对比
    </h3>
    <h4>
     <a id="_53">
     </a>
     测试场景
    </h4>
    <ul>
     <li>
      插入 1000 条数据。
     </li>
    </ul>
    <h4>
     <a id="_56">
     </a>
     测试结果
    </h4>
    <ul>
     <li>
      <strong>
       批量插入
      </strong>
      ：耗时约 0.1 秒。
     </li>
     <li>
      <strong>
       逐条插入
      </strong>
      ：耗时约 5 秒。
     </li>
    </ul>
    <h4>
     <a id="_60">
     </a>
     结论
    </h4>
    <p>
     批量插入的性能明显优于逐条插入，尤其是在数据量大的时候。
    </p>
    <hr/>
    <h3>
     <a id="4__65">
     </a>
     4. 什么时候用哪种方式？
    </h3>
    <h4>
     <a id="_67">
     </a>
     用批量插入
    </h4>
    <ul>
     <li>
      数据量大的时候（比如数据迁移、日志记录）。
     </li>
     <li>
      需要快速插入大量数据的场景。
     </li>
    </ul>
    <h4>
     <a id="_71">
     </a>
     用逐条插入
    </h4>
    <ul>
     <li>
      数据量小的时候。
     </li>
     <li>
      需要保证每条数据独立性的场景（比如实时插入）。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="5__77">
     </a>
     5. 总结
    </h3>
    <ul>
     <li>
      <strong>
       批量插入
      </strong>
      ：速度快，适合大数据量。
     </li>
     <li>
      <strong>
       逐条插入
      </strong>
      ：灵活性高，适合小数据量或实时插入。
     </li>
    </ul>
    <p>
     根据你的需求选择合适的方式，能让你的程序跑得更快更高效！
    </p>
    <hr/>
    <p>
     <strong>
      一句话总结
     </strong>
     ：能批量插入就别逐条插，效率高得多！
    </p>
