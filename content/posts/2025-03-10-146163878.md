     HTTP拾技杂谈
    </h2>
    <p>
     简单聊聊HTTP中的那些东西

    <hr/>

    <hr/>
    <h2>
     <a id="_10">
     </a>
     前言
    </h2>
    <p>
     超文本传输协议（Hypertext Transfer Protocol ，HTTP）是一个标准，定义了Web客户端如何与服务器对话，以及数据如何从服务器传回客户端。尽管通常任务HTTP是一种传输HTML文件以及文件中内嵌图片的方法，但实际上HTTP是一个数据格式。——《java网络编程》
    </p>
    <hr/>
    <h2>
     <a id="HTTP_16">
     </a>
     HTTP协议
    </h2>
    <p>
     HTTP连接使用TCP/IP来传输数据。
    </p>
    <h3>
     <a id="14_19">
     </a>
     1.请求从客户端到服务器端的4个步骤
    </h3>
    <ol>
     <li>
      默认情况下，客户端在80端口与服务器打开一个TCP连接，URL可以指定其他端口
     </li>
     <li>
      客户端向服务器发送消息，请求指定路径上的资源。请求分为首部、可选地址、请求数据。
     </li>
     <li>
      服务器向客户端回复响应。以响应码开头，后面是包含元数据的首部、一个空行、所请求的文档(数据)或错误消息。
     </li>
     <li>
      服务器关闭连接。
      <br/>
      以上就是基本HTTP1.0过程，在HTTP1.1及以后版本中，可以复用TCP连接实现发送多个请求。在1.1中请求和响应可以分为多个块发送。
     </li>
    </ol>
    <h4>
     <a id="_27">
     </a>
     一般客户端请求如下：
    </h4>
    <pre><code class="prism language-java"><span class="token constant">POST</span> <span class="token operator">/</span>api<span class="token operator">/</span>data <span class="token constant">HTTP</span><span class="token operator">/</span><span class="token number">1.1</span>          # 请求行（方法 <span class="token operator">+</span> <span class="token constant">URI</span> <span class="token operator">+</span> <span class="token constant">HTTP</span>版本）
<span class="token class-name">Host</span><span class="token operator">:</span> www<span class="token punctuation">.</span>example<span class="token punctuation">.</span>com            # 目标主机（必需首部）
<span class="token class-name">User</span><span class="token operator">-</span><span class="token class-name">Agent</span><span class="token operator">:</span> <span class="token class-name">Mozilla</span><span class="token operator">/</span><span class="token number">5.0</span>          # 客户端标识
<span class="token class-name">Content</span><span class="token operator">-</span><span class="token class-name">Type</span><span class="token operator">:</span> application<span class="token operator">/</span>json   # 消息体格式说明
<span class="token class-name">Content</span><span class="token operator">-</span><span class="token class-name">Length</span><span class="token operator">:</span> <span class="token number">27</span>               # 消息体字节长度（必需首部）
<span class="token class-name">Accept</span><span class="token operator">:</span> application<span class="token operator">/</span>json         # 期望的响应格式
<span class="token class-name">Authorization</span><span class="token operator">:</span> <span class="token class-name">Bearer</span> abc123     # 认证令牌
<span class="token class-name">Connection</span><span class="token operator">:</span> keep<span class="token operator">-</span>alive           # 连接控制
                                  #注意这里的空一行是标准
<span class="token punctuation">{<!-- --></span><span class="token string">"key"</span><span class="token operator">:</span> <span class="token string">"value"</span><span class="token punctuation">,</span> <span class="token string">"num"</span><span class="token operator">:</span> <span class="token number">42</span><span class="token punctuation">}</span>      # 消息体（实际传输的数据）

</code></pre>
    <p>
     第一行为请求行，包括方法、从服务器获取资源路径以及HTTP版本。
     <br/>
     其他则是以 Key：Value组成的信息。
     <br/>
     像User-Agent代表浏览器版本，host服务器名，Accept，告诉服务器，客户端可以处理什么样的数据（定义服务器返回格式）。
     <br/>
     这些类型统称为MIME类型，分为两级：类型和子类型。例如：application/json
    </p>
    <pre><code>text/*
image/*
application/*
...等
</code></pre>
    <p>
     当然也可以自定义非标准的定制类型和子类型，只要以x-开头。例如flash文件，使用application/x-shockwave-flash类型。
     <br/>
     最后以空行，或者空行+数据体
    </p>
    <h4>
     <a id="_53">
     </a>
     服务端响应如下
    </h4>
    <pre><code class="prism language-java"><span class="token constant">HTTP</span><span class="token operator">/</span><span class="token number">1.1</span> <span class="token number">200</span> <span class="token constant">OK</span>                          # 状态行（协议版本 <span class="token operator">+</span> 状态码 <span class="token operator">+</span> 原因短语）
<span class="token class-name">Server</span><span class="token operator">:</span> nginx<span class="token operator">/</span><span class="token number">1.18</span><span class="token number">.0</span>                     # 服务器软件及版本
<span class="token class-name">Date</span><span class="token operator">:</span> <span class="token class-name">Fri</span><span class="token punctuation">,</span> <span class="token number">13</span> <span class="token class-name">Oct</span> <span class="token number">2023</span> <span class="token number">05</span><span class="token operator">:</span><span class="token number">30</span><span class="token operator">:</span><span class="token number">15</span> <span class="token constant">GMT</span>      # 响应生成时间
<span class="token class-name">Content</span><span class="token operator">-</span><span class="token class-name">Type</span><span class="token operator">:</span> text<span class="token operator">/</span>html<span class="token punctuation">;</span> charset<span class="token operator">=</span><span class="token constant">UTF</span><span class="token operator">-</span><span class="token number">8</span>   # 消息体的格式和编码
<span class="token class-name">Content</span><span class="token operator">-</span><span class="token class-name">Length</span><span class="token operator">:</span> <span class="token number">1234</span>                     # 消息体的字节长度
<span class="token class-name">Connection</span><span class="token operator">:</span> keep<span class="token operator">-</span>alive                   # 连接控制（保持活跃）
<span class="token class-name">Cache</span><span class="token operator">-</span><span class="token class-name">Control</span><span class="token operator">:</span> max<span class="token operator">-</span>age<span class="token operator">=</span><span class="token number">3600</span>              # 缓存策略（有效期<span class="token number">1</span>小时）
<span class="token class-name">ETag</span><span class="token operator">:</span> <span class="token string">"abc123xyz"</span>                        # 资源版本标识符
<span class="token class-name">Last</span><span class="token operator">-</span><span class="token class-name">Modified</span><span class="token operator">:</span> <span class="token class-name">Mon</span><span class="token punctuation">,</span> <span class="token number">09</span> <span class="token class-name">Oct</span> <span class="token number">2023</span> <span class="token number">12</span><span class="token operator">:</span><span class="token number">00</span><span class="token operator">:</span><span class="token number">00</span> <span class="token constant">GMT</span>  # 资源最后修改时间

<span class="token operator">&lt;</span><span class="token operator">!</span><span class="token constant">DOCTYPE</span> html<span class="token operator">&gt;</span>                          # 空行（分隔首部和消息体）
<span class="token generics"><span class="token punctuation">&lt;</span>html<span class="token punctuation">&gt;</span></span>                                   # 消息体（实际返回的内容）
  <span class="token generics"><span class="token punctuation">&lt;</span>head<span class="token punctuation">&gt;</span></span><span class="token generics"><span class="token punctuation">&lt;</span>title<span class="token punctuation">&gt;</span></span><span class="token class-name">Example</span> <span class="token class-name">Page</span><span class="token operator">&lt;</span><span class="token operator">/</span>title<span class="token operator">&gt;</span><span class="token operator">&lt;</span><span class="token operator">/</span>head<span class="token operator">&gt;</span>
  <span class="token generics"><span class="token punctuation">&lt;</span>body<span class="token punctuation">&gt;</span></span><span class="token generics"><span class="token punctuation">&lt;</span>h1<span class="token punctuation">&gt;</span></span><span class="token class-name">Hello</span> <span class="token class-name">World</span><span class="token operator">&lt;</span><span class="token operator">/</span>h1<span class="token operator">&gt;</span><span class="token operator">&lt;</span><span class="token operator">/</span>body<span class="token operator">&gt;</span>
<span class="token operator">&lt;</span><span class="token operator">/</span>html<span class="token operator">&gt;</span>
</code></pre>
    <p>
     第一行表示服务器使用的协议HTTP1.1，紧跟一个响应码。其他首部行则指出请求的日期、消息体字节长度等
    </p>
    <h3>
     <a id="2KeepAlive_72">
     </a>
     2.Keep-Alive
    </h3>
    <p>
     在HTTP请求的首部中大家是否注意到Keep-Alive，他表示可以重用一个socket：
     <br/>
     Connection：Keep-Alice。
     <br/>
     在HTTP1.0会为每个请求打开一个新的连接，而这样打开和关闭消耗的资源远远大于实际传输数据的开销。所以在HTTP1.1以后就默认支持，复用socket连接，如果需要关闭则使用上述key-value显示控制。
    </p>
    <ul>
     <li>
      在java中可以通过http.keepAlive控制true/false，来控制socket复用
     </li>
     <li>
      http.maxConnections可以表示希望同时打开的socket的数量，来保证服务器不会被大量连接冲崩溃。
     </li>
    </ul>
    <h3>
     <a id="HTTP_80">
     </a>
     HTTP方法
    </h3>
    <p>
     HTTP服务器的遵循请求-响应模式：无状态请求+无状态响应。
     <br/>
     HTTP的方法主要有4个：
     <br/>
     POST、GET、PUT、DELETE
     <br/>
     对于这些方法的使用，再谈谈几点容易被忽略的细节：
    </p>
    <ul>
     <li>
      GET方法获取一个资源，它没有副作用，如果失败可以反复执行，而不用担心有什么问题
     </li>
     <li>
      GET输出可能会被缓存，可以通过首部取消缓存进行禁用
     </li>
     <li>
      PUT方法表示将资源上传到服务器，但它具有幂等性。重复该方法也不用担心是否失败，两次将同一个文档放在同一个位置并不影响。
     </li>
     <li>
      DELETE与PUT一样具有幂等性，重复执行，也只会将资源删除一次。
     </li>
     <li>
      POST是最常用的方法，POST要用于不能重复的不安全的操作，如完成一个交易。
     </li>
    </ul>
    <h3>
     <a id="Cookie_92">
     </a>
     Cookie
    </h3>
    <p>
     很多网站使用一些
     <strong>
      小文本串
     </strong>
     在连接之间存储
     <strong>
      持久的客户端状态
     </strong>
     ，这些称为cookie。
     <br/>
     cookie在请求和响应的HTTP首部中，由服务器传递到客户端，再从客户端传回服务器。
     <br/>
     cookie可以是标识会话ID、购物车、登录凭据等信息，也可以存无意义的值。
     <br/>
     要在浏览器设置一个cookie，则需要服务器端在响应中使用Set-Cookie首部行:
    </p>
    <pre><code class="prism language-java"><span class="token constant">HTTP</span><span class="token operator">/</span><span class="token number">1.1</span> <span class="token number">200</span> <span class="token constant">OK</span>                          # 状态行（协议版本 <span class="token operator">+</span> 状态码 <span class="token operator">+</span> 原因短语）
<span class="token class-name">Content</span><span class="token operator">-</span><span class="token class-name">Type</span><span class="token operator">:</span> text<span class="token operator">/</span>html<span class="token punctuation">;</span> charset<span class="token operator">=</span><span class="token constant">UTF</span><span class="token operator">-</span><span class="token number">8</span>   # 消息体的格式和编码
<span class="token class-name">Set</span><span class="token operator">-</span><span class="token class-name">Cookie</span><span class="token operator">:</span> xxx<span class="token operator">=</span>dsahjfkeqyowqlf   # 设置cookie
</code></pre>
    <p>
     而后，客户端再向同一个服务发送请求：
    </p>
    <pre><code class="prism language-java"><span class="token constant">GET</span> <span class="token operator">/</span>index<span class="token punctuation">.</span>html <span class="token constant">HTTP</span><span class="token operator">/</span><span class="token number">1.1</span>          # 请求行（方法 <span class="token operator">+</span> <span class="token constant">URI</span> <span class="token operator">+</span> <span class="token constant">HTTP</span>版本）
<span class="token class-name">Host</span><span class="token operator">:</span> www<span class="token punctuation">.</span>example<span class="token punctuation">.</span>com            # 目标主机（必需首部）
<span class="token class-name">Cookie</span><span class="token operator">:</span>cart <span class="token operator">=</span> dsahjfkeqyowqlf   #cookie要与服务器给的一致
</code></pre>
    <p>
     当然服务器可以设置不止一个cookie，可以有多个Set-Cookie，用于控制过期时间、路径、端口、值等信息。
    </p>
    <p>
     默认情况下，cookie来自哪个服务器则应用于哪个服务器。网站也可以指示一个cookie用于整个子域，例如从www.foo.example.com响应的：
     <br/>
     Set-Cookie：user = elharo;Domain=.foo.example.com
     <br/>
     表示浏览器除了可以用cookie返回最初的www.foo.example.com，还可发送给*.foo.example.com
     <br/>
     当然还可以缩小它的使用域
     <br/>
     例如：Set-Cookie: user=elharo; Path=/restricted，表示请求服务器上子树/restricted,可以。而相同网站的其他目录中就不能使用这个cookie。
     <br/>
     在Set-Cookie中使用expires = Wdy, DD-Mon-YYYY HH:MM:SS GMT 可以设置cookie的过期时间。
     <br/>
     设置Max-Age = 3600 ，则表示3600秒后过期，此时浏览器会从缓存中删除这些cookie。
    </p>
    <hr/>
    <h2>
     <a id="_121">
     </a>
     总结
    </h2>
    <p>
     本章节介绍了HTTP协议是现代Web通信的核心，其设计简洁高效，支持多种请求方法和数据格式，并通过Cookie机制实现了客户端状态的持久化。随着HTTP版本的演进，连接复用等优化机制进一步提升了通信性能。本文详细介绍了HTTP协议的基本原理、请求响应格式、方法特性以及Cookie机制。
    </p>
