     遍历聚合对象的统一方式
    </h3>
    <hr/>
    <h3>
     <a id="1__4">
     </a>
     1. 模式定义与用途
    </h3>
    <h4>
     <a id="_5">
     </a>
     核心思想
    </h4>
    <ul>
     <li>
      ​
      <strong>
       迭代器模式
      </strong>
      ：提供一种方法顺序访问聚合对象的元素，而无需暴露其内部表示。
     </li>
     <li>
      <strong>
       关键用途
      </strong>
      ：
      <br/>
      1.​
      <strong>
       统一遍历接口
      </strong>
      ：为不同数据结构（如数组、链表、树）提供一致的遍历方式。
      <br/>
      ​2.
      <strong>
       支持多种遍历策略
      </strong>
      ：前向、反向、条件过滤等。
      <br/>
      ​3.
      <strong>
       简化聚合类设计
      </strong>
      ：将遍历逻辑从聚合类中分离。
     </li>
    </ul>
    <h4>
     <a id="_11">
     </a>
     经典场景
    </h4>
    <ul>
     <li>
      STL容器的迭代器（如std::vector::iterator）。
     </li>
     <li>
      自定义集合类（如链表、图）的遍历。
     </li>
     <li>
      数据库查询结果的逐行遍历。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2__16">
     </a>
     2. 模式结构解析
    </h3>
    <h4>
     <a id="UML_17">
     </a>
     UML类图
    </h4>
    <pre><code class="prism language-plaintext">+---------------------+          +---------------------+  
|      Aggregate      |          |       Iterator       |  
+---------------------+          +---------------------+  
| + createIterator()  |&lt;&gt;-------&gt;| + next(): void       |  
+---------------------+          | + hasNext(): bool    |  
          ^                      +---------------------+  
          |                                ^  
          |                                |  
  +-------+-------+              +---------+---------+  
  |               |              |                   |  
+---------------------+    +-------------------+ +----------------+  
| ConcreteAggregate   |    |  ConcreteIterator | |     Client      |  
+---------------------+    +-------------------+ +----------------+  
| + createIterator()  |    | + next()          | | 通过迭代器遍历聚合对象 |  
+---------------------+    | + hasNext()       | +----------------+  
                           +-------------------+  
</code></pre>
    <h4>
     <a id="_36">
     </a>
     角色说明
    </h4>
    <ol>
     <li>
      <code>
       Aggregate
      </code>
      ：聚合接口，定义创建迭代器的方法（如
      <code>
       createIterator()
      </code>
      ）。
     </li>
     <li>
      ​
      <code>
       ConcreteAggregate
      </code>
      ：具体聚合类，实现迭代器创建逻辑。
     </li>
     <li>
      <code>
       Iterator
      </code>
      ：迭代器接口，定义遍历方法（如
      <code>
       next()
      </code>
      、
      <code>
       hasNext()
      </code>
      ）。
     </li>
     <li>
      <code>
       ConcreteIterator
      </code>
      ：具体迭代器，实现特定遍历逻辑。
     </li>
     <li>
      <code>
       Client
      </code>
      ：通过迭代器访问聚合对象，无需依赖其内部结构。
     </li>
    </ol>
