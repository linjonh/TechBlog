 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="./../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="./../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     <strong>
      设置全局变量 main.ts
     </strong>
    </p>
    <blockquote>
     <p>
      app.config.globalProperties.$category = { index: 0 }
     </p>
    </blockquote>
    <p>
     <strong>
      获取全局变量
     </strong>
    </p>
    <blockquote>
     <p>
      const { appContext } = getCurrentInstance() as ComponentInternalInstance
     </p>
     <p>
      console.log(appContext.config.globalProperties.$category)
     </p>
    </blockquote>
    <p>
     或是
    </p>
    <blockquote>
     <p>
      const { proxy } = getCurrentInstance() as ComponentInternalInstance
     </p>
     <p>
      console.log(proxy?.$category)
     </p>
    </blockquote>
    <p>
     第二种方法如果遇到 类型ComponentPublicInstance上不存在属性“$category”问题，解决办法就是添加一个 .d.ts声明文件，添加全局挂载属性：
    </p>
    <pre><code class="hljs">declare module '@vue/runtime-core' {
  interface ComponentCustomProperties {
    $category: object
  }
}</code></pre>
    <p>
     <strong>
      还有一种方法小程序有
      <a class="link-info" href="https://uniapp.dcloud.net.cn/collocation/App.html#globaldata" rel="nofollow" title="globalData">
       globalData
      </a>
      ，这是一种简单的全局变量机制。这套机制在uni-app里也可以使用，并且全端通用。
     </strong>
    </p>
    <p>
     首页 index.vue中设置值
    </p>
    <p>
     getApp().globalData = { ll: 'sd' }
    </p>
    <p>
     使用的页面中获取值
    </p>
    <p>
     console.log(getApp().globalData)
    </p>
