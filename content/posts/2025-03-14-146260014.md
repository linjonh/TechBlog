     <a id="Git_0">

     配置Git
    </h2>
    <h3>
     <a id="_1">
     </a>
     忽略特殊文件
    </h3>
    <p>
     在日常开发中，有些文件不想或不应该提交到远端，如保存数据库密码的配置文件。
    </p>
    <p>
     在Git工作区的根目录下创建一个特殊的.gitignore文件，把要忽略的文件名填进去，Git就会自动忽略这些文件。
    </p>
    <p>
     不需要从头写
     <code>
      .gitignore
     </code>
     文件，gitee在创建仓库时可为我们生成，但需要主动勾选：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/84cd3707b63c47b69952d2ec7125c93f.png"/>
    </p>
    <p>
     如果当时没有选择这个选择，在工作区创建一个也是可以的。
    </p>
    <p>
     无论哪种方式，最终都可以得到一个完整的
     <code>
      .gitignore
     </code>
     文件，例如我们想忽略以
     <code>
      .so
     </code>
     和
     <code>
      .ini
     </code>
     结尾所有⽂件，
     <code>
      .gitignore
     </code>
     的内容如下：
    </p>
    <pre><code># 省略选择模板的内容
...
# My configurations：
*.ini
*.so
</code></pre>
    <p>
     在.gitignore文件中也可以指定某个确定的文件。
    </p>
    <p>
     最后，把.gitignore提交到远端：
    </p>
    <pre><code class="prism language-bash">$ vim.gitignore
$ <span class="token function">git</span> add.

    <p>
     接着验证
     <code>
      .gitignore
     </code>
     文件的能力，在工作区新增两个文件a.so b.ini：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">touch</span> a.so b.ini

$ <span class="token function">git</span> status
On branch master

</code></pre>
    <p>
     检验
     <code>
      .gitignore
     </code>
     的标准就是
     <code>
      git status
     </code>
     命令显示
     <code>
      working tree clean
     </code>
     。
    </p>
    <p>
     若想强制添加被忽略的文件，可用
     <code>
      -f
     </code>
     强制添加：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">git</span> <span class="token function">add</span> <span class="token parameter variable">-f</span> <span class="token punctuation">[</span>filename<span class="token punctuation">]</span>
</code></pre>
    <p>
     若怀疑.gitignore写得有问题，可用git check-ignore命令检查，如检查a.so文件：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">git</span> check-ignore <span class="token parameter variable">-v</span> a.so
.gitignore:3:*.so         a.so
</code></pre>
    <p>
     当编写规则排除部分文件（如排除所有.开头的隐藏文件：.* ），却发现
     <code>
      .gitignore
     </code>
     也被排除，可添加例外规则：
    </p>
    <pre><code># 排除所有.开头的隐藏文件：
.*

# 不排除.gitignore
!.gitignore
</code></pre>
    <p>
     把指定文件排除
     <mark>
      在.gitignore规则外的写法是
      <code>
       !+文件名
      </code>
      。
     </mark>
    </p>
    <h3>
     <a id="_80">
     </a>
     给命令配置别名
    </h3>
    <p>
     使用Git时，有些命令较长，git支持对命令进行简化。
    </p>
    <ul>
     <li>
      将git status简化为git st，对应的命令为：
     </li>
    </ul>
    <pre><code class="prism language-bash">$ <span class="token function">git</span> config <span class="token parameter variable">--global</span> alias.st status
</code></pre>
    <p>
     –global参数是全局参数，这些命令在这台电脑的所有Git仓库都有用。不加，则只针对当前仓库起作用。
     <br/>
     使用git st的效果：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">git</span> st
On branch master
Your branch is up to <span class="token function">date</span> with <span class="token string">'origin/master'</span><span class="token builtin class-name">.</span>

nothing to commit, working tree clean
</code></pre>
    <ul>
     <li>
      配置一个git last，让其显示最后一次提交信息：
     </li>
    </ul>
    <pre><code class="prism language-bash">$ <span class="token function">git</span> config <span class="token parameter variable">--global</span> alias.last <span class="token string">'log -1'</span>
</code></pre>
    <p>
     使用git last显示最近一次的提交：
    </p>
    <pre><code class="prism language-bash">hyb@139-159-150-152:~/git_teaching$ <span class="token function">git</span> last
