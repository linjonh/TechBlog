    <h2 id="2.%E6%80%9D%E8%B7%AF%E5%88%86%E6%9E%90%C2%A0">
     2.思路分析
    </h2>
    <p>
     <span style="color:#0d0016;">
      <strong>
       说我的rabbitmq连接不上，去配置检查了一下，发现没啥大问题
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016;">
      <strong>
       IP，用户名，密码，端口号都吻合，而且web界面也能进入
      </strong>
     </span>
    </p>
    <p>
     <img alt="" class="left" height="793" src="https://i-blog.csdnimg.cn/blog_migrate/5c15f9dbdd3e467bdb3a73fbcbd751b6.png" width="1200"/>
    </p>
    <h2 id="3.%E8%BF%9B%E5%85%A5%E7%99%BD%E7%83%AD%E5%8C%96%C2%A0">
     3.进入白热化
    </h2>
    <p>
     <strong>
      <span style="color:#0d0016;">
       由于rabbitmq默认用户root是只能允许本地
       <em>
        主机localhost连接的，而
       </em>
       我的rabbitmq是在虚拟机上以
       <a href="https://so.csdn.net/so/search?q=docker&amp;spm=1001.2101.3001.7020" title="docker">
        docker
       </a>
       创建的，属于远程访问，因此连接报错。
      </span>
     </strong>
    </p>
    <ul>
     <li>
      <span style="color:#fe2c24;">
       <strong>
        在web页面添加新用户
       </strong>
      </span>
     </li>
    </ul>
    <p>
     <img alt="" class="left" height="575" src="https://i-blog.csdnimg.cn/blog_migrate/a9fcb89a63ebf3c3dc3b4c98f77cd546.png" width="1127"/>
    </p>
    <ul>
     <li>
      <span style="color:#fe2c24;">
       <strong>
        配置新加的用户信息
       </strong>
      </span>
     </li>
    </ul>
    <p style="text-align:center;">
     <img alt="" class="left" src="https://i-blog.csdnimg.cn/blog_migrate/4849ab362509b8521ba6038b8047e3bd.png"/>
    </p>
    <p>
     但是发现还是不行~，所以不是用户的原因~
    </p>
    <h2 id="4.%E5%85%A8%E6%96%B9%E4%BD%8D%E7%93%A6%E8%A7%A3%E5%AF%B9%E6%89%8B">
     4.全方位瓦解对手
    </h2>
    <pre><code class="language-Groovy">server:
  port: 8801

spring:
  application:
    name: cloud-stream-provider
  cloud:
    stream:
      binders: # 在此处配置要绑定的rabbitmq的服务信息；
        defaultRabbit: # 表示定义的名称，用于于binding整合
          type: rabbit # 消息组件类型
          environment: # 设置rabbitmq的相关的环境配置
            spring:
              rabbitmq:
                host: 192.168.20.129
                port: 5672
                username: xz
                password: 123456
      bindings: # 服务的整合处理
        output: # 这个名字是一个通道的名称
          destination: studyExchange # 表示要使用的Exchange名称定义
          content-type: application/json # 设置消息类型，本次为json，文本则设置“text/plain”
          binder: defaultRabbit # 设置要绑定的消息服务的具体设置

eureka:
  client: # 客户端进行Eureka注册的配置
    service-url:
      defaultZone: http://eureka7001.com:7001/eureka,http://eureka7002.com:7002/eureka,http://eureka7003.com:7003/eureka
  instance:
    lease-renewal-interval-in-seconds: 2 # 设置心跳的时间间隔（默认是30秒）
    lease-expiration-duration-in-seconds: 5 # 如果现在超过了5秒的间隔（默认是90秒）
    instance-id: send-8801.com  # 在信息列表时显示主机名称
    prefer-ip-address: true     # 访问的路径变为IP地址</code></pre>
    <blockquote>
     <ul>
      <li>
       <span style="color:#0d0016;">
        <strong>
         如果你是用的
        </strong>
       </span>
       <span style="color:#fe2c24;">
        <strong>
         其他服务器上
        </strong>
       </span>
       <span style="color:#0d0016;">
        <strong>
         的rabbitmq，比如我是在Linux上用docker直接拉去的，按上面配置方式的话，启动时会
        </strong>
       </span>
       <span style="color:#fe2c24;">
        <strong>
         试图连接两次
        </strong>
       </span>
       <span style="color:#0d0016;">
        <strong>
         rabbitmq。
        </strong>
       </span>
      </li>
      <li>
       <span style="color:#0d0016;">
        <strong>
         程序
        </strong>
       </span>
       <span style="color:#fe2c24;">
        <strong>
         第一次
        </strong>
       </span>
       <span style="color:#0d0016;">
        <strong>
         试图连接访问的就是
        </strong>
       </span>
       <span style="color:#4da8ee;">
        <strong>
         application.yml中配置的地址
        </strong>
       </span>
       <span style="color:#0d0016;">
        <strong>
         ,此时已经订阅成功了
        </strong>
       </span>
       ，
       <span style="color:#0d0016;">
        <strong>
         但是程序还会在之后进行
        </strong>
       </span>
       <span style="color:#fe2c24;">
        <strong>
         第二次
        </strong>
       </span>
       <span style="color:#0d0016;">
        <strong>
         连接,此时访问的地址就是
        </strong>
       </span>
       <span style="color:#4da8ee;">
        <strong>
         localhost:5672
        </strong>
       </span>
      </li>
     </ul>
     <p>
      <span style="color:#0d0016;">
       <strong>
        在我的环境中,我
       </strong>
      </span>
      <span style="color:#fe2c24;">
       <strong>
        本地没有rabbitmq环境
       </strong>
      </span>
      <span style="color:#0d0016;">
       <strong>
        ,所以直接报IOException所以,如果是使用的自己的服务器来配置,则需要修改配置文件。
       </strong>
      </span>
     </p>
    </blockquote>
    <p>
     修改后的配置文件：
    </p>
    <pre><code class="language-Groovy">server:
  port: 8801

spring:
  application:
    name: cloud-stream-provider
  rabbitmq:
    host: 192.168.20.129
    port: 5672
    username: root
    password: 123456
  cloud:
    stream:
      binders:
        defaultRabbit:
          type: rabbit
      bindings:
        output:
          destination: studyExchange
          content-type: application/json
          binder: defaultRabbit

eureka:
  client: # 客户端进行Eureka注册的配置
    service-url:
      defaultZone: http://eureka7001.com:7001/eureka,http://eureka7002.com:7002/eureka,http://eureka7003.com:7003/eureka
  instance:
    lease-renewal-interval-in-seconds: 2 # 设置心跳的时间间隔（默认是30秒）
    lease-expiration-duration-in-seconds: 5 # 如果现在超过了5秒的间隔（默认是90秒）
    instance-id: send-8801.com  # 在信息列表时显示主机名称
    prefer-ip-address: true     # 访问的路径变为IP地址</code></pre>
