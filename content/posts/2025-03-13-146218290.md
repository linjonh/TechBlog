    <h4>
     <a id="_Tauri__asset_not_found_indexhtml__0">
     </a>
     如何修复 Tauri 发布后程序运行时显示
     <code>
      asset not found: index.html
     </code>
     的问题
    </h4>
    <p>
     在使用 Tauri 发布应用程序时，如果运行时出现
     <code>
      asset not found: index.html
     </code>
     的错误，通常是因为 Tauri 无法找到或正确加载前端资源文件（如
     <code>
      index.html
     </code>
     ）。以下是详细的修复步骤和解释。
    </p>
    <hr/>
    <p>
     检查时直接查看 编译后包所在位置下面有没有index.html文件生成即可。
     <br/>
     路径比如build,dist等，据配置文件配置指定。
    </p>
    <h5>
     <a id="_8">
     </a>
     <strong>
      问题原因
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       <code>
        frontendDist
       </code>
       配置错误
      </strong>
      ：
      <code>
       tauri.conf.json
      </code>
      中的
      <code>
       build.frontendDist
      </code>
      路径未正确指向包含
      <code>
       index.html
      </code>
      的前端资源文件夹。
     </li>
     <li>
      <strong>
       前端资源未正确打包
      </strong>
      ：前端构建工具（如 Vite、Webpack）未将资源文件输出到 Tauri 预期的目录。
     </li>
     <li>
      <strong>
       路径问题
      </strong>
      ：在开发环境和生产环境中，路径配置不一致，导致资源加载失败。
     </li>
    </ol>
    <hr/>
    <h5>
     <a id="_15">
     </a>
     <strong>
      修复步骤
     </strong>
    </h5>
    <h6>
     <a id="1__tauriconfjson__17">
     </a>
     <strong>
      1. 检查
      <code>
       tauri.conf.json
      </code>
      配置
     </strong>
    </h6>
    <p>
     确保
     <code>
      tauri.conf.json
     </code>
     中的
     <code>
      build.frontendDist
     </code>
     路径正确指向包含
     <code>
      index.html
     </code>
     的文件夹。例如：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"build"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"frontendDist"</span><span class="token operator">:</span> <span class="token string">"../dist"</span><span class="token punctuation">,</span> <span class="token comment">// 指向前端资源文件夹</span>
    <span class="token string-property property">"devPath"</span><span class="token operator">:</span> <span class="token string">"http://localhost:1420"</span><span class="token punctuation">,</span> <span class="token comment">// 开发环境路径</span>
    <span class="token string-property property">"beforeDevCommand"</span><span class="token operator">:</span> <span class="token string">"npm run dev"</span><span class="token punctuation">,</span>
    <span class="token string-property property">"beforeBuildCommand"</span><span class="token operator">:</span> <span class="token string">"npm run build"</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      如果使用 Vite，默认输出目录为
      <code>
       dist
      </code>
      ，确保
      <code>
       frontendDist
      </code>
      路径正确。
     </li>
     <li>
      如果使用其他工具（如 Webpack），确保路径与输出目录一致。
     </li>
    </ul>
    <h6>
     <a id="2__32">
     </a>
     <strong>
      2. 确保前端资源正确打包
     </strong>
    </h6>
    <p>
     运行前端构建命令，确保资源文件生成到正确目录。例如：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> run build
</code></pre>
    <ul>
     <li>
      检查
      <code>
       dist
      </code>
      目录下是否包含
      <code>
       index.html
      </code>
      及其他资源文件。
     </li>
     <li>
      如果
      <code>
       dist
      </code>
      目录为空，检查前端构建工具的配置文件（如
      <code>
       vite.config.js
      </code>
      或
      <code>
       webpack.config.js
      </code>
      ），确保输出目录正确。
     </li>
    </ul>
    <h6>
     <a id="3__40">
     </a>
     <strong>
      3. 检查路径问题
     </strong>
    </h6>
    <ul>
     <li>
      在开发环境中，Tauri 使用
      <code>
       devPath
      </code>
      加载前端资源（如
      <code>
       http://localhost:1420
      </code>
      ）。
     </li>
     <li>
      在生产环境中，Tauri 使用
      <code>
       frontendDist
      </code>
      加载本地资源文件。
     </li>
     <li>
      确保
      <code>
       index.html
      </code>
      及其依赖的资源文件（如 CSS、JS）路径正确。
     </li>
    </ul>
    <h6>
     <a id="4__45">
     </a>
     <strong>
      4. 清理缓存并重新构建
     </strong>
    </h6>
    <p>
     有时缓存文件可能导致资源加载失败，清理缓存并重新构建：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 清理缓存</span>
<span class="token function">rm</span> <span class="token parameter variable">-rf</span> dist
<span class="token function">rm</span> <span class="token parameter variable">-rf</span> src-tauri/target

<span class="token comment"># 重新构建</span>
<span class="token function">npm</span> run build
npx tauri build
</code></pre>
    <h6>
     <a id="5__57">
     </a>
     <strong>
      5. 检查文件权限
     </strong>
    </h6>
    <p>
     在部分操作系统（如 Linux）中，文件权限可能导致资源加载失败。确保
     <code>
      dist
     </code>
     目录及其文件具有正确的权限：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> <span class="token parameter variable">-R</span> <span class="token number">755</span> dist
</code></pre>
