    <h4>
     <a id="_React__Hydration_Failed__0">
     </a>
     <strong>
      解决 React 中的 Hydration Failed 错误
     </strong>
    </h4>
    <p>
     React 的
     <strong>
      服务器端渲染
     </strong>
     （SSR）通过在服务器端生成 HTML 并将其发送给客户端，帮助提高页面加载速度和搜索引擎优化（SEO）。然而，在进行 SSR 后，React 需要进行
     <strong>
      水合
     </strong>
     （hydration）操作，即将服务器渲染的静态 HTML 转换为可交互的 React 组件。这一过程中，如果服务器端渲染的 HTML 和客户端渲染的 HTML 内容不一致，就会出现
     <strong>
      Hydration Failed
     </strong>
     错误。
    </p>
    <p>
     本文将详细解析
     <strong>
      Hydration Failed
     </strong>
     错误的发生原因以及解决方法，帮助你有效避免和排查这个问题。
    </p>
    <hr/>
    <h4>
     <a id="_Hydration_Failed__8">
     </a>
     <strong>
      什么是 Hydration Failed 错误？
     </strong>
    </h4>
    <p>
     在 React 中，服务器端渲染的页面先生成静态 HTML，并发送给浏览器。接着，React 会在客户端执行水合操作，将这些静态 HTML 元素转化为 React 可以管理的动态组件。如果服务器端和客户端渲染的 HTML 内容不一致，就会触发
     <strong>
      Hydration Failed
     </strong>
     错误。
    </p>
    <h4>
     <a id="_Hydration_Failed__12">
     </a>
     <strong>
      为什么会发生 Hydration Failed 错误？
     </strong>
    </h4>
    <h5>
     <a id="1__14">
     </a>
     <strong>
      1. 动态内容导致的差异
     </strong>
    </h5>
    <p>
     最常见的原因是
     <strong>
      动态内容
     </strong>
     ，即依赖于客户端环境的数据（如
     <code>
      Math.random()
     </code>
     、
     <code>
      Date.now()
     </code>
     、
     <code>
      window
     </code>
     、
     <code>
      document
     </code>
     等），这些内容在服务器端渲染时会有所不同，从而导致水合时的 HTML 不匹配。
    </p>
    <p>
     例如：
    </p>
    <pre><code class="prism language-jsx">function TimeComponent() {
  return &lt;p&gt;当前时间: {Date.now()}&lt;/p&gt;;
}
</code></pre>
    <p>
     在服务器端，
     <code>
      Date.now()
     </code>
     会获取服务器的时间戳，而在客户端，
     <code>
      Date.now()
     </code>
     会获取浏览器的时间戳。这两者不一致，就会导致 HTML 的差异。
    </p>
    <h5>
     <a id="2__ID__26">
     </a>
     <strong>
      2. 不稳定的 ID 或随机数据
     </strong>
    </h5>
    <p>
     如果你在渲染过程中使用了不稳定的 ID 或随机数（例如
     <code>
      Math.random()
     </code>
     ），这些内容会在每次渲染时变化，导致服务器和客户端渲染的 HTML 不一致，从而触发水合失败。
    </p>
    <h5>
     <a id="3__30">
     </a>
     <strong>
      3. 客户端特有的操作
     </strong>
    </h5>
    <p>
     <code>
      window
     </code>
     、
     <code>
      document
     </code>
     和
     <code>
      localStorage
     </code>
     等浏览器对象只存在于客户端，在服务器端渲染时这些对象不可用。如果在服务器端渲染时使用了这些浏览器对象，就会导致水合时的差异，进而引发错误。
    </p>
    <h5>
     <a id="4__34">
     </a>
     <strong>
      4. 异步数据加载问题
     </strong>
    </h5>
    <p>
     如果组件依赖于异步数据（例如通过 API 请求获取数据），而这个数据没有在服务器端渲染完成前加载完毕，就会造成服务器端和客户端渲染的 HTML 不一致，导致水合错误。
    </p>
    <hr/>
    <h4>
     <a id="_Hydration_Failed__40">
     </a>
     <strong>
      如何解决 Hydration Failed 错误？
     </strong>
    </h4>
    <h5>
     <a id="1__42">
     </a>
     <strong>
      1. 避免动态内容
     </strong>
    </h5>
    <p>
     对于依赖于动态内容的部分（如时间戳、随机数等），你应该确保这些内容只在客户端渲染时生成，而不是在服务器端渲染时生成。
    </p>
    <p>
     <strong>
      解决方法
     </strong>
     ：在客户端使用
     <code>
      useEffect()
     </code>
     来延迟执行需要依赖客户端环境的数据操作。
    </p>
    <p>
     例如，避免直接在渲染中使用
     <code>
      Date.now()
     </code>
     ，改为使用
     <code>
      useEffect()
     </code>
     ：
    </p>
    <pre><code class="prism language-jsx">import { useState, useEffect } from "react";

function TimeComponent() {
  const [time, setTime] = useState(null);

  useEffect(() =&gt; {
    setTime(Date.now());
  }, []);

  return &lt;p&gt;当前时间: {time ? time : "加载中..."}&lt;/p&gt;;
}
</code></pre>
    <p>
     这样，服务器端会渲染
     <code>
      "加载中..."
     </code>
     ，客户端加载后再更新为真实时间。
    </p>
    <h5>
     <a id="2__useId__ID_64">
     </a>
     <strong>
      2. 使用
      <code>
       useId()
      </code>
      生成稳定的 ID
     </strong>
    </h5>
    <p>
     如果在渲染过程中需要使用唯一的 ID（如表单元素的
     <code>
      id
     </code>
     和
     <code>
      htmlFor
     </code>
     配对），避免使用
     <code>
      Math.random()
     </code>
     或其他随机数生成器，因为它们在服务器端和客户端渲染时的值可能不同，导致不匹配。
    </p>
    <p>
     <strong>
      解决方法
     </strong>
     ：React 18 提供了
     <code>
      useId()
     </code>
     Hook，它会确保生成的 ID 在服务器端和客户端一致。
    </p>
    <pre><code class="prism language-jsx">import { useId } from "react";

function MyComponent() {
  const id = useId();

  return &lt;div id={id}&gt;唯一 ID: {id}&lt;/div&gt;;
}
</code></pre>
    <p>
     <code>
      useId()
     </code>
     会生成一个稳定的唯一 ID，确保服务器端和客户端渲染的 HTML 一致。
    </p>
    <h5>
     <a id="3__useEffect_81">
     </a>
     <strong>
      3. 客户端特有操作使用
      <code>
       useEffect()
      </code>
     </strong>
    </h5>
    <p>
     对于依赖于客户端环境的操作（如
     <code>
      window
     </code>
     、
     <code>
      document
     </code>
     等），应该使用
     <code>
      useEffect()
     </code>
     来确保只有在客户端渲染时才进行这些操作。
    </p>
    <p>
     <strong>
      解决方法
     </strong>
     ：将浏览器特有的逻辑放入
     <code>
      useEffect()
     </code>
     中：
    </p>
    <pre><code class="prism language-jsx">import { useState, useEffect } from "react";

function WindowSizeComponent() {
  const [windowWidth, setWindowWidth] = useState(0);

  useEffect(() =&gt; {
    setWindowWidth(window.innerWidth);
  }, []);

  return &lt;div&gt;当前窗口宽度: {windowWidth}px&lt;/div&gt;;
}
</code></pre>
    <p>
     这样，
     <code>
      window.innerWidth
     </code>
     只会在客户端获取，避免了在服务器端渲染时访问无效的浏览器对象。
    </p>
    <h5>
     <a id="4__101">
     </a>
     <strong>
      4. 确保异步数据在服务器端渲染时加载完成
     </strong>
    </h5>
    <p>
     如果组件需要依赖异步数据，在 SSR 时要确保数据加载完成后再进行渲染。你可以使用
     <code>
      getServerSideProps
     </code>
     （对于 Next.js）或其他类似的 API 来确保异步数据在渲染之前已准备好。
    </p>
    <p>
     <strong>
      解决方法
     </strong>
     ：
    </p>
    <pre><code class="prism language-jsx">export async function getServerSideProps() {
  const data = await fetchDataFromAPI();
  return { props: { data } };
}

function DataComponent({ data }) {
  return &lt;div&gt;数据: {data}&lt;/div&gt;;
}
</code></pre>
    <p>
     这样，数据会在服务器端准备好后再进行渲染，确保服务器端和客户端渲染的 HTML 一致。
    </p>
    <hr/>
    <h4>
     <a id="_Hydration_Failed__120">
     </a>
     <strong>
      如何调试 Hydration Failed 错误？
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        检查浏览器控制台
       </strong>
       ：如果发生水合错误，浏览器控制台通常会提供详细的错误信息，指示 HTML 内容的具体差异。
      </p>
     </li>
     <li>
      <p>
       <strong>
        检查渲染的 HTML 是否一致
       </strong>
       ：可以查看浏览器中的“查看页面源代码”并与 React 渲染后的内容进行对比，找出差异。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用稳定的生成方式
       </strong>
       ：确保 ID、时间戳、随机数等只在客户端渲染时生成，避免使用会在不同环境中变化的内容。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_130">
     </a>
     <strong>
      总结
     </strong>
    </h4>
    <p>
     <code>
      Hydration Failed
     </code>
     错误是由于服务器端和客户端渲染的 HTML 不一致造成的。常见的原因包括依赖动态内容、使用不稳定的 ID、客户端特有的操作以及异步数据加载问题。通过确保动态内容只在客户端渲染时生成、使用
     <code>
      useId()
     </code>
     生成稳定的 ID、将客户端特有的操作放入
     <code>
      useEffect()
     </code>
     中、以及确保异步数据在服务器端渲染时加载完成，可以有效避免和解决 Hydration Failed 错误。
    </p>
