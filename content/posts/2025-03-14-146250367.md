    <h6>
     <a id="_2">
     </a>
     <strong>
      引言：为什么你需要关心网络安全？
     </strong>
    </h6>
    <p>
     想象一下这样的场景：
    </p>
    <ul>
     <li>
      你在咖啡店连接公共Wi-Fi时，输入了银行卡密码；
     </li>
     <li>
      你的手机收到一条“快递丢失理赔”短信，点击链接后账户被盗；
     </li>
     <li>
      公司服务器因未修复漏洞被黑客入侵，客户数据全部泄露……
     </li>
    </ul>
    <p>
     这些并非虚构，而是每天都在发生的真实事件。
     <strong>
      网络安全不是技术人员的专属领域，而是数字时代每个人的生存技能
     </strong>
     。本篇将带你从零开始，认识网络威胁的全貌，并掌握核心安全概念。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/df8f67b1cb5ca31c009e43a0782fcb72.jpeg"/>
    </p>
    <hr/>
    <h4>
     <a id="_15">
     </a>
     <strong>
      一、网络威胁全景：攻击者的“武器库”
     </strong>
    </h4>
    <p>
     网络攻击类型多样，以下是
     <strong>
      最常见的四类威胁
     </strong>
     及其真实案例：
    </p>
    <h5>
     <a id="1_Malware_18">
     </a>
     <strong>
      1. 恶意软件（Malware）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：通过感染设备窃取数据、破坏系统或勒索钱财的程序。
     </li>
     <li>
      <strong>
       常见形式
      </strong>
      ：
      <ul>
       <li>
        <strong>
         勒索软件
        </strong>
        （如WannaCry）：加密用户文件，要求支付比特币赎金（2017年全球爆发，瘫痪英国医疗系统）。
       </li>
       <li>
        <strong>
         木马病毒
        </strong>
        （如Emotet）：伪装成正常软件，窃取银行账号与密码。
       </li>
       <li>
        <strong>
         蠕虫病毒
        </strong>
        （如Stuxnet）：无需用户操作即可自动传播（曾破坏伊朗核设施离心机）。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2_Data_Breach_25">
     </a>
     <strong>
      2. 数据泄露（Data Breach）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：敏感信息（如用户密码、身份证号）被非法获取。
     </li>
     <li>
      <strong>
       典型案例
      </strong>
      ：
      <ul>
       <li>
        <strong>
         Facebook数据门
        </strong>
        ：8700万用户数据被Cambridge Analytica滥用，影响美国大选。
       </li>
       <li>
        <strong>
         某酒店集团泄露
        </strong>
        ：5亿客户入住记录被黑客售卖，包含身份证号、家庭住址。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="3_Phishing_31">
     </a>
     <strong>
      3. 网络钓鱼（Phishing）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：通过伪装成可信来源（如银行、同事邮件）诱导用户点击恶意链接。
     </li>
     <li>
      <strong>
       攻击特征
      </strong>
      ：
      <ul>
       <li>
        <strong>
         紧迫性话术
        </strong>
        ：“账户异常！立即验证！”
       </li>
       <li>
        <strong>
         伪造链接
        </strong>
        ：
        <code>
         www.paypa1.com
        </code>
        （数字“1”替换字母“l”）。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="4_APT_37">
     </a>
     <strong>
      4. 高级持续性威胁（APT）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：针对特定目标（如政府、企业）的长期隐蔽攻击。
     </li>
     <li>
      <strong>
       典型流程
      </strong>
      ：
      <ol>
       <li>
        <strong>
         渗透
        </strong>
        ：通过钓鱼邮件植入后门。
       </li>
       <li>
        <strong>
         潜伏
        </strong>
        ：长期窃取数据或监控系统。
       </li>
       <li>
        <strong>
         爆发
        </strong>
        ：窃取核心资料或瘫痪基础设施。
       </li>
      </ol>
     </li>
     <li>
      <strong>
       案例
      </strong>
      ：SolarWinds供应链攻击（2020年，黑客通过软件更新入侵美国多个政府部门）。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="CIA_47">
     </a>
     <strong>
      二、网络安全核心概念：CIA三要素
     </strong>
    </h4>
    <p>
     <strong>
      CIA模型
     </strong>
     是网络安全的基石，包含以下三个维度：
    </p>
    <h5>
     <a id="1_Confidentiality_50">
     </a>
     <strong>
      1. 机密性（Confidentiality）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       目标
      </strong>
      ：确保数据仅被授权者访问。
     </li>
     <li>
      <strong>
       破坏案例
      </strong>
      ：
      <ul>
       <li>
        员工将公司数据库密码泄露至GitHub。
       </li>
       <li>
        未加密的聊天记录被中间人窃听。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       防护手段
      </strong>
      ：加密技术（如AES）、访问控制（如RBAC权限模型）。
     </li>
    </ul>
    <h5>
     <a id="2_Integrity_57">
     </a>
     <strong>
      2. 完整性（Integrity）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       目标
      </strong>
      ：防止数据被篡改或破坏。
     </li>
     <li>
      <strong>
       破坏案例
      </strong>
      ：
      <ul>
       <li>
        黑客修改电商平台订单金额（如将100元改为1元）。
       </li>
       <li>
        病毒篡改系统文件导致蓝屏崩溃。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       防护手段
      </strong>
      ：数字签名、哈希校验（如SHA-256）。
     </li>
    </ul>
    <h5>
     <a id="3_Availability_64">
     </a>
     <strong>
      3. 可用性（Availability）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       目标
      </strong>
      ：确保授权用户能正常使用资源。
     </li>
     <li>
      <strong>
       破坏案例
      </strong>
      ：
      <ul>
       <li>
        DDoS攻击使网站无法访问（如2016年Dyn事件导致Twitter、Netflix宕机）。
       </li>
       <li>
        勒索软件加密企业文件导致业务停滞。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       防护手段
      </strong>
      ：负载均衡、流量清洗、数据备份。
     </li>
    </ul>
    <p>
     <strong>
      CIA模型关系图
     </strong>
     ：
    </p>
    <pre><code class="prism language-plain">机密性 → 保护数据隐私  
完整性 → 确保数据可信  
可用性 → 维持服务稳定  
三者缺一不可，共同构成安全防线！
</code></pre>
    <hr/>
    <h4>
     <a id="WiresharkHTTP_82">
     </a>
     <strong>
      三、动手实验：用Wireshark抓包分析HTTP明文传输风险
     </strong>
    </h4>
    <p>
     <strong>
      实验目的
     </strong>
     ：理解未加密通信的危险性，直观感受“机密性”的重要性。
    </p>
    <h5>
     <a id="1Wireshark_85">
     </a>
     <strong>
      步骤1：安装Wireshark
     </strong>
    </h5>
    <ul>
     <li>
      访问官网（
      <a href="https://www.wireshark.xn--org-128dw03dn9di17gcil6w8v." rel="nofollow">
       https://www.wireshark.org）下载并安装。
      </a>
     </li>
     <li>
      Windows用户需额外安装WinPcap驱动。
     </li>
    </ul>
    <h5>
     <a id="2HTTP_89">
     </a>
     <strong>
      步骤2：捕获HTTP流量
     </strong>
    </h5>
    <ol>
     <li>
      打开Wireshark，选择正在使用的网卡（如“Wi-Fi”或“以太网”）。
     </li>
     <li>
      在过滤栏输入
      <code>
       http
      </code>
      ，仅显示HTTP协议流量。
     </li>
     <li>
      访问一个HTTP网站（非HTTPS），例如：
      <a href="http://example.com." rel="nofollow">
       http://example.com。
      </a>
     </li>
    </ol>
    <h5>
     <a id="3_94">
     </a>
     <strong>
      步骤3：分析数据包
     </strong>
    </h5>
    <ul>
     <li>
      找到目标HTTP请求（如
      <code>
       GET / HTTP/1.1
      </code>
      ），双击查看详情：
      <ul>
       <li>
        <strong>
         明文暴露
        </strong>
        ：URL路径、Cookie、表单数据（如搜索关键词）均可被直接读取。
       </li>
       <li>
        <strong>
         风险示例
        </strong>
        ：若登录页面使用HTTP，用户名和密码将以明文传输！
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      实验结论
     </strong>
     ：
    </p>
    <ul>
     <li>
      HTTPS通过TLS加密保护数据传输，避免中间人窃听。
     </li>
     <li>
      <strong>
       行动建议
      </strong>
      ：永远不要在HTTP页面输入敏感信息！
     </li>
    </ul>
