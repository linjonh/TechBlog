    <h2>
     <a id="_29">
     </a>
     二、数据传输的基本概念
    </h2>
    <h3>
     <a id="21__30">
     </a>
     2.1 数据与信号
    </h3>
    <p>
     数据是信息的载体，可以是文本、图像、音频或视频等形式。在网络中，数据需要通过信号进行传输。信号分为两种类型：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        模拟信号
       </strong>
       ：连续变化的信号，例如电话线中的声音信号。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数字信号
       </strong>
       ：离散的信号，由二进制数据（0 和 1）表示。
      </p>
     </li>
    </ul>
    <h3>
     <a id="22__37">
     </a>
     2.2 数据传输的基本单位
    </h3>
    <p>
     数据传输的基本单位是比特（bit），即二进制数的一位（0 或 1）。多个比特组合成更大的单位，例如：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        字节（Byte）
       </strong>
       ：8 个比特。
      </p>
     </li>
     <li>
      <p>
       <strong>
        千字节（KB）
       </strong>
       ：1024 字节。
      </p>
     </li>
     <li>
      <p>
       <strong>
        兆字节（MB）
       </strong>
       ：1024 KB。
      </p>
     </li>
    </ul>
    <h3>
     <a id="23__46">
     </a>
     2.3 数据传输的度量指标
    </h3>
    <p>
     数据传输的性能通常通过以下指标衡量：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        带宽（Bandwidth）
       </strong>
       ：单位时间内传输的数据量，通常以 bps（比特每秒）为单位。
      </p>
     </li>
     <li>
      <p>
       <strong>
        延迟（Latency）
       </strong>
       ：数据从发送端到接收端所需的时间。
      </p>
     </li>
     <li>
      <p>
       <strong>
        吞吐量（Throughput）
       </strong>
       ：实际传输的有效数据量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        误码率（Bit Error Rate, BER）
       </strong>
       ：传输过程中出错的比特比例。
      </p>
     </li>
    </ul>
    <h2>
     <a id="_57">
     </a>
     三、数据在网络中的传输过程
    </h2>
    <h3>
     <a id="31__58">
     </a>
     3.1 数据封装与解封装
    </h3>
    <p>
     数据在网络中传输时，需要按照协议栈的层次进行封装和解封装。以 TCP/IP 模型为例：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        应用层
       </strong>
       ：生成原始数据（如 HTTP 请求）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        传输层
       </strong>
       ：添加 TCP 头部（包括端口号、序列号等）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        网络层
       </strong>
       ：添加 IP 头部（包括源 IP 地址、目标 IP 地址等）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数据链路层
       </strong>
       ：添加帧头部和尾部（包括 MAC 地址、校验和等）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        物理层
       </strong>
       ：将数据转换为信号并通过物理介质传输。
      </p>
     </li>
    </ul>
    <p>
     接收端则按照相反的顺序解封装数据。
    </p>
    <center>
     数据封装与解封装
    </center>
    <p>
     <img alt="图1：数据封装与解封装过程示意图" src="https://i-blog.csdnimg.cn/direct/aef2f6b0c39649e19ea8d2e405669823.png"/>
    </p>
    <h3>
     <a id="32__79">
     </a>
     3.2 数据分片与重组
    </h3>
    <p>
     当数据包的大小超过网络的最大传输单元（MTU）时，数据会被分片传输。接收端收到所有分片后，根据分片信息进行重组。
    </p>
    <h3>
     <a id="33__82">
     </a>
     3.3 数据路由与转发
    </h3>
    <p>
     数据包在网络中传输时，路由器根据目标 IP 地址查找路由表，决定下一跳的路径。这一过程称为路由与转发。
    </p>
    <h2>
     <a id="_85">
     </a>
     四、数据传输的编码方式
    </h2>
    <h3>
     <a id="41__86">
     </a>
     4.1 数字信号与模拟信号
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        数字信号
       </strong>
       ：适合计算机处理，抗干扰能力强。
      </p>
     </li>
     <li>
      <p>
       <strong>
        模拟信号
       </strong>
       ：适合传输连续变化的信号，但易受噪声干扰。
      </p>
     </li>
    </ul>
    <h3>
     <a id="42__91">
     </a>
     4.2 常见的编码方式
    </h3>
    <h4>
     <a id="421_NRZ_92">
     </a>
     4.2.1 不归零编码（NRZ）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：用高电平表示 1，低电平表示 0。
      </p>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：简单易实现。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：无法区分连续的 0 或 1，可能导致时钟同步问题。
      </p>
     </li>
    </ul>
    <h4>
     <a id="422__99">
     </a>
     4.2.2 曼彻斯特编码
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：每个比特周期中间有一个跳变，从高到低表示 0，从低到高表示 1。
      </p>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：自带时钟信号，适合同步传输。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：带宽利用率较低。
      </p>
     </li>
    </ul>
    <h4>
     <a id="423__106">
     </a>
     4.2.3 差分曼彻斯特编码
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：每个比特周期开始时有跳变表示 0，无跳变表示 1。
      </p>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：抗干扰能力强。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：实现复杂。
      </p>
     </li>
    </ul>
    <h4>
     <a id="424_4B5B__113">
     </a>
     4.2.4 4B/5B 编码
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：将 4 位数据映射为 5 位编码，确保足够的跳变。
      </p>
     </li>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：提高带宽利用率。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：需要额外的编码表。
      </p>
     </li>
    </ul>
    <h3>
     <a id="43__120">
     </a>
     4.3 编码方式的选择与优化
    </h3>
    <p>
     选择编码方式时，需考虑以下因素：
    </p>
    <ul>
     <li>
      <p>
       带宽利用率
      </p>
     </li>
     <li>
      <p>
       时钟同步能力
      </p>
     </li>
     <li>
      <p>
       抗干扰能力
      </p>
     </li>
     <li>
      <p>
       实现复杂度
      </p>
     </li>
    </ul>
    <h2>
     <a id="_131">
     </a>
     五、数据传输的模式
    </h2>
    <h3>
     <a id="51__132">
     </a>
     5.1 单工、半双工与全双工
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        单工
       </strong>
       ：数据只能单向传输（如广播）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        半双工
       </strong>
       ：数据可以双向传输，但不能同时进行（如对讲机）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        全双工
       </strong>
       ：数据可以同时双向传输（如电话）。
      </p>
     </li>
    </ul>
    <h3>
     <a id="52__139">
     </a>
     5.2 串行传输与并行传输
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        串行传输
       </strong>
       ：逐比特传输，适合远距离通信。
      </p>
     </li>
     <li>
      <p>
       <strong>
        并行传输
       </strong>
       ：多比特同时传输，适合短距离高速通信。
      </p>
     </li>
    </ul>
    <h3>
     <a id="53__144">
     </a>
     5.3 同步传输与异步传输
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        同步传输
       </strong>
       ：发送端和接收端使用统一的时钟信号。
      </p>
     </li>
     <li>
      <p>
       <strong>
        异步传输
       </strong>
       ：每个数据包附带起始位和停止位。
      </p>
     </li>
    </ul>
    <h2>
     <a id="_149">
     </a>
     六、数据传输中的关键技术
    </h2>
    <h3>
     <a id="61__150">
     </a>
     6.1 差错控制
    </h3>
    <h4>
     <a id="611__151">
     </a>
     6.1.1 奇偶校验
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：通过添加一个校验位，使数据中 1 的个数为奇数或偶数。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：简单场景下的错误检测。
      </p>
     </li>
    </ul>
    <h4>
     <a id="612_CRC_156">
     </a>
     6.1.2 循环冗余校验（CRC）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：通过多项式除法生成校验码。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：广泛应用于数据链路层。
      </p>
     </li>
    </ul>
    <h4>
     <a id="613_FEC_161">
     </a>
     6.1.3 前向纠错（FEC）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：通过添加冗余信息，使接收端能够纠正错误。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：无线通信和卫星通信。
      </p>
     </li>
    </ul>
    <h3>
     <a id="62__166">
     </a>
     6.2 流量控制
    </h3>
    <h4>
     <a id="621__167">
     </a>
     6.2.1 停止-等待协议
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：发送端发送一个数据包后，等待接收端的确认。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：简单但效率低。
      </p>
     </li>
    </ul>
    <h4>
     <a id="622__172">
     </a>
     6.2.2 滑动窗口协议
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：允许发送端连续发送多个数据包，接收端通过窗口大小控制流量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：TCP 协议中的流量控制。
      </p>
     </li>
    </ul>
    <h3>
     <a id="63__177">
     </a>
     6.3 拥塞控制
    </h3>
    <h4>
     <a id="631__178">
     </a>
     6.3.1 拥塞检测与避免
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：通过监测网络状况动态调整发送速率。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：TCP 协议中的拥塞控制。
      </p>
     </li>
    </ul>
    <h4>
     <a id="632_TCP__183">
     </a>
     6.3.2 TCP 拥塞控制算法
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        慢启动
       </strong>
       ：逐步增加发送速率。
      </p>
     </li>
     <li>
      <p>
       <strong>
        拥塞避免
       </strong>
       ：在接近拥塞时降低发送速率。
      </p>
     </li>
    </ul>
    <h2>
     <a id="_188">
     </a>
     七、数据传输的实际应用
    </h2>
    <h3>
     <a id="71__189">
     </a>
     7.1 以太网中的数据传输
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：采用 CSMA/CD 协议，支持全双工通信。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：局域网中的主要传输技术。
      </p>
     </li>
    </ul>
    <h3>
     <a id="72__194">
     </a>
     7.2 无线网络中的数据传输
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：采用无线电波传输，易受干扰。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：Wi-Fi、蓝牙等。
      </p>
     </li>
    </ul>
    <h3>
     <a id="73__199">
     </a>
     7.3 互联网中的数据传输
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：基于 TCP/IP 协议，支持全球范围内的数据传输。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用
       </strong>
       ：网页浏览、文件传输、视频流等。
      </p>
     </li>
    </ul>
