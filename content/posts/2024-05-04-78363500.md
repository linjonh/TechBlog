# 程序员怎么转到大数据方向

有小伙伴通过“程序视界”的有问有答问如何转型做大数据：

![](https://img-blog.csdnimg.cn/img_convert/be1b673b1c5d238429d4d2c008fbf4a1.png;wxfrom=5&wx_lazy=1)

我自己不懂大数据，就到 GitChat 找来一个 Chat ，专门讲如何转型大数据的。

分享者是
Fickr孫啟誠
，一家互联网金融公司的大数据研发工程师，他负责的项目，处理的数据一般在50T以上，日均数据增长在20G以上。

以下是 Chat 的介绍。

大数据相关的工作也是分为很多方向，甚至不同公司同一个职位的工作内容都会有很大的差异。所以本场 Chat 旨在为准备转型大数据的朋友确定一个适合自己的方向（最近有很多通过知乎加我的小伙伴一直在咨询关于转型方向的问题，可能很多人会有这样的疑问）。主要内容如下:

一、非程序员转型大数据

1. 金融财会，统计，其他商科转型大数据。
2. 非科班理工科转型大数据
3. 除了第 1 条以外的文科专业同学转型大数据。

二、程序员转型大数据

1. Java 后端 / Java Web 程序员转型大数据。
2. Python 程序员转型大数据。
3. 数据库相关工作人员转型大数据。
4. 运维相关工作人员转型大数据。
5. 其他（如前端，实施人员等）

对于每个转型都会包括一下三个方面的内容:

a. 最佳职位推荐，推荐原因分析。

b. 结合自身情况和行业现状做一个 SWOT 分析。

c. 推荐职位应该具有的能力（技术能力，综合能力）与资源分析。
