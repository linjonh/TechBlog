    <h2>
     <a id="_1">
     </a>
     故障切换
    </h2>
    <p>
     主：M (10.10.10.1)
     <br/>
     从：S1 (10.10.10.2) 、S2 (10.10.10.2)
     <br/>
     由于主库M已经故障，所以只要把S1或者S2提升为主库，假如提升S1为主库，修改S2的主库为S1，即可。
    </p>
    <h3>
     <a id="1_5">
     </a>
     1.保证从库数据已更新完毕
    </h3>
    <p>
     在从库S1、S2上操作，保证S1、S2已经执行了relay log中的全部更新，在每个从库上先停止io线程：
    </p>
    <pre><code>mysql&gt;stop slave io_thread;
</code></pre>
    <p>
     然后执行
    </p>
    <pre><code>mysql&gt; show processlist \G;
</code></pre>
    <p>
     看到下面的语句，表示全部更新执行完毕。
    </p>
    <p>
     Has read all relay log waiting for the slave I/O thread to update it
    </p>
    <h3>
     <a id="2S1_18">
     </a>
     2、从库S1重置为主库
    </h3>
    <pre><code>mysql&gt; stop slave;
mysql&gt; reset master;
mysql&gt; reset slave; 
mysql&gt; show master status;           #记录FILE Position 
</code></pre>
    <h3>
     <a id="3S2S1_25">
     </a>
     3、从库S2重置，改为S1为主库：
    </h3>
    <pre><code>mysql&gt; stop slave;
mysql&gt; reset master;
mysql&gt; reset slave; 
mysql&gt;change master to master_host='10.10.10.2',master_port=3308, master_user='repl',master_password='pwd', master_log_file='mysql-bin.xxxxx',master_log_pos=xxxxx; 
mysql&gt; start slave; 
</code></pre>
    <h3>
     <a id="4S2S1_33">
     </a>
     4、主库恢复后，同S2，改主库为S1
    </h3>
