    <h2>
     <a id="Vue_3__script_setup__props__4">
     </a>
     Vue 3 中
     <code>
      &lt;script setup&gt;
     </code>
     与 props 的"双向绑定"实现原理
    </h2>
    <h3>
     <a id="Props__6">
     </a>
     Props 的单向数据流特性
    </h3>
    <ul>
     <li>
      <strong>
       不可变性原则
      </strong>
      <br/>
      props 是父组件传递给子组件的
      <strong>
       只读数据
      </strong>
      ，遵循单向数据流原则。
     </li>
     <li>
      <strong>
       通信方式
      </strong>
      <br/>
      子组件不能直接修改 props，需通过**事件发射（emit）**通知父组件更新数据。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_14">
     </a>
     模拟双向绑定的实现方案
    </h3>
    <p>
     Vue 提供
     <code>
      v-model
     </code>
     语法糖，通过约定式 props 和事件实现类似双向绑定的效果：
    </p>
    <h4>
     <a id="_17">
     </a>
     实现步骤
    </h4>
    <ol>
     <li>
      <strong>
       父组件
      </strong>
      通过
      <code>
       v-model
      </code>
      绑定数据到子组件
     </li>
     <li>
      <strong>
       子组件
      </strong>
      接收默认名为
      <code>
       modelValue
      </code>
      的 props
     </li>
     <li>
      <strong>
       子组件
      </strong>
      触发
      <code>
       update:modelValue
      </code>
      事件传递新值
     </li>
    </ol>

     <a id="_22">

    <h2>
     <a id="_57">
     </a>
     总结：
    </h2>
    <p>
     在这个例子中，父组件的 inputValue 被绑定到了子组件的 v-model。在子组件中，我们接收了一个名为 modelValue 的 props，并在输入框的值改变时发射了一个 update:modelValue 事件。父组件监听这个事件并更新它的 inputValue。
    </p>
    <p>
     尽管这不是真正的双向绑定，但它提供了一种在 Vue 组件之间模拟双向数据流动的有效方式。记住，props 应该是不可变的，子组件应该通过事件来与父组件通信。
    </p>
