    <p>
     <font size="3">
      性质：移动行业应用开发商
      <br/>
      受访对象：北京联合信息集团副总 熊军
      <br/>
      <font color="#ff0000">
       观点：
      </font>
      1. 移动行业应用的开发，需要与不同的行业的SI或者ISV合作，靠Mobile单打独斗还是不行的，通过互相推广，将移动应用这个蛋糕做大。
      <br/>
      2. 整个企业移动应用市场还处于萌芽阶段，要将行业应用推广开，牵涉到移动产业链上各方的利益。这些角色应该互相支持合作，先把整个市场作大。
      <br/>
     </font>
    </p>
    <p>
     <font size="3">
      公司背景：联合电信移动应用事业部的前身是北京四维时代，是国内最早从事移动终端和无线网络软件开发和行业应用开发的单位之一，致力于面向政府、企业和行业客户，提供全方位的移动、无线和嵌入式应用软件和解决方案。
     </font>
    </p>
    <p>
     <font size="3">
      采访正文：
     </font>
    </p>
    <p>
     <font size="3">
      1. CSDN：联合电信的"移动警务系统"获得微软移动应用开发大赛行业应用大奖。请谈一下公司在行业软件开发方面的成功经验。
      <br/>
      联合电信：
      <br/>
      我们作为开发行业应用的ISV，特别关注移动应用，是因为我们有一些相关的从业经验。我们最早在PDA上作开发，从Palm的开发逐渐过渡到PocketPC的开发，现在过渡到了PocketPC Phone和Smartphone。这个过程伴随着技术的进步，伴随着市场的变化，我们也在不断调整自己的步伐。
      <br/>
      原来做基于PDA的有一些小的软件，可以让企业利用这个PDA，在办公室外面做一些工作。但这并不能满足企业真正的移动需求。做到01、02年的时候，我们意识到移动设备必须要跟无线网络通信紧密结合，才能够充分满足客户的需要。
      <br/>
      刚开始也尝试了许多方法，原来PDA用红外线通过手机拨号上网，这种模式的应用其实很不好。随着技术的进步，这种无线网络的发展给我们带来了很多机会，例如GPRS和马上要启动的3G。
      <br/>
      根据我们的研发和市场经验，我们更看重智能手机的发展前景。原来的终端是个PDA，它没有无线功能，现在的智能终端有很多无线功能，这种无线功能不只是GPRS、CDMA或3G，其实还有很大一块是无线局域网的东西，比如：WiFi（802.11b）和蓝牙等。这种无线的概念不仅仅是类似GPRS这样的广域无线通信网络，还包括无线局域网，无线个人区域网，如蓝牙、UWB等无线网络技术。现在的很多移动终端集成了多种功能无线网络功能，比如既有GPRS的功能又有WIFI的功能。
      <br/>
      我们更看重智能移动终端给行业带来的变化。现在做移动商务的，主要有两类：一类是电信的增值服务商，他们考虑问题是站在电信的角度，怎么用电信的网络去为企业客户和个人消费者服务。一类是我们这样在PDA上开发应用的厂商，我们站的角度是找合适的技术，合适的网络去适应客户的需求，而不受限于电信运营商的网络。
      <br/>
      其实移动警务并不是我们做得最好的产品，我们做得最好的是数据采集类产品，比如移动销售、移动巡检等。警务这块的业务需求和技术方案比较成熟，这套系统是完全基于.NET和.NET CompactFramwork的技术体系开发的，并采用了基于GPRS和CDMA 1X网络的最新的移动流媒体技术。当时我们采用的技术方案是最先进的，实现的功能也很完善，比如我们把移动的视频监控功能集成进去了，而且可以对视频进行动态切换。当然，移动视频通讯协议这个底层的技术不是我们自己实现的，是采用Windows Mobile这个平台本身提供的Windows Media来实现的。Windows Mobile作为企应用终端是非常合适的，它把很多企业应用方案需要的功能内置了。比如说数据库、VPN、还有流媒体，还有非常完善的网络通信功能。这样你根本就不用担心你使用不同的网络技术需要调用不同的接口，不会在一些基础的技术上遇到障碍。在企业应用里面数据库是很基本的一个东西，而J2ME、Brew上可能找不到合适的数据库。另外Windows Mobile在扩展外设的应用上也很方便，比如企业应用涉及到的GPS、条码、或者数码相机这样一些设备，J2ME根本调不了GPS，用J2ME来调条码，也不可能像用Windows Mobile来调条码功能那么强。所以说，Windows Mobile平台确实底层更开放一些，功能更强大一些。而J2ME更多是面向消费领域的东西，来做游戏很好，因为游戏不需要太多的外设的扩展，没有太多本地的存储，也不需要天天跟数据库打交道，或者做一些网上浏览应用也很简单。消费领域更强调计算能力和图形处理能力，本地动态存储用的较少，顶多需要保存游戏进度进度等等。但是企业应用需要保存大量的数据，动态存储要占到很大的比例。而且需要很好的通过无线通讯，和后台中间件将这些数据保存在已有的系统和数据库中。Windows Mobile的ActiveSync，它是一套非常完整的中间件体系。年初Symbian和Nokia获得ActiveSync的授权，那是因为这可以帮助他们更好的实现企业信息系统的应用，将Outlook、Exchange、SQL Sever很好的融合在一起。这是微软在这块的优势，同步中间件在企业的应用优势非常明显。企业客户需要的不仅是电子邮件，还有更多的数据库需要去同步，作为应用开发商，我们不期望做太多底层工作，我们只是把客户的需求设计实现。
      <br/>
      2. CSDN：谈一下跟数据库厂商的合作
      <br/>
      联合电信：
      <br/>
      我们在终端上的数据库，DB2、Oracle都有现成的Windows Mobile上的数据库，而Windows Mobile 5已经把数据库集成到系统里了。微软提供的免费的移动数据库对企业应用吸引很大，因为其他的移动数据库的授权费用都比较昂贵。我们跟数据库厂商都有很好的合作，例如Sybase、Oracle等。Sybase的移动数据库有他的优势，但主要针对高端用户。Oracle数据库的优点在于与企业已有的Oracle数据库有很好的交互性。企业应用中，考虑和已有产品的结合很重要，我们现在和行业软件开发商合作，互相推广。这些行业软件开发商可以将我们的一些产品模块集成到他们的方案里，这样可以互利。
      <br/>
      3. CSDN：能谈一下开发不同行业应用的异同吗？
      <br/>
      联合电信：
      <br/>
      对不同的客户来说，技术和应用模式没有太大区别，都涉及到数据采集、查询以及现场处理，而且基本是这三种模式的融合。我们的移动巡检系统就偏重数据的采集，移动警务偏重数据查询，移动销售偏重于数据的现场处理。不同行业之间的应用模式，技术平台、技术选型上没有太大区别，唯一的区别是不同行业采取的数据不一样，不同行业的业务要求不一样，比如对数据的安全性要求。另外，不同行业间的业务流程差别比较大，所以分析业务流程也是行业应用开发的关键。
      <br/>
      4. CSDN：公司以后业务的重点在哪里？
      <br/>
      联合电信：
      <br/>
      公司现在的业务主要有两块，一块是移动应用，另一块是企业消息平台。这两块下面分别有很多针对不用行业的产品，另外也想和不同的行业的SI或者ISV合作，把我们在移动技术领域的优势推广。
      <br/>
      目前对移动应用需求较大的行业主要集中在外资企业以及国内垄断性的行业，比如快速消费品，即零售业，或一些数据采集需求比较多的行业，另外物流行业的需求也很大。
      <br/>
      那么，公司现在面临一个如何定位的问题。不可能参与到所有的行业中，这样就不精了。所以我们现在跟不同行业的软件开发伙伴合作，互相推广，通过不同的合作模式，将移动应用这个蛋糕做大。
      <br/>
      产品的提炼和推广过程非常痛苦。首先，定义这个产品很困难，然后还要说服客户用我们的平台，这在前期投入非常大。因为移动应用的推广都需要一个过程，涉及到一个用户习惯问题。
      <br/>
      从全球看，移动应用总体上可以划分为分美国、欧洲、中国三种类型。传统类型的移动应用，最领先的在美国，他们移动应用作得很好，提供基于多种系统的应用，而且非常完善。欧洲的应用比较单一，从手机角度出发的应用比较多，美国主要是PDA延伸出来的。中国现在是分成两帮，一帮是做电信、增值服务过来的，一帮是从传统的IT过来的，从PDA角度去延伸，其实现在两帮都没有找到出路。我们也是不停在找新的增长点和产品机会。现在最大的趋势还是移动终端和电信结合起来带来的应用模式上的增长点。
      <br/>
      5. CSDN：公司在开发移动客户关系管理系统方面做了哪些工作？
      <br/>
      联合电信：
      <br/>
      我们会跟一些CRM公司合作，比如SAP。这类似于跟行业SI的合作，我们会互相做一些产品和解决方案的推广。我们意识到仅仅要靠Mobile这块单打独斗还是不行的，因为在移动终端上跑的数据还是在这些传统的CRM、ERP里面，这就是为什么一些Mobile的供应商长不大，甚至死掉的原因。因为Mobile只是很小的一块，更多的只是一些其他系统的延伸。
      <br/>
      当然，移动商务的概念大得多，但现阶段还停留在这块。在联合电信看来，终端是要随着个人走的，PC是固定的，缺乏移动性，而大部分人都在流动状态。如果移动能够满足应用需求，企业可能会将信息系统完全转移到移动终端上。比如，完全基于移动的CRM方案。移动商务比电子商务的面更广，移动商务可以在任何时间任何地点完成，可能还需要5--10年的时间。国外有人将所有的企业信息数据用PDA实现，而且跟PC没有区别。中国很多企业并没有完全信息化，可能会一次扩展到下一代，只需要一台服务器，其他所有的工作都用智能手机来做。中国的移动商务是否能实现这种跨越，还需要很多的努力。
      <br/>
      6. CSDN：能谈谈企业应用中数据安全性的保证吗？
      <br/>
      联合电信：
      <br/>
      无线网络的安全性是非常高的。用户担心的安全性的问题主要存在于两个地方，第一，手机丢了上面的数据怎么办，第二，服务器上数据的安全性。但是这两块都有成熟的解决方案。无线终端上的安全体系跟PC上的完全一样，只是移动终端的安全级做不到PC上那么高，这是因为计算能力的原因，但也是可以解决的。从技术上没问题，更多的是应用模式上的，怎么让客户相信，这种方式是安全的。手机丢了，手机上的数据也是可以保密的。比如，输三次密码，不对就将数据全部清掉，或者将手机锁死。其实，手机比PC更加安全，因为这是随着自己走的，比固定的PC的安全性更有保障的。
      <br/>
      7. CSDN：公司现在还需要需要哪些支持？
      <br/>
      联合电信：
      <br/>
      从现在的状况看来，整个企业移动应用市场还处于萌芽阶段，整个产业链上的关系非常复杂。比如运营商的费用、不同平台的支持。操作系统厂商需要推广自己的操作系统概念，我们作为应用开发商在后面跟着走。微软在这块作得还不太够，很多用户还不知道Windows Mobile的概念，比如：这个系统有什么好处、能解决什么问题，为什么比其他的平台好，这是微软应该做的，但是现在的情况是我们需要给客户作很多宣传工作。
      <br/>
      对运营商来说，他们应该宣传企业级GPRS的使用，而终端厂商要保证够便宜，售后服务等。而且手机设计也要考虑到不同用户的需求。所以要将行业应用推广开，牵涉到各方的利益，很难平衡。这些角色应该互相支持合作，先把整个市场作大。
     </font>
