    <h3>
     <a id="1__22">
     </a>
     1. 背景
    </h3>
    <p>
     开发环境或者测试环境，有的时候需要用到
     <code>
      HTTPS
     </code>
     ，在时间和成本上考虑，我们利用
     <code>
      Java
     </code>
     密码体系结构(
     <code>
      JCA
     </code>
     ,
     <code>
      Java Cryptography Architecture
     </code>
     ) 完成建设可用的加密通信库。
    </p>
    <p>
     利用
     <code>
      Java
     </code>
     自带
     <code>
      keytool.exe
     </code>
     生成证书，
     <code>
      Keytool
     </code>
     是一个
     <code>
      Java
     </code>
     数据证书的管理工具，在命令行方式下，生成服务器证书申请文
     <code>
      CSR
     </code>
     ，然后到
     <code>
      CA
     </code>
     签发服务器证书。
    </p>
    <h3>
     <a id="2__28">
     </a>
     2. 演示环境
    </h3>
    <ul>
     <li>
      Windows 10
     </li>
     <li>
      Jdk 1.8.0.131
     </li>
    </ul>
    <h3>
     <a id="3__33">
     </a>
     3. 命令
    </h3>
    <ul>
     <li>
      genkey：定义将要创建一个新的密钥
     </li>
     <li>
      alias：别名，
      <code>
       alias
      </code>
      ，不区分大小写
     </li>
     <li>
      keyalg：指定加密的算法，这里是
      <code>
       RSA
      </code>
     </li>
     <li>
      keystore：指定密钥保存路径，在 D:盘目录下的 Code 文件中
     </li>
     <li>
      dname：表示密钥的
      <code>
       Distinguished Names
      </code>
     </li>
     <li>
      keypass：私钥的密码
     </li>
     <li>
      storepass：密钥库密码
     </li>
     <li>
      validity：创建的证书有效期，单位为 天
     </li>
     <li>
      list：列出证书
     </li>
     <li>
      rfc：可编码方式打印证书
     </li>
     <li>
      v：显示详细信息
     </li>
     <li>
      import：将已签名数字证书导入密钥库
     </li>
     <li>
      export：将别名指定的证书导出到文件
     </li>
     <li>
      keysize：指定密钥长度
     </li>
     <li>
      delete：删除密钥库中某条目
     </li>
     <li>
      file：参数指定导出到文件的文件名
     </li>
     <li>
      keypasswd：修改密钥库中指定条目的密码
     </li>
     <li>
      storepasswd：修改密钥库的密码
     </li>
    </ul>
    <h3>
     <a id="4__54">
     </a>
     4. 创建证书
    </h3>
    <pre><code>
keytool -genkey -alias eviltop -keyalg RSA -keystore D:\Code\eviltop.keystore -dname "CN=localhost, OU=localhost, O=localhost, L=SH, ST=SH, C=CN" -keypass 123456 -storepass 123456 -validity 180

</code></pre>
    <p>
     当不指定
     <code>
      keypass
     </code>
     和
     <code>
      storepass
     </code>
     ，创建密钥过程中需要手工输入。
    </p>
