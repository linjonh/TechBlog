    <h3>
     <a id="_2">
     </a>
     <strong>
      引言
     </strong>
    </h3>
    <p>
     本文的目标是演示如何创建一个集成了
     <strong>
      NocoBase、LangFlow 和 VectorDB
     </strong>
     工具的 AI 助手。作为基础，我将使用自己在
     <strong>
      NocoBase
     </strong>
     中开发的系统，该系统用于管理架构数据，通过添加一个 AI 助手来生成来自该系统中数据的洞察，例如
     <strong>
      应用、API 目录、解决方案设计和故事
     </strong>
     。
    </p>
    <p>
     在本文中，我们将使用以下技术：
    </p>
    <ul>
     <li>
      <strong>
       NocoBase、PostgreSQL 和 Docker
      </strong>
      （本文中将展示如何安装它们）。
     </li>
     <li>
      <strong>
       LangFlow
      </strong>
      ，本地运行，可通过 Docker 安装（安装指南在此处）。
     </li>
     <li>
      <strong>
       向量数据库
      </strong>
      ，通过注册
      <strong>
       AstraDB
      </strong>
      账户实现，本文将使用该数据库作为向量存储。
     </li>
    </ul>
    <h3>
     <a id="_12">
     </a>
     <strong>
      架构概览
     </strong>
    </h3>
    <p>
     <img alt="Architecture Overview.png" src="https://i-blog.csdnimg.cn/img_convert/bccc7d0618e29dfe74f01a8fb14b4246.png"/>
    </p>
    <p>
     此图展示了新数据（或更新数据）如何被转换为
     <strong>
      嵌入（embeddings）
     </strong>
     并存储到
     <strong>
      向量数据库
     </strong>
     中。
    </p>
    <ol>
     <li>
      <strong>
       用户 → NocoBase 平台
      </strong>
      <br/>
      用户与 NocoBase 平台交互（例如在集合中添加或更新记录）。
     </li>
     <li>
      <strong>
       触发 CRUD 操作
      </strong>
      <br/>
      NocoBase 中的 CRUD（增删改查）操作会触发事件或内部工作流。
     </li>
     <li>
      <strong>
       工作流（NocoBase）
      </strong>
      <br/>
      NocoBase 配置了一个响应数据变更的工作流。当检测到记录创建或修改时，工作流启动下一步。
     </li>
     <li>
      <strong>
       API 流程：加载数据
      </strong>
      <br/>
      NocoBase 工作流调用 LangFlow 的 API 或外部服务，发送待转换为嵌入的数据。
     </li>
     <li>
      <strong>
       LangFlow — 数据转嵌入
      </strong>
      <br/>
      LangFlow 接收数据，并利用语言模型将内容转换为向量（嵌入）。这些嵌入以数值形式表示文本的语义或上下文，支持语义搜索。
     </li>
     <li>
      <strong>
       向量数据库（AstraDB）
      </strong>
      <br/>
      嵌入随后存储到向量数据库（AstraDB）中，每个嵌入与原始内容（如文档、记录、文本）关联。
     </li>
    </ol>
    <p>
     <img alt="diagram.png" src="https://i-blog.csdnimg.cn/img_convert/2334bacb9a3f78536dc359ea3b525885.png"/>
    </p>
    <p>
     此图还展示了用户如何通过向量数据库进行语义查询并获取相关结果。
    </p>
    <ol>
     <li>
      <strong>
       用户 → NocoBase 平台
      </strong>
      <br/>
      用户再次与 NocoBase 平台交互，但此次通过嵌入式聊天小部件（或其他搜索界面）。
     </li>
     <li>
      <strong>
       嵌入式聊天小部件
      </strong>
      <br/>
      用户输入问题或指令。例如：
      <em>
       “展示关于应用 X 的信息。”
      </em>
      小部件向 LangFlow 发送请求以处理查询。
     </li>
     <li>
      <strong>
       LangFlow — 数据转嵌入
      </strong>
      <br/>
      LangFlow 将用户指令转换为嵌入，以向量格式表示搜索意图。
     </li>
     <li>
      <strong>
       向量数据库（AstraDB）— 相似性搜索
      </strong>
      <br/>
      使用指令的嵌入在 AstraDB 中搜索最相似的向量（即语义最接近的内容）。
     </li>
     <li>
      <strong>
       在模型中精炼搜索结果
      </strong>
      <br/>
      根据 AstraDB 返回的结果，LangFlow 使用 OpenAI 模型（或其他 LLM）精炼搜索结果。
     </li>
     <li>
      <strong>
       响应用户
      </strong>
      <br/>
      最终结果（文本、文档或生成的回答）返回至 NocoBase 聊天小部件，向用户展示响应。
     </li>
    </ol>
    <h3>
     <a id="_NocoBase__48">
     </a>
     <strong>
      使用 NocoBase 的应用
     </strong>
    </h3>
    <p>
     本文将沿用先前创建的应用。该应用旨在展示 NocoBase 的功能，并将在其中实现 AI 助手。下图展示了助手的工作方式。
    </p>
    <p>
     目标是通过提问获取功能相关的洞察。可用功能包括：应用、API 目录、解决方案设计和故事。
    </p>
    <p>
     <img alt="Application Using NocoBase.webp" src="https://i-blog.csdnimg.cn/img_convert/85b13f0ea090677e3a3880b19418b971.webp?x-oss-process=image/format,png"/>
    </p>
    <h3>
     <a id="_LangFlow__AstraDB_56">
     </a>
     <strong>
      什么是 LangFlow 和 AstraDB？
     </strong>
    </h3>
    <p>
     <strong>
      LangFlow
     </strong>
     是一款由巴西开发者创建的开源工具，提供图形化界面以构建、可视化和调试涉及语言模型的工作流。基于 LangChain 生态系统，LangFlow 简化了自然语言处理（NLP）流程和生成式 AI 应用的创建，允许开发者连接不同组件（如 API 调用、文本转换和业务逻辑），而无需大量编码。
    </p>
    <p>
     <img alt="LangFlow and AstraDB.png" src="https://i-blog.csdnimg.cn/img_convert/6593e111e0f33b73e6a60aac28cf4eb4.png">
      <strong>
       AstraDB
      </strong>
      最初是基于 Apache Cassandra 的分布式数据库服务，后扩展至支持非结构化数据存储和向量搜索。其向量数据库功能特别适用于机器学习应用、语义搜索、推荐系统和高维数据处理。
     </img>
    </p>
    <p>
     <img alt="AstraDB.png" src="https://i-blog.csdnimg.cn/img_convert/211bdf19af1aa08a0d81da47fdfcff6e.png"/>
    </p>

      LangFlow

     <img alt="Installing LangFlow.png" src="https://i-blog.csdnimg.cn/img_convert/cde8ce4c2260637417a22c6697d0427d.png"/>

    <h3>
     <a id="_AstraDB_74">
     </a>
     <strong>
      配置 AstraDB
     </strong>
    </h3>
    <p>
     在 AstraDB 注册账户后，可按以下步骤配置数据库和集合：
    </p>
    <ol>
     <li>
      选择
      <strong>
       服务提供商
      </strong>
     </li>
     <li>
      选择
      <strong>
       区域
      </strong>
     </li>
     <li>
      创建
      <strong>
       数据库
      </strong>
     </li>
    </ol>
    <p>
     <img alt="Configuring AstraDB.png" src="https://i-blog.csdnimg.cn/img_convert/2c53d848b5bd6995f55a28cabf3b70cd.png"/>
    </p>
    <p>
     由于集合存储向量化数据，需配置
     <strong>
      Embedding
     </strong>
     功能，即负责向量化的 LLM 模型。可选的嵌入模型包括 OpenAI、Nvidia 和 Google 的模型。本文使用
     <strong>
      text-embedding-ada-002
     </strong>
     进行数据转换。
    </p>
    <p>
     <img alt="Configuring AstraDB.png" src="https://i-blog.csdnimg.cn/img_convert/efdd21faa29039a40ad52df42547864b.png"/>
    </p>
    <h3>
     <a id="_88">
     </a>
     <strong>
      向向量数据库加载数据
     </strong>
    </h3>
    <p>
     此步骤将待搜索数据输入向量数据库：
    </p>
    <ol>
     <li>
      LangFlow 工作流通过 URL 接收数据。
     </li>
     <li>
      处理文本。
     </li>
     <li>
      使用 text-embedding-ada-002 转换为向量。
     </li>
     <li>
      存储至 AstraDB 的配置集合中。
     </li>
    </ol>
    <p>
     <img alt="Loading Data into the Vector Database.png" src="https://i-blog.csdnimg.cn/img_convert/04ad654b2c2ec50bb7384d2bc78822aa.png"/>
    </p>
    <h4>
     <a id="_99">
     </a>
     <strong>
      关键点
     </strong>
    </h4>
    <ul>
     <li>
      需在数据库连接组件中定义
      <strong>
       AstraDB Token
      </strong>
      。生成 Token 需在 AstraDB 集合页面点击
      <strong>
       Generate Token
      </strong>
      。
     </li>
    </ul>
    <p>
     <img alt="Important Points.png" src="https://i-blog.csdnimg.cn/img_convert/86faa8436d9ab20e4d522b4b13e6eda1.png"/>
    </p>
    <ul>
     <li>
      源文件可在此
      <a href="https://github.com/leandro-jm/">
       仓库
      </a>
      找到。
     </li>
     <li>
      使用 OpenAI 模型需为其 API 添加额度，并生成 Token 以配置嵌入生成组件。具体流程参考
      <a href="https://platform.openai.com/docs/quickstart" rel="nofollow">
       此文档
      </a>
      。
     </li>
    </ul>
    <p>
     通过 Postman 调用 LangFlow 的 API（通过
     <strong>
      API &gt; cURL
     </strong>
     获取 URL）即可测试。下图展示 Postman 请求示例：
    </p>
    <p>
     <img alt="Important Points.png" src="https://i-blog.csdnimg.cn/img_convert/b1378e2a5bb492210049c93d7f09560d.png"/>
    </p>
    <h3>
     <a id="_112">
     </a>
     <strong>
      通过模型执行搜索
     </strong>
    </h3>
    <p>
     此步骤负责从向量数据库检索数据，并通过
     <strong>
      RAG（检索增强生成）
     </strong>
     精炼结果。
    </p>
    <p>
     <img alt="Conducting Research Using a Model.png" src="https://i-blog.csdnimg.cn/img_convert/7de0b3d8ecbc34370218a2cdba81bea9.png"/>
    </p>
    <p>
     此阶段同样需在组件中定义 OpenAI 和 AstraDB 的 Token。源文件可在此
     <a href="https://github.com/leandro-jm/">
      仓库
     </a>
     找到。
    </p>
    <p>
     测试时可通过 LangFlow 的
     <strong>
      Playground
     </strong>
     标签页使用内置聊天功能。
    </p>
    <h3>
     <a id="_NocoBase__122">
     </a>
     <strong>
      在 NocoBase 中配置工作流
     </strong>
    </h3>
    <p>
     此步骤将在应用中创建触发器，以发送需向量化的数据。即每当插入新数据时，数据将被发送至 LangFlow API 处理。
    </p>
    <p>
     使用 NocoBase 的
     <strong>
      工作流
     </strong>
     功能实现此目标。工作流详情参考
     <a href="https://docs.nocobase.com/handbook/workflow" rel="nofollow">
      此文档
     </a>
     。
    </p>
    <p>
     <img alt="Configuring Workflow in NocoBase.png" src="https://i-blog.csdnimg.cn/img_convert/62eec8ee9cba5251af4d34ee9e1cb200.png"/>
    </p>
    <p>
     我们将创建一个工作流，将应用集合的数据发送至向量化集合。
    </p>
    <p>
     <strong>
      步骤：
     </strong>
    </p>
    <ol>
     <li>
      创建新工作流，选择
      <strong>
       集合事件
      </strong>
      。
      <br/>
      <img alt="Configuring Workflow in NocoBase.png" src="https://i-blog.csdnimg.cn/img_convert/d4b64547ed6dd03a7aa53839a2eb3c79.png"/>
     </li>
     <li>
      将
      <strong>
       触发器
      </strong>
      定义为
      <strong>
       应用
      </strong>
      集合的插入操作。
      <br/>
      <img alt="Configuring Workflow in NocoBase.png" src="https://i-blog.csdnimg.cn/img_convert/d2852472ce7d03248c1047b6a387e0a0.png"/>
     </li>
     <li>
      创建
      <strong>
       请求节点
      </strong>
      以发送应用的向量化数据。本文以
      <strong>
       标题
      </strong>
      和
      <strong>
       描述
      </strong>
      为例，实际可根据业务规则发送任意信息。
      <br/>
      <img alt="Configuring Workflow in NocoBase.png" src="https://i-blog.csdnimg.cn/img_convert/c47be5da2d28cf69bab30d3912a13285.png"/>
     </li>
     <li>
      <strong>
       通知步骤
      </strong>
      为可选，无需执行。
     </li>
    </ol>
