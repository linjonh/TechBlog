    <h3 name="1.%20%E5%BC%95%E8%A8%80">
     <span style="color:#ed7976">
      1. 引言
     </span>
    </h3>
    <p>
     Android Media3 是 Google 推出的新一代媒体处理框架，ExoPlayer 作为其核心组件，提供了比传统 MediaPlayer 更强大的功能和更高的灵活性。Media3 ExoPlayer 的主要优势包括：
    </p>
    <ul>
     <li>
      <p>
       支持 HLS、DASH、SmoothStreaming 等现代流媒体协议
      </p>
     </li>
     <li>
      <p>
       高度可定制的播放器架构
      </p>
     </li>
     <li>
      <p>
       自适应码率切换（ABR）优化
      </p>
     </li>
     <li>
      <p>
       完善的 DRM 支持
      </p>
     </li>
     <li>
      <p>
       改进的音频焦点处理
      </p>
     </li>
    </ul>
    <h3 id="2.%20%E6%B7%BB%E5%8A%A0%E4%BE%9D%E8%B5%96" name="2.%20%E6%B7%BB%E5%8A%A0%E4%BE%9D%E8%B5%96">
     <span style="color:#ed7976">
      2. 添加依赖
     </span>
    </h3>
    <p>
     首先，在
     <code>
      build.gradle
     </code>
     (模块级)中添加 Media3 ExoPlayer 依赖：
    </p>
    <pre><code class="language-Groovy">dependencies {
    //这里版本过高需要升级 SDK 至34 ，JDK 要 17
    def media3_version = "1.2.0"
    
    // 核心库
    implementation "androidx.media3:media3-exoplayer:$media3_version"
    // UI 组件
    implementation "androidx.media3:media3-ui:$media3_version"
    // 可选 DASH 支持
    implementation "androidx.media3:media3-exoplayer-dash:$media3_version"
    // 可选 HLS 支持
    implementation "androidx.media3:media3-exoplayer-hls:$media3_version"
    // 可选，支持 RTSP
    implementation 'androidx.media3:media3-exoplayer-rtsp:$media3_versio' 
}</code></pre>
    <blockquote>
     <p>
      这里版本过高需要升级 SDK 至34 ，JDK 要 17，否则会报如下错误：
     </p>
     <p>
      Dependency 'androidx.media3:media3-ui:1.2.0' requires libraries and applications that
      <br/>
      depend on it to compile against version 34 or later of the Android APIs.
     </p>
     <p>
      如果不想使用高版本可以使用1.0.0。
     </p>
    </blockquote>

    <h4 id="2.2%20%E5%88%9D%E5%A7%8B%E5%8C%96ExoPlayer" name="2.2%20%E5%88%9D%E5%A7%8B%E5%8C%96ExoPlayer">
     <span style="color:#956fe7">
      3.2 初始化ExoPlayer
     </span>
    </h4>
    <p>
     在
     <code>
      MainActivity.kt
     </code>
     （或
     <code>
      MainActivity.java
     </code>
     ）中创建
     <code>
      ExoPlayer
     </code>
     并加载视频：
    </p>
    <pre><code class="language-Kotlin">class ExoPlayerActivity : BaseActivity&lt;ActivityExoplayerBinding&gt;() {

    private var player: ExoPlayer? = null;
    override fun getLayoutID(): Int {
        return R.layout.activity_exoplayer
    }

    override fun init() {
        initExoPlayer()
    }

    private fun initExoPlayer() {
        player = ExoPlayer.Builder(this).build().also { exoPlayer -&gt;
            mBind.playerView.player = exoPlayer
            //加载 raw 资源，网络资源替换Uri即可。
            val mediaItem: MediaItem = MediaItem.fromUri("android.resource://" + packageName + "/" + R.raw.sv_ad_video)
            exoPlayer.setMediaItem(mediaItem)
            exoPlayer.prepare()
            exoPlayer.playWhenReady = true

        }
    }

    override fun onDestroy() {
        super.onDestroy()
        // 确保在Activity销毁时释放播放器资源
        player?.release()
        player = null
    }
}</code></pre>
    <h3 id="4.%20%E6%8E%A7%E5%88%B6%E6%92%AD%E6%94%BE" name="4.%20%E6%8E%A7%E5%88%B6%E6%92%AD%E6%94%BE">
     <span style="color:#ed7976">
      4. 控制播放
     </span>
    </h3>
    <p>
     ExoPlayer 提供了一系列方法来控制播放状态：
    </p>
    <pre><code class="language-Kotlin">player?.play()         // 播放
player?.pause()        // 暂停
player?.seekTo(20000)  // 跳转到 20 秒
player?.stop()         // 停止
player?.release()      // 释放资源</code></pre>
    <h3 id="5.%20%E7%9B%91%E5%90%AC%E6%92%AD%E6%94%BE%E7%8A%B6%E6%80%81" name="5.%20%E7%9B%91%E5%90%AC%E6%92%AD%E6%94%BE%E7%8A%B6%E6%80%81">
     <span style="color:#ed7976">
      5. 监听播放状态
     </span>
    </h3>
    <p>
     您可以监听
     <code>
      ExoPlayer
     </code>
     播放状态，如播放错误、缓冲状态等：
    </p>
    <pre><code class="language-Kotlin">        player?.addListener(object : Player.Listener {
            override fun onPlaybackStateChanged(state: Int) {
                when (state) {
                    ExoPlayer.STATE_READY -&gt; {
                        println("播放器已准备好")
                    }
                    ExoPlayer.STATE_BUFFERING -&gt; {
                        println("正在缓冲")
                    }
                    ExoPlayer.STATE_ENDED -&gt; {
                        println("播放完成")
                    }
                }
            }

            override fun onPlayerError(error: PlaybackException) {
                println("播放错误: ${error.message}")
            }
        })</code></pre>

     <span style="color:#ed7976">
      6. 播放网络流（HLS / DASH / RTSP）
     </span>
    </h3>
    <p>
     ExoPlayer 支持 HTTP Live Streaming (HLS) 和 DASH 等流媒体格式。只需更改
     <code>
      MediaItem
     </code>
     即可：
    </p>
    <pre><code class="language-Kotlin">// HLS 直播流
val mediaItem = MediaItem.fromUri("https://www.example.com/sample.m3u8") 
player?.setMediaItem(mediaItem)
player?.prepare()
player?.play()
</code></pre>

    <h4 id="7.1%20%E6%89%8B%E5%8A%A8%E5%88%87%E6%8D%A2%E5%8A%9F%E8%83%BD" name="7.1%20%E6%89%8B%E5%8A%A8%E5%88%87%E6%8D%A2%E5%8A%9F%E8%83%BD">

      7.1 手动切换功能

    <p>
     如果视频流支持多种权限，可以手动选择播放质量：
    </p>
    <pre><code class="language-Kotlin">val trackSelector = DefaultTrackSelector(this).apply {
    parameters = buildUponParameters().setMaxVideoSizeSd().build()
}
player = ExoPlayer.Builder(this).setTrackSelector(trackSelector).build()
</code></pre>

    <h4 id="7.2%20DRM%20%E4%BF%9D%E6%8A%A4" name="7.2%20DRM%20%E4%BF%9D%E6%8A%A4">
     <span style="color:#956fe7">
      7.2 DRM 保护
     </span>
    </h4>
    <p>
     ExoPlayer也支持DRM（如Widevine），您可以这样加载受DRM保护的内容：
    </p>
    <pre><code class="language-Kotlin">val drmSessionManager = DefaultDrmSessionManager.Builder()
    .setKeyRequestParameters(mapOf("Content-Type" to "application/json"))
    .build()
val mediaItem = MediaItem.Builder()
    .setUri(videoUri)
    .setDrmUuid(C.WIDEVINE_UUID)
    .setDrmSessionManager(drmSessionManager)
    .build()
player?.setMediaItem(mediaItem)
</code></pre>
    <h3 id="8.%20%E9%87%8A%E6%94%BE%E6%92%AD%E6%94%BE%E5%99%A8%E8%B5%84%E6%BA%90" name="8.%20%E9%87%8A%E6%94%BE%E6%92%AD%E6%94%BE%E5%99%A8%E8%B5%84%E6%BA%90">
     <span style="color:#ed7976">
      8. 释放播放器资源
     </span>
    </h3>
    <p>
     当
     <code>
      Activity
     </code>
     或
     <code>
      Fragment
     </code>
     关闭时，需要释放
     <code>
      ExoPlayer
     </code>
     ，避免内存泄漏：
    </p>
    <pre><code class="language-Kotlin">    override fun onDestroy() {
        super.onDestroy()
        // 确保在Activity/Fragment销毁时释放播放器资源
        player?.release()
        player = null
    }</code></pre>
