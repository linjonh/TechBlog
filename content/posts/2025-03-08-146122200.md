      自我简介：4年导游，10年程序员，最近6年一直深耕低代码领域，分享低代码和AI领域见解。

    </p>
    <p>
     <code>
      JSON Web Token
     </code>
     缩写 (JWT)，以加密后的JSON对象传输信息
    </p>
    <h6>
     <a id="_8">
     </a>
     应用场景
    </h6>
    <ol>
     <li>
      授权登录（SSO）
     </li>
     <li>
      信息交换
     </li>
    </ol>
    <h6>
     <a id="_13">
     </a>
     构成
    </h6>
    <p>
     包含三部分，以
     <code>
      .
     </code>
     分割
    </p>
    <ul>
     <li>
      Header（头部）
     </li>
     <li>
      Payload（负载）
     </li>
     <li>
      Signature（签名）
     </li>
    </ul>
    <p>
     常见形式：
     <code>
      xxxxxx.yyyyyy.zzzzzz
     </code>
    </p>
    <p>
     实例
    </p>
    <pre><code>eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.cThIIoDvwdueQB468K5xDc5633seEFoqwxjF_xSJyQQ
</code></pre>
    <p>
     Header:
     <code>
      eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
     </code>
    </p>
    <p>
     Payload:
     <code>
      eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ
     </code>
    </p>
    <p>
     Signature:
     <code>
      cThIIoDvwdueQB468K5xDc5633seEFoqwxjF_xSJyQQ
     </code>
    </p>
    <p>
     字符串通过Base64解码后得到如下信息：
     <code>
      （所以隐秘信息不能放在这里！）
     </code>
    </p>
    <p>
     <code>
      Header
     </code>
    </p>
    <pre><code>{
  "alg": "HS256", // 加密算法
  "typ": "JWT"    // token 类型
}
</code></pre>
    <p>
     <code>
      Payload
     </code>
    </p>
    <pre><code>{
  "sub": "1234567890",
  "name": "John Doe",
  "iat": 1516239022
}

JWT规定了7个官方字段供选用

iss (issuer) : 签发人 
exp (expiration time) : 过期时间 
sub (subject) : 主题 
aud (audience) : 受众 
nbf (Not Before) : 生效时间 
iat (Issued At) : 签发时间 
jti (JWT ID) : 编号

开发者也可以自己指定字段和内容

</code></pre>
    <p>
     <code>
      Signature
     </code>
    </p>
    <pre><code>HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  secret
)

通过签名算法 HMACSHA256（Header里指定） 得到的签名字符串：

cThIIoDvwdueQB468K5xDc5633seEFoqwxjF_xSJyQQ

其中：secret 密钥由服务器指定，不能泄漏给用户

</code></pre>
    <h6>
     <a id="_88">
     </a>
     传输途径
    </h6>
    <p>
     Http 请求头配置
     <code>
      Authorization
     </code>
    </p>
    <pre><code>Authorization: Bearer &lt;token&gt;
</code></pre>
    <p>
     服务端获取
     <code>
      token
     </code>
     后可以 通过
     <code>
      srcret
     </code>
     验签比对，防止篡改
    </p>
