    <h1 class="title-article" id="articleContentId">
     Linux 找回 Root 密码（多发行版本）
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     适用于大多数 Linux 发行版（Ubuntu、CentOS、Debian 等），无需第三方工具，
     <strong>
      需物理接触服务器或系统启动权限
     </strong>
     。
    </p>
    <hr/>
    <h5>
     <strong>
      步骤 1：重启系统并进入 GRUB 引导菜单
     </strong>
    </h5>
    <ol>
     <li>
      重启系统，看到启动画面时，
      <strong>
       快速按下
       <code>
        Esc
       </code>
       、
       <code>
        Shift
       </code>
       或
       <code>
        E
       </code>
      </strong>
      （不同系统按键不同，可多试几次）。
     </li>
     <li>
      进入
      <strong>
       GRUB 菜单
      </strong>
      后，选择要修改的系统内核选项（通常第一个），按
      <code>
       e
      </code>
      进入编辑模式。
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      步骤 2：修改内核启动参数
     </strong>
    </h5>
    <ol>
     <li>
      找到以
      <code>
       linux
      </code>
      或
      <code>
       linux16
      </code>
      或
      <code>
       linuxefi
      </code>
      开头的行（可能显示为
      <code>
       ro quiet splash
      </code>
      ），
      <strong>
       将
       <code>
        ro
       </code>
       改为
       <code>
        rw init=/bin/bash
       </code>
      </strong>
      （部分系统需改为
      <code>
       rw single
      </code>
      ）。
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>linux /vmlinuz-xxx root=/dev/sda1 **rw init=/bin/bash**
</code></pre>
      <ul>
       <li>
        <strong>
         目的
        </strong>
        ：以可读写（
        <code>
         rw
        </code>
        ）模式启动，并直接进入
        <strong>
         单用户模式
        </strong>
        （无需密码）。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      步骤 3：挂载文件系统并修改密码
     </strong>
    </h5>
    <ol>
     <li>
      按下
      <code>
       Ctrl+X
      </code>
      或
      <code>
       F10
      </code>
      启动到命令行界面。
     </li>
     <li>
      <strong>
       重新挂载根分区为读写模式
      </strong>
      （部分系统已通过参数自动挂载）：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>mount -o remount,rw /
</code></pre>
     </li>
     <li>
      <strong>
       修改 Root 密码
      </strong>
      ：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>passwd root    # 输入两次新密码
</code></pre>
     </li>
     <li>
      <strong>
       确保 SELinux 重新标记
      </strong>
      （如系统使用 SELinux）：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>touch /.autorelabel  # CentOS/RHEL 需要此步骤
</code></pre>
     </li>
     <li>
      重启系统：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>exec /sbin/init      # 或直接按 Ctrl+Alt+Delete
</code></pre>
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      其他可能场景处理
     </strong>
    </h5>
    <h6>
     <strong>
      场景 1：系统使用 systemd（如 Ubuntu 18.04+、CentOS 7+）
     </strong>
    </h6>
    <ol>
     <li>
      在 GRUB 编辑模式中，修改
      <code>
       linux
      </code>
      行末尾为：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>rw init=/bin/bash
</code></pre>
     </li>
     <li>
      若提示文件系统错误，需指定根分区：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>mount -o remount,rw /sysroot
chroot /sysroot
passwd root
touch /.autorelabel  # 仅 SELinux 系统需要
exit
reboot
</code></pre>
     </li>
    </ol>
    <h6>
     <strong>
      场景 2：使用 Live CD/USB 修改密码
     </strong>
    </h6>
    <ol>
     <li>
      用 Linux Live USB 启动系统。
     </li>
     <li>
      挂载原系统的根分区（需替换
      <code>
       /dev/sda1
      </code>
      为实际分区）：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>sudo mount /dev/sda1 /mnt
</code></pre>
     </li>
     <li>
      修改密码：
      <pre></pre>
      <p>
       &lt;BASH&gt;
      </p>
      <pre><code>sudo chroot /mnt
passwd root
exit
sudo reboot
</code></pre>
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      注意事项
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       安全限制
      </strong>
      ：
      <ul>
       <li>
        要求
        <strong>
         物理访问机器
        </strong>
        ，虚拟机需控制台权限。
       </li>
       <li>
        若启用了全盘加密（如 LUKS），需先解密磁盘。
       </li>
       <li>
        <strong>
         Secure Boot 可能阻止此操作
        </strong>
        ，需在 BIOS/UEFI 中临时关闭。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       生产环境
      </strong>
      ：
      <ul>
       <li>
        修改密码后检查系统日志（
        <code>
         /var/log/secure
        </code>
        或
        <code>
         /var/log/auth.log
        </code>
        ）。
       </li>
       <li>
        启用 SSH 密钥认证或
        <code>
         sudo
        </code>
        权限，减少依赖 Root 密码。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <p>
     <strong>
      示例流程图
     </strong>
    </p>
    <pre></pre>
    <p>
     &lt;PLAINTEXT&gt;
    </p>
    <pre><code>重启 → GRUB菜单 → 修改启动参数 → 进入单用户模式 → 挂载rw → passwd → 重启生效
</code></pre>
    <p>
     通过上述步骤，可快速恢复 Root 权限，建议操作后加固系统安全！ 🔐
    </p>
