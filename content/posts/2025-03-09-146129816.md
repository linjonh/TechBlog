      <font face="华文楷体">
       <font size="6">
        <strong>
         1. URL 基本格式
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     平时我们俗称的 “网址” 其实就是说的
     <code>
      URL
     </code>
     (Uniform Resource Locator
     <code>
      统一资源定位符
     </code>
     )；
    </p>
    <p>
     互联网上的每个文件都有一个唯一的URL，它包含的信息指出文件的位置，以及浏览器应该怎么处理它；
    </p>
    <p>
     URL的详细规则由因特网标准
     <code>
      RFC1738
     </code>
     进行了约定.(
     <a href="https://datatracker.ietf.org/doc/html/rfc1738" rel="nofollow">
      因特网标准 RFC1738
     </a>
     )
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/49724fd589c649318a7ed94b25936b34.png"/>
    </p>
    <p>
     一个具体的URL:
    </p>
    <blockquote>
     <p>
      <a href="https://v.bitedu.vip/personInf/student?userId=10000&amp;classId=100" rel="nofollow">
       https://v.bitedu.vip/personInf/student?userId=10000&amp;classId=100
      </a>
     </p>
    </blockquote>
    <p>
     可以看到，在这个URL中有些信息被省略了；
    </p>
    <hr/>
    <p>
     <strong>
      https:
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ffb0688c17484d0cbaba87e4dde804a7.png"/>
    </p>
    <ul>
     <li>
      协议方案名；
     </li>
     <li>
      常见的有http和https，也有其他的类型；(例如访问mysql时用的jdbc:mysql)
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      user:pass:
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4780e4c45a7b4428aa365c175ffb14ca.png"/>
    </p>
    <ul>
     <li>
      登陆信息；
     </li>
     <li>
      现在的网站进行身份认证，一般不再通过URL进行了，一般都会省略
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      v.bitedu.vip:
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f3ec2dd75c4943bea045b3aa9279d1af.png"/>
    </p>
    <ul>
     <li>
      <font color="red">
       服务器地址（IP地址），用于定位一个主机；
      </font>
     </li>
     <li>
      此处是一个"域名"，域名会通过
      <code>
       DNS 系统
      </code>
      解析成一个具体的IP地址；
     </li>
     <li>
      (通过
      <code>
       ping
      </code>
      命令可以看到，v.bitedu.vip 的真实IP地址为
      <code>
       118.24.113.28
      </code>
      )
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      端口号：
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1935af283aa2401081d019cb24495e87.png"/>
    </p>
    <ul>
     <li>
      每个应用程序在使用网络时，都会关联一个空闲的端口号；
     </li>
     <li>
      <font color="red">
       通过 IP 定位一台主机，通过端口号区分主机上使用网络的多个应用程序；
      </font>
     </li>
     <li>
      当端口号省略的时候，浏览器会根据
      <code>
       协议类型
      </code>
      自动决定使用哪个端口；
     </li>
     <li>
      例如 http 协议默认使用
      <code>
       80端口
      </code>
      ，https协议默认使用
      <code>
       443端口
      </code>
      ；
     </li>
    </ul>
    <blockquote>
     <p>
      回忆五元组：源端口，源 ip，目的端口，目的 ip，协议类型；
     </p>
     <p>
      http 协议默认使用
      <code>
       80端口
      </code>
      ，https协议默认使用
      <code>
       443端口
      </code>
      ，站在客户端的角度，两种协议访问的端口是
      <code>
       远端服务器的端口(目的端口)
      </code>
      ，不是浏览器自身的
      <code>
       客户端端口
      </code>
      (源端口)；
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      /personInf/student:
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ca79cb2e905a4242a9c187f1841e0fa8.png"/>
    </p>
    <ul>
     <li>
      带层次结构的
      <code>
       文件路径
      </code>
      ，每一层都是一个目录；
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      userId=10000&amp;classId=100:
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3122aa56c22d4202a7a79db9fc05f577.png"/>
    </p>
    <ul>
     <li>
      查询字符串
      <code>
       (query string)
      </code>
      ，本质是一个
      <code>
       键值对
      </code>
      结构；
     </li>
     <li>
      键值对之间使用
      <code>
       &amp;
      </code>
      分隔；
     </li>
     <li>
      键和值之间使用
      <code>
       =
      </code>
      分隔.
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      片段标识：
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cb76f578cbeb4de89ee4cb1eef92eb46.png"/>
    </p>
    <ul>
     <li>
      片段标识主要用于
      <code>
       页面内跳转
      </code>
      .
     </li>
    </ul>
    <blockquote>
     <p>
      例如Vue官方文档：
     </p>
     <p>
      <a href="https://cn.vuejs.org/v2/guide/#%E8%B5%B7%E6%AD%A5" rel="nofollow">
       https://cn.vuejs.org/v2/guide/#%E8%B5%B7%E6%AD%A5
      </a>
     </p>
    </blockquote>
    <ul>
     <li>
      通过不同的片段标识跳转到文档的不同章节
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="font_color__bluefont_facefont_size6strong2__ping_IP_strongfontfontfont_128">
     </a>
     <font color="blue">
      <font face="华文楷体">
       <font size="6">
        <strong>
         2. 使用 ping 命令查看域名对应的IP地址
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <ol>
     <li>
      在开始菜单中输入
      <code>
       cmd
      </code>
      ，打开命令提示符
     </li>
     <li>
      在
      <code>
       cmd
      </code>
      中输入
      <code>
       ping v.bitedu.vip
      </code>
      ，即可看到
      <code>
       域名解析
      </code>
      的结果.
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/85d6703a22394371b22ad20ba8f8b32e.png"/>
    </p>
    <blockquote>
     <p>
      PS：
     </p>
     <p>
      有的电脑上 ping 命令会报错 ping
      <code>
       不是内部或外部命令，也不是可运行的程序或批处理文件
      </code>
      ；
     </p>
     <p>
      这种情况是因为有的 Windows10 默认没有启用 ping 命令；
     </p>
     <p>
      百度搜索 windows10 启用 ping 即可；
     </p>
    </blockquote>
    <hr/>
    <h4>
     <a id="font_color__bluefont_facefont_size6strong3_query_stringstrongfontfontfont_147">
     </a>
     <font color="blue">
      <font face="华文楷体">
       <font size="6">
        <strong>
         3. query string
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <ul>
     <li>
      query string中的内容是
      <code>
       键值对结构
      </code>
      ；
     </li>
     <li>
      其中的 key 和 value 的
      <code>
       取值
      </code>
      和
      <code>
       个数
      </code>
      ，完全都是程序猿自己约定的；
     </li>
     <li>
      我们可以通过这样的方式来自定制传输我们需要的信息给服务器.
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="font_color__bluefont_facefont_size6strong4_URLstrongfontfontfont_157">
     </a>
     <font color="blue">
      <font face="华文楷体">
       <font size="6">
        <strong>
         4. URL中的可省略部分
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     <strong>
      协议名：
     </strong>
    </p>
    <ul>
     <li>
      可以省略，省略后默认为http://
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      ip地址/域名：
     </strong>
    </p>
    <ul>
     <li>
      在
      <code>
       HTML
      </code>
      中可以省略(比如 img , link , script , a 标签的
      <code>
       src
      </code>
      或者
      <code>
       href
      </code>
      属性)；
     </li>
     <li>
      省略后，表示
      <code>
       服务器的 ip/域名
      </code>
      与当前
      <code>
       HTML所属的 ip/域名
      </code>
      一致
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      端口号：
     </strong>
    </p>
    <ul>
     <li>
      可以省略，省略后如果是
      <code>
       http
      </code>
      协议，端口号自动设为
      <code>
       80
      </code>
      ;
     </li>
     <li>
      如果是
      <code>
       https
      </code>
      协议，端口号自动设为
      <code>
       443
      </code>
      .
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      带层次的文件路径：
     </strong>
    </p>
    <ul>
     <li>
      可以省略，省略后相当于
      <code>
       /
      </code>
      ；
     </li>
     <li>
      有些服务器会在发现
      <code>
       /路径
      </code>
      的时候自动访问
      <code>
       /index.html
      </code>
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      查询字符串：
     </strong>
    </p>
    <ul>
     <li>
      可以省略
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      片段标识：
     </strong>
    </p>
    <ul>
     <li>
      可以省略
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="font_color__bluefont_facefont_size6strong5_URL_encodestrongfontfontfont_200">
     </a>
     <font color="blue">
      <font face="华文楷体">
       <font size="6">
        <strong>
         5. 关于URL encode
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     像
     <code>
      /
     </code>
     <code>
      ?
     </code>
     <code>
      :
     </code>
     <code>
      #
     </code>
     <code>
      &amp;
     </code>
     <code>
      =
     </code>
     …等这样的字符，已经被 URL 当做特殊意义理解了；因此这些字符不能随意出现；
    </p>
    <hr/>
    <table>
     <tbody>
      <tr>
       <td bgcolor="pink">
        <font color="red">
         <font face="华文楷体">
          <font size="4">
           <strong>
            某个参数中需要带有这些特殊字符，就必须先对特殊字符进行转义；
           </strong>
          </font>
         </font>
        </font>
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <ul>
      <li>
       <p>
        一个
        <code>
         中文字符
        </code>
        等非英文系的文字，由
        <code>
         UTF-8
        </code>
        或者
        <code>
         GBK
        </code>
        这样的编码方式构成，虽然在URL中没有特殊含义，但是仍然需要进行
        <code>
         转义
        </code>
        ；
       </p>
      </li>
      <li>
       <p>
        否则
        <code>
         浏览器
        </code>
        可能把 UTF-8/GBK 编码中的
        <code>
         某个字节
        </code>
        ，当做
        <code>
         URL中的特殊符号
        </code>
        ；
       </p>
      </li>
      <li>
       <p>
        只不过很多浏览器为了用户看起来方便，显示 URL 某个参数的时候，
        <code>
         显示的是转义之前的字符
        </code>
        ;
       </p>
      </li>
      <li>
       <p>
        实际上，我们可以
        <code>
         通过抓包查看真实的数据
        </code>
        ，已经是转义之后的数据；
       </p>
      </li>
     </ul>
     <hr/>
     <p>
      我们在搜索框中输入中文，可以看见 URL 的其中一个
      <code>
       query string 键值对
      </code>
      中，对
      <code>
       属性
      </code>
      的赋值还是中文；
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/642034264d5f430888b92bef73a9620e.png"/>
     </p>
     <hr/>
     <p>
      我们使用
      <code>
       fiddler
      </code>
      来对刚刚的访问操作进行抓包：
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f27d0794d7674a1d852613a9f3c26b1c.png"/>
     </p>
     <hr/>
     <p>
      在记事本中打开
      <code>
       HTTP请求
      </code>
      的详细信息，我们发现属性赋值的真实数据是”你好“转义后的结果：
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bad9047c5d4f4f45b14506e834ccb82a.png"/>
     </p>
     <hr/>
     <p>
      <font color="red">
       转义这样的操作非常重要，如果没有进行转义，就可能会使浏览器或者服务器无法正确解析，请求就会失败。
      </font>
     </p>
    </blockquote>
    <hr/>
    <table>
     <tbody>
      <tr>
       <td bgcolor="pink">
        <font color="red">
         <font face="华文楷体">
          <font size="4">
           <strong>
            转义的规则如下：
           </strong>
          </font>
         </font>
        </font>
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <ul>
      <li>
       <p>
        将需要转码的字符转为
        <code>
         16进制
        </code>
        ，然后
        <code>
         从右到左
        </code>
        ；
       </p>
      </li>
      <li>
       <p>
        取4位(不足4位直接处理)，每2位做一位，前面加上%，编码成
        <code>
         %XY
        </code>
        格式；
       </p>
      </li>
     </ul>
     <hr/>
     <p>
      例如：
     </p>
     <p>
