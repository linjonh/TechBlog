    <h2>
     <a id="_5">
     </a>
     搭建平台
    </h2>
    <p>
     按照
     <a href="https://github.com/mengning/DevStar">
      DevStar的Github仓库
     </a>
     要求，在终端中执行下列命令，即可成功安装DevStar到本地部署。
     <strong>
      如遇网络问题，需要更换可用的docker镜像源。
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token function">wget</span> <span class="token parameter variable">-c</span> https://devstar.cn/assets/install.sh <span class="token operator">&amp;&amp;</span> <span class="token function">chmod</span> +x install.sh <span class="token operator">&amp;&amp;</span> <span class="token function">sudo</span> ./install.sh
devstar <span class="token builtin class-name">help</span>
<span class="token function">sudo</span> devstar start <span class="token parameter variable">--port</span><span class="token operator">=</span><span class="token number">8081</span>
</code></pre>
    <p>
     执行结果如下图所示：
     <br/>
     <img alt="运行结果1" src="https://i-blog.csdnimg.cn/direct/6273eae6db4344abba8ade9ded356627.png">
      <br/>
      <img alt="运行结果2" src="https://i-blog.csdnimg.cn/direct/76fe9a87d00b42998356d1e7f02c15b6.png">
       <br/>
       这里我选择在8081号端口运行，故加了后缀
       <code>
        --port=8081
       </code>
       。
       <br/>
       随后在浏览器中访问
       <a href="http://localhost:8081" rel="nofollow">
        http://localhost:8081
       </a>
       即可进入DevStar平台，如下图所示。
       <br/>
       <img alt="配置页面" src="https://i-blog.csdnimg.cn/direct/1b28faeb374a473cb65511dbc354a3bc.png">
        <br/>
        这里的配置选项默认没有改动，点击最下方按钮
        <code>
         Install Gitea
        </code>
        ，等待几秒会跳转到注册页面，按要求扫码后再注册账号，第一个注册的账号会默认为管理员。
        <br/>
        <img alt="注册页面" src="https://i-blog.csdnimg.cn/direct/697ddc5abd6046b7aec36eb140bca05d.png">
         <br/>
         <img alt="登录页面" src="https://i-blog.csdnimg.cn/direct/c3005468543b4bc59b540bc2d47d5c25.png">
          <br/>
          至此，DevStar平台的本地部署已完成。
         </img>
        </img>
       </img>
      </img>
     </img>

    <h2>
     <a id="Git_23">
     </a>
     Git基本命令的使用
    </h2>
    <h3>
     <a id="_24">
     </a>
     本地仓库的创建
    </h3>
    <p>
     在页面左上角找到按钮
     <code>
      New Repository
     </code>
     ，进入配置页。
     <br/>
     <img alt="创建仓库" src="https://i-blog.csdnimg.cn/direct/2bd4b5c79cf64c899575f8f80ffedaa3.png">
      <br/>
      <img alt="仓库配置" src="https://i-blog.csdnimg.cn/direct/318e6c671ab44cdc9fabdb8d4331fcf0.png"/>
     </img>
    </p>
    <p>
     图中修改已用红框标出。这里把
     <code>
      Default Branch
     </code>
     的内容修改成了
     <code>
      master
     </code>
     ，因为
     <strong>
      本地Git默认的分支名称为master，改成一致为了输入指令时不用再修改
     </strong>
     。点击
     <code>
      Create Repository
     </code>
     即可创建仓库。
     <br/>
     <img alt="仓库界面" src="https://i-blog.csdnimg.cn/direct/5acb0c489c0842259ac46d172b85ca11.png"/>
    </p>
    <h3>
     <a id="_31">
     </a>
     代码提交
    </h3>
    <p>
     首先在本地创建一个新的文件夹用于存放文件，然后在终端中运行下列指令。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> log <span class="token comment"># 查看git日志</span>
<span class="token function">git</span> init <span class="token comment"># 初始化当前目录的本地仓库，生成.git文件</span>
<span class="token function">git</span> remote <span class="token function">add</span> origin http://localhost:8081/ltdz/TestRepo.git <span class="token comment"># 将本地仓库与远程仓库关联，名为origin</span>
<span class="token function">git</span> <span class="token function">add</span> hello.c <span class="token comment"># 把当前目录中的hello.c文件加入到暂存区，便于后续提交</span>
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Added hello.c"</span> <span class="token comment"># 将暂存区提交到本地仓库</span>
<span class="token function">git</span> push <span class="token parameter variable">-u</span> origin master <span class="token comment"># 将本地分支的提交推送到远程仓库，，第一次推送要用 -u 确认关系</span>
</code></pre>
    <p>
     <img alt="命令行1" src="https://i-blog.csdnimg.cn/direct/2e0fa40257e14bb68783c80ee5b1ae05.png"/>
     <br/>
     <img alt="命令行2" src="https://i-blog.csdnimg.cn/direct/5449f33ba1cf4a83a642932878c053f6.png"/>
     <br/>
     <strong>
      注意在最后一步推送到远程仓库时需要验证DevStar的身份，用户名和密码即注册时的数据
     </strong>
     ，随后即可在仓库中发现新添加的文件
     <code>
      hello.c
     </code>
     。
    </p>
    <p>
     <img alt="提交代码" src="https://i-blog.csdnimg.cn/direct/4e737d721cfd4fcea189f47355810639.png"/>
    </p>
    <h3>
     <a id="_49">
     </a>
     代码合并
    </h3>
    <p>
     在终端中运行下列命令，给仓库创建新的分支：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> branch main <span class="token comment"># 创建新分支main</span>
<span class="token function">git</span> checkout main <span class="token comment"># 切换到分支main</span>
<span class="token function">git</span> branch <span class="token comment"># 查看当前分支</span>
<span class="token function">git</span> push <span class="token parameter variable">-u</span> origin main <span class="token comment"># 将新创建的分支推送到远程仓库</span>
</code></pre>
    <p>
     <img alt="创建分支" src="https://i-blog.csdnimg.cn/direct/043e1c68fd294421898e479d22320e33.png"/>
     <br/>
     随后即可在仓库中发现新创建的
     <code>
      main
     </code>
     分支。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/061d1cbf6e324735873f8f2136924cbe.png"/>
     <br/>
     再运行下列指令，在新创建的
     <code>
      main
     </code>
     分支下提交已经修改后的
     <code>
      hello.c
     </code>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> hello.c
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Modified hello.c from branch main"</span>
<span class="token function">git</span> push origin main
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/974ebe77c52a4b6e948950bce6346faa.png"/>
     <br/>
     随后即可在仓库中的
     <code>
      main
     </code>
     分支下发现对代码的修改，
     <strong>
      注意此时
      <code>
       master
      </code>
      分支下的代码内容没有变化
     </strong>
     ，图中区别已用红框标出。
     <br/>
     <img alt="master分支" src="https://i-blog.csdnimg.cn/direct/88287752fb5447b4af5cfd6970dc55f4.png"/>
     <br/>
     <img alt="main分支" src="https://i-blog.csdnimg.cn/direct/82ef1e98a80b46369b58bdf244984ab2.png"/>
     <br/>
     此时再切换回
     <code>
      master
     </code>
     分支，执行下列指令，即可将
     <code>
      main
     </code>
     分支的改动合并到
     <code>
      master
     </code>
     分支。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> merge main <span class="token comment"># 将main分支的改动合并过来</span>
<span class="token function">git</span> push origin master <span class="token comment"># 推送到远程仓库</span>
</code></pre>
    <p>
     <img alt="分支合并" src="https://i-blog.csdnimg.cn/direct/a82b1ca76c854e20b7673dd149e0d137.png"/>
     <br/>
     此时
     <code>
      master
     </code>
     分支下的代码内容已经因为代码合并更新。
     <br/>
     <img alt="合并结果" src="https://i-blog.csdnimg.cn/direct/f09851cbc52943c2b8453e3ab1eaf1a9.png"/>
    </p>
    <h3>
     <a id="_83">
     </a>
     版本发布
    </h3>
    <p>
     在终端中运行下列指令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> tag <span class="token parameter variable">-a</span> v1.0.0 <span class="token parameter variable">-m</span> <span class="token string">"Release Version 1.0.0"</span> <span class="token comment"># 定义标签</span>
<span class="token function">git</span> push origin v1.0.0 <span class="token comment"># 推送到远程仓库</span>
<span class="token function">git</span> tag <span class="token comment"># 查看所有标签</span>
</code></pre>
    <p>
     <img alt="版本发布" src="https://i-blog.csdnimg.cn/direct/8911ae89a01342599c1d5c1f4c267424.png"/>
     <br/>
     对应在仓库中也会显示新的版本：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b96d14d5531a45efb94eb33c9eee25de.png"/>
     <br/>
     在对应的Releases界面点击
     <code>
      New Release
     </code>
     即可创建新的Release并发布。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6f3a37d6a26a4a16a6c6a6e35348d1ce.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e51808d377714fe7afa76ba9af5baf32.png"/>
    </p>
    <h2>
     <a id="_98">
     </a>
     总结
    </h2>
    <p>
     通过本次实验，我深入学习了如何在本地搭建Git仓库并熟练掌握了Git的基本命令，包括仓库的创建、代码的提交、分支的合并以及版本的发布。我通过实际操作掌握了
     <code>
      git init
     </code>
     、
     <code>
      git add
     </code>
     、
     <code>
      git commit
     </code>
     、
     <code>
      git push
     </code>
     、
     <code>
      git branch
     </code>
     、
     <code>
      git merge
     </code>
     和
     <code>
      git tag
     </code>
     等核心命令的使用方法；我通过创建和切换分支学会了如何在不同的分支上进行开发，并将修改合并到主分支。这让我理解了Git分支管理的强大功能，尤其是在多人协作开发中的应用。我通过打标签和发布版本，掌握了如何为代码库的重要节点做标记，并通过代码仓库的Release功能发布版本。这让我对软件版本管理有了更清晰的认识。本次实验的设计非常贴合实际开发需求，内容丰富且循序渐进，让我从零开始逐步掌握了Git的核心技能。希望未来还能有更多这样高质量的实验课程！
    </p>
