    <h4>
     <a id="_usrlocal__15">
     </a>
     关于 /usr/local 目录
    </h4>
    <p>
     macOS 和 Linux 系统比较接近，其软件安装目录是也是有讲究的，理解这一点，对系统管理是很有帮助的：
    </p>
    <ol>
     <li>
      /usr：系统级的目录，可以理解为C:/Windows/，/usr/lib理解为C:/Windows/System32。
     </li>
     <li>
      /usr/local：用户级的程序目录，可以理解为C:/Progrem Files/。用户自己编译的软件默认会安装到这个目录下。
     </li>
     <li>
      /opt：用户级的程序目录，可以理解为D:/Software，opt有可选的意思，这里可以用于放置第三方大型软件（或游戏），当你不需要时，直接 rm -rf 掉即可。在硬盘容量不够时，也可将/opt单独挂载到其他磁盘上使用。
     </li>
    </ol>
    <h4>
     <a id="_usrlocal__24">
     </a>
     打开 /usr/local 所在目录的方法
    </h4>
    <ol>
     <li>
      macOS 系统的 /usr/local 目录默认在 Finder 下是隐藏的，如果需要到 /usr/local，可以打开 Finder，然后使用 command + shift + G，在弹出的目录中填写 /usr/local就可以了，也可以在 Finder 上右键点击前往文件夹（相当于 command + shift + G）。
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4cfc6a00db754626a76f76a3addeee60.png"/>

    <ol start="2">
     <li>
      还有一种查看方式就是在命令行（终端）中：
      <code>
       cd /usr/local
      </code>
      <br/>
      然后再使用 ls 就可以查看下面安装的东西了。
     </li>
    </ol>

    <h4>
     <a id="macOS__44">
     </a>
     macOS 下配置环境变量
    </h4>
    <p>
     macOS 的环境变量一般配置在 .bash_profile 或者 .zshrc 文件里，所以只要将下载的安装包路径写入 .bash_profile 或者 .zshrc 并完成保存就好了。
    </p>
    <h4>
     <a id="macOS__bash_profile__zshrc__48">
     </a>
     macOS 下 .bash_profile 和 .zshrc 两者之间的区别
    </h4>
    <p>
     .bash_profile 中修改环境变量只对当前窗口有效，当关闭窗口后再使用可能会报
     <code>
      zsh: command not found: XXX
     </code>
     ; 而且需要
     <code>
      source ~/.bash_profile
     </code>
     才能使用;
    </p>
    <p>
     .zshrc 则相当于 windows 的开机启动的环境变量, 永久有效。
    </p>
    <p>
     所以建议尽量只使用 .zshrc，然后加一行
     <code>
      source .bash_profile
     </code>
     以兼容不小心加入 .bash_profile 文件的变量。
    </p>
    <h4>
     <a id="_bash_profile__56">
     </a>
     编辑 .bash_profile 文件
    </h4>
    <ol>
     <li>
      打开终端
     </li>
     <li>
      vi ~/.bash_profile 或者 open ~/.bash_profile 打开文件
     </li>
     <li>
      修改后 通过 :wq 保存
     </li>
     <li>
      source ~/.bash_profile 生效
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ebc437cb10d34c748f99f229cd2f8271.png"/>

    <h4>
     <a id="_zshrc__66">
     </a>
     编辑 .zshrc 文件
    </h4>
    <p>
     只有当在 Mac OS 上使用 zsh shell 时，才会获得 ~/.zshrc 文件，如果你不确定自己使用的是哪个 shell，请打开终端并发出以下命令：
    </p>
    <pre><code>echo $SHELL
</code></pre>
    <p>
     若结果显示为 /bin/zsh， 说明你在 macOS 上使用的是 zsh shell
    </p>
    <ol>
     <li>
      open ~/.zshrc 或者 vim ~/.zshrc
     </li>
     <li>
      在打开的 .zshrc 文件窗口中进行更改
     </li>
     <li>
      通过 :wq 保存
     </li>
     <li>
      source ~/.zshrc
     </li>
    </ol>

    <h4>
     <a id="homebrew__npm__96">
     </a>
     homebrew 和 npm 的区别？
    </h4>
    <ol>
     <li>
      <p>
       homebrew，可以理解成 macOS 的软件管理工具，粗俗点说就是 mac 界的 qq 软件助手之类的东西。所以通过 brew，安装什么 chrome浏览器啊、atom 编辑器之类的可视化工具也是可以的。
      </p>
     </li>
     <li>
      <p>
       npm，是 node.js 界的程序/模块管理工具，也就是说 npm 只管理那些服务于 JavaScript 社区的程序。而且跨平台，windows 和 macOS，以及其他类 unix 操作系统都可以用。
      </p>
     </li>
     <li>
      <p>
       npm是用于 NodeJS 语言的包管理器，NodeJS 是跨平台的；而 homebrew 是用于OS X系统的包管理器，类似Windows的各种软件管理工具（所谓XX软件市场之类）和Linux的apt-get/yum/pacman等。
      </p>
     </li>
     <li>
      <p>
       npm 是 node.js 的包管理工具，只要有 node 环境，不管是 windows, macOS, 还是 linux 都可以使用 npm 下载模块，brew 是 mac 的包管理工具，只有 macOS 上才有。
      </p>
     </li>
    </ol>
    <p>
     引用链接：https://www.jianshu.com/p/131dda5e51fe
    </p>
    <p>
     npm部分命令如下:
    </p>
    <pre><code>npm -l    用于查看各个命令的简单用法（所以下面的可以用这个命令来查看）
npm init    用来初始化生成一个新的 package.json文件。
它会向用户提问一系列问题，如果你觉得不用修改默认配置，一路回车就可以了。
npm -h 或 npm help    查看npm命令的帮助信息
npm ls 或 npm list    查看npm已安装的包信息
npm -v 或 npm --version    查看npm版本信息
npm install npm -g   npm更新自身
npm info &lt;pkg&gt; version    查看某个模块最新发布版本信息，如npm info underscore version
npm search &lt;keyword&gt;    查找与keyword匹配的模块信息
npm view &lt;pkg&gt; version    查看一个包的最新发布版本
npm i 或 npm install    npm安装当前目录package.json里面的所有包，
下面的i同样可以用install代替，当卸载时，i用uninstall代替
npm update &lt;pkg&gt; [-g]  更新指定模块，有-g表示全局
npm i &lt;pkg&gt; [-g]    安装指定模块，有-g表示全局
npm i &lt;pkg&gt;@version [-g]   安装指定版本的模块，有-g表示全局
npm i &lt;pkg&gt; --save    安装包的同时自动更新package.json的依赖
npm i &lt;pkg&gt; --save-dev    安装包的同时自动更新package.json的开发依赖
npm i &lt;pkg&gt; --save-optional    安装包的同时自动更新package.json的可选版本依赖
npm i &lt;pkg&gt; --save-exact   安装包并写入确切版本依赖，而不是一个可选的版本范围.
</code></pre>
    <p>
     homebrew部分命令如下:
    </p>
    <pre><code>brew install &lt;pkg&gt;    安装软件包
brew uninstall &lt;pkg&gt;    卸载软件包
brew search &lt;keyword&gt;    查询软件包
brew list 或brew ls    列出已安装的软件包
brew update    更新brew
brew home    用浏览器打开brew的官方网站
brew info    显示软件信息
brew deps    显示包依赖
</code></pre>

    <h4>
     <a id="nrm__187">
     </a>
     nrm 的安装与使用
    </h4>
    <p>
     nrm 是一个用来管理和快速切换私人 npm 配置的工具，可用来随意的切换 npm 镜像源，比如公司内网镜像，淘宝镜像等，建议全局安装;
    </p>
    <p>
     nrm 是 node模块，所以通过 npm 来安装，它是一个命令行模块，需要使用–global参数进行全局安装，命令如下：
    </p>
    <pre><code>npm install nrm -g --save 全局安装

nrm ls 查看默认配置，这里会列出所有的镜像

nrm current 查看当前使用的是哪个源

nrm use &lt;源名&gt; 切换到其它源，如 nrm use cnpm

nrm add 命令可添加公司私有 npm 源，如 http://baozun.com(随便写的)，起个别名叫 baobao，nrm add baobao http://baozun.com
此时再执行 nrm ls，发现 baobao 已添加成功

nrm test npm 测速

nrm del baobao 删除源配置
</code></pre>

     </li>
     <li>
      点击 Sign in &amp; Turn on 按钮，选择微软或者 github 账号进行登录，登陆成功之后，Settings Sync就是开启状态了，VSCode会在后台自动同步你刚才所选的内容。
