    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       1.如何优化docker镜像的大小？
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      使用多阶段构建（multi-stage build）
     </li>
     <li style="text-align:justify">
      选择更小的基础镜像（如alpine）
     </li>
     <li style="text-align:justify">
      减少镜像层数，合并RUN命令
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       2.请解释Docker中的网络模式（如bridge,host,none)
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      Bridge：默认模式，容器通过虚拟网络桥接与主机通信
     </li>
     <li style="text-align:justify">
      Host：容器直接使用主机的网络栈
     </li>
     <li style="text-align:justify">
      none：容器没有网络接口
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       3.如何将本地开发的Docker容器部署到Kubernetes集群中？
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     将Docker镜像推送到镜像仓库（如Docker Hub）。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Kubernetes中创建Deployment，指定镜像地址。
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       4.你如何设计一个高可用的系统架构？请结合Kubernetes和MySQL等技术说明。
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      使用Kubernetes部署无状态应用，确保Pod的高可用性。
     </li>
     <li style="text-align:justify">
      使用MySQL主从复制或集群实现数据库高可用。
     </li>
     <li style="text-align:justify">
      使用负载均衡器分发流量。
     </li>
     <li style="text-align:justify">
      设置监控和自动恢复机制。
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       5.如果生产环境中的某个服务器突然出现性能下降，你会如何排查问题？
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      检查Kubernetes Pod的状态和日志。
     </li>
     <li style="text-align:justify">
      使用kubectl top 查看资源使用情况。
     </li>
     <li style="text-align:justify">
      检查数据库的慢查询。
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       6.你如何实现CI/CD流水线，并将其与Kubernetes集成？
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      使用GitLab构建CI/CD流水线。
     </li>
     <li style="text-align:justify">
      在流水线中构建Docker镜像并推送到镜像仓库。
     </li>
     <li style="text-align:justify">
      部署到Kubernetes：使用kubectl或HELM将镜像部署到Kubernetes集群。
     </li>
    </ol>
