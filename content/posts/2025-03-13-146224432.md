    <h2>
     <a id="ChromeOS_134__0">
     </a>
     ChromeOS 134 版本更新
    </h2>
    <h3>
     <a id="ChromeOS_134__1">
     </a>
     一、ChromeOS 134 更新内容
    </h3>
    <h4>
     <a id="1_ChromeOS_Kiosk_Web_Isolated_Web_Apps_2">
     </a>
     1.
     <strong>
      ChromeOS 自助终端（Kiosk）模式支持隔离 Web 应用（Isolated Web Apps）
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 134
     </strong>
     开始，自助终端（Kiosk）模式支持
     <strong>
      隔离 Web 应用（Isolated Web Apps，IWA）
     </strong>
     ，提供
     <strong>
      更安全、更强大
     </strong>
     的应用解决方案，并可
     <strong>
      深度集成系统功能
     </strong>
     。
    </p>
    <h5>
     <a id="11__5">
     </a>
     1.1. 主要特点
    </h5>
    <ol>
     <li>
      <strong>
       更高的安全性
      </strong>
      ：IWA 运行在
      <strong>
       隔离环境
      </strong>
      中，减少安全风险。
     </li>
     <li>
      <strong>
       更强的系统集成能力
      </strong>
      ：支持
      <strong>
       访问系统级功能
      </strong>
      ，增强 Kiosk 设备的应用能力。
     </li>
     <li>
      <strong>
       适用于 Kiosk 和数字标牌（Digital Signage）
      </strong>
      ：管理员可通过
      <strong>
       Google 管理控制台（Admin Console）
      </strong>
      配置
      <strong>
       IWAs
      </strong>
      ，用于
      <strong>
       自助终端和数字标牌部署
      </strong>
      。
     </li>
    </ol>
    <h4>
     <a id="2__10">
     </a>
     2.
     <strong>
      为毕业生迁移数据
     </strong>
    </h4>
    <p>
     <strong>
      最早从 2025 年 3 月起
     </strong>
     ，ChromeOS 将推出
     <strong>
      新的内容迁移工具
     </strong>
     ，帮助
     <strong>
      毕业生或其他受教育机构管理（EDU-managed）用户
     </strong>
     通过
     <strong>
      更新的数据迁移流程
     </strong>
     迁移其数据。
    </p>
    <h5>
     <a id="21__13">
     </a>
     2.1. 功能介绍
    </h5>
    <ul>
     <li>
      <strong>
       引导学生迁移数据
      </strong>
      ：支持将
      <strong>
       Docs、Sheets、Slides 和 Gmail 内容
      </strong>
      迁移到
      <strong>
       个人 Gmail 账户
      </strong>
      。
     </li>
     <li>
      <strong>
       学校管理员可进行配置
      </strong>
      ：
      <ul>
       <li>
        <strong>
         在 Chromebook 任务栏（Shelf）固定迁移工具图标
        </strong>
        ，方便学生使用。
       </li>
       <li>
        <strong>
         向学生和教职员工发送 Chromebook 迁移通知
        </strong>
        ，提醒他们及时转移数据。
       </li>
       <li>
        <strong>
         设置提醒时间
        </strong>
        ，在指定日期触发通知，鼓励学生使用迁移工具。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/fb0003fc687b79cf8c18cb52dc3f54f8.png"/>
    </p>
    <h4>
     <a id="3_Slow_Keys_22">
     </a>
     3.
     <strong>
      慢速按键（Slow Keys）
     </strong>
    </h4>
    <p>
     <strong>
      慢速按键（Slow Keys）
     </strong>
     是一项
     <strong>
      辅助功能
     </strong>
     ，旨在帮助
     <strong>
      手部灵活性受限的用户
     </strong>
     更准确地输入文本。
    </p>
    <h5>
     <a id="31__25">
     </a>
     3.1. 功能介绍
    </h5>
    <ul>
     <li>
      <strong>
       适用于手部颤抖、关节炎、手指麻木等情况
      </strong>
      ，帮助用户更精确地按键。
     </li>
     <li>
      <strong>
       按键延迟机制
      </strong>
      ：
      <ul>
       <li>
        用户必须
        <strong>
         按住按键
        </strong>
        一定时间，键才会被注册输入。
       </li>
       <li>
        <strong>
         防止误触
        </strong>
        ，减少因
        <strong>
         手部不稳
        </strong>
        造成的错误输入。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/351e9bc2d4aebc13dce17881a09bfb6f.png"/>
    </p>
    <h4>
     <a id="4_Quick_Insert_GIF__33">
     </a>
     4.
     <strong>
      快速插入（Quick Insert）支持 GIF 添加
     </strong>
    </h4>
    <p>
     <strong>
      Quick Insert
     </strong>
     即将支持
     <strong>
      直接插入 GIF
     </strong>
     ！
    </p>
    <h3>
     <a id="Chrome_OS__36">
     </a>
     二、Chrome OS 即将更新的内容
    </h3>
    <h4>
     <a id="1_ChromeOS__37">
     </a>
     1.
     <strong>
      ChromeOS 电池寿命管理策略
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 135
     </strong>
     开始，新增
     <strong>
      电池充电优化策略
     </strong>
     —— DevicePowerBatteryChargingOptimization，提供更多优化选项，以
     <strong>
      延长 Chromebook 的电池寿命
     </strong>
     。
    </p>
    <h5>
     <a id="11__40">
     </a>
     1.1. 主要功能
    </h5>
    <ul>
     <li>
      <strong>
       支持管理员设置最大充电限制
      </strong>
      ，默认值为
      <strong>
       100%
      </strong>
      ，可减少电池老化，提高长期可靠性。
     </li>
     <li>
      <strong>
       适用于大规模设备管理
      </strong>
      ，特别是
      <strong>
       教育机构或企业环境
      </strong>
      ，帮助优化设备使用寿命。
     </li>
     <li>
      <strong>
       适用于个人用户
      </strong>
      ，希望最大化
      <strong>
       Chromebook 的电池健康度
      </strong>
      。
     </li>
     <li>
      <strong>
       策略自动生效，无需用户干预
      </strong>
      。
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/54a3a149c0674168ed4f35b273846e28.png"/>
    </p>
    <h4>
     <a id="2_Kiosk_Heartbeat_48">
     </a>
     2.
     <strong>
      Kiosk 设备心跳（Heartbeat）更新
     </strong>
    </h4>
    <p>
     去年 Chrome 向客户发送了
     <strong>
      MSA（管理服务公告）
     </strong>
     ，通知
     <strong>
      Kiosk Heartbeat API 迁移
     </strong>
     ，要求设备
     <strong>
      更新至 ChromeOS 126
     </strong>
     以保持兼容性。
    </p>
    <h5>
     <a id="21__51">
     </a>
     2.1. 重要更新要求
    </h5>
    <ul>
     <li>
      <strong>
       如果您的设备运行的 ChromeOS 版本低于 126
      </strong>
      ，请尽快升级至：
      <ul>
       <li>
        <strong>
         ChromeOS 126
        </strong>
        或
       </li>
       <li>
        <strong>
         ChromeOS LTS 126
        </strong>
        或
       </li>
       <li>
        <strong>
         当前最新的 ChromeOS 稳定版本
        </strong>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       否则，Kiosk 设备的心跳功能将无法正常工作。
      </strong>
     </li>
    </ul>
    <h5>
     <a id="22__58">
     </a>
     2.2. 需要采取的措施
    </h5>
    <ul>
     <li>
      <strong>
       确保已激活 DeviceChromeVariation（设置值为 0），直到所有设备升级至 LTS 132。
      </strong>
     </li>
     <li>
      <strong>
       一旦所有设备更新到 LTS 132，即可移除 DeviceChromeVariation 限制，并将其更改为任意值。
      </strong>
     </li>
    </ul>
    <h5>
     <a id="23_DeviceChromeVariation__62">
     </a>
     2.3. DeviceChromeVariation 关闭后会发生什么？
    </h5>
    <ul>
     <li>
      <strong>
       在 2025 年 4 月 7 日之前，运行 LTS 132 之前的版本 的设备仍将继续使用旧的 Kiosk Heartbeat 基础架构。
      </strong>
     </li>
     <li>
      <strong>
       从 2025 年 4 月 8 日起，所有 ChromeOS 版本低于 LTS 132 的设备将 停止发送设备心跳信号，影响 Kiosk 设备的监控和管理。
      </strong>
     </li>
    </ul>
    <h4>
     <a id="3_Admin_Console_66">
     </a>
     3.
     <strong>
      基于事件的设备日志收集（用于故障排查，Admin Console）
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 135
     </strong>
     开始，为了帮助
     <strong>
      排查设备问题
     </strong>
     （如
     <strong>
      崩溃或操作系统更新失败
     </strong>
     ），系统将自动收集并上传设备日志。
    </p>
    <h5>
     <a id="31__69">
     </a>
     3.1. 触发日志收集的条件
    </h5>
    <p>
     <strong>
      系统日志收集仅在以下设置开启时生效：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       启用设备系统日志上传（Enable device system log upload）
      </strong>
      作用：开启整体日志收集功能。
     </li>
     <li>
      <strong>
       上报设备 OS 信息 → OS 更新状态（Report device OS information → OS update status）
      </strong>
      作用：收集
      <strong>
       操作系统更新相关数据
      </strong>
      ，帮助排查更新失败问题。
     </li>
     <li>
      <strong>
       上报设备遥测数据 → 崩溃信息（Report device telemetry → Crash information）
      </strong>
      作用：收集
      <strong>
       严重崩溃信息
      </strong>
      ，协助调试设备故障。
     </li>
    </ol>
    <h5>
     <a id="32__76">
     </a>
     3.2. 管理员如何查看日志？
    </h5>
    <ul>
     <li>
      当设备发生问题时，
      <strong>
       IT 管理员
      </strong>
      将在
      <strong>
       Admin Console（管理控制台）收到通知
      </strong>
      ，并可以在
      <strong>
       设备详情页
      </strong>
      直接查看上传的日志。
     </li>
     <li>
      <strong>
       管理员还可以查看设备事件历史记录
      </strong>
      ，帮助分析问题发生的时间和频率。
     </li>
    </ul>
    <h5>
     <a id="33__80">
     </a>
     3.3. 日志上传规则
    </h5>
    <ul>
     <li>
      <strong>
       每天最多上传 2 次
      </strong>
      ，避免占用过多带宽。
     </li>
     <li>
      <strong>
       单个日志文件大小一般在 400KB ~ 1MB 之间
      </strong>
      。
     </li>
    </ul>
    <h4>
     <a id="4_AI__84">
     </a>
     4.
     <strong>
      AI 壁纸与视频通话背景
     </strong>
    </h4>
    <p>
     最早从
     <strong>
      ChromeOS 136
     </strong>
     开始，Chrome 计划引入
     <strong>
      高分辨率的 AI 生成壁纸
     </strong>
     和
     <strong>
      视频通话背景
     </strong>
     。
    </p>
    <h5>
     <a id="41__87">
     </a>
     4.1. 功能介绍
    </h5>
    <ul>
     <li>
      <strong>
       个性化体验
      </strong>
      ：使用
      <strong>
       生成式 AI
      </strong>
      创建独特的
      <strong>
       壁纸
      </strong>
      和
      <strong>
       视频通话背景
      </strong>
      ，让你的 Chromebook 更具个性。
     </li>
     <li>
      <strong>
       丰富的模板选择
      </strong>
      ：提供多种风格的 AI 生成模板，可快速定制符合
      <strong>
       情绪、兴趣或个性
      </strong>
      的背景。
     </li>
     <li>
      <strong>
       仅适用于 Chromebook Plus 设备
      </strong>
      ，提供更高级的视觉体验。
     </li>
    </ul>
    <h4>
     <a id="5_ChromeOS__Chrome_Apps_92">
     </a>
     5.
     <strong>
      ChromeOS 即将停止支持 Chrome Apps
     </strong>
    </h4>
    <p>
     <strong>
      早在 2016 年
     </strong>
     ，Chrome 宣布逐步淘汰
     <strong>
      Chrome Apps
     </strong>
     ，并推广
     <strong>
      Web Apps
     </strong>
     作为替代方案。
    </p>
    <p>
     <strong>
      2021 年
     </strong>
     ，Chromium 官方博客宣布，ChromeOS 企业和教育用户以及开发者的
     <strong>
      Chrome Apps 支持
     </strong>
     将至少延长至
     <strong>
      2025 年 1 月
     </strong>
     。
    </p>
    <p>
     随着大多数用户已迁移至 Web Apps，现在已确认
     <strong>
      Chrome Apps 彻底停用的时间表
     </strong>
     。
    </p>
    <h5>
     <a id="51_Chrome_Apps__99">
     </a>
     5.1. Chrome Apps 停用时间表
    </h5>
    <h6>
     <a id="511__2025__7_ChromeOS_M139_100">
     </a>
     5.1.1. 📅 2025 年 7 月（ChromeOS M139）
    </h6>
    <ul>
     <li>
      <strong>
       用户自行安装的 Chrome Apps 停止支持
      </strong>
      。
     </li>
     <li>
      <strong>
       管理员强制安装（force-install）的 Chrome Apps 仍可继续使用
      </strong>
      。
     </li>
    </ul>
    <h6>
     <a id="512__2026__7_ChromeOS_M150_104">
     </a>
     5.1.2. 📅 2026 年 7 月（ChromeOS M150）
    </h6>
    <ul>
     <li>
      <strong>
       Kiosk 模式下的 Chrome Apps 支持将终止
      </strong>
      。
     </li>
     <li>
      <strong>
       LTS（长期支持）通道上的 Kiosk 设备可继续使用 Chrome Apps，支持延长至 2027 年 4 月
      </strong>
      。
     </li>
    </ul>
    <h6>
     <a id="513__2028__2_ChromeOS_M168_108">
     </a>
     5.1.3. 📅 2028 年 2 月（ChromeOS M168）
    </h6>
    <ul>
     <li>
      <strong>
       ChromeOS 最后一版支持 Chrome Apps 的版本
      </strong>
      ，正式宣布
      <strong>
       Chrome Apps 完全终止
      </strong>
      。
     </li>
     <li>
      <strong>
       LTS 设备仍可运行 Chrome Apps，直至 2028 年 10 月
      </strong>
      。
     </li>
    </ul>
    <h6>
     <a id="514___112">
     </a>
     5.1.4. 📌 官方声明：不提供任何例外情况。
    </h6>
