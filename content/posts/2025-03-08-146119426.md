    <h1 class="title-article" id="articleContentId">
     ES Filter Query 区别
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     在
     <strong>
      Elasticsearch（ES）
     </strong>
     中，
     <strong>
      Filter
     </strong>
     和
     <strong>
      Query
     </strong>
     是两种常用的数据检索方式，它们的主要区别在于
     <strong>
      是否计算相关性分数（Score）
     </strong>
     以及
     <strong>
      是否使用缓存
     </strong>
     。以下是它们的详细区别和应用场景：
    </p>
    <hr/>
    <h3>
     1.
     <strong>
      核心区别
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        Filter（过滤器）
       </th>
       <th>
        Query（查询）
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         相关性分数
        </strong>
       </td>
       <td>
        不计算分数，只判断文档是否匹配。
       </td>
       <td>
        计算分数，用于衡量文档与查询的相关性。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         性能
        </strong>
       </td>
       <td>
        性能更高，适合精确匹配。
       </td>
       <td>
        性能较低，适合全文检索。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         缓存
        </strong>
       </td>
       <td>
        结果会被缓存，重复查询时更快。
       </td>
       <td>
        结果不会被缓存。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         使用场景
        </strong>
       </td>
       <td>
        精确匹配（如范围查询、Term 查询）。
       </td>
       <td>
        全文检索（如 Match 查询）。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         排序
        </strong>
       </td>
       <td>
        不参与排序。
       </td>
       <td>
        参与排序（基于分数）。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     2.
     <strong>
      详细说明
     </strong>
    </h3>
    <h4>
     （1）
     <strong>
      Filter（过滤器）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         只判断文档是否匹配，不计算相关性分数。
        </p>
       </li>
       <li>
        <p>
         结果会被缓存，适合重复查询的场景。
        </p>
       </li>
       <li>
        <p>
         性能更高，因为不需要计算分数。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         精确匹配（如 Term 查询、Range 查询）。
        </p>
       </li>
       <li>
        <p>
         布尔条件判断（如
         <code>
          must
         </code>
         、
         <code>
          should
         </code>
         、
         <code>
          must_not
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         过滤不需要排序的数据。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <p>
       json
      </p>
      <p>
       复制
      </p>
      <pre>{
  "query": {
    "bool": {
      "filter": [
        { "term": { "status": "active" } },
        { "range": { "age": { "gte": 18, "lte": 30 } } }
      ]
    }
  }
}</pre>
     </li>
    </ul>
    <h4>
     （2）
     <strong>
      Query（查询）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         计算文档与查询的相关性分数（Score）。
        </p>
       </li>
       <li>
        <p>
         结果不会被缓存。
        </p>
       </li>
       <li>
        <p>
         适合全文检索和模糊匹配。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         全文检索（如 Match 查询）。
        </p>
       </li>
       <li>
        <p>
         需要排序的场景（基于分数）。
        </p>
       </li>
       <li>
        <p>
         模糊匹配（如 Fuzzy 查询）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <p>
       json
      </p>
      <p>
       复制
      </p>
      <pre>{
  "query": {
    "match": {
      "title": "elasticsearch"
    }
  }
}</pre>
     </li>
    </ul>
    <hr/>
    <h3>
     3.
     <strong>
      Filter 和 Query 的结合使用
     </strong>
    </h3>
    <p>
     在实际应用中，Filter 和 Query 通常会结合使用，以兼顾性能和相关性。例如：
    </p>
    <ul>
     <li>
      <p>
       使用
       <strong>
        Filter
       </strong>
       过滤掉不符合条件的文档。
      </p>
     </li>
     <li>
      <p>
       使用
       <strong>
        Query
       </strong>
       对剩余文档计算相关性分数。
      </p>
     </li>
    </ul>
    <h4>
     示例：
    </h4>
    <p>
     json
    </p>
    <p>
     复制
    </p>
    <pre>{
  "query": {
    "bool": {
      "must": [
        { "match": { "title": "elasticsearch" } }  // Query
      ],
      "filter": [
        { "term": { "status": "active" } },       // Filter
        { "range": { "age": { "gte": 18 } } }     // Filter
      ]
    }
  }
}</pre>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         must
        </code>
       </strong>
       ：用于 Query，计算相关性分数。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         filter
        </code>
       </strong>
       ：用于 Filter，不计算分数，但会缓存结果。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     4.
     <strong>
      性能对比
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        Filter
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         性能更高，适合精确匹配。
        </p>
       </li>
       <li>
        <p>
         结果缓存可以显著提升重复查询的性能。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Query
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         性能较低，因为需要计算相关性分数。
        </p>
       </li>
       <li>
        <p>
         适合全文检索和排序场景。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     5.
     <strong>
      总结
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        Filter
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         适合精确匹配和过滤场景。
        </p>
       </li>
       <li>
        <p>
         不计算分数，性能更高，结果会被缓存。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Query
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         适合全文检索和排序场景。
        </p>
       </li>
       <li>
        <p>
         计算分数，性能较低，结果不会被缓存。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        结合使用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         在实际应用中，通常将 Filter 和 Query 结合使用，以兼顾性能和相关性。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     通过合理使用 Filter 和 Query，可以显著提升 Elasticsearch 的查询性能和准确性。
    </p>
