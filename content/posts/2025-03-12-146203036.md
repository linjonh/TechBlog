    <p>
     C# 使用 AOT（Ahead - Of - Time，提前编译）生成的 "Hello, World!" 可执行文件在运行时占用的内存会受到多种因素的影响，以下是详细分析：
    </p>
    <h4>
     影响内存占用的因素
    </h4>
    <p>
    </p>
    <ol>
     <li>
      <strong>
       操作系统
      </strong>
      ：不同的操作系统（如 Windows、Linux、macOS）对进程的内存管理机制不同，会导致内存占用有所差异。
     </li>
     <li>
      <strong>
       运行环境
      </strong>
      ：包括系统中已运行的其他程序、系统的内存管理策略等。
     </li>
     <li>
      <strong>
       编译器和运行时配置
      </strong>
      ：不同版本的 .NET SDK 以及编译时的配置选项（如优化级别）会影响生成的可执行文件的内存占用。
     </li>
    </ol>
    <h4>
     大致的内存占用情况
    </h4>
    <ul>
     <li>
      <strong>
       Windows 系统
      </strong>
      <ul>
       <li>
        在 Windows 上，使用 .NET 6 及更高版本的 AOT 编译生成的简单 "Hello, World!" 程序，初始内存占用通常在几兆字节左右。具体来说，可能在 2 - 5 MB 之间。这其中包含了程序本身的代码段、数据段所占用的内存，以及操作系统为进程分配的一些基本资源（如栈空间等）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Linux 系统
      </strong>
      <ul>
       <li>
        在 Linux 系统中，内存占用一般会相对更低一些。同样的 "Hello, World!" AOT 程序，初始内存占用可能在 1 - 3 MB 左右。这是因为 Linux 的内存管理机制更加轻量级，并且对于一些系统资源的分配更为高效。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     示例代码及编译运行步骤
    </h4>
    <h5>
     示例代码（
     <code>
      Program.cs
     </code>
     ）
    </h5>
    <p>
     csharp
    </p>
    <pre><code>using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello, World!");
    }
}
</code></pre>
    <p>
     编译为 AOT 可执行文件
    </p>
    <p>
     确保你已经安装了 .NET 6 或更高版本的 SDK。在终端中执行以下命令：
    </p>
    <pre><code>dotnet publish -r &lt;runtime-identifier&gt; -p:PublishAot=true
</code></pre>
    <p>
     其中
     <code>
      &lt;runtime-identifier&gt;
     </code>
     根据你的目标系统进行替换，例如：
    </p>
    <p>
    </p>
    <ul>
     <li>
      对于 Windows x64 系统，使用
      <code>
       win-x64
      </code>
      。
     </li>
     <li>
      对于 Linux x64 系统，使用
      <code>
       linux-x64
      </code>
      。
     </li>
    </ul>
    <h5>
     查看内存占用
    </h5>
    <ul>
     <li>
      <strong>
       Windows
      </strong>
      ：可以使用任务管理器查看程序的内存占用情况。在任务管理器的 “详细信息” 选项卡中找到对应的可执行文件，查看其 “内存（专用工作集）” 列。
     </li>
     <li>
      <strong>
       Linux
      </strong>
      ：可以使用
      <code>
       top
      </code>
      或
      <code>
       ps
      </code>
      命令查看程序的内存占用。例如，使用
      <code>
       ps -p &lt;进程 ID&gt; -o %mem,rss
      </code>
      命令，其中
      <code>
       &lt;进程 ID&gt;
      </code>
      是程序的进程 ID。
     </li>
    </ul>
