    <h3>
     Docker 和 Nginx 简介
    </h3>
    <p>
     Docker 是一个开源的容器化平台，能够让你以轻量级的方式打包、分发和运行应用。想象一下，你的应用就像一个快递包裹，Docker 确保它在任何地方都能完好无损地到达。它的隔离性和便携性让开发者们爱不释手。而 Nginx 则是一个高性能的 HTTP 服务器，广泛用于反向代理和负载均衡。它的高效性让它成为了现代 web 服务的宠儿。结合 Docker 和 Nginx，开发者可以快速搭建稳定的服务环境，简化部署流程，提升开发效率。
    </p>
    <p>
     本次实验我将采用三丰云的免费服务器，配置为 1 核 CPU、1GB 内存和 5Mbps 带宽，对于此次的实验来说，能够满足需求。
    </p>
    <h3>
     在 Ubuntu 下通过 Docker 部署 Nginx 的详细步骤
    </h3>
    <h4>
     步骤 1: 安装 Docker
    </h4>
    <p>
     确保你的 Ubuntu 系统是最新的。运行以下命令：
    </p>
    <pre><code>sudo apt update
sudo apt upgrade
</code></pre>
    <p>
     然后安装 Docker：
    </p>
    <pre><code>sudo apt install docker.io
</code></pre>
    <p>
     安装完成后，启动 Docker 并设置开机自启：
    </p>
    <pre><code>sudo systemctl start docker
sudo systemctl enable docker
</code></pre>
    <h4>
     步骤 2: 拉取 Nginx 镜像
    </h4>
    <p>
     从 Docker Hub 拉取最新的 Nginx 镜像：
    </p>
    <pre><code>sudo docker pull nginx
</code></pre>
    <h4>
     步骤 3: 创建 Nginx 配置文件
    </h4>
    <p>
     在
     <code>
      /etc/nginx/conf.d/
     </code>
     目录下创建一个配置文件，命名为
     <code>
      default.conf
     </code>
     ：
    </p>
    <pre><code>sudo mkdir -p /etc/nginx/conf.d
sudo nano /etc/nginx/conf.d/default.conf
</code></pre>
    <p>
     在文件中添加以下内容：
    </p>
    <pre><code>server {
    listen 80;
    server_name localhost;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    error_page 404 /404.html;
    location = /404.html {
        internal;
    }
}
</code></pre>
    <h4>
     步骤 4: 运行 Nginx 容器
    </h4>
    <p>
     使用以下命令启动 Nginx 容器：
    </p>
    <pre><code>sudo docker run --name mynginx -v /etc/nginx/conf.d/default.conf:/etc/nginx/conf.d/default.conf -p 80:80 -d nginx
</code></pre>
    <h5>
     参数说明：
    </h5>
    <ul>
     <li>
      <code>
       --name mynginx
      </code>
      : 指定容器名称为
      <code>
       mynginx
      </code>
      。
     </li>
     <li>
      <code>
       -v /etc/nginx/conf.d/default.conf:/etc/nginx/conf.d/default.conf
      </code>
      : 将本地配置文件挂载到容器内。
     </li>
     <li>
      <code>
       -p 80:80
      </code>
      : 将主机的 80 端口映射到容器的 80 端口。
     </li>
     <li>
      <code>
       -d
      </code>
      : 在后台运行容器。
     </li>
    </ul>
    <h4>
     步骤 5: 验证 Nginx 是否成功部署
    </h4>
    <p>
     在浏览器中输入你的服务器 IP 地址，应该能看到 Nginx 的欢迎页面。如果一切正常，恭喜你，Nginx 已成功部署在 Docker 容器中！
    </p>
    <p>
     这就是在 Ubuntu 下通过 Docker 部署 Nginx 服务器的全过程。保持你的环境干净整洁，安全高效，继续探索吧！
    </p>
