     win10 win+shift+s 无法立即连续截图 第二次截图需要等很久
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     意外的win10做了一次更新，然后系统的截图工具就出现了这个问题。
    </p>
    <p>
     windows的截图功能，是由 Microsoft Text Input Application 来维护的，是 Windows 系统内置的输入法管理程序，负责协调输入法与系统之间的交互‌
    </p>
    <p>
    </p>
    <p>
     ScreenClippingHost.exe 是 Windows 原生截图工具（如 Snipping Tool）的后台进程，用于支持屏幕剪裁、截图编辑及剪贴板操作‌，触发截图快捷键（Win+Shift+S）时自动激活，具体过程是由Microsoft Text Input Application 启动一个临时的新的进程。截图完后就进程就自动结束了，正常来说这个进程结束会特别快。
     <br/>
     <img alt="" height="103" src="https://i-blog.csdnimg.cn/direct/4bccbcba467c4405bafdc855af25002f.png" width="1172">
      <br/>
      <img alt="" height="84" src="https://i-blog.csdnimg.cn/direct/9e7c7bec807b44b9b90195c8272e4f4e.png" width="1035"/>
     </img>
    </p>
    <p>
     但是不知道为何，更新后自动系统自动安装了Snip &amp; Sketch后导致截图无法连续。
     <br/>
     解决办法，win+I -&gt; 应用 -&gt;应用和功能
     <br/>
     在应用和功能的搜索框中找 关键字  “截图”
     <br/>
     出现 截图工具 和 截图与草图 。全部卸载，重启电脑，截图功能就能正常了。
    </p>
    <p>
    </p>
    <p>
     Windows 10/11 的截图功能深度集成于系统底层，即使卸载「截图工具」（
     <code>
      SnippingTool.exe
     </code>
     ）和「截图与草图」（
     <code>
      ScreenClippingHost.exe
     </code>
     ），其核心组件（如
     <code>
      ScreenClippingHost.exe
     </code>
     ）仍作为系统服务保留，支持快捷键 ‌
     <strong>
      Win+Shift+S
     </strong>
     ‌ 的底层截屏功能‌。
    </p>
    <p>
     ‌
     <strong>
      卸载后：底层快捷键直接调用核心功能
     </strong>
     ‌。
     <code>
      Win+Shift+S
     </code>
     的截图功能由系统资源管理器（
     <code>
      explorer.exe
     </code>
     ）直接驱动，卸载应用仅移除前端界面，底层截屏逻辑仍保留。卸载后，系统不再加载冗余的 UI 组件（如标注工具栏、通知弹窗），直接通过剪贴板缓存截图，响应速度更快‌
    </p>
