    <h4>
     <a id="_uniapp__4">
     </a>
     一、 uni-app 简介
    </h4>
    <p>
     uni-app 是一个使用 Vue.js 开发所有前端应用的框架。开发者编写一套代码，可发布到 iOS、Android、H5、以及各种小程序（微信/支付宝/百度/头条/QQ/钉钉/淘宝）、快应用等多个平台。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1562103e405edb4b27fab7311d06f9d8.png"/>
    </p>
    <p>
     详细的 uni-app 官方文档，请翻阅
     <a href="https://uniapp.dcloud.net.cn/" rel="nofollow">
      uniapp官网
     </a>
    </p>
    <h4>
     <a id="HBuilderX__12">
     </a>
     二、开发工具（HBuilderX ）
    </h4>
    <p>
     uni-app 官方推荐使用 HBuilderX 来开发 uni-app 类型的项目。主要好处：
    </p>
    <p>
     模板丰富
    </p>
    <p>
     完善的智能提示
    </p>
    <p>
     一键运行
    </p>
    <p>
     当然，你依然可以根据自己的喜好，选择使用 VS Code、Sublime、记事本… 等自己喜欢的编辑器！
    </p>
    <h4>
     <a id="__HBuilderX_23">
     </a>
     三、 下载 HBuilderX
    </h4>
    <blockquote>
     <p>
      访问 HBuilderX 的官网首页 https://www.dcloud.io/hbuilderx.html
     </p>
    </blockquote>
    <p>
     点击首页的 DOWNLOAD 按钮
    </p>
    <p>
     选择下载 正式版 -&gt; App 开发版
    </p>
    <h4>
     <a id="__HBuilderX_30">
     </a>
     四、 安装 HBuilderX
    </h4>
    <ol>
     <li>
      <p>
       将下载的 zip包 进行解压缩
      </p>
     </li>
     <li>
      <p>
       将解压之后的文件夹，存放到纯英文的目录中（且不能包含括号等特殊字符）
      </p>
     </li>
     <li>
      <p>
       双击 HBuilderX.exe 即可启动 HBuilderX
      </p>
     </li>
    </ol>

    <pre><code class="prism language-md">一个 uni-app 项目，默认包含如下目录及文件：

┌─components            uni-app组件目录

├─pages                 业务页面文件存放的目录
│  ├─index
│  │  └─index.vue       index页面
│  └─list
│     └─list.vue        list页面
├─static                存放应用引用静态资源（如图片、视频等）的目录，注意：静态资源只能存放于此
├─main.js               Vue初始化入口文件
├─App.vue               应用配置，用来配置小程序的全局样式、生命周期函数等
├─manifest.json         配置应用名称、appid、logo、版本等打包信息
└─pages.json            配置页面路径、页面窗口样式、tabBar、navigationBar 等页面类信息

    <h4>
     <a id="_Git__139">
     </a>
     八、使用 Git 管理项目
    </h4>
    <ul>
     <li>
      <p>
       实战项目开发必须用到Git版本控制工具
      </p>
     </li>
     <li>
      <p>
       学习Git安装配置参考：
       <br/>
       参考另一篇文章进行安装 git 、配置公钥、以及仓库的创建以及托管开发流程：
       <a href="https://blog.csdn.net/Forever_Hopeful/article/details/135977257?spm=1001.2014.3001.5501">
        Git项目开发从0到1实战教程
       </a>
      </p>
      <p>
       <strong>
        本地管理
       </strong>
       <br/>
       在项目根目录中新建 .gitignore 忽略文件，并配置如下：
       <br/>
       <strong>
        忽略 node_modules 目录
       </strong>
       <br/>
       <code>
        /node_modules
       </code>
       <br/>
       <code>
        /unpackage/dist
       </code>
       <br/>
       注意：由于我们忽略了 unpackage 目录中仅有的 dist 目录，因此默认情况下， unpackage 目录不会被 Git 追踪
      </p>
     </li>
    </ul>
    <p>
     此时，为了让 Git 能够正常追踪 unpackage 目录，按照惯例，我们可以在 unpackage 目录下创建一个叫做 .gitkeep 的文件进行占位
    </p>
    <ul>
     <li>
      <strong>
       初始化仓库
      </strong>
      <br/>
      打开终端，切换到项目根目录中，运行如下的命令，初始化本地 Git 仓库：
      <br/>
      <code>
       git init
      </code>
      <br/>
      将所有文件都加入到暂存区：
      <br/>
      <code>
       git add .
      </code>
      <br/>
      本地提交更新：
      <br/>
      <code>
       git commit -m "init project"
      </code>
     </li>
     <li>
      <strong>
       把项目托管到码云
      </strong>
      <br/>
      注册并激活码云账号（ 注册页面地址：https://gitee.com/signup ）
     </li>
    </ul>
    <p>
     生成并配置 SSH 公钥
     <br/>
     <a href="https://blog.csdn.net/Forever_Hopeful/article/details/135988610?spm=1001.2014.3001.5501">
      生成配置ssh公钥的方法请参考该文章
     </a>
    </p>
    <p>
     创建空白的码云仓库
    </p>
    <p>
     把本地项目上传到码云对应的空白仓库中
    </p>
