    <h4>
     <a id="create_react_agentmodel_tools__graph_builderadd_conditional_edges__0">
     </a>
     <strong>
      <code>
       create_react_agent(model, tools)
      </code>
      和
      <code>
       graph_builder.add_conditional_edges
      </code>
      的联系和区别
     </strong>
    </h4>
    <p>
     这两者都是在
     <strong>
      LangGraph
     </strong>
     框架下用于构建智能代理（Agent）工作流的，但它们的作用完全不同：
    </p>
    <table>
     <thead>
      <tr>
       <th>
       </th>
       <th>
        <code>
         create_react_agent(model, tools)
        </code>
       </th>
       <th>
        <code>
         graph_builder.add_conditional_edges
        </code>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         作用
        </strong>
       </td>
       <td>
        创建一个
        <strong>
         可以调用工具
        </strong>
        的智能代理（Agent）
       </td>
       <td>
        在 LangGraph 中
        <strong>
         定义工作流中的条件分支
        </strong>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         本质
        </strong>
       </td>
       <td>
        让 AI 通过
        <strong>
         ReAct 机制
        </strong>
        决定如何调用
        <code>
         tools
        </code>
        处理任务
       </td>
       <td>
        让任务执行时，根据不同的
        <strong>
         条件
        </strong>
        决定下一步执行哪个节点
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        需要让 AI 代理根据用户输入
        <strong>
         选择并执行合适的工具
        </strong>
       </td>
       <td>
        需要在一个
        <strong>
         有条件分支的工作流
        </strong>
        里决定下一个执行的步骤
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         是否依赖 AI 解析
        </strong>
       </td>
       <td>
        是，AI 决定何时调用工具
       </td>
       <td>
        否，条件逻辑通常由代码或数据判断
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <a id="_13">
     </a>
     <strong>
      联系
     </strong>
    </h3>
    <p>
     通常，你会先用
     <code>
      create_react_agent(model, tools)
     </code>
     <strong>
      创建一个可以调用工具的 AI 代理
     </strong>
     ，然后在
     <strong>
      LangGraph 的工作流
     </strong>
     里用
     <code>
      graph_builder.add_conditional_edges
     </code>
     <strong>
      控制代理的执行流程
     </strong>
     。
    </p>
    <p>
     <strong>
      换句话说：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       create_react_agent(model, tools)
      </code>
      让 AI 代理能使用工具来处理任务。
     </li>
     <li>
      <code>
       graph_builder.add_conditional_edges
      </code>
      则决定 AI 代理完成任务后，
      <strong>
       下一个应该执行的步骤
      </strong>
      。
     </li>
    </ul>
    <p>
     它们经常配合使用，比如：
    </p>
    <ol>
     <li>
      <strong>
       先用
       <code>
        create_react_agent(model, tools)
       </code>
      </strong>
      让 AI 代理能查询数据库、调用 API 等。
     </li>
     <li>
      <strong>
       再用
       <code>
        add_conditional_edges
       </code>
      </strong>
      让 AI 代理在不同情况下执行不同的逻辑。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_26">
     </a>
     <strong>
      举例说明
     </strong>
    </h3>
    <h4>
     <a id="_27">
     </a>
     <strong>
      场景：智能宠物助手
     </strong>
    </h4>
    <p>
     我们想创建一个
     <strong>
      智能宠物助手
     </strong>
     ，它能够：
    </p>
    <ul>
     <li>
      <strong>
       查询用户的宠物喜好
      </strong>
      （如果用户有存过宠物，就告诉他喜欢什么）
     </li>
     <li>
      <strong>
       推荐宠物
      </strong>
      （如果用户没有存过宠物，就推荐一些宠物给他）
     </li>
    </ul>
    <h4>
     <a id="1__create_react_agentmodel_tools_32">
     </a>
     <strong>
      1. 使用
      <code>
       create_react_agent(model, tools)
      </code>
     </strong>
    </h4>
    <pre><code class="prism language-python">graph <span class="token operator">=</span> create_react_agent<span class="token punctuation">(</span>model<span class="token punctuation">,</span> tools<span class="token punctuation">)</span>
</code></pre>
    <p>
     这行代码创建了一个
     <strong>
      AI 代理
     </strong>
     ，它可以调用
     <code>
      tools
     </code>
     （比如
     <code>
      list_favorite_pets
     </code>
     查询宠物、
     <code>
      update_favorite_pets
     </code>
     存储宠物）。
    </p>
    <ul>
     <li>
      <strong>
       如果用户问“我喜欢什么宠物？”
      </strong>
      <ul>
       <li>
        代理会调用
        <code>
         list_favorite_pets
        </code>
        工具来查询数据
       </li>
      </ul>
     </li>
     <li>
      <strong>
       如果用户说“我喜欢猫和狗。”
      </strong>
      <ul>
       <li>
        代理会调用
        <code>
         update_favorite_pets
        </code>
        工具来存储信息
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      ➡️ 这一步只是让 AI 具备调用工具的能力
     </strong>
     ，但它并不决定
     <strong>
      下一步要执行哪个任务
     </strong>
     。
    </p>
    <hr/>
    <h4>
     <a id="2__graph_builderadd_conditional_edges_47">
     </a>
     <strong>
      2. 使用
      <code>
       graph_builder.add_conditional_edges
      </code>
     </strong>
    </h4>
    <pre><code class="prism language-python">graph_builder<span class="token punctuation">.</span>add_conditional_edges<span class="token punctuation">(</span>
    <span class="token string">"check_pets"</span><span class="token punctuation">,</span>  <span class="token comment"># 当前步骤</span>
    <span class="token keyword">lambda</span> data<span class="token punctuation">:</span> <span class="token string">"recommend_pets"</span> <span class="token keyword">if</span> <span class="token keyword">not</span> data<span class="token punctuation">[</span><span class="token string">"pets"</span><span class="token punctuation">]</span> <span class="token keyword">else</span> <span class="token string">"show_pets"</span><span class="token punctuation">,</span>  <span class="token comment"># 条件判断</span>
<span class="token punctuation">)</span>
</code></pre>
    <p>
     这行代码的作用是：
    </p>
    <ul>
     <li>
      <strong>
       如果
       <code>
        data["pets"]
       </code>
       为空
      </strong>
      （用户没有存过宠物），执行
      <code>
       "recommend_pets"
      </code>
      （推荐宠物）。
     </li>
     <li>
      <strong>
       否则，执行
       <code>
        "show_pets"
       </code>
       （告诉用户他喜欢的宠物）。
      </strong>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_60">
     </a>
     <strong>
      完整的流程
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       用户输入：“我喜欢什么宠物？”
      </strong>
     </li>
     <li>
      <strong>
       AI 代理调用
       <code>
        list_favorite_pets
       </code>
       查询宠物
      </strong>
     </li>
     <li>
      <strong>
       工作流分支
      </strong>
      <ul>
       <li>
        <strong>
         如果有宠物信息
        </strong>
        ，执行
        <code>
         show_pets
        </code>
        ，回复“你喜欢猫和狗”。
       </li>
       <li>
        <strong>
         如果没有宠物信息
        </strong>
        ，执行
        <code>
         recommend_pets
        </code>
        ，回复“你可以试试养猫或狗”。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_69">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         功能
        </strong>
       </th>
       <th>
        <code>
         create_react_agent(model, tools)
        </code>
       </th>
       <th>
        <code>
         graph_builder.add_conditional_edges
        </code>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         作用
        </strong>
       </td>
       <td>
        让 AI 代理能够调用工具处理任务
       </td>
       <td>
        让 AI 代理在任务完成后决定执行哪个步骤
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         是否涉及 AI 解析
        </strong>
       </td>
       <td>
        是，AI 自己决定是否调用工具
       </td>
       <td>
        否，执行路径由代码逻辑决定
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         使用场景
        </strong>
       </td>
       <td>
        让 AI 代理能查询数据库、调用 API、处理用户请求
       </td>
       <td>
        在一个工作流中，定义执行路径的分支逻辑
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         举例
        </strong>
       </td>
       <td>
        AI 代理查询用户喜欢的宠物
       </td>
       <td>
        AI 代理根据查询结果决定是显示宠物还是推荐宠物
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     <a id="_77">
     </a>
     <strong>
      一句话总结
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       create_react_agent(model, tools)
      </code>
      <strong>
       让 AI 代理能调用工具
      </strong>
      ，但不管执行顺序。
     </li>
     <li>
      <code>
       graph_builder.add_conditional_edges
      </code>
      <strong>
       决定执行顺序
      </strong>
      ，但不涉及 AI 解析。
     </li>
    </ul>
    <p>
     它们经常搭配使用，
     <code>
      create_react_agent
     </code>
     负责
     <strong>
      执行任务
     </strong>
     ，
     <code>
      add_conditional_edges
     </code>
     负责
     <strong>
      控制任务流
     </strong>
     。
    </p>
