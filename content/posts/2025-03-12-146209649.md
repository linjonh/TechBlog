    <h3>
     <a id="_27">
     </a>
     重点讲解自动应答：
    </h3>
    <h4>
     <a id="_28">
     </a>
     一、开启自动应答功能
    </h4>
    <h5>
     <a id="1_29">
     </a>
     1.进入规则设置界面
    </h5>
    <p>
     打开软件后，点击顶部菜单栏的「设置」或「工具」选项，选择「自动应答规则」或「脚本规则」功能入口，以服务端为例：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b57a53999d474931ba006e83f049e664.png"/>
    </p>
    <h4>
     <a id="_36">
     </a>
     二、配置触发条件与应答内容
    </h4>
    <h5>
     <a id="1_37">
     </a>
     1.设置触发条件
    </h5>
    <p>
     –匹配模式：支持全匹配、部分匹配或正则表达式（如设置接收数据包含”HELLO”时触发）。
     <br/>
     –数据格式：选择ASCII或HEX格式（需与接收数据格式一致）。
     <br/>
     –触发范围：可指定特定客户端IP或端口（默认对所有连接生效）。
    </p>
    <p>
     触发条件：ASCII数据包含"PING"
    </p>
    <p>
     匹配模式：部分匹配
    </p>
    <h5>
     <a id="2_46">
     </a>
     2.定义应答内容
    </h5>
    <p>
     –在「应答数据」框中输入回复内容，支持ASCII、HEX或转义字符（如\r\n）。
     <br/>
     –高级选项可添加动态脚本（如时间戳、计数器）。
    </p>
    <h5>
     <a id="3_49">
     </a>
     3.示例图如下
    </h5>
    <p>
     在规则管理界面点击「新增规则」，输入规则名称（例如“我是匹配规则”）并选择生效协议（TCP/UDP）。
     <img alt="新建规则" src="https://i-blog.csdnimg.cn/direct/9b954482299549f1b484eec32b0a306f.png"/>

     图一：新建规则

     <br/>
     图二：规则信息配置
    </p>
    <h4>
     <a id="_55">
     </a>
     三、测试与调试
    </h4>
    <h5>
     <a id="1_56">
     </a>
     1.启用规则
    </h5>
    <p>
     勾选规则前的复选框，确保规则状态为「启用」。
     <br/>
     <img alt="配置完毕后勾选规则" src="https://i-blog.csdnimg.cn/direct/6def1884cad349d4822bc772101aadcc.png"/>
    </p>
    <h5>
     <a id="2_59">
     </a>
     2.模拟测试
    </h5>
    <p>
     –使用另一台设备或本机客户端发送触发指令（如发送”PING”）。
     <br/>
     –观察接收窗口是否自动返回预设应答内容（如”PONG”）。
     <br/>
     <img alt="发送流程的响应截图" src="https://i-blog.csdnimg.cn/direct/8307a29836054955a8581a925fc18019.png"/>
     <br/>
     图：发送流程的响应信息的截图
    </p>
    <h5>
     <a id="3_65">
     </a>
     3.日志检查
    </h5>
    <p>
     若规则未生效，可在「日志」标签中查看匹配记录和错误提示，一般情况不会发生错误。
    </p>

     <a id="_68">
     </a>
     四、高级功能（可选）
    </h4>
    <h5>
     <a id="1_69">
     </a>
     1.批量规则管理
    </h5>
    <p>
     支持同时启用多条规则，优先级可拖动调整（从上至下依次匹配）。
    </p>

    <h5>
     <a id="3_73">
     </a>
     3.数据校验
    </h5>
    <p>
     可配置CRC校验、异或校验等，确保指令合法性。
    </p>
    <h4>
     <a id="_76">
     </a>
     注意事项
    </h4>
    <p>
     1.若使用HEX格式，需确保收发双方数据格式一致（如”AA BB CC”对应十六进制字节流）。
     <br/>
     2.复杂规则建议先保存配置文件（通过「导出规则」备份），避免意外丢失。
     <br/>
     3.若需调试工业设备协议，可结合「间隔发送」和「循环发送」功能模拟真实场景。
