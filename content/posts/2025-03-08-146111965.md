    <p>
     在 Go 语言中，
     <code>
      container/list
     </code>
     包提供了一个双向链表的实现。链表是一种常见的数据结构，适用于频繁插入和删除操作的场景。
     <code>
      container/list
     </code>
     包中的链表是双向的，意味着每个元素都包含指向前一个和后一个元素的指针。
    </p>
    <h3>
     <a id="146_LRU__3">
     </a>
     力扣：146. LRU 缓存
    </h3>
    <p>
     <strong>
      力扣算法链接：
      <a href="https://leetcode.cn/problems/lru-cache/?envType=study-plan-v2&amp;envId=top-100-liked" rel="nofollow">
       https://leetcode.cn/problems/lru-cache/?envType=study-plan-v2&amp;envId=top-100-liked
      </a>
     </strong>
    </p>
    <p>
     请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。
     <br/>
     实现 LRUCache 类：
     <br/>
     LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存。
     <br/>
     int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
     <br/>
     void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。
    </p>
    <p>
     函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。
    </p>
    <p>
     输入
     <br/>
     [“LRUCache”, “put”, “put”, “get”, “put”, “get”, “put”, “get”, “get”, “get”]
     <br/>
     [[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]
    </p>
    <p>
     输出
     <br/>
     [null, null, null, 1, null, -1, null, -1, 3, 4]
    </p>
    <p>
     解释
     <br/>
     LRUCache lRUCache = new LRUCache(2);
     <br/>
     lRUCache.put(1, 1); // 缓存是 {1=1}
     <br/>
     lRUCache.put(2, 2); // 缓存是 {1=1, 2=2}
     <br/>
     lRUCache.get(1); // 返回 1
     <br/>
     lRUCache.put(3, 3); // 该操作会使得关键字 2 作废，缓存是 {1=1, 3=3}
     <br/>
     lRUCache.get(2); // 返回 -1 (未找到)
     <br/>
     lRUCache.put(4, 4); // 该操作会使得关键字 1 作废，缓存是 {4=4, 3=3}
     <br/>
     lRUCache.get(1); // 返回 -1 (未找到)
     <br/>
     lRUCache.get(3); // 返回 3
     <br/>
     lRUCache.get(4); // 返回 4
    </p>

    <h4>
     <a id="_201">
     </a>
     使用场景
    </h4>
    <ul>
     <li>
      <strong>
       频繁插入和删除
      </strong>
      : 链表在插入和删除操作上比数组更高效，尤其是在中间位置。
     </li>
     <li>
      <strong>
       实现队列和栈
      </strong>
      : 链表可以用来实现队列（FIFO）和栈（LIFO）等数据结构。
     </li>
     <li>
      <strong>
       动态数据存储
      </strong>
      : 当数据量不确定或需要动态调整时，链表是一个很好的选择。
     </li>
    </ul>
    <h4>
     <a id="_207">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      <strong>
       内存开销
      </strong>
      : 链表的每个元素都需要额外的内存来存储前后指针，因此内存开销比数组大。
     </li>
     <li>
      <strong>
       随机访问性能差
      </strong>
      : 链表不支持随机访问，访问某个元素需要从头或尾开始遍历。
     </li>
    </ul>
