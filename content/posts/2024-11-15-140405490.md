     <h3>
      <a id="tar_23">
      </a>
      🩷tar指令
     </h3>
     <p>
      <code>
       tar [-cxtzjvf] 文件与目录 .... 参数
      </code>
      ：
     </p>
     <ol>
      <li>
       -c ：建立一个压缩文件的参数指令(create 的意思)；
      </li>
      <li>
       -x ：解开一个压缩文件的参数指令！
      </li>
      <li>
       -t ：查看 tarfile 里面的文件！
      </li>
      <li>
       -z ：是否同时具有 gzip 的属性？亦即是否需要用 gzip 压缩？
      </li>
      <li>
       -j ：是否同时具有 bzip2 的属性？亦即是否需要用 bzip2 压缩？
      </li>
      <li>
       -v ：压缩的过程中显示文件！这个常用，但不建议用在背景执行过程！
      </li>
      <li>
       -f ：使用档名，请留意，在 f 之后要立即接档名喔！不要再加参数！
      </li>
      <li>
       -C ： 解压到指定目录
      </li>
     </ol>
     <p>
      这里只需要重要掌握czxv即可。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5a5473648003412489ce9b4c4474d6b7.png">

       c表示建立一个压缩文件，z表示看是否具有这个gzip的属性，f表示在之后添加解压后的文件的名称。

     <h2>
      <a id="_Shell__72">
      </a>
      ❤️什么是 Shell 命令
     </h2>
     <p>
      Shell 命令是用户与操作系统进行交互的方式。Shell 是一个命令行解释器，它接受用户输入的命令，并将这些命令传递给操作系统进行执行。常见的 Shell 包括 Bash、Zsh、Sh、Csh 等。
     </p>
     <p>
      在 Shell 中，用户可以输入各种命令来执行不同的任务，如文件操作、进程管理、系统监控等。例如：
     </p>
     <ul>
      <li>
       <code>
        ls
       </code>
       ：列出目录内容
      </li>
      <li>
       <code>
        cd
       </code>
       ：改变当前目录
      </li>
      <li>
       <code>
        cp
       </code>
       ：复制文件或目录
      </li>
      <li>
       <code>
        mv
       </code>
       ：移动或重命名文件或目录
      </li>
      <li>
       <code>
        rm
       </code>
       ：删除文件或目录
      </li>
      <li>
       <code>
        chmod
       </code>
       ：改变文件或目录的权限
      </li>
      <li>
       <code>
        ps
       </code>
       ：显示当前运行的进程
      </li>
     </ul>
     <p>
      Shell 还支持编写脚本，通过编写一系列命令来自动化执行任务，从而提高工作效率。
     </p>
     <p>
      简单来说，shell指令就是命令行解释器（command Interpreter）。
     </p>
     <p>
      <strong>
       主要包含：
      </strong>
     </p>
     <ul>
      <li>
       将使用者的命令翻译给核心（kernel）处理。
      </li>
      <li>
       同时，将核心的处理结果翻译给使用者。
      </li>
     </ul>
     <blockquote>
      <p>
       对比windows GUI，我们操作windows 不是直接操作windows内核，而是通过图形接口，点击，从而完成我们的
       <br/>
       操作（比如进入D盘的操作，我们通常是双击D盘盘符.或者运行起来一个应用程序）。
      </p>
     </blockquote>
     <blockquote>
      <p>
       shell 对于Linux，有相同的作用，主要是对我们的指令进行解析，解析指令给Linux内核。反馈结果在通过内核运
       <br/>
       行出结果，通过shell解析给用户。
      </p>
     </blockquote>
     <h2>
      <a id="Linux_102">
      </a>
      ❤️Linux权限管理的概念
     </h2>
     <p>
      Linux中分为两种用户：
     </p>
     <ol>
      <li>
       超级用户：级用户被称为"root"，也被称为系统管理员。root拥有系统中的最高权限，可以执行所有操作，包括修改系统设置、安装和删除软件、访问系统文件等。root用户对系统的操作有无限制的权限，因此在使用root用户时需要非常谨慎，以防止不慎造成系统损坏。（可以再linux系统下做任何事情，不受限制）
      </li>
      <li>
       普通用户：在Linux系统中创建的其他用户账号。普通用户在系统中有受限的权限，不能进行对系统关键部分的操作。普通用户可以执行常规的任务，如创建、编辑和删除自己的文件、运行应用程序、访问网络和打印机等。普通用户不能修改系统级别的设置，也不能访问其他用户的文件和目录，保证系统的安全性和稳定性。（在linux下做有限的事情）
      </li>
     </ol>
     <p>
      <strong>
       命令
      </strong>
      ：
      <code>
       su [用户名]
      </code>
      <br/>
      <strong>
       功能
      </strong>
      ：切换用户
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d3b4f034be324c169966dc7a75d20779.png"/>
     </p>
     <p>
      可以看见我们现在就将root用户切换为了newuser用户。

      su -可以切换为超级用户。

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/55fc063be197432585b8aba4a81c9fb0.png"/>
     </p>

     <h3>
      <a id="_124">
      </a>
      🩷文件类型的访问权限
     </h3>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e91217c5212449f48f44843ee51cdd4e.png"/>
     </p>
     <p>
      上面是大部分的信息组成，下面重点说一下权限信息：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8ca9fdcc329644f59cd4a380f164bbfe.png"/>
      <br/>
      三个为一组，从左到右分别是所有者信息，所属组信息，其他的信息。
     </p>
     <p>
      <strong>
       文件信息：
      </strong>
     </p>
     <ol>
      <li>
       d：文件夹
      </li>
      <li>
       -：普通文件
      </li>
      <li>
       l：软链接（类似Windows的快捷方式）
      </li>
      <li>
       b：块设备文件（例如硬盘、光驱等）
      </li>
      <li>
       p：管道文件
      </li>
      <li>
       c：字符设备文件（例如屏幕等串口设备）
      </li>
      <li>
       s：套接口文件
      </li>
     </ol>
     <p>
      <strong>
       基本权限：
      </strong>
     </p>
     <ol>
      <li>
       i.读（r/4）：Read对文件而言，具有读取文件内容的权限；对目录来说，具有浏览该目录信息的权限
      </li>
      <li>
       ii.写（w/2）：Write对文件而言，具有修改文件内容的权限；对目录来说具有删除移动目录内文件的权限
      </li>
      <li>
       iii.执行（x/1）：execute对文件而言，具有执行文件的权限；对目录来说，具有进入目录的权限
      </li>
      <li>
       iv.“—”表示不具有该项权限
      </li>
     </ol>
     <p>
      我们来看看下面的：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f8ce8ecce63d469a8585f43225287cb7.png"/>
     </p>
     <p>
      对于权限信息来说，每个信息位都是按照顺序，读—写—执行，这样的顺序来的，每个信息位要么是可以要么是不可以，所以这里我们可以用两态来表示，也就是0/1。
     </p>

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/29f46539cd194c57b5a82f669a7be6ac.png"/>

     <h3>
      <a id="chmod_157">
      </a>
      🩷chmod指令
     </h3>
     <p>
      <strong>
       功能
      </strong>
      ：设置文件的访问权限
      <br/>
      <strong>
       格式
      </strong>
      ：chmod [参数] 权限 文件名
      <br/>
      <strong>
       常用选项
      </strong>
      ：
     </p>
     <blockquote>
      <p>
       R -&gt; 递归修改目录文件的权限
       <br/>
       说明：只有文件的拥有者和root才可以改变文件的权限
      </p>
     </blockquote>
     <p>
      <strong>
       用户表示符+/-=权限字符
      </strong>
     </p>
     <ol>
      <li>
       +:向权限范围增加权限代号所表示的权限
      </li>
      <li>
       -:向权限范围取消权限代号所表示的权限
      </li>
      <li>
       =:向权限范围赋予权限代号所表示的权限
      </li>
     </ol>
     <p>
      <strong>
       用户符号：
      </strong>
     </p>
     <ol>
      <li>
       u：拥有者
      </li>
      <li>
       g：拥有者同组用
      </li>
      <li>
       o：其它用户
      </li>
      <li>
       a：所有用户
      </li>
     </ol>
     <p>
      <strong>
       示例：
      </strong>
      <br/>
      chmod u+rwx 将所有者改为可读可写可执行
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6cf7cc009a7a4f40b6b036d28ac0c206.png"/>
      <br/>
      同样对于所属者的缩写g和其他的缩写o也可以执行相同的操作。
     </p>

     <p>
      <mark>
       注意：这里如果所有者和所属组是同一个的话，我们的权限如果所有者是不可读和不可写的话，是写入不了东西的，因为判断身份是按照
       <code>
        所有者、所属组、其他
       </code>
       进行判断的，所以这里先判断所有者成立之后，就不会继续判断了只会判断一次。
