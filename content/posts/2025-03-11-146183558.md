    <h2 style="text-align:justify">
     1、下载Android Studio 和Java JDK 并配置
    </h2>
    <p style="text-align:justify">
     （安装教程看百度），我这里使用的是Android Studio 2024.01版本，java17 版本
    </p>
    <h2 style="text-align:justify">
     2、下载Android SDK并配置
    </h2>
    <blockquote>
     <p style="text-align:justify">
      参考：
     </p>
     <p style="text-align:justify">
      <a href="https://blog.csdn.net/2401_83004375/article/details/139300339" title="Android SDK下载安装(_指定版本)-CSDN博客">
       Android SDK下载安装(_指定版本)-CSDN博客
      </a>
     </p>
    </blockquote>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     3、下载gradle
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     进入https://services.gradle.org/distributions/下载，我这里下载的是gradle-8.7-bin.zip
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     4、配置本地gradle地址（镜像源不好拉，建议使用本地）
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     打开项目的根目录，找到 gradle/wrapper/gradle-wrapper.properties 文件。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     找到以下行：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     distributionUrl=https\://services.gradle.org/distributions/gradle-8.7-bin.zip
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     将其替换为本地地址：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     distributionUrl=file:///E:/ZY/gradle/gradle-8.7-bin.zip
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     保存文件，重新同步 Gradle 依赖。（最上面的那个小象按钮）
    </p>
    <h2 style="text-align:justify">
     5、项目gradle设置
    </h2>
    <p style="text-align:justify">
     <img alt="" height="894" src="https://i-blog.csdnimg.cn/direct/f642434a7c0548738587aade5582d268.png" width="1295"/>
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     6、同步依赖：
    </h2>
    <blockquote>
     <ol>
      <li>
       <strong>
        同步 Gradle 依赖
       </strong>
       ：
       <ul>
        <li>
         在 Android Studio 工具栏中，点击
         <code>
          Sync Project with Gradle Files
         </code>
         按钮（通常是一个蓝色的圆形按钮）。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        等待同步完成
       </strong>
       ：
       <ul>
        <li>
         Android Studio 会根据新的配置下载或使用本地的 Gradle 分发包，完成项目同步
        </li>
       </ul>
      </li>
      <li>
       <p>
        注意事项
       </p>
       <p>
        <strong>
         路径格式
        </strong>
        ：
       </p>
       <ul>
        <li>
         Windows 系统中，路径应使用反斜杠
         <code>
          \\
         </code>
         或者双反斜杠
         <code>
          \\\\
         </code>
         ，确保 URL 格式正确。
        </li>
        <li>
         macOS 和 Linux 系统中，路径使用正斜杠
         <code>
          /
         </code>
         。
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
    <p>
    </p>
    <h2>
     7、启动项目
    </h2>
    <blockquote>
     <ol>
      <li>
       <strong>
        打开 Android Studio
       </strong>
       ：
       <ul>
        <li>
         启动 Android Studio 并确保项目已成功导入。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        选择设备
       </strong>
       ：
       <ul>
        <li>
         连接 Android 设备或启动 Android 模拟器。
        </li>
        <li>
         在 Android Studio 的工具栏中，点击设备选择器，选择要使用的设备。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        构建项目
       </strong>
       ：
       <ul>
        <li>
         点击工具栏中的
         <code>
          Build
         </code>
         按钮（或使用快捷键
         <code>
          Ctrl + F9
         </code>
         ）。
        </li>
        <li>
         等待构建完成，确保没有错误。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        运行应用
       </strong>
       ：
       <ul>
        <li>
         点击工具栏中的
         <code>
          Run
         </code>
         按钮（绿色三角形），或使用快捷键
         <code>
          Shift + F10
         </code>
         。
        </li>
        <li>
         Android Studio 将会编译、构建并安装应用到选定的设备上，随后启动应用
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
    <h2>
     8、调试项目
    </h2>
    <blockquote>
     <ol>
      <li>
       <strong>
        设置断点
       </strong>
       ：
       <ul>
        <li>
         在代码编辑器中，点击行号左侧的空白区域，设置断点。
        </li>
        <li>
         断点会在程序执行到该位置时暂停，方便查看变量和调试逻辑。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        启动调试模式
       </strong>
       ：
       <ul>
        <li>
         点击工具栏中的
         <code>
          Debug
         </code>
         按钮（带有虫子图标的按钮），或使用快捷键
         <code>
          Shift + F9
         </code>
         。
        </li>
        <li>
         应用会在断点处暂停，进入调试模式。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        查看变量值
       </strong>
       ：
       <ul>
        <li>
         在调试模式下，点击右侧的
         <code>
          Variables
         </code>
         标签，查看当前作用域内的变量及其值。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        查看日志信息
       </strong>
       ：
       <ul>
        <li>
         打开
         <code>
          Logcat
         </code>
         窗口（快捷键
         <code>
          Alt + 6
         </code>
         ）。
        </li>
        <li>
         查看应用输出的日志信息，帮助分析问题。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        单步调试
       </strong>
       ：
       <ul>
        <li>
         使用调试工具栏中的按钮进行单步调试：
         <ul>
          <li>
           <code>
            Step Over
           </code>
           （F8）：执行当前行，跳到下一行。
          </li>
          <li>
           <code>
            Step Into
           </code>
           （F7）：进入当前行的方法内部。
          </li>
          <li>
           <code>
            Step Out
           </code>
           （Shift + F8）：跳出当前方法，返回调用处。
          </li>
          <li>
           <code>
            Resume
           </code>
           （F9）：继续执行程序，直到下一个断点。
          </li>
         </ul>
        </li>
       </ul>
      </li>
     </ol>
     <h4>
     </h4>
    </blockquote>
    <h2>
     常用调试技巧
    </h2>
    <ol>
     <li>
      <strong>
       打印日志
      </strong>
      ：
      <ul>
       <li>
        在代码中添加
        <code>
         Log.d()
        </code>
        、
        <code>
         Log.e()
        </code>
        等日志语句，输出关键信息。
       </li>
       <li>
        示例：
        <pre><code class="language-java">Log.d("TAG", "Variable value: " + variableName);</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用 Logcat 过滤日志
      </strong>
      ：
      <ul>
       <li>
        在
        <code>
         Logcat
        </code>
        窗口中，输入过滤关键词（如包名或标签）以快速定位相关日志。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       检查布局问题
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         Layout Inspector
        </code>
        工具（快捷键
        <code>
         Shift + Ctrl + L
        </code>
        ）查看应用的实时布局结构。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       分析内存和性能
      </strong>
      ：
      <ul>
       <li>
        使用 Android Profiler（快捷键
        <code>
         Shift + Alt + P
        </code>
        ）监控应用的内存、CPU 和网络使用情况。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h2>
     查看构建日志
    </h2>
    <ol>
     <li>
      <strong>
       打开构建日志窗口
      </strong>
      ：
      <ul>
       <li>
        点击工具栏中的
        <code>
         Toggle Build Output
        </code>
        按钮（通常显示为
        <code>
         &gt;
        </code>
        ），或使用快捷键
        <code>
         Alt + 0
        </code>
        。
       </li>
       <li>
        构建日志会显示在底部窗口中，帮助诊断构建过程中的问题。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       分析错误信息
      </strong>
      ：
      <ul>
       <li>
        如果构建失败，查看构建日志中的错误信息，根据提示修复代码或依赖项。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h2>
     注意事项
    </h2>
    <ul>
     <li>
      <strong>
       确保设备已授权
      </strong>
      ：
      <ul>
       <li>
        在物理设备上运行应用前，确保设备已启用“USB 调试”功能。
       </li>
       <li>
        在模拟器上运行时，确保模拟器已正确配置并启动。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       保持代码整洁
      </strong>
      ：
      <ul>
       <li>
        定期清理不必要的代码和注释，提高可读性和维护性。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       版本控制
      </strong>
      ：
      <ul>
       <li>
        使用 Git 等版本控制系统管理代码，方便回溯和协作。
       </li>
      </ul>
     </li>
    </ul>
