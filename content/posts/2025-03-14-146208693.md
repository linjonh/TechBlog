     docker构建镜像时总会有部分文件没有被更新，解决办法
    </h1>

  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     在构建镜像的时候总会遇到明明更新了里面的某些库文件，但是打包镜像以后发现镜像里面的库文件还是没有被更新，依然使用的旧文件。
    </p>
    <p>
     试过了很多办法，比如 在构建镜像的时候使用 --no-cache
    </p>
    <pre><code class="language-bash">docker build --no-cache . -t 192.168.11.111:11211/comm:3.2</code></pre>
    <p>
     然而并没有效果，新打包的镜像还是存在部分旧文件，最后无奈把Dokerfile文件换了一种写法，解决问题，当然在此之前试过使用笨办法也能解决；
    </p>
    <p>
     解决办法：
    </p>
    <p>
     原本我的 Dockerfile
    </p>
    <pre><code class="language-bash">FROM 192.168.10.110:1199/std-base:4.0

ADD bin /home/huu/app/bin
ADD appdata /home/huu/app/appdata
ADD lib/* /home/huu/app/lib/
ADD lib/plugin/* /home/huu/app/lib/plugin/
ENV PROJECTHOME="/home/huu/app"
RUN chmod 755 $PROJECTHOME/bin/start-app.sh
RUN chmod 755 $PROJECTHOME/bin/appinstance
CMD ["/home/huu/app/bin/start-app.sh"]

</code></pre>
    <p>
     重新修改后的 Dockerfile
    </p>
    <pre><code class="language-bash">FROM 192.168.10.110:1199/std-base:4.0

ADD bin /home/huu/app/bin
ADD appdata /home/huu/app/appdata
ADD lib/ /home/huu/app/lib/
#ADD lib/plugin/* /home/huu/app/lib/plugin/
ENV PROJECTHOME="/home/huu/app"
RUN chmod 755 $PROJECTHOME/bin/start-app.sh
RUN chmod 755 $PROJECTHOME/bin/appinstance
CMD ["/home/huu/app/bin/start-app.sh"]

</code></pre>
    <p>
     主要区别就是 lib目录，使用了 lib/  直接添加了整个目录，包括整个目录的文件和文件夹
    </p>
    <p>
     而原来的是为了添加 plugin目录，又把文件重新加了一遍。经过这样修改以后，之前总是没有更新的几个库文件这次都更新了；
    </p>
    <p>
    </p>
    <p>
     笨办法：
     <br/>
     使用原本的Dockerfile文件构建一次镜像，例如：
    </p>
    <pre><code class="language-bash">docker  build .  -t  192.168.10.110:1199/app:2.0</code></pre>
    <p>
     然后运行
    </p>
    <pre><code class="language-bash">docker run -it &lt;镜像ID&gt;   /bin/bash</code></pre>
    <p>
     最后将需要更新的文件，复制到这个容器中
    </p>
    <pre><code class="language-bash">docker cp libodbc.so 123456789abc:/app/lib</code></pre>
    <p>
     将容器打包成一个新的镜像：
    </p>
    <pre><code class="language-bash">docker commit &lt;容器id&gt;  192.168.10.110:1199/app:2.1</code></pre>
    <p>
     然后检查新打包的镜像
    </p>
    <pre><code class="language-bash">docker images</code></pre>
    <p>
     笨办法适用于临时使用，最好还是使用Dockerfile构建镜像
    </p>
