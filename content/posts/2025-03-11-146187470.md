    <h2>
     <a id="Git_10">
     </a>
     一、初识Git
    </h2>
    <h3>
     <a id="_11">
     </a>
     问题引入
    </h3>
    <blockquote>
     <p>
      想象一下，你现在在写一份材料报告，交了第一版后被mentor打回，改到第十版又被要求还是提交第一版，这个时候你估计想刀Ta的心就有了
     </p>
    </blockquote>
    <p>
     诸如此类，在日常工作或学习中，我们常常遇到需要保存多个版本文件的情况，例如文档编写时为了防止丢失或误改而产生的多个副本（如“报告-v1”、“报告-v2”等）。随着版本数量增加，不仅文件数量增多，而且难以记住每个版本具体修改了什么内容。同样地，项目代码也存在同样的问题
    </p>
    <h3>
     <a id="_16">
     </a>
     解决方案
    </h3>
    <p>
     为了能够更⽅便我们管理这些不同版本的⽂件，便有了
     <strong>
      版本控制器
     </strong>
     。所谓的版本控制器，就是能让你了解到⼀个⽂件的历史，以及它的发展过程的系统。
    </p>
    <p>
     通俗的讲就是⼀个可以记录⼯程的每⼀次改动和版本迭代的⼀个管理系统，同时也⽅便多⼈协同作业。
    </p>
    <p>
     ⽬前最主流的
     <strong>
      版本控制器就是 Git
     </strong>
     。Git 可以控制电脑上所有格式的⽂件，例如 doc、excel、dwg、dgn、rvt等等。对于我们程序员来说，Git 最重要的就是可以帮助我们管理软件开发项⽬中的源代码⽂件！
    </p>
    <h3>
     <a id="_23">
     </a>
     注意事项
    </h3>
    <p>
     还需要再明确⼀点，所有的版本控制系统，Git 也不例外，
     <strong>
      其实只能跟踪⽂本⽂件的改动
     </strong>
     ，⽐如 TXT ⽂件，⽹⻚，所有的程序代码等等。版本控制系统可以告诉你每次的改动，⽐如在第5⾏加了⼀个单词“Linux”，在第8⾏删了⼀个单词 “Windows”。
    </p>
    <p>
     ⽽图⽚、视频这些⼆进制⽂件，虽然也能由版本控制系统管理，但没法跟踪⽂件的变化，只能把⼆进制⽂件每次改动串起来，也就是只知道图⽚从100KB改成了120KB，但到底改了啥，版本控制系统不知道，也没法知道。
    </p>

    <h2>
     <a id="Git_46">
     </a>
     三、Git配置与基本操作
    </h2>
    <h3>
     <a id="Git_47">
     </a>
     Git创建
    </h3>
    <p>
     要提前说的是，仓库是进⾏版本控制的⼀个⽂件⽬录。我们要想对⽂件进⾏版本控制，就必须先创建⼀个仓库出来
    </p>
    <p>
     创建本地Git仓库的命令是git init。此命令需要在你希望作为仓库根目录的文件夹内执行
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c95a46d54f0c4cc3b97e2f143dfea864.png">
      <br/>
      执行上述命令后，你会注意到当前目录下出现了一个名为.git的隐藏目录，而我已经创建过，所以提示重复初始化git仓库
     </img>
    </p>
    <p>
     .git目录是Git用来跟踪和管理仓库的地方。它包含了所有必要的元数据和配置文件。注意不要手动修改这个目录中的文件，以免破坏Git仓库的结构。
    </p>
    <p>
     对于有兴趣深入了解的 uu，可以在不影响项目的情况下探索.git目录的内容，以增进对Git工作原理的理解。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a9bd8d63a919434baee70e35ee4977c4.png"/>
    </p>
    <h3>
     <a id="Git_60">
     </a>
     Git配置用户名称和地址
    </h3>
    <p>
     当安装完 Git 后，首要任务是设置你的用户名称和 E-mail 地址。这是非常重要的，因为每次 Git 提交都会使用这些信息。
    </p>
    <ol>
     <li>
      全局配置（适用于所有仓库）：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"Your Name"</span>
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"email@example.com"</span>
</code></pre>
    <ol start="2">
     <li>
      本地配置（仅适用于当前仓库）：如果希望在不同仓库中使用不同的用户名或邮箱，可以在特定仓库内执行以下命令，而不带 --global 选项：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">git</span> config user.name <span class="token string">"Your Name"</span>
<span class="token function">git</span> config user.email <span class="token string">"email@example.com"</span>
</code></pre>
    <ol start="3">
     <li>
      查看配置：使用以下命令可以列出所有的 Git 配置项：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">-l</span>
<span class="token function">git</span> config <span class="token parameter variable">--list</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1194dfe62d32463f85e536ee81c2f851.png"/>
    </p>
    <ol start="4">
     <li>
      删除配置：若要删除某个配置项，可以使用 --unset 选项：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> <span class="token parameter variable">--unset</span> user.name
<span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> <span class="token parameter variable">--unset</span> user.email
</code></pre>
    <h3>
     <a id="_93">
     </a>
     认识工作区、暂存区、版本库
    </h3>
    <ul>
     <li>
      工作区：这是你在电脑上编写代码或文件的目录。
     </li>
     <li>
      暂存区（Stage/Index）：一般存放于 .git/index 文件中，用于存储即将提交到版本库中的更改。
     </li>
     <li>
      版本库（Repository）：位于工作区的隐藏 .git 目录下，用来管理所有文件的修改和历史记录。
     </li>
    </ul>
    <p>
     当你创建一个新的 Git 版本库时，Git 会自动创建一个名为 master 的分支及指向该分支的 HEAD 指针。对工作区的文件进行修改后，需先通过 git add 命令将更改添加到暂存区，再通过 git commit 将暂存区的内容提交至版本库。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2d2e1537a0bc40baa3fe4c93d79a2803.png"/>
    </p>
    <p>
     .git 是 Git 用来跟踪和管理仓库的地方，包含了所有必要的元数据和配置文件。不要手动修改这个目录中的文件，以免破坏 Git 仓库的结构。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/440f8d15469149d786b6ecdb11ec36a0.png"/>
    </p>
    <h3>
     <a id="_106">
     </a>
     添加文件到仓库
    </h3>
    <p>
     要将新文件或修改后的文件添加到 Git 管理中，需要经历两个步骤：add 和 commit。
    </p>
    <h4>
     <a id="_109">
     </a>
     添加文件到暂存区
    </h4>
    <p>
     添加单个或多个文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token punctuation">[</span>file1<span class="token punctuation">]</span> <span class="token punctuation">[</span>file2<span class="token punctuation">]</span> <span class="token punctuation">..</span>.
</code></pre>
    <p>
     添加指定目录及其子目录：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token punctuation">[</span>dir<span class="token punctuation">]</span>
</code></pre>
    <p>
     添加当前目录下的所有改动：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>
</code></pre>
    <h4>
     <a id="_126">
     </a>
     提交暂存区内容到本地仓库
    </h4>
    <p>
     提交暂存区的所有内容（默认）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"message"</span>
</code></pre>
    <p>
     提交暂存区的指定文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> commit <span class="token punctuation">[</span>file1<span class="token punctuation">]</span> <span class="token punctuation">[</span>file2<span class="token punctuation">]</span> <span class="token punctuation">..</span>. <span class="token parameter variable">-m</span> <span class="token string">"message"</span>
</code></pre>
    <p>
     注意，git commit 后面的 -m 参数用于提供描述性提交信息，这部分内容非常重要，因为它记录了你所做的更改细节。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c53ebe09ce134844a4be1251f436247d.png"/>
    </p>
    <h3>
     <a id="_142">
     </a>
     查看提交历史
    </h3>
    <p>
     你可以使用 git log 查看项目的提交历史。为了简化输出，还可以加上 --pretty=oneline 参数以获得更简洁的格式。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> log <span class="token parameter variable">--pretty</span><span class="token operator">=</span>oneline
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/81742b1169ba4196b0ffef879561c2c6.png"/>
     <br/>
     这将显示每个提交的简短摘要（漂亮的一行），包括 提交 ID（SHA1 格式的哈希值），它是一个由 Git 自动生成的
     <strong>
      唯一标识符
     </strong>
     ，用以区分不同的提交。
    </p>
    <blockquote>
     <p>
      你看到的 commit ID 肯定跟我不一样，这以你的为准
     </p>
    </blockquote>
    <h2>
     <a id="Git_HEAD_153">
     </a>
     四、Git 暂存区、HEAD、对象库及文件
    </h2>
    <h3>
     <a id="Git_154">
     </a>
     Git内部结构概览
    </h3>
    <ol>
     <li>
      暂存区（Index）
     </li>
    </ol>
    <ul>
     <li>
      index 文件位于 .git 目录下，是暂存区的物理存储位置。
     </li>
     <li>
      使用 git add 命令后，修改或新增的内容会被添加到这里。暂存区的作用是在提交前收集所有要纳入版本控制的更改。
     </li>
    </ul>
    <ol start="2">
     <li>
      HEAD 指针
     </li>
    </ol>
    <ul>
     <li>
      默认情况下，HEAD 是指向当前分支的一个指针，通常是指向 master 分支。
     </li>
     <li>
      可以通过查看 .git/HEAD 文件来确认 HEAD 当前所指向的分支。
     </li>
    </ul>
    <ol start="3">
     <li>
      master 分支
     </li>
    </ol>
    <ul>
     <li>
      master 分支是一个特殊的默认分支，代表项目的主线发展。
     </li>
     <li>
      查看 master 分支最新的提交 ID（commit id），可以通过查看 .git/refs/heads/master 文件内容
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ebdd29c4a4734086ad2a5f243485ae1c.png"/>
    </p>
    <ol start="4">
     <li>
      对象库（Objects）
     </li>
    </ol>
    <ul>
     <li>
      .git/objects 目录存放了所有被 Git 管理的对象，包括提交（commits）、树（trees）、标签（tags）和 blob（文件内容）。
     </li>
     <li>
      对象名由 SHA-1 哈希值表示，前两位作为子目录名，剩余部分作为文件名。例如，对于提交 ID c9091af21fd2929fd9a8c42876a1f6ba686eb3b9，其路径为 .git/objects/c9/091af21fd2929fd9a8c42876a1f6ba686eb3b9。
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/048dfde4662e4236a0428db0ca4fb205.png"/>
    </p>
    <h3>
     <a id="_Git__177">
     </a>
     查看和理解 Git 对象
    </h3>
    <ul>
     <li>
      使用 git cat-file -p &lt; object-id &gt; 命令可以解码并查看对象的内容。
     </li>
     <li>
      提交对象包含元数据（如作者信息、时间戳）以及指向树对象的引用
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5f897ec377bf4c35998266dd70c0346f.png"/>
    </p>
    <h3>
     <a id="_182">
     </a>
     工作区、暂存区、版本库的操作流程
    </h3>
    <p>
     <strong>
      添加文件到仓库
     </strong>
    </p>
    <ul>
     <li>
      新建或修改文件后，使用 git add 将更改添加到暂存区。
     </li>
     <li>
      使用 git commit 提交暂存区的内容到本地仓库中。注意，只有在暂存区中的文件才会被提交。
     </li>
    </ul>
    <p>
     <strong>
      处理未添加到暂存区的文件
     </strong>
    </p>
    <ul>
     <li>
      如果有新的文件没有通过 git add 添加到暂存区，那么这些文件不会被 git commit 捕获。
     </li>
     <li>
      解决方法是再次执行 git add 后再进行 git commit。
     </li>
    </ul>
    <p>
     <strong>
      修改文件
     </strong>
    </p>
    <ul>
     <li>
      Git 跟踪的是修改而非文件本身。
      <strong>
       任何对文件内容的变更（如新增行、删除行、更改字符等）都被视为一次修改。
      </strong>
     </li>
     <li>
      修改文件后，可以使用 git status 查看哪些文件已被修改但尚未添加到暂存区。
     </li>
     <li>
      使用 git diff [file] 显示工作区与暂存区之间的差异，帮助理解具体做了哪些改动。
     </li>
     <li>
      使用 git diff HEAD – [file] 查看工作区与最近一次提交之间的区别。
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cf5bab8fabfe4e0495dc16953f983e0f.png"/>
     <br/>
     <strong>
      提交修改后的文件
     </strong>
    </p>
    <ul>
     <li>
      修改文件后，先用 git add 将更改添加到暂存区，然后用 git commit 提交更改。
     </li>
     <li>
      提交完成后，git status 应显示
      <strong>
       nothing to commit, working tree clean
      </strong>
      ，表明没有待提交的更改。
     </li>
    </ul>
