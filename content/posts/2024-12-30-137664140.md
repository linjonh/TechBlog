    <p>
     <a href="" rel="nofollow">
     </a>
     一，WXSS
    </p>
    <hr/>
    <p>
     首先我们先介绍一下
     <code>
      wxss
     </code>
     ，
     <code>
      wxss
     </code>
     是微信小程序独有的模板样式，它类似于web开发中的
     <code>
      CSS
     </code>
     ，具备
     <code>
      CSS
     </code>
     大部分的功能，同时，它还拓展出了几个内容功能，接下来牛牛介绍一下新增的
     <code>
      rpx尺寸单位
     </code>
     以及
     <code>
      样式导入@import
     </code>
     。
    </p>
    <h4>
     <a id="_11_rpx_40">
     </a>
     <a href="" rel="nofollow">
     </a>
     1.1 rpx尺寸单位
    </h4>
    <p>
     <code>
      rpx
     </code>
     尺寸单位是微信独有的，主要用来解决屏适配问题。怎么回事呢？我们知道，手机款式很多，手机屏幕的大小也并不一致，我们想要让小程序根据屏幕宽度自动完成屏适配，于是有了
     <code>
      rpx
     </code>
     ，
     <code>
      rpx
     </code>
     将屏幕宽度等分为750份，我们在开发的时候使用
     <code>
      rpx
     </code>
     尺寸单位，到使用的时候就会将其换算成像素单位，实现屏适配。
    </p>
    <p>
     tip:
    </p>
    <p>
     微信建议开发者， 开发微信小程序时设计师可以用
     <code>
      iPhone6
     </code>
     作为视觉稿的标准。
    </p>
    <h4>
     <a id="_12_import_54">
     </a>
     <a href="" rel="nofollow">
     </a>
     1.2 样式导入@import
    </h4>
    <p>
     在
     <code>
      CSS
     </code>
     中，我们用
     <code>
      &lt;link&gt;
     </code>
     标签导入外联样式表，在微信小程序中，单个页面的
     <code>
      wxss
     </code>
     文件渲染页面并不用我们操心，不过有时候我们需要格外导入外联样式，而在
     <code>
      WXSS
     </code>
     中，我们使用
     <code>
      @import
     </code>
     实现。
    </p>
    <p>
     语法：
     <code>
      @import "相对路径"
     </code>
    </p>
    <h4>
     <a id="_13__66">
     </a>
     <a href="" rel="nofollow">
     </a>
     1.3 全局样式
    </h4>
    <p>
     微信小程序的全局样式是写在根目录的
     <code>
      app.wxss
     </code>
     文件中的，全局样式就意味着里面的样式将在每一个页面均生效。
    </p>
    <p>
     需要注意的是，当全局样式与页面的局部样式发生冲突时，但局部选择器权重大于或等于全局样式选择器权重的时候，会采取就近原则，用局部样式的定义渲染。
    </p>
    <p>
     <a href="" rel="nofollow">
     </a>
     二，全局配置
    </p>
    <hr/>
    <p>
     全局配置我们在之前的文章也有做过一定的了解，根目录下的
     <code>
      app.json
     </code>
     文件就是专门放全局配置的地方。
    </p>
    <p>
     <code>
      app.json
     </code>
     文件中主要有六部分。
    </p>
    <ol>
     <li>
      <code>
       pages
      </code>
      ，用来存放页面的路径，第一个放的就是小程序的首页。
     </li>
    </ol>
    <pre><code>
  "pages": [

    "pages/index/index",

    "pages/list/list"

  ],



</code></pre>
    <ol start="2">
     <li>
      <code>
       window
      </code>
      ，页面的窗口外观，如：
     </li>
    </ol>
    <pre><code>
  "window": {

    "navigationBarBackgroundColor": "#fff",

    "navigationBarTextStyle": "black",

    "navigationBarTitleText": "第一个小程序"

  },



</code></pre>
    <ol start="3">
     <li>
      <code>
       tabBar
      </code>
      ，页面的底部栏，微信小程序中这部分经常作为跳转页面的工具。
     </li>
    </ol>
    <pre><code>
  "tabBar": {

    "list": [

      {

        "pagePath": "pages/index/index",

        "text": "首页"

      },

      {

        "pagePath": "pages/list/list",

        "text": "目录"

      }

    ],

    "color": "#fff",

    "selectedColor": "#999",

    "backgroundColor": "#111"

  },



</code></pre>
    <ol start="4">
     <li>
      还有一个常见的就是
      <code>
       style
      </code>
      ，用来设置是否启用新版组件样式，如果需要，需要让
      <code>
       style
      </code>
      的值为
      <code>
       v2
      </code>
      ,如果不采用，直接把
      <code>
       style
      </code>
      那一行删掉即可。
     </li>
    </ol>
