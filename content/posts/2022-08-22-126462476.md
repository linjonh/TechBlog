    <h2>
     <a id="_26">
     </a>
     图像基本概念
    </h2>
    <h3>
     <a id="_27">
     </a>
     图像
    </h3>
    <ol>
     <li>
      <strong>
       视频
      </strong>
      是由
      <strong>
       一组图像组成
      </strong>
      ，为了传输被
      <strong>
       压缩
      </strong>
      ，最终
      <strong>
       在显示设备
      </strong>
      上显示；
     </li>
     <li>
      <strong>
       图像
      </strong>
      是由
      <strong>
       像素
      </strong>
      组成，像素是由
      <strong>
       RGB
      </strong>
      组成；
      <ul>
       <li>
        <p>
         RGB即三原色（红、绿、蓝），可以混合成世界上所有的颜色。具体RGB值表示如下图所示：
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b8cc10e8650b3dddea1e69dee14b6d84.png"/>
        </p>
        <ul>
         <li>
          彩色图像中每个点，有R、G、B三个分量组成；以RGB24为例，图像像素数据的
          <strong>
           存储方式
          </strong>
          如下：
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/708b7cb6481904d542d9d8330d0cfb68.png"/>
         </li>
        </ul>
        <p>
         从图中可以看出，RGB24一次存储了每个像素点的R、G、B信息。
        </p>
        <ul>
         <li>
          与RGB类似的还有BGR格式，可以互相转换。转换方式可以自己写转换程序，也可以使用有swscale相关的ffmpeg库。
         </li>
         <li>
          RGB
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          分辨率
         </strong>
         表示横向的像素点个数与纵向的像素点个数
        </p>
       </li>
       <li>
        <p>
         <strong>
          位深
         </strong>
         ：每一个像素占用的
         <strong>
          位数
         </strong>
         ；最常见的是24位-RGB888，每个颜色都是8位；还有32位-RGBA，A表示alpha（透明度）。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_43">
     </a>
     屏幕
    </h3>
    <ul>
     <li>
      <p>
       一个像素是三个发光二极管组成，可以展示一组RGB值，如下图所示：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d5965d57eceba39c59b9805863fc4a4a.png"/>
      </p>
     </li>
     <li>
      <p>
       屏幕上的
       <strong>
        每个像素由三个发光二极管
       </strong>
       组成，整个屏幕即可表示为下图：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/673be14fc493aba4f04b236921528640.png"/>
      </p>
     </li>
     <li>
      <p>
       图像与屏幕的关系就是：图像是
       <strong>
        数据
       </strong>
       ，屏幕是
       <strong>
        显示设备
       </strong>
       ，图像数据经过驱动程序让屏幕显示图像。
      </p>
     </li>
     <li>
      <p>
       表示屏幕质量的指标：
      </p>
      <ul>
       <li>
        PPI（pixels per inch）：每英寸的像素数
       </li>
       <li>
        DPI(Dots pen inch)：每英寸的点数，基本上DPI=PPI
       </li>
       <li>
        PPI&gt;300就属于视网膜级别，人眼区分不出来，认为是一体的
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="_56">
     </a>
     图像码流如何计算？
    </h2>
    <p>
     要求的
     <strong>
      平滑度越高，帧率就越大
     </strong>
     ；要求的
     <strong>
      清晰度越高，分辨率就越大
     </strong>
    </p>
    <h3>
     <a id="_58">
     </a>
     分辨率
    </h3>
    <ul>
     <li>
      分辨率即是：
      <strong>
       X轴的像素个数 * Y轴的像素个数
      </strong>
     </li>
     <li>
      常见的宽高比
      <strong>
       16：9
      </strong>
      以及
      <strong>
       4：3
      </strong>
      <ul>
       <li>
        16：9的分辨率有360P（640
        <em>
         360）、720P（1280
        </em>
        720）、1K（1080P 1920
        <em>
         1080）、2K（1440P，2560
        </em>
        1440）、4K（4096*2160；如今大多显示器都是用此宽高比
       </li>
       <li>
        4：3常见的是640*480；旧有的老电视机会采样这个宽高比
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="_63">
     </a>
     帧率
    </h3>
    <ul>
     <li>
      帧率是：
      <strong>
       每秒钟采集/播放图像的个数
      </strong>
      。
     </li>
     <li>
      动画的帧率是：25帧/s
     </li>
     <li>
      常见的帧率是：15帧/s，30帧/s，60帧/s
     </li>
     <li>
      实时通信一般是15帧/s（帧率越大，占用的传输带宽就越大）
     </li>
     <li>
      录课一般30帧/s能够满足需求
     </li>
     <li>
      电影一般是60帧/s
     </li>
    </ul>
    <h3>
     <a id="_71">
     </a>
     码流计算
    </h3>
    <ul>
     <li>
      未编码的RGB码流 = 分辨率（宽*高）*3（Byte）*25（fps）
      <ul>
       <li>
        例如：1280
        <em>
         720
        </em>
        3
        <em>
         25=69120000 约69兆Byte/s；码流是以位为单位的，则69M
        </em>
        8=552Mbit/s
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="_76">
     </a>
     图像显示
    </h3>
    <ul>
     <li>
      图像显示的三种情况
      <ul>
       <li>
        <p>
         图像大小等于显示区域大小
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2602b1659306c43224b0d1033f7b99af.png"/>
        </p>
       </li>
       <li>
        <p>
         图像小于显示区域（拉伸/留白）
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/1a4e3478128cae0c49b62a1380d81367.png"/>
        </p>
       </li>
       <li>
        <p>
         图像大于显示区域（缩小/截断）
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ba8e34dd10176cc6b223add90f6ca405.png"/>
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="YUV_87">
     </a>
     YUV
    </h2>
    <h3>
     <a id="YUV_88">
     </a>
     YUV格式
    </h3>
    <ul>
     <li>
      相关实验表明，人眼对亮度敏感而
      <strong>
       对色度不敏感
      </strong>
      。因而可以
      <strong>
       将亮度信息和色度信息分
      </strong>
      离，并对色度信息采用更“狠”一点的压缩方案，从而提高压缩效率。
     </li>
     <li>
      YUV（也称YCbCr）格式中，
      <strong>
       Y只包含亮度信息，而UV只包含色度信息
      </strong>
      。
     </li>
     <li>
      主要的采样格式有YUV4:2:0、YUV4:2:2和YUV4:4:4。
      <ul>
       <li>
        YUV444：与RGB88同等概念
       </li>
       <li>
        YUV422：每一行来说，隔一行有个U/V，数据量：1280
        <em>
         720 +2
        </em>
        1280/2
        <em>
         720=2
        </em>
        1280*720，
        <strong>
         数据量减少了2/3
        </strong>
        。
       </li>
       <li>
        YUV420：
        <strong>
         应用最广泛的格式
        </strong>
        ，播放器必须支持
       </li>
      </ul>
     </li>
     <li>
      为什么使用YUV? 显示器从黑白显示器演变过来的,为了和以前格式兼容以及
      <strong>
       YUV存储的数据比RGB要少很多
      </strong>
      。
     </li>
    </ul>
    <h3>
     <a id="RGBYUV_97">
     </a>
     RGB与YUV的转换
    </h3>
    <ul>
     <li>
      <p>
       RGB与YUV关系：
       <strong>
        RGB用于屏幕图像的展示，YUV用于采集与编码。
       </strong>
      </p>
     </li>
     <li>
      <p>
       YUV转RGB：
      </p>
      <ul>
       <li>
        R = Y + 1.140*v
       </li>
       <li>
        G = Y - 0.394
        <em>
         U -0.581
        </em>
        v
       </li>
       <li>
        B = Y + 2.032*u
       </li>
      </ul>
     </li>
     <li>
      <p>
       RGB转YUV
      </p>
      <ul>
       <li>
        Y = 0.299
        <em>
         R + 0.587
        </em>
        G + 0.114*B
       </li>
       <li>
        U = -0.147
        <em>
         R - 0.289
        </em>
        G + 0.436*B = 0.492（B-Y）
       </li>
       <li>
        V = 0.615R - 0.515
        <em>
         G - 0.100
        </em>
        B = 0.877*（R-Y）
       </li>
      </ul>
     </li>
     <li>
      <p>
       4：2：0并
       <strong>
        不意味着只有Y、Cb两个分量，而没有Cr分量
       </strong>
       。它实际指的是每行扫描线来说，只有一种色度分量，它以2：1的抽样率存储。相邻的扫描行存储不同的色度分量，也就是说，如果一行是4：2：0的话，下一行就是4：0：2，再下一行是4：2：0…以此类推
      </p>
     </li>
    </ul>
    <h3>
     <a id="YUV_110">
     </a>
     YUV存储
    </h3>
    <ul>
     <li>
      以
      <strong>
       YUV420P
      </strong>
      为例，图像像素数据的存储方式如图所示。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/20d3ffdd6ac77404206c4ac1a24a0423.png">
       <br/>
       从图中可以看出，YUV420P
       <strong>
        首先存储整张图像的Y信息
       </strong>

       <strong>

       </strong>
       ，
       <strong>
        最后存储了整张图像的V信息
       </strong>
       。
      </img>
     </li>
    </ul>
