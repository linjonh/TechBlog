    <p>
     Java Virtual Machine（JVM）是Java程序的运行环境（java二进制字节码的运行环境）
     <br/>
     好处：
    </p>
    <ul>
     <li>
      一次编写，到处运行
     </li>
     <li>
      自动内存管理，垃圾回收机制
      <br/>
      JVM由哪些部分组成，运行流程是什么？
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b1b5198b07434ea6a72f063b60d722c4.png"/>
     </li>
    </ul>
    <hr/>
    <h2>
     <a id="1__10">
     </a>
     1. 程序计数器
    </h2>
    <p>
     程序计数器：线程私有（不存在线程安全），
    </p>
    <ol>
     <li>
      原理：内部记录正在执行的字节码指令的地址（行号）。
     </li>
     <li>
      功能：保存当前线程的行号，如果时间片被抢占，再次执行会从记录的行号继续执行，不会从头执行
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c1e9ee53854a41949329ded27a8f5468.png"/>
     </li>
    </ol>
    <h2>
     <a id="2_Java_17">
     </a>
     2. Java堆
    </h2>
    <h3>
     <a id="21__18">
     </a>
     2.1 结构
    </h3>
    <p>
     线程共享的区域（存在线程安全）
     <br/>
     主要用来保存对象实例，数组等，当堆中没有内存空间可分配给实例，也无法再扩展时，则抛出OutOfMemoryError异常。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/54987f656f88483b930c2160dd7ddaf7.png"/>
    </p>
    <p>
     年轻代被划分为三部分，Eden区和两个大小严格相同的Survivor区，
     <br/>
     根据JVM的策略，在经过几次垃圾收集后，任然存活于Survivor的对象将被移动到老年代区间。
     <strong>
      老年代主要保存生命周期长的对象，一般是一些老的对象
     </strong>
    </p>
    <h3>
     <a id="22_Java_1718_25">
     </a>
     2.2 Java 1.7与1.8堆的区别
    </h3>
    <ul>
     <li>
      1.7中有有一个永久代，存储的是类信息、静态变量、常量、编译后的代码
     </li>
     <li>
      1.8移除了永久代，把数据存储到了本地内存的元空间中，防止内存溢出
      <br/>
      优化点：1.7堆的永久代在1.8更新成了本地内存中的元空间，解决永久代多容易OOM，少了会浪费内存的问题
     </li>
    </ul>
    <h2>
     <a id="3__29">
     </a>
     3. 虚拟机栈
    </h2>
    <p>
     Java Virtual machine Stacks（java 虚拟机栈）
    </p>
    <ul>
     <li>
      每个线程运行时所需要的内存，称为虚拟机栈，先进后出（多个线程运行就会创建多个虚拟机栈——
      <strong>
       线程安全
      </strong>
      ）
     </li>
     <li>
      每个栈由多个栈帧（frame）组成，对应着每次方法调用时所占用的内存
     </li>
     <li>
      每个线程只能有一个活动栈帧，对应着当前正在执行的那个方法
     </li>
    </ul>
    <h3>
     <a id="31__34">
     </a>
     3.1 垃圾回收是否涉及栈内存
    </h3>
    <p>
     不涉及。垃圾回收主要指就是堆内存，当栈帧弹栈以后，内存就会释放
    </p>
    <h3>
     <a id="32__36">
     </a>
     3.2 栈内存分配越大越好吗
    </h3>
    <p>
     视情况分析，默认的栈内存通常为1024k
     <br/>
     栈帧过大会导致线程数变少，例如，机器总内存为512m，目前能活动的线程数则为512个，如果把栈内存改力2048k，那么能活动的栈帧就会减半
    </p>
    <h3>
     <a id="33__39">
     </a>
     3.3 方法内的局部变量是否线程安全？
    </h3>
    <ul>
     <li>
      如果方法内局部变量没有逃离方法的作用范围，它是线程安全的
     </li>
     <li>
      如果是局部变量引用了对象，并逃离方法的作用范围，需要考虑线程安全
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2b3e9ac8bc3745eaae531fa149d95651.png"/>
     </li>
    </ul>
    <h3>
     <a id="34__45">
     </a>
     3.4 栈内存溢出情况
    </h3>
    <ul>
     <li>
      栈帧过多导致栈内存溢出，典型问题：递归调用
     </li>
     <li>
      栈帧过大导致栈内存溢出
     </li>
    </ul>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">static</span> <span class="token keyword">void</span> <span class="token function">m4</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>
	<span class="token function">m4</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">//java.lang.StackOverflowError</span>

    <h3>
     <a id="35__53">
     </a>
     3.5 堆栈的区别是什么
    </h3>
    <ul>
     <li>
      栈内存一般会用来存储局部变量和方法调用，但堆内存是用来存储Java对象和数组的的。堆会GC垃圾回收，而栈不会。
     </li>
     <li>
      栈内存是线程私有的，而堆内存是线程共有的。
     </li>
     <li>
      两者异常错误不同，但如果栈内存或者堆内存不足都会抛出异常。
      <br/>
      栈空间不足：java.lang.StackOverFlowError。
      <br/>
      堆空间不足：javalang.OutOfMemoryError。
     </li>
    </ul>
    <h2>
     <a id="4__59">
     </a>
     4. 方法区
    </h2>
    <ul>
     <li>
      方法区（Method Area）是各个线程共享的内存区域
     </li>
     <li>
      主要存储类的信息、运行时常量池
     </li>
     <li>
      虚拟机启动的时候创建，关闭虚拟机时释放
     </li>
     <li>
      如果方法区域中的内存无法满足分配请求，则会抛出OutOfMemoryError: Metaspace
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/54095c3a82c740af9dd9f22cc3e28c4f.png"/>
     </li>
    </ul>
    <p>
     jdk1.7到1.8把这个区从堆移到元空间（本地空间）
    </p>
    <h3>
     <a id="41__67">
     </a>
     4.1 常量池
    </h3>
    <p>
     可以看作是一张表，虚拟机指令根据这张常量表找到要执行的类名、方法名、参数类型、字面量等信息
    </p>
    <pre><code class="prism language-java">javap <span class="token operator">-</span>v <span class="token class-name">Application</span><span class="token punctuation">.</span><span class="token keyword">class</span>
</code></pre>
    <p>
     查看字节码结构（类的基本信息、常量池、方法定义）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4882bb3457204290828d64413e3fde32.png"/>
    </p>
    <p>
     <strong>
      运行时常量池
     </strong>
     <br/>
     常量池是
     <code>
      *.class
     </code>
     文件中的，当该类被加载，它的常量池信息就会放入运行时常量池，并把里面的符号地址变为真实地址
    </p>
    <h2>
     <a id="5__79">
     </a>
     5. 直接内存
    </h2>
    <ul>
     <li>
      直接内存并不属于JVM中的内存结构，不由JVM进行管理。是虚拟机的系统内存
     </li>
     <li>
      常见于NIO操作时，用于数据缓冲区，它分配回收成本较高，但读写性能高
     </li>
    </ul>
    <h3>
     <a id="51_IO_82">
     </a>
     5.1 IO数据拷贝流程（常规）
    </h3>
    <p>
     操作磁盘文件中需要存入两次到缓存区（系统缓存区、java缓存区）
     <br/>
     因为java代码无法直接操作系统造成了java缓存区，这样导致了一次不必要的复制，导致效率低
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e353fee3a8ac4f4897d53bc7438b193f.png"/>
    </p>
    <h3>
     <a id="52_NIO_87">
     </a>
     5.2 NIO数据拷贝流程
    </h3>
    <p>
     直接内存（共享的内存区域）让系统和java代码都可以直接访问。
     <br/>
     相较IO少了一次数据复制，效率成倍的提升。所以NIO适合文件操作
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b95ba2bd35a64074b5d9db809e372929.png"/>
    </p>
