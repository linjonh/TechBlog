    <h3>
     <a id="_1">
     </a>
     内容渲染指令
    </h3>
    <p>
     内容渲染指令用来辅助开发者渲染 DOM 元素的文本内容。常用的内容渲染指令有如下2 个：
    </p>
    <h4>
     <a id="vtextinnerText_5">
     </a>
     v-text（类似innerText）
    </h4>
    <ul>
     <li>
      <ul>
       <li>
        使用语法：
        <code>
         &lt;p v-text="name"&gt;hello&lt;/p&gt;
        </code>
        ，意思是将 name 值渲染到 p 标签中
       </li>
       <li>
        类似 innerText，使用该语法，会覆盖 p 标签原有内容
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="vhtml_innerHTML_11">
     </a>
     v-html（类似 innerHTML）
    </h4>
    <ul>
     <li>
      <ul>
       <li>
        使用语法：
        <code>
         &lt;p v-html="intro"&gt;hello&lt;/p&gt;
        </code>
        ，意思是将 intro 值渲染到 p 标签中
       </li>
       <li>
        类似 innerHTML，使用该语法，会覆盖 p 标签原有内容
       </li>
       <li>
        类似 innerHTML，使用该语法，能够将HTML标签的样式呈现出来。
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="_18">
     </a>
     条件渲染指令
    </h3>
    <p>
     条件判断指令，用来辅助开发者按需控制 DOM 的显示与隐藏。条件渲染指令有如下两个，分别是：
    </p>
    <h4>
     <a id="vshow_22">
     </a>
     v-show
    </h4>
    <ol>
     <li>
      作用： 控制元素显示隐藏
     </li>
     <li>
      语法： v-show = “表达式” 表达式值为 true 显示， false 隐藏
     </li>
     <li>
      原理： 切换 display:none 控制显示隐藏
     </li>
     <li>
      场景：频繁切换显示隐藏的场景
     </li>
    </ol>
    <h4>
     <a id="vif_29">
     </a>
     v-if
    </h4>
    <ol>
     <li>
      作用： 控制元素显示隐藏（条件渲染）
     </li>
     <li>
      语法： v-if= “表达式” 表达式值 true显示， false 隐藏
     </li>
     <li>
      原理： 基于条件判断，是否创建 或 移除元素节点
     </li>
     <li>
      场景： 要么显示，要么隐藏，不频繁切换的场景
     </li>
    </ol>

     <a id="_computed__124">
     </a>
     <strong>
      与 computed 的区别
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th align="left">
        <strong>
         <code>
          methods
         </code>
        </strong>

        <strong>
         <code>
          computed
         </code>
        </strong>

      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        每次调用都会执行
       </td>
       <td align="left">

       </td>
      </tr>
      <tr>
       <td align="left">

       </td>
      </tr>
     </tbody>

    <h5>
     <a id="_212">
     </a>
     <strong>
      最佳实践
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       单一职责
      </strong>
      ：每个方法专注于一个任务，提高可维护性。
     </li>
     <li>
      <strong>
       避免直接操作 DOM
      </strong>
      ：优先通过数据驱动视图，而非直接操作 DOM。
     </li>
     <li>
      <strong>
       命名清晰
      </strong>
      ：使用动词命名方法（如
      <code>
       handleClick
      </code>
      、
      <code>
       fetchData
      </code>
      ）。
     </li>
     <li>
      <strong>
       复用逻辑
      </strong>
      ：复杂逻辑封装为方法，避免在模板中写过多表达式。
     </li>
    </ol>

    <h4>
     <a id="computed_256">
     </a>
     computed
    </h4>
    <p>
     在 Vue 中，
     <code>
      computed
     </code>
     是用于定义
     <strong>
      计算属性
     </strong>
     的核心选项，它基于响应式依赖进行缓存，适合处理需要根据其他数据动态计算的场景。
    </p>
    <hr/>
    <h5>
     <a id="_262">
     </a>
     <strong>
      核心特性
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       响应式依赖
      </strong>
      ：
      <code>
       computed
      </code>
      属性会追踪其依赖的响应式数据（如
      <code>
       data
      </code>
      、
      <code>
       props
      </code>
      、其他
      <code>
       computed
      </code>
      ），当依赖变化时自动重新计算。
     </li>
     <li>
      <strong>
       缓存机制
      </strong>
      ：只有当依赖变化时才会重新计算，否则直接返回缓存结果，性能高效。
     </li>
     <li>
      <strong>
       声明式逻辑
      </strong>
      ：将复杂逻辑封装为计算属性，避免模板中写过多表达式。
     </li>
    </ul>
    <hr/>

     <a id="_270">

    <h5>
     <a id="_426">
     </a>
     <strong>
      常见问题
     </strong>
    </h5>
    <p>
     <strong>
      Q1:
      <code>
       computed
      </code>
      中可以调用异步方法吗？
     </strong>
    </p>
    <ul>
     <li>
      不建议！
      <code>
       computed
      </code>
      设计用于同步计算，异步操作应放在
      <code>
       methods
      </code>
      或
      <code>
       watch
      </code>
      中。
     </li>
    </ul>
    <p>
     <strong>
      Q2:
      <code>
       computed
      </code>
      依赖非响应式数据会怎样？
     </strong>
    </p>
    <ul>
     <li>
      如果依赖的数据不是响应式的（如未在
      <code>
       data
      </code>
      中定义），计算属性不会自动更新。
     </li>
    </ul>
    <p>
     <strong>
      Q3:
      <code>
       computed
      </code>
      和
      <code>
       watch
      </code>
      的区别？
     </strong>
    </p>
    <ul>
     <li>
      <code>
       computed
      </code>
      ：声明式派生数据，依赖多个值。
     </li>
     <li>
      <code>
       watch
      </code>
      ：监听单个值的变化，执行副作用（如异步请求、复杂逻辑）。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      总结
     </strong>
     ：
    </p>
    <p>
     <code>
      computed
     </code>
     是 Vue 中处理复杂数据逻辑的利器，通过缓存机制提升性能，保持代码简洁。
    </p>
    <p>
     优先使用
     <code>
      computed
     </code>
     替代模板内复杂表达式！
    </p>

    <h3>
     <a id="_606">
     </a>
     属性绑定指令
    </h3>
    <h4>
     <a id="vbind_608">
     </a>
     v-bind
    </h4>
    <p>
     <code>
      v-bind
     </code>
     是 Vue.js 中用于
     <strong>
      动态绑定属性
     </strong>
     或
     <strong>
      组件 Props
     </strong>
     的指令，它可以将 Vue 实例中的数据与 HTML 属性或组件 Props 进行绑定，实现数据到视图的动态响应。
    </p>
    <hr/>
    <h5>
     <a id="_614">
     </a>
     <strong>
      核心功能
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        动态绑定 HTML 属性
       </strong>
       <br/>
       将 HTML 元素的属性（如
       <code>
        href
       </code>
       、
       <code>
        src
       </code>
       、
       <code>
        class
       </code>
       、
       <code>
        style
       </code>
       等）绑定到 Vue 实例的数据，当数据变化时，属性自动更新。
      </p>
     </li>
     <li>
      <p>
       <strong>
        传递 Props 到子组件
       </strong>
       <br/>
       将父组件的数据传递给子组件的 Props，实现父子组件通信。
      </p>
     </li>
     <li>
      <p>
       <strong>
        动态绑定属性名
       </strong>
       <br/>
       根据数据动态决定绑定的属性名（如
       <code>
        :[attributeName]="value"
       </code>
       ）。
      </p>
     </li>
    </ol>
    <hr/>

     <a id="_627">
     </a>
     <strong>
      基本语法
     </strong>

    </h3>
    <h4>
     <a id="vfor_761">
     </a>
     v-for
    </h4>
    <p>
     在 Vue.js 中，
     <code>
      v-for
     </code>
     是用于
     <strong>
      循环渲染列表数据
     </strong>
     的核心指令，可以遍历数组、对象或数字，生成动态的 DOM 元素或组件。

     </strong>

     </a>

    </h3>
    <h4>
     <a id="vmodel_928">
     </a>
     v-model
    </h4>
    <p>
     <code>
      v-model
     </code>
     是 Vue.js 中用于实现
     <strong>
      表单输入元素与数据双向绑定
     </strong>
     的核心指令，它能将表单控件（如
     <code>
      &lt;input&gt;
     </code>
     、
     <code>
      &lt;textarea&gt;
     </code>
     、
     <code>
      &lt;select&gt;
     </code>
     ）的值与 Vue 实例的数据自动同步，无需手动操作 DOM。
    </p>
    <hr/>
    <h5>
     <a id="_934">
     </a>
     <strong>
      核心机制
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       双向绑定
      </strong>
      ：
      <br/>
      <code>
       v-model
      </code>
      是
      <code>
       v-bind
      </code>
      （数据到视图）和
      <code>
       v-on
      </code>
      （视图到数据）的语法糖。
      <ul>
       <li>
        <strong>
         数据 → 视图
        </strong>
        ：将数据绑定到表单元素的
        <code>
         value
        </code>
        /
        <code>
         checked
        </code>
        属性。
       </li>
       <li>
        <strong>
         视图 → 数据
        </strong>
        ：监听表单元素的
        <code>
         input
        </code>
        或
        <code>
         change
        </code>
        事件，更新数据。
       </li>
      </ul>
     </li>
    </ul>

    <h5>
     <a id="vmodel_1106">
     </a>
     v-model的原理拆解
    </h5>
    <p>
     Vue 的
     <code>
      v-model
     </code>
     是一个语法糖，用于实现表单元素和组件的
     <strong>
      双向数据绑定
     </strong>
     。其核心原理可以拆解为以下部分：
    </p>
    <hr/>
    <h6>
     <a id="1__1112">
     </a>
     <strong>
      1. 原生表单元素的双向绑定
     </strong>
    </h6>
    <p>
     对于原生表单元素（如
     <code>
      &lt;input&gt;
     </code>
     、
     <code>
      &lt;textarea&gt;
     </code>
     、
     <code>
      &lt;select&gt;
     </code>
     ），
     <code>
      v-model
     </code>
     会根据元素类型自动绑定对应的属性和事件，实现双向更新：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        文本输入（text、textarea）
       </strong>
       ：
      </p>
      <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>input</span> <span class="token attr-name">v-model</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>message<span class="token punctuation">"</span></span> <span class="token punctuation">/&gt;</span></span>

     <strong>
      4. 修饰符处理
     </strong>
    </h6>
    <p>
     <code>
      v-model
     </code>
     支持修饰符，用于对输入值进行处理：
    </p>
    <ul>
     <li>
      <strong>
       <code>
        .lazy
       </code>
      </strong>
      ：将
      <code>
       input
      </code>
      事件改为
      <code>
       change
      </code>
      事件（延迟同步）。
     </li>
     <li>
      <strong>
       <code>
        .trim
       </code>
      </strong>
      ：自动去除首尾空格。
     </li>
     <li>
      <strong>
       <code>
        .number
       </code>
      </strong>
      ：将输入转为数值类型。
     </li>
    </ul>
    <pre><code class="prism language-html"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>input</span> <span class="token attr-name">v-model.lazy.trim.number</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>value<span class="token punctuation">"</span></span> <span class="token punctuation">/&gt;</span></span>
