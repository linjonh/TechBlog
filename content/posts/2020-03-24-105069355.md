    <h2>
     <a id="_7">
     </a>
     数据字典
    </h2>
    <p>
     <strong>
      定义
     </strong>
     ：数据字典是描述数据的信息的集合，是对 系统中使用的所有数据元素/数据流图中包含的所有元素 的定义的集合。是为了描述在结构化分析过程中定义对象的内容时，使用的一种半形式化的工具。
     <br/>
     <br/>
    </p>
    <p>
     <strong>
      主要用途
     </strong>
     ：在软件分析和设计的过程中给人提供关于数据的描述信息。即，你可以查询任何在项目中你无法理解的各种数据，从而最大程度的消除 开发人员或不同开发小组之间的 歧义和交流不畅问题。
     <br/>
     <br/>
    </p>
    <p>
     <strong>
      举个例子
     </strong>
     ：E-R图大家都知道（不知道也要装作知道），它是用来表示实体及实体间关系的图。如“人有身份证”，“身份证”就是一个实体，“身份证”有"身份证号"这个属性。那么这时问题就来了，你如何定义"身份证号"？
     <strong>
      即：光有一个身份证号，你不给出身份证号 有几位、由字母还是数字组成、每段数字代表什么意思，你根本无法知道“身份证号”这个属性是个什么东西
     </strong>
     。而数据字典就是用来说明这些东西的（不仅限于此，因为如果你不对“人”等其他元素进行定义的话，你甚至都不知道“人”是什么东西）。
    </p>
    <blockquote>
     <p>
      数据字典和数据流图共同构成系统的逻辑模型。没有数据流图，数据字典难以发挥作用。没有数据字典，数据流图就不严格。只有数据流图和对数据流图中每个元素的精确定义放在一起，才能共同构成系统的规格说明书。所以说数据字典与数据流图是相互补充的。
     </p>
    </blockquote>
    <p>
     <br/>

    <h3>
     <a id="1_23">
     </a>
     1-数据字典的内容
    </h3>
    <p>
     因为数据字典需要对数据流图中所有的数据进行定义，所以
     <strong>
      为了方便定义，数据字典中把数据分为了4类
     </strong>
     。数据字典
     <strong>
      主要
     </strong>
     对系统中的以下四类元素的定义组成：
    </p>
    <ol>
     <li>
      数据流。
     </li>
     <li>
      数据元素：数据流由哪些元素组成。
     </li>
     <li>
      系统中的数据存储。
     </li>
     <li>
      系统的处理。
      <br/>
      <br/>
     </li>
    </ol>
    <h4>
     <a id="11_33">
     </a>
     1.1-数据流的描述
    </h4>
    <p>
     即，定义
     <strong>
      数据流
     </strong>
     时需要指明哪些东西。系统当中的每个数据流都需要在数据字典中做出相应描述。
    </p>
    <ul>
     <li>
      数据流名：数据流的名称。
     </li>
     <li>
      说明：简要介绍作用即它产生的原因和结果。
     </li>
     <li>
      数据流来源：即该数据流来自何方。
     </li>
     <li>
      数据流去向：去向何处。
     </li>
     <li>
      数据流组成：数据结构。
     </li>
     <li>
      每个数据量流通量（根据实际情况）：数据量、流通量。
      <br/>
      <br/>
     </li>
    </ul>
    <h4>
     <a id="12_45">
     </a>
     1.2-数据元素的描述
    </h4>
    <p>
     即，定义
     <strong>
      数据元素
     </strong>
     时需要指明哪些东西。
    </p>
    <ul>
     <li>
      数据元素名
     </li>
     <li>
      类型：数字（离散值、连续值），文字（编码类型）
     </li>
     <li>
      长度
     </li>
     <li>
      取值范围
     </li>
     <li>
      相关的数据元素及数据结构
      <br/>
      <br/>
     </li>
    </ul>
    <h4>
     <a id="13_56">
     </a>
     1.3-数据存储的描述
    </h4>
    <ul>
     <li>
      数据存储名
     </li>
     <li>
      简述：存放的是什么数据。
     </li>
     <li>
      输入数据
     </li>
     <li>
      输出数据
     </li>
     <li>
      数据文件组成：数据结构。
     </li>
     <li>
      存储方式：顺序，直接，关键码。
     </li>
     <li>
      存取频率
      <br/>
      <br/>
     </li>
    </ul>
    <h4>
     <a id="14_67">
     </a>
     1.4-处理的描述
    </h4>
    <p>
     能
     <strong>
      把输入数据流转变为输出数据流的机制
     </strong>
     ，我们称之为“处理”。
    </p>
    <ul>
     <li>
      处理名
     </li>
     <li>
      处理编号：反映该处理的层次
     </li>
     <li>
      简要描述：加工逻辑及功能简述
     </li>
     <li>
      输入数据流
     </li>
     <li>
      输出数据流
     </li>
     <li>
      加工逻辑：简述加工程序、加工顺序
      <br/>
      <br/>
      <br/>
      <br/>
     </li>
    </ul>
    <h3>
     <a id="2_80">
     </a>
     2-如何定义数据
    </h3>
    <h4>
     <a id="21_81">
     </a>
     2.1-为什么要定义数据
    </h4>
    <p>
     因为在一个系统当中，数据通常都是复合的——由很多简单的数据元素组成。那我们
     <strong>
      如何使用简单的元素去描述这些复杂的数据
     </strong>
     呢？数据定义解决的就是这个问题。
    </p>
    <p>
     定义数据采用的原则：自顶向下分解。
     <br/>
     数据元素组成数据的方式：
    </p>
    <ol>
     <li>
      顺序：（复合数据）以确定的次序连接两个或多个数据元素。
     </li>
     <li>
      选择：（复合数据）从两个或多个数据元素中选一个。
     </li>
     <li>
      重复：（复合数据）是 把指定的数据元素重复两次或多次 而组成的。
     </li>
     <li>
      可选：（在复合数据中）此数据元素是可有可无的。
      <br/>
      <br/>
     </li>
    </ol>
    <h4>
     <a id="22_93">
     </a>
     2.2-如何定义
    </h4>
    <p>
     由于文字会具有歧义，所以数据字典中同样采用符号的形式来定义这四类元素。有消除歧义、方便理解、清晰简洁等功效（滑稽）。
    </p>
