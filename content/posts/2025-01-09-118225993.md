    </h2>
    <h3>
     <a id="_1">
     </a>
     前言
    </h3>
    <p>
     目前网上大多数驱动OLED屏都采用软件IIC，因为HAL库的升级使得硬件IIC的稳定性得到了保障，所以想采用硬件IIC+DMA的方式控制OLED，在最初的时候，换用硬件IIC极其的方便，换到HAL库的接口函数就好了，但打开DMA就炸，仔细想了一下，知道原因
     <strong>
      （由于DMA为非阻塞模式，反复调用函数但硬件并未准备好，所以并未执行）
     </strong>
     经过拼包DMA发送,CPU的负担大幅降低，显示效果良好。
     <br/>
     下文从CubeMX的配置开始说起，并介绍两个DMA发送函数以及相应的发送完成回调函数，最后附上代码实体。（根据中景园历程修改的DMA发送版本）。
    </p>

    <h3>
     <a id="HAL_27">
     </a>
     HAL库函数
    </h3>
    <p>
     主要的DMA发送函数有两个
    </p>
    <pre><code class="prism language-c"><span class="token function">HAL_I2C_Mem_Write_DMA</span><span class="token punctuation">(</span>I2C_HandleTypeDef <span class="token operator">*</span>hi2c<span class="token punctuation">,</span> uint16_t DevAddress<span class="token punctuation">,</span> uint16_t MemAddress<span class="token punctuation">,</span>uint16_t MemAddSize<span class="token punctuation">,</span> uint8_t <span class="token operator">*</span>pData<span class="token punctuation">,</span> uint16_t Size<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token function">HAL_I2C_Master_Transmit_DMA</span><span class="token punctuation">(</span>I2C_HandleTypeDef <span class="token operator">*</span>hi2c<span class="token punctuation">,</span> uint16_t DevAddress<span class="token punctuation">,</span> uint8_t <span class="token operator">*</span>pData<span class="token punctuation">,</span>uint16_t Size<span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     两函数具有较为明显的区别，就是在
     <strong>
      HAL_I2C_Mem_Write_DMA
     </strong>
     函数中有
     <strong>
      MemAddress
     </strong>
     这一参数，
     <strong>
      MemAddress
     </strong>
     代表了传输到目标机的内存地址，而使用
     <strong>
      HAL_I2C_Master_Transmit_DMA
     </strong>
     函数需要在
     <strong>
      pData
     </strong>
     的第一位加入目标内存地址。
     <br/>
     这两个函数同时有着各自的完成回调函数分别是
     <br/>
     HAL_I2C_MemTxCpltCallback
     <br/>
     HAL_I2C_MasterTxCpltCallback
    </p>
    <p>
     因为对OLED操作需要先发送cmd发送坐标位置，在发送数据，这种方式刷新一次屏幕需要发送多次信息，所以为了保证信息都可以被接收，使用完成回调函数，发送下次的数据。
    </p>
