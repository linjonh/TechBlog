      <span style="color:#956fe7">
       ​6.7 ​​MDI（Medium Dependent Interface，媒介依赖接口）​
      </span>
     </a>
    </h6>

    <h2 id="%E5%89%8D%E8%A8%80" name="%E5%89%8D%E8%A8%80">
     前言
    </h2>
    <p>
     TCP/IP四层模型传输控制协议是网络通信的核心框架，包括
     <strong>
      数据链路层 / 网络接口层（链路层与物理层）​
     </strong>
     、
     <strong>
      网络层
     </strong>
     、
     <strong>
      传输层
     </strong>
     和
     <strong>
      应用层
     </strong>
     。这里将结合TCP/IP四层模型，深入探讨物理层（PHY）、链路层（MAC）、媒介独立层（MII）及其相关技术（如ODDR、IDDR、MDI、MAC、ARP、UDP等），帮助我们全面理解网络通信的底层实现及其在TCP/IP模型中的位置和作用。
     <span style="background-color:#ffd900">
      虽然上篇对TCP/IP四层模型传输控制协议有过分享，但是鉴于TCP/IP模型的广泛应用，这里对其进行详细介绍。
     </span>

     一.
     <strong>
      TCP/IP四层模型概述
     </strong>
    </h2>
    <h3 id="1.1%20%E6%A8%A1%E5%9E%8B%E5%88%86%E5%B1%82" name="1.1%20%E6%A8%A1%E5%9E%8B%E5%88%86%E5%B1%82">
     <strong>
      1.1 模型分层
     </strong>
    </h3>
    <p>
     <span style="background-color:#ffd900">
      TCP/IP四层模型是网络通信的分层架构，总共有四层，分别为
     </span>
     <strong>
      <span style="background-color:#ffd900">
       网络接口层、
      </span>
     </strong>
     <span style="background-color:#ffd900">
      <strong>
       网络层、传输层
      </strong>
      和
      <strong>
       应用层
      </strong>
      。
     </span>
     各层功能如下：
    </p>
    <blockquote>
     <ol>
      <li>
       <span style="color:#0d0016">
        <strong>
         ​网络接口层（物理层与链路层）​：
        </strong>
       </span>
       <ul>
        <li>
         <span style="color:#0d0016">
          ​
          <strong>
           物理层（PHY）​
          </strong>
          ：负责将数据转换为适合物理媒介传输的信号。
         </span>
        </li>
        <li>
         <span style="color:#0d0016">
          ​
          <strong>
           链路层（MAC）​
          </strong>
          ：负责数据帧的封装、解封装、错误检测和流量控制。
         </span>
        </li>
       </ul>
      </li>
      <li>
       <span style="color:#0d0016">
        ​
        <strong>
         网络层
        </strong>
        ：负责数据包的路由和转发，核心协议为IP（Internet Protocol）。
       </span>
      </li>
      <li>
       <span style="color:#0d0016">
        ​
        <strong>
         传输层
        </strong>
        ：负责端到端的数据传输，核心协议为TCP（Transmission Control Protocol）和UDP（User Datagram Protocol）。
       </span>
      </li>
      <li>
       <span style="color:#0d0016">
        ​
        <strong>
         应用层
        </strong>
        ：负责应用程序之间的通信，核心协议包括HTTP、FTP、DNS等。
       </span>
      </li>
     </ol>
    </blockquote>
    <p>
     ISO/OSI七层模型 与 TCP/IP四层模型对比：
    </p>
    <p class="img-center">
     <img alt="" height="500" src="https://i-blog.csdnimg.cn/direct/4dbe7f87a2614418805661f74af6a8c3.png" width="673"/>

     1.2
     <strong>
      <strong>
       媒介独立层
      </strong>
     </strong>
    </h3>
    <p>
     网络接口层 &gt; 媒介独立层
    </p>
    <p>
     <span style="background-color:#ffd900">
      在TCP/IP的模型中将 物理层和数据链路层 合称网络接口层。网络接口层一般分为一层和二层，一层又称为物理层，二层又称为数据链路层，在以太网中又称为MAC层。由于一层与二层之间的设计往往由不同的厂家实现，为了方便一层与二层交换数据和控制，因此又出现了
     </span>
     <strong>
      <span style="background-color:#ffd900">
       媒介独立层
      </span>
     </strong>
     <span style="background-color:#ffd900">
      或 媒介无关层
     </span>
     <strong>
      <span style="background-color:#ffd900">
       MII（Media Independent Interface）
      </span>
     </strong>
     ，媒介独立层与一层和二层之间的关系，如下图所示。由于在嵌入式系统 或 FPGA的编程中媒介独立层是经常要处理的一层，因此在讨论其它层之前先讨论该层。
    </p>
    <p class="img-center">
     <img alt="" height="336" src="https://i-blog.csdnimg.cn/direct/7e5c5b85cd4146c494a2171dc37db6a2.png" width="419"/>
    </p>
    <p class="img-center">
     <img alt="" height="178" src="https://i-blog.csdnimg.cn/direct/b7831b8210214539a2de0295e87210b3.png" width="419"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">

    <h2 id="%E4%BA%8C.%C2%A0%E7%BD%91%E7%BB%9C%E6%8E%A5%E5%8F%A3%E5%B1%82" name="%E4%BA%8C.%C2%A0%E7%BD%91%E7%BB%9C%E6%8E%A5%E5%8F%A3%E5%B1%82">
     二.
     <strong>
      网络接口层
     </strong>
    </h2>
    <p>
     网络接口层（ 物理层 PHY、链路层 MAC 与 媒介独立层 MII ）
    </p>
    <p>
     <img alt="" height="547" src="https://i-blog.csdnimg.cn/direct/423a67a32db84566b78cfe87caa45a15.png" width="682"/>
    </p>
    <h3 id="%E2%80%8B2.1%20%E7%89%A9%E7%90%86%E5%B1%82%EF%BC%88%20PHY%20%EF%BC%89%E2%80%8B" name="%E2%80%8B2.1%20%E7%89%A9%E7%90%86%E5%B1%82%EF%BC%88%20PHY%20%EF%BC%89%E2%80%8B">
     <span style="background-color:null">
      ​
     </span>
     <strong>
      <span style="background-color:null">
       2.1 物理层（ PHY ）​
      </span>
     </strong>
    </h3>
    <p>
     <span style="background-color:#ffd900">
      物理层是网络接口层的最底层，负责将MAC层的并行数据转换为适合物理媒介传输的串行信号。
     </span>
     PHY层通常分为三个子层：
    </p>
    <ul>
     <li>
      ​
      <strong>
       PCS子层（Physical Coding Sublayer）​
      </strong>
      ：负责信道编码，如4B/5B、8B/10B、64B/66B等。
     </li>
     <li>
      ​
      <strong>
       PMA子层（Physical Medium Attachment）​
      </strong>
      ：负责信号调制，如3阶/5阶编码、PAM调制。
     </li>
     <li>
      ​
      <strong>
       PMD子层（Physical Medium Dependent）​
      </strong>
      ：负责与物理媒介相关的信号传输，如MDI接口。
     </li>
    </ul>
    <h3>
     <strong>
      2.2 链路层（ MAC ）
     </strong>
    </h3>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       链路层
      </span>
     </strong>
     <span style="background-color:#ffd900">
      也叫做
     </span>
     <strong>
      <span style="background-color:#ffd900">
       数据链路层，
      </span>
     </strong>
     <span style="background-color:#ffd900">
      英文名称：
     </span>
     <strong>
      <span style="background-color:#ffd900">
       MAC。
      </span>
     </strong>
     <span style="background-color:#ffd900">
      链路层是网络接口层的重要组成部分，负责帧的封装、错误检测和流量控制。它通过MAC地址实现设备间的通信，并通过ARP协议解析IP地址到MAC地址的映射。
     </span>
    </p>
    <h6>
     <strong>
      2.3.1 MAC（Media Access Control）
     </strong>
    </h6>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       管理数据帧传输，确保设备通信。
      </span>
     </strong>
    </p>
    <p>
     <strong>
      具体功能：
     </strong>
    </p>
    <blockquote>
     <ol>
      <li>
       帧封装 ：将上层数据封装为以太网帧，包括源MAC地址、目标MAC地址和帧校验序列（FCS）。
      </li>
      <li>
       冲突检测 ：在半双工模式下，通过 CSMA/CD（载波侦听多路访问/冲突检测）机制避免数据冲突。
      </li>
      <li>
       流量控制 ：使用 PAUSE 帧实现流量控制，防止接收端缓冲区溢出。
      </li>
     </ol>
    </blockquote>
    <p>
     <strong>
      与 MII 层的关系：
     </strong>
    </p>
    <blockquote>
     <ol>
      <li>
       MAC 层通过 MII 接口与 PHY 层交互。
      </li>
      <li>
       MII 接口提供了标准化的信号定义（如 TXD、RXD），使得 MAC 和 PHY 可以无缝协作。
      </li>
     </ol>
    </blockquote>
    <p>
     <strong>
      应用场景：
     </strong>
    </p>
    <blockquote>
     <ol>
      <li>
       在嵌入式系统或 FPGA 设计中，MAC 模块通常由硬件逻辑实现。
      </li>
      <li>
       在现代交换机中，MAC 地址表用于管理设备间的通信。
      </li>
     </ol>
    </blockquote>
    <p>
     <strong>
      2.3.2 ARP（Address Resolution Protocol）
     </strong>
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       将IP地址解析为MAC地址，支持局域网通信。
      </span>
     </strong>
     <br/>
     <strong>
      工作原理 ：
     </strong>
    </p>
    <blockquote>
     <p>
      请求过程 ：主机发送一个 ARP 请求广播帧，询问目标 IP 地址对应的 MAC 地址。
      <br/>
      响应过程 ：目标设备收到 ARP 请求后，返回一个 ARP 响应帧，包含其 MAC 地址。
     </p>
    </blockquote>
    <p>
     <strong>
      数据包格式 ：
     </strong>
    </p>
    <blockquote>
     <ol>
      <li>
       硬件类型 ：指定网络类型（如以太网）。
      </li>
      <li>
       协议类型 ：指定协议类型（如 IPv4）。
      </li>
      <li>
       操作码 ：指示是 ARP 请求还是 ARP 响应。
      </li>
      <li>
       发送方 MAC 地址和 IP 地址 。
      </li>
      <li>
       目标 MAC 地址和 IP 地址 。
      </li>
     </ol>
    </blockquote>
    <p>
     <strong>
      应用场景 ：
     </strong>
    </p>
    <blockquote>
     <ol>
      <li>
       在局域网中，ARP 用于建立 IP 地址与 MAC 地址的映射关系。
      </li>
      <li>
       在路由器中，ARP 表用于加速数据包转发。
      </li>
     </ol>
    </blockquote>
    <h3 id="2.2%20%E5%AA%92%E4%BB%8B%E7%8B%AC%E7%AB%8B%E5%B1%82%EF%BC%88%20MII%20%EF%BC%89%E2%80%8B" name="2.2%20%E5%AA%92%E4%BB%8B%E7%8B%AC%E7%AB%8B%E5%B1%82%EF%BC%88%20MII%20%EF%BC%89%E2%80%8B">
     <strong>
      <span style="background-color:null">
       2.3 媒介独立层（ MII ）
      </span>
      ​
     </strong>
    </h3>
    <p>
     虽然上篇有过介绍，但是这里再来将其详细介绍
    </p>
    <p>
     <span style="background-color:#ffd900">
      MII层是连接MAC层和PHY层的桥梁，确保两者之间的数据传输和控制。随着网络技术的发展，MII接口衍生出多种类型，以适应不同的网络速率和场景。
     </span>
    </p>
    <p>
     常见的MII接口包括：
    </p>
    <blockquote>
     <ul>
      <li>
       ​
       <strong>
        MII
       </strong>
       ：用于10M/100M以太网。
      </li>
      <li>
       ​
       <strong>
        RMII
       </strong>
       ：MII的简化版本，引脚数量减少。
      </li>
      <li>
       ​
       <strong>
        GMII
       </strong>
       ：用于千兆以太网，数据位宽扩展到8位。
      </li>
      <li>
       ​
       <strong>
        RGMII
       </strong>
       ：GMII的简化版本，采用双数据速率（DDR）技术。
      </li>
     </ul>
    </blockquote>
    <h4 id="%E2%91%A0%20MII%EF%BC%88Media%20Independent%20Interface%EF%BC%89%E2%80%8B" name="%E2%91%A0%20MII%EF%BC%88Media%20Independent%20Interface%EF%BC%89%E2%80%8B">
     <strong>
      ① MII（Media Independent Interface）​
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     媒介无关层，MII接口主要用在10M/100M以太网 二层MAC 到一层Phy的接口。
    </p>
    <p class="img-center">
     <img alt="" height="272" src="https://i-blog.csdnimg.cn/direct/b7e38e0e85974a6c88fe76e760f96b01.png" width="367"/>
    </p>
    <p>
     <span style="color:#1a439c">
      <strong>
       MII接口信号（分类顺序）:
      </strong>
     </span>
    </p>
    <blockquote>
     <p>
      <strong>
       发送方向信号：
      </strong>
     </p>
     <p>
      ●
      <strong>
       <strong>
        TX_ER(Transmit Error)
       </strong>
      </strong>
      ：发送数据错误提示信号，同步于TX_CLK，高电平有效，表示TX_ER有效期内传输的数据无效。对于10Mbps速率下，TX_ER不起作用；
     </p>
     <p>
      ●
      <strong>
       TX_EN(Transmit Enable)
      </strong>
      ：发送数据使能信号，同步于TX_CLK，高电平有效，表示TXD[3:0]上的数据有效。
     </p>
     <p>
      ●
      <strong>
       <strong>
        TX_CLK
       </strong>
      </strong>
      ： 发送时钟，用Phy芯片产生，10M以太网为2.5MHz时钟,100M 以太网时钟为25MHz.方向由Phy（Physical）指向Host（主机）的MAC。
     </p>
     <p>
      ●
      <strong>
       <strong>
        TXD [3:0]:
       </strong>
      </strong>
      发送的数据，与TX_CLK 同步，表示mac 层发送数据到phy层（对于MAC层的8位数据，TXD遵循先发
      <strong>
       <strong>
        低4位再发高4位
       </strong>
      </strong>
      的原则）。
     </p>
     <p>
      <br/>
      <strong>
       接收方向信号：
      </strong>
     </p>
     <p>
      ●
      <strong>
       <strong>
        RX_ER(Receive Error)
       </strong>
      </strong>
      ： 接收数据错误提示信号，同步于RX_CLK，高电平有效，表示RX_ER有效期内传输的数据无效。对于10Mbps速率下，RX_ER不起作用
      <span style="color:#494949">
       ；（一般发生在转发过程，一旦信号拉高，整个数据包都不能用了）。
      </span>
     </p>
     <p>
      ● ​
      <strong>
       RX_DV(Receive Data Valid)
      </strong>
      ：接收数据有效信号，同步于RX_CLK，高电平有效，表示RXD[3:0]上的数据有效。
     </p>
     <p>
      ●
      <strong>
       <strong>
        RX_CLK
       </strong>
      </strong>
      :  MAC 层的接收时钟，一般由phy产生，mac 层使用这个时钟。
     </p>
     <p>
      ●
      <strong>
       <strong>
        RXD[3:0]
       </strong>
      </strong>
      ：接收数据，与RX_CLK同步，表示由phy层发送数据到mac层。
     </p>
     <p>
      <br/>
      <strong>
       控制信号：
      </strong>
     </p>
     <p>
     </p>
     <p style="margin-left:.0001pt; margin-right:0; text-align:left">
      ●
      <strong>
       <strong>
        CRS
       </strong>
      </strong>

     </p>

     </p>
     <p style="margin-left:.0001pt; margin-right:0; text-align:left">
     </p>
     <p style="margin-left:.0001pt; margin-right:0; text-align:left">
      <span style="color:#0d0016">

    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:left">

      </span>
     </strong>
     <span style="color:#1a439c">
      <strong>
       图片顺序
      </strong>
     </span>
     <strong>
      <span style="color:#1a439c">
       ）：
      </span>
     </strong>
    </p>
    <blockquote>
     <p style="margin-left:.0001pt; margin-right:0; text-align:left">
      ●
      <strong>
       <strong>
        TX_EN：
       </strong>
      </strong>
      发送数据使能，高电平有效，只有TX_EN 为高电平时发送的数据才是有效的。在数据发送期间TX_EN 可以为低电平，此时为等待期。
     </p>

      <strong>
       <strong>
        TXD[1:0]：
       </strong>
      </strong>
      发送数据，在RMII接口中发送数据每次只要两位，对于8位数据，发送顺序是从低到高的原则，如数据a[7:0], TXD 按照a[1:0]–&gt;a[3:2]–&gt;a[5:4]–&gt;a[7:6]的顺序发送。数据发送与TX_EN配合，只有TX_EN为高电平时，数据才是有效的。

      </strong>

     </p>

      ●
      <strong>
       <strong>
        RXD[1:0]：
       </strong>
      </strong>
      每次只接收两位，也是遵循从低位到高位的原则，与CRS_DV配合完成数据接收。
     </p>

       </strong>
      </strong>

    </h4>
    <p>

    </p>

    <h4 id="%E2%91%A4%20%E5%85%B6%E4%BB%96MII%E6%8E%A5%E5%8F%A3" name="%E2%91%A4%20%E5%85%B6%E4%BB%96MII%E6%8E%A5%E5%8F%A3">
     ⑤ 其他MII接口
    </h4>
    <p>
     <span style="background-color:#ffffff">
      在MII的接口中，还有其它的接口如SGMII，XGMII，SFP SGMII等类型，
     </span>
     <span style="background-color:#ffffff">
      主要用于光纤通信中。
     </span>
    </p>
    <ul>
     <li>
      ​
      <strong>
       SGMII（Serial GMII）​
      </strong>
      ：用于光纤通信，数据通过串行方式传输。
     </li>
     <li>
      ​
      <strong>
       XGMII（10 Gigabit MII）​
      </strong>
      ：用于10G以太网。
     </li>
     <li>
      ​
      <strong>
       SFP SGMII
      </strong>
      ：用于SFP模块的光纤通信。
     </li>
    </ul>
    <p>
     <span style="background-color:#ffd900">
      物理层（PHY）和媒介独立层（MII）是网络通信的基础技术。物理层PHY通过信道编码和信号处理确保数据传输的可靠性，而媒介独立层MII则通过标准化接口实现MAC层和PHY层之间的高效通信。理解这些技术对于网络设计和优化具有重要意义。
     </span>
    </p>
    <p>
     <span style="background-color:#ffd900">
      网络接口层作为TCP/IP模型的最底层，涵盖了物理层（PHY）和链路层（数据链路层，MAC）的核心功能。通过物理层的编码、调制和信号处理，以及媒介独立层（MII）的标准化接口设计，确保了数据能够在物理媒介上高效、可靠地传输。同时，链路层通过MAC地址管理和ARP协议解析，实现了设备间的通信和数据包的正确传递。
     </span>
    </p>
    <p>
    </p>

    <h2 id="%E2%80%8B%E4%B8%89.%20%E7%BD%91%E7%BB%9C%E5%B1%82" name="%E2%80%8B%E4%B8%89.%20%E7%BD%91%E7%BB%9C%E5%B1%82">
     ​
     <strong>
      三. 网络层
     </strong>
    </h2>
    <p>
     网络层：ARP与IP
    </p>
    <h4 id="%E2%80%8B3.1%20%E2%80%8BARP%EF%BC%88Address%20Resolution%20Protocol%EF%BC%8C%E5%9C%B0%E5%9D%80%E8%A7%A3%E6%9E%90%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B" name="%E2%80%8B3.1%20%E2%80%8BARP%EF%BC%88Address%20Resolution%20Protocol%EF%BC%8C%E5%9C%B0%E5%9D%80%E8%A7%A3%E6%9E%90%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B">
     ​
     <strong>
      3.1
     </strong>
     ​
     <strong>
      ARP（Address Resolution Protocol，地址解析协议）​
     </strong>
    </h4>
    <p>
     ARP是网络接口层的协议，用于将IP地址解析为MAC地址，以便在局域网中传输数据。当设备需要发送数据时，首先检查ARP缓存中是否存在目标IP地址对应的MAC地址。如果没有，设备会广播一个ARP请求，询问“谁拥有这个IP地址？”。目标设备收到ARP请求后，会回复一个ARP响应，包含其MAC地址。发送方将IP地址和MAC地址的映射存储在ARP缓存中，以便后续通信时直接使用。ARP协议是局域网通信的基础，确保数据能够准确传输到目标设备。
    </p>
    <p>
     <span style="background-color:#ffd900">
      ARP协议用于将IP地址解析为MAC地址，以便在局域网中传输数据。它解决了网络层（IP地址）与数据链路层（MAC地址）之间的地址映射问题。
     </span>
     <strong>
      <span style="background-color:#ffd900">
       将IP地址解析为MAC地址，支持局域网通信。
      </span>
     </strong>
    </p>
    <p>
     ​
     <strong>
      在TCP/IP模型中的位置
     </strong>
     ：虽然ARP协议在功能上属于网络接口层，但它主要为网络层提供服务，帮助实现IP地址到MAC地址的转换。
    </p>
    <p>
     ​
     <strong>
      工作流程
     </strong>
     ：
    </p>
    <blockquote>
     <ol>
      <li>
       ​
       <strong>
        ARP请求
       </strong>
       ：当设备需要发送数据时，首先检查ARP缓存中是否存在目标IP地址对应的MAC地址。如果没有，设备会广播一个ARP请求，询问“谁拥有这个IP地址？”。
      </li>
      <li>
       ​
       <strong>
        ARP响应
       </strong>
       ：目标设备收到ARP请求后，会回复一个ARP响应，包含其MAC地址。
      </li>
      <li>
       ​
       <strong>
        ARP缓存
       </strong>
       ：发送方将IP地址和MAC地址的映射存储在ARP缓存中，以便后续通信时直接使用。
      </li>
     </ol>
    </blockquote>
    <p>
     <strong>
      应用场景
     </strong>
     ：ARP协议广泛应用于局域网（LAN）中，特别是在以太网环境中，确保数据能够准确传输到目标设备。
    </p>
    <h4 id="%E2%80%8B3.2%20%E2%80%8BIP%EF%BC%88Internet%20Protocol%EF%BC%8C%E7%BD%91%E9%99%85%E7%BD%91%E7%BB%9C%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B" name="%E2%80%8B3.2%20%E2%80%8BIP%EF%BC%88Internet%20Protocol%EF%BC%8C%E7%BD%91%E9%99%85%E7%BD%91%E7%BB%9C%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B">
     ​
     <strong>
      3.2
     </strong>
     ​
     <strong>
      IP（Internet Protocol，网际网络协议）​
     </strong>
    </h4>
    <p>

    </p>

     <strong>
      4.1 TCP（Transmission Control Protocol，传输控制协议）
     </strong>
    </h4>
    <p>
     TCP（Transmission Control Protocol，传输控制协议）​​ 是一种面向连接、可靠的传输层协议，主要用于确保数据在网络上准确、有序地传输。它通过三次握手建立连接，并通过确认机制、重传机制和错误检测来保证数据的完整性和顺序。TCP还支持流量控制和拥塞控制，能够动态调整数据传输速率，避免网络拥塞和接收方缓冲区溢出。尽管TCP的开销较大，延迟较高，但其高可靠性使其成为Web浏览、文件传输、电子邮件等需要可靠数据传输的应用场景的首选协议。

     </strong>

    </p>

     <strong>
      MAC（Media Access Control，媒体访问控制）
     </strong>
    </h4>
    <p>
     在以太网中，MAC层（介质访问控制层）作为数据链路层的核心组件，负责数据帧的封装、解封装、错误检测和流量控制。它不仅管理数据帧的发送和接收，还通过MII接口与PHY层（物理层）紧密协作，确保数据能够正确地在局域网中传输。简而言之，MAC层处理数据帧的各项操作，并与物理层配合实现数据的有效传输。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       管理数据帧传输，确保设备通信。
      </span>

    <h4 id="%E2%80%8B6.2%C2%A0ARP%EF%BC%88Address%20Resolution%20Protocol%EF%BC%8C%E5%9C%B0%E5%9D%80%E8%A7%A3%E6%9E%90%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B" name="%E2%80%8B6.2%C2%A0ARP%EF%BC%88Address%20Resolution%20Protocol%EF%BC%8C%E5%9C%B0%E5%9D%80%E8%A7%A3%E6%9E%90%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B">

     </strong>
    </h4>
    <p>
     在局域网中，ARP（地址解析协议）用于将IP地址解析为对应的MAC地址，以实现数据的传输。ARP通过广播方式查询目标设备的MAC地址，并将解析得到的结果存储于ARP缓存中。这样，当需要向同一网络中的目标设备发送数据时，ARP协议可以确保正确地定位目标设备的物理地址。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       将IP地址解析为MAC地址，支持局域网通信。
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       ARP
      </span>
     </strong>
     <span style="background-color:#ffd900">
      用于将IP地址映射到对应的MAC物理地址，这是因为在局域网内部通信时需要知道目标设备的MAC地址。ARP协议通过广播查询请求，获取目标设备的MAC地址，并将这个信息存储在本地的ARP缓存中以便后续快速访问。
     </span>
    </p>

     <strong>

     </strong>
     ​
     <strong>

    <p>
     <span style="color:#494949">
      在RGMII接口中，ODDR用于将8位数据分为高低4位，分别在时钟的上升沿和下降沿传输。它是一种在FPGA中实现双数据速率输出的
      <span style="background-color:null">
       技术，用于在时钟的上升沿和下降沿分别输出数据。
      </span>
     </span>
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       在时钟上升沿和下降沿输出数据，提高传输效率。
      </span>
     </strong>
    </p>
    <h4 id="%E2%80%8B6.4%C2%A0%E2%80%8BIDDR%EF%BC%88Input%20Double%20Data%20Rate%EF%BC%8C%E8%BE%93%E5%85%A5%E5%8F%8C%E6%95%B0%E6%8D%AE%E9%80%9F%E7%8E%87%EF%BC%89%E2%80%8B%E2%80%8B" name="%E2%80%8B6.4%C2%A0%E2%80%8BIDDR%EF%BC%88Input%20Double%20Data%20Rate%EF%BC%8C%E8%BE%93%E5%85%A5%E5%8F%8C%E6%95%B0%E6%8D%AE%E9%80%9F%E7%8E%87%EF%BC%89%E2%80%8B%E2%80%8B">
     ​6
     <strong>
      .4
     </strong>
     ​
     <strong>
      IDDR（Input Double Data Rate，输入双数据速率）​​
     </strong>
    </h4>
    <p>
     <span style="color:#494949">
      在RGMII接口中，IDDR用于将接收到的4位数据分别在时钟的上升沿和下降沿捕获，并合并为8位数据。它是一种在FPGA中实现双数据速率输入的技术
      <span style="background-color:null">
       ，用于在时钟的上升沿和下降沿分别捕获数据。
      </span>
     </span>
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       在时钟上升沿和下降沿捕获数据，提高接收效率。
      </span>
     </strong>
    </p>
    <p>
    </p>
    <h4 id="6.5%20%E2%80%8BDDR%EF%BC%88Double%20Data%20Rate%EF%BC%8C%E5%8F%8C%E6%95%B0%E6%8D%AE%E9%80%9F%E7%8E%87%EF%BC%89%E2%80%8B" name="6.5%20%E2%80%8BDDR%EF%BC%88Double%20Data%20Rate%EF%BC%8C%E5%8F%8C%E6%95%B0%E6%8D%AE%E9%80%9F%E7%8E%87%EF%BC%89%E2%80%8B">
     6.5 ​
     <strong>
      DDR（Double Data Rate，双数据速率）​
     </strong>
    </h4>
    <p>
     DDR是一种高速数据传输技术，通过在时钟的上升沿和下降沿分别传输数据，将数据传输速率提高一倍。DDR技术广泛应用于内存（如DDR SDRAM）和高速接口（如RGMII）中，以提高数据传输效率。在FPGA设计中，DDR技术通过ODDR（输出双数据速率）和IDDR（输入双数据速率）实现，分别用于双数据速率的发送和接收。DDR技术的核心优势是提高带宽和传输速率，适用于对性能要求较高的场景。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       通过时钟双沿传输数据，提高传输效率。
      </span>
     </strong>
    </p>
    <p>
     <span style="background-color:#ffd900">
      在
     </span>
     <strong>
      <span style="background-color:#ffd900">
       RGMII接口
      </span>
     </strong>
     <span style="background-color:#ffd900">
      中，
     </span>
     <strong>
      <span style="background-color:#ffd900">
       ODDR
      </span>
     </strong>
     <span style="background-color:#ffd900">
      和
     </span>
     <strong>
      <span style="background-color:#ffd900">
       IDDR
      </span>
     </strong>
     <span style="background-color:#ffd900">
      技术主要用于处理双倍数据速率
     </span>
     <span style="color:#fe2c24">
      <strong>
       <span style="background-color:#ffd900">
        DDR
       </span>
      </strong>
     </span>
     <span style="background-color:#ffd900">
      （Double Data Rate）
     </span>
     <span style="color:#fe2c24">
      <strong>
       <span style="background-color:#ffd900">
       </span>
      </strong>
     </span>
     <span style="background-color:#ffd900">
      信号。ODDR用于在时钟的上升沿和下降沿分别输出数据，而IDDR则用于在同一时钟周期内捕获两次数据。这里的DDR是基础概念，而ODDR和IDDR则是其具体应用形式，分别针对数据输出和输入优化，通过双边沿采样技术实现了更高效的数据传输速率。
     </span>
    </p>
    <p>
     <span style="background-color:#ffd900">
      ODDR和IDDR技术被用来实现高效的4位宽度数据传输。具体来说，8位宽的数据通过ODDR技术被分割成高低4位，在时钟的上升沿和下降沿分别发送出去；接收端则使用IDDR技术在时钟的两个边沿捕获4位数据，并将其合并为8位数据。
     </span>
    </p>
    <p>
    </p>
    <h4 id="%E2%80%8B6.6%20%E2%80%8BUDP%EF%BC%88User%20Datagram%20Protocol%EF%BC%8C%E7%94%A8%E6%88%B7%E6%95%B0%E6%8D%AE%E6%8A%A5%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B" name="%E2%80%8B6.6%20%E2%80%8BUDP%EF%BC%88User%20Datagram%20Protocol%EF%BC%8C%E7%94%A8%E6%88%B7%E6%95%B0%E6%8D%AE%E6%8A%A5%E5%8D%8F%E8%AE%AE%EF%BC%89%E2%80%8B">
     ​6
     <strong>
      .6
     </strong>
     ​
     <strong>
      UDP（User Datagram Protocol，用户数据报协议）​
     </strong>
    </h4>
    <p>
     UDP是一种无连接、不可靠的传输层协议，主要用于高效传输数据。它不建立连接，直接将数据封装成数据报发送，适用于对实时性要求较高的场景，如视频流、语音通信、在线游戏等。UDP协议头开销小，传输效率高，但由于缺乏可靠性机制，可能会出现数据丢失、重复或乱序的情况。尽管不可靠，UDP在广播、多播和简单查询（如DNS、DHCP）等场景中表现出色，成为实时应用和轻量级通信的首选协议。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       UDP协议
      </span>
     </strong>
     <span style="background-color:#ffd900">
      提供了一种灵活的数据传输方式，特别适合那些对实时性有较高要求的应用。
     </span>
     <strong>
      <span style="background-color:#ffd900">
       快速传输数据，但不可靠。
      </span>
     </strong>
    </p>
    <p>
    </p>
    <h4 id="%E2%80%8B6.7%C2%A0%E2%80%8B%E2%80%8BMDI%EF%BC%88Medium%20Dependent%20Interface%EF%BC%8C%E5%AA%92%E4%BB%8B%E4%BE%9D%E8%B5%96%E6%8E%A5%E5%8F%A3%EF%BC%89%E2%80%8B" name="%E2%80%8B6.7%C2%A0%E2%80%8B%E2%80%8BMDI%EF%BC%88Medium%20Dependent%20Interface%EF%BC%8C%E5%AA%92%E4%BB%8B%E4%BE%9D%E8%B5%96%E6%8E%A5%E5%8F%A3%EF%BC%89%E2%80%8B">
     ​6
     <strong>
      .7 ​
     </strong>
     ​
     <strong>
      MDI（Medium Dependent Interface，媒介依赖接口）​
     </strong>
    </h4>
    <p>
     MDI是物理层（PHY）与物理媒介（如网线、光纤）之间的接口，负责将PHY芯片生成的信号转换为适合物理媒介传输的形式。MDI接口的主要功能是实现信号的电平转换和阻抗匹配，确保信号在物理媒介上能够高效、可靠地传输。常见的MDI接口包括以太网中的RJ-45接口，用于连接网线和网络设备。MDI接口的设计和实现直接影响网络的传输性能和稳定性，是物理层与物理媒介之间的关键桥梁。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       MDI
      </span>
     </strong>
     <span style="background-color:#ffd900">
      确保信号能够在物理媒介上高效、稳定地传播（FPGA 31 中也有提到）。
     </span>
     <strong>
      <span style="background-color:#ffd900">
       连接PHY与物理媒介，确保信号传输。
      </span>
     </strong>
    </p>
    <p>
    </p>
    <h4 id="6.8%20SOC%EF%BC%88System%20on%20Chip%EF%BC%8C%E7%89%87%E4%B8%8A%E7%B3%BB%E7%BB%9F%EF%BC%89%E2%80%8B" name="6.8%20SOC%EF%BC%88System%20on%20Chip%EF%BC%8C%E7%89%87%E4%B8%8A%E7%B3%BB%E7%BB%9F%EF%BC%89%E2%80%8B">
     <strong>
      6.8 SOC（System on Chip，片上系统）​
     </strong>
    </h4>
    <p>
     SOC是一种将处理器、内存、外设和其他功能模块集成在单一芯片上的技术。它通过高度集成，减少芯片面积、功耗和成本，同时提高系统性能和可靠性。SOC广泛应用于嵌入式系统、移动设备（如智能手机、平板电脑）和物联网设备中。SOC的核心优势是高度集成和定制化设计，能够满足特定应用场景的需求。SOC设计通常包括硬件描述、软件开发和系统验证等环节，是现代电子系统设计的核心技术之一。
    </p>
    <p>
     FPGA（现场可编程门阵列）和SoC（系统级芯片）是两种不同的集成电路技术，但它们之间确实存在一定的关系，并且在某些情况下可以结合使用以形成更强大的解决方案。
    </p>
    <p>
     <strong>
      <span style="background-color:#ffd900">
       将多种功能集成在单一芯片上，实现高效、低功耗的系统设计。
      </span>
     </strong>
    </p>
    <p>
    </p>

     七. 本文总结
    </h2>
    <p>
     在TCP/IP模型中，
     <strong>
      网络接口层
     </strong>
     将
     <strong>
      物理层
     </strong>
     和
     <strong>
      数据链路层
     </strong>
     合并为一个整体，负责将数据转换为适合物理媒介传输的信号，并确保数据帧的正确传输。
     <strong>
      媒介独立层（MII）​
     </strong>
     作为连接
     <strong>
      物理层PHY
     </strong>
     和​
     <strong>
      链路层MAC
     </strong>
     的桥梁，通过标准化的接口信号和灵活的速率适配，确保两者之间的高效通信。理解MII层的作用及其在嵌入式系统和FPGA编程中的重要性，对于网络设计和优化具有重要意义。
    </p>
