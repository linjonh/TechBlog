    <p>
     想要训练一个好用的机器学习模型该怎么办？
    </p>
    <center>
     <b>
      标注数据去吧
     </b>
    </center>
    <p>
     是呀，数据不行，模型怎么能行？可是面对成千上万的数据只能看模型而叹气了。
     <br/>
     有钱一起花，有数据当然一起标了，赶紧做一个简单的数据标注系统，发动组里的人一起标数据吧！
    </p>
    <h3>
     <a id="_8">
     </a>
     基本构造
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/58086907a351630b0ef3b5a796b4f9b1.png">
      <br/>
      还是传统的构造方法，有显示，有控制，还有数据库
     </img>
    </p>
    <h5>
     <a id="_11">
     </a>
     前端页面
    </h5>
    <p>
     用于展现你要标注的数据，提交标注的结果(是，否，疑惑)的按钮，标注规则
    </p>
    <h5>
     <a id="_13">
     </a>
     后端控制
    </h5>
    <p>
     处理前端的标注结果并让前端发过去需要标注的数据
    </p>
    <h5>
     <a id="_15">
     </a>
     数据库
    </h5>
    <p>
     存放这要标记的数据，并且有这条数据的id 号，数据内容，标注结果，标注状态
    </p>
    <h3>
     <a id="_18">
     </a>
     实现工具
    </h3>
    <ul>
     <li>
      前端：HTML, jQuery，bootstrap
     </li>
     <li>
      后端 : python flask
     </li>
     <li>
      数据库：mysql
     </li>
    </ul>
    <p>
     前端基本样式：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/783179438954505daa7812db6a038af4.png">
      <br/>
      主要写的内容就是js
      <br/>
      首先在加载的页面的时候需要让后台访问一次数据库，并把需要标注的数据返回
      <br/>
      点击对应的按钮，给后台发送对应的数据，然后后台根据前端的指示进行数据库的操作并返回数据
     </img>
    </p>
    <p>
     后台的功能主要有两个
    </p>
    <ul>
     <li>
      1 用于页面初始化的查询操作
     </li>
     <li>
      2 用于点击按钮的处理操作
     </li>
    </ul>
    <p>
     查询操作是数据库的select操作，而处理操作就是先把前端做的处理对数据库进行更新，然后查询一条没有标注的数据进行返回。
    </p>
    <p>
     数据库的设计只需要有一个id主键，数据内容，标注的结果，还有对应的标注状态这么几条。
    </p>
    <p>
     开始标注数据吧！
    </p>
    <p>
     实现快，疗效好，这里就不放代码了。如果有人感兴趣可以留言，我给你们进行单独的介绍
