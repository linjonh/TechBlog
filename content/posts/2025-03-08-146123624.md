    <hr/>
    <h4>
     <a id="ArrayList__Vector__8">
     </a>
     ArrayList 与 Vector 的区别详解
    </h4>
    <p>
     <strong>
      ArrayList
     </strong>
     和
     <strong>
      Vector
     </strong>
     都是 Java 中基于动态数组实现的
     <code>
      List
     </code>
     接口的实现类，但它们在设计、性能和线程安全性上有显著差异。以下是两者的核心区别：
    </p>
    <hr/>
    <h5>
     <a id="1__14">
     </a>
     <strong>
      1. 线程安全性
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         特性
        </strong>
       </th>
       <th>
        <strong>
         ArrayList
        </strong>
       </th>
       <th>
        <strong>
         Vector
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         线程安全
        </strong>
       </td>
       <td>
        <strong>
         非线程安全
        </strong>
        （方法未同步）
       </td>
       <td>
        <strong>
         线程安全
        </strong>
        （方法通过
        <code>
         synchronized
        </code>
        同步）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        单线程环境，性能优先
       </td>
       <td>
        多线程环境（但实际开发中不推荐）
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      示例对比
     </strong>
     ：

     <a id="2__33">
     </a>
     <strong>
      2. 扩容机制
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         特性
        </strong>
       </th>
       <th>
        <strong>
         ArrayList

        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         默认容量
        </strong>
       </td>
       <td>
        10
       </td>
       <td>
        10
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         扩容公式
        </strong>
       </td>

        <code>
         newCapacity = oldCapacity + (oldCapacity &gt;&gt; 1)

        </strong>

        <code>
         newCapacity = oldCapacity * 2

        </strong>

       <td>
        内存利用率较高，但扩容次数可能较多
       </td>
       <td>
        扩容次数较少，但内存浪费较多
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      代码示例
     </strong>

    </p>
    <pre><code class="prism language-java"><span class="token comment">// ArrayList 扩容逻辑</span>

    <h4>
     <a id="_112">
     </a>
     <strong>
      总结与选型建议
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       单线程环境
      </strong>
      ：
      <ul>
       <li>
        优先使用
        <strong>
         ArrayList
        </strong>
        （性能更高，内存更优）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       多线程环境
      </strong>
      ：
      <ul>
       <li>
        <strong>
         避免使用 Vector
        </strong>
        （性能差，锁粒度粗）。
       </li>
       <li>
        替代方案：
        <ul>
         <li>
          <strong>
           <code>
            Collections.synchronizedList(new ArrayList&lt;&gt;())
           </code>
          </strong>
          （灵活控制同步范围）。
         </li>
         <li>
          <strong>
           <code>
            CopyOnWriteArrayList
           </code>
          </strong>
          （读多写少场景）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       历史代码兼容
      </strong>
      ：
      <ul>
       <li>
        仅在维护旧系统时使用 Vector。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_Vector__125">
     </a>
     <strong>
      为什么 Vector 已被弃用？
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       锁粒度过大
      </strong>
      ：所有方法同步导致高并发时性能差。
     </li>
     <li>
      <strong>
       功能冗余
      </strong>
      ：现代集合框架（如
      <code>
       java.util.concurrent
      </code>
      ）提供更高效的线程安全类。
     </li>
     <li>
      <strong>
       设计陈旧
      </strong>
      ：未充分利用现代 JVM 优化（如 CAS 操作）。
     </li>
    </ul>
    <hr/>

     </strong>
