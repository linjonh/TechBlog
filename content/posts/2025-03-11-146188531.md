    <h2>
     <a id="_0">
     </a>
     简介
    </h2>
    <p>
     TDengine ODBC 是为 TDengine 实现的 ODBC 驱动程序，支持 Windows 系统的应用（如
     <a href="https://powerbi.microsoft.com/zh-cn/" rel="nofollow">
      PowerBI
     </a>
     等）以及用户自定义开发的应用程序，通过 ODBC 标准接口访问本地、远程和云服务的 TDengine 数据库。
    </p>
    <p>
     TDengine ODBC 提供基于 WebSocket（推荐）和 原生连接两种方式连接 TDengine 数据库，使用时可以为 TDengine 数据源设置不同的连接方式。访问云服务时必须使用 WebSocket 连接方式。
    </p>
    <p>
     TDengine ODBC 提供 64 位 和 32 位 两种驱动程序。但 32 位仅企业版支持，且仅支持 WebSocket 连接方式。
    </p>
    <p>
     <strong>
      注意：
     </strong>
    </p>
    <ul>
     <li>
      驱动管理器：确保使用与应用程序架构匹配的 ODBC 驱动管理器。32 位应用程序需要使用 32 位 ODBC 驱动管理器，64 位应用程序需要使用 64 位 ODBC 驱动管理器。
     </li>
     <li>
      数据源名称（DSN）：32 位和 64 位 ODBC 驱动管理器都可以看到所有 DSN，用户 DSN 标签页下的 DSN 如果名字相同会共用，因此需要在 DSN 名称上去区分。
     </li>
    </ul>
    <p>
     想更多了解 TDengine 时序时序数据库的使用，可访问
     <a href="https://docs.taosdata.com/intro/" rel="nofollow">
      TDengine官方文档
     </a>
     。
    </p>
    <h3>
     <a id="ODBC__14">
     </a>
     ODBC 版本兼容性
    </h3>
    <p>
     支持 ODBC 所有版本
    </p>
    <h3>
     <a id="_18">
     </a>
     安装
    </h3>
    <ol>
     <li>
      <p>
       仅支持 Windows 平台。Windows 上需要安装过 VC 运行时库，可在此下载安装
       <a href="https://learn.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist?view=msvc-170" rel="nofollow">
        VC运行时库
       </a>
       如果已经安装VS开发工具可忽略。
      </p>
     </li>
     <li>
      <p>
       安装 TDengine Windows 客户端。版本在 3.2.1.0 或以上，包含 TDengine 的 ODBC 64 位驱动；版本在 3.3.3.0 或以上，包含 TDengine 的 ODBC 32/64 位驱动。
      </p>
     </li>
    </ol>
    <h3>
     <a id="_24">
     </a>
     配置数据源
    </h3>
    <h4>
     <a id="_26">
     </a>
     数据源连接类型与区别
    </h4>
    <p>
     TDengine ODBC 支持两种连接 TDengine 数据库方式：WebSocket 连接与 Native 连接，其区别如下：
    </p>
    <ol>
     <li>
      <p>
       访问云服务仅支持使用 WebSocket 连接方式。
      </p>
     </li>
     <li>
      <p>
       32 位应用程序仅支持使用 WebSocket 连接方式。
      </p>
     </li>
     <li>
      <p>
       WebSocket 连接的兼容性更好，一般不需要随着 TDengine 数据库服务端升级而升级客户端的库。
      </p>
     </li>
     <li>
      <p>
       Native 连接通常性能更好一点，但是必须与 TDengine 数据库服务端的版本保持一致。
      </p>
     </li>
     <li>
      <p>
       对于一般用户，建议使用
       <strong>
        WebSocket
       </strong>
       连接方式，性能与 Native 差别不大，兼容性更好。
      </p>
     </li>
    </ol>
    <h4>
     <a id="WebSocket__40">
     </a>
     WebSocket 连接
    </h4>
    <ol>
     <li>
      <p>
       【开始】菜单搜索打开【ODBC 数据源(32 位)】或者【ODBC 数据源(64 位)】管理工具
      </p>
     </li>
     <li>
      <p>
       选中【用户 DSN】标签页，通过【添加(D)】按钮进入"创建数据源"界面
      </p>
     </li>
     <li>
      <p>
       选择想要添加的数据源，这里我们选择【TDengine】
      </p>
     </li>
     <li>
      <p>
       点击完成，进入 TDengine ODBC 数据源配置页面，填写如下必要信息
      </p>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cb7c4905d220465eba7880b26655342e.png"/>
    </p>
    <pre><code>4.1 【DSN】：Data Source Name 必填，为新添加的 ODBC 数据源命名

4.2【连接类型】：必选，选择连接类型，这里选择 【WebSocket】

4.3【URL】必填，ODBC 数据源 URL，示例：`http://localhost:6041`， 云服务的 url 示例：`https://gw.cloud.taosdata.com?token=your_token`

4.4【数据库】选填，需要连接的默认数据库

4.5【用户名】仅供第5步测试连接使用，选填，数据库用户名，如果不填，TDengine 默认 root

4.6【密码】仅供第5步测试连接使用，选填，数据库用户密码，如果不填，TDengine 默认 taosdata

4.7【兼容软件】支持对工业软件 KingSCADA、Kepware 等的兼容性适配，通常情况下，选择默认值 General 即可
</code></pre>
    <ol start="5">
     <li>
      <p>
       点【测试连接】测试连接情况，如果成功，提示"成功连接到URL"
      </p>
     </li>
     <li>
      <p>
       点【确定】，即可保存配置并退出
      </p>
     </li>
     <li>
      <p>
       也可以在第2步选择已经配置好的数据源名通过【配置】按钮进入配置页面，修改已有配置
      </p>
     </li>
    </ol>
    <h4>
     <a id="_32__73">
     </a>
     原生连接（不支持云服务和 32 位应用程序）
    </h4>
    <ol>
     <li>
      <p>
       【开始】菜单搜索打开【ODBC 数据源(64 位)】管理工具（注意不要选择ODBC 数据源(32 位)）
      </p>
     </li>
     <li>
      <p>
       选中【用户 DSN】标签页，通过【添加(D)】按钮进入"创建数据源"界面
      </p>
     </li>
     <li>
      <p>
       选择想要添加的数据源，这里我们选择【TDengine】
      </p>
     </li>
     <li>
      <p>
       点击完成，进入 TDengine ODBC 数据源配置页面，填写如下必要信息
      </p>
      <p>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7e65826f6b8447c99a74aa730ea30975.png"/>
      </p>
      <p>
       4.1 【DSN】：Data Source Name 必填，为新添加的 ODBC 数据源命名
      </p>
      <p>
       4.2 【连接类型】：必选，选择连接类型，这里选择 【Native】 原生连接；
      </p>
      <p>
       4.3 【服务器】必填，ODBC 数据源 服务器 地址，示例：
       <code>
        localhost:6030
       </code>
      </p>
      <p>
       4.4 【数据库】选填，需要连接的默认数据库
      </p>
      <p>
       4.5 【用户名】仅供第5步测试连接使用，选填，数据库用户名，如果不填，TDengine 默认 root
      </p>
      <p>
       4.6 【密码】仅供第5步测试连接使用，选填，数据库用户密码，如果不填，TDengine 默认 taosdata
      </p>
      <p>
       4.7【兼容软件】支持对工业软件 KingSCADA、Kepware 等的兼容性适配，通常情况下，选择默认值 General 即可
      </p>
     </li>
     <li>
      <p>
       点【测试连接】测试连接情况，如果成功，提示"连接成功"
      </p>
     </li>
     <li>
      <p>
       点【确定】，即可保存配置并退出
      </p>
     </li>
     <li>
      <p>
       也可以在第2步选择已经配置好的数据源名通过【配置】按钮进入配置页面，修改已有配置
      </p>
     </li>
    </ol>
    <h3>
     <a id="_107">
     </a>
     支持的平台
    </h3>
    <ul>
     <li>
      原生连接方式支持的平台和 TDengine Windows X64版 客户端驱动支持的平台一致。
     </li>
     <li>
      WebSocket 连接方式除此之外还支持 Windows X64系统上运行的 32 位应用程序上使用。
     </li>
    </ul>
