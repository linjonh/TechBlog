    <h3>
     一、IP地址
    </h3>
    <h4>
     1. 什么是IP地址？
    </h4>
    <p>
     IP地址（Internet Protocol Address）是互联网中设备的唯一逻辑标识符，类似于现实生活中的“门牌号”。它分为
     <strong>
      IPv4
     </strong>
     （32位，如
     <code>
      192.168.1.1
     </code>
     ）和
     <strong>
      IPv6
     </strong>
     （128位，如
     <code>
      2001:0db8::1
     </code>
     ）两种版本。
    </p>
    <h4>
     2. IP地址的作用
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        设备标识
       </strong>
       ：在互联网中唯一标识一台设备。
      </p>
     </li>
     <li>
      <p>
       <strong>
        路由寻址
       </strong>
       ：帮助数据包找到从源设备到目标设备的传输路径。
      </p>
     </li>
     <li>
      <p>
       <strong>
        网络分段
       </strong>
       ：通过子网掩码划分网络，优化管理。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     3. 在Windows中查看与配置IP地址
    </h4>
    <h5>
     查看IP地址
    </h5>
    <ol>
     <li>
      <p>
       打开命令提示符（
       <code>
        Win + R
       </code>
       → 输入
       <code>
        cmd
       </code>
       ）。
       <img alt="" src="https://i-blog.csdnimg.cn/direct/185f87161f9545948e687afbb6eb0d08.png"/>
      </p>
     </li>
     <li>
      <p>
       输入命令：ipconfig
      </p>
      <p>
       在输出中查找
       <strong>
        IPv4 Address
       </strong>
       或
       <strong>
        IPv6 Address
       </strong>
       。
       <img alt="" src="https://i-blog.csdnimg.cn/direct/47fb8d0a4d5f4cd2b4d425c9b895ec45.png"/>
      </p>
     </li>
    </ol>
    <h5>
     配置IP地址
    </h5>
    <ol>
     <li>
      <p>
       打开“网络和共享中心” → 选择当前连接的网络 → 点击“属性”。
      </p>
     </li>
     <li>
      <p>
       双击
       <strong>
        Internet 协议版本 4 (TCP/IPv4)
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       选择
       <strong>
        使用下面的IP地址
       </strong>
       ，手动输入IP、子网掩码和默认网关。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     4. 未手动配置IP地址为何能上网？
    </h4>
    <p>
     设备默认启用
     <strong>
      DHCP（动态主机配置协议）
     </strong>
     ，会自动从路由器或运营商服务器获取以下信息：
    </p>
    <ul>
     <li>
      <p>
       IP地址
      </p>
     </li>
     <li>
      <p>
       子网掩码
      </p>
     </li>
     <li>
      <p>
       默认网关
      </p>
     </li>
     <li>
      <p>
       DNS服务器地址
       <br/>
       这一过程称为
       <strong>
        动态IP分配
       </strong>
       ，无需用户手动配置。
      </p>
     </li>
    </ul>
    <h4>
    </h4>
    <hr/>
    <h3>
     二、端口
    </h3>
    <h4>
     1. 什么是端口号？
    </h4>
    <p>
     端口号是网络通信中应用程序的“逻辑通道”，范围
     <code>
      0~65535
     </code>
     。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        知名端口
       </strong>
       ：
       <code>
        0~1023
       </code>
       （如 HTTP 的
       <code>
        80
       </code>
       、HTTPS 的
       <code>
        443
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        动态端口
       </strong>
       ：
       <code>
        1024~65535
       </code>
       （临时分配给客户端程序）。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     2. 测试特定端口是否通
    </h4>
    <p>
     使用
     <code>
      telnet
     </code>
     或
     <code>
      nc
     </code>
     命令（需确保工具已安装）：
    </p>
    <p>
     telnet 目标IP 端口号 # 或 nc -zv 目标IP 端口号
    </p>
    <ul>
     <li>
      <p>
       若返回
       <strong>
        Connected
       </strong>
       ，表示端口开放。
      </p>
     </li>
     <li>
      <p>
       若返回
       <strong>
        Connection refused
       </strong>
       ，表示端口关闭或被防火墙拦截。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     三、MAC地址
    </h3>
    <h4>
     1. 什么是MAC地址？
    </h4>
    <p>
     MAC地址（Media Access Control Address）是设备的物理地址，固化在网卡中，格式为
     <code>
      00-1A-2B-3C-4D-5E
     </code>
     （48位十六进制数）。
    </p>
    <h4>
     2. MAC地址的作用
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        局域网通信
       </strong>
       ：在数据链路层直接标识设备。
      </p>
     </li>
     <li>
      <p>
       <strong>
        ARP协议依赖
       </strong>
       ：通过MAC地址实现IP到物理地址的映射。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     3. 查看MAC地址
    </h4>
    <h5>
     Windows
    </h5>
    <p>
     ipconfig /all # 或在命令提示符输入： getmac
    </p>
    <h5>
     Linux/macOS
    </h5>
    <p>
     ifconfig # 或 ip link show
    </p>
    <hr/>
    <h3>
     四、数据包如何传给下一个节点？——ARP协议
    </h3>
    <h4>
     1. 什么是ARP？
    </h4>
    <p>
     ARP（Address Resolution Protocol）用于将
     <strong>
      IP地址
     </strong>
     解析为
     <strong>
      MAC地址
     </strong>
     ，实现局域网内设备通信。
    </p>
    <hr/>
    <h4>
     2. ARP工作流程
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        ARP请求
       </strong>
       ：源设备广播询问：“谁的IP是
       <code>
        192.168.1.20
       </code>
       ？请告诉
       <code>
        192.168.1.10
       </code>
       ”。
      </p>
     </li>
     <li>
      <p>
       <strong>
        ARP响应
       </strong>
       ：目标设备单播回复：“IP
       <code>
        192.168.1.20
       </code>
       的MAC地址是
       <code>
        00-1A-2B-3C-4D-5E
       </code>
       ”。
      </p>
     </li>
     <li>
      <p>
       <strong>
        更新ARP缓存
       </strong>
       ：源设备将IP-MAC映射存入本地ARP表，后续通信直接使用。
      </p>
     </li>
    </ol>
