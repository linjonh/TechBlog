    <h4>
     一、前言
    </h4>
    <p>
     java项目开发完成，部署上线，进入项目运维阶段，日常工作需要准备哪些资料和文档?当项目上线后，运行一段时间，或多或少会遇到一些运维上的问题，比如服务器磁盘饱满，服务器CPU，内存使用率过高，应用存在安全漏洞，应用报错，临时需求编个变更等等。诸多问题。那么若想快速响应，平时就需要准备好与项目运维相关的文档和资料。我总结了以下一些内容。
    </p>
    <p>
     <img alt="" height="339" src="https://i-blog.csdnimg.cn/blog_migrate/300ffc428fbdb63bf7f00730bace72e2.png" width="856"/>
    </p>
    <h4>
     二、运维需要的资料和文档
    </h4>
    <p>
     1、测试环境和生产环境部署说明文档
    </p>
    <p>
     2、测试环境，生产环境服务器台账
    </p>
    <p>
     3、项目中使用到的域名和端口
    </p>
    <p>
     4、各系统的地址和账号信息
    </p>
    <p>
     5、系统使用手册
    </p>
    <p>
    </p>
    <p>
     部署文档范例：
    </p>
    <h3 style="text-align:justify;">
     <strong>
      1.1服务器资源
     </strong>
    </h3>
    <p>
     操作系统版本：Linux CentOS 7.6
    </p>
    <p>
     JAVA环境：openjdk version "1.8.0_312"
    </p>
    <p>
     数据库：MYSQL8.0 或PostgreSQL 12.8
    </p>
    <h3 style="text-align:justify;">
     <strong>
      1.2 yum准备
     </strong>
    </h3>
    <h4 style="text-align:justify;">
     <strong>
      1.2.1 更新yum
     </strong>
    </h4>
    <p>
     yum update yum
    </p>
    <h4 style="text-align:justify;">
     <strong>
      1.2.2 安装yum工具
     </strong>
    </h4>
    <p>
     yum -y install yum-utils
    </p>
    <h4 style="text-align:justify;">
     <strong>
      1.2.3查看yum版本
     </strong>
    </h4>
    <p>
     yum –version
    </p>
    <p>
     版本为3.4.3或之后的最新版本即可
    </p>
    <h3 style="text-align:justify;">
     <strong>
      <a name="_Toc99015461">
      </a>
      1.3 JAVA部署
     </strong>
    </h3>
    <p>
     JAVA安装命令:
    </p>
    <p style="margin-left:21pt;">
     yum install java-1.8.0-openjdk.x86_64
    </p>
    <p>
     查看JAVA版本: java -version
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <p style="margin-left:.0001pt;text-align:justify;">
    </p>
    <h3 style="text-align:justify;">
     <strong>
      <a name="_Toc99015462">
      </a>
      1.4 docker环境及部署
     </strong>
    </h3>
    <h4 style="text-align:justify;">
     <strong>
      <a name="_Toc99015463">
      </a>
      1.4.1版本
     </strong>
    </h4>
    <p>
     Client: Docker Engine – Community version: 20.10.12
    </p>
    <p>
     Server: Docker Engine – Community version: 20.10.12
    </p>
    <h4 style="text-align:justify;">
     <strong>
      <a name="_Toc99015464">
      </a>
      1.4.2 部署
     </strong>
    </h4>
    <p>
     1.用yum拉取阿里云镜像:
    </p>
    <p style="margin-left:21pt;">
     yum-config-manager --add-repo
     <a href="http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo" rel="nofollow" title="http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo">
      http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
     </a>
    </p>
    <p>
     2. yum安装:
    </p>
    <p style="margin-left:42pt;">
     yum install docker-ce
    </p>
    <p>
     3.查看docker版本:
    </p>
    <p style="margin-left:42pt;">
     docker version
    </p>
    <p>
     4.启动docker:
    </p>
    <p style="margin-left:42pt;">
     systemctl start docker
    </p>
    <p>
     5.设置docker开机自启
    </p>
    <p style="margin-left:42pt;">
     systemctl enable docker
    </p>
    <h4 style="text-align:justify;">
     <strong>
      <a name="_Toc99015465">
      </a>
      1.4.3 文件夹
     </strong>
    </h4>
    <p style="margin-left:21pt;">
     1.在系统的/home路径下创建一个命名为“docker”的文件夹，用于存放docker相关文件。
    </p>
    <p style="margin-left:21pt;">
     2.在/home/docker路径下创建两个文件夹“script”和“apps”，用于存放脚本文件与镜像文件。
    </p>
    <h4 style="text-align:justify;">
     <strong>
      <a name="_Toc99015466">
      </a>
      1.4.4 docker常用命令
     </strong>
    </h4>
    <p style="margin-left:21pt;">
     1.查看docker镜像: docker images
    </p>
    <p style="margin-left:21pt;">
     2.查看docker正在运行容器: docker ps
    </p>
    <p style="margin-left:21pt;">
     3.查看docker里的容器: docker ps -a
    </p>
    <h3 style="text-align:justify;">
     <strong>
      <a name="_Toc99015467">
      </a>
      1.5 nginx部署
     </strong>
    </h3>
    <p>
     端口需要:80
    </p>
    <h4 style="text-align:justify;">
     <strong>
      <a name="_Toc99015468">
      </a>
      1.5.1 docker部署nginx
     </strong>
    </h4>
    <p>
     docker pull nginx:latest
    </p>
    <h4 style="text-align:justify;">
     <strong>
      1.5.2运行nginx
     </strong>
    </h4>
    <p>
     1.通过脚本文件nginx.sh运行，脚本文件的运行需授予权限，在文件目录下命令：
    </p>
    <p>
     chmod 755 nginx.sh
    </p>
    <p>
     2.运行脚本命令：./nginx.sh
    </p>
    <h4 style="text-align:justify;">
     <strong>
      1.5.3nginx常用命令
     </strong>
    </h4>
    <p style="margin-left:21pt;">
     1.启动: docker start nginx
    </p>
    <p style="margin-left:21pt;">
     2.重启: docker restart nginx
    </p>
    <p style="margin-left:21pt;">
     3.停止: docker stop nginx
    </p>
