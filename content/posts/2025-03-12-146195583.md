    <h3>
     <a id="_0">
     </a>
     导致报错的操作
    </h3>
    <ol>
     <li>
      在已安装了
      <code>
       oracle11g
      </code>
      版本后再安装
      <code>
       oracle9i
      </code>
      客户端
     </li>
     <li>
      在安装了
      <code>
       oracle11g
      </code>
      后修改了主机名称
     </li>
    </ol>
    <h3>
     <a id="_5">
     </a>
     解决方法
    </h3>
    <ol>
     <li>
      修改配置文件的主机名称
      <br/>
      可能是因为你的Oracle文件配置的主机名称出了问题，在你的Oarcle安装路径
      <code>
       D:\app\lenovo\product\11.2.0\dbhome_1\NETWORK\ADMIN
      </code>
      下找到
      <code>
       listener.ora
      </code>
      ，
      <br/>
      将
      <code>
       HOST
      </code>
      的值修改成你自己电脑的名称(右键我的电脑，属性，查看主机名称）
     </li>
     <li>
      配置环境变量
     </li>
    </ol>
    <pre><code class="prism language-bash">ORACLE_HOME：D:<span class="token punctuation">\</span>app<span class="token punctuation">\</span>lenovo<span class="token punctuation">\</span>product<span class="token punctuation">\</span><span class="token number">11.2</span>.0<span class="token punctuation">\</span>dbhome_1
TNS_ADMIN：D:<span class="token punctuation">\</span>app<span class="token punctuation">\</span>lenovo<span class="token punctuation">\</span>product<span class="token punctuation">\</span><span class="token number">11.2</span>.0<span class="token punctuation">\</span>dbhome_1<span class="token punctuation">\</span>NETWORK<span class="token punctuation">\</span>ADMIN
</code></pre>
    <ol start="3">
     <li>
      修改注册表
     </li>
    </ol>
    <p>
     <code>
      HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\OracleOraDb11g_home1TNSListener
     </code>
     注册表的ImagePath值，
     <code>
      D:\app\lenovo\product\11.2.0\dbhome_1\BIN\TNSLSNR
     </code>
     中没有’
     <code>
      .exe
     </code>
     ’，所以也加上后缀；
     <br/>
     4. 重新配置监听程序
     <br/>
     删除
     <code>
      D:\app\lenovo\product\11.2.0 \ db \ NETWORK \ ADMIN
     </code>
     下的
     <code>
      listener.ora
     </code>
     文件
     <br/>
     选择
     <code>
      Net Configuration Assistant
     </code>
     监听，一直无脑下一步，新建了个
     <code>
      listener.ora
     </code>
     文件，启动
     <code>
      OracleOraDb11g_home1TNSListener
     </code>
     服务还是报错，但是此时多了个
     <code>
      OracleTNSListener
     </code>
     服务，可以正常启动，然后试着
     <code>
      ODBC
     </code>
     和数据库连接，还是报错
     <br/>
     5. 重启数据库
    </p>
    <pre><code class="prism language-sql">sqlplus <span class="token operator">/</span> <span class="token keyword">as</span> sysdba
startup <span class="token keyword">force</span>
</code></pre>
    <p>
     再重新连接数据库和
     <code>
      ODBC
     </code>
     ，就没有问题了他
    </p>

    </p>
