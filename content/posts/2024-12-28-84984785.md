    <h3 id="项目介绍：">
     项目介绍：
    </h3>
    <hr/>
    <p>
     1.项目分为一个前台系统，一个后台管理系统，后台管理一些测试数据和用户数据，前台不提供注册，全部由后台创建用户
     <br/>
     2.系统首页可创建，删除，修改项目
     <br/>
     3.点击项目后，分为项目首页，测试地址管理，接口管理，自动化管理，项目动态，项目成员
     <br/>
     4.接口管理中，提供类似postman的快速测试方式，录入的接口可进行测试，修改，查看测试记录，修改记录及删除；还可直接导出接口文档，后续引入新功能导入swagger的接口信息
     <br/>
     5.自动化测试管理可直接从接口管理中导入接口信息，也可新建接口；单条用例内，后面的接口可关联前面接口的返回结果，前置条件是前面的接口校验方式选择json校验，并填写正确的json格式的数据。目前接口返回结果校验只提供5种方式，分别是不校验，校验HTTP状态，校验JSON格式，完全校验与正则校验。可手动测试生成测试报告。自动化测试需要部署在Linux环境下，使用的是crontab作为定时任务与循环任务，目前自动化测试报告的模板还在开发中。。。。
    </p>

     系统声明：

    <p>

     2.初步学习web开发，接口统一采用基于方法的方式编写，后续引入权限系统，并修改成基于类的方法

    <h3 id="使用方法：">
     使用方法：
    </h3>
    <hr/>
    <h4 id="1.安装Python3环境（未在Python2上运行后，不知道有没有问题）">
     1.安装Python3环境（未在Python2上运行后，不知道有没有问题）
    </h4>
    <h4 id="2.下载代码到本地并解压">
     2.下载代码到本地并解压
    </h4>
    <h4 id="3.cmd到根目录下安装相关依赖包">
     3.cmd到根目录下安装相关依赖包
    </h4>
    <pre class="has"><code>pip install -r requirements.txt
pip install https://github.com/darklow/django-suit/tarball/v2
</code></pre>
    <h4 id="4.安装mysql数据库，配置数据库连接，进入api_automation_test/settings.py">
     4.安装mysql数据库，配置数据库连接，进入api_automation_test/settings.py
    </h4>
    <pre class="has"><code>DATABASES = {
    'default': {
        # 'ENGINE': 'django.db.backends.sqlite3',
        # 'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
        'ENGINE':'django.db.backends.mysql',     # 数据库类型，mysql
        'NAME':'api_test',            #  database名
        'USER':'root',               # 登录用户
        'PASSWORD':'123456',        #  登录用户名
        'HOST':'127.0.0.1',        # 数据库地址
        'PORT':'3306'              # 数据库端口
    }
}
</code></pre>
    <h4>
     5.cmd到根目录下，让 Django 知道我们在我们的模型有一些变更
    </h4>
    <pre class="has"><code>python manage.py makemigrations
</code></pre>
    <h4 id="6.创造或修改表结构">
     6.创造或修改表结构
    </h4>
    <pre class="has"><code>python manage.py migrate 
</code></pre>
    <h4 id="7.创建超级用户，用于登录后台管理">
     7.创建超级用户，用于登录后台管理
    </h4>
    <pre class="has"><code>python manage.py createsuperuser
</code></pre>
    <h4 id="8.安装VUE环境，下载node.js并配置环境，下载npm包管理器，安装vue脚手架用于生成vue工程模板">
     8.安装VUE环境，下载node.js并配置环境，下载npm包管理器，安装vue脚手架用于生成vue工程模板
    </h4>
    <pre class="has"><code>npm install --global vue-cli
</code></pre>
    <h4>
     9.cmd进入frontend目录下，运行npm install安装相关依赖包
    </h4>
    <h4 id="10.打包">
     10.打包
    </h4>
    <pre class="has"><code>npm run build
</code></pre>
    <h4 id="11.运行启动django服务">
     11.运行启动django服务
    </h4>
    <pre class="has"><code>python manage.py runserver 0.0.0.0:8000 
</code></pre>
    <h4>
     12.现在就可以访问
     <a href="http://127.0.0.1:8000/" rel="nofollow">
      http://127.0.0.1:8000
     </a>
     进行登录，
     <a href="http://127.0.0.1:8000/admin" rel="nofollow">
      http://127.0.0.1:8000/admin
     </a>
     为后台管理平台(注：如果登录不了的话，可能是接口请求地址错误，修改api_automation_test/frontend/src/api/api.js 注释其他IP，保留127.0.0.1
    </h4>
    <pre class="has"><code>export const test = 'http://127.0.0.1:8000';
</code></pre>
