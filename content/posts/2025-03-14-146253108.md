    <h2>
     <a id="_5">
     </a>
     写作结构
    </h2>
    <h3>
     <a id="_6">
     </a>
     摘要
    </h3>
    <ol>
     <li>
      项目简介：背景、功能介绍。
     </li>
     <li>
      本人工作角色、工作内容
     </li>
     <li>
      采用了（技术、方法……），解决了……
     </li>
     <li>
      效果、问题、展望
     </li>
    </ol>
    <h3>
     <a id="_12">
     </a>
     正文
    </h3>
    <ol>
     <li>
      项目简介在稍微详细地写一遍
      <ol>
       <li>
        项目概要-
        <strong>
         what
        </strong>
       </li>
       <li>
        作者担任的工作-
        <strong>
         who
        </strong>
       </li>
       <li>
        开发体制、项目在系统设计方面的情况
       </li>
      </ol>
     </li>
     <li>
      过渡段：识别关键因素、出了什么问题？（
      <strong>
       可选
      </strong>
      ）
     </li>
     <li>
      理论部分：引入理论，并简要阐述-
      <strong>
       why
      </strong>
     </li>
     <li>
      实践部分：详细阐述理论如何指导实践-
      <strong>
       how
      </strong>
     </li>
    </ol>
    <h3>
     <a id="_21">
     </a>
     结尾
    </h3>
    <ol>
     <li>
      项目总结：已投产运行*月，取得了……成果。
     </li>
     <li>
      遗留问题、建议、展望。
     </li>
     <li>
      本人的收获、不足、下一步努力方向。
     </li>
    </ol>
    <h2>
     <a id="_26">
     </a>
     写作要求
    </h2>
    <h3>
     <a id="_27">
     </a>
     理论知识
    </h3>
    <ol>
     <li>
      科目1、科目2同时准备，理解基础上记住要点。
     </li>
     <li>
      <strong>
       不要死记硬背
      </strong>
     </li>
    </ol>
    <h3>
     <a id="_31">
     </a>
     熟悉擅长
    </h3>
    <ol>
     <li>
      项目内容最好选择自己熟悉的、亲历过的。
     </li>
     <li>
      技术内容最好选择自己熟悉的、最擅长的。
     </li>
    </ol>
    <h3>
     <a id="_35">
     </a>
     精写调整
    </h3>
    <ol>
     <li>
      手敲练习，内容完整。
     </li>
     <li>
      找专业老师进行批改，反复调整。
     </li>
    </ol>
    <h2>
     <a id="_39">
     </a>
     写作步骤
    </h2>
    <ol>
     <li>
      选试题（3~5分钟）
     </li>
     <li>
      论文构思（5分钟）
     </li>
     <li>
      写摘要（20分钟）
     </li>
     <li>
      正文撰写（80分钟）
     </li>
     <li>
      检查修正（10分钟）
     </li>
    </ol>
    <p>
     题目都不会的话，优先选理论性题目。
     <br/>
     理论性题目对实践性要求不高。
    </p>
    <h2>
     <a id="_49">
     </a>
     注意事项
    </h2>
    <ol>
     <li>
      以我为中心
     </li>
     <li>
      站在高级工程师的高度
     </li>
     <li>
      忠实于论点
     </li>
     <li>
      条理清晰，开门见山
     </li>
     <li>
      首尾一致
     </li>
     <li>
      切忌空有理论没有实际
     </li>
     <li>
      注意回答小标题
     </li>
    </ol>
