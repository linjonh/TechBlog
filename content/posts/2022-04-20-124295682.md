    <h3>
     <strong>
      1、Robot Framework简介
     </strong>
    </h3>
    <p class="img-center">
     <img alt="" height="468" src="https://i-blog.csdnimg.cn/blog_migrate/d795386c34ec48c5776fa99da55b765a.png" width="595"/>
    </p>
    <p>
     <span style="color:#fe2c24;">
      Robot Framework是一个基于Python的可扩展关键字驱动的自动化框架，用于验收测试，验收测试驱动开发（ATDD），行为驱动开发（BDD）和机器人流程自动化（RPA）。
     </span>
     它可以用于分布式异构环境，其中自动化需要使用不同的技术和接口。由Robot Framework基金会支持。许多行业领先的公司在其软件开发中使用该工具。
    </p>
    <p>
     Robot Framework是开放和可扩展的。可以与几乎任何其他工具集成，以创建强大而灵活的自动化解决方案而著称。Robot Framework可以免费使用，无需许可费用。
    </p>
    <p>
    </p>
    <h3>
     <strong>
      2、
     </strong>
     <strong>
      Robot Framework
     </strong>
     <strong>
      框架特点
     </strong>
    </h3>
    <p>
     1、RF支持易于使用的表格语法，以便以统一的方式创建测试用例。
    </p>
    <p>
     2、RF提供从现有关键字创建可重用的高级关键字的功能。
    </p>
    <p>
     3、提供易于阅读的HTML 格式结果报告和日志。
    </p>
    <p>
     4、独立于平台和应用程序。
    </p>
    <p>
     5、提供用于创建可使用 Python 本机实现的自定义测试库。
    </p>
    <p>
     6、提供命令行界面和基于 XML 的输出文件，以便集成到现有的生成基础结构（持续集成系统）中。
    </p>
    <p>
     7、支持测试 Web 应用程序、rest API、移动应用程序、正在运行的进程、通过 Telnet 或 SSH 连接到远程系统等。
    </p>
    <p>
     8、支持创建数据驱动的测试用例。
    </p>
    <p>
     9、具有对变量的内置支持，特别适用于在不同环境中进行测试。
    </p>
    <p>
     10、提供标记以分类和选择要执行的测试用例。
    </p>
    <p>
     11、支持集成：测试套件只是可与生产代码一起进行版本控制的文件和目录。
    </p>
    <p>
     12、提供测试用例和测试套件级别的设置和拆卸。
    </p>
    <p>
     13、模块化架构支持创建测试，甚至适用于具有多个不同接口的应用程序。
    </p>
    <p>
    </p>
    <h3>
     <strong>
      3、
     </strong>
     <strong>
      Robot Framework
     </strong>
     <strong>
      框架的体系结构
     </strong>
    </h3>
    <p>
     Robot Framework是一个通用的，独立于应用程序和技术的框架。它具有高度模块化的体系结构，如下图所示。
    </p>
    <p class="img-center">
     <img alt="" height="405" src="https://i-blog.csdnimg.cn/blog_migrate/093d2aa0b813e43c7fd92358c19f31c5.png" width="676"/>
    </p>
    <p>
     测试数据采用简单、易于编辑的表格格式。当Robot Framework启动时，它会处理数据，执行测试用例并生成日志和报告。核心框架对被测目标一无所知，与它的交互由库处理。库可以直接使用应用程序接口，也可以使用较低级别的测试工具作为驱动程序。
    </p>
    <p>
    </p>

    <h3>
     <strong>
      6、

     <strong>
      Robot Framework
     </strong>
     <strong>
      简单示例
     </strong>
    </h3>
    <p>
     <strong>
      案例中包含2个文件：
     </strong>
    </p>
    <p>
     Test_suite.robot文件
    </p>
    <p>
     Keywords.resource文件
    </p>
    <p class="img-center">
     <img alt="" height="290" src="https://i-blog.csdnimg.cn/blog_migrate/1e712884c887c2d075607aa5d6d90bf4.png" width="789"/>

    <p>
     <strong>
      1）创建一个robot文件（代码如下所示）：
     </strong>
    </p>
    <pre><code>*** Settings ***</code><code>Documentation     A test suite for valid login.</code><code>...</code><code>...               Keywords are imported from the resource file</code><code>Resource          keywords.resource</code><code>Default Tags      positive</code>
<code>*** Test Cases ***</code><code>Login User with Password</code><code>    Connect to Server</code><code>    Login User            ironman    1234567890</code><code>    Verify Valid Login    Tony Stark</code><code>    [Teardown]    Close Server Connection</code>
<code>Denied Login with Wrong Password</code><code>    [Tags]    negative</code><code>    Connect to Server</code><code>    Run Keyword And Expect Error    *Invalid Password    Login User    ironman    123</code><code>    Verify Unauthorised Access</code><code>    [Teardown]    Close Server Connection</code>
<code>2）创建一个关键字文件：</code>
<code>*** Settings ***</code><code>Documentation     This is a resource file, that can contain variables and keywords.</code><code>...               Keywords defined here can be used where this Keywords.resource in loaded.</code><code>Library           CustomLibrary.py</code>

<code>*** Keywords ***</code><code>Connect to Server</code><code>    Connect    fe80::aede:48ff:fe00:1122</code>
<code>Close Server Connection</code><code>    Disconnect</code>
<code>Login User</code><code>    [Arguments]    ${login}    ${password}</code><code>    Set Login Name    ${login}</code><code>    Set Password    ${password}</code><code>    Execute Login</code>
<code>Verify Valid Login</code><code>    [Arguments]    ${exp_full_name}</code><code>    ${version}=    Get Server Version</code><code>    Should Not Be Empty    ${version}</code><code>    ${name}=    Get User Name</code><code>    Should Be Equal    ${name}    ${exp_full_name}</code>
<code>Verify Unauthorised Access</code><code>    Run Keyword And Expect Error    PermissionError*    Get Server Version</code>
<code>Login Admin</code><code>    [Documentation]    'Login Admin' is a Keyword.</code><code>    ...                It calls 'Login User' from 'CustomLibrary.py'</code><code>    Login User    admin    @RBTFRMWRK@</code><code>    Verify Valid Login    Administrator</code></pre>
