    <h2>
     <a id="_0">
     </a>
     序言：
    </h2>
    <p>
     本文介绍通过SSM架构 +Nginx+FFmpeg实现rtsp流转hls流，在前端html上实现视频播放功能。此方法可用于网络摄像头RTSP视频流WEB端实时播放。（海康和大华都可以），我使用的是海康
    </p>
    <h3>
     <a id="_FFmpegwindows_3">
     </a>
     步骤一：安装软件 FFmpeg(windows)
    </h3>
    <p>
     1.
     <strong>
      下载并解压:
     </strong>
     FFmpeg文件夹
     <br/>
     2.
     <strong>
      配置环境变量：
     </strong>
     在“Path”变量原有变量值内容上加上
     <strong>
      d:\ffmpeg\bin
     </strong>
     ，
     <br/>
     3.
     <strong>
      验证：
     </strong>
     ffmpeg -version 出现版本号则成功。
    </p>
    <h3>
     <a id="_Nginxwindows_10">
     </a>
     步骤二：安装软件 Nginx(windows)

     2.

     </strong>

    <h2>
     <a id="RTSP_38">
     </a>
     RTSP地址获取流程
    </h2>
    <p>
     例如：
     <strong>
      rtsp://admin:bfm100866@192.168.1.64:554/streaming/channels/101
     </strong>
    </p>
    <h3>
     <a id="RTSP_43">
     </a>
     一、RTSP地址基本格式
    </h3>
    <p>
     海康摄像头的RTSP地址通常遵循以下基本格式：
    </p>
    <pre><code class="prism language-java">
rtsp<span class="token operator">:</span><span class="token operator">/</span><span class="token operator">/</span><span class="token punctuation">[</span>username<span class="token punctuation">]</span><span class="token operator">:</span><span class="token punctuation">[</span>password<span class="token punctuation">]</span>@<span class="token punctuation">[</span>ip<span class="token punctuation">]</span><span class="token operator">:</span><span class="token punctuation">[</span>port<span class="token punctuation">]</span><span class="token operator">/</span><span class="token punctuation">[</span>path<span class="token punctuation">]</span>


<span class="token operator">-</span>username：摄像头的用户名，默认为“admin”。

<span class="token operator">-</span>password：摄像头的密码，需根据设备实际设置而定。

<span class="token operator">-</span>ip：摄像头的<span class="token constant">IP</span>地址。

<span class="token operator">-</span>port：<span class="token constant">RTSP</span>端口号，默认为<span class="token number">554</span>，若未修改则可不写。

<span class="token operator">-</span>path：流媒体的访问路径，根据摄像头的型号和配置不同而有所区别。


</code></pre>
    <h3>
     <a id="RTSP_65">
     </a>
     二、RTSP地址获取步骤
    </h3>
    <h4>
     <a id="1_67">
     </a>
     1.登录摄像头管理系统
    </h4>
    <p>
     首先，通过浏览器登录摄像头的Web管理系统，通常使用摄像头的默认IP地址（如192.168.1.64）和默认用户名密码（admin/admin或admin/12345）进行登录。
    </p>
    <h4>
     <a id="2_71">
     </a>
     2.查看网络配置
    </h4>
    <p>
     在摄像头管理系统中，找到网络配置部分，查看并记录摄像头的IP地址、端口号等信息。
    </p>
    <h4>
     <a id="3RTSP_75">
     </a>
     3.确定RTSP路径
    </h4>
    <p>
     RTSP路径可能因摄像头型号和固件版本而异，一般有以下几种格式：
    </p>
    <p>
     -新格式：
     <code>
      rtsp://[username]:[password]@[ip]:[port]/streaming/channels/
     </code>
    </p>
    <p>
     -例如，取主码流：
     <code>
      rtsp://admin:123456@192.168.1.64:554/streaming/channels/101
     </code>
    </p>
    <p>
     -参数``中，通道1的主码流为“101”，子码流为“102”。
    </p>
    <p>
     -老格式：
     <code>
      rtsp://[username]:[password]@[ip]:[port]/[videotype]/ch/[streamtype]/av_stream
     </code>
    </p>
    <p>
     -例如，主码流取流：
     <code>
      rtsp://admin:12345@192.168.0.1:554/h264/ch1/main/av_stream
     </code>
    </p>
    <p>
     -参数中，
     <code>
      videotype
     </code>
     为视频编码类型（如h264、mpeg4），
     <code>
      streamtype
     </code>
     为码流类型（主码流为main，子码流为sub）。
    </p>
    <h4>
     <a id="4RTSP_91">
     </a>
     4.测试RTSP地址
    </h4>
    <p>
     使用支持RTSP的播放器（如
     <strong>
      VLCMediaPlayer
     </strong>
     ）输入RTSP地址进行测试，确认视频流是否能够正常获取和播放。
    </p>
    <p>
     VLC安装包链接：
     <a href="https://download.csdn.net/download/qq_39951524/90465347">
      VLC安装包
     </a>
     https://download.csdn.net/download/qq_39951524/90465347
    </p>
    <h3>
     <a id="FFmpegrtsp__hls_97">
     </a>
     FFmpeg命令行转流(rtsp 转 hls流)
    </h3>
    <pre><code class="prism language-java">
ffmpeg <span class="token operator">-</span>fflags <span class="token operator">+</span>genpts
 <span class="token operator">-</span>i <span class="token string">"rtsp://admin:bfm100866@192.168.1.64:554/streaming/channels/101"</span> 
 <span class="token operator">-</span>c copy 
 <span class="token operator">-</span>f hls 
 <span class="token operator">-</span>hls_time <span class="token number">10</span> 
 <span class="token operator">-</span>hls_list_size <span class="token number">10</span> 
  <span class="token class-name">D</span><span class="token operator">:</span><span class="token operator">/</span>nginx<span class="token operator">/</span>html<span class="token operator">/</span>hls<span class="token operator">/</span>test<span class="token punctuation">.</span>m3u8

</code></pre>
    <p>
     命令解释
    </p>
    <pre><code class="prism language-java">
<span class="token operator">-</span>fflags <span class="token operator">+</span>genpts<span class="token operator">:</span> 这个选项告诉 <span class="token class-name">FFmpeg</span> 在处理输入文件时生成 <span class="token constant">PTS</span>。这对于某些编码的视频文件（特别是那些 <span class="token constant">PTS</span><span class="token operator">/</span><span class="token constant">DTS</span> 不完全正确或缺失的）非常有用。它可以确保 <span class="token constant">PTS</span> 被正确生成，这对于 <span class="token constant">HLS</span> 流的正确播放很重要。

<span class="token operator">-</span>i <span class="token string">"rtsp:sdd.mp4"</span><span class="token operator">:</span> 指定输入文件，这里是 rtsp视频流。

<span class="token operator">-</span>c copy<span class="token operator">:</span> 使用复制流的方式来编码视频和音频，这意味着不对数据进行重新编码，只是简单地复制数据。这可以减少处理时间和提高效率，但前提是输入和输出容器及编解码器必须兼容。

<span class="token operator">-</span>f hls<span class="token operator">:</span> 指定输出格式为 <span class="token constant">HLS</span>。

<span class="token operator">-</span>hls_time <span class="token number">2.0</span><span class="token operator">:</span> 设置每个 <span class="token constant">HLS</span> 分片的时长为 <span class="token number">2</span> 秒。

<span class="token operator">-</span>hls_list_size <span class="token number">1</span><span class="token operator">:</span> 指定播放列表中最多包含一个分片。这对于事件驱动的直播非常有用，因为你可以在事件发生时快速更新播放列表。

output<span class="token operator">/</span><span class="token operator">:</span> 指定输出目录，所有生成的 <span class="token constant">HLS</span> 文件和播放列表将被放置在此目录中。


</code></pre>
