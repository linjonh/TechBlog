    <h2>
     【算法实战】日期转二进制：两种解法的思路与优化（附代码解析）
    </h2>
    <h3>
     一、问题描述
    </h3>
    <p>
     给定一个
     <code>
      yyyy-mm-dd
     </code>
     格式的日期字符串，要求将年、月、日分别转为
     <strong>
      无前导零的二进制
     </strong>
     ，并保持
     <code>
      year-month-day
     </code>
     格式。
     <br/>
     <strong>
      示例
     </strong>
     ：输入
     <code>
      2025-03-15
     </code>
     ，输出
     <code>
      11111101001-11-1111
     </code>
     （2025→11111101001，3→11，15→1111）。
    </p>
    <h3>
     二、解法一：直接分割转换（新手友好）
    </h3>
    <h4>
     思路分析
    </h4>
    <ol>
     <li>
      <strong>
       分割日期
      </strong>
      ：按
      <code>
       -
      </code>
      拆分为年、月、日三部分。
     </li>
     <li>
      <strong>
       转二进制
      </strong>
      ：利用
      <code>
       Integer.toBinaryString
      </code>
      直接转换。
     </li>
     <li>
      <strong>
       拼接结果
      </strong>
      ：按格式拼接，无需处理前导零（Java 方法自动省略）。
     </li>
    </ol>
    <h4>
     代码实现
    </h4>
    <pre><code class="language-java">class Solution {
    public String convertDateToBinary(String date) {
        String[] parts = date.split("-"); // 分割日期
        return Integer.toBinaryString(Integer.parseInt(parts[0])) + "-" +
               Integer.toBinaryString(Integer.parseInt(parts[1])) + "-" +
               Integer.toBinaryString(Integer.parseInt(parts[2]));
    }
}
</code></pre>
    <h4>
     代码亮点
    </h4>
    <ul>
     <li>
      <strong>
       简洁直观
      </strong>
      ：一行代码完成核心逻辑，适合快速开发。
     </li>
     <li>
      <strong>
       自动去前导零
      </strong>
      ：Java 的
      <code>
       toBinaryString
      </code>
      天然支持，无需额外处理。
     </li>
    </ul>
    <h4>
     复杂度分析
    </h4>
    <ul>
     <li>
      时间：O (1)（日期长度固定，拆分和转换为常数操作）
     </li>
     <li>
      空间：O (1)（除输入外，额外空间固定）
     </li>
    </ul>
    <h3>
     三、解法二：逆向字符填充（性能优化）
    </h3>
    <h4>
     思路升级
    </h4>
    <p>
     观察发现：日期各部分的二进制长度有限（年≤13 位，月≤4 位，日≤5 位），可预先分配固定大小的字符数组，
     <strong>
      从后向前填充
     </strong>
     ，避免字符串拼接的性能损耗。
    </p>
    <h4>
     关键步骤拆解（以
     <code>
      2025-03-15
     </code>
     为例）
    </h4>
    <ol>
     <li>
      <strong>
       提取日（15）
      </strong>
      ：转为二进制
      <code>
       1111
      </code>
      ，逆向填充到数组末尾。
     </li>
     <li>
      <strong>
       插入
       <code>
        -
       </code>
      </strong>
      ：在日的二进制前添加分隔符。
     </li>
     <li>
      <strong>
       提取月（3）
      </strong>
      ：转为
      <code>
       11
      </code>
      ，填充到
      <code>
       -
      </code>
      前。
     </li>
     <li>
      <strong>
       插入
       <code>
        -
       </code>
      </strong>
      ：在月的二进制前添加分隔符。
     </li>
     <li>
      <strong>
       提取年（2025）
      </strong>
      ：转为
      <code>
       11111101001
      </code>
      ，填充到最前面。
     </li>
    </ol>
    <h4>
     代码实现（含详细注释）
    </h4>
    <pre><code class="language-java">class Solution {
    private static final int MAX_LEN = 32; // 足够容纳最大可能的二进制（年13位 + 2个- + 月4位 + 日5位 = 25位）

    public String convertDateToBinary(String date) {
        char[] newDate = new char[MAX_LEN]; // 预分配固定大小数组
        int begin = MAX_LEN; // 从数组末尾开始填充
        
        // 处理日（索引8-9，即"dd"）
        begin = appendBinary(date, newDate, begin, 8, 9);
        newDate[--begin] = '-'; // 插入日-月分隔符
        
        // 处理月（索引5-6，即"mm"）
        begin = appendBinary(date, newDate, begin, 5, 6);
        newDate[--begin] = '-'; // 插入月-年分隔符
        
        // 处理年（索引0-3，即"yyyy"）
        appendBinary(date, newDate, begin, 0, 3);
        
        // 从begin到数组末尾的有效字符转为字符串
        return new String(newDate, begin, MAX_LEN - begin);
    }

    private int appendBinary(String date, char[] arr, int ptr, int start, int end) {
        int num = 0;
        // 1. 提取日期部分并转为整数（如"2025"→2025）
        for (int i = start; i &lt;= end; i++) {
            num = num * 10 + (date.charAt(i) - '0');
        }
        // 2. 整数转二进制，逆向填充字符数组
        while (num &gt; 0) {
            arr[--ptr] = (char) ((num % 2) + '0'); // 取最低位，从后往前填
            num /= 2;
        }
        return ptr; // 返回新的填充起点
    }
}
</code></pre>
    <h4>
     核心优化点
    </h4>
    <ol>
     <li>
      <strong>
       避免字符串拼接
      </strong>
      ：通过字符数组直接操作，减少
      <code>
       String
      </code>
      对象创建开销。
     </li>
     <li>
      <strong>
       逆向填充
      </strong>
      ：二进制低位在后（如 15→1111，填充顺序是
      <code>
       1→1→1→1
      </code>
      ），天然符合数组从后往前填充的逻辑。
     </li>
     <li>
      <strong>
       固定长度数组
      </strong>
      ：预先计算最大可能长度（32 位足够），空间利用更高效。
     </li>
    </ol>
