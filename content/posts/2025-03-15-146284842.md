    <p>
    </p>
    <p>
    </p>
    <p>
     在Kubernetes（简称K8s）中，Endpoint是一个非常重要的资源，它主要用于表示和管理服务（Service）所暴露的网络地址（IP地址和端口）。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6c079fdd66d140709b1015fa55154e68.png#pic_center"/>
    </p>
    <h4>
     <a id="Endpoint_9">
     </a>
     一、Endpoint的定义与作用
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        定义
       </strong>
       ：Endpoint是K8s中用于描述Service实际访问点的资源。它包含了提供服务的Pod的IP地址和端口信息，这些信息是K8s实现服务发现和流量分发的关键依据。
      </p>
     </li>
     <li>
      <p>
       <strong>
        作用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         服务发现
        </strong>
        ：通过记录具体Pod的网络信息，让外部请求能够准确找到提供服务的实例。
       </li>
       <li>
        <strong>
         负载均衡
        </strong>
        ：K8s使用Endpoints来将流量路由到正确的Pod上，以实现负载均衡。
       </li>
       <li>
        <strong>
         关联Service与Pod
        </strong>
        ：Endpoint的主要作用是将Service与后端Pod关联起来，确保服务请求能够被正确地路由到可用的Pod实例上，实现了服务的高可用性和负载均衡。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/671d07c9f4be4ab3937cf063f082d89a.png#pic_center"/>
    </p>
    <h4>
     <a id="Endpoint_20">
     </a>
     二、Endpoint的创建与管理
    </h4>
    <ol>
     <li>
      <strong>
       自动创建
      </strong>
      ：在K8s中，当创建一个Service时，K8s会根据Service的选择器（Selector）自动发现匹配的Pod，并创建相应的Endpoint。这种自动化的创建机制大大简化了服务部署和管理的流程。
     </li>
     <li>
      <strong>
       手动创建
      </strong>
      ：虽然不常见，但Endpoint也可以手动创建。这通常用于特殊情况，比如需要显式指定Endpoint的IP地址和端口，例如在与外部非K8s管理的服务进行集成时，可能需要手动创建Endpoint来定义服务的访问信息，以便将其纳入K8s的服务管理体系中。
     </li>
     <li>
      <strong>
       Endpoint控制器
      </strong>
      ：K8s的Endpoint控制器负责监控Service和Pod的变化，并自动更新Endpoint的信息。例如，当一个新的Pod创建时，Endpoint控制器会将该Pod的IP地址和端口添加到对应的Endpoint中；同样，当Pod被删除或发生故障时，Endpoint控制器也会及时更新Endpoint信息，确保服务的流量始终能够分发到健康的Pod实例上。
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4a57afd848ff4f698dd72e26499cae78.png#pic_center"/>
    </p>
    <h4>
     <a id="Endpoint_28">
     </a>
     三、Endpoint的查看与组成
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        查看Endpoint
       </strong>
       ：可以使用
       <code>
        kubectl get endpoints
       </code>
       命令来查看默认命名空间的Endpoint。输出结果会列出每个Service对应的Endpoint，包括IP地址和端口信息。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Endpoint的组成
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         IP地址
        </strong>
        ：Pod或其他网络服务的IP地址。
       </li>
       <li>
        <strong>
         端口
        </strong>
        ：服务暴露的端口。
       </li>
       <li>
        <strong>
         TargetRef
        </strong>
        ：指向实际目标（Pod）的引用，通常包括Pod的名称和命名空间。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8f9217fadb2a45d0bf312e020f0b05e7.png#pic_center"/>
    </p>
    <h4>
     <a id="EndpointSlice_39">
     </a>
     四、EndpointSlice
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        概念
       </strong>
       ：为了提高Endpoint的管理效率，K8s 1.21引入了EndpointSlice资源。EndpointSlice将Endpoint分片管理，每个EndpointSlice包含一组Endpoint，避免了单个Endpoint资源过大带来的性能问题。
      </p>
     </li>
     <li>
      <p>
       <strong>
        优势
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         增强扩展性
        </strong>
        ：支持更大规模的集群和更多的Endpoint。
       </li>
       <li>
        <strong>
         提高性能
        </strong>
        ：在大规模集群环境下，EndpointSlice能够更好地适应不断增长的服务和Pod数量，保证服务发现和流量分发的高效运行。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7e2a565611b54143973b68159f8de7e9.png#pic_center"/>
    </p>
    <h4>
     <a id="Endpoint_49">
     </a>
     五、Endpoint的使用场景
    </h4>
    <ol>
     <li>
      <strong>
       集群内部服务访问
      </strong>
      ：在K8s集群内部，Pod可以通过Service的名称访问到后端的Pod，而Service则是通过Endpoint来实现这一功能的。
     </li>
     <li>
      <strong>
       集群外部服务访问
      </strong>
      ：在实际使用场景中，K8s集群有时需要访问集群外部的服务。这时，可以通过手动创建Endpoint或使用endpoints-operator等工具来自动管理外部服务的Endpoint，实现集群内部Pod对外部服务的访问。
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/83e3e0e04e8e488f92a9134c80fac29a.png#pic_center"/>
    </p>
    <h4>
     <a id="EndpointService_57">
     </a>
     六、Endpoint与Service的关系
    </h4>
    <p>
     在Kubernetes（K8s）中，Service和Endpoint是两个紧密相关但功能不同的资源。以下是它们之间的主要区别：
    </p>
    <h5>
     <a id="1_61">
     </a>
     1、定义与功能
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        Service
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         定义
        </strong>
        ：Service是K8s中的一个抽象层，它定义了一种访问Pod集合的方式。
       </li>
       <li>
        <strong>
         功能
        </strong>
        ：Service允许通过统一的接口访问一组执行相同功能的Pod，无论这些Pod如何变化、扩展或缩减。它提供了服务发现和负载分发的功能，通过负载均衡器将网络流量分发到后端的Pod集合。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Endpoint
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         定义
        </strong>
        ：Endpoint是K8s中的一种资源，用于描述Service的实际访问点。
       </li>
       <li>
        <strong>
         功能
        </strong>
        ：Endpoint包含了提供服务的Pod的IP地址和端口信息，这些信息是K8s实现服务发现和流量分发的关键依据。它像一座桥梁，连接了抽象的服务访问入口（Service）和实际提供服务的工作负载（Pod）。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="2_73">
     </a>
     2、创建与管理
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        Service
       </strong>
       ：
      </p>
      <ul>
       <li>
        通常通过YAML文件或kubectl命令创建。
       </li>
       <li>
        创建时，需要指定选择器（Selector）以匹配后端Pod。
       </li>
       <li>
        K8s会根据选择器自动发现匹配的Pod，并创建相应的Endpoint。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Endpoint
       </strong>
       ：
      </p>
      <ul>
       <li>
        在大多数情况下，Endpoint是自动创建的，与Service相关联。
       </li>
       <li>
        也可以手动创建Endpoint，但这通常用于特殊情况，如与外部非K8s管理的服务进行集成。
       </li>
       <li>
        Endpoint的信息由Endpoint控制器监控和更新，以确保与Service和Pod的状态保持一致。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="3_87">
     </a>
     3、关系与交互
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        关系
       </strong>
       ：
      </p>
      <ul>
       <li>
        Service是抽象层，定义了访问Pod集合的策略。
       </li>
       <li>
        Endpoint是实际实现，包含了Service的流量分发目标（即Pod的IP地址和端口）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        交互
       </strong>
       ：
      </p>
      <ul>
       <li>
        当外部请求访问Service时，K8s会根据Endpoint信息将请求路由到正确的Pod实例上。
       </li>
       <li>
        Service通过与Endpoint的映射关系，实现了服务的抽象化和负载均衡。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="4_99">
     </a>
     4、使用场景与特点
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        Service
       </strong>
       ：
      </p>
      <ul>
       <li>
        适用于需要访问一组Pod的场景，提供了负载均衡、服务发现和抽象等功能。
       </li>
       <li>
        支持多种访问模式，如ClusterIP、NodePort、LoadBalancer和ExternalName。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Endpoint
       </strong>
       ：
      </p>
      <ul>
       <li>
        主要用于描述Service的实际访问点，通常不需要直接管理。
       </li>
       <li>
        在特殊情况下，如与外部服务集成时，可能需要手动创建和管理Endpoint。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     Service和Endpoint在K8s中扮演着不同的角色。Service是抽象层，定义了访问Pod集合的策略；而Endpoint是实际实现，包含了Service的流量分发目标。它们通过紧密协作，实现了服务发现和负载均衡等功能，为K8s集群中的服务提供了高效、稳定的运行环境。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f341bc4c24554b6285b13d1aaa0fcf51.png#pic_center"/>
    </p>
    <h4>
     <a id="Endpointkubectl_115">
     </a>
     七、Endpoint的kubectl命令
    </h4>
    <p>
     与Kubernetes（K8s）中的Endpoint相关的
     <code>
      kubectl
     </code>
     命令主要用于查看、创建、编辑和删除Endpoint资源。以下是一些常用的
     <code>
      kubectl
     </code>
     命令及其用法：
    </p>
    <h5>
     <a id="1_Endpoint_119">
     </a>
     1. 查看Endpoint
    </h5>
    <p>
     要查看K8s集群中的Endpoint列表，可以使用以下命令：
    </p>
    <pre><code class="prism language-bash">kubectl get endpoints
</code></pre>
    <p>
     这将显示集群中所有Endpoint的列表，包括它们的名称、命名空间、IP地址和端口号等信息。如果只想查看特定命名空间中的Endpoint，可以添加
     <code>
      -n
     </code>
     选项，后面跟命名空间的名称：
    </p>
    <pre><code class="prism language-bash">kubectl get endpoints <span class="token parameter variable">-n</span> your-namespace
</code></pre>
    <h5>
     <a id="2_Endpoint_133">
     </a>
     2. 创建Endpoint
    </h5>
    <p>
     虽然Endpoint通常是由K8s自动创建的，与Service相关联，但在某些特殊情况下，也可以手动创建Endpoint。这通常涉及编写一个YAML文件来描述Endpoint资源，然后使用
     <code>
      kubectl apply
     </code>
     命令将其应用到集群中。
    </p>
    <p>
     以下是一个创建Endpoint的YAML文件示例：
    </p>
    <pre><code class="prism language-yaml"><span class="token key atrule">apiVersion</span><span class="token punctuation">:</span> v1
<span class="token key atrule">kind</span><span class="token punctuation">:</span> Endpoints
<span class="token key atrule">metadata</span><span class="token punctuation">:</span>
  <span class="token key atrule">name</span><span class="token punctuation">:</span> my<span class="token punctuation">-</span>endpoint
<span class="token key atrule">subsets</span><span class="token punctuation">:</span>
  <span class="token punctuation">-</span> <span class="token key atrule">addresses</span><span class="token punctuation">:</span>
      <span class="token punctuation">-</span> <span class="token key atrule">ip</span><span class="token punctuation">:</span> 192.168.1.1
    <span class="token key atrule">ports</span><span class="token punctuation">:</span>
      <span class="token punctuation">-</span> <span class="token key atrule">port</span><span class="token punctuation">:</span> <span class="token number">80</span>
        <span class="token key atrule">protocol</span><span class="token punctuation">:</span> TCP
</code></pre>
    <p>
     应用这个YAML文件到集群中：
    </p>
    <pre><code class="prism language-bash">kubectl apply <span class="token parameter variable">-f</span> my-endpoint.yaml
</code></pre>
    <h5>
     <a id="3_Endpoint_158">
     </a>
     3. 编辑Endpoint
    </h5>
    <p>
     要编辑现有的Endpoint资源，可以使用
     <code>
      kubectl edit
     </code>
     命令。这将打开一个文本编辑器（通常是系统默认的编辑器，如vim或nano），允许你修改Endpoint的相关信息。
    </p>
    <pre><code class="prism language-bash">kubectl edit endpoints my-endpoint
</code></pre>
    <p>
     在编辑器中，你可以找到和修改Endpoints的相关信息，如subsets、addresses和ports。完成编辑后，保存并退出编辑器，K8s将应用你的更改。
    </p>
    <h5>
     <a id="4_Endpoint_168">
     </a>
     4. 删除Endpoint
    </h5>
    <p>
     要删除特定的Endpoint资源，可以使用
     <code>
      kubectl delete endpoints
     </code>
     命令。这将从K8s集群中移除指定的Endpoint。
    </p>
    <pre><code class="prism language-bash">kubectl delete endpoints my-endpoint
</code></pre>
    <p>
     请注意，手动创建、编辑和删除Endpoint通常不是K8s的标准做法。在大多数情况下，K8s会自动管理Endpoint资源，以确保它们与Service和Pod的状态保持一致。手动操作Endpoint可能会导致服务发现和负载均衡问题，因此应谨慎进行。
    </p>
    <p>
     Endpoint在K8s中扮演着至关重要的角色，它实现了Service与后端Pod的关联、服务发现和负载均衡等功能。通过合理管理和使用Endpoint，可以确保K8s集群中的服务能够高效、稳定地运行。
    </p>
