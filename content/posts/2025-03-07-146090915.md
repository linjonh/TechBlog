     redis热key

    <p>
     在 Redis 中，
     <strong>
      热 Key（Hot Key）
     </strong>
     是指被频繁访问的 Key，可能会导致以下问题：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        性能瓶颈
       </strong>
       ：单个 Redis 实例的 CPU 或网络带宽被耗尽。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数据倾斜
       </strong>
       ：在 Redis 集群中，热 Key 可能导致某个节点的负载过高。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缓存击穿
       </strong>
       ：热 Key 过期或被删除时，大量请求直接打到数据库，导致数据库压力骤增。
      </p>
     </li>
    </ul>
    <p>
     以下是处理热 Key 的常见方法和最佳实践：
    </p>
    <hr/>
    <h4>
     1.
     <strong>
      检测热 Key
     </strong>
    </h4>
    <h5>
     <strong>
      1.1 使用 Redis 自带的命令
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         redis-cli --hotkeys
        </code>
       </strong>
       （Redis 6.0 及以上版本）：
      </p>
      <ul>
       <li>
        <p>
         扫描 Redis 实例中的热 Key。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         bash
        </p>
        <pre>redis-cli --hotkeys</pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         MONITOR
        </code>
        命令
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         实时监控 Redis 的命令执行情况，找出频繁访问的 Key。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         bash
        </p>
        <pre>redis-cli MONITOR</pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      1.2 使用第三方工具
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Redis 慢查询日志
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         通过分析慢查询日志，找出频繁访问的 Key。
        </p>
       </li>
       <li>
        <p>
         配置慢查询日志：
        </p>
        <p>
         bash
        </p>
        <pre>CONFIG SET slowlog-log-slower-than 1000  -- 记录超过 1ms 的查询
CONFIG SET slowlog-max-len 1000         -- 最多记录 1000 条慢查询
SLOWLOG GET                             -- 查看慢查询日志</pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        监控工具
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         使用 Prometheus、Grafana 等监控工具，分析 Redis 的访问模式。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     2.
     <strong>
      处理热 Key 的方法
     </strong>
    </h4>
    <h5>
     <strong>
      2.1 本地缓存
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        在应用层缓存热 Key 的数据
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         使用本地缓存（如 Guava Cache、Caffeine）缓存热 Key 的数据，减少对 Redis 的访问。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         java
        </p>
        <pre>Cache&lt;String, String&gt; localCache = Caffeine.newBuilder()
    .maximumSize(1000)
    .expireAfterWrite(10, TimeUnit.MINUTES)
    .build();

String value = localCache.get(key, k -&gt; redis.get(k));</pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      2.2 拆分热 Key
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        将热 Key 拆分为多个子 Key
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         例如，将一个热 Key 拆分为多个子 Key，分散访问压力。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         bash
        </p>
        <pre># 原始 Key
GET hotkey

# 拆分后的 Key
GET hotkey:part1
GET hotkey:part2
GET hotkey:part3</pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      2.3 使用 Redis 集群
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        将热 Key 分散到多个 Redis 实例
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         在 Redis 集群中，使用哈希标签（Hash Tag）将热 Key 分散到多个节点。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         bash
        </p>
        <pre># 使用哈希标签
SET {hotkey}:part1 value1
SET {hotkey}:part2 value2
SET {hotkey}:part3 value3</pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      2.4 限流和降级
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        对热 Key 的访问进行限流
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         使用限流算法（如令牌桶、漏桶）限制对热 Key 的访问频率。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         java
        </p>
        <pre>RateLimiter rateLimiter = RateLimiter.create(100);  // 每秒允许 100 次访问
if (rateLimiter.tryAcquire()) {
    String value = redis.get(hotkey);
} else {
    // 降级处理
}</pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        降级处理
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         当热 Key 的访问压力过大时，返回默认值或错误信息，避免系统崩溃。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      2.5 异步更新缓存
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        使用异步任务更新热 Key
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         当热 Key 过期时，使用异步任务更新缓存，避免大量请求直接打到数据库。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         java
        </p>
        <pre>String value = redis.get(hotkey);
if (value == null) {
    executor.submit(() -&gt; {
        String dbValue = loadFromDatabase(hotkey);
        redis.set(hotkey, dbValue);
    });
    value = getDefaultValue();
}</pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      2.6 设置合理的过期时间
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        避免热 Key 同时过期
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         为热 Key 设置随机的过期时间，避免大量 Key 同时过期导致缓存击穿。
        </p>
       </li>
       <li>
        <p>
         示例：
        </p>
        <p>
         java
        </p>
        <pre>int ttl = 3600 + new Random().nextInt(600);  // 过期时间在 3600~4200 秒之间
redis.setex(hotkey, ttl, value);</pre>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     3.
     <strong>
      预防热 Key 的产生
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        设计时考虑访问模式
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         在设计系统时，预估数据的访问模式，避免单个 Key 被频繁访问。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用多级缓存
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         结合本地缓存和分布式缓存，减少对 Redis 的直接访问。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        监控和预警
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         实时监控 Redis 的访问情况，及时发现和处理热 Key。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     4.
     <strong>
      总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        检测热 Key
       </strong>
       ：使用
       <code>
        redis-cli --hotkeys
       </code>
       、
       <code>
        MONITOR
       </code>
       命令或监控工具。
      </p>
     </li>
     <li>
      <p>
       <strong>
        处理热 Key
       </strong>
       ：本地缓存、拆分热 Key、使用 Redis 集群、限流和降级、异步更新缓存。
      </p>
     </li>
     <li>
      <p>
       <strong>
        预防热 Key
       </strong>
       ：设计时考虑访问模式，使用多级缓存，监控和预警。
      </p>
     </li>
    </ul>
