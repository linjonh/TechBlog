    <p>
     在系统开发过程中，如何优雅地管理错误信息一直是个棘手问题。传统的错误处理方式往往存在不统一、难以维护等缺点。而
     <strong>
      errcode
     </strong>
     模块通过对错误码进行规范化管理，为系统级和业务级错误提供了统一的编码标准。本文将带您深入了解 errcode 的设计原理、错误码结构以及详细的使用示例，帮助您构建高效稳定的错误处理机制。
    </p>
    <h3>
     <a id="errcode__3">
     </a>
     一、errcode 模块设计原理解析
    </h3>
    <p>
     在一个复杂系统中，错误码不仅用于指示错误类型，更承载着对错误原因、模块归属以及自定义描述的综合信息。errcode 模块规定了总共六位十进制数字错误码，并按照以下结构进行划分：
    </p>
    <ul>
     <li>
      <strong>
       第一位数字
      </strong>
      ：表示错误所属的协议类型，如
      <code>
       1
      </code>
      表示 HTTP 系统级错误、
      <code>
       2
      </code>
      表示 HTTP 业务级错误、
      <code>
       3
      </code>
      表示 gRPC 系统级错误、
      <code>
       4
      </code>
      表示 gRPC 业务级错误。
     </li>
     <li>
      <strong>
       中间三位数字
      </strong>
      ：标识错误发生的模块或表，范围为 1 至 1000，帮助开发者快速定位错误来源。
     </li>
     <li>
      <strong>
       最后两位数字
      </strong>
      ：用于自定义错误细节，范围为 1 至 100。
     </li>
    </ul>
    <p>
     此外，errcode 模块还对不同服务类型设定了错误码的具体取值范围：
    </p>
    <ul>
     <li>
      <strong>
       HTTP 服务
      </strong>
      <ul>
       <li>
        系统级错误码范围：100000 ~ 200000
       </li>
       <li>
        业务级错误码范围：200000 ~ 300000
       </li>
      </ul>
     </li>
     <li>
      <strong>
       gRPC 服务
      </strong>
      <ul>
       <li>
        系统级错误码范围：300000 ~ 400000
       </li>
       <li>
        业务级错误码范围：400000 ~ 500000
       </li>
      </ul>
     </li>
    </ul>
    <p>
     这种设计不仅使错误码具备清晰的层次结构，还能在跨系统、跨模块的场景下实现高效、统一的错误管理。
    </p>
    <h3>
     <a id="errcode__22">
     </a>
     二、errcode 模块的实际使用示例
    </h3>
    <p>
     errcode 模块在使用上极为简便，并支持对错误消息进行动态重写，以便更好地向客户端传递友好信息。下面将通过 HTTP 和 gRPC 两种场景的代码示例，演示如何在实际项目中集成和应用 errcode 模块。
    </p>
    <h4>
     <a id="1_HTTP__26">
     </a>
     1. HTTP 错误码示例
    </h4>
    <p>
     在基于 SQL 构建的 Web 服务(gin)中，我们可以使用 errcode 来返回统一格式的错误码。示例代码如下：
    </p>
    <pre><code class="prism language-go"><span class="token keyword">package</span> main

<span class="token keyword">import</span> <span class="token punctuation">(</span>
    <span class="token string">"github.com/gin-gonic/gin"</span>
    <span class="token string">"github.com/go-dev-frame/sponge/pkg/gin/response"</span>
    <span class="token string">"github.com/go-dev-frame/sponge/pkg/errcode"</span>
<span class="token punctuation">)</span>

<span class="token keyword">func</span> <span class="token function">handler</span><span class="token punctuation">(</span>c <span class="token operator">*</span>gin<span class="token punctuation">.</span>Context<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 返回默认错误</span>
    response<span class="token punctuation">.</span><span class="token function">Error</span><span class="token punctuation">(</span>c<span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>InvalidParams<span class="token punctuation">)</span>
    
    <span class="token comment">// 自定义错误消息</span>
    response<span class="token punctuation">.</span><span class="token function">Error</span><span class="token punctuation">(</span>c<span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>InvalidParams<span class="token punctuation">.</span><span class="token function">RewriteMsg</span><span class="token punctuation">(</span><span class="token string">"参数格式不正确"</span><span class="token punctuation">)</span><span class="token punctuation">)</span>
    
    <span class="token comment">// 转换为标准 HTTP 状态码输出</span>
    response<span class="token punctuation">.</span><span class="token function">Out</span><span class="token punctuation">(</span>c<span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>InvalidParams<span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在这里，
     <code>
      InvalidParams
     </code>
     是一个预定义的错误码（100001）。
     <code>
      response.Error
     </code>
     直接返回错误，而
     <code>
      RewriteMsg
     </code>
     允许动态调整提示信息。
     <code>
      response.Out
     </code>
     则将错误码映射为 HTTP 状态码（如 400 Bad Request）。
    </p>
    <h4>
     <a id="2_gRPC__53">
     </a>
     2. gRPC 错误码示例
    </h4>
    <p>
     对于 gRPC 服务，同样可以使用 errcode 模块管理错误。示例代码展示了如何在返回错误时选择不同的转换方法：
    </p>
    <pre><code class="prism language-go"><span class="token keyword">package</span> main

<span class="token keyword">import</span> <span class="token punctuation">(</span>
    <span class="token string">"github.com/go-dev-frame/sponge/pkg/errcode"</span>
<span class="token punctuation">)</span>

<span class="token keyword">func</span> <span class="token punctuation">(</span>s <span class="token operator">*</span>server<span class="token punctuation">)</span> <span class="token function">SomeMethod</span><span class="token punctuation">(</span>ctx context<span class="token punctuation">.</span>Context<span class="token punctuation">,</span> req <span class="token operator">*</span>pb<span class="token punctuation">.</span>Request<span class="token punctuation">)</span> <span class="token punctuation">(</span><span class="token operator">*</span>pb<span class="token punctuation">.</span>Response<span class="token punctuation">,</span> <span class="token builtin">error</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 返回默认错误</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">Err</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
    
    <span class="token comment">// 自定义消息</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">Err</span><span class="token punctuation">(</span><span class="token string">"参数校验失败"</span><span class="token punctuation">)</span>
    
    <span class="token comment">// 将错误码转换为标准 gRPC 状态码</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">ToRPCErr</span><span class="token punctuation">(</span><span class="token punctuation">)</span>

    <span class="token comment">// 将错误码转换为标准 gRPC 状态码并自定义消息</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">ToRPCErr</span><span class="token punctuation">(</span><span class="token string">"自定义错误信息"</span><span class="token punctuation">)</span>

    <span class="token comment">// ---------------------</span>

    <span class="token comment">// 如果是 gRPC 网关调用，转为 HTTP 状态码</span>
    <span class="token comment">// 将错误码转换为标准 HTTP 状态码</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">ErrToHTTP</span><span class="token punctuation">(</span><span class="token punctuation">)</span>

    <span class="token comment">// 重写错误消息并转换为 HTTP 状态码</span>
    <span class="token keyword">return</span> <span class="token boolean">nil</span><span class="token punctuation">,</span> errcode<span class="token punctuation">.</span>StatusInvalidParams<span class="token punctuation">.</span><span class="token function">ErrToHTTP</span><span class="token punctuation">(</span><span class="token string">"自定义错误信息"</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     <code>
      StatusInvalidParams
     </code>
     （预定义的错误码300003）会被转换为 gRPC 的标准状态码（如
     <code>
      InvalidArgument
     </code>
     ）。如果需要与 HTTP 兼容，还可以用
     <code>
      ErrToHTTP
     </code>
     。
    </p>
    <h3>
     <a id="_errcode__90">
     </a>
     三、为何选择 errcode 模块？
    </h3>
    <p>
     在实际项目开发中，错误处理往往是系统稳定性的关键。使用 errcode 模块的主要优势包括：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        统一性与标准化
       </strong>
       <br/>
       通过统一的错误码结构，使得开发者在跨模块、跨服务时可以快速定位问题，降低了错误处理的复杂度。
      </p>
     </li>
     <li>
      <p>
       <strong>
        灵活性
       </strong>
       <br/>
       支持重写错误消息和状态码转换，能够根据不同业务场景返回更加友好、易于理解的错误信息。
      </p>
     </li>
     <li>
      <p>
       <strong>
        可扩展性
       </strong>
       <br/>
       错误码设计预留了足够的扩展空间，无论是系统级错误还是业务级错误，都能轻松适配新需求，避免了后续维护中的混乱。
      </p>
     </li>
    </ul>

     四、总结
    </h3>
