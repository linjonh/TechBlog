    <p>
     <code>
      application.yml
     </code>
     和
     <code>
      bootstrap.yml
     </code>
     文件都是 Spring Boot 项目中用于配置应用程序的 YAML 或 Properties 文件，但它们在
     <strong>
      加载顺序、用途和优先级
     </strong>
     上存在关键的区别。
    </p>
    <p>
     <strong>
      核心区别概览:
     </strong>
    </p>
    <table>
     <thead>
      <tr>
       <th>

       </th>
       <th>
        <code>
         bootstrap.yml
        </code>
        /
        <code>

        </code>
       </th>
       <th>
        <code>
         application.yml
        </code>
        /
        <code>
         application.properties
        </code>
       </th>
      </tr>

      </tr>

        </strong>

    <p>
     <strong>
      详细解释和区别：
     </strong>
    </p>
    <p>
     <strong>
      1. 加载时机 (Loading Order):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       (Bootstrap Context):
      </strong>
      在 Spring Boot 应用程序启动的
      <strong>
       非常早期
      </strong>
      阶段加载。 这是通过
      <strong>
       Bootstrap Context
      </strong>
      实现的，它是一个特殊的父级 Spring 上下文，在主 Application Context 之前创建。
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       (Application Context):
      </strong>
      在
      <code>
       bootstrap.yml
      </code>
      加载之后，
      <strong>
       在 Application Context 初始化时
      </strong>
      加载。 这是 Spring Boot 应用的主上下文。
     </li>
    </ul>
    <p>
     <strong>
      理解 Bootstrap Context 和 Application Context:
     </strong>
    </p>
    <p>
     你可以将 Bootstrap Context 理解为 Spring Boot 启动的
     <strong>
      引导阶段
     </strong>
     ，它负责初始化一些
     <strong>
      基础且关键
     </strong>
     的配置，例如：
    </p>
    <ul>
     <li>
      <strong>
       配置 Spring Cloud Config Client 或 Nacos Config Client 的连接信息:
      </strong>
      Bootstrap Context 需要先知道如何连接配置中心，才能加载外部化配置。
     </li>
     <li>
      <strong>
       加载共享配置:
      </strong>
      在多个应用之间共享的配置通常需要在 Bootstrap Context 中加载，以便在应用启动的早期阶段就可用。
     </li>
     <li>
      <strong>
       设置
       <code>
        spring.application.name
       </code>
       和
       <code>
        spring.profiles.active
       </code>
       等基础属性:
      </strong>
      这些属性通常需要在 Bootstrap Context 中确定，以便后续的配置加载和应用初始化能够正确进行。
     </li>
    </ul>
    <p>
     Application Context 则是 Spring Boot 应用的
     <strong>
      主应用上下文
     </strong>
     ，负责加载和管理应用程序中的所有 Bean、处理请求、执行业务逻辑等。
     <code>
      application.yml
     </code>
     主要用于配置应用程序自身的通用设置。
    </p>
    <p>
     <strong>
      2. 用途 (Purpose):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       : 引导引导 (Bootstrapping) 和 外部化配置:
      </strong>
      <ul>
       <li>
        <strong>
         配置 Spring Cloud Config Client 或 Nacos Config Client:
        </strong>
        例如
        <code>
         spring.cloud.config.uri
        </code>
        或
        <code>
         spring.cloud.nacos.config.server-addr
        </code>
        等配置。
       </li>
       <li>
        <strong>
         加载共享配置 (
         <code>
          sharedConfigs
         </code>
         ):
        </strong>
        使用 Spring Cloud Alibaba Nacos Config 的
        <code>
         sharedConfigs
        </code>
        功能时，需要在
        <code>
         bootstrap.yml
        </code>
        中配置。
       </li>
       <li>
        <strong>
         配置应用程序名称 (
         <code>
          spring.application.name
         </code>
         ):
        </strong>
        通常在
        <code>
         bootstrap.yml
        </code>
        中设置，用于标识应用，方便配置中心识别。
       </li>
       <li>
        <strong>
         激活 Profile (
         <code>
          spring.profiles.active
         </code>
         ):
        </strong>
        在
        <code>
         bootstrap.yml
        </code>
        中激活 Profile，可以影响后续的配置加载。
       </li>
       <li>
        <strong>
         其他需要在应用启动早期就加载的配置:
        </strong>
        例如，一些自定义的 Bootstrap 监听器或初始化器需要的配置。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       : 应用程序通用配置:
      </strong>
      <ul>
       <li>
        <strong>
         应用程序自身的通用配置:
        </strong>
        例如，端口号、数据源连接信息 (如果不是共享的)、日志级别、国际化配置、Thymeleaf 模板配置、Spring MVC 配置、Bean 的定义等。
       </li>
       <li>
        <strong>
         业务逻辑相关的配置:
        </strong>
        例如，一些业务开关、参数配置等。
       </li>
       <li>
        <strong>
         大部分应用程序的配置都应该放在
         <code>
          application.yml
         </code>
         中。
        </strong>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      3. 上下文 (Context):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       : Bootstrap Context
      </strong>
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       : Application Context
      </strong>
     </li>
    </ul>
    <p>
     这意味着
     <code>
      bootstrap.yml
     </code>
     中的配置主要作用于 Bootstrap Context，而
     <code>
      application.yml
     </code>
     中的配置作用于 Application Context。 Bootstrap Context 的配置会影响 Application Context 的创建和初始化。
    </p>
    <p>
     <strong>
      4. 优先级 (Priority):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       的优先级更高 (在 Bootstrap Context 中):
      </strong>
      Bootstrap Context 的配置会优先加载，并影响 Application Context 的创建。
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       的优先级较低 (在 Application Context 中):
      </strong>
      Application Context 的配置在 Bootstrap Context 之后加载。
     </li>
    </ul>
    <p>
     <strong>
      5. 配置来源 (Configuration Sources):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       : 倾向于配置
      </strong>
      外部配置来源**，例如 Spring Cloud Config Server, Nacos Config Server 等。 也用于配置一些需要在引导阶段就确定的
      <strong>
       基础属性
      </strong>
      。
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       : 主要用于配置
      </strong>
      应用自身**的配置，通常来源于本地文件系统、命令行参数、环境变量等。
     </li>
    </ul>
    <p>
     <strong>
      6. 适用场景 (Use Cases):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        bootstrap.yml
       </code>
       /
       <code>
        bootstrap.properties
       </code>
       : 适用于:
      </strong>
      <ul>
       <li>
        配置 Spring Cloud Config 或 Nacos Config Client 连接信息。
       </li>
       <li>
        配置
        <code>
         sharedConfigs
        </code>
        (Nacos Config)。
       </li>
       <li>
        设置
        <code>
         spring.application.name
        </code>
        和
        <code>
         spring.profiles.active
        </code>
        。
       </li>
       <li>
        配置需要在 Bootstrap Context 中使用的属性。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       /
       <code>
        application.properties
       </code>
       : 适用于:
      </strong>
      <ul>
       <li>
        配置应用程序的通用设置。
       </li>
       <li>
        定义 Bean。
       </li>
       <li>
        配置业务逻辑相关的参数。
       </li>
       <li>
        大部分应用程序的配置。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      7. 配置覆盖 (Configuration Overriding):
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        application.yml
       </code>
       中的配置会覆盖
       <code>
        bootstrap.yml
       </code>
       中
      </strong>
      相同属性的配置。 这是因为
      <code>
       application.yml
      </code>
      在
      <code>
       bootstrap.yml
      </code>
      之后加载。
     </li>
     <li>
      <strong>
       命令行参数、环境变量等
      </strong>
      可以进一步覆盖
      <code>
       application.yml
      </code>
      和
      <code>
       bootstrap.yml
      </code>
      中的配置。 Spring Boot 的配置优先级顺序是：
      <ol>
       <li>
        命令行参数
       </li>
       <li>
        环境变量
       </li>
       <li>
        <code>
         application.yml
        </code>
        /
        <code>
         application.properties
        </code>
        (profile-specific 和 default)
       </li>
       <li>
        <code>
         bootstrap.yml
        </code>
        /
        <code>
         bootstrap.properties
        </code>
        (profile-specific 和 default)
       </li>
       <li>
        默认配置 (Spring Boot 框架内部的默认值)
       </li>
      </ol>
     </li>
    </ul>
    <p>
     <strong>
      8. 默认位置 (Default Location):
     </strong>
    </p>
    <ul>
     <li>
      <code>
       application.yml
      </code>
      和
      <code>
       bootstrap.yml
      </code>
      的默认位置都是
      <code>
       src/main/resources/
      </code>
      目录下。
     </li>
    </ul>
    <p>
     <strong>
      9. 是否必须存在 (Required or Optional):
     </strong>
    </p>
    <ul>
     <li>
      <code>
       application.yml
      </code>
      和
      <code>
       bootstrap.yml
      </code>
      都不是
      <strong>
       必须
      </strong>
      存在的。 Spring Boot 应用在没有这些配置文件的情况下也可以启动，它会使用默认配置。
     </li>
     <li>
      然而，在实际开发中，为了进行各种自定义配置，
      <code>
       application.yml
      </code>
      (或
      <code>
       application.properties
      </code>
      ) 通常是
      <strong>
       必需
      </strong>
      存在的。
      <code>
       bootstrap.yml
      </code>
      (或
      <code>
       bootstrap.properties
      </code>
      ) 则在需要使用 Spring Cloud Config 或 Nacos Config 等外部化配置管理时才
      <strong>
       常用
      </strong>
      。
     </li>
    </ul>
    <p>
     <strong>
      总结:
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       如果你不需要使用 Spring Cloud Config 或 Nacos Config 等外部化配置管理，那么你可能只需要
       <code>
        application.yml
       </code>
       (或
       <code>
        application.properties
       </code>
       ) 文件。
      </strong>
      大部分应用程序的配置都放在
      <code>
       application.yml
      </code>
      中即可。
     </li>
     <li>
      <strong>
       如果你需要使用 Spring Cloud Config 或 Nacos Config 等外部化配置管理，或者需要在应用启动的早期阶段加载一些关键配置 (例如共享配置、应用名、Profile 等)，那么你就需要使用
       <code>
        bootstrap.yml
       </code>
       (或
       <code>
        bootstrap.properties
       </code>
       ) 文件。
      </strong>
      <code>
       bootstrap.yml
      </code>
      主要用于配置引导阶段的配置，以及外部化配置的连接信息和共享配置。
     </li>
    </ul>
    <p>
     <strong>
      简单记忆口诀:
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       Bootstrap 先行，Application 后续:
      </strong>
      <code>
       bootstrap.yml
      </code>
      先加载，
      <code>
       application.yml
      </code>
      后加载。
     </li>
     <li>
      <strong>
       Bootstrap 引导，Application 应用:
      </strong>
      <code>
       bootstrap.yml
      </code>
      负责引导启动，
      <code>
       application.yml
      </code>
      负责应用配置。
     </li>
     <li>
      <strong>
       Bootstrap 外部，Application 自身:
      </strong>
      <code>
       bootstrap.yml
      </code>
      配置外部配置，
      <code>
       application.yml
      </code>
      配置自身应用。
     </li>
    </ul>
