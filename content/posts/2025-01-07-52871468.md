# SQL Server数据库结构（MSSQL入门）

**学习数据库，先了解一下sql server 的界面，毕竟人性化的界面可以帮助初学者快速上手这门技术。**

**数据库有系统数据库、数据库快照、reportserver 、reportserver tempDB

和自定义数据库组成（这里的例子：employee、mysql、students）。**

**注意：SQL Server有很多个版本，一些快速“上手版”以及“单机版”为了缩小文件大小会相应地减少一些内容；**

**数据库快照需要自己做，好比windows系统的备份；**

**刚开始学习建议下载个官方提供的示例数据库，看看人家的关系设置和命令规范；**

**这里提供2008/2008R2示例数据库的下载（http://download.csdn.net/detail/yang\_jesus/9659014），

其他版本也可以尝试使用，关于SQL Server的下载安装和示例数据库的添加在SQL文章中有。**

**我把它分为四部分：系统数据库，数据库快照，ssrs，自定义数据库**

**![](https://img-my.csdn.net/uploads/201206/20/1340158634_8115.png)**

**一、系统数据库**

**![](https://img-my.csdn.net/uploads/201206/20/1340158660_9897.png)
  
master 数据库：记录所有系统信息，：登陆，系统设置，初 始化信息，其他系统数据库及用户的信息。**

**model 数据库，所有 数据库的模板数据库。可以根据自己的需要修改。一直存在在SQl 中。**

**msdb 数据库：代理服务数据库。**

**tempdb 数据库：临时数据库，临时存储空间。只有一个，数据多时，可以自行调整。当sql server 关闭时，数据就会丢失。**

**二、数据库快照**

**数据库快照是数据库（源数据库）的只读、静态视图。多个快照可以位于一个源数据库中，并且可以作为数据库始终驻留在同一服务器实例上。创建快照时，每个数据库快照在事务上与源数据库一致。在被数据库所有者显式删除之前，快照始终存在。**

**三、ssrs**

**数据库发布报表的工具。Reportserver tempDB数据库是reportserver数据库的临时缓冲数据库。**

**![](https://img-my.csdn.net/uploads/201206/20/1340158696_7445.png)**

**四、自定义数据库：**

**employee，students，mysql**

**系统数据库vs自定义数据库**

**系统：**

**![](https://img-my.csdn.net/uploads/201206/20/1340158728_1910.png)**

**自定义**

**![](https://img-my.csdn.net/uploads/201206/20/1340158786_1201.png)**

**同义词相当于对象的别名,起著连结

数据库对象

和

应用程序

的作用,假如对象需要更换,则不用修改应用程序而直接修改同义词就可以了。目前只是初级的数据库学习阶段的话，貌似用不到。**

**Service broker：也是跟服务器和外部服务器工作有关的，可以简化工作的工具。**

**初学者常用到的也就是表，视图，可编程性，安全性里的一些东西。**

**不同：**

**1、系统数据库没有‘数据库关系图’——多表组成的关系图。**

**2、数据库对象；系统只有系统表，系统视图。自定义有包含系统的，同时有自定义的表跟视图。**

**3、可编程性：系统中多了一个扩展存储过程。扩展存储过程使您能够在编程语言（例如，C）中创建自己的外部例程。**

**在SQL server中，数据库就看完了，在我的脑海中也就不是一堆一堆的长长的陌生的，系统的非系统的，还有神马表，约束，角色之类的东东了。这样一分析，我想用什么就可以找到什么，虽然有系统非系统之分，可是我们知道什么我们用的着什么我们用不着，还有就是这样一比较之后，我们只需要多留意一下不同点，相同的就省去了重复记忆的麻烦了。**
