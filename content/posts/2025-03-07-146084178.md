    <h1 class="title-article" id="articleContentId">
     UDP学习笔记（一）为什么UDP需要先将数据转换为字节数组
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     UDP 发送数据时需要
     <strong>
      先将数据转换为字节数组
     </strong>
     再发送，主要是因为
     <strong>
      计算机网络传输的最基本单位是“字节”
     </strong>
     （Byte）。让我们从以下几个方面来深入理解这个设计选择：
    </p>
    <hr/>
    <h3>
     <a id="1__4">
     </a>
     <strong>
      1. 计算机网络只能传输“字节”
     </strong>
    </h3>
    <p>
     在网络通信中，无论是 TCP 还是 UDP，最终传输的都是
     <strong>
      二进制数据
     </strong>
     。计算机不能直接传输字符串、整数或对象，而是需要
     <strong>
      转换为字节数组
     </strong>
     。
    </p>
    <p>
     📌
     <strong>
      示例：发送字符串
      <code>
       "Hello"
      </code>
     </strong>
    </p>
    <pre><code class="prism language-kotlin"><span class="token keyword">val</span> <span class="token keyword">data</span> <span class="token operator">=</span> <span class="token string-literal singleline"><span class="token string">"Hello"</span></span><span class="token punctuation">.</span><span class="token function">toByteArray</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token comment">// 转换为字节数组</span>
</code></pre>
    <ul>
     <li>
      计算机存储
      <code>
       "Hello"
      </code>
      不是存字符串，而是存储对应的
      <strong>
       ASCII/UTF-8 字节
      </strong>
      ：
      <pre><code>H -&gt; 72 (0x48)
e -&gt; 101 (0x65)
l -&gt; 108 (0x6C)
l -&gt; 108 (0x6C)
o -&gt; 111 (0x6F)
</code></pre>
     </li>
     <li>
      <code>
       toByteArray()
      </code>
      生成的字节数组：
      <pre><code class="prism language-kotlin"><span class="token punctuation">[</span><span class="token number">72</span><span class="token punctuation">,</span> <span class="token number">101</span><span class="token punctuation">,</span> <span class="token number">108</span><span class="token punctuation">,</span> <span class="token number">108</span><span class="token punctuation">,</span> <span class="token number">111</span><span class="token punctuation">]</span>
</code></pre>
     </li>
     <li>
      这些字节才是
      <strong>
       真正可以通过 UDP 发送的数据
      </strong>
      。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2_UDP__27">
     </a>
     <strong>
      2. UDP 只是负责“搬运”字节数据
     </strong>
    </h3>
    <p>
     UDP 协议本质上是一个
     <strong>
      简单的“快递员”
     </strong>
     ，它的任务是：
    </p>
    <ol>
     <li>
      <strong>
       接收
      </strong>
      一串二进制数据（字节数组）。
     </li>
     <li>
      <strong>
       打包
      </strong>
      数据，加上 IP 头、UDP 头（源端口、目标端口、数据长度等）。
     </li>
     <li>
      <strong>
       发送
      </strong>
      到网络。
     </li>
    </ol>
    <p>
     UDP 不关心数据的类型，它只知道：
    </p>
    <ul>
     <li>
      这是一串二进制数据（字节数组）。
     </li>
     <li>
      需要发送到某个 IP 和端口。
     </li>
    </ul>
    <p>
     这就像：
    </p>
    <ul>
     <li>
      <strong>
       快递员不管你寄的是书、本子还是手机
      </strong>
      ，他们只在乎包装好的箱子（字节数组）。
     </li>
     <li>
      <strong>
       UDP 也不管数据是不是字符串、图片或视频
      </strong>
      ，它只是传输“字节”。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="3__43">
     </a>
     <strong>
      3. 统一格式，适用于所有数据类型
     </strong>
    </h3>
    <p>
     因为
     <strong>
      所有数据最终都能转换成字节数组
     </strong>
     ，所以 UDP 传输时不需要关心数据格式。例如：
    </p>
    <ul>
     <li>
      <strong>
       字符串
      </strong>
      ：转换为字节数组
      <code>
       toByteArray()
      </code>
     </li>
     <li>
      <strong>
       整数
      </strong>
      ：转换为字节数组
      <code>
       ByteBuffer.allocate(4).putInt(value).array()
      </code>
     </li>
     <li>
      <strong>
       图片
      </strong>
      ：JPEG/PNG 本质上也是字节数组
     </li>
     <li>
      <strong>
       音视频流
      </strong>
      ：H.264/MP3 等数据流都是字节数组
     </li>
    </ul>
    <p>
     不管是什么数据，
     <strong>
      只要能变成字节数组，UDP 就能传输
     </strong>
     。
    </p>
    <p>
     📌
     <strong>
      示例：发送整数
      <code>
       1234
      </code>
     </strong>
    </p>
    <pre><code class="prism language-kotlin"><span class="token keyword">val</span> buffer <span class="token operator">=</span> ByteBuffer<span class="token punctuation">.</span><span class="token function">allocate</span><span class="token punctuation">(</span><span class="token number">4</span><span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">putInt</span><span class="token punctuation">(</span><span class="token number">1234</span><span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">array</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</code></pre>
    <ul>
     <li>
      结果是：
      <pre><code class="prism language-kotlin"><span class="token punctuation">[</span><span class="token number">0</span><span class="token punctuation">,</span> <span class="token number">0</span><span class="token punctuation">,</span> <span class="token number">4</span><span class="token punctuation">,</span> <span class="token number">210</span><span class="token punctuation">]</span> <span class="token comment">// 这是 1234 的二进制表示</span>
</code></pre>
     </li>
     <li>
      这个字节数组就可以通过 UDP 发送了。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="4__TCP_64">
     </a>
     <strong>
      4. 其他协议（如 TCP）也是传输字节
     </strong>
    </h3>
    <p>
     不仅是 UDP，
     <strong>
      所有网络协议最终传输的都是字节
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       HTTP 发送网页数据
      </strong>
      ：HTML 文件也是字节数据。
     </li>
     <li>
      <strong>
       FTP 传输文件
      </strong>
      ：文件本质上也是字节流。
     </li>
     <li>
      <strong>
       WebSocket 传输消息
      </strong>
      ：消息数据同样要转换为字节数组。
     </li>
    </ul>
    <p>
     无论是
     <strong>
      字符串、图片、视频、音频
     </strong>
     ，在网络上传输时都必须转换为字节流，这样才能统一处理。
    </p>
    <hr/>
    <h3>
     <a id="5__74">
     </a>
     <strong>
      5. 提高数据传输的灵活性
     </strong>
    </h3>
    <p>
     如果 UDP 允许直接发送
     <strong>
      字符串、整数、对象
     </strong>
     ，那么：
    </p>
    <ul>
     <li>
      每种数据类型的
      <strong>
       发送方式都不同
      </strong>
      ，协议设计会变得复杂。
     </li>
     <li>
      <strong>
       不同设备可能理解不了数据
      </strong>
      ，比如一个 Java 设备发送
      <code>
       int
      </code>
      ，一个 Python 设备接收，它就可能无法解析。
     </li>
    </ul>
    <p>
     而
     <strong>
      字节数组是最通用的格式
     </strong>
     ，不同设备（Windows、Linux、Android、iOS）都能理解字节数据，这样 UDP 就可以跨平台工作。
    </p>
    <hr/>
    <h3>
     <a id="_83">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       计算机只能传输字节数据
      </strong>
      ，所以 UDP 只能发送
      <strong>
       字节数组
      </strong>
      。
     </li>
     <li>
      <strong>
       UDP 只是一个“快递员”
      </strong>
      ，它不关心数据类型，只搬运二进制数据。
     </li>
     <li>
      <strong>
       所有数据都能转换为字节数组
      </strong>
      ，所以 UDP 只需要支持
      <strong>
       发送字节
      </strong>
      ，而不用关心具体的数据类型。
     </li>
     <li>
      <strong>
       网络上所有协议（HTTP、FTP、TCP、UDP）最终传输的都是字节数据
      </strong>
      ，这样能兼容不同操作系统和设备。
     </li>
     <li>
      <strong>
       这种设计让 UDP 更加通用和高效
      </strong>
      ，无论是发送文本、图片还是音视频，都可以用
      <strong>
       相同的方式
      </strong>
      传输。
     </li>
    </ol>
