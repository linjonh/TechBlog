    <h1 class="title-article" id="articleContentId">
     SpringBoot加载配置文件的优先级
    </h1>
   </div>
  </div>
 </div>

    <blockquote>
     <p>
      在 SpringBoot 应用中，配置文件的加载顺序（优先级）决定了不同来源的配置属性如何覆盖彼此，但是，在实践中，也会经常遇到。下面文章将分析 SpringBoot 配置文件加载的优先级，从高到低排列。
     </p>
    </blockquote>
    <p>
    </p>
    <p>
    </p>
    <h2>
     <a id="1__4">
     </a>
     1. 命令行参数
    </h2>
    <ul>
     <li>
      <p>
       通过命令行（Command-line arguments）传递的参数具有最高优先级。
      </p>
     </li>
     <li>
      <p>
       如下示例，在启动应用时传递命令行参数
       <code>
        --server.port=8081
       </code>
       ：
      </p>
     </li>
    </ul>
    <pre><code class="prism language-java">java <span class="token operator">-</span>jar myapp<span class="token punctuation">.</span>jar <span class="token operator">--</span>server<span class="token punctuation">.</span>port<span class="token operator">=</span><span class="token number">8081</span>
</code></pre>
    <h2>
     <a id="2_Java__14">
     </a>
     2. Java 系统属性
    </h2>
    <ul>
     <li>
      <p>
       在启动应用时通过
       <code>
        -D
       </code>
       选项设置的系统属性。
      </p>
     </li>
     <li>
      <p>
       如下示例，在启动应用时设置
       <code>
        -Dserver.port=8081
       </code>
       系统属性：
      </p>
     </li>
    </ul>
    <pre><code class="prism language-java">java <span class="token operator">-</span><span class="token class-name">Dserver</span><span class="token punctuation">.</span>port<span class="token operator">=</span><span class="token number">8081</span> <span class="token operator">-</span>jar myapp<span class="token punctuation">.</span>jar
</code></pre>
    <h2>
     <a id="3__24">
     </a>
     3. 操作系统环境变量
    </h2>
    <ul>
     <li>
      设置在操作系统级别的环境变量。例如，
      <code>
       SERVER_PORT=8081
      </code>
      。
     </li>
    </ul>
    <h2>
     <a id="4_random_Random_Configuration_Property_Sources_28">
     </a>
     4. random 配置属性源（Random Configuration Property Sources）：
    </h2>
    <ul>
     <li>
      包括
      <code>
       random.*
      </code>
      属性，用于生成随机数或其他随机相关的配置。
     </li>
    </ul>
    <h2>
     <a id="5_applicationprofileproperties__applicationprofileyml__32">
     </a>
     5. application.{profile}.properties 或 application.{profile}.yml 文件
    </h2>
    <ul>
     <li>
      基于当前激活的 Spring Profile 加载的配置文件。例如，如果激活了
      <code>
       dev
      </code>
      环境，则加载
      <code>
       application-dev.properties
      </code>
      。
     </li>
    </ul>
    <h2>
     <a id="6_applicationproperties__applicationyml__36">
     </a>
     6. application.properties 或 application.yml 文件
    </h2>
    <ul>
     <li>
      默认的配置文件，通常位于
      <code>
       src/main/resources
      </code>
      目录下。
     </li>
    </ul>
    <h2>
     <a id="7_PropertySource__40">
     </a>
     7. @PropertySource 注解指定的属性文件
    </h2>
    <ul>
     <li>
      通过在配置类上使用
      <code>
       @PropertySource
      </code>
      注解加载的自定义属性文件。
     </li>
    </ul>
    <h2>
     <a id="8__44">
     </a>
     8. 默认属性
    </h2>
    <ul>
     <li>
      使用
      <code>
       SpringApplication.setDefaultProperties
      </code>
      设定的默认属性。
     </li>
    </ul>
    <h3>
     <a id="_48">
     </a>
     配置文件的搜索顺序
    </h3>
    <ul>
     <li>
      Spring Boot 在不同位置搜索
      <code>
       application.properties
      </code>
      或
      <code>
       application.yml
      </code>
      文件，以下是搜索的顺序，从高到低优先级：
     </li>
    </ul>
    <ol>
     <li>
      <p>
       <strong>
        外部位置（当前目录、上级目录等）：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         file:./config/
        </code>
       </li>
       <li>
        <code>
         file:./
        </code>
       </li>
       <li>
        <code>
         file:../config/
        </code>
       </li>
       <li>
        <code>
         file:../
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <ul>
     <li>
      如果你在应用程序的当前目录或其上级目录中有配置文件，这些文件的优先级高于类路径下的配置文件。
     </li>
    </ul>
    <ol start="2">
     <li>
      <p>
       <strong>
        类路径下的
        <code>
         /config
        </code>
        子目录：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         classpath:/config/
        </code>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        类路径根目录：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         classpath:/
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_69">
     </a>
     配置优先级示例
    </h3>
    <p>
     假设存在以下配置来源：
    </p>
    <ul>
     <li>
      <code>
       application.properties
      </code>
      在类路径根目录中设置
      <code>
       server.port=8080
      </code>
     </li>
     <li>
      <code>
       application-dev.properties
      </code>
      在类路径下的
      <code>
       config
      </code>
      目录中设置
      <code>
       server.port=8081
      </code>
     </li>
     <li>
      外部
      <code>
       application.properties
      </code>
      文件设置
      <code>
       server.port=8082
      </code>
     </li>
     <li>
      命令行参数传递
      <code>
       --server.port=8083
      </code>
     </li>
    </ul>
    <p>
     在激活
     <code>
      dev
     </code>
     Profile 的情况下，最终
     <code>
      server.port
     </code>
     的值将是
     <code>
      8083
     </code>
     。这是因为命令行参数的优先级最高，覆盖了其他所有配置来源。
    </p>
    <h3>
     <a id="_Profile_80">
     </a>
     激活 Profile
    </h3>
    <p>
     可以通过以下方式激活特定的 Spring Profile：
    </p>
    <ul>
     <li>
      <strong>
       命令行参数：
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-java">java <span class="token operator">-</span>jar myapp<span class="token punctuation">.</span>jar <span class="token operator">--</span>spring<span class="token punctuation">.</span>profiles<span class="token punctuation">.</span>active<span class="token operator">=</span>dev
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        环境变量：
       </strong>
       <br/>
       设置
       <code>
        SPRING_PROFILES_ACTIVE=dev
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        配置文件：
       </strong>
       <br/>
       在
       <code>
        application.properties
       </code>
       中设置
       <code>
        spring.profiles.active=dev
       </code>
      </p>
     </li>
    </ul>
    <h3>
     <a id="_96">
     </a>
     属性覆盖规则
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        同一属性名在多个来源中定义时，优先级高的来源会覆盖低的。例如：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         application.properties
        </code>
        中定义
        <code>
         app.name=MyApp
        </code>
       </li>
       <li>
        外部
        <code>
         application.properties
        </code>
        中定义
        <code>
         app.name=ExternalApp
        </code>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     若外部位置的配置文件优先级更高，则最终
     <code>
      app.name
     </code>
     为
     <code>
      ExternalApp
     </code>
     。
    </p>
    <h2>
     <a id="_105">
     </a>
     总结
    </h2>
    <p>
     Spring Boot 的配置文件加载优先级有助于：
    </p>
    <ul>
     <li>
      <strong>
       正确组织和管理配置文件
      </strong>
      ，确保不同环境下的配置正确应用。
     </li>
     <li>
      <strong>
       避免意外覆盖
      </strong>
      ，通过明确的优先级控制哪些配置生效。
     </li>
     <li>
      <strong>
       优化配置管理流程
      </strong>
      ，尤其是在复杂项目或多环境部署时。
     </li>
    </ul>
