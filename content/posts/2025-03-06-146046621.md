    <p>
     每一个线程自带事件循环
     <br/>
     线程函数，不会在main的事件循环中执行
     <br/>
     所以，我们碰到比较耗时的操作，我们就可以把这个操作放到线程中去完成，避免程序一卡一卡的
    </p>
    <h2>
     线程类：QThread
    </h2>
    <p>
     只要创建出一个QThread对象，就会立刻创建出一个线程
     <br/>
     只不过该线程没有开始运行而已
    </p>
    <blockquote>
     <p>
      QThread* thread = new QThread(this)
     </p>
    </blockquote>
    <h2 style="background-color:transparent">
     QThread线程如何运行
    </h2>
    <blockquote>
     <p>
      thread-&gt;start();
     </p>
    </blockquote>
    <p>
     即使没写线程主函数，线程依旧能够正常运行
    </p>
    <blockquote>
     <p>
      QThread start之后，会做一个判断：
      <br/>
      if(有没有写线程主函数 == 1){
      <!-- -->
      <br/>
      运行线程主函数
      <br/>
      }else{
      <!-- -->
      <br/>
      运行线程的事件循环
      <br/>
      }
     </p>
    </blockquote>
    <h2>
     线程的2条使用路径
    </h2>
    <h3 style="background-color:transparent">
     写一个线程主函数
    </h3>
    <blockquote>
     <p>
      QThread 类有虚函数run
     </p>
     <p>
      只要重写了run函数，就相当于写了一个线程主函数
     </p>
     <p>
      那么当 线程.start()之后，就会运行这个run函数
     </p>
     <p>
      所以，我们要写一个 myThread,继承QThread，就可以重写myThread的run函数了
     </p>
    </blockquote>
    <p>
     那么创建线程的时候，就不要再创建QThread了，而是创建myThread
    </p>
    <h4 style="background-color:transparent">
     QT、c的线程区别
    </h4>
    <p>
     1：一个QThread或者一个myThread，只会有一个线程
     <br/>
     2：c语言线程，只要线程主函数结束运行，线程就结束运行了，剩下的就是资源回收的问题
     <br/>
     QT线程，只要QThread对象一直存在，线程就一直存在，无论run函数是否结束
     <br/>
     如果run函数结束运行了，想要重新运行的话，重新start就好了
     <br/>
     3：c语言线程，只要有地址，什么数据都能访问
     <br/>
     QT线程，不允许跨线程访问ui界面中的窗口的
    </p>
    <h4>
     线程如何结束运行
    </h4>
    <blockquote>
     <p>
      thread-&gt;terminate();
     </p>
    </blockquote>
    <pre><code class="language-cpp">#ifndef MYTHREAD_H
#define MYTHREAD_H


#include&lt;QThread&gt;

#include&lt;QScreen&gt;
#include&lt;QApplication&gt;
#include&lt;QPixmap&gt;

class myThread : public QThread
{
    Q_OBJECT
public:
    myThread(QObject*parent);
    void run() override;
    QScreen* screen;
signals:
    void sndSignal(QPixmap pic);
};

#endif // MYTHREAD_H
#include "mythread.h"

myThread::myThread(QObject*parent):QThread(parent){
    screen=QApplication::primaryScreen();
}
void myThread::run(){
    while(1){
        QPixmap pic=screen-&gt;grabWindow(0);
        emit sndSignal(pic);
        QThread::usleep(100);
    }
}

    <h3>
     将一个事件，移动到线程的事件循环中去执行
    </h3>
    <blockquote>
     <p>
      QObject::moveToThread(目标线程地址 t)
      <br/>
      谁调用这个函数，谁的所有事件，都会被移动到t的事件循环中去执行
     </p>
    </blockquote>
    <p>
     但是，moveToThread（）有限制：
    </p>
    <p>
     1：调用moveToThread函数的组件，必须直接继承QObject，不能继承自QWidget 获取其他基类 2：调用moveToThread函数的组件，不能有父组件
     <br/>
     所以，要自己写一个类，直接继承自QObject，还不能有父组件
    </p>
    <pre><code class="language-cpp">class Worker : public QObject {
    Q_OBJECT
public slots:
    void doWork() {
        // 耗时操作
        emit workFinished();
    }
signals:
    void workFinished();
};

QThread* thread = new QThread;
Worker* worker = new Worker;
worker-&gt;moveToThread(thread);
connect(thread, &amp;QThread::started, worker, &amp;Worker::doWork);
connect(worker, &amp;Worker::workFinished, thread, &amp;QThread::quit);
connect(worker, &amp;Worker::workFinished, worker, &amp;Worker::deleteLater);
connect(thread, &amp;QThread::finished, thread, &amp;QThread::deleteLater);
thread-&gt;start();</code></pre>
