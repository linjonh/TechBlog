    <p>
     <strong>
      一·。Docker的构建参数
     </strong>
    </p>
    <p>
     <img alt="" height="667" src="https://i-blog.csdnimg.cn/direct/5f5fe665dcb74427ada60b970cc82b5c.png" width="961"/>
    </p>
    <p>
     <img alt="" height="279" src="https://i-blog.csdnimg.cn/direct/fe01c55c8075435cbd8bfccdb1dd7e43.png" width="1020"/>
    </p>
    <p>
     注释：
    </p>
    <p>
     1.对于CMD，如果不想显示，而是使用交互界面：docker run -ti --rm --name test2 busybox:v5 sh
    </p>
    <p>
     2.对于CMD，一个交互界面只可以使用一个，如果想多次使用CMD，则用ENTRYPOINT进行替换
    </p>
    <p>
    </p>
    <p>
     <strong>
      FROM的基本使用
     </strong>
    </p>
    <p>
     <span style="color:#fe2c24">
      mkdir /docker
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      cd /docker
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      vi /Dockerfile
     </span>
    </p>
    <p>
     <img alt="" height="75" src="https://i-blog.csdnimg.cn/direct/c2be755424394adbbfe355629c54d511.png" width="277"/>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        FROM busybox
       </code>
      </strong>
      ：这是
      <code>
       Dockerfile
      </code>
      中的一条指令，
      <code>
       FROM
      </code>
      用于指定基础镜像，
      <code>
       busybox
      </code>
      是一个轻量级的 Linux 工具集镜像，许多 Docker 镜像都会基于它来构建，以此为基础可以在其之上添加额外的功能。
     </li>
     <li>
      <strong>
       <code>
        RUN touch /leefile
       </code>
      </strong>
      ：
      <code>
       RUN
      </code>
      指令用于在镜像构建过程中执行命令。
      <code>
       touch
      </code>
      是 Linux 系统中用于创建空文件的命令，
      <code>
       /leefile
      </code>
      是要创建的文件的路径，这里表示在镜像中创建一个名为
      <code>
       leefile
      </code>
      的空文件。
     </li>
    </ul>
    <p>
     <span style="color:#fe2c24">
      docker build -t busybox:v2 .
     </span>
     <span style="color:#0d0016">
      (或者使用:docker build -f /docker/Dockerfile1 -t busybox:v3 .
     </span>
     <span style="color:#0d0016">
      )
     </span>
    </p>
    <p>
     <code>
      docker build
     </code>
     是 Docker 用于构建镜像的命令，
     <code>
      -t
     </code>
     选项用于给构建的镜像指定一个标签（tag），
     <code>
      busybox:v2
     </code>
     表示镜像的名称为
     <code>
      busybox
     </code>
     ，版本号为
     <code>
      v2
     </code>
     。
    </p>
    <p>
     <span style="color:#fe2c24">
      docker run -it --rm --name test1 busybox:v2
     </span>
    </p>
    <p>
     <code>
      --rm
     </code>
     选项表示容器停止运行后自动删除容器，避免占用过多磁盘空间。
    </p>
    <p>
     综上所述，这一系列操作的整体流程是：先创建一个目录，然后在该目录下创建一个
     <code>
      Dockerfile
     </code>
     来定义镜像的构建步骤，接着使用
     <code>
      docker build
     </code>
     命令基于
     <code>
      Dockerfile
     </code>
     构建一个名为
     <code>
      busybox:v2
     </code>
     的镜像，最后使用
     <code>
      docker run
     </code>
     命令基于这个镜像启动一个交互式的容器，并将其命名为
     <code>
      test1
     </code>
     ，容器停止后会自动删除。
    </p>
    <p>
     <span style="color:#fe2c24">
      docker history busybox:v2
     </span>
     ：可以用于查看该镜像版本的信息
    </p>
    <p>
     <strong>
      COPY的使用
     </strong>
    </p>
    <p>
     将passwd复制到容器内
    </p>
    <p>
     1.首先进行文件的复制，copy /etc/passwd  /docker
    </p>
    <p>
     2.vi /docker/Dockerfile1
    </p>
    <p>
     <img alt="" height="109" src="https://i-blog.csdnimg.cn/direct/5fbc5ed7c0784a9082a63058a8ebb991.png" width="276"/>
    </p>
    <p>
     3.
     <span style="color:#0d0016">
      docker build -f /docker/Dockerfile1 -t busybox:v3 .
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      4.docker run -it --rm --name test1 busybox:v2
     </span>
    </p>
    <p>
     ADD和COPY相似，可以进行压缩文件
    </p>
    <p>
     <img alt="" height="112" src="https://i-blog.csdnimg.cn/direct/7f4f442fc202477ca5d922555f319459.png" width="367"/>
    </p>
    <p>
    </p>
    <p>
     <strong>
      二。docker内部署centos
     </strong>
    </p>
    <p>
     <img alt="" height="238" src="https://i-blog.csdnimg.cn/direct/ec6fc2512438401881ffb38fbba95800.png" width="1887"/>
    </p>
    <p>
    </p>
    <p>
     <strong>
      三。搭建docker的私有仓库
     </strong>
    </p>
    <p>
     1.
     <span style="color:#fe2c24">
      docker pull registry
     </span>
     ：拉取文件
    </p>
    <p>
     2.
     <span style="color:#fe2c24">
      docker run -d -p 5000:5000 --restart=always registry
     </span>
    </p>
    <p>
     创建register容器，-p端口5000，
     <strong>
      <code>
       --restart=always
      </code>
     </strong>
     ：这是一个重启策略选项，指定容器在退出后总是自动重启，确保镜像仓库服务能够持续运行
    </p>
    <p>
     3.
     <span style="color:#fe2c24">
      docker tag busybox:latest 192.168.142.135:5000/busybox:latest
     </span>
    </p>
    <ul>
     <li>
      <code>
       docker tag
      </code>
      ：
      <u>
       用于为镜像添加标签，方便对镜像进行标识和管理。
      </u>
     </li>
     <li>
      <code>
       busybox:latest
      </code>
      ：是源镜像的名称和标签，这里表示本地的
      <code>
       busybox
      </code>
      镜像的
      <code>
       latest
      </code>
      版本。
     </li>
     <li>
      <code>
       192.168.142.135:5000/busybox:latest
      </code>
      ：是目标标签，指定了将本地的
      <code>
       busybox:latest
      </code>
      镜像标记为可以推送到地址为
      <code>
       192.168.142.135
      </code>
      、端口为 5000 的本地镜像仓库中的
      <code>
       busybox:latest
      </code>
      镜像。
     </li>
    </ul>
    <p>
     4.
     <span style="color:#fe2c24">
      docker push 192.168.142.135:5000/busybox:latest
     </span>
    </p>
    <ul>
     <li>
      <code>
       docker push
      </code>
      ：用于将本地的镜像推送到指定的镜像仓库。
     </li>
     <li>
      <code>
       192.168.142.135:5000/busybox:latest
      </code>
      ：指定了要推送的镜像的目标地址和名称标签
     </li>
    </ul>
    <p>
     5.vi /etc/docker/daemon.json
    </p>
    <p>
     <img alt="" height="193" src="https://i-blog.csdnimg.cn/direct/50e83201985d48cb9a86dd44781265d6.png" width="702"/>
    </p>
    <p>
     6.测试：
    </p>
    <p>
     <img alt="" height="76" src="https://i-blog.csdnimg.cn/direct/3178a33ac5b9418f9c681a19df02777b.png" width="693"/>
    </p>
    <p>
     注释：register文件在：
    </p>
    <p>
     /var/lib/docker/volumes/a03953bf68bf99f30e7529aac89ed497fbfbd7af6c55abc71330123b6ce60f82/_data/docker/registry/v2/repositories
    </p>
    <p>
    </p>
    <p>
     <strong>
      拓展：私有仓库直接存入，风险太大，需要进行加密保证安全性：
     </strong>
    </p>
    <p>
     1.
     <span style="color:#fe2c24">
      mkdir certs
     </span>
     ：创建用于存储共公私钥的目录
    </p>
    <p>
     2.
     <span style="color:#fe2c24">
      openssl req -newkey rsa:4096 -nodes -sha256 -keyout certs/jiami.org.key -addext "subjectAltName =DNS:reg.jiami.org" -x509 -days 365 -out certs/jiami.org.crt
     </span>
    </p>
    <p>
     注释：使用openssl req生成一个4096位的私钥，采用sha256哈希算法最终将私钥存储与certs/jiami.org.key文件内，指定解析后，生成-x509证书，最终将证书保存于后续文件
    </p>
    <p>
     3.
     <span style="color:#fe2c24">
      docker run -d -p 443:443 --restart=always -v /root/certs:/certs
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      -e REGISTRY_HTTP_ADDR=0.0.0.0:443
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      -e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/jiami.org.crt
     </span>
    </p>
    <p>
     <span style="color:#fe2c24">
      -e REGISTRY_HTTP_TLS_KEY=/certs/jiami.org.key registry:latest
     </span>
    </p>
    <p>
     注释：-v /root/certs:/certs，-v表挂载，将主机上的/root/cets挂载于容器内的certs，-e表设置环境变量，设置端口，证书和私钥
    </p>
    <p>
     <img alt="" height="171" src="https://i-blog.csdnimg.cn/direct/efa1fb7f678740afbdf9df7e37a52c84.png" width="1843"/>
    </p>
    <p>
     4.
     <span style="color:#fe2c24">
      mkdir -p /etc/docker/certs.d/reg.jiami.org
     </span>
     ：创建一个docker子目录用于存储证书
    </p>
    <p>
     5.
     <span style="color:#fe2c24">
      cp /root/certs/jiami.org.crt /etc/docker/certs.d/reg.jiami.org/ca.crt
     </span>
     ：将证书复制于文件
    </p>
    <p>
     6.
     <span style="color:#fe2c24">
      systemctl restart docker
     </span>
    </p>
    <p>
     7.
     <span style="color:#fe2c24">
      docker tag nginx:v1 reg.jiami.org/nginx:v1
     </span>
     ：打标签
    </p>
    <p>
     8.
     <span style="color:#fe2c24">
      docker push reg.jiami.org/nginx:v1
     </span>
     ：进行推送
    </p>
    <p>
     <img alt="" height="271" src="https://i-blog.csdnimg.cn/direct/e356a187c04d4cc687199e08e35a1c6b.png" width="1081"/>
    </p>
    <p>
    </p>
    <p>
     <strong>
      四。为仓库建立登陆认证：（前提创建好私钥和证书）
     </strong>
    </p>
    <p>
     1.
     <span style="color:#fe2c24">
      yum install httpd-tools
     </span>
     ：下载登陆工具
    </p>
    <p>
     2.
     <span style="color:#fe2c24">
      mkdir auth
     </span>
    </p>
    <p>
     3.
     <span style="color:#fe2c24">
      htpasswd -Bc auth/.htpasswd lee
     </span>
     ：为目录创建密码和用户lee
     <br/>
     <img alt="" height="54" src="https://i-blog.csdnimg.cn/direct/23553d19e2d34e63bce6b29427d30e1c.png" width="748"/>
    </p>
    <p>
     注释：再次添加密码和用户不需要添加c，只要-B
    </p>
    <p>
     4.
     <span style="color:#fe2c24">
      docker run -d \ -p 443:443 \ --restart=always \ -v /root/certs:/certs \ -e REGISTRY_HTTP_ADDR=0.0.0.0:443 \ -e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/jiami.org.crt \ -e REGISTRY_HTTP_TLS_KEY=/certs/jiami.org.key \ -v /root/auth:/auth \ -e REGISTRY_AUTH=htpasswd \ -e "REGISTRY_AUTH_HTPASSWD_REALM=Registry Realm" \ -e "REGISTRY_AUTH_HTPASSWD_PATH=/auth/.htpasswd" \ -e REGISTRY_HTTP_SECRET=$(openssl rand -hex 32) \ registry
     </span>
     <br/>
     <img alt="" height="54" src="https://i-blog.csdnimg.cn/direct/7d3d2fc062b048c595b4bc78ea464c6b.png" width="949"/>
    </p>
    <p>
     注释：必须拥有此文件，提供http加密
    </p>
    <p>
     <img alt="" height="217" src="https://i-blog.csdnimg.cn/direct/e2e05103624c4d13936ad8e7b744fbde.png" width="943"/>
    </p>
    <p>
     注释：在以后上传任何东西前，必须进行登陆：docker login reg.jiami.org，否则无法上床内容于私有仓库
    </p>
