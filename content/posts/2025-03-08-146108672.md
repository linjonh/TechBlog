     </a>
     【APP/小程序嵌入H5页面】
    </h2>
    <p>
     <a href="https://www.bilibili.com/video/BV1PE421A7Pt?spm_id_from=333.788.videopod.sections&amp;vd_source=c919d6976fd77ac77f9860cf2e7e0e11" rel="nofollow">

     </a>
    </p>
    <h3>
     <a id="_7">
     </a>
     背景
    </h3>
    <p>
     产品APP：安卓端，ios端和小程序端。产品要做一个活动页，需要在3个端侧把活动页展示出来。但活动有效期只有3天。
    </p>
    <p>
     可供选择的方案包括：
    </p>
    <ul>
     <li>
      安卓端、ios和小程序的代码都开发此活动，活动结束后再将这部分代码逻辑屏蔽。每端添加活动后都需要重新提交版本审核上架。
     </li>
     <li>
      做一个H5页面，在3个端侧分别将此H5页面嵌入进去。
     </li>
    </ul>
    <h3>
     <a id="H5_15">
     </a>
     小程序内嵌H5页面具体实现
    </h3>
    <p>
     使用uniapp的web-view实现。
    </p>
    <p>
     <strong>
      操作
     </strong>
     ：通过webview内置组件进行H5页面嵌入，src填入要打开的H5页面的网址
    </p>
    <p>
     <strong>
      示例
     </strong>
    </p>
    <pre><code>&lt;web-view src="https://xxx.com/xxx/xxx"&gt;&lt;/web-view&gt;
</code></pre>
    <br/>
    <br/>

    <h3>
     <a id="1_H5_45">
     </a>
     案例1 微信H5页面跳转第三方小程序
    </h3>
    <p>
     <a href="https://www.bilibili.com/video/BV1DS4y1w7yf/?spm_id_from=333.1391.0.0&amp;vd_source=c919d6976fd77ac77f9860cf2e7e0e11" rel="nofollow">
      如何在微信H5页面链接跳转到第三方小程序的任意页面
     </a>
    </p>
    <p>
     在微信的H5页，跳转第三方小程序。常用于社群引流等。
    </p>
    <p>
     跳转过程如下图。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dd0467c526674f7384bd9d8df2e39c63.png"/>
    </p>
    <h4>
     <a id="_52">
     </a>
     前提条件
    </h4>
    <p>
     在微信的H5页，跳转第三方小程序，前提条件：
    </p>
    <ul>
     <li>
      <ol>
       <li>
        拥有已认证的服务号
        <br/>
        服务号添加“JS接口安全域名”，即访问H5界面的域名。
       </li>
      </ol>
     </li>
     <li>
      <ol start="2">
       <li>
        获取第三方小程序的原始id
        <br/>
        只能手动获取，微信官方没有给出api接口动态调用。
        <br/>
        获取方法，以小红书为例，gh_52be748ce5b7。
       </li>
      </ol>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/14c44daed21b4041934fea4a8bd30898.png"/>
    </p>
    <ul>
     <li>
      <ol start="3">
       <li>
        获取第三方小程序的页面路径
       </li>
      </ol>
     </li>
    </ul>
    <p>
     如果页面路径获取不到，默认跳转的是第三方小程序的首页。
     <br/>
     官方也提供了手动获取的方法，但是没有暴露的api方法，手动获取方法不友好，需要去小程序业务后台添加开发者，然后下图会多一个页面路径。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8ed5d606b3d44c49b44abcf066dd167b.png"/>
    </p>
    <blockquote>
     <p>
      注意这里有个复制链接，对于开发者来说是没用的。
     </p>
    </blockquote>
    <p>
     这里的up写了一个动态获取的功能，需要去公众号——数字门店营销（没找到）。
    </p>
    <p>
     以上就是跳转的必要条件。
    </p>

    <p>
     一、概念解析
    </p>
    <ol>
     <li>
      什么是静态网站？
      <br/>
      特点：
      <mark>
       内容固定，无需服务器端处理（如 PHP、数据库）
      </mark>
      ，用户访问时直接加载 HTML、CSS、JavaScript 文件。
      <br/>
      常见形式：企业官网、个人博客、产品展示页等。
      <br/>
      技术栈：纯前端技术（HTML/CSS/JavaScript），可能搭配静态网站生成器（如 Hugo、Jekyll）。
     </li>
     <li>
      什么是 H5？
      <br/>
      H5 的两种常见含义：
      <br/>
      广义：HTML5 技术标准，包含新的语义标签（如
      <code>
       &lt;header&gt;
      </code>
      、
      <code>
       &lt;section&gt;
      </code>
      ）、多媒体支持（
      <code>
       &lt;video&gt;
      </code>
      、
      <code>
       &lt;canvas&gt;
      </code>
      ）、本地存储等。
      <br/>
      <mark>
       狭义（国内语境）：特指移动端交互式营销页面（如滑动翻页、动画效果），但严格来说这是对 HTML5 技术的场景化应用。
      </mark>
     </li>
    </ol>
    <p>
     二、静态网站 H5 的核心特点
    </p>
    <ol>
     <li>
      技术优势
      <br/>
      响应式设计：通过 HTML5 的 和 CSS3 媒体查询适配多端（PC/手机/平板）。
      <br/>
      多媒体支持：直接嵌入视频/音频（无需 Flash）：
     </li>
     <li>
      局限性
      <br/>
      内容更新不便：需手动修改 HTML 或重新生成。
      <br/>
      功能受限：无法实现用户登录、实时数据交互等动态功能。
     </li>
    </ol>

    <h4>
     <a id="_402">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      微信开放标签
      <code>
       "wx-open-launch-weapp"
      </code>
      有最低微信版本要求，以及最低系统版本要求。
      <ul>
       <li>
        微信版本要求：7.0.12及以上
       </li>
       <li>
        系统版本：ios 10.3以及上、Android 5.0以及上
       </li>
      </ul>
     </li>
     <li>
      根据自己的系统，做好友好提示
     </li>
     <li>
      测试环境：唤起小程序，需要在测试环境或正式环境进行测试，开发环境无法测试
     </li>
    </ul>
    <h4>
     <a id="_411">
     </a>
     常见问题
    </h4>
    <p>
     按钮渲染失败，排查
    </p>
    <ul>
     <li>
      域名错误（本地开发，控制台报域名不对）
      <ul>
       <li>
        公众号是否绑定了正确的域名
       </li>
       <li>
        初始化时传的域名参数是否与前端域名一致
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      为什么要在测试/正式环境进行测试，因为开发环境与公众号绑定的域名不是一致的，因此按钮渲染失败。
      <br/>
      解决：提交代码，发布测试环境进行测试。
      <br/>
      让前端域名地址与后端接口前缀地址保持一致。
     </p>
    </blockquote>
    <ul>
     <li>
      签名错误（大部分是ios手机）
      <br/>
      进入页面1的域名是
      <code>
       https://test.zyy.com/#/nav1
      </code>
      <br/>
      进入页面2的域名是
      <code>
       https://test.zyy.com/#/nav2
      </code>
     </li>
    </ul>
    <p>
     进入上述2个页面时，分别初始化js-sdk，那么ios手机容易报签名错误。
    </p>
    <p>
     如何解决？
    </p>
    <ul>
     <li>
      初始化的入参保持一致
     </li>
    </ul>
    <blockquote>
     <p>
      推荐使用主域名(#之前的部分)
      <code>
       https://test.zyy.com/
      </code>
     </p>
    </blockquote>
    <ul>
     <li>
      每次都是最新的初始化返回值
     </li>
    </ul>
    <blockquote>
     <p>
      从页面A跳页面B时，采用
      <code>
       window.location.href = 页面2
      </code>
      ，或
      <code>
       window.location.replace(页面2)
      </code>
      <br/>
      即通过跳转链接实现，相当于手动刷新页面，先跳到页面1，再跳转到页面2
     </p>
    </blockquote>
    <ul>
     <li>
      第一次请求后端接口时，本地缓存后端接口返回的4个值(appId, timestamp, nonceStr, signature)
     </li>
    </ul>
    <br/>
    <br/>

    <h2>
     <a id="_454">
     </a>
     【小程序一键登录】
    </h2>
    <h3>
     <a id="1__456">
     </a>
     1 获取登录临时凭证
    </h3>
    <ul>
     <li>
      步骤1 首先调用api-
      <a href="https://uniapp.dcloud.net.cn/api/plugins/login.html#login" rel="nofollow">
       uni.login()
      </a>
      ，获取到code，此code是用户登录的临时凭证。
     </li>
    </ul>
    <pre><code class="prism language-javascript"><span class="token literal-property property">methods</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
	<span class="token keyword">async</span> <span class="token function">login</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
		<span class="token keyword">let</span> <span class="token punctuation">{<!-- --></span> code <span class="token punctuation">}</span> <span class="token operator">=</span> <span class="token keyword">await</span> uni<span class="token punctuation">.</span><span class="token function">login</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
	<span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <h3>
     <a id="2__468">
     </a>
     2 请求后台信息
    </h3>
    <ul>
     <li>
      步骤2 调用后端接口，将code作为入参请求接口
     </li>
    </ul>
    <p>
     <strong>
      接口信息
     </strong>
     <br/>
     请求url： /u/loginByWechat
     <br/>
     请求方式：GET
     <br/>
     参数：code，必传
    </p>
    <p>
     <strong>
      代码示例
     </strong>
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">async</span> <span class="token function">loginWechat</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

<span class="token punctuation">}</span>
</code></pre>
    <p>
     <strong>
      接口返回
     </strong>
     <br/>
     此时后端返回的数据有2种情况：
    </p>
    <ol>
     <li>
      用户已经注册或已经登录过，此时接口返回用户信息和token，此时进入下一步登录后的操作即可。
     </li>
     <li>
      用户之前没有注册或登录过，此时需要进入注册环节。
     </li>
    </ol>
    <h3>
     <a id="3__489">
     </a>
     3 注册环节
    </h3>
    <p>
     如果走注册环节，那么步骤2接口返回的字段信息会有openid，sessionKey，unionid.
    </p>
    <ul>
     <li>
      步骤3 注册环节，请求后台注册接口
     </li>
    </ul>
    <p>
     <strong>
      注册接口
     </strong>
     <br/>
     注册微信用户（将微信信息注册到平台）
     <br/>
     请求url：/u/wechat/register
     <br/>
     请求方式：POST
     <br/>
     参数：unionId，openId，sessionKey, signature，rawData，encryptedData
    </p>
    <p>
     <strong>
      参数获取
     </strong>
     <br/>
     unionId，openId，sessionKey已经通过步骤2的接口返回了，此时还剩下signature，rawData，encryptedData。通过官方api-
     <a href="https://uniapp.dcloud.net.cn/api/plugins/login.html#getuserinfo" rel="nofollow">
      uni.getUserInfo()
     </a>
     获取。
    </p>
    <p>
     <strong>
      代码示例
