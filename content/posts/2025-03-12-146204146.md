  <div class="article_content clearfix" id="article_content">

    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     如果一台电脑只有一个github账号，那么进行默认的ssh配置，通过git拉取和提交代码即可，但在实际的工作中，有时候需要在一台电脑登录多个github账号，将不同的项目代码提交到不同的github账号，这个时候如果仅仅只是使用ssh默认配置，私钥和公钥将无法完成
     <code>
      一对一
     </code>
     配对，为此我们需要进行ssh的``多对多```配置，简单来讲就是原本两个人用一双筷子，现在两个人用两双筷子。
     <br/>
     当你看到这篇文章时，默认你已经有了一些git相关的基础知识，这篇文章里不进行赘述。
     <br/>
     为了举例方便，这里使用 “one” 和 “two” 两个账户。
    </p>
    <h4>
     <a id="_cd_ssh_3">
     </a>
     步骤一 ：使用cd ~/.ssh切换工作目录，然后使用如下命令生成两个钥匙，中间一路回车。
    </h4>
    <pre><code class="prism language-python">ssh<span class="token operator">-</span>keygen <span class="token operator">-</span>t rsa <span class="token operator">-</span>f <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_one  <span class="token operator">-</span>C <span class="token string">"one@qq.com"</span>     <span class="token comment">#-C 参数里面是注释，可以随便填</span>
ssh<span class="token operator">-</span>keygen <span class="token operator">-</span>t rsa <span class="token operator">-</span>f <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_two  <span class="token operator">-</span>C <span class="token string">"two@qq.com"</span>
</code></pre>
    <p>
     这样会在~/.ssh目录下生成四个文件：
    </p>
    <pre><code class="prism language-python">id_rsa<span class="token punctuation">.</span>one      <span class="token comment">#账号 one 的私钥</span>
id_rsa<span class="token punctuation">.</span>one<span class="token punctuation">.</span>pub  <span class="token comment">#账号 one 的公钥</span>
id_rsa<span class="token punctuation">.</span>two
id_rsa<span class="token punctuation">.</span>two<span class="token punctuation">.</span>pub
</code></pre>
    <h4>
     <a id="_config_19">
     </a>
     步骤二：创建配置文件 config
    </h4>
    <p>
     在 ~/.ssh目录下新建 config 文件，令不同 Host 实际映射到同一 HostName，但密钥文件不同。
    </p>
    <pre><code class="prism language-python">touch config    <span class="token comment">#创建 config</span>
vim config      <span class="token comment">#编辑 config</span>
</code></pre>
    <p>
     写入以下内容
    </p>
    <pre><code class="prism language-python">Host  one<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com      <span class="token comment">#one.github.com是自己取的名，可以随便取</span>
HostName github<span class="token punctuation">.</span>com
PreferredAuthentications publickey
IdentityFile <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_one


Host two<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com
HostName github<span class="token punctuation">.</span>com
PreferredAuthentications publickey
IdentityFile <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_two
</code></pre>
    <h4>
     <a id="ssh_42">
     </a>
     步骤三：添加秘钥到ssh识别列表
    </h4>
    <pre><code class="prism language-python">ssh<span class="token operator">-</span>add <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_one
ssh<span class="token operator">-</span>add <span class="token operator">~</span><span class="token operator">/</span><span class="token punctuation">.</span>ssh<span class="token operator">/</span>id_rsa_two
</code></pre>
    <p>
     步骤四：添加 SSH key 及 测试
     <br/>
     分别登陆两个 github 账号，在 Settings —&gt; SSH and GPG keys 中，点击 “new SSH key”，把 “id_rsa.one.pub” 和 "id_rsa.two.pub"这两个公钥的内容分别添加到相应的账号中。
    </p>
    <h4>
     <a id="_SSH__github_51">
     </a>
     步骤五：为了确认我们可以通过 SSH 连接 github，可通过输入下面命令来验证
    </h4>
    <pre><code class="prism language-python">ssh <span class="token operator">-</span>T git@one<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com
ssh <span class="token operator">-</span>T git@two<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com
</code></pre>
    <p>
     如果看到下面信息，就说明连接正常。
    </p>
    <pre><code class="prism language-python">Hi one! You’ve successfully authenticated<span class="token punctuation">,</span> but GitHub does <span class="token keyword">not</span> provide shell access<span class="token punctuation">.</span>
Hi  two! You’ve successfully authenticated<span class="token punctuation">,</span> but GitHub does <span class="token keyword">not</span> provide shell access<span class="token punctuation">.</span>
</code></pre>
    <h4>
     <a id="_65">
     </a>
     步骤六：用户名和邮箱配置
    </h4>
    <p>
     注意：因为一台电脑上配置了多个 github 账号，所以就不能再配置全局的用户名和邮箱了，而是在不同的仓库下，如果需要连接不同的 git 账号，配置相应的局部用户名和邮箱即可，如果之前配置过全局的用户名和邮箱，需要取消配置。
     <br/>
     取消全局 用户名/邮箱 配置
    </p>
    <pre><code class="prism language-python">git config <span class="token operator">-</span><span class="token operator">-</span><span class="token keyword">global</span> <span class="token operator">-</span><span class="token operator">-</span>unset user<span class="token punctuation">.</span>name
git config <span class="token operator">-</span><span class="token operator">-</span><span class="token keyword">global</span> <span class="token operator">-</span><span class="token operator">-</span>unset user<span class="token punctuation">.</span>email
</code></pre>
    <p>
     设置局部 用户名/邮箱 配置
    </p>
    <pre><code class="prism language-python">git config user<span class="token punctuation">.</span>name “xxxx”
git config user<span class="token punctuation">.</span>email “xxxx@xx<span class="token punctuation">.</span>com”
</code></pre>
    <h4>
     <a id="_git_81">
     </a>
     步骤七：使用 git克隆仓库
    </h4>
    <p>
     原来写法
    </p>
    <pre><code class="prism language-python">git clone git@github<span class="token punctuation">.</span>com<span class="token punctuation">:</span> 用户名<span class="token operator">/</span>learngit<span class="token punctuation">.</span>git
</code></pre>
    <p>
     现在写法（将github.com 替换为之前设置的别名，这里替换为one.github.com）
    </p>
    <pre><code class="prism language-python">git clone git@one<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com<span class="token punctuation">:</span> 用户名<span class="token operator">/</span>learngit<span class="token punctuation">.</span>git
</code></pre>
    <h4>
     <a id="_95">
     </a>
     步骤八：远程地址添加或者修改
    </h4>
    <p>
     添加
    </p>
    <pre><code class="prism language-python">git remote add origin git@one<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com<span class="token punctuation">:</span>用户名<span class="token operator">/</span>项目名<span class="token punctuation">.</span>git    
</code></pre>
    <p>
     修改
    </p>
    <pre><code class="prism language-python">git remote <span class="token builtin">set</span><span class="token operator">-</span>url origin git@one<span class="token punctuation">.</span>github<span class="token punctuation">.</span>com<span class="token punctuation">:</span>用户名<span class="token operator">/</span>项目名<span class="token punctuation">.</span>git
</code></pre>
