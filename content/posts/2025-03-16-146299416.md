    <h2>
     Django模型
    </h2>
    <ul>
     <li>
      <p>
       定义：模型是 Django 中用于定义数据库结构的 Python 类。每个模型类对应数据库中的一张表，类的属性对应表的字段。
      </p>
     </li>
     <li>
      <p>
       作用：通过模型，Django 可以将 Python 代码与数据库表结构关联起来，开发者无需直接编写 SQL 语句即可操作数据库。
      </p>
     </li>
    </ul>

    <p>
     注意：id 字段会自动生成，无需显式定义（Django 默认会添加一个自增的 id 主键，如果不是这种情况需要手动添加字段）
    </p>
    <pre><code class="language-python">from django.db import models

class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    password = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    first_name = models.CharField(max_length=100)
    last_name = models.CharField(max_length=100)

    def __str__(self):
        return self.username</code></pre>
    <p>
     举个例子models中添加一个商品 类中有id(自动生成)、name、code、price、crate_time、update_time大家自己创建下 按照上面的命令
    </p>
    <pre><code class="language-python">class Product(models.Model):
    # id 字段会自动生成，无需显式定义（Django 默认会添加一个自增的 id 主键）
    name = models.CharField(max_length=100, verbose_name="商品名称")
    code = models.CharField(max_length=50, unique=True, verbose_name="商品编码")
    price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name="商品价格")
    create_time = models.DateTimeField(auto_now_add=True, verbose_name="创建时间")
    update_time = models.DateTimeField(auto_now=True, verbose_name="更新时间")</code></pre>
    <h2>
     <strong>
      Migrations
     </strong>
    </h2>
    <h2>
    </h2>
    <ul>
     <li>
      <p>
       定义：迁移是 Django 用来管理数据库模式变更的机制。每次修改模型后，Django 会生成一个迁移文件，记录模型的变化。
      </p>
     </li>
     <li>
      <p>
       作用：迁移文件允许你将数据库模式从一种状态更新到另一种状态，同时保留现有数据。
      </p>
     </li>
     <li>
      <p>
       生成迁移文件：使用
       <code>
        python manage.py makemigrations
       </code>
       命令，Django 会检测模型的变更，并生成相应的迁移文件。
      </p>
      <ul>
       <li>
        <p>
         例如，修改user模型后，运行
         <code>
          makemigrations
         </code>
         会生成一个迁移文件，记录这些变更。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <img alt="" height="879" src="https://i-blog.csdnimg.cn/direct/f11296730dcc4191bfeab32aa1bda618.png" width="1769"/>
    </p>
    <p>
     product例子生成表名：myapp_product
    </p>
    <p>
     <img alt="" height="942" src="https://i-blog.csdnimg.cn/direct/587790a69b1648078dda549496645bb3.png" width="1859"/>
    </p>
    <p>
     <code>
      makemigrations
     </code>
     和
     <code>
      migrate
     </code>
     的关系
    </p>
    <ul>
     <li>
      <p>
       <code>
        makemigrations
       </code>
       ：生成迁移文件，记录模型的变更。
      </p>
     </li>
     <li>
      <p>
       <code>
        migrate
       </code>
       ：应用迁移文件，将变更同步到数据库中。
      </p>
     </li>
     <li>
      <p>
       流程：
      </p>
      <ol>
       <li>
        <p>
         修改模型（如添加字段、修改字段类型等）。
        </p>
       </li>
       <li>
        <p>
         运行
         <code>
          python manage.py makemigrations
         </code>
         ，生成迁移文件。(注意如果说有报错需要查看自己的settings.py中是否包含INSTALLED_APPS是否包含了myapp)
        </p>
       </li>
       <li>
        <p>
         运行
         <code>
          python manage.py migrate
         </code>
         ，将迁移应用到数据库（可以查看数据库中建立的表，生成的表是app名称_对象名）。
        </p>
       </li>
      </ol>
     </li>
    </ul>
