    <blockquote>
     <div style="text-align:left">
      MySQL的binlog（二进制日志）是MySQL数据库中非常重要的日志文件，它记录了所有对数据库的修改操作（如INSERT、UPDATE、DELETE等）。通过 binlog，我们可以实现数据恢复、主从复制、数据审计等功能。因此，定期备份binlog是数据库运维中的关键任务之一。
     </div>
    </blockquote>
    <h2 id="2wJo-1742052920185" style="text-align:left">
     <strong>
      1 binlog基本概念
     </strong>
    </h2>
    <h3 id="nw2H-1742052920187" style="text-align:left">
     <strong>
      1.1 binlog的作用
     </strong>
    </h3>
    <blockquote>
     <div style="text-align:left">
      binlog是MySQL的二进制日志文件，记录了所有对数据库的修改操作。具有以下特点：
     </div>
     <div style="text-align:left">
      <strong>
       记录内容：
      </strong>
      SQL语句或行数据的变化（取决于binlog_format配置)
     </div>
     <div style="text-align:left">
      <strong>
       作用：
      </strong>
     </div>
     <div>
      <ul style="margin-left:0; margin-right:0">
       <li style="text-align:left">
        数据恢复：当数据库发生误操作或数据丢失时，可以通过binlog恢复到指定时间点
       </li>
      </ul>
     </div>
     <div>
      <ul style="margin-left:0; margin-right:0">
       <li style="text-align:left">
        主从复制：binlog是实现MySQL主从复制的核心，从库通过读取主库的binlog来同步数据
       </li>
       <li style="text-align:left">
        数据审计：binlog记录了所有对数据库的修改操作，可以用于数据审计和问题排查
       </li>
      </ul>
     </div>
    </blockquote>
    <h3 id="Jiul-1742052920202" style="text-align:left">
     <strong>
      1.2 binlog的三种格式
     </strong>
    </h3>
    <div>
     <blockquote>
      <ul style="margin-left:0; margin-right:0">
       <li style="text-align:left">
        STATEMENT：记录SQL语句
       </li>
       <li style="text-align:left">
        ROW：记录每一行数据的变化
       </li>
       <li style="text-align:left">
        MIXED：混合模式，默认记录SQL语句，但在某些情况下记录行数据
       </li>
      </ul>
     </blockquote>
    </div>

+---------------+-------+
| log_bin       | ON    |
+---------------+-------+

    <h2 id="SL6d-1742052962212">
     2 启用binlog
    </h2>
    <p>
     在MySQL中，binlog默认是关闭的，要启用binlog，需要修改MySQL配置文件.
    </p>
    <h3 id="Iy6L-1742052962217">
     2.1 启用binlog操作步骤
    </h3>
    <pre><code>1.编辑MySQL配置文件（通常是/etc/my.cnf）
vim /etc/my.cnf
# [mysqld] 部分添加以下配置：
[mysqld]
log_bin = /data/mysql/log/mysql-bin.log  # 启用binlog，并指定文件路径
server_id = 1                        # 设置服务器唯一 ID（主从复制时需要）
binlog_format = STATEMENT                  # 设置binlog格式为STATEMENT
expire_logs_days = 7                 # binlog文件保留7天

2.重启MySQL服务
systemctl restart mysqld</code></pre>
    <h3 id="3PCf-1742052982495">
     2.2 验证是否启用
    </h3>
    <pre><code># 命令
SHOW VARIABLES LIKE 'log_bin';

# 输出示例
mysql&gt; SHOW VARIABLES LIKE 'log_bin';
+---------------+-------+
| Variable_name | Value |
+---------------+-------+
| log_bin       | ON    |
+---------------+-------+
1 row in set (0.01 sec)

mysql&gt; </code></pre>

    <h2>
     4 手动刷新binlog

    <h2 id="Ualg-1742053091970">
     5 binlog备份数据
    </h2>
    <h3 id="AkK6-1742053091972">
     5.1 导出指定时间范围的binlog
    </h3>
    <pre><code>mysqlbinlog --start-datetime="2025-03-15 00:00:00" --stop-datetime="2025-03-15 19:00:00" /data/mysql/log/mysql-bin.000002 &gt; /data/mysql/backup/binlog_20250315.sql

参数说明：
--start-datetime：指定开始时间
--stop-datetime：指定结束时间
/data/mysql/log/mysql-bin.000005：要导出的binlog文件
/data/mysql/tmp/backup/binlog_20250314.sql：导出的SQL文件路径</code></pre>
    <h3 id="6Bn6-1742053111591">
     5.2 导出指定位置范围的binlog
    </h3>
    <pre><code>mysqlbinlog --start-position=123 --stop-position=456 /data/mysql/log/mysql-bin.000002 &gt; /data/mysql/backup/binlog_position.sql</code></pre>
    <h3 id="NJwk-1742053148398">
     5.3 解码ROW格式的binlog
    </h3>
    <pre><code># 如果binlog格式为 ROW，需要使用--base64-output=DECODE-ROWS选项解码
mysqlbinlog --base64-output=DECODE-ROWS --verbose /data/mysql/log/mysql-bin.000002 &gt; /backup/binlog_decoded.sql</code></pre>
