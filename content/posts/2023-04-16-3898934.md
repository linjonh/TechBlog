     <span style="color: #666699;">
      <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
       今天将“软件测试和质量专栏”改为“软件工程专栏”，意味着自己的博客将扩展到一个更广的范围，虽然核心还围绕测试和质量，但包括软件需求工程、系统设计和项目管理等。所以特写一篇文章，作为开头和备忘。
      </span>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 24.25pt; text-align: justify;">
     什么是软件工程？
    </p>
    <p class="MsoNormal" style="text-indent: 24.25pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      <br/>
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 24.25pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      从上世纪
      <span lang="EN-US">
       90
      </span>
      年代初起，计算学科（
      <span lang="EN-US">
       Computing
      </span>
      ）的发展远远超越了计算机科学的边界，形成了计算机科学、计算机工程、软件工程、信息系统与信息技术等若干独立学科。计算机工程从电子工程学科中分离出来，旨在研究计算机硬件的相关工程问题，而软件工程从计算机科学中的一个学科方向发展成为与之并重的一门独立学科，重点研究如何以系统的、可控的、高效的方式开发和维护高质量软件的问题。
     </span>
    </p>
    <p style="margin: 0pt 0pt 0pt 20.5pt;">
     <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: ˎ̥;">
     </span>
    </p>
    <p style="margin: 0pt 0pt 0pt -1.55pt;">
     <strong>
      <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: ˎ̥;">
       1
      </span>
     </strong>
     <strong>
      <span style="font-size: 10.5pt; color: black;">
       ．软件工程的定义
      </span>
     </strong>
     <strong>
     </strong>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      软件工程学科诞生后，人们为软件工程给出了不同的定义，例如最早的定义是由
     </span>
     <span lang="EN" style="font-size: 10.5pt; color: black; font-family: Arial;">
      <a href="http://en.wikipedia.org/wiki/F.L._Bauer" rel="nofollow" title="F.L. Bauer">
       <span style="color: black;">
        F.L. Bauer
       </span>
      </a>
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      给出的，即“
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      软件工程是为了经济地获得能够在实际机器上高效运行的、可靠的软件而建立和应用一系列坚实的软件工程原则
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      ”。而美国
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      梅隆卡耐基大学软件工程研究所（
     </span>
     <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: Arial;">
      SEI
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      ）给出的定义则是
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      软件工程是以工程的形式应用计算机科学和数学原理，从而经济有效地解决软件问题。但目前普遍使用的软件工程定义是由
      <span lang="EN-US">
       IEEE
      </span>
      给出的，即软件工程是将系统性的、规范化的、可定量的方法应用于软件的开发、运行和维护。
     </span>
     <span style="font-size: 12.5pt; color: black; font-family: Arial;">
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      软件工程概念实际存在两层含义，从狭义概念看，软件工程着重体现在软件过程中所采用的工程方法和管理体系，例如，引入成本核算、质量管理和项目管理等，即将软件产品开发看作是一项工程项目所需要的系统工程学和管理学。从广义概念看，软件工程涵盖了软件生命周期中所有的工程方法、技术和工具，包括需求工程、设计、编程、测试和维护的全部内容，即完成一个软件产品所必备的思想、理论、方法、技术和工具。
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: 宋体;">
     </span>
    </p>
    <p style="margin: 0pt 0pt 0pt -1.55pt;">
     <strong>
      <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: ˎ̥;">
       2
      </span>
     </strong>
     <strong>
      <span style="font-size: 10.5pt; color: black;">
       ．软件工程的内涵
      </span>
     </strong>
     <strong>
     </strong>
    </p>
    <p class="MsoNormal" style="margin-bottom: 3.25pt; text-indent: 23.85pt; line-height: 120%;">
     <span style="font-size: 10.5pt; color: black; line-height: 120%; font-family: 宋体;">
      软件工程学科包含为完成软件需求、设计、构建、测试和维护所需的知识、方法和工具。
     </span>
     <span style="font-size: 10.5pt; color: black; line-height: 120%; font-family: 宋体;">
      软件工程不局限在理论之上，更重要在实践上，能够帮助软件组织协调团队、运用有限的资源，遵守已定义的软件工程规范，通过一系列可复用的、有效的方法，在规定的时间内达到预先设定的目标。针对软件工程的实施，无论是采用什么样的方法和工具，先进的软件工程思想始终是最重要的。只有在正确的工程思想指导下，才能制定正确的技术路线，才能正确地运用方法和工具达到软件工程或项目管理的既定目标。
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: 宋体;">
     </span>
    </p>
    <p style="margin: 0pt 0pt 0pt -1.55pt;">
     <strong>
      <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: ˎ̥;">
       3
      </span>
     </strong>
     <strong>
      <span style="font-size: 10.5pt; color: black;">
       ．软件工程是一门交叉性学科
      </span>
     </strong>
     <strong>
     </strong>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black;">
      软件工程是一门交叉性的工程学科，它是将计算机科学、数学、工程学和管理学等基本原理应用于软件的开发与维护中，其重点在于大型软件的分析与评价、规格说明、设计和演化，同时涉及管理、质量、创新、标准、个人技能、团队协作和专业实践等。从这个意义上看，软件工程可以看作由下列3部分组成。
     </span>
    </p>
    <ul>
     <li>
      <span style="font-size: 10.5pt; color: black;">
       计算机科学和数学用于构造软件的模型与算法；
      </span>
     </li>
     <li>
      <span style="font-size: 10.5pt; color: black;">
       工程科学用于制定规范、设计范型、评估成本以及确定权衡等；
      </span>
     </li>
     <li>
      <span style="font-size: 10.5pt; color: black;">
       管理科学用于计划、资源、质量、成本等管理。
      </span>
     </li>
    </ul>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black;">
      例如，计算机辅助软件工程（Computer Aided Software Engineering，CASE）是一组工具和方法的集合
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      ，可以辅助软件生命周期各阶段进行的软件开发活动。
     </span>
     <span lang="EN-US" style="font-size: 10.5pt; color: black;">
      CASE
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      吸收了
     </span>
     <span lang="EN-US" style="font-size: 10.5pt; color: black;">
      CAD
     </span>
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      （计算机辅助设计）、软件工程、操作系统、数据库、网络和许多其他计算机领域的原理和技术。这个例子也体现了这一点——软件工程是学科交叉的、集成和综合的领域。
     </span>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <img alt="" src="https://p-blog.csdn.net/images/p_blog_csdn_net/KerryZhu/EntryImages/20081019/SE.gif"/>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: 宋体;">
     </span>
    </p>
    <p style="margin: 0pt 0pt 0pt -1.55pt;">
     <strong>
      <span lang="EN-US" style="font-size: 10.5pt; color: black; font-family: ˎ̥;">
       4
      </span>
     </strong>
     <strong>
      <span style="font-size: 10.5pt; color: black;">
       ．软件工程学科范围
      </span>
     </strong>
     <strong>
     </strong>
    </p>
    <p class="MsoNormal" style="text-indent: 21.85pt; text-align: justify;">
     <span style="font-size: 10.5pt; color: black; font-family: 宋体;">
      如果从知识领域看，软件工程学科是以软件方法和技术为核心，涉及计算机的硬件体系、系统基础平台等相关领域，同时还要涉及一些应用领域和通用的管理学科、组织行为学科。例如，通过应用领域的知识帮助我们理解用户的需求，从而可以根据需求来设计软件的功能。在软件工程中必然要涉及组织中应用系统的部署和配置所面临的实际问题，同时又必须不断促进知识的更新和理论的创新。为了真正解决实际问题，需要在理论和应用上获得最佳平衡。
     </span>
    </p>
