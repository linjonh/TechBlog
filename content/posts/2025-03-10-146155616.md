    <p>
     在面试中，关于
     <strong>
      React 中的
      <code>
       useMemo
      </code>
      和
      <code>
       useCallback
      </code>
      的区别
     </strong>
     是一个常见的问题。
    </p>
    <hr/>
    <h4>
     <a id="useMemo__useCallback__4">
     </a>
     <strong>
      <code>
       useMemo
      </code>
      和
      <code>
       useCallback
      </code>
      的区别
     </strong>
    </h4>
    <h5>
     <a id="1__6">
     </a>
     <strong>
      1. 功能定义
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         useMemo
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        用于缓存计算结果，避免在每次组件渲染时重新计算复杂的值。
       </li>
       <li>
        它接受一个计算函数和一个依赖数组，只有当依赖项发生变化时，才会重新计算。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         useCallback
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        用于缓存函数实例，避免在每次组件渲染时创建新的函数。
       </li>
       <li>
        它接受一个函数和一个依赖数组，只有当依赖项发生变化时，才会返回新的函数。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2__15">
     </a>
     <strong>
      2. 使用场景
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         useMemo
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        适用于需要缓存计算结果的场景，尤其是计算开销较大的值，如复杂的数学计算、对象创建等。
       </li>
       <li>
        示例：
        <pre><code class="prism language-javascript"><span class="token keyword">const</span> memoizedValue <span class="token operator">=</span> <span class="token function">useMemo</span><span class="token punctuation">(</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token operator">=&gt;</span> <span class="token function">computeExpensiveValue</span><span class="token punctuation">(</span>a<span class="token punctuation">,</span> b<span class="token punctuation">)</span><span class="token punctuation">,</span> <span class="token punctuation">[</span>a<span class="token punctuation">,</span> b<span class="token punctuation">]</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         useCallback
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        适用于需要缓存函数引用的场景，特别是当函数作为
        <code>
         props
        </code>
        传递给子组件时，防止子组件因函数引用变化而重新渲染。
       </li>
       <li>
        示例：
        <pre><code class="prism language-javascript"><span class="token keyword">const</span> memoizedCallback <span class="token operator">=</span> <span class="token function">useCallback</span><span class="token punctuation">(</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token operator">=&gt;</span> <span class="token punctuation">{<!-- --></span>
  <span class="token function">doSomething</span><span class="token punctuation">(</span>a<span class="token punctuation">,</span> b<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">,</span> <span class="token punctuation">[</span>a<span class="token punctuation">,</span> b<span class="token punctuation">]</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="3__32">
     </a>
     <strong>
      3. 性能优化
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         useMemo
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过缓存计算结果，减少不必要的计算，从而提高性能。
       </li>
       <li>
        适用于依赖项变化较少的场合。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         useCallback
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过缓存函数引用，避免子组件因父组件的重新渲染而收到新的函数引用，从而减少子组件的不必要渲染。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="4__40">
     </a>
     <strong>
      4. 本质关系
     </strong>
    </h5>
    <ul>
     <li>
      本质上，
      <code>
       useCallback(fn, deps)
      </code>
      与
      <code>
       useMemo(() =&gt; fn, deps)
      </code>
      是相同的，都是返回一个记忆化的值或函数。
     </li>
    </ul>
    <h5>
     <a id="5__43">
     </a>
     <strong>
      5. 注意事项
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        过度使用问题
       </strong>
       ：
      </p>
      <ul>
       <li>
        不要过度使用
        <code>
         useMemo
        </code>
        和
        <code>
         useCallback
        </code>
        ，因为它们会增加代码复杂性。
       </li>
       <li>
        对于简单的函数或计算，React 通常会自动优化，无需手动缓存。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        依赖项管理
       </strong>
       ：
      </p>
      <ul>
       <li>
        确保依赖数组中的值是必要的，否则可能导致缓存失效或不必要的重新计算。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_53">
     </a>
     <strong>
      回答要点
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        useMemo
       </code>
      </strong>
      用于缓存计算结果，避免在每次渲染时重新计算复杂的值。
     </li>
     <li>
      <strong>
       <code>
        useCallback
       </code>
      </strong>
      用于缓存函数实例，避免因函数引用变化导致子组件的不必要渲染。
     </li>
     <li>
      两者本质上相似，
      <code>
       useCallback(fn, deps)
      </code>
      可以通过
      <code>
       useMemo(() =&gt; fn, deps)
      </code>
      来实现。
     </li>
     <li>
      使用时需注意依赖项的管理，避免过度使用。
     </li>
    </ul>
