    <h3>
     <a id="_1">
     </a>
     一、基本概念
    </h3>
    <h4>
     <a id="11_PageRequest_3">
     </a>
     1.1 PageRequest分页
    </h4>
    <p>
     PageRequest是Spring Data提供的分页实现，它是基于页码的分页方式（Page-based pagination）。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">new</span> <span class="token class-name">PageRequest</span><span class="token punctuation">(</span>page<span class="token punctuation">,</span> size<span class="token punctuation">)</span>
<span class="token comment">// page: 页码，从0开始</span>
<span class="token comment">// size: 每页大小</span>
</code></pre>
    <h4>
     <a id="12_SkipLimit_11">
     </a>
     1.2 Skip/Limit分页
    </h4>
    <p>
     Skip/Limit是MongoDB原生的分页方式，它是基于偏移量的分页（Offset-based pagination）。
    </p>
    <pre><code class="prism language-java">query<span class="token punctuation">.</span><span class="token function">skip</span><span class="token punctuation">(</span>offset<span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">limit</span><span class="token punctuation">(</span>size<span class="token punctuation">)</span>
<span class="token comment">// offset: 跳过的记录数</span>
<span class="token comment">// size: 获取的记录数</span>
</code></pre>
    <h3>
     <a id="_19">
     </a>
     二、主要区别
    </h3>
    <h4>
     <a id="21__21">
     </a>
     2.1 使用方式
    </h4>
    <ol>
     <li>
      PageRequest:
     </li>
    </ol>
    <pre><code class="prism language-java"><span class="token comment">// 获取第一页，每页10条</span>
<span class="token class-name">Pageable</span> pageable <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">PageRequest</span><span class="token punctuation">(</span><span class="token number">0</span><span class="token punctuation">,</span> <span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 获取第二页，每页10条</span>
<span class="token class-name">Pageable</span> pageable <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">PageRequest</span><span class="token punctuation">(</span><span class="token number">1</span><span class="token punctuation">,</span> <span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <ol start="2">
     <li>
      Skip/Limit:
     </li>
    </ol>
    <pre><code class="prism language-java"><span class="token comment">// 获取前10条</span>
query<span class="token punctuation">.</span><span class="token function">skip</span><span class="token punctuation">(</span><span class="token number">0</span><span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">limit</span><span class="token punctuation">(</span><span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token comment">// 获取第11-20条</span>
query<span class="token punctuation">.</span><span class="token function">skip</span><span class="token punctuation">(</span><span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">limit</span><span class="token punctuation">(</span><span class="token number">10</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <h4>
     <a id="22__38">
     </a>
     2.2 参数计算
    </h4>
    <ol>
     <li>
      PageRequest:
     </li>
    </ol>
    <pre><code class="prism language-java">page <span class="token operator">=</span> pageNumber <span class="token operator">-</span> <span class="token number">1</span>  <span class="token comment">// pageNumber从1开始</span>
size <span class="token operator">=</span> pageSize
</code></pre>
    <ol start="2">
     <li>
      Skip/Limit:
     </li>
    </ol>
    <pre><code class="prism language-java">offset <span class="token operator">=</span> <span class="token punctuation">(</span>pageNumber <span class="token operator">-</span> <span class="token number">1</span><span class="token punctuation">)</span> <span class="token operator">*</span> pageSize
limit <span class="token operator">=</span> pageSize
</code></pre>
    <h4>
     <a id="23__51">
     </a>
     2.3 适用场景
    </h4>
    <h5>
     <a id="PageRequest_53">
     </a>
     PageRequest适用场景：
    </h5>
    <ol>
     <li>
      需要与Spring Data框架深度集成
     </li>
     <li>
      需要获取分页的额外信息（总页数、是否是最后一页等）
     </li>
     <li>
      数据量较小，页码跨度不大的场景
     </li>
     <li>
      用户界面基于页码导航的场景
     </li>
    </ol>
    <h5>
     <a id="SkipLimit_59">
     </a>
     Skip/Limit适用场景：
    </h5>
    <ol>
     <li>
      需要更灵活的分页控制
     </li>
     <li>
      数据量大，但主要关注连续分页的场景
     </li>
     <li>
      无限滚动加载的场景
     </li>
     <li>
      需要优化性能的场景
     </li>
    </ol>
    <h3>
     <a id="_65">
     </a>
     三、性能考虑
    </h3>
    <h4>
     <a id="31_PageRequest_67">
     </a>
     3.1 PageRequest的性能特点
    </h4>
    <ol>
     <li>
      <p>
       优点：
      </p>
      <ul>
       <li>
        与Spring Data完美集成
       </li>
       <li>
        提供丰富的分页信息
       </li>
       <li>
        代码可读性好
       </li>
      </ul>
     </li>
     <li>
      <p>
       缺点：
      </p>
      <ul>
       <li>
        大页码时性能较差
       </li>
       <li>
        内存占用相对较高
       </li>
       <li>
        不适合大数据量分页
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <a id="32_SkipLimit_78">
     </a>
     3.2 Skip/Limit的性能特点
    </h4>
    <ol>
     <li>
      <p>
       优点：
      </p>
      <ul>
       <li>
        实现简单直接
       </li>
       <li>
        内存占用低
       </li>
       <li>
        适合大数据量分页
       </li>
      </ul>
     </li>
     <li>
      <p>
       缺点：
      </p>
      <ul>
       <li>
        skip值较大时性能会下降
       </li>
       <li>
        不提供总页数等信息
       </li>
       <li>
        需要自行处理边界情况
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_89">
     </a>
     四、最佳实践建议
    </h3>
    <h4>
     <a id="41__91">
     </a>
     4.1 选择建议
    </h4>
    <ol>
     <li>
      <p>
       小型应用，数据量不大：
      </p>
      <ul>
       <li>
        推荐使用PageRequest，开发更便捷
       </li>
      </ul>
     </li>
     <li>
      <p>
       大型应用，数据量大：
      </p>
      <ul>
       <li>
        推荐使用Skip/Limit，性能更好
       </li>
       <li>
        考虑使用游标或时间戳分页
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <a id="42__99">
     </a>
     4.2 性能优化建议
    </h4>
    <ol>
     <li>
      添加适当的索引
     </li>
     <li>
      避免大的偏移量
     </li>
     <li>
      考虑使用游标分页
     </li>
     <li>
      合理设置每页大小
     </li>
    </ol>

    <h3>
     <a id="_125">
     </a>
     五、总结
    </h3>
    <ol>
     <li>
      <p>
       PageRequest和Skip/Limit各有优势，选择时需要考虑：
      </p>
      <ul>
       <li>
        数据量大小
       </li>
       <li>
        性能要求
       </li>
       <li>
        业务场景
       </li>
       <li>
        开发便利性
       </li>
      </ul>
     </li>
     <li>
      <p>
       实际应用建议：
      </p>
      <ul>
       <li>
        数据量&lt;10万，用户习惯页码翻页：选择PageRequest
       </li>
       <li>
        数据量&gt;10万，或需要无限滚动：选择Skip/Limit
       </li>
       <li>
        数据量&gt;100万：考虑使用游标分页
       </li>
      </ul>
     </li>
     <li>
      <p>
       性能优化核心：
      </p>
      <ul>
       <li>
        合理使用索引
       </li>
       <li>
        避免大偏移量
       </li>
       <li>
        控制每页数据量
       </li>
       <li>
        考虑缓存策略
       </li>
      </ul>
     </li>
    </ol>
