    <h3>
     <a id="1__28">
     </a>
     1. 概述
    </h3>
    <h4>
     <a id="11_Interceptor_30">
     </a>
     1.1 什么是拦截器（Interceptor）？
    </h4>
    <p>
     拦截器是
     <strong>
      Spring MVC
     </strong>
     框架的组件，基于
     <strong>
      AOP（面向切面编程）
     </strong>
     实现。它允许在请求处理的不同阶段（如Controller方法执行前后）插入自定义逻辑。
    </p>
    <h4>
     <a id="12_Filter_33">
     </a>
     1.2 什么是过滤器（Filter）？
    </h4>
    <p>
     过滤器是
     <strong>
      Java Servlet规范
     </strong>
     定义的组件，作用于所有进入容器的请求（如Tomcat）。它可以在请求到达Servlet前或响应返回客户端前进行预处理和后处理。
    </p>
    <h4>
     <a id="13__36">
     </a>
     1.3 核心区别
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        拦截器（Interceptor）
       </th>
       <th>
        过滤器（Filter）
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         所属框架
        </strong>
       </td>
       <td>
        Spring MVC
       </td>
       <td>
        Servlet API
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         作用范围
        </strong>
       </td>
       <td>
        仅Spring MVC管理的请求
       </td>
       <td>
        所有请求（包括静态资源）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         依赖
        </strong>
       </td>
       <td>
        依赖Spring容器
       </td>
       <td>
        依赖Servlet容器（如Tomcat）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         执行时机

    <h3>
     <a id="2__47">
     </a>
     2. 使用场景
    </h3>
    <h4>
     <a id="21__49">
     </a>
     2.1 拦截器的典型应用
    </h4>
    <ul>
     <li>
      <strong>
       日志记录
      </strong>
      ：记录请求参数、响应时间。
     </li>
     <li>
      <strong>
       权限验证
      </strong>
      ：检查用户是否登录或拥有权限。
     </li>
     <li>
      <strong>
       事务管理
      </strong>
      ：在Controller方法前后开启/提交事务。
     </li>
     <li>
      <strong>
       性能监控
      </strong>
      ：统计接口耗时。
     </li>
    </ul>
    <h4>
     <a id="22__55">
     </a>
     2.2 过滤器的典型应用
    </h4>
    <ul>
     <li>
      <strong>
       全局字符编码
      </strong>
      ：统一设置请求/响应的编码（如UTF-8）。
     </li>
     <li>
      <strong>
       跨域处理
      </strong>
      ：添加CORS响应头。
     </li>
     <li>
      <strong>
       XSS防御
      </strong>
      ：过滤请求参数中的恶意脚本。
     </li>
     <li>
      <strong>
       请求压缩
      </strong>
      ：对响应内容进行GZIP压缩。
     </li>
    </ul>

    <hr/>
    <h3>
     <a id="4__174">
     </a>
     4. 执行顺序与流程
    </h3>
    <h4>
     <a id="41__176">
     </a>
     4.1 执行顺序
    </h4>
    <ol>
     <li>
      <strong>
       过滤器（FilterChain）
      </strong>
      → 2.
      <strong>
       拦截器（preHandle）
      </strong>
      → 3.
      <strong>
       Controller方法
      </strong>
      → 4.
      <strong>
       拦截器（postHandle）
      </strong>
      → 5.
      <strong>
       视图渲染
      </strong>
      → 6.
      <strong>
       拦截器（afterCompletion）
      </strong>
      → 7.
      <strong>
       过滤器后续处理
      </strong>
     </li>
    </ol>
    <h4>
     <a id="42__179">
     </a>
     4.2 流程图
    </h4>
    <pre><code>客户端 → Filter.doFilter() → Interceptor.preHandle()
       → Controller → Interceptor.postHandle()
       → 视图渲染 → Interceptor.afterCompletion()
       → Filter.doFilter()后续处理 → 客户端
</code></pre>

     5. 常见问题与解决方案
    </h3>

    <ul>
     <li>
      <strong>

      </strong>

      <code>
       registry.addInterceptor()

     <li>
      <strong>
       过滤器
      </strong>

      <code>

    <h3>
     <a id="6__249">
     </a>
     6. 总结
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        选择拦截器还是过滤器？
       </strong>
      </p>
      <ul>
       <li>
        需要访问Spring上下文或Controller信息 → 拦截器。
       </li>
       <li>
        需处理所有请求（包括静态资源） → 过滤器。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        最佳实践
       </strong>
       ：
      </p>
      <ul>
       <li>
        优先使用拦截器处理业务相关逻辑。
       </li>
       <li>
        使用过滤器处理底层Servlet容器的任务（如编码、压缩）。
       </li>
      </ul>
     </li>
    </ul>
