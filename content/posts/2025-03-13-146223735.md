   <div class="htmledit_views" id="content_views">
    <p>
     <em>
      我们付出一些成本，时间的或者其他，最终总能收获一些什么。
     </em>
    </p>
    <p>
     <strong>
      升级背景：
     </strong>
    </p>
    <div>
     <span style="color:#000000">
      近日，新华三盾山实验室监测到 Apache 官方修复了一个远程代码执行漏洞
     </span>
     <span style="color:#000000">
      (CVE-2025-24813)
     </span>
     <span style="color:#000000">
      ，其CVSS3 漏洞评分为
     </span>
     <span style="color:#000000">
      7.5
     </span>
     <span style="color:#000000">
      。
     </span>
    </div>
    <div>
     <div>
     </div>
     <div>
      <span style="color:#000000">
       <strong>
        影响范围
       </strong>
      </span>
     </div>
     <div>
      <span style="color:#000000">
       9.0.0.M1    ≤ version   ≤
      </span>
      <span style="color:#000000">
       9.0.98
      </span>
     </div>
     <div>
      <span style="color:#000000">
       10.1.0-M1  ≤
      </span>
      <span style="color:#000000">
       version
      </span>
      <span style="color:#000000">
       ≤
      </span>
      <span style="color:#000000">
       10.1.34
      </span>
     </div>
     <div>
      <span style="color:#000000">
       11.0.0-M1  ≤
      </span>
      <span style="color:#000000">
       version
      </span>
      <span style="color:#000000">
       ≤
      </span>
      <span style="color:#000000">
       11.0.2
      </span>
     </div>
    </div>
    <div>
     <div>
     </div>
     <div>
      <span style="color:#0d0d0d">
       <strong>
        官方补丁
       </strong>
      </span>
     </div>
     <div>
      <span style="color:#000000">
       Apache 基金会官方已发布漏洞公告，请尽快前往下载更新补丁：
      </span>
     </div>
     <div>
      <span style="color:#0000ff">
       https://lists.apache.org/thread/j5fkjv2k477os90nczf2v9l61fb0kkgq
      </span>
     </div>
     <div>
      <span style="color:#000000">
       或升级至如下版本：
      </span>
     </div>
     <div>
      <span style="color:#000000">
       Apache Tomcat &gt;= 11.0.3
      </span>
     </div>
     <div>
      <span style="color:#000000">
       Apache Tomcat &gt;= 10.1.35
      </span>
     </div>
     <div>
      <span style="color:#000000">
       Apache Tomcat &gt;= 9.0.99
      </span>
     </div>
     <div>
     </div>
     <div>
      <strong>
       参考链接
      </strong>
     </div>
     <div>
      <div>
       <span style="color:#0000ff">
        https://lists.apache.org/thread/j5fkjv2k477os90nczf2v9l61fb0kkgq
       </span>
      </div>
      <div>
       <span style="color:#0000ff">
        https://nvd.nist.gov/vuln/detail/CVE-2025-24813
       </span>
      </div>
      <div>
      </div>
      <div>
       <span style="color:#0d0016">
       </span>
       <span style="color:#0d0016">
        根据以上的背景，需要自行检查，tomcat的版本。
       </span>
      </div>
      <div>
       直接通过 https://ip，查看tomcat版本为9.0.84，无更新补丁，只能走升级
      </div>
      <div>
       <img alt="" height="907" src="https://i-blog.csdnimg.cn/direct/3b6aea4a05bc42e8836be98168a57938.png" width="1290">
       </img>
      </div>
      <div>
       <a href="https://tomcat.apache.org/download-90.cgi" rel="nofollow" title="Apache Tomcat® - Apache Tomcat 9 Software Downloads">
        Apache Tomcat® - Apache Tomcat 9 Software Downloads
       </a>
       ，进行下载，由于是linux系统下载的是apache-tomcat-9.0.102.tar.gz
      </div>
      <div>
       1.将tomcat9.0.102的tar包上传到服务器中，并在该目录下，对其进行解压
      </div>
      <div>
      </div>
      <div>
       cd /home/soft/tomcat9
      </div>
      <div>
       tar -zxvf apache-tomcat-9.0.102.tar.gz
      </div>
      <div>
      </div>
      <div>
       2.找到在原9.0.84版本conf下的server.xml配置文件，以此确定pfx证书的路径，确认是否需要修改server.xml配置文件中证书的路径。
      </div>
      <div>
       <img alt="" height="227" src="https://i-blog.csdnimg.cn/direct/a5c8027eb30541a3abbe241859199489.png" width="679">
       </img>
      </div>
      <div>
      </div>
      <div>
       3.取消GeoScene server /portal联合配置的web adaptor
      </div>
      <div>
       <a class="link-info" href="https://blog.csdn.net/weixin_40625478/article/details/112871472" title="可查看该文章">
        可查看该文章
       </a>
      </div>
      <div>
      </div>
      <div>
       解除server的联合
      </div>
      <div>
       <a href="https://10.240.71.11/server/admin/" rel="nofollow" title="https://IP或域名/server/admin/">
        https://IP或域名/server/admin/
       </a>
      </div>
      <div>
       根据server的用户名和密码进行登录，登录后进入system--webadaptors，如下进行注销操作
      </div>
      <div>
       <img alt="" height="262" src="https://i-blog.csdnimg.cn/direct/97ce24aef45d45629ae9821cac3d3263.png" width="690">
       </img>
      </div>
      <div>
       根据portal的用户名和密码进行登录，登录后，如上进行注销操作
      </div>
      <div>
       <img alt="" height="467" src="https://i-blog.csdnimg.cn/direct/bcf17d119fd54cb9a01a82d4055f62c7.png" width="959">
       </img>
      </div>
      <div>
       4.注销联合托管后，将tomcat9.0.84进行停止
      </div>
      <div>
       cd /home/soft/tomcat9/bin
      </div>
      <div>
       ./shutdown.sh
      </div>
      <div>
      </div>
      <div>
       5.将tomcat9.0.84中的两份war文件拷贝到tomcat9.0.102版本的webapps下，将原server.xml拷贝到conf下，注意里面证书的路径是不是对的。
      </div>
      <div>
      </div>
      <div>
       6.启动tomcat9.0.102，并在网页上进行验证
      </div>
      <div>
      </div>
      <div>
       <img alt="" height="304" src="https://i-blog.csdnimg.cn/direct/941bfec9f3e740b0924f33d42ec646de.png" width="756">
       </img>
      </div>
      <div>
       7.重新为portal和server配置GeoSceneWebAdaptor
      </div>
      <div>
      </div>
      <div>
      </div>
     </div>
    </div>
   </div>
