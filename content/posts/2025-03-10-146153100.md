    <h3>
     <a id="_21">
     </a>
     论文
    </h3>
    <p>
     论文：https://arxiv.org/pdf/2311.10982
     <br/>
     项目：https://makepixelsdance.github.io/
    </p>
    <h3>
     <a id="_24">
     </a>
     摘要
    </h3>
    <p>
     目前的视频生成方法
     <strong>
      主要侧重于文本到视频的生成
     </strong>
     ，这样生成的
     <strong>
      视频片段往往动作极少
     </strong>
     。作者认为，仅依赖文本指令进行视频生成是不够且并非最优的。
    </p>
    <p>
     在本文中，作者介绍了PixelDance，这是一种
     <strong>
      基于扩散模型
     </strong>
     的新颖方法，它
     <strong>
      将视频生成的第一帧和最后一帧的图像指令与文本指令相结合。
     </strong>
    </p>
    <blockquote>
     <p>
      这并不是首例提出依赖文本指令进行视频生成是不够的论文，在Meta的Emu Video中也提到了，但是两者的生成策略略有不同：
     </p>
     <ul>
      <li>
       Emu Video根据输入文本提示生成图像；之后再基于图像和文本的强条件生成视频。
      </li>
      <li>
       输入的内容包括，文本，作为视频第一帧的图像，作为视频最后一帧的图像。
      </li>
     </ul>
     <p>
      Emu Video模型的实现参考历史文章：
      <a href="https://blog.csdn.net/haopinglianlian/article/details/145239505">
       AIGC视频生成明星——Emu Video模型
       <br/>
      </a>
     </p>
    </blockquote>
    <h3>
     <a id="_36">
     </a>
     引言
    </h3>
    <h4>
     <a id="_37">
     </a>
     输入
    </h4>
    <p>
     作者提出了一种
     <code>
      新颖的视频生成输入
     </code>
     ，其输入包括三部分：
    </p>
    <ol>
     <li>
      <strong>
       文本指令
      </strong>
      ；
     </li>
     <li>
      <strong>
       视频片段的第一帧
      </strong>
      引入图像指令。第一帧的图像指令描绘了视频片段的主要场景。
     </li>
     <li>
      <strong>
       视频片段的最后一帧
      </strong>
      引入图像指令。最后一帧的图像指令（在训练和推理中可选使用）描绘了片段的结尾，并为生成提供了额外的控制。
     </li>
    </ol>
    <p>
     这种方法还可以
     <code>
      创建更长的视频
     </code>
     ，在这种情况下，模型会被多次应用，
     <strong>
      前一个片段的最后一帧将作为后续片段的第一帧指令
     </strong>
     。
    </p>
    <h4>
     <a id="_46">
     </a>
     训练和推理时的数据处理
    </h4>
    <p>
     相比于其他模型提出使用高度描述性的文本注释以更好地遵循文本指令。图像指令使用真实视频帧，更加直接和易于获取。
    </p>
    <p>
     PixelDance是一种基于
     <strong>
      潜在扩散模型
     </strong>
     的视频生成方法，以
     <strong>
      &lt;文本，第一帧，最后帧&gt;指令
     </strong>
     为条件。
    </p>
    <p>
     <strong>
      文本指令
     </strong>
     由预训练的文本编码器编码，并通过
     <strong>
      交叉注意力机制
     </strong>
     集成到扩散模型中。
     <br/>
     <strong>
      图像指令
     </strong>
     由预训练的
     <strong>
      VAE编码器编码
     </strong>
     ，并与扰动后的视频
     <strong>
      潜在表示或高斯噪声连接，作为扩散模型的输入
     </strong>
     ，如下所示：【在下节“方法”中会详细介绍】。
    </p>
    <ul>
     <li>
      <p>
       第一帧
      </p>
      <ul>
       <li>
        在训练中，使用（真实）第一帧来强制模型严格遵循指令，保持连续视频片段之间的连续性。
       </li>
       <li>
        在推理中，这个指令可以来源：
        <ol>
         <li>
          <strong>
           从文本到图像（T2I）模型中获得
          </strong>
          ；
         </li>
         <li>
          <strong>
           由用户提供
          </strong>
          。
         </li>
        </ol>
       </li>
      </ul>
     </li>
     <li>
      <p>
       最后一帧
      </p>
      <ul>
       <li>
        在训练中，我们有意
        <strong>
         避免鼓励模型完全复制最后一帧指令
        </strong>
        ，因为
        <strong>
         在推理中很难提供完美的最后一帧
        </strong>
        ，在最后一帧的处理上，作者开发了三种技术：
        <ul>
         <li>
          第一，在训练中，最后一帧指令是从视频片段的最后三帧（真实帧）中随机选择的。
         </li>
         <li>
          第二，我们向指令中引入噪声，以减少对指令的依赖并提高模型的鲁棒性。
         </li>
         <li>
          第三，在训练中，我们以一定的概率（例如25%）随机丢弃最后一帧指令。
         </li>
        </ul>
       </li>
       <li>
        在推理中，用户可以
        <strong>
         使用基本的图像编辑工具
        </strong>
        创建这种指令。模型能够适应用户提供的粗略草图作为指导，而不需要提供完美的最后一帧。简单而有效的推理采样策略如下（最后一帧指令的影响可以通过T来调整。）：
        <ul>
         <li>
          在前T步去噪步骤中，利用最后一帧指令引导视频生成朝着期望的结束状态发展。
         </li>
         <li>
          在剩余的步骤中，丢弃该指令，使模型能够生成更具时间连贯性的视频。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_69">
     </a>
     总结
    </h4>
    <p>
     作者在文中说明了其主要贡献可以总结如下：
    </p>
    <ul>
     <li>
      提出了一种基于扩散模型的新颖视频生成方法PixelDance，它将第一帧和最后一帧的图像指令与文本指令相结合（还可以生成长视频，这一点是
      <strong>
       最重要
      </strong>
      的）。
     </li>
     <li>
      为PixelDance开发了训练和推理技术，这不仅有效地提高了生成视频的质量，而且为用户提供了对视频生成过程更多的控制。
     </li>
     <li>
      在公开数据上训练的模型在生成具有复杂场景和动作的高动态视频方面表现出色，为视频生成设定了新的标准。
     </li>
    </ul>
    <h3>
     <a id="_75">
     </a>
     相关工作
    </h3>
    <h4>
     <a id="_76">
     </a>
     视频生成
    </h4>
    <p>
     扩散模型在逼真的文本到图像生成方面取得了显著进展，其表现出比GAN更强的鲁棒性，并且与基于Transformer的模型相比需要更少的参数。潜在扩散模型被提出通过在压缩的低维潜在空间中训练扩散模型来减轻计算负担。
    </p>
    <p>
     对于视频生成，
     <strong>
      以往的研究
     </strong>
     通常在
     <strong>
      预训练的文本到图像扩散模型的2D UNet
     </strong>
     中
     <strong>
      添加时间卷积层和时间注意力层
     </strong>
     【如Gen-1和Emu Video等】。尽管这些进展通过集成超分辨率模块为高分辨率视频的生成铺平了道路，但生成的视频具有简单、动作极少的特点。
    </p>
    <p>
     最近，视频编辑领域取得了显著进展，特别是在保持视频原始结构和运动的同时进行内容修改方面，但存在下面的问题：
    </p>
    <ul>
     <li>
      寻找合适的参考视频进行编辑非常耗时。
     </li>
     <li>
      限制了创作的范围，因为它排除了合成全新内容（例如，一只北极熊走在长城上）的可能性。
     </li>
    </ul>
    <h4>
     <a id="_85">
     </a>
     长视频生成
    </h4>
    <p>
     长视频生成是一项更具挑战性的任务，它要求连续的视频片段之间无缝过渡，并且场景和角色具有长期一致性。通常有两种方法：
    </p>
    <ol>
     <li>
      自回归方法采用滑动窗口，以前一个片段为条件生成新的片段。自回归方法由于时间上的误差累积容易导致质量下降。
     </li>
     <li>
      分层方法首先生成稀疏帧，然后插值中间帧。需要长视频进行训练，由于在线视频中频繁的镜头变化，这些长视频很难获得。
     </li>
    </ol>
    <p>
     在本文中，PixelDance以
     <strong>
      自回归的方式生成连续的视频片段
     </strong>
     ，并且在合成长期一致的帧方面比现有模型
     <strong>
      表现更优
     </strong>
     。同时，我们提倡用户积极参与生成过程，就像电影导演一样，以确保生成的内容与用户的期望紧密一致。
    </p>
    <h3>
     <a id="_92">
     </a>
     方法
    </h3>
    <p>
     本文试图让模型
     <strong>
      专注于学习视频内容的动态变化
     </strong>
     ，以生成具有丰富动作的视频。将
     <strong>
      第一帧和最后一帧的图像指令与文本指令相结合
     </strong>
     用于视频生成，并且我们有效地利用
     <strong>
      公开视频数据
     </strong>
     进行训练。我们将在本节详细阐述
     <strong>
      模型架构
     </strong>
     和
     <strong>
      量身定制的训练和推理技术
     </strong>
     。
    </p>
    <h4>
     <a id="_95">
     </a>
     模型架构
    </h4>
    <h5>
     <a id="_96">
     </a>
     潜在扩散架构
    </h5>
    <p>
     论文中使用变分自动编码器（VAE）在潜在空间中进行去噪训练，以减轻计算负担。同时采用广泛使用的
     <strong>
      2D UNet作为扩散模型
     </strong>
     ，它由一系列
     <strong>
      空间下采样层
     </strong>
     和一系列带有
     <strong>
      跳跃连接
     </strong>
     的
     <strong>
      空间上采样层
     </strong>
     构成。它由两个基本块组成：
    </p>
    <ul>
     <li>
      2D卷积块
     </li>
     <li>
      2D注意力块。
     </li>
    </ul>
    <p>
     通过
     <strong>
      插入时间层将2D UNet扩展为3D变体
     </strong>
     ，实现如下：
    </p>
    <ul>
     <li>
      在2D卷积层之后
      <strong>
       添加沿时间维度的1D卷积层
      </strong>
      ；
     </li>
     <li>
      在2D注意力层之后
      <strong>
       添加沿时间维度的1D注意力层(
       <code>
        双向自注意力机制
       </code>
       )
      </strong>
      。
     </li>
    </ul>
    <blockquote>
     <p>
      Runway的Gen-1和Meta的Emu Video也是如此实现的。
     </p>
    </blockquote>
    <p>
     该模型进行
     <strong>
      图像和视频联合训练
     </strong>
     ，以在空间维度上保持高保真的生成能力。对于图像输入，1D时间操作被禁用。
    </p>
    <blockquote>
     <p>
      图像输入主要涉及两类：训练时的
      <strong>
       图像指令输入
      </strong>
      以及
      <strong>
       联合训练时的普通图像输入
      </strong>
      。
     </p>
    </blockquote>
    <p>
     我们使用预训练的
     <strong>
      CLIP文本编码器对文本指令进行编码
     </strong>
     ，并通过UNet中的
     <strong>
      交叉注意力层
     </strong>
     将嵌入文本注入，其中
     <strong>
      隐藏状态作为查询
     </strong>
     ，
     <strong>
      文本作为键和值
     </strong>
     。
    </p>

     两种生成模式
    </h4>
    <p>
     第一种是基础模式(Basic Mode)【和Emu Video 一样】，用户只需要提供一张指导图片+文本描述，PixelDance 就可以生成有高度一致性且有丰富动态性的视频，其中指导图片可以是真实图片，也可以利用现有的文生图模型生成，如下所示：

    </p>
    <p>
     第二种是高级魔法模式（Magic Mode），给了用户更多发挥想象力和创造力的空间。在这种模式下，用户需要提供两张指导图片+文本描述，可以更好地生成更有难度的各种炫酷特效镜头，如下所示：
     <br/>

    </p>
    <h4>
     <a id="_144">
     </a>
     更多应用
    </h4>
    <h5>
     <a id="_145">
     </a>
     草图指令
    </h5>
    <p>
     我们提出的方法可以扩展到其他类型的图像指令，如语义图、图像草图、人体姿态和边界框。为了证明这一点，我们以图像草图为例，使用图像草图作为最后一帧指令对PixelDance进行微调。结果如图10的前两行所示，表明一个简单的草图图像能够指导视频生成过程。
    </p>
    <h5>
     <a id="_147">
     </a>
     零样本视频编辑
    </h5>
    <p>
     PixelDance无需任何训练即可执行视频编辑，这通过将视频编辑任务转化为图像编辑任务来实现。如图10的最后一个示例所示，通过编辑给定视频的第一帧和最后一帧，PixelDance生成了与用户视频编辑期望一致的时间连贯视频。
    </p>
    <h3>
     <a id="_150">
     </a>
     结论
    </h3>
    <p>
     在本文中，作者提出了一种基于扩散模型的新颖视频生成方法PixelDance，它将第一帧和最后一帧的图像指令与文本指令相结合。作者还为该方法开发了专门的训练和推理技术。主要在WebVid - 10M上训练的PixelDance在合成具有复杂场景和动作的视频方面表现出卓越的熟练度，为视频生成设定了新的标准。
    </p>
    <p>
     尽管我们的方法取得了显著成果，但仍有进一步改进的空间：
    </p>
    <ul>
     <li>
      首先，模型可以从高质量的开放域视频数据训练中受益。
     </li>
     <li>
      其次，在特定领域对模型进行微调可以进一步增强其能力。
     </li>
     <li>
      第三，结合概述视频关键元素和动作的注释文本可以提高与用户指令的匹配度。
     </li>
