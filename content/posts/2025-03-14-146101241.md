    <h2>
     <a id="_1">
     </a>
     管理方式
    </h2>
    <p>
     利用ansible实现管理的主要方式：
    </p>
    <ul>
     <li>
      <strong>
       Ad-Hoc
      </strong>
      即利用ansible命令，主要用于临时命令使用场景.(一行一行代码执行）
     </li>
     <li>
      <strong>
       Ansible-playbook
      </strong>
      主要用于长期规划好的，大型项目的场景，需要有前期的规划过程
     </li>
    </ul>
    <h2>
     <a id="_6">
     </a>
     相关工具
    </h2>
    <ul>
     <li>
      <code>
       /usr/bin/ansible
      </code>
      主程序，临时命令执行工具
     </li>
     <li>
      <code>
       /usr/bin/ansible-doc
      </code>
      查看配置文档，模块功能查看工具
     </li>
     <li>
      <code>
       /usr/bin/ansible-galaxy
      </code>
      下载/上传优秀代码或Roles模块的官网平台
     </li>
     <li>
      <code>
       /usr/bin/ansible-playbook
      </code>
      定制自动化任务，编排剧本工具
     </li>
     <li>
      <code>
       /usr/bin/ansible-pull
      </code>
      远程执行命令的工具
     </li>
     <li>
      <code>
       /usr/bin/ansible-vault
      </code>
      文件加密工具
     </li>
     <li>
      <code>
       /usr/bin/ansible-console
      </code>
      基于Console界面与用户交互的执行工具
     </li>
    </ul>

    <h2>
     <a id="ansible_55">
     </a>
     ansible命令
    </h2>
    <h3>
     <a id="_56">
     </a>
     基本格式：
    </h3>
    <pre><code class="prism language-bash">ansible <span class="token operator">&lt;</span>host-pattern<span class="token operator">&gt;</span> <span class="token punctuation">[</span>-m module_name<span class="token punctuation">]</span> <span class="token punctuation">[</span>-a args<span class="token punctuation">]</span>
</code></pre>
    <ul>
     <li>
      ：所属组
     </li>
     <li>
      [-m module_name]：模块
     </li>
     <li>
      [-a args] ：参数
     </li>
    </ul>
    <h3>
     <a id="_65">
     </a>
     选项说明：
    </h3>
    <blockquote>
     <p>
      –version #显示版本
      <br/>
      -m module #指定模块，默认为command
      <br/>
      -v #详细过程 –vv -vvv更详细
      <br/>
      –list-hosts #显示主机列表，可简写 --list
      <br/>
      -k, --ask-pass #提示输入ssh连接密码，默认Key验证
      <br/>
      -C, --check #检查，并不执行
      <br/>
      -T, --timeout=TIMEOUT #执行命令的超时时间，默认10s
      <br/>
      -u, --user=REMOTE_USER #执行远程执行的用户
      <br/>
      -b, --become #代替旧版的sudo 切换
      <br/>
      –become-user=USERNAME #指定sudo的runas用户，默认为root
      <br/>
      -K, --ask-become-pass #提示输入sudo时的口令
     </p>
    </blockquote>
    <pre><code class="prism language-bash"> ansible all <span class="token parameter variable">-m</span> <span class="token function">ping</span>

    <h3>
     <a id="ansible_202">
     </a>
     ansible命令执行过程
    </h3>
    <p>
     1.加载自己的配置文件 默认
     <code>
      /etc/ansible/ansible.cfg
     </code>
    </p>
    <p>
     2.加载自己对应的模块文件，如：
     <code>
      command
     </code>
    </p>
    <p>
     3.通过ansible将模块或命令生成对应的临时py文件，并将该文件传输至远程服务器的对应执行用户
     <code>
      $HOME/.ansible/tmp/ansible-tmp-数字/XXX.PY文件
     </code>
    </p>
    <p>
     4.给文件+x执行
    </p>
    <p>
     5.执行并返回结果
    </p>
    <p>
     6.删除临时py文件，退出
    </p>

    <h3>
     <a id="ansible__219">
     </a>
     ansible 的执行状态
    </h3>
    <p>
     绿色：执行成功并且不需要做改变的操作
     <br/>
     黄色：执行成功并且对目标主机做变更
     <br/>
     红色：执行失败
    </p>
