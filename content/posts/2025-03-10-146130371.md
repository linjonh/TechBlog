    <h2>
     <a id="1_7">
     </a>
     1.创建数据库
    </h2>
    <p>
     语法：
    </p>
    <pre><code>CREATE DATABASE [IF NOT EXISTS] db_name [create_specification [,
create_specification] ...]
create_specification:
[DEFAULT] CHARACTER SET charset_name
[DEFAULT] COLLATE collation_name
</code></pre>
    <p>
     <code>
      CREATE
     </code>
     、
     <code>
      DATABASE
     </code>
     ： MySQL 内定的关键字，不区分大小写。
    </p>
    <p>
     <code>
      db_name
     </code>
     新创建的数据库的名字。
    </p>
    <p>
     <code>
      []
     </code>
     表示是可选项。
    </p>
    <p>
     <code>
      [IF NOT EXISTS]
     </code>
     可选项表示如果数据库不存在就创建，反之不执行任何操作。
    </p>
    <p>
     <code>
      [create_specification]
     </code>
     是指创建数据库时可使用的一些创建选项。
    </p>
    <p>
     例如：
     <br/>
     <code>
      [DEFAULT] CHARACTER SET charset_name
     </code>
     表示指定默认字符集。
    </p>
    <p>
     <code>
      [DEFAULT] COLLATE collation_name
     </code>
     表示指定默认校验集。
    </p>
    <p>
     <strong>
      案例
     </strong>
    </p>
    <ul>
     <li>
      创建名为
      <code>
       db1
      </code>
      的数据库
     </li>
    </ul>
    <pre><code>create database db1;
</code></pre>
    <ul>
     <li>
      增加
      <code>
       [if not exists]
      </code>
      选项再次创建名字为db1的数据库。
     </li>
    </ul>
    <pre><code>create database if not exists db1;
</code></pre>
    <p>
     SQL语句执行结果显示，语句执行成功，有1行被影响，有1个警告,说明操作有问题但不影响后续执行。
    </p>
    <p>
     ps:但如果不增加
     <code>
      [if not exists]
     </code>
     选项再次创建，就会报错。
    </p>
    <ul>
     <li>
      创建一个使用utf8字符集，并带校对规则的
      <code>
       db2
      </code>
      数据库。
     </li>
    </ul>
    <pre><code>create database db2 charset=utf8 collate utf8_general_ci;


</code></pre>
    <p>
     （关于字符集和校对规则的解释，请看后文介绍）
    </p>
    <p>
     <strong>
      创建数据库的本质：在/var/lib/MySQL中创建一个目录
     </strong>
    </p>
    <h2>
     <a id="2_89">
     </a>
     2.操纵数据库
    </h2>
    <h3>
     <a id="21_90">
     </a>
     2.1查看数据库
    </h3>
    <pre><code>show databases;      
</code></pre>
    <p>
     ps:注意是
     <code>
      databases
     </code>
    </p>
    <h3>
     <a id="22_95">
     </a>
     2.2显示创建语句
    </h3>
    <pre><code>show create database 数据库名；
</code></pre>
    <p>
     示例：
    </p>
    <pre><code>mysql &gt; show create database mytest;
+----------+----------------------------------------------------------------+
| Database | Create Database |
+----------+----------------------------------------------------------------+
| mysql | CREATE DATABASE `mytest` /*!40100 DEFAULT CHARACTER SET utf8 */ |
+----------+----------------------------------------------------------------+
</code></pre>
    <h3>
     <a id="23_109">
     </a>
     2.3修改数据库
    </h3>
    <p>
     语法：
    </p>
    <pre><code>ALTER DATABASE db_name 
[alter_spacification [,alter_spacification]...] 
alter_spacification:
[DEFAULT] CHARACTER SET charset_name
[DEFAULT] COLLATE collation_name
</code></pre>
    <p>
     <code>
      ALTER
     </code>
     意思指改变
    </p>
    <blockquote>
     <p>
      说明：
      <br/>
      对数据库的修改主要为修改数据库的字符集、校验规则
     </p>
    </blockquote>
    <p>
     示例： 将 mytest 数据库字符集改成 gbk
    </p>
    <pre><code>ALTER DATABASE mytest charset=gbk;
</code></pre>
    <h2>
     <a id="3_130">
     </a>
     3.删除数据库
    </h2>
    <pre><code>DROP DATABASE [IF EXISTS] db_ name;
</code></pre>
    <p>
     执行删除之后的结果:
    </p>
    <ul>
     <li>
      数据库内部看不到对应的数据库
     </li>
     <li>
      对应的数据库文件夹被删除，级联删除，里面的数据表全部被删
     </li>
    </ul>
    <p>
     示例：删除数据库
     <code>
      mytest
     </code>
    </p>
    <pre><code>DROP DATABASE mytest;     
</code></pre>
    <p>
     实例：如果数据库
     <code>
      mytest1
     </code>
     存在的话，就删除
    </p>
    <pre><code>DROP DATABASE IF EXSITS mytest1;
</code></pre>
    <p>
     <strong>
      删除数据库的本质：在/var/lib/MySQL中删除该数据库对应的目录
     </strong>
    </p>
    <p>
     ** 注意：不要随意删除数据库！**
    </p>
    <h2>
     <a id="4_150">
     </a>
     4.字符集和校验规则
    </h2>
    <h3>
     <a id="41Character_Set_151">
     </a>
     4.1字符集（Character Set）
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：字符集是一个系统支持的所有抽象字符的集合，它规定了哪些字符可以被存储和处理。
     </li>
    </ul>
    <p>
     比如常见的字符集有ASCII、Unicode、GBK等。
    </p>
    <ul>
     <li>
      <p>
       <code>
        ASCII
       </code>
       字符集主要包含了
       <code>
        英文字母
       </code>
       、
       <code>
        数字
       </code>
       、
       <code>
        标点符号
       </code>
       等基本字符，总共128个字符。
      </p>
     </li>
     <li>
      <p>
       <code>
        Unicode
       </code>
       字符集则是为了统一表示世界上所有语言的字符而设计的，它包含了几乎所有国家和民族使用的字符，涵盖范围极广。
      </p>
     </li>
     <li>
      <p>
       <code>
        GBK
       </code>
       是
       <code>
        汉字
       </code>
       内码扩展规范，它在ASCII的基础上，增加了大量的中文汉字和其他字符。
      </p>
     </li>
     <li>
      <p>
       <strong>
        作用
       </strong>
       ：字符集决定了数据库能够存储和处理哪些字符。
      </p>
      <p>
       如果数据库使用的字符集不支持某些特定字符，那么在存储或处理这些字符时就会出现问题，比如乱码或者直接无法存储。
      </p>
      <p>
       <strong>
        eg:
       </strong>
       若数据库采用ASCII字符集，就无法正确存储中文汉字。
      </p>
     </li>
    </ul>
    <h3>
     <a id="42_Collation_166">
     </a>
     4.2 校验规则（Collation）
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：校验规则也称为排序规则，它定义了字符如何进行比较、排序和校对的规则。
     </li>
     <li>
      校验规则是基于字符集的，它决定了字符在数据库中的比较和排序顺序。
     </li>
     <li>
      常见的校验规则有utf8_general_ci、utf8_unicode_ci等，其中 “ci” 表示不区分大小写（case insensitive），如果是 “cs” 则表示区分大小写（case sensitive）。
     </li>
     <li>
     </li>
     <li>
      <strong>
       作用
      </strong>
      <ul>
       <li>
        <strong>
         数据比较
        </strong>
        ：在进行查询、过滤、分组等操作时，校验规则决定了数据库如何比较字符数据。比如在执行
        <code>
         SELECT * FROM table WHERE column = 'value'
        </code>
        这样的查询时，数据库会根据校验规则来判断
        <code>
         column
        </code>
        中的值与
        <code>
         'value'
        </code>
        是否相等。
       </li>
       <li>
        <strong>
         数据排序
        </strong>
        ：当使用
        <code>
         ORDER BY
        </code>
        子句对数据进行排序时，校验规则决定了字符数据的排序顺序。例如，在一个包含人名的列上按照字母顺序排序，不同的校验规则可能会导致不同的排序结果。
       </li>
      </ul>
     </li>
    </ul>
    <h3>
     <a id="43_178">
     </a>
     4.3字符集与编码集的关系
    </h3>
    <p>
     字符集和编码集是相辅相成的关系。
    </p>
    <p>
     字符集定义了字符的集合，而编码集则规定了这些字符如何在计算机中表示和存储。
    </p>
    <p>
     可以把字符集想象成一个字典，里面列出了所有可用的字符，而编码集就是给这些字符编号的规则，通过这个规则，计算机才能知道如何存储和处理这些字符。
    </p>
