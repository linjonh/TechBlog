    <p style="margin-left:0; margin-right:0">
     在前端开发中，AJAX（Asynchronous JavaScript and XML）和 XHR（XMLHttpRequest）是两个密切相关但有些不同的概念。以下是对这两者的详细解释及其关系：
    </p>
    <p style="margin-left:0; margin-right:0">
     <strong>
      1. AJAX
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：AJAX 是一个用于创建异步 web 应用程序的技术。它允许网页在不完全重新加载整页的情况下，与服务器交换数据并更新部分网页内容。
     </li>
     <li>
      <strong>
       主要特征
      </strong>
      ：
      <ul>
       <li>
        <strong>
         异步操作
        </strong>
        ：AJAX 使得页面可以在后台与服务器进行数据交换，用户仍然可以与页面进行交互，而无需等待响应。
       </li>
       <li>
        <strong>
         支持多种格式
        </strong>
        ：虽然 AJAX 的名称中包含 XML，但它实际上可以处理多种数据格式，包括 JSON、HTML 以及纯文本。
       </li>
       <li>
        <strong>
         提高用户体验
        </strong>
        ：通过动态内容加载，AJAX 使得用户操作更流畅。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用示例
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="hljs">// 使用 fetch API 进行 AJAX 请求 

fetch('https://api.example.com/data') 

    .then(response =&gt; response.json()) 

    .then(data =&gt; { 

        console.log(data); 

        // 动态更新页面内容 

    }) 

    .catch(error =&gt; { 

        console.error('Error:', error); 

    }); </code></pre>
    <p style="margin-left:0; margin-right:0">
     <strong>
      2. XMLHttpRequest（XHR）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      ：XMLHttpRequest 是一种用于在 JavaScript 中与服务器进行交互的 API。在 AJAX 早期，XHR 是实现 AJAX 功能的主要方式。
     </li>
     <li>
      <strong>
       主要特征
      </strong>
      ：
      <ul>
       <li>
        <strong>
         用于发送和接收数据
        </strong>
        ：XHR 可以以异步方式发送 HTTP 请求并接收响应，允许 JS 程序与远程服务器进行数据交换。
       </li>
       <li>
        <strong>
         兼容性
        </strong>
        ：XHR 在大多数主流浏览器中都得到了良好的支持，是 AJAX 的基础。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用示例
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="hljs">const xhr = new XMLHttpRequest(); 

xhr.open('GET', 'https://api.example.com/data', true); 

xhr.onreadystatechange = function () { 

    if (xhr.readyState === 4 &amp;&amp; xhr.status === 200) { 

        const data = JSON.parse(xhr.responseText); 

        console.log(data); 

        // 动态更新页面内容 

    } 

}; 

xhr.send(); </code></pre>
    <p style="margin-left:0; margin-right:0">
     <strong>
      3. AJAX 和 XHR 的关系
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       AJAX 是一种技术，而 XHR 是实现这一技术的工具
      </strong>
      ：
      <ul>
       <li>
        AJAX 是一系列技术的组合，目的在于提高 Web 应用程序的用户体验和交互性能。
       </li>
       <li>
        XHR 是 AJAX 技术中的核心组成部分之一，负责实际的 HTTP 请求和响应处理。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       现代发展
      </strong>
      ：
      <ul>
       <li>
        随着技术的发展，现代浏览器引入了更简洁的 API，如
        <strong>
         Fetch API
        </strong>
        ，取代了原本需要使用 XHR 的很多功能。Fetch 具有更直观的使用方式和更强的功能，可以更方便地处理 JSON、流等。
       </li>
      </ul>
     </li>
     <li>
      虽然 XHR 在 AJAX 发展和使用中扮演了重要角色，但在今天的开发中，我们通常会使用更高级的工具（如 Fetch API）进行 AJAX 请求。尽管如此，了解 XHR 仍然非常重要，尤其是在需要处理与旧浏览器兼容性时。
     </li>
    </ul>

     希望这能帮助你更好地理解 AJAX 和 XHR 的关系！如果你有其他问题，随时可以问。
