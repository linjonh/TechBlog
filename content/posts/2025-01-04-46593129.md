    <p>
     最近在使用Linux操作系统执行一个可执行文件，结果出现了No such file or directory的提示，表示很疑惑。
    </p>
    <pre class="prettyprint"><code class="hljs livecodeserver">./tshrf

bash: ./tshref: No such <span class="hljs-built_in">file</span> <span class="hljs-operator">or</span> <span class="hljs-built_in">directory</span></code></pre>
    <p>
     查看文件信息，可以看到文件是存在的，并且是可以执行的。
    </p>
    <pre class="prettyprint"><code class="hljs lasso"><span class="hljs-attribute">-rwxr</span><span class="hljs-attribute">-xr</span><span class="hljs-attribute">-x</span> <span class="hljs-number">1</span> yuan yuan <span class="hljs-number">20581</span>  <span class="hljs-number">4</span>月 <span class="hljs-number">29</span>  <span class="hljs-number">2004</span> tshref</code></pre>
    <p>
     查阅资料后，原因是系统位数与该可执行文件需要的lib库位数不匹配。
    </p>
    <p>
     用uname命令打印系统信息，发现系统是64位系统
    </p>
    <pre class="prettyprint"><code class="hljs lasso">uname <span class="hljs-attribute">-a</span>

Linux yuan<span class="hljs-attribute">-vm</span> <span class="hljs-number">3.13</span><span class="hljs-number">.0</span><span class="hljs-subst">-</span><span class="hljs-number">32</span><span class="hljs-attribute">-generic</span> <span class="hljs-variable">#57</span><span class="hljs-attribute">-Ubuntu</span> SMP Tue Jul <span class="hljs-number">15</span> <span class="hljs-number">03</span>:<span class="hljs-number">51</span>:<span class="hljs-number">08</span> UTC <span class="hljs-number">2014</span> x86_64 x86_64 x86_64 GNU/Linux</code></pre>
    <p>
     用file命令查看文件信息，发现是一个32位可执行文件。
    </p>
    <pre class="prettyprint"><code class="hljs vhdl"><span class="hljs-keyword">file</span> ./tshref

./tshref: ELF <span class="hljs-number">32</span>-<span class="hljs-typename">bit</span> LSB  executable, Intel <span class="hljs-number">80386</span>, version <span class="hljs-number">1</span> (SYSV), 
dynamically linked (uses <span class="hljs-keyword">shared</span> libs), <span class="hljs-keyword">for</span> GNU/Linux <span class="hljs-number">2.2</span><span class="hljs-number">.5</span>, <span class="hljs-keyword">not</span> stripped</code></pre>
    <p>
     要想在64位系统上与运行32位程序，则需要安装32位lib库。
     <br/>
     对于Ubuntu用户可以使用下面的命令安装。
    </p>
    <pre class="prettyprint"><code class="hljs applescript">sudo apt-<span class="hljs-keyword">get</span> install ia32-libs

Reading package lists... Done
Building dependency tree       
Reading state information... Done
Package ia32-libs <span class="hljs-keyword">is</span> <span class="hljs-keyword">not</span> available, <span class="hljs-keyword">but</span> <span class="hljs-keyword">is</span> referred <span class="hljs-keyword">to</span> <span class="hljs-keyword">by</span> another package.
This may mean <span class="hljs-keyword">that</span> <span class="hljs-keyword">the</span> package <span class="hljs-keyword">is</span> missing, has been obsoleted, <span class="hljs-keyword">or</span>
<span class="hljs-keyword">is</span> only available <span class="hljs-keyword">from</span> another source
However <span class="hljs-keyword">the</span> following packages replace <span class="hljs-keyword">it</span>:
  lib32z1 lib32ncurses5 lib32bz2-<span class="hljs-number">1.0</span></code></pre>
    <p>
     过程中有可能找不到需要的库，但是会有几个替代包，选择安装其中一个。
    </p>
    <pre class="prettyprint"><code class="hljs bash"><span class="hljs-built_in">sudo</span> apt-get install lib32bz2-<span class="hljs-number">1.0</span>

lib32bz2-<span class="hljs-number">1.0</span></code></pre>
    <p>
     然后就可以正常运行之前的可执行文件了。
    </p>
    <p>
     遇到这种问题其实还有可能是其他原因，例如文本的编码格式问题等，本文仅提出了一种解决方法，读者遇到相同问题要具体问题具体分析。
    </p>
