    <h1 class="title-article" id="articleContentId">
     golang中具有 “no copy“的类型
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     在 Go 语言中，某些类型由于特殊用途或底层实现，可能会被标记为
     <strong>
      “no copy”
     </strong>
     ，即它们不能被复制，通常是因为复制会导致意外的行为或错误。这些类型主要包括：
    </p>
    <hr/>
    <h4>
     <a id="1_syncMutexsyncRWMutex_4">
     </a>
     1.
     <strong>
      <code>
       sync.Mutex
      </code>
      、
      <code>
       sync.RWMutex
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：Mutex 是用于同步的锁，复制后可能会导致多个实例操作同一个底层锁的情况，从而引发竞争条件或死锁。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">var</span> mu sync<span class="token punctuation">.</span>Mutex
copyMu <span class="token operator">:=</span> mu <span class="token comment">// ❌ 错误，拷贝 Mutex 会导致不可预测行为</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_syncCond_14">
     </a>
     2.
     <strong>
      <code>
       sync.Cond
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：
      <code>
       sync.Cond
      </code>
      依赖于
      <code>
       sync.Mutex
      </code>
      ，如果复制，会导致多个
      <code>
       Cond
      </code>
      变量竞争同一个锁。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">var</span> cond <span class="token operator">=</span> sync<span class="token punctuation">.</span><span class="token function">NewCond</span><span class="token punctuation">(</span><span class="token operator">&amp;</span>sync<span class="token punctuation">.</span>Mutex<span class="token punctuation">{<!-- --></span><span class="token punctuation">}</span><span class="token punctuation">)</span>
copyCond <span class="token operator">:=</span> <span class="token operator">*</span>cond <span class="token comment">// ❌ 不应该复制</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_syncOnce_24">
     </a>
     3.
     <strong>
      <code>
       sync.Once
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：
      <code>
       sync.Once
      </code>
      用于确保某段代码只执行一次，复制
      <code>
       Once
      </code>
      可能导致相同的初始化逻辑执行多次。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">var</span> once sync<span class="token punctuation">.</span>Once
copyOnce <span class="token operator">:=</span> once <span class="token comment">// ❌ 拷贝可能会导致 `Do` 不能正确保证只执行一次</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="4_syncWaitGroup_34">
     </a>
     4.
     <strong>
      <code>
       sync.WaitGroup
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：
      <code>
       sync.WaitGroup
      </code>
      维护一个内部计数器，拷贝后多个
      <code>
       WaitGroup
      </code>
      可能操作同一个计数器，导致错误。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">var</span> wg sync<span class="token punctuation">.</span>WaitGroup
copyWg <span class="token operator">:=</span> wg <span class="token comment">// ❌ 复制 WaitGroup 可能导致等待逻辑混乱</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="5_syncPool_44">
     </a>
     5.
     <strong>
      <code>
       sync.Pool
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：
      <code>
       sync.Pool
      </code>
      是一个对象池，拷贝会导致多个
      <code>
       Pool
      </code>
      可能共享相同的对象存储区，破坏内存管理。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">var</span> pool sync<span class="token punctuation">.</span>Pool
copyPool <span class="token operator">:=</span> pool <span class="token comment">// ❌ 复制可能导致多个 Pool 共享底层存储，导致数据混乱</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="6_contextContext_54">
     </a>
     6.
     <strong>
      <code>
       context.Context
      </code>
      （通常不应该被复制）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：
      <code>
       context.Context
      </code>
      用于控制超时、取消等，复制
      <code>
       Context
      </code>
      可能会导致取消信号不生效。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go">ctx <span class="token operator">:=</span> context<span class="token punctuation">.</span><span class="token function">Background</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
copyCtx <span class="token operator">:=</span> ctx <span class="token comment">// ❌ 复制 context 可能导致错误的取消行为</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="7__runtimeinternalatomicNoCopy__64">
     </a>
     7.
     <strong>
      带
      <code>
       runtime/internal/atomic.NoCopy
      </code>
      结构体的类型
     </strong>
    </h4>
    <ul>
     <li>
      Go 内部提供了
      <code>
       runtime/internal/atomic.NoCopy
      </code>
      结构体，嵌入该类型的结构体会在
      <code>
       go vet
      </code>
      检查时报错，防止误复制。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-go"><span class="token keyword">type</span> myStruct <span class="token keyword">struct</span> <span class="token punctuation">{<!-- --></span>
    noCopy runtime<span class="token operator">/</span>internal<span class="token operator">/</span>atomic<span class="token punctuation">.</span>NoCopy
<span class="token punctuation">}</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_75">
     </a>
     <strong>
      如何防止结构体被拷贝？
     </strong>
    </h4>
    <p>
     如果你定义的结构体不希望被复制，可以嵌入
     <code>
      sync.NoCopy
     </code>
     （Go 1.20 之后正式提供）：
    </p>
    <pre><code class="prism language-go"><span class="token keyword">import</span> <span class="token string">"sync"</span>

<span class="token keyword">type</span> MyStruct <span class="token keyword">struct</span> <span class="token punctuation">{<!-- --></span>
    noCopy sync<span class="token punctuation">.</span>NoCopy
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这样，
     <code>
      go vet
     </code>
     工具会在发现
     <code>
      MyStruct
     </code>
     被复制时发出警告。
    </p>
    <hr/>
    <h4>
     <a id="_88">
     </a>
     <strong>
      总结
     </strong>
    </h4>
    <p>
     在 Go 语言中，以下类型通常不能被复制：
    </p>
    <ul>
     <li>
      <strong>
       同步相关
      </strong>
      ：
      <ul>
       <li>
        <code>
         sync.Mutex
        </code>
       </li>
       <li>
        <code>
         sync.RWMutex
        </code>
       </li>
       <li>
        <code>
         sync.Cond
        </code>
       </li>
       <li>
        <code>
         sync.Once
        </code>
       </li>
       <li>
        <code>
         sync.WaitGroup
        </code>
       </li>
       <li>
        <code>
         sync.Pool
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       上下文管理
      </strong>
      ：
      <ul>
       <li>
        <code>
         context.Context
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用
       <code>
        sync.NoCopy
       </code>
       或
       <code>
        runtime/internal/atomic.NoCopy
       </code>
       防止拷贝的结构体
      </strong>
     </li>
    </ul>
    <p>
     一般来说，如果一个类型涉及
     <strong>
      锁、并发同步、状态管理
     </strong>
     ，就要特别注意
     <strong>
      避免拷贝
     </strong>
     ，否则可能会导致竞态条件、锁丢失或不一致行为。
    </p>
