     能简述一下动态 SQL 的执行原理吗
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     MyBatis 的动态 SQL 是一种强大的功能，允许开发者根据条件动态生成 SQL 语句。它的执行原理主要涉及以下几个步骤：
    </p>
    <p>
     ### **1. 解析映射文件**
     <br/>
     当 MyBatis 启动时，会加载并解析映射文件（`Mapper.xml`），提取其中的 SQL 语句和动态标签（如 `&lt;if&gt;`、`&lt;foreach&gt;` 等）。
    </p>
    <p>
     - MyBatis 使用 XML 解析器（如 DOM 解析器）读取映射文件，并将 SQL 语句和动态标签解析为 **抽象语法树（AST）**。
     <br/>
     - 每个动态标签（如 `&lt;if&gt;`、`&lt;choose&gt;` 等）会被解析为一个节点，并存储在 AST 中。
    </p>
    <p>
     ### **2. 构建动态 SQL**
     <br/>
     当执行一个动态 SQL 查询时，MyBatis 会根据传入的参数（如方法的输入参数）动态构建最终的 SQL 语句。
    </p>
    <p>
     #### **步骤：**
     <br/>
     1. **参数绑定**：MyBatis 将方法的输入参数传递给动态 SQL 处理器。
     <br/>
     2. **遍历 AST**：动态 SQL 处理器遍历抽象语法树，根据每个节点的类型（如 `&lt;if&gt;`、`&lt;foreach&gt;`）和条件判断，动态生成 SQL 片段。
     <br/>
     3. **拼接 SQL**：将生成的 SQL 片段拼接成完整的 SQL 语句。
    </p>
    <p>
     ### **3. 示例解析**
     <br/>
     假设有一个动态 SQL 查询：
     <br/>
     ```xml
     <br/>
     &lt;select id="findUsers" parameterType="map" resultType="User"&gt;
     <br/>
     SELECT * FROM users
     <br/>
     WHERE 1=1
     <br/>
     &lt;if test="name != null"&gt;
     <br/>
     AND name = #{name}
     <br/>
     &lt;/if&gt;
     <br/>
     &lt;if test="age != null"&gt;
     <br/>
     AND age = #{age}
     <br/>
     &lt;/if&gt;
     <br/>
     &lt;/select&gt;
     <br/>
     ```
    </p>
    <p>
     #### **执行过程：**
     <br/>
     1. **解析映射文件**：
     <br/>
     - MyBatis 将 `&lt;if&gt;` 标签解析为节点，并存储在 AST 中。
     <br/>
     - SQL 模板被解析为一个基础结构。
    </p>
    <p>
     2. **参数绑定**：
     <br/>
     - 假设传入的参数是 `{name: "Alice", age: null}`。
    </p>
    <p>
     3. **遍历 AST**：
     <br/>
     - 遍历第一个 `&lt;if&gt;` 标签，判断 `name != null` 是否成立。因为 `name` 不为 `null`，所以生成 SQL 片段 `AND name = #{name}`。
     <br/>
     - 遍历第二个 `&lt;if&gt;` 标签，判断 `age != null` 是否成立。因为 `age` 为 `null`，所以跳过该片段。
    </p>
    <p>
     4. **拼接 SQL**：
     <br/>
     - 最终生成的 SQL 语句为：
     <br/>
     ```sql
     <br/>
     SELECT * FROM users WHERE 1=1 AND name = #{name}
     <br/>
     ```
    </p>
    <p>
     5. **预编译和执行**：
     <br/>
     - MyBatis 使用 `PreparedStatement` 预编译生成的 SQL 语句，并绑定参数值（如 `name = "Alice"`）。
     <br/>
     - 执行 SQL 语句并返回结果。
    </p>
    <p>
     ### **4. 动态 SQL 的执行原理总结**
     <br/>
     - **解析阶段**：MyBatis 在启动时解析映射文件，将动态 SQL 语句转换为抽象语法树（AST）。
     <br/>
     - **运行时构建**：在执行 SQL 时，根据传入的参数动态遍历 AST，生成 SQL 片段并拼接成完整的 SQL 语句。
     <br/>
     - **预编译和执行**：生成的 SQL 语句通过 `PreparedStatement` 预编译并执行，确保性能和安全性。
    </p>
    <p>
     ### **5. 动态 SQL 的优势**
     <br/>
     - **灵活性**：可以根据条件动态生成 SQL，满足复杂的业务需求。
     <br/>
     - **性能优化**：避免了不必要的 SQL 片段拼接，减少了数据库的负担。
     <br/>
     - **安全性**：通过预编译和参数绑定，避免了 SQL 注入风险。
    </p>
    <p>
     通过动态 SQL，MyBatis 提供了一种强大且灵活的方式来处理复杂的 SQL 查询，同时保持代码的简洁性和可维护性。
    </p>
