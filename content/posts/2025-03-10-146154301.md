# 多线程--参数传递之间的关系

在C++中创建线程时，传递参数的方式会影响参数的生命周期、线程的安全性和性能。以下是几种常见的传递方式及其适用情况：

#### 1. 值传递

值传递会创建参数的副本，并在线程函数内部使用该副本。这种方式可以避免线程之间的竞态条件，因为每个线程都有自己独立的参数副本。

**适用情况**
：

* **参数是简单的基本类型（如整数、浮点数等）或小型结构体。**
* **你希望每个线程都有参数的独立副本，不会影响其他线程。**
* **参数的拷贝开销不大。**

```
#include <iostream>
#include <thread>

void threadFunction(int value) {
    std::cout << "Value: " << value << std::endl;
}

int main() {
    int value = 42;
    std::thread t(threadFunction, value);
    t.join();
    return 0;
}

```

#### 2. 引用传递

引用传递允许线程直接访问原始参数，可以避免拷贝大对象的开销。但这也意味着如果主线程修改了参数，子线程可能会遇到未定义行为。

**适用情况**
：

* **参数是大型对象，拷贝开销很大。**
* **确保参数在整个线程执行期间保持有效（主线程不会销毁或修改该参数）。**
* **需要在多个线程间共享数据，并通过同步机制（如互斥锁）来保护数据**
  。

```
#include <iostream>
#include <thread>

void threadFunction(int& value) {
    std::cout << "Value: " << value << std::endl;
}

int main() {
    int value = 42;
    std::thread t(threadFunction, std::ref(value));
    t.join();
    return 0;
}

```

#### 3. 指针传递

指针传递和引用传递类似，也允许线程直接访问原始参数。但需要注意指针的生命周期管理，避免悬空指针。

**适用情况**
：

* 参数是动态分配的对象，传递指针避免拷贝。
* **需要传递数组等数据结构，使用指针进行操作**
  。
* 确保指针在整个线程执行期间保持有效，避免悬空指针。

```
#include <iostream>
#include <thread>

void threadFunction(int* value) {
    std::cout << "Value: " << *value << std::endl;
}

int main() {
    int value = 42;
    std::thread t(threadFunction, &value);
    t.join();
    return 0;
}

```

#### 小结

* **值传递**
  ：适用于简单类型和小型结构体，避免竞态条件。
* **引用传递**
  ：适用于大型对象，共享数据时需要同步机制。
* **指针传递**
  ：适用于动态分配的对象和数组，需注意指针的生命周期管理。

选择合适的传递方式取决于具体的使用场景和性能需求。
