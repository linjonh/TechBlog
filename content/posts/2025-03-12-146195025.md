    <p>
     在 Qt 6.8.2（C++）中，两个线程之间的同步可以通过以下机制实现。以下是常用方法及代码示例：
    </p>
    <h4>
     1. ‌
     <strong>
      QMutex（互斥锁）
     </strong>
     ‌
    </h4>
    <p>
     用于保护共享资源，确保同一时间只有一个线程访问资源。
    </p>
    <pre></pre>
    <pre><code class="language-cpp">#include &lt;QMutex&gt;
#include &lt;QThread&gt;

QMutex mutex;
int sharedData = 0;

class WorkerThread : public QThread {
protected:
    void run() override {
        mutex.lock();
        sharedData++;
        mutex.unlock();
    }
};

// 使用 QMutexLocker（更安全，自动管理锁）
void safeIncrement() {
    QMutexLocker locker(&amp;mutex);
    sharedData++;
}
</code></pre>
    <hr/>
    <h4>
     2. ‌
     <strong>
      QReadWriteLock（读写锁）
     </strong>
     ‌
    </h4>
    <p>
     允许多个线程同时读取，但写入时独占。
    </p>
    <pre><code class="language-cpp">#include &lt;QReadWriteLock&gt;

QReadWriteLock rwLock;
QString sharedString;

class ReaderThread : public QThread {
    void run() override {
        rwLock.lockForRead();
        qDebug() &lt;&lt; "Read:" &lt;&lt; sharedString;
        rwLock.unlock();
    }
};

class WriterThread : public QThread {
    void run() override {
        rwLock.lockForWrite();
        sharedString = "New Data";
        rwLock.unlock();
    }
};
</code></pre>
    <hr/>
    <h4>
     3. ‌
     <strong>
      QSemaphore（信号量）
     </strong>
     ‌
    </h4>
    <p>
     控制对多个相同资源的访问。
    </p>
    <pre><code class="language-cpp">#include &lt;QReadWriteLock&gt;

QReadWriteLock rwLock;
QString sharedString;

class ReaderThread : public QThread {
    void run() override {
        rwLock.lockForRead();
        qDebug() &lt;&lt; "Read:" &lt;&lt; sharedString;
        rwLock.unlock();
    }
};

class WriterThread : public QThread {
    void run() override {
        rwLock.lockForWrite();
        sharedString = "New Data";
        rwLock.unlock();
    }
};
</code></pre>
    <hr/>
    <h4>
     4. ‌
     <strong>
      QWaitCondition（条件变量）
     </strong>
     ‌
    </h4>
    <p>
     让线程等待某个条件满足后再执行。
    </p>
    <pre><code class="language-cpp">#include &lt;QWaitCondition&gt;
#include &lt;QMutex&gt;

QMutex mutex;
QWaitCondition condition;
bool dataReady = false;

// 生产者线程
void Producer::run() {
    mutex.lock();
    dataReady = true;
    condition.wakeAll(); // 唤醒所有等待线程
    mutex.unlock();
}

// 消费者线程
void Consumer::run() {
    mutex.lock();
    while (!dataReady) {
        condition.wait(&amp;mutex); // 释放mutex并等待
    }
    // 处理数据
    mutex.unlock();
}
</code></pre>
    <hr/>
    <h4>
     5. ‌
     <strong>
      信号槽跨线程通信
     </strong>
     ‌
    </h4>
    <p>
     Qt 的信号槽机制默认是线程安全的，可通过
     <code>
      QueuedConnection
     </code>
     实现跨线程通信。
    </p>
    <pre><code class="language-cpp">// Worker对象在子线程中执行任务
class Worker : public QObject {
    Q_OBJECT
public slots:
    void doWork() {
        // 耗时操作
        emit resultReady(result);
    }
signals:
    void resultReady(int);
};

// 主线程中
QThread thread;
Worker worker;
worker.moveToThread(&amp;thread);
connect(&amp;worker, &amp;Worker::resultReady, this, &amp;MainWindow::handleResult);
thread.start();

// 触发任务
QMetaObject::invokeMethod(&amp;worker, "doWork", Qt::QueuedConnection);
</code></pre>
    <hr/>
    <h4>
     关键注意事项：
    </h4>
    <ul>
     <li>
      ‌
      <strong>
       线程亲和性
      </strong>
      ‌：使用
      <code>
       moveToThread()
      </code>
      将对象移动到目标线程，避免直接跨线程访问成员。
     </li>
     <li>
      ‌
      <strong>
       避免死锁
      </strong>
      ‌：确保锁的获取和释放成对出现，优先使用
      <code>
       QMutexLocker
      </code>
      。
     </li>
     <li>
      ‌
      <strong>
       跨平台性
      </strong>
      ‌：Qt 的同步机制封装了系统原生 API（如 Windows 的临界区、事件），保证跨平台行为一致。
     </li>
    </ul>
    <p>
     根据需求选择合适机制：互斥锁适合简单共享资源，信号量控制资源池，条件变量适合复杂等待逻辑，信号槽适合解耦通信。
    </p>
