    <h2>
     <a id="1_2">
     </a>
     1.接口测试的介绍
    </h2>
    <h3>
     <a id="11_3">
     </a>
     1.1接口测试的定义
    </h3>
    <p>
     接口测试是测试系统组件间接口的一种测试。
    </p>
    <p>
     接口测试主要用于检测外部系统与系统之间以及内部各个子系统之间的交互点。测试的重点是要检查数据的交换，传递和控制管理过程，以及系统间的相互逻辑依赖关系等。
    </p>
    <h3>
     <a id="12_8">
     </a>
     1.2接口的类型
    </h3>
    <p>
     ①程序内部的接口
    </p>
    <p>
     ②系统对外的接口
    </p>
    <h3>
     <a id="13_13">
     </a>
     1.3接口的分类
    </h3>
    <p>
     ①webService接口，走soap协议通过http传输，请求报文和返回报文都是xml格式的，在测试的时候用通过工具才能进行调用、测试。
    </p>
    <p>
     ②http api接口，走http协议，通过路径来区分调用的方法，请求报文都是key-value形式的，返回报文一般都是json串，有get和post等方法，这也是最常用的两种请求方式。
    </p>
    <h3>
     <a id="14_18">
     </a>
     1.4请求的结构
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce7375caba19a4724860c08e8a5fd475.png">
      <br/>
      请求协议：
      <br/>
      http — 普通的http请求
      <br/>
      https — 加密的http请求，传输数据更加安全
      <br/>
      ftp — 文件传输协议，主要用来传输文件
     </img>
    </p>
    <p>
     请求IP：指提供接口的系统所部署的服务器地址。
    </p>
    <p>
     请求端口：如果不填端口，默认是80，否则需要填写端口号。
    </p>
    <p>
     接口地址：指系统提供的接口的地址。
    </p>
    <p>
     接口参数：参数在接口路径后，用“?”来表示路径地址完了，剩下的都是参数，用“&amp;”来区分参数个数。
    </p>
    <h2>
     <a id="2Postman_33">
     </a>
     2.Postman的介绍和常用功能
    </h2>
    <h3>
     <a id="21postman_34">
     </a>
     2.1postman是什么
    </h3>
    <p>
     Postman是一款功能强大的网页调试、HTTP请求发送及接口测试用例运行的工具，能够模拟各种HTTP Request，实现特定的接口测试，能够高效的帮助后端开发人员独立进行接口测试。Postman提供了云服务，支持数据同步及团队协作等，提供了丰富的HTML格式的报告模板。postman不仅可进行接口测试，还是一个API管理工具。
    </p>
    <p>
     Postman有两种形式的产品：
     <br/>
     1、本地应用程序
     <br/>
     2、Chrome浏览器的插件。
    </p>
    <p>
     <img alt="postman" src="https://i-blog.csdnimg.cn/blog_migrate/6a2297050af5fd9498fb2d025ae801e3.png">
      <br/>
      下载地址：
      <a href="https://www.postman.com/downloads/" rel="nofollow">
       https://www.postman.com/downloads/
      </a>
     </img>
    </p>
    <h3>
     <a id="22postman_44">
     </a>
     2.2postman的常用功能
    </h3>
    <p>
     1、抓取和分析现有网站请求
     <br/>
     2、创建Http请求
     <br/>
     3、管理Http请求(Collections)
     <br/>
     4、变量
     <br/>
     5、导出和导入
     <br/>
     6、生成在线API文档
     <br/>
     7、自动化测试
     <br/>
     8、Mock server
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/29c2983c7eb16e0f46c93cba2b9f4c41.png"/>
    </p>

    <h3>
     <a id="33Mock_64">
     </a>
     3.3Mock功能的使用
    </h3>
    <p>
     Mock常见的使用场景：
    </p>
    <p>
     1、无法控制第三方系统某接口的返回，返回的数据不满足要求。
     <br/>
     2、某依赖系统还未开发完成，就需要对被测系统进行测试。
     <br/>
     3、某些系统不支持重复请求，如支付功能。
     <br/>
     4、系统功能有访问频次限制，限制获取敏感信息的接口访问频次。
    </p>
    <p>
     <strong>
      方法一：新建API
     </strong>
    </p>
    <p>
     1、首先点击Add-Mock Server，创建Mock Server
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a1827ef9e302430400714dbe387d5537.png">
      <br/>
      2、填写API，这里请求方法是GET，请求路径是test，响应码是200，响应体是ok。
     </img>
    </p>
    <p>
     （Ps：{
     <!-- -->
     {url}}/不可去掉）
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/834967a9611746d6ff34e7d19eac1836.png"/>
     <br/>
     然后mock server命名，点击创建，得到一个Postman分配的网址，这个网址是以mock.pstmn.io域名结尾的，就是刚才创建的API对应的mock地址。
    </p>
    <p>
     在创建mock的界面有一项是Make this mock server private，不勾选为将mock地址公布出去，任何人都可以访问。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e40042760e15300344cf11296f7a6404.png"/>
     <br/>
     下面这个就是路径
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5c9c3b2e8b0506eeb5770fecefffbe21.png"/>
     <br/>
     在Postman的左侧能看到刚刚创建的集合，点击小三角，点击Mocks，可以看到刚刚创建的mock server。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2d22e91e05bb3e04a795bd474a4246c5.png"/>
    </p>
    <p>
     3、打开创建的请求test，运行环境选择创建的testServer，点击发送，返回的结果就是刚刚填入的值ok。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5c866ac4d324e932edca53fe889ca8fe.png"/>
    </p>
    <p>
     4、点击右侧Examples可以看到test
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/14a9b03190ce59295a0d901005c53df9.png"/>
     <br/>
     点击test打开编辑，编辑完成后点击保存
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4f82a118fa16af9db5de68d5fdb0837e.png"/>
     <br/>
     PS：
     <br/>
     每个request都可以有多个example
     <br/>
     每个example创建之后都可以修改
     <br/>
     Mock server只会采用最后创建的example
    </p>
    <p>
     5、再次发送请求{
     <!-- -->
     {url}}/test?name=aaa，返回刚刚编辑的Examples。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/685eb9eed2b71d4b8bb784123b4d87f6.png"/>
     <br/>
     <strong>
      方法二：使用工作区的集合
     </strong>
    </p>
    <p>
     1、点击左侧集合的小三角，点击Mocks，点击Create a mock server
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8cee80705a9d34cd118d77fc09ce6497.png"/>
    </p>
    <p>
     填写mock server的名称，选择环境，点击创建即可
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/23dac0b6c88e9290c4169acf0fa9e015.png"/>
     <br/>
     上一步勾选了Make this mock server private，可点击Generate Postman API Key设置API key。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/21c3bcc0658c1ac202c828652ce8cc67.png"/>
     <br/>
     2、如果勾选Make this mock server private，设为私有，打开https://go.pstmn.io/pro-api-keys，点击Generate API key
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/748959c1a5e56671651159a15c8acb9b.png"/>
     <br/>
     填写key的名字，点击创建，生成API key
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d993f076fc3575ccae83508e2610fdcf.png"/>
     <br/>
     这里可以复制
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/95e748ea41d3461d438ade74e359e772.png"/>
     <br/>
     3、创建完成后，可在mocks下看到创建好的mock server，以将其保存在环境的变量中，也可以直接使用。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/133f78f0171d87bb54fa5d7303ffdf6b.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/91d28fe1b322073ec485151d4dfc9cc6.png"/>
     <br/>
     4、为request创建一个example，选择requeset添加example，然后保存。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/40fc92798e693bf10dbd81f0871d9c34.png"/>
     <br/>
     5、Mock server设为私有后请求时要附上对应的API keys，否则请求找不到Mock服务器。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b0c5a159c4675c5b0ca1cc406a11e88d.png"/>
    </p>
    <p>
     在请求头附上key：x-api-key，value：api key后请求则可。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9dd4a210d5df493a8d0f03347a7d4d63.png"/>
    </p>
