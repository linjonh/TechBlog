    <p>
     <strong>
      看这一篇吧 https://blog.csdn.net/u014686399/article/details/105097217， 是最新的。这篇太老了
     </strong>
    </p>
    <p>
     这两天废了九牛二虎之力完成了python3.5接入cas server， 废话少说，步骤如下:
    </p>
    <h4>
     1.基础环境
    </h4>
    <div>
     <table border="1" cellpadding="1" cellspacing="1" style="width:200px;">
      <tbody>
       <tr>
        <td>
         python
        </td>
        <td>
         3.5.4
        </td>
       </tr>
       <tr>
        <td>
         django
        </td>
        <td>
         2.0
        </td>
       </tr>
       <tr>
        <td>
         对接的cas server
        </td>
        <td>
         3.5.2
        </td>
       </tr>
      </tbody>
     </table>
     <span style="color:#ff0000;">
      注:cas对外提供的是https
     </span>
    </div>
    <div>
     python的环境算是比较新的了，所以坑比较多。cas我偷懒了用了个老版本
    </div>
    <div>
    </div>
    <h4>
     2、django对接cas环境的安装
    </h4>
    <div>
     (1)安装python-cas,这个建议下载1.2.0版本以上的
    </div>
    <div>
     1.2.0版本的链接:https://pypi.python.org/pypi/python-cas/1.2.0
    </div>
    <div>
     然后 解压---&gt; python setup.py install
    </div>
    <div>
    </div>
    <div>
     (2)安装django_cas_ng 这个建议3.5.8以上，这时候坑就来了
    </div>
    <div>
     3.5.8版本的链接: https://pypi.python.org/pypi/django-cas-ng
    </div>
    <div>
     然后 解压 ---&gt; python setup.py install
    </div>
    <div>
     <span style="color:#ff0000;">
      <strong>
      </strong>
     </span>
    </div>
    <div>
     <span style="color:#ff0000;">
      <strong>
       -------------------------------------------------------------------------------------------------
      </strong>
     </span>
    </div>
    <div>
     这时候你要看django的日志了，因为很可能会报错的
    </div>
    <div>
     1、如果出现ImportError: cannot import name 'CASClient'
    </div>
    <div>
     说明你的python-cas安装不成功，或者是版本太低
    </div>
    <div>
     2、TypeError: __init__() missing 1 required positional argument: 'on_delete'
    </div>
    <div>
     找到你的python安装包的路径下的site-packages/django_cas_ng-3.5.0-py3.5.egg/django_cas_ng/models.py
    </div>
    <div>
     修改为：
    </div>
    <div>
     <pre class="has"><code class="language-python">unique_together = ('session_key', 'user')
    session_key = models.CharField(max_length=255, blank=True, null=True)
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        related_name="+",
        null=True,
        blank=True,on_delete=models.PROTECT
    )</code></pre>
     3、django_cas_ng.ProxyGrantingTicket.date: (fields.E160) The options auto_now, auto_now_add, and default are       mutually exclusive. Only one of these options may be present.
    </div>
    <div>
     找到你的python安装包的路径下的site-packages/django_cas_ng-3.5.0-py3.5.egg/django_cas_ng/models.py
    </div>
    <div>
     修改：
    </div>
    <div>
     找到31行，删除前面的add_now部分，最后：
    </div>
    <div>
     <pre class="has"><code class="language-python">pgtiou = models.CharField(max_length=255, null=True, blank=True)
pgt = models.CharField(max_length=255, null=True, blank=True)</code></pre>
     <pre class="has"><code class="language-python">date = models.DateTimeField(auto_now_add=True)</code></pre>
    </div>
    <div>
     4、AttributeError: module 'django_cas_ng' has no attribute 'views'
    </div>
    <div>
     找到你报错的url.py,
    </div>
    <div>
     修改：
    </div>
    <div>
     import django_cas_ng.views
    </div>
    <div>
     <span style="color:#ff0000;">
      <strong>
       -------------------------------------------------------------------------------------------------
      </strong>
     </span>
    </div>
    <div>
    </div>
    <div>
     （3）配置的setting.py，其中AUTHENTICATION_BACKENDS可能没有，需要你自行添加
    </div>
    <div>
    </div>
    <div>
     <pre class="has"><code class="language-python">INSTALLED_APPS = (
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'django_cas_ng',
    ...
)

MIDDLEWARE_CLASSES = (
    'django.middleware.common.CommonMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    ...
)

AUTHENTICATION_BACKENDS = (
    'django.contrib.auth.backends.ModelBackend',
    'django_cas_ng.backends.CASBackend',
)</code></pre>
     <pre class="has"><code class="language-python">添加 CAS_SERVER_URL="cas server的地址"</code></pre>
     <pre class="has"><code class="language-python">添加 CAS_REDIRECT_URL="你网站的主页"
</code></pre>
    </div>
    <div>
     （4）配置url.py，增加
    </div>
    <div>
     <pre class="has"><code class="language-python">    url(r'^accounts/login$', django_cas_ng.views.login, name='cas_ng_login'),
    url(r'^accounts/logout$', django_cas_ng.views.logout, name='cas_ng_logout'),</code></pre>
     <pre class="has"><code class="language-python">url(r'^accounts/callback$', django_cas_ng.views.callback, name='cas_ng_proxy_callback'),</code></pre>
    </div>
    <div>
     （5）配置数据库
    </div>
    <div>
     首先查看你的数据库是否有,下表:
    </div>
    <div>
    </div>
    <div>
     auth_group
     <br/>
     auth_group_permissions
     <br/>
     auth_permission
     <br/>
     auth_user
     <br/>
     auth_user_groups
     <br/>
     auth_user_user_permissions
    </div>
    <div>
     要是没有的话请执行：
    </div>
    <div>
     python manager.py makemigrations
    </div>
    <div>
     python manager.py migrate
    </div>
    <div>
     然后查看是否有：
    </div>
    <div>
     django_cas_ng_proxygrantingticket
    </div>
    <div>
     django_cas_ng_sessionticket
    </div>
    <div>
     要是没有的话，请执行：
    </div>
    <div>
     python manager.py makemigrations  django_cas_ng
    </div>
    <div>
     python manager.py migrate    django_cas_ng
    </div>
    <div>
     （6）
    </div>
    <div>
     要是你的cas server 是http的，现在你就可以访问http://django的ip+port/accounts/login,要是转到cas上，就说明你成功了
    </div>
    <div>
     <span style="color:#3366ff;">
      要是你的cas server 是https的，请在django的主机是添加环境变量
     </span>
    </div>
    <div>
     <span style="color:#3366ff;">
      export REQUESTS_CA_BUNDLE="证书的路径"
     </span>
    </div>
    <div>
     <span style="color:#3366ff;">
      这里的证书必须是PEM格式的，否则会报requests.exceptions.SSLError: unknown error (_ssl.c:3165)
     </span>
    </div>
    <div>
     <span style="color:#3366ff;">
      如果你的证书使用keytool转换的，请再用下面的命令把之前的证书转换为PEM格式的
     </span>
    </div>
    <div>
     <span style="color:#3366ff;">
     </span>
     <pre class="has"><code class="language-python">openssl x509 -inform der -in server.cer -out server.pemaccounts/login</code></pre>
    </div>
    <div>
     <span style="color:#3366ff;">
      到此配置完毕，你可以访问http://django的ip+port/accounts/login，要是转到cas上，就说明你成功了
     </span>
    </div>
