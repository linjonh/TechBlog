# 业内首款云原生技术中台产品云原生 Stack 来了

**简介：**
云原生 Stack 满足了各种典型场景下客户对于线下高集成平台的诉求，让企业数字话转型不受技术约束，专注业务本身，加速企业数字化迭代。

今天，企业数字化转型依然面临很大的挑战，虽然有很多新技术不断涌现，云厂商、ISV 在帮助企业做转型，但是在实际落地过程中，企业仍然需要解决一些痛点，比如：当前，业界技术产品和理念非常繁多，怎么用新技术实现业务快速创新？目前的路径还不是很清晰。

其次，分布式技术复杂度高，运维成本高，业务稳定性挑战大，系统搭建出来之后，存在稳定性不好的问题，在后续运行中也可能出现多种问题。还有就是技术组合集成度不足，缺乏统一的规划和端到端一致的解决方案。

正是因为看到企业在数字化转型中面临的技术难点，阿里云发布业界首款云原生技术中台产品——
**云原生 Stack（简称 CNStack）**
。

云原生 Stack，是云原生时代的技术中台，可以线下轻量、敏捷的输出。技术中台基于 ACK 敏捷版，在用户的基础设施环境，轻量敏捷地享受到与 ACK一样安全、稳定的容器服务，天生具有多集群管理能力。

同时，云原生 Stack 也是业务应用平台，可以满足线下各行各业客户在数字化转型中遇到的所有技术挑战。云原生 Stack 不但可以敏捷输出，也可以规模化输出，更可以在异构 IaaS 上输出。

云原生 Stack 补齐了云原生线下能力，提供了四大核心产品形态：

![图片1.png](https://i-blog.csdnimg.cn/blog_migrate/841108a59b18324124e3a10c8490eabd.png)

**第一个是云原生 Stack for Application：**
它是企业数字化的一站式技术底座，把 ACK 敏捷版、EDAS 应用平台、应用高可用服务 AHAS、可观测 ARMS 集成在一个平台上，具备较高的集成度和一致的体验。云原生 Stack for Application 解决了开发运维一体化，加速业务迭代，内置分布式中间件，一站式满足行业应用云原生的所有诉求，并且提供了大量的解决方案，包括异地多活方案、企业互联网架构方案、业务中台方案、安全生产方案、资源混部方案等。

**以阿里云资源混部解决方案为例，它可以将 IT 基础设施资源的日平均利用率从 10% 的业界平均水平提升到 45%，帮助企业降低 50% 的 IT 成本。**
让企业数字化不受技术约束，更轻量、更简单。

**第二个形态是云原生 Stack for SaaS：**
以容器为代表的云原生技术带来的核心价值是：向下管理资源，向上支撑应用，水平管理软件生命周期。过去在单机时代，软件的安装管理比较容易。但是到了云计算时代，在分布式软件、分布式环境、分布式算力的背景下，软件想要部署发布，升级维护就会变得很复杂，以前行业中完全没有这个问题的解法。

今天，阿里云开创性地提出云原生 Stack for SaaS。让 SaaS 软件的交付更简单，它屏蔽了 IaaS 的差异性，什么环境都可以部署，什么环境都可以适配，每一种硬件、芯片都可以适配。
**云原生 Stack for SaaS可以让部署提效 5 倍，集群测试提效 5 倍，并且降低依赖组件 90% 运维工作量。**
现在已经有超过 50 家企业软件在对接、使用、落地这个产品。

**第三个形态是云原生 Stack for Edge：**
当下，边缘的场景也非常流行，云原生 Stack 是边缘场景和边缘应用的全生命周期管理平台，可以实现边缘资源 1 分钟自动纳管；具备低时延、安全通信能力，消息到达率 99.9%；组件丰富，内置数十款边缘中间件。

**第四个形态是云原生 Stack 社区版：**
社区版是帮助企业和开发者快速试用云原生 Stack 的技术平台，可以免费下载、一键部署，一站式满足研发测试能力，并且可以无缝迁移到商业版、迁移到生产环境，这个版本会在 12 月在阿里云官网公开提供下载。

云原生 Stack 满足了各种典型场景下客户对于线下高集成平台的诉求，让企业数字化转型不受技术约束，专注业务本身，加速企业数字化迭代。
