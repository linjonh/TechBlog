# 【IntelliJ IDEA】idea导入node\_modules卡死，idea 忽略 node\_modules 索引

## 一、背景描述

今天使用到了前端开发的项目，而项目里有node\_modules，在idea打开时，idea扫描该目录并进行索引。然后就出现了，Idea卡死，电脑CPU直接飙到100%，于是赶紧使用任务管理器把Idea干掉。

[原文链接](https://blog.csdn.net/weixin_44299027/article/details/115870890)

## 二、原因分析

node\_modules一般都很大，而且一般不需要对其索引。npm构建的文件包含node\_modules依赖库，文件众多，导入idea中会造成会卡死现象，所以我们需要把node\_modules依赖库设置成忽略文件。

## 三、解决方案

解决此问题可以设置忽略 node\_modules文件夹。以下是详细步骤，只需三步，即可轻松简单彻底搞定。

步骤一：点击 node\_modules 文件夹 ----> 右键 ----> Mark Directory as ----> Excluded

步骤二：File ----> Settings ----> Editor ----> File Types，将node\_modules设置为忽略文件，如下图：

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/35af985a441c40239d9dc1af0be4379f.png)
  
![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/875b4af01da545deb892505c0661fb60.png)
  
步骤三、修改 项目.iml 文件

```bash
修改 项目.iml 文件中，在content节点增加 <excludeFolder url="file://$MODULE_DIR$/node_modules" />

```
