    <p>
     最近学习react，看到别人有在用
     <code>
      craco.config.js
     </code>
     ，于是了解一下，这是一篇小白篇！！
    </p>
    <p>
     在使用
     <code>
      create-react-app
     </code>
     构建项目时，会把
     <code>
      webpack
     </code>
     相关配置都隐藏起来，若需要修改配置可以执行以下命令将配置文件弹出来，但此操作是不可逆的
    </p>
    <pre><code class="prism language-javascript">npm run eject
</code></pre>
    <p>
     需要注意的是，执行 eject 后，你将不再享受 Create React App 提供的抽象和自动化功能，同时也需要自行处理配置和构建的相关问题。这意味着你对项目的配置和构建有更多的责任和掌控，但也能满足更灵活的定制需求。
    </p>
    <p>
     <code>
      craco.config.js
     </code>
     是一个用于自定义 Create React App（CRA）项目配置的文件，通过
     <strong>
      CRACO（Create React App Configuration Override）
     </strong>
     工具实现。CRACO 允许开发者在不执行
     <code>
      eject
     </code>
     命令的情况下，覆盖和扩展 CRA 的默认配置，从而满足更复杂的项目需求。
    </p>
    <h4>
     <a id="cracoconfigjs__12">
     </a>
     <strong>
      <code>
       craco.config.js
      </code>
      的功能
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        自定义 Webpack 配置
       </strong>
       ：
      </p>
      <ul>
       <li>
        可以修改 Webpack 的模块解析规则、添加或替换 Loader，以及配置 Webpack 插件。
       </li>
       <li>
        例如，设置路径别名（如
        <code>
         @
        </code>
        指向
        <code>
         src
        </code>
        目录）或优化输出配置。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        修改 Babel 配置
       </strong>
       ：
      </p>
      <ul>
       <li>
        支持自定义 Babel 的预设和插件，以支持新语法或优化代码。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        集成 ESLint 和 Stylelint
       </strong>
       ：
      </p>
      <ul>
       <li>
        自定义 ESLint 和 Stylelint 的配置规则，以统一代码风格。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        PostCSS 配置
       </strong>
       ：
      </p>
      <ul>
       <li>
        自定义 PostCSS 插件和配置，优化样式处理流程。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        环境特定配置
       </strong>
       ：
      </p>
      <ul>
       <li>
        支持根据开发环境（开发、生产、测试）动态调整配置。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        插件系统
       </strong>
       ：
      </p>
      <ul>
       <li>
        支持通过插件扩展功能，例如集成 Less、Sass 或其他工具。
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="_74">
     </a>
     <strong>
      优势
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       无需
       <code>
        eject
       </code>
      </strong>
      ：保留 CRA 的便利性和后续升级能力。
     </li>
     <li>
      <strong>
       高度可定制
      </strong>
      ：支持 Webpack、Babel、ESLint 等的自定义。
     </li>
     <li>
      <strong>
       插件生态
      </strong>
      ：通过插件扩展功能，满足多样化需求。
     </li>
    </ul>
    <p>
     通过
     <code>
      craco.config.js
     </code>
     ，开发者可以灵活地定制 React 项目的构建过程，而无需深入了解底层实现。
