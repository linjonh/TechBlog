    <p>
     Docker运行hello-world镜像失败或超时，报错：Unable to find image ‘hello-world:latest’ locally
     <br/>
     Trying to pull repository docker.io/library/hello-world …
     <br/>
     /usr/bin/docker-current: missing signature key.
     <br/>
     See ‘/usr/bin/docker-current run --help’.
     <br/>
     如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9a0fcb3770bf4f13ab133ee5f8804bce.png">
      <br/>
      在安装 Docker 并尝试运行 hello-world 时，遇到
      <strong>
       连接超时
      </strong>
      的问题，这通常是因为
      <strong>
       默认的 Docker 镜像源访问速度较慢或不稳定
      </strong>
      所导致的。为了加速 Docker 镜像的下载和提升稳定性，解决这个问题的一种有效方法就是
      <strong>
       更换镜像源
      </strong>
      。
     </img>
    </p>
    <p>
     可以参考官方文档（阿里云）解决步骤：
     <br/>
     <a href="https://cr.console.aliyun.com/cn-shenzhen/instances/mirrors" rel="nofollow">
      https://cr.console.aliyun.com/cn-shenzhen/instances/mirrors
     </a>
    </p>
    <p>
     <strong>
      我的解决方案
     </strong>
     ：
     <br/>
     第一步：配置加速地址：设置registry mirror
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mkdir</span> -p /etc/docker
<span class="token function">sudo</span> <span class="token function">tee</span> /etc/docker/daemon.json <span class="token operator">&lt;&lt;-</span><span class="token string">'EOF'
{
    "registry-mirrors": [
        "https://do.nark.eu.org",
        "https://dc.j8.work",
        "https://docker.m.daocloud.io",
        "https://dockerproxy.com",
        "https://docker.mirrors.ustc.edu.cn",
        "https://docker.nju.edu.cn"
    ]
}
EOF</span>
<span class="token function">sudo</span> systemctl daemon-reload
<span class="token function">sudo</span> systemctl restart docker
systemctl status docker
</code></pre>
    <p>
     第二步：重启完docker之后检查registry mirror刚刚配置的加速地址是否成功
    </p>
    <pre><code class="prism language-bash">docker info
</code></pre>
    <p>
     看到：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a76cb733ba434a8d80462e8d36361aea.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/da40767863db4ceb919a2f11b25ca2a7.png">
       <br/>
       这里出现了Registy Mirrors。说明通过上面的命令已经配置成功。
      </img>
     </img>
    </p>
    <p>
     第三步：运行docker run hello-world
    </p>
    <pre><code class="prism language-bash">docker run hello-world
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/36b1a7fb82e2431c8c09e8fed12e3aae.png"/>
    </p>
    <p>
     此时我们也可以查看是否成功拉取hello-world镜像：
    </p>
    <pre><code class="prism language-bash">docker images
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/31c8f8d75d424732ac7f57bfa2431d35.png">

      Docker运行hello-world镜像失败或超时的问题终于解决了。
