    <p>
     此博客为一份详细的 Markdown 语法入门指南，专门针对在 VSCode 上使用 Markdown 的零基础用户。这份指南将包括 Markdown 的基础语法、在 VSCode 中的安装与使用方式、常见问题及注意事项。
    </p>
    <p>
     Markdown 是一种
     <strong>
      轻量级标记语言
     </strong>
     ，使用纯文本符号来标记格式，使文档既易读又易写。它常用于编写技术文档、博客文章、GitHub 项目README等，在各类平台上广泛流行 (
     <a href="http://www.runoob.com/markdown/md-tutorial.html#:~:text=Markdown%20%E8%83%BD%E8%A2%AB%E4%BD%BF%E7%94%A8%E6%9D%A5%E6%92%B0%E5%86%99%E7%94%B5%E5%AD%90%E4%B9%A6%EF%BC%8C%E5%A6%82%EF%BC%9AGitbook%E3%80%82" rel="nofollow" title="Markdown 教程 | 菜鸟教程">
      Markdown 教程 | 菜鸟教程
     </a>
     )。本指南将按照提纲，帮助零基础用户快速上手 Markdown，并结合 VSCode 编辑器介绍其用法和技巧。
    </p>
    <h3>
     1. 什么是 Markdown？
    </h3>
    <p>
     <strong>
      Markdown
     </strong>
     由约翰·格鲁伯（John Gruber）于 2004 年创建，是一种旨在让人们用纯文本格式编写文档的语言 (
     <a href="http://www.runoob.com/markdown/md-tutorial.html#:~:text=Markdown%20%E6%98%AF%E4%B8%80%E7%A7%8D%E8%BD%BB%E9%87%8F%E7%BA%A7%E6%A0%87%E8%AE%B0%E8%AF%AD%E8%A8%80%EF%BC%8C%E5%AE%83%E5%85%81%E8%AE%B8%E4%BA%BA%E4%BB%AC%E4%BD%BF%E7%94%A8%E6%98%93%E8%AF%BB%E6%98%93%E5%86%99%E7%9A%84%E7%BA%AF%E6%96%87%E6%9C%AC%E6%A0%BC%E5%BC%8F%E7%BC%96%E5%86%99%E6%96%87%E6%A1%A3%E3%80%82" rel="nofollow" title="Markdown 教程 | 菜鸟教程">
      Markdown 教程 | 菜鸟教程
     </a>
     )。它使用直观的符号（如
     <code>
      #
     </code>
     、
     <code>
      *
     </code>
     等）来表示标题、列表等格式，以确保
     <strong>
      可读性
     </strong>
     ：Markdown 文件可以直接以纯文本形式阅读，不显得杂乱，同时也能转换为 HTML 等格式用于展示。
    </p>
    <p>
     Markdown 的优势和用途主要包括：
    </p>
    <ul>
     <li>
      <strong>
       用途广泛
      </strong>
      ：适用于撰写各种内容，如技术文档、笔记、博客文章、电子书等。许多网站和工具（如 GitHub、GitBook、简书等）都支持 Markdown 来书写说明文档或帖子 (
      <a href="http://www.runoob.com/markdown/md-tutorial.html#:~:text=Markdown%20%E8%83%BD%E8%A2%AB%E4%BD%BF%E7%94%A8%E6%9D%A5%E6%92%B0%E5%86%99%E7%94%B5%E5%AD%90%E4%B9%A6%EF%BC%8C%E5%A6%82%EF%BC%9AGitbook%E3%80%82" rel="nofollow" title="Markdown 教程 | 菜鸟教程">
       Markdown 教程 | 菜鸟教程
      </a>
      )。
     </li>
     <li>
      <strong>
       简洁高效
      </strong>
      ：相比富文本编辑器，Markdown 语法简单明了，格式操作主要依靠键盘输入符号，极大提高书写效率。
     </li>
     <li>
      <strong>
       易于转换
      </strong>
      ：Markdown 文档可以轻松转换为 HTML、PDF、Word 等多种格式进行发布和分享。由于 Markdown 文件本质是文本，和版本控制系统（如 Git）配合也非常友好，方便追踪更改历史。
     </li>
    </ul>
    <p>
     总之，Markdown 让创作者专注于内容本身，而无需过多关心排版细节，是编写各类文档的理想选择。
    </p>
    <h3>
     2. 在 VSCode 上使用 Markdown
    </h3>
    <p>
     <strong>
      Visual Studio Code (VSCode)
     </strong>
     对 Markdown 有开箱即用的支持。只需在 VSCode 中
     <strong>
      打开或新建一个扩展名为
      <code>
       .md
      </code>
      的文件
     </strong>
     ，即可开始编写 Markdown。VSCode 默认内置了 Markdown 语法高亮和基本预览功能 (
     <a href="http://www.runoob.com/markdown/md-tutorial.html#:~:text=%E6%9C%AC%E6%95%99%E7%A8%8B%E5%B0%86%E4%BD%BF%E7%94%A8%20VSCode%20%E7%BC%96%E8%BE%91%E5%99%A8%E6%9D%A5%E8%AE%B2%E8%A7%A3%20Markdown%20%E7%9A%84%E8%AF%AD%E6%B3%95%EF%BC%8CVSCode,%E6%94%AF%E6%8C%81%20MacOS%20%E3%80%81Windows%E3%80%81Linux%20%E5%B9%B3%E5%8F%B0%EF%BC%8C%E4%B8%94%E5%8C%85%E5%90%AB%E5%A4%9A%E7%A7%8D%E4%B8%BB%E9%A2%98%E3%80%82" rel="nofollow" title="Markdown 教程 | 菜鸟教程">
      Markdown 教程 | 菜鸟教程
     </a>
     )。
    </p>
    <p>
     为了更好地在 VSCode 中编辑和预览 Markdown，推荐安装以下扩展插件：
    </p>
    <ul>

      <strong>
       Markdown All in One
      </strong>

      <a href="https://www.cnblogs.com/zhangyi1357/p/17940788#:~:text=%E8%AF%A5%E6%8F%92%E4%BB%B6%E5%8A%9F%E8%83%BD%E9%9D%9E%E5%B8%B8%E5%BC%BA%E5%A4%A7%EF%BC%8C%E6%8F%90%E4%BE%9B%E4%BA%86%E5%BF%AB%E6%8D%B7%E9%94%AE%E3%80%81%E7%9B%AE%E5%BD%95%E3%80%81%E8%87%AA%E5%8A%A8%E9%A2%84%E8%A7%88%E3%80%81%E5%9B%BE%E7%89%87%E5%BF%AB%E9%80%9F%E4%B8%8A%E4%BC%A0%E7%AD%89%E5%8A%9F%E8%83%BD%E3%80%82" rel="nofollow" title="VSCode 中优雅地编写 Markdown - zhangyi1357 - 博客园">

     </li>
     <li>
      <strong>
       Markdown Preview Enhanced
      </strong>
      ：功能强大的 Markdown 增强插件，支持数学公式、Mermaid 流程图、PlantUML 绘图、PDF 导出等高级特性 (
      <a href="https://marketplace.visualstudio.com/items?itemName=shd101wyy.markdown-preview-enhanced#:~:text=Markdown%20Preview%20Enhanced%20is%20an,Preview%20Plus%20and%20RStudio%20Markdown" rel="nofollow" title=" Markdown Preview Enhanced - Visual Studio Marketplace ">
       Markdown Preview Enhanced - Visual Studio Marketplace
      </a>
      ) (
      <a href="https://blog.csdn.net/while0/article/details/124677531#:~:text=MPE%20%E5%86%85%E5%B5%8C%E4%BA%86%E5%BE%88%E5%A4%9A%E5%B7%A5%E5%85%B7%EF%BC%8C%E5%8F%AF%E4%BB%A5%E5%B0%86%E7%AD%96%E5%88%92%E4%BB%8E%E5%B0%B4%E5%B0%AC%E7%9A%84p%E5%9B%BE%E5%B7%A5%E4%BD%9C%E4%B8%AD%E8%A7%A3%E6%94%BE%E5%87%BA%E6%9D%A5%EF%BC%88%E6%8A%95%E5%85%A5%E5%B0%B4%E5%B0%AC%E7%9A%84%E7%BC%96%E7%A8%8B%E5%B7%A5%E4%BD%9C%E7%9A%84%E6%80%80%E6%8A%B1%EF%BC%89%E3%80%82" title="在 VSCode 下用 Markdown Preview Enhanced 愉快地写文档-CSDN博客">

     </li>
    </ul>
    <p>
     安装上述插件后，VSCode 将更加得心应手地编辑 Markdown。另外，还可根据需要安装 MarkdownLint（语法风格检查）、Markdown PDF（将文档导出为 PDF）等插件，这里不再详述。
    </p>
    <p>
     <strong>
      在 VSCode 中预览 Markdown 文档
     </strong>
     ：VSCode 自带 Markdown 文件预览功能。打开一个
     <code>
      .md
     </code>
     文件后，可以通过以下方式调出预览：
    </p>

    </p>
    <h3>
     3. Markdown 基本语法
    </h3>
    <p>
     下面介绍 Markdown 的基本语法，包括标题、段落、文本格式等常用元素。每种语法会附带示例，帮助你了解
     <strong>
      Markdown 源码
     </strong>
     和
     <strong>
      预览效果
     </strong>
     之间的对应关系。
    </p>
    <h4>
     标题（Headers）
    </h4>
    <p>
     Markdown 使用
     <code>
      #
     </code>
     符号表示标题级别。
     <code>
      #
     </code>
     的个数对应标题级别：
    </p>
    <ul>
     <li>
      <code>
       # 标题1
      </code>
      表示一级标题（通常文档标题）。
     </li>
     <li>
      <code>
       ## 标题2
      </code>
      表示二级标题（章节标题）。
     </li>
     <li>
      以此类推，最多支持六级标题（
      <code>
       ###### 标题6
      </code>
      ）。
     </li>
    </ul>
    <p>
     示例：
    </p>
    <pre><code># 这是一级标题
## 这是二级标题
### 这是三级标题
</code></pre>
    <p>
     在预览中，一级标题通常字体最大、加粗显示；二级标题次之，以此递减。注意在
     <code>
      #
     </code>
     和标题文字之间
     <strong>
      需要有空格
     </strong>
     。此外，尽量按顺序使用标题级别，不要跳级使用，以保证文档结构清晰。
    </p>
    <h4>
     段落和换行（Paragraphs &amp; Line Breaks）
    </h4>
    <p>
     普通文本直接书写就是一个段落。
     <strong>
      段落之间要空一行
     </strong>
     ，否则会被连成同一段。要在段落内强制换行（即不新起一段），有以下方法：
    </p>
    <ul>
     <li>
      在行尾加
      <strong>
       两个及以上空格
      </strong>
      然后换行，可以插入一个换行（）。
     </li>
     <li>
      使用显式的换行标签
      <code>
       &lt;br&gt;
      </code>
      插入换行。
     </li>
    </ul>
    <p>
     例如，在以下Markdown源码中第二句末尾有两个空格，将产生换行：
    </p>
    <pre><code>这是第一行文本。  
这是第二行文本（同段落换行）。
</code></pre>
    <p>
     预览效果会将两行分别显示但仍属于同一段落。若省略两个空格，上例两行将在预览中合并成一行连续文本。
    </p>
    <h4>
     文本格式（Bold, Italic, Strikethrough）
    </h4>
    <p>
     Markdown 提供基本的
     <strong>
      文字样式
     </strong>
     ，通过符号包围文本实现：
    </p>
    <ul>
     <li>
      <strong>
       粗体
      </strong>
      ：使用双星号
      <code>
       **粗体文本**
      </code>
      （或双下划线
      <code>
       __粗体文本__
      </code>
      ）来加粗文字。
     </li>
     <li>
      <em>
       斜体
      </em>
      ：使用单星号
      <code>
       *斜体文本*
      </code>
      （或单下划线
      <code>
       _斜体文本_
      </code>
      ）来倾斜文字。
     </li>
     <li>
      删除线：使用双波浪线
      <code>
       ~~删除线文本~~
      </code>
      来表示删除或划掉的文字。
     </li>
    </ul>
    <p>
     示例：
    </p>
    <pre><code>这是**加粗**的文字，这是*斜体*，这是~~删除线~~。
</code></pre>
    <p>
     预览中，加粗文本通常以粗体显示，斜体文本以斜体显示，删除线文本会被中间划一条横线。在需要强调或表明修改的场景下，这些格式非常有用。
    </p>
    <h4>
     引用块（Blockquote）
    </h4>
    <p>
     使用
     <code>
      &gt;
     </code>
     引导的段落会被渲染为
     <strong>
      引用块
     </strong>
     。引用块通常用于引用他人的文章、提示信息等。可以在
     <code>
      &gt;
     </code>
     后直接书写内容。引用块也可以
     <strong>
      嵌套
     </strong>
     ：在已有
     <code>
      &gt;
     </code>
     的段落前再添加一个
     <code>
      &gt;
     </code>
     ，就形成嵌套引用。
    </p>
    <p>
     示例：
    </p>
    <pre><code>&gt; 这是一个引用段落，用于引用他人的文字或强调重点。
&gt; 
&gt; 引用可以包含多个段落。在每个段落前都加上`&gt;`。
&gt;&gt; 嵌套引用：在一个引用内再引用。
</code></pre>
    <p>
     预览效果：引用块通常有缩进和左侧的竖线样式。嵌套引用会进一步缩进。通过引用块，可以将引用内容与正文区分开。
    </p>
    <h4>
     代码块与行内代码（Code Blocks &amp; Inline Code）
    </h4>
    <p>
     Markdown 使用反引号 (`) 来表示代码。
    </p>
    <ul>
     <li>
      <strong>
       行内代码
      </strong>
      ：用单个反引号把代码围起来，例如 `printf("Hello");`。行内代码会在预览中以等宽字体显示，背景略有高亮，适合短代码或命令的引用。
     </li>
     <li>
      <strong>
       代码块
      </strong>
      ：使用三个反引号开头和结尾来创建多行代码块。你可以在开头的反引号后指定代码语言（例如 "```python"），以启用语法高亮。
     </li>
    </ul>
    <p>
     示例（行内代码和代码块）：
    </p>
    <pre><code>在 C 语言中，你可以调用 `printf("Hello");` 来打印输出。

```c
#include &lt;stdio.h&gt;
int main() {
    printf("Hello, Markdown!\n");
    return 0;
}
</code></pre>
    <pre><code>
在以上示例中，第一行的 *printf("Hello");* 被渲染为行内代码。随后是一段 C 代码块（用 ```c 标记），在预览中会按C语言语法进行高亮显示。使用代码块可以方便地展示代码片段、终端输出等内容，而不会被Markdown解释为格式标记。

### 列表（Lists）

Markdown 支持**无序列表**和**有序列表**：

- **无序列表**：使用 `-`、`*` 或 `+` 加空格作为列表项开头。多个列表项将形成一个无序列表。  
- **有序列表**：使用数字后紧跟一个英文句点和空格（如 `1. `、`2. `）作为列表项开头。数字的实际值一般不影响输出（所有项用`1.`也会自动编号），但为了清晰通常按照顺序编号。

示例：

```markdown
无序列表示例：
- 项目 A
- 项目 B
  - 子项目 B1  （缩进两个空格形成子列表）
  - 子项目 B2

有序列表示例：
1. 步骤一
2. 步骤二
3. 步骤三
</code></pre>
    <p>
     预览中，无序列表项通常以圆点或小符号显示；有序列表则按顺序编号。注意为了正确形成列表，列表项标记后需接空格，与上一段落之间要有空行分隔。子列表通过在标记前缩进（一般两个或四个空格）来创建。
    </p>
    <h4>
     表格（Tables）
    </h4>
    <p>
     Markdown 可以创建简单的表格。使用管道符号
     <code>
      |
     </code>
     分隔不同的列，使用短横线
     <code>
      -
     </code>
     构成表头下的分隔线。语法如下：
    </p>
    <pre><code>| 商品   | 价格   | 数量 |
| ------ | ------ | ---- |
| 苹果   | 5 元   | 3    |
| 香蕉   | 2 元   | 5    |
</code></pre>
    <p>
     说明：第一行定义表头各列，第二行使用
     <code>
      ---
     </code>
     表示表头与表格主体的分隔（至少三个短横线，可在两侧加
     <code>
      :
     </code>
     来控制列内容对齐方式：居左
     <code>
      :-
     </code>
     , 居中
     <code>
      :-:
     </code>
     , 居右
     <code>
      -:
     </code>
     ），后续行每行一条记录。以上示例将生成一个三列的表格，包含两行数据。
    </p>
    <p>
     在预览中，表格会以网格形式显示，首行通常加粗作为表头。确保每行的管道数量一致，否则可能无法正确渲染表格。此外，Markdown 原生表格不支持复杂的单元格合并等功能，仅适用于简单数据展示。
    </p>
    <h4>
     链接与图片（Links &amp; Images）
    </h4>
    <p>
     编写超链接和插入图片是 Markdown 的重要功能：
    </p>
    <ul>
     <li>
      <strong>
       链接
      </strong>
      ：使用
      <code>
       [链接文本](URL)
      </code>
      的格式创建。点击预览中的链接文本会打开对应的 URL。例如：
      <code>
       [访问百度](https://www.baidu.com)
      </code>
      会显示为一个可点击的“访问百度”超链接。
     </li>
     <li>
      <strong>
       图片
      </strong>
      ：语法与链接类似，但在最前面加一个感叹号
      <code>
       !
      </code>
      。格式为：
      <code>
       ![替代文本](图片URL)
      </code>
      。替代文本会在图片无法加载时显示，也用于屏幕阅读器。示例：
      <code>
       ![Markdown 标志](https://markdown-here.com/img/icon256.png)
      </code>
      会插入一张来自网络的 Markdown 标志图片。
     </li>
    </ul>
    <p>
     请注意，若使用本地图片，可以使用相对路径或绝对路径，例如：
     <code>
      ![本地图片](./images/pic01.png)
     </code>
     。在 VSCode 的 Markdown 预览中，本地图片路径是相对于当前 Markdown 文件的位置的，需要确保路径正确。若图片无法显示，可能需要检查
     <strong>
      路径或启用不安全内容
     </strong>
     （VSCode 默认 Markdown 预览出于安全限制，不加载本地磁盘以外的资源，可通过命令面板搜索“Markdown: Change Preview Security Settings”调整设置以允许本地文件）。
    </p>
    <h4>
     分割线（Horizontal Rule）
    </h4>
    <p>
     分割线可用于分隔内容、强调主题转换。在 Markdown 中，任意一行包含三个或以上的连续符号如
     <code>
      ***
     </code>
     、
     <code>
      ---
     </code>
     或
     <code>
      ___
     </code>
     （星号、短横线、下划线）都将渲染为一条水平分割线。
    </p>
    <p>
     示例：
    </p>
    <pre><code>内容段落1

---

内容段落2
</code></pre>
    <p>
     上述
     <code>
      ---
     </code>
     将被转换为一条水平直线，将段落1与段落2分隔开。分割线前后需各空一行。常用的分割线写法是三个短横线，如上例所示。
    </p>
    <h4>
     任务列表（Task List）
    </h4>
    <p>
     任务列表是 Markdown 的扩展语法（在 GitHub Flavored Markdown 等平台支持）。在无序列表的基础上，通过在列表项前加
     <code>
      [ ]
     </code>
     或
     <code>
      [x]
     </code>
     来表示复选框（未选中或选中状态），可以创建待办事项列表。
    </p>
    <p>
     示例：
    </p>
    <pre><code>待办事项：
- [x] 完成 Markdown 基础语法学习
- [ ] 安装 VSCode 的 Markdown 插件
- [ ] 用 Markdown 撰写一篇笔记
</code></pre>
    <p>
     在预览中，上述列表会显示为带复选框的任务清单，其中已完成的第一项显示为勾选状态。这个功能常用于编写计划、任务清单等。要注意在方括号与列表文本之间留有一个空格。VSCode 的内置预览支持任务列表渲染（它遵循 GitHub 风格 Markdown），但纯文本查看时你仍然会看到原始的
     <code>
      [x]
     </code>
     标记。
    </p>
    <h3>
     4. 高级 Markdown 语法
    </h3>
    <p>
     除了上述基本语法，Markdown 还可以通过一些
     <strong>
      扩展
     </strong>
     实现更高级的排版需求。在 VSCode 中，借助插件和一些技巧，可以使用图表、公式以及自定义样式等高级功能。
    </p>
    <h4>
     Mermaid 图表和数学公式
    </h4>
    <p>
     在文档中插入
     <strong>
      流程图、序列图等图表
     </strong>
     以及
     <strong>
      数学公式
     </strong>
     ，可以使用 VSCode 插件
     <em>
      Markdown Preview Enhanced (MPE)
     </em>
     实现。MPE 插件支持直接使用 Mermaid 语法绘制图表，以及使用 LaTeX 语法插入数学公式 (
     <a href="https://blog.csdn.net/while0/article/details/124677531#:~:text=MPE%20%E5%86%85%E5%B5%8C%E4%BA%86%E5%BE%88%E5%A4%9A%E5%B7%A5%E5%85%B7%EF%BC%8C%E5%8F%AF%E4%BB%A5%E5%B0%86%E7%AD%96%E5%88%92%E4%BB%8E%E5%B0%B4%E5%B0%AC%E7%9A%84p%E5%9B%BE%E5%B7%A5%E4%BD%9C%E4%B8%AD%E8%A7%A3%E6%94%BE%E5%87%BA%E6%9D%A5%EF%BC%88%E6%8A%95%E5%85%A5%E5%B0%B4%E5%B0%AC%E7%9A%84%E7%BC%96%E7%A8%8B%E5%B7%A5%E4%BD%9C%E7%9A%84%E6%80%80%E6%8A%B1%EF%BC%89%E3%80%82" title="在 VSCode 下用 Markdown Preview Enhanced 愉快地写文档-CSDN博客">
      在 VSCode 下用 Markdown Preview Enhanced 愉快地写文档-CSDN博客
     </a>
     )。
    </p>
    <ul>
     <li>
      <strong>
       Mermaid 图表
      </strong>
      ：Mermaid 是一种用文本描述图表的工具。通过编写
      <code>
       mermaid
      </code>
      代码块，在其中写入图表定义，Markdown 预览将渲染出相应的图形。例如，在 Markdown 中输入：
     </li>
    </ul>
    <pre><code>```mermaid
graph LR
    A[开始] --&gt; B[过程] --&gt; C[结束]
```
</code></pre>
    <p>
     以上代码将绘制一个从“开始”到“过程”再到“结束”的流程图（LR表示从左到右）。你可以用类似的方法画流程图、甘特图、时序图等多种图表。需要确保已安装并启用了 MPE 插件，否则普通 Markdown 不会识别
     <code>
      mermaid
     </code>
     代码块。
    </p>
    <ul>
     <li>
      <strong>
       数学公式
      </strong>
      ：使用数学公式时，一般采用 LaTeX 语法。MPE 插件允许我们在 Markdown 中通过
      <code>
       $$ ... $$
      </code>
      包围块级公式，通过
      <code>
       $ ... $
      </code>
      包围行内公式，然后在预览中渲染为数学符号。例如：
     </li>
    </ul>
    <pre><code>这是行内公式 $E=mc^2$ 示例。

$$
E = mc^2
$$
</code></pre>
    <p>
     上述行内公式将在文本中显示为 $E=mc^2$，而独立居中的块级公式会显示为\(E = mc^2\)。你可以编写更复杂的公式，比如矩阵、积分符号等，MPE 将调用数学排版引擎（如 KaTeX）进行渲染，使其显示为标准的数学格式。
    </p>
    <p>
     <strong>
      提示
     </strong>
     ：Mermaid 图表和数学公式都是 Markdown 的扩展功能，需要预览工具的支持。VSCode 官方内置预览并不直接支持它们，但通过安装 MPE 插件即可。使用这些功能时，建议先在 VSCode 中确认预览效果是否符合预期，如果不生效请检查插件是否正确安装并启用。
    </p>
    <h4>
     使用 HTML 语法扩展 Markdown
    </h4>
    <p>
     Markdown 本质上会被转换为 HTML 来显示。因此，你可以在 Markdown 文档中直接编写
     <strong>
      HTML 标签
     </strong>
     来实现一些 Markdown 无法覆盖的布局或格式。
    </p>
    <p>
     例如：
    </p>
    <ul>
     <li>
      如果想插入
      <strong>
       视频
      </strong>
      或更复杂的内容，可以直接粘贴相应的
      <code>
       &lt;video&gt;
      </code>
      、
      <code>
       &lt;iframe&gt;
      </code>
      等 HTML 标签。
     </li>
     <li>
      如果需要控制某段文字的颜色或大小（Markdown 无直接语法），可以使用内联的
      <code>
       &lt;span style="..."&gt;...&lt;/span&gt;
      </code>
      或
      <code>
       &lt;font&gt;
      </code>
      标签（不推荐，已过时）来调整样式。
     </li>
     <li>
      可以通过 HTML 实现更复杂的表格布局，或者定义折叠面板、按钮等交互元素（前提是目标输出环境支持这些 HTML）。
     </li>
    </ul>
    <p>
     需要注意的是，并非所有 Markdown 渲染环境都会完全信任并执行HTML。例如 GitHub的README会屏蔽某些HTML标签出于安全考虑。但在 VSCode 本地预览中，大部分简单 HTML 标签都会按预期渲染。另外，为了保持 Markdown 文档的可读性，只有在确有必要时才掺入 HTML，并且尽量在块级元素（如段落之间）使用，以免破坏 Markdown 原有结构。
    </p>
    <h4>
     自定义 CSS 样式美化 Markdown
    </h4>
    <p>
     VSCode 的 Markdown 预览支持通过自定义 CSS 来调整样式。如果你对默认的预览样式不满意（例如想更改背景颜色、字体、代码块主题等），可以使用以下方法进行美化：
    </p>
    <ul>
     <li>
      <strong>
       更改预览主题
      </strong>
      ：一些插件（如 MPE）内置了多种主题样式，可在其设置中选择。例如 MPE 提供了 GitHub 风格、黑色主题等预览主题，直接修改插件设置
      <code>
       markdown-preview-enhanced.previewTheme
      </code>
      即可应用不同的配色 (
      <a href="https://www.cnblogs.com/zhangyi1357/p/17940788#:~:text=" rel="nofollow" title="VSCode 中优雅地编写 Markdown - zhangyi1357 - 博客园">
       VSCode 中优雅地编写 Markdown - zhangyi1357 - 博客园
      </a>
      )。
     </li>
     <li>
      <strong>
       加载自定义 CSS
      </strong>
      ：VSCode 内置预览允许在设置中指定自定义CSS文件。步骤为：新建一个 CSS 文件，编写你希望覆盖的样式规则（例如修改
      <code>
       h1, h2
      </code>
      标题颜色，调整表格样式等）；然后在 VSCode
      <strong>
       设置
      </strong>
      中搜索 “Markdown &gt; Styles”，将刚才CSS文件的路径添加进去 (
      <a href="https://mazhuang.org/fragment/vscode-markdown-preview-custom-css/#:~:text=VSCode%20%E9%87%8C%E9%BB%98%E8%AE%A4%E7%9A%84Markdown%20%E9%A2%84%E8%A7%88%E6%A0%B7%E5%BC%8F%E4%B8%8D%E5%A4%AA%E5%A5%BD%E7%9C%8B%EF%BC%8C%E5%8F%AF%E4%BB%A5%E8%87%AA%E5%AE%9A%E4%B9%89%E6%A0%B7%E5%BC%8F%E3%80%82%20%E6%96%B9%E6%B3%95%E6%98%AF%E5%9C%A8VSCode%20%E7%9A%84%E9%85%8D%E7%BD%AE%E9%87%8C%EF%BC%8C%E6%89%BE%E5%88%B0Extensions,Markdown%3A%20Styles%EF%BC%8C%E7%84%B6%E5%90%8EAdd%20Item%EF%BC%8C%E5%8F%AF%E4%BB%A5%E6%B7%BB%E5%8A%A0%E5%BD%93%E5%89%8D%E5%B7%A5%E4%BD%9C" rel="nofollow" title="VSCode 自定义Markdown 预览样式 - 码志">
       VSCode 自定义Markdown 预览样式 - 码志
      </a>
      )。再次打开 Markdown 预览时，就会应用你的定制样式。这样你可以彻底掌控预览的外观，使之符合你的审美或阅读习惯。
     </li>
    </ul>
    <p>
     通过自定义 CSS，你甚至可以使 VSCode 的 Markdown 预览风格与特定的网站风格一致（比如 GitHub 风格），或者实现夜间深色模式预览等。但请注意，此操作仅影响本地预览，对实际导出的HTML或其他平台上的显示无直接影响（除非那些平台也引用了你的CSS）。
    </p>
    <h3>
     5. VSCode Markdown 常见问题及解决方案
    </h3>
    <p>
     在使用 VSCode 编辑 Markdown 时，可能会遇到一些常见问题。以下列出几个问题及对应的解决方法：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        预览无法打开
       </strong>
       ：当按下快捷键或命令后，Markdown 预览没有出现时，首先确认文件已保存且扩展名为
       <code>
        .md
       </code>
       。然后确保 VSCode 的 Markdown 预览功能正常启用（可尝试通过菜单
       <strong>
        “打开预览”
       </strong>
       来验证）。如果安装了第三方 Markdown 插件，某些插件设置可能影响内置预览，尝试禁用冲突的插件。最后，检查命令是否正确，例如应使用
       <code>
        Markdown: Open Preview to the Side
       </code>
       而非其它相似命令。
      </p>
     </li>
     <li>
      <p>
       <strong>
        图片无法显示
       </strong>
       ：预览中图片不显示通常有两种情况：其一，图片路径错误或图片文件不存在。请检查 Markdown 中引用的路径是否有效（相对路径要相对于当前 Markdown 文件）。其二，VSCode 出于安全设置阻止加载本地或非安全链接图片。如果确定路径无误却仍不显示，可在命令面板运行 “Markdown: Change Preview Security Settings”（更改预览安全设置），选择放宽安全限制（例如允许本地文件或不安全内容），然后重新加载预览 (
       <a href="https://juejin.cn/post/7094968757256192037#:~:text=VSCode%E5%BC%80%E5%90%AF%E4%BE%A7%E8%BE%B9%E9%A2%84%E8%A7%88%EF%BC%8CMarkDown%E6%96%87%E4%BB%B6%E4%B8%AD%E5%9B%BE%E7%89%87%E6%97%A0%E6%B3%95%E6%98%BE%E7%A4%BA%20,%E6%89%93%E5%BC%80%E5%91%BD%E4%BB%A4%E6%A1%86%20%C2%B7%20%E6%90%9C%E7%B4%A2MarkDown%EF%BC%8C%E6%89%BE%E5%88%B0%E6%9B%B4%E6%94%B9%E9%A2%84%E8%A7%88%E5%AE%89%E5%85%A8%E8%AE%BE%E7%BD%AE%E3%80%82%20%C2%B7%20%E9%80%89%E6%8B%A9%E5%85%81%E8%AE%B8%E4%B8%8D%E5%AE%89%E5%85%A8%E5%86%85%E5%AE%B9%E5%8D%B3%E5%8F%AF%E3%80%82" rel="nofollow" title="VSCode开启侧边预览，MarkDown文件中图片无法显示 - 稀土掘金">
        VSCode开启侧边预览，MarkDown文件中图片无法显示 - 稀土掘金
       </a>
       )。一般来说，本地图片确保路径正确即可显示，网络图片则要求有稳定的外链地址。
      </p>
     </li>
     <li>
      <p>
       <strong>
        代码高亮失效
       </strong>
       ：代码块未呈现出应有的高亮可能有几种原因。首先，确认你在三反引号后正确标注了语言（例如
       <code>
        java、
       </code>
       python 等）；如果未标注语言，预览可能默认不高亮或只能做基本高亮。其次，可能是 VSCode 使用的主题对代码块配色不明显，你可以切换 VSCode 主题或 Markdown 预览主题来查看效果。另外，若使用了 Markdown 扩展插件，检查其设置中是否关闭了代码高亮功能。一般VSCode内置预览使用
       <strong>
        markdown-it
       </strong>
       引擎遵循 CommonMark 标准并支持大部分语言的高亮，如发现某种语言不高亮，可以考虑安装相应语言的 VSCode 扩展以提供语法支持。
      </p>
     </li>
     <li>
      <p>
       <strong>
        表格格式问题
       </strong>
       ：有时 Markdown 表格在预览中未正确显示，常见原因是表格语法不规范。确保表格有正确的管道符和分隔线——尤其是第二行的
       <code>
        ---
       </code>
       分隔符，每列至少三个短横线，且管道符数量匹配。表格前后需要空行分隔，否则可能被识别成上一段内容的一部分。此外，如果表格内容较长，换行处可能需要在单元格内使用
       <code>
        &lt;br&gt;
       </code>
