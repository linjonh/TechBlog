    <h1 class="title-article" id="articleContentId">
     scala 中的@BeanProperty
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     <strong>
      在 Scala 中，
      <code>
       @BeanProperty
      </code>
      是一个注解，用于自动生成符合 Java Bean 规范的 getter 和 setter 方法。Java Bean 规范要求类的属性具有标准的
      <code>
       get
      </code>
      和
      <code>
       set
      </code>
      方法，而 Scala 默认生成的 getter 和 setter 方法并不符合这一规范。通过使用
      <code>
       @BeanProperty
      </code>
      注解，可以方便地为类的字段生成符合 Java Bean 规范的 getter 和 setter 方法。
     </strong>
    </p>
    <h4>
     <strong>
      使用
      <code>
       @BeanProperty
      </code>
     </strong>
    </h4>
    <p>
     <strong>
      <code>
       @BeanProperty
      </code>
      注解可以应用于类的字段（
      <code>
       val
      </code>
      或
      <code>
       var
      </code>
      ）。它会自动生成以下方法：
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        对于
        <code>
         var
        </code>
        字段：生成
        <code>
         get
        </code>
        和
        <code>
         set
        </code>
        方法。
       </strong>
      </p>
     </li>
     <li>
      <p>
       <strong>
        对于
        <code>
         val
        </code>
        字段：仅生成
        <code>
         get
        </code>
        方法（因为
        <code>
         val
        </code>
        是不可变的）。
       </strong>
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      示例
     </strong>
    </h5>
    <pre><code class="language-Scala">import scala.beans.BeanProperty

class Person {
  @BeanProperty var name: String = _
  @BeanProperty val age: Int = 25
}</code></pre>
    <h5>
     <strong>
      生成的代码
     </strong>
    </h5>
    <p>
     <strong>
      上述代码等价于以下 Java 代码：
     </strong>
    </p>
    <pre><code class="language-Scala">public class Person {
  private String name;
  private final int age = 25;

  // 对于 name 字段
  public String getName() { return this.name; }
  public void setName(String name) { this.name = name; }

  // 对于 age 字段
  public int getAge() { return this.age; }
}</code></pre>
    <h5>
     <strong>
      使用示例
     </strong>
    </h5>
    <pre><code class="language-Scala">val person = new Person()
person.setName("Alice")  // 调用生成的 setter 方法
println(person.getName)  // 调用生成的 getter 方法，输出: Alice
println(person.getAge)   // 调用生成的 getter 方法，输出: 25</code></pre>
    <h4>
     <strong>
      注意事项
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         val
        </code>
        和
        <code>
         var
        </code>
        的区别：
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          对于
          <code>
           val
          </code>
          字段，只会生成
          <code>
           get
          </code>
          方法，因为
          <code>
           val
          </code>
          是不可变的。
         </strong>
        </p>
       </li>
       <li>
        <p>
         <strong>
          对于
          <code>
           var
          </code>
          字段，会生成
          <code>
           get
          </code>
          和
          <code>
           set
          </code>
          方法。
         </strong>
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        默认值：
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          如果字段没有初始化值（如
          <code>
           var name: String = _
          </code>
          ），Scala 会为其赋予默认值（
          <code>
           null
          </code>
          用于引用类型，
          <code>
           0
          </code>
          用于数值类型，
          <code>
           false
          </code>
          用于布尔类型等）。
         </strong>
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        与 Scala 原生方法的共存：
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          使用
          <code>
           @BeanProperty
          </code>
          注解后，Scala 仍然会生成原生的 getter 和 setter 方法（例如
          <code>
           name
          </code>
          和
          <code>
           name_=
          </code>
          ）。
         </strong>
        </p>
       </li>
       <li>
        <p>
         <strong>
          因此，你可以同时使用 Java Bean 风格的方法和 Scala 风格的方法。
         </strong>
        </p>
       </li>
      </ul>
      <pre><code class="language-Scala">val person = new Person()
person.setName("Bob")  // Java Bean 风格的 setter
person.name = "Charlie" // Scala 风格的 setter
println(person.getName) // Java Bean 风格的 getter
println(person.name)    // Scala 风格的 getter</code></pre>
     </li>
     <li>
      <p>
       <strong>
        适用场景：
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          当你需要与 Java 代码交互时（例如在 Java 中调用 Scala 类的方法），
          <code>
           @BeanProperty
          </code>
          非常有用。
         </strong>
        </p>
       </li>
       <li>
        <p>
         <strong>
          如果你只在 Scala 中使用该类，通常不需要使用
          <code>
           @BeanProperty
          </code>
          ，直接使用 Scala 原生的 getter 和 setter 即可。
         </strong>
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <strong>
      总结
     </strong>
    </h4>
    <p>
     <strong>
      <code>
       @BeanProperty
      </code>
      是 Scala 提供的一个便捷工具，用于生成符合 Java Bean 规范的 getter 和 setter 方法。它特别适用于需要与 Java 代码交互的场景。通过使用
      <code>
       @BeanProperty
      </code>
      ，你可以避免手动编写冗长的 getter 和 setter 方法，同时保持代码的简洁性和兼容性。
     </strong>
    </p>
