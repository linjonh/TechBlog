    <h2>
     1.RabbitMq介绍
    </h2>
    <p>
     RabbitMQ 是一款开源的消息队列中间件，基于 AMQP（高级消息队列协议）实现，支持多种编程语言和平台。以下是其核心特点和介绍：
    </p>
    <h4>
     <strong>
      核心特点
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        多语言支持
       </strong>
       <br/>
       提供 Java、Python、C#、Go、JavaScript 等语言的客户端库，适配不同开发场景。
      </p>
     </li>
     <li>
      <p>
       <strong>
        高可靠性
       </strong>
      </p>
      <ul>
       <li>
        支持消息持久化，确保数据不丢失。
       </li>
       <li>
        提供消息确认机制（ACK），保证消息被正确处理。
       </li>
       <li>
        通过集群模式实现高可用性和负载均衡。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        灵活的路由机制
       </strong>
      </p>
      <ul>
       <li>
        通过
        <strong>
         Exchange（交换器）
        </strong>
        管理消息路由规则，支持 Direct、Fanout、Topic、Headers 等类型。
       </li>
       <li>
        消息最终被路由到
        <strong>
         Queue（队列）
        </strong>
        中供消费者处理。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        支持多种协议
       </strong>
       <br/>
       除 AMQP 外，还支持 MQTT、STOMP、HTTP 等协议，适配物联网、实时通信等场景。
      </p>
     </li>
     <li>
      <p>
       <strong>
        轻量级与低延迟
       </strong>
       <br/>
       适合中小企业和对延迟敏感的应用，如订单处理、支付系统等。
      </p>
     </li>
    </ol>
    <h4>
     <strong>
      架构组成
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Producer（生产者）
      </strong>
      ：发送消息到 RabbitMQ。
     </li>
     <li>
      <strong>
       Consumer（消费者）
      </strong>
      ：从队列中获取消息并处理。
     </li>
     <li>
      <strong>
       Broker
      </strong>
      ：RabbitMQ 服务器实例，负责接收、存储和转发消息。
     </li>
     <li>
      <strong>
       Exchange
      </strong>
      ：根据路由规则将消息分发到队列。
     </li>
     <li>
      <strong>
       Queue
      </strong>
      ：存储消息的缓冲区，消费者从队列中拉取消息。
     </li>
    </ul>
    <h4>
     <strong>
      典型应用场景
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        异步处理
       </strong>
       <br/>
       解耦服务间依赖，例如用户下单后异步发送短信或邮件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        削峰填谷
       </strong>
       <br/>
       应对突发流量，如秒杀活动中暂存订单请求，避免系统过载。
      </p>
     </li>
     <li>
      <p>
       <strong>
        日志处理
       </strong>
       <br/>
       集中收集和分发日志，支持多服务订阅。
      </p>
     </li>
     <li>
      <p>
       <strong>
        微服务通信
       </strong>
       <br/>
       作为服务间异步通信的桥梁，降低耦合度。
      </p>
     </li>
    </ol>
    <h4>
     <strong>
      优缺点
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：
      </p>
      <ul>
       <li>
        高可靠性和灵活的路由策略。
       </li>
       <li>
        社区活跃，文档完善。
       </li>
       <li>
        支持多种协议和编程语言。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：
      </p>
      <ul>
       <li>
        吞吐量相对 Kafka 较低，不适合超大规模数据处理。
       </li>
       <li>
        部署和配置相对复杂（尤其在集群模式下）。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <strong>
      总结
     </strong>
    </h4>
    <p>
     RabbitMQ 是一款功能强大、成熟稳定的消息队列，适合对可靠性和灵活性要求较高的中小型项目。如果需要处理海量数据或追求极致性能，可考虑 Kafka 或 RocketMQ 等其他方案。
    </p>

#include &lt;amqpcpp.h&gt;
#include &lt;amqpcpp/libev.h&gt;
#include &lt;openssl/ssl.h&gt;
#include &lt;openssl/opensslv.h&gt;

{
    // 创建一个事件循环对象

    // 创建一个LibEvHandler对象

        return -1;

        std::cout &lt;&lt; "Exchange declared" &lt;&lt; std::endl;
    });

    channel.declareQueue("test_queue")\
    .onError([](const std::string&amp; message){
        std::cout &lt;&lt; "Error2: " &lt;&lt; message &lt;&lt; std::endl;
        return -1;

        return -1;
    })\
    .onSuccess([](){
        std::cout &lt;&lt; "Queue bind" &lt;&lt; std::endl;
