    <h2>
     <span style="font-size:18px">
      MPP数据库定义
     </span>
    </h2>
    <p>
     <span style="font-size:18px">
      MPP即大规模并行处理（Massively Parallel Processor ）。 在数据库非共享集群中，每个节点都有独立的磁盘存储系统和内存系统，业务数据
     </span>
     <span style="font-size:18px">
      根据数据库模型和应用特点划分到各个节点上，每台数据节点通过专用网络或者商业通用网络互相连接，彼此协同计算，作为整体提供数据 库服务。非共享数据库集群有完全的可伸缩性、高可用、高性能、优秀的性价比、资源共享等优势。
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      <img alt="" src="https://img-blog.csdn.net/20150413222859097?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvbHljNDE3MzU2OTM1/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center">
       <br/>
      </img>
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      大规模并行处理(MPP)架构
      <br/>
     </span>
    </p>
    <p>
     <img alt="" src="https://img-blog.csdn.net/20150413222935866?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvbHljNDE3MzU2OTM1/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center">
      <br/>
     </img>
    </p>
    <p>
     <br/>
    </p>
    <p>
     <span style="font-size:18px">
      Vertica无共享MPP的特点
      <br/>
     </span>
    </p>
    <ul>
     <li>
      <span style="font-size:18px">
       无特殊节点
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       所有节点对等
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       可通过任意节点查询或加载数据
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       实时加载与查询同步进行
      </span>
     </li>
    </ul>
    <p>
     <span style="font-size:18px">
      目前常见的MPP架构数据库
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      1.GREENPLUM
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      2.DB2 DPF架构数据库
     </span>
    </p>
    <p>
     <span style="font-size:18px">
      3.vertica mpp数据库
     </span>
     <br/>
    </p>
    <h2>
     为什么选择MPP
    </h2>
    <h3>
     • PB级别的分析需求
    </h3>
    <ul>
     <li>
      <span style="font-size:18px">
       传统数据库无法支持大规模集群与PB级别数据量
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       性能受限
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       扩展性受限
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       成本高昂
      </span>
     </li>
    </ul>
    <h3>
     • 软硬件一体机成本高昂、扩展受限
    </h3>
    <ul>
     <li>
      <span style="font-size:18px">
       生产扩容、测试、开发、容灾都需新购同型号一体机（机柜）
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       跨代兼容性问题
      </span>
     </li>
    </ul>
    <h3>
     • In-memory技术太贵而且不成熟
    </h3>
    <ul>
     <li>
      <span style="font-size:18px">
       内存成本高
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       TB级别以下，不适合大数据量
      </span>
     </li>
    </ul>
    <h3>
     • Hadoop技术的先天不足
    </h3>
    <ul>
     <li>
      <span style="font-size:18px">
       Hive等sql-on-hadoop性能太慢
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       SQL兼容性与支持不足
      </span>
     </li>
     <li>
      <span style="font-size:18px">
       数据安全性无法保证
      </span>
     </li>
    </ul>
    <p>
     <span style="font-size:18px">
      大数据之惑
     </span>
     ，
     <span style="font-size:18px">
      面对越来越庞大的数据，企业如何利用好大数据值得思考，HADOOP，MPP，PPT在精彩，还是一步步的转变。
     </span>
     <br/>
    </p>
