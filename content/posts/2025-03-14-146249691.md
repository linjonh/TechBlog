     Python中列表、元组和字典的对比
    </h2>
    <p>
     以下是Python中列表（List）、元组（Tuple）和字典（Dictionary）的对比表格，帮助理解它们的特性与使用场景：
    </p>
    <h3>
     <a id="_4">
     </a>
     基本特性对比
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        列表（List）
       </th>
       <th>
        元组（Tuple）
       </th>
       <th>
        字典（Dictionary）
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        可变性
       </td>
       <td>
        ✅ 可修改（增删改元素）
       </td>
       <td>
        ❌ 不可修改（创建后固定）
       </td>
       <td>
        ✅ 键不可变，值可变（键必须是不可变类型，如字符串、数字、元组）
       </td>

      <tr>
       <td>
        有序性
       </td>
       <td>
        ✅ 有序（元素按插入顺序存储）
       </td>
       <td>
        ✅ 有序（元素按插入顺序存储）
       </td>

        ❌ 无序（Python 3.7+ 后保留插入顺序，但设计上仍视为"无序"）

       <td>

       </td>

       </td>
      </tr>

       <td>
        语法符号
       </td>
       <td>
        方括号
        <code>
         []
        </code>
        ，如
        <code>
         [1, 2, 3]
        </code>
       </td>
       <td>
        圆括号
        <code>
         ()
        </code>
        ，如
        <code>
         (1, 2, 3)

       </td>
       <td>
        花括号
        <code>
         {}
        </code>

        <code>
         {'a': 1, 'b': 2}

       <td>
        操作特点
       </td>
       <td>
        支持
        <code>
         append()
        </code>
        、
        <code>
         insert()
        </code>
        、
        <code>
         pop()
        </code>
        等增删操作；索引和切片灵活
       </td>

        仅支持查询（如
        <code>
         index()
        </code>
        、
        <code>
         count()

       <td>

       </td>
       <td>
        较低（因动态分配内存）
       </td>
       <td>

       <td>

       </td>

      <tr>
       <td>
        使用场景
       </td>
       <td>
        动态数据集合（如用户输入列表、待处理数据）

       </td>
       <td>
        键值映射（如数据库记录、JSON数据、缓存）
       </td>

     注意事项

       </td>

    <h3>
     <a id="_32">
     </a>
     总结对比
    </h3>
    <h4>
     <a id="_vs__34">
     </a>
     列表 vs 元组
    </h4>
    <ul>
     <li>
      列表用于
      <strong>
       需要修改的动态数据
      </strong>
      ，元组用于
      <strong>
       不可变数据
      </strong>
      （如函数多返回值、字典键）
     </li>
     <li>
      元组性能更优，适合存储大量静态数据
     </li>
    </ul>
    <h4>
     <a id="_vs__38">
     </a>
     字典 vs 列表/元组
    </h4>
    <ul>
     <li>
      字典通过
      <strong>
       键快速定位值
      </strong>
      ，适合关联数据；列表/元组通过
      <strong>
       索引或顺序
      </strong>
      访问元素
     </li>
     <li>
      字典的键不可变，但值可以是任意类型
     </li>
    </ul>
    <h3>
     <a id="_42">
     </a>
     使用场景示例
    </h3>
    <ul>
     <li>
      <strong>
       列表
      </strong>
      ：处理用户输入、循环中动态添加数据
     </li>
     <li>
      <strong>
       元组
      </strong>
      ：定义常量、多返回值函数、作为字典的键
     </li>
     <li>
      <strong>
       字典
      </strong>
      ：JSON数据处理、缓存系统、快速查找映射关系
     </li>
    </ul>
