     <a id="_1">
     </a>
     一、百度搜索框测试
    </h3>
    <p>
     功能测试：
     <br/>
     （1）正常测试
    </p>
    <ul>
     <li>
      输入字符类型
     </li>
     <li>
      输入长度范围限制
     </li>
     <li>
      是否支持复制粘贴
     </li>
     <li>
      输入搜索能否正确跳转至对应内容页面
     </li>
     <li>
      输入时是否有智能搜索推荐
     </li>
     <li>
      输入内容和推荐内容是否相关
     </li>
     <li>
      点击输入框时是否有光标显示
     </li>
     <li>
      多次输入相同内容，搜索前后是否一致
     </li>
     <li>
      输入删除再次输入搜索
     </li>
     <li>
      输入长度大于百度搜索框可显示长度，前面内容是否被覆盖
      <br/>
      （2）异常测试
     </li>
     <li>
      无网络搜索
     </li>
     <li>
      弱网搜索（弱网：信号差、网速慢的情况，一般低于2G的速率都属于弱网）
     </li>
     <li>
      手机来电、关机、重启、系统升级
     </li>
    </ul>
    <p>
     性能测试：
    </p>
    <ul>
     <li>
      最多可允许输入的字符数
     </li>
     <li>
      搜索后页面跳转速度
     </li>
     <li>
      不同网络响应速度：网络正常：2G、3G、4G和5G；不同宽带
     </li>
     <li>
      多用户同一时间并发操作响应时间
     </li>
    </ul>
    <p>
     兼容性测试：
    </p>
    <ul>
     <li>
      不同操作系统
     </li>
     <li>
      不同浏览器
     </li>
     <li>
      不同百度版本
     </li>
     <li>
      不同手机类型
     </li>
     <li>
      不同分辨率
     </li>
    </ul>
    <p>
     安全性：
    </p>
    <ul>
     <li>
      敏感隐私内容禁止搜索
     </li>
     <li>
      被删除、加密文件搜索不出内容
     </li>
    </ul>
    <p>
     易用性：
    </p>
    <ul>
     <li>
      输入搜索内容后，可以点击回车键开始搜索
     </li>
     <li>
      支持复制粘贴
     </li>
     <li>
      输入内容有智能推荐
     </li>
    </ul>
    <p>
     界面：
    </p>
    <ul>
     <li>
      界面布局、颜色搭配合理
     </li>
     <li>
      无错别字
     </li>
    </ul>
    <p>
     使用百度搜索进行性能测试1；http://t.zoukankan.com/Grace7582-p-4804100.html
     <br/>
     使用百度搜索进行性能测试2。https://www.pianshen.com/article/7718934683/
    </p>
    <h3>
     <a id="_49">
     </a>
     二、银行卡转账功能测试
    </h3>
    <p>
     <code>
      通过选择付款账户，填写转账金额、输入收款账户相关信息以及附加信息，填写好相关信息后，进入安全认证页面选择安全工具、填写安全认证页面选择安全工具、填写安全密码，最后提交。
     </code>
     <br/>
     功能测试：
     <br/>
     （1）正常测试
    </p>
    <ul>
     <li>
      付款账户、金额、收款账户、安全认证全部输入正确，可以转账成功
     </li>
     <li>
      收款账户、收款人姓名、开户行需要全部对应
     </li>
     <li>
      金额在0-账户余额之间，同时需要对边界值进行测试
     </li>
     <li>
      如果收款账户与付款账户属于同一银行，只需进行收款账户姓名匹配即可，无需选择开户行信息
     </li>
     <li>
      收款账户需要考虑：行内账户、跨行转账、异地转账、借记卡、信用卡账号、同名账户（本人、本行）、转活期存折、转定期存折（变定期）、转定期存单
     </li>
     <li>
      安全认证：U盾、手机验证码
      <br/>
      （2）异常测试
     </li>
     <li>
      收款账户、姓名和开户行三者任意不匹配
     </li>
     <li>
      付款账户金额不够
     </li>
     <li>
      小于等于0的情况需要边界测试
     </li>
     <li>
      信用卡、定期存折不能转出，一般会在账户选择是进行屏蔽
     </li>
     <li>
      转出账户被销户、冻结、挂失等异常状态不能转出
     </li>
     <li>
      正常状态、挂失状态的卡可以转入金额，冻结、销户卡不能收到转入金额
     </li>
     <li>
      U盾密码错误、使用他人U盾、验证码错误、超时、为空都会导致交易失败
     </li>
    </ul>
    <p>
     特殊业务场景：
    </p>
    <ul>
     <li>
      手续费**元封顶
     </li>
     <li>
      最低手续费
     </li>
     <li>
      手续费折扣
     </li>
    </ul>
    <p>
     其他特殊场景：
    </p>
    <ul>
     <li>
      转账当时限额、单笔限额（具体安全认证工具）
     </li>
     <li>
      转账对公账户
     </li>
     <li>
      转账汇路考虑，普通转账、实时转账的测试
     </li>
     <li>
      外币转账
     </li>
     <li>
      转入账户与转出账户相同
     </li>
     <li>
      转账超时会导致转账失败（手续费照扣）
     </li>
     <li>
      输入框验证和用户体验等非功能考虑
     </li>
    </ul>
    <h3>
     <a id="_82">
     </a>
     三、微信发送图片测试
    </h3>
    <p>
     功能测试：
    </p>
    <ul>
     <li>
      单个图片不以原图发出，图片是否清楚
     </li>
     <li>
      单个图片以原图发出，接收图片是否是原图
     </li>
     <li>
      一次做多可以发多少张图片
     </li>
     <li>
      一次发出图片超出是否限制，是否发出正常提示
     </li>
     <li>
      未勾选的图片是否会发出
     </li>
     <li>
      图片尺寸超级大是否可以正常发送
     </li>
     <li>
      图片尺寸超级小是否可以正常发送
     </li>
     <li>
      图片大小超过1G是否可以发送
     </li>
     <li>
      不同图片格式如：jpg、png、jpeg等格式是否可以发出
     </li>
    </ul>
    <p>
     兼容性测试：
    </p>
    <ul>
     <li>
      不同操作平台是否可以接收图片
     </li>
     <li>
      不同操作系统是否可以接收图片
     </li>
     <li>
      不同微信版本是否可以接收图片
     </li>
    </ul>
    <p>
     性能测试：
    </p>
    <ul>
     <li>
      多人同时发送图片，是否正常
     </li>
     <li>
      最大图片发送出去是否有影响
     </li>
     <li>
      发送出去后，流量是否正常
     </li>
     <li>
      发送出去耗时多久
     </li>
    </ul>
    <p>
     界面测试：
    </p>
    <ul>
     <li>
      发送图片以及选择图片发送是否符合需求和设计文档
     </li>
     <li>
      是否存在错别字
     </li>
    </ul>
    <p>
     网络测试：
    </p>
    <ul>
     <li>
      弱网下，发送图片是否正常
     </li>
     <li>
      在发送图片切换移动网络时，WiFi是否失效
     </li>
    </ul>
    <p>
     安全测试：
    </p>
    <ul>
     <li>
      图片设计少儿不宜的图片是否提示
     </li>
     <li>
      发送图片是否存在丢失资源
     </li>
    </ul>
    <h3>
     <a id="_117">
     </a>
     四、淘宝支付功能测试
    </h3>
    <p>
     <code>
      根据不同维度，我们可以把淘宝支付分为如下所示的种类：
     </code>
     <br/>
     支付金额：
    </p>
    <ul>
     <li>
      小于最小值：如0.01
     </li>
     <li>
      大于最大值或金额上线
     </li>
     <li>
      无意义金额，如0元
     </li>
     <li>
      格式错误（负数、非数字）
     </li>
     <li>
      余额小于实际支付金额
     </li>
     <li>
      超过第三方支付接口当日消费或单笔消费金额
     </li>
     <li>
      第三方接口，如微信、支付宝、网银、post机终端服务
     </li>
    </ul>
    <p>
     支付操作：
    </p>
    <ul>
     <li>
      账号+免密支付
     </li>
     <li>
      指纹、人脸支付
     </li>
     <li>
      免密支付
     </li>
     <li>
      银行卡密码支付
     </li>
     <li>
      信用卡支付
     </li>
    </ul>
    <p>
     异常处理：
    </p>
    <ul>
     <li>
      退款处理
     </li>
     <li>
      支付数据交换时终端（断网、断电、弱网），重新启动能否再支付
     </li>
     <li>
      支付失败后如何处理
     </li>
     <li>
      支付金额不足时，充值后可否继续支付
     </li>
     <li>
      持续点击
     </li>
     <li>
      多次扣款如何处理退款
     </li>
     <li>
      取消支付后是否能再次支付
     </li>
     <li>
      第三方支付未登录时支付
     </li>
    </ul>
    <p>
     兼容性：
    </p>
    <ul>
     <li>
      不同操作平台是否可以支付（PC/笔记本/平板/手机端）
     </li>
     <li>
      不同操作系统是否可以支付
     </li>
     <li>
      不同支付宝版本是否可以支付
     </li>
    </ul>
    <p>
     后台处理订单：
    </p>
    <ul>
     <li>
      成功订单财务处理
     </li>
     <li>
      失败订单财务处理
     </li>
     <li>
      退款订单财务处理
     </li>
     <li>
      差错账单如何处理等
     </li>
    </ul>
