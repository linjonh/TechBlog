    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       2024 年对
       <a href="https://glasskube.dev/" rel="nofollow" title="Glasskube">
        Glasskube
       </a>
       来说是充满活力的一年。2 月，我们推出了一个 Kubernetes 管理器，它最初是由我们的团队在产品发布失败后内部开发的，部分原因是当前使用的包管理工具的限制。在这次不冷不热的 Kubernetes 操作员首次亮相后，Glasskube 创始人
       <a href="https://www.linkedin.com/in/pmigat/" rel="nofollow" title="Philip">
        Philip
       </a>
       和
       <a href="https://www.linkedin.com/in/louisnweston/" rel="nofollow" title="Louis">
        Louis
       </a>
       在 Hacker News 上分享了他们的发布事后分析。他们发现，许多其他人对现有的 Kubernetes 包管理解决方案也遇到了类似的挫折。认识到这一共同的痛苦，产品转型变得不可避免，Glasskube 包管理器开始形成。
      </span>
     </span>
    </p>
    <p class="img-center">
     <a class="article-body-image-wrapper" href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F8z9kjsjsff7p8jtertau.png" rel="nofollow">
      <img alt="the-start （开始）" height="535" src="https://img-blog.csdnimg.cn/img_convert/bfc2c9ad8f2c9e75d7b18d618025e9b7.png" width="800"/>
     </a>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       现在，才同年 8 月中旬，我们已经接近 150 万次下载、2.6k 个 GitHub 星和 50+ 贡献者。我们见证了采用、功能和用例的显著增长。随着我们的不断发展，我们想花点时间回顾一下我们迄今为止完成的工作。这篇博文是一次全面的更新，是一面旗帜，旨在分享您需要了解的有关 Glasskube 当前状态、即将发生的事情以及您如何帮助塑造 Kubernetes 包管理的未来的所有信息。
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       欢迎来到 Kubernetes 包管理的新前沿。
      </span>
     </span>
    </p>
    <h3 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       介绍
      </span>
     </span>
    </h3>
    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       为什么 Kubernetes 需要更好的包管理
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       自诞生以来，
       <a href="https://kubernetes.io/" rel="nofollow" title="Kubernetes">
        Kubernetes
       </a>
       作为容器编排系统从未停止过普及，其采用率不断提高。近年来，
       <a href="https://landscape.cncf.io/" rel="nofollow" title="可用软件包">
        可用软件包
       </a>
       的数量已从大约 100 个增加到 800 多个。这种增长无论如何都非常令人印象深刻，显示了 Kubernetes 生态系统的成熟度，但它也揭示了当前包管理工具的问题。开发人员经常在以下方面苦苦挣扎：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       复杂且高度定制的工作流程
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       耗时且多次手动流程
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       过度依赖当前的包管理器工具，这还有很多不足之处。
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       这些问题可能导致：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       运营团队的更多工作
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       不可扩展的工作流程
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       可能的安全风险
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       很明显，Kubernetes 需要一种方法来管理以现代 Kubernetes 集群使用方式发展的软件包，即以更加自动化和整体声明的方式。
      </span>
     </span>
    </p>
    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       什么是 Glasskube？
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 是为 Kubernetes 打造的开源包管理器。它旨在使安装、更新和设置包更容易、更快捷。事实上，它可以比 Helm 或 Kustomize 等工具快 20 倍。
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 从易于使用的包管理器（如 和 ）中汲取了灵感。它提供两种管理包的方法：
       <code>
        Homebrew
       </code>
       <code>
        npm
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        图形用户界面 （GUI）
       </strong>
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
    </p>
    <p class="img-center">
     <a class="article-body-image-wrapper" href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fjq8s43psmq7iirfelktz.png" rel="nofollow">
      <img alt="Glasskube-用户界面" height="442" src="https://img-blog.csdnimg.cn/img_convert/0dc72e7b3e4c093337f2c12801d49fe4.png" width="800"/>
     </a>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        命令行界面 （CLI）
       </strong>
      </span>
     </span>
    </p>
    <div style="margin-left:0; margin-right:0; text-align:start">
     <pre class="has" style="margin-left:0; margin-right:0"><code class="language-highlight language-Go">glasskube serve # to access the Glasskube UI</code></pre>
     <div>
      <div>
      </div>
     </div>
    </div>
    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       当前工具：Helm 和 Kustomize
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Helm 和 Kustomize 是管理
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Kubernetes
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       包的两个主要工具。它们有助于在 Kubernetes 中设置和运行应用程序，但工作方式不同。
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        Helm
       </strong>
       使用模板创建
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Kubernetes yaml
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       文件。它将应用程序打包到 “图表” 中，这些图表是 YAML 文件的集合。它的一些好处是：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       配置复杂应用程序
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       它在应用程序安装方面大放异彩
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       回滚更改非常简单
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       它被广泛采用
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        Kustomize
       </strong>
       采用不同的方法，并且多次与
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Helm
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       结合使用。它允许用户：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       设置基本应用程序配置
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       为不同环境添加更改
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       避免复杂的模板
      </p>
     </li>
    </ul>
    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       现有工具的问题
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       虽然
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Helm
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       和
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Kustomize
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       很有用，但它们可能会给开发人员带来问题。
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        Helm
       </strong>
       可能很棘手：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       由于模板复杂，难以学习
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       针对不同的设置自定义应用程序可能会令人困惑
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       无法更新
       <span style="color:#be191c">
        CRD
       </span>
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       更新很笨拙
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       Helm 本质上是片面的，安装后它的工作就完成了。
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        Kustomize
       </strong>
       有它自己的问题：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       没有内置的方法来管理包
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       用户需要其他工具来共享其设置
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       处理复杂的依赖关系可能很困难
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       在管理应用程序生命周期方面的自动化程度低于 Helm
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       这些问题说明了为什么需要一个更简单的
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       <strong>
        Kubernetes
       </strong>
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       包管理器。这就是
      </span>
     </span>
     <span style="color:#be191c">
      <span style="background-color:#ffffff">
       Glasskube
      </span>
     </span>
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       的用武之地，旨在让整个过程更轻松。
      </span>
     </span>
    </p>
    <p>
    </p>
    <h3 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 解释
      </span>
     </span>
    </h3>
    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 的工作原理
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 使管理 Kubernetes 软件包变得更加容易。它提供了两种使用方法：
      </span>
     </span>
    </p>
    <ol style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       图形界面 （GUI）
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       命令行界面 （CLI）
      </p>
     </li>
    </ol>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 使用一个名为
       <a href="https://glasskube.dev/products/hub/" rel="nofollow" title="Glasskub Hub">
        Glasskub Hub
       </a>
       的中央软件包存储库，它可以：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       了解包依赖项
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       与 GitOps 配合良好
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       自动更新包
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       允许引用多个公共和私有软件包存储库
      </p>
     </li>
    </ul>
    <p style="margin-left:0; margin-right:0; text-align:start">

       可能最重要的是，Glasskube 具有客户端和服务器端组件。服务器端组件会持续跟踪已安装软件包的运行状况和所需状态。服务器端组件包括：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">

        PackageController
       </strong>

       </span>
       通信以部署和协调包。
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       <strong>
        PackageInfoController：
       </strong>
       从配置的后端包存储库中读取包清单，以确定集群包的真实状态。
      </p>
     </li>
    </ul>

    <h4 style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       解决常见问题
      </span>
     </span>
    </h4>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       Glasskube 直接解决了其他 Kubernetes 包管理器存在的差距：
      </span>
     </span>
    </p>
    <ul style="margin-left:0; margin-right:0">
     <li>
      <p style="margin-left:0; margin-right:0">
       它比 Helm 更易于使用，而 Helm 对新用户来说可能很难
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       它比 Kustomize 更好地处理依赖项，
       <span style="color:#be191c">
        Kustomize
       </span>
       没有内置的包管理
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       它与 GitOps 配合得很好，可以更轻松地：
      </p>
      <ul style="margin-left:0; margin-right:0">
       <li>
        回滚更改
       </li>
       <li>
        升级应用
       </li>
       <li>
        团队合作
       </li>
      </ul>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       包范围有助于提高包安装和包共享的效率。例如，如果多个包都依赖于 cert-manager，则一个实例足以为多个包提供服务。
      </p>
     </li>
     <li>
      <p style="margin-left:0; margin-right:0">
       轻松访问带有前端的软件包，而无需手动前移
      </p>
     </li>
    </ul>

    <p style="margin-left:0; margin-right:0; text-align:start">

     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">

      </div>

    <div style="margin-left:0; margin-right:0; text-align:start">

     <div>
      <div>
      </div>
     </div>
    </div>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       然后，此命令将在安装之前请求确认软件包版本。
      </span>
     </span>
    </p>
    <p style="margin-left:0; margin-right:0; text-align:start">
     <span style="color:#171717">
      <span style="background-color:#ffffff">
       <strong>
        要删除包：
       </strong>
      </span>
     </span>
    </p>
    <div style="margin-left:0; margin-right:0; text-align:start">
     <pre class="has" style="margin-left:0; margin-right:0"><code class="language-highlight language-Go">glasskube uninstall</code></pre>
     <div>
      <div>
      </div>
     </div>
