    <h3 style='margin:10px 0px; font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; line-height:40px; color:rgb(51,51,51); font-size:20px'>
     新的挑战
    </h3>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     微服务和传统的单块应用相比，在测试策略上，会有一些不太一样的地方。简单来说，在微服务架构中，测试的层次变得更多，而且对环境的搭建要求更高。比如对单块应用，在一个机器上就可以setup出所有的依赖，但是在微服务场景下，由于依赖的服务往往很多，要搭建一个完整的环境非常困难，这对团队的DevOps的能力也有比较高的要求。
    </p>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     相对于单块来说，微服务架构具有以下特点：
    </p>
    <ul style='padding:0px; margin:0px 0px 10px 25px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     <li style="line-height:20px">
      每个微服务在物理上分属不同进程
     </li>
     <li style="line-height:20px">
      服务间往往通过RESTful来集成
     </li>
     <li style="line-height:20px">
      多语言，多数据库，多运行时
     </li>
     <li style="line-height:20px">
      网络的不可靠特性
     </li>
     <li style="line-height:20px">
      不同的团队和交付周期
     </li>
    </ul>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     上述的这些微服务环境的特点，决定了在微服务场景中进行测试自然会面临的一些挑战：
    </p>
    <ul style='padding:0px; margin:0px 0px 10px 25px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     <li style="line-height:20px">
      服务间依赖关系复杂
     </li>
     <li style="line-height:20px">
      需要为每个不同语言，不同数据库的服务搭建各自的环境
     </li>
     <li style="line-height:20px">
      端到端测试时，环境准备复杂
     </li>
     <li style="line-height:20px">
      网络的不可靠会导致测试套件的不稳定
     </li>
     <li style="line-height:20px">
      团队之间的沟通成本
     </li>
    </ul>
    <h4 style='margin:10px 0px; font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; line-height:20px; color:rgb(51,51,51); font-size:16px'>
     测试的分层
    </h4>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     相比于常见的三层测试金字塔，在微服务场景下，这个层次可以被扩展为5层（如果将UI测试单独抽取出来，可以分为六层）。
    </p>
    <ul style='padding:0px; margin:0px 0px 10px 25px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     <li style="line-height:20px">
      单元测试
     </li>
     <li style="line-height:20px">
      集成测试
     </li>
     <li style="line-height:20px">
      组件测试
     </li>
     <li style="line-height:20px">
      契约测试
     </li>
     <li style="line-height:20px">
      端到端测试
     </li>
    </ul>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>

    </p>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     和测试金字塔的基本原则相同：
    </p>
    <ol style='padding:0px; margin:0px 0px 10px 25px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     <li style="line-height:20px">
      越往上，越接近业务/最终用户；越往下，越接近开发
     </li>
     <li style="line-height:20px">
      越往上，测试用例越少
     </li>
     <li style="line-height:20px">
      越往上，测试成本越高（越耗时，失败时的信息越模糊，越难跟踪）
     </li>
    </ol>
    <h5 style='margin:10px 0px; font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; line-height:20px; color:rgb(51,51,51); font-size:18px'>
     单元测试
    </h5>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     单元测试，即每个微服务内部，对于领域对象，领域逻辑的测试。它的隔离性比较高，无需其他依赖，执行速度较快。
    </p>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     对于业务规则：
    </p>

     集成测试
    </h5>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>

    </p>
    <ul style='padding:0px; margin:0px 0px 10px 25px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     <li style="line-height:20px">
      数据库访问模块与数据库的集成
     </li>
     <li style="line-height:20px">
      对外部service依赖的测试，比如对第三方支付，通知等服务的集成
     </li>

    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     集成测试强调模块和外部的交互的验证，在集成测试时，通常会涉及到外部的组件，比如数据库，第三方服务。这时候需要尽可能真实的去与外部组件进行交互，比如使用和真实环境相同类型的数据库，采用独立模式（Standalone）的WireMock来启动外部依赖的RESTful系统。
    </p>

     前后端分离
    </h6>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>

    </p>

     要确保这些组件组合起来还能如预期的执行，相关测试必不可少。这篇文章详细讨论了前后端分离之后的测试及开发实践。
    </p>
    <h5 style='margin:10px 0px; font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; line-height:20px; color:rgb(51,51,51); font-size:18px'>
     契约测试
    </h5>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>
     在微服务场景中，服务之间会有很多依赖关系。根据消费者驱动契约，我们可以将服务分为消费者端和生产者端，通常消费者自己会定义需要的数据格式以及交互细节，并生成一个契约文件。然后生产者根据自己的契约来实现自己的逻辑，并在持续集成环境中持续验证。
    </p>
    <p style='margin-top:0px; margin-bottom:10px; color:rgb(51,51,51); font-family:"Helvetica Neue",Helvetica,Arial,"WenQuanYi Micro Hei","Microsoft Yahei",SimSun,sans-serif; font-size:14px'>

    </p>
