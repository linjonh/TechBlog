    需求：查询模型butone下的student表，用db2admin用户登录
    <p>
    </p>
    <p>
     连接所需要的jar：
     <span style="color:#FF0000">
      db2jcc.jar
     </span>
     <span style="color:#000000">
      、
      <span style="color:#FF0000">
       db2jcc_license_cu.jar
      </span>
      注意jar包在db2安装路径下：如：D:\Program Files\IBM\SQLLIB\java
     </span>
    </p>
    <p>
     <span style="color:#000000">
     </span>
    </p>
    <p>
     <span style="color:#000000">
      <strong>
       连接代码：
      </strong>
     </span>
    </p>
    <pre><code class="language-java">public static void main(String[] args) {
		try {
			Class.forName("com.ibm.db2.jcc.DB2Driver").newInstance();
			String url = "jdbc:db2://localhost:50000/forum";
			String userName = "DB2ADMIN";
			String passWord = "admin";
			Connection conn = DriverManager.getConnection(url, userName,
					passWord);
			Statement st = conn.createStatement();
			ResultSet set = st.executeQuery("select * from butone.student");
			while(set.next()){
				System.out.println(set.getString(1));
				System.out.println(set.getInt(2));
			}
		} catch (InstantiationException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		} catch (IllegalAccessException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		} catch (ClassNotFoundException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		} catch (SQLException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}</code></pre>
    <p>
     <br/>
    </p>
    <p>
     刚开始接触db2,所以连接这个出现了问题；需求用db2admin登录，那么登录后就会找默认的模型：db2admin
    </p>
    <p>
     但是我想查找butone下的student表，sql这样写是不行的：
    </p>
    <p>
    </p>
    <pre><code class="language-sql">select * from student</code></pre>
    错误信息提示：
    <p>
    </p>
    <p>
    </p>

    <p>
     根据错误信息：
    </p>
    <pre><code class="language-java">DB2ADMIN.STUDENT</code></pre>
    <p>
     那么要改为：
    </p>
    <p>
    </p>
    <pre><code class="language-sql">select * from butone.student</code></pre>
    <p>
    </p>
    <pre></pre>
    <p>
    </p>

     这样子有比较麻烦，推荐两种解决方案：
