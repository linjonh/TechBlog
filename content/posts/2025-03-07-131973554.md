    <h2>
     <a id="_2">
     </a>
     前言
    </h2>
    <p>
     在通过OTA的方式对设备进行升级，若在使用内部FLASH装载固件程序的方式下，需要擦写 内部FLASH 从而实现把新的固件程序写入到 内部FLASH。
    </p>
    <p>
     <a href="https://www.st.com/resource/en/programming_manual/cd00283419-stm32f10xxx-flash-memory-microcontrollers-stmicroelectronics.pdf" rel="nofollow">
      参考文档：ST的闪存编程文档
     </a>
    </p>
    <h2>
     <a id="STM32F105ROM_8">
     </a>
     STM32F105的内部ROM分布
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/47261e2fe36be7f6b431326561b235b1.png">
      <br/>
      F105属于互联型设备，其内部的主存储器ROM有256KB。
      <br/>
      这256KB 由 128页 ，每页2KB的页面组成
     </img>
    </p>
    <p>
     Information block 分为两个部分：System memory 和 Option bytes
    </p>
    <ul>
     <li>
      System memory：
      <br/>
      这个部分存储了是STM32官方系统的bootloader程序，在通过选择对应的BOOT模式可以进入该bootloader程序；该bootloader程序是通过 把USART1的传输过来的程序 写入到Main memory中。
      <br/>
      在互联型产品中，bootloader程序 还可以读取 USART2(重映射)等接口，从而烧录程序。
     </li>
     <li>
      Option bytes
      <br/>
      有两种保护方式：①写包含；②读保护
     </li>
    </ul>
    <p>
     注意：
     <br/>
     1.
     <strong>
      只有在写FLASH完毕后，才能正确的读取FLASH
     </strong>
     。在 写FLASH时读取FLASH，将会造成写失败；
     <br/>
     2.只有在内部高速晶振开启时，才能写入/擦除内部FLASH。
     <br/>
     3.在低功耗模式下，对FLASH的任何访问将被中止。
    </p>
    <h2>
     <a id="STM32F10x_32">
     </a>
     STM32F10x的闪存擦写
    </h2>
    <p>
     STM32F10x的闪存擦写由 FPEC 控制。
    </p>
    <p>
     FPEC指Flash program and erase controller，FPEC由7个32位的寄存器组成。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ed9fa6018d5195714db888a0f534e919.png">
      <br/>
      注意：只有CPU没有访问 正在操作的FLASH，CPU将不会因操作FLASH而阻塞。
     </img>
    </p>
    <h3>
     <a id="FPEC_39">
     </a>
     解锁FPEC
    </h3>
    <p>
     在复位后，FPEC是被保护着。
    </p>
    <p>
     需要往 FLASH_KEYR 寄存器写入固定的值，才可解开FPEC的锁。
    </p>
    <p>
     如果这个值写错了，那么只有在下一次复位后，才能再次接口FPEC的锁。
    </p>
    <h3>
     <a id="Main_FLASH__46">
     </a>
     Main FLASH 的编写
    </h3>
    <p>
     每次只能往main flash写入16位数据。
    </p>
    <p>
     在写入main flash的指定地址时，若该地址并未被擦除，则FLASH_SR的 PGERR 将会被置位。
     <br/>
     若该地址是写保护的(通过 FLASH_WRPR 设置)，则FLASH_SR的 WRPRTERR 将会被置位。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0930138851d42cc6f9caa7b0eba8ed0c.png"/>
    </p>

     标志的擦写 main Flash的步骤如下：

    <h2>
     <a id="main_Flash_56">
     </a>
     main Flash的擦除
    </h2>
    <p>
     在编写 main Flash 之前，需要先擦除。
    </p>
    <p>
     擦除 main Flash 的方式有两种：页擦除，全擦除。
    </p>
    <hr/>
    <p>
     页擦除：页擦除是擦除 main Flash的一个页面。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0262c3fb95f03bf436d5a85dd6baf78f.png"/>
    </p>

</code></pre>
    <h2>
     <a id="_70">
     </a>
     注意点
    </h2>
    <p>
     在使用stm32自带的flash保存数据时候，如下特点必须知道：
    </p>
    <p>
     1、必须是先擦除一个扇区，才能写入
    </p>
    <p>
     2、读数据没有限制
    </p>
    <p>
     3、
     <strong>
      写数据必须是偶数（2字节、4字节）
     </strong>
     ，同时写入地址以一定要考虑字节对齐，
    </p>
    <p>
     4、一般都是在最后几页进行数据保存的，确保数据量不超过flash的的大小，比如f103大容量是2k字节，其实一个扇区只能写入1k数量的2字节的数据。
    </p>
    <pre><code class="prism language-c">http<span class="token operator">:</span><span class="token comment">//eeskill.com/article/id/29479</span>
</code></pre>
    <p>
     5、写入数据时，采用大端模式，如想往 0x0800 0000写入
     <code>
      0x31 0x32 0x33 0x34
     </code>
     ，则写入的数据是
     <code>
      uint32_t data = 0x31 + 0x32&lt;&lt;8 + 0x33&lt;&lt;16 + 0x34&lt;&lt;24;
     </code>
     即写入
     <code>
      0x34333231
     </code>
    </p>
    <p>
     6、一旦写入数据，无论你写的是FLASH擦除后的默认值
     <code>
      0xff
     </code>
     还是其他值，若在不擦除的情况下再次往该地址写数据，必定失败。
    </p>
    <p>
     7、若遇到写入 或 擦除失败的清空，需要清除标志位后才能再继续写入或擦除。
    </p>
