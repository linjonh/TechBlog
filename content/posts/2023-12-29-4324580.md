    <p>
     引言：目前的WEB应用，很多请求和返回都可以通过工具查看，方便我们深入的进行测试工作。
    </p>
    <p>
     方法1：firefox下的插件：Web Developer
     <br/>
     Web Developer，非常强大（推荐使用，页面设计调试利器），查看Header只是功能之一。
     <br/>
     操作方法：
     <br/>
     1）打开WEB DEVELOPER
     <br/>
     2)   Information-&gt;View Response Headers.
     <br/>
     返回结果如下：
    </p>
    <p>
     <img alt="p1" class="alignnone size-full wp-image-2526" height="249" src="http://rdc.taobao.com/blog/qa/wp-content/uploads/2009/07/p1.png" width="485"/>
    </p>
    <p>
     方法2：firefox下的插件：httpfox
     <br/>
     操作方法：
     <br/>
     1）打开httpfox
     <br/>
     2 ）启动start按钮
     <br/>
     3） 打开你的页面
     <br/>
     4）查看type=text/html  的你要查看的url地址。并选中它
     <br/>
     5）选中”headers”标签 ，查看”request header”或”response header”。我这里关注的的返回结果。
     <br/>
     查看at_ssid，确认返回结果正确。
     <br/>
     界面如下：
    </p>
    <p>
     <img alt="p21" class="alignnone size-full wp-image-2528" height="390" src="http://rdc.taobao.com/blog/qa/wp-content/uploads/2009/07/p21.png" width="1101">
      <br/>
      方法3：firefox下的firebug工具
      <br/>
      操作方法
      <br/>
      1.打开firebug
      <br/>
      2.选择网络
      <br/>
      3.选择html（如果网络下没有打开网络面板）不会展示出html标签。可以选中这三项，启动”为XXX启动选中的面板”
      <br/>
      4.刷新要查看的页面.
      <br/>
      5.查看get请求中的headers信息，里面就有我们要用的信息了。
     </img>
    </p>
