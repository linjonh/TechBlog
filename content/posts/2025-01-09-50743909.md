# 对腾讯云qcloud音视频通信SDK的调研(优缺点)

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 腾讯云音视频SDK的调研报告之优缺点 | | | |  |
|  | 缺点 |  | 优点 |  |
|  |  |  |  |  |
|  | * 2015/05/20腾讯云正式对外开放QQ音视频，此项服务目前推出时间算是比较短，使用其SDK且有所名气的App目前市场上还几乎没有看到。 |  | * 通过腾讯云通信解决方案，用户可高效低成本的受QQ同等优质的聊天能力； |  |
|  |  |  |  |
|  |  | * 互动直播将服务音视频社交通过SDK可集成跨平台，进行高质量的音视频通信；   见附录[1][2] |  |
|  |  |  |  |
|  | * SDK的Demo代码工程文件不全，将Eclipse下的项目转出为Gradle形式，存在项目jar包不全需要自行上网自己下载的情况，不知道为啥。 |  |  |  |
|  |  | * 稳定可靠的账号登录集成。   见附录一[3][4][5] |  |
|  |  |  |  |
|  |  |  | * 拥有完备的资料关系链体系，增量更新用户数据。   见附录一[6][7] |  |
|  | * 移动端的SDK里所给的Demo版本比较旧而且是Eclipse版的，开发者希望能直接体验接入Android Studio版的Demo，并且官网SDK应该及时更新Demo。 |  |  |  |
|  |  | * 拥有成熟的垂直社交的视频解决方案。   <http://www.qcloud.com/product/ilvb.html#wiki> |  |
|  |  |  |
|  |  |  |  |  |
|  | * 对微信api的调用只写了一半。 |  | * 技术论坛内容比较丰富。   <http://bbs.qcloud.com/forum.php> |  |
|  |  |  |  |  |
|  |  |  |  |  |
|  |  |  |  |  |
| **附录一** | | | |  |
| 互动直播 | [1]. 腾讯云互动直播服务具备400ms超低延时，可多人音视频互动，服务覆盖全球的特点，支持单聊，群聊的音视频社交场景。 | | |  |
| [2]. 只需集成SDK，就可实现音视频的采集、编解码、传输、渲染播放等工作，业界领先的云端流控策略，卓越的音视频技术，将保障您的服务在Android、   iOS、Windows、Web主流平台及不同终端机型上都有着优异的音视频通信效果。 | | |  |
|  |
| 账号登录集  成 | [3]. 丰富的账号体系支持:支持手机号注册登录；支持字符串注册登录；支持游客/匿名登录；支持QQ/微信/新浪微博等第三方账号；支持独立模式和托管模式两种集成方式。 | | |  |
|  |
| [4]. 快捷的账号登录集成:开发者只需调用少量接口，即可快速实现注册/登录能力，无需关注细节，1天内完成账号登录集成。 | | |  |
| [5]. 安全的账号保护:腾讯账号登录体系采用高级别的安全加密逻辑，有效提升APP的账号安全能力。开发者不用考虑密码存储，网络传输加密，拖库等问题。 | | |  |
| 用户资料和  关系链管理 | [6]. 使用了腾讯QQ同样完备的资料关系链服务体系，包括：关系链管理、好友推荐、黑名单管理、最近联系人、资料管理，并且提供自定义的资料关系链扩展能力。 | | |  |
| [7]. 支持用户资料和关系链增量更新，最大限度节省流量。 | | |  |
