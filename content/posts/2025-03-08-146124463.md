    <h2>
     一、什么是动态规划：
    </h2>
    <p>
     动态规划（Dynamic Programming，简称 DP）是一种用于求解最优化问题的方法，它通过将问题分解为子问题并逐步求解这些子问题来优化问题的解决过程。动态规划适用于那些可以通过将问题分解为重叠子问题来解决的问题。
    </p>
    <h5>
     动态规划的核心思想：
    </h5>
    <ol>
     <li>
      <strong>
       最优子结构
      </strong>
      ：问题的最优解可以通过子问题的最优解来构建。
     </li>
     <li>
      <strong>
       重叠子问题
      </strong>
      ：子问题会被重复计算，因此通过记忆化（存储子问题的答案）来避免重复计算，从而节省时间。
     </li>
    </ol>
    <p>
     动态规划的目标是通过将大问题转化为小问题的求解，最终得到最优解。
    </p>
    <h6>
     <strong>
      动态规划的做题步骤
     </strong>
     ：
    </h6>
    <ol>
     <li>
      <strong>
       定义状态
      </strong>
      ：将问题转化为状态表示，通常是通过一个数组或二维数组来表示问题的各个状态。
     </li>
     <li>
      <strong>
       状态转移方程
      </strong>
      ：根据问题的约束条件，推导出从一个状态到另一个状态的转移规律。
     </li>
     <li>
      <strong>
       边界条件
      </strong>
      ：确定递归的终止条件或基础情况。
     </li>
     <li>
      <strong>
       优化
      </strong>
      ：如果有冗余的计算，可以通过空间优化或减少状态存储的维度等手段进一步优化。
     </li>
    </ol>
    <p>
     常见的模板就有：
    </p>
    <pre><code class="language-java">public int dpSolution(int n) {
    int[] dp = new int[n + 1];
    
    // 初始化边界条件
    dp[0] = 0;  // 或者其他适当的值
    dp[1] = 1;  // 或者其他适当的值
    
    // 状态转移方程
    for (int i = 2; i &lt;= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];  // 这是一个示例，根据具体问题的状态转移方程修改
    }
    
    return dp[n];  // 返回最终的结果
}
</code></pre>

    <h3>
     <a id="2_41">
     </a>
     二、核心思想：
    </h3>
    <p>
     这道题其实真的挺有难度的在我看来，因为这道题如果暴力去写的话，求最大我们要枚举出很多很多种情况，这显然是不现实的，可能会用递归来一直重复此过程，而这道题选择动态规划显然是必要的：
    </p>
    <p>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        最优子结构
       </strong>
       <br/>
       要最大化"ovo"的数量，每个位置的选择（是否消耗修改次数形成ovo）会影响后续决策。通过将全局问题拆解为“是否在第i位结束一个ovo”的子问题，可以利用子问题的最优解组合得到全局最优解。
      </p>
     </li>
     <li>
      <p>
       ​
       <strong>
        重叠子问题
       </strong>
       <br/>
       不同位置的决策可能涉及相同的修改次数和字符检查（例如连续多个ovo可能共享部分计算），动态规划通过存储中间状态避免重复计算，提升效率。
      </p>
     </li>
     <li>
      <p>
       ​
       <strong>
        资源分配
       </strong>
       <br/>
       修改次数k是有限资源，动态规划的二维状态
       <code>
        dp[i][j]
       </code>
       能精确追踪：处理到第i个字符时，消耗j次修改能达到的最大ovo数，从而高效分配修改资源。
      </p>
     </li>
     <li>
      <p>
       ​
       <strong>
        无后效性
       </strong>
       <br/>
       一旦确定i位置的状态（如使用j次修改），后续决策仅依赖当前状态，不会影响之前的状态，符合动态规划的应用条件。
      </p>
     </li>
    </ol>
    <h3>
     三、本题的具体实现：
    </h3>
    <p>
     首先我们肯定要定义出来状态
    </p>
    <p>
     <strong>
      定义状态
     </strong>
     <br/>
     定义
     <code>
      dp[i][j]
     </code>
     表示处理到字符串第
     <code>
      i
     </code>
     个字符时，使用
     <code>
      j
     </code>
     次修改后能获得的最多不相交 "ovo" 子串数量。注意i要等于3（
    </p>
    <ul>
     <li>
      <span style="color:#fe2c24">
       "ovo" 是三个连续字符组成的子串，因此至少需要处理到第3个字符才能形成第一个可能的子串。
      </span>
     </li>
     <li>
      <span style="color:#fe2c24">
       例如字符串索引为
       <code>
        1-3
       </code>
       、
       <code>
        2-4
       </code>
       等，必须从i=3开始检查。
      </span>
     </li>
    </ul>
    <p>
     ）
    </p>
    <p>
     接着进行状态的更新：
    </p>
    <p>
     将
     <code>
      dp[i][j]
     </code>
     初始化为
     <code>
      dp[i-1][j]
     </code>
     ，表示不选择以
     <code>
      i
     </code>
     结尾的子串时的最优解。
    </p>
    <pre><code class="language-java">for (int j = 0; j &lt;= k; j++) {
    dp[i][j] = dp[i-1][j];
}</code></pre>
    <p>
     <strong>
      尝试更新状态
     </strong>
     <br/>
     检查能否以
     <code>
      i
     </code>
     结尾形成一个 "ovo" 子串：
    </p>
    <ul>
     <li>
      ​
      <strong>
       计算代价
       <code>
        cost
       </code>
      </strong>
      ：将
      <code>
       i-2, i-1, i
      </code>
      修改为 "ovo" 所需的操作次数。
     </li>
     <li>
      ​
      <strong>
       状态转移
      </strong>
      ：若当前剩余操作次数
      <code>
       j ≥ cost
      </code>
      ，则比较继承值和选择当前子串后的值，取最大。
     </li>
    </ul>
    <p>
     仅当
     <code>
      j ≥ cost
     </code>
     时，允许转移
    </p>
    <pre><code class="language-java">for (int j = cost; j &lt;= k; j++) {
    dp[i][j] = Math.max(dp[i][j], dp[i-3][j-cost] + 1);
}</code></pre>
    <p>
     计算修改代价：
    </p>
    <pre><code class="language-java">int cost = 0;
if (str.charAt(i-2) != 'o') cost++; // 第一个字符需是o
if (str.charAt(i-1) != 'v') cost++; // 第二个字符需是v
if (str.charAt(i) != 'o') cost++;   // 第三个字符需是o</code></pre>
    <p>
     不断更新最大值：
    </p>
    <pre><code class="language-java">ans = Math.max(ans, dp[i][j]); // 记录所有可能的最大值</code></pre>
    <p>
    </p>
    <h2>
     四、完整代码实现：
    </h2>
    <pre><code class="language-java">import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();
        while (T-- &gt; 0) {
            int n = sc.nextInt();
            int k = sc.nextInt();
            String str = " " + sc.next(); // 转换为1-based索引

            int[][] dp = new int[n+1][k+1];
            int ans = 0;

            for (int i = 3; i &lt;= n; i++) {
                // 继承i-1位置的状态
                for (int j = 0; j &lt;= k; j++) {
                    dp[i][j] = dp[i-1][j];
                }

                // 计算将i-2,i-1,i三个字符变为vov的代价
                int cost = 0;
                if (str.charAt(i-2) != 'o') cost++;
                if (str.charAt(i-1) != 'v') cost++;
                if (str.charAt(i) != 'o') cost++;

                // 状态转移：消耗cost次操作，数量+1
                for (int j = cost; j &lt;= k; j++) {
                    dp[i][j] = Math.max(dp[i][j], dp[i-3][j-cost] + 1);
                    ans = Math.max(ans, dp[i][j]);
                }
            }
            System.out.println(ans);
        }
    }
}</code></pre>
    <h5>
     要点：
    </h5>
    <ul>
     <li>
      <strong>
       状态设计
      </strong>
      ：二维数组跟踪位置和修改次数。
     </li>
     <li>
      ​
      <strong>
       转移逻辑
      </strong>
      ：分不选/选当前子串两种情况，确保不重叠。
     </li>
     <li>
      ​
      <strong>
       边界处理
      </strong>
      ：从
      <code>
       i=3
      </code>
      开始，避免越界。
     </li>
     <li>
      ​
      <strong>
       复杂度
      </strong>
      ：时间复杂度
      <code>
       O(nk)
      </code>
      ，空间复杂度
      <code>
       O(nk)
      </code>
     </li>
    </ul>
