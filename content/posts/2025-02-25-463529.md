    <h1 class="title-article" id="articleContentId">
     微软赞助 LinuxWorld 分析称可能入侵 Linux 业
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    中国开源软件推进联盟主席助理宋可为一大早就来到中国 LinuxWorld 会场。当看到微软公司的标志出现在顶级赞助商之列时，他和他的朋友们都吃了一惊。
    <br/>
    <br/>
    “在前五届大会中，微软都不是赞助商。这一次微软突然成为最大赞助商，我们觉得太突然了。”第二天接受《财经时报》采访时，宋可为依然感到有一些不可思议。
    <br/>
    <br/>
    同样感到大惑不解还有许多自发前往的Linux爱好者。“无论是 Linux 社区、企业，还是像我们这样的个人，潜意识里都对微软有种敌对情绪。主办方允许微软赞助这个大会，我们感到很不理解。”面对记者，来自清华大学的一名学生则有些忿忿不满。
    <br/>
    <br/>
    作为 Linux 阵营最大的竞争对手，微软公司过去几年一直试图将包括 Linux 在内的开源应用扼杀在摇篮之中，并为此不惜代价在全球大做宣传。而现在，微软对于 Linux 的姿态多少有些耐人寻味。
    <br/>
    <br/>
    “这并不奇怪，微软显然接受了 Linux 崛起的事实，它在转向支持 Linux 产业。但我有一个更可怕的直觉，微软很有可能侵入 Linux 产业。”计世资讯总经理曲晓东向《财经时报》透露了他的猜想。
    <br/>
    <br/>
    <br/>
    Linux 变“硬”，微软变“软”
    <br/>
    <br/>
    影响微软态度的，是 Linux 技术的成熟以及市场的走热。
    <br/>
    <br/>
    今年 8 初，中国人民银行网络处的小张和他的同事们长长松了口气。在年初，他们首次采用 Linux 系统为人行总部以及全国 9 大分行承担某项关键应用。经过最近几个月的使用和观察后，他们发现 Linux 系统的运行表现相当令人满意。
    <br/>
    <br/>
    这只是最近掀起的“Linux 风暴”的“冰山一角”。从今年起，包括交通银行、中国工商银行、中国农业银行、中国建设银行、中国银行等在内的其他五大银行都在尝试把部分应用向 Linux 平台迁移——在此之前，银行后台几乎清一色都是 Unix 系统，而现在，基于 Linux 的应用已经越来越多。
    <br/>
    <br/>
    不仅是在金融业，在电信、政府、零售业等其他行业，Linux 应用也获得极大的发展。受此影响，众多 Linux 企业纷纷宣布盈利状况好转。而据 IDC 最近预测显示，未来 Linux 市场将以每年 25% 的速度增长，2008 年其市场将达到 350 亿美元。
    <br/>
    <br/>
    Linux 的崛起，对于微软而言无疑是一个尴尬的事实。“尽管 Windows 垄断了桌面操作系统，但在服务器以及Web应用领域，Linux 系统的优势地位要比 Windows 明显。”一位业界人士分析说。他援引的一份国外有关报告显示，去年第 4 季度服务器市场上，Linux 销售增长率高达 35.6%，远远高于 Unix 2.7% 以及 Windows 15.5% 的增长率。
    <br/>
    <br/>
    在这种局面下，微软继续站在 Linux 对立面显然是个不明智的做法，调整策略成为必然之举。
    <br/>
    <br/>
    今年 3 月，微软首席执行官史蒂夫·鲍尔默主动与红帽首席执行官马修·祖尼克联系，两人在纽约进行了长久密谈。鲍尔默一直将 Linux 以及开源视为“毒瘤”，欲灭之而后快。但现在，鲍尔默的口辞却发生了巨大变化：“与我们竞争的是产品，因此我们不会抵制潮流。”
    <br/>
    <br/>
    最近，微软还在美国成立了一个 Linux 与开源软件实验室。联系起微软去年尝试将 Linux 整合到 Windows，并发布了部分源代码的消息，我们不难看出，微软对 Linux 的态度已经趋向好转。
    <br/>
    <br/>
    微软中国公司平台策略总监潘迪对此认为：“商业软件和开源软件各有优势，不同的发展模式能够、也应该在健康的竞争氛围中并存。” 这种观点，显然与以往有所不同。
    <br/>
    <br/>
    “微软改变与 Linux 对立的策略，这有助于保障微软自身的利益。一旦 Linux 真正成了气候，微软也能尽量减少损失。” 曲晓东这样认为。
    <br/>
    <br/>
    微软密谋鲸吞 Linux？
    <br/>
    <br/>
    但可以肯定的是，微软对于 Linux 的态度绝非“认同”这么简单。
    <br/>
    <br/>
    曾有这样一个故事在坊间流传：某国政府去年决定放弃 Windows，全面采购 Linux 系统，结果在一些重大项目开发时却遭遇难题。无可奈何之下，该国政府还是借助了微软的研发力量，才最终完成 Linux 项目的开发。
    <br/>
    <br/>
    这个故事如果属实，那么对于众多 Linux 厂商而言无疑是个讽刺：Linux 最大的对头，却在 Linux 方面拥有第一流的研发实力。
    <br/>
    <br/>
    实际上，微软尽管一直反对 Linux，但在招揽 Linux 及开源人才方面却不惜重金。近年来，大量开源社区中的知名人士都悄然转到微软麾下，如 Gentoo 创始人 Daniel Robbins 和曾在整个 python 和 Java 社区引起巨大轰动的 Jim Hugunin 等人。
    <br/>
    <br/>
    “默认敌人成长肯定不是微软的性格，微软最擅长的做法是用类似产品整垮敌人，就像当年微软用 IE 整垮网景那样。”某熟悉微软的业内人士声称，他认为，微软利用 Linux 消灭其他 Linux 企业“并非没有可能”。
    <br/>
    <br/>
    曲晓东则认为，凭借微软在开发操作系统和应用软件的实力，它完全可以轻松进入 Linux 产业，并一举盖过所有 Linux 企业的风头，成为全球最大的 Linux 巨头。“当然，这只是我个人的想法，至少微软目前还没有说这样的话。”
    <br/>
    <br/>
    对于这种猜想，倪光南表示异议：“微软如果真的加入了，这对 Linux 和开源倒是件好事，至少能起到很大的推动作用。但是，微软绝对不会这么做。”他的理由是，开源软件企业主要依靠服务赢得利润，自身软件价格并不贵。微软靠卖应用软件就远比 Linux 企业活得滋润。“微软怎么会放弃高利润，而转向低利润呢？”他在电话里反问记者。
    <br/>
    <br/>
    尽管目前微软与 Linux 相处融洽，但对于未来双方关系将如何发展，专家们并没有一致看法。“无论如何，微软对于 Linux 的野心还是值得警惕的。”某位人士这样评论说。
