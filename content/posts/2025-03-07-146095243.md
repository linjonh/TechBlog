    <h1 class="title-article" id="articleContentId">
     git如何解除远程仓库 改变远程仓库地址
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     这三个命令通常一起使用来更改项目的远程仓库地址：
    </p>
    <ol>
     <li>
      <strong>
       查看当前远程仓库配置
      </strong>
      ：
      <code>
       git remote -v
      </code>
     </li>
     <li>
      <strong>
       移除旧的远程仓库配置
      </strong>
      ：
      <code>
       git remote remove origin
      </code>
     </li>
     <li>
      <strong>
       添加新的远程仓库配置
      </strong>
      ：
      <code>
       git remote add origin 新的远程仓库地址
      </code>
      <hr/>
      <p>
      </p>
     </li>
    </ol>
    <p>
     要解除当前项目的远程仓库关联，你可以按照以下步骤操作：
    </p>
    <ul>
     <li>
      <h4>
       1.
       <code>
        git remote -v
       </code>
      </h4>
      <ul>
       <li>
        <p>
         <strong>
          功能
         </strong>
         ：显示当前项目中所有远程仓库的配置信息。
        </p>
       </li>
       <li>
        <p>
         <strong>
          输出内容
         </strong>
         ：
        </p>
        <ul>
         <li>
          远程仓库的名称（例如
          <code>
           origin
          </code>
          ）。
         </li>
         <li>
          远程仓库的 URL（用于 fetch 和 push 操作）。
         </li>
        </ul>
        <p>
         示例输出：
        </p>
        <pre><code class="hljs">origin  git@gitee.com:xiaoliu_l/game-lianliankan.git (fetch)
origin  git@gitee.com:xiaoliu_l/game-lianliankan.git (push)
//前面的 origin 就是仓库名字，也就是说仓库名字就是 origin</code></pre>
        <p>
         这个命令可以帮助你确认当前项目的远程仓库地址是否正确，或者查看是否有多个远程仓库。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <code>
      <font face="-apple-system, SF UI Text, Arial, PingFang SC, Hiragino Sans GB, Microsoft YaHei, WenQuanYi Micro Hei, sans-serif, SimHei, SimSun">
       2.
      </font>
      git remote remove origin
     </code>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        功能
       </strong>
       ：移除名为
       <code>
        origin
       </code>
       的远程仓库配置。
      </p>
     </li>
     <li>
      <p>
       <strong>
        作用
       </strong>
       ：
      </p>
      <ul>
       <li>
        执行此命令后，Git 会删除与
        <code>
         origin
        </code>
        相关的所有配置信息。
       </li>
       <li>
        如果你想更换远程仓库或完全移除远程仓库关联，这个命令非常有用。
       </li>
      </ul>
      <p>
       示例：
      </p>
      <pre><code class="hljs">git remote remove origin</code></pre>
      <p>
       执行后，再次运行
       <code>
        git remote -v
       </code>
       将不会看到
       <code>
        origin
       </code>
       的相关信息。
      </p>
     </li>
    </ul>
    <p>
    </p>
    <h4>
     3.
     <code>
      git remote add origin 新的远程仓库地址
     </code>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        功能
       </strong>
       ：添加一个新的远程仓库，并将其命名为
       <code>
        origin
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        参数
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         origin
        </code>
        ：这是远程仓库的名称，通常默认为
        <code>
         origin
        </code>
        ，但你可以使用其他名称。
       </li>
       <li>
        <code>
         新的远程仓库地址
        </code>
        ：这是你要关联的新远程仓库的 URL。
       </li>
      </ul>
      <p>
       示例：
      </p>
      <pre><code class="hljs">git remote add origin git@*********/new-repo.git
// git@*********/new-repo.git 是仓库远程链接</code></pre>
      <p>
       <img alt="" height="573" src="https://i-blog.csdnimg.cn/direct/5df902012761464ab15bca1c8edc95dc.png" width="585"/>
      </p>
     </li>
    </ul>
    <p>
     也就是仓库的这个链接 ，这条命令会将新的远程仓库地址添加到本地项目的配置中，并将其命名为
     <code>
      origin
     </code>
     。之后，你可以通过
     <code>
      git push origin main
     </code>
     等命令将代码推送到新的远程仓库。
    </p>
    <p>
    </p>
