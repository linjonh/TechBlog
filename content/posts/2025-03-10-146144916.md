     <code>
      npm run serve
     </code>

     <code>
      EADDRINUSE
     </code>
     ？😱
    </p>
    <p>

    </p>

     </a>

    <h4>
     <a id="_49">
     </a>
     错误信息解析：
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        EADDRINUSE
       </code>
      </strong>
      ：这是一个常见的网络错误，表示“地址已被占用”。
     </li>
     <li>
      <strong>
       <code>
        0.0.0.0:8080
       </code>
      </strong>
      ：这是 Vue 开发服务器默认使用的端口号。错误表明 8080 端口已经被其他进程占用了。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_55">
     </a>
     二、为什么会出现这个问题？
    </h3>
    <p>
     在开发过程中，端口占用问题通常有以下几种原因：
    </p>
    <ol>
     <li>
      <strong>
       其他 Vue 项目正在运行
      </strong>
      ：你可能同时打开了多个 Vue 项目，而它们都试图使用 8080 端口。
     </li>
     <li>
      <strong>
       其他服务占用了端口
      </strong>
      ：比如本地运行的 Java 服务、Node.js 服务或其他应用程序。
     </li>
     <li>
      <strong>
       上次关闭项目时端口未释放
      </strong>
      ：有时系统可能没有完全释放端口，导致再次启动时冲突。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_64">
     </a>
     三、解决方案：三步搞定端口占用
    </h3>
    <p>
     接下来，我们分三步来解决这个问题。无论你是 Windows、Mac 还是 Linux 用户，都能找到适合自己的方法！
    </p>
    <h4>
     <a id="_1_68">
     </a>
     方法 1：找到并终止占用端口的进程
    </h4>
    <h5>
     <a id="Windows__70">
     </a>
     Windows 用户：
    </h5>
    <ol>
     <li>
      <p>
       打开命令提示符（CMD）。
      </p>
     </li>
     <li>
      <p>
       输入以下命令，查找占用 8080 端口的进程 ID（PID）：
      </p>
      <pre><code class="prism language-bash"><span class="token function">netstat</span> <span class="token parameter variable">-ano</span> <span class="token operator">|</span> findstr :8080
</code></pre>
      <p>
       你会看到类似以下的输出：
      </p>
      <pre><code class="prism language-bash">TCP    <span class="token number">0.0</span>.0.0:8080            <span class="token number">0.0</span>.0.0:0              LISTENING       <span class="token number">1234</span>
</code></pre>
      <p>
       其中
       <code>
        1234
       </code>
       就是占用端口的进程 ID。
      </p>
     </li>
     <li>
      <p>
       使用以下命令终止该进程：
      </p>
      <pre><code class="prism language-bash">taskkill /PID <span class="token number">1234</span> /F
</code></pre>
      <p>
       <strong>
        注意
       </strong>
       ：
       <code>
        /F
       </code>
       表示强制终止进程。
      </p>
     </li>
    </ol>
    <h5>
     <a id="MacLinux__94">
     </a>
     Mac/Linux 用户：
    </h5>
    <ol>
     <li>
      <p>
       打开终端。
      </p>
     </li>
     <li>
      <p>
       输入以下命令，查找占用 8080 端口的进程：
      </p>
      <pre><code class="prism language-bash"><span class="token function">lsof</span> <span class="token parameter variable">-i</span> :8080
</code></pre>
      <p>
       你会看到类似以下的输出：
      </p>
      <pre><code class="prism language-bash">COMMAND   PID <span class="token environment constant">USER</span>   FD   TYPE DEVICE SIZE/OFF NODE NAME
<span class="token function">node</span>     <span class="token number">1234</span> user   20u  IPv4 <span class="token number">123456</span>      0t0  TCP *:http-alt <span class="token punctuation">(</span>LISTEN<span class="token punctuation">)</span>
</code></pre>
      <p>
       其中
       <code>
        1234
       </code>
       就是占用端口的进程 ID。
      </p>
     </li>
     <li>
      <p>
       使用以下命令终止该进程：
      </p>
      <pre><code class="prism language-bash"><span class="token function">kill</span> <span class="token parameter variable">-9</span> <span class="token number">1234</span>
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_2_Vue__119">
     </a>
     方法 2：更改 Vue 项目的端口号
    </h4>
    <p>
     如果你不想终止其他进程，或者你希望保留当前运行的服务，可以通过修改 Vue 项目的配置文件来更改端口号。
    </p>
    <h5>
     <a id="_123">
     </a>
     操作步骤：
    </h5>
    <ol>
     <li>
      <p>
       找到项目根目录下的
       <code>
        vue.config.js
       </code>
       文件。如果该文件不存在，可以在根目录创建一个。
      </p>
     </li>
     <li>
      <p>
       在
       <code>
        vue.config.js
       </code>
       文件中添加或修改如下内容：
      </p>
      <pre><code class="prism language-javascript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token literal-property property">devServer</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
    <span class="token literal-property property">port</span><span class="token operator">:</span> <span class="token number">3000</span>  <span class="token comment">// 将 8080 更改为你希望使用的端口号</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
      <p>
       这里我们将端口号改为 3000，你可以根据需要选择其他端口。
      </p>
     </li>
     <li>
      <p>
       保存文件并重新启动项目：
      </p>
      <pre><code class="prism language-bash"><span class="token function">npm</span> run serve
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_3_npm__145">
     </a>
     方法 3：清除 npm 缓存
    </h4>
    <p>
     有时，npm 缓存中的一些问题也可能导致开发服务器无法启动。你可以尝试清除 npm 缓存并重新安装依赖：
    </p>
    <ol>
     <li>
      <p>
       清除 npm 缓存：
      </p>
      <pre><code class="prism language-bash"><span class="token function">npm</span> cache clean <span class="token parameter variable">--force</span>
</code></pre>
     </li>
     <li>
      <p>
       重新安装依赖：
      </p>
      <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span>
</code></pre>
     </li>
     <li>
      <p>
       再次尝试启动项目：
      </p>
      <pre><code class="prism language-bash"><span class="token function">npm</span> run serve
</code></pre>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_169">
     </a>
     四、重启项目大法
    </h3>
    <p>
     重启项目，项目可能会重新选择一个端口，就不会有端口占用问题。
    </p>
    <p>
     无论你选择哪种方法，都能让你从“端口占用”的泥潭中解脱出来，继续愉快地开发 Vue 项目！🎉
    </p>
    <hr/>
    <p>
     <strong>
      小贴士：
     </strong>
    </p>
    <ul>
     <li>
      如果你经常遇到端口冲突问题，可以养成习惯，每次启动项目前检查端口占用情况。
     </li>
     <li>
      使用
      <code>
       vue.config.js
      </code>
      自定义端口号时，记得选择一个不常用的端口，避免与其他服务冲突。
     </li>
    </ul>
