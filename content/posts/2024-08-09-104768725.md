    <h4>
     <a id="_0">
     </a>
     数据通信系统的三大部分
    </h4>
    <p>
     1.源系统
     <br/>
     2.传输系统
     <br/>
     3.目的系统
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d50d2f78668fb94bfc5197f3e10f53ee.png">
      <br/>
      下面对三大部分进行分析。
     </img>
    </p>
    <p>
    </p>
    <h4>
     <a id="_8">
     </a>
     源系统
    </h4>
    <p>
     源系统即为发送端、发送方。
     <br/>
     源系统包括两个部分：
     <br/>
     1.源点（源站、信源）
     <br/>
     2.发送器
    </p>
    <p>
     源点用来干嘛呢？
     <br/>
     源点设备产生要传输的数据。
    </p>
    <p>
     那发送器呢？
     <br/>
     源点生成的数字比特流要通过发送器
     <strong>
      编译
     </strong>
     之后才能在传输系统中进行传输。典型的发送器：调制器。
    </p>
    <p>
    </p>
    <h4>
     <a id="_22">
     </a>
     传输系统
    </h4>
    <p>
     传输系统又名传输网络。
     <br/>
     可以是简单的传输线，也可以是复杂的网络系统。
     <br/>
     其作用也就不用多说了，当然是用来传输数据的。
    </p>
    <p>
    </p>
    <h4>
     <a id="_28">
     </a>
     目的系统
    </h4>
    <p>
     目的系统即接收端、接收方。
     <br/>
     目的系统包括两个部分：
     <br/>
     1.接收器
     <br/>
     2.终点（目的站、信宿）
    </p>
    <p>
     接收器用来干什么？
     <br/>
     接收器呢，和发送器做着相反的工作。接收传输系统传送过来的信号，并把它转换为能被目的设备处理的信息。发送器编译，接收器解调。
     <br/>
     所以典型的接收器为解调器。还原发送端产生的数字比特流。
    </p>
    <p>
     终点呢？
     <br/>
     终点设备从接收器接收数字比特流，然后将信息输出。
    </p>
