    <div style="text-align:start">
     <p style="margin-left:0; margin-right:0; text-align:justify">
      <span style="color:#000000">
       Modbus TCP 转Modbus RTU网关快速配置案例
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="background-color:#fafbfc">
        在工业自动化领域，
        <strong>
         <span style="color:#ff0000">
          Modbus 协议
         </span>
        </strong>
        以其简洁和高效而著称，成为众多设备通信的首选。
       </span>
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="background-color:#fafbfc">
        随着技术的发展和应用场景的变化，
        <strong>
         <span style="color:#ff0000">
          Modbus
         </span>
        </strong>
        协议也发展出了不同的版本，其中
        <strong>
         <span style="color:#ff0000">
          Modbus TCP 和 Modbus RTU
         </span>
        </strong>
        是两种常见的变种。
       </span>
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/5a8211f000b94f6dba8550f23fb8b002.png"/>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="background-color:#fafbfc">
        本文将深入探讨这两种协议的特点、应用场景以及它们之间的转换方法，帮助工程师更好地理解和应用这些技术。
       </span>
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/19d6a5291d0749ceaeb361496ee927b3.png"/>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="color:#1f4e79">
        1、
       </span>
       打开
       <strong>
        <span style="color:#ff0000">
         Modbus TCP 转Modbus RTU
        </span>
       </strong>
       配置软件用户界面主要有三部分构成，如下图：
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       设备窗口：用来列举设备信息，包括：端口、协议、命令等；
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       配置窗口：用来配置参数；
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       描述窗口：用来显示描述信息。
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/6df061d2c76945acbc8af7d4ea297249.png"/>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       2、设备窗口采用树形结构，根节点为选中的网关设备。
      </span>
     </p>
     <p style="margin-left:0; margin-right:89.9pt; text-align:justify">
      <span style="color:#000000">
       比如
       <strong>
       </strong>
       <strong>
        <span style="color:#ff0000">
         <strong>
          Modbus RTU
         </strong>
        </span>
       </strong>
       协议，作为主站时，子节点为在此主站下的各个 Modbus 从站，而从站的子节点又为此从站配置的命令。如果设置各层次节点参数，点击此节点即可，右侧配置窗口， 可显示相应参数。设备窗口如下所示：
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/ef904578ed104a3b96496f993aae2571.png"/>
     </p>
     <p style="margin-left:0; margin-right:89.9pt; text-align:justify">
      <span style="color:#000000">
       3、增加节点操作：在子网或节点上单击鼠标左键，选中该节点，然后执行增加节点操作。在子网下增加一个名字为“Node”的节点；
      </span>
     </p>
     <p style="margin-right:90.25pt; text-align:left">
      <span style="color:#000000">
       删除节点操作：单击鼠标左键，选中待删除节点，然后执行删除节点操作。该节点以及所属命令节点全部被删除；
      </span>
     </p>
     <p style="margin-right:90.25pt; text-align:left">
      <span style="color:#000000">
       增加命令操作：在节点上单击鼠标左键，然后执行增加命令操作，为该节点添加命令，弹出选择命令对话框，供用户选择如下图所示：
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/85203fcdbce741cdaba65057f8220a0c.png"/>
     </p>
     <p style="margin-right:90.25pt; text-align:left">
      <span style="color:#000000">
       4、
       <strong>
        <span style="color:#ff0000">
         <strong>
          modbus TCP
         </strong>
        </span>
       </strong>
       配置
      </span>
     </p>
    </div>
    <div style="text-align:start">
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       IP 地 址：设备 IP 地址；
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       子网掩码：设备子网掩码；
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       网关地址：设备在局域网的网关地址；
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/f8cb350b49c143abbad3ff35523f9421.png"/>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       5、可配置参数为：Modbus 通讯波特率、数据位、奇偶校验方式、停止位、通讯传输模式、响应等待时间、轮询延时时间、输出命令轮询模式、脉冲输出时间、扫描比率，配置界面如下：
      </span>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/933fff38b65f4e77856452ad1069d667.png"/>
     </p>
     <p style="margin-right:90.2pt; text-align:left">
      <span style="color:#000000">
       Modbus 通 讯 波 特 率 ：
       <span style="color:#313131">
        1200bit/s, 2400bit/s, 4800bit/s, 9600bit/s, 19200bit/s, 38400bit/s
       </span>
       <span style="color:#313131">
        ,
       </span>
       <span style="color:#313131">
        57600bit/s, 115200bit/。
       </span>
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       数据位：7、8、9 位。
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       奇偶校验方式：无、奇、偶。
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       停止位：1 位、2 位。
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       通讯传输模式：RTU。
      </span>
     </p>
    </div>
    <div style="text-align:start">
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       响应等待时间：当 Modbus 主站发送命令后， 等待从站响应的时间， 范围：
      </span>
     </p>
     <p style="margin-left:44pt; margin-right:0; text-align:left">
      <span style="color:#000000">
       100ms~50000ms。
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       轮询延时时间：当 Modbus 主站发送命令后，收到正确响应或响应超时后，发送下一条
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       Modbus 命令之前的延迟时间，范围：0~2500ms。
      </span>
     </p>
     <p style="margin-right:90.05pt; text-align:left">
      <span style="color:#000000">
       输出命令轮询模式：Modbus 写命令，有四种输出模式：连续输出、禁止输出、逢变输出和脉冲输出。
      </span>
     </p>
     <p style="text-align:left">
      <span style="color:#000000">
       连续输出：与 Modbus 读命令输出方式相同，根据扫描比率进行扫描输出； 禁止输出：禁止输出 Modbus 写命令；
      </span>
     </p>
     <p style="text-align:left">
      <span style="color:#000000">
       逢变输出：输出数据有变化时，输出写命令，并在接收到正确响应数据后停止输出； 脉冲输出：按照脉冲周期，输出写命令。
      </span>
     </p>
     <p style="margin-right:0pt; text-align:left">
      <span style="color:#000000">
       脉冲输出时间：脉冲输出方式的脉冲时间。
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="background-color:#fafbfc">
        <strong>
         <span style="background-color:#fafbfc">
          <span style="color:#ff0000">
           <strong>
            Modbus TCP转Modbus RTU
           </strong>
          </span>
         </span>
        </strong>
        <span style="background-color:#fafbfc">
         <span style="color:#252a3e">
          的转换是一个涉及多方面考量的复杂过程，需要深入了解两种协议的工作原理及应用场景。
         </span>
        </span>
       </span>
      </span>
     </p>
     <p style="margin-left:0; margin-right:0; text-align:left">
      <span style="color:#000000">
       <span style="background-color:#fafbfc">
        <span style="background-color:#fafbfc">
         <span style="color:#252a3e">
          通过合理的规划与实施，可以有效促进新旧设备间的无缝对接，提升工业自动化系统的整体效率和灵活性。
         </span>
        </span>
       </span>
      </span>
     </p>
