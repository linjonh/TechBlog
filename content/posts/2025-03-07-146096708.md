    <h3>
     <a id="_0">
     </a>
     情况描述
    </h3>
    <p>
     非常蛋疼的情况。在外地，
     <code>
      mac
     </code>
     本机的网络是可以正常联网并且访问
     <code>
      google
     </code>
     的，但是需要通过
     <code>
      easyconnect
     </code>
     连接校园里的服务器。而服务器自身的网络又无法访问
     <code>
      google
     </code>
     ，这就导致网上的方案异或是
     <code>
      gpt
     </code>
     等的方案给出的注入
     <code>
      ssh -D
     </code>
     或者
     <code>
      ssh -R
     </code>
     等方案都无法成功，因为走
     <code>
      easyconnect
     </code>
     的话，没法访问外网。（这
     <code>
      easyconect
     </code>
     是真的很流氓，装完像是代理了所有流量，而且还没办法设置什么）
    </p>
    <h3>
     <a id="_3">
     </a>
     解决方案
    </h3>
    <p>
     使用
     <code>
      tinyproxy
     </code>
     ，这个方案需要你在
     <code>
      gpt
     </code>
     那里详细描述下细节迭代几次，才能搞到。这里直接贴
     <code>
      gpt
     </code>
     的原话吧。
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 你现在的网络情况是：</span>
<span class="token comment"># 服务器 必须通过 EasyConnect 才能联网（即，它的所有流量都会走 EasyConnect）。</span>
<span class="token comment"># Mac 可以直接访问 Google，但 EasyConnect 无法访问 Google。</span>
<span class="token comment"># 服务器的流量 必须走 EasyConnect，所以它的代理流量最终还是受 EasyConnect 限制，无法访问 Google。</span>
<span class="token comment"># 💡 可能的解决方案</span>

<span class="token comment"># ✅ 方案 1：在 Mac 上开启 HTTP 代理，并让服务器走这个代理</span>
<span class="token comment"># 目前你在 Mac 上开的 ssh -D 是 SOCKS5 代理，但 很多软件（包括 EasyConnect）会限制 SOCKS5 流量。</span>
<span class="token comment"># 一个可能的解决方案是：在 Mac 上开启 HTTP 代理，让服务器用 curl --proxy http://... 直接走 Mac，而不是 EasyConnect。</span>

<span class="token comment"># 📌 在 Mac 上开启 HTTP 代理</span>
<span class="token comment"># Mac 终端安装 tinyproxy（轻量级 HTTP 代理）：</span>
brew <span class="token function">install</span> tinyproxy
<span class="token comment"># 修改 tinyproxy 配置文件：</span>
<span class="token function">nano</span> /opt/homebrew/etc/tinyproxy.conf
找到：
<span class="token comment"># Allow 127.0.0.1</span>
修改为：
Allow <span class="token number">0.0</span>.0.0/0
<span class="token comment">#（这样允许所有 IP 访问 tinyproxy，否则服务器访问会被拒）</span>
<span class="token comment"># 启动 tinyproxy：</span>
<span class="token function">sudo</span> brew services start tinyproxy
<span class="token comment"># 确保 tinyproxy 在监听：</span>
<span class="token function">netstat</span> <span class="token parameter variable">-an</span> <span class="token operator">|</span> <span class="token function">grep</span> <span class="token number">8888</span>
<span class="token comment"># 如果看到 0.0.0.0:8888 LISTEN，说明 HTTP 代理已启动。</span>
<span class="token comment"># 📌 在服务器上使用 HTTP 代理</span>
<span class="token comment"># 在服务器上，用 curl 访问 Google：</span>

<span class="token comment"># 这里填你的easyconnect的虚拟地址，可以点开软件看到</span>
<span class="token function">curl</span> <span class="token parameter variable">--proxy</span> http://xx.x.xx.xxx:8888 https://www.google.com <span class="token parameter variable">-v</span>
<span class="token comment"># 之后直接export http_proxy=xx.x.xx.xxx:8888</span>
<span class="token comment">#    	export http_proxy=xx.x.xx.xxx:8888 就行</span>

<span class="token comment"># 如果成功，说明 tinyproxy 已经绕过 EasyConnect 访问 Google！</span>
<span class="token comment"># 如果失败，尝试 telnet 看服务器是否能连通：</span>
telnet xx.x.xx.xxx <span class="token number">8888</span>
<span class="token comment"># 如果 Connection refused，说明 tinyproxy 没有正常工作，检查防火墙和配置。</span>
<span class="token comment"># 如果 Connected，但 curl 失败，可能 EasyConnect 还在拦截 HTTP 代理流量。</span>
</code></pre>
    <p>
     注意本地连接服务器时走
     <code>
      easyconnect
     </code>
     相当于就走过一次代理，所以要用
     <code>
      easyconnect
     </code>
     分配的虚拟地址去做操作，本质上原理应该是
     <code>
      mac
     </code>
     通过
     <code>
      tinyproxy
     </code>
     自己起了一个代理，级别更高以绕过，监听其他访问网站的操作，让
     <code>
      easyconnect
     </code>
     的流量也走到
     <code>
      mac
     </code>
     本机的网络上。但奇怪的是我访问某些
     <code>
      mac
     </code>
     本机才能连上的网址，仍然是处于被屏蔽状态，还是理解的不对？网络还得学。
