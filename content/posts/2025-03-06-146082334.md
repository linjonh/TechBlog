    <p>
     MVC（Model-View-Controller）是一种
     <strong>
      分层架构设计模式
     </strong>
     ，核心思想是通过职责分离提升代码的可维护性和扩展性。它的三个组件分工如下：
    </p>
    <h5>
     1.
     <strong>
      Model（模型）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        职责
       </strong>
       ：管理数据和业务逻辑，与数据库或外部服务交互。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         不依赖 View 和 Controller，是纯粹的数据层。
        </p>
       </li>
       <li>
        <p>
         提供数据校验、持久化（如保存到数据库）、状态通知（通过观察者模式）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：用户注册时，Model 负责验证邮箱格式、密码强度，并将数据存入数据库。
      </p>
     </li>
    </ul>
    <h5>
     2.
     <strong>
      View（视图）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        职责
       </strong>
       ：负责数据展示和用户界面（UI），不处理业务逻辑。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         被动接收 Model 的数据更新（如通过绑定或事件监听）。
        </p>
       </li>
       <li>
        <p>
         支持多形态展示（同一数据可以渲染为网页、移动端页面或 PDF）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：用户个人主页的 HTML 模板，展示用户的头像、昵称和动态列表。
      </p>
     </li>
    </ul>
    <h5>
     3.
     <strong>
      Controller（控制器）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        职责
       </strong>
       ：接收用户输入，协调 Model 和 View。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         解析请求参数（如 HTTP 请求中的表单数据）。
        </p>
       </li>
       <li>
        <p>
         调用 Model 处理业务，根据结果选择渲染哪个 View。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：用户点击“提交订单”按钮时，Controller 调用 Model 扣减库存，成功后跳转到支付页面。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     🌰
     <strong>
      MVC 工作流程示例（用户登录场景）
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        用户操作
       </strong>
       ：在登录页面输入账号密码，点击“登录”按钮。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Controller
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         接收 HTTP 请求，解析账号密码参数。
        </p>
       </li>
       <li>
        <p>
         <strong>
          校验参数格式
         </strong>
         （如是否为空、长度是否合法）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Model
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         根据账号查询数据库，验证密码哈希是否匹配。
        </p>
       </li>
       <li>
        <p>
         返回验证结果（成功或失败）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Controller
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         根据 Model 的返回结果：
        </p>
        <ul>
         <li>
          <p>
           成功：调用 View 渲染用户主页。
          </p>
         </li>
         <li>
          <p>
           失败：调用 View 渲染错误提示页。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        View
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         生成 HTML 页面（主页或错误页），返回给用户浏览器。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     ⚡️
     <strong>
      MVC 的优缺点
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         优点
        </strong>
       </th>
       <th>
        <strong>
         缺点
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        分离关注点，便于团队协作
       </td>
       <td>
        复杂场景下 Controller 可能臃肿
       </td>
      </tr>
      <tr>
       <td>
        视图可替换（如 Web/APP 切换）
       </td>
       <td>
        Model 和 View 可能产生间接耦合（如双向绑定）
       </td>
      </tr>
      <tr>
       <td>
        易于单元测试（分层明确）
       </td>
       <td>
        小型项目可能过度设计
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     🆚
     <strong>
      MVC 与 MVVM 的区别
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        MVC
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         View 直接依赖 Model（如通过数据绑定）。
        </p>
       </li>
       <li>
        <p>
         Controller 负责手动更新 View（如调用
         <code>
          render()
         </code>
         方法）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        MVVM
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         引入
         <strong>
          ViewModel
         </strong>
         层，通过双向绑定自动同步 View 和 Model。
        </p>
       </li>
       <li>
        <p>
         典型应用：Vue.js、Angular（减少手动 DOM 操作）。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <h2>
     模拟提问：如果让你用 MVC 设计一个TODO列表应用，你会如何划分模块？
    </h2>
    <ol>
     <li>
      <p>
       <strong>
        Model
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         定义
         <code>
          TodoItem
         </code>
         类，包含标题、状态（完成/未完成）、创建时间等属性。
        </p>
       </li>
       <li>
        <p>
         提供
         <code>
          save()
         </code>
         （保存到本地存储）、
         <code>
          loadAll()
         </code>
         （加载所有待办事项）等方法。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        View
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         HTML 页面结构，包含输入框、添加按钮、待办列表。
        </p>
       </li>
       <li>
        <p>
         渲染方法：根据 Model 数据生成列表项，标记完成状态（如划掉文字）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Controller
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         监听“添加”按钮点击事件，获取输入内容，调用 Model 的
         <code>
          save()
         </code>
         。
        </p>
       </li>
       <li>
        <p>
         监听“删除”或“切换状态”操作，更新 Model 后触发 View 重新渲染。
        </p>
       </li>
      </ul>
     </li>
    </ol>
