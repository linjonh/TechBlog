    <p>
     对于链表我们有很多种，有带头和不带头，双向和单项，循环和不循环。
    </p>
    <p>
     我们实现的单向链表是不带头单向不循环链表。
    </p>
    <p>
     链表不比顺序表，它可以连续也可以不连续，是链子型的每条链子两边都有节点（除首尾)。
    </p>
    <h2>
     单向链表的优缺点
    </h2>
    <h3>
     1.它适用于增删，修改数据，时间复杂度是O（1）。
    </h3>
    <h3>
     2.不适用于查找，时间复杂度是O（n），因为要从头找起。
    </h3>
    <h2>
     单向链表的实现
    </h2>
    <p>
     单向链表有两个属性：一个是当前的数值，另一个是下一个节点的地址。
    </p>
    <pre><code class="hljs">public class MySingleList {
    static class ListNode{
        public ListNode next;
        int value;


        public ListNode(int val){
            this.value = val;
        }
    }
 public ListNode head;
}</code></pre>
    <p>
     这里的head代表头节点。
    </p>
    <h3>
     一.增
    </h3>
    <h4>
     1.头插和尾插
    </h4>
    <pre><code class="hljs"> public void addFirst(int data){
        ListNode listNode = new ListNode(data);
        listNode.next = head;
        head = listNode;

    }</code></pre>
    <p>
     头插只需将创建好的节点连接到当前的头节点，再将头节点设为新插入的节点。
    </p>
    <p>
     其实链表的next就像一个带有方向的链子，通过next来使链子指向你想要指向的地址，我们会经常忽略这个问题，经常因为没有处理好next使链表变成了环形循环链表。
    </p>
    <pre><code class="hljs"> public void addLast(int data){
        ListNode listNode = new ListNode(data);
        ListNode cur = head;
        if (cur == null){
            head = listNode;
            return;
        }
        while(cur.next != null){
            cur=cur.next;
        }
        cur.next = listNode;
    }
</code></pre>
    <p>
     尾插就是通过头节点一直找到最后一个节点，进行插入操作，这也是链表不足的地方，因为顺序表有明确的位置可以直接找到尾部，而链表则需要从头找起，时间复杂度达到了O（n）。
    </p>
    <h4>
     2.任意插入
    </h4>

     <img alt="" src="https://i-blog.csdnimg.cn/direct/d44323f8e93f4ca6b136a7f93984e99f.png"/>

    <p>
     既然是任意插入插入的位置就得有讲究，我们得先判断是否合法，就像是顺序表处理下标一样。
    </p>
    <p>
     在位置是头时，我们就用了头插的方法，这里我们用到了find..方法，该方法也体现了链表的特性。
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/48231331f946427a8bffb62a7cd456c7.png">
     </img>
    </p>
    <p>
     作为链形的连接方式，就注定了一个节点断了，链表就毁了，所以在插入时我们得先找到前一个节点，让他先跟新节点连接起来，再让新节点连接下一个节点。
    </p>
