    <h2>
     <a id="09_0">
     </a>
     谷粒商城09——人人开源框架的快速入门
    </h2>
    <p>
     前言：这个系列将使用最前沿的cursor作为辅助编程工具，来快速开发一些基础的编程项目。目的是为了在真实项目中，帮助初级程序员快速进阶，以最快的速度，效率，快速进阶到中高阶程序员。
    </p>
    <p>
     本项目将基于谷粒商城项目，并且对谷粒商城项目进行二次重构，使其满足最新的主流技术栈要求。
    </p>
    <p>
     这篇文章主要介绍，人人开源框架的快速入门，使用它快速搭建项目后台。后面下篇文章考虑使用若依重构。敬请期待。
    </p>
    <h3>
     <a id="1_7">
     </a>
     1、人人开源框架简介
    </h3>
    <p>
     人人开源（Renren Open Source）是一个专注于Java开发的开源社区，提供一系列旨在提高开发效率、降低开发成本的开源项目。这些项目涵盖权限管理、快速开发平台、代码生成等多个方面，帮助开发者快速构建和部署应用。
    </p>
    <p>
     我们将借助它来实现我们后台管理系统的快速搭建。
    </p>
    <p>
     <img alt="image-20250306101327269" src="https://i-blog.csdnimg.cn/img_convert/8248af615651693e4482277af5f42556.png"/>
    </p>
    <p>
     至于里面各个项目的详细介绍，可以自行了解。
    </p>
    <p>
     网址：https://gitee.com/renrenio
    </p>

    <p>
     将其整个拖到我们之前建立的guilimall后端工程文件夹下面。

    <p>
     在项目结构目录下，导入这个项目模块。

    <p>
     打开navicat，连接我们前面创建的数据库，执行下面sql。当然，实际上，我们前面一篇学习数据库设计，已经执行过了，所以这里就不用重复执行了。

     更改下数据库配置，先看application.yml。默认使用的是dev环境。

    <p>
     更改下数据库配置信息。将其替换为你自己创建的数据库。
    </p>
    <p>
     <img alt="image-20250306105729916" src="https://i-blog.csdnimg.cn/img_convert/8c2d4a443249008aa121ca9b335f5c35.png"/>

    <p>
     idea编辑器可能出现如下报错：

     <code>
      &lt;parent&gt;

     标签中加上
     <code>
      &lt;relativePath /&gt;

    <pre><code class="prism language-xml"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>parent</span><span class="token punctuation">&gt;</span></span>
        <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>groupId</span><span class="token punctuation">&gt;</span></span>org.springframework.boot<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>groupId</span><span class="token punctuation">&gt;</span></span>

</code></pre>
    <p>
     如果项目出现大量爆红，开源reload下maven project。清理idea缓存重新打开项目。

    <h3>
     <a id="3_93">
     </a>
     3、运行前端项目
    </h3>
    <p>
     使用vscode打开前端项目。
    </p>
    <p>
     需要具有node环境，我们前面文章早就教过了，这里不再赘述了。没有的同学自己安装。
    </p>
    <p>
     执行命令。
    </p>
    <p>
     <img alt="image-20250306173842393" src="https://i-blog.csdnimg.cn/img_convert/30bea2d856c8da5b9ea50f6e157ffebe.png"/>
    </p>
    <p>
     发现报错。
    </p>
    <p>
     <img alt="image-20250306174146998" src="https://i-blog.csdnimg.cn/img_convert/4a3f6cc2b9496cfaec45068f1fa06f5a.png"/>
    </p>
    <p>
     出现报错，看错误信息，是需要环境中python。
    </p>
    <p>
     安装一个。记得添加到环境变量。如果自动安装报错，可以去官网下周zip包解压，手动添加导PATH环境变量。
    </p>
    <p>
     安装官网：https://www.python.org/downloads/windows/
    </p>
    <p>
     <img alt="image-20250307100152296" src="https://i-blog.csdnimg.cn/img_convert/e04d12f527f19bce9045affcd8293e81.png"/>
    </p>
    <p>
     测试下。
    </p>
    <p>
     <img alt="image-20250307101439011" src="https://i-blog.csdnimg.cn/img_convert/b405ce3ab80b93e543bb7417881f3c22.png"/>
    </p>
    <p>
     关闭vscode，重新打开，在vscode终端也测试下，确保能找到python。
    </p>
    <p>
     重新执行
     <code>
      pnpm install
     </code>
     ，报错。错误信息如下。
    </p>
    <p>
     <img alt="image-20250307102901606" src="https://i-blog.csdnimg.cn/img_convert/570577b7c27313d212af0c0f3c0d9790.png"/>
    </p>
    <p>
     从错误信息来看，
     <code>
      node-gyp
     </code>
     无法找到合适的 Visual Studio 安装来编译原生模块。这是因为
     <code>
      node-gyp
     </code>
     需要 Visual Studio 的 C++ 构建工具来编译某些依赖项。
    </p>
    <p>
     下周安装：https://visualstudio.microsoft.com/zh-hans/visual-cpp-build-tools/
    </p>
    <p>
     <img alt="image-20250307103210332" src="https://i-blog.csdnimg.cn/img_convert/0c4ceb68e2394734bc84eabfc8aada15.png"/>
    </p>
    <p>
     在安装过程中，确保勾选以下组件：
    </p>
    <p>
     <img alt="image-20250307111257819" src="https://i-blog.csdnimg.cn/img_convert/0fe883f878708247f797ed10c0205df5.png"/>
    </p>
    <p>
     <img alt="image-20250307114014557" src="https://i-blog.csdnimg.cn/img_convert/1f02482f72d67277ad2b3fb86b1cec4b.png"/>
    </p>
    <p>
     可能会弹窗，需要重启电脑。那你就按提示重启电脑再安装。
    </p>
    <p>
     <img alt="image-20250307111853950" src="https://i-blog.csdnimg.cn/img_convert/628cd8014f319845f6775ff9f1df59d5.png"/>
    </p>
    <p>
     再执行，发现还是不行。
    </p>
    <p>
     仔细观察错误信息：
    </p>
    <pre><code>gyp ERR! UNCAUGHT EXCEPTION
│ gyp ERR! stack TypeError: Cannot assign to read only property 'cflags' of object '#&lt;Object&gt;'
│ gyp ERR! stack     at createConfigFile (C:\Users\半旧\Desktop\wz\javawrokspace\renren-fast-vue\node_modules\.pnpm\node-gyp@7.1.2\node_modules\…  
│ gyp ERR! stack     at process.processTicksAndRejections (node:internal/process/task_queues:85:11)
│ gyp ERR! System Windows_NT 10.0.26100
│ gyp ERR! command "C:\\Program Files\\nodejs\\node.exe" "C:\\Users\\半旧\\Desktop\\wz\\javawrokspace\\renren-fast-vue\\node_modules\\.pnpm\\nod…  
│ gyp ERR! cwd C:\Users\半旧\Desktop\wz\javawrokspace\renren-fast-vue\node_modules\.pnpm\node-sass@6.0.1\node_modules\node-sass
│ gyp ERR! node -v v22.14.0
│ gyp ERR! node-gyp -v v7.1.2
│ gyp ERR! Node-gyp failed to build your package.
│ gyp ERR! Try to update npm and/or node-gyp and if it does not help file an issue with the package author.
│ Build failed with error code: 7
</code></pre>
    <p>
     核心问题：
    </p>
    <p>
     <strong>
      Node.js 版本过高
     </strong>
     <br/>
     你使用的 Node.js v22.14.0 已超出
     <code>
      node-sass@6.0.1
     </code>
     的兼容范围。
     <code>
      node-sass
     </code>
     官方明确表示其最高仅支持到 Node.js 16。
    </p>
    <p>
     <strong>
      node-gyp 配置冲突
     </strong>
     <br/>
     错误
     <code>
      Cannot assign to read only property 'cflags'
     </code>
     表明
     <code>
      node-gyp
     </code>
     在生成构建配置时遇到权限或语法冲突，可能与高版本 Node.js 的模块加载机制不兼容有关。
    </p>
    <p>
     <strong>
      Windows 编译工具链缺失
     </strong>
     <br/>
     <code>
      node-sass
     </code>
     需要 Visual Studio Build Tools 和 Python 2.x 支持，但新版本 Node.js 默认不再集成这些工具。
    </p>
    <p>
     （1）开代理。进行如下操作,降级 Node.js 至兼容版本：
    </p>
    <pre><code class="prism language-cmd"># 安装 nvm-windows（需管理员权限）
choco install nvm
# 安装并切换到 Node.js v16
nvm install 16.20.2
nvm use 16.20.2
</code></pre>
    <p>
     验证环境：
    </p>
    <pre><code class="prism language-bash"><span class="token function">node</span> <span class="token parameter variable">-v</span>  <span class="token comment"># 应显示 v16.x.x</span>
<span class="token function">npm</span> <span class="token parameter variable">-v</span>   <span class="token comment"># 应显示 6.x.x 或 8.x.x</span>
</code></pre>
    <p>
     (2). 替换 node-sass 为 sass
    </p>
    <p>
     <code>
      node-sass
     </code>
     已弃用，建议改用官方推荐的
     <code>
      sass
     </code>
     （Dart Sass）：
    </p>
    <p>
     卸载旧依赖：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> uninstall node-sass
</code></pre>
    <p>
     安装 sass：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> <span class="token function">install</span> sass --save-dev
</code></pre>
    <p>
     修改项目代码：
     <br/>
     将所有
     <code>
      node-sass
     </code>
     的引用替换为
     <code>
      sass
     </code>
     （例如在 Webpack 或 Vue CLI 配置中）。
    </p>
    <p>
     （3）. 安装 Windows 编译工具链
    </p>
    <p>
     若仍需使用
     <code>
      node-sass
     </code>
     ，需补充环境依赖：
    </p>
    <p>
     安装 Visual Studio Build Tools（网页1、网页4）：
    </p>
    <ul>
     <li>
      勾选「Desktop development with C++」工作负载。
     </li>
     <li>
      安装 Python 3（从
      <a href="https://www.python.org/downloads/release/python-2718/" rel="nofollow">
       Python 官网
      </a>
      下载）。
     </li>
    </ul>
    <p>
     配置环境变量：
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> python <span class="token string">"C:\Python313\python.exe"</span>
<span class="token function">npm</span> config <span class="token builtin class-name">set</span> msvs_version <span class="token number">2022</span>
</code></pre>
    <p>
     <code>
      npm install
     </code>
     没有报错.
    </p>
    <p>
     <img alt="image-20250307141432665" src="https://i-blog.csdnimg.cn/img_convert/653f7b03846e3e77cffeeed9849c1ac6.png"/>
    </p>
    <p>
     咱们接下来运行下这个项目。
    </p>
    <pre><code>npm run dev
</code></pre>
    <p>
     没有问题。
