    <h2>
     <a id="git__noff__0">
     </a>
     git合并 --no-ff 的作用？
    </h2>
    <p>
     在Git中，–no-ff参数用于合并分支时禁止快进（Fast-Forward）模式，强制创建一个新的合并提交。以下是详细说明：
    </p>
    <p>
     ​作用：git merge --no-ff
     <br/>
     ​
     <strong>
      禁止快进合并
     </strong>
     ：即使当前分支可以快进到目标分支（即没有分叉），也会创建一个新的合并提交节点。
     <br/>
     <strong>
      ​保留分支历史
     </strong>
     ：在提交历史中明确记录分支的存在和合并操作，使历史更清晰。
    </p>
    <blockquote>
     <p>
      如果你不知道合并的时候有没有加上 --no-ff ，那你大概率就是没加
     </p>
    </blockquote>
    <h2>
     <a id="FastForward_8">
     </a>
     快进（Fast-Forward）模式
    </h2>
    <h3>
     <a id="_9">
     </a>
     作用：​
    </h3>
    <p>
     快进（Fast-Forward）是 Git 合并分支时的一种默认行为，​当目标分支的提交历史直接领先于当前分支，且没有分叉时，Git 会直接将当前分支的指针（HEAD）移动到目标分支的最新提交，​不会生成新的合并提交。
     <br/>
     本质上，这是一种“指针直接前移”的操作，保持提交历史的线性。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/eaf77306d2514a9baab02ff384ae6492.png"/>
    </p>
    <h2>
     <a id="gitnoff__15">
     </a>
     为什么git合并要加–no-ff ？
    </h2>
    <h3>
     <a id="_16">
     </a>
     明确合并记录
    </h3>
    <p>
     快进合并（不加–no-ff）会直接移动分支指针，导致提交历史呈线性，无法区分是普通提交还是合并操作。使用–no-ff会生成一个合并提交，明确标记分支合并点。
    </p>
    <h3>
     <a id="_19">
     </a>
     保留上下文信息
    </h3>
    <p>
     合并提交的日志可以包含分支名称（如Merge branch ‘feature’），方便后续追踪功能来源或修复的代码。
    </p>
    <h3>
     <a id="_22">
     </a>
     ​简化回滚操作
    </h3>
    <p>
     如果合并后发现问题，可以通过回滚合并提交快速撤销整个分支的改动，而非逐个提交回退。
    </p>
    <h2>
     <a id="noff_25">
     </a>
     不加–no-ff的风险
    </h2>
    <h3>
     <a id="_26">
     </a>
     丢失分支历史
    </h3>
    <p>
     快进合并后，分支的提交历史与主分支线性融合，难以从历史中看出曾存在过的分支结构。
    </p>
    <h3>
     <a id="_28">
     </a>
     调试困难
    </h3>
    <p>
     若某功能引入问题，无法快速定位到合并点，可能需要遍历多个提交才能找到问题根源。
    </p>
    <h2>
     <a id="_noff_31">
     </a>
     不同编辑器、合并方式下增加 –no-ff方式
    </h2>
    <h3>
     <a id="1__32">
     </a>
     1. 全局配置默认禁用
    </h3>
    <h4>
     <a id="_33">
     </a>
     步骤一：
    </h4>
    <p>
     .gitconfig 文件增加两行配置
    </p>
    <pre><code>[merge]
        ff = false
</code></pre>
    <h4>
     <a id="_39">
     </a>
     步骤二：
    </h4>
    <p>
     cmd窗口运行
     <code>
      git config --global merge.ff false
     </code>
     # 所有合并默认禁用快进
    </p>
    <h3>
     <a id="2__41">
     </a>
     2. 命令行合并示例
    </h3>
    <p>
     <code>
      git merge --no-ff &lt;branch&gt;
     </code>
    </p>
