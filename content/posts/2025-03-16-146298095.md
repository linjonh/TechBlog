    <h2>
     本文档旨在提供 Git 的常用命令及其使用示例，涵盖全局参数配置、获取本地仓库、基本概念、本地仓库操作、远程仓库操作和分支操作等内容。
    </h2>
    <hr/>
    <h3>
     1. 全局参数配置
    </h3>
    <p>
     Git 允许用户配置全局参数，以便在所有的仓库中共享这些设置。
    </p>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 设置用户名
git config --global user.name "Your Name"

# 设置邮箱
git config --global user.email "your.email@example.com"

# 查看所有配置
git config --list

# 查看特定配置
git config user.name

# 设置默认编辑器为 VSCode
git config --global core.editor "code --wait"

# 设置默认的合并工具
git config --global merge.tool vscode

# 启用颜色输出
git config --global color.ui auto</code></pre>
    <hr/>
    <h3>
     2. 获取本地仓库
    </h3>
    <h4>
     初始化仓库
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 在当前目录初始化一个新的 Git 仓库
git init</code></pre>
    <h4>
     克隆仓库
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 克隆远程仓库到本地
git clone &lt;repository_url&gt;

# 克隆指定分支
git clone -b &lt;branch_name&gt; &lt;repository_url&gt;</code></pre>
    <hr/>
    <h3>
     3. 基本概念
    </h3>
    <p>
     Git 是一个分布式版本控制系统，用于跟踪文件的变化并协作开发。为了更好地理解 Git 的工作机制，以下是 Git 中的一些核心概念：
    </p>
    <p>
    </p>
    <h4>
     3.1 工作区（Working Directory）.git所在目录就是工作区
    </h4>
    <p>
     <img alt="" height="349" src="https://i-blog.csdnimg.cn/direct/3fcd5018edd74f298a025ce8009463b6.png" width="754"/>
    </p>
    <ul>
     <li>
      工作区是用户当前正在编辑的目录，包含项目的所有文件和文件夹。
     </li>
     <li>
      工作区中的文件可以是未跟踪的文件（未被 Git 管理）或已跟踪的文件（已纳入 Git 管理）。
     </li>
     <li>
      在工作区中修改文件后，需要通过
      <code>
       git add
      </code>
      将更改添加到暂存区。
     </li>
    </ul>
    <h4>
     3.2 暂存区（Staging Area / Index）
    </h4>
    <ul>
     <li>
      暂存区是一个临时区域，用于保存用户打算提交的更改。
     </li>
     <li>
      通过
      <code>
       git add
      </code>
      命令将工作区中的修改添加到暂存区。
     </li>
     <li>
      暂存区允许用户选择性地提交某些更改，而不是一次性提交所有修改。
     </li>
     <li>
      可以通过
      <code>
       git status
      </code>
      查看暂存区和工作区的状态。
     </li>
    </ul>
    <h4>
     3.3 本地仓库（Local Repository）
    </h4>
    <ul>
     <li>
      本地仓库是 Git 存储项目历史记录的地方，通常位于项目根目录下的
      <code>
       .git
      </code>
      文件夹中。
     </li>
     <li>
      每次通过
      <code>
       git commit
      </code>
      提交更改时，Git 会将暂存区的内容保存到本地仓库，并生成一个唯一的提交记录（Commit）。
     </li>
     <li>
      本地仓库包含了项目的完整历史记录、分支信息、标签等。
     </li>
     <li>
      <img alt="" height="185" src="https://i-blog.csdnimg.cn/direct/65c5d184fc1547b387bdf800652f580c.png" width="650"/>
     </li>
    </ul>
    <h4>
     3.4 提交（Commit）
    </h4>
    <ul>
     <li>
      提交是 Git 中的一个基本单位，代表一次对项目的更改。
     </li>
     <li>
      每次提交都会生成一个唯一的 SHA-1 哈希值，用于标识该提交。
     </li>
     <li>
      提交包含以下信息：
      <ul>
       <li>
        作者和提交者的信息。
       </li>
       <li>
        提交的时间戳。
       </li>
       <li>
        提交的更改内容（通过
        <code>
         diff
        </code>
        查看）。
       </li>
       <li>
        提交的注释信息（通过
        <code>
         git commit -m
        </code>
        添加）。
       </li>
       <li>
        <img alt="" height="319" src="https://i-blog.csdnimg.cn/direct/af02e4a9c1fd4f628608ac2a033fbf02.png" width="538"/>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     3.5 远程仓库（Remote Repository）
    </h4>
    <ul>
     <li>
      远程仓库是托管在服务器上的 Git 仓库，用于团队协作和代码共享。
     </li>
     <li>
      通过
      <code>
       git clone
      </code>
      命令将远程仓库克隆到本地，或者通过
      <code>
       git remote add
      </code>
      添加新的远程仓库。
     </li>
     <li>
      通过
      <code>
       git push
      </code>
      将本地仓库的更改推送到远程仓库，通过
      <code>
       git pull
      </code>
      或
      <code>
       git fetch
      </code>
      获取远程仓库的更改。
     </li>
     <li>
      常见的远程仓库托管平台包括 GitHub、GitLab 和 Bitbucket。
     </li>
    </ul>
    <h4>
     3.6 分支（Branch）
    </h4>
    <ul>
     <li>
      分支是 Git 中用于并行开发的核心功能。
     </li>
     <li>
      默认情况下，Git 会创建一个
      <code>
       master
      </code>
      或
      <code>
       main
      </code>
      分支。
     </li>
     <li>
      每个分支代表一个独立的开发线，可以在分支上进行修改而不影响其他分支。
     </li>
     <li>
      通过
      <code>
       git branch
      </code>
      创建新分支，通过
      <code>
       git checkout
      </code>
      切换分支，通过
      <code>
       git merge
      </code>
      合并分支。
     </li>
     <li>
      分支的轻量级特性使得 Git 在并行开发中非常高效。
     </li>
    </ul>
    <h4>
     3.7 标签（Tag）
    </h4>
    <ul>
     <li>
      标签是 Git 中用于标记特定提交的功能，通常用于发布版本（如
      <code>
       v1.0.0
      </code>
      ）。
     </li>
     <li>
      标签分为轻量标签（Lightweight Tag）和附注标签（Annotated Tag）。
      <ul>
       <li>
        轻量标签：只是一个指向特定提交的指针。
       </li>
       <li>
        附注标签：包含额外的信息，如标签名、作者、日期和注释。
       </li>
      </ul>
     </li>
     <li>
      通过
      <code>
       git tag
      </code>
      创建标签，通过
      <code>
       git push --tags
      </code>
      将标签推送到远程仓库。
     </li>
    </ul>
    <h4>
     3.8 HEAD
    </h4>
    <ul>
     <li>
      <code>
       HEAD
      </code>
      是一个指针，指向当前工作区所在的分支或提交。
     </li>
     <li>
      通常情况下，
      <code>
       HEAD
      </code>
      指向当前分支的最新提交。
     </li>
     <li>
      通过
      <code>
       git checkout &lt;commit&gt;
      </code>
      可以分离
      <code>
       HEAD
      </code>
      ，使其直接指向某个提交而不是分支。
     </li>
    </ul>
    <hr/>
    <h3>
     4. 本地仓库操作
    </h3>
    <h4>
     查看状态
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 查看当前仓库状态
git status</code></pre>
    <h4>
     添加文件到暂存区
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 添加单个文件
git add &lt;file_name&gt;

# 添加所有文件
git add .</code></pre>
    <h4>
     提交更改
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 提交到本地仓库
git commit -m "Commit message"

# 提交并跳过暂存区
git commit -a -m "Commit message"</code></pre>
    <h4>
     查看提交历史
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 查看提交历史
git log

# 查看简洁的提交历史
git log --oneline

# 查看某个文件的修改历史
git log &lt;file_name&gt;</code></pre>
    <h4>
     撤销更改
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 撤销工作区的更改
git checkout -- &lt;file_name&gt;

# 撤销暂存区的更改
git reset HEAD &lt;file_name&gt;

# 修改最后一次提交
git commit --amend</code></pre>
    <h4>
     删除文件
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 从 Git 和文件系统中删除文件
git rm &lt;file_name&gt;

# 从 Git 中删除文件，但保留在文件系统中
git rm --cached &lt;file_name&gt;</code></pre>
    <hr/>
    <h3>
     5. 远程仓库操作
    </h3>
    <h4>
     添加远程仓库
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 添加远程仓库
git remote add origin &lt;repository_url&gt;</code></pre>
    <h4>
     查看远程仓库
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 查看远程仓库信息
git remote -v</code></pre>
    <h4>
     推送更改到远程仓库
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 推送当前分支到远程仓库
git push origin &lt;branch_name&gt;

# 推送所有分支到远程仓库
git push --all origin</code></pre>
    <h4>
     拉取远程仓库的更改
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 拉取远程仓库的更改并合并
git pull origin &lt;branch_name&gt;

# 拉取远程仓库的更改但不合并
git fetch origin</code></pre>
    <h4>
     删除远程分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 删除远程分支
git push origin --delete &lt;branch_name&gt;</code></pre>
    <hr/>
    <h3>
     6. 分支操作
    </h3>
    <h4>
     创建分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 创建新分支
git branch &lt;branch_name&gt;

# 创建并切换到新分支
git checkout -b &lt;branch_name&gt;</code></pre>
    <h4>
     切换分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 切换到指定分支
git checkout &lt;branch_name&gt;</code></pre>
    <h4>
     查看分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 查看所有分支
git branch

# 查看远程分支
git branch -r

# 查看所有分支（包括远程）
git branch -a</code></pre>
    <h4>
     合并分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 将指定分支合并到当前分支
git merge &lt;branch_name&gt;

# 查看合并冲突
git status</code></pre>
    <h4>
     删除分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 删除本地分支
git branch -d &lt;branch_name&gt;

# 强制删除本地分支
git branch -D &lt;branch_name&gt;</code></pre>
    <h4>
     重命名分支
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 重命名当前分支
git branch -m &lt;new_branch_name&gt;

# 重命名其他分支
git branch -m &lt;old_branch_name&gt; &lt;new_branch_name&gt;</code></pre>
    <h4>
     查看分支历史
    </h4>
    <p>
     &lt;BASH&gt;
    </p>
    <pre><code># 查看分支的提交历史
git log --graph --oneline --all</code></pre>
