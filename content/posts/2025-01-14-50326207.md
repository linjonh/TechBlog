    UML和建模软件Rose的使用已在上一篇介绍过了，这一篇我们继续介绍画图方法。
    <br/>
    我们介绍“
    <strong>
     类图
    </strong>
    ”:
    <br/>
    <p>
     用于描述系统中的类与类之间的各种关系。
    </p>
    <p>
     继承:父子类关系;
    </p>
    实现:类与接口关系;
    <br/>
    关联(全局变量, 组合/聚合): 值、引用;
    <br/>
    依赖:局部变量(非Spring中 依赖注入)。
    <br/>
    <br/>

          其中图中的"关联"和"依赖"的不同:
          <br/>

          依赖：局部性
          <br/>
          <br/>
          然后我们解释一下图中的各种关系:
          <br/>
          (1)继承关系
          <br/>
          <pre><code class="language-java">public abstract class BaseDaoImpl{
	
    public void save(Object entity){
    }
	
    public Object findObject(Object id){
    }


}


public class UserDaoImpl extends BaseDaoImpl{
}</code></pre>
          <img alt="" src="https://img-blog.csdn.net/20151216122407276?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
          <br/>
          <br/>
          <br/>
          (2)实现关系
          <br/>
          <pre><code class="language-java">public interface UserDao {
}
public class UserDaoImpl 
implements UserDao {
}</code></pre>
          <img alt="" src="https://img-blog.csdn.net/20151216122425862?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
          <br/>
          <br/>
          <br/>
          (3)关联关系
          <br/>
          关联关系实际上就是类与类之间的联系;
          <br/>
          关联具有导航性：即双向关系或单向关系。
          <br/>
          <img alt="" src="https://img-blog.csdn.net/20151216122433985?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
          <br/>
          <br/>
          <br/>
          (4)关联之聚合关系(By ref)
          <br/>
          聚合关系,表示的是整体和部分的关系，整体与部分可以分开。
          <br/>
          <br/>
          使用带空心菱形的实线来表示：
          <br/>
          <img alt="" src="https://img-blog.csdn.net/20151216122455557?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
          <br/>
          <br/>
          (5)组合关系(By value)
          <br/>
          也是整体与部分的关系，但是整体与部分不可以分开;部分离开整体没有实际意义。
          <br/>
          使用带实心菱形的实线来表示:
          <br/>
          <img alt="" src="https://img-blog.csdn.net/20151216122503391?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQv/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/Center"/>
          <br/>
          <br/>
          (6)依赖关系
          <br/>
          A类使用到了另一个B类型。
          <br/>
          <br/>
          关系具有:临时性的、偶然性,但是B类的变
          <br/>
          <p>
           会影响到A。代码层面，为类B作为参数被类A在某个method方法中使用。
          </p>
