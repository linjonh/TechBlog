    <h2>
     <a id="_0">
     </a>
     错误记录
    </h2>
    <p>
     当代码仓里分支信息非常多时，如果每次都用git pull更新信息，会耗费很长时间，因此可以用git fetch把我们要checkout的分支线fetch下来，再去checkout。但是，我遇到一个问题，这个看起来已经fetch到指定分支了，但是无法checkout，本文记录一下这个问题的解决方案
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f2cf14dd3f0b482884b86a78814a5cb2.png"/>
    </p>
    <h2>
     <a id="_3">
     </a>
     解决方案
    </h2>
    <p>
     原因：git 的配置可能影响 git pull 的行为。比如 remote.origin.fetch 配置项指定了 git fetch 时要获取的引用，若配置不当，可能导致无法获取新的分支
     <br/>
     使用下面的指令查看自己的本地仓库配置问题
    </p>
    <pre><code class="prism language-c">git config <span class="token operator">--</span>get remote<span class="token punctuation">.</span>origin<span class="token punctuation">.</span>fetch
</code></pre>
    <p>
     将配置改为，不要指向具体分支，
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config remote.origin.fetch <span class="token string">"+refs/heads/:refs/remotes/origin/"</span>
</code></pre>
    <p>
     再次
     <code>
      git config --get remote.origin.fetch
     </code>
     <br/>
     可以看到 你的本地仓库指向已经修改成，就可以正常fetch，checkout了
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ce53bf86a2d54ab78381b0030b163d12.png"/>
    </p>
