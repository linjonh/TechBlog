    <h2>
     <a id="_11">
     </a>
     创建本地仓库
    </h2>
    <p>
     创建一个VSCode_test文件夹
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4c722947f74744cb88f092b7475ced77.png"/>
    </p>
    <p>
     打开VSCode
     <br/>
     选择创建的文件夹
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2e28ce7a937b48ddb0a319dfe9b9b414.png"/>
    </p>
    <p>
     创建一个
     <code>
      .c
     </code>
     文件
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5aec9b5f41c149b1b23f0f3d20f4682b.png"/>
    </p>
    <p>
     随便写一段代码
     <br/>
     然后按
     <strong>
      Ctrl+s
     </strong>
     保存
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4b11437e91a0494d970a99759edd2577.png"/>
    </p>
    <p>
     然后点击Git的图标
     <br/>
     点初始化仓库，这一步相当于命令行的
     <code>
      git init
     </code>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3dd192ed75e54568b8a1fea7a6df3f7e.png"/>
    </p>
    <p>
     在文件夹中多出了一个
     <code>
      .git
     </code>
     文件
     <br/>
     即
     <strong>
      本地仓库
     </strong>
     创建成功
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b9e9398d7d004632a0f352514dfc7a6a.png"/>
    </p>
    <p>
     这里相当于两个命令
    </p>
    <pre><code class="prism language-c">git add <span class="token punctuation">.</span>
</code></pre>
    <pre><code class="prism language-c">git commit <span class="token operator">-</span>m <span class="token char">'first add'</span>
</code></pre>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/6fd0b09183634fedb56a365cd8a52f03.png"/>
    </p>
    <p>
     在下方
     <strong>
      源代码管理图
     </strong>
     中可以看到我们在
     <strong>
      本地仓库
     </strong>
     的第一次提交
     <br/>
     且分支名为
     <strong>
      main
     </strong>
    </p>
    <p>
     这里相当于使用了命令
     <code>
      git-log
     </code>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2fb42c7008894461869e55ff0c3da178.png"/>
    </p>
    <h2>
     <a id="_61">
     </a>
     推送远程仓库
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/54e17a0b71374cee88d5538edfaca04a.png"/>
    </p>
    <p>
     在Git中创建一个仓库
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/765028dd263741b5a1c59603c18ac8a2.png"/>
    </p>
    <p>
     这里选择
     <code>
      HTTPS
     </code>
     <br/>
     与在
     <code>
      Git Bash
     </code>
     中不一样
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f2649c1410364741a79f9049deefe11c.png"/>
    </p>
    <p>
     复制这个
     <strong>
      链接
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f47bb05278e6436fba76f568b0901a63.png"/>
    </p>
    <p>
     在VSCode中粘贴到此处
     <br/>
     我认为下面的步骤就是
    </p>
    <p>
     命令
     <code>
      git remote add origin git@gitee.com:jian-haohua/vscode_test.git
     </code>
     <br/>
     即添加远程仓库
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/16c2742894b34ccbaacfc05a2b1289c4.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3f8c3d619d174169a06d7199d3c52dd5.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d64d61deef634af0ab2ef38f21664327.png"/>
    </p>
    <p>
     推送到远程仓库中
     <br/>
     即命令
     <code>
      git push --set-upstream origin main:main
     </code>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b3ddc8bc9a4247a3860e3008ef6696d6.png"/>
    </p>
    <p>
     如果是第一次点击
     <strong>
      发布Branch
     </strong>
     <br/>
     可能会报错
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/7b39c5f458d34abab4621deb8066afea.png"/>
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/a940e4b8b3f949efbe136811eb3a2162.png"/>
    </p>
    <p>
     原因：
     <code>
      HTTPS缓存不够
     </code>
     <br/>
     解决：进入
     <strong>
      Git bash
     </strong>
     中执行命令
     <code>
      git config --global http.postBuffer 524288000
     </code>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/834dfe0803e54839bcf41e26a7f1df6f.png"/>
    </p>
    <p>
     点击
     <strong>
      发布Branch
     </strong>
     后下方源代码管理图会多出一个像
     <strong>
      云端
     </strong>
     的图标
     <br/>
     说明成功推动到远程仓库中
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7ee1fcfd4d4f4bc18b51683706937ab1.png"/>
    </p>
    <p>
     在
     <strong>
      Gitee
     </strong>
     中也可以看到成功推送
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c4e710d909784e3bbf9f535dbdbdab50.png"/>
    </p>
    <p>
     当我们对内容进行了修改
     <br/>
     相应的地方会有提示
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/160dcf40914a4342b2da907c32cdb679.png"/>
    </p>
    <p>
     可以再次提交
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/56ddfd72e4684d1cab4dfd0121b4edc8.png"/>
    </p>
    <p>
     下方也会显示出对应的修改记录
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0b361ebfa56b408d861aba88b8c2eb99.png"/>
    </p>
    <p>
     点击同步即将修改后的再次推送到远程仓库
     <br/>
     相当于使用命令
     <code>
      git push
     </code>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5d69023bdaf945e08bbb21440387998a.png"/>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5449890981334d2e9b3f89887f56829a.png"/>
    </p>
    <p>
     至此就完成了Git的基本操作
    </p>
    <h2>
     <a id="_140">
     </a>
     界面功能
    </h2>
    <p>
     这里包含了Git的所有命令
     <br/>
     通过图形化界面替代了命令行
     <br/>
     相比于用
     <code>
      Git Bash
     </code>
     会方便快捷很多
     <br/>
     但是跟Linux一样，用命令行可能会更好
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/663702256bfc47588c46913330ca65eb.png"/>
    </p>
    <p>
     这里也可以进行快捷操作
     <br/>
     将鼠标放到对应的图标
     <br/>
     即会显示对应的功能
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a4461bb8ae8c42f7947905467f4924ea.png"/>
    </p>
    <p>
     当然也可以调出终端使用命令行
     <br/>
     使用快捷键
     <strong>
      Ctrl+Shift+`
     </strong>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cb10a089751a4c2483abb889d95fbdd2.png"/>
    </p>
    <p>
     但是在VSCode中是用不了命令
     <code>
      git-log
     </code>
     的
     <br/>
     因为这个命令要自己提前配置
     <br/>
     如果要在终端命令行中用这个命令
     <br/>
     就先输入
     <code>
      bash
     </code>
     进入Git的命令终端
     <br/>
     然后就相当于使用
     <code>
      bash
     </code>
     了
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0d2b2cb0031d439a88b8fb02ced64000.png"/>
    </p>
    <blockquote>
     <p>
      <code>
       Git bash
      </code>
      是一个为 Windows 提供的命令行工具，它结合了 Git 和一个类 Unix 的 Bash 环境。通过 Git Bash，用户可以在 Windows 上运行类似 Linux 或 macOS 的
      <strong>
       命令行操作
      </strong>
      ，使用 Git 来进行版本控制。具体来说，Git Bash 提供了以下功能：
      <br/>
      <strong>
       Git 命令
      </strong>
      ：让用户在 Windows 上运行 Git 命令，如 git clone、git commit、git push 等。
      <br/>
      <strong>
       Bash 环境
      </strong>
      ：提供类似 Unix 的 shell 环境，可以运行常见的 Unix 命令，如 ls、cp、rm 等。
      <br/>
      <strong>
       支持脚本
      </strong>
      ：可以运行 Shell 脚本（.sh），这对于自动化任务非常有用。
     </p>
    </blockquote>
    <p>
     简单来说，
     <code>
      Git Bash 是 Windows 用户使用 Git 和类 Unix 环境的工具
     </code>
    </p>
