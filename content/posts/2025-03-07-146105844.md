    <p>
     在C#中，命名空间（Namespace）是一种封装和组织代码的方式，它允许将相关的类、接口、结构体和枚举等类型组织在一起，以避免命名冲突，并提供了一种逻辑上的分组方式。命名空间的使用有助于提高代码的可读性、可维护性和复用性。
    </p>
    <h2>
     <a id="_1">
     </a>
     为什么要使用命名空间
    </h2>
    <p>
     使用命名空间是为了避免命名冲突。举个例子
    </p>
    <pre><code class="prism language-csharp"><span class="token keyword">namespace</span> <span class="token namespace">M1</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">C</span> <span class="token punctuation">{<!-- --></span> <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
<span class="token keyword">namespace</span> <span class="token namespace">M1</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">C</span> <span class="token punctuation">{<!-- --></span> <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     如上面代码所示，如果定义了两个C类，由于名字上产生了冲突，编译器无法识别该调用哪个C类，就无法通过编译。但是，如果两个C类位于不同的命名空间下，就不存在命名冲突了，因为一个是M1.C，另一个是M2.C。
     <br/>
     另外一点就是，如果自定义的类型和.NET框架提供的类型名称相同，使用命名空间来包装自定义的类型就十分有必要了。如System命名空间的有个Array类，而开发者自己也定义一个Array类，为了避免冲突可以把自己定义的Array类放到一个命名空间内Lib，（使用时 Lib.Array就可以了）
    </p>
    <h2>
     <a id="_16">
     </a>
     命名空间的基本概念
    </h2>
    <ol>
     <li>
      <p>
       <strong>
        定义：
       </strong>
       命名空间通过namespace关键字定义。每个命名空间都可以包含多个类和接口等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用：
       </strong>
       通过using指令可以引入所需的命名空间，这样就可以访问该命名空间中的所有公共成员。
      </p>
     </li>
     <li>
      <p>
       <strong>
        嵌套：
       </strong>
       命名空间可以嵌套，即一个命名空间内部可以包含另一个命名空间。
      </p>
     </li>
     <li>
      <p>
       <strong>
        全局命名空间：
       </strong>
       C# 有一个全局的global命名空间，它允许你在任何地方访问全局变量和函数
      </p>
     </li>
    </ol>
    <h2>
     <a id="_26">
     </a>
     命名空间的示例
    </h2>
    <pre><code class="prism language-csharp"><span class="token keyword">using</span> <span class="token namespace">System</span><span class="token punctuation">;</span>
 
<span class="token keyword">namespace</span> <span class="token namespace">MyApplication</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token keyword">class</span> <span class="token class-name">Program</span>
    <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">static</span> <span class="token return-type class-name"><span class="token keyword">void</span></span> <span class="token function">Main</span><span class="token punctuation">(</span><span class="token class-name"><span class="token keyword">string</span><span class="token punctuation">[</span><span class="token punctuation">]</span></span> args<span class="token punctuation">)</span>
        <span class="token punctuation">{<!-- --></span>
            Console<span class="token punctuation">.</span><span class="token function">WriteLine</span><span class="token punctuation">(</span><span class="token string">"Hello, World!"</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在这个例子中，MyApplication是一个命名空间，它包含了Program类。通过using System;引入了System命名空间，这样就可以使用Console类。
    </p>
    <h2>
     <a id="_43">
     </a>
     命名空间的嵌套
    </h2>
    <pre><code class="prism language-csharp"><span class="token keyword">namespace</span> <span class="token namespace">OuterNamespace</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token keyword">namespace</span> <span class="token namespace">InnerNamespace</span>
    <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">class</span> <span class="token class-name">MyClass</span>
        <span class="token punctuation">{<!-- --></span>
            <span class="token comment">// 类成员定义</span>
        <span class="token punctuation">}</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在这个例子中，InnerNamespace是嵌套在OuterNamespace内部的。要访问MyClass，需要使用完整的命名空间路径：OuterNamespace.InnerNamespace.MyClass。
    </p>
    <h2>
     <a id="_58">
     </a>
     使用外部命名空间中的类型
    </h2>
    <pre><code class="prism language-csharp"><span class="token keyword">using</span> <span class="token namespace">System</span><span class="token punctuation">;</span> <span class="token comment">// 引入System命名空间，可以使用Console类等</span>
<span class="token keyword">using</span> <span class="token namespace">System<span class="token punctuation">.</span>Collections<span class="token punctuation">.</span>Generic</span><span class="token punctuation">;</span> <span class="token comment">// 引入System.Collections.Generic命名空间，可以使用List&lt;T&gt;等集合类型</span>
</code></pre>
    <h2>
     <a id="_64">
     </a>
     默认命名空间和别名使用
    </h2>
    <pre><code class="prism language-csharp"><span class="token keyword">using</span> <span class="token class-name">Project</span> <span class="token operator">=</span> <span class="token class-name">MyProjectNamespace<span class="token punctuation">.</span>Project</span><span class="token punctuation">;</span> <span class="token comment">// 为MyProjectNamespace.Project指定别名Project</span>
</code></pre>
    <p>
     这样，就可以使用Project来代替完整的命名空间路径。
    </p>
    <h2>
     <a id="_71">
     </a>
     注意事项和最佳实践
    </h2>
    <ul>
     <li>
      <p>
       <strong>
        避免全局命名冲突：
       </strong>
       合理使用命名空间可以有效避免不同库之间的类名冲突。
      </p>
     </li>
     <li>
      <p>
       <strong>
        保持一致的命名空间结构：
       </strong>
       根据项目或组织的结构来组织命名空间，例如按照功能模块或组件来划分。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用静态导入：
       </strong>
       对于常用的类或方法，可以使用静态导入（如上例中的using static System.Math;），这样可以直接调用静态成员而不需要实例化。
      </p>
     </li>
     <li>
      <p>
       <strong>
        全局访问：
       </strong>
       虽然全局访问在某些情况下很有用（例如全局变量），但应谨慎使用以避免潜在的命名冲突和代码混乱。
      </p>
     </li>
    </ul>
    <p>
     通过合理使用C#的命名空间，可以大大提高代码的组织性和可维护性。
    </p>
