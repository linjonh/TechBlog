    <h1 class="title-article" id="articleContentId">
     JVM RuntimeDataArea 成分
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="./../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     根据 JVM 规范，
     <strong>
      运行时数据区（Runtime Data Area）
     </strong>
     是 JVM 内存管理的核心模块，分为以下 5 个主要部分（按线程共享性分类）：
    </p>
    <hr/>
    <h4>
     <strong>
      一、线程私有区域
     </strong>
    </h4>
    <h5>
     1.
     <strong>
      程序计数器（Program Counter Register）---- PC
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        记录当前线程执行字节码指令的地址（行号指示器）。
       </li>
       <li>
        执行 Native 方法时，计数器值为
        <code>
         Undefined
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       特性
      </strong>
      ：
      <ul>
       <li>
        唯一不会发生
        <code>
         OutOfMemoryError
        </code>
        的区域。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     2.
     <strong>
      Java 虚拟机栈（Java Virtual Machine Stack）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        存储方法调用的栈帧（Frame），每个栈帧包含：
        <ul>
         <li>
          <strong>
           局部变量表
          </strong>
          （基本类型、对象引用）。
         </li>
         <li>
          <strong>
           操作数栈
          </strong>
          （计算过程的临时数据）。
         </li>
         <li>
          <strong>
           动态链接
          </strong>
          （指向方法区常量池的符号引用）。
         </li>
         <li>
          <strong>
           方法返回地址
          </strong>
          。
         </li>
        </ul>
       </li>
       <li>
        方法调用对应栈帧的入栈和出栈。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       异常
      </strong>
      ：
      <ul>
       <li>
        <code>
         StackOverflowError
        </code>
        ：栈深度超过限制（如无限递归）。
       </li>
       <li>
        <code>
         OutOfMemoryError
        </code>
        ：栈扩展失败（如线程过多）。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     3.
     <strong>
      本地方法栈（Native Method Stack）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        为 Native 方法（如 C/C++ 实现的方法）提供服务。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       特性
      </strong>
      ：
      <ul>
       <li>
        HotSpot JVM 将虚拟机栈与本地方法栈合并。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       异常
      </strong>
      ：与虚拟机栈相同。
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      二、线程共享区域
     </strong>
    </h4>
    <h5>
     4.
     <strong>
      Java 堆（Java Heap）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        存放对象实例和数组（所有对象分配内存的主区域）。
       </li>
       <li>
        GC 的主要管理区域（分代回收机制）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       结构
      </strong>
      ：
      <ul>
       <li>
        <strong>
         新生代
        </strong>
        （Eden、Survivor 区）。
       </li>
       <li>
        <strong>
         老年代
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       异常
      </strong>
      ：
      <code>
       OutOfMemoryError
      </code>
      （堆内存不足）。
     </li>
    </ul>
    <h5>
     5.
     <strong>
      方法区（Method Area）
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：
      <ul>
       <li>
        存储类元数据（类名、字段、方法、接口等）。
       </li>
       <li>
        运行时常量池（字面量、符号引用）。
       </li>
       <li>
        JIT 编译后的代码缓存。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       实现演变
      </strong>
      ：
      <ul>
       <li>
        JDK 7 及之前：永久代（PermGen）。
       </li>
       <li>
        JDK 8 及之后：元空间（Metaspace，使用本地内存）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       异常
      </strong>
      ：
      <code>
       OutOfMemoryError
      </code>
      （类元数据过多）。
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      三、补充：直接内存（Direct Memory）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       特性
      </strong>
      ：
      <ul>
       <li>
        非 JVM 运行时数据区，但通过 NIO 的
        <code>
         DirectByteBuffer
        </code>
        分配。
       </li>
       <li>
        避免 Java 堆与 Native 堆间的数据复制，提升 I/O 性能。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       异常
      </strong>
      ：
      <code>
       OutOfMemoryError
      </code>
      （本地内存不足）。
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      总结：内存区域对比
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        区域
       </th>
       <th>
        线程共享性
       </th>
       <th>
        存储内容
       </th>
       <th>
        异常类型
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        程序计数器
       </td>
       <td>
        私有
       </td>
       <td>
        指令地址
       </td>
       <td>
        无
       </td>
      </tr>
      <tr>
       <td>
        虚拟机栈
       </td>
       <td>
        私有
       </td>
       <td>
        方法栈帧
       </td>
       <td>
        Stack/OutOfMemoryError
       </td>
      </tr>
      <tr>
       <td>
        本地方法栈
       </td>
       <td>
        私有
       </td>
       <td>
        Native 方法栈帧
       </td>
       <td>
        Stack/OutOfMemoryError
       </td>
      </tr>
      <tr>
       <td>
        堆
       </td>
       <td>
        共享
       </td>
       <td>
        对象实例
       </td>
       <td>
        OutOfMemoryError
       </td>
      </tr>
      <tr>
       <td>
        方法区
       </td>
       <td>
        共享
       </td>
       <td>
        类元数据、常量池
       </td>
       <td>
        OutOfMemoryError
       </td>
      </tr>
      <tr>
       <td>
        直接内存（补充）
       </td>
       <td>
        共享
       </td>
       <td>
        NIO 缓冲区
       </td>
       <td>
        OutOfMemoryError
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      提示
     </strong>
     ：各区域的内存分配参数可通过 JVM 选项调整（如
     <code>
      -Xmx
     </code>
     设置堆最大值，
     <code>
      -XX:MaxMetaspaceSize
     </code>
     限制元空间）
    </p>
