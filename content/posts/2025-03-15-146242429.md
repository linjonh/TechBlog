     C++初阶——类和对象（二）
    </h2>
    <p>
     本期内容书接上回，继续讨论类和对象相关内容。类和对象属于C++初阶部分，主要反映了面向对象编程的三大基本特点之一——封装，在C++的学习中占有举足轻重的地位！

    <h3>
     <a id="_2">
     </a>
     一、类对象模型
    </h3>
    <h4>
     <a id="1_3">
     </a>
     1.如何计算基于一个类创建出的对象大小？
    </h4>
    <h5>
     <a id="1sizeof_4">
     </a>
     （1）使用sizeof计算
    </h5>
    <p>
     我们先来复习一下什么是类，什么是对象。类定义了一个新的
     <mark>
      作用域
     </mark>
     ，类的所有成员都在类的作用域中。通过class关键字将
     <code>
      数据
     </code>
     与操作（
     <code>
      成员函数
     </code>
     ）绑定。类定义仅仅是逻辑的
     <mark>
      蓝图
     </mark>
     ，就像是一个房间的
     <mark>
      设计图纸
     </mark>
     ，并不能真正的住进去，因此，我们需要
     <mark>
      对类进行实例化
     </mark>
     才会创建出具体的对象，
     <strong>
      分配内存空间
     </strong>
     。
     <mark>
      同一个类可以实例化出多个不同的对象
     </mark>
     。因此，从实际意义上来说，我们所计算的内存空间，并不是类本身，而是基于类创建出的对象的大小。当然，仅仅从计算这个层面出发，我们可以通过类来计算如果用这个类来创建对象，对象应该会占用多少空间，就好比我们可以通过房屋的设计图纸来计算房屋建成后，理论上会占用多大的面积，我们来举个例子，分别看一看：
    </p>
    <ul>
     <li>
      第一种计算方法：
      <br/>
      <img alt="示例1" src="https://i-blog.csdnimg.cn/direct/621296c6e34c4316a448f9cdf00b059c.png">
       <br/>
       在这里，我们先
       <mark>
        通过类实例化一个对象
       </mark>
       ，就像是先根据图纸把房子建好，然后再测量房子的占地面积。
      </img>
     </li>
     <li>
      第二种计算方法：

    <h5>
     <a id="2_14">
     </a>
     （2）直接计算
    </h5>
    <p>
     直接计算就涉及到
     <mark>
      结构体的内存对齐
     </mark>
     规则了，知识C语言方面的知识。在这里也简单的复习一下：
    </p>
    <ul>
     <li>
      第一个成员在与结构体偏移量为0的地址处。
     </li>
     <li>
      其他成员变量要对齐到某个数字（对齐数）的
      <strong>
       整数倍
      </strong>
      的地址处。
     </li>
     <li>
      <strong>
       对齐数 = 编译器默认的一个对齐数与该成员大小的较小值
      </strong>
      （VS中默认的对齐数为8）
     </li>
     <li>
      结构体总大小为：
      <code>
       最大对齐数
      </code>
      （所有变量类型最大者与默认对齐参数取较小）
      <code>
       的整数倍
      </code>
      。
     </li>
     <li>
      如果嵌套了结构体的情况，嵌套的结构体对齐到自己的最大对齐数的整数倍处，结构体的整体大小就是所有最大对齐数（含嵌套结构体的对齐数）的整数倍。
      <br/>
      我们来举个例子：
      <br/>
      <img alt="示例4" src="https://i-blog.csdnimg.cn/direct/6f4db3c7669348e39f98212603e7cc09.png#pic_center">
       <br/>
       根据
       <strong>
        对齐数 = 编译器默认的一个对齐数与该成员大小的较小值。（VS中默认的对齐数为8）
       </strong>
       ，int a的对齐数为4（int a本身大小为4，默认对齐数为8，取较小值）；同理，int b对齐数也为4；char c的对齐数为1（char c本身大小为1，默认对齐数为8，取较小值）。 根据
       <strong>
        结构体总大小为：最大对齐数（所有变量类型最大对齐数与默认对齐数的较小值）的整数倍
       </strong>
       ，这里的成员变量的对齐数最大值为4，和默认对齐数8相比较小，因此最终结构体对齐方式为4个字节，a占4字节，b占4字节,c占4字节（用1个字节，空3个字节）。
      </img>
     </li>
    </ul>
    <h6>
     <a id="_24">
     </a>
     为什么需要结构体内存对齐规则？
    </h6>
    <p>
     CPU是
     <mark>
      一次性读取
     </mark>
     一定的字节数，我们以这样的一个结构体为例：
     <br/>
     <img alt="示例5" src="https://i-blog.csdnimg.cn/direct/e869a3f0820f4612a9121a8dad7c7a1f.png">
      <br/>
      char型占用一个字节，int型占用4个字节，假设CPU一次读取4个字节。
     </img>
    </p>
    <ul>
     <li>
      如果是连着存储，那么就是CPU一次读取的4个字节中有一个字节属于char a，另外三个字节属于int b的一部分，b并没有被一下子读取完整，因此需要再读一次，然后将两次读取的结果拼接，然而
      <mark>
       拼接数据的耗时会大大增加
      </mark>
      。
     </li>
     <li>
      如果使用内存对齐，那么前4个字节中，有一个字节是char，后三个字节空着，然后接下来的四个字节完整的存储一个int b，CPU一下子就能把b的值读完整，
      <strong>
       避免了拼接
      </strong>
      ，
      <strong>
       提高了效率
      </strong>
      ，这是一种以空间换时间的方式。
     </li>
    </ul>
    <h4>
     <a id="2_30">
     </a>
     2.类和对象相关的存储方式
    </h4>
    <p>
     刚才在计算的过程中，我们好像只在讨论类中成员变量对应的空间，就算出了对象的大小，但是类里面还有
     <code>
      成员函数
     </code>
     呢，它们占据空间吗？
    </p>
    <ul>
     <li>
      我们以这样一段代码为例：
      <br/>
      <img alt="示例6" src="https://i-blog.csdnimg.cn/direct/28aab811a522428ca276bb4d35dfe917.png">
       <br/>
       根据上文的计算，这里的对象是12个字节，函数并没有占用空间。其实，这也是很好理解的。成员函数和成员变量不同，它是一个
       <mark>
        方法
       </mark>
       。基于同一个类创建出的不同的对象各自的成员变量是自己的，但是操作方法是一样的，是共同的，比如说这里的一个函数，它就是一个打印两个数相加的功能，不管什么对象都一样。就好比一个小区，每家每户的房子是自个儿的，但是那些篮球场，游泳池是大家共用的，没有必要每建一间房，都要配上一个篮球场，游泳池。在这里也一样，每个对象中成员变量是不同的，但是调用同一份函数，如果按照此种方式存储，当一个类创建多个对象时，每个对象中都会保存一份代码，相同代码保存多次，浪费空间。
      </img>
     </li>
     <li>
      如果一个类中没有成员变量，只有成员函数或者什么都没有，如图所示：
      <br/>
      <img alt="示例7" src="https://i-blog.csdnimg.cn/direct/cc585af703494d4fb5746b4925add588.png">
       <br/>
       <img alt="示例8" src="https://i-blog.csdnimg.cn/direct/abc6cbbe67314acba4b6ceb399cbef14.png">
        <br/>
        没有成员变量的类对象，需要1字节，是为了
        <mark>
         占位
        </mark>
        ，表示对象存在，并没有存储有效数据。
       </img>
      </img>
     </li>
    </ul>
    <h3>
     <a id="this_39">
     </a>
     二、this指针
    </h3>
    <p>
     我们以日期类为例：
     <br/>
     <img alt="示例9" src="https://i-blog.csdnimg.cn/direct/92ea52d9b3104a7f9e5c73da87e60788.png"/>
     <br/>
     在这个日期类中，我们定义了年、月、日三个成员变量，并且还有
     <code>
      初始化
     </code>
     以及
     <code>
      打印
     </code>
     这两个函数。我们再
     <code>
      实例化
     </code>
     几个对象：
     <br/>
     <img alt="示例10" src="https://i-blog.csdnimg.cn/direct/7fa7cc68ffa04f64adc4c3d616790514.png"/>
     <br/>
     Date类中有
     <code>
      Init
     </code>
     与
     <code>
      Print
     </code>
     两个成员函数，函数体中没有关于不同对象的区分，那当
     <code>
      d1
     </code>
     调用
     <code>
      Init
     </code>
     函数时，该函数是如何知道应该设置
     <code>
      d1
     </code>
     对象，而不是设置
     <code>
      d2
     </code>
     对象呢？这里，我们就要引入一个新的内容——
     <code>
      this指针
     </code>
     ！
     <br/>
     C++编译器给每个“非静态的成员函数“增加了一个
     <strong>
      隐藏的指针参数
     </strong>
     ，让该指针
     <mark>
      指向当前对象(函数运行时调用该函数的对象)
     </mark>
     ，在函数体中所有“成员变量”的操作，都是
     <strong>
      通过该指针去访问
     </strong>
     。只不过所有的操作对用户是透明的，即用户
     <strong>
      不需要传递
     </strong>
     ，编译器自动完成。
     <br/>
     如图所示：
     <br/>
     <img alt="示例11" src="https://i-blog.csdnimg.cn/direct/7b233d41a37e4c9aa466e0f74b904880.png"/>
     <br/>
     比如：
    </p>
    <pre><code class="prism language-bash">Date d1<span class="token punctuation">;</span>
d1.Init<span class="token punctuation">(</span><span class="token number">2025</span>, <span class="token number">3</span>, <span class="token number">9</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
d1.Print<span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这里
     <code>
      this
     </code>
     指针就是指向的
     <code>
      d1
     </code>
     ，然后通过
     <code>
      this
     </code>
     指针访问
     <code>
      d1
     </code>
     里的成员变量。
     <br/>
     <code>
      this
     </code>
     指针的特性
    </p>
    <ul>
     <li>
      <code>
       this
      </code>
      指针的类型：
      <code>
       类类型* const
      </code>
      ，即成员函数中，不能给this指针赋值。也就是说
      <code>
       this指针
      </code>
      在使用时不能指向别的对象，只能指向当前对象，但是可以修改当前对象的内容。
     </li>
     <li>
      只能在“成员函数”的内部使用
     </li>
     <li>
      <code>
       this指针
      </code>
      本质上是“成员函数”的形参，当对象调用成员函数时，将对象地址作为实参传递给this形参。所以对象中不存储this指针。
     </li>
     <li>
      <code>
       this指针
      </code>
      是“成员函数”第一个隐含的指针形参，一般情况由编译器通过ecx寄存器自动传递，不需要用户传递
     </li>
    </ul>
