    <h4>
     <a id="_CentOS_9_NAT__NAT__1">
     </a>
     在 CentOS 9 虚拟机中进行网络配置，可根据不同的网络连接模式（如桥接模式、NAT 模式等）来操作，以下以常见的 NAT 模式为例，详细介绍配置步骤。
    </h4>
    <h5>
     <a id="1__2">
     </a>
     1. 确认虚拟机网络连接模式
    </h5>
    <p>
     打开虚拟机软件（如 VMware Workstation 或 VirtualBox），设置 CentOS 9 虚拟机的网络连接模式为 NAT 模式。
     <br/>
     <strong>
      VMware Workstation
     </strong>
     ：右键点击虚拟机，选择 “设置”，在 “网络适配器” 选项中选择 “NAT 模式”。
     <br/>
     <strong>
      VirtualBox
     </strong>
     ：选择虚拟机，点击 “设置”，在 “网络” 选项卡中，将 “连接方式” 设置为 “网络地址转换（NAT）”。
    </p>
    <h5>
     <a id="2__6">
     </a>
     2. 编辑网络配置文件
    </h5>
    <p>
     CentOS 9 使用 NetworkManager 来管理网络连接，网络配置文件通常位于 /etc/sysconfig/network-scripts/ 目录下，文件名一般以 ifcfg- 开头，后面跟着网络接口名称（如 eth0、ens33 等）。
    </p>
    <ol>
     <li>
      打开终端，使用以下命令查看当前网络接口名称：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">ip</span> <span class="token function">link</span> show
</code></pre>
    <p>
     假设网络接口名称为 ens33。
     <br/>
     2. 使用文本编辑器（如 vim 或 nano）打开对应的网络配置文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">vim</span> /etc/sysconfig/network-scripts/ifcfg-ens33
</code></pre>
    <p>
     修改配置文件内容，确保以下参数正确设置：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">TYPE</span><span class="token operator">=</span>Ethernet
<span class="token assign-left variable">PROXY_METHOD</span><span class="token operator">=</span>none
<span class="token assign-left variable">BROWSER_ONLY</span><span class="token operator">=</span>no
<span class="token assign-left variable">BOOTPROTO</span><span class="token operator">=</span>dhcp  <span class="token comment"># 使用 DHCP 自动获取 IP 地址</span>
<span class="token assign-left variable">DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV4_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6INIT</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_AUTOCONF</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6_ADDR_GEN_MODE</span><span class="token operator">=</span>stable-privacy
<span class="token assign-left variable">NAME</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">UUID</span><span class="token operator">=</span>xxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  <span class="token comment"># 保持原有的 UUID</span>
<span class="token assign-left variable">DEVICE</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">ONBOOT</span><span class="token operator">=</span>yes  <span class="token comment"># 开机自动启用网络接口	</span>
</code></pre>
    <p>
     保存并退出编辑器。如果使用 vim，按 Esc 键，输入 :wq 并回车；如果使用 nano，按 Ctrl + X，然后按 Y 确认保存，最后回车。
     <br/>
     3. 重启网络服务
     <br/>
     修改配置文件后，需要重启网络服务使配置生效：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart NetworkManager
</code></pre>
    <ol start="4">
     <li>
      验证网络连接
      <br/>
      使用以下命令检查网络连接是否正常：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">ping</span> www.baidu.com
</code></pre>
    <p>
     如果能够正常收到响应，说明网络连接配置成功。
     <br/>
     手动配置静态 IP 地址
     <br/>
     如果需要手动配置静态 IP 地址，可以将 BOOTPROTO 参数修改为 static，并添加以下参数：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">IPADDR</span><span class="token operator">=</span><span class="token number">192.168</span>.1.100  <span class="token comment"># 静态 IP 地址</span>
<span class="token assign-left variable">NETMASK</span><span class="token operator">=</span><span class="token number">255.255</span>.255.0  <span class="token comment"># 子网掩码</span>
<span class="token assign-left variable">GATEWAY</span><span class="token operator">=</span><span class="token number">192.168</span>.1.1    <span class="token comment"># 网关地址</span>
<span class="token assign-left variable">DNS1</span><span class="token operator">=</span><span class="token number">8.8</span>.8.8           <span class="token comment"># DNS 服务器地址</span>
</code></pre>
    <p>
     修改完成后，保存文件并重启网络服务：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart NetworkManager
</code></pre>
    <h4>
     <a id="_60">
     </a>
     使用桥接模式进行网络配置
    </h4>
    <p>
     在 CentOS 9 虚拟机中使用桥接模式进行网络配置，可让虚拟机直接连接到物理网络，就像一台独立的物理设备一样，以下是详细的配置步骤。
    </p>
    <h5>
     <a id="1___62">
     </a>
     1. 设置虚拟机网络连接模式为桥接模式
    </h5>
    <p>
     <strong>
      VMware Workstation：
     </strong>
     <br/>
     右键点击虚拟机，选择 “设置”。
     <br/>
     在 “网络适配器” 选项中，选择 “桥接模式”，还可以根据需要选择桥接到的物理网络接口（如无线网卡或有线网卡）。
     <br/>
     <strong>
      VirtualBox
     </strong>
     ：
     <br/>
     选择虚拟机，点击 “设置”。
     <br/>
     在 “网络” 选项卡中，将 “连接方式” 设置为 “桥接网卡”，并从 “界面名称” 下拉列表中选择要桥接的物理网络接口。
    </p>
    <h5>
     <a id="2__69">
     </a>
     2. 编辑网络配置文件
    </h5>
    <p>
     确定网络接口名称：
    </p>
    <ol>
     <li>
      打开终端，使用以下命令查看当前网络接口名称：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">ip</span> <span class="token function">link</span> show
</code></pre>
    <p>
     假设网络接口名称为 ens33。
     <br/>
     2. 编辑配置文件：
     <br/>
     使用文本编辑器（如 vim 或 nano）打开对应的网络配置文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">vim</span> /etc/sysconfig/network-scripts/ifcfg-ens33
</code></pre>
    <p>
     自动获取 IP 地址（DHCP 方式）：
     <br/>
     若希望通过 DHCP 自动获取 IP 地址，确保配置文件内容如下：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">TYPE</span><span class="token operator">=</span>Ethernet
<span class="token assign-left variable">PROXY_METHOD</span><span class="token operator">=</span>none
<span class="token assign-left variable">BROWSER_ONLY</span><span class="token operator">=</span>no
<span class="token assign-left variable">BOOTPROTO</span><span class="token operator">=</span>dhcp
<span class="token assign-left variable">DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV4_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6INIT</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_AUTOCONF</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6_ADDR_GEN_MODE</span><span class="token operator">=</span>stable-privacy
<span class="token assign-left variable">NAME</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">UUID</span><span class="token operator">=</span>xxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  <span class="token comment"># 保持原有的 UUID</span>
<span class="token assign-left variable">DEVICE</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">ONBOOT</span><span class="token operator">=</span>yes
</code></pre>
    <p>
     手动配置静态 IP 地址：
     <br/>
     如果需要手动配置静态 IP 地址，将 BOOTPROTO 参数修改为 static，并添加以下参数：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">TYPE</span><span class="token operator">=</span>Ethernet
<span class="token assign-left variable">PROXY_METHOD</span><span class="token operator">=</span>none
<span class="token assign-left variable">BROWSER_ONLY</span><span class="token operator">=</span>no
<span class="token assign-left variable">BOOTPROTO</span><span class="token operator">=</span>static
<span class="token assign-left variable">DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV4_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6INIT</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_AUTOCONF</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_DEFROUTE</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPV6_FAILURE_FATAL</span><span class="token operator">=</span>no
<span class="token assign-left variable">IPV6_ADDR_GEN_MODE</span><span class="token operator">=</span>stable-privacy
<span class="token assign-left variable">NAME</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">UUID</span><span class="token operator">=</span>xxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  <span class="token comment"># 保持原有的 UUID</span>
<span class="token assign-left variable">DEVICE</span><span class="token operator">=</span>ens33
<span class="token assign-left variable">ONBOOT</span><span class="token operator">=</span>yes
<span class="token assign-left variable">IPADDR</span><span class="token operator">=</span><span class="token number">192.168</span>.1.100  <span class="token comment"># 静态 IP 地址，需与物理网络在同一网段</span>
<span class="token assign-left variable">NETMASK</span><span class="token operator">=</span><span class="token number">255.255</span>.255.0  <span class="token comment"># 子网掩码</span>
<span class="token assign-left variable">GATEWAY</span><span class="token operator">=</span><span class="token number">192.168</span>.1.1    <span class="token comment"># 网关地址，通常是路由器的 IP 地址</span>
<span class="token assign-left variable">DNS1</span><span class="token operator">=</span><span class="token number">8.8</span>.8.8           <span class="token comment"># DNS 服务器地址</span>
<span class="token assign-left variable">DNS2</span><span class="token operator">=</span><span class="token number">8.8</span>.4.4           <span class="token comment"># 备用 DNS 服务器地址</span>
</code></pre>
    <p>
     注意，IPADDR、NETMASK、GATEWAY 和 DNS 等参数需要根据你的实际网络环境进行修改。
     <br/>
     3. 保存并退出配置文件
     <br/>
     若使用 vim 编辑器，按 Esc 键，输入 :wq 并回车。
     <br/>
     若使用 nano 编辑器，按 Ctrl + X，然后按 Y 确认保存，最后回车。
     <br/>
     4. 重启网络服务
     <br/>
     修改配置文件后，需要重启网络服务使配置生效：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart NetworkManager
</code></pre>
    <ol start="5">
     <li>
      验证网络连接
      <br/>
      使用以下命令检查网络连接是否正常：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">ping</span> www.baidu.com
</code></pre>
    <p>
     若能正常收到响应，说明网络连接配置成功。你还可以使用 ip addr show 命令查看当前网络接口的 IP 地址信息。
    </p>
