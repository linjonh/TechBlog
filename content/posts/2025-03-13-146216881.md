    <h3>
     <a id="1_Thread_Pool_1">
     </a>
     1. 线程池（Thread Pool）
    </h3>
    <p>
     核心目的：避免频繁创建/销毁线程，复用固定数量的线程处理任务队列。
    </p>
    <h4>
     <a id="11__4">
     </a>
     1.1 线程池结构要素
    </h4>
    <ul>
     <li>
      任务队列：存储待执行的任务（函数对象）
     </li>
     <li>
      工作线程集合：执行任务的线程
     </li>
     <li>
      同步机制：互斥锁（std::mutex）、条件变量（std::condition_variable）
     </li>
     <li>
      停止标志：安全终止池的开关（std::atomic）
     </li>
    </ul>
    <h4>
     <a id="12__9">
     </a>
     1.2 线程池实现步骤
    </h4>
    <ul>
     <li>
      步骤1：定义任务队列和工作线程
     </li>
     <li>
      步骤2：初始化工作线程，循环等待任务
     </li>
     <li>
      步骤3：提交任务到队列，唤醒线程
     </li>
     <li>
      步骤4：安全关闭池，等待剩余任务完成
      <br/>
      代码示例:
     </li>
    </ul>

    <h3>
     <a id="_218">
     </a>
     多选题答案
    </h3>
    <ol>
     <li>
      答案：A、C、E
     </li>
    </ol>
    <p>
     解析：
    </p>
    <p>
     A. 工作线程在弹出任务时未处理条件变量的虚假唤醒，可能导致线程错误地尝试处理空队列中的任务，从而崩溃或逻辑错误。
     <br/>
     C. 提交任务时不检查线程池是否关闭，可能导致任务被添加到已关闭的队列中，无法执行或引发异常。
     <br/>
     E. 析构线程池时未优雅关闭线程，可能导致资源泄漏或线程访问已销毁的数据。
     <br/>
     B错误：shared_ptr保存锁不会直接导致死锁，锁本身设计或使用顺序错误才会导致死锁。shared_ptr管理锁可能导致所有权混乱，但题目中“导致死锁”的描述不准确。
     <br/>
     D错误：互斥锁粒度过大会降低并发性能，但不会直接导致任务无法执行。
    </p>
    <ol start="2">
     <li>
      答案：A、B
     </li>
    </ol>
    <p>
     解析：
    </p>
    <p>
     A错误：atomic标志位无法强制终止阻塞中的线程（如等待条件变量或I/O），此为协作式中断，选项描述错误。
     <br/>
     B错误：C++标准库的std::thread不原生支持中断，需通过标志位实现协作式中断。
     <br/>
     C正确：任务应包含中断检查点。
     <br/>
     D正确：中断阻塞在I/O的线程可能引发资源泄漏。
     <br/>
     E正确：条件变量需结合谓词避免虚假唤醒。
    </p>
    <ol start="3">
     <li>
      答案：A、B、C、D、E
     </li>
    </ol>
    <p>
     解析：
    </p>
    <p>
     A. 检查队列是否为空需同步，以避免数据竞争。
     <br/>
     B. 多个线程提取任务需同步，防止并发修改队列。
     <br/>
     C. 提交任务到队列尾部需同步，保证队列的线程安全。
     <br/>
     D. 调整线程数量需同步，避免中间状态不一致。
     <br/>
     E. 修改运行状态标志需同步，确保可见性与原子性。
    </p>
    <ol start="4">
     <li>
      答案：A、B、D
     </li>
    </ol>
    <p>
     解析：
    </p>
    <p>
     A. 停止接受新任务是优雅关闭的第一步。
     <br/>
     B. 等待已提交任务完成保证任务全部处理。
     <br/>
     D. 通过条件变量通知线程退出循环是关键步骤。
     <br/>
     C错误：强制终止线程是暴力方式，不优雅。
     <br/>
     E错误：重置未完成任务非必要，取决于设计。
    </p>
    <ol start="5">
     <li>
      答案：B、C、D
     </li>
    </ol>
    <p>
     解析：
    </p>
    <p>
     B. 频繁竞争同一锁（如任务队列锁）是典型性能瓶颈。
     <br/>
     C. 动态调整线程数未负载均衡，可能导致资源浪费或竞争。
     <br/>
     D. 轮询策略可能导致负载不均，影响性能。
     <br/>
     A正确：无锁队列可提升性能。
     <br/>
     E正确：线程本地存储优化高频数据访问，反而是优化点。
