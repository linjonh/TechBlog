    <h3>
     <a id="_2">
     </a>
     合并分支
    </h3>
    <p>
     为了在master主分支上能看到新的提交，就需要将dev分支合并到master分支，示例如下：
    </p>
    <pre><code>$ git branch
* dev
  master
  
$ git checkout master  # 切换到master上进行合并
Switched to branch'master'

$ git merge dev  # 合并dev分支
Updating 166-250e1..374d5ce
Fast-forward
 ReadMe | 1 +
 1 file changed, 1 insertion(+)
 
$ cat ReadMe
hello bit
hello git
hello world
hello version1
hello version2
hello version3
write New content for new branch dev
</code></pre>
    <p>
     <mark>
      <code>
       git merge
      </code>
      命令用于合并指定分支到当前分支
     </mark>
     。
    </p>
    <p>
     合并后，master就能看到dev分支提交的内容了。此时的状态如图如下所示。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/64d3f22ad3ae462ea5f173a65d5eb658.jpeg"/>
    </p>
    <p>
     <code>
      Fast-forward
     </code>
     代表“快进模式”，也就是直接把master指向dev的当前提交，所以合并速度非常快。
     <br/>
     当然，也不是每次合并都能Fast-forward，我们后面会讲其他方式的合并。
    </p>
    <h3>
     <a id="_41">
     </a>
     删除分支
    </h3>
    <p>
     分支完成后，dev分支对于我们来说就没用了，那么dev分支就可以被删除掉，
     <mark>
      注意如果当前正处于某分支下，就不能删除当前分支
     </mark>
     ，
     <br/>
     如：
    </p>
    <pre><code>$ git branch
* dev
  master
  
$ git branch -d dev
error: Cannot delete branch 'dev' checked out at '/home/hyb/gitcode'
</code></pre>
    <p>
     而可以在其他分支下删除当前分支，如：
    </p>
    <pre><code>$ git checkout master
Switched to branch'master'

$ git branch -d dev
Deleted branch dev (was 374d5ce).

$ git branch
* master
</code></pre>
    <p>
     此时的状态如图如下所示。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6d88fd83d40a4995ad6974a39f74f269.jpeg"/>
    </p>
    <p>
     因为创建、合并和删除分支非常快，所以Git鼓励你使用分支完成某个任务，合并后再删掉分支，这和直接在master分支上工作效果是一样的，但过程更安全。
    </p>
