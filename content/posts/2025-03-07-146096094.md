    <h3>
     <strong>
      5. 物理安全
     </strong>
     详细解读
    </h3>
    <p>
     本章节主要评估xxx 有限公司在
     <strong>
      物理安全
     </strong>
     方面的控制措施，包括对敏感资产（物理和信息）的保护、访问控制、监控以及应急响应等方面。以下是详细解读：
    </p>
    <h4>
     1. 安全计划 (5.1 Security plan)
    </h4>
    <p>
     <strong>
      评估结果:
     </strong>
     <strong>
      C- (基本合规)
     </strong>
    </p>
    <p>
     <strong>
      主要内容:
     </strong>
    </p>
    <ul>
     <li>
      <p>
       安全区域划分:
      </p>
      <ul>
       <li>
        <p>
         站点基于安全概念进行设计，
        </p>
        <p>
         采用了适当的安全区域划分系统
        </p>
        <p>
         。
        </p>
        <ul>
         <li>
          <p>
           整个站点被划分为
          </p>
          <p>
           七个级别的安全区域
          </p>
          <p>
           ，
          </p>
          <p>
           并
          </p>
          <p>
           在站点平面图上进行了映射
          </p>
          <p>
           。
          </p>
          <ul>
           <li>
            <p>
             例如，办公区、非安全生产区、高安全生产区等。
            </p>
           </li>
          </ul>
         </li>
        </ul>
       </li>
       <li>
        <p>
         控制矩阵:
        </p>
        <ul>
         <li>
          <p>
           开发了一个控制矩阵，
           <strong>
            清晰显示了哪些物理安全控制措施适用于不同的安全区域
           </strong>
           。
          </p>
         </li>
         <li>
          <p>
           <strong>
            不足:
           </strong>
           控制矩阵
           <strong>
            未包含或引用在站点安全政策中
           </strong>
           ，这可能导致安全策略与实际执行之间存在脱节。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      改进建议:
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        建议将控制矩阵纳入站点安全政策
       </strong>
       ，并确保所有相关人员都了解其内容，以便在日常运营中有效实施。
      </p>
     </li>
     <li>
      <p>
       <strong>
        建议定期审查和更新安全区域划分和控制矩阵
       </strong>
       ，以反映站点布局或安全需求的变化。
      </p>
     </li>
    </ul>

    </h4>

     </strong>

    </p>
    <p>
     <strong>
      主要内容:
     </strong>
    </p>
    <ul>
     <li>
      <p>
       建筑安全:
      </p>
      <ul>
       <li>
        <p>
         站点位于
        </p>
        <p>
         独立工业园区/科技园的多租户建筑内
        </p>
        <p>
         。
        </p>
        <ul>
         <li>
          <p>

           <strong>
            3 号楼
           </strong>

         </li>
         <li>
          <p>

        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       人员入口控制:
      </p>
      <ul>
       <li>
        <p>
         xxx xxx 区域有
         <strong>
          一个人员入口点
         </strong>
         ，通过一个
         <strong>
          受控的门
         </strong>
         从一个
         <strong>
          专用楼梯间
         </strong>
         进入，该楼梯间连接到
         <strong>
          一楼公共可进入的共享入口大厅
         </strong>
         （地面层）。
        </p>
       </li>
       <li>
        <p>
         改进措施:
        </p>
        <ul>
         <li>
          <p>
           xxx 对旋转门结构进行了改造，增加了
           <strong>
            不锈钢栏杆
           </strong>
           。
          </p>
         </li>
         <li>
          <p>
           栏杆的定位
           <strong>
            防止未经授权的人员使用旋转门的背面绕过访问控制机制
           </strong>
           。
          </p>
         </li>
         <li>
          <p>
           不足:
          </p>
          <ul>
           <li>
            <p>
             审核员指出，旋转门
             <strong>
              未安装任何物理屏障
             </strong>
             ，以防止人员在旋转门内完成一次完整旋转。
            </p>
           </li>
           <li>
            <p>
             <strong>
              未经保护的反向流动
             </strong>
             可能会被用来以与授权用户相反的方向通过旋转门。
            </p>
           </li>
           <li>
            <p>
             改进措施:
            </p>
            <ul>
             <li>
              <p>
               由于每次访问的旋转有限，未经授权的通过
               <strong>
                只有在两名授权用户以相反方向通过时才能实现
               </strong>
               。
              </p>
             </li>
            </ul>
           </li>
          </ul>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       高安全区域访问控制:
      </p>
      <ul>
       <li>
        <p>
         高安全区域
        </p>
        <p>
         通过仓库走廊
        </p>
        <p>
        </p>
        <p>
         进入。
        </p>
        <ul>
         <li>
          <p>
           仓库走廊
           <strong>
            通过一堵墙上的门
           </strong>
           从办公室和非安全生产区进入，该墙还将走廊分为两个不同的区域。
          </p>
         </li>
         <li>
          <p>
           墙壁上还包含一个
           <strong>
            紧急出口门
           </strong>
           （见下文）。
          </p>
         </li>
         <li>
          <p>
           门禁控制:
          </p>
          <ul>
           <li>
            <p>
             两扇门都作为 SAS-UP 区域访问控制系统的一部分受到监控。
            </p>
           </li>
           <li>
            <p>
             进入高安全生产区域是通过
             <strong>
              单向人阱门
             </strong>
             作为站点电子访问控制系统的一部分。
            </p>
           </li>
           <li>
            <p>
             一旦通过人阱，就进入
             <strong>
              IT1 和外包装室
             </strong>
             。 一个小型洁净环境更衣室也位于该区域。
            </p>
           </li>
          </ul>
         </li>
         <li>
          <p>
           人阱管理:
          </p>
          <ul>
           <li>
            <p>
             改进措施:
            </p>
            <ul>
             <li>
              <p>
               xxx 修订了人阱手动覆盖的管理方式。
              </p>
             </li>
             <li>
              <p>
               安全控制室工作人员现在必须
               <strong>
                每次使用人阱的手动覆盖功能时填写纸质登记表
               </strong>
               。
              </p>
             </li>
             <li>
              <p>
               使用覆盖的原因
               <strong>
                必须记录在登记表中
               </strong>
               。
              </p>
             </li>
             <li>
              <p>
               不足:
              </p>
              <ul>
               <li>
                <p>
                 审核期间，审核员演示了
                 <strong>
                  绕过视频分析系统
                 </strong>
                 的可能性，使得两名审核员能够
                 <strong>
                  同时进入
                 </strong>
                 。
                </p>
               </li>
               <li>
                <p>
                 审核员使用 xxx 员工对陷阱门进行重新测试时，
                 <strong>
                  无法重现漏洞
                 </strong>
                 。
                </p>
               </li>
               <li>
                <p>
                 审核员指出，
                </p>
                <p>
                 人阱手动覆盖事件的管理流程
                </p>
                <p>
                 需要改进
                </p>
                <p>
                 。
                </p>
                <ul>
                 <li>
                  <p>
                   每次使用手动覆盖功能时，
                   <strong>
                    必须在登记表中记录使用原因
                   </strong>
                   。
                  </p>
                 </li>
                 <li>
                  <p>
                   <strong>
                    站点安全经理应每周审查系统事件日志
                   </strong>
                   ，并将手动覆盖事件与安全控制室操作员维护的记录进行比较。
                  </p>
                 </li>
                </ul>
               </li>
              </ul>
             </li>
            </ul>
           </li>
          </ul>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       紧急出口:
      </p>
      <ul>
       <li>
        <p>
         改进措施:
        </p>
        <ul>
         <li>
          <p>
           审核期间，审核员观察到，紧急出口门的锁定机制
           <strong>
            在审核结束前已修复
           </strong>
           。
          </p>
         </li>
        </ul>
       </li>
       <li>
        <p>
         紧急出口配置:
        </p>
        <ul>
         <li>
          <p>
           从高安全区域有两个紧急出口。
          </p>
         </li>

          <p>
           一扇门提供了一条从高安全个性化车间到 xxx 用于非安全生产活动的通用测试车间的逃生路线。 这条紧急出口路线从非安全生产车间继续延伸到仓库走廊。

         </li>
         <li>
          <p>
           第二扇紧急出口门安装在单向人阱旁边，直接通向仓库走廊。
          </p>
         </li>
         <li>
          <p>
           从仓库到仓库走廊有一个紧急出口。
          </p>
         </li>
        </ul>
       </li>
       <li>
        <p>
         监控:
        </p>
        <ul>
         <li>
          <p>
           所有紧急出口门都作为站点入侵者检测系统的一部分受到监控。
          </p>
         </li>
         <li>
          <p>
           紧急出口门
           <strong>
            通常看起来是适当的构造
           </strong>
           。
          </p>
         </li>
        </ul>
       </li>
       <li>
        <p>

        </p>
        <ul>
         <li>
          <p>
           打开紧急出口会触发
           <strong>
            本地报警
           </strong>
           和
           <strong>
            安全控制室的报警
           </strong>
           。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>

        </p>

          </ul>
         </li>

             </strong>
             。
            </p>
           </li>
          </ul>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

     <strong>
      改进建议:
     </strong>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        建议改进人阱管理流程
       </strong>
       ，例如，实施更严格的访问控制措施或定期审查人阱事件日志。
      </p>
     </li>
     <li>
      <p>
       <strong>

      </p>
     </li>

      <p>
       <strong>

       ，以确保能够有效监控所有关键区域。
      </p>
     </li>
     <li>
      <p>
       <strong>
        建议定期测试 CCTV 系统的图像检索功能
       </strong>
       ，以确保其可靠性。
      </p>
     </li>
    </ul>
    <h4>
     3. 访问控制 (5.3 Access control)
    </h4>
    <p>
     <strong>
      评估结果:
     </strong>
     <strong>
      NC → C- (从不符合到基本合规)
     </strong>
    </p>
    <p>
     <strong>
      主要内容:
     </strong>
    </p>
    <ul>
     <li>
      <p>
       人员访问控制:
      </p>
      <ul>
       <li>
        <p>
         改进措施:
        </p>
        <ul>
         <li>
          <p>
           <strong>

           </strong>

         </li>
         <li>
          <p>
