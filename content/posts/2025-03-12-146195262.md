    <blockquote>
     <h4>
      <span style="color:#ff9900">
       SCSS @import 警告与解决⽅案
      </span>
     </h4>
     <h4>
      <span style="color:#ff9900">
       ⚠ 警告信息
      </span>
     </h4>
     <p>
      在 SCSS 中，使⽤  @import  可能会产⽣以下警告：
     </p>
     <pre><code class="language-css">Deprecation Warning: Sass @import rules are deprecated and will be removed in Dart Sass 3.0.0.</code></pre>
    </blockquote>
    <blockquote>
     <h4>
      <span style="color:#ff9900">
       ? 为什么会有这个警告？
      </span>
     </h4>
     <p>
      Sass 官⽅已经废弃  @import ，推荐使⽤  @use  和  @forward  进⾏模块化管理。其主要原因包括：
      <br/>
      1.
      <strong>
       命名冲突
      </strong>
      ： @import  允许多个⽂件共享变量和混合宏，可能导致命名冲突。
      <br/>
      2.
      <strong>
       重复导⼊
      </strong>
      ： @import  可能导致相同的⽂件被导⼊多次，影响编译性能。
      <br/>
      3.
      <strong>
       作⽤域问题
      </strong>
      ： @import  导⼊的所有内容都是全局的，⽽  @use  采⽤更严格的作⽤域管理，避免变量污染。
     </p>
    </blockquote>
    <blockquote>
     <h4>
      <span style="color:#ff9900">
       ? 如何复现这个警告？
      </span>
     </h4>
     <p>
      创建以下 SCSS ⽂件，并尝试编译：
     </p>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _variables.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">$primary-color: blue;
</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         style.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'variables';
body {
 background-color: $primary-color;
}
</code></pre>
     <p>
      <strong>
       然后运⾏：
      </strong>
     </p>
     <pre><code class="language-css">sass style.scss style.css</code></pre>
     <p>
      <strong>
       你将看到 @import  相关的警告信息。
      </strong>
     </p>
     <p>
      <img alt="" height="732" src="https://i-blog.csdnimg.cn/direct/2d412c6336b8497eb8eec052d5e2f749.png" width="2274"/>
     </p>
    </blockquote>
    <blockquote>
     <h3>
      <span style="color:#ff9900">
       ✅ 如何正确替换 @import ？
      </span>
     </h3>
     <p>
     </p>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _variables.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">$primary-color: blue;
</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         style.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@use 'variables' as v;
body {
 background-color: v.$primary-color;
}</code></pre>
     <p>
      <strong>
       重新编辑后，警告消失
      </strong>
     </p>
     <p>
      <img alt="" height="502" src="https://i-blog.csdnimg.cn/direct/38a90868c65f463eaa65e24d4fb79f89.png" width="1290"/>
     </p>
    </blockquote>
    <blockquote>
     <h4>
      <span style="color:#ff9900">
       @import 的问题示例
      </span>
     </h4>
     <h4>
      <span style="color:#ff9900">
       1. 命名冲突
      </span>
     </h4>
     <h5>
      问题
      <br/>
      <span style="color:#ff9900">
       @import  可能导致相同变量或 Mixin 被覆盖。
      </span>
      <br/>
      示例
     </h5>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#d7d8d9">
         _colors.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">$primary-color: blue;</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _theme.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">$primary-color: red;</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         style.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'colors';
@import 'theme';
body {
 background-color: $primary-color;
}</code></pre>
     <h5>
      <span style="color:#ff9900">
       结果
      </span>
     </h5>
     <p>
      最终  body  的  background-color  变为 red，因为  theme.scss  覆盖了  colors.scss  的值。
     </p>
     <p>
      <img alt="" height="644" src="https://i-blog.csdnimg.cn/direct/bfa683a675d74d4dbafbb401e8a9b21a.png" width="2174"/>
     </p>
     <h4>
      <span style="color:#ff9900">
       2. 重复导⼊
      </span>
     </h4>
     <p>
      问题
      <br/>
      <span style="color:#ff9900">
       @import  可能导致相同的⽂件被多次导⼊，影响编译效率。
      </span>
     </p>
     <p>
      示例
      <br/>
      <span style="color:#0d0016">
       <strong>
        <span style="background-color:#cccccc">
         _buttons.scss
        </span>
       </strong>
      </span>
     </p>
     <pre><code class="language-css">.button {
 padding: 10px;
 border-radius: 5px;
}</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _components.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'buttons';</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         style.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'buttons';
@import 'components';</code></pre>
     <h5>
      <span style="color:#ff9900">
       结果
      </span>
     </h5>
     <p>
      .button  的样式被重复⽣成两次，导致冗余代码。
     </p>
     <p>
      <img alt="" height="562" src="https://i-blog.csdnimg.cn/direct/0f8bbe9a7e1f4d558d9b325e2fd968ac.png" width="2668"/>
     </p>
     <h4>
      <span style="color:#ff9900">
       3. 作⽤域问题
      </span>
     </h4>
     <p>
      问题
      <br/>
      <span style="color:#ff9900">
       @import  导⼊的变量和 Mixin 作⽤域是全局的，可能污染其他⽂件的变量。
      </span>
     </p>
     <p>
      示例
      <br/>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _variables.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">$padding: 20px;</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _card.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'variables';
.card {
 padding: $padding;
}</code></pre>
     <p>
      <strong>
       <span style="color:#0d0016">
        <span style="background-color:#cccccc">
         _button.scss
        </span>
       </span>
      </strong>
     </p>
     <pre><code class="language-css">@import 'variables';
.button {
 padding: $padding; // 可能需要不同的 padding，但被全局变量影响
}</code></pre>
     <p>
      <span style="color:#ff9900">
       由于变量是全局的，未来如果  variables.scss  修改  $padding ，所有引⽤的⽂件都会受影响，可能导致意外的样式变更。
      </span>
     </p>
    </blockquote>
    <blockquote>
     <h4>
      <span style="color:#ff9900">
       ✅@use  的优势
      </span>
     </h4>
     <p>
      <strong>
       避免全局污染
      </strong>
      ： @use  采⽤命名空间（如  v.$primary-color ），防⽌变量污染全局作⽤域。
      <br/>
      <strong>
       避免重复导⼊
      </strong>
      ： @use  只会导⼊⼀次，优化编译性能。
      <br/>
      <strong>
       模块化管理
      </strong>
      ：使⽤  @use  让代码更结构化，便于维护。
     </p>
     <h4>
      <span style="color:#ff9900">
       ✅结论
      </span>
     </h4>
     <p>
      <img alt="" height="176" src="https://i-blog.csdnimg.cn/direct/f7b9f2c428bf46e1b5553907a450c175.png" width="712"/>
     </p>
     <h4>
      <span style="color:#ff9900">
       ✅ 尽早升级到 @use ，避免未来维护成本增加！
      </span>
     </h4>
    </blockquote>
