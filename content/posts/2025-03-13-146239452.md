     构建及运行命令
    </h3>
    <h4>
     <a id="_100">
     </a>
     构建镜像
    </h4>
    <p>
     在构建镜像时，可以使用
     <code>
      --cache-from
     </code>
     参数来利用之前构建的缓存，加快构建速度。
    </p>
    <pre><code class="prism language-bash"><span class="token function">docker</span> build --cache-from mysql:10.11-alpine <span class="token parameter variable">-t</span> mysql:10.11-alpine <span class="token builtin class-name">.</span>
</code></pre>
    <ul>
     <li>
      使用
      <code>
       docker build
      </code>
      命令构建镜像，
      <code>
       -t
      </code>
      参数指定镜像的标签为
      <code>
       mysql:10.11-alpine
      </code>
      ，
      <code>
       .
      </code>
      表示当前目录为构建上下文。
     </li>
    </ul>
    <h4>
     <a id="root_110">
     </a>
     运行容器（需设置root密码）
    </h4>
    <pre><code class="prism language-bash"><span class="token function">docker</span> run <span class="token parameter variable">-d</span> <span class="token punctuation">\</span>
    <span class="token parameter variable">--name</span> mysql <span class="token punctuation">\</span>
    <span class="token parameter variable">-e</span> <span class="token assign-left variable">MYSQL_ROOT_PASSWORD</span><span class="token operator">=</span><span class="token string">'Tech@12345'</span> <span class="token punctuation">\</span>
    <span class="token parameter variable">-v</span> mysql_data:/var/lib/mysql <span class="token punctuation">\</span>
    <span class="token parameter variable">-p</span> <span class="token number">3306</span>:3306 <span class="token punctuation">\</span>
    mysql:10.11-alpine
</code></pre>
    <ul>
     <li>
      使用
      <code>
       docker run
      </code>
      命令运行容器，
      <code>
       -d
      </code>
      参数表示容器在后台运行。
     </li>
     <li>
      <code>
       --name mysql
      </code>
      指定容器的名称为
      <code>
       mysql8
      </code>
      。
     </li>
     <li>
      <code>
       -e MYSQL_ROOT_PASSWORD=
      </code>
      设置环境变量
      <code>
       MYSQL_ROOT_PASSWORD
      </code>
      ，用于指定MySQL的
      <code>
       root
      </code>
      用户密码。
     </li>
     <li>
      <code>
       -v mysql_data:/var/lib/mysql
      </code>
      将容器内的
      <code>
       /var/lib/mysql
      </code>
      目录挂载到名为
      <code>
       mysql_data
      </code>
      的数据卷上，实现数据持久化。
     </li>
     <li>
      <code>
       -p 3306:3306
      </code>
      将容器的3306端口映射到宿主机的3306端口，方便外部访问。
     </li>
    </ul>
    <h3>
     <a id="Dockerfile_129">
     </a>
     下面是Dockerfile内容解说
    </h3>
    <h5>
     <a id="_131">
     </a>
     基础镜像选择
    </h5>
    <pre><code class="prism language-markdown">FROM alpine:3.18
</code></pre>
    <ul>
     <li>
      我们选择
      <code>
       alpine:3.18
      </code>
      作为基础镜像，因为Alpine Linux的镜像非常小巧，这意味着构建出来的MySQL镜像也会相对较小，从而减少磁盘空间的占用，同时也能加快镜像的拉取和容器的启动速度。
     </li>
    </ul>
    <h5>
     <a id="_139">
     </a>
     仓库切换与依赖安装
    </h5>
    <pre><code class="prism language-bash"><span class="token comment"># 切换至旧版本仓库并安装依赖</span>
RUN <span class="token function">sed</span> <span class="token parameter variable">-i</span> <span class="token string">'s/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g'</span> /etc/apk/repositories <span class="token punctuation">\</span>
    <span class="token operator">&amp;&amp;</span> <span class="token builtin class-name">echo</span> <span class="token string">"http://mirrors.aliyun.com/alpine/v3.18/community"</span> <span class="token operator">&gt;&gt;</span> /etc/apk/repositories <span class="token punctuation">\</span>
    <span class="token operator">&amp;&amp;</span> apk update <span class="token punctuation">\</span>
    <span class="token operator">&amp;&amp;</span> apk <span class="token function">add</span> --no-cache <span class="token punctuation">\</span>
        <span class="token assign-left variable">mysql</span><span class="token operator">=</span><span class="token number">10.11</span>.11-r0 <span class="token punctuation">\</span>
        mysql-client<span class="token operator">=</span><span class="token number">10.11</span>.11-r0 <span class="token punctuation">\</span>
        mariadb-connector-c-dev<span class="token operator">=</span><span class="token number">3.3</span>.5-r0 <span class="token punctuation">\</span>
    <span class="token operator">&amp;&amp;</span> <span class="token function">mkdir</span> <span class="token parameter variable">-p</span> /var/run/mysqld /var/lib/mysql <span class="token punctuation">\</span>
    <span class="token operator">&amp;&amp;</span> <span class="token function">chown</span> <span class="token parameter variable">-R</span> mysql:mysql /var/run/mysqld /var/lib/mysql
</code></pre>
    <ul>
     <li>
      使用
      <code>
       sed
      </code>
      命令将Alpine Linux的软件源替换为阿里云的镜像源，这样可以加快软件包的下载速度。
     </li>
     <li>
      添加
      <code>
       http://mirrors.aliyun.com/alpine/v3.18/community
      </code>
      仓库，确保可以获取到所需的软件包。
     </li>
     <li>
      使用
      <code>
       apk update
      </code>
      更新软件包列表，然后使用
      <code>
       apk add --no-cache
      </code>
      安装MySQL服务、MySQL客户端以及
      <code>
       mariadb-connector-c-dev
      </code>
      。
      <code>
       --no-cache
      </code>
      参数可以避免在镜像中缓存安装包，进一步减小镜像体积。
     </li>
     <li>
      创建MySQL运行和数据目录
      <code>
       /var/run/mysqld
      </code>
      和
      <code>
       /var/lib/mysql
      </code>
      ，并将其所有者设置为
      <code>
       mysql
      </code>
      用户，确保MySQL服务有正确的权限。
     </li>
    </ul>
    <h5>
     <a id="_159">
     </a>
     配置文件和入口脚本复制
    </h5>
    <pre><code class="prism language-bash">COPY my.cnf /etc/mysql/my.cnf
COPY docker-entrypoint.sh /usr/local/bin/
</code></pre>
    <ul>
     <li>
      将本地的
      <code>
       my.cnf
      </code>
      配置文件复制到容器内的
      <code>
       /etc/mysql/my.cnf
      </code>
      路径，用于配置MySQL服务。
     </li>
     <li>
      将
      <code>
       docker-entrypoint.sh
      </code>
      脚本复制到容器内的
      <code>
       /usr/local/bin/
      </code>
      路径，该脚本将作为容器的入口点，用于初始化数据库和启动MySQL服务。
     </li>
    </ul>
    <h5>
     <a id="_169">
     </a>
     数据卷和端口暴露
    </h5>
    <pre><code class="prism language-bash">VOLUME /var/lib/mysql
EXPOSE <span class="token number">3306</span>
</code></pre>
    <ul>
     <li>
      使用
      <code>
       VOLUME
      </code>
      指令声明
      <code>
       /var/lib/mysql
      </code>
      为数据卷，这样可以将MySQL的数据存储在宿主机上，避免容器删除时数据丢失。
     </li>
     <li>
      使用
      <code>
       EXPOSE
      </code>
      指令暴露MySQL服务的默认端口3306，方便容器与外部进行通信。
     </li>
    </ul>
    <h5>
     <a id="_179">
     </a>
     入口点和默认命令
    </h5>
    <pre><code class="prism language-bash">ENTRYPOINT <span class="token punctuation">[</span><span class="token string">"docker-entrypoint.sh"</span><span class="token punctuation">]</span>
CMD <span class="token punctuation">[</span><span class="token string">"mysqld"</span>, <span class="token string">"--user=mysql"</span><span class="token punctuation">]</span>
</code></pre>
    <ul>
     <li>
      使用
      <code>
       ENTRYPOINT
      </code>
      指令指定容器启动时执行的脚本为
      <code>
       docker-entrypoint.sh
      </code>
      。
     </li>
     <li>
      使用
      <code>
       CMD
      </code>
      指令指定容器启动后默认执行的命令为
      <code>
       mysqld --user=mysql
      </code>
      ，即启动MySQL服务。
     </li>
    </ul>
    <h4>
     <a id="mycnf_189">
     </a>
     my.cnf配置文件详解
    </h4>
    <p>
     mysqld配置段
    </p>
    <pre><code class="prism language-ini">[mysqld]
user=mysql
datadir=/var/lib/mysql
port=3306
character-set-server=utf8
collation-server=utf8_general_ci
default-authentication-plugin=mysql_native_password

[client]
default-character-set=utf8
</code></pre>
    <ul>
     <li>
      <code>
       user=mysql
      </code>
      ：指定MySQL服务运行的用户为
      <code>
       mysql
      </code>
      。
     </li>
     <li>
      <code>
       datadir=/var/lib/mysql
      </code>
      ：指定MySQL数据存储的目录为
      <code>
       /var/lib/mysql
      </code>
      。
     </li>
     <li>
      <code>
       port=3306
      </code>
      ：指定MySQL服务监听的端口为3306。
     </li>
     <li>
      <code>
       character-set-server=utf8
      </code>
      ：设置MySQL服务器的字符集为
      <code>
       utf8
      </code>
      。
     </li>
     <li>
      <code>
       collation-server=utf8_general_ci
      </code>
      ：设置MySQL服务器的字符排序规则为
      <code>
       utf8_general_ci
      </code>
      ，支持更广泛的字符范围。
     </li>
     <li>
      <code>
       default-authentication-plugin=mysql_native_password
      </code>
      ：强制使用
      <code>
       mysql_native_password
      </code>
      认证方式，避免部分客户端因兼容性问题无法连接。
     </li>
    </ul>
    <p>
     client配置段
    </p>
    <ul>
     <li>
      <code>
       default-character-set=utf8
      </code>
      ：设置MySQL客户端的默认字符集为
      <code>
       utf8
      </code>
      。
     </li>
    </ul>

    <h5>
     <a id="_252">
     </a>
     数据库初始化流程
    </h5>
    <ul>
     <li>
      使用
      <code>
       if [ -z "$(ls -A /var/lib/mysql)" ]
      </code>
      判断
      <code>
       /var/lib/mysql

     </li>
     <li>
      使用
      <code>
       mysql_install_db
      </code>
      初始化MySQL数据库。
     </li>
     <li>
      启动MySQL服务，并设置
      <code>
       --skip-networking
      </code>
      参数，避免在初始化过程中接受外部连接。
     </li>
     <li>
      使用
      <code>

      <code>
       root

      <code>
       MYSQL_ROOT_PASSWORD
      </code>
      的值，并刷新权限。
     </li>
     <li>
      最后杀死临时启动的MySQL进程，并等待其退出。
