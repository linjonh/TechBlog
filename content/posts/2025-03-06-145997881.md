    <h3>
     <a id="1_7">
     </a>
     1.如何理解版本控制？
    </h3>
    <p>
     版本控制是追踪和管理文件变更的系统，用于记录代码的历史修改、协调多人协作、快速回滚错误等场景。常见的版本控制系统分为两类：
    </p>
    <ul>
     <li>
      <strong>
       集中式版本控制（如 SVN）
      </strong>
      ：所有操作依赖中央服务器，存在单点故障风险。
     </li>
     <li>
      <strong>
       分布式版本控制（如 Git）
      </strong>
      ：每个开发者拥有完整的仓库副本，独立性更强。
     </li>
    </ul>
    <p>
     <strong>
      Git 的核心特性：
     </strong>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        分布式架构
       </strong>
      </p>
      <ul>
       <li>
        每个本地仓库都是完整的代码库，包含全部历史记录。
       </li>
       <li>
        支持离线操作（提交、分支切换等）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        高效性能
       </strong>
      </p>
      <ul>
       <li>
        基于快照存储（非传统差异存储），操作速度快。
       </li>
       <li>
        优化大型项目和二进制文件处理。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数据完整性
       </strong>
      </p>
      <ul>
       <li>
        使用 SHA-1 哈希算法确保每次提交的唯一性，防止数据篡改。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        灵活的分支模型
       </strong>
      </p>
      <ul>
       <li>
        分支创建、合并成本极低，鼓励频繁分支操作（如功能分支、Bug 修复分支）。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="2Git_27">
     </a>
     2.Git的操作
    </h3>
    <ol>
     <li>
      <p>
       安装Git
       <br/>
       <code>
        yum install git
       </code>
      </p>
     </li>
     <li>
      <p>
       在Gitee创建项目
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c325703311324c21836b308ff53e5107.png"/>
      </p>
     </li>
     <li>
      <p>
       在创建好的项目页面中复制项目的链接。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/df45bf3496d646619b60c803994f4e6f.png">
        <br/>
        下载项目到本地
        <br/>
        创建好一个放置代码的目录.
        <br/>
        <code>
         git clone [url]
        </code>
        <br/>
        这里的
        <code>
         url
        </code>
        就是刚刚建立好的项目的链接.
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b3c5ff3a6c2349e992d61e87aaa2b47d.png">
         <br/>
         把远端仓库拉取到本地文件。
        </img>
       </img>
      </p>
     </li>
     <li>
      <p>
       这个远端仓库在哪里呢？
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f90f5742f7bf4d05b3270cfa406516fb.png">
        <br/>
        我们会发现在在当前目录下多了一个
        <code>
         .git
        </code>
        目录，严格意义上讲，这个
        <code>
         .git
        </code>
        才是当前新建的仓库，我们把
        <code>
         .git
        </code>
        称之为隐藏的本地仓库。
        <br/>
        <strong>
         注意：
        </strong>
        git提交的时候，只会提交变化的部分！
       </img>
      </p>
     </li>
     <li>
      <p>
       当我们把远端仓库拉取到本地时，我们就可以在当前的目录下新增代码，我们首先在当前目录下创建一个
       <code>
        test.c
       </code>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a86be210e1064902998ca13ccd169564.png">
        <br/>
        我们刚刚创建的
        <code>
         test.c
        </code>
        现在并没有被本地仓库管理起来，当我们把
        <code>
         test.c
        </code>
        添加到本地仓库里的时候，这时候
        <code>
         push
        </code>
        才能把本地仓库推送到远端。
       </img>
      </p>
     </li>
     <li>
      <p>
       我们一般把
       <code>
        .git
       </code>
       称之为
       <strong>
        目录
       </strong>
       ，把当前目录名称之为
       <strong>
        工作区
       </strong>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2d9f315e258541da96cf561d83586532.png"/>
      </p>
     </li>
     <li>
      <p>
       把
       <code>
        test.c
       </code>
       添加到本地仓库
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5c4618cf1dff4992a1953b91d681e74f.png">
        <br/>
        严格意义上讲，此时的
        <code>
         test.c
        </code>
        并没有真正意义上添加到本地仓库，而是放到一个叫
        <strong>
         暂存区
        </strong>
        里面。这个暂存区允许我们多次添加，一次提交到仓库。
       </img>
      </p>
     </li>
     <li>
      <p>
       <code>
        git status
       </code>
       可以看到当前的一个状态
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e79f2511fe2141688e9fd70bf0fc93cd.png"/>
       <br/>
       <code>
        new file
       </code>
       说明次文件是新增的。
      </p>
     </li>
     <li>
      <p>
       <code>
        git commit
       </code>
       真正意义上的提交到本地仓库，我们直接输入
       <code>
        git commit
       </code>
       是不行的，我们需要带上
       <code>
        -m
       </code>
       选项，我们称之为
       <strong>
        日志信息
       </strong>
       或者
       <strong>
        提交日志信息
       </strong>
       。注意：-m后面跟的内容不能胡乱写，必须写清楚本次提交都干了什么。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9192bb72a3064db49e0a82b400237fd9.png"/>
      </p>
     </li>
     <li>
      <p>
       <code>
        git log
       </code>
       可以看到日志，比如我们刚才的推送信息
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/70ff91a6027f4195b05f6e981afc9d1a.png"/>
      </p>
     </li>
     <li>
      <p>
       <code>
        git push
       </code>
       使本地仓库和远端仓库同步
      </p>
     </li>
     <li>
      <p>
       如果我们不小心删除了本地仓库，我们想获得我们历史上的提交信息，不用担心，我们直接复制链接，进行
       <code>
        git clone + 链接
       </code>
       ，就可以把我们之前的记录重新拉取下来。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/61fef65a62784370b5046f6e4ca8b220.png"/>
       <br/>
       以上就是一次
       <code>
        Liunx
       </code>
       下的
       <code>
        git
       </code>
       的基本提交，总结一下就是
       <code>
        git三板斧
       </code>
       ：
      </p>
     </li>
    </ol>
    <ul>
     <li>
      <code>
       git add
      </code>
     </li>
     <li>
      <code>
       git commit
      </code>
     </li>
     <li>
      <code>
       git push
      </code>
      <br/>
      附加：
      <code>
       git log
      </code>
      :查看日志
      <code>
       git status
      </code>
      :查看状态
     </li>
    </ul>

    <h3>
     <a id="_69">
     </a>
     补充细节问题
    </h3>
    <ol>
     <li>
      <code>
       git
      </code>
      命令行安装
     </li>
    </ol>
    <pre><code class="prism language-cpp">sudo yum install git
sudo apt install <span class="token operator">-</span>y git
</code></pre>
    <p>
     查看是否安装
     <code>
      git --version
     </code>
     ,出现下面带有版本号的表明已经安装。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7154c68e717a46ef9769c1c604c447f5.png"/>
    </p>
    <ol start="2">
     <li>
      首次使用：需要输入
      <code>
       username
      </code>
      ,
      <code>
       email
      </code>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4e33d7f894fa4f46b4f1cb6b93241500.png"/>
     </li>
     <li>
      <code>
       git
      </code>
      版本管理，只进行管理源文件
     </li>
     <li>
      如果像以下这样的临时文件不想提交到本地仓库并且提交到远端呢？
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/213badc8862c41d6a1ad3894aba9f21f.png"/>
     </li>
     <li>
      我们在构建仓库时存在一个
      <code>
       .gitignore
      </code>
      ,这个文件是帮助我们进行本地文件后缀过滤的，意思是如果临时文件对应的后缀在此文件中有的话，对应的文件都不会被提交。（忽略特定后缀的文件）
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7383943c0d714b4bb4e3b5472fedeef5.png"/>
     </li>
     <li>
      打开
      <code>
       .gitignore
      </code>
      文件，这都是些临时文件的后缀，当然我们也可以自己向里面添加。
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a47a38247dec47ebb953f42dd6c90176.png"/>
     </li>
     <li>
      如果一个人在一台机器上提交了代码，另一个人在另一台机器上提交代码，就会出现下面情况，提交冲突了，我们就需要
      <code>
       git pull
      </code>
      将远端仓库和本地仓库进行一次同步。然后提交。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/45da88311c1c476ea661aee92700b47c.png"/>
      <br/>
      总结：远端仓库相比较于任何人都是最新的!
      <br/>
      为什么冲突呢？答案是提醒本地用户，你需要和远端仓库同步。
     </li>
    </ol>
