    <p>
     raync 备份 clickhouse 到远程主机
     <br/>
     📌 使用 rsync 备份 ClickHouse 数据到远程主机
     <br/>
     ClickHouse 的数据通常存储在 /var/lib/clickhouse/ 目录下，如果你想将 ClickHouse 数据备份到远程主机，可以使用 rsync。
     <br/>
     ✅ 1. rsync 备份 ClickHouse 数据
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> <span class="token parameter variable">--delete</span> /var/lib/clickhouse/ user@remote_host:/backup/clickhouse/
</code></pre>
    <p>
     🔹 参数解释：
    </p>
    <p>
     -a：归档模式，保留权限、时间戳、符号链接等（相当于 -rlptgoD）。
     <br/>
     -v：显示详细信息。
     <br/>
     -z：压缩数据，提高传输效率。
     <br/>
     –progress：显示进度。
     <br/>
     –delete：同步删除远程已删除的文件（可选）。
     <br/>
     /var/lib/clickhouse/：本机 ClickHouse 数据目录。
     <br/>
     user@remote_host:/backup/clickhouse/：远程主机的备份目录。
     <br/>
     🚀 示例（远程主机 IP 为 192.168.1.100，用户名为 backup）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/ backup@192.168.1.100:/backup/clickhouse/
</code></pre>
    <p>
     ✅ 2. 仅备份 store/ 目录（数据表）
     <br/>
     ClickHouse 主要的表数据存储在 store/ 目录，你也可以仅备份该目录：
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/store/ backup@192.168.1.100:/backup/clickhouse/store/
</code></pre>
    <p>
     ✅ 3. 仅备份 metadata/（表结构）
     <br/>
     ClickHouse 表结构 存储在 /var/lib/clickhouse/metadata/ 目录：
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/metadata/ backup@192.168.1.100:/backup/clickhouse/metadata/
</code></pre>
    <p>
     ✅ 4. 计划任务定期备份
     <br/>
     如果你希望每天凌晨 2 点自动备份 ClickHouse 数据到远程主机，可以添加 crontab 任务：
    </p>
    <pre><code class="prism language-bash"><span class="token function">crontab</span> <span class="token parameter variable">-e</span>
</code></pre>
    <pre><code class="prism language-bash"><span class="token number">0</span> <span class="token number">2</span> * * * <span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/ backup@192.168.1.100:/backup/clickhouse/
</code></pre>
    <p>
     ✅ 5. 使用 rsync 增量备份
     <br/>
     如果你想使用 增量备份（仅传输新数据），可以加上 --link-dest 选项：
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> <span class="token parameter variable">--delete</span> --link-dest<span class="token operator">=</span>/backup/clickhouse/latest /var/lib/clickhouse/ backup@192.168.1.100:/backup/clickhouse/<span class="token variable"><span class="token variable">$(</span><span class="token function">date</span> +<span class="token punctuation">\</span>%F<span class="token variable">)</span></span>/
<span class="token function">ln</span> <span class="token parameter variable">-sfn</span> /backup/clickhouse/<span class="token variable"><span class="token variable">$(</span><span class="token function">date</span> +<span class="token punctuation">\</span>%F<span class="token variable">)</span></span> /backup/clickhouse/latest
</code></pre>
    <p>
     这样每次备份只会存储增量数据，但可以恢复完整的数据。
     <br/>
     ✅ 6. 备份 ClickHouse 并压缩
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> /var/lib/clickhouse/ backup@192.168.1.100:/backup/clickhouse/ <span class="token operator">&amp;&amp;</span> <span class="token function">tar</span> <span class="token parameter variable">-czvf</span> /backup/clickhouse.tar.gz /backup/clickhouse/
</code></pre>
    <p>
     这将：
     <br/>
     先 rsync 备份数据到远程主机
     <br/>
     再使用 tar 打包 gzip 压缩备份文件
     <br/>
     📌 总结
     <br/>
     操作 命令
     <br/>
     完整备份 ClickHouse
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/ backup@remote_host:/backup/clickhouse/
</code></pre>
    <p>
     仅备份 store/（数据表）
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/store/ backup@remote_host:/backup/clickhouse/store/
</code></pre>
    <p>
     仅备份 metadata/（表结构）
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/metadata/ backup@remote_host:/backup/clickhouse/metadata/
</code></pre>
    <p>
     增量备份
    </p>
    <pre><code class="prism language-bash"><span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> --link-dest<span class="token operator">=</span>/backup/clickhouse/latest /var/lib/clickhouse/ backup@remote_host:/backup/clickhouse/<span class="token variable"><span class="token variable">$(</span><span class="token function">date</span> +<span class="token punctuation">\</span>%F<span class="token variable">)</span></span>/
</code></pre>
    <p>
     自动定时备份 crontab -e 添加
    </p>
    <pre><code class="prism language-bash"><span class="token number">0</span> <span class="token number">2</span> * * * <span class="token function">rsync</span> <span class="token parameter variable">-avz</span> <span class="token parameter variable">--progress</span> /var/lib/clickhouse/ backup@remote_host:/backup/clickhouse/
</code></pre>
    <p>
     🚀 推荐使用 rsync 进行增量备份，并结合 crontab 实现自动定期备份！ 🎯
     <br/>
     警告
     <br/>
     备份完成后，如果想用这个备份数据直接启动新的 clickhouse 需要删除 store 目录下的空目录（因为在备份过程中源 clickhouse 如果是启动状态就会有空目录被备份）
    </p>
    <pre><code class="prism language-bash"><span class="token function">find</span> /tmp/test <span class="token parameter variable">-type</span> d <span class="token parameter variable">-empty</span> <span class="token parameter variable">-delete</span>
</code></pre>
