     如何用Docker部署Mysql

    <h4>
     <strong>
      1. 安装 Docker
     </strong>
    </h4>
    <p>
     确保已安装 Docker，并启动 Docker 服务。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        Linux
       </strong>
       ：
      </p>
      <p>
       bash
      </p>
      <p>
       复制
      </p>
      <pre>sudo apt update
sudo apt install docker.io
sudo systemctl start docker
sudo systemctl enable docker</pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      2. 拉取 MySQL 镜像
     </strong>
    </h4>
    <p>
     从 Docker Hub 拉取官方 MySQL 镜像。
    </p>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker pull mysql:latest</pre>
    <hr/>
    <h4>
     <strong>
      3. 启动 MySQL 容器
     </strong>
    </h4>
    <p>
     使用
     <code>
      docker run
     </code>
     命令启动 MySQL 容器。
    </p>
    <h5>
     <strong>
      基本命令
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker run -d \
  --name mysql-container \
  -e MYSQL_ROOT_PASSWORD=my-secret-pw \
  -p 3306:3306 \
  mysql:latest</pre>
    <h5>
     <strong>
      参数说明
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        参数
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         -d
        </code>
       </td>
       <td>
        后台运行容器（detached mode）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         --name mysql-container
        </code>
       </td>
       <td>
        为容器指定名称（可自定义）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -e MYSQL_ROOT_PASSWORD
        </code>
       </td>
       <td>
        设置 MySQL 的 root 用户密码（必须设置）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -p 3306:3306
        </code>
       </td>
       <td>
        将容器的 3306 端口映射到主机的 3306 端口（主机端口:容器端口）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         mysql:latest
        </code>
       </td>
       <td>
        使用的镜像名称及版本（
        <code>
         latest
        </code>
        表示最新版本）。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      4. 连接到 MySQL
     </strong>
    </h4>
    <h5>
     <strong>
      (1) 进入容器内部
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker exec -it mysql-container mysql -u root -p</pre>
    <p>
     输入密码（
     <code>
      my-secret-pw
     </code>
     ）后即可进入 MySQL 命令行。
    </p>
    <h5>
     <strong>
      (2) 使用外部客户端
     </strong>
    </h5>
    <p>
     通过主机 IP 和端口（
     <code>
      localhost:3306
     </code>
     ）连接 MySQL，使用工具如：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        MySQL CLI
       </strong>
       ：
      </p>
      <p>
       bash
      </p>
      <p>
       复制
      </p>
      <pre>mysql -h 127.0.0.1 -P 3306 -u root -p</pre>
     </li>
     <li>
      <p>
       <strong>
        图形化工具
       </strong>
       ：如 MySQL Workbench、DBeaver 等。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      5. 持久化数据
     </strong>
    </h4>
    <p>
     默认情况下，容器停止后数据会丢失。可以通过挂载卷（Volume）将数据持久化到主机。
    </p>
    <h5>
     <strong>
      挂载数据卷
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker run -d \
  --name mysql-container \
  -e MYSQL_ROOT_PASSWORD=my-secret-pw \
  -p 3306:3306 \
  -v /path/on/host:/var/lib/mysql \
  mysql:latest</pre>
    <ul>
     <li>
      <p>
       <code>
        /path/on/host
       </code>
       ：主机上的目录路径（需提前创建）。
      </p>
     </li>
     <li>
      <p>
       <code>
        /var/lib/mysql
       </code>
       ：容器内 MySQL 数据存储路径。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      6. 自定义配置文件
     </strong>
    </h4>
    <p>
     如果需要使用自定义的 MySQL 配置文件（如
     <code>
      my.cnf
     </code>
     ），可以挂载配置文件到容器。
    </p>
    <h5>
     <strong>
      挂载配置文件
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker run -d \
  --name mysql-container \
  -e MYSQL_ROOT_PASSWORD=my-secret-pw \
  -p 3306:3306 \
  -v /path/on/host/my.cnf:/etc/mysql/my.cnf \
  mysql:latest</pre>
    <hr/>
    <h4>
     <strong>
      7. 环境变量配置
     </strong>
    </h4>
    <p>
     MySQL 镜像支持通过环境变量配置数据库。
    </p>
    <table>
     <thead>
      <tr>
       <th>
        环境变量
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         MYSQL_ROOT_PASSWORD
        </code>
       </td>
       <td>
        root 用户密码（必须设置）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         MYSQL_DATABASE
        </code>
       </td>
       <td>
        创建指定名称的数据库。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         MYSQL_USER
        </code>
       </td>
       <td>
        创建新用户。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         MYSQL_PASSWORD
        </code>
       </td>
       <td>
        为新用户设置密码。
       </td>
      </tr>
     </tbody>
    </table>
    <h5>
     <strong>
      示例
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker run -d \
  --name mysql-container \
  -e MYSQL_ROOT_PASSWORD=my-secret-pw \
  -e MYSQL_DATABASE=mydb \
  -e MYSQL_USER=myuser \
  -e MYSQL_PASSWORD=mypassword \
  -p 3306:3306 \
  mysql:latest</pre>
    <hr/>
    <h4>
     <strong>
      8. 常用管理命令
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        命令
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         docker ps
        </code>
       </td>
       <td>
        查看运行中的容器。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker stop mysql-container
        </code>
       </td>
       <td>
        停止容器。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker start mysql-container
        </code>
       </td>
       <td>
        启动已停止的容器。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker restart mysql-container
        </code>
       </td>
       <td>
        重启容器。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker logs mysql-container
        </code>
       </td>
       <td>
        查看容器日志。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker rm mysql-container
        </code>
       </td>
       <td>
        删除容器（需先停止）。
       </td>
      </tr>
      <tr>
       <td>
        <code>
         docker rmi mysql:latest
        </code>
       </td>
       <td>
        删除镜像。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      9. 使用 Docker Compose 部署
     </strong>
    </h4>
    <p>
     对于复杂场景（如多容器应用），推荐使用 Docker Compose。
    </p>
    <h5>
     <strong>
      <code>
       docker-compose.yml
      </code>
      示例
     </strong>
    </h5>
    <p>
     yaml
    </p>
    <p>
     复制
    </p>
    <pre>version: '3.8'
services:
  mysql:
    image: mysql:latest
    container_name: mysql-container
    environment:
      MYSQL_ROOT_PASSWORD: my-secret-pw
      MYSQL_DATABASE: mydb
      MYSQL_USER: myuser
      MYSQL_PASSWORD: mypassword
    ports:
      - "3306:3306"
    volumes:
      - /path/on/host:/var/lib/mysql</pre>
    <h5>
     <strong>
      启动服务
     </strong>
    </h5>
    <p>
     bash
    </p>
    <p>
     复制
    </p>
    <pre>docker-compose up -d</pre>
    <hr/>
    <h4>
     <strong>
      10. 总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       使用 Docker 部署 MySQL 简单高效，适合开发、测试和生产环境。
      </p>
     </li>
     <li>
      <p>
       通过挂载卷和配置文件，可实现数据持久化和自定义配置。
      </p>
     </li>
     <li>
      <p>
       对于复杂场景，推荐使用 Docker Compose 管理多容器应用。
      </p>
     </li>
    </ul>
