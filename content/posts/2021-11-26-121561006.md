    <h4>

     需求整理
    </h4>
    <h5>
     <a id="_1">
     </a>
     需求文档编辑
    </h5>
    <h6>
     <a id="_2">
     </a>
     概述：
    </h6>
    <p>
     需求文档是将概念化的想法变成可直观阅读的叙述文档。
    </p>
    <h6>
     <a id="_5">
     </a>
     具备内容：
    </h6>
    <p>
     产品描述、用户分析、场景分析、行业分析、详细功能、竞品调研。
    </p>
    <h6>
     <a id="_8">
     </a>
     文档目标：
    </h6>
    <p>
     清晰描述产品定位、用户群体、核心内容。
    </p>
    <h6>
     <a id="_11">
     </a>
     要求：
    </h6>
    <p>
     全员编辑
    </p>
    <h4>
     <a id="_16">
     </a>
     原型图编辑
    </h4>
    <h5>
     <a id="_17">
     </a>
     原型图编辑（功能流程图）
    </h5>
    <h6>
     <a id="_18">
     </a>
     概述：
    </h6>
    <p>
     将文字叙述的产品描述变为可直观参照的产品原型及逻辑图
    </p>
    <h6>
     <a id="_21">
     </a>
     图形界面推荐工具：
    </h6>
    <p>
     墨刀、Axure
    </p>
    <h6>
     <a id="_24">
     </a>
     功能流程图推荐工具：
    </h6>
    <p>
     ProcessON
    </p>
    <h6>
     <a id="_27">
     </a>
     要求
    </h6>
    <p>
     全员编辑
    </p>
    <h4>
     <a id="_32">
     </a>
     项目架构
    </h4>
    <h5>
     <a id="_33">
     </a>
     项目架构设计
    </h5>
    <h6>
     <a id="_34">
     </a>
     概述：
    </h6>
    <p>
     选择制定产品开发所需硬件、环境、技术栈、开发规范、主要类、函数等。
    </p>
    <h6>
     <a id="_37">
     </a>
     要求：
    </h6>
    <p>
     编码规范、开发环境、工具版本、编码、语言、类图、函数、数据库设计、协作工具、项目工期、验收标准、例行会议、开发日志等。
    </p>
    <h6>
     <a id="_40">
     </a>
     负责人：
    </h6>
    <p>
     技术官
    </p>
    <h6>
     <a id="_43">
     </a>
     类图工具：
    </h6>
    <p>
     ProcessON
    </p>
    <h6>
     <a id="_46">
     </a>
     协作工具：
    </h6>
    <p>
     TAPD
    </p>

     </a>

     Git

     <li>
      ​每日会议、验收任务
     </li>
     <li>
      预备方案
     </li>
     <li>
      负责人：组长、技术官
