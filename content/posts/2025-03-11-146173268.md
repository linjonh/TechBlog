    <p>
     一、微服务概念
    </p>
    <p>
     微服务是一种软件架构风格，ta是以专注于单一职责的很多小型项目为基础，组合出复杂的大型应用。
    </p>
    <p>
     微服务
     <br/>
     - 服务拆分
     <br/>
     - 远程调用
     <br/>
     - 服务治理
     <br/>
     - 请求路由
     <br/>
     - 身份认证
     <br/>
     - 配置管理
     <br/>
     - 服务保护
     <br/>
     - 分布式事务
     <br/>
     - 异步通信
     <br/>
     - 消息可靠性
     <br/>
     - 延迟消息
     <br/>
     - 分布式搜素
     <br/>
     - 倒排索引
     <br/>
     - 数据聚合
    </p>
    <p>
     单体架构
     <br/>
     单体架构：将业务的所有功能集中在一个项目中开发，打成一个包部署。
     <br/>
     优点：架构简单；部署成本低
     <br/>
     缺点：团队写作成本高；系统发布效率低；系统可用性差
     <br/>
     总结：单体架构适合开发功能相对简单，规模较小的项目。
    </p>
    <p>
     微服务架构
     <br/>
     微服务架构：是服务化思想指导下的一套最佳实践架构方案。服务化，就是把单体架构中的功能模块拆分为多个独立项目。
     <br/>
     优点：粒度小；团队自治；服务自治。
    </p>
    <p>
     <br/>
     SpringCloud
     <br/>
     SpringCloud是目前国内使用最广泛的微服务框架。
     <br/>
     SpringCloud集成了各种微服务功能组件，并基于SpringBoot实现了这些组件的自动装配，从而提供了良好的开箱即用体验。
     <br/>
     - 服务注册发现：Eureka、Nacos、Consul
     <br/>
     - 服务远程调用：OpenFeign、Dubbo
     <br/>
     - 服务链路监控：Zipkin、Sleuth
     <br/>
     - 统一配置管理：SpringCloudConfig、Nacos
     <br/>
     - 统一网关路由：SpringCloudGateway、Zuul
     <br/>
     - 流控、降级、保护：Hystix、Sentinel
    </p>
    <p>
     <br/>
     服务治理：
    </p>
    <p>
     注册服务 -&gt; 订阅服务 -&gt; 负载均衡 -&gt; 远程调用
    </p>
    <p>
     服务提供者：注册服务；心跳续约
    </p>
    <p>
     服务调用者：订阅服务；负载均衡；远程调用
    </p>
    <p>
     注册中心：推送变更
    </p>
    <p>
     <br/>
     Nacos是目前国内企业中占比最多的注册中心组件。是阿里巴巴的产品，目前已经加入SpringCloudAlibaba中。
     <br/>
     Nacos：一个更易于构建云原生应用的动态服务发现、配置管理和服务管理平台。
    </p>
