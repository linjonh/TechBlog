     <p>
      开源文化的发展历史
     </p>
     <div class="e3lan-post">
      <div>
       <span class="bt-uid-tg">
       </span>
      </div>
     </div>
     <p>
      去年，我在
      <a href="https://corporate.walmart.com/" rel="nofollow noopener noreferrer" target="_blank">
       沃尔玛总部发表了
      </a>
      有关Fuse开源集成和消息传递，使用开源的好处以及为Apache Software Foundation贡献的好处的演讲。 您可以在
      <a href="http://www.slideshare.net/rajdavies/how-opensourceworks" rel="nofollow noopener noreferrer" target="_blank">
       slideshare.net
      </a>
      上找到经过清理的版本。 我还介绍了开放源代码的历史，就像所有历史一样，没有一个“正确的”故事，但是我试图尽力提炼时间表。 如果这对其他人有用，我在这里讲故事。 您坐得舒适吗？ 然后我将开始……
     </p>
     <p>
      <a href="https://en.wikipedia.org/wiki/IBM_704" rel="nofollow noopener noreferrer" target="_blank">
       IBM 704
      </a>
      是第一台具有浮点运算硬件的量产计算机，并于1954年被引入新兴的计算机市场。IBM设法在1955年至1960年之间出售了多达123种这类野兽。IBM将这些机器与免费软件和源捆绑在一起代码– 1955年
      <br/>
      <br/>
      成立了
      <a href="http://www.share.org/" rel="nofollow noopener noreferrer" target="_blank">
       SHARE
      </a>
      组织，以允许志趣相投的个人共享和交换704机器的代码。 SHARE是第一个公开开源组织，并且仍然存在。
      <a href="//www.javacodegeeks.com/wp-content/uploads/2013/02/HistoryOfOpenSource.png" rel="nofollow" style="display:block;text-align:center;">
       <img alt="" border="0" class="alignright" height="170" src="https://i-blog.csdnimg.cn/blog_migrate/081328f61cfd3ddff6eb9bfac4f760e3.png" width="400"/>
      </a>
      在开放源代码领域，一切都非常好（尽管这对IBM竞争对手不利）–直到1969年，司法部提起诉讼，声称IBM垄断了计算机市场。 诉讼拖延了13年，但申请的结果是1969年，IBM将其软件从计算机的发行版中分离出来。 到目前为止，该软件是免费的，源代码也已分发。
     </p>
     <p>
      现在我们进入70年代至80年代，以及UNIX战争，
      <a href="https://en.wikipedia.org/wiki/Wars_of_the_Roses" rel="nofollow noopener noreferrer" target="_blank">
       与玫瑰
      </a>
      战争相比，这些战争的难处仅略少一些-因此，我将尽力提炼要点。 UNIX于1960年代在Bell Labs（当时是AT＆T的一部分）的
      <a href="https://en.wikipedia.org/wiki/Multics" rel="nofollow noopener noreferrer" target="_blank">
       Multics
      </a>
      处开始了开发。 它最初是用汇编语言编写的，并于1972年移植到C编程语言–使其变得更加可移植。 由于1950年代的一次反托拉斯案，AT＆T被禁止销售计算机和计算机软件。 UNIX是免费授予大学和教育机构的许可，并成为
      <a href="https://en.wikipedia.org/wiki/Berkeley_Software_Distribution" rel="nofollow noopener noreferrer" target="_blank">
       Berkley软件发行版
      </a>
      （BSD）-1984年由Berkley大学首次发行。
     </p>
     <p>
      在切线时间轴上，
      <a href="https://stallman.org/" rel="nofollow noopener noreferrer" target="_blank">
       Richard Stallman
      </a>
      在1983年创建了
      <a href="https://www.gnu.org/" rel="nofollow noopener noreferrer" target="_blank">
       GNU
      </a>
      （GNU不是UNIX），这将导致获得自由软件基金会和GPL（和LGPL）许可。 追溯到1984年，美国监管机构将AT＆T划分为区域电信公司（“婴儿铃”），但也允许AT＆T进入计算和软件市场，并开始出售自己的AT＆T UNIX发行版，该公司积极地这样做。 当AT＆T在90年代初起诉Berkley违反许可证时，UNIX市场陷入混乱。 在每一次冲突中，都有机会，
      <a href="https://en.wikipedia.org/wiki/Linus_Torvalds" rel="nofollow noopener noreferrer" target="_blank">
       Linus Torvalds
      </a>
      在1991年开发LINUX引发了一场开源大火，最终导致了最成功的开源公司
      <a href="https://www.redhat.com/" rel="nofollow noopener noreferrer" target="_blank">
       Red Hat的
      </a>
      诞生。
     </p>
     <p>
      Apache HTTP服务器从1994年开始要求维护旧的NCSA HTTP守护程序。 这项合作导致了
      <a href="https://www.apache.org/" rel="nofollow noopener noreferrer" target="_blank">
       Apache Software Foundation
      </a>
      于1999年
      <a href="https://www.apache.org/" rel="nofollow noopener noreferrer" target="_blank">
       成立
      </a>
      。开源的最大革命是通过创建
      <a href="https://github.com/" rel="noopener noreferrer" target="_blank">
       GitHub
      </a>
      （2008）进行的
      <a href="https://github.com/" rel="noopener noreferrer" target="_blank">
       -GitHub
      </a>
      几乎是我在开源开发中最重要的公司，这主要是因为它使用了社交网络技术来鼓励协作–在短短的几年内，它拥有超过100万用户和超过200万回购协议。 因此，我们走了一圈，根据Gartner的预测，在全球2000强企业中，有99％的企业将在关键任务软件组合中包含开源软件。
      <br/>
     </p>
