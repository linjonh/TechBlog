    <h2>
     <a id="1__1">
     </a>
     1. 概述
    </h2>
    <p>
     数据库服务器是所有应用的数据中转站，若数据库服务被恶意攻击，可能会造成数据泄露、数据丢失、数据被恶意篡改等诸多无法挽回的损失；
     <br/>
     所以，对数据库进行安全设置是每一个数据库管理人员都应掌握的知识；
    </p>
    <h2>
     <a id="2__4">
     </a>
     2. 更改登录用户验证方式
    </h2>
    <p>
     SQL Server 登录模式分为 “Windows 身份验证模式” 和 “SQL Server 身份验证模式” 两种，若在安装 SQL Server 时选择的是 “Windows 身份验证模式 ”，则 sa 登录账号被禁用；若想开启 sa 账户，则能使用 ALTER LOGIN 语句；
    </p>
    <ol>
     <li>
      <p>
       在 SQL Server 对象资源管理器中，右击服务器，属性；
       <img alt="20220327" src="https://i-blog.csdnimg.cn/blog_migrate/b1da58bc0b1bdc9d7a6beede7c841d40.png"/>
      </p>
     </li>
     <li>
      <p>
       安全性 → 服务器身份验证 ，选择新的服务器身份验证 → 确定；
       <img alt="20220327" src="https://i-blog.csdnimg.cn/blog_migrate/a2c7565c05a3bfd0483eee2723415788.png"/>
      </p>
     </li>
     <li>
      <p>
       确定，重启 SQL Server ，新的验证方式被启用；
      </p>
     </li>
    </ol>
    <h2>
     <a id="3__12">
     </a>
     3. 创建与删除登录用户
    </h2>
    <p>
     在 SQL Server 中能创建多个登录用户来访问数据库服务器，SQL Server 能对创建的登录用户做严格的设置来控制账户的访问权限、密码策略等；
     <br/>
     在 SQL Server 数据库中创建新的登录用户步骤如下：
    </p>
    <ol>
     <li>
      <p>
       在对象资源管理器中找到安全性节点，展开
       <strong>
        登录名
       </strong>
       ，选择
       <strong>
        新建登录名
       </strong>
       ；
       <img alt="20220327" src="https://i-blog.csdnimg.cn/blog_migrate/c06c15f136ec7a3f31d68f0bdef9129b.png"/>
      </p>
     </li>
     <li>
      <p>
       在
       <strong>
        登录名-新建
       </strong>
       对话框 →
       <strong>
        常规
       </strong>
       ，输入新建账户的名字 Tim →
       <strong>
        SQL Server 身份验证
       </strong>
       ，输入密码；
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/58c5adb496307668d45b0361f33cd79d.png"/>
      </p>
     </li>
    </ol>
    <p>
     若用户在上述操作中选中了
     <strong>
      强制实施密码策略
     </strong>
     ，系统就会对所设置的密码的长度组合复杂度哟一个强制性的要求，提高密码的安全性；
    </p>
    <ol start="3">
     <li>
      打开
      <strong>
       服务器角色
      </strong>
      ，对服务器角色用户向用户授予服务器范围内的安全特权方式进行设定；
      <br/>
      <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/ca3bfc23055a5275ed7186db4d63af06.png"/>
     </li>
    </ol>
    <p>
     <strong>
      注意：
     </strong>
     选中的是 pubic 特权，这是 SQL Server 中的一类默认角色，若想让角色拥有服务器管理的最高权限，可选择
     <code>
      sysadmin
     </code>
     ；
    </p>
    <ol start="4">
     <li>
      <p>
       打开
       <strong>
        用户映射
       </strong>
       → 在右上部分选中此账户能操作的数据库 → 在右下部分选中定义登陆者的角色身份；
       <br/>
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/b1fbc7b2eef509b5fb48ab8c51348b59.png"/>
      </p>
     </li>
     <li>
      <p>
       最后选择
       <strong>
        状态
       </strong>
       ，在此页中选中授予允许连接到数据库引擎和已启用登录；
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/ec34255a0a60346fbecb03ca07d060ec.png"/>
      </p>
     </li>
     <li>
      <p>
       设置完毕后单击
       <strong>
        确定
       </strong>
       ，新的登录账户创建完毕，可在登录名节点下找到新建的登录用户 Tim；
       <br/>
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/92f34968ae5ea1b5d82aeaa5b368634f.png"/>
      </p>
     </li>
    </ol>
    <p>
     若需
     <strong>
      删除
     </strong>
     某个账户，只需右击登录名，选择
     <strong>
      删除
     </strong>
     ；
     <br/>
     <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/f014da6fffb5ad2b079f5954fe6b2667.png"/>
    </p>
    <h2>
     <a id="4__38">
     </a>
     4. 创建与删除数据库用户
    </h2>
    <p>
     数据库用户是映射到登录账户上的；
     <br/>
     在 SQL Server 数据库中能为
     <strong>
      一个数据库创建多个数据库用户
     </strong>
     ，
     <br/>
     创建数据库用户的操作步骤如下所示：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        数据库
       </strong>
       →
       <strong>
        安全性
       </strong>
       →
       <strong>
        用户
       </strong>
       ，右击选择
       <strong>
        新建用户
       </strong>
       ；
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/b92da82e8d48b8209e4933b0becc1a82.png"/>
      </p>
     </li>
     <li>
      <p>
       <strong>
        常规
       </strong>
       →
       <strong>
        登录名
       </strong>
       → 输入
       <strong>
        登录名
       </strong>
       ；
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/7f796a2e33ee1dc39cd963cb3e351c9a.png"/>
      </p>
     </li>
     <li>
      <p>
       在
       <strong>
        此用户拥有的架构
       </strong>
       中勾线 db_accessadmin；
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/a5523c28f95e5f14cdae9438119010f3.png"/>
      </p>
     </li>
     <li>
      <p>
       在数据库角色成员身份中勾选 db_accessadmin；
       <br/>
       <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/f89030073f92719675aad8fd45c42c8f.png"/>
      </p>
     </li>
     <li>
      <p>
       单击
       <strong>
        确定
       </strong>
       ，即可完成数据库用户的添加；
      </p>
     </li>
    </ol>
    <p>
     若想删除数据库，只需要右击要删除的用户名，选择
     <strong>
      删除
     </strong>
     ，即可；
    </p>
    <p>
     <strong>
      注意：
     </strong>
     除了使用用户资源管理器进行创建用户之外，新建用户还能用 T-SQL 语句
     <code>
      CREATE USER
     </code>
     、
     <code>
      CREATE LOGIN
     </code>
     语句进行创建；
    </p>
    <h2>
     <a id="5__57">
     </a>
     5. 设置服务器角色权限
    </h2>
    <p>
     当几个用户需要在某个特定的数据库中执行类型的动作时，能向该数据库中添加一个角色；
     <br/>
     数据库角色指定了能访问相同数据库对象的一组数据库用户；
     <br/>
     <strong>
      固定服务器角色
     </strong>
     已经具备了执行指定操作的权限，能把其他登录名作为成员添加到固定服务器角色中，这样该登录名就能继承固定服务器角色的权限；
     <br/>
     在 SQL Server 数据库中默认的服务器角色如下：
     <br/>
     <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/38bbc970d562488ae159fb4a81952526.png"/>
    </p>
    <p>
     服务器角色的作用和权限，如下所示：
    </p>
    <ul>
     <li>
      <strong>
       bulkadmin：
      </strong>
      这个服务器角色的成员能运行 BULK INSERT 语句；这条语句运行从文本文件中将数据导入 SQL Server 数据库中，为需要执行大容量插入数据的域账户而设计；
     </li>
     <li>
      <strong>
       dbcreator：
      </strong>
      这个服务器角色成员能创建、更改、删除和还原任何数据库；这既是适合助理 DBA 的角色，也可能是适合开发人员的角色；
     </li>
     <li>
      <strong>
       diskadmin：
      </strong>
      这个服务器角色能用于管理磁盘文件，它适合助理 DBA；
     </li>
     <li>
      <strong>
       processadmin：
      </strong>
      SQL Server 数据库能够多任务化，就是能通过执行多个进程做多个事件；
     </li>
     <li>
      <strong>
       serveradmin：
      </strong>
      这个数据库角色的成员能更改服务器范围的配置选项和关闭服务器；
     </li>
     <li>
      <strong>
       setupadmin：
      </strong>
      为需要管理链接服务器和控制启动的存储过程的用户而设计，这个角色的成员能添加到 setupadmin，能增加、删除和配置服务器，并控制启动过程；
     </li>
     <li>
      <strong>
       sysadmin：
      </strong>
      这个服务器角色有权在 SQL Server 数据库中执行任何任务；
     </li>
     <li>
      <strong>
       pubilc：
      </strong>
      两大特点，一个初始状态时没有权限，二是所有的数据库用户都是它的成员；
     </li>
    </ul>
    <h3>
     <a id="51__77">
     </a>
     5.1.
     <strong>
      查看角色属性
     </strong>
    </h3>
    <p>
     要查看服务器角色的属性时，右击需要查看的角色名 →
     <strong>
      属性
     </strong>
     ；
     <img alt="20220328" src="https://i-blog.csdnimg.cn/blog_migrate/c92ad9726995c9f4cb44fb435eef8a73.png"/>
    </p>
    <h3>
     <a id="52___80">
     </a>
     5.2 操作权限
    </h3>
    <p>
     对角色权限的操作分为 3 种状态，
     <strong>
      授予
     </strong>
     、
     <strong>
      撤销
     </strong>
     、
     <strong>
      拒绝
     </strong>
     ，分别用
     <code>
      GRANT
     </code>
     、
     <code>
      REVOKE
     </code>
     、
     <code>
      DENY
     </code>
     语句进行操作；
     <br/>
     授予权限的基本语法格式如下：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">GRANT</span>
{<!-- --><span class="token keyword">ALL</span><span class="token operator">|</span>statement <span class="token punctuation">[</span><span class="token punctuation">,</span><span class="token punctuation">.</span><span class="token punctuation">.</span>n<span class="token punctuation">]</span> }
<span class="token keyword">TO</span> security_account <span class="token punctuation">[</span><span class="token punctuation">,</span><span class="token punctuation">.</span><span class="token punctuation">.</span>n<span class="token punctuation">]</span>
</code></pre>
    <p>
     <strong>
      注意：
     </strong>
     将权限授予角色，而不是单独的登录名或用户；当某个用户由其他人取代时，能从角色中删除离开的用户，并向角色中添加新用户，与该角色关联的许多权限都将自动应用于新用户；
    </p>
    <h2>
     <a id="6__92">
     </a>
     6. 密码策略
    </h2>
    <p>
     在 Windows Server 2003 或更高版本中运行时，SQL Server 数据库能使用 Windows 密码策略机制；
     <br/>
     SQL Server 数据库能将在内部使用的密码应用在 Windows Server 2003 中，两者使用相同的复杂性策略和过期策略；
     <br/>
     此功能需要通过 NetValidatePasswordPolicy API 实现，该 API 只在 Windows Server 2003 和更高版本中提供；
    </p>
    <h3>
     <a id="61__96">
     </a>
     6.1. 密码复杂性
    </h3>
    <p>
     密码复杂性策略通过增加可能密码的数量来阻止强力攻击；实施密码复杂性策略时，新密码必须符合密码不能包含全部或部分用户账户名的原则，部分账户名是指
     <code>
      3 个或 3 个以上
     </code>
     两端用
     <strong>
      “空白”
     </strong>
     （
     <strong>
      空格、制表符、回车符等
     </strong>
     ）或任何以下字符分隔的连续字母数字字符：
     <strong>
      逗号（,）、句号（.）、连字符（-)、下划线（_）或数字字符号（#）
     </strong>
     ；
    </p>
    <p>
     密码的设置时要注意：
    </p>
    <ul>
     <li>
      密码长度至少为 8 个字符；
     </li>
     <li>
      密码包含以下 4 类字符中的 3 类：拉丁文大写字母（A
      <sub>
       Z）、拉丁文小写字母（a
      </sub>
      z）、10 个基本数字（0~9）；
     </li>
     <li>
      非字母数字字符，如感叹号（!）、美元符号（$）、数字字符（#）或百分号（%）；
     </li>
    </ul>
    <p>
     密码最长可为 128 个字符；使用的密码应尽可能长、尽可能复杂；
    </p>
    <h3>
     <a id="62__107">
     </a>
     6.2. 密码过期
    </h3>
    <p>
     密码过期策略用于管理密码的使用过程；若 SQL Server 数据库中试试密码过期策略，则系统将提醒用户更改旧密码，并禁用带有过期密码的账户；
     <br/>
     用户能为每个 SQL Server 登录名单独配置密码策略实施，通过使用
     <code>
      ALTER LOGIN(T-SQL)
     </code>
     来配置 SQL Server 登录名的密码策略选项；
    </p>
    <p>
     配置密码策略实施时，适用以下规则，
    </p>
    <ol>
     <li>
      若
      <code>
       CHECK_POLICY
      </code>
      改为
      <strong>
       ON
      </strong>
      ，则将出现以下行为：
      <br/>
      除非将 CHECK_EXPIRATION 显示地设置为 OFF，否则也会将其设置为 ON；
      <br/>
      密码历史使用当前的密码哈希值初始化；
     </li>
     <li>
      若
      <code>
       CHECK_POLICY
      </code>
      改为
      <strong>
       OFF
      </strong>
      ，则将出现以下行为：
      <br/>
      CHECK_EXPIRATION 也设置为 OFF；
      <br/>
      清除密码历史；
      <br/>
      <code>
       lockout_time
      </code>
      的值被重置；
     </li>
    </ol>
    <p>
     若指定
     <code>
      MUST_cHANGE
     </code>
     ，则
     <code>
      CHECK_EXPIRATION
     </code>
     和
     <code>
      CHECK_POLICY
     </code>
     必须设置为
     <strong>
      ON
     </strong>
     ；否则，该语句将失败；
     <br/>
     若
     <code>
      CHECK_POLICY
     </code>
     设置为
     <strong>
      OFF
     </strong>
     ，则
     <code>
      CHECK_EXPIRATION
     </code>
     不能设置为
     <strong>
      ON
     </strong>
     ，包含此选项组合的
     <code>
      ALTER LOGIN
     </code>
     语句将失败；
    </p>
