    <h2>
     <a id="Linux_docker_1">
     </a>
     Linux 安装升级docker
    </h2>
    <h3>
     <a id="_2">
     </a>
     背景
    </h3>
    <ul>
     <li>
      centos系统
     </li>
     <li>
      旧版本
      <code>
       docker --version
      </code>
      Docker version 20.10.18, build b40c2f6
     </li>
    </ul>
    <h3>
     <a id="_7">
     </a>
     升级
    </h3>
    <p>
     如果没有旧版本可以跳过升级部分直接看安装即可
    </p>
    <h4>
     <a id="docker_9">
     </a>
     停止docker服务
    </h4>
    <p>
     首先先把服务停止了，不要直接卸载
     <br/>
     <code>
      systemctl stop docker
     </code>
    </p>
    <h4>
     <a id="docker_12">
     </a>
     备份原docker数据目录
    </h4>
    <p>
     把默认的docker目录改一下名称。
     <br/>
     <code>
      mv /var/lib/docker /var/lib/docker-bak
     </code>
    </p>
    <h4>
     <a id="docker_15">
     </a>
     移除旧版本docker
    </h4>
    <p>
     现在就可以安全地删除旧版本的docker服务了。
    </p>
    <pre><code>yum -y remove docker
yum -y remove docker-client
yum -y remove docker-common
</code></pre>
    <p>
     或者
    </p>
    <pre><code>yum -y remove docker*
</code></pre>
    <h4>
     <a id="docker_ce_27">
     </a>
     安装docker ce
    </h4>
    <pre><code>yum -y install yum-utils
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
yum -y install docker-ce-18.06.3.ce-3.el7
</code></pre>
    <h4>
     <a id="_34">
     </a>
     恢复数据目录
    </h4>
    <p>
     安装完docker-ce后，系统会创建新的docker目录，删除新的，如何把备份的改回docker名称。
     <br/>
     切记：不要启动docker！！
    </p>
    <pre><code>rm -rf /var/lib/docker
mv /var/lib/docker-bak /var/lib/docker
</code></pre>
    <h4>
     <a id="docker_41">
     </a>
     启动docker
    </h4>
    <pre><code>systemctl start docker
</code></pre>
    <h4>
     <a id="_45">
     </a>
     参考
    </h4>
    <p>
     <a href="https://blog.csdn.net/weixin_43431218/article/details/135107454">
      升级docker或者docker到docker-ce完全保留镜像和容器，不影响原容器使用方法
     </a>
    </p>
    <h3>
     <a id="_47">
     </a>
     安装
    </h3>
    <h4>
     <a id="docker_48">
     </a>
     找到docker官网
    </h4>
    <p>
     <a href="https://www.docker.com/" rel="nofollow">
      官网
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/af0f846fa55c40b29c0712c7714f0464.png"/>
    </p>
    <h4>
     <a id="docker_52">
     </a>
     找到docker文档
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fc544891131b4bcebf23b9985ff332b5.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d1c4e2da998c4e56b0324d5ea4b08561.png"/>
     </img>
    </p>
    <h4>
     <a id="docker_57">
     </a>
     删除旧版本docker
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d529a8377d224638a81e10d7ad49ba78.png"/>
    </p>
    <h4>
     <a id="docker_yum_60">
     </a>
     配置docker yum源
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/74e39939c32f46c8b70d2cc6400efef5.png">
      <br/>
      当我们使用官网命令安装时可以看到这里有两个命令，第一个是
     </img>
    </p>
    <pre><code>sudo dnf -y install dnf-plugins-core
</code></pre>
    <p>
     这是安装软件包（提供用于管理 您的 DNF 存储库），第二个是
    </p>
    <pre><code>sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
</code></pre>
    <p>
     设置存储库的地址，由于这个地址是docker的官网地址，下载比较慢，因此我们在这里更换一个国内镜像地址
    </p>
    <pre><code># 配置docker yum源。
sudo yum-config-manager \
--add-repo \
http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
</code></pre>
    <p>
     这两个命令有啥区别？
     <br/>
     ps 这里的官网图片突然变成中文是因为我用浏览器翻译了一下，并不是有中文官网
    </p>
    <h4>
     <a id="docker_79">
     </a>
     参考官网继续安装docker
    </h4>
    <pre><code># 安装 最新 docker
sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
</code></pre>
    <p>
     这里一共安装了
    </p>
    <ul>
     <li>
      <code>
       docker-ce
      </code>
      docker 引擎
     </li>
     <li>
      <code>
       docker-ce-cli
      </code>
      docker引擎的命令行程序，cli=client 客户端的意思
     </li>
     <li>
      <code>
       containerd
      </code>
      docker运行时容器环境
     </li>
     <li>
      <code>
       docker-build-plugin
      </code>
      docker用来构建镜像的插件库
     </li>
     <li>
      <code>
       docker-compose-plugin
      </code>
      docker用来做批量的工具库
     </li>
    </ul>
    <p>
     sudo 使用超级管理员权限，如果当前登录的是root用户则不加sudo也可以
    </p>
    <h4>
     <a id="_92">
     </a>
     设置开机自启
    </h4>
    <pre><code># 启动&amp; 开机启动docker； enable + start 二合一
systemctl enable docker --now
</code></pre>
    <h4>
     <a id="_98">
     </a>
     配置加速
    </h4>
    <pre><code># 配置加速
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json &lt;&lt;-'EOF'
{
    "registry-mirrors": [
        "https://mirror.ccs.tencentyun.com",
        "https://docker.m.daocloud.io"
    ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
</code></pre>
    <p>
     docker 下载镜像默认从docker hub下载，速度非常慢，我们这里换一下镜像源
    </p>
    <h4>
     <a id="_114">
     </a>
     测试
    </h4>
    <p>
     <code>
      docker ps
     </code>
     或者
     <code>
      docker --version
     </code>
     或者
     <code>
      sudo docker run hello-world
     </code>
