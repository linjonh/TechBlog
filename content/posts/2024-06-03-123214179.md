    <h2>
     <a id="_6">
     </a>
     前言
    </h2>
    <p>
     <font color="#999AAA">
      自然语言处理(Natural Language Processing，NLP)是一门融合了计算机科学、人工智能以及语言学的交叉学科，这门学科研究的是如何通过机器学习等技术,让计算机学会处理人类语言，乃至实现终极目标——理解人类语言或人工智能。
      <br/>
     </font>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4b30638accb220d906fab54176a0cbfd.png"/>
    </p>
    <hr color="#000000" size='1"'/>
    <h2>
     <a id="1__15">
     </a>
     1. 自然语言与编程语言
    </h2>
    <ol>
     <li>
      词汇量
     </li>
    </ol>
    <ul>
     <li>
      C语言32个关键字
     </li>
     <li>
      Java语言有50个关键字
     </li>
    </ul>
    <ol start="2">
     <li>
      结构化
     </li>
    </ol>
    <p>
     自然语言是非结构化的，而编程语言是结构化的。
    </p>
    <pre><code class="prism language-python"><span class="token keyword">class</span> <span class="token class-name">Company</span><span class="token punctuation">(</span><span class="token builtin">object</span><span class="token punctuation">)</span><span class="token punctuation">:</span>
    <span class="token keyword">def</span> <span class="token function">__init__</span><span class="token punctuation">(</span>self<span class="token punctuation">,</span> founder<span class="token punctuation">,</span> logo<span class="token punctuation">)</span> <span class="token operator">-</span><span class="token operator">&gt;</span> <span class="token boolean">None</span><span class="token punctuation">:</span>
    	self<span class="token punctuation">.</span>founder <span class="token operator">=</span> founder
    	self<span class="token punctuation">.</span>logo <span class="token operator">=</span> logo
    	apple <span class="token operator">=</span> Company<span class="token punctuation">(</span>founder<span class="token operator">=</span><span class="token string">'乔布斯'</span><span class="token punctuation">,</span> logo<span class="token operator">=</span><span class="token string">'apple'</span><span class="token punctuation">)</span>
</code></pre>
    <ol start="3">
     <li>
      歧义性
     </li>
    </ol>
    <p>
     他说：“她这个人真有意思（funny）。”她说：“他这个人怪有意思的（funny）。”于是人们以为他们有了意思（wish），并让他向她意思意思（express）。他火了：“我根本没有那个意思（thought）！”她也生气了：“你们这么说是什么意思（intention）？”事后有人说：“真有意思（funny）。”也有人说：“真没意思（nonsense）”。（原文见《生活报》1994.11.13.第六版）［吴尉天，1999］
     <br/>
     4. 容错性
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0a649aac2034afa2fae5e254ea3bb227.png"/>
    </p>
    <ol start="5">
     <li>
      <p>
       易变性
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a651d60c44b6f12bb7ef07e62cd830de.png"/>
      </p>
     </li>
     <li>
      <p>
       简略性
       <br/>
       由于说话速度和听话速度、书写速度和阅读速度的限制，人类语言往往简洁、干练。我们经常省略大量背景知识或常识。
      </p>
     </li>
    </ol>
    <ul>
     <li>
      比如我们会对朋友说“老地方见"，而不必指出“老地方”在哪里。
     </li>
     <li>
      对于机构名称，我们经常使用简称，比如“工行”“地税局”，假定对方熟悉该简称。
     </li>
     <li>
      如果上文提出一个对象作为话题，则下文经常使用代词。
     </li>
     <li>
      在连续的新闻报道或者一本书的某一页中,并不需要重复前面的事实，而假定读者已经熟知。
     </li>
    </ul>

     2. 自然语言处理层次
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/377d8fbd00c0871a2ca95d8579d9bc43.png"/>

    <h3>
     <a id="21__49">
     </a>
     2.1 语音、图像和文本
    </h3>
    <p>
     自然语言处理系统的输入源一共有三个，即语音、图像与文本。
    </p>
    <ul>
     <li>
      通过
      <strong>
       语音识别
      </strong>
      将语音转化为文本
     </li>
     <li>
      通过
      <strong>
       光学字符
      </strong>
      将图像转化为文本
     </li>
    </ul>
    <h3>
     <a id="22__54">
     </a>
     2.2 中文分词、词性标注和命名实体识别
    </h3>
    <p>
     这3个任务都是围绕词语进行的分析，所以统称
     <strong>
      词法分析
     </strong>
     。
     <br/>
     词法分析的主要任务：
    </p>
    <ul>
     <li>
      将文本分隔为有意义的词语（中文分词)
     </li>
     <li>
      确定每个词语的类别和浅层的歧义消除（词性标注)
     </li>
     <li>
      并且识别出一些较长的专有名词（命名实体识别)。
     </li>
    </ul>
    <h3>
     <a id="23__61">
     </a>
     2.3 信息抽取
    </h3>
    <p>
     经过词法分析后，文本已经呈现出部分结构化的趋势。
     <br/>
     根据这些单词和标签抽取一部分有用的信息
    </p>
    <ul>
     <li>
      关键词
     </li>
     <li>
      实体抽取
     </li>
     <li>
      关系抽取
     </li>
    </ul>
    <h3>
     <a id="24__67">
     </a>
     2.4 文本分类与文本聚类
    </h3>
    <p>
     <strong>
      文本分类
     </strong>
     ：想知道一段话是褒义还是贬义的，判断一封邮件是否是垃圾邮件，想把许多文档分门别类地整理一下。
     <br/>
     <strong>
      文本聚类
     </strong>
     ：只想把相似的文本归档到一起，或者排除重复的文档，而不关心具体类别。
    </p>

     <a id="25__71">

    <h3>
     <a id="26__75">
     </a>
     2.6 语义分析与篇章分析
    </h3>
    <ul>
     <li>
      词义消歧（确定一个词在语境中的含义，而不是简单的词性）
     </li>
     <li>
      语义角色标注（标注中句子中的谓语与其它部分的关系）
     </li>
     <li>
      语义依存分析（分析句子中词语之间的语义关系）
     </li>

    <h2>
     <a id="5__93">
     </a>
     5. 语料库
    </h2>
    <p>
     语料库作为自然语言处理领域中的数据集，是我们教机器理解语言不可或缺的习题集。
     <br/>
     <strong>
      中文分词语料库
     </strong>
     ：由人工正确切分后的句子集合
     <br/>
     在2005年的第二届国际中文分词比赛中，曾经公开过约1个月份的语料。其中的一句样例为:
     <br/>
     先有通货膨胀干扰，后有通货紧缩叫板。
    </p>
    <p>
     <strong>
      词性标注语料库
     </strong>
     ：切分并为每个词语指定一个词性的语料
     <br/>
     <strong>
      命名实体识别语料库
     </strong>
     ：人工标注了文本内部制作者关心的实体名词以及实体类别
    </p>
