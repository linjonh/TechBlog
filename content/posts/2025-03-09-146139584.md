    <h4>
     1.创建项目
    </h4>
    <pre><code>npx create-react-app my-app --template typescript</code></pre>
    <h4>
     2.
     <strong>
      安装依赖
     </strong>
    </h4>
    <p>
    </p>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         eslint
        </code>
       </strong>
       ：核心代码质量工具。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         prettier
        </code>
       </strong>
       ：代码格式化工具。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         eslint-plugin-prettier
        </code>
       </strong>
       ：将 Prettier 的规则集成到 ESLint 中。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         eslint-config-prettier
        </code>
       </strong>
       ：禁用与 Prettier 冲突的 ESLint 规则。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         @typescript-eslint/parser
        </code>
       </strong>
       ：TypeScript 的 ESLint 解析器。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         @typescript-eslint/eslint-plugin
        </code>
       </strong>
       ：为 TypeScript 提供额外的 ESLint 规则。
      </p>
     </li>
    </ul>
    <pre><code>npm install --save-dev eslint prettier eslint-plugin-prettier eslint-config-prettier @typescript-eslint/parser @typescript-eslint/eslint-plugin</code></pre>
    <h4>
     3.
     <strong>
      配置 ESLint
     </strong>
    </h4>
    <p>
     在项目根目录下创建
     <code>
      .eslintrc.js
     </code>
     文件，配置 ESLint 以支持 Prettier 和 TypeScript：
    </p>
    <pre><code>module.exports = {
  root: true, // 确保 ESLint 不会向上查找配置文件
  parser: '@typescript-eslint/parser', // 使用 TypeScript 解析器
  parserOptions: {
    ecmaVersion: 2020, // 支持 ES2020 语法
    sourceType: 'module', // 支持 ES 模块
    ecmaFeatures: {
      jsx: true, // 支持 JSX
    },
  },
  env: {
    browser: true, // 浏览器环境
    node: true, // Node.js 环境
    es6: true, // ES6+ 语法支持
  },
  extends: [
    'eslint:recommended', // ESLint 推荐规则
    'plugin:@typescript-eslint/recommended', // TypeScript 推荐规则
    'plugin:react/recommended', // React 推荐规则
    'plugin:react/jsx-runtime', // 支持 React 17+ 的 JSX 运行时
    'plugin:prettier/recommended', // Prettier 推荐规则
    'prettier', // 禁用与 Prettier 冲突的 ESLint 规则
  ],
  plugins: [
    'react', // React 插件
    'prettier', // Prettier 插件
    '@typescript-eslint', // TypeScript ESLint 插件
  ],
  rules: {
    // ESLint 规则
    'no-unused-vars': 'warn', // 允许未使用的变量，但发出警告
    'no-console': 'warn', // 允许 console，但发出警告
    'react/react-in-jsx-scope': 'off', // React 17+ 不需要此规则
    'react/jsx-uses-react': 'off', // React 17+ 不需要此规则
    'react/jsx-filename-extension': [1, { extensions: ['.tsx', '.jsx'] }], // 允许在 JSX 文件中使用 TS/JS

    // Prettier 规则
    'prettier/prettier': [
      'error',
      {
        singleQuote: true, // 使用单引号
        semi: true, // 使用分号
        trailingComma: 'all', // 在对象或数组的最后一个元素后添加逗号
        printWidth: 80, // 每行的最大字符数
        tabWidth: 2, // 缩进宽度
        useTabs: false, // 不使用制表符缩进
        endOfLine: 'auto', // 自动处理换行符
      },
    ],
  },
  settings: {
    react: {
      version: 'detect', // 自动检测 React 版本
    },
  },
};
</code></pre>
    <h4>
     4.
     <strong>
      配置 Prettier
     </strong>
    </h4>
    <p>
     在项目根目录下创建
     <code>
      .prettierrc.js
     </code>
     文件，配置 Prettier 的格式化规则：
    </p>
    <pre><code>module.exports = {
  singleQuote: true, // 使用单引号
  semi: true, // 使用分号
  trailingComma: 'all', // 在对象或数组的最后一个元素后添加逗号
  printWidth: 80, // 每行的最大字符数
  tabWidth: 2, // 缩进宽度
  useTabs: false, // 不使用制表符缩进
  jsxSingleQuote: true, // JSX 中使用单引号
  bracketSpacing: true, // 对象字面量的括号是否换行
  jsxBracketSameLine: false, // JSX 的闭合括号是否在同一行
  endOfLine: 'auto' // 自动处理换行符
};</code></pre>
    <h4>
     5.
     <strong>
      配置 VS Code（可选）
     </strong>
    </h4>
    <p>
    </p>
    <p>
     如果你使用的是 VS Code，可以通过以下配置自动格式化代码：
    </p>
    <h5>
     <strong>
      安装插件
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        ESLint 插件
       </strong>
       ：自动检查代码质量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Prettier 插件
       </strong>
       ：自动格式化代码。
      </p>
     </li>
    </ol>
    <h5>
     <strong>
      配置
      <code>
       .vscode/settings.json
      </code>
     </strong>
    </h5>
    <p>
     在项目根目录下创建
     <code>
      .vscode
     </code>
     文件夹，并在其中创建
     <code>
      settings.json
     </code>
     文件：
    </p>
    <pre><code>{
  "editor.formatOnSave": true, // 保存时自动格式化
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true // 保存时自动修复 ESLint 问题
  },
  "prettier.configPath": "./.prettierrc.js", // 指定 Prettier 配置文件路径
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode" // 设置 Prettier 为默认格式化工具
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[javascriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}</code></pre>
    <h4>
     6.
     <strong>
      测试配置
     </strong>
    </h4>
    <pre><code>npx eslint . --ext .ts,.tsx,.js,.jsx --fix
npx prettier . --write</code></pre>
    <h4 style="background-color:transparent">
     <strong>
      7. 配置 Husky 和 Lint-staged（可选）
     </strong>
    </h4>
    <p>
     为了确保提交的代码符合格式要求，可以使用
     <code>
      husky
     </code>
     和
     <code>
      lint-staged
     </code>
     在提交前自动运行 ESLint 和 Prettier。
    </p>
    <pre><code>npm install --save-dev husky lint-staged</code></pre>
    <p>
     在
     <code>
      package.json
     </code>
     中添加以下配置：
    </p>
    <pre><code>{
  "husky": {
    "hooks": {
      "pre-commit": "lint-staged"
    }
  },
  "lint-staged": {
    "*.{ts,tsx,js,jsx,json,css,scss,md}": [
      "eslint --fix",
      "prettier --write",
      "git add"
    ]
  }
}</code></pre>
    <p>
     检查配置是否生效
    </p>
    <pre><code>//都得代码在暂存区才能用下面方法测试
npx lint-staged
</code></pre>
    <p>
     如果没有生效？？？？
    </p>
    <h5>
     1.用 npx husky --version  检查，如何显示如下需要安装 npx husky install
    </h5>
    <p>
     Usage:
     <br/>
     husky install [dir] (default: .husky)
     <br/>
     husky uninstall
     <br/>
     husky set|add &lt;file&gt; [cmd]
    </p>
    <h5>
     2.然后关联 npx husky add .husky/pre-commit "npx lint-staged"
    </h5>
    <p>
    </p>
