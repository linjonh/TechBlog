    <p>
    </p>
    <p>
    </p>
    <p>
     本文将详细介绍，在目标检测模型（如YOLO、Faster R-CNN或SSD）中的核心核心评估指标mAP的相关概念。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/img_convert/5f082f303df05b6d686bef3f40d9e383.jpeg"/>
    </p>
    <h2>
     <a id="_71">
     </a>
     目标检测比分类更难
    </h2>
    <p>
     在分类问题中，你只需预测一个标签。在目标检测中，必须：
    </p>
    <ul>
     <li>
      <strong>
       找到对象所在的位置（定位：绘制边界框）。
      </strong>
     </li>
     <li>
      <strong>
       确定对象是什么分类。
      </strong>
     </li>
    </ul>
    <p>
     其评估指标包括
     <strong>
      精确度
     </strong>
     、
     <strong>
      召回率
     </strong>
     和
     <strong>
      mAP
     </strong>
     。
    </p>
    <h2>
     <a id="_80">
     </a>
     精确度与召回率
    </h2>
    <h3>
     <a id="_82">
     </a>
     <strong>
      精确度
     </strong>
    </h3>
    <p>
     定义：
     <strong>
      在你的模型检测到的所有物体中，有多少是正确的？
     </strong>
    </p>
    <p>
     公式：
     <code>
      True Positives / (True Positives + False Positives)
     </code>
    </p>
    <h3>
     <a id="_88">
     </a>
     <strong>
      召回率
     </strong>
    </h3>
    <p>
     用于评估你的模型有多全面。定义：
     <strong>
      在所有存在的物体中，你的模型找到了多少？
     </strong>
    </p>
    <p>
     公式：
     <code>
      True Positives / (True Positives + False Negatives)
     </code>
    </p>
    <p>
     但是，仅仅靠精确度和召回率并不能说明全部问题。如果你的模型很擅长找物体，但画检测框的时候很糟糕怎么办？
    </p>
    <blockquote>
     <p>
      <strong>
       这就是Intersection over Union（IOU）的由来。
      </strong>
     </p>
    </blockquote>
    <h2>
     <a id="IoU_98">
     </a>
     IoU：检测质量的衡量标准
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a56ce6dfc4fa44f5a422b1f8b4a860e0.png">
      <br/>
      交并比（IoU）是衡量预测边界框与真实框（实际对象位置）对齐的准确程度的
      <em>
       关键指标
      </em>
      。它的工作原理如下：
     </img>
    </p>
    <h3>
     <a id="_104">
     </a>
     计算方式
    </h3>
    <ul>
     <li>
      <strong>
       交集
      </strong>
      ：预测框和真实框之间的重叠区域。
     </li>
     <li>
      <strong>
       并集
      </strong>
      ：两个方框所覆盖的总面积。
     </li>
    </ul>
    <p>
     公式：IoU =相交面积/合并面积
    </p>
    <p>
     <strong>
      举例来说：
     </strong>
    </p>
    <ul>
     <li>
      如果你的模型预测出一个与真实框完全重叠的框，
      <strong>
       IoU = 1.0。
      </strong>
     </li>
     <li>
      如果没有重叠，
      <strong>
       IoU = 0.0。
      </strong>
     </li>
     <li>
      如果预测的盒子覆盖了一半的真实框，
      <strong>
       IoU = 0.5。
      </strong>
     </li>
    </ul>
    <h3>
     <a id="IoU_117">
     </a>
     为什么IoU阈值很重要
    </h3>
    <p>
     1.IoU阈值（例如，0.5)作为检测的通过/未通过标准：
    </p>
    <ul>
     <li>
      <strong>
       真阳性（TP）
      </strong>
      ：IoU ≥阈值（例如，≥0.5）。
     </li>
     <li>
      <strong>
       假阳性（FP）
      </strong>
      ：IoU &lt;阈值（例如，重叠面积太小了）。
     </li>
    </ul>
    <p>
     2.满足不同的定制化需求
    </p>
    <ul>
     <li>
      <em>
       mAP@0.5
      </em>
      是宽松的（框只需要50%重叠）。
     </li>
     <li>
      <em>
       mAP@0.75
      </em>
      需要精确定位（75%重叠）。
     </li>
     <li>
      <em>
       mAP@0.95
      </em>
      非常严格（用于医疗成像等安全关键任务）。
     </li>
    </ul>
    <p>
     让我们来做比喻，想想IoU的门槛，比如考试的“及格分数”：
    </p>
    <ul>
     <li>
      阈值为0.5就像得分50%通过（对大多数情况来说足够好）。
     </li>
     <li>
      0.9的阈值就像需要90%才能通过（为精英表现保留）。
     </li>
    </ul>
    <p>
     那么我们该如何解释模型的性能？我有Precision、Recall和IoU，但我该如何处理它们？
    </p>
    <blockquote>
     <p>
      这就是平均精度（AP）的作用。
     </p>
    </blockquote>
    <h2>
     <a id="AP_139">
     </a>
     平均精度（AP）：曲线下面积
    </h2>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/img_convert/3483723631ff050f8bfac2efe15bbd73.png"/>
    </p>
    <p>
     对于单个类（例如，“cat”），这就是你如何找到AP（精确率-召回曲线下的面积）：
    </p>
    <p>
     一.按置信度对检测进行排序：从模型最高置信度的预测开始。
    </p>
    <p>
     二.计算每一步的精确率召回率：当你沿着列表往下走（降低置信阈值）时，你：
    </p>
    <ul>
     <li>
      <strong>
       提高召回率
      </strong>
      （找到更多对象，但有更多误报风险）。
     </li>
     <li>
      <strong>
       降低精确度
      </strong>
      （更多检测，但有些可能是错误的）。
     </li>
    </ul>
    <p>
     三.绘制精确-召回（PR）曲线：
    </p>
    <ul>
     <li>
      X轴=召回率（0到1）。
     </li>
     <li>
      Y轴=精确度（0到1）。
     </li>
     <li>
      一个完美的模特有一个拥抱右上角的PR曲线。
     </li>
    </ul>
    <p>
     四. 计算AP（PR曲线下面积）
    </p>
    <ul>
     <li>
      AP将整个PR曲线总结为一个数字（0到1）。
     </li>
    </ul>
    <p>
     <strong>
      对于平均精确度，AUC计算如下：
     </strong>
    </p>
    <ul>
     <li>
      PR曲线通过在固定的召回率水平上插入精确度来“平滑”。
     </li>
     <li>
      AP = 11个等间隔召回点（0.0，0.1，…，1.0）处的精确度值的平均值。
     </li>
     <li>
      更简单的方法：使用原始PR曲线下的积分（面积）。
     </li>
    </ul>
    <blockquote>
     <p>
      完美的
      <strong>
       PR曲线下面积
      </strong>
      = AP = 1.0（在所有召回水平下100%的准确率）。
     </p>
    </blockquote>
    <h3>
     <a id="AP_172">
     </a>
     为什么AP很重要
    </h3>
    <p>
     <strong>
      一.平衡精确度和召回率
     </strong>
     ：高AP意味着模型：
    </p>
    <ul>
     <li>
      检测大多数对象（高召回率）。
     </li>
     <li>
      很少出错（高精度）。
     </li>
    </ul>
    <p>
     <strong>
      二.可观察不同类别性能
     </strong>
     ：AP是按类计算的。“猫”的低AP意味着你的模型对猫的检测效果不佳。
    </p>
    <p>
     <strong>
      三.身份不可知
     </strong>
     ：与固定阈值度量（例如，准确性），AP在
     <em>
      所有置信水平
     </em>
     上评估性能。
    </p>
    <p>
     <strong>
      举例来说
     </strong>
     ：
    </p>
    <p>
     <strong>
      高平均精度（例如，0.9）
     </strong>
    </p>
    <ul>
     <li>
      在
      <em>
       每一个召回级别
      </em>
      ，精确度都很高。
     </li>
     <li>
      如果模型检测到90%的对象（召回率=0.9），精确度仍然是90%。
     </li>
    </ul>
    <p>
     <strong>
      低平均精度（例如，0.3）
     </strong>
     ：
    </p>
    <ul>
     <li>
      随着召回率的增加，精确度急剧下降。
     </li>
     <li>
      检测到80%的对象（召回率=0.8）可能意味着准确率下降到20%。
     </li>
    </ul>
    <h2>
     <a id="mAPMean_Average_Precision_195">
     </a>
     mAP（Mean Average Precision）
    </h2>
    <p>
     <strong>
      mAP（Mean Average Precision）
     </strong>
     是
     <em>
      所有类别
     </em>
     的平均值。
    </p>
    <ul>
     <li>
      例如：如果模型检测到猫、狗和汽车，则mAP =（AP_cat + AP_dog + AP_car）/ 3。
     </li>
    </ul>
    <h2>
     <a id="mAP05mAP095_202">
     </a>
     mAP@0.5与mAP@0.95区别
    </h2>
    <p>
     <strong>
      mAP@0.5
     </strong>
     ：
    </p>
    <ul>
     <li>
      使用宽松的IoU阈值（50%重叠）。
     </li>
     <li>
      在通用检测中常见（例如，PASCAL VOC数据集）。
     </li>
     <li>
      支持检测对象的模型*，即使检测框稍微偏离*。
     </li>
    </ul>
    <p>
     <strong>
      mAP@0.95
     </strong>
     ：
    </p>
    <ul>
     <li>
      使用严格的IoU阈值（95%重叠）。
     </li>
     <li>
      支持具有
      <em>
       接近完美定位的
      </em>
      模型。
     </li>
     <li>
      用于高风险应用（例如，医学成像、机器人学）。
     </li>
    </ul>
    <p>
     <strong>
      COCO mAP
     </strong>
     ：跨0.5到0.95（增量为0.05）的IoU阈值的mAP平均值。这是严与宽的平衡。
    </p>
    <h2>
     <a id="mAP_218">
     </a>
     为什么mAP是最终的评估指标
    </h2>
    <ol>
     <li>
      <strong>
       平衡精确召回
      </strong>
      ：与准确性不同，mAP会惩罚错过对象（低召回）或垃圾邮件错误检测（低精度）的模型。
     </li>
     <li>
      <strong>
       根据需求设定IOU
      </strong>
      ：通过使用IoU，mAP确保检测框不仅“足够好”，而且
      <em>
       与您的门槛要求一样精确
      </em>
      。
     </li>
    </ol>
    <p>
     对于优先考虑速度
     <em>
      和
     </em>
     准确性的YOLO模型，通过mAP可以知道：
    </p>
    <ul>
     <li>
      检测的可靠性（精度）。
     </li>
     <li>
      错过的物体有多少（召回）。
     </li>
     <li>
      边界框有多贴合（IoU）。
     </li>
    </ul>
    <h2>
     <a id="_230">
     </a>
     结论
    </h2>
    <p>
     目标检测是一项复杂的任务，评估其性能需要一个平衡精度，召回率和定位准确性的指标。这就是
     <strong>
      mAP
     </strong>
     的闪光点。它不仅仅是一个数字-它是一个全面的衡量模型检测对象、绘制边界框和处理多个类的能力的指标。
    </p>
    <p>
     无论您使用的是YOLO、Faster R-CNN还是任何其他对象检测框架，mAP都能为您提供一个可靠的指标来比较模型、调整超参数，并将性能提升到一个新的水平。通过
     <strong>
      mAP@0.5
     </strong>
     和
     <strong>
      mAP@0.95
     </strong>
     等变量，您可以定制评估，以满足特定应用的精度要求。
    </p>
