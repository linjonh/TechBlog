    <blockquote>
     <p>
      在数据库流水线开发中，代码复用与动态配置是提升效率的核心诉求。SQLMesh以其独特的宏系统与用户定义变量机制，重新定义了SQL生成的灵活性。与传统模板引擎不同，SQLMesh的宏并非简单的字符串替换，而是基于语义理解的智能代码重构——通过
      <code>
       sqlglot
      </code>
      库解析SQL结构，结合Python逻辑处理能力，让用户能够以声明式语法实现复杂查询的动态组装。
     </p>
    </blockquote>
    <h3>
     <a id="_6">
     </a>
     引言
    </h3>
    <p>
     无论是全局配置、网关级参数还是模型内局部变量，SQLMesh的三层变量体系（Global &gt; Gateway &gt; Local）提供了细粒度的数据管控能力。通过
     <code>
      @DEF
     </code>
     宏定义和
     <code>
      @VAR()
     </code>
     函数调用，开发者可以轻松将业务规则、环境参数与SQL逻辑解耦，实现“一次编写，多环境适配”。而宏函数的引入更进一步，允许将条件判断、数学计算等逻辑封装为可复用的代码片段，彻底告别重复代码。例如，通过嵌套宏函数
     <code>
      @container_volume
     </code>
     调用
     <code>
      @area
     </code>
     ，SQLMesh不仅生成最终SQL，更赋予代码可读性与可维护性。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3a28af92b5144ff1b6a885505319d027.png"/>
    </p>
    <h3>
     <a id="_12">
     </a>
     宏系统：两种方法
    </h3>
    <p>
     SQLMesh 中的宏与像 Jinja 这样的模板系统的宏表现不同。
     <br/>
     宏系统基于字符串替换。宏系统会扫描代码文件，识别表示宏内容的特殊字符，并将宏元素替换为其他文本。
     <br/>
     从广义上讲，这就是模板系统的全部功能。它们具有提供控制流逻辑（if-then）和其他功能的工具，但这些功能仅用于支持替换正确的字符串。
    </p>
    <p>
     模板系统有意对所使用的编程语言保持中立，大多数模板系统适用于从博客文章到 HTML 再到 SQL 的各种内容。相比之下，SQLMesh 宏专门设计用于生成 SQL 代码。它们通过使用 Python 的 sqlglot 库对正在创建的 SQL 代码进行分析，从而具有对 SQL 代码的语义理解，并且允许使用 Python 代码，这样用户就可以整洁地实现复杂的宏逻辑。
    </p>
    <h3>
     <a id="SQLMesh__20">
     </a>
     SQLMesh 宏方法
    </h3>
    <p>
     本节描述SQLMesh宏在底层是如何工作的。你可以随意跳过这一节，并在有用的时候返回。使用SQLMesh宏不需要此信息，但它对于调试任何表现出令人困惑行为的宏非常有用。
    </p>
    <p>
     SQLMesh宏方法和模板系统之间的关键区别在于字符串替换所起的作用。在模板系统中，字符串替换是全部和唯一的要点。
    </p>
    <p>
     在SQLMesh中，字符串替换只是修改SQL查询的语义表示的一个步骤。SQLMesh宏通过构建和修改SQL查询的语义表示来工作。
    </p>
    <p>
     在处理完所有非sql文本之后，它使用替换的值将查询的语义表示修改为其最终状态。
    </p>
    <p>
     它采用以下五个步骤来实现这一目标：
    </p>
    <ol>
     <li>
      <p>
       使用适当的 SQLGlot SQL 语境（例如，Postgres、BigQuery 等）解析文本。在解析过程中，它检测特殊宏符号 @ 以区分非 SQL 和 SQL 文本。解析器构建 SQL 代码结构的语义表示，将非 SQL 文本捕获为“占位符”值，以便在后续步骤中使用。
      </p>
     </li>
     <li>
      <p>
       检查占位符值，将其分类为以下类型之一：
      </p>
      <ul>
       <li>
        <p>
         使用 @DEF 操作符创建用户自定义宏变量（有关用户自定义宏变量的更多信息请参阅后面章节）
        </p>
       </li>
       <li>
        <p>
         宏变量：SQLMesh 预定义的、用户自定义的局部变量以及用户自定义的全局变量
        </p>
       </li>
       <li>
        <p>
         宏函数，包括 SQLMesh 提供的以及用户自定义的
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       替换检测到的宏变量值。在大多数情况下，这是与模板系统一样的直接字符串替换。
      </p>
     </li>
     <li>
      <p>
       执行任何宏函数并替换返回值。
      </p>
     </li>
     <li>
      <p>
       用(3)中替换的变量值和(4)中的函数修改SQL查询的语义表示。
      </p>
     </li>
    </ol>
    <h3>
     <a id="_48">
     </a>
     用户定义的变量
    </h3>
    <p>
     SQLMesh支持三种用户定义的宏变量：global、gateway和local。
    </p>
    <p>
     全局和网关宏变量在项目配置文件中定义，可以在任何项目模型中访问。局部宏变量在模型定义中定义，并且只能在该模型中访问。
    </p>
    <p>
     可以在任何或所有全局、网关和本地级别指定具有相同名称的宏变量。当在多个级别上指定变量时，最特定级别的值优先。例如，局部变量的值优先于同名网关变量的值，网关变量的值优先于全局变量的值。
    </p>
    <h4>
     <a id="_56">
     </a>
     全局变量
    </h4>
    <p>
     全局变量在项目配置文件变量键中定义。
    </p>
    <p>
     全局变量值可以是以下任何数据类型或包含这些类型的列表或字典：int， float, bool, str。
    </p>
    <p>
     在模型定义中使用 @&lt;VAR_NAME&gt; 宏或 @VAR() 函数来访问全局变量值。后者函数要求将变量名称以单引号括起作为第一个参数，并将可选的默认值作为第二个参数。默认值是一种安全机制，用于在项目配置文件中未找到变量名称的情况下使用。
     <br/>
     例如，以下这个 SQLMesh 配置键定义了六个不同数据类型的变量：
    </p>
    <pre><code class="prism language-yaml"><span class="token key atrule">variables</span><span class="token punctuation">:</span>
  <span class="token key atrule">int_var</span><span class="token punctuation">:</span> <span class="token number">1</span>
  <span class="token key atrule">float_var</span><span class="token punctuation">:</span> <span class="token number">2.0</span>
  <span class="token key atrule">bool_var</span><span class="token punctuation">:</span> <span class="token boolean important">true</span>
  <span class="token key atrule">str_var</span><span class="token punctuation">:</span> <span class="token string">"cat"</span>
  <span class="token key atrule">list_var</span><span class="token punctuation">:</span> <span class="token punctuation">[</span><span class="token number">1</span><span class="token punctuation">,</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token number">3</span><span class="token punctuation">]</span>
  <span class="token key atrule">dict_var</span><span class="token punctuation">:</span>
    <span class="token key atrule">key1</span><span class="token punctuation">:</span> <span class="token number">1</span>
    <span class="token key atrule">key2</span><span class="token punctuation">:</span> <span class="token number">2</span>
</code></pre>
    <p>
     python版本：
    </p>
    <pre><code class="prism language-python">variables <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string">"int_var"</span><span class="token punctuation">:</span> <span class="token number">1</span><span class="token punctuation">,</span>
    <span class="token string">"float_var"</span><span class="token punctuation">:</span> <span class="token number">2.0</span><span class="token punctuation">,</span>
    <span class="token string">"bool_var"</span><span class="token punctuation">:</span> <span class="token boolean">True</span><span class="token punctuation">,</span>
    <span class="token string">"str_var"</span><span class="token punctuation">:</span> <span class="token string">"cat"</span><span class="token punctuation">,</span>
    <span class="token string">"list_var"</span><span class="token punctuation">:</span> <span class="token punctuation">[</span><span class="token number">1</span><span class="token punctuation">,</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token number">3</span><span class="token punctuation">]</span><span class="token punctuation">,</span>
    <span class="token string">"dict_var"</span><span class="token punctuation">:</span> <span class="token punctuation">{<!-- --></span><span class="token string">"key1"</span><span class="token punctuation">:</span> <span class="token number">1</span><span class="token punctuation">,</span> <span class="token string">"key2"</span><span class="token punctuation">:</span> <span class="token number">2</span><span class="token punctuation">}</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>

config <span class="token operator">=</span> Config<span class="token punctuation">(</span>
    variables<span class="token operator">=</span>variables<span class="token punctuation">,</span>
    <span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span> <span class="token comment"># other Config arguments</span>
<span class="token punctuation">)</span>

</code></pre>
    <p>
     模型定义可以像这样在WHERE子句中访问int_var值：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token operator">*</span>
<span class="token keyword">FROM</span> <span class="token keyword">table</span>
<span class="token keyword">WHERE</span> int_variable <span class="token operator">=</span> <span class="token variable">@INT_VAR</span>
</code></pre>
    <p>
     或者，可以通过将变量名传递给@VAR（）宏函数来访问相同的变量。注意，在调用@VAR（‘int_var’）时，变量名是用单引号括起来的：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token operator">*</span>
<span class="token keyword">FROM</span> <span class="token keyword">table</span>
<span class="token keyword">WHERE</span> int_variable <span class="token operator">=</span> <span class="token variable">@VAR</span><span class="token punctuation">(</span><span class="token string">'int_var'</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     默认值可以作为第二个参数传递给@VAR（）宏函数，如果配置文件中缺少该变量，该参数将用作回退值。
    </p>
    <p>
     在这个例子中，WHERE子句将呈现为WHERE some_value = 0，因为在项目配置文件中没有定义名为missing_var的变量：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token operator">*</span>
<span class="token keyword">FROM</span> <span class="token keyword">table</span>
<span class="token keyword">WHERE</span> some_value <span class="token operator">=</span> <span class="token variable">@VAR</span><span class="token punctuation">(</span><span class="token string">'missing_var'</span><span class="token punctuation">,</span> <span class="token number">0</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     对于 Python 宏函数，可通过
     <code>
      evalutor.var
     </code>
     方法获取类似的 API，对于 Python 模型，则可通过
     <code>
      context.var
     </code>
     方法获取类似的 API。
    </p>
    <h4>
     <a id="_124">
     </a>
     网关变量
    </h4>
    <p>
     与全局变量一样，网关变量在项目配置文件中定义。但是，它们是在特定网关的变量键中指定的：
    </p>
    <pre><code class="prism language-yaml"><span class="token key atrule">gateways</span><span class="token punctuation">:</span>
  <span class="token key atrule">my_gateway</span><span class="token punctuation">:</span>
    <span class="token key atrule">variables</span><span class="token punctuation">:</span>
      <span class="token key atrule">int_var</span><span class="token punctuation">:</span> <span class="token number">1</span>
    <span class="token punctuation">...</span>
</code></pre>
    <p>
     python代码：
    </p>
    <pre><code class="prism language-python">gateway_variables <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  <span class="token string">"int_var"</span><span class="token punctuation">:</span> <span class="token number">1</span>
<span class="token punctuation">}</span>

config <span class="token operator">=</span> Config<span class="token punctuation">(</span>
    gateways<span class="token operator">=</span><span class="token punctuation">{<!-- --></span>
      <span class="token string">"my_gateway"</span><span class="token punctuation">:</span> GatewayConfig<span class="token punctuation">(</span>
        variables<span class="token operator">=</span>gateway_variables
        <span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span> <span class="token comment"># other GatewayConfig arguments</span>
        <span class="token punctuation">)</span><span class="token punctuation">,</span>
      <span class="token punctuation">}</span>
<span class="token punctuation">)</span>
</code></pre>
    <p>
     在模型中使用与全局变量相同的访问方法来访问它们。
     <br/>
     特定于网关的变量值优先于在根变量键中指定的具有相同名称的变量值。
    </p>
    <h4>
     <a id="_156">
     </a>
     局部变量
    </h4>
    <p>
     局部宏变量是在模型中定义的。局部变量的值优先于具有相同名称的全局变量或网关特定变量。
     <br/>
     使用 @DEF 宏运算符定义您自己的局部宏变量。例如，你可以将宏变量 macro_var 设置为值 1 的方式如下：
    </p>
    <pre><code class="prism language-sql"><span class="token variable">@DEF</span><span class="token punctuation">(</span>macro_var<span class="token punctuation">,</span> <span class="token number">1</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     SQLMesh 对使用 @DEF 运算符有着三项基本要求：
    </p>
    <ol>
     <li>
      MODEL 语句必须以分号结尾；
     </li>
     <li>
      所有 @DEF 使用项都必须置于 MODEL 语句之后、 SQL 查询语句之前。
     </li>
     <li>
      每个 @DEF 使用都必须以分号结尾；
      <br/>
      例如，请参考 SQLMesh 快速入门指南中的以下模型 sqlmesh_example.full_model：
     </li>
    </ol>
    <pre><code class="prism language-sql">MODEL <span class="token punctuation">(</span>
  name sqlmesh_example<span class="token punctuation">.</span>full_model<span class="token punctuation">,</span>
  kind <span class="token keyword">FULL</span><span class="token punctuation">,</span>
  cron <span class="token string">'@daily'</span><span class="token punctuation">,</span>
  audits <span class="token punctuation">(</span>assert_positive_order_ids<span class="token punctuation">)</span><span class="token punctuation">,</span>
<span class="token punctuation">)</span><span class="token punctuation">;</span>

<span class="token keyword">SELECT</span>
  item_id<span class="token punctuation">,</span>
  <span class="token function">count</span><span class="token punctuation">(</span><span class="token keyword">distinct</span> id<span class="token punctuation">)</span> <span class="token keyword">AS</span> num_orders<span class="token punctuation">,</span>
<span class="token keyword">FROM</span>
  sqlmesh_example<span class="token punctuation">.</span>incremental_model
<span class="token keyword">GROUP</span> <span class="token keyword">BY</span> item_id
</code></pre>
    <p>
     这个模型可以用一个用户定义的宏变量来扩展，以根据item_size过滤查询结果，如下所示：
    </p>
    <pre><code class="prism language-sql">MODEL <span class="token punctuation">(</span>
  name sqlmesh_example<span class="token punctuation">.</span>full_model<span class="token punctuation">,</span>
  kind <span class="token keyword">FULL</span><span class="token punctuation">,</span>
  cron <span class="token string">'@daily'</span><span class="token punctuation">,</span>
  audits <span class="token punctuation">(</span>assert_positive_order_ids<span class="token punctuation">)</span><span class="token punctuation">,</span>
<span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">-- NOTE: semi-colon at end of MODEL statement</span>

<span class="token variable">@DEF</span><span class="token punctuation">(</span>size<span class="token punctuation">,</span> <span class="token number">1</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">-- NOTE: semi-colon at end of @DEF operator</span>

<span class="token keyword">SELECT</span>
  item_id<span class="token punctuation">,</span>
  <span class="token function">count</span><span class="token punctuation">(</span><span class="token keyword">distinct</span> id<span class="token punctuation">)</span> <span class="token keyword">AS</span> num_orders<span class="token punctuation">,</span>
<span class="token keyword">FROM</span>
  sqlmesh_example<span class="token punctuation">.</span>incremental_model
<span class="token keyword">WHERE</span>
  item_size <span class="token operator">&gt;</span> <span class="token variable">@size</span> <span class="token comment">-- Reference to macro variable `@size` defined above with `@DEF()`</span>
<span class="token keyword">GROUP</span> <span class="token keyword">BY</span> item_id
</code></pre>
    <p>
     本例使用@DEF（size, 1）定义宏变量size。当模型运行时，SQLMesh将在where子句中出现@size的地方替换数字1。
    </p>
    <h4>
     <a id="_212">
     </a>
     宏函数
    </h4>
    <p>
     除了内联用户定义变量外，SQLMesh还支持内联宏函数。与单独使用变量相比，可以使用这些函数来表达更具可读性和可重用性的逻辑。让我们来看一个例子：
    </p>
    <pre><code class="prism language-sql">MODEL<span class="token punctuation">(</span><span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">.</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

    <h3>
     <a id="_272">
     </a>
     最后总结
    </h3>
    <p>
     SQLMesh的宏与变量系统，本质上是将SQL开发从“硬编码”推向“声明式编程”的桥梁。其核心价值体现在三方面：
    </p>
    <ol>
     <li>
      <strong>
       语义级宏处理
      </strong>
      ：通过解析SQL结构而非单纯文本替换，确保宏操作不会破坏查询逻辑，同时支持复杂函数嵌套与Python代码注入。
     </li>
     <li>
      <strong>
       三层变量优先级
      </strong>
      ：全局配置提供基础参数，网关变量细化环境差异，局部宏则实现模型级定制，层级间遵循“就近原则”，避免配置冲突。
     </li>
     <li>
      <strong>
       函数式宏编程
      </strong>
      ：通过
      <code>
       @DEF
      </code>
      定义的宏函数支持多参数、条件分支与嵌套调用，将业务逻辑封装为SQL内的“插件”，显著提升复杂查询的复用性。
     </li>
    </ol>
    <p>
     无论是处理多租户环境的动态表名替换，还是实现基于参数的查询条件分支，SQLMesh的宏系统都能以声明式语法简化开发流程。其设计哲学——
     <strong>
      “让SQL回归逻辑，让配置远离代码”
     </strong>
     ，正在成为构建灵活、可扩展数据库流水线的标配工具。
    </p>
