# 微软虚拟技术打造IT新生态

微软随Windows Server 2008推出Hyper-V虚拟服务器技术，不仅在虚拟硬件方面得到突破，同时为企业重新定义了虚拟服务平台，使虚拟化技术的应用变成企业信息化新的生态系统（应用平台）。为什么微软虚拟技术正在被企业广泛的使用，对传统虚拟软件如VMware的市场进行蚕食？我认为除了微软虚拟技术的比较优势外（这在前一篇博文已讨论过），更重要的是微软能提供除了服务器虚拟以外，更多的虚拟化服务和解决方案，比如虚拟桌面、虚拟显示、虚拟应用程序等。微软除了在服务器（包括Hyper-V虚拟技术）、数据库、开发平台软件(Visual Studio .Net)等上重拳出击外，CRM、ERP、Project、Office等企业应用系统也能良好接口和交互数据。微软可以为企业提供单一平台的绝大多数企业应用，这就是笔者看好微软虚拟技术平台的理由。

和其它虚拟技术厂商主要提供硬件虚拟平台（如服务器等）不同，微软目前已经将软件虚拟化放在了产品策略上的绝对优先地位。微软具有全面而灵活的端到端的解决方案，微软的投入不只局限在服务器虚拟化上，在展现层虚拟化、桌面虚拟化、应用虚拟化上都有投入，并通过System Center实现整合的管理，在客户中进行广泛的推动与部署。微软在虚拟化方面的优势首先是基于拥有广泛用户基础的Window平台上，并从数据中心到桌面给用户提供了灵活的全方位的灵活选择，同时，微软具有良好的合作伙伴生态圈，比如IBM、HP、SUN等已陆续宣布全面支持微软虚拟技术。微软在IT软件业的领导地位，使其它硬件商、软件商会主动向其靠拢，这和其它微软技术提供商需要寻求第三方支持和兼容完全不可同日而语。

微软虚拟技术具有整合的管理和最佳的整体拥有成本(TCO)，微软提供的不仅仅是虚拟技术，而是全面的虚拟化解决方案，这和当前企业在经济危机下开始重视“绿色IT”，尽可能提高硬件、软件的运行效率的需求十分契合。2007年IDC发布关于虚拟化的研究报告，IDC认为现在的虚拟化生态链已经成熟，“虚拟化为’绿色数据中心’提供了必要的条件，从CPU多核，到IT硬件产品的丰富化、多样化，再到软件厂商多方面的虚拟化支持，使得虚拟化的生态链系统逐渐完善与成熟。”报告中提到，全世界90%的服务器尚未采用虚拟化技术。从美国的情况看，90%的业务正在考虑虚拟化和服务器整合；50%的组织正在实施或利用虚拟化。虚拟化一方面能够减少服务器采购的数量，另一方面在减少服务器数量的同时能够降低能耗。而从产业链上看，多核计算和硬件厂商的支持等，都在促使虚拟化将被用户广泛接受。

IBM、HP、SUN等传统高端服务器商，对虚拟化的支持可谓不遗余力，为什么它们不担心虚拟化的广泛应用减少服务器订单呢？为什么他们要高调支持微软虚拟化技术？其实，除目前多核CPU的能力越来越强大外，PC Server大行其道且拼装服务器正在开始流行。微软在PC Server操作系统的垄断地位，使微软虚拟化技术的潜在用户众多。通过微软Windows Server 2008和Hyper-V，中小企业有望使用更强大的专业服务器来整合原来的PC Server以及上面的应用，降低系统软硬件采购和维护成本，提高数据安全级别。高端服务器制造商对微软虚拟技术的鼎力支持，将使企业和微软以及硬件厂商形成多赢的局面。微软虚拟技术的范围非常广泛，不只局限于服务器的虚拟化，更多的将是各种应用的虚拟化。例如在今后的环境中，如果不做虚拟，就无法运行两个版本的应用，但是如果有了应用的虚拟化，你就可以去这么做。从服务器到应用的虚拟化，微软虚拟化的核心就在于虚拟软件产品既能管理物理的也可以管理虚拟的环境，只要采用服务系统就可以实现这一目标。采用单一的产品就能实现物理和虚拟环境的管理，这将为用户带来更好的选择，他们就不需要花大量的投资去购买别的产品。

微软在虚拟化方面的视角是非常广泛的，虚拟化涵盖从PC、服务器到数据中心等的虚拟化，目前微软的重点开始转向虚拟化的管理方面。如果IT管理做的不好，实用的虚拟化的成本比不用还要高。所以微软非常重视在微拟化应用中既可以管理物理的环境，也能管理虚拟的环境。除了在虚拟化管理方面的优势外，微软的虚拟化技术的另一优势就是集成。微软技术方面最大的优点，在于有一个完整的、集成的平台，这是所有第三方产品不具备的。微软的虚拟化架构，也许单独拿出其中的某个功能或者组件，并不一定都是No.1。但只有微软才能提供完整的架构平台，企业不用担心要向多个供应商采购才能拼凑出一套完整的虚拟平台、不用担心各个组件之间是否可以整合、更不用担心各个组件之间是否可以兼容。最重要的是，因为来自微软，企业不必担心未来升级和业务的迁移问题。

笔者相信，微软虚拟技术必将在打造企业IT生态系统中发挥越来越重要的作用。
