     1.1 创建流程
    </h4>
    <p>
     device_add -》 error = device_create_file(dev, &amp;dev_attr_dev);
    </p>
    <p>
     先来看看 dev_attr_dev 的定义
     <br/>
     static DEVICE_ATTR_RO(dev);
    </p>
    <pre><code class="prism language-c"> <span class="token number">637</span> #define <span class="token function">DEVICE_ATTR_RO</span><span class="token punctuation">(</span>_name<span class="token punctuation">)</span> \

    <p>

    </p>
    <p>
     <code>
      __ATTR_RO

    <p>
     device_create_file -&gt; sysfs_create_file(&amp;dev-&gt;kobj, &amp;attr-&gt;attr);

    <p>
     sysfs_create_file_ns -&gt; sysfs_add_file_mode_ns (主体函数)
    </p>
    <p>
     在 sysfs_add_file_mode_ns 函数中：
     <br/>
     假设获取到的 sysfs_ops 为 dev_sysfs_ops
     <br/>
     那么，struct kernfs_ops ops 为 sysfs_file_kfops_rw
     <br/>
     以这个ops 作为参数调用 ：
     <code>
      __kernfs_create_file
     </code>
    </p>
    <p>
     <code>
      __kernfs_create_file
     </code>
     创建一个新的 kernfs_node 实例 kn，其 parent 为 device.kobj.sd
     <br/>
     并设置 kn-&gt;attr.ops = ops; 就是 sysfs_file_kfops_rw，如此在sysfs创立属性对应的文件
    </p>
    <p>
     （bus_create_file和device_create_file类似）
    </p>
    <h4>
     <a id="12_open_51">
     </a>
     1.2 open流程
    </h4>
    <p>
     kernfs_file_fops 这个 file_operation 实例是在哪里赋值的：
    </p>
    <p>
     kernfs_iop_lookup （kernfs_dir_iops.lookup）
     <br/>
     kernfs_get_inode
     <br/>
     kernfs_init_inode
    </p>
    <p>
     先看open:
     <br/>
     <code>
      SYSCALL_DEFINE3(open, const char __user *, filename, int, flags, umode_t, mode)
     </code>
     <br/>
     do_sys_open(dfd, filename, flags, mode);
     <br/>
     do_filp_open
     <br/>
     path_openat
     <br/>
     do_last
    </p>
    <p>
     do_last 函数中有两个关键调用：lookup_open、vfs_open
    </p>
    <p>
     先看 lookup_open：
     <br/>
     调用：
     <code>
      dir_inode-&gt;i_op-&gt;lookup
     </code>
     对于kernfs，dir_inode-&gt;i_op 为 kernfs_dir_iops（目录ops），其 lookup hook 为：kernfs_iop_lookup。kernfs_iop_lookup -》 kernfs_get_inode -》 kernfs_init_inode，在 kernfs_init_inode 中：
     <code>
      inode-&gt;i_fop = &amp;kernfs_file_fops;
     </code>
     ,得到文件操作集。
    </p>
    <p>
     继续看vfs_open：
     <br/>
     vfs_open -》 do_dentry_open
     <br/>
     do_dentry_open函数中，会执行：
     <code>
      f-&gt;f_op = fops_get(inode-&gt;i_fop);
     </code>
     将上面inode的fops赋值给file，就是 kernfs_file_fops
    </p>
    <p>
     do_dentry_open 下面会执行：
    </p>
    <pre><code class="prism language-c"> <span class="token number">806</span>     <span class="token keyword">if</span> <span class="token punctuation">(</span><span class="token operator">!</span>open<span class="token punctuation">)</span> <span class="token comment">// open 作为入参是NULL</span>
 <span class="token number">807</span>         open <span class="token operator">=</span> f<span class="token operator">-&gt;</span>f_op<span class="token operator">-&gt;</span>open<span class="token punctuation">;</span>
 <span class="token number">808</span>     <span class="token keyword">if</span> <span class="token punctuation">(</span>open<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
 <span class="token number">809</span>         error <span class="token operator">=</span> <span class="token function">open</span><span class="token punctuation">(</span>inode<span class="token punctuation">,</span> f<span class="token punctuation">)</span><span class="token punctuation">;</span>

 <span class="token number">812</span>     <span class="token punctuation">}</span>
</code></pre>
    <p>
     会执行到底层 open hook，即 kernfs_file_fops的open hook，kernfs_fop_open
    </p>
    <p>
     接下来看 kernfs_fop_open：
     <br/>
     首先从（当前操作文件对应的kernfs_node 实例）获取（kernfs_ops实例）
     <code>
      ops = kernfs_ops(kn);
     </code>
     就是：kn-&gt;attr.ops。
     <br/>
     在上面创建属性文件的流程中：__kernfs_create_file 函数会设置这个ops，为 sysfs_file_kfops_rw
     <br/>
     回到 kernfs_fop_open，会执行如下代码段：
    </p>
    <pre><code class="prism language-c"><span class="token number">703</span>     <span class="token keyword">if</span> <span class="token punctuation">(</span>ops<span class="token operator">-&gt;</span>seq_show<span class="token punctuation">)</span>
<span class="token number">704</span>         error <span class="token operator">=</span> <span class="token function">seq_open</span><span class="token punctuation">(</span>file<span class="token punctuation">,</span> <span class="token operator">&amp;</span>kernfs_seq_ops<span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 设置seq_file的ops</span>
</code></pre>
    <p>
     在seq_open函数中：会从slab缓存控制器里面分配一个 seq_file实例，并设置其ops为kernfs_seq_ops，并将这个seq_file作为其所属file实例的私有数据 private_data成员。
    </p>
    <p>
     打开流程就大概分析上面这些，主要是为了搞明白file的操作集是怎么来的，接下来read和write会调用到什么底层hook
    </p>
    <h4>
     <a id="13_read_99">
     </a>
     1.3 read流程
    </h4>
    <p>
     接下来看看读流程，是怎么调用到最底层的show hook的:
     <br/>
     <code>
      SYSCALL_DEFINE3(read, unsigned int, fd, char __user *, buf, size_t, count)
     </code>
     <br/>
     ksys_read(fd, buf, count);
     <br/>
     vfs_read
     <br/>
     <code>
      __vfs_read
     </code>
     <br/>
     file-&gt;f_op-&gt;read：由上面的open流程可知，f_op为 kernfs_file_fops，read hook 为 kernfs_fop_read
     <br/>
     kernfs_fop_read
     <br/>
     seq_read：主要是调用 m-&gt;op-&gt;show，在上面的open流程中，m-&gt;op为kernfs_seq_ops，show hook为 kernfs_seq_show
     <br/>
     kernfs_seq_show
     <br/>
     of-&gt;kn-&gt;attr.ops-&gt;seq_show：在创建流程中，可知为of-&gt;kn-&gt;attr.ops 为 sysfs_file_kfops_rw，seq_show hook为 sysfs_kf_seq_show
     <br/>
     sysfs_kf_seq_show:先获取当前kernfs_node对应的kobj，获取kobj-&gt;ktype-&gt;sysfs_ops，在创建流程中可知为 dev_sysfs_ops，再调用其 show hook，为 dev_attr_show
     <br/>
     dev_attr_show：由 attribute实例 找到 device_attribute 实例，再调用其show hook
     <br/>
     dev_attr-&gt;show
    </p>
    <p>
     上面 dev_attr-&gt;show 这个hook，可以通过 DEVICE_ATTR_RO 来定义，通过 device_create_file 来注册（具体看创建流程）
    </p>
