    <h3>
     <a id="_0">
     </a>
     出现问题的状态：
    </h3>
    <ul>
     <li>
      ALSA
      <strong>
       已正确
      </strong>
      识别到 ZOOM H2n 设备（card 1）
     </li>
     <li>
      sounddevice 库（依赖 PortAudio）
      <strong>
       未能正确枚举
      </strong>
      设备
     </li>
    </ul>
    <h3>
     <a id="_4">
     </a>
     修复方法：
    </h3>
    <p>
     <strong>
      1. 强制 sounddevice 使用 ALSA 后端
     </strong>
     <br/>
     默认情况下，sounddevice 可能尝试通过 PulseAudio 访问设备，导致冲突。需强制使用 ALSA：
     <br/>
     方法一：通过环境变量设置
     <br/>
     在运行 Python 脚本前设置环境变量：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">SDL_AUDIODRIVER</span><span class="token operator">=</span>alsa
<span class="token builtin class-name">export</span> <span class="token assign-left variable">AUDIODEV</span><span class="token operator">=</span>hw:1,0  <span class="token comment"># 对应你的 H2n 设备</span>
python your_script.py
</code></pre>
    <p>
     方法二：在代码中指定设备名称
     <br/>
     修改音频流初始化代码，直接使用 ALSA 设备名称：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 将 DEVICE_INDEX 替换为 ALSA 设备名称（如 'hw:1,0'）</span>
stream <span class="token operator">=</span> sd.InputStream<span class="token punctuation">(</span>device<span class="token operator">=</span><span class="token string">'hw:1,0'</span>, <span class="token assign-left variable">samplerate</span><span class="token operator">=</span>RATE, <span class="token assign-left variable">channels</span><span class="token operator">=</span><span class="token number">1</span>, <span class="token assign-left variable">blocksize</span><span class="token operator">=</span>CHUNK<span class="token punctuation">)</span>
</code></pre>
    <p>
     tips:但是我的sounddevice显示设备始终为空，指定也没用
     <br/>
     <strong>
      2.修复 PortAudio/ALSA 兼容性
     </strong>
     （成功修复）
     <br/>
     sounddevice 依赖 PortAudio 的 ALSA 支持，需确保系统已正确配置：
     <br/>
     步骤 1：安装 PortAudio 的 ALSA 开发库
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> libasound2-dev portaudio19-dev
</code></pre>
    <p>
     步骤 2：重新编译 sounddevice 的 PortAudio 后端（最终修复步骤）
     <br/>
     如果问题持续，可能需要手动编译 PortAudio：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 下载 PortAudio 源码</span>
<span class="token function">wget</span> http://files.portaudio.com/archives/pa_stable_v190700_20210406.tgz
<span class="token function">tar</span> <span class="token parameter variable">-xvf</span> pa_stable_v190700_20210406.tgz
<span class="token builtin class-name">cd</span> portaudio

<span class="token comment"># 配置并编译（强制启用 ALSA）</span>
./configure --with-alsa
<span class="token function">make</span>
<span class="token function">sudo</span> <span class="token function">make</span> <span class="token function">install</span>

<span class="token comment"># 重新安装 sounddevice</span>
pip uninstall <span class="token parameter variable">-y</span> sounddevice
pip <span class="token function">install</span> sounddevice --no-binary :all:
</code></pre>
    <h3>
     <a id="_49">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <pre><code class="prism language-bash"><span class="token function">import</span> sounddevice as sd
print<span class="token punctuation">(</span>sd.query_devices<span class="token punctuation">(</span><span class="token punctuation">))</span>  <span class="token comment"># 打印所有设备及其索引</span>
</code></pre>
    <p>
     用上面的代码输出始终为空，但是系统中又可以通过命令行录制音频，就怀疑是sounddevice的依赖有问题，重新编译安装依赖库，最终问题得以解决
    </p>
