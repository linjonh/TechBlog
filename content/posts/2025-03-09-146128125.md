    <p>
     <a href="https://arthas.aliyun.com/" rel="nofollow" title="Arthas">
      Arthas
     </a>
     是阿里开源的 Java 诊断工具，能够帮助开发者快速定位和解决 Java 应用的性能问题、内存泄漏、CPU 飙升等问题。以下是使用 Arthas 排查常见问题的详细步骤。
    </p>
    <hr/>
    <h3>
     <strong>
      1. 安装与启动 Arthas
     </strong>
    </h3>
    <h4>
     安装
    </h4>
    <ul>
     <li>
      <p>
       下载 Arthas：
      </p>
      <pre><code class="hljs">curl -O https://arthas.aliyun.com/arthas-boot.jar</code></pre>
      <p>
      </p>
     </li>
    </ul>
    <h4>
     启动
    </h4>
    <ul>
     <li>
      <p>
       启动 Arthas：
      </p>
      <pre><code class="hljs">java -jar arthas-boot.jar</code></pre>
      <p>
      </p>
     </li>
     <li>
      <p>
       选择需要诊断的 Java 进程（输入进程编号）。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      2. 常用命令
     </strong>
    </h3>
    <h4>
     <strong>
      dashboard
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       查看实时系统状态，包括 CPU、内存、线程等信息。
      </p>
      <pre><code class="hljs">dashboard</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          CPU
         </strong>
         ：查看占用 CPU 高的线程。
        </p>
       </li>
       <li>
        <p>
         <strong>
          Memory
         </strong>
         ：查看堆内存使用情况。
        </p>
       </li>
       <li>
        <p>
         <strong>
          Threads
         </strong>
         ：查看活跃线程。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      thread
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       查看线程信息，定位高 CPU 线程。
      </p>
      <pre><code class="hljs">thread</code></pre>
      <ul>
       <li>
        <p>
         查看所有线程：
        </p>
        <pre><code class="hljs">thread</code></pre>
        <p>
        </p>
       </li>
       <li>
        <p>
         查看 CPU 占用最高的线程：
        </p>
        <pre><code class="hljs">thread -n 3</code></pre>
        <p>
        </p>
       </li>
       <li>
        <p>
         查看某个线程的堆栈：
        </p>
        <pre><code class="hljs">thread &lt;线程ID&gt;</code></pre>
        <p>
        </p>
       </li>
       <li>
        <p>
         查找阻塞的线程：
        </p>
        <pre><code class="hljs">thread -b</code></pre>
        <p>
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      jvm
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       查看 JVM 信息，包括内存、GC、类加载等。
      </p>
      <pre><code class="hljs">jvm</code></pre>
      <p>
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      heapdump
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       导出堆内存快照，用于分析内存泄漏或 OOM。
      </p>
      <pre><code class="hljs">heapdump /path/to/heapdump.hprof</code></pre>
      <ul>
       <li>
        <p>
         使用
         <strong>
          MAT
         </strong>
         或
         <strong>
          JVisualVM
         </strong>
         分析
         <code>
          heapdump.hprof
         </code>
         文件。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      sc 和 sm
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        sc
       </strong>
       ：查看已加载的类信息。
      </p>
      <pre><code class="hljs">sc com.example.MyClass</code></pre>
      <p>
      </p>
     </li>
     <li>
      <p>
       <strong>
        sm
       </strong>
       ：查看类的方法信息。
      </p>
      <pre><code class="hljs">sm com.example.MyClass</code></pre>
      <p>
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      watch
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       监控方法的入参、返回值、异常等信息。
      </p>
      <pre><code class="hljs">watch com.example.MyClass myMethod "{params, returnObj, throwExp}" -n 5</code></pre>
      <ul>
       <li>
        <p>
         <code>
          params
         </code>
         ：方法参数。
        </p>
       </li>
       <li>
        <p>
         <code>
          returnObj
         </code>
         ：返回值。
        </p>
       </li>
       <li>
        <p>
         <code>
          throwExp
         </code>
         ：异常信息。
        </p>
       </li>
       <li>
        <p>
         <code>
          -n 5
         </code>
         ：监控 5 次调用。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      trace
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       追踪方法调用链路，分析性能瓶颈。
      </p>
      <pre><code class="hljs">trace com.example.MyClass myMethod</code></pre>
      <p>
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      monitor
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       监控方法的调用统计（调用次数、成功率、平均耗时等）。
      </p>
      <pre><code class="hljs">monitor -c 5 com.example.MyClass myMethod</code></pre>
      <ul>
       <li>
        <p>
         <code>
          -c 5
         </code>
         ：每 5 秒统计一次。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      ognl
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       执行 OGNL 表达式，查看或修改运行时的变量
      </p>
      <pre><code class="hljs">ognl '@com.example.MyClass@myStaticField'</code></pre>
      <p>
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      3. 排查常见问题
     </strong>
    </h3>
    <h4>
     <strong>
      CPU 飙升
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       使用
       <code>
        dashboard
       </code>
       查看 CPU 占用高的线程。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        thread -n 3
       </code>
       查看 CPU 占用最高的 3 个线程。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        thread &lt;线程ID&gt;
       </code>
       查看线程堆栈，定位问题代码。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        trace
       </code>
       或
       <code>
        watch
       </code>
       进一步分析问题方法。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      内存泄漏
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       使用
       <code>
        dashboard
       </code>
       查看内存使用情况。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        heapdump
       </code>
       导出堆内存快照。
      </p>
     </li>
     <li>
      <p>
       使用
       <strong>
        MAT
       </strong>
       或
       <strong>
        JVisualVM
       </strong>
       分析
       <code>
        heapdump.hprof
       </code>
       ，查找内存泄漏的根源。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        sc
       </code>
       和
       <code>
        sm
       </code>
       查看相关类的加载和方法调用情况。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      方法性能问题
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       使用
       <code>
        trace
       </code>
       追踪方法调用链路，分析耗时操作。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        monitor
       </code>
       监控方法的调用统计，查看平均耗时和调用次数。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        watch
       </code>
       监控方法的入参和返回值，定位问题。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      死锁
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       使用
       <code>
        thread -b
       </code>
       查找死锁线程。
      </p>
     </li>
     <li>
      <p>
       使用
       <code>
        thread &lt;线程ID&gt;
       </code>
       查看线程堆栈，分析锁竞争情况。
      </p>
     </li>
    </ol>
