     如何将一个项目推送到gitlab
    </h1>
   </div>

 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

   <div class="htmledit_views" id="content_views">
    <h4>
     1. 初始化本地项目为 Git 仓库
    </h4>
    <p>
     若本地项目还不是 Git 仓库，要先将其初始化为 Git 仓库。在项目根目录下打开终端，执行如下命令：
    </p>
    <pre><code class="hljs">git init</code></pre>
    <h4>
     2. 添加文件到暂存区
    </h4>
    <p>
     使用
     <code>
      git add
     </code>
     命令把项目中的文件添加到暂存区。若要添加所有文件，可执行：
    </p>
    <pre><code class="hljs">git add .</code></pre>
    <h4>
     3. 提交暂存区的文件到本地仓库
    </h4>
    <p>
     添加文件到暂存区后，使用
     <code>
      git commit
     </code>
     命令将这些文件提交到本地仓库，同时要添加有意义的提交信息：
    </p>
    <pre><code class="hljs">git commit -m "Initial commit"</code></pre>
    <h4>
     4. 添加远程仓库地址
    </h4>
    <p>
     使用
     <code>
      git remote add
     </code>
     命令将 GitLab 上的项目仓库地址添加为本地仓库的远程仓库。假设远程仓库地址为
     <code>
      http://10.10.103.61:31200/cna/microservice.git
     </code>
     （具体地址要依据实际情况调整），执行以下命令：
    </p>
    <pre><code class="hljs">git remote add origin http://10.10.103.61:31200/cna/microservice.git</code></pre>
    <h4>
     5. 推送本地仓库内容到远程仓库
    </h4>
    <p>
     执行
     <code>
      git push
     </code>
     命令把本地仓库的内容推送到远程仓库。初次推送时，要指定远程仓库名称（通常是
     <code>
      origin
     </code>
     ）和本地分支名称（通常是
     <code>
      main
     </code>
     或者
     <code>
      master
     </code>
     ，具体取决于你的配置），并且使用
     <code>
      -u
     </code>
     参数关联本地分支和远程分支：
    </p>
    <pre><code class="hljs">git push -u origin main</code></pre>
    <p>
     若本地分支名称不是
     <code>
      main
     </code>
     ，需将上述命令中的
     <code>
      main
     </code>
     替换为实际的分支名称。
    </p>
