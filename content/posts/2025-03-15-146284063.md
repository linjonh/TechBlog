    <p>
     内部底层实现互通
    </p>
    <p>
     <strong>
      需求：实现内网到外网的通信
     </strong>
    </p>
    <p>
     与外网的配置
    </p>
    <p>
     AR1：interface GigabitEthernet0/0/2
     <br/>
     ip address 202.100.1.1 255.255.255.0
     <br/>
     #
     <br/>
     ospf 1
     <br/>
     default-route-advertise
     <br/>
     #
     <br/>
     ip route-static 0.0.0.0 0.0.0.0 202.100.1.2
    </p>
    <p>
     AR4：
    </p>
    <p>
     interface GigabitEthernet0/0/0
     <br/>
     ip address 202.100.1.2 255.255.255.0
     <br/>
     #
     <br/>
     interface GigabitEthernet0/0/1
     <br/>
     ip address 200.1.1.2 255.255.255.0
     <br/>
     #
     <br/>
     interface GigabitEthernet0/0/2
     <br/>
     ip address 100.1.1.2 255.255.255.0
    </p>
    <p>
     一、配置ACL匹配上网路由（AR1）
    </p>
    <p>
     acl number 2000
     <br/>
     rule 5 permit source 192.168.1.0 0.0.0.255
     <br/>
     rule 10 permit source 192.168.2.0 0.0.0.255
    </p>
    <p>
     二、配置NAT进行地址转换
    </p>
    <p>
     1、
     <span style="color:#000000">
      easy-ip 出接口转换
     </span>
    </p>
    <p>
     ①配置（AR1）
    </p>
    <p>
     <span style="color:#000000">
      interface GigabitEthernet0/0/2
      <br/>
      nat outbound 2000
     </span>
    </p>
    <p>
     <span style="color:#000000">
      ②查看
     </span>
    </p>
    <p>
     <img alt="" height="145" src="https://i-blog.csdnimg.cn/direct/e51ef59d2d604788b1dd93aa9fd406da.png" width="761"/>
    </p>
    <p>
     ③验证：内部能ping通外部202.100.1.2的地址
    </p>
    <p>
     <img alt="" height="342" src="https://i-blog.csdnimg.cn/direct/f5aa2b4f34c347448387e94f587b8bf4.png" width="769"/>
    </p>
    <p>
     2、地址池转换--NAPT，增加需求PC4不可以上网
    </p>
    <p>
     ①配置，按从小到大顺序匹配，所以先拒绝192.168.2.1，再允许192.168.2.0/24的其他IP可以上网
     <br/>
     acl number 2000
     <br/>
     rule 5 permit source 192.168.1.0 0.0.0.255
     <br/>
     rule 10 deny source 192.168.2.1 0
     <br/>
     rule 15 permit source 192.168.2.0 0.0.0.255
    </p>
    <p>
     nat address-group 1 202.100.1.3 202.100.1.10
     <br/>
     #
     <br/>
     interface GigabitEthernet0/0/2
     <br/>
     nat outbound 2000 address-group 1
     <br/>
     ②查看
    </p>
    <p>
     <img alt="" height="146" src="https://i-blog.csdnimg.cn/direct/a66aafff990343c49fd03a6dd6ba9485.png" width="779"/>
    </p>
    <p>
     ③验证：增加需求PC4不可以上网，其他网段正常上网
    </p>
    <p>
     <img alt="" height="365" src="https://i-blog.csdnimg.cn/direct/feb92d7b22ff4cedb48d5c947c8aacee.png" width="745"/>
    </p>
    <p>
     <img alt="" height="325" src="https://i-blog.csdnimg.cn/direct/ccd598a0e67544f79d61b6a444ccab4b.png" width="755"/>
    </p>
    <p>
     <img alt="" height="501" src="https://i-blog.csdnimg.cn/direct/57e0c0cc6ced41eea5df08fff4964bbd.png" width="725"/>
    </p>
    <p>
     <strong>
      需求：将内网服务器映射到公网，公网通过公网地址访问内网服务
     </strong>
     <br/>
     <strong>
      nat server
     </strong>
    </p>
    <p>
     <strong>
      ①配置
     </strong>
    </p>
    <p>
     interface GigabitEthernet0/0/2
     <br/>
     nat server protocol tcp global current-interface 1111 inside 192.168.1.2 ftp
     <br/>
     nat server protocol tcp global current-interface 2222 inside 192.168.1.2 www
    </p>
