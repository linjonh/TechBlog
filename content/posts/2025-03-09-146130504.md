    <h2 id="server-review">
     服务器审查
    </h2>
    <p>
     在 Linux 中审查服务器的过程包括评估服务器的性能、安全性和配置，以确定需要改进的领域或任何潜在问题。审查的范围可以包括检查安全增强功能、检查日志文件、审查用户帐户、分析服务器的网络配置以及检查其软件版本。
    </p>
    <p>
     Linux 以其稳定性和安全性而闻名，已成为全球许多网络和服务器后端的主打产品。根据您使用的发行版，Linux 提供了多种工具和命令来执行全面的服务器审查。
    </p>
    <pre><code># A command often used for showing memory information
free -m

# A command for showing disk usage
df -h

# A command for showing CPU load
uptime
</code></pre>
    <p>
     系统管理员和 DevOps 专业人员定期进行服务器审查以确保服务器的最佳性能、安全性和可靠性是一项关键任务。
    </p>
    <p>
     <code>
      Linux 服务器审查
     </code>
     就像给服务器做“全身体检”——检查它的健康状况、安全漏洞和性能瓶颈，确保它稳定高效地工作。以下是通俗易懂的步骤和常用命令：
    </p>
    <hr/>
    <h4>
     <strong>
      一、基础检查：看“生命体征”
     </strong>
    </h4>
    <p>
     就像测体温、血压，先看服务器的基本状态：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        内存使用
       </strong>
       （查是否“贫血”）：
      </p>
      <pre><code>free -m  # 查看内存总量、已用、空闲（单位MB）</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          关键指标
         </strong>
         ：
         <code>
          available
         </code>
         （可用内存）是否充足。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        磁盘空间
       </strong>
       （查“胃容量”）：
      </p>
      <pre><code>df -h  # 查看各磁盘分区的使用情况（单位易读：GB/MB）</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          重点关注
         </strong>
         ：
         <code>
          Use%
         </code>
         （使用率）超过 80% 需清理。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        CPU 负载
       </strong>
       （查“心脏压力”）：
      </p>
      <pre><code>uptime      # 查看负载平均值（1/5/15分钟）
top 或 htop  # 实时监控CPU占用高的进程（按q退出）</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          负载解读
         </strong>
         ：负载值 ≤ CPU核心数 为正常（如4核CPU负载≤4）。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      二、安全审查：防“小偷入侵”
     </strong>
    </h4>
    <p>
     检查门锁（防火墙）、访客记录（日志）和可疑人员（用户账户）：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        用户账户检查
       </strong>
       （查“谁有钥匙”）：
      </p>
      <pre><code>sudo less /etc/passwd      # 查看所有用户
sudo less /etc/shadow      # 查看用户密码状态（需权限）
who                        # 查看当前登录用户
last                       # 查看登录历史</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          重点排查
         </strong>
         ：陌生用户、空密码账户、root远程登录。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        防火墙与端口
       </strong>
       （查“门窗是否关好”）：
      </p>
      <pre><code>sudo ufw status          # 查看防火墙状态（Ubuntu）
sudo iptables -L -n -v  # 查看详细防火墙规则（通用）
ss -tuln                # 查看所有监听端口（替代netstat）</code></pre>
      <ul>
       <li>
        <p>
         <strong>
          风险端口
         </strong>
         ：关闭不必要的端口（如默认的22端口可改为非标准）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        日志分析
       </strong>
       （查“监控录像”）：
      </p>
      <pre><code>sudo tail -100 /var/log/auth.log  # 查看认证日志（SSH登录记录）
sudo journalctl -u sshd          # 查看SSH服务日志（Systemd系统）
grep "Failed" /var/log/auth.log  # 过滤登录失败的记录（防暴力破解）</code></pre>
      <pre><code>🔹 如何检测可疑攻击？
🔸 1️⃣ 统计失败登录次数：
sh
复制
编辑
grep "Failed" /var/log/auth.log | wc -l
📌 作用：统计包含 "Failed" 的行数，即失败尝试次数。
🔍 如果失败次数异常多（比如几百次），可能是暴力破解！

🔸 2️⃣ 查看攻击来源 IP
sh
复制
编辑

📌 作用：

解析 Failed password for ... from 192.168.1.100 这类日志

复制

🔍 如果某个 IP 失败次数异常多，说明它在恶意尝试登录！

sh
复制
编辑

📌 作用：阻止 203.0.113.42 访问服务器（彻底封锁）。

     </li>
    </ol>

    <h4>
     <strong>
      三、性能调优：治“慢性病”

     检查拖慢服务器的“慢性问题”：
    </p>

      <p>

        进程与服务
       </strong>
       （查“谁在吃资源”）：
      </p>

      <ul>
       <li>
        <p>
         <strong>

       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        定时任务
       </strong>
       （查“自动任务”）：
      </p>

       <li>
        <p>

        </p>

    <h2 id="uptime-load" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        正常运行时间负载
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在管理 Linux 服务器时，一个值得密切关注的关键指标是“正常运行时间”。Linux 中的命令提供有关系统在不关闭或重新启动的情况下运行了多长时间的信息，以及系统负载平均值。
       <code>
        uptime
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       系统负载平均值是一个重要的指标，它说明了计算机系统执行的计算工作量。它反映了有多少进程正在排队等待获取 CPU 时间。系统负载平均值通常显示 1 分钟、5 分钟和 15 分钟的持续时间。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       通过持续分析 Linux 服务器上的正常运行时间和负载，管理员可以识别系统使用模式、诊断可能的性能问题并确定有效的容量规划策略。如果服务器的平均负载较高，则可能表明系统资源不足或配置错误，从而导致性能降低或系统无响应。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       以下是该命令及其输出的示例：
       <code>
        uptime
       </code>
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">$ uptime
 10:58:35 up 2 days, 20 min,  1 user,  load average: 0.00, 0.01, 0.05</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在上面的输出中，“2 days， 20 min” 告诉我们系统运行了多长时间，而 “0.00， 0.01， 0.05” 分别显示了系统在过去 1 分钟、5 分钟和 15 分钟内的平均负载。
      </span>
     </span>
    </p>
    <p>
     <code>
      uptime
     </code>
     是 Linux 中一个快速查看
     <strong>
      服务器健康状态
     </strong>
     的命令，它能告诉你两件关键事：
     <strong>
      服务器运行了多久
     </strong>
     和
     <strong>
      当前系统的“忙碌程度”
     </strong>
     。就像汽车的仪表盘，一眼就能看出是否需要“检修”！
    </p>
    <hr/>
    <h4>
     <strong>
      命令输出示例
     </strong>
    </h4>
    <p>
    </p>
    <pre><code>$ uptime
 10:58:35 up 2天, 20分钟,  1用户,  平均负载: 0.00, 0.01, 0.05</code></pre>
    <pre></pre>
    <p>
     <strong>
      翻译成大白话
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        10:58:35
       </strong>
       ：现在是上午10点58分35秒（系统时间）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        up 2天, 20分钟
       </strong>
       ：服务器已经连续运行了2天零20分钟没重启。
      </p>
     </li>
     <li>
      <p>
       <strong>
        1用户
       </strong>
       ：当前有1个用户登录（比如通过SSH）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        平均负载: 0.00, 0.01, 0.05
       </strong>
       ：过去1分钟、5分钟、15分钟的“负载指数”。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      “平均负载”通俗解释
     </strong>
    </h4>
    <p>
     你可以把
     <strong>
      负载（Load Average）
      <strong>
       想象成
      </strong>
      超市收银台的排队情况
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        负载值 = 正在结账的顾客 + 排队等待的顾客
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       假设服务器是
       <strong>
        4个收银台（4核CPU）
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          负载 ≤4
         </strong>
         ：顾客来了能立刻结账，无需排队（健康）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          负载 &gt;4
         </strong>
         ：开始排队，数值越大队伍越长（可能卡顿）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        三个数值的意义
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <code>
          0.00
         </code>
         （1分钟负载）：刚刚的排队情况（最敏感，可能突发高峰）。
        </p>
       </li>
       <li>
        <p>
         <code>
          0.01
         </code>
         （5分钟负载）：短期的平均排队情况。
        </p>
       </li>
       <li>
        <p>
         <code>
          0.05
         </code>
         （15分钟负载）：长期的趋势（更稳定）。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      如何判断负载是否过高？
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        单核CPU
       </strong>
       ：负载超过1.0 表示过载。
      </p>
     </li>
     <li>
      <p>
       <strong>
        4核CPU
       </strong>
       ：负载超过4.0 表示满负荷。
      </p>
     </li>
     <li>
      <p>
       <strong>
        举例
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <code>

         </code>

        </p>
        <ul>
         <li>

          <p>

           </strong>

          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

    <h2 id="auth-logs" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        身份验证日志
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       在处理 Linux 服务器及其维护时，需要定期审查的最关键组件之一是身份验证日志。这些日志通常位于 /var/log/auth.log（对于基于 Debian 的发行版）或 /var/log/secure（对于 Red Hat 和 CentOS）中，记录服务器上发生的所有与身份验证相关的事件和活动。其中包括系统登录、密码更改和发出的 sudo 命令等。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       身份验证日志是监控和分析 Linux 服务器安全性的宝贵工具。它们可以指示暴力登录攻击、未经授权的访问尝试和任何可疑行为。定期分析这些日志是确保服务器安全性和数据完整性的一项基本任务。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       以下是如何使用命令查看身份验证日志的最后几个条目的示例：
       <code>
        tail
       </code>
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">tail /var/log/auth.log</code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       熟悉阅读和理解身份验证日志，因为这是确保服务器安全的重要方法。
      </span>
     </span>
    </p>
    <p>
     <code>
      Linux 身份验证日志
     </code>
     就像服务器的“
     <strong>
      安全监控录像
     </strong>
     ”，详细记录了谁在什么时候、以什么方式登录过系统，以及做了哪些需要权限的操作。它是排查黑客入侵、追踪异常行为的“破案神器”！以下是通俗解读：
    </p>
    <hr/>
    <h4>
     <strong>
      一、日志文件在哪？
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Debian/Ubuntu
       </strong>
       ：
       <code>
        /var/log/auth.log
       </code>
      </p>
     </li>
     <li>
      <p>
       <strong>
        RedHat/CentOS
       </strong>
       ：
       <code>
        /var/log/secure
       </code>
      </p>
     </li>
     <li>
      <p>
       查看最新日志（实时监控）：
      </p>
      <pre><code>sudo tail -f /var/log/auth.log  # 实时滚动显示最新日志（按Ctrl+C退出）</code></pre>
      <pre></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>

     </strong>
    </h4>

      <ul>
       <li>
        <p>
         <strong>
          关键信息
         </strong>
         ：用户
         <code>
          root
         </code>
         从 IP
         <code>
          192.168.1.100
         </code>
         通过密码登录成功。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        登录失败
       </strong>
       ：
      </p>
      <pre><code>May 25 14:31:05 server sshd[5678]: Failed password for invalid user hacker from 203.0.113.5 port 22 ssh2</code></pre>
      <pre></pre>
      <ul>
       <li>
        <p>
         <strong>
          危险信号
         </strong>

         <code>

         </code>

    </ul>

    <h5>

    </h5>
    <ul>
     <li>
      <p>
       <strong>
        用户提权操作
       </strong>

     <li>
      <p>
       <strong>
        创建/删除用户
       </strong>

      </p>

May 25 16:05:00 server userdel[3456]: delete user 'test'</code></pre>

     </li>

    <h4>
     <strong>
      三、如何快速分析日志？
     </strong>
    </h4>
    <h5>
     1️⃣
     <strong>
      找“暴力破解”攻击
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       统计某个 IP 的失败登录次数：
      </p>
      <pre><code>grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr
# 输出示例：
# 100 203.0.113.5
# 2   192.168.1.100</code></pre>
      <pre></pre>
      <ul>
       <li>
        <p>
         <strong>

         </strong>
         ：IP
         <code>
          203.0.113.5
         </code>

      </ul>
     </li>
    </ul>
    <h5>
     2️⃣

     </strong>
    </h5>
    <ul>
     <li>
      <p>
       查看所有 sudo 提权记录：
      </p>
      <pre><code>grep "sudo:" /var/log/auth.log</code></pre>
      <pre></pre>
     </li>
    </ul>
    <h5>
     3️⃣

    <ul>
     <li>
      <p>

      <pre><code>grep "Accepted" /var/log/auth.log</code></pre>
      <pre></pre>
     </li>
    </ul>

    <h2 id="services-running" style="margin-left:.625rem; margin-right:0; text-align:start">
     <strong>
      <span style="color:var(--tw-prose-headings)">
       <span style="background-color:#ffffff">
        正在运行的服务
       </span>
      </span>
     </strong>
    </h2>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       Linux 服务器因其稳定性和灵活性而广受欢迎，这些因素使它们成为企业和组织在管理各种服务时的首选。在 Linux 服务器下运行的服务范围从 Web 服务到数据库服务、DNS 服务器、邮件服务器等。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       作为 Linux 系统管理员，定期查看这些正在运行的服务以管理资源、检查其状态和解决问题，从而确保服务器的运行状况和性能非常重要。
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       Linux 有多种工具可以实现此目的，例如：、、 和 。
       <code>
        systemctl
       </code>
       <code>
        service
       </code>
       <code>
        netstat
       </code>
       <code>
        ss
       </code>
       <code>
        lsof
       </code>
      </span>
     </span>
    </p>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       例如，该命令在 Linux 系统上广泛使用，用于列出所有正在运行的服务：
       <code>
        systemctl
       </code>
      </span>
     </span>
    </p>
    <pre class="has" style="margin-left:0; margin-right:0; text-align:start"><code class="language-astro-code">systemctl --type=service </code></pre>
    <p style="margin-left:.5rem; margin-right:0; text-align:start">
     <span style="color:#374151">
      <span style="background-color:#ffffff">
       此命令将显示所有活动服务的列表及其当前状态。它是服务器管理的必需品，应该成为任何 Linux 系统管理员工具箱的一部分。
      </span>
     </span>
    </p>
    <p>
     在 Linux 服务器中，**“正在运行的服务”**就像一台多功能机器上的各种“工作模块”——比如负责网站服务的“模块”、管理数据库的“模块”、处理邮件的“模块”等。作为管理员，你需要知道哪些模块正在运行、是否健康，以及它们占用了多少资源。以下是通俗易懂的检查方法：
    </p>
    <hr/>
    <h4>
     <strong>
      一、查看服务列表：用“服务遥控器”
     </strong>
    </h4>
    <h5>
     <strong>
      1.
      <code>
       systemctl
      </code>
      命令
     </strong>
     （主流工具，适用于大多数新系统）
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        列出所有正在运行的服务
       </strong>
       ：
      </p>
      <pre><code>systemctl list-units --type=service --state=running</code></pre>
      <pre></pre>
      <p>
       <strong>
        输出示例
       </strong>
       ：
      </p>
      <pre><code>UNIT                LOAD   ACTIVE SUB     DESCRIPTION
nginx.service       loaded active running Nginx Web Server
mysql.service       loaded active running MySQL Database
ssh.service         loaded active running OpenSSH Server</code></pre>
      <pre></pre>
      <ul>
       <li>
        <p>
         <strong>
          关键信息
         </strong>
         ：
         <code>
          ACTIVE
         </code>
         显示服务状态（
         <code>
          active
         </code>
         表示正常运行）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        检查某个服务的详细状态
       </strong>
       （如Nginx）：
      </p>
      <pre><code>systemctl status nginx</code></pre>
      <pre></pre>
      <p>
       <strong>
        输出示例
       </strong>
       ：
      </p>
      <pre><code>● nginx.service - Nginx Web Server
   Loaded: loaded (/lib/systemd/system/nginx.service; enabled; vendor preset: enabled)
   Active: active (running) since Mon 2024-05-27 10:00:00 CST; 2 days ago
   Main PID: 1234 (nginx)
   CGroup: /system.slice/nginx.service
           ├─1234 nginx: master process /usr/sbin/nginx
           └─1235 nginx: worker process</code></pre>
      <pre></pre>
      <ul>
       <li>
        <p>
         <strong>
          关注点
         </strong>
         ：
         <code>
          Active
         </code>
         （运行状态）、
         <code>
          Main PID
         </code>
         （进程ID）、日志片段。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      2.
      <code>
       service
      </code>
      命令
     </strong>
     （旧系统兼容工具，操作更简单）
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        列出所有服务
       </strong>
       （仅显示部分）：
      </p>
      <pre><code>service --status-all</code></pre>
      <pre></pre>
      <p>
       <strong>
        输出示例
       </strong>
       ：
      </p>
      <pre><code>[ + ]  nginx
[ - ]  mysql
[ ? ]  docker</code></pre>
      <pre></pre>
      <ul>
       <li>
        <p>
         <strong>
          符号解读
         </strong>
         ：
         <code>
          +
         </code>
         表示运行中，
         <code>
          -
         </code>
         表示已停止，
         <code>
          ?
         </code>
         表示状态未知。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      二、检查网络服务：用“网络监控器”
     </strong>
    </h4>
    <h5>
     <strong>
      1.
      <code>
       ss
      </code>
      或
      <code>
       netstat
      </code>

    <ol>
     <li>
      <p>
       <strong>
        释放缓存
       </strong>
       （临时救急）：
      </p>
      <pre><code>sync &amp;&amp; echo 3 | sudo tee /proc/sys/vm/drop_caches  # 清理缓存（不影响程序运行）</code></pre>
      <pre><code>📌 sync &amp;&amp; echo 3 | sudo tee /proc/sys/vm/drop_caches 详解
该命令用于清理 Linux 系统的缓存，但不会影响正在运行的程序。

🔹 1️⃣ sync —— 确保数据写入磁盘
✅ 作用：sync 命令会强制将所有待写入磁盘的数据（缓存中的数据）真正写入磁盘，确保数据不会因为清理缓存而丢失。

🔹 2️⃣ echo 3 —— 生成缓存清理指令
✅ 作用：echo 命令用于输出 3，这里 3 是一个指令参数，用于通知 Linux 清除所有类型的缓存。

数值	作用
1	只清除 页缓存（Page Cache）
2	只清除 目录项和 inode 缓存
3	清除所有缓存（推荐使用）

🔹 3️⃣ |（管道）—— 连接 echo 和 tee
echo 3 | sudo tee /proc/sys/vm/drop_caches
✅ 作用：
echo 3 生成指令
| 管道符：将 echo 3 的输出传递给 tee 命令

🔹 4️⃣ sudo tee /proc/sys/vm/drop_caches —— 执行缓存清理
sudo tee /proc/sys/vm/drop_caches
✅ 作用：
sudo —— 以 超级用户权限 执行（普通用户无权修改 /proc/sys/vm/drop_caches）。
tee —— 既输出内容到终端，也写入文件：
/proc/sys/vm/drop_caches 是 Linux 内核参数文件，写入 3 之后，系统会执行缓存清理。
tee 作用类似于 &gt;，但可以同时显示和写入文件，&gt; 直接写入而不显示。

🔹 5️⃣ sync &amp;&amp; echo 3 | sudo tee /proc/sys/vm/drop_caches 总结
✅ 完整流程：
sync 确保所有数据写入磁盘，防止未保存数据丢失。
echo 3 生成清理指令。
| 把指令传递给 tee。
sudo tee /proc/sys/vm/drop_caches 清理所有缓存。

🔹 6️⃣ 注意事项
📌 不会影响正在运行的程序：
该命令仅清理缓存，不会影响应用程序的正常运行。
清理缓存后，系统可能会变慢，因为需要重新加载数据到缓存。

📌 适用于哪些场景？
释放内存：如果 Linux 系统长时间运行，占用大量缓存，可以手动释放缓存。
测试性能：在进行性能测试时，希望得到不受缓存影响的结果。
查看真实的可用内存：缓存占用内存时，可释放后查看真实的空闲内存。

🔹 7️⃣ 扩展：如何验证缓存清理？
📌 清理缓存前后，查看内存占用
free -h    # 查看内存使用情况
sync &amp;&amp; echo 3 | sudo tee /proc/sys/vm/drop_caches
free -h    # 再次查看变化
🔹 清理前：cached 部分较高
🔹 清理后：cached 变少，free 变多 ✅

🔹 8️⃣ 更安全的写法
为了防止错误，可以使用更安全的方法：
sudo sh -c "echo 3 &gt; /proc/sys/vm/drop_caches"
✅ 区别：

sudo 只影响 sh -c 里的 echo，避免 tee 额外权限问题。
🔹 9️⃣ 总结
命令部分	作用
sync	将缓存数据写入磁盘，防止数据丢失
echo 3	生成清理指令（3 代表清理所有缓存）
`	`
sudo tee /proc/sys/vm/drop_caches	执行缓存清理，释放内存
💡 这个命令适用于手动释放缓存，特别是在长时间运行的 Linux 服务器上。 </code></pre>

    <pre><code class="hljs">ps (Process Status，进程状态) 是 Linux 用于查看进程信息的命令。
aux 选项组合：
a (All Users) → 显示所有用户的进程（不仅限于当前用户）。
u (User Format) → 以用户友好的格式显示，包括 CPU、内存、进程名等信息。
