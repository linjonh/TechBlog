     <h4>
      <a id="11__7">

      1.1. 大端序与小端序
     </h4>
     <p>
      <strong>
       大端序
      </strong>
      ：内存地址低位存储数据的高位、内存地址高位存储数据的低位
      <br/>
      <strong>
       小端序
      </strong>
      ：内存地址高位存储数据的高位、内存地址低位存储数据的低位
     </p>
     <p>
      采用大端序保存多字节数据非常直观，它常用于大型UNIX服务器的RISC系列的CPU中。此外，
      <strong>
       网络协议中也经常采用大端序方式
      </strong>
      。了解这些，对从事x86系列应用程序的开发人员以及代码逆向分析人员具有非常重要的意义，因为通过网络传输应用程序使用数据时，往往都需要修改字节序。
     </p>

     <h2>
      <a id="2IA32_40">
      </a>
      2.IA-32寄存器
     </h2>
     <h3>
      <a id="21_CPU_41">
      </a>
      2.1. 什么是CPU寄存器
     </h3>
     <p>
      寄存器（Register）是CPU内部用来存放数据的一些小型存储区域，它与我们常说的RAM（RandomAccessMemory，随机存储器、内存）略有不同。CPU访问（Access）
      <strong>
       RAM中的数据时要经过较长的物理路径，所以花费的时间要长一些；而寄存器集成在CPU内部，拥有非常高的读写速度。
      </strong>
     </p>
     <h3>
      <a id="22_IA32_44">
      </a>
      2.2. IA-32寄存器
     </h3>
     <p>
      IA-32是英特尔推出的32位元架构，属于复杂的指令集架构，它提供了非常丰富的功能，且支持多种寄存器。
     </p>
     <p>
      通用寄存器（General PurposeRegisters,32位，8个）
      <br/>
      段寄存器（Segment Registers，16位，6个)
      <br/>
      程序状态与控制寄存器（ProgramStatus and Control Registers，32位，1个）
      <br/>
      指令指针寄存器（InstructionPointer，32位，1个)
     </p>
     <h4>
      <a id="221__51">
      </a>
      2.2.1. 通用寄存器
     </h4>
     <p>
      <img alt="Pasted image 20250311164546" src="https://i-blog.csdnimg.cn/img_convert/512688fda3538d0c8d3753ca7ba41499.png"/>
     </p>
     <blockquote>
      <p>
       为了实现对低16位的兼容，各寄存器又可以分为高（H:High)低（L：Low)
       <br/>
       几个独立寄存器。下面以EAX为例讲解。
       <br/>
       EAX:（0~31）32位
       <br/>
       AX:（0~15）EAX的低16位
       <br/>
       AH:（8~15）AX的高8位
       <br/>
       AL：（0~7）AX的低8位
       <br/>
       若想全部使用4个字节（32位），则使用EAX;若只想使用2个字节（16位)，只
       <br/>
       要使用EAX的低16位部分AX就可以了。
       <br/>
       AX又分为高8位的AH与低8位的AL两个独立寄存器。借助这种方式，可以根据不同情况把一个32位的寄存器分别用作8位、16位、32位寄存器。后面的程序调试中，我们分析汇编代码就能很容易地理解它们。
      </p>
     </blockquote>
     <p>
      EAX：（针对操作数和结果数据的）累加器
      <br/>
      EBX：（DS段中的数据指针）基址寄存器
      <br/>
      ECX：（字符串和循环操作的）计数器
      <br/>
      EDX：（IO指针）数据寄存器
      <br/>
      EBP：（SS段中栈内数据指针）扩展基址指针寄存器
      <br/>
      ESI:（字符串操作源指针）源变址寄存器
      <br/>
      EDI：（字符串操作目标指针）目的变址寄存器
      <br/>
      ESP：（SS段中栈指针）栈指针寄存器
      <br/>
      此外：EAX一般用在函数返回值中。所有的win32API 函数都会先把返回值保存到EAX再返回
     </p>
     <h4>
      <a id="222__73">
      </a>
      2.2.2. 段寄存器
     </h4>
     <p>
      IA-32的保护模式中，段是一种内存保护技术，它把内存划分为多个区段，并为每个区段赋、范围、访问权限等，以保护内存。
      <br/>
      此外，它还同分页技术（Paging）一起用于将虚拟内存变更为实际物理内存。段内存记录在
      <strong>
       SDT
      </strong>
      （SegmentDescriptorTable，段描述符表）中，而段寄存器就持有这些SDT的索引（index)。
     </p>
     <p>
      CS:Code Segment，代码段寄存器
      <br/>
      SS：Stack Segment,栈段寄存器
      <br/>
      DS：Data Segment，数据段寄存器
      <br/>
      ES:Extra（Data）Segment，附加（数据）段寄存器
      <br/>
      FS：DataSegment，数据段寄存器
      <br/>
      GS:DataSegment，数据段寄存器
     </p>
     <h4>
      <a id="223__84">
      </a>
      2.2.3. 程序状态与控制寄存器
     </h4>
     <p>
      <strong>
       EFLAGS：FlagRegister，标志寄存器
      </strong>
      <br/>
      大小为4字节（32位）由原来的16位FLAGS寄存器拓展而来
      <br/>
      学习代码逆向分析技术的初级阶段，只要掌握3个与程序调试相关的标志即可，
      <br/>
      分别为
      <strong>
       ZF（ZeroFlag，零标志)
      </strong>
      、
      <strong>
       OF（OverflowFlag，溢出标志)
      </strong>
      、
      <strong>
       CF（CarryFlag，进位标志)
      </strong>
      。
     </p>
     <h4>
      <a id="224__90">
      </a>
      2.2.4. 指令指针寄存器
     </h4>
     <p>
      <strong>
       ●EIP：InstructionPointer，指令指针寄存器
      </strong>
      <br/>
      指令指针寄存器保存着CPU要执行的指令地址，其大小为32位（4个字节），由原16位IP寄存器扩展而来。程序运行时，CPU会读取EIP中一条指令的地址，传送指令到指令缓冲区后，EIP寄存器的值自动增加，增加的大小即是读取指令的字节大小。
      <br/>
      这样，CPU每次执行完一条指令，就会通过EIP寄存器读取并执行下一条指令。
      <br/>
      <strong>
       与通用寄存器不同，我们不能直接修改EIP的值，只能通过其他指令间接修改
      </strong>
      ，这些特定指令包括JMP、Jcc、CALL、RET。此外，我们还可以通过中断或异常来修改EIP的值。
     </p>
     <h2>
      <a id="3__96">
      </a>
      3. 栈
     </h2>
     <p>
      <strong>
       栈内存在进程中的作用如下：
      </strong>
      <br/>
      (1)暂时保存函数内的局部变量。
      <br/>
      (2)调用函数时传递参数。
      <br/>
      (3)保存函数返回后的地址。
      <br/>
      栈其实是一种数据结构，它按照FILO（FirstInLastOut，后进先出）的原则存储数据。
     </p>
     <h4>
      <a id="11__103">
      </a>
      1.1. 栈的特征
     </h4>
     <p>
      <img alt="Pasted image 20250311170849" src="https://i-blog.csdnimg.cn/img_convert/787673dee62e07d36217f4bccd905759.png">
       <br/>
       一个进程中，
       <strong>
        栈顶指针（ESP）初始状态指向栈底端
       </strong>
       。执行PUSH命令将数据压人栈时，栈顶指针就会上移到栈顶端。执行POP命令从栈中弹出数据时，若栈为空，则栈顶指针重新移动到栈底端。
       <br/>
       换言之，
       <strong>
        栈是一种由高地址向低地址扩展的数据结构
       </strong>
       ，图中，栈是由下往上扩展的。
       <br/>
       由于栈具有这种特征，所以我们常常说“
       <strong>
        栈是逆向扩展的
       </strong>
       ”，向栈中压数据就像一层层砌砖，每向上砌一层，砖墙就增高一点儿。
      </img>
     </p>

       函数调用时的参数顺序（正序）与参数入栈时的顺序（逆序）相反。
      </strong>
     </p>
     <blockquote>
      <p>
       这里就是因为栈的结构是
       <strong>
        FILO
       </strong>
       先进后出，即出入栈会交换顺序
       <br/>
       我们是要先把参数入栈，然后再出栈传递给函数进行调用。所以需要逆序入栈，从而使得是正序出栈
      </p>
     </blockquote>
     <p>
      我们把程序执行到EIP=
      <code>
       40100E
      </code>
      处观察栈窗口
      <br/>
      <img alt="Pasted image 20250311181809" src="https://i-blog.csdnimg.cn/img_convert/dcd3862569bedac9947009341053ada0.png"/>
      <br/>
      从这个栈中取出数据就是正序的了。也就可以正确的传递给函数参数进行调用
     </p>
     <blockquote>
      <p>
       个人理解：因为栈是向低地址延申的。所以低地址是栈顶，高地址是栈底。所以出来的顺序就是
       <code>
        NULL
       </code>
       <code>
        Make me ....
       </code>
       也就是正序的了。
      </p>
     </blockquote>
     <h2>
      <a id="5__165">
      </a>
      5. 栈帧
     </h2>
     <p>
      <strong>
       作用
      </strong>
      ： 栈帧在程序中用于声明局部变量、调用函数。
      <br/>
      栈帧就是利用EBP寄存器访问栈内局部变量、参数、函数返回地址等的手段
      <br/>
      与ESP寄存器不同：ESP承担栈顶指针的作用。而EBP寄存器则负责行驶栈帧指针的职能
     </p>
     <p>
      程序运行中，ESP寄存器的值随时变化，访问栈中函数的局部变量、参数时，若以ESP值为基准编写程序会十分困难，并且也很难使CPU引用到准确的地址。
      <br/>
      所以，调用某函数时，
      <strong>
       先要把用作基准点（函数起始地址）的ESP值保存到EBP，并维持在函数内部
      </strong>
      。
      <br/>
      这样，无论ESP的值如何变化，以EBP的值为基准（base）能够安全访问到相关函数的局部变量、参数、返回地址，这就是EBP寄存器作为栈帧指针的作用。
     </p>
     <blockquote>
      <p>
       ●最新的编译器中都带有一个“优化”（Optimization）选项，使用该选项编译简单的函数将不会生成栈帧。
      </p>
      <p>
       ●在栈中保存函数返回地址是系统安全隐患之一，攻击者使用缓冲区溢出技术能够把保存在栈内存的返回地址更改为其他地址。
      </p>
     </blockquote>

      然后执行retn命令后。存储在栈中的返回地址即被返回， 此时调用栈也已经完全返回到被调用到
      <code>
       add
      </code>
      函数之前的状态
      <br/>
      <img alt="Pasted image 20250311230223" src="https://i-blog.csdnimg.cn/img_convert/546413e6427c4c943572e64ea22f110b.png"/>
     </p>
     <h4>
      <a id="518_add_327">
      </a>
      5.1.8. 从栈中删除函数add的参数（整理栈）
     </h4>
     <p>
      现在，函数执行流已经返回到
      <code>
       main
      </code>
      函数中
      <br/>
      然后执行
      <code>
       ADD esp,0x8
      </code>
      即回收空间。不需要a b参数了
      <br/>
      这里与前面开辟空间的
      <code>
       SUB esp,0x8
      </code>
      相对应
     </p>
     <blockquote>
      <p>
       被调函数执行完毕后，
       <strong>
        函数的调用者（Caller）
       </strong>
       负责清理存储在栈中的参数，这种方式称为
       <strong>
        cdecl
       </strong>
       方式；
       <br/>
       反之，
       <strong>
        被调用者（Callee）
       </strong>
       负责清理保存在栈中的参数，这种方式称为
       <strong>
        stdcall
       </strong>
       方式。
       <br/>
       这些函数调用规则统称为调用约定（CallingConvention)，这在程序开发与分析中是一个非常重要的概念
      </p>
     </blockquote>
     <h4>
      <a id="519_printf_336">
      </a>
      5.1.9. 调用printf函数
     </h4>
     <p>
      对应源代码
     </p>
     <pre><code class="prism language-cpp"><span class="token function">printf</span><span class="token punctuation">(</span><span class="token string">"%d\n"</span><span class="token punctuation">,</span><span class="token function">add</span><span class="token punctuation">(</span>a<span class="token punctuation">,</span>b<span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
     <p>
      <img alt="Pasted image 20250311231113" src="https://i-blog.csdnimg.cn/img_convert/a7743e00015baef529826316508ea34d.png"/>
     </p>
     <p>
      由于上面的
      <code>
       printf
      </code>
      函数有2个参数，大小为8个字节（32位寄存器+32位常量=64位=8字节），
      <br/>
      所以在
      <code>
       40104F
      </code>
      地址处使用
      <code>
       ADD
      </code>
      命令，将ESP加上8个字节，把函数的参数从栈中删除。函数
      <code>
       printf
      </code>
      执行完毕并通过
      <code>
       ADD
      </code>
      命令删除参数后 栈内情况
      <br/>
      <img alt="Pasted image 20250311231350" src="https://i-blog.csdnimg.cn/img_convert/5227b100469e19a0127c3e26367eb3f9.png"/>
     </p>
     <h4>
      <a id="5110__346">
      </a>
      5.1.10. 设置返回值
     </h4>
     <p>
      对应源代码
     </p>
     <pre><code class="prism language-cpp"><span class="token keyword">return</span> <span class="token number">0</span><span class="token punctuation">;</span>
</code></pre>
     <p>
      <code>
       main
      </code>
      函数使用该语句设置返回值0
      <br/>
      汇编代码
      <code>
       xor eax,eax
      </code>
      异或清0 且比
      <code>
       MOV eax,0
      </code>
      更快
     </p>
     <h4>
      <a id="5111_main_353">
      </a>
      5.1.11. 删除栈中&amp;main函数终止
     </h4>
     <p>
      对应源代码
     </p>
     <pre><code class="prism language-cpp"><span class="token keyword">return</span> <span class="token number">0</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
     <p>
      主函数终止，同
      <code>
       add
      </code>
      函数一样，需要先从栈中删除其对应的栈帧
      <br/>
      对应汇编代码
      <br/>
      <code>
       MOV ESP，EBP
      </code>
      <br/>
      <code>
       POP EBP
      </code>
      <br/>
      执行后，
      <code>
       main
      </code>
      函数栈帧被删除。其局部变量，a b 也不再生效。
      <br/>
      <img alt="Pasted image 20250311231845" src="https://i-blog.csdnimg.cn/img_convert/63586fef93996ae71b9aff9e7f0f5eec.png"/>
      <br/>
      此时与
      <code>
       main
      </code>
      函数开始的栈内情形是一样的
     </p>
     <p>
      然后执行
      <code>
       retn
      </code>
      程序流跳转到返回地址
      <code>
       401250
      </code>
      处，该地址指向Visual C++的启动函数区域。随后执行进程终止代码。
     </p>
