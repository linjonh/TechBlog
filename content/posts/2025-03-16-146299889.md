     Centos 7 升级 GCC
    </h1>

  <div class="article_content clearfix" id="article_content">

    <h2>
     一、环境准备
    </h2>
    <h3>
     1. 检查当前 GCC 版本
    </h3>
    <pre><code class="hljs">gcc --version</code></pre>
    <p>
     <img alt="" height="103" src="https://i-blog.csdnimg.cn/direct/90d4dbb96ccf4849818a0c6bf3354899.png" width="952"/>
    </p>
    <h3>
     2. 清理系统环境
    </h3>
    <pre><code class="hljs">sudo yum clean all

sudo yum update -y</code></pre>
    <hr/>
    <h2>
     二、安装 SCL 仓库
    </h2>
    <blockquote>
     <p>
      此处我们使用scl来切换gcc版本
     </p>
     <p>
      SCL（Software Collections）提供高版本软件包，且与系统默认工具链隔离。
     </p>
    </blockquote>
    <pre><code class="hljs">sudo yum install -y centos-release-scl</code></pre>
    <hr/>
    <h2>
     三、安装 GCC 9.3
    </h2>
    <h3>
     1. 安装 devtoolset-9
    </h3>
    <pre><code class="hljs">sudo yum install -y devtoolset-9-gcc devtoolset-9-gcc-c++</code></pre>
    <blockquote>
     <p>
      如果报错如下，是因为CentOS 7 的SCL仓库已停止维护，默认的镜像服务器已失效，导致无法通过yum安装
     </p>
     <p>
     </p>
    </blockquote>
    <p>
     <img alt="" height="1193" src="https://i-blog.csdnimg.cn/direct/b53fd814301e42e79cd9fccef466c6b8.png" width="1080"/>
    </p>
    <h3>
     解决方案 (切换为阿里云镜像环境)
    </h3>
    <h4>
     1、查看自己的源环境
    </h4>
    <blockquote>
     <p>
      主要查看 base是否为阿里云环境 以及 sclo rh 是否为阿里云环境(以下是配图)
     </p>
    </blockquote>
    <pre><code class="hljs">yum repolist</code></pre>
    <p>
     <img alt="" height="473" src="https://i-blog.csdnimg.cn/direct/6aa86279e5804a68b5803932b9d334cc.png" width="1505"/>
    </p>
    <h4>
     2、切换配置信息
    </h4>
    <pre><code class="hljs"># 使用阿里云镜像
sudo wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

修改所有仓库配置为阿里云地址：
sudo sed -i 's|vault.centos.org|mirrors.aliyun.com/centos|g' /etc/yum.repos.d/*.repo</code></pre>
    <h4>
     3、验证scl源 (如果和我的不一致可能回导致404 小伙伴可直接复制粘贴)
    </h4>
    <pre><code class="hljs">sudo vi /etc/yum.repos.d/CentOS-SCLo-scl-rh.repo</code></pre>
    <blockquote>
     <p>
      [centos-sclo-rh]
      <br/>
      name=CentOS-7 - SCLo rh
      <br/>
      baseurl=http://mirrors.aliyun.com/centos/7/sclo/x86_64/rh/
      <br/>
      enabled=1
      <br/>
      gpgcheck=1
      <br/>
      gpgkey=http://mirrors.aliyun.com/centos/centos/RPM-GPG-KEY-CentOS-SIG-SCLo
     </p>
    </blockquote>
    <p>
     <img alt="" height="176" src="https://i-blog.csdnimg.cn/direct/f224fc4667f7452dab4bab371a61a33c.png" width="1286"/>
    </p>
    <h4>
     4、重新构建
    </h4>
    <blockquote>
     <p>
      切记一定要多
     </p>
     <p>
      sudo yum clean all # 清除
      <br/>
      sudo yum update -y # 更新
     </p>
     <p>
      sudo yum makecache #构建
     </p>
    </blockquote>
    <pre><code class="hljs">sudo yum clean all
sudo yum makecache</code></pre>
    <p>
     证明没有任何问题
     <img alt="" height="608" src="https://i-blog.csdnimg.cn/direct/af18794883db4846821fedfb891607b5.png" width="837"/>
    </p>
    <h4>
     5、手动下载安装文件
    </h4>
    <blockquote>
     <p>
      理论上各位不需要执行这个步骤 重新执行  sudo yum install -y devtoolset-9-gcc devtoolset-9-gcc-c++  命令即可 如果出现像我一样 可依照我如此下载将文件名称换成自己对于的即可
     </p>
     <p>
     </p>
     <p>
      注意：我当时不知道为什么没有下载下下来 在执行命令时显示有五个文件404不找到  这是我手动配置的地址可以直接下载下来
     </p>
    </blockquote>
    <pre><code class="hljs">wget http://vault.centos.org/centos/7/sclo/x86_64/rh/Packages/d/devtoolset-9-runtime-9.1-0.el7.x86_64.rpm
wget http://vault.centos.org/centos/7/sclo/x86_64/rh/Packages/d/devtoolset-9-binutils-2.32-16.el7.x86_64.rpm
wget http://vault.centos.org/centos/7/sclo/x86_64/rh/Packages/d/devtoolset-9-gcc-9.3.1-2.2.el7.x86_64.rpm
wget http://vault.centos.org/centos/7/sclo/x86_64/rh/Packages/d/devtoolset-9-gcc-c++-9.3.1-2.2.el7.x86_64.rpm
wget http://vault.centos.org/centos/7/sclo/x86_64/rh/Packages/d/devtoolset-9-libstdc++-devel-9.3.1-2.2.el7.x86_64.rpm</code></pre>
    <h4>
     6、安装环境
    </h4>
    <pre><code class="hljs">sudo yum install -y policycoreutils-python
sudo yum install -y scl-utils</code></pre>
    <h4>
     7、编译安装文件
    </h4>
    <blockquote>
     <p>
      注意：如果你有其他文件 也仿照此格式添加上
     </p>
    </blockquote>
    <pre><code class="hljs">sudo rpm -Uvh --force --nodeps devtoolset-9-runtime-9.1-0.el7.x86_64.rpm \
           devtoolset-9-binutils-2.32-16.el7.x86_64.rpm \
           devtoolset-9-gcc-9.3.1-2.2.el7.x86_64.rpm \
           devtoolset-9-gcc-c++-9.3.1-2.2.el7.x86_64.rpm \
           devtoolset-9-libstdc++-devel-9.3.1-2.2.el7.x86_64.rpm</code></pre>
    <h5>
     8、验证 GCC 是否安装成功
    </h5>
    <pre><code class="hljs">sudo yum install -y devtoolset-9-gcc devtoolset-9-gcc-c++ </code></pre>
    <p>
     <img alt="" height="320" src="https://i-blog.csdnimg.cn/direct/4d24028cebe84b2b956922f8c288213c.png" width="1123"/>
    </p>
    <p>
    </p>
    <h3>
     2. 激活 GCC 9.3 环境
    </h3>
    <pre><code class="hljs">scl enable devtoolset-9 bash

# 验证版本
gcc --version</code></pre>
    <blockquote>
     <p>
      切换成功
     </p>
    </blockquote>
    <p>
     <img alt="" height="156" src="https://i-blog.csdnimg.cn/direct/9931b7ae85464f2a89aa2915e13dba67.png" width="953"/>
    </p>
    <hr/>
    <h2>
     四、永久切换为GCC 9.3
    </h2>
    <blockquote>
     <p>
      正常上述步骤后你的机器已经拥有了 9.3的环境,但是因为  scl enable devtoolset-9 bash  只是当前会话有效 当你新开会话之后 gcc 版本就会失效(小伙伴们可以打开一个新的窗口验证一下)，所以建议设置成永久使用
     </p>
    </blockquote>
    <h3>
     1. 修改用户环境变量
    </h3>
    <pre><code class="hljs"># 写入配置信息
echo 'source /opt/rh/devtoolset-9/enable #This is  gcc version' &gt;&gt; ~/.bashrc

# 验证是否添加成功
vim ~/.bashrc

# 刷新配置
source ~/.bashrc
</code></pre>
    <h3>
     2. 验证生效
    </h3>
    <blockquote>
     <p>
      打开新客户端 验证 gcc 版本是否生效
     </p>
    </blockquote>
    <p>
     <img alt="" height="142" src="https://i-blog.csdnimg.cn/direct/be8e9b76617149d09fad401d32071f51.png" width="1137"/>
    </p>
    <blockquote>
     <p>
      亲测可用
     </p>
     <p>
     </p>
     <p>
      注：这里如果不想启动 只需要把 写入的数据删除掉，打开新窗口就回变回原来的版本
     </p>
     <p>
      <br/>
      注：大家安装时 一定要注意自己的环境，目前我安装redis用到了 9.X的GCC环境就不做其他演示，个人理解如果想要升级至 GCC 11 也是此流程，文件不同罢了。
     </p>
    </blockquote>
