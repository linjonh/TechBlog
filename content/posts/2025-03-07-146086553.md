     K8S高可用Web应用部署方案
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     以下是基于 Kubernetes 的高可用 Web 应用部署方案，满足多副本、跨可用区容灾、HTTPS 配置以及滚动更新和回滚的需求：
    </p>
    <hr/>
    <h4>
     <strong>
      一、架构设计
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        多副本 + 跨可用区容灾
       </strong>
      </p>
      <ul>
       <li>
        <p>
         使用 Kubernetes 的
         <strong>
          Deployment
         </strong>
         部署多副本应用，并通过
         <strong>
          Pod 反亲和性
         </strong>
         将 Pod 分散到不同可用区。
        </p>
       </li>
       <li>
        <p>
         使用
         <strong>
          Cluster Autoscaler
         </strong>
         自动扩缩节点，确保资源充足。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通过 Ingress 暴露服务并配置 HTTPS
       </strong>
      </p>
      <ul>
       <li>
        <p>
         使用
         <strong>
          Ingress Controller
         </strong>
         （如 Nginx Ingress）暴露服务，并通过
         <strong>
          Cert-Manager
         </strong>
         自动管理 TLS 证书。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        滚动更新和回滚机制
       </strong>
      </p>
      <ul>
       <li>
        <p>
         使用 Deployment 的滚动更新策略，逐步替换旧版本 Pod。
        </p>
       </li>
       <li>
        <p>
         通过
         <code>
          kubectl rollout
         </code>
         命令实现版本回滚。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      二、详细实现步骤
     </strong>
    </h4>
    <h5>
     <strong>
      1. 多副本 + 跨可用区容灾
     </strong>
    </h5>
    <h6>
     <strong>
      1.1 创建 Deployment
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       使用以下 YAML 文件创建 Deployment，部署 3 个副本：
      </p>
      <pre>apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
  namespace: default
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web-app
  template:
    metadata:
      labels:
        app: web-app
    spec:
      affinity:
        podAntiAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
          - labelSelector:
              matchExpressions:
              - key: app
                operator: In
                values:
                - web-app
            topologyKey: topology.kubernetes.io/zone
      containers:
      - name: web-app
        image: my-web-app:1.0.0
        ports:
        - containerPort: 80
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "500m"
            memory: "512Mi"</pre>
      <ul>
       <li>
        <p>
         <strong>
          关键点
         </strong>
         ：
        </p>
        <ul>
         <li>
          <p>
           <code>
            replicas: 3
           </code>
           ：部署 3 个副本。
          </p>
         </li>
         <li>
          <p>
           <code>
            podAntiAffinity
           </code>
           ：确保 Pod 分散到不同可用区。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <h6>
     <strong>
      1.2 创建 Service
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       使用以下 YAML 文件创建 Service，暴露 Deployment：
      </p>
      <pre>apiVersion: v1
kind: Service
metadata:
  name: web-app
  namespace: default
spec:
  selector:
    app: web-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80</pre>
     </li>
    </ul>
    <hr/>
    <h5>
     <strong>
      2. 通过 Ingress 暴露服务并配置 HTTPS
     </strong>
    </h5>
    <h6>
     <strong>
      2.1 安装 Ingress Controller
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       使用 Helm 安装 Nginx Ingress Controller：
      </p>
      <pre>helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm install ingress-nginx ingress-nginx/ingress-nginx</pre>
     </li>
    </ul>
    <h6>
     <strong>
      2.2 安装 Cert-Manager
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       使用 Helm 安装 Cert-Manager：
      </p>
      <pre>helm repo add jetstack https://charts.jetstack.io
helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --create-namespace \
  --set installCRDs=true</pre>
     </li>
    </ul>
    <h6>
     <strong>
      2.3 创建 Ingress 资源
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       使用以下 YAML 文件创建 Ingress，配置 HTTPS：
      </p>
      <pre>apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: web-app
  namespace: default
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  tls:
  - hosts:
    - my-web-app.example.com
    secretName: web-app-tls
  rules:
  - host: my-web-app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: web-app
            port:
              number: 80</pre>
      <ul>
       <li>
        <p>
         <strong>
          关键点
         </strong>
         ：
        </p>
        <ul>
         <li>
          <p>
           <code>
            tls
           </code>
           ：配置 HTTPS，使用 Cert-Manager 自动签发 Let's Encrypt 证书。
          </p>
         </li>
         <li>
          <p>
           <code>
            host
           </code>
           ：替换为实际域名。
          </p>
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h5 style="background-color:transparent">
     <strong>
      3. 实现滚动更新和回滚机制
     </strong>
    </h5>
    <h6 style="background-color:transparent">
     <strong>
      3.1 滚动更新
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       更新 Deployment 的镜像版本：
      </p>
      <pre>kubectl set image deployment/web-app web-app=my-web-app:2.0.0</pre>
     </li>
     <li>
      <p>
       Kubernetes 会自动执行滚动更新，逐步替换旧版本 Pod。
      </p>
     </li>
    </ul>
    <h6 style="background-color:transparent">
     <strong>
      3.2 回滚机制
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       查看 Deployment 的更新历史：
      </p>
      <pre>kubectl rollout history deployment/web-app</pre>
     </li>
     <li>
      <p>
       回滚到上一个版本：
      </p>
      <pre>kubectl rollout undo deployment/web-app</pre>
     </li>
     <li>
      <p>
       回滚到指定版本：
      </p>
      <pre>kubectl rollout undo deployment/web-app --to-revision=2</pre>
     </li>
    </ul>
    <hr/>
    <h4 style="background-color:transparent">
     <strong>
      三、验证与测试
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        验证多副本与跨可用区容灾
       </strong>
      </p>
      <ul>
       <li>
        <p>
         查看 Pod 分布：
        </p>
        <pre>kubectl get pods -o wide</pre>
       </li>
       <li>
        <p>
         确认 Pod 分散到不同可用区。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        验证 HTTPS 配置
       </strong>
      </p>
      <ul>
       <li>
        <p>
         访问
         <code>
          https://my-web-app.example.com
         </code>
         ，确认证书有效且服务正常。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        验证滚动更新与回滚
       </strong>
      </p>
      <ul>
       <li>
        <p>
         更新镜像版本，观察 Pod 替换过程。
        </p>
       </li>
       <li>
        <p>
         执行回滚，确认应用恢复到旧版本。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      四、总结
     </strong>
    </h4>
    <p>
     通过以上方案，可以实现高可用的 Web 应用部署，满足多副本、跨可用区容灾、HTTPS 配置以及滚动更新和回滚的需求。关键点包括：
    </p>
    <ul>
     <li>
      <p>
       使用 Deployment 和 Pod 反亲和性实现多副本与跨可用区容灾。
      </p>
     </li>
     <li>
      <p>
       使用 Ingress 和 Cert-Manager 配置 HTTPS。
      </p>
     </li>
     <li>
      <p>
       利用 Deployment 的滚动更新和回滚机制实现无缝升级与回退。
      </p>
     </li>
    </ul>
