    <p>
     最近开始学习微信小程序，初步掌握了应用小程序在项目中的业务开发，但还是感觉有很多需要掌握的原理等知识点，所以就通过面试题来总结一下吧。（本菜鸟比较懒，之后有时间再慢慢写。。。）
    </p>
    <h2>
     一、小程序中有几种文件？
    </h2>
    <ul>
     <li>
      json文件：
      <code>
       json
      </code>
      是一种
      <strong>
       数据格式
      </strong>
      ，并不是编程语言，在小程序中，它扮演的是
      <strong>
       静态配置
      </strong>
      的角色。常见的json文件有3种：
      <ul>
       <li>
        <p>
         小程序配置
         <span style="color:#ed7976;">
          <code>
           <span style="background-color:#fef2f0;">
            app.json
           </span>
          </code>
         </span>
         ：做全局配置；
        </p>
       </li>
       <li>
        <p>
         页面配置
         <span style="color:#ed7976;">
          <code>
           <span style="background-color:#fef2f0;">
            page.json
           </span>
          </code>
         </span>
         ：对小程序具体页面的配置；
        </p>
       </li>
       <li>
        <p>
         工具配置
         <code>
          project.config.json
         </code>
         ：对开发者工具的个性化配置，如域名校验、代码上传时自动压缩等（参考
         <a href="https://developers.weixin.qq.com/miniprogram/dev/devtools/projectconfig.html" rel="nofollow" title="项目配置文件 | 微信开放文档">
          项目配置文件 | 微信开放文档
         </a>
         ）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       WXML模板：全称
       <span style="color:#ff9900;">
        （WeiXin Markup Language）
       </span>
       微信标记语言 ，结合小程序的基础组件、事件系统，可以构建出页面的结构。
      </p>
     </li>
     <li>
      WXSS：
      <span style="color:#ff9900;">
       （WeiXin Style Sheets）
      </span>
      小程序的样式语言，用于描述
      <font face="monospace">
       wxml
      </font>
      的组件的视觉效果，就相当于网页的css。
     </li>
     <li>
      JavaScript脚本语言：逻辑处理，网络请求。
     </li>
    </ul>
    <p>
    </p>
    <h2>
     二、小程序中的自定义事件，如何传值到js？
    </h2>
    <p>
     通过data-key实现，key的名称可以自定义。
    </p>
    <p>
     在js中通过event参数获取，event.currentTarget.dataset.key。
    </p>
    <p>
     注意：key不允许大写，不可以存放对象，有两种支持的写法：
    </p>
    <ul>
     <li>
      <code>
       data-element-type
      </code>
      ，最终会呈现为：
      <code>
       event.currentTarget.dataset.elementType
      </code>
      ；
     </li>
     <li>
      <code>
       data-elementType
      </code>
      ，最终会呈现为
      <code>
       event.currentTarget.dataset.elementtype
      </code>
      。
     </li>
    </ul>
    <p>
    </p>
    <h2>
     三、wxss和css有无不同？
    </h2>
    <p>
     为了适应广大的前端开发者，WXSS 具有 CSS 大部分特性。同时为了更适合开发微信小程序，WXSS 对 CSS 进行了扩充以及修改。
    </p>
    <p>
     与 CSS 相比，WXSS 扩展的特性有：
    </p>
    <ul>
     <li>
      尺寸单位：
      <span style="color:#494949;">
       支持rpx，可以根据屏幕宽度进行自适应，规定屏幕宽度为750rpx；
      </span>
     </li>
     <li>
      样式导入：
      <span style="color:#494949;">
       使用
       <code>
        @import
       </code>
       语句可以导入外联样式表，
       <code>
        @import
       </code>
       后跟需要导入的外联样式表的相对路径，用
       <code>
        ;
       </code>
       表示语句结束。
      </span>
     </li>
    </ul>
    <p>
     <span style="color:#494949;">
      区别：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#494949;">
       wxss 中的本地资源图片无法通过 WXSS 获取，可以使用网络图片，或者 base64，或者使用&lt;image/&gt;标签。
      </span>
     </li>
    </ul>
    <p>
    </p>
    <h2>
     四、小程序关联微信公众号如何确定用户的唯一性？
    </h2>
    <p>
     在小程序中有openid、union_id
    </p>
    <p>
     openid是同一用户同一应用唯一；
    </p>
    <p>
     union_id是同一用户不同应用唯一。
    </p>
    <p>
     所以，想要确定用户的唯一性，只能用union_id。
    </p>
    <p>
     使用
     <code>
      wx.getUserInfo
     </code>
     方法
     <code>
      withCredentials
     </code>
     为
     <code>
      true
     </code>
     时 可获取
     <code>
      encryptedData
     </code>
     ，里面有
     <code>
      union_id
     </code>
     。后端需要进行对称解密
    </p>
    <p>
    </p>
    <h2>
     五、小程序页面的生命周期
    </h2>
    <p>
     页面生命周期首先考虑三个事件：
    </p>
    <ul>
     <li>
      <p>
       监听页面初次加载：
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onLoad
         </span>
        </code>
       </span>
       ，在页面没被销毁之前只会触发1次，可以在
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onLoad
         </span>
        </code>
       </span>
       的参数中获取打开当前页面路径中的参数；
      </p>
     </li>
     <li>
      <p>
       监听页面显示：
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onShow
         </span>
        </code>
       </span>
       ，页面显示/切入前台时触发；
      </p>
     </li>
     <li>
      <p>
       监听页面初次渲染完成：
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onReady
         </span>
        </code>
       </span>
       ，在页面没被销毁前只会触发1次，代表页面已经准备妥当，可以和视图层进行交互。
      </p>
     </li>
    </ul>
    <p>
     页面显示后，随着用户的操作，还会触发其他事件：
    </p>
    <ul>
     <li>
      <p>
       监听页面隐藏：
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onHide
         </span>
        </code>
       </span>
       ，页面隐藏/切入后台时触发，
       <span style="color:#ed7976;">
        <span style="background-color:#fef2f0;">
         wx.navigateTo
        </span>
       </span>
       切换到其他页面、底部
       <span style="color:#ed7976;">
        <span style="background-color:#fef2f0;">
         tab
        </span>
       </span>
       切换时触发；
      </p>
     </li>
     <li>
      <p>
       监听页面卸载：
       <span style="color:#ed7976;">
        <code>
         <span style="background-color:#fef2f0;">
          onUnload
         </span>
        </code>
       </span>
       ，返回到其它页时，
       <span style="color:#ed7976;">
        <span style="background-color:#fef2f0;">
         wx.redirectTo
        </span>
       </span>
       或
       <span style="color:#ed7976;">
        <span style="background-color:#fef2f0;">
         wx.navigateBack
        </span>
       </span>
       使当前页面被微信客户端销毁回收时触发。
      </p>
     </li>
    </ul>
