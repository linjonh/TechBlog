    <h2 id="%E4%B8%80%E3%80%81%E5%AE%89%E8%A3%85%E5%89%8D%E6%8F%90%E6%9D%A1%E4%BB%B6">

    <h3 id="1.1%20Java%E7%8E%AF%E5%A2%83">

     </span>
    </h3>
    <p>
     Neo4j需要Java运行环境来运行，因此首先需要安装JDK。可以从Oracle官网或OpenJDK网站下载并安装适合Mac的JDK版本。假设你选择的是 Neo4j 5.25.1 ，应使用JDK 17或更高版本，选择的是 Neo4j 4.4.39，应使用JDK 11。
    </p>
    <h3 id="1.2%20Homebrew%EF%BC%88%E5%8F%AF%E9%80%89%EF%BC%89">
     <span style="color:#4da8ee;">
      1.2 Homebrew（可选）
     </span>
    </h3>
    <p>
     Homebrew是Mac上的一个包管理器，可以简化安装和管理软件的过程。如果你还没有安装Homebrew，可以通过在终端中输入以下命令来安装：
    </p>
    <pre><code class="language-bash">/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"</code></pre>
    <h2 id="%E4%BA%8C%E3%80%81%E4%B8%8B%E8%BD%BD%E5%B9%B6%E5%AE%89%E8%A3%85Neo4j">

    <h4 id="3.1%C2%A0%E8%AE%BE%E7%BD%AE%E7%8E%AF%E5%A2%83%E5%8F%98%E9%87%8F(%E5%8F%AF%E9%80%89)">
     <span style="color:#956fe7;">
      3.1 设置环境变量(可选)
     </span>
    </h4>
    <p>
     如果你打算从
     <span style="color:#fe2c24;">
      命令行启动Neo4j服务
     </span>
     ，你可能需要将其可执行文件的路径添加到你的系统PATH环境变量中。通常，Neo4j的可执行文件位于：
    </p>
    <ul>
     <li>
      Neo4j 桌面端：
      <code>
       /Applications/Neo4j/Desktop.app/Contents/Resources/app/bin
      </code>
      目录下。
     </li>
     <li>
      非桌面端：你解压保存的路径
      <code>
       /bin
      </code>
      目录下。
     </li>
    </ul>
    <p>
     你可以通过以下方式将其添加到PATH中（这里以bash shell为例）：
    </p>
    <pre><code class="language-bash">export PATH="/Applications/Neo4j Desktop.app/Contents/Resources/app/bin:$PATH"</code></pre>
    <p>
     将上述命令添加到你的
     <span style="color:#fe2c24;">
      <code>
       ~/.bash_profile
      </code>
      或
      <code>
       ~/.zshrc
      </code>
      文件
     </span>
     中（取决于你使用的shell），然后
     <span style="color:#fe2c24;">
      运行
      <code>
       source ~/.bash_profile
      </code>
      或
      <code>
       source ~/.zshrc
      </code>
      来使其生效
     </span>
     。
    </p>

     <span style="color:#ed7976;">
      五、创建一个简单的图
     </span>
    </h2>
    <p>
     Neo4j的Cypher查询语言是一套用于构建、查询和操作图数据库的声明式语言。
    </p>
    <p>
     在 Neo4j 中，图是由节点（Node）、关系（Relationship）和属性（Property）组成的。我们可以创建一个简单的图来练习 Cypher 查询。
    </p>
    <p class="img-center">
     <img alt="" height="359" src="https://i-blog.csdnimg.cn/direct/f57307db3b204d658eac17afa3665e62.png" width="600"/>
    </p>
    <h3 id="5.1%20%E5%88%9B%E5%BB%BA%E8%8A%82%E7%82%B9%2F%E5%85%B3%E7%B3%BB">
     <span style="color:#4da8ee;">
      5.1 创建节点/关系
     </span>
    </h3>
    <h4 id="5.1.1%C2%A0%E5%88%9B%E5%BB%BA%E8%8A%82%E7%82%B9%E5%B9%B6%E7%BB%99%E8%8A%82%E7%82%B9(n)%E5%88%86%E9%85%8D%E4%B8%80%E4%B8%AA%E5%B1%9E%E6%80%A7(name)">
     <span style="color:#956fe7;">
      5.1.1 创建节点并给节点(n)分配一个属性(name)
     </span>
    </h4>
    <blockquote>
     <p>
      <span style="color:#956fe7;">
      </span>
      CREATE (n {name:"曹操"})
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="204" src="https://i-blog.csdnimg.cn/direct/c5b7280994d248c3b629127e3be16618.png" width="600"/>
    </p>
    <p>
     这就表示我们创建成功了，在左侧 Node labels 中未发现该节点。我们执行查询节点是能操作到的，如下图：
    </p>
    <p class="img-center">
     <img alt="" height="201" src="https://i-blog.csdnimg.cn/direct/b0b17a1aaf334d778d8de94b3557970f.png" width="600"/>
    </p>
    <h4 id="%C2%A05.1.2%C2%A0%E5%88%9B%E5%BB%BA%E8%8A%82%E7%82%B9%E5%B9%B6%E5%88%86%E9%85%8D%E5%B1%9E%E6%80%A7%E5%92%8C%E6%A0%87%E7%AD%BE(Person)">
     <span style="color:#956fe7;">
      5.1.2 创建节点并分配属性和标签(Person)
     </span>
    </h4>
    <blockquote>
     <p>
      <span style="color:#956fe7;">
      </span>
      CREATE (p:Person{name:"曹操"})
     </p>
    </blockquote>
    <p>
     <img alt="" height="744" src="https://i-blog.csdnimg.cn/direct/08e3d43221b84d3290f2e8a9c9492448.png" width="1200"/>
    </p>
    <p>
     执行完后，你就可以看到左侧节点列表有数据了，还包括我们之前创建的节点。
    </p>

     <span style="color:#956fe7;">
      5.1.3 批量创建节点
     </span>
    </h4>
    <p>
     <span style="color:#956fe7;">
     </span>
     UNWIND是一种操作，用于将列表中的元素解构为多行。它允许按行访问和处理列表中的每个元素，更多地用于数据导入、转换和过滤等场景。
    </p>
    <p>
     无标签
    </p>
    <blockquote>
     <p>
      UNWIND [{name: '张辽'}, {name: '关羽'}] AS myNodes
     </p>
     <p>
      CREATE (n) SET n = myNodes
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="167" src="https://i-blog.csdnimg.cn/direct/c53f720bdd09457587db7606835506f7.png" width="600"/>
    </p>
    <p>
     有标签：如果我们只处理统一标签（
     <code>
      Person）
     </code>
     节点，我们可以这样的：
    </p>
    <blockquote>
     <p>
      UNWIND [{name: '貂蝉'}, {name: '吕布'}] AS myNode
     </p>
     <p>
      CREATE (n:Person {name: myNode.name})
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="341" src="https://i-blog.csdnimg.cn/direct/9cef914a28be4713a7bcceef5fc5b728.png" width="600"/>
    </p>
    <h4 id="5.1.4%C2%A0%E5%88%9B%E5%BB%BA%E4%B8%80%E4%B8%AA%E5%85%B3%E7%B3%BB">
     <span style="color:#956fe7;">
      5.1.4 创建一个关系
     </span>
    </h4>
    <p>
     <span style="color:#956fe7;">
     </span>
     给已经存在的两个节点创建一个关系。
    </p>
    <blockquote>
     <p>
      MATCH (a:Person),(b:Person)
      <br/>
      WHERE a.name = '吕布' AND b.name = '貂蝉'
      <br/>
      CREATE (a)-[r:妻子]-&gt;(b)
      <br/>
      RETURN r
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="117" src="https://i-blog.csdnimg.cn/direct/b0b3a20872234eae946ec1d2ff3b3de7.png" width="400"/>
    </p>
    <h3 id="5.2%20%E4%BF%AE%E6%94%B9%E8%8A%82%E7%82%B9%2F%E5%85%B3%E7%B3%BB">
     <span style="color:#4da8ee;">
      5.2 修改节点/关系
     </span>
    </h3>
    <h4 id="5.2.1%20%E4%B8%BA%E8%8A%82%E7%82%B9%E6%B7%BB%E5%8A%A0%E5%B1%9E%E6%80%A7">
     <span style="color:#956fe7;">
      5.2.1 为节点添加属性
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n {name:'吕布'})
      <br/>
      SET n.age=18
      <br/>
      RETURN n
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="193" src="https://i-blog.csdnimg.cn/direct/280a0b75ccb44851a1fa9e5431205893.png" width="350"/>
    </p>
    <h4 id="5.2.2%C2%A0%E4%B8%BA%E8%8A%82%E7%82%B9%E6%B7%BB%E5%8A%A0%E6%A0%87%E7%AD%BE">
     <span style="color:#956fe7;">
      5.2.2 为节点添加标签
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n{name:'关羽'}) SET n:Person RETURN n
     </p>
    </blockquote>
    <p>
     <img alt="" height="141" src="https://i-blog.csdnimg.cn/direct/ca0c0baa9fb14d6e8c5fdbebe0cf5c35.png" width="330"/>
     <img alt="" height="151" src="https://i-blog.csdnimg.cn/direct/ff22908045a546a28061a731ecc7bd63.png" width="330"/>
    </p>
    <h3 id="5.3%20%E5%8C%B9%E9%85%8D(%E6%9F%A5%E6%89%BE)%E8%8A%82%E7%82%B9%2F%E5%85%B3%E7%B3%BB">
     <span style="color:#4da8ee;">
      5.3 匹配(查找)节点/关系
     </span>
    </h3>
    <p>
     Cypher 的
     <span style="color:#fe2c24;">
      MATCH 语句和 SQL 的 SELECT 语句类似,都是查询数据的表达式。
     </span>
     而Cypher语句中的 WHERE 和 SQL 语句中的 WHERE 语句由这相同的作用,其作用是用来过滤匹配结的,在Cypher 语句中 WHERE 是在 MATCH、OPTIONAL MATCH、WITH 或 START 语句中使用的过滤限定语句。
    </p>
    <p>
     <code>
      匹配语句结构
     </code>
    </p>
    <ul>
     <li>
      [MATCH WHERE]
     </li>
     <li>
      [OPTIONAL MATCH WHERE]
     </li>
     <li>
      [WITH [ORDER BY] [SKIP] [LIMIT]]
     </li>
     <li>
      RETURN [ORDER BY] [SKIP] [LIMIT]
     </li>
    </ul>
    <h4 id="5.3.1%C2%A0%E6%A0%B9%E6%8D%AE%E5%B1%9E%E6%80%A7%E5%8C%B9%E9%85%8D%E8%8A%82%E7%82%B9%E4%BF%A1%E6%81%AF">
     <span style="color:#956fe7;">
      5.3.1 根据属性匹配节点信息
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n{name:"吕布"}) RETURN n
     </p>
     <p>
      MATCH (n{age:18}) RETURN n
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="188" src="https://i-blog.csdnimg.cn/direct/8e5cd249ee0748b5a75394a47b01d539.png" width="400"/>
    </p>
    <h4 id="5.3.2%C2%A0%E6%A0%B9%E6%8D%AE%E6%A0%87%E7%AD%BE%E5%92%8C%E5%B1%9E%E6%80%A7%E5%8C%B9%E9%85%8D%E8%8A%82%E7%82%B9%E4%BF%A1%E6%81%AF">
     <span style="color:#956fe7;">
      5.3.2 根据标签和属性匹配节点信息
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n:Person{name:"吕布"}) RETURN n
     </p>
    </blockquote>
    <h4 id="%C2%A05.3.3%C2%A0%E5%8C%B9%E9%85%8D%E4%BB%BB%E6%84%8F%E8%8A%82%E7%82%B9">
     <span style="color:#956fe7;">
      5.3.3 匹配任意节点
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n) RETURN n
     </p>
    </blockquote>
    <h4 id="%C2%A05.3.4%20%E5%8C%B9%E9%85%8D%E4%BB%BB%E6%84%8F%E5%85%B3%E7%B3%BB">
     <span style="color:#956fe7;">
      5.3.4 匹配任意关系
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH p = (n)-[r]-&gt;(m) RETURN p
     </p>
    </blockquote>
    <h4 id="5.3.5%C2%A0%E5%8C%B9%E9%85%8D%E5%AD%97%E7%AC%A6%E4%B8%B2%E7%9A%84%E5%BC%80%E5%A4%B4">
     <span style="color:#956fe7;">
      5.3.5 匹配字符串的开头
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n)
     </p>
     <p>
      WHERE n.name
      <span style="color:#fe2c24;">
       STARTS
      </span>
      WITH '吕'
     </p>
     <p>
      RETURN n
     </p>
    </blockquote>
    <h4 id="5.3.6%C2%A0%E5%8C%B9%E9%85%8D%E5%AD%97%E7%AC%A6%E4%B8%B2%E7%9A%84%E6%9C%AB%E5%B0%BE">
     <span style="color:#956fe7;">
      5.3.6 匹配字符串的末尾
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n)
     </p>
     <p>
      WHERE n.name
      <span style="color:#fe2c24;">
       ENDS
      </span>
      WITH '布'
     </p>
     <p>
      RETURN n
     </p>
    </blockquote>
    <h4 id="5.3.7%C2%A0%E5%8C%B9%E9%85%8D%E5%AD%97%E7%AC%A6%E4%B8%B2%E7%9A%84%E5%8C%85%E5%90%AB%E5%8C%B9%E9%85%8D">
     <span style="color:#956fe7;">
      5.3.7 匹配字符串的包含匹配
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n)
     </p>
     <p>
      WHERE n.name
      <span style="color:#fe2c24;">
       CONTAINS
      </span>
      WITH '吕'
     </p>
     <p>
      RETURN n
     </p>
    </blockquote>
    <h4 id="5.3.8%20%E5%AD%97%E7%AC%A6%E4%B8%B2%E6%8E%92%E9%99%A4%E5%8C%B9%E9%85%8D%C2%A0">
     <span style="color:#956fe7;">
      5.3.8 字符串排除匹配
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n)
     </p>
     <p>
      WHERE NOT n.name STARTS WITH '吕'
     </p>
     <p>
      RETURN n
     </p>
    </blockquote>
    <p class="img-center">
     <img alt="" height="359" src="https://i-blog.csdnimg.cn/direct/af75fbdb89c0409dbfbacf7ce9cf9ba3.png" width="500"/>
    </p>
    <p>
     其他还有很多，如字符串正则表达式等，就不一一列举了。
    </p>
    <h3 id="5.4%20%E5%88%A0%E9%99%A4%E8%8A%82%E7%82%B9%2F%E5%85%B3%E7%B3%BB%2F%E8%A1%A8">
     <span style="color:#4da8ee;">
      5.4 删除节点/关系/表
     </span>
    </h3>
    <h4 id="5.4.1%C2%A0%E5%88%A0%E9%99%A4%E8%8A%82%E7%82%B9%E5%B1%9E%E6%80%A7">
     <span style="color:#956fe7;">
      5.4.1 删除节点属性
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n{name:'夏侯淳'}) remove n.age RETURN n
     </p>
    </blockquote>
    <ul>
     <li>
      <code>
       MATCH语句
      </code>
      用于找到满足条件的节点。在这里，它查找所有标签为
      <code>
       Person
      </code>
      且属性
      <code>
       name
      </code>
      值为"'夏侯淳'"的节点。
     </li>
    </ul>
    <h4 id="5.4.2%C2%A0%E5%88%A0%E9%99%A4%E8%8A%82%E7%82%B9%E6%A0%87%E7%AD%BE">
     <span style="color:#956fe7;">
      5.4.2 删除节点标签
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (n{name:'夏侯淳'}) REMOVE n:Person RETURN n
     </p>
    </blockquote>
    <h4 id="5.4.3%C2%A0%E5%88%A0%E9%99%A4%E8%8A%82%E7%82%B9">
     <span style="color:#956fe7;">
      5.4.3 删除节点
     </span>
    </h4>
    <blockquote>
     <p>
      MATCH (p:Person {name: '夏侯淳'})
     </p>
     <p>
      DELETE p
     </p>
    </blockquote>
    <ul>
     <li>
      <code>
       DELETE语句
      </code>
      用于删除找到的节点。由于Neo4j中的关系是基于节点的，因此删除节点也会删除与该节点相连的所有关系。
     </li>
    </ul>
    <h4 id="5.4.4%20%E5%88%A0%E9%99%A4%E5%85%B3%E7%B3%BB">
     <span style="color:#956fe7;">
      5.4.4 删除关系
     </span>
    </h4>
    <p>
     仅删除a=&gt;b关系(所有)，
     <span style="color:#fe2c24;">
      保留节点
     </span>
     。
    </p>
    <blockquote>
     <p>
      MATCH (a)-[r]-&gt;(b)
      <br/>
      WHERE a.name = '节点A的名称' AND b.name = '节点B的名称'
      <br/>
      DELETE r
     </p>
     <p>
     </p>
     <p>
      MATCH (a)-[r]-&gt;(b)
     </p>
     <p>
      WHERE a.name = '吕布' AND b.name = '貂蝉'
     </p>
     <p>
      DELETE r
     </p>
    </blockquote>
    <h4 id="5.4.5%20%E5%88%A0%E9%99%A4%E5%85%B3%E7%B3%BB%E5%8F%8A%E8%8A%82%E7%82%B9">
     <span style="color:#956fe7;">
      5.4.5 删除关系及节点
     </span>
    </h4>
    <p>
     删除关系以及
     <span style="color:#fe2c24;">
      至少一端没有其他关系的节点
     </span>
     。
    </p>
    <blockquote>
     <p>
      MATCH (a)-[r]-&gt;(b)
      <br/>
      WHERE a.name = '节点A的名称' AND b.name = '节点B的名称'
      <br/>
      DETACH DELETE r
     </p>
    </blockquote>
    <h4 id="5.4.6%20%E5%88%A0%E9%99%A4%E8%8A%82%E7%82%B9%E6%89%80%E6%9C%89%E5%85%B3%E7%B3%BB">
     <span style="color:#956fe7;">
      5.4.6 删除节点所有关系
     </span>
    </h4>
    <p>
     删除节点
     <code>
      a
     </code>
     和
     <code>
      b
     </code>
     以及它们与其他任何节点之间的所有关系。
    </p>
    <blockquote>
     <p>
      MATCH (a), (b)
      <br/>
      WHERE a.name = '节点A的名称' AND b.name = '节点B的名称'
      <br/>
      DELETE a, b
     </p>
    </blockquote>
    <h4 id="5.4.7%20%E5%88%A0%E9%99%A4%E6%89%80%E6%9C%89%E8%8A%82%E7%82%B9%E5%92%8C%E5%85%B3%E7%B3%BB">
     <span style="color:#956fe7;">
      5.4.7
      <strong>
       删除所有节点和关系
      </strong>
     </span>
    </h4>
    <p>
     如果你想要清空整个图数据库，可以删除所有的带有关系的节点及关系。但请注意，这是一个
     <span style="color:#fe2c24;">
      非常危险的操作，因为它会永久删除所有数据
     </span>
     。
    </p>
    <p>
     独立没有关系的节点会被保留下来。
    </p>
    <blockquote>
     <p>
      MATCH (n)-[r]-()
      <br/>
      DELETE n, r
     </p>
    </blockquote>
    <h4 id="5.4.7%20%E5%88%A0%E9%99%A4%E6%89%80%E6%9C%89%E8%8A%82%E7%82%B9">
     <span style="color:#956fe7;">
      5.4.7 删除所有节点
     </span>
    </h4>
    <p>
     如果你想要清空整个图数据库，可以删除所有的节点。但请注意，这是一个
     <span style="color:#fe2c24;">
      非常危险的操作，因为它会永久删除所有数据
     </span>
     。
    </p>
    <blockquote>
     <p>
      MATCH (n) DELETE n
     </p>
    </blockquote>
