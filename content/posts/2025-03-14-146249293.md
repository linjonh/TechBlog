    <h2>
     以太网协议
    </h2>
    <ul>
     <li>
      以太网是当今现有局域网(Local Area Network,LAN)采用的最通用的通信协议标准，该标准定义了在局域网中采用的电缆类型和信号处理方法。
     </li>
     <li>
      以太网是建立在CSMA/CD(Carrier Sense Multiple Access/Collision Detection,载波监听多路访问/冲突检测)机制上的广播型网络。
     </li>
    </ul>
    <p>
    </p>
    <ul>
     <li>
      先听后发
     </li>
     <li>
      边听边发
     </li>
     <li>
      冲突避退
     </li>
     <li>
      随机重发
     </li>
    </ul>
    <h2>
     以太网卡
    </h2>
    <p>
     网络接口卡(Network Interface Card,NIC)也称为“网卡”是计算机、交换机、路由器等网络设备与外部网络世界相连的关键部件。
    </p>
    <h2>
     以太网帧
    </h2>
    <ul>
     <li>
      以太网技术所使用的帧称为以太网帧(Ethernet Frame)，或简称以太帧  Type &gt;= 1536
     </li>
     <li>
      以太帧的格式有两个标准:EthernetII格式和IEEE802.3格式。                Type&lt;1536
     </li>
    </ul>
    <p>
     <img alt="" height="244" src="https://i-blog.csdnimg.cn/direct/2731c911b81d4fa0bc0503b7b7b5814b.png" width="876"/>
    </p>
    <p>
     MTU = 1500
    </p>
    <p>
    </p>
    <h2>
     MAC地址
    </h2>
    <p>
     48bit
    </p>
    <h3>
     分类
    </h3>
    <ul>
     <li>
      OUI(organizationally Unique Identifier):厂商代码，由IEEE分配，3 Byte,24 bit。
     </li>
     <li>
      制造商分配:3Byte，24 bit
     </li>
    </ul>

     <li>

      丢弃（入接口和出接口是一个）
     </li>

    <h2>
     VLAN（虚拟局域网）
    </h2>
    <ul>
     <li>
      虚拟局域网VLAN可以隔离广播域。
     </li>
    </ul>
    <p>
     特点:
    </p>
    <ul>
     <li>
      不受地域限制。
     </li>
     <li>
      同一VLAN内的设备才能直接进行二层通信。
     </li>
    </ul>
    <h3>
     VLAN标签
    </h3>
    <p>
     4bits
    </p>
    <h3>
     VLAN划分
    </h3>
    <p>
     <img alt="" height="398" src="https://i-blog.csdnimg.cn/direct/4ae606f781d04f7b9890d4eb45bf946c.png" width="894"/>
    </p>
    <p>
     先在交换机接口配置PVID  就是VLAN num
    </p>
    <h2>
     以太网二层接口类型
    </h2>
    <p>
     Access接口
     <br/>
     交换机上常用来连接用户PC、服务器等终端设备的接口。Access接口所连接的这些设备的网卡往往只收发无标记帧。Access接口只能加入一个VLAN。
     <br/>
     Trunk接口
     <br/>
     Trunk接口允许多个VLAN的数据帧通过，这些数据帧通过802.1QTag实现区分。Trumk接口常用于交换机之间的互联，也用于连接路由器防火墙等设备的子接口。
     <br/>
     Hybrid接口（混合）
     <br/>
     Hybrid接口与Trunk接口类似，也允许多个VLAN的数据帧通过，这些数据帧通过802.1Q Tag实现区分。用户可以灵活指定Hybrid接口在发送某个(或某些)VLAN的数据帧时是否携带Tag。
    </p>

    <h3>
     配置BPDU的比较原则
    </h3>
    <p>
     STP按照如下顺序选择最优的配置BPDU:
    </p>
    <ul>
     <li>
      1.最小的根桥ID
     </li>
     <li>
      2.最小的RPC
     </li>
     <li>
      3.最小的网桥ID
     </li>
     <li>
      4.最小的接口ID
     </li>
    </ul>
    <h3>
     STP计算过程
    </h3>
    <ul>
     <li>
      选择根桥
     </li>
     <li>
      选择根端口          非根桥上里根桥最近的端口  比较收到的BPDU
     </li>
     <li>
      选择指定端口       每一条链路选一个（非根桥上除了根端口 剩下的端口比较   一般根桥的端口都为指定端口   比较收到的BPDU）
     </li>
     <li>
      阻塞非指定端口
     </li>
    </ul>
