     (一)自定义镜像
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b0d7dc92bd624bf8a5d8170b09b1588d.png" width="360"/>

    <h3>
     <a id="_2">
     </a>
     一、本地目录挂载
    </h3>
    <p>
     将来我们自己开发好了java应用并且要用docker去部署，那么就需要去自行制作成镜像，也就是自定义镜像。
    </p>
    <blockquote>
     <p>
      假设我们需要往Linux服务器上直接部署java应用而不使用docker，那么在这之前就需要给该环境安装好JRE，也就是JVM运行环境，再把jar包传上去并启动就能部署成功。
      <br/>
      但是我们要想该系统拿到我们的jar包就能直接运行，不需要以上步骤，那么只要使用docker并向java项目镜像文件中准备好JRE、jar包就可以了吗?我们都知道java应用之所以能跨系统运行就是因为JRE与JVM，但是JVM最终是要跟操作系统交互的，不同操作系统的JVM也不一样。那么在制作镜像时就需要针对将要部署的Linux发行版来准备JRE以及该系统的全部函数库，那么它就可以不依赖于所部署的操作系统。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ac2fdbb9d8144f808d7c0423d8a0afc3.png" width="460">
      <br/>
      假设现在要基于Ubuntu发行版来制作自定义java镜像
     </img>
    </p>
    <ol>
     <li>
      准备Linux运行环境及函数库
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e8f23e5144b340f6a59caf8613bb413f.png" width="360"/>
     </li>
     <li>
      准备JRE并配置环境变量
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0ec3d41a8172421d9cdbf42516628c06.png" width="360"/>
     </li>
     <li>
      拷贝jar包
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a0e2d090a12d4d25b5a8ed1b285fa6fc.png" width="360"/>
     </li>
     <li>
      编写运行脚本
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f48488fb4f8440f38913d5a35c904c44.png" width="360"/>
     </li>
    </ol>
    <p>
     最终构建好java镜像
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e94682fa0952405dbb49e3d43b0bdd0b.png" width="260"/>
    </p>
    <blockquote>
     <p>
      实际上docker是把该应用的不同部分分别打成包最后再合并在一起称为一个完整的包，也将此一份份包称之为
      <strong>
       层(Layer)
      </strong>
      。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/183d0d16b3e64b5b9afab09ad7af0f2d.png" width="460">
      <br/>
      像这样分层后，假设我们需要针对不同Linux系统去制作java镜像，我们只需要去修改(重新制作)基础镜像层的内容即可，而不需要又去重新制作上层镜像内容，上传镜像也就只需要去上传基础镜像并共享前面几层镜像即可。
      <br/>
      这样制作的好处：分层解耦、复用、节省空间。
      <br/>
      假设现在我们去拉取一个Redis镜像，就能看到它分为几层来进行下载，已下载的层也不会再重复下载。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/aa7c1b511004405d8ceacf27f712d19b.png" width="460"/>
     </img>
    </p>
    <h3>
     <a id="Dockerfile_29">
     </a>
     二、Dockerfile
    </h3>
    <p>
     可以看到在docker中把一整个镜像文件分为了多层结构，不过实际上我们并不需要亲手去制作镜像，只需要借助Dockerfile去描述清楚
     <strong>
      镜像的结构(入口、层、基础镜像)
     </strong>
     ，docker就可以自动帮我们完成整个镜像的构建。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8eb7baab512049e7a8c4341ff71569d6.png" width="560"/>
    </p>
    <ul>
     <li>
      From指的是希望镜像从哪开始，可以去指定需要的Linux发行版，docker就会自动去找对应的基础镜像，在此基础上去完成对应操作。
     </li>
     <li>
      Entrypoint指的是入口，比如java应用的入口启动脚本就是java -jar xxx.jar。
      <br/>
      中间层就是要把java、jar包拷入，再来配置环境变量。
     </li>
    </ul>
    <h4>
     <a id="1_35">
     </a>
     (1)案例示范
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8257a0a232e74d0ab17eed937356820f.png" width="460"/>
     <br/>
     像这样的java应用部署过程还是有点繁琐，因为不同java应用都要用到JDK，而不同java应用变化的仅仅是jar包的名称，所以在这整个镜像结构当中将来在部署java应用时真正会变化的就是架构的名称，所以如果每次构建java应用镜像都得重新写这名字就会比较繁琐。
     <br/>
     也就是说可以把其他不变的配置，比如JDK、环境变量等，都抽取出来，制作为一个基础镜像。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/68f01b88b89544b1adb75654efd6e25a.png" width="560"/>
     <br/>
     在这个openjdk基础镜像当中就已经配置好了所有JDK的环境，以后只需要执行拷贝jar包以及设置入口的命令即可。
    </p>

    <h2>

     (二)容器网络
    </h2>
    <h3>
     <a id="_67">
     </a>
     一、网络介绍
    </h3>
    <p>
     分别查看mysql与dd容器的网络配置状态
    </p>
    <ol>
     <li>
      mysql容器
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6a4e080f90e64e068189b2164f1aadb1.png" width="360"/>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a1b3f74edfa24623a6617b8cf50f5755.png" width="560"/>
      <br/>
      还拥有自己的IP地址
     </li>
     <li>
      dd容器
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/191627eccea9408a8e91bb62f055396c.png" width="360"/>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4902241b52da419db7a4bc574bb02457.png" width="560"/>
      <br/>
      同样拥有自己的IP地址。
     </li>
    </ol>
    <p>
     我们可以发现这两个IP地址非常相似，只有最后一位不相同，说明这两个容器其实是在同一个网段当中，也就意味着它们之间可以互相访问。但是前面提到docker容器之间都是自己独立的隔离空间，为什么还有相同网段可以互相访问，这是因为他们都有相同的网关"Gateway"。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5602a5d9c12c4b68b914a69dee35f03f.png" width="560"/>
     <br/>
     在我们安装docker时，它会在虚拟机内创建一张虚拟网卡，名字默认为docker0，它拥有自己的范围地址。并且还会给该网卡创建一个虚拟的网桥，前面mysql与dd容器的IP地址都在docker0的地址范围内，也就说明它们都与docker0通过网桥连上了，也说明它们被分配了一个在该范围内的IP地址。
    </p>
    <p>
     通过ip addr命令就能查看到当前虚拟机上的网卡。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5424a59d5beb477b9f223d267c92c8b4.png" width="560"/>
     <br/>
     可以看到docker0的网卡。
    </p>
    <p>
     此时进入到dd容器当中再来ping mysq容器的IP地址，发现能成功ping通。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/00aa0aaccd794752bda8477cba2938e5.png" width="560"/>
     <br/>
     但是并不能说明该dd服务就能与mysql数据库连接，因为前面说到该IP地址是由docker网桥进行分配的，若该服务重新启动，那么它的IP地址就会改变或被其他服务占用。说明以后在java项目当中若要配置MySQL的地址就不能直接配置为该固定地址。
    </p>
    <blockquote>
     <p>
      因此原来这种IP地址连接的方式并不靠谱，也就需要去用到自定义网络了。
     </p>
    </blockquote>
    <h3>
