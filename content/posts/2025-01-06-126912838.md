    <h2>
     一、数据库安全概况
    </h2>
    <h3>
     1.1 数据库安全概念
    </h3>
    <p>
     数据库是网络信息系统的基础性软件，
     <span style="color:#ad720d;">
      承载着各种各样的数据
     </span>
     ，成为应用系统的支撑平台
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        国外主流
       </strong>
      </span>
      的数据库系统：有 MS SQL、MySQL、Oracle、DB2 等
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        国产
       </strong>
      </span>
      数据库系统：主要有人大金仓、达梦等
     </li>
    </ul>
    <p>
     <span style="color:#1c7892;">
      <strong>
       数据库安全：
      </strong>
     </span>
     是指数据库的机密性、完整性、可用性能够得到保障，其
     <strong>
      主要涉及
     </strong>
     <span style="color:#ad720d;">
      数据库管理安全、数据安全、数据库应用安全以及数据库运行安全
     </span>
    </p>
    <h3>
     1.2 数据库安全威胁
    </h3>
    <p>
     数据库所处的环境日益开放，所面临的数据库安全威胁日益增多，
     <strong>
      包括：
     </strong>
    </p>
    <ol>
     <li>
      <span style="color:#1c7892;">
       <strong>
        授权的误用
       </strong>
      </span>
      (Misuses of Authority)：
      <strong>
       合法用户
      </strong>
      <span style="color:#ad720d;">
       越权获得他们不应该获得的资源
      </span>
      ，窃取程序或存储介质，修改或破坏数据。
      <strong>
       授权用户
      </strong>
      <span style="color:#ad720d;">
       将自身的访问特权不适当地授予其他用户
      </span>
      ，导致系统安全策略受到威胁，使用户数据泄露
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        逻辑推断和汇聚
       </strong>
      </span>
      (Logical Inference and Aggregation)：
      <span style="color:#ad720d;">
       利用逻辑推理，把不太敏感的数据结合起来可以推断出敏感信息
      </span>
      。进行逻辑推断也可能要用到某些数据库系统以外的知识。与逻辑推断紧密相关的是数据汇聚安全问题，即个别的数据项是不敏感的，但是当足够多的个别数据值收集在一起时，就成为敏感的数据集
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        伪装
       </strong>
      </span>
      (Masquerade)：攻击者
      <span style="color:#ad720d;">
       假冒用户身份获取数据库系统的访问权限
      </span>
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        旁路控制
       </strong>
      </span>
      (Bypassing Controls)：在数据库
      <span style="color:#ad720d;">
       设置后门，绕过数据库系统的安全访问控制机制
      </span>
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        隐蔽信道
       </strong>
      </span>
      (Covert Channels)：通常储存在数据库中的数据
      <strong>
       经由合法的数据信道被取出
      </strong>
      。与正常的合法信道相反，
      <strong>
       隐蔽信道
      </strong>
      <span style="color:#ad720d;">
       利用非正常的通信途径传输数据以躲避数据库安全机制的控制
      </span>
      ，如共享内存、临时文件
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        SQL注入攻击
       </strong>
      </span>
      (SQL Injection)：攻击者
      <span style="color:#ad720d;">
       利用数据库应用程序的输入未进行安全检查的漏洞，欺骗数据库服务器执行恶意的SQL命令
      </span>
      。SQL注入攻击常常导致数据库信息泄露，甚至会造成数据系统的失控
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库口令密码破解：
       </strong>
      </span>
      <span style="color:#ad720d;">
       利用口令字典或手动猜测数据库用户密码，以达到非授权访问数据库系统的目的
      </span>
      。互联网常见的
      <strong>
       黑客攻击技术手段
      </strong>
      有
      <strong>
       “撞库”，其
       <span style="color:#ad720d;">
        技术原理
       </span>
      </strong>
      就是通过收集互联网己泄露的用户+口令密码信息，生成对应的字典表，尝试批量登录其他网站后，得到一系列可以登录的用户
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        硬件及介质攻击：
       </strong>
      </span>
      <span style="color:#ad720d;">
       对数据库系统相关的设备和存储介质的物理攻击
      </span>
     </li>
    </ol>
    <h3>
     1.3 数据库安全隐患
    </h3>
    <ol>
     <li>
      <strong>
       数据库用户账号和密码隐患：
      </strong>
      虽然多数数据库提供基本安全功能，但是
      <span style="color:#ad720d;">
       没有机制来限制用户必须选择健壮的密码
      </span>
      。多数数据库系统
      <span style="color:#ad720d;">
       有公开的默认账号和默认密码
      </span>
     </li>
     <li>
      <strong>
       数据库系统扩展存储过程隐患：
      </strong>
      多数数据库系统提供了“扩展存储过程”的服务以满足数据库管理，但是这也成为数据库系统的后门
     </li>
     <li>
      <strong>
       数据库系统软件和应用程序漏洞：
      </strong>
      软件程序漏洞造成数据安全机制或 OS 安全机制失效，攻击者可以获取远程访问权限
     </li>
     <li>
      <strong>
       数据库系统权限分配隐患：
      </strong>
      数据库管理员分配给用户的权限过大，导致用户误删除数据库系统数据，或者泄露数据库敏感数据。
     </li>
     <li>
      <strong>
       数据库系统用户安全意识薄弱：
      </strong>
      数据库系统用户选择弱口令或者口令保管不当都会给攻击者提供进入系统的机会
     </li>
     <li>
      <strong>
       网络通信内容是明文传递：
      </strong>
      利用数据库和应用程序之间网络通信内容未经加密的漏洞，网络窃听者窃取诸如应用程序特定数据或数据库登录凭据等敏感数据
     </li>
     <li>
      <strong>
       数据库系统安全机制不健全：
      </strong>
      导致安全策略无法实施
     </li>
    </ol>
    <h3>
     1.4 数据库安全需求
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       数据库系统的安全目标：
      </strong>
     </span>
     是
     <span style="color:#ad720d;">
      保护数据库系统的安全运行
     </span>
     及
     <span style="color:#ad720d;">
      数据资源的安全性
     </span>
     ，
     <strong>
      包括
     </strong>
    </p>
    <ol>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库标识与鉴别：
       </strong>
      </span>
      用于数据库用户身份识别和认证
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库访问控制：
       </strong>
      </span>
      对数据资源及系统操作进行访问授权及违规控制。数据库系统一般提供
      <span style="color:#ad720d;">
       自主访问控制、强制访问控制、角色访问控制
      </span>
      等
      <strong>
       多种访问控制模式
      </strong>
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库安全审计：
       </strong>
      </span>
      按照审计安全策略，对相关的数据库操作进行记录，形成
      <strong>
       数据库审计文件
      </strong>
      。
      <strong>
       审计记录的主要信息
      </strong>
      包括审计的操作类型、执行操作的用户标识、操作的时间、客体对象等用户行为相关信息
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库备份与恢复：
       </strong>
      </span>
      当数据库系统出现故障时，可以实现对备份数据的还原以及利用数据库日志进行数据库恢复重建
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库加密：
       </strong>
      </span>
      对数据库中的敏感数据进行加密处理，并提供密钥管理服务
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        资源限制：
       </strong>
      </span>
      防止授权用户无限制地使用数据库服务器处理器(CPU)、共享缓存、数据库存储介质等数据库服务器资源，限制每个授权用户/授权管理员的并行会话数等功能
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库安全加固：
       </strong>
      </span>
      对数据库系统进行安全漏洞检查和修补，防止安全漏洞引入数据库系统
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        数据库安全管理：
       </strong>
      </span>
      数据库系统提供
      <strong>
       安全集中管理机制
      </strong>
      ，实现数据库的安全策略集中配置和管理
     </li>
    </ol>

    <h3>
     2.2 数据库加密
    </h3>
    <p>
     <span style="color:#1a439c;">
      <strong>
       数据库加密方式分为两种
      </strong>
     </span>
    </p>
    <ul>
     <li>
      一是数据库
      <span style="color:#1c7892;">
       <strong>
        网上传输的数据
       </strong>
      </span>
      ，通常
      <span style="color:#ad720d;">
       利用SSL协议来实现
      </span>
     </li>
     <li>
      二是数据库
      <span style="color:#1c7892;">
       <strong>
        存储的数据
       </strong>
      </span>
      ，通过
      <span style="color:#ad720d;">
       数据库存储加密来实现
      </span>
     </li>
    </ul>
    <p>
     按照
     <strong>
      加密组件
     </strong>
     与
     <strong>
      数据库管理系统的关系
     </strong>
     ，
     <span style="color:#1c7892;">
      <strong>
       数据库存储加密
      </strong>
     </span>
     <span style="color:#1a439c;">
      <strong>
       可以分成两种加密方式：
      </strong>
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#1c7892;">
       <strong>
        库内加密：
       </strong>
      </span>
      是指在DBMS内部实现支持加密的模块
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        库外加密：
       </strong>
      </span>
      指在DBMS范围之外，由专门的加密部件完成加密/解密操作
     </li>
    </ul>
    <p>
     <span style="color:#1a439c;">
      <strong>
       数据库存储加密的常用技术方法有
      </strong>
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ad720d;">
       <strong>
        基于文件的
       </strong>
      </span>
      <strong>
       数据库加密技术：
      </strong>
      将数据库
      <span style="color:#ad720d;">
       文件作为整体
      </span>
      ，对整个数据库文件进行加密，形成密文来保证数据的机密性
     </li>
     <li>
      <strong>
       <span style="color:#ad720d;">
        基于记录的
       </span>
       数据库加密技术：
      </strong>
      将数据库的
      <span style="color:#ad720d;">
       每一个记录加密成密文
      </span>
      并存放于数据库文件中
     </li>
     <li>
      <strong>
       <span style="color:#ad720d;">
        基于字段的
       </span>
       数据库加密技术：
      </strong>
      加密数据库的字段，以
      <span style="color:#ad720d;">
       不同记录的不同字段为基本加密单元进行加密
      </span>
     </li>
    </ul>
    <h3>
     2.3 数据库防火墙
    </h3>
    <p>
     根据
     <span style="color:#ad720d;">
      预定义的
      <strong>
       禁止和许可策略
      </strong>
      让合法的SQL操作通过，阻断非法违规操作
     </span>
     ，形成数据库的
     <strong>
      外围防御圈
     </strong>
     ，实现SQL危险操作的
     <strong>
      主动预防、实时审计
     </strong>
    </p>
    <p>
     面对来自
     <span style="color:#1c7892;">
      <strong>
       外部的入侵行为
      </strong>
     </span>
     ，数据库防火墙提供
     <strong>
      SQL注入禁止
     </strong>
     和
     <strong>
      数据库虚拟补丁包
     </strong>
     功能。通过虚拟补丁包，数据库系统不用升级、打补丁，即可完成对主要数据库漏洞的防控
    </p>
    <p>
     <span style="color:#1a439c;">
      <strong>
       数据库防火墙的安全作用如下
      </strong>
     </span>
    </p>
    <ol>
     <li>
      <strong>
       屏蔽直接访问数据库的通道：
       <span style="color:#1c7892;">
        数据库防火墙部署
       </span>
      </strong>
      介于
      <span style="color:#ad720d;">
       数据库服务器
      </span>
      和
      <span style="color:#ad720d;">
       应用服务器
      </span>
      之间，屏蔽直接访问的通道，防止数据库隐通道对数据库的攻击
     </li>
     <li>
      <strong>
       增强认证：
      </strong>
      应用程序对数据库的访问，必须经过
      <span style="color:#ad720d;">
       数据库防火墙和数据库自身
       <strong>
        两层身份认证
       </strong>
      </span>
     </li>
     <li>
      <strong>
       攻击检测：
      </strong>
      可
      <span style="color:#1c7892;">
       <strong>
        实时检测
       </strong>
      </span>
      用户对数据库进行的
      <span style="color:#ad720d;">
       SQL注入和缓冲区溢出攻击
      </span>
      ，并
      <span style="color:#ad720d;">
       报警或阻止攻击行为
      </span>
      ，记录攻击操作发生的时间、来源IP、登录数据库的用户名、攻击代码等详细信息
     </li>
     <li>
      <strong>
       防止漏洞利用：

      </span>
      <span style="color:#ad720d;">
       user、db、host、tables_priv和columns_priv
      </span>
      。通过授权表，MySQL提供非常灵活的安全机制
      <ol>
       <li>
        MySQL 具有grant和revoke命令，可以用来创建和删除用户权限，便于分配用户权限
       </li>
       <li>
        MySQL管理员可以用grant和revoke来创建用户、授权和撤权、删除用户
       </li>
      </ol>
     </li>
     <li>
      <span style="color:#1c7892;">
       <strong>
        安全审计：
       </strong>
      </span>
      MySQL
      <strong>
       内置了审计机制
      </strong>
      ，可以记录MySQL的运行状况
     </li>
    </ol>
    <h3>
     5.3 MySQL 安全最佳实践
    </h3>
    <ol>
     <li>
      <strong>
       MySQL安装：
      </strong>
      建立单独启动的用户和组，安装最新的软件包，选择合适的静态参数编译数据库
     </li>
     <li>
      <strong>
       建立MySQL Chrooting运行环境：
      </strong>
      形成“沙箱”保护机制，增强系统抗渗透能力
     </li>
     <li>
      <strong>
       关闭MySQL的远程连接：
      </strong>
      关闭MySQL的
      <span style="color:#ad720d;">
       默认监听端口3306
      </span>
      ，避免增加MySQL的远程攻击风险
     </li>
     <li>
      <strong>
       禁止MySQL导入本地文件：
      </strong>
      禁止MySQL中用“LOAD DATA LOCAL INFILE” 命令，防范攻击者利用此命令把本地文件读到数据库中，阻止用户非法获取敏感信息
     </li>
     <li>
      <strong>
       修改MySQL的root用户ID和密码
      </strong>
     </li>
     <li>
      <strong>
       删除MySQL 的默认用户和db：
      </strong>
      删除MySQL的
      <span style="color:#ad720d;">
       默认数据库test
      </span>
      。除了root用户外，其他的用户都去掉
     </li>
     <li>
      <strong>
       更改MySQL的root用户名，防止口令暴力破解
      </strong>
     </li>
     <li>
      <strong>
       建立应用程序独立使用数据库和用户账号：
      </strong>
      建立应用程序所需要的所有数据库和账号，要求这些账号只能访问应用程序用到的数据库，限制访问 MySQL 数据库和任何系统不能够拥有特权
     </li>
     <li>
      <strong>
       安全监测：
      </strong>
      安全监控MySQL数据库运行，及时修补MySQL数据库漏洞
     </li>
     <li>
      <strong>
       安全备份：
      </strong>
      定期备份MySQL数据库系统及数据
     </li>
    </ol>
    <h3>
     5.4 MySQL 漏洞修补
    </h3>
    <p>
     MySQL的
     <span style="color:#1c7892;">
      <strong>
       安全漏洞主要类型是：
      </strong>
     </span>
     拒绝服务、代码执行、溢出、逃避、特权获取等，因此，要及时安装MySQL漏洞补丁包，防止漏洞被利用
    </p>
    <h2>
     六、国产数据库安全分析与防护
    </h2>
    <h3>
     6.1 国产数据库概况
    </h3>
    <p>
     <span style="color:#1c7892;">
      <strong>
       国产数据库：
      </strong>
     </span>
     是指由
     <strong>
      国家自主研发力量研制
     </strong>
     的数据库系统，具有
     <strong>
      较强的可控性和安全性
     </strong>
     。现已形成
     <span style="color:#ad720d;">
      神舟数据、人大金仓、达梦
     </span>
     等
     <strong>
      传统数据库
     </strong>
     以及
     <span style="color:#ad720d;">
      中科院软件所安捷
     </span>
     <strong>
      实时数据库
     </strong>
    </p>
    <p>
     围绕数据库安全，我国制定了国家标准《信息安全技术数据库管理系统安全技术要求》，该标准规定了
     <strong>
      数据库管理系统的五个安全等级
     </strong>
     及其所需要的安全技术要求
    </p>
    <h3>
     6.2 国产数据库安全分析
    </h3>
    <ol>
     <li>
      <span style="color:#0d0016;">
       <strong>
        国产数据库
       </strong>
      </span>
      <span style="color:#1c7892;">
       <strong>
        安全漏洞：
       </strong>
      </span>
      国产数据库的设计和代码实现可能存在安全漏洞
     </li>
     <li>
      <span style="color:#0d0016;">
       <strong>
        国产数据库
       </strong>
      </span>
      <span style="color:#1c7892;">
       <strong>
        依赖第三方系统组件的安全：
       </strong>
      </span>
      第三方系统组件存在安全漏洞引发的安全风险问题，例如Open SSL协议安全漏洞，对国产数据库网络传输数据有安全影响
     </li>
     <li>
      <span style="color:#0d0016;">
       <strong>
        国产数据库
       </strong>
      </span>
      <span style="color:#1c7892;">
       <strong>
        系统安全配置的安全：
       </strong>
      </span>
      对国产数据库系统的安全敏感配置不当，构成系统安全威胁。
      <span style="color:#1c7892;">
       <strong>
        常见的安全配置不当
       </strong>
      </span>
      包括
      <span style="color:#ad720d;">
       未启用国产数据库安全功能、设置数据库弱口令、开放过多的服务端口、使用非安全远程登录工具
      </span>
      等
     </li>
     <li>
      <span style="color:#0d0016;">
       <strong>
        国产数据库
       </strong>
      </span>
      <span style="color:#1c7892;">
       <strong>
        支持平台的安全：
       </strong>
      </span>
      国产数据库受制于操作系统而产生的安全问题。例如，操作系统的安全问题导致数据库文件被窃取或破坏，甚至失去控制
     </li>
    </ol>
