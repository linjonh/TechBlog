     </h2>

     地理数据库是用于保存数据集集合的“容器”。有以下三种类型：
    </p>
    <ol id="GUID-DC5EF936-3525-44A7-80B3-E26FF78F9978" style="margin:0em 0px 1.2em; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; list-style-position:outside; color:rgb(77,77,77); line-height:17.9200000762939px">
     <li id="GUID-1663438F-448F-44D3-9054-49FD3276ADFD" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      文件地理数据库 - 在文件系统中以文件夹形式存储。每个数据集都以文件形式保存，该文件大小最多可扩展至 1 TB。建议使用文件地理数据库而不是个人地理数据库。
     </li>
     <li id="GUID-B330DCCF-05ED-4D27-8AFB-3CAEDBE19C2F" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      个人地理数据库 - 所有的数据集都存储于 Microsoft Access 数据文件内，该数据文件的大小最大为 2 GB。
     </li>
     <li id="GUID-6091AAC5-8761-4491-AF10-54E2B65B4E70" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      ArcSDE 地理数据库 - 也称作多用户地理数据库。这种类型的数据库使用 Oracle、Microsoft SQL Server、IBM DB2、IBM Informix 或 PostgreSQL 存储于关系数据库中。这些地理数据库需要使用 ArcSDE，并且在大小和用户数量方面没有限制。
     </li>
    </ol>

     <a class="xref" href="http://resources.arcgis.com/zh-cn/help/main/10.1/003n/003n00000023000000.htm" rel="nofollow noopener noreferrer" style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; color:rgb(0,122,194); text-decoration:none" target="_blank">
      了解有关创建地理数据库的信息

    </p>
    <div class="section1" id="GUID-8942E55D-FD88-4171-BAF3-EF44303C4D24" style="margin:0px 0px 0.3em 10px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; color:rgb(77,77,77); line-height:17.9200000762939px">
     <h3 style="margin:0px 0px 0.2em; padding:0px; border:0px; outline:0px; font-size:16px; vertical-align:baseline; line-height:1.2em; font-family:arial,helvetica,sans-serif; color:rgb(0,0,0)">
      文件地理数据库和个人地理数据库
     </h3>
     <p id="GUID-D8A6930F-081C-4954-AB02-F1626F4DF916" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      文件地理数据库和个人地理数据库是专为支持地理数据库的完整信息模型而设计的，它包含拓扑、栅格目录、网络数据集、terrain 数据集、地址定位器等，
      <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       ArcGIS for Desktop Basic
      </span>
      、
      <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       标准版
      </span>
      和
      <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       高级版
      </span>
      的所有用户可免费获取这两种地理数据库。单用户可以对文件地理数据库和个人地理数据进行编辑，这两种地理数据库不支持地理数据库版本管理。使用文件地理数据库，如果要在不同的要素数据集、独立要素类或表中进行编辑，则可以同时存在多个编辑器。
     </p>
     <p id="GUID-0EC1BC8E-B693-4DFA-BC80-D556A83DC8F0" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      文件地理数据库是 ArcGIS 9.2 发布的新地理数据库类型。其旨在执行以下操作：
     </p>
     <ul id="UL_A9EE17BD0D824325B89BF022ED855308" style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; list-style-position:outside">
      <li id="GUID-DD970863-60E8-46A9-9BEF-37C16E7EA35F" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       为所有用户提供可用范围广泛、简单且可扩展的地理数据库解决方案。
      </li>
      <li id="GUID-6B5F4E83-3AEE-4D61-B559-5A1798FED6AC" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       提供能够跨操作系统工作的可移植地理数据库。
      </li>
      <li id="GUID-7A05EB13-11DD-4C13-A3A1-13BBBE006D0F" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       通过扩展可处理非常大的数据集。
      </li>
      <li id="GUID-DF63654E-1C07-42C9-BD63-29B08CCF7291" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       性能和可扩展性极佳。例如，要支持包含超过 3 亿个要素的单个数据集，并支持可扩展为每个文件超过 500 GB（且可获得极佳的性能）的数据集。
      </li>
      <li id="GUID-DB97B794-46F7-4550-A9AC-E976E0EF25B1" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       使用性能和存储能力都得到优化的高效数据结构。文件地理数据库所使用的存储空间约为 shapefile 和个人地理数据库所必需的要素几何存储空间的三分之一。文件地理数据库还允许用户将矢量数据压缩为只读格式，以进一步降低存储要求。
      </li>
      <li id="GUID-C5902D9E-243D-4C2A-8DF6-BC40CCAF64A9" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       在涉及属性的操作方面优于 shapefile，数据大小限制可进行扩展，可使其超出 shapefile 限制。
      </li>
     </ul>
     <p id="GUID-EBBA146C-A6E0-4FF1-95C4-9F29A501531B" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      自从个人地理数据库最初在 ArcGIS 8.0 版本中首次发布以来，ArcGIS 中一直在使用个人地理数据库，该地理数据库使用了 Microsoft Access 数据文件结构（.mdb 文件）。它们支持的地理数据库的大小最大为 2 GB。不过，在数据库性能开始降低之前，有效的数据库大小会较小（介于 250 和 500 MB 之间）。个人地理数据库只能在 Microsoft Windows 操作系统下使用。用户喜欢他们能够通过 Microsoft Access 针对个人地理数据库执行的表操作。许多用户确实喜欢 Microsoft Access 中用于处理属性值的文本处理功能。
     </p>
     <p id="GUID-693CBFAE-FBB7-4E72-9EA4-6ED18D6A5B85" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      出于很多用途，ArcGIS 将继续支持个人地理数据库。不过，多数情况下，
      <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       Esri
      </span>
      推荐使用文件地理数据库，因为文件地理数据库的大小具有可扩展性、性能也会显著提高并可跨平台使用。文件地理数据库非常适合处理用于 GIS 投影的基于文件的数据集，非常适合个人使用以及在小型工作组中使用。它具有很高的性能，在不需要使用 DBMS 的情况下能够进行很好的扩展以存储大量数据。另外，还可跨多个操作系统对其进行移植。
     </p>
     <p id="GUID-06BCE292-1245-4DD8-8C15-8060DB3E46F4" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      通常，用户会针对数据集合使用多个文件或个人地理数据库，并针对他们的 GIS 工作同时访问这些地理数据库。
     </p>
    </div>
    <div class="section1" id="GUID-3566B58A-5BA3-48BF-91C3-5659948BD619" style="margin:0px 0px 0.3em 10px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; color:rgb(77,77,77); line-height:17.9200000762939px">
     <h3 style="margin:0px 0px 0.2em; padding:0px; border:0px; outline:0px; font-size:16px; vertical-align:baseline; line-height:1.2em; font-family:arial,helvetica,sans-serif; color:rgb(0,0,0)">
      ArcSDE 地理数据库
     </h3>
     <p id="GUID-D3C3ECBF-F07B-4480-BF7B-630471F5B922" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      如果需要一种多位用户可同时编辑和使用的大型多用户地理数据库，则 ArcSDE 地理数据库可提供一种极佳的解决方案。新增的功能可用于管理共享式多用户地理数据库和支持多种基于版本的关键性 GIS 工作流。从而使利用贵组织企业关系数据库的能力成为 ArcSDE 地理数据库的一项重要优势。
     </p>
     <p id="GUID-54B8DEF1-1236-4B0A-9A3D-00BE0198A7B1" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      ArcSDE 地理数据库适用于多种 DBMS 存储模型（IBM DB2、Informix、Oracle、PostgreSQL 和 SQL Server）。ArcSDE 地理数据库使用范围广泛，主要适用于个人、工作组、部门和企业设置。它们充分利用 DBMS 的基础架构以支持以下内容：
     </p>
     <ul id="UL_4A2DF6A68B09452FB4D617DBF4D71739" style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; list-style-position:outside">
      <li id="GUID-FA37BD0A-E561-4A8C-8CD1-AEE598796091" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       超大型连续 GIS 数据库
      </li>
      <li id="GUID-783AE5A8-D747-49F9-98FE-90FB1BAC73BE" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       多位同步用户
      </li>
      <li id="GUID-09AC673D-F334-4B7F-80D2-C6E808A37430" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       长事务和版本化工作流
      </li>
      <li id="GUID-376B53D1-3615-4BCF-90A5-F4287CBDDB13" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       对 GIS 数据管理的关系数据库支持（为保证可伸缩性、可靠性、安全性、备份以及完整性等提供建立关系数据库的优势）
      </li>
      <li id="GUID-1F782877-5AE6-42D8-9397-B34AB16F1D26" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       所有支持的 DBMS（Oracle、SQL Server、PostgreSQL、Informix 和 DB2）中的 SQL 空间类型
      </li>
      <li id="GUID-FC5F965F-1480-4B6D-B0ED-76CE5E06ADE4" style="margin:0px 0px 5px 30px; padding:0px 0px 0px 5px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       可适应大量用户不同要求的高性能
      </li>
     </ul>
     <p id="GUID-ECF9DC6F-5949-4B0B-B9D9-A0A0CE1C4460" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      通过许多大型地理数据库的安装启用，我们发现在将 GIS 数据所需的大型二进制对象移入和移出表格时 DBMS 的效率极高。此外，与基于文件的 GIS 数据集相比，GIS 数据库的容量更大且支持的用户数量也更多。
     </p>
     <p id="GUID-1048D58E-7538-4BA0-8D22-36BAF512D32B" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      有关 ArcSDE 地理数据库架构以及 ArcSDE 地理数据库如何利用关系数据库技术的信息，请参见
      <a class="xref" href="http://resources.arcgis.com/zh-cn/help/main/10.1/003n/003n000000r4000000.htm" rel="nofollow noopener noreferrer" style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; color:rgb(0,122,194); text-decoration:none" target="_blank">
       地理数据库的架构
      </a>
      。
     </p>
     <div class="section2" id="GUID-C19D1FFD-B4C8-4AAC-9E6D-A324913792CA" style="margin:0px 0px 0.3em 15px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
      <h4 style="margin:0px 0px 0.2em; padding:0px; border:0px; outline:0px; font-size:14px; vertical-align:baseline; line-height:1.2em; font-family:arial,helvetica,sans-serif; color:rgb(0,0,0)">
       ArcSDE 可对 DBMS 事务框架进行长事务管理和短事务管理
      </h4>
      <p id="GUID-C3D9F544-4EA7-4F91-B6ED-369E5218F255" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       ArcSDE 的主要角色之一就是支持每个 DBMS 中的地理数据库版本管理框架。
      </p>
      <p id="GUID-202EB195-FACE-4BD2-A6AA-FC718ED7AF40" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       绝大多数情况下，GIS 中的单个编辑事务可能涉及对多个表中的多个行进行更改。例如，更新宗地可能需要更改面的表示，并更改相应的边界线和宗地拐角。此外，还必须更新这些要素中每个要素的属性记录。此编辑操作需要对多个表中的多条记录进行更改。在这些情况下，用户希望将此编辑集合视为单个事务。提交或回滚这些更改时，会将它们视为一个统一的操作来进行管理。
      </p>
      <p id="GUID-829F3B51-3D8B-4DA5-BB89-4E2FC26B88E2" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       同时，用户希望能够在一个编辑会话中撤消和重做单个编辑操作。为了使这种情况变得更为复杂，可能需要在与中央共享数据库断开连接的系统中执行编辑操作。
      </p>
      <p id="GUID-AF0C1027-97D2-4BAB-BF5C-75FB8C7422C8" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       而且，在这些专门化的 GIS 数据维护过程中，GIS 数据库必须持续保持对日常操作可用，而在这些日常操作中，每位用户都有可能获取共享 GIS 数据库的个人视图或状态。
      </p>
      <p id="GUID-87952299-CB88-4608-8D19-3D715FA7E786" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       通过使用一种称为
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        版本管理
       </span>
       的方法，ArcSDE 地理数据库支持在多用户环境下对这些数据管理情景及许多其他数据管理情景进行管理和更新。在版本管理这种机制下，所有的数据库更改都作为表中的行进行记录。例如，每次更新某一行中的某个值时，旧值即会失效，并会新增一个更新行。
      </p>
      <p id="GUID-03624C6A-D91B-4039-948E-4124B142F591" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       这样，通过将更改信息以增量记录的方式存储在数据库中，ArcSDE 技术就能在简单 DBMS 事务框架中管理复杂的高级 GIS 事务。
      </p>
      <div class="notes" id="GUID-E584027D-4224-4BC9-9EC4-3B5CC2EB5CA9" style="margin:10px; padding:0px; border-width:1px 0px; border-top-style:solid; border-bottom-style:solid; border-top-color:rgb(204,204,204); border-bottom-color:rgb(204,204,204); outline:0px; font-size:12.8000001907349px; vertical-align:baseline; width:1003.67498779297px; clear:both; display:inline-block">
       <div class="note" style="margin:0px 0px 0.3em; padding:5px 5px 20px 0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; float:left; font-weight:bold; height:16px; display:inline">
        <img alt="注" class="note_img" src="https://i-blog.csdnimg.cn/blog_migrate/292228b43142208c1bebe7f4b30379d0.png" style="margin:0px 5px 0px 0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; float:left; position:relative; top:2px" title="注">

        <p id="GUID-BBCE4E84-AFB8-4EF8-A575-046BE334FA21" style="margin:0px 0.5em; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
         此机制用于维护所有更改的记录及其元数据，其是术语
         <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
          版本管理
         </span>
         的起源。
        </p>

       ArcSDE 使用版本的元数据来隔离多个编辑会话、支持复杂事务、共享复本、同步多个数据库之间的内容、执行自动存档并支持历史查询。
      </p>
      <p id="GUID-81A08B57-BF0B-4369-BB39-D2E0EC5BC1EC" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       有关详细信息，请参阅
       <a class="xref" href="http://resources.arcgis.com/zh-cn/help/main/10.1/003n/003n000000wq000000.htm" rel="nofollow noopener noreferrer" style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline; color:rgb(0,122,194); text-decoration:none" target="_blank">
        编辑和维护数据概述

       。

      <h4 style="margin:0px 0px 0.2em; padding:0px; border:0px; outline:0px; font-size:14px; vertical-align:baseline; line-height:1.2em; font-family:arial,helvetica,sans-serif; color:rgb(0,0,0)">
       ArcGIS 中如何包含 ArcSDE 技术？
      </h4>
      <p id="GUID-B121FF8D-0826-4D74-8029-F67E1D2ACB02" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       过去，ArcSDE 作为单独的 Esri 产品进行销售。从 ArcGIS 9.2 开始，ArcSDE 技术成为 ArcGIS 内含的功能，不再单独提供。ArcSDE 技术已集成到了
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Desktop
       </span>
       、ArcGIS Engine 和
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Server
       </span>
       中，以便用户能够在其组织内扩展基于 DBMS 的地理数据库。
      </p>
      <p id="GUID-872E1298-A1F1-4DC0-976B-73431DC93296" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       从 ArcGIS 9.2 开始，
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Desktop Standard
       </span>
       和
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        高级版
       </span>
       包含 Microsoft SQL Server Express 的安装程序。这些软件产品还包括支持 SQL Server Express 中 ArcSDE 地理数据库的 ArcSDE 功能，最多可同时支持三个桌面用户。
      </p>
      <p id="GUID-338C09F2-9C72-48E4-A999-5412946CF484" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       从 ArcGIS 9.3 开始，ArcGIS Engine 包含 SQL Server Express 的安装程序。对于用于编辑地理数据库的 ArcGIS Engine 应用程序，ArcGIS Engine 的可选 Geodatabase Update 扩展模块是必需的模块。
      </p>
      <p id="GUID-E8FF4332-002D-401C-95C6-73C7081E8153" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       Microsoft 对 SQL Server Express 的使用做出了限制，要求只能使用一个 CPU（或套接字中的内核）和大小为 1GB 的 RAM。SQL Server 2005 Express 或 SQL Server 2008 Express 数据库最大只能为 4GB。SQL Server 2008 Express R2 数据库最大为 10GB。（有关具体实施时的实例大小限制的特定信息，请参阅 Esri 许可协议。）
      </p>
      <p id="GUID-F7FE5139-7BFE-449A-8350-2EEA7E5BC9A5" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       在
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Desktop Standard
       </span>
       和
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        高级版
       </span>
       中，可通过 ArcCatalog 应用程序使用 SQL Server Express 全面管理 ArcSDE 地理数据库。这样，一次就可为多个用户提供全面的 ArcSDE 地理数据库功能。可在 ArcCatalog 中或目录窗口中设置和管理这些 ArcSDE 地理数据库。不需要具有额外的软件或数据库管理方面的专业知识。
      </p>
      <p id="GUID-16C69EA7-D31A-4A8F-817D-723D72DEC00B" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       ArcGIS for Server Workgroup 还包括 ArcSDE 对 SQL Server Express 的支持。使用此级别的 ArcSDE，通过使用 SQL Server Express，最多可同时支持 10 个 Windows Desktop 用户和编辑人员（
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Desktop Basic
       </span>
       、
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        标准版
       </span>
       、
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        高级版
       </span>
       、自定义 ArcGIS Engine 应用程序、AutoCAD、MicroStation 等的用户），并可支持任意数量的来自 Web 应用程序的额外服务器连接。（有关实现时所需的连接数量的特定信息，请参见许可协议。）
      </p>
      <p id="GUID-9AB24A95-0AC5-4529-B758-3C0C0E2E75ED" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       对于 ArcGIS for Server Workgroup，您可使用
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Desktop Standard
       </span>
       或
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        高级版
       </span>
       在 ArcCatalog 中或目录窗口中创建和管理 SQL Server Express 的 ArcSDE 地理数据库。不需要具有额外的数据库管理方面的专业知识。
      </p>
      <p id="GUID-741ED5E4-77D7-4866-A158-F325F9AD26F2" style="margin-top:0.5em; margin-bottom:0.5em; margin-left:10px; padding-top:0px; padding-bottom:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
        ArcGIS for Server
       </span>
       中包含完整的企业级 ArcSDE 技术，未做任何限制。您仍可针对 Oracle、SQL Server、PostgreSQL、IBM DB2 和 IBM Informix 运行传统的 ArcSDE 技术。企业服务器级别的 ArcSDE 支持可以扩展为任意大小的数据库以及任意数量的用户，并可在任意大小和配置的计算机上运行。使用
       <span style="margin:0px; padding:0px; border:0px; outline:0px; font-size:12.8000001907349px; vertical-align:baseline">
