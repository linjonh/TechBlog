    <h2>
     一、指定php版运行，
    </h2>
    <pre><code>cd /www/wwwroot/www.xxx.com/ &amp;&amp; /www/server/php/56/bin/php think timedtasks start &gt;&gt; /tmp/timedtasks.log 2&gt;&amp;1</code></pre>
    <h2>
     二、不指定php版
    </h2>
    <pre><code>cd /www/wwwroot/www.xxx.com/ &amp;&amp; php think timedtasks start &gt;&gt; /tmp/timedtasks.log 2&gt;&amp;1</code></pre>
    <p>
     或
    </p>
    <pre><code># 使用绝对路径（通过 which php 获取）
cd /www/wwwroot/www.xxx.com/ &amp;&amp; /usr/bin/php think timedtasks start &gt;&gt; /tmp/timedtasks.log 2&gt;&amp;1</code></pre>
    <p>
    </p>
    <p>
    </p>
    <h3>
     切换工作目录
     <br/>
     cd /www/wwwroot/www.xxx.com/
     <br/>
     确保 PHP 脚本在项目根目录下执行，避免因相对路径导致的文件找不到问题（常见于框架的入口文件、配置文件加载）。
    </h3>
    <p>
     指定 PHP 版本路径
     <br/>
     /www/server/php/56/bin/php
     <br/>
     直接使用绝对路径调用 PHP 5.6 的解释器，避免系统默认 PHP 版本冲突。
    </p>
    <p>
     执行命令
     <br/>
     think timedtasks start
     <br/>
     假设这是你的项目自定义命令（如 ThinkPHP 框架的定时任务）。
    </p>
    <p>
     日志记录（可选但强烈建议）
     <br/>
     &gt;&gt; /tmp/timedtasks.log 2&gt;&amp;1
     <br/>
     将标准输出和错误信息重定向到日志文件，便于后续排查问题
    </p>
    <p>
    </p>
    <p>
    </p>
    <h2>
     三、常见问题排查
     <br/>
     <br/>
     3.1 权限问题
     <br/>
     确保宝塔的 www 用户（或其他运行用户）有权限访问项目目录和 PHP 可执行文件：
     <br/>
     chown -R www:www /www/wwwroot/www.xxx.com/
     <br/>
     chmod +x /www/server/php/56/bin/php
     <br/>
     3.2 路径错误
     <br/>
     手动在 SSH 中执行命令测试：
     <br/>
     /www/server/php/56/bin/php -v  # 确认 PHP 版本
     <br/>
     cd /www/wwwroot/www.xxx.com/ &amp;&amp; /www/server/php/56/bin/php think timedtasks start
     <br/>
     3.3 依赖缺失
     <br/>
     如果脚本依赖环境变量（如数据库配置），可在命令前通过 export 声明：
     <br/>
     export ENV=production &amp;&amp; cd /path &amp;&amp; php command
    </h2>
