     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/28c62dae68f773dac74d1bd865974a24.png">
      第一个是显示没有找到"D:\Python27"，但是这个是个文件夹，是不是npm自己知道要加载python，我们只给路径就行，它自己找对应的目录呢，但是事实我们配置的环境变量，第一处是系统环境变量：配置了python的环境变量为D:\Python27\python.exe这个东西，但是貌似这块好像没有用到。第二处是npm的python配置，我们配置了npm config set python "D:\Python27"这个路径，感觉应该是这个，改一下加上文件名：
     </img>
    </p>
    <pre><code class="prism language-bash"><span class="token function">npm</span> config <span class="token builtin class-name">set</span> python <span class="token string">"D:\Python27\python.exe"</span>
</code></pre>
    <pre><code class="prism language-bash"><span class="token function">npm</span> rebuild node-sass
</code></pre>
    <p>
     我们重构一下上边这个东西，然后在执行一下install这个命令

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4210d18f52873125a714d09e6adb92bc.png">
      <br/>
      成功了啊，所以问题出在python的环境变量配置，当然是npm的，而不是nodeJs的版本的问题。
     </img>
