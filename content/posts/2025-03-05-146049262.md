    <h1 class="title-article" id="articleContentId">
     对象头（Object Header）中的 Mark Word 在32位和64位系统中有显著区别
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

   <div class="htmledit_views" id="content_views">
    <p>
     对象头（Object Header）中的
     <strong>
      Mark Word
     </strong>
     在32位和64位系统中有显著区别，尤其是在开启或未开启
     <strong>
      指针压缩
     </strong>
     （Compressed Oops）时。以下是详细说明：
    </p>
    <hr/>
    <h4>
     <strong>
      1. 32位系统与64位系统的Mark Word区别
     </strong>
    </h4>
    <h5>
     <strong>
      32位系统
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Mark Word 大小
       </strong>
       ：32位（4字节）
      </p>
     </li>
     <li>
      <p>
       <strong>
        结构
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         存储对象的哈希码、锁状态、GC分代年龄等信息。
        </p>
       </li>
       <li>
        <p>
         锁状态（如偏向锁、轻量级锁、重量级锁）会占用部分位。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      64位系统
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Mark Word 大小
       </strong>
       ：64位（8字节）
      </p>
     </li>
     <li>
      <p>
       <strong>
        结构
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         与32位类似，但可存储更多的信息。
        </p>
       </li>
       <li>
        <p>
         由于指针长度增加（64位），对象头的大小会显著增大，导致内存开销增加。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      2. 开启指针压缩（Compressed Oops）与未开启的区别
     </strong>
    </h4>
    <p>
     指针压缩是JVM在64位系统下的一种优化技术，目的是减少内存占用。
    </p>
    <h5>
     <strong>
      未开启指针压缩
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        指针大小
       </strong>
       ：64位（8字节）
      </p>
     </li>
     <li>
      <p>
       <strong>
        对象头大小
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         Mark Word：8字节
        </p>
       </li>
       <li>
        <p>
         类型指针（Klass Pointer）：8字节
        </p>
       </li>
       <li>
        <p>
         总计：16字节（不包括对齐填充）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         内存占用较大，尤其是小对象（如只有几个字段的对象）会显得浪费。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <strong>
      开启指针压缩
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        指针大小
       </strong>
       ：32位（4字节）
      </p>
     </li>
     <li>
      <p>
       <strong>
        对象头大小
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         Mark Word：8字节
        </p>
       </li>
       <li>
        <p>
         类型指针（Klass Pointer）：4字节
        </p>
       </li>
       <li>
        <p>
         总计：12字节（不包括对齐填充）
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         类型指针被压缩为32位，减少了内存占用。
        </p>
       </li>
       <li>
        <p>
         压缩后的指针可以表示最大35位的地址空间（32GB堆内存），通过左移3位（乘以8）还原为64位地址。
        </p>
       </li>
       <li>
        <p>
         显著减少小对象的内存开销。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      3. 指针压缩的适用条件
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        堆内存限制
       </strong>
       ：通常堆内存小于32GB时，指针压缩才能生效。
      </p>
     </li>
     <li>
      <p>
       <strong>
        对齐要求
       </strong>
       ：JVM要求对象在内存中按8字节对齐，因此即使开启指针压缩，对象头大小仍可能对齐到16字节。
      </p>
     </li>
    </ul>
    <hr/>

     <strong>
      4. 总结
     </strong>

      </p>
     </li>
     <li>
      <p>
       <strong>
        64位系统（未压缩）
       </strong>
       ：对象头较大，适合大内存应用。
      </p>
     </li>
     <li>
      <p>
       <strong>
        64位系统（压缩）
       </strong>
       ：通过压缩类型指针，减少内存占用，适合堆内存小于32GB的场景。
      </p>
     </li>
    </ul>
    <p>
     在实际开发中，建议在64位系统上开启指针压缩（默认开启），以减少内存开销并提升性能。
    </p>
