#### 1.match\_all

```apl
GET /book/_search
{
    "query": {
        "match_all": {}
    }
}

```

#### 2.match

```apl
GET /book/_search
{
	"query": {
		"match": {
			"description": "java程序员"
		}
	}
}

```

#### 3.multi\_match

```apl
GET /book/_search
{
  "query": {
    "multi_match": {
      "query": "java程序员",
      "fields": ["name", "description"]
    }
  }
}

```

#### 4.range query 范围查询

```apl
GET /book/_search
{
  "query": {
    "range": {
      "price": {
        "gte": 80,
		"lte": 90
      }
    }
  }
}

```

#### 5.term query

字段为 keyword 时，存储和搜索都不分词

```apl
GET /book/_search
{
  "query": {
    "term": {
      "description": "java程序员"
    }
  }
}

```

#### 6.terms query

```apl
GET /book/_search
{
    "query": { "terms": { "tag": [ "search", "full_text", "nosql" ] }}
}


```

#### 7.exist query

查询有某些字段值的文档

```apl
GET /_search
{
    "query": {
        "exists": {
            "field": "join_date"
        }
    }
}

```

#### 8.Fuzzy query

返回包含与搜索词类似的词的文档，该词由 Levenshtein 编辑距离度量。

包括以下几种情况：

* 更改角色（box→fox）
* 删除字符（aple→apple）
* 插入字符（sick→sic）
* 调换两个相邻字符（ACT→CAT）

```apl
GET /book/_search
{
    "query": {
        "fuzzy": {
            "description": {
                "value": "jave"
            }
        }
    }
}

```

#### 9.IDs

```apl
GET /book/_search
{
    "query": {
        "ids" : {
            "values" : ["1", "4", "100"]
        }
    }
}

```

#### 10.prefix 前缀查询

```apl
GET /book/_search
{
    "query": {
        "prefix": {
            "description": {
                "value": "spring"
            }
        }
    }
}

```

#### 11.regexp query

正则查询

```apl
GET /book/_search
{
    "query": {
        "regexp": {
            "description": {
                "value": "j.*a",
                "flags" : "ALL",
                "max_determinized_states": 10000,
                "rewrite": "constant_score"
            }
        }
    }
}

```
