    <h2 id="%E4%B8%80%E3%80%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E7%9A%84%E5%8E%9F%E7%90%86" name="%E4%B8%80%E3%80%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E7%9A%84%E5%8E%9F%E7%90%86">
     一、共享内存的原理
    </h2>
    <p>
     共享内存是通过在物理内存上开辟一块空间，然后让需要通信的进程都映射到这一块空间，这样就使它们看到同一块资源了。
    </p>
    <p>
     <img alt="" height="709" src="https://i-blog.csdnimg.cn/direct/e50dc6181d7440f8af28783f505f208d.png" width="1510"/>
    </p>
    <p>
     共享内存通信是
     <strong>
      <span style="color:#ff9900">
       双向
      </span>
     </strong>
     的，也就是说一个进程可以既读又写，使用起来就和C语言的malloc申请到的内存差不多。这种通信方式存在着数据安全问题，会在下文细说。
    </p>
    <h2 id="%E4%BA%8C%E3%80%81%E4%BF%A1%E9%81%93%E7%9A%84%E5%BB%BA%E7%AB%8B" name="%E4%BA%8C%E3%80%81%E4%BF%A1%E9%81%93%E7%9A%84%E5%BB%BA%E7%AB%8B">
     二、信道的建立
    </h2>
    <h3 id="1.%E5%88%9B%E5%BB%BA%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98" name="1.%E5%88%9B%E5%BB%BA%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98">
     1.创建共享内存
    </h3>
    <p>
     创建共享内存使用
     <strong>
      shmget
     </strong>
     函数，它的作用是创建或获取共享内存段的系统调用。
    </p>
    <p>
     对于shmget的使用来说，虽然操作起来相对简单，但要完全理解其各种参数的设定则较为困难。不过接下来我会进行详细讲解。
    </p>
    <blockquote>
     <p>
      shmget声明如下：
     </p>
     <pre><code class="language-cpp">#include &lt;sys/ipc.h&gt;
#include &lt;sys/shm.h&gt;

int shmget(key_t key, size_t size, int shmflg);
</code></pre>
     <ul>
      <li>
       参数key：用户设定任意一个数，用于区分不同共享内存，通常由
       <strong>
        ftok
       </strong>
       生成。
      </li>
      <li>
       参数size：设定共享内存的大小。
      </li>
      <li>
       参数shmflg：标志位，用于指定共享内存段的创建方式和权限。常见的标志包括：
       <ul>
        <li>
         <code>
          <span style="color:#fe2c24">
           IPC_CREAT
          </span>
          ：
         </code>
         如果共享内存段不存在，则创建它。
        </li>
        <li>
         <code>
          <span style="color:#fe2c24">
           IPC_EXCL
          </span>
          ：
         </code>
         与
         <code>
          IPC_CREAT
         </code>
         一起使用，
         <strong>
          确保创建的共享内存段是新的
         </strong>
         。
        </li>
        <li>
         权限标志：如
         <code>
          0666
         </code>
         ，表示所有用户都有读写权限。
        </li>
       </ul>
      </li>
      <li>
       返回值：
       <ul>
        <li>
         成功时返回共享内存段的标识符（
         <span style="color:#fe2c24">
          <strong>
           shmid
          </strong>
         </span>
         ）。
        </li>
        <li>
         失败时返回
         <code>
          -1
         </code>
         ，并设置
         <code>
          errno
         </code>
         以指示错误类型。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h4 id="1.key%E7%9A%84%E4%BD%9C%E7%94%A8" name="1.key%E7%9A%84%E4%BD%9C%E7%94%A8">
     <strong>
      1.key的作用
     </strong>
    </h4>
    <ul>
     <li>
      思考1：在用户层面如何让两个独立进程共享同一块内存？
     </li>
     <li>
      思考2：在匿名管道和命名管道中，用户层面是如何让两个进程确定同一个资源的？
     </li>
    </ul>
    <p>
     问题2很显然，管道的本质是文件，用户通过让两个程序打开同一个文件名来实现看到同一个资源。
    </p>
    <p>
     因此，共享内存同样需要一个key来充当类似文件名的功能。
    </p>
    <h4 id="2.key%E7%9A%84%E9%80%89%E5%8F%96" name="2.key%E7%9A%84%E9%80%89%E5%8F%96">
     <strong>
      2.key的选取
     </strong>
    </h4>
    <p>
     key参数本质是一个int类型，我们可以直接指定一个数值传入，当然，为了更规范，更专业，我们通常都会使用ftok来生成。
    </p>
    <blockquote>
     <p>
      ftok的声明如下：
     </p>
     <pre><code class="language-cpp">#include &lt;sys/types.h&gt;
#include &lt;sys/ipc.h&gt;

key_t ftok(const char *pathname, int proj_id);
</code></pre>
     <ul>
      <li>
       参数pathnme：一个存在的文件路径（例如
       <code>
        /tmp/myfile
       </code>
       ），文件必须存在，否则
       <code>
        ftok
       </code>
       会失败。
      </li>
      <li>
       参数proj_id：一个整数，用于进一步区分不同的 IPC 对象。
      </li>
      <li>
       返回值：
       <ul>
        <li>
         <p>
          <strong>
           成功
          </strong>
          ：返回生成的
          <code>
           key_t
          </code>
          键值。
         </p>
        </li>
        <li>
         <p>
          <strong>
           失败
          </strong>
          ：返回
          <code>
           -1
          </code>
          ，并设置
          <code>
           errno
          </code>
          以指示错误原因。
         </p>
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h4 id="3.shmid%E7%9A%84%E4%BD%9C%E7%94%A8" name="3.shmid%E7%9A%84%E4%BD%9C%E7%94%A8">
     <strong>
      3.shmid的作用
     </strong>
    </h4>
    <p>
     <strong>
      shmid
     </strong>
     是一个int类型，由shmget返回，在作用上和物理意义上与文件系统中的fd类似。它的作用主要是让用户找到指定的共享内存。而它的物理意义在这里就不提了，感兴趣的话可以等我的下一篇博文（IPC系统）发布。
    </p>
    <h4 id="4.key%E5%92%8Cshmid%E7%9A%84%E5%8C%BA%E5%88%AB" name="4.key%E5%92%8Cshmid%E7%9A%84%E5%8C%BA%E5%88%AB">
     <strong>
      4.key和shmid的区别
     </strong>
    </h4>
    <p>
     <strong>
     </strong>
     key最终成为
     <span style="color:#ff9900">
      系统层
     </span>
     区分不同IPC的标志，而shmid则是
     <span style="color:#ff9900">
      用户层
     </span>
     用来区分不同IPC的标志。
    </p>
    <h4 id="5.%E5%86%85%E5%AD%98%E8%AE%BE%E5%AE%9A%E7%9A%84%E7%89%B9%E6%80%A7" name="5.%E5%86%85%E5%AD%98%E8%AE%BE%E5%AE%9A%E7%9A%84%E7%89%B9%E6%80%A7">
     <strong>
      5.内存设定的特性
     </strong>
    </h4>
    <p>
     这里的内存设定指的是shmget函数中的参数size。
    </p>
    <p>
     <u>
      当传入的内存不足4096字节（4KB）的倍数时，会扩到4096倍数。但是只会提供size大小的使用空间。这样做可以规避掉一些因为共享内存过多带来的问题。
     </u>
    </p>
    <h4 id="6.shmflg%E7%9A%84%E8%AE%BE%E5%AE%9A" name="6.shmflg%E7%9A%84%E8%AE%BE%E5%AE%9A">
     6.shmflg的设定
    </h4>
    <p>
     对于共享内存，我们可以将程序简单分为创建端和使用端，它们的shmflg设定通常是：
    </p>
    <ul>
     <li>
      <strong>
       创建端
      </strong>
      ：IPC_CREAT | IPC_EXCL | 0666
     </li>
     <li>
      <strong>
       使用端
      </strong>
      ：IPC_CREAT
     </li>
    </ul>
    <p>
     创建端要保证IPC是最新的，所以需要加IPC_EXCL，然后还需要设定权限。
    </p>
    <p>
     使用端只需要获取共享内存段的系统调用，因此只用一个IPC_CREAT即可。
    </p>
    <h3 id="2.%E7%BB%91%E5%AE%9A%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98" name="2.%E7%BB%91%E5%AE%9A%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98">
     2.绑定共享内存
    </h3>
    <p>
     以上我们完成的只是共享内存的创建，接下来还需要把进程绑定到共享内存，使用函数
     <strong>
      shmat
     </strong>
     ，其中at指的是单词attach。
    </p>
    <blockquote>
     <p>
      shmat的声明：
     </p>
     <pre><code class="language-cpp">#include &lt;sys/types.h&gt;
#include &lt;sys/shm.h&gt;

void *shmat(int shmid, const void *shmaddr, int shmflg);
</code></pre>
     <ul>
      <li>
       参数shmid：传入从shmget中返回的shmid来指定共享内存。
      </li>
      <li>
       参数shmaddr：指定共享内存段附加到进程地址空间的位置，通常设为nullptr，系统会自动选择一个合适的地址。
      </li>
      <li>
       参数shmflg：读写方式，常用的有：
       <ul>
        <li>
         <code>
          SHM_RDONLY
         </code>
         ：以只读方式附加共享内存段。
        </li>
        <li>
         <code>
          0
         </code>
         ：以读写方式附加共享内存段。
        </li>
       </ul>
      </li>
      <li>
       返回值：
       <ul>
        <li>
         成功时，返回共享内存段附加到进程地址空间的起始地址。
        </li>
        <li>
         失败时，返回
         <code>
          (void *) -1
         </code>
         ，并设置
         <code>
          errno
         </code>
         。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h3 id="3.%E4%BB%A3%E7%A0%81%E7%A4%BA%E4%BE%8B" name="3.%E4%BB%A3%E7%A0%81%E7%A4%BA%E4%BE%8B">
     3.代码示例
    </h3>
    <p>
     创建端程序：
    </p>
    <pre><code class="language-cpp">int main()
{
    //生成一个key
    int key = ftok(".", 48);
    //创建共享内存
    int shmid = shmget(key, 4069, IPC_CREAT | IPC_EXCL | 0666);
    //连接到共享内存
    void* p = shmat(shmid,nullptr,0);
    //使用共享内存
    //... ...
    return 0;
}</code></pre>
    <p>
     使用端程序：
    </p>
    <pre><code class="language-cpp">int main()
{
    //生成一个相同key
    int key = ftok(".", 48);
    //获取到共享内存的系统调用
    int shmid = shmget(key, 4069, IPC_CREAT);
    //连接到共享内存
    void* p = shmat(shmid,nullptr,0);
    //使用共享内存
    //... ...
    return 0;
}</code></pre>
    <p>
     <span style="color:#fe2c24">
      <strong>
       注意
      </strong>
     </span>
     ：为了简洁和方便说明问题，以上代码省略了头文件的包含和返回值有效性的判断等等，在实际开发中可不敢省略。
    </p>
    <h2 id="%E4%B8%89%E3%80%81%E5%88%A9%E7%94%A8%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E9%80%9A%E4%BF%A1" name="%E4%B8%89%E3%80%81%E5%88%A9%E7%94%A8%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E9%80%9A%E4%BF%A1">
     三、利用共享内存通信
    </h2>
    <h3 id="1.%E9%80%9A%E4%BF%A1" name="1.%E9%80%9A%E4%BF%A1">
     1.通信
    </h3>
    <p>
     上文我们只是完成了信道的建立，接下来我们进行通信，通过上面的操作，我们已经获取到共享内存的起始地址。
    </p>
    <p>
     <u>
      它的用法与C语言的malloc申请的内存用法相同，只是共享内存可以同时被两个进程访问。
     </u>
    </p>
    <p>
     如下写端：
    </p>
    <pre><code class="language-cpp">int main()
{
    int key = ftok(".", 48);
    int shmid = shmget(key, 4069, IPC_CREAT | IPC_EXCL | 0666);
    void* p = shmat(shmid,nullptr,0);
    //使用共享内存
    char* chp = (char*)p;
    for(int i='a';i&lt;='z';i++)
    {
        sleep(1);
        *chp=i;
        chp++;
    }
    return 0;
}</code></pre>
    <p>
     读端：
    </p>
    <pre><code class="language-cpp">int main()
{
    int key = ftok(".", 48);
    int shmid = shmget(key, 4069, IPC_CREAT);
    void* p = shmat(shmid,nullptr,0);
    //使用共享内存
    char* chp = (char*)p;
    while(true)
    {
        sleep(1);
        cout&lt;&lt;chp&lt;&lt;endl;
    }
    return 0;
}</code></pre>
    <p>
     <strong>
      <span style="color:#fe2c24">
       注意
      </span>
     </strong>
     ：为了获取到同一个共享内存，我们设定的key必须一致。
    </p>
    <h3 id="2.%E8%A7%A3%E9%99%A4%E7%BB%91%E5%AE%9A" name="2.%E8%A7%A3%E9%99%A4%E7%BB%91%E5%AE%9A">
     2.解除绑定
    </h3>
    <p>
     如果进程退出时没有解除绑定，共享内存段仍然会保留在系统的共享内存资源中，直到显式删除（通过
     <code>
      shmctl
     </code>
     或系统重启）。
    </p>
    <p>
     使用
     <strong>
      shmdt
     </strong>
     来解除绑定，其中dt代表单词delete。
    </p>
    <blockquote>
     <p>
      shmdt的声明：
     </p>
     <pre><code class="language-cpp">int shmdt(const void *shmaddr);</code></pre>
     <p>
      参数shmaddr：需要断开连接的共享内存的起始地址。
     </p>
     <p>
      返回值：
     </p>
     <ul>
      <li>
       成功：返回0。
      </li>
      <li>
       失败：返回-1，并设置errno以指示错误原因。
      </li>
     </ul>
    </blockquote>
    <p>
     一个共享内存，与它绑定的程序的个数是由一个
     <span style="color:#fe2c24">
      引用计数
     </span>
     机制进行维护的，当shmdt成功，引用计数减1。
    </p>
    <h3 id="3.%E9%94%80%E6%AF%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98" name="3.%E9%94%80%E6%AF%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98">
     3.销毁共享内存
    </h3>
    <p>
     <strong>
      <span style="color:#fe2c24">
       共享内存不会随程序的结束而销毁，它是随内核的
      </span>
     </strong>
     ，因此需要显式地进行销毁，可以使用shmctl函数。或在命令行中使用指令进行销毁。
    </p>
    <h4 id="1.%E5%91%BD%E4%BB%A4%E8%A1%8C%E9%94%80%E6%AF%81" name="1.%E5%91%BD%E4%BB%A4%E8%A1%8C%E9%94%80%E6%AF%81">
     1.命令行销毁
    </h4>
    <p>
     1.1.查看共享内存信息
    </p>
    <pre><code class="language-cpp">ipcs -m
</code></pre>
    <p>
     如下：
    </p>
    <p>
     <img alt="" height="208" src="https://i-blog.csdnimg.cn/direct/7900256b1bd04151bf64e070f170c6d8.png" width="1025"/>
    </p>
    <p>
     这里解释一下nattch信息：它表示与该共享内存连接的程序个数。
    </p>
    <p>
     1.2.销毁共享内存
    </p>
    <pre><code class="language-cpp">ipcrm -m 2</code></pre>
    <p>
     这里需要填入shmid（即这里的2）来指定共享内存。
    </p>
    <h4 id="2.%E7%A8%8B%E5%BA%8F%E4%B8%AD%E9%94%80%E6%AF%81" name="2.%E7%A8%8B%E5%BA%8F%E4%B8%AD%E9%94%80%E6%AF%81">
     2.程序中销毁
    </h4>
    <p>
     在程序中销毁我们使用函数
     <strong>
      shmctl
     </strong>
     ，其中ctl代表单词control。
    </p>
    <blockquote>
     <p>
      shmctl声明如下：
     </p>
     <pre><code class="language-cpp">#include &lt;sys/ipc.h&gt;
#include &lt;sys/shm.h&gt;

int shmctl(int shmid, int cmd, struct shmid_ds *buf);
</code></pre>
     <ul>
      <li>
       参数shmid：传入从shmget返回的shmid来指定需要销毁的共享内存。
      </li>
      <li>
       参数cmd：需要传入一个操作选项，操作选项很多，而IPC_RMID就是用来销毁共享内存的。
      </li>
      <li>
       参数shmid_ds：这是一个输出型参数，如果你需要获取共享内存的信息，则传入一个shmid_ds类型的指针来接收，如果不是通常传入nullptr即可。
      </li>
      <li>
       返回值：
       <ul>
        <li>
         成功时返回
         <code>
          0
         </code>
         。
        </li>
        <li>
         失败时返回
         <code>
          -1
         </code>
         ，并设置
         <code>
          errno
         </code>
         以指示错误类型。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <p>
     注：命令行销毁和程序中销毁效果是一样的，因为命令行销毁底层还是调用了shmctl函数。
    </p>
    <h2 id="%E5%9B%9B%E3%80%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E7%9A%84%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F" name="%E5%9B%9B%E3%80%81%E5%85%B1%E4%BA%AB%E5%86%85%E5%AD%98%E7%9A%84%E7%94%9F%E5%91%BD%E5%91%A8%E6%9C%9F">
     四、共享内存的生命周期
    </h2>
    <p>
     共享内存的生命周期是不随进程的，而是随内核，如果没有显示删除它就会一直存在，尽管相关的进程已经退出。直到重装系统才得以释放。
    </p>
    <p>
     使用shmctl释放共享内存存在的情况
    </p>
    <p>
     <strong>
      1.正常释放
     </strong>
    </p>
    <p>
     <strong>
      当nattach（引用计数）为0时，即没有进程与它绑定，被正常释放。
     </strong>
    </p>
    <p>
     <strong>
      2.共享内存段被标记为已删除，但仍有进程附加（shmat）
     </strong>
    </p>
    <p>
     <strong>
      共享内存段已经被标记为已删除（不能附加到新的进程），但之前仍有一些进程附加到该共享内存段并正在使用。所以共享内存段不会被立即释放。只有当所有附加的进程都调用
      <code>
       shmdt
      </code>
      分离后，系统才会释放资源。
     </strong>
    </p>
    <h2 id="%E4%BA%94%E3%80%81%E6%95%B0%E6%8D%AE%E5%AE%89%E5%85%A8%E9%97%AE%E9%A2%98" name="%E4%BA%94%E3%80%81%E6%95%B0%E6%8D%AE%E5%AE%89%E5%85%A8%E9%97%AE%E9%A2%98">
     五、数据安全问题
    </h2>
    <p>
     <u>
      <span style="color:null">
       共享内存最大的优点就是
      </span>
      <span style="color:#fe2c24">
       快
      </span>
      <span style="color:null">
       ，相比使用管道技术，它减少了中间复杂转化和拷贝工作，而是直接对物理内存进行访问。
      </span>
     </u>
    </p>
    <p>
     但它也有一个致命的缺点，相比管道技术，共享内存它的读端和写端是不带有
     <span style="color:#fe2c24">
      <strong>
       同步机制
      </strong>
     </span>
     的，这就很容易使得数据混乱，也就是造成数据不一致问题。
    </p>
    <p>
     比如我们写端写入“hello world”，而读端读到的可能是“he”，“ll”，“o wor”，“ld”等等无法预测的奇葩数据。 读端一个劲地读，不会管写端这句话是否已经说完，而且也无法知道。
    </p>
    <p>
     当我们不了解锁的情况下想要解决这个问题，可以利用命名管道来解决，因为命名管道带有同步机制，我们用它的write和read函数来保护数据的安全，当然write和read并不用写或读什么有意义的数据。
    </p>

    <h3 id="1.comm.hpp" name="1.comm.hpp">
     1.comm.hpp
    </h3>
    <pre><code class="language-cpp">#pragma
#include &lt;iostream&gt;
#include &lt;string&gt;
#include &lt;cstdio&gt;
#include &lt;sys/ipc.h&gt;
#include &lt;sys/shm.h&gt;
#include &lt;sys/types.h&gt;
#include &lt;unistd.h&gt;
#define SIZE 1024
#define KEY_NUM 0x66
#define PATH "."
#define CREATE "create"
#define USE "user"
#define ERROR(str)   \
	{                \
		perror(str); \
		exit(1);     \
	}

using namespace std;
class Shm
{
private:
	void Create(int flg)
	{
		_shmid = shmget(_key, _size, flg);
		if (_shmid &lt; 0)
		{
			ERROR("shmget");
		}
		printf("shmget success id:%d\n", _shmid);
	}
	void Attach()
	{
		_start_mem = shmat(_shmid, nullptr, 0);
		if ((long long)_start_mem &lt; 0)
		{
			ERROR("shmat");
		}
	}
	void Destroy()
	{
		int m = shmdt(_start_mem);
		if (m &lt; 0)
		{
			ERROR("shmdt");
		}
		int n = shmctl(_shmid, IPC_RMID, nullptr);
		if (n &lt; 0)
		{
			ERROR("shmctl");
		}
	}

public:
	Shm(string path, int projid, string user)
		: _shmid(-1), _size(SIZE), _usertype(user)
	{
		_key = ftok(path.c_str(), projid);
		if (_key &lt; 0)
		{
			ERROR("ftok");
		}
		printf("ftok success id:%d\n", _key);
		if (_usertype == CREATE)
			Create(IPC_CREAT | IPC_EXCL | 0666);
		else
			Create(IPC_CREAT);
		Attach();
	}
	void *VirtualAddr()
	{
		return _start_mem;
	}
	~Shm()
	{
		Destroy();
	}

private:
	int _shmid;
	int _size;
	int _key;
	string _usertype;
	void *_start_mem;
};</code></pre>
