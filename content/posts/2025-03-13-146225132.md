    <h2>
     <a id="_5">
     </a>
     前言
    </h2>
    <p>
     JSONL（JSON Lines） 和 JSON 都是 JSON（JavaScript Object Notation）格式的变体，但它们的结构和使用场景有所不同。
    </p>
    <hr/>
    <h2>
     <a id="_13">
     </a>
     一、区别
    </h2>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        JSON
       </th>
       <th>
        JSONL
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         格式
        </strong>
       </td>
       <td>
        整个文件是一个标准的 JSON 结构
       </td>
       <td>
        每一行是一个独立的 JSON 对象
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         数据结构
        </strong>
       </td>
       <td>
        [] 代表数组，{} 代表对象
       </td>
       <td>
        纯文本，每行是一个 JSON 对象，不包含 []
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         解析方式
        </strong>
       </td>
       <td>
        需要完整解析整个
       </td>
       <td>
        JSON 可以逐行解析，每行都是一个独立的 JSON
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        配置文件、小型数据存储
       </td>
       <td>
        大规模数据处理、流式处理（日志、机器学习数据）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         处理方式
        </strong>
       </td>
       <td>
        一次性加载到内存
       </td>
       <td>
        可逐行读取，节省内存
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="1JSON__21">
     </a>
     1.JSON 示例
    </h3>
    <p>
     JSON 文件通常是一个完整的对象或数组：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
  <span class="token string-property property">"users"</span><span class="token operator">:</span> <span class="token punctuation">[</span>
    <span class="token punctuation">{<!-- --></span>
      <span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">1</span><span class="token punctuation">,</span>
      <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Alice"</span>
    <span class="token punctuation">}</span><span class="token punctuation">,</span>
    <span class="token punctuation">{<!-- --></span>
      <span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">2</span><span class="token punctuation">,</span>
      <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Bob"</span>
    <span class="token punctuation">}</span>
  <span class="token punctuation">]</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     或：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">[</span>
  <span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">1</span><span class="token punctuation">,</span>
    <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Alice"</span>
  <span class="token punctuation">}</span><span class="token punctuation">,</span>
  <span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">2</span><span class="token punctuation">,</span>
    <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Bob"</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">]</span>
</code></pre>
    <p>
     这种格式
     <strong>
      必须整体解析
     </strong>
     ，如果数据量过大，解析会占用大量内存。
    </p>
    <h3>
     <a id="2JSONL__52">
     </a>
     2.JSONL 示例
    </h3>
    <p>
     JSONL（JSON Lines）格式是一种按行存储 JSON 对象的纯文本格式，每一行都是一个完整的 JSON 对象：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span><span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">1</span><span class="token punctuation">,</span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Alice"</span><span class="token punctuation">}</span>
<span class="token punctuation">{<!-- --></span><span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Bob"</span><span class="token punctuation">}</span>
<span class="token punctuation">{<!-- --></span><span class="token string-property property">"id"</span><span class="token operator">:</span> <span class="token number">3</span><span class="token punctuation">,</span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Charlie"</span><span class="token punctuation">}</span>
</code></pre>
    <pre><code>注意：JSONL 文件没有逗号，每行是独立的 JSON 对象，不是 JSON 数组！
</code></pre>
    <p>
     <strong>
      JSONL 的特点
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       支持流式处理
      </strong>
      ：可以逐行读取，而不需要一次性加载整个文件。
     </li>
     <li>
      <strong>
       适合大规模数据
      </strong>
      ：如日志处理、大规模数据存储（如 Elasticsearch、BigQuery）。
     </li>
     <li>
      <strong>
       易扩展
      </strong>
      ：每行独立，便于追加和更新数据。
     </li>
    </ul>

    <h2>
     <a id="_101">
     </a>
     总结
    </h2>
    <p>
     ✅ JSON 适用于：结构化、小规模数据，一次性加载解析。
     <br/>
     ✅ JSONL 适用于：大规模数据、流式处理，每行是独立 JSON。
    </p>
    <p>
     如果你的数据量很大，或者是日志流式数据（比如 RAG 应用存储检索数据），建议使用 JSONL。
    </p>
