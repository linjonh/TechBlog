     一.冒泡排序
    </h3>
    <blockquote>
     <p>
      <strong>
       冒泡排序即
       <span style="color:#ed7976">
        qsort排序
       </span>
       ，其在几大排序中属于那种
       <span style="color:#ed7976">
        效率较低
       </span>
       的排序方式，相同条件下，qsort排序的时间要比其他排序多几十乃至几千倍，但由于其在排序过程中不会轻易改变原来数据的相对位置，因此它也算是一种
       <span style="color:#ed7976">
        较为稳定的函数
       </span>
       （关于排序函数的稳定性我会在下文带着慢慢说明），以下是冒泡排序的具体原理与代码：
      </strong>
     </p>
     <p>
      <span style="color:#79c6cd">
       <strong>
        在升序条件下，不断遍历左边的数据以找其中最大者然后不断循环放在右边
       </strong>
      </span>
     </p>
     <p>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/c3ea7b0bbe514faa8440c952a2e904ce.gif"/>
     </p>
     <pre><code class="language-cs">//冒泡排序
void BubbleSort(int* arr, int n)
{
	int end = n;
	while (end)
	{
		int flag = 0;
		for (int i = 1; i &lt; end; ++i)//代表每一次进入循环左边数据都会少一个
		{
			if (arr[i - 1] &gt; arr[i])
			{
				int tem = arr[i];
				arr[i] = arr[i - 1];
				arr[i - 1] = tem;//右边比左边大就交换
				flag = 1;
			}
		}
		if (flag == 0)
		{
			break;
		}
		--end;
	}
}
</code></pre>
     <p>
      <strong>
       看懂这样的代码之后可以仔细想想：冒泡排序在
       <span style="color:#79c6cd">
        最差的情况
       </span>
       下（即恰好每一轮的找最大值都需要它一个不漏地遍历其当时的所有数据）那么冒泡排序的时间复杂度为O（n^2）
       <span style="color:#98c091">
        (即1一直累加到n-1)
       </span>
       ，
       <span style="color:#79c6cd">
        最好的情况
       </span>
       下就是O（n），同时，在冒泡排序的每一趟过程中，假设有两个连续的一样的数据，那么冒泡排序在遍历过程中，是
       <span style="color:#ed7976">
        不会改变这两个数据的相对位置
       </span>
       的，因此在不考虑时间复杂度的情况下，所以我称冒泡排序这种排序方式是一种
       <span style="color:#faa572">
        稳定的排序方式
       </span>
      </strong>
     </p>
    </blockquote>
    <p>
    </p>
    <h3 id="%E4%BA%8C.%E5%A0%86%E6%8E%92%E5%BA%8F" name="%E4%BA%8C.%E5%A0%86%E6%8E%92%E5%BA%8F" style="background-color:transparent; text-align:center">
     二.堆排序
    </h3>
    <p>
    </p>
    <blockquote>
     <p>
      <strong>
       堆排序的具体使用我在以下文章
       <a class="link-info" href="https://blog.csdn.net/2403_87691282/article/details/145888014?spm=1001.2014.3001.5501" title="https://blog.csdn.net/2403_87691282/article/details/145888014?spm=1001.2014.3001.5501">
        https://blog.csdn.net/2403_87691282/article/details/145888014?spm=1001.2014.3001.5501
       </a>
       里有过说明，所以这里就允许我稍微偷下懒展示一下现成代码吧：
      </strong>
     </p>
     <pre><code class="language-cs">//向下调整算法
void AdjustDown(HP* hp, int parent, int n)//n是向下调整的下界范围
{
	int child = parent * 2 + 1;//公式
	while (child &lt; n)
	{
		//建小根堆
		if (child + 1 &lt; n &amp;&amp; hp-&gt;arr[child + 1] &lt; hp-&gt;arr[child])//child+1的目的是确保两个孩子结点都是有意义的结点
		{
			child++;
		}
		if (hp-&gt;arr[parent] &gt; hp-&gt;arr[child])
		{
			swap(&amp;hp-&gt;arr[parent], &amp;hp-&gt;arr[child]);
			parent = child;
			child = parent * 2 + 1;
		}
		else
		{
			//我最小的孩子都比父节点大，那么这个堆就是正常堆了，直接退出
			break;
		}
	}
}


// 升序，建⼤堆
// 降序，建⼩堆
// O(N*logN)

//基于数组的堆排序
void HeapSort(int* a, int n)
{
 
     // a数组直接建堆 O(N)
 for (int i = (n-1-1)/2; i &gt;= 0; --i)
 {
     AdjustDown(a, n, i);
 }
     // O(N*logN)
     int end = n - 1;
 while (end &gt; 0)
 {
     Swap(&amp;a[0], &amp;a[end]);
     AdjustDown(a, end, 0);
     --end;
 }
}</code></pre>
     <p>
      <strong>
       堆排序的时间复杂度也是比较好理解的：
       <span style="color:#ed7976">
        n*log（n）
       </span>
       ，log（n）是向下调整算法每一层遍历需要的时间，n则是每一次为了出堆顶数据而进行的循环调用，由于每一次的出堆顶数据都需要调用一次向下调整算法，因此堆排序的时间复杂度为n*log（n），而且需要补充的一点是由于每一次的取堆顶出堆顶，可能会导致原待排序数据中相同数据的
       <span style="color:#79c6cd">
        相对位置在堆排序后出现改变
       </span>
       ，因此也可以说
       <span style="color:#e6b223">
        尽管堆排序的效率在排序里算不错
       </span>
       的，但其
       <span style="color:#e6b223">
        稳定性是属于不稳定
       </span>
       的那一批
      </strong>
     </p>
    </blockquote>
    <p>
    </p>
    <h3 id="%E4%B8%89.%E6%8F%92%E5%85%A5%E6%8E%92%E5%BA%8F" name="%E4%B8%89.%E6%8F%92%E5%85%A5%E6%8E%92%E5%BA%8F" style="background-color:transparent; text-align:center">
     <strong>
      三.插入排序
     </strong>
    </h3>
    <blockquote>
     <p>
      <strong>
       插入排序的逻辑其实也不难：
      </strong>
     </p>
     <p>
      <strong>
       1.从第一个元素开始，
       <span style="color:#ed7976">
        假设该元素已经被排序
       </span>
       <br/>
       2.取下一个元素tem，从已排序的元素序列
       <span style="color:#ed7976">
        从后往前
       </span>
       遍历
       <br/>
       3.如果该元素大于tem，则将该元素移到下一位
       <br/>
       4.重复步骤3，直到找到已排序元素中小于等于tem的元素
       <br/>
       5.tem插入到该元素的后面，若已排序所有元素都大于tem，则将tem插入到下标为0的位置
       <br/>
       6.循环步骤2~5
      </strong>
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/direct/8feaf2f978f345b3938923ed240400d4.gif"/>
     </p>
     <pre><code class="language-cs">void InsertSort(int* arr, int n)
{
	for (int i = 0; i &lt; n - 1; ++i)
	{
		//记录有序序列最后一个元素的下标
		int end = i;
		//待插入的元素
		int tem = arr[end + 1];
		//单趟排
		while (end &gt;= 0)
		{
			//比插入的数大就向后移
			if (tem &lt; arr[end])
			{
				arr[end + 1] = arr[end];
				end--;
			}
			//比插入的数小，跳出循环
			else
			{
				break;
			}
		}
		//tem放到比插入的数小的数的后面
		arr[end  + 1] = tem;
		//代码执行到此位置有两种情况:
		//1.待插入元素找到应插入位置（break跳出循环到此）
		//2.待插入元素比当前有序序列中的所有元素都小（while循环结束后到此）
	}
}
</code></pre>
     <p>
      <strong>
       代码补充解释：由于end作为记录有序数据的最后一个数据的变量，所以在大循环for中，
       <span style="color:#faa572">
        end的实际值是一直在改变的
       </span>
       ，因此就需要在单趟排之后
       <span style="color:#98c091">
        随时重置tmp的位置
       </span>
       ，而且tmp作为始终处于end后一位的变量（作为end的侦察兵，始终为了确定end是否需要与后一位交换位置而存在），在小循环while里end值每交换一次，就代表其与tmp值交换位置，因此需要在
       <span style="color:#ed7976">
        每个小循环的最后再将end值--使其回到正确的实际值，
       </span>
       <span style="color:#0d0016">
        同样还有一点需要注意的就是直接插入排序的时间复杂度为O（n^2）,且由于其在遍历过程中不会改变相同数据的相对位置，所以其排序性质是稳定的
       </span>
      </strong>
     </p>
    </blockquote>
    <h3 id="" name="" style="text-align:center">
    </h3>
    <h3 id="%E5%9B%9B.%E9%80%89%E6%8B%A9%E6%8E%92%E5%BA%8F" name="%E5%9B%9B.%E9%80%89%E6%8B%A9%E6%8E%92%E5%BA%8F" style="text-align:center">
     四.选择排序
    </h3>
    <blockquote>
     <p>
      <strong>
       选择排序的思路也不算难，主要就是通过每次从待排序序列中挑一个最大再挑一个最小然后分别放在数组的两端，依次类推：
      </strong>
     </p>
     <p style="text-align:center">
      <img alt="" src="https://i-blog.csdnimg.cn/direct/5f2ac0adc4404df88c068fb185c7712f.gif"/>
     </p>
     <p style="text-align:center">
      <strong>
       具体代码：
      </strong>
     </p>
     <pre><code class="language-cs">//选择排序
void swap(int* a, int* b)
{
	int tem = *a;
	*a = *b;
	*b = tem;
}
void SelectSort(int* arr, int n)
{
	//保存参与单趟排序的第一个数和最后一个数的下标
	int begin = 0, end = n - 1;
	while (begin &lt; end)
	{
		//保存最大值的下标
		int maxi = begin;
		//保存最小值的下标
		int mini = begin;
		//找出最大值和最小值的下标
		for (int i = begin; i &lt;= end; ++i)
		{
			if (arr[i] &lt; arr[mini])
			{
				mini = i;
			}
			if (arr[i] &gt; arr[maxi])
			{
				maxi = i;
			}
		}
		//最小值放在序列开头
		swap(&amp;arr[mini], &amp;arr[begin]);
		//最大值放在序列结尾
		swap(&amp;arr[maxi], &amp;arr[end]);
		++begin;
		--end;
	}
}
</code></pre>
     <p>
      <strong>
       当理解了选择排序后有一点需要注意的就是
       <span style="color:#98c091">
        直接选择排序
       </span>
       的时间复杂度无论是
       <span style="color:#98c091">
        最佳情况
       </span>
       （即待排序数据已经排序完毕）还是
       <span style="color:#98c091">
        最差情况
       </span>
       （每一次找最大最小都要遍历全部待排序数据）都是
       <span style="color:#98c091">
        O（n^2）
       </span>
       ,这点与冒泡排序不同，
       <span style="color:#ed7976">
        冒泡排序
       </span>
       虽然最差情况时与直接排序相同，但在最好情况下的
       <span style="color:#ed7976">
        时间复杂度却是O（n）
       </span>
       ，主要还是因为冒泡排序如果排已经拍好的数组时，其中的小循环几乎已经不需要进入执行了，但直接选择排序即便是遍历已经排好的数组也仍旧需要从头到尾遍历一遍，并且直接选择在遇到相同数据时同样是有可能会改变原始数据的相对顺序的，
       <span style="color:#6eaad7">
        是不稳定的
       </span>
      </strong>
     </p>
    </blockquote>
    <p>
    </p>
    <h3 id="%E4%BA%94.%E5%B8%8C%E5%B0%94%E6%8E%92%E5%BA%8F" name="%E4%BA%94.%E5%B8%8C%E5%B0%94%E6%8E%92%E5%BA%8F" style="background-color:transparent; text-align:center">
     五.希尔排序
    </h3>
    <p>
    </p>
    <blockquote>
     <p>
      <strong>
       希尔排序的逻辑可能就有些复杂：其基本思想为先选定⼀个整数（通常是
       <span style="color:#ed7976">
        gap=n/3+1
       </span>
       ），把 待排序⽂件所有记录分成各组，所有的
       <span style="color:#faa572">
        距离相等的记录分在同⼀组内
       </span>
       ，并对每⼀组内的记录进行排序，然后gap=gap/3+1得到下⼀个整数，再将数组分成各组，进行插入排序，当gap=1时，就相当于直接插入排序，
      </strong>
     </p>
     <p>
      <strong>
       再简单点来说就是
       <span style="color:#ed7976">
        先将待排序列进行预排序，使待排序列接近有序，然后再对该序列进行一次插入排序，此时插入排序的时间复杂度为O(N)
       </span>
      </strong>
     </p>
     <p style="text-align:center">
      <strong>
       由于它是在直接插入排序算法的基础上进行改进⽽来的，综合来说它的效率肯定是更高的
      </strong>
      <img alt="" height="246" src="https://i-blog.csdnimg.cn/direct/11d51eaf6d8541359aeaab1b948f121f.png" width="258">
       <strong>
        <img alt="" height="346" src="https://i-blog.csdnimg.cn/direct/23e8499c6aa744ea816b6cafa144e848.gif" width="642"/>
       </strong>
      </img>
     </p>
     <pre><code class="language-cs">//希尔排序
void ShellSort(int* arr, int n)
{
	int gap = n;
	while (gap&gt;1)
	{
		//每次对gap折半操作
		gap = gap / 2;
		//单趟排序
		for (int i = 0; i &lt; n - gap; ++i)
		{
			int end = i;
			int tem = arr[end + gap];
			while (end &gt;= 0)
			{
				if (tem &lt; arr[end])
				{
					arr[end + gap] = arr[end];
					end -= gap;
				}
				else
				{
					break;
				}
			}
			arr[end + gap] = tem;
		}
	}
}
</code></pre>
     <p>
      <strong>
       代码方面的解释也不算很难，希尔排序通过对
       <span style="color:#ed7976">
        原始数据不断的分小组细化
       </span>
       ，然后对每一个小组使用
       <span style="color:#98c091">
        直接插入排序
       </span>
       ，从而减少直接插入排序算法里循环的次数，希尔算法里对于其中小循环的解释跟直接插入排序算法是有一样的，都是基于end与tmp的相对位置不变，然后小的往前放，大的往后放并且随时重置tmp（希尔里就是end+gap）使其一直处于end的前面而实现的算法，同样，我们再说到其时间复杂度，这里希尔算法的时间复杂度由于数学理论的局限，因此：
      </strong>
      <img alt="" height="395" src="https://i-blog.csdnimg.cn/direct/7c504e8bde654d70aff7f26ba9b87117.png" width="1349"/>
     </p>
     <p>
      <strong>
       但唯一可以确定的是，希尔排序确实在很大程度上优化了直接插入算法
      </strong>
     </p>
    </blockquote>
    <p>
    </p>
    <h3 id="%E5%85%AD.%E5%BF%AB%E9%80%9F%E6%8E%92%E5%BA%8F" name="%E5%85%AD.%E5%BF%AB%E9%80%9F%E6%8E%92%E5%BA%8F" style="text-align:center">
     <strong>
      六.快速排序
     </strong>
    </h3>
    <blockquote>
     <p>
      <span style="color:#98c091">
       <em>
        <strong>
         快速排序我认为是几大排序里最复杂也是探究的最深的一个排序算法思想，其
        </strong>
       </em>
      </span>
      <span style="color:#ed7976">
       <em>
        <strong>
         根本的逻辑
        </strong>
       </em>
      </span>
      <span style="color:#98c091">
       <em>
        <strong>
         就是通过先在原始数据任意建立一个参考点，然后通过各种方法使参考点的左边的数据小于参考点，然后使参考点右边的数据大于参考点
        </strong>
       </em>
      </span>
     </p>
     <p>
      <img alt="" height="645" src="https://i-blog.csdnimg.cn/direct/5ac68a48dcd1463fa3a7707c462c896a.png" width="1185"/>
     </p>
    </blockquote>
    <h4 id="1.Lomuto%E7%89%88%E6%9C%AC%EF%BC%88%E5%89%8D%E5%90%8E%E6%8C%87%E9%92%88%E6%B3%95%EF%BC%89" name="1.Lomuto%E7%89%88%E6%9C%AC%EF%BC%88%E5%89%8D%E5%90%8E%E6%8C%87%E9%92%88%E6%B3%95%EF%BC%89" style="background-color:transparent">
     1.Lomuto版本（前后指针法）
    </h4>
    <blockquote>
     <p>
      <strong>
       思路：
       <br/>
       1.
       <span style="color:#ed7976">
        选出一个keyi
       </span>
       ，一般是最左边或是最右边的。
       <br/>
       2.起始时，prev指针指向序列开头，
       <span style="color:#fbd4d0">
        cur
       </span>
       指针指向
       <span style="color:#fbd4d0">
        prev+1
       </span>
       。
       <br/>
       3.若cur指向的内容小于keyi，则prev先向后移动一位，然后交换prev和cur指针指向的内容，然后cur指针++；若cur指向的内容大于key，则cur指针直接++，
       <span style="color:#79c6cd">
        （因此在排序过程中在没遇到大于keyi参考值时，prev和cur都会在下一步之前重合，只有当遇到大于参考值时，cur才会多走一步从而拉开与prev的距离，而这样拉开一步的解雇就是使prev的下一步指向值是一个比参考值大的数，因此此时交换prev和cur就可以实现数据大的往后移动）
       </span>
       如此进行下去，直到cur到达end+1(出界)位置，此时将keyi和prev指针指向的内容交换即可。
      </strong>
     </p>
     <p>
      <strong>
       经过一次单趟排序，最终也能使得key左边的数据全部都小于key，key右边的数据全部都大于key。
      </strong>
     </p>
     <p>
      <strong>
       4.然后也还是将key的左序列和右序列再次进行这种单趟排序，如此反复操作下去，直到左右序列只有一个数据，或是左右序列不存在时，便停止操作
      </strong>
     </p>
     <pre><code class="language-cs">//Lomuto前后指针
int _QuickSort2(int* arr, int left, int right)
{
	int keyi = left;
	int prev = left, cur = prev + 1;
	while(cur &lt;= right)
	{
		if (arr[cur] &lt; arr[keyi] &amp;&amp; ++prev != cur)
		{
			Swap(&amp;arr[prev], &amp;arr[cur]);
		}
		++cur;
	}
	Swap(&amp;arr[prev], &amp;arr[keyi]);
	return prev;
}
</code></pre>
    </blockquote>

	StackPussh(&amp;st, left);

	{
		//取栈顶两次
		int begin = StackTop(&amp;st);
		StackPop(&amp;st);

		int end = StackTop(&amp;st);
		StackPop(&amp;st);

		int key1 = begin;

				Swap(&amp;arr[prev], &amp;arr[cur]);
			}
			cur++;
		}
		Swap(&amp;arr[prev], &amp;arr[key1]);


		key1 = prev;//重新设定基准值以区分两个数组
		//begin key1 end
		//左序列【begin，key1-1】  右序列【key1+1，end】

			StackPush(&amp;st, begin);

	}

void QuickSort(int* arr, int begin, int end)
{
	//只有一个数或区间不存在
	if (begin &gt;= end)
		return;
	int left = begin;
	int right = end;
	//选左边为key
	int keyi = begin;
	while (begin &lt; end)
	{
		//右边选小   等号防止和key值相等    防止顺序begin和end越界
		while (arr[end] &gt;= arr[keyi] &amp;&amp; begin &lt; end)
		{
			--end;
		}
		//左边选大
		while (arr[begin] &lt;= arr[keyi] &amp;&amp; begin &lt; end)
		{
			++begin;
		}
		//小的换到右边，大的换到左边
		swap(&amp;arr[begin], &amp;arr[end]);
	}
	swap(&amp;arr[keyi], &amp;arr[end]);
	keyi = end;
	//[left,keyi-1]keyi[keyi+1,right]
	QuickSort(arr, left, keyi - 1);
	QuickSort(arr,keyi + 1,right);
}

{
	if (left &gt;= right)
	{
		return;
	}
	int mid = (right + left) / 2;
	//[left,mid] [mid+1,right]
	_MergeSort(a, left, mid, tmp);
	_MergeSort(a, mid + 1, right, tmp);

	int begin1 = left, end1 = mid;
	int begin2 = mid + 1, end2 = right;
	int index = begin1;

	 //合并两个有序数组为⼀个数组 
	while (begin1 &lt;= end1 &amp;&amp; begin2 &lt;= end2)
	{
		if (a[begin1] &lt; a[begin2])
		{
			tmp[index++] = a[begin1++];
		 }
		else
		{
			tmp[index++] = a[begin2++];
		 }
	}
	while (begin1 &lt;= end1)
	{
		tmp[index++] = a[begin1++];
	}
    while (begin2 &lt;= end2)
	{
		tmp[index++] = a[begin2++];
	}
	for (int i = left; i &lt;= right; i++)
	{
		a[i] = tmp[i];
	}
}
