    <h2>
     一、创建公司数据库表格
    </h2>
    <p>
     <strong>
      创建employee表格
     </strong>
    </p>
    <pre><code class="language-sql">create table employee(
	emp_id int primary key,
    `name` varchar(20),
    birth_date date,
    sex varchar(20),
    salary int,
    branch_id int,
    sup_id int
);
</code></pre>
    <p>
     在这里并没有将branch_id和sup_id设计为外键，是应为其branch和employee表格还没有创建，所以之后会再返回来设计。
    </p>
    <p>
     <strong>
      创建branch表格
     </strong>
    </p>
    <pre><code class="language-sql">create table branch(
	branch_id int primary key,
    branch_name varchar(20),
    manager_id int,
    foreign key (manager_id) references employee(emp_id) on delete set null
);</code></pre>
    <p>
     这里设置manager_id为外键，references关联上employee表格的emp_id属性，这里先不用知道为什么加上 on delete set null，之后会详细的说。
    </p>
    <p>
     现在补上branch_id和sup_id为外键，因为现在branch和employee表格已经创建好了。
    </p>
    <p>
     <strong>
      branch_id：
     </strong>
    </p>
    <pre><code class="language-sql">alter table employee 
add foreign key (branch_id) 
references branch(branch_id) 
on delete set null;</code></pre>
    <p>
     <strong>
      sup_id：
     </strong>
    </p>
    <pre><code class="language-sql">alter table employee 
add foreign key (sup_id) 
references employee(emp_id) 
on delete set null;</code></pre>
    <p>
     <strong>
      创建clint表格
     </strong>
    </p>
    <pre><code class="language-sql">create table `client`(
	client_id int primary key,
    client_name varchar(20),
    phone varchar(20)
);
</code></pre>
    <p>
     <strong>
      创建works_with表格
     </strong>
    </p>
    <pre><code class="language-sql">create table `works_with`(
	emp_id int,
    client_id int,
    total_sales int,
    primary key(emp_id, client_id),
    foreign key (emp_id) references employee(emp_id) on delete cascade,
    foreign key (client_id) references `client`(client_id) on delete cascade
);</code></pre>
    <p>
     同样的这里先不用知道为什么加上 on delete cascade，之后会详细的说。
    </p>

    <h2>
     六、Union
    </h2>
    <p>
     通过union操作，就可以将搜寻结果结合在一起。
    </p>
    <p>
     <strong>
      -- 1.员工名字 union 客户名字
     </strong>
    </p>
    <pre><code class="language-sql">select `name` 
from employee
union 
select `client_name`
from `client`;</code></pre>
    <p class="img-center">
     <img alt="" height="300" src="https://i-blog.csdnimg.cn/direct/885b96b232484e9dbd73cdce9f646816.png" width="242"/>
    </p>
    <p>
     在使用union得时候，其属性得数量是要求一致的，不能属性数量不一致，否则就会导致出错。
    </p>
    <p>
     <strong>
      -- 2.员工id +员工名子 union 客户id + 客户名字
     </strong>
    </p>
    <pre><code class="language-sql">select `emp_id`, `name` 
from employee
union 
select `client_id`, `client_name`
from `client`;</code></pre>
    <p class="img-center">
     <img alt="" height="297" src="https://i-blog.csdnimg.cn/direct/5bd0939125de4e789c7e989317c09a3a.png" width="300"/>
    </p>
    <p>
     可以看见这样返回的结果其实是以第一个用了第一个选择的属性的，如果想改变其和并后的属性名字的话，使用以下代码就可以：
    </p>
    <pre><code class="language-sql">select `emp_id` as `total_id`, `name`as `total_name`
from employee
union 
select `client_id`, `client_name`
from `client`;</code></pre>
    <p class="img-center">
     <img alt="" height="229" src="https://i-blog.csdnimg.cn/direct/27de1c3ac3504761b02387f1fa31eec0.png" width="300"/>

    <h2>
     七、join 连接
    </h2>
    <p>
     可以帮助我们将两个表格连接在一起
    </p>
    <p>
     <strong>
      -- 取得所有部门经理的名字
     </strong>
    </p>
    <p>
     要想取得部门经理的名字，那我们就得先去找部门（branch）的表格，而branch表格中之后中只有manmager_id，并没有manager的名字，所以要通过manager_id找到对应的名字。
    </p>
    <pre><code class="language-sql">select *
from employee
join branch on manager_id = emp_id;</code></pre>
    <p class="img-center">
     <img alt="" height="208" src="https://i-blog.csdnimg.cn/direct/f8173e18b37a4816966cab6a0e18d699.png" width="600"/>
    </p>
    <p>
     为什么只显示了三条数据，是因为再进行连接的时候是通过manager_id = emp_id进行连接的，所以只有206、207、208匹配上了。
    </p>
    <p>
     现在不想显示那么多的数据，就可以通过以下代码来实现：
     <br/>
    </p>
    <pre><code class="language-sql">select emp_id, `name`, branch_name
from employee
join branch on manager_id = emp_id;</code></pre>
    <p class="img-center">
     <img alt="" height="249" src="https://i-blog.csdnimg.cn/direct/12aaf6b5a896476a9005fc271b835eb1.png" width="400"/>
    </p>
    <p>
     若在连接的时候，两个表格中的属性，有重复，比如假设branch表中的manager_id属性叫id，而employee表中emp_id也叫id的话，那再连接的时候应该改成
     <strong>
      `employee`.`emp_id` = `branch`.`brach_id`
     </strong>
     ，这样指定好属性来自于哪个表格。
    </p>

    <h2>
     八、subquery 子查询
    </h2>
    <p>
     是表示在一个查询语句中可以使用另外一个查询语句。
    </p>
    <p>
     <strong>
      -- 1.找出研发部门的经理名字
     </strong>
    </p>
    <pre><code class="language-sql">select `name` 
from employee
where emp_id = (
	select manager_id
	from branch
	where branch_name = '研发'
);</code></pre>

     <img alt="" height="337" src="https://i-blog.csdnimg.cn/direct/6bf9a3d5eb12487498df6183ffb9df66.png" width="400"/>
    </p>
    <pre><code class="language-sql">select manager_id
	from branch
	where branch_name = '研发'</code></pre>
    <p>
     首选通过子查询，查询到了研发部门的manager_id，然后再通过这个子查询的结果去查询该id在employee中对应的名称。
     <br/>
     -- 2.找出对单一位客戶销售金额超过50000的员工名字
    </p>
    <pre><code class="language-sql">select `name` 
from employee
where emp_id in (
	select emp_id 
	from works_with
	where total_sales &gt; 50000
);</code></pre>
    <p class="img-center">

    <h2>
     九、解释on delete set null 和on delete cascade
    </h2>
    <p class="img-center">
     <img alt="" height="285" src="https://i-blog.csdnimg.cn/direct/23c5924da8e94c58b29b082211c1116d.png" width="400"/>
    </p>
    <p>
     <strong>
      on delete set null：
     </strong>
     当时在设置branch表格的时候，其中的外键manager_id是referneces到了employee表格的emp_id的，那假设employee中的id为206的员工离职了，那branch表格中的manager_id该怎么办呢？所以加上on delete set null表示在外键manager_id所referneces的employee表格中的emp_id被删除后，自动将manager_id设置为null。
    </p>
    <p>
     <strong>
      测试：
     </strong>
    </p>
    <pre><code class="language-sql">delete from employee
where emp_id = 207;
select * from employee;</code></pre>
    <p class="img-center">
     <img alt="" height="221" src="https://i-blog.csdnimg.cn/direct/4ace67b3ff7a4bb588ae221ce0d0e8ef.png" width="400"/>
    </p>
    <p>
    </p>
    <p>
     当207被删除后，其外键sup_id中是207的都被设置为了空（即209和210）同时branch表中的外键manager_id也被设置为了空。
    </p>
    <p class="img-center">
     <img alt="" height="174" src="https://i-blog.csdnimg.cn/direct/839591e0b80847cda8b3802f4cd4af3e.png" width="250"/>
    </p>
    <p>
     <strong>
      on delete cascade：
     </strong>
     当时在设置works_with表格的时候，其中的外键emp_id是referneces到了employee表格的emp_id的，那假设employee中的id为206的员工离职了，那works_with表格中的emp_id该怎么办呢？这里加上on delete cascade表示在外键emp_id所referneces的employee表格中的emp_id被删除后，自动将works_with这行删除。
     <strong>
      <span style="color:#be191c">
       <span style="background-color:#fef2f0">
        注意：这里不能设置为on delete set null，因为works_with表格中的emp_id同时是主键和外键，主键不能为空，所以只能是被删除该记录而不能被设置为空null。
       </span>
      </span>
     </strong>
    </p>
