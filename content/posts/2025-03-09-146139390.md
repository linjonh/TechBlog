    <h2>
     一、同步调用
    </h2>
    <blockquote>
     <p>
      优势：时效性强，等待到结果后才返回
     </p>
    </blockquote>
    <blockquote>
     <p>
      缺点：
     </p>
     <ol>
      <li>
       拓展性差
      </li>
      <li>
       性能下降
      </li>
      <li>
       级联失败问题
      </li>
     </ol>
    </blockquote>
    <h2>
     二、异步调用
    </h2>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/56c87a8238be4067bf7e21e5136a569a.png"/>
    </p>
    <blockquote>
     <p>
      优势：
     </p>
     <ol>
      <li>
       耦合度低，拓展性强
      </li>
      <li>
       异步调用，无需等待，性能好
      </li>
      <li>
       故障隔离，下游服务故障不影响上游业务
      </li>
      <li>
       缓存消息，流量削峰填谷
      </li>
     </ol>
    </blockquote>
    <blockquote>
     <p>
      缺点：
     </p>
     <ol>
      <li>
       不能立即得到调用结果，时效性差
      </li>
      <li>
       不确定下游业务执行是否成功
      </li>
      <li>
       业务安全依赖于Broker（消息代理）的可靠性
      </li>
     </ol>
    </blockquote>
    <h2>
     三、初识MQ
    </h2>
    <p>
     MQ（
     <strong>
      M
     </strong>
     essage
     <strong>
      Q
     </strong>
     ueue），中文是
     <strong>
      消息队列
     </strong>
     ，字面来看就是存放消息的队列。也就是异步调用中的Broker
    </p>
    <h2>
     四、RabbitMQ
    </h2>
    <p>
     RabbitMQ是基于Erlang语言开发的开源消息通信中间件
    </p>
    <p>
     <a class="link-info" href="https://www.rabbitmq.com/" rel="nofollow" title="RabbitMQ官网">
      RabbitMQ官网
     </a>
    </p>
    <h3>
     1.安装
    </h3>
    <p>
     基于Docker来安装RabbitMQ
    </p>
    <pre><code class="hljs">docker run \
 -e RABBITMQ_DEFAULT_USER=itheima \
 -e RABBITMQ_DEFAULT_PASS=123321 \
 -v mq-plugins:/plugins \
 --name mq \
 --hostname mq \
 -p 15672:15672 \
 -p 5672:5672 \
 --network hm-net\
 -d \
 rabbitmq:3.8-management</code></pre>
