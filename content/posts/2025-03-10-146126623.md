     2.什么是 Spring Web MVC？
    </h3>
    <blockquote>
     <p>
      Spring Web MVC 是基于 Servlet API 构建的原始 Web 框架，从⼀开始就包含在 Spring 框架中。它的正式名称“Spring Web MVC”来⾃其源模块的名称(Spring-webmvc)，但它通常被称为"Spring MVC"。
     </p>
    </blockquote>
    <p>
     从上述定义我们可以得出⼀个信息： Spring Web MVC 是⼀个 Web 框架.
     <br/>
     下⾯咱们简称之为: Spring MVC
     <br/>
     然而要真正的理解什么是 Spring MVC？我们首先要搞清楚什么是 MVC。
    </p>
    <h4>
     <a id="21_MVC__23">
     </a>
     2.1 MVC 定义
    </h4>
    <blockquote>
     <p>
      MVC 是 Model View Controller 的缩写，它是软件工程中的⼀种软件架构设计模式，它把软件系统分为模型、视图和控制器三个基本部分.
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9f846cb2bcd94c4bad2296989f7f6a01.png"/>
    </p>
    <p>
     <font color="blue">
      • View(视图) 指在应用程序中专门用来与浏览器进行交互，展示数据的资源。
      <br/>
      • Model(模型) 是应用程序的主体部分，用来处理程序中数据逻辑的部分。
      <br/>
      • Controller（控制器）可以理解为⼀个分发器，用来决定对于视图发来的请求，需要用哪⼀个模型来处理，以及处理完后需要跳回到哪⼀个视图。即用来连接视图和模。
     </font>
    </p>
    <blockquote>
     <p>
      比如: 厨房可以用来做饭, 但真实实现做饭功能的是火以及各种做饭相关的⻝材和工具。
      <br/>
      厨房就好比是SpringBoot, 厨房可以装柜子, 实现收纳功能, 装燃气灶等, 实现做饭功能。
      <br/>
      做饭这个事, 就是MVC, 在几千年前, 有火有食材就可以实现。
     </p>
    </blockquote>
    <p>
     学习Spring MVC, 重点也就是学习如何通过浏览器和用户程序进行交互。
     <br/>
     主要分以下
     <strong>
      三个方面
     </strong>
     :
    </p>
    <ol>
     <li>
      <strong>
       建立连接
      </strong>
      ：将用户（浏览器）和 Java 程序连接起来，也就是访问⼀个地址能够调用到我们的Spring 程序。
     </li>
     <li>
      <strong>
       请求
      </strong>
      : 用户请求的时候会带⼀些参数，在程序中要想办法获取到参数, 所以请求这块主要是 获取参数的功能。
     </li>
     <li>
      <strong>
       响应:
      </strong>
      执行了业务逻辑之后，要把程序执行的结果返回给用户, 也就是响应。
     </li>
    </ol>
    <blockquote>
     <p>
      比如用户去银行存款
      <br/>
      1.建立连接: 去柜台
      <br/>
      2.请求: 带着银行卡, 身份证去存款
      <br/>
      3.响应: 银行返回⼀张存折
      <br/>
      对于 Spring MVC 来说，掌握了以上 3 个功能就相当于掌握了 Spring MVC。
     </p>
    </blockquote>

     <code>

      </strong>

    </ul>

     返回字符串：可以直接返回一个字符串，@ResponseBody会将字符串内容写入到响应的正文中。

    <h4>
     <a id="35_GetMappingPostMapping__137">
     </a>
     3.5
     <code>
      @GetMapping
     </code>
     、
     <code>
      @PostMapping
     </code>
     等
    </h4>
    <p>
     当然，以下是对
     <code>
      @GetMapping
     </code>
     和
     <code>
      @PostMapping
     </code>
     的更详细说明：
    </p>
    <hr/>
    <ol>
     <li>
      <code>
       @GetMapping
      </code>
     </li>
    </ol>
    <ul>
     <li>
      <p>
       <strong>
        用途
       </strong>
       ：
       <br/>
       用于映射 HTTP 的 GET 请求，processing incoming HTTP GET requests.
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        GET 请求是通过 URL 参数传递数据的。
       </li>
       <li>
        适用于查询、获取资源、检索数据等场景。
       </li>
       <li>
        数据是可见的，通常用于安全性要求不高的场景。
       </li>
       <li>
        GET 请求是
        <strong>
         幂等
        </strong>
        的（Idempotent），即多次调用同一个接口，结果不会改变。
       </li>
       <li>
        GET 请求的数据大小有限制，因为数据是通过 URL 传递的。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <pre><code class="prism language-java"><span class="token annotation punctuation">@GetMapping</span><span class="token punctuation">(</span><span class="token string">"/users"</span><span class="token punctuation">)</span>
<span class="token keyword">public</span> <span class="token class-name">List</span><span class="token generics"><span class="token punctuation">&lt;</span><span class="token class-name">User</span><span class="token punctuation">&gt;</span></span> <span class="token function">getAllUsers</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        常用属性
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         value
        </code>
        或
        <code>
         path
        </code>
        ：指定映射的 URL路径。
        <pre><code class="prism language-java"><span class="token annotation punctuation">@GetMapping</span><span class="token punctuation">(</span>path <span class="token operator">=</span> <span class="token string">"/users/{id}"</span><span class="token punctuation">)</span>

    <span class="token keyword">return</span> userService<span class="token punctuation">.</span><span class="token function">getUserById</span><span class="token punctuation">(</span>id<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ol start="2">
     <li>
      <code>
       @PostMapping
      </code>
     </li>
    </ol>
    <ul>
     <li>
      <p>
       <strong>
        用途
       </strong>
       ：
       <br/>
       用于映射 HTTP 的 POST 请求，processing incoming HTTP POST requests.
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        POST 请求通常通过请求体（Body）传递数据。
       </li>
       <li>
        适用于提交表单、创建新资源、更新数据等场景。
       </li>
       <li>
        POST 请求是
        <strong>
         非幂等
        </strong>
        的（Non-Idempotent），即多次调用同一个接口，可能会创建多个资源。
       </li>
       <li>
        POST 请求可以传递更大的数据，适合复杂的提交操作。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例
       </strong>
       ：
      </p>
      <pre><code class="prism language-java"><span class="token annotation punctuation">@PostMapping</span><span class="token punctuation">(</span><span class="token string">"/users"</span><span class="token punctuation">)</span>
<span class="token keyword">public</span> <span class="token class-name">User</span> <span class="token function">createUser</span><span class="token punctuation">(</span><span class="token annotation punctuation">@RequestBody</span> <span class="token class-name">User</span> user<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">return</span> userService<span class="token punctuation">.</span><span class="token function">createUser</span><span class="token punctuation">(</span>user<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        常用属性
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         value
        </code>
        或
        <code>
         path
        </code>
        ：指定映射的 URL路径。
       </li>
       <li>
        <code>
         consumes
        </code>
        ：指定接收的内容类型（如
        <code>
         application/json
        </code>
        ）。
       </li>
       <li>
        <code>
         produces
        </code>
        ：指定返回的内容类型（如
        <code>
         application/json

    <h3>
     <a id="5_MVC__338">
     </a>
     5. MVC 和三层架构的设计模式
    </h3>
    <h4>
     <a id="51_MVC__340">
     </a>
     5.1 MVC 模式
    </h4>
    <ul>
     <li>
      <strong>
       Model（模型）
      </strong>
      ：负责数据管理和业务逻辑。
     </li>
     <li>
      <strong>
       View（视图）
      </strong>
      ：负责展示数据。
     </li>
     <li>
      <strong>
       Controller（控制器）
      </strong>
      ：负责处理用户请求，并协调 Model 和 View。
     </li>
    </ul>
    <h4>
     <a id="52__346">
     </a>
     5.2 三层架构
    </h4>
    <ul>
     <li>
      <strong>
       表现层（Web层）
      </strong>
      ：负责与用户交互，接收请求并返回响应。
     </li>
     <li>
      <strong>
       业务逻辑层（Service层）
      </strong>
      ：负责处理业务逻辑。
     </li>
     <li>
      <strong>
       数据访问层（DAO层）
      </strong>
      ：负责与数据库交互。
     </li>
    </ul>

     <a id="6__354">
     </a>
     6. 企业开发中的命名规范
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        类名
       </strong>
      </p>
      <ul>
       <li>
        使用大驼峰命名法（首字母大写）。
       </li>
       <li>
        例如：
        <code>
         UserController
        </code>
        、
        <code>
         UserServiceImpl
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        方法名
       </strong>
      </p>
      <ul>
       <li>
        使用小驼峰命名法（首字母小写）。
       </li>
       <li>
        例如：
        <code>
         getUserById
        </code>
        、
        <code>
         saveUser
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        变量名
       </strong>
      </p>
      <ul>
       <li>
        使用小驼峰命名法。
       </li>
       <li>
        例如：
        <code>
         userName
        </code>
        、
        <code>
         userEmail
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        包名
       </strong>
      </p>
      <ul>
       <li>
        使用小写字母，分层命名。
       </li>
       <li>
        例如：
        <code>
         com.example.controller
        </code>
        、
        <code>
         com.example.service
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        API 命名
       </strong>
      </p>
      <ul>
       <li>
        使用动词 + 名词的形式。
       </li>
       <li>
        例如：
        <code>
         GET /api/users
        </code>
        、
        <code>
         POST /api/users
        </code>
        。
       </li>
      </ul>
