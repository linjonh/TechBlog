    <h3 id="%E5%89%8D%E8%A8%80%EF%BC%9A">
     前言：
    </h3>
    <blockquote>
     <p>
      接口其实是为了弥补继承的缺点：
      <strong>
       <span style="color:#fe2c24">
        我们无法让父类完美的适应每一个子类的要求
       </span>
      </strong>
      。例如我们有一个动物类的父类，有一个蝌蚪类，鱼类，兔子类。如果我们要在蝌蚪和鱼类里面添加一个游泳方法，但是父类中没有，此时我们只能够在鱼类里面实现一个游泳方法，在蝌蚪类里面提供一个游泳方法，这就又拉低了代码的执行效率，因此我们就创建了接口这个概念。
     </p>
    </blockquote>
    <h3>
     接口：
    </h3>
    <p>
     <span style="background-color:#ffd900">
     </span>
     <strong>
      <span style="background-color:#ffd900">
       接口（Interface）是面向对象编程中的一个重要概念，它定义了一个类的外部形式，以定规范使用这个类的方式
      </span>
     </strong>
     <span style="background-color:#ffd900">
      。
     </span>
     可以理解为一种约定，规定类能够提供哪些方法和属性。
     <strong>
      接口通常只包含方法的声明，而不包含具体实现。
     </strong>
     它们在编程中被用来描述一个或多个类应该支持哪些方法以及这些方法应该如何调用。
    </p>
    <p>
     在Java中，接口是使用关键字“
     <strong>
      interface
     </strong>
     ”来声明的。一个接口的方法必须在实现它的类中进行实现，否则编译器就会报错。在实现类中，需要使用“
     <strong>
      implements
     </strong>
     ”关键字来实现接口的方法。
    </p>
    <p>
     需要注意的是，
     <strong>
      一个类可以实现多个接口，但只能继承一个父类。这使得接口成为一种弥补java单继承限制的方式，通过实现多个接口可以获得多种特性。
     </strong>
    </p>
    <p>
     总之，接口为我们提供了一种规范、高度抽象的编程方式，可以实现代码的灵活性和可扩展性。
    </p>
    <p>
     其实换句话来讲：
     <strong>
      <span style="background-color:#ffd900">
       接口弥补了只能继承一个父类的缺点，为其他类提供了一个统一的功能函数。
      </span>
     </strong>
    </p>
    <p>
     回到刚才的例子，我们就可以创建一个游泳接口，让鱼类和蝌蚪类都使用这个接口，我们就实现了提高效率的目的。
    </p>
    <h3>
     如何定义一个接口:
    </h3>
    <p>
     1. 使用关键字
     <strong>
      interface
     </strong>
     创建接口。
     <br/>
     2. 接口中可以声明常量和方法，但是不能声明成员变量和构造方法，接口中的方法不能有具体实现。
     <br/>
     3. 通过接口中方法的声明，来规范类的使用方式。
    </p>
    <p>
     下面是一个示例接口的定义：
    </p>
    <pre><code class="hljs">public interface Animal {
    static final String TYPE = "animal";
    void run();
    void eat(String food);
}</code></pre>
    <h3>
     如何使用一个接口：
    </h3>
    <p>
     1. 类通常必须实现接口中的所有方法才能使用该接口。
     <br/>
     2. 使用关键字implements进行接口的实现。
     <br/>
     3. 实现类必须覆盖接口中的所有方法，以提供具体的实现。
     <br/>
    </p>
    <p>
     下面是一个示例实现类的定义：
    </p>
    <pre><code class="hljs">public class Dog implements Animal {
    public void run() {
        System.out.println("我是一条狗，我会跑！");
    }
    public void eat(String food) {
        System.out.println("我是一条狗，我正在吃" + food);
    }
 
}</code></pre>
    <h3 id="%E6%8E%A5%E5%8F%A3%E4%B8%AD%E6%88%90%E5%91%98%E7%9A%84%E7%89%B9%E7%82%B9%EF%BC%9A">
     接口中成员的特点：
    </h3>
    <p>
     成员变量：
     <br/>
     只能是常量，默认修饰符是public static final。
    </p>
    <p>
     构造方法：
     <br/>
     没有
    </p>
    <p>
     方法成员：
     <br/>
     只能是
     <a href="https://so.csdn.net/so/search?q=%E6%8A%BD%E8%B1%A1%E6%96%B9%E6%B3%95&amp;spm=1001.2101.3001.7020" title="抽象方法">
      抽象方法
     </a>
     ，默认修饰符：public abstract。
    </p>
    <p>
     1. 接口中可以声明常量和方法，但是不能声明具体实现。
     <br/>
     2. 接口中声明的方法必须在实现类中进行实现。
     <br/>
     3. 接口中声明的常量和方法默认为 public static final 和 public abstract 类型。
     <br/>
     4. 接口中可以包含 default 和 static 方法的实现，这些方法允许在接口中提供默认实现，而无需实现它们的类进行实现。
     <br/>
     5. 接口不能被实例化，但可以通过引用实现它的类类型来调用其中声明的方法或访问其中的常量。
     <br/>
     6. 接口可以继承一个或多个父接口，继承后可以使用父接口中的方法和常量。
    </p>
    <h3 id="%E6%8E%A5%E5%8F%A3%E4%B8%8E%E7%B1%BB%E7%9A%84%E5%8C%BA%E5%88%AB%EF%BC%9A%C2%A0%C2%A0">
     接口与类的区别：
    </h3>
    <p>
     在面向对象编程中，接口与类是两个重要的概念。虽然它们有一些相似之处，但它们之间也存在一些区别。
    </p>
    <p>
     1.
     <strong>
      继承关系
     </strong>
     ：类可以继承另一个类，也可以实现一个或多个接口。但
     <strong>
      接口不能直接继承类，只能继承接口
     </strong>
     。一个类可以继承一个类，并实现多个接口。
    </p>
    <p>
     2. 相似之处：
     <strong>
      接口与类都可以包含方法和属性。但是接口中的方法只有方法签名，没有方法实现
     </strong>
     ；而类可以包含方法实现。
    </p>
    <p>
     3. 区别：接口与类的最大区别是类有构造函数，而接口无法创建实例。此外，类可以定义变量，但接口只能定义常量，也就是 final 变量。另外，类可以使用 private 和 protected 变量，而接口只有 public 变量。接口定义的方法只能是 public abstract，而类可以定义各种类型的方法。接口继承接口时，可以使用 extends 关键字，类继承类时可以使用 extends 关键词，类实现接口时可以使用 implements 关键词。
    </p>
    <p>
     因此，接口与类之间有一些相似之处，但是它们还是有很多区别。接口提供了一种规范，允许多种不同的实现，而类则提供了一个具体的实现。可以通过继承和实现接口的方式，使得代码更加灵活和可扩展。
    </p>
    <blockquote>
     <p>
      JDK7以前：接口中只能定义抽象方法。
     </p>
     <p>
      JDK8以后：接口中可以定义有方法体的方法。（默认，多态）
     </p>
     <p>
      JDK9之后：接口中可以定义私有方法。
     </p>
    </blockquote>
    <p>
     这些新增特点都是为了方便接口升级。也就是在版本更新的时候新增接口方法。
    </p>
    <h3>
     接口的应用：
    </h3>
    <p>
     <strong>
      1. 多态性：
     </strong>
    </p>
    <p>
     通过接口的实现，实现了不同对象之间的多态性。可以将一个接口的实现传递给一个方法，这个方法可以处理所有实现了这个接口的类的对象。
    </p>
    <p>
     <strong>
      2. 规范：
     </strong>
    </p>
    <p>
     通过接口声明的方法，规范了类的使用方式，使得各个类在实现接口的方法时有一个明确的标准。
    </p>
    <p>
     <strong>
      3. 组合：
     </strong>
    </p>
    <p>
     通过同时实现多个接口，实现了类的组合。这种方式比类的继承更加灵活，可以在不同的情况下选择不同的组合方式。
    </p>
    <p>
     <strong>
      4. 扩展：
     </strong>
    </p>
    <p>
     通过新增接口中的方法，扩展了类的功能，同时不需要对类本身进行修改。使用这种方式可以实现类的开放-关闭原则。
    </p>
    <p>
     <strong>
      5. 解耦（面向接口编程）：
     </strong>
    </p>
    <p>
     接口的使用可以将代码解耦，实现了模块之间的独立性。在接口与实现类之间形成了一个中间层，使得实现类可以独立于接口的具体实现。
    </p>
    <p>
     总之，接口是面向对象编程中非常重要的一种概念，通过使用接口可以提高代码的可读性、可维护性、可扩展性和可复用性。
    </p>
