    <h1 class="title-article" id="articleContentId">
     Information:java: Multiple encodings set for module chunk platf "GBK" will be used by compile
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     Intellij IDEA 在引入代码后，出现编译错误。
    </p>
    <p>
    </p>
    <p>
    </p>
    <pre><code class="language-html hljs"> </code></pre>
    <ol>
     <li>
      <p>
       <code>
        Information:java: Multiple encodings set for module chunk platf "GBK" will be used by compile
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        Error:(221, 70) java: unclosed string literal
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        Error:(221, 84) java: ';' expected
       </code>
      </p>
     </li>
    </ol>
    <p>
     <br/>
     猜想是编码问题。
    </p>
    <p>
     项目的默认编码设置的UTF-8
    </p>
    <p>
     在Settings - Editor - File Encodings 中设置了
    </p>
    <p>
     IDEA  Encoding UTF-8
    </p>
    <p>
     Project Encoding UTF-8
    </p>
    <p>
    </p>
    <p>
     改变每个引入.java文件的编码为UTF-8 但是编译还是会出现以上异常。
    </p>
    <p>
    </p>
    <p>
     因为在复制引入的时候，IDEA 会在.idea/encoding.xml中制定文件的编码。
    </p>
    <p>
     将其中内容删除即可。
    </p>
    <p>
     原：
    </p>
    <p>
    </p>
    <pre><code class="language-html hljs"> </code></pre>
    <ol>
     <li>
      <p>
       <code>
        &lt;?xml version="1.0" encoding="UTF-8"?&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;project version="4"&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;component name="Encoding" defaultCharsetForPropertiesFiles="UTF-8"&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$" charset="UTF-8" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$/src/main/java/com/bestpay/model" charset="GBK" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$/src/main/java/com/bestpay/utils/DateUtils.java" charset="GBK" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$/src/main/java/com/bestpay/utils/HttpUtils.java" charset="GBK" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$/src/main/java/com/bestpay/utils/MD5.java" charset="GBK" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="PROJECT" charset="UTF-8" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;/component&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;/project&gt;
       </code>
      </p>
     </li>
    </ol>
    <p>
     删除后：
    </p>
    <p>
    </p>
    <pre><code class="language-html hljs"> </code></pre>
    <ol>
     <li>
      <p>
       <code>
        &lt;?xml version="1.0" encoding="UTF-8"?&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;project version="4"&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;component name="Encoding" defaultCharsetForPropertiesFiles="UTF-8"&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="file://$PROJECT_DIR$" charset="UTF-8" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;file url="PROJECT" charset="UTF-8" /&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;/component&gt;
       </code>
      </p>
     </li>
     <li>
      <p>
       <code>
        &lt;/project&gt;
       </code>
      </p>
     </li>
    </ol>
    <p>
    </p>
    <p>
     即可编译正常。
    </p>
    <p>
    </p>
    <p>
     IDEA 进行编译代码的时候，特别是新项目 特别容易出现 编码错误，但是 File-Encoding中设置的又没有问题，而且maven 是能打包的，就是用 idea 自带的 编译的时候 就会出现提示 找不到字符，错误开头会提示
    </p>
    <pre class="has"><code>java: Multiple encodings set for module chunk test  

"GBK" will be used by compiler  
</code></pre>
    <p>
     但是我们的
     <strong>
      设置里面 设置的 是 UTF-8 那么显然哪个地方把它设置为 GBK 了
     </strong>
     ；
    </p>
    <p>
     最后解决办法 是按连接中的答案进行操作的：
     <strong>
      检查.idea/encodings.xml是否对某些类指定了charset,删除即可
     </strong>
    </p>
