# 测试各阶段的check List

常用测试checkList，可继续补充

![](https://i-blog.csdnimg.cn/blog_migrate/c66e5e247cd24f2992cb701c0676cf75.png)

### 测试前

|  |  |  |
| --- | --- | --- |
| 迭代测试前CheckList | | |
| 检查时间 |  | |
| 检查人 |  | |
| 序号 | 内容 | 检查结果 |
| 1 | 提前确定测试范围以及提交测试包的时间和方式 |  |
| 2 | 评估工作量，确定测试周期，确定测试重点以及测试策略 |  |
| 3 | 安排测试计划，并邮件发出 |  |
| 4 | 根据测试内容，准备测试环境和测试数据 |  |
| 5 | 跟踪测试包提交时间 |  |

**测试中**

|  |  |  |
| --- | --- | --- |
| 接受测试包CheckList | | |
| 检查时间 |  | |
| 检查人 |  | |
| 序号 | 内容 | 检查结果 |
| 1 | 检查是否有“测试通知”邮件 |  |
| 2 | 检查测试包路径是否正确 |  |
| 3 | 检查测试包命名是否符合规范 |  |
| 4 | 检查版本号是否正确 |  |
| 5 | 检查更新说明或者安装说明是否齐全 |  |
| 6 | 检查新增功能或者修复问题清单是否齐全 |  |
| 7 | 检查是否有自测报告 |  |
| 8 | 检查日志文件或者调试文件是否清除 |  |
| 9 | 检查测试包的配置文件是否遗漏或者正确 |  |

**测试后**

|  |  |  |
| --- | --- | --- |
| 测试发布CheckList | | |
| 检查时间 |  | |
| 检查人 |  | |
| 序号 | 内容 | 检查结果 |
| 1 | 检查发布包是否经过测试的测试包 |  |
| 2 | 检查发布包命名是否符合规范 |  |
| 3 | 检查日志文件或者调试文件是否清除 |  |
| 4 | 检查发布包的配置文件是否遗漏或者正确 |  |
| 5 | 检查版本号是否正确 |  |
| 6 | 检查新增功能或者修复问题清单是否齐全 |  |
| 7 | 检查操作手册是否齐全 |  |
| 8 | 检查更新说明或者安装说明是否齐全 |  |
| 9 | 检查是否准备了一份遗留bug确认清单 |  |
| 10 | 检查发布地址是否正确 |  |
| 11 | 检查是否有“发布通知”邮件 |  |
| 12 | 检查“发布通知”邮件内容是否包含：测试通过的内容、发布地址、测试说明、测试范围、测试环境、遗留bug清单（或者附件） | |
| 13 | 检查是否更新发布记录列表 |  |
