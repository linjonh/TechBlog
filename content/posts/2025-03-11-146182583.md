     1 双端栈/队列
    </h3>
    <p>
     （1）一般形态的讨论：经过大量编程训练后，不禁会有疑问：
     <br/>
     【Q】为何ArrayList，LinkedList，Stack、Queue这四种基本形态以及它们的变种被统称为“线性表”呢？涉及到“双端”话题时，又应该如何正确理解呢？
    </p>
    <p>
     【A】很多文献中总是给出大量的理论化论述，但是我认为可以简单的归纳如下：
     <br/>
     （1）一个线性表两头都能进出。（注意红色箭头代表的数据进出方向。）
     <br/>
     （2）可灵活设定元素增长方向：两头向中间 or 中间向两头；当然，都要处理越界/碰头问题。
     <br/>
     （3）人们也习惯将带回绕的线性表当做一个环来看待。（老生常谈的话题，后续看案例。）

     2 栈与队列的相互转化
    </h3>
    <p>
     这是一对经典的面试问题，在Leecode上有对应练习，大家可以试试看。

    <h4>
     <a id="21__17">
     </a>
     2-1 栈转化成队列
    </h4>
    <p>
     【Q】给定两个栈，你能否组装成一个队列？
     <br/>
     【A】设置一个栈作入队用，另一个栈作出队用：
     <br/>
     （1）入队直接压到输入栈。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/17486b555ce844929a9d5fb629c832ee.png">
      <br/>
      （2）出队时：
      <br/>
      &lt;1&gt; 若输出栈为空则将输入栈的全部数据依次弹出并压入输出栈。
      <br/>
      &lt;2&gt; 此时输出栈从栈顶往栈底的顺序就是队列从队首往队尾的顺序。
      <br/>
      &lt;3&gt; 若输出栈不为空，直接弹出即可。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bb8479d3f10e4a2db6712aff22b4a0aa.png"/>
     </img>
    </p>
    <h4>
     <a id="22__28">
     </a>
     2-2 队列转化成栈
    </h4>
    <p>
     【Q】给你两个队列，你能否组装成一个栈？
     <br/>
     【A1】常规解法
     <br/>
     （1）queue1用于存储栈内的元素，queue2作为入栈操作的辅助队列。
     <br/>
     （2）入栈操作时，首先将元素入队到queue2，然后将queue1的全部元素依次出队并入队到queue2，此时queue2的前端的元素即为新入栈的元素，再将queue1和queue2互换，则queue1的元素即为栈内的元素，queue1的前端和后端分别对应栈顶和栈底。
     <br/>
     （3）由于每次入栈操作都确保queue1的前端元素为栈顶元素，因此出栈操作和获得栈顶元素操作都可以简单实现。出栈操作只需要移除queue1的前端元素并返回即可，获得栈顶元素操作只需要获得queue1的前端元素并返回即可（不移除元素）。
     <br/>
     （4）由于queue1用于存储栈内的元素，判断栈是否为空时，只需要判断queue1是否为空即可。
     <br/>

    <h4>
     <a id="32_MapReducekv_48">
     </a>
     3-2 MapReduce中的缓冲区对&lt;k,v&gt;键值对排序
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3efda31d00504265a950bf5be75beb28.png"/>
     <br/>
     【Q1】设想现在给你一个Byte数组（线性数据结构）如何把&lt;k1，v1&gt; ~ &lt;kn，vn&gt;n个键值对数据存入其中？
     <br/>
     【Q2】此时若直接排序会导致频繁的内存腾挪（联想动态数组时我们实现的resize操作？），如何避免？
    </p>
    <p>
     【A】在实际工程设计中，将数据转化为两部分的组合：
     <br/>
     （1） 固定长度的&lt;k，v&gt;键值对。
     <br/>
     （2）非固定长度meta元数据段。
     <br/>
     （3）&lt;k，v&gt;键值对还存储了meta的起始位置。
    </p>
    <p>
     为了提高buffer的利用率，考虑使用环形buffer，那么：
     <br/>
     （1）index = 0处就相当于一条分割线，称为赤道（equator）。
     <br/>
     （2）&lt;k，v&gt;和meta都以equator为起点，分别朝两个方向存放数据。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7c0596cd2fd449d4b70e0382e93d1e08.png"/>
     <br/>
     盗一张图，更加形象的表达
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1122b366100b4a3698113456f07d4e6f.png"/>
     <br/>
     【效果】
     <br/>
     （1）排序时仅需要调整&lt;k,v&gt;的顺序即可，极大减少内存腾挪。
     <br/>
     （2）排序完成后，外部读取数据时，采用间接寻址能够以正确的顺序获取到完整的数据集合。
    </p>
