    <h2>
     <a id="_1">
     </a>
     问题
    </h2>
    <p>
     使用BatchAcknowledgingMessageListener 批量消费Kafka消息，成功则手动提交offset，失败则重试。消费成功的情况下没有问题，但消费失败情况下，调用nack方法重试时则报异常。
    </p>
    <h3>
     <a id="_3">
     </a>
     示例
    </h3>
    <pre><code class="prism language-java">
  <span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">BatchCustomMessageListener</span> <span class="token keyword">implements</span> <span class="token class-name">BatchAcknowledgingMessageListener</span> <span class="token punctuation">{<!-- --></span>

    <span class="token keyword">private</span> <span class="token class-name">MessageHandler</span> messageHandler<span class="token punctuation">;</span>

    <span class="token keyword">public</span> <span class="token class-name">BatchCustomMessageListener</span><span class="token punctuation">(</span><span class="token class-name">MessageHandler</span> messageHandler<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">this</span><span class="token punctuation">.</span>messageHandler <span class="token operator">=</span> messageHandler<span class="token punctuation">;</span>
    <span class="token punctuation">}</span>

    <span class="token annotation punctuation">@Override</span>
    <span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">onMessage</span><span class="token punctuation">(</span><span class="token class-name">List</span> data<span class="token punctuation">,</span> <span class="token class-name">Acknowledgment</span> acknowledgment<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">try</span> <span class="token punctuation">{<!-- --></span>
            messageHandler<span class="token punctuation">.</span><span class="token function">handle</span><span class="token punctuation">(</span>data<span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 处理多条消息</span>
            acknowledgment<span class="token punctuation">.</span><span class="token function">acknowledge</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// 成功处理后提交偏移量</span>

    <p>
     上边的代码乍一看没啥问题，编译，启动也都没报错。但是在执行nack的时候进到了Acknowledgment接口默认nack(sleep) 方法里边，并抛出异常。
    </p>
    <pre><code class="prism language-bash">nack<span class="token punctuation">(</span>sleep<span class="token punctuation">)</span> is not supported by this Acknowledgment
</code></pre>
    <h3>
     <a id="_33">
     </a>
     异常
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f42e6f8ce7294e7c84b712017981aceb.png">
      <br/>
      <img alt="![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/8bbf4f37af134374bf1f4e34c2687dbe.png" src="https://i-blog.csdnimg.cn/direct/00f673c87faa4769941bffdec50ad7c8.png"/>
     </img>
    </p>
    <h2>
     <a id="_36">
     </a>
     原因
    </h2>
    <p>
     Acknowledgment接口有两个nack方法：
     <strong>
      nack(long sleep)
     </strong>
     和
     <strong>
      nack(int index, long sleep)
     </strong>
     ， 以及两个实现类
     <strong>
      ConsumerAcknowledgment
     </strong>
     和
     <strong>
      ConsumerBatchAcknowledgment
     </strong>
     。ConsumerAcknowledgment仅实现了nack(long sleep)，而ConsumerBatchAcknowledgment仅实现了nack(int index, long sleep)。
    </p>
    <h3>
     <a id="nack_38">
     </a>
     nack方法
    </h3>
    <p>
     <strong>
      注意：调用nack方法后，将会在指定sleep时间后，重新消费消息。在sleep期间内，不会消费新消息。
     </strong>
    </p>
    <h3>
     <a id="Acknowledgment_40">
     </a>
     Acknowledgment接口
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bcf4bdae87af433fb882deea92294896.png"/>
    </p>
    <h3>
     <a id="ConsumerAcknowledgment_42">
     </a>
     实现类：ConsumerAcknowledgment
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0cb8b6af40cd4d1781690f6e1f2aeb1a.png"/>
    </p>
    <h3>
     <a id="ConsumerBatchAcknowledgment_45">
     </a>
     实现类：ConsumerBatchAcknowledgment
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/78f4d1c447a64c1bb4ec796654e860cd.png"/>
    </p>
    <p>
     这样的设计也很好理解。。
     <br/>
     当BatchAcknowledgingMessageListener批量消费消息时， 使用的是ConsumerBatchAcknowledgment，重试时需要告诉ConsumerBatchAcknowledgment要从这批量消息中的哪条开始重试消费，即要指定index值。我的例子中调用的是nack(long sleep)，没有指定index，所以进到了默认方法里，抛了异常。
    </p>
    <p>
     而使用AcknowledgingMessageListener消费单条消息时，使用的是ConsumerAcknowledgment，重试时它知道重试当前的消息，因为就这一条，所以只需要指定重试时间就可以了。
    </p>
    <p>
     <strong>
      也就是说批量消费时，重试要调用nack(int index, long sleep)，单条消费时，重试要调用nack(long sleep)，二者不搭配，就会抛不支持该方法的异常。
     </strong>
    </p>
