    <h5>
     <strong>
      办法： 手动添加设备支持文件（暂时解决方式）
     </strong>
    </h5>
    <p>
     如果您无法立即升级 Xcode，也可以通过下载设备支持文件来暂时解决问题。
    </p>
    <ol>
     <li>
      <p>
       <strong>
        检查当前设备的 iOS 版本：
       </strong>
      </p>
      <ul>
       <li>
        连接设备到 Mac，打开 Xcode 查看提示的 iOS 版本。
       </li>
       <li>
        例如，如果您的设备是 iOS 18，但 Xcode 不支持，说明缺少
        <code>
         Device Support
        </code>
        文件。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        下载对应的设备支持文件：
       </strong>
      </p>
      <ul>
       <li>
        访问以下 GitHub 仓库，查找对应的 iOS Device Support 文件：
        <br/>
        <a href="https://github.com/iGhibli/iOS-DeviceSupport" title="GitHub - iGhibli/iOS-DeviceSupport: This repository holds the device support files for the iOS, and I will update it regularly.">
         GitHub - iGhibli/iOS-DeviceSupport: This repository holds the device support files for the iOS, and I will update it regularly.
        </a>
       </li>
       <li>
        下载与设备 iOS 版本对应的文件夹，例如
        <code>
         18.x
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        拷贝设备支持文件到 Xcode：
       </strong>
      </p>
      <ul>
       <li>
        打开 Finder，按以下路径查找 Xcode 的
        <code>
         DeviceSupport
        </code>
        文件夹：
        <pre></pre>
        <p>
         <code>
          /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/DeviceSupport/
         </code>
        </p>
       </li>
       <li>
        将下载的设备支持文件
        <strong>
         解压缩
        </strong>
        ，然后拷贝到
        <code>
         DeviceSupport
        </code>
        文件夹中。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        重新启动 Xcode 并连接设备：
       </strong>
      </p>
      <ul>
       <li>
        现在 Xcode 应该可以识别新版本的设备。
       </li>
      </ul>
     </li>
    </ol>
