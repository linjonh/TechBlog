    <h2>
     <a id="_9">
     </a>
     一、分布式版本控制系统的概念
    </h2>
    <ul>
     <li>
      <p>
       本地操作：所有操作（工作区、暂存区、版本库等）都是在本地完成，即你的笔记本或计算机上。Git作为一个分布式版本控制系统，意味着每个用户的电脑上都有一个完整的版本库副本。
      </p>
     </li>
     <li>
      <p>
       协作机制：多用户协作时，各成员可以在自己的电脑上修改文件，并通过推送和拉取操作与其他成员交换更改。这使得即使没有互联网连接也可以继续工作，因为版本库就在本地。
      </p>
     </li>
     <li>
      <p>
       安全性：由于每个参与者的电脑里都有完整的版本库，所以即使某台电脑出现问题，其他任何一台都可以复制整个项目的历史记录。
      </p>
     </li>
    </ul>
    <p>
     实际使用中，我们会使用到一种中央服务器，这是为了方便团队成员之间的交流，通常会设立一台充当“中央服务器”的电脑。但这台服务器仅用于促进大家的修改交换，而非必要的工作依赖。
    </p>
    <p>
     这就要提到一个“远程仓库”的概念了：
    </p>
    <ul>
     <li>
      分布方式：从一台原始机器克隆出多个副本到不同的机器上。每台机器上的版本库都是平等的，无主次之分。
     </li>
     <li>
      实际应用：一般会选择一台稳定的服务器作为远程仓库，供团队成员克隆、推送和拉取代码。例如，GitHub提供免费的Git仓库托管服务，而国内则常用码云Gitee。
     </li>
    </ul>
    <blockquote>
     <p>
      大家可以自行选择喜欢的代码托管平台来尝试一下创建远程仓库
     </p>
    </blockquote>
    <h2>
     <a id="_26">
     </a>
     二、克隆远程仓库
    </h2>
    <p>
     使用git clone命令加上远程仓库链接来下载仓库到本地。可以采用SSH协议或HTTPS协议进行传输。
     <strong>
      SSH更安全但需要配置公钥；HTTPS更简单直接但每次推送需输入密码。
     </strong>
    </p>
    <blockquote>
     <p>
      第一步
     </p>
    </blockquote>
    <p>
     如果选择SSH协议，需先生成SSH密钥对并添加到远程仓库的服务端。步骤包括检查是否存在已有密钥、生成新密钥、将公钥添加到远程仓库设置中
    </p>
    <pre><code class="prism language-bash">ssh-keygen <span class="token parameter variable">-t</span> rsa <span class="token parameter variable">-C</span> <span class="token string">"Your Email"</span>
</code></pre>
    <blockquote>
     <p>
      第二步
     </p>
    </blockquote>
    <p>
     顺利的话，可以在⽤⼾主⽬录⾥找到 .ssh ⽬录，⾥⾯有 id_rsa 和 id_rsa.pub 两个⽂件，这两 个就是SSH Key的秘钥对，
     <strong>
      id_rsa 是私钥，不能泄露出去， id_rsa.pub 是公钥
     </strong>
     ，可以放⼼地告 诉任何⼈。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/eb0cad23f3bc470baaa4528bc30e3534.png"/>
    </p>
    <p>
     接下来我们就可以将我们的公钥放到远程仓库里面进去，点击确认后，需要认证你的身份，这个时候你只需要输入就可以了，待一切完成后，你就可以开始 clone 了
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8da0ea615416458b922ce0fa5b3366e4.png"/>
    </p>
    <blockquote>
     <p>
      注意：SSH协议每次推送是不用输入密码的，但如果是HTTPS协议，每次推送都必须输入密码
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4056e064e6b24aebb534d9efaa582b5d.png"/>
    </p>
    <h2>
     <a id="_51">
     </a>
     三、多用户协作与公钥管理
    </h2>
    <p>
     如果有多个人员协作开发，GitHub/Gitee允许添加多个公钥。只需将每个人的电脑上的Key都添加到GitHub/Gitee，这样就可以在每台电脑上往GitHub/Gitee上提交推送了。
    </p>
    <p>
     克隆后的本地与远程分支对应
    </p>
    <ol>
     <li>
      <strong>
       自动对应：
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      当我们从远程仓库克隆后，实际上Git会自动把本地的master分支和远程的master分支对应起来。
     </li>
     <li>
      远程仓库的默认名称是origin。
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2a4d577fd9ae4dd5bb2207f5411c092e.png"/>
    </p>
    <ol start="2">
     <li>
      <strong>
       查看远程库消息：
      </strong>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">git</span> remote
<span class="token function">git</span> remote <span class="token parameter variable">-v</span>
</code></pre>
    <ol start="3">
     <li>
      <strong>
       向远程仓库推送：
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      前提：在本地仓库配置了和远端仓库一样的用户名和邮箱
     </li>
     <li>
      将本地仓库的内容推送至远端仓库：git push
     </li>
     <li>
      详细命令格式：git push &lt;远程主机名&gt; &lt;本地分支名&gt;:&lt;远程分支名&gt;
     </li>
     <li>
      如果本地分支名与远程分支名相同，则可以省略冒号：git push &lt;远程主机名&gt; &lt;本地分支名&gt;
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d5a6d17138cd4a9dae8e0289ea201ef6.png"/>
    </p>
    <ol start="4">
     <li>
      <strong>
       拉取远程仓库更新：
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      如果远程仓库是要领先于本地仓库⼀个版本，为了使本地仓库保持最新的版本，需要拉取下远端代码，并合并到本地
     </li>
     <li>
      从远程获取代码并合并本地的版本：git pull
     </li>
     <li>
      详细命令格式：git pull &lt;远程主机名&gt; &lt;远程分支名&gt;:&lt;本地分支名&gt;
     </li>
     <li>
      如果远程分支名与本地分支名相同，则可以省略冒号：git pull &lt;远程主机名&gt; &lt;远程分支名&gt;
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b1bde7ad00cf4996af1e8e45e8cbf5c0.png"/>
    </p>
    <h2>
     <a id="Git_86">
     </a>
     四、配置Git忽略特殊文件
    </h2>
    <ol>
     <li>
      创建 .gitignore 文件：
     </li>
    </ol>
    <p>
     在工作区根目录下创建一个特殊的 .gitignore 文件，并列出不想或不应该提交到远端的文件模式。示例内容：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1c28b0ae1f9e4e03a7ea281028d9aa7c.png"/>
    </p>
    <ol start="2">
     <li>
      处理.gitignore规则中的例外情况
     </li>
    </ol>
    <pre><code class="prism language-vim"># 排除所有.开头的隐藏文件:
.*
# 不排除.gitignore
!.gitignore
</code></pre>
    <blockquote>
     <p>
      如果已经忽略了某个类型的文件，但是又想追踪管理某个特定的该类型文件，该怎么办？、
     </p>
    </blockquote>
    <p>
     方法一：使用
     <strong>
      git add -f file_name
     </strong>
     强制添加（推荐）
     <br/>
     方法二：在
     <strong>
      .gitignore
     </strong>
     中添加配置
     <strong>
      !file_name
     </strong>
     即可让Git不忽略该文件
    </p>
    <ul>
     <li>
      可以使用
      <strong>
       git check-ignore -v file_name
      </strong>
      来检查相关配置
     </li>
    </ul>
    <h2>
     <a id="_110">
     </a>
     五、给命令配置别名
    </h2>
    <p>
     使用
     <strong>
      git config --global alias.&lt; alias-name &gt; &lt; command &gt;
     </strong>
     为常用命令设置别名。
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> alias.st status
<span class="token function">git</span> config <span class="token parameter variable">--global</span> alias.last <span class="token string">'log -1'</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/afb0f12a62c74bde9d8bbe8e453f176c.png"/>
    </p>
    <blockquote>
     <p>
      目前所有的命令建议自己手动完成，尽快适应Git的操作，
      <strong>
       建议在熟悉基本命令后再考虑使用命令别名简化工作流程。
      </strong>
     </p>
    </blockquote>
