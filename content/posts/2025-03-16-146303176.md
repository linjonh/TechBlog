     animes 和 css对比
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     <strong>
      Anime.js
     </strong>
     并不是纯粹的 CSS 动画库，而是一个基于 JavaScript 的动画库。它可以通过操作 DOM 元素的属性（如
     <code>
      style
     </code>
     、
     <code>
      transform
     </code>
     、
     <code>
      opacity
     </code>
     等）来实现动画效果。虽然 Anime.js 的某些功能与 CSS 动画类似，但它的工作原理和功能范围与 CSS 动画有显著区别。
    </p>
    <p>
     以下是对 Anime.js 和 CSS 动画的详细对比，以及为什么有人可能会误认为 Anime.js 就是 CSS 动画：
    </p>
    <hr/>
    <h3>
     <strong>
      1. Anime.js 的特点
     </strong>
    </h3>
    <h4>
     <strong>
      1.1 基于 JavaScript
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 是一个 JavaScript 库，通过操作 DOM 元素的属性来实现动画。
      </p>
     </li>
     <li>
      <p>
       它可以直接操作元素的
       <code>
        style
       </code>
       属性，或者通过 JavaScript 控制 SVG、CSS 变量等。
      </p>
     </li>
    </ul>
    <h4>
     <strong>
      1.2 强大的动画控制
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 提供了丰富的 API，支持复杂的动画序列、时间轴、回调函数等。
      </p>
     </li>
     <li>
      <p>
       示例：
      </p>
      <p>
       javascript
      </p>
      <p>
       复制
      </p>
      <pre>anime({
  targets: '.box',
  translateX: 250,
  rotate: '1turn',
  duration: 1000,
  easing: 'easeInOutSine',
});</pre>
     </li>
    </ul>
    <h4>
     <strong>
      1.3 跨浏览器兼容性
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 会自动处理浏览器兼容性问题，确保动画在不同浏览器中表现一致。
      </p>
     </li>
    </ul>
    <h4>
     <strong>
      1.4 支持多种目标
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 不仅可以操作 DOM 元素，还可以操作 JavaScript 对象、CSS 变量、SVG 属性等。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      2. CSS 动画的特点
     </strong>
    </h3>
    <h4>
     <strong>
      2.1 基于 CSS
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       CSS 动画是通过
       <code>
        @keyframes
       </code>
       和
       <code>
        animation
       </code>
       属性实现的，完全由浏览器渲染引擎处理。
      </p>
     </li>
     <li>
      <p>
       示例：
      </p>
      <p>
       css
      </p>
      <p>
       复制
      </p>
      <pre>@keyframes slide {
  from { transform: translateX(0); }
  to { transform: translateX(250px); }
}
.box {
  animation: slide 1s ease-in-out;
}</pre>
     </li>
    </ul>
    <h4>
     <strong>
      2.2 性能优化
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       CSS 动画由浏览器原生支持，通常性能较好，尤其是在涉及硬件加速的属性（如
       <code>
        transform
       </code>
       、
       <code>
        opacity
       </code>
       ）时。
      </p>
     </li>
    </ul>
    <h4>
     <strong>
      2.3 简单易用
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       对于简单的动画效果，CSS 动画的代码更简洁，易于维护。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      3. 为什么有人会认为 Anime.js 就是 CSS 动画？
     </strong>
    </h3>
    <h4>
     <strong>
      3.1 相似的动画效果
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 和 CSS 动画都可以实现类似的视觉效果（如平移、旋转、缩放等），因此容易混淆。
      </p>
     </li>
    </ul>
    <h4>
     <strong>
      3.2 操作 CSS 属性
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 可以直接操作元素的 CSS 属性（如
       <code>
        style.transform
       </code>
       、
       <code>
        style.opacity
       </code>
       等），这与 CSS 动画的操作对象相同。
      </p>
     </li>
    </ul>
    <h4>
     <strong>
      3.3 部分功能重叠
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       Anime.js 支持 CSS 动画的缓动函数（如
       <code>
        ease-in-out
       </code>
       、
       <code>
        linear
       </code>
       等），并且可以通过
       <code>
        translateX
       </code>
       、
       <code>
        rotate
       </code>
       等属性实现类似 CSS 动画的效果。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      4. Anime.js 与 CSS 动画的区别
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        Anime.js
       </th>
       <th>
        CSS 动画
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         实现方式
        </strong>
       </td>
       <td>
        基于 JavaScript
       </td>
       <td>
        基于 CSS
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         控制能力
        </strong>
       </td>
       <td>
        支持复杂动画序列、时间轴、回调函数
       </td>
       <td>
        仅支持简单的
        <code>
         @keyframes
        </code>
        动画
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         性能
        </strong>
       </td>
       <td>
        依赖 JavaScript 执行，性能稍低
       </td>
       <td>
        浏览器原生支持，性能较高
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         兼容性
        </strong>
       </td>
       <td>
        自动处理浏览器兼容性问题
       </td>
       <td>
        需要手动处理浏览器前缀
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         目标对象
        </strong>
       </td>
       <td>
        DOM 元素、JavaScript 对象、SVG 等
       </td>
       <td>
        仅支持 DOM 元素
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         动态控制
        </strong>
       </td>
       <td>
        支持动态修改动画参数
       </td>
       <td>
        不支持动态修改
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <strong>
      5. 何时使用 Anime.js？
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        复杂动画
       </strong>
       ：需要实现复杂的动画序列或时间轴。
      </p>
     </li>
     <li>
      <p>
       <strong>
        动态控制
       </strong>
       ：需要在运行时动态修改动画参数。
      </p>
     </li>
     <li>
      <p>
       <strong>
        跨浏览器兼容性
       </strong>
       ：需要确保动画在所有浏览器中表现一致。
      </p>
     </li>
     <li>
      <p>
       <strong>
        非 CSS 目标
       </strong>
       ：需要操作 JavaScript 对象或 SVG 属性。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      6. 何时使用 CSS 动画？
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        简单动画
       </strong>
       ：只需要实现简单的视觉效果（如 hover 效果）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        性能优化
       </strong>
       ：需要最大化动画性能，尤其是涉及硬件加速的属性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        静态动画
       </strong>
       ：动画参数在运行时不需要修改。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      总结
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       Anime.js 是一个基于 JavaScript 的动画库，虽然它可以操作 CSS 属性并实现类似 CSS 动画的效果，但它的功能更强大，适用于更复杂的场景。
      </p>
     </li>
     <li>
      <p>
       CSS 动画则更适合简单的、静态的动画效果，并且性能更好。
      </p>
     </li>
    </ul>
