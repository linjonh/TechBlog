      <h2>
       <a id="font_colorDarkOrchid_font_9">
       </a>
       <font color="DarkOrchid">
        一：🔥 原理
       </font>
      </h2>
      <p>
       🔬
       <strong>
        <font color="#41acs">
         原理就是让你需要访问的内网可以被其他内网访问到， 其实就是让内网经过一个公网服务器的转发，使得能够被访问。
        </font>
       </strong>
      </p>
      <p>
       <mark>
        这里我们需要用到 frp：
       </mark>
      </p>
      <p>
       <strong>
        <code>
         FRP（Fast Reverse Proxy）
        </code>
       </strong>
       是一款开源的反向代理工具，主要用于实现内网穿透。它通过在公网服务器和内网设备之间建立一个代理通道，使得处于内网中的设备能够被外部网络访问，同时也允许内网设备主动与外部网络通信。
      </p>
      <p>
       <strong>
        实现步骤
       </strong>
      </p>
      <ol>
       <li>
        <p>
         <strong>
          <font color="#FF4500">
           部署公网服务器端
          </font>
         </strong>
        </p>
        <ul>
         <li>
          在公网服务器上下载并安装
          <strong>
           <code>
            FRP
           </code>
          </strong>
          文件。
         </li>
         <li>
          配置
          <strong>
           <code>
            FRP
           </code>
          </strong>
          的服务器端配置文件（
          <code>
           frps.toml
          </code>
          ），设置监听端口、认证信息等参数。
         </li>
         <li>
          启动
          <strong>
           <code>
            FRP
           </code>
          </strong>
          服务器端程序，使其开始监听来自内网设备的连接请求。
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          <font color="#FF4500">
           配置内网客户端
          </font>
         </strong>
        </p>
        <ul>
         <li>
          在需要被访问的内网设备上下载并安装
          <strong>
           <code>
            FRP
           </code>
          </strong>
          文件。
         </li>
         <li>
          配置
          <strong>
           <code>
            FRP
           </code>
          </strong>
          的客户端配置文件（
          <code>
           frpc.toml
          </code>
          ），指定内网服务的 IP 地址和端口，并将其映射到公网服务器的对应端口。
         </li>
         <li>
          启动
          <strong>
           <code>
            FRP
           </code>
          </strong>
          客户端程序，使其连接到公网服务器，并建立隧道。
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          <font color="#FF4500">
           数据转发与通信
          </font>
         </strong>
        </p>
        <ul>
         <li>
          当外部用户访问公网服务器的指定端口时，
          <strong>
           <code>
            FRP
           </code>
          </strong>
          服务器会将请求转发到内网设备的对应 IP 和端口。
         </li>
         <li>
          同样，内网设备发送的数据也会通过
          <strong>
           <code>
            FRP
           </code>
          </strong>
          服务器转发到外部网络，从而实现双向通信。
         </li>
        </ul>
       </li>
      </ol>

           <code>

           ： 这是重定向操作， 用于将命令的标准输出（stdout） 和标准错误（stderr） 都重定向到

            /dev/null

           <code>
            /dev/null
           </code>
           是一个特殊的设备文件， 向它写入的内容都会被丢弃， 读取它则会立即返回文件结束。 因此， 这个操作的作用是忽略命令的所有输出信息。
          </font>
         </strong>
        </li>
       </ul>
      </blockquote>
