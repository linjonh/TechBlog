    <p>
     <strong>
      懒加载（Lazy Loading）
     </strong>
     是一种优化网页性能的技术，主要用于延迟加载非关键资源（如图片、视频、脚本等），直到它们真正需要被使用时才加载。懒加载可以显著减少页面初始加载时间，降低带宽消耗，并提升用户体验。
    </p>
    <hr/>
    <h3>
     <a id="1__5">
     </a>
     1. 懒加载的核心思想
    </h3>
    <p>
     懒加载的核心思想是
     <strong>
      按需加载
     </strong>
     。具体来说：
    </p>
    <ul>
     <li>
      <strong>
       初始加载
      </strong>
      ：只加载当前视口（Viewport）内的资源，或者只加载用户可见的内容。
     </li>
     <li>
      <strong>
       延迟加载
      </strong>
      ：当用户滚动页面或资源进入视口时，再动态加载这些资源。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2__14">
     </a>
     2. 懒加载的应用场景
    </h3>
    <ol>
     <li>
      <strong>
       图片懒加载
      </strong>
      ：延迟加载页面中的图片，直到图片进入视口。
     </li>
     <li>
      <strong>
       视频懒加载
      </strong>
      ：延迟加载视频资源，直到用户点击播放按钮。
     </li>
     <li>
      <strong>
       脚本懒加载
      </strong>
      ：延迟加载非关键脚本，直到页面主要内容加载完成。
     </li>
     <li>
      <strong>
       分页加载
      </strong>
      ：在长列表或分页内容中，只加载当前可见的部分，滚动时再加载更多内容。
     </li>
    </ol>
    <hr/>

    <h5>
     <a id="_86">
     </a>
     代码解析
    </h5>
    <ol>
     <li>
      <strong>
       <code>
        data-src
       </code>
       属性
      </strong>
      ：将图片的真实 URL 存储在
      <code>
       data-src
      </code>
      属性中，而不是直接放在
      <code>
       src
      </code>
      属性里。
     </li>
     <li>
      <strong>
       <code>
        IntersectionObserver
       </code>
      </strong>
      ：
      <ul>
       <li>
        监听每个图片是否进入视口。
       </li>
       <li>
        当图片进入视口时，将
        <code>
         data-src
        </code>
        的值赋给
        <code>
         src
        </code>
        属性，触发图片加载。
       </li>
       <li>
        加载完成后，停止观察该图片。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       性能优化
      </strong>
      ：通过
      <code>
       rootMargin
      </code>
      和
      <code>
       threshold
      </code>
      参数，可以控制图片进入视口的触发条件。
     </li>
    </ol>

    <h3>
     <a id="5__167">
     </a>
     5. 懒加载的优缺点
    </h3>
    <h4>
     <a id="_169">
     </a>
     优点
    </h4>
    <ul>
     <li>
      <strong>
       减少初始加载时间
      </strong>
      ：只加载用户可见的内容，提升页面加载速度。
     </li>
     <li>
      <strong>
       节省带宽
      </strong>
      ：减少不必要的资源加载，降低服务器压力。
     </li>
     <li>
      <strong>
       提升用户体验
      </strong>
      ：避免页面卡顿，提供更流畅的交互体验。
     </li>
    </ul>
    <h4>
     <a id="_174">
     </a>
     缺点
    </h4>
    <ul>
     <li>
      <strong>
       实现复杂度
      </strong>
      ：需要额外的代码逻辑来实现懒加载。
     </li>
     <li>
      <strong>
       SEO 影响
      </strong>
      ：如果搜索引擎爬虫无法正确解析懒加载的内容，可能会影响 SEO。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="6__180">
     </a>
     6. 总结
    </h3>
    <p>
     懒加载是一种非常实用的性能优化技术，特别适用于包含大量图片或长列表的页面。通过
     <code>
      IntersectionObserver
     </code>
     或滚动事件监听，可以轻松实现懒加载功能。结合占位符、预加载和错误处理，可以进一步提升用户体验。
    </p>
    <p>
     在实际项目中，可以根据需求选择合适的懒加载方案，并结合其他优化技术（如防抖、节流）来达到最佳性能。
    </p>
