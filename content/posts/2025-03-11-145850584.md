     一.Git 初识
    </h3>
    <h4>
     <a id="11__26">
     </a>
     1.1 提出问题
    </h4>
    <p>
     不知道你工作或学习时，有没有遇到这样的情况：我们在编写各种文档时，为了防止文档丢失，更改失误，失误后能恢复到原来的版本，不得不复制出⼀个副本，比如：
     <br/>
     “报告-v1”
     <br/>
     “报告-v2”
     <br/>
     “报告-v3”
     <br/>
     “报告-确定版”
     <br/>
     “报告-最终版”
     <br/>
     “报告-究极进化版”
     <br/>
     …
    </p>
    <blockquote>
     <p>
      每个版本有各自的内容，但最终会只有⼀份报告需要被我们使用。但在此之前的工作都需要这些不同版本的报告，于是每次都是复制粘贴副本，产出的文件就越来越多，文件多不是问题，问题是：随着版本数量的不断增多，你还记得这些版本各自都是修改了什么吗？
      <br/>
      文档如此，我们写的项目代码，也是存在这个问题的！！
     </p>
    </blockquote>
    <h4>
     <a id="11__40">
     </a>
     1.1 如何解决–版本控制器
    </h4>
    <blockquote>
     <p>
      为了能够更方便我们管理这些不同版本的文件，便有了版本控制器。所谓的版本控制器，就是能让你了解到⼀个文件的历史，以及它的发展过程的系统。通俗的讲就是⼀个可以记录工程的每⼀次改动和版本迭代的⼀个管理系统，同时也方便多人协同作业。
     </p>
    </blockquote>
    <p>
     目前最主流的版本控制器就是Git。Git可以控制电脑上所有格式的文件，例如doc、excel、dwg、dgn、rvt等等。对于我们开发人员来说，Git最重要的就是可以帮助我们管理软件开发项目中的源代码文件！
    </p>
    <ul>
     <li>
      注意事项
     </li>
    </ul>
    <p>
     还需要再明确⼀点，所有的版本控制系统，Git也不例外，其实只能跟踪文本文件的改动，比如TXT文件，网页，所有的程序代码等等。版本控制系统可以告诉你每次的改动，比如在第5行加了⼀个单词“Linux”，在第8行删了⼀个单词“Windows”。而图篇、视频这些二进制文件，虽然也能由版本控制系统管理，但没法跟踪文件的变化，只能把二进制文件每次改动串起来，也就是只知道图篇从100KB改成了120KB，但到底改了啥，版本控制系统不知道，也没法知道。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/6f79742bc3364efabb652aece4a8d185.png"/>
    </p>

    <h4>
     <a id="32_Git_134">
     </a>
     3.2. 配置Git
    </h4>
    <p>
     当安装Git后首先要做的事情是设置你的用户名称和e-mail地址，这是非常重要的。配置命令为：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> user.name <span class="token string">"Your Name"</span>   
<span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> user.email <span class="token string">"email@example.com"</span>  
<span class="token comment"># 把 Your Name 改成你的昵称</span>
<span class="token comment"># 把 email@example.com 改成邮箱的格式，只要格式正确即可。</span>
</code></pre>
    <p>
     其中–global 是⼀个可选项。如果使用哪个了该选项，表示这台机器上所有的Git仓库都会使用这个配置。如果你希望在不同仓库中使用不同的name 或e-mail ，可以不要–global 选项，注意的是，执行命令时必须要在仓库里。
    </p>
    <p>
     查看配置命令为：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">-l</span>
</code></pre>
    <p>
     删除对应的配置命令为：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> <span class="token parameter variable">--unset</span> user.name
 <span class="token function">git</span> config <span class="token punctuation">[</span>--global<span class="token punctuation">]</span> <span class="token parameter variable">--unset</span> user.email
</code></pre>

    <h3>
     <a id="__158">
     </a>
     四. 认识工作区、暂存区、版本库
    </h3>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/166b7e1a892f49aea6025763cab40814.png"/>
    </p>
    <ul>
     <li>
      工作区:是在电脑上你要写代码或文件的目录。
     </li>
     <li>
      暂存区:英文叫 stage 或index。一般存放在 .git 目录下的index文件(.git/index)中，我们
      <br/>
      把暂存区有时也叫作索引(index)。
     </li>
     <li>
      版本库:又名仓库，英文名 repository 。工作区有一个隐藏目录 .git，它不算工作区，而是 Git 的版本库。这个版本库里面的所有文件都可以被 Git 管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”下面这个图展示了工作区、暂存区和版本库之间的关系:
     </li>
     <li>
      图中左侧为工作区，右侧为版本库。Git的版本库里存了很多东西，其中最重要的就是暂存区。
     </li>
     <li>
      在创建 Git 版本库时，Git 会为我们自动创建一个唯一的 master分支，以及指向 master 的一个指
      <br/>
      针叫 HEAD。(分支和HEAD的概念后面再说)
     </li>
     <li>
      当对工作区修改(或新增)的文件执行 git add 命令时，暂存区目录树的文件索引会被更新。
     </li>
     <li>
      当执行提交操作 git commit 时，master分支会做相应的更新，可以简单理解为暂存区的目录
      <br/>
      树才会被真正写到版本库中
     </li>
    </ul>
    <p>
     由上述描述我们便能得知：通过新建或粘贴进目录的文件，并不能称之为向仓库中新增文件，而只是在工作区新增了文件。必须要通过使用git add 和git commit 命令才能将文件添加到仓库中进行管理！！！
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/5da0020671204c779b1a9e5ce9451679.png"/>
    </p>

     4.1 添加文件–场景⼀
    </h4>
    <p>
     在包含.git的目录下新建⼀个ReadMe文件，我们可以使用git add 命令可以将文件添加到暂存区：
    </p>

      <p>

     </li>
     <li>
      <p>

      </p>
     </li>
     <li>
      <p>
       添加当前目录下的所有文件改动到暂存区:
       <strong>
        git add
       </strong>
       <br/>

       <strong>
        git commt
       </strong>

       <strong>
        git commit

       <br/>

        -m “message”
       </strong>
      </p>

    </p>

     </li>
    </ol>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ cat.git / HEAD

     而默认的 master 分支，其实就是:
    </p>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">cat</span>

     打印的

      23807c536969cd886c4fb624b997ca575756eed6

    <ol start="3">
     <li>

     </li>
    </ol>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">ls</span>

    <p>
     总结一下，在本地的 git 仓库中，有几个文件或者目录很特殊
    </p>
    <ul>
     <li>
      index:暂存区， git add 后会更新该内容。
     </li>
     <li>
      HEAD: 默认指向 master分支的一个指针。
     </li>
     <li>
      refs/heads/master:文件里保存当前 master

      分支的最新 commit id。
     </li>
     <li>
      objects: 包含了创建的各种版本库对象及内容，可以简单理解为放了 git 维护的所有修改。

    <h4>
     <a id="43_360">
     </a>
     4.3添加文件–场景二
    </h4>
    <p>
     学习到这里，我们已经清楚了如何向仓库中添加文件，并且对于工作区、暂存区、版本库也有了一定的认识。那么我们再展示一种添加文件的场景，能加深对工作区、暂存区、版本库的理解，示例如
    </p>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">touch</span> file4
qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">git</span> <span class="token function">add</span> file4
qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">touch</span> file5
qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">git</span> commit - m<span class="token string">"add file"</span>  
<span class="token punctuation">[</span>master 3d406c0<span class="token punctuation">]</span> <span class="token function">add</span> <span class="token function">file</span>
<span class="token number">1</span> <span class="token function">file</span> changed, <span class="token number">0</span> insertions<span class="token punctuation">(</span>+<span class="token punctuation">)</span>, <span class="token number">0</span> deletions<span class="token punctuation">(</span>-<span class="token punctuation">)</span>
create mode <span class="token number">100644</span> file4
</code></pre>
    <p>
     提交后发现打印了
     <br/>
     1 file changed insertions(+)0 deletions(-),
     <br/>
     意思是只有一个文件改变了，这时我们提出了疑问，
     <br/>
     不是新增了两个文件吗 ?
    </p>
    <blockquote>
     <p>
      再来回忆下，git add 是将文件添加到暂存区，git commit 是将暂存区的内容添加到本地仓库中。由于我们并没有使用 git add file5 ，file5 就不在暂存区中维护，所以我们commit 的时候其实只是把已经在暂存区的 file4 提交了，而遗漏了工作区的 fie5。如何提交 file5 呢 ? 很简单，再次add, commit 即可
     </p>
    </blockquote>
