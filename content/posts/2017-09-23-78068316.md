    <p>
     Well, Java 9 is available so let’s make modules!
    </p>
    <p>
     模块化在Java9中是炒鸡炒鸡重要的！今后，模块将是应用程序的基础构建单位。无论是使用动态语言或静态语言编写，模块都是应用程序的一个重要组成部分，且模块遵循单一功能原则。你可以将任意方法称为模块，正如函数式编程也是基于模块一样。好了，总之，你get到这个点之后就记住：以后模块很重要很重要很重要 The important thing is need to be said 3 times！
    </p>
    <p>
     <img alt="这里写图片描述" src="https://img-blog.csdn.net/20170923021145739?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvcmlja2l5ZWF0/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast" title=""/>
    </p>
    <p>
     步骤：
    </p>
    <ul>
     <li>
      步骤1：下载Java 9并安装
     </li>
     <li>
      步骤2：创建一个模块java文件
     </li>
     <li>
      步骤3：模块文件路径
     </li>
     <li>
      步骤4：编写一个模块
     </li>
     <li>
      步骤5：向我们的模块添加代码
     </li>
     <li>
      步骤6：编译我们的模块
     </li>
     <li>
      步骤7：运行我们的模块
     </li>
    </ul>
    <p>
     <strong>
      1、下载Java 9并安装
     </strong>
    </p>
    <p>
     下载地址：
     <a href="http://www.oracle.com/technetwork/java/javase/downloads/jdk9-downloads-3848520.html" rel="nofollow noopener noreferrer" target="_blank">
      http://www.oracle.com/technetwork/java/javase/downloads/jdk9-downloads-3848520.html
     </a>
    </p>
    <p>
     下载后，只需点击它即可进行安装（如果您在MacOS上），并确认已安装：
    </p>
    <pre class="prettyprint"><code class="hljs lasso">tomerb@tomerb<span class="hljs-attribute">-mac</span><span class="hljs-built_in">.</span><span class="hljs-built_in">local</span>:~$ java <span class="hljs-subst">--</span>version
java <span class="hljs-number">9</span><span class="hljs-attribute">-ea</span>
Java(TM) SE Runtime Environment (build <span class="hljs-number">9</span><span class="hljs-attribute">-ea</span><span class="hljs-subst">+</span><span class="hljs-number">164</span>)
Java HotSpot(TM) <span class="hljs-number">64</span><span class="hljs-attribute">-Bit</span> Server VM (build <span class="hljs-number">9</span><span class="hljs-attribute">-ea</span><span class="hljs-subst">+</span><span class="hljs-number">164</span>, mixed mode)
tomerb@tomerb<span class="hljs-attribute">-mac</span><span class="hljs-built_in">.</span><span class="hljs-built_in">local</span>:~$ cd ~/tmp
tomerb@tomerb<span class="hljs-attribute">-mac</span><span class="hljs-built_in">.</span><span class="hljs-built_in">local</span>:~$ mkdir <span class="hljs-attribute">-p</span> ~/tmp/java9<span class="hljs-attribute">-modules</span>
tomerb@tomerb<span class="hljs-attribute">-mac</span><span class="hljs-built_in">.</span><span class="hljs-built_in">local</span>:~$ cd ~/tmp/java9<span class="hljs-attribute">-modules</span></code></pre>
    <p>
     <strong>
      2、创建一个模块java文件
     </strong>
    </p>
    <p>
     在Java 9中，为了定义模块，需要为java文件设定一个特别的名字，建议为：
     <code>
      module-info.java
     </code>
    </p>
    <p>
     <strong>
      3、模块文件路径
     </strong>
    </p>
    <p>
     <code>
      module-info.java
     </code>
     应该放在哪里呢？按照惯例，应该把它放在和模块名相同的目录下。
    </p>
    <pre class="prettyprint"><code class="hljs avrasm">如果你的模块名称是

<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>

那么你的模块  module-info<span class="hljs-preprocessor">.java</span> 应该放在：

src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>

这使您的  module-info<span class="hljs-preprocessor">.java</span> 文件放在：

src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/module-info<span class="hljs-preprocessor">.java</span> 路径。

得到它了？ &lt;module-path&gt; == &lt;module name&gt; </code></pre>
    <p>
     <strong>
      4、编写一个模块
     </strong>
    </p>
    <p>
     现在我们知道我们的模块 filename 和我们的模块 filepath，我们来编写一个具有这个命名和文件夹约定的模块：
    </p>
    <pre class="prettyprint"><code class="hljs avrasm">tomerb@tomerb-mac<span class="hljs-preprocessor">.local</span>:~/tmp/java9-modules$ mkdir -p src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>
tomerb@tomerb-mac<span class="hljs-preprocessor">.local</span>:~/tmp/java9-modules$ vi src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/module-info<span class="hljs-preprocessor">.java</span></code></pre>
    <pre class="prettyprint"><code class="hljs avrasm">module <span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span> { }</code></pre>
    <p>
     至此，我们就完成了一个模块的编写！
    </p>
    <p>
     <strong>
      5、向我们的模块添加代码
     </strong>
    </p>
    <p>
     在这一步中，我们将向我们的模块添加一些代码！在同一目录下创建一个新的java文件来作为我们的模块：
    </p>
    <pre class="prettyprint"><code class="hljs avrasm">$ mkdir -p src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/<span class="hljs-keyword">com</span>/me/mymodule
$ vi src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/<span class="hljs-keyword">com</span>/me/mymodule/Main<span class="hljs-preprocessor">.java</span></code></pre>
    <p>
     注意我们代码中的目录名称。为什么这样做呢？ 我们首先输入我们模块所在的路径，然后我们为源代码创建完整的包名称。在这种情况下，
     <code>
      /com/me/mymodule
     </code>
     在
     <code>
      /com.me.mymodule
     </code>
     的上层。只是我们的源文件属于我们的模块，并且模块已经在Java 9惯例的标准模块目录中。
    </p>
    <p>
     所以，我们就在Main.java编写hello world：
    </p>
    <pre class="prettyprint"><code class="hljs cs">package com.me.mymodule;
<span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> Main {
    <span class="hljs-keyword">public</span> <span class="hljs-keyword">static</span> <span class="hljs-keyword">void</span> <span class="hljs-title">main</span>(String[] args) {
        System.<span class="hljs-keyword">out</span>.println(<span class="hljs-string">"Hello World from Module! :)"</span>); <span class="hljs-comment">// nothing special here, standard java &lt; 9 class.</span>
    }
}</code></pre>
    <p>
     <strong>
      6、编译我们的模块
     </strong>
    </p>
    <p>
     首先制作
     <code>
      mods
     </code>
     目录（我们将进一步传递给：
     <code>
      java --module-path
     </code>
     ）：
    </p>
    <pre class="prettyprint"><code class="hljs avrasm">$ mkdir -p mods/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>
$ javac -d mods/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span> \
          src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/module-info<span class="hljs-preprocessor">.java</span> \
          src/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/<span class="hljs-keyword">com</span>/me/mymodule/Main<span class="hljs-preprocessor">.java</span></code></pre>
    <p>
     轻动你高贵的手指，让编译器再跑一会儿！
    </p>
    <p>
     <strong>
      7、运行我们的模块
     </strong>
    </p>
    <pre class="prettyprint"><code class="hljs avrasm">$ java --module-path mods -m <span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span>/<span class="hljs-keyword">com</span><span class="hljs-preprocessor">.me</span><span class="hljs-preprocessor">.mymodule</span><span class="hljs-preprocessor">.Main</span>
Hello World from Module! :)</code></pre>
    <p>
     <strong>
      总结
     </strong>
    </p>
    <p>
     在这部分中，我们下载了Java 9，创建了一个模块，向其中添加了一个源文件，并运行它。我们看到在创建源代码时创建模块路径和类似的东西时，需要遵循一个命名约定。
    </p>
