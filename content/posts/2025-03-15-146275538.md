    <h4>
     <a id="_100">
     </a>
     功能说明
    </h4>
    <h5>
     <a id="1__101">
     </a>
     1.
     <strong>
      按日期生成日志
     </strong>
    </h5>
    <ul>
     <li>
      日志文件保存在
      <code>
       ./logs
      </code>
      目录（可通过
      <code>
       --log-dir
      </code>
      修改路径）
     </li>
     <li>
      每日生成一个文件，例如
      <code>
       cpu_mem_20231001.csv
      </code>
     </li>
     <li>
      文件结构示例：
      <pre><code class="prism language-csv">Timestamp,CPU%,Memory%,Memory_Used_GB,Memory_Total_GB,Alert
2023-10-01 14:30:00,15.2,32.1,4.2,13.0,
2023-10-01 14:30:05,89.5,85.3,11.1,13.0,"CPU超过阈值(80.0%) 内存超过阈值(80.0%)"
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="2__111">
     </a>
     2.
     <strong>
      阈值报警
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       默认阈值
      </strong>
      ：CPU &gt; 80% 或 内存 &gt; 80%（可命令行修改）
     </li>
     <li>
      <strong>
       报警记录位置
      </strong>
      ：
      <code>
       logs/alert.log
      </code>
      ，内容示例：
      <pre><code>[2023-10-01 14:30:05] CPU超过阈值(80.0%) 内存超过阈值(80.0%)
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3__118">
     </a>
     3.
     <strong>
      命令行参数
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token comment"># 示例：设置 CPU 阈值 70%，内存阈值 90%，间隔 10 秒</span>
python3 monitor.py --cpu-threshold <span class="token number">70</span> --mem-threshold <span class="token number">90</span> <span class="token parameter variable">--interval</span> <span class="token number">10</span>
</code></pre>
    <ul>
     <li>
      <strong>
       参数列表
      </strong>
      ：
      <ul>
       <li>
        <code>
         --cpu-threshold
        </code>
        ：CPU 报警阈值（百分比）
       </li>
       <li>
        <code>
         --mem-threshold
        </code>
        ：内存报警阈值（百分比）
       </li>
       <li>
        <code>
         --interval
        </code>
        ：监控间隔（秒）
       </li>
       <li>
        <code>
         --log-dir
        </code>
        ：日志目录路径
       </li>
      </ul>
     </li>
    </ul>
