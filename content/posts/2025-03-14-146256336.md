    <h2>
     <a id="_14">
     </a>
     实现的目标
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7f91944c902343c9a80c44d9c8704eb0.png"/>
    </p>
    <h2>
     <a id="PS_17">
     </a>
     <mark>
      PS：特别重要的点
     </mark>
    </h2>
    <ul>
     <li>
      光猫只要一根网线到路由器的wan口
     </li>
     <li>
      我这里的情况是IPTV机顶盒直接接在路由器的lan口下播放和回放都没问题
     </li>
     <li>
      但是机顶盒通过交换机后一起接入到路由器的lan口，播放10秒左右就会卡死，尝试过更换交换机，结果是一样的，千兆和2.5G交换机都会卡死。
     </li>
     <li>
      所以：好在我的软路由有4个口，刚好1个给wan，1个给交换机，2个给2台机顶盒
     </li>
     <li>
      但是我没测试过2个机顶盒通过一个独立的交换机接入软路由是否会播放卡死。
     </li>
    </ul>
    <h2>
     <a id="_23">
     </a>
     创建接口
    </h2>
    <h3>
     <a id="vlan_24">
     </a>
     添加vlan接口
    </h3>
    <pre><code>/interface vlan add interface=eth3 name=eth3.85 vlan-id=85
/interface vlan add interface=br-lan name=br-lan.85 vlan-id=85
</code></pre>
    <h3>
     <a id="briptv_29">
     </a>
     添加br-iptv桥
    </h3>
    <pre><code>/interface bridge add name=br-iptv igmp-snooping=yes comment=br-iptv protocol-mode=none
</code></pre>
    <h3>
     <a id="briptv_32">
     </a>
     把接口添加到br-iptv桥上
    </h3>
    <pre><code>/interface bridge port add bridge=br-iptv interface=eth3.85 pvid=85
/interface bridge port add bridge=br-iptv interface=br-lan.85 pvid=85
</code></pre>
    <h2>
     <a id="_36">
     </a>
     添加接口列表
    </h2>
    <pre><code>/interface list add comment="defconf:IPTV list" name=IPTV
</code></pre>
    <h3>
     <a id="_38">
     </a>
     设置接口列表
    </h3>
    <p>
     /interface list member add comment=“defconf:iptv member” interface=br-iptv list=IPTV
    </p>
    <h2>
     <a id="option125_41">
     </a>
     设置option125
    </h2>
    <h3>
     <a id="125mac0x_42">
     </a>
     把125的mac地址中间冒号去掉，前面添加0x
    </h3>
    <pre><code>/ip dhcp-server option add code=125 force=yes name=option125 value=0x000000001a02064847572d43540a0220000b0200550d02002e
</code></pre>
    <h3>
     <a id="dhcpservernetworkoption125_45">
     </a>
     在原有的dhcp-server的network上添加option125
    </h3>
    <ul>
     <li>
      number是第几个network的意思,一般情况默认0，除非你有额外的network设置
      <pre><code>/ip dhcp-server network set dhcp-option=option125 number=0 
</code></pre>
     </li>
    </ul>
    <h2>
     <a id="_50">
     </a>
     防火墙设置
    </h2>
    <h3>
     <a id="IPTV_51">
     </a>
     允许入站IPTV接口的流量
    </h3>
    <ul>
     <li>
      这条要放在“丢弃所有非 LAN 接口进入的流量”规则上面，不然看10秒左右会卡死
      <pre><code>/ip firewall filter add action=accept chain=input comment="iptv: accept iptv" in-interface-list=IPTV
</code></pre>
     </li>
    </ul>
