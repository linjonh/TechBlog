    <h2>
     <span style="color:#fe2c24">
      一、套接字通信
     </span>
    </h2>
    <p>
     <span style="color:#0d0016">
      内核开发工程师将网络部分的头文件存储到一个专门的目录include/net中，而不是存储到标准位置include/linux。
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      计算机之间通信是一个非常复杂的问题：
     </span>
    </p>
    <ul>
     <li>
      如何建立物理连接？
     </li>
     <li>
      使用什么样的线缆？
     </li>
     <li>
      通信介质有那些限制和特殊的要求？
     </li>
     <li>
      如何处理传输错误的问题？
     </li>
     <li>
      如何识别计算机网络中的每一台计算机？
     </li>
     <li>
      如果两台计算机通过其他计算机连接，二者之间的数据是如何进行交换？
     </li>
     <li>
      如何查找最佳的路由？
     </li>
     <li>
      如何打包数据，使它不依赖于特定计算机的特性？
     </li>
    </ul>
    <h3>
     1.ISO/OSI和TCP/IP参考模型
    </h3>
    <p>
     ISO设计一种参考模型，定义组成网络的各个层，该层由7层组成，成为OSI（开放系统互联）模型如下：
    </p>
    <p class="img-center">
     <img alt="" height="401" src="https://i-blog.csdnimg.cn/direct/b3a42061ed954a608a4f13ddbd393f90.png" width="599"/>
    </p>
    <ul>
     <li>
      应用层：网络服务与最终用户的接口；
     </li>
     <li>
      表示层：数据的表示、安全及压缩；格式（jpeg, ascii 等）；
     </li>
     <li>
      会话层：建立、管理及终止传话；
     </li>
     <li>
      传输层：定义传输数据的协议端口，以及流控和差错校验；（数据包一旦离开网卡进入网络传输层）
     </li>
     <li>
      网络层：进行逻辑地址建立、实现不同网络之间的路由选择；
     </li>
     <li>
      数据链路层：建立逻辑连接、进行硬件地址寻址、差错检验等功能（由底层网络定义协议），将比特组合成字节进而组合成帧，用 MAC 地址访问介质，错误发现但不能纠正。
     </li>
    </ul>
    <p>
     <strong>
      <span style="color:#956fe7">
       其中核心基本术语如下：
      </span>
     </strong>
    </p>
    <ul>
     <li>
      数据帧（Frame）：指起始点和目的点都是数据链路层的信息单元。
     </li>
     <li>
      数据包（Packet）：指起始点和目的地是网络层的信息单元。
     </li>
     <li>
      数据报（Datagram）：指起始点和目的地都使用无连接网络服务的网络层的信息单元。
     </li>
     <li>
      段（Segment）：指起始点和目的地都是传输层的信息单元。
     </li>
     <li>
      消息（Message）：指起始点和目的地都在网络层以上的信息单元（经常在应用层）。
     </li>
     <li>
      元素（Cell）：指的是一种固定长度的信息，它的起始点和目的地都是数据链路层。元素通常用于异步传输模式（ATM）和交换多兆位数据服务（SMDS）网络等交换环境。
     </li>
     <li>
      数据单元（Data unit）：常用的数据单元有服务数据单元（SDU）、协议数据单元（PDU）。
     </li>
     <li>
      数据帧：帧数据由两部分组成：帧头部和帧数据，帧头部包括接收方主机物理地址的定位及其它网络信息，帧数据区含有一个数据体。
     </li>
     <li>
      IP 数据体由两个部分组成：数据体头部和数据体的数据区，数据体头部包括 IP 源地址和 IP 目标地址及其它信息，数据体的数据区包括用户数据协议、传输控制协议，还有数据包及其它信息。
     </li>
    </ul>
    <blockquote>
     <h4>
      1. 数据帧（Frame）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：数据链路层的信息单元，负责在相邻设备间传输数据，包含物理地址（如 MAC 地址）等链路层信息。
      <br/>
      <strong>
       例子
      </strong>
      ：好比快递包裹在某个快递网点内部流转。包裹上贴着收件网点的地址标签（类似 MAC 地址），快递员按这个标签在网点内传递包裹，这个带标签的包裹就像数据帧。
     </p>
     <h4>
      2. 数据包（Packet）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：网络层的信息单元，关注逻辑地址（如 IP 地址），用于不同网络间的路由选择。
      <br/>
      <strong>
       例子
      </strong>
      ：假设快递要从北京发往上海，快递系统根据目的地城市（类似 IP 地址）规划运输路线，这个按城市地址规划路线运输的快递包裹，就是数据包。
     </p>
     <h4>
      3. 数据报（Datagram）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：网络层无连接服务的信息单元，发送方不确认接收方是否准备好，类似 “发了就不管”。
      <br/>
      <strong>
       例子
      </strong>
      ：像寄平信，写信人把信投入邮筒后，不跟踪信是否送达、是否丢失，只负责发送，这封信就像数据报。
     </p>
     <h4>
      4. 段（Segment）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：传输层的信息单元，负责拆分大数据，确保可靠传输（如 TCP 协议分段）。
      <br/>
      <strong>
       例子
      </strong>
      ：传输一部 2GB 的电影时，传输层会把电影拆分成多个小段（类似拆分成多个小文件），每个小段添加传输控制信息（如编号），这些小段就是段。
     </p>
     <h4>
      5. 消息（Message）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：网络层以上（常为应用层）的信息单元，直接面向用户需求。
      <br/>
      <strong>
       例子
      </strong>
      ：用微信发送一段文字 “你好”，这段文字对用户是直观的信息，应用层处理后发送，这个文字内容就是消息，用户不关心底层如何传输。
     </p>
     <h4>
      6. 元素（Cell）
     </h4>
     <p>
      <strong>
       讲解
      </strong>
      ：数据链路层的固定长度信息单元，用于 ATM 等交换网络（与可变帧长的传统以太网相比，ATM固定心愿长度传输更适用于实时性、稳定性要求极高的业务；但因技术复杂、成本高，逐渐被以太网等技术取代，目前更多用于特定专业领域），传输高效、规则。
      <br/>
      <strong>
       例子
      </strong>
      ：像标准化的货柜，每个货柜大小、规格固定（固定长度）。在 ATM 网络中，数据像货物一样被装进这些固定规格的货柜传输，确保快速交换，这个货柜就是元素。
     </p>
     <h4>
      7. 数据单元（SDU、PDU）
     </h4>
     <ul>
      <li>
       <strong>
        服务数据单元（SDU）
       </strong>
       ：上层协议交给下层处理的数据，关注 “服务内容”。
       <br/>
       <strong>
        例子
       </strong>
       ：用户在网盘上传文件，文件内容就是 SDU，网盘底层协议（如 HTTP）会处理它。
      </li>
      <li>
       <strong>
        协议数据单元（PDU）
       </strong>
       ：加入协议控制信息后的数据单元，关注 “协议处理”。
       <br/>
       <strong>
        例子
       </strong>
       ：还是上传文件，底层协议给文件添加头部信息（如传输规则、地址），组合后的整体（文件 + 头部）就是 PDU，类似给文件装个 “协议信封” 再发送。
      </li>
     </ul>
    </blockquote>
    <p>
     <span style="color:#fe2c24">
      <strong>
       各层执行的任务：
      </strong>
     </span>
    </p>
    <p>
     主机到网络层负责将信息从一台计算机传输到远程计算机。它处理传输介质的物理性质，并将数据流换分为定长的帧，以便在发生传输错误时重传数据块。假设几台电脑共享同一传输线程，网络接口卡必须有唯一一个ID号，MAC地址。
    </p>
    <p>
     IP使用一定格式的地址来寻址计算机，比如192.168.186.138，这些地址由正式注册权威机构或提供者分配（有时为动态的）。
    </p>
    <h3>
     2.创建套接字
    </h3>
    <p>
     套接字不仅可以用于各种传输协议的IP连接，也可以用于内核支持的所有其他地址和协议类型（例如：IPX、Appletalk、本地UNIX套接字，还有在&lt;socket.h&gt;中列出的许多其他类型）。
    </p>
    <p class="img-center">
     <img alt="" height="124" src="https://i-blog.csdnimg.cn/direct/ee94bd5334654f9ea301acfdc515236c.png" width="755"/>
    </p>
    <ul>
     <li>
      sa_family：是地址家族，一般都是以 AF_XXX 形式出现，比如大多数都是用 AF_INET，代表 TCP/IP 协议中用于ipv4的地址家族。
     </li>
     <li>
      Sa_data：是 14 字节协议地址。这个数据结构用做 bind, connect, recvfrom, sendto 等函数的参数，指明地址信息。
     </li>
    </ul>
    <p class="img-center">
     <img alt="" height="133" src="https://i-blog.csdnimg.cn/direct/a2d6f20bcf6a4454be59622d5ded7e39.png" width="676"/>
    </p>
    <p>
     short sin_family; // 一般是指：AF_INET 地址族 PF_INET 协议族
     <br/>
     unsigned short sin_port;// 端口号（必须要采用网络数据格式，普通数字大家可以使用 htons () 函数转换成网络数据格式的数字）
     <br/>
     struct in_addr sin_addr; // IP 地址 32 位
     <br/>
     unsigned char sin_zero [8]; // 它没有实际意义，只是为了跟 sockaddr 结构在内存中对齐
    </p>
    <p class="img-center">
     <img alt="" height="97" src="https://i-blog.csdnimg.cn/direct/e9172f4c4fb04567bf276fcee4ba8eae.png" width="626"/>
    </p>
    <p>
     in_addr_t s_addr就是存储32位的ipv4地址，明确区分IPv4地址与其他数据，避免直接操作原始整数，并与特定API兼容（如bind、connect）。
    </p>
    <p>
    </p>
    <p>
     <span style="color:#fe2c24">
      BSD 网络软件中包含两个重要的函数：inet_addr，inet_ntoa。用来在二进制地址格式和点分十进制字符串格式之间转换，仅支持 IPv4。也有两个函数同时支持 IPv4 和 IPv6：inet_ntop, inet_pton。
     </span>
    </p>
    <p>
    </p>
    <h3>
     3.使用套接字
    </h3>
    <p>
     简要说明套接字如何表示到内核网络子系统的接口，设计简短程序。例如简单的并发服务器模型如下图所示。在服务器端，主程序提前构建多个子进程，当客户端的请求到来的时候，系统从进程池中选取一个子进程处理客户端的连接，每个子进程处理一个客户端的请求，在全部子进程的处理能力得到满足之前，服务器的网络负载是基本不变的。
    </p>
    <p class="img-center">
     <img alt="" height="253" src="https://i-blog.csdnimg.cn/direct/cdb2c466745e46009da5c4bfc0737376.png" width="400"/>
    </p>
    <p>
     TCP 并发服务器，在处理客户端请求之前，程序先分叉为 3 个子进程，对应多个客户端的请求，由多个子进程进行处理。与循环服务器相比较，并发的 TCP 程序，在处理客户端请求的时候，不再简单地使用一个 while 进行客户端请求的串行处理，而是 fork () 一个进程，将客户端的请求放到一个进程中进行处理。
    </p>
    <p>
     （1）server.c
    </p>
    <pre><code class="language-cpp">#include &lt;sys/types.h&gt;
#include &lt;sys/socket.h&gt;
#include &lt;netinet/in.h&gt;
#include &lt;time.h&gt;
#include &lt;string.h&gt;
#include &lt;unistd.h&gt;
#include &lt;stdlib.h&gt;
#include &lt;stdio.h&gt;
#include &lt;signal.h&gt;

#define BUFFLEN 1024
#define SERVER_PORT 8888
#define BACKLOG 5
#define PIDNUMB 3
static void handle_connect(int s_s)
{	
	int s_c;	
	struct sockaddr_in from;				
	socklen_t len = sizeof(from);
	
	while(1)
	{
		s_c = accept(s_s, (struct sockaddr*)&amp;from, &amp;len);
		time_t now;								
		char buff[BUFFLEN];						
		int n = 0;
		memset(buff, 0, BUFFLEN);			
		n = recv(s_c, buff, BUFFLEN,0);		
		if(n &gt; 0 &amp;&amp; !strncmp(buff, "TIME", 4))	
		{
			memset(buff, 0, BUFFLEN);			
			now = time(NULL);					
			sprintf(buff, "%24s\r\n",ctime(&amp;now));	
			send(s_c, buff, strlen(buff),0);	
		}		
		close(s_c);	
	}
		
}
void sig_int(int num)
{
	exit(1);	
}
int main(int argc, char *argv[])
{
	int s_s;									
	struct sockaddr_in local;					
	signal(SIGINT,sig_int);  //捕获ctrl+C,使用sig_int函数处理
	
	s_s = socket(AF_INET, SOCK_STREAM, 0);
	
	memset(&amp;local, 0, sizeof(local));			
	local.sin_family = AF_INET;					
	local.sin_addr.s_addr = htonl(INADDR_ANY);	
	local.sin_port = htons(SERVER_PORT);		
	
	bind(s_s, (struct sockaddr*)&amp;local, sizeof(local));
	listen(s_s, BACKLOG);				
	
	pid_t pid[PIDNUMB];
	int i =0;
	for(i=0;i&lt;PIDNUMB;i++)
	{
		pid[i] = fork();
		if(pid[i] == 0)						
		{
			handle_connect(s_s);
		}
	}
    
	while(1);	
	close(s_s);
	
	return 0;
}</code></pre>

    <h3>
     4.数据报套接字
    </h3>
    <p>
     UDP是建立在IP连接之上的第二种广泛使用的传输协议。UDP表示User Datagram Protocal（用户数据报协议）。UDP通常用于视频会议、音频流以及类似的服务。
    </p>
    <p>
     <span style="color:#956fe7">
      <strong>
       UDP 与 TCP 有所不同
      </strong>
     </span>
     <br/>
     ● UDP 是面向分组的，在发送数据之前，无须建立显式的连接；
     <br/>
     ● 分组可以在传输期间丢失，不保证数据一定能够到达其目的地；
     <br/>
     ● 分组接收的次序不一定与发送的次序相同。
    </p>
    <p>
    </p>
    <h2>
     <span style="color:#fe2c24">
      二、网络分层模型
     </span>
    </h2>
    <h3>
     <span style="color:#0d0016">
      1.模型
     </span>
    </h3>
    <p>
     内核网络子系统的实现与刚介绍的TCP\IP参考模型非常相似。相关的C语言代码划分为不同层次，各层次都有明确定义的任务，各个层次只能通过明确定义的接口与上下文紧邻的层次通信。这种做法的好处在于，可以组合使用各种设备、传输机制和协议。
    </p>

    <h3>
     2.历史上的协议
    </h3>
    <blockquote>
     <h4>
      <strong>
       一、AppleTalk 协议
      </strong>
     </h4>
     <h5>
      1.
      <strong>
       背景与历史
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         开发者
        </strong>
        ：Apple 公司（1985年推出）。
       </p>
      </li>
      <li>
       <p>
        <strong>
         用途
        </strong>
        ：专为早期
        <strong>
         Macintosh 计算机
        </strong>
        设计的局域网协议，支持文件共享、打印服务等。
       </p>
      </li>
      <li>
       <p>
        <strong>
         现状
        </strong>
        ：已被 TCP/IP 取代，现代 macOS 不再原生支持。
       </p>
      </li>
     </ul>
     <h5>
      2.
      <strong>
       协议组成
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         DDP（Datagram Delivery Protocol）
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           作用
          </strong>
          ：网络层协议，提供无连接的数据报传输（类似 IP）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           寻址
          </strong>
          ：使用
          <strong>
           网络号 + 节点号
          </strong>
          （如
          <code>
           Network 100, Node 5
          </code>
          ）。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         ATP（AppleTalk Transaction Protocol）
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           作用
          </strong>
          ：传输层协议，提供可靠的事务处理（类似 TCP）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           特点
          </strong>
          ：基于请求-响应模型，确保数据完整性。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         其他协议
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           ZIP（Zone Information Protocol）
          </strong>
          ：管理逻辑网络分区（Zone）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           NBP（Name Binding Protocol）
          </strong>
          ：将设备名解析为网络地址。
         </p>
        </li>
       </ul>
      </li>
     </ul>
     <h5>
      3.
      <strong>
       网络拓扑
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         物理层
        </strong>
        ：支持 LocalTalk（专用串行接口，速度 230 Kbps）、EtherTalk（基于以太网）。
       </p>
      </li>
      <li>
       <p>
        <strong>
         局限性
        </strong>
        ：低速、封闭生态，难以适应互联网时代需求。
       </p>
      </li>
     </ul>
     <hr/>
     <h4>
      <strong>
       二、IPX/SPX 协议
      </strong>
     </h4>
     <h5>
      1.
      <strong>
       背景与历史
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         开发者
        </strong>
        ：Novell 公司（1980年代）。
       </p>
      </li>
      <li>
       <p>
        <strong>
         用途
        </strong>
        ：为
        <strong>
         Novell NetWare
        </strong>
        网络操作系统设计的核心协议。
       </p>
      </li>
      <li>
       <p>
        <strong>
         现状
        </strong>
        ：逐渐被 TCP/IP 取代，Windows 通过
        <strong>
         NWLink
        </strong>
        兼容 IPX/SPX。
       </p>
      </li>
     </ul>
     <h5>
      2.
      <strong>
       协议组成
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         IPX（Internetwork Packet Exchange）
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           作用
          </strong>
          ：网络层协议，提供无连接的数据包路由（类似 IP）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           寻址
          </strong>
          ：使用
          <strong>
           网络号 + 节点号（MAC地址）
          </strong>
          （如
          <code>
           Network 0x1A2B, Node 00:1A:2B:3C:4D:5E
          </code>
          ）。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         SPX（Sequenced Packet Exchange）
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           作用
          </strong>
          ：传输层协议，提供面向连接的可靠传输（类似 TCP）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           特点
          </strong>
          ：支持数据分段、按序传递和重传机制。
         </p>
        </li>
       </ul>
      </li>
     </ul>
     <h5>
      3.
      <strong>
       优势与局限
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         优势
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           高效路由
          </strong>
          ：IPX 的路由协议（RIP）简单高效，适合局域网。
         </p>
        </li>
        <li>
         <p>
          <strong>
           即插即用
          </strong>
          ：节点自动获取网络号，无需手动配置。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         局限
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           缺乏互联网兼容性
          </strong>
          ：IPX 地址格式与 IP 不兼容，无法直接接入互联网。
         </p>
        </li>
        <li>
         <p>
          <strong>
           协议臃肿
          </strong>
          ：SPX 的可靠性机制增加了开销。
         </p>
        </li>
       </ul>
      </li>
     </ul>
     <hr/>
     <h4>
      <strong>
       三、SDLC 与 HDLC 协议
      </strong>
     </h4>
     <h5>
      1.
      <strong>
       SDLC（Synchronous Data Link Control）
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         背景
        </strong>
        ：IBM 为
        <strong>
         SNA（Systems Network Architecture）
        </strong>
        设计的数据链路层协议。
       </p>
      </li>
      <li>
       <p>
        <strong>
         特点
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           同步传输
          </strong>
          ：时钟信号同步数据帧。
         </p>
        </li>
        <li>
         <p>
          <strong>
           拓扑支持
          </strong>
          ：点对点、多点（轮询机制）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           帧格式
          </strong>
          ：帧头 + 数据 + CRC 校验 + 帧尾。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         应用场景
        </strong>
        ：早期 IBM 大型机与终端通信。
       </p>
      </li>
     </ul>
     <h5>
      2.
      <strong>
       HDLC（High-Level Data Link Control）
      </strong>
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         背景
        </strong>
        ：基于 SDLC 改进，由 ISO 标准化（ISO 13239）。
       </p>
      </li>
      <li>
       <p>
        <strong>
         改进点
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           协议独立性
          </strong>
          ：支持多种网络层协议。
         </p>
        </li>
        <li>
         <p>
          <strong>
           帧类型扩展
          </strong>
          ：新增监控帧（Supervisory Frame）和无编号帧（Unnumbered Frame）。
         </p>
        </li>
        <li>
         <p>
          <strong>
           多点连接优化
          </strong>
          ：支持平衡模式（两台设备平等通信）。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         应用场景
        </strong>
        ：广域网（如 PPP 协议基于 HDLC 设计）、路由器间通信。
       </p>
      </li>
     </ul>

     <h4>
      <strong>
       五、为何这些协议被淘汰？
      </strong>
     </h4>
     <ol>
      <li>
       <p>
        <strong>
         TCP/IP 的崛起
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           开放性
          </strong>
          ：TCP/IP 是开放标准，跨平台兼容性强。
         </p>
        </li>
        <li>
         <p>
          <strong>
           互联网支持
          </strong>
          ：IP 地址体系天然适配全球互联网。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         技术局限性
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          AppleTalk 和 IPX 的封闭生态难以扩展。
         </p>
        </li>
        <li>
         <p>
          SDLC/HDLC 复杂度高，被更简单的 PPP、以太网取代。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         硬件演进
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          高速以太网和无线网络需要更高效的协议栈。
         </p>
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
