    <h2>
     <a id="1__0">
     </a>
     1. 什么是红队
    </h2>
    <p>
     红队（Red Team)是安全团队最大化模仿真实世界里的入侵事件，包括入侵者的战术技术流程（TTPs)，检验蓝队（Blue Team)的威胁检测及应急响应效率，帮助企业或政府提高整个安全建设及安全运营能力。
    </p>
    <h2>
     <a id="2__4">
     </a>
     2. 红队产生的背景
    </h2>
    <ul>
     <li>
      红队产生的背景主要是全球网络空间安全威胁加剧
      <ul>
       <li>
        全球肆虐的勒索病毒
       </li>
       <li>
        核武器级的网络军火库
       </li>
       <li>
        国家级的APT攻击
       </li>
       <li>
        组织严密的黑客团体
       </li>
      </ul>
     </li>
     <li>
      新型IT基础设施带来的新挑战
      <ul>
       <li>
        人工知识、大数据、云计算、5G 物联网， 从传统的信息安全变成网络空间安全
       </li>
      </ul>
     </li>
     <li>
      传统安全技术与管理应对乏力
      <ul>
       <li>
        企业或政府买了安全设备、招了安全人才，颁布了安全规范，但还是被黑客入侵，导致信息泄漏等
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="3__16">
     </a>
     3. 如何建立红队的知识体系？
    </h2>
    <h3>
     <a id="31_PTES_18">
     </a>
     3.1. PTES
    </h3>
    <p>
     <a href="http://www.pentest-standard.org/index.php/PTES_Technical_Guidelines" rel="nofollow">
      PTES（Penetration Tseting Execution Standard）渗透测试执行标准
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/12a33f6c41e2ec76f033f507ed3decf3.png">
      <br/>
      它的核心理念是通过建立起进行渗透测试所要求的基本准则基线定义一次真正的渗透测试过程，得到了安全业界的广泛认同。
     </img>
    </p>
    <p>
     此标准将渗透测试分为
     <strong>
      七个阶段
     </strong>
     ，
     <strong>
      在每个阶段中定义不同的扩展级别，选择哪种级别由被攻击测试的客户组织决定
     </strong>
     。
     <br/>
     详细参见：
     <a href="https://blog.csdn.net/zqzqzqzqwuwuwu/article/details/115612049">
      渗透测试PTES标准流程（超详细）
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/9b7724818abe31ea0bf1b456523accc6.png"/>
    </p>
    <h3>
     <a id="32__CKC_29">
     </a>
     3.2. 网络杀伤链 CKC
    </h3>
    <p>
     CKC: 网络杀伤链 Cyber Kill Chain ，2011年提出的一个网络攻击模型，对于真实世界中的每个攻击阶段，包括目标侦察、武器研制、载荷投递、渗透利用、安装植入、命令控制、任务执行 七个阶段
    </p>
    <p>
     如何利用CKC进行网络防御，详见
     <a href="https://www.lockheedmartin.com/content/dam/lockheed-martin/rms/documents/cyber/Gaining_the_Advantage_Cyber_Kill_Chain.pdf" rel="nofollow">
      Applying Cyber Kill Chain® Methodology to Network Defense
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5390c8261ce2268a5fdb0f3fce8e4945.png"/>
    </p>
    <h3>
     <a id="33_ATTCK_34">
     </a>
     3.3. ATT&amp;CK
    </h3>
    <p>
     <a href="http://attack.mitre.org/" rel="nofollow">
      ATT&amp;CK官网
     </a>
     <br/>
     MITRE ATT&amp;CK®是一个基于真实世界观察的对手战术和技术的全球可访问知识库。ATT&amp;CK知识库被用作私营部门、政府以及网络安全产品和服务社区开发特定威胁模型和方法的基础。
    </p>
    <ul>
     <li>
      ATT&amp;CK（Adversarial Tactics, Techniques, And Common Knowledge) 是由MITRE公司2013年提出的
      <strong>
       对抗战术、技术和常识
      </strong>
      框架，基于真实世界的攻防数据来描述和分类对抗行为，目的是不同的红队、蓝队、组织机构等构建一套通用语言
     </li>
     <li>
      ATT&amp;CK框架可以看成是在Cyber Kill Chain的基础上，提供了更细粒度、更易共享的战术、技术、文档、工具等资源
     </li>
     <li>
      ATT&amp;CK框架分为ATT&amp;CK for Enterprise、ATT&amp;CK for mobile
     </li>
    </ul>
