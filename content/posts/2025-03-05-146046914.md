    <p>
     分布式网络（
     <strong>
      Distributed Network
     </strong>
     ）指的是一种计算机网络架构，其中计算资源（计算、存储、数据处理等）分布在多个物理或逻辑上的节点上，而不是集中在单一的服务器或数据中心中。这种架构的主要目标是
     <strong>
      提高系统的可靠性、可扩展性和性能
     </strong>
     。
    </p>
    <h3>
     <strong>
      1. 分布式网络的特点
     </strong>
    </h3>
    <p>
     ✅
     <strong>
      去中心化（Decentralization）
     </strong>
    </p>
    <ul>
     <li>
      传统的
      <strong>
       集中式网络
      </strong>
      （如单台服务器）会有单点故障（SPOF，Single Point of Failure）。
     </li>
     <li>
      <strong>
       分布式网络没有单一的控制中心
      </strong>
      ，多个节点可以协作完成任务。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      高可用性（High Availability）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       多个副本
      </strong>
      （如数据库的主从复制、数据分片）保证系统在部分节点失效时仍能运行。
     </li>
     <li>
      <strong>
       容错机制
      </strong>
      （如Leader选举、心跳检测）提高网络稳定性。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      可扩展性（Scalability）
     </strong>
    </p>
    <ul>
     <li>
      通过
      <strong>
       增加节点
      </strong>
      （横向扩展，scale-out）提升计算能力，而不是仅依赖于单个服务器的升级（纵向扩展，scale-up）。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      数据一致性（Consistency）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       CAP 定理
      </strong>
      （一致性 Consistency、可用性 Availability、分区容忍性 Partition Tolerance）在分布式网络设计中非常重要，通常需要权衡。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      异地容灾（Disaster Recovery）
     </strong>
    </p>
    <ul>
     <li>
      分布式架构通常会在多个数据中心部署，防止某个区域的故障影响整个系统。
     </li>
    </ul>

    <h3>
     <strong>
      3. 分布式网络的关键技术
     </strong>
    </h3>
    <h4>
     <strong>
      (1) 分布式存储
     </strong>
    </h4>
    <p>
     分布式存储系统能够跨多个服务器存储数据，保证数据冗余，提高读写性能：
    </p>
    <ul>
     <li>
      <strong>
       HDFS（Hadoop Distributed File System）
      </strong>
      ：大数据存储
     </li>
     <li>
      <strong>
       Ceph、GlusterFS
      </strong>
      ：对象存储
     </li>
     <li>
      <strong>
       Amazon S3、腾讯云 COS
      </strong>
      ：云存储
     </li>
    </ul>
    <h4>
     <strong>
      (2) 分布式计算
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       MapReduce
      </strong>
      （Hadoop 生态）：将计算任务拆分成多个任务并行执行。
     </li>
     <li>
      <strong>
       Spark
      </strong>
      ：基于内存的分布式计算引擎，比 MapReduce 更快。
     </li>
     <li>
      <strong>
       Flink
      </strong>
      ：专注于实时流计算。
     </li>
    </ul>
    <h4>
     <strong>
      (3) 分布式数据库
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       OLTP（事务型数据库）
      </strong>
      ：MySQL + 分片（Sharding）、TiDB、CockroachDB
     </li>
     <li>
      <strong>
       OLAP（分析型数据库）
      </strong>
      ：ClickHouse、Druid、Presto
     </li>
     <li>
      <strong>
       NoSQL 数据库
      </strong>
      ：HBase（列存储）、Cassandra（分布式 KV 存储）
     </li>
    </ul>
    <h4>
     <strong>
      (4) 分布式消息队列
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Kafka
      </strong>
      ：高吞吐日志收集、数据流处理
     </li>
     <li>
      <strong>
       RabbitMQ、RocketMQ
      </strong>
      ：事件驱动架构
     </li>
    </ul>
    <h4>
     <strong>
      (5) 分布式协调 &amp; 负载均衡
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Zookeeper
      </strong>
      ：管理分布式系统中的 leader 选举、配置管理
     </li>
     <li>
      <strong>
       Nginx、Envoy
      </strong>
      ：负载均衡，均衡分发流量
     </li>
     <li>
      <strong>
       Kubernetes（K8S）
      </strong>
      ：容器编排，自动伸缩服务
     </li>
    </ul>

      4. 分布式网络的典型应用场景

     ✅
     <strong>
      云计算
     </strong>
     ：腾讯云、阿里云、AWS 等提供的云存储、云计算服务

     ✅
     <strong>
      CDN（内容分发网络）
     </strong>
     ：加速网站访问，如 Cloudflare、腾讯 CDN
     <br/>
     ✅
     <strong>
      微服务架构
     </strong>
     ：多个服务部署在不同机器上，使用 gRPC、HTTP 进行通信

    <h3>
     <strong>
      分布式存储原理
     </strong>
    </h3>
    <p>
     分布式存储（
     <strong>
      Distributed Storage
     </strong>
     ）是一种数据存储架构，它将数据分布在多个物理或逻辑服务器上，而不是集中在单台机器上。这样可以提高
     <strong>
      数据可用性、扩展性、容错能力
     </strong>
     ，适用于大规模数据存储需求。
    </p>
    <hr/>
    <h3>
     <strong>
      1. 分布式存储的核心概念
     </strong>
    </h3>
    <p>
     ✅
     <strong>
      数据分片（Sharding）
     </strong>
     <br/>
     将大规模数据拆分成小块（Shard/Chunk），分布到不同的存储节点上，提高存储性能和扩展性。
    </p>
    <p>
     ✅
     <strong>
      数据副本（Replication）
     </strong>
     <br/>
     同一份数据存储多个副本，保证在部分节点故障时，数据仍然可用，提高数据可靠性。
    </p>
    <p>
     ✅
     <strong>
      一致性（Consistency）
     </strong>
     <br/>
     分布式存储系统需要确保多副本之间的数据一致性，常见策略：
    </p>
    <ul>
     <li>
      <strong>
       强一致性（Strong Consistency）
      </strong>
      ：所有副本必须同步，保证最新数据（如分布式事务）。
     </li>
     <li>
      <strong>
       最终一致性（Eventual Consistency）
      </strong>
      ：允许短时间的数据不一致，最终会同步（如 DNS、Cassandra）。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      CAP 定理
     </strong>
     （Consistency、Availability、Partition Tolerance）
    </p>
    <ul>
     <li>
      <strong>
       一致性（C）
      </strong>
      ：所有副本的数据相同
     </li>
     <li>
      <strong>
       可用性（A）
      </strong>
      ：即使部分节点故障，系统仍然可以读写
     </li>
     <li>
      <strong>
       分区容忍性（P）
      </strong>
      ：即使网络分区（节点之间网络断开），系统仍能运行
     </li>
    </ul>
    <p>
     <strong>
      CAP 定理告诉我们：分布式存储系统不能同时保证 C、A、P，只能二者兼得。
     </strong>
     <br/>
     例如：
    </p>
    <ul>
     <li>
      <strong>
       HDFS、BigTable
      </strong>
      偏向
      <strong>
       CP（强一致性 + 分区容忍性）
      </strong>
     </li>
     <li>
      <strong>
       Cassandra、DynamoDB
      </strong>
      偏向
      <strong>
       AP（高可用 + 分区容忍性）
      </strong>
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      2. 分布式存储架构
     </strong>
    </h3>
    <h4>
     <strong>
      (1) 主从架构（Master-Slave）
     </strong>
    </h4>
    <ul>
     <li>
      数据存储在
      <strong>
       主节点（Master）
      </strong>
      ，从节点（Slave）同步数据
     </li>
     <li>
      适用于
      <strong>
       数据库读多写少的场景
      </strong>
     </li>
     <li>
      例如
      <strong>
       MySQL 主从复制
      </strong>
     </li>
    </ul>
    <p>
     ✅
     <strong>
      优点
     </strong>
     ：
    </p>
    <ul>
     <li>
      读写分离，提高读取性能
     </li>
     <li>
      主节点崩溃后，可以手动或自动选举新的主节点
     </li>
    </ul>
    <p>
     ❌
     <strong>
      缺点
     </strong>
     ：
    </p>
    <ul>
     <li>
      主节点负载大，写入性能有限
     </li>
     <li>
      主从同步可能存在数据延迟
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      (2) 去中心化架构（P2P / Shared-Nothing）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       无中心化主节点，每个节点都有独立的存储和计算能力
      </strong>
     </li>
     <li>
      采用
      <strong>
       一致性哈希（Consistent Hashing）
      </strong>
      进行数据分布
     </li>
     <li>
      适用于
      <strong>
       海量数据存储 &amp; 分布式文件系统
      </strong>
     </li>
     <li>
      例如
      <strong>
       Cassandra、Ceph
      </strong>
     </li>
    </ul>
    <p>
     ✅
     <strong>
      优点
     </strong>
     ：
    </p>
    <ul>
     <li>
      易于扩展，动态增加节点
     </li>
     <li>
      容错能力强，某些节点失效不影响整体
     </li>
    </ul>
    <p>
     ❌
     <strong>
      缺点
     </strong>
     ：
    </p>
    <ul>
     <li>
      复杂的路由机制
     </li>
     <li>
      需要高效的副本同步和负载均衡
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      (3) 分布式文件系统
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       数据切片存储到多个存储节点
      </strong>
      ，并生成多个副本
     </li>
     <li>
      适用于
      <strong>
       大规模非结构化数据
      </strong>
     </li>
     <li>
      例如
      <strong>
       HDFS（Hadoop Distributed File System）
      </strong>
     </li>
    </ul>
    <p>
     ✅
     <strong>
      优点
     </strong>
     ：
    </p>
    <ul>
     <li>
      支持超大规模存储（如 PB 级数据）
     </li>
     <li>
      适用于
      <strong>
       流式读取
      </strong>
      场景（如大数据分析）
     </li>
    </ul>
    <p>
     ❌
     <strong>
      缺点
     </strong>
     ：
    </p>
    <ul>
     <li>
      不适合低延迟场景
     </li>
     <li>
      写入吞吐量有限
     </li>
    </ul>

    <h3>
     <strong>
      4. 分布式存储的关键技术
     </strong>
    </h3>
    <h4>
     <strong>
      (1) 数据分片（Sharding）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Hash 分片
      </strong>
      ：使用 Hash 函数分配数据，如
      <code>
       key % num_shards
      </code>
     </li>
     <li>
      <strong>
       范围分片（Range Partitioning）
      </strong>
      ：按数据范围分片，如
      <code>
       user_id &lt; 1000 -&gt; shard_1
      </code>
     </li>
     <li>
      <strong>
       一致性哈希（Consistent Hashing）
      </strong>
      ：减少数据迁移，提高扩展性（如 Cassandra）
     </li>
    </ul>
    <h4>
     <strong>
      (2) 数据副本（Replication）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       同步复制（Strong Consistency）
      </strong>
      ：主节点写入数据，立即同步到从节点（如 MySQL）
     </li>
     <li>
      <strong>
       异步复制（Eventually Consistent）
      </strong>
      ：主节点写入数据，从节点异步更新（如 Cassandra）
     </li>
     <li>
      <strong>
       多主复制（Multi-Master）
      </strong>
      ：多个节点可同时写入，提高吞吐量（如 TiDB）
     </li>
    </ul>
    <h4>
     <strong>
      (3) 数据一致性
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Paxos / Raft 共识协议
      </strong>
      ：保证数据一致性（如 Etcd、ZooKeeper）
     </li>
     <li>
      <strong>
       Quorum 机制
      </strong>
      ：确保多数派副本更新后才确认写入（如 Cassandra）
     </li>
    </ul>
    <h4>
     <strong>
      (4) 负载均衡 &amp; 容灾
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       负载均衡（Load Balancing）
      </strong>
      ：均衡多个存储节点的压力，如 Nginx + LVS
     </li>
     <li>
      <strong>
       数据修复（Self-Healing）
      </strong>
      ：检测数据丢失并自动恢复，如 HDFS DataNode 检测
     </li>
    </ul>
    <hr/>
    <h3>
     <strong>
      5. 分布式存储的应用场景
     </strong>
    </h3>
    <p>
     ✅
     <strong>
      大数据分析（HDFS、Hive）
     </strong>
     ：存储 TB 级数据，支持并行计算
     <br/>
     ✅
     <strong>
      日志存储（Kafka + S3）
     </strong>

     <strong>
