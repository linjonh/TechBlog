# 【AWS入门】创建并使用AWS Builder ID

## 【AWS入门】创建并使用AWS Builder ID

### Create and Use AWS Builder ID

By Jackson@ML

#### 1\. AWS Builder ID简介

> **AWS Builder ID** ，是提供访问Amazon所选工具及服务(例如：**Amazon CodeCatalyst, Amazon Q
> Developer, AWS Training and Certification** 等)的**个人用户配置文件** 。

AWS Builder ID代表个人独立于已注册AWS账户的任何凭据和数据。和其它个人资料一样，AWS Builder
ID由个人保留，不管是否完成个人、教育和职业目标。

众所周知，AWS账户可充当用户创建的AWS资源的容器，并为这些资源提供安全边界，但用户的AWS Builder ID代表用户个人。

AWS Builder ID是**免费的** ，使用时仅需支付AWS账户中消耗的AWS相关资源。

#### 2\. 创建AWS Builder ID

登录AWS工具及服务时，可以创建AWS Builder ID。创建该账户需要Email地址，姓名和密码。

注册完毕，进入AWS Builder ID（中文名为“AWS构建者ID”），访问“我的详细信息”，查询个人资料信息，如下图所示：

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/723b603e86dc4b33a8d9a74d01a9db64.png)

#### 3\. 用AWS Builder ID启用AWS工具和服务

默认情况下,使用AWS Builder ID登录AWS工具或服务时,会话持续时间为30天（除了 Amazon Q
Developer的会话持续时间为90天）。

该会话结束后,需要再次登录，才能继续访问AWS工具和服务。

可以使用AWS生成器ID登录，以访问以下AWS工具和服务。

##### 1) AWS Cloud Community

<https://community.aws>是一个AWS构建者的社区平台，可用AWS Builder
ID来访问。在此可以探索发现教育内容，分享个人思路和项目，评论他人帖子，关注喜欢的AWS构建者。

AWS Cloud Community社区Web站点如下图所示。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/1d3fb9e033e04eb3ae17e9ad7f473247.png)

##### 2) Amazon CodeCatalyst

访问Amazon CodeCatalyst时，用户会创建AWS Builder ID，可选择一个别名，用以关联诸如问题，代码提交和提交请求等活动。

可邀请其他人到你的Amazon CodeCatalyst空间，它含有工具，基础架构和必要的团队环境，用以构建成功项目。

用户需要一个AWS账户来将项目部署到云端。

Amazon CodeCatalyst官方文档站点如下图。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/508deb9cfed84b9dbdfec08f183a423d.png)

##### 3) AWS Migration Hub

用AWS Builder ID可访问**AWS Migration Hub** （迁移集线器），如下图所示。  
AWS Migration Hub提供了一个单一的位置来发现用户现有的服务器、规划迁移，并跟踪每个应用程序迁移的状态。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/6b9fc0f7886e4d0bb3bdd5ec723a22cd.png)

##### 4) Amazon Q Developer

**Amazon Q Developer** 是一个由生成式AI驱动的会话助手。该助手可以帮助用户**理解、构建、扩展和操作** AWS应用程序。

Amazon Q Developer操作步骤，可详见Amazon Q Developer用户指南。  
以下是AWS Q Developer官方文档站点。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/ca7aef7d022c4e40b78121c5c241e36d.png)

##### 5) AWS re:Post

**AWS re:Post** 为用户提供专家级技术指导，以便能够更快创新，以及改善AWS服务的运营效率。

可用AWS Builder ID登录并在没有AWS账户或者信用卡的情况下，通过AWS
re:Post加入社区。这里有热门话题，以及顶级贡献者，他们都在技术上各有千秋，对开发者交流非常方便。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/6987e30200ab43e5b5c75a15daa338a4.png)

##### 6) AWS Startups

用AWS Builder ID登录，可以加入AWS Startups，这样可以利用学习内容，工具和资源；也能够支持在AWS快速成长。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/63243544e479429e8be24157572eae74.png)

##### 7) AWS Training and Certification

**AWS Training and Certification**( AWS培训和认证)网站<https://www.aws.training/>
，可由AWS Builder ID一键登入访问；  
这里汇聚了所有AWS培训和认证内容。包括 _**AWS云从业者认证，AWS认证解决方案架构师，AWS认证开发者**_
等十几个AWS官方认证的培训资料和备考资料等。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/463f447348834e85b18a0fe1070287b2.png)

##### 8) Website Registration Portal (WRP)

**Website Registration Portal**(**Web站点注册门户** ，缩写**WRP**)可使用AWS Builder
ID作为AWS营销网站的持久客户身份和注册配置文件。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/6218a2881f664cb39a32d03a3fce5b60.png)  
如果需要的话，请注册新的网络研讨会，并查看注册或参加过的所有网络研讨会。

##### 9) 删除AWS Builder ID

进入AWS配置文件页面<https://profile.aws.amazon.com>，选择AWS Builder ID页面隐私和数据，在其页面删除AWS
Builder ID，确定执行。

*注：该账户删除后，用户不再有权限访问AWS工具和服务。
