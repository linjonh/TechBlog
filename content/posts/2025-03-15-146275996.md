    <h3>
     <a id="Create_and_Use_AWS_Builder_ID_1">
     </a>
     Create and Use AWS Builder ID
    </h3>
    <p>
     By Jackson@ML
    </p>
    <h4>
     <a id="1%09AWS_Builder_ID_4">
     </a>
     1. AWS Builder ID简介
    </h4>
    <blockquote>
     <p>
      <strong>
       AWS Builder ID
      </strong>
      ，是提供访问Amazon所选工具及服务(例如：
      <strong>
       Amazon CodeCatalyst, Amazon Q Developer, AWS Training and Certification
      </strong>
      等)的
      <strong>
       个人用户配置文件
      </strong>
      。
     </p>
    </blockquote>
    <p>
     AWS Builder ID代表个人独立于已注册AWS账户的任何凭据和数据。和其它个人资料一样，AWS Builder ID由个人保留，不管是否完成个人、教育和职业目标。
    </p>
    <p>
     众所周知，AWS账户可充当用户创建的AWS资源的容器，并为这些资源提供安全边界，但用户的AWS Builder ID代表用户个人。
    </p>
    <p>
     AWS Builder ID是
     <strong>
      免费的
     </strong>
     ，使用时仅需支付AWS账户中消耗的AWS相关资源。
    </p>
    <h4>
     <a id="2%09AWS_Builder_ID_14">
     </a>
     2. 创建AWS Builder ID
    </h4>
    <p>
     登录AWS工具及服务时，可以创建AWS Builder ID。创建该账户需要Email地址，姓名和密码。
    </p>
    <p>
     注册完毕，进入AWS Builder ID（中文名为“AWS构建者ID”），访问“我的详细信息”，查询个人资料信息，如下图所示：
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/723b603e86dc4b33a8d9a74d01a9db64.png"/>
    </p>
    <h4>
     <a id="3%09AWS_Builder_IDAWS_21">
     </a>
     3. 用AWS Builder ID启用AWS工具和服务
    </h4>
    <p>
     默认情况下,使用AWS Builder ID登录AWS工具或服务时,会话持续时间为30天（除了 Amazon Q Developer的会话持续时间为90天）。
    </p>
    <p>
     该会话结束后,需要再次登录，才能继续访问AWS工具和服务。
    </p>
    <p>
     可以使用AWS生成器ID登录，以访问以下AWS工具和服务。
    </p>
    <h5>
     <a id="1%09AWS_Cloud_Community_27">
     </a>
     1) AWS Cloud Community
    </h5>
    <p>
     <a href="https://community.aws" rel="nofollow">
      https://community.aws
     </a>
     是一个AWS构建者的社区平台，可用AWS Builder ID来访问。在此可以探索发现教育内容，分享个人思路和项目，评论他人帖子，关注喜欢的AWS构建者。
    </p>
    <p>
     AWS Cloud Community社区Web站点如下图所示。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1d3fb9e033e04eb3ae17e9ad7f473247.png"/>
    </p>
    <h5>
     <a id="2%09Amazon_CodeCatalyst_34">
     </a>
     2) Amazon CodeCatalyst
    </h5>
    <p>
     访问Amazon CodeCatalyst时，用户会创建AWS Builder ID，可选择一个别名，用以关联诸如问题，代码提交和提交请求等活动。
    </p>
    <p>
     可邀请其他人到你的Amazon CodeCatalyst空间，它含有工具，基础架构和必要的团队环境，用以构建成功项目。
    </p>
    <p>
     用户需要一个AWS账户来将项目部署到云端。
    </p>
    <p>
     Amazon CodeCatalyst官方文档站点如下图。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/508deb9cfed84b9dbdfec08f183a423d.png"/>
    </p>
    <h5>
     <a id="3%09AWS_Migration_Hub_45">
     </a>
     3) AWS Migration Hub
    </h5>
    <p>
     用AWS Builder ID可访问
     <strong>
      AWS Migration Hub
     </strong>
     （迁移集线器），如下图所示。
     <br/>
     AWS Migration Hub提供了一个单一的位置来发现用户现有的服务器、规划迁移，并跟踪每个应用程序迁移的状态。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6b9fc0f7886e4d0bb3bdd5ec723a22cd.png"/>
    </p>
    <h5>
     <a id="4%09Amazon_Q_Developer_50">
     </a>
     4) Amazon Q Developer
    </h5>
    <p>
     <strong>
      Amazon Q Developer
     </strong>
     是一个由生成式AI驱动的会话助手。该助手可以帮助用户
     <strong>
      理解、构建、扩展和操作
     </strong>
     AWS应用程序。
    </p>
    <p>
     Amazon Q Developer操作步骤，可详见Amazon Q Developer用户指南。
     <br/>
     以下是AWS Q Developer官方文档站点。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ca7aef7d022c4e40b78121c5c241e36d.png"/>
    </p>
    <h5>
     <a id="5%09AWS_rePost_58">
     </a>
     5) AWS re:Post
    </h5>
    <p>
     <strong>
      AWS re:Post
     </strong>
     为用户提供专家级技术指导，以便能够更快创新，以及改善AWS服务的运营效率。
    </p>
    <p>
     可用AWS Builder ID登录并在没有AWS账户或者信用卡的情况下，通过AWS re:Post加入社区。这里有热门话题，以及顶级贡献者，他们都在技术上各有千秋，对开发者交流非常方便。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6987e30200ab43e5b5c75a15daa338a4.png"/>
    </p>
    <h5>
     <a id="6%09AWS_Startups_66">
     </a>
     6) AWS Startups
    </h5>
    <p>
     用AWS Builder ID登录，可以加入AWS Startups，这样可以利用学习内容，工具和资源；也能够支持在AWS快速成长。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/63243544e479429e8be24157572eae74.png"/>
    </p>
    <h5>
     <a id="7%09AWS_Training_and_Certification_72">
     </a>
     7) AWS Training and Certification
    </h5>
    <p>
     <strong>
      AWS Training and Certification
     </strong>
     ( AWS培训和认证)网站
     <a href="https://www.aws.training/" rel="nofollow">
      https://www.aws.training/
     </a>
     ，可由AWS Builder ID一键登入访问；
     <br/>
     这里汇聚了所有AWS培训和认证内容。包括
     <em>
      <strong>
       AWS云从业者认证，AWS认证解决方案架构师，AWS认证开发者
      </strong>
     </em>
     等十几个AWS官方认证的培训资料和备考资料等。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/463f447348834e85b18a0fe1070287b2.png"/>
    </p>
    <h5>
     <a id="8%09Website_Registration_Portal_WRP_77">
     </a>
     8) Website Registration Portal (WRP)
    </h5>
    <p>
     <strong>
      Website Registration Portal
     </strong>
     (
     <strong>
      Web站点注册门户
     </strong>
     ，缩写
     <strong>
      WRP
     </strong>
     )可使用AWS Builder ID作为AWS营销网站的持久客户身份和注册配置文件。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6218a2881f664cb39a32d03a3fce5b60.png">
      <br/>
      如果需要的话，请注册新的网络研讨会，并查看注册或参加过的所有网络研讨会。
     </img>
    </p>
    <h5>
     <a id="9%09AWS_Builder_ID_83">
     </a>
     9) 删除AWS Builder ID
    </h5>
    <p>
     进入AWS配置文件页面
     <a href="https://profile.aws.amazon.com" rel="nofollow">
      https://profile.aws.amazon.com
     </a>
     ，选择AWS Builder ID页面隐私和数据，在其页面删除AWS Builder ID，确定执行。
    </p>
    <p>
     *注：该账户删除后，用户不再有权限访问AWS工具和服务。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/97758fd9fb804adfa34cb058636531d9.png">
      <br/>
      后续文章，将开启在AWS做进一步开发实践的探讨。
     </img>
    </p>
