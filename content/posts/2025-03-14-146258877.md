   <div class="htmledit_views" id="content_views">
    <p>
     CentOS 7误删GRUB2可以通过以下步骤恢复：
     <br/>
     <br/>
     进入救援模式
     <br/>
     <br/>
     1. 插入CentOS 7安装光盘，重启系统。在开机时按BIOS设置对应的按键（通常是F2等），将启动顺序调整为CD - ROM优先。
     <br/>
     <br/>
     2. 系统从光盘启动后，选择“Troubleshooting”，然后选择“Rescue a CentOS system”，按提示选择语言等设置，进入救援模式。
     <br/>
     <br/>
     切根与安装GRUB2
     <br/>
     4个选项选第一个
     <br/>
     1. 进入救援模式后，通常系统会自动将硬盘上的系统挂载到 /mnt/sysimage 目录，执行 chroot /mnt/sysimage 切换到系统根目录。
     <br/>
     <br/>
     2. 执行 grub2-install /dev/sda 命令来安装GRUB2到硬盘的MBR（如果是UEFI环境则不用指定设备）。
     <br/>
     <br/>
     安装内核与生成配置文件
     <br/>
     <br/>
     1. 挂载光盘，执行 mount /dev/sr0 /mnt 。如果光盘设备不是 /dev/sr0 ，需要根据实际情况修改。
     <br/>
     <br/>
     2. 执行 yum reinstall kernel 命令重新安装内核。
     <br/>
     <br/>
     3. 执行 grub2-mkconfig -o /boot/grub2/grub.cfg 命令重新生成GRUB2的配置文件。
     <br/>
     <br/>
     退出并重启
     <br/>
     <br/>
     执行 exit 命令退出 chroot 环境，再次执行 exit 退出救援模式，然后执行 reboot 命令重启系统，移除光盘，系统将从硬盘正常启动，此时GRUB2应该已恢复正常。
