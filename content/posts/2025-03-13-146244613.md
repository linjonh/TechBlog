    <p>
     Django的视图(view)是处理业务逻辑的核心，它负责处理用户的请求并返回响应数据。Django提供了两种编写视图的方式：基于函数的视图和基于类的视图。本章会详细介绍如何编写视图以及如何使用Django提供的通用类视图。
    </p>
    <h3 id="什么是视图view及其工作原理" name="什么是视图view及其工作原理">
     什么是视图(View)及其工作原理
    </h3>
    <p>
     Django的Web开发也遵循经典软件设计MVC模式开发的。View (视图) 主要根据用户的请求返回数据，用来展示用户可以看到的内容(比如网页，图片)，也可以用来处理用户提交的数据，比如保存到数据库中。Django的视图(
     <code>
      views.py
     </code>
     ）通常和URL路由(URLconf)一起工作的。服务器在收到用户通过浏览器发来的请求后，会根据用户请求的url地址和
     <code>
      urls.py
     </code>
     里配置的url-视图映射关系，去执行相应视图函数或视图类，从而返回给客户端响应数据。
    </p>
    <p>
     我们先看一个最简单的函数视图。当用户发来一个请求
     <code>
      request
     </code>
     时，我们通过
     <code>
      HttpResponse
     </code>
     打印出
     <code>
      Hello， World!
     </code>
     。
    </p>
    <pre><code># views.py
from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello， World!")
</code></pre>
    <p>
     <strong>
      提示
     </strong>
     ：每个视图函数的第一个默认参数都必需是
     <code>
      request
     </code>
     , 它是一个全局变量。Django把每个用户请求封装成了
     <code>
      request
     </code>
     对象，它包含里当前请求的所有信息，比如请求路径
     <code>
      request.path
     </code>
     , 当前用户
     <code>
      request.user
     </code>
     以及用户通过POST提交的数据
     <code>
      request.POST
     </code>
     。
    </p>
    <p>
     上面例子过于简单。在实际Web开发过程中，我们的View不仅要负责从数据库读写数据，还需要指定显示内容的模板，并提供模板渲染页面所需的内容对象(
     <code>
      context object
     </code>
     )。接下来我们要看一个更接近现实的案例。
    </p>
    <h3 id="接近现实的函数视图" name="接近现实的函数视图">
     接近现实的函数视图
    </h3>
    <p>
     我们依然以
     <code>
      blog
     </code>
     为例，需要编写两个视图函数，一个用于展示文章列表，一个用于展示文章详情，你的
     <code>
      urls.py
     </code>
     和
     <code>
      views.py
     </code>
     正常情况下应如下所示：
    </p>
    <pre><code># blog/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('blog/', views.index, name='index'),
    path('blog/articles/&lt;int:id&gt;/', views.article_detail, name='article_detail'),
]

# blog/views.py
from django.shortcuts import render, get_object_or_404
from .models import Article

# 展示所有文章
def index(request):
    latest_articles = Article.objects.all().order_by('-pub_date')
    return render(request, 'blog/article_list.html', {"latest_articles": latest_articles})

# 展示所有文章
def article_detail(request, id):
    article = get_object_or_404(Article, pk=id)
    return render(request, 'blog/article_detail.html', {"article": article})
</code></pre>
    <p>
     那么上面这段代码是如何工作的呢？
    </p>
    <ul>
     <li>
      当用户在浏览器输入
      <code>
       /blog/
      </code>
      时，URL收到请求后会调用视图
      <code>
       views.py
      </code>
      里的
      <code>
       index
      </code>
      函数，展示所有文章。
     </li>
     <li>
      当用户在浏览器输入
      <code>
       /blog/article/5/
      </code>
      时，URL不仅调用了
      <code>
       views.py
      </code>
      里的
      <code>
       article_detail
      </code>
      函数，而且还把参数文章id通过
      <code>
       &lt;int:id&gt;
      </code>
      括号的形式传递给了视图里的
      <code>
       article_detail
      </code>
      函数。。
     </li>
     <li>
      views.py里的index函数先提取要展示的数据对象列表
      <code>
       latest_articles
      </code>
      , 然后通过
      <code>
       render
      </code>
      方法传递给模板
      <code>
       blog/article_list.html
      </code>
      .。
     </li>
     <li>
      views.py里的
      <code>
       article_detail
      </code>
      方法先通过
      <code>
       get_object_or_404
      </code>
      方法和id调取某篇具体的文章对象article，然后通过render方法传递给模板
      <code>
       blog/article_detail.html
      </code>
      显示。
     </li>
    </ul>
    <p>
     在本例中，我们使用了视图函数里常用的2个便捷方法
     <code>
      render()
     </code>
     和
     <code>
      get_object_or_404()
     </code>
     。
    </p>
    <ul>
     <li>
      <code>
       render
      </code>
      方法有4个参数。第一个是
      <code>
       request
      </code>
      , 第二个是模板的名称和位置，第三个是需要传递给模板的内容, 也被称为
      <code>
       context object
      </code>
      。第四个参数是可选参数
      <code>
       content_type
      </code>
      （内容类型), 我们什么也没写。
     </li>
     <li>
      <code>
       get_object_or_404
      </code>
      方法第一个参数是模型Models或数据集queryset的名字，第二个参数是需要满足的条件（比如pk = id, title = ‘python’)。当需要获取的对象不存在时，给方法会自动返回Http 404错误。
     </li>
    </ul>
    <p>
     下图是模板的代码。模板可以直接调用通过视图传递过来的内容。
    </p>
    <pre><code># blog/article_list.html
 
{% block content %}
{% for article in latest_articles %}
     {<!-- -->{ article.title }}
     {<!-- -->{ article.pub_date }}
{% endfor %}
{% endblock %}

# blog/article_detail.html
{% block content %}
   {<!-- -->{ article.title }}
   {<!-- -->{ article.pub_date }}
   {<!-- -->{ article.body }}
{% endblock %}
</code></pre>
    <h3 id="更复杂的案例-视图处理用户提交的数据" name="更复杂的案例-视图处理用户提交的数据">
     更复杂的案例: 视图处理用户提交的数据
    </h3>
    <p>
     视图View不仅用于确定给客户展示什么内容，以什么形式展示，而且也用来处理用户通过表单提交的数据。我们再来看两个创建和修改文章的视图函数
     <code>
      article_create
     </code>
     和
     <code>
      article_update
     </code>
     ，看看它们是如何处理用户通过表单POST提交过来的数据。
    </p>
    <pre><code> from django.shortcuts import render, redirect, get_object_or_404
 from django.urls import reverse
 from .models import Article
 from .forms import ArticleForm
 
 # 创建文章
 def article_create(request):
     # 如果用户通过POST提交，通过request.POST获取提交数据
     if request.method == "POST":
         # 将用户提交数据与ArticleForm表单绑定
         form = ArticleForm(request.POST)
         # 表单验证，如果表单有效，将数据存入数据库
         if form.is_valid():
             form.save()
             # 创建成功，跳转到文章列表
             return redirect(reverse("blog:article_list"))
     else:
         # 否则空表单
         form = ArticleForm()
     return render(request, "blog/article_form.html", { "form": form, })

 # 更新文章
 def article_update(request, pk):
     # 从url里获取单篇文章的id值，然后查询数据库获得单个对象实例
     article = get_object_or_404(Article, pk=id)
     
     # 如果用户通过POST提交，通过request.POST获取提交数据
     if request.method == 'POST':
         # 将用户提交数据与ArticleForm表单绑定，进行验证
         form = ArticleForm(instance=article, data=request.POST)
         if form.is_valid():
             form.save()
             # 更新成功，跳转到文章详情
             return redirect(reverse("blog:article_detail", args=[pk,]))
     else:
         # 否则用实例生成表单
         form = ArticleForm(instance=article)

     return render(request, "blog/article_form.html", { "form": form, "object": article})
</code></pre>
    <p>
     我们给每一行代码添加了说明。值得一提的是在创建和更新文章时我们向模板传递了
     <code>
      form
     </code>
     这个变量，模板会根据我们自定义的Form类自动生成表单。我们还使用了自定义的Form类对用户提交的数据(
     <code>
      request.POST
     </code>
     )进行验证,并将通过验证的数据存入数据库。
    </p>
    <p>
     这里所使用
     <code>
      ArticleForm
     </code>
     实际上是非常简单的，仅包含了
     <code>
      title
     </code>
     和
     <code>
      body
     </code>
     两个字段。
    </p>
    <pre><code>#blog/forms.py
from .models import Article
from django import forms

class ArticleForm(forms.ModelForm):
     class Meta:
         model = Article
         fields = ['title', 'body']
</code></pre>
    <h3 id="基于函数的视图和基于类的视图" name="基于函数的视图和基于类的视图">
     基于函数的视图和基于类的视图
    </h3>
    <p>
     Django提供了两种编写视图的方式: 基于函数的视图(Function Base View, FBV)和基于类的视图(Class Based View, CBV)。前面案例中的
     <code>
      index
     </code>
     ，
     <code>
      article_detail
     </code>
     和
     <code>
      article_update
     </code>
     的方法都是基于函数的视图。函数视图的优点是比较直接，容易读者理解, 缺点是不便于继承和重用。
    </p>
    <p>
     基于类的视图以
     <code>
      class
     </code>
     定义，而不是函数视图的
     <code>
      def
     </code>
     定义。使用类视图后可以将视图对应的不同请求方式以类中的不同方法来区别定义(get方法处理GET请求，post方法处理POST请求)，相对于函数视图逻辑更清晰，代码也有更高的复用性。
    </p>
    <pre><code>from django.views.generic import View

class MyClassView(View):
    """类视图"""
    def get(self, request):
        """处理GET请求"""
        return render(request, 'register.html')

    def post(self, request):
        """处理POST请求"""
        return ...
</code></pre>
    <p>
     <strong>
      注意
     </strong>
     ：在URL配置文件中使用类视图时，需要使用
     <code>
      as_view()
     </code>
     将其伪装成方法：
    </p>
    <pre><code># blog/urls.py
from django.urls import path, re_path
from . import views
 
urlpatterns = [
    path('', views.MyClassView.as_view()),
]
</code></pre>
    <h3 id="django通用类视图" name="django通用类视图">
     Django通用类视图
    </h3>
    <p>
     在实际Web开发过程中，我们对不同的数据或模型总是反复进行以下同样的操作，使用通用的类视图可以大大简化我们的代码量。
    </p>
    <ul>
     <li>
      展示对象列表（比如所有用户，所有文章）
     </li>
     <li>
      查看某个对象的详细信息（比如用户资料，比如文章详情)
     </li>
     <li>
      通过表单创建某个对象（比如创建用户，新建文章）
     </li>
     <li>
      通过表单更新某个对象信息（比如修改密码，修改文字内容）
     </li>
     <li>
      用户填写表单提交后转到某个完成页面
     </li>
     <li>
      删除某个对象
     </li>
    </ul>
    <p>
     Django提供了很多通用的基于类的视图，来帮我们简化视图的编写。这些View与上述操作的对应关系如下:
    </p>
    <ul>
     <li>
      <p>
       展示对象列表（比如所有用户，所有文章）-
       <code>
        ListView
       </code>
      </p>
     </li>
     <li>
      <p>
       展示某个对象的详细信息（比如用户资料，比如文章详情) -
       <code>
        DetailView
       </code>
      </p>
     </li>
     <li>
      <p>
       通过表单创建某个对象（比如创建用户，新建文章）-
       <code>
        CreateView
       </code>
      </p>
     </li>
     <li>
      <p>
       通过表单更新某个对象信息（比如修改密码，修改文字内容）-
       <code>
        UpdateView
       </code>
      </p>
     </li>
     <li>
      <p>
       用户填写表单后转到某个完成页面 -
       <code>
        FormView
       </code>
      </p>
     </li>
     <li>
      <p>
       删除某个对象 -
       <code>
        DeleteView
       </code>
      </p>
     </li>
    </ul>
    <p>
     上述常用通用视图一共有6个，前2个属于展示类视图(Display view), 后面4个属于编辑类视图(Edit view)。下面我们就来看下这些通用视图是如何工作的，如何简化我们代码的。
    </p>
    <p>
     注意：如果你要使用Edit view，请务必在模型里里定义
     <code>
      get_absolute_url()
     </code>
     方法，否则会出现错误。这是因为通用视图在对一个对象完成编辑后，需要一个返回链接。
     <code>
      get_absolute_url()
     </code>
     可以为某个对象生成独一无二的url。
    </p>
    <h4 id="listview" name="listview">
     a. ListView
    </h4>
    <p>
     <code>
      ListView
     </code>
     用来展示一个对象的列表。它只需要一个参数模型名称即可。比如我们希望展示所有文章列表，我们的
     <code>
      views.py
     </code>
     可以简化为:
    </p>
    <pre><code># Create your views here.
from django.views.generic import ListView
from .models import Article

class IndexView(ListView):
    model = Article
</code></pre>
    <p>
     上述代码等同于:
    </p>
    <pre><code># 展示所有文章
def index(request):
    queryset = Article.objects.all()
    return render(request, 'blog/article_list.html', {"article_list": queryset})
</code></pre>
    <p>
     尽管我们只写了一行
     <code>
      model = Article
     </code>
     ,
     <code>
      ListView
     </code>
     实际上在背后做了很多事情：
    </p>
    <ul>
     <li>
      提取了需要显示的对象列表或数据集
      <code>
       queryset: Article.objects.all()
      </code>
     </li>
     <li>
      指定了用来显示对象列表的模板名称: 默认
      <code>
       app_name/model_name_list.html
      </code>
      , 即
      <code>
       blog/article_list.html
      </code>
      .
     </li>
     <li>
      指定了内容对象名称(context object name):默认值
      <code>
       object_list
      </code>
     </li>
    </ul>
    <p>
     你或许已经注意到了2个问题：需要显示的文章对象列表并没有按发布时间逆序排列，默认内容对象名称
     <code>
      object_list
     </code>
     也不友好。或许你也不喜欢默认的模板名字，还希望通过这个视图给模板传递额外的内容(比如现在的时间)。你可以轻易地通过重写
     <code>
      queryset
     </code>
     ,
     <code>
      template_name
     </code>
     和
     <code>
      context_object_name
     </code>
     来完成ListView的自定义。
    </p>
    <p>
     如果你还需要传递模型以外的内容，比如现在的时间，你还可以通过重写
     <code>
      get_context_data
     </code>
     方法传递额外的参数或内容。
    </p>
    <pre><code># Create your views here.
from django.views.generic import ListView
from .models import Article
from django.utils import timezone

class IndexView(ListView):

    queryset = Article.objects.all().order_by("-pub_date")
    template_name = 'blog/article_list.html'
    context_object_name = 'latest_articles'

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['now'] = timezone.now()
        return context
</code></pre>
    <p>
     如果上述的queryset还不能满足你的要求，比如你希望一个用户只看到自己发表的文章清单，你可以通过更具体的
     <code>
      get_queryset()
     </code>
     方法来返回一个需要显示的对象列表。
    </p>
    <pre><code># Create your views here.
from django.views.generic import ListView
from .models import Article
from django.utils import timezone

class IndexView(ListView):

    template_name = 'blog/article_list.html'
    context_object_name = 'latest_articles'

    def get_queryset(self):
        return Article.objects.filter(author=self.request.user).order_by('-pub_date')

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['now'] = timezone.now()
        return context
</code></pre>
    <h4 id="detailview" name="detailview">
     b. DetailView
    </h4>
    <p>
     <code>
      DetailView
     </code>
     用来展示一个具体对象的详细信息。它需要URL传递某个对象的具体参数（如id, pk, slug值）。本例中用来展示某篇文章详细内容的view可以简写为:
    </p>
    <pre><code># Create your views here.
from django.views.generic import DetailView
from .models import Article

class ArticleDetailView(DetailView):
    model = Article
</code></pre>
    <p>
     <code>
      DetailView
     </code>
     默认的模板是
     <code>
      app/model_name_detail.html
     </code>
     ,默认的内容对象名字
     <code>
      context_object_name
     </code>
     是model_name。本例中默认模板是
     <code>
      blog/article_detail.html
     </code>
     , 默认对象名字是
     <code>
      article
     </code>
     , 在模板里可通过 ``获取文章标题。
    </p>
    <p>
     你同样可以通过重写
     <code>
      queryset
     </code>
     ,
     <code>
      template_name
     </code>
     和
     <code>
      context_object_name
     </code>
     来完成DetailView的自定义。你还可以通过重写
     <code>
      get_context_data
     </code>
     方法传递额外的参数或内容。如果你指定了queryset, 那么返回的object是queryset.get(pk = id), 而不是model.objects.get(pk = id)。
    </p>
    <pre><code># Create your views here.
from django.views.generic import ListView，DetailView
from .models import Article
from django.utils import timezone

class ArticleDetailView(DetailView):

    queryset = Article.objects.all().order_by("-pub_date") # 一般不写
    template_name = 'blog/article_detail.html'
    context_object_name = 'article'

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['now'] = timezone.now()
        return context
</code></pre>
    <h4 id="createview" name="createview">
     c. CreateView
    </h4>
    <p>
     <code>
      CreateView
     </code>
     一般通过某个表单创建某个对象，通常完成后会转到对象列表。比如一个最简单的文章创建CreateView可以写成：
    </p>
    <pre><code>from django.views.generic.edit import CreateView
from .models import Article

class ArticleCreateView(CreateView):
    model = Article
    fields = ['title', 'body',]
</code></pre>
    <p>
     CreateView默认的模板是
     <code>
      model_name_form.html,
     </code>
     即
     <code>
      article_form.html
     </code>
     。这里CreateView还会根据
     <code>
      fields
     </code>
     自动生成表单字段。默认的context_object_name是
     <code>
      form
     </code>
     。模板代码如下图所示:
    </p>
    <pre><code># blog/article_form.html

&lt;form method="post"&gt;{% csrf_token %}
    {<!-- -->{ form.as_p }}
    &lt;input type="submit" value="Save" /&gt;
&lt;/form&gt;
</code></pre>
    <p>
     如果你不想使用默认的模板和默认的表单，你可以通过重写
     <code>
      template_name
     </code>
     和
     <code>
      form_class
     </code>
     来完成CreateView的自定义。
    </p>
    <p>
     对于
     <code>
      CreateView
     </code>
     , 重写它的
     <code>
      form_valid
     </code>
     方法不是必需，但很有用。当用户提交的数据是有效的时候，执行该方法。你可以通过定义此方法做些别的事情，比如发送邮件，存取额外的数据。
    </p>
    <pre><code>from django.views.generic.edit import CreateView
from .models import Article
from .forms import ArticleCreateForm

class ArticleCreateView(CreateView):
    model = Article
    template_name = 'blog/article_create_form.html'
    form_class = ArticleCreateForm

    def form_valid(self, form):
       form.do_sth()
       return super(ArticleCreateView, self).form_valid(form)
</code></pre>
    <p>
     form_valid方法一个常见用途就是就是将创建对象的用户与model里的user结合（需要用户先登录再提交)。见下面例子。
    </p>
    <pre><code>from django.contrib.auth.mixins import LoginRequiredMixin
from django.views.generic.edit import CreateView
from .models import Article

class ArticleCreate(LoginRequiredMixin, CreateView):
    model = Article
    fields = ['title', 'body']

    def form_valid(self, form):
        form.instance.author = self.request.user
        return super().form_valid(form)
</code></pre>
    <h4 id="updateview" name="updateview">
     d. UpdateView
    </h4>
    <p>
     UpdateView一般通过某个表单更新现有对象的信息，更新完成后会转到对象详细信息页面。它需要URL提供访问某个对象的具体参数（如pk, slug值）。比如一个最简单的文章更新的UpdateView如下所示。
    </p>
    <pre><code>from django.views.generic.edit import UpdateView
from .models import Article

class ArticleUpdateView(UpdateView):
    model = Article
    fields = ['title', 'body',]
</code></pre>
    <p>
     UpdateView和CreateView很类似，比如默认模板都是
     <code>
      model_name_form.html
     </code>
     , 因此它们可以共用一个模板。但是区别有两点:
    </p>
    <ul>
     <li>
      CreateView显示的表单是空表单，UpdateView中的表单会显示现有对象的数据。
     </li>
     <li>
      用户提交表单后，CreateView转向对象列表，UpdateView转向对象详细信息页面。
     </li>
    </ul>
    <p>
     你可以通过重写
     <code>
      template_name
     </code>
     和
     <code>
      form_class
     </code>
     来完成UpdateView的自定义。
    </p>
    <ul>
     <li>
      本例中默认的模板是
      <code>
       article_form.html
      </code>
      , 你可以改为
      <code>
       article_update_form.html
      </code>
      。
     </li>
     <li>
      虽然form_valid方法不是必需，但很有用。当用户提交的数据是有效的时候，你可以通过定义此方法做些别的事情，比如发送邮件，存取额外的数据。
     </li>
    </ul>
    <pre><code>from django.views.generic.edit import UpdateView
from .models import Article
from .forms import ArticleUpdateForm

class ArticleUpdateView(UpdateView):
    model = Article
    template_name = 'blog/article_update_form.html'
    form_class = ArticleUpdateForm

    def form_valid(self, form):
       form.do_sth()
       return super(ArticleUpdateView, self).form_valid(form)
</code></pre>
    <p>
     另一个进行UpdateView的常用自定义方法是
     <code>
      get_object
     </code>
     方法。比如你希望一个用户只能编辑自己发表的文章对象。当一个用户尝试编辑别人的文章时，返回http 404错误。这时候你可以通过更具体的
     <code>
      get_object()
     </code>
     方法来返回一个更具体的对象。代码如下:
    </p>
    <pre><code>from django.views.generic.edit import UpdateView
from .models import Article
from .forms import ArticleUpdateForm

class ArticleUpdateView(UpdateView):
    model = Article
    template_name = 'blog/article_update_form.html'
    form_class = ArticleUpdateForm
 
    def get_object(self, queryset=None):
        obj = super().get_object(queryset=queryset)
        if obj.author != self.request.user:
            raise Http404()
        return obj
</code></pre>
    <h4 id="formview" name="formview">
     e. FormView
    </h4>
    <p>
     FormView一般用来展示某个表单，而不是用于创建或更新某个模型对象。当用户输入信息未通过表单验证，显示错误信息。当用户输入信息通过表单验证提交后，跳到其它页面。使用FormView一般需要定义
     <code>
      template_name
     </code>
     ,
     <code>
      form_class
     </code>
     和
     <code>
      success_url
     </code>
     .
    </p>
    <p>
     见下面代码。
    </p>
    <pre><code># views.py - Use FormView
from myapp.forms import ContactForm
from django.views.generic.edit import FormView

class ContactView(FormView):
    template_name = 'contact.html'
    form_class = ContactForm
    success_url = '/thanks/'

    def form_valid(self, form):
        # This method is called when valid form data has been POSTed.
        # It should return an HttpResponse.
        form.send_email()
        return super().form_valid(form)
</code></pre>
    <h4 id="deleteview" name="deleteview">
     f. DeleteView
    </h4>
    <p>
     DeleteView一般用来删除某个具体对象。它要求用户点击确认后再删除一个对象。使用这个通用视图，你需要定义模型的名称model和成功删除对象后的返回的URL。默认模板是
     <code>
      myapp/model_confirm_delete.html
     </code>
     。默认内容对象名字是model_name。本例中为article。
    </p>
    <p>
     本例使用了默认的模板
     <code>
      blog/article_confirm_delete.html
     </code>
     ，删除文章后通过
     <code>
      reverse_lazy
     </code>
     方法返回到index页面。
    </p>
    <pre><code>from django.urls import reverse_lazy
from django.views.generic.edit import DeleteView
from .models import Article

class ArticleDelete(DeleteView):
    model = Article
    success_url = reverse_lazy('index')
</code></pre>
    <p>
     模板内容如下:
    </p>
    <pre><code># blog/article_confirm_delete.html

&lt;form method="post"&gt;{% csrf_token %}
    &lt;p&gt;Are you sure you want to delete "{<!-- -->{ article }}"?&lt;/p&gt;
    &lt;input type="submit" value="Confirm" /&gt;
&lt;/form&gt;
</code></pre>
