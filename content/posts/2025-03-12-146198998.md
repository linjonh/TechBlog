    <h1 class="title-article" id="articleContentId">
     Buildroot编译生成的output路径下 linux-headers-custom 与 linux-custom 有什么区别
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     在 Buildroot 生成的
     <code>
      output
     </code>
     目录下，
     <code>
      linux-headers-custom
     </code>
     和
     <code>
      linux-custom
     </code>
     是不同的组件，它们的作用如下：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         linux-headers-custom
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        这是
        <strong>
         Linux 内核头文件
        </strong>
        的存放目录，主要用于构建用户空间的 C 库（如 glibc、musl）和用户态程序（如编译某些需要内核头文件的库或驱动）。
       </li>
       <li>
        其中包含的是从 Linux 内核提取出来的头文件，而不是完整的内核源码。
       </li>
       <li>
        这些头文件用于用户空间和内核的接口，比如系统调用、设备驱动的 API 等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         linux-custom
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        这是
        <strong>
         完整的 Linux 内核源码目录
        </strong>
        ，用于编译实际运行的 Linux 内核。
       </li>
       <li>
        这个目录包含完整的内核源代码，配合
        <code>
         Buildroot
        </code>
        提供的
        <code>
         .config
        </code>
        配置文件，可以用于编译内核镜像（如
        <code>
         zImage
        </code>
        、
        <code>
         uImage
        </code>
        ）。
       </li>
       <li>
        <code>
         linux-custom
        </code>
        目录是 Buildroot 用于构建
        <strong>
         内核镜像
        </strong>
        的工作目录，其中可能包含编译后的内核和模块文件。
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     主要区别：
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        目录
       </th>
       <th>
        作用
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         linux-headers-custom
        </code>
       </td>
       <td>
        仅包含用于构建用户空间程序的内核头文件
       </td>
      </tr>
      <tr>
       <td>
        <code>
         linux-custom
        </code>
       </td>
       <td>
        包含完整的 Linux 内核源码，用于编译 Linux 内核
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      总结
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       如果你要修改 Linux 内核代码或配置内核功能，应修改
       <code>
        linux-custom
       </code>
       目录中的源码
      </strong>
      。
     </li>
     <li>
      <strong>
       如果你要调整用户空间程序的编译环境（如 glibc 需要不同的内核头文件版本），可能涉及
       <code>
        linux-headers-custom
       </code>
      </strong>
      ，但通常不需要手动修改它。
     </li>
    </ul>
