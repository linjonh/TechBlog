    <h3>
     <a id="MySQL_9">
     </a>
     新建MySQL连接
    </h3>
    <p>
     工具使用TCP/IP协议进行数据库连接，与navicate等数据库工具连接方式一致，若安装有其他数据库工具，可以参考其数据库连接信息进行填写。
    </p>
    <p>
     <strong>
      服务器：
     </strong>
     <br/>
     填写安装有mysql数据库的服务器地址。
     <br/>
     如果是连接安装在本地电脑上的数据库，可以填localhost或者127.0.0.1或者电脑的ip地址
     <br/>
     如果是连接安装在远程服务器上的数据库，需填写服务器ip地址
     <br/>
     <strong>
      端口：
     </strong>
     <br/>
     填写mysql端口号，默认安装端口号为3306，若安装时自定义了非默认端口号，请填写自定义的端口号
     <br/>
     <strong>
      用户名：
     </strong>
     <br/>
     填写允许连接到mysql数据库的用户名
     <br/>
     <strong>
      密码：
     </strong>
     <br/>
     填写所填用户名的对应密码，若连接报错，请检查密码是否填写正确。
     <br/>
     <strong>
      数据库：
     </strong>
     <br/>
     填写要导入数据的数据库名称。
    </p>
    <p>
     第一步：填写必要的连接信息并点击测试连接
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/da27221fa10fa2300991219f95adee20.png">
      <br/>
      第二步：提示连接成功后，填写连接名称并选择要导入的数据库，点击确定即可
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cf15f3704f75222980e4a0aa9653e415.png">
       <br/>
       <strong>
        常见问题：
       </strong>
       <br/>
       1.ERROR 1045: Access denied for user ‘root’@‘localhost’ (using password: YES)
       <br/>
       密码不对，请仔细核对密码并重新填写
      </img>
     </img>
    </p>
    <p>
     2.Host is not allowed to connect to this MySQL server
     <br/>
     连接远程数据库时，需要远程数据库设置允许用户进行远程连接，
     <br/>
     具体可以参考：
     <a href="https://blog.csdn.net/weixin_43989637/article/details/112009123">
      Host is not allowed to connect to this MySQL server
     </a>
    </p>
    <h3>
     <a id="Oracle_36">
     </a>
     新建Oracle连接
    </h3>
    <p>
     工具与navicate，pl/sql developer等数据库工具连接方式一致，若安装有其他数据库工具，可以参考其数据库连接信息进行填写。
     <br/>
     <strong>
      注意：
      <br/>
      1.工具所在目录请不要包含中文字符，否则无法连接
     </strong>
     <br/>
     <strong>
      服务器名称：
     </strong>
     <br/>
     填写安装有oracle数据库的服务器地址。
     <br/>
     如果是连接安装在本地电脑上的数据库，可以填LOCALHOST或者127.0.0.1或者电脑的ip地址
     <br/>
     如果是连接安装在远程服务器上的数据库，需填写服务器ip地址
     <br/>
     <strong>
      端口：
     </strong>
     <br/>
     填写oracle端口号，默认安装端口号为1521，若安装时自定义了非默认端口号，请填写自定义的端口号
     <br/>
     <strong>
      服务名/SID：
     </strong>
     <br/>
     填写服务名或者SID。
     <br/>
     <strong>
      用户名：
     </strong>
     <br/>
     填写允许连接到oracle数据库的用户名
     <br/>
     <strong>
      密码：
     </strong>
     <br/>
     填写所填用户名的对应密码，若连接报错，请检查密码是否填写正确。
     <br/>
     <strong>
      表所属用户：
     </strong>
     <br/>
     选择要导入导出的表归属用户，默认为连接用户，可以选择其他用户。
    </p>
    <p>
     第一步：填写必要的连接信息并点击测试连接
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d90ce67a3bfe415ea839eb00a881ba3f.png"/>
    </p>
    <p>
     第二步：提示连接成功后，填写连接名称，若要导入导出的表归属是非连接用户，则选择一个表所属用户，最后点击确定即可
    </p>
    <p>
     <strong>
      常见问题：
     </strong>
     <br/>
     1.DPI-1047: Cannot locate a 64-bit Oracle Client library
     <br/>
     工具所在目录不能包含中文，请将工具移动到纯英文目录下再重试
    </p>
    <h3>
     <a id="SQL_Server_64">
     </a>
     新建SQL Server连接
    </h3>
    <p>
     工具同时支持Windows身份验证和SQL Server身份验证，两种方式的填写方式不同，可以参考SQL Server Management Studio或者Navicate等其他数据库工具的数据库连接进行填写。
     <br/>
     一、Windows身份验证：
     <br/>
     <strong>
      服务器名称：
     </strong>
     <br/>
     可以填电脑的主机名称或者localhost或者127.0.0.1或者电脑的ip地址或者英文句号.
     <br/>
     端口默认为1433，默认可以不填。如果是非默认端口，则用逗号,跟在ip地址后面，如：192.168.126.111,15304
     <br/>
     <strong>
      用户名：
     </strong>
     <br/>
     默认为电脑主机名\电脑开机登录时的用户名
     <br/>
     <strong>
      密码：
     </strong>
     <br/>
     默认不填
     <br/>
     <strong>
      数据库：
     </strong>
     <br/>
     填写要导入的数据库名称。
     <br/>
     <strong>
      模式：
     </strong>
     <br/>
     创建数据库时默认模式为dbo，若要导入其他模式，请填写其他模式名
     <br/>
     SQL Server Management Studio Windows身份验证如下图
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3958218b833b9ba64c8671940037ab9d.png">
      <br/>
      第一步：直接点击测试连接
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/4abdae1da3fe5319b1f4e73903890be6.png">
       <br/>
       第二步：选择要导入数据的数据库和模式，填写连接名称，点击确定即可
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/069f763c7c60f2a014f7f8d60a3e0c81.png"/>
      </img>
     </img>
    </p>
    <p>
     二、SQL Server身份验证
     <br/>
     需要在安装数据库时指定了混合身份验证，否则无法连接
     <br/>
     <strong>
      服务器：
     </strong>
     <br/>
     填写安装有sql server数据库的服务器地址。
     <br/>
     如果是连接安装在本地电脑上的数据库，可以填电脑的主机名称或者localhost或者127.0.0.1或者电脑的ip地址
     <br/>
     如果是连接安装在远程服务器上的数据库，需填写服务器ip地址
     <br/>
     如果是非默认端口，则用逗号,跟在ip地址后面，如：192.168.126.111,15304
     <br/>
     <strong>
      用户名：
     </strong>
     <br/>
     填写允许连接到sql server数据库的用户名
     <br/>
     <strong>
      密码：
     </strong>
     <br/>
     填写所填用户名的对应密码，若连接报错，请检查密码是否填写正确。
     <br/>
     <strong>
      数据库：
     </strong>
     <br/>
     填写已创建好的数据库名称。
     <br/>
     <strong>
      模式：
     </strong>
     <br/>
     创建数据库时默认模式为dbo，若要导入其他模式，请填写其他模式名
     <br/>
     SQL Server Management Studio SQL Server身份验证如下图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/fabc67f9e35ae4a27f8c2165f5831028.png"/>
    </p>
    <p>
     第一步：身份验证选择SQL Server身份验证，填入连接信息，点击测试连接
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b760d5584fc037425a00162c237e3b54.png"/>
     <br/>
     第二步：选择要导入数据的数据库和模式，填写连接名称，点击确定即可
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/81ee7841fbfc062ba375d881fd8c7857.png"/>
    </p>
    <p>
     <strong>
      ODBC驱动程序：
     </strong>
     <br/>
     在高级选项界面的ODBC驱动程序选项，列出了你电脑上已安装的ODBC驱动程序，工具默认选择版本最高的一个来进行连接。若该选项为空，表示电脑上没有可用的ODBC驱动程序，需先安装ODBC驱动程序才能进行连接。若ODBC驱动程序版本过低，也可能无法连接或者出现导入失败等问题，请下载安装最新的ODBC驱动程序在重启工具重试。
     <a href="https://go.microsoft.com/fwlink/?linkid=2242886" rel="nofollow">
      下载 Microsoft ODBC Driver 18 for SQL Server (x64)
     </a>
    </p>
    <p>
     <strong>
      常见问题：
     </strong>
     <br/>
     1.点击测试连接报错，请检查数据库连接信息是否填写正确。
     <br/>
     并确认SQL Server的TCP/IP连接已开启
     <br/>
     开启方法可以参考：
     <a href="https://blog.csdn.net/Hehuyi_In/article/details/96122942">
      SQL Server的TCP/IP连接的开启方法
     </a>
     ，开启后最好重启电脑重试
    </p>

    </p>
    <h3>
     <a id="Hive_180">
     </a>
     新建Hive连接
    </h3>
    <p>
     工具与datagrip等数据库工具连接方式一致，若安装有其他数据库工具，可以参考其数据库连接信息进行填写。
     <br/>
     <strong>
      服务器：
     </strong>
     <br/>
     填写安装有hive数据库的服务器地址。
     <br/>
     如果是连接安装在本地电脑上的数据库，可以填localhost或者127.0.0.1或者电脑的ip地址
     <br/>
     如果是连接安装在远程服务器上的数据库，需填写服务器ip地址
     <br/>
     <strong>
      端口：
     </strong>
     <br/>
     填写hive端口号，默认安装端口号为10000，若安装时自定义了非默认端口号，请填写自定义的端口号
     <br/>
     <strong>
      用户名：
     </strong>
     <br/>
     填写允许连接到hive数据库的用户名
     <br/>
     <strong>
      密码：
     </strong>
     <br/>
     填写所填用户名的对应密码，若连接报错，请检查密码是否填写正确。
     <br/>
     <strong>
      数据库：
     </strong>
     <br/>
     填写已创建好的数据库名称。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/2f36df56368b42c04d4c9fd97e660fe7.png"/>
     <br/>
     高级选项：
     <br/>
     默认情况下，导入数据时使用insert into语句进行写入，该方式适合数据量较小的文件。
     <br/>
     如果数据量较大，建议使用load写入方式。
     <br/>
     使用load写入时，需要在高级选项界面填写安装hive所在服务器的服务器信息
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bb78cf9f1e44a18a31f16a112fa0d8e2.png"/>
     <br/>
     同时，首次使用需要使用winscp登录服务器并保存会话。
    </p>
