    <h2>
     <a id="__2">
     </a>
     一 什么是提示工程
    </h2>
    <p>
     【提示工程】也叫【指令工程】
     <br/>
     prompt就是给大模型发的指令，如“给我讲个笑话”
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5311ab207cc94238b46fe68f4c8356f6.png"/>
    </p>
    <h4>
     <a id="_6">
     </a>
     懂得提示工程原理会带来什么优势
    </h4>
    <p>
     <strong>
      懂得原理
     </strong>
     <br/>
     为什么有的指令是有效的，有的指令是无效的
     <br/>
     为什么同样的指令有时候有效，有时候无效
     <br/>
     怎么提升指令有效的概率
    </p>
    <h4>
     <a id="prompt_11">
     </a>
     使用prompt的两种目的
    </h4>
    <p>
     <strong>
      获得具体问题的具体结果
     </strong>
     <br/>
     如：php是不是世界上最好的语言
     <br/>
     <strong>
      固化一套prompt到程序中，成为系统功能的一部分
     </strong>
     <br/>
     如：生成本公司的简报，发送给老板
    </p>
    <h4>
     <a id="prompt_16">
     </a>
     高质量prompt核心要点
    </h4>
    <p>
     具体，丰富，少歧义
    </p>
    <h2>
     <a id="_prompt_19">
     </a>
     二 prompt的典型构成
    </h2>
    <p>
     <strong>
      如果底层大模型换了，prompt需要重新调优
     </strong>
     <br/>
     <strong>
      大模型对prompt开头和结尾的内容更加敏感，注意重要信息放在开头和结尾（lost in middle：how language use long context）
     </strong>
    </p>
    <h4>
     <a id="prompt_22">
     </a>
     prompt的典型构成
    </h4>
    <ul>
     <li>
      角色：给ai一个最匹配任务的角色，比如：【你是一个软件工程师】（大家玩出来的窍门，可以在开头把问题域收窄，减少二义性）
     </li>
     <li>
      指令：对任务进行描述
     </li>
     <li>
      上下文：给出任务相关的背景知识
     </li>
     <li>
      例子：必要时给出举例
     </li>
     <li>
      输入：任务的输入信息
     </li>
     <li>
      输出：输出的格式描述
     </li>
    </ul>

    <h4>
     <a id="_44">
     </a>
     核心思路
    </h4>
    <p>
     1、把输入的自然语言对话，转化为结构化的表示
     <br/>
     2、从结构化的表示，生成策略
     <br/>
     3、把策略转为自然语言输出
    </p>
    <h4>
     <a id="prompt_49">
     </a>
     用prompt进行实现
    </h4>
    <p>
     <strong>
      1、先用一个简单的提取用户输入关键词的prompt检测模型是否能理解需求功能，不能理解考虑换模型
     </strong>
    </p>
    <pre><code class="prism language-bash">你的任务是识别用户对于手机流量套餐产品的选择条件。
每种流量套餐产品包含三个属性：名称、月费价格、月流量
根据用户的输入，识别用户在上述三种属性上的倾向
</code></pre>
    <p>
     输入信息：办个100g的套餐
     <br/>
     输出：
    </p>
    <ul>
     <li>
      名称：用户倾向选择100g的套餐
     </li>
     <li>
      月费价格：用户未提及对月费价格的倾向
     </li>
     <li>
      月流量：用户倾向选择100g的套餐
     </li>
    </ul>
    <p>
     <strong>
      2、调整输出格式，方便后续处理
     </strong>
    </p>
    <pre><code class="prism language-bash">你的任务是识别用户对于手机流量套餐产品的选择条件。
每种流量套餐产品包含三个属性：名称<span class="token punctuation">(</span>name<span class="token punctuation">)</span>、月费价格<span class="token punctuation">(</span>price<span class="token punctuation">)</span>、月流量<span class="token punctuation">(</span>data<span class="token punctuation">)</span>
根据用户的输入，识别用户在上述三种属性上的倾向
以json格式输出
<span class="token number">1</span>.name字段取值为string类型，取值必须为以下之一：经济套餐，畅游套餐，无限套餐，校园套餐
<span class="token number">2</span>.price字段的取值为一个结构体或者null，包含两个字段：
（1）operator，string类型，取值范围为<span class="token string">'&lt;='</span>（小于等于<span class="token punctuation">)</span>,<span class="token string">'&gt;='</span><span class="token punctuation">(</span>大于等于<span class="token punctuation">)</span>
（2）value，int类型
<span class="token number">3</span>.data字段取值为结构体或者null，包含两个字段
（1）operator，string类型，取值范围为<span class="token string">'&lt;='</span>（小于等于<span class="token punctuation">)</span>,<span class="token string">'&gt;='</span><span class="token punctuation">(</span>大于等于<span class="token punctuation">)</span>
（2）value，int类型或string类型，string类型只能是’无上限‘
<span class="token number">4</span>.用户意图可以按照price或data排序，以sort字段标识，取值为一个结构体
（1）结构体以<span class="token string">"ordering"</span><span class="token operator">=</span><span class="token string">"descend"</span>表示降序排序，以”value“字段存储待排序的字段
（2）结构体以<span class="token string">"ordering"</span><span class="token operator">=</span><span class="token string">"ascend"</span>表示升序排序，以”value“字段存储待排序的字段
输出中只包含用户提及的字段，不要猜测任何用户没有直接提及的字段，不输出值为null的字段
</code></pre>
    <p>
     输入信息：我要个最便宜的套餐
     <br/>
     输出信息：
     <br/>
     {“name”:经济套餐}
    </p>
