    <p>
     SSL（
     <strong>
      Secure Sockets Layer
     </strong>
     ，安全套接层）认证是一种用于
     <strong>
      加密网络通信
     </strong>
     和
     <strong>
      验证服务器身份
     </strong>
     的安全技术。它是TLS（
     <strong>
      Transport Layer Security
     </strong>
     ，传输层安全协议）的前身，虽然现在大多数应用使用的是TLS，但仍习惯性地称之为SSL认证。
    </p>
    <h2>
     <a id="SSL__1">
     </a>
     SSL 认证
    </h2>
    <h3>
     <a id="SSL_2">
     </a>
     <strong>
      SSL认证的核心作用
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       数据加密
      </strong>
      ：防止数据在传输过程中被窃取或篡改，保证机密性。
     </li>
     <li>
      <strong>
       身份验证
      </strong>
      ：验证服务器身份，防止用户连接到伪造的服务器（如中间人攻击）。
     </li>
     <li>
      <strong>
       数据完整性
      </strong>
      ：确保传输的数据未被篡改，提高数据的可信度。
     </li>
    </ol>
    <h3>
     <a id="SSL_6">
     </a>
     <strong>
      SSL证书的工作原理
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       服务器部署SSL证书
      </strong>
      ：
      <ul>
       <li>
        服务器向CA（
        <strong>
         Certificate Authority
        </strong>
        ，证书颁发机构）申请SSL证书。
       </li>
       <li>
        CA验证服务器身份后颁发证书，包含公钥、服务器信息等。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       客户端（浏览器）与服务器建立SSL/TLS连接
      </strong>
      ：
      <ul>
       <li>
        客户端向服务器请求安全连接（HTTPS）。
       </li>
       <li>
        服务器发送SSL证书，客户端验证证书的有效性（是否由可信CA签发、是否过期等）。
       </li>
       <li>
        若验证通过，客户端和服务器进行
        <strong>
         密钥交换
        </strong>
        ，协商使用对称加密算法。
       </li>
       <li>
        之后，所有通信数据均被加密，确保安全。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="SSL_15">
     </a>
     <strong>
      SSL证书的类型
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       域名验证（DV）证书
      </strong>
      ：仅验证域名所有权，成本低，适合个人网站。
     </li>
     <li>
      <strong>
       企业验证（OV）证书
      </strong>
      ：验证企业身份，适用于公司官网、B2B平台。
     </li>
     <li>
      <strong>
       扩展验证（EV）证书
      </strong>
      ：最高级别的身份认证，适用于金融、电商等对安全要求高的网站。
     </li>
     <li>
      <strong>
       通配符（Wildcard）证书
      </strong>
      ：可用于多个子域名（如
      <code>
       *.example.com
      </code>
      ）。
     </li>
     <li>
      <strong>
       多域名（SAN）证书
      </strong>
      ：支持多个不同的域名绑定在同一证书上。
     </li>
    </ol>
    <h3>
     <a id="SSL_21">
     </a>
     <strong>
      常见的SSL证书提供商
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       DigiCert
      </strong>
     </li>

      <strong>
       GlobalSign

     </li>
     <li>

       Let’s Encrypt（免费证书）
      </strong>
     </li>
     <li>
      <strong>
       GeoTrust
      </strong>
     </li>
    </ul>
    <h3>
     <a id="SSL_27">
     </a>
     <strong>
      如何检查网站是否使用SSL认证？
     </strong>
    </h3>
    <ul>
     <li>
      看浏览器地址栏是否以**
      <code>
       https://
      </code>
      **开头。
     </li>
     <li>
      查看浏览器的🔒锁🔒图标，点击可查看证书详情。
     </li>
     <li>
      使用
      <code>
       openssl
      </code>
      命令检查：
      <pre><code class="prism language-bash">openssl s_client <span class="token parameter variable">-connect</span> example.com:443
</code></pre>
     </li>
    </ul>
    <h3>
     <a id="SSLTLS_34">
     </a>
     <strong>
      SSL与TLS的区别
     </strong>
    </h3>
    <ul>
     <li>
      SSL 2.0 和 SSL 3.0 已被废弃，TLS 1.2 和 TLS 1.3 是当前主流协议。
     </li>
     <li>
      TLS 比 SSL 更安全，支持更强的加密算法和防御机制（如防止BEAST、POODLE等攻击）。
     </li>
    </ul>
    <h3>
     <a id="_37">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <p>
     SSL认证是保护网站数据安全的关键技术，网站通过部署SSL证书实现加密通信、身份验证和数据完整性。现代互联网基本都采用
     <strong>
      TLS协议
     </strong>
     ，但仍常称为SSL证书。
    </p>
    <h2>
     <a id="TLS_39">
     </a>
     TLS
    </h2>
    <h3>
     <a id="TLSTransport_Layer_Security_40">
     </a>
     <strong>
      TLS（Transport Layer Security，传输层安全协议）
     </strong>
    </h3>
    <p>
     TLS（传输层安全协议）是
     <strong>
      SSL（Secure Sockets Layer，安全套接层）
      <strong>
       的升级版本，用于在网络通信中提供
      </strong>
      加密、数据完整性和身份验证
     </strong>
     。TLS 目前是互联网安全通信的主流标准，广泛应用于
     <strong>
      HTTPS、邮件传输、VPN、VoIP 等场景
     </strong>
     。
    </p>
    <h3>
     <a id="TLS_42">
     </a>
     <strong>
      TLS的主要作用
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       加密数据
      </strong>
      ：防止数据在传输过程中被窃取或篡改，保障通信的机密性。
     </li>
     <li>
      <strong>
       身份认证
      </strong>
      ：使用
      <strong>
       数字证书
      </strong>
      （如 SSL 证书）验证服务器（或客户端）的真实性，防止中间人攻击。
     </li>
     <li>
      <strong>
       数据完整性
      </strong>
      ：使用消息认证码（MAC）确保数据在传输过程中没有被篡改。
     </li>
    </ol>
    <h3>
     <a id="TLS_46">
     </a>
     <strong>
      TLS的工作流程
     </strong>
    </h3>
    <p>
     TLS 建立安全连接的过程称为
     <strong>
      TLS握手（TLS Handshake）
     </strong>
     ，通常包括以下步骤：
    </p>
    <h4>
     <a id="1_Hello_48">
     </a>
     <strong>
      1. 客户端向服务器发送“Hello”
     </strong>
    </h4>
    <ul>
     <li>
      客户端（如浏览器）发送
      <code>
       ClientHello
      </code>
      ，包含支持的
      <strong>
       TLS 版本、加密算法（Cipher Suites）
      </strong>
      和
      <strong>
       随机数（Client Random）
      </strong>
      。
     </li>
    </ul>
    <h4>
     <a id="2_Hello_50">
     </a>
     <strong>
      2. 服务器回应“Hello”
     </strong>
    </h4>
    <ul>
     <li>
      服务器返回
      <code>
       ServerHello
      </code>
      ，选择加密算法，并发送
      <strong>
       数字证书（SSL 证书）
      </strong>
      ，其中包含服务器的公钥和身份信息。
     </li>
    </ul>
    <h4>
     <a id="3__52">
     </a>
     <strong>
      3. 证书验证
     </strong>
    </h4>
    <ul>
     <li>
      客户端验证证书是否由**受信任的CA（证书颁发机构）**签发，是否过期，是否匹配访问的域名。
     </li>
     <li>
      如果证书无效，连接可能会被阻止（浏览器会显示**“不安全”警告**）。
     </li>
    </ul>
    <h4>
     <a id="4__55">
     </a>
     <strong>
      4. 密钥交换
     </strong>
    </h4>
    <ul>
     <li>
      客户端生成
      <strong>
       会话密钥（Session Key）
      </strong>
      ，并使用服务器的公钥加密发送给服务器（或使用 DH/ECDH 进行密钥协商）。
     </li>
     <li>
      服务器使用私钥解密，获得相同的会话密钥。
     </li>
    </ul>
    <h4>
     <a id="5__58">
     </a>
     <strong>
      5. 生成会话密钥，开始安全通信
     </strong>
    </h4>
    <ul>
     <li>
      客户端和服务器使用商定的**对称加密算法（如 AES）**进行数据加密，后续通信使用该会话密钥加密数据。
     </li>
    </ul>
    <h3>
     <a id="TLS_60">
     </a>
     <strong>
      TLS版本
     </strong>
    </h3>

       <th>

        发布时间
       </th>
       <th>

       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        TLS 1.0
       </td>

        1999

       </td>

       </td>

       </td>
       <td>
        增加对 CBC（Cipher Block Chaining）模式的安全改进，已弃用。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         TLS 1.2
        </strong>
       </td>
       <td>
        <strong>
         2008
        </strong>
       </td>
       <td>
        目前广泛使用的版本，支持更强的加密算法（AES-GCM），引入 HMAC 进行消息认证。
       </td>
      </tr>

       <td>
        <strong>
         TLS 1.3
        </strong>
       </td>
       <td>
        <strong>
         2018
        </strong>
       </td>

       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      🔥
      <strong>
       TLS 1.3 是当前最安全、性能最优的版本，推荐使用！
      </strong>
     </p>
    </blockquote>
    <h3>
     <a id="TLS_vs_SSL_69">
     </a>
     <strong>
      TLS vs SSL
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        对比
       </th>
       <th>
        SSL
       </th>
       <th>
        TLS
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        安全性
       </td>
       <td>
        存在较多漏洞（如 POODLE）
       </td>
       <td>
        更安全，移除不安全算法
       </td>
      </tr>
      <tr>
       <td>
        速度
       </td>
       <td>
        握手过程较慢
       </td>
       <td>
        TLS 1.3 速度更快（减少 RTT）
       </td>
      </tr>
      <tr>
       <td>
        版本
       </td>
       <td>
        最高为 SSL 3.0（已弃用）
       </td>
       <td>
        最新版为 TLS 1.3
       </td>
      </tr>
      <tr>
       <td>
        加密算法
       </td>
       <td>
        支持较旧的加密算法
       </td>
       <td>
        仅支持强加密（如 AES-GCM、ChaCha20）
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      💡
      <strong>
       TLS 1.0 和 1.1 已被废弃，TLS 1.2 和 TLS 1.3 是当前主流标准！
      </strong>
     </p>
    </blockquote>
    <h3>
     <a id="TLS_78">
     </a>
     <strong>
      TLS应用场景
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       HTTPS（安全网页通信）
      </strong>
      ：如
      <code>
       https://example.com
      </code>
     </li>
     <li>
      <strong>
       VPN（虚拟专用网络）
      </strong>
      ：如 OpenVPN
     </li>
     <li>
      <strong>
       电子邮件
      </strong>
      ：如 SMTPS、IMAPS、POP3S
     </li>
     <li>
      <strong>
       VoIP（网络电话）
      </strong>
      ：如 SIP over TLS
     </li>
     <li>
      <strong>
       消息传输
      </strong>
      ：如 MQTT over TLS
     </li>
    </ul>
    <h3>
     <a id="TLS_84">
     </a>
     <strong>
      如何检查TLS版本？
     </strong>
    </h3>
    <h4>
     <a id="1__OpenSSL_85">
     </a>
     <strong>
      1. 使用 OpenSSL
     </strong>
    </h4>
    <pre><code class="prism language-bash">openssl s_client <span class="token parameter variable">-connect</span> example.com:443 <span class="token parameter variable">-tls1_3</span>
</code></pre>
    <p>
     如果服务器支持 TLS 1.3，会成功建立连接，否则会失败。
    </p>
    <h4>
     <a id="2__90">
     </a>
     <strong>
      2. 使用浏览器
     </strong>
    </h4>
    <ul>
     <li>
      在
      <strong>
       Chrome/Firefox
      </strong>
      中，访问
      <code>
       https://example.com
      </code>
     </li>
     <li>
      点击🔒锁图标 &gt; 查看证书 &gt; 检查
      <strong>
       TLS 版本
      </strong>
     </li>
    </ul>
    <h3>
     <a id="_93">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       TLS 是 SSL 的升级版
      </strong>
      ，比 SSL 更安全、更高效，目前广泛用于 HTTPS 和其他加密通信。
     </li>
     <li>
      <strong>
       TLS 1.2 和 TLS 1.3 是当前主流标准
      </strong>
      ，TLS 1.3 具有更快的握手速度和更强的安全性。
     </li>
     <li>
      <strong>
       检查网站是否支持 TLS
      </strong>
      ，可以使用
      <code>
       openssl
      </code>
      或浏览器查看证书。
     </li>
    </ul>
    <blockquote>
     <p>
      🚀
      <strong>
       建议所有应用尽快升级到 TLS 1.3，确保安全！
      </strong>
     </p>
    </blockquote>
