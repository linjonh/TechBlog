        一：🔥 计算机网络背景
       </font>
      </h2>
      <h3>
       <a id="_11__10">
       </a>
       🦋 1-1 网络发展
      </h3>
      <img src="https://i-blog.csdnimg.cn/direct/26b3e67962184983808e1f35bc136fb4.png#pic_center" width="70%">
       <br/>
       <p>
        🛜
        <strong>
         网络互联: 多台计算机连接在一起, 完成数据共享;
        </strong>
       </p>
       <img src="https://i-blog.csdnimg.cn/direct/9e0ee5f2c5314b61b5d3f92fdd0aea6d.png#pic_center" width="80%">
        <br/>
        <p>
         <strong>
          <code>
           局域网 （Local Area Network，LAN）
          </code>
          : 计算机数量更多了, 通过交换机和路由器连接在一起；
         </strong>
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b26748841eac45d3a05d7e3c8190f619.png">
          <br/>
          <br/>
         </img>
        </p>
        <p>
         📚
         <strong>
          <code>
           广域网 （Wide Area Network，WAN）
          </code>
          : 将远隔千里的计算机都连在一起;
         </strong>
        </p>
        <p>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0722dad221a4488ca92eeef1f2e63572.png">
          <br/>
          <br/>
         </img>
        </p>
        <p>
         <strong>
          所谓 “局域网” 和 “广域网” 只是一个相对的概念. 比如, 我们有 “天朝特色” 的广域网, 也可以看做一个比较大的局域网.
         </strong>
        </p>
        <ul>
         <li>
          <strong>
           计算机是人的工具， 人要协同工作， 注定了网络的产生是必然的
          </strong>
         </li>
        </ul>
        <h2>
         <a id="font_colorDarkOrchid___font_36">
         </a>
         <font color="DarkOrchid">
          二：🔥 初识协议
         </font>
        </h2>
        <ul>
         <li>
          <strong>
           "
           <code>
            协议
           </code>
           " 是一种约定.
          </strong>
         </li>
         <li>
          <strong>
           打电话约定电话铃响的次数的约定
          </strong>
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2233f60c50604db28533286f3f17b146.png">
           <br/>
           <strong>
            计算机之间的传输媒介是 光信号 和 电信号. 通过 “频率” 和 “强弱” 来表示 0 和 1 这样的信息. 要想传递各种不同的信息, 就需要约定好双方的数据格式.
           </strong>
          </img>
         </li>
        </ul>
        <blockquote>
         <p>
          <strong>
           🎯 思考: 只要通信的两台主机, 约定好协议就可以了么?
          </strong>
         </p>
         <ul>
          <li>
           <strong>
            定好协议， 但是你用频率表示 01， 我用强弱表示 01， 就好比我用中国话， 你用葡萄牙语一样， 虽然大家可能遵守的一套通信规则， 但是语言不同，即是订好了基本的协议， 也是无法正常通信的。
           </strong>
           <br/>
           <br/>
           <strong>
            <font color="#C71585">
             所以， 完善的协议， 需要更多更细致的规定， 并让参与的人都要遵守。
            </font>
           </strong>
          </li>
          <li>
           <strong>
            计算机生产厂商有很多;
           </strong>
          </li>
          <li>
           <strong>
            计算机操作系统, 也有很多;
           </strong>
          </li>
          <li>
           <strong>
            计算机网络硬件设备, 还是有很多;
           </strong>
          </li>
          <li>
           <strong>
            <font color="#C71585">
             如何让这些不同厂商之间生产的计算机能够相互顺畅的通信? 就需要有人站出来, 约定一个共同的标准, 大家都来遵守, 这就是
             <code>
              网络协议
             </code>
             ;
            </font>
           </strong>
          </li>
         </ul>
        </blockquote>
        <p>
         📚
         <strong>
          网络协议的核心作用包括：
         </strong>
        </p>
        <ul>
         <li>
          <strong>
           <code>
            标准化
           </code>
           ：使不同厂商的设备能够互联互通。
          </strong>
         </li>
         <li>
          <strong>
           <code>
            规范性
           </code>
           ：定义了数据在传输过程中的行为。
          </strong>
         </li>
         <li>
          <strong>
           <code>
            可靠性
           </code>
           ：通过校验和重传机制，确保数据正确传输。
          </strong>
         </li>
        </ul>
        <p>
         📚
         <strong>
          举例：
         </strong>
        </p>
        <ul>
         <li>
          <strong>
           <code>
            HTTP协议
           </code>
           ：定义了网页浏览器与服务器之间的通信规则。
          </strong>
         </li>
         <li>
          <strong>
           <code>
            TCP协议
           </code>
           ：确保数据在传输中无错误且按顺序到达。
          </strong>
         </li>
        </ul>
        <h3>
         <a id="_21__64">
         </a>
         🦋 2-1 协议分层
        </h3>
        <p>
         <strong>
          <font color="#FF4500">
           网络通信是一个复杂的过程，为了降低设计难度并提高效率，网络协议通常采用分层设计的理念。分层设计将复杂的通信任务划分为若干独立的功能模块，每个模块专注于特定任务。
          </font>
         </strong>
        </p>
        <img src="https://i-blog.csdnimg.cn/direct/046edab52e694c1eb07950feed334114.png#pic_center" width="60%">
         <ul>
          <li>
           <strong>
            在这个例子中, 我们的"协议"只有两层:语言层、 通信设备层。
           </strong>
          </li>
          <li>
           <strong>
            但是实际的网络通信协议， 设计的会更加复杂, 需要分更多的层
           </strong>
          </li>
          <li>
           <strong>
            但是通过上面的简单例子， 我们是能理解， 分层可以实现解耦合， 让软件维护的成本更低
           </strong>
          </li>
         </ul>
         <p>
          <strong>
           协议分层的特点：
          </strong>
         </p>
         <ol>
          <li>
           <p>
            <strong>
             <code>
              模块化设计
             </code>
            </strong>
            <br/>
            <strong>
             每一层完成特定功能，互不干扰。例如，传输层关注数据的可靠传输，而物理层则负责信号的传递。
            </strong>
           </p>
          </li>
          <li>
           <p>
            <strong>
             <code>
              接口标准化
             </code>
            </strong>
            <br/>
            <strong>
             各层之间通过接口传递数据和信息，无需了解其他层的具体实现。例如，应用层无需关注底层硬件如何工作。
            </strong>
           </p>
          </li>
          <li>
           <p>
            <strong>
             <code>
              便于开发与维护
             </code>
            </strong>
            <br/>
            <strong>
             协议的分层设计使得开发者可以专注于某一层的优化，而不影响其他层。
            </strong>
           </p>
          </li>
         </ol>
         <h4>
          <a id="_vs__87">
          </a>
          协议分层 vs. 软件分层
         </h4>
         <p>
          协议分层与操作系统的软件分层理念相似。例如，在Linux系统中：
         </p>
         <ul>
          <li>
           <strong>
            <font color="#C71585">
             操作系统的分层包括内核层、硬件抽象层和用户层。
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#C71585">
             协议的分层则对应了从物理传输到应用服务的不同阶段。
            </font>
           </strong>
          </li>
         </ul>
         <p>
          二者的共同点在于通过分层实现复杂任务的简化和模块化。例如：
         </p>
         <ul>
          <li>
           <strong>
            <font color="#C71585">
             协议分层：物理层负责电信号传输，传输层确保数据可靠到达。
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#C71585">
             软件分层：驱动程序负责硬件控制，用户应用只需调用接口。
            </font>
           </strong>
          </li>
         </ul>
         <p>
          通过分层设计，网络协议不仅实现了多厂商设备的互通，还为Linux等操作系统提供了结构化的网络通信支持。
         </p>
         <h3>
          <a id="_22_OSI_100">
          </a>
          🦋 2-2 OSI七层网络模型
         </h3>
         <p>
          <strong>
           <code>
            OSI（Open Systems Interconnection）
           </code>
          </strong>
          模型是国际标准化组织（ISO）提出的网络通信框架，是一种理论上的参考模型，用于定义网络通信的结构和分层。
         </p>
         <p>
          OSI 模型将网络通信划分为七个层次，从硬件传输到应用服务，层与层之间通过标准接口交互。其核心目的是规范和简化不同设备、协议之间的通信，促进互联互通。
         </p>
         <p>
          它的最大优点是将服务、 接口和协议这三个概念明确地区分开来， 概念清楚，理论也比较完整. 通过七个层次化的结构模型使不同的系统不同的网络之间实现可靠的通讯;
         </p>
         <p>
          • 但是, 它既复杂又不实用; 所以我们按照 TCP/IP 四层模型来讲解
         </p>
         <table>
          <thead>
           <tr>
            <th align="left">
             层次
            </th>
            <th align="left">
             主要功能
            </th>
            <th align="center">
             示例协议/设备
            </th>
           </tr>
          </thead>
          <tbody>
           <tr>
            <td align="left">
             应用层
            </td>
            <td align="left">
             面向用户，提供网络服务接口，如文件传输、邮件、远程登录等。
            </td>
            <td align="center">
             HTTP、FTP、SMTP
            </td>
           </tr>
           <tr>
            <td align="left">
             表示层
            </td>
            <td align="left">
             数据格式的转换、加密、解密和压缩，确保不同设备之间的数据表示一致。
            </td>
            <td align="center">
             JPEG、GIF、TLS
            </td>
           </tr>
           <tr>
            <td align="left">
             会话层
            </td>
            <td align="left">
             管理会话建立、维持和断开，如数据同步和传输控制。
            </td>
            <td align="center">
             NetBIOS、RPC
            </td>
           </tr>
           <tr>
            <td align="left">
             传输层
            </td>
            <td align="left">
             提供端到端的可靠数据传输和错误检测，确保数据完整到达目标端。
            </td>
            <td align="center">
             TCP、UDP
            </td>
           </tr>
           <tr>
            <td align="left">
             网络层
            </td>
            <td align="left">
             负责路由和逻辑地址（IP）的分配，确保数据能跨网络到达目标设备。
            </td>
            <td align="center">
             IP、ICMP、IGRP
            </td>
           </tr>
           <tr>
            <td align="left">
             数据链路层
            </td>
            <td align="left">
             负责数据帧的组装、传输与错误检测，在局域网中完成数据的正确传输。
            </td>
            <td align="center">
             Ethernet、PPP、Wi-Fi
            </td>
           </tr>
           <tr>
            <td align="left">
             物理层
            </td>
            <td align="left">
             负责比特流的传输，定义物理介质（如光纤、电缆）和信号规范（如电压、频率）。
            </td>
            <td align="center">
             光纤、网线、无线信号
            </td>
           </tr>
          </tbody>
         </table>
         <p>
          其实在网络角度， OSI 定的协议 7 层模型其实非常完善， 但是在实际操作的过程中， 会话层、 表示层是不可能接入到操作系统中的， 所以在工程实践中， 最终落地的是 5 层协议。
         </p>
         <ul>
          <li>
           但是要理解上面的话， 需要我们学习完网络才可以理解， 这里就知道就可以
          </li>
         </ul>
         <h3>
          <a id="_23_TCPIP__122">
          </a>
          🦋 2-3 TCP/IP 五层(或四层)模型
         </h3>
         <p>
          TCP/IP 是一组协议的代名词， 它还包括许多协议， 组成了 TCP/IP 协议簇.
          <br/>
          TCP/IP 通讯协议采用了 5 层的层级结构， 每一层都呼叫它的下一层所提供的网络来完成自己的需求
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             <code>
              物理层
             </code>
             : 负责光 / 电信号的传递方式. 比如现在以太网通用的网线(双绞 线)、 早期以太网采用的的同轴电缆(现在主要用于有线电视)、 光纤, 现在的 wifi 无线网使用电磁波等都属于物理层的概念。 物理层的能力决定了最大传输速率、 传输距离、 抗干扰性等. 集线器 (Hub) 工作在物理层.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              数据链路层
             </code>
             : 负责设备之间的数据帧的传送和识别. 例如网卡设备的驱动、 帧同步(就是说从网线上检测到什么信号算作新帧的开始)、 冲突检测(如果检测到冲突就自动重发)、 数据差错校验等工作. 有以太网、 令牌环网, 无线 LAN 等标准. 交换机 (Switch) 工作在数据链路层.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              网络层
             </code>
             : 负责地址管理和路由选择. 例如在 IP 协议中, 通过 IP 地址来标识一台主机, 并通过路由表的方式规划出两台主机之间的数据传输的线路(路由). 路由器 (Router) 工作在网路层.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              传输层
             </code>
             : 负责两台主机之间的数据传输. 如传输控制协议 (TCP), 能够确保数据可靠的从源主机发送到目标主机.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              应用层
             </code>
             : 负责应用程序间沟通，如简单电子邮件传输（SMTP）、文件传输协议（FTP）、网络远程访问协议（Telnet） 等. 我们的网络编程主要就是针对应用层.
            </strong>
           </li>
          </ul>
         </blockquote>
         <p>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6cb6a47514ea46f09070df49d4fd4659.png"/>
          <br/>
          物理层我们考虑的比较少， 我们只考虑软件相关的内容. 因此很多时候我们直接称为 TCP / IP 四层模型.
         </p>
         <p>
          一般而言
         </p>
         <ul>
          <li>
           <strong>
            <font color="DarkOrchid">
             对于一台主机, 它的操作系统内核实现了从传输层到物理层的内容;
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="DarkOrchid">
             对于一台路由器, 它实现了从网络层到物理层;
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="DarkOrchid">
             对于一台交换机, 它实现了从数据链路层到物理层;
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="DarkOrchid">
             对于集线器, 它只实现了物理层;
            </font>
           </strong>
           <br/>
           但是并不绝对. 很多交换机也实现了网络层的转发; 很多路由器也实现了部分传输层的内容(比如端口转发);
          </li>
         </ul>

          🦋 5-1 理解源 IP 地址和目的 IP 地址
         </h3>
         <ul>
          <li>
           IP 在网络中， 用来标识主机的唯一性
          </li>
          <li>

         </p>
         <p>
          但是人是怎么看到聊天信息的呢？ 怎么执行下载任务呢？ 怎么浏览网页信息呢？ 通过启动的 qq， 迅雷， 浏览器。
         </p>
         <p>
          而启动的 qq， 迅雷， 浏览器都是进程。 换句话说， 进程是人在系统中的代表， 只要把数据给进程， 人就相当于就拿到了数据。
         </p>
         <p>
          <strong>
           所以： 数据传输到主机不是目的， 而是手段。 到达主机内部， 在交给主机内的进程，才是目的。
          </strong>
         </p>
         <p>
          但是系统中， 同时会存在非常多的进程， 当数据到达目标主机之后， 怎么转发给目标进程？ 这就要在网络的背景下， 在系统中， 标识主机的唯一性。
         </p>
         <p>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8765e6cfe18a4bfbb3b6692cd74324c3.png"/>
         </p>
         <h3>
          <a id="_52__285">
          </a>
          🦋 5-2 认识端口号
         </h3>
         <p>
          端口号(port)是传输层协议的内容.
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             端口号是一个 2 字节 16 位的整数;
            </strong>
           </li>
           <li>
            <strong>
             端口号用来标识一个进程, 告诉操作系统, 当前的这个数据要交给哪一个进程来处理;
            </strong>
           </li>
           <li>
            <strong>
             IP 地址 + 端口号能够标识网络上的某一台主机的某一个进程;
            </strong>
           </li>
           <li>
            <strong>
             一个端口号只能被一个进程占用
            </strong>
           </li>
          </ul>
         </blockquote>
         <p>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5b25ecc8e1e042658977e68ad00fe219.png"/>
         </p>
         <h4>
          <a id="521__293">
          </a>
          5-2-1 端口号范围划分
         </h4>
         <blockquote>
          <ul>
           <li>
            <strong>
             <code>
              0 - 1023: 知名端口号
             </code>
             , HTTP, FTP, SSH 等这些广为使用的应用层协议, 他们的端口号都是固定的.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              1024 - 65535
             </code>
             : 操作系统动态分配的端口号. 客户端程序的端口号, 就是由操作系统从这个范围分配的.
            </strong>
           </li>
          </ul>
         </blockquote>
         <h4>
          <a id="522_____ID_298">
          </a>
          5-2-2 理解 “端口号” 和 “进程 ID”
         </h4>
         <p>
          我们之前在学习系统编程的时候, 学习了 pid 表示唯一一个进程; 此处我们的端口号也是唯一表示一个进程. 那么这两者之间是怎样的关系?
         </p>
         <p>
          10086 例子
          <br/>
          <strong>
           另外, 一个进程可以绑定多个端口号; 但是一个端口号不能被多个进程绑定;
          </strong>
         </p>
         <ul>
          <li>
           <strong>
            <code>
             进程 ID 属于系统概念， 技术上也具有唯一性， 确实可以用来标识唯一的一个进程， 但是这样做， 会让系统进程管理和网络强耦合， 实际设计的时候， 并没有选择这样做。
            </code>
           </strong>
          </li>
         </ul>
         <h4>
          <a id="523__305">
          </a>
          5-2-3 理解源端口号和目的端口号
         </h4>
         <p>
          📚
          <strong>
           传输层协议(TCP 和 UDP)的数据段中有两个端口号, 分别叫做 源端口号 和 目的端口号. 就是在描述 “数据是谁发的, 要发给谁”;
          </strong>
         </p>
         <h3>
          <a id="_53__socket_308">
          </a>
          🦋 5-3 理解 socket
         </h3>
         <ul>
          <li>
           <strong>
            <font color="#9400D3">
             综上， IP 地址用来标识互联网中唯一的一台主机， port 用来标识该主机上唯一的一个网络进程
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#9400D3">
             IP+Port 就能表示互联网中唯一的一个进程
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#9400D3">
             所以， 通信的时候， 本质是两个互联网进程代表人来进行通信， {srcIp，srcPort， dstIp， dstPort}这样的 4 元组就能标识互联网中唯二的两个进程
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#9400D3">
             所以， 网络通信的本质， 也是进程间通信
            </font>
           </strong>
          </li>
          <li>
           <strong>
            <font color="#9400D3">
             我们把 ip+port 叫做套接字 socket
            </font>
           </strong>
          </li>
         </ul>
         <h3>
          <a id="_54__315">
          </a>
          🦋 5-4 传输层的典型代表
         </h3>
         <p>
          如果我们了解了系统， 也了解了网络协议栈， 我们就会清楚， 传输层是属于内核的， 那么我们要通过网络协议栈进行通信， 必定调用的是传输层提供的系统调用， 来进行的网络通信。

          <a id="541__TCP__320">
          </a>
          5-4-1 认识 TCP 协议
         </h4>
         <p>
          <strong>
           此处我们先对
           <code>
            TCP(Transmission Control Protocol 传输控制协议)
           </code>
           有一个直观的认识; 后面我们再详细讨论 TCP 的一些细节问题.
          </strong>
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             传输层协议

           </li>
           <li>
            <strong>
             有连接
            </strong>

            <strong>

         <h4>
          <a id="542__UDP__326">
          </a>
          5-4-2 认识 UDP 协议
         </h4>
         <p>
          <strong>
           此处我们也是对
           <code>
            UDP(User Datagram Protocol 用户数据报协议)
           </code>
           有一个直观的认识; 后面再详细讨论.
          </strong>
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             传输层协议
            </strong>
           </li>
           <li>
            <strong>
             无连接
            </strong>
           </li>
           <li>
            <strong>
             不可靠传输
            </strong>
           </li>
           <li>
            <strong>

            </strong>

         <p>
          因为我们暂时还没有深入了解 tcp、 udp 协议， 此处只做了解即可
         </p>
         <h3>
          <a id="_55__335">
          </a>
          🦋 5-5 网络字节序
         </h3>
         <p>
          我们已经知道,内存中的多字节数据相对于内存地址有大端和小端之分, 磁盘文件中的多字节数据相对于文件中的偏移地址也有大端小端之分, 网络数据流同样有大端小端之分. 那么如何定义网络数据流的地址呢?
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             发送主机通常将发送缓冲区中的数据按内存地址从低到高的顺序发出;
            </strong>
           </li>
           <li>
            <strong>
             接收主机把从网络上接到的字节依次保存在接收缓冲区中,也是按内存地址从低到高的顺序保存;
            </strong>
           </li>
           <li>
            <strong>
             因此,网络数据流的地址应这样规定:先发出的数据是低地址,后发出的数据是高地址.
            </strong>
           </li>
           <li>
            <strong>
             <code>
              TCP / IP 协议规定,网络数据流应采用大端字节序,即低地址高字节.
             </code>
            </strong>
           </li>
           <li>
            <strong>
             不管这台主机是大端机还是小端机, 都会按照这个 TCP/IP 规定的网络字节序来发送/接收数据;
            </strong>
           </li>
           <li>
            <strong>
             如果当前发送主机是小端, 就需要先将数据转成大端; 否则就忽略, 直接发送即可
            </strong>
           </li>
          </ul>
         </blockquote>
         <p>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3ee259672269420a9c86914c2e86f50f.png"/>
          <br/>
          <br/>
         </p>
         <p>
          <strong>
           🎯 为使网络程序具有可移植性,使同样的 C 代码在大端和小端计算机上编译后都能正常运行,可以调用以下库函数做网络字节序和主机字节序的转换。
          </strong>
         </p>
         <p>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ebe6531deda44489b87efb59c713df3b.png"/>
         </p>
         <blockquote>
          <ul>
           <li>
            <strong>
             这些函数名很好记,
             <code>
              h 表示 host
             </code>
             ,
             <code>
              n 表示 network
             </code>
             ,
             <code>
              l 表示 32 位长整数
             </code>
             ,
             <code>
              s 表示 16 位短整数
             </code>
             。
            </strong>
           </li>
           <li>
            <strong>
             例如
             <code>
              htonl 表示将 32 位的长整数从主机字节序转换为网络字节序
             </code>
             , 例如将 IP 地址转换后准备发送。
            </strong>
           </li>
           <li>
            <strong>
             如果主机是小端字节序, 这些函数将参数做相应的大小端转换然后返回;
            </strong>
           </li>
           <li>
            <strong>
             如果主机是大端字节序, 这些函数不做转换, 将参数原封不动地返回。
            </strong>
           </li>
          </ul>
         </blockquote>

         <h4>
          <a id="562_sockaddr__378">
          </a>
          5-6-2 sockaddr 结构
         </h4>
         <p>
          <strong>
           🦁 socket API 是一层抽象的网络编程接口,适用于各种底层网络协议,如 IPv4、 IPv6, 以及后面要讲的 UNIX Domain Socket. 然而, 各种网络协议的地址格式并不相同.
          </strong>
         </p>
         <img src="https://i-blog.csdnimg.cn/direct/c4d250b398c44e34afa2b282cb3efae9.png#pic_center" width="60%"/>
         <blockquote>
          <ul>
           <li>
            <strong>
             <font color="#ff0000">
              Pv4 和 IPv6 的地址格式定义在 netinet / in.h 中,IPv4 地址用 sockaddr_in 结构体表示,包括 16 位地址类型, 16 位端口号和 32 位 IP 地址.
             </font>
            </strong>
           </li>
           <li>
            <strong>
             IPv4、 IPv6 地址类型分别定义为常数
             <code>
              AF_INET
             </code>
             、
             <code>
              AF_INET6
             </code>
             . 这样,只要取得某种 sockaddr 结构体的首地址,不需要知道具体是哪种类型的 sockaddr 结构体,就可以根据地址类型字段确定结构体中的内容.
            </strong>
           </li>
           <li>
            <strong>
             socket API 可以都用
             <code>
              struct sockaddr \*
             </code>
             类型表示, 在使用的时候需要强制转化成
             <code>
              sockaddr_in
             </code>
             ; 这样的好处是程序的通用性, 可以接收 IPv4, IPv6, 以及 UNIX Domain Socket 各种类型的 sockaddr 结构体指针做为参数;
            </strong>
           </li>
          </ul>
         </blockquote>
