    <h2>
     <a id="1_MQTT_3">
     </a>
     1. MQTT是什么？
    </h2>
    <p>
     MQTT（Message Queuing Terlemetry Transport）消息队列遥测协议；是一种轻量级的发布/订阅消息传输协议，专为IOT、低带宽、高延迟的网络环境设计，具有高效、低耗、海量设备连接特性。
    </p>
    <p>
     <strong>
      1.通讯原理
     </strong>
    </p>
    <ul>
     <li>
      发布（Publisher）：发布消息到特定主题（Topic）
     </li>
     <li>
      订阅（Subscriber）：订阅主题接收消息
     </li>
     <li>
      代理（Broker）：消息路由管理，接收发布的消息，颁发给所有订阅者
     </li>
    </ul>
    <p>
     示例：传感器（Publisher）发送消息到主题（Topic），代理（Broker）接收消息并检查该主题的所有客户端，订阅者（Subscriber）实时接收消息
    </p>
    <p>
     <strong>
      2.消息结构
     </strong>
    </p>
    <ul>
     <li>
      主题（topic）+负载（payload）
     </li>
    </ul>
    <p>
     <strong>
      3.必要条件
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       MQTT Broker：
      </strong>
      消息代理服务器
     </li>
     <li>
      <strong>
       客户端库：
      </strong>
      设备或应用需集成MQTT客户端
     </li>
     <li>
      <strong>
       端口：
      </strong>
      默认非加密端口 1883，加密端口 8883（TLS/SSL）
     </li>
     <li>
      <strong>
       连接认证：
      </strong>
      支持用户名/密码、客户端证书等认证方式
     </li>
     <li>
      <strong>
       QoS服务质量/策略
      </strong>
     </li>
    </ul>
    <table>
     <thead>
      <tr>
       <th align="left">
        QoS等级
       </th>
       <th align="left">
        描述
       </th>
       <th align="left">
        适用场景
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        0
       </td>
       <td align="left">
        最多一次（无确认）
       </td>
       <td align="left">
        实时性高，允许丢数据
       </td>
      </tr>
      <tr>
       <td align="left">
        1
       </td>
       <td align="left">
        至少一次（需确认）
       </td>
       <td align="left">
        数据需可靠但不重复
       </td>
      </tr>
      <tr>
       <td align="left">
        2
       </td>
       <td align="left">
        精确一次（握手确认）
       </td>
       <td align="left">
        关键数据，严格不重复
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      4. 关键特性
     </strong>
     <br/>
     遗嘱消息（Last Will）
    </p>
    <ul>
     <li>
      设备异常断开时，Broker 自动发布预设消息（如“设备离线”），通知其他客户端。
     </li>
    </ul>
    <p>
     保留消息（Retained）
    </p>
    <ul>
     <li>
      Broker 保存主题的最新消息，新订阅者首次连接时立即获取。
     </li>
    </ul>
    <p>
     主题层级（Topic Hierarchy）
    </p>
    <ul>
     <li>
      支持多级通配符（+ 单层，# 多层）
     </li>
     <li>
      例如：
      <code>
       home/floor1/temperature; home/+/status（匹配所有楼层状态）
      </code>
     </li>
    </ul>
    <p>
     <strong>
      5.应用场景
     </strong>
    </p>
    <ul>
     <li>
      移动设备远程监控（AGV状态上报）
     </li>
     <li>
      跨厂区数据汇聚（通过云平台中转）
     </li>
    </ul>

     3. 服务器核心代码
    </h2>
    <p>
     <strong>
      初始化
     </strong>
     -public FrmServer(){…}
    </p>
    <ul>
     <li>
      获取IP集合
      <code>
       （Dns.GetHostAddresses）
      </code>
     </li>
     <li>
      绑定控件
      <code>
       （cmb_iplist.DataSource、.SelectedIndex）
      </code>
     </li>
    </ul>
    <p>
     <strong>
      创建服务器对象（IMqttServer）
     </strong>
     （-&gt;服务启动按钮点击事件）
    </p>
    <ul>
     <li>
      创建服务器配置 _
      <code>
       var optionsBuilder = new MqttServerOptionsBuilder()
      </code>
      <br/>
      – 验证用户密码_
      <code>
       .WithConnectionValidator(c =&gt;...
      </code>
     </li>
     <li>
      <abbr title="先初始化对象，在对象的成员.ClientConnected 应用前先实例化 new">
       实例化服务对象
      </abbr>
      _
      <code>
       mqttServer = new MqttFactory().CreateMqttServer();
      </code>
     </li>
     <li>
      创建MQTT事件方法_
      <code>
       mqttServer.ClientConnected += MqttServer_ClientConnected;
      </code>
      <br/>
      – 方法日志显示_
      <code>
       this.AddLog(0, "MQTT客户端已连接" + "ClietID:" + e.ClientId.Length);
      </code>
     </li>
     <li>
      启动服务_
      <code>
       mqttServer.StartAsync(optionsBuilder.Build());
      </code>
     </li>
    </ul>
    <p>
     <strong>
      日志对象
     </strong>
    </p>
    <ul>
     <li>
      <abbr title="创建委托(日志信息的委托)一个命名空间只需要创建一个委托；client再创建“委托对象”、“委托方法体”、“初始化绑定”，应用即可">
       创建委托
      </abbr>
      _
      <code>
       public delegate void AddlogDelegate(int index, string info);
      </code>
     </li>
     <li>
      委托方法_
      <code>
       private void AddlogMothod(int index, string info){...}
      </code>
     </li>
     <li>
      委托对象_
      <code>
       private AddlogDelegate AddLog;
      </code>
     </li>
     <li>
      对象绑定方法 _
      <code>
       this.AddLog = this.AddlogMothod;
      </code>
     </li>
     <li>
      对象应用eg:_
      <code>
       this.AddLog(0, "MQTT服务端已停止");
      </code>
     </li>
    </ul>
    <p>
     <strong>
      状态栏
     </strong>
    </p>
    <ul>
     <li>
      系统时间_
      <code>
       this.tss_time.Text = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
      </code>
     </li>
     <li>
      连接个数（连接/断开事件更新）_
      <code>
       this.tss_connnum.Text = mqttServer.GetClientSessionsStatusAsync().Result.Count.ToString();
      </code>
     </li>
    </ul>
    <p>
     <strong>
      其他
     </strong>

    <ul>
     <li>

      </code>
     </li>
     <li>
      清空日志_
      <code>
       this.lst_info.Items.Clear();
      </code>
     </li>
     <li>
      打开客户端（注意“重新生成解决方案”）_
      <code>
       new FrmClient().Show();
      </code>
     </li>
     <li>
      当前时间_
      <code>

     5. 消息订阅与发布
    </h2>
    <p>
     客户端可以正确连接后，只有订阅与发布消息，才算真正进行数据通讯；消息的订阅和发布均在客户端进行，服务端只需提供一个服务供客户连接（桥梁的作用）；小节解释消息订阅发布的核心代码。

     <strong>
      QoS服务策略
     </strong>
     （下拉框获取）

    </ul>
    <p>
     <strong>
      消息订阅
     </strong>
    </p>

      消息发布
