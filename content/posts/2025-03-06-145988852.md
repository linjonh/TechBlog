    <h4>
     <a id="1_GCC_2">
     </a>
     1. GCC编译器
    </h4>
    <ul>
     <li>
      <strong>
       GCC
      </strong>
      （GNU Compiler Collection）是GNU项目的一部分，支持多种编程语言，包括C、C++、Java、Fortran、Ada等。
     </li>
     <li>
      <strong>
       编译过程
      </strong>
      ：GCC将源程序转换为可执行程序的过程包括预处理、编译、汇编和链接四个步骤。
     </li>
    </ul>
    <h5>
     <a id="11__5">
     </a>
     1.1 编译流程
    </h5>
    <ul>
     <li>
      <strong>
       预处理
      </strong>
      ：处理宏定义和头文件，生成
      <code>
       .i
      </code>
      文件
     </li>
    </ul>
    <pre><code class="prism language-bash"> gcc <span class="token parameter variable">-E</span> hello.c <span class="token parameter variable">-o</span> hello.i
</code></pre>
    <ul>
     <li>
      <strong>
       编译
      </strong>
      ：生成汇编代码
      <code>
       .s
      </code>
      文件
     </li>
    </ul>
    <pre><code class="prism language-csharp">gcc <span class="token operator">-</span>S hello<span class="token punctuation">.</span>i <span class="token operator">-</span>o hello<span class="token punctuation">.</span>s
</code></pre>
    <ul>
     <li>
      <strong>
       汇编
      </strong>
      ：生成目标文件
      <code>
       .o
      </code>
      文件
     </li>
    </ul>
    <pre><code class="prism language-csharp">gcc <span class="token operator">-</span>c hello<span class="token punctuation">.</span>s <span class="token operator">-</span>o hello<span class="token punctuation">.</span>o
</code></pre>
    <ul>
     <li>
      <strong>
       链接
      </strong>
      ：生成可执行文件
     </li>
    </ul>
    <pre><code class="prism language-csharp">gcc hello<span class="token punctuation">.</span>o <span class="token operator">-</span>o hello
</code></pre>
    <h5>
     <a id="12_27">
     </a>
     1.2常用选项
    </h5>
    <ul>
     <li>
      <strong>
       预处理控制
      </strong>
      ：
      <ul>
       <li>
        <code>
         -E
        </code>
        ：仅进行预处理，输出到标准输出。
       </li>
       <li>
        <code>
         -Dname
        </code>
        ：定义宏
        <code>
         name
        </code>
        ，值为1。
       </li>
       <li>
        <code>
         -Dname=definition
        </code>
        ：定义宏
        <code>
         name
        </code>
        ，值为
        <code>
         definition
        </code>
        。
       </li>
       <li>
        <code>
         -Uname
        </code>
        ：取消宏
        <code>
         name
        </code>
        的定义。
       </li>
       <li>
        <code>
         -undef
        </code>
        ：取消所有与操作系统和GCC相关的宏定义。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       编译及警告信息控制
      </strong>
      ：
      <ul>
       <li>
        <code>
         -o filename
        </code>
        ：指定输出文件名。
       </li>
       <li>
        <code>
         -c
        </code>
        ：仅编译为目标文件，不进行汇编和链接。
       </li>
       <li>
        <code>
         -S
        </code>
        ：仅编译为汇编文件。
       </li>
       <li>
        <code>
         -Wall
        </code>
        ：显示所有警告信息。
       </li>
       <li>
        <code>
         -W
        </code>
        ：禁止所有警告信息。
       </li>
       <li>
        <code>
         -Werror
        </code>
        ：将所有警告信息转换为错误信息。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       C语言标准控制
      </strong>
      ：
      <ul>
       <li>
        <code>
         -ansi
        </code>
        ：使用ISO C90标准。
       </li>
       <li>
        <code>
         -std=c89
        </code>
        ：使用ISO C89标准。
       </li>
       <li>
        <code>
         -std=c99
        </code>
        ：使用ISO C99标准。
       </li>
       <li>
        <code>
         -std=gnu89
        </code>
        ：使用GNU扩展的ISO C89标准。
       </li>
       <li>
        <code>
         -std=gnu99
        </code>
        ：使用GNU扩展的ISO C99标准。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       程序调试及优化
      </strong>
      ：
      <ul>
       <li>
        <code>
         -g
        </code>
        ：生成调试信息，用于GDB调试。
       </li>
       <li>
        <code>
         -O0
        </code>
        ：不进行优化（默认）。
       </li>
       <li>
        <code>
         -O1
        </code>
        ：基本优化。
       </li>
       <li>
        <code>
         -O2
        </code>
        ：进一步优化。
       </li>
       <li>
        <code>
         -O3
        </code>
        ：最高优化级别。
       </li>
       <li>
        <code>
         -Os
        </code>
        ：优化代码大小。
       </li>
      </ul>
     </li>
    </ul>

    <h5>
     <a id="13_68">
     </a>
     1.3静态库与动态库
    </h5>
    <ul>
     <li>
      <strong>
       静态库
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         ar
        </code>
        工具创建静态库文件（
        <code>
         .a
        </code>
        ）。
       </li>
       <li>
        静态库在编译时将代码复制到可执行文件中。
       </li>
       <li>
        示例命令：
        <pre><code class="prism language-bash">gcc <span class="token parameter variable">-c</span> file.c
ar rcs libname.a file.o
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       动态库
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         gcc -shared
        </code>
        创建动态库文件（
        <code>
         .so
        </code>
        ）。
       </li>
       <li>
        动态库在运行时动态加载，节省磁盘空间和内存。
       </li>
       <li>
        示例命令：
        <pre><code class="prism language-bash">gcc <span class="token parameter variable">-fPIC</span> <span class="token parameter variable">-c</span> file.c
gcc <span class="token parameter variable">-shared</span> <span class="token parameter variable">-o</span> libname.so file.o
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li>
      静态库（.a文件）:
     </li>
    </ul>
    <pre><code class="prism language-csharp"><span class="token preprocessor property"># 生成目标文件</span>
gcc <span class="token operator">-</span>c lib1<span class="token punctuation">.</span>c lib2<span class="token punctuation">.</span>c
<span class="token preprocessor property"># 打包为静态库</span>
ar rcs libmylib<span class="token punctuation">.</span>a lib1<span class="token punctuation">.</span>o lib2<span class="token punctuation">.</span>o
<span class="token preprocessor property"># 使用静态库</span>
gcc main<span class="token punctuation">.</span>c <span class="token operator">-</span>L<span class="token punctuation">.</span> <span class="token operator">-</span>lmylib <span class="token operator">-</span>o main
</code></pre>
    <ul>
     <li>
      动态库（.so文件）：
     </li>
    </ul>
    <pre><code class="prism language-csharp"><span class="token preprocessor property"># 生成位置无关代码</span>
gcc <span class="token operator">-</span>fPIC <span class="token operator">-</span>c lib1<span class="token punctuation">.</span>c lib2<span class="token punctuation">.</span>c
<span class="token preprocessor property"># 生成动态库</span>
gcc <span class="token operator">-</span>shared <span class="token operator">-</span>o libmylib<span class="token punctuation">.</span>so lib1<span class="token punctuation">.</span>o lib2<span class="token punctuation">.</span>o

gcc main<span class="token punctuation">.</span>c <span class="token operator">-</span>L<span class="token punctuation">.</span> <span class="token operator">-</span>lmylib <span class="token operator">-</span>o main
</code></pre>

     2. GDB调试器
    </h4>
    <h5>
     <a id="21__112">
     </a>
     2.1 基本使用
    </h5>
    <ul>
     <li>
      <strong>
       GDB
      </strong>
      （GNU Debugger）是GNU项目提供的调试工具，支持C、C++、Ada、Pascal等多种语言。
     </li>
     <li>
      <strong>
       主要功能
      </strong>
      ：
      <ul>
       <li>
        启动程序并影响其行为。
       </li>
       <li>
        根据条件停止程序执行。
       </li>
       <li>
        检查程序崩溃时的状态。
       </li>
       <li>
        修改程序内部状态。
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li>
      编译时添加调试信息：
     </li>
    </ul>
    <pre><code class="prism language-csharp">gcc <span class="token operator">-</span>g <span class="token operator">-</span>o debug_prog debug_prog<span class="token punctuation">.</span>c
</code></pre>
    <ul>
     <li>
      启动GDB：
     </li>
    </ul>
    <pre><code class="prism language-csharp">gdb debug_prog
</code></pre>

    <h5>
     <a id="1_361">
     </a>
     1.变量定义
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：变量用于简化
      <code>
       Makefile
      </code>
      的编写，避免重复代码，提高可维护性。
     </li>
     <li>
      <strong>
       类型
      </strong>
      ：
      <ul>
       <li>
        <strong>
         递归展开变量
        </strong>
        ：使用
        <code>
         =
        </code>
        赋值，变量值在使用时展开。
        <pre><code class="prism language-makefile">CC = gcc
CFLAGS = -Wall -g
</code></pre>
       </li>
       <li>
        <strong>
         简单变量
        </strong>
        ：使用
        <code>
         :=
        </code>
        赋值，变量值在定义时立即展开。
        <pre><code class="prism language-makefile">CC := gcc
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       使用场景
      </strong>
      ：
      <ul>
       <li>
        在命令中通过
        <code>
         $(变量名)
        </code>
        使用变量，例如：
        <pre><code class="prism language-makefile">main: main.o
    $(CC) -o main main.o $(CFLAGS)
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2_381">
     </a>
     2.隐式规则
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：隐式规则允许Make自动推导目标文件的生成规则，减少
      <code>
       Makefile
      </code>
      的编写工作量。
     </li>
     <li>
      <strong>
       示例
      </strong>
      ：
      <ul>
       <li>
        将
        <code>
         .c
        </code>
        文件编译为
        <code>
         .o
        </code>
        文件：
        <pre><code class="prism language-makefile">%.o: %.c
    $(CC) -c $&lt; -o $@ $(CFLAGS)
</code></pre>
       </li>
       <li>
        使用隐式规则简化目标文件的生成：
        <pre><code class="prism language-makefile">main: main.o hello1.o hello2.o
    $(CC) -o main main.o hello1.o hello2.o $(CFLAGS)
</code></pre>
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="1__878">
     </a>
     1. 静态链接库和动态链接库之间的相同点和不同点
    </h4>
    <p>
     <strong>
      相同点：
     </strong>
    </p>
    <ul>
     <li>
      都是共享代码的方式，避免重复编写和编译相同的代码。
     </li>
     <li>
      允许程序使用库中定义的函数和变量。
     </li>
    </ul>
    <p>
     <strong>
      不同点：
     </strong>
    </p>
    <ul>
     <li>
      静态链接库在编译时被复制到最终的可执行文件中，增加可执行文件的大小，运行时不需要库文件。
     </li>
     <li>
      动态链接库在程序运行时加载，减小可执行文件的大小，运行时需要库文件。
     </li>
     <li>
      静态链接库更新需要重新编译所有使用该库的程序；动态链接库更新只需替换库文件，无需重新编译程序。
     </li>
    </ul>
    <h4>
     <a id="2_Make_889">
     </a>
     2. Make命令工具如何确定哪些文件需要重新生成，而哪些不需要生成
    </h4>
    <ul>
     <li>
      Make工具通过比较目标文件和其依赖文件的时间戳来确定。
     </li>
     <li>
      如果目标文件不存在或其修改时间晚于任一依赖文件，则需要重新生成。
     </li>
     <li>
      如果目标文件比所有依赖文件新，则不需要重新生成。
     </li>
     <li>
      Makefile中的规则和变量也会影响哪些文件需要重新生成。
     </li>
    </ul>
    <h4>
     <a id="3_Make_896">
     </a>
     3. Make中的伪目标的作用是什么
    </h4>
    <ul>
     <li>
      伪目标用于定义不对应实际文件的目标。
     </li>
     <li>
      用于执行操作，如构建多个目标或执行复杂构建步骤。
     </li>
     <li>
      避免与实际文件名冲突，确保Make能执行所需操作。
     </li>
     <li>
      伪目标通常以
      <code>
       .PHONY
      </code>
      声明，例如
      <code>
       .PHONY: clean
      </code>
      ，确保
      <code>
       make clean
      </code>
      执行清理操作。
     </li>
    </ul>

    <h4>
     <a id="2Makefile_1001">
     </a>
     问题2，不习惯使用Makefile的自动化编译
    </h4>
    <p>
     使用Makefile时经常出现各种错误，比如链接库失败，依赖缺失等。当程序报错的时候，可以将报错信息发给AI，AI会分析报错原因，并提供正确的做法，如：

    <pre><code class="prism language-csharp">编译错误：undefined reference to xxx

</code></pre>
    <p>
     AI还会生成各种学习资源，以及对当前更好的修改方法。而且AI可以生成基础语法教程，如果对哪一种命令感到疑惑，也可以问AI，让AI生成具体用法并给出示例，学习事半功倍。
