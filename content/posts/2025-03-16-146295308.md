    <h3>
     <a id="vscode__2">
     </a>
     vscode 自带的格式化
    </h3>
    <p>
     vscode 默认支持导入语句排序和删除未使用的导入。
    </p>
    <p>
     对应的配置是：
     <code>
      source.organizeImports
     </code>
    </p>
    <p>
     可选值：
    </p>
    <ol>
     <li>
      never：关闭
     </li>
     <li>
      explicit：仅在显式手动保存时触发
     </li>
     <li>
      always：自动保存时也会触发
     </li>
    </ol>
    <p>
     我喜欢在手动 Ctrl + S 保存时，自动格式化当前代码。因此可以如下配置：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"editor.formatOnSave"</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span> <span class="token comment">// 保存时格式化</span>
    <span class="token string-property property">"editor.codeActionsOnSave"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string-property property">"source.organizeImports"</span><span class="token operator">:</span> <span class="token string">"explicit"</span> <span class="token comment">// 在显示保存时，整理导入语句</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <h3>
     <a id="eslint__32">
     </a>
     eslint 实现整理导入语句
    </h3>
    <p>
     vscode 的导入语句排序，没有强制性，在协作中为了规范其他人要整理导入语句，我们往往会使用 eslint 对排序进行校验。
    </p>
    <p>
     并且 vscode 是按字母顺序排序的，这可能不符合要求，或者想要自定义排序规则。我们往往也会使用 eslint 来执行整理代码的操作。
    </p>
    <p>
     这依赖两个 eslint 插件：
    </p>
    <ol>
     <li>
      导入排序：
      <code>
       eslint-plugin-simple-import-sort
      </code>
     </li>
     <li>
      删除未使用的导入：
      <code>
       eslint-plugin-unused-imports
      </code>
     </li>
    </ol>
    <p>
     安装后， eslint 配置：
    </p>
    <pre><code class="prism language-typescript">module<span class="token punctuation">.</span>exports <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
  plugins<span class="token operator">:</span> <span class="token punctuation">[</span>
    <span class="token string">"unused-imports"</span><span class="token punctuation">,</span> 
    <span class="token string">"simple-import-sort"</span>
  <span class="token punctuation">]</span><span class="token punctuation">,</span>
  rules<span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
   <span class="token string-property property">"simple-import-sort/imports"</span><span class="token operator">:</span> <span class="token string">"error"</span><span class="token punctuation">,</span> <span class="token comment">// 强制导入语句排序</span>
    <span class="token comment">// "simple-import-sort/exports": "error", // 强制导出语句排序</span>
    <span class="token string-property property">"unused-imports/no-unused-imports"</span><span class="token operator">:</span> <span class="token string">"error"</span><span class="token punctuation">,</span> <span class="token comment">// 禁止未使用的导入，eslint修复时会自动删除未使用的导入</span>
    <span class="token string-property property">"unused-imports/no-unused-vars"</span><span class="token operator">:</span> <span class="token punctuation">[</span>
        <span class="token comment">// 不禁止未使用的变量，只是警告，避免 eslint 修复时删除它</span>
        <span class="token string">"warn"</span><span class="token punctuation">,</span>
        <span class="token punctuation">{<!-- --></span> vars<span class="token operator">:</span> <span class="token string">"all"</span><span class="token punctuation">,</span> varsIgnorePattern<span class="token operator">:</span> <span class="token string">"^_"</span><span class="token punctuation">,</span> args<span class="token operator">:</span> <span class="token string">"after-used"</span><span class="token punctuation">,</span> argsIgnorePattern<span class="token operator">:</span> <span class="token string">"^_"</span> <span class="token punctuation">}</span>
    <span class="token punctuation">]</span>
  <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     配置规则后，当代码不符合规范时，eslint 就会抛出对应的错误或者警告。
    </p>
    <p>
     执行 eslint --fix 修复时，两个插件就会按配置的规则整理导入语句。
    </p>
    <p>
     显然执行 eslint 命令修复太麻烦了，我希望和 vscode 自带的整理功能一样，手动执行 Ctrl + S 保存时，就整理代码。
    </p>
    <p>
     要实现这一点，只需要配置
     <code>
      source.fixAll.eslint
     </code>
     。指定在显示保存时，使用 eslint --fix 来格式化代码即可。
    </p>
    <p>
     并且注意，要关闭 vscode 自带的整理导入语句功能，要不然会有冲突，因为两个排序规则不一样。
    </p>
    <ul>
     <li>
      VSCode 默认的导入整理是按
      <strong>
       文件路径字母顺序
      </strong>
      排序，
     </li>
     <li>
      <code>
       &lt;font style="background-color:rgb(252, 252, 252);"&gt;simple-import-sort&lt;/font&gt;
      </code>
      插件要求按
      <strong>
       自定义分组规则
      </strong>
      （默认是内置模块优先、外部依赖次之、本地文件最后）排序。
     </li>
    </ul>
    <pre><code class="prism language-typescript"><span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"editor.formatOnSave"</span><span class="token operator">:</span> <span class="token boolean">true</span><span class="token punctuation">,</span> <span class="token comment">// 保存时自动格式化</span>
    <span class="token string-property property">"editor.codeActionsOnSave"</span><span class="token operator">:</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string-property property">"source.organizeImports"</span><span class="token operator">:</span> <span class="token string">"never"</span><span class="token punctuation">,</span> <span class="token comment">// 关闭 vscode 默认的导入语句整理功能：1.import 语句按字母顺序排序, 2.删除未使用的导入语句</span>
        <span class="token string-property property">"source.fixAll.eslint"</span><span class="token operator">:</span> <span class="token string">"explicit"</span> <span class="token comment">// 让 ESLint 处理所有修复（包括排序）</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
