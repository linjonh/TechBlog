     web 图像技术：前端引入图片的各种方式及其优缺点

     <p style="text-align: left">
      前端开发人员在构建网站时需要做的一个决定是引入图片的方式。它可以是
      <code>
       &lt;img&gt;
      </code>
      标签，或者是通过CSS
      <code>
       background
      </code>
      属性，还可以使用 SVG
      <code>
       &lt;image&gt;
      </code>
      。选择正确的方式是很重要的，它对性能和可访问性有很大的影响。
     </p>
     <p style="text-align: left">
      在本文中，我们会学习引入图像的各种方式，以及每种方式的优点和缺点，以及何时使用和为什么使用它们。
     </p>
     <h4>
      HTML img 元素
     </h4>
     <p style="text-align: left">
      最简单的情况下，
      <code>
       img
      </code>
      元素必须包含
      <code>
       src
      </code>
      属性:
     </p>
     <pre class="has"><code class="language-go">&lt;img src="cool.jpg" alt=""&gt;
</code></pre>
     <h5>
      设置宽度和高度属性
     </h5>
     <p style="text-align: left">
      在页面加载时，它们会在页面图像加载时发生一些布局变化。为了避免这种情况，我们可以为它设置
      <code>
       width
      </code>
      和
      <code>
       height
      </code>
      属性:
     </p>
     <pre class="has"><code class="language-go">&lt;img src="cool.jpg" width="200" height="100" alt=""&gt;
</code></pre>
     <p style="text-align: left">
      虽然对某些人来说，这可能看起来太过简单了，但它是有用的。我们用图例的方式来理清这个概念：
     </p>
     <p style="text-align: center">
      <img src="https://i-blog.csdnimg.cn/blog_migrate/f07d384c16ed03b5028bef64a0cd5914.png"/>
     </p>
     <p style="text-align: left">
      ‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍我们看到到右侧图片即使尚未加载仍保留了空间？那是因为设置了宽度和高度。
     </p>
     <h5>
      通过 CSS 隐藏图像
     </h5>
     <p style="text-align: left">
      图像可以用 CSS 隐藏。但是，它仍将加载在页面中。因此，在执行此操作时请小心。如果一个图像应该被隐藏，那么它可能是出于装饰的目的。
     </p>
     <pre class="has"><code class="language-go">img {
    display: none;
}
</code></pre>
     <p style="text-align: left">
      同样，上面的方法也不能阻止浏览器加载图像，即使它在视觉上是隐藏的。原因是
      <code>
       &lt;img&gt;
      </code>
      被认为是一个被替换的元素，所以我们无法控制它所加载的内容。
     </p>
     <h5>
      可访问性问题
     </h5>
     <p style="text-align: left">
      通过将
      <code>
       alt
      </code>
      属性设置为有意义的描述，用来访问 HTML 图像，这对于屏幕阅读器用户非常有帮助。
     </p>
     <p style="text-align: left">
      然而，如果一个
      <code>
       alt
      </code>
      描述是不需要的，请不要删除它，如果你这样做，图像
      <code>
       src
      </code>
      将被读出,这对可访问性非常不利。
     </p>
     <p style="text-align: left">
      不仅如此，如果图像因为某种原因没有加载，并且它有一个清晰的
      <code>
       alt
      </code>
      ，它将作为一个回退显示。同样通过图例来演示一下。
     </p>
     <p style="text-align: left">
      假设我们有以下图片：
     </p>
     <pre class="has"><code class="language-go">&lt;img class="food-thumb" width="300" height="200" src="cheescake.jpg"&gt;

&lt;img class="food-thumb" width="300" height="200" src="cheescake.jpg" alt=""&gt;
</code></pre>
<p style="text-align: left">
当
<code>
src
</code>
无效，图片没有加载时。第一个没有
<code>
alt
</code>
属性，而第二个是空的
<code>
alt
</code>
属性，下面是它们的视觉效果：
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/73226b8d2e8cadcb71a48bcb9841f9aa.png"/>
</p>
<p style="text-align: left">
没有
<code>
alt
</code>
的图片仍然保留了它的空间，这很混乱，而且不利于访问。而另一个
<code>
alt
</code>
为空的图片，会折叠起来看起来像一个小点，这里因为它有边框。
</p>
<p style="text-align: left">
但是，当
<code>
alt
</code>
属性有值，它看起来是这样的：
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/b687dbec605f48bff5825a9374e4a520.png"/>
</p>
<p style="text-align: left">
这不是一个很好的反馈吗?此外，当图像源失败时，可以向它们添加伪元素。
</p>
<h5>
响应式图像
</h5>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/c5d3557efcceaefbd9a80e1264d9cff7.png"/>
</p>
<p style="text-align: left">
<code>
&lt;img&gt;
</code>
的优点在于可以针对特定视口大小将其扩展为具有多个版本的照片。我们有两种不同的方式来生成一组响应式图像：
</p>
<p style="text-align: left"> 1.
<strong>
<code>
srcset
</code>
属性
</strong>
</p>
<pre class="has"><code class="language-go">&lt;img src="small.jpg" srcset="medium.jpg 500w, large.jpg 800w" alt=""&gt;
</code></pre>
<p style="text-align: left">
这很简单。对我来说，
<code>
srcset
</code>
可以根据屏幕宽度显示多个图像尺寸，这并不是一种完美的解决方案。它让浏览器选择合适的图像，而我们对此无能为力。
</p>
<p style="text-align: left"> 2.
<strong>
<code>
picture
</code>
标签
</strong>
</p>
<pre class="has"><code class="language-go">&lt;picture&gt;
  &lt;source srcset="large.jpg" media="(min-width: 800px)" /&gt;
  &lt;source srcset="medium.jpg" media="(min-width: 500px)" /&gt;
  &lt;img src="small.jpg" /&gt;
&lt;/picture&gt;
</code></pre>
<p style="text-align: left">
另一种选择是使用
<code>
&lt;picture&gt;
</code>
元素。我更喜欢这种方式，因为它更容易预测。
</p>
<p style="text-align: left">
事例源码：https://codepen.io/shadeed/pen/d703aee137f38c138f2323a0252548ac?editors=1100
</p>
<h5>
调整图像大小
</h5>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/20c4729621b440545021beacddb93850.png"/>
</p>
<p style="text-align: left">
对于
<code>
&lt;img&gt;
</code>
，我们还可以使用的一组很好的特性
<code>
object-fit
</code>
和
<code>
object-position
</code>
。它们可以控制
<code>
&lt;img&gt;
</code>
的大小和定位，就像 CSS 背景图像。
</p>
<p style="text-align: left">
<code>
object-fit
</code>
值有：
<code>
fill, contain, cover, none, scale-down
</code>
。例如：
</p>
<pre class="has"><code class="language-go">img {
    object-fit: cover;
    object-position: 50% 50%;
}
</code></pre>
<h4>
CSS 背景图片
</h4>
<p style="text-align: left">
使用 CSS 背景显示图像时，它需要一个具有内容或特定宽度或高度的元素。通常，背景图像的主要用途应该是用于装饰目的。
</p>
<h5>
何使用 CSS 背景图片
</h5>
<p style="text-align: left">
首先，我们需要一个元素
</p>
<pre class="has"><code class="language-go">// html
&lt;div class="element"&gt;Some content&lt;/div&gt;

// css
.element {
    background: url('cool.jpg');
}
</code></pre>
<h5>
多个背景
</h5>
<p style="text-align: left">
使用 CSS 背景图片的好处是可以设置多个背景。考虑以下示例：
</p>
<pre class="has"><code class="language-go">.element {
    background: url('cool-1.jpg'), url('cool-2.jpg');
}
</code></pre>
<h5>
隐藏图像
</h5>
<p style="text-align: left">
我们可以在特定视口中隐藏和显示图像，如果未使用 CSS 设置图片，则不会下载该图片。与使用
<code>
&lt;img&gt;
</code>
相比，这是一个额外的好处。
</p>
<p style="text-align: left">
在上面的示例中，我们有一个背景图像，仅在视口宽度大于
<code>
700px
</code>
时显示。
</p>
<h5>
可访问性问题
</h5>
<p style="text-align: left">
如果使用不当，背景图像可能会影响可访问性。例如，将其用于文章主题，这对于文章至关重要。
</p>
<h5>
非开发人员无法下载
</h5>
<p style="text-align: left">
普通人知道，如果要保存图像，只需单击鼠标左键，然后选择保存即可。CSS 　背景图片并非如此，我们必须先检查元素，然后在
<code>
DevTools
</code>
中的
<code>
url
</code>
中打开链接，然后才能下载随 CSS 添加的图像。
</p>
<h5>
伪元素
</h5>
<p style="text-align: left">
可以将伪元素与 CSS 背景图像一起使用，例如在图像顶部显示覆盖图。对于
<code>
&lt;img&gt;
</code>
这是不可能的，直到我们为叠加层添加单独的元素。
</p>
<h4>
SVG Image
</h4>
<p style="text-align: left">
<strong>
SVG
</strong>
被认为是一种图像，它最大的功能是在不影响质量的情况下进行缩放。此外，使用
<code>
SVG
</code>
，我们可以嵌入
<code>
JPG
</code>
、
<code>
PNG
</code>
或
<code>
SVG
</code>
图像。参见下面的 HTML:
</p>
<pre class="has"><code class="language-go">&lt;svg width="200" height="200"&gt;
  &lt;image href="cheesecake.jpg" height="100%" width="100%" preserveAspectRatio="xMidYMid slice" /&gt;
&lt;/svg&gt;
</code></pre>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/e6abe18df6c692ebc65a4a5fe22e850e.png"/>
</p>
<p style="text-align: left">
<strong>
你是否注意到了
<code>
prepareAspectRatio
</code>
？它的作用是可以让图像占据 SVG 的整个宽度和高度，而不会被拉伸或压缩。
</strong>
</p>
<p style="text-align: left">
当
<code>
&lt;image&gt;
</code>
宽度较大时，它将填充其父级（SVG）宽度而不会拉伸。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/c1ca0f8d9ef1c89d36527c070e290910.png"/>
</p>
<p style="text-align: left">
这非常类似于 CSS 中的
<code>
object-fit: cover
</code>
或
<code>
background-size: cover
</code>
。
</p>
<h5>
可访问性问题
</h5>
<p style="text-align: left">
关于 SVG 的可访问性，这使我想起了
<code>
&lt;title&gt;
</code>
元素。例如，我们可以像下面这样添加它：
</p>
<pre class="has"><code class="language-go">&lt;svg width="200" height="200"&gt;
   &lt;title&gt;A photo of blueberry Cheescake&lt;/title&gt;
   &lt;image href="cheesecake.jpg" height="100%" width="100%" preserveAspectRatio="xMidYMid slice" /&gt;
&lt;/svg&gt;
</code></pre>
<p style="text-align: left">
我们还可以使用
<code>
&lt;desc&gt;
</code>
元素
</p>
<pre class="has"><code class="language-go">&lt;svg width="200" height="200"&gt;
   &lt;title&gt;A photo of blueberry Cheescake&lt;/title&gt;
   &lt;desc&gt;A meaningful description about the image&lt;/desc&gt;
   &lt;image href="cheesecake.jpg" height="100%" width="100%" preserveAspectRatio="xMidYMid slice" /&gt;
&lt;/svg&gt;
</code></pre>
<h5>
非开发人员无法下载
</h5>
<p style="text-align: left">
必须先检查元素并复制图像的
<code>
URL
</code>
，然后才能下载嵌入
<code>
SVG
</code>
的图像。但是，如果我们要防止用户下载特定的图像，这可能是一件好事。
</p>
<h4>
用例
</h4>
<h5>
Hero Section
</h5>
<p style="text-align: left">
在构建
<code>
hero p
</code>
时，有时我们需要在标题和其他内容下方的图像。参见下图：
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/13ebf6a5065e058fa1b9b3a20707d127.png"/>
</p>
<p style="text-align: left">
一些要求：
</p>
<ul>
<li>
<p style="text-align: left">
背景图片能够动态替换
</p>
</li>
<li>
<p style="text-align: left">
图片有一个覆盖层，让阅读更容易
</p>
</li>
<li>
<p style="text-align: left">
图像有三种尺寸:小号、中号和大号。每一个都是针对一个特定的视口。
</p>
</li>
</ul>
<p style="text-align: left">
在开始解决方案之前，让我们先问问自己这种背景的性质。这是一些入门问题：
</p>
<ol>
<li>
<p style="text-align: left">
为用户保留这个图像很重要吗，还是可以跳过它？
</p>
</li>
<li>
<p style="text-align: left">
我们是否需要在所有视口尺寸上使用它？
</p>
</li>
<li>
<p style="text-align: left">
它是静态的还是动态变化的？
</p>
</li>
</ol>
<h5>
解决方案 1
</h5>
<p style="text-align: left">
通过使用多个 CSS 背景，我们可以将一个背景作为叠加层，将另一个背景作为实际图像。请看下面的 CSS：
</p>
<pre class="has"><code class="language-go">.hero {
    background-image: linear-gradient(rgba(0, 0, 0, 0.4), rgba(0, 0, 0, 0.4)), var('landscape.jpg');
    background-repeat: no-repeat;
    background-size: 100%, cover;
}
</code></pre>
<p style="text-align: left">
虽然此解决方案有效，但可以使用 JavaScript 动态更改背景图片。见下面：
</p>
<pre class="has"><code class="language-go">&lt;p class="hero" style="background: linear-gradient(rgba(0, 0, 0, 0.5), rgba(0, 0, 0, 0.5)), url('landscape.jpg');"&gt;
  &lt;!-- Hero content --&gt;
&lt;/p&gt;
</code></pre>
<p style="text-align: left">
这里添加了一个内联样式。虽然这是可行的，但它看起来很丑，而且不实用。
</p>
<p style="text-align: left">
也许我们可以使用 CSS 变量？让我们来探索一下。
</p>
<pre class="has"><code class="language-go">&lt;p class="hero" style="--bg-url: url('landscape.jpg')"&gt;
  &lt;!-- Hero content --&gt;
&lt;/p&gt;
</code></pre>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/e63a119485a207d586edeb0e9dafcd9f.png"/>
</p>
<p style="text-align: left">
现在，我们可以轻松地更新
<code>
--bg-url
</code>
变量来动态更改背， 这比内联的东西好一百万倍。
</p>
<p style="text-align: left">
小结：
</p>
<ol>
<li>
<p style="text-align: left">
这种方案只有在图片不重要才适用
</p>
</li>
<li>
<p style="text-align: left">
这种方案只适合一些静态网站，因为没有从后台拉取图片
</p>
</li>
</ol>
<p style="text-align: left">
事例源码：https://codepen.io/shadeed/pen/17978a2d824fd51a3b27c2c2d099a522
</p>
<h5>
解决方案 2
</h5>
<p style="text-align: left">
该方案是可以使用一个
<code>
img
</code>
标签：
</p>
<pre class="has"><code class="language-go">&lt;p class="hero"&gt;
  &lt;h2 class="hero**title"&gt;Using Images in CSS&lt;/h2&gt;
  &lt;p class="hero**desc"&gt;An article about which and when to use&lt;/p&gt;
  &lt;img src="landscape.jpg" alt=""&gt;
&lt;/p&gt;
</code></pre>
<p style="text-align: left">
在 CSS 中，我们需要将图片绝对定位在内容下方，并且还需要使用伪元素作为叠加层。
</p>
<pre class="has"><code class="language-go">.hero {
    position: relative;
}

.hero img {
    position: absolute;
    left: 0;
    top: 0;
    z-index: -1;
    width: 100%;
    height: 100%;
    object-fit: cover;
}

.hero:after {
    content: "";
    position: absolute;
    left: 0;
    top: 0;
    z-index: -1;
    width: 100%;
    height: 100%;
    background: rgba(0, 0, 0, 0.4);
}
</code></pre>
<p style="text-align: left">
此解决方案的优点在于，可以轻松更改图片的
<code>
src
</code>
属性。同样，如果图像很重要，它将会更加有用。
</p>
<p style="text-align: left">
另外，我喜欢使用 HTML
<code>
&lt;img&gt;
</code>
的功能是能够在未加载图片的情况下添加回退。回退至少可以使内容保持可读性。
</p>
<pre class="has"><code class="language-go">.hero img {
    /_ Other styles _/
    background: #2962ff;
  }
</code></pre>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/fa9cb786a389ff42c38436814b52104b.png"/>
</p>
<p style="text-align: left">
在图片源路径不对时，背景颜色是一个代替方面。
<br/>
</p>
<p style="text-align: left">

</p>
<h4>
网站 Logo
</h4>
<p style="text-align: left">
网站
<code>
Logo
</code>
是很重要的，因为它可以将网站与其他网站区分开。要嵌入徽标，我们有一些选择：
</p>
<ul>
<li>
<p style="text-align: left">
<code>
img
</code>
-&gt;
<code>
png,jpg, orsvg
</code>
</p>
</li>
<li>
<p style="text-align: left">
内联 SVG
</p>
</li>
<li>
<p style="text-align: left">
背景图
</p>
</li>
</ul>
<p style="text-align: left">
接下来，我们来看看哪种方式更合适。
</p>
<h5>
带有很多细节的 Logo
</h5>
<p style="text-align: left">
当徽标具有许多细节或形状时，将其用作嵌入式
<code>
SVG
</code>
可能没有好处。我建议使用
<code>
&lt;img&gt;
</code>
，图像类型可以是
<code>
png
</code>
，
<code>
jpg
</code>
或
<code>
svg
</code>
。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/28b709792d4ea133dee19a19fc043750.png"/>
</p>
<h5>
需要动画的简单 Logo
</h5>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/05d6d20d4d961478bcc9b116ae2fe20b.png"/>
</p>
<p style="text-align: left">
我们有一个简单的 logo ，其中包含形状和文字。悬停时，形状和文本需要更改颜色。怎么做？对我来说最好的解决方案是使用嵌入式 SVG。
</p>
<p style="text-align: left">
<strong>
HTML
</strong>
</p>
<pre class="has"><code class="language-go">&lt;a href="#"&gt;
    &lt;svg class="logo" width="115" height="47" xmlns="http://www.w3.org/2000/svg"&gt;
      &lt;g transform="translate(-5 -5)" fill="none" fill-rule="evenodd"&gt;
        &lt;rect fill="#D8D8D8" transform="rotate(45 28.5 28.5)" x="9" y="9" width="39" height="39" rx="11" /&gt;
        &lt;text font-family="Rubik-Medium, Rubik" font-size="25" font-weight="400" fill="#6F6F6F"&gt;
          &lt;tspan x="63.923" y="36.923"&gt;Rect&lt;/tspan&gt;
        &lt;/text&gt;
      &lt;/g&gt;
    &lt;/svg&gt;
&lt;/a&gt;
</code></pre>
<p style="text-align: left">
<strong>
CSS
</strong>
</p>
<pre class="has"><code class="language-go">.logo rect,
.logo text {
  transition: 0.3s ease-out;
}

.logo:hover rect,
.logo:hover text {
  fill: #4a7def;
}
</code></pre>
<p style="text-align: left">
事例源码：https://codepen.io/shadeed/pen/4005077cc543647148007f4834c0585c?editors=0100
</p>
<h5>
响应 Logo
</h5>
<p style="text-align: left">
这让我想起了
<strong>
Smashing Magazine
</strong>
的 logo。我喜欢它从一个小图标变成一个完整的徽标。参见下面的模型：
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/47c8af48141cd84f722393e4ef244b13.png"/>
</p>
<p style="text-align: left">
完美的解决方案是使用
<code>
&lt;picture&gt;
</code>
元素，可以在其中添 加 logo 的两个版本。如下所示：
</p>
<pre class="has"><code class="language-go">&lt;a class="logo" href="/"&gt;
    &lt;picture&gt;
      &lt;source media="(min-width: 1350px)" srcset="sm-logo--full.svg"&gt;&lt;img src="sm-logo.svg" alt="Smashing Magazine"&gt;&lt;/picture&gt;
  &lt;/a&gt;
</code></pre>
<p style="text-align: left">
在 CSS 中，我们需要将视口的宽度更改为等于或大于
<code>
1350px
</code>
。
</p>
<pre class="has"><code class="language-go">.logo {
  display: inline-block;
  width: 45px;
}

@media (min-width: 1350px) {
  .logo {
    width: 180px;
  }
}
</code></pre>
<p style="text-align: left">
这是简单而直接的解决方案。
</p>
<p style="text-align: left">
事例源码：https://codepen.io/shadeed/pen/6cf55d4e87b7c443820bd5f8694587a8?editors=1100
</p>
<h5>
带有渐变的 Logo
</h5>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/6ab28f35f5e2028c75be284ac84187e4.png"/>
</p>
<p style="text-align: left">
当 logo 具有渐变时，从
<code>
Illustrator
</code>
或
<code>
Sketch
</code>
等设计应用程序将其导出的过程可能并不完美，有时会中断。
</p>
<p style="text-align: left">
使用 SVG，我们可以轻松地为
<code>
logo
</code>
添加渐变。我添加了
<code>
&lt;linearGradient&gt;
</code>
并将其用作文本填充。
</p>
<pre class="has"><code class="language-go">&lt;svg class="logo" width="115" height="47" xmlns="http://www.w3.org/2000/svg"&gt;
    &lt;defs&gt;
      &lt;linearGradient id="gradient" x1="0%" y1="100%" x2="0%" y2="0%"&gt;
        &lt;stop offset="0%" stop-color="#4a7def"&gt;&lt;/stop&gt;
        &lt;stop offset="50%" stop-color="#ab4787"&gt;&lt;/stop&gt;
      &lt;/linearGradient&gt;
    &lt;/defs&gt;
      &lt;g transform="translate(-5 -5)" fill="none" fill-rule="evenodd"&gt;
        &lt;rect fill="#AB4787" transform="rotate(45 28.5 28.5)" x="9" y="9" width="39" height="39" rx="11" /&gt;
        &lt;text font-family="Rubik-Medium, Rubik" font-size="30" font-weight="400" fill="url(#gradient)"&gt;
          &lt;tspan x="63.923" y="36.923"&gt;Rect&lt;/tspan&gt;
        &lt;/text&gt;
      &lt;/g&gt;
&lt;/svg&gt;
</code></pre>
<p style="text-align: left">
事例源码：https://codepen.io/shadeed/pen/9bf3bee3d08a40411effb5d65f25b5c1?editors=1100
</p>
<h5>
用户头像
</h5>
<p style="text-align: left">
对于用户头像，它们具有很多形状，但最常见的是矩形或圆形。在此用例中，会介绍一个对你有用的重要技巧。
</p>
<p style="text-align: left">
首先，让我们看下面的模型。请注意，我们有一个完美的化身，并且
<code>
100％
</code>
清晰。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/f298970d2352384c26f513fa82a89fa7.png"/>
</p>
<p style="text-align: left">
但是，当用户上传半白色头像或非常浅的头像时，这个设计就不太好了。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/c119bb1c5e90b303d8bde09eb3a8c5ac.png"/>
</p>
<p style="text-align: left">
请注意，在上面的模型中，除非真正专注看，否则尚不清楚其中是否有一个圆圈。这是一个问题。为了解决这个问题，我们应该在头像内添加边框，以防图像太亮看不清除。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/68c4fcf7b4687bddfa11f24168575c23.png"/>
</p>
<p style="text-align: left">
我们有几个选择
<br/>
</p>
<ul>
<li>
<p style="text-align: left">
<code>
img
</code>
元素
</p>
</li>
<li>
<p style="text-align: left">
<code>
img
</code>
和
<code>
div
</code>
元素
</p>
</li>
<li>
<p style="text-align: left">
<code>
div
</code>
与 CSS 背景
</p>
</li>
<li>
<p style="text-align: left">
SVG
<code>
image
</code>
</p>
</li>
</ul>
<p style="text-align: left">
哪一个最好？我们来探索探索。
</p>
<h5>
使用 HTML `&lt;img&gt;`
</h5>
<p style="text-align: left">
你可能首先想到的是添加一个边框，对吧?让我们来探讨一下这个问题。
</p>
<pre class="has"><code class="language-go">.avatar {
    border: 2px solid #f2f2f2;
}
</code></pre>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/6efdeff88d0e2772bb7d7c14969b54e8.png"/>
</p>
<p style="text-align: left">
我们的目标是使内部边框与图像融合在一起，这种不太实用。
</p>
<h5>
使用 `&lt;img&gt;` 和 `&lt;div&gt;` 元素
</h5>
<p style="text-align: left">
现在的问题是，要添加内部边框，我们不能使用
<code>
box-shadow
</code>
，因为它无法在图像上使用。解决方案用
<code>
&lt;div&gt;
</code>
包裹 头像中，并添加专用于内部边框的元素。
</p>
<p style="text-align: left">
<strong>
HTML
</strong>
</p>
<pre class="has"><code class="language-go">&lt;div class="avatar-wrapper"&gt;
     &lt;img class="avatar" src="shadeed2.jpg" alt="A photo of Ahmad Shadeed"&gt;
     &lt;div class="avatar-border"&gt;&lt;/div&gt;
&lt;/div&gt;
</code></pre>
<p style="text-align: left">
<strong>
CSS
</strong>
</p>
<pre class="has"><code class="language-go">.avatar-wrapper {
  position: relative;
  width: 150px;
  height: 150px;
}

.avatar-border {
  position: absolute;
  left: 0;
  top: 0;
  width: 100%;
  height: 100%;
  border-radius: 50%;
  border: 2px solid rgba(0, 0, 0, 0.1);
}
</code></pre>
<p style="text-align: left">
通过在
<code>
&lt;div&gt;
</code>
上使用透明度 10%黑色的边框，我们可以确保边框与深色图像融合，并且只有在图像较亮的情况下才可见。参见下面的模型：
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/9e3746ac1b40468ae518da8aa3535fb7.png"/>
</p>
<p style="text-align: left">
事例地址：https://codepen.io/shadeed/pen/da23d9a18dac14692a97e1bc6e86a5ff?editors=1100
</p>
<h5>
使用`&lt;div&gt;`与 CSS 背景
</h5>
<p style="text-align: left">
如果我要使用
<code>
&lt;div&gt;
</code>
来显示头像，则可能表示该图像具有装饰性。我记得一个用例，它是分散在页面中的随机头像。
</p>
<p style="text-align: center">
<img src="https://i-blog.csdnimg.cn/blog_migrate/5b6c79cf3b6881f5222f4ce165c4f1d9.png"/>
</p>
<p style="text-align: left">
<strong>
HTML
</strong>
</p>
<pre class="has"><code class="language-go">&lt;div class="avatar" style="--img-url: url(shadeed2.jpg)"&gt;&lt;/div&gt;
</code></pre>
<p style="text-align: left">
<strong>
CSS
</strong>
</p>
<pre class="has"><code class="language-go">.avatar {
  background: var(--img-url) center/cover;
  width: 150px;
  height: 150px;
  border-radius: 50%;
  box-shadow: inset 0 0 0 2px rgba(#000, 0.1);
}
</code></pre>
<p style="text-align: left">
事例地址:https://codepen.io/shadeed/pen/39eb9dac364ec15b9ab9bae7fe3a7148?editors=0100
</p>
<h5>
使用  SVG `&lt;image&gt;`
</h5>
<p style="text-align: left">
对我来说，这是最有趣的解决方案。我在检查 Facebook 的新设计时注意到了它。
</p>
<pre class="has"><code class="language-go">&lt;svg role="none" style="height: 36px; width: 36px;"&gt;
  &lt;mask id="avatar"&gt;
    &lt;circle cx="18" cy="18" fill="white" r="18"&gt;&lt;/circle&gt;
  &lt;/mask&gt;
  &lt;g mask="url(#avatar)"&gt;
    &lt;image x="0" y="0" height="100%" preserveAspectRatio="xMidYMid slice" width="100%" xlink:href="avatar.jpg" style="height: 36px; width: 36px;"&gt;&lt;/image&gt;
    &lt;circle cx="18" cy="18" r="18"&gt;&lt;/circle&gt;
  &lt;/g&gt;
&lt;/svg&gt;
</code></pre>
<p style="text-align: left">
先对其进行剖析，它包含以下内容：
</p>
<ul>
<li>
<p style="text-align: left">
用于将图像剪切为圆形(circle )的蒙版(mask )
</p>
</li>
<li>
<p style="text-align: left">
对其应用了蒙版的组
</p>
</li>
<li>
<p style="text-align: left">
<code>
image
</code>
本身带有
<code>
preserveAspectRatio =“ xMidYMid”
</code>
</p>
</li>
<li>
<p style="text-align: left">
用于内边界的圆
</p>
</li>
</ul>
<pre class="has"><code class="language-go">circle {
  stroke-width: 2;
  stroke: rgba(0, 0, 0, 0.1);
  fill: none;
}
</code></pre>
<p style="text-align: left">
事例地址：https://codepen.io/shadeed/pen/b17d34b5c23cc90fdc4573779544c8c7?editors=0100
</p>
