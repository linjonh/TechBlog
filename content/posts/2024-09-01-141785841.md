    <h4>
     <strong>
      2. 网易考拉 Megalo
     </strong>
    </h4>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="323" src="https://img-blog.csdnimg.cn/img_convert/858e47d2840eda128591409772dca449.webp?x-oss-process=image/format,png" width="1200"/>
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="152" src="https://img-blog.csdnimg.cn/img_convert/72799142e9112b845814a924094d1a25.webp?x-oss-process=image/format,png" width="1200"/>
    </p>
    <p>
     <code>
      Megalo
     </code>
     是基于 Vue（Vue@2.6.6） 的小程序开发框架，让开发者可以用 Vue 的开发方式开发小程序应用。Megalo 是为了跨 H5 和小程序两端的应用提供一个高效的解决方案，只需要少量改动即可完成 H5 和小程序之间的代码迁移。
    </p>
    <p>
     <code>
      Megalo
     </code>
     目前支持微信小程序，
     <a href="https://zhida.zhihu.com/search?q=%E6%94%AF%E4%BB%98%E5%AE%9D%E5%B0%8F%E7%A8%8B%E5%BA%8F&amp;zhida_source=entity&amp;is_preview=1" rel="nofollow" title="支付宝小程序">
      支付宝小程序
     </a>
     ，百度智能小程序，字节跳动小程序。
    </p>
    <p>
     github:
     <a href="https://link.zhihu.com/?target=https%3A//github.com/kaola-fed/megalo" title="https://github.com/kaola-fed/megalo">
      https://github.com/kaola-fed/megalo
     </a>

    <h4>
     <strong>
      3. 京东Taro
     </strong>
    </h4>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="648" src="https://img-blog.csdnimg.cn/img_convert/b284b24ea8f396a4286250f26541072c.webp?x-oss-process=image/format,png" width="1200"/>
    </p>
    <p>
     <code>
      Taro
     </code>
     是一个开放式跨端跨框架解决方案，支持使用 React/Vue/Nerv 等框架来开发 微信 / 京东 / 百度 / 支付宝 / 字节跳动 / QQ / 飞书 小程序 / H5 / RN 等应用。
    </p>
    <p>
     现如今市面上端的形态多种多样，Web、React Native、微信小程序等各种端大行其道。当业务要求同时在不同的端都要求有所表现的时候，针对不同的端去编写多套代码的成本显然非常高，这时候只编写一套代码就能够适配到多端的能力就显得极为需要。
    </p>
    <p>
     同时它还提供了一套京东风格的组件库
     <code>
      NutUI
     </code>
     :
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="521" src="https://img-blog.csdnimg.cn/img_convert/9f16f66495e7f644156ab08dfbeff262.webp?x-oss-process=image/format,png" width="1200"/>
    </p>
    <p>
     <code>
      NutUI
     </code>
     从功能、组件、性能、官网四方面放入手，不断在组件库的丰富性、轻量化、易用性上进行改进，力求打造一款好用的移动端组件库，为开发者提效，为业务赋能，为开源贡献一份力量。
    </p>
    <p>
     github地址:
     <a href="https://link.zhihu.com/?target=https%3A//github.com/nervjs/taro" title="https://github.com/nervjs/taro">
      https://github.com/nervjs/taro
     </a>
    </p>
    <h4>
     <strong>
      4. Remax
     </strong>
    </h4>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="494" src="https://img-blog.csdnimg.cn/img_convert/a13787f8b6196421e64444ba90d4dbef.webp?x-oss-process=image/format,png" width="1200"/>
    </p>
    <p>
     <code>
      remax
     </code>
     将 React 运行在小程序环境中，让我们可以使用完整的 React 进行小程序开发。 其具有如下特点:
    </p>
    <ul>
     <li>
      真正的 React - 不同于
      <a href="https://zhida.zhihu.com/search?q=%E9%9D%99%E6%80%81%E7%BC%96%E8%AF%91&amp;zhida_source=entity&amp;is_preview=1" rel="nofollow" title="静态编译">
       静态编译
      </a>
      的方案，在 Remax 中使用 React 没有任何限制，包括 React Hooks。我们可以把 Remax 理解为针对小程序的 React Native
     </li>
     <li>
      多端支持 - 使用 Remax 把代码转换到多个小程序平台
     </li>
     <li>
      TypeScript - 完整的 TypeScript 支持，给你满满的安全感
     </li>
    </ul>
    <p>
     github地址:
     <a href="https://link.zhihu.com/?target=https%3A//github.com/remaxjs" title="https://github.com/remaxjs">
      https://github.com/remaxjs
     </a>
    </p>
