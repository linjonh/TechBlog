     el-table 插槽踩过的坑 ：slot-scope 和#default的区别
    </h1>
   </div>

    <p>
     <strong>
      <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=slot-scope&amp;rsv_pq=cb40eb0200102c15&amp;oq=el-table%20slot-scope%20%E5%92%8C%23default%E7%9A%84%E5%8C%BA%E5%88%AB&amp;rsv_t=9898jTbxAzqdX23GTide1M7FAnTB97m2TAWvpRxWn4NwUh48KGAl0U9DC5s&amp;tn=baidu&amp;ie=utf-8" rel="nofollow" title="slot-scope">
       slot-scope
      </a>
      和#default是Vue中用于定义插槽的两种不同语法，它们在Vue 2和Vue 3中有不同的应用场景和语法规则。
     </strong>
     ‌
    </p>
    <h4>
     slot-scope
    </h4>
    <p>
     在‌
     <strong>
      <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Vue%202.x&amp;rsv_pq=cb40eb0200102c15&amp;oq=el-table%20slot-scope%20%E5%92%8C%23default%E7%9A%84%E5%8C%BA%E5%88%AB&amp;rsv_t=9898jTbxAzqdX23GTide1M7FAnTB97m2TAWvpRxWn4NwUh48KGAl0U9DC5s&amp;tn=baidu&amp;ie=utf-8" rel="nofollow" title="Vue 2.x">
       Vue 2.x
      </a>
     </strong>
     ‌中，slot-scope是用于声明具名插槽并获取父组件传递过来的数据的主要方式。通过slot-scope可以定义一个变量scope，该变量包含了当前行的数据（row）、列数据（column）和行索引（$index）等属性。例如：
    </p>
    <pre><code class="language-javascript">&lt;el-table-column label="性别"&gt;
  &lt;template slot-scope="scope"&gt;
    {<!-- -->{ scope.row.gender ? '男' : '女' }}
  &lt;/template&gt;
&lt;/el-table-column&gt;
</code></pre>
    <p>
     在这种写法中，slot-scope是标准的具名插槽数据访问方式，适用于Vue 2项目‌1。
    </p>
    <h4>
     #default
    </h4>
    <p>
     在‌
     <strong>
      <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Vue%203.0&amp;rsv_pq=cb40eb0200102c15&amp;oq=el-table%20slot-scope%20%E5%92%8C%23default%E7%9A%84%E5%8C%BA%E5%88%AB&amp;rsv_t=9898jTbxAzqdX23GTide1M7FAnTB97m2TAWvpRxWn4NwUh48KGAl0U9DC5s&amp;tn=baidu&amp;ie=utf-8" rel="nofollow" title="Vue 3.0">
       Vue 3.0
      </a>
     </strong>
     ‌中，#default是具名插槽的语法糖，提供了一种更简洁的方式来访问插槽数据。通过解构赋值，可以直接从插槽的作用域上下文中提取出需要的属性。例如：
    </p>
    <pre><code class="language-javascript">&lt;el-table-column label="性别"&gt;
  &lt;template #default="{ row }"&gt;
    {<!-- -->{ row.gender ? '男' : '女' }}
  &lt;/template&gt;
&lt;/el-table-column&gt;
</code></pre>
    <p>
     这种写法在需要提取多个属性时更加清晰和简洁，尤其是当需要访问index时，可以写成`template #default="{ row, $index }"`‌1。
    </p>
    <h4>
     使用场景和语法区别
    </h4>
    <ul>
     <li>
      ‌
      <strong>
       slot-scope
      </strong>
      ‌：适用于‌
      <strong>
       <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Vue%202&amp;rsv_pq=cb40eb0200102c15&amp;oq=el-table%20slot-scope%20%E5%92%8C%23default%E7%9A%84%E5%8C%BA%E5%88%AB&amp;rsv_t=9898jTbxAzqdX23GTide1M7FAnTB97m2TAWvpRxWn4NwUh48KGAl0U9DC5s&amp;tn=baidu&amp;ie=utf-8" rel="nofollow" title="Vue 2">
        Vue 2
       </a>
      </strong>
      ‌项目，是标准的具名插槽数据访问方式。如果项目使用Vue 2框架，必须使用slot-scope来实现对el-table单元格内容的自定义‌1。
     </li>
     <li>
      ‌
      <strong>
       #default
      </strong>
      ‌：适用于‌
      <strong>
       <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Vue%203&amp;rsv_pq=cb40eb0200102c15&amp;oq=el-table%20slot-scope%20%E5%92%8C%23default%E7%9A%84%E5%8C%BA%E5%88%AB&amp;rsv_t=9898jTbxAzqdX23GTide1M7FAnTB97m2TAWvpRxWn4NwUh48KGAl0U9DC5s&amp;tn=baidu&amp;ie=utf-8" rel="nofollow" title="Vue 3">
        Vue 3
       </a>
      </strong>
      ‌项目，提供了更简洁的语法糖，尤其适合需要提取多个属性的场景。这种新的语法让代码看起来更加简洁，尤其是在需要提取多个属性时，解构赋值的方式更加清晰‌1。
     </li>
    </ul>
