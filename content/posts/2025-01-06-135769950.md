    <p>
     我们通过下面的动图来回顾下手机本地模拟器的使用效果。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/7c157188d71c6968521fcf2b095127b1.gif">
      <br/>
      本期，我们将为大家介绍HarmonyOS本地模拟器的版本演进，并手把手教大家使用HarmonyOS本地模拟器。
     </img>
    </p>
    <p>
     一、本地模拟器的版本演进
     <br/>
     2021年12月31日，经过一个版本的迭代优化，随DevEco Studio 3.0 Beta2版本发布的本地模拟器新增了智能手表和智慧屏的模拟器品类。
    </p>
    <p>
     本地模拟器分别针对智能手表和智慧屏的特征，定制了其独有的特性。
     <br/>
     智能手表：支持心率、步数的模拟数据注入，支持旋转表冠按钮的模拟。
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/6707f9a36fdcbd760d6eacb9afe4867e.jpeg">
      <br/>
      图1 智能手表
      <br/>
      智慧屏：支持遥控器的模拟。
     </img>
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/a1d3bdba56ef337533e579c1d7c6500a.jpeg">
      <br/>
      图2 智慧屏
      <br/>
      下面，我们通过动图来看看智能手表和智慧屏品类本地模拟器的使用效果。
      <br/>
      <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/2b6018e0e7133413ec2ac79113479fbe.gif">
       <br/>
       通过以上对本地模拟器的介绍，不难看出本地模拟器具有以下优势：
       <br/>
       直接运行在用户本地计算机上，不需要登录授权，零延迟，也不受使用时长限制，拥有和真机一样的性能。
       <br/>
       提供audio、battery、location等多种通用器件模拟，为开发者提供了一个近似真机的运行环境。
      </img>
     </img>
    </p>
    <p>
     对华为设备提供产品专属模拟，如智能手表的旋转按压功能键，以及智慧屏的遥控器模拟等。
    </p>
    <p>
     支持Java、JS、C/C++单语言调试和JS+Java跨语言调试能力，帮助开发者更方便、高效地调试应用。
    </p>
    <p>
     二、使用前的准备工作
     <br/>
     了解了本地模拟器之后，下面我们要重点为大家讲解如何使用本地模拟器。在使用本地模拟器之前，我们需要做好以下准备工作：
    </p>
    <ol>
     <li>
      本地模拟器是创建和运行在本地计算机上的，需确保本地计算机满足以下资源要求：
      <br/>
      Windows系统：8GB及以上
      <br/>
      macOS系统：8GB及以上
     </li>
    </ol>
    <p>
     说明：不支持在虚拟机系统上运行本地模拟器，例如不支持在Ubuntu系统上，通过安装Windows虚拟机，然后使用Windows系统安装和运行模拟器。
    </p>
    <ol>
     <li>
      检查DevEco Studio的版本：
      <br/>
      DevEco Studio V3.0 Beta1及更高版本才支持本地模拟器，当前推荐使用DevEco Studio V3.0 Beta2版本。
     </li>
     <li>
      使用本地模拟器前，需要安装相应的镜像包，具体操作如下：
      <br/>
      （1）打开DevEco Studio，选择“Files &gt; Settings”（macOS系统则选择“DevEco Studio &gt; Preferences”）。
     </li>
    </ol>
    <p>
     （2）如图3所示，在“SDK Manager &gt; HarmonyOS Legacy SDK”页签，勾选“Platforms”下的镜像包。不同的的镜像包对应不同的设备类型，System-image-phone镜像包对应手机，System-image-tv镜像包对应智慧屏，System-image-wearable镜像包对应智能手表。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/6de6fbe226bdc52183732be5ab3c9beb.jpeg">
      <br/>
      图3 下载镜像包
      <br/>
      （3）点击“Apply”进行安装。
     </img>
    </p>
    <ol>
     <li>
      安装本地模拟器。
      <br/>
      在菜单栏选择“Tools &gt; Device Manager”，进入图4所示界面，点击“Install”即可安装本地模拟器。
     </li>
    </ol>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/2e87e9797a0960107fbb09e78a208ef4.jpeg">
      <br/>
      图4 安装本地模拟器
     </img>
    </p>
    <p>
     三、创建和使用本地模拟器
     <br/>
     完成了上面的准备工作，接下来就是本文的重点，教大家如何创建和使用本地模拟器。
    </p>
    <p>
     1.（可选）自定义本地模拟器文件存放路径。
     <br/>
     本地模拟器文件的默认存放路径为：C:\Users\用户名\AppData\Local\Huawei\HarmonyOSEmulator\deployed。如果C盘空间紧张，可以通过新增一个系统的环境变量HarmonyOS_HVD_HOME自定义存放路径，指向空间充裕的目录。
    </p>
    <ol>
     <li>
      创建本地模拟器。
      <br/>
      （1）打开DevEco Studio，选择“Tools &gt; Device Manager”。如图3所示，在Local Emulator页签中，点击右下角的New Emulator按钮，开始创建本地模拟器。
     </li>
    </ol>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/b65a65c4db868c701b4c760c4a7c22fa.jpeg">
      <br/>
      图5 创建本地模拟器
      <br/>
      （2）选择一个设备来创建本地模拟器。
     </img>
    </p>
    <p>
     如图4所示，DevEco Studio默认提供Huawei_Phone、Huawei_TV和Huawei_Wearable三个设备，分别对应手机、智慧屏和智能手表。针对华为设备的特征，DevEco Studio对这三个设备预置了尺寸、分辨率等参数。
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/c53e30696a3ca1da6a3fbb6f59e8fa43.jpeg">
      <br/>
      图6 选择一个设备
      <br/>
      开发者点击New Hardware或Huawei_Phone后的克隆 图标，可以创建新的手机设备，自定义设备的名称、尺寸、分辨率、内存等参数（如图7）。
      <br/>
      <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/8216102decd3e653029d4aace14f6a95.jpeg"/>
      <br/>
      图7 自定义手机参数
      <br/>
      开发者从默认的三个设备或者创建的手机设备中，选择一个设备来创建本地模拟器。
      <br/>
      （3）选择好设备后，点击Next，可以看到本地模拟器的镜像信息。
      <br/>
      （4）再次点击Next，检查本地模拟器的配置信息，如有需要此时还可以修改配置信息。确认无误后，点击Finish完成本地模拟器的创建。
      <br/>
      创建成功后，在Local Emulator页签可以看到新创建的本地模拟器，如图8所示。
      <br/>
      <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/94c73dbd86c7e726b405dfd8bfe9bad5.jpeg"/>
      <br/>
      图8 本地模拟器创建成功
     </img>
    </p>
    <ol>
     <li>
      运行本地模拟器。
      <br/>
      （1）在Local Emulator页签（如图8），点击即可启动本地模拟器。
     </li>
    </ol>
    <p>
     （2）点击DevEco Studio工具栏中的按钮运行工程，或使用默认快捷键Shift+F10（Mac为Control+R）运行工程。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/faa04c14b99f8296efecb3dac7d986a7.jpeg"/>
    </p>
    <p>
     图9 运行工程
     <br/>
     （3）DevEco Studio会启动应用的编译构建，完成后应用即可运行在本地模拟器上。以手机设备为例，运行效果如图10所示。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/c05dc407e2c53ce4d833fdba8f6ad714.jpeg"/>
     <br/>
     图10 运行本地模拟器
     <br/>
     图8所示的手机本地模拟器，除了可以像真机一样直接在手机屏幕内操作，右侧的工具栏提供了操作便捷、丰富和数据注入能力，包括：调整音量、电池模拟、GPS模拟、网络模拟、虚拟传感器模拟等。
     <br/>
     本地模拟器目前支持手机、智能手表和智慧屏三种设备类型。图11展示了同一套代码同时在智能手表、手机和智慧屏三个设备上的运行效果。
     <br/>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/051e83d65199e51d74527c63d68a8b19.jpeg"/>
     <br/>
     图11 本地模拟器运行效果
    </p>
