    <h2>
     <a id="_258">
     </a>
     自定义指令
    </h2>
    <p>
     我理解的自定义指令就是你可以把
     <strong>
      功能或者组件单独写出来
     </strong>
     ，以供后期
     <strong>
      复用
     </strong>
     ，相当于
     <strong>
      vue的component组件
     </strong>
    </p>
    <h3>
     <a id="_260">
     </a>
     介绍
    </h3>
    <ul>
     <li>
      <p>
       restrict
      </p>
      <ul>
       <li>
        E ： 标签形式 eg: &lt;hello&gt;
       </li>
       <li>
        A : 属性形式 eg： &lt;div hello /&gt;
       </li>
       <li>
        C: class形式 eg &lt;div class=“hello” /&gt;
       </li>
       <li>
        M: 注释写法 eg： &lt;-- directive:hello --&gt;
        <strong>
         注释需要replace: true 可以组合使用
        </strong>
        <br/>
        <mark>
         一般会使用前两个也就是‘E’和‘A’,一般A是功能性的使用较多，E一般是模板替代形式跟vue中组件似的
        </mark>
       </li>
      </ul>
     </li>
     <li>
      <p>
       replace
      </p>
      <ul>
       <li>
        true : 里面内容直接替换外面标签，要不总在写的里面
       </li>
      </ul>
     </li>
     <li>
      <p>
       template
      </p>
     </li>
     <li>
      <p>
       templateUrl
      </p>
     </li>
     <li>
      <p>
       scope
      </p>
      <ul>
       <li>
        true 独立作用域，如果不写这个scope的话就会所有组件公用一套数据，在一个上面改，其他的都会变
       </li>
       <li>
        {} 隔离作用域，不再跟外层作用于有任何瓜葛，独立作用域还会集成上面的Aaa里面数据，这个隔离就不会继承了，只能使用什么就传过来
        <ul>
         <li>
          @ 就是解析字符串
         </li>
         <li>
          = 找变量
         </li>
         <li>
          &amp; 函數
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       link —（scope, element, attr,reController） (reController就是拿别的组件里面的属性值，下面讲),主要用于操作dom的,跟controller的区别自己感觉是controller是可以吧公共逻辑都放在里面,当如果集成另一个指令,想拿另一个指令的定义的变量就可以用link的reController么
      </p>
     </li>
     <li>
      <p>
       controller --就写逻辑的地方
      </p>
     </li>
    </ul>
