    <h2>
     什么是配置文件
    </h2>
    <p>
     “配置文件”是一个用来保护程序或者系统设置信息的文件，它的作用是让程序在启动或者运行中，能够读取这些设置并按预期进行工作，而不需要手动的设置。
    </p>
    <h2>
     Spring Boot 配置文件
    </h2>
    <ul>
     <li>
      设置服务器端口、编码格式
     </li>
     <li>
      配置数据库连接
     </li>
     <li>
      控制日志级别
     </li>
     <li>
      设置缓存、邮件、消息队列等服务参数
     </li>
     <li>
      管理不同环境（开发、测试、生产）下的配置
     </li>
    </ul>
    <p>
     <img alt="" height="89" src="https://i-blog.csdnimg.cn/direct/40e2ef9f66624dd0920962b6da504c29.png" width="328"/>
    </p>
    <p>
     Spring Boot中的配置文件一共有两种形式
    </p>
    <h2>
     <span style="color:#fe2c24">
      <strong>
       .properties
      </strong>
     </span>
     文件 和
     <span style="color:#fe2c24">
      <strong>
       .yml  /.yaml
      </strong>
     </span>
     文件
    </h2>
    <h3>
     properties 基本语法
    </h3>
    <p>
     properties 是以键值的形式配置的，key和value之间是以"="连接的
    </p>
    <pre><code class="language-java">spring.application.name=spring-ioc demo
server.port=8081


spring.datasource.url=jdbc:mysql://127.0.0.1:3306/testdb?characterEncoding=utf8&amp;useSSL=false
spring.datasource.username=root
spring.datasource.password=root


#自定义
my.key=10
my.key2=true</code></pre>
    <p>
    </p>
    <h3>
     yml基本语法
    </h3>
    <p>
     yml是树形结构的配置⽂件，它的基础语法是"key:value"
    </p>
    <p>
     <span style="color:#fe2c24">
      <strong>
       记住：在key：value中“ ：”后一定要加空格
      </strong>
     </span>
    </p>
    <pre><code class="language-java">server:
  port: 8081

my:
  key: 11


spring.datasource.url=jdbc:mysql://127.0.0.1:3306/testdb?characterEncoding=utf8&amp;useSSL=false

spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/testdb?characterEncoding=utf8&amp;useSSL=false
    username: admin
    password: 123456


#自定义的key
aa.bb.cc=5
aa.bb.dd=6
aa.bb.cc: 5
aa.bb.dd: 6

#字符串
string.value: hello

#布尔值，true/false
boolean.value: false
boolean.value1: true

#整数
int.value: 12

#浮点数
float.value: 12.12


#Null，~代表Null
null.value: ~

  "" 空字符串
empty.value: ''
#里面什么都不用写</code></pre>
    <p>
    </p>
    <h3>
     yml的配置读取
    </h3>
    <p>
     <strong>
      用@value注解和@ConfigurationProperties注解来读取文件
     </strong>
    </p>
    <p>
     <strong>
      两个注解的区别：@value注解用于读取单个配置项，@ConfigurationProperties注解用于批量注入多个配置项
     </strong>
    </p>
    <h4>
     <strong>
      @value注解
     </strong>
    </h4>
    <pre><code class="language-java">@Controller
public class YmlController {

    @Value("${my.key3}")
    private String myKey3;

    @Value("${my.key4}")
    private String myKey4;

    @PostConstruct
    public void print(){
        System.out.println(myKey3);
        System.out.println(myKey4);
    }
}</code></pre>
    <pre><code class="language-java">my:
  key3: Hello
  key4: 5
</code></pre>
    <p>
     <img alt="" height="55" src="https://i-blog.csdnimg.cn/direct/4c70242498a9451cb0440dfa46a3d137.png" width="458"/>
    </p>
    <h4>
     <strong>
      @ConfigurationProperties注解
     </strong>
    </h4>
    <p>
     <strong>
      集合和Map的配置
     </strong>
    </p>
    <pre><code class="language-java">@Data
@ConfigurationProperties(prefix = "dbtypes")
@Component
public class DbTypes {
    private Integer id;
    private List&lt;String&gt; name;
    private Map&lt;String,String&gt; ball;
}

@Data
@Component
@ConfigurationProperties(prefix = "person")
public class Person {
    private Integer id;
    private String name;
    private Integer age;
}

</code></pre>
    <pre><code class="language-java">@Controller
public class YmlController {

   
    @Autowired
    private Person person;

    @Autowired
    private DbTypes dbTypes;

    @PostConstruct
    public void print(){
        System.out.println(person);
        System.out.println(dbTypes);
    }
}</code></pre>
    <p>
     在yml配置文件中
    </p>
    <pre><code class="language-java">person:
  id: 1
  name: zhangsan
  age: 15

dbtypes:
  id: 4
  name:
    - 123
    - sa
    - ss
  ball:
    k1: v1
    k2: v2
    k3: v3</code></pre>
    <p>
     <img alt="" height="61" src="https://i-blog.csdnimg.cn/direct/379bef1382a743d6a1c26e32bf23e7f8.png" width="748"/>
    </p>
    <p>
    </p>
    <h3>
     yml的优缺点：
    </h3>
    <h4>
     优点
    </h4>
    <p>
     1.结构清晰，一目了然
    </p>
    <p>
     <img alt="" height="165" src="https://i-blog.csdnimg.cn/direct/e25b57dfab8f4da38c3afa7d1913aff5.png" width="947"/>
    </p>
    <p>
     2.支持复杂数据类型（List、Map 等）
    </p>
    <p>
     <img alt="" height="185" src="https://i-blog.csdnimg.cn/direct/16a607120ad94b3eaf5b2c632d90a6ad.png" width="380"/>
    </p>
    <p>
     3.语法简洁，没有重复前缀
    </p>
    <p>
     4.可读性强
    </p>
    <p>
    </p>
    <h4>
     缺点
    </h4>
    <p>
     1.缩进必须正确，容易出错
    </p>
    <p>
     2.不适合写大量注释
    </p>
    <p>
     3.调试时不容易发现问题
    </p>
    <p>
    </p>
    <h2>
     验证码案例
    </h2>
    <p>
     随着安全性的要求越来越⾼,⽬前项⽬中很多都使⽤了验证码,验证码的形式也是多种多样,更复杂的图 形验证码和⾏为验证码已经成为了更流⾏的趋势。
    </p>
    <p>
     我们通过Hutool提供的小工具来实现验证码
    </p>
    <h3>
     需求：
    </h3>
    <p>
     在页面上生成验证码图案，然后通过输入验证码图形输入正确的验证码通过验证。
    </p>
    <p>
     首先我们打开Hutool，找到图形验证码
    </p>
    <p>
     <img alt="" height="827" src="https://i-blog.csdnimg.cn/direct/07bc5fc7d4f347c8ba409d76f889e625.png" width="1699"/>
    </p>
    <p>
    </p>
    <h3>
     操作流程：
    </h3>
    <p>
     导入包，引入依赖
    </p>
    <pre><code class="language-java">        &lt;dependency&gt;
            &lt;groupId&gt;cn.hutool&lt;/groupId&gt;
            &lt;artifactId&gt;hutool-all&lt;/artifactId&gt;
            &lt;version&gt;5.8.16&lt;/version&gt;
        &lt;/dependency&gt;</code></pre>
    <p>
     首先我们先生成验证码
    </p>
    <p>
     我们生成的是圆圈干扰图形的验证码
    </p>
    <p>
     <img alt="" height="276" src="https://i-blog.csdnimg.cn/direct/fd74fcadc39f40aba11fc40861919cb0.png" width="491"/>
    </p>
    <pre><code class="language-java"> CircleCaptcha captcha = CaptchaUtil.createCircleCaptcha(captchaProperties.getWidth(),captchaProperties.getLength());
 String code=captcha.getCode();</code></pre>
    <p>
     由于用户输入完验证码，我们需要对比是否输入正确，所以我们需要得到正确的验证码，所以我们需要在生成验证码后，将正确的验证码存到Session中
    </p>
    <pre><code class="language-java">     session.setAttribute(captchaProperties.getSession().getKey(),code);
     session.setAttribute(captchaProperties.getSession().getDate(),new Date());
     captcha.write(response.getOutputStream());</code></pre>
    <p>
     然后我们需要进行一个校验 ,判断用户输入的验证码是否正确
    </p>
    <pre><code class="language-java">if (StringUtil.isNullOrEmpty(captcha)){
            return false;
        }
        String code =(String) session.getAttribute(captchaProperties.getSession().getKey());
        Date date =(Date) session.getAttribute(captchaProperties.getSession().getDate());
        if(captcha.equalsIgnoreCase(code) &amp;&amp; date!=null  &amp;&amp; System.currentTimeMillis()-date.getTime()&lt;VALID_TIME){
            return true;
        }
        return false;</code></pre>
    <p>
    </p>
    <p>
     我们将图形的大小写入配置文件中
    </p>
    <p>
     <img alt="" height="34" src="https://i-blog.csdnimg.cn/direct/3658f6ba202749e7b7ebcbb673b5ce6d.png" width="746"/>
    </p>
    <pre><code class="language-java">captcha:
  width: 150
  length: 100
  session:
    key: CAPTCHA_SESSION_KEY
    date: CAPTCHA_SESSION_DATE</code></pre>
    <p>
     并且用户在输入验证码时有时间限制，超过了一定的时间，就会失效，所以我们需要定义一段时间限制
    </p>
    <p>
     一分钟
    </p>
    <pre><code class="language-java">private final static Long VALID_TIME=60*30*1000L;</code></pre>
    <p>
     系统现在的时间 - 输入开始的时间 &lt; 一分钟
    </p>
    <pre><code class="language-java">System.currentTimeMillis()-date.getTime()&lt;VALID_TIME</code></pre>
    <p>
     然后在前端代码中写入接口
    </p>
    <pre><code class="language-javascript"> $.ajax({
        type:"post",
        url:"/captcha/check",
        data:{
          captcha:$("#inputCaptcha").val()
        },
        success:function(result){
          if(result){
            location.href="success.html";
          }else{
            alert("输入失败，请重新输入");
          }
        }</code></pre>
    <p>
     并且在前段代码中写入
    </p>
    <pre><code class="language-html">  &lt;meta charset="utf-8"&gt;
  &lt;meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate" /&gt;
  &lt;meta http-equiv="Pragma" content="no-cache" /&gt;
  &lt;meta http-equiv="Expires" content="0" /&gt;</code></pre>
    <p>
     它们用于防止浏览器缓存网页内容，确保每次都从服务器加载最新资源（特别适合验证码、表单、动态页面等场景）。
    </p>

    <pre><strong>CaptchaController</strong>
</pre>
    <pre><code class="language-java">package com.blame.springcaptcha.captcahController;

import ch.qos.logback.core.util.StringUtil;
import cn.hutool.captcha.CaptchaUtil;
import cn.hutool.captcha.CircleCaptcha;
import com.blame.springcaptcha.model.CaptchaProperties;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import java.io.IOException;
import java.util.Date;

@RequestMapping("captcha")
@RestController
public class CaptchaController {

    @Autowired
    private CaptchaProperties captchaProperties;

    private final static Long VALID_TIME=60*30*1000L;

    @RequestMapping("/getCaptcha")
    public void getCaptcha(HttpServletResponse response, HttpSession session) throws IOException {
        response.setContentType("image/jpeg");
        CircleCaptcha captcha = CaptchaUtil.createCircleCaptcha(captchaProperties.getWidth(),captchaProperties.getLength());
        String code=captcha.getCode();
        session.setAttribute(captchaProperties.getSession().getKey(),code);
        session.setAttribute(captchaProperties.getSession().getDate(),new Date());
        captcha.write(response.getOutputStream());

    }

    @RequestMapping("/check")
    public boolean check(HttpSession session,String captcha){
        if (StringUtil.isNullOrEmpty(captcha)){
            return false;
        }
        String code =(String) session.getAttribute(captchaProperties.getSession().getKey());
        Date date =(Date) session.getAttribute(captchaProperties.getSession().getDate());
        if(captcha.equalsIgnoreCase(code) &amp;&amp; date!=null  &amp;&amp; System.currentTimeMillis()-date.getTime()&lt;VALID_TIME){
            return true;
        }
        return false;
    }

}
</code></pre>
