    <h2 id="FTQ4K">
     起因
    </h2>
    在工作中使用git pull突然发现仓库出现了找不到代码库问题，但是这个时候有个对策又急着需要，于是乎，就需要备份，拷贝给另一个工程师输出。
    <h2 id="IOYPZ">
     git 打补丁操作
    </h2>
    工程师A生成补丁文件
    <pre><code class="prism language-git">touch a.txt

git add a.txt

git commit -m "aaa"

#前三条命令是模拟一个提交，最后一条是生成patch文件
git format-patch -1
</code></pre>
    <p>
     工程师A然后讲这个patch文件发送给工程师B
    </p>
    <p>
     工程师B然后操作，之后就将补丁给打上了。
    </p>
    <pre><code class="prism language-git">git apply patch文件名
#或者使用下面这个
git am patch文件名
</code></pre>
    <p>
     <strong>
      两个命令区别：git apply
     </strong>
     命令会修改本地文件而不创建提交，而
     <strong>
      git am
     </strong>
     会修改文件并创建提交
    </p>
