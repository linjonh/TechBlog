    <h4>
     <a id="_1">
     </a>
     参数解释
    </h4>
    <ul>
     <li>
      <code>
       enable_stmt_track
      </code>
      ：是否启用Full/Slow SQL特性
     </li>
     <li>
      <code>
       log_min_duration_statement
      </code>
      ：指定慢 SQL 时间阈值
     </li>
     <li>
      <code>
       instr_unique_sql_count
      </code>
      ：当系统中产生的unique SQL条目数量（dbe_perf.statement/dbe_perf.summary_statement统计）大于instr_unique_sql_count时，若开启了unique SQL自动淘汰，则系统会按unique SQL的更新时间由远到近自动淘汰一定比例的条目，使得新产生的unique SQL信息可以继续被统计。若没有开启自动淘汰，则系统产生的新的unique SQL信息将不再被统计。
     </li>
     <li>
      <code>
       enable_auto_clean_unique_sql
      </code>
      ：当系统中产生的unique SQL条目数量大于等于instr_unique_sql_count时，是否启用unique SQL自动淘汰功能。
     </li>
     <li>
      <code>
       track_stmt_retention_time
      </code>
      ：组合参数，控制全量/慢SQL记录的保留时间
     </li>
     <li>
      <code>
       track_stmt_stat_level
      </code>
      ：组合参数，控制语句执行跟踪的级别，全量/慢SQL跟踪级别。参数级别为L1则记录query_plan。
     </li>
     <li>
      <code>
       track_activity_query_size
      </code>
      ：设置用于跟踪每一个活动会话的当前正在执行命令的字节数
     </li>
    </ul>
