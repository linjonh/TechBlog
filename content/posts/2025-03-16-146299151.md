    <h2>
     <a id="Solidity__17">
     </a>
     Solidity 基础知识点总览
    </h2>
    <h3>
     <a id="1_Solidity__19">
     </a>
     1. Solidity 简介
    </h3>
    <p>
     Solidity 是一种用于编写智能合约的高级编程语言，专为以太坊虚拟机（EVM）设计。智能合约是自动执行和控制的计算机程序，它们在区块链上运行，无需中间人。
    </p>
    <h3>
     <a id="2__23">
     </a>
     2. 环境设置
    </h3>
    <ul>
     <li>
      安装 Node.js 和 npm（Node Package Manager）。
     </li>
     <li>
      安装 Truffle 或 Hardhat 等开发框架。
     </li>
     <li>
      配置以太坊测试网络（如 Ganache）或连接到真实的以太坊网络。
     </li>
    </ul>
    <h3>
     <a id="3__29">
     </a>
     3. 智能合约基础
    </h3>
    <h4>
     <a id="31__31">
     </a>
     3.1 合约结构
    </h4>
    <p>
     一个基本的 Solidity 智能合约通常包含以下几个部分：
    </p>
    <ul>
     <li>
      <code>
       pragma solidity ^版本号;
      </code>
      ：指定 Solidity 编译器版本。
     </li>
     <li>
      <code>
       contract 合约名 { ... }
      </code>
      ：定义合约。
     </li>
     <li>
      状态变量（state variables）：存储在区块链上的变量。
     </li>
     <li>
      函数（functions）：可执行的操作。
     </li>
    </ul>
    <h4>
     <a id="32__40">
     </a>
     3.2 示例合约：简单的存储合约
    </h4>
    <pre><code class="prism language-solidity">// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 storedData; // 状态变量

    // 读取存储的数据
    function retrieve() public view returns (uint256) {
        return storedData;
    }

    // 存储数据
    function store(uint256 x) public {
        storedData = x;
    }
}
</code></pre>
    <h4>
     <a id="33__61">
     </a>
     3.3 合约部署与交互
    </h4>
    <p>
     使用 Truffle 或 Hardhat 等框架可以方便地部署和与智能合约交互。以下是一个使用 Truffle 部署上述合约的简要步骤：
    </p>
    <ol>
     <li>
      创建一个新的 Truffle 项目。
     </li>
     <li>
      将上述合约代码保存到
      <code>
       contracts/SimpleStorage.sol
      </code>
      文件中。
     </li>
     <li>
      编写迁移脚本（migrations）来部署合约。
     </li>
     <li>
      使用
      <code>
       truffle compile
      </code>
      编译合约。
     </li>
     <li>
      使用
      <code>
       truffle migrate
      </code>
      部署合约到测试网络。
     </li>
     <li>
      使用
      <code>
       truffle console
      </code>
      或其他工具与合约交互。
     </li>
    </ol>
    <h3>
     <a id="4_Solidity__72">
     </a>
     4. Solidity 关键点
    </h3>
    <h4>
     <a id="41__74">
     </a>
     4.1 数据类型
    </h4>
    <ul>
     <li>
      值类型（Value Types）：如
      <code>
       uint
      </code>
      ,
      <code>
       int
      </code>
      ,
      <code>
       address
      </code>
      ,
      <code>
       bool
      </code>
      等。
     </li>
     <li>
      引用类型（Reference Types）：如
      <code>
       mapping
      </code>
      ,
      <code>
       array
      </code>
      ,
      <code>
       struct
      </code>
      等。
     </li>
    </ul>
    <h4>
     <a id="42__79">
     </a>
     4.2 函数修饰符
    </h4>
    <ul>
     <li>
      <code>
       public
      </code>
      ：函数可以从合约外部和内部调用。
     </li>
     <li>
      <code>
       private
      </code>
      ：函数只能从合约内部调用。
     </li>
     <li>
      <code>
       external
      </code>
      ：函数只能从合约外部调用。
     </li>
     <li>
      <code>
       view
      </code>
      ：表示函数不会修改区块链上的状态。
     </li>
     <li>
      <code>
       pure
      </code>
      ：表示函数既不读取也不修改区块链上的状态。
     </li>
     <li>
      <code>
       payable
      </code>
      ：允许函数接收以太币。
     </li>
    </ul>
    <h4>
     <a id="43_Events_88">
     </a>
     4.3 事件（Events）
    </h4>
    <p>
     事件是合约中用于记录日志的特殊函数，它们可以被区块链外部的应用程序监听。
    </p>
    <pre><code class="prism language-solidity">event DataStored(uint256 indexed value);

function storeWithEvent(uint256 x) public {
    storedData = x;
    emit DataStored(x); // 触发事件
}
</code></pre>
    <h4>
     <a id="44_Errors_101">
     </a>
     4.4 错误处理（Errors）
    </h4>
    <p>
     Solidity 提供了自定义错误的功能，可以在合约中定义错误并在触发时抛出。
    </p>
    <pre><code class="prism language-solidity">error InsufficientFunds(uint256 required, uint256 provided);

function withdraw(uint256 amount) public {
    if (amount &gt; msg.value) {
        revert InsufficientFunds({required: amount, provided: msg.value});
    }
    // 处理提款逻辑
}
</code></pre>
