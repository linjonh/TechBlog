    <h3>
     一、NVM和Yarn
    </h3>
    <p>
    </p>
    <p>
     主要是后端开发，但是最近写毕业设计发现前端这边对环境的要求比较严格，比如去执行 npm install 的时候，会出现20的版本可以拉下来依赖，但是npm run dev 时候却启动不了，要换成10的版本去启动。
    </p>
    <p>
    </p>
    <p>
     那么这里来来回回换版本就不舒服，于是就得使用 nvm 和 yarn这俩工具，看看它们的作用以及区别在哪里。
    </p>
    <p>
    </p>
    <h3>
     二、NVM 的作用
    </h3>
    <p>
    </p>
    <p>
     <code>
      nvm
     </code>
     ，即
     <code>
      Node Version Manager
     </code>
     ，是一个用于管理
     <code>
      Node.js
     </code>
     版本的工具。在前端开发中，不同的项目可能依赖于不同版本的
     <code>
      Node.js
     </code>
     。有些老项目可能需要较低版本的
     <code>
      Node.js
     </code>
     才能正常运行，而一些新项目则可能需要最新版本的
     <code>
      Node.js
     </code>
     以享受新特性和性能优化。
    </p>
    <p>
    </p>
    <p>
     <code>
      nvm
     </code>
     的主要作用体现在以下几个方面：
    </p>
    <p>
    </p>
    <ol>
     <li>
      <strong>
       多版本切换
      </strong>
      ：通过
      <code>
       nvm
      </code>
      ，开发者可以轻松地在不同版本的
      <code>
       Node.js
      </code>
      之间进行切换。例如，当我们要开发一个基于旧版
      <code>
       Node.js
      </code>
      的项目时，只需使用
      <code>
       nvm use &lt;旧版本号&gt;
      </code>
      命令，就可以切换到相应的版本。当项目完成后，又可以切换回最新版本继续其他项目的开发。
     </li>
     <li>
      <strong>
       安装特定版本
      </strong>
      ：
      <code>
       nvm
      </code>
      允许我们根据项目需求安装特定版本的
      <code>
       Node.js
      </code>
      。我们可以使用
      <code>
       nvm install &lt;版本号&gt;
      </code>
      命令来安装所需的版本，无需担心与系统中已安装的其他版本产生冲突。
     </li>
     <li>
      <strong>
       管理全局包
      </strong>
      ：在不同版本的
      <code>
       Node.js
      </code>
      下，我们可能需要安装不同的全局包。
      <code>
       nvm
      </code>
      可以帮助我们在不同版本之间独立管理这些全局包，确保每个项目都能使用到正确的依赖。
     </li>
    </ol>
    <p>
    </p>
    <h3>
     三、Yarn 的作用
    </h3>
    <p>
    </p>
    <p>
     <code>
      yarn
     </code>
     是一个快速、可靠、安全的
     <code>
      JavaScript
     </code>
     包管理器。在前端项目中，我们通常需要使用各种第三方包来实现不同的功能，而
     <code>
      yarn
     </code>
     就是用来管理这些包的工具。
    </p>
    <p>
    </p>
    <ol>
     <li>
      <strong>
       快速安装
      </strong>
      ：
      <code>
       yarn
      </code>
      采用了并行安装的方式，能够同时下载多个包，大大提高了安装速度。与传统的
      <code>
       npm
      </code>
      相比，
      <code>
       yarn
      </code>
      在安装大型项目的依赖时表现得更加出色。
     </li>
     <li>
      <strong>
       版本管理
      </strong>
      ：
      <code>
       yarn
      </code>
      可以精确地管理包的版本。它会根据项目的
      <code>
       package.json
      </code>
      文件中的配置，安装指定版本的包，并且在升级包时会提示可能的兼容性问题，确保项目的稳定性。
     </li>
     <li>
      <strong>
       离线模式
      </strong>
      ：在一些网络环境较差的情况下，
      <code>
       yarn
      </code>
      的离线模式非常有用。只要我们之前已经安装过某个包，
      <code>
       yarn
      </code>
      就可以在离线状态下从缓存中获取该包，避免了重复下载。
     </li>
     <li>
      <strong>
       工作区
      </strong>
      ：
      <code>
       yarn
      </code>
      支持工作区功能，这对于管理多个相关的项目非常方便。我们可以在一个根目录下创建多个子项目，并且通过
      <code>
       yarn
      </code>
      来统一管理它们的依赖，减少了重复安装和管理的麻烦。
     </li>
    </ol>
    <p>
    </p>
    <h3>
     四、NVM 和 Yarn 的区别
    </h3>
    <p>
    </p>
    <ol>
     <li>
      <strong>
       功能领域
      </strong>
      ：
      <code>
       nvm
      </code>
      主要专注于
      <code>
       Node.js
      </code>
      版本的管理，解决的是不同项目对
      <code>
       Node.js
      </code>
      版本的依赖问题。而
      <code>
       yarn
      </code>
      则是用于管理项目中的第三方包，包括包的安装、更新、卸载等操作。
     </li>
     <li>
      <strong>
       操作对象
      </strong>
      ：
      <code>
       nvm
      </code>
      的操作对象是
      <code>
       Node.js
      </code>
      本身，它可以安装、切换、卸载不同版本的
      <code>
       Node.js
      </code>
      。而
      <code>
       yarn
      </code>
      的操作对象是项目中的各种
      <code>
       JavaScript
      </code>
      包，通过
      <code>
       package.json
      </code>
      文件来管理包的依赖关系。
     </li>
     <li>
      <strong>
       使用场景
      </strong>
      ：当我们遇到项目对
      <code>
       Node.js
      </code>
      版本有特殊要求时，就需要使用
      <code>
       nvm
      </code>
      来切换版本。而在项目开发过程中，当我们需要添加、更新或删除第三方包时，就会用到
      <code>
       yarn
      </code>
      。
     </li>
    </ol>
    <p>
    </p>
    <p>
     <code>
      nvm
     </code>
     和
     <code>
      yarn
     </code>
     都是前端开发中非常重要的工具。
     <code>
      nvm
     </code>
     帮助我们解决了
     <code>
      Node.js
     </code>
     版本的管理问题，而
     <code>
      yarn
     </code>
     则让我们在管理项目依赖时更加得心应手。熟练掌握这两个工具的使用，能够大大提高我们的开发效率，让我们的前端开发工作更加顺畅。
    </p>
    <p>
    </p>
    <p>
     希望通过这篇博客，大家对
     <code>
      nvm
     </code>
     和
     <code>
      yarn
     </code>
     有了更清晰的认识和理解。在今后的开发中，能够合理地运用它们，为项目的成功保驾护航。
    </p>
    <p>
    </p>
