    <h1 class="title-article" id="articleContentId">
     如何重置 MySQL root 用户的登录密码？
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <h4>
     重置 MySQL root 密码的核心步骤是
     <strong>
      绕过权限验证登录数据库
     </strong>
     并
     <strong>
      更新密码字段
     </strong>
     。以下是具体方法：
    </h4>
    <hr/>
    <h5>
     <strong>
      方法一：通过
      <code>
       --SKIP-GRANT-TABLES
      </code>
      模式修改密码
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        停止 MySQL 服务
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         Windows
        </strong>
        ：在命令行执行
        <code>
         net stop mysql
        </code>
        （服务名可能为
        <code>
         mysql80
        </code>
        或
        <code>
         mysql57
        </code>
        ，需根据实际名称调整）
        <em>
         4
        </em>
        <em>
         5
        </em>
        。
       </li>
       <li>
        <strong>
         Linux
        </strong>
        ：执行
        <code>
         sudo systemctl stop mysql
        </code>
        或
        <code>
         sudo service mysql stop
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        以跳过权限验证模式启动 MySQL
       </strong>
      </p>
      <ul>
       <li>
        打开命令行（Windows）或终端（Linux），输入：
        <pre></pre>
        <p>
         Bash
        </p>
        <p>
         <a id="copy-1741843605668">
         </a>
        </p>
        <code id="code-1741843605668">
         mysqld --skip-grant-tables --shared-memory
        </code>
       </li>
       <li>
        此时 MySQL 会忽略权限验证，允许无密码登录。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        登录并修改密码
       </strong>
      </p>
      <ul>
       <li>
        新开一个命令行/终端窗口，输入：
        <pre></pre>
        <p>
         Bash
        </p>
        <p>
         <a id="copy-1741842072591">
         </a>
        </p>
        <code id="code-1741842072591">
         mysql -u root
        </code>
       </li>
       <li>
        执行以下 SQL 语句（注意版本差异）：
        <ul>
         <li>
          <strong>
           MySQL 5.7 及以上
          </strong>
          ：
          <pre></pre>
          <p>
           Sql
          </p>
          <p>
           <a id="copy-1741838767880">
           </a>
          </p>
          <code id="code-1741838767880">
           UPDATE mysql.user SET authentication_string=PASSWORD('新密码') WHERE User='root'; FLUSH PRIVILEGES;
          </code>
         </li>
         <li>
          <strong>
           MySQL 8.0 及以上
          </strong>
          ：
          <pre></pre>
          <p>
           Sql
          </p>
          <p>
           <a id="copy-1741836160266">
           </a>
          </p>
          <code id="code-1741836160266">
           ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '新密码'; FLUSH PRIVILEGES;
          </code>
         </li>
        </ul>
        <em>
         3
        </em>
        <em>
         4
        </em>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        重启 MySQL 服务
       </strong>
      </p>
      <ul>
       <li>
        关闭所有 MySQL 进程，重新启动服务：
        <pre></pre>
        <p>
         Bash
        </p>
        <p>
         <a id="copy-1741843815088">
         </a>
        </p>
        <code id="code-1741843815088">
         net start mysql # Windows sudo systemctl start mysql # Linux
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      方法二：使用
      <code>
       --INIT-FILE
      </code>
      参数（推荐用于生产环境）
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        创建密码修改脚本
       </strong>
      </p>
      <ul>
       <li>
        新建文件
        <code>
         reset_password.sql
        </code>
        ，内容为：
        <pre></pre>
        <p>
         Sql
        </p>
        <p>
         <a id="copy-1741839049572">
         </a>
        </p>
        <code id="code-1741839049572">
         ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
        </code>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        停止 MySQL 服务
       </strong>
       （同上）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        启动 MySQL 并指定初始化文件
       </strong>
      </p>
      <pre></pre>
      <p>
       Bash
      </p>
      <p>
       <a id="copy-1741844945271">
       </a>
      </p>
      <code id="code-1741844945271">
       mysqld --init-file=/path/to/reset_password.sql
      </code>
      <ul>
       <li>
        服务启动后会自动执行脚本更新密码
        <em>
         1
        </em>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        重启服务正常模式
       </strong>
       。
      </p>
     </li>
    </ol>
    <hr/>
    <h5>
     <strong>
      注意事项
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       如果出现
       <code>
        authentication_string
       </code>
       字段更新失败，尝试先置空密码再修改：
      </p>
      <pre></pre>
      <p>
       Sql
      </p>
      <p>
       <a id="copy-1741840060141">
       </a>
      </p>
      <code id="code-1741840060141">
       UPDATE mysql.user SET authentication_string='' WHERE User='root'; FLUSH PRIVILEGES;
      </code>
     </li>
     <li>
      <p>
       修改完成后，检查
       <code>
        mysql.user
       </code>
       表确保权限生效：
      </p>
      <pre></pre>
      <p>
       Sql
      </p>
      <p>
       <a id="copy-1741843767115">
       </a>
      </p>
      <code id="code-1741843767115">
       SELECT user, host, authentication_string FROM mysql.user;
      </code>
     </li>
    </ol>
