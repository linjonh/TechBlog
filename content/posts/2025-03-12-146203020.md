     一、DMA与ADC基础介绍
    </h3>
    <h4>
     <a id="1_DMACPU_2">
     </a>
     1. DMA：解放CPU的“数据搬运工”
    </h4>
    <p>
     DMA（Direct Memory Access） 是STM32中用于在外设与内存之间直接传输数据的硬件模块。其核心优势在于无需CPU干预，可显著提升系统效率。
    </p>
    <ul>
     <li>
      <p>
       功能特点：
      </p>
      <ul>
       <li>
        <p>
         支持存储器↔外设、存储器↔存储器的数据传输。
        </p>
       </li>
       <li>
        <p>
         多通道管理，每个通道独立控制一个外设的数据传输。
        </p>
       </li>
       <li>
        <p>
         传输完成后触发中断通知CPU。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       典型应用场景：
      </p>
      <ul>
       <li>
        <p>
         ADC多通道连续采样数据的自动存储。
        </p>
       </li>
       <li>
        <p>
         串口大数据收发、SPI/I2C通信等。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="2_ADC_19">
     </a>
     2. ADC：模拟世界的“数字翻译器”
    </h4>
    <p>
     ADC（Analog-to-Digital Converter） 负责将模拟信号（如电压、温度）转换为数字信号。STM32的ADC模块支持多通道、高精度采样。
    </p>
    <ul>
     <li>
      <p>
       关键参数：
      </p>
      <ul>
       <li>
        <p>
         分辨率：12位（0~4095）。
        </p>
       </li>
       <li>
        <p>
         采样速率：最高1MHz（STM32F1系列）。
        </p>
       </li>
       <li>
        <p>
         输入通道：16个外部通道 + 2个内部通道（温度传感器、V
         <sub>
          REF
         </sub>
         ）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       工作模式：
      </p>
      <ul>
       <li>
        单次转换、连续转换、扫描模式（多通道轮询）。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="3__134">
     </a>
     3. 优化技巧与常见问题
    </h4>
    <h5>
     <a id="ADC_135">
     </a>
     提升ADC精度
    </h5>
    <ul>
     <li>
      <p>
       硬件优化：
      </p>
      <ul>
       <li>
        <p>
         添加0.1μF滤波电容到ADC输入引脚。
        </p>
       </li>
       <li>
        <p>
         单独为VDDA和VSSA供电。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       软件优化：
      </p>
      <ul>
       <li>
        丢弃前几次采样值（避免电源不稳定）。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_146">
     </a>
     常见问题排查
    </h5>
    <ul>
     <li>
      <p>
       数据错位：检查DMA数据宽度（需与ADC对齐方式一致）。
      </p>
     </li>
     <li>
      <p>
       采样值跳动：增加采样时间或添加软件滤波（如滑动平均）。
      </p>
     </li>
     <li>
      <p>
       DMA不触发：确认DMA通道与ADC的映射关系（参考数据手册）。
      </p>
     </li>
    </ul>
