    <h2>
     MySQL体系结构
    </h2>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/0289e95ee92944f08815fff8eed602d3.png"/>
    </p>
    <p>
     MySQL的系统体系结构一般来说分为一下几层：
    </p>
    <ul>
     <li>
      连接层
     </li>
     <li>
      服务层
     </li>
     <li>
      引擎层
     </li>
     <li>
      储存层
     </li>
    </ul>
    <h2 id="%E5%AD%98%E5%82%A8%E5%BC%95%E6%93%8E%E7%AE%80%E4%BB%8B" name="%E5%AD%98%E5%82%A8%E5%BC%95%E6%93%8E%E7%AE%80%E4%BB%8B">
     存储引擎简介
    </h2>
    <p>
     存储引擎是储存数据，建立索引，更新/查询数据的实现方式。引擎没有好坏之分，不同的场景应该选择合适的引擎。
     <strong>
      <span style="color:#fe2c24">
       引擎是基于表的，而不是基于库的，所以也被称为表类型
      </span>
     </strong>
     。
    </p>
    <p>
     在创建表时，指定储存引擎类型：
    </p>
    <blockquote>
     <p>
      <span style="color:null">
       <strong>
        create  table  表名  （
       </strong>
      </span>
     </p>
     <p>
      <span style="color:null">
       <strong>
        字段1，字段类型  [ 注释 ]  ,
       </strong>
      </span>
     </p>
     <p>
      <span style="color:null">
       <strong>
        字段2，字段类型  [ 注释 ]
       </strong>
      </span>
     </p>
     <p>
      <span style="color:null">
       <strong>
        ）
       </strong>
      </span>
      <span style="color:#fe2c24">
       <strong>
        Engine = 储存类型
       </strong>
      </span>
      <span style="color:null">
       <strong>
        [ 注释 ] ;
       </strong>
      </span>
     </p>
    </blockquote>
    <p>
     查看当前数据库支持的储存引擎：
    </p>
    <pre><code>show engines;</code></pre>
    <ul>
     <li>
      <h3 id="InnoDB" name="InnoDB">
       InnoDB
      </h3>
     </li>
    </ul>
    <h4 id="%E4%BB%8B%E7%BB%8D" name="%E4%BB%8B%E7%BB%8D">
     介绍
    </h4>
    <p>
     InnoDB是一种高可靠性和高性能性的储存引擎，自从MySQL 5.5之后，InnoDB就是MySQL的默认储存引擎。
    </p>
    <h4 id="%E7%89%B9%E7%82%B9" name="%E7%89%B9%E7%82%B9">
     特点
    </h4>
    <ul>
     <li>
      DML操作遵循ACID原则
     </li>
     <li>
      行级锁，提高并发性能
     </li>
     <li>
      支持外键约束，提高数据的一致性和完整性
     </li>
    </ul>
    <h4 id="%E6%96%87%E4%BB%B6" name="%E6%96%87%E4%BB%B6">
     文件
    </h4>
    <p>
     xxx.ibd，innnoDB的
     <strong>
      <span style="background-color:#ff9900">
       每张表都有一个对应的表文件，储存表结构，数据和索引
      </span>
     </strong>
     。
    </p>
    <p>
     逻辑储存结构
    </p>
    <p>
     MyISAM和Memory
    </p>
    <ul>
     <li>
      <h3 id="MyISAM" name="MyISAM">
       MyISAM
      </h3>
     </li>
    </ul>
    <h4 id="%E4%BB%8B%E7%BB%8D" name="%E4%BB%8B%E7%BB%8D">
     介绍
    </h4>
    <p>
     是早期MySQL的默认储存引擎
    </p>
    <h4 id="%E7%89%B9%E7%82%B9" name="%E7%89%B9%E7%82%B9">
     特点
    </h4>
    <p>
     <strong>
      <span style="background-color:#ff9900">
       支持表锁，不支持行锁
      </span>
     </strong>
    </p>
    <p>
     访问速度更快
    </p>
    <h4 id="%E6%96%87%E4%BB%B6" name="%E6%96%87%E4%BB%B6">
     文件
    </h4>
    <p>
     xxx.sdi 储存表结构信息
    </p>
    <p>
     xxx.MYD 储存数据
    </p>
    <p>
     xxx.MYI 储存索引
    </p>
    <ul>
     <li>
      <h3 id="Memory" name="Memory">
       Memory
      </h3>
     </li>
    </ul>
    <h4 id="%E4%BB%8B%E7%BB%8D" name="%E4%BB%8B%E7%BB%8D">
     介绍
    </h4>
    <p>
     Memory时储存在内存中的，由于受到硬件影响或者是断电等问题，所以
     <span style="color:#fe2c24">
      <strong>
       一般储存临时信息、临时表
      </strong>
     </span>
    </p>
    <h4 id="%E7%89%B9%E7%82%B9" name="%E7%89%B9%E7%82%B9">
     特点
    </h4>
    <p>
     内存储存
    </p>
    <p>
     hash索引（默认）
    </p>
    <ul>
     <li>
      也是表锁
     </li>
    </ul>
    <h4 id="%E6%96%87%E4%BB%B6" name="%E6%96%87%E4%BB%B6">
     文件
    </h4>
    <p>
     xxx.sdi  储存表结构信息
    </p>
    <h2 id="%E9%80%89%E6%8B%A9" name="%E9%80%89%E6%8B%A9">
     选择
    </h2>
    <p>
     针对应用系统的选择合适的储存引擎，当然也可以根据实际系统的情况，自由的对储存引擎进行组合
    </p>
    <blockquote>
     <ul>
      <li>
       <strong>
        InnoDB：是MySQL的默认储存引擎，支持事务，外键。
        <span style="color:#fe2c24">
         如果对于事物的完整性，并发事务条件下希望保证数据的一致性
        </span>
        ，除了查询插入操作还有大量删除更新操作，那么InnoDB是最好的选择。
       </strong>
      </li>
      <li>
      </li>
      <li>
       <strong>
        MyISAM：一般是大量查询插入操作，少量删除更新操作，并且对于事务的完整性、并发性并没有许多要求，那么这个将会是最好的选择。
       </strong>
      </li>
      <li>
      </li>
      <li>
       <strong>
        Memor：储存在内存中，
        <span style="color:#fe2c24">
         访问速度快，通常用于临时表及缓存
        </span>
        。缺点是有表的大小限制，不能存储大量的数据，无法保证数据的安全性。
       </strong>
      </li>
     </ul>
    </blockquote>
