    <h3>
     <a id="_4">
     </a>
     下载工具的安装
    </h3>
    <p>
     先来补充一个小知识：Ubuntu系统和CentOS系统的
     <strong>
      包管理机制不同，用来查询软件源的命令也不同：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       Ubuntu系统使用的是apt包管理系统
      </strong>
      ：
      <code>
       rpm
      </code>
      命令主要用于基于RPM包管理的系统，比如CentOS、Red Hat Enterprise Linux等。而Ubuntu 20.04默认使用的是
      <code>
       apt
      </code>
      （Advanced Package Tool）包管理工具，它的数据库结构和查询命令与
      <code>
       rpm
      </code>
      是完全不同的体系。所以使用
      <code>
       rpm -qa | grep wget
      </code>
      在Ubuntu系统中查询不到结果，因为Ubuntu并不依赖
      <code>
       rpm
      </code>
      来管理已安装的软件包信息。
     </li>
     <li>
      <strong>
       正确的查询方式
      </strong>
      ：在Ubuntu 20.04中，要查看是否安装了某个命令，可以使用
      <code>
       dpkg -l | grep 指令
      </code>
      命令。
      <code>
       dpkg
      </code>
      是Debian系操作系统（Ubuntu基于Debian）中用于管理软件包的基础工具，
      <code>
       dpkg -l
      </code>
      会列出系统中已安装软件包的相关信息，再通过管道符
      <code>
       |
      </code>
      配合
      <code>
       grep
      </code>
      命令筛选出包含对应命令的相关内容，就能确认是否安装了该命令及其版本等情况。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_9">
     </a>
     更换软件源
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        需要更换软件源的情况
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         网络访问限制
        </strong>
        ：默认的Ubuntu官方软件源服务器可能位于国外，国内用户访问时可能由于网络环境等因素（比如网络带宽限制、连接不稳定等）导致下载软件包速度很慢，甚至出现下载失败的情况。此时更换为国内的镜像源，像阿里云、清华源等，能显著提升下载速度，使软件安装、更新等操作更顺畅。
       </li>
       <li>
        <strong>
         特定软件版本需求
        </strong>
        ：有时候官方源里提供的软件版本未必是你期望的，比如某些软件在官方源中版本较旧，而一些国内的镜像源可能会对部分常用软件进行更及时的版本同步，更换软件源后就有机会获取到更新的版本。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        可以不更换的情况
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         网络环境良好
        </strong>
        ：如果所在网络环境对访问Ubuntu官方软件源没有限制，且下载速度能满足日常使用需求，例如在一些网络配置较高且国际网络访问顺畅的办公或科研环境中，那么不更换软件源也是可以正常进行软件的安装、更新等操作的。
       </li>
       <li>
        <strong>
         对软件版本无特殊要求
        </strong>
        ：若平时只是使用一些常规的、稳定版本的软件，且对获取最新版本没有迫切需求，依靠官方源提供的软件版本就能满足日常使用，也可不更换软件源。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_19">
     </a>
     更换软件源的步骤
    </h4>
    <p>
     以下是在Ubuntu 20.04系统下更换软件源的常见操作步骤：
    </p>
    <ol>
     <li>
      <strong>
       备份原有的软件源列表文件
      </strong>
      ：
      <br/>
      打开终端，输入以下命令备份默认的软件源配置文件
      <code>
       sources.list
      </code>
      ：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">cp</span> /etc/apt/sources.list /etc/apt/sources.list.bak
</code></pre>
    <p>
     这样做是为了在后续若因更换软件源出现问题时，可以方便地恢复到原来的配置。
    </p>
    <ol start="2">
     <li>
      <strong>
       选择合适的镜像源并编辑
       <code>
        sources.list
       </code>
       文件
      </strong>
      ：
      <br/>
      常见的国内镜像源有阿里云源、清华源、中科大源等，以阿里云源为例来介绍编辑过程。
     </li>
    </ol>
    <ul>
     <li>
      首先打开
      <code>
       sources.list
      </code>
      文件进行编辑，可以使用文本编辑器（如
      <code>
       nano
      </code>
      或
      <code>
       vim
      </code>
      ），这里以
      <code>
       nano
      </code>
      为例：
     </li>
    </ul>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">nano</span> /etc/apt/sources.list
</code></pre>
    <ul>
     <li>
      然后将原文件中的内容全部删除（可以使用快捷键，如在
      <code>
       nano
      </code>
      中按
      <code>
       Ctrl + K
      </code>
      删除整行内容，多次操作来清除全部内容），接着将以下适合Ubuntu 20.04的清华源内容复制粘贴进去（不同Ubuntu版本对应的源内容有差别，需注意选择正确的版本）：
     </li>
    </ul>
    <pre><code class="prism language-bash">deb http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse
</code></pre>
    <ol start="3">
     <li>
      <strong>
       更新软件源列表
      </strong>
      ：
      <br/>
      在终端中输入以下命令来更新软件源列表，让系统获取新的软件包信息：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
</code></pre>
    <p>
     至此，软件源就更换完成了，后续可以使用
     <code>
      apt
     </code>
     相关命令进行软件的安装、更新等操作，并且下载速度等情况通常会根据所选用的镜像源情况有所改善。
    </p>
    <p>
     当然，你也可以选择其他镜像源按照类似的步骤进行更换，只是在编辑
     <code>
      sources.list
     </code>
     文件时将对应镜像源的正确内容填写进去即可。
    </p>

    <h2>
     <a id="WebSocketpp_97">
     </a>
     WebSocketpp
    </h2>
    <h3>
     <a id="WebSocket_98">
     </a>
     WebSocket介绍
    </h3>
    <p>
     在传统的 web 程序架构中，其交互模式呈现出典型的“
     <strong>
      一问一答
     </strong>
     ”形态。具体而言，客户端会向服务器发送一个 HTTP 请求，随后服务器针对该请求向客户端返回一个 HTTP 响应。在这样的模式下，服务器往往处于相对被动的地位，因为倘若客户端不主动发起请求，服务器便无法自主地向客户端推送响应消息。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d93c1b1f2d4743d9bed3275f03d7629a.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3d980a991ea54f6d9b3a74843ae6c3c0.png">
       <br/>
       当我们着手开发诸如网页实时聊天或者像本项目所涉及的五子棋游戏这类对实时性要求颇高的程序时，“
       <strong>
        消息推送
       </strong>
       ”功能就显得尤为关键，即
       <strong>
        服务器需要能够主动地将消息推送到客户端
       </strong>
       。然而，若依旧沿用传统的 C/S 通信方案，也就是基于原生的 HTTP 协议来实现这一需求的话，那就只能借助“
       <strong>
        轮询
       </strong>
       ”的方式了。
      </img>
     </img>
    </p>
    <p>
     所谓轮询，就是客户端要不停地向服务器发送请求，只有这样，当服务器接收到发送方传来的消息后，才能够将新消息推送给接收方。不过，这种轮询的方式存在着明显的弊端：
    </p>
    <ul>
     <li>
      <strong>
       轮询的成本颇高
      </strong>
      。对于
      <strong>
       客户端
      </strong>
      而言，为了保障消息能够及时被获取，需要每隔较短的时间就向服务器发送一个请求，而且这个时间间隔自然是越短越好，如此一来，便会给客户端带来较大的资源消耗和性能压力，成本也就随之升高了。而从
      <strong>
       服务器端
      </strong>
      来看，它需要应对众多客户端发起的大量请求，这无疑会给服务器造成极大的负担，使其面临严峻的性能考验。
     </li>
     <li>
      采用轮询方式会导致通信双方
      <strong>
       无法及时获取到消息的响应
      </strong>
      。毕竟是依靠客户端不断发起请求来询问服务器是否有新消息，在两次请求的间隔期间，即便服务器已经有了新消息，客户端也无法第一时间知晓，存在消息延迟的问题。
     </li>
    </ul>
    <p>
     那么，为什么
     <strong>
      HTTP 不支持服务器主动向客户端进行消息推送
     </strong>
     呢？这是由 HTTP 协议本身的设计特点所决定的。HTTP 协议构建在
     <strong>
      请求 - 响应
     </strong>
     的模型之上，其设计初衷就是客户端发起请求，服务器针对请求进行响应，整个流程是单向触发的，并没有为服务器主动向客户端推送消息预留相应的机制和通道，所以在原生的 HTTP 协议下，服务器很难做到主动向客户端发送消息，除非借助像轮询这样额外的变通手段，但轮询又存在诸多弊端，这也正是后续 WebSocket 应运而生的重要原因所在。
    </p>
    <ul>
     <li>
      WebSocket是从 HTML5 开始支持的⼀种网页端和服务端保持长连接的 消息推送机制。
     </li>
     <li>
      WebSocket 更接近于TCP这种级别的通信方式，一旦连接建立完成客户端或者服务器都可以主动地向对方发送数据。
     </li>
    </ul>
    <h3>
     <a id="_116">
     </a>
     原理解析
    </h3>
    <p>
     WebSocket 协议是用来解决HTTP不支持消息推送问题的，通常使用WebSocket协议是从HTTP协议转换的（WebSocket也能单独使用）。
    </p>
    <p>
     所以，为了建立一个 WebSocket 连接，客户端浏览器首先要向服务器发起一个 HTTP 请求，这个请求和通常的 HTTP 请求不同，该请求包含了一些附加头信息，通过这个附加头信息完成握手过程并进行升级协议的过程。具体过程如下：
    </p>
    <ul>
     <li>
      首先，进行 TCP 的三次握手建立连接。
     </li>
     <li>
      然后使用 HTTP 协议进行通信，如完成用户登录、用户注册等工作。
     </li>
     <li>
      在 HTTP 协议中，如果想使用长连接、消息推送的功能，就需要将当前 HTTP 协议的通信连接切换成WebSocket 协议的通信连接。
     </li>
     <li>
      切换协议的过程为：
      <ol>
       <li>
        客户端向服务器端发送一个 HTTP 请求，该请求是 WebSocket 协议的切换请求；
       </li>
       <li>
        服务器端收到请求后，服务器端就会向客户端发送一个状态码为 101 的响应；
       </li>
       <li>
        客户端收到响应后，协议切换过程就结束了，之后就可以使用 WebSocket 协议进行通信了。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/75d6647b96bc4121a02764f284bd0691.png"/>
       </li>
      </ol>
     </li>
    </ul>
    <p>
     在协议切换时，
     <strong>
      客户端发起的请求
     </strong>
     中的字段有：
    </p>
    <ul>
     <li>
      <strong>
       请求行：
      </strong>
      <code>
       GET /ws HTTP/1.1
      </code>
      ，请求方法为 GET、URI通常为 /ws、HTTP版本为 HTTP/1.1。
     </li>
     <li>
      <strong>
       请求报头：
      </strong>
      <ul>
       <li>
        <strong>
         Connection：
        </strong>
        <code>
         Upgrade
        </code>
        ，表示当前连接需要进行升级/切换。
       </li>
       <li>
        <strong>
         Upgrade：
        </strong>
        <code>
         WebSocket
        </code>
        ，表示当前连接需要升级/切换到 WebSocket 协议。
       </li>
       <li>
        <strong>
         Sec-WebSocket-Version：
        </strong>
        <code>
         xxx
        </code>
        ，表示 WebSocket 协议的版本（用来让服务器检查当前 WebSocket 协议版本是否支持）。
       </li>
       <li>
        <strong>
         Sec-WebSocket-key：
        </strong>
        <code>
         xxx

    <h3>
     <a id="WebSocket_146">
     </a>
     WebSocket报文格式
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f0d1f1b58b44493cbd59219b1421f68c.png">
      <br/>
      报文字段比较多，我们重点关注这几个字段：
     </img>
    </p>
    <ul>
     <li>
      <strong>
       FIN：
      </strong>
      WebSocket 传输数据以消息为概念单位，一个消息有可能由一个或多个帧组成，FIN字段为1，表示末尾帧，为0表示中间帧。
     </li>
     <li>
      <strong>
       RSV1~3：
      </strong>
      保留字段，只在扩展时使用，若未启用扩展则应置1，若收到不全为0的数据帧，且未协商扩展则立即终止连接。
     </li>
     <li>
      <strong>
       opcode：
      </strong>
      标志当前数据帧的类型
      <ul>
       <li>
        0x0: 表示这是个延续帧，当 opcode 为 0 表示本次数据传输采用了数据分片，当前收到的帧为，其中⼀个分片。
       </li>
       <li>
        0x1: 表示这是一个文本帧。
       </li>
       <li>
        0x2: 表示这是一个二进制帧。
       </li>
       <li>
        0x3-0x7: 保留，暂未使用。
       </li>
       <li>
        0x8: 表示断开连接。
       </li>
       <li>
        0x9: 表示 ping 帧。
       </li>
       <li>
        0xa: 表示 pong 帧。
       </li>
       <li>
        0xb-0xf: 保留，暂未使用。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       mask：
      </strong>
      表示 Payload 数据是否被编码，若为1则必有Mask-Key，用于解码 Payload 数据。仅客户端发送给服务端的消息时需要设置。
     </li>
     <li>
      <strong>
       Payload length：
      </strong>
      数据载荷的长度，单位是字节， 有可能为7位、7+16位、7+64位。假设Payload length = x，则有以下几种情况：
      <ul>
       <li>
        x为0~126：数据的长度为x字节。
       </li>
       <li>
        x为126：后续2个字节代表⼀个16位的无符号整数，该无符号整数的值为数据的长度。
       </li>
       <li>
        x为127：后续8个字节代表⼀个64位的⽆符号整数（最高位为0），该无符号整数的值为数据的长度。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Mask-Key：
      </strong>
      当mask为1时存在，长度为4字节，解码规则：
      <code>
       DECODED[i] = ENCODED[i] ^ MASK[i% 4]
      </code>
      。
     </li>
     <li>
      <strong>
       Payload data:
      </strong>
      报文携带的载荷数据。
     </li>
    </ul>

      <strong>

      </strong>

        </code>

        </strong>

        </code>

     Json介绍
    </h3>
    <p>
     <code>
      Json
     </code>
     是一种数据交换格式，它采用完全独立于编程语言的文本格式来存储和表示数据。
    </p>
    <p>
     当有多个数据对象在进行传输或者进行持久化存储时，可以采用
     <code>
      Json
     </code>
     数据交换格式进行序列化，转换为一个整体的二进制数据串进行传输。
    </p>
    <p>
     <code>
      Json
     </code>
     的数据类型包括对象，数组，字符串，数字等：
    </p>
    <ul>
     <li>
      对象：使用花括号
      <code>
       { }
      </code>
      括起来的表示一个对象。
     </li>
     <li>
      数组：使用中括号
      <code>
       [ ]
      </code>
      括起来的表示一个数组。
     </li>
     <li>
      字符串：使用常规双引号
      <code>
       " "
      </code>
      括起来的的表示一个字符串。
     </li>
     <li>
      数字：包括整形和浮点数，可以直接使用。
     </li>
    </ul>

       Json::Value

    <h2>
     <a id="MYSQL_API_775">
     </a>
     MYSQL API
    </h2>
    <p>
     MySQL 是基于 C/S（客户端/服务器）模型实现的，在该架构下，MySQL 数据库的管理由本机的 MySQL 服务器负责。若要对数据库中的数据进行操作，就需要创建一个客户端，客户端通过向服务器发起请求（以 SQL 语句的形式），服务器在接收到请求并执行相应的 SQL 语句后，会将对应的结果返回给客户端。
    </p>
    <p>
     MySQL API 接口则是帮助开发者实现 MySQL 客户端的关键工具。
    </p>
    <p>
     下面详细介绍 MySQL 数据库操作流程：
    </p>

    </h3>
    <ol>
     <li>

       初始化 MYSQL 操作句柄
      </strong>
      ：首先要初始化一个 MYSQL 操作句柄，这个句柄包含了众多重要字段，例如其中的 socket 字段用于网络通信，它是客户端与服务器进行数据传输的关键通道。通过初始化句柄，为后续的数据库操作奠定基础。
     </li>
     <li>
      <strong>
       连接 MYSQL 服务器
      </strong>
      ：利用初始化好的句柄，建立与 MySQL 服务器的连接。这一步需要提供正确的服务器地址、端口号、用户名以及密码等信息，确保能够成功连接到目标服务器，从而开启与数据库的交互通道。
     </li>
     <li>
      <strong>
       设置客户端的字符集
      </strong>
      ：为了防止在数据传输和处理过程中出现乱码问题，需要设置客户端的字符集。通常使用统一的字符集，如 UTF - 8，以确保客户端与服务器之间能够正确地识别和处理各种字符数据。
     </li>
     <li>
      <strong>
       选择要操作的数据库
      </strong>
      ：在成功连接到服务器后，需要明确指定要操作的数据库。每个 MySQL 服务器可能管理着多个数据库，通过选择特定的数据库，使得后续的操作都在该指定的数据库环境中进行。
     </li>
    </ol>
    <h3>
     <a id="_788">
     </a>
     客户端对数据库中数据的操作
    </h3>
    <p>
     客户端对数据库数据的操作主要分为两大类：数据变更操作（新增、修改、删除数据）和数据查询操作。
    </p>
    <p>
     对于数据变更操作（新增、修改、删除数据），只要对应的 SQL 语句在服务器端成功执行，数据库中的数据就会发生相应的变化，这类操作通常不需要返回特定的结果集，因为其目的在于改变数据库的状态。
    </p>
    <p>
     而对于数据查询操作，除了要确保查询 SQL 语句执行成功外，还需要将查询结果集保存到客户端本地，以便进一步处理和展示。具体步骤如下：
    </p>
    <ol start="5">
     <li>
      <strong>
       执行 SQL 语句
      </strong>
      ：无论是数据变更操作还是查询操作，都需要通过 MySQL API 接口向服务器发送 SQL 语句，服务器接收到语句后进行解析和执行。
     </li>
     <li>
      <strong>
       将查询结果集保存到本地
      </strong>
      ：若执行的是查询操作，在服务器成功执行 SQL 语句并生成结果集后，客户端需要将该结果集保存到本地内存或其他存储介质中，以便后续操作。
     </li>
     <li>
      <strong>
       获取结果集中的结果条数
      </strong>
      ：保存好结果集后，可以通过相应的函数或方法获取结果集中的行数和列数信息。这对于了解查询结果的规模和结构非常重要，有助于后续的遍历和处理。
     </li>
     <li>
      <strong>
       遍历访问每一条结果
      </strong>
      ：根据获取到的结果集行数和列数，使用循环结构遍历结果集，逐行访问每一条数据记录。在遍历过程中，可以根据业务需求对每条数据进行进一步的处理，如提取特定字段的值、进行数据计算等。
     </li>
     <li>
      <strong>
       释放结果集
      </strong>
      ：当对查询结果集的处理完成后，为了避免内存泄漏和资源浪费，需要及时释放结果集所占用的内存空间。通过调用特定的释放函数，将结果集占用的内存归还给系统。
     </li>
     <li>
      <strong>
       释放 MYSQL 句柄
      </strong>
      ：在完成所有数据库操作后，最后要释放初始化的 MYSQL 操作句柄。这一步同样是为了释放资源，关闭与服务器的连接通道，确保程序在结束数据库操作后能够正确地清理所占用的系统资源。
     </li>
    </ol>

       定义
      </strong>
      ：CSS 是一种用来为 HTML 文档添加样式的语言，它可以控制网页元素的外观，如颜色、字体、大小、位置、边框、背景等。
     </li>
     <li>
      <strong>
       作用
      </strong>
      ：使网页更加美观和易于阅读，实现网页的布局和排版，将网页的内容与表现形式分离，方便维护和修改网页的样式。例如，可以通过 CSS 设置一个段落的字体颜色为红色，字体大小为 16 像素，段落的外边距为 10 像素等。

    </ul>

    <h3>
     <a id="WebSocket_1412">
     </a>
     WebSocket
    </h3>
    <p>
     创建一个 websocket 请求，请求服务器创建一个 websocket 长连接，进行持久通信。
     <strong>
      主要用于聊天输入框内容的获取。
     </strong>
    </p>
    <p>
     以下是使用 WebSocket 对象发送请求的过程：
    </p>
    <p>
     <strong>
      一、创建 WebSocket 连接
     </strong>
     ：
    </p>
    <p>
     首先，在 JavaScript 中创建一个 WebSocket 对象。需要提供一个 WebSocket 服务器的 URL，该 URL 以
     <code>
      ws://
     </code>
     或
     <code>
      wss://
     </code>
     开头（
     <code>
      wss://
     </code>
     用于加密的 WebSocket 连接）。例如：
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">let</span> socket <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">WebSocket</span><span class="token punctuation">(</span><span class="token string">'ws://example.com/socket'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这行代码创建了一个新的 WebSocket 对象，它将尝试连接到
     <code>
      ws://example.com/socket
     </code>
     的 WebSocket 服务器。
    </p>
    <p>
     <strong>
      二、监听事件
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       onopen 事件
      </strong>
      ：
     </li>
    </ul>
    <p>
     当 WebSocket 连接成功建立时，会触发
     <code>
      onopen
     </code>
     事件。可以在此事件处理函数中执行一些操作，例如发送初始消息或设置标志位表示连接已打开。例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function-variable function">onopen</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token parameter">event</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token string">'WebSocket 连接已打开'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token comment">// 可以在此发送初始消息</span>
    socket<span class="token punctuation">.</span><span class="token function">send</span><span class="token punctuation">(</span><span class="token string">'Hello, Server!'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     当连接成功打开时，将在控制台打印
     <code>
      WebSocket 连接已打开
     </code>
     ，并向服务器发送消息
     <code>
      Hello, Server!
     </code>
     。
    </p>
    <ul>
     <li>
      <strong>
       onmessage 事件
      </strong>
      ：
     </li>
    </ul>
    <p>
     当从服务器接收到消息时，会触发
     <code>
      onmessage
     </code>
     事件。该事件处理函数的参数包含了从服务器接收到的数据，可以是文本或二进制数据。例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function-variable function">onmessage</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token parameter">event</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token string">'收到服务器消息: '</span> <span class="token operator">+</span> event<span class="token punctuation">.</span>data<span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token comment">// 处理接收到的消息</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这里将接收到的消息打印到控制台，并可以对消息进行进一步处理，比如更新网页的内容。
    </p>
    <ul>
     <li>
      <strong>
       onerror 事件
      </strong>
      ：
     </li>
    </ul>
    <p>
     如果在连接过程中出现错误，会触发
     <code>
      onerror
     </code>
     事件。可以在此处理错误情况，例如显示错误信息。例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function-variable function">onerror</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token parameter">event</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">error</span><span class="token punctuation">(</span><span class="token string">'WebSocket 错误: '</span> <span class="token operator">+</span> event<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     此代码将错误信息输出到控制台，以便调试和处理错误。
    </p>
    <ul>
     <li>
      <strong>
       onclose 事件
      </strong>
      ：
     </li>
    </ul>
    <p>
     当 WebSocket 连接关闭时，会触发
     <code>
      onclose
     </code>
     事件。可以在此进行一些清理工作或通知用户连接已关闭。例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function-variable function">onclose</span> <span class="token operator">=</span> <span class="token keyword">function</span><span class="token punctuation">(</span><span class="token parameter">event</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    console<span class="token punctuation">.</span><span class="token function">log</span><span class="token punctuation">(</span><span class="token string">'WebSocket 连接已关闭'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <strong>
      三、发送消息
     </strong>
     ：
    </p>
    <p>
     使用
     <code>
      send()
     </code>
     方法发送消息。可以发送文本消息或二进制数据，例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function">send</span><span class="token punctuation">(</span><span class="token string">'This is a message from client'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这将发送文本消息
     <code>
      This is a message from client
     </code>
     到服务器。
    </p>
    <p>
     <strong>
      四、关闭连接
     </strong>
     ：
    </p>
    <p>
     可以使用
     <code>
      close()
     </code>
     方法关闭 WebSocket 连接，例如：
    </p>
    <pre><code class="prism language-javascript">socket<span class="token punctuation">.</span><span class="token function">close</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这将关闭当前的 WebSocket 连接。
    </p>

     <strong>
      五、示例代码
     </strong>
     ：
    </p>
    <pre><code class="prism language-javascript"><span class="token keyword">let</span> socket <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">WebSocket</span><span class="token punctuation">(</span><span class="token string">'ws://example.com/socket'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

    <p>
     <strong>
      六、WebSocket 优点和应用场景
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       优点
      </strong>
      ：
      <ul>
       <li>
        全双工通信：允许客户端和服务器之间同时进行双向通信，比 AJAX 的请求/响应模式更加灵活。
       </li>
       <li>
        实时性高：适用于实时通信场景，如聊天应用、实时数据更新（如股票价格、实时监控数据）。
       </li>
       <li>
        低延迟：避免了 HTTP 的开销，提高了通信效率。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       应用场景
      </strong>
      ：
      <ul>
       <li>
        在线聊天应用：用户之间可以实时发送和接收消息。
       </li>
       <li>
        在线游戏：玩家之间的实时操作和数据交互。
       </li>
       <li>
        实时数据更新：如金融数据、传感器数据的实时更新等。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     WebSocket 提供了一种强大的客户端-服务器通信机制，尤其适用于需要实时双向通信的场景，通过创建 WebSocket 对象，监听相关事件，并使用
     <code>
      send()
     </code>
     方法发送消息，可以实现高效、实时的网络通信。
    </p>
