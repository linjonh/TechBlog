     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b78891378f6346fa974442dae79fa307.png#pic_center">
      <br/>
      主要掌握一主多从模式，每个从机设定一个唯一的设备地址，主机会叫从机的名字即发出从机的地址，从机会比较此地址是否跟自己的一样，如果一样就相应。一般地址的最后几位可以改变，可变的地址位对应相应的引脚。下为硬件电路
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/68712b041f164a3483d834a7791a0fd3.png#pic_center"/>
     </img>
    </p>
    <p>
     下图这种情况就是短路，需要避免，所以需要禁止所有设备输出强上拉的高电平，采用外置弱上拉电阻加开漏输出的结构。上图右边是具体结构。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b8a52390037a4a998a002701d964431d.png#pic_center">
      <br/>
      好处：
      <br/>
      第一，完全杜绝了电源短路现象
      <br/>
      第二，避免了引脚模式的频繁切换
      <br/>
      第三，线与现象，只有都输出高电平才输出高电平
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/63796fecefe64478a2c86d224c776b51.png#pic_center">
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9d890ceb68ae4651b0623b72120312f9.png#pic_center">
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7fa4a09a3a2c45cfb015ef68448c287a.png#pic_center">
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/67156fd5d5c449d3bb5e31e817b91e14.png#pic_center">
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1dd990f880da41d3bd9432dbb27adcd6.png#pic_center"/>
          <br/>
          指定地址写，指定的从机指定的寄存器地址下写入。首先拉低SDA产生起始条件，然后发送一个字节，即从机地址7位加读写位1位，读写位置1或者置0控制读写。主机接受从机产生应答，再发送的字节送到了指定设备的内部就是寄存器的地址，接着就是应答，表示收到了，然后就是发送的数据，再产生接收应答，然后是终止条件。
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bc83dd96996a4877a172de5d3e76a6d2.png#pic_center"/>
          <br/>
          当前地址读不用指定当前地址。直接发送数据。
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ad340e2e9130458ea19e6d75cc8cd5d3.png#pic_center"/>
          <br/>
          指定地址写+当前地址读=指定地址读，即再指定地址写的时候加上当前地址读。指定地址写和指定地址读用的比较多。
