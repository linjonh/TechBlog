    <h2>
     <a id="Linux__1">
     </a>
     Linux 环境变量快速上手
    </h2>
    <h3>
     <a id="1__3">
     </a>
     1. 什么是环境变量
    </h3>
    <ul>
     <li>
      环境变量（Environment Variables）是操作系统中用于存储配置信息的一种机制，可以在运行时被进程读取和使用。
     </li>
     <li>
      常见环境变量示例：
      <ul>
       <li>
        <code>
         PATH
        </code>
        : 存放可执行文件搜索路径。
       </li>
       <li>
        <code>
         HOME
        </code>
        : 当前用户的家目录路径。
       </li>
       <li>
        <code>
         USER
        </code>
        、
        <code>
         LANG
        </code>
        、
        <code>
         SHELL
        </code>
        等。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2__12">
     </a>
     2. 查看环境变量
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        查看所有环境变量
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token function">printenv</span>
</code></pre>
      <p>
       或者
      </p>
      <pre><code class="prism language-bash"><span class="token function">env</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        查看单个环境变量
       </strong>
      </p>
      <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token environment constant">$PATH</span>
<span class="token builtin class-name">echo</span> <span class="token environment constant">$HOME</span>
</code></pre>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="3__31">
     </a>
     3. 设置环境变量
    </h3>
    <h4>
     <a id="31__33">
     </a>
     3.1 临时设置（仅在当前终端会话中有效）
    </h4>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">VAR_NAME</span><span class="token operator">=</span><span class="token string">"value"</span>
</code></pre>
    <ul>
     <li>
      <strong>
       示例
      </strong>
      ：
      <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token string">"<span class="token environment constant">$HOME</span>/.local/bin:<span class="token environment constant">$PATH</span>"</span>
</code></pre>
     </li>
     <li>
      <strong>
       解释
      </strong>
      ：
      <ul>
       <li>
        <strong>
         作用
        </strong>
        ：将用户家目录下的
        <code>
         .local/bin
        </code>
        添加到
        <code>
         PATH
        </code>
        环境变量中。
       </li>
       <li>
        <strong>
         路径说明
        </strong>
        ：
        <ul>
         <li>
          <code>
           $HOME/.local/bin
          </code>
          ：用户私有的二进制文件目录，常用于存放用户安装的工具或脚本（如通过
          <code>
           pip install --user
          </code>
          安装的 Python 包）。
         </li>
         <li>
          <code>
           $PATH
          </code>
          ：系统当前的
          <code>
           PATH
          </code>
          变量，包含默认的可执行文件搜索路径。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         语法解析
        </strong>
        ：
        <ul>
         <li>
          <code>
           export
          </code>
          ：设置环境变量并使其在当前 Shell 会话及其子进程中生效。
         </li>
         <li>
          <code>
           "$HOME/.local/bin:$PATH"
          </code>
          ：将
          <code>
           $HOME/.local/bin
          </code>
          添加到
          <code>
           $PATH
          </code>
          最前面，优先查找此目录中的可执行文件。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         使用场景
        </strong>
        ：确保用户安装的工具（如
        <code>
         jupyter-notebook
        </code>
        ）无需完整路径即可运行。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       注意
      </strong>
      ：关闭终端后，此设置会失效。
     </li>
    </ul>
    <h4>
     <a id="32__Shell__52">
     </a>
     3.2 永久设置（在 Shell 配置文件中）
    </h4>
    <p>
     不同 Shell 的配置文件不同，以下以
     <strong>
      Zsh
     </strong>
     为例：
    </p>
    <ol>
     <li>
      编辑
      <code>
       ~/.zshrc
      </code>
      <pre><code class="prism language-bash"><span class="token function">nano</span> ~/.zshrc
</code></pre>
     </li>
     <li>
      在文件末尾添加：
      <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token string">"<span class="token environment constant">$HOME</span>/.local/bin:<span class="token environment constant">$PATH</span>"</span>
</code></pre>
     </li>
     <li>
      保存并退出，然后让配置立即生效：
      <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> ~/.zshrc
</code></pre>
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       效果
      </strong>
      ：每次打开新的 Zsh 终端时，
      <code>
       ~/.zshrc
      </code>
      会自动加载，设置环境变量。
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       Bash 用户
      </strong>
      ：配置文件通常是
      <code>
       ~/.bashrc
      </code>
      或
      <code>
       ~/.bash_profile
      </code>
      ，步骤类似。
     </p>
    </blockquote>
    <hr/>
    <h3>
     <a id="4__73">
     </a>
     4. 常见环境变量说明
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        PATH
       </strong>
      </p>
      <ul>
       <li>
        作用：存放可执行文件的搜索路径。输入命令时，系统按
        <code>
         PATH
        </code>
        中列出的目录顺序查找。
       </li>
       <li>
        示例：添加
        <code>
         ~/.local/bin
        </code>
        后，可直接运行该目录下的脚本或工具。
       </li>
       <li>
        <strong>
         建议
        </strong>
        ：将用户自定义路径放在
        <code>
         $PATH
        </code>
        前面，确保优先调用用户安装的工具。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        HOME
       </strong>
       <br/>
       当前用户的家目录，通常是
       <code>
        /home/用户名
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        SHELL
       </strong>
       <br/>
       当前使用的 Shell 类型，如
       <code>
        /bin/zsh
       </code>
       或
       <code>
        /bin/bash
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        LANG
       </strong>
       <br/>
       语言及字符编码设置，如
       <code>
        en_US.UTF-8
       </code>
       、
       <code>
        zh_CN.UTF-8
       </code>
       。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="5__91">
     </a>
     5. 验证与排查
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        查看是否生效
       </strong>
      </p>
      <ul>
       <li>
        检查变量：
        <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token variable">$VAR_NAME</span>
</code></pre>
       </li>
       <li>
        查找命令路径：
        <pre><code class="prism language-bash"><span class="token function">which</span> <span class="token operator">&lt;</span>command<span class="token operator">&gt;</span>  <span class="token comment"># 如 which jupyter-notebook</span>
</code></pre>
       </li>
       <li>
        或使用更全面的工具：
        <pre><code class="prism language-bash"><span class="token builtin class-name">type</span> <span class="token operator">&lt;</span>command<span class="token operator">&gt;</span>  <span class="token comment"># 显示命令类型和位置</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        多 Shell 环境
       </strong>
      </p>
      <ul>
       <li>
        如果命令找不到，可能是 Shell 未加载配置文件。
       </li>
       <li>
        检查当前 Shell：
        <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token environment constant">$SHELL</span>
</code></pre>
       </li>
       <li>
        切换默认 Shell：
        <pre><code class="prism language-bash">chsh <span class="token parameter variable">-s</span> /bin/zsh  <span class="token comment"># 或 /bin/bash</span>
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        系统级环境变量
       </strong>
      </p>
      <ul>
       <li>
        可编辑
        <code>
         /etc/profile
        </code>
        （所有用户生效）或
        <code>
         /etc/environment
        </code>
        （全局变量），但用户级配置（如
        <code>
         ~/.zshrc
        </code>
        ）更常见。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="6__123">
     </a>
     6. 现代开发环境中的环境变量
    </h3>
    <ul>
     <li>
      <strong>
       虚拟环境和容器
      </strong>
      <ul>
       <li>
        在 Python
        <code>
         venv
        </code>
        、Conda 或 Docker 中，环境变量会根据激活环境自动调整。
       </li>
       <li>
        示例：激活 Conda 环境后，
        <code>
         PATH
        </code>
        自动包含该环境的
        <code>
         bin
        </code>
        目录。
       </li>
       <li>
        <strong>
         注意
        </strong>
        ：运行命令前需确保环境已激活。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="7__132">
     </a>
     7. 小结
    </h3>
    <ol>
     <li>
      <strong>
       核心
      </strong>
      ：区分“临时 vs. 永久”，明确配置文件（
      <code>
       ~/.zshrc
      </code>
      或
      <code>
       ~/.bashrc
      </code>
      ）。
     </li>
     <li>
      <strong>
       永久设置
      </strong>
      ：
      <ul>
       <li>
        在配置文件中添加
        <code>
         export
        </code>
        命令，使用
        <code>
         source
        </code>
        或重启终端生效。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       排查工具
      </strong>
      ：
      <ul>
       <li>
        <code>
         which
        </code>
        、
        <code>
         type
        </code>
        和
        <code>
         echo $PATH
        </code>
        可快速定位问题。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       保持简洁
      </strong>
      ：
      <ul>
       <li>
        避免重复添加路径，确保配置文件清晰。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <blockquote>
     <p>
      <strong>
       温馨提示
      </strong>
      ：
      <br/>
      在虚拟环境、Conda 或 Docker 等多环境中，注意初始化脚本和激活流程。基本原理不变：通过环境变量（如
      <code>
       PATH
      </code>
      ）让系统找到可执行文件。
      <br/>
      遇到“找不到命令”时，检查
      <code>
       PATH
      </code>
      和 Shell 配置，问题多半迎刃而解！
     </p>
    </blockquote>
    <p>
     祝学习愉快，掌握环境变量后，Linux 操作将更加得心应手！
    </p>
