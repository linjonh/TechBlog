    <h2>
     一，磁盘设备
    </h2>
    <p>
     <img alt="" height="1039" src="https://i-blog.csdnimg.cn/direct/928889c487984ad79e89558810b22941.png" width="1912"/>

    <h3>
     (一).
     <strong>
      硬盘（HDD）存储区域构成
     </strong>
    </h3>
    <p>
     硬盘的存储区域主要由
     <strong>
      扇区
     </strong>
     、
     <strong>
      簇
     </strong>
     、
     <strong>
      分区
     </strong>
     和
     <strong>
      文件系统
     </strong>
     等组成。
    </p>
    <h4>
     1.
     <strong>
      扇区（Sector）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：扇区是硬盘存储的最小单位，通常一个扇区的大小为
       <strong>
        512字节
       </strong>
       或
       <strong>
        4KB
       </strong>
       。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：硬盘通过扇区将数据存储在盘片上。每个扇区都包含一定量的数据，操作系统和硬盘控制器会通过扇区来读取和写入数据。
      </li>
     </ul>
    </blockquote>
    <h4>
     2.
     <strong>
      簇（Cluster）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：簇是文件系统管理的最小存储单位，通常由多个扇区组成，常见的簇大小为
       <strong>
        2KB、4KB、8KB、16KB
       </strong>
       等。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：簇是操作系统进行文件存储时的单位。当文件被写入硬盘时，操作系统并不会按单个扇区存储，而是按簇来分配磁盘空间。如果文件大小小于簇的大小，剩余空间就会浪费掉。
      </li>
     </ul>
    </blockquote>
    <h4>
     3.
     <strong>
      分区（Partition）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：分区是硬盘的逻辑划分，目的是将硬盘划分为多个独立的区域，可以存储不同的数据或安装不同的操作系统。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：每个分区有自己的文件系统，可以独立管理和使用。硬盘通常可以有多个分区（如主分区、扩展分区、逻辑分区等），并且每个分区都有不同的存储区域和配置。
      </li>
     </ul>
    </blockquote>
    <h4>
     4.
     <strong>
      文件系统（File System）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：文件系统是对存储在分区上的数据进行管理和组织的结构。常见的文件系统有
       <strong>
        FAT32
       </strong>
       、
       <strong>
        NTFS
       </strong>
       、
       <strong>
        exFAT
       </strong>
       等。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：文件系统负责组织存储区域上的数据，使得文件能够在硬盘上有效地存储、读取、修改和删除。它通过目录结构、文件分配表（FAT）等机制来管理数据存储和访问。
      </li>
     </ul>
    </blockquote>
    <h4>
     5.
     <strong>
      主引导记录（MBR）和GUID分区表（GPT）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：MBR和GPT是两种分区表格式，用来描述硬盘的分区结构。
       <ul>
        <li>
         <strong>
          MBR（Master Boot Record）
         </strong>
         ：较旧的分区表格式，支持最多4个主分区或3个主分区和1个扩展分区。
        </li>
        <li>
         <strong>
          GPT（GUID Partition Table）
         </strong>
         ：较新的分区表格式，支持更大的硬盘和更多的分区，通常用于容量大于2TB的硬盘。
        </li>
       </ul>
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：分区表记录了硬盘的分区结构和分区的起始位置，帮助操作系统找到和访问每个分区。
      </li>
     </ul>
    </blockquote>

    <h3>
     (二).
     <strong>
      固态硬盘（SSD）存储区域构成
     </strong>
    </h3>
    <p>
     SSD的存储区域结构不同于硬盘（HDD），主要由
     <strong>
      页（Page）
     </strong>
     、
     <strong>
      块（Block）
     </strong>
     、**存储单元（Cell）
     <strong>
      和
     </strong>
     闪存控制器（Controller）**等组成。
    </p>
    <h4>
     1.
     <strong>
      存储单元（Cell）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：SSD中的存储单元是存储数据的最基本单位，每个存储单元存储一定量的二进制信息（0或1）。
      </li>
      <li>
       <strong>
        类型
       </strong>
       ：根据技术的不同，存储单元有不同的类型，如
       <strong>
        SLC（单层单元）
       </strong>
       、
       <strong>
        MLC（多层单元）
       </strong>
       、
       <strong>
        TLC（三层单元）
       </strong>
       、
       <strong>
        QLC（四层单元）
       </strong>
       等。每种类型的存储单元能存储的位数不同。
       <ul>
        <li>
         <strong>
          SLC
         </strong>
         ：每个存储单元存储1位数据，速度快、耐用，但成本高。
        </li>
        <li>
         <strong>
          MLC
         </strong>
         ：每个存储单元存储2位数据，速度和耐用性较低，但成本较低。
        </li>
        <li>
         <strong>
          TLC
         </strong>
         ：每个存储单元存储3位数据，速度和耐用性进一步降低，但存储密度更高。
        </li>
        <li>
         <strong>
          QLC
         </strong>
         ：每个存储单元存储4位数据，存储密度最高，但耐用性最低。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h4>
     2.
     <strong>
      页（Page）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：页是SSD存储数据的基本单元，通常每个页的大小为
       <strong>
        4KB
       </strong>
       、
       <strong>
        8KB
       </strong>
       或更大。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：数据在SSD中按页存储，每次读写操作都是按页为单位进行的。一个页可能包含多个存储单元。
      </li>
     </ul>
    </blockquote>
    <h4>
     3.
     <strong>
      块（Block）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：块是SSD中存储的较大单位，每个块通常包含多个页。SSD的擦除操作是按块进行的，即必须擦除整个块才能重新写入数据。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：在SSD中，数据的写入和删除操作是按块进行的，而不是按单个页进行的。这也是SSD和传统硬盘的一个重要区别。
      </li>
     </ul>
    </blockquote>
    <h4>
     4.
     <strong>
      闪存控制器（Controller）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：闪存控制器是SSD的“大脑”，它负责管理存储单元、页和块的读写操作，执行垃圾回收（Garbage Collection）、磨损均衡（Wear Leveling）等操作。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：控制器决定数据如何在SSD中分配、读取、写入和擦除。它通过优化存储和减少写入次数，延长了SSD的使用寿命。
      </li>
     </ul>
    </blockquote>
    <h4>
     5.
     <strong>
      垃圾回收（Garbage Collection）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：垃圾回收是SSD管理的一个重要过程，用来清理无效或删除的数据。由于SSD不能像硬盘那样直接删除数据，它必须将无效的数据标记为可擦除，等到适当时机进行清理。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：垃圾回收优化了存储空间的使用，避免了“存储块”中长期存在无效数据。
      </li>
     </ul>
    </blockquote>
    <h4>
     6.
     <strong>
      磨损均衡（Wear Leveling）
     </strong>
    </h4>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：磨损均衡是SSD控制器的一项技术，用于均匀分配写入次数，避免某些存储单元过早损坏。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：SSD的存储单元有限次写入，磨损均衡算法确保每个存储单元都被均匀使用，从而提高SSD的寿命和可靠性。
      </li>
     </ul>
    </blockquote>
    <h3>
     (三).
     <strong>
      U盘
     </strong>
    </h3>
    <h4>
     <strong>
      1.u盘的存储单元
     </strong>
    </h4>
    <p>
     U盘的存储介质主要由
     <strong>
      NAND闪存芯片
     </strong>
     组成，它是一种非易失性存储器，能够在断电的情况下保持数据。NAND闪存芯片的存储结构通常由多个
     <strong>
      存储单元
     </strong>
     组成，这些存储单元以页（Page）为单位进行数据存储，并按块（Block）进行管理。
    </p>
    <h5>
     1.1
     <strong>
      存储单元（Cell）
     </strong>
    </h5>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：存储单元是闪存中最小的数据存储单元，每个存储单元负责存储二进制数据（0或1）。
      </li>
      <li>
       <strong>
        类型
       </strong>
       ：根据存储密度的不同，存储单元可以分为
       <strong>
        SLC（单层单元）
       </strong>
       、
       <strong>
        MLC（多层单元）
       </strong>
       、
       <strong>
        TLC（三层单元）
       </strong>
       和
       <strong>
        QLC（四层单元）
       </strong>
       等。
       <ul>
        <li>
         <strong>
          SLC
         </strong>
         ：每个存储单元存储1位数据，速度快，耐用，适用于高性能应用。
        </li>
        <li>
         <strong>
          MLC
         </strong>
         ：每个存储单元存储2位数据，耐用性较差，但存储密度较高，常见于消费级产品。
        </li>
        <li>
         <strong>
          TLC
         </strong>
         ：每个存储单元存储3位数据，进一步提高存储密度，但寿命较短。
        </li>
        <li>
         <strong>
          QLC
         </strong>
         ：每个存储单元存储4位数据，存储密度更高，但耐用性和性能较低。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h5>
     1.2
     <strong>
      页（Page）
     </strong>
    </h5>
    <blockquote>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：页是U盘中存储数据的最小单位，每个页由多个存储单元组成，常见的页大小为
       <strong>
        4KB
       </strong>
       或
       <strong>
        8KB
       </strong>
       。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：数据的读取和写入是按页进行的，通常一个操作会将数据从一个页中读取或写入。
      </li>
     </ul>
    </blockquote>
    <h5>
     1.3
     <strong>
      块（Block）
     </strong>
    </h5>
    <blockquote>
     <p>
     </p>
     <ul>
      <li>
       <strong>
        定义
       </strong>
       ：块是U盘中较大的存储单位，每个块通常包含多个页，常见的块大小为
       <strong>
        128KB
       </strong>
       、
       <strong>
        256KB
       </strong>
       或更大。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：U盘的擦除操作是按块进行的，即需要擦除整个块才能重新写入数据。由于闪存的特性，无法直接覆盖原数据，因此需要先擦除整个块，再将数据写入。
      </li>
     </ul>
    </blockquote>
    <h4>
     2.
     <strong>
      U盘的管理结构：
     </strong>
    </h4>
    <h5>
     2.1
     <strong>
      闪存控制器（Controller）
     </strong>
    </h5>
    <blockquote>
     <ul>
      <li>
       <strong>
        功能
       </strong>
       ：闪存控制器是U盘的“大脑”，它负责管理闪存芯片上的数据存储和读写操作。控制器通过硬件和算法（如垃圾回收、磨损均衡等）来优化数据的存储和读取过程。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：控制器管理数据的读写、删除和清理过程，确保数据的可靠性，并延长U盘的使用寿命。
      </li>
     </ul>
    </blockquote>
    <h5>
     2.2
     <strong>
      磨损均衡（Wear Leveling）
     </strong>
    </h5>
    <blockquote>
     <ul>
      <li>
       <strong>
        功能
       </strong>
       ：闪存芯片的存储单元有限次写入，磨损均衡是一种算法，用来确保每个存储单元的写入次数尽可能均匀，避免某些存储单元过早损坏。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：通过均衡地分配写入操作，避免部分存储单元频繁写入，而其他单元几乎不被使用，延长U盘的使用寿命。
      </li>
     </ul>
    </blockquote>
    <h5>
     2.3
     <strong>
      垃圾回收（Garbage Collection）
     </strong>
    </h5>
    <blockquote>
     <ul>
      <li>
       <strong>
        功能
       </strong>
       ：垃圾回收是一种技术，用于清理无效的数据块，释放空间，以便存储新的数据。由于U盘中的数据不能像传统硬盘那样直接删除，控制器需要定期执行垃圾回收操作，标记无效数据块并重新利用它们。
      </li>
      <li>
       <strong>
        作用
       </strong>
       ：垃圾回收优化了存储空间的使用，减少了写入操作的成本，提高了存储效率。
      </li>
     </ul>
    </blockquote>
    <h4>
     3.
     <strong>
      文件系统：
     </strong>
    </h4>
    <blockquote>
     <p>
      U盘的存储空间通常会被格式化为某种文件系统，以便于操作系统能够管理和访问存储在其中的数据。常见的文件系统包括：
     </p>
     <ul>
      <li>
       <strong>
        FAT32
       </strong>
       ：广泛支持的文件系统，适用于较小的存储设备，最大支持的单个文件大小为
       <strong>
        4GB
       </strong>
       。
      </li>
      <li>
       <strong>
        exFAT
       </strong>
       ：为了解决FAT32的文件大小限制，exFAT适用于大容量U盘，支持大于4GB的文件。
      </li>
      <li>
       <strong>
        NTFS
       </strong>
       ：虽然NTFS主要用于硬盘，但某些U盘也可以使用NTFS格式，适合需要更高数据保护和文件系统特性的用户。
      </li>
     </ul>
     <p>
      文件系统将U盘的存储空间划分为多个逻辑块（簇）并管理它们的访问。它还负责跟踪文件的存储位置、文件大小和其他元数据。
     </p>
    </blockquote>
    <h4>
     4.
     <strong>
      U盘的分区：
     </strong>
    </h4>
    <blockquote>
     <p>
      U盘通常会包含一个或多个分区。每个分区都有独立的文件系统，可以存储不同的数据或支持不同的操作系统。
     </p>
     <ul>
      <li>
       <strong>
        主分区
       </strong>
       ：通常是U盘的默认分区，用于存储文件和数据。
      </li>
      <li>
       <strong>
        扩展分区
       </strong>
       （较少见）：类似硬盘的扩展分区，U盘可以通过扩展分区来创建更多的逻辑分区。
      </li>
     </ul>
    </blockquote>

    <h2>
     二，FAT32
    </h2>
    <p>
     <img alt="" height="580" src="https://i-blog.csdnimg.cn/direct/685f9d90e1874425909c0c4c6ac8b1b1.png" width="1169"/>
    </p>
    <h3 style="background-color:transparent">
     (一).什么是FAT32
    </h3>
    <blockquote>
     <p>
      FAT32(
      <strong>
       File Allocation Table 32
      </strong>
      )，中文名：32位文件分配表。
      <span style="color:#fe2c24">
       FAT
      </span>
      是一种表格，它记录了文件在存储介质上的位置。每个
      <span style="color:#fe2c24">
       文件或文件夹
      </span>
      在磁盘上都有
      <span style="color:#fe2c24">
       对应的存储位置
      </span>
      ，而这个位置是通过
      <span style="color:#fe2c24">
       FAT 表
      </span>
      来管理的。FAT表中的每一项都记录了文件的一部分位置和是否被占用等信息。
      <span style="color:#fe2c24">
       <strong>
        32
       </strong>
      </span>
      ：数字 32 指的是文件系统中使用的
      <span style="color:#fe2c24">
       “簇”（cluster）地址的位数
      </span>
      。FAT32 使用 32 位来表示簇地址，这意味着它可以寻址更多的存储空间，从而支持更大的磁盘和更大的文件。相较于老旧的 FAT16（16 位寻址）和 FAT12（12 位寻址），FAT32 能够支持更大容量的存储设备。
     </p>
    </blockquote>
    <h3>
     (二)，什么是簇
    </h3>
    <blockquote>
     <p>
      在文件系统中，**簇（Cluster）**是存储数据的最小单位。它是由多个扇区（Sector）组成的一个块。在硬盘或其他存储介质上，数据并不是按字节直接存储的，而是按簇进行分配的。
     </p>
     <ul>
      <li>
       <p>
        <strong>
         扇区与簇
        </strong>
        ：
       </p>
       <ul>
        <li>
         **扇区（Sector）**是硬盘的最小存储单位，通常一个扇区的大小是
         <strong>
          512字节
         </strong>
         （虽然一些新硬盘可能是4KB）。
        </li>
        <li>
         <span style="color:#fe2c24">
          注：扇区是计算机读写的最小单位
         </span>
        </li>
        <li>
         <strong>
          簇
         </strong>
         是由一个或多个扇区组成的。当文件系统需要存储一个文件时，它会将文件分配给一个或多个簇，而不是直接分配给一个个独立的扇区。簇的大小一般是2的幂次方，比如 1KB、2KB、4KB、8KB 等，具体大小取决于分区的总大小。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         簇的作用
        </strong>
        ：
       </p>
       <ul>
        <li>
         <strong>
          管理文件存储
         </strong>
         ：每个文件会占用至少一个簇。如果文件的大小小于簇的大小，剩余的空间则浪费掉。例如，如果簇的大小是4KB，但文件只有2KB，那么仍然会占用4KB的空间，剩下的2KB就无法用来存储其他文件。
        </li>
        <li>
         <strong>
          提高效率
         </strong>
         ：簇的使用减少了管理大量单个扇区的复杂性，因此提高了文件系统的效率。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         簇的大小
        </strong>
        ：
       </p>
       <ul>
        <li>
         簇的大小通常由文件系统在格式化时决定，并且与磁盘的总大小以及使用的文件系统类型（如FAT32、NTFS等）有关。一般来说，较大的磁盘使用较大的簇，以提高访问速度和减少管理开销。
        </li>
        <li>
         在FAT32中，簇的大小可能是1KB、2KB、4KB、8KB、16KB、32KB等。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         簇和文件存储
        </strong>
        ：
       </p>
       <ul>
        <li>
         当你保存一个文件时，操作系统会找到足够的簇来存储它。如果文件很小，它可能只占用一个簇。如果文件较大，它可能会跨越多个簇存储。盘
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h2>
     三，NTFS
    </h2>
    <h3>
     (一).什么是NTFS
    </h3>
    <p>
     <strong>
      NTFS
     </strong>
     （New Technology File System），中文名：新科技文件系统，是由微软开发的一种文件系统。
    </p>
    <h3>
     (二).NTFS的特点和优点：
    </h3>
    <blockquote>
     <ol>
      <li>
       <p>
        <strong>
         支持大文件和大分区
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS支持非常大的文件和分区，最大支持单个文件大小达
         <strong>
          16 EB（Exabytes）
         </strong>
         ，而最大分区大小可以达到
         <strong>
          256 TB
         </strong>
         。相比之下，FAT32的最大文件大小限制为4GB。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         文件和文件夹权限
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS允许为文件和文件夹设置
         <strong>
          访问控制列表（ACLs）
         </strong>
         ，从而对文件的读取、写入、执行等权限进行细粒度的控制。这使得NTFS在安全性方面比FAT32更强大。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         支持文件压缩和加密
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS支持
         <strong>
          文件级别的压缩
         </strong>
         ，可以节省磁盘空间，同时还支持文件加密（通过
         <strong>
          EFS
         </strong>
         ，即加密文件系统）。这增强了数据的安全性和存储效率。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         日志记录功能（Journaling）
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS具备
         <strong>
          日志记录功能
         </strong>
         ，它可以记录文件系统的操作日志，以帮助恢复损坏的数据或避免数据丢失。这是NTFS区别于FAT32的重要特点之一，因为FAT32没有此功能。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         硬链接和符号链接
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS支持
         <strong>
          硬链接
         </strong>
         和
         <strong>
          符号链接
         </strong>
         （symlinks），这使得文件可以在不同位置共享或引用，提供了更灵活的文件管理方式。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         分区和存储效率
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS使用更先进的方式来管理磁盘空间，相比FAT32，它能更有效地管理存储设备，特别是对于大容量磁盘和高性能需求的场景。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         兼容性
        </strong>
        ：
       </p>
       <ul>
        <li>
         虽然NTFS在Windows系统中得到广泛使用，但其他操作系统（如macOS和Linux）对NTFS的支持相对有限。macOS不能原生写入NTFS磁盘，Linux则可以通过安装额外的驱动来读写NTFS格式的磁盘。
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         碎片整理
        </strong>
        ：
       </p>
       <ul>
        <li>
         NTFS支持自动碎片整理功能，虽然现代操作系统会定期优化磁盘，但NTFS的设计本身便减少了磁盘碎片的产生。
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>

    <h2>
     五，主分区和逻辑分区
    </h2>
    <p>
     **主分区（Primary Partition）
     <strong>
      和
     </strong>
     逻辑分区（Logical Partition）**是硬盘分区类型的两种不同形式，它们之间有一些重要的区别，主要体现在如何分配和管理硬盘的空间。
    </p>
    <h3>
     (一)
     <strong>
      主分区（Primary Partition）
     </strong>
    </h3>
    <blockquote>
     <p>
      <strong>
       定义
      </strong>
      ： 主分区是硬盘分区的基本类型，它可以直接被操作系统识别和使用。一个硬盘最多可以有
      <strong>
       4个主分区
      </strong>
      ，也可以将硬盘中的一部分空间划分为扩展分区，以便创建更多的逻辑分区。
     </p>
     <p>
      <strong>
       特点
      </strong>
      ：
     </p>
     <ul>
      <li>
       每个硬盘最多可以有
       <strong>
        4个主分区
       </strong>
       。如果需要更多的分区，则可以将其中一个主分区转换为扩展分区。
      </li>
      <li>
       主分区直接用于存储数据或安装操作系统，操作系统能够识别并直接访问主分区。
      </li>
      <li>
       主分区中的一个可以被设置为启动分区，也就是安装操作系统的分区。
      </li>
     </ul>
     <p>
      <strong>
       为什么有主分区
      </strong>
      ： 主分区的设计是为了确保操作系统能直接访问硬盘上的存储区域，且支持启动操作系统。最初的MBR（主引导记录）分区表标准仅允许最多四个分区，因此引入了主分区。
     </p>
    </blockquote>
    <h3>
     (二).
     <strong>
      逻辑分区（Logical Partition）
     </strong>
    </h3>
    <blockquote>
     <p>
      <strong>
       定义
      </strong>
      ： 逻辑分区是通过扩展分区来创建的，它并不是直接被操作系统识别的，而是依赖于扩展分区的结构。逻辑分区是从扩展分区中分出来的，用于在硬盘上创建更多的分区。
     </p>
     <p>
      <strong>
       特点
      </strong>
      ：
     </p>
     <ul>
      <li>
       逻辑分区必须位于扩展分区中，而扩展分区本身只能有一个。扩展分区可以包含多个逻辑分区。
      </li>
      <li>
       逻辑分区的数量理论上没有硬性限制，通常可以创建多个逻辑分区，这对于需要更多分区的用户非常有用。
      </li>
      <li>
       逻辑分区和主分区一样，也可以用于存储数据、安装操作系统等，但它们需要通过扩展分区来管理。
      </li>
     </ul>
     <p>
      <strong>
       为什么有逻辑分区
      </strong>
      ： 由于传统的MBR分区表最多只支持4个分区，为了扩展硬盘分区的数量，逻辑分区便应运而生。逻辑分区使得用户可以在一个扩展分区内创建多个分区，从而突破了MBR的限制。
     </p>
    </blockquote>

    <h3>
     (四).
     <strong>
      为什么要有主分区和逻辑分区的区分？
     </strong>
    </h3>
    <blockquote>
     <ul>
      <li>
       <strong>
        兼容性与扩展性
       </strong>
       ：传统的MBR分区表只允许最多4个主分区，因此为了在有限的4个分区中使用更多的分区，扩展分区和逻辑分区应运而生。扩展分区使得用户能够创建多个逻辑分区，打破了MBR分区表的4分区限制。
      </li>
      <li>
       <strong>
        启动分区管理
       </strong>
       ：为了让操作系统能够顺利启动，主分区支持标记为启动分区。因此，主分区在操作系统安装和启动过程中扮演着重要角色。
      </li>
      <li>
       <strong>
        分区管理的灵活性
       </strong>
       ：通过使用逻辑分区，用户可以在同一个扩展分区中管理多个逻辑分区，这使得磁盘分区的管理更灵活，尤其适用于需要创建多个数据存储区的情况。
      </li>
     </ul>
    </blockquote>
