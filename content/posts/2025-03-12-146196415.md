    <h2>
     <a id="MongoDB__Schemaless_0">
     </a>
     <strong>
      MongoDB 的 Schema-less（无模式）特性
     </strong>
    </h2>
    <p>
     MongoDB
     <strong>
      不需要提前定义表结构（Schema）
     </strong>
     ，可以存储
     <strong>
      不同结构
     </strong>
     的文档（JSON/BSON），这与传统的 SQL 关系型数据库（RDBMS）有很大不同。
    </p>
    <h3>
     <a id="1_MongoDB__4">
     </a>
     <strong>
      1. MongoDB 的无模式特性
     </strong>
    </h3>
    <p>
     在 MongoDB 中：
    </p>
    <ul>
     <li>
      每个
      <strong>
       集合（Collection）
      </strong>
      类似于 SQL 的表（Table）。
     </li>
     <li>
      但集合中的
      <strong>
       文档（Document）
      </strong>
      （类似于 SQL 的行 Row）可以有
      <strong>
       不同的字段（Columns）
      </strong>
      ，无需固定结构。
     </li>
     <li>
      你可以向同一个集合中插入不同结构的数据，MongoDB 会自动处理这些数据。
     </li>
    </ul>
    <h5>
     <a id="MongoDB__12">
     </a>
     <strong>
      示例：MongoDB 允许不同结构的数据
     </strong>
    </h5>
    <pre><code class="prism language-json"><span class="token comment">// 同一个集合（people），但结构不同</span>
<span class="token punctuation">{<!-- --></span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Alice"</span><span class="token punctuation">,</span> <span class="token string-property property">"age"</span><span class="token operator">:</span> <span class="token number">25</span> <span class="token punctuation">}</span>
<span class="token punctuation">{<!-- --></span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Bob"</span><span class="token punctuation">,</span> <span class="token string-property property">"age"</span><span class="token operator">:</span> <span class="token number">30</span><span class="token punctuation">,</span> <span class="token string-property property">"email"</span><span class="token operator">:</span> <span class="token string">"bob@example.com"</span> <span class="token punctuation">}</span>
<span class="token punctuation">{<!-- --></span> <span class="token string-property property">"name"</span><span class="token operator">:</span> <span class="token string">"Charlie"</span><span class="token punctuation">,</span> <span class="token string-property property">"age"</span><span class="token operator">:</span> <span class="token number">28</span><span class="token punctuation">,</span> <span class="token string-property property">"address"</span><span class="token operator">:</span> <span class="token string">"New York"</span><span class="token punctuation">,</span> <span class="token string-property property">"phone"</span><span class="token operator">:</span> <span class="token string">"123456789"</span> <span class="token punctuation">}</span>
</code></pre>
    <p>
     这里：
    </p>
    <ul>
     <li>
      Alice 只有
      <code>
       name
      </code>
      和
      <code>
       age
      </code>
     </li>
     <li>
      Bob 额外有
      <code>
       email
      </code>
     </li>
     <li>
      Charlie 额外有
      <code>
       address
      </code>
      和
      <code>
       phone
      </code>
     </li>
    </ul>
    <p>
     MongoDB
     <strong>
      不会报错
     </strong>
     ，而 SQL 关系数据库（如 MySQL、PostgreSQL）要求所有记录（Row）遵循
     <strong>
      相同的列结构
     </strong>
     ，否则会导致插入失败。
    </p>
    <hr/>
    <h3>
     <a id="2_MongoDB_vs_SQL_31">
     </a>
     <strong>
      2. MongoDB vs SQL：结构对比
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        MongoDB（NoSQL）
       </th>
       <th>
        SQL 关系数据库（RDBMS）
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         数据存储单位
        </strong>
       </td>
       <td>
        文档（JSON/BSON）
       </td>
       <td>
        表（Table）中的行（Row）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         模式（Schema）
        </strong>
       </td>
       <td>
        无模式（Schema-less），字段可以不固定
       </td>
       <td>
        需要预定义模式，所有行必须符合表结构
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         字段约束
        </strong>
       </td>
       <td>
        没有强制约束，字段可以缺失或不同
       </td>
       <td>
        需要在表定义时固定字段和数据类型
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         数据查询
        </strong>
       </td>
       <td>
        使用 BSON 文档查询（MongoDB Query Language）
       </td>
       <td>
        使用 SQL 查询
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         扩展性
        </strong>
       </td>
       <td>
        水平扩展（Sharding）较容易
       </td>
       <td>
        垂直扩展为主，水平扩展较难
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         事务
        </strong>
       </td>
       <td>
        事务支持较弱（支持 ACID 事务，但不如 SQL 强大）
       </td>
       <td>
        强事务支持（ACID）
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <a id="3_MongoDB__SQL__44">
     </a>
     <strong>
      3. MongoDB 与 SQL 主要不同点
     </strong>
    </h3>
    <h4>
     <a id="_46">
     </a>
     <strong>
      插入数据
     </strong>
    </h4>
    <h5>
     <a id="MongoDB_48">
     </a>
     <strong>
      MongoDB
     </strong>
    </h5>
    <p>
     可以向同一个集合（Collection）中插入不同结构的文档：
    </p>
    <pre><code class="prism language-python"><span class="token keyword">from</span> pymongo <span class="token keyword">import</span> MongoClient

client <span class="token operator">=</span> MongoClient<span class="token punctuation">(</span><span class="token string">"mongodb://localhost:27017/"</span><span class="token punctuation">)</span>
db <span class="token operator">=</span> client<span class="token punctuation">[</span><span class="token string">"testdb"</span><span class="token punctuation">]</span>
collection <span class="token operator">=</span> db<span class="token punctuation">[</span><span class="token string">"people"</span><span class="token punctuation">]</span>

<span class="token comment"># 插入不同结构的数据</span>
collection<span class="token punctuation">.</span>insert_one<span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token string">"name"</span><span class="token punctuation">:</span> <span class="token string">"Alice"</span><span class="token punctuation">,</span> <span class="token string">"age"</span><span class="token punctuation">:</span> <span class="token number">25</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
collection<span class="token punctuation">.</span>insert_one<span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token string">"name"</span><span class="token punctuation">:</span> <span class="token string">"Bob"</span><span class="token punctuation">,</span> <span class="token string">"age"</span><span class="token punctuation">:</span> <span class="token number">30</span><span class="token punctuation">,</span> <span class="token string">"email"</span><span class="token punctuation">:</span> <span class="token string">"bob@example.com"</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
collection<span class="token punctuation">.</span>insert_one<span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token string">"name"</span><span class="token punctuation">:</span> <span class="token string">"Charlie"</span><span class="token punctuation">,</span> <span class="token string">"age"</span><span class="token punctuation">:</span> <span class="token number">28</span><span class="token punctuation">,</span> <span class="token string">"address"</span><span class="token punctuation">:</span> <span class="token string">"New York"</span><span class="token punctuation">}</span><span class="token punctuation">)</span>

<span class="token comment"># 查询所有数据</span>
<span class="token keyword">for</span> doc <span class="token keyword">in</span> collection<span class="token punctuation">.</span>find<span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">:</span>
    <span class="token keyword">print</span><span class="token punctuation">(</span>doc<span class="token punctuation">)</span>
</code></pre>
    <p>
     <strong>
      特点：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       无需创建表结构
      </strong>
     </li>
     <li>
      <strong>
       不同文档的字段可以不同
      </strong>
     </li>
     <li>
      <strong>
       直接插入数据，MongoDB 会自动存储
      </strong>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="SQLMySQL__77">
     </a>
     <strong>
      SQL（MySQL 示例）
     </strong>
    </h5>
    <pre><code class="prism language-sql"><span class="token keyword">CREATE</span> <span class="token keyword">TABLE</span> people <span class="token punctuation">(</span>
    id <span class="token keyword">INT</span> <span class="token keyword">AUTO_INCREMENT</span> <span class="token keyword">PRIMARY</span> <span class="token keyword">KEY</span><span class="token punctuation">,</span>
    name <span class="token keyword">VARCHAR</span><span class="token punctuation">(</span><span class="token number">100</span><span class="token punctuation">)</span> <span class="token operator">NOT</span> <span class="token boolean">NULL</span><span class="token punctuation">,</span>
    age <span class="token keyword">INT</span> <span class="token operator">NOT</span> <span class="token boolean">NULL</span><span class="token punctuation">,</span>
    email <span class="token keyword">VARCHAR</span><span class="token punctuation">(</span><span class="token number">255</span><span class="token punctuation">)</span><span class="token punctuation">,</span>
    address <span class="token keyword">VARCHAR</span><span class="token punctuation">(</span><span class="token number">255</span><span class="token punctuation">)</span>
<span class="token punctuation">)</span><span class="token punctuation">;</span>

<span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> people <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'Alice'</span><span class="token punctuation">,</span> <span class="token number">25</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> people <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">,</span> email<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'Bob'</span><span class="token punctuation">,</span> <span class="token number">30</span><span class="token punctuation">,</span> <span class="token string">'bob@example.com'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token keyword">INSERT</span> <span class="token keyword">INTO</span> people <span class="token punctuation">(</span>name<span class="token punctuation">,</span> age<span class="token punctuation">,</span> address<span class="token punctuation">)</span> <span class="token keyword">VALUES</span> <span class="token punctuation">(</span><span class="token string">'Charlie'</span><span class="token punctuation">,</span> <span class="token number">28</span><span class="token punctuation">,</span> <span class="token string">'New York'</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <strong>
      特点：
     </strong>
    </p>
    <ul>
     <li>
      必须
      <strong>
       先定义表结构
      </strong>
      （CREATE TABLE）
     </li>
     <li>
      插入数据时，
      <strong>
       必须符合表的字段
      </strong>
     </li>
     <li>
      不能动态增加字段，除非
      <strong>
       ALTER TABLE
      </strong>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_101">
     </a>
     <strong>
      数据查询
     </strong>
    </h4>
    <h5>
     <a id="MongoDB_103">
     </a>
     <strong>
      MongoDB
     </strong>
    </h5>
    <pre><code class="prism language-python"><span class="token comment"># 查询 name 为 "Alice" 的人</span>
result <span class="token operator">=</span> collection<span class="token punctuation">.</span>find_one<span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token string">"name"</span><span class="token punctuation">:</span> <span class="token string">"Alice"</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
<span class="token keyword">print</span><span class="token punctuation">(</span>result<span class="token punctuation">)</span>
</code></pre>
    <p>
     使用
     <strong>
      JSON/BSON 格式查询
     </strong>
    </p>
    <hr/>
    <h5>
     <a id="SQL_115">
     </a>
     <strong>
      SQL
     </strong>
    </h5>
    <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> <span class="token operator">*</span> <span class="token keyword">FROM</span> people <span class="token keyword">WHERE</span> name <span class="token operator">=</span> <span class="token string">'Alice'</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     如果字段不存在（例如
     <code>
      phone
     </code>
     ），需要
     <strong>
      ALTER TABLE
     </strong>
     添加
    </p>
    <hr/>
    <h4>
     <a id="_125">
     </a>
     <strong>
      结构变化的影响
     </strong>
    </h4>
    <h5>
     <a id="MongoDB_127">
     </a>
     <strong>
      MongoDB
     </strong>
    </h5>
    <p>
     如果需要给某些文档增加
     <code>
      phone
     </code>
     字段，可以直接更新：
    </p>
    <pre><code class="prism language-python">collection<span class="token punctuation">.</span>update_one<span class="token punctuation">(</span><span class="token punctuation">{<!-- --></span><span class="token string">"name"</span><span class="token punctuation">:</span> <span class="token string">"Alice"</span><span class="token punctuation">}</span><span class="token punctuation">,</span> <span class="token punctuation">{<!-- --></span><span class="token string">"$set"</span><span class="token punctuation">:</span> <span class="token punctuation">{<!-- --></span><span class="token string">"phone"</span><span class="token punctuation">:</span> <span class="token string">"123456789"</span><span class="token punctuation">}</span><span class="token punctuation">}</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     <strong>
      MongoDB 允许文档的字段动态增加
     </strong>
     ，不影响已有数据。
    </p>
    <hr/>
    <h5>
     <a id="SQL_139">
     </a>
     <strong>
      SQL
     </strong>
    </h5>
    <p>
     如果想给
     <code>
      people
     </code>
     表增加
     <code>
      phone
     </code>
     字段：
    </p>
    <pre><code class="prism language-sql"><span class="token keyword">ALTER</span> <span class="token keyword">TABLE</span> people <span class="token keyword">ADD</span> <span class="token keyword">COLUMN</span> phone <span class="token keyword">VARCHAR</span><span class="token punctuation">(</span><span class="token number">20</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <strong>
      SQL 需要修改整个表结构
     </strong>
     ，否则无法存储新的字段。
    </p>
    <hr/>
    <h3>
     <a id="4__MongoDB_SQL_151">
     </a>
     <strong>
      4. 什么时候选择 MongoDB？什么时候选择 SQL？
     </strong>
    </h3>
    <h4>
     <a id="_MongoDB__153">
     </a>
     <strong>
      适合 MongoDB 的场景
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       数据结构不固定
      </strong>
      ，经常变化（如电商商品、社交媒体动态）
     </li>
     <li>
      <strong>
       高并发读写
      </strong>
      ，需要快速查询和写入（如日志、IoT 设备数据）
     </li>
     <li>
      <strong>
       海量数据存储
      </strong>
      ，需要水平扩展（如大数据分析）
     </li>
     <li>
      <strong>
       JSON 格式存储需求
      </strong>
      ，适用于
      <strong>
       RESTful API
      </strong>
     </li>
    </ul>
    <h4>
     <a id="_SQL__160">
     </a>
     <strong>
      适合 SQL 关系数据库的场景
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       数据结构固定，表结构清晰
      </strong>
      （如银行系统、ERP）
     </li>
     <li>
      <strong>
       强事务支持（ACID）
      </strong>
      （如金融支付系统）
     </li>
     <li>
      <strong>
       复杂的多表 JOIN 查询
      </strong>
      （如关系紧密的数据管理）
     </li>
    </ul>
    <hr/>

      灵活性高、数据结构多变
     </strong>
     的场景，而 SQL 关系数据库更适合
     <strong>
      结构清晰、事务要求严格
     </strong>
     的应用。
    </p>
    <h2>
     <a id="MongoDB__183">
     </a>
     <strong>
      MongoDB 环境部署
     </strong>
    </h2>
    <h3>
     <a id="1__MongoDB_185">
     </a>
     <strong>
      1. 安装 MongoDB
     </strong>
    </h3>
    <h5>
     <a id="Ubuntu__Debian_187">
     </a>
     <strong>
      Ubuntu / Debian
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> mongodb
</code></pre>
    <h5>
     <a id="CentOS__RHEL_194">
     </a>
     <strong>
      CentOS / RHEL
     </strong>
    </h5>
    <pre><code>sudo yum install -y mongodb-org
</code></pre>
    <h5>
     <a id="Windows_200">
     </a>
     <strong>
      Windows
     </strong>
    </h5>
    <ol>
     <li>
      前往
      <a href="https://www.mongodb.com/try/download/community" rel="nofollow">
       MongoDB 官网
      </a>
      下载安装包
     </li>
     <li>
      安装后，确保 MongoDB 在系统可用：
     </li>
    </ol>
    <pre><code class="prism language-bash">mongod <span class="token parameter variable">--version</span>
</code></pre>
    <h3>
     <a id="2__MongoDB_209">
     </a>
     <strong>
      2. 启动 MongoDB
     </strong>
    </h3>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl start mongod
<span class="token function">sudo</span> systemctl <span class="token builtin class-name">enable</span> mongod
</code></pre>
    <p>
     确保 MongoDB 成功启动：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl status mongod
</code></pre>
    <h3>
     <a id="3__MongoDB_222">
     </a>
     <strong>
      3. 初始化 MongoDB
     </strong>
    </h3>
    <p>
     MongoDB 需要预先定义数据库文件夹，可以手动创建并在配置文件中修改，也可以按照默认的文件夹路径。
    </p>
    <p>
     如果需要定制数据目录，可以修改配置文件
     <code>
      /etc/mongod.conf
     </code>
     ：
    </p>
    <pre><code class="prism language-ini">storage:
  dbPath: /var/lib/mongodb
systemLog:
  destination: file
  path: /var/log/mongodb/mongod.log
  logAppend: true
</code></pre>
    <p>
     修改配置后，重启 MongoDB：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart mongod
</code></pre>
    <blockquote>
     <p>
      [!TIP]
     </p>
     <p>
      有时可能会遇到文件权限问题，可以首先手动启动数据库
      <code>
       mongod
      </code>
      查看报错信息。如果实在不行可以修改守护进程配置文件，将启动命令的配置删除。
      <code>
       --config /etc/mongod.conf
      </code>
     </p>
    </blockquote>
