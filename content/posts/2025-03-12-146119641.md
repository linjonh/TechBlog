    </h2>
    <h3 id="1.1%20listen%E7%9B%91%E5%90%AC%E5%87%BD%E6%95%B0" name="1.1%20listen%E7%9B%91%E5%90%AC%E5%87%BD%E6%95%B0">
     1.1 listen监听函数
    </h3>
    <p>
     Tcp协议相较于Udp协议，在双方进行通信之前，需要进行连接。正如你是一家饭店老板，客人在用餐前，都需要先跟你进行协商。
    </p>
    <p>
     <img alt="" height="183" src="https://i-blog.csdnimg.cn/direct/6982970028124ee691fb26406588bc95.png" width="776"/>
    </p>
    <p>
     listen函数的作用就是将套接字设置为监听状态，以便随时接收传过来的连接。
    </p>
    <ul>
     <li>
      该函数的第一个参数，是连接方套接字的文件描述符。想象一下，如果你有一家饭店位于市区，人潮涌动，竞争也十分激烈。这时，你可能会派遣一位机灵的小伙子到外面去招揽顾客。这个文件描述符就相当于这位负责招揽顾客的小伙子。
     </li>
     <li>
      第二个参数，是在连接请求过多，服务器处理不过来时，能够容纳的新连接的数量。就好比您的餐厅位于人流量极大的市区，一到饭点，餐厅就座无虚席。当餐厅内没有空位时，服务员通常会为等待的顾客提供号码牌，让他们稍作休息。
     </li>
     <li>
      但是，不可能无限制地提供号码牌，因此会有一个上限。这个参数的作用就是限制操作系统接收但尚未处理的连接数量。通常情况下，这个值会设置在10个左右。
     </li>
    </ul>
    <p>
    </p>
    <p>
     <img alt="" height="72" src="https://i-blog.csdnimg.cn/direct/e72454fdbde14795a866379c63b9fede.png" width="867"/>
    </p>
    <p>
     该函数返回值是一个整数。如果调用成功，返回0；如果调用失败，返回-1，并且错误码会被设置。
    </p>
    <p>
    </p>
    <p>
    </p>
    <h3 id="1.2%20accept%E5%87%BD%E6%95%B0" name="1.2%20accept%E5%87%BD%E6%95%B0">
     1.2 accept函数
    </h3>
    <p>
     <img alt="" height="182" src="https://i-blog.csdnimg.cn/direct/646558669a364391b9693e134b8784d8.png" width="749"/>
    </p>
    <p>
     accept函数用于套接字接受连接，并创建新的文件描述给新连接。
    </p>
    <ul>
     <li>
      该函数的第一个参数是连接方套接字的文件描述符，就是上面所说的招揽顾客的小伙子。
     </li>
     <li>
      第二个参数是套接字地址类型的指针变量，是一个输出型参数，用于带出发起连接的主机信息。
     </li>
     <li>
      第三个参数是也是一个输出型参数，用于带出该套接字结构体类型大小。
     </li>
    </ul>
    <p>
     <img alt="" height="104" src="https://i-blog.csdnimg.cn/direct/554314d5906a4dc683d2bf849eb8b93d.png" width="752"/>
    </p>
    <p>
     这就是像上面的listen函数，它的作用是把顾客成功吸引到餐厅。这时，就需要餐厅内的服务员来接待他们。因此，accept函数的返回值是一个文件描述符，它用于处理这些连接的数据读写操作。如果该函数调用失败，会返回-1，错误码会被设置。
    </p>
    <p>
    </p>
    <p>
    </p>
    <h3 id="1.3%20connect%E5%87%BD%E6%95%B0" name="1.3%20connect%E5%87%BD%E6%95%B0">
     1.3 connect函数
    </h3>
    <p>
     <img alt="" height="188" src="https://i-blog.csdnimg.cn/direct/b897c8957b6b421081441c6cf3f9eefe.png" width="713"/>
    </p>
    <p>
     connect函数用于建立一个与指定地址的连接。该函数通常由客户端发起，连接指定的服务端，且使用的是Tcp套接字。
    </p>
    <ul>
     <li>
      sockfd参数是一个已经创建的文件描述符。
     </li>
     <li>
      addr参数是一个指向sockaddr的指针变量，里面需要包含要连接的远程服务器的IP地址和端口号。
     </li>
     <li>
      addrlen是表明addr结构体的大小。
     </li>
    </ul>
    <p>
    </p>
    <p>
    </p>
    <h3 id="1.4%20inet_ntop%E5%87%BD%E6%95%B0" name="1.4%20inet_ntop%E5%87%BD%E6%95%B0">
     1.4 inet_ntop函数
    </h3>
    <p>
     <img alt="" height="179" src="https://i-blog.csdnimg.cn/direct/1658482ba2794b759eb3dba4b43f89a6.png" width="772"/>
    </p>
    <p>
     <code>
      inet_ntop
     </code>
     是一个在计算机网络编程中常用的函数，用于将网络地址转换成字符串形式。以下是函数参数的说明：
    </p>
    <ul>
     <li>
      <code>
       af
      </code>
      ：地址族（Address Family）。它指定了源地址
      <code>
       src
      </code>
      的类型，可以是
      <code>
       AF_INET
      </code>
      （用于IPv4），
      <code>
       AF_INET6
      </code>
      （用于IPv6）等。
     </li>
     <li>
      <code>
       src
      </code>
      ：指向包含原始网络地址的指针。对于IPv4，这应该是一个
      <code>
       struct in_addr
      </code>
      类型；对于IPv6，则是一个
      <code>
       struct in6_addr
      </code>
      类型。
     </li>
     <li>
      <code>
       dst
      </code>
      ：指向目标缓冲区的指针，该缓冲区用于存储转换后的字符串形式的地址。
     </li>
     <li>
      <code>
       size
      </code>
      ：目标缓冲区
      <code>
       dst
      </code>
      的大小。
     </li>
    </ul>

    <p>

     <li>

     </li>
     <li>

    <h3 id="2.1%20TcpServer.hpp" name="2.1%20TcpServer.hpp">

    <h4 id="2.1.1%20TcpServer%E7%B1%BB" name="2.1.1%20TcpServer%E7%B1%BB">
     2.1.1 TcpServer类
    </h4>
    <p>
     TcpServer.hpp文件中，主要包含TcpServer类。下面介绍
    </p>
    <ul>
     <li>
      TcpServer类中的成员变量有监听文件描述符、服务启动的端口号和服务运行状态。
     </li>
     <li>
      构造函数是一个全缺省函数，可以使用gdefaultport中的端口，也可以外部传参。监听文件描述符初始化为gsockfd，gsockfd是一个全局变量，避免硬编码。服务器运行状态初始化为false，表示未启动。
     </li>
    </ul>
    <pre><code class="language-cpp">#ifndef __TCP_SERVER__HPP
#define __TCP_SERVER__HPP

#include &lt;iostream&gt;
#include &lt;string&gt;
#include &lt;cstring&gt;
#include &lt;memory&gt;
#include &lt;string.h&gt;

#include &lt;sys/types.h&gt;
#include &lt;sys/socket.h&gt;
#include &lt;netinet/in.h&gt;
#include &lt;arpa/inet.h&gt;
#include &lt;unistd.h&gt;
#include &lt;sys/wait.h&gt;
#include "Common.hpp"

using namespace LogModule;

#define BACKLOG 8

const static int gsockfd = -1;
const static uint16_t gdefaultport = 8888;

class TcpServer
{
public:
    TcpServer(const uint16_t port = gdefaultport)
        :_listen_sockfd(gsockfd)
        ,_port(port)
        ,_isrunning(false)
    {}

    ~TcpServer()
    {
        if (_listen_sockfd &gt; gsockfd)
            ::close(_listen_sockfd);
    }
private:
    int _listen_sockfd;
    uint16_t _port;  //服务器端口号
    bool _isrunning; //服务器运行状态
};

#endif</code></pre>

    </p>
    <h4 id="2.1.2%20InitServer%E5%87%BD%E6%95%B0" name="2.1.2%20InitServer%E5%87%BD%E6%95%B0">
     2.1.2 InitServer函数
    </h4>
    <p>
     InitServer函数，用于初始化整个Tcp服务。
    </p>
    <ul>
     <li>
      首先使用socket函数创建一个套接字，第一个参数传AF_INET，表示网络通信。第二个参数传SOCK_STREAM，表示使用Tcp协议，第三个参数默认传0即可。不过这个函数返回的文件描述符不用于进行IO操作，而是监听新连接。
     </li>
     <li>
      接着，填充网络信息。定义一个sockaddr_in结构体，该结构体内部有三个字段需要填充。第一个字段表示什么通信，填AF_INET，表示网络通信。第二字段填端口号，但是得使用htons函数将主机字节序转换成网络字节序，变成大端模式。
     </li>
     <li>
      第三个字段一般来说需要填写机器的IP地址，但是作为服务器，可能客户端发来的消息需要多台服务器处理不同的信息，如果服务器填写固定的IP地址，那么客户端接受服务器的消息后，只能返回给一台服务器。所以服务器的sin_addr中的s_addr一般设置INADDR_ANY。
     </li>
     <li>
      作为Tcp套接字，不仅需要调用socket和bind函数，还需要调用listen函数设置监听状态。
     </li>
    </ul>
    <pre><code class="language-cpp">    void InitServer()
    {
        // 1.创建套接字
        _listen_sockfd = ::socket(AF_INET, SOCK_STREAM, 0);
        if(_listen_sockfd &lt; 0)
        {
            std::cout &lt;&lt; "socket:" &lt;&lt; strerror(errno) &lt;&lt; std::endl;
            exit(SOCKET_ERR);
        }
        std::cout &lt;&lt; "socket success, _listen_sockfd is: " &lt;&lt; _listen_sockfd &lt;&lt; std::endl;
        
        // 2、填充网络信息，
        // 2.1 设置进入内核中，只是填充了结构体!
        struct sockaddr_in loacl;
        memset(&amp;loacl, 0, sizeof(loacl));
        loacl.sin_family = AF_INET;
        loacl.sin_port = ::htons(_port); //要被发送给对方的，即要发送到网络中！
        loacl.sin_addr.s_addr = INADDR_ANY;
      
        // 2.2 bind 设置如内核中
        int n = ::bind(_listen_sockfd, CONV(&amp;loacl), sizeof(loacl));
        if(n &lt; 0)
        {
            std::cout &lt;&lt; "bind: "&lt;&lt; errno &lt;&lt; " " &lt;&lt;strerror(errno) &lt;&lt; std::endl;
            exit(BIND_ERR);
        }
        std::cout &lt;&lt; "bind success" &lt;&lt; std::endl;

        // 3.设置为监听状态
        int sockfd = ::listen(_listen_sockfd, BACKLOG);
        if(sockfd &lt; 0)
        {
            std::cout &lt;&lt; "listen: " &lt;&lt; errno &lt;&lt; " " &lt;&lt; strerror(errno) &lt;&lt; std::endl;
            exit(LISTEN_ERR);
        }
        std::cout &lt;&lt; "listen success" &lt;&lt; std::endl;
    }</code></pre>

    <h4 id="2.1.3%20Start%E5%87%BD%E6%95%B0" name="2.1.3%20Start%E5%87%BD%E6%95%B0">
     2.1.3 Start函数
    </h4>
    <p>
     Start函数用于启动该服务。
    </p>
    <ul>
     <li>
      首先，将_isrunning变量设置为true，表示服务正在运行。
     </li>
     <li>
      接着，写上while循环。调用accept函数，为新连接创建专属的文件描述符，进行通信。accept函数中，第二个参数是可以传入sockadd_in结构体变量指针，使用CONV宏进行强转。可以获取新连接的IP地址和端口号等信息。
     </li>
     <li>
      如果accept函数调用成功，peer结构体变量中会填充客户端的信息。可以通过inet_ntop函数将peer中无符号整数IP地址转换成点分十进制的字符串形式。ntohs函数用于网络字节序列转换成主机字节序列。
     </li>
     <li>
      当连接成功后，我们可以直接调用HandleRquest函数处理请求，进行网络通信。但是单进程通信，只能处理一个连接。可以使用多进程、多线程和线程池。但是这些方案在处理大量请求时，还是不合适，需要使用多路转接技术。
     </li>
     <li>
      多线程版本，使用fork函数创建子进程，但是父进程需要调用waitpid函数阻塞等待子进程，再回收。所以，我们可以让子进程再创建一个子进程，称为孙子进程，然后子进程退出。那么孙子进程变成孤儿进程，由系统的init进程等待它的退出。而父进程不再需要等待子进程，可以继续去执行accept函数，为新连接创建文件描述符。
     </li>
     <li>
      多线程版本，是一有新连接就创建线程去处理。可以调用pthread库中的pthread_create函数创建线程，其中第三个参数使用lambada表达式，调用HandleRequest函数。最后使用pthreate_detach函数是新线程和主线程分离。
     </li>
    </ul>
    <pre><code class="language-cpp">    void Start()
    {
        _isrunning = true;
        while(true)
        {
            // 1.获取新连接
            struct sockaddr_in peer;
            socklen_t len = sizeof(peer); //必须设定

            std::cout &lt;&lt; "waiting for connection..." &lt;&lt; std::endl;
            int sockfd = accept(_listen_sockfd, CONV(&amp;peer), &amp;len);
            if(sockfd &lt; 0)
            {
                std::cout &lt;&lt; "accept: " &lt;&lt; errno &lt;&lt; " " &lt;&lt; strerror(errno) &lt;&lt; std::endl;
                continue;
            }
            std::cout &lt;&lt; "accept success, new sockfd is: " &lt;&lt; sockfd &lt;&lt; std::endl;

            // 2.获取客户端信息
            char ipbuffer[64];
            const char *ip = ::inet_ntop(AF_INET, &amp;peer.sin_addr, ipbuffer, sizeof(ipbuffer));
            std::string clientaddr = ip;
            clientaddr += ":";
            clientaddr += std::to_string(ntohs(peer.sin_port));
            std::cout &lt;&lt; "client addr is: " &lt;&lt; clientaddr &lt;&lt; std::endl;

            // 3.处理请求
            // version 0 单进程处理
            // HandleRequset(sockfd);

            //version 1 多进程模式 
            pid_t pid = fork();
            if(pid == 0)
            {
                ::close(_listen_sockfd); //子进程关闭监听套接字
                //
                if(fork() &gt; 0) exit(0); // 子进程退出，父进程不用等待

                // 父进程-&gt;子进程退出-&gt;孙子进程，
                // 孙子进程会变成孤儿进程，由操作系统等待
                HandleRequset(sockfd);
                exit(0); //子进程退出
            }

            int rid = waitpid(pid, NULL, 0);
            if(rid &lt; 0)
            {
                std::cout &lt;&lt; "waitpid: " &lt;&lt; errno &lt;&lt; " " &lt;&lt; strerror(errno) &lt;&lt; std::endl;
            }

            //version 2 多线程模式
            pthread_t tid;
            pthread_create(&amp;tid, NULL, [](void* arg){
                int sockfd = *(int*)arg;
                HandleRequset(sockfd);
            }, (void*)sockfd);
            pthread_detach(tid);
            
        }
        _isrunning = false;
    }
</code></pre>

    <h4 id="2.1.4%20HandlerRequest%E5%87%BD%E6%95%B0" name="2.1.4%20HandlerRequest%E5%87%BD%E6%95%B0">
     2.1.4 HandlerRequest函数
    </h4>
    <p>
     HandlerRequest函数用于处理与客户端的通信。我们协定传输的数据都是字符串。
    </p>
    <ul>
     <li>
      在开始时，我们定义一个inbuffer字符数组，用于存储从TCP连接中接收到的数据。由于TCP协议是基于字节流的，这意味着数据的传输是连续的，而且没有固定的边界。因此，我们需要使用recv函数来从连接中读取数据，并使用send函数来发送数据。
     </li>
     <li>
      由于TCP连接是全双工的，我们可以在同一个文件描述符上进行读写操作。在这个文件描述符下，有两个独立的缓冲区：一个用于接收数据，另一个用于发送数据。这允许我们在不中断数据流的情况下，同时处理输入和输出。
     </li>
     <li>
      面向字节流类似于读文本文件，我们一次读取的数据不固定，可以多也可以少，所以读取数据的时候可能会读不完整。如果想要保证数据的完整性，我们需要引入协议，就是做一种约定，比如一个报文数据开头是数据的长度，再跟\n接上有效数据，最后结尾再跟\n。这里暂时不对数据完整性进行考虑。
     </li>
     <li>
      recv函数的返回值跟read函数类似，返回值大于0，表明是数据的字节数大小。如果等于0，表示读到文件末尾，即客户端关闭。如果小于0，就是调用recv函数失败。
     </li>
    </ul>
    <pre><code class="language-cpp">    void HandleRequset(int sockfd)
    {
        char inbuffer[4096];
        while(true)
        {
            ssize_t n = ::recv(sockfd, inbuffer, sizeof(inbuffer) - 1, 0);
            if (n &gt; 0)
            {
                // 在后面字符串后面加上反斜杠0，以便于输出
                inbuffer[n] = 0;
                std::cout &lt;&lt; "sockfd-" &lt;&lt; sockfd &lt;&lt; ": " &lt;&lt; inbuffer &lt;&lt; std::endl;

                std::string echo_str = "echo# ";
                echo_str += inbuffer;
                ::send(sockfd, echo_str.c_str(), echo_str.size(), 0);
            }
            else if (n == 0)
            {
                std::cout &lt;&lt; "client closed" &lt;&lt; std::endl;
                break;
            }
            else
            {
                std::cout &lt;&lt; "recv: " &lt;&lt; errno &lt;&lt; " " &lt;&lt; strerror(errno) &lt;&lt; std::endl;
                break;
            }
        }
        // 一定要记得关闭文件描述符
        // 不然会造成文件描述符泄漏，也是内存泄漏！！！
        ::close(sockfd);  
    }</code></pre>

    <h3 id="2.4%20TcpClient.cc" name="2.4%20TcpClient.cc">
     2.4 TcpClient.cc
    </h3>
    <p>
     Tcp服务的客户端，命令行启动该程序时，需要传入访问的IP地址和端口号。
    </p>
    <ul>
     <li>
      首先创建套接字。再填充服务端的网络信息，调用connect函数进行连接。客户端不需要绑定端口号，操作系统会在客户端第一次发起连接时，随机分配一个端口号给客户端。
     </li>
     <li>
      因为Tcp协议是基于字节流的，所以可以使用write函数，直接向套接字绑定的文件描述符中写入数据，就会发送给服务端。读取数据可以使用read函数。
     </li>
     <li>
      我们只处理返回值大于0的情况，如果返回值不大于0，说明服务端有问题。
     </li>
    </ul>
    <pre><code class="language-cpp">#include &lt;iostream&gt;

    std::string serverip = argv[1];
    uint16_t serverport = std::stoi(argv[2]);

        std::cout &lt;&lt; "create socket error" &lt;&lt; std::endl;

    struct sockaddr_in server_addr;
    memset(&amp;server_addr, 0, sizeof(server_addr));

    server_addr.sin_port = htons(serverport);
    server_addr.sin_addr.s_addr = ::inet_addr(serverip.c_str());

            else

        }
    }

    </h2>
    <p>
     客户端发送指令，让服务端执行，并把执行结果返回给客户端。这其实就类似我们使用Xshell登录云服务器，然后使用命令行输入执行，并返回结果给我们。
    </p>
    <h3 id="3.1%20CommandExec.hpp" name="3.1%20CommandExec.hpp">
     3.1 CommandExec.hpp
    </h3>
    <p>
     下面是CommandExec.hpp代码内容。包含Command类，内部成员变量是一个set容器，专门存储可以执行的指令名字，其实也可以不使用白名单策略，但是得注意别乱输入指令。
    </p>
    <p>
     SafeCheck函数用来检查传进来的指令是否在_while_list中。
    </p>
    <p>
     popen函数C 语言标准库中的一个函数，用于执行一个命令并获取命令的输出或者向命令发送输入。它会创建一个管道，然后读写端打开。
    </p>
    <pre><code class="language-cpp">#pragma once

#include &lt;iostream&gt;
#include &lt;string&gt;
#include &lt;cstdio&gt;
#include &lt;set&gt;

const int line_size = 1024;

class Command
{     
public:
    Command()
    {
        _while_list.insert("ls");
        _while_list.insert("pwd");
        _while_list.insert("ls -l");
        _while_list.insert("ll");
        _while_list.insert("touch");
        _while_list.insert("whoami");
    }

    bool SafeCheck(std::string &amp;comstr)
    {
        auto iter = _while_list.find(comstr);
        return iter == _while_list.end() ? false : true;
    }

    // 给你一个命令字符串“ls -l”，让你执行，执行完后，把结果返回
    std::string Execute(std::string comstr)
    {
        if(!SafeCheck(comstr))
        {
            return std::string(cmd + "不支持");
        }

        FILE *fp = popen(comstr.c_str(), "r");
        if(nullptr == fp)
        {
            return std::string("Failed");
        }

        char buffer[line_size];
        std::string result;
        while(true)
        {
            char *ret = :: fgets(buffer, sizeof(buffer), fp);
            if(!ret) break;
            result += ret;
        }

        pclose(fp);
        return result.empty() ? std::string("Done") : result;
    }
private:
    std::set&lt;std::string&gt; _while_list;
};</code></pre>
