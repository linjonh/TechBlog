    <h2>
     <a id="_1">
     </a>
     分布式数据库体系结构
    </h2>
    <h3>
     <a id="_2">
     </a>
     全局外模式
    </h3>
    <p>
     全局外模式是全局应用的用户视图，是全局概念模式的子集，该层直接与用户（或应用程序）交互。
    </p>
    <h3>
     <a id="_4">
     </a>
     全局概念模式
    </h3>
    <p>
     全局概念模式定义分布式数据库中数据的整体逻辑结构，
     <strong>
      数据就如同根本没有分布一样
     </strong>
     ，可用传统的集中式数据库中所采用的方法进行定义。
    </p>
    <h3>
     <a id="_6">
     </a>
     分片模式
    </h3>
    <p>
     将一个关系模式分解称为几个数据片。
    </p>
    <h3>
     <a id="_8">
     </a>
     分布模式
    </h3>
    <p>
     分布式数据库的本质特性就是数据分布在不同的物理位置。
    </p>
    <p>
     分布模式的主要职责是定义数据片段（即分片模式的处理结果）的存放节点。
    </p>
    <h3>
     <a id="_12">
     </a>
     局部概念模式
    </h3>
    <p>
     局部概念模式是局部数据库的概念模式。
    </p>
    <h3>
     <a id="_14">
     </a>
     局部内模式
    </h3>
    <p>
     局部内模式是局部数据库的内模式。
    </p>
    <h2>
     <a id="_16">
     </a>
     分布式数据库特点
    </h2>
    <table>
     <thead>
      <tr>
       <th>
        特点
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        共享性
       </td>
       <td>
        不同的结点的数据共享。
       </td>
      </tr>
      <tr>
       <td>
        自治性
       </td>
       <td>
        每个结点对本地数据都能独立管理。
       </td>
      </tr>
      <tr>
       <td>
        可用性
       </td>
       <td>
        某一场地故障时，可以使用其他场地上的副本而不至于使整个系统瘫痪。
       </td>
      </tr>
      <tr>
       <td>
        分布性
       </td>
       <td>
        数据分布在不同场地上的存储。
       </td>
      </tr>
     </tbody>
    </table>
    <h2>
     <a id="_24">
     </a>
     分布透明性
    </h2>
    <p>
     包括：分片透明性、位置透明性和局部数据模型（逻辑）透明性。
    </p>
    <p>
     层次划分：
    </p>
    <ul>
     <li>
      分片透明性是最高层次。
     </li>
     <li>
      位置透明性是下一层次。
     </li>
     <li>
      局部数据模型（逻辑）透明性是最低层次。
     </li>
    </ul>
    <h3>
     <a id="_32">
     </a>
     分片透明性
    </h3>
    <p>
     分片透明性是指，用户或应用程序只对全局关系进行操作，而
     <strong>
      不必考虑数据的分片
     </strong>
     。
    </p>
    <h3>
     <a id="_34">
     </a>
     位置透明性
    </h3>
    <p>
     位置透明性是指，用户或应用程序应当了解分片情况，但
     <strong>
      不必了解片段的存储场地
     </strong>
     。
    </p>
    <h3>
     <a id="_36">
     </a>
     局部数据模型（逻辑）透明性
    </h3>
    <p>
     局部数据模型（逻辑）透明性是指，用户或应用程序应当了解分片及各片段的存储场地，但
     <strong>
      不必了解
     </strong>
     局部场地上使用的是
     <strong>
      何种数据模型
     </strong>
     。
    </p>
