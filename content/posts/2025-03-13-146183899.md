     <strong>
      1. 什么是 Node.js？（JavaScript 服务器环境）
     </strong>
    </h2>
    <h3>
     <a id="11_Nodejs__2">
     </a>
     <strong>
      1.1 Node.js 定义
     </strong>
    </h3>
    <p>
     <strong>
      Node.js
     </strong>
     是一个
     <strong>
      基于 Chrome V8 引擎的 JavaScript 运行环境
     </strong>
     ，用于运行 JavaScript 代码，特别适合
     <strong>
      服务器端开发
     </strong>
     。
    </p>
    <p>
     <strong>
      特点：
     </strong>
     <br/>
     ✅
     <strong>
      非阻塞（异步 I/O）
     </strong>
     → 处理高并发请求更高效。
     <br/>
     ✅
     <strong>
      事件驱动
     </strong>
     → 采用
     <strong>
      Event Loop
     </strong>
     机制，无需创建多个线程。
     <br/>
     ✅
     <strong>
      单线程
     </strong>
     → 适用于 I/O 密集型应用（如 Web 服务器、API）。
     <br/>
     ✅
     <strong>
      npm（Node Package Manager）
     </strong>
     → 内置
     <strong>
      npm
     </strong>
     ，可安装丰富的第三方库。
    </p>

     <strong>
      3. 什么是 API 路由？（定义 API 访问路径）
     </strong>
    </h2>
    <h3>
     <a id="31_API__53">
     </a>
     <strong>
      3.1 API 路由定义
     </strong>
    </h3>
    <p>
     <strong>
      API 路由
     </strong>
     指的是
     <strong>
      定义 URL 访问规则
     </strong>
     ，让前端或第三方系统可以正确访问 API。
    </p>
    <p>
     <strong>
      特点：
     </strong>
     <br/>
     ✅
     <strong>
      清晰的 URL 设计
     </strong>
     → 避免
     <code>
      /getUser
     </code>
     这种不规范的写法。
     <br/>
     ✅
     <strong>
      基于资源的 RESTful 路由
     </strong>
     →
     <code>
      /users/:id
     </code>
     获取特定用户数据。
     <br/>
     ✅
     <strong>
      参数化 URL
     </strong>
     → 例如
     <code>
      /users/:id/orders
     </code>
     获取某个用户的订单。
    </p>

    <h2>
     <a id="4__113">

     <strong>
      4. 什么是反向代理？（负载均衡和安全优化）
     </strong>
    </h2>
    <h3>
     <a id="41__114">
     </a>
     <strong>
      4.1 反向代理定义
     </strong>
    </h3>
    <p>
     <strong>
      反向代理（Reverse Proxy）
     </strong>
     是
     <strong>
      位于客户端和服务器之间的代理服务器
     </strong>
     ，用于
     <strong>
      提高性能、安全性、负载均衡
     </strong>
     。
    </p>
    <p>
     <strong>
      特点：
     </strong>
     <br/>
     ✅
     <strong>
      隐藏后端服务器
     </strong>
     → 提供统一入口，保护真实服务器 IP。
     <br/>
     ✅
     <strong>
      负载均衡
     </strong>
     → 把请求分配到不同服务器，减少单台服务器压力。
     <br/>
     ✅
     <strong>
      SSL 终止
     </strong>
     → 代理服务器处理 HTTPS，减少应用服务器计算压力。
    </p>
    <hr/>
    <h3>
     <a id="42__124">
     </a>
     <strong>
      4.2 企业中的反向代理应用
     </strong>
    </h3>
    <h4>
     <a id="__4Nginx__125">
     </a>
     <strong>
      📌 案例 4：Nginx 负载均衡
     </strong>
    </h4>
    <p>
     <strong>
      场景
     </strong>
     ：企业需要高并发 API 服务器，防止单点故障。
     <br/>
     <strong>
      解决方案
     </strong>
     ：使用
     <strong>
      Nginx 作为反向代理，分发请求到多个服务器
     </strong>
     。
    </p>
    <p>
     <strong>
      Nginx 配置：
     </strong>
    </p>
    <pre><code class="prism language-nginx">server {
    listen 80;
    location /api/ {
        proxy_pass http://backend_servers;
    }
}

upstream backend_servers {
    server backend1.example.com;
    server backend2.example.com;
}
</code></pre>
    <p>
     ✅
     <strong>
      企业价值
     </strong>
     ：分流流量，提高系统稳定性，防止服务器崩溃。
    </p>
    <hr/>
    <h4>
     <a id="__147">
     </a>
     <strong>
      📌 讲义示例
     </strong>
    </h4>
    <pre><code>ifn666.com {
  handle /locallib/* {
    reverse_proxy localhost:3000
  }
  handle /carhire/* {
    reverse_proxy localhost:3005
  }
}
</code></pre>
    <p>
     💡
     <strong>
      示例展示了如何使用反向代理分发流量，提升后端 API 服务器性能。
     </strong>
    </p>
