    <h2>
     <a id="_5">
     </a>
     一、概念
    </h2>
    <p>
     二叉搜索树又称二叉排序树，它或者是一棵空树，或者是具有以下性质的二叉树
    </p>
    <ul>
     <li>
      若它的左子树不为空，则左子树上所有节点的值都小于根节点的值
     </li>
     <li>
      若它的右子树不为空，则右子树上所有节点的值都大于根节点的值
     </li>
     <li>
      它的左右子树也分别为二叉搜索树
     </li>
    </ul>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/0b5771b01ce8af44a2164f175851a1d4.png"/>
    </p>
    <p>
     从二又搜索树的定义可知，它的前提是二叉树，并且采用了
     <strong>
      递归的方式
     </strong>
     进行定义，它的结点间满足一个偏序关系，左子树根结点的值定比父结点小，右子树根结点的值一定比父结点大。
    </p>
    <p>
     正如它的名字所说，构造这样一棵树的目的是为了提高搜索的速度，如果对二叉搜索树进行
     <strong>
      中序遍历
     </strong>
     ，我们可以发现，得到的序列是个递增序列。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/95241f12dde660f08eedf0e7dd63f44b.png"/>
    </p>
    <blockquote>
     <p>
      <strong>
       递归：
      </strong>
     </p>
     <h5 id="LN6Ch">
      (1) 左右子树本身是二叉搜索树
     </h5>
     无论从哪个节点开始，只需考察：
     <ul>
      <li>
       当前节点的值是否满足与左子树和右子树的比较关系；
      </li>
      <li>
       然后递归到左右子树去判断它们是否分别满足二叉搜索树的性质。
      </li>
     </ul>
     <p>
      这种对左右子树重复性质的要求是递归定义的直接体现。
     </p>
     <h5 id="TSqxb">
      (2) 以根节点为核心的分层定义
     </h5>
     + 当前树的性质由当前根节点与左右子树的性质共同决定； + 左子树和右子树本身可以被视为规模更小的二叉搜索树，这种嵌套结构直接表明递归定义的存在。
    </blockquote>

    <h3>
     <a id="BSTree__default_93">
     </a>
     强制生成默认构造BSTree() = default
    </h3>
    <p>
     在上面完整的代码中，
     <code>
      BSTree()
     </code>
     默认构造函数是通过以下代码定义的：
    </p>
    <pre><code class="prism language-cpp"><span class="token function">BSTree</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token operator">=</span> <span class="token keyword">default</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     <code>
      = default
     </code>
     的作用
    </p>
    <ul>
     <li>
      生成默认构造函数：
      <ul>
       <li>
        当我们写
        <code>
         BSTree() = default;
        </code>
        时，编译器会为类自动生成一个默认的构造函数。
       </li>
       <li>
        它的行为等价于一个空的用户定义构造函数，但由编译器自动完成。
       </li>
      </ul>
     </li>
     <li>
      为什么用
      <code>
       = default
      </code>
      ：
      <ul>
       <li>
        显式告诉编译器生成默认构造函数，表示我们明确需要这个默认的构造行为。
       </li>
       <li>
        避免用户误以为我们忘记定义构造函数。
       </li>
       <li>
        保持代码简单，同时遵守编译器生成的高效实现。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <p>
     为什么需要显式生成默认构造函数？
    </p>
    <p>
     <code>
      BSTree
     </code>
     类包含一个私有成员变量
     <code>
      _root
     </code>
     ：
    </p>
    <pre><code class="prism language-cpp">Node<span class="token operator">*</span> _root <span class="token operator">=</span> <span class="token keyword">nullptr</span><span class="token punctuation">;</span>
</code></pre>
    <ul>
     <li>
      原因 1: 明确默认构造需求
      <br/>
      如果没有声明任何构造函数，编译器会自动生成默认构造函数，且会将
      <code>
       _root
      </code>
      初始化为
      <code>
       nullptr
      </code>
      （因为你用了类内初始化
      <code>
       Node* _root = nullptr;
      </code>
      ）。但如果你显式定义了其他构造函数（比如拷贝构造、赋值重载），编译器将不再自动生成默认构造函数。为避免丢失默认构造函数的功能，就需要显式使用
      <code>
       = default;
      </code>
      。
     </li>
     <li>
      原因 2: 保持代码的可读性与语义
      <br/>
      通过显式声明
      <code>
       BSTree() = default;
      </code>
      ，代码更直观地表明：
      <br/>
      <em>
       “这个类的默认构造行为是由编译器生成的，我没有进行额外的修改。”
      </em>
     </li>
    </ul>
    <hr/>
    <p>
     <code>
      = default
     </code>
     和手动定义的区别
    </p>
    <p>
     如果不使用
     <code>
      = default
     </code>
     而是手动定义默认构造函数：
    </p>
    <pre><code class="prism language-cpp"><span class="token function">BSTree</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token operator">:</span> 
    <span class="token function">_root</span><span class="token punctuation">(</span><span class="token keyword">nullptr</span><span class="token punctuation">)</span> 
    <span class="token punctuation">{<!-- --></span><span class="token punctuation">}</span>
</code></pre>
    <p>
     行为与
     <code>
      = default
     </code>
     等价，但显式手动定义不如
     <code>
      = default
     </code>
     简洁。
    </p>
    <p>
     <code>
      = default
     </code>
     的好处包括：
    </p>
    <ol>
     <li>
      让编译器自动生成，减少代码量和维护负担。
     </li>
     <li>
      避免意外错误，例如忘记初始化成员变量。
     </li>
    </ol>
    <hr/>
    <p>
     适用场景
    </p>
    <p>
     使用
     <code>
      = default
     </code>
     常见于以下场景：
    </p>
    <ol>
     <li>
      需要默认构造函数，但有其他特殊构造函数：
      <br/>
      如果定义了其他构造函数（如拷贝构造、移动构造等），而仍需要默认构造函数，必须显式声明它。
     </li>
     <li>
      需要提高代码语义的清晰度：
      <br/>
      即使编译器会自动生成默认构造函数，显式
      <code>
       = default
      </code>
      更直观。
     </li>
    </ol>

       BSTree&lt;K&gt; t
      </code>
      ，因此会调用
      <strong>
       复制构造函数
      </strong>
      ，为
      <code>
       t
      </code>
      创建一个
      <code>
       tree2
      </code>
      的副本。
     </li>
     <li>
      在此过程中，
      <code>
       t._root
      </code>
      指向了
      <code>
       tree2
      </code>
      的根节点（即
      <code>
       10
      </code>
      ），但
      <code>
       t
      </code>
      是一个独立的对象，内存完全独立于
      <code>
       tree2
      </code>
      。
     </li>
    </ul>
    <p>
     变量状态：
    </p>
    <ul>
     <li>
      <code>
       this
      </code>
      ：指向
      <code>
       tree1
      </code>
      ，
      <code>
       this-&gt;_root = 5
      </code>
      。
     </li>
     <li>
      <code>
       t._root
      </code>
      ：指向
      <code>
       tree2
      </code>
      的副本，即根节点为
      <code>
       10
      </code>
      。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      第二步：执行
      <code>
       swap
      </code>
     </strong>
    </p>
    <pre><code class="prism language-cpp"><span class="token function">swap</span><span class="token punctuation">(</span>_root<span class="token punctuation">,</span> t<span class="token punctuation">.</span>_root<span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     这里使用的是标准库中的
     <code>
      std::swap
     </code>
     函数。该函数的核心是交换两个对象的值，具体步骤如下：
    </p>
    <ol>
     <li>
      创建临时变量
      <code>
       temp
      </code>
      ：
      <code>
       temp
      </code>
      保存
      <code>
       this-&gt;_root
      </code>
      （即
      <code>
       tree1
      </code>
      的
      <code>
       _root
      </code>
      ），此时
      <code>
       temp = 5
      </code>
      。
     </li>
     <li>
      将
      <strong>
       <code>
        t._root
       </code>
      </strong>
      赋给
      <code>
       this-&gt;_root
      </code>
      ：此时
      <code>
       this-&gt;_root
      </code>
      指向
      <code>
       t._root
      </code>
      的值（即
      <code>
       10
      </code>
      ），也就是
      <code>
       tree2
      </code>
      的根节点。
     </li>
     <li>
      将
      <strong>
       <code>
        temp
       </code>
      </strong>
      赋给
      <code>
       t._root
      </code>
      ：最后，
      <code>
       t._root
      </code>
      变为
      <code>
       5
      </code>
      ，即之前
      <code>
       tree1
      </code>
      的根节点。
     </li>
    </ol>
    <p>
     <code>
      swap
     </code>
     操作结束后，
     <code>
      tree1
     </code>
     和
     <code>
      t
     </code>
     的根节点已经交换。
    </p>
    <p>
     变量状态：
    </p>
    <ul>
     <li>
      <code>
       this-&gt;_root
      </code>
      ：现在指向
      <code>
       10
      </code>
      ，即
      <code>
       tree2
      </code>
      的根节点。
     </li>
     <li>
      <code>
       t._root
      </code>
      ：现在指向
      <code>
       5
      </code>
      ，即
      <code>
       tree1
      </code>
      原来的根节点。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      第三步：返回当前对象（链式调用支持）
     </strong>
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">return</span> <span class="token operator">*</span><span class="token keyword">this</span><span class="token punctuation">;</span>
</code></pre>
    <ul>
     <li>
      <code>
       operator=
      </code>
      函数返回当前对象，即
      <code>
       tree1
      </code>
      ，此时
      <code>
       tree1._root = 10
      </code>
      。
     </li>
     <li>
      <code>
       t
      </code>
      是传递进来的临时变量，会在函数结束时自动销毁。由于
      <code>
       t._root
      </code>
      现在指向的是
      <code>
       5
      </code>
      （即
      <code>
       tree1
      </code>
      原来的根节点），当
      <code>
       t
      </code>
      离开作用域时，其析构函数会被调用，自动释放
      <code>
       5
      </code>
      这个节点及其子树（如果有）。
     </li>
    </ul>
    <p>
     变量状态：
    </p>
    <ul>
     <li>
      <code>
       this-&gt;_root
      </code>
      ：保持指向
      <code>
       10
      </code>
      ，即
      <code>
       tree2
      </code>
      的根节点。
     </li>
     <li>
      <code>
       t
      </code>
      ：将在函数结束时销毁，
      <code>
       t._root
      </code>
      指向
      <code>
       5
      </code>
      的那部分内存会被释放。
     </li>
    </ul>
    <hr/>
    <p>
     最
     <strong>
      终结果
     </strong>
    </p>
    <ul>
     <li>
      <code>
       tree1
      </code>
      ：现在拥有
      <code>
       tree2
      </code>
      的树结构，根节点为
      <code>
       10
      </code>
      。
     </li>
     <li>
      <code>
       tree2
      </code>
      ：原来的结构不变，仍然指向
      <code>
       10
      </code>
      ，但由于我们是通过副本
      <code>
       t
      </code>
      操作，
      <code>
       tree2
      </code>
      本身不会受到影响。
     </li>
     <li>
      <strong>
       旧的
       <strong>
        <code>
         tree1
        </code>
       </strong>
       的树（即以
       <strong>
        <code>
         5
        </code>
       </strong>
       为根的树）
      </strong>
      ：在
      <code>
       t
      </code>
      的生命周期结束时，随着
      <code>
       t
      </code>
      的析构函数被调用，旧的
      <code>
       tree1
      </code>
      树被销毁。
     </li>
    </ul>
    <hr/>
    <p>
     关键点总结：
    </p>
    <ol>
     <li>
      <strong>
       临时副本
      </strong>
      ：通过传值的方式，将
      <code>
       tree2
      </code>
      的副本传递给
      <code>
       t
      </code>
      ，
      <code>
       t
      </code>
      独立于
      <code>
       tree2
      </code>
      。
     </li>
     <li>
      <strong>
       高效的
       <code>
        swap
       </code>
      </strong>
      ：
      <code>
       swap
      </code>
      交换了
      <code>
       tree1
      </code>
      和
      <code>
       t
      </code>
      的根节点指针，而不是深拷贝整个树，效率更高。
     </li>
     <li>
      <strong>
       资源管理
      </strong>
      ：由于
      <code>
       t
      </code>
      是传入的临时变量，赋值过程结束后
      <code>
       t
      </code>
      会被自动销毁，释放旧的
      <code>
       tree1
      </code>
      的内存，避免了手动
      <code>
       Destroy
      </code>
      的麻烦。
     </li>
     <li>
      <strong>
       链式赋值
      </strong>
      ：
      <code>
       return *this
      </code>
      支持链式调用，如
      <code>
       tree1 = tree2 = tree3;
      </code>
      。
     </li>
    </ol>
    <p>
     这种
     <code>
      swap
     </code>
     的写法，简洁、高效，并且通过临时对象
     <code>
      t
     </code>
     自动管理资源，是现代 C++ 中推荐的写法。
    </p>
    <h3>
     <a id="return_this_302">
     </a>
     传统深拷贝写法中为什么需要return *this?
    </h3>
    <p>
     <strong>
      为什么返回
      <strong>
       <code>
        *this
       </code>
      </strong>
      ？即使
      <strong>
       <code>
        tree1
       </code>
      </strong>
      的内存已被释放？
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       释放的只是原树的节点，树本身仍然存在
      </strong>
     </li>
    </ol>
    <p>
     调用
     <code>
      Destory(_root)
     </code>
     时，仅释放了
     <code>
      tree1
     </code>
     原有的节点内存，但
     <code>
      tree1
     </code>
     本身（即当前对象）仍然存在。
     <code>
      _root
     </code>
     只是一个指针成员变量，而非整个
     <code>
      tree1
     </code>
     对象的内存。
    </p>
    <p>
     例如：
    </p>
    <ul>
     <li>
      如果
      <code>
       tree1
      </code>
      原本有数据，它们会被
      <code>
       Destory
      </code>
      清空，变成一个空树（
      <code>
       _root == nullptr
      </code>
      ）。
     </li>
     <li>
      然后通过
      <code>
       Copy(t._root)
      </code>
      ，重新构造
      <code>
       tree1
      </code>
      的数据结构，使其变得与
      <code>
       tree2
      </code>
      相同。
     </li>
    </ul>
    <p>
     最终，
     <code>
      tree1
     </code>
     并没有被销毁，而是被重建。
    </p>
    <hr/>
    <ol start="2">
     <li>
      <strong>
       返回值的意义：支持链式赋值
      </strong>
     </li>
    </ol>
    <p>
     返回
     <code>
      *this
     </code>
     是赋值运算符的标准设计，用于支持链式赋值。链式赋值允许连续的赋值操作，如：
    </p>
    <pre><code class="prism language-cpp">BSTree<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> a<span class="token punctuation">,</span> b<span class="token punctuation">,</span> c<span class="token punctuation">;</span>
a <span class="token operator">=</span> b <span class="token operator">=</span> c<span class="token punctuation">;</span>
</code></pre>
    <ul>
     <li>
      <strong>
       流程
      </strong>
      ：
      <ol>
       <li>
        <code>
         b = c
        </code>
        执行后，返回
        <code>
         b
        </code>
        的引用。
       </li>
       <li>
        然后将
        <code>
         b
        </code>
        的引用作为参数传递给
        <code>
         a = b
        </code>
        。
       </li>
      </ol>
     </li>
    </ul>
    <p>
     如果不返回
     <code>
      *this
     </code>
     ，链式赋值将无法工作。
    </p>
    <hr/>
    <ol start="3">
     <li>
      <strong>
       为什么不是返回销毁的内容？
      </strong>
     </li>
    </ol>
    <p>
     <code>
      Destory(_root)
     </code>
     只是清空了树的原有数据，
     <code>
      tree1
     </code>
     的对象本身还存在。返回
     <code>
      *this
     </code>
     是返回当前对象的引用，表示赋值操作完成，当前对象的状态被更新为目标对象的状态。
    </p>
    <p>
     例如：
    </p>
    <pre><code class="prism language-cpp">BSTree<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span> tree1<span class="token punctuation">,</span> tree2<span class="token punctuation">;</span>
<span class="token comment">// 假设 tree1 有旧的节点，tree2 是目标树</span>
tree1 <span class="token operator">=</span> tree2<span class="token punctuation">;</span>
</code></pre>
    <p>
     在
     <code>
      tree1 = tree2
     </code>
     过程中：
    </p>
    <ol>
     <li>
      清空
      <code>
       tree1
      </code>
      的原节点，但
      <code>
       tree1
      </code>
      本身还在。
     </li>
     <li>
      将
      <code>
       tree2
      </code>
      的数据复制到
      <code>
       tree1
      </code>
      。
     </li>
     <li>
      返回
      <code>
       tree1
      </code>
      的引用，表示
      <code>
       tree1
      </code>
      现在是
      <code>
       tree2
      </code>
      的深拷贝。
     </li>
    </ol>
    <hr/>
    <ol start="4">
     <li>
      <strong>
       示意图说明
      </strong>
     </li>
    </ol>
    <p>
     <strong>

    </p>

    <h3>
     <a id="InOrder_419">
     </a>
     InOrder()为何这样设计？
    </h3>
    <p>
     <code>
      InOrder()
     </code>
     函数设计的目的是为了
     <strong>
      封装实现细节
     </strong>
     ，让用户只需调用接口函数即可完成对二叉树的中序遍历，而不必了解树的内部数据结构。这种设计的优势在于以下几点：
    </p>
    <ol>
     <li>
      <strong>
       隐藏内部实现细节
      </strong>
     </li>
    </ol>
    <p>
     在代码中，
     <code>
      InOrder()
     </code>
     调用了一个私有的辅助递归函数
     <code>
      _InOrder(Node* root)
     </code>
     ：
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">void</span> <span class="token function">InOrder</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token function">_InOrder</span><span class="token punctuation">(</span>_root<span class="token punctuation">)</span><span class="token punctuation">;</span>
    cout <span class="token operator">&lt;&lt;</span> endl<span class="token punctuation">;</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     用户只需要调用
     <code>
      tree.InOrder()
     </code>
     就可以触发中序遍历，输出节点值，而不需要知道：
    </p>
    <ul>
     <li>
      树的节点结构 (
      <code>
       BSTreeNode&lt;K&gt;
      </code>
      的定义)。
     </li>
     <li>
      遍历的逻辑细节。
     </li>
     <li>
      遍历需要从根节点
      <code>
       _root
      </code>
      开始。
     </li>
    </ul>
    <p>
     这种封装遵循了
     <strong>
      面向对象编程中的封装原则
     </strong>
     ，屏蔽了用户不需要知道的复杂实现。
    </p>
    <hr/>
    <ol start="2">
     <li>
      <strong>
       简化用户操作
      </strong>
     </li>
    </ol>
    <p>
     对于用户来说，他们关心的是功能，而非具体实现。用户只需要：
    </p>
    <pre><code class="prism language-cpp">tree<span class="token punctuation">.</span><span class="token function">InOrder</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     而不需要自己写出复杂的递归逻辑去遍历树，例如：
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">void</span> <span class="token function">InOrderManual</span><span class="token punctuation">(</span>BSTreeNode<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span><span class="token operator">*</span> root<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

    <p>
     封装之后，用户无需处理树的节点指针，也不用理解递归过程，降低了使用门槛。
    </p>
    <hr/>
    <ol start="3">
     <li>
      <strong>
       降低错误概率
      </strong>
     </li>
    </ol>
    <p>
     让用户直接访问和操作树的内部结构（如
     <code>
      _root
     </code>
     指针）可能导致以下问题：
    </p>
    <ul>
     <li>
      <strong>
       不小心破坏了树的完整性
      </strong>
      （如误修改了
      <code>
       _root
      </code>
      或其他节点的子指针）。
     </li>
     <li>
      <strong>
       错误遍历逻辑
      </strong>
      ：用户可能误用树的节点指针，造成逻辑混乱。
     </li>
    </ul>
    <p>
     通过只暴露
     <code>
      InOrder()
     </code>
     方法，用户无法直接接触内部的
     <code>
      _root
     </code>
     或树节点，减少了误操作的风险。
    </p>
    <hr/>
    <ol start="4">
     <li>
      <strong>
       增强代码的可维护性和扩展性
      </strong>
     </li>
    </ol>
    <p>
     如果以后需要修改中序遍历的实现（例如加入线程或并行化），我们只需改动私有的
     <code>
      _InOrder()
     </code>
     方法，而无需修改
     <code>
      InOrder()
     </code>
     或影响用户代码。
    </p>
    <p>
     用户调用
     <code>
      InOrder()
     </code>
     的方式保持不变，代码改动对外透明。这种设计符合
     <strong>
      模块化设计
     </strong>
     的思想。
    </p>

    <h3 id="IeyYn">
     2.插入
    </h3>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/img_convert/e6b35375c4c3eec612b04bce6443f2b8.png"/>
    </p>
    <ol>
     <li>
      特殊情况处理：
      <br/>
      如果树是空的（即 root == nullptr），那么直接把要插入的元素作为根节点插入即可。
     </li>
     <li>
      基本规则：
      <br/>
      ○ 因为是二叉搜索树，
      <strong>
       所以不能插入重复元素
      </strong>
      。
      <br/>
      ○ 插入的元素一定会成为某个叶子节点（即没有子节点的位置）。
     </li>
     <li>
      具体操作思路：
      <br/>
      ○ 从根节点开始定义一个指针变量 cur 来遍历树，初始时 cur = root。
      <br/>
      ○ 如果插入的值比当前节点的值小，就向左子树移动；如果插入的值比当前节点的值大，就向右子树移动。
      <br/>
      ○ 在移动时，还需要定义另一个变量 parent 记录当前节点的“父节点”，也就是 cur 移动前的位置。这样，当找到一个空位时，知道应该插入在 parent 的左子树还是右子树。
     </li>
     <li>
      最后一步：
      <br/>
      ○ 当 cur 移动到空位置（即 cur == nullptr）时，根据 parent 的位置决定插入新元素：
      <br/>
      ■ 如果插入的值小于 parent 的值，插入到 parent 的左子树；
      <br/>
      ■ 如果插入的值大于 parent 的值，插入到 parent 的右子树。
      <br/>
      这样就完成了插入操作。
     </li>
    </ol>

     四、删除
    </h2>
    <p>
     首先查找元素是否在二叉搜索树中，如果不存在，则返回, 否则要删除的结点可能分下面四种情
     <br/>
     况：
     <br/>
     a. 要删除的结点无孩子结点
     <br/>
     b. 要删除的结点只有左孩子结点
     <br/>
     c. 要删除的结点只有右孩子结点
     <br/>
     d. 要删除的结点有左、右孩子结点
     <br/>
     看起来有待删除节点有4中情况，实际情况a可以与情况b或者c合并起来，因此真正的删除过程
     <br/>
     如下：
     <br/>
     情况b：删除该结点且使被删除节点的双亲结点指向被删除节点的左孩子结点–直接删除
     <br/>
     情况c：删除该结点且使被删除节点的双亲结点指向被删除结点的右孩子结点–直接删除
     <br/>
     情况d：在它的右子树中寻找中序下的第一个结点(关键码最小)，用它的值填补到被删除节点
     <br/>
     中，再来处理该结点的删除问题–替换法删除

    <h2>
     <a id="KV_806">
     </a>
     六、应用——KV模型
    </h2>
    <ol>
     <li>
      <strong>
       K模型
      </strong>
      ：K模型即只有key作为关键码，结构中只需要存储Key即可，关键码即为需要搜索到
      <br/>
      的值。
      <br/>
      比如：给一个单词word，判断该单词是否拼写正确，具体方式如下：
      <br/>
      以词库中所有单词集合中的每个单词作为key，构建一棵二叉搜索树
      <br/>
      在二叉搜索树中检索该单词是否存在，存在则拼写正确，不存在则拼写错误。
     </li>
     <li>
      <strong>
       KV模型
      </strong>
      ：每一个关键码key，都有与之对应的值Value，即&lt;Key, Value&gt;的键值对。该种方
      <br/>
      式在现实生活中非常常见：
      <br/>
      比如英汉词典就是英文与中文的对应关系，通过英文可以快速找到与其对应的中文，英
      <br/>
      文单词与其对应的中文&lt;word, chinese&gt;就构成一种键值对；
      <br/>
      再比如统计单词次数，统计成功后，给定单词就可快速找到其出现的次数，单词与其出
      <br/>
      现次数就是&lt;word, count&gt;就构成一种键值对。
     </li>

     八、代码中重难点
    </h2>
    <h3>
     <a id="template_class_K_1558">
     </a>
     为什么有两处template &lt;class K
    </h3>
    <p>
     在上面的代码中，出现两处
     <code>
      template &lt;class K&gt;
     </code>
     是因为它们的作用域和含义不同：
    </p>
    <ol>
     <li>
      第一处
      <code>
       template &lt;class K&gt;
      </code>
     </li>
    </ol>
    <pre><code class="prism language-cpp"><span class="token keyword">template</span> <span class="token operator">&lt;</span><span class="token keyword">class</span> <span class="token class-name">K</span><span class="token operator">&gt;</span>
<span class="token keyword">struct</span> <span class="token class-name">BSTreeNode</span>
</code></pre>
    <p>
     这一部分定义了一个模板结构体
     <code>
      BSTreeNode
     </code>
     ，它接受一个模板参数
     <code>
      K
     </code>
     ，用来指定二叉搜索树节点的关键字类型。模板参数
     <code>
      K
     </code>
     是用来描述节点内部的
     <code>
      _key
     </code>
     数据成员类型。
    </p>
    <ol start="2">
     <li>
      第二处
      <code>
       template &lt;class K&gt;
      </code>
     </li>
    </ol>
    <pre><code class="prism language-cpp"><span class="token keyword">template</span> <span class="token operator">&lt;</span><span class="token keyword">class</span> <span class="token class-name">K</span><span class="token operator">&gt;</span>
<span class="token keyword">class</span> <span class="token class-name">BSTree</span>
</code></pre>
    <p>
     这里定义了一个模板类
     <code>
      BSTree
     </code>
     ，它也接受一个模板参数
     <code>
      K
     </code>
     ，用来指定树中的节点关键字类型。
     <code>
      BSTree
     </code>
     内部的
     <code>
      Node
     </code>
     类型被定义为
     <code>
      BSTreeNode&lt;K&gt;
     </code>
     ，即二叉树节点的模板参数类型与
     <code>
      BSTree
     </code>
     的模板参数一致。
    </p>
    <p>
     <strong>
      为什么需要两次
      <code>
       template &lt;class K&gt;
      </code>
      ？
     </strong>
    </p>
    <p>
     这是因为
     <strong>
      模板定义的作用域是独立的
     </strong>
     ，模板参数的声明只能在其所属的模板中生效。也就是说：
    </p>
    <ul>
     <li>
      <code>
       BSTreeNode&lt;K&gt;
      </code>
      的
      <code>
       K
      </code>
      和
      <code>
       BSTree&lt;K&gt;
      </code>
      的
      <code>
       K
      </code>
      是两个不同的模板参数，尽管它们名字相同，但彼此独立。
     </li>
     <li>
      在
      <code>
       BSTree
      </code>
      中使用
      <code>
       BSTreeNode&lt;K&gt;
      </code>
      时，需要显式传递模板参数
      <code>
       K
      </code>
      ，以表明
      <code>
       BSTreeNode
      </code>
      的模板实例化。
     </li>
    </ul>
    <p>
     如果模板类
     <code>
      BSTree
     </code>
     不单独定义自己的模板参数，而直接使用
     <code>
      BSTreeNode&lt;K&gt;
     </code>
     的模板参数，会导致代码难以扩展和维护。
    </p>
    <h3>
     <a id="boolstatue_1589">
     </a>
     bool和statue的区别
    </h3>
    <p>
     <code>
      bool
     </code>
     和
     <code>
      status
     </code>
     的区别主要取决于上下文中的定义和使用方式。以下是它们之间的一些典型区别：
    </p>
    <hr/>
    <p>
     <strong>
      1.
      <strong>
       <code>
        bool
       </code>
      </strong>
      是一种数据类型
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       描述
      </strong>
      :
      <code>
       bool
      </code>
      是一种内置的布尔类型，用来表示逻辑上的真假值，取值范围是
      <code>
       true
      </code>
      或
      <code>
       false
      </code>
      。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 用于条件判断、布尔表达式、逻辑运算等。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      :
      <ul>
       <li>
        判断条件是否成立。
       </li>
       <li>
        返回是否成功的标志，例如函数的执行结果。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      示例
     </strong>
     :
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">bool</span> isReady <span class="token operator">=</span> <span class="token boolean">true</span><span class="token punctuation">;</span>  <span class="token comment">// 表示某个条件成立</span>
<span class="token keyword">if</span> <span class="token punctuation">(</span>isReady<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 做某些事情</span>
<span class="token punctuation">}</span>
</code></pre>
    <hr/>
    <p>
     <strong>
      2.
      <strong>
       <code>
        status
       </code>
      </strong>
      是一种抽象的状态表示
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       描述
      </strong>
      :
      <code>
       status
      </code>
      一般用于表示系统、进程、任务、操作等的状态，通常不局限于
      <code>
       true
      </code>
      和
      <code>
       false
      </code>
      两种值。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 表示多种状态（例如成功、失败、处理中、错误等），通常会被定义为枚举或特定类型。
     </li>
     <li>
      <strong>
       常见场景
      </strong>
      :
      <ul>
       <li>
        描述复杂的操作结果或状态。
       </li>
       <li>
        提供更多上下文信息。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      示例
     </strong>

     <code>
