    <h2>
     一. 什么是红黑树？
    </h2>
    <p>
     红黑树也是一种二叉搜索树，红黑树中的节点有红色与黑色，通过控制节点的颜色来保证
     <strong>
      最长路径节点数不会超过最短路径节点数的2倍。
     </strong>
     从而保证该二叉搜索树是
     <strong>
      非常接近
     </strong>
     <strong>
      平衡的。
     </strong>
    </p>
    <h3>
     <strong>
      4.1 红黑树的性质
     </strong>
    </h3>
    <blockquote>
     <p>
      <strong>
       1 根节点是黑色的
      </strong>
     </p>
     <p>
      <strong>
       2 每一条路径上的黑色节点数量相同
      </strong>
     </p>
     <p>
      <strong>
       3 红色节点的孩子只能为黑色节点（无连续的红色节点）
      </strong>
     </p>
     <p>
      <strong>
       4 叶子节点是黑色的（这里的叶子指的是最后的空节点）
      </strong>
     </p>
    </blockquote>
    <p>
     通过上面的性质可以总结出来：
    </p>
    <p>
     <strong>
      每条路径黑色节点数量相同
     </strong>
    </p>
    <p>
     <strong>
      最短路径为：全部是黑色节点
     </strong>
     <strong>
     </strong>
     ​​​​​​​        ​​​​​​​        ​​​​​​​
    </p>
    <p>
     <strong>
      最长路径为：一红一黑
     </strong>
    </p>
    <p>
     <strong>
     </strong>
     这样就能保证最长路径不会最短路径的二倍，从而保证红黑树是近似平衡的。
    </p>
    <p>
     <img alt="" height="439" src="https://i-blog.csdnimg.cn/direct/1c8432bca12a4e67a5da1f9ae752c731.png" width="817"/>
    </p>
    <h2>
     二. 有AVL树为何还要红黑树？
    </h2>
    <p>
     AVL树通过平衡因子和旋转操作来保证整棵树是
     <strong>
      高度平衡的，可以解决二叉搜索树退化为链表的问题。保证其增删查改效率为O(log N)。
     </strong>
    </p>
    <p>
     而红黑树通过控制节点颜色来保证整棵树是近
     <strong>
      似平衡的（最长路径不超过最短路径二倍），能够保证其增删查改的效率为O(log N) ~ 2*O(log N)
     </strong>
    </p>
    <blockquote>
     <p>
      既然如此，为何还要红黑树呢？
     </p>
     <p>
      <strong>
       1 红黑树的平均效率是比AVL较低，但是现代计算机CPU效率非常高，logN和2 *log N的差异不是很大
      </strong>
     </p>
     <p>
      <strong>
       2 AVL树严格的平衡是需要不断的旋转操作得到的，旋转操作是比较耗时的。而红黑树的旋转操作比AVL树少得多。
      </strong>
     </p>
     <p>
      <strong>
      </strong>
      所以在很多情况下，
      <strong>
       红黑树的效率更高，综合性能更好，且实现简单。
      </strong>
     </p>
    </blockquote>
    <h2 style="background-color:transparent">
     三. 红黑树的代码框架
    </h2>
    <h3>
     3.1 红黑树的节点
    </h3>
    <p>
     红黑树节点只需增加一个表示颜色的变量即可
    </p>
    <pre><code class="language-cpp">enum  Color
{
	RED,
	Block
};

template&lt;class K,class V&gt;
struct rbTreeNode
{
	rbTreeNode&lt;K, V&gt;* _left;
	rbTreeNode&lt;K, V&gt;* _right;
	rbTreeNode&lt;K, V&gt;* _parent;

	pair&lt;K, V&gt; _kv;
	Color col;

	rbTreeNode(const pair&lt;K, V&gt;&amp; _kv = pair&lt;K, V&gt;())
		:_left(nullptr), _right(nullptr), _parent(nullptr), _kv(kv), _col(RED)
	{}
};

</code></pre>
    <p>
     这里默认将节点设置为红色，因为红色节点更温和，插入新节点容易调整红黑树。
     <strong>
      如果默认节点是黑色，每一次插入数据都要调整各个路径的值，操作复杂。
     </strong>
    </p>
    <h3>
     <strong>
      3.2 红黑树框架
     </strong>
    </h3>
    <pre><code class="language-cpp">template&lt;class K,class V&gt;
class RBTree
{
	using Node = rbTreeNode&lt;K, V&gt;;
public:
	bool Insert(const pair&lt;K, V&gt;&amp; kv)
	{

	}

	bool Find(const pair&lt;K, V&gt;&amp; kv)
	{

	}
private:
	Node* _root;
};</code></pre>
    <p>
     如果想要增加其他功能，可以直接在类内部增加。
     <strong>
      本文主要说明红黑树的节点的插入。
     </strong>
    </p>
    <h2 style="background-color:transparent">
     <strong>
      四. 红黑树节点的插入操作⭐
     </strong>
    </h2>
    <p>
     <strong>
     </strong>
     首先安装二叉搜索树的性质进行插入数据，插入数据后再根据
     <strong>
      红黑树的性质进行调整
     </strong>
    </p>
    <h3 style="background-color:transparent">
     <strong>
      3.1 插入节点父亲为黑色（情况一）
     </strong>
    </h3>
    <p>
     <strong>
      如果插入节点的父亲为黑色，插入节点为红色。直接插入即可，无论是其左孩子还是右孩子。这样不会破坏红黑树的任意一个条件。
     </strong>
    </p>
    <h3>
     <strong>
      3.2 插入节点父亲为红色
     </strong>
    </h3>
    <p>
     <strong>
     </strong>
     由于父亲为红色，插入节点也为红色。破坏了红黑树的条件（
     <strong>
      红色节点的孩子必须为黑色
     </strong>
     ）。此时就需要进行调整。这里有很多种情况，读者可以根据图解快速理解。
    </p>
    <p>
     注意：
    </p>
    <p>
     <strong>
      1 如果父亲为红色，那么爷爷一定是黑色
     </strong>
    </p>
    <p>
     <strong>
      2
     </strong>
     所以红黑树的调整主要是：
     <strong>
      叔叔节点的颜色，位置（爷爷的左右孩子），是否存在
     </strong>
    </p>
    <h4>
     a 叔叔存在且颜色为红色（情况二与三）
    </h4>
    <p>
     如果叔叔存在且为红色，则会有下面两种情况。
    </p>
    <p>
     <img alt="" height="652" src="https://i-blog.csdnimg.cn/direct/ac641b7034134e4ca44b644a74f118da.png" width="1158"/>
    </p>
    <blockquote>
     <p>
      上面这两种情况，
      <strong>
       我们只需要将父亲p和叔叔u变为黑色，然后将爷爷变为黑色。
      </strong>
      这样就能保证从爷爷开始的整颗树满足红黑树
     </p>
    </blockquote>
    <p>
     如下图：
    </p>
    <p>
     <img alt="" height="763" src="https://i-blog.csdnimg.cn/direct/623a7cef6b4a429da45f88be8658ed6e.png" width="960">
     </img>
    </p>
    <p>
     调整完之后，还要继续向上调整。
    </p>
    <blockquote>
     <p>
      <strong>
       如果爷爷是根节点，则直接将爷爷变为黑色，调整结束。否则需要继续向上调整，因为爷爷是红色节点，如果爷爷的父亲是红色则需要重新调整（不能有连续的红节点）。
      </strong>
     </p>
     <p>
      <strong>
       此时爷爷g就是新的cur。
      </strong>
     </p>
    </blockquote>
    <p>
     就像下图：
    </p>
    <p>
     <img alt="" height="539" src="https://i-blog.csdnimg.cn/direct/c8331a33e345430f9ad7a5e35248386e.png" width="1542">
     </img>
    </p>
    <h4>
     <strong>
      b 叔叔不存在或者叔叔是黑色节点（情况三与四）
     </strong>
    </h4>
    <p>
     <strong>
      图解如下：解释在图解中
     </strong>
    </p>
    <p>
     <img alt="" height="764" src="https://i-blog.csdnimg.cn/direct/7fecac3b0d81410fb049127a1029d962.png" width="1540"/>
    </p>
    <blockquote>
     <p>
      <strong>
       所以这种只需要对g进行右旋转然后再让p变黑色，g变红色即可
      </strong>
     </p>
    </blockquote>
    <p>
     如果cur此时是父亲的右侧，则需要先对p进行左旋。如下图：
    </p>
    <p>
     <img alt="" height="717" src="https://i-blog.csdnimg.cn/direct/eeb5fe44b66640648a5d532313dcf46b.png" width="1879">
     </img>
    </p>
    <p>
     上面展示的是cur父亲是cur爷爷左孩子的情况，对于右孩子的情况也是一样。
     <strong>
      只是旋转的方向改一改就行。
     </strong>
    </p>
    <p>
     <strong>
      c 插入操作的代码
     </strong>
    </p>
    <pre><code class="language-cpp">	bool Insert(const pair&lt;K, V&gt;&amp; kv)
	{
		//1.按照搜索树的规则进行插入
		if (_root == nullptr)
		{
			_root = new Node(kv);
			_root-&gt;_col = BLACK;
			return true;
		}

		Node* parent = nullptr;
		Node* cur = _root;
		while (cur)
		{
			if (cur-&gt;_kv.first &lt; kv.first)
			{
				parent = cur;
				cur = cur-&gt;_right;
			}
			else if (cur-&gt;_kv.first &gt; kv.first)
			{
				parent = cur;
				cur = cur-&gt;_left;
			}
			else
			{
				return false;
			}
		}

		cur = new Node(kv);
		if (cur-&gt;_kv.first &gt; parent-&gt;_kv.first)
		{
			parent-&gt;_right = cur;
		}
		else
		{
			parent-&gt;_left = cur;
		}
		cur-&gt;_parent = parent;

		//2.调整颜色
		//新增结点是红色的(因为红色节更温和，影响红黑树的结构小)
		cur-&gt;_col = RED;

		while (parent &amp;&amp; parent-&gt;_col == RED)
		{
			Node* grandfather = parent-&gt;_parent;

			//父亲在祖父左边
			if (parent == grandfather-&gt;_left)
			{
				//红黑树的调节关键看叔叔
				Node* uncle = grandfather-&gt;_right;

				if (uncle &amp;&amp; uncle-&gt;_col == RED)
				{
					parent-&gt;_col = uncle-&gt;_col = BLACK;
					grandfather-&gt;_col = RED;

					//继续向上调整
					cur = grandfather;
					parent = grandfather-&gt;_parent;
				}
				else
				{
					//情况二三，叔叔是黑色或者不存在

					//情况三，cur在parent右边（折线），此时对parent左旋，再对grandfather右旋
					if (parent-&gt;_right == cur)
					{
						RotateLeft(parent);
						swap(parent, cur);	//只是交换了指针，kv节点并没有交换（更换了父亲和孩子的位置）
					}


					//情况二，cur在parent左边（直线）,此时对grandfather右旋即可
					//再让parent变黑，grandfather变红
					RotateRight(grandfather);
					parent-&gt;_col = BLACK;
					grandfather-&gt;_col = RED;
					break;
				}
			}
			else  //说明父亲在祖父的右边
			{
				Node* uncle = grandfather-&gt;_left;

				//情况一
				if (uncle &amp;&amp; uncle-&gt;_col == RED)
				{
					parent-&gt;_col = uncle-&gt;_col = BLACK;
					grandfather-&gt;_col = RED;

					cur = grandfather;
					parent = grandfather-&gt;_parent;
				}
				else
				{
					//情况二三

					//情况三（折线），此时父亲在右边，cur在父亲左边，对parent右旋，再对grandfath左旋
					if (cur == parent-&gt;_left)
					{
						RotateRight(parent);
						swap(cur, parent);
					}
					//情况二（直线）						
					RotateLeft(grandfather);
					grandfather-&gt;_col = RED;
					parent-&gt;_col = BLACK;
					break;	//此时颜色已经调整好了，不需要继续调整
				}

			}
		}
		_root-&gt;_col = BLACK;	//无论如何变色，最后让根为黑色。就不会出错
		return true;
	}</code></pre>
    <p>
     旋转操作的代码：
    </p>
    <pre><code class="language-cpp">void RotateLeft(Node* parent)
	{
		if (!parent)
			return;
		Node* ppNode = parent-&gt;_parent;//要旋转节点的父亲
		Node* SubR = parent-&gt;_right;//要旋转节点的右孩子
		Node* SubRLeft = SubR-&gt;_left;//要旋转节点右孩子的左孩子


		//一:调整节点
		parent-&gt;_right = SubRLeft;
		if (SubRLeft)
			SubRLeft-&gt;_parent = parent;

		SubR-&gt;_left = parent;
		parent-&gt;_parent = SubR;

		//1.parent是根，现在SubR是根
		//2.parent是整棵树的子树，找到其父亲，旋转完成后，让subR与其父亲相连接
		if (_root == parent)
		{
			_root = SubR;
			SubR-&gt;_parent = nullptr;
		}
		else
		{
			if (ppNode-&gt;_left == parent)
			{
				ppNode-&gt;_left = SubR;
			}
			else
			{
				ppNode-&gt;_right = SubR;
			}

			SubR-&gt;_parent = ppNode;
		}
	}

	//左左，右单旋
	void RotateRight(Node* parent)
	{
		if (!parent)
			return;
		Node* ppNode = parent-&gt;_parent;
		Node* SubL = parent-&gt;_left;
		Node* SubLRight = SubL-&gt;_right;

		//1.旋转,调整节点
		parent-&gt;_left = SubLRight;
		if (SubLRight)
			SubLRight-&gt;_parent = parent;

		SubL-&gt;_right = parent;
		parent-&gt;_parent = SubL;

		if (_root == parent)
		{
			_root = SubL;
			SubL-&gt;_parent = nullptr;
		}
		else
		{
			if (ppNode-&gt;_left == parent)
			{
				ppNode-&gt;_left = SubL;
			}
			else
			{
				ppNode-&gt;_right = SubL;
			}
			SubL-&gt;_parent = ppNode;
		}
	}</code></pre>
