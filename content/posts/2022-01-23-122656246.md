    <h2>
     <a id="_6">
     </a>
     一：面向对象分析的基本过程
    </h2>
    <h3>
     <a id="1_7">
     </a>
     （1）定义
    </h3>
    <p>
     <strong>
      <font color="ff0000">
       面向对象分析
      </font>
      ：就是抽取和整理
      <font color="0000ff">
       用户需求
      </font>
      并建立问题域精确模型的过程
     </strong>
    </p>
    <h3>
     <a id="235_12">
     </a>
     （2）3个子模型与5个层次
    </h3>
    <h4>
     <a id="A3_13">
     </a>
     A：3个子模型
    </h4>
    <p>
     面向对象建模得到的模型包含系统的三个要素：
    </p>
    <ul>
     <li>
      <strong>
       <font color="0000ff">
        静态结构（对象模型）
       </font>
       ：解决任一问题，都需要从客观世界实体及实体间相互关系抽象出极有价值的对象模型
      </strong>
     </li>
     <li>
      <strong>
       <font color="0000ff">
        交互次序（动态模型）
       </font>
       ：当问题涉及交互作用和时序时，动态模型是重要的
      </strong>
     </li>
     <li>
      <strong>
       <font color="0000ff">
        数据变换（功能模型）
       </font>
       ：解决运算量很大的问题，则涉及重要的功能模型
      </strong>
     </li>
    </ul>
    <h4>
     <a id="B5_22">
     </a>
     B：5个层次
    </h4>
    <p>
     复杂问题的对象模型由5个层次组成:
     <strong>
      主题层、类与对象层、结构层、属性层和服务层
     </strong>
     。这5个层次一层比一层显现出对象模型的更多细节
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/22cefb70039733eca0581d1088160b50.png"/>
    </p>
    <h5>
     <a id="_28">
     </a>
     ①：主题
    </h5>
    <p>
     主题是
     <strong>
      指导读者理解大型、复杂模型的一种机制
     </strong>
     。即通过划分主题把一个大型、复杂的对象模型分解成
     <strong>
      几个不同的概念范畴
     </strong>
    </p>
    <h5>
     <a id="7_pm_2_32">
     </a>
     ②：7
     <span class="katex--inline">
      <span class="katex">
       <span class="katex-mathml">
        ± 
        
       
      
        \pm
       </span>
       <span class="katex-html">
        <span class="base">
         <span class="strut" style="height: 0.6667em; vertical-align: -0.0833em;">
         </span>
         <span class="mord">
          ±
         </span>
        </span>
       </span>
      </span>
     </span>
     2原则
    </h5>
    <p>
     心理研究表明，人类的短期记忆能力一般限于
     <strong>
      一次记忆5~9个对象
     </strong>
     ，面向对象从两个方面体现这条原则
    </p>
    <ul>
     <li>
      <strong>
       控制可见性
      </strong>
      ：控制读者能见到的层次数目来控制可见性
     </li>
     <li>
      <strong>
       指导读者注意力
      </strong>
      ：增加了主题层，可从高层次描述总体模型，并指导读者的注意力
     </li>
    </ul>
    <h5>
     <a id="_40">
     </a>
     ③：面向对象分析顺序
    </h5>
    <ol>
     <li>
      寻找类与对象
     </li>
     <li>
      识别结构
     </li>
     <li>
      识别主题
     </li>
     <li>
      定义属性
     </li>
     <li>
      建立动态模型
     </li>
     <li>
      建立功能模型
     </li>
     <li>
      定义服务
     </li>
    </ol>
    <h5>
     <a id="_53">
     </a>
     ④：建模要点
    </h5>
    <ul>
     <li>
      面向对象分析
      <strong>
       不可能严格按照顺序线性进行
      </strong>
     </li>
     <li>
      必须在反复分析中
      <strong>
       对初始模型中不准确、不完整和错误
      </strong>
      的内容加以扩充和更正
     </li>
     <li>
      仔细研究
      <strong>
       类似问题的分析结果
      </strong>
      ，尽可能
      <strong>
       重用
      </strong>
      这些结果
     </li>
    </ul>
    <h2>
     <a id="_61">
     </a>
     二：需求陈述
    </h2>
    <h3>
     <a id="1_63">
     </a>
     （1）内容
    </h3>
    <ul>
     <li>
      阐明“
      <strong>
       做什么
      </strong>
      ”而不是“怎样做”
     </li>
     <li>
      描述
      <strong>
       用户的需求
      </strong>
      而不是提出解决问题的方法
     </li>
     <li>
      指出哪些是
      <strong>
       系统必要的性质
      </strong>
      ，哪些是任选的性质
     </li>
     <li>
      避免对
      <strong>
       设计策略施加过多的约束
      </strong>
      ，不描述系统的
      <strong>
       内部结构
      </strong>
     </li>
     <li>
      描
      <strong>
       述系统性能
      </strong>
      及
      <strong>
       系统与外界环境交互协议
      </strong>
     </li>
     <li>
      描述采用的
      <strong>
       软件工程标准、模块构造准则、将来的扩充以及可维护性要求
      </strong>
      等方面
     </li>
    </ul>
    <h3>
     <a id="2_72">
     </a>
     （2）书写规范
    </h3>
    <ul>
     <li>
      做到
      <strong>
       语法正确
      </strong>
      ，而且应该
      <strong>
       慎重选用名词、动词、形容词和同义词
      </strong>
     </li>
     <li>
      必须把
      <strong>
       需求与实现策略区分开
      </strong>
      ，后者不是问题域的本质性质
     </li>
     <li>
      需求陈述
      <strong>
       可简可繁
      </strong>
     </li>
     <li>
      避免出现具有
      <strong>
       二义性的、不完整的、不一致
      </strong>
      的内容
     </li>
    </ul>
