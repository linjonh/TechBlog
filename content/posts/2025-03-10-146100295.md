    <blockquote>
     <p>
      我们在开发中经常会写日志，所以需要有个日志可视化界面管理，使用ELK可以实现高效集中化的日志管理与分析，提升性能稳定性，满足安全合规要求，支持开发运维工作。

     下述是我在搭建ELK时遇到的许许多多的坑，希望能替大家避开.

    <h3>
     1. 环境准备
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        Elasticsearch
       </strong>
       : 用于存储和检索日志数据。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Logstash
       </strong>
       : 用于收集、处理和转发日志数据。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Kibana
       </strong>
       : 用于可视化和分析日志数据。
      </p>

    <h3>
     2. 安装 ELK 组件
    </h3>
    <p>
     你可以通过 Docker 快速安装 ELK 组件：
    </p>
    <pre><code class="language-java"># 拉取 ELK 镜像
docker pull docker.elastic.co/elasticsearch/elasticsearch:7.10.0
docker pull docker.elastic.co/logstash/logstash:7.10.0
docker pull docker.elastic.co/kibana/kibana:7.10.0

# 启动 Elasticsearch
docker run -d --name elasticsearch -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" docker.elastic.co/elasticsearch/elasticsearch:7.10.0

# 启动 Logstash
docker run -d --name logstash -p 5044:5044 docker.elastic.co/logstash/logstash:7.10.0

# 启动 Kibana
docker run -d --name kibana -p 5601:5601 --link elasticsearch:elasticsearch docker.elastic.co/kibana/kibana:7.10.0</code></pre>
    <p>

    <p>
     <strong>
      避坑点1：
     </strong>
    </p>
    <p>
     在启动时，发现老是报错，查了很久都没有结果，最终终于排查出来是环境变量配置的问题.
    </p>
    <p>
     我们可以看下述它自带的配置文件，文件中说明了若配置了ES_JAVA_HOME，则会使用此配置指向的JDK，但是此处JAVA_HOME和CLASS_PATH的环境变量会影响启动，所以如果想要直接使用它自带的JDK，需要将JAVA_HOME和CLASS_PATH的环境变量暂时注释掉.
    </p>
    <pre><code class="language-java">set SCRIPT=%0

rem determine Elasticsearch home; to do this, we strip from the path until we
rem find bin, and then strip bin (there is an assumption here that there is no
rem nested directory under bin also named bin)
for %%I in (%SCRIPT%) do set ES_HOME=%%~dpI

:es_home_loop
for %%I in ("%ES_HOME:~1,-1%") do set DIRNAME=%%~nxI
if not "%DIRNAME%" == "bin" (
  for %%I in ("%ES_HOME%..") do set ES_HOME=%%~dpfI
  goto es_home_loop
)
for %%I in ("%ES_HOME%..") do set ES_HOME=%%~dpfI

rem now set the classpath
set ES_CLASSPATH=!ES_HOME!\lib\*
set ES_MODULEPATH=!ES_HOME!\lib
set LAUNCHERS_CLASSPATH=!ES_CLASSPATH!;!ES_HOME!\lib\launchers\*;!ES_HOME!\lib\java-version-checker\*
set SERVER_CLI_CLASSPATH=!ES_CLASSPATH!;!ES_HOME!\lib\tools\server-cli\*

set HOSTNAME=%COMPUTERNAME%

if not defined ES_PATH_CONF (
  set ES_PATH_CONF=!ES_HOME!\config
)

rem now make ES_PATH_CONF absolute
for %%I in ("%ES_PATH_CONF%..") do set ES_PATH_CONF=%%~dpfI

set ES_DISTRIBUTION_TYPE=zip

cd /d "%ES_HOME%"

rem now set the path to java, pass "nojava" arg to skip setting ES_JAVA_HOME and JAVA
if "%1" == "nojava" (
   exit /b
)

rem comparing to empty string makes this equivalent to bash -v check on env var
rem and allows to effectively force use of the bundled jdk when launching ES
rem by setting ES_JAVA_HOME=
if defined ES_JAVA_HOME (
  set JAVA="%ES_JAVA_HOME%\bin\java.exe"
  set JAVA_TYPE=ES_JAVA_HOME
  if not exist !JAVA! (
    echo "could not find java in !JAVA_TYPE! at !JAVA!" &gt;&amp;2
    exit /b 1
  )

  rem check the user supplied jdk version
  !JAVA! -cp "%ES_HOME%\lib\java-version-checker\*" "org.elasticsearch.tools.java_version_checker.JavaVersionChecker" || exit /b 1
) else (
  rem use the bundled JDK (default)
  set JAVA="%ES_HOME%\jdk\bin\java.exe"
  set "ES_JAVA_HOME=%ES_HOME%\jdk"
  set JAVA_TYPE=bundled JDK
)

rem do not let JAVA_TOOL_OPTIONS slip in (as the JVM does by default)
if defined JAVA_TOOL_OPTIONS (
  echo warning: ignoring JAVA_TOOL_OPTIONS=%JAVA_TOOL_OPTIONS%
  set JAVA_TOOL_OPTIONS=
)

rem warn that we are not observing the value of $JAVA_HOME
if defined JAVA_HOME (
  echo warning: ignoring JAVA_HOME=%JAVA_HOME%; using %JAVA_TYPE% &gt;&amp;2
)

rem JAVA_OPTS is not a built-in JVM mechanism but some people think it is so we
rem warn them that we are not observing the value of %JAVA_OPTS%
if defined JAVA_OPTS (
  (echo|set /p=warning: ignoring JAVA_OPTS=%JAVA_OPTS%; )
  echo pass JVM parameters via ES_JAVA_OPTS
)

cd %ES_HOME%
</code></pre>

    <h4>
     2. 配置文件编写
    </h4>
    <p>
     Logstash 的配置文件分为两类：主配置文件（
     <code>
      logstash.conf
     </code>
     ）和管道配置文件（
     <code>
      pipelines.yml
     </code>
     ）。
    </p>
    <p>
    </p>
    <h5>
     1）
     <strong>
      <code>
       logstash.conf
      </code>
      配置文件
     </strong>
    </h5>
    <p>
     <code>
      logstash.conf
     </code>
     是 Logstash 的核心配置文件，定义了数据的输入、过滤和输出。它通常位于 Logstash 安装目录的
     <code>
      config
     </code>
     文件夹中。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        输入（Input）
       </strong>
       ：定义数据源，例如日志文件、网络端口等。
       <br/>
       示例：
      </p>
      <pre><code>input {
    file {
        path =&gt; "/var/log/*.log"
        start_position =&gt; "beginning"
    }
}</code></pre>
     </li>
     <li>
      <p>
       <strong>
        过滤（Filter）
       </strong>
       ：对数据进行解析和处理，例如使用 Grok 插件解析日志。
       <br/>
       示例：
      </p>
      <pre><code>filter {
    grok {
        match =&gt; { "message" =&gt; "%{IP:client} %{WORD:method} %{URIPATHPARAM:request}" }
    }
}</code></pre>
     </li>
     <li>
      <p>
       <strong>
        输出（Output）
       </strong>
       ：定义数据的输出目标，例如 Elasticsearch、文件或控制台。
       <br/>
       示例：
      </p>
     </li>
     <li>
      <pre><code>output {
    elasticsearch {
        hosts =&gt; ["http://localhost:9200"]
        index =&gt; "logstash-%{+YYYY.MM.dd}"
    }
    stdout {
        codec =&gt; rubydebug
    }
}</code></pre>
     </li>
    </ul>
    <p>
    </p>
    <h5>
     （2）
     <strong>
      <code>
       pipelines.yml
      </code>
      配置文件
     </strong>
    </h5>
    <p>
     <code>
      pipelines.yml
     </code>
     文件用于定义管道的配置，包括管道的名称和配置文件路径。它通常位于 Logstash 安装目录的
     <code>
      config
     </code>
     文件夹中。
    </p>
    <pre><code class="language-java">- pipeline.id: main
  path.config: "config/logstash-sample.conf"</code></pre>
    <p>
     <strong>
      避坑点2：
     </strong>
    </p>
    <p>
     此处不知道怎么配置都会显示
     <strong>
      No configuration found in the configured sources
     </strong>
     . 一直没找出原因
    </p>
    <p>
     <img alt="" height="595" src="https://i-blog.csdnimg.cn/direct/ab3c226cdcbb4369925f5a5fb3b0a69f.png" width="1104"/>
    </p>
    <p>
     重新从配置的信息中找原因，最终发现是conf配置文件的问题.
    </p>
    <p>
     <strong>
      logstash-sample.conf
     </strong>
     示例如下：
    </p>
    <pre><code class="language-java">input {
  # 示例3：HTTP 接收数据
  http {
    port =&gt; 8080
    codec =&gt; "json"
  }
}

output {
  # 示例1：输出到 Elasticsearch（带安全认证）
  elasticsearch {
    hosts =&gt; ["http://localhost:9200"]
    index =&gt; "logs-%{+YYYY.MM.dd}"
    user =&gt; "elastic"
    password =&gt; "password"
    ssl =&gt; false
  }
}</code></pre>
    <blockquote>
     <p>
      <strong>
       logstash.yml  此文件需要配置上去，否则会一直报错，很多教程中没有指这一点！！！
      </strong>
     </p>
    </blockquote>
    <pre><code class="language-java">
# 节点名称（用于集群标识）
node.name: "my-logstash-node"

# 管道配置路径（可指定多个管道）
path.config: "config/pipelines/*.conf"

# 数据存储路径（队列和持久化数据）
path.data: "data/"

# 日志路径
path.logs: "logs/"

# HTTP API 配置（用于监控和管理）
api.http.host: "0.0.0.0"
api.http.port: 9600

# 性能调优（根据硬件调整线程和批处理大小）
pipeline.workers: 2          # 工作线程数（建议等于CPU核心数）
pipeline.batch.size: 125     # 每批处理事件数</code></pre>
    <p>
     上述坑点避开后，基本上都能启动了。效果如下：

      原因：kibana连接Elasticsearch不允许直接使用超级账号elastics，需要按照下述方式创建专用账号

    <h5>
     使用
     <code>
      elasticsearch-service-tokens
     </code>
     CLI工具
    </h5>
    <ol>
     <li>
      <p>
       打开终端或命令提示符。
      </p>
     </li>
     <li>
      <p>
       运行以下命令来创建服务账号令牌：
      </p>
      <p>
       bash复制
      </p>
      <pre><code>elasticsearch-service-tokens create elastic/kibana &lt;token_name&gt;</code></pre>
      <p>
       其中
       <code>
        elastic/kibana
       </code>
       是Kibana的服务账号，
       <code>
        &lt;token_name&gt;
       </code>
       是你为这个令牌指定的名称，例如
       <code>
        kibana-token
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       该命令会在
       <code>
        $ES_HOME/config/service_tokens
       </code>
       文件中保存新的服务令牌，并在终端输出Bearer令牌。
      </p>
     </li>
    </ol>
    <pre><code># 配置Kibana服务运行的主机地址，允许远程访问时可设置为"0.0.0.0"
server.host: "localhost"

# 配置Kibana服务运行的端口
server.port: 5601

# 配置Elasticsearch的地址，如果有多个节点，可以像下面这样列出
elasticsearch.hosts:
  - "http://localhost:9200"

elasticsearch.serviceAccountToken: "AAEAAWVsYXN0aXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX1NldHZlZjdm9B"</code></pre>

    <h3>
     <strong>
      3. 集成项目

    <p>
     上述三个服务全部安装完成之后，我们就可以开始对接项目了.

    <h4>
    </h4>
    <h4>
     Spring Boot 项目配置
    </h4>
    <h4>
     3.1 添加依赖
    </h4>
    <p>
     在
     <code>
      pom.xml
     </code>
     中添加依赖：
    </p>
    <pre><code>&lt;dependency&gt;
    &lt;groupId&gt;net.logstash.logback&lt;/groupId&gt;
    &lt;artifactId&gt;logstash-logback-encoder&lt;/artifactId&gt;
    &lt;version&gt;6.6&lt;/version&gt;
&lt;/dependency&gt;</code></pre>
    <h5>
     <strong>
      3.2 添加配置
     </strong>
    </h5>
    <p>
     在
     <code>
      src/main/resources
     </code>
     目录下创建
     <code>
      logback-spring.xml
     </code>
     文件（此处需注意，若已存在logback相关xml文件，则不需要重新创建）：
    </p>
    <pre><code>&lt;?xml version="1.0" encoding="UTF-8"?&gt;
&lt;configuration&gt;
    &lt;include resource="org/springframework/boot/logging/logback/base.xml"/&gt;

    &lt;appender name="LOGSTASH" class="net.logstash.logback.appender.LogstashTcpSocketAppender"&gt;
        &lt;destination&gt;localhost:5044&lt;/destination&gt;
        &lt;encoder class="net.logstash.logback.encoder.LogstashEncoder"/&gt;
    &lt;/appender&gt;

    &lt;root level="INFO"&gt;
        &lt;appender-ref ref="LOGSTASH"/&gt;
        &lt;appender-ref ref="console"/&gt;
    &lt;/root&gt;
&lt;/configuration&gt;</code></pre>
    <p>
     添加上述配置，注意root节点不能重复，重复的话会以最后一个root节点为主.
    </p>
    <p>
    </p>
    <h4>
     <strong>
      3.3 测试：
     </strong>
    </h4>
    <p>
     在你的项目中将日志打印出来
    </p>
    <pre><code>logger.info("用户列表日志记录！！！");</code></pre>
    <p>
     若上述有问题，则需要检查是否已经创建了索引，先检查logstash的配置文件：
    </p>
    <pre><code>input {
  tcp {
    port =&gt; 5044
    codec =&gt; json_lines
  }
}

output {
  # 示例1：输出到 Elasticsearch（带安全认证）
  elasticsearch {
    hosts =&gt; ["http://localhost:9200"]
    index =&gt; "springboot-logs-%{+YYYY.MM.dd}"
    user =&gt; "elastic"
    password =&gt; "XXX"
    ssl =&gt; false
  }
}</code></pre>
    <p>
     此处的tcp监听是监听springboot配置文件中对应的端口.
