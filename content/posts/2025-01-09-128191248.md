    <h2>
     DNS简介
    </h2>
    <p>
     DNS，全称 Domain Name System，域名系统。
    </p>
    <p>
     DNS是将域名解析成IP地址，然后找到IP对应的主机或者服务器。我们平常上网查找资料的时候，总是在浏览器的搜索栏输入 www.baidu.com ，当出现百度一下的界面，我们就进行搜索，实际上，在输入 www.baidu.com 按下回车键的时候，就已经开始了域名解析的过程，最后解析成一个ip地址，然后找到ip对应的百度的服务器，将页面呈现。
    </p>
    <p>
     DNS分为两种解析方式：
    </p>
    <p>
     正向解析：将域名解析成IP地址
     <br/>
     逆向解析：将IP地址解析成域名
     <br/>
     我们通常使用的都是正向解析，即将域名www.baidu.com输入浏览器，然后解析成IP地址，进行访问。
    </p>
    <p>
     DNS递归查询和迭代查询，简单的来说：
    </p>
    <p>
     递归查询：就是主机发送DNS请求给本地的DNS服务器，本地服务器有记录，就直接回给主机；若没有，本地的DNS服务器就会以DNS客户端的身份向根服务器发送请求，根没有，就像其他顶级服务器查询，直到查到结果，返还给本地DNS服务器，然后交由本地服务器发给主机，整个过程中，主机只和本地服务器建立连接。
    </p>
    <p>
     迭代查询：主机发送请求给本地服务器，若本地没有，则返回一个可以进行查询的其他的服务器的列表，然后主机再向列表中的服务器进行请求，直至查出，返回给主机。整个过程，主机不知建立和本地服务器的连接，还会和其他的服务器进行连接。
    </p>
    <p>
     记录类型：A,AAAA,PTR,SOA,NS,CNAME,MX
    </p>
    <p>
     SOA：Start Of Authority，起始授权记录；一个区域解析库有且仅能有一个SOA记录，必须位于解析库的第一条记录SOA，是起始授权机构记录，说明了在众多 NS 记录里哪一台才是主要的服务器。在任何DNS记录文件中，都是以SOA ( Startof Authority )记录开始。SOA资源记录表明此DNS名称服务器是该DNS域中数据信息的最佳来源。
    </p>
    <p>
     A：域名解析成IP地址
    </p>
    <p>
     AAAA：IPV6
    </p>
    <p>
     PTR：反向解析，ip地址解析成域名
    </p>
    <p>
     NS：专用于标明当前区域的DNS服务器，服务器类型为域名服务器
    </p>
    <p>
     CNAME：别名记录
    </p>
    <p>
     MX：邮件交换器
    </p>
    <p>
     <strong>
      SOA记录与NS记录的区别：NS记录表示域名服务器记录，用来指定该域名由哪个DNS服务器来进行解析；SOA记录设置一些数据版本和更新以及过期时间等信息。
     </strong>
    </p>
    <h2>
     <strong>
      安装DNS服务器
     </strong>
    </h2>
    <h3>
     安装dns服务
    </h3>
    <blockquote>
     <p>
      yum install bind
     </p>
     <p>
      yum install bind-utils
     </p>
    </blockquote>
    <p>
     <img alt="" height="434" src="https://i-blog.csdnimg.cn/blog_migrate/c581dff801a29c1760b3b55d1f659cde.png" width="645"/>
    </p>
    <p>
     <em>
      如果遇到镜像无法访问的话，可以检查一下虚拟机的镜像路径是否正确，重启虚拟机应该就可以解决了。
     </em>
    </p>
    <h3>
     配置文件
    </h3>
    <p>
     编辑named.conf文件
    </p>
    <blockquote>
     <p>
      vim /etc/named.conf
     </p>
    </blockquote>
    <p>
     配置文件
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/efbd6460900d7f6796147d09f2c9cd9b.png"/>
    </p>
    <pre><code>options {
        listen-on port 53 { 127.0.0.1; };
        listen-on port 53 { any; };
//      listen-on-v6 port 53 { ::1; };
        directory       "/var/named";
        dump-file       "/var/named/data/cache_dump.db";
        statistics-file "/var/named/data/named_stats.txt";
        memstatistics-file "/var/named/data/named_mem_stats.txt";
        secroots-file   "/var/named/data/named.secroots";
        recursing-file  "/var/named/data/named.recursing";
//      allow-query     { localhost; };
        allow-query     { any; };
</code></pre>
    <p>
     检查语法错误
    </p>
    <blockquote>
     <p>
      named-checkconf
     </p>
    </blockquote>
    <p>
     <em>
      没有错误不会报错，有错误按照提示修改即可
     </em>
    </p>
    <p>
     编写正向解析配置文件
    </p>
    <p>
     查看本机ip
    </p>
    <blockquote>
     <p>
      ifconfig
     </p>
    </blockquote>
    <p>
     <img alt="" height="380" src="https://i-blog.csdnimg.cn/blog_migrate/b212565b7af8717eadd2cfac8724c488.png" width="655"/>
    </p>
    <blockquote>
     <p>
      cd /var/named/
     </p>
     <p>
      vim iris.com.zone
     </p>
    </blockquote>
    <pre><code>$TTL 1D
@	IN SOA	@ rname.invalid. (
					0	; serial
					1D	; refresh
					1H	; retry
					1W	; expire
					3H )	; minimum
	NS	@
	A	192.168.1.128           //添加A记录，为本机IP
www	A	192.168.1.128    //添加www的A记录，即www.iris.com
	AAAA	::1

</code></pre>
    <p>
     编写反向解析配置文件
    </p>
    <blockquote>
     <p>
      cd /var/named/
     </p>
     <p>
      vim iris.com.local
     </p>
    </blockquote>
    <pre><code>$TTL 1D
@       IN SOA  @ rname.invalid. (
                                        0       ; serial
                                        1D      ; refresh
                                        1H      ; retry
                                        1W      ; expire
                                        3H )    ; minimum
        NS      @
        AAAA    ::1
128     PTR     www.iris.com
</code></pre>
    <p>
     编写服务器配置文件
    </p>
    <blockquote>
     <p>
      vim /etc/named.rfc1912.zones
     </p>
    </blockquote>
    <p>
     在文件末尾添加配置
    </p>
    <pre><code>//正向区域配置
zone "iris.com"IN {
        type master;
        file "iris.com.zone"; 
        allow-update{none;};
};
//反向区域配置
zone "1.168.192.in-addr.arpa"IN{
        type master;
        file "iris.com.local"; 
        allow-update{none;};
};

</code></pre>
    <p>
     重启dns服务
    </p>
    <blockquote>
     <p>
      systemctl restart named
     </p>
    </blockquote>
    <p>
     <em>
      如果还在named下记得先用cd退出
     </em>
    </p>

     测试
    </h3>

    </p>
    <blockquote>
     <p>
      vim etc/resolv.conf
     </p>
    </blockquote>
