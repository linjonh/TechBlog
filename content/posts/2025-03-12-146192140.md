    <h2 id="1.%20Shell%E7%9A%84%E8%BF%90%E8%A1%8C%E5%8E%9F%E7%90%86" name="1.%20Shell%E7%9A%84%E8%BF%90%E8%A1%8C%E5%8E%9F%E7%90%86">
     <span style="color:#be191c">
      1. Shell的运行原理
     </span>
    </h2>
    <p>
     我们都知道Windows以图形化界面为交互方式，而Linux以命令行界面为交互方式。
    </p>
    <p>
     图形化界面和命令行界面都是为了让用户进行相关操作，而图形化界面和命令行界面就是我们所说的“
     <span style="color:#511b78">
      <strong>
       外壳程序
      </strong>
     </span>
     ”。
    </p>
    <p>
     <img alt="" height="413" src="https://i-blog.csdnimg.cn/direct/93510b632c3d46778504737e642d8b45.png" width="1354"/>
    </p>
    <p>
     Linux严格意义上说是一个操作系统，我们称之为“核心（kernel）”，但我们一般用户不能直接使用kernel，而是通过kernel的“外壳程序”，也就是所谓的
     <span style="color:#fe2c24">
      <strong>
       Shell
      </strong>
     </span>
     ，来与kernel沟通。
    </p>
    <p>
     Shell最简单的定义就是“
     <strong>
      命令行解释器
     </strong>
     ”：
    </p>
    <p>
     1）将使用者的命令翻译给核心（kernel）处理。
     <br/>
     2）将核心的处理结果翻译给使用者。
    </p>
    <p>
     对比Windows中的图形化界面（GUI），我们操作Windows并不是直接操作Windows内核，而是通过图形接口，点击，从而完成我们的操作。
    </p>
    <p>
     Shell对于Linux具有相同的作用，主要是对我们的指令进行解析，
     <strong>
      解析指令给Linux内核，反馈结果再通过内核运行出结果
     </strong>
     ，通过Shell解析给用户。
    </p>
    <p>
     那么上面我们认识了Shell是什么，下面我们来谈它的运行原理。
    </p>
    <p>
     <strong>
      Shell运行原理：
     </strong>
     <br/>
     1）创建子进程，让子进程进行命令行解释。
     <br/>
     2）子进程出现任何问题，都不影响父进程Shell。
    </p>
    <p>
     这里我们简单理解，其实就是子进程与父进程是相互独立的，我们来Windows下启动一个程序，本质上是创建了一个子进程；比如我们打开微信，再打开QQ；如果此时微信程序终止了，或者我们退出了，QQ不会受到任何影响，这就是因为微信和QQ是两个独立的子进程，谁都不影响谁。
    </p>
    <p>
     <strong>
      注意：
     </strong>
     Shell只是所有外壳程序的统称，例如在centos 7当中的外壳程序名叫bash。
    </p>
    <h2 id="2.%20Linux%E4%B8%AD%E7%9A%84%E6%9D%83%E9%99%90%E9%97%AE%E9%A2%98" name="2.%20Linux%E4%B8%AD%E7%9A%84%E6%9D%83%E9%99%90%E9%97%AE%E9%A2%98">
     <span style="color:#be191c">
      2. Linux中的权限问题
     </span>
    </h2>
    <h3 id="2.1%20Linux%E6%9D%83%E9%99%90%E7%9A%84%E6%A6%82%E5%BF%B5" name="2.1%20Linux%E6%9D%83%E9%99%90%E7%9A%84%E6%A6%82%E5%BF%B5">
     <span style="color:#956fe7">
      2.1 Linux权限的概念
     </span>
    </h3>
    <p>
     <span style="color:#0d0016">
      关于权限的概念，这里其实简单理解：
      <strong>
       ”权限“就是能或者不能做什么
      </strong>
      。大家需要知道：
     </span>
     <span style="color:#511b78">
      <strong>
       权限=角色+目标属性。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      在Linux下有两种用户，分别是超级用户（root）和普通用户。超级用户可以在Linux下做任何事情，几乎不受限制，而普通用户一般只能在自己的工作目录下（/home/xxx）工作，以及在系统上做有限的工作。
      <br/>
      换句话来说，所有的权限的概念都是用来限制普通用户的，而超级用户几乎不受限制。
     </span>
     <br/>
     <span style="color:#ff9900">
      <strong>
       超级用户的命令提示符是“#”
      </strong>
      。
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       root@hcss-ecs-348a:~# whoami
       <br/>
       root
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#ff9900">
      <strong>
       普通用户的命令提示符是“$”。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:/root$ whoami
       <br/>
       cp
      </strong>
     </span>
    </p>
    <h4 id="2.1.1%20%E5%A6%82%E4%BD%95%E5%AE%9E%E7%8E%B0%E7%94%A8%E6%88%B7%E8%B4%A6%E5%8F%B7%E4%B9%8B%E9%97%B4%E7%9A%84%E5%88%87%E6%8D%A2" name="2.1.1%20%E5%A6%82%E4%BD%95%E5%AE%9E%E7%8E%B0%E7%94%A8%E6%88%B7%E8%B4%A6%E5%8F%B7%E4%B9%8B%E9%97%B4%E7%9A%84%E5%88%87%E6%8D%A2">
     <span style="color:#fe2c24">
      2.1.1 如何实现用户账号之间的切换
     </span>
    </h4>
    <p>
     <strong>
      语法：
     </strong>
     su 用户名/su
     <br/>
     <strong>
      功能：
     </strong>
     用户切换。
     <br/>
     从普通账号切换为root账号。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:/root$ su
       <br/>
       Password:
      </strong>
     </span>
    </p>
    <p>
     上面是我们从普通用户切换到超级用户的命令行操作，在Linux中密码是不回显的。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       root@hcss-ecs-348a:~# su cp
       <br/>
       cp@hcss-ecs-348a:/root$
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      上面是从超级用户切换到普通用户的命令行操作，超级切换普通是不需要输入密码的。
     </span>
    </p>
    <h4 id="2.1.2%20%E5%A6%82%E4%BD%95%E5%B0%86%E6%99%AE%E9%80%9A%E7%94%A8%E6%88%B7%E6%B7%BB%E5%8A%A0%E5%88%B0%E4%BF%A1%E4%BB%BB%E5%88%97%E8%A1%A8%E4%B8%AD" name="2.1.2%20%E5%A6%82%E4%BD%95%E5%B0%86%E6%99%AE%E9%80%9A%E7%94%A8%E6%88%B7%E6%B7%BB%E5%8A%A0%E5%88%B0%E4%BF%A1%E4%BB%BB%E5%88%97%E8%A1%A8%E4%B8%AD">
     <span style="color:#fe2c24">
      2.1.2 如何将普通用户添加到信任列表中
     </span>
    </h4>
    <p>
     首先我们要切换到超级用户，然后使用vim打开文件sudoers进行添加（vim /etc/sudoers
     <br/>
     ）。打开后往下滑动，你会看见类似下面的命令行，这时按“i”切换到插入模式，将自己的普通用户名添加上去就OK了，我这里是ubuntu22.04系统下运行的。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       # User privilege specification
       <br/>
       root    ALL=(ALL:ALL) ALL
       <br/>
       cp       ALL=(ALL:ALL) ALL
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      添加完成后，我们按“Esc”退出插入模式，再按“Shift+:”进入底行模式，你会在最后一行看见“:”，然后输入“wq!”保存退出即可。
     </span>
    </p>
    <h4 id="2.1.3%C2%A0%E5%A6%82%E4%BD%95%E4%BB%85%E6%8F%90%E5%8D%87%E5%BD%93%E5%89%8D%E6%8C%87%E4%BB%A4%E7%9A%84%E6%9D%83%E9%99%90" name="2.1.3%C2%A0%E5%A6%82%E4%BD%95%E4%BB%85%E6%8F%90%E5%8D%87%E5%BD%93%E5%89%8D%E6%8C%87%E4%BB%A4%E7%9A%84%E6%9D%83%E9%99%90">
     <span style="color:#fe2c24">
      2.1.3 如何仅提升当前指令的权限
     </span>
    </h4>
    <p>
     但可能某些情况下，你只想提升当前指令的权限，那么不必切换到超级用户。
    </p>
    <p>
     <strong>
      语法：
     </strong>
     sudo 指令
     <br/>
     <strong>
      功能：
     </strong>
     提升当前指令的权限。
     <br/>
     例如，我现在要以一名普通用户的身份，修改另一个普通用户的账号密码。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:/root$ ls /home
       <br/>
       cp  xjy
       <br/>
       cp@hcss-ecs-348a:/root$ whoami
       <br/>
       cp
       <br/>
       cp@hcss-ecs-348a:/root$ sudo passwd xjy
       <br/>
       [sudo] password for cp:
       <br/>
       New password:
       <br/>
       Retype new password:
       <br/>
       passwd: password updated successfully
      </strong>
     </span>
    </p>
    <p>
    </p>
    <h3 id="2.2%20Linux%E6%9D%83%E9%99%90%E7%AE%A1%E7%90%86%C2%A0" name="2.2%20Linux%E6%9D%83%E9%99%90%E7%AE%A1%E7%90%86%C2%A0">
     <span style="color:#956fe7">
      2.2 Linux权限管理
     </span>
    </h3>
    <p>
     <span style="color:#0d0016">
      那么Linux中为什么要设计权限呢？其实就是为了限制用户的行为，防止错误的发生。
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      那么结合上面的内容，我们来理解一下权限是怎么来”限制“的，分两点：
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      首先，权限限制的角色（人）；
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      其次，权限要求目标必须具有对应属性。
     </span>
    </p>
    <h4 id="2.2.1%C2%A0%E6%96%87%E4%BB%B6%E8%AE%BF%E9%97%AE%E8%80%85%E7%9A%84%E5%88%86%E7%B1%BB%EF%BC%88%E4%BA%BA%EF%BC%89" name="2.2.1%C2%A0%E6%96%87%E4%BB%B6%E8%AE%BF%E9%97%AE%E8%80%85%E7%9A%84%E5%88%86%E7%B1%BB%EF%BC%88%E4%BA%BA%EF%BC%89">
     <span style="color:#fe2c24">
      2.2.1 文件访问者的分类（人）
     </span>
    </h4>
    <p>
     对于用户来说，权限可以将用户分为三大类：
     <br/>
     1）文件和文件目录的所有者（
     <strong>
      文件拥有者
     </strong>
     ）。
     <br/>
     2）文件拥有者所在的组的用户（
     <strong>
      文件所属组
     </strong>
     ）。
     <br/>
     3）其他用户（
     <strong>
      other
     </strong>
     ）。
    </p>
    <p>
     <strong>
      注意：
     </strong>
     <br/>
     1）对于某一文件而言，其拥有者、所属组和other就是由超级用户（root）和普通用户所扮演。
     <br/>
     2）在Linux当中，所有用户都要隶属于某一个组，哪怕这个组只有你一个人（此时该组就以你的用户名为组名）。
    </p>
    <p>
     这里我们可以通过“ll”命令来查看文件的拥有者和所属组。
    </p>
    <p>
     <img alt="" height="289" src="https://i-blog.csdnimg.cn/direct/0ad29912426946e7b02f1beb3ffb88f8.png" width="1019"/>
    </p>
    <p>
     那么大家思考一下为什么要由所属组这个概念呢？
    </p>
    <p>
     其实这里也很好理解，如果我们没有所属组这个概念，那么当我们创建一个文件后，要么就是只有你自己（拥有者）能看到，要么就是其他人（other）也都能看到。而你所希望的是你自己和你的小组成员能看到，剩下的人看不到。
    </p>
    <p>
     于是就有了所属组这个概念，这时你就可以将文件设置为拥有者和所属组可见，而other不可见。所以所属组的存在是为了更灵活的进行权限配置，满足团队协作。
    </p>
    <p>
     <strong>
      注：
     </strong>
     除了文件拥有者和文件所属组之外的都叫other。
    </p>
    <h4 id="2.2.2%C2%A0%E6%96%87%E4%BB%B6%E7%B1%BB%E5%9E%8B%E5%92%8C%E8%AE%BF%E9%97%AE%E6%9D%83%E9%99%90%EF%BC%88%E4%BA%8B%E7%89%A9%E5%B1%9E%E6%80%A7%EF%BC%89" name="2.2.2%C2%A0%E6%96%87%E4%BB%B6%E7%B1%BB%E5%9E%8B%E5%92%8C%E8%AE%BF%E9%97%AE%E6%9D%83%E9%99%90%EF%BC%88%E4%BA%8B%E7%89%A9%E5%B1%9E%E6%80%A7%EF%BC%89">
     <span style="color:#fe2c24">
      2.2.2 文件类型和访问权限（事物属性）
     </span>
    </h4>
    <p>
     <span style="color:#511b78">
      <strong>
       在Linux中，一切皆文件。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      那么根据上面的结论，权限要求的”目标“当然也是文件，那么要求其有对应属性，文件有哪些属性呢？
     </span>
    </p>
    <p>
     在Linux中，文件的属性就三种：
     <span style="color:#511b78">
      <strong>
       读、写、执行。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      同样还是使用指令“ll”来查看文件属性
     </span>
    </p>
    <p>
     <img alt="" height="334" src="https://i-blog.csdnimg.cn/direct/f87874ee974d49f192f0e82e00f784f3.png" width="1051"/>
    </p>
    <p>
     这串字符由10个字符组成的。其中第一个字符所代表的就是该文件的文件类型。
    </p>
    <p>
     <img alt="" height="171" src="https://i-blog.csdnimg.cn/direct/1f96616910b4455985ffaac80572c692.png" width="853"/>
    </p>
    <p>
     不同的字符代表不同的文件类型。
     <br/>
     <strong>
      1）-：代表普通文件。
      <br/>
      2）d：代表目录。
     </strong>
     <br/>
     3）l：代表链接文件（类似于Windows当中的快捷方式）。
     <br/>
     4）b：代表块设备文件（例如硬盘、光驱等）。
     <br/>
     5）p：管道文件。
     <br/>
     6）c：字符设备文件。
     <br/>
     7）s：套接口文件。
     <br/>
     注意： 在Linux当中，文件类型与文件后缀无关。
    </p>
    <p>
     剩下的9个字符每三个为一组，分别代表该文件相对于其拥有者、所属组以及other是否拥有某种属性。
    </p>
    <p>
     <img alt="" height="189" src="https://i-blog.csdnimg.cn/direct/f77e80fb784c482c99d6d27fd5f0f2fa.png" width="853"/>
    </p>
    <p>
     每一组的三个字符的第一个字符代表该文件是否具有可读属性，第二个代表是否具有可写属性，第三个代表是否具有可执行属性。
    </p>
    <p>
     <img alt="" height="133" src="https://i-blog.csdnimg.cn/direct/018db830bda4437cba110fc527393342.png" width="745">
      若是具有可读属性，则第一个位置的字符为r；若是具有可写属性，则第二个位置的字符为w；若是具有可执行属性，则第三个位置的字符为x。若某一位置为字符 - ，则说明不具有对应位置的属性。
     </img>
    </p>
    <p>
     下面我们举个例子，来描述一个文件的权限：
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-r-- 1 cp   cp      0 Mar 12 12:08 test4.txt
       <br/>
       -rw-r--r-- 1 root root    0 Mar 12 12:10 test5.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       对于test4.txt来说，test.txt是一个普通文件，该文件的拥有者和所属组对其都是可读可写的，但该文件的other对其只有读的权限。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       对于test5.txt来说，test5.txt是一个普通文件，该文件的拥有者对其有读和写的权限；该文件的所属组和other都只对其有读的权限。
      </strong>
     </span>
    </p>
    <h4 id="2.2.3%C2%A0%E6%96%87%E4%BB%B6%E6%9D%83%E9%99%90%E5%80%BC%E7%9A%84%E8%A1%A8%E7%A4%BA%E6%96%B9%E6%B3%95" name="2.2.3%C2%A0%E6%96%87%E4%BB%B6%E6%9D%83%E9%99%90%E5%80%BC%E7%9A%84%E8%A1%A8%E7%A4%BA%E6%96%B9%E6%B3%95">
     <span style="color:#fe2c24">
      2.2.3 文件权限值的表示方法
     </span>
    </h4>
    <h5 id="2.2.3.1%20%E5%AD%97%E7%AC%A6%E8%A1%A8%E7%A4%BA%E6%B3%95" name="2.2.3.1%20%E5%AD%97%E7%AC%A6%E8%A1%A8%E7%A4%BA%E6%B3%95">
     <span style="color:#4da8ee">
      2.2.3.1 字符表示法
     </span>
    </h5>
    <p>
     ll指令打印文件权限值时的表示方法就是字符表示法。例如
    </p>
    <p>
     <img alt="" height="587" src="https://i-blog.csdnimg.cn/direct/4d6edf026e694ecabd30b856fb6bed10.png" width="1630"/>
    </p>
    <h5 id="2.2.3.2%20%E5%85%AB%E8%BF%9B%E5%88%B6%E8%A1%A8%E7%A4%BA%E6%B3%95%C2%A0" name="2.2.3.2%20%E5%85%AB%E8%BF%9B%E5%88%B6%E8%A1%A8%E7%A4%BA%E6%B3%95%C2%A0">
     <span style="color:#4da8ee">
      2.2.3.2 八进制表示法
     </span>
    </h5>
    <p>
     字符表示法中的每一个字符所在位置所表示的结果只有两种可能，要么为真，要么为假，因此我们可以将这三个字符换为三个二进制位，进而换为一个八进制位进行表示。例如
    </p>
    <p>
     <img alt="" height="587" src="https://i-blog.csdnimg.cn/direct/67d9855851694cbbbcc3ccd8cf50044a.png" width="1621"/>
    </p>
    <h4 id="2.2.4%C2%A0%E6%96%87%E4%BB%B6%E8%AE%BF%E9%97%AE%E6%9D%83%E9%99%90%E7%9A%84%E7%9B%B8%E5%85%B3%E8%AE%BE%E7%BD%AE%E6%96%B9%E6%B3%95%C2%A0" name="2.2.4%C2%A0%E6%96%87%E4%BB%B6%E8%AE%BF%E9%97%AE%E6%9D%83%E9%99%90%E7%9A%84%E7%9B%B8%E5%85%B3%E8%AE%BE%E7%BD%AE%E6%96%B9%E6%B3%95%C2%A0">
     <span style="color:#fe2c24">
      2.2.4 文件访问权限的相关设置方法
     </span>
    </h4>
    <h5 id="2.2.4.1%20chmod%E6%8C%87%E4%BB%A4%E6%9D%83%E9%99%90%E5%80%BC" name="2.2.4.1%20chmod%E6%8C%87%E4%BB%A4%E6%9D%83%E9%99%90%E5%80%BC">
     <span style="color:#4da8ee">
      <strong>
       2.2.4.1 chmod指令权限值
      </strong>
     </span>
    </h5>
    <p>
     <strong>
      语法：
     </strong>
     chmod 选项 权限 文件名或目录名
     <br/>
     <strong>
      功能：
     </strong>
     设置文件的访问权限。
    </p>
    <p>
     <strong>
      格式一：
     </strong>
     用户符号 +/-/= 权限字符
    </p>
    <p>
     1）+：向权限范围增加权限代号所表示的权限。
     <br/>
     2）-：向权限范围取消权限代号所表示的权限。
     <br/>
     3）=：向权限范围赋予权限代号所表示的权限。
     <br/>
     用户符号：
     <br/>
     1）u：拥有者。
     <br/>
     2）g：所属组。
     <br/>
     3）o：other。
     <br/>
     4）a：所有用户。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#956fe7">
      <strong>
       -rw-rw-r-- 1 cp   cp      0 Mar 12 12:08 test4.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ chmod o+wx test4.txt（指令修改）
      </strong>
     </span>
    </p>
    <p>
     修改后：
     <br/>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rwx 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
     <br/>
    </p>
    <p>
     <strong>
      若要同时设置不同类用户的访问权限，则需用逗号隔开。
     </strong>
    </p>
    <p>
     <span style="color:#0d0016">
      修改前：
     </span>
    </p>
    <p>
     <span style="color:#4da8ee">
      <strong>
       -rw-rw-rwx 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ chmod o-wx,u+x test4.txt（指令修改）
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      修改后：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rwxrw-r-- 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
     <br/>
    </p>
    <p>
     <strong>
      格式二：
     </strong>
     三位八进制数字
    </p>
    <p>
     将对应的八进制数转换为二进制，进而设置对应权限值。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#4da8ee">
      <strong>
       -rwxrw-r-- 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ chmod 667 test4.txt（指令修改）
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       这里的667是八进制数字，换成二进制就是110110100，就代表rw-rw-rwx。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      修改后：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rwx 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <h5 id="2.2.4.2%20%E6%94%B9%E5%8F%98%E6%96%87%E4%BB%B6%E7%9A%84%E6%8B%A5%E6%9C%89%E8%80%85" name="2.2.4.2%20%E6%94%B9%E5%8F%98%E6%96%87%E4%BB%B6%E7%9A%84%E6%8B%A5%E6%9C%89%E8%80%85">
     <span style="color:#4da8ee">
      <strong>
       2.2.4.2 改变文件的拥有者
      </strong>
     </span>
    </h5>
    <p>
     <strong>
      语法：
     </strong>
     chown 选项 用户名 文件名或目录名
     <br/>
     <strong>
      功能：
     </strong>
     修改文件的拥有者。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#956fe7">
      <strong>
       -rw-rw-rwx 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       修改指令：cp@hcss-ecs-348a:~$ sudo chown root test4.txt
       <br/>
       [sudo] password for cp:
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      修改后：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rwx 1 root cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       这里大家会发现这个文件的拥有者被改成了root。
      </strong>
     </span>
    </p>
    <p>
     <strong>
      注意：
     </strong>
     修改文件的拥有者需要root用户进行操作，若是普通用户则需要进行权限提升。
    </p>
    <p>
    </p>
    <p>
     也可以
     <strong>
      使用chown指令同时修改文件的拥有者和所属组
     </strong>
     ，将拥有者和所属组的用户名用冒号隔开即可。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#956fe7">
      <strong>
       -rw-rw-rwx 1 cp   cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       修改指令：cp@hcss-ecs-348a:~$ sudo chown root:root test4.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      修改后：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rwx 1 root root    0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <h5 id="2.2.4.3%20%E4%BF%AE%E6%94%B9%E6%96%87%E4%BB%B6%E7%9A%84%E6%89%80%E5%B1%9E%E7%BB%84" name="2.2.4.3%20%E4%BF%AE%E6%94%B9%E6%96%87%E4%BB%B6%E7%9A%84%E6%89%80%E5%B1%9E%E7%BB%84">
     <span style="color:#4da8ee">
      2.2.4.3 修改文件的所属组
     </span>
    </h5>
    <p>
     <strong>
      语法：
     </strong>
     chgrp 选项 用户名 文件名或目录名
     <br/>
     <strong>
      功能：
     </strong>
     修改文件的所属组。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#956fe7">
      <strong>
       -rw-rw-rwx 1 root root    0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       修改指令：cp@hcss-ecs-348a:~$ sudo chgrp cp test4.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      修改后：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rwx 1 root cp      0 Mar 12 12:08 test4.txt*
      </strong>
     </span>
    </p>
    <h5 id="2.2.4.4%20%E4%BF%AE%E6%94%B9%E6%96%87%E4%BB%B6%E7%9A%84%E6%9D%83%E9%99%90%E6%8E%A9%E7%A0%81" name="2.2.4.4%20%E4%BF%AE%E6%94%B9%E6%96%87%E4%BB%B6%E7%9A%84%E6%9D%83%E9%99%90%E6%8E%A9%E7%A0%81">
     <span style="color:#4da8ee">
      <strong>
       2.2.4.4 修改文件的权限掩码
      </strong>
     </span>
    </h5>
    <p>
     <span style="color:#0d0016">
      这里大家需要思考一个问题，为什么新建的文件和目录都有默认的权限？
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       drwxrwxr-x 2 cp   cp   4096 Mar 12 12:02 mytest/
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-r-- 1 cp   cp      0 Mar 12 18:06 test6.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      这里我们看到的权限是最终权限，而
     </span>
     <span style="color:#511b78">
      <strong>
       最终权限=默认权限&amp;(~umask)
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      在Linux中，
     </span>
     新建文件的默认权限为666，新建目录的默认权限为777；
    </p>
    <p>
     <img alt="" height="440" src="https://i-blog.csdnimg.cn/direct/dc0a18c18959437b8bd3e1ee14ec5af2.png" width="1164"/>
    </p>
    <p>
     <span style="color:#0d0016">
      这里提到的umask是系统内默认存在的一个叫作
      <strong>
       权限掩码
      </strong>
      的概念：
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      我们可以通过指令来查看我们系统权限掩码：
     </span>
    </p>
    <p>
     <strong>
      语法：
     </strong>
     umask 权限值
     <br/>
     <strong>
      功能：
     </strong>
     查看或修改文件掩码。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ umask
       <br/>
       0002
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       这里我的系统的权限掩码就是0002。
      </strong>
     </span>
    </p>
    <p>
     <img alt="" height="201" src="https://i-blog.csdnimg.cn/direct/5ca4f05ba3d14d358a9a4c7a8c09318d.png" width="956"/>
     <br/>
    </p>
    <p>
     <img alt="" height="398" src="https://i-blog.csdnimg.cn/direct/0b9441ed23924bf4accf85e370262691.png" width="1157"/>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       那么权限掩码的目的是什么呢？
      </strong>
     </span>
    </p>
    <p>
     其实通过上面的图，大家可以发现其目的就在于
     <span style="color:#be191c">
      <strong>
       希望权限掩码中出现的权限不在最终权限中出现。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       那为什么要设计权限掩码呢？
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      第一点在于，最终权限由操作系统自主决定，无法在创建前进行修改——系统可配置、可灵活满足需要的一种表现。
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      第二点在于，特殊情况下，我们可以通过配置umask来控制文件的最终权限，让代码实现可控。
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      有了上面的铺垫，想必大家也可以想到，我们可以通过改变权限掩码来设置文件的访问权限：
     </span>
     <br/>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ umask 000
      </strong>
     </span>
     <br/>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ touch data.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       -rw-rw-rw- 1 cp   cp      0 Mar 12 18:26 data.txt
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       大家可以看到，data.txt文件的最终权限是受到权限掩码的影响的。
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~$ umask 777
       <br/>
       cp@hcss-ecs-348a:~$ mkdir file
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       d--------- 2 cp   cp   4096 Mar 12 18:28 file/
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       这里同上，我们创建的新目录file的
      </strong>
     </span>
     <span style="color:#511b78">
      <strong>
       最终权限是受到权限掩码的影响的。
      </strong>
     </span>
    </p>
    <p>
     <strong>
      注意：
     </strong>
     超级用户的默认掩码为0022，普通用户的默认掩码为0002。
    </p>
    <h4 id="2.2.5%C2%A0%E7%9B%AE%E5%BD%95%E7%9A%84%E6%9D%83%E9%99%90" name="2.2.5%C2%A0%E7%9B%AE%E5%BD%95%E7%9A%84%E6%9D%83%E9%99%90">
     <span style="color:#fe2c24">
      2.2.5 目录的权限
     </span>
    </h4>
    <p>
     <span style="color:#0d0016">
      对于文件来说，其可读可写可执行的属性我们都知道分别代表着什么对应的操作，那对于目录来说可读可写可执行又分别代表着什么呢？
      <br/>
      1）可读权限： 如果用户没有该目录的可读权限，则无法通过ls指令查看目录中的文件内容。
      <br/>
      2）可写权限： 如果用户没有该目录的可写权限，则无法通过一系列指令在目录中创建文件或删除文件。
      <br/>
      3）可执行权限： 如果用户没有该目录的可执行权限，则无法通过cd指令进入到目录当中。
     </span>
    </p>
    <p>
     <strong>
      那么这就会出现一个问题：
     </strong>
     <br/>
     只要用户拥有某目录的可写权限，就可以删除该目录当中的文件，而不论该用户是否拥有该文件的可写权限，这显然是不合理的。
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       drwxr-xrwx 2 root root 4096 Mar 12 12:14 dir/
      </strong>
     </span>
     <br/>
     <span style="color:#ff9900">
      <strong>
       cp@hcss-ecs-348a:~$ cd dir
       <br/>
       cp@hcss-ecs-348a:~/dir$ touch test7.txt
       <br/>
       cp@hcss-ecs-348a:~/dir$ ll
       <br/>
       total 8
       <br/>
       drwxr-xrwx 2 root root 4096 Mar 12 18:42 ./
       <br/>
       drwxr-x--- 5 cp   cp   4096 Mar 12 18:28 ../
      </strong>
     </span>
     <br/>
     <span style="color:#0d0016">
      <strong>
       -r--r--r-- 1 cp   cp      0 Mar 12 18:48 test7.txt（这里经过处理，让我们对其没有“写”权限）
      </strong>
     </span>
     <br/>
     <span style="color:#ff9900">
      <strong>
       cp@hcss-ecs-348a:~/dir$ rm -f test7.txt
       <br/>
       cp@hcss-ecs-348a:~/dir$ ll
       <br/>
       total 8
       <br/>
       drwxr-xrwx 2 root root 4096 Mar 12 18:43 ./
       <br/>
       drwxr-x--- 5 cp   cp   4096 Mar 12 18:28 ../
      </strong>
     </span>
    </p>
    <p>
     大家发现，我们对dir目录来说是other，我们拥有对其写的权限，那么我在dir目录下创建一个新文件test7.txt，此时大家注意这个文件没有写权限，但是我们还是能将它删除，所以这个现象就比较奇怪了，我不能“写”这个文件，但是却可以直接删除它，为了解决这个不合理的问题，Linux引入了
     <strong>
      粘滞位
     </strong>
     的概念。
    </p>
    <h4 id="2.2.6%20%E7%B2%98%E6%BB%9E%E4%BD%8D" name="2.2.6%20%E7%B2%98%E6%BB%9E%E4%BD%8D">
     <span style="color:#fe2c24">
      2.2.6 粘滞位
     </span>
    </h4>
    <p>
     <strong>
      语法：
     </strong>
     chmod +t 目录名
     <br/>
     <strong>
      功能：
     </strong>
     给目录加上粘滞位。
    </p>
    <p>
     当一个用户将某一个目录加上粘滞位后，该目录的权限值的最后一位变为字符“t”。
    </p>
    <p>
     修改前：
    </p>
    <p>
     <span style="color:#4da8ee">
      <strong>
       drwxr-xrwx 2 root root 4096 Mar 12 18:56 dir/
      </strong>
     </span>
     <br/>
     <span style="color:#511b78">
      <strong>
       修改指令：root@hcss-ecs-348a:/home/cp# chmod +t dir
      </strong>
     </span>
    </p>
    <p>
     修改后：
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       drwxr-xrwt 2 root root 4096 Mar 12 18:56 dir/
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      此时我们再次来删除文件，大家会看到下面的报错：
     </span>
    </p>
    <p>
     <span style="color:#511b78">
      <strong>
       cp@hcss-ecs-348a:~/dir$ rm -f test7.txt
       <br/>
       rm: cannot remove 'test7.txt': Operation not permitted
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#be191c">
      这里就是粘滞位起到作用了，我们无法像上面那样去删除文件test7.txt了。
     </span>
    </p>
    <p>
     <span style="color:#be191c">
      总结：
     </span>
    </p>
    <p>
     当一个目录被设置为粘滞位，则该目录下的文件只能由：
     <br/>
     1）超级用户删除。
     <br/>
     2）该目录的拥有者删除。
     <br/>
     3）该文件的拥有者删除。
    </p>
