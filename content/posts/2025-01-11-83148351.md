     1.腾讯的GT测试工具,使用方便,无需联网

    <h3>
     什么是GT？
    </h3>
    <p>
     GT（随身调）是APP的随身调试平台，它是直接运行在手机上的“集成调试环境”(IDTE, Integrated Debug Environment)。
    </p>
    <p>
     利用GT，仅凭一部手机，无需连接电脑，即可对APP进行快速的性能测试(CPU、内存、流量、电量、帧率/流畅度等等)、开发日志的查看、Crash日志查看、网络数据包的抓取、APP内部参数的调试、真机代码耗时统计等。
    </p>
    <p>
     如果您觉得GT提供的功能还不够满足您的需要，您还可以利用GT提供的基础API自行开发有特殊功能的GT插件，帮助您解决更加复杂的APP调试问题  ,有专门的APP测试讨论群
    </p>
    <p>
     github地址:
     <a href="https://github.com/Tencent/GT">
      https://github.com/Tencent/GT
     </a>
    </p>

    <h3>
     1, 官方工具
    </h3>
    <p>
     一般来说, 学习一门新的技术, 最应该做的就是阅读其官方文档, 那是最权威的.
     <br/>
     Android本身给我们提供了很多App性能测试和分析工具, 而且大部分都集成到Android Studio或DDMS中, 非常方便使用.
    </p>
    <h4>
     1.1 StrictMode
    </h4>
    <ul>
     <li>
      <p>
       说明
       <br/>
       顾名思义, "严格模式", 主要用来限制应用做一些不符合性能规范的事情. 一般用来检测主线程中的耗 时操作和阻塞. 开启StrictMode后, 如果线程中做一些诸如读写文件, 网络访问等操作, 将会在Log console输出一些警告, 警告信息包含Stack Trace来显示哪个地方出了问题.
      </p>
     </li>
     <li>
      <p>
       文档
      </p>
      <ul>
       <li>
        <a href="https://link.jianshu.com?t=https://developer.android.com/reference/android/os/StrictMode.html" rel="nofollow">
         https://developer.android.com/reference/android/os/StrictMode.html
        </a>
       </li>
      </ul>
     </li>
     <li>
      <p>
       作用
      </p>
      <ul>
       <li>
        主要用来做主线程优化分析
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     1.2 Systrace
    </h4>
    <ul>

      <br/>
      其以trace文件(html)的方式记录. 可以直接用Chrome浏览器打开查看. 界面如下:
     </li>
    </ul>

    <h4>
     1.3 Hierarchy Viewer
    </h4>
    <ul>
     <li>
      说明
      <br/>
      Hierarchy Viewer提供了一个可视化的界面来观测布局的层级, 让我们可以优化布局层级, 删除多余的不必要的View层级, 提升布局速度.
     </li>
    </ul>
    <p>
     <img alt="" class="has" src="//upload-images.jianshu.io/upload_images/851999-62c25b5d13bbe59d.png?imageMogr2/auto-orient/strip%7CimageView2/2/w/962/format/webp"/>
    </p>
    <p>
     Hierarchy Viewer
    </p>
    <p>
     有必要说明下的是:
     <br/>
     上图红框标出的三个点是关键分析数据. 左起依次代表View的Measure, Layout和Draw的性能. 另外颜色表示该View的该项时间指数, 分为:
     <br/>
     * 绿色, 表示该View的此项性能比该View Tree中超过50%的View都要快.
     <br/>
     * 黄色, 表示该View的此项性能比该View Tree中超过50%的View都要慢.
     <br/>
     * 红色, 表示该View的此项性能是View Tree中最慢的.
    </p>
    <ul>
     <li>
      <p>
       文档
      </p>
      <ul>
       <li>
        <a href="https://link.jianshu.com?t=https://developer.android.com/studio/profile/hierarchy-viewer.html" rel="nofollow">
         https://developer.android.com/studio/profile/hierarchy-viewer.html
        </a>
       </li>
       <li>
        <a href="https://link.jianshu.com?t=https://developer.android.com/studio/profile/hierarchy-viewer-walkthru.html" rel="nofollow">
         https://developer.android.com/studio/profile/hierarchy-viewer-walkthru.html
        </a>
       </li>
       <li>
        <a href="https://link.jianshu.com?t=https://developer.android.com/studio/profile/hierarchy-viewer-setup.html" rel="nofollow">
         https://developer.android.com/studio/profile/hierarchy-viewer-setup.html
        </a>
       </li>
       <li>
        <a href="https://link.jianshu.com?t=https://developer.android.com/studio/profile/optimize-ui.html#HierarchyViewer" rel="nofollow">
         https://developer.android.com/studio/profile/optimize-ui.html#HierarchyViewer
        </a>
       </li>
      </ul>
     </li>
     <li>
      <p>
       作用
      </p>
      <ul>
       <li>
        用来做View层级分析, 可以分析出View Tree中的性能阻塞点, 以便对症下药, 提升布局性能.
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      Hierarchy Viewer需要Root的机器(产品机没有开启ViewServer)才可以执行.
      <br/>
      可以使用
      <a href="https://link.jianshu.com?t=https://github.com/romainguy/ViewServer">
       第三方的开源的ViewServer
      </a>
      来协助我们在未Root的机器上使用Hierarchy Viewer分析.
