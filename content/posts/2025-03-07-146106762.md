    <p>
     不小心把工作区的代码全删掉了
    </p>
    <p>
     首先是名字出错，不知为何gitee任意把我的名字更改。
    </p>
    <p>
     导致无法push验证
    </p>
    <pre>git push -u origin "master</pre>
    <p>
     显示：fatal: Authentication failed for 'https://gitee.com/zhang-great/stm32-smart-security-system.git/'
    </p>
    <p>
     我以为需要拉取在gitee创建的仓库。
    </p>
    <p>
     就 git pull origin master.
    </p>
    <p>
     最后又看网上的博客 说要输入 git pull --rebase origin master
    </p>
    <p>
     <code>
      --rebase
     </code>
     选项表示以
     <strong>
      变基（Rebase）
     </strong>
     的方式整合远程分支的更新到本地分支。它的核心作用是
     <strong>
      重新调整本地提交的基准。 这样就把与拉取的远程仓库不一致的文件都删了。
     </strong>
    </p>
    <p>
     <strong>
      之后又 git add . 导致彻底地删除了。
     </strong>
    </p>
    <p>
     <strong>
      随后输入 git status 。  发现已经是删除的状态。
     </strong>
    </p>
    <p>
     <img alt="" height="278" src="https://i-blog.csdnimg.cn/direct/b6daf38473494585aaed003681691b0d.png" width="378"/>
    </p>
    <p>
     我问了Deepseek 这样如何恢复。
    </p>
    <p>
     <img alt="" height="354" src="https://i-blog.csdnimg.cn/direct/2aa520e9c5fb4463ba89e87a5564df1c.png" width="1208"/>
    </p>
    <p>
     先输入 git reset HEAD 文件夹   回退到上一阶段。 HEAD 是 Git 用来跟踪
     <strong>
      当前检出的提交或分支
     </strong>
     的指针。 可用于回退
    </p>
    <p>
     之后 输入 git checkout -- 文件夹    就恢复成功。checkout  丢弃工作区中某个文件的未暂存修改，恢复到最近一次
     <code>
      git commit
     </code>
     或
     <code>
      git add
     </code>
     的状态。
    </p>
    <p>
     因此文件恢复过来。
    </p>
    <p>
     git的正确使用办法
    </p>
    <p>
     Git 全局设置:
    </p>
    <pre>git config --global user.name 名字
git config --global user.email emial </pre>
    <p>
     创建 git 仓库:
    </p>
    <pre>mkdir asa
cd asa
git init 
touch README.md
git add README.md
git commit -m "first commit"
git remote add origin https://gitee.com/zhang-great/asa.git
git push -u origin "master"</pre>
