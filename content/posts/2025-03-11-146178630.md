    <h4>
     <a id="1__sshpass_2">
     </a>
     1. 安装
     <code>
      sshpass
     </code>
    </h4>
    <p>
     <code>
      sshpass
     </code>
     是一个可以自动输入密码的工具。如果未安装，运行以下命令安装：
     <br/>
     • 对于 Debian/Ubuntu 系统：
    </p>
    <pre><code class="prism language-bash"><span class="token function">apt</span> update <span class="token operator">&amp;&amp;</span> <span class="token function">apt</span> <span class="token function">install</span> sshpass
</code></pre>
    <p>
     • 对于 CentOS/RHEL 系统：
    </p>
    <pre><code class="prism language-bash">yum <span class="token function">install</span> sshpass
</code></pre>
    <h4>
     <a id="2__rsync__15">
     </a>
     2. 编写
     <code>
      rsync
     </code>
     命令
    </h4>
    <p>
     假设您需要将远程服务器的
     <code>
      /root/vanblog
     </code>
     目录同步到本地的
     <code>
      /root/vanblog
     </code>
     目录，命令如下：
    </p>
    <pre><code class="prism language-bash">/usr/bin/sshpass <span class="token parameter variable">-p</span> <span class="token string">'your_password'</span> /usr/bin/rsync <span class="token parameter variable">-avz</span> <span class="token parameter variable">-e</span> <span class="token string">"ssh -i /root/id_rsa"</span> root@11.11.11.11:/root/vanblog /root/vanblog
</code></pre>
    <p>
     将
     <code>
      your_password
     </code>
     替换为您的 SSH 密码。
    </p>
    <h4>
     <a id="3__crontab__22">
     </a>
     3. 添加
     <code>
      crontab
     </code>
     任务
    </h4>
    <p>
     编辑
     <code>
      crontab
     </code>
     文件：
    </p>
    <pre><code class="prism language-bash"><span class="token function">crontab</span> <span class="token parameter variable">-e</span>
</code></pre>
    <p>
     添加以下行，每天下午 15:00 执行任务：
    </p>
    <pre><code class="prism language-bash"><span class="token number">0</span> <span class="token number">15</span> * * * /usr/bin/sshpass <span class="token parameter variable">-p</span> <span class="token string">'your_password'</span> /usr/bin/rsync <span class="token parameter variable">-avz</span> <span class="token parameter variable">-e</span> <span class="token string">"ssh -i /root/id_rsa"</span> root@11.11.11.11:/root/vanblog /root/vanblog <span class="token operator">&gt;&gt;</span> /root/rsync.log <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span>
</code></pre>
    <p>
     将
     <code>
      your_password
     </code>
     替换为您的 SSH 密码。
    </p>
    <h4>
     <a id="4__33">
     </a>
     4. 验证任务
    </h4>
    <p>
     查看
     <code>
      crontab
     </code>
     任务：
    </p>
    <pre><code class="prism language-bash"><span class="token function">crontab</span> <span class="token parameter variable">-l</span>
</code></pre>
    <p>
     等待 15:00 后，检查
     <code>
      /root/rsync.log
     </code>
     文件，确认任务是否执行成功。
    </p>
    <hr/>
    <h3>
     <a id="_42">
     </a>
     常见问题
    </h3>
    <h4>
     <a id="1_cron__44">
     </a>
     1.
     <code>
      cron
     </code>
     服务未找到
    </h4>
    <p>
     如果运行
     <code>
      systemctl status cron
     </code>
     时提示
     <code>
      Unit cron.service could not be found
     </code>
     ，检查
     <code>
      crond
     </code>
     服务：
    </p>
    <pre><code class="prism language-bash">systemctl status crond
</code></pre>
    <p>
     如果未运行，启动并启用它：
    </p>
    <pre><code class="prism language-bash">systemctl start crond
systemctl <span class="token builtin class-name">enable</span> crond
</code></pre>
    <h4>
     <a id="2__55">
     </a>
     2. 任务未执行
    </h4>
    <p>
     • 检查
     <code>
      cron
     </code>
     服务是否正常运行：
    </p>
    <pre><code class="prism language-bash">systemctl status crond
</code></pre>
    <p>
     • 确保任务路径正确，使用绝对路径。
     <br/>
     • 查看日志文件（如
     <code>
      /var/log/cron
     </code>
     或
     <code>
      /var/log/syslog
     </code>
     ）排查问题。
    </p>
