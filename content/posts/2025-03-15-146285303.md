    <h2>
     <a id="_1">
     </a>
     <strong>
      排序算法对比：快速排序、归并排序、堆排序
     </strong>
    </h2>
    <h3>
     <a id="1_Quick_Sort_3">
     </a>
     <strong>
      1. 快速排序（Quick Sort）
     </strong>
    </h3>
    <h4>
     <a id="_4">
     </a>
     <strong>
      原理
     </strong>
    </h4>
    <p>
     快速排序采用
     <strong>
      分治法
     </strong>
     （Divide and Conquer），通过选取基准值（pivot），将数组划分为
     <strong>
      小于基准值
     </strong>
     和
     <strong>
      大于基准值
     </strong>
     的两个部分，并递归排序。
    </p>
    <h4>
     <a id="_7">
     </a>
     <strong>
      特点
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       时间复杂度
      </strong>
      ：
      <ul>
       <li>
        最优：
        <code>
         O(n log n)
        </code>
       </li>
       <li>
        平均：
        <code>
         O(n log n)
        </code>
       </li>
       <li>
        最差：
        <code>
         O(n²)
        </code>
        （当选取的
        <code>
         pivot
        </code>
        总是最小或最大值时，会退化成冒泡排序）
       </li>
      </ul>
     </li>
     <li>
      <strong>
       空间复杂度
      </strong>
      ：
      <code>
       O(log n)
      </code>
      （递归调用栈）
     </li>
     <li>
      <strong>
       是否稳定
      </strong>
      ：❌
      <strong>
       不稳定
      </strong>
      （交换过程中可能改变相同元素的相对位置）
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：
      <ul>
       <li>
        适用于大规模数据排序
       </li>
       <li>
        数据较随机时性能较优
       </li>
       <li>
        <strong>
         不适用于数据接近有序
        </strong>
        或
        <strong>
         需要稳定性的场景
        </strong>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2_Merge_Sort_21">
     </a>
     <strong>
      2. 归并排序（Merge Sort）
     </strong>
    </h3>
    <h4>
     <a id="_22">
     </a>
     <strong>
      原理
     </strong>
    </h4>
    <p>
     归并排序同样采用
     <strong>
      分治法
     </strong>
     ，将数组拆分成
     <strong>
      左右两部分
     </strong>
     ，分别排序后再
     <strong>
      合并（merge）
     </strong>
     ，确保整体有序。
    </p>
    <h4>
     <a id="_25">
     </a>
     <strong>
      特点
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       时间复杂度
      </strong>
      ：
      <ul>
       <li>
        <strong>
         最佳、最差、平均
        </strong>
        ：
        <code>
         O(n log n)
        </code>
        （即使是最坏情况下也能保持
        <code>
         O(n log n)
        </code>
        ）
       </li>
      </ul>
     </li>
     <li>
      <strong>
       空间复杂度
      </strong>
      ：
      <code>
       O(n)
      </code>
      （额外存储归并时的数组）
     </li>
     <li>
      <strong>
       是否稳定
      </strong>
      ：✅
      <strong>
       稳定
      </strong>
      （归并过程不会打乱相同元素的相对顺序）
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：
      <ul>
       <li>
        适用于
        <strong>
         数据量大且需要稳定性
        </strong>
        的情况
       </li>
       <li>
        适用于
        <strong>
         链表排序
        </strong>
       </li>
       <li>
        适用于
        <strong>
         外部排序（大数据排序）
        </strong>
        ，因其可并行处理数据
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="3_Heap_Sort_37">
     </a>
     <strong>
      3. 堆排序（Heap Sort）
     </strong>
    </h3>
    <h4>
     <a id="_38">
     </a>
     <strong>
      原理
     </strong>
    </h4>
    <p>
     堆排序基于
     <strong>
      二叉堆（Binary Heap）
     </strong>
     数据结构，先将数组构造成
     <strong>
      最大堆（Max Heap）
     </strong>
     ，然后依次取出堆顶元素（最大值），调整堆结构，最终得到一个有序数组。
    </p>
    <h4>
     <a id="_41">
     </a>
     <strong>
      特点
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       时间复杂度
      </strong>
      ：
      <ul>
       <li>
        最优、平均、最差：
        <code>
         O(n log n)
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       空间复杂度
      </strong>
      ：
      <code>
       O(1)
      </code>
      （原地排序，不需要额外空间）
     </li>
     <li>
      <strong>
       是否稳定
      </strong>
      ：❌
      <strong>
       不稳定
      </strong>
      （堆调整过程中可能改变相同元素的相对位置）
     </li>
     <li>
      <strong>
       适用场景
      </strong>
      ：
      <ul>
       <li>
        适用于
        <strong>
         大规模数据排序
        </strong>
       </li>
       <li>
        适用于
        <strong>
         对最坏情况有较好保证
        </strong>
        的场景
       </li>
       <li>
        适用于
        <strong>
         优先队列的实现
        </strong>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>

    <hr/>
    <h3>
     <a id="5__63">
     </a>
     <strong>
      5. 选择哪种排序？
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       快速排序
      </strong>
      ：一般情况下最快，适用于数据规模大、数据随机分布的情况。
     </li>
     <li>
      <strong>
       归并排序
      </strong>
      ：适用于数据量大、
      <strong>
       稳定性要求高
      </strong>
      的情况，如数据库排序。
     </li>
     <li>
      <strong>
       堆排序
      </strong>
      ：适用于
      <strong>
       大规模数据排序
      </strong>
      ，适用于
      <strong>
       时间复杂度需要稳定的情况
      </strong>
      。
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
    </p>
    <ul>
     <li>
      如果
      <strong>
       数据量大
      </strong>
      ，推荐
      <strong>
       快速排序
      </strong>
      （但注意避免最坏情况）。
     </li>
     <li>
      如果
      <strong>
       要求稳定排序
      </strong>
      ，推荐
      <strong>
       归并排序
      </strong>
      。
     </li>
     <li>
      如果
      <strong>
       空间受限
      </strong>
      ，推荐
      <strong>
       堆排序
      </strong>
      （因为它是
      <code>
       O(1)
      </code>
      空间复杂度的
      <code>
       O(n log n)
      </code>
      排序算法）。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      总结：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       快速排序
      </strong>
      是平均情况下最快的
      <code>
       O(n log n)
      </code>
      排序，但最坏情况下退化为
      <code>
       O(n²)
      </code>
      。
     </li>
     <li>
      <strong>
       归并排序
      </strong>
      <strong>
       始终
      </strong>
      是
      <code>
       O(n log n)
      </code>
      ，但需要额外
      <code>
       O(n)
      </code>
      空间，是稳定排序。
     </li>
     <li>
      <strong>
       堆排序
      </strong>
      适用于
      <strong>
       大数据且需要
       <code>
        O(1)
       </code>
       额外空间
      </strong>
      ，但不稳定。
     </li>
    </ol>
