    <h3>
     <a id="_2">
     </a>
     <strong>
      一、代理模式概述
     </strong>
    </h3>
    <p>
     代理模式（Proxy Pattern）是一种结构型设计模式，它允许一个对象（代理）来控制对另一个对象（目标对象）的访问。
    </p>
    <h4>
     <a id="_5">
     </a>
     <strong>
      代理模式的作用
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       控制访问
      </strong>
      ：用于权限控制，例如用户权限验证。
     </li>
     <li>
      <strong>
       延迟初始化
      </strong>
      ：按需创建对象，优化资源使用。
     </li>
     <li>
      <strong>
       日志记录
      </strong>
      ：拦截方法调用，记录日志。
     </li>
     <li>
      <strong>
       远程代理
      </strong>
      ：用于分布式系统，如WCF、gRPC。
     </li>
     <li>
      <strong>
       缓存代理
      </strong>
      ：缓存数据库查询结果，提高性能。
     </li>
    </ul>
    <h4>
     <a id="_12">
     </a>
     <strong>
      代理模式的关键角色
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       抽象主题（Subject）
      </strong>
      ：定义代理和真实对象的公共接口。
     </li>
     <li>
      <strong>
       真实主题（RealSubject）
      </strong>
      ：执行实际业务逻辑的类。
     </li>
     <li>
      <strong>
       代理类（Proxy）
      </strong>
      ：持有真实对象的引用，并控制对其访问。
     </li>
    </ol>

    <h4>
     <a id="_66">
     </a>
     <strong>
      优缺点
     </strong>
    </h4>
    <p>
     ✅
     <strong>
      优点
     </strong>
    </p>
    <ul>
     <li>
      代码结构清晰，便于管理。
     </li>
     <li>
      适用于权限控制、日志等场景。
     </li>
    </ul>
    <p>
     ❌
     <strong>
      缺点
     </strong>
    </p>
    <ul>
     <li>
      需要手动编写代理类，代码量增加。
     </li>
     <li>
      代理类与真实类耦合度高。
     </li>
    </ul>

    <h3>
     <a id="_vs__167">
     </a>
     <strong>
      四、静态代理 vs. 动态代理
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        静态代理
       </th>
       <th>
        动态代理
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        代码量
       </td>
       <td>
        需要手写代理类
       </td>
       <td>
        代码复用高
       </td>
      </tr>
      <tr>
       <td>
        灵活性
       </td>
       <td>
        低，需手动编写代理
       </td>
       <td>
        高，代理类可动态生成
       </td>
      </tr>
      <tr>
       <td>
        性能开销
       </td>
       <td>
        低
       </td>
       <td>
        运行时反射有一定开销
       </td>
      </tr>
      <tr>
       <td>
        适用场景
       </td>
       <td>
        日志、权限控制
       </td>
       <td>
        AOP、远程调用
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <a id="_177">
     </a>
     <strong>
      五、总结
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       静态代理
      </strong>
      适用于少量代理对象，代码清晰但不够灵活。
     </li>
     <li>
      <strong>
       动态代理
      </strong>
      适用于需要大量代理、AOP编程，灵活但需要额外的运行时开销。
     </li>
     <li>
      选择哪种方式，取决于具体的需求和性能考虑。
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      推荐使用动态代理实现AOP，如日志、事务管理等，减少重复代码，提高可维护性。
     </strong>
    </p>
