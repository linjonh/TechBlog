# comfyUI笔记1-开启comfyUI

系统ubuntu

- 激活一个名为 comfyui 的 Conda 环境，这通常是为了确保运行程序时所需的依赖项都在正确的位置。

`conda activate comfyui`

- 进入到 ComfyUI 程序的根目录。  
`cd ~/ComfyUI`

- 启动 ComfyUI 程序，并且使用了 --cpu 参数，这表示程序将使用 CPU 来运行，而不是 GPU。  
`python main.py --cpu`

- default location  
http://127.0.0.1:8188/

- 这部分笔记是 ComfyUI 界面里的操作。在程序里添加了一个叫做 "Load Checkpoint" 或 "CheckpointLoaderSimple" 的节点，这个节点就是用来加载模型文件的。  
> Load Checkpoint  
>: CheckpointLoaderSimple

问题：模型文件识别问题

Q：遇到的问题是，虽然已经有了 sd-v1-4-test.safetensors 这个模型文件，但在 ComfyUI 界面中却找不到它。

A：解决这个问题的方法：

- 执行下面步骤，就能识别到sd-v1-4-test.safetensors了：

1. 创建一个名为 checkpoints 的新文件夹，它位于 ~/ComfyUI/models/ 目录下。  
`mkdir -p ~/ComfyUI/models/checkpoints`  
2. 将你的模型文件 移动 到了刚刚创建的 checkpoints 文件夹里。  
`mv ~/ComfyUI/models/Stable-diffusion/sd-v1-4/sd-v1-4-test.safetensors ~/ComfyUI/models/checkpoints/  
`

备注： ComfyUI 的默认设置

ComfyUI 默认会在特定的文件夹里寻找不同类型的文件。对于 模型（checkpoint） 文件，它会去 ComfyUI/models/checkpoints/ 这个目录下找。

所以，当把 sd-v1-4-test.safetensors 文件移动到正确的位置后，ComfyUI 就能顺利识别并加载它了。
