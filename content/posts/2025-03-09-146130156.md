    <p>
     程序猿大佬有言：”
     <code>
      天下方法共十斗，GET 独占八斗，POST 占一斗，剩下的方法占一斗。
     </code>
     “
    </p>
    <p>
     因此，我们只要掌握了
     <code>
      GET
     </code>
     和
     <code>
      POST
     </code>
     ，足以应付绝大多数工作情景。
    </p>

     <font color="gree">
      <font face="华文楷体">
       <font size="6">
        <strong>
         1. GET方法
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     <code>
      GET
     </code>
     是最常用的 HTTP 方法，常用于
     <code>
      获取服务器上的某个资源
     </code>
     ；
    </p>
    <p>
     在浏览器中直接
     <code>
      输入URL
     </code>
     ，此时
     <code>
      浏览器
     </code>
     就会发送出一个
     <code>
      GET请求
     </code>
     ；
    </p>
    <p>
     另外，HTML中的
     <code>
      link
     </code>
     <code>
      img
     </code>
     <code>
      script
     </code>
     等标签，也会
     <code>
      触发GET请求
     </code>
     ；
    </p>
    <blockquote>
     <p>
      后面我们还会学习，使用 JavaScript 中的
      <code>
       ajax
      </code>
      也能
      <code>
       构造 GET请求
      </code>
     </p>
    </blockquote>
    <hr/>
    <table>
     <tbody>
      <tr>
       <td bgcolor="pink">
        <font color="red">
         <font face="华文楷体">
          <font size="4">
           <strong>
            使用强制刷新操作，让 fiddler 抓取到包含静态资源的请求结果
           </strong>
          </font>
         </font>
        </font>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <blockquote>
     <p>
      在平时的 fiddler 中，抓包的数据中基本都是
      <code>
       HTML
      </code>
      ，基本没有
      <code>
       CSS
      </code>
      或者
      <code>
       JS
      </code>
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5628cc42132b4a3f8233e130b5cba955.png"/>
     </p>
     <hr/>
     <p>
      这时候就有一个小技巧，可以回到浏览器中，按住
      <code>
       ctrl后再刷新页面
      </code>
      ，重新打开 fiddler：
     </p>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a11b97fd86d547ddae248ec8ee879352.png"/>
     </p>
     <p>
      此时就会发现，fiddler 抓包明显抓到了更多的 css 和 js
     </p>
     <hr/>
     <p>
      <strong>
       原理
      </strong>
      ：
     </p>
     <p>
      F5 表示
      <code>
       刷新
      </code>
      ，而 ctrl+F5 表示的是
      <code>
       强制刷新
      </code>
      ；
     </p>
     <hr/>
     <p>
      <strong>
       引入浏览器的缓存机制
      </strong>
      ：
     </p>
     <ul>
      <li>
       浏览器是从
       <code>
        服务器
       </code>
       ，也就是通过
       <code>
        网络
       </code>
       加载页面的；
      </li>
      <li>
       我们知道，缓存数据的速度为
       <code>
        CPU&gt;内存&gt;硬盘&gt;网络带宽
       </code>
       ，所以通常来说，
       <code>
        从网络带宽中加载数据的速度是慢于硬盘的
       </code>
       ；
      </li>
      <li>
       浏览器为了加快访问网页的速度，就会把页面依赖的一些
       <code>
        静态资源
       </code>
       （css，js，图片，字体，mp3…）缓存到
       <code>
        硬盘
       </code>
       上；
      </li>
      <li>
       因此在我们第一次访问服务器时，需要加载包含静态资源的数据；
      </li>
      <li>
       <font color="red">
        这些静态资源会在第一次访问服务器时，加载到硬盘上；后续再次访问，就不会再重新加载静态资源的数据了；
       </font>
      </li>
     </ul>
     <hr/>
     <ul>
      <li>
       <p>
        如果要在后续的访问中，
        <code>
         重新加载静态资源
        </code>
        ，就需要使用强制刷新操作；
       </p>
      </li>
      <li>
       <p>
        <font color="red">
         强制刷新会忽略本地缓存，所有资源都从服务器中重新获取一遍；
        </font>
       </p>
      </li>
      <li>
       <p>
        因此使用强制刷新操作，我们就可以使用 fiddler 抓取到
        <code>
         更丰富的请求结果
        </code>
        ；
       </p>
      </li>
     </ul>
    </blockquote>

    <hr/>
    <p>
     <strong>
      GET :
     </strong>
    </p>
    <blockquote>
     <p>
      <code>
       首行的第一部分
      </code>
      为 GET
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      header
     </strong>
     ：
    </p>
    <blockquote>
     <p>
      header 部分有若干个
      <code>
       键值对
      </code>
      结构；
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      body ：
     </strong>
    </p>
    <blockquote>
     <p>
      <code>
       body 部分为空
      </code>
      ：
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      URL query string :
     </strong>
    </p>
    <blockquote>
     <p>
      因为 body 部分为空，如果需要通过 GET
      <code>
       给服务器发送数据
      </code>
      ，可以
      <code>
       通过 query string 传递
      </code>
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      关于 GET 请求的 URL 长度问题：
     </strong>
    </p>
    <blockquote>
     <p>
      网上有些资料上描述：get请求长度最多 1024kb 这样的说法是错误的.
     </p>
     <hr/>
     <p>
      HTTP协议由
      <code>
       RFC2616
      </code>
      标准定义：
     </p>
     <blockquote>
      <p>
       "Hypertext Transfer Protocol–HTTP/1.1,“does not specify any requirement for URL length.”
      </p>
     </blockquote>
     <p>
      <code>
       标准原文中明确说明，没有对 URL 的长度有任何的限制
      </code>
      ；
     </p>
     <hr/>
     <ul>
      <li>
       <p>
        实际 URL 的长度，取决于
        <code>
         浏览器的实现
        </code>
        和
        <code>
         HTTP服务器端的实现
        </code>
        ；
       </p>
      </li>
      <li>
       <p>
        在浏览器端，
        <code>
         不同的浏览器最大长度是不同的
        </code>
        ，但是现代浏览器支持的长度一般都很长；
       </p>
      </li>
      <li>
       <p>
        在
        <code>
         服务器端
        </code>
        ，一般
        <code>
         这个长度是可以配置的
        </code>
        .
       </p>
      </li>
     </ul>
    </blockquote>

        <strong>
         2. POST方法
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     POST 方法也是一种常见的方法，多用于
     <code>
      提交用户输入的数据给服务器
     </code>
     (例如
     <code>
      登陆页面
     </code>
     或者
     <code>
      上传数据
     </code>
     ).
    </p>
    <p>
     通过HTML中的
     <code>
      form标签
     </code>
     可以构造POST请求，或者使用JavaScript的
     <code>
      ajax
     </code>
     也可以构造POST请求;
    </p>

     <strong>
      URL query string

     <p>

     <strong>

      <code>

      </code>
      结构；
     </p>

     <strong>
      body ：
     </strong>
    </p>

       <code>

       </code>
       ：
      </li>
      <li>
       body 内的
       <code>
        数据格式
       </code>
       通过 header 中的
       <code>
        Content-Type
       </code>

      </li>
      <li>
       body的
       <code>
        长度
       </code>

       <code>
        Content-Length
       </code>
       指定；
      </li>
     </ul>
    </blockquote>

        <strong>
         3. 经典面试题：谈谈GET和POST的区别
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <table>
     <tbody>
      <tr>
       <td bgcolor="pink">
        <font color="red">
         <font face="华文楷体">
          <font size="4">
           <strong>
            通常情况
           </strong>
          </font>
         </font>
        </font>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <p>
     <strong>
      从HTTP的定义来看
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       GET
      </strong>
      ：
      <ul>
       <li>
        GET 用于
        <code>
         获取资源
        </code>
        ；
       </li>
       <li>
        通常情况下，GET
        <code>
         用于请求数据
        </code>
        而
        <code>
         不改变服务器状态
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       POST
      </strong>
      ：
      <ul>
       <li>
        POST 用于
        <code>
         提交数据到服务器
        </code>
        ；
       </li>
       <li>
        通常情况下，
        <code>
         会改变服务器的状态
        </code>
        或
        <code>
         产生副作用
        </code>
        （如创建或更新资源）。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <p>
     由于
     <code>
      HTTP
     </code>
     和
     <code>
      浏览器
     </code>
     等规定，它们在应用过程中会出现一些区别:
    </p>
    <p>
     <strong>
      参数传递方式
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        GET
       </strong>
       ：
      </p>
      <ul>
       <li>
        参数放在 query String 中，
        <code>
         通过 URL 拼接传递
        </code>
       </li>
       <li>
        暴露在请求 URL中，具有
        <code>
         可见性
        </code>
        ，
        <code>
         长度有限
        </code>
        （取决于浏览器和服务器)。
       </li>
       <li>
        body 一般为空，需要传递的数据通过query string传递
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        POST
       </strong>
       ：
      </p>
      <ul>
       <li>
        参数
        <code>
         放在请求体 body 中
        </code>
       </li>
       <li>
        通常
        <code>
         不可见
        </code>
        ，
        <code>
         长度理论上没有限制
        </code>
        ；
       </li>
       <li>
        query string 一般为空，需要传递的数据通过 body 传递
       </li>
       <li>
        更适合传递大量数据（但是注意，POST也可以在URL上放参数！)。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      安全性（对于登录场景）
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        GET
       </strong>
       ：
      </p>
      <ul>
       <li>
        参数可见，数据
        <code>
         容易暴露在浏览器历史记录、日志和缓存中
        </code>
       </li>
       <li>
        不适合传递敏感信息。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        POST
       </strong>
       ：
      </p>
      <ul>
       <li>
        数据放在请求体中，相对安全，但
        <code>
         需要 HTTPS 才能保证数据加密传输
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      幂等性（如果多次请求得到的结果一样，就视为请求是幂等的）
     </strong>
     ：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        GET
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         幂等
        </code>
        （重复请求不会改变服务器状态）；
       </li>
       <li>
        GET 设计成幂等，就可以
        <code>
         允许 GET 请求被缓存
        </code>
        ；
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        POST
       </strong>
       ：
      </p>
      <ul>
       <li>
        <code>
         非幂等
        </code>
        （多次请求可能导致重复创建资源或执行多次相同操作）；
       </li>
       <li>
        POST 由于不要求幂等，所以 POST 经常是不幂等的，因此
        <code>
         POST 请求不能被缓存
        </code>
        ；
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <table>
     <tbody>
      <tr>
       <td bgcolor="pink">
        <font color="red">
         <font face="华文楷体">
          <font size="4">
           <strong>
            补充说明
           </strong>
          </font>
         </font>
        </font>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <blockquote>
     <p>
      <strong>
       关于语义：
      </strong>
     </p>
     <ul>
      <li>
       GET完全可以用于
       <code>
        提交数据
       </code>
       ，POST也完全可以用于
       <code>
        获取数据
       </code>
      </li>
     </ul>
     <hr/>
     <p>
      <strong>
       关于幂等性：
      </strong>
     </p>
     <ul>
      <li>
       标准建议 GET 实现为幂等的；
      </li>
      <li>
       实际开发中 GET 也不必完全遵守这个规则
      </li>
      <li>
       主流网站都有
       <code>
        "猜你喜欢"
       </code>
       功能，会根据用户的历史行为，实时更新现有的结果；
      </li>
     </ul>
     <hr/>
     <p>
      <strong>
       关于安全性：
      </strong>
     </p>
     <ul>
      <li>
       有些资料上说"POST比GET请安全"，这样的说法是不科学的；
      </li>
     </ul>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a6f43fa1e7eb47b78a2db4de8d2c24ea.png"/>
     </p>
     <ul>
      <li>
       是否安全取决于
       <code>
        前端在传输密码等敏感信息时，是否进行加密
       </code>
       ，和GET/ POST无关；
      </li>
     </ul>
     <hr/>
     <p>
      <strong>
       关于传输数据量：
      </strong>
     </p>
     <ul>
      <li>
       有的资料上说" GET传输的数据量小，POST传输数据量大 "，这个也是不科学的；
      </li>
     </ul>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/833397668a4b4ace9fef80d5803661d6.png"/>
     </p>
     <ul>
      <li>
       标准没有规定 GET 的 URL 的长度，也没有规定 POST 的 body 的长度；
      </li>
      <li>
       传输数据量多少，完全取决于
       <code>
        不同浏览器和不同服务器之间的实现区别
       </code>
       ；
      </li>
     </ul>
     <hr/>
     <p>
      <strong>
       关于传输数据类型：
      </strong>
     </p>
     <ul>
      <li>
       <p>
        有的资料上说"GET 只能传输文本数据，POST 可以传输二进制数据"，这个也是不科学的；
       </p>
      </li>
      <li>
       <p>
        GET 的 URL 确实只能放文本（
        <code>
         query string
        </code>
        无法直接传输二进制数据），但是可以针对二进制数据进行urlencode（通过
        <code>
         base64
        </code>
        把二进制转码成文本）；
       </p>
      </li>
     </ul>
    </blockquote>
    <hr/>
    <h4>
     <a id="font_color__greefont_facefont_size6strong4_strongfontfontfont_345">
     </a>
     <font color="gree">
      <font face="华文楷体">
       <font size="6">
        <strong>
         4. 其他方法
        </strong>
       </font>
      </font>
     </font>
    </h4>
    <hr/>
    <p>
     <strong>
      PUT
     </strong>
    </p>
    <ul>
     <li>
      与POST相似，只是具有幂等特性，一般用于更新
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      DELETE
     </strong>
    </p>
    <ul>
     <li>
      删除服务器指定资源
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/0d992736cb40444aa44bace0ef4c8814.png"/>
    </p>
    <hr/>
    <p>
     <strong>
      OPTIONS
     </strong>
    </p>
    <ul>
     <li>
      返回服务器所支持的请求方法
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      HEAD
     </strong>
    </p>
    <ul>
     <li>
      类似于GET，只不过响应体不返回，只返回响应头
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      TRACE
     </strong>
    </p>
    <ul>
     <li>
      回显服务器端收到的请求，测试的时候会用到这个
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      CONNECT
     </strong>
    </p>
    <ul>
     <li>
      预留，暂无使用
     </li>
    </ul>
    <hr/>
    <p>
     这些方法的
     <code>
      HTTP请求
     </code>
     可以使用
     <code>
      ajax
     </code>
     来构造 (也可以通过一些第三方工具)
    </p>
    <blockquote>
     <ul>
      <li>
       <p>
        任何一个能进行网络编程的语言，都可以构造HTTP请求；
       </p>
      </li>
      <li>
       <p>
        本质上就是通过
        <code>
         TCP socket
        </code>
        写入一个符合
        <code>
         HTTP 协议
        </code>
        规则的
        <code>
         字符串
        </code>
        .
       </p>
      </li>
     </ul>
    </blockquote>
