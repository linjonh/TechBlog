     <strong>
      2、安装Git
     </strong>
     ‌：
    </h3>
    <p>
     双击下载的安装包，按照提示进行安装。选择安装路径时，建议使用默认路径（如C:\Program Files\Git），除非您有特殊需求。
    </p>
    <p>
     在安装过程中，建议选择默认配置，包括git命令行、图形窗口等选项。对于编辑器选择，可以使用Vim或选择现代的GUI编辑器如Notepad++等。
    </p>
    <p>
     完成安装后，可以在桌面右键菜单中打开Git的命令行窗口，或者通过命令提示符输入
    </p>
    <pre><code class="language-bash">git --version</code></pre>
    <p>
     来验证是否安装成功。‌
    </p>

     四、git常用指令
    </h2>
    <h3 id="1%E3%80%81%E4%BB%93%E5%BA%93%E5%88%9D%E5%A7%8B%E5%8C%96%E4%B8%8E%E7%AE%A1%E7%90%86" name="1%E3%80%81%E4%BB%93%E5%BA%93%E5%88%9D%E5%A7%8B%E5%8C%96%E4%B8%8E%E7%AE%A1%E7%90%86">
     1、仓库初始化与管理
    </h3>
    <ol>
     <li>
      <strong>
       git init
      </strong>
      ：在当前目录初始化一个新的Git仓库。
     </li>
     <li>
      <strong>
       git clone [url]
      </strong>
      ：克隆远程仓库到本地，[url]为远程仓库的地址。
     </li>
    </ol>
    <h3 id="2%E3%80%81%E9%85%8D%E7%BD%AE" name="2%E3%80%81%E9%85%8D%E7%BD%AE">
     2、配置
    </h3>
    <ol>
     <li>
      <strong>
       git config --list
      </strong>
      ：显示当前的Git配置。
     </li>
     <li>
      <strong>
       git config -e [--global]
      </strong>
      ：编辑Git配置文件。
     </li>
     <li>
      <strong>
       git config [--global] user.name "[name]"
      </strong>
      ：设置提交代码时的用户名。
     </li>
     <li>
      <strong>
       git config [--global] user.email "[email address]"
      </strong>
      ：设置提交代码时的邮箱地址。
     </li>
    </ol>
    <h3 id="3%E3%80%81%E5%B7%A5%E4%BD%9C%E5%8C%BA%E4%B8%8E%E6%9A%82%E5%AD%98%E5%8C%BA%E7%AE%A1%E7%90%86" name="3%E3%80%81%E5%B7%A5%E4%BD%9C%E5%8C%BA%E4%B8%8E%E6%9A%82%E5%AD%98%E5%8C%BA%E7%AE%A1%E7%90%86">
     3、工作区与暂存区管理
    </h3>
    <ol>
     <li>
      <strong>
       git status
      </strong>
      ：查看当前工作目录下文件的状态。
     </li>
     <li>
      <strong>
       git add [file]
      </strong>
      ：将文件添加到暂存区。使用
      <code>
       .
      </code>
      代替[file]可添加当前目录中的所有文件。
     </li>
     <li>
      <strong>
       git reset [file]
      </strong>
      ：重置暂存区的指定文件，与上一次commit保持一致，但工作区不变。
     </li>
     <li>
      <strong>
       git reset --hard
      </strong>
      ：重置暂存区与工作区，与上一次commit保持一致。
     </li>
     <li>
      <strong>
       git rm [file]
      </strong>
      ：删除工作区文件，并且将这次删除放入暂存区。
     </li>
     <li>
      <strong>
       git mv [current path] [new path]
      </strong>
      ：移动或重命名文件。
     </li>
    </ol>
    <h3 id="4%E3%80%81%E6%8F%90%E4%BA%A4" name="4%E3%80%81%E6%8F%90%E4%BA%A4">
     4、提交
    </h3>
    <ol>
     <li>
      <strong>
       git commit -m "[message]"
      </strong>
      ：将暂存区的文件提交到仓库，并附上一个说明性的消息。
     </li>
     <li>
      <strong>
       git commit -a -m "[message]"
      </strong>
      ：提交自上次commit之后工作区的所有变化，直接到仓库区。
     </li>
     <li>
      <strong>
       git commit --amend -m "[message]"
      </strong>
      ：使用一次新的commit，替代上一次提交。如果代码没有任何新变化，则用来改写上一次commit的提交信息。
     </li>
    </ol>
    <h3 id="5%E3%80%81%E5%88%86%E6%94%AF%E7%AE%A1%E7%90%86" name="5%E3%80%81%E5%88%86%E6%94%AF%E7%AE%A1%E7%90%86">
     5、分支管理
    </h3>
    <ol>
     <li>
      <strong>
       git branch
      </strong>
      ：查看当前仓库的所有分支。
     </li>
     <li>
      <strong>
       git branch -a
      </strong>
      ：查看本地和远程的所有分支。
     </li>
     <li>
      <strong>
       git branch -r
      </strong>
      ：查看远程的所有分支。
     </li>
     <li>
      <strong>
       git checkout [branch]
      </strong>
      ：切换到指定分支。
     </li>
     <li>
      <strong>
       git checkout -b [branch]
      </strong>
      ：创建一个新分支并切换到该分支。
     </li>
     <li>
      <strong>
       git branch -D [branch]
      </strong>
      ：删除一个分支。
     </li>
     <li>
      <strong>
       git merge [branch]
      </strong>
      ：将指定分支的修改合并到当前分支。
     </li>
     <li>
      <strong>
       git rebase [branch]
      </strong>
      ：将一系列提交移动或组合到新的基本提交上。
     </li>
    </ol>
    <h3 id="6%E3%80%81%E8%BF%9C%E7%A8%8B%E4%BB%93%E5%BA%93%E7%AE%A1%E7%90%86" name="6%E3%80%81%E8%BF%9C%E7%A8%8B%E4%BB%93%E5%BA%93%E7%AE%A1%E7%90%86">
     6、远程仓库管理
    </h3>
    <ol>
     <li>
      <strong>
       git remote add [remote name] [url]
      </strong>
      ：添加远程仓库。
     </li>
     <li>
      <strong>
       git remote -v
      </strong>
      ：查看当前远程地址。
     </li>
     <li>
      <strong>
       git remote set-url origin [new-remote-url]
      </strong>
      ：修改远程仓库地址。
     </li>
     <li>
      <strong>
       git remote remove [remote name]
      </strong>
      ：删除远程仓库。
     </li>
     <li>
      <strong>
       git remote rename [old name] [new name]
      </strong>
      ：重命名远程存储库。
     </li>
     <li>
      <strong>
       git fetch [remote name]
      </strong>
      ：从远程仓库获取更改。
     </li>
     <li>
      <strong>
       git pull [remote name] [branch]
      </strong>
      ：从远程仓库拉取最新的修改并合并到当前分支。
     </li>
     <li>
      <strong>
       git push [-u origin] [branch]
      </strong>
      ：将本地仓库的修改推送到远程仓库。
     </li>
    </ol>
    <h3 id="7%E3%80%81%E7%89%88%E6%9C%AC%E6%8E%A7%E5%88%B6" name="7%E3%80%81%E7%89%88%E6%9C%AC%E6%8E%A7%E5%88%B6">
     7、版本控制
    </h3>
    <ol>
     <li>
      <strong>
       git log
      </strong>
      ：查看提交日志，包括提交信息、时间等。
     </li>
     <li>
      <strong>
       git diff
      </strong>
      ：查看文件的修改内容。使用
      <code>
       --staged
      </code>
      标志可查看暂存区的修改。
     </li>
     <li>
      <strong>
       git reset [commit]
      </strong>
      ：恢复到指定的提交，同时将该提交之后的修改移除。
     </li>
     <li>
      <strong>
       git revert [commit]
      </strong>
      ：创建一个新的提交来撤销之前的更改。
     </li>
     <li>
      <strong>
       git checkout [commit id]
      </strong>
      ：查看之前的提交。
     </li>
    </ol>
    <h3 id="8%E3%80%81%E5%85%B6%E4%BB%96%E9%AB%98%E7%BA%A7%E6%93%8D%E4%BD%9C" name="8%E3%80%81%E5%85%B6%E4%BB%96%E9%AB%98%E7%BA%A7%E6%93%8D%E4%BD%9C">
     8、其他高级操作
    </h3>
    <ol>
     <li>
      <strong>
       git stash
      </strong>
      ：保存当前工作目录的修改，可以在需要的时候再恢复。
     </li>
     <li>
      <strong>
       git stash list
      </strong>
      ：列出所有存储的工作进度。
     </li>
     <li>
      <strong>
       git stash apply [stash id]
      </strong>
      ：恢复之前保存的工作进度。
     </li>
     <li>
      <strong>
       git stash pop [stash id]
      </strong>
      ：恢复并删除指定的工作进度。
     </li>
     <li>
      <strong>
       git stash drop [stash id]
      </strong>
      ：删除指定的工作进度。
     </li>
     <li>
      <strong>
       git stash clear
      </strong>
      ：删除所有的工作进度。
     </li>
     <li>
      <strong>
       git tag
      </strong>
      ：用于标记代码库中重要的版本或里程碑的指针。
     </li>
    </ol>
