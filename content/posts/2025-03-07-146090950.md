     三、延迟等级深度优化
    </h3>
    <h4>
     <a id="1__85">
     </a>
     1. 时间梯度设计原则
    </h4>
    <pre><code class="prism language-mermaid">gantt
    title RocketMQ延迟等级时间分布
    dateFormat  X
    axisFormat %s秒
    
    section 短期延迟
    等级1 : 0, 1
    等级2 : 1, 5
    等级3 : 5, 10
    等级4 : 10, 30
    
    section 中期延迟
    等级5 : 30, 60
    等级6 : 60, 120
    等级7 : 120, 180
    
    section 长期延迟
    等级8 : 180, 300
    ...
    等级18 : 7200, 7200
</code></pre>
    <h5>
     <a id="_109">
     </a>
     设计考量：
    </h5>
    <ul>
     <li>
      <strong>
       指数退避
      </strong>
      ：满足常见业务场景
     </li>
     <li>
      <strong>
       存储效率
      </strong>
      ：固定等级减少元数据开销
     </li>
     <li>
      <strong>
       扫描性能
      </strong>
      ：减少时间轮复杂度
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2__116">
     </a>
     2. 自定义延迟等级配置
    </h4>
    <pre><code class="prism language-properties"># broker.conf
messageDelayLevel=1s 5s 10s 30s 1m 2m 3m 4m 5m 6m 7m 8m 9m 10m 20m 30m 1h 2h

     </a>
     调整注意事项：
    </h5>

     <li>
      需要集群所有Broker统一配置
     </li>
     <li>
      修改后需重启生效
     </li>
     <li>
      等级数量不超过原始18级
     </li>
    </ol>
