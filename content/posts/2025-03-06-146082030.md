    <h4>
     <a id="_391">
     </a>
     异或数列(位运算)
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/10993beb8b1e45b1b23683eab2bb3c4e.png"/>
     <br/>
     示例 1
    </p>
    <blockquote>
     <p>
      输入
      <br/>
      4
      <br/>
      1 1
      <br/>
      1 0
      <br/>
      2 2 1
      <br/>
      7 992438 1006399 781139 985280 4729 872779 563580
     </p>
    </blockquote>
    <blockquote>
     <p>
      输出
      <br/>
      1
      <br/>
      0
      <br/>
      1
      <br/>
      1
     </p>
    </blockquote>
    <p>
     <mark>
      解题思路
     </mark>
    </p>
    <p>
     我们设在游戏结束后 Alice 的数变为
     <code>
      c
     </code>
     ，Bob 的数变为
     <code>
      d
     </code>
     。
    </p>
    <p>
     我们先来解决平局的情况：
    </p>
    <p>
     根据异或性质可得：若
     <code>
      c = d
     </code>
     ，则
     <code>
      c ⊕ d = 0
     </code>
     。
    </p>
    <p>
     而
     <code>
      c ⊕ d = X1 ⊕ X2 ⊕ ⋯ ⊕ Xn
     </code>
     ，所以要使
     <code>
      c = d
     </code>
     ，当且仅当
     <code>
      X1 ⊕ X2 ⊕ ⋯ ⊕ Xn = 0
     </code>
     。
    </p>
    <p>
     接下来定输赢：
    </p>
    <p>
     我们将
     <code>
      c, d
     </code>
     转换成二进制数。对于二进制数的比较，我们是从高位往低位开始的。所以要使自己的数最大，就需要从高位开始。
    </p>
    <p>
     设当前枚举到二进制的第
     <code>
      i
     </code>
     位。设
     <code>
      X1, X2, X3, ..., Xn
     </code>
     中一共有
     <code>
      cnt1
     </code>
     个数在该位的值为
     <code>
      1
     </code>
     ，
     <code>
      cnt2
     </code>
     个数在该位的值为
     <code>
      0
     </code>
     。（
     <code>
      cnt1 + cnt2 = n
     </code>
     ）
    </p>
    <p>
     <mark>
      结论一
     </mark>
     ：
    </p>
    <p>
     如果
     <code>
      cnt1
     </code>
     为偶数，则 Alice 和 Bob 无法在该位分出胜负。
    </p>
    <p>
     <strong>
      证明方法和上述平局情况相同。
     </strong>
     或者也可以这么想：
     <code>
      cnt1
     </code>
     为偶数，那么 Alice 和 Bob 要么都从这
     <code>
      cnt1
     </code>
     个数中分到偶数个，要么 Alice 和 Bob 都在这
     <code>
      cnt1
     </code>
     个数中分到奇数个；所以无论怎么分配，
     <code>
      c, d
     </code>
     在该位的异或值都必然相同。
    </p>
    <p>
     反之当
     <code>
      cnt1
     </code>
     为奇数时，必然能决出胜负。证明方法和上述类似，就不再给出。
    </p>
    <p>
     那么
     <code>
      cnt1
     </code>
     为奇数时如何判断谁输谁赢呢？
    </p>
    <p>
     我们先定义，对于当前第
     <code>
      i
     </code>
     位，如果能让自己的数值从
     <code>
      0 → 1
     </code>
     ，或者能让对手的数值从
     <code>
      1 → 0
     </code>
     ，则自己的胜率
     <code>
      +1
     </code>
     ；如果让自己的数值从
     <code>
      1 → 0
     </code>
     ，或者让对手的数值从
     <code>
      0 → 1
     </code>
     ，则自己的胜率
     <code>
      -1
     </code>
     ；如果既不改变自己的数值，也不改变对手的数值，则自己的胜率不变。显然，游戏结束时，胜率越高的一方获胜。
    </p>
    <p>
     <mark>
      结论二
     </mark>
     ：
    </p>
    <p>
     当
     <code>
      cnt1
     </code>
     为奇数，
     <code>
      cnt2 = 0
     </code>
     时，先手必胜。
    </p>
    <p>
     <strong>
      证明：
     </strong>
    </p>
    <p>
     模拟一下可以发现先手后手走的每一步都必然是让自己胜率增加的一步。由于
     <code>
      cnt1
     </code>
     为奇数，所以先手可以比后手多走一步，所以先手的胜率必然会比后手高。
    </p>
    <p>
     那么什么情况下必然会使自己的胜率减少呢？即当自己的数值为
     <code>
      1
     </code>
     ，且对手的数值为
     <code>
      0
     </code>
     ，且公共数列中只有
     <code>
      1
     </code>
     可以选取时。
    </p>
    <p>
     <mark>
      结论三
     </mark>
     ：
    </p>
    <p>
     谁的胜率率先减少，则谁必败。
    </p>
    <p>
     <strong>
      证明：
     </strong>
     由于一方胜率减少了，所以可得公共数列中只有
     <code>
      1
     </code>
     可以选取，没有
     <code>
      0
     </code>
     可以选取。
    </p>
    <p>
     设胜率率先减少的
     <code>
      Alice
     </code>
     ，那么此时
     <code>
      Alice
     </code>
     和
     <code>
      Bob
     </code>
     的数值只有两种可能：
    </p>
    <ol>
     <li>
      <code>
       Alice
      </code>
      的数值为
      <code>
       0
      </code>
      ，
      <code>
       Bob
      </code>
      的数值为
      <code>
       0
      </code>
      ；
     </li>
     <li>
      <code>
       Alice
      </code>
      的数值为
      <code>
       1
      </code>
      ，
      <code>
       Bob
      </code>
      的数值为
      <code>
       1
      </code>
      。
     </li>
    </ol>
    <p>
     由于
     <code>
      Alice
     </code>
     和
     <code>
      Bob
     </code>
     的数值相同，所以公共序列中使用的
     <code>
      1
     </code>
     的个数必然为偶数，剩余的
     <code>
      1
     </code>
     的个数必然为奇数。且此时是
     <code>
      Bob
     </code>
     先手，根据结论二，
     <code>
      Bob
     </code>
     必胜，
     <code>
      Alice
     </code>
     必败，证明完毕。
    </p>
    <p>
     那么谁的胜率会先减少的呢？
    </p>
    <p>
     <mark>
      结论四
     </mark>
     ：
    </p>
    <p>
     当
     <code>
      cnt1
     </code>
     、
     <code>
      cnt2
     </code>
     为奇数时且
     <code>
      cnt1 &gt; 1
     </code>
     时，先手的胜率会率先减少。
    </p>
    <p>
     <strong>
      证明：
     </strong>
     当
     <code>
      cnt2
     </code>
     为奇数时，先手第一步只能选取
     <code>
      0
     </code>
     或是
     <code>
      1
     </code>
     ：
    </p>
    <ul>
     <li>
      若先手先取
      <code>
       1
      </code>
      则后手取
      <code>
       0
      </code>
      。此时先手的数值为
      <code>
       1
      </code>
      ，后手的数值为
      <code>
       0
      </code>
      。为了不让自己的胜率降低，先手只能取
      <code>
       0
      </code>
      ，而后手也接着取
      <code>
       0
      </code>
      。由于
      <code>
       0
      </code>
      个为奇数，所以先手将率先无法取
      <code>
       0
      </code>
      ，只能取
      <code>
       1
      </code>
      ，使得自己的胜率降低。
     </li>
     <li>
      若先手取
      <code>
       0
      </code>
      则后手也取
      <code>
       0
      </code>
      。此时场面还是
      <code>
       1
      </code>
      的个数为奇数，
      <code>
       0
      </code>
      的个数为奇数的情况。若先手率先取了
      <code>
       1
      </code>
      ，则就回到了上述的情况，先手必败。所以先手只能不断取
      <code>
       0
      </code>
      ，而后手也跟着不断取
      <code>
       0
      </code>
      。最后先手取完
      <code>
       0
      </code>
      ，将剩余奇数个
      <code>
       1
      </code>
      ，回到了结论三的情况。由于此时到了后手的轮次，所以先手必败。
     </li>
    </ul>
    <p>
     <mark>
      结论五
     </mark>
     ：
    </p>
    <p>
     当
     <code>
      cnt1 = 1
     </code>
     时，先手必胜。
    </p>
    <p>
     <strong>
      证明略。
     </strong>
    </p>
    <p>
     根据上述五个结论，模拟一遍即可。
    </p>

    <p>
     <mark>
      最大公因数的基本概念
     </mark>
     ：
     <br/>
     最大公因数（GCD）是指两个数的最大共有因子。比如，
     <code>
      gcd(12, 15) = 3
     </code>
     ，因为
     <code>
      12
     </code>
     和
     <code>
      15
     </code>
     都能被
     <code>
      3
     </code>
     整除，而没有比
     <code>
      3
     </code>
     更大的共同因子。
    </p>
    <p>
     <mark>
      思路分析
     </mark>
     ：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        最大公因数的性质
       </strong>
       ：
      </p>
      <ul>
       <li>
        假设我们有两个正整数
        <code>
         a
        </code>
        和
        <code>
         b
        </code>
        ，如果它们的 GCD 很大，那么这两个数的因子也应该尽量重合。
       </li>
       <li>
        如果我们选取
        <code>
         a = n
        </code>
        和
        <code>
         b = n / 2
        </code>
        ，那么它们的 GCD 通常会比较大。具体来说，
        <code>
         gcd(n, n/2)
        </code>
        总是
        <code>
         n/2
        </code>
        （这是因为
        <code>
         n/2
        </code>
        是
        <code>
         n
        </code>
        的因子，并且它们共享
        <code>
         n/2
        </code>
        作为共同因子）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        为什么选择
        <code>
         n
        </code>
        和
        <code>
         n / 2
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          选择
          <code>
           n
          </code>
          和
          <code>
           n / 2
          </code>
          作为候选
         </strong>
         ：
        </p>
        <ul>
         <li>
          如果我们选择两个数
          <code>
           a = n
          </code>
          和
          <code>
           b = n / 2
          </code>
          ，这两个数之间的最大公因数是
          <code>
           n / 2
          </code>
          。这是因为：
          <ul>
           <li>
            <code>
             n
            </code>
            是
            <code>
             n / 2
            </code>
            的倍数，
            <code>
             n
            </code>
            和
            <code>
             n / 2
            </code>
            的最大公因数就是
            <code>
             n / 2
            </code>
            。
           </li>
          </ul>
         </li>
         <li>
          例如，当
          <code>
           n = 10
          </code>
          时，
          <code>
           n = 10
          </code>
          和
          <code>
           n / 2 = 5
          </code>
          ，这两个数的 GCD 是 5。
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          为什么
          <code>
           n / 2
          </code>
          会是最大值
         </strong>
         ：
        </p>
        <ul>
         <li>
          当我们选择两个数时，我们希望它们的最大公因数尽可能大。
          <code>
           n / 2
          </code>
          是
          <code>
           n
          </code>
          最大的因子之一，所以选择
          <code>
           n
          </code>
          和
          <code>
           n / 2
          </code>
          总是能得到最大的 GCD。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        其他可能的组合
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果选择
        <code>
         a = n
        </code>
        和
        <code>
         b = n - 1
        </code>
        ，它们的最大公因数一般会较小，因为相邻的整数的 GCD 总是
        <code>
         1
        </code>

       </li>
       <li>
        同理，其他的一些数对，如
        <code>
         a = n
        </code>
        和
        <code>
         b = n - 2
        </code>

        <code>
         n / 2
        </code>

        <code>

       </li>
      </ul>
     </li>

    <p>
     暴力做法
     <br/>
     变量含义：
     <br/>
     • n 表示大楼数量，d 表示要查询的天数。
     <br/>
     • 数组 h 存储每栋大楼的高度，数组 t 存储每个查询日对应的海平面高度。
     <br/>
     • 布尔数组 st 标记某天是否“未被淹没”（true 为未淹没）。
    </p>
    <p>
     对每个查询日的处理：
     <br/>
     (1) 将大楼中高 &lt;= 当前海平面的全部标记为 false（表示已淹没）。
     <br/>
     (2) 随后扫描所有大楼，累积计算未淹没大楼所形成的连续“区域”数量：
    </p>
    <ul>
     <li>
      如果遇到一段连续的 true（未淹没大楼），则算作一个区域；
     </li>
     <li>
      当连续的 true 被一个 false（淹没大楼）打断时，再出现下一段 true 时，就会有一个新的区域。
      <br/>
      (3) 将最终区域数输出。
     </li>

    <p>
     双指针+排序
    </p>
    <ol>
     <li>
      首先，将所有大楼 (高度, 下标) 按高度从小到大排序，便于后续根据海平面高度逐步淹没。
     </li>
     <li>
      随后，有一个双指针循环：当海平面上升到 t[i] 时，就把所有高度 ≤ t[i] 的大楼“标记”为淹没（分别存储到 drown[i] 列表）。
     </li>
     <li>
      利用一个布尔数组 st[] 来标记下标为 x 的大楼是否已被淹没；给 0 与 n+1 这两个“边界”强制设为已淹没状态（true），方便识别某大楼左右是否都是淹没状态。
     </li>
     <li>
      遍历 drown[i]，对于每一个新淹没的大楼 x：
      <br/>
      • 若 x 左右均尚未淹没，则此次淹没会把原来的一个区域分割成两个，因此区域计数 ans 增加 1。
      <br/>
      • 若 x 左右均已经淹没，则原先的两个淹没区在 x 处“连接”为一个区，区域计数 ans 减少 1。
      <br/>
      • 最后将 x 标记为已淹没。
     </li>
     <li>
      每次处理完后，输出当前 ans 值，即此刻剩余未淹没大楼的整体区域数量。
     </li>
