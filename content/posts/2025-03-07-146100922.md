    <h1 class="title-article" id="articleContentId">
     redis的淘汰策略
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     Redis 的淘汰策略（Eviction Policy）是指当 Redis 内存不足时，如何选择删除哪些数据来腾出空间。以下是 Redis 支持的几种淘汰策略，用通俗的话解释它们的特点：
    </p>
    <hr/>
    <h4>
     1.
     <strong>
      noeviction（不淘汰）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         当内存不足时，新写入的数据会报错，旧数据不会被删除。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         数据非常重要，绝对不能丢失。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “内存满了？那就别写新数据了，直接报错！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     2.
     <strong>
      allkeys-lru（最近最少使用）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从所有 Key 中，删除最近最少使用的 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         适合大多数场景，优先保留最近常用的数据。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “谁最近没用过，就把谁踢出去！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     3.
     <strong>
      volatile-lru（最近最少使用，仅限过期 Key）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从设置了过期时间的 Key 中，删除最近最少使用的 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         只淘汰那些可能会过期的数据，保留永久数据。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “只踢那些可能会过期的，而且最近没用的！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     4.
     <strong>
      allkeys-random（随机淘汰）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从所有 Key 中，随机删除一个 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         数据没有明显的访问模式，随机淘汰即可。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “随便踢一个，看谁倒霉！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     5.
     <strong>
      volatile-random（随机淘汰，仅限过期 Key）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从设置了过期时间的 Key 中，随机删除一个 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         只淘汰那些可能会过期的数据，随机选择。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “只踢那些可能会过期的，随便踢一个！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     6.
     <strong>
      volatile-ttl（最短存活时间）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从设置了过期时间的 Key 中，删除剩余存活时间最短的 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         优先淘汰即将过期的数据。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “谁快过期了，就先踢谁！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     7.
     <strong>
      allkeys-lfu（最少频率使用）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从所有 Key 中，删除访问频率最低的 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         适合访问频率有明显差异的场景。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “谁用得最少，就把谁踢出去！”
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     8.
     <strong>
      volatile-lfu（最少频率使用，仅限过期 Key）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         从设置了过期时间的 Key 中，删除访问频率最低的 Key。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         只淘汰那些可能会过期的数据，优先淘汰访问频率低的。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        通俗解释
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         “只踢那些可能会过期的，而且用得最少的！”
        </p>
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     10.
     <strong>
      如何选择淘汰策略？
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       如果数据非常重要，不能丢失，选择
       <strong>
        noeviction
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       如果数据有明确的访问模式，选择
       <strong>
        LRU
       </strong>
       或
       <strong>
        LFU
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       如果数据没有明显的访问模式，选择
       <strong>
        random
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       如果数据有过期时间，选择
       <strong>
        volatile
       </strong>
       相关的策略。
      </p>
     </li>
    </ul>
