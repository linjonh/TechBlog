# 前后端联调小细节

前端向后端发数据，有时候前端和后端的字段是一样的，字段没有问题，前端发过来的载荷也没有问题，但后端接收到的字段是null的，这时候可以排查前端发过来的数据是不是多嵌套了一层，比如发过来的是。

实例如下

```javascript
data: data
// 上面和下面两种方式接收是不一样的，下面这种方式多嵌套了一层
data: {
	data
}
```

多嵌套了一层导致后端对应的controller的方法的方法形参

使用@RequestBody注解接收的实例对象对应不上前端发过来的数据
