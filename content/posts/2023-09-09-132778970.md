    <h2>
     <a id="_1">
     </a>
     前言
    </h2>
    <p>
     大量的项目，使用不同的版本，打包部署极大地浪费了时间和成本。
    </p>
    <h2>
     <a id="Python_3">
     </a>
     一、Python虚拟环境是什么？
    </h2>
    <p>
     Python虚拟环境是一个独立的、隔离的Python运行环境，允许你在同一台机器上同时管理和运行多个不同版本的Python及其包/库。
    </p>
    <h2>
     <a id="_5">
     </a>
     二、环境版本
    </h2>
    <blockquote>
     <p>
      window 10
      <br/>
      Python 3.10.0
      <br/>
      Visual Studio Code 1.82
     </p>
    </blockquote>
    <h2>
     <a id="_10">
     </a>
     三、搭建步骤
    </h2>
    <h3>
     <a id="1_11">
     </a>
     1.创建工作空间
    </h3>
    <p>
     创建一个工作空间：名为为vsspace；
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/30bebb100d9dcd8d8ce221dbfbd214d9.png"/>
    </p>
    <h3>
     <a id="2_14">
     </a>
     2.创建虚拟环境
    </h3>
    <p>
     在vs命令行（terminal）中输入命令:
    </p>
    <blockquote>
     <p>
      python -m venv vsvenv
     </p>
    </blockquote>
    <p>
     创建Python虚拟环境文件夹 vsvenv；
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0c1e84db59ebc4a330d7ee839060d8dd.png">
      <br/>
      创建完成后，左侧菜单会出现一个虚拟目录列表，也就是项目的虚拟环境；
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/bc3264b9c1f2b785c06753cb2879914e.png"/>
     </img>
    </p>
    <h3>
     <a id="3_21">
     </a>
     3.激活虚拟环境
    </h3>
    <ol>
     <li>
      在命令行中输入命令cd .\vsvenv\Scripts，进入虚拟环境目录；
     </li>
     <li>
      在命令行中输入命令 .\Activate.ps1 ，激活当前虚拟环境；
     </li>
    </ol>
    <blockquote>
     <p>
      如出现如下错误，按照下方步骤操作：
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/297e79b2aec84dc531b3a43643eff609.png">
       <br/>
       a. 使用管理员运行PowerShell
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a739d09ff5218c677c3cc4a930f0c11d.png">
        <br/>
        b. 输入命令：Get-ExecutionPolicy，输出的是：”Restricted"；
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7499d43e913bc8cf8e0d5564a86a01c0.png">
         <br/>
         c. 输入命令：Set-ExecutionPolicy RemoteSigned，并在输入框中输入“Y”；
         <br/>
         <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c0d7c6250620c6f82e66f1bd841614da.png">
          <br/>
          d. 输入命令：Get-ExecutionPolicy，输出的是：”RemoteSigned"；
          <br/>
          <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e8fc5d9ff1412759b4d29a16a5656668.png"/>
         </img>
        </img>
       </img>
      </img>
     </p>
    </blockquote>
    <ol start="3">
     <li>
      再次输入 .\Activate.ps1命令，出现（vsvenv）虚拟目录，到此就算激活成功了。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/82d1de7cd84b2591691f35c6d988d604.png"/>
      <br/>
      至此，Python虚拟环境搭建完毕。在虚拟环境中pip安装的第三方库都只作用于该虚拟环境，不影响系统环境。
     </li>
    </ol>

    <h2>
     <a id="_48">
     </a>
     五、总结
    </h2>
    <ol>
     <li>
      包管理与隔离：使用虚拟环境可以方便地管理和隔离不同项目的包依赖关系。每个虚拟环境都有独立的Python解释器和包安装目录，这意味着你可以在不同的虚拟环境中安装和管理不同版本的包，而不会相互干扰或冲突。
     </li>
     <li>
      版本控制：通过创建不同的虚拟环境，你可以轻松切换Python版本。每个虚拟环境都可以使用不同的Python版本，并且可以在项目之间进行切换，而无需全局更改Python版本。
     </li>
     <li>
      简化部署：使用虚拟环境可以简化项目的部署过程。你可以将虚拟环境与项目一起打包，并在其他机器上轻松地重新创建相同的开发环境。这样可以确保项目在不同环境中具有相同的依赖关系，减少了配置和依赖问题。
     </li>
     <li>
      操作灵活性：通过激活或退出虚拟环境，你可以快速切换项目之间的工作环境。这样你可以有效地管理多个项目，而无需全局更改设置或手动安装特定版本的包。
     </li>
     <li>
      清理和维护：在虚拟环境中安装的包不会影响全局Python环境，这使得清理和维护变得更加方便。如果你不再需要某个虚拟环境，只需删除相应的目录即可。
     </li>
    </ol>
