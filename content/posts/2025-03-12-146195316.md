    <p>
     桥接模式（Bridge Pattern）是一种
     <strong>
      结构型设计模式
     </strong>
     ，其核心思想是
     <strong>
      将抽象（Abstraction）与实现（Implementation）分离，使它们可以独立变化
     </strong>
     。通过组合而非继承的方式，桥接模式解决了多层继承带来的类爆炸问题，并提高了系统的扩展性。
    </p>
    <h4>
     <strong>
      1. 核心角色
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        角色
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         抽象化（Abstraction）
        </strong>
       </td>
       <td>
        定义抽象接口，并持有
        <strong>
         实现化对象
        </strong>
        的引用。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         扩展抽象化（Refined Abstraction）
        </strong>
       </td>
       <td>
        对抽象化的扩展，提供更具体的业务逻辑。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         实现化接口（Implementor）
        </strong>
       </td>
       <td>
        定义实现类的接口，供抽象化角色调用。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         具体实现化（Concrete Implementor）
        </strong>
       </td>
       <td>
        实现实现化接口的具体类。
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      2. 应用场景
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        多维度的独立变化
       </strong>
       ：当一个类存在多个独立变化的维度，且需要灵活组合时（如手机品牌与手机软件）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        避免类爆炸
       </strong>
       ：防止因多层继承导致子类数量指数级增长。
      </p>
     </li>
     <li>
      <p>
       <strong>
        运行时动态切换实现
       </strong>
       ：允许在运行时切换抽象与实现的具体组合。
      </p>
     </li>
    </ul>

    <h4>
     <strong>
      5. 桥接模式优缺点
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         优点
        </strong>
       </th>
       <th>
        <strong>
         缺点
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        解耦抽象与实现，独立扩展维度。
       </td>
       <td>
        增加系统设计复杂度。
       </td>
      </tr>
      <tr>
       <td>
        避免多层继承，减少子类数量。
       </td>
       <td>
        需正确识别系统中独立变化的维度。
       </td>
      </tr>
      <tr>
       <td>
        支持运行时动态组合抽象与实现。
       </td>
       <td>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <strong>
      6. 桥接模式 vs. 适配器模式
     </strong>
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         模式
        </strong>
       </th>
       <th>
        <strong>
         目的
        </strong>
       </th>
       <th>
        <strong>
         应用阶段
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         桥接模式
        </strong>
       </td>
       <td>
        分离抽象与实现，允许独立扩展。
       </td>
       <td>
        系统设计阶段预先规划。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适配器模式
        </strong>
       </td>
       <td>
        解决接口不兼容问题，使类协同工作。
       </td>
       <td>
        系统维护阶段兼容已有代码。
       </td>
      </tr>
     </tbody>
    </table>

    <h4>
     <strong>
      7. 实际应用场景
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        JDBC 驱动程序
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          抽象化
         </strong>
         ：
         <code>
          Connection
         </code>
         接口（如
         <code>
          java.sql.Connection
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          实现化
         </strong>
         ：不同数据库的驱动实现（如
         <code>
          MySQLConnection
         </code>
         、
         <code>
          OracleConnection
         </code>
         ）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        GUI 跨平台框架
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          抽象化
         </strong>
         ：窗口组件（如按钮、文本框）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          实现化
         </strong>
         ：不同操作系统的绘制实现（如Windows、Linux）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        消息通知系统
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          抽象化
         </strong>
         ：消息类型（文本、图片）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          实现化
         </strong>
         ：发送渠道（短信、邮件、App推送）。
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      8. 总结
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        核心价值
       </strong>
       ：通过组合替代继承，解决多维度变化的扩展问题。
      </p>
     </li>
     <li>
      <p>
       <strong>
        关键设计
       </strong>
       ：抽象化类持有实现化接口的引用，通过委托调用实现功能。
      </p>
     </li>
     <li>
      <p>
       <strong>
        适用性
       </strong>
       ：当系统中存在多个独立变化的维度，且需要灵活组合时优先考虑桥接模式。
      </p>
     </li>
    </ul>
