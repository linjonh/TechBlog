    <h4>
     <a id="11__24">
     </a>
     1.1 版本回退
    </h4>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/8e7fc3e0553a42839cc45bda2b9f2498.png"/>
    </p>
    <p>
     之前我们也提到过，Git 能够管理文件的历史版本，这也是版本控制器重要的能力。如果有一天你发现之前的工作做的出现了很大的问题，需要在某个特定的历史版本重新开始，这个时候，就需要版本回退的功能了。
    </p>
    <p>
     执行 git reset 命令用于回退版本，可以指定退回某一次提交的版本。要解释一下“回退”本质是要将版本库中的内容进行回退，工作区或暂存区是否回退由命令参数决定:
     <br/>
     git reset 命令语法格式为:
     <strong>
      git reset [–soft | --mixed |- -hard] [HEAD]
     </strong>
    </p>
    <ul>
     <li>
      –mixed 为默认选项，使用时可以不用带该参数。该参数将暂存区的内容退回为指定提交版本内容，工作区文件保持不变。
     </li>
     <li>
      –soft 参数对于工作区和暂存区的内容都不变，只是将版本库回退到某个指定版本。
     </li>
     <li>
      –hard 参数将暂存区与工作区都退回到指定版本。切记工作区有未提交的代码时不要用这个命令，因为工作区会回滚，你没有提交的代码就再也找不回了，所以使用该参数前一定要慎重。
     </li>
     <li>
      HEAD 说明:
      <br/>
      可直接写成commitid，表示指定退回的版本
     </li>
    </ul>
    <ol>
     <li>
      HEAD 表示当前版本
     </li>
     <li>
      HEAD^ 上一个版本
     </li>
     <li>
      HEAD^^ 上上一个版本
      <br/>
      以此类推…
     </li>
    </ol>
    <ul>
     <li>
      可以使用 ~数字表示:
     </li>
    </ul>
    <ol>
     <li>
      HEAD~0 表示当前版本
     </li>
     <li>
      HEAD~1 上一个版本
     </li>
     <li>
      HEAD^2 上上一个版本
      <br/>
      以此类推…
     </li>
    </ol>

    <h4>
     <a id="21__389">

     2.1 理解分支
    </h4>
    <p>
     本章开始介绍 Git 的杀手级功能之一(注意是之一，也就是后面还有之二，之三……):分支。分支就是科幻电影里面的平行宇宙，当你正在电脑前努力学习 C++ 的时候，另一个你正在另一个平行宇宙里努力学习 JAVA。
    </p>
    <p>
     如果两个平行宇宙互不干扰，那对现在的你也没啥影响。不过，在某个时间点，两个平行宇宙合并了，结果，你既学会了 C++ 又学会了 JAVA!
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/3200e9dd702a4f358e63e68e56e9283b.png"/>
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/edfb39102495430f8e44ae5448a05d4d.png"/>
    </p>
    <p>
     在版本回退里，你已经知道，每次提交，Git都把它们串成一条时间线，这条时间线就可以理解为是个分支。截止到目前，只有一条时间线，在Git里，这个分支叫主分支，即master分支。
    </p>
    <p>
     再来理解一下HEAD，HEAD 严格来说不是指向提交，而是指向master，master才是指向提交的，所以，HEAD 指向的就是当前分支。
    </p>
    <p>
     每次提交，master分支都会向前移动一步，这样，随着你不断提交，master分支的线也越来越长,而HEAD只要一直指向master分支即可指向当前分支。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/1d27505f2c8045238dfce69c0777eeff.png"/>
    </p>
    <p>
     通过查看当前的版本库，我们也能清晰的理出思路:
    </p>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ cat.git / HEAD

    <p>
     当切换到 master分支之时，HEAD 就指向了 master，当然看不到提交了!
    </p>
    <h4>
     <a id="24__514">
     </a>
     2.4 合并分支
    </h4>
    <p>
     为了在 master 主分支上能看到新的提交，就需要将 dev 分支合并到 master 分支，示例如下:
    </p>
    <pre><code class="prism language-bash">qcj139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">git</span> branch
* dev
master
qcje139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span> <span class="token builtin class-name">:</span> ~/ gitcode$ <span class="token function">git</span> checkout master
<span class="token comment"># 切换到 master 上进行合并</span>
Switched to branch <span class="token string">'master'</span>
qcje139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcodes <span class="token function">git</span> merge dev <span class="token comment">#合并 dev 分支</span>
Updating 16623e1<span class="token punctuation">..</span>3740dce
Fast - forward
ReadMe1 +
l <span class="token function">file</span> changed, <span class="token number">1</span> insertion<span class="token punctuation">(</span>+<span class="token punctuation">)</span>
qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">cat</span> ReadMe
hello bit
hello <span class="token function">git</span>
hello world
hello versionl
hello version2
hello version3
<span class="token function">write</span> aaa <span class="token keyword">for</span> new branch
</code></pre>
    <p>
     git merge 命令用于合并指定分支到当前分支。合并后，master就能看到dev分支提交的内容了。此时的状态如图如下所示。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/7ef0ed1dc984442e9703dee54b4848ef.png"/>
     <br/>
     Fast-forward 代表“快进模式”也就是直接把master指向dev的当前提交，所以合并速度非常快。当然，也不是每次合并都能 Fast-forward，我们后面会讲其他方式的合并。
    </p>
    <h4>
     <a id="25__543">
     </a>
     2.5 删除分支
    </h4>
    <p>
     合并完成后,dev分支对于我们来说就没用了，那么dev分支就可以被删除掉，注意如果当前正处于某分支下，就不能删除当前分支，如:
    </p>
    <pre><code class="prism language-bash">qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">git</span> branch
* dev
master
qcj@139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$
<span class="token function">git</span> branch - d dev
error <span class="token builtin class-name">:</span> Cannot delete branch <span class="token string">'dev'</span> checked out at <span class="token string">'/home/qcj/gitcode'</span>
</code></pre>
    <p>
     而可以在其他分支下删除当前分支，如:
    </p>
    <pre><code class="prism language-bash">qcje139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span>:~/ gitcode$ <span class="token function">git</span> checkout master
Switched to branch 'master
qcje139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span> <span class="token builtin class-name">:</span> ~/ gitcode$ <span class="token function">git</span> branch - d dev
Deleted branch dev<span class="token punctuation">(</span>was bdaf528<span class="token punctuation">)</span>.
qcje139 - <span class="token number">159</span> - <span class="token number">150</span> - <span class="token number">152</span> <span class="token builtin class-name">:</span> ~/ gitcode$ <span class="token function">git</span> branch
* master
</code></pre>
    <p>
     此时的状态如图如下所示。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/b293aa751ec64a1391ba09a7014032f1.png"/>
    </p>
    <p>
     因为创建、合并和删除分支非常快，所以Git鼓励你使用分支完成某个任务，合并后再删掉分支，这和直接在master分支上工作效果是一样的，但过程更安全。
    </p>
