    <h4>
     <a id="_129">
     </a>
     <strong>
      命名规范总结
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       包名
      </strong>
      ：全小写，层级清晰（如
      <code>
       com.example.project.module
      </code>
      ）。
     </li>
     <li>
      <strong>
       类名
      </strong>
      ：大驼峰（如
      <code>
       UserController
      </code>
      ）。
     </li>
     <li>
      <strong>
       接口与实现类
      </strong>
      ：
      <ul>
       <li>
        接口：
        <code>
         XxxService.java
        </code>
       </li>
       <li>
        实现类：
        <code>
         XxxServiceImpl.java
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Mapper 接口
      </strong>
      ：
      <code>
       XxxMapper.java
      </code>
     </li>
     <li>
      <strong>
       实体类
      </strong>
      ：
      <code>
       Xxx.java
      </code>
     </li>
     <li>
      <strong>
       DTO/VO
      </strong>
      ：
      <code>
       XxxDTO.java
      </code>
      、
      <code>
       XxxVO.java
      </code>
     </li>
     <li>
      <strong>
       测试类
      </strong>
      ：
      <code>
       XxxTest.java
      </code>
      （如
      <code>
       UserServiceTest.java
      </code>
      ）
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_142">
     </a>
     <strong>
      注意事项
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       避免过度使用 XML
      </strong>
      ：MyBatis-Plus 的
      <code>
       BaseMapper
      </code>
      和
      <code>
       QueryWrapper
      </code>
      已覆盖 90% 的 SQL 场景。
     </li>
     <li>
      <strong>
       统一异常处理
      </strong>
      ：在
      <code>
       exception
      </code>
      包下定义全局异常处理器
      <code>
       GlobalExceptionHandler.java
      </code>
      。
     </li>
     <li>
      <strong>
       分页插件配置
      </strong>
      ：在
      <code>
       config
      </code>
      包下添加
      <code>
       MyBatisPlusConfig.java
      </code>
      配置分页插件。
     </li>
    </ol>
    <p>
     通过以上结构，项目将具备良好的可维护性和扩展性。
    </p>
