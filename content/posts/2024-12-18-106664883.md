    <p>
     大家好，我是 JackTian。
    </p>
    <p>
     作为一名运维工程师，大家都知道。早在几年前，偏传统运维，以 cacti、nagios 为主流，到后来的 zabbix、Prometheus、Open-Falcon 等，也是现在大多数企业用的偏多的运维监控平台。甚至有些企业，都是自主研发。
    </p>
    <p>
     不管是自主研发还是用开源的，其最终目的都是为了提高日常运维工作效率。那么，今天杰哥给大家推荐一款 GitHub 热榜开源运维平台 —— spug。
    </p>
    <p>
     这款开源运维平台是：
     <code>
      面向中小型企业设计的轻量级无 Agent 的自动化运维平台，其主要功能包括：主机管理、主机批量执行、主机在线终端、文件在线上传下载、应用发布部署、在线任务计划、配置中心、监控、报警等。
     </code>
    </p>
    <p>
     <strong>
      主要特征：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       批量执行:
      </strong>
      主机命令在线批量执行；
     </li>
     <li>
      <strong>
       在线终端:
      </strong>
      主机支持浏览器在线终端登录；
     </li>
     <li>
      <strong>
       文件管理:
      </strong>
      主机文件在线上传下载；
     </li>
     <li>
      <strong>
       任务计划:
      </strong>
      灵活的在线任务计划；
     </li>
     <li>
      <strong>
       发布部署:
      </strong>
      支持自定义发布部署流程；
     </li>
     <li>
      <strong>
       配置中心:
      </strong>
      支持 KV、文本、JSON 等格式的配置；
     </li>
     <li>
      <strong>
       监控中心:
      </strong>
      支持站点、端口、进程、自定义等监控；
     </li>
     <li>
      <strong>
       报警中心:
      </strong>
      支持短信、邮件、钉钉、微信等报警方式；
     </li>
     <li>
      <strong>
       优雅美观:
      </strong>
      基于 Ant Design 的 UI 界面；
     </li>
     <li>
      <strong>
       开源免费:
      </strong>
      前后端代码完全开源；
     </li>
    </ul>
    <p>
     <strong>
      依赖环境：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       Python 3.6 及以上
      </strong>
     </li>
     <li>
      <strong>
       Redis 3.x 及以上
      </strong>
     </li>
     <li>
      <strong>
       Nodejs 12.14 TLS
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      安装：
     </strong>
    </p>
    <p>
     1、Clone 项目代码
    </p>
    <pre><code>$ git clone https://github.com/openspug/spug /data/spug
</code></pre>
    <p>
     2、创建运行环境
    </p>
    <pre><code>$ cd /data/spug/spug_api
$ python3 -m venv venv 
$ source venv/bin/activate 
$ pip install -r requirements.txt -i https://pypi.doubanio.com/simple/
</code></pre>
    <p>
     3、初始化数据库
    </p>
    <pre><code>$ python manage.py initdb
</code></pre>
    <p>
     4、创建默认管理员账户
    </p>
    <pre><code>$ python manage.py useradd -u admin -p admin123 -s -n 管理员 
</code></pre>
    <p>
     各参数所代表的含义：
    </p>
    <p>
     -u：用户名
     <br/>
     -p：密码
     <br/>
     -s：超级管理员
     <br/>
     -n：用户昵称
    </p>
    <p>
     5、启动 api 开发环境服务
    </p>
    <pre><code>$ python manage.py runserver
</code></pre>
    <p>
     6、安装前端依赖
    </p>
    <pre><code>$ cd /data/spug/spug_web 
$ npm install --registry=https://registry.npm.taobao.org
</code></pre>
    <p>
     7、启动前端
    </p>
    <pre><code>$ npm start
</code></pre>
    <p>
     8、访问测试
    </p>
    <p>
     执行完上述启动前端命令时，将自动在浏览器中打开该项目。
    </p>
    <p>
     如未正常打开，可在浏览器中输入：
     <code>
      http://localhost:3000
     </code>
     输入
     <code>
      用户名、密码
     </code>
     进行访问。
    </p>
    <p>
     除上述安装外，还有一些其他服务功能需要依赖额外的服务，例如：
     <code>
      批量执行的任务卡住无法看到执行输出
     </code>
     、
     <code>
      任务计划模块添加的任务不会执行
     </code>
     、
     <code>
      监控中心模块添加的监控任务不会执行
     </code>
     ，具体可参考官网文档。
    </p>
