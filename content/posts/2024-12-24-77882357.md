    <h3 id="一什么是自动化测试" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:2.13em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <span style="">
      一、什么是自动化测试
     </span>
    </h3>
    <p>
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     自动化
     <a class="replace_word" href="http://lib.csdn.net/base/softwaretest" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="软件测试知识库">
      测试
     </a>
     是把以人为驱动的测试行为改成机器执行，通过精心设计的测试用例，由机器按照测试用例的执行步骤对应用进行自动操作，然后输出结果，由测试人员进行比较。自动化测试可以极大的节省人力、时间和硬件资源，提高测试效率。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     大白话就是: 写好自动操作app的脚本，运行起来，输出结果，结果正确，app正常测试没问题。
    </p>
    <h3 id="二移动自动化测试的流程" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:2.13em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t1" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      二、移动自动化测试的流程
     </span>
    </h3>
    <ol style="color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       通过id、name、xpath等方法定位页面元素
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       对定位到的页面元素执行相应的操作
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       对操作后出现的结果和预期结果做一个比较
      </p>
     </li>
    </ol>
    <h3 id="三移动自动化测试的优点" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:2.13em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t2" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      三、移动自动化测试的优点
     </span>
    </h3>
    <ol style="color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       对程序员来说，在提交代码之前，可以先进行回归测试，这样能提高代码质量，加快代码上线速度。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       自动化测试能够很方便的自动完成安装/卸载、启动/运行、UI适配等枯燥的手工测试环节，节省很多时间，在准确编写测试脚本的前提下，能够帮助测试人员完成更多要求。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       脚本和框架一般在
       <a class="replace_word" href="http://lib.csdn.net/base/ios" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="iOS知识库">
        iOS
       </a>
       和
       <a class="replace_word" href="http://lib.csdn.net/base/android" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Android知识库">
        Android
       </a>
       平台是通用的，同一测试脚本适用于所有
       <a class="replace_word" href="http://lib.csdn.net/base/android" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Android知识库">
        android
       </a>
       机型，可快速完成兼容性测试。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       可以模拟产生数据。通过一些重复的操作，产生一些简单的数据。能够保存截图、输出log、输出错误信息。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       可以长时间运行，并保证数据的正确性，避免人工测试的人为错误。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       Android可以通过局域网，多设备多脚本多case连续测试。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       可以借助透视宝对应用性能进行全面诊断。比如，用户提供安装有透视宝SDK的apk/ipa的安装包，就可以对用户的APP进行一次整体测试，产生一份包括崩溃、内存泄漏、网络监控和UI性能测试的具有洞察力的详细报告。目前云智慧的很多客户在产品发布之前都要先绑定透视宝SDK，之后在 testin、百度云测、腾讯优测等平台进行测试之后才评估是否使用。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       不需要重新编译APP。
      </p>
     </li>
    </ol>
    <h3 id="四移动自动化测试的缺点" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:2.13em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t3" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      四、移动自动化测试的缺点
     </span>
    </h3>
    <ol style="color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       需要借助Appium工具来录制脚本，而Appium是第三方的开源工具，需要布置环境。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       可通过Xcode自带的Instruments或者Android自带的UIAutomator工具录制脚本，要会简单使用xcode和eclipse。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       录制成功的脚本代码需要重新编写，来输出结果、进行截图、输出错误信息等，需要会编写简单的
       <a class="replace_word" href="http://lib.csdn.net/base/python" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Python知识库">
        Python
       </a>
       或者
       <a class="replace_word" href="http://lib.csdn.net/base/javascript" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="JavaScript知识库">
        js
       </a>
       脚本。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       每次要测试一个新app的时候，都要重新设计测试用例，重新设计输出规范和记录内容，重新生成标准化的测试报告。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       需要对结果进行逻辑处理，需要人工比较、分析。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       没办法手机录脚本，需要配合客户端。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       框架维护不易，复用率低。
      </p>
     </li>
    </ol>

     <span style="">
      1. 我选择的是Appium，为什么呢?
     </span>
    </h4>
    <blockquote style="margin:0px 0px 1.1em; padding:15px 20px; border-left-width:10px; border-left-style:solid; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">

    <h4 id="2-appium简介" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:1.73em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t7" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      2. Appium简介
     </span>
    </h4>
    <blockquote style="margin:0px 0px 1.1em; padding:15px 20px; border-left-width:10px; border-left-style:solid; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <ul style="">
      <li style="">
       Appium 是由Sauce Labs 提供支持的一款自动化测试框架，针对原生(native)、混血(hybrid)、和移动Web应用，使用WebDriver协议来驱动ios和Android
      </li>
      <li style="">
       Appium的设计理念是测试原生应用，不应该要求用户引入额外的SDK或者重新编译应用，另外Appium应该能与测试人员喜欢的测试实践、测试框架、测试工具一起使用。
      </li>
      <li style="">
       Appium 旨在用户可以通过任何语言以及任何测试框架去自动化测试任何移动应用，另外通过测试代码可以访问后端的API和DB
      </li>
     </ul>
    </blockquote>
    <h4 id="3-appium的要求" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:1.73em; color:rgb(63,63,63); font-family:'microsoft yahei'">

    <h4 id="4-appium概念" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:1.73em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t9" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      4. Appium概念
     </span>
    </h4>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <span style="">
      1.服务端/服务器
      <a class="replace_word" href="http://lib.csdn.net/base/architecture" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52)" target="_blank" title="大型网站架构知识库">
       架构
      </a>
     </span>
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     appium的核心其实是一个暴露了一系列REST API的server。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     这个server的功能其实很简单：监听一个端口，然后接收由client发送来的command。翻译这些command，把这些command转成移动设备可以理解的形式发送给移动设备，然后移动设备执行完这些command后把执行结果返回给appium server，appium server再把执行结果返回给client。
     <br style=""/>
     在这里client其实就是发起command的设备，一般来说就是我们代码执行的机器，执行appium测试代码的机器。狭义点理解，可以把client理解成是代码，这些代码可以是
     <a class="replace_word" href="http://lib.csdn.net/base/java" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Java 知识库">
      Java
     </a>
     /ruby/
     <a class="replace_word" href="http://lib.csdn.net/base/python" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Python知识库">
      python
     </a>
     /js的，只要它实现了webdriver标准协议就可以。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     这样的设计思想带来了一些好处:
     <br style=""/>
     1. 可以带来多语言的支持
     <br style=""/>
     2. 可以把server放在任意机器上，哪怕是云服务器都可以；（是的，appium和webdriver天生适合云测试）
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <span style="">
      2. Session
     </span>
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     自动化测试总是基于会话内容来实现的，客户端与服务端初始化会话方式，具体到不同的库有区别，但它们最终都会发送一个post/session 请求到服务器，携带一个名为Desired Capabilities的JSON对象，服务器将会自动化会话并响应一个Session ID用于发送后续命令
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <span style="">
      3. Desired Capabilities
     </span>
     <br style=""/>
     Desired Capabilities是一组发送到服务器的键和值(map and hash),告诉服务器测试人员期望建立的自动化会话类型，这里有各种类型的属性，可以用来改变服务器在测试期间的行为。例如，可以设置platformName属性值为Android，以便告诉Appium我们期望Android会话而不是IOS
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <span style="">
      4. Appium Server
     </span>
     <br style=""/>
     Appium是用
     <a class="replace_word" href="http://lib.csdn.net/base/nodejs" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="Node.js知识库">
      Node.js
     </a>
     写的服务器,它可以通过源码构建和安装或者直接通过NPM安装，这里不建议用命令安装，外国网你懂的，下一篇文章教大家安装。
    </p>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <span style="">
      5. Appium Clients
     </span>
     <br style=""/>
     Appium提供了Java、Ruby、Python、
     <a class="replace_word" href="http://lib.csdn.net/base/php" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="PHP知识库">

    <h3 id="七什么是黑盒白盒测试" style="margin:0.8em 0px; padding:0px; font-weight:100; line-height:1.3em; font-size:2.13em; color:rgb(63,63,63); font-family:'microsoft yahei'">
     <a name="t11" style="color:rgb(12,137,207)" target="_blank">
     </a>
     <span style="">
      七、什么是黑盒白盒测试
     </span>
    </h3>
    <ul style="color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       <span style="">
        黑盒测试
       </span>
       ：已知产品的功能设计规格，可以进行测试证明每个实现了的功能是否符合要求。并没有通过检查程序的源代码或者很清楚地了解该软件的源代码程序具体是怎样设计的。测试人员通过输入他们的数据然后看输出的结果从而了解软件怎样工作。
      </p>
     </li>
     <li style="">
      <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px">
       <span style="">
        白盒测试
       </span>
       ：是通过程序的源代码进行测试而不使用用户界面。这种类型的测试需要从代码句法发现内部代码在
       <a class="replace_word" href="http://lib.csdn.net/base/datastructure" rel="nofollow noopener noreferrer" style="text-decoration:none; color:rgb(223,52,52); font-weight:bold" target="_blank" title="算法与数据结构知识库">
        算法
       </a>
       ，溢出，路径，条件等等中的缺点或者错误，进而加以修正。
      </p>
     </li>
    </ul>
    <p style="margin-top:0px; margin-bottom:1.1em; padding-top:0px; padding-bottom:0px; color:rgb(63,63,63); font-family:'microsoft yahei'; font-size:15px; line-height:35px">
     二者最大的区别应该就是测试对象不一样，白盒测试主要针对的是程序代码逻辑，黑盒测试主要针对的是程序所展现给用户的功能，简单的说就是白盒测试后台程序，黑盒测试前台展示功能
    </p>
