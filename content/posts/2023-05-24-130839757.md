# 微软 x OpenAI 祭出杀招！GPT-4 全面进入 Windows 11

> 作者 | 王启隆
>   
> 责编 | 屠敏
>   
> 出品 | CSDN（ID：CSDNnews）

二十余年前，微软曾发起过一场著名的“浏览器战争”，用 IE 4.0 成功打赢了浏览器市场份额超过 90% 以上的网景浏览器。当时，微软用的就是“将 IE 放在 Windows 98 ”这样的“禁忌武器”，还因此遭到了日后的反垄断起诉。

在今天凌晨开幕的微软 Build 大会，微软携手 OpenAI 再次祭出当年的大杀招，用 AI 结合操作系统，为 Windows 系列带来了一次前所未有的智能革命。

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/0037c837565f4865a153bbf9aec55e7c.png#pic_center)

微软用两个简单的字母准确传达了今年技术领域的重大转变：A 和 I。我们熟悉的微软董事长兼 CEO 萨提亚·纳德拉（Satya Nadella）在主题演讲上总结了每一场技术时代的变革历程，并在时间线推进到 AI 时代后顺势宣布了 Bing 的 50 项更新，其中最重大的更新，自然便是 Bing 将引入 ChatGPT 成为默认搜索体验。

紧接着，微软携手 OpenAI 连续发布了数项重大更新：打造共享插件平台，为 ChatGPT 构建 AI 插件生态系统；Azure AI 迎来 Studio 和内容安全等颠覆级更新；端到端的数据分析平台 Microsoft Fabric；以及真正的重头戏，适用于 Windows 11 的 Windows Copilot。

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/4b8ca1cbfdd902fb9848ca3bf1bee3e9.png#pic_center)

* Microsoft Build 2023 官方网址：https://build.microsoft.com/en-US/home

### 无缝过渡，Windows 11 开发者=AI 开发者

Windows Copilot 意味着 Windows 11 将成为世界上第一个接入集中式 AI 协助的 PC 平台。微软还将 Bing Chat 插件扩展到 Windows，使开发人员能够将他们的应用程序集成到 Windows Copilot 中，同时支持跨平台和跨 Azure 的 AI 开发。

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/c56ea092bf6b073023faa7e16a64ca75.png#pic_center)

调用 Windows Copilot 既熟悉又简单，Copilot 按钮位于任务栏的前面和中央，非常易于用户查找和使用。打开后，Windows Copilot 侧边栏会在应用程序、程序和窗口中保持一致，并和复制/粘贴、Snap Assist、截图工具、个性化这些老功能无缝结合，成为新时代的 AI 个人助理。
  
例如，你不仅可以复制/粘贴内容到 Copilot 的窗口，还可以要求 Copilot 重写、总结或解释上面的内容。
  
Windows Copilot 将会集成 Bing 和 ChatGPT 插件，用户不仅可以访问增强的 AI 功能和体验，开发者对于这两个插件的投入也会全部延续到 Windows Copilot 当中，每一位 Windows 11 开发者都可以是 AI 开发者。

Windows Copilot 将于 6 月开始提供 Windows 11 的预览版。

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/7e7a2489611ccb3d29047a4dd719e14c.png#pic_center)

### Bing x ChatGPT

从 1 月开始，微软就宣布延长与 OpenAI 的合作伙伴关系。到了 2 月，微软宣布推出全新的人工智能 Bing 搜索引擎和 Edge 浏览器，以改变世界上最大的软件类别——搜索。
  
![图片](https://i-blog.csdnimg.cn/blog_migrate/9d8452ba33a9db586cf9821091158580.png#pic_center)

从今年 5 月起，OpenAI 将开始使用 Bing 作为 ChatGPT 的默认搜索体验。新的搜索功能将于今天向 ChatGPT Plus 用户推出，并会很快通过 ChatGPT 中的插件为所有免费的 ChatGPT 用户启用。

就像 Bing 的聊天体验由 GPT-4 提供支持一样，OpenAI 的 ChatGPT 将会整合 Bing 的搜索引擎和网络数据，和 Bing 一样，ChatGPT 也会提供数据的引用链接。

### AI 插件生态系统

在大会上，微软承诺使用与 ChatGPT 相同的开放插件标准，从而实现 Bing Chat、微软 Copilot 平台和 ChatGPT 插件之间的互操作性。开发人员如今能只用一个平台构建适用于消费者和企业界面的插件，包括 ChatGPT、Bing、Dynamics 365 Copilot 和 Microsoft 365 Copilot。

作为这个 AI 插件生态系统的一份子，Bing 自然也扩展了更多的插件支持。你可以用 Expedia 享受对话式的旅行规划，也可以借助 Instacart 让 Bing 规划晚餐菜单，并且网购买菜送货到家。

![图片](https://i-blog.csdnimg.cn/blog_migrate/522060438e129a2640a4fc7cf28417dd.png#pic_center)

Bing 的插件合作伙伴

开发者还能用插件将他们的应用和服务集成到 Microsoft 365 Copilot 中。适用于 Microsoft 365 Copilot 的插件包括 ChatGPT、Bing、Teams 和 Power Platform。开发者将能够使用 Microsoft Teams Toolkit for Visual Studio Code 和 Visual Studio 轻松为 Microsoft 365 Copilot 构建新插件。

微软还将其通用插件平台原生集成到了 Edge 浏览器中，使它成为首个集成 AI 搜索的浏览器。
  
![图片](https://i-blog.csdnimg.cn/blog_migrate/560ada3237229dbb193a4b78df62d5bd.png#pic_center)

### Azure 大更新 - 企业级 AI 革命

Azure OpenAI 服务是 Microsoft 完全托管的、以企业为中心的产品，旨在让企业访问 AI 实验室 OpenAI 的技术并增加治理功能。今天，微软推出了 Azure AI Studio，这是 Azure OpenAI 服务中的一项新功能，可让客户将 OpenAI 的 ChatGPT 或 GPT-4 等模型与他们自己的数据（无论是文本还是图像）结合起来，并构建一个聊天助手或另一种“推理”私人数据的应用程序。

从此，开发者可以轻松、安全地将 Azure OpenAI 服务模型建立在他们的数据上，而无需再查看该数据或在数据上训练模型。同时，开发者还能将基于云的存储添加到使用 Azure AI Studio 创建的 AI Copilots，以跟踪与用户的对话并以适当的上下文和意识进行响应。

![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/3e757044fc85959c0397dc67e2a5467a.png#pic_center)

除了革命级的技术更新，微软也没有忘记在社会责任上将面临的那些风险。人工智能审核服务 Azure AI Content Safety（Azure 内容安全）是 Azure AI 的另一项新产品，该服务旨在营造更安全的在线环境和社区。

它提供了一系列经过训练的 AI 模型，可以检测图像和文本中的“不当”内容。这些模型——可以理解英语、西班牙语、德语、法语、日语、葡萄牙语、意大利语和中文的文本——为被标记的内容严重性打分，指示哪些内容需要采取行动。

这次发布还包括 Azure Machine Learning 的新工具和 Microsoft Designer 以及 Bing Image Creator 的新功能。Azure Machine Learning 新增了对文本和图像数据的支持，帮助用户评估大型模型的错误、公平性和解释。Prompt Flow 提供了简化的体验，用于提示、评估和调整大型语言模型，并整合了内容安全功能。此外，Microsoft Designer 和 Bing Image Creator 将引入媒体溯源功能，用于验证由 AI 生成的图像或视频。

### 全新的统一分析平台：Fabric

Microsoft Fabric 是微软今日推出的端到端数据和分析平台，包括数据工程、数据集成、数据仓库、数据科学、实时分析、应用观测和业务智能，所有这些功能都会连接到一个被称为 OneLake 的单一数据存储库。它使得各种技术水平的客户都可以在一个统一的体验中体验到这些功能。

![图片](https://i-blog.csdnimg.cn/blog_migrate/52eac579be08b57591cf0ea58bdcef06.png#pic_center)

Fabric 在每个层面都融入了 Azure OpenAI 服务，帮助客户充分发掘数据的潜力，使开发人员能够利用生成式人工智能在数据中找到洞见。在 Microsoft Fabric 的每个数据体验中，客户可以通过 Copilot 创建数据流和数据管道，生成代码，构建机器学习模型或可视化结果。客户甚至可以创建自己的对话式语言体验，将 Azure OpenAI 服务模型与他们的数据结合起来，并发布插件。

![图片](https://i-blog.csdnimg.cn/blog_migrate/3e78546ee3be6d772078004e548688f7.png#pic_center)

### Dev Box & Dev Home：完整的开发生态

在 2022 年 8 月首次推出开发者预览版后，微软宣布其 Dev Box 云驱动工作站将很快在 2023 年 7 月亮相。Dev Box 的诞生是为了节省开发者在不同项目之间切换的时间，省去了安装和配置时所需的数小时（有时甚至浪费数天）的工作。
  
Dev Box 提供预配置、集中管理和基于项目的即用型开发环境。它为开发者提供了一个可定制的开发盒子，可以满足不同开发需求，并通过 Azure Marketplace 提供多个可供定制的预设镜像。开发者可以使用 Dev Box 进行开发工作，并通过 Azure Deployment Environments 管理和定制开发环境。

为了建设 Windows 11 的生态，微软还推出了 Dev Home。它将以预览版的形式推出，开发者可以从 Microsoft Store 获取这一全新的 Windows 体验。Dev Home 促进开发者与 GitHub 进行连接，并配置如 Microsoft Dev Box 和 GitHub Codespaces 的云端开发环境。作为开源和可扩展的工具，Dev Home 还允许开发者通过定制化的仪表板和所需工具来提升开发体验。

![图片](https://i-blog.csdnimg.cn/blog_migrate/045ea464034f93a1ffe34a7b01589c5b.png#pic_center)

微软意欲打造的 AI 生态已经完成了雏形。在未来，AI 将会重新定义开发者构建的内容和软件的构建方式。
