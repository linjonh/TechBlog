    <h2>
     <a id="conda_pip_list__0">
     </a>
     conda 虚拟环境内，
     <code>
      pip list
     </code>
     展示全局的包
    </h2>
    <p>
     网上一大堆垃圾文章，解决不了问题，贼讨厌那种看文章还得关注博主的
    </p>
    <p>
     我的环境 在
     <code>
      linux
     </code>
     和
     <code>
      win
     </code>
     上使用
     <code>
      miniconda
     </code>
     的时候遇到
    </p>
    <p>
     <strong>
      有两种情况会造成上面的问题
     </strong>
    </p>
    <h3>
     <a id="1base__6">
     </a>
     1，在base 环境内执行创建其他虚拟环境的操作
    </h3>
    <pre><code>(base) C:\Users\admin&gt; conda create -n xxxx python=3.10
</code></pre>
    <p>
     当你直接在base 环境内执行
    </p>
    <pre><code>(base) C:\Users\admin&gt; conda activate xxxxx
</code></pre>
    <p>
     执行
     <code>
      pip list
     </code>
     显示的是
     <code>
      base
     </code>
     环境内的包
    </p>
    <p>
     <strong>
      请你退出
      <code>
       base
      </code>
      环境，删掉之前创建的虚拟环境重新创建
     </strong>
    </p>
    <h3>
     <a id="2_python_18">
     </a>
     2，创建虚拟环境没有指定
     <code>
      python
     </code>
     具体的版本
    </h3>
    <pre><code>C:\Users\admin&gt; conda create -n xxxx
</code></pre>
    <p>
     这种会默认使用全局的 默认
     <code>
      python
     </code>
     版本，所以一定得指定具体的
     <code>
      python
     </code>
     版本
    </p>
