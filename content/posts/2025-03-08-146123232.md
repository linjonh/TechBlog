    <h2>
     <a id="1_JavaScript__7">
     </a>
     1. JavaScript 事件系统基础
    </h2>
    <p>
     JavaScript 事件系统是前端开发的核心机制之一，它允许网页对用户交互做出响应。简单来说，事件是在浏览器中发生的特定动作，如点击按钮、提交表单、加载页面等。
    </p>
    <h3>
     <a id="11_11">
     </a>
     1.1事件的本质
    </h3>
    <p>
     从本质上讲，JavaScript 事件是一种观察者模式（Observer Pattern）的实现。在这种模式中：
    </p>
    <ul>
     <li>
      DOM 元素作为被观察者（Subject）
     </li>
     <li>
      事件处理函数作为观察者（Observer）
     </li>
     <li>
      当特定动作发生时，浏览器通知所有注册的观察者
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e9c06bdc671c48a892d4c4ddc4f8367f.png#pic_center"/>
     </li>
    </ul>

    <h2>
     <a id="3__95">

     3. 事件传播机制：捕获与冒泡
    </h2>
    <p>
     DOM 事件传播过程中有三个阶段：
    </p>
    <ol>
     <li>
      <strong>
       捕获阶段
      </strong>
      ：事件从 Window 对象向下传递到目标元素
     </li>
     <li>
      <strong>
       目标阶段
      </strong>
      ：事件到达目标元素
     </li>
     <li>
      <strong>
       冒泡阶段
      </strong>
      ：事件从目标元素向上冒泡到 Window 对象
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fac7c00268904c80a73a3cc4000e8392.png#pic_center"/>
     </li>
    </ol>
    <h3>
     <a id="31__104">
     </a>

     <code>

     <li>
      <strong>

      </strong>

    </ol>

     7. React 合成事件系统
    </h2>
    <p>
     React 实现了自己的事件系统，称为"合成事件"（Synthetic Events）。它是对原生浏览器事件的跨浏览器包装，旨在使事件在不同浏览器中的行为一致。

    </h3>
    <ol>
     <li>
      <strong>
       跨浏览器一致性
      </strong>
      ：抹平不同浏览器的差异
     </li>
     <li>
      <strong>
       性能优化
      </strong>
      ：使用事件委托和事件池
     </li>
     <li>
      <strong>
       自动绑定
      </strong>
      ：React 组件中的事件处理方法可以自动绑定到组件实例
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a90bf0ad284c47ca9f8601a2e2f17758.png#pic_center"/>
    </p>
    <h3>
     <a id="72_React__366">
     </a>
     7.2 React 事件与原生事件的区别
    </h3>
    <ol>
     <li>
      <strong>
       命名约定
      </strong>
      ：React 使用驼峰命名（如
      <code>
       onClick
      </code>
      而非
      <code>
       onclick
      </code>
      ）
     </li>
     <li>
      <strong>
       传递函数
      </strong>
      ：React 传递函数作为事件处理程序，而不是字符串
     </li>
     <li>
      <strong>
       返回 false 不阻止默认行为
      </strong>
      ：必须显式调用
      <code>
       preventDefault()
      </code>
     </li>
     <li>
      <strong>
       事件委托
      </strong>
      ：React 将大多数事件委托到根节点（document），而不是实际的 DOM 元素
     </li>
     <li>
      <strong>
       合成事件对象
      </strong>
      ：React 的事件对象是合成的，不是原生的
     </li>
    </ol>
    <h3>
     <a id="73__React__374">
     </a>
     7.3 在 React 中使用事件
    </h3>
    <pre><code class="prism language-jsx">class ClickCounter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    
    // 绑定 this
    this.handleClick = this.handleClick.bind(this);
  }
  
  handleClick(event) {
    // event 是 React 的合成事件对象
    console.log('事件类型:', event.type);
    console.log('目标元素:', event.target);
    
    // 更新状态
    this.setState(prevState =&gt; ({
      count: prevState.count + 1
    }));
    
    // 阻止默认行为
    event.preventDefault();
  }
  
  render() {
    return (
      &lt;button onClick={this.handleClick}&gt;
        点击了 {this.state.count} 次
      &lt;/button&gt;
    );
  }
}
</code></pre>
    <p>
     在函数组件中：
    </p>
    <pre><code class="prism language-jsx">function ClickCounter() {

    <h3>
     <a id="74__this__428">
     </a>
     7.4 事件处理函数中的
     <code>
      this
     </code>
     绑定
    </h3>
    <p>
     在 React 类组件中，事件处理函数的
     <code>
      this
     </code>
     默认不指向组件实例，有以下几种解决方法：
    </p>
    <ul>
     <li>
      <strong>
       在构造函数中绑定
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="prism language-jsx">constructor(props) {
  super(props);
  this.handleClick = this.handleClick.bind(this);
}
</code></pre>
    <ul>
     <li>
      <strong>
       使用箭头函数
      </strong>
      ：
     </li>
    </ul>
    <pre><code class="prism language-jsx">// 在类中使用箭头函数定义方法
handleClick = (event) =&gt; {
  this.setState({ count: this.state.count + 1 });
};
</code></pre>
    <ul>
     <li>
      <strong>
       在渲染时使用箭头函数
      </strong>
      （不推荐，每次渲染会创建新函数）：
     </li>

    <h3>
     <a id="75__458">
     </a>
     7.5 合成事件与原生事件的交互
    </h3>
    <p>
     在某些情况下，需要同时使用 React 合成事件和 DOM 原生事件：
    </p>
    <pre><code class="prism language-jsx">class HybridEventComponent extends React.Component {

  
  constructor(props) {
    super(props);
    this.buttonRef = React.createRef();
    
    // 绑定 this
    this.handleResize = this.handleResize.bind(this);
    this.handleClick = this.handleClick.bind(this);
    this.handleNativeClick = this.handleNativeClick.bind(this);
  }
  componentDidMount() {
    // 添加原生事件监听器
    window.addEventListener('resize', this.handleResize);
    // 添加到 DOM 元素
    this.buttonRef.current.addEventListener('click', this.handleNativeClick);
  }
  
  componentWillUnmount() {
    // 记得清理！
    window.removeEventListener('resize', this.handleResize);
    this.buttonRef.current.removeEventListener('click', this.handleNativeClick);
  }
  
  handleResize(event) {
    console.log('窗口大小改变 - 原生事件');
    // 这里的 event 是原生 DOM 事件对象
  }
  
  handleClick(event) {
    console.log('按钮点击 - React 合成事件');
    // 这里的 event 是 React 合成事件对象
  }
  
  handleNativeClick(event) {
    console.log('按钮点击 - 原生事件');
    // 这里的 event 是原生 DOM 事件对象
  }
  
  render() {
    return (
      &lt;button 
        ref={this.buttonRef} 
        onClick={this.handleClick}
      &gt;
        点击我
      &lt;/button&gt;
    );
  }
}
</code></pre>
    <h3>
     <a id="76_React_17__516">
     </a>
     7.6 React 17 中的事件系统更新
    </h3>
    <p>
     在 React 17 中，React 的事件系统进行了重大更新：
    </p>
    <ol>
     <li>
      <strong>
       事件委托位置变更
      </strong>
      ：从
      <code>
       document
      </code>
      移动到了 React 树的根 DOM 容器，这使得在同一页面上运行多个 React 版本成为可能
     </li>
     <li>
      <strong>
       去除事件池
      </strong>
      ：合成事件对象不再被复用，不需要调用
      <code>
       e.persist()
      </code>
     </li>
     <li>
      <strong>
       对齐原生浏览器行为
      </strong>
      ：如
      <code>
       onScroll
      </code>
      不再冒泡，
      <code>
       onFocus
      </code>
      和
      <code>
       onBlur
      </code>
      使用原生 focusin/focusout 事件
     </li>
    </ol>
    <p>
     React 18 及更高版本继续保持这些更改，并进一步优化了事件系统的性能。
    </p>
