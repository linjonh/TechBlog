    <h2>
     <a id="Fungible_Tokens_0">
     </a>
     一、同质化代币（Fungible Tokens）
    </h2>
    <h3>
     <a id="1_ERC20_1">
     </a>
     1. ERC-20（基础标准）
    </h3>
    <p>
     用途：同质化代币（如 USDT、UNI）。
     <br/>
     特点：标准化转账和授权接口。
    </p>
    <h3>
     <a id="2_ERC777_ERC20_4">
     </a>
     2. ERC-777（增强版 ERC-20）
    </h3>
    <p>
     改进：
     <br/>
     支持 钩子函数（Hooks），在转账前后触发自定义逻辑（如黑名单检查）。
     <br/>
     操作更高效，减少 Gas 消耗。
     <br/>
     案例：DAI 稳定币的部分实现。
    </p>
    <h3>
     <a id="3_ERC2612_Gas__9">
     </a>
     3. ERC-2612（无 Gas 转账）
    </h3>
    <p>
     改进：允许用户通过链下签名（类似 EIP-712）授权转账，无需提前支付 Gas 费。
     <br/>
     场景：优化用户体验，适用于去中心化交易所（DEX）。
    </p>
    <h2>
     <a id="NFTNonFungible_Tokens_12">
     </a>
     二、非同质化代币（NFT，Non-Fungible Tokens）
    </h2>
    <h3>
     <a id="1_ERC721_NFT__13">
     </a>
     1. ERC-721（基础 NFT 标准）
    </h3>
    <p>
     用途：唯一性资产（如数字艺术品、游戏道具）。
     <br/>
     核心函数：
     <br/>
     ownerOf(tokenId)：查询 NFT 所有者。
     <br/>
     safeTransferFrom()：安全转账（防丢失）。
     <br/>
     案例：CryptoPunks、Bored Ape Yacht Club。
    </p>
    <h3>
     <a id="2_ERC1155_19">
     </a>
     2. ERC-1155（多合一混合代币）
    </h3>
    <p>
     改进：
     <br/>
     单个合约可同时管理 同质化和非同质化代币。
     <br/>
     批量转账（一次交易转移多个代币）。
     <br/>
     场景：游戏道具（如 Decentraland 的虚拟物品）、批量空投。
     <br/>
     案例：Enjin 游戏平台、OpenSea 的部分 NFT。
    </p>
    <h3>
     <a id="3_ERC4907NFT__25">
     </a>
     3. ERC-4907（NFT 租赁标准）
    </h3>
    <p>
     用途：分离 NFT 的所有权和使用权。
     <br/>
     功能：
     <br/>
     设置“使用者”角色（如租户）和到期时间。
     <br/>
     到期后自动收回使用权。
     <br/>
     案例：虚拟土地租赁、音乐版权临时授权。
    </p>
    <h2>
     <a id="_31">
     </a>
     三、金融与治理协议
    </h2>
    <h3>
     <a id="1_ERC4626_32">
     </a>
     1. ERC-4626（收益金库标准）
    </h3>
    <p>
     用途：统一 DeFi 收益聚合器的接口（如 Yearn Finance）。
     <br/>
     功能：标准化存款、取款和收益计算方式，提升协议兼容性。
    </p>
    <h3>
     <a id="2_ERC1400_35">
     </a>
     2. ERC-1400（证券型代币标准）
    </h3>
    <p>
     用途：合规化证券代币发行（STO）。
     <br/>
     功能：
     <br/>
     内置权限控制（如 KYC/AML 验证）。
     <br/>
     支持法律文件附加（如投资者条款）。
     <br/>
     案例：Polymath 平台的证券代币。
    </p>
    <h2>
     <a id="_41">
     </a>
     四、扩展功能与元交易
    </h2>
    <h3>
     <a id="1_ERC2981NFT__42">
     </a>
     1. ERC-2981（NFT 版税标准）
    </h3>
    <p>
     用途：定义 NFT 转售时的版税分成规则。
     <br/>
     功能：每次 NFT 转售时自动向创作者支付版税（如 OpenSea 的创作者分成）。
    </p>
    <h3>
     <a id="2_ERC5169_45">
     </a>
     2. ERC-5169（跨链执行标准）
    </h3>
    <p>
     用途：允许智能合约在多个链上同步执行逻辑。
     <br/>
     场景：跨链 DAO 治理、多链资产管理。
    </p>
    <h3>
     <a id="3_ERC20Permit_48">
     </a>
     3. ERC-20-Permit（链下授权）
    </h3>
    <p>
     改进：类似 ERC-2612，允许通过签名授权代币转账，无需提前上链操作。
     <br/>
     案例：Uniswap 的 Permit2 合约。
    </p>
    <h2>
     <a id="Layer2__51">
     </a>
     五、Layer2 与扩容相关
    </h2>
    <h3>
     <a id="1_ERC4337_52">
     </a>
     1. ERC-4337（账户抽象）
    </h3>
    <p>
     用途：将钱包变为智能合约账户，支持更灵活的交易逻辑。
     <br/>
     功能：
     <br/>
     无 Gas 交易（由第三方支付）。
     <br/>
     社交恢复（用好友签名找回账户）。
     <br/>
     案例：Argent 智能钱包。
    </p>
    <h3>
     <a id="2_Optimism__OVM__58">
     </a>
     2. Optimism 的 OVM 标准
    </h3>
    <p>
     用途：适配 Layer2 网络 Optimism 的合约开发。
     <br/>
     特点：兼容 EVM，但 Gas 成本更低。
    </p>
    <h2>
     <a id="_61">
     </a>
     六、其他创新标准
    </h2>
    <h3>
     <a id="1_ERC4804Web3__62">
     </a>
     1. ERC-4804（Web3 访问协议）
    </h3>
    <p>
     用途：通过 HTTP URL 直接访问链上资源（如 NFT 图片）。
     <br/>
     场景：简化 Web2 开发者与区块链的交互。
    </p>
    <h3>
     <a id="2_ERC6551NFT__65">
     </a>
     2. ERC-6551（NFT 绑定账户）
    </h3>
    <p>
     用途：为每个 NFT 分配一个独立的以太坊账户。
     <br/>
     功能：NFT 可持有资产、与其他合约交互（如游戏角色自主交易道具）。
    </p>
    <h2>
     <a id="_69">
     </a>
     总结
    </h2>
    <p>
     以太坊生态的代币标准持续演进，核心逻辑是解决特定场景的痛点：
    </p>
    <p>
     ERC-20 → 基础流动性。
     <br/>
     ERC-721/1155 → 数字所有权。
     <br/>
     ERC-4337/2612 → 用户体验优化。 开发者需根据业务需求（如合规性、功能复杂度、Gas 成本）选择最适配的标准。
    </p>
    <h3>
     <a id="_76">
     </a>
     如何选择代币标准？
    </h3>
    <p>
     发行普通代币（如治理币） ERC-20、ERC-777
     <br/>
     唯一性资产（如艺术品） ERC-721、ERC-1155
     <br/>
     复杂金融协议 ERC-4626、ERC-1400
     <br/>
     多链互操作 ERC-5169、ERC-4337
     <br/>
     低 Gas 或 Layer2 ERC-2612、OVM 标准
    </p>
