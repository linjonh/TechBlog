    <h2>
     <a id="DockerRagflowDev_1">
     </a>
     Docker快速启动Ragflow:Dev
    </h2>
    <h3>
     <a id="_3">
     </a>
     系统准备
    </h3>
    <p>
     ubuntu server 24.04
    </p>
    <ul>
     <li>
      CPU ≥ 4 cores (x86);
     </li>
     <li>
      RAM ≥ 16 GB;
     </li>
     <li>
      Disk ≥ 100 GB;
     </li>
    </ul>
    <h5>
     <a id="_9">
     </a>
     更新系统
    </h5>
    <pre><code>sudo apt update 
</code></pre>
    <h3>
     <a id="_14">
     </a>
     下载源码
    </h3>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone https://github.com/infiniflow/ragflow.git
<span class="token builtin class-name">cd</span> ragflow/docker
<span class="token comment"># 切换稳定版本分支</span>
<span class="token function">git</span> checkout <span class="token parameter variable">-f</span> v0.17.0
</code></pre>
    <h3>
     <a id="docker_22">
     </a>
     安装docker全家桶
    </h3>
    <p>
     这里我选择直接安装1panel
    </p>
    <ol>
     <li>
      因为1panel会自动根据你系统，选择最佳的docker(rocky linux不行)
     </li>
     <li>
      当然，也包含了docker-compose
     </li>
     <li>
      如果是自己安装的docker-compose有可能是旧版本的，导致构建镜像失败也是有可能的
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">curl</span> <span class="token parameter variable">-sSL</span> https://resource.fit2cloud.com/1panel/package/quick_start.sh <span class="token parameter variable">-o</span> quick_start.sh <span class="token operator">&amp;&amp;</span> <span class="token function">sudo</span> <span class="token function">bash</span> quick_start.sh
</code></pre>
    <h3>
     <a id="_32">
     </a>
     开始构建镜像
    </h3>
    <pre><code class="prism language-bash"><span class="token comment"># 需要提前切换到 ragflow/docker 目录执行下面命令</span>
<span class="token function">sudo</span> <span class="token function">docker</span> compose <span class="token parameter variable">-f</span> docker-compose.yml up <span class="token parameter variable">-d</span>
</code></pre>
    <h3>
     <a id="_38">
     </a>
     访问测试
    </h3>
    <p>
     http://IP/
    </p>
    <h4>
     <a id="502_Bad_Gateway_42">
     </a>
     502 Bad Gateway解决办法
    </h4>
    <p>
     说明有个Base服务没有启动，我们检测一下后端是否正常
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">docker</span> logs <span class="token parameter variable">-f</span> ragflow-server
</code></pre>
    <p>
     如果是以下输出，说明一切正常，就不会502错误
    </p>
    <pre><code class="prism language-bash">     ____   ___    ______ ______ __
    / __ <span class="token punctuation">\</span> /   <span class="token operator">|</span>  / ____// ____// /____  _      __
   / /_/ // /<span class="token operator">|</span> <span class="token operator">|</span> / / __ / /_   / // __ <span class="token punctuation">\</span><span class="token operator">|</span> <span class="token operator">|</span> /<span class="token operator">|</span> / /
  / _, _// ___ <span class="token operator">|</span>/ /_/ // __/  / // /_/ /<span class="token operator">|</span> <span class="token operator">|</span>/ <span class="token operator">|</span>/ /
 /_/ <span class="token operator">|</span>_<span class="token operator">|</span>/_/  <span class="token operator">|</span>_<span class="token operator">|</span><span class="token punctuation">\</span>____//_/    /_/ <span class="token punctuation">\</span>____/ <span class="token operator">|</span>__/<span class="token operator">|</span>__/

 * Running on all addresses <span class="token punctuation">(</span><span class="token number">0.0</span>.0.0<span class="token punctuation">)</span>
</code></pre>
    <p>
     那么我们就需要额外启动一些base服务
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 需要提前切换到 ragflow/docker 目录执行下面命令</span>
<span class="token function">sudo</span> <span class="token function">docker</span> compose <span class="token parameter variable">-f</span> docker-compose-base.yml <span class="token parameter variable">--profile</span> infinity up <span class="token parameter variable">-d</span>
</code></pre>
