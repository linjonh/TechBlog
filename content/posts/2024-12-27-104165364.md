    <h2>
     <strong>
      <span style="color:#303030;">
       什么是Java / JVM中的-Xms和-Xms参数（已更新至Java 13）
      </span>
     </strong>
    </h2>
    <p>
     <span style="color:#515151;">
      简而言之，
     </span>
    </p>
    <ul>
     <li>
      <code class="language-plaintext">
       Xmx
      </code>
      指定应用程序可用的
      <strong>
       最大
      </strong>
      堆大小
     </li>
     <li>
      <code class="language-plaintext">
       Xms
      </code>
      指定应用程序可用的
      <strong>
       最小
      </strong>
      堆大小
     </li>
    </ul>
    <p>
     <span style="color:#515151;">
      这些是Java虚拟机（JVM）参数，用于指定Java应用程序的内存边界。在对性能问题或
      <a href="https://docs.oracle.com/javase/9/docs/api/java/lang/OutOfMemoryError.html" rel="nofollow">
       OutOfMemoryError进行
      </a>
      故障排除时，通常使用它们。它们
      <strong>
       控制
      </strong>
      Java应用程序可用
      <strong>
       的内存量
      </strong>
      。该
      <code class="language-plaintext">
       Xmx
      </code>
      参数指定
      <strong>
       最大存储器
      </strong>
      的应用程序都可以使用，其中作为
      <code class="language-plaintext">
       Xms
      </code>
      指定的
      <strong>
       最小
      </strong>
      或初始内存池。如果您的应用程序超出了最大内存（使用分配
      <code class="language-plaintext">
       Xmx
      </code>
      ），并且垃圾回收器无法释放内存，则JVM将崩溃，并显示
      <a href="https://docs.oracle.com/javase/9/docs/api/java/lang/OutOfMemoryError.html" rel="nofollow">
       OutOfMemoryError
      </a>
      。如果您有兴趣，我写了
      <strong>
       一篇文章，
      </strong>
      并举例说明垃圾收集的工作方式及其产生的年代。
     </span>
    </p>
    <pre class="has"><code class="language-prettyprint">&lt;span style="color:#515151"&gt; 
$ java -Xmx256m Xmx1024m -jar yourapp.jar
 
&lt;/span&gt;</code></pre>
    <p>
     <span style="color:#515151;">
      在上面的示例中，应用程序
      <code class="language-plaintext">
       yourapp.jar
      </code>
      将获得256 MB的初始内存池，最大内存池最大为1024 MB。在中
      <code class="language-plaintext">
       256m
      </code>
      ，
      <code class="language-plaintext">
       m
      </code>
      代表兆字节。您可以使用
      <code class="language-plaintext">
       g
      </code>
      或
      <code class="language-plaintext">
       G
      </code>
      表示千兆字节。
     </span>
    </p>
    <ul>
     <li>
      <code class="language-plaintext">
       Xmx1g
      </code>
      或
      <code class="language-plaintext">
       Xmx1G
      </code>
      ：将最大内存大小设置为1 GB。
     </li>
     <li>
      <code class="language-plaintext">
       Xmx1024m
      </code>
      或
      <code class="language-plaintext">
       Xmx1024M
      </code>
      ：将最大内存大小设置为1024兆字节。
     </li>
     <li>
      <code class="language-plaintext">
       Xmx1024000k
      </code>
      或
      <code class="language-plaintext">
       Xmx1024000K
      </code>
      ：将最大内存大小设置为1024000千字节。
     </li>
    </ul>
    <p>
     <span style="color:#515151;">
      请务必注意，
      <code class="language-plaintext">
       Xmx
      </code>
      和
      <code class="language-plaintext">
       Xms
      </code>
      都是可选的。如果未提供这些，则Java虚拟机（JVM）将为其使用
      <strong>
       默认值
      </strong>
      。默认值取决于您的计算机和JVM配置。对于服务器类机器，这些
      <a href="https://docs.oracle.com/javase/9/gctuning/ergonomics.htm#JSGCT-GUID-83551BA5-ADEA-4E2E-B60A-3A953DA8FD02" rel="nofollow">
       定义为
      </a>
      ：
     </span>
    </p>
    <ul>
     <li>
      初始堆大小为物理内存的1/64（用于
      <code class="language-plaintext">
       Xms
      </code>
      ）
     </li>
     <li>
      最大堆大小为物理内存的1/4（用于
      <code class="language-plaintext">
       Xmx
      </code>
      ）
     </li>
    </ul>
    <p>
     <span style="color:#515151;">
      在下面的示例中，最大内存将被限制为1024 MB。初始内存（
      <code class="language-plaintext">
       Xms
      </code>
      参数代表的将使用默认值。）
     </span>
    </p>
    <pre class="has"><code class="language-prettyprint">&lt;span style="color:#515151"&gt; 
java -Xmx1024m -jar yourapp.jar
 
&lt;/span&gt;</code></pre>
    <p>
     <span style="color:#515151;">
      这是一个
      <strong>
       不错的YouTube视频
      </strong>
      ，介绍了解决与内存相关的错误的过程，并通过示例演示了如何解决这些错误。
     </span>
    </p>
    <p>
     <strong>
      <span style="color:#313131;">
       Java 13和Z垃圾收集器
      </span>
     </strong>
    </p>
    <p>
     <span style="color:#515151;">
      Java 13引入了一个称为ZGC的新垃圾收集器。其功能之一包括优化，以将未使用的内存返回到操作系统。默认情况下启用此功能，它不会返回内存，以使堆大小减小到小于
      <code class="language-plaintext">
       Xms
      </code>
      。因此，如果您将其设置
      <code class="language-plaintext">
       Xms
      </code>
      为相等
      <code class="language-plaintext">
       Xmx
      </code>
      （就像许多开发人员一样），它将实质上禁用该功能。
     </span>
    </p>
    <p>
     <span style="color:#515151;">
      如果要查看所有可用的JVM参数，则可以使用
      <code class="language-plaintext">
       java -X
      </code>
      开关，例如
     </span>
    </p>
    <pre class="has"><code class="language-prettyprint">$ java -X
    -Xmixed混合模式执行（默认）
    仅-xint解释模式执行
    -Xbootclasspath：&lt;目录和zip / jar文件，用：&gt;分隔

    -Xbootclasspath / a：&lt;用：&gt;分隔的目录和zip / jar文件

    -Xbootclasspath / p：&lt;用：&gt;分隔的目录和zip / jar文件
