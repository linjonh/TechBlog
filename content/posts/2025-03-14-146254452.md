     CentOS 系统安装 docker 以及常用插件

 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     博主用的的是WindTerm软件链接的服务器，因为好用
    </p>
    <h4>
     <span style="color:null">
      1.链接上服务器登入后，在/root/目录下
     </span>
    </h4>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/d2c1abfac85f427aa608458e776eea8c.png"/>
    </p>
    <hr/>
    <h4>
     <span style="color:null">
      2.执行以下命令安装docker
     </span>
    </h4>
    <pre><code>sudo yum install -y yum-utils

sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

sudo systemctl enable docker --now</code></pre>
    <p>
     可以检查下是否安装成功
    </p>
    <pre><code>docker ps</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/deb5db5a8e794adf8d69e6bf7747df0e.png">
      这样就是安装成功了
     </img>
    </p>
    <p>
     有时候会出现启动不了 ，可能是sudo systemctl enable docker --now（立即执行启动docker的意思）没有执行
    </p>
    <hr/>
    <p>
     3. 切换镜像源，编辑
     <strong>
      <code>
       /etc/docker/daemon.json
      </code>
      文件，没有就创建一个
      <code>
       daemon.json
      </code>
      文件
     </strong>
    </p>
    <p>
     <strong>
      镜像源加速需要到自己的云服务器账号查看
     </strong>
    </p>
    <pre><code>{
    "registry-mirrors": [ "阿里云的、华为云或者其他的都可以.mirror.swr.myhuaweicloud.com" ]
}</code></pre>
    <h4>
     重新加载 Docker 守护进程配置并重启 Docker 服务
    </h4>
    <pre><code>sudo systemctl daemon - reload
sudo systemctl restart docker</code></pre>
    <p>
     查看镜像源是否配置成功
    </p>
    <pre><code>docker info</code></pre>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/1fca80db4aa4404e8c016bbe03910039.png"/>
    </p>
    <p>
     出现这个恭喜你配置成功
    </p>
    <p>
     <strong>
      <span style="color:#fe2c24">
       以后prometheus.yml和docker-compose.yml依赖文件都创建在/root/prod/目录下，自己新建prod目录
      </span>
     </strong>
    </p>
    <hr/>
    <p>
     现在安装常用插件
    </p>
    <p>
     <strong>
      <span style="color:#fe2c24">
       在prod目录
      </span>
     </strong>
     创建prometheus.yml文件
    </p>
    <pre><code class="hljs">vim prometheus.yml</code></pre>
    <p>
     按
     <span style="color:#fe2c24">
      <strong>
       i
      </strong>
     </span>
     进入编辑模式 ，粘贴下面的代码
    </p>
    <pre><code class="hljs">global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'redis'
    static_configs:
      - targets: ['redis:6379']

  - job_name: 'kafka'
    static_configs:
      - targets: ['kafka:9092']</code></pre>
    <p>
     按
     <span style="color:#fe2c24">
      esc
     </span>
     健输入，
     <span style="color:#fe2c24">
      :wq
     </span>
     ，保存并退出，如果退出不了输入，
     <span style="color:#fe2c24">
      :q!
     </span>
     <span style="color:null">
      ，强行退出不做更改(再重新vim创建)
     </span>
    </p>
    <p>
     创建docker-compose.yml文件
    </p>
    <p>
    </p>
    <pre><code class="hljs">vim docker-compose.yml</code></pre>
    <p>
     按
     <span style="color:#fe2c24">
      <strong>
       i
      </strong>
     </span>
     进入编辑模式 ，粘贴下面的代码
    </p>
    <p>
     按
     <span style="color:#fe2c24">
      esc
     </span>
     健输入，
     <span style="color:#fe2c24">
      :wq
     </span>
     ，保存并退出，如果退出不了输入，
     <span style="color:#fe2c24">
      :q!
     </span>
     <span style="color:null">
      ，强行退出不做更改(再重新vim创建)
     </span>
    </p>
    <hr/>
    <p>
     启动环境，就会下载
    </p>
    <pre><code class="hljs">docker compose -f docker-compose.yml up -d</code></pre>
    <p>
     可以查看下载了哪些插件
    </p>
    <pre><code class="hljs"> docker ps</code></pre>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/91f783a783d1400e9c62d3f4211324f3.png">
      <span style="color:#fe2c24">
       <strong>
        插件名称以及端口号还有限制ip访问等等消息
       </strong>
      </span>
     </img>
    </p>
    <p>
     <strong>
      <span style="color:null">
       比如我们访问kafka ui插件，
      </span>
     </strong>
     http://服务器ip地址:8080
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/75742fc05c2848fa81f1c1f21d7df473.png"/>
    </p>
    <p>
     OK成功了，其他的插件一样
    </p>
