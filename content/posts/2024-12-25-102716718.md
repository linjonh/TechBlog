    <h2>
     <a id="1_1">
     </a>
     1.嵌入式是啥
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5e503aab4789f35f5e9c76080c87af1e.png">
      <br/>
      重点描述如下：
      <br/>
      软硬件可裁剪
      <br/>
      对功能、可靠性、成本、体积、功耗等严格要求
      <br/>
      嵌入式系统是软件和硬件的综合体
      <br/>
      有些嵌入式系统还包含操作系统，但大多数嵌入式系统都是由单个程序实现整个控制逻辑。
     </img>
    </p>
    <p>
     <strong>
      归纳
     </strong>
     ：需要关注硬件，注意可靠性，功耗，成本。这就是为什么你家洗衣机里面用嵌入式而不放一台Intel电脑。
    </p>
    <p>
     <strong>
      嵌入式开发设备有哪些：
     </strong>
    </p>
    <p>
     单片机/DSP/FPGA/ARM/arduino/树莓派 …
    </p>
    <p>
     <strong>
      单片机：
     </strong>
     <br/>
     单片机又称单片微控制器，它不是完成某一个逻辑功能的芯片，而是把一个计算机系统集成到一个芯片上。相当于一个微型的计算机
     <br/>
     <img alt="在这里插入图片描述" height="300" src="https://i-blog.csdnimg.cn/blog_migrate/e900bd41ce09804d31454b5ff67f3b35.png" width="400">
      <br/>
      <strong>
       DSP
      </strong>
      <br/>
      数字信号处理器（digital signal processor) 由大规模或超大规模集成电路芯片组成的用来完成某种信号处理任务的处理器。它是为适应高速实时信号处理任务的需要而逐渐发展起来的。硬件乘法累加操作/哈佛结构/特殊寻址模式…
     </img>
    </p>
    <p>
     <strong>
      FPGA
     </strong>
     <br/>
     Field Programmable Gate Array，中文名是现场可编程门阵列
     <br/>
     “FPGA写的不是代码，是电路”。
     <br/>
     <img alt="在这里插入图片描述" height="200" src="https://i-blog.csdnimg.cn/blog_migrate/af3e3c681e43a5dbfeb5c0e070e47c71.png" width="300">
      <br/>
      下面资料/图片来源：
      <a href="http://www.elecfans.com/d/716715.html" rel="nofollow">
       FPGA是什么？为什么要使用它
      </a>
      <br/>
      CPU、GPU 都属于冯·诺依曼结构，指令译码执行、共享内存。FPGA 之所以比 CPU 甚至 GPU 能效高，本质上是无指令、无需共享内存的体系结构带来的福利。
      <br/>
      FPGA 每个逻辑单元的功能在重编程（烧写）时就已经确定，不需要指令。
      <br/>
      FPGA 每个逻辑单元与周围逻辑单元的连接在重编程（烧写）时就已经确定，并不需要通过共享内存来通信。
      <br/>
      <img alt="在这里插入图片描述" height="220" src="https://i-blog.csdnimg.cn/blog_migrate/d78c1c22386d2fcead40012bebffdda5.png" width="500">
       <br/>
       <strong>
        ARM
       </strong>
       <br/>
       Advanced RISC Machine，RISC：精简指令集计算机。使用了经过精简设计的指令系统，使得整个处理器的设计可以更加简洁，功耗、体积大大缩小。
       <br/>
       对比DSP，ARM具有比较强的事务管理功能，可以用来跑界面以及应用程序等，其优势主要体现在控制方面，它的速度和数据处理能力一般，但是外围接口比较丰富，标准化和通用性做的很好，而且在功耗等方面做得也比较好，所以适合用在一些消费电子品方面。
      </img>
     </img>
    </p>
    <p>
     <strong>
      Arduino与树莓派
     </strong>
     <br/>
     Arduino是一款基于微控制器（单片机）的电子开发板，它可以运行一些相对比较简单的应用程序。与树莓派不同，Arduino无法运行操作系统。由于没有操作系统，Arduino的程序运行更加直接，更适合用于与传感器和外围电路进行通信，其程序体积较小，程序运行效率相对较高。Arduino的硬件原理图、电路图、IDE软件及核心库文件都是开源的，在开源协议范围内里可以任意修改原始设计及相应代码。
     <br/>
     树莓派与Arduino不同，树莓派是一款超小体积的计算机，它需要利用TF卡装载操作系统才能够运行。树莓派利用ARM作为处理器，还具有HDMI接口、USB接口、网络接口等。
     <br/>
     <a href="http://baijiahao.baidu.com/s?id=1605212801170379398&amp;wfr=spider&amp;for=pc" rel="nofollow">
      Arduino与树莓派 哪款开发板更适合你
     </a>
    </p>
    <p>
     这些芯片各有优势，经常是互相借鉴，你中有我我中有你。设计产品的时候也可以充分发挥各自的优势所在。
     <br/>
     <img alt="在这里插入图片描述" height="200" src="https://i-blog.csdnimg.cn/blog_migrate/52ae47f9d3af0f7bb051e35eb88022b7.png" width="400"/>
    </p>

    <h2>
     <a id="2_43">

     2.嵌入式应用
    </h2>
    <p>
     <strong>
      单片机控制鱼缸生态系统
     </strong>
     <br/>
     设备：51单片机
     <br/>
     功能：时间显示，定时控制电机旋转投喂饲料，定时开启灯光/过滤系统/增氧系统。
     <br/>
     延申产品：空调/洗衣机/电子表
     <br/>
     <img alt="在这里插入图片描述" height="400" src="https://i-blog.csdnimg.cn/blog_migrate/fd03f9838f5ed2b34828a254fb2bb132.jpeg" width="300"/>
    </p>
    <hr/>
    <p>
     <strong>
      DSP数字信号处理
     </strong>
     <br/>
     设备：DSP+51单片机
     <br/>
     功能：dsp做信号采集，频谱变化，滤波，分析电流/电压/频率等参数。数据传输到单片机控制液晶显示。
     <br/>
     延申产品：数字功放，主动降噪耳机
     <br/>
     <img alt="在这里插入图片描述" height="400" src="https://i-blog.csdnimg.cn/blog_migrate/c1a58ed87e433e81aadcfef8b023a42e.png" width="300"/>
    </p>
    <hr/>
    <p>
     <strong>
      FPGA信号发生器
     </strong>
     <br/>
     延申产品：数字电源，视频编解码（低延时，大数据并行计算）
     <br/>
     <img alt="在这里插入图片描述" height="300" src="https://i-blog.csdnimg.cn/blog_migrate/075b011b6311d04b25fea8e2cf26d7d2.png" width="400"/>
     <br/>
     嵌入式的应用可以说充满了我们生活各个方面，小到手表/路由器，大到飞机卫星都有应用。
    </p>

     3.PIN 引脚
    </h2>
    <p>
     PIN:引脚，一个集成度这么高的芯片，封装这么好为什么要伸出些引脚来？这是用来与外设打交道，如同我们写代码，封装接口，也是要留出传入参数传出参数。不同引脚会有不同的功能。
     <br/>
     树莓派封装好的HDMI口/USB口这些其实最终也是连接到芯片的某些特定功能的PIN脚。侧边用排针引出来的引脚，只是主芯片没使用的一些引脚可以给开发者自定义使用的。所以说主芯片还是通过PIN脚与外界打交道的。

    <h3>
     <a id="_119">
     </a>
     继电器
    </h3>
    <p>
     家用设备220V，我们的嵌入式设备是3.3V，如果嵌入式设备直接接上去，估计要冒烟。如何用嵌入式设备去控制这些几百伏的设备呢？
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/50a202b7865cff08bf017122cd026343.png"/>
     <br/>
     作用：小电流去控制大电流
     <br/>
     电磁继电器
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/0084e8b967b9718a592213fd01e87158.png"/>
     <br/>
     原理
     <br/>
     通过小电流控制电磁铁吸附，从而控制大电流电路。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ebca19e2a607f03ac98e97ac8f394af3.png"/>
     <br/>
     由于电磁线圈是感性器件，IO口控制D/E接通和断开是瞬间的事情，这样线圈就会产生很高的反电动势，容易导致嵌入式IO口损坏。所以一般在嵌入式与线圈间加入光耦隔离或者二极管保护。
    </p>

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/52e5ac798ff1d4efc1393a1a0a9ba6f0.png"/>
