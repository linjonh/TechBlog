    <p>
     将错误边界与 React 的
     <code>
      Suspense
     </code>
     结合使用，可以有效地处理组件中的异步操作和错误。
     <code>
      Suspense
     </code>
     允许你指定一个加载状态，而错误边界则可以捕获渲染过程中发生的错误。这种结合可以提升用户体验，使应用更具鲁棒性。
    </p>
    <h3>
     <a id="1__Suspense__2">
     </a>
     1. 理解 Suspense 和错误边界的作用
    </h3>
    <ul>
     <li>
      <strong>
       Suspense
      </strong>
      ：用于处理异步加载的组件，允许你在加载时展示一个占位 UI。
     </li>
     <li>
      <strong>
       错误边界
      </strong>
      ：捕获子组件树中发生的 JavaScript 错误，并展示回退 UI。
     </li>
    </ul>

    <h3>
     <a id="5__136">
     </a>
     5. 总结
    </h3>
    <p>
     将错误边界与
     <code>
      Suspense
     </code>
     结合使用，可以有效地处理异步加载和错误情况。错误边界捕获组件中的错误，而
     <code>
      Suspense
     </code>
     则处理数据加载时的等待状态。通过这种方式，用户可以获得更好的体验，避免了应用崩溃的情况。
    </p>
    <h4>
     <a id="_140">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      确保在
      <code>
       DataFetchingComponent
      </code>
      中抛出一个 Promise，
      <code>
       Suspense
      </code>
      会等待它解析。
     </li>
     <li>
      确保错误边界组件包裹了所有需要捕获错误的子组件。
     </li>
     <li>
      记得处理好清理工作，以避免内存泄漏。
     </li>
    </ul>
