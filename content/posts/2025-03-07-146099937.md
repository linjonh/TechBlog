    <h2>
     <a id="1HyperV_backend_7">
     </a>
     1.Hyper-V backend
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9d9f724c98594cc2be37be00627f3377.png">
      <br/>
      我们知道，Docker Desktop 最开始的架构的后端是采用的 Hyper-V。 Docker daemon (dockerd) 运行在一个 Linux distro (LinuxKit build) 中，Docker client (docker) 则运行在 Windows 中。
     </img>
    </p>
    <h2>
     <a id="WSL_2_backend_12">
     </a>
     WSL 2 backend
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f48612b49e874fcf8f857691940fdb65.png">
      <br/>
      在 Docker Desktop WSL 2 backend 中，Docker 官方由根据 WSL 2 构建了新的 WSL2 distro (Bootstrapping distro)，运行在 WSL 2 Utility VM 中。
     </img>
    </p>
    <blockquote>
     <p>
      需要注意的是任何 WSL 2 distro 的 kernel 由微软提供。
     </p>
    </blockquote>
    <p>
     在这个名为 “docker-desktop” 的 WSL 2 distro 中运行着 Docker daemon，你可以通过执行命令来查看详细信息
    </p>
    <pre><code class="prism language-powershell">docker info
</code></pre>
