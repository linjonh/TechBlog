    <h2>
     <a id="_Python_Python_Dependencies_1">
     </a>
     一、什么是 Python 依赖（Python Dependencies）？
    </h2>
    <p>
     Python
     <strong>
      依赖（dependencies）
     </strong>
     指的是
     <strong>
      某个 Python 项目运行所需的外部库或模块
     </strong>
     。这些依赖通常由
     <code>
      pip
     </code>
     从
     <strong>
      Python Package Index（PyPI）
     </strong>
     或其他包管理器（如 Conda）下载和安装。
    </p>
    <hr/>
    <h3>
     <a id="1__6">
     </a>
     1. 依赖的作用
    </h3>
    <p>
     Python 标准库提供了一些基础功能（如
     <code>
      math
     </code>
     ,
     <code>
      os
     </code>
     ,
     <code>
      sys
     </code>
     ），但复杂的功能通常依赖
     <strong>
      第三方库
     </strong>
     ，如：
    </p>
    <ul>
     <li>
      <strong>
       Web 开发
      </strong>
      :
      <code>
       Flask
      </code>
      ,
      <code>
       Django
      </code>
     </li>
     <li>
      <strong>
       数据科学
      </strong>
      :
      <code>
       numpy
      </code>
      ,
      <code>
       pandas
      </code>
      ,
      <code>
       matplotlib
      </code>
     </li>
     <li>
      <strong>
       机器学习
      </strong>
      :
      <code>
       scikit-learn
      </code>
      ,
      <code>
       tensorflow
      </code>
      ,
      <code>
       torch
      </code>
     </li>
     <li>
      <strong>
       数据库
      </strong>
      :
      <code>
       SQLAlchemy
      </code>
      ,
      <code>
       PyMySQL
      </code>
     </li>
    </ul>
    <p>
     这些库通常
     <strong>
      不是 Python 内置
     </strong>
     的，需要额外安装，成为该项目的“依赖”。
    </p>
    <hr/>
    <h3>
     <a id="2__Python__17">
     </a>
     2. 如何管理 Python 依赖
    </h3>
    <p>
     <strong>
      （1）安装依赖
     </strong>
    </p>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> requests
</code></pre>
    <p>
     会安装
     <code>
      requests
     </code>
     及其所有依赖项。
    </p>
    <p>
     <strong>
      （2）列出已安装的依赖
     </strong>
    </p>
    <pre><code class="prism language-bash">pip list
</code></pre>
    <p>
     <strong>
      （3）导出依赖（生成 requirements.txt）
     </strong>
     <br/>
     如果希望在
     <strong>
      其他环境中安装相同的依赖
     </strong>
     ，可以使用：
    </p>
    <pre><code class="prism language-bash">pip freeze <span class="token operator">&gt;</span> requirements.txt
</code></pre>
    <p>
     <code>
      requirements.txt
     </code>
     示例：
    </p>
    <pre><code>numpy==1.24.2
pandas==1.5.3
requests==2.28.2
</code></pre>
    <p>
     <strong>
      （4）从 requirements.txt 安装依赖
     </strong>
    </p>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> <span class="token parameter variable">-r</span> requirements.txt
</code></pre>
    <hr/>
    <h3>
     <a id="3__47">
     </a>
     3. 依赖管理问题
    </h3>
    <p>
     （1）版本冲突
    </p>
    <ul>
     <li>
      项目 A 需要
      <code>
       numpy==1.21.0
      </code>
     </li>
     <li>
      项目 B 需要
      <code>
       numpy==1.24.0
      </code>
     </li>
     <li>
      如果两个项目共享环境，可能会导致
      <strong>
       版本冲突
      </strong>
      。
     </li>
    </ul>
    <p>
     <strong>
      解决方案：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       使用虚拟环境（venv）
      </strong>
      :
      <pre><code class="prism language-bash">python <span class="token parameter variable">-m</span> venv my_env
<span class="token builtin class-name">source</span> my_env/bin/activate  <span class="token comment"># Mac/Linux</span>
my_env<span class="token punctuation">\</span>Scripts<span class="token punctuation">\</span>activate  <span class="token comment"># Windows</span>
</code></pre>
     </li>
     <li>
      <strong>
       使用
       <code>
        pipenv
       </code>
      </strong>
      :
      <pre><code class="prism language-bash">pipenv <span class="token function">install</span>
pipenv shell
</code></pre>
     </li>
    </ul>
    <p>
     <strong>
      （2）依赖递归
     </strong>
     <br/>
     安装
     <code>
      pandas
     </code>
     可能会自动安装
     <code>
      numpy
     </code>
     ，因为
     <code>
      pandas
     </code>
     依赖于
     <code>
      numpy
     </code>
     。某些库的依赖会影响项目的整体结构。
    </p>
    <p>
     <strong>
      解决方案
     </strong>
     ：
    </p>
    <ul>
     <li>
      定期使用
      <code>
       pip list --outdated
      </code>
      检查更新。
     </li>
     <li>
      使用
      <code>
       pipdeptree
      </code>
      可视化依赖关系：
      <pre><code class="prism language-bash">pip <span class="token function">install</span> pipdeptree
pipdeptree
</code></pre>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="4__79">
     </a>
     4. 依赖锁定
    </h3>
    <p>
     在团队开发或生产环境中，建议使用
     <strong>
      依赖锁定
     </strong>
     确保环境一致：
    </p>
    <ul>
     <li>
      <strong>
       pip
      </strong>
      :
      <code>
       requirements.txt
      </code>
     </li>
     <li>
      <strong>
       pipenv
      </strong>
      :
      <code>
       Pipfile.lock
      </code>
     </li>
     <li>
      <strong>
       poetry
      </strong>
      :
      <code>
       poetry.lock
      </code>
     </li>
    </ul>
    <p>
     示例：
    </p>
    <pre><code class="prism language-bash">pip freeze <span class="token operator">&gt;</span> requirements.txt
pip <span class="token function">install</span> <span class="token parameter variable">-r</span> requirements.txt
</code></pre>

        </code>

      </strong>
     </p>
    </blockquote>
    <h2>
     <a id="pipvenv_106">
     </a>
     二、使用pip安装包+venv隔离环境
    </h2>
    <p>
     使用
     <code>
      pip
     </code>
     安装包时，推荐结合
     <strong>
      虚拟环境（virtual environment）
     </strong>
     来管理 Python 依赖，避免不同项目的包版本冲突。主要方法包括
     <strong>
      venv
     </strong>
     （Python 内置）、
     <strong>
      virtualenv
     </strong>
     （更强大）、
     <strong>
      pipenv
     </strong>
     （自动管理依赖）。
    </p>
    <h3>
     <a id="_1_venv_108">
     </a>
     方法 1：使用 venv（推荐）
    </h3>
    <p>
     <code>
      venv
     </code>
     是 Python 内置的虚拟环境工具，适用于大多数情况。
    </p>
    <h4>
     <a id="_111">
     </a>
     创建虚拟环境
    </h4>
    <pre><code class="prism language-bash">python <span class="token parameter variable">-m</span> venv my_env  <span class="token comment"># 创建名为 my_env 的环境</span>
</code></pre>
    <h4>
     <a id="_116">
     </a>
     激活环境
    </h4>
    <ul>
     <li>
      <strong>
       Windows（CMD）
      </strong>
      :
      <pre><code class="prism language-bash">my_env<span class="token punctuation">\</span>Scripts<span class="token punctuation">\</span>activate
</code></pre>
     </li>
     <li>
      <strong>
       Windows（PowerShell）
      </strong>
      :
      <pre><code class="prism language-powershell">my_env\Scripts\Activate<span class="token punctuation">.</span>ps1
</code></pre>
     </li>
     <li>
      <strong>
       Mac/Linux
      </strong>
      :
      <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> my_env/bin/activate
</code></pre>
     </li>
    </ul>
    <h4>
     <a id="_130">
     </a>
     安装包
    </h4>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> package_name
</code></pre>
    <h4>
     <a id="_135">
     </a>
     冻结（导出）环境
    </h4>
    <pre><code class="prism language-bash">pip freeze <span class="token operator">&gt;</span> requirements.txt
</code></pre>
    <p>
     可用于在新环境中复现安装：
    </p>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> <span class="token parameter variable">-r</span> requirements.txt
</code></pre>
    <h4>
     <a id="_144">
     </a>
     退出环境
    </h4>
    <pre><code class="prism language-bash">deactivate
</code></pre>
    <h4>
     <a id="_149">
     </a>
     删除环境
    </h4>
    <pre><code class="prism language-bash"><span class="token function">rm</span> <span class="token parameter variable">-rf</span> my_env  <span class="token comment"># Mac/Linux</span>
rd /s /q my_env  <span class="token comment"># Windows</span>
</code></pre>
    <hr/>
    <h3>
     <a id="_2_virtualenv_Python_2_157">
     </a>
     方法 2：使用 virtualenv（适用于 Python 2）
    </h3>
    <p>
     <code>
      virtualenv
     </code>
     与
     <code>
      venv
     </code>
     类似，但支持更多 Python 版本，创建的环境更独立。
    </p>
    <h4>
     <a id="_virtualenv_159">
     </a>
     安装 virtualenv
    </h4>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> virtualenv
</code></pre>
    <h4>
     <a id="_164">
     </a>
     创建环境
    </h4>
    <pre><code class="prism language-bash">virtualenv my_env
</code></pre>
    <blockquote>
     <p>
      <strong>
       与
       <code>
        venv
       </code>
       类似的激活、安装、退出操作
      </strong>
     </p>
    </blockquote>
    <hr/>
    <h3>
     <a id="_3_pipenv_172">
     </a>
     方法 3：使用 pipenv（自动管理依赖）
    </h3>
    <p>
     <code>
      pipenv
     </code>
     结合了
     <code>
      pip
     </code>
     和
     <code>
      venv
     </code>
     ，自动管理虚拟环境，并创建
     <code>
      Pipfile
     </code>
     追踪依赖。
    </p>
    <h4>
     <a id="_pipenv_175">
     </a>
     安装 pipenv
    </h4>
    <pre><code class="prism language-bash">pip <span class="token function">install</span> pipenv
</code></pre>
    <h4>
     <a id="_180">
     </a>
     创建和激活环境
    </h4>
    <pre><code class="prism language-bash">pipenv <span class="token function">install</span>
pipenv shell
</code></pre>
    <h4>
     <a id="_186">
     </a>
     安装包
    </h4>
    <pre><code class="prism language-bash">pipenv <span class="token function">install</span> package_name
</code></pre>
    <h4>
     <a id="_191">
     </a>
     导出依赖
    </h4>
    <pre><code class="prism language-bash">pipenv lock
</code></pre>
    <h4>
     <a id="_196">
     </a>
     退出环境
    </h4>
    <pre><code class="prism language-bash"><span class="token builtin class-name">exit</span>
</code></pre>
    <hr/>

    <h2>
     <a id="Conda_212">
     </a>
     三、使用Conda进行包安装和环境管理
    </h2>
    <p>
     Conda是Anaconda中一个强大的包和环境管理工具，可以在Windows的Anaconda Prompt命令行使用，也可以在macOS或者Linux系统的终端窗口(terminal window)的命令行使用。
     <br/>
     如何下载及使用Anaconda进行包管理和环境管理，推荐文章：https://blog.csdn.net/chenxy_bwave/article/details/119996001
    </p>
    <h2>
     <a id="pip_installconda_install_215">
     </a>
     四、pip install和conda install
    </h2>
    <h3>
     <a id="41_pip_install_vs_conda_install_217">
     </a>
     4.1 pip install vs conda install
    </h3>
    <p>
     pip install
     <code>
      和
     </code>
     conda install` 都是 Python 包管理工具，但它们的工作方式和适用场景不同：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        <code>
         pip install
        </code>
       </th>
       <th>
        <code>
         conda install
        </code>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         管理工具
        </strong>
       </td>
       <td>
        <code>
         pip
        </code>
        (Python 包管理器)
       </td>
       <td>
        <code>
         conda
        </code>
        (Anaconda 包管理器)
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         包来源
        </strong>
       </td>
       <td>
        PyPI（Python Package Index）
       </td>
       <td>
        Anaconda 仓库（含 Conda Forge、defaults）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         支持的包类型
        </strong>
       </td>
       <td>
        仅 Python 相关的包
       </td>
       <td>
        不仅支持 Python 包，还支持 C/C++ 依赖包（如
        <code>
         numpy
        </code>
        、
        <code>
         scipy
        </code>
        等的底层依赖）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         依赖管理
        </strong>
       </td>
       <td>
        仅管理 Python 依赖，需手动解决非 Python 依赖问题
       </td>
       <td>
        自动管理 Python 和非 Python 依赖
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         虚拟环境管理
        </strong>
       </td>
       <td>
        结合
        <code>
         venv
        </code>
        或
        <code>
         virtualenv
        </code>
        使用
       </td>
       <td>
        结合
        <code>
         conda
        </code>
        环境使用
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         安装方式
        </strong>
       </td>
       <td>
        直接安装
        <code>
         .whl
        </code>
        或
        <code>
         .tar.gz
        </code>
       </td>
       <td>
        使用预编译的二进制包（更快、更稳定）
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         速度和兼容性
        </strong>
       </td>
       <td>
        可能因编译问题导致安装失败
       </td>
       <td>
        通常安装更快，兼容性更好
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="42_PyPIAnaconda__230">
     </a>
     4.2 PyPI和Anaconda 仓库
    </h3>
    <h4>
     <a id="421_PyPIPython_Package_Index_232">
     </a>
     4.2.1 PyPI（Python Package Index）
    </h4>
    <p>
     <strong>
      Python Package Index（PyPI）
     </strong>
     是 Python 官方的软件包仓库，类似于软件商店，开发者可以在上面发布 Python 库，用户可以通过
     <code>
      pip install
     </code>
     下载和安装这些库。
    </p>
    <ul>
     <li>
      <strong>
       官网
      </strong>
      :
      <a href="https://pypi.org" rel="nofollow">
       https://pypi.org
      </a>
     </li>
     <li>
      <strong>
       安装命令
      </strong>
      :
      <code>
       pip install package_name
      </code>
     </li>
     <li>
      <strong>
       包格式
      </strong>
      :
      <code>
       .whl
      </code>
      （Wheel）或
      <code>
       .tar.gz
      </code>
      （源代码）
     </li>
     <li>
      <strong>
       特点
      </strong>
      :
      <ul>
       <li>
        <strong>
         通用性强
        </strong>
        ，支持所有 Python 发行版（CPython、PyPy等）
       </li>
       <li>
        <strong>
         只包含 Python 相关的包
        </strong>
        ，不提供 C/C++ 依赖项
       </li>
       <li>
        <strong>
         可能需要编译
        </strong>
        ，某些包依赖于 C/C++ 库，需要额外安装依赖（如
        <code>
         numpy
        </code>
        、
        <code>
         scipy
        </code>
        ）
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="422_Anaconda__243">
     </a>
     4.2.2 Anaconda 仓库
    </h4>
    <p>
     Anaconda 主要面向
     <strong>
      数据科学
     </strong>
     和
     <strong>
      机器学习
     </strong>
     ，它的包管理系统
     <strong>
      Conda
     </strong>
     维护了多个官方和社区仓库，包括defaluts仓库和Conda Forge仓库。
    </p>
    <h5>
     <a id="1defaults__245">
     </a>
     （1）defaults 仓库
    </h5>
    <ul>
     <li>
      由
      <strong>
       Anaconda 官方维护
      </strong>
      ，默认包含的包源。预编译好的二进制包，安装速度快，兼容性强
     </li>
     <li>
      适用于数据科学、机器学习的常见库（如
      <code>
       numpy
      </code>
      ,
      <code>
       pandas
      </code>
      ,
      <code>
       scipy
      </code>
      ,
      <code>
       matplotlib
      </code>
      ）
     </li>
     <li>
      <strong>
       安装命令
      </strong>
      :
      <code>
       conda install package_name
      </code>
     </li>
    </ul>
    <h5>
     <a id="2Conda_Forge_250">
     </a>
     （2）Conda Forge
    </h5>
    <ul>
     <li>
      <strong>
       社区驱动
      </strong>
      的 Conda 仓库，包含更多第三方库，维护更新比
      <code>
       defaults
      </code>
      更快，包的种类更多
     </li>
     <li>
      安装时可以用
      <code>
       -c conda-forge
      </code>
      指定
      <pre><code class="prism language-bash">conda <span class="token function">install</span> <span class="token parameter variable">-c</span> conda-forge package_name
</code></pre>
     </li>
     <li>
      适用于
      <code>
       defaults
      </code>
      没有的包，或者需要较新版本的包
     </li>
    </ul>

    <h3>
     <a id="43_pip_install__conda_install_272">
     </a>
     4.3 pip install 和 conda install的安装位置
    </h3>
    <ul>
     <li>
      conda install xxx：这种方式安装的库都会放在anaconda3/pkgs目录下，这样的好处就是，当在某个环境下已经下载好了某个库，再在另一个环境中还需要这个库时，就可以直接从pkgs目录下将该库复制至新环境而不用重复下载。
     </li>
     <li>
      pip install xxx：分两种情况，一种情况就是当前conda环境的python是conda安装的，和系统的不一样，那么xxx会被安装到anaconda3/envs/current_env/lib/python3.x/site-packages文件夹中，如果当前conda环境用的是系统的python，那么xxx会通常会被安装到~/.local/lib/python3.x/site-packages文件夹中
     </li>
    </ul>

    </h3>
    <ul>

      <strong>
       Anaconda
      </strong>

      </strong>

      <strong>

      </strong>
      、
      <strong>
       机器学习
      </strong>
      或
      <strong>

      <code>

     五、其他包管理工具

    <h3>
     <a id="uv_298">
     </a>
     uv
    </h3>
    <p>
     是一个高性能的 Python 包管理工具，旨在替代 pip，提供更快的包安装速度。
    </p>
    <h3>
     <a id="poetry_300">
     </a>
     poetry
    </h3>
    <p>
     poetry 是一个现代化的 Python 包管理和构建工具，集成了依赖管理、虚拟环境管理和包发布功能。特点如下：
    </p>
    <ul>
     <li>
      功能：集成依赖管理、虚拟环境管理和包发布。
     </li>
     <li>
      依赖管理：使用 pyproject.toml 和 poetry.lock 文件记录依赖，自动解析并安装所需库。
     </li>
     <li>
      环境管理：自动为每个项目创建和管理虚拟环境，确保项目独立性。
     </li>
     <li>
      包发布：简化包的构建和发布流程，支持发布到 PyPI 等包索引。
     </li>
     <li>
      适用场景：适合大型或复杂项目，需要严格依赖管理和包发布功能的情况。
     </li>
    </ul>
