    <h2>
     <a id="_9">
     </a>
     为什么需要动态内存管理
    </h2>
    <h3>
     <a id="_10">
     </a>
     静态分配的局限性
    </h3>
    <p>
     静态数组：
     <code>
      int arr[100]
     </code>
     需要编译时确定，且空间开辟是固定的
     <br/>
     自动变量：函数内定义的局部变量生命周期受限于函数栈帧
    </p>
    <p>
     但是对于空间的需求，不仅仅是上述的情况。有时候我们需要的空间大小在程序运行的时候才能知道，那数组的编译时开辟空间的方式就不能满足了。这时候就只能试试动态存开辟了。
    </p>
    <h3>
     <a id="_16">
     </a>
     动态分配的优势
    </h3>
    <ul>
     <li>
      运行时按需分配（如根据用户输入决定数据规模）
     </li>
     <li>
      灵活调整内存大小（动态扩容/缩容）
     </li>
     <li>
      突破栈空间限制（堆空间通常比栈大得多）
     </li>
     <li>
      实现跨函数的数据持久化（如链表节点）
     </li>
    </ul>
    <h2>
     <a id="_22">
     </a>
     动态内存函数
    </h2>
    <h3>
     <a id="malloc_23">
     </a>
     malloc
    </h3>
    <h4>
     <a id="_24">
     </a>
     函数介绍
    </h4>
    <p>
     <mark>
      函数地址
     </mark>
     ：
     <a href="https://legacy.cplusplus.com/reference/cstdlib/malloc/" rel="nofollow">
      malloc
     </a>
     <br/>
     <mark>
      函数格式
     </mark>
     ：
     <code>
      void* malloc (size_t size);
     </code>
     <br/>
     <mark>
      功能
     </mark>
     ：分配一个大小字节的内存块，返回一个指向该块开头的指针。新分配的内存块的内容没有初始化，保留不确定的值。
     <br/>
     <mark>
      返回值
     </mark>
     ：如果成功，则返回一个指向函数分配的内存块的指针。如果函数分配请求的内存块失败，则返回空指针。
     <br/>
     <mark>
      头文件
     </mark>
     ：
     <code>
      strstr
     </code>
     函数的声明在
     <code>
      stdlib.h
     </code>
     的头文件中，在使用时，要使用
     <code>
      #include &lt;stdlib.h&gt;
     </code>
     操作包含头文件。
     <br/>
     <mark>
      特别注意
     </mark>
     ：
    </p>
    <ol>
     <li>
      如果size为0，则返回值取决于特定的库实现（它可能是也可能不是空指针），但是返回的指针不能被解引用。
     </li>
     <li>
      该指针的类型始终为void*，可以将其强制转换为所需的数据指针类型，以实现可解引用。
     </li>
    </ol>

    <h3>
     <a id="free_62">
     </a>
     free
    </h3>
    <h4>
     <a id="_63">
     </a>
     函数介绍
    </h4>
    <p>
     <mark>
      函数地址
     </mark>
     ：
     <a href="https://legacy.cplusplus.com/reference/cstdlib/free/" rel="nofollow">
      free
     </a>
     <br/>
     <mark>
      函数格式
     </mark>
     ：
     <code>
      void free (void* ptr);
     </code>
     <br/>
     <mark>
      功能
     </mark>
     ：释放了先前通过调用
     <code>
      malloc
     </code>
     、
     <code>
      calloc
     </code>
     或
     <code>
      realloc
     </code>
     分配的内存块，使其再次可用以进行进一步分配。
     <br/>
     <mark>
      返回值
     </mark>
     ：
     <code>
      void
     </code>
     <br/>
     <mark>
      头文件
     </mark>
     ：
     <code>
      strstr
     </code>
     函数的声明在
     <code>
      stdlib.h
     </code>
     的头文件中，在使用时，要使用
     <code>
      #include &lt;stdlib.h&gt;
     </code>
     操作包含头文件。
     <br/>
     <mark>
      特别注意
     </mark>
     ：
    </p>
    <ol>
     <li>
      如果
      <code>
       ptr
      </code>
      没有指向由上述函数分配的内存块，则会导致未定义的行为。
     </li>
     <li>
      如果
      <code>
       ptr
      </code>
      是空指针，则该函数不执行任何操作。
     </li>
     <li>
      <code>
       free
      </code>
      函数没有改变
      <code>
       ptr
      </code>
      本身的值，因此它仍然指向相同的（现在无效的）位置。
     </li>
    </ol>

    <h3>
     <a id="realloc_119">
     </a>
     realloc
    </h3>
    <h4>
     <a id="C11_120">
     </a>
     函数介绍(C11标准)
    </h4>
    <p>
     <mark>
      函数地址
     </mark>
     ：
     <a href="https://legacy.cplusplus.com/reference/cstdlib/realloc/" rel="nofollow">
      realloc
     </a>
     <br/>
     <mark>
      函数格式
     </mark>
     ：
     <code>
      void* realloc (void* ptr, size_t size);
     </code>
     <br/>
     <mark>
      功能
     </mark>
     ：改变
     <code>
      ptr
     </code>
     所指向的内存块的大小。
     <br/>
     <mark>
      返回值
     </mark>
     ：如果成功，则返回一个指向函数分配的内存块的指针。如果函数分配请求的内存块失败，则返回空指针，表示
     <code>
      ptr
     </code>
     所指向的块没有被修改。
     <br/>
     <mark>
      头文件
     </mark>
     ：
     <code>
      strstr
     </code>
     函数的声明在
     <code>
      stdlib.h
     </code>
     的头文件中，在使用时，要使用
     <code>
      #include &lt;stdlib.h&gt;
     </code>
     操作包含头文件。
     <br/>
     <mark>
      特别注意
     </mark>
     ：
    </p>
    <ol>
     <li>
      如果
      <code>
       size
      </code>
      为0，则返回值取决于特定的库实现：它可以是空指针，也可以是其他不能解引用的位置。
     </li>
     <li>
      如果
      <code>
       ptr
      </code>
      是空指针，则该函数的行为类似于
      <code>
       malloc
      </code>
      ，分配一个新的大小字节块并返回一个指向其起点的指针。
     </li>
     <li>
      如果函数分配请求的内存块失败，则返回空指针，并且参数
      <code>
       ptr
      </code>
      指向的内存块不会被释放（它仍然有效，其内容不变）。
     </li>
     <li>
      <strong>
       这个函数调整原内存空间大小的基础上，还会将原来内存中的数据移动到
       <code>
        新
       </code>
       的空间。
      </strong>
     </li>
     <li>
      <strong>
       realloc在调整内存空间的是存在两种情况：
      </strong>
      <br/>
      情况1：原有空间之后有足够大的空间
      <br/>
      情况2：原有空间之后没有足够大的空间
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9d5e16aee46c4ec3b26f5752e570b4fa.png">
       <br/>
       <strong>
        情况1
       </strong>
       ：当是情况1 的时候，要扩展内存就直接原有内存之后直接追加空间，原来空间的数据不发生变化。
       <br/>
       <strong>
        情况2
       </strong>
       ：当是情况2 的时候，原有空间之后没有足够多的空间时，扩展的方法是：在堆空间上另找一个合适大小的连续空间来使用。这样函数返回的是一个新的内存地址。
       <br/>
       由于上述的两种情况，
       <code>
        realloc
       </code>
       函数的使用就要注意一些。
      </img>
     </li>
    </ol>
    <h4>
     <a id="_138">
     </a>
