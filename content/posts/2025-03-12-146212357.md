    <h4>
     <a id="_0">
     </a>
     <strong>
      消息重复消费的原因
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       生产者重试
      </strong>
      ：网络波动导致生产者未收到 Broker 确认，重复发送消息。
     </li>
     <li>
      <strong>
       消费者失败
      </strong>
      ：消费者处理消息后未发送 ACK，消息重新入队。
     </li>
     <li>
      <strong>
       集群故障转移
      </strong>
      ：主节点宕机，未确认消息被重新投递。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_7">
     </a>
     <strong>
      解决方案
     </strong>
    </h4>

     </strong>

    <ul>
     <li>
      <strong>

      </strong>

     </li>
     <li>
      <strong>
       组合使用多种方案
      </strong>
      ：例如“手动ACK + Redis去重”兼顾可靠性与性能。
     </li>
     <li>
      <strong>
       监控与告警
