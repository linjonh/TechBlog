     Jenkins 安装插件后构建成功但未启动容器的解决方法
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <p>
     在使用 Jenkins 构建 Docker 容器时，可能会遇到构建成功但没有容器启动的情况。这通常是由于在安装完 Docker 插件后未重启 Jenkins 服务，导致插件未生效。
    </p>
    <h3>
     问题描述
    </h3>
    <p>
     在 Jenkins 中安装了 Docker 插件后，直接进行构建，构建结果显示成功，但没有 Docker 容器启动。这是因为插件在安装后需要 Jenkins 重启才能生效。
    </p>
    <h3>
     解决步骤
    </h3>
    <ol>
     <li>
      <p>
       ​
       <strong>
        安装 Docker 插件
       </strong>
      </p>
      <ul>
       <li>
        登录 Jenkins 管理界面。
       </li>
       <li>
        导航到 ​
        <strong>
         Manage Jenkins
        </strong>
        -&gt; ​
        <strong>
         Manage Plugins
        </strong>
        。
       </li>
       <li>
        在 ​
        <strong>
         Available
        </strong>
        标签页中搜索并安装 ​
        <strong>
         Docker Plugin
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       ​
       <strong>
        重启 Jenkins 服务
       </strong>
      </p>
      <ul>
       <li>
        安装完插件后，必须重启 Jenkins 以使插件生效。
       </li>
      </ul>
     </li>
     <li>
      <ul>
       <li>
        重启 Jenkins 后，重新构建你的项目。
       </li>
       <li>
        确保构建脚本中正确配置了 Docker 相关的命令，例如
        <code>
         docker build
        </code>
        和
        <code>
         docker run
        </code>
        。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     注意事项
    </h3>
    <ul>
     <li>
      ​
      <strong>
       插件安装后必须重启
      </strong>
      ：Jenkins 插件在安装后需要重启才能生效，特别是像 Docker 插件这样涉及到系统集成的插件。
     </li>
     <li>
      ​
      <strong>
       检查 Jenkins 日志
      </strong>
      ：如果问题仍然存在，可以查看 Jenkins 的日志文件，通常位于
      <code>
       /var/log/jenkins/jenkins.log
      </code>
      ，以获取更多信息。
     </li>
     <li>
      ​
      <strong>
       确保 Docker 服务正常运行
      </strong>
      ：在 Jenkins 服务器上确保 Docker 服务已启动并正常运行。
     </li>
    </ul>
