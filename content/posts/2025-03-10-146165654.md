     <h4>
      <a id="LeetCode_860__6">
      </a>
      LeetCode 860. 柠檬水找零题解
     </h4>
     <hr/>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7f6dc2b32f5e43b493eaf68ff80a61f8.png"/>
     </p>
     <h5>
      <a id="_14">
      </a>
      问题描述
     </h5>
     <p>
      在柠檬水摊上，每杯柠檬水售价 5 美元。顾客按顺序支付 5、10 或 20 美元，你需要正确找零（开始时无零钱）。若所有顾客都能成功找零，返回
      <code>
       true
      </code>
      ，否则返回
      <code>
       false
      </code>
      。
     </p>
     <hr/>
     <h5>
      <a id="_19">
      </a>
      解题思路
     </h5>
     <p>
      <strong>
       贪心算法
      </strong>
      是本题的最优解，核心在于
      <strong>
       优先使用高面额零钱
      </strong>
      ，保留灵活的小面额零钱应对后续找零需求。
     </p>
     <h6>
      <a id="_22">
      </a>
      关键步骤：
     </h6>
     <ol>
      <li>
       <strong>
        初始化零钱数量
       </strong>
       ：用
       <code>
        five
       </code>
       和
       <code>
        ten
       </code>
       分别记录 5 美元和 10 美元的数量。
      </li>
      <li>
       <strong>
        遍历账单
       </strong>
       ：
       <ul>
        <li>
         <strong>
          收到 5 美元
         </strong>
         ：直接收下，无需找零。
        </li>
        <li>
         <strong>
          收到 10 美元
         </strong>
         ：需找 5 美元，若无则失败。
        </li>
        <li>
         <strong>
          收到 20 美元
         </strong>
         ：优先找 10+5 美元的组合，若无则找 3 个 5 美元。若均无法满足，失败。
        </li>
       </ul>
      </li>
     </ol>

     <h5>
      <a id="_63">
      </a>
      算法分析
     </h5>
     <ul>
      <li>
       <strong>
        时间复杂度
       </strong>
       ：O(n)，仅需遍历一次账单数组。
      </li>
      <li>
       <strong>
        空间复杂度
       </strong>
       ：O(1)，仅用两个变量记录零钱数量。
      </li>
     </ul>

     <h5>
      <a id="_84">
      </a>
      贪心策略的正确性证明
     </h5>
     <ul>
      <li>
       <strong>
        为何优先用 10+5 找零 20 美元？
       </strong>
       <br/>
       5 美元可应对更多场景（如找零 10 美元需 1 张，找零 20 美元需 3 张），而 10 美元只能用于找零 20 美元。优先消耗 10 美元可保留 5 美元的灵活性。
      </li>
     </ul>
     <hr/>
     <h5>
      <a id="_90">
      </a>
      边界条件与测试用例
     </h5>
     <ol>
      <li>
       <strong>
        首顾客支付非 5 美元
       </strong>
       ：直接失败（如
       <code>
        bills = [10]
       </code>
       ）。
      </li>
      <li>
       <strong>
        中途零钱不足
       </strong>
       ：如
       <code>
        [5,5,10,10,20]
       </code>
       ，最后一位顾客需找 15 美元但只剩 10 美元，失败。
      </li>
      <li>
       <strong>
        大额支付需组合找零
       </strong>
       ：如
       <code>
        [5,10,20]
       </code>
       ，需正确处理 20 美元的两种找零方式。
      </li>
     </ol>
     <hr/>

      <a id="_97">
      </a>
      总结
     </h5>
     <p>
      本题通过贪心策略，在保证每一步局部最优（保留最多 5 美元）的同时实现全局最优解。关键点在于
      <strong>
       零钱面额的使用优先级
      </strong>
      和
      <strong>
       边界条件的处理
      </strong>
      。
     </p>
