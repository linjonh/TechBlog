    <h2>
     <a id="_0">
     </a>
     移动布局基本概念
    </h2>
    <h3>
     <a id="_viewport_2">
     </a>
     视口 viewport
    </h3>
    <ul>
     <li>
      概念：视口就是浏览器显示页面内容的屏幕区域
     </li>
     <li>
      分类
     </li>
    </ul>
    <table>
     <thead>
      <tr>
       <th>
        类型
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        布局视口 layout viewport
       </td>
       <td>
        解决早期的PC端页面在手机上显示的问题，一般为980px
       </td>
      </tr>
      <tr>
       <td>
        视觉视口 visual viewport
       </td>
       <td>
        是用户正在看到的网站的区域
       </td>
      </tr>
      <tr>
       <td>
        理想视口 ideal viewport
       </td>
       <td>
        最理想的视口尺寸，通常等于设备宽度
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      视口标签
     </li>
    </ul>
    <pre><code class="prism language-css">&lt;meta name=<span class="token string">"viewport"</span> content=<span class="token string">"width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0"</span>&gt;
</code></pre>
    <table>
     <thead>
      <tr>
       <th>
        属性
       </th>
       <th>
        说明
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        width
       </td>
       <td>
        设置视口的宽度，可以设置device-width的特殊值
       </td>
      </tr>
      <tr>
       <td>
        initial-scale
       </td>
       <td>
        初始缩放比，一般为1
       </td>
      </tr>
      <tr>
       <td>
        maximum-scale
       </td>
       <td>
        最大缩放比，一般为1
       </td>
      </tr>
      <tr>
       <td>
        minimum-scale
       </td>
       <td>
        最小缩放比，一般为1
       </td>
      </tr>
      <tr>
       <td>
        user-scalable
       </td>
       <td>
        是否允许用户缩放，一般为no或0，也可以是yes
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="_27">
     </a>
     二倍图
    </h3>
    <ul>
     <li>
      物理像素：是屏幕上真实存在的像素
     </li>
     <li>
      物理像素比：一个px能显示的物理像素点的个数，称为物理像素比
     </li>
     <li>
      在标准的viewport设置中，使用倍图来提高图片质量，解决在高清设备中的模糊问题
     </li>
    </ul>
    <p>
     当手机的物理像素比不为1时，就要使用倍图，例如二倍图中：
    </p>
    <ul>
     <li>
      <p>
       img插入图片：使用width和height属性指定图片缩放后的尺寸
      </p>
     </li>
     <li>
      <p>
       bg背景图片：使用background-size属性指定缩放后的尺寸
      </p>
     </li>
    </ul>
    <p>
     可选的参数：百分比、长度px、cover、contain
    </p>
    <p>
     cover：扩展图片，完全覆盖
    </p>
    <p>
     contain：扩展图片，让宽度或高度适应内容区域，不一定填满
    </p>
