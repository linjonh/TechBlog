     一、聚簇索引与非聚簇索引
    </h3>
    <h4>
     <a id="1__4">
     </a>
     1. 聚簇索引结构
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        存储方式
       </strong>
       <br/>
       InnoDB 中，聚簇索引的叶子节点直接存储
       <strong>
        完整数据行
       </strong>
       ，数据按主键值物理排序存储。
      </p>
      <ul>
       <li>
        主键索引即数据文件，非叶子节点存储主键范围和子节点指针
       </li>
       <li>
        数据行与主键索引绑定，主键顺序决定磁盘存储顺序
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        示例存储结构
       </strong>
      </p>
      <pre><code class="prism language-plaintext">B+树结构：
根节点 → [id&lt;100, page1] [100≤id&lt;200, page2]
叶子节点（page1）→ | id=50 | Alice | 25 | → | id=80 | Bob | 30 |  
（数据行按id排序存储）
</code></pre>
     </li>
    </ul>
    <h4>
     <a id="2__vs__18">
     </a>
     2. 聚簇索引 vs 非聚簇索引查询速度对比
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         查询类型
        </strong>
       </th>
       <th>
        聚簇索引性能
       </th>
       <th>
        非聚簇索引性能
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        主键查询
       </td>
       <td>
        ⚡️
        <strong>
         O(1)
        </strong>
        直接定位数据
       </td>
       <td>
        🐢 需两次查找（索引+回表）
       </td>
      </tr>
      <tr>
       <td>
        范围查询
       </td>
       <td>
        ⚡️ 顺序I/O快（数据连续存储）
       </td>
       <td>
        🐢 随机I/O多（数据分散）
       </td>
      </tr>
      <tr>
       <td>
        覆盖查询
       </td>
       <td>
        ✅ 无需回表
       </td>
       <td>
        若索引包含查询字段可避免回表
       </td>
      </tr>
     </tbody>
    </table>
    <blockquote>
     <p>
      <strong>
       结论
      </strong>
      ：主键查询优先使用聚簇索引，二级索引需评估回表代价
     </p>
    </blockquote>

     二、事务日志机制
    </h3>
    <h4>
     <a id="1_Undo_Log_31">
     </a>
     1. Undo Log（回滚日志）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        核心作用
       </strong>
      </p>
      <ul>
       <li>
        事务回滚：记录数据修改前的旧值
       </li>
       <li>
        MVCC 支持：存储多版本数据实现非阻塞读
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        存储结构
       </strong>
      </p>
      <pre><code class="prism language-plaintext">┌───────────────┐
| Undo Log Segment |
| - INSERT → 记录主键值         |
| - UPDATE → 旧数据镜像         |
| - DELETE → 数据完整拷贝       |
└───────────────┘
</code></pre>
      <ul>
       <li>
        通过 purge 线程定期清理已提交事务的旧版本
       </li>
      </ul>
     </li>
    </ul>

     五、性能优化实践
    </h3>
    <h4>
     <a id="1__128">
     </a>
     1. 索引优化技巧
    </h4>
    <ul>
     <li>
      主键设计为自增整型（减少页分裂）
     </li>
     <li>
      联合索引遵循最左前缀原则
      <pre><code class="prism language-sql"><span class="token comment">-- 有效使用索引</span>
<span class="token keyword">SELECT</span> <span class="token operator">*</span> <span class="token keyword">FROM</span> users <span class="token keyword">WHERE</span> name<span class="token operator">=</span><span class="token string">'Alice'</span> <span class="token operator">AND</span> age<span class="token operator">&gt;</span><span class="token number">20</span><span class="token punctuation">;</span> 
<span class="token comment">-- 索引设计为 (name, age)</span>
</code></pre>
     </li>
    </ul>
    <h4>
     <a id="2__137">
     </a>
     2. 事务控制建议
    </h4>
    <ul>
     <li>
      控制事务粒度：单个事务执行时间 &lt; 1秒
     </li>
     <li>
      批量操作分批次提交
      <pre><code class="prism language-sql"><span class="token keyword">SET</span> autocommit<span class="token operator">=</span><span class="token number">0</span><span class="token punctuation">;</span>

     </li>
    </ul>

     <a id="3__146">

     3. 锁优化策略
