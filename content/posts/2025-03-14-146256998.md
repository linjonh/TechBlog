     <a id="_32">
     </a>
     （二）核心文件详解
    </h4>
    <h5>
     <a id="1_Dockerfile_34">
     </a>
     1. Dockerfile
    </h5>
    <pre><code class="prism language-dockerfile"># 第一阶段：构建阶段
FROM alpine:latest AS builder

# 安装必要的依赖
RUN apk update &amp;&amp; apk add --no-cache \
    build-base \
    pcre-dev \
    zlib-dev \
    openssl-dev \
    wget \
    unzip \
    git

# 下载Nginx 1.25源码
RUN wget http://nginx.org/download/nginx-1.25.0.tar.gz &amp;&amp; \
    tar -zxvf nginx-1.25.0.tar.gz &amp;&amp; \
    rm nginx-1.25.0.tar.gz

# 下载Redis插件源码
RUN git clone https://github.com/openresty/redis2-nginx-module.git &amp;&amp; \
    cd redis2-nginx-module &amp;&amp; \
    git checkout v0.15

# 编译并安装Nginx
WORKDIR /nginx-1.25.0
RUN ./configure \
    --with-http_ssl_module \
    --add-module=../redis2-nginx-module \
    --prefix=/etc/nginx \
    --sbin-path=/usr/sbin/nginx \
    --conf-path=/etc/nginx/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/run/nginx.lock &amp;&amp; \
    make &amp;&amp; \
    make install

# 第二阶段：运行阶段
FROM alpine:latest

# 更换软件源
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories

# 安装运行时必要的依赖
RUN apk update &amp;&amp; apk add --no-cache \
    pcre \
    zlib \
    openssl

# 创建新用户
RUN addgroup -S nginxgroup &amp;&amp; \
    adduser -S -G nginxgroup nginxuser &amp;&amp; \
    id nginxuser

# 复制编译好的Nginx文件
COPY --from=builder /etc/nginx /etc/nginx
COPY --from=builder /usr/sbin/nginx /usr/sbin/nginx

# 复制自定义的 Nginx 配置文件
COPY nginx.conf /etc/nginx/nginx.conf

# 复制 SSL 证书
COPY ssl/cert.pem /etc/nginx/ssl/cert.pem
COPY ssl/key.pem /etc/nginx/ssl/key.pem

# 创建必要的目录
RUN mkdir -p /var/log/nginx
RUN mkdir -p /var/run/nginx
RUN mkdir -p /var/cache/nginx

# 以 root 用户执行权限修复
USER root
RUN chown -R nginxuser:nginxgroup /etc/nginx /var/log/nginx /var/run/nginx /var/cache/nginx /etc/nginx/ssl

# 复制启动脚本
COPY start_nginx.sh /start_nginx.sh
# 以 root 用户修改启动脚本权限
RUN chmod +x /start_nginx.sh

# 暴露 80 和 443 端口
EXPOSE 80 443

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s \
    CMD curl -f http://localhost/ || exit 1

# 以 root 用户启动脚本
CMD ["/start_nginx.sh"]
</code></pre>
    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       多阶段构建
      </strong>
      <ul>
       <li>
        目的：将构建过程分为两个阶段，第一阶段（
        <code>
         builder
        </code>
        ）用于构建 Nginx 并安装 Redis 模块，包含了许多构建工具和依赖，比较臃肿；第二阶段只复制第一阶段构建好的必要文件和运行时依赖，生成的最终镜像会非常精简，减少了镜像的体积，提高了安全性和部署效率。
       </li>
       <li>
        实现方式：通过
        <code>
         FROM alpine:latest AS builder
        </code>
        定义第一阶段，使用
        <code>
         COPY --from=builder
        </code>
        从第一阶段复制文件到第二阶段。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       基础镜像选择
      </strong>
      <ul>
       <li>
        选择
        <code>
         alpine:latest
        </code>
        作为基础镜像，因为 Alpine Linux 非常轻量级，它的镜像体积很小，适合作为基础镜像来构建 Docker 镜像，能减少最终镜像的大小。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        HEALTHCHECK
       </code>
       指令
      </strong>
      <ul>
       <li>
        作用：用于定期检查容器的健康状态。
       </li>
       <li>
        配置：
        <code>
         HEALTHCHECK --interval=30s --timeout=3s CMD curl -f http://localhost/ || exit 1
        </code>
        ，表示每隔 30 秒检查一次，如果 3 秒内
        <code>
         curl
        </code>
        请求本地地址失败，就认为容器不健康。
       </li>
       <li>
        应用场景：在容器编排系统（如 Docker Compose、Kubernetes）中自动重启不健康的容器。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2_nginxconf_140">
     </a>
     2. nginx.conf
    </h5>
    <pre><code class="prism language-nginx">user nginxuser nginxgroup;  # 指定用户和用户组
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

# 引入环境变量，放在 main 上下文
env BACKEND_SERVER;

events {
    worker_connections 1024;
    multi_accept on;
    use epoll;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    server_tokens off;

    # 启用 Gzip 压缩
    gzip on;
    gzip_disable "msie6";
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_buffers 16 8k;
    gzip_http_version 1.1;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 缓存配置
    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m use_temp_path=off;

    # 定义 upstream 块
    upstream backend {
        server $BACKEND_SERVER;
    }

    # HTTP 服务器配置
    server {
        listen 80;
        server_name _;

        # 重定向 HTTP 请求到 HTTPS
        #return 301 https://$host$request_uri;
        proxy_redirect http:// https://;
    }

    # HTTPS 服务器配置
    server {
        listen 443 ssl;
        server_name _;

        ssl_certificate /etc/nginx/ssl/cert.pem;
        ssl_certificate_key /etc/nginx/ssl/key.pem;

        # 尝试代理请求到后端服务
        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # 缓存配置
            proxy_cache my_cache;
            proxy_cache_valid 200 302 60m;
            proxy_cache_valid 404 1m;

            # 如果后端服务不可用，返回默认页面
            error_page 502 503 504 = @fallback;
        }

        # 错误页面配置
        error_page 500 /50x.html;
        location = /50x.html {
            root /usr/share/nginx/html;
        }

        # 备用页面配置
        location @fallback {
            root /usr/share/nginx/html;
            index index.html;
        }
    }
}
</code></pre>
    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       环境变量引入
      </strong>
      <ul>
       <li>
        方式：在
        <code>
         nginx.conf
        </code>
        中使用
        <code>
         env BACKEND_SERVER;
        </code>
        引入环境变量，然后在
        <code>
         upstream
        </code>
        块中使用
        <code>
         $BACKEND_SERVER
        </code>
        作为后端服务器地址。
       </li>
       <li>
        作用：可以在运行容器时通过环境变量动态指定后端服务器地址，提高了配置的灵活性。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Gzip 压缩配置
      </strong>
      <ul>
       <li>
        启用：
        <code>
         gzip on;
        </code>
        开启 Gzip 压缩功能。
       </li>
       <li>
        参数配置：如
        <code>
         gzip_comp_level 6
        </code>
        表示压缩级别为 6，
        <code>
         gzip_types
        </code>
        定义了需要压缩的文件类型。
       </li>
       <li>
        作用：减少数据传输量，提高网站的加载速度，尤其是对于文本类型的文件（如 HTML、CSS、JavaScript 等）效果明显。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       缓存配置
      </strong>
      <ul>
       <li>
        缓存路径定义：
        <code>
         proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=100m inactive=60m use_temp_path=off;
        </code>
        定义了缓存路径、缓存区大小等参数。
       </li>
       <li>
        缓存规则配置：在
        <code>
         location
        </code>
        块中使用
        <code>
         proxy_cache my_cache;
        </code>
        和
        <code>
         proxy_cache_valid 200 302 60m;
        </code>
        配置缓存规则。
       </li>
       <li>
        作用：减少对后端服务器的请求次数，提高响应速度，尤其是对于一些静态资源或频繁访问的页面。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="3_start_nginxsh_257">
     </a>
     3. start_nginx.sh
    </h5>
    <pre><code class="prism language-bash"><span class="token shebang important">#!/bin/sh</span>

<span class="token comment"># 检查环境变量是否设置</span>
<span class="token keyword">if</span> <span class="token punctuation">[</span> <span class="token parameter variable">-z</span> <span class="token string">"<span class="token variable">$BACKEND_SERVER</span>"</span> <span class="token punctuation">]</span><span class="token punctuation">;</span> <span class="token keyword">then</span>
    <span class="token builtin class-name">echo</span> <span class="token string">"Warning: BACKEND_SERVER environment variable is not set. Using localhost (127.0.0.1) as the backend server."</span>
    <span class="token assign-left variable">BACKEND_SERVER</span><span class="token operator">=</span><span class="token string">"127.0.0.1"</span>
<span class="token keyword">fi</span>

<span class="token comment"># 替换 nginx.conf 中的占位符</span>
<span class="token function">sed</span> <span class="token parameter variable">-i</span> <span class="token string">"s|\<span class="token variable">$BACKEND_SERVER</span>|<span class="token variable">$BACKEND_SERVER</span>|g"</span> /etc/nginx/nginx.conf

<span class="token comment"># 检查配置文件是否有效</span>
nginx <span class="token parameter variable">-t</span>

<span class="token comment"># 以 root 用户启动 Nginx，Nginx 会根据 nginx.conf 中的 user 指令切换用户</span>
<span class="token builtin class-name">exec</span> nginx <span class="token parameter variable">-g</span> <span class="token string">'daemon off;'</span>
</code></pre>
    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       环境变量检查与替换
      </strong>
      <ul>
       <li>
        检查：使用
        <code>
         if [ -z "$BACKEND_SERVER" ]; then
        </code>
        检查
        <code>
         BACKEND_SERVER
        </code>
        环境变量是否设置。
       </li>
       <li>
        替换：如果未设置，则使用默认值
        <code>
         127.0.0.1
        </code>
        ，然后使用
        <code>
         sed
        </code>
        命令替换
        <code>
         nginx.conf
        </code>
        中的占位符。
       </li>
       <li>
        作用：确保在不同环境中 Nginx 都能正确连接到后端服务器。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       配置文件检查
      </strong>
      <ul>
       <li>
        命令：
        <code>
         nginx -t
        </code>
        用于检查配置文件是否有效。
       </li>
       <li>
        作用：避免因配置错误导致 Nginx 启动失败。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_288">
     </a>
     （三）生成自签名证书（测试环境）
    </h4>
    <pre><code class="prism language-bash"><span class="token function">mkdir</span> ssl
openssl req <span class="token parameter variable">-x509</span> <span class="token parameter variable">-nodes</span> <span class="token parameter variable">-days</span> <span class="token number">365</span> <span class="token parameter variable">-newkey</span> rsa:2048 <span class="token punctuation">\</span>
    <span class="token parameter variable">-keyout</span> ssl/key.pem <span class="token punctuation">\</span>
    <span class="token parameter variable">-out</span> ssl/cert.pem <span class="token punctuation">\</span>
    <span class="token parameter variable">-subj</span> <span class="token string">"/C=CN/ST=Zhejiang/L=Hangzhou/O=Example Company/OU=IT Department/CN=example.com/emailAddress=admin@example.com"</span>
</code></pre>
    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       OpenSSL 命令
      </strong>
      <ul>
       <li>
        作用：使用 OpenSSL 生成自签名证书。
       </li>
       <li>
        参数解释：
        <code>
         -x509
        </code>
        表示生成自签名证书，
        <code>
         -nodes
        </code>
        表示不加密私钥，
        <code>
         -days 365
        </code>
        表示证书有效期为 365 天，
        <code>
         -newkey rsa:2048
        </code>
        表示生成 2048 位的 RSA 密钥，
        <code>
         -subj
        </code>
        用于一次性设置证书的各种信息（如国家、省份、公司等）。
       </li>
       <li>
        应用场景：在测试环境中可以用来测试 HTTPS 连接，但在生产环境中建议使用正规的 CA 机构颁发的证书。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_305">
     </a>
     （四）构建镜像
    </h4>
    <pre><code class="prism language-bash"><span class="token function">docker</span> build --cache-from nginx-1.25-redis-alpine <span class="token parameter variable">-t</span> nginx-1.25-redis-alpine <span class="token builtin class-name">.</span> 
</code></pre>
    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       <code>
        docker build
       </code>
       命令
      </strong>
      <ul>
       <li>
        参数解释：
        <code>
         --cache-from
        </code>
        用于指定缓存镜像，
        <code>
         -t
        </code>
        用于指定镜像的标签，最后的
        <code>
         .
        </code>
        表示使用当前目录作为构建上下文。
       </li>
       <li>
        作用：根据 Dockerfile 构建 Docker 镜像。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_317">
     </a>
     （五）运行容器
    </h4>
    <pre><code class="prism language-bash"><span class="token function">docker</span> run <span class="token parameter variable">-d</span> <span class="token parameter variable">-p</span> <span class="token number">80</span>:80 <span class="token parameter variable">-p</span> <span class="token number">443</span>:443 nginx-1.25-redis-alpine

     <code>
      -e
     </code>
     参数：
    </p>
    <pre><code class="prism language-bash"><span class="token function">docker</span> run <span class="token parameter variable">-d</span> <span class="token parameter variable">-p</span> <span class="token number">80</span>:80 <span class="token parameter variable">-p</span> <span class="token number">443</span>:443 <span class="token parameter variable">-e</span> <span class="token assign-left variable">BACKEND_SERVER</span><span class="token operator">=</span>backend_server_ip:backend_server_port nginx-1.25-redis-alpine

    <p>
     <strong>
      知识点讲解
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       端口映射
      </strong>
      <ul>
       <li>
        方式：使用
        <code>
         docker run -p 80:80 -p 443:443
        </code>
        将容器的 80 和 443 端口映射到主机的对应端口。
       </li>
       <li>
        作用：通过主机的 IP 地址访问容器内的 Nginx 服务。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       环境变量传递
      </strong>
      <ul>
       <li>
        方式：使用
        <code>
         -e BACKEND_SERVER=backend_server_ip:backend_server_port
        </code>
        在运行容器时传递环境变量。
       </li>
       <li>
        作用：动态指定后端服务器地址。
       </li>
      </ul>
     </li>
    </ul>
