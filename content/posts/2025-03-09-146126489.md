    <h4>
     准备打包
    </h4>
    <p>
     笔者通过vite打包生成的目录如下，
    </p>
    <p>
     <img alt="" height="165" src="https://i-blog.csdnimg.cn/direct/7f30dee3790e4af29af9c26079ec785d.png" width="278"/>
    </p>
    <p>
     可惜没有一个关键的东西，需要在01-dist目录下新建一个package.json，才能运行。感觉很麻烦。
    </p>
    <p>
     但是，笔者突然想到
     <strong>
      只要在publice目录下写下package.json，打包之后不就有package.json
     </strong>
     了。
    </p>
    <p>
     <img alt="" height="116" src="https://i-blog.csdnimg.cn/direct/0ffbca2b320742e5b87b87d222b3292e.png" width="258"/>
    </p>
    <blockquote>
     <p>
      先public目录下新建一个package.json文件，内容如下
     </p>
    </blockquote>
    <pre><code>{
  "name": "01-dist",
  "version": "1.0.0",
  "main": "index.html",
  "window": {
    "icon": "vite.svg"
  }
}
</code></pre>
    <p>
     上面四点name，version，main，icon，缺一不可。
    </p>
    <p>
     少了一个，要么无法运行，要么报错。
    </p>
    <p>
     再次打包
    </p>
    <p>
     <img alt="" height="196" src="https://i-blog.csdnimg.cn/direct/22fcfbde7f434d3ba79869e16f18d04b.png" width="258"/>
    </p>
    <p>
     完美。
    </p>
    <h4>
     运行命令
    </h4>
    <pre><code>nwbuild --mode=build --glob=false --flavor=sdk --cacheDir=./node_modules/nw --outDir=../build ./</code></pre>
    <p>
     具体命令的含义可参考官网，也许可以把运行命令也写入public/package.json中
    </p>
    <pre><code>{
  "name": "01-dist",
  "main": "index.html",
  "version": "1.0.0",
  "window": {
    "icon": "vite.svg"
  },
  "scripts": {
    "build": "nwbuild --mode=build --glob=false --flavor=sdk --cacheDir=./node_modules/nw --outDir=../build ./"
  }
}
</code></pre>
    <p>
     再次打包。进入01-dist目录， 运行
     <strong>
      pnpm run build
     </strong>
    </p>

     再次思考，我是通过全局nw-builder进行打包。如果不使用全局，懂了，因此
    </p>
    <p>
     再次修改public/package.json目录，使其拥有nw-builder依赖。
    </p>
    <p>
     最终内容如下
    </p>
    <blockquote>
     <p>
      public/package.json
     </p>
    </blockquote>
    <pre><code>{
  "name": "01-dist",
  "main": "index.html",
  "version": "1.0.0",
  "window": {
    "icon": "vite.svg"
  },
  "dependencies": {
    "nw-builder": "^4.13.9"
  },
  "scripts": {
    "build": "nwbuild --mode=build --glob=false --flavor=sdk --cacheDir=./node_modules/nw --outDir=../build ./"
  }
}
</code></pre>
    <h3>
     工作流打包
    </h3>
    <h4>
     流程
    </h4>
    <p>
     <strong>
      1、读取仓库代码
     </strong>
    </p>
    <p>
     <strong>
      2、安装node
     </strong>
    </p>
    <p>
     <strong>
      3、第一次打包(vite）
     </strong>
    </p>
    <p>
     <strong>
      4、第二次打包(nw-builder)
     </strong>
    </p>
    <p>
     <strong>
      5、上传打包结果
     </strong>
    </p>
    <p>
     因此
    </p>
    <blockquote>
     <p>
      .github/workflows/01-start.yaml文件的内容如下
     </p>
    </blockquote>
    <pre><code>name: 打包01项目
on:
  workflow_dispatch:
    inputs:
      ad:
        description: '打包'
        required: true
        default: 'true'
        type: boolean
jobs:
  build:
    runs-on: windows-latest
    steps:
      - name: 读取项目
        uses: actions/checkout@v4

      - name: 设置node版本
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: 安装依赖并第一次打包
        run: |
          cd 01-quick-start
          npm install
          npm run build

      - name: 第二次打包
        run: |
          cd 01-dist
          npm install
          npm run build
      - name: 上传
        uses: actions/upload-artifact@v4
        with:
          name: build
          path: build
</code></pre>
