</code></pre>
    <h4>
     <a id="docker_run_32">
     </a>
     命令解析：
     <code>
      docker run
     </code>
    </h4>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：创建并启动一个心得容器
     </li>
     <li>
      <strong>
       参数
      </strong>
      ：
      <ul>
       <li>
        <code>
         -d
        </code>
        ：表示容器在后台运行（守护模式）。
       </li>
       <li>
        <code>
         -p 8080:80
        </code>
        ：将容器的 80 的端口映射到宿主机的 8080端口。这样，你可以通过访问宿主机的
        <code>
         8080
        </code>
        端口来访问容器内 Nginx 的服务。
       </li>
       <li>
        <code>
         --name my_test_nginx
        </code>
        ：为容器指定一个自定义名称。如果不指定，Docker 会自动生成一个随机名称。
       </li>
       <li>
        <code>
         nginx
        </code>
        ：指定使用的镜像名称。这里使用的是官方的 Nginx镜像（默认为最新版本
        <code>
         latest
        </code>
        ）
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a97e01d06c4f405d96b294c488ee3dea.png"/>
       </li>
      </ul>
     </li>
    </ul>

    <h5>
     <a id="docker_exec_82">
     </a>
     命令解析：docker exec
    </h5>
    <ul>
     <li>
      <strong>
       作用
      </strong>
      ：在运行中的 Docker 容器内部执行命令。
     </li>
     <li>
      <strong>
       参数
      </strong>
      ：
      <ul>
       <li>
        <code>
         -it
        </code>
        ：两个选项的组合：
        <ul>
         <li>
          <code>
           -i
          </code>
          ：（interactive）：保持 STDIN
         </li>
         <li>
          <code>
           -t
          </code>
          ：（tty）：分配一个伪终端，让你能够像操作普通终端一样操作容器内部的命令行。
         </li>
        </ul>
       </li>
       <li>
        <code>
         my_test_nginx
        </code>
        ：容器的名称或 ID。这是之前通过
        <code>
         doker
        </code>
       </li>
       <li>
        <code>
         /bin/bash
        </code>
        ：在容器内部执行的命令，这里指启动一个 Bash 终端
        <br/>
        这将打开一个交互式的 Bash 终端，让你可以在容器内部执行命令。
       </li>
      </ul>
     </li>
    </ul>

    <h2>
     <a id="3__107">
     </a>
     <strong>
      3. 使用自定义文件配置数据卷
     </strong>
    </h2>
    <p>
     使用 Docker 部署 Nginx 时，合理配置数据卷（Volumes）可以实现配置文件、静态资源和日志的持久化，同时便于管理和更新。
    </p>
    <h3>
     <a id="31__109">
     </a>
     <strong>
      3.1 创建数据卷文件
     </strong>
    </h3>
    <h4>
     <a id="311__Docker__110">
     </a>
     <strong>
      3.1.1 创建 Docker 数据卷
     </strong>
    </h4>
    <pre><code class="prism language-bash"><span class="token comment"># 在使用 docker volume create 创建的数据卷（Volumes）默认存储在 Docker 的内部目录中，而不是直接映射到宿主机的某个特定路径</span>
<span class="token function">docker</span> volume create nginx-config   <span class="token comment"># 配置文件</span>
<span class="token function">docker</span> volume create nginx-html     <span class="token comment"># 静态资源</span>
<span class="token function">docker</span> volume create nginx-logs     <span class="token comment"># 日志文件</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1e833860879d4125803cb4e06d456449.png"/>
    </p>
    <h4>
     <a id="312__Docker__118">
     </a>
     <strong>
      3.1.2 查看 Docker 数据卷内部路径
     </strong>
    </h4>
    <p>
     可以通过 docker volume inspect 命令查看数据卷的详细信息，包括其在宿主机上的实际存储路径。
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 查看数据卷的存储路径</span>
<span class="token function">docker</span> volume inspect nginx-config
</code></pre>

    <h3>
     <a id="32__125">
     </a>
     <strong>
      3.2 准备宿主机上的文件
     </strong>
    </h3>
    <p>
     在宿主机上准备需要挂载的文件和目录：
    </p>
    <ol>
     <li>
      <strong>
       配置文件
      </strong>
      ：将 Nginx 的配置文件（如
      <code>
       nginx.conf
      </code>
      和
      <code>
       default.conf
      </code>
      ）放在宿主机的某个目录中，例如
      <code>
       /home/user/nginx/nginx-config
      </code>
      。
     </li>
     <li>
      <strong>
       静态资源
      </strong>
      ：将静态文件（如 HTML、图片等）放在
      <code>
       /home/user/nginx/nginx-html
      </code>
      目录中。
     </li>
     <li>
      <strong>
       日志目录
      </strong>
      ：创建一个日志目录
      <code>
       /home/user/nginx/nginx-logs
      </code>
      ，用于存储 Nginx 的日志文件。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7096bf8402c7410990eb0e95ce10c5b9.png"/>
     </li>
    </ol>
    <h3>
     <a id="33__131">
     </a>
     <strong>
      3.3 将文件复制到数据卷
     </strong>
    </h3>
    <p>
     将本地文件复制到 Nginx 容器内部的原因主要有以下几点，这些原因涉及配置管理、数据持久化、开发便利性以及安全性等多个方面。以下是一些具体场景和原因：
    </p>
    <h4>
     <a id="331__134">
     </a>
     <strong>
      3.3.1. 配置管理
     </strong>
    </h4>
    <p>
     Nginx 的行为主要由其配置文件（如
     <code>
      nginx.conf
     </code>
     和站点配置文件）控制。在开发和部署过程中，你可能需要频繁修改这些配置文件以适应不同的需求。将本地配置文件复制到容器内部，可以实现以下目标：
    </p>
    <ul>
     <li>
      <strong>
       快速更新配置
      </strong>
      ：无需重新构建镜像，直接通过
      <code>
       docker cp
      </code>
      将修改后的配置文件复制到容器中，并重新加载 Nginx 服务即可生效。
     </li>
     <li>
      <strong>
       版本控制
      </strong>
      ：将配置文件存储在宿主机上，便于使用版本控制系统（如 Git）管理配置文件的变更历史。
     </li>
     <li>
      <strong>
       一致性
      </strong>
      ：确保开发环境和生产环境使用相同的配置文件，减少因环境差异导致的问题。
     </li>
    </ul>
    <h5>
     <a id="3311_nginxconfig_139">
     </a>
     3.3.1.1 首先启动容器并绑定数据卷nginx-config
    </h5>
    <pre><code class="prism language-bash"><span class="token comment"># </span>
<span class="token function">docker</span> run <span class="token parameter variable">-d</span> <span class="token parameter variable">-p</span> <span class="token number">8080</span>:80 <span class="token parameter variable">--name</span> my_test_nginx <span class="token parameter variable">-v</span> nginx-config:/etc/nginx nginx <span class="token function">tail</span> <span class="token parameter variable">-f</span> /dev/null
</code></pre>
    <h5>
     <a id="3312_nginxconfdefaultconf_144">
     </a>
     3.3.1.2 复制配置文件:nginx.conf和default.conf
    </h5>
    <pre><code class="prism language-bash"><span class="token comment"># 将宿主机本地的文件nginx.conf复制到docker里面</span>
<span class="token function">docker</span> <span class="token function">cp</span> /home/dgs/nginx/nginx-config/nginx.conf  my_test_nginx:/etc/nginx/nginx.conf

<span class="token function">docker</span> <span class="token function">cp</span> /home/dgs/nginx/nginx-config/default.conf my_test_nginx:/etc/nginx/conf.d/default.conf
</code></pre>
    <h5>
     <a id="3313__nginx__151">
     </a>
     3.3.1.3 重启 nginx 容器
    </h5>
    <pre><code class="prism language-bash"><span class="token comment"># docker exec -it &lt;容器名称&gt; nginx -s reload</span>
<span class="token function">docker</span> <span class="token builtin class-name">exec</span> <span class="token parameter variable">-it</span> my_test_nginx nginx <span class="token parameter variable">-s</span> reload
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/886619651978458a920abe790112ff9e.png"/>
    </p>
    <h4>
     <a id="332__157">
     </a>
     <strong>
      3.3.2. 静态资源管理
     </strong>
    </h4>
    <p>
     Nginx 通常用于提供静态资源（如 HTML 文件、图片、CSS 文件等）。将本地静态资源复制到容器内部，可以实现以下目标：
    </p>
    <ul>
     <li>
      <strong>
       快速部署
      </strong>
      ：直接将静态资源复制到容器的
      <code>
       /usr/share/nginx/html
      </code>
      目录中，无需重新构建镜像，即可快速更新网站内容。
     </li>
     <li>
      <strong>
       开发便利性
      </strong>
      ：在开发过程中，你可以直接在宿主机上修改静态资源，然后通过
      <code>
       docker cp
      </code>

     </li>

    <h4>
     <a id="333__183">
     </a>
     <strong>
      3.3.3. 日志管理
     </strong>
    </h4>
    <p>
     Nginx 的日志文件（如
     <code>
      access.log
     </code>
     和
     <code>
      error.log
     </code>
     ）通常存储在
     <code>
      /var/log/nginx
     </code>
     目录中。将日志文件存储在宿主机上，可以实现以下目标：
    </p>
    <ul>
     <li>
      <strong>
       持久化日志
      </strong>
      ：即使容器被删除，日志文件仍然保留在宿主机上，便于后续分析和审计。
     </li>
     <li>
      <strong>
       集中管理
      </strong>
      ：将日志文件存储在宿主机的某个目录中，便于使用日志管理工具（如
      <code>
       logrotate
      </code>
      ）进行集中管理和轮转。
     </li>
     <li>
      <strong>
       安全审计
      </strong>
      ：日志文件是安全审计的重要依据，将其存储在宿主机上可以提高安全性。
     </li>
    </ul>

    <h2>
     <a id="4__207">
     </a>
     <strong>
      4 重点总结
     </strong>
    </h2>
    <p>
     通过本次学习，我们全面掌握了 Docker 的基础命令，从镜像的拉取与管理，到容器的创建、启动、停止与删除，再到日志与数据的管理以及配置文件的操作。以 Nginx 为例的实践操作，让我们能够将理论知识应用到实际场景中，为后续的开发与部署工作打下了坚实的基础。Docker 的这些基础命令是日常开发和运维工作中最常用的工具，熟练掌握它们将大大提高我们的工作效率。
    </p>
    <ul>
     <li>
      搜索镜像：
      <code>
       docker search nginx
      </code>
      ：用于在 Docker Hub 上搜索 Nginx 镜像。
     </li>
     <li>
      拉取镜像：
      <code>
       docker pull nginx
      </code>
      ：从 Docker Hub 拉取官方的 Nginx 镜像。
     </li>
     <li>
      查看本地镜像：
      <code>
       docker images
      </code>
      ：列出本地存储的所有镜像。
     </li>
     <li>
      删除镜像：
      <code>
       docker rmi nginx
      </code>
      ：删除本地的 Nginx 镜像。
     </li>
     <li>
      运行容器：
      <code>
       docker run -d -p 8080:80 --name my_test_nginx nginx
      </code>
      ：在后台运行一个 Nginx 容器，并将容器的 80 端口映射到宿主机的 8080 端口。
     </li>
     <li>
      查看运行中的容器：
      <code>
       docker ps
      </code>
      ：列出当前正在运行的容器。
     </li>
     <li>
      查看所有容器：
      <code>
       docker ps -a
      </code>
      ：列出所有容器，包括已停止的。
     </li>
     <li>
      停止容器：
      <code>
       docker stop my_test_nginx
      </code>
      ：停止名为 my_test_nginx 的容器。
     </li>
     <li>
      启动容器：
      <code>
       docker start my_test_nginx
      </code>
      ：启动已停止的容器。
     </li>
     <li>
      删除容器：
      <code>
       docker rm my_test_nginx
      </code>
      ：删除容器。
     </li>
     <li>
      进入容器：
      <code>
       docker exec -it my_test_nginx /bin/bash
      </code>
      ：进入正在运行的容器并启动交互式 Bash 终端。
     </li>
     <li>
      查看容器日志：
      <code>
       docker logs my_test_nginx
      </code>
      ：查看 Nginx 容器的日志输出。
     </li>
     <li>
      将日志复制到宿主机：
      <code>
       docker cp my_test_nginx:/var/log/nginx/access.log /path/to/destination/access.log
      </code>
      ：将容器内的日志文件复制到宿主机指定路径。
     </li>
     <li>
      使用数据卷挂载日志目录：
      <code>
       docker run -d --name my_test_nginx -v /home/user/nginx-logs:/var/log/nginx nginx
      </code>
      ：将容器的日志目录挂载到宿主机的指定目录。
     </li>
     <li>
      创建独立的数据卷并指定文件：
      <code>
       docker volume create nginx-config
      </code>
      ：创建一个名为 nginx-config 的数据卷用于存储 Nginx 配置文件。
     </li>
     <li>
      将配置文件复制到数据卷：
      <code>
       docker cp /home/user/nginx-config/nginx.conf my_test_nginx:/etc/nginx/nginx.conf
      </code>
      ：将宿主机上的配置文件复制到数据卷中。
     </li>
     <li>
      挂载配置文件数据卷：
      <code>
       docker run -d --name my_test_nginx -v nginx-config:/etc/nginx nginx
      </code>
      ：运行容器时挂载配置文件数据卷。
     </li>
     <li>
      直接在容器中替换：进入容器后，使用
      <code>
       docker cp
      </code>
      命令将新的 index.html 文件复制到容器的
      <code>
       /usr/share/nginx/html
      </code>
      目录。
     </li>
     <li>
      使用数据卷挂载：在启动容器时挂载包含新 index.html 文件的目录到
      <code>
       /usr/share/nginx/html
      </code>
      。
     </li>
    </ul>
