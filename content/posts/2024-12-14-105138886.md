    <h4 style="margin-left:0cm;">
     <strong>
      第三题:
     </strong>
    </h4>
    <p style="text-indent:33px;">
     (1).为
     <strong>
      每种类型的模块内聚
     </strong>
     举一个例子。
    </p>
    <p style="margin-left:0cm;">
     (2).为
     <strong>
      每种类型的模块耦合
     </strong>
     举一个具体的例子。
    </p>
    <table border="1" cellspacing="0">
     <tbody>
      <tr>
       <td style="vertical-align:top;width:414.8pt;">
        <p style="margin-left:0cm;">
         <strong>
          解：
         </strong>
        </p>
        <p style="margin-left:0cm;">
         <strong>
          （1）内聚
          <a href="#_ftn1" name="_ftnref1" rel="nofollow">
           <strong>
            [1]
           </strong>
          </a>
          ,每种类型的模块内聚举例说明如下：
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          1
         </strong>
         <strong>
          、偶然内聚：
         </strong>
         如果一个模块完成一组任务，这些任务彼此间即使有关系，关系也是很松散的。这就叫做偶然内聚
        </p>
        <p style="text-indent:33px;">
         偶然内聚的例子：在模块T 中有A,B,C三条语句，至少从表面上看来这三条语句没什么联系，只是因为D,E,F,G中都有这三条语句，为了节省空间才把这三条语句作为一个模板放在一起。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          2
         </strong>
         <strong>
          、逻辑内聚：
         </strong>
         如果一个模块完成的任务在逻辑上属于相同或相似的一类(例如一个模块产生各种类型的全部输出)，称为逻辑内聚
        </p>
        <p style="text-indent:33px;">
         逻辑内聚的例子：某一个模块将打印年、月、日，具体打印什么，将由传入的控制标志所决定。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          3
         </strong>
         <strong>
          、时间内聚：
         </strong>
         一个模块包含的任务必须在同一段时间内执行（例如，模块完成各种初始化工作），称为时间内聚
        </p>
        <p style="text-indent:33px;">
         时间内聚的例子：将多个变量的初始化放在同一个模块中实现。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          4
         </strong>
         <strong>
          、过程内聚：
         </strong>
         如果一个模块内的处理元素是相关的。而且必须以特定次序执行。称为过程内聚。
        </p>
        <p style="text-indent:33px;">
         过程内聚的例子：一个子程序，将开始读取学生的学号，然后是姓名，最后将读取分数，是由于特定的顺序而将这些操作组合在一起的
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          5
         </strong>
         <strong>
          、通讯内聚：
         </strong>
         如果模块中所有的元素都使用同一个输入数据和（或）产生同一个输出数据，则称为通讯内聚。
        </p>
        <p style="text-indent:33px;">
         通讯内聚的例子：有一个子程序，它将打印实验报告，并且在完成后重新初始化传进来的实验数据。这个程序具有通讯内聚性。因为这两个操作由于使用同一个数据源联系在了一起。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          6
         </strong>
         <strong>
          、顺序内聚：
         </strong>
         如果一个模块内的处理元素和同一个功能密切相关，而且这些处理必须顺序执行（通常一个处理元素的输出数据作为下一个处理元素的输入数据），则称为顺序内聚。
        </p>
        <p style="text-indent:33px;">
         顺序内聚的例子：有一个子程序，通过给出的生日，先计算出年龄。再根据年龄算出退休的时间，则这个程序具有顺序内聚性。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          7
         </strong>
         <strong>
          、功能内聚：
         </strong>
         如果模块内所有的元素属于一个整体完成一个单一的功能，则成为功能内聚。
        </p>
        <p style="text-indent:33px;">
         功能内聚的例子：一个程序中所有的操作都是为了算出一个人的年龄
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="margin-left:0cm;">
         <strong>
          （2）耦合
          <a href="#_ftn2" name="_ftnref2" rel="nofollow">
           <strong>
            [2]
           </strong>
          </a>
          ,每种类型的模块耦合举例说明如下：
         </strong>
        </p>
        <p style="text-indent:33px;">
         一般模块之间的可能的连接方式有七种，构成耦合的七种类型，它们的关系如下图：
        </p>
        <p style="margin-left:0cm;">
         <img alt="" height="248" src="https://i-blog.csdnimg.cn/blog_migrate/4ce739cf44c6fe985575a04db1e3c530.png" width="1113"/>
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="text-indent:33px;">
         下面举例说明以上几种耦合：
        </p>
        <p style="text-indent:33px;">
         非直接耦合：两模块之间没直接关系（模块1和模块2），独立性最强
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         数据耦合：即一个模块访问另一个模块的时候，彼此之间是通过数据参数来交换输入、输出信息的，这种耦合为数据耦合。这种耦合较为松散，模块间独立性较强。
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         特征耦合：即一组模块通过参数传递记录信息，用户情况是个数据结构，图中模块都与此有关，“计算水费”和“计算电费”本没有关系，由于引用了此数据结构产生了依赖关系
         <br/>
        </p>
        <p style="text-indent:33px;">
         控制耦合：即如果一个模块通过传送开关、标志、名字等控制信息，明显地控制选择另一模块的功能，就是控制耦合
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         <strong>
          外部耦合：
          <span style="color:#333333;">
           一组模块都访问同一全局简单变量而不是同一全局数据结构，而且不是通过参数表传递该全局变量的信息，则称之为外部耦合。
          </span>
         </strong>
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         公共耦合:
         <span style="color:#333333;">
          若一组模块都访问同一个公共数据环境，则它们之间的耦合就称为公共耦合。
         </span>
        </p>
        <p style="text-indent:33px;">
        </p>
        <p style="text-indent:33px;">
         内容耦合：如果出现以下情况之一，两个模块就发生了内容耦合：
        </p>
        <ol>
         <li>
          一个模块访问另一个模块的内部数据。
         </li>
         <li>
          一个模块不通过正常入口儿转到另一个模块的内部
         </li>
         <li>
          两个模块有一部分程序代码重叠（只可能发生在汇编程序中）
         </li>
         <li>
          一个模块有多个入口（这意味着一个模块有几种功能）
         </li>
        </ol>
       </td>
      </tr>
     </tbody>
    </table>

    <p style="margin-left:0cm;">
     <strong>
      (2).在什么情况下应该使用形式化说明技术？使用形式化说明技术应遵守哪些规则？
     </strong>
    </p>

     <tbody>
      <tr>

        <p style="margin-left:0cm;">
         <strong>
          解：（1）形式化说明技术和非形式化方法的优缺点如下：
         </strong>
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="text-align:center;">
         <img alt="" height="271" src="https://i-blog.csdnimg.cn/blog_migrate/c5ced767163bf350c322693e87a1cb0f.png" width="721"/>
        </p>
        <p style="text-indent:33px;">
         <strong>
          以一
         </strong>
         <strong>
          个简单的打飞机游戏系统规格说明为例，用自然语言描述如下：
         </strong>
        </p>
        <p style="text-indent:33px;">
         游戏的每个状态对应一个游戏界面，开始状态下，变量cd_start=1是进入正常游戏的状态，cd_start=2时，进入读取游戏状态，cd_start=3是进入得分榜界面查看。在得分榜界面按任意键返回开始界面，在读取游戏界面，当游戏数据读取完成后进入正常游戏状态，正常游戏状态下，同时按下左键和右键进入储存游戏界面，数据储存结束后返回正常游戏状态，在正常游戏状态下，如果变量game_res=0，则游戏结束，进入游戏结束画面。
        </p>
        <p style="text-indent:33px;">
         <span style="color:#ff0000;">
          可见，用自然语言书写的系统规格说明书，啰嗦繁杂，并且可能存在矛盾，二义性，含糊性，不完整性及抽象层次混乱等问题。
         </span>
        </p>
        <p style="margin-left:0cm;">
        </p>
        <p style="text-indent:0;">
         <strong>
          （2）应用形式化方法的应该遵循的准则如下：
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          1，应该选用适当的表示方法；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          2，应该形式化，但不要过分形式化；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          3, 应该估算成本；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          4, 应该有形式化的方法顾问随时提供咨询；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          5, 不应该放弃传统的开发方式；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          6，应该建立详尽的文档；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          7，不应该放弃质量标准；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          8，不应该盲目依赖形式化方法；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          9，应该测试，测试再测试；
         </strong>
        </p>
        <p style="text-indent:33px;">
         <strong>
          10，应该重用。
