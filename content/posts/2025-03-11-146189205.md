     二、docker 存储
    </h1>

    <p>
     docker四种方式：默认、volumes数据卷、bind mounts挂载、tmpfs mount(仅在linux环境中提供)，其中volumes、bind mounts两种实现持久化容器数据；
    </p>
    <ul>
     <li>
      <p>
       默认：数据保存在运行的容器中，容器删除后，数据也随之删除；
      </p>
     </li>
     <li>
      <p>
       volumes：数据卷，数据存放在主机文件系统/var/lib/docker/volumes/目录下，该目录由docker管理，其它进程不允许修改，推荐该种方式持久化数据；
      </p>
     </li>
     <li>
      <p>
       Bind mounts：直接挂载主机文件系统的任何目录或文件，类似主机和容器的共享目录，主机上任何进程都可以访问修改，容器中也可以看到修改，这种方式最简单。
      </p>
     </li>
     <li>
      <p>
       tmpfs：数据暂存在主机内存中，不会写入文件系统，重启后，数据删除。
      </p>
     </li>
    </ul>
    <h3>
     <a id="_10">
     </a>
     （一）卷管理以及文件系统挂在语法
    </h3>
    <h4>
     <a id="1Docker__12">
     </a>
     1、Docker 卷管理语法
    </h4>
    <pre><code>docker  volume  create ：创建一个新卷
docker  volume  ls：列出本地docker主机上的卷
docker  volume  imspect：查看卷的详细信息，包括卷在Docker主机文件系统中的具体位置
docker  volume  prune：删除未被容器或者服务副本使用的全部卷
docker  volume  rm：删除未被使用的指定卷
</code></pre>
    <h4>
     <a id="2_22">
     </a>
     2、容器的文件系统挂在语法
    </h4>
    <h5>
     <a id="1v__24">
     </a>
     （1）-v 选项的语法
    </h5>
    <pre><code>-v  [主机中的挂在源]容器中的挂在目标[:&lt;选项&gt;]
</code></pre>
    <p>
     注：第三个选项是ro，默认是rw
    </p>
    <h5>
     <a id="2mount__32">
     </a>
     （2）–mount 选项的语法格式
    </h5>
    <pre><code>--mount  &lt;键&gt;=&lt;值&gt;,&lt;键&gt;=&lt;值&gt;

type：指定要挂在的类型，值可以是 bind(绑定挂载)、volume(卷)或 tmpfs。默认使用volume
source（或sc）：指定挂载源，对于卷，挂载源卷，挂载源为主机上的目录或者文件
destionation（或dist、target）：指定挂载目标，即容器中的挂在点，必须采用绝对路径的形式
readonly：指定只读选项，表示源以只读方式挂在到容器中
其他键：可以被多次指定，由若干键值对组成。卷和绑定挂载有不同的键
</code></pre>
    <h3>
     <a id="_44">
     </a>
     （二）案例
    </h3>
    <h4>
     <a id="1_46">
     </a>
     1、创建和管理卷
    </h4>
    <p>
     （1）创建卷
    </p>
    <pre><code># docker volume create volume-test01
</code></pre>
    <p>
     （2）查看卷的详细信息
    </p>
    <pre><code># docker volume inspect volume-test01
</code></pre>
    <p>
     （3）启动容器时挂载卷
    </p>
    <pre><code># docker run -it -d -v volume-test01:/txt centos /bin/bash
</code></pre>
    <p>
     （4）删除卷
    </p>
    <pre><code>docker volume  rm  volume-test01
</code></pre>
    <p>
     注：没有容器使用该卷以后，才可以删除卷。
    </p>
    <p>
     （5）启动容器时自动创建卷
    </p>
    <p>
     启动带有卷的容器时，如果卷不存在，则docker会自动创建这个卷，即在docker根目录的volumes子目录下生成相应的目录结构。
    </p>
    <pre><code># docker run -it -d --name  com-test01 --mount source=vol-test01,target=/app  nginx
</code></pre>
    <h4>
     <a id="2_82">
     </a>
     2、使用容器填充卷、使用只读卷和使用匿名卷
    </h4>
    <p>
     （1）使用容器填充卷
    </p>
    <p>
     如果容器启动时挂载已经存在并包含数据的卷，则容器不会将其挂载点目录的数据复制到该卷，而是直接使用卷。
    </p>
    <p>
     如果容器启动时挂载空白卷（卷已经存在但是没有任何数据）或者自动创建新卷，而容器的挂在点目录下已经有文件或者目录，则该挂载卷目录的内容会被复制到卷中，即将容器中挂在点目录的数据填充到卷中。
    </p>
    <p>
     （2）使用只读卷
    </p>
    <pre><code># docker rm -it -d --name=nginx --mount soruce=nginx-vol01,destination=/usr/share/nginx/html,readonly  nginx:latest
</code></pre>
    <p>
     （3）使用匿名卷
    </p>
    <pre><code># docker run -it -d --mount destination=/app  nginx
</code></pre>
    <p>
     注：匿名卷并不是没有名称的，而是docker自动为匿名卷生成一个UUID作为名称。
    </p>
    <h4>
     <a id="3_104">
     </a>
     3、使用容器进行绑定挂载
    </h4>
    <pre><code># docker run -it -d --name  con-test-01 --mount type=bind,source="$(pwd)"/nginx,target=/app  nginx
</code></pre>
    <h4>
     <a id="4_110">
     </a>
     4、创建、备份、恢复卷容器
    </h4>
    <pre><code>docker run --rm ：停止容器时自动删除容器
			--volumes-from ：基于卷容器创建新的容器，并挂载卷容器提供的卷
</code></pre>
    <h5>
     <a id="1_117">
     </a>
     （1）创建卷容器
    </h5>
    <pre><code># docker create --name data-store01 -v /data01 ubuntu /bin/bash
</code></pre>
    <h5>
     <a id="2_123">
     </a>
     （2）备份卷容器
    </h5>
    <p>
     ① 启动新的容器并从data-store01容器中挂载卷
    </p>
    <p>
     ② 将本地主机的当前目录挂载为/backup
    </p>
    <p>
     ③ 传送命令，将data01卷的内容打包为/backup/backup.tar文件
    </p>
    <pre><code># docker run --rm --volumes-from data-store01 -v $(pwd):/backup  ubuntu tar -cvf /backup/backup.tar  /data01
</code></pre>
    <h5>
     <a id="3_135">
     </a>
     （3）从备份中恢复卷容器
    </h5>
    <pre><code># docker run -v /data01 --name data-store02 ubuntu /bin/bash
# docker run --rm --volumes-from data-store02 -v $(pwd):/backup ubuntu bash -c "cd /data01 &amp;&amp; tar -xvf /backup/backup.tar  --strip 1"
</code></pre>
