     Java集合
    </h2>
    <p>
     在Java开发中，集合框架是面试和实际开发中非常重要的内容。以下是一些常见的Java集合八股文问题和场景题，以及详细答案和示例代码。

    <h4>
     <a id="1_Java_5">
     </a>
     <strong>
      1. Java集合框架的结构是什么？
     </strong>
    </h4>
    <p>
     <strong>
      答案：
     </strong>
     <br/>
     Java集合框架主要分为三大接口：
     <code>
      Collection
     </code>
     、
     <code>
      Map
     </code>
     和
     <code>
      Queue
     </code>
     。
    </p>
    <ul>
     <li>
      <strong>
       Collection
      </strong>
      ：是集合的顶级接口，它又分为
      <code>
       List
      </code>
      和
      <code>
       Set
      </code>
      。
      <ul>
       <li>
        <code>
         List
        </code>
        ：有序集合，允许重复元素，如
        <code>
         ArrayList
        </code>
        、
        <code>
         LinkedList
        </code>
        和
        <code>
         Vector
        </code>
        。
       </li>
       <li>
        <code>
         Set
        </code>
        ：无序集合，不允许重复元素，如
        <code>
         HashSet
        </code>
        、
        <code>
         LinkedHashSet
        </code>
        和
        <code>
         TreeSet
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Map
      </strong>
      ：键值对集合，不允许键重复，但值可以重复，如
      <code>
       HashMap
      </code>
      、
      <code>
       TreeMap
      </code>
      和
      <code>
       LinkedHashMap
      </code>
      。
     </li>
     <li>
      <strong>
       Queue
      </strong>
      ：队列接口，用于实现先进先出（FIFO）的集合，如
      <code>
       LinkedList
      </code>
      和
      <code>
       PriorityQueue
      </code>
      。
     </li>
    </ul>

    <h4>
     <a id="2_ArrayListLinkedList_49">
     </a>
     <strong>
      2. ArrayList和LinkedList的区别是什么？
     </strong>
    </h4>
    <p>
     <strong>
      答案：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       存储结构
      </strong>
      ：
      <ul>
       <li>
        <code>
         ArrayList
        </code>
        基于动态数组实现，支持快速随机访问，但插入和删除效率较低。
       </li>
       <li>
        <code>
         LinkedList
        </code>
        基于双向链表实现，插入和删除效率高，但不支持快速随机访问。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       性能差异
      </strong>
      ：
      <ul>
       <li>
        <code>
         ArrayList
        </code>
        适合频繁读取，
        <code>
         LinkedList
        </code>
        适合频繁插入和删除。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       内存占用
      </strong>
      ：
      <ul>
       <li>
        <code>
         LinkedList
        </code>
        每个节点需要额外存储前后指针，内存占用较大。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="3_HashMap_83">
     </a>
     <strong>
      3. HashMap的底层实现原理是什么？
     </strong>
    </h4>
    <p>
     <strong>
      答案：
     </strong>
     <br/>
     <code>
      HashMap
     </code>
     基于哈希表实现，主要由两部分组成：数组和链表（或红黑树）。
    </p>
    <ul>
     <li>
      <strong>
       哈希冲突
      </strong>
      ：当两个键的哈希值相同或哈希值对数组取模后相同，会发生冲突。
     </li>
     <li>
      <strong>
       解决冲突
      </strong>
      ：
      <ul>
       <li>
        JDK 1.7及之前：链表法，将冲突的元素存储在链表中。
       </li>
       <li>
        JDK 1.8及之后：链表+红黑树，当链表长度超过8时，链表会转换为红黑树，以优化查找效率。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       扩容机制
      </strong>
      ：当负载因子（
      <code>
       size / capacity
      </code>
      ）达到阈值（默认0.75）时，数组会扩容为原来的两倍。
     </li>
    </ul>

    <h4>
     <a id="4_HashMapArrayListVector_114">
     </a>
     <strong>
      4. 如何保证线程安全？HashMap、ArrayList和Vector的区别是什么？
     </strong>
    </h4>
    <p>
     <strong>
      答案：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       线程安全
      </strong>
      ：
      <ul>
       <li>
        <code>
         HashMap
        </code>
        和
        <code>
         ArrayList
        </code>
        不是线程安全的，需要手动同步。
       </li>
       <li>
        <code>
         Vector
        </code>
        是线程安全的，但性能较差，因为它的每个方法都加了
        <code>
         synchronized
        </code>
        锁。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       区别
      </strong>
      ：
      <ul>
       <li>
        <code>
         HashMap
        </code>
        和
        <code>
         ArrayList
        </code>
        ：非线程安全，性能高。
       </li>
       <li>
        <code>
         Vector
        </code>
        ：线程安全，性能低。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       线程安全的替代品
      </strong>
      ：
      <ul>
       <li>
        使用
        <code>
         Collections.synchronizedMap()
        </code>
        或
        <code>
         ConcurrentHashMap
        </code>
        替代
        <code>
         HashMap
        </code>
        。
       </li>
       <li>
        使用
        <code>
         CopyOnWriteArrayList
        </code>
        替代
        <code>
         ArrayList
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      示例代码：
     </strong>
    </p>
