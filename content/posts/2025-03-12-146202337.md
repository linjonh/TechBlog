    <h4 id="问题描述">
     问题描述
    </h4>
    <p>
     小蓝最近在研究一种浮点数的表示方法：RR 格式。对于一个大于 0 的浮点数 dd，可以用 RR 格式的整数来表示。给定一个转换参数 nn，将浮点数转换为 RR 格式整数的做法是:
    </p>
    <ol>
     <li>
      <p>
       将浮点数乘以 2n2n；
      </p>
     </li>
     <li>
      <p>
       四舍五入到最接近的整数。
      </p>
     </li>
    </ol>
    <h4 id="输入格式">
     输入格式
    </h4>
    <p>
     一行输入一个整数 nn 和一个浮点数 dd，分别表示转换参数，和待转换的浮点数。
    </p>
    <h4 id="输出格式">
     输出格式
    </h4>
    <p>
     输出一行表示答案：dd 用 RR 格式表示出来的值。
    </p>
    <h4 id="样例输入">
     样例输入
    </h4>
    <pre><code>2 3.14
</code></pre>
    <p>
    </p>
    <h4 id="样例输出">
     样例输出
    </h4>
    <pre><code>13
</code></pre>
    <p>
    </p>
    <h4 id="样例说明">
     样例说明
    </h4>
    <p>
     3.14×22=12.563.14×22=12.56，四舍五入后为 1313。
    </p>
    <h4 id="评测用例规模与约定">
     评测用例规模与约定
    </h4>
    <p>
     对于 50%50% 的评测用例：1≤n≤10,1≤1≤n≤10,1≤ 将 dd 视为字符串时的长度 ≤15≤15。
    </p>
    <p>
     对于 100%100% 的评测用例：1≤n≤1000,1≤1≤n≤1000,1≤ 将 dd 视为字符串时的长度 ≤1024≤1024；保证 dd 是小数，即包含小数点。
    </p>
    <h3>
     代码展示
    </h3>
    <pre><code class="language-cpp">#include &lt;iostream&gt;
#include&lt;cmath&gt;
using namespace std;
int main()
{
  ios::sync_with_stdio(0);cin.tie(0);
  int n;
  float d;
  cin&gt;&gt;n&gt;&gt;d;
  while(n&gt;0){
    d=d*2;
    n--;

  }
  int a=lround(d);
  cout&lt;&lt;a;
  // 请在此输入您的代码
  return 0;
}</code></pre>
    <h4>
     <strong>
      代码功能总结
     </strong>
    </h4>
    <ol>
     <li>
      输入整数
      <code>
       n
      </code>
      和浮点数
      <code>
       d
      </code>
      。
     </li>
     <li>
      对
      <code>
       d
      </code>
      连续乘以 2 的
      <code>
       n
      </code>
      次方（通过循环
      <code>
       d = d * 2
      </code>
      执行
      <code>
       n
      </code>
      次）。
     </li>
     <li>
      将最终结果
      <code>
       d
      </code>
      四舍五入到最近的整数，输出该整数。
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      <code>
       lround
      </code>
      函数详解
     </strong>
    </h4>
    <h5>
     函数原型：
    </h5>
    <p>
     <code id="code_copy_1741762950160">
      long int lround(double x); // 处理 double 类型
     </code>
    </p>
    <p>
     <code>
      long int lroundf(float x); // 处理 float 类型（代码中实际调用）
     </code>
    </p>
    <p>
     <code>
      long int lroundl(long double x); // 处理 long double 类型
     </code>
    </p>
    <h5>
     参数：
    </h5>
    <ul>
     <li>
      <code>
       x
      </code>
      ：需要四舍五入的浮点数（代码中
      <code>
       d
      </code>
      是
      <code>
       float
      </code>
      类型，调用
      <code>
       lroundf
      </code>
      ）。
     </li>
    </ul>
    <h5>
     返回值：
    </h5>
    <ul>
     <li>
      返回四舍五入后的
      <code>
       long int
      </code>
      类型整数。
     </li>
     <li>
      如果
      <code>
       x
      </code>
      超出
      <code>
       long int
      </code>
      的表示范围，返回值未定义（可能返回
      <code>
       LONG_MAX
      </code>
      或
      <code>
       LONG_MIN
      </code>
      ）。
     </li>
    </ul>
    <h5>
     <strong>
      关键行为
     </strong>
     ：
    </h5>
    <ul>
     <li>
      对
      <code>
       x
      </code>
      进行
      <strong>
       四舍五入
      </strong>
      ，但
      <strong>
       中间值（如 0.5）会向远离零的方向取整
      </strong>
      。
      <br/>
      例如：
      <ul>
       <li>
        <code>
         lround(2.3) → 2
        </code>
       </li>
       <li>
        <code>
         lround(2.5) → 3
        </code>
        （注意：与
        <code>
         std::round
        </code>
        的行为不同！）
       </li>
       <li>
        <code>
         lround(-1.7) → -2
        </code>
       </li>
      </ul>
     </li>
    </ul>

    <hr/>
    <h4>
     <strong>
      代码中为何使用
      <code>
       lround
      </code>
      ？
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        直接返回整数
       </strong>
       ：
       <br/>
       代码需要将最终结果以整数形式输出（
       <code>
        cout &lt;&lt; a
       </code>
       ），使用
       <code>
        lround
       </code>
       避免了额外的类型转换（例如
       <code>
        round
       </code>
       返回
       <code>
        double
       </code>
       ，还需转换为
       <code>
        int
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        明确的四舍五入逻辑
       </strong>
       ：
       <br/>
       如果
       <code>
        d * 2^n
       </code>
       的小数部分为
       <code>
        0.5
       </code>
       ，
       <code>
        lround
       </code>
       会向更大的整数方向取整（例如
       <code>
        2.5 → 3
       </code>
       ），符合常见的数学四舍五入预期。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <strong>
      潜在问题
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        溢出风险
       </strong>
       ：
       <br/>
       如果
       <code>
        d * 2^n
       </code>
       结果过大（超过
       <code>
        long int
       </code>
       的范围），
       <code>
        lround
       </code>
       的返回值会溢出，导致未定义行为。
       <br/>
       <strong>
        改进建议
       </strong>
       ：检查范围或使用
       <code>
        try-catch
       </code>
       （需启用异常处理）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        负数的中间值处理
       </strong>
       ：
       <br/>
       对于负数中间值（如
       <code>
        -2.5
       </code>
       ），
       <code>
        lround
       </code>
       会向更小的方向取整（即
       <code>
        -3
       </code>
       ），需要注意是否符合业务逻辑。
      </p>
     </li>
    </ol>

    <h4>
     <strong>
      总结
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        lround
       </code>
       的作用
      </strong>
      ：将浮点数四舍五入到最接近的整数，直接返回
      <code>
       long int
      </code>
      类型结果。
     </li>
     <li>
      <strong>
       代码逻辑
      </strong>
      ：通过循环计算
      <code>
       d * 2^n
      </code>
      ，并用
      <code>
       lround
      </code>
      确保结果为整数。
     </li>
     <li>
      <strong>
       注意事项
      </strong>
      ：关注溢出和负数中间值的处理。
     </li>
    </ul>
