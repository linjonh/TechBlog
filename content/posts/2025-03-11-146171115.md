     1.
     <strong>
      在桌面和 iOS 设备上使用 Google Lens 进行搜索
     </strong>
    </h4>
    <p>
     管理员可以通过名为
     <strong>
      LensOverlaySettings
     </strong>
     的策略控制该功能的所有元素。
    </p>
    <p>
     执行搜索时，屏幕截图会被发送到 Google 服务器，但不会与任何 ID 或账户关联，也不会被任何人查看，且其内容数据不会被记录。
    </p>
    <p>
     为了使搜索能够结合用户当前浏览的文档或网站的上下文，PDF 文件的字节数据或网站 HTML 也会被发送到 Google 服务器，但同样不会与任何 ID 或账户关联，不会被任何人查看，且其内容数据或生成的数据不会被记录。
    </p>

    <h4>
     <a id="2_macOS__PWA_Web_Shim__19">
     </a>
     2.
     <strong>
      macOS 上 PWA（渐进式 Web 应用）Shim 的临时代码签名调整
     </strong>
    </h4>
    <p>
     在 macOS 上安装
     <strong>
      渐进式 Web 应用（PWA）
     </strong>
     时，系统会创建一个
     <strong>
      应用程序 Shim（代理程序）
     </strong>
     ，其
     <strong>
      代码签名方式
     </strong>
     将发生变化。
    </p>
    <p>
     新的方式将使用
     <strong>
      安装时生成的临时（Ad-hoc）代码签名
     </strong>
     ，而不再使用统一的代码签名。
    </p>
    <p>
     在 macOS 生态系统中，代码签名用于
     <strong>
      标识应用程序
     </strong>
     。这次更新后，每个
     <strong>
      PWA Shim
     </strong>
     都会有一个
     <strong>
      唯一的身份
     </strong>
     ，而此前所有的 PWA 在 macOS 上看起来是同一个应用程序。
    </p>
    <h5>
     <a id="21__26">
     </a>
     2.1. 更新的主要影响
    </h5>
    <ul>
     <li>
      <strong>
       解决 macOS “开机启动项” 中多个 PWA 无法同时加入的问题
      </strong>
      。
     </li>
     <li>
      <strong>
       改进 macOS 对 PWA 内用户通知的处理
      </strong>
      ，为未来的功能升级铺平道路。
     </li>
    </ul>

    <h4>
     <a id="3_Chrome_Sync__Chrome_37">
     </a>
     3.
     <strong>
      Chrome Sync 将停止支持超过四年的旧版 Chrome
     </strong>
    </h4>
    <p>
     从
     <strong>
      2025 年 2 月
     </strong>
     开始，
     <strong>
      Chrome Sync（在 Google 账户中使用和保存数据的功能）
     </strong>
     将
     <strong>
      不再支持超过四年的旧版 Chrome
     </strong>
     。
    </p>
    <p>
     如果要继续使用
     <strong>
      Chrome Sync
     </strong>
     ，用户需要
     <strong>
      升级至更新版本的 Chrome
     </strong>
     。
    </p>
    <h5>
     <a id="31__42">
     </a>
     3.1. 受影响范围
    </h5>
    <ul>
     <li>
      <strong>
       Chrome 133
      </strong>
      及更高版本：不会受到影响
     </li>
     <li>
      <strong>
       旧版 Chrome（超过四年未更新）
      </strong>
      ：
      <ul>
       <li>
        <strong>
         Chrome Sync 功能将不可用
        </strong>
       </li>
       <li>
        需要
        <strong>
         升级 Chrome
        </strong>
        以恢复同步功能
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="4_HttpsOnlyMode__54">
     </a>
     4.
     <strong>
      HttpsOnlyMode 策略新增选项
     </strong>
    </h4>
    <p>
     <strong>
      Ask Before HTTP (ABH)
     </strong>
     （先前称为
     <strong>
      HTTPS Only/First Modes
     </strong>
     ）允许 Chrome 在发送不安全的
     <strong>
      HTTP 内容
     </strong>
     之前请求用户同意。
    </p>
    <p>
     <strong>
      HttpsOnlyMode 策略
     </strong>
     允许
     <strong>
      强制启用或禁用
     </strong>
     ABH 机制。
    </p>
    <h5>
     <a id="41__Balanced_Mode_59">
     </a>
     4.1. 新增平衡模式 (Balanced Mode)
    </h5>
    <ul>
     <li>
      在
      <strong>
       Chrome 129
      </strong>
      ，ABH 机制新增了
      <strong>
       “平衡模式” (Balanced Mode)
      </strong>
      ，该模式旨在减少用户的不便：
      <ul>
       <li>
        大多数情况下，它的行为与
        <strong>
         严格 ABH
        </strong>
        一致。
       </li>
       <li>
        但当 Chrome
        <strong>
         已知无法建立 HTTPS 连接
        </strong>
        （例如访问
        <strong>
         单标签主机名
        </strong>
        ，如 internal/）时，将不再询问用户。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       新增策略：force_balanced_enabled
      </strong>
      <ul>
       <li>
        <strong>
         用于强制启用
        </strong>
        “平衡模式”。
       </li>
       <li>
        在
        <strong>
         Chrome 129 之前的版本
        </strong>
        设置该选项
        <strong>
         不会生效
        </strong>
        ，将默认不对 ABH 进行企业限制。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="5__74">
     </a>
     5.
     <strong>
      节能模式下的标签页冻结机制
     </strong>
    </h4>
    <p>
     当
     <strong>
      节能模式（Energy Saver）
     </strong>
     激活时，Chrome 会对
     <strong>
      隐藏且静音超过 5 分钟
     </strong>
     ，并且
     <strong>
      占用大量 CPU
     </strong>
     的标签页进行
     <strong>
      冻结
     </strong>
     ，以减少 CPU 负载，延长电池寿命，提高浏览器性能。
    </p>
    <h5>
     <a id="51__77">
     </a>
     5.1. 例外情况
    </h5>
    <p>
     以下情况的标签页
     <strong>
      不会被冻结
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       提供音视频会议功能
      </strong>
      （通过以下方式检测到）：
      <ul>
       <li>
        使用
        <strong>
         麦克风
        </strong>
        或
        <strong>
         摄像头
        </strong>
       </li>
       <li>
        进行
        <strong>
         屏幕、窗口或标签页捕获
        </strong>
       </li>
       <li>
        使用
        <strong>
         RTCPeerConnection
        </strong>
        并具有
        <strong>
         活动的 RTCDataChannel 或 MediaStreamTrack
        </strong>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       控制外部设备
      </strong>
      （通过以下 API 进行检测）：
      <ul>
       <li>
        <strong>
         Web USB
        </strong>
       </li>
       <li>
        <strong>
         Web Bluetooth
        </strong>
       </li>
       <li>
        <strong>
         Web HID
        </strong>
       </li>
       <li>
        <strong>
         Web Serial
        </strong>
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="6_Android__Chrome__V8__96">
     </a>
     6.
     <strong>
      Android 版 Chrome 的 V8 安全设置
     </strong>
    </h4>
    <p>
     <strong>
      V8
     </strong>
     是
     <strong>
      Chrome 的 JavaScript 和 WebAssembly 引擎
     </strong>
     ，用于提升网页性能。
    </p>
    <p>
     为了
     <strong>
      减少 Chrome 的攻击面
     </strong>
     ，
     <strong>
      Chrome 133（Android 版）
     </strong>
     在
     <strong>
      chrome://settings/security
     </strong>
     中新增了一项安全设置，允许
     <strong>
      禁用 V8 的 Just-in-Time（JIT）优化器
     </strong>
     ，但仍
     <strong>
      保持对 WebAssembly 的兼容性
     </strong>
     。
    </p>
    <h5>
     <a id="61__101">
     </a>
     6.1. 管理员控制选项
    </h5>
    <p>
     可以通过以下策略
     <strong>
      管理 JIT 相关设置
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       DefaultJavaScriptJitSetting
      </strong>
      —— 控制 JIT 默认启用或禁用
     </li>
     <li>
      <strong>
       JavaScriptJitAllowedForSites
      </strong>
      —— 允许特定网站使用 JIT
     </li>
     <li>
      <strong>
       JavaScriptJitBlockedForSites
      </strong>
      —— 阻止特定网站使用 JIT
     </li>
    </ul>
