    <blockquote>
     <p>
      本文应网友
      <code>
       畅天
      </code>
      的要求折腾。他遇到了跨域问题，所以老苏找了个二合一的镜像来规避。其中使用的
      <code>
       Baikal
      </code>
      和
      <code>
       InfCloud
      </code>
      都是最新的版本；
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8a5e740c43e0e26a1c02624624ceec4b.png#pic_center"/>
    </p>
    <p>
     <strong>
      什么是 Baikal ？
     </strong>
    </p>
    <blockquote>
     <p>
      <code>
       Baikal
      </code>
      是一个免费的开源自托管
      <code>
       CalDAV
      </code>
      和
      <code>
       CardDAV
      </code>
      服务器，适用于想要管理其数据并确保其数据是私有的用户。
     </p>
    </blockquote>
    <p>
     <strong>
      什么是 InfCloud ?
     </strong>
    </p>
    <blockquote>
     <p>
      <code>
       InfCloud
      </code>
      是一个开源的
      <code>
       CalDAV/CardDAV
      </code>
      网页客户端。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/6884d2566e548e6667581ff2be250824.png#pic_center"/>
    </p>
    <h2>
     <a id="_19">
     </a>
     安装
    </h2>
    <p>
     在群晖上以 Docker 方式安装。
    </p>
    <p>
     在注册表中搜索
     <code>
      baikal-infcloud
     </code>
     ，选择第一个
     <code>
      squibbles/baikal-infcloud
     </code>
     ，版本选择
     <code>
      v1.1
     </code>
     。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/7726fe073b005dd123a21d6fb51e3fca.png#pic_center"/>
    </p>
    <h3>
     <a id="_28">
     </a>
     卷
    </h3>
    <p>
     在
     <code>
      docker
     </code>
     文件夹中，创建一个新文件夹
     <code>
      baikal-infcloud
     </code>
     ，并在其中建一个子文件夹
     <code>
      data
     </code>
    </p>

    <h2>
     <a id="_64">
     </a>
     命令行安装
    </h2>
    <p>
     如果你熟悉命令行，可能用
     <code>
      docker cli
     </code>
     更快捷
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 新建文件夹 baikal-infcloud 和 子目录</span>
<span class="token function">mkdir</span> <span class="token parameter variable">-p</span> /volume1/docker/baikal-infcloud/data

<span class="token comment"># 进入 baikal-infcloud 目录</span>
<span class="token builtin class-name">cd</span> /volume1/docker/baikal-infcloud

<span class="token comment"># 运行容器</span>
<span class="token function">docker</span> run <span class="token parameter variable">-d</span> <span class="token punctuation">\</span>
<span class="token parameter variable">--restart</span> unless-stopped <span class="token punctuation">\</span>
<span class="token parameter variable">--name</span> baikal-infcloud <span class="token punctuation">\</span>
<span class="token parameter variable">-p</span> <span class="token number">8801</span>:8800 <span class="token punctuation">\</span>
<span class="token parameter variable">-v</span> <span class="token variable"><span class="token variable">$(</span><span class="token builtin class-name">pwd</span><span class="token variable">)</span></span>/data:/var/www/baikal/Specific/ <span class="token punctuation">\</span>
<span class="token parameter variable">-e</span> <span class="token assign-left variable">TIMEZONE</span><span class="token operator">=</span>Asia/Shanghai <span class="token punctuation">\</span>
squibbles/baikal-infcloud:v1.1
</code></pre>
<p>
也可以用
<code>
docker-compose
</code>
安装，将下面的内容保存为
<code>
docker-compose.yml
</code>
文件
</p>
<pre><code class="prism language-yaml"><span class="token key atrule">version</span><span class="token punctuation">:</span> <span class="token string">"3.8"</span>

<span class="token key atrule">services</span><span class="token punctuation">:</span>
<span class="token key atrule">baikal-infcloud</span><span class="token punctuation">:</span>
<span class="token key atrule">image</span><span class="token punctuation">:</span> squibbles/baikal<span class="token punctuation">-</span>infcloud<span class="token punctuation">:</span>v1.1
<span class="token key atrule">container_name</span><span class="token punctuation">:</span> baikal<span class="token punctuation">-</span>infcloud
<span class="token key atrule">restart</span><span class="token punctuation">:</span> unless<span class="token punctuation">-</span>stopped
<span class="token key atrule">ports</span><span class="token punctuation">:</span>
<span class="token punctuation">-</span> 8801<span class="token punctuation">:</span><span class="token number">8800</span>
<span class="token key atrule">volumes</span><span class="token punctuation">:</span>
<span class="token punctuation">-</span> ./data<span class="token punctuation">:</span>/var/www/baikal/Specific/
<span class="token key atrule">environment</span><span class="token punctuation">:</span>
<span class="token punctuation">-</span> TIMEZONE=Asia/Shanghai
</code></pre>
<p>
然后执行下面的命令
</p>
<pre><code class="prism language-bash"><span class="token comment"># 新建文件夹 baikal-infcloud 和 子目录</span>
<span class="token function">mkdir</span> <span class="token parameter variable">-p</span> /volume1/docker/baikal-infcloud/data

<span class="token comment"># 进入 baikal-infcloud 目录</span>
<span class="token builtin class-name">cd</span> /volume1/docker/baikal-infcloud

<span class="token comment"># 将 docker-compose.yml 放入当前目录</span>

<span class="token comment"># 一键启动</span>
<span class="token function">docker-compose</span> up <span class="token parameter variable">-d</span>
</code></pre>

<h3>
<a id="InfCloud_146">
</a>
InfCloud
</h3>
<p>
之前老苏写过
<code>
AgenDAV
</code>
，跟
<code>
InfCloud
</code>
是差不多的应用，区别在于
<code>
AgenDAV
</code>
只支持
<code>
CalDAV
</code>
，而
<code>
InfCloud
</code>
同时支持
<code>
CalDAV/CardDAV
</code>
</p>
<blockquote>
<p>
文章传送门：
<a href="https://laosu.cf/2023/01/06/CalDAV%E7%BD%91%E9%A1%B5%E5%AE%A2%E6%88%B7%E7%AB%AFAgenDAV" rel="nofollow">
CalDAV 网页客户端 AgenDAV
</a>
</p>
</blockquote>
<p>
在浏览器中输入
<code>
http://群晖 IP:8801/infcloud/
</code>
就能看到登录界面
</p>
<blockquote>
<p>
语言默认为
<code>
English
</code>
，可以下拉选择
<code>
中文
</code>
</p>
</blockquote>
