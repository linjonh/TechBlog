      5.1 网络概述

    <h4 style="text-align:justify">
     <strong>
      5.1.1 计算机网络的概念
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. 计算机网络的发展
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     计算机网络是计算机技术与通信技术相结合的产物，它实现了远程通信、远程信息处理和资源共享。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 计算机网络的功能
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     数据通信、资源共享、管理集中化、实现分布式处理、负载均衡。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       3. 网络有关指标
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·网络
     <strong>
      <strong>
       性能指标
      </strong>
     </strong>
     ：速率、带宽(频带宽度或传送线路速率)、吞吐量、时延、往返时间、利用率。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·网络
     <strong>
      <strong>
       非性能指标
      </strong>
     </strong>
     ：费用、质量、标准化、可靠性、可扩展性、可升级性、易管理性和可维护性
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     按
     <strong>
      <strong>
       通信距离
      </strong>
     </strong>
     可将计算机网络分为
     <strong>
      <strong>
       局域网
      </strong>
     </strong>
     、城域网和广域网。
    </p>

    <h4 style="text-align:justify">
     <strong>
      5.1.3 网络的拓扑结构
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       网络拓扑结构
      </strong>
     </strong>
     是指网络中通信线路和结点的几何排序，用于
     <strong>
      <strong>
       表示整个网络的结构外貌
      </strong>
     </strong>
     ，反映各结点之间的结构关系。常用的网络拓扑结构有：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·总线型(利用率低、干扰大、价格低)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·星型(交换机形成的局域网、中央单元负荷大)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·环型(流动方向固定、效率低扩充难)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·树型(总线型的扩充、分级结构)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·分布式(任意结点连接、管理难成本高)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="438" src="https://i-blog.csdnimg.cn/direct/d1ef093a4cdf49ebaf9a8f642b91f208.png" width="698"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      5.1.4 通信技术
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     计算机网络是
     <strong>
      <strong>
       利用通信技术将数据从一个结点传送到另一结点的过程
      </strong>
     </strong>
     。
     <strong>
      <strong>
       通信技术
      </strong>
     </strong>
     是计算机网络的基础。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. 信道
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     信道可分为
     <strong>
      <strong>
       物理信道和逻辑信道
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     物理信道由传输介质和设备组成，根据传输介质的不同，分为无线信道和有线信道。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     逻辑信道是指在数据发送端和接收端之间存在的一条虚拟线路，可以是有连接的或无连接的。逻辑信道以物理信道为载体。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 发信机
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       发信机进行的信号处理
      </strong>
     </strong>
     包括信源编码、信道编码、交织、脉冲成形和调制。相反地，收信机进行的信号处理包括解调、采样判决、去交织、信道译码和信源译码。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       3. 复用技术和多址技术
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     如果
     <strong>
      <strong>
       同时传递多路数据就需要复用技术和多址技术
      </strong>
     </strong>
     。复用技术是指在一条信道上同时传输多路数据的技术，如TDM时分复用、FDM 频分复用和CDM 码分复用等。多址技术是指在一条线上同时传输多个用户数据的技术，在接收端把多个用户的数据分离(TDMA时分多址、FDMA 频分多址和CDMA 码分多址)。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       4. 5G通信网络
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     作为新一代的移动通信技术，
     <strong>
      <strong>
       5G特征
      </strong>
     </strong>
     体现在以下方面：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     1)基于OFDM（正交频分复用技术）优化的波形和多址接入；2)实现可扩展的OFDM间隔参数配置；3)OFDM加窗提高多路传输效率；4)灵活框架设计；5)大规模MIMO：最多256根天线；6)毫米波：频率大于24GHz以上的频段；7)频谱共享；8)先进的信道编码设计。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     5G网络的主要特征：
     <strong>
      <strong>
       服务化架构、网络切片
      </strong>
     </strong>
     。
    </p>
    <h4 style="text-align:justify">
     <strong>
      5.1.5 ISO/OSI网络体系结构
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. ISO/OSI参考模型
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ISO/OSI的参考模型一共有7层，
     <strong>
      <strong>
       由低层到高层分别为：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）
     <strong>
      <strong>
       物理层
      </strong>
     </strong>
     ：为数据链路层提供一个物理连接以及它们的机械、电气、功能和过程特性。传输的数据单位是
     <strong>
      <strong>
       比特
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）
     <strong>
      <strong>
       数据链路层
      </strong>
     </strong>
     ：负责在两个相邻结点间的线路上
     <strong>
      <strong>
       无差错
      </strong>
     </strong>
     地传输帧数据，并进行
     <strong>
      <strong>
       流量控制
      </strong>
     </strong>
     。传输的数据单位是
     <strong>
      <strong>
       帧
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）
     <strong>
      <strong>
       网络层
      </strong>
     </strong>
     ：将数据链路层的帧数据组成数据包，数据包中有
     <strong>
      <strong>
       源网络地址
      </strong>
     </strong>
     和
     <strong>
      <strong>
       目的网络地址
      </strong>
     </strong>
     ，
     <strong>
      <strong>
       主要提供路由选择功能，除此之外还有差错检测和恢复、流量/拥塞控制、激活和终止网络连接等功能
      </strong>
     </strong>
     。传输的数据单位是
     <strong>
      <strong>
       数据包
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）
     <strong>
      <strong>
       传输层
      </strong>
     </strong>
     ：为会话层提供透明、可靠的端到端的
     <strong>
      <strong>
       数据传输服务
      </strong>
     </strong>
     。其功能有数据分段、数据传输、数据组装、
     <strong>
      <strong>
       差错控制、流量/拥塞控制
      </strong>
     </strong>
     等。传输的数据单位是
     <strong>
      <strong>
       报文
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （5）
     <strong>
      <strong>
       会话层
      </strong>
     </strong>
     ：为表示层提供建立、维护和结束
     <strong>
      <strong>
       会话连接
      </strong>
     </strong>
     的功能，提供
     <strong>
      <strong>
       会话管理功能
      </strong>
     </strong>
     。传输的数据单位是报文。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （6）
     <strong>
      <strong>
       表示层
      </strong>
     </strong>
     ：提供格式化的表示和
     <strong>
      <strong>
       转换数据服务
      </strong>
     </strong>
     ，如
     <strong>
      <strong>
       数据的压缩、解压缩、加密和解密
      </strong>
     </strong>
     等。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （7）
     <strong>
      <strong>
       应用层
      </strong>
     </strong>
     ：
     <strong>
      <strong>
       面向用户服务
      </strong>
     </strong>
     ，提供网络与用户应用软件之间的接口服务，如
     <strong>
      <strong>
       文件传输、电子邮件、远程登录
      </strong>
     </strong>
     等。
    </p>

     <strong>
      5.2 网络互连硬件
     </strong>
    </h3>
    <h4 style="text-align:justify">
     <strong>
      5.2.1 网络的设备
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     构建一个实际的网络需要网络的传输介质、网络互连设备作为支持。

     <strong>
      5.3 网络的协议与标准
     </strong>
    </h3>
    <h4 style="text-align:justify">
     <strong>
      5.3.1 TCP/IP协议族
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     TCP/IP是Internet的核心协议，被广泛应用于局域网和广域网中，已成为事实上的国际标准。
    </p>
    <p class="img-center">
     <img alt="" height="344" src="https://i-blog.csdnimg.cn/direct/cbc72e3877b04665b9841c78ee2ea7f7.png" width="647"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       应用层
      </strong>
     </strong>
     ：在模型的最高层，用户调用应用程序来访问TCP/IP互连网络，以享受网络上提供的各种服务。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       传输层
      </strong>
     </strong>
     ：提供应用程序之间的通信服务。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       网际层
      </strong>
     </strong>
     ：又称IP层，它主要有三个功能：①把数组分组封装到IP数据报中，填入数据报的首部，使用路由算法选择路由，送达机器；②处理接收到的数据报，检验其正确性；③发出ICMP差错和控制报文，并处理收到的ICMP报文。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       网络接口层
      </strong>
     </strong>
     ：处于TCP/IP的最下层，主要负责为物理网络准备数据所需的全部服务程序和功能。
    </p>
    <h4 style="text-align:justify">

     <strong>
      <strong>

      </strong>
     </strong>

    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ping：测试目标IP是否可达。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 传输层协议
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       ①传输控制协议TCP
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       TCP为应用程序提供了一个可靠的、面向连接的、全双工的数据传输服务
      </strong>
     </strong>
     。使用TCP在源主机和目的主机之间建立和关闭连接时，均需要通过
     <strong>
      <strong>
       三次握手
      </strong>
     </strong>
     来确认建立和关闭是否成功，如下图所示。
    </p>
    <p class="img-center">
     <img alt="" height="292" src="https://i-blog.csdnimg.cn/direct/4f193f6d1c754fce807e4d546b903f03.png" width="431"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     TCP是如何实现可靠传输的呢？它使用了
     <strong>
      <strong>
       重发
      </strong>
     </strong>
     技术。在TCP传输过程中，发送方会启动一个定时器，然后把数据包发送出去，如果发送方没有在定时器到点之前收到来自接收方的确认，就会重发这个数据包。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       ②用户数据报协议UDP
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       UDP是一种不可靠、无连接的协议
      </strong>
     </strong>
     。UDP协议软件的主要作用是将UDP消息展示给应用层，并不负责重新发送丢失或出错的数据消息，不对接收到的无序IP数据报重新排序。
    </p>

    <h3 style="text-align:justify">
     <strong>
      5.4 Internet地址
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     无论是在网上检索信息还是发送电子邮件，都必须知道对方的
     <strong>
      <strong>
       Internet地址
      </strong>
     </strong>
     ，它能唯一确定Internet上的每一台计算机、每个用户的位置。
     <strong>
      <strong>
       Internet地址有两种书写形式：域名形式和IP地址形式
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. 域名
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     一个完整、通用的层次性主机域名由4个部分组成：计算机主机名.本地名.组名.最高层域名。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如： www.qinghua.edu.cn   cn是地理性顶级域名，表示“中国”。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     www.263.net  net是组织性顶级域名，表示“网络技术组织机构”。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     常用域名后缀：
     <strong>
      <strong>
       com（商业组织、公司），top（顶级、高端、适用于任何商业、公司、个人），edu（教育机构）
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. IP地址
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       Internet中的主机地址是用IP地址来标识的
      </strong>
     </strong>
     。这是因为因特网使用的网络协议是TCP/IP协议，因此每个主机必须用IP地址来标识。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       IP地址（IPv4）由4个8位的二进制数来表示，共32位
      </strong>
     </strong>
     。为了便于使用，通常将二进制“翻译”成十进制，数字之间用“.”分开。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如：

      <strong>
       3. 子网划分
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     为了节约资源、提高效率和安全性，在原有A,B,C类IP地址基础上进行子网划分，根据
     <strong>
      <strong>
       主机个数来划分最合适的方案
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     将
     <strong>
      <strong>
       主机号
      </strong>
     </strong>

     <strong>
      <strong>
       子网号
      </strong>
     </strong>
     ，就可以划分出多个子网，此时IP地址组成为：
     <strong>
      <strong>
       网络号+子网号+主机号
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       子网掩码：网络号、子网号均为1+主机号均为0
      </strong>
     </strong>
     。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

     <strong>
      <strong>
       5. IPv6简介
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       IPv6地址长度为128位
      </strong>
     </strong>
     ，每组由4个十六进制数表示，用“:”分开。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如：FF05:0000:0000:0000:0000:0000:0000:00B3或FF05::00B3（0压缩，连续的0用“:”取代）
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     IPv6数据包有一个
     <strong>
      <strong>
       40字节的基本首部
      </strong>
     </strong>
     ，其后允许有0个或多个扩展首部，再后面是数据。
    </p>
    <p class="img-center">
     <img alt="" height="332" src="https://i-blog.csdnimg.cn/direct/1614f41cd616440ab73d095cb5b9aecf.png" width="571"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     IPv6地址的分类：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ①单播地址：传统的点对点通信。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ②多播地址：一对多的通信，数据包交付到一组计算机中。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ③任播地址：IPv6新增的类型。任播的目的站是一组计算机，但只交付其中一个，通常是距离最近的那个。
    </p>
    <h3 style="text-align:justify">
     <strong>
      5.5 组网技术
     </strong>
    </h3>
    <h4 style="text-align:justify">
     <strong>
      5.5.1 交换技术
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     数据在网络中转发通常离不开交换机。人们日常使用的计算机通常就是通过
     <strong>
      <strong>
       交换机
      </strong>
     </strong>
     接入网络的。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       交换机功能
      </strong>
     </strong>
     包括：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）集线功能。提供大量可供线缆连接的端口达到部署星状拓扑网络的目的。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）中继功能。在转发帧时重新产生不失真的电信号。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）桥接功能。在内置的端口上使用相同的转发和过滤逻辑。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）隔离冲突域功能。将部署好的局域网分为多个冲突域，而每个冲突域都有自己独立的带宽，以提高交换机整体宽带利用效率。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     交换机需要实现的功能：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）转发路径学习。根据收到数据帧中的源MAC地址建立该地址同交换机端口的映射，写入MAC地址表中。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）数据转发。如果交换机根据数据帧中的目的MAC地址在建立好的MAC地址表中查询到了，就向对应端口进行转发。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）数据泛洪。如果数据帧中的目的MAC地址不在MAC地址表中，则向所有端口转发，也就是泛洪。广播帧和组播帧向所有端口(不包括源端口)进行转发。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）链路地址更新。MAC地址表会每隔一定时间(如300s)更新一次。
    </p>
    <h4 style="text-align:justify">
     <strong>
      5.5.2 路由技术
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     路由功能由路由器来提供，具体包括：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）异种网络互连，比如具有异种子网协议的网络互连；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）子网协议转换，不同子网间包括局域网和广域网之间的协议转换；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）数据路由，即将数据从一个网络依据路由规则转发到另一个网络；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）速率适配，利用缓存和流控协议进行适配；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （5）隔离网络，防止广播风暴，实现防火墙；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （6）报文分片和重组，超过接口的MTU(最大传输单元)报文被分片，到达目的地之后的报文被重组；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （7）备份、流量控制，如主备线路的切换和复杂流量控制等。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     路由原理：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     路由器工作在OSI七层协议中的第3层，即
     <strong>
      <strong>
       网络层
      </strong>
     </strong>
     。其主要任务是接收来源于一个网络接口的数据包，通常根据此数据包的目的地址决定待转发的下一个地址(即下一跳地址)。路由器中维持着数据转发所需的
     <strong>
      <strong>
       路由表
      </strong>
     </strong>
     ，所有数据包的发送或转发都通过查找路由表来实现。这个路由表可以静态配置，也可以通过动态路由协议自动生成。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     路由器协议：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）
     <strong>
      <strong>
       内部网关协议
      </strong>
     </strong>
     (Interior Gateway Protocol，
     <strong>
      <strong>
       IGP
      </strong>
     </strong>
     )，是指在一个自治系统（Autonomous System，AS）内运行的路由协议。常见IGP协议：
     <strong>
      <strong>
       RIP
      </strong>
     </strong>
     (路由信息协议)、
     <strong>
      <strong>
       OSPF
      </strong>
     </strong>
     (开放最短路径优先)、
     <strong>
      <strong>
       IS-IS
      </strong>
     </strong>
     (中间系统到中间系统)。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）
     <strong>
      <strong>
       外部网关协议
      </strong>
     </strong>
     (Exterior Gateway Protocol，
     <strong>
      <strong>
       EGP
      </strong>
     </strong>
     )，是指在自治系统AS之间的路由协议。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）
     <strong>
      <strong>
       边界网关协议
      </strong>
     </strong>
     (Border Gateway Protocol，
     <strong>
      <strong>
       BGP
      </strong>
     </strong>
     )，Internet规模庞大、情况复杂，EGP已不适用，已被BGP取代。是指在互联网中的不同自治系统AS之间进行路由选择和路径控制，以确保数据包能够正确地到达目的地。
    </p>
    <h3 style="text-align:justify">
     <strong>
      5.6 网络工程
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       网络建设工程
      </strong>
     </strong>
     可分为
     <strong>
      <strong>
       网络规划、网络设计和网络实施
      </strong>
     </strong>
     三个环节。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. 网络规划
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     网络规划需要以需求为导向，兼顾技术和工程可行性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 网络设计
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     网络设计一般采用分层的方式，分为接入层、汇聚层、核心层。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）
     <strong>
      <strong>
       接入层
      </strong>
     </strong>
     ：主要面向用户端，解决相邻用户之间的互访需求，并且为这些访问提供足够的带宽，接入层还应当适当负责一些用户管理功能(如地址认证、用户认证、计费管理等)，以及用户信息收集工作(如用户的IP地址、MAC地址、访问日志等)。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）
     <strong>
      <strong>
       汇聚层
      </strong>
     </strong>
     ：是核心层和接入层的分界面，将网络业务连接到接入层，并且实施与安全、流量、负载和路由相关的策略。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）
     <strong>
      <strong>
       核心层
      </strong>
     </strong>
     ：是网络主干部分，提供不同区域之间的最佳路由和高速数据传送。因此，核心层交换机应拥有可靠性、性能和吞吐量。核心层的设备采用双机冗余热备份是非常必要的，也可以使用负载均衡功能来改善网络性能。
    </p>
    <p class="img-center">
     <img alt="" height="372" src="https://i-blog.csdnimg.cn/direct/736149f3e63e4a089d29183f6008c326.png" width="645"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       3. 网络实施
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     网络实施是依据网络设计结果进行设备采购、安装、调试和系统切换（需对原有系统改造升级时）等。
    </p>
    <h3 style="text-align:justify">
     <strong>
      5.7 网络存储技术
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     一般来说，计算机访问磁盘存储有3种方式：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·
     <strong>
      <strong>
       直连式存储
      </strong>
     </strong>
     （Direct-Attached Storage，
     <strong>
      <strong>
       DAS
      </strong>
     </strong>
     ），是指将存储设备通过SCSI接口直接连接到一台服务器上使用，其本身是硬件的堆叠，存储操作依赖于服务器，不带有任何存储操作系统。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     存在问题：在传递距离、连接数量、传输速率等方面都受到限制。容量难以扩展升级；数据处理和传输能力较低；服务器异常会波及存储器。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·
     <strong>
      <strong>
       网络连接存储
      </strong>
     </strong>
     （Network-Attached Storage，
     <strong>
      <strong>
       NAS
      </strong>
     </strong>
     ），是通过网络接口与网络直接相连，由用户通过网络访问（支持多种TCP/IP协议），有独立的存储系统。NAS设备有自己的OS，类似于一个专用的文件服务器，一般存储信息采用RAID进行管理，即插即用。不仅响应速度快，而且数据传输速率也很高。NAS的性能特点是进行小文件级的共享存取；支持即插即用；可以很经济的解决存储容量不足的问题，但难以获得满意的性能。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·
     <strong>
      <strong>
       存储区域网络
      </strong>
     </strong>
     （Storage Area Network，
     <strong>
      <strong>
       SAN
      </strong>
     </strong>
     ），是通过专用交换机将磁盘阵列与服务器连接起来的高速专用子网。它没有采用文件共享存取方式，而是采用块(block)级别存储。SAN是通过专用高速网将一个或多个网络存储设备和服务器连接起来的专用存储系统，其最大特点是将存储设备从传统的以太网中分离了出来，成为独立的存储区域网络SAN的系统结构。目前主要使用以太网（IP SAN）和光纤（FC SAN）两类环境。
    </p>
    <p class="img-center">
     <img alt="" height="406" src="https://i-blog.csdnimg.cn/direct/d94729691836429f996c831fefab735d.png" width="703"/>
    </p>
    <h3 style="text-align:justify">
     <strong>
      5.8 磁盘冗余阵列RAID
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       RAID
      </strong>
     </strong>
     ，即
     <strong>
      <strong>
       磁盘冗余阵列技术
      </strong>
     </strong>
     ，将数据分散存储在不同磁盘中，可并行读取，可冗余存储，提高磁盘访问速度，保障数据安全性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID0将数据分散地存储在不同磁盘中，磁盘利用率100%，访问速度最快，但是没有提供冗余和错误修复技术；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID1在成对的独立磁盘上产生互为备份的数据，增加存储可靠性，可以纠错，但磁盘利用率只有50%；
    </p>
    <p class="img-center">
     <img alt="" height="355" src="https://i-blog.csdnimg.cn/direct/496e905a6cfb465db265e6773d395ff2.png" width="594"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID2将数据条块化地分布于不同硬盘上，并使用海明码校验；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID3使用奇偶校验，并用单块磁盘存储奇偶校验信息(可靠性低于RAID5)；
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID5在所有磁盘上交叉地存储数据及奇偶校验信息(所有校验信息存储总量为一个磁盘容量，但分布式存储在不同的磁盘上)，读/写指针可同时操作；
    </p>
    <p class="img-center">
     <img alt="" height="380" src="https://i-blog.csdnimg.cn/direct/c632ed0a786245b7bbc421c0e815c606.png" width="575"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID0+1(是两个RAID0，若一个磁盘损坏，则当前RAID0无法工作，即有一半的磁盘无法工作)；
    </p>
    <p class="img-center">
     <img alt="" height="393" src="https://i-blog.csdnimg.cn/direct/2dd06030979449f6a8a22deae876fcbe.png" width="659"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ·RAID1+0(是两个RAID1，不允许同一组中的两个磁盘同时损坏)与RAID1原理类似，磁盘利用率都只有50%，但安全性更高。
    </p>
    <p class="img-center">
     <img alt="" height="393" src="https://i-blog.csdnimg.cn/direct/695021615f084ef389e4dc9a5e85338f.png" width="661"/>
    </p>
    <h3 style="text-align:justify">
     <strong>
      考点补充
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       1. 网络地址转换NAT
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       NAT
      </strong>
     </strong>
     是用于
     <strong>
      <strong>
       在本地网络中使用私有地址，在连接互联网时转而使用全局 IP 地址的技术
      </strong>
     </strong>
     。NAT实际上是为解决IPv4地址短缺而开发的技术。NAT通常部署在一个组织的网络出口位置，通过将内部网络IP地址替换为出口的IP地址提供公网可达性和上层协议的连接能力。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       2. 默认网关
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     一台主机可以有多个网关。默认网关的意思是一台主机如果找不到可用的网关，就把数据包发给默认指定的网关，由这个网关来处理数据包。现在主机使用的网关，一般指的是默认网关。默认网关的IP地址必须与本机IP地址在同一个网段内，即同网络号。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       3. 虚拟局域网VLAN
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     VLAN是一组逻辑上的设备和用户，这些设备和用户并不受物理位置的限制，可以根据功能、部门及应用等因素将它们组织起来，相互之间的通信就好像它们在同一个网段中一样。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       虚拟局域网VLAN
      </strong>
     </strong>
     工作在OSI参考模型的第2层和第3层，一个VLAN就是一个广播域，VLAN之间的通信是通过第3层的路由器来完成的。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     与传统的局域网技术相比较，VLAN技术更加灵活，它具有以下优点：网络设备的移动、添加和修改的管理开销减少；可以控制广播活动；可提高网络的安全性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       4. 虚拟专用网VPN
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       虚拟专用网VPN是在公用网络上建立专用网络的技术
      </strong>
     </strong>
     。其之所以称为虚拟网，主要是因为整个VPN网络的任意两个节点之间的连接并没有传统专网所需的端到端的物理链路，而是架构在公用网络服务商所提供的网络平台，如Internet、ATM(异步传输模式)、Frame Relay (帧中继)等之上的逻辑网络，用户数据在逻辑链路中传输。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       5. 冲突域和广播域
      </strong>
     </strong>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       路由器可以阻断广播域和冲突域，交换机只能阻断冲突域
      </strong>
     </strong>
     ，因此一个路由器下可以划分多个广播域和多个冲突域；一个交换机下整体是一个广播域，但可以划分多个冲突域；而物理层设备集线器下整体可以作为一个冲突域和一个广播域。
    </p>
