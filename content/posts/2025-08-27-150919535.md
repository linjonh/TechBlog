# res.rows和res.data区别

写前端的时候，傻傻的分不清

#### 1. res.rows 格式

这是分页查询的标准返回格式，通常用于列表页面：

```

{
  "code": 200,
  "msg": "操作成功",
  "total": 100,        // 总记录数
  "rows": [            // 当前页的数据列表
    {
      "id": 1,
      "name": "手机",
      "shopId": 1
    },
    {
      "id": 2, 
      "name": "电脑",
      "shopId": 1
    }
  ]
}
```

使用场景：

* 分页列表查询

* 表格数据展示

* 需要总记录数的场景

#### 2. res.data 格式

这是单个对象或数组的返回格式，通常用于详情、树形结构等：

```

{
  "code": 200,
  "msg": "操作成功",
  "data": [            // 直接返回数据
    {
      "id": 1,
      "name": "手机",
      "children": [
        {
          "id": 11,
          "name": "小米"
        }
      ]
    }
  ]
}
```

使用场景：

* 树形结构数据

* 单个对象详情

* 下拉选项数据

* 不需要分页的场景
