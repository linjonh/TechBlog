    <h2>
     <a id="_0">
     </a>
     前言
    </h2>
    <p>
     Context是鸿蒙中及其重要的Api包括了非常多的接口和功能，而且在官方文档中也是优先推荐使用Context中的接口
    </p>
    <h2>
     <a id="Stage_2">
     </a>
     在Stage模型中基本介绍
    </h2>
    <h4>
     <a id="1_UI_3">
     </a>
     1.
     <strong>
      UI实例与窗口的一一关联
     </strong>
    </h4>
    <ul>
     <li>
      在Stage模型中，
      <code>
       WindowStage
      </code>
      或
      <code>
       Window
      </code>
      是用于管理窗口的类，它们通过
      <code>
       loadContent
      </code>
      接口加载页面内容。
     </li>
     <li>
      当调用
      <code>
       loadContent
      </code>
      时，系统会创建一个UI实例，这个实例是页面内容的具体表现形式，包括布局、控件、样式等。
     </li>
     <li>
      这个UI实例被绑定到调用
      <code>
       loadContent
      </code>
      的窗口上，因此每个窗口都有一个独立的UI实例与之关联。这种一一对应的关系确保了窗口的内容和行为是独立且可管理的。
     </li>
    </ul>
    <h4>
     <a id="2_UIUI_8">
     </a>
     2.
     <strong>
      全局UI接口与UI实例的上下文关联
     </strong>
    </h4>
    <ul>
     <li>
      在Stage模型中，有一些全局的UI接口（例如修改UI样式、更新布局、触发事件等）。
     </li>
     <li>
      这些接口的执行依赖于具体的UI实例上下文。所谓“上下文”，是指当前UI实例的状态、布局、控件等信息。
     </li>
     <li>
      当调用这些全局UI接口时，系统会通过调用链（即代码的执行路径）追溯到当前的UI实例上下文。例如，如果在一个按钮点击事件中调用全局UI接口，系统会根据事件的来源（按钮所在的UI实例）找到对应的上下文。
     </li>
    </ul>
    <h4>
     <a id="3_UI_13">
     </a>
     3.
     <strong>
      非UI页面或异步回调中的问题
     </strong>
    </h4>
    <ul>
     <li>
      在非UI页面（例如纯逻辑代码模块、工具类等）中调用全局UI接口时，由于这些代码没有直接关联到某个UI实例，系统无法通过调用链找到对应的上下文。
     </li>
     <li>
      在异步回调中（例如定时器回调、网络请求完成后的回调等），调用全局UI接口也可能出现问题。因为异步回调的执行时机和上下文可能与最初调用时的上下文不一致。
     </li>
     <li>
      如果系统无法找到正确的UI实例上下文，就会导致全局UI接口无法正确执行，从而引发接口执行失败。
     </li>
    </ul>
    <h4>
     <a id="4__18">
     </a>
     4.
     <strong>
      解决方法
     </strong>
    </h4>
    <p>
     为了避免这种问题，可以采用以下方法：
    </p>
    <ul>
     <li>
      <strong>
       在UI页面中调用全局接口
      </strong>
      ：确保全局UI接口的调用发生在与UI实例直接相关的代码中。
     </li>
     <li>
      <strong>
       手动传递上下文
      </strong>
      ：在异步回调中，可以手动将UI实例的上下文传递给回调函数，以便在回调中正确调用全局UI接口。
     </li>
     <li>
      <strong>
       使用绑定机制
      </strong>
      ：某些框架支持将异步回调与UI实例绑定，确保回调执行时能够正确找到上下文。
     </li>
    </ul>

    <hr/>
    <h4>
     <a id="_121">
     </a>
     <strong>
      组件概述
     </strong>
    </h4>
    <p>
     <code>
      AnimateToExample
     </code>
     是一个标记为入口组件 (
     <code>
      @Entry
     </code>
     ) 和可复用组件 (
     <code>
      @Component
     </code>
     ) 的结构体，用于展示按钮动画效果。它通过
     <code>
      UIContext
     </code>
     提供的动画 API 实现了按钮大小变化和旋转动画的功能。
    </p>
    <hr/>

    <h5>
     <a id="_1change_size_150">
     </a>
     <strong>
      按钮 1：
      <code>
       change size
      </code>
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        功能
       </strong>
       ：点击按钮时，按钮的宽度和高度会在两种状态之间切换。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         UIContext
        </code>
        的使用
       </strong>
       ：
      </p>
      <pre><code class="prism language-arkts">this.uiContext?.animateTo({
  duration: 2000,
  curve: Curve.EaseOut,
  iterations: 1,
  playMode: PlayMode.Normal,
  onFinish: () =&gt; {
    console.info('play end');
  }
}, () =&gt; {
  this.widthSize = 150;
  this.heightSize = 60;
});
</code></pre>
      <ul>
       <li>
        调用
        <code>
         uiContext.animateTo
        </code>
        方法执行动画：
        <ul>
         <li>
          <code>
           duration: 2000
          </code>
          ：动画持续时间为 2000 毫秒。
         </li>
         <li>
          <code>
           curve: Curve.EaseOut
          </code>
          ：动画曲线为
          <code>
           EaseOut
          </code>
          。
         </li>
         <li>
          <code>
           iterations: 1
          </code>
          ：动画循环次数为 1 次。
         </li>
         <li>
          <code>
           playMode: PlayMode.Normal
          </code>
          ：动画播放模式为
          <code>
           Normal
          </code>
          。
         </li>
         <li>
          <code>
           onFinish
          </code>
          ：动画结束时回调函数打印日志
          <code>
           "play end"
          </code>
          。
         </li>
        </ul>
       </li>
       <li>
        动画更新逻辑中，修改按钮的宽度为
        <code>
         150
        </code>
        ，高度为
        <code>
         60
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="_2stop_rotating_176">
     </a>
     <strong>
      按钮 2：
      <code>
       stop rotating
      </code>
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        功能
       </strong>
       ：点击按钮时，停止旋转动画并将旋转角度重置为 0。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         UIContext
        </code>
        的使用
       </strong>
       ：
      </p>
      <pre><code class="prism language-arkts">this.uiContext?.animateTo({
  duration: 1200,
  curve: Curve.Friction,
  delay: 500,
  iterations: -1,
  playMode: PlayMode.Alternate,
  expectedFrameRateRange: {
    min: 10,
    max: 120,
    expected: 60,
  }
}, () =&gt; {
  this.rotateAngle = 90;
});
</code></pre>
      <ul>
       <li>
        <p>
         在组件出现时（
         <code>
          onAppear
         </code>
         方法），调用
         <code>
          uiContext.animateTo
         </code>
         方法启动无限循环的旋转动画：
        </p>
        <ul>
         <li>
          <code>
           duration: 1200
          </code>
          ：动画持续时间为 1200 毫秒。
         </li>
         <li>
          <code>
           curve: Curve.Friction
          </code>
          ：动画曲线为
          <code>
           Friction
          </code>
          。
         </li>
         <li>
          <code>
           delay: 500
          </code>
          ：动画延迟 500 毫秒后开始。
         </li>
         <li>
          <code>
           iterations: -1
          </code>
          ：动画无限循环（
          <code>
           -1
          </code>
          表示无限）。
         </li>
         <li>
          <code>
           playMode: PlayMode.Alternate
          </code>
          ：动画播放模式为
          <code>
           Alternate
          </code>
          （来回交替）。
         </li>
         <li>
          <code>
           expectedFrameRateRange
          </code>
          ：设置期望的帧率范围为
          <code>
           min: 10
          </code>
          ,
          <code>
           max: 120
          </code>
          ,
          <code>
           expected: 60
          </code>
          。
         </li>
         <li>
          动画更新逻辑中，修改旋转角度为
          <code>
           90
          </code>
          度。
         </li>
        </ul>
       </li>
       <li>
        <p>
         点击按钮时，调用
         <code>
          uiContext.animateTo
         </code>
         方法停止旋转动画：
        </p>
        <pre><code class="prism language-arkts">this.uiContext?.animateTo({ duration: 0 }, () =&gt; {
  this.rotateAngle = 0;
});
</code></pre>
        <ul>
         <li>
          设置动画持续时间为 0。
         </li>
         <li>
          将旋转角度重置为
          <code>
           0
          </code>
          度。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>

    <p>

     <code>
      UIContext

    </p>
    <ol>
     <li>
      <strong>
       获取上下文

       getUIContext()
      </code>
      方法获取当前的 UI 上下文。
     </li>
     <li>
      <strong>
       执行动画
      </strong>
      ：通过
      <code>
       animateTo
      </code>
      方法实现平滑的动画效果，支持多种参数配置（如
      <code>
       duration
      </code>
      、
      <code>
       curve
      </code>
      、
      <code>
       iterations
      </code>
      等）。
     </li>
     <li>
      <strong>
       动态更新状态
      </strong>
      ：在动画更新逻辑中，动态修改状态变量（如
      <code>
       widthSize
      </code>
      、
      <code>
       heightSize
      </code>
      和
      <code>
       rotateAngle
      </code>
      ），从而实现按钮大小变化和旋转动画的效果。
     </li>
    </ol>
    <p>
     通过
     <code>
      UIContext
     </code>
