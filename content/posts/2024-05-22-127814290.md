     前端如何在没有后端配合的情况下获取服务器时间？
    </h1>
   </div>

 </div>

    <p>
     现在有一个需求，前端需要同步服务器的时间，因为前端的时间是不可靠的，解决方案有很多，但是不管什么方案，都是需要和服务端进行交互，那么怎么减轻后端的工作量直接实现呢？
    </p>
    <h3>
     <a id="HTTP_Header__Date_4">
     </a>
     HTTP Header 中的 Date
    </h3>
    <p>
     HTTP Header 中有一个字段叫做 Date，它的格式是
     <code>
      RFC 1123
     </code>
     ，也就是
     <code>
      Fri, 30 Sep 2022 02:30:19 GMT
     </code>
     ，这个时间是服务器的时间，我们可以直接从 HTTP Header 中获取这个时间，然后前端直接使用这个时间即可。
    </p>
    <p>
     打开控制台，查看一下请求的 HTTP Header，可以看到 Date 字段，这个就是服务器的时间：
    </p>
    <img src="https://i-blog.csdnimg.cn/blog_migrate/1362bb5ceb83b2bb613c343dd1f08e2a.png">
     <p>
      这里有一个细节，可以看到这个时间的结尾有
      <code>
       GMT
      </code>
      的字母，代表这个时间是
      <code>
       GMT
      </code>
      时间，也就是格林威治时间，我们需要将这个时间转换成本地时间，因为我们前端的时间是本地时间，如果不转换，那么前端的时间就会和服务器的时间不一致。
     </p>
     <img src="https://i-blog.csdnimg.cn/blog_migrate/931d67834f71f6ceb570fb8586368a90.png">
      <p>
       如上图，带了
       <code>
        GMT
       </code>
       的后缀的时间是ok的，不带的就需要转换了。
      </p>
      <h3>
       <a id="_GMT__19">
       </a>
       本地时间和 GMT 时间
      </h3>
      <p>
       我们知道，本地时间和 GMT 时间是有区别的，比如北京时间和 GMT 时间的区别是 8 小时，那么我们如何获取本地时间和 GMT 时间的区别呢？
      </p>
      <pre><code>const date = new Date()
const offset = date.getTimezoneOffset() // 480 
</code></pre>
      <p>
       这里的
       <code>
        offset
       </code>
       就是本地时间和 GMT 时间的区别，单位是分钟，如果是北京时间，那么
       <code>
        offset
       </code>
       就是 480，如果是东京时间，那么
       <code>
        offset
       </code>
       就是 540。
      </p>
      <h3>
       <a id="_GMT__31">
       </a>
       本地时间和 GMT 时间的转换
      </h3>
      <p>
       我们知道了本地时间和 GMT 时间的区别，那么我们就可以将 GMT 时间转换成本地时间了，这里有一个小技巧，我们可以直接使用
       <code>
        new Date()
       </code>
       来转换，因为
       <code>
        new Date()
       </code>
       可以接受
       <code>
        GMT
       </code>
       格式的时间，然后会自动转换成本地时间。
      </p>
      <pre><code>const date = new Date('Fri, 30 Sep 2022 02:30:19 GMT')
console.log(date) // Fri Sep 30 2022 10:30:19 GMT+0800 (中国标准时间) 
</code></pre>
      <p>
       这里我们可以看到，
       <code>
        new Date()
       </code>
       接受了
       <code>
        GMT
       </code>
       格式的时间，然后自动转换成了本地时间。
      </p>
      <h3>
       <a id="_43">
       </a>
       前端实现
      </h3>
      <p>
       现在我们已经知道了如何获取服务器的时间，以及如何将 GMT 时间转换成本地时间，那么我们就可以实现前端获取服务器时间了。
      </p>
      <pre><code>axios.head('https://www.baidu.com?_=' + new Date().getTime()).then(res =&gt; {const date = new Date(res.headers.date)console.log(date) // Fri Sep 30 2022 10:30:19 GMT+0800 (中国标准时间)
}) 
</code></pre>
      <p>
       这里使用
       <code>
        axios
       </code>
       发送一个
       <code>
        head
       </code>
       请求，然后获取
       <code>
        Date
       </code>
       字段，然后转换成本地时间，这样就可以获取到服务器的时间了，加时间戳是为了防止浏览器缓存。
      </p>
      <h3>
       <a id="_55">
       </a>
       优化
      </h3>
      <p>
       上面的代码是可以正常工作的，但是有一个问题，就是每次都要发送一个
       <code>
        head
       </code>
       请求，这样会增加服务器的压力，所以我们可以优化一下，比如我们可以在页面加载的时候就获取服务器的时间，然后每隔一段时间再获取一次，这样就可以减少服务器的压力了。
      </p>
      <pre><code>let date = new Date()
function getServerTime() {setTimeout(() =&gt; {axios.head('https://www.baidu.com?_=' + new Date().getTime()).then(res =&gt; {date = new Date(res.headers.date)console.log(date) // Fri Sep 30 2022 10:30:19 GMT+0800 (中国标准时间)}).finally(() =&gt; {getServerTime()})}, 1000 * 60 * 5)
}
getServerTime()

function timeStep(time) {date.setTime(date.getTime() + time)setTimeout(() =&gt; {timeStep(time)}, time)
}
timeStep(1000);
</code></pre>
<p>
这里我们使用
<code>
setTimeout
</code>
来模拟服务器的时间，然后每隔一段时间就获取一次服务器的时间，这样就可以减少服务器的压力了。
</p>
<h3>
<a id="_73">
</a>
总结
</h3>
<p>
没啥好总结的，就是获取服务器的时间，然后转换成本地时间，然后模拟服务器的时间，这样就可以实现前端获取服务器时间了。
</p>
