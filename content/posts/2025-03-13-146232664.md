    <p>
     如果你想试用或学习 Linux 环境，但又不想破坏现有系统，在虚拟机里安装 Ubuntu（或其他 Linux 发行版）是一种非常安全、方便的方式。通过 VMware Workstation / VMware Player 等虚拟机软件，可以在 Windows 或 macOS 等操作系统上快速创建并运行一个 “软件沙箱”，其中安装和使用 Ubuntu 的过程非常直观且不会影响到主机系统。本文将结合个人经验，为你呈现一篇图文并茂（文字说明）的详细教程，帮助你从 “零基础” 到 “能用 Ubuntu”。
    </p>
    <h3>
     <a id="1__5">
     </a>
     1. 安装准备
    </h3>
    <p>
     在开始之前，我们需要先准备好以下两项内容：
     <strong>
      VMware 虚拟机软件
     </strong>
     和
     <strong>
      Ubuntu 系统镜像
     </strong>
     。此外，还需确保你的主机（电脑）上有足够的硬盘空间与内存资源，以容纳一个完整的 Ubuntu 系统以及其他后续可能安装的软件。
    </p>
    <h4>
     <a id="VMware__9">
     </a>
     VMware 软件获取
    </h4>
    <ul>
     <li>
      <strong>
       VMware Workstation Pro
      </strong>
      （付费版本，功能更丰富）：适用于经常使用虚拟机、需要专业功能（如快照、多虚拟网络配置等）以及商业用途的人群。
     </li>
     <li>
      <strong>
       VMware Workstation Player
      </strong>
      （免费版，非商业用途）：适用于个人学习、开发测试，以及简单的虚拟机场景。
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       下载地址（官方）不能下载了
      </strong>
      <br/>
      访问
      <a href="https://www.vmware.com/" rel="nofollow">
       VMware 官网
      </a>
      （需自行科学上网或搜索国内镜像）下载对应版本。
      <br/>
      下载完成后，按照常规的软件安装流程 “下一步-下一步” 即可完成，建议全程使用默认选项，不做过多修改。
      <br/>
      具体下载步骤参考：
      <a href="https://blog.csdn.net/cute_neko/article/details/140409619">
       [2024年]最新VMware Workstation虚拟机下载并安装
      </a>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b1e6b0fd9fb14dcaa8b8d7e50047cacc.png"/>
     </p>
    </blockquote>
    <h4>
     <a id="Ubuntu__20">
     </a>
     Ubuntu 镜像获取
    </h4>
    <p>
     Ubuntu 提供了多个版本（LTS 长期支持版和最新正式版）。通常情况下，
     <strong>
      LTS 版
     </strong>
     更适合对系统稳定性和长周期更新有需求的场景，而
     <strong>
      最新正式版
     </strong>
     通常包含更多新特性。
    </p>
    <blockquote>
     <p>
      <strong>
       下载地址（官方）
      </strong>
      <br/>
      访问
      <a href="https://ubuntu.com/download" rel="nofollow">
       Ubuntu 官网
      </a>
      下载最新的 ISO 镜像或 LTS 版本。文件大小一般在 2~3 GB 之间。
     </p>
    </blockquote>
    <p>
     下载完成后，建议核对一下镜像文件的 MD5 或 SHA256，确保下载文件无损坏（如果环境允许的话），然后就可以拿这个镜像来安装了。
    </p>
    <hr/>
    <h3>
     <a id="2__31">
     </a>
     2. 创建新的虚拟机
    </h3>
    <p>
     以
     <strong>
      VMware Workstation
     </strong>
     为例，打开软件后，我们可以按照下面的步骤创建一个新的 Ubuntu 虚拟机。
    </p>
    <h4>
     <a id="_35">
     </a>
     基础配置
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        启动 VMware
       </strong>
       ：双击图标进入主界面。 ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/2dbeb67573f94642a5adfa3509cbcd02.png
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/fbcfb628468d4036b0338821c1f0da24.png"/>
      </p>
     </li>
     <li>
      <p>
       <strong>
        创建新虚拟机
       </strong>
       ：点击 “File（文件）→ New Virtual Machine（新建虚拟机）” 或在主界面中选择 “Create a New Virtual Machine”。
      </p>
     </li>
     <li>
      <p>
       <strong>
        选择创建类型
       </strong>
       ：一般会出现两个选项：
      </p>
      <ul>
       <li>
        <strong>
         Typical（典型）
        </strong>
        ：大多数情况下选择典型就可以满足需求，它会自动帮你完成常规设置。
       </li>
       <li>
        <strong>
         Custom（自定义）
        </strong>
        ：如果你对网络、硬件兼容性、SCSI 控制器类型、固件类型等有特殊要求，可以使用自定义模式。
       </li>
      </ul>
     </li>
    </ol>
    <blockquote>
     <p>
      <strong>
       建议
      </strong>
      ：初次接触或并无特别需求时，选择 “Typical（典型）” 即可。
     </p>
    </blockquote>
    <ol start="4">
     <li>
      <strong>
       选择安装来源
      </strong>
      ：VMware 会提示是否从光盘（ISO）或物理驱动器安装，或者稍后安装操作系统。我们选择已经下载好的
      <strong>
       Ubuntu ISO 镜像
      </strong>
      文件。
     </li>
     <li>
      <strong>
       选择操作系统类型
      </strong>
      ：如果 VMware 能自动检测到是 Ubuntu，就会在下拉框中自动选中 “Linux → Ubuntu 64-bit”。如果没有检测到，可以手动选择对应的版本。
     </li>
     <li>
      <strong>
       虚拟机名称与存储位置
      </strong>
      ：给虚拟机起一个有意义的名称，比如
      <code>
       Ubuntu_22.04_LTS_VM
      </code>
      ，并选择一个硬盘位置来存放虚拟机文件（vmdk、vmx 等）。
     </li>
     <li>
      <strong>
       磁盘容量设置
      </strong>
      ：通常，默认 20GB 或 30GB 就可以满足基本的测试需求，但如果你要在 Ubuntu 中安装大量软件或进行开发，建议酌情增加到 40~60GB 甚至更多。
      <ul>
       <li>
        <strong>
         单个文件存储
        </strong>
        vs
        <strong>
         分割成多个 2GB 文件
        </strong>
        ：两者对性能影响不大，但分割成多个文件在一些文件系统上更兼容，单个文件则更方便管理和移动。根据个人习惯选择即可。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     完成以上步骤后，VMware 会生成一个基本配置的虚拟机，接下来我们可以再做一些硬件自定义设置。
    </p>
    <h4>
     <a id="_54">
     </a>
     自定义硬件设置
    </h4>
    <ol>
     <li>
      <strong>
       CPU 和内存
      </strong>
      ：在新虚拟机向导的最后一页或虚拟机 “设置 (Settings)” 中，可以选择给虚拟机分配多少 CPU 核心和多少内存。
      <ul>
       <li>
        如果你的主机有 8GB 内存，建议给 Ubuntu 虚拟机分配
        <strong>
         2GB~4GB
        </strong>
        （最少 2GB，建议 4GB 以保证流畅度）；如果主机内存更大，也可以分配更多。
       </li>
       <li>
        CPU 可以根据需要选择 1~2 个核心起步，若主机配置较高，也可适当增加。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       网络适配器
      </strong>
      ：默认采用
      <strong>
       NAT
      </strong>
      模式，Ubuntu 虚拟机可以通过主机共享网络访问外网，主机和虚拟机之间也能互相通信。若有特殊需求，可改为
      <strong>
       Bridge
      </strong>
      （直通网络）或
      <strong>
       Host-Only
      </strong>
      （仅主机互通）等。
     </li>
     <li>
      <strong>
       其他设置
      </strong>
      ：
      <ul>
       <li>
        <strong>
         显示 (Display)
        </strong>
        ：如果要启用 3D 加速，可以勾选相关选项，不过对于多数场景而言，默认设置就足够了。
       </li>
       <li>
        <strong>
         声卡 (Sound card)
        </strong>
        、
        <strong>
         USB 控制器
        </strong>
        、
        <strong>
         光驱
        </strong>
        等保持默认即可。
       </li>
       <li>
        <strong>
         ISO 文件
        </strong>
        确保已选中 Ubuntu 的镜像，如果没有可以在 “CD/DVD” 设备中重新选择。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     至此，我们已经完成了一台 “准备用来安装 Ubuntu 的虚拟机” 的基本配置。
    </p>
    <hr/>
    <h3>
     <a id="3_Ubuntu__69">
     </a>
     3. Ubuntu 系统安装过程
    </h3>
    <h4>
     <a id="_71">
     </a>
     启动虚拟机
    </h4>
    <ol>
     <li>
      <strong>
       打开虚拟机
      </strong>
      ：点击 VMware 主界面中的虚拟机列表，选中刚才创建的 Ubuntu 虚拟机，然后点击 “Power on this virtual machine” 或 “启动虚拟机”。
     </li>
     <li>
      <strong>
       引导进入安装界面
      </strong>
      ：如果一切正常，VMware 会检测到 ISO 镜像并从中引导，出现 Ubuntu 的启动菜单。通常会看到 “Try Ubuntu without installing”（试用模式） 和 “Install Ubuntu”（安装模式）等选项。
     </li>
    </ol>
    <blockquote>
     <p>
      如果没有出现该界面，可能是启动顺序或引导方式出了问题，请检查虚拟机的 “CD/DVD” 设置是否加载了 Ubuntu 的 ISO，并保证它的启动优先级在硬盘之前。
     </p>
    </blockquote>
    <h4>
     <a id="_Ubuntu_78">
     </a>
     正式安装 Ubuntu
    </h4>
    <ol>
     <li>
      <strong>
       选择语言
      </strong>
      ：在最初的欢迎界面上，通常会让你选择系统安装语言。这里可以选择
      <strong>
       简体中文
      </strong>
      （或英文，视个人习惯而定）。
     </li>
     <li>
      <strong>
       键盘布局
      </strong>
      ：如果你希望使用中文或英文键盘布局，可在此选择。如果默认 US 键盘即可，也可保持默认。
     </li>
     <li>
      <strong>
       尝试或直接安装
      </strong>
      ：通常我们会选择
      <strong>
       直接安装 Ubuntu
      </strong>
      ，当然也可以先试用一下（Live 模式），确认硬件兼容后再安装。
     </li>
     <li>
      <strong>
       安装类型
      </strong>
      ：在虚拟机中安装时，一般选择
      <strong>
       Erase disk and install Ubuntu
      </strong>
      （清空磁盘并安装）。由于这是个虚拟磁盘，不会影响真实主机。
     </li>
     <li>
      <strong>
       分区
      </strong>
      ：Ubuntu 安装器会为你自动分配根目录
      <code>
       /
      </code>
      、swap 分区（如果需要）等，一般直接下一步即可。如果你比较熟悉 Linux 分区结构，可以根据需要执行手动分区（/boot, /home, /var 等分开），但对初学者而言，“自动分区” 已经足够。
     </li>
     <li>
      <strong>
       时区
      </strong>
      ：选择所在区域（如 Shanghai）或根据实际位置进行设置。
     </li>
     <li>
      <strong>
       创建用户
      </strong>
      ：设置一个用户名、计算机名和密码。记好这个密码，后续你需要它来安装软件、提升权限（sudo）等操作。
     </li>
     <li>
      <strong>
       开始安装
      </strong>
      ：点击 “Install Now” 或 “开始安装”，Ubuntu 安装程序会正式往虚拟磁盘中写入文件，根据你的硬件配置速度，大约需要 5~20 分钟。
     </li>
    </ol>
    <h4>
     <a id="_89">
     </a>
     安装过程中常见问题
    </h4>
    <ul>
     <li>
      <strong>
       提示缺少空间
      </strong>
      ：如果在前面没有设置足够大的虚拟磁盘，可能会弹出提示空间不足。可以回到虚拟机设置中修改磁盘大小，或在创建时就给出更大的空间。
     </li>
     <li>
      <strong>
       安装卡顿或闪屏
      </strong>
      ：可能是显卡驱动或 VM 兼容性引起的临时现象，一般安装完成后再更新驱动即可。若安装卡死，建议在重启后尝试 “safe graphics” 模式。
     </li>
    </ul>
    <p>
     安装完成后，会提示重启 Ubuntu 虚拟机。此时，记得在 VMware 虚拟机设置里移除（或取消挂载）ISO 镜像，以免下次启动时又进到安装界面。
    </p>
    <hr/>
    <h3>
     <a id="4__98">
     </a>
     4. 安装后优化
    </h3>
    <p>
     成功进入 Ubuntu 系统桌面后，你就可以享受到一套功能完备的 Linux 系统了。不过，为了更好的使用体验，可以进行以下优化操作。
    </p>
    <h4>
     <a id="_VMware_Tools_102">
     </a>
     安装 VMware Tools
    </h4>
    <p>
     VMware Tools 可以帮助虚拟机和宿主机间更好地交互，比如：
    </p>
    <ul>
     <li>
      在虚拟机中实现自动分辨率调整、鼠标无缝切换
     </li>
     <li>
      共享文件夹、剪贴板
     </li>
     <li>
      提高图形性能
     </li>
    </ul>
    <p>
     <strong>
      安装方法
     </strong>
     ：（以 Ubuntu 22.04 为例）
    </p>
    <ol>
     <li>
      在 VMware 菜单栏中选择：
      <strong>
       “VM” → “Install VMware Tools”
      </strong>
      （如果显示 “Reinstall VMware Tools”，说明之前已经挂载了）
     </li>
     <li>
      Ubuntu 中会自动挂载一个光盘镜像（通常在 /media/ 目录下），里面包含
      <code>
       VMwareTools-xxx.tar.gz
      </code>
      压缩包。
     </li>
     <li>
      将该文件复制到任意位置（例如
      <code>
       ~/Documents
      </code>
      ），然后解压：
      <pre><code class="prism language-bash"><span class="token function">tar</span> <span class="token parameter variable">-zxvf</span> VMwareTools-xxx.tar.gz
</code></pre>
     </li>
     <li>
      进入解压后文件夹，运行安装脚本：
      <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> vmware-tools-distrib
<span class="token function">sudo</span> ./vmware-install.pl
</code></pre>
     </li>
     <li>
      安装脚本会提示一系列问题，通常直接按
      <strong>
       Enter
      </strong>
      接受默认选项即可。
     </li>
     <li>
      安装完成后，建议重启 Ubuntu：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">reboot</span>
</code></pre>
     </li>
    </ol>
    <p>
     若安装过程非常繁琐，也可以在 Ubuntu 软件源中安装
     <strong>
      open-vm-tools
     </strong>
     和
     <strong>
      open-vm-tools-desktop
     </strong>
     （该套件也是 VMware Tools 的开源版本）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> open-vm-tools open-vm-tools-desktop <span class="token parameter variable">-y</span>
<span class="token function">sudo</span> <span class="token function">reboot</span>
</code></pre>
    <h4>
     <a id="_137">
     </a>
     系统更新与软件安装
    </h4>
    <p>
     Ubuntu 下的软件中心和命令行包管理器（
     <code>
      apt
     </code>
     ）都非常方便。安装完成后，建议先进行一次系统更新：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> upgrade <span class="token parameter variable">-y</span>
</code></pre>
    <p>
     然后可以根据需要安装常用软件，比如
     <code>
      git
     </code>
     ,
     <code>
      curl
     </code>
     ,
     <code>
      vim
     </code>
     ,
     <code>
      build-essential
     </code>
     等：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token function">git</span> <span class="token function">curl</span> <span class="token function">vim</span> build-essential <span class="token parameter variable">-y</span>
</code></pre>
    <h4>
     <a id="_152">
     </a>
     分辨率与显示设置
    </h4>
    <p>
     如果已经安装 VMware Tools 或 open-vm-tools-desktop，Ubuntu 会根据 VM 窗口大小自动调整分辨率。若出现分辨率异常，可进入
     <strong>
      Settings → Displays
     </strong>
     （设置→显示）里手动选择合适的分辨率。
    </p>
    <hr/>
    <h3>
     <a id="5__158">
     </a>
     5. 常见故障及解决方案
    </h3>
    <h4>
     <a id="_160">
     </a>
     黑屏或安装卡顿
    </h4>
    <ul>
     <li>
      <strong>
       原因
      </strong>
      ：常见于在 VM 中启动安装时，显卡兼容或某些硬件选项冲突。
     </li>
     <li>
      <strong>
       解决
      </strong>
      ：
      <ol>
       <li>
        重启时在引导菜单选择 “Safe graphics” 或 “nomodeset” 参数安装；
       </li>
       <li>
        在 VMware 设置里调整显卡 3D 加速开关；
       </li>
       <li>
        提高给虚拟机分配的内存或 CPU 资源。
       </li>
      </ol>
     </li>
    </ul>
    <h4>
     <a id="_168">
     </a>
     网络配置问题
    </h4>
    <ul>
     <li>
      <strong>
       NAT 模式无法上网
      </strong>
      ：重启网络或尝试
      <code>
       ping www.google.com
      </code>
      （若无法访问谷歌可换国内网站），检查 DNS 设置（在
      <code>
       /etc/resolv.conf
      </code>
      ）或 VMware 虚拟网卡是否启用。
     </li>
     <li>
      <strong>
       Bridge 模式获取不到 IP
      </strong>
      ：可能是宿主机网络配置冲突，需要在 VMware Virtual Network Editor 中查看网桥是否绑定了正确的物理网卡。
     </li>
    </ul>
    <h4>
     <a id="_173">
     </a>
     其他可能的问题
    </h4>
    <ul>
     <li>
      <strong>
       无法复制粘贴
      </strong>
      ：检查是否安装了 VMware Tools，并在虚拟机设置中打开共享剪贴板功能。
     </li>
     <li>
      <strong>
       共享文件夹无法访问
      </strong>
      ：在 VMware 设置中启用 “Shared Folders”，并在 Ubuntu 中挂载对应目录；或者手动使用
      <code>
       vmhgfs
      </code>
      挂载。
     </li>
     <li>
      <strong>
       磁盘空间不足
      </strong>
      ：可以使用 VMware 的磁盘扩展功能或在创建时就分配更大空间。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="6__181">
     </a>
     6. 总结
    </h3>
    <p>
     通过本文的步骤，你已经能够在 VMware 中顺利创建并安装一个全新的 Ubuntu 系统。这个虚拟机环境可用于：
    </p>
    <ul>
     <li>
      <strong>
       学习 Linux 命令行和系统操作
      </strong>
     </li>
     <li>
      <strong>
       进行软件开发、编译、调试
      </strong>
     </li>
     <li>
      <strong>
       搭建服务器环境（例如 LAMP, Docker 等）
      </strong>
     </li>
     <li>
      <strong>
       安全地测试未知软件或脚本
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      VMware + Ubuntu
     </strong>
     的组合让我们在不影响主机系统的情况下，充分体验 Linux 的强大和灵活。如果在此过程中遇到什么问题，可以多搜索 Ubuntu 官方社区或 VMware 论坛，几乎所有问题都能找到相应的解决方案。
    </p>
