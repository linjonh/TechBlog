    <h3>
     <a id="_0">
     </a>
     目标
    </h3>
    <p>
     理解
    </p>
    <ol>
     <li>
      在 Go 中，赋值操作既包括最基本的形式（左边一个变量，右边一个表达式），也包括复合赋值、元组赋值和隐式赋值。
     </li>
     <li>
      表达式求值的顺序、变量更新时的副作用以及如何处理多返回值和下划线（_）丢弃不需要的值
     </li>
    </ol>
    <h3>
     <a id="_7">
     </a>
     概念
    </h3>
    <ol>
     <li>
      <p>
       可赋值性
      </p>
      <p>
       赋值语句要求左侧变量和右侧表达式的最终值必须有相同的数据类型：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          严格的类型匹配
         </strong>
        </p>
        <p>
         只有当右侧的值对于左侧变量是可赋值的，赋值才是允许的。如果类型不匹配，编译器会报错。
        </p>
       </li>
       <li>
        <p>
         <strong>
          特殊情况：nil 与常量
         </strong>
        </p>
        <ul>
         <li>
          nil 可以赋值给任何指针或引用类型的变量。
         </li>
         <li>
          常量赋值有更灵活的规则，允许在一定条件下进行隐式转换，以避免额外的类型转换操作。
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          与相等比较的关系
         </strong>
        </p>
        <p>
         对于使用
         <code>
          ==
         </code>
         或
         <code>
          !=
         </code>
         进行比较时，只有当右侧的值对于左侧变量可赋值时，相等比较才有意义。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       表达式求值顺序：
      </p>
      <ul>
       <li>
        在元组赋值中，右侧所有表达式会先求值，然后再一次性将结果赋给左侧的各个变量。这保证了在交换变量值或多个变量依赖的情况下不会出现意外的中间状态。
       </li>
      </ul>
     </li>
     <li>
      <p>
       变量的概念与内存模型：每个变量在内存中都有固定位置，赋值语句实际上更新该位置中的数据。
      </p>
     </li>
    </ol>
    <h3>
     <a id="_28">
     </a>
     要点
    </h3>
    <h4>
     <a id="_30">
     </a>
     显式赋值
    </h4>
    <p>
     最基本的赋值形式是将一个表达式的值赋给一个变量或可寻址的表达式。例如：
    </p>
    <ul>
     <li>
      <p>
       基本变量赋值
      </p>
      <pre><code class="prism language-go">x <span class="token operator">=</span> <span class="token number">1</span>
</code></pre>
      <p>
       将整数 1 赋给变量 x。
      </p>
     </li>
     <li>
      <p>
       指针赋值
      </p>
      <pre><code class="prism language-go"><span class="token operator">*</span>p <span class="token operator">=</span> <span class="token boolean">true</span>
</code></pre>
      <p>
       通过指针 p 间接将布尔值 true 赋给 p 指向的变量。
      </p>
     </li>
     <li>
      <p>
       结构体字段赋值
      </p>
      <pre><code class="prism language-go">person<span class="token punctuation">.</span>name <span class="token operator">=</span> <span class="token string">"bob"</span>
</code></pre>
     </li>
     <li>
      <p>
       数组、slice 或 map 元素赋值
      </p>
      <pre><code class="prism language-go">count<span class="token punctuation">[</span>x<span class="token punctuation">]</span> <span class="token operator">=</span> count<span class="token punctuation">[</span>x<span class="token punctuation">]</span> <span class="token operator">*</span> scale
</code></pre>
     </li>
    </ul>
    <ol>
     <li>
      <p>
       复合赋值运算符
      </p>
      <p>
       对于某些二元算术运算和赋值的组合操作，Go 提供了简洁的写法
      </p>
      <pre><code class="prism language-go">count<span class="token punctuation">[</span>x<span class="token punctuation">]</span> <span class="token operator">*=</span> scale
</code></pre>
      <p>
       这种复合赋值可以减少对变量表达式的重复计算，使代码更简洁。
      </p>
     </li>
    </ol>
    <h4>
     <a id="_73">
     </a>
     自增与自减
    </h4>
    <pre><code class="prism language-go">v <span class="token operator">:=</span> <span class="token number">1</span>
v<span class="token operator">++</span>    <span class="token comment">// 等价于 v = v + 1，v 变为 2</span>
v<span class="token operator">--</span>    <span class="token comment">// 等价于 v = v - 1，v 变为 1</span>
</code></pre>
    <ul>
     <li>
      自增和自减是语句而非表达式，不能像其它语言那样嵌入表达式中。
      <ul>
       <li>
        不能将自增、自减操作用于赋值表达式中，如不能写成
        <code>
         x = i++
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_84">
     </a>
     元组赋值（多重赋值）
    </h4>
    <p>
     元组赋值允许同时更新多个变量的值，且在更新前
     <strong>
      右边所有表达式都先计算好，再统一更新左边对应的变量
     </strong>
     。这对于解决一些变量相互依赖的问题非常有用：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        交换两个变量
       </strong>
      </p>
      <pre><code class="prism language-go">
x<span class="token punctuation">,</span> y <span class="token operator">=</span> y<span class="token punctuation">,</span> x
</code></pre>
      <p>
       同时交换 x 与 y 的值，而不需要临时变量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        交换数组或 slice 中的元素
       </strong>
      </p>
      <pre><code class="prism language-go">
a<span class="token punctuation">[</span>i<span class="token punctuation">]</span><span class="token punctuation">,</span> a<span class="token punctuation">[</span>j<span class="token punctuation">]</span> <span class="token operator">=</span> a<span class="token punctuation">[</span>j<span class="token punctuation">]</span><span class="token punctuation">,</span> a<span class="token punctuation">[</span>i<span class="token punctuation">]</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        实际应用：计算最大公约数（GCD）
       </strong>
      </p>
      <pre><code class="prism language-go">
<span class="token keyword">func</span> <span class="token function">gcd</span><span class="token punctuation">(</span>x<span class="token punctuation">,</span> y <span class="token builtin">int</span><span class="token punctuation">)</span> <span class="token builtin">int</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">for</span> y <span class="token operator">!=</span> <span class="token number">0</span> <span class="token punctuation">{<!-- --></span>
        x<span class="token punctuation">,</span> y <span class="token operator">=</span> y<span class="token punctuation">,</span> x<span class="token operator">%</span>y
    <span class="token punctuation">}</span>
    <span class="token keyword">return</span> x
<span class="token punctuation">}</span>

</code></pre>
      <p>
       每次循环中先计算 x%y，再将旧的 y 和 x%y 分别赋值给 x 和 y，实现算法逻辑。
      </p>
     </li>
     <li>
      <p>
       <strong>
        计算斐波那契数
       </strong>
      </p>
      <pre><code class="prism language-go">
<span class="token keyword">func</span> <span class="token function">fib</span><span class="token punctuation">(</span>n <span class="token builtin">int</span><span class="token punctuation">)</span> <span class="token builtin">int</span> <span class="token punctuation">{<!-- --></span>
    x<span class="token punctuation">,</span> y <span class="token operator">:=</span> <span class="token number">0</span><span class="token punctuation">,</span> <span class="token number">1</span>
    <span class="token keyword">for</span> i <span class="token operator">:=</span> <span class="token number">0</span><span class="token punctuation">;</span> i <span class="token operator">&lt;</span> n<span class="token punctuation">;</span> i<span class="token operator">++</span> <span class="token punctuation">{<!-- --></span>
        x<span class="token punctuation">,</span> y <span class="token operator">=</span> y<span class="token punctuation">,</span> x<span class="token operator">+</span>y
    <span class="token punctuation">}</span>
    <span class="token keyword">return</span> x
<span class="token punctuation">}</span>

</code></pre>
      <p>
       每次迭代更新 x 和 y，用于生成斐波那契数列。
      </p>
     </li>
     <li>
      <p>
       <strong>
        元组赋值在 for 循环中也可使用
       </strong>
      </p>
      <pre><code class="prism language-go">
i<span class="token punctuation">,</span> j<span class="token punctuation">,</span> k <span class="token operator">=</span> <span class="token number">2</span><span class="token punctuation">,</span> <span class="token number">3</span><span class="token punctuation">,</span> <span class="token number">5</span>

</code></pre>
      <p>
       注意：当赋值表达式较复杂时，过度使用元组赋值可能影响代码的可读性，简单情况优先考虑单个赋值。
      </p>
     </li>
    </ul>
    <h4>
     <a id="_146">
     </a>
     处理多个返回值
    </h4>
    <p>
     在 Go 中，很多函数会返回多个值，例如错误处理常见于函数调用。元组赋值允许将这些值同时赋给多个变量：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        函数返回两个值
       </strong>
      </p>
      <pre><code class="prism language-go">
f<span class="token punctuation">,</span> err <span class="token operator">=</span> os<span class="token punctuation">.</span><span class="token function">Open</span><span class="token punctuation">(</span><span class="token string">"foo.txt"</span><span class="token punctuation">)</span>
</code></pre>
      <p>
       要求左侧的变量数量与函数返回值数量匹配。
      </p>
     </li>
     <li>
      <p>
       <strong>
        map 查找、类型断言、通道接收
       </strong>
      </p>
      <p>
       这些操作有时会产生两个结果：
      </p>
      <pre><code class="prism language-go">
v<span class="token punctuation">,</span> ok <span class="token operator">=</span> m<span class="token punctuation">[</span>key<span class="token punctuation">]</span>     <span class="token comment">// map 查找，ok 表示是否找到对应的键</span>
v<span class="token punctuation">,</span> ok <span class="token operator">=</span> x<span class="token punctuation">.</span><span class="token punctuation">(</span>T<span class="token punctuation">)</span>      <span class="token comment">// 类型断言，ok 表示断言是否成功</span>
v<span class="token punctuation">,</span> ok <span class="token operator">=</span> <span class="token operator">&lt;-</span>ch       <span class="token comment">// 通道接收，ok 表示接收是否成功（通道关闭时返回零值和 false）</span>
</code></pre>
      <p>
       但它们也可以只产生一个结果：
      </p>
      <pre><code class="prism language-go">
v <span class="token operator">=</span> m<span class="token punctuation">[</span>key<span class="token punctuation">]</span>         <span class="token comment">// 查找失败时返回零值</span>
v <span class="token operator">=</span> x<span class="token punctuation">.</span><span class="token punctuation">(</span>T<span class="token punctuation">)</span>          <span class="token comment">// 类型断言失败时会触发 panic</span>
v <span class="token operator">=</span> <span class="token operator">&lt;-</span>ch           <span class="token comment">// 如果通道阻塞，则等待（不算失败）</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        使用空白标识符丢弃不需要的值
       </strong>
      </p>
      <p>
       如果不需要某个返回值，可以使用下划线
       <code>
        _
       </code>
       来丢弃：
      </p>
      <pre><code class="prism language-go">
<span class="token boolean">_</span><span class="token punctuation">,</span> err <span class="token operator">=</span> io<span class="token punctuation">.</span><span class="token function">Copy</span><span class="token punctuation">(</span>dst<span class="token punctuation">,</span> src<span class="token punctuation">)</span> <span class="token comment">// 丢弃字节数</span>
<span class="token boolean">_</span><span class="token punctuation">,</span> ok <span class="token operator">=</span> x<span class="token punctuation">.</span><span class="token punctuation">(</span>T<span class="token punctuation">)</span> <span class="token comment">// 只检测类型，忽略具体值</span>
</code></pre>
     </li>
    </ul>
    <h4>
     <a id="_190">
     </a>
     隐式赋值行为
    </h4>
    <p>
     赋值不仅仅通过显式的赋值语句实现，很多场景中会隐式地发生赋值：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        函数调用
       </strong>
      </p>
      <p>
       调用函数时，调用参数的值会隐式地赋给函数的参数变量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        返回语句
       </strong>
      </p>
      <p>
       返回值会隐式赋值给结果变量。
      </p>
     </li>
     <li>
      <p>
       <strong>
        复合类型字面量
       </strong>
      </p>
      <p>
       创建数组、slice、map 或结构体时，字面量初始化会隐式对每个元素或字段进行赋值。例如：
      </p>
      <pre><code class="prism language-go">medals <span class="token operator">:=</span> <span class="token punctuation">[</span><span class="token punctuation">]</span><span class="token builtin">string</span><span class="token punctuation">{<!-- --></span><span class="token string">"gold"</span><span class="token punctuation">,</span> <span class="token string">"silver"</span><span class="token punctuation">,</span> <span class="token string">"bronze"</span><span class="token punctuation">}</span>
</code></pre>
      <p>
       相当于
      </p>
      <pre><code class="prism language-go">medals<span class="token punctuation">[</span><span class="token number">0</span><span class="token punctuation">]</span> <span class="token operator">=</span> <span class="token string">"gold"</span>
medals<span class="token punctuation">[</span><span class="token number">1</span><span class="token punctuation">]</span> <span class="token operator">=</span> <span class="token string">"silver"</span>
medals<span class="token punctuation">[</span><span class="token number">2</span><span class="token punctuation">]</span> <span class="token operator">=</span> <span class="token string">"bronze"</span>
</code></pre>
     </li>
    </ul>
    <h3>
     <a id="_219">
     </a>
     语言特性
    </h3>
    <ol>
     <li>
      <p>
       在有命名返回值的函数中，当执行返回语句时，返回值会自动赋值给这些命名的返回变量，不需要在返回语句中单独写出它们。因为在函数体内，这些结果变量已经赋得了值。
      </p>
      <p>
       在 Go 语言中，函数可以定义命名返回值，也就是在函数签名中为返回值起一个名字。这样做的效果是，这些命名的返回值在函数体内就像局部变量一样存在。当执行 return 语句时，如果没有显式指定返回值，那么当前这些变量的值就会自动（隐式地）作为函数的返回值。
      </p>
      <p>
       例如，考虑下面的例子：
      </p>
      <pre><code class="prism language-go">
<span class="token keyword">func</span> <span class="token function">add</span><span class="token punctuation">(</span>a<span class="token punctuation">,</span> b <span class="token builtin">int</span><span class="token punctuation">)</span> <span class="token punctuation">(</span>sum <span class="token builtin">int</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    sum <span class="token operator">=</span> a <span class="token operator">+</span> b <span class="token comment">// 这里把 a+b 的结果赋给了命名返回变量 sum</span>
    <span class="token keyword">return</span>      <span class="token comment">// 隐式返回 sum，即返回 a+b 的结果</span>
<span class="token punctuation">}</span>
</code></pre>
      <p>
       在这个函数中，我们声明了返回值变量
       <code>
        sum
       </code>
       。在函数体内，我们将
       <code>
        a+b
       </code>
       的值赋给
       <code>
        sum
       </code>
       ，然后直接使用
       <code>
        return
       </code>
       语句返回。这里的
       <code>
        return
       </code>
       没有跟随任何值，但 Go 会自动把
       <code>
        sum
       </code>
       的当前值作为函数的返回值返回给调用者。
      </p>
      <p>
       这种方式的好处是：
      </p>
      <ul>
       <li>
        <strong>
         简洁
        </strong>
        ：在函数的最后可以直接写
        <code>
         return
        </code>
        ，不必重复写返回变量。
       </li>
       <li>
        <strong>
         可读性
        </strong>
        ：当返回值具有名称时，函数的返回含义更加明确。
       </li>
      </ul>
      <p>
       需要注意的是，如果在函数中使用了命名返回值，那么在函数结束时，无论是通过
       <code>
        return
       </code>
       语句还是到达函数末尾，Go 都会使用这些命名返回值的当前值作为最终的返回结果。这就是所谓“返回值会隐式赋值给结果变量”的含义。
      </p>
     </li>
    </ol>
