    <h3>
     <a id="_0">
     </a>
     目标
    </h3>
    <p>
     学习Go 的编译方式、包的组织方式以及工具链的统一调用方式
    </p>
    <h3>
     <a id="_4">
     </a>
     概念与定义
    </h3>
    <ol>
     <li>
      package
      <ul>
       <li>
        Go 语言通过包来组织代码。包类似于其它语言的库librarries或模块modules，每个包通常对应一个目录，目录中的所有 .go 文件都属于同一个包。
       </li>
       <li>
        <strong>
         特殊的 main 包 :
        </strong>
        <br/>
        当代码使用
        <code>
         package main
        </code>
        声明时，表示这是一个可独立执行的程序而非一个库。程序的执行入口就是
        <code>
         main
        </code>
        函数
       </li>
      </ul>
     </li>
     <li>
      import
      <ul>
       <li>
        通过 import 语句，编译器能够识别代码中所需的外部依赖，若导入错误或缺少必要包，程序将无法编译。这种机制保证了代码的整洁和依赖的明确性
       </li>
      </ul>
     </li>
     <li>
      主函数 func main
      <ul>
       <li>
        <code>
         func main()
        </code>
        是程序的执行入口。所有可执行程序都必须包含一个 main 函数。一般情况下，main 函数会调用其它函数和包中的功能来实现实际业务逻辑
       </li>
      </ul>
     </li>
     <li>
      编译、运行与工具链：Go 语言是
      <strong>
       编译型语言
      </strong>
      ，它会将源码编译为静态链接的二进制可执行文件。
     </li>
     <li>
      Go 语言的工具链将源代码及其依赖转换成计算机的机器指令（译注：静态编译）。Go 语言提供的工具都通过一个单独的命令
      <code>
       go
      </code>
      调用，
      <code>
       go
      </code>
      命令有一系列子命令。
      <ul>
       <li>
        go run：临时编译与执行
        <ul>
         <li>
          编译一个或多个以。
          <code>
           .go
          </code>
          结尾的源文件，链接库文件，并运行最终生成的可执行文件，适合临时实验和调试。
         </li>
        </ul>
       </li>
       <li>
        go build：编译生成可执行二进制文件
        <ul>
         <li>
          在 Linux 或 macOS 下，生成的文件名与源码文件相同；在 Windows 下则会加上
          <code>
           .exe
          </code>
          后缀。
         </li>
         <li>
          由于是静态编译，
          <strong>
           生成的可执行文件包含了所有依赖的代码（go程序可以独立运行）
          </strong>
          ，因此运行时无需担心系统库更新造成冲突或环境依赖问题，便于分发和部署。
         </li>
        </ul>
       </li>
       <li>
        源码获取：go get。
        <br/>
        可以自动下载源码到
        <code>
         $GOPATH/src/...
        </code>
        目录。需要先安装版本管理工具（如 Git 或 Hg）并设置好 GOPATH 环境变量。
       </li>
      </ul>
     </li>
     <li>
      自动插入分号：编译器会根据特定符号后的换行符自动插入分号。这要求开发者在书写代码时注意换行位置，否则可能导致解析错误。
      <ul>
       <li>
        函数的左大括号
        <code>
         {
         <!-- -->
        </code>
        必须与函数声明在同一行上。
       </li>
       <li>
        在表达式中，换行位置也有讲究：例如，在运算符
        <code>
         +
        </code>
        后换行通常不会有问题，而在运算符
        <code>
         +
        </code>
        前换行则可能被错误地插入分号。
       </li>
      </ul>
     </li>
     <li>
      <code>
       gofmt
      </code>
      工具会自动将代码格式化成统一的风格
      <ul>
       <li>
        许多文本编辑器可以配置在保存文件时自动运行
        <code>
         gofmt
        </code>
        ，确保每次提交的代码都符合标准格式。
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     <a id="_29">
     </a>
     要点（案例）
    </h3>
    <pre><code class="prism language-go"><span class="token keyword">package</span> main

<span class="token keyword">import</span> <span class="token string">"fmt"</span>

<span class="token keyword">func</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">{<!-- --></span>
	fmt<span class="token punctuation">.</span><span class="token function">Println</span><span class="token punctuation">(</span><span class="token string">"hello, world"</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     使用
     <code>
      go run helloworld.go
     </code>
     命令可以编译并立即运行该程序
    </p>
    <p>
     运行
     <code>
      go build helloworld.go
     </code>
     会编译生成一个可执行二进制文件。
    </p>
    <p>
     示例代码仓库托管在 gopl.io 上。通过
     <code>
      go get gopl.io/ch1/helloworld
     </code>
     命令，可以自动下载源码到
     <code>
      $GOPATH/src/gopl.io/ch1/helloworld
     </code>
     目录。需要先安装版本管理工具（如 Git 或 Hg）并设置好 GOPATH 环境变量。
    </p>
    <p>
     文件中 package 声明必须位于最上方，紧随其后的是 import 声明。之后是函数func、变量var、常量const和类型type的声明。虽然这些声明的顺序不影响编译，但规范化的顺序有助于代码的可读性和维护性。
    </p>
    <p>
     函数的定义由
     <code>
      func
     </code>
     关键字开始，后面跟上函数名、参数列表、返回值列表和函数体。
    </p>
