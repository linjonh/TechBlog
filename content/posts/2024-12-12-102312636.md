    <h4>
     数据模型:（Data model）
    </h4>
    <p style="text-indent:33px;">
     数据模型是数据库结构的基础，是用来描述数据的一组概念和定义，数据模型主要有
     <span style="color:#f33b45;">
      <strong>
       三个要素
      </strong>
     </span>
     ：数据结构，数据操作，数据约束条件。
    </p>
    <blockquote>
     <p style="text-indent:0;">
      <span style="color:#ffbb66;">
       数据结构
      </span>
      <span style="color:#86ca5e;">
       (表里面的字段)
      </span>
      <span style="color:#ffbb66;">
       :
      </span>
     </p>
     <p style="text-indent:33px;">
      对象类型的集合，是对静态属性的描述。
      <br/>
      <span style="color:#ffbb66;">
       数据操作
      </span>
      <span style="color:#86ca5e;">
       (表里面的数据)
      </span>
      <span style="color:#ffbb66;">
       :
      </span>
     </p>
     <p style="text-indent:33px;">
      是对数据库中的各种对象性数据，允许执行的操作的集合，如增删改查等；数据操作是对系统动态热性的描述。
      <br/>
      <span style="color:#ffbb66;">
       数据约束条件
      </span>
      <span style="color:#86ca5e;">
       (表里边的约束)
      </span>
      <span style="color:#ffbb66;">
       :
      </span>
     </p>
     <p style="text-indent:33px;">
      <span style="color:#ffbb66;">
      </span>
      是一组完整性规则的集合，也就是说，对于具体的应用必须先遵循特定的语义约束条件。 比如：性别只能取"男"或者"女"中的之一。考试成绩：（满分100）只能是0-100的数值。
      <br/>
     </p>
     <p style="text-indent:33px;">
      <span style="color:#f33b45;">
       总结: 创建一张数据表, 这张表的整体内容(字段, 内容, 约束)就是一个数据模型.
      </span>
     </p>
    </blockquote>
    <h4>
     数据库:
    </h4>
    <p style="text-indent:33px;">
     数据库（database）是长期存储在计算机外存上有结构，可共享的数据集合；数据库中的数据按照一定的数据模型描述、 组织和存储，具有较小的冗余度，较高的数据独立性和可扩展性，并可以为多个用户共享。
    </p>
    <p>
     <strong>
      常见数据库分类：
     </strong>
    </p>
    <blockquote>
     <p>
      <span style="color:#ffbb66;">
       关系型数据库：
      </span>
      MySQL，oracle，sqlserver
     </p>
     <p>
      <span style="color:#ffbb66;">
       非关系型数据库：
      </span>
      redis，hbase等
     </p>
    </blockquote>
    <h4>
     数据库管理系统:（DBMS, DataBase Managerment System）
    </h4>
    <p style="text-indent:33px;">
     是指数据库系统中对数据库进行管理的软件系统，是数据库系统的核心组成部分。数据库的一切操作，如增删改查以及各种控制，都是通过DBMS进行的。
    </p>
    <p style="text-indent:0;">
     <span style="color:#f33b45;">
      具有以下4个基本的功能
     </span>
     ：
    </p>
    <blockquote>
     <p style="text-indent:0;">
      <span style="color:#ffbb66;">
       1) 数据定义功能:
      </span>
     </p>
     <p style="text-indent:33px;">
      用户可以通过DBMS提供的数据定义语言对数据库的数据进行定义。
      <br/>
      <span style="color:#ffbb66;">
       2) 数据操纵功能:
      </span>
     </p>
     <p style="text-indent:33px;">
      用户可以通过数据操纵语言实现对数据库的增删改查操作
      <br/>
      <span style="color:#ffbb66;">
       3) 数据库运行管理:
      </span>
     </p>
     <p style="text-indent:33px;">
      管理数据库的运行是DBMS运行时的核心工作。所有访问数据库的操作都要在DBMS的统一管理下进行，以保证数据的安全性、完整性、一致性以及多用户对数据库的并发使用。
      <br/>
      <span style="color:#ffbb66;">
       4) 数据库的建立和维护:
      </span>
     </p>
     <p style="text-indent:33px;">
      建立数据库，包括数据库初始数据的输入与数据转换等。维护数据库，包括数据库的转储与恢复，数据库的重组织，性能监控和分析。
     </p>
    </blockquote>
    <h4>
     数据库系统相关管理人员:
    </h4>
    <p style="text-indent:33px;">
     概述:  数据库系统的相关人员是数据库系统的重要组成部分，具体可以分为以下的三类人员:数据库管理员 ，应用程序开发人员，最终用户。
    </p>
    <blockquote>
     <ul>
      <li>
       数据库管理员  (DBA)
      </li>
     </ul>
     <p>
      职责：负责数据库的建立、使用、维护的专门人员
     </p>
     <ul>
      <li>
       应用程序开发人员  (程序员)
      </li>
     </ul>
     <p>
      职责：开发数据库应用程序的人员，可以使用数据库管理系统的所有功能。
     </p>
     <ul>
      <li>
       最终用户  (普通用户)
      </li>
     </ul>
     <p>
      职责：一般来说，是通过应用程序使用数据库的人员，最终用户无需自己编写应用程序。
     </p>
    </blockquote>
    <p>
    </p>
    <h4>
     数据库系统
    </h4>
    <p style="text-indent:33px;">
     数据库系统（database system DBS）是由硬件系统，数据库管理系统，数据库，数据库应用程序， 数据库系统相关人员构成的人-机系统，是指有数据库的整个计算机系统。
     <span style="color:#86ca5e;">
      注意：在许多场合下，数据库，数据库管理系统，数据库系统不做严格区分。
     </span>
    </p>
