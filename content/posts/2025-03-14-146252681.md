    <h4>
     💻
     <strong>
      Webpack 构建流程及 Loader 和 Plugin 区别
     </strong>
    </h4>
    <p>
     Webpack 是现代前端项目的
     <strong>
      打包工具
     </strong>
     ，通过模块化的方式将多个文件打包成一个或多个文件。Webpack 的强大之处在于其灵活性和插件机制。
    </p>
    <hr/>
    <h3>
     🏗️
     <strong>
      一、Webpack 构建流程
     </strong>
    </h3>
    <p>
     Webpack 的整个构建流程大致可以分为以下几个阶段：
    </p>
    <ol>
     <li>
      <strong>
       初始化
      </strong>
     </li>
     <li>
      <strong>
       编译
      </strong>
     </li>
     <li>
      <strong>
       构建模块
      </strong>
     </li>
     <li>
      <strong>
       模块打包
      </strong>
     </li>
     <li>
      <strong>
       输出文件
      </strong>
     </li>
    </ol>
    <hr/>
    <h4>
     🚀
     <strong>
      1. 初始化阶段
     </strong>
    </h4>
    <ul>
     <li>
      读取配置文件（如
      <code>
       webpack.config.js
      </code>
      ），合并默认配置和自定义配置。
     </li>
     <li>
      创建
      <code>
       Compiler
      </code>
      对象，加载所有配置的
      <strong>
       插件（Plugin）
      </strong>
      。
     </li>
    </ul>
    <hr/>
    <h4>
     ⚙️
     <strong>
      2. 编译阶段
     </strong>
    </h4>
    <ul>
     <li>
      从
      <strong>
       入口文件
      </strong>
      开始，根据模块导入关系，递归地解析每个模块。
     </li>
     <li>
      通过
      <strong>
       Loader
      </strong>
      对模块进行转换，如转译
      <code>
       ES6
      </code>
      、处理
      <code>
       SCSS
      </code>
      、图片等。
     </li>
    </ul>
    <hr/>
    <h4>
     🗃️
     <strong>
      3. 构建模块
     </strong>
    </h4>
    <ul>
     <li>
      使用配置的
      <strong>
       Loader
      </strong>
      对文件进行转换，生成标准的
      <strong>
       模块对象
      </strong>
      。
     </li>
     <li>
      依赖模块进行递归构建，形成依赖树。
     </li>
    </ul>
    <hr/>
    <h4>
     📦
     <strong>
      4. 打包阶段
     </strong>
    </h4>
    <ul>
     <li>
      将所有模块按照依赖关系打包到一个或多个文件中。
     </li>
     <li>
      生成浏览器可识别的代码块（Chunk）。
     </li>
    </ul>
    <hr/>
    <h4>
     📝
     <strong>
      5. 输出阶段
     </strong>
    </h4>
    <ul>
     <li>
      根据配置输出打包后的文件，通常是**
      <code>
       dist
      </code>
      目录**。
     </li>
     <li>
      执行
      <strong>
       优化操作
      </strong>
      （如压缩、代码分割等）。
     </li>
    </ul>
    <hr/>
    <h3>
     🔄
     <strong>
      二、Loader 和 Plugin 的区别
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        方面
       </th>
       <th>
        Loader
       </th>
       <th>
        Plugin
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        作用
       </td>
       <td>
        <strong>
         文件加载与转换
        </strong>
       </td>
       <td>
        <strong>
         扩展 Webpack 构建功能
        </strong>
       </td>
      </tr>
      <tr>
       <td>
        处理范围
       </td>
       <td>
        <strong>
         单个文件
        </strong>
        （模块）
       </td>
       <td>
        <strong>
         整个构建流程
        </strong>
       </td>
      </tr>
      <tr>
       <td>
        使用方式
       </td>
       <td>
        <strong>
         module.rules
        </strong>
        中配置
       </td>
       <td>
        <strong>
         plugins
        </strong>
        中配置
       </td>
      </tr>
      <tr>
       <td>
        执行阶段
       </td>
       <td>
        <strong>
         模块加载
        </strong>
        阶段
       </td>
       <td>
        <strong>
         构建和打包
        </strong>
        的任意阶段
       </td>
      </tr>
      <tr>
       <td>
        典型场景
       </td>
       <td>
        <code>
         babel-loader
        </code>
        、
        <code>
         css-loader
        </code>
        、
        <code>
         file-loader
        </code>
       </td>
       <td>
        <code>
         HtmlWebpackPlugin
        </code>
        、
        <code>
         CleanWebpackPlugin
        </code>
        、
        <code>
         DefinePlugin
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     🔍
     <strong>
      1. Loader 详解
     </strong>
    </h4>
    <p>
     <strong>
      作用：
     </strong>
    </p>
    <ul>
     <li>
      主要用于将
      <strong>
       非 JS 文件
      </strong>
      转换为可被 Webpack 处理的模块。
     </li>
     <li>
      例如：把 SCSS 编译成 CSS、把 TypeScript 转换成 JS。
     </li>
    </ul>
    <p>
     <strong>
      配置示例：
     </strong>
    </p>
    <pre><code>module.exports = {
    module: {
        rules: [
            {
                test: /\.scss$/,      // 匹配 .scss 文件
                use: [
                    'style-loader',   // 将 CSS 以 &lt;style&gt; 标签插入到 DOM 中
                    'css-loader',     // 处理 @import 和 url()
                    'sass-loader'     // 将 SCSS 转换为 CSS
                ]
            }
        ]
    }
};
</code></pre>
    <hr/>
    <h4>
     🛠️
     <strong>
      2. Plugin 详解
     </strong>
    </h4>
    <p>
     <strong>
      作用：
     </strong>
    </p>
    <ul>
     <li>
      扩展 Webpack 功能，如
      <strong>
       文件压缩
      </strong>
      、
      <strong>
       注入环境变量
      </strong>
      、
      <strong>
       生成 HTML
      </strong>
      等。
     </li>
    </ul>
    <p>
     <strong>
      配置示例：
     </strong>
    </p>
    <pre><code>const HtmlWebpackPlugin = require('html-webpack-plugin');
const { CleanWebpackPlugin } = require('clean-webpack-plugin');

module.exports = {
    plugins: [
        new CleanWebpackPlugin(),              // 清理输出目录
        new HtmlWebpackPlugin({                // 生成 HTML 文件
            template: './src/index.html',
            filename: 'index.html'
        })
    ]
};
</code></pre>
    <hr/>
    <h3>
     💡
     <strong>
      三、Loader 和 Plugin 的区别详解
     </strong>
    </h3>
    <h4>
     <strong>
      1. 执行阶段不同
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Loader
      </strong>
      ：在
      <strong>
       模块加载
      </strong>
      时对文件进行
      <strong>
       预处理
      </strong>
      。
     </li>
     <li>
      <strong>
       Plugin
      </strong>
      ：在
      <strong>
       整个构建生命周期
      </strong>
      的任意阶段进行扩展。
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      2. 关注点不同
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Loader
      </strong>
      ：
      <strong>
       文件转换
      </strong>
      ，例如：
      <ul>
       <li>
        <strong>
         Babel
        </strong>
        将 ES6 转换为 ES5
       </li>
       <li>
        <strong>
         CSS 预处理器
        </strong>
        （如 SCSS、LESS）转换为 CSS
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Plugin
      </strong>
      ：
      <strong>
       流程控制和功能扩展
      </strong>
      ，例如：
      <ul>
       <li>
        <strong>
         HtmlWebpackPlugin
        </strong>
        自动生成 HTML 文件
       </li>
       <li>
        <strong>
         CleanWebpackPlugin
        </strong>
        清理输出目录
       </li>
       <li>
        <strong>
         MiniCssExtractPlugin
        </strong>
        抽取 CSS
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      3. 使用方式不同
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Loader
      </strong>
      ：通过
      <code>
       module.rules
      </code>
      进行配置，通常使用链式调用。
     </li>
     <li>
      <strong>
       Plugin
      </strong>
      ：通过
      <code>
       plugins
      </code>
      进行配置，实例化后传入构造函数。
     </li>
    </ul>
    <hr/>
    <h3>
     📝
     <strong>
      四、实际场景对比
     </strong>
    </h3>
    <h4>
     <strong>
      1. Loader 使用场景：
     </strong>
    </h4>
    <ul>
     <li>
      处理
      <strong>
       样式文件
      </strong>
      ：
      <code>
       sass-loader
      </code>
      、
      <code>
       css-loader
      </code>
      、
      <code>
       style-loader
      </code>
      。
     </li>
     <li>
      处理
      <strong>
       图片文件
      </strong>
      ：
      <code>
       file-loader
      </code>
      、
      <code>
       url-loader
      </code>
      。
     </li>
     <li>
      处理
      <strong>
       JS 转换
      </strong>
      ：
      <code>
       babel-loader
      </code>
      、
      <code>
       ts-loader
      </code>
      。
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      2. Plugin 使用场景：
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       优化打包体积
      </strong>
      ：
      <code>
       TerserWebpackPlugin
      </code>
      压缩 JS、
      <code>
       MiniCssExtractPlugin
      </code>
      抽离 CSS。
     </li>
     <li>
      <strong>
       自动生成 HTML
      </strong>
      ：
      <code>
       HtmlWebpackPlugin
      </code>
      。
     </li>
     <li>
      <strong>
       清理构建目录
      </strong>
      ：
      <code>
       CleanWebpackPlugin
      </code>
      。
     </li>
     <li>
      <strong>
       环境变量注入
      </strong>
      ：
      <code>
       DefinePlugin
      </code>
      。
     </li>
    </ul>
    <hr/>
    <h3>
     🔥
     <strong>
      五、面试要点总结
     </strong>
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        Webpack 构建流程：
       </strong>
      </p>
      <ul>
       <li>
        初始化、编译、模块构建、打包和输出。
       </li>
       <li>
        重点理解
        <strong>
         多入口、多输出和模块依赖分析
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        Loader 和 Plugin 的区别：
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         Loader 是文件转换器
        </strong>
        ，在模块加载时处理文件。
       </li>
       <li>
        <strong>
         Plugin 是功能扩展器
        </strong>
        ，在整个构建生命周期中发挥作用。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        常见面试题：
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         Loader 和 Plugin 有什么区别？
        </strong>
        <blockquote>
         <p>
          Loader 负责
          <strong>
           文件转换
          </strong>
          ，Plugin 负责
          <strong>
           功能扩展
          </strong>
          。
         </p>
        </blockquote>
       </li>
       <li>
        <strong>
         Webpack 如何进行性能优化？
        </strong>
        <blockquote>
         <p>
          使用
          <strong>
           代码拆分
          </strong>
          、
          <strong>
           按需加载
          </strong>
          、
          <strong>
           持久化缓存
          </strong>
          和
          <strong>
           Tree Shaking
          </strong>
          。
         </p>
        </blockquote>
       </li>
       <li>
        <strong>
         如何配置多入口打包？
        </strong>
        <blockquote>
         <p>
          使用
          <code>
           entry
          </code>
          字段指定多个入口点，并在
          <code>
           output
          </code>
          中使用占位符。
         </p>
        </blockquote>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     🎯
     <strong>
      总结
     </strong>
    </h3>
    <ol>
     <li>
      <strong>
       Loader
      </strong>
      解决
      <strong>
       文件转换
      </strong>
      ，如 SCSS -&gt; CSS、ES6 -&gt; ES5。
     </li>
     <li>
      <strong>
       Plugin
      </strong>
      实现
      <strong>
       功能增强
      </strong>
      ，如清理目录、打包优化、注入变量。
     </li>
     <li>
      <strong>
       构建流程
      </strong>
      ：
      <strong>
       初始化
      </strong>
      -&gt;
      <strong>
       编译
      </strong>
      -&gt;
      <strong>
       模块构建
      </strong>
      -&gt;
      <strong>
       打包
      </strong>
      -&gt;
      <strong>
       输出
      </strong>
      。
     </li>
     <li>
      <strong>
       性能优化
      </strong>
      ：
      <strong>
       多入口
      </strong>
      配置、
      <strong>
       代码拆分
      </strong>
      、
      <strong>
       持久化缓存
      </strong>
      、
      <strong>
       Tree Shaking
      </strong>
      。
     </li>
    </ol>
    <p>
     希望这份讲解能帮助你彻底理解 Webpack 的构建流程和 Loader、Plugin 的区别！
    </p>
