    <h3>
     <a id="_82">
     </a>
     安装过程中遇到的错误
    </h3>
    <h4>
     <a id="ORACLE_HOME_TNS_ADMIN_83">
     </a>
     安装过其他版本数据库，设置了ORACLE_HOME或 TNS_ADMIN
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ef281c121492454193760140df004848.png"/>
     <br/>
     根据如下安装文档，安装23ai时候，如果设置ORACLE_HOME或 TNS_ADMIN，需要删除。
     <br/>
     https://docs.oracle.com/en/database/oracle/oracle-database/23/xeinw/installing-oracle-database-xe.html#GUID-7BF9ACDC-1281-494C-AF96-77A25B1BD67D
    </p>
    <blockquote>
     <p>
      If the ORACLE_HOME or TNS_ADMIN environment variables have been set, then delete them.
     </p>
    </blockquote>
    <h5>
     <a id="_89">
     </a>
     解决方法
    </h5>
    <pre><code class="prism language-clike">修改Windows环境变量

打开环境变量设置界面：在桌面上右键“我的电脑”，并在弹出的右键菜单中选择“属性”。在弹出的设置界面中，点击“高级系统设置”。

修改环境变量PATH：在弹出的系统属性界面中先后选择“高级”、“环境变量”。
删除ORACLE_HOME 变量
</code></pre>
    <h4>
     <a id="Windows_Installer_Serviece_error_1719_100">
     </a>
     无法访问Windows Installer Serviece (error 1719)
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9a9f0718441c41199a0949825c203271.png"/>
     <br/>
     错误 1719 通常与 Windows Installer 服务有关，而不是直接与 Oracle 数据库相关。
    </p>
    <h5>
     <a id="_104">
     </a>
     解决方法
    </h5>
    <pre><code class="prism language-clike">启用 Windows Installer 服务

按 Win <span class="token operator">+</span> R 打开运行窗口。
输入 services<span class="token punctuation">.</span>msc 并回车。
在服务列表中找到 Windows Installer 服务。
右键点击该服务，选择“属性”。
将启动类型设置为“自动”，然后点击“启动”按钮。
点击“确定”保存更改。
</code></pre>
    <h4>
     <a id="_116">
     </a>
     其他注意
    </h4>
    <p>
     1.安装Oracle 23ai时，需要确保安装目录具有适当的用户权限。
     <br/>
     2.如果防火墙阻止了Oracle服务的访问，请确保在防火墙设置中允许Oracle服务的通信。
    </p>
