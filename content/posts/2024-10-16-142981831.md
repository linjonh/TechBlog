    <h2>
     <a id="_5">
     </a>
     前言
    </h2>
    <p>
     按照gpt给的流程安装docker,到
     <code>
      docker run hello-world
     </code>
     时一直失败，试了一下直接拉取docker资源
     <code>
      docker pull xxx
     </code>
     还是不行，看了一下别的帖子都说需要换源，但很多docker源都失效了，为了方便后续自己再安装换源记录一下，只想看怎么换源的可以直接跳到第2部分
    </p>
    <hr/>
    <h2>
     <a id="docker__9">
     </a>
     一、docker 安装
    </h2>
    <p>
     以下步骤由gpt生成，亲测步骤10之前都没问题
     <br/>
     1: 更新系统包
     <br/>
     首先，打开终端并更新你的包索引：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> upgrade <span class="token parameter variable">-y</span>
</code></pre>
    <p>
     2: 安装必要的依赖
    </p>
    <p>
     安装一些必要的包，以便让
     <code>
      apt
     </code>
     能够通过 HTTPS 使用仓库：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> apt-transport-https ca-certificates <span class="token function">curl</span> software-properties-common <span class="token parameter variable">-y</span>
</code></pre>
    <p>
     3: 添加 Docker 的官方 GPG 密钥
    </p>
    <p>
     运行以下命令来添加 Docker 的 GPG 密钥：
    </p>
    <pre><code class="prism language-bash"><span class="token function">curl</span> <span class="token parameter variable">-fsSL</span> https://download.docker.com/linux/ubuntu/gpg <span class="token operator">|</span> <span class="token function">sudo</span> apt-key <span class="token function">add</span> -
</code></pre>
    <p>
     4: 添加 Docker 的 APT 仓库
    </p>
    <p>
     添加 Docker 的稳定版仓库：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> add-apt-repository <span class="token string">"deb [arch=amd64] https://download.docker.com/linux/ubuntu <span class="token variable"><span class="token variable">$(</span>lsb_release <span class="token parameter variable">-cs</span><span class="token variable">)</span></span> stable"</span>
</code></pre>
    <p>
     5: 更新包索引
    </p>
    <p>
     再次更新包索引以包含 Docker 的软件包：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
</code></pre>
    <p>
     6: 安装 Docker
    </p>
    <p>
     现在，你可以安装 Docker：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> docker-ce <span class="token parameter variable">-y</span>
</code></pre>
    <p>
     7: 启动 Docker 服务
    </p>
    <p>
     确保 Docker 服务正在运行：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl start <span class="token function">docker</span>
<span class="token function">sudo</span> systemctl <span class="token builtin class-name">enable</span> <span class="token function">docker</span>
</code></pre>
    <p>
     检查docker状态:
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl status <span class="token function">docker</span>
</code></pre>
    <p>
     8: 验证 Docker 是否安装成功
    </p>
    <p>
     你可以通过以下命令检查 Docker 是否安装成功：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">docker</span> <span class="token parameter variable">--version</span>
</code></pre>
    <p>
     9: 以非 root 用户运行 Docker（可选）
    </p>
    <p>
     如果你希望以非 root 用户运行 Docker，可以将当前用户添加到
     <code>
      docker
     </code>
     组：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">usermod</span> <span class="token parameter variable">-aG</span> <span class="token function">docker</span> <span class="token environment constant">$USER</span>
</code></pre>
    <p>
     使用以下命令立即应用更改：
    </p>
    <pre><code class="prism language-bash">newgrp <span class="token function">docker</span>
</code></pre>
    <p>
     <em>
      <strong>
       ！！！！！注意：这种方式单次有效，下次再运行docker还是必须用sudo
       <br/>
       想永久有效的方式是在输入
       <code>
        sudo usermod -aG docker $USER
       </code>
       指令后立即输入
       <code>
        sudo reboot
       </code>
       对系统进行重启！！！
      </strong>
     </em>
     <br/>
     重启之后打开终端验证是否设置成功,输出中应当包含docker：
    </p>
    <pre><code class="prism language-python">groups $USER
</code></pre>
    <p>
     10: 测试 Docker
    </p>
    <p>
     运行以下命令来测试 Docker 是否工作正常：
    </p>
    <pre><code class="prism language-bash"><span class="token function">docker</span> run hello-world
</code></pre>

    <h2>
     <a id="_docker_pull_timeout_111">
     </a>
     二 、docker pull 失败，提示timeout
    </h2>
    <p>
     <strong>
      <code>
       主要是网络问题，需要换源
      </code>
     </strong>
    </p>
    <h4>
     <a id="1_etcdockerdaemonjson_114">
     </a>
     1. 打开/新建/etc/docker/daemon.json文件
    </h4>
    <pre><code class="prism language-powershell">sudo nano <span class="token operator">/</span>etc/docker/daemon<span class="token punctuation">.</span>json 
</code></pre>
    <h4>
     <a id="2_docker_120">
     </a>
     2. 将最新的docker源地址粘贴进去，并将更改后的文件保存
    </h4>
    <pre><code class="prism language-powershell"> <span class="token punctuation">{<!-- --></span> <span class="token string">"registry-mirrors"</span>: <span class="token punctuation">[</span> <span class="token string">"https://dockerpull.com"</span> <span class="token punctuation">]</span> <span class="token punctuation">}</span>
</code></pre>
    <p>
     Ctrl + O保存，Enter确认保存，Ctrl + X退出nano
    </p>
    <p>
     <em>
      <strong>
       <code>
        注意：由于docker源经常更新，请自行检索确认可用网址，下面的github链接是比较全面的docker源网址汇总，可以在这里面找。
       </code>
      </strong>
     </em>
     <a href="https://github.com/dongyubin/DockerHub">
      https://github.com/dongyubin/DockerHub
     </a>
    </p>
    <h4>
     <a id="3_docker_129">
     </a>
     3. 应用更改并重启docker
    </h4>
    <pre><code class="prism language-powershell">sudo systemctl daemon-reload <span class="token comment">#重载systemd管理守护进程配置文件</span>
sudo systemctl restart docker <span class="token comment">#重启 Docker 服务</span>
</code></pre>
    <h4>
     <a id="4_136">
     </a>
     4.重新测试
    </h4>
    <pre><code class="prism language-powershell">docker run hello-world
</code></pre>

     得到如下结果，大功告成！！！

    <h2>
     <a id="docker_149">
     </a>
     三、构建docker存储空间不足
    </h2>
    <p>
     运行完上述测试后在
     <code>
      docker build xxx
     </code>
     的时候报错：
     <code>
      InvalidArchiveError,No space left on device
     </code>
     ,提示存储空间不足
    </p>
    <p>
     首先检查磁盘空间使用情况
    </p>
    <pre><code class="prism language-powershell">df <span class="token operator">-</span>h
</code></pre>
    <p>
     得到如下结果：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ee9b9970d7394944916d7338380551a9.png">
      <br/>
      docker默认情况下会占用根分区的存储空间，也就是上图
      <code>
       /dev/nvme0n1p10
      </code>
      ，显示剩余8.9G，空间不足，两种解决方法。
     </img>
    </p>
    <h4>
     <a id="1_Docker__159">
     </a>
     1. 清理Docker 镜像和容器
    </h4>
    <pre><code class="prism language-powershell">docker system prune <span class="token operator">-</span>a  <span class="token comment"># 删除所有未使用的镜像、悬挂的卷和停止的容器</span>
</code></pre>
    <h4>
     <a id="2__Docker__home__165">
     </a>
     2. 将 Docker 存储目录移动到 /home 分区
    </h4>
    <p>
     编辑/etc/docker/daemon.json文件，添加如下内容：
    </p>
    <pre><code class="prism language-powershell"><span class="token punctuation">{<!-- --></span>
  <span class="token string">"data-root"</span>: <span class="token string">"/home/docker-data"</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     最终该文件内容如下：
    </p>
    <pre><code class="prism language-powershell"> <span class="token punctuation">{<!-- --></span> 
 <span class="token string">"registry-mirrors"</span>: <span class="token punctuation">[</span> <span class="token string">"https://dockerpull.com"</span> <span class="token punctuation">]</span> <span class="token punctuation">,</span>
 <span class="token string">"data-root"</span>: <span class="token string">"/home/docker-data"</span>
 <span class="token punctuation">}</span>
</code></pre>
    <p>
     像第二部分1～3步一样保存更改并重启docker，再重新构建你所需的docker
    </p>
    <pre><code class="prism language-powershell">docker bulid xxx
</code></pre>

    <h2>
     <a id="_189">
     </a>
     四、运行容器
    </h2>
    <h4>
     <a id="1_docker__191">
     </a>
     1. 查看刚刚构建好的docker 镜像
    </h4>
    <pre><code class="prism language-powershell">docker images
</code></pre>
    <p>
     得到结果如下表示构建成功：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/854f253efc0b49a7913d95d909bda554.png"/>
    </p>
    <h4>
     <a id="2__198">
     </a>
     2. 用构建的镜像来启动一个新的容器
    </h4>
    <pre><code class="prism language-powershell">docker run <span class="token operator">--</span>gpus all <span class="token operator">-</span>it <span class="token operator">-</span>v <span class="token operator">/</span>home/cslearner/PycharmProjects/tdmpc2-main:<span class="token operator">/</span>app cslearner/tdmpc2:1<span class="token punctuation">.</span>0<span class="token punctuation">.</span>0 <span class="token operator">/</span>bin/bash  
<span class="token comment"># 将cslearner/tdmpc2:1.0.0 替换成你的镜像名+TAG</span>
</code></pre>
    <p>
     <code>
      /home/PycharmProjects/tdmpc2-main:/app
     </code>
     表示将主机上/home/PycharmProjects/tdmpc2-main目录的内容挂载到容器/app 目录
     <br/>
     不用GPU的话可以将
     <code>
      --gpus all
     </code>
     去掉，还有就是/bin/bash表示进入容器的交互式终端，可以执行命令及脚本文件。
     <br/>
     但是这步加了gpu之后报错了，如下：
    </p>
    <pre><code class="prism language-powershell">docker: Error response <span class="token keyword">from</span> daemon: could not <span class="token function">select</span> device driver <span class="token string">""</span> with capabilities: <span class="token punctuation">[</span><span class="token namespace">[gpu]</span><span class="token punctuation">]</span><span class="token punctuation">.</span>
</code></pre>
    <p>
     原因是没装
     <code>
      NVIDIA Container Toolkit
     </code>
    </p>
    <ul>
     <li>
      添加添加 NVIDIA 的 GPG 密钥和仓库
     </li>
    </ul>
    <pre><code class="prism language-powershell">distribution=$<span class="token punctuation">(</span><span class="token punctuation">.</span> <span class="token operator">/</span>etc/os-release<span class="token punctuation">;</span><span class="token function">echo</span> <span class="token variable">$ID</span><span class="token variable">$VERSION_ID</span><span class="token punctuation">)</span>
curl <span class="token operator">-</span>s <span class="token operator">-</span>L https:<span class="token operator">/</span><span class="token operator">/</span>nvidia<span class="token punctuation">.</span>github<span class="token punctuation">.</span>io/nvidia-docker/gpgkey <span class="token punctuation">|</span> sudo apt-key add <span class="token operator">-</span>
curl <span class="token operator">-</span>s <span class="token operator">-</span>L https:<span class="token operator">/</span><span class="token operator">/</span>nvidia<span class="token punctuation">.</span>github<span class="token punctuation">.</span>io/nvidia-docker/<span class="token variable">$distribution</span><span class="token operator">/</span>nvidia-docker<span class="token punctuation">.</span>list <span class="token punctuation">|</span> sudo <span class="token function">tee</span> <span class="token operator">/</span>etc/apt/sources<span class="token punctuation">.</span>list<span class="token punctuation">.</span>d/nvidia-docker<span class="token punctuation">.</span>list
</code></pre>
    <p>
     执行上述指令后在终端看到下图，是/etc/apt/sources.list.d/nvidia-docker.list文件中的内容，但是这里生成的网址有些已经弃用了，所以需要修改
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e7cb9afc513d4706a7aae3bf66fe21e7.png">
      <br/>
      打开文件
     </img>
    </p>
    <pre><code class="prism language-powershell">sudo nano <span class="token operator">/</span>etc/apt/sources<span class="token punctuation">.</span>list<span class="token punctuation">.</span>d/nvidia-docker<span class="token punctuation">.</span>list
</code></pre>
    <p>
     修改文件内容如下：
    </p>
    <pre><code class="prism language-powershell">deb https:<span class="token operator">/</span><span class="token operator">/</span>nvidia<span class="token punctuation">.</span>github<span class="token punctuation">.</span>io/libnvidia-container/stable/deb/$<span class="token punctuation">(</span>ARCH<span class="token punctuation">)</span> <span class="token operator">/</span>
<span class="token comment">#deb https://nvidia.github.io/libnvidia-container/experimental/deb/$(ARCH) /</span>
</code></pre>
    <p>
     之后再
    </p>
    <pre><code class="prism language-powershell">sudo apt-get update
</code></pre>
    <p>
     我在sudo apt-get update之后发现还有错误，一个是
     <code>
      W: 目标 CNF (stable/cnf/Commands-all) 在 /etc/apt/sources.list:58 和 /etc/apt/sources.list.d/docker.list:1 中被配置了多次
     </code>
     ，我首先是把sources.list文件里面和docker.list重复的部分注释掉了，然后把docker.list里面的源也换掉，如下:
    </p>
    <pre><code class="prism language-powershell">deb <span class="token namespace">[arch=amd64]</span> https:<span class="token operator">/</span><span class="token operator">/</span>mirrors<span class="token punctuation">.</span>ustc<span class="token punctuation">.</span>edu<span class="token punctuation">.</span>cn/docker-ce/linux/ubuntu focal stable
</code></pre>
    <p>
     之后又报错
     <code>
      http://security.ubuntu.com/ubuntu/dists/focal-security/InRelease 连接失败 [IP: 185.125.190.83 80]
     </code>
     ，我再次打开sources.list，把
     <code>
      http://security.ubuntu.com/ubuntu
     </code>
     全删除了，替换成国内源如下：
    </p>
    <pre><code class="prism language-powershell">deb http:<span class="token operator">/</span><span class="token operator">/</span>mirrors<span class="token punctuation">.</span>ustc<span class="token punctuation">.</span>edu<span class="token punctuation">.</span>cn/ubuntu/ focal-security main restricted
deb http:<span class="token operator">/</span><span class="token operator">/</span>mirrors<span class="token punctuation">.</span>ustc<span class="token punctuation">.</span>edu<span class="token punctuation">.</span>cn/ubuntu/ focal-security universe
deb http:<span class="token operator">/</span><span class="token operator">/</span>mirrors<span class="token punctuation">.</span>ustc<span class="token punctuation">.</span>edu<span class="token punctuation">.</span>cn/ubuntu/ focal-security multiverse
</code></pre>
    <p>
     这个时候再重启一下docker
    </p>
    <pre><code class="prism language-powershell">sudo systemctl restart docker
docker run <span class="token operator">--</span>gpus all <span class="token operator">-</span>it &lt;user&gt;<span class="token operator">/</span>tdmpc2:1<span class="token punctuation">.</span>0<span class="token punctuation">.</span>0 <span class="token operator">/</span>bin/bash
</code></pre>
    <p>
     看到下图，成功！其中root@后面的一串15d32f77ff51是容器的ID
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d1567f0f26b3411a872e085a568ec99c.png"/>
    </p>
    <h4>
     <a id="3_python_259">
     </a>
     3. 检查待执行脚本所需的环境是否安装好(python)
    </h4>
    <pre><code class="prism language-powershell">python <span class="token operator">--</span>version
conda list
</code></pre>
    <p>
     输入conda list 之后又得到报错：
     <code>
      Error while loading conda entry point: conda-libmamba-solver (libarchive.so.19: cannot open shared object file: No such file or directory)
     </code>
     ，原因是conda 缺少了依赖库
     <code>
      libarchive.so.19
     </code>
     ，在docker中执行以下命令来安装：
    </p>
    <pre><code class="prism language-powershell">apt-get update
apt-get install <span class="token operator">-</span>y libarchive-dev
</code></pre>
    <p>
     再次出现
     <code>
      W: Failed to fetch
     </code>
     ，需要换源，在docker中打开/etc/apt/sources.list文件，删除全部，然后替换成：
    </p>
    <pre><code class="prism language-powershell">deb http:<span class="token operator">/</span><span class="token operator">/</span>mirrors<span class="token punctuation">.</span>ustc<span class="token punctuation">.</span>edu<span class="token punctuation">.</span>cn/ubuntu/ focal main restricted universe multiverse
