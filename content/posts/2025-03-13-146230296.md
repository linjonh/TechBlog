       <span style="color:#ff0000">
        <strong>
         自动属性
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     下面两种写法作用相同.前者属于自动属性.在使用自动属性时应该注意:get和set访问器不能有具体的实现.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <img alt="" height="158" src="https://i-blog.csdnimg.cn/direct/ed8a472f4c26414f8a61f65574ffff5a.png" width="584"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812896">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         可空类型(可以为空的值类型)
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     对于
     <span style="color:#c00000">
      值类型
     </span>
     而言,C#规定:在定义变量的同时必须要对其赋初值,否则可能无法通过编译器的编译.但是在某些情况下,用户事先是不知道变量的初始值是多少的.因为可以为空的值类型就显得很重要.
    </p>
    <ol>
     <li style="text-align:justify">
      值类型包括:int\double\float\decimal\bool\char
     </li>
     <li style="text-align:justify">
      可为空的语法为:Nullable&lt;T&gt;,其中T指代上面具体的某种值类型
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     比如,可以为空的int类型:Nullable&lt;int&gt;;可以为空的bool类型:Nullable&lt;bool&gt;
    </p>
    <ol>
     <li style="text-align:justify">
      可以为空的变量,在赋值时,可以将null赋值给它.
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如:Nullable&lt;int&gt;  age  =  null;
    </p>
    <ol>
     <li style="text-align:justify">
      Nullable&lt;T&gt;可以简写为:T?
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如: Nullable&lt;int&gt; age = null;等价于int? age = null;
    </p>
    <ol>
     <li style="text-align:justify">
      下面两种写法都是错误的.
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     string? name = null;  Student? Stu = null;
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     因为string和Student都是引用类型.
    </p>
    <ol>
     <li style="text-align:justify">
      可空值类型的常用属性:
     </li>
    </ol>
    <ol>
     <li style="text-align:justify">
      HasValue:布尔类型,如果字段不为null返回true,否则返回false.
     </li>
     <li style="text-align:justify">
      Value:获取字段的值,如果该字段的值为null,那么调用Value属性会报错
     </li>
    </ol>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812897">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         泛型类
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     List&lt;T&gt;:T可以指代任何类型,比如:int\bool\string\Student\...
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Dictionary&lt;T, K&gt;:字典类,其中T和K可以指代任何类型
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     KeyValuePair&lt;Tkey, Tvalue&gt;:键值对类型
    </p>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812898">

    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     利用对象初始化器,在实例化一个类对象的同时可以为它的属性赋值.
    </p>

     <img alt="" height="164" src="https://i-blog.csdnimg.cn/direct/f7e8c13f5afc4f4fbe4b9a8860a2b5a6.png" width="563"/>

       <span style="color:#ff0000">
        <strong>
         Linq及其扩展
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Linq是一种数据查询语言(它能够从多种数据源中查询数据). 现在基于Linq的扩展有:
    </p>
    <ol>
     <li style="text-align:justify">
      Linq To Object:主要是从内存对象中查询数据
     </li>
     <li style="text-align:justify">

     </li>
     <li style="text-align:justify">
      Linq To XML:主要是从XML片段或XML文档中查询数据
     </li>
    </ol>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812921">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         Linq To Sql简介
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Linq To Sql是一个微软在.NET Framework 3.5中推出的一款ORM(Object  Relation  Model)实体框架.一个好的实体框架除了能够将数据表映射为实体类,还应该支持使用面向的思想去操纵关系型数据库.现在比较流行的ORM框架包括:Linq To Sql\Entity Framework\Nhibernate等.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     我们在使用Linq To Sql的过程中,很少去手写SQL语句就能实现常见的数据增删改差操作.但是它的底层实现仍然是Ado.net.那么如何检测Linq To Sql生成的SQL语句呢?通过DataContext的Log属性即可实现检测功能.
    </p>

     <img alt="" height="297" src="https://i-blog.csdnimg.cn/direct/3f90123c70ae4ed891c6c01293e345ea.png" width="595"/>

      <a name="_Toc398812922">

        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     他们的作用分别是:DataContext类用于和数据库交互(增删改查),每次进行数据库操作时,都应该先创建该类对象;
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     而每一个实体类则和数据表相对应.

     DataContext的特性标记为Database,表示与数据库相关

    <p>
     <img alt="" height="353" src="https://i-blog.csdnimg.cn/direct/9ca5124c8b40482790b511e93aca0655.png" width="604"/>

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     上述两种做法都可以实现功能.做法一是先实现工资模版的新增,然后再新增工资条目.做法二是向工资模版对象中添加多个工资条目,然后向数据库提交工资模版.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     需要注意的是,做法一中为工资条目对象设置了工资模版ID,而做法二并没有为工资条目设置工资模版ID.但为什么最后效果却是一样的呢?

     <strong>
      <a name="_Toc398812931">

     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     分页功能需要使用两个函数进行组合:Skip()和Take()
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Skip(int count):跳过count条数据
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Take(int count):从开始位置提取count条记录
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     分页查询: dataContext.XXX.Skip(pageIndex * pageSize).Take(pageSize)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     pageIndex是要提取的第几页记录.pageIndex &gt;= 0
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     pageSize是每页显示的记录条数
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812932">
      </a>
      <strong>
       <strong>
        场景3:在分层中如何使用Linq To Sql
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Linq To Sql是ADO.NET的一种替代方案,而ADO.NET属于数据层的范畴,因此应该将Linq To Sql添加到数据层.又因为Linq To Sql生成的类既包括了DataContext,又包括了实体类,所以如果要利用Linq To Sql生成的实体类,就需要将这些实体类剪切到实体层中,然后在向DataContext中添加实体层的引用即可.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     当然,我们也可以不将Linq To Sql生成的实体类剪切到实体层中,那么这样我们就需要额外重新编写实体层需要的实体类.
    </p>

     <strong>
      <a name="_Toc398812933">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         其他
        </strong>
       </span>
      </strong>
     </strong>
    </h2>
    <ul>
     <li style="text-align:justify">
      EntitySet&lt;T&gt;:该类表示数据实体的集合
     </li>
     <li style="text-align:justify">
      System.Data.Linq.Table&lt;T&gt;:该类表示数据表对象
     </li>
     <li style="text-align:justify">
      如何判定通过Linq To Sql进行数据的修改\删除\新增是否成功?
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     因为InsertOnSubmit方法和DataContext.SubmitChanges()方法都没有任何返回值,所以无法通过他们判断出是否操作成功.判断的标准就是:如果不抛出异常就成功,抛出异常肯定就失败了.
    </p>
    <ul>
     <li style="text-align:justify">
      对于标识列(自动生成列)而言,在调用DataContext.XXX.InsertOnSubmit()后是无法获取标识列的值的;只有在调用DataContext.SubmitChanges()后方能获取标识列的值.
     </li>
     <li style="text-align:justify">
      在项目中,我们可以利用VS提供的Linq To Sql设计器,快速的生成DataContext和实体类.也可以手写相应的类代码.在这里推荐大家使用VS提供的Linq To Sql设计器,因为它更加方便快捷,基本都是通过拖拽的方式就实现我们想要的效果.除非设计器生成的代码不满足你的要求,我们才会采用第二种方案.

    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812935">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         MVC与ASP.NET MVC基础概念
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     MVC是Model-View-Controller的缩写.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     MVC将应用程序划分为3大组件:模型\视图\控制器.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     MVC不是ASP.NET所特有,它只是一种开发理念.java中的struts2也是一种MVC模型.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ASP.NET MVC从2008年发布1.0版以来,截至2014年ASP.NET MVC最新版本已经是5.0.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ASP.NET MVC从1.0版开始,就已经开放源代码（源码地址：aspnetwebstack.codeplex.com）.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ASP.NET MVC官网地址：http://www.asp.net/mvc
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <a href="https://dotnet.microsoft.com/zh-cn/apps/aspnet/mvc" rel="nofollow" title="ASP.NET MVC 模式 | .NET">
      ASP.NET MVC 模式 | .NET
     </a>
    </p>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812936">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         MVC三大组件的相互关系
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      在控制器中可以直接调用视图和模型
     </li>
     <li style="text-align:justify">
      在视图中可以调用模型.
     </li>
     <li style="text-align:justify">
      模型不能调用视图
     </li>
     <li style="text-align:justify">
      模型能够限定视图中使用的数据,但视图中使用的模型应由控制器提供
     </li>
     <li style="text-align:justify">
      在视图中可以调用控制器（通过视图中表单的提交和点击超链接的方式调用）
     </li>
    </ul>
    <p>
     <img alt="" height="350" src="https://i-blog.csdnimg.cn/direct/6dda8a8b26484f3984cdcf3d00f2f903.png" width="597"/>

      <strong>
       <span style="color:#ff0000">
        <strong>
         ASP.NET Webform模型和ASP.NET MVC模型的关系
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     二者都是基于ASP.NET Web框架构建的开发模型.所以ASP.NET中的一些功能可以被二者公用.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Webform编程模型是典型的以事件驱动的web模型,而MVC则不是.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Webform的URL地址是基于文件系统的,而MVC则是基于Action.
    </p>
    <p>
     <img alt="" height="419" src="https://i-blog.csdnimg.cn/direct/d029f870af784cd4b1b184f940aaf904.png" width="586"/>

    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812938">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         ASP.NET MVC中的约定
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      所有的控制器必须放到Controllers文件夹下
     </li>
     <li style="text-align:justify">
      所有的控制器类名必须以Controller结尾
     </li>
     <li style="text-align:justify">
      所有的模型应该放到Models目录下
     </li>
     <li style="text-align:justify">
      所有的视图文件都应该放到Views目录下.
     </li>
     <li style="text-align:justify">
      所有的控制器类都应该继承自Controller类(本质上是要继承Icontroller接口)
     </li>
     <li style="text-align:justify">
      在控制器类中的public方法被成为Action(行为)
     </li>
     <li style="text-align:justify">
      如果在相应的视图目录中没有找到视图文件,那么会寻找Views\Shared目录下名称相同的视图文件
     </li>
     <li style="text-align:justify">
      在Action中return view(),默认会返回和Action名称一样的视图文件.
     </li>
     <li style="text-align:justify">
      在Global.asax全局应用程序类中注册了默认路由（名称为Default）,默认路由指定了Controller默认为Home,Action默认为Index,参数id为可选参数.所以在URL地址中如果不输入controller默认访问Home控制器;如果不输入action默认访问名称为Index的action.
     </li>
    </ul>
    <ul>
     <li style="text-align:justify">
      http://localhost:54321/   解释：
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     根据默认路由规则，等价于 =&gt; http://localhost:54321/Home/Index
    </p>
    <ul>
     <li style="text-align:justify">
      http://localhost:54321/Home/Index/5  解释：
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     数值5会自动映射到action中名称为id的参数中。
    </p>
    <ul>
     <li style="text-align:justify">
      Http://localhost:54321/Home/Index/5?name=jack&amp;age=20   解释：
     </li>

         控制器相关
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      在Controller类中方法访问级别为public的方法,就是行为(Action).
     </li>
     <li style="text-align:justify">
      如果不希望Controller类中的方法成为Action(可以在地址栏中被访问),有两种实现方式:
     </li>
    </ul>
    <ol>
     <li style="text-align:justify">
      将方法的访问级别设置为private
     </li>
     <li style="text-align:justify">
      在方法上添加特性标记[NonAction]
     </li>
    </ol>

     </li>

       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     页面中如果使用了MVC验证功能，那么在后台对应的Action中必须通过ModelState.IsValid判断是否通过了服务器端验证。只有通过了服务器端验证，才能执行其他的业务逻辑代码。
    </p>
    <p>
     <img alt="" height="232" src="https://i-blog.csdnimg.cn/direct/7cf2e2ef2be2488e83c15db6bba83db9.png" width="420"/>

     <strong>
      使用要点:
     </strong>
    </p>
    <ol>
     <li style="text-align:justify">
      被验证的标记不能是纯html,必须使用相应的html辅助方法
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     例如:Html.TextBoxFor(model=&gt;model.UserName),则验证有效;
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     &lt;input type=”text” id=”UserName” name=”UserName” /&gt;,则验证无效;
    </p>
    <ol>
     <li style="text-align:justify">
      必须使用相应的html辅助方法显示验证消息(Html.ValidationMessage)
     </li>
     <li style="text-align:justify">
      被验证的控件必须放在有html.beginform的辅助方法内部(不能是纯&lt;form&gt;标记).
     </li>
     <li style="text-align:justify">
      客户端验证是不安全的，容易被禁用或欺骗，所以一定要在服务器端判断ModelState.IsValid是否为true.
     </li>
    </ol>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812951">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         Action与View的数据传递
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在某些情况下,视图需要显示多份数据(比如被修改的员工信息\岗位列表\部门列表),但是Action在返回视图时,View()方法只能传递1个object类型的数据,那么其他的数据该如何传递?
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ASP.NET MVC为我们提供了3种方案:ViewData\TempData\ViewBag.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      重要提示：
     </strong>
     在Controller和View中都有这三个属性，在Controller的相应属性中存放了数据后，就可以在View的相应属性中提取这些数据。
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812952">
      </a>
      <strong>
       <strong>
        ViewData
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ViewData是一个字典类型.向里面添加数据时要以键值对的形式添加.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     首先在Action中向ViewData中添加,其次就可以在View中访问ViewData来提取数据.
    </p>
    <p>
     <img alt="" height="647" src="https://i-blog.csdnimg.cn/direct/00c7fcfad2a34faabde4b96ce5e017cd.png" width="623"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812953">
      </a>
      <strong>
       <strong>
        ViewBag
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ViewBag是一个动态类型的属性。在ASP.NET MVC 3.0中才加入了ViewBag属性.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     动态类型:dynamic，就是可以事先不为其定义属性,通过直接给属性赋值的方式就可以随时为它添加新的属性.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     下面的代码演示了向ViewBag中存放cls和stu两个对象，这两个对象分别放到了属性ClassInfo和StudentInfo中。
    </p>
    <p>
     <img alt="" height="631" src="https://i-blog.csdnimg.cn/direct/04b34e0760a14b48b1119764c661f8f7.png" width="621"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812954">
      </a>
      <strong>
       <strong>
        TempData
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     TempData的用法和ViewData很相似，都是字典类型.那他们的区别是什么?
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ViewData中存储的数据仅仅对本次访问的当前action有效.而TempData中存储的数据,除了对本次访问的当前action有效外,还对本次访问的其他action有效（比如在Action1中向TempData中存放了数据，即使从Action1跳转至Action2后，仍然可以在Action2中从TempData提取数据。而ViewData和ViewBag都无法做到这一点）.因此,TempData中数据的生命周期比ViewData|ViewBag中的生命周期要长.
    </p>
    <h3 style="text-align:justify">
     <strong>
      <a name="_Toc398812955">
      </a>
      <strong>
       <span style="color:#ff0000">
        <strong>
         ActionResult的实现类
        </strong>
       </span>
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在ASP.NET MVC模型中，Action用于响应用户的各种请求，比如向客户端返回html文档、html片段、json数据、纯文本、文件等结果。在ASP.NET MVC中通过在Action（行为或操作）方法中返回ActionResult类型的对象来实现向客户端响应上面的各种结果。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ActionResult类是个抽象类(抽象类无法直接被实例化).
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ActionResult有很多实现类，不同的实现类用于响应客户端不同的结果。并且在Controller类中提供了大量的辅助方法,这些辅助方法可以快速的创建各种ActionResult.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在实际开发中，我们更多的是使用Controller类中的辅助方法，而非自己手工创建ActionResult的实现类。
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812956">
      </a>
      <strong>
       <strong>
        ViewResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     ViewResult类的作用是向客户端响应View目录中一个视图文件.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Controller类中提供了View()方法，来快速的返回一个ViewResult类对象.
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812957">
      </a>
      <strong>
       <strong>
        ContentResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     该类用于向客户端返回一段文本内容(纯文本\HTML...).
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Controller类中的辅助方法为:Controller.Content(string  content);
    </p>
    <p>
     <img alt="" height="114" src="https://i-blog.csdnimg.cn/direct/c2fb63ef05684de991a6207c4a2dacf4.png" width="633"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812958">
      </a>
      <strong>
       <strong>
        RedirectToRouteResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     该类的作用是从当前action跳转至其他action.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Controller类中的辅助方法为:RedirectToAction(“ActionName”, “ControllerName”)
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812959">
      </a>
      <strong>
       <strong>
        RedirectResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在服务器端从当前Action跳转到其他URL地址.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Controller类中的辅助方法为:Controller.Redirect(string  url)
    </p>
    <p>
     <img alt="" height="97" src="https://i-blog.csdnimg.cn/direct/29a387be7f0c42db8bb652ab6001d672.png" width="456"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812960">
      </a>
      <strong>
       <strong>
        JsonResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     该类用于向客户端返回一段Json格式的字符串.
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Controller类中的辅助方法为:Controller.Json(object  data);
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       注意：
      </strong>
     </strong>
     默认情况下mvc拒绝响应以get方式发送的ajax请求，需要使用JsonRequestBehavior.AllowGet。
    </p>
    <p>
     <img alt="" height="168" src="https://i-blog.csdnimg.cn/direct/7f2ce6c6b8264022bac54e68a8ca7f2c.png" width="613"/>
    </p>
    <h4 style="text-align:justify">
     <strong>
      <a name="_Toc398812961">
      </a>
      <strong>
       <strong>
        PartialViewResult类
       </strong>
      </strong>
     </strong>
    </h4>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

    </p>
    <ul>
     <li style="text-align:justify">
      Confirm:在发送ajax请求前,弹出确认对话框,该属性就是设置对话框中的提示消息
     </li>
     <li style="text-align:justify">

     </li>

     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     InsertionMode.Replace:用新值替换旧值(默认值)
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     InsertionMode.InsertAfter:将新值插入到目标元素的底部
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     InsertionMode.InsertBefore:将新值插入到目标元素的顶部
    </p>
    <ul>
     <li style="text-align:justify">
      OnBegin:当执行ajax请求前,执行的js函数
     </li>
     <li style="text-align:justify">
      OnSuccess:当执行ajax请求成功时,执行的js函数
     </li>
     <li style="text-align:justify">
      OnFailure:当ajax请求失败时,执行的js函数
     </li>
     <li style="text-align:justify">
      OnComplate: 当ajax请求完成后,执行的js函数(不管请求失败或成功,都会执行)
     </li>
     <li style="text-align:justify">
      LoadingElementId:在ajax请求期间,可以显示一段友好的提示,该属性用于设置提示元素的ID
     </li>
     <li style="text-align:justify">
      LoadingElementDuration: 提示元素在显示和隐藏时,可以执行一段动画,该属性用于设置提示元素的动画执行时间
     </li>
    </ul>
