    <h2>
     <a id="_0">
     </a>
     题目描述
    </h2>
    <p>
     公有云的某个region内，
     <code>
      N
     </code>
     个网络节点组网情况可以使用一个
     <code>
      N×N
     </code>
     的矩阵
     <code>
      matrix
     </code>
     表示。在这个组网图中，
     <code>
      matrix[i][j] = p
     </code>
     时，表示用户在编号为
     <code>
      i
     </code>
     的节点访问编号为
     <code>
      j
     </code>
     的节点时，必须在
     <code>
      i
     </code>
     节点上具有
     <code>
      ≥p
     </code>
     的权限等级（
     <code>
      p=0
     </code>
     时表示无法通过
     <code>
      i
     </code>
     节点访问
     <code>
      j
     </code>
     节点）。如果用户成功访问了
     <code>
      j
     </code>
     节点，那么它在
     <code>
      j
     </code>
     节点上的权限等级调整为
     <code>
      p
     </code>
     。
    </p>
    <p>
     <code>
      exposed
     </code>
     为一个整数数组，表示暴露在公网上的网络节点的编号列表。某天扫描发现这批暴露在公网的节点存在被外部恶意攻击风险，且该攻击会影响到可访问的其他节点，并可以持续传递进行攻击。被恶意攻击的节点从公网访问时，攻击者获得了ROOT权限（权限等级为10，即最大值）。
    </p>
    <p>
     小明是一名网络安全工程师，为了在有限的时间内尽可能减少故障带来的损失，需要立即将某个节点从公网“下线”。假设攻击结束时，被攻击过的节点数量为
     <code>
      R
     </code>
     ，请帮小明计算出将哪个节点下线能使
     <code>
      R
     </code>
     尽可能小。如果答案有多个节点，返回索引最小的那个节点。请注意：从公网“下线”的节点，不会受到来自公网的攻击，但仍然可能被“可访问”的其他节点传递攻击。
    </p>
    <h3>
     <a id="_7">
     </a>
     输入描述
    </h3>
    <ol>
     <li>
      第一行：一个整数
      <code>
       N
      </code>
      ，表示网络节点数量（
      <code>
       2 ≤ N ≤ 24
      </code>
      ）。
     </li>
     <li>
      接下来
      <code>
       N
      </code>
      行：每行
      <code>
       N
      </code>
      个数字
      <code>
       v
      </code>
      ，以空格分割，形成一个
      <code>
       N×N
      </code>
      的矩阵，表示网络节点组网的矩阵（
      <code>
       0 ≤ v ≤ 10
      </code>
      ）。
     </li>
     <li>
      最后一行：输入
      <code>
       exposed
      </code>
      数组，表示暴露在公网上的网络节点的编号列表，数组元素不会重复（
      <code>
       0 ≤ exposed[i] ≤ N-1
      </code>
      ）。
     </li>
    </ol>
    <h3>
     <a id="_12">
     </a>
     输出描述
    </h3>
    <p>
     输出在
     <code>
      exposed
     </code>
     数组中，计划“下线”的那个节点的编号。
    </p>
    <h3>
     <a id="_15">
     </a>
     用例输入
    </h3>
    <pre><code class="prism language-bash"><span class="token number">4</span>
<span class="token number">1</span> <span class="token number">0</span> <span class="token number">0</span> <span class="token number">0</span>
<span class="token number">0</span> <span class="token number">1</span> <span class="token number">2</span> <span class="token number">0</span>
<span class="token number">0</span> <span class="token number">1</span> <span class="token number">1</span> <span class="token number">4</span>
<span class="token number">0</span> <span class="token number">0</span> <span class="token number">3</span> <span class="token number">1</span>
<span class="token number">1</span> <span class="token number">3</span>
</code></pre>
    <pre><code class="prism language-bash"><span class="token number">3</span>
</code></pre>
    <ul>
     <li>
      <code>
       1
      </code>
      和
      <code>
       3
      </code>
      是公网暴露的节点。
     </li>
     <li>
      <code>
       1
      </code>
      、
      <code>
       2
      </code>
      、
      <code>
       3
      </code>
      三个节点是连通的，但相互访问需要考虑权限等级限制。
     </li>
     <li>
      例如从
      <code>
       1
      </code>
      节点登录，访问到
      <code>
       2
      </code>
      节点后，权限等级不足以访问
      <code>
       3
      </code>
      号节点。
     </li>
     <li>
      如果将
      <code>
       1
      </code>
      号节点从公网下线，那
      <code>
       3
      </code>
      号节点可以先访问
      <code>
       2
      </code>
      号再访问
      <code>
       1
      </code>
      号，此时
      <code>
       R
      </code>
      的值为
      <code>
       3
      </code>
      。
     </li>
     <li>
      如果将
      <code>
       3
      </code>
      号节点从公网下线，则只能通过
      <code>
       1
      </code>
      号节点访问到
      <code>
       2
      </code>
      号节点。
     </li>
    </ul>
    <h2>
     <a id="_35">
     </a>
     解题思路
    </h2>
    <ol>
     <li>
      <p>
       <strong>
        问题建模
       </strong>
       ：
      </p>
      <ul>
       <li>
        该问题可以看作是一个
        <strong>
         图论问题
        </strong>
        ，目标是找到一个节点，将其从公网下线后，被攻击的节点数量
        <code>
         R
        </code>
        最小。
       </li>
       <li>
        使用优先队列（大顶堆）模拟攻击过程，从权限等级高的节点开始传播攻击。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数据结构
       </strong>
       ：
      </p>
      <ul>
       <li>
        使用二维数组
        <code>
         mp
        </code>
        存储网络节点组网的矩阵。
       </li>
       <li>
        使用
        <code>
         vector&lt;int&gt;
        </code>
        存储暴露在公网上的节点编号
        <code>
         exposed
        </code>
        。
       </li>
       <li>
        使用
        <code>
         vector&lt;int&gt;
        </code>
        存储每个节点的最大权限等级
        <code>
         vis
        </code>
        。
       </li>
       <li>
        使用优先队列
        <code>
         priority_queue
        </code>
        模拟攻击传播过程。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        模拟攻击传播
       </strong>
       ：
      </p>
      <ul>
       <li>
        对于每个暴露的节点，如果该节点不是被下线的节点，则将其加入优先队列。
       </li>
       <li>
        从优先队列中取出当前节点，尝试访问其所有可达的节点。
       </li>
       <li>
        如果当前节点的权限等级足够访问目标节点，则将目标节点加入优先队列，并更新目标节点的最大权限等级。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        计算被攻击的节点数量
       </strong>
       ：
      </p>
      <ul>
       <li>
        遍历所有节点，统计被攻击的节点数量
        <code>
         R
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        选择最优节点
       </strong>
       ：
      </p>
      <ul>
       <li>
        遍历所有暴露的节点，计算每个节点下线后的被攻击节点数量
        <code>
         R
        </code>
        。
       </li>
       <li>
        选择使
        <code>
         R
        </code>
        最小的节点，如果有多个节点使
        <code>
         R
        </code>
        最小，则选择索引最小的节点。
       </li>
      </ul>
     </li>
    </ol>
