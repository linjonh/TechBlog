    <h2>
     <a id="_6">
     </a>
     第一部分：文件操作
    </h2>
    <h3>
     <a id="1__0_8">
     </a>
     <strong>
      1. 清空文件（清除文件大小为 0）
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token operator">&gt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     这行命令使用
     <strong>
      输出重定向
     </strong>
     操作符
     <code>
      &gt;
     </code>
     。
     <strong>
      输出重定向
     </strong>
     造成文件被打开并写入。如果文件不存在就创建它，若存在就清除文件大小为 0。由于重定向没给文件任何内容，结果就清空了文件。
    </p>
    <p>
     如果想用一些字符串文本替换文件里的内容，可用如下命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token string">"some string"</span> <span class="token operator">&gt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     这将
     <code>
      some string
     </code>
     字符串写入文件
    </p>
    <h3>
     <a id="2__24">
     </a>
     <strong>
      2. 追加字符串写入文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token string">"foo bar baz"</span> <span class="token operator">&gt;&gt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     这行命令使用另一个
     <strong>
      输出重定向
     </strong>
     操作符
     <code>
      &gt;&gt;
     </code>
     ，意思追加内容到文件。如果文件不存在就创建它并写入。
     <strong>
      追加
     </strong>
     的内容放到文件尾并放到新行（另起一行）。若不想另起一行，可以使用
     <code>
      echo
     </code>
     命令选项
     <code>
      -n
     </code>
     ，形如
     <code>
      echo -n
     </code>
     命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token parameter variable">-n</span> <span class="token string">"foo bar baz"</span> <span class="token operator">&gt;&gt;</span> <span class="token function">file</span>
</code></pre>
    <h3>
     <a id="3__36">
     </a>
     <strong>
      3. 读取文件首行并赋值到一个变量
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     本命令使用内建命令
     <code>
      read
     </code>
     和重定向符
     <code>
      &lt;
     </code>
     。
    </p>
    <p>
     <strong>
      解释：
     </strong>
     <code>
      read
     </code>
     命令从标准输入读取一行文字到
     <code>
      line
     </code>
     变量。
     <code>
      -r
     </code>
     命令选项意味着输入保留原始（raw）-- 意思是字符串中的
     <strong>
      转义字符反斜杠 ()
     </strong>
     保持原样。重定向命令
     <code>
      &lt; file
     </code>
     意味着标准输入被
     <code>
      file
     </code>
     代替 – 从
     <code>
      file
     </code>
     文件中读取数据。
    </p>
    <p>
     <code>
      read
     </code>
     命令读入字符串会按照
     <code>
      IFS
     </code>
     环境变量设置，删除读取的一行字符串中行首、行尾的分隔符。
     <strong>
      IFS
     </strong>
     表示
     <strong>
      间隔符（Internal Field Separator）
     </strong>
     的含义。用于命令行扩展分隔单词和断行，默认
     <strong>
      IFS
     </strong>
     设置为
     <strong>
      空格（space）、制表符（tab）、回车换行（newline \n）
     </strong>
     。如果想保留这些，可利用
     <code>
      IFS
     </code>
     即时清除设置命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable"><span class="token environment constant">IFS</span></span><span class="token operator">=</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     上面命令仅限当前命令，并不会改变
     <strong>
      IFS
     </strong>
     的设置。只用于将文件中的一行文本真正原样读出。
    </p>
    <p>
     另外的从文件中读出第一行的方法：
    </p>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable">line</span><span class="token operator">=</span><span class="token variable"><span class="token variable">$(</span><span class="token function">head</span> <span class="token parameter variable">-1</span> <span class="token function">file</span><span class="token variable">)</span></span>
</code></pre>
    <p>
     上面方法使用
     <strong>
      命令执行替换
     </strong>
     操作符
     <code>
      $(...)
     </code>
     ，它执行括号中
     <code>
      ...
     </code>
     的命令返回其输出。本例中，命令
     <code>
      head -1 file
     </code>
     输出文件的第一行，之后赋值给
     <code>
      line
     </code>
     变量。 使用
     <code>
      $(...)
     </code>
     等同于
     <strong>
      <code>
       ...
      </code>
     </strong>
     ，故也可写成：
    </p>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable">line</span><span class="token operator">=</span><span class="token variable"><span class="token variable">`</span><span class="token function">head</span> <span class="token parameter variable">-1</span> <span class="token function">file</span><span class="token variable">`</span></span>
</code></pre>
    <p>
     然而，
     <code>
      $(...)
     </code>
     是优选方法，它简洁，易于嵌套。
    </p>
    <h3>
     <a id="4__68">
     </a>
     <strong>
      4. 一行行地读取文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token keyword">while</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $line</span>
<span class="token keyword">done</span> <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     这是唯一正确的一行行地读取文件的方法。它将
     <code>
      read
     </code>
     命令放到
     <code>
      while
     </code>
     循环中，当
     <code>
      read
     </code>
     命令遇到文件结束，它返回一个非零正数（代码失败），循环结束。
    </p>
    <p>
     记住
     <code>
      read
     </code>
     命令会删除前导、结尾的分隔符（空格，tab，回车），若想保留分隔符，使用
     <code>
      IFS
     </code>
     即时清除设置命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token keyword">while</span> <span class="token assign-left variable"><span class="token environment constant">IFS</span></span><span class="token operator">=</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $line</span>
<span class="token keyword">done</span> <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     若不喜欢循环语句尾加
     <code>
      &lt; file
     </code>
     重定向操作，也可使用管道命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token function">cat</span> <span class="token function">file</span> <span class="token operator">|</span> <span class="token keyword">while</span> <span class="token assign-left variable"><span class="token environment constant">IFS</span></span><span class="token operator">=</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $line</span>
<span class="token keyword">done</span>
</code></pre>
    <h3>
     <a id="5__94">
     </a>
     <strong>
      5. 从文件中随机读取一行并赋值变量
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> random_line <span class="token operator">&lt;</span> <span class="token operator">&lt;</span><span class="token punctuation">(</span><span class="token function">shuf</span> <span class="token function">file</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     这命令对于 bash ，不是一个明晰的从文件随机读取行的方法。因为它需要借助某些外部程序的帮助。在一台运行现代 Linux 的机器上使用来自 GNU 的核心工具
     <code>
      shut
     </code>
     命令。
    </p>
    <p>
     这条命令使用
     <strong>
      进程执行
     </strong>
     替换操作符
     <code>
      &lt;(...)
     </code>
     ，这个
     <strong>
      进程执行
     </strong>
     替换操作创建匿名管道并连接
     <strong>
      进程执行
     </strong>
     的输出部分到某个命名管道的写入部分，然后 bash 执行这个进程，这样就把执行进程的匿名管道替换成了命名管道（用设备文件描述符表示）。
    </p>
    <p>
     当 bash 解析
     <code>
      &lt;(shuf file)
     </code>
     命令部分时，它打开一个特殊的设备文件
     <code>
      /dev/fd/n
     </code>
     ，其中
     <code>
      n
     </code>
     是一个未使用的文件描述符，然后运行
     <code>
      shut file
     </code>
     进程并将它的输出连接到
     <code>
      /dev/fd/n
     </code>
     ，这样
     <code>
      &lt;(shut file)
     </code>
     被替换成
     <code>
      &lt; /dev/fd/n
     </code>
     。实际效果如下：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> random_line <span class="token operator">&lt;</span> /dev/fd/n
</code></pre>
    <p>
     这将从这个被行随机化过的文件中读取第一行。
    </p>
    <p>
     下面是借助 GNU 的
     <code>
      sort
     </code>
     命令完成同样目标的另一解决方案。GNU 的
     <code>
      sort
     </code>
     命令通过
     <code>
      -R
     </code>
     命令选项随机化输入。
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> random_line <span class="token operator">&lt;</span> <span class="token operator">&lt;</span><span class="token punctuation">(</span><span class="token function">sort</span> <span class="token parameter variable">-R</span> <span class="token function">file</span><span class="token punctuation">)</span>
</code></pre>
    <p>
     另外的获取文件中随机一行并赋值变量的方法：
    </p>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable">random_line</span><span class="token operator">=</span><span class="token variable"><span class="token variable">$(</span><span class="token function">sort</span> <span class="token parameter variable">-R</span> <span class="token function">file</span> <span class="token operator">|</span> <span class="token function">head</span> <span class="token parameter variable">-1</span><span class="token variable">)</span></span>
</code></pre>
    <p>
     上面命令通过
     <code>
      sort -R
     </code>
     命令随机化文件后，执行
     <code>
      head -1
     </code>
     取出其第一行。
    </p>
    <h3>
     <a id="6__3__3__126">
     </a>
     <strong>
      6. 从文件读取一行并分割前 3 个单词赋值给 3 个变量
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token keyword">while</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> field1 field2 field3 throwaway<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $field1, $field2, and $field3</span>
<span class="token keyword">done</span> <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     如果在
     <code>
      read
     </code>
     命令中指定超过一个的变量名，将会把输入字符串分割成符合变量个数的字段（也称域，依据
     <code>
      IFS
     </code>
     设置的分隔符进行分割。默认是空格、tab、回车），之后，把第 1 个字段分配给第 1 个变量，把第 2 个字段分配给第 2 个变量，以此类推。但是，最后一个字段是字符串剩下的全部。这就是我们最后使用一个不用的变量（throwaway）的原因，若没有它，也许
     <code>
      field3
     </code>
     中并不是我们期望的值。
    </p>
    <p>
     经常我们会用
     <code>
      _
     </code>
     更短的形式替换
     <code>
      throwaway
     </code>
     做为弃用的变量名：
    </p>
    <pre><code class="prism language-shell">$ <span class="token keyword">while</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> field1 field2 field3 _<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $field1, $field2, and $field3</span>
<span class="token keyword">done</span> <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     当然，若你能保证字符串中只包含 3 个字段，你完全可以不安排这个无用的变量名：
    </p>
    <pre><code class="prism language-shell">$ <span class="token keyword">while</span> <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> field1 field2 field3<span class="token punctuation">;</span> <span class="token keyword">do</span>
    <span class="token comment"># do something with $field1, $field2, and $field3</span>
<span class="token keyword">done</span> <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     这里有个例子。我们知道，当想知道一个文件内容有多少行、多少单词，多少字符时，在文件上运行
     <code>
      wc
     </code>
     命令，将输出这 3 个数值和文件名，如下所示：
    </p>
    <pre><code class="prism language-shell">$ <span class="token function">cat</span> file-with-5-lines
x <span class="token number">1</span>
x <span class="token number">2</span>
x <span class="token number">3</span>
x <span class="token number">4</span>
x <span class="token number">5</span>

$ <span class="token function">wc</span> file-with-5-lines
 <span class="token number">5</span> <span class="token number">10</span> <span class="token number">20</span> file-with-5-lines
</code></pre>
    <p>
     因此，这个文件有 5 行，10 个单词和 20 个字符（空格、回车都算）。我们可使用
     <code>
      read
     </code>
     命令获取这些数值并放到变量中。演示如下：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> lines words chars _ <span class="token operator">&lt;</span> <span class="token operator">&lt;</span><span class="token punctuation">(</span><span class="token function">wc</span> file-with-5-lines<span class="token punctuation">)</span>

$ <span class="token builtin class-name">echo</span> <span class="token variable">$lines</span>
<span class="token number">5</span>
$ <span class="token builtin class-name">echo</span> <span class="token variable">$words</span>
<span class="token number">10</span>
$ <span class="token builtin class-name">echo</span> <span class="token variable">$chars</span>
<span class="token number">20</span>
</code></pre>
    <p>
     类似地，可以使用
     <strong>
      即入字符串（here-strings）
     </strong>
     将字符串分割成多个字段赋值给变量。假设有个字符串
     <strong>
      “20 packets in 10 seconds”
     </strong>
     保存在
     <code>
      $info
     </code>
     变量中，现将
     <strong>
      20
     </strong>
     和
     <strong>
      10
     </strong>
     取出放到 2 个变量中。不久前这个任务我写过：
    </p>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable">packets</span><span class="token operator">=</span><span class="token variable"><span class="token variable">$(</span><span class="token builtin class-name">echo</span> $info <span class="token operator">|</span> <span class="token function">awk</span> <span class="token string">'{ print $1 }'</span><span class="token variable">)</span></span>
$ <span class="token assign-left variable">duration</span><span class="token operator">=</span><span class="token variable"><span class="token variable">$(</span><span class="token builtin class-name">echo</span> $info <span class="token operator">|</span> <span class="token function">awk</span> <span class="token string">'{ print $4 }'</span><span class="token variable">)</span></span>
</code></pre>
    <p>
     然而，若善用
     <code>
      read
     </code>
     命令和 bash 的能力，使用如下一条简单命令可达成目标：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">read</span> packets _ _ duration _ <span class="token operator">&lt;&lt;&lt;</span> <span class="token string">"<span class="token variable">$info</span>"</span>
</code></pre>
    <p>
     这里的
     <code>
      &lt;&lt;&lt;
     </code>
     是 bash 中的一种重定向机制，被称为
     <strong>
      即入字符串（here-strings）重定向
     </strong>
     操作符，用于直接将一个字符串重定向到标准输入传递给命令。
    </p>
    <h3>
     <a id="7__194">
     </a>
     <strong>
      7. 获取文件大小并赋值变量
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token assign-left variable">size</span><span class="token operator">=</span><span class="token variable"><span class="token variable">$(</span><span class="token function">wc</span> <span class="token parameter variable">-c</span> <span class="token operator">&lt;</span> <span class="token function">file</span><span class="token variable">)</span></span>
</code></pre>
    <p>
     上面的一行命令使用命令执行替换操作符
     <code>
      $(...)
     </code>
     （在第 3 段 --[3. 读取文件首行并赋值到一个变量] – 中已讲过）。本命令中将
     <code>
      wc -c &lt; file
     </code>
     执行结果，即得出的
     <strong>
      file
     </strong>
     的大小赋值给变量
     <strong>
      size
     </strong>
     。（
     <strong>
      译注：
     </strong>
     思考并实验这里为什么使用
     <code>
      wc -c &lt; file
     </code>
     而不使用
     <code>
      wc -c file
     </code>
     ？）
    </p>
    <h3>
     <a id="8__202">
     </a>
     <strong>
      8. 从全路径字符串中取出文件名
     </strong>
    </h3>
    <p>
     让我们看个例子，假设有个
     <strong>
      “/path/to/file.txt”
     </strong>
     字符串表示文件的全路径，只想取出其中的文件名
     <strong>
      file.txt
     </strong>
     ，该如何做呢？ 一个好的解决方案使用 bash 外壳程序的
     <strong>
      参数展开（parameter expansion）
     </strong>
     机制。演示如下：
    </p>
    <pre><code class="prism language-c">$ pvar <span class="token operator">=</span> <span class="token string">"/path/to/file.txt"</span>
$ filename<span class="token operator">=</span>$<span class="token punctuation">{<!-- --></span>pvar##<span class="token operator">*</span><span class="token operator">/</span><span class="token punctuation">}</span>
</code></pre>
    <p>
     如上命令，使用形如
     <code>
      ${varname##pattern}
     </code>
     的
     <strong>
      参数展开
     </strong>
     操作符。这个展开操作试图从
     <code>
      $varname
     </code>
     变量表示的字符串开始按
     <strong>
      pattern
     </strong>
     进行模式匹配，若匹配成功，将从
     <strong>
      $varname
     </strong>
     字符串删除掉匹配的子字符串后的剩余部分返回。（删除匹配）
    </p>
    <p>
     上面例子的情况
     <code>
      */
     </code>
     模式将在 “
     <strong>
      /path/to/file.txt
     </strong>
     ” 中
     <strong>
      从头到尾
     </strong>
     匹配任意字符后跟
     <strong>
      ‘/’
     </strong>
     的模式，由于
     <code>
      */
     </code>
     是贪心匹配，故匹配结果是
     <strong>
      /path/to/
     </strong>
     。按照展开逻辑，表达式将返回
     <strong>
      删除匹配
     </strong>
     的子串，所以，变量
     <strong>
      $filename
     </strong>
     将等于
     <strong>
      file.txt
     </strong>
     。（其中
     <strong>
      ##
     </strong>
     表示
     <strong>
      从头到尾
     </strong>
     开始模式匹配）
    </p>
    <h3>
     <a id="9__215">
     </a>
     <strong>
      9. 从全路径字符串中取出目录名
     </strong>
    </h3>
    <p>
     与前一任务命令相似，这次让我们从
     <strong>
      “/path/to/file.txt”
     </strong>
     中取出目录名
     <strong>
      /path/to/
     </strong>
     子串。同样使用
     <strong>
      参数展开
     </strong>
     操作，命令如下：
    </p>
    <pre><code class="prism language-c">$ pvar <span class="token operator">=</span> <span class="token string">"/path/to/file.txt"</span>
$ dirname<span class="token operator">=</span>$<span class="token punctuation">{<!-- --></span>pvar<span class="token operator">%</span><span class="token comment">/*}
</span></code></pre>
    <p>
     这次
     <code>
      ${varname%pattern}
     </code>
     的
     <strong>
      参数展开
     </strong>
     操作是
     <strong>
      从尾到头
     </strong>
     的匹配模式（其中
     <strong>
      %
     </strong>
     表示
     <strong>
      从尾到头
     </strong>
     开始模式匹配）。
    </p>
    <p>
     例子中，，模式为
     <strong>
      /*
     </strong>
     。
     <strong>
      从尾到头
     </strong>
     匹配后，匹配结果为
     <strong>
      /file.txt
     </strong>
     。将匹配删除后的结果为
     <strong>
      /path/to
     </strong>
     。
    </p>
    <h3>
     <a id="10__228">
     </a>
     <strong>
      10. 快速创建文件副本
     </strong>
    </h3>
    <p>
     让我们看一下将
     <strong>
      /path/to/file
     </strong>
     表示的文件，在同目录下拷贝一个副本命令。通常你会写成如下命令：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">cp</span> /path/to/file /path/to/file_copy
</code></pre>
    <p>
     然而，你可以使用
     <strong>
      大括号展开
     </strong>
     机制，命令写成如下简短形式：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">cp</span> /path/to/file<span class="token punctuation">{<!-- --></span>,_copy<span class="token punctuation">}</span>
</code></pre>
    <p>
     <strong>
      大括号展开
     </strong>
     机制是将大括号中的每个条目依次展开（枚举每个条目）。例子中
     <code>
      /path/to/file{,_copy}
     </code>
     将会被展开成
     <code>
      /path/to/file /path/to/file_copy
     </code>
     ，整个命令将变成
     <code>
      cp /path/to/file /path/to/file_copy
     </code>
     。
    </p>
    <p>
     相似地，可以快速移动（改名）一个文件，如下命令：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">mv</span> /path/to/file<span class="token punctuation">{<!-- --></span>,_old<span class="token punctuation">}</span>
</code></pre>
    <p>
     命令展开成
     <code>
      mv /path/to/file /path/to/file_old
     </code>
     。
    </p>
    <h2>
     <a id="_252">
     </a>
     第二部分：字符串操作
    </h2>
    <h3>
     <a id="1__254">
     </a>
     <strong>
      1. 创建输出字母表
     </strong>
    </h3>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span>
</code></pre>
    <p>
     以上命令使用 bash 的
     <strong>
      大括号展开（brace expansion）
     </strong>
     机制，其含义是生成「字面字符串」序列（枚举每个条目）。此例中表达式是
     <strong>
      {x…z}
     </strong>
     ，其中 x 和 z 是单个字符，这个表示字母表中从 x 到 z 的所有字符（包含 x 和 z）（注意：x 和 z 之间是 2 个
     <strong>
      .
     </strong>
     符号）。
    </p>
    <p>
     若运行以上命令，其输出 a-z 的所有字母：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span>
a b c d e f g h i j k l m n o p q r s t u <span class="token function">v</span> w x y z
</code></pre>
    <h3>
     <a id="2__269">
     </a>
     <strong>
      2. 无空格分隔输出字母表中字母
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">printf</span> <span class="token string">"%c"</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span>
</code></pre>
    <p>
     这个神奇的 bash 技巧 99.99% 的使用者不知道。如果把一个列表传递个
     <code>
      printf
     </code>
     命令，
     <strong>
      printf
     </strong>
     命令会循环依次处理列表的每个条目，循环结束，命令退出。这是一个极佳的技巧。
    </p>
    <p>
     以上命令使用
     <strong>
      “%c”
     </strong>
     作为
     <strong>
      printf
     </strong>
     命令的格式输出控制符。（与 C 语言中函数类似）它将列表序列中的字母依次输出。
    </p>
    <p>
     下面是命令运行输出结果：
    </p>
    <pre><code class="prism language-shell">abcdefghijklmnopqrstuvwxyz
</code></pre>
    <p>
     这个输出后没有回车，因为列表序列中不包含回车符（
     <strong>
      ‘\n’
     </strong>
     ）。加入行终止，只需在字符序列尾加入回车符，方法是在列表序列表达式尾加上
     <code>
      $'\n'
     </code>
     ，命令如下：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">printf</span> <span class="token string">"%c"</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span> <span class="token string">$'<span class="token entity" title="\n">\n</span>'</span>
</code></pre>
    <p>
     <code>
      $'\n'
     </code>
     是 bash 方便表示，代表回车符。命令输出 {a…z} 后，接着输出回车符。（注意：printf 类似 C 语言中的 printf 函数，接受可变参数）。
    </p>
    <p>
     另一方法使用
     <code>
      echo
     </code>
     命令输出
     <code>
      printf "%c" {a..z}
     </code>
     命令结果。这里使用了命令执行替换机制（第一篇文讲解过）。命令如下：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token variable"><span class="token variable">$(</span><span class="token builtin class-name">printf</span> <span class="token string">"%c"</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span><span class="token variable">)</span></span>
</code></pre>
    <p>
     如上命令，使用命令执行替换（command substitution）输出字母表传递给
     <code>
      echo
     </code>
     命令输出，
     <code>
      echo
     </code>
     命令默认在命令后输出回车。
    </p>
    <p>
     想以列方式输出字母？在
     <code>
      printf
     </code>
     命令的格式输出控制中加入回车符！
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">printf</span> <span class="token string">"%c<span class="token entity" title="\n">\n</span>"</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span>
a
b
<span class="token punctuation">..</span>.
z
</code></pre>
    <p>
     想将
     <code>
      printf
     </code>
     命令展开的列表赋值给变量？使用
     <code>
      -v
     </code>
     命令选项：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">printf</span> <span class="token parameter variable">-v</span> alphabet <span class="token string">"%c"</span> <span class="token punctuation">{<!-- --></span>a<span class="token punctuation">..</span>z<span class="token punctuation">}</span>
</code></pre>
    <p>
     这将
     <strong>
      “abcdefghijklmnopqrstuvwxyz”
     </strong>
     字符串赋值给
     <code>
      $alphabet
     </code>
     变量。
    </p>
    <p>
     类似地，可创建整数列表序列，我们试验一下从 1 到 100 ：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span><span class="token number">1</span><span class="token punctuation">..</span><span class="token number">100</span><span class="token punctuation">}</span>
</code></pre>
    <p>
     输出：
    </p>
    <pre><code class="prism language-bash"><span class="token number">1</span> <span class="token number">2</span> <span class="token number">3</span> <span class="token punctuation">..</span>. <span class="token number">100</span>
</code></pre>
    <p>
     也可使用
     <code>
      seq
     </code>
     命令工具创建数字序列，替换上述的方法：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">seq</span> <span class="token number">1</span> <span class="token number">100</span>
</code></pre>
    <h3>
     <a id="3__0__9__0_337">
     </a>
     <strong>
      3. 给 0 到 9 数字序列中每个数加前导 0
     </strong>
    </h3>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">printf</span> <span class="token string">"%02d "</span> <span class="token punctuation">{<!-- --></span><span class="token number">0</span><span class="token punctuation">..</span><span class="token number">9</span><span class="token punctuation">}</span>
</code></pre>
    <p>
     这里我们再次使用
     <code>
      printf
     </code>
     命令的循环能力。 这次
     <strong>
      printf
     </strong>
     的格式控制字符串是
     <strong>
      “”%02d"
     </strong>
     ，它的意思是输出的数字以 0 前导补足数字到 2 位。后面的命令参数通过大括号展开传递进来从 0 到 9 的数字。
     <br/>
     输出：
    </p>
    <pre><code class="prism language-bash">00 01 02 03 04 05 06 07 08 09
</code></pre>
    <p>
     如果使用的 bash 版本大于 4，也可使用下面的大括号展开表达式：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span>00<span class="token punctuation">..</span>09<span class="token punctuation">}</span>
</code></pre>
    <p>
     老的 bash 版本没有这个特性。
    </p>
    <h3>
     <a id="4__30__358">
     </a>
     <strong>
      4. 生成 30 个英语单词
     </strong>
    </h3>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span>w,t,<span class="token punctuation">}</span>h<span class="token punctuation">{<!-- --></span>e<span class="token punctuation">{<!-- --></span>n<span class="token punctuation">{<!-- --></span>,ce<span class="token punctuation">{<!-- --></span>,forth<span class="token punctuation">}</span><span class="token punctuation">}</span>,re<span class="token punctuation">{<!-- --></span>,in,fore,with<span class="token punctuation">{<!-- --></span>,al<span class="token punctuation">}</span><span class="token punctuation">}</span><span class="token punctuation">}</span>,ither,at<span class="token punctuation">}</span>
</code></pre>
    <p>
     上面的命令是对
     <strong>
      大括号展开（brace expansion）
     </strong>
     机制滥用（也能很好理解它）。来看看它的输出：
    </p>
    <pre><code class="prism language-shell">when whence whenceforth where wherein wherefore wherewith wherewithal whither what <span class="token keyword">then</span> thence thenceforth there therein therefore therewith therewithal thither that hen hence henceforth here herein herefore herewith herewithal hither hat
</code></pre>
    <p>
     太神奇了！
    </p>
    <p>
     它怎样做到的！在
     <strong>
      大括号展开
     </strong>
     机制中，在大括号中的每个被逗号
     <strong>
      ,
     </strong>
     分隔的条目都可以分别交叉放置。例如，如果像下面的命令：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token punctuation">{<!-- --></span>a,b,c<span class="token punctuation">}</span><span class="token punctuation">{<!-- --></span><span class="token number">1,2</span>,3<span class="token punctuation">}</span>
</code></pre>
    <p>
     上面的大括号表达式将会产生
     <strong>
      a1 a2 a3 b1 b2 b3 c1 c2 c3
     </strong>
     ，其首先使用第 1 个大括号中的
     <strong>
      a
     </strong>
     依次和第 2 个大括号中的
     <strong>
      1,2,3
     </strong>
     组合，构成
     <strong>
      a1 a2 a3
     </strong>
     ，接着使用第 1 个大括号中的
     <strong>
      b
     </strong>
     依次和第 2 个大括号中的
     <strong>
      1,2,3
     </strong>
     组合，构成
     <strong>
      b1 b2 b3
     </strong>
     ，接着是
     <strong>
      c
     </strong>
     。（矩阵乘的概念）
    </p>
    <p>
     所以，上面生成单词的命令就会构成上面那些单词的输出！
    </p>
    <h3>
     <a id="5__1__10__382">
     </a>
     <strong>
      5. 生成 1 个字符串的 10 个副本
     </strong>
    </h3>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> foo<span class="token punctuation">{<!-- --></span>,,,,,,,,,,<span class="token punctuation">}</span>
</code></pre>
    <p>
     上述命令再次使用大括号展开机制。前面的字符串
     <strong>
      foo
     </strong>
     和后面的 10 个条目都为空的列表的每个条目组合，形成字符串
     <strong>
      foo
     </strong>
     的 10 个副本：
    </p>
    <pre><code class="prism language-shell">foo foo foo foo foo foo foo foo foo foo foo
</code></pre>
    <h3>
     <a id="6__2__394">
     </a>
     <strong>
      6. 合并连接 2 个字符串
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token string">"<span class="token variable">$x</span><span class="token variable">$y</span>"</span>
</code></pre>
    <p>
     这条命令简单合并连接 2 个变量，如果
     <code>
      x
     </code>
     变量包含字符串
     <strong>
      foo
     </strong>
     ，且
     <code>
      y
     </code>
     变量包含字符串
     <strong>
      bar
     </strong>
     ，结果为
     <strong>
      foobar
     </strong>
     。
    </p>
    <p>
     注意命令中
     <strong>
      “
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         x 
         
        
       
         x
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.4306em;">
          </span>
          <span class="mord mathnormal">
           x
          </span>
         </span>
        </span>
       </span>
      </span>
      y”
     </strong>
     是被双引号括住的。若不括住它，
     <code>
      echo
     </code>
     命令会优先将
     <strong>
      <span class="katex--inline">
       <span class="katex">
        <span class="katex-mathml">
         x 
         
        
       
         x
        </span>
        <span class="katex-html">
         <span class="base">
          <span class="strut" style="height: 0.4306em;">
          </span>
          <span class="mord mathnormal">
           x
          </span>
         </span>
        </span>
       </span>
      </span>
      y
     </strong>
     变量中的值解释为命令行参数或命令选项。因此，若
     <strong>
      $x
     </strong>
     变量的值与命令行参数或命令选项相同（以
     <strong>
      -
     </strong>
     开始的字符等），将被按命令参数或命令选项解释，使结果不能按我们的意愿输出：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">x</span><span class="token operator">=</span>-n
<span class="token assign-left variable">y</span><span class="token operator">=</span><span class="token string">" bar"</span>
<span class="token builtin class-name">echo</span> <span class="token variable">$x</span><span class="token variable">$y</span>
</code></pre>
    <p>
     输出：
    </p>
    <pre><code class="prism language-shell">foo
</code></pre>
    <p>
     相反，合并连接 2 个字符串方法：
    </p>
    <pre><code class="prism language-shell"><span class="token assign-left variable">x</span><span class="token operator">=</span>-n
<span class="token assign-left variable">y</span><span class="token operator">=</span><span class="token string">" foo"</span>
<span class="token builtin class-name">echo</span> <span class="token string">"<span class="token variable">$x</span><span class="token variable">$y</span>"</span>
</code></pre>
    <p>
     输出：
    </p>
    <pre><code class="prism language-shell"><span class="token punctuation">\</span>-n foo
</code></pre>
    <p>
     若想合并连接 2 个字符串赋值到另一个变量，就不必要使用双引号括住：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">var</span><span class="token operator">=</span><span class="token variable">$x</span><span class="token variable">$y</span>

      文件描述符
     </strong>
     ，也可以从它那里读或写入到它。
    </p>
    <p>
     <strong>
      文件描述符
     </strong>
     总是指向某些文件（除非文件被关闭）。通常，bash 启动时会把 3 个
     <strong>
      标准文件描述符
     </strong>
     ：stdin，stdout，stderr 通通指向终端。输入（stdin）来自终端的键入内容，所有的输出（stdout，stderr）显示在终端里。
    </p>
    <p>
     假定你的终端是
     <strong>
      /dev/tty0
     </strong>
     ，下面是
     <strong>
      终端
     </strong>
     与
     <strong>
      标准文件描述符
     </strong>
     之间的关系示意图：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/b36edc6be96d8ee7686890ed71d6f9aa.png"/>
    </p>
    <p>
     当 Bash 执行一条命令时，它会
     <strong>
      分支（fork）
     </strong>
     一个子进程，这个子进程继承
     <strong>
      fork
     </strong>
     它的父进程的一切标准文件描述符（关于
     <strong>
      fork
     </strong>
     ，详情参考手册命令
     <code>
      man 2 fork
     </code>
     ）。首先按照命令行指示进行
     <strong>
      重定向输入 / 输出
     </strong>
     后才
     <strong>
      执行（exec）
     </strong>
     命令（关于
     <strong>
      exec
     </strong>
     ，详情参考手册命令
     <code>
      man 3 exec
     </code>
     ）。
    </p>
    <p>
     对于想
     <strong>
      bash 重定向
     </strong>

     <strong>
      当重定向发生时，标准文件描述符如何变化
     </strong>
     --，这个蓝图将极大地帮助到你。
    </p>
    <h3>
     <a id="1__614">
     </a>
     <strong>
      1. 重定向标准输出到一个文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&gt;</span>file
</code></pre>
    <p>
     符号
     <strong>
      &gt;
     </strong>
     是输出重定向操作符。bash 先尝试打开重定向指向的文件（这里是文件
     <strong>
      file
     </strong>
     ）并可写入，如果成功，Bash 将
     <code>
      command
     </code>
     命令子进程的标准输出文件描述符指向打开的文件。如果失败，整个命令失败退出，结束子进程。
    </p>
    <p>
     命令行
     <code>
      command &gt;file
     </code>
     等同于命令行
     <code>
      command 1&gt;file
     </code>
     。数字
     <strong>
      1
     </strong>
     代表
     <strong>
      stdout
     </strong>
     ，它是标准输出文件描述符
     <strong>
      stdout
     </strong>
     的文件描述符号。
    </p>
    <p>
     下面是标准文件描述符改变的蓝图示意。Bash 打开文件
     <strong>
      file
     </strong>
     并确定可写入后，替换子进程的标准输出文件描述符
     <strong>
      1
     </strong>
     为
     <strong>
      file
     </strong>
     文件的文件描述符，自此，命令的所有输出将写入文件
     <strong>
      file
     </strong>
     中。
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/78339f354bc846a47f74a281ee89bbe1.png"/>
    </p>
    <p>
     通常，你可使用形如 “command
     <strong>
      n
     </strong>
     &gt;file” 的命令，表示此命令的
     <strong>
      n
     </strong>
     号标准输入 / 输出文件描述符指向
     <strong>
      file
     </strong>
     。
    </p>
    <p>
     <strong>
      举例：
     </strong>
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">ls</span> <span class="token operator">&gt;</span> file_list
</code></pre>
    <p>
     重定向
     <code>
      ls
     </code>
     命令的输出到
     <strong>
      file_list
     </strong>
     文件（命令的输出写入到文件）。
    </p>

</code></pre>
    <p>
     这里，将命令的标准错误输出（stderr 号 2）文件描述符重定向到文件。这里的数字
     <strong>
      2
     </strong>
     代表标准错误输出文件描述符。
    </p>
    <p>
     下面是标准文件描述符表改变蓝图：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/f4cafcb7596f7e9da18ed849e2572881.png"/>
    </p>
    <p>
     Bash 打开文件
     <strong>
      file
     </strong>
     并确定可写入后，替换子进程的标准错误输出文件描述符
     <strong>
      2
     </strong>

    <h3>
     <a id="3__652">
     </a>
     <strong>
      3. 重定向标准输出和标准错误输出到文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&amp;&gt;</span>file
</code></pre>
    <p>
     上面这行命令中
     <strong>
      “&amp;&gt;”
     </strong>
     表示把所有的标准输出文件描述符（
     <strong>
      stdout
     </strong>
     和
     <strong>
      stderr
     </strong>
     ）重定向到
     <strong>
      file
     </strong>
     文件描述符。
     <strong>
      &amp;&gt;
     </strong>
     是 Bash 快速引用所有标准输出描述符（
     <strong>
      1
     </strong>
     和
     <strong>
      2
     </strong>
     ）的简写。
    </p>
    <p>
     下图是标准文件描述符改变示意图：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/c811f96e13a7846fe081bb35c33d2bfa.png"/>
    </p>
    <p>
     这里你看到所有的标准输出（ stdout 和 stderr ）都指向
     <strong>
      file
     </strong>
     文件的文件描述符。
    </p>
    <p>
     有不同的几个方法将所有标准输出重定向到一个目标。可以按顺序一个接一个地重定向：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&gt;</span>file <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span>
</code></pre>
    <p>
     上面命令是较通用的重定向所有标准输出到文件的方法。首先，标准输出（stdout）重定向到 file 文件（这步是先运行 “
     <strong>
      &gt;file
     </strong>
     ” 的结果）。之后，标准错误输出（stderr）重定向到
     <strong>
      已重定向过的 stdout (&amp;1) 上 （通过 “2&gt;&amp;1”）
     </strong>
     。因此，所有的标准输出都重定向到 file 上。
    </p>
    <p>
     Bash 在命令行解析时，发现有多个重定向操作时，按从左到右的顺序依次解释处理。让我们按 Bash 执行顺序，单步分析，更好地理解它。一开始，Bash 在执行命令前，其 3 个标准输入 / 输出文件描述符都指向终端。示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/b36edc6be96d8ee7686890ed71d6f9aa.png"/>
    </p>
    <p>
     首先，Bash 处理第一个重定向 “
     <strong>
      &gt;file
     </strong>
     ” 。之前，我们见过这个示意图。它的标准输出（stdout）指向终端：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/78339f354bc846a47f74a281ee89bbe1.png"/>
    </p>
    <p>
     下一步解释处理重定向操作 “
     <strong>
      2&gt;&amp;1
     </strong>
     ”，这个操作的含义是使标准错误输出文件描述符（stderr 号 2）成为 1 号标准输出文件描述符（stdout）的副本（重定向 2 到 1 的当前引用）。我们得到如下示意图：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/c811f96e13a7846fe081bb35c33d2bfa.png"/>
    </p>
    <p>
     所有的标准输出描述符都重定向到 file 文件。
    </p>
    <p>
     然而，小心不要把
     <code>
      command &gt;file 2&gt;&amp;1
     </code>
     写成
     <code>
      command 2&gt;&amp;1 &gt;file
     </code>
     ，它们是不同的！
    </p>
    <p>
     Bash 对重定向指令的顺序敏感！后一命令仅仅把
     <strong>
      标准输出 1（stdout）
     </strong>
     重定向到 file 文件，而
     <strong>
      标准输出 2（stderr）
     </strong>
     仍然输出到终端。理解为什么，让我们再进行单步分析，看看发生了什么。初始示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/b36edc6be96d8ee7686890ed71d6f9aa.png"/>
    </p>
    <p>
     Bash 从左到右一步步处理重定向，首先遇到 “
     <strong>
      2&gt;&amp;1
     </strong>
     ” ，因此将
     <strong>
      2 (stderr)
     </strong>
     重定向与
     <strong>
      1(stdout)
     </strong>
     相同，即都指向终端，示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/b569bad4ae99e5eef6a396ad96fe78b9.png"/>
    </p>
    <p>
     现在，Bash 看到第 2 个重定向操作符 “
     <strong>
      &gt;file
     </strong>
     ” ，它重定向
     <strong>
      1(stdout)
     </strong>
     到 file 文件，示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/23317ce43ea7279ae346dccf07bd86f0.png"/>
    </p>
    <p>
     看到了什么，
     <strong>
      1(stdout)
     </strong>
     重定向到了 file 文件，但
     <strong>
      2(stderr)
     </strong>
     仍指向终端，所有写到
     <strong>
      stderr
     </strong>
     的都显示到屏幕。所以，在使用重定向时，要非常非常小心给出它的操作循序！
    </p>
    <p>
     另外在 Bash 中要注意：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&amp;&gt;</span>file
</code></pre>
    <p>
     操作含义，与下面命令形式完全相同：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&gt;&amp;</span><span class="token function">file</span>
</code></pre>
    <p>
     但推荐第一种形式写法。
    </p>

     <a id="5__744">

     <strong>
      5. 重定向文件内容作为命令的输入
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&lt;</span>file
</code></pre>
    <p>
     这里， Bash 在执行命令前，试图打开
     <strong>
      file
     </strong>
     文件并可读。若打开文件失败，命令失败退出。若打开读取文件成功，将使用打开的文件描述符作为命令的标准输入描述符（stdin 号 0）。
    </p>
    <p>
     上述动作完成后的标准输入输出描述符表示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/d98ff5857cce0b12751eaa159c083049.png"/>
    </p>
    <p>
     下面是个例子，假定你想从 file 中读取其第 1 行赋值给变量，命令如下：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-r</span> line <span class="token operator">&lt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     Bash 的内建命令
     <code>
      read
     </code>
     从标准输入读取单行数据。 使用输入重定向操作符
     <strong>
      &lt;
     </strong>
     使它从 file 文件中读取第 1 行数据。

    <h3>
     <a id="6__764">
     </a>
     <strong>
      6. 重定向多行文本到命令的输入
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&lt;&lt;</span>EOL
<span class="token operator">&gt;</span>你
<span class="token operator">&gt;</span>多行
<span class="token operator">&gt;</span>文本
<span class="token operator">&gt;</span>输入
<span class="token operator">&gt;</span>到这儿
<span class="token operator">&gt;</span>EOL不是唯一文字
<span class="token operator">&gt;</span>  EOL
<span class="token operator">&gt;</span>EOL
</code></pre>
    <p>
     这里，我们使用
     <strong>
      即入文档（here-document）
     </strong>
     重定向操作符 “
     <strong>
      &lt;&lt;MARKER
     </strong>
     ” 的功能。这个操作指示 bash 从标准输入（stdin 0）读取多行输入，直到某行（最后一行）只包含
     <strong>
      MARKER
     </strong>
     且无前导空白即退出输入。最后一行的终止输入标志不会附加到输入中。将最终的多行输入给命令。
    </p>
    <p>
     下面是个一般的例子。假设你拷贝了一堆的网上的网址（URLs）在系统剪贴板里，你想去掉所有的前缀
     <strong>
      http://
     </strong>
     ，快速的单行命令如下：
    </p>
    <pre><code class="prism language-bash">$ <span class="token function">sed</span> <span class="token string">'s|http://||'</span> <span class="token operator">&lt;&lt;</span><span class="token string">EOL
http://url1.com
http://url2.com
http://url3.com
EOL</span>
</code></pre>
    <p>
     这里使用即入文档重定向符，将多行网址记录输入给
     <code>
      sed
     </code>
     命令，“
     <strong>
      sed
     </strong>
     ” 命令使用正则表达式将所有行记录中的
     <strong>
      http://
     </strong>
     删除。
    </p>
    <p>
     以上例子输出：
    </p>
    <pre><code class="prism language-shell">url1.com
url2.com
url3.com
</code></pre>

    <h3>
     <a id="8__820">
     </a>
     <strong>
      8. 重定向所有的错误输出到一个文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span>file
$ command1
$ command2
$ <span class="token punctuation">..</span>.
</code></pre>
    <p>
     上述第一个命令行使用了 Bash 内建命令
     <code>
      exec
     </code>
     的功能，如果使用它重定向了标准输入输出（演示中是 2 号标准输出 stderr ），它在本 shell 中永久有效！除非你再次修改它或退出这个 shell 。
    </p>
    <p>
     例子中，使用
     <code>
      exec 2&gt;file
     </code>
     命令将
     <strong>
      2
     </strong>
     号标准输出（stderr）重定向到 file 文件。之后，在这个 shell 环境里所有命令的标准错误输出都重定向到 file 文件中。这是非常有用的一个技巧。当你想把所有命令或脚本中的运行记录到一个日志文件里，而又不想在每条命令中都繁琐地输入重定向操作。
    </p>
    <p>
     概括地讲，
     <code>
      exec
     </code>
     命令的主要功能是不创建子进程地调用一个 bash 命令或脚本并执行，若命令行中指定了命令或脚本，当前 shell 将被替换。我们演示的命令，在
     <code>
      exec
     </code>
     后并未给出任何命令，这仅仅利用了
     <code>
      exec
     </code>
     的执行重定向的功能，并无 shell 被替换。（译注：个人觉得
     <strong>
      exec
     </strong>
     是个较分裂的命令。推荐参考这个
     <a href="https://www.computerhope.com/unix/bash/exec.htm" rel="nofollow">
      链接
     </a>
     中对
     <strong>
      exec
     </strong>
     命令的解释。或直接看这个
     <a href="https://askubuntu.com/questions/525767/what-does-an-exec-command-do" rel="nofollow">
      AskUbuntu
     </a>
     中的回答）
    </p>
    <h3>
     <a id="9__835">
     </a>
     <strong>
      9. 创建用户的文件描述符做为自定义的输入描述符
     </strong>
    </h3>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&lt;</span>file
</code></pre>
    <p>
     这里，我们再次使用
     <code>
      exec
     </code>
     命令进行重定向的设置，指定
     <code>
      3&lt;file
     </code>
     的意思是打开 file 文件可读取，并将其分配给当前 shell 的
     <strong>
      3
     </strong>
     号输入文件描述符。当前 shell 的输入输出文件描述符示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/ea1dc0650e3ecd22ccb0fa315f3a670b.png"/>
    </p>
    <p>
     现在就可以从
     <strong>
      3
     </strong>
     号输入文件描述符读取，如下命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-u</span> <span class="token number">3</span> line
</code></pre>
    <p>
     上面的命令从 3 号输入文件描述符（重定向到 file 文件）中读取。
    </p>
    <p>
     或用常规命令，诸如
     <code>
      grep
     </code>
     操作 3 号输入文件描述符：
    </p>
    <pre><code class="prism language-shell">$ <span class="token function">grep</span> <span class="token string">"foo"</span> <span class="token operator">&lt;</span><span class="token file-descriptor important">&amp;3</span>
</code></pre>
    <p>
     这里，3 号输入文件描述符通过重定向代替了
     <strong>
      grep
     </strong>
     命令的默认 stdin 输入。一定要记住，资源有限，故使用完用户自打开的文件描述符后，要及时关闭它，释放描述符号。可以再次使用。
    </p>
    <p>
     使用完 3 号文件描述符后，可以关闭它，使用如下命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&gt;&amp;</span>-
</code></pre>
    <p>
     这里似乎 3 号文件描述符被重定向给
     <strong>
      &amp;-
     </strong>
     实际上，
     <strong>
      &amp;-
     </strong>
     就是
     <strong>
      关闭这个文件描述符
     </strong>
     的一种语法。
    </p>
    <h3>
     <a id="10__869">
     </a>
     <strong>
      10. 创建用户的文件描述符做为自定义的输出描述符
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">4</span>&gt;</span>file
</code></pre>
    <p>
     这里，简单地指示 bash 打开可写 file 文件，并将其文件描述符重定向成为 4 号输出描述符。当前 shell 的输入输出文件描述符示意图如下：
    </p>
    <p>
     <img alt="img" src="https://i-blog.csdnimg.cn/blog_migrate/604b145d321771a161582ef3a2d53c01.png"/>
    </p>
    <p>
     正如你在上图所见，输入输出文件描述符并未按顺序来，用户可自由指定 0 只 255 之间的数字做为自定义打开的文件描述符号。
    </p>
    <p>
     现在，简单往 4 号输出文件描述符中写入：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token string">"foo"</span> <span class="token operator">&gt;</span><span class="token file-descriptor important">&amp;4</span>
</code></pre>
    <p>
     同样，可以关闭 4 号输出文件描述符：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">4</span>&gt;&amp;</span>-
</code></pre>
    <p>
     学会使用自定义输入输出文件描述符，一切都是那么简单！
    </p>
    <h3>
     <a id="11__895">
     </a>
     <strong>
      11. 打开文件用于读写
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&lt;&gt;</span>file
</code></pre>
    <p>
     这里使用 bash 的菱形操作符
     <code>
      &lt;&gt;
     </code>
     ，菱形操作符打开 file 文件用于读写。
    </p>
    <p>
     所以，可以执行如下例子：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token string">"foo bar"</span> <span class="token operator">&gt;</span> <span class="token function">file</span>   <span class="token comment"># 将字符串  "foo bar" 写入 file 文件。</span>
$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">5</span>&lt;&gt;</span> <span class="token function">file</span>           <span class="token comment"># 以读写方式打开 file 文件并重定向到5号描述符</span>
$ <span class="token builtin class-name">read</span> <span class="token parameter variable">-n</span> <span class="token number">3</span> var <span class="token operator">&lt;</span><span class="token file-descriptor important">&amp;5</span>       <span class="token comment"># 从5号输入输出描述符中读取前3个字符。</span>
$ <span class="token builtin class-name">echo</span> <span class="token variable">$var</span>
</code></pre>
    <p>
     上面最后命令将输出 “
     <strong>
      foo
     </strong>
     ” ，只读出前 3 个字符。
    </p>
    <p>
     也可写一些内容到 file 文件:
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token parameter variable">-n</span> + <span class="token operator">&gt;</span><span class="token file-descriptor important">&amp;5</span>           <span class="token comment"># 在文件中第4字符位置写入 "+"</span>
$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">5</span>&gt;&amp;</span>-               <span class="token comment"># 关闭5号文件描述符</span>
$ <span class="token function">cat</span> <span class="token function">file</span>
</code></pre>
    <p>
     上面会输出
     <strong>
      foo+bar
     </strong>
     。（译注：实践发现以永久重定向文件描述符读写文件，会保持读写位置）
    </p>

     <strong>
      12. 多个命令输出重定向到文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token punctuation">(</span>command1<span class="token punctuation">;</span> command2<span class="token punctuation">)</span> <span class="token operator">&gt;</span>file

    <p>

     <code>
      (commands)
     </code>
     运行命令操作，其中 bash 会将

    <p>
     所以这里的
     <code>

    <h3>
     <a id="13__shell__934">
     </a>
     <strong>
      13. 通过文件 shell 之间传递命令执行
     </strong>
    </h3>
    <p>
     打开 2 个 shell 模拟终端，在第 1 个 shell 中，输入命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token function">mkfifo</span> fifo
$ <span class="token builtin class-name">exec</span> <span class="token operator">&lt;</span> fifo
</code></pre>
    <p>
     在第 2 个 shell 中，输入命令：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&gt;</span> fifo<span class="token punctuation">;</span> <span class="token builtin class-name">echo</span> <span class="token string">'echo test'</span> <span class="token operator">&gt;</span><span class="token file-descriptor important">&amp;3</span>
</code></pre>
    <p>
     现在看第 1 个 shell ，发现
     <code>
      echo test
     </code>
     命令执行输出在第 1 个 shell 里。你可以在第 2 个 shell 中发送命令字符串给 3 号描述符的方式给第 1 个 shell 发送命令。
    </p>
    <p>
     下面讲解它是如何做到的。
    </p>
    <p>
     在 shell 1 中，使用
     <code>
      mkfifo
     </code>
     创建
     <strong>
      命名管道
     </strong>
     –
     <strong>
      fifo
     </strong>
     。一个命名管道（也称为 FIFO）性质与一般的管道相同，除了它是通过系统文件系统访问之外（创建命名管道将会在当前目录下创建一个以命名管道名命名的文件，这个文件第一个属性为
     <strong>
      p
     </strong>
     表示它为一个
     <strong>
      管道（pipe）
     </strong>
     ，并且文件大小始终显示为 0）。它可供多进程读写（可供进程间通讯）。当进程间通过
     <strong>
      FIFO
     </strong>
     – 命名管道 – 交换数据时，系统内核并不会把数据写入文件存储系统。因此，命名管道文件大小永远为 0 。仅仅把文件系统中的文件名做为进程引用它的一个名称罢了。
    </p>
    <p>
     下一命令
     <code>
      exec &lt; fifo
     </code>
     重定向 shell 1 的输入为
     <strong>
      fifo
     </strong>
     。
    </p>
    <p>
     接着，shell 2 重定向这个命名管道做为用户输出文件描述符，并分配号为 3 。然后给 3 号输出发送字符串
     <code>
      echo test
     </code>
     ，这将传给
     <strong>
      fifo
     </strong>
     文件。
    </p>
    <p>
     因此，shell 1 连接到
     <strong>
      fifo
     </strong>
     的标准输入进入 shell 1 命令行并执行！很容易吧！
    </p>
    <p>
     （译注：原理很简单，自己开两个 shell 去实践练习一下。别的不多说，只提醒一点：当 shell 1 把自己的标准输入重定向成命名管道之后，它就不能接受它自己终端的输入了。如何恢复它回到原始的输入？2 个办法，1. 是重定向前备份。2. 是使用它原始的名称恢复，提示：输入仅可用
     <strong>
      /dev/tty
     </strong>
     。至于怎样让它执行命令？自己想想，你棒棒哒！）
    </p>
    <h3>
     <a id="14__bash__963">
     </a>
     <strong>
      14. 通过 bash 访问网站
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&lt;&gt;</span>/dev/tcp/www.bing.com/80
$ <span class="token builtin class-name">echo</span> <span class="token parameter variable">-e</span> <span class="token string">"GET / HTTP/1.1<span class="token entity" title="\n">\n</span><span class="token entity" title="\n">\n</span>"</span> <span class="token operator">&gt;</span><span class="token file-descriptor important">&amp;3</span>
$ <span class="token function">cat</span> <span class="token operator">&lt;</span><span class="token file-descriptor important">&amp;3</span>
</code></pre>
    <p>
     Bash 处理
     <strong>
      /dev/tcp/host/port
     </strong>
     作为特殊文件。它不需要在你的文件系统中存在。这个只为 Bash 通过它打开指定主机的网络接口。
    </p>
    <p>
     以上例子，首先打开可读写 3 号输入输出文件描述符指向
     <strong>
      /dev/tcp/www.bing.com/80
     </strong>
     ，它将链接到
     <strong>
      www.bing.com
     </strong>
     网站的端口 80 。
    </p>
    <p>
     下一步，向 3 号文件描述符写入 ‘
     <strong>
      GET / HTTP/1.1\n\n
     </strong>
     ’ （发送 HTTP 请求），然后使用
     <code>
      cat
     </code>
     命令简单读取 3 号文件描述符的内容。
    </p>
    <p>
     类似地，你可以通过
     <strong>
      /dev/udp/host/port
     </strong>
     创建特殊文件用于 UDP 连接。
    </p>
    <p>
     使用
     <strong>
      /dev/tcp/host/port
     </strong>
     的方法，你甚至可以在 Bash 中写出端口扫描的命令或脚本！
    </p>
    <p>
     （译注：原文示例使用的是谷歌，我这里换成了必应。实验了一下，可能由于现在网站都使用 SSL 的缘故，第一次会获得一个包含错误提示的 HTTP 响应内容，连接就关闭了）
    </p>
    <h3>
     <a id="15__983">
     </a>
     <strong>
      15. 当重定向输出时阻止写入已有文件
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">set</span> <span class="token parameter variable">-o</span> noclobber
</code></pre>
    <p>
     上面的命令打开当前 shell 的
     <strong>
      noclobber
     </strong>
     选项。这个选项阻止当前 shell 使用重定向 ‘
     <strong>
      &gt;
     </strong>
     ’ 操作覆盖写入已有的文件。
    </p>
    <p>
     如果你输出重定向的文件是一个已有文件，将会得到一个错误提示：
    </p>
    <pre><code class="prism language-shell">$ <span class="token operator">&gt;</span> exFile
$ <span class="token builtin class-name">echo</span> <span class="token builtin class-name">test</span> <span class="token operator">&gt;</span> exFile
bash: exFile: cannot overwrite existing <span class="token function">file</span>
<span class="token comment">#bash: exFile: 不能覆写已有文件</span>
</code></pre>
    <p>
     如果你完全确定就是要覆盖写入已有文件，可以使用
     <strong>
      &gt;|
     </strong>
     重定向符：
    </p>
    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> <span class="token builtin class-name">test</span> <span class="token operator">&gt;|</span> exFile
$ <span class="token function">cat</span> exFile
<span class="token builtin class-name">test</span>
</code></pre>
    <p>
     这个操作符成功超越
     <strong>
      noclobber
     </strong>
     选项。
    </p>

     <strong>

     <strong>
      20. 重定向操作的顺序
     </strong>
    </h3>
    <p>
     你可以把重定向操作放到命令行的任何位置。看看下面 3 个例子，它们效果一样：

    <pre><code class="prism language-bash">$ <span class="token builtin class-name">echo</span> hello <span class="token operator">&gt;</span>/tmp/example

$ <span class="token operator">&gt;</span>/tmp/example <span class="token builtin class-name">echo</span> hello
</code></pre>
    <p>
     喜欢上了 bash ！

    <h3>
     <a id="22__stdout_stderr__1120">
     </a>
     <strong>
      22. 重定向 stdout 到一个进程，stderr 到另一进程
     </strong>
    </h3>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&gt;</span> <span class="token operator">&gt;</span><span class="token punctuation">(</span>stdout_cmd<span class="token punctuation">)</span> <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span> <span class="token operator">&gt;</span><span class="token punctuation">(</span>stderr_cmd<span class="token punctuation">)</span>
</code></pre>
    <p>
     这行命令使用了重定向命令替换展开操作符
     <code>
      &gt;()
     </code>
     。它将会运行
     <strong>
      ()
     </strong>
     中的命令。而其标准输入通过匿名管道连接到了
     <code>
      command
     </code>
     的标准输出。接着下一个
     <code>
      &gt;()
     </code>
     操作符将其标准输入连接到了
     <code>
      command
     </code>
     的标准错误输出。
    </p>
    <p>
     对于上面的例子，第一个命令替换
     <code>
      &gt;(stdout_cmd)
     </code>
     可能使 bash 返回
     <code>
      /dev/fd/60
     </code>
     文件描述符。同时，第二个命令替换
     <code>
      &gt;(stderr_cmd)
     </code>
     可能使 bash 返回
     <code>
      /dev/fd/61
     </code>
     文件描述符。这 2 个文件描述符由 bash 实时创建为等待读取的命名管道。它们都等待某些命令进程往里写，以便它们读取。
    </p>
    <p>
     所以，上述命令展开后可能是这样：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">command</span> <span class="token operator">&gt;</span> /dev/fd/60 <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span> /dev/fd/61
</code></pre>
    <p>
     现在可看得清楚些，
     <strong>
      command
     </strong>
     的 stdout 重定向到
     <strong>
      /dev/fd/60
     </strong>
     ， stderr 重定向到
     <strong>
      /dev/fd/61
     </strong>
     。
    </p>
    <p>
     当
     <strong>
      command
     </strong>
     输出，内部进程 ‘
     <strong>
      stdout_cmd
     </strong>
     ’ 的命令执行将接受。当
     <strong>
      command
     </strong>
     的错误输出，内部进程 ‘
     <strong>
      stderr_cmd
     </strong>
     ’ 的命令执行将接受。
    </p>
    <h3>
     <a id="23__1140">
     </a>
     <strong>
      23. 找出每个管道命令的退出码
     </strong>
    </h3>
    <p>
     让我们看一下几个命令用管道串起来的例子，如下命令：
    </p>
    <pre><code class="prism language-shell">$ cmd1 <span class="token operator">|</span> cmd2 <span class="token operator">|</span> cmd3 <span class="token operator">|</span> cmd4
</code></pre>
    <p>
     你想找出这里每个命令的退出状态码，该如何做？有没有一个简便的方法获取每个命令的退出状态码，而不是 bash 简单给出的最后一条命令的退出状态码。
    </p>
    <p>
     Bash 的开发者想到了这点，他们加入了一个名为 ‘
     <strong>
      PIPESTATUS
     </strong>
     ’ 的数组变量来保留管道命令流中每个命令的运行退出状态码。
    </p>
    <p>
     数组变量
     <strong>
      PIPESTATUS
     </strong>
     中的每个数字都对应于相应位置命令的退出状态码。下面是个例子：
    </p>
    <pre><code class="prism language-shell">$ <span class="token builtin class-name">echo</span> <span class="token string">'men are cool'</span> <span class="token operator">|</span> <span class="token function">grep</span> <span class="token string">'moo'</span> <span class="token operator">|</span> <span class="token function">sed</span> <span class="token string">'s/o/x/'</span> <span class="token operator">|</span> <span class="token function">awk</span> <span class="token string">'{ print $1 }'</span>
$ <span class="token builtin class-name">echo</span> <span class="token variable">${<!-- --><span class="token environment constant">PIPESTATUS</span><span class="token punctuation">[</span>@<span class="token punctuation">]</span>}</span>
<span class="token number">0</span> <span class="token number">1</span> <span class="token number">0</span> <span class="token number">0</span>
</code></pre>
    <p>
     上面例子中，命令
     <code>
      grep 'moo'
     </code>
     失败，因此数组变量
     <strong>
      PIPESTATUS
     </strong>
     中的第 2 个数为 1 。
    </p>

    <h3>
     <a id="stdin_stdout_stderr_1283">
     </a>
     stdin, stdout, stderr
    </h3>
    <p>
     stdin、stdout、stderr
    </p>
    <p>
     When Bash starts, normally, 3 file descriptors are opened,
     <code>
      0
     </code>
     ,
     <code>
      1
     </code>
     and
     <code>
      2
     </code>
     also known as standard input (
     <code>
      stdin
     </code>
     ), standard output (
     <code>
      stdout
     </code>
     ) and standard error (
     <code>
      stderr
     </code>
     ).
     <br/>
     当 Bash 启动时，通常会打开 3 个文件描述符，
     <code>
      0
     </code>
     、
     <code>
      1
     </code>
     和
     <code>
      2
     </code>
     也称为标准输入 (
     <code>
      stdin
     </code>
     )、标准输出 (
     <code>
      stdout
     </code>
     ) 和标准错误 (
     <code>
      stderr
     </code>
     )。
    </p>
    <p>
     For example, with Bash running in a Linux terminal emulator, you’ll see:
     <br/>

     </code>

    <h2>
     <a id="Simple_Redirections_1318">
     </a>
     Simple Redirections
    </h2>
    <p>
     简单重定向
    </p>
    <h3>
     <a id="Output_Redirection_n_file_1321">
     </a>
     Output Redirection “n&gt; file”
    </h3>
    <p>
     输出重定向 “n&gt; file”
    </p>
    <p>
     <code>
      &gt;
     </code>
     is probably the simplest redirection.
     <br/>
     <code>
      &gt;
     </code>
     可能是最简单的重定向。
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">echo</span> foo <span class="token operator">&gt;</span> <span class="token function">file</span>
</code></pre>
    <p>
     the
     <code>
      &gt; file
     </code>
     after the command alters the file descriptors belonging to the command
     <code>
      echo
     </code>
     . It changes the file descriptor
     <code>
      1
     </code>
     (
     <code>
      &gt; file
     </code>
     is the same as
     <code>
      1&gt;file
     </code>
     ) so that it points to the file
     <code>
      file
     </code>
     . They will look like:
     <br/>
     命令后面的
     <code>
      &gt; file
     </code>
     会改变属于命令
     <code>
      echo
     </code>
     的文件描述符。它更改文件描述符
     <code>
      1
     </code>
     （
     <code>
      &gt; file
     </code>
     与
     <code>
      1&gt;file
     </code>
     相同），使其指向文件
     <code>
      file
     </code>
     。它们将如下所示：
    </p>
    <pre><code class="prism language-shell">  ---       +-----------------------+
standard input   <span class="token punctuation">(</span> <span class="token number">0</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard output  <span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> <span class="token function">file</span>  <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard error   <span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
</code></pre>
    <p>
     Now characters written by our command,
     <code>
      echo
     </code>
     , that are sent to the standard output, i.e., the file descriptor
     <code>
      1
     </code>
     , end up in the file named
     <code>
      file
     </code>
     .
     <br/>
     现在，由我们的命令
     <code>
      echo
     </code>
     写入的字符被发送到标准输出，即文件描述符
     <code>
      1
     </code>
     ，最终出现在名为
     <code>
      file
     </code>
     的文件中。
    </p>
    <p>
     In the same way, command
     <code>
      2&gt; file
     </code>
     will change the standard error and will make it point to
     <code>
      file
     </code>
     . Standard error is used by applications to print errors.
     <br/>
     同样，命令
     <code>
      2&gt; file
     </code>
     将更改标准误差并使其指向
     <code>
      file
     </code>
     。应用程序使用标准错误来打印错误。
    </p>
    <p>
     What will
     <code>
      command 3&gt; file
     </code>
     do? It will open a new file descriptor pointing to
     <code>
      file
     </code>
     . The command will then start with:
     <br/>
     <code>
      command 3&gt; file
     </code>
     会做什么？它将打开一个指向
     <code>
      file
     </code>
     的新文件描述符。然后，该命令将以以下内容开头：
    </p>
    <pre><code class="prism language-shell">  ---       +-----------------------+
standard input   <span class="token punctuation">(</span> <span class="token number">0</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard output  <span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard error   <span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
new descriptor   <span class="token punctuation">(</span> <span class="token number">3</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> <span class="token function">file</span>  <span class="token operator">|</span>
  ---       +-----------------------+
</code></pre>
    <p>
     What will the command do with this descriptor? It depends. Often nothing. We will see later why we might want other file descriptors.
     <br/>
     命令将对此描述符执行什么操作？这要看情况。通常什么都没有。我们稍后会明白为什么我们可能需要其他文件描述符。
    </p>
    <h3>
     <a id="Input_Redirection_n_file_1373">
     </a>
     Input Redirection “n&lt; file”
    </h3>
    <p>
     输入重定向 “n&lt; file”
    </p>
    <p>
     When you run a command using
     <code>
      command &lt; file
     </code>
     , it changes the file descriptor
     <code>
      0
     </code>
     so that it looks like:
     <br/>
     当您使用
     <code>
      command &lt; file
     </code>
     运行命令时，它会更改文件描述符
     <code>
      0
     </code>
     ，使其如下所示：
    </p>
    <pre><code class="prism language-shell">  ---       +-----------------------+
standard input   <span class="token punctuation">(</span> <span class="token number">0</span> <span class="token punctuation">)</span> <span class="token operator">&lt;</span>----<span class="token operator">|</span> <span class="token function">file</span>  <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard output  <span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
standard error   <span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5            <span class="token operator">|</span>
  ---       +-----------------------+
</code></pre>
    <p>
     If the command reads from
     <code>
      stdin
     </code>
     , it now will read from
     <code>
      file
     </code>
     and not from the console.
     <br/>
     如果命令从
     <code>
      stdin
     </code>
     读取，它现在将从
     <code>
      file
     </code>
     读取，而不是从控制台读取。
    </p>
    <p>
     As with
     <code>
      &gt;
     </code>
     ,
     <code>
      &lt;
     </code>
     can be used to open a new file descriptor for reading,
     <code>
      command 3&lt;file
     </code>
     . Later we will see how this can be useful.
     <br/>
     与
     <code>
      &gt;
     </code>
     一样，
     <code>
      &lt;
     </code>
     可用于打开新的文件描述符进行读取，
     <code>
      command 3&lt;file
     </code>
     。稍后我们将看到这有什么用处。
    </p>
    <h3>
     <a id="Pipes__1397">
     </a>
     Pipes |
    </h3>
    <p>
     管道 |
    </p>
    <p>
     What does this
     <code>
      |
     </code>
     do? Among other things, it connects the standard output of the command on the left to the standard input of the command on the right. That is, it creates a special file, a pipe, which is opened as a write destination for the left command, and as a read source for the right command.
     <br/>
     这个
     <code>
      |
     </code>
     有什么作用？此外，它还将左侧命令的标准输出连接到右侧命令的标准输入。也就是说，它会创建一个特殊文件，即管道，该文件将作为左侧命令的写入目标打开，并作为右侧命令的读取源打开。
    </p>
    <pre><code class="prism language-shell">           <span class="token builtin class-name">echo</span> foo               <span class="token operator">|</span><span class="token function">cat</span>
 ---       +--------------+               ---       +--------------+

 ---       +--------------+    /          ---       +--------------+
              /
 ---       +--------------+  /            ---       +--------------+
<span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> pipe <span class="token punctuation">(</span>write<span class="token punctuation">)</span> <span class="token operator">|</span> /            <span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts     <span class="token operator">|</span>
 ---       +--------------+               ---       +--------------+
 ---       +--------------+               ---       +--------------+
<span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5   <span class="token operator">|</span>              <span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/    <span class="token operator">|</span>
 ---       +--------------+               ---       +--------------+
</code></pre>
    <p>
     This is possible because the redirections are set up by the shell
     <strong>
      before
     </strong>
     the commands are executed, and the commands inherit the file descriptors.
     <br/>
     这是可能的，因为重定向是在执行命令之前由 shell 设置的，并且命令继承了文件描述符。
    </p>
    <h2>
     <a id="More_On_File_Descriptors_1420">
     </a>
     More On File Descriptors
    </h2>
    <p>
     有关文件描述符的更多信息
    </p>
    <h3>
     <a id="Duplicating_File_Descriptor_21_1423">
     </a>
     Duplicating File Descriptor 2&gt;&amp;1
    </h3>
    <p>
     复制文件描述符 2&gt;&amp;1
    </p>
    <p>
     We have seen how to open (or redirect) file descriptors. Let us see how to duplicate them, starting with the classic
     <code>
      2&gt;&amp;1
     </code>
     . What does this mean? That something written on the file descriptor
     <code>
      2
     </code>
     will go where file descriptor
     <code>
      1
     </code>
     goes. In a shell
     <code>
      command 2&gt;&amp;1
     </code>
     is not a very interesting example so we will use
     <code>
      ls /tmp/ doesnotexist 2&gt;&amp;1 | less
     </code>
     <br/>
     我们已经了解了如何打开（或重定向）文件描述符。让我们看看如何复制它们，从经典的
     <code>
      2&gt;&amp;1
     </code>
     开始。这是什么意思？文件描述符
     <code>
      2
     </code>
     上写入的内容将转到文件描述符
     <code>
      1
     </code>
     所在的位置。在 shell 中
     <code>
      command 2&gt;&amp;1
     </code>
     不是一个非常有趣的例子，所以我们将使用
     <code>
      ls /tmp/ doesnotexist 2&gt;&amp;1 | less
     </code>
    </p>
    <pre><code class="prism language-shell">   <span class="token function">ls</span> /tmp/ doesnotexist <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span>     <span class="token operator">|</span>   <span class="token function">less</span>
 ---       +--------------+              ---       +--------------+
<span class="token punctuation">(</span> <span class="token number">0</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/5   <span class="token operator">|</span>     ------<span class="token operator">&gt;</span> <span class="token punctuation">(</span> <span class="token number">0</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span>from the pipe <span class="token operator">|</span>
 ---       +--------------+    /   ---<span class="token operator">&gt;</span>  ---       +--------------+
              /   /
 ---       +--------------+  /   /       ---       +--------------+
<span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> to the pipe  <span class="token operator">|</span> /   /       <span class="token punctuation">(</span> <span class="token number">1</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span>  /dev/pts    <span class="token operator">|</span>
 ---       +--------------+    /         ---       +--------------+
      	       	       	      /
 ---       +--------------+  /           ---       +--------------+
<span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span>  to the pipe <span class="token operator">|</span> /           <span class="token punctuation">(</span> <span class="token number">2</span> <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /dev/pts/    <span class="token operator">|</span>
 ---       +--------------+              ---       +--------------+
</code></pre>
    <p>
     Why is it called
     <em>
      duplicating
     </em>
     ? Because after
     <code>
      2&gt;&amp;1
     </code>
     , we have 2 file descriptors pointing to the same file. Take care not to call this “File Descriptor Aliasing”; if we redirect
     <code>
      stdout
     </code>
     after
     <code>
      2&gt;&amp;1
     </code>
     to a file
     <code>
      B
     </code>
     , file descriptor
     <code>
      2
     </code>
     will still be opened on the file
     <code>
      A
     </code>
     where it was. This is often misunderstood by people wanting to redirect both standard input and standard output to the file. Continue reading for more on this.
     <br/>
     为什么叫复制？因为在
     <code>
      2&gt;&amp;1
     </code>
     之后，我们有 2 个文件描述符指向同一个文件。注意不要将此称为“文件描述符别名”；如果我们将
     <code>
      2&gt;&amp;1
     </code>
     之后的
     <code>
      stdout
     </code>
     重定向到文件
     <code>
      B
     </code>
     ，文件描述符
     <code>
      2
     </code>
     仍将在文件
     <code>
      A
     </code>
     上打开。想要将标准输入和标准输出重定向到文件的人经常误解这一点。继续阅读以了解更多信息。
    </p>
    <p>
     So if you have two file descriptors
     <code>
      s
     </code>
     and
     <code>
      t
     </code>
     like:
     <br/>
     因此，如果你有两个文件描述符
     <code>
      s
     </code>
     和
     <code>
      t
     </code>
     ，如下所示：
    </p>
    <pre><code class="prism language-shell">  ---       +-----------------------+
 a descriptor    <span class="token punctuation">(</span> s <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /some/file            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
 a descriptor    <span class="token punctuation">(</span> t <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /another/file         <span class="token operator">|</span>
  ---       +-----------------------+
</code></pre>
    <p>
     Using a
     <code>
      t&gt;&amp;s
     </code>
     (where
     <code>
      t
     </code>
     and
     <code>
      s
     </code>
     are numbers) it means:
     <br/>
     使用
     <code>
      t&gt;&amp;s
     </code>
     （其中
     <code>
      t
     </code>
     和
     <code>
      s
     </code>
     是数字） 表示：
    </p>
    <blockquote>
     <p>
      Copy whatever file descriptor
      <code>
       s
      </code>
      contains into file descriptor
      <code>
       t
      </code>
      <br/>
      将 s 包含的任何文件描述符复制到文件描述符 t 中
     </p>
    </blockquote>
    <p>
     So you got a copy of this descriptor:
     <br/>
     所以你得到了这个描述符的副本：
    </p>
    <pre><code class="prism language-shell">  ---       +-----------------------+
 a descriptor    <span class="token punctuation">(</span> s <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /some/file            <span class="token operator">|</span>
  ---       +-----------------------+
  ---       +-----------------------+
 a descriptor    <span class="token punctuation">(</span> t <span class="token punctuation">)</span> ----<span class="token operator">&gt;|</span> /some/file            <span class="token operator">|</span>
  ---       +-----------------------+
</code></pre>
    <p>
     Internally each of these is represented by a file descriptor opened by the operating system’s
     <code>
      fopen
     </code>
     calls, and is likely just a pointer to the file which has been opened for reading (
     <code>
      stdin
     </code>
     or file descriptor
     <code>
      0
     </code>
     ) or writing (
     <code>
      stdout
     </code>
     /
     <code>
      stderr
     </code>
     ).
     <br/>
     在内部，它们中的每一个都由操作系统的
     <code>
      fopen
     </code>
     调用打开的文件描述符表示，并且可能只是一个指向已打开进行读取（
     <code>
      stdin
     </code>
     或文件描述符
     <code>
      0
     </code>
     ）或写入（
     <code>
      stdout
     </code>
     /
     <code>
      stderr
     </code>
     ）的文件的指针。
    </p>
    <p>
     Note that the file reading or writing positions are also duplicated. If you have already read a line of
     <code>
      s
     </code>
     , then after
     <code>
      t&gt;&amp;s
     </code>
     if you read a line from
     <code>
      t
     </code>
     , you will get the second line of the file.
     <br/>
     请注意，文件读取或写入位置也是重复的。如果你已经阅读了
     <code>
      s
     </code>
     行，那么在
     <code>

    <h3>
     <a id="Why_sed_sfoobar_file_file_Doesnt_Work_1589">
     </a>
     Why sed ‘s/foo/bar/’ file &gt;file Doesn’t Work
    </h3>
    <p>
     为什么 sed ‘s/foo/bar/’ 文件 &gt; 文件不起作用
    </p>
    <p>
     This is a common error, we want to modify a file using something that reads from a file and writes the result to
     <code>
      stdout
     </code>
     . To do this, we redirect stdout to the file we want to modify. The problem here is that, as we have seen, the redirections are setup before the command is actually executed.
     <br/>
     这是一个常见的错误，我们想要使用从文件中读取并将结果写入
     <code>
      stdout
     </code>
     的内容来修改文件。为此，我们将 stdout 重定向到我们想要修改的文件。这里的问题是，正如我们所看到的，重定向是在实际执行命令之前设置的。
    </p>
    <p>
     So
     <strong>
      BEFORE
     </strong>
     sed starts, standard output has already been redirected, with the additional side effect that, because we used &gt;, “file” gets truncated. When
     <code>
      sed
     </code>
     starts to read the file, it contains nothing.
     <br/>
     因此，在 sed 启动之前，标准输出已经被重定向，还有一个额外的副作用，因为我们使用了 &gt;，“file” 被截断。当
     <code>
      sed
     </code>
     开始读取文件时，它不包含任何内容。
    </p>

    <h3>
     <a id="Closing_The_File_Descriptors_1682">
     </a>
     Closing The File Descriptors
    </h3>
    <p>
     关闭文件描述符
    </p>
    <p>
     Closing a file through a file descriptor is easy, just make it a duplicate of -. For instance, let’s close
     <code>
      stdin &lt;&amp;-
     </code>
     and
     <code>
      stderr 2&gt;&amp;-
     </code>
     :
     <br/>
     通过文件描述符关闭文件很容易，只需将其与 - 重复即可。例如，让我们关闭
     <code>
      stdin &lt;&amp;-
     </code>
     和
     <code>
      stderr 2&gt;&amp;-
     </code>
     ：
    </p>
    <pre><code class="prism language-shell"> <span class="token function">bash</span> <span class="token parameter variable">-c</span> <span class="token string">'{ lsof -a -p $$ -d0,1,2 ;} &lt;&amp;- 2&gt;&amp;-'</span>
 COMMAND   PID <span class="token environment constant">USER</span>   FD   TYPE DEVICE SIZE NODE NAME
 <span class="token function">bash</span>    <span class="token number">10668</span> pgas    1u   CHR  <span class="token number">136,2</span>         <span class="token number">4</span> /dev/pts/2
</code></pre>
    <p>
     we see that inside the
     <code>
      {}
     </code>
     that only
     <code>
      1
     </code>
     is still here.
     <br/>
     Though the OS will probably clean up the mess, it is perhaps a good idea to close the file descriptors you open. For instance, if you open a file descriptor with
     <code>
      exec 3&gt;file
     </code>
     , all the commands afterwards will inherit it. It’s probably better to do something like:
     <br/>
     尽管操作系统可能会清理混乱，但关闭您打开的文件描述符可能是个好主意。例如，如果你用
     <code>
      exec 3&gt;file
     </code>
     打开一个文件描述符，之后的所有命令都将继承它。执行如下操作可能更好：
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&gt;</span>file
<span class="token punctuation">..</span><span class="token punctuation">..</span>
<span class="token comment">#commands that uses 3</span>
<span class="token punctuation">..</span><span class="token punctuation">..</span>
<span class="token builtin class-name">exec</span> <span class="token operator"><span class="token file-descriptor important">3</span>&gt;&amp;</span>-
<span class="token comment">#we don't need 3 any more</span>
</code></pre>
    <p>
     I’ve seen some people using this as a way to discard, say stderr, using something like: command 2&gt;&amp;-. Though it might work, I’m not sure if you can expect all applications to behave correctly with a closed stderr.
     <br/>
     我见过有些人用这个作为丢弃的方法，比如 stderr，使用类似命令 2&gt;&amp;- 的东西。虽然它可能有效，但我不确定您是否可以期望所有应用程序在关闭 stderr 的情况下都能正常运行。
    </p>
    <p>
     When in doubt, I use 2&gt;/dev/null.
     <br/>
     如有疑问，我使用 2&gt;/dev/null。
    </p>

    <h2>
     <a id="Syntax_1844">
     </a>
     Syntax
    </h2>
    <p>
     语法
    </p>
    <p>
     I used to have trouble choosing between
     <code>
      0&amp;&lt;3
     </code>
     <code>
      3&amp;&gt;1
     </code>
     <code>
      3&gt;&amp;1
     </code>
     <code>
      -&gt;2
     </code>
     <code>
      -&lt;&amp;0
     </code>
     <code>
      &amp;-&lt;0
     </code>
     <code>
      0&lt;&amp;-
     </code>
     etc… (I think probably because the syntax is more representative of the result, i.e., the redirection, than what is done, i.e., opening, closing, or duplicating file descriptors).
     <br/>
     我以前在
     <code>
      0&amp;&lt;3
     </code>
     <code>
      3&amp;&gt;1
     </code>
     <code>
      3&gt;&amp;1
     </code>
     <code>
      -&gt;2
     </code>
     <code>
      -&lt;&amp;0
     </code>
     <code>
      &amp;-&lt;0
     </code>
     <code>
      0&lt;&amp;-
     </code>
     等之间很难选择…（我认为可能是因为语法更能代表结果，即重定向，而不是所做的，即打开、关闭或复制文件描述符）。
    </p>
    <p>
     If this fits your situation, then maybe the following “rules” will help you, a redirection is always like the following:
     <br/>
     如果这符合您的情况，那么以下 “规则” 可能会对您有所帮助，重定向总是如下所示：
    </p>
    <pre><code class="prism language-shell"> lhs <span class="token function">op</span> rhs
</code></pre>
    <ul>
     <li>
      <code>
       lhs
      </code>
      is always a file description, i.e., a number:
      <ul>
       <li>
        Either we want to open, duplicate, move or we want to close. If the op is
        <code>
         &lt;
        </code>
        then there is an implicit 0, if it’s
        <code>
         &gt;
        </code>
        or
        <code>
         &gt;&gt;
        </code>
        , there is an implicit 1.
       </li>
       <li>
        要么我们想要打开、复制、移动，要么我们想要关闭。如果 op 为 &lt;，则存在隐式 0，如果为 &gt; 或 &gt;&gt;，则存在隐式 1。
       </li>
      </ul>
     </li>
     <li>
      <code>
       op
      </code>
      is
      <code>
       &lt;
      </code>
      ,
      <code>
       &gt;
      </code>
      ,
      <code>
       &gt;&gt;
      </code>
      ,
      <code>
       &gt;|
      </code>
      , or
      <code>
       &lt;&gt;
      </code>
      :
      <ul>
       <li>
        <code>
         &lt;
        </code>
        if the file decriptor in
        <code>
         lhs
        </code>
        will be read,
        <code>
         &gt;
        </code>
        if it will be written,
        <code>
         &gt;&gt;
        </code>
        if data is to be appended to the file,
        <code>
         &gt;|
        </code>
        to overwrite an existing file or
        <code>
         &lt;&gt;
        </code>
        if it will be both read and written.
       </li>
       <li>
        &lt;是否将读取 LHS 中的文件描述器，&gt;是否将写入该文件，&gt;&gt;是否将数据附加到文件中，&gt;| 覆盖现有文件，&lt;&gt;是否将读取和写入该文件。
       </li>
      </ul>
     </li>
     <li>
      <code>
       rhs
      </code>
      is the thing that the file descriptor will describe:
      <ul>
       <li>
        It can be the name of a file, the place where another descriptor goes (
        <code>
         &amp;1
        </code>
        ), or,
        <code>
         &amp;-
        </code>
        , which will close the file descriptor.
       </li>
       <li>
        它可以是文件名、另一个描述符所在的位置 （&amp;1） 或 &amp;-，这将关闭文件描述符。
        <br/>
        You might not like this description, and find it a bit incomplete or inexact, but I think it really helps to easily find that, say
        <code>
         &amp;-&gt;0
        </code>
        is incorrect.
        <br/>
        你可能不喜欢这个描述，觉得它有点不完整或不精确，但我认为很容易找到它真的很有帮助，比如说
        <code>
         &amp;-&gt;0
        </code>
        是不正确的。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="A_note_on_style_1869">
     </a>
     A note on style
    </h4>
    <p>
     关于风格的说明
    </p>
    <p>
     The shell is pretty loose about what it considers a valid redirect. While opinions probably differ, this author has some (strong) recommendations:
     <br/>
     shell 对它认为有效的重定向非常宽松。虽然意见可能不同，但笔者提出了一些（强烈）的建议：
    </p>
    <ul>
     <li>
      <strong>
       Always
      </strong>
      keep redirections “tightly grouped” – that is,
      <strong>
       do not
      </strong>
      include whitespace anywhere within the redirection syntax except within quotes if required on the RHS (e.g. a filename that contains a space). Since shells fundamentally use whitespace to delimit fields in general, it is visually much clearer for each redirection to be separated by whitespace, but grouped in chunks that contain no unnecessary whitespace.
      <br/>
      始终保持重定向“紧密分组”——也就是说，不要在重定向语法中的任何位置包含空格，除非在 RHS 上需要时在引号内（例如，包含空格的文件名）。由于 shell 通常从根本上使用空格来分隔字段，因此每个重定向都由空格分隔，但分组为不包含不必要空格的块，这在视觉上要清晰得多。
     </li>
     <li>
      <strong>
       Do
      </strong>
      always put a space between each redirection, and between the argument list and the first redirect.
      <br/>
      务必始终在每次重定向之间以及参数列表和第一个重定向之间放置一个空格。
     </li>
     <li>
      <strong>
       Always
      </strong>
      place redirections together at the very end of a command after all arguments. Never precede a command with a redirect. Never put a redirect in the middle of the arguments.
      <br/>

    <p>
     Jan Schampera, 2012/02/10 05:46
     <br/>
     You pump
     <code>
      STDERR
     </code>
     of the command to descriptor 1, so that it can be transported by the pipe and seen as input by the
     <code>
      tee
     </code>
     command. At the same time you redirect the original
     <code>
      STDOUT
     </code>
     to descriptor 3. The
     <code>
      tee
     </code>
     command writes your original standard error output to the file plus outputs it to its
     <code>
      STDOUT
     </code>
     .
     <br/>
     您将命令的
     <code>
      STDERR
     </code>
     抽取到描述符 1，以便它可以通过管道传输，并被视为
     <code>
      tee
     </code>
     命令的输入。同时，您将原始
     <code>
      STDOUT
     </code>
     重定向到描述符 3。
     <code>
      tee
     </code>
     命令将原始标准错误输出写入文件，并将其输出到其
     <code>
      STDOUT
     </code>
     。
    </p>
    <p>
     Outside the whole construct you collect your original standard output (descriptor 3) and your original standard error output (descriptor 1 - through
     <code>
      tee
     </code>
     ) to the normal descriptors (1 and 2), the rest is a simple file redirection for both descriptors.
     <br/>
     在整个结构之外，您将原始标准输出（描述符 3）和原始标准错误输出（描述符 1 - 到
     <code>
      tee
     </code>
     ）收集到普通描述符（1 和 2），其余部分是两个描述符的简单文件重定向。
    </p>
    <p>
     In short, you use a third descriptor to switch a bypass through
     <code>
      tee
     </code>
     .
     <br/>
     简而言之，您可以使用第三个描述符来切换通过
     <code>
      tee
     </code>
     的旁路。
    </p>
    <p>
     I don’t know a global method (
     <code>
      exec
     </code>
     or the like) off my head. I can imagine that you can hack something with
     <a href="http://wiki.bash-hackers.org/syntax/expansion/proc_subst" rel="nofollow">
      process substitution
     </a>
     , but I’m not sure.
     <br/>
     我不知道一个全局方法（
     <code>
      exec
     </code>
     或类似方法）。我可以想象你可以用进程替换来破解一些东西，但我不确定。
    </p>
