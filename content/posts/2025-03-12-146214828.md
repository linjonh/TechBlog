     最小花费爬楼梯（动态规划）
    </h2>
    <p>
     <a href="https://www.nowcoder.com/practice/6fe0302a058a4e4a834ee44af88435c7" rel="nofollow">
      题目链接
     </a>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/632fcffeb7c34bbda3f19baded761809.png"/>
    </p>
    <h3>
     <a id="_5">
     </a>
     题解
    </h3>
    <p>
     <font size="3">
      1. 状态表示：以i位置为结尾的最小花费
      <br/>
      2. 状态转移方程：
      <br/>
      dp[i] = min(dp[i-1] + cost[i-1,dp[i-2] + cost[i-2])
      <br/>
      可以从 i-1 位置和 i-2 到达 i 位置
      <br/>
      注意 dp[i] 表示的是 i 位置之前的最小花费，还要加上该点的位置才是到达这个点的最小花费
      <br/>
      注意楼顶的位置是n下标的位置
      <br/>
      3.从左往右开始填表
      <br/>
      4. 初始化：dp[0] = dp[1] = 0，因为从0或者1位置开始向后走，之前是没有花费的
     </font>
    </p>

    <h3>
     <a id="_43">
     </a>
     题解
    </h3>
    <p>
     <font size="3">
      1. 第二种解法是：
      <br/>
      <font color="Red">
       <strong>
        使用两个额外的变量来记录两个字符串的下标，每遇到其中一个字符串就去这个字符串的前面找另一个字符串，记录两个字符串之间的最小距离，记得找完后要更新下标
       </strong>
      </font>
      <br/>
      2. 这样不用暴力地固定一个字符串找另一个字符串了，时间复杂度优化为了O(N)
     </font>
    </p>
