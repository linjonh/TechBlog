    <p>
     在复杂的前端应用中，组件之间的通信往往需要一种灵活且解耦的方式。传统的 Vue 2 中，我们常使用全局事件总线来实现这种通信，但在 Vue 3 中，由于架构和 API 的变化，全局事件总线并非内置方案。本文将为你详细介绍如何在 Vue 3 中实现事件总线，并通过代码示例展示基于
     <a href="https://github.com/developit/mitt">
      mitt
     </a>
     的轻量级事件总线实现，以及自定义实现的方法。
    </p>
    <hr/>
    <h3>
     <a id="_6">
     </a>
     为什么需要事件总线？
    </h3>
    <p>
     在组件间通信场景中，当组件之间没有直接的父子关系时，我们可以通过事件总线来实现数据传递。事件总线能够实现以下效果：
    </p>
    <ul>
     <li>
      <strong>
       解耦合通信：
      </strong>
      发送者与接收者无需相互依赖，只需关注事件名称与数据内容。
     </li>
     <li>
      <strong>
       灵活扩展：
      </strong>
      对于简单的跨组件通信需求，不必引入状态管理库（如 Vuex/Pinia）。
     </li>
     <li>
      <strong>
       简化代码逻辑：
      </strong>
      通过统一的事件中转，便于维护与调试。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_mitt__16">
     </a>
     使用 mitt 实现事件总线
    </h3>
    <p>
     <a href="https://github.com/developit/mitt">
      mitt
     </a>
     是一个仅 200 行左右代码的极简事件触发器，非常适合用作 Vue 3 的事件总线。
    </p>
    <h4>
     <a id="1__mitt_20">
     </a>
     1. 安装 mitt
    </h4>
    <p>
     首先，通过 npm 或 yarn 安装 mitt：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 使用 npm 安装</span>
<span class="token function">npm</span> <span class="token function">install</span> mitt

<span class="token comment"># 或者使用 yarn</span>
<span class="token function">yarn</span> <span class="token function">add</span> mitt
</code></pre>
    <h4>
     <a id="2__32">
     </a>
     2. 创建事件总线
    </h4>
    <p>
     在项目中创建一个单独的事件总线文件，如
     <code>
      eventBus.js
     </code>
     ：
    </p>
    <pre><code class="prism language-js"><span class="token comment">// eventBus.js</span>
<span class="token keyword">import</span> mitt <span class="token keyword">from</span> <span class="token string">'mitt'</span>

<span class="token keyword">const</span> emitter <span class="token operator">=</span> <span class="token function">mitt</span><span class="token punctuation">(</span><span class="token punctuation">)</span>

<span class="token keyword">export</span> <span class="token keyword">default</span> emitter
</code></pre>
    <h4>
     <a id="3__45">
     </a>
     3. 在组件中使用事件总线
    </h4>
    <h5>
     <a id="_ComponentAvue_47">
     </a>
     发送端组件（例如 ComponentA.vue）
    </h5>
    <pre><code class="prism language-vue">&lt;template&gt;
  &lt;div&gt;
    &lt;h2&gt;组件 A&lt;/h2&gt;
    &lt;button @click="sendMessage"&gt;发送消息&lt;/button&gt;
  &lt;/div&gt;
&lt;/template&gt;

&lt;script setup&gt;
import emitter from '@/eventBus'  // 根据项目实际路径引入

const sendMessage = () =&gt; {
  // 触发事件 'custom-event'，传递消息数据
  emitter.emit('custom-event', 'Hello from Component A')
}
&lt;/script&gt;
</code></pre>
    <h5>
     <a id="_ComponentBvue_67">
     </a>
     接收端组件（例如 ComponentB.vue）
    </h5>
    <pre><code class="prism language-vue">&lt;template&gt;
  &lt;div&gt;
    &lt;h2&gt;组件 B&lt;/h2&gt;
    &lt;p&gt;收到的消息：{<!-- -->{ message }}&lt;/p&gt;
  &lt;/div&gt;
&lt;/template&gt;

&lt;script setup&gt;
import { ref, onMounted, onUnmounted } from 'vue'
import emitter from '@/eventBus'

const message = ref('')

// 定义事件处理函数
const updateMessage = (payload) =&gt; {
  message.value = payload
}

onMounted(() =&gt; {
  // 监听 'custom-event' 事件
  emitter.on('custom-event', updateMessage)
})

onUnmounted(() =&gt; {
  // 组件销毁时注销事件监听，避免内存泄漏
  emitter.off('custom-event', updateMessage)
})
&lt;/script&gt;
</code></pre>
    <p>
     通过上述代码示例，我们实现了组件间的简单通信：当 ComponentA 中点击按钮时，会通过事件总线发送消息；而 ComponentB 监听到该消息后，自动更新显示的内容。
    </p>

    <hr/>
    <h3>
     <a id="_142">
     </a>
     总结
    </h3>
    <p>
     本文介绍了 Vue 3 中实现事件总线的两种方式：
    </p>
    <ul>
     <li>
      使用轻量级库
      <a href="https://github.com/developit/mitt">
       mitt
      </a>
      实现高效解耦的事件通信；
     </li>
     <li>
      基于 Vue 3 响应式 API 自定义一个简单的事件总线。
     </li>
    </ul>
    <p>
     事件总线对于非父子组件间的通信场景十分适用，但在大型应用中，建议结合状态管理方案（如 Pinia 或 Vuex）进行更系统化的数据管理。希望这篇文章能帮助你更好地理解并应用 Vue 3 中的事件总线，为组件间通信搭建高效桥梁！
    </p>

     快试试以上代码示例，体验 Vue 3 中事件总线带来的灵活与便捷吧！
