     1. 业务理解与需求分析的重要性
    </h6>
    <ul>
     <li>
      <strong>
       根本认知
      </strong>
      ：
      <ul>
       <li>
        软件本质是解决业务问题的工具
       </li>
       <li>

       </li>
      </ul>
     </li>
     <li>
      <strong>
       面试标准
      </strong>
      ：
      <ul>
       <li>
        架构师面试先考察业务能力（业务背景/架构/模块关系）
       </li>
       <li>
        业务不过关直接淘汰（水货/PPT架构师识别标准）
       </li>
      </ul>
     </li>
     <li>
      <strong>
       阶段关联
      </strong>
      ：
      <ul>
       <li>
        需求分析是架构设计基本功
       </li>
       <li>
        需求分析阶段是架构师理解业务的关键窗口
       </li>
      </ul>
     </li>
    </ul>

     <li>

      </strong>

      <ul>

       <li>

      </ul>

       方法论破除神秘感

       </li>
      </ul>
     </li>
    </ul>

     3. 学习重点说明
    </h6>
    <ul>
     <li>

       </li>
      </ul>
     </li>
     <li>
      <strong>
       迭代学习模式
      </strong>
      ：
      <ul>
