     2.2.1、main函数
    </h4>
    <pre><code class="language-cpp">#include "mainwindow.h"

#include &lt;QApplication&gt;

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);
    MainWindow w;
    w.show();//相当于绘制函数，没有它窗口都看不见
    return a.exec();//用来启动应用程序的事件循环的。当你调用这个函数
                      时，它会开始处理和分发事件，如用户的点击、键盘输入等
}</code></pre>
    <ul>
     <li>
      <span style="color:#333333">
       <strong>
        MainWindow
       </strong>
      </span>
      <span style="color:#333333">
       ：这是一个包含完整菜单栏、工具栏和状态栏的主窗口应用程序框架。它适合于更复 杂的应用程序，需要这些额外的用户界面元素来提供丰富的功能和交互
      </span>
     </li>
     <li>
      <span style="color:#333333">
       <strong>
        Widget
       </strong>
      </span>
      <span style="color:#333333">
       ：这通常是一个简单的窗口，没有内置的菜单栏、工具栏或状态栏。它适合于更简单或专用 的应用程序，不需要复杂的用户界面组件。
      </span>
     </li>
     <li>
      <div>
       <span style="color:#333333">
        <strong>
         QApplication a(argc, argv)
        </strong>
        ;
       </span>
       <span style="color:#333333">
        这行代码的作用是创建一个
       </span>
       <span style="color:#333333">
        QApplication
       </span>
       <span style="color:#333333">
        类的
       </span>
      </div>
      <div>
       <span style="color:#333333">
        实例。这是几乎每个
       </span>
       <span style="color:#333333">
        Qt
       </span>
       <span style="color:#333333">
        应用程序必须做的第一步，因为它负责管理应用程序的许多核心功能。
       </span>
      </div>
     </li>
    </ul>

    <pre><code class="language-cpp">#ifndef WIDGET_H
#define WIDGET_H

#include &lt;QWidget&gt;

QT_BEGIN_NAMESPACE
namespace Ui { class Widget; }
QT_END_NAMESPACE

class Widget : public QWidget //MainWindow公有继承QMainWindow，QMainWindow继承Q_OBJECT

private:
    Ui::Widget *ui;

        <span style="color:#333333">

        </span>
        <span style="color:#333333">
         parent
        </span>
        <span style="color:#333333">

     2.3.2、信号与槽
    </h4>
    <p>
     简单来说信号就是触发槽，槽是一个执行的动作，信号就是触发槽的信号。
    </p>
    <ul>
     <li>
      <div>
       <span style="color:#333333">
        <strong>
         信号
        </strong>
       </span>
       <span style="color:#333333">
        <strong>
         (Signals)
        </strong>
       </span>
       <span style="color:#333333">
        ：是由对象在特定事件发生时发出的消息。例如，
       </span>
       <span style="color:#333333">
        QPushButton
       </span>
       <span style="color:#333333">
        有一个
       </span>
      </div>
      <div>
       <span style="color:#333333">
        clicked()
       </span>
       <span style="color:#333333">
        信号，当用户点击按钮时发出。
       </span>
      </div>
     </li>
     <li>
      <div>
       <div>
        <span style="color:#333333">
         <strong>
          槽
         </strong>
        </span>
        <span style="color:#333333">
         <strong>
          (Slots)
         </strong>
        </span>
        <span style="color:#333333">
         ：是用来响应信号的方法。一个槽可以是任何函数，当其关联的信号被发出时，该槽函数 将被调用。
        </span>
       </div>
      </div>
     </li>
    </ul>
    <p>
     <strong>
      简单使用一下信号与槽：我们需要在类Widget中添加一下信号和槽的成员：
     </strong>
    </p>
    <pre><code class="language-cpp">class Widget : public QWidget
{
    Q_OBJECT

public:
    Widget(QWidget *parent = nullptr);
    ~Widget();

//类成员信号定义
signals:
    void mysignal();//无参类型信号
    void mysignalparams(int value);//有参类型信号

//类成员槽定义
private slots:
    void myslot();//无参类型槽
    void myslotparams(int value);//有参类型槽

private:
    Ui::Widget *ui;
};</code></pre>
    <p>
     然后在.cpp文件中定义槽的函数：
    </p>
    <pre><code class="language-cpp">//槽函数定义
void Widget::myslot()
{
    std::cout &lt;&lt; "myslot" &lt;&lt; std::endl;
}
//槽函数定义
void Widget::myslotparams(int value)
{
    std::cout &lt;&lt; "myslotparams" &lt;&lt; std::endl;
    std::cout &lt;&lt; value &lt;&lt; std::endl;
}</code></pre>
    <pre>然后将信号和槽“连接起来（connect）”：
</pre>
    <pre><code class="language-cpp">Widget::Widget(QWidget *parent): QWidget(parent), ui(new Ui::Widget)
{
    ui-&gt;setupUi(this);

    connect(this,SIGNAL(mysignal()),this,SLOT(myslot()));//将信号和槽联系起来
    connect(this,SIGNAL(mysignalparams(int)),this,SLOT(myslotparams(int)));//将信号和槽联系起来

    //代码触发信号
    emit mysignal();
    emit mysignalparams(99);
}</code></pre>

      第一种：右键点击按键，选择转到槽：

void MainWindow::on_pushButton_clicked()

}</code></pre>
    <p>
     <strong>
      第二种：
      <span style="color:#333333">
       使用 QObject::connect（是否使用函数指针）
      </span>
     </strong>
    </p>
    <p>
     我们需要在.h头文件中声明槽，第一种系统会自动帮我们声明

    </p>
    <p>
     然后我们也是去.cpp文件中定义内容并且使用connect函数“连接” 信号与槽

    </p>
    <p>
     <strong>
      第四种：
      <span style="color:#333333">
       使用
      </span>
      <span style="color:#333333">

      </span>
     </strong>
    </p>
    <p>
     <span style="color:#333333">
      <strong>
       优点：不需要在头文件中声明槽函数

    <pre><code class="language-cpp">//第三方式：lambda表达式：QObject::connect(sender, &amp;Sender::signal, [=]() { /* lambda body */ });

    {
        std::cout &lt;&lt; "第三种方法按下响应" &lt;&lt; std::endl;
    });
    //不需要在h头文件中声明槽——响应函数</code></pre>

    <h4 id="2.6.2%E3%80%81QList" name="2.6.2%E3%80%81QList">
     2.6.2、
     <span style="color:#333333">
      <strong>
       QList
      </strong>
     </span>
    </h4>
    <div>
     <span style="color:#333333">
      在
     </span>
     <span style="color:#333333">
      Qt
     </span>
     <span style="color:#333333">
      框架中，
     </span>
     <span style="color:#333333">
      QList
     </span>
     <span style="color:#333333">
      是一个容器类，它在内部实现上类似于一个数组，但也提供了一些链表的特性。
     </span>
    </div>
    <div>
     <span style="color:#333333">
      QList
     </span>
     <span style="color:#333333">
      的设计旨在提供一个在多数情况下既高效又方便的通用列表容器。用于存储元素列表。它提供了 丰富的功能，包括添加、移除、访问元素等。
     </span>
    </div>
    <div>
     <span style="color:#333333">
      QList
     </span>
     <span style="color:#333333">
      <strong>
       的内部工作原理：
      </strong>
     </span>
    </div>
    <ol>
     <li>
      <div>
       <span style="color:#333333">
        <strong>
         数组式存储
        </strong>
       </span>
       <span style="color:#333333">
        ：
       </span>
       <span style="color:#333333">
        QList
       </span>
       <span style="color:#333333">
        在大多数情况下使用连续内存存储其元素，类似于数组。这意味着它提供了快 速的索引访问（通过下标操作符 []
       </span>
       <span style="color:#333333">
        ），以及相对高效的迭代性能。
       </span>
      </div>
     </li>
     <li>
      <span style="color:#333333">
       <strong>
        动态调整大小
       </strong>
      </span>
      <span style="color:#333333">
       ：与静态数组不同，
      </span>
      <span style="color:#333333">
       QList
      </span>
      <span style="color:#333333">
       可以动态增长和缩减，自动管理内存分配。
      </span>
     </li>
     <li>
      <span style="color:#333333">
       <strong>
        链表特性
       </strong>
      </span>
      <span style="color:#333333">
       ：虽然
      </span>
      <span style="color:#333333">
       QList
      </span>
      <span style="color:#333333">
       主要基于数组，但它也提供了一些链表的操作，比如在列表的开始或结束 处添加和移除元素。这些操作通常比在数组中间插入或删除元素更高效。
      </span>
     </li>
     <li>
      <div>
       <span style="color:#333333">
        <strong>
         复制时共享内存
        </strong>
       </span>
       <span style="color:#333333">
        ：
       </span>
       <span style="color:#333333">
        QList
       </span>
       <span style="color:#333333">
        使用一种称为
       </span>
       <span style="color:#333333">
        “
       </span>
       <span style="color:#333333">
        隐式共享
       </span>
       <span style="color:#333333">
        ”
       </span>
       <span style="color:#333333">
        （
       </span>
       <span style="color:#333333">
        implicit sharing
       </span>
       <span style="color:#333333">
        ）或
       </span>
       <span style="color:#333333">
        “
       </span>
       <span style="color:#333333">
        写时复制
       </span>
       <span style="color:#333333">
        ”
       </span>
       <span style="color:#333333">
        （
       </span>
       <span style="color:#333333">
        copy-onwrite）的技术。这意味着当你复制一个
       </span>
       <span style="color:#333333">
        QList
       </span>
       <span style="color:#333333">
        时，它不会立即复制所有元素，而是共享相同的数 据，直到你尝试修改其中一个列表，此时才进行实际的复制。这使得复制 QList
       </span>
       <span style="color:#333333">
        变得非常高效。
       </span>
      </div>
     </li>
    </ol>
    <p>
     基本用法：
    </p>
    <p>
     <span style="color:#333333">

     2.10、QT事件
    </h3>
    <div>
     <span style="color:#333333">
      <strong>
       事件处理过程：
      </strong>
     </span>
    </div>
    <div>
     <div>
      <span style="color:#333333">
       众所周知
      </span>
      <span style="color:#333333">
       Qt
      </span>
      <span style="color:#333333">
       是一个基于
      </span>
      <span style="color:#333333">
       C++
      </span>
      <span style="color:#333333">
       的框架，主要用来开发带窗口的应用程序（不带窗口的也行，但不是主流）。 我们使用的基于窗口的应用程序都是基于事件，其目的主要是用来实现回调（因为只有这样程序的效率 才是最高的）。所以在Qt
      </span>
      <span style="color:#333333">
       框架内部为我们提供了一些列的事件处理机制，当窗口事件产生之后，事件会 经过：
      </span>
      <span style="color:#fe2c24">
       <strong>
        事件派发 -&gt; 事件过滤-&gt;事件分发-&gt;事件处理
       </strong>
      </span>
      <span style="color:#333333">
      </span>
      <span style="color:#333333">
       几个阶段。
      </span>
      <strong>
       <span style="color:#38d8f0">
        Qt窗口中对于产生的一系列事件都有默认 的处理动作，如果我们有特殊需求就需要在合适的阶段重写事件的处理动作
       </span>
      </strong>
      <span style="color:#333333">
       ，
       <strong>
        比如信号与槽就是一种
       </strong>
      </span>
     </div>
     <div>
      <div>
       <span style="color:#333333">
        事件（
       </span>
       <span style="color:#333333">
        event
       </span>
       <span style="color:#333333">
        ）是由系统或者
       </span>
       <span style="color:#333333">
        Qt
       </span>
       <span style="color:#333333">
        本身在不同的场景下发出的。当用户按下
       </span>
       <span style="color:#333333">
        /
       </span>
       <span style="color:#333333">
        移动鼠标、敲下键盘，或者是 窗口关闭/
       </span>
       <span style="color:#333333">
        大小发生变化
       </span>
       <span style="color:#333333">
        /
       </span>
       <span style="color:#333333">
        隐藏或显示都会发出一个相应的事件。一些事件在对用户操作做出响应时发出， 如鼠标/
       </span>
       <span style="color:#333333">
        键盘事件等；另一些事件则是由系统自动发出，如计时器事件。
       </span>
      </div>
      <div>
       <div>
        <span style="color:#333333">
         每一个
        </span>
        <span style="color:#333333">
         Qt
        </span>
        <span style="color:#333333">
         应用程序都对应一个唯一的
        </span>
        <span style="color:#333333">
         QApplication
        </span>
        <span style="color:#333333">
         应用程序对象，然后调用这个对象的
        </span>
        <span style="color:#333333">
         exec()
        </span>
        <span style="color:#333333">
         函
        </span>
       </div>
       <div>
        <span style="color:#333333">
         数，这样
        </span>
        <span style="color:#333333">
         Qt
        </span>
        <span style="color:#333333">
         框架内部的事件检测就开始了（
        </span>
        <span style="color:#333333">
         程序将进入事件循环来监听应用程序的事件
        </span>
        <span style="color:#333333">
         ）。
        </span>
       </div>
      </div>
     </div>
    </div>
    <div>
    </div>
    <pre><code class="language-cpp">int main(int argc, char *argv[])
{
    QApplication a(argc, argv);
    MainWindow* w = new MainWindow;
    w.show();
    return a.exec();
}</code></pre>
    <p>
     <span style="color:#333333">
      事件在
     </span>
     <span style="color:#333333">
      Qt
     </span>
     <span style="color:#333333">
      中产生之后，的分发过程是这样的：
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#333333">
       当事件产生之后，
      </span>
      <span style="color:#333333">
       Qt
      </span>
      <span style="color:#333333">
       使用用应用程序对象调用
      </span>
      <span style="color:#333333">
       notify()
      </span>
      <span style="color:#333333">
       函数将事件发送到指定的窗口：
      </span>
      <pre><code class="language-cpp">[override virtual] bool QApplication::notify(QObject *receiver, QEvent *e);</code></pre>
     </li>
     <li>
      <div>
       <span style="color:#333333">
        事件在发送过程中可以通过事件过滤器进行过滤，默认不对任何产生的事件进行过滤。
       </span>
       <pre><code class="language-cpp">// 需要先给窗口安装过滤器, 该事件才会触发
[virtual] bool QObject::eventFilter(QObject *watched, QEvent *event)</code></pre>
      </div>
     </li>
     <li>
      <div>
       <span style="color:#333333">
        当事件发送到指定窗口之后，窗口的事件分发器会对收到的事件进行分类
       </span>
       <span style="color:#333333">
        :
       </span>
       <pre><code class="language-cpp">[override virtual protected] bool QWidget::event(QEvent *event);</code></pre>
      </div>
     </li>
     <li>
      <span style="color:#333333">
       事件分发器会将分类之后的事件（鼠标事件、键盘事件、绘图事件。。。）分发给对应的事件处理 器函数进行处理，每个事件处理器函数都有默认的处理动作（我们也可以重写这些事件处理器函 数），比如：鼠标事件：

#ifndef MYTEXTEDIT_H
#define MYTEXTEDIT_H

#include &lt;QTextEdit&gt;

class MyTextEdit : public QTextEdit
{
public:
    MyTextEdit(QWidget *parent);//需要添加代码的地方



protected:
    void wheelEvent(QWheelEvent *e) override;
    void keyPressEvent(QKeyEvent *e) override;
    void keyReleaseEvent(QKeyEvent *e) override;

private:
    bool ctrlKeyPressed = 0;//Ctrl是否被按下标志位

};

#endif // MYTEXTEDIT_H



mytextedit.c

#include "mytextedit.h"

#include &lt;QWheelEvent&gt;
#include &lt;QDebug&gt;

MyTextEdit::MyTextEdit(QWidget *parent) : QTextEdit(parent)
{

}

void MyTextEdit::wheelEvent(QWheelEvent *e)
{
    qDebug() &lt;&lt; "wheelEvent";
    if(ctrlKeyPressed == 1){
        if(e-&gt;angleDelta().y() &gt; 0){
            //放大字体
            qDebug() &lt;&lt; "fangda";
        }else if(e-&gt;angleDelta().y() &lt; 0){
            //缩小字体
            qDebug() &lt;&lt; "suoxiao";
        }
        e-&gt;accept();//事件处理完成
    }else{
        QTextEdit::wheelEvent(e);//没有按下Ctrl默认怎么处理就怎么处理
    }
}

void MyTextEdit::keyPressEvent(QKeyEvent *e)
{
    qDebug() &lt;&lt; "keyPressEvent";
    if(e-&gt;key() == Qt::Key_Control){
           // qDebug() &lt;&lt; "ctrl Pressed";
            ctrlKeyPressed = 1;
        }
        QTextEdit::keyPressEvent(e);
}

void MyTextEdit::keyReleaseEvent(QKeyEvent *e)
{
    qDebug() &lt;&lt; "keyReleaseEvent";
    if(e-&gt;key() == Qt::Key_Control){
       // qDebug() &lt;&lt; "ctrl Release";
        ctrlKeyPressed = 0;
    }
    QTextEdit::keyPressEvent(e);
}
