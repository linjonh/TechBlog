    <p>
     分支 two
    </p>
    <p>
     查看所有分支
     <strong>
      git branch
     </strong>
     <br/>
     切换分支（two）
     <strong>
      git checkout two
     </strong>
     <br/>
     创建并切换到新分支（two）
     <strong>
      git checkout -b two
     </strong>
     <br/>
     提交当前代码到到源分支
     <strong>
      git push --set-upstream origin two
     </strong>
    </p>
    <h3>
     <a id="_8">
     </a>
     合并分支
    </h3>
    <p>
     <strong>
      1. 切换到目标分支
     </strong>
     <br/>
     例如，如果你想将feature-branch合并到main分支，你应该先切换到main分支
    </p>
    <pre><code>git checkout main
</code></pre>
    <p>
     <strong>
      2. 拉取最新的更改
     </strong>
     <br/>
     在合并之前，确保你的分支是最新的。拉取远程仓库的最新更改到你的本地仓库：
    </p>
    <pre><code>git pull origin main
</code></pre>
    <p>
     <strong>
      3. 合并分支
     </strong>
     <br/>
     现在，你可以将另一个分支合并到当前分支。例如，将feature-branch合并到main分支 。 使用以下命令：
    </p>
    <pre><code>git merge feature-branch
</code></pre>
    <p>
     <strong>
      4. 解决冲突（如果存在）
     </strong>
     <br/>
     如果在合并过程中出现冲突，Git会暂停合并过程并通知你。你需要手动解决这些冲突。解决完冲突后，你需要标记冲突为已解决：
    </p>
    <pre><code>git add &lt;冲突文件&gt;
</code></pre>
    <p>
     然后继续合并过程：
    </p>
    <pre><code>git merge --continue
</code></pre>
    <p>
     <strong>
      5. 提交合并（如果需要）
     </strong>
     <br/>
     如果合并过程中没有冲突，或者你解决了所有冲突并提交了更改，Git会自动完成合并。但是，有时候你可能想在合并后提交一个合并提交（merge commit），特别是当你想要记录这次合并的理由时。你可以使用：
    </p>
    <pre><code>git commit -m "合并描述信息"
</code></pre>
    <p>
     <strong>
      6. 推送更改到远程仓库
     </strong>
     <br/>
     最后，将合并后的更改推送到远程仓库：
    </p>
    <pre><code>git push origin &lt;目标分支名&gt;
</code></pre>
    <p>
     例如，对于main分支：
    </p>
    <pre><code>git push origin main
</code></pre>
