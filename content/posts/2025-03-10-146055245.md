      <br/>

    <h3>
     <a id="2_4">
     </a>
     2.构造函数
    </h3>
    <blockquote>
     <p>
      概念：构造函数是一个特殊的成员函数，名字与类名相同,创建类类型对象时由编译器自动调用，以保证每个数据成员都有一个合适的初始值，并且
      <strong>
       在对象整个生命周期内只调用一次
      </strong>
      。需要注意的是，构造函数虽然名称叫构造，但是构造函数的主要任务并不是开空间创建对象，而是
      <strong>
       初始化对象
      </strong>
      。
      <br/>
      特点：
     </p>
     <ol>
      <li>
       函数名与类名相同。
      </li>
      <li>
       无返回值。
      </li>
      <li>
       对象实例化时编译器自动调用对应的构造函数。
      </li>
      <li>
       构造函数可以重载。
      </li>
      <li>
       如果类中没有显式定义构造函数，则C++编译器会自动生成一个无参的默认构造 函数，一旦用户显式定义编译器将不再生成。
       <br/>
       6.C++把类型分成
       <strong>
        内置类型
       </strong>
       (基本类型)和
       <strong>
        自定义类型
       </strong>
       。内置类型就是语言提供的数据类型，如：int/char…，自定义类型就是我们使用class/struct/union等自己定义的类型，
       <strong>
        编译器生成默认的构造函数只会对自定类型成员调用的它的默认成员函数
       </strong>
       <br/>
       7.无参的构造函数和全缺省的构造函数都称为默认构造函数，并且默认构造函数只能有一个。
       <br/>
       <strong>
        注意：无参构造函数、全缺省构造函数、我们没写编译器默认生成的构造函数，都可以认为是默认构造函数。
       </strong>
       <br/>
       <strong>
        调用顺序：
       </strong>
       <br/>
       <strong>
        全局对象先于局部对象进行构造，局部对象按照出现的顺序进行构造，无论是否为static
       </strong>
      </li>
     </ol>
    </blockquote>

    <h3>
     <a id="3_37">
     </a>
     3.析构函数
    </h3>
    <blockquote>
     <p>
      概念：特殊的成员函数，析构函数不是完成对对象本身的销毁，局部对象销毁工作是由编译器完成的。而
      <strong>
       对象在销毁时会自动调用析构函数，完成对象中资源的清理工作
      </strong>
      。
      <br/>
      特点：
     </p>
     <ol>
      <li>
       析构函数名是在类名前加上字符 ~。
      </li>
      <li>
       无参数无返回值类型。
      </li>
      <li>
       <strong>
        一个类只能有一个析构函数
       </strong>
       。若未显式定义，系统会自动生成默认的析构函数。注意：析构函数不能重载
      </li>
      <li>
       <strong>
        对象生命周期结束时，C++编译系统系统自动调用析构函数
       </strong>
       <br/>
       5.内置类型成员不做处理，自定义类型成员会去调用他的析构函数
      </li>
     </ol>
     <blockquote>
      <p>
       总结：
       <br/>
       1.一般情况下，
       <strong>
        有动态申请资源，就需要显示写析构函数释放资源
       </strong>
       ，如Stack类。
       <br/>
       2.
       <strong>
        没有动态申请的资源，不需要写析构
       </strong>
       ，如Data类
       <br/>
       3.
       <strong>
        需要释放资源的成员都是自定义类型不需要析构
       </strong>
       ，如用栈实现队列
      </p>
     </blockquote>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       调用顺序：
       <br/>
       与构造函数相反，从最后派生的类开始调用，逐步向上调用到基类，局部对象按照出现的顺序从后往前调用，最后是全局变量，只需注意static改变对象的生存作用域之后，会放在局部对象之后进行析构。
      </strong>
     </p>
    </blockquote>
    <h3>
     <a id="4_52">
     </a>
     4.拷贝构造函数
    </h3>
    <blockquote>
     <p>
      为了创建一个与已存在对象一样的新对象并在此基础上进行操作，引入该函数。
      <br/>
      概念：
      <strong>
       只有单个形参
      </strong>
      ，该形参是对
      <strong>
       本类类型对象的引用(一般常用const修饰)
      </strong>
      ，在用
      <strong>
       已存在的类类型对象创建新对象时由编译器自动调用
      </strong>
      。
      <br/>
      特点：
     </p>
     <ol>
      <li>
       拷贝构造函数是
       <strong>
        构造函数
       </strong>
       的一个
       <strong>
        重载形式
       </strong>
      </li>
      <li>
       拷贝构造函数的
       <strong>
        参数只有一个
       </strong>
       且
       <strong>
        必须是类类型对象的引用
       </strong>
       ，使用
       <strong>
        传值方式编译器直接报错
       </strong>
       ，因为会引发无穷递归调用。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/bb1436ff56244c67b514c3572880c66b.png">
        <br/>
        行为机制，
        <strong>
         实参复制到形参，需调用拷贝构造函数
        </strong>
        ，于是触发无穷递归，每次递归建立新的函数栈帧，直到栈空间耗尽。
        <br/>
        正确方式如下，以Date类为例
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4886ad9b89dc4aa197474145c11e1fbe.png"/>
        <br/>
        这里d是d1的别名，d2是this指针。形参为实参的别名，无对象复制，安全。可以常量引用const&amp;进一步禁止修改原对象，增强安全性
        <br/>
        3.
        <strong>
         若未显式定义，编译器会生成默认的拷贝构造函数
        </strong>
        。默认的拷贝构造函数对象按内存存储按字节序完成拷贝，这种拷贝叫做浅拷贝，或者值拷贝。
        <br/>
        4.C++规定
        <br/>
        1.内置类型直接拷贝
        <br/>
        2.自定义类型必须用拷贝构造完成拷贝(深拷贝)。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/232babd9a15d45ba99739154db21b0d0.png"/>
        <br/>
        该图存在问题 1.析构两次会报错 2.其中一个修改会影响另外一个(指向同一块空间)
        <br/>
        所以注意：
        <br/>
        类中如果没有涉及资源申请时，拷贝构造函数是否写都可以；
        <strong>
         一旦涉及到资源申请时，则拷贝构造函数是一定要写的，否则就是浅拷贝
        </strong>
        。
       </img>
      </li>
      <li>
       拷贝构造函数典型调用场景：
       <br/>
       a.使用已存在对象创建新对象
       <br/>
       b.函数参数类型为类类型对象
       <br/>
       c.函数返回值类型为类类型对象
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/61304beaca73425d9ab40a7bd289530a.png"/>
       <br/>
       这里以Date类为例感受拷贝构造函数的调用场景，同时注意为了提高程序效率，
       <strong>
        一般对象传参时，尽量使用引用类型，返回时根据实际场景，能用引用尽量使用引用
       </strong>
       。
      </li>
     </ol>
    </blockquote>
    <h3>
     <a id="5_77">
     </a>
     5.运算符重载
    </h3>
    <blockquote>
     <p>
      C++为了增强代码的可读性引入了运算符重载，运算符重载是具有特殊函数名的函数,
      <br/>
      也具
      <strong>
       有其返回值类型
      </strong>
      ，
      <strong>
       函数名字以及参数列表
      </strong>
      ，其返回值类型与参数列表与普通的函数类似。函数名字为：关键字operator后面接需要重载的运算符符号。
      <br/>
      函数原型：返回值类型 operator操作符(参数列表)
      <br/>
      注意：
      <br/>
      1.不能通过连接其他符号来创建新的操作符：比如operator@ ，要重载有意义的
      <br/>
      2.重载操作符必须有一个类类型参数
      <br/>
      3.用于内置类型的运算符，其含义不能改变，例如：内置的整型+，不能改变其含义
      <br/>
      4.作为类成员函数重载时，其形参看起来比操作数数目少1，因为成员函数的第一个参数为隐藏的this
     </p>
    </blockquote>
    <ol start="5">
     <li>
      <em>
       <em>
        .
       </em>
       :: sizeof ?: .以上5个运算符不能重载
      </em>
      *
     </li>
    </ol>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d14469c810834ebea546f36545047ea7.png"/>
      <br/>
      以&lt;号为例，bool operator&lt;(const Date&amp;x)实际上是bool operator&lt;(Data*this,const Date&amp;x),调用函数时可以直接(d1&lt;d2),但
      <strong>
       从汇编指令看调用过程中其实直接call运算符重载的地址，两种方式都可以写，但前者更简单。
      </strong>
      <br/>
      注意若把运算符重载函数定义在
      <strong>
       全局
      </strong>
      的就需要
      <strong>
       成员变量是公有的
      </strong>
     </p>
    </blockquote>

    <h3>
     <a id="6_119">
     </a>
     6.赋值运算符重载
    </h3>
    <blockquote>
     <p>
      格式：
      <br/>
      1.
      <strong>
       参数类型
      </strong>
      ：const T&amp;，传递引用可以提高传参效率
      <br/>
      2.
      <strong>
       返回值类型
      </strong>
      ：T&amp;，返回引用可以提高返回的效率，有返回值目的是为了支持连续赋值
      <br/>
      3.
      <strong>
       检测是否自己给自己赋值
      </strong>
      ，涉及到动态资源管理时需加入检查
      <br/>
      4.
      <strong>
       返回*this
      </strong>
      ：要复合连续赋值的含义
      <br/>
      特征：
      <br/>
      1.赋值运算符
      <strong>
       只能重载成类的成员函数不能重载成全局函数
      </strong>
      <br/>
      原因：赋值运算符如果不显式实现，编译器会生成一个默认的。此时用户再在类外自己实现一个全局的赋值运算符重载，就和编译器在类中生成的默认赋值运算符重载冲突了，故赋值运算符重载只能是类的成员函数。
      <br/>
      2.用户没有显式实现时，编译器会生成一个默认赋值运算符重载，以值的方式逐字节拷贝。
      <br/>
      <strong>
       默认生成赋值重载跟拷贝构造行为一样
       <br/>
       a.内置类型成员：直接赋值，浅拷贝
       <br/>
       b.自定义类型成员会调用他的赋值重载
      </strong>
      <br/>
      3.区分：
      <br/>
      <strong>
       运算符重载函数：已经存在的两个对象之间复制拷贝
       <br/>
       构造函数：用一个以及存在的对象初始化另一个对象
      </strong>
     </p>
    </blockquote>
    <blockquote>
     <p>
      需注意：
      <br/>
      如果类中未涉及到资源管理，赋值运算符是否实现都可以；一旦涉及到资源管理则必须要实现。(与拷贝构造函数相同)

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/38a4b8fa3376460e8629fe6cbafd59d1.png"/>
     </p>
    </blockquote>
    <h3>
     <a id="7const_139">
     </a>
     7.const成员
    </h3>
    <blockquote>
     <p>
      <strong>
       将const修饰的“成员函数”称之为const成员函数
      </strong>
      ，const修饰类成员函数，
      <strong>
       实际修饰该成员函数隐含的this指针
      </strong>
      ，表明在该成员函数中
      <strong>
       不能对类的任何成员进行修改
      </strong>
      。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/caa67768df8f4b99af30ebc2183db1af.png"/>
      <br/>
      从下图可发现几个问题
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2c1b438331b04ee9ae4174238a114abb.png"/>
      <br/>
      <strong>
       1.const对象不能调用非const成员函数
      </strong>
      。const对象状态不可修改，这样会造成权限放大的问题。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3518d3e4f4b340759cd1e94501d8204a.png"/>
      <br/>
      <strong>
       2.非const成员可以调用const成员函数
      </strong>
      。const成员函数不会修改对象的成员变量，这样是权限缩小没问题。
      <br/>
      再来一张图来简单感受一下权限问题
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ba68a313579e42fa8af4c204787ea5b0.png"/>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       总结：
       <br/>
       只要成员内部不修改成员变量，都应该加const，这样const对象和普通对象都可以调用。权限可以缩小和转移，不能放大。
      </strong>
     </p>
    </blockquote>
    <h3>
     <a id="8const_152">
     </a>
     8.取地址及const取地址操作符重载
    </h3>
    <blockquote>
     <p>
      意义：
      <strong>
       控制对象获取行为
      </strong>
      <br/>
      <strong>
       一般情况：使用编译器生成的默认取地址的重载，直接返回的this指针
      </strong>
      (即对象的内存地址)。
      <br/>
      <strong>
       const版本的意义
      </strong>
      ：
      <br/>
      1.保证常量性，防止通过指针修改对象内容
      <br/>
      2.权限控制：const成员函数内部只能调用其他const成员函数
      <br/>
      <strong>
       需要手动重载的场景
      </strong>
      ：
      <br/>
      1.隐藏真实地址。安全敏感场景下避免外部直接操作内存对象
      <br/>
      2.一些附加功能的拓展
      <br/>
      3.返回特定内容：比如内部某个子成员地址
     </p>
    </blockquote>
