    <h2>
     <a id="IP__1">
     </a>
     IP 地址
    </h2>
    <p>
     IP（Internet Protocol）地址是计算机网络中的标识符，用于唯一标识网络中的设备。它可以是 IPv4 或 IPv6 地址，负责网络通信中的寻址和数据传输。
    </p>
    <p>
     在 TCP/IP 设计之初，为了标识互联网中的每台主机的身份，设计人员为每个接入网络中的主机都分配一个IP地址（Internet Protocol Address），是一个32位的整数地址，只有合法的IP地址才能接入互联网中并且与其他主机进行通信，IP地址是软件地址，不是硬件地址，硬件 MAC 地址是存储在网卡中的，应用于本地网络中寻找目标主机。而IP地址能让一个网络中的主机能够与另一个网络中的主机进行通信，无需理会这些主机之间的MAC地址。
    </p>
    <h3>
     <a id="IP__7">
     </a>
     IP 地址的分类
    </h3>
    <h4>
     <a id="IPv4__8">
     </a>
     IPv4 地址
    </h4>
    <p>
     IPv4 地址由 32 位二进制组成，通常以 点分十进制表示，例如：
     <br/>
     192.168.1.1
     <br/>
     IPv4 地址范围从 0.0.0.0 到 255.255.255.255，总共有大约 42 亿个地址。
    </p>
    <p>
     IPv4 地址的分类
     <br/>
     在全球的互联网中，每个主机都要唯一的一个IP地址作为身份识别，那么这么多IP地址是怎么样分配的呢？这不可能是随意自由选择IP地址的，实际上每个主机的IP地址的一部分都由其所在的子网决定的，所以又出现了IP地址分类编址的概念，网络的类型决定了IP 地址将如何划分成网络部分和节点部分，在分类编址中，设计者把所有的IP地址划分为5大类，分别为A、B、C、D、E五类，每一类地址都觉定了其中IP地址的一部分组成，具体见下图
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2b8bff2f97214e6d82a4b795bfb1c69c.png"/>
    </p>
    <p>
     类别地址范围主要用途
    </p>
    <ul>
     <li>
      A 类1.0.0.0 – 126.255.255.255大型网络（政府、企业）
     </li>
     <li>
      B 类128.0.0.0 – 191.255.255.255中型网络（高校、公司）
     </li>
     <li>
      C 类192.0.0.0 – 223.255.255.255小型网络（家庭、办公室）
     </li>
     <li>
      D 类224.0.0.0 – 239.255.255.255组播（Multicast）
     </li>
     <li>
      E 类240.0.0.0 – 255.255.255.255预留（研究用途）
     </li>
    </ul>
    <p>
     📌 示意图：IPv4 地址分类
     <br/>
     A 类： 10.0.0.0
     <br/>
     B 类： 172.16.0.0
     <br/>
     C 类： 192.168.1.0
    </p>

     各类 IP 地址的特点
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/96f932a88903464a8ed2231d6126cb77.png"/>
    </p>
    <h4>
     <a id="IPv6__34">
     </a>
     IPv6 地址
    </h4>
    <p>
     由于 IPv4 地址已接近耗尽，IPv6 采用 128 位地址，可以提供 2¹²⁸ 个地址。
     <br/>
     IPv6 示例：
     <br/>
     2001:0db8:85a3:0000:0000:8a2e:0370:7334
     <br/>
     IPv6 采用 冒号分隔的十六进制表示，并支持地址缩写：
    </p>
    <pre><code class="prism language-c"><span class="token number">2001</span><span class="token operator">:</span>db8<span class="token operator">::</span><span class="token number">8</span>a2e<span class="token operator">:</span><span class="token number">370</span><span class="token operator">:</span><span class="token number">7334</span>
</code></pre>
    <h3>
     <a id="_IP__IP_43">
     </a>
     公有 IP 与私有 IP
    </h3>
    <p>
     类型适用范围例子公有 IP互联网设备8.8.8.8（Google DNS）私有 IP局域网（LAN）192.168.1.1、10.0.0.1
     <br/>
     私有 IP 地址范围（不能用于公网）：
    </p>
    <ul>
     <li>
      A 类： 10.0.0.0 – 10.255.255.255
     </li>
     <li>
      B 类： 172.16.0.0 – 172.31.255.255
     </li>
     <li>
      C 类： 192.168.0.0 – 192.168.255.255
      <br/>
      📌 示意图：私有与公有 IP
      <br/>
      互联网（公有 IP） &lt;——&gt; 路由器（公有 IP） &lt;——&gt; 内部网络（私有 IP）
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_IP__IP_52">
     </a>
     静态 IP 与动态 IP
    </h3>
    <p>
     类型说明适用场景静态 IP手动分配，固定不变服务器、企业网络动态 IP由 DHCP 自动分配家庭宽带、手机网络
    </p>
    <h3>
     <a id="_55">
     </a>
     子网与子网掩码
    </h3>
    <p>
     子网掩码用于区分 网络部分 和 主机部分，例如：
     <br/>
     IP 地址：192.168.1.10
     <br/>
     子网掩码：255.255.255.0
     <br/>
     这表示 192.168.1.x 属于同一子网。
     <br/>
     CIDR 表示法：
    </p>
    <pre><code class="prism language-c"><span class="token number">192.168</span><span class="token number">.1</span><span class="token number">.10</span><span class="token operator">/</span><span class="token number">24</span>  <span class="token punctuation">(</span>子网掩码 <span class="token number">255.255</span><span class="token number">.255</span><span class="token number">.0</span><span class="token punctuation">)</span>
</code></pre>
    <h3>
     <a id="_IP__64">
     </a>
     常见 IP 地址用途
    </h3>
    <p>
     IP 地址用途127.0.0.1本机回环地址（Loopback）8.8.8.8Google DNS255.255.255.255广播地址169.254.x.xAPIPA（自动分配 IP）
    </p>
    <h3>
     <a id="IP__67">
     </a>
     IP 地址的工作方式
    </h3>
    <p>
     IP 地址用于网络通信，基本流程如下：
     <br/>
     1.设备分配 IP 地址（手动或动态获取）。
     <br/>
     2.DNS 解析将域名转换为 IP 地址（例如 www.google.com -&gt; 142.250.190.14）。
     <br/>
     3.数据包通过路由器转发到目标 IP 地址。
     <br/>
     4.目标设备接收数据并返回响应。
     <br/>
     📌 示意图：数据包传输过程
    </p>
    <pre><code class="prism language-c">用户电脑 <span class="token punctuation">(</span><span class="token number">192.168</span><span class="token number">.1</span><span class="token number">.2</span><span class="token punctuation">)</span>  <span class="token operator">--</span><span class="token operator">-&gt;</span>  路由器  <span class="token operator">--</span><span class="token operator">-&gt;</span>  互联网 <span class="token punctuation">(</span><span class="token number">8.8</span><span class="token number">.8</span><span class="token number">.8</span><span class="token punctuation">)</span>
</code></pre>
    <h3>
     <a id="_78">
     </a>
     总结
    </h3>
    <p>
     ✅ IPv4 由 32 位组成，IPv6 由 128 位组成。
     <br/>
     ✅ IP 地址有公有、私有、静态、动态等类型。
     <br/>
     ✅ 子网掩码用于区分网络部分和主机部分。
     <br/>
     ✅ IP 地址用于标识设备，路由器负责数据包的转发。
     <br/>
     📌 掌握 IP 地址的基础概念，有助于理解计算机网络的核心原理！
