  <div class="article_content clearfix" id="article_content">

   <div class="markdown_views prism-atom-one-dark" id="content_views">
    <svg style="display: none;" xmlns="http://www.w3.org/2000/svg">
     <path d="M5,0 0,2.5 5,5z" id="raphael-marker-block" stroke-linecap="round" style="-webkit-tap-highlight-color: rgba(0, 0, 0, 0);">
     </path>
    </svg>
    <p>
     之前有写过Ubuntu安装驱动的手册： https://blog.csdn.net/qq_50247813/article/details/146062785
    </p>
    <p>
     <strong>
      到官网寻找合适的驱动：
     </strong>
     <br/>
     https://www.nvidia.cn/drivers/lookup
    </p>
    <blockquote>
     <p>
      直接选择
      <code>
       linux 64-bit
      </code>
      适用于所有x86构架的linux，arm需要选择arm架构。
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7203c3f961784af494815cf50fecf94b.png"/>
    </p>
    <p>
     <strong>
      这次选择
      <code>
       550.90.07
      </code>
      版本下载，将下载的驱动上传到服务器：
     </strong>
    </p>
    <blockquote>
     <p>
      实验主机选择ubuntu 22.04 不区分发现版本centos也行
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1606f7e2848142bf9716e1c815b1e2b0.png"/>
    </p>
    <p>
     在安装驱动前，将开源的
     <code>
      nouveau
     </code>
     禁用掉。
    </p>
    <pre><code class="prism language-bash">root@gpu-server:/home/gpu-admin<span class="token comment"># rmmod nouveau</span>
root@gpu-server:/home/gpu-admin<span class="token comment">#  echo "blacklist nouveau</span>
options nouveau <span class="token assign-left variable">modeset</span><span class="token operator">=</span><span class="token number">0</span>" <span class="token operator">&gt;&gt;</span> /etc/modprobe.d/blacklist-nouveau.conf
</code></pre>
    <p>
     上传驱动安装包到服务器。
    </p>
    <pre><code class="prism language-bash">gpu-admin@gpu-server:~$ <span class="token function">ls</span>
NVIDIA-Linux-x86_64-550.90.07.run
</code></pre>
    <p>
     <code>
      .run 包
     </code>
     是需要编译的，所以下载一下gcc编译器，切换到root编译需要root权限。
    </p>
    <pre><code class="prism language-bash">root@gpu-server:/home/gpu-admin<span class="token comment"># apt-get install gcc g++ make cmake -y</span>
</code></pre>
    <p>
     通过file命令可以看出
     <code>
      NVIDIA-Linux-x86_64-550.90.07.run
     </code>
     是一个可执行的脚本
    </p>
    <pre><code class="prism language-bash">root@gpu-server:/home/gpu-admin<span class="token comment"># file NVIDIA-Linux-x86_64-550.90.07.run </span>
NVIDIA-Linux-x86_64-550.90.07.run: POSIX shell script executable <span class="token punctuation">(</span>binary data<span class="token punctuation">)</span>
</code></pre>
    <p>
     运行脚本
    </p>
    <pre><code class="prism language-bash">root@gpu-server:/home/gpu-admin<span class="token comment"># bash NVIDIA-Linux-x86_64-550.90.07.run</span>
</code></pre>
    <ul>
     <li>
      <p>
       安装程序检测到软件源仓库已存在驱动，建议从源仓库获取更好的兼容性，这里选择继续安装。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c891b7cb571a42a8814fbb3608e98056.png"/>
      </p>
     </li>
     <li>
      <p>
       图中的警告信息表明 NVIDIA 驱动程序安装程序（nvidia-installer）无法自动检测到 X Window 系统的库文件路径和模块路径。它猜测了路径 /usr/lib64 和 /usr/lib64/xorg/modules，但这些路径可能不正确或不完整。如果 X 系统无法找到 NVIDIA X 驱动模块，你需要安装 pkg-config 工具和适合你发行版的 X.Org SDK/开发包。
       <strong>
        这个不影响点击OK即可
       </strong>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/abffeb2494d8484d87ca17492baea229.png"/>
      </p>
     </li>
     <li>
      <p>
       图片显示的是在安装 NVIDIA 驱动程序时的一个选择提示。这个提示询问用户是否要安装 NVIDIA 的 32 位兼容性库。
       <strong>
        选择yes
       </strong>
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4a87907b1223461ebda62234f6d93f95.png"/>
      </p>
     </li>
     <li>
      <p>
       图片显示了一条警告信息：你正在安装的 NVIDIA 驱动程序包包含了 Vulkan 图形 API 的组件，但是，系统上没有检测到 Vulkan ICD（安装客户端驱动）加载器。我们不需要这个功能
       <strong>
        点击OK
       </strong>
       即可
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a69c942f3b284bff84b01fb457c94066.png"/>
      </p>
     </li>
     <li>
      <p>
       initramfs：是 Linux 系统启动时使用的一个临时文件系统，包含启动过程中所需的基本文件和驱动程序，通常情况下，建议选择 Rebuild initramfs。这样可以确保在启动时正确加载 NVIDIA 专有驱动程序，避免潜在的冲突或问题。重建 initramfs 通常是一个安全的操作，不会影响系统的正常运行。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/35e37c1923a941ce8f029e8efb114c0b.png"/>
      </p>
     </li>
    </ul>
    <p>
     安装完成：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ab81ae3db82d4537b31e0b9d04a424c7.png">
      <br/>
      使用
      <code>
       nvidia-smi
      </code>
      查看GPU信息
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/5abb34f4eae1412b9dbebc37f7c2b0ca.png"/>
      <br/>
      重启一下系统
     </img>
    </p>
    <pre><code class="prism language-bash">root@gpu-server:/home/gpu-admin<span class="token comment"># reboot</span>
</code></pre>
