     如何阅读webpack-bundle-analyzer分析生成的图
    </h1>

 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <ul>
     <li>
      <p>
       <strong>
        颜色表示
       </strong>
       ： 各个模块会用不同的颜色表示，可以帮助您快速区分不同的模块。
      </p>
     </li>
     <li>
      <p>
       <strong>
        大小表示
       </strong>
       ： 每个模块的大小在图表中会直接反映出来。较大的方块表示占用空间较大的模块，较小的方块表示占用空间较小的模块。
      </p>
     </li>
     <li>
      <p>
       <strong>
        模块名称
       </strong>
       ： 您可以看到每个模块的名称和路径，这样可以帮助您定位具体的模块。
      </p>
     </li>
     <li>
      <p>
       <strong>
        鼠标悬停查看详细信息
       </strong>
       ： 将鼠标悬停在某个模块上，会显示该模块的详细信息，包括模块名、路径、大小等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        点击展开子模块
       </strong>
       ： 点击某个模块，会展开并显示该模块的子模块详细信息，帮助您深入了解模块内部的结构。
      </p>
     </li>
    </ul>
    <p>
     <img alt="" height="241" src="https://i-blog.csdnimg.cn/direct/aee67eb0d87e46f98f76ce8d92c6dc75.png" width="865">
     </img>
    </p>
    <p>
     分析页面中，您会看到不同模块的大小统计信息。这里是解释
     <code>
      Stat size
     </code>
     ,
     <code>
      Parsed size
     </code>
     , 和
     <code>
      Gzipped size
     </code>
     的含义
    </p>
    <h4>
     Stat Size
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Stat size（原始大小）
       </strong>
       ：这是 Webpack 统计的原始文件大小，不包括任何加载或处理后的内容。这个大小通常表示的是打包前的模块大小。
      </p>
     </li>
     <li>
      <p>
       <strong>
        您的项目
       </strong>
       ：250.45 KB
      </p>
     </li>
    </ul>
    <h4>
     Parsed Size
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Parsed size（解析大小）
       </strong>
       ：这是经过 Webpack 和浏览器解析后的文件大小，通常会较小，因为 Webpack 会处理并优化代码。
      </p>
     </li>
     <li>
      <p>
       <strong>
        您的项目
       </strong>
       ：112.76 KB
      </p>
     </li>
    </ul>
    <h4>
     Gzipped Size
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Gzipped size（压缩大小）
       </strong>
       ：这是经过 gzip 压缩后的文件大小，通常是发布到生产环境时的文件大小。浏览器会解压这些文件并加载它们。
      </p>
     </li>
     <li>
      <p>
       <strong>
        您的项目
       </strong>
       ：20.46 KB
      </p>
     </li>
    </ul>
    <h4>
     Path
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        Path（路径）
       </strong>
       ：表示模块在项目中的文件路径。这有助于快速定位并优化占用较大空间的模块。
      </p>
     </li>
    </ul>
