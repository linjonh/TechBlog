    <h2>
     <a id="_0">
     </a>
     简介
    </h2>
    <p>
     最近翻到了吃灰已久的泰山派，是刚出来的时候用优惠券买的，当时价格挺便宜的，最近给它翻出来了，打算试试做个项目。买的泰山派容量是2G+16G，SOC芯片使用的是RK3566，搭载1TOP算力的NPU，并且具有丰富的接口，详细参数如下：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/189866d581ba4b579dcea88340b383fc.png#pic_center"/>
    </p>
    <p>
     官方的参考文档如下：https://wiki.lckfb.com/zh-hans/tspi-rk3566/
     <br/>
     由于官方给的是基于Ubuntu18.04的开发环境搭建步骤，然而Ubuntu目前稳定版已经出到了24.04，重新专门为它安装一个18.04版本的系统显然不合适。因此，在这里记录了一下基于Ubuntu24.04进行泰山派Linux开发环境搭建的步骤。
    </p>
    <h2>
     <a id="_6">
     </a>
     环境准备
    </h2>
    <ul>
     <li>
      Ubuntu24.04
     </li>
     <li>
      泰山派
     </li>
     <li>
      tspi_linux_sdk_repo_20240131.tar.gz
     </li>
     <li>
      buildroot_dl_4c7c9df616fb.tar.gz
     </li>
    </ul>
    <h2>
     <a id="python2_12">
     </a>
     python2安装
    </h2>
    <p>
     由于python2目前官方已经结束支持，Ubuntu24.04中已经不再默认安装python2了。然而泰山派的SDK编译依然要用到python2，因此我们这里首先进行python2的安装。
    </p>
    <h3>
     <a id="python27_14">
     </a>
     安装python2.7
    </h3>
    <p>
     首先运行如下命令，进行系统的更新。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update <span class="token operator">&amp;&amp;</span> <span class="token function">sudo</span> <span class="token function">apt</span> upgrade
</code></pre>
    <p>
     之后安装相关依赖
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> build-essential checkinstall libncursesw5-dev libssl-dev libsqlite3-dev tk-dev libgdbm-dev libc6-dev libbz2-dev libffi-dev
</code></pre>
    <p>
     然后我们下载python2的安装包，这里我选择的是2.7.18版本，也是python2的最后一个版本。
     <br/>
     下载页面：https://www.python.org/downloads/release/python-2716/
     <br/>
     下载链接：https://www.python.org/ftp/python/2.7.18/Python-2.7.18.tgz
     <br/>
     下载完成后，我们对压缩包进行解压
    </p>
    <pre><code class="prism language-bash"><span class="token function">tar</span> <span class="token parameter variable">-xvf</span> Python-2.7.18.tgz
</code></pre>
    <p>
     之后，我们进入解压后的目录，检查依赖项和编译器设置，使用configure命令生成 Makefile 文件，并且使用CFLAGS=-fPIC设置编译器生成位置无关的代码，否则可能会报错
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> Python-2.7.18
./configure --enable-optimizations <span class="token assign-left variable">CFLAGS</span><span class="token operator">=</span>-fPIC
</code></pre>
    <p>
     在完成上述准备后，我们就能够执行如下命令进行编译安装了
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">make</span> altinstall
</code></pre>
    <p>
     其中altinstall可以保证不会覆盖系统默认的 python 命令，从而避免影响系统自带的python3。
     <br/>
     最后，我们使用whereis命令，查看安装好的pyhon2.7的路径
    </p>
    <pre><code class="prism language-bash"><span class="token function">whereis</span> python2.7
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/eb204af72b514166857546e0e1b285ac.png#pic_center"/>
    </p>
    <p>
     可以看到我们的python2.7已经成功安装到了/usr/local目录下。
     <br/>
     下面我们运行如下命令，进入到python环境中，测试python2.7是否能够正常使用
    </p>
    <pre><code class="prism language-bash">python2.7
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/15ba9200da4c43cda7dd16d29bfd526d.png#pic_center"/>
    </p>
    <p>
     可以看到我们的python2.7已经能够正常运行。
    </p>
    <h3>
     <a id="pip_54">
     </a>
     pip安装
    </h3>
    <p>
     目前我们仅仅是安装了python2.7本体，还有许多python的功能包需要使用pip工具进行安装。接下来我们就来安装python2.7的pip工具，运行命令如下：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token function">curl</span>
<span class="token function">curl</span> https://bootstrap.pypa.io/pip/2.7/get-pip.py <span class="token parameter variable">-o</span> get-pip.py
<span class="token function">sudo</span> python2.7 get-pip.py
</code></pre>
    <p>
     安装完成后，我们运行如下命令进行测试
    </p>
    <pre><code class="prism language-bash">pip2.7 <span class="token parameter variable">--version</span>
</code></pre>
    <h3>
     <a id="python2_65">
     </a>
     添加python2软链接
    </h3>
    <p>
     虽然我们上面完成了python2.7的安装，但是系统默认依然还是python3。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/65bd06c2e84443c98813625a0aeb2166.png#pic_center"/>
    </p>
    <p>
     我们还需要使用如下命令添加软连接，使得python3和python2分别对应不同版本的python
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">ln</span> <span class="token parameter variable">-s</span> /usr/local/bin/python2.7 python2
</code></pre>
    <p>
     其中的/usr/local/bin/python2.7对应于你自己实际的python2.7安装路径
    </p>
    <h2>
     <a id="SDK_74">
     </a>
     SDK编译
    </h2>
    <h3>
     <a id="SDK_75">
     </a>
     解压SDK
    </h3>
    <p>
     首先运行如下命令，解压下载的SDK压缩包
    </p>
    <pre><code class="prism language-bash"><span class="token function">tar</span> <span class="token parameter variable">-xvzf</span> tspi_linux_sdk_repo_20240131.tar.gz
</code></pre>
    <p>
     之后运行如下命令，进行同步
    </p>
    <pre><code class="prism language-bash">.repo/repo/repo <span class="token function">sync</span> <span class="token parameter variable">-l</span> <span class="token parameter variable">-j88</span>
</code></pre>
    <p>
     同步完成后，会出现如下successfully界面
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/cc00cb77b2554303ac4f29c1d3a81c4b.png#pic_center"/>
    </p>
    <h3>
     <a id="dl_87">
     </a>
     dl库替代
    </h3>
    <p>
     由于buildroot编译时需要下载很多库，而如果网络不好的话，可能会出现buildroot的库下载失败报错的问题，因此我们使用了立创提供的已经下载好的库，直接进行替换即可。
    </p>
    <p>
     首先将buildroot_dl_4c7c9df616fb.tar.gz复制到SDK目录中，并运行如下命令，解压buildroot_dl_4c7c9df616fb文件到buildroot/dl目录下
    </p>
    <pre><code class="prism language-bash"><span class="token function">tar</span> <span class="token parameter variable">-xzf</span> buildroot_dl_4c7c9df616fb.tar.gz
</code></pre>
    <p>
     解压完成后，buildroot/dl目录下的部分文件展示如下：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3850e0d4ba774bf293783a08d9adb2cb.png#pic_center"/>
    </p>
    <h3>
     <a id="_97">
     </a>
     编译准备
    </h3>
    <p>
     在完成上述步骤后，我们就能够进行编译环境的安装了。
    </p>
    <h4>
     <a id="_99">
     </a>
     安装环境依赖
    </h4>
    <p>
     首先运行如下命令，安装编译环境所需要的依赖。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> <span class="token function">git</span> <span class="token function">make</span> openjdk-8-jdk git-core gnupg flex bison gperf build-essential <span class="token function">zip</span> <span class="token function">curl</span> zlib1g-dev libgl1-mesa-dev gcc-multilib g++-multilib libc6-dev-i386 x11proto-core-dev libx11-dev lib32z1-dev libxml2-utils xsltproc <span class="token function">unzip</span> <span class="token function">bc</span> imagemagick ccache schedtool libssl-dev clang
<span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> whiptail <span class="token function">ssh</span> libssl-dev liblz4-tool <span class="token function">expect</span> patchelf chrpath <span class="token function">gawk</span> texinfo chrpath diffstat binfmt-support qemu-user-static live-build bison flex fakeroot cmake gcc-multilib g++-multilib <span class="token function">unzip</span> device-tree-compiler ncurses-dev
</code></pre>
    <h4>
     <a id="_105">
     </a>
     板级配置
    </h4>
    <p>
     运行如下命令进行泰山派SDK的板级配置，并在弹出来的选项卡中选择BoardConfig-rk3566-tspi-v10.mk
    </p>
    <pre><code class="prism language-bash">./build.sh lunch
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7d5d4a96b7ca451196995d430c6f0e45.png#pic_center"/>
    </p>
    <h3>
     <a id="buildroot_111">
     </a>
     编译buildroot系统
    </h3>
    <p>
     在完成以上准备后，我们就能够编译buildroot系统了。
    </p>
    <p>
     由于buildroot中使用的python3.8.6，因此我们在编译时也必须使用python3.8环境才能正常进行，否则在编译时可能会报错
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/796e9728b91b483bb3f95f2e4cf2fecf.png#pic_center"/>
    </p>
    <p>
     这里我们使用pyenv工具，安装python3.8.19并使其生效。
    </p>
    <pre><code class="prism language-bash">pyenv <span class="token function">install</span> <span class="token number">3.8</span>.19
pyenv <span class="token builtin class-name">local</span> <span class="token number">3.8</span>.19  <span class="token comment"># 仅当前目录生效</span>
</code></pre>
    <p>
     之后需要设置环境变量RK_ROOTFS_SYSTEM为buildroot
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">RK_ROOTFS_SYSTEM</span><span class="token operator">=</span>buildroot
</code></pre>
    <p>
     为避免编译buildroot时报错multiple definition of bwriter_buffer，可以将如下代码添加到buildroot/package/squashfs/0001-include-sysmacros.patch文件的末尾（未验证是否为必须步骤）
    </p>
    <pre><code>diff -ruN squashfs-3de1687d7432ea9b302c2db9521996f506c140a3/squashfs-tools/mksquashfs.h squashfs-3de1687d7432ea9b302c2db9521996f506c140a3-patch/squashfs-tools/mksquashfs.h
--- squashfs-3de1687d7432ea9b302c2db9521996f506c140a3/squashfs-tools/mksquashfs.h       2015-12-07 09:42:03.000000000 +0800
+++ squashfs-3de1687d7432ea9b302c2db9521996f506c140a3-patch/squashfs-tools/mksquashfs.h 2025-01-02 16:59:52.994326895 +0800
@@ -133,7 +133,7 @@
 #define BLOCK_OFFSET 2
 
 extern struct cache *reader_buffer, *fragment_buffer, *reserve_cache;
-struct cache *bwriter_buffer, *fwriter_buffer;
+extern struct cache *bwriter_buffer, *fwriter_buffer;
 extern struct queue *to_reader, *to_deflate, *to_writer, *from_writer,
        *to_frag, *locked_fragment, *to_process_frag;
 extern struct append_file **file_mapping;
</code></pre>
    <p>
     最后运行如下命令，进行全编译。
    </p>
    <pre><code class="prism language-bash">./build.sh all
</code></pre>
    <p>
     如果只编译（u-Boot，kernel，Rootfs，Recovery），则在编译完成以后还需要再执⾏./mkfirmware.sh 进⾏固件打包，与./build.sh updateimg生成镜像，否则rockdev目录下生成img镜像不是最新的。
    </p>

     在首次进行编译时，会遇到电源配置选项。这里除了VCCIO4和VCCIO6选1.8V其他都选3V3。如果没有弹出电源选择框，则需要安装sudo apt install whiptail后再重新进行编译。

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1e4dfc5348644e168871ef5577615226.png#pic_center"/>

    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b18afc0382274654b7614fdb423a9b5b.png#pic_center"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/32aabe416bc34834a8c74e8cd796042a.png#pic_center"/>
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8275fce4063943b5908ec44a5e21d13c.png#pic_center"/>

     在Ubuntu24.04环境下，可能会在编译完kernel或buildroot后中断并且无报错信息，这时只需要重新执行build命令即可。
     <br/>
     编译成功后的输出如下：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9967b9767102434ba9e2a96b118dfa61.png#pic_center"/>

    <h3>
     <a id="_163">
     </a>
     打包固件
    </h3>
    <p>
     之后运行如下命令进行固件打包
    </p>
    <pre><code class="prism language-bash">./mkfirmware.sh
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7537c9cbb5e84be2a317727b9dd8001d.png#pic_center"/>
     <br/>
     运行如下命令查看固件，可以看到固件已经是最新的了。
    </p>
    <pre><code class="prism language-bash"><span class="token function">ls</span> ./rockdev <span class="token parameter variable">-l</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6490da5b8aeb47e8bc7f733bad86705d.png#pic_center"/>
    </p>
    <h3>
     <a id="_175">
     </a>
     打包镜像
    </h3>
    <p>
     生成的固件是零散的，并且不便于下载，因此我们可以使用如下命令将打包好的固件生成一个update.img镜像，便于后续的镜像下载。
    </p>
    <pre><code class="prism language-bash">./build.sh updateimg
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/940bca2f8088456c82bff6cf138178d6.png#pic_center"/>
    </p>
    <p>
     执行完毕后，就能够在rockdev生成update.img镜像文件了。
    </p>
    <h2>
     <a id="_184">
     </a>
     小结
    </h2>
    <p>
     至此，我们就完成了Ubuntu24.04环境下泰山派的SDK搭建。其中最值得注意的是在Ubuntu24.04环境下执行build命令可能会意外中断，如果未出现error报错，则无需执行clean操作，只需直接重新执行build命令即可。这样可以避免全量编译，从而大幅缩短构建时间，尤其是buildroot根文件系统的编译过程，因为它需要对所有功能包逐一编译，极为耗时。下一博客中，我们将介绍如何在linux下进行固件的烧录。
    </p>
