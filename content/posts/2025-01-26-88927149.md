    <h6>
     <a id="Cisco_Packet_Tracer_0">
     </a>
     实验环境：思科模拟器Cisco Packet Tracer
    </h6>
    <h6>
     <a id="IP_1">
     </a>
     IP地址划分：
    </h6>
    <pre><code>WEB：192.168.1.1
DNS：192.168.1.2
PC：192.168.1.3
公网IP：100.0.0.1  100.0.0.2  100.0.0.3  100.0.0.4
</code></pre>
    <h6>
     <a id="_6">
     </a>
     实验目的：
    </h6>
    <pre><code>内网能正常访问外网，外网只能访问内网的DNS服务器和WEB服务器
</code></pre>
    <p>
     实验拓扑图：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/43df381a77feb1a8b9845aaeb6070295.png"/>
    </p>
    <h6>
     <a id="_10">
     </a>
     在路由器上的配置：
    </h6>
    <pre><code> 1、添加标准ACL列表，允许所有
 Router(config)#access-list 1 permit any
 2、配置动态NAT，通过路由器g0/1的公网地址访问外网
 Router(config)#ip nat inside source list 1 interface gigabitEthernet 0/1 overload
 3、g0/1为出口，g0/0为入口
 Router(config)#interface gigabitEthernet 0/1
 Router(config-if)#ip nat outside
 Router(config)#interface gigabitEthernet 0/0
 Router(config-if)#ip nat inside
</code></pre>
    <p>
     到此，内网可以正常访问外网。接下来需要在外网服务器上正常访问内网的域名www.nb.com
     <br/>
     DNS服务器添加一个域名解析条目：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/49957454ebee8e27546f0931e35cd2a2.png"/>
    </p>
    <h6>
     <a id="_23">
     </a>
     在路由器上配置：
    </h6>
    <pre><code>1、静态NAT：WEB服务器---&gt;100.0.0.3
Router(config)#ip nat inside source static 192.168.1.1 100.0.0.3
2、静态NAT：DNS服务器---&gt;100.0.0.4
Router(config)#ip nat inside source static 192.168.1.2 100.0.0.4
外网服务器的DNS设为100.0.0.4(非常关键)
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/df733cc7aaea42771ed9dfb481e10d58.png">
      <br/>
      在外网服务器上输入域名：www.nb.com，可以正常访问
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/3c9a2085d2a4a2a683f7afe5b067160e.png"/>
     </img>
    </p>
    <h6>
     <a id="wwwnbcom_32">
     </a>
     外网访问内网www.nb.com流程：
    </h6>
    <p>
     外网服务器—&gt;查找DNS（100.0.0.4）—&gt;NAT转换为192.168.1.2—&gt;解析出www.nb.com指向的IP地址100.0.0.3（由于外网服务器是通过静态NAT与192.168.1.1通信，因此只能通过100.0.0.3去访问192.168.1.1）—&gt;NAT转换为192.168.1.1—&gt;实现WEB服务器的正常访问
