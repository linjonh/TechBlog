    <p>
     在面试中，关于
     <strong>
      Webpack 中
      <code>
       file-loader
      </code>
      和
      <code>
       url-loader
      </code>
      的区别
     </strong>
     是一个常见的问题。
    </p>
    <hr/>
    <h4>
     <a id="fileloader__urlloader__4">
     </a>
     <strong>
      <code>
       file-loader
      </code>
      和
      <code>
       url-loader
      </code>
      的区别
     </strong>
    </h4>
    <h5>
     <a id="1__6">
     </a>
     <strong>
      1. 功能定义
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         file-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        主要用于处理静态资源文件（如图片、字体等），将其复制到输出目录，并返回文件的 URL。
       </li>
       <li>
        适用于较大文件或需要单独存储的资源。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         url-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        是
        <code>
         file-loader
        </code>
        的扩展，它在处理文件时会先判断文件大小。
       </li>
       <li>
        如果文件大小小于指定的限制（
        <code>
         limit
        </code>
        ），则会将文件内容转换为 Base64 编码的 Data URL，直接嵌入到代码中；如果文件大小超过限制，则退化为
        <code>
         file-loader
        </code>
        的行为。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="2__15">
     </a>
     <strong>
      2. 使用场景
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         file-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        适用于需要将文件作为独立资源输出的场景，例如图片、字体文件等。
       </li>
       <li>
        通过配置可以指定文件的输出路径、文件名模板等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         url-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        适用于小文件，通过将文件内容直接嵌入到代码中，减少 HTTP 请求次数。
       </li>
       <li>
        例如，对于小图标或小图片，使用
        <code>
         url-loader
        </code>
        可以将它们转换为 Base64 编码，直接嵌入到 CSS 或 JavaScript 中。
       </li>
      </ul>
     </li>
    </ul>

    <h5>
     <a id="4__73">
     </a>
     <strong>
      4. 性能与优化
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         file-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        适合较大文件，避免将文件内容嵌入到代码中，减少打包体积。
       </li>
       <li>
        通过哈希值优化缓存，确保文件更新后客户端能够正确加载。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         url-loader
        </code>
       </strong>
       ：
      </p>
      <ul>
       <li>
        通过 Base64 编码减少 HTTP 请求次数，适合小文件。
       </li>
       <li>
        需要合理设置
        <code>
         limit
        </code>
        ，避免过多嵌入大文件导致打包体积过大。
       </li>
      </ul>
     </li>
    </ul>
