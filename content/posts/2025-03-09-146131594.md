    <h2>
     <a id="sphinxpdf_5">
     </a>
     sphinx支持导出pdf
    </h2>
    <p>
     sphinx本身是支持导出pdf的，命令如下：
    </p>
    <pre><code class="prism language-shell"><span class="token function">make</span> clean latexpdf
</code></pre>
    <p>
     支持中文的话，需要在
     <code>
      conf.py
     </code>
     增加如下信息：
    </p>
    <pre><code class="prism language-python">language <span class="token operator">=</span> <span class="token string">'zh_CN'</span>
<span class="token comment"># https://sphinx-rtd-trial.readthedocs.io/en/1.0.8/config.html</span>
<span class="token comment"># support chinese</span>
latex_elements <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
    <span class="token string">'preamble'</span><span class="token punctuation">:</span> <span class="token triple-quoted-string string">r'''
    \hypersetup{unicode=true}
    \usepackage{CJKutf8}
    \DeclareUnicodeCharacter{00A0}{\nobreakspace}
    \DeclareUnicodeCharacter{2203}{\ensuremath{\exists}}
    \DeclareUnicodeCharacter{2200}{\ensuremath{\forall}}
    \DeclareUnicodeCharacter{2286}{\ensuremath{\subseteq}}
    \DeclareUnicodeCharacter{2713}{x}
    \DeclareUnicodeCharacter{27FA}{\ensuremath{\Longleftrightarrow}}
    \DeclareUnicodeCharacter{221A}{\ensuremath{\sqrt{}}}
    \DeclareUnicodeCharacter{221B}{\ensuremath{\sqrt[3]{}}}
    \DeclareUnicodeCharacter{2295}{\ensuremath{\oplus}}
    \DeclareUnicodeCharacter{2297}{\ensuremath{\otimes}}
    \begin{CJK}{UTF8}{gbsn}
    \AtEndDocument{\end{CJK}}
    '''</span><span class="token punctuation">,</span>
    <span class="token string">'extraclassoptions'</span><span class="token punctuation">:</span> <span class="token string">'openany,oneside'</span><span class="token punctuation">,</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     需要安装如下几个软件：
    </p>
    <pre><code class="prism language-sh"><span class="token function">apt-get</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> texlive-fonts-recommended texlive-fonts-extra
<span class="token function">apt-get</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> texlive-latex-extra latexmk latex-cjk-all 
</code></pre>
    <p>
     PS：这几个软件都有点大，增加的时候注意看看是否需要。

    <h2>
     <a id="sphinx_49">
     </a>
     sphinx支持中文搜索
    </h2>
    <p>
     需要安装这个分词软件
    </p>
    <pre><code class="prism language-shell">pip <span class="token function">install</span> jieba
</code></pre>
    <p>
     需要在
     <code>
      conf.py
     </code>
     增加如下信息：
    </p>
    <pre><code class="prism language-python"><span class="token keyword">try</span><span class="token punctuation">:</span>
    <span class="token keyword">import</span> jieba
    html_search_options <span class="token operator">=</span> <span class="token punctuation">{<!-- --></span>
        <span class="token string">'type'</span><span class="token punctuation">:</span> <span class="token string">'test'</span><span class="token punctuation">,</span>
        <span class="token string">'language'</span><span class="token punctuation">:</span> <span class="token string">'zh'</span><span class="token punctuation">,</span>
        <span class="token string">'tokenizer'</span><span class="token punctuation">:</span> <span class="token string">'jieba.sphinx_zh'</span>
    <span class="token punctuation">}</span>
<span class="token keyword">except</span><span class="token punctuation">:</span>
    <span class="token keyword">pass</span>
</code></pre>
    <p>
     在 Sphinx 搜索引擎中配置 ‘tokenizer’: ‘jieba.sphinx_zh’ 以支持中文搜索，其核心原理是通过集成 中文分词技术 和 搜索引擎的索引机制，将中文文本拆解为有意义的词语单元，从而实现对中文内容的精准检索。以下是具体实现原理的分步解释：
    </p>
    <ol>
     <li>
      中文分词的挑战
     </li>
    </ol>
    <p>
     中文文本没有像英文那样的空格分隔符，因此需要借助分词工具将连续的字序列切分为独立的词语。例如：
    </p>
    <p>
     原始文本：“我爱自然语言处理”
     <br/>
     分词结果：[“我”, “爱”, “自然语言处理”] 或 [“我”, “爱”, “自然”, “语言”, “处理”]（取决于分词模式）
     <br/>
     分词质量直接影响搜索的准确性和召回率。错误的分词（如 “自然语” 和 “言处理”）会导致搜索失败。
    </p>
    <ol start="2">
     <li>
      Jieba 分词器的角色
     </li>
    </ol>
    <p>
     Jieba 是一个高效的中文分词工具，提供以下能力：
    </p>
    <p>
     精确模式：将句子最精确地切分（适合文本分析）。
     <br/>
     全模式：扫描所有可能的词语组合（速度快，可能冗余）。
     <br/>
     搜索引擎模式：在精确模式基础上，对长词再次切分（适合检索场景）。
     <br/>
     在 Sphinx 中，jieba.sphinx_zh 模块是专门为 Sphinx 设计的适配接口，使其能够调用 Jieba 的分词功能。
    </p>
    <ol start="3">
     <li>
      Sphinx 的索引和搜索流程
     </li>
    </ol>
    <p>
     配置 tokenizer: jieba.sphinx_zh 后，Sphinx 的工作流程如下：
    </p>
    <p>
     索引阶段（Indexing）
     <br/>
     文本预处理：读取文档中的中文文本。
     <br/>
     调用 Jieba 分词：使用 Jieba 将文本切分为词语列表。例如：
     <br/>
     输入：“清华大学位于北京”
     <br/>
     分词结果：[“清华大学”, “位于”, “北京”]
     <br/>
     构建倒排索引：将分词后的词语作为索引项（terms），记录它们在文档中的位置和频率，形成倒排索引结构。
     <br/>
     搜索阶段（Searching）
     <br/>
     用户输入查询：例如 “北京 大学”。
     <br/>
     查询分词：Jieba 将查询词切分为 [“北京”, “大学”]。
     <br/>
     匹配倒排索引：Sphinx 查找包含这些词语的文档，计算相关性并返回结果。
    </p>
    <ol start="4">
     <li>
      关键技术与优化
     </li>
    </ol>
    <p>
     词典机制：Jieba 内置核心词典（如 dict.txt），包含常见词语及其词频，确保基础分词的准确性。用户也可扩展自定义词典（如添加专业术语 “自然语言处理”）。
     <br/>
     算法优化：
     <br/>
     基于前缀词典的贪婪匹配：最大化匹配长词（如优先匹配 “清华大学” 而非 “清华” + “大学”）。
     <br/>
     隐马尔可夫模型（HMM）：识别未登录词（如人名、机构名）。
     <br/>
     停用词过滤：Sphinx 可配置忽略常见无意义词（如 “的”、“了”），减少索引冗余。
     <br/>
     词性标注（可选）：通过 Jieba 的词性标注功能，实现更细粒度的搜索过滤（如仅搜索名词）。
    </p>
    <ol start="5">
     <li>
      为何需要显式配置 Tokenizer？
     </li>
    </ol>
    <p>
     Sphinx 默认使用基于空格和标点的简单分词（适合英文），无法处理中文连续文本。通过替换为 jieba.sphinx_zh，Sphinx 的底层分词逻辑被重定向到 Jieba 的接口，从而支持中文语义单元的分词。
    </p>
    <ol start="6">
     <li>
      实际依赖和扩展
     </li>
    </ol>
    <p>
     安装 Jieba：需通过 pip install jieba 安装库，并确保 Sphinx 能调用 Python 模块。
     <br/>
     自定义词典：可通过 jieba.load_userdict(“custom_dict.txt”) 添加领域专有词汇。
     <br/>
     性能调优：调整 Jieba 的缓存机制和并行分词模式，以提升索引速度。
    </p>
    <p>
     总的来说，通过集成 Jieba 分词器，Sphinx 将中文文本转化为有意义的词语单元，解决了中文全文检索的核心难题。这一配置直接影响了索引构建和查询处理的底层逻辑，使 Sphinx 能够像处理英文一样高效地支持中文搜索，核心依赖分词准确性和搜索引擎的倒排索引技术
    </p>
    <h2>
     <a id="sphinxJS_127">
     </a>
     sphinx支持自定义样式/JS
    </h2>
    <p>
     需要在
     <code>
      conf.py
     </code>
     增加如下信息：
    </p>
    <pre><code class="prism language-python">html_static_path <span class="token operator">=</span> <span class="token punctuation">[</span><span class="token string">'_static'</span><span class="token punctuation">]</span>
html_js_files <span class="token operator">=</span> <span class="token punctuation">[</span><span class="token string">'yeshen.js'</span><span class="token punctuation">,</span><span class="token string">'yeshen_1.js'</span><span class="token punctuation">]</span>
html_css_files <span class="token operator">=</span> <span class="token punctuation">[</span><span class="token string">'yeshen.css'</span><span class="token punctuation">]</span>
</code></pre>
    <p>
     创建
     <code>
      _static
     </code>
     目录，然后在其中增加
     <code>
      yeshen.js
     </code>
     <code>
      yeshen.css
     </code>
     <code>
      yeshen_1.js
     </code>
     等几个文件
     <br/>
     这几个文件会在生成的index.html中引用。
    </p>
    <p>
     目录说明：
    </p>
    <pre><code>conf.py
_static
    |-- yeshen.js
    |-- yeshen.css
    |__ yeshen_1.js
</code></pre>
    <h2>
     <a id="sphinx_151">
     </a>
     sphinx支持自定义模版
    </h2>
    <ol>
     <li>
      创建模板目录
     </li>
    </ol>
    <p>
     在项目根目录下创建
     <code>
      _templates
     </code>
     文件夹：
    </p>
    <pre><code class="prism language-bash"><span class="token function">mkdir</span> source/_templates
</code></pre>
    <ol start="2">
     <li>
      配置模板路径
     </li>
    </ol>
    <p>
     在
     <code>
      conf.py
     </code>
     中指定模板路径：
    </p>
    <pre><code class="prism language-python">templates_path <span class="token operator">=</span> <span class="token punctuation">[</span><span class="token string">'_templates'</span><span class="token punctuation">]</span>
</code></pre>
    <ol start="3">
     <li>
      覆盖现有模板
     </li>
    </ol>
    <p>
     查找目标模板：确定你要修改的模板文件（如 layout.html, footer.html 等）。参考你所使用的主题的模板结构。
     <br/>
     创建同名文件：在 _templates 下创建同名文件，Sphinx将优先使用你的模板。
     <br/>
     示例：修改页脚（覆盖 footer.html）
    </p>
    <pre><code class="prism language-html"><span class="token comment">&lt;!-- source/_templates/footer.html --&gt;</span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>div</span> <span class="token attr-name">class</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>footer<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>
    © 2025 @yeshen.org | 自定义页脚内容
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>div</span><span class="token punctuation">&gt;</span></span>
</code></pre>
    <ol start="4">
     <li>
      扩展现有模板（继承并修改）
      <br/>
      使用Jinja2的 {% extends %} 继承原模板，然后覆盖特定块。
     </li>
    </ol>
    <p>
     示例：在侧边栏添加内容
    </p>
    <pre><code class="prism language-html"><span class="token comment">&lt;!-- source/_templates/sidebar.html --&gt;</span>
{% extends "!sidebar.html" %}

{% block sidebarlogo %}
    {<!-- -->{ super() }}  <span class="token comment">&lt;!-- 保留原有内容 --&gt;</span>
    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>div</span> <span class="token attr-name">class</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>custom-sidebar-section<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>
        你的自定义内容
    <span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>div</span><span class="token punctuation">&gt;</span></span>
{% endblock %}
</code></pre>
    <ol start="5">
     <li>
      创建全新模板
     </li>
    </ol>
    <p>
     在 _templates 中新建文件，并通过 … template:: 指令在RST中引用。
    </p>
    <ol start="6">
     <li>
      使用自定义模板
     </li>
    </ol>
    <p>
     HTML主题：确保 conf.py 中设置了正确的主题（如 html_theme = ‘alabaster’）。
     <br/>
     构建文档：
    </p>
    <pre><code class="prism language-sh">sphinx-build <span class="token parameter variable">-b</span> html <span class="token builtin class-name">source</span> build
</code></pre>
    <p>
     常用模板文件
     <br/>
     layout.html: 整体布局
     <br/>
     header.html: 页眉
     <br/>
     footer.html: 页脚
     <br/>
     sidebar.html: 侧边栏
     <br/>
     page.html: 单页内容
     <br/>
     提示
     <br/>
     使用 sphinx-build -b html source build 重新生成文档以查看更改。
     <br/>
     查看主题的原始模板文件（如经典主题模板）获取可覆盖的块名。
     <br/>
     通过以上步骤，你可以灵活定制Sphinx文档的样式和结构
    </p>

</code></pre>
