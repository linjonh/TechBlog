    <h2>
     定位
    </h2>
    <p>
     鸿蒙OS分布式操作系统简介鸿蒙系统（HarmonyOS)，是第一款基于微内核的全场景分布式OS，是华为自主研发的操作系统。而且HarmonyOS是一款面向万物互联时代的、全新的分布式操作系统。
    </p>
    <p>
     在传统的单设备系统能力基础上，HarmonyOS提出了基于同一套系统能力、适配多种终端形态的分布式理念，能够支持手机、平板、智能穿戴、智慧屏、车机、PC、智能音箱、耳机、AR/VR眼镜等多种终端设备，提供全场景（移动办公、运动健康、社交通信、媒体娱乐等）业务能力。
    </p>
    <p>
     <img alt="" height="1080" src="https://i-blog.csdnimg.cn/blog_migrate/f3cb6c882e59e2775d08deedb9df5a3a.png" width="1200"/>
    </p>
    <h2 id="ZH-CN_TOPIC_0000001175301813__p1364801515501">
     HarmonyOS有三大特征：
    </h2>
    <ul id="ZH-CN_TOPIC_0000001175301813__ul176481115165011">
     <li id="ZH-CN_TOPIC_0000001175301813__li8648121515508">
      搭载该操作系统的设备在系统层面融为一体、形成超级终端，让设备的硬件能力可以弹性扩展，实现设备之间
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="硬件互助，资源共享">
       硬件互助，资源共享
      </a>
      。
      <p id="ZH-CN_TOPIC_0000001175301813__p1648151511504">
       对消费者而言，HarmonyOS能够将生活场景中的各类终端进行能力整合，实现不同终端设备之间的快速连接、能力互助、资源共享，匹配合适的设备、提供流畅的全场景体验。
      </p>
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li7648215145013">
      面向开发者，实现
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="一次开发，多端部署">
       一次开发，多端部署
      </a>
      。
      <p id="ZH-CN_TOPIC_0000001175301813__p264841575014">
       对应用开发者而言，HarmonyOS采用了多种分布式技术，使应用开发与不同终端设备的形态差异无关，从而让开发者能够聚焦上层业务逻辑，更加便捷、高效地开发应用。
      </p>
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li18648131515505">
      一套操作系统可以满足不同能力的设备需求，实现
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="统一OS，弹性部署">
       统一OS，弹性部署
      </a>
      。
      <p id="ZH-CN_TOPIC_0000001175301813__p86482015165010">
       对设备开发者而言，HarmonyOS采用了组件化的设计方案，可根据设备的资源能力和业务特征灵活裁剪，满足不同形态终端设备对操作系统的要求。
      </p>
     </li>
    </ul>
    <p id="ZH-CN_TOPIC_0000001175301813__p264817152507">
     HarmonyOS提供了支持多种开发语言的API，供开发者进行应用开发。支持的开发语言包括ArkTS、JS（JavaScript）、C/C++ 、Java。
    </p>
    <h3 id="section42071731152111">
     技术架构
    </h3>
    <p>
    </p>
    <p id="ZH-CN_TOPIC_0000001175301813__p2923125218406">
     HarmonyOS整体遵从分层设计，从下向上依次为：内核层、系统服务层、框架层和应用层。系统功能按照“系统 &gt; 子系统 &gt; 功能/模块”逐级展开，在多设备部署场景下，支持根据实际需求裁剪某些非必要的子系统或功能/模块。HarmonyOS技术架构如下所示。
    </p>
    <p id="ZH-CN_TOPIC_0000001175301813__p2066663713410">
    </p>
    <p class="img-center">
     <img alt="" height="831" src="https://i-blog.csdnimg.cn/blog_migrate/a7107ef598fece87a77fbd1243f69e3a.png" width="1200"/>
    </p>
    <h4 id="section17198101363120">
     内核层
    </h4>
    <p>
    </p>
    <ul id="ZH-CN_TOPIC_0000001175301813__ul37592349313">
     <li id="ZH-CN_TOPIC_0000001175301813__li4297131383112">
      内核子系统：HarmonyOS采用多内核设计，支持针对不同资源受限设备选用适合的OS内核。内核抽象层（KAL，Kernel Abstract Layer）通过屏蔽多内核差异，对上层提供基础的内核能力，包括进程/线程管理、内存管理、文件系统、网络管理和外设管理等。
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li11760183493113">
      驱动子系统：
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="硬件驱动框架（HDF）">
       硬件驱动框架（HDF）
      </a>
      是HarmonyOS硬件生态开放的基础，提供统一外设访问能力和驱动开发、管理框架。
     </li>
    </ul>
    <h4 id="section81951740203118">
     系统服务层
    </h4>
    <p>
    </p>
    <p id="ZH-CN_TOPIC_0000001175301813__p10298213103118">
     系统服务层是HarmonyOS的核心能力集合，通过框架层对应用程序提供服务。该层包含以下几个部分：
    </p>
    <ul id="ZH-CN_TOPIC_0000001175301813__ul329931323113">
     <li id="ZH-CN_TOPIC_0000001175301813__li22981313113117">
      系统基本能力子系统集：为分布式应用在HarmonyOS多设备上的运行、调度、迁移等操作提供了基础能力，由分布式软总线、分布式数据管理、分布式任务调度、方舟多语言运行时、公共基础库、多模输入、图形、安全、AI等子系统组成。其中，方舟运行时提供了C/C++/JS多语言运行时和基础的系统类库，也为使用方舟编译器静态化的Java程序（即应用程序或框架层中使用Java语言开发的部分）提供运行时。
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li8299101316313">
      基础软件服务子系统集：为HarmonyOS提供公共的、通用的软件服务，由事件通知、电话、多媒体、DFX（Design For X） 、
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="MSDP">
       MSDP
      </a>
      &amp;
      <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="DV">
       DV
      </a>
      等子系统组成。
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li6299151393118">
      增强软件服务子系统集：为HarmonyOS提供针对不同设备的、差异化的能力增强型软件服务，由智慧屏专有业务、穿戴专有业务、IoT专有业务等子系统组成。
     </li>
     <li id="ZH-CN_TOPIC_0000001175301813__li529921319317">
      硬件服务子系统集：为HarmonyOS提供硬件服务，由位置服务、生物特征识别、穿戴专有硬件服务、IoT专有硬件服务等子系统组成。
     </li>
    </ul>
    <p id="ZH-CN_TOPIC_0000001175301813__p18846133113211">
     根据不同设备形态的部署环境，基础软件服务子系统集、增强软件服务子系统集、硬件服务子系统集内部可以按子系统粒度裁剪，每个子系统内部又可以按功能粒度裁剪。
    </p>
    <h4 id="section1946017713210">
     框架层
    </h4>
    <p>
    </p>
    <p id="ZH-CN_TOPIC_0000001175301813__p2904156324">
     框架层为HarmonyOS应用开发提供了ArkTS/JS/C/C++/Java等多语言的用户程序框架，两种UI框架（包括适用于ArkTS/JS语言的方舟开发框架即ArkUI、适用于Java语言的Java UI框架），以及各种软硬件服务对外开放的多语言框架API。根据系统的组件化裁剪程度，HarmonyOS设备支持的API也会有所不同。
    </p>
    <h4 id="section16208165063117">
     应用层
    </h4>
    <p>
    </p>
    <p id="ZH-CN_TOPIC_0000001175301813__p106141929123216">
     应用层包括系统应用和第三方非系统应用。HarmonyOS的应用由一个或多个
     <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="FA（Feature Ability）">
      FA（Feature Ability）
     </a>
     或
     <a href="https://docs.qq.com/doc/DSHlBQWh2d3lmYVlp" rel="nofollow" title="PA（Particle Ability）">
      PA（Particle Ability）
     </a>
     组成。其中，FA有UI界面，提供与用户交互的能力；而PA无UI界面，提供后台运行任务的能力以及统一的数据访问抽象。FA在进行用户交互时所需的后台数据访问也需要由对应的PA提供支撑。基于FA/PA开发的应用，能够实现特定的业务功能，支持跨设备调度与分发，为用户提供一致、高效的应用体验。
    </p>
