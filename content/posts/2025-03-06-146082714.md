    <h3>
     <a id="_0">
     </a>
     题目：
    </h3>
    <p>
     表：Tweets
    </p>
    <p>
     ±---------------±--------+
     <br/>
     | Column Name | Type |
     <br/>
     ±---------------±--------+
     <br/>
     | tweet_id | int |
     <br/>
     | content | varchar |
     <br/>
     ±---------------±--------+
     <br/>
     在 SQL 中，tweet_id 是这个表的主键。
     <br/>
     content 只包含美式键盘上的字符，不包含其它特殊字符。
     <br/>
     这个表包含某社交媒体 App 中所有的推文。
    </p>
    <p>
     查询所有无效推文的编号（ID）。当推文内容中的字符数严格大于 15 时，该推文是无效的。
    </p>
    <p>
     以任意顺序返回结果表。
    </p>
    <p>
     <strong>
      示例 1：
     </strong>
    </p>
    <p>
     输入：
     <br/>
     Tweets 表：
     <br/>
     ±---------±---------------------------------+
     <br/>
     | tweet_id | content |
     <br/>
     ±---------±---------------------------------+
     <br/>
     | 1 | Vote for Biden |
     <br/>
     | 2 | Let us make America great again! |
     <br/>
     ±---------±---------------------------------+
    </p>
    <p>
     输出：
     <br/>
     ±---------+
     <br/>
     | tweet_id |
     <br/>
     ±---------+
     <br/>
     | 2 |
     <br/>
     ±---------+
     <br/>
     解释：
     <br/>
     推文 1 的长度 length = 14。该推文是有效的。
     <br/>
     推文 2 的长度 length = 32。该推文是无效的。
    </p>
    <h3>
     <a id="_38">
     </a>
     思路：
    </h3>
    <p>
     这里考虑到要判断字符的长度，所以引入了Len()函数（mysql中引入length（）函数），返回文本字段中值的长度。
     <br/>
     由于需要找的是无效推文的编号（ID），那么select寻找的就是tweet_id，其次给出符合的条件为”推文内容中的字符数严格大于 15 时，该推文是无效“，所以代码中的判断条件为：LEN(content) &gt; 15，使用where关键字在表中过滤不符合条件的表格行即可。
    </p>
    <h3>
     <a id="_43">
     </a>
     代码：
    </h3>
    <pre><code class="prism language-sql"><span class="token keyword">select</span> tweet_id
<span class="token keyword">from</span> Tweets
<span class="token keyword">where</span> <span class="token function">LEN</span><span class="token punctuation">(</span>content<span class="token punctuation">)</span> <span class="token operator">&gt;</span> <span class="token number">15</span>
</code></pre>
