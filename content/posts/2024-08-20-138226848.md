    <h2 id="bSOsi">
     三、
     <strong>
      安装
     </strong>
     <strong>
      nginx
     </strong>
     <strong>
      （
     </strong>
     <strong>
      web
     </strong>
     <strong>
      服务器）
     </strong>
    </h2>
    <p id="u0cfb5acf">
     在部署flask项目时候，因为flask是一个轻量级的Python Web框架，其本身并不包含一个成熟的Web服务器组件，而是依赖于内置的简易服务器（werkzeug）
    </p>
    <h3 id="ACcS5">
     1. 更新系统软件包：
    </h3>
    <pre id="i8UZv"><code>sudo yum update</code></pre>
    <h3 id="WlmHp">
     2. 安装EPEL存储库
    </h3>
    <pre id="x3GdT"><code>sudo yum install epel-release</code></pre>
    <p id="u11b8dbef">
    </p>
    <p class="img-center">
     <img alt="" height="494" id="u3327c55d" src="https://i-blog.csdnimg.cn/blog_migrate/f2a2496bdcae38ce829bc9539dcaddbd.png" width="939"/>
    </p>
    <h3 id="hgkc4">
     3. 安装Nginx
    </h3>
    <pre id="BUP9w"><code>sudo yum install nginx</code></pre>
    <p id="ue4e19a4f">
    </p>
    <p class="img-center">
     <img alt="" height="261" id="uf3026cc6" src="https://i-blog.csdnimg.cn/blog_migrate/d5e784e668725e8eaf4c82dbfe572a77.png" width="1026"/>
    </p>
    <h3 id="kndRl">
     4. 启动Nginx服务并设置开机自启
    </h3>
    <pre id="JoF3N"><code>sudo systemctl start nginx</code></pre>
    <p id="u1b5c27c0">
     为了让Nginx在系统启动时自动运行，执行：
    </p>
    <pre id="VVPYe"><code>sudo systemctl enable nginx</code></pre>
    <h3 id="WNy85">
     5. 验证Nginx安装
    </h3>
    <p id="ue0ecdc47">
     (1) 命令行检查Nginx服务状态：
    </p>
    <pre id="tZ3U3"><code>sudo systemctl status nginx</code></pre>
    <p id="u553575f8">
    </p>
    <p class="img-center">
     <img alt="" height="359" id="u2ee6c95a" src="https://i-blog.csdnimg.cn/blog_migrate/cc1a28142e63bf72d45119c9f4e41693.png" width="1107"/>
    </p>
    <p id="u96d0d34e">
     (2) 浏览器访问服务器的公共IP地址
    </p>
    <p id="u2e21db29">
     Nginx默认监听在80端口，外部想使用nginx就需要云服务器开发80端口，在云服务器控制台的左侧导航栏找到安全组，点进安全组
    </p>
    <p id="u5071e279">
    </p>
    <p class="img-center">
     <img alt="" height="365" id="ub1382b53" src="https://i-blog.csdnimg.cn/blog_migrate/c793660d20d4248003a88c6851d08e73.png" width="1108"/>
    </p>
    <p id="ubb169c0c">
     点进后：
    </p>
    <p id="u1d22e332">
    </p>
    <p class="img-center">
     <img alt="" height="258" id="ud1032760" src="https://i-blog.csdnimg.cn/blog_migrate/4bd125aee3aa29e8294cef7e373bd0b0.png" width="1106"/>
    </p>
    <p id="ue3d36a64">
     点击手动添加，授权策略：默认（允许）；优先级默认：默认（1）；协议类型：默认（TCP）；端口： 80 ；授权对象：0.0.0.0/0。
    </p>
    <p id="ua27abcd9">
     表示允许所有外部IP地址访问，然后点击保存。
    </p>
    <p id="u51854058">
    </p>
    <p class="img-center">
     <img alt="" height="248" id="u4ef0c090" src="https://i-blog.csdnimg.cn/blog_migrate/f886718230e68013886190a9edb463bf.png" width="1107"/>
    </p>
    <p id="u5c5a4730">
     如果不能访问，可能是防火墙的问题，关闭防火墙或者允许nginx端口。
    </p>
    <h2 id="X8Zjp">
     四、
     <strong>
      安装
     </strong>
     <strong>
      python
     </strong>
     <strong>
      虚拟环境
     </strong>
    </h2>
    <h3 id="GZ80e">
     1.检查系统是否已安装Python3及其pip3:
    </h3>
    <pre id="da2wM"><code>python3 --version
pip3 --version</code></pre>
    <p id="uc7c490a5">
    </p>
    <p class="img-center">
     <img alt="" height="166" id="ude446d63" src="https://i-blog.csdnimg.cn/blog_migrate/be849652606e8eb5c833f1653b83ce74.png" width="876"/>
    </p>
    <p id="u19e60ae1">
     如果没有安装或版本过低：
    </p>
    <pre id="MWhWW"><code>sudo yum install python3 python3-devel python3-pip</code></pre>
    <h3 id="Ug178">
     2. 创建Python虚拟环境
    </h3>
    <p id="ufa7bcc4c">
     在root目录下创建一个新的虚拟环境：
    </p>
    <pre id="F3z3q"><code>mkdir my_flaskProject
cd my_flaskProject
python3 -m venv my_venv</code></pre>
    <p id="u6e599acc">
     这里my_venv是虚拟机环境的名称，可以自己命名。
    </p>
    <p id="ufcbd41f1">
    </p>
    <p class="img-center">
     <img alt="" height="198" id="u073a3421" src="https://i-blog.csdnimg.cn/blog_migrate/1d928397b27afd960d3766a08c199691.png" width="1028"/>
    </p>
    <h3 id="bik0m">
     3. 激活虚拟环境：
    </h3>
    <pre id="nU4EP"><code>source my_venv/bin/activate</code></pre>
    <p id="uc59b45ef">
    </p>
    <p class="img-center">
     <img alt="" height="61" id="ub3a26f8c" src="https://i-blog.csdnimg.cn/blog_migrate/a72889958ed23049195a7850b4c1a869.png" width="1108"/>
    </p>
    <p id="u4aed9ba9">
     激活后，命令行提示符前会显示虚拟环境的名字（my_venv）,这表明当前正处于此虚拟环境中运行命令，此时安装的Python包将只在这个环境中生效，不会影响全局Python环境。
    </p>
    <h3 id="ODmwa">
     4. 安装项目所需Python包
    </h3>
    <p id="ub786e0cd">
     在虚环境中，可以使用pip安装Flask和其他项目依赖：
    </p>
    <pre id="H07hq"><code>pip install Flask
pip install -r requirements.txt  # 如果存在requirements.txt文件</code></pre>
    <h3 id="Lztgv">
     5. 退出虚拟环境
    </h3>
    <p id="u4bb8aaf2">
     完成开发或部署任务后，要退出当前虚拟环境，只需运行：
    </p>
    <pre id="A76Hr"><code>deactivate</code></pre>
    <h2 id="Ea9L9">
     五、
     <strong>
      导入项目文件
     </strong>
    </h2>
    <h3 id="IGwf2">
     1. 确保虚拟环境已激活
    </h3>
    <p id="udab82c0d">
     如果未激活：
    </p>
    <pre id="H3IjZ"><code>source /root/my_flaskProject/my_venv/bin/activate</code></pre>
    <p id="u981d0c20">
     可能会提示错误信息：Permission denied 表示当前没有足够的权限来执行 activate 脚本
    </p>
    <p id="u6715f138">
    </p>
    <p class="img-center">
     <img alt="" height="51" id="uf78de02a" src="https://i-blog.csdnimg.cn/blog_migrate/ed03507c1ef2ddea4f20d8a2ae9f5f12.png" width="1108"/>
    </p>
    <p id="uadbd614a">
     运行如下命令：
    </p>
    <pre id="RKzKd"><code>chmod +x /root/my_flaskProject/my_venv/bin/activate</code></pre>
    <p id="ud3a57eab">
     赋予 activate 脚本执行权限
    </p>
    <p id="uf0e27386">
     再次激活，前面出现（my_venv）激活成功！
    </p>
    <p id="ue9b0ae83">
    </p>
    <h3 id="ivRFz">
     2. 配置nginx
    </h3>
    <p id="ue4d04343">
     编辑Nginx的服务器配置文件，通常位于/etc/nginx/sites-available/目录下，
     <strong>
      如果没有sites-available这个目录，可按照如下进行配置：
     </strong>
    </p>
    <p id="u07c995a7">
     打开/etc/nginx/nginx.conf文件，在http{...}块内添加
    </p>
    <pre id="FtQld"><code>server {
  listen 80;  # 监听公网端口（此处为标准HTTP端口80）
  server_name your_domain.com;  # 替换为您的域名或公网IP
  
  location / {
  proxy_pass http://localhost:5000;  # 替换为项目实际监听的本地端口
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  }
}</code></pre>
    <p id="u253d9667">
     这段配置是告诉Nginx监听公网端口80，并将所有到此服务器的请求转发到本地端口8080（是你的python项目本地运行的端口）。同时设置了必要的代理头信息。
    </p>
    <p id="ud56dbc29">
     重启Nginx服务：
    </p>
    <pre id="Ux6Nd"><code>sudo systemctl restart nginx</code></pre>
    <p id="u5717cf87">
     检查配置和日志
    </p>
    <p id="ube6c4ec9">
     nginx -t
    </p>
    <p id="u3623e958">
    </p>
    <h3 id="ftFq8">
     3. 确保flask项目中入口文件host='0.0.0.0'
    </h3>
    <p id="ue2a8f6a7">
     端口号为上述在nginx配置的项目实际监听的端口号。
    </p>
    <pre id="mVY9m"><code>if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=True)</code></pre>
    <p id="u57f41a30">
    </p>
    <h3 id="ZdCBy">
     4. 将本地flask项目文件（包括源代码、配置文件、静态资源等）复制到虚拟环境所在的目录下
    </h3>
    <pre id="w4w9X"><code>cp -r /path/to/your_project /root/my_flaskProject/my_venv/</code></pre>
    <p id="ub7c9e2af">
     设置环境变量：如果应用初始化在app.py文件中则：
    </p>
    <pre id="paMXr"><code>export FLASK_APP=app.py</code></pre>
    <p id="u3210a34f">
     如果是其他文件，例如入口在myflask.py中则：
    </p>
    <p id="ube76db94">
     export FLASK_APP=myflask （去掉.py）
    </p>
    <p id="ue5c9c571">
     运行：flask run
    </p>
    <p id="uce93485e">
    </p>
    <pre id="EQGtz"><code>netstat -tulpn | grep :5000</code></pre>
    <p id="ua7f6b37b">
     查看端口5000进程
    </p>
    <p id="ubb9fe27d">
     杀死进程：kill -9 进程id
    </p>
    <p id="u9a83785b">
    </p>
    <h3 id="iAfvW">
     5. 在虚拟环境中安装Gumicorn
    </h3>
    <p id="u275faef0">
     虚拟环境（如使用 virtualenv 或 conda 创建的环境）为项目提供了独立的Python运行环境。在其中安装Gunicorn和其他依赖，可以确保项目使用的是特定版本的软件包，避免与其他项目或系统全局包产生版本冲突。这样，每个Flask应用可以拥有自己的依赖集，易于管理和更新，且不会影响到服务器上其他应用程序。
    </p>
    <pre id="oPhbQ"><code>pip install gunicorn</code></pre>
    <p id="u6d60b9b8">
     配置Gunicorn运行Flask应用： 创建一个Gunicorn启动脚本：
    </p>
    <pre id="AaWUs"><code>vim gunicorn_start.sh</code></pre>
    <pre id="qg9NV"><code>#!/bin/bash

# 设置工作目录（这里替换成您的Flask应用所在的目录）
working_directory=" /root/my_flaskProject/my_venv/"

# 切换到工作目录
cd "$working_directory"

# 设置环境变量，如果适用
export FLASK_APP=myflask.py
export FLASK_ENV=production</code></pre>
    <p id="u9c065c5a">
    </p>
    <p id="u581d37d2">
     # 启动Gunicorn，指定监听地址和端口，以及应用入口点
    </p>
    <pre id="XckkM"><code>exec gunicorn --bind 0.0.0.0:5000 myflask:app</code></pre>
    <p id="ua8a90a48">
    </p>
    <p id="u9c88e224">
     给启动脚本赋予权限：
    </p>
    <pre id="arC9k"><code>chmod +x gunicorn_start.sh</code></pre>
    <p id="uf45360f0">
    </p>
    <p id="u07f68490">
     启动Gunicorn服务：
    </p>
    <pre id="jwFec"><code>./gunicorn_start.sh</code></pre>
    <p id="ub5a3b2a1">
    </p>
    <p id="u02e1d67d">
     成功后，可以查看进程
    </p>
    <p id="u07087595">
     然后在nginx.conf配置（刚刚在导入项目时其实已经配置过...）：
    </p>
    <pre id="s9H9S"><code>server {
		listen 80;  # 监听公网端口（此处为标准HTTP端口80）
		server_name 114.55.118.198;  # 替换为您的域名或公网IP

		location / {
			proxy_pass http://localhost:5000;  # 替换为项目实际监听的本地端口
			proxy_set_header Host $host;
			proxy_set_header X-Real-IP $remote_addr;
			proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
			proxy_set_header X-Forwarded-Proto $scheme;
			proxy_redirect off;
			proxy_buffering off;
			proxy_http_version 1.1;
			proxy_set_header Upgrade $http_upgrade;
			proxy_set_header Connection "upgrade";
			proxy_read_timeout 600s;
		}
	}</code></pre>
    <p id="u51a25919">
    </p>
    <p id="ue700d41a">
     完成配置后，执行以下操作以检查配置文件的正确性并重启Nginx服务：
