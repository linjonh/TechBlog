    <h2 id="ccmon" name="ccmon" style="background-color:transparent">
     一、前言
    </h2>
    <p id="udfca14a6">
     C# 是一种面向对象的编程语言。在面向对象的程序设计方法中，程序由各种相互交互的对象组成。相同种类的对
    </p>
    <p id="u6258c74b">
     象通常具有相同的类型，或者说，是在相同的 class 中。
    </p>
    <p id="u8fe44c6d">
     例如，以 Rectangle（矩形）对象为例。
    </p>
    <p id="u09ddef33">
     它具有 length 和 width 属性。根据设计，它可能需要接受这些属性值、计算面积和显示细节。
    </p>
    <p id="uaf3a06a1">
     让我们来看看一个 Rectangle（矩形）类的实现，并借此讨论 C# 的基本语法：
    </p>
    <p id="u7d61108e">
     <strong>
      实例
     </strong>
    </p>
    <pre id="VPpli"><code>using System;
namespace RectangleApplication
{
    class Rectangle
    {
        // 成员变量
        double length;
        double width;
        public void Acceptdetails()
        {
            length = 4.5;    
            width = 3.5;
        }
        public double GetArea()
        {
            return length * width;
        }
        public void Display()
        {
            Console.WriteLine("Length: {0}", length);
            Console.WriteLine("Width: {0}", width);
            Console.WriteLine("Area: {0}", GetArea());
        }
    }
    
    class ExecuteRectangle
    {
        static void Main(string[] args)
        {
            Rectangle r = new Rectangle();
            r.Acceptdetails();
            r.Display();
            Console.ReadLine();
        }
    }
}</code></pre>
    <p id="u4fff2559">
     当上面的代码被编译和执行时，它会产生下列结果：
    </p>
    <pre id="ndVRO"><code>Length: 4.5
Width: 3.5
Area: 15.75</code></pre>

    <h2 id="d3cHU" name="d3cHU">
     八、标识符
    </h2>
    <p id="udb754690">
     标识符是用来识别类、变量、函数或任何其它用户定义的项目。在 C# 中，类的命名必须遵循如下基本规则：
    </p>
    <ul>
     <li id="ucd8427a4">
      标识符必须以字母、下划线或 @ 开头，后面可以跟一系列的字母、数字（ 0 - 9 ）、下划线（ _ ）、@。
     </li>
     <li id="u85df8f94">
      标识符中的第一个字符不能是数字。
     </li>
     <li id="ue3710e5d">
      标识符必须不包含任何嵌入的空格或符号，比如 ? - +! # % ^ &amp; * ( ) [ ] { } . ; : " ' / \。
     </li>
     <li id="u4d7087ab">
      标识符不能是 C# 关键字。除非它们有一个 @ 前缀。 例如，@if 是有效的标识符，但 if 不是，因为 if 是关键字。
     </li>
     <li id="u68c85e0b">
      标识符必须区分大小写。大写字母和小写字母被认为是不同的字母。
     </li>
     <li id="u4b36dc28">
      不能与C#的类库名称相同。
     </li>
    </ul>

    <h2 id="QkJaW" name="QkJaW">
     十一、顶级语句（Top-Level Statements）
    </h2>
    <p id="u07ce2c7a">
     在 C# 9.0 版本中，引入了顶级语句（Top-Level Statements）的概念，这是一种新的编程范式，允许开发者在文
    </p>
    <p id="u4a5fb654">
     件的顶层直接编写语句，而不需要将它们封装在方法或类中。
    </p>
    <p id="u9180a81a">
     <strong>
      特点：
     </strong>
    </p>
    <ul>
     <li id="u0fb00aad">
      <strong>
       无需类或方法
      </strong>
      ：顶级语句允许你直接在文件的顶层编写代码，无需定义类或方法。
     </li>
     <li id="uad543b56">
      <strong>
       文件作为入口点
      </strong>
      ：包含顶级语句的文件被视为程序的入口点，类似于 C# 之前的 Main 方法。
     </li>
     <li id="u5b365553">
      <strong>
       自动 Main 方法
      </strong>
      ：编译器会自动生成一个 Main 方法，并将顶级语句作为 Main 方法的主体。
     </li>
     <li id="ub2065dfb">
      <strong>
       支持局部函数
      </strong>
      ：尽管不需要定义类，但顶级语句的文件中仍然可以定义局部函数。
     </li>
     <li id="u177197d5">
      <strong>
       更好的可读性
      </strong>
      ：对于简单的脚本或工具，顶级语句提供了更好的可读性和简洁性。
     </li>
     <li id="u82102ac8">
      <strong>
       适用于小型项目
      </strong>
      ：顶级语句非常适合小型项目或脚本，可以快速编写和运行代码。
     </li>
     <li id="udb439714">
      <strong>
       与现有代码兼容
      </strong>
      ：顶级语句可以与现有的 C# 代码库一起使用，不会影响现有代码。
     </li>
    </ul>
    <p id="ud4178347">
     <strong>
      传统 C# 代码
     </strong>
     - 在使用顶级语句之前，你必须像这样编写一个 C# 程序：
    </p>
    <h3 id="F2555" name="F2555">
     实例一
    </h3>
    <pre id="r7pLo"><code>using System;

namespace MyApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}</code></pre>
    <p id="u92346dab">
     <strong>
      使用顶级语句的 C# 代码
     </strong>
     - 使用顶级语句，可以简化为：
    </p>
    <h3 id="SpwkL" name="SpwkL">
     实例二
    </h3>
    <pre id="hCMkh"><code>using System;

Console.WriteLine("Hello, World!");</code></pre>
    <p id="u121eab61">
     顶级语句支持所有常见的 C# 语法，包括声明变量、定义方法、处理异常等。
    </p>
    <h3 id="EHgrS" name="EHgrS">
     实例三
    </h3>
    <pre id="A7Xnw"><code>using System;
using System.Linq;

// 顶级语句中的变量声明
int number = 42;
string message = "The answer to life, the universe, and everything is";

// 输出变量
Console.WriteLine($"{message} {number}.");

// 定义和调用方法
int Add(int a, int b) =&gt; a + b;
Console.WriteLine($"Sum of 1 and 2 is {Add(1, 2)}.");

// 使用 LINQ
var numbers = new[] { 1, 2, 3, 4, 5 };
var evens = numbers.Where(n =&gt; n % 2 == 0).ToArray();
Console.WriteLine("Even numbers: " + string.Join(", ", evens));

// 异常处理
try
{
    int zero = 0;
    int result = number / zero;
}
catch (DivideByZeroException ex)
{
    Console.WriteLine("Error: " + ex.Message);
}</code></pre>
    <h3 id="uTFt0" name="uTFt0">
     注意事项
    </h3>
    <ul>
     <li id="ucd44211c">
      <strong>
       文件限制：
      </strong>
      顶级语句只能在一个源文件中使用。如果在一个项目中有多个使用顶级语句的文件，会导致编译错误。
     </li>
     <li id="u14e8824e">
      <strong>
       程序入口：
      </strong>
      如果使用顶级语句，则该文件会隐式地包含 Main 方法，并且该文件将成为程序的入口点。
     </li>
     <li id="u53d91abb">
      <strong>
       作用域限制：
      </strong>
      顶级语句中的代码共享一个全局作用域，这意味着可以在顶级语句中定义的变量和方法可以在整个文件中访问。
     </li>
    </ul>
    <p id="u58fa184d">
     顶级语句在简化代码结构、降低学习难度和加快开发速度方面具有显著优势，特别适合于编写简单程序和脚本。
    </p>
    <h2 id="Z2164" name="Z2164">
     十二、Java与C#的语法区别
    </h2>
    <p id="u4554b183">
     由Java转入C#一段时间了，总结下个人认为的Java同C#语法之间的不同之处，有不同意见之处还望各位海涵，刚
    </p>
    <p id="u3b3cc83c">
     学Java时觉得语法同C#大致是相同的(应该说C#同Java大致相同
    </p>
    <p id="ue050274a">
     由Java转入C#一段时间了，总结下个人认为的C#同Java语法之间的不同之处，有不同意见之处还望各位海涵
    </p>
    <p id="u140fc69a">
     刚学C#时觉得Java语法同大致是相同的(应该说C#同Java大致相同，毕竟人家微软的C#是有意模仿Java的语法习
    </p>
    <p id="u8a12f9ed">
     惯的)
    </p>
    <p id="u1221c3a3">
     比尔.盖茨曾经说过：“Java是最卓越的程序设计语言”
    </p>
    <p id="u2fb1c12b">
     言归正传，下面探讨Java同C#的语法不同之处。。。
    </p>
    <h3 id="I8sSY" name="I8sSY">
     1. 命名空间与包
    </h3>
    <p id="uc913fde2">
     C#为了把实现相似功能的类组织在一起，引入了命名空间的概念（namespace）
    </p>
    <p id="u38544f58">
     Java中与此对应的东西叫做包（package）
    </p>
    <h3 id="mLH27" name="mLH27">
     2. 类的访问控制方面的不同
    </h3>
    <p id="uf1f4036e">
     C#只有两种：public和默认(同internal)
    </p>
    <p id="ud3bd3007">
     public能被所有的类（同项目中和不同项目中）访问
    </p>
    <p id="u29475d60">
     internal（在class关键字前不加控制符时默认为internal），表明类只能在同一项目中访问
    </p>
    <p id="uc7f58b57">
    </p>
    <p id="ubcc6a2b0">
     Java也只有两种：public和默认
    </p>
    <p id="u922d2f17">
     public能被所有的类访问
    </p>
    <p id="ufcb6b471">
     默认（在class关键字前不加控制符时）只能被同一包中的所有类访问
    </p>
    <h3 id="DmY5P" name="DmY5P">
     3. 类成员的访问控制
    </h3>
    <p id="uc69321bf">
     C#中有四种：public,protected,private（默认）,internal（注意这里internal和默认是不同的）
    </p>
    <ul>
     <li id="u3ae35234">
      public能被所有的类访问
     </li>
     <li id="ue6ec8764">
      protected只能被子类访问
     </li>
     <li id="u85ca19d1">
      private（也即不写任何控制符时的默认情况下）只能类内部访问
     </li>
     <li id="ua07ad651">
      internal能被同一项目中的类访问
     </li>
    </ul>
    <p id="u326040ec">
     Java也有四种：public,protected,private和默认
    </p>
    <ul>
     <li id="u6d6ba04e">
      public能被所有的类访问
     </li>
     <li id="u4277b4d2">
      protected既能被同一个包中的其他类访问，也可以被不同包中的子类访问
     </li>
     <li id="uf8a170d7">
      private只能在类内部使用
     </li>
     <li id="u634a7622">
      默认可以被这个包中的其他类访问，如果一个子类与父类位于不同的包中，子类也不能访问父类中的默认访问控制成员
     </li>
    </ul>
    <h3 id="KMa7X" name="KMa7X">
     4. C#中类的继承用通过冒号:实现，在Java中用extends
    </h3>
    <p id="u9dd94f20">
     C#中实现接口通过冒号:实现，在Java中用implements
    </p>
    <p id="uf62d638d">
     C#中密封类用sealed实现，在Java中用final
    </p>
    <p id="ud1a840f2">
     C#中常数用const实现，在Java中用final
    </p>
    <p id="u894df2f1">
     C#中属性用set,get代码块实现，在Java中一般用类似于C#中的字段代表属性，或者用setter,getter构造器实现
    </p>
    <p id="u68209c88">
     C#中有部分类（partial）的概念，Java中没有
    </p>
    <p id="u932790dd">
     C#中有readonly修饰属性只读，Java中没有
    </p>
    <p id="u2796e63a">
     C#中有virtual和override修饰虚方法和重写方法，Java中没有，Java中默认父类中的方法都是virtual的
    </p>
    <p id="u32af2e91">
     Java中有static{},synchroized{}代码块的概念，C#中没有
    </p>
    <p id="u30effcc0">
     Java中有标签（如labelA:）的概念，C#中没有
    </p>
    <p id="u87343129">
     C#中子类调用父类的方法用base.method()，Java中用super.method()
    </p>
    <p id="ub6099015">
     C#中用is来判断一个实例是否是某一个类的，Java中用instanceof
    </p>
    <p id="u4506c1b6">
     C#中用foreach(int i in array)来对数组中每一个元素的遍历，Java中用for(int i : array)
    </p>
    <h3 id="MBBfW" name="MBBfW">
     5. 基本结构对比
    </h3>
    <p id="uabf83072">
     在C#中，类和接口的概念与Java中的类似，但有一些细微的差别。
    </p>
    <p id="uff8b6b28">
     下面是Java中的一些基本概念与C#中对应概念的对比：
    </p>
    <ol>
     <li id="uff0c68a8">
      <strong>
       类 (Class)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u1ceba7f3">
      <strong>
       Java
      </strong>
      : 用于创建对象的模板。可以包含字段、方法、构造函数等。
     </li>
     <li id="u8ee0ef2f">
      <strong>
       C#
      </strong>
      : 同样用于创建对象的模板。可以包含字段、属性、方法、构造函数等。
     </li>
    </ul>
    <ol>
     <li id="uac668fcf">
      <strong>
       抽象类 (Abstract Class)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u98a9a646">
      <strong>
       Java
      </strong>
      : 不能直接实例化，用于提供一种模板，让其他类继承并实现其方法。
     </li>
     <li id="u0d73eae2">
      <strong>
       C#
      </strong>
      : 不能直接实例化，用于提供一种模板，让其他类继承并实现其方法。
     </li>
    </ul>
    <ol>
     <li id="u4cc82997">
      <strong>
       接口 (Interface)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u618479d6">
      <strong>
       Java
      </strong>
      : 定义了一个类必须实现的方法，但没有提供这些方法的实现。
     </li>
     <li id="u259e71bb">
      <strong>
       C#
      </strong>
      : 同样定义了一个类必须实现的方法，但没有提供这些方法的实现。C#中的接口还可以包含属性、方法、事件和索引器（但不包含字段）。
     </li>
    </ul>
    <ol>
     <li id="u3ac8e772">
      <strong>
       枚举 (Enum)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u3eee811a">
      <strong>
       Java
      </strong>
      : 用于定义变量，该变量可以是预定义的一组常量之一。
     </li>
     <li id="uac89af81">
      <strong>
       C#
      </strong>
      : 用于定义变量，该变量可以是预定义的一组常量之一。
     </li>
    </ul>
    <ol>
     <li id="u1e471c70">
      <strong>
       注解 (Annotation)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u79223134">
      <strong>
       Java
      </strong>
      : 用于为代码提供元数据信息，例如标记代码元素（如类、方法、字段等）以供编译器或运行时使用。
     </li>
     <li id="u34b8a175">
      <strong>
       C#
      </strong>
      : 同样用于为代码提供元数据信息，例如特性（Attributes）在C#中用来提供类似的功能。
     </li>
    </ul>
    <p id="ueee5be32">
     例如，[Serializable] 在C#中用来标记类是可序列化的。
    </p>
    <ol>
     <li id="u858b1907">
      <strong>
       数组 (Array)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u1030c91d">
      <strong>
       Java
      </strong>
      : 用于存储固定大小的同类型元素序列。
     </li>
     <li id="u89498e4b">
      <strong>
       C#
      </strong>
      : 同样用于存储固定大小的同类型元素序列。
     </li>
    </ul>
    <ol>
     <li id="ud3eeee4b">
      <strong>
       泛型 (Generics)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="uf2242923">
      虽然不是直接的对应概念，但Java和C#都支持泛型编程，允许类型安全地操作数据结构而不必使用类型强制转换。
     </li>
    </ul>
    <ol>
     <li id="u98201dcf">
      <strong>
       委托 (Delegate) 和 事件 (Event)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u15eedec1">
      这些是C#特有的概念，类似于Java中的接口，但专门用于定义回调方法，特别是在事件驱动编程中。在Java中，通常使用接口和匿名内部类来实现类似的功能。
     </li>
    </ul>
    <ol>
     <li id="ue9186fa0">
      <strong>
       属性 (Property)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u9f6b189a">
      在C#中，属性是类的成员，它封装了类的字段，并提供了一种访问这些字段的机制。这在C#中非常常见，而在Java中通常使用私有字段和公共的getter/setter方法来实现类似的功能。
     </li>
    </ul>
    <ol>
     <li id="u8927eada">
      <strong>
       结构体 (Struct)
      </strong>
      :
     </li>
    </ol>
    <ul>
     <li id="u9ee5a90d">
      在C#中，struct 关键字用于定义一个简单的数据类型，类似于Java中的类，但它是值类型而不是引用类型。这在C#中是存在的，而在Java中没有直接对应的结构体概念（Java只有类）。
     </li>
    </ul>
    <p id="udcbee205">
     通过上述对比，可以看出Java和C#在面向对象编程的许多方面有着相似之处，尽管在某些具体语法和概念上存在
    </p>
    <p id="u53b37cf9">
     差异。了解这些差异有助于在两种语言之间进行代码迁移或开发时更加得心应手。
    </p>
