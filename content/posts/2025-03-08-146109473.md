    <h1 class="title-article" id="articleContentId">
     ORACLE 执行查询语句慢（不走对应索引）
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <h2>
     <br/>
     1. 索引未被创建或未正确创建
    </h2>
    <p>
     <br/>
     确保为查询中涉及的列创建了索引。例如，如果你经常需要按column_name列进行查询，确保已经为该列创建了索引,索引创建语句
    </p>
    <blockquote>
     <p>
      CREATE INDEX idx_column_name ON table_name(column_name);
     </p>
    </blockquote>
    <h2>
     2、索引不可用
    </h2>
    <p>
     <br/>
     原因:索引可能被标记为不可用（UNUSABLE）通常是由于索引重建失败或数据导入操作导致的。
     <br/>
     解决方法:检查索引状态并重建索引
     <br/>
     检查索引状态
    </p>
    <blockquote>
     <p>
      SELECT INDEX_NAME, STATUS FROM USER_INDEXES WHERE TABLE_NAME = 'TABLE_NAME';
     </p>
    </blockquote>
    <p>
     如果索引不可用，重建索引
    </p>
    <blockquote>
     <p>
      ALTER INDEX INDEX_NAME REBUILD;
     </p>
    </blockquote>
    <h2>
     3、
     <strong>
      统计信息不准确
     </strong>
    </h2>
    <p>
     <strong>
      原因
     </strong>
     :
     <br/>
     Oracle 优化器依赖统计信息来决定执行计划。如果表的统计信息不准确或过时，优化器可能会错误地选择不使用索引。
    </p>
    <p>
     所以创建索引并且执行语句没有问题，则 可以使用DBMS_STATS包来收集最新的统计信息：
     <br/>
     解决方法:
    </p>
    <blockquote>
     <p>
      EXEC DBMS_STATS.GATHER_TABLE_STATS('SCHEMA_NAME', 'TABLE_NAME');
      <br/>
      EXEC DBMS_STATS.GATHER_TABLE_STATS('SYMDR', 'LAB_REPORT_INFO');
     </p>
    </blockquote>
    <h2>
     4、如果收集最新的统计信息执行报错
    </h2>
    <p>
     错误信息：
    </p>
    <p>
     ORA-20005: object statistics are locked (stattype = ALL) 则看是否有死锁
    </p>
    <blockquote>
     <p>
      SELECT s.sid, s.serial#, l.object_id, o.object_name, l.session_id blocking_sid
      <br/>
      FROM v$locked_object l
      <br/>
      JOIN dba_objects o ON l.object_id = o.object_id
      <br/>
      JOIN v$session s ON l.session_id = s.sid
      <br/>
      WHERE o.object_type = 'LAB_REPORT_INFO';
     </p>
    </blockquote>
    <h2>
     5、终止会话
    </h2>
    <p>
     如果找到死锁可以使用如下命令终止会话
    </p>
    <blockquote>
     <p>
      ALTER SYSTEM KILL SESSION 'sid,serial#';
     </p>
    </blockquote>
    <h2>
     6、强制更新统计信息
    </h2>
    <p>
     如果确定没有其他会话正在使用统计信息，或者已经终止了阻塞会话，可以尝试强制更新统计信息：
    </p>
    <blockquote>
     <p>
      <br/>
      BEGIN
      <br/>
      DBMS_STATS.GATHER_TABLE_STATS(
      <br/>
      ownname       =&gt; 'OWNER',
      <br/>
      tabname       =&gt; 'TABLE_NAME',
      <br/>
      estimate_percent =&gt; DBMS_STATS.AUTO_SAMPLE_SIZE,
      <br/>
      method_opt    =&gt; 'FOR ALL COLUMNS SIZE AUTO',
      <br/>
      cascade       =&gt; TRUE,
      <br/>
      force         =&gt; TRUE);
      <br/>
      END;
      <br/>
      /
      <br/>
      BEGIN
      <br/>
      DBMS_STATS.GATHER_TABLE_STATS(
      <br/>
      ownname       =&gt; 'SYMDR',
      <br/>
      tabname       =&gt; 'LAB_REPORT_INFO',
      <br/>
      estimate_percent =&gt; DBMS_STATS.AUTO_SAMPLE_SIZE,
      <br/>
      method_opt    =&gt; 'FOR ALL COLUMNS SIZE AUTO',
      <br/>
      cascade       =&gt; TRUE,
      <br/>
      force         =&gt; TRUE);
      <br/>
      END;
      <br/>
      /
     </p>
    </blockquote>
    <h2>
     7、 检查执行计划
    </h2>
    <p>
     使用 EXPLAIN PLAN 或 DBMS_XPLAN 查看查询的执行计划，了解优化器为何选择不使用索引：
    </p>
    <blockquote>
     <p>
      EXPLAIN PLAN FOR SELECT * FROM TABLE_NAME WHERE COLUMN_NAME = 'VALUE';
      <br/>
      SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
     </p>
     <p>
     </p>
     <p>
      EXPLAIN PLAN FOR
      <br/>
      select * from lab_report_info where org_code='XX'   and request_no='XX'      and local_id='XX' ;
      <br/>
      SELECT * FROM table(DBMS_XPLAN.DISPLAY());
     </p>
    </blockquote>
