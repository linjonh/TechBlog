    <h2>
     <a id="Qtrelease_15">
     </a>
     Qt生成release版
    </h2>
    <ol>
     <li>
      在 Qt Creator 左下角改一个设置：
      <br/>
      <img alt="release" src="https://i-blog.csdnimg.cn/blog_migrate/a05c2aeaa3a7b0936f69ab151f411cc8.png"/>
     </li>
     <li>
      点绿色三角形编译运行过之后，在和工程文件同目录下，有个装着编译出来的可执行文件的文件夹，形如：
      <br/>
      <img alt="release_diretory" src="https://i-blog.csdnimg.cn/blog_migrate/70ecfb1ba800630bca04750b346212fd.png">
       <br/>
       里面有两个子文件夹，一个叫
       <code>
        debug
       </code>
       ，另一个叫
       <code>
        release
       </code>
       ，因为先前选了release，故可执行文件在
       <code>
        release
       </code>
       那个子目录下，但
       <strong>
        里面那个还不想要的release版的
       </strong>
      </img>
     </li>
     <li>
      将
      <code>
       release
      </code>
      里那个
      <code>
       .exe
      </code>
      文件复制到一个
      <strong>
       空文件夹
      </strong>
      下，如在桌面新建一个叫
      <code>
       TomChat
      </code>
      的新文件夹，复制到里面
     </li>
     <li>
      Qt 有个自带的终端（Terminal / Console），应该在开始菜单里可以找到
      <br/>
      <img alt="Qt_Term" src="https://i-blog.csdnimg.cn/blog_migrate/06086d66bc0f252dfe421fa683668d22.png"/>
     </li>
     <li>
      在终端里更改工作目录到刚刚那个新建的文件夹里，如执行
      <br/>
      <code>
       cd /d C:\Users\Tom\Desktop\TomChat
      </code>
      <br/>
      然后用 Qt 自带的工具
      <code>
       windeployqt
      </code>
      来生成release版，执行
      <br/>
      <code>
       windeployqt iTom.exe
      </code>
      （
      <em>
       iTom.exe
      </em>
      是那个可执行文件的名字）
      <br/>
      命令执行完成后，就成功生成了release版，类似于免安装的软件那样，把文件夹拷走就可以用。
      <img alt="windeployqt" src="https://i-blog.csdnimg.cn/blog_migrate/7568ae3575b3f9ba187e9bba8508baa2.png">
       <br/>
       用
       <em>
        windeployqt
       </em>
       时可能会弹框说缺少若干个什么什么
       <code>
        .dll
       </code>
       文件的，但去看Qt的目录发现这些
       <code>
        .dll
       </code>
       文件其实是有的，我当时是通过
       <strong>
        环境变量path中把Qt那条上移到第一条
       </strong>
       解决的（之前为了给vim配gcc，添加了codeblocks里mingw的目录进去，不知道是不是跟这有关）
       <br/>
       <img alt="qt_path" src="https://i-blog.csdnimg.cn/blog_migrate/ba5262828632e103e1323da2e144abb4.png">
        <br/>
        #程序架构
       </img>
      </img>
     </li>
     <li>
      登录界面（LoginDialog）
     </li>
    </ol>
    <ul>
     <li>
      最先出现，用于获取用户本次登录所起的用户名，并传递给主界面；
     </li>
    </ul>
    <ol start="2">
     <li>
      主界面（iTom）
     </li>
    </ol>
    <ul>
     <li>
      主界面是一个叫
      <code>
       iTom
      </code>
      的类，主要包含一个
      <em>
       好友列表（FriendList）
      </em>
      、一个退出按钮；
     </li>
     <li>
      主界面拥有QUdpSocket、QTcpServer、QTcpSocket，在后台负责广播上线消息（登录时）、广播下线消息（退出时）、回复上线广播（收到别人上线广播后）、统一接收聊天信息并转传给相应的
      <em>
       好友名片（FriendButton）
      </em>
      、接收文件；
     </li>
    </ul>
    <ol start="3">
     <li>
      好友列表（FriendList）
     </li>
    </ol>
    <ul>
     <li>
      朋友列表就是盗版QQ的朋友列表（但没有分组），用于存放
      <em>
       好友名片（FriendButton）
      </em>
      ，主要是展示功能；
     </li>
     <li>
      列表里默认有个
      <code>
       Broadcast
      </code>
      的好友名片，用于广播信息；
     </li>
    </ul>
    <ol start="4">
     <li>
      好友名片（FriendButton）
     </li>
    </ol>
    <ul>
     <li>
      好友名片就是像QQ好友列表里展示好友的那种东西，之所以叫
      <em>
       button
      </em>
      是因为继承自
      <code>
       QPushButton
      </code>
      ，点击呼出与相应好友的
      <em>
       聊天窗口（ChatWindow）
      </em>
      ；
     </li>
    </ul>
    <ol start="5">
     <li>
      聊天窗口（ChatWindow）
     </li>
    </ol>
    <ul>
     <li>
      包含显示聊天信息的显示框、信息编辑框、发送信息按钮、发送文件按钮；
     </li>
     <li>
      聊天窗口
      <strong>
       不负责
      </strong>
      接收信息，而是来自主界面的转传，但负责发信息、选择并发送文件；
     </li>
     <li>
      广播的窗口
      <strong>
       不能
      </strong>
      发送文件，所以它的文件发送按钮
      <strong>
       被禁用
      </strong>
      ；
     </li>
    </ul>
