    <h2>
     一.ip
    </h2>
    <h3>
     1.ip的定义
    </h3>
    <p>
     IP是“Internet Protocol”的缩写，即“互联网协议”。它是用于计算机网络通信的基础协议之一，属于
     <strong>
      TCP/IP
     </strong>
     协议族中的
     <strong>
      网络层协议
     </strong>
     。IP协议的主要功能是负责将数据包从源主机传输到目标主机，并确保数据能够在复杂的网络环境中正确地路由和转发。
    </p>
    <h3>
     2.ip地址
    </h3>
    <p>
     IP地址是IP协议的核心组成部分，它是一个用于唯一标识网络设备（如计算机、服务器、路由器等）的地址。IP地址分为两种主要版本：
    </p>
    <h4>
     （1）IPv4（Internet Protocol version 4）
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        格式
       </strong>
       ：IPv4地址是一个32位的二进制数，通常以点分十进制格式表示，例如：
       <code>
        192.168.1.1
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        地址范围
       </strong>
       ：IPv4地址的范围是从
       <code>
        0.0.0.0
       </code>
       到
       <code>
        255.255.255.255
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        分类
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          A类地址
         </strong>
         ：范围是
         <code>
          1.0.0.0
         </code>
         到
         <code>
          126.0.0.0
         </code>
         ，默认子网掩码为
         <code>
          255.0.0.0
         </code>
         。
        </p>
       </li>
       <li>
        <p>
         <strong>
          B类地址
         </strong>
         ：范围是
         <code>
          128.0.0.0
         </code>
         到
         <code>
          191.255.0.0
         </code>
         ，默认子网掩码为
         <code>
          255.255.0.0
         </code>
         。
        </p>
       </li>
       <li>
        <p>
         <strong>
          C类地址
         </strong>
         ：范围是
         <code>
          192.0.0.0
         </code>
         到
         <code>
          223.255.255.0
         </code>
         ，默认子网掩码为
         <code>
          255.255.255.0
         </code>
         。
        </p>
       </li>
       <li>
        <p>
         <strong>
          D类地址
         </strong>
         ：范围是
         <code>
          224.0.0.0
         </code>
         到
         <code>
          239.255.255.255
         </code>
         ，用于多播（组播）。
        </p>
       </li>
       <li>
        <p>
         <strong>
          E类地址
         </strong>
         ：范围是
         <code>
          240.0.0.0
         </code>
         到
         <code>
          255.255.255.255
         </code>
         ，目前保留未使用。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：IPv4地址数量有限（约43亿个），随着互联网的快速发展，IPv4地址逐渐耗尽。
      </p>
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      ip4里面的子网掩码是什么？
     </strong>
    </p>
    <p>
     子网掩码将ip地址划分为网络部分和本地部分，帮网络设备确定ip地址所属的子网。子网通过其二进制形式中的“1”和“0”来区分这两个部分。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        网络部分
       </strong>
       ：子网掩码中为“1”的部分表示IP地址的网络部分，用于标识网络。
      </p>
     </li>
     <li>
      <p>
       <strong>
        主机部分
       </strong>
       ：子网掩码中为“0”的部分表示IP地址的主机部分，用于标识网络中的设备。
      </p>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       子网掩码的分类
      </strong>
     </p>
     <p>
      根据IPv4地址的分类（A、B、C类），默认子网掩码如下：
     </p>
     <ul>
      <li>
       <p>
        <strong>
         A类地址
        </strong>
        ：默认子网掩码为
        <code>
         255.0.0.0
        </code>
        ，表示前8位为网络部分，后24位为主机部分。
       </p>
      </li>
      <li>
       <p>
        <strong>
         B类地址
        </strong>
        ：默认子网掩码为
        <code>
         255.255.0.0
        </code>
        ，表示前16位为网络部分，后16位为主机部分。
       </p>
      </li>
      <li>
       <p>
        <strong>
         C类地址
        </strong>
        ：默认子网掩码为
        <code>
         255.255.255.0
        </code>
        ，表示前24位为网络部分，后8位为主机部分。
       </p>
      </li>
     </ul>
    </blockquote>
    <h4>
     （2）
     <strong>
      IPv6（Internet Protocol version 6）
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        格式
       </strong>
       ：IPv6地址是一个128位的二进制数，通常以冒号分隔的十六进制格式表示，例如：
       <code>
        2001:0db8:85a3:0000:0000:8a2e:0370:7334
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        特点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         提供了几乎无限的地址空间（约3.4×10³⁸个地址），解决了IPv4地址不足的问题。
        </p>
       </li>
       <li>
        <p>
         支持更高效的路由和自动配置功能。
        </p>
       </li>
       <li>
        <p>
         内置了更好的安全性（如IPsec）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        推广现状
       </strong>
       ：尽管IPv6已经存在多年，但由于兼容性和成本问题，目前全球范围内的IPv6普及率仍在逐步提高。
      </p>
     </li>
    </ul>
    <h4>
     （3）特殊的地址 127.0.0.01
    </h4>
    <p>
     <code>
      127.0.0.1
     </code>
     是IPv4协议中保留的本地回环地址（Loopback Address）。它被用于标识本机上的网络服务，主要用于测试和调试网络程序。
    </p>
    <hr/>
    <p>
     <strong>
      那么为什么我一定要用这个ip地址呢？用别的地址不行吗？
     </strong>
    </p>
    <p>
     使用
     <code>
      127.0.0.1
     </code>
     有其特定的优势和必要性。
     <code>
      127.0.0.1
     </code>
     是一个特殊的回环地址，用于本地通信，可以避免网络冲突和配置问题，确保测试环境的一致性和隔离性。因此，推荐使用
     <code>
      127.0.0.1
     </code>
     进行本地调试和测试。
    </p>
    <h2>
     二. 端口
    </h2>
    <h3>
     1.端口的定义
    </h3>
    <p>
     端口是一个16位的数字，范围从0到65535。它用于
     <strong>
      标识应用程序
     </strong>
     或服务在某个IP地址上的具体位置。每个IP地址可以有多个端口，每个端口可以运行不同的应用程序或服务。
    </p>
    <h3>
     2.端口的分类
    </h3>
    <p>
     端口可以分为以下几类：
    </p>
    <p>
     <strong>
      （1）系统端口（0-1023）
     </strong>
    </p>
    <p>
     这些端口通常由系统或操作系统保留，用于运行系统服务和关键应用程序。例如：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        端口80
       </strong>
       ：用于HTTP协议，提供网页访问服务。
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口443
       </strong>
       ：用于HTTPS协议，提供安全的网页访问服务。
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口21
       </strong>
       ：用于FTP协议，提供文件传输服务。
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口25
       </strong>
       ：用于SMTP协议，提供电子邮件服务。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      （2） 用户端口（1024-49151）
     </strong>
    </p>
    <p>
     这些端口可以由用户自定义或由应用程序使用。许多应用程序会使用这些端口进行网络通信。例如：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        端口3306
       </strong>
       ：用于MySQL数据库服务。
      </p>
     </li>
     <li>
      <p>
       <strong>
        端口5432
       </strong>
       ：用于PostgreSQL数据库服务。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      （3）动态端口（49152-65535）
     </strong>
    </p>
    <p>
     这些端口通常由操作系统动态分配，用于临时通信。例如，当你使用浏览器访问一个网页时，浏览器会从这些端口中选择一个临时端口作为通信的源端口。
    </p>
    <h2>
     三.通讯接口
    </h2>
    <h3>
     1.通讯接口的定义
    </h3>
    <ul>
     <li>
      <p>
       通讯接口是一个实现通讯协议的
       <strong>
        接口
       </strong>
       ，使得不同设备或应用程序能够按照
       <strong>
        协议
       </strong>
       的规定进行通信。
      </p>
     </li>
     <li>
      <p>
       通讯接口可以是
       <strong>
        软件接口
       </strong>
       （如API、库）或
       <strong>
        硬件接口
       </strong>
       （如USB、RS-232）。
      </p>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       软件接口和硬件接口之间有什么关系呢？
      </strong>
     </p>
     <p>
      <a href="https://blog.csdn.net/m0_59247951/article/details/146093028?sharetype=blogdetail&amp;sharerId=146093028&amp;sharerefer=PC&amp;sharesource=m0_59247951&amp;spm=1011.2480.3001.8118" title="为什么会有软件接口和硬件接口？-CSDN博客">
       为什么会有软件接口和硬件接口？-CSDN博客
      </a>
     </p>
    </blockquote>
    <h3>
     2.通讯接口分类
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        软件接口
       </strong>
       ：如Socket、Snap7、Modbus-TCP等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        硬件接口
       </strong>
       ：如USB、RS-232、RS-485等。
      </p>
     </li>
    </ul>
    <h3>
     3.接口Snap7和Socket
    </h3>
    <h4>
     （1）
     <strong>
      Snap7
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        定义
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         Snap7 是一个用于与西门子 S7 系列 PLC（可编程逻辑控制器）进行通信的开源库。它提供了与 PLC 设备进行数据交换的接口。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        接口类型
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          PLC 接口
         </strong>
         ：Snap7 主要用于与 PLC 设备进行通信，因此可以被视为一种 PLC 接口。
        </p>
       </li>
       <li>
        <p>
         <strong>
          软件接口
         </strong>
         ：Snap7 提供了软件接口，允许开发者通过编程与 PLC 设备进行通信。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        作用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         提供了与西门子 PLC 设备进行高效、可靠通信的接口，适用于工业自动化领域。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <strong>
      （2）Socket
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        定义
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         Socket 是计算机网络中的一种通用通信接口，可以用于实现网络通信。它提供了与网络中其他设备或应用程序进行数据交换的接口。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        接口类型
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          计算机软件接口
         </strong>
         ：Socket 是计算机网络中的一种通用接口，适用于各种网络应用。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        作用
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         提供了网络通信的接口，使得不同设备或应用程序之间可以进行数据传输，适用于多种网络通信场景。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     四.通讯协议
    </h2>
    <h3>
     1.通讯协议的定义
    </h3>
    <ul>
     <li>
      <p>
       通讯协议是一组
       <strong>
        规则和标准
       </strong>
       ，用于定义两个或多个
       <strong>
        通信实体
       </strong>
       （如设备、应用程序）之间如何进行
       <strong>
        信息交换
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       协议规定了通信的内容、格式、顺序以及错误处理等细节。
      </p>
     </li>
    </ul>
    <p>
     <img alt="" height="446" src="https://i-blog.csdnimg.cn/direct/bd850426fce84afe81dc8f517a148b11.png" width="1277"/>
    </p>
    <h3>
     2.通讯协议分类
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        网络协议
       </strong>
       ：如TCP/IP、UDP、HTTP、FTP等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        工业协议
       </strong>
       ：如Modbus、Profibus、Profinet、S7等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        应用层协议
       </strong>
       ：如HTTP、HTTPS、SMTP、POP3等。
      </p>
     </li>
    </ul>
    <blockquote>
     <p>
      <strong>
       可是为什么会有这么多通讯协议的分类呢？
      </strong>
     </p>
     <p>
      因为通讯协议的多样性是由于不同的应用场景、历史背景和技术发展导致的。
     </p>
     <p>
      <a href="https://blog.csdn.net/m0_59247951/article/details/146093954?sharetype=blogdetail&amp;sharerId=146093954&amp;sharerefer=PC&amp;sharesource=m0_59247951&amp;spm=1011.2480.3001.8118" title="通讯协议的分类及原因-CSDN博客">
       通讯协议的分类及原因-CSDN博客
      </a>
     </p>
    </blockquote>
    <h2>
     五.UDP和TCP
    </h2>
    <ul>
     <li>
      <p>
       <strong>
        UDP
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          定义
         </strong>
         ：UDP是一种无连接的传输层协议，它不提供可靠的数据传输服务，也不保证数据能够成功到达接收方。它在发送数据前不需要与接收方建立连接，发送数据后也不会等待接收方的确认。
        </p>
       </li>
       <li>
        <p>
         <strong>
          特点
         </strong>
         ：UDP协议简单高效，适用于实时性要求较高的应用场景，如视频会议、在线游戏等。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li>
      <p>
       <strong>
        TCP
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          定义
         </strong>
         ：TCP是一种面向连接的传输层协议，它在发送数据前需要与接收方建立可靠的连接。TCP通过三次握手（SYN、ACK、SYN-ACK）来建立连接，通过四次挥手（FIN、ACK、FIN-ACK）来断开连接。
        </p>
       </li>
       <li>
        <p>
         <strong>
          特点
         </strong>
         ：TCP提供可靠的数据传输服务，能够保证数据的按序到达和完整性。它适用于对数据可靠性要求较高的场景，如文件传输、电子邮件等。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     协议的比较与区别
    </h4>

    <h4>
     协议的报文格式和字段含义
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        UDP 报文格式
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          首部
         </strong>
         ：8字节，包含以下字段：
        </p>
        <ul>
         <li>
          <p>
           <strong>
            源端口
           </strong>
           ：2字节，发送方的端口号。
          </p>
         </li>
         <li>
          <p>
           <strong>
            目的端口
           </strong>
           ：2字节，接收方的端口号。
          </p>
         </li>
         <li>
          <p>
           <strong>
            长度
           </strong>
           ：2字节，UDP用户数据报的总长度（包括首部和数据）。
          </p>
         </li>
         <li>
          <p>
           <strong>
            校验和
           </strong>
           ：2字节，用于检测UDP用户数据报在传输过程中是否有错误。
          </p>
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          数据区
         </strong>
         ：存放实际要传输的数据。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        TCP 报文格式
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          首部
         </strong>
         ：通常20-60字节，包含以下字段：
        </p>
        <ul>
         <li>
          <p>
           <strong>
            源端口
           </strong>
           ：2字节，发送方的端口号。
          </p>
         </li>
         <li>
          <p>
           <strong>
            目的端口
           </strong>
           ：2字节，接收方的端口号。
          </p>
         </li>
         <li>
          <p>
           <strong>
            序号
           </strong>
           ：4字节，用于标识TCP报文段的顺序。
          </p>
         </li>
         <li>
          <p>
           <strong>
            确认号
           </strong>
           ：4字节，用于确认接收到的报文段。
          </p>
         </li>
         <li>
          <p>
           <strong>
            首部长度
           </strong>
           ：1字节，标识TCP首部的长度。
          </p>
         </li>
         <li>
          <p>
           <strong>
            标志位
           </strong>
           ：6位，用于控制TCP连接的状态（如SYN、ACK、FIN等）。
          </p>
         </li>
         <li>
          <p>
           <strong>
            窗口大小
           </strong>
           ：2字节，用于流量控制。
          </p>
         </li>
         <li>
          <p>
           <strong>
            校验和
           </strong>
           ：2字节，用于检测TCP报文段在传输过程中是否有错误。
          </p>
         </li>
         <li>
          <p>
           <strong>
            紧急指针
           </strong>
           ：2字节，用于紧急数据的传输。
          </p>
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          数据区
         </strong>
         ：存放实际要传输的数据。
        </p>
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     协议的使用方式和编程示例
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        UDP 编程示例
       </strong>
      </p>
      <p>
       Python复制
      </p>
      <pre><code class="language-python">import socket

# 创建UDP套接字
udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 绑定端口
udp_socket.bind(('localhost', 12345))

# 发送数据
udp_socket.sendto(b'Hello, UDP!', ('localhost', 12346))

# 接收数据
data, addr = udp_socket.recvfrom(1024)
print(f"Received from {addr}: {data.decode()}")

# 关闭套接字
udp_socket.close()</code></pre>
     </li>
     <li>
      <p>
       <strong>
        TCP 编程示例
       </strong>
      </p>
      <p>
       Python复制
      </p>
      <pre><code class="language-python">import socket

# 创建TCP套接字
tcp_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 绑定端口
tcp_socket.bind(('localhost', 12345))

# 监听连接
tcp_socket.listen(5)

# 接受连接
conn, addr = tcp_socket.accept()
print(f"Connected by {addr}")

# 发送数据
conn.sendall(b'Hello, TCP!')

# 接收数据
data = conn.recv(1024)
print(f"Received from {addr}: {data.decode()}")

# 关闭连接
conn.close()
tcp_socket.close()</code></pre>
     </li>
    </ul>
