    </p>
    <p>
    </p>
    <hr/>
    <p>
     因为虚拟机重新安装了一回，所以需要重新和远程仓库连接，在网上找了很久没有找到相关操作，自己实操成功，记录下本博客，帮助有需要的人
    </p>
    <hr/>
    <ol>
     <li>
      <p>
       <strong>
        确保新虚拟机安装Git
       </strong>
       <br/>
       在新虚拟机上，首先检查是否安装了Git。如果没有，安装它：
      </p>
      <ul>
       <li>
        对于Ubuntu/Debian系统：
        <code>
         sudo apt update &amp;&amp; sudo apt install git
        </code>
       </li>
       <li>
        对于CentOS/RHEL系统：
        <code>
         sudo yum install git
        </code>
        或
        <code>
         sudo dnf install git
        </code>
       </li>
       <li>
        安装完成后，运行
        <code>
         git --version
        </code>
        确认安装成功。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        配置Git基本信息
       </strong>
       <br/>
       如果你之前在远程仓库上有提交记录，建议配置相同的用户名和邮箱：
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"你的名字"</span>
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"你的邮箱"</span>
</code></pre>
      <p>
       这样可以保持提交记录的一致性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        克隆远程仓库到新虚拟机
       </strong>
       <br/>
       假设你之前用的是GitHub、GitLab或Gitee之类的平台，找到你远程仓库的URL（比如
       <code>
        https://gitee.com/用户名/仓库名.git
       </code>
       或 SSH 格式
       <code>
        git@gitee.com:用户名/仓库名.git
       </code>
       ）。然后在新虚拟机上运行：
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> clone 远程仓库URL
</code></pre>
      <p>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a339e1391e0942a28c6f39f8e46d0c62.png"/>
      </p>
      <ul>
       <li>
        如果使用 HTTPS URL，可能会提示输入用户名和密码（或个人访问令牌）。
       </li>
       <li>
        如果使用 SSH URL，需要在新虚拟机上生成SSH密钥并添加到远程仓库平台：
        <pre><code class="prism language-bash">ssh-keygen <span class="token parameter variable">-t</span> rsa <span class="token parameter variable">-b</span> <span class="token number">4096</span> <span class="token parameter variable">-C</span> <span class="token string">"你的邮箱"</span>
</code></pre>
        然后将
        <code>
         ~/.ssh/id_rsa.pub
        </code>
        的内容添加到远程仓库平台的SSH设置中。
        <br/>
        <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/faac140516a84a23a34800bf3923d80a.png"/>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        验证连接
       </strong>
       <br/>
       克隆完成后，进入仓库目录：
      </p>
      <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> 仓库名
<span class="token function">git</span> remote <span class="token parameter variable">-v</span>
</code></pre>
      <p>
       这会显示远程仓库的URL，确认是否正确连接到你之前的仓库。
      </p>
     </li>
     <li>
      <p>
       <strong>
        继续工作
       </strong>
       <br/>
       如果之前本地仓库有未推送的更改，但虚拟机已删除，那些更改可能已丢失。不过，远程仓库的内容应该都还在。你可以在新虚拟机上直接基于远程仓库的内容继续工作：
      </p>
      <ul>
       <li>
        查看分支：
        <code>
         git branch -a
        </code>
       </li>
       <li>
        切换分支（如果需要）：
        <code>
         git checkout 分支名
        </code>
       </li>
       <li>
        拉取最新代码：
        <code>
         git pull origin 分支名
        </code>
       </li>
      </ul>
     </li>
    </ol>
    <p>
     注意事项
    </p>
    <ul>
     <li>
      如果你之前在远程仓库启用了两步验证，可能需要使用个人访问令牌（Personal Access Token）而不是密码。
     </li>
     <li>
      如果不确定远程仓库URL，可以登录Gitee（或其他平台）查看仓库详情。
     </li>
    </ul>
