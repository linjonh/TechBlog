    为什么越来越多的Java工程师转向大数据
    <br/>
    <br/>
    Java语言在编程中的地位不言而喻，本文分析一下为什么越来越多的java工程师开始转向hadoop？
    <br/>
    Hadoop是Apache软件基金会的顶级开源项目，是由原雅虎公司Doug Cutting根据Google发布的学术论文而创建的开源项目。Hadoop项目是一套可靠的，可扩展的，支持分布式计算的开源软件。
    <br/>
    要从Java转向hadoop的原因进行交流，总结起来，主要有以下4点：
    <br/>
    <br/>
    1大数据是未来趋势
    <br/>
    <br/>
    <p>
     做Java也是不错的，不过目前大数据是个趋势，Hadoop本身又是Java开发的，再加上Hadoop工程师普遍比纯Java开发要高3k以上，所以有很多搞Java的都在往这hadoop大数据方向转。
    </p>
    <img alt="" src="https://img-blog.csdn.net/20180225191336638">
     <br/>
     2大数据岗位工资高
     <br/>
     <br/>
     做Java的人已经比较多了，但是能上2.5万的寥寥。而Hadoop很多人只1年经验就拿2万以上了。所以很多现在待遇还不错的人也还来学Hadoop。
     <br/>
     <br/>
     3大数据职位发展空间大
     <br/>
     <br/>
     Java这块如果做5~6年到管理岗位的话，薪资基本可以达到2万-2.5万了。但是2.5万基本上是Java技术人员的天花板，除非是架构师或者做底层的开发。但Hadoop这块2万多的薪资只能算一般，后面还有很大发展空间。
     <br/>
     <br/>
     4大数据不受年龄限制
     <br/>
     <br/>
     年龄大对搞技术的来说是个比较大的问题，Java工程师满大街都是，Hadoop这块年龄影响比较小，因为搞大数据不是简单的编程，编程的份量连1/6都不到，很多时候需要你从服务器、计算、运维等多个方面来分析问题解决问题，年龄越大经验越丰富，也越吃香。
