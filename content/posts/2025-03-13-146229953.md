    <p>
     来源GPT，仅记录学习
    </p>
    <hr/>
    <p>
     <code>
      yield return WaitForEndOfFrame()
     </code>
     适用于
     <strong>
      渲染结束后再执行代码
     </strong>
     ，但 WebGL 可能不适合这个操作，
     <strong>
      会拖慢帧率
     </strong>
     。(渲染得太慢）
    </p>
    <p>
     <code>
      yield return null;
     </code>
     让代码在
     <strong>
      下一帧的
      <code>
       Update()
      </code>
      里继续运行
     </strong>
     ，更加流畅。
    </p>
    <p>
     <code>
      yield return WaitForFixedUpdate()
     </code>
     <strong>
      （更平稳），
     </strong>
     如果你的游戏
     <strong>
      物理计算较多
     </strong>
     ，可以使用
     <code>
      WaitForFixedUpdate()
     </code>
     ，让协程与
     <strong>
      FixedUpdate()
     </strong>
     同步：
    </p>
    <p>
     <strong>
      适用情况：
     </strong>
    </p>
    <ul>
     <li>
      如果你的移动涉及到
      <strong>
       刚体物理（Rigidbody）
      </strong>
      。
     </li>
     <li>
      需要更平稳的帧率，但可能不如
      <code>
       yield return null;
      </code>
      灵敏。
     </li>
    </ul>
