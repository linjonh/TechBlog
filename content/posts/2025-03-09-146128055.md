    <p>
     Linux是一个多任务操作系统，肯定会存在多个任务共同操作同一段内存或者设备的情况，多个任务甚至中断都能访问的资源叫做共享资源，就和共享单车一样。在驱动开发中要注意对共享资源的保护，也就是要处理对共享资源的并发访问。
     <mark>
      比如共享单车，大家按照谁扫谁骑走的原则来共用这个单车，如果没有这个并发访问共享单车的原则存在，只怕到时候为了一辆单车要打起来了。
     </mark>
    </p>
    <h3>
     <a id="1_1">
     </a>
     1、并发与竞争简介
    </h3>
    <p>
     <img alt="Pasted image 20250306135841.png" src="https://i-blog.csdnimg.cn/img_convert/052732f517ada3e2d307917315f8b27d.png">
      <br/>
      <mark>
       打印机必须保证一次只能打印一份文档，只有打印完成以后才能打印其他的文档。
      </mark>
      <br/>
      <img alt="Pasted image 20250306140238.png" src="https://i-blog.csdnimg.cn/img_convert/ad73563b874a58010b40598cc72008c5.png">
       <br/>
       <mark>
        临界区
       </mark>
       就是共享数据段，对于临界区必须保证一次只有一个线程访问。
       <br/>
       也就是要保证
       <mark>
        临界区是原子访问的
       </mark>
       ，注意这里的==“原子==”不是正点原子的“原子”。
       <br/>
       我们都知道，原子是化学反应不可再分的基本微粒，这里的
       <mark>
        原子访问
       </mark>
       就表示这一个访问
       <mark>
        是一个步骤
       </mark>
       ，
       <mark>
        不能再进行拆分
       </mark>
       。如果
       <mark>
        多个线程同时操作临界区就表示存在竞争
       </mark>
       ，我们在编写驱动的时候一
       <mark>
        定要注意避免并发和防止竞争访问
       </mark>
       。
       <br/>
       <mark>
        重点
       </mark>
       ：我们一般在编写驱动的时候就要考虑到并发与竞争，而不是驱动都编写完了然后再处理并发与竞争。
       <br/>
       <img alt="Pasted image 20250306141039.png" src="https://i-blog.csdnimg.cn/img_convert/164c07f566dcafc527adc1cf38b19c90.png"/>
      </img>
     </img>
    </p>

    <h4>
     <a id="31API_29">
     </a>
     3.1、自旋锁API函数
    </h4>
    <p>
     <img alt="Pasted image 20250306144347.png" src="https://i-blog.csdnimg.cn/img_convert/81cf361da41ac45361bcccb2b17732c8.png"/>
     <br/>
     <img alt="Pasted image 20250306144837.png" src="https://i-blog.csdnimg.cn/img_convert/35a60812643931b7eb10229d3d050a61.png"/>
     <br/>
     <img alt="Pasted image 20250306145328.png" src="https://i-blog.csdnimg.cn/img_convert/e7ae4121c33885e567108678561a8e6c.png"/>
     <br/>
     <img alt="Pasted image 20250306145356.png" src="https://i-blog.csdnimg.cn/img_convert/92abd7ec48948951deeab2c2b2bb252b.png"/>
     <br/>
     <mark>
      这个意思就是
     </mark>
     ：假如没有开启禁止本地中断，在线程A在获取锁后，中断开始想获取这个锁，但是因为线程A占了这个锁，所以中断没法执行，但是中断调用函数优先权比线程A高，线程A不可能执行，所以僵持下去了！
     <br/>
     <mark>
      假如没有这个锁，线程A一旦被中断事件中断，就必须让出cpu使用。先中断后线程A。
     </mark>
     <br/>
     最好的解决方法就是获取锁之前关闭本地中断。
     <br/>
     <mark>
      上面有API函数进行处理！
     </mark>
     <br/>
     <img alt="Pasted image 20250306151442.png" src="https://i-blog.csdnimg.cn/img_convert/7296730f313450e3bc35fae6aa423a0f.png"/>
     <br/>
     当线程开始时，禁止本地中断，获取自旋锁；当中断开始时，激活本地中断，释放自旋锁。
     <br/>
     一般在线程中使用
     <code>
      spin_lock_irqsave/ spin_unlock_irqrestore
     </code>
     ，在中断中使用
     <code>
      spin_lock/spin_unlock
     </code>
     。
     <br/>
     例如：

     <a id="32_44">

     3.2、其他类型的锁
    </h4>
    <h5>
     <a id="321_45">
     </a>
     3.2.1、读写自旋锁
    </h5>
    <p>
     <img alt="Pasted image 20250306152930.png" src="https://i-blog.csdnimg.cn/img_convert/3bd565ef56abccfecdbd167e5d46c2be.png"/>
     <br/>
     <mark>
      这个意思是
     </mark>
     ：只能一个线程进行写锁操作，支持多个线程进行读锁操作。修改数据只可以一个线程进行，读取数据可以多个线程同时进行。
     <br/>
     <img alt="Pasted image 20250306153112.png" src="https://i-blog.csdnimg.cn/img_convert/2554f6bdbd83ad2b2bd332e26fd8bf60.png"/>
     <br/>
     读写锁操作 API 函数分为两部分，一个是给读使用的，一个是给写使用的。
     <br/>
     <img alt="Pasted image 20250306153151.png" src="https://i-blog.csdnimg.cn/img_convert/3f62b3d7089e79e5456c77a3d58badab.png"/>

    <h5>
     <a id="322_52">
     </a>
     3.2.2、顺序锁
    </h5>
    <p>
     顺序锁在读写锁的基础上衍生而来的，
     <mark>
      使用读写锁的时候读操作和写操作不能同时进行
     </mark>
     。使用顺序锁的话可以允许在写的时候进行读操作，也就是实现同时读写，
     <mark>
      但是不允许同时进行并发的写操作
     </mark>
     <br/>
     。
     <br/>
     <mark>
      虽然顺序锁的读和写操作可以同时进行，但是如果在读的过程中发生了写操作
     </mark>
     ，
     <mark>
      最好重新进行读取
     </mark>
     ，保证数据完整性。
     <br/>
     <mark>
      顺序锁保护的资源不能是指针，因为如果在写操作的时候可能会导致指针无效，而这个时候恰巧有读操作访问指针的话就可能导致意外发生，比如读取野指针导致系统崩溃
     </mark>
     。
     <br/>
     Linux 内核使用 seqlock_t 结构体表示顺序锁。

     <img alt="Pasted image 20250306154743.png" src="https://i-blog.csdnimg.cn/img_convert/13467af32b86f3934ea642452d149418.png"/>

    <h3>
     <a id="5_72">
     </a>
     5、互斥体
    </h3>
    <p>
     将信号量的值设置为 1 就可以使用信号量进行互斥访问了。
     <br/>
     虽然可以通过信号量实现互斥，但是 Linux 提供了一个比信号量更专业的机制来进行互斥，它就是互斥体—mutex。
     <br/>
     互斥访问表示
     <mark>
      一次只有一个线程
     </mark>
     可以访问共享资源，不能递归申请互斥体。
     <br/>
     在我们编写 Linux 驱动的时候遇到需要互斥访问的地方建议使用
     <mark>
      mutex
     </mark>
     。
     <br/>
     使用 mutex 结构体表示互斥体，定义如下：
