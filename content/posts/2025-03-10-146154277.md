    <h2>
     <a id="vue3__9">
     </a>
     vue3 的生命周期有哪些？
    </h2>
    <p>
     很多人可能是从咱们的上一节
     <a href="https://blog.csdn.net/General_D/article/details/146140862?fromshare=blogdetail&amp;sharetype=blogdetail&amp;sharerId=146140862&amp;sharerefer=PC&amp;sharesource=General_D&amp;sharefrom=from_link">
      vue2篇
     </a>
     过来的，不过不管3还是2，都一样重点讲组件的四个阶段八个周期：
     <strong>
      初始化阶段、挂载阶段、更新阶段、注销阶段
     </strong>
    </p>
    <h3>
     <a id="1_12">
     </a>
     1.组件创建时（初始化阶段）
    </h3>
    <p>
     <code>
      setup()
     </code>
     （替代 beforeCreate 和 created）
    </p>
    <ul>
     <li>
      触发时机：组件初始化时最先执行。
     </li>
     <li>
      作用：
      <br/>
      定义响应式数据（ref, reactive）。
      <br/>
      声明方法。
      <br/>
      组合逻辑（Composition API 的核心）。
      <br/>
      <code>
       注意：setup() 执行时机早于 beforeCreate，created 逻辑需写在 setup 中
      </code>
     </li>
    </ul>
    <h3>
     <a id="2__20">
     </a>
     2. 组件被挂载时（挂载阶段）
    </h3>
    <p>
     <code>
      onBeforeMount()
     </code>
    </p>
    <p>
     <strong>
      使用场景
     </strong>
     ：
    </p>
    <ul>
     <li>
      几乎不用，类似 Vue2 的 beforeMount。
     </li>
    </ul>
    <p>
     <code>
      onMounted()
     </code>
    </p>
    <p>
     <strong>
      使用场景：
     </strong>
    </p>
    <ul>
     <li>
      操作 DOM（如初始化 ECharts 图表）。
     </li>
     <li>
      发起依赖 DOM 的请求（获取元素尺寸）。
     </li>
     <li>
      <strong>
       添加全局事件监听
      </strong>
      （window.addEventListener）。
     </li>
    </ul>
    <h3>
     <a id="3__35">
     </a>
     3. 数据更新时（更新阶段）
    </h3>
    <p>
     <code>
      onBeforeUpdate()
     </code>
    </p>
    <p>
     <strong>
      使用场景：
     </strong>
    </p>
    <ul>
     <li>
      记录更新前的 DOM 状态（如滚动位置）。
     </li>
     <li>
      移除旧的 DOM 事件监听。
     </li>
    </ul>
    <p>
     <code>
      onUpdated()
     </code>
    </p>
    <p>
     <strong>
      使用场景：
     </strong>
    </p>
    <ul>
     <li>
      操作更新后的 DOM（如滚动到最新内容）。
     </li>
     <li>
      同步第三方库状态（如更新地图位置）。
     </li>
     <li>
      ⚠️ 避免在此修改数据（可能导致无限循环）！
     </li>
    </ul>
    <h3>
     <a id="4__52">
     </a>
     4. 组件卸载（卸载阶段）
    </h3>
    <p>
     <code>
      onBeforeUnmount()
     </code>
    </p>
    <p>
     <strong>
      使用场景：
     </strong>
    </p>
    <ul>
     <li>
      清除定时器（clearInterval）。
     </li>
     <li>
      解绑全局事件（window.removeEventListener）。
     </li>
     <li>
      释放外部资源（如销毁 ECharts 实例）。
     </li>
    </ul>
    <p>
     <code>
      onUnmounted()
     </code>
    </p>
    <p>
     <strong>
      使用场景：
     </strong>
    </p>
    <ul>
     <li>
      几乎不用（此时组件已销毁）。
     </li>
    </ul>

    <p>
     <strong>
      注意事项
     </strong>
     🚧
    </p>
    <ul>
     <li>
      优先使用 Composition API：更灵活地组织代码逻辑。
     </li>
     <li>
      避免在 onUpdated 中修改状态：可能导致无限更新循环。
     </li>
     <li>
      资源清理应在 onBeforeUnmount 完成：onUnmounted 阶段可能已无法访问组件实例。
     </li>
    </ul>

    <h2>
     <a id="_125">
     </a>
     关键记忆点（面试重点）：
    </h2>
    <p>
     <strong>
      命名变化：
     </strong>
    </p>
    <ul>
     <li>
      beforeDestroy → beforeUnmount
     </li>
     <li>
      destroyed → unmounted（强调“卸载”而非“销毁”）。
     </li>
    </ul>
    <p>
     <strong>
      Composition API：
     </strong>
    </p>
    <ul>
     <li>
      Vue3 通过 onXxx 函数（如 onMounted）在 setup 中注册生命周期。 set
     </li>
     <li>
      up 替代了 beforeCreate 和 created，直接在此初始化逻辑。
     </li>
    </ul>
    <p>
     <strong>
      执行顺序：
     </strong>
    </p>
    <ul>
     <li>
      setup() 执行时机早于 beforeCreate，created 逻辑需写在 setup 中。
     </li>
    </ul>
