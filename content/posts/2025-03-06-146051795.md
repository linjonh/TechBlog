    <h2>
     <a id="_Cell_Ranger_Linux__0">
     </a>
     高效使用 Cell Ranger：安装、参数解析及 Linux 后台运行指南
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/945d377836a848a9b08ce29c88baca76.png"/>
    </p>
    <h3>
     <a id="_3">
     </a>
     背景介绍
    </h3>
    <p>
     Cell Ranger 是 10x Genomics 开发的一套用于单细胞转录组测序数据处理的软件。它可以对 10x Genomics 平台生成的
     <code>
      FASTQ
     </code>
     文件进行对齐、UMI 计数和基因表达量计算，是单细胞 RNA-seq 数据分析的第一步。由于 Cell Ranger 对输入数据格式有严格要求，并且计算资源需求较高，因此在使用时需要注意安装环境、文件命名规范以及后台运行的方式。
    </p>
    <p>
     本指南将介绍 Cell Ranger 的安装方法、数据处理流程、文件命名规则、运行参数介绍以及如何在 Linux 端高效运行 Cell Ranger。
    </p>
    <h3>
     <a id="1__Cell_Ranger_9">
     </a>
     1. 安装 Cell Ranger
    </h3>
    <h4>
     <a id="11__11">
     </a>
     1.1 下载与解压安装
    </h4>
    <pre><code class="prism language-bash"><span class="token comment"># 下载 Cell Ranger 7.2.0 版本</span>
<span class="token function">wget</span> <span class="token parameter variable">-O</span> cellranger-7.2.0.tar.gz <span class="token string">"https://cf.10xgenomics.com/releases/cell-exp/cellranger-7.2.0.tar.gz"</span>

<span class="token comment"># 解压并移动到合适的目录</span>
<span class="token function">tar</span> <span class="token parameter variable">-xzvf</span> cellranger-7.2.0.tar.gz
<span class="token function">mv</span> cellranger-7.2.0 /opt/cellranger-7.2.0

<span class="token comment"># 添加路径（建议写入 ~/.bashrc 以便每次启动生效）</span>
<span class="token builtin class-name">echo</span> <span class="token string">'export PATH=/opt/cellranger-7.2.0:$PATH'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
    <h4>
     <a id="12__26">
     </a>
     1.2 下载参考基因库
    </h4>
    <p>
     可以选择不同版本的参考基因库：
    </p>
    <ul>
     <li>
      <a href="https://www.10xgenomics.com/support/software/cell-ranger/downloads" rel="nofollow" title="2024-A 版本">
       2024-A 版本
      </a>
     </li>
     <li>
      <a href="https://www.10xgenomics.com/support/software/cell-ranger/latest/release-notes/cr-reference-release-notes#2020-a" rel="nofollow" title="2020-A 版本">
       2020-A 版本
      </a>
     </li>
    </ul>
    <p>
     下载并解压：
    </p>
    <pre><code class="prism language-bash"><span class="token function">wget</span> <span class="token parameter variable">-O</span> refdata-gex-GRCh38-2020-A.tar.gz <span class="token string">"https://cf.10xgenomics.com/releases/cell-exp/refdata-gex-GRCh38-2020-A.tar.gz"</span>
<span class="token function">tar</span> <span class="token parameter variable">-xzvf</span> refdata-gex-GRCh38-2020-A.tar.gz <span class="token parameter variable">-C</span> /home/user/reference/
</code></pre>
    <h3>
     <a id="2__Cell_Ranger_count_38">
     </a>
     2. 运行 Cell Ranger
     <code>
      count
     </code>
    </h3>
    <h4>
     <a id="21_cellranger_count__40">
     </a>
     2.1
     <code>
      cellranger count
     </code>
     主要参数介绍
    </h4>
    <pre><code class="prism language-bash">cellranger count <span class="token parameter variable">--id</span><span class="token operator">=</span>run_count_sample1 <span class="token punctuation">\</span>
   <span class="token parameter variable">--fastqs</span><span class="token operator">=</span>/home/user/yard/run_cellranger_count/sample1 <span class="token punctuation">\</span>
   <span class="token parameter variable">--sample</span><span class="token operator">=</span>sample1 <span class="token punctuation">\</span>
   <span class="token parameter variable">--transcriptome</span><span class="token operator">=</span>/home/user/yard/run_cellranger_count/refdata-gex-GRCh38-2020-A
</code></pre>
    <p>
     <strong>
      参数说明：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       --id
      </code>
      ：为该运行设置一个唯一的名称，所有输出文件将存储在
      <code>
       --id
      </code>
      目录下。
     </li>
     <li>
      <code>
       --fastqs
      </code>
      ：指定存放
      <code>
       FASTQ
      </code>
      文件的路径。
     </li>
     <li>
      <code>
       --sample
      </code>
      ：指定样本名称，必须匹配
      <code>
       FASTQ
      </code>
      文件的前缀。
     </li>
     <li>
      <code>
       --transcriptome
      </code>
      ：指定参考基因组路径。
     </li>
     <li>
      <code>
       --nosecondary
      </code>
      （可选）：跳过降维分析，减少计算时间和存储需求。
     </li>
     <li>
      <code>
       --localcores=N
      </code>
      （可选）：指定使用的 CPU 核心数，默认使用所有可用核心。
     </li>
     <li>
      <code>
       --localmem=N
      </code>
      （可选）：指定使用的最大内存（GB）。
     </li>
    </ul>
    <p>
     如果不需要使用 Cell Ranger 的降维结果，可以添加
     <code>
      --nosecondary
     </code>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">title</span><span class="token operator">=</span><span class="token string">"每次在下面进行修改，然后粘贴到终端运行"</span>
cellranger count <span class="token parameter variable">--id</span><span class="token operator">=</span>run_count_sample2 <span class="token punctuation">\</span>
   <span class="token parameter variable">--fastqs</span><span class="token operator">=</span>/home/user/yard/test <span class="token punctuation">\</span>
   <span class="token parameter variable">--sample</span><span class="token operator">=</span>sample2 <span class="token punctuation">\</span>
   <span class="token parameter variable">--transcriptome</span><span class="token operator">=</span>/home/user/reference/refdata-gex-GRCh38-2020-A <span class="token punctuation">\</span>
   <span class="token parameter variable">--nosecondary</span>
</code></pre>
    <h4>
     <a id="22__68">
     </a>
     2.2 后台运行方式对比
    </h4>
    <p>
     在 Linux 端，后台运行
     <code>
      cellranger
     </code>
     可采用多种方式，以下是常见方法的对比：
    </p>
    <h5>
     <a id="_1_nohup_72">
     </a>
     <strong>
      方法 1：使用
      <code>
       nohup
      </code>
     </strong>
    </h5>
    <p>
     适用于简单的后台运行，任务不会因退出终端而中断。
    </p>
    <pre><code class="prism language-bash"><span class="token function">nohup</span> cellranger count <span class="token parameter variable">--id</span><span class="token operator">=</span>run_count_sample3 <span class="token punctuation">\</span>
   <span class="token parameter variable">--fastqs</span><span class="token operator">=</span>/home/user/yard/run_cellranger_count/sample3 <span class="token punctuation">\</span>
   <span class="token parameter variable">--sample</span><span class="token operator">=</span>sample3 <span class="token punctuation">\</span>
   <span class="token parameter variable">--transcriptome</span><span class="token operator">=</span>/home/user/yard/run_cellranger_count/refdata-gex-GRCh38-2020-A <span class="token punctuation">\</span>
   <span class="token parameter variable">--nosecondary</span> <span class="token operator">&amp;&gt;</span> run_sample3.log <span class="token operator">&amp;</span>
</code></pre>
    <h5>
     <a id="_2_screen_82">
     </a>
     <strong>
      方法 2：使用
      <code>
       screen
      </code>
     </strong>
    </h5>
    <p>
     适用于长时间运行的任务，可随时恢复。
    </p>
    <pre><code class="prism language-bash"><span class="token function">screen</span> <span class="token parameter variable">-S</span> cellranger_run
cellranger count <span class="token parameter variable">--id</span><span class="token operator">=</span>run_count_sample4 <span class="token punctuation">\</span>
   <span class="token parameter variable">--fastqs</span><span class="token operator">=</span>/home/user/yard/sample4 <span class="token punctuation">\</span>
   <span class="token parameter variable">--sample</span><span class="token operator">=</span>sample4 <span class="token punctuation">\</span>
   <span class="token parameter variable">--transcriptome</span><span class="token operator">=</span>/home/user/yard/run_cellranger_count/refdata-gex-GRCh38-2020-A
</code></pre>
    <p>
     退出
     <code>
      screen
     </code>
     但保持进程运行：
    </p>
    <pre><code class="prism language-bash">Ctrl + A, 然后 D
</code></pre>
    <p>
     重新进入
     <code>
      screen
     </code>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token function">screen</span> <span class="token parameter variable">-r</span> cellranger_run
</code></pre>
    <h5>
     <a id="_3_tmux_100">
     </a>
     <strong>
      方法 3：使用
      <code>
       tmux
      </code>
     </strong>
    </h5>
    <p>
     比
     <code>
      screen
     </code>
     更强大，支持多个窗口管理。
    </p>
    <pre><code class="prism language-bash">tmux new <span class="token parameter variable">-s</span> cellranger_run
cellranger count <span class="token parameter variable">--id</span><span class="token operator">=</span>run_count_sample5 <span class="token punctuation">\</span>
   <span class="token parameter variable">--fastqs</span><span class="token operator">=</span>/home/user/yard/sample5 <span class="token punctuation">\</span>
   <span class="token parameter variable">--sample</span><span class="token operator">=</span>sample5 <span class="token punctuation">\</span>
   <span class="token parameter variable">--transcriptome</span><span class="token operator">=</span>/home/user/yard/reference/refdata-gex-GRCh38-2020-A
</code></pre>
    <p>
     退出
     <code>
      tmux
     </code>
     但保持进程运行：
    </p>
    <pre><code class="prism language-bash">Ctrl + B, 然后 D
</code></pre>
    <p>
     重新进入
     <code>
      tmux
     </code>
     会话：
    </p>
    <pre><code class="prism language-bash">tmux attach <span class="token parameter variable">-t</span> cellranger_run
</code></pre>
    <h3>
     <a id="3_FASTQ__118">
     </a>
     3. FASTQ 文件命名规则与注意事项
    </h3>
    <p>
     Cell Ranger 需要特定格式的
     <code>
      FASTQ
     </code>
     文件名，以确保正确识别数据：
    </p>
    <pre><code class="prism language-text">[Sample Name]_S[Sample Number]_L[Lane Number]_R[Read Type]_001.fastq.gz
</code></pre>
    <p>
     示例：
    </p>
    <pre><code class="prism language-text">sample1_S1_L001_R1_001.fastq.gz
sample1_S1_L001_R2_001.fastq.gz
</code></pre>
    <h4>
     <a id="31__131">
     </a>
     3.1 注意事项
    </h4>
    <ul>
     <li>
      <strong>
       文件名必须符合 10x Genomics 规定
      </strong>
      ，否则
      <code>
       cellranger
      </code>
      无法识别。
     </li>
     <li>
      <strong>
       避免空格和特殊字符
      </strong>
      ，所有文件名应严格按照
      <code>
       S1_L001_R1_001.fastq.gz
      </code>
      这种格式命名。
     </li>
     <li>
      <strong>
       确保所有 FASTQ 文件在同一目录下
      </strong>
      ，并正确指定
      <code>
       --fastqs
      </code>
      参数。
     </li>
    </ul>
    <h4>
     <a id="32__136">
     </a>
     3.2 检查命名是否符合格式
    </h4>
    <pre><code class="prism language-bash"><span class="token function">ls</span> /path/to/fastq_files <span class="token operator">|</span> <span class="token function">grep</span> <span class="token parameter variable">-E</span> <span class="token string">".*_S[0-9]+_L[0-9]{3}_R[12]_001.fastq.gz"</span>
</code></pre>
    <h3>
     <a id="4__142">
     </a>
     4. 目录结构建议
    </h3>
    <p>
     建议使用标准化的目录结构管理数据：
    </p>
    <pre><code class="prism language-bash">├── data/
│   ├── raw_fastq/
│   ├── processed/
├── reference/
│   ├── GRCh38-2020-A/
├── scripts/
├── results/
</code></pre>
    <p>
     这样可以更方便管理数据，避免混乱。
    </p>
