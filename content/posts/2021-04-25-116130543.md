    <h2>
     1、什么是物联网平台
    </h2>
    <p>
     阿里云物联网平台为设备提供安全可靠的连接通信能力，向下连接海量设备，支撑设备数据采集上云；向上提供云端API，服务端通过调用云端API将指令下发至设备端，实现远程控制。
    </p>
    <p id="p-cyw-cen-2mi">
     物联网平台消息通信流程图如下。
    </p>
    <p>
     <a href="https://static-aliyun-doc.oss-accelerate.aliyuncs.com/assets/img/zh-CN/3199158061/p132750.png" rel="nofollow">
      <img alt="企业基于物联网平台的业务链路" height="332" id="image-f6b-am4-tvi" src="https://i-blog.csdnimg.cn/blog_migrate/7082ade7d1a29bce38e40194c5e817f4.png" width="813"/>
     </a>
    </p>
    <p id="p-6kj-z2d-kk4">
     实现设备消息的完整通信流程，需要您完成设备端的设备开发、云端服务器的开发（云端SDK的配置）、数据库的创建、手机App的开发。
    </p>
    <p id="p-hda-vl2-5l6">
     下文介绍物联网平台消息通信链路。
    </p>
    <p id="p-rc1-x4s-ydc">
     关于设备通信的更多信息，请参见
     <a href="https://help.aliyun.com/document_detail/146382.htm#concept-2358399" rel="nofollow">
      通信方式
     </a>
     。
    </p>
    <p>
    </p>
    <h3 id="title-b0c-oxw-e13">
     上行数据链路
    </h3>
    <ul id="ul-hdg-xvy-rph">
     <li id="li-kli-mkg-rjj">
      设备通过MQTT协议与物联网平台建立长连接，上报数据（通过Publish发布Topic和Payload）到物联网平台。
     </li>
     <li id="li-217-3l3-zpx">
      您可配置规则引擎，编写SQL对上报数据进行处理，并配置转发规则，将处理后的数据转发到RDS、表格存储、函数计算、TSDB、企业版实例内的时序数据存储、DataHub、消息队列RocketMQ等云产品中，或通过AMQP消费组流转到您的ECS服务器上。
     </li>
     <li>
     </li>
    </ul>
    <h3 id="title-bm5-qmv-dow">
     下行指令链路
    </h3>
    <ul id="ul-3nq-t4w-w3a">
     <li id="li-r85-v0o-ovt">
      ECS业务服务器调用基于HTTPS协议的API接口Pub，给Topic发送指令，将数据发送到物联网平台。
     </li>
     <li id="li-lul-eid-54h">
      物联网平台通过MQTT协议，使用Publish发送数据（指定Topic和Payload）到设备端。
     </li>
    </ul>
    <p>
    </p>
    <h2>
     2、产品功能
    </h2>
    <p>
     物联网平台主要提供了设备接入、设备管理、规则引擎等能力，为各类IoT场景和行业开发者赋能。
    </p>
    <p>
    </p>
    <h3 id="title-7ok-vib-rya">
     设备接入
    </h3>
    <p id="p-mum-42n-51a">
     物联网平台支持海量设备连接上云，设备与云端通过IoT Hub进行稳定可靠地双向通信。
    </p>
    <ul id="ul-3f9-sk4-f87">
     <li id="li-wxa-v5t-mw4">
      <a href="https://help.aliyun.com/document_detail/42648.htm#concept-jlk-mjl-vdb" rel="nofollow">
       设备开发
      </a>
      ：提供设备端SDK、驱动等，帮助不同设备、网关轻松接入阿里云。
     </li>
     <li id="li-0ao-grf-vgq">
      <a href="https://help.aliyun.com/document_detail/115438.htm#concept-2050364" rel="nofollow">
       设备接入引导
      </a>
      ：提供蜂窝（2G、3G、4G、5G）、NB-IoT、LoRaWAN、Wi-Fi等不同网络设备接入方案，解决企业异构网络管理设备接入的痛点。
     </li>
     <li id="li-afm-txh-h8x">
      提供
      <a href="https://help.aliyun.com/document_detail/30540.htm#concept-jfq-vjw-vdb" rel="nofollow">
       MQTT
      </a>
      、
      <a href="https://help.aliyun.com/document_detail/57699.htm#concept-tcf-yq5-wdb" rel="nofollow">
       CoAP
      </a>
      、
      <a href="https://help.aliyun.com/document_detail/58036.htm#concept-l14-nt5-wdb" rel="nofollow">
       HTTP、HTTPS
      </a>
      等多种协议的设备端SDK，既满足长连接的实时性需求，也满足短连接的低功耗需求。
     </li>
     <li id="li-qyi-b4n-02h">
      <a href="https://help.aliyun.com/document_detail/100576.htm#concept-2050349" rel="nofollow">
       SDK不同语言或平台功能汇总
      </a>
      ：开源多种平台设备端代码，提供跨平台移植指导，赋能企业基于多种平台做设备接入。
     </li>
    </ul>
    <p>
    </p>
    <h3 id="title-rv1-m3q-fmj">
     设备管理
    </h3>
    <p id="p-68t-bog-gi4">
     物联网平台提供完整的设备生命周期管理功能，支持
     <a href="https://help.aliyun.com/document_detail/73729.htm#task-yk1-rnl-vdb" rel="nofollow">

     <a href="https://help.aliyun.com/document_detail/68702.htm#concept-rhj-535-42b" rel="nofollow">

     </a>
     、
     <a href="https://help.aliyun.com/document_detail/99981.htm#task-wpk-dtg-cgb" rel="nofollow">
      在线调试
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/84891.htm#concept-om1-tgs-c2b" rel="nofollow">
      远程配置
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/58328.htm#task-prw-fzz-xdb" rel="nofollow">
      OTA升级
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/141582.htm#concept-2321153" rel="nofollow">
      实时监控
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/90386.htm#task-ejm-qp2-cfb" rel="nofollow">
      设备分组
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/113275.htm#concept-h5h-q44-hhb" rel="nofollow">
      设备删除
     </a>
     等功能。
    </p>
    <p id="p-fma-rvi-yy2">
     功能特性如下：
    </p>
    <ul id="ul-7de-ynn-uua">
     <li id="li-zn0-5pj-yb7">
      提供设备物模型，简化应用开发。
     </li>
     <li id="li-dfk-poc-jul">
      提供设备上下线变更通知服务，方便实时获取设备状态。
     </li>
     <li id="li-bb8-hj4-slv">
      提供数据存储能力，方便用户海量设备数据的存储及实时访问。
     </li>
     <li id="li-avr-tr1-6l6">
      支持OTA升级，赋能设备远程升级。
     </li>
     <li id="li-0nb-zsq-tb4">
      提供设备影子缓存机制，将设备与应用解耦，解决不稳定无线网络下的通信不可靠痛点。
     </li>
    </ul>

    <h3 id="title-xjr-pdj-hmn">
     安全能力
    </h3>
    <p id="p-ngz-2tj-ggq">
     物联网平台提供多重防护，有效保障设备和云端数据的安全。
    </p>
    <ul id="ul-69q-9xd-rqg">
     <li id="li-enw-tun-xhj">
      <a href="https://help.aliyun.com/document_detail/42649.htm#concept-fqy-pjl-vdb" rel="nofollow">
       身份认证
      </a>
      <ul id="ul-jt9-qq1-4nb">
       <li id="li-arr-o4s-mv3">
        提供芯片级安全存储方案（ID²）及设备密钥安全管理机制，防止设备密钥被破解。安全级别很高。
       </li>
       <li id="li-8u5-du0-p3d">
        提供一机一密的设备认证机制，降低设备被攻破的安全风险。适合有能力批量预分配设备证书（ProductKey、DeviceName和DeviceSecret），将设备证书信息烧录到每个设备的芯片。安全级别高。
       </li>
       <li id="li-ev5-whl-var">
        提供一型一密的设备认证机制。设备预烧产品证书（ProductKey和ProductSecret），认证时动态获取设备证书（包括ProductKey、DeviceName和DeviceSecret）。适合批量生产时无法将设备证书烧录每个设备的情况。安全级别普通。
       </li>
       <li id="li-w3z-rt8-n0q">
        提供X.509证书的设备认证机制，支持基于MQTT协议直连的设备使用X.509证书进行认证。安全级别很高。
       </li>
      </ul>
     </li>
     <li id="li-ypq-dh5-28l">
      通信安全
      <ul id="ul-lpm-48e-94v">
       <li id="li-whp-wza-tmb">
        支持TLS（
        <a href="https://help.aliyun.com/document_detail/30540.htm#concept-jfq-vjw-vdb" rel="nofollow">
         MQTT
        </a>
        、
        <a href="https://help.aliyun.com/document_detail/58036.htm#concept-l14-nt5-wdb" rel="nofollow">
         HTTPS
        </a>
        ）、DTLS（
        <a href="https://help.aliyun.com/document_detail/57699.htm#concept-tcf-yq5-wdb" rel="nofollow">
         CoAP
        </a>
        ）数据传输通道，保证数据的机密性和完整性，适用于硬件资源充足、对功耗不是很敏感的设备。安全级别高。
       </li>
       <li id="li-5lu-k5t-nqx">
        支持设备权限管理机制，保障设备与云端安全通信。
       </li>
       <li id="li-l77-azx-ooj">
        支持设备级别的通信资源（Topic等）隔离，防止设备越权等问题。
       </li>
      </ul>
     </li>
    </ul>
    <p>

    <h2>
     3、产品架构
    </h2>
    <p>
     设备连接物联网平台，与物联网平台进行数据通信。物联网平台可将设备数据流转到其他阿里云产品中进行存储和处理。这是构建物联网应用的基础。
    </p>
    <p>
     <a href="https://static-aliyun-doc.oss-accelerate.aliyuncs.com/assets/img/zh-CN/7769728061/p202711.png" rel="nofollow">

     </a>
    </p>

    <h3 id="title-2oc-ihh-9pu">
     IoT SDK
    </h3>
    <p id="p-1rp-lv9-szg">
     物联网平台提供IoT SDK，设备集成SDK后，即可安全接入物联网平台，使用设备管理、数据流转等功能。
    </p>
    <p id="p-c6o-5hg-zli">
     只有支持TCP/IP协议的设备可以集成IoT SDK。
    </p>
    <p>
     具体操作，请参见
     <a href="https://help.aliyun.com/document_detail/42648.htm#concept-jlk-mjl-vdb" rel="nofollow">
      设备端SDK开发指南
     </a>
     。
    </p>
    <p>
    </p>
    <h3 id="title-0hc-45c-4ha">
     边缘计算
    </h3>
    <p id="p-c0n-gip-ix4">
     边缘计算能力允许您在最靠近设备的地方构建边缘计算节点，过滤清洗设备数据，并将处理后的数据上传至云平台。更多信息，请参见
     <a href="https://help.aliyun.com/document_detail/85155.htm#concept-egf-t13-d2b" rel="nofollow">
      物联网边缘计算
     </a>
     。
    </p>
    <p>
    </p>
    <h3 id="title-464-rjh-p95">
     设备接入
    </h3>
    <p id="p-wo6-3lh-f7r">
     物联网平台提供各类
     <a href="https://help.aliyun.com/document_detail/42648.htm#concept-jlk-mjl-vdb" rel="nofollow">
      设备端SDK
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/42649.htm#concept-fqy-pjl-vdb" rel="nofollow">
      设备认证方式
     </a>
     ，支持
     <a href="https://help.aliyun.com/document_detail/30540.htm#concept-jfq-vjw-vdb" rel="nofollow">
      MQTT
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/57699.htm#concept-tcf-yq5-wdb" rel="nofollow">
      CoAP
     </a>
     、
     <a href="https://help.aliyun.com/document_detail/58036.htm#concept-l14-nt5-wdb" rel="nofollow">
      HTTP
     </a>
     等多种协议，实现设备快速上云。
    </p>
    <p id="p-44i-ybn-mer">
     设备上云后与云端通过IoT Hub进行稳定可靠地双向通信。
    </p>
    <p>
     IoT Hub具有下列特性：
    </p>
    <ul id="ul-3xv-3tt-r00">
     <li id="li-tbz-a6s-7j8">
      高性能扩展：支持线性动态扩展，可以支撑十亿设备同时连接。
     </li>
     <li id="li-8rq-u2o-epf">
      全链路加密：整个通信链路以RSA、AES加密，保证数据传输的安全。
     </li>
     <li id="li-849-ydm-qaf">
      消息实时到达：当设备与IoT Hub成功建立数据通道后，两者间将保持长连接，以减少握手时间，保证消息实时到达。
     </li>
     <li id="li-fjp-558-p2r">
      支持数据透传： IoT Hub支持将数据以二进制透传的方式传到自己的服务器上，不保存设备数据，从而保证数据的安全可控性。
     </li>
     <li id="li-pt6-rxv-45e">
      支持多种通信模式：IoT Hub支持RRPC和PUB/SUB两种通信模式，以满足您在不同场景下的需求。其中，PUB/SUB是基于Topic进行的消息路由。
     </li>
     <li id="li-g4w-cyx-k1x">
      支持多种设备接入协议：支持设备使用CoAP、MQTT、HTTPS协议接入物联网平台。
     </li>
    </ul>

    <h3 id="title-6xx-5qx-vkf">
     安全认证和权限策略
    </h3>
    <p id="p-v28-b7t-25u">
     安全是IoT的重要话题。阿里云物联网平台提供多重防护，保障设备和云端数据的安全。
    </p>
    <ul id="ul-pcq-mpp-tdb">
     <li id="li-ghy-ory-l3i">
      物联网平台为每个设备颁发唯一证书，设备使用证书进行身份验证连接物联网平台。
     </li>
     <li id="li-l2s-ids-l6z">
      针对不同安全等级和产线烧录的要求，物联网平台为开发者提供了多种设备认证方式。
     </li>
     <li id="li-d6g-k9b-mzj">
      授权粒度精确到设备级别，任何设备只能对自己的Topic发布、订阅消息。服务端凭借阿里云AccessKey对账号下所属的Topic进行操作。
     </li>
    </ul>
