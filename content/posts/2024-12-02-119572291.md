     HarmonyOS 系统架构

    <p>
     HarmonyOS是一款面向
     <code>
      万物互联
     </code>
     时代的、全新的
     <code>
      分布式
     </code>
     操作系统。
    </p>
    <p>
     在传统的单设备系统能力基础上，HarmonyOS提出了基于
     <code>
      同一套系统能力
     </code>
     、
     <code>
      适配多种终端形态
     </code>
     的分布式理念。能够支持手机、平板、智能穿戴、智慧屏、车机等多种终端设备，提供全场景（移动办公、运动健康、社交通信、媒体娱乐等）业务能力。
    </p>
    <p>
     HarmonyOS具有三大特征：
    </p>
    <ul>
     <li>
      搭载该操作系统的设备在系统层面融为一体、形成超级终端，让设备的硬件能力可以弹性扩展，实现设备之间硬件互助，资源共享。
      <br/>
      对消费者而言，HarmonyOS能够将生活场景中的各类终端进行能力整合，实现不同终端设备之间的快速连接、能力互助、资源共享，匹配合适的设备、提供流畅的全场景体验。
     </li>
     <li>
      面向开发者，实现一次开发，多端部署。
      <br/>
      对应用开发者而言，HarmonyOS采用了多种分布式技术，使应用开发与不同终端设备的形态差异无关，从而让开发者能够聚焦上层业务逻辑，更加便捷、高效地开发应用。
     </li>
     <li>
      一套操作系统可以满足不同能力的设备需求，实现统一OS，弹性部署。
      <br/>
      对设备开发者而言，HarmonyOS采用了组件化的设计方案，可根据设备的资源能力和业务特征灵活裁剪，满足不同形态终端设备对操作系统的要求。
     </li>
    </ul>
    <h3>
     <a id="_14">
     </a>
     一、系统架构
    </h3>
    <p>
     HarmonyOS整体遵从分层设计依次为：
     <code>
      内核层
     </code>
     、
     <code>
      系统服务层
     </code>
     、
     <code>
      框架层
     </code>
     、
     <code>
      应用层
     </code>
     。
     <br/>
     HarmonyOS系统功能上按照
     <code>
      系统 &gt; 子系统 &gt; 功能/模块
     </code>
     逐级展开，在多设备部署场景下，支持根据实际需求裁剪某些非必要的子系统或功能/模块。
    </p>
    <p>
     <img alt="系统架构" src="https://i-blog.csdnimg.cn/blog_migrate/ba5909e17990da45d5cb998aeade818b.png"/>
    </p>
    <h4>
     <a id="11__23">
     </a>
     1.1 内核层
    </h4>
    <p>
     HarmonyOS采用多内核设计，支持针对不同资源受限设备选用适合的OS内核。
    </p>
    <p>
     <img alt="内核层" src="https://i-blog.csdnimg.cn/blog_migrate/b3fee51f4fdd47e2544fcb43c850cb37.png"/>
    </p>
    <ul>
     <li>
      内核抽象层（KAL，Kernel Abstract Layer）
      <br/>
      通过屏蔽多内核差异，对上层提供基础的内核能力，包括进程/线程管理、内存管理、文件系统、网络管理和外设管理等；
     </li>
     <li>
      硬件驱动框架（HDF）
      <br/>
      硬件驱动框架（HDF）是HarmonyOS硬件生态开放的基础，
      <code>
       提供统一外设访问能力和驱动开发、管理框架
      </code>
      。
     </li>
    </ul>
    <h4>
     <a id="12__35">
     </a>
     1.2 系统服务层
    </h4>
    <p>
     系统服务层是HarmonyOS的核心能力集合，系统服务层通过框架层对应用程序提供服务。
     <br/>
     系统服务层根据不同设备形态的部署环境，基础软件服务子系统集、增强软件服务子系统集、硬件服务子系统集内部可以
     <code>
      按子系统粒度裁剪
     </code>
     ，
     <code>
      每个子系统内部又可以按功能粒度裁剪
     </code>
     （HarmonyOS系统功能上按照
     <code>
      系统 &gt; 子系统 &gt; 功能/模块
     </code>
     逐级展开）。
    </p>
    <p>
     <img alt="系统服务层" src="https://i-blog.csdnimg.cn/blog_migrate/b75b9fbd6e69e6c75120a176ffe74e39.png"/>
    </p>
    <ul>
     <li>
      系统基本能力子系统集：
      <br/>
      系统基本能力子系统集为分布式应用在HarmonyOS多设备上的运行、调度、迁移等操作提供了基础能力。由
      <code>
       分布式软总线
      </code>
      、
      <code>
       分布式数据管理
      </code>
      、
      <code>
       分布式任务调度
      </code>
      、
      <code>
       方舟多语言运行时
      </code>
      、
      <code>
       公共基础库
      </code>
      、
      <code>
       多模输入
      </code>
      、
      <code>
       图形
      </code>
      、
      <code>
       安全
      </code>
      、
      <code>
       AI等子系统
      </code>
      组成。
      <br/>
      <code>
       方舟运行时
      </code>
      提供了
      <code>
       C/C++/JS多语言运行时
      </code>
      和
      <code>
       基础的系统类库
      </code>
      ，也为使用方舟编译器静态化的
      <code>
       Java程序
      </code>
      提供运行时。
     </li>
     <li>
      基础软件服务子系统集：
      <br/>
      基础软件服务子系统集为HarmonyOS提供
      <code>
       公共的
      </code>
      、
      <code>
       通用的
      </code>
      软件服务。由
      <code>
       事件通知
      </code>
      、
      <code>
       电话
      </code>
      、
      <code>
       多媒体
      </code>
      、
      <code>
       DFX（Design For X）
      </code>
      、
      <code>
       MSDP&amp;DV
      </code>
      等子系统组成。
     </li>
     <li>
      增强软件服务子系统集：
      <br/>
      为HarmonyOS提供
      <code>
       针对不同设备的
      </code>
      、
      <code>
       差异化的能力
      </code>
      增强型软件服务。由智慧屏专有业务、穿戴专有业务、IoT专有业务等子系统组成。
     </li>
     <li>
      硬件服务子系统集：
      <br/>
      为HarmonyOS提供硬件服务。由位置服务、生物特征识别、穿戴专有硬件服务、IoT专有硬件服务等子系统组成。
     </li>
    </ul>
    <h4>
     <a id="13__54">
     </a>
     1.3 框架层
    </h4>
    <p>
     框架层为HarmonyOS应用开发提供了
     <code>
      Java/C/C++/JS
     </code>
     等多语言的
     <code>
      用户程序框架
     </code>
     、
     <code>
      Ability框架
     </code>
     、
     <code>
      两种UI框架
     </code>
     ，以及各种软硬件服务对外开放的多语言框架API（根据系统的组件化裁剪程度，HarmonyOS设备支持的API也会有所不同）。
    </p>
    <p>
     <img alt="框架层" src="https://i-blog.csdnimg.cn/blog_migrate/efa263c6bb9b2e502c8e1ba4639e6ef3.png"/>
    </p>
    <ul>
     <li>
      用户程序框架：
      <br/>
      为HarmonyOS应用开发提供了Java/C/C++/JS等多语言的程序框架。
     </li>
     <li>
      UI框架：
      <br/>
      包括适用于Java语言的
      <code>
       Java UI框架
      </code>
      、适用于JS语言的
      <code>
       JS UI框架
      </code>
      ，以及各种软硬件服务对外开放的多语言框架API
     </li>
     <li>
      Ability框架：
      <br/>
      HarmonyOS支持应用以
      <code>
       Ability为单位
      </code>
      进行
      <code>
       部署
      </code>
      ，一个应用可以具备多种能力（即
      <code>
       可以包含多个Ability
      </code>
      ）。
      <br/>
      <code>
       Ability
      </code>
      可以分为
      <code>
       FA（Feature Ability）
      </code>
      和
      <code>
       PA（Particle Ability）
      </code>
      两种类型。
      <br/>
      <code>
       FA（Feature Ability）
      </code>
      包含一组显示页面；
      <br/>
      <code>
       PA（Particle Ability）
      </code>
      用于提供后台运行任务的能力，不包含显示页面。
     </li>
    </ul>
    <h4>
     <a id="14__72">
     </a>
     1.4 应用层
    </h4>
    <p>
     应用层包括系统应用和第三方非系统应用。
    </p>
    <p>
     <img alt="应用层" src="https://i-blog.csdnimg.cn/blog_migrate/240a6e5479800206252010bf401e1d81.png"/>
    </p>
    <p>
     HarmonyOS的应用由一个或多个
     <code>
      FA（Feature Ability）
     </code>
     或
     <code>
      PA（Particle Ability）
     </code>
     组成。
    </p>
    <ul>
     <li>
      <code>
       FA（Feature Ability）
      </code>
      有UI界面，提供与用户交互的能力；
     </li>
     <li>
      <code>
       PA（Particle Ability）
      </code>
      无UI界面，提供后台运行任务的能力以及统一的数据访问抽象。
     </li>
    </ul>
    <p>
     <code>
      FA（Feature Ability）
     </code>
     在进行用户交互时所需的后台数据访问也需要由对应的
     <code>
      PA（Particle Ability）
     </code>
     提供支撑。基于 FA/PA 开发的应用，能够实现特定的业务功能，支持跨设备调度与分发，为用户提供一致、高效的应用体验。
    </p>
    <p>
     <img alt="HarmonyOS应用程序" src="https://i-blog.csdnimg.cn/blog_migrate/45f0c862b090b26c1f980f2dcc728785.png"/>
    </p>
    <h3>
     <a id="_89">
     </a>
     二、开发语言
    </h3>
    <p>
     HarmonyOS提供了支持多种开发语言的API，供开发者进行应用开发。
     <br/>
     支持的开发语言包括
     <code>
      Java
     </code>
     、
     <code>
      JS（JavaScript）
     </code>
     、
     <code>
      XML（Extensible Markup Language）
     </code>
     、
     <code>
      C/C++
     </code>
     、
     <code>
      CSS（Cascading Style Sheets）
     </code>
     和
     <code>
      HML（HarmonyOS Markup Language）
     </code>
     。
    </p>
    <p>
     <img alt="多种语言开发环境" src="https://i-blog.csdnimg.cn/blog_migrate/0f1b35b136e6810b00156ddb2ad86328.png"/>
    </p>
