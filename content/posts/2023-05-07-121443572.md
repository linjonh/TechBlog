    <h4>
     <a id="_0">
     </a>
     嵌入式系统的定义和体系结构
    </h4>
    <h5>
     <a id="_2">
     </a>
     嵌入式系统的定义
    </h5>
    <ul>
     <li>
      IEEE定义：嵌入式系统是用于控制、监视或者辅助操作机器和设备的装置。（是软件和硬件的综合体，还可以涵盖机电等附属装置）
     </li>
     <li>
      以
      <u>
       应用
      </u>
      为中心，以计算机技术为基础，
      <u>
       软硬件可剪裁
      </u>
      ，功能、可靠性、成本、体积、功耗严格要求的
      <u>
       专用
      </u>
      计算机系统。
     </li>
     <li>
      以
      <u>
       嵌入式应用
      </u>
      为目的的计算机系统（系统及、板级、片级）。
     </li>
     <li>
      <u>
       嵌入
      </u>
      到
      <u>
       对象体系
      </u>
      内部的
      <u>
       专用计算机系统。
      </u>
     </li>
     <li>
      广义：带有微处理器的专用软硬件系统都可以称为嵌入式系统
     </li>
     <li>
      狭义：
      <u>
       具有自己操作系统，具有特定功能，用于滕鼎场合的嵌入式体统。
      </u>
     </li>
    </ul>
    <h5>
     <a id="_10">
     </a>
     实时系统
    </h5>
    <p>
     定义：实时系统RTOS需要在规定时间内完成任务，并进行确定性预测
     <br/>
     <strong>
      正确性分为：
     </strong>
    </p>
    <ul>
     <li>
      逻辑正确性
     </li>
     <li>
      时间正确性
      <br/>
      <strong>
       实行性精度分为：
      </strong>
     </li>
     <li>
      硬实时：时间精度高
     </li>
     <li>
      软实时：时间精度要求不太高，没有满足时间约束并不会导致灾难性的后果
     </li>
     <li>
      区别：时间精度的区别
     </li>
    </ul>
    <h5>
     <a id="_20">
     </a>
     嵌入式系统简介
    </h5>
    <p>
     <strong>
      嵌入式系统的重要特征：
     </strong>
    </p>
    <ul>
     <li>
      系统精简
     </li>
     <li>
      系统内核小
     </li>
     <li>
      高实时性0S
     </li>
     <li>
      专用性高
     </li>
     <li>
      嵌入式软件开发走向标准化
     </li>
     <li>
      嵌入式系统需要开发工具与环境
     </li>
    </ul>
    <p>
     <strong>
      嵌入式系统的组成：
     </strong>
     <br/>
     软件子系统
    </p>
    <ul>
     <li>
      操作系统（驱动程序）
     </li>
     <li>
      应用软件
     </li>
    </ul>
    <p>
     硬件子系统
    </p>
    <ul>
     <li>
      存储器
     </li>
     <li>
      处理器
     </li>
     <li>
      输入输出接口
     </li>
     <li>
      外设
     </li>
    </ul>
    <h5>
     <a id="_41">
     </a>
     嵌入式处理器
    </h5>
    <p>
     <strong>
      嵌入式系统的核心是微处理器
     </strong>
    </p>
    <p>
     <strong>
      嵌入式微处理器的特点：
     </strong>
    </p>
    <ul>
     <li>
      对实时多任务有很强的支持能力
     </li>
     <li>
      具有功能很强的存储区保护功能
     </li>
     <li>
      可扩展的处理器结构
     </li>
     <li>
      嵌入式未处理器功耗很低
     </li>
    </ul>
    <p>
     <strong>
      嵌入式处理器分类：
     </strong>
    </p>
    <ul>
     <li>
      MCU嵌入式微控制器：单片机，单片化，体积小，功耗、成本低
     </li>
     <li>
      DSP嵌入式DSP处理器：用于信号处理
     </li>
     <li>
      MPU嵌入式处理器：由通用处理器演化而来，只保留了嵌入式相关硬件
     </li>
     <li>
      SoC嵌入式片上系统：嵌入式片上系统，一种电路系统，将功能区只做在一个芯片上
     </li>
    </ul>
    <h5>
     <a id="RTOS32CPU_58">
     </a>
     RTOS（实时操作系统）是32位嵌入式CPU的软件基础
    </h5>
    <p>
     <strong>
      RTOS内核 提供CPU的管理：
     </strong>
     硬件初始化，MMU(存储器管理单元)，定时器，中断
    </p>
    <p>
     <strong>
      VxWorks所具有的显著特点是：
     </strong>
    </p>
    <ul>
     <li>
      可靠性、实时性和可裁剪性。
     </li>
     <li>
      它支持多种处理器，如x86、i960、Sun Sparc、Motorola MC68xxx、MIPS 、POWER PC等等。
     </li>
    </ul>
    <p>
     WindowsCE嵌入但不够实时，属于软实时操作系统，
    </p>
    <p>
     <strong>
      嵌入式系统的软件基础是RTOS，其主要功能为：
     </strong>
    </p>
    <ul>
     <li>
      处理器管理
     </li>
     <li>
      内存管理
     </li>
     <li>
      设备管理
     </li>
     <li>
      编程接口
     </li>
    </ul>
    <p>
     μCOS：微控制器操作系统的1
    </p>
    <p>
     <strong>
      嵌入式系统的优势：和PC，单片机比的优势
     </strong>
    </p>
    <ul>
     <li>
      性能:嵌入式系统的处理性能远大于单片机，与PC机相似但体积更小
     </li>
     <li>
      实时性：RTOS强实时性
     </li>
     <li>
      人机交互：支持图形用户界面，与PC相似
     </li>
     <li>
      系统升级：可快速升级和更新
     </li>
    </ul>
    <p>
     <strong>
      嵌入式系统≠单片机系统
     </strong>
    </p>

     <thead>
      <tr>
       <th>
       </th>

      </tr>
      <tr>

       </td>

       </td>

       </td>
      </tr>
      <tr>
       <td>

        基于平台软硬协同，系统设计核心是软件设计

    <ul>
     <li>
      目前嵌入式系统的主流是以32位嵌入式微处理器为核心的硬件设计和基于实时操作系统（RTOS）的软件设计
     </li>
     <li>
      单片机系统多为4位、8位、16位机，不适合运行操作系统，难以进行复杂的运算及处理功能
     </li>
     <li>
      嵌入式系统强调基于平台的设计、软硬件协同设计，单片机大多采用软硬件流水设计
     </li>
     <li>
      嵌入式系统设计的核心是软件设计（占70%左右的工作量），单片机系统软硬件设计所占比例基本相同
     </li>
    </ul>
    <p>
     <strong>
      嵌入式系统≠PC机
     </strong>
    </p>

    <ul>
     <li>
      嵌入式系统一般是专用系统，而PC是通用计算平台
     </li>
     <li>
      嵌入式系统的资源比PC少得多
     </li>
     <li>
      嵌入式系统软件故障带来的后果比PC机大得多
     </li>
     <li>
      嵌入式系统一般采用实时操作系统
     </li>
     <li>
      嵌入式系统大都有成本、功耗的要求
     </li>
     <li>

     </li>

    <h5>
     <a id="_114">
     </a>
     嵌入式系统的组成
    </h5>
    <p>
     硬件结构：处理器、存储器、输出输出接口
     <br/>
     软件结构：应用程序、操作系统
    </p>
    <p>
     嵌入式系统的软硬件框架：层次：功能层→软件层→中间层→硬件层
    </p>
    <h5>
     <a id="_120">
     </a>
     嵌入式系统简介
    </h5>
    <ul>
     <li>
      嵌入式外围设备
     </li>
     <li>
      分类
      <ul>
       <li>
        存储器类型：静态易失型存储器(RAM/SRAM)，动态存储器(DRAM)，非易失型存储器(ROM、EPROM、EEPROM、FLASH)。其中，FLASH(闪存)以可擦写次数多，存储速度快，容量大及价格便宜等有点在嵌入式领域得到广泛的应用。
       </li>
       <li>
        接口类型：目前存在的所有接口在嵌入式系统中都有其广泛的应用，但是以下几种接口的应用最为广泛，包括RS-232接口(串口)、IrDA(红外)、SPI(串行设备接口)、I2C、USB、Ethernet和普通并口。
       </li>
       <li>
        显示类型：CRT、LCD和触摸屏等外围显示设备。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     最小硬件系统
    </p>
    <p>
     <strong>
      嵌入式操作系统的特点
     </strong>
    </p>
    <ul>
     <li>
      可装卸性。
     </li>
     <li>
      强实时性。
     </li>
     <li>
      统一的接口。
     </li>
     <li>
      操作方便、简单、提供友好的图形GUI，图形界面，追求易学易用。
     </li>
     <li>
      提供强大的网络功能，支持TCP/IP协议及其它协议，提供TCP/UDP/IP/PPP协议支持及统一的MAC访问层接口，为各种移动计算设备预留接口
     </li>
     <li>
      强稳定性，弱交互性。
     </li>
     <li>
      固化代码。
     </li>
     <li>
      更好的硬件适应性，也就是良好的移植性。
     </li>
    </ul>
    <p>
     <strong>
      嵌入式外围设备：存储器、接口、显示设备等
     </strong>
    </p>
    <p>
     <strong>
      中间层→硬件抽象层，位于软件层和硬件层之间
     </strong>
     <br/>
     硬件层和软件层之间为中间层，也称为硬件抽象层（Hardware Abstract Layer，HAL）或板级支持包(Board Support Package，BSP)。
    </p>
    <ul>
     <li>
      <pre><code>硬件相关性
</code></pre>
     </li>
     <li>
      <pre><code>操作系统相关性
</code></pre>
     </li>
    </ul>
    <p>
     <strong>
      设备驱动程序功能：一组库函数，用来对硬件进行初始化和管理，并向上层软件提供良好的访问接口。
     </strong>
    </p>
    <ul>
     <li>
      硬件启动：在开机上电或系统重启的时候，对硬件进行初始化；
     </li>
     <li>
      硬件关闭：将硬件设置为关机状态；
     </li>
     <li>
      硬件停用：暂停使用这个硬件；
     </li>
     <li>
      硬件启用：重新启用这个硬件；
     </li>
     <li>
      读操作：从硬件中读取数据；
     </li>
     <li>
      写操作：往硬件中写入数据。
     </li>
    </ul>
