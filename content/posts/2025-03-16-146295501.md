     正则表达式小结

    <blockquote>
     <p>
      正则表达式是一种用于描述文本模式的特殊字符串，它由一系列字符和特殊字符组成，用于匹配和操作文本数据。下面是正则表达式的一些常见规则：
     </p>
    </blockquote>
    <ol>
     <li>
      <p>
       字符匹配：
      </p>
      <ul>
       <li>
        普通字符：正则表达式中的普通字符（字母、数字、符号）将按照字面意义进行匹配。
       </li>
       <li>
        元字符：正则表达式中的一些特殊字符具有特殊含义，例如
        <code>
         ^
        </code>
        、
        <code>
         $
        </code>
        、
        <code>
         .
        </code>
        、
        <code>
         *
        </code>
        、
        <code>
         +
        </code>
        、
        <code>
         ?
        </code>
        等。
       </li>
      </ul>
     </li>
     <li>
      <p>
       字符类：
      </p>
      <ul>
       <li>
        方括号
        <code>
         []
        </code>
        ：用于定义一个字符类，匹配方括号中列出的任意一个字符。
       </li>
       <li>
        范围表示：可以在字符类中使用连字符
        <code>
         -
        </code>
        表示字符的范围。例如，
        <code>
         [0-9]
        </code>
        表示匹配任意一个数字字符。
       </li>
      </ul>
     </li>
     <li>
      <p>
       量词：
      </p>
      <ul>
       <li>
        <code>
         *
        </code>
        ：匹配前面的元素零次或多次。
       </li>
       <li>
        <code>
         +
        </code>
        ：匹配前面的元素一次或多次。
       </li>
       <li>
        <code>
         ?
        </code>
        ：匹配前面的元素零次或一次。
       </li>
       <li>
        <code>
         {n}
        </code>
        ：匹配前面的元素恰好n次。
       </li>
       <li>
        <code>
         {n,}
        </code>
        ：匹配前面的元素至少n次。
       </li>
       <li>
        <code>
         {n,m}
        </code>
        ：匹配前面的元素至少n次且不超过m次。
       </li>
      </ul>
     </li>
     <li>
      <p>
       特殊字符：
      </p>
      <ul>
       <li>
        <code>
         .
        </code>
        ：匹配任意单个字符，除了换行符。
       </li>
       <li>
        <code>
         ^
        </code>
        ：匹配输入字符串的开始位置。
       </li>
       <li>
        <code>
         $
        </code>
        ：匹配输入字符串的结束位置。
       </li>
       <li>
        <code>
         \
        </code>
        ：用于转义下一个字符，使其失去特殊含义。
       </li>
      </ul>
     </li>
     <li>
      <p>
       逻辑操作：
      </p>
      <ul>
       <li>
        <code>
         |
        </code>
        ：用于分隔多个模式，匹配任意一个模式。
       </li>
       <li>
        <code>
         ()
        </code>
        ：用于创建分组，可以对其中的模式进行分组和操作。
       </li>
      </ul>
     </li>
     <li>
      <p>
       边界匹配：
      </p>
      <ul>
       <li>
        <code>
         \b
        </code>
        ：匹配单词的边界，即单词的开始或结束位置。
       </li>
       <li>
        <code>
         \B
        </code>
        ：匹配非单词边界的位置。
       </li>
      </ul>
     </li>
     <li>
      <p>
       特殊字符简写：
      </p>
      <ul>
       <li>
        <code>
         \d
        </code>
        或者
        <code>
         [:digit:]
        </code>
        ：匹配任意一个数字字符，相当于
        <code>
         [0-9]
        </code>
        。
       </li>
       <li>
        <code>
         \D
        </code>
        ：匹配任意一个非数字字符，相当于
        <code>
         [^0-9]
        </code>
        。
       </li>
       <li>
        <code>
         \w
        </code>
        ：匹配任意一个单词字符（字母、数字、下划线），相当于
        <code>
         [A-Za-z0-9_]
        </code>
        。
       </li>
       <li>
        <code>
         \W
        </code>
        ：匹配任意一个非单词字符，相当于
        <code>
         [^A-Za-z0-9_]
        </code>
        。
       </li>
       <li>
        <code>
         \s
        </code>
        ：匹配任意一个空白字符（空格、制表符、换行符等）。
       </li>
       <li>
        <code>
         \S
        </code>
        ：匹配任意一个非空白字符。
       </li>
      </ul>
     </li>
     <li>
      <p>
       反向引用：
      </p>
      <ul>
       <li>
        <code>
         ( )
        </code>
        ：用于创建捕获组，并将匹配的内容存储在临时变量中。
       </li>
       <li>
        <code>
         \n
        </code>
        ：用于引用前面的捕获组中的内容（n表示捕获组的编号）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       前后查找：
      </p>
      <ul>
       <li>
        <code>
         (?=...)
        </code>
        ：正向前查找，匹配紧跟在指定模式之前的位置，但不包括该模式。
       </li>
       <li>
        <code>
         (?&lt;=...)
        </code>
        ：正向后查找，匹配紧跟在指定模式之后的位置，但不包括该模式。
       </li>
       <li>
        <code>
         (?!...)
        </code>
        ：负向前查找，匹配不紧跟在指定模式之前的位置。
       </li>
       <li>
        <code>
         (?&lt;!...)
        </code>
        ：负向后查找，匹配不紧跟在指定模式之后的位置。
       </li>
      </ul>
     </li>
     <li>
      <p>
       贪婪与非贪婪匹配：
      </p>
      <ul>
       <li>
        <code>
         *
        </code>
        、
        <code>
         +
        </code>
        、
        <code>
         ?
        </code>
        、
        <code>
         {}
        </code>
        默认是贪婪匹配，尽可能匹配更多的字符。
       </li>
       <li>
        <code>
         *?
        </code>
        、
        <code>
         +?
        </code>
        、
        <code>
         ??
        </code>
        、
        <code>
         {}?
        </code>
        表示非贪婪匹配，尽可能匹配更少的字符。
       </li>
      </ul>
     </li>
    </ol>
    <blockquote>
     <p>
      这些是正则表达式的一些常见规则和特性。不同的正则表达式引擎和工具可能支持不同的规则和扩展，因此在具体使用时应查阅相关文档以获得更准确的信息和语法。
     </p>
    </blockquote>
    <hr/>
    <blockquote>
     <p>
      正则表达式与命令配合使用：
     </p>
    </blockquote>
    <p>
     下面是一些在Linux中使用正则表达式的常见命令和示例：
    </p>
    <ol>
     <li>
      grep命令：
      <code>
       grep
      </code>
      命令用于在文件中搜索匹配正则表达式的文本行。
     </li>
    </ol>
    <p>
     示例：
    </p>
    <pre><code class="prism language-perl">grep <span class="token string">'pattern'</span> file<span class="token operator">.</span>txt
</code></pre>
    <p>
     这将在
     <code>
      file.txt
     </code>
     文件中查找匹配
     <code>
      pattern
     </code>
     的文本行。
    </p>
    <ol start="2">
     <li>
      sed命令：
      <code>
       sed
      </code>
      命令用于对文本进行模式匹配和替换操作。
     </li>
    </ol>
    <pre><code class="prism language-perl">sed <span class="token string">'s/pattern/replacement/'</span> file<span class="token operator">.</span>txt
</code></pre>
    <p>
     这将在
     <code>
      file.txt
     </code>
     文件中查找匹配
     <code>
      pattern
     </code>
     的文本，并将其替换为
     <code>
      replacement
     </code>
     。
    </p>
    <ol start="3">
     <li>
      awk命令：
      <code>
       awk
      </code>
      命令是一种强大的文本处理工具，支持使用正则表达式进行模式匹配和操作。
     </li>
    </ol>
    <pre><code class="prism language-perl">awk <span class="token string">'/pattern/ { print $0 }'</span> file<span class="token operator">.</span>txt
</code></pre>
    <p>
     这将在
     <code>
      file.txt
     </code>
     文件中查找匹配
     <code>
      pattern
     </code>
     的文本行，并将其打印出来。
    </p>
    <blockquote>
     <p>
      这只是一些常见的命令和示例，正则表达式在Linux中的应用非常广泛，可以根据具体的需求和工具使用文档来了解更多用法和选项。
     </p>
    </blockquote>
