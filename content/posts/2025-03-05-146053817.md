    <h3>
     <a id="_2">
     </a>
     题目：
    </h3>
    <p>
     584.寻找用户推荐人
     <br/>
     表: Customer
    </p>
    <p>
     ±------------±--------+
     <br/>
     | Column Name | Type |
     <br/>
     ±------------±--------+
     <br/>
     | id | int |
     <br/>
     | name | varchar |
     <br/>
     | referee_id | int |
     <br/>
     ±------------±--------+
     <br/>
     在 SQL 中，id 是该表的主键列。
     <br/>
     该表的每一行表示一个客户的 id、姓名以及推荐他们的客户的 id。
     <br/>
     找出那些 没有被 id = 2 的客户 推荐 的客户的姓名。
    </p>
    <h3>
     <a id="_18">
     </a>
     思路：
    </h3>
    <p>
     首先需要找的是客户姓名，那么select寻找的就是name，其次给出符合的条件为”没有被id = 2 的客户推荐“，那么就存在两种可能：
     <br/>
     a)当前客户没有被其他客户推荐
     <br/>
     b)推荐当前客户的不是id = 2的客户
     <br/>
     这两种可能是或的关系，只要一个为真，那么这个式子结果就为真
     <br/>
     所以用or逻辑运算符将两种可能组合起来，再使用where关键字在表中过滤不符合条件的表格行，确保最后结果是对的。
    </p>
    <h3>
     <a id="_26">
     </a>
     代码：
    </h3>
    <pre><code class="prism language-sql"><span class="token keyword">select</span> name 
<span class="token keyword">from</span> Customer 
<span class="token keyword">where</span> referee_id <span class="token operator">!=</span> <span class="token number">2</span> <span class="token operator">or</span> referee_id <span class="token operator">is</span> <span class="token boolean">null</span>
</code></pre>
