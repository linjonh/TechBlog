    <h2>
     <a id="_1">
     </a>
     静态库
    </h2>
    <p>
     <strong>
      核心特性：
     </strong>
    </p>
    <ul>
     <li>
      链接时机：在编译时将库代码直接嵌入到可执行文件中。
     </li>
     <li>
      文件格式：.a（Linux）、.lib（Windows）。
     </li>
     <li>
      内存占用：每个使用该库的可执行文件都会包含一份完整的库代码副本。
     </li>
    </ul>
    <p>
     <strong>
      创建静态库（.a文件）
     </strong>
    </p>
    <ol>
     <li>
      编写源代码：首先，编写你需要的库函数的源代码文件，例如mylib.c
     </li>
     <li>
      编译为目标文件：使用gcc/g++编译源码为目标文件（.o文件）
     </li>
    </ol>
    <pre><code class="prism language-bash">g++ <span class="token parameter variable">-c</span> mylib.cpp <span class="token parameter variable">-o</span> mylib.o
</code></pre>
    <ol start="3">
     <li>
      创建静态库：使用
      <code>
       ar
      </code>
      工具将目标文件打包为静态库（.a文件）
     </li>
    </ol>
    <pre><code class="prism language-bash">ar rcs libmylib.a mylib.o
</code></pre>
    <ul>
     <li>
      rcs：替换旧文件（r）、创建新库（c）、生成索引（s）
     </li>
    </ul>
    <h2>
     <a id="_24">
     </a>
     动态库
    </h2>
    <p>
     <strong>
      核心特性：
     </strong>
    </p>
    <ul>
     <li>
      链接时机：在运行时由操作系统动态加载到内存（映射到地址空间的共享区）。
     </li>
     <li>
      文件格式：.so（Linux）、.dll（Windows）。
     </li>
     <li>
      内存占用：多个进程可共享同一份库的内存实例。
     </li>
    </ul>
    <p>
     <strong>
      创建动态库（.so文件）
     </strong>
    </p>
    <ol>
     <li>
      编写源代码：首先编写库函数的源代码文件，例如mylib.c
     </li>
     <li>
      编译并生成位置无关代码(PIC)：在编译为目标文件时，需要添加
      <code>
       -fPIC
      </code>
      选项以生成位置无关代码
     </li>
    </ol>
    <pre><code class="prism language-bash">g++ <span class="token parameter variable">-c</span> <span class="token parameter variable">-fPIC</span> mylib.cpp <span class="token parameter variable">-o</span> mylib.o
</code></pre>
    <ol start="3">
     <li>
      创建动态库：使用gcc/g++的
      <code>
       -shared
      </code>
      选项将目标文件链接为动态库（.so文件）
     </li>
    </ol>
    <pre><code class="prism language-bash">g++ <span class="token parameter variable">-shared</span> mylib.o <span class="token parameter variable">-o</span> libmylib.so
</code></pre>
    <h2>
     <a id="_43">
     </a>
     库的链接
    </h2>
    <p>
     在gcc/g++的编译和链接过程中，-I（大写i）、-L、-l（小写L）是三个常用的选项，它们分别用于
     <strong>
      指定头文件的搜索路径
     </strong>
     、
     <strong>
      库文件的搜索路径
     </strong>
     以及
     <strong>
      要链接的库的名称
     </strong>
     。
    </p>
    <pre><code class="prism language-bash">gcc <span class="token parameter variable">-I</span> /home/user/include <span class="token parameter variable">-L</span> /home/user/lib <span class="token parameter variable">-lmylib</span> my_program.c <span class="token parameter variable">-o</span> my_program


-I/home/user/include 告诉编译器在 /home/user/include目录下查找头文件。
-L/home/user/lib 告诉链接器在 /home/user/lib目录下查找库文件。
-Imylib告诉链接器链接名为mylib的库 （即查找libmylib.a或libmylib.so文件）。
</code></pre>
    <p>
     【
     <strong>
      注
     </strong>
     】：
    </p>
    <ul>
     <li>
      库就是功能性代码块的包装，也是包含：头文件 和 源文件（所谓的：库文件），头文件和库文件一般都是存储在各自的目录下，所以需要分别链接。
     </li>
     <li>
      链接库名时要去掉前后缀，前缀lib，后缀 .a / .so。
     </li>
    </ul>
    <p>
     上述操作是让编译器知道了文件路径，但链接完成后的程序运行时动态库还需要加载，
     <strong>
      加载器还不知道所使用的第三方动态库的路径。
     </strong>
    </p>
    <p>
     <strong>
      五种方法解决：
     </strong>
    </p>
    <ol>
     <li>
      将动态库文件拷贝到系统默认的库路径 /lib64/ 或 /usr/lib64/；
     </li>
     <li>
      建立系统默认的库路径下的软链接指向第三方库；
     </li>
     <li>
      将库的路径添加进环境变量 LD_LIBRARY_PATH 中
     </li>
    </ol>
    <pre><code class="prism language-bash">//临时追加
<span class="token builtin class-name">export</span> <span class="token assign-left variable">LD_LIBRARY_PATH</span><span class="token operator">=</span>/path/to/your/lib:<span class="token variable">$LD_LIBRARY_PATH</span>
./your_program
</code></pre>
    <ol start="4">
     <li>
      编译时指定 rpath（硬编码路径到可执行文件）
     </li>
    </ol>
    <pre><code class="prism language-bash">g++ main.cpp -L./mylib <span class="token parameter variable">-lmylib</span> -Wl,-rpath<span class="token operator">=</span><span class="token string">'$ORIGIN/mylib'</span> <span class="token parameter variable">-o</span> main

//-Wl,-rpath<span class="token operator">=</span><span class="token punctuation">..</span>.：将 rpath 写入可执行文件，<span class="token variable">$ORIGIN</span> 表示可执行文件所在目录。
</code></pre>
    <ol start="5">
     <li>
      配置/etc/ld.so.conf.d/，ldconfig更新
      <br/>
      在/etc/ld.so.conf.d/路径下创建一个 .conf文件，把动态库所在的路径放到这个.conf文件中，然后ldconfig 重新加载一份即可。
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span>  /etc/ld.so.conf.d/
<span class="token function">touch</span>  myname.conf
<span class="token function">vim</span> myname.conf  //打开后把路径复制粘贴进去就行
</code></pre>
    <p>
     <strong>
      怎么简化链接库的命令呢？
     </strong>
    </p>
    <ol>
     <li>
      如果自己创建的头文件和库文件 分别放在系统中存储库的文件的目录中时，就不需要-I和-L了，仅需链接指定库文件即可（就像使用C语言库直接编译那样，还是需要指定库名称）。
     </li>
     <li>
      建立系统标准库目录下的软链接
      <br/>
      eg：
     </li>
    </ol>
    <pre><code class="prism language-c"><span class="token macro property"><span class="token directive-hash">#</span><span class="token directive keyword">include</span> <span class="token expression">“myinc<span class="token operator">/</span>mymath<span class="token punctuation">.</span>h”</span></span>
gcc main<span class="token punctuation">.</span>c <span class="token operator">-</span>o main <span class="token operator">-</span>lmymath

<span class="token comment">//在系统头文件和库文件的标准库目录下 分别创建指向自己创建的库的头文件和库文件的软链接</span>
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/773c264fdeb648959043bdd5acbab2e7.png"/>
    </p>
    <p>
     <strong>
      动态库在系统中加载之后，会被所有链接了该库的进程共享，这是如何被共享的？
     </strong>
    </p>
    <p>
     答：动态库从磁盘加载到物理内存，然后这一份动态库文件可以与所有链接该库的进程建立映射，映射到每个进程的进程地址空间的共享区中。
    </p>
