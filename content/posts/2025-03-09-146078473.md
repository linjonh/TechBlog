    <h2>

     </a>
     问题
    </h2>
    <p>
     在Postman里可成功执行的POST请求：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/56ae829ab145403abc8f53869ce7d99f.png">

      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8912508a2389492d8a2b52834bc6d7a6.png">
       <br/>
       因为cURL基本上算是行业标准，所以Postman默认选中cURL，支持切换不同的开发语言：
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a52c8a87196a4f098c6ae4deaf803ac7.png">
        <br/>
        点击上图右上角的复制按钮，得到cURL脚本。
       </img>
      </img>
     </img>
    </p>
    <p>
     Windows 11家庭版，打开Git Bash客户端，版本为：
     <br/>
     <code>
      git version 2.47.1.windows.1
     </code>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/240f4168dba8415a97a381c1a2fb8b64.png">
      <br/>
      执行上述cURL脚本异常：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7c45a4e8305349c8971ef2abfd4d447e.png">
       <br/>
       报错信息：
      </img>
     </img>
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span><span class="token string-property property">"code"</span><span class="token operator">:</span><span class="token number">400</span><span class="token punctuation">,</span><span class="token string-property property">"msg"</span><span class="token operator">:</span><span class="token string">"JSON parse error: Invalid UTF-8 start byte 0xb4; nested exception is com.fasterxml.jackson.core.JsonParseException: Invalid UTF-8 start byte 0xb4\n at [Source: (org.springframework.util.StreamUtils$NonClosingInputStream); line: 3, column: 18]"</span><span class="token punctuation">}</span>
</code></pre>
    <h3>
     <a id="_18">
     </a>
     分析
    </h3>
    <p>
     根据上面几行简短的报错信息，提取几个关键内容：
    </p>
    <ul>
     <li>
      <code>
       code=400
      </code>
      ：表明这是客户端异常，此处的客户端是Git Bash；
     </li>
     <li>
      JsonParseException和StreamUtils：表明服务端确实有接收到客户端提交的请求，要不然客户端也无法得知Jackson框架里的类JsonParseException，当然也无法得知Spring框架里的StreamUtils工具类；
     </li>
     <li>
      报错原因是非法JSON：
      <code>
       JSON parse error
      </code>
      ；
     </li>
     <li>
      编码异常：
      <code>
       Invalid UTF-8 start byte 0xb4
      </code>
      。
     </li>
    </ul>
    <p>
     分析起来，看起来头头是道，怎么解决问题呢？
    </p>

     <a id="_26">
     </a>

      一一尝试：

    </p>
    <ul>
     <li>
      很明确是小写双引号，不是大写双引号；
     </li>
     <li>
      将上述cURL脚本使用txt文件保存，并保存为不带
      <a href="https://blog.csdn.net/lonelymanontheway/article/details/117649518">
       BOM
      </a>
      的格式，没有解决问题；

    <h3>
     <a id="_37">
     </a>
     方法一
    </h3>
    <p>
     反复尝试+试错，浪费不少时间，ChatGPT终于给出一个可行的解决方案：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/76af7efb5c7542b4a6c15eecf0f16ac0.png"/>
     <br/>
     也就是在Git Bash执行的目录下，新增一个
     <code>
      payload.json
     </code>
     文件，内容就是POST请求的requestBody：
    </p>
    <pre><code class="prism language-json"><span class="token punctuation">{<!-- --></span>
    <span class="token string-property property">"retriever_type"</span><span class="token operator">:</span> <span class="token string">"TEXT"</span><span class="token punctuation">,</span>
    <span class="token string-property property">"question"</span><span class="token operator">:</span> <span class="token string">"大模型技术如何帮助数据中心实现高效减碳？"</span><span class="token punctuation">,</span>
    <span class="token string-property property">"min_score"</span><span class="token operator">:</span> <span class="token string">"0.2"</span><span class="token punctuation">,</span>
    <span class="token string-property property">"max_results"</span><span class="token operator">:</span> <span class="token string">"10"</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     然后将命令：
    </p>
    <pre><code class="prism language-shell"><span class="token parameter variable">--data</span> <span class="token string">'{
    "retriever_type": "TEXT",
    "question": "大模型技术如何帮助数据中心实现高效减碳？",
    "min_score": "0.2",
    "max_results": "10"
}'</span>
</code></pre>
    <p>
     修改为（其他部分不变）：
     <code>
      --data @payload.json
     </code>
    </p>
    <p>
     执行效果：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/dc6a83df26a6470c894102b98f2d4e4d.png"/>
    </p>
    <h3>
     <a id="_62">
     </a>
     方法二
    </h3>
    <p>
     为了执行一个在Postman可以成功执行的cURL脚本，我需要另存为一个JSON文件，感觉非常反人类。于是继续追问ChatGPT，给出一个不使用文件的方法：
    </p>
    <pre><code class="prism language-shell"><span class="token builtin class-name">printf</span> <span class="token string">'%s'</span> <span class="token string">'{
  "retriever_type": "TEXT",
  "question": "大模型技术如何帮助数据中心实现高效减碳？",
  "min_score": "0.2",
  "max_results": "10"
}'</span> <span class="token operator">|</span> <span class="token function">curl</span> <span class="token parameter variable">--location</span> <span class="token string">'http://api.test.tesla.com/rag_online/rag/retrieval'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--header</span> <span class="token string">'tesla-token: 1111222233334444'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--header</span> <span class="token string">'Content-Type: application/json'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--data</span> @-
</code></pre>
    <p>
     和上面的解决方法非常类似。
    </p>

    <h2>
     <a id="_100">
     </a>
     解决方案
    </h2>
    <h3>
     <a id="Git_Bash_101">
     </a>
     Git Bash
    </h3>
    <p>
     如果是GET请求，直接可使用Git Bash。但是对于POST请求，如果坚持要使用Git Bash客户端，有两种方法：
    </p>
    <ul>
     <li>
      使用文件
     </li>
    </ul>
    <pre><code class="prism language-shell"><span class="token function">curl</span> <span class="token parameter variable">--location</span> <span class="token string">'http://api.test.tesla.com/rag_online/rag/retrieval'</span> <span class="token punctuation">\</span>
<span class="token parameter variable">--header</span> <span class="token string">'tesla-token: 1111222233334444'</span> <span class="token punctuation">\</span>
<span class="token parameter variable">--header</span> <span class="token string">'Content-Type: application/json'</span> <span class="token punctuation">\</span>
<span class="token parameter variable">--data</span> @payload.json
</code></pre>
    <ul>
     <li>
      使用管道符
     </li>
    </ul>
    <pre><code class="prism language-shell"><span class="token builtin class-name">printf</span> <span class="token string">'%s'</span> <span class="token string">'{
  "retriever_type": "TEXT",
  "question": "大模型技术如何帮助数据中心实现高效减碳？",
  "min_score": "0.2",
  "max_results": "10"
}'</span> <span class="token operator">|</span> <span class="token function">curl</span> <span class="token parameter variable">--location</span> <span class="token string">'http://api.test.tesla.com/rag_online/rag/retrieval'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--header</span> <span class="token string">'ecmas-token: 1111222233334444'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--header</span> <span class="token string">'Content-Type: application/json'</span> <span class="token punctuation">\</span>
  <span class="token parameter variable">--data</span> @-
</code></pre>
    <p>

    <h3>
     <a id="WSL_125">
     </a>
     WSL
    </h3>
    <p>
     Windows 8不知道什么时候开始支持WSL，另外貌似有不少
     <a href="https://github.com/microsoft/WSL/issues/3115">
      问题
     </a>
     ，Windows 10/11好像也不是一开始就支持WSL的。
    </p>
    <p>
     总而言之，如果可以的话，建议升级Windows版本，安装WSL，比CMD、PowerShell功能强大：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/c8993323ffbf454b91f6bf28d88e6809.png"/>
     <br/>
     使用WSL可执行多行cURL脚本。当然，WSL是一个Ubuntu系统，默认自带cURL命令。
    </p>
    <p>
     关于WSL，参考我写的另一篇
     <a href="https://blog.csdn.net/lonelymanontheway/article/details/128423060">
      Windows 10/11安装使用WSL
     </a>
     。
