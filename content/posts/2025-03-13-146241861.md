     【报错问题】在visual studio 终端使用npm -v后报错禁止运行脚本怎么处理

  <div class="article_content clearfix" id="article_content">

    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/07f22aeaea1043b483efa70601fd247c.png">
      <br/>
      PS C:\Users\10449&gt; npm -v
      <br/>
      npm : 无法加载文件 D:\node\npm.ps1，因为在此系统上禁止运行脚本。有关详细信息，请参阅 https:/go.microsoft.com/fwlink/?LinkID=135170 中的 about_Execution_Policies。
      <br/>
      所在位置 行:1 字符: 1
     </img>
    </p>
    <ul>
     <li>
      npm -v
     </li>
     <li>
      <pre><code>  + CategoryInfo          : SecurityError: (:) []，PSSecurityException
  + FullyQualifiedErrorId : UnauthorizedAccess
</code></pre>
     </li>
    </ul>
    <p>
     这个错误是因为 PowerShell 的执行策略（Execution Policy）限制了脚本的运行。默认情况下，PowerShell 的执行策略可能是
     <code>
      Restricted
     </code>
     ，这会阻止任何脚本的运行，包括
     <code>
      npm
     </code>
     命令。
    </p>
    <p>
     以下是解决这个问题的步骤：
    </p>
    <hr/>
    <h4>
     <a id="1__14">
     </a>
     <strong>
      1. 检查当前的执行策略
     </strong>
    </h4>
    <ol>
     <li>
      打开 PowerShell（以管理员身份运行）。
     </li>
     <li>
      运行以下命令检查当前的执行策略：
      <pre><code class="prism language-powershell"><span class="token function">Get-ExecutionPolicy</span>
</code></pre>
      <ul>
       <li>
        常见的执行策略包括：
        <ul>
         <li>
          <code>
           Restricted
          </code>
          ：禁止运行任何脚本。
         </li>
         <li>
          <code>
           AllSigned
          </code>
          ：只允许运行经过数字签名的脚本。
         </li>
         <li>
          <code>
           RemoteSigned
          </code>
          ：允许运行本地脚本，但远程脚本必须经过数字签名。
         </li>
         <li>
          <code>
           Unrestricted
          </code>
          ：允许运行所有脚本。
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="2__28">
     </a>
     <strong>
      2. 更改执行策略
     </strong>
    </h4>
    <p>
     为了允许运行
     <code>
      npm
     </code>
     命令，你可以将执行策略更改为
     <code>
      RemoteSigned
     </code>
     或
     <code>
      Unrestricted
     </code>
     。
    </p>
    <h5>
     <a id="_1_31">
     </a>
     <strong>
      方法 1：临时更改执行策略
     </strong>
    </h5>
    <ol>
     <li>
      在 PowerShell 中运行以下命令：
      <pre><code class="prism language-powershell"><span class="token function">Set-ExecutionPolicy</span> RemoteSigned <span class="token operator">-</span>Scope <span class="token keyword">Process</span>
</code></pre>
      <ul>
       <li>
        这将仅在当前 PowerShell 会话中更改执行策略，关闭 PowerShell 后会恢复默认设置。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_2_38">
     </a>
     <strong>
      方法 2：永久更改执行策略
     </strong>
    </h5>
    <ol>
     <li>
      以管理员身份运行 PowerShell。
     </li>
     <li>
      运行以下命令：
      <pre><code class="prism language-powershell"><span class="token function">Set-ExecutionPolicy</span> RemoteSigned
</code></pre>
      <ul>
       <li>
        这将永久更改执行策略。
       </li>
       <li>
        如果系统提示确认，输入
        <code>
         Y
        </code>
        并按回车。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="3__49">
     </a>
     <strong>
      3. 验证更改
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       运行以下命令检查执行策略是否已更改：
      </p>
      <pre><code class="prism language-powershell"><span class="token function">Get-ExecutionPolicy</span>
</code></pre>
      <ul>
       <li>
        如果显示
        <code>
         RemoteSigned
        </code>
        或
        <code>
         Unrestricted
        </code>
        ，说明更改成功。
       </li>
      </ul>
     </li>
     <li>
      <p>
       再次运行
       <code>
        npm -v
       </code>
       检查是否正常工作：
      </p>
      <pre><code class="prism language-powershell">npm <span class="token operator">-</span>v
</code></pre>
      <ul>
       <li>
        如果显示 npm 的版本号，说明问题已解决。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="4__64">
     </a>
     <strong>
      4. 如果问题仍然存在
     </strong>
    </h4>
    <p>
     如果更改执行策略后问题仍然存在，可以尝试以下方法：
    </p>
    <h5>
     <a id="_1CMD_67">
     </a>
     <strong>
      方法 1：使用命令提示符（CMD）
     </strong>
    </h5>
    <ol>
     <li>
      打开命令提示符（CMD），而不是 PowerShell。
     </li>
     <li>
      运行以下命令：
      <pre><code class="prism language-cmd">npm -v
</code></pre>
      <ul>
       <li>
        在 CMD 中，执行策略不会影响
        <code>
         npm
        </code>
        命令。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_2_Nodejs__75">
     </a>
     <strong>
      方法 2：修复 Node.js 安装
     </strong>
    </h5>
    <ol>
     <li>
      重新安装 Node.js：
      <ul>
       <li>
        访问
        <a href="https://nodejs.org/" rel="nofollow">
         Node.js 官网
        </a>
        下载最新版本。
       </li>
       <li>
        运行安装程序，确保选择
        <strong>
         Add to PATH
        </strong>
        选项。
       </li>
      </ul>
     </li>
     <li>
      重新打开 PowerShell 或 CMD，运行
      <code>
       npm -v
      </code>
      检查是否正常工作。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="5__83">
     </a>
     <strong>
      5. 恢复执行策略（可选）
     </strong>
    </h4>
    <p>
     如果你不希望永久更改执行策略，可以在完成操作后将其恢复为默认值：
    </p>
    <ol>
     <li>
      以管理员身份运行 PowerShell。
     </li>
     <li>
      运行以下命令：
      <pre><code class="prism language-powershell"><span class="token function">Set-ExecutionPolicy</span> Restricted
</code></pre>
     </li>
    </ol>
