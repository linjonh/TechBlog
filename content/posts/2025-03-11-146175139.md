     <li>

        <code>

        panic("error message")
       </code>
       主动抛出的错误，只要在
       <strong>

       </strong>
       的
       <code>
        defer
       </code>
       链中调用
       <code>
        recover
       </code>
       ，即可捕获并恢复。
      </p>

     </li>
     <li>
      <p>
       <strong>
        运行时错误（部分）
       </strong>
       <br/>
       如切片越界、类型断言失败、空指针解引用等运行时
       <code>
        panic
       </code>
       ，只要在
       <code>
        defer
       </code>
       中正确使用
       <code>
        recover
       </code>
       ，
       <strong>

       </strong>
       。但程序可能处于不可信状态，继续执行需谨慎。
      </p>
      <pre><code class="prism language-go"><span class="token keyword">defer</span> <span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>

       </code>
       可以捕获并处理。
      </p>

    <h4>
     <a id="recover_33">
     </a>
     <strong>
      不能
      <code>
       recover
      </code>
      的错误
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        其他goroutine未处理的
        <code>
         panic
        </code>
       </strong>
       <br/>
       每个goroutine需独立处理自己的
       <code>
        panic
       </code>
       。若某goroutine未在其内部
       <code>
        defer
       </code>
       中调用
       <code>
        recover
       </code>
       ，该
       <code>
        panic
       </code>
       会导致
       <strong>
        整个程序崩溃
       </strong>
       ，主goroutine无法捕获。
      </p>
      <pre><code class="prism language-go"><span class="token keyword">go</span> <span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token function">panic</span><span class="token punctuation">(</span><span class="token string">"goroutine panic"</span><span class="token punctuation">)</span> <span class="token comment">// 主goroutine无法捕获此panic</span>
<span class="token punctuation">}</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        程序已终止的情况
       </strong>
       <br/>
       若已调用
       <code>
        os.Exit()
       </code>
       或发生致命错误（如内存耗尽），
       <code>
        recover
       </code>
       无法恢复。
      </p>
     </li>
     <li>
      <p>
       <strong>
        非
        <code>
         defer
        </code>
        上下文中的
        <code>
         panic
        </code>
       </strong>
       <br/>
       <code>
        recover
       </code>
       仅在
       <code>
        defer
       </code>
       函数中调用有效，且必须位于触发
       <code>
        panic
       </code>
       的同一函数调用栈中。
      </p>
      <pre><code class="prism language-go"><span class="token keyword">func</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token function">panic</span><span class="token punctuation">(</span><span class="token string">"panic outside defer"</span><span class="token punctuation">)</span> <span class="token comment">// 无法被后续的defer捕获</span>
    <span class="token keyword">defer</span> <span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span> <span class="token function">recover</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">}</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token comment">// 此处defer不会执行</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        重复调用
        <code>
         panic
        </code>
       </strong>
       <br/>
       若在
       <code>
        defer
       </code>
       中再次触发
       <code>
        panic
       </code>
       且未被捕获，程序仍会崩溃。
      </p>
      <pre><code class="prism language-go"><span class="token keyword">defer</span> <span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token function">recover</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
    <span class="token function">panic</span><span class="token punctuation">(</span><span class="token string">"re-panic"</span><span class="token punctuation">)</span> <span class="token comment">// 未被捕获，导致程序终止</span>
<span class="token punctuation">}</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token function">panic</span><span class="token punctuation">(</span><span class="token string">"initial panic"</span><span class="token punctuation">)</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        CGO或系统级错误（部分）
       </strong>
       <br/>
       某些底层系统错误（如栈溢出、内存段错误）可能绕过Go的
       <code>
        panic/recover
       </code>
       机制，直接终止程序。
      </p>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_69">
     </a>
     <strong>
      关键规则总结
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        作用域限制
       </strong>
       <br/>
       <code>
        recover
       </code>
       仅在当前goroutine的
       <code>
        defer
       </code>
       函数中有效。
      </p>
     </li>
     <li>
      <p>
       <strong>
        运行时错误的可恢复性
       </strong>
       <br/>
       技术上可捕获，但程序状态可能损坏，需谨慎处理。
      </p>
     </li>
     <li>
      <p>
       <strong>
        设计建议
       </strong>
      </p>
      <ul>
       <li>
        仅对可预测的错误使用
        <code>
         panic/recover
        </code>
        （如输入验证失败）。
       </li>
       <li>
        避免依赖
        <code>
         recover
        </code>
        处理不可恢复错误（如内存越界），应确保程序健壮性。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <p>
     通过合理使用
     <code>
      defer
     </code>
     和
     <code>
      recover
     </code>
     ，可以优雅处理局部错误，但需理解其局限性以避免误用。
    </p>
