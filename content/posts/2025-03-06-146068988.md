    <p>
     ## Flutter 开发常见问题排查与解决
    </p>
    <p>
     ### 1. 项目初始化与依赖问题
     <br/>
     ```bash
     <br/>
     # 清理项目
     <br/>
     flutter clean
    </p>
    <p>
     # 获取依赖
     <br/>
     flutter pub get
    </p>
    <p>
     # 详细日志运行
     <br/>
     flutter run -v
     <br/>
     ```
    </p>
    <p>
     ### 2. 网络和下载问题
     <br/>
     - 网络慢可能导致依赖下载卡住
     <br/>
     - 使用 `-v` 参数可查看详细日志
     <br/>
     - 检查网络连接
     <br/>
     - 可以尝试使用国内镜像源
    </p>
    <p>
     ### 3. Android 设备认证与配置
     <br/>
     #### 设备连接前准备
     <br/>
     1. 确保安装 Android SDK Command-line Tools
     <br/>
     2. 开启开发者模式
     <br/>
     3. 允许USB调试
    </p>
    <p>
     ### 4. 系统状态检查
     <br/>
     ```bash
     <br/>
     # 检查Flutter开发环境状态
     <br/>
     flutter doctor
     <br/>
     ```
    </p>
    <p>
     ### 5. Gradle 配置优化
    </p>
    <p>
     #### build.gradle 配置
     <br/>
     ```gradle
     <br/>
     // 调整最小SDK版本
     <br/>
     defaultConfig {
     <!-- -->
     <br/>
     minSdkVersion 34  // 根据实际设备调整
     <br/>
     targetSdkVersion 35
     <br/>
     }
     <br/>
     ```
    </p>
    <p>
     #### gradle-wrapper.properties 修改
     <br/>
     ```properties
     <br/>
     # 修改下载链接为 bin 版本
     <br/>
     distributionUrl=https\://services.gradle.org/distributions/gradle-8.10.2-bin.zip
     <br/>
     ```
    </p>
    <p>
     ### 6. 常见卡住情况处理
     <br/>
     - 网络问题：检查代理、切换网络
     <br/>
     - 依赖下载：使用 `flutter pub get --verbose`
     <br/>
     - 编译卡住：查看详细日志 `flutter run -v`
    </p>
    <p>
     ### 7. 网络加速建议
     <br/>
     - 使用国内镜像
     <br/>
     - 配置 pub 源
     <br/>
     ```bash
     <br/>
     # 配置国内镜像
     <br/>
     flutter pub config proxy=http://your-proxy-server
     <br/>
     ```
    </p>
    <p>
     ### 8. 设备连接troubleshooting
     <br/>
     - 确保USB调试已开启
     <br/>
     - 检查驱动程序
     <br/>
     - 尝试更换USB接口
     <br/>
     - 重启ADB服务 `adb kill-server &amp;&amp; adb start-server`
    </p>
    <p>
     ### 9. 常用调试命令
     <br/>
     ```bash
     <br/>
     # 列出可用设备
     <br/>
     flutter devices
    </p>
    <p>
     # 查看详细诊断信息
     <br/>
     flutter doctor -v
     <br/>
     ```
    </p>
    <p>
     ### 10. 最佳实践
     <br/>
     - 定期更新Flutter和Dart SDK
     <br/>
     - 保持Gradle和依赖版本兼容
     <br/>
     - 使用稳定的网络环境
     <br/>
     - 备份项目依赖配置
    </p>
    <p>
     ## 温馨提示
     <br/>
     - 遇到问题先看日志
     <br/>
     - 不要盲目尝试，逐步排查
     <br/>
     - 必要时可以清理缓存重新开始
    </p>
