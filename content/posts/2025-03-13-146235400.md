     linux中yum和wget指令的区别
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">

    <p>
     <code>
      yum
     </code>
     和
     <code>
      wget
     </code>
     都是 Linux 上的下载工具，但它们的用途、下载方式和适用场景不同。以下是它们的
     <strong>
      主要区别
     </strong>
     ：
    </p>
    <hr/>
    <h3>
     <strong>
      1.
      <code>
       yum
      </code>
      是软件包管理器，
      <code>
       wget
      </code>
      是文件下载工具
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         功能
        </strong>
       </th>
       <th>
        <strong>
         <code>
          yum
         </code>
        </strong>
       </th>
       <th>
        <strong>
         <code>
          wget
         </code>
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         用途
        </strong>
       </td>
       <td>
        <strong>
         安装、更新和管理 RPM 软件包
        </strong>
       </td>
       <td>
        <strong>
         从 HTTP/HTTPS/FTP 下载文件
        </strong>
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         工作方式
        </strong>
       </td>
       <td>
        通过
        <code>
         yum
        </code>
        仓库获取软件包及其依赖项
       </td>
       <td>
        仅下载文件，不管理安装
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         依赖管理
        </strong>
       </td>
       <td>
        <strong>
         自动解析依赖
        </strong>
        ，安装所需的软件包
       </td>
       <td>
        只下载文件，不解析依赖
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        需要安装软件，例如
        <code>
         yum install wget
        </code>
       </td>
       <td>
        下载网页、文件、脚本、二进制文件等
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         协议支持
        </strong>
       </td>
       <td>
        只支持访问 Yum 仓库（通常是 HTTP/HTTPS）
       </td>
       <td>
        支持 HTTP、HTTPS、FTP
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         是否需要 root 权限
        </strong>
       </td>
       <td>
        需要 (
        <code>
         sudo
        </code>
        或 root 用户)
       </td>
       <td>
        不需要 root
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     <strong>
      示例
     </strong>
    </h4>
    <h5>
     <strong>
      使用
      <code>
       yum
      </code>
      安装软件
     </strong>
    </h5>
    <pre><code>yum install wget -y
</code></pre>
    <p>
     📌
     <strong>
      作用
     </strong>
     ：安装
     <code>
      wget
     </code>
     ，并自动安装所需的依赖项。
    </p>
    <h5>
     <strong>
      使用
      <code>
       wget
      </code>
      下载文件
     </strong>
    </h5>
    <pre><code>wget https://example.com/file.tar.gz
</code></pre>
    <p>
     📌
     <strong>
      作用
     </strong>
     ：下载
     <code>
      file.tar.gz
     </code>
     ，但不会自动安装它。
    </p>
    <hr/>
    <h3>
     <strong>
      2.
      <code>
       yum
      </code>
      主要用于软件包管理
     </strong>
    </h3>
    <h4>
     <strong>
      （1）安装软件
     </strong>
    </h4>
    <pre><code>yum install nginx -y
</code></pre>
    <p>
     📌 作用：安装
     <code>
      nginx
     </code>
     ，并自动安装所有依赖项。
    </p>
    <h4>
     <strong>
      （2）更新系统
     </strong>
    </h4>
    <pre><code>yum update -y
</code></pre>
    <p>
     📌 作用：更新所有软件包到最新版本。
    </p>
    <h4>
     <strong>
      （3）搜索软件包
     </strong>
    </h4>
    <pre><code>yum search git
</code></pre>
    <p>
     📌 作用：查找包含 "git" 关键字的软件包。
    </p>
    <h4>
     <strong>
      （4）查看已安装软件
     </strong>
    </h4>
    <pre><code>yum list installed | grep wget
</code></pre>
    <p>
     📌 作用：检查
     <code>
      wget
     </code>
     是否已安装。
    </p>
    <hr/>
    <h3>
     <strong>
      3.
      <code>
       wget
      </code>
      主要用于下载文件
     </strong>
    </h3>
    <h4>
     <strong>
      （1）下载单个文件
     </strong>
    </h4>
    <pre><code>wget https://example.com/file.zip
</code></pre>
    <p>
     📌 作用：从
     <code>
      example.com
     </code>
     下载
     <code>
      file.zip
     </code>
     到本地。
    </p>
    <h4>
     <strong>
      （2）断点续传
     </strong>
    </h4>
    <pre><code>wget -c https://example.com/file.zip
</code></pre>
    <p>
     📌 作用：如果下载中断，继续下载文件。
    </p>
    <h4>
     <strong>
      （3）下载网页
     </strong>
    </h4>
    <pre><code>wget -O index.html https://example.com
</code></pre>
    <p>
     📌 作用：将网页保存为
     <code>
      index.html
     </code>
     。
    </p>
    <h4>
     <strong>
      （4）递归下载整个网站
     </strong>
    </h4>
    <pre><code>wget -r -np -k https://example.com
</code></pre>
    <p>
     📌 作用：下载整个
     <code>
      example.com
     </code>
     站点。
    </p>
    <hr/>
    <h3>
     <strong>
      4.
      <code>
       yum
      </code>
      和
      <code>
       wget
      </code>
      的典型使用场景
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         场景
        </strong>
       </th>
       <th>
        <strong>
         应该用
         <code>
          yum
         </code>
         还是
         <code>
          wget
         </code>
         ？
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        安装
        <code>
         nginx
        </code>
       </td>
       <td>
        <code>
         yum install nginx
        </code>
       </td>
      </tr>
      <tr>
       <td>
        更新系统
       </td>
       <td>
        <code>
         yum update -y
        </code>
       </td>
      </tr>
      <tr>
       <td>
        下载一个
        <code>
         .tar.gz
        </code>
        文件
       </td>
       <td>
        <code>
         wget https://example.com/file.tar.gz
        </code>
       </td>
      </tr>
      <tr>
       <td>
        安装
        <code>
         wget
        </code>
        本身
       </td>
       <td>
        <code>
         yum install wget
        </code>
       </td>
      </tr>
      <tr>
       <td>
        下载
        <code>
         .rpm
        </code>
        包并手动安装
       </td>
       <td>
        <code>
         wget https://example.com/package.rpm
        </code>
        +
        <code>
         rpm -ivh package.rpm
        </code>
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <strong>
      5.
      <code>
       yum
      </code>
      和
      <code>
       wget
      </code>
      结合使用
     </strong>
    </h3>
    <p>
     有时候你需要
     <strong>
      先用
      <code>
       wget
      </code>
      下载
      <code>
       .rpm
      </code>
      文件，然后用
      <code>
       yum
      </code>
      或
      <code>
       rpm
      </code>
      安装
     </strong>
     ：
    </p>
    <pre><code>wget http://mirrors.aliyun.com/nginx/nginx-1.20.1.rpm
yum localinstall nginx-1.20.1.rpm -y
</code></pre>
    <p>
     📌
     <strong>
      作用
     </strong>
     ：
    </p>
    <ol>
     <li>
      用
      <code>
       wget
      </code>
      下载
      <code>
       nginx
      </code>
      的
      <code>
       .rpm
      </code>
      文件。
     </li>
     <li>
      用
      <code>
       yum localinstall
      </code>
      安装
      <code>
       .rpm
      </code>
      ，并解析依赖。
     </li>
    </ol>
    <hr/>
    <h3>
     <strong>
      6.
      <code>
       curl
      </code>
      vs
      <code>
       wget
      </code>
      vs
      <code>
       yum
      </code>
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         工具
        </strong>
       </th>
       <th>
        <strong>
         主要用途
        </strong>
       </th>
       <th>
        <strong>
         是否自动安装
        </strong>
       </th>
       <th>
        <strong>
         是否多线程
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         yum
        </code>
       </td>
       <td>
        安装和管理软件包
       </td>
       <td>
        ✅ 是
       </td>
       <td>
        ❌ 否
       </td>
      </tr>
      <tr>
       <td>
        <code>
         wget
        </code>
       </td>
       <td>
        下载文件（支持 HTTP/FTP）
       </td>
       <td>
        ❌ 否
       </td>
       <td>
        ❌ 否（单线程）
       </td>
      </tr>
      <tr>
       <td>
        <code>
         curl
        </code>
       </td>
       <td>
        发送 HTTP 请求、API 调用
       </td>
       <td>
        ❌ 否
       </td>
       <td>
        ❌ 否（默认单线程）
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h3>
     <strong>
      7. 总结
     </strong>
    </h3>
    <p>
     ✅
     <strong>
      <code>
       yum
      </code>
      适用于
     </strong>
     ：
    </p>
    <ul>
     <li>
      安装、更新、删除软件包
     </li>
     <li>
      自动解决软件依赖关系
     </li>
     <li>
      适用于
      <code>
       CentOS/RHEL
      </code>
     </li>
    </ul>
    <p>
     ✅
     <strong>
      <code>
       wget
      </code>
      适用于
     </strong>
     ：
    </p>
    <ul>
     <li>
      下载网页、文件、脚本、压缩包
     </li>
     <li>
      支持 HTTP、HTTPS、FTP
     </li>
     <li>
      断点续传
     </li>
    </ul>
    <p>
     如果你想安装软件
     <strong>
      （如
      <code>
       wget
      </code>
      本身）
     </strong>
     ，应该使用：
    </p>
    <pre><code>yum install wget
</code></pre>
    <p>
     如果你只是想下载一个
     <code>
      .tar.gz
     </code>
     文件，而不是安装它，应该使用：
    </p>
    <pre><code>wget https://example.com/file.tar.gz
</code></pre>
    <p>
     🚀
     <strong>
      所以
      <code>
       yum
      </code>
      适合安装软件，而
      <code>
       wget
      </code>
      适合下载文件！
     </strong>
    </p>
    <p>
    </p>
    <p>
    </p>
    <p>
     是的，
     <strong>
      <code>
       wget
      </code>
      只负责下载文件
     </strong>
     ，如果你下载的是一个
     <strong>
      RPM 安装包
     </strong>
     ，那么你还需要使用
     <code>
      yum
     </code>
     或
     <code>
      rpm
     </code>
     来安装它。
    </p>
    <hr/>
    <h3>
     <strong>
      🔹 示例：使用
      <code>
       wget
      </code>
      下载并安装软件
     </strong>
    </h3>
    <p>
     如果官方
     <code>
      yum
     </code>
     源没有你需要的软件，你可以先用
     <code>
      wget
     </code>
     下载 RPM 包，再手动安装。
    </p>
    <h4>
     <strong>
      1️⃣ 使用
      <code>
       wget
      </code>
      下载 RPM 安装包
     </strong>
    </h4>
    <pre><code>wget http://mirrors.aliyun.com/nginx/nginx-1.20.1.rpm
</code></pre>
    <p>
     📌 这里
     <code>
      wget
     </code>
     只是把
     <code>
      nginx-1.20.1.rpm
     </code>
     <strong>
      下载到本地
     </strong>
     。
    </p>
    <hr/>
    <h4>
     <strong>
      2️⃣ 使用
      <code>
       yum
      </code>
      安装该 RPM 包
     </strong>
    </h4>
    <pre><code>yum localinstall nginx-1.20.1.rpm -y
</code></pre>
    <p>
     📌
     <strong>
      <code>
       yum localinstall
      </code>
      会自动解析和安装依赖项
     </strong>
     ，确保安装成功。
    </p>
    <hr/>
    <h4>
     <strong>
      3️⃣ 如果
      <code>
       yum
      </code>
      源不可用，可以用
      <code>
       rpm
      </code>
      安装
     </strong>
    </h4>
    <p>
     如果你不想使用
     <code>
      yum
     </code>
     ，也可以用
     <code>
      rpm
     </code>
     命令直接安装：
    </p>
    <pre><code>rpm -ivh nginx-1.20.1.rpm
</code></pre>
    <p>
     📌 但是
     <code>
      rpm
     </code>
     <strong>
      不会自动解决依赖问题
     </strong>
     ，如果缺少依赖，需要手动安装！
    </p>
    <p>
     <strong>
      如果缺少依赖，可以先运行
     </strong>
     ：
    </p>
    <pre><code>rpm -ivh nginx-1.20.1.rpm
</code></pre>
    <p>
     如果报错，例如：
    </p>
    <pre><code>error: Failed dependencies:
    libssl.so.10 is needed by nginx-1.20.1.rpm
</code></pre>
    <p>
     你需要手动安装依赖：
    </p>
    <pre><code>yum install libssl.so.10
</code></pre>
    <p>
     然后再安装
     <code>
      nginx-1.20.1.rpm
     </code>
     。
    </p>
    <hr/>
    <h3>
     <strong>
      🔹 总结
     </strong>
    </h3>
    <table>
     <thead>
      <tr>
       <th>

        </strong>
       </th>

       <th>
        <strong>
         作用
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>

       </td>

        </code>

       </td>
      </tr>
      <tr>
       <td>
        用
        <code>
         yum
        </code>
        安装
       </td>
       <td>
        <code>
         yum localinstall package.rpm -y
        </code>
       </td>

        </strong>
        ，自动解析依赖

       </td>
      </tr>
     </tbody>
    </table>
