    <p>
     大家做智能体应该很久了
     <br/>
     我们发现人类预制的规则总会因为稀疏区域导致失效
     <br/>
     所以大家展开了思考
     <br/>
     一个智能体可以根据场景描述自动规划路径
     <br/>
     一个智能体可以根据场景描述自动规划路径，其核心流程大致如下：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        场景解析与约束提取
       </strong>
      </p>
      <ul>
       <li>
        利用自然语言处理（NLP）技术，解析输入的场景描述，提取关键目标、约束条件和环境信息（如路况、天气、时间要求等）。
       </li>
       <li>
        示例：在描述“城市中心高峰期交通拥堵、雨天出行”的场景时，智能体将识别出“交通拥堵”、“雨天”、“高峰期”等关键词，并提取出时间、地点及安全要求等约束。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        数据检索与知识库支持
       </strong>
      </p>
      <ul>
       <li>
        调用数据库和知识库接口，查询历史数据、实时信息或文档记录，获取当前环境下的相关信息。
       </li>
       <li>
        支持多种数据源，例如：关系型数据库（MySQL、PostgreSQL、TiDB）、NoSQL（MongoDB、Redis）、搜索引擎（Elasticsearch）、图数据库（Neo4j）以及文档内容（PDF、PPT、DOCX、Excel转换的结构化数据）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        路径生成与搜索算法
       </strong>
      </p>
      <ul>
       <li>
        结合场景约束和检索到的数据信息，应用路径规划算法（如 A*、Dijkstra 或基于强化学习的方法）生成备选路径。
       </li>
       <li>
        根据各方案在安全性、效率、成本等指标上的得分，确定最优方案。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        动态反馈与迭代调整
       </strong>
      </p>
      <ul>
       <li>
        在执行过程中，实时监控反馈（例如交通状态、天气变化、意外事件），并利用 Agentic Manus 的反馈机制调整规划。
       </li>
       <li>
        实现步骤结果驱动：每一步执行后，智能体根据实际结果重新规划下一步行动，确保整个任务链始终处于最优状态。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        OpenAI 协议接口支持
       </strong>
      </p>
      <ul>
       <li>
        对外暴露类似 OpenAI Chat Completion 的接口，支持 prompt 输入和函数调用，允许用户通过标准化 API 与智能体进行交互，并获取路径规划建议和动态调整反馈。
       </li>
      </ul>
     </li>
    </ol>
    <h4>
     <a id="_25">
     </a>
     示例应用场景
    </h4>
    <p>
     假设智能体接收到如下场景描述：
    </p>
    <blockquote>
     <p>
      “请规划一条从 A 点到 B 点的路径，目前处于城市中心高峰期，且预报显示接下来1小时内有暴雨，要求尽可能避开拥堵路段，保证安全。”
     </p>
    </blockquote>
    <p>
     智能体处理流程：
    </p>
    <ul>
     <li>
      <strong>
       解析场景
      </strong>
      ：提取关键词“城市中心”、“高峰期”、“暴雨”、“避开拥堵”。
     </li>
     <li>
      <strong>
       数据检索
      </strong>
      ：查询实时交通数据（通过 Elasticsearch 或实时 API），检索天气预报数据（例如从内部接口获取暴雨信息）。
     </li>
     <li>
      <strong>
       路径生成
      </strong>
      ：基于 A* 算法生成多个备选路径，并对各路径进行安全、时间、成本的评估。
     </li>
     <li>
      <strong>
       方案选择与反馈
      </strong>
      ：选择最优路径后，实时监控执行情况，如遇突发状况（例如某条备选路段出现事故），智能体会调用函数重新规划，输出更新后的最佳路径建议。
     </li>
     <li>
      <strong>
       接口交互
      </strong>
      ：整个过程通过 OpenAI 协议接口进行交互，用户可以查询当前规划状态，甚至手动输入反馈调整要求。
     </li>
    </ul>
    <p>
     这种全流程闭环的智能路径规划，不仅依赖于高效的数据检索和搜索算法，还依托于动态反馈与迭代机制，确保在复杂多变的场景下依然能够输出最优的路径规划方案。
    </p>
    <p>
     而随着manus的出现
    </p>
    <p>
     这种思想在很短的时间内成为了主流。
     <br/>
     今天我们来介绍两个开源工作。
    </p>
    <ul>
     <li>
      OpenManus 是一个开源项目，允许用户创建自己的 AI 代理，无需邀请码。
     </li>
     <li>
      它由 MetaGPT 团队在 3 小时内开发，简单易用，欢迎贡献和反馈。
     </li>
     <li>
      用户可以配置 LLM API（如 OpenAI 的 GPT-4o）来运行 AI 代理，适合开发者探索 AI 技术。
     </li>
     <li>
      安装简单，需设置 Conda 环境和 API 密钥，运行后可通过终端输入任务。
     </li>
    </ul>
    <p>
     <strong>
      项目概述
     </strong>
     <br/>
     OpenManus 是一个开源工具，旨在让用户无需邀请码即可创建 AI 代理。它与 Manus（一个需要邀请码的 AI 代理）相对，提供了更开放的访问方式。项目由 MetaGPT 社区成员开发，快速完成，适合技术爱好者尝试 AI 自动化。
    </p>
    <p>
     <strong>
      如何开始
     </strong>
     <br/>
     用户需创建 Python 3.12 的 Conda 环境，克隆 GitHub 仓库
     <a href="https://github.com/mannaandpoem/OpenManus">
      OpenManus 仓库
     </a>
     ，安装依赖，并配置
     <code>
      config.toml
     </code>
     文件以添加 API 密钥。运行
     <code>
      python main.py
     </code>
     或
     <code>
      python run_flow.py
     </code>
     （不稳定版）后，可通过终端输入任务让 AI 代理执行。
    </p>
    <p>
     <strong>
      未来发展
     </strong>
     <br/>
     项目计划包括优化任务规划、添加实时演示、重放功能、强化学习模型调整及全面基准测试，显示其持续发展的潜力。
    </p>

    <h4>

     详细报告
    </h4>
    <p>
     OpenManus 是一个开源项目，旨在让用户能够创建自己的 AI 代理，无需邀请码，相比于需要邀请码的 Manus AI 提供了更开放的访问方式。该项目由 MetaGPT 社区的成员开发，包括 @mannaandpoem、@XiangJinyu、@MoshiQAQ 和 @didiforgithub，他们在短短 3 小时内完成了开发，体现了高效的协作和 AI 技术能力。以下是项目的详细介绍，涵盖其功能、安装步骤、未来规划及社区参与方式。
    </p>
    <h5>
     <a id="_68">
     </a>
     项目背景与意义
    </h5>
    <p>
     OpenManus 的灵感来源于 Manus AI，这是一个由中国公司 Monica AI 开发的通用 AI 代理，能够自主处理复杂现实任务，如旅行规划、股票分析和教育材料创建。根据
     <a href="https://manus.im/" rel="nofollow">
      Manus AI 官网
     </a>
     和相关新闻报道（如
     <a href="https://www.newsweek.com/manus-new-china-ai-agent-explained-2040445" rel="nofollow">
      Newsweek 文章
     </a>
     ），Manus 当前为邀请制，限制了用户访问。而 OpenManus 作为开源替代方案，消除了这一障碍，允许任何有兴趣的用户参与和使用。
    </p>
    <p>
     该项目的开源性质使其成为 AI 技术民主化的重要一步，适合开发者、技术爱好者及学生探索 AI 代理的潜力。其简单实现也降低了学习和贡献的门槛，鼓励社区协作创新。
    </p>
    <h5>
     <a id="_73">
     </a>
     功能与特性
    </h5>
    <p>
     OpenManus 支持通过配置 LLM API（如 OpenAI 的 GPT-4o）运行 AI 代理，用户可以通过终端输入任务，驱动智能体执行操作。根据
     <a href="https://github.com/mannaandpoem/OpenManus">
      GitHub 仓库
     </a>
     的描述，其主要特性包括：
    </p>
    <ul>
     <li>
      <strong>
       项目演示
      </strong>
      ：提供视频演示（如 seo_website.mp4），展示其功能。
     </li>
     <li>
      <strong>
       安装步骤
      </strong>
      ：需创建 Python 3.12 的 Conda 环境，克隆仓库
      <a href="https://github.com/mannaandpoem/OpenManus">
       OpenManus 仓库
      </a>
      ，并用
      <code>
       pip install -r requirements.txt
      </code>
      安装依赖。
     </li>
     <li>
      <strong>
       配置要求
      </strong>
      ：用户需编辑
      <code>
       config.toml
      </code>
      文件，添加 API 密钥，示例文件为
      <code>
       config/config.example.toml
      </code>
      ，支持 gpt-4o，最大令牌数为 4096。
     </li>
     <li>
      <strong>
       快速启动
      </strong>
      ：运行
      <code>
       python main.py
      </code>
      即可使用，或用
      <code>
       python run_flow.py
      </code>
      运行不稳定版本。
     </li>
     <li>
      <strong>
       社区参与
      </strong>
      ：欢迎用户通过 GitHub 提交建议或拉取请求，或通过电子邮件（mannaandpoem@gmail.com）联系 @mannaandpoem。
     </li>
     <li>
      <strong>
       路线图
      </strong>
      ：计划包括更好的任务规划、实时演示、重放功能、强化学习（RL）微调模型及全面基准测试。
     </li>
    </ul>

     此外，项目还感谢

      anthropic-computer-use

    <h5>
     <a id="_85">
     </a>
     安装与使用详解
    </h5>
    <p>
     安装 OpenManus 的过程如下，适合有一定编程基础的用户：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        环境准备
       </strong>
       ：
      </p>
      <ul>
       <li>
        打开终端，运行
        <code>
         conda create -n open_manus python=3.12
        </code>
        创建新环境。
       </li>
       <li>
        激活环境：
        <code>
         conda activate open_manus
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        克隆仓库
       </strong>
       ：
      </p>
      <ul>
       <li>
        运行
        <code>
         git clone https://github.com/mannaandpoem/OpenManus.git
        </code>
        下载代码。
       </li>
       <li>
        进入目录：
        <code>
         cd OpenManus
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        安装依赖
       </strong>
       ：
      </p>
      <ul>
       <li>
        运行
        <code>
         pip install -r requirements.txt
        </code>
        安装所需库。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        配置 API
       </strong>
       ：
      </p>
      <ul>
       <li>
        复制示例配置文件：
        <code>
         cp config/config.example.toml config/config.toml
        </code>
        。
       </li>
       <li>
        编辑
        <code>
         config.toml
        </code>
        ，添加你的 API 密钥，例如：
        <pre><code>[llm]
model = "gpt-4o"
base_url = "https://api.openai.com/v1"
api_key = "sk-..."  # 替换为实际 API 密钥
max_tokens = 4096
temperature = 0.0
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        运行项目
       </strong>
       ：
      </p>
      <ul>
       <li>
        稳定版：运行
        <code>
         python main.py
        </code>
        ，然后通过终端输入任务。
       </li>
       <li>
        不稳定版：运行
        <code>
         python run_flow.py
        </code>
        ，适用于测试新功能。
       </li>
      </ul>
     </li>
    </ol>

    <p>
     OpenManus 是一个由 MetaGPT 社区开发的开源 AI 代理工具，旨在让用户无需邀请码即可创建自己的 AI 代理。它基于大型语言模型（LLM）如 GPT-4o 运行，适合开发者和技术爱好者探索 AI 自动化。根据 2025 年 3 月 6 日的最新信息，其当前功能主要包括网页数据获取和代码执行，但用户表达了对生成多个决策路径并在实施过程中优化的更高需求的担忧。以下是详细分析和解决方案建议，旨在为用户提供全面的指导。
    </p>
    <h5>
     <a id="_183">
     </a>
     项目背景与当前能力
    </h5>
    <p>
     OpenManus 的 GitHub 仓库（
     <a href="https://github.com/mannaandpoem/OpenManus">
      OpenManus GitHub 仓库
     </a>
     ）显示，它由团队成员 @mannaandpoem、@XiangJinyu、@MoshiQAQ 和 @didiforgithub 在 3 小时内开发完成。这一高效性体现了开源社区的协作能力，但也表明其功能较为基础。安装过程简单，包括创建 Python 3.12 的 Conda 环境、克隆仓库并配置 LLM API 密钥（如 OpenAI 的 GPT-4o）。用户可以通过终端输入任务，代理会执行如网页搜索或代码运行的操作。
    </p>
    <p>
     然而，用户的期望是让 AI 根据场景生成多个决策路径，并在实施过程中优化这些路径。这需要代理具备高级规划和决策能力，而当前 OpenManus 的底层组件（如仅支持网页数据获取和代码执行）显得不足以满足这一需求。
    </p>
    <h5>
     <a id="_188">
     </a>
     规划能力的扩展可能性
    </h5>
    <p>
     研究表明，LLM 可以作为规划工具，生成初步计划并辅助决策，但其效果有限，尤其是在需要解决子目标交互的复杂规划任务中（
     <a href="https://yochan-lab.github.io/tutorial/LLMs-Planning/index.html" rel="nofollow">
      LLMs-Planning Tutorial
     </a>
     ）。为了实现用户的目标，可以考虑以下方法：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        整合外部规划库
       </strong>
      </p>
      <ul>
       <li>
        Python 社区提供了多个 AI 规划库，如 Poodle 和 pyplanning。
       </li>
       <li>
        Poodle 是一个 Python 到 PDDL 的编译器，支持自动化规划，适用于机器人规划、调度和决策优化（
        <a href="https://pythonawesome.com/python-framework-for-ai-planning-and-automated-programming/" rel="nofollow">
         Poodle 框架
        </a>
        ）。
       </li>
       <li>
        pyplanning 实现了 AI 规划算法，支持 PDDL 1.2 语法，适合嵌入现有应用（
        <a href="https://github.com/kbvatral/pyplanning">
         pyplanning GitHub
        </a>
        ）。
       </li>
       <li>
        用户可以修改 OpenManus 的代码，将这些库集成，生成决策路径并根据执行反馈优化。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        利用 LLM 进行规划
       </strong>
      </p>
      <ul>
       <li>
        LLM 可以通过提示词生成任务计划，例如使用“链式思考”（Chain of Thought）或 ReAct 框架，让模型逐步推理并生成步骤（
        <a href="https://dki-lab.github.io/LLM-Planner/" rel="nofollow">
         LLM-Planner 研究
        </a>
        ）。
       </li>
       <li>
        例如，用户可以设计提示词如“为任务‘预订从纽约到伦敦的航班’生成一个步骤计划”，然后让代理执行每个步骤，并在执行失败时重新规划。
       </li>
       <li>
        这需要添加记忆功能以跟踪状态，并可能结合外部验证器确保计划的可行性（
        <a href="https://arxiv.org/html/2402.01817v3" rel="nofollow">
         LLM-Modulo Frameworks 论文
        </a>
        ）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        社区与路线图支持
       </strong>
      </p>
      <ul>
       <li>
        OpenManus 的路线图中提到“更好的规划”，表明项目团队计划增强规划能力（
        <a href="https://github.com/mannaandpoem/OpenManus">
         OpenManus GitHub 路线图
        </a>
        ）。
       </li>
       <li>
        用户可以参与社区，通过 GitHub 提交议题或拉取请求，与开发者合作实现规划功能。
       </li>
      </ul>
     </li>
    </ol>

     详细报告
    </h4>
    <p>
     OpenManus 是一个由 MetaGPT 社区开发的开源 AI 代理工具，旨在让用户无需邀请码即可创建自己的 AI 代理。它基于大型语言模型（LLM）如 GPT-4o 运行，适合开发者和技术爱好者探索 AI 自动化。根据 2025 年 3 月 6 日的最新信息，其当前功能主要包括网页数据获取和代码执行，但用户表达了对生成多个决策路径并在实施过程中优化的更高需求的担忧。以下是详细分析和解决方案建议，旨在为用户提供全面的指导，满足二次开发的需要。
    </p>
    <h5>
     <a id="_292">
     </a>
     项目背景与当前能力
    </h5>
    <p>
     OpenManus 的 GitHub 仓库（
     <a href="https://github.com/mannaandpoem/OpenManus">
      OpenManus GitHub 详细描述
     </a>
     ）显示，它由团队成员 @mannaandpoem、@XiangJinyu、@MoshiQAQ 和 @didiforgithub 在 3 小时内开发完成。这一高效性体现了开源社区的协作能力，但也表明其功能较为基础。安装过程简单，包括创建 Python 3.12 的 Conda 环境、克隆仓库并配置 LLM API 密钥（如 OpenAI 的 GPT-4o）。用户可以通过终端输入任务，代理会执行如网页搜索或代码运行的操作。
    </p>
    <p>
     然而，用户的期望是让 AI 根据场景生成多个决策路径，并在实施过程中优化这些路径。这需要代理具备高级规划和决策能力，而当前 OpenManus 的底层组件（如仅支持网页数据获取和代码执行）显得不足以满足这一需求。
    </p>
    <h5>
     <a id="_297">
     </a>
     规划能力的扩展可能性
    </h5>
    <p>
     研究表明，LLM 可以作为规划工具，生成初步计划并辅助决策，但其效果有限，尤其是在需要解决子目标交互的复杂规划任务中（
     <a href="https://github.com/AGI-Edgerunners/LLM-Planning-Papers">
      LLM-based Planning Papers 必读论文
     </a>
     ）。为了实现用户的目标，可以考虑以下方法：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        利用 LLM 进行规划
       </strong>
      </p>
      <ul>
       <li>
        LLM 可以通过提示词生成任务计划，例如使用“链式思考”（Chain of Thought）或 ReAct 框架，让模型逐步推理并生成步骤（
        <a href="https://dki-lab.github.io/LLM-Planner/" rel="nofollow">
         LLM-Planner 研究论文
        </a>
        ）。
       </li>
       <li>
        例如，用户可以设计提示词如“为任务‘预订从纽约到伦敦的航班’生成一个步骤计划”，然后让代理执行每个步骤，并在执行失败时重新规划。
       </li>
       <li>
        这需要添加记忆功能以跟踪状态，并可能结合外部验证器确保计划的可行性（
        <a href="https://arxiv.org/html/2402.01817v3" rel="nofollow">
         LLM-Modulo Frameworks 规划论文
        </a>
        ）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        整合外部规划库
       </strong>
      </p>
      <ul>
       <li>
        Python 社区提供了多个 AI 规划库，如 Poodle 和 pyplanning。
       </li>
       <li>
        Poodle 是一个 Python 到 PDDL 的编译器，支持自动化规划，适用于机器人规划、调度和决策优化（
        <a href="https://pythonawesome.com/python-framework-for-ai-planning-and-automated-programming/" rel="nofollow">
         Poodle Python 规划框架
        </a>
        ）。
       </li>
       <li>
        pyplanning 实现了 AI 规划算法，支持 PDDL 1.2 语法，适合嵌入现有应用（
        <a href="https://github.com/kbvatral/pyplanning">
         pyplanning AI 规划算法实现
        </a>
        ）。
       </li>
       <li>
        用户可以修改 OpenManus 的代码，将这些库集成，生成决策路径并根据执行反馈优化。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        社区与路线图支持
       </strong>
      </p>
      <ul>
       <li>
        OpenManus 的路线图中提到“更好的规划”，表明项目团队计划增强规划能力（
        <a href="https://github.com/mannaandpoem/OpenManus">
         OpenManus GitHub 路线图
        </a>
        ）。
       </li>
       <li>
        用户可以参与社区，通过 GitHub 提交议题或拉取请求，与开发者合作实现规划功能。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_315">
     </a>
     实施步骤与技术细节
    </h5>
    <p>
     以下是用户可能采取的具体步骤：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        理解 OpenManus 架构
       </strong>
      </p>
      <ul>
       <li>
        研究 OpenManus 的代码，了解任务处理和 LLM 交互的逻辑。
       </li>
       <li>
        当前代码主要支持通过终端输入任务并执行简单操作，缺乏规划模块。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        设计规划模块
       </strong>
      </p>
      <ul>
       <li>
        创建一个新模块，负责生成计划。例如，使用 LLM 生成计划的伪代码如下：
        <pre><code class="prism language-python"><span class="token keyword">class</span> <span class="token class-name">PlanningAgent</span><span class="token punctuation">:</span>
    <span class="token keyword">def</span> <span class="token function">generate_plan</span><span class="token punctuation">(</span>self<span class="token punctuation">,</span> task<span class="token punctuation">)</span><span class="token punctuation">:</span>
        prompt <span class="token operator">=</span> <span class="token string-interpolation"><span class="token string">f"生成一个步骤计划来完成任务：</span><span class="token interpolation"><span class="token punctuation">{<!-- --></span>task<span class="token punctuation">}</span></span><span class="token string">"</span></span>
        plan <span class="token operator">=</span> self<span class="token punctuation">.</span>llm_client<span class="token punctuation">.</span>generate_text<span class="token punctuation">(</span>prompt<span class="token punctuation">)</span>
        <span class="token keyword">return</span> plan<span class="token punctuation">.</span>split<span class="token punctuation">(</span><span class="token string">"\n"</span><span class="token punctuation">)</span>  <span class="token comment"># 解析为步骤列表</span>
</code></pre>
       </li>
       <li>
        计划应包括多个决策路径，例如“通过搜索引擎查找航班信息”或“直接访问航空公司网站”。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        执行与优化
       </strong>
      </p>
      <ul>
       <li>
        代理按步骤执行计划，每步完成后检查结果。如果失败（如无法找到航班），反馈给 LLM 生成新计划。
       </li>
       <li>
        需要实现状态跟踪功能，确保代理记住已完成步骤，优化后续决策。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        测试与迭代
       </strong>
      </p>
      <ul>
       <li>
        使用测试场景（如旅行规划或项目管理）验证规划功能，观察是否能生成多个路径并优化。
       </li>
       <li>
        根据结果调整提示词或整合规划库以提高效率。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_341">
     </a>
     挑战与注意事项
    </h5>
    <ul>
     <li>
      LLM 生成的计划可能不完美，尤其在复杂任务中，需要外部验证器或传统规划算法的支持（
      <a href="https://arxiv.org/html/2402.01817v2" rel="nofollow">
       LLMs Can’t Plan, But Can Help Planning 论文
      </a>
      ）。
     </li>
     <li>
      整合规划库可能增加复杂性，需要用户具备一定编程经验。
     </li>
     <li>
      当前 OpenManus 社区规模较小，相关文档和示例可能有限，建议参考通用 LLM 规划教程（如
      <a href="https://hlfshell.ai/posts/llm-task-planner/" rel="nofollow">
       Utilizing LLMs as a Task Planning Agent for Robotics
      </a>
      ）。
     </li>
    </ul>
    <h5>
     <a id="_346">
     </a>
     社区参与与未来展望
    </h5>
    <p>
     OpenManus 鼓励用户贡献，社区可以通过 GitHub 提交建议或联系 @mannaandpoem（邮箱：mannaandpoem@gmail.com）获取支持。路线图显示未来可能包括实时演示和强化学习模型，这些可能进一步支持规划优化。
    </p>
    <h5>
     <a id="_349">
     </a>
     意想不到的细节
    </h5>
    <p>
     尽管 OpenManus 规划能力有限，但其在 3 小时内开发完成，这一高效性在开源项目中较为少见，体现了 MetaGPT 团队的协作能力。这可能激励用户积极参与，扩展其功能以满足更高需求。
    </p>
