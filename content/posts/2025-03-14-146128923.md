    <h3>
     <a id="1__5">
     </a>
     1. 备份类型
    </h3>
    <ul>
     <li>
      <strong>
       逻辑备份
      </strong>
      ：导出数据库的逻辑结构，以
      <code>
       SQL语句
      </code>
      的形式呈现，可读性强。适用于小数据量或跨版本迁移。常见工具包括
      <code>
       mysqldump
      </code>
      、
      <code>
       mysqlpump
      </code>
      、
      <code>
       mydumper
      </code>
      。其特点是兼容性好，但备份和恢复速度较慢，支持单表、单库或全库备份。
     </li>
     <li>
      <strong>
       物理备份
      </strong>
      ：直接复制数据库的
      <code>
       物理文件
      </code>
      ，如数据文件、日志文件。备份和恢复速度快，适合大数据量。
     </li>
     <li>
      <strong>
       增量备份
      </strong>
      ：仅备份自上次全量或增量备份后的变更数据。适用于大数据量且每日变更较少的场景，常用工具为
      <code>
       XtraBackup
      </code>
      ，基于 InnoDB 的 LSN 日志。
     </li>
    </ul>
    <p>
     <br/>
     <br/>
    </p>
    <h3>
     <a id="2__11">
     </a>
     2. 备份策略设计
    </h3>
    <ul>
     <li>
      <strong>
       全量备份 + 增量备份
      </strong>
      ：每周进行一次全量备份，每日进行一次增量备份。恢复时，先恢复全量备份，再应用所有增量备份，最后恢复未备份的 Binlog。
     </li>
     <li>
      <strong>
       Binlog 备份
      </strong>
      ：结合全量备份可实现时间点恢复。需确保开启 Binlog（
      <code>
       log_bin=ON
      </code>
      ）。
     </li>
    </ul>
    <p>
     <br/>
     <br/>
    </p>
    <h3>
     <a id="3__16">
     </a>
     3. 恢复方法
    </h3>
    <ul>
     <li>
      <strong>
       逻辑备份恢复
      </strong>
      ：使用
      <code>
       mysql
      </code>
      命令执行备份文件中的 SQL 语句进行恢复。
     </li>
     <li>
      <strong>
       物理备份恢复
      </strong>
      ：以
      <code>
       XtraBackup
      </code>
      为例，需要先准备备份数据，再将文件复制到 MySQL 数据目录。
     </li>
     <li>
      <strong>
       时间点恢复（PITR）
      </strong>
      ：在误操作等场景下，可恢复到误操作前的状态。先恢复最近的全量备份，再应用 Binlog 到误操作前的时刻。
     </li>
    </ul>
    <p>
     <br/>
     <br/>
    </p>
    <h3>
     <a id="4__22">
     </a>
     4. 备份工具对比
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        工具
       </th>
       <th>
        类型
       </th>
       <th>
        特点
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         mysqldump
        </strong>
       </td>
       <td>
        逻辑备份
       </td>
       <td>
        兼容性好，支持单表备份，速度慢，适合小数据量。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         mydumper
        </strong>
       </td>
       <td>
        逻辑备份
       </td>
       <td>
        多线程备份，效率高，支持并行导出和压缩。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         XtraBackup
        </strong>
       </td>
       <td>
        物理备份
       </td>
       <td>
        热备份，支持增量备份，适合大数据量，恢复速度快。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         MySQL Shell
        </strong>
       </td>
       <td>
        逻辑/物理
       </td>
       <td>
        8.0+ 支持，可并行备份（
        <code>
         util.dumpInstance()
        </code>
        ），集成度高。
       </td>
      </tr>
     </tbody>
    </table>

     <br/>

    <h3>
     <a id="5__32">
     </a>
     5. 逻辑备份
    </h3>
    <pre><code class="prism language-bash"><span class="token comment"># 备份整个数据库</span>
mysqldump <span class="token parameter variable">-u</span> root <span class="token parameter variable">-p</span> <span class="token parameter variable">--databases</span> shop_db <span class="token operator">&gt;</span> shop_db_backup.sql

<span class="token comment"># 仅备份表</span>
mysqldump <span class="token parameter variable">-u</span> root <span class="token parameter variable">-p</span> shop_db orders <span class="token operator">&gt;</span> orders_backup.sql

</code></pre>
    <p>
     <br/>
     <br/>
    </p>
    <h3>
     <a id="6__43">
     </a>
     6. 恢复
    </h3>
    <pre><code class="prism language-bash"><span class="token comment"># 逻辑备份恢复整个数据库</span>
mysql <span class="token parameter variable">-u</span> root <span class="token parameter variable">-p</span> shop_db <span class="token operator">&lt;</span> shop_db_backup.sql

<span class="token comment"># 逻辑备份恢复单表</span>
mysql <span class="token parameter variable">-u</span> root <span class="token parameter variable">-p</span> shop_db <span class="token parameter variable">-e</span> <span class="token string">"source orders_backup.sql"</span>

</code></pre>
