    <h2>
     <a id="_3">
     </a>
     近距离无线通信技术
    </h2>
    <ul>
     <li>
      近距离无线通信技术是实现无线局域网、无线个域网中节点、设备组网的常用技术，用于将传感器，RFID以及手机等移动感知设备的感知数据进行数据汇聚，并通过网关传输到上层网络中。
     </li>
     <li>
      <strong>
       近距离无线通信技术通常有Wi-Fi技术、蓝牙技术、（LiFi）和ZigBee技术
      </strong>
      。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b6c85bdbf3e8fc174b656117418ae22b.png"/>
     </li>
    </ul>
    <h3>
     <a id="WIFI_7">
     </a>
     WIFI
    </h3>
    <ul>
     <li>
      无线保真Wi-Fi(Wireless Fidelity)技术是一种将PC、笔记本、移动手持设备（如PDA、手机）等终端以无线方式互相连接的
      <strong>
       短距离无线电通信技术
      </strong>
      。
     </li>
     <li>
      Wi-Fi是一个无线网络通信技术的品牌，由
      <strong>
       Wi-Fi联盟(Wi-Fi Alliance)所持有
      </strong>
      。
      <strong>
       目的是改善基于IEEE 802.11标准的无线网络产品之间的互通性。
      </strong>
     </li>
     <li>
      Wi-Fi于1999年由Wi-Fi联盟发布，WI-FI联盟最初为无线以太网相容联盟WECA（Wireless Ethernet Compatibility Alliance），因此，Wi-Fi技术又称为无线相容性认证技术。
     </li>
    </ul>

    <h4>
     <a id="WiFi_15">
     </a>
     Wi-Fi的信道
    </h4>
    <ul>
     <li>
      工作频段：WLAN工作频段：2.4Ghz和5Ghz
     </li>
    </ul>
    <ol>
     <li>
      工作在2.4Ghz频段的设备：ZigBee设备、微波炉、双向寻呼系统、脉冲雷达系统、支持802.11a/b/g/n/ac协议的WLAN设备
     </li>
     <li>
      5Ghz频段：支持802.11/n/ac协议的WLAN设备、其他多媒体通信设备
     </li>
    </ol>
    <hr/>
    <ul>
     <li>
      信道的划分：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/954303909b771f6e7c3f52060b72142d.png"/>
     </li>
     <li>
      2.4GHz信道的划分：
     </li>
    </ul>
    <ol>
     <li>
      2.4Ghz每个信道频段宽度为20MHz
     </li>
     <li>
      在多个频道同时工作的情况下，为保证频道之间不相互干扰
      <strong>
       要求两个频道的中心频率间隔不能低于25MHz
      </strong>
      ，在实际使用中一般选择
      <strong>
       1，6，11三个频道进行频率规划
      </strong>
      ，同频道间必须两两隔离。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8fc92627a4f5e753a4bae912a61d8236.png"/>
     </li>
    </ol>
    <ul>
     <li>
      <strong>
       信道的使用：相邻的AP应尽量避免使用同一频点应主要频率间隔和功率控制
      </strong>
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/30baa3c3402d3ed32841bec0250e3daa.png"/>
     </li>
     <li>
      相邻的AP应尽量避免使用同一频点，同一区域相同频率覆盖的AP数量建议不超过3个，由于无线信号的传播是3维球形，在同一建筑物内部部署多个AP时，可以采用下图频率规划，避免干扰。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e585609583abeeddfd649cab40415b4a.png"/>
     </li>
    </ul>
    <h4>
     <a id="WiFi_30">
     </a>
     Wi-Fi技术的术语
    </h4>
    <ul>
     <li>
      <p>
       LAN：局域网，路由和主机组成的内部局域网，一般为有线网络。
      </p>
     </li>
     <li>
      <p>
       WAN：广域网，是外部一个更大的局域网。
      </p>
     </li>
     <li>
      <p>
       WLAN（Wireless LAN，即无线局域网）：LAN是局域网，其实大多数指的是有线网络中的局域网，无线网络中的局域网，一般用WLAN。
      </p>
     </li>
     <li>
      <p>
       AP（Access point，访问点，接入点）：是一个无线网络中的特殊节点，通过这个节点，无线网络中的其它类型节点可以和无线网络外部以及内部进行通信。
       <strong>
        AP通过无线链路和STA进行通信。AP上行方向与AC通过有线链路连接。
       </strong>
      </p>
     </li>
     <li>
      <p>
       功能：中继、桥连、主从模式；
      </p>
     </li>
     <li>
      <p>
       Station（工作站）：表示连接到无线网络中的设备，这些设备通过AP，可以和内部其它设备或者无线网络外部通信。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/8c62df14df7bf7cc4924a6bb6737d1ed.png"/>
      </p>
     </li>
     <li>
      <p>
       Assosiate：连接，如果一个Station想要加入到无线网络中，需要和这个无线网络中的AP关联（即Assosiate）。
      </p>
     </li>
     <li>
      <p>
       SSID（服务集标识）：用来
       <strong>
        标识一个无线网络
       </strong>
       。可看作局域网的名称，
       <strong>
        每个无线网络都有它自己的SSID，32个字符
       </strong>
       。在同一SS内的所有STA和AP必须具有相同的SSID，否则无法进行通信。
      </p>
     </li>
     <li>
      <p>
       BSSID（基本服务集标识）：
       <strong>
        用来标识一个BSS，其格式和MAC地址一样，是48位的地址格式
       </strong>
       。一般来说，它就是所处的无线接入点的MAC地址。
      </p>
     </li>
     <li>
      <p>
       基本服务集（BSS）：基本服务集是8O2.11LAN的基本组成模块。
       <strong>
        能互相进行无线通信的STA可以组成一个BSS（BasiCService Set）
       </strong>
       。如果一个站移出BSS的覆盖范围，它将不能再与BSS的其它成员通信。
      </p>
     </li>
     <li>
      <p>
       扩展服务集（ESS）：多个BSS可以构成一个扩展网络，称为扩展服务集（ESS）网络，一个ESS网络内部的STA可以互相通信，是采用相同的SSID的多个BSS形成的更大规模的虚拟BSS。连接BSS的组件称为分布式系统（Distribution SystemDS）。
       <br/>
       <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c9e82ebf80efcfa0dcc7578cc2b93a4e.png"/>
      </p>
     </li>
     <li>
      <p>
       BSSID实际上就是AP的MAC地址，用来标识AP管理的BSS，在同一个AP内BSSID和SSID一一映射。在一个ESS内SSID是相同的，但对于ESS内的每个AP与之对应的BSSID是不相同的。如果一个AP可以同时支持多个SSID的话，则AP会分配不同的BSSID来对应这些SSID。
      </p>
     </li>
     <li>
      <p>
       DS（Distributed System）：分布式系统。分步式系统骨干网络（例如以太网）做为媒介，连接各个接入点，每个接入点与其内的station可构成BSS，各个接入点中的桥接控制器有到达骨干网络和其内部BSS无线网的接口（类似两个MAC地址），station通信需要通过分布式系统。
      </p>
     </li>
    </ul>

      Wi-Fi的工作模式：工作站Station、无线接入点-AP
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/d806c04a0f4d26626f80aac987e8d068.png"/>

    <h5>
     <a id="Adhoc_53">
     </a>
     Ad-hoc模式
    </h5>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/68c527b60c2168c80a074140a1a57e96.png"/>
    </p>
    <ul>
     <li>
      由无线工作站组成，用于一台无线工作站和另一台或多台其他无线工作站的直接通讯，该网络
      <strong>
       无法接入到有线网络
      </strong>
      中，
      <strong>
       只能独立使用
      </strong>
      。
      <strong>
       无需AP，安全由各个客户端自行维护。
      </strong>
     </li>
     <li>
      采用这种拓扑结构的网络，
      <strong>
       各站点竞争公用信道
      </strong>
      ，但站点数过多时，信道竞争成为限制网络性能的要害，因此，这种拓扑结构比较
      <strong>
       适合小规模、小范围的WLAN系统组网
      </strong>
      。
     </li>
     <li>
      点对点模式中的一个节点必需能同时“看”到网络中的其他节点，否则就认为网络中断，因此对等网络
      <strong>
       只能用于少数用户的组网环境
      </strong>
      ，比如4至8个用户。
     </li>
    </ul>
    <h5>
     <a id="AP_58">
     </a>
     无线接入点-AP
    </h5>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/729e26b3f3072273c05d2223d73ea74e.png"/>
    </p>
    <ul>
     <li>
      WLAN终端设备：用户可通过内置
      <strong>
       WLAN无线模块
      </strong>
      的终端（如PC、PDA、手机等）或终端+
      <strong>
       WLAN网卡
      </strong>
      的方式获得互联网接入服务和数据通信服务。目前WLAN网卡主要支持IEEE802.11a/b/g协议，后续应能支持IEEE802.11n协议，同时应能支持IEEE802.11i协议进行认证加密。
     </li>
     <li>
      WLAN接入系统：由接入点设备AP和业务接入控制设备（AC）组成，完成WLAN用户的接入控制，WLAN接入认证点则由AP设备实现。
     </li>
    </ul>
    <h5>
     <a id="APFAT_AP_vs_FIT_AP_62">
     </a>
     AP：FAT AP vs FIT AP
    </h5>
    <ul>
     <li>
      接入控制器（AC）Access Controller：在无线局域网和外部网之间充当网关的功能。AC将来自不同AP的数据进行汇聚，与Internet相连。AC支持用户安全控制、业务控制、计费信息采集以及网络的监控。
     </li>
     <li>
      胖AP组网：
      <ul>
       <li>
        每个AP都是一个单独的节点，独立配置其信道和功率；安装简便
       </li>
       <li>
        每个AP独立工作，较难扩展到大型、连续、协调的无线局域网和增加高级应用
       </li>
       <li>
        每个AP都需要独立配置安全策略，如果AP数量增加，将会给网络管理、维护及升级带来较大的困难
       </li>
       <li>
        很难进行无线网络质量的优化数据的采集瘦
       </li>
      </ul>
     </li>
     <li>
      AP组网：
      <ul>
       <li>
        通过AC对AP群组进行自动信道分配和选择，及自动调整发射功率，降低AP之间的互干扰，提高网络动态覆盖特性
       </li>
       <li>
        支持二层/三层漫游切换
       </li>
       <li>
        容易实现非法AP检测和处理管理节点上移后，运维数据采集针对AC而非AP，解决了网管系统受限于AP处理能力和性能的问题
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     <a id="WiFi_78">
     </a>
     Wi-Fi的安全技术
    </h4>
    <ul>
     <li>
      加入网络之前需要进行身份认证。IEEE 802.11b标准定义
      <strong>
       开放式和共享密钥
      </strong>
      两种身份认证方法。
     </li>
     <li>
      由于Wi-Fi是采用射频技术进行网络连接及传输的开放式物理系统，其
      <strong>
       安全性主要表现在物理层、链路层和网络层
      </strong>
      。
      <strong>
       核心是数据链路层安全
      </strong>
      。WEP（Wired Equivalent Privacy，WEP）
      <ul>
       <li>
        <strong>
         IEEE 802.11b标准规定了一种称为有线等价保密协议WEP的加密方案，提供了确保无线局域网数据流的安全机制。
        </strong>
       </li>
       <li>
        WEP采用的加密算法是由Ron Rivets of RSA Data Security Inc在1987年设计的40位的RC4加密算法。RC4算法属于对称流密码，支持可变长度密钥
       </li>
      </ul>
     </li>
     <li>
      对于64位加密，加密密钥为10个十六进制字符（0-9和A-F）或5个 ASCII 字符。
     </li>
     <li>
      对于128位加密，加密密钥为26个十六进制字符或13个ASCII字符。
     </li>
     <li>
      64位加密又称40位加密，128位加密又称104位加密。
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      <strong>
       WPA（Wireless Protected Access，无线保护访问）是由Wi-Fi 联盟提出的一个无线安全访问保护协议。
      </strong>
     </li>
    </ul>
    <blockquote>
     <ul>
      <li>
       WPA协议的提出旨在克服所有WEP协议的安全缺陷，WPA协议大大改进了之前的无线网络安全保护能力和访问控制技术，使无线网络数据的安全级别提高。
      </li>
      <li>
       WPA 主要解决了WEP中在客户端与缺乏身份认证的访问点之间使用相同静态密钥和网络接入时身份认证方面存在的缺陷问题。
      </li>
     </ul>
    </blockquote>

    <h3>
     <a id="Bluetooth_100">
     </a>
     Bluetooth
    </h3>
    <h4>
     <a id="_101">
     </a>
     蓝牙技术概论
    </h4>
    <ul>
     <li>
      蓝牙（Bluetooth）是一种支持设备短距离通信（10cm-10m）的
      <strong>
       无线个域网WPAN技术
      </strong>
      ，能在包括移动电话、PDA、无线耳机、笔记本计算机、相关外设等众多设备之间进行无线信息交换。
     </li>
     <li>
      利用蓝牙技术，能够有效地简化移动通信终端设备之间的通信，也能够简化设备与Internet之间的通信，从而使数据传输变得更加迅速、高效。
     </li>
     <li>
      蓝牙是一个开放性的
      <strong>
       无线通信标准（802.15）
      </strong>
      ，设计者的初衷是用隐形的连接线代替线缆。
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/91c3ed334be2be1ee6fa5a0d19b0519f.png"/>
    </p>
    <ul>
     <li>
      蓝牙 4.0 是第一个蓝牙综合协议规范，将三种规格集成在一起。其中最重要的变化就是 BLE（Bluetooth Low Energy）
      <strong>
       低功耗功能，提出了低功耗蓝牙、传统蓝牙和高速蓝牙三种模式
      </strong>
      。
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      蓝牙Mesh网络是用于建立多对多设备通信的低能耗蓝牙（Bluetooth Low Energy，Bluetooth LE）新的网络拓扑。它允许用户创建基于多个设备的大型网络，网络可以包含数十台，数百甚至数千台蓝牙Mesh设备，这些设备之间可以相互进行信息的传递。
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      Mesh 网状网络：实现物联网的关键“钥匙”
     </li>
    </ul>

       对物理层和数据链路层
      </strong>
      进行了标准化，

    <h4>

     蓝牙的组网技术
    </h4>
    <h5>
     <a id="piconet_116">
     </a>
     微微网piconet(了解)
    </h5>
    <ul>
     <li>
      1台主设备和1台以上从设备构成的网络称为微网（Piconet）。
     </li>
     <li>
      一个蓝牙单元作为微网的主单元，其余的可作为从单元看待。在一个微网中最多可有七个活动从单元。另外，更多的从单元被锁定在休眠状态中。这些处于休眠状态的从单元在该信道中不能被激活，但对主单元来讲它们仍由主单元同步。无论对激活或休眠状态来讲，信道访问都由主单元控制
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/f71538fecc796c89d56d8cf2c3a3ad00.png"/>
     </li>
     <li>
      一个微微网可由8个蓝牙设备组成。
     </li>
     <li>
      在同一个微微网中，主设备为所有的设备提供时钟和跳频同步序列。
     </li>
     <li>
      在同一个微微网中，所有的设备有同样的跳频序列。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/c1cbd9c760ff4f917518de3d3718e186.png"/>
     </li>
     <li>
      微微网中，所有设备的级别是相同的，具有相同的权限（如手机可通过蓝牙音箱播放音乐，同时与其他手机交换数据）。
     </li>
     <li>
      微微网A中的主设备可以成为微微网B的从设备
     </li>
     <li>
      每个微微网都有独立的跳频序列，微微网间不会发生跳频同步，无同频干扰。
     </li>
    </ul>
    <h5>
     <a id="scatternet_127">
     </a>
     散射网scatternet
    </h5>
    <ul>
     <li>
      散射网是由多个独立的非同步的微微网组成的，比单一微微网覆盖更多设备，更大范围
     </li>
     <li>
      一个散射网最多可连10个微微网
     </li>
     <li>
      不同微微网之间有互联的桥接设备，桥接设备可以由主设备或从设备充当
     </li>
     <li>
      一个设备不能同时担任两个微微网的主设备，
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/5a29615c679cefefaea2dee9467a24e2.png"/>

     可见光通信技术
    </h3>
    <h4>
     <a id="_159">
     </a>
     可见光通信概述
    </h4>
    <ol>
     <li>
      可见光通信技术（Visible Light Communication，VLC）是指
      <strong>
       利用可见光波段的光作为信息载体
      </strong>
      ，无需光纤等有线信道的传输介质，
      <strong>
       在空气中直接传输光信号的通信方式
      </strong>
     </li>
     <li>
      可见光通信技术绿色低碳、可实现近乎零耗能通信，有效避免无线电通信电磁信号泄露等弱点，快速构建抗干扰、抗截获的安全信息空间。
     </li>
     <li>
      可见光通信Li-Fi作为Wi-Fi的有效补充手段，可用于对射频无线通信敏感场合，如机舱、矿井、核电站和医疗设备工作区域。
     </li>
    </ol>
    <h4>
     <a id="_163">
     </a>
     可见光通信优缺点
    </h4>
    <hr/>
    <ul>
     <li>
      Li-Fi优点：丰富的频谱资源、低成本、安全性、无电磁辐射
     </li>
     <li>
      Li-Fi缺点：易被遮挡、光源间断性问题、频繁切换、环境干扰、用户友好的反向通信
     </li>
    </ul>
    <hr/>
    <h4>

    </h4>

     </li>
     <li>
      码间干扰消除技术
     </li>
     <li>
      全双工通信
     </li>
    </ol>
    <h4>
     <a id="_172">
     </a>
     可见光通信应用
    </h4>
    <ul>
     <li>
      随着LED在照明、显示上替代传统光源，使得这些设施在原有基础上具备传输信息的功能。另外，由于图像传感器在VLC领域的应用，使得接收端除了能够接收到数据外还能够准确判断发射端与接收端的相对位置，这就为VLC应用于室内导航、机器人或车辆之间的精确控制、准确的位置测量等提供了可能。
