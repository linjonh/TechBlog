    <h2>
     Jenkins简介
    </h2>
    <blockquote>
     <p>
      Jenkins 是一款广泛使用的开源持续集成和持续交付（CI/CD）工具，以下是对它的详细介绍：
     </p>
     <h3>
      基本信息
     </h3>
     <ul>
      <li>
       <strong>
        起源与发展
       </strong>
       ：Jenkins 最早起源于 Hudson 项目，后来从 Hudson 项目中分离出来独立发展。自 2011 年正式成为独立项目后，Jenkins 凭借其强大的功能和活跃的社区支持，迅速成为全球最受欢迎的 CI/CD 工具之一，在软件开发领域得到了广泛的应用和推广。
      </li>
      <li>
       <strong>
        主要功能
       </strong>
       ：核心功能是持续集成，它可以定期自动从代码仓库拉取最新代码，然后执行构建、测试等一系列任务。并且支持持续交付和持续部署，能将构建好的软件自动部署到测试环境或生产环境，实现软件的快速迭代和交付。还提供了丰富的插件扩展机制，通过安装各种插件，可以与众多工具和平台集成，如代码仓库 Git、Subversion，测试框架 JUnit、TestNG，容器平台 Docker 等，满足不同项目的多样化需求。
      </li>
     </ul>
     <h3>
      特点
     </h3>
     <ul>
      <li>
       <strong>
        高度可扩展性
       </strong>
       ：拥有大量的插件，涵盖了从代码管理、构建工具、测试框架到部署工具等各个方面。用户可以根据具体项目需求，轻松安装和配置插件，扩展 Jenkins 的功能，使其能够适应不同技术栈和开发流程的项目。
      </li>
      <li>
       <strong>
        多平台支持
       </strong>
       ：可以在多种操作系统上运行，包括 Windows、Linux、Mac OS 等。这使得它能够很好地适配不同的开发和运维环境，无论团队使用何种操作系统，都能方便地部署和使用 Jenkins。
      </li>
      <li>
       <strong>
        易于配置和使用
       </strong>
       ：提供了直观的 Web 界面，用户通过浏览器即可方便地进行各种配置和操作。在 Web 界面中，用户可以轻松创建和管理任务（Jobs），配置构建步骤、触发条件、环境变量等，无需编写复杂的脚本或命令，降低了使用门槛。
      </li>
      <li>
       <strong>
        强大的社区支持
       </strong>
       ：拥有庞大且活跃的社区，这意味着用户在使用过程中遇到问题时，能够很容易地在社区中找到解决方案、获取相关文档和教程，也可以参与社区的讨论和贡献，推动 Jenkins 的不断发展和完善。
      </li>
     </ul>
     <h3>
      应用场景
     </h3>
     <ul>
      <li>
       <strong>
        软件开发流程集成
       </strong>
       ：在软件开发的整个生命周期中，Jenkins 可以与开发、测试、运维等各个环节的工具和流程紧密集成。例如，在开发阶段，它可以与代码编辑器集成，实时检测代码提交并触发构建；在测试阶段，能与各种测试工具配合，自动执行单元测试、集成测试等；在运维阶段，可与部署工具结合，实现自动化部署。
      </li>
      <li>
       <strong>
        多项目管理与协作
       </strong>
       ：对于拥有多个项目的团队或企业，Jenkins 可以统一管理所有项目的 CI/CD 流程。通过创建不同的任务和视图，对每个项目进行独立配置和管理，同时又能在一个平台上进行整体监控和调度，方便团队成员之间的协作和沟通。
      </li>
      <li>
       <strong>
        持续部署与自动化运维
       </strong>
       ：Jenkins 能够实现软件的持续部署，将构建好的应用程序自动部署到生产环境中，减少人工干预，提高部署效率和准确性。配合自动化运维工具，还可以实现服务器的自动配置、环境搭建等任务，提升整个运维过程的自动化水平。
      </li>
     </ul>
    </blockquote>
    <h3>
    </h3>
    <h2>
     Maven 简介
    </h2>
    <blockquote>
     <p>
      Maven 是一个基于项目对象模型（POM，Project Object Model）的项目管理和构建自动化工具，由 Apache 软件基金会开发。它主要用于 Java 项目，但也可以用于其他语言的项目。Maven 的核心功能和优势体现在以下几个方面：
     </p>
     <ul>
      <li>
       <strong>
        依赖管理
       </strong>
       ：Maven 能够自动下载项目所需的依赖库，并处理依赖之间的版本冲突。开发者只需在项目的 POM 文件中声明所需的依赖，Maven 会从中央仓库或其他指定的仓库中下载这些依赖。
      </li>
      <li>
       <strong>
        项目构建
       </strong>
       ：Maven 定义了一套标准的项目构建生命周期，包括清理、编译、测试、打包、部署等阶段。通过执行简单的命令，开发者可以完成整个项目的构建过程，保证构建过程的一致性和可重复性。
      </li>
      <li>
       <strong>
        项目管理
       </strong>
       ：Maven 提供了统一的项目结构和配置文件（POM.xml），使得项目的管理和维护更加方便。团队成员可以根据 POM 文件快速了解项目的依赖、插件和构建配置等信息。
      </li>
     </ul>
     <h3 style="background-color:transparent">
      Maven 使用方法
     </h3>
     <h4 style="background-color:transparent">
      1. 安装 Maven
     </h4>
     <ul>
      <li>
       <strong>
        下载
       </strong>
       ：访问 Maven 官方网站（
       <a href="https://maven.apache.org/download.cgi" rel="nofollow" title="Download Apache Maven – Maven">
        Download Apache Maven – Maven
       </a>
       ），下载适合你操作系统的二进制发行版。
      </li>
      <li>
       <strong>
        解压
       </strong>
       ：将下载的压缩包解压到指定目录，例如
       <code>
        C:\apache-maven-3.8.6
       </code>
       （Windows）或
       <code>
        /usr/local/apache-maven-3.8.6
       </code>
       （Linux）。
      </li>
      <li>
       <strong>
        配置环境变量
       </strong>
       ：
       <ul>
        <li>
         <strong>
          Windows
         </strong>
         ：在系统环境变量中添加
         <code>
          MAVEN_HOME
         </code>
         变量，值为 Maven 的安装目录，然后在
         <code>
          Path
         </code>
         变量中添加
         <code>
          %MAVEN_HOME%\bin
         </code>
         。
        </li>
        <li>
         <strong>
          Linux
         </strong>
         ：编辑
         <code>
          ~/.bashrc
         </code>
         或
         <code>
          /etc/profile
         </code>
         文件，添加以下内容：
        </li>
       </ul>
      </li>
     </ul>
     <pre><code>export MAVEN_HOME=/usr/local/apache-maven-3.8.6
export PATH=$PATH:$MAVEN_HOME/bin
</code></pre>
     <p>
      保存文件后，执行
      <code>
       source ~/.bashrc
      </code>
      或
      <code>
       source /etc/profile
      </code>
      使配置生效。
     </p>
     <ul>
      <li>
       <strong>
        验证安装
       </strong>
       ：打开终端或命令提示符，执行
       <code>
        mvn -version
       </code>
       命令，如果显示 Maven 的版本信息，则说明安装成功。
      </li>
     </ul>
     <h4 style="background-color:transparent">
      2. 创建 Maven 项目
     </h4>
     <ul>
      <li>
       <strong>
        使用命令行创建
       </strong>
       ：打开终端，执行以下命令创建一个简单的 Java 项目：
      </li>
     </ul>
     <pre><code>mvn archetype:generate -DgroupId=com.example -DartifactId=my-project -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false
</code></pre>
     <ul>
      <li>
       <code>
        groupId
       </code>
       ：项目的组 ID，通常是公司或组织的域名倒序，如
       <code>
        com.example
       </code>
       。
      </li>
      <li>
       <code>
        artifactId
       </code>
       ：项目的唯一标识符，如
       <code>
        my-project
       </code>
       。
      </li>
      <li>
       <code>
        archetypeArtifactId
       </code>
       ：项目的原型，
       <code>
        maven-archetype-quickstart
       </code>
       是一个简单的 Java 项目原型。
      </li>
      <li>
       <code>
        interactiveMode=false
       </code>
       ：表示不使用交互式模式。
      </li>
      <li>
       <strong>
        使用 IDE 创建
       </strong>
       ：大多数集成开发环境（如 IntelliJ IDEA、Eclipse）都支持创建 Maven 项目。在 IDE 中选择创建 Maven 项目，按照向导填写
       <code>
        groupId
       </code>
       、
       <code>
        artifactId
       </code>
       等信息即可。
      </li>
     </ul>
     <h4 style="background-color:transparent">
      3. 配置 POM 文件
     </h4>
     <p>
      POM 文件（
      <code>
       pom.xml
      </code>
      ）是 Maven 项目的核心配置文件，以下是一个简单的 POM 文件示例：
     </p>
     <pre><code>&lt;project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd"&gt;
    &lt;modelVersion&gt;4.0.0&lt;/modelVersion&gt;

    &lt;groupId&gt;com.example&lt;/groupId&gt;
    &lt;artifactId&gt;my-project&lt;/artifactId&gt;
    &lt;version&gt;1.0-SNAPSHOT&lt;/version&gt;

    &lt;dependencies&gt;
        &lt;!-- 添加项目依赖 --&gt;
        &lt;dependency&gt;
            &lt;groupId&gt;junit&lt;/groupId&gt;
            &lt;artifactId&gt;junit&lt;/artifactId&gt;
            &lt;version&gt;4.13.2&lt;/version&gt;
            &lt;scope&gt;test&lt;/scope&gt;
        &lt;/dependency&gt;
    &lt;/dependencies&gt;

    &lt;build&gt;
        &lt;plugins&gt;
            &lt;!-- 配置编译插件 --&gt;
            &lt;plugin&gt;
                &lt;groupId&gt;org.apache.maven.plugins&lt;/groupId&gt;
                &lt;artifactId&gt;maven-compiler-plugin&lt;/artifactId&gt;
                &lt;version&gt;3.8.1&lt;/version&gt;
                &lt;configuration&gt;
                    &lt;source&gt;1.8&lt;/source&gt;
                    &lt;target&gt;1.8&lt;/target&gt;
                &lt;/configuration&gt;
            &lt;/plugin&gt;
        &lt;/plugins&gt;
    &lt;/build&gt;
&lt;/project&gt;
</code></pre>
     <ul>
      <li>
       <strong>
        <code>
         &lt;dependencies&gt;
        </code>
       </strong>
       ：用于声明项目的依赖，每个
       <code>
        &lt;dependency&gt;
       </code>
       元素包含
       <code>
        groupId
       </code>
       、
       <code>
        artifactId
       </code>
       和
       <code>
        version
       </code>
       等信息。
      </li>
      <li>
       <strong>
        <code>
         &lt;build&gt;
        </code>
       </strong>
       ：用于配置项目的构建过程，包括插件的使用。
      </li>
     </ul>
     <h4 style="background-color:transparent">
      4. 执行 Maven 命令
     </h4>
     <ul>
      <li>
       <strong>
        清理项目
       </strong>
       ：执行
       <code>
        mvn clean
       </code>
       命令，删除项目中的编译输出文件，如
       <code>
        target
       </code>
       目录。
      </li>
      <li>
       <strong>
        编译项目
       </strong>
       ：执行
       <code>
        mvn compile
       </code>
       命令，将项目的源代码编译为字节码文件。
      </li>
      <li>
       <strong>
        运行测试
       </strong>
       ：执行
       <code>
        mvn test
       </code>
       命令，运行项目中的单元测试用例。
      </li>
      <li>
       <strong>
        打包项目
       </strong>
       ：执行
       <code>
        mvn package
       </code>
       命令，将项目打包为 JAR 或 WAR 文件。
      </li>
      <li>
       <strong>
        安装项目到本地仓库
       </strong>
       ：执行
       <code>
        mvn install
       </code>
       命令，将项目打包并安装到本地 Maven 仓库，以便其他项目可以引用。
      </li>
      <li>
       <strong>
        部署项目到远程仓库
       </strong>
       ：执行
       <code>
        mvn deploy
       </code>
       命令，将项目打包并部署到远程 Maven 仓库。
      </li>
     </ul>
     <p>
      通过以上步骤，你可以使用 Maven 来管理和构建 Java 项目。在实际开发中，你可以根据项目的需求，进一步配置 POM 文件，添加更多的依赖和插件。
     </p>
    </blockquote>
