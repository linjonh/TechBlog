    <h2>
     <a id="Spring_Boot_StarterStarter_0">
     </a>
     深入理解Spring Boot Starter及如何自定义Starter
    </h2>
    <p>
     Spring Boot Starter本质上是一种依赖管理和自动配置机制。通过引入一个Starter依赖，可以自动引入并配置一系列相关组件，极大地简化开发流程。比如使用
     <code>
      spring-boot-starter-web
     </code>
     ，可以自动引入Spring MVC、Tomcat服务器、JSON解析器等，无需手动逐个配置。
    </p>
    <h3>
     <a id="Starter_4">
     </a>
     Starter的自动配置原理
    </h3>
    <p>
     <strong>
      Spring Boot 会扫描所有
      <code>
       classpath
      </code>
      下 JAR 包中的
      <code>
       META-INF/spring.factories
      </code>
      文件
     </strong>
     ，并按照其中的配置进行自动加载。
    </p>
    <h4>
     <a id="__8">
     </a>
     <strong>
      📌 详细解析
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        扫描机制
       </strong>
      </p>
      <ul>
       <li>
        Spring Boot 使用
        <strong>
         <code>
          SpringFactoriesLoader
         </code>
        </strong>
        类来加载
        <code>
         spring.factories
        </code>
        文件中的配置。
       </li>
       <li>
        它会扫描
        <strong>
         所有
        </strong>
        依赖（包括
        <code>
         libs/
        </code>
        和
        <code>
         dependencies
        </code>
        目录中的 JAR 包），寻找
        <code>
         META-INF/spring.factories
        </code>
        ，然后合并所有的配置项。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        合并机制
       </strong>
      </p>
      <ul>
       <li>
        如果多个 JAR 包（如
        <code>
         spring-boot-starter-web
        </code>
        、
        <code>
         spring-boot-starter-data-jpa
        </code>
        等）都定义了
        <code>
         spring.factories
        </code>
        ，Spring Boot 会把
        <strong>
         所有 JAR 包中的配置合并
        </strong>
        ，然后执行加载。
       </li>
       <li>
        <strong>
         示例：
        </strong>
        <ul>
         <li>
          <code>
           spring-boot-autoconfigure.jar
          </code>
          的
          <code>
           META-INF/spring.factories
          </code>
          可能包含：
          <pre><code class="prism language-properties">org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration,\
org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration
</code></pre>
         </li>
         <li>
          另一个 JAR 包
          <code>
           my-custom-starter.jar
          </code>
          可能包含：
          <pre><code class="prism language-properties">org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
com.example.custom.MyCustomAutoConfiguration
</code></pre>
         </li>
         <li>
          <strong>
           Spring Boot 会把这两部分合并
          </strong>
          ，最终加载：
          <pre><code class="prism language-properties">org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration,\
org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration,\
com.example.custom.MyCustomAutoConfiguration
</code></pre>
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        加载顺序
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         Spring Boot 不保证
         <code>
          spring.factories
         </code>
         配置的执行顺序
        </strong>
        ，如果多个 JAR 包中存在相同的 Key（比如
        <code>
         EnableAutoConfiguration
        </code>
        ），它们都会被加载，但
        <strong>
         执行顺序不可预测
        </strong>
        。
       </li>
       <li>
        <strong>
         如果需要控制加载顺序，可以使用
         <code>
          @AutoConfigureBefore
         </code>
         和
         <code>
          @AutoConfigureAfter
         </code>
        </strong>
        。
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="METAINFspringfactories__42">
     </a>
     <strong>
      <code>
       META-INF/spring.factories
      </code>
      文件的作用
     </strong>
    </h4>
    <p>
     <code>
      META-INF/spring.factories
     </code>
     是
     <strong>
      Spring Boot 的自动配置机制
     </strong>
     之一，它的主要作用是：
    </p>
    <ul>
     <li>
      <strong>
       启用 Spring Boot 的自动配置（AutoConfiguration）
      </strong>
     </li>
     <li>
      <strong>
       自动注册 Spring 组件（如监听器、工厂类等）
      </strong>
     </li>
     <li>
      <strong>
       扩展 Spring Boot 功能
      </strong>
     </li>
    </ul>
    <h4>
     <a id="_springfactories__48">

     <strong>

    <h4>

     </a>
     <strong>
      📌
      <code>

      </code>

     </strong>
    </h4>
    <p>
     Spring Boot 使用
     <code>
      SpringFactoriesLoader.loadFactoryNames(Class&lt;?&gt; factoryType, ClassLoader classLoader)

     方法来扫描所有
     <code>
      META-INF/spring.factories
     </code>
     ：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">static</span> <span class="token class-name">List</span><span class="token generics"><span class="token punctuation">&lt;</span><span class="token class-name">String</span><span class="token punctuation">&gt;</span></span> <span class="token function">loadFactoryNames</span><span class="token punctuation">(</span><span class="token class-name">Class</span><span class="token generics"><span class="token punctuation">&lt;</span><span class="token operator">?</span><span class="token punctuation">&gt;</span></span> factoryType<span class="token punctuation">,</span> <span class="token class-name">ClassLoader</span> classLoader<span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token class-name">String</span> factoryTypeName <span class="token operator">=</span> factoryType<span class="token punctuation">.</span><span class="token function">getName</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token keyword">return</span> <span class="token function">loadSpringFactories</span><span class="token punctuation">(</span>classLoader<span class="token punctuation">)</span><span class="token punctuation">.</span><span class="token function">getOrDefault</span><span class="token punctuation">(</span>factoryTypeName<span class="token punctuation">,</span> <span class="token class-name">Collections</span><span class="token punctuation">.</span><span class="token function">emptyList</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

     <li>
      <code>

     </li>

    <h4>
     <a id="__springfactories__72">
     </a>
     <strong>
      📌 如何避免
      <code>
       spring.factories
      </code>
      冲突？
     </strong>
    </h4>
    <p>
     如果多个 JAR 包的
     <code>
      spring.factories
     </code>
     中包含
     <strong>
      相同的 Key
     </strong>
     ，但不同的实现（比如多个 JAR 里都有
     <code>
      EnableAutoConfiguration
     </code>
     ），可能会导致：
    </p>
    <ul>
     <li>
      <strong>
       重复加载
      </strong>
      （多个类都被注册）
     </li>
     <li>
      <strong>
       冲突
      </strong>
      （不同的类覆盖了同一个 Bean）
     </li>
    </ul>
    <h5>
     <a id="_77">
     </a>
     <strong>
      解决方案
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        使用
        <code>
         spring-boot-autoconfigure
        </code>
        提供的新方式
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         Spring Boot 2.7+ 推荐使用
         <code>
          META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
         </code>
        </strong>
       </li>
       <li>
        这样可以更好地控制哪些配置应该被加载，不会导致
        <code>
         spring.factories
        </code>
        的全局合并问题。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        使用
        <code>
         @ConditionalOnMissingBean
        </code>
        避免重复加载
       </strong>
      </p>
      <ul>
       <li>
        在自动配置类中使用
        <code>
         @ConditionalOnMissingBean
        </code>
        ，避免同一个 Bean 被多次注册：
       </li>
      </ul>
      <pre><code class="prism language-java"><span class="token annotation punctuation">@Configuration</span>
<span class="token annotation punctuation">@ConditionalOnClass</span><span class="token punctuation">(</span><span class="token class-name">MyService</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">)</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">MyAutoConfiguration</span> <span class="token punctuation">{<!-- --></span>
    <span class="token annotation punctuation">@Bean</span>
    <span class="token annotation punctuation">@ConditionalOnMissingBean</span>
    <span class="token keyword">public</span> <span class="token class-name">MyService</span> <span class="token function">myService</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">return</span> <span class="token keyword">new</span> <span class="token class-name">MyService</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        使用
        <code>
         @AutoConfigureBefore
        </code>
        或
        <code>
         @AutoConfigureAfter
        </code>
       </strong>
      </p>
      <ul>
       <li>
        控制自动配置类的加载顺序：
       </li>
      </ul>
      <pre><code class="prism language-java"><span class="token annotation punctuation">@Configuration</span>
<span class="token annotation punctuation">@AutoConfigureBefore</span><span class="token punctuation">(</span><span class="token class-name">DataSourceAutoConfiguration</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">)</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">MyCustomAutoConfiguration</span> <span class="token punctuation">{<!-- --></span>
    <span class="token comment">// 自定义配置</span>
<span class="token punctuation">}</span>
</code></pre>
     </li>
    </ol>

    <h4>
     <a id="__108">
     </a>
     <strong>
      📌 结论
     </strong>
    </h4>
    <p>
     ✅
     <strong>
      Spring Boot 确实会扫描所有 JAR 包的
      <code>
       META-INF/spring.factories
      </code>
      文件，并合并所有配置项
     </strong>
     。
     <br/>
     ✅
     <strong>
      这使得 Starter 组件可以自动注册需要的 Bean 和配置，但也可能导致 Bean 冲突或重复加载的问题
     </strong>
     。
     <br/>
     ✅
     <strong>
      Spring Boot 2.7+ 推荐使用
      <code>
       AutoConfiguration.imports
      </code>
      作为更可控的替代方案
     </strong>
     。
    </p>

    <h3>
     <a id="Starter_173">

     五、Starter的命名规范建议
    </h3>
    <p>

     </strong>
     ，以提高可读性和易用性。
    </p>

     Spring Boot 官方 Starter 的命名规则通常是：
    </p>
    <pre><code>spring-boot-starter-&lt;技术名称&gt;
</code></pre>

    <h4>
     <a id="2__Starter__195">
     </a>
     <strong>
      2️⃣ 自定义 Starter 的推荐命名规则
     </strong>
    </h4>
    <p>
     对于
     <strong>
      自定义 Starter
     </strong>
     ，社区推荐使用：
    </p>
    <pre><code>&lt;公司/团队前缀&gt;-starter-&lt;功能名称&gt;
</code></pre>
    <p>
     或者：
    </p>
    <pre><code>&lt;组织名称&gt;-spring-boot-starter-&lt;功能名称&gt;
</code></pre>
    <h5>
     <a id="__205">
     </a>

     </strong>
    </h5>

       </td>

    <h5>
     <a id="__213">
     </a>
     <strong>
      📌 推荐命名规则
     </strong>
    </h5>
    <p>
     ✅
     <strong>
      避免以
      <code>
       spring-boot-starter-
      </code>
      开头
     </strong>
    </p>
    <ul>
     <li>
      这样可以
      <strong>
       避免和官方 Starter 混淆
      </strong>
      ，让用户明确知道它是
      <strong>
       第三方 Starter
      </strong>
      。
     </li>
     <li>
      <strong>
       推荐：
      </strong>
      <code>
       mycompany-starter-xxx
      </code>
      或
      <code>
       xxx-spring-boot-starter
      </code>
     </li>
     <li>
      <strong>
       不推荐：
      </strong>
      <code>
       spring-boot-starter-xxx
      </code>
      ❌（可能导致误以为是官方的）
     </li>
    </ul>
    <p>
     ✅
     <strong>
      前缀代表团队或组织名称
     </strong>
    </p>
    <ul>
     <li>
      例如：
      <ul>
       <li>
        <code>
         alibaba-starter-seata
        </code>
        （阿里巴巴 Seata 事务管理）
       </li>
       <li>
        <code>
         baidu-starter-paddlepaddle
        </code>
        （百度 PaddlePaddle AI 框架）
       </li>
       <li>
        <code>
         mycompany-starter-wechat
        </code>
        （企业内部的微信消息推送）
       </li>
      </ul>
     </li>
    </ul>
    <p>
     ✅
     <strong>
      后缀描述功能
     </strong>
    </p>
    <ul>
     <li>
      <code>
       starter-dingtalk
      </code>
      （钉钉通知）
     </li>
     <li>
      <code>
       starter-mq
      </code>
      （消息队列支持）
     </li>
     <li>
      <code>
       starter-log-enhancer
      </code>
      （日志增强）
     </li>
    </ul>

    <h4>
     <a id="3__232">
     </a>
     <strong>
      3️⃣ 约束和注意事项
     </strong>
    </h4>
    <h5>
     <a id="__233">
     </a>
     <strong>
      🚀 官方约束
     </strong>
    </h5>
    <p>
     Spring Boot
     <strong>
      不会
     </strong>
     强制要求 Starter 必须符合某种命名规则，但
     <strong>
      官方有一些约定
     </strong>
     ：
    </p>
    <ol>
     <li>
      <strong>
       官方 Starter 需要由 Spring 团队维护
      </strong>
      （放在
      <code>
       org.springframework.boot
      </code>
      下面）。
     </li>
     <li>
      <strong>
       社区或第三方 Starter 不能以
       <code>
        spring-boot-starter-
       </code>
       开头
      </strong>
      ，避免误导用户。
     </li>
    </ol>
    <h5>
     <a id="__238">
     </a>
     <strong>
      🚀 依赖命名
     </strong>
    </h5>
    <p>
     如果 Starter 需要提供
     <strong>
      自动配置
     </strong>
     ，通常需要提供：
    </p>
    <ul>
     <li>
      <strong>
       核心依赖（starter 依赖）
      </strong>
      ：
      <pre><code class="prism language-xml"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>artifactId</span><span class="token punctuation">&gt;</span></span>mycompany-starter-dingtalk<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>artifactId</span><span class="token punctuation">&gt;</span></span>
</code></pre>
     </li>
     <li>
      <strong>
       自动配置依赖（autoconfigure 依赖）
      </strong>
      ：
      <pre><code class="prism language-xml"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>artifactId</span><span class="token punctuation">&gt;</span></span>mycompany-starter-dingtalk-autoconfigure<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>artifactId</span><span class="token punctuation">&gt;</span></span>
</code></pre>
     </li>
    </ul>
    <p>
     这样可以
     <strong>
      拆分 Starter 的自动配置和核心功能
     </strong>
     ，提高灵活性。
    </p>
    <h4>
     <a id="__251">
     </a>
     <strong>
      📌 结论
     </strong>
    </h4>
    <p>
     ✅
     <strong>
      官方 Starter
     </strong>
     以
     <code>
      spring-boot-starter-
     </code>
     开头，后面跟具体技术，如
     <code>
      spring-boot-starter-web
     </code>
     。
     <br/>
     ✅
     <strong>
      自定义 Starter
     </strong>
     推荐
     <code>
      mycompany-starter-xxx
     </code>
     或
     <code>
      xxx-spring-boot-starter
     </code>
     ，避免与官方混淆。
     <br/>
     ✅
     <strong>
      组织前缀
     </strong>
     可加
     <code>
      mycompany-
     </code>
     或
     <code>
      com-example-
     </code>
     ，以区分不同的公司或团队。
     <br/>
     ✅
     <strong>
      最好拆分
      <code>
       starter
      </code>
      和
      <code>
       starter-autoconfigure
      </code>
      ，增强模块化和可扩展性
     </strong>
     。
    </p>
