    <p>
     废话不多说直接上代码：
    </p>
    <pre><code class="language-javascript">function fileToBase64(file) {
	return new Promise((resolve, reject) =&gt; {
		const reader = new FileReader();
		reader.readAsDataURL(file);
		reader.onload = function (event) {
			const base64Data = event.target.result.split(',')[1];
			resolve(base64Data);
		};
		reader.onerror = function (error) {
			reject(error);
		};
	});
}</code></pre>

    <p>
     <strong>
      <code>
       FileReader

     </strong>
     接口允许 Web 应用程序异步读取存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/File" rel="nofollow" title="File">
      File
     </a>
     或
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/Blob" rel="nofollow" title="Blob">
      Blob
     </a>
     对象指定要读取的文件或数据。
    </p>
    <p>
     文件对象可以从用户使用
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/input" rel="nofollow" title="&lt;input&gt;">
      &lt;input&gt;
     </a>
     元素选择文件而返回的
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/FileList" rel="nofollow" title="FileList">
      FileList
     </a>
     对象中获取，或者从拖放操作的
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/DataTransfer" rel="nofollow" title="DataTransfer">
      DataTransfer
     </a>
     对象中获取。
    </p>
    <p>
     <code>
      FileReader
     </code>
     只能访问用户明确选择的文件内容，无论是使用 HTML
     <code>
      &lt;input type="file"&gt;
     </code>
     元素还是通过拖放。它不能用于从用户的文件系统中按路径名读取文件。要按路径名读取客户端文件系统上的文件，请使用
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/File_System_API" rel="nofollow" title="文件系统访问 API">
      文件系统访问 API
     </a>

     <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/Window/fetch" rel="nofollow" title="fetch()">
      fetch()
     </a>
     ，如果跨源读取，则需要
     <a href="https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/CORS" rel="nofollow" title="CORS">
      CORS
     </a>
     权限。
