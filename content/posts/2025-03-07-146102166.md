    <h1 class="title-article" id="articleContentId">
     正则表达式（2）匹配规则
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     <strong>
      正则表达式的匹配规则定义了如何识别字符串中的特定模式。这些规则包括字符类匹配、元字符匹配、数量词、字符转义和分组。
     </strong>
    </p>
    <p>
    </p>
    <h2>
     字符类匹配
    </h2>
    <p>
     字符类匹配允许你指定一个字符集合，并匹配该集合中的任意单个字符。这是通过方括号
     <code>
      []
     </code>
     来实现的。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        简单字符类
       </strong>
       ：
       <code>
        [abc]
       </code>
       匹配 "a"、"b" 或 "c" 中的任意一个字符。
      </p>
     </li>
     <li>
      <p>
       <strong>
        范围字符类
       </strong>
       ：
       <code>
        [a-z]
       </code>
       匹配任意小写字母，
       <code>
        [0-9]
       </code>
       匹配任意数字。
      </p>
     </li>
     <li>
      <p>
       <strong>
        否定字符类
       </strong>
       ：
       <code>
        [^abc]
       </code>
       匹配除了 "a"、"b"、"c" 之外的任意字符。
      </p>
     </li>
     <li>
      <p>
       <strong>
        多字符集合
       </strong>
       ：
       <code>
        [abc123]
       </code>
       匹配 "a"、"b"、"c"、"1"、"2" 或 "3" 中的任意一个字符。
      </p>
     </li>
    </ul>
    <h2>
     元字符匹配
    </h2>
    <p>
     元字符是正则表达式中具有特殊含义的字符，用于指定复杂的匹配模式。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         .
        </code>
        （点）
       </strong>
       ：匹配除换行符以外的任意单个字符。
      </p>
     </li>
     <li>
      <p>
       <code>
        ^
       </code>
       ：匹配输入字符串的开始位置。
      </p>
     </li>
     <li>
      <p>
       <code>
        $
       </code>
       ：匹配输入字符串的结束位置。
      </p>
     </li>
     <li>
      <p>
       <code>
        *
       </code>
       ：匹配前面的字符零次或多次。
      </p>
     </li>
     <li>
      <p>
       <code>
        +
       </code>
       ：匹配前面的字符一次或多次。
      </p>
     </li>
     <li>
      <p>
       <code>
        ?
       </code>
       ：匹配前面的字符零次或一次。
      </p>
     </li>
     <li>
      <p>
       <code>
        |
       </code>
       ：逻辑或操作符，匹配两个表达式中的任意一个。
      </p>
     </li>
    </ul>
    <h2>
     数量词
    </h2>
    <p>
     数量词指定一个字符或字符组出现的次数。
    </p>
    <ul>
     <li>
      <p>
       <code>
        n*
       </code>
       ：匹配前面的字符零次或多次（n 为任意正整数）。
      </p>
     </li>
     <li>
      <p>
       <code>
        n+
       </code>
       ：匹配前面的字符一次或多次（n 为任意正整数）。
      </p>
     </li>
     <li>
      <p>
       <code>
        n?
       </code>
       ：匹配前面的字符零次或一次（n 为任意正整数）。
      </p>
     </li>
     <li>
      <p>
       <code>
        n{m}
       </code>
       ：匹配前面的字符恰好 m 次（m 为任意正整数）。
      </p>
     </li>
     <li>
      <p>
       <code>
        n{m,}
       </code>
       ：匹配前面的字符至少 m 次。
      </p>
     </li>
     <li>
      <p>
       <code>
        n{m,n}
       </code>
       ：匹配前面的字符至少 m 次，最多 n 次。
      </p>
     </li>
    </ul>
    <h2>
     字符转义
    </h2>
    <p>
     在正则表达式中，一些字符具有特殊含义。如果你想要匹配这些特殊字符本身，需要使用反斜杠
     <code>
      \
     </code>
     进行转义。
    </p>
    <ul>
     <li>
      <p>
       <code>
        \d
       </code>
       ：匹配一个数字字符（等同于
       <code>
        [0-9]
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <code>
        \s
       </code>
       ：匹配任何空白字符（包括空格、制表符、换页符等）。
      </p>
     </li>
     <li>
      <p>
       <code>
        \w
       </code>
       ：匹配任何字母数字字符（等同于
       <code>
        [a-zA-Z0-9_]
       </code>
       ）。
      </p>
     </li>
     <li>
      <p>
       <code>
        \n
       </code>
       ：匹配换行符。
      </p>
     </li>
     <li>
      <p>
       <code>
        \.
       </code>
       ：匹配点字符（
       <code>
        .
       </code>
       ）。
      </p>
     </li>
    </ul>
    <h2>
     分组
    </h2>
    <p>
     分组允许你将多个字符视为一个整体进行匹配和操作。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        捕获组
       </strong>
       ：
       <code>
        (...)
       </code>
       将括号内的内容视为一个整体进行匹配，并且可以被后续引用。
      </p>
     </li>
     <li>
      <p>
       <strong>
        非捕获组
       </strong>
       ：
       <code>
        (?:...)
       </code>
       将括号内的内容视为一个整体进行匹配，但不捕获匹配的内容。
      </p>
     </li>
     <li>
      <p>
       <strong>
        选择组
       </strong>
       ：
       <code>
        (a|b)
       </code>
       匹配 "a" 或 "b"。
      </p>
     </li>
     <li>
      <p>
       <strong>
        反向引用
       </strong>
       ：
       <code>
        \1
       </code>
       引用第一个捕获组匹配的内容。
      </p>
     </li>
    </ul>
