   <div class="htmledit_views" id="content_views">
    <p>
     Influxdb2支持cli命令维护数据库，以删除数据为例，步骤如下：
    </p>
    <p>
     1、下载cli客户端 https://dl.influxdata.com/influxdb/releases/influxdb2-client-2.7.5-windows-amd64.zip，解压到指定目录
     <br/>
     2、web界面创建token
     <br/>
     3、运行powershell，进入第1步解压的客户端目录
     <br/>
     4、配置token，以本机访问为例
     <br/>
    </p>
    <pre><code class="language-bash">  .\influx config create --config-name myconfig`--host-url "http://localhost:8086" `--org "组织名称" --token "web界面配置的api token串" `--active</code></pre>
    <p>
     <br/>
     5、执行处理命令，命令格式建议参考https://docs.influxdata.com/influxdb/v2/reference/cli/influx/
     <br/>
     下面命令是删除所有指定库和测量名数据的命令
    </p>
    <pre><code class="language-bash">  .\influx delete --bucket 库名 --start 1970-01-01T00:00:00Z --stop $(date -u +"%Y-%m-%dT%H:%M:%SZ") --predicate '_measurement="测量值"'</code></pre>
    <p>
     删除命令格式参考：
     <a href="https://docs.influxdata.com/influxdb/v2/reference/cli/influx/delete/" rel="nofollow" title="influx delete | InfluxDB OSS v2 Documentation">
      influx delete | InfluxDB OSS v2 Documentation
     </a>
    </p>
    <p>
     官方说明有如下警告：
    </p>
    <h5 id="deleting-data-without-a-delete-predicate">
     <a href="https://docs.influxdata.com/influxdb/v2/reference/cli/influx/delete/#deleting-data-without-a-delete-predicate" rel="nofollow" title="Deleting data without a delete predicate">
      Deleting data without a delete predicate
     </a>
    </h5>
    <p>
     Running
     <code>
      influx delete
     </code>
     without the
     <code>
      -p
     </code>
     or
     <code>
      --predicate
     </code>
     flag deletes all data with timestamps between the specified
     <code>
      --start
     </code>
     and
     <code>
      --stop
     </code>
     times in the specified bucket.
    </p>
    <h5 id="cannot-delete-data-by-field">
     <a href="https://docs.influxdata.com/influxdb/v2/reference/cli/influx/delete/#cannot-delete-data-by-field" rel="nofollow" title="Cannot delete data by field">
      Cannot delete data by field
     </a>
    </h5>
    <p>
     InfluxDB 2.7 does not support deleting data
     <strong>
      by field
     </strong>
     .
    </p>
    <p>
     6、参考概念-数据建模
    </p>
    <ul>
     <li>
      <strong>
       合理设计测量（Measurement）
      </strong>
      <ul>
       <li>
        测量类似于传统数据库中的表，要根据数据的业务含义进行合理划分。例如，将不同类型的监控数据（如服务器性能监控、网络流量监控）分别存储在不同的测量中，方便数据的管理和查询。
       </li>
       <li>
        避免创建过多或过少的测量，过多会增加管理成本，过少则会导致数据结构混乱。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       标签（Tag）和字段（Field）的使用
      </strong>
      <ul>
       <li>
        <strong>
         标签
        </strong>
        ：标签用于存储具有低基数（即不同取值较少）的元数据，如设备名称、地理位置等。标签会被自动索引，因此使用标签进行过滤和分组查询会非常高效。例如，在监控多个服务器时，将服务器名称作为标签，可以快速筛选出特定服务器的数据。
       </li>
       <li>
        <strong>
         字段
        </strong>
        ：字段用于存储实际的测量值，如温度、湿度等。字段值不会被自动索引，适合存储高基数的数据。避免将高基数的数据作为标签，否则会导致索引过大，影响性能。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       时间戳的处理
      </strong>
      <ul>
       <li>
        确保数据的时间戳准确无误，时间戳是 InfluxDB 中数据排序和查询的重要依据。
       </li>
       <li>
        尽量使用 UTC 时间，避免因时区问题导致数据不一致。
       </li>
      </ul>
     </li>
    </ul>
