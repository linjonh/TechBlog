    <h5>
     1.1. 处理缺失值的相关函数
    </h5>
    <ul>
     <li>
      <strong>
       判断缺失值
      </strong>
      <ul>
       <li>
        <strong>
         <code>
          pd.isnull(df)
         </code>
        </strong>
        ：用于判断 DataFrame
        <code>
         df
        </code>
        中的元素是否为缺失值（NaN ），返回一个与
        <code>
         df
        </code>
        形状相同的布尔型 DataFrame，元素为
        <code>
         True
        </code>
        表示对应位置是缺失值。
       </li>
       <li>
        <strong>
         <code>
          pd.notnull(df)
         </code>
        </strong>
        ：与
        <code>
         pd.isnull(df)
        </code>
        相反，判断 DataFrame
        <code>
         df
        </code>
        中的元素是否不是缺失值，返回布尔型 DataFrame，元素为
        <code>
         True
        </code>
        表示对应位置不是缺失值 。
       </li>
       <li>
        之后利用np.all（）或者np.any就可以知道数据中是否存在NaN。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       删除缺失值
      </strong>
      <ul>
       <li>
        <strong>
         <code>
          dropna(axis='rows')
         </code>
        </strong>
        ：用于删除存在缺失值的行（当
        <code>
         axis='rows'
        </code>
        时 ）或列（当
        <code>
         axis='columns'
        </code>
        时 ）。默认
        <code>
         axis='rows'
        </code>
        ，不会修改原数据，需要接收返回值。例如
        <code>
         new_df = df.dropna()
        </code>
        ，将返回删除缺失值行后的新 DataFrame。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       填充缺失值
      </strong>
      <ul>
       <li>
        <strong>
         <code>
          fillna(value, inplace=False)
         </code>
        </strong>
        ：用指定的
        <code>
         value
        </code>
        填充缺失值。
        <code>
         inplace=False
        </code>
        （默认 ）时，不修改原数据，返回填充后的新对象；
        <code>
         inplace=True
        </code>
        时，直接在原数据上进行填充修改。例如
        <code>
         df.fillna(0, inplace=True)
        </code>
        ，将把
        <code>
         df
        </code>
        中的缺失值都填充为 0 。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       替换数据
      </strong>
      <ul>
       <li>
        <strong>
         <code>
          replace(to_replace, value)
         </code>
        </strong>
        ：用于替换数据。
        <code>
         to_replace
        </code>
        是要被替换的值，
        <code>
         value
        </code>
        是替换后的值。例如
        <code>
         df.replace('?', np.nan)
        </code>
        ，将把
        <code>
         df
        </code>
        中所有的
        <code>
         '?'
        </code>
        替换为
        <code>
         NaN
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>

     1.2. 电影数据的缺失值处理示例

# 读取电影数据

# 使用pd.isnull判断

#另一种方法
is_null_result = pd.isnull(movie)
#返回一个和movie的shape一样的数组，全是布尔型，True为不空，False为空，
#只要结合np.any即可判断是否有空，如果有就是返回True
</code></pre>

     <li>
      <strong>
       存在缺失值且为
       <code>
        np.nan
       </code>
       时的处理

       <li>
        <strong>
         删除缺失值
        </strong>
       </li>
      </ul>
     </li>

    <pre><code class="language-python"># 不修改原数据
new_movie = movie.dropna()
# 也可重新赋值给原变量
movie = movie.dropna()

    <ul>
     <li>
      <strong>
       填充缺失值

# 遍历填充所有存在缺失值的列
for i in movie.columns:
    if np.all(pd.notnull(movie[i])) == False:
        movie[i].fillna(movie[i].mean(), inplace=True)

    <ul>
     <li>
      <strong>
       缺失值不是
       <code>
        np.nan

import numpy as np

# 替换'?'为np.nan
wis = wis.replace(to_replace='?', value=np.nan)
# 后续再进行缺失值填充等操作，例如填充某列缺失值为均值

    <h5>
     1.3. 处理缺失值的步骤总结
    </h5>
    <ol>
     <li>
      <strong>
       统一缺失值形式
      </strong>
      ：若存在缺失值但不是
      <code>
       np.nan
      </code>
      的形式，使用
      <code>
       data.replace(to_replace, value=np.nan)
      </code>
      将其替换为
      <code>
       np.nan
      </code>
      。
     </li>
     <li>
      <strong>
       判断是否存在缺失值
      </strong>
      ：使用
      <code>
       np.all(pd.notnull(data))
      </code>
      ，若结果为
      <code>
       True
      </code>
      ，表示不存在缺失值，不做处理；若为
      <code>
       False
      </code>
      ，表示存在缺失值，继续下一步。
     </li>
     <li>
      <strong>
       填充缺失值
      </strong>
      ：使用统计学指标值（如均值
      <code>
       mean
      </code>
      、中位数
      <code>
       median
      </code>
      等 ）对缺失值进行填充，如
      <code>
       data['column_name'].fillna(data['column_name'].mean(), inplace=True)
      </code>
      。
     </li>
    </ol>

     2.数据的离散化
    </p>
    <h5>
     2.1. 为什么要离散化
    </h5>
    <p>
     连续属性离散化目的是简化数据结构，减少给定
     <span style="color:#fe2c24">
      连续属性
     </span>
     值个数，常作为数据挖掘工具。比如在分析用户年龄与消费习惯时，年龄是连续属性，将年龄离散化为儿童、青年、中年、老年等区间，可更方便分析不同年龄段消费特征。
    </p>
    <h5>
     2.2. 什么是数据的离散化
    </h5>
    <p>
     连续属性离散化是在连续属性值域上划分若干离散区间，用不同符号或整数值代表落在各子区间的属性值。例如原始身高数据 165, 174, 160, 180, 159, 163, 192, 184 ，按 150 - 165, 165 - 180, 180 - 195 划分区间，可标记为矮、中、高类别。
    </p>
    <h5>
     2.3. 具体实现
    </h5>
    <ul>
     <li>
      <strong>
       读取股票数据
      </strong>
     </li>
    </ul>
    <pre><code class="language-python">import pandas as pd
# 读取股票数据
data = pd.read_csv("./data/stock_day.csv")
# 筛选出p_change数据
p_change = data['p_change']
</code></pre>
    <p>
     此代码先读取股票 CSV 文件数据，再提取出涨跌幅数据
     <code>
      p_change

    <ul>
     <li>
      <strong>
       将股票涨跌幅数据进行分组

    </ul>
    <pre><code class="language-python"># 自行分组，将数据分为10组
qcut = pd.qcut(p_change, 10)
# 计算分到每个组数据个数
qcut.value_counts()
</code></pre>
    <p>
     <code>
      pd.qcut(data, q)
     </code>
     中，
     <code>
      data
     </code>
     是要分组的数据（这里是
     <code>
      p_change
     </code>
     ），
     <span style="color:#fe2c24">
      <code>
       q
      </code>
      是要划分的组数
     </span>
     （这里为 10 ），它会使每组数据数量尽量相等。
     <span style="color:#fe2c24">
      <code>
       value_counts()
      </code>
      用于统计每组数据个数，经常结合离散化一起使用

    <ul>
     <li>
      <strong>
       自定义区间分组（
       <code>
        cut

    <pre><code class="language-python"># 自己指定分组区间
bins = [-100, -7, -5, -3, 0, 3, 5, 7, 100]

</code></pre>
    <p>
     <code>
      pd.cut(data, bins)
     </code>
     中，
     <code>
      data
     </code>
     是数据，
     <code>
      bins
     </code>
     是自定义区间边界列表（这里
     <code>
      bins
     </code>
     指定了具体区间 ），按此区间对数据进行分组。也经常结合value_counts（）一起使用。

     现在我们就通过统计个数得到了一下形式

dummies = pd.get_dummies(p_counts, prefix="rise")

      prefix

     3.数据的合并

    <p>
     如果数据由多张表构成，需要进行数据合并

    <h5>
     1.
     <code>
      pd.concat
     </code>
     实现数据合并
    </h5>
    <ul>
     <li>
      <strong>
       函数语法及参数
      </strong>
      <br/>
      <code>
       pd.concat(objs, axis=0, join='outer', ignore_index=False)
      </code>
      <ul>
       <li>
        <strong>
         <code>
          objs
         </code>
        </strong>
        ：必填参数，是一个由 DataFrame 或 Series 组成的
        <span style="color:#fe2c24">
         列表
        </span>
        ，如
        <code>
         [data1, data2]
        </code>
        ，表示要合并的对象。
       </li>
       <li>
        <strong>
         <code>
          axis
         </code>
        </strong>
        ：可选参数，用于指定合并的轴方向。
        <code>
         axis=0
        </code>
        （默认值）表示按列索引合并（纵向合并 ），即上下拼接；
        <code>
         axis=1
        </code>
        表示按行索引合并（横向合并 ），即左右拼接。
       </li>
       <li>
        <strong>
         <code>
          join
         </code>
        </strong>
        ：可选参数，指定合并时的连接方式，有
        <code>
         'outer'
        </code>
        （默认值，取并集 ）和
        <code>
         'inner'
        </code>
        （取交集 ）两种。
       </li>
       <li>
        <strong>
         <code>
          ignore_index
         </code>
        </strong>
        ：可选参数，布尔值，默认为
        <code>
         False
        </code>
        。若为
        <code>
         True
        </code>
        ，合并后将重新生成行索引。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       代码示例
      </strong>

    <h5>
     2.
     <code>
      pd.merge
     </code>
     实现数据合并
    </h5>
    <ul>
     <li>
      <strong>
       函数语法及参数
      </strong>
      <br/>
      <code>
       pd.merge(left, right, how='inner', on=None, left_on=None, right_on=None)
      </code>
      <ul>
       <li>
        <strong>
         <code>
          left
         </code>
        </strong>
        ：必填参数，第一个 DataFrame。
       </li>
       <li>
        <strong>
         <code>
          right
         </code>
        </strong>
        ：必填参数，第二个 DataFrame。
       </li>
       <li>
        <strong>
         <code>
          how
         </code>
        </strong>
        ：可选参数，指定连接方式，常见取值有：
        <ul>
         <li>
          <strong>
           <code>
            'inner'
           </code>
          </strong>
          ：内连接（默认值），使用左右 DataFrame 键的交集。
         </li>
         <li>
          <strong>
           <code>
            'left'
           </code>
          </strong>
          ：左连接，使用左 DataFrame 的键。
         </li>
         <li>
          <strong>
           <code>
            'right'
           </code>
          </strong>
          ：右连接，使用右 DataFrame 的键。
         </li>
         <li>
          <strong>
           <code>
            'outer'
           </code>
          </strong>
          ：外连接，使用左右 DataFrame 键的并集。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         <code>
          on
         </code>
        </strong>
        ：可选参数，指定用于合并的共同键列。
        <span style="color:#fe2c24">
         要求左右 DataFrame 都存在该列
        </span>
        。
       </li>
       <li>
        <strong>
         <code>
          left_on
         </code>
        </strong>
        ：可选参数，指定左 DataFrame 用于合并的键列。
       </li>
       <li>
        <strong>
         <code>
          right_on
         </code>
        </strong>
        ：可选参数，指定右 DataFrame 用于合并的键列。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       代码示例
      </strong>
     </li>
    </ul>
    <pre><code class="language-python">import pandas as pd
left = pd.DataFrame({'key1': ['K0', 'K0', 'K1', 'K2'],
                     'key2': ['K0', 'K1', 'K0', 'K1'],
                     'A': ['A0', 'A1', 'A2', 'A3'],
                     'B': ['B0', 'B1', 'B2', 'B3']})
right = pd.DataFrame({'key1': ['K0', 'K1', 'K1', 'K2'],
                      'key2': ['K0', 'K0', 'K0', 'K0'],
                      'C': ['C0', 'C1', 'C2', 'C3'],
                      'D': ['D0', 'D1', 'D2', 'D3']})
# 默认内连接
result_inner = pd.merge(left, right, on=['key1', 'key2'])
# 左连接
result_left = pd.merge(left, right, how='left', on=['key1', 'key2'])
# 右连接
result_right = pd.merge(left, right, how='right', on=['key1', 'key2'])
print("内连接结果：\n", result_inner)
print("左连接结果：\n", result_left)
print("右连接结果：\n", result_right)
</code></pre>
    <p>
     此代码先创建了两个 DataFrame
     <code>
      left
     </code>
     和
     <code>
      right
     </code>
     ，然后分别展示了内连接、左连接和右连接的合并结果。
    </p>
    <blockquote>
     <p>

    <blockquote>
     <p>
      为什么要在group后边加上具体的列呢？
     </p>
     <h4>
      明确聚合对象
     </h4>
     <p>
      <code>
       groupby
      </code>
      函数实现分组后，需要对特定的列进行聚合计算（这里是求平均值，使用
      <code>
       mean
      </code>
      函数 ）。
      <code>
       col
      </code>
      这个 DataFrame 可能包含多列数据（如案例中还有
      <code>
       object
      </code>
      、
      <code>
       price2
      </code>
      等列 ），通过指定
      <code>
       ['price1']
      </code>
      ，就明确告诉程序是要对
      <code>
       price1
      </code>
      这一列的数据进行聚合操作。如果不指定，程序就不知道具体要对哪列数据进行后续的聚合计算，会引发错误。
     </p>
     <h4>
      符合逻辑操作流程
     </h4>
     <p>
      分组聚合操作的逻辑是先分组，然后对组内的某列或某些列数据进行聚合。比如在这个案例中，我们想知道不同颜色对应的
      <code>
       price1
      </code>
      价格的平均值，所以要指定
      <code>
       price1
      </code>
      列。如果我们想分析
      <code>
       price2
      </code>
      列，就需要指定
      <code>
       ['price2']
      </code>
      ，如
      <code>
       col.groupby(['color'])['price2'].mean()
      </code>
      。
     </p>
    </blockquote>
    <blockquote>
     <p>
      注意，仔细观察两种方法的不同之处，group里的代码一个直接用列名一个不能：
     </p>
     <p>
      在
      <code>
       col['price1'].groupby(col['color']).mean()
      </code>
      这种针对 Series 对象进行
      <code>
       groupby
      </code>
      操作的情境下，不能直接用列名。
     </p>
     <p>
      因为
      <code>
       col['price1']
      </code>
      已经是一个 Series 对象，当对它调用
      <code>
       groupby
      </code>
      方法时，
      <code>
       groupby
      </code>
      函数期望传入一个与该 Series 长度一致的可迭代对象（通常也是一个 Series ）来确定分组规则。
      <span style="color:#fe2c24">
       如果直接写列名，比如
       <code>
        col['price1'].groupby('color').mean()
       </code>
       ，程序会报错，因为
       <code>
        'color'
       </code>
       只是一个字符串，不是与
       <code>
        col['price1']
       </code>
       长度相同的可迭代对象，无法明确分组逻辑。
      </span>
     </p>
     <p>
      而在基于 DataFrame 进行
      <code>
       groupby
      </code>
      操作时，像
      <code>
       col.groupby(['color'])['price1'].mean()
      </code>
      ，可以直接用列名（以列表形式 ）来指定分组依据，
      <span style="color:#fe2c24">
       这是因为 DataFrame 的
       <code>
        groupby
       </code>
       方法有相应的机制去识别和处理列名列表作为分组键
      </span>
      。
     </p>
    </blockquote>
    <blockquote>
     <p>
      当group中分组依据是一个列表的时候就好像是先按第一个分，然后一次分组：
