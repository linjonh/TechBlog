    <h3>
     <a id="1__2">
     </a>
     1 介绍
    </h3>
    <p>
     <strong>
      Sentinel
     </strong>
     是阿里开源的分布式系统流量防卫组件，专注于
     <strong>
      流量控制、熔断降级、系统保护
     </strong>
     。
    </p>
    <p>
     官网：https://sentinelguard.io/zh-cn/index.html
    </p>
    <p>
     wiki：
     <a href="https://github.com/alibaba/Sentinel/wiki/%E5%A6%82%E4%BD%95%E4%BD%BF%E7%94%A8">
      https://github.com/alibaba/Sentinel/wiki
     </a>
    </p>
    <p>
     <strong>
      对比同类产品
     </strong>
     ：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        产品
       </th>
       <th>
        特点
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         Hystrix
        </strong>
       </td>
       <td>
        Netflix 开源，功能全面但已停止维护，基于线程池隔离，资源消耗较大
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         Resilience4j
        </strong>
       </td>
       <td>
        轻量级，基于函数式编程，但需要结合其他组件实现完整功能
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         Sentinel
        </strong>
       </td>
       <td>
        轻量级，实时监控和控制，支持动态规则配置，与 Spring Cloud 深度集成
       </td>
      </tr>
     </tbody>
    </table>

      </strong>

     <li>

      </strong>

     <li>

      </strong>

    <h3>
     <a id="_300">
     </a>
     重点总结
    </h3>
    <ol>
     <li>
      <strong>
       核心功能
      </strong>
      ：
      <ul>
       <li>
        流量控制（QPS/线程数）、熔断降级（慢调用/异常比例）、热点参数限流。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       优势
      </strong>
      ：
      <ul>
       <li>
        动态规则配置，实时监控，与 Spring Cloud 无缝集成。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       实操关键
      </strong>
      ：
      <ul>
       <li>
        控制台启动后需配置
        <code>
         spring.cloud.sentinel.transport.dashboard
        </code>
        。
       </li>
       <li>
        使用
        <code>
         @SentinelResource
        </code>
        定义资源和降级逻辑。
       </li>
       <li>
        OpenFeign 需结合 Fallback 类实现服务降级。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       注意事项
      </strong>
      ：
      <ul>
       <li>
        规则配置后需持久化到 Nacos/Redis，避免重启丢失。
       </li>
       <li>
        生产环境建议开启 Sentinel 的集群流控功能。
       </li>
      </ul>
     </li>
    </ol>
