     <span style="color:#0d0016">
      1.1  序列式容器和关联式容器
     </span>
    </h5>
    <p>
     <span style="color:#0d0016">
      今天我们学习的map和set是一种新容器：
     </span>
    </p>
    <blockquote>
     <div>
      <span style="color:#1f2329">
       1. 前⾯我们已经接触过STL中的部分容器如：string、vector、list、deque、array、forward_list等，这些容器统称为序列式容器，因为逻辑结构为线性序列的数据结构，两个位置存储的值之间⼀般没有紧密的关联关系，⽐如交换⼀下，他依旧是序列式容器。顺序容器中的元素是按他们在容器中的存储位置来顺序保存和访问的。
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       2. 关联式容器也是⽤来存储数据的，与序列式容器不同的是，关联式容器逻辑结构通常是⾮线性结构，两个位置有紧密的关联关系，交换⼀下，他的存储结构就被破坏了。顺序容器中的元素是按关键字来保存和访问的。关联式容器有map/set系列和unordered_map/unordered_set系列。
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       3. 本章节讲解的map和set底层是红⿊树，红⿊树是⼀颗平衡⼆叉搜索树。set是key搜索场景的结构，map是key/value搜索场景的结构。
      </span>
     </div>
    </blockquote>

     <span style="color:#0d0016">
      2.3 set的构造和迭代器
     </span>

    <p>
    </p>
    <blockquote>
     <div>
      <span style="color:#1f2329">
       set的构造我们关注以下⼏个接⼝即可。
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       set的⽀持正向和反向迭代遍历，遍历默认按升序顺序，因为底层是⼆叉搜索树，迭代器遍历⾛的中序；⽀持迭代器就意味着⽀持范围for，set的iterator和const_iterator都不⽀持迭代器修改数据，修改关键字数据，破坏了底层搜索树的结构。
      </span>
     </div>
    </blockquote>

    <h5 id="2.6%20find%E5%92%8Cerase%E4%BD%BF%E2%BD%A4%E6%A0%B7%E4%BE%8B%EF%BC%9A" name="2.6%20find%E5%92%8Cerase%E4%BD%BF%E2%BD%A4%E6%A0%B7%E4%BE%8B%EF%BC%9A">

{

    <p>
     迭代器是按照set的中序遍历走的，所以set中的最小值就是迭代器的起始位置，也就是begin()，当我们输入值进行删除时，返回值是该数在set中的个数，对于使用迭代器进行删除，在删除后，迭代器会失效。相比算法库的查找和set自身的查找，自身的查找是根据二叉搜索树来进行的，所以最多查找logN次，而算法库的查找是一个一个进行查找，最多查找N次。
    </p>
    <pre><code class="hljs">int main()
{
	std::set&lt;int&gt; myset;

	for (auto e : myset)
	{
		cout &lt;&lt; e &lt;&lt; " ";
	}
	cout &lt;&lt; endl;
	// 实现查找到的[itlow,itup)包含[30, 60]区间
	// 返回 &gt;= 30
	auto itlow = myset.lower_bound(30);
	// 返回 &gt; 60
	auto itup = myset.upper_bound(60);
	// 删除这段区间的值
	myset.erase(itlow, itup);
	for (auto e : myset)
	{
		cout &lt;&lt; e &lt;&lt; " ";
	}
	cout &lt;&lt; endl;
	return 0;
}</code></pre>

     <span style="color:#0d0016">
      2.7 multiset和set的差异
     </span>
    </h5>
    <div>
     <span style="color:#1f2329">
      multiset和set的使⽤基本完全类似，主要区别点在于multiset⽀持值冗余，那么
     </span>
    </div>
    <div>
     <span style="color:#1f2329">
      insert/find/count/erase都围绕着⽀持值冗余有所差异，具体参看下⾯的样例代码理解。
     </span>
    </div>
    <div>
     <pre><code class="hljs">int main()
{
	multiset&lt;int&gt; s = { 4,2,7,2,4,8,4,5,4,9 };
	auto it = s.begin();
	while (it != s.end())
	{
		cout &lt;&lt; *it &lt;&lt; " ";
		++it;
	}
	cout &lt;&lt; endl;
	// 相⽐set不同的是，x可能会存在多个，find查找中序的第⼀个
	int x;
	cin &gt;&gt; x;
	auto pos = s.find(x);
	while (pos != s.end() &amp;&amp; *pos == x)
	{
		cout &lt;&lt; *pos &lt;&lt; " ";
		++pos;
	}
	cout &lt;&lt; endl;
	// 相⽐set不同的是，count会返回x的实际个数
	cout &lt;&lt; s.count(x) &lt;&lt; endl;
	// 相⽐set不同的是，erase给值时会删除所有的x
	s.erase(x);
	for (auto e : s)
	{
		cout &lt;&lt; e &lt;&lt; " ";
	}
	cout &lt;&lt; endl;
	return 0;
}</code></pre>
     <p>
      count函数会返回在multiset中该数据的个数，在set中只有0和1两种结果，但在multiset中有更多种结果，在multiset中的erase函数会删除给定的所有值。
     </p>
     <h5 id="2.8%20%E5%85%B3%E4%BA%8Eset%E7%9A%84%E9%A2%98%E7%9B%AE" name="2.8%20%E5%85%B3%E4%BA%8Eset%E7%9A%84%E9%A2%98%E7%9B%AE" style="background-color:transparent">

     <p>
      两个数组的交集：
     </p>
     <pre><code class="hljs">class Solution {
public:
    vector&lt;int&gt; intersection(vector&lt;int&gt;&amp; nums1, vector&lt;int&gt;&amp; nums2) {
        set&lt;int&gt; s1(nums1.begin(),nums1.end());
        set&lt;int&gt; s2(nums2.begin(),nums2.end());
        vector&lt;int&gt; tmp;
        auto it1=s1.begin();
        auto it2=s2.begin();
        while(it1!=s1.end()&amp;&amp;it2!=s2.end())
        {
            if(*it1==*it2)
            {
                tmp.push_back(*it1);
                it1++;
                it2++;
            }
            else if(*it1&lt;*it2)
            {
                it1++;
            }
            else{
                it2++;
            }
        }
        return tmp;
    }
};</code></pre>
     <p>
      我们使用两个set分别存放两个数组的数据，然后两个set分别创建一个迭代器进行比较，当迭代器指向的值相同的时候，则该值为交集，随后两个迭代器都向后移动，当迭代器指向的值不相同的时候，我们让指向值小的迭代器向后移动，因为迭代器是按照中序遍历走的，小值的后面是大值，但大值后面不可能是小值。
     </p>

     </span>

    <blockquote>
     <div>
      <span style="color:#1f2329">
       前⾯提到map⽀持修改mapped_type 数据，不⽀持修改key数据，修改关键字数据，破坏了底层搜索树的结构。
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       map第⼀个⽀持修改的⽅式时通过迭代器，迭代器遍历时或者find返回key所在的iterator修改，map还有⼀个⾮常重要的修改接⼝operator[]，但是operator[]不仅仅⽀持修改，还⽀持插⼊数据和查找数据，所以他是⼀个多功能复合接⼝
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       需要注意从内部实现⻆度，map这⾥把我们传统说的value值，给的是T类型，typedef为
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       mapped_type。⽽value_type是红⿊树结点中存储的pair键值对值。⽇常使⽤我们还是习惯将这⾥的T映射值叫做value。
      </span>
     </div>
    </blockquote>
    <p>
     map中有一个运算符重载函数operator[]，该函数不仅可以修改，还可以进行插入和查找。
    </p>
    <pre><code class="hljs">// operator的内部实现

    <p>
     该运算符重载和前面的insert有关，当我们输入一个key值后，会进行insert：
    </p>
    <blockquote>
     <div>
      <span style="color:#0d0016">
       1、如果key已经在map中，插⼊失败，则返回⼀个pair&lt;iterator,bool&gt;对象，返回pair对象
      </span>
     </div>
     <div>
      <span style="color:#0d0016">
       first是key所在结点的迭代器，second是false
      </span>
     </div>
     <div>
      <span style="color:#0d0016">
       2、如果key不在在map中，插⼊成功，则返回⼀个pair&lt;iterator,bool&gt;对象，返回pair对象
      </span>
     </div>
     <div>
      <span style="color:#0d0016">
       first是新插⼊key所在结点的迭代器，second是true
      </span>
     </div>
     <div>
      <span style="color:#0d0016">
       3、也就是说⽆论插⼊成功还是失败，返回pair&lt;iterator,bool&gt;对象的first都会指向key所在的迭代器，那么也就意味着insert插⼊失败时充当了查找的功能，正是因为这⼀点，insert可以⽤来实现operator[]
      </span>
     </div>
     <div>
      <span style="color:#0d0016">
       1、如果k不在map中，insert会插⼊k和mapped_type默认值，同时[]返回结点中存储mapped_type值的引⽤，那么我们可以通过引⽤修改返映射值。所以[]具备了插⼊+修改功能 2、如果k在map中，insert会插⼊失败，但是insert返回pair对象的first是指向key结点的迭代器，返回值同时[]返回结点中存储mapped_type值的引⽤，所以[]具备了查找+修改的功能
      </span>
     </div>
    </blockquote>

     <span style="color:#1f2329">
      multimap和map的差异
     </span>

    <blockquote>
     <div>

    <pre><code class="hljs">class Solution {

    Node* copyRandomList(Node* head) {
        map&lt;Node*,Node*&gt; nodemap;
        Node* cur=head;

        {

                copytail=copyhead=new Node(cur-&gt;val);
            }
            else{
                copytail-&gt;next=new Node(cur-&gt;val);
                copytail=copytail-&gt;next;
            }

            cur=cur-&gt;next;

        cur=head;
        Node* copy=copyhead;
        while(cur)
        {

                copy-&gt;random=nullptr;
            }

    <pre><code class="hljs">class Solution {

    struct compare{
        bool operator()(const pair&lt;string,int&gt;&amp; p1,const pair&lt;string,int&gt;&amp; p2)

            return p1.second&gt;p2.second||(p1.second==p2.second&amp;&amp;p1.first&lt;p2.first);

            m[e]++;
        }
        vector&lt;pair&lt;string,int&gt;&gt; v(m.begin(),m.end());
        sort(v.begin(),v.end(),compare());
        vector&lt;string&gt; vv;

     <span style="color:#1f2329">
      本题⽬我们利⽤map统计出次数以后，返回的答案应该按单词出现频率由⾼到低排序，有⼀个特殊要求，如果不同的单词有相同出现频率，按字典顺序排序。

      <span style="color:#1f2329">
       ⽤排序找前k个单词，因为map中已经对key单词排序过，也就意味着遍历map时，次数相同的单词，字典序⼩的在前⾯，字典序⼤的在后⾯。那么我们将数据放到vector中⽤⼀个稳定的排序就可以实现上⾯特殊要求，但是sort底层是快排，是不稳定的，所以我们要⽤stable_sort，他是稳定的。
      </span>
     </div>
     <div>
      <span style="color:#1f2329">
       当然也可以实现仿函数改变比较的规则。这里就用到了仿函数，当出现次数不同的时候，按照出现次数进行比较，若出现次数相同，则按照字典序进行比较。
