     【MySQL】MySQL是如何传输数据的？

    <h3>
     <strong>
      1. 客户端-服务器通信（连接层）
     </strong>
    </h3>
    <p>
     MySQL 客户端（如
     <code>
      mysql
     </code>
     命令行、GUI 工具或应用程序）与 MySQL 服务器之间通过
     <strong>
      网络
     </strong>
     或
     <strong>
      本地套接字
     </strong>
     进行通信。
    </p>
    <h4>
     1.1 本地和远程通信协议：TCP/IP（Transmission Control Protocol/Internet Protocol）
    </h4>
    <p>
     （1）TCP/IP是互联网的基础通信协议，用于
     <strong>
      本地（LAN）
     </strong>
     和
     <strong>
      远程（WAN）
     </strong>
     的数据传输。它包含：
    </p>
    <ul>
     <li>
      <strong>
       IP（Internet Protocol，互联网协议）
      </strong>
      ：负责寻址和路由数据包。
     </li>
     <li>
      <strong>
       TCP（Transmission Control Protocol，传输控制协议）
      </strong>
      ：提供可靠的端到端通信，确保数据不丢失、不重复、按顺序到达。
     </li>
    </ul>
    <p>
     （2）使用IP 地址 &amp; DNS 主机名来识别主机
    </p>
    <ul>
     <li>
      <strong>
       IP 地址
      </strong>
      ：用于唯一标识网络中的设备（如
      <code>
       192.168.1.100
      </code>
      ）。
     </li>
     <li>
      <strong>
       DNS（域名系统）
      </strong>
      ：将主机名（如
      <code>
       mysql.example.com
      </code>
      ）解析为 IP 地址，方便访问远程服务器。
     </li>
    </ul>
    <p>
     （3）使用TCP 端口号区分同一主机上的不同服务。
    </p>
    <p>
     （4）本地通信：
     <strong>
      同一台主机内
     </strong>
     进程之间的通信，常用于
     <strong>
      进程间通信（IPC）
     </strong>
     或
     <strong>
      测试网络服务
     </strong>
     。
    </p>
    <ul>
     <li>
      仅在
      <strong>
       本机
      </strong>
      上运行的进程间通信，
      <strong>
       不经过物理网卡
      </strong>
      。
     </li>
     <li>
      使用
      <strong>
       回环地址（127.0.0.1）
      </strong>
      进行通信。
     </li>
     <li>
      <strong>
       低延迟、高速、安全
      </strong>
      ，不会经过外部网络。
     </li>
     <li>
      通过
      <strong>
       Unix Socket
      </strong>
      或
      <code>
       127.0.0.1:3306
      </code>
      进行连接。
      <pre><code class="language-bash">mysql -u root -p -h 127.0.0.1 -P 3306</code></pre>
      <p>
       如果使用
       <span style="color:#494949">
        --host=
       </span>
       <span style="color:#0d0016">
        <code>
         localhost
        </code>
       </span>
       ，MySQL 可能会使用 Unix 套接字连接，而不是 TCP。
      </p>
     </li>
    </ul>
    <p>
     （5）远程通信：不同设备间通过 TCP 进行通信，通常使用
     <strong>
      公网 IP（如 8.8.8.8）或局域网 IP（如 192.168.1.100）
     </strong>
     。
    </p>
    <ul>
     <li>
      需要网络连接（局域网或互联网）。
     </li>
     <li>
      可能受到防火墙、NAT 限制。
     </li>
     <li>
      存在一定的网络延迟。
     </li>
     <li>
      可能需要身份验证、加密保护（如 SSH、TLS）。
     </li>
    </ul>
    <h4>
     1.2 本地通信协议：Socket
    </h4>
    <p>
     （1）Socket 主要用于
     <strong>
      同一台计算机上
     </strong>
     的
     <strong>
      进程间通信
     </strong>
     （IPC），可以通过：
    </p>
    <ul>
     <li>
      <strong>
       Unix 域套接字（UDS)：不使用IP地址和端口，
      </strong>
      基于
      <strong>
       文件路径
      </strong>
      通信（如
      <code>
       /tmp/mysqld
       <strong>
        .sock
       </strong>
      </code>
      ）。只在 Linux/macOS 可用（Windows 没有 UDS）。低延迟，高性能，比 TCP 更快。
     </li>
     <li>
      <strong>
       本地TCP套接字：
      </strong>
      使用
      <code>
       127.0.0.1
      </code>
      作为 IP 地址。支持所有操作系统，性能稍低于 Unix 套接字
     </li>
     <li>
      <strong>
       本地UDP套接字：
      </strong>
      使用
      <code>
       127.0.0.1
      </code>
      作 IP 地址，UDP 是
      <strong>
       无连接的协议
      </strong>
      ，适用于实时通信（如游戏、本地日志传输）。适合小数据传输，但可能丢包。
     </li>
     <li>
      使用/var/lib/mysql/mysql.sock 套接字文件的示例：
     </li>
    </ul>
    <pre><code class="language-bash"> mysql -S /var/lib/mysql/mysql.sock -uroot -p</code></pre>
    <ul>
     <li>
      使用默认套接字文件/tmp/mysql.sock 的示例：
      <pre><code class="language-bash"> mysql -uroot -p
</code></pre>
      <p>
       如果没有指定主机，mysql 假定-h localhost
      </p>
     </li>
    </ul>
    <h4>
     1.3 SSL
    </h4>
    <p>
     连接层在可用时使用安全、加密的连接SSL。
    </p>
    <ul>
     <li>
      如果服务器主机上安装了OpenSSL，MySQL包安装程序将创建SSL密钥，安装程序调用mysql_ssl_rsa_setup 实用程序来创建密钥，如果从二进制存档安装，需要手动调用此实用程序。
     </li>
     <li>
      如果密钥可用，MySQL客户机将使用SSL，密钥位于数据目录中，将客户端密钥复制到远程客户端以启用加密的远程连接。
     </li>
     <li>
      如果SSL不可用，则连接将不加密，可以将服务器和客户机配置为强制使用SSL。
     </li>
    </ul>
    <h4>
     1.4连接线程
    </h4>
    <p>
     在 MySQL
     <strong>
      默认模式
     </strong>
     下，每个客户端连接都会创建一个
     <strong>
      独立的服务器线程
     </strong>
     来处理 SQL 语句。然而，在高并发场景下，这种模式会带来
     <strong>
      线程创建和销毁的开销
     </strong>
     ，导致性能下降。
    </p>
    <p>
     为了解决这个问题，MySQL 提供了
     <strong>
      线程池（Thread Pool）
     </strong>
     ，它
     <strong>
      分离
     </strong>
     了
     <strong>
      连接管理
     </strong>
     和
     <strong>
      线程管理，
     </strong>
     通过
     <strong>
      线程组（Thread Group）
     </strong>
     共享线程资源，提高服务器的可扩展性和并发处理能力。
    </p>
    <p>
     <strong>
      （1）线程池插件
     </strong>
     管理连接和服务器线程：
    </p>
    <ul>
     <li>
      线程池不会为每个连接创建独立线程，而是通过线程组（Thread Group）共享线程。
     </li>
     <li>
      连接请求进入线程池
      <strong>
       等待调度
      </strong>
      ，然后由
      <strong>
       可用线程
      </strong>
      处理 SQL 语句。
     </li>
    </ul>
    <p>
     <strong>
      （2）线程组（Thread Group）
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       每个线程组
      </strong>
      处理多个客户端连接，但
      <strong>
       任何时刻只允许一个线程运行一个短时 SQL 语句
      </strong>
      。
     </li>
     <li>
      <strong>
       短时语句优先
      </strong>
      ，避免长时间 SQL 语句阻塞其他查询。
     </li>
    </ul>
    <p>
     <strong>
      （3）事务优先级管理
     </strong>
     ：
    </p>
    <ul>
     <li>
      <strong>
       运行中的事务具有高优先级
      </strong>
      ，保证事务一致性和响应速度。
     </li>
     <li>
      <strong>
       长时间运行的事务不会阻塞短语句
      </strong>
      ，提高数据库吞吐量。
     </li>
    </ul>
