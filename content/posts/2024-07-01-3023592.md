    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        在 北京创业6年了，其中感受无法形容，这是我第一次承接微软的技术外包项目，种种感想尽在其中，首先我非常感谢北京、上海Microsoft公司的朋友，是 他们为我们提供了这次机会，其次，我祝福我们团队中的年轻人，通过这次项目开发，他们得到了充分的锻炼……。国庆长假期间，我有充分的时间回顾项目，其中 的诸多感想写出来与朋友们分享。创业是很辛苦的事情，其中的很多无奈你必须面对，我已经感觉到自己在逐步的远离技术开发，但看到我们团队中年轻人越来越 强，我既感慨，又兴奋，通过这次外包，我回到了几年前，年轻人提高了自信，我坚信我们的技术团队是最好的，团队的未来属于其中的年轻人……
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        今年端午的前二天我接到北京
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft
        </font>
       </span>
       <span style="font-family: 宋体;">
        一位朋友的电话，问我是否对
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft
        </font>
       </span>
       <span style="font-family: 宋体;">
        的一个项目有兴趣，这是一个电信方面的项目，建立在
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft
        </font>
       </span>
       <span style="font-family: 宋体;">
        统一通讯技术框架之上。放下电话我与助手去了
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft
        </font>
       </span>
       <span style="font-family: 宋体;">
        ，打算先了解一些项目的具体事宜。我办公室在鸟巢边上，与盘古大观大约
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         30
        </font>
       </span>
       <span style="font-family: 宋体;">
        米的距离，因此距离微软也不算很远，出租车通常情况下也就
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         30
        </font>
       </span>
       <span style="font-family: 宋体;">
        分钟。微软所在的现代汽车大厦大概每年都去过几回，我对微软也不是很陌生，但熟悉的人西格玛里面的稍多一些（可能是离我家近的缘故）。虽然与微软的技术打交道十几年了，但与微软具体的项目接触却是头一回，到朋友的办公室之后，几句简单的寒暄就直接切入主题了。
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        也许有一种预感，我似乎察觉到这个项目不是那么容易。朋友首先与我谈到
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Office Communicator 2007
        </font>
       </span>
       <span style="font-family: 宋体;">
        ，实话实说，我只是在技术资料上读过有限的关于
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Office Communicator
        </font>
       </span>
       <span style="font-family: 宋体;">
        的信息，感性的体验完全没有，朋友几分钟的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Demo
        </font>
       </span>
       <span style="font-family: 宋体;">
        也就给我留下一个轮廓性质的印象，但是我明确了一个问题：这个项目本质上是对
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         OC
        </font>
       </span>
       <span style="font-family: 宋体;">
        （
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Office Communicator
        </font>
       </span>
       <span style="font-family: 宋体;">
        ）的一种再定制，如同
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN Shell
        </font>
       </span>
       <span style="font-family: 宋体;">
        与
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        之间的关系。谈完项目之后，一种潜意识告诉我虽然无法当时确认这个项目，但我必须做好承接的准备……
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        第二天，朋友的助手到我办公室给我们安装了一套完整的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         OC
        </font>
       </span>
       <span style="font-family: 宋体;">
        环境，同时在我笔记本电脑中也搭建了一套虚拟机服务环境，当时我们的办公室刚刚安置完毕，因此，
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         OC
        </font>
       </span>
       <span style="font-family: 宋体;">
        的细节只能在端午前的晚上进行深入的了解。我对
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        的了解在那个时刻也仅限于具体的使用，说来很巧，我的一位哈尔滨的朋友到北京工作之后，建议我使用
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN Shell
        </font>
       </span>
       <span style="font-family: 宋体;">
        ，也许是因为他工作单位的某些原因，他希望我们之间的聊天要进行一些“加密”处理，因此对
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN Shell
        </font>
       </span>
       <span style="font-family: 宋体;">
        我还是有很好的感性认识。
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        虽然对
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        的了解不多，但出于技术上“解剖”的习惯，我也曾经考察过
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         UI
        </font>
       </span>
       <span style="font-family: 宋体;">
        结构特征，
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        是
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft Direct UI
        </font>
       </span>
       <span style="font-family: 宋体;">
        技术框架的典型产品，虽然网络上有很多关于
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Direct UI
        </font>
       </span>
       <span style="font-family: 宋体;">
        的探讨，但大多数都是似是而非的讨论，其根本原因是
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         DirectUI
        </font>
       </span>
       <span style="font-family: 宋体;">
        是
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Microsoft
        </font>
       </span>
       <span style="font-family: 宋体;">
        私有的技术框架，外界基本上不可能拿到有价值的信息。微软的朋友给我提供了他能够拿到的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         DirectUI
        </font>
       </span>
       <span style="font-family: 宋体;">
        的相关资料，遗憾的是文档虽然很全，但缺乏必要的开发库与头文件，尽管如此，我们也可以窥视
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         DirectUI
        </font>
       </span>
       <span style="font-family: 宋体;">
        的“一斑”了。如果我可以接下微软的项目，那么一个基本的前提是我们必须找到一种类似
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN Shell
        </font>
       </span>
       <span style="font-family: 宋体;">
        扩展
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         MSN
        </font>
       </span>
       <span style="font-family: 宋体;">
        的途径，因此一个最基本的问题就是：我们能否在
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         Office Communicator
        </font>
       </span>
       <span style="font-family: 宋体;">
        上实现自定义的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         UI
        </font>
       </span>
       <span style="font-family: 宋体;">
        元素，这个问题是我必须克服的第一个问题。
       </span>
      </font>
     </font>
    </p>

    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <font size="3">
      <font color="#000000">
       <span style="font-family: 宋体;">
        端 午节是传统的节日，当然要陪家人简单的过过节。微软的朋友也许是顾及情面，并没有急于催促这件事情，但也无意中流露出急迫的成分，我一时之间也无法判断出 是否可以如法炮制，只能凭感觉了。果然端午假期的第二天朋友来电话了，我只好说还没有明确的技术方案，毕竟时间太短了，但可以就一些技术感想说说思路，我 的基本感想是：一、
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         HOOK
        </font>
       </span>
       <span style="font-family: 宋体;">
        几个基本的
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         API
        </font>
       </span>
       <span style="font-family: 宋体;">
        函数，感觉上应该是或者起码要
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         HOOK
        </font>
       </span>
       <span style="font-family: 宋体;">
        的几个
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         API
        </font>
       </span>
       <span style="font-family: 宋体;">
        要包括：
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         FindResource
        </font>
       </span>
       <span style="font-family: 宋体;">
        、
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         LoadResource
        </font>
       </span>
       <span style="font-family: 宋体;">
        、
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         SizeofResource
        </font>
       </span>
       <span style="font-family: 宋体;">
        ；二、
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         DirectUI
        </font>
       </span>
       <span style="font-family: 宋体;">
        技术的基本出发点是
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         UI
        </font>
       </span>
       <span style="font-family: 宋体;">
        元素“
       </span>
       <span lang="EN-US">
        <font face="Calibri">
         XML
        </font>
       </span>
       <span style="font-family: 宋体;">
        描 述化”，可以动态的修改资源描述，在其中增加新元素的插入点；三、最关键的问题是，如何给“新元素”匹配我们希望的“动作”。更多的感觉还无法找到，微软 的朋友在电话里听完后感觉我的感想应该是可行的，约我能否再去微软，我只好说，这些感觉仅仅是感觉而已，需要进一步的整理，这样再次见面才会有比较实际的 想法或方案，于是我们约定端午假期的最后一天在微软相见，我意识到基本上没有退路了，这个感觉必须要落实到真实的一面。
       </span>
      </font>
     </font>
    </p>
    <p class="MsoNormal" style="margin: 0cm 0cm 0pt; text-indent: 21pt;">
     <span style="font-family: 宋体;">
      <font color="#000000" size="3">
       应 该说在想法上，我们没有充分的准备时间，在技术上更谈不上充分（毕竟没有真正的实现这些想法）。在这样的前提下，我再次与朋友在微软见面了，由于还是假 期，微软里面的人不多，我们在会议室里经过一小时的探讨，虽然没有把握，但也形成了初步的方案，离承接项目还有相当的距离，可以说只是项目的“预研”实 验，可能是出于鼓励，微软的朋友说，一定可以实现这些设想，而我则不是那么轻松，我知道，这是第一次项目接触，如果没有结果，应该不会有下一次……
      </font>
     </span>
    </p>
