    <h1 class="title-article" id="articleContentId">
     在 CentOS 上，常用几种方法来确保 Python 脚本在断开终端后继续运行
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

    <p>
     在 CentOS 上，你可以使用以下几种方法来确保 Python 脚本在断开终端后继续运行：
    </p>
    <h4>
     <a id="1__nohup__2">
     </a>
     1. 使用
     <code>
      nohup
     </code>
     命令
    </h4>
    <p>
     <code>
      nohup
     </code>
     命令可以让进程在终端关闭后继续运行。
    </p>
    <pre><code class="prism language-bash"><span class="token function">nohup</span> python main.py <span class="token operator">&gt;</span> output.log <span class="token operator"><span class="token file-descriptor important">2</span>&gt;</span><span class="token file-descriptor important">&amp;1</span> <span class="token operator">&amp;</span>
</code></pre>
    <ul>
     <li>
      <code>
       nohup
      </code>
      ：忽略挂断信号，使进程在终端关闭后继续运行。
     </li>
     <li>
      <code>
       &gt; output.log
      </code>
      ：将标准输出重定向到
      <code>
       output.log
      </code>
      文件。
     </li>
     <li>
      <code>
       2&gt;&amp;1
      </code>
      ：将标准错误输出重定向到标准输出（即
      <code>
       output.log
      </code>
      文件）。
     </li>
     <li>
      <code>
       &amp;
      </code>
      ：将进程放到后台运行。
     </li>
    </ul>
    <h4>
     <a id="2__screen__14">
     </a>
     2. 使用
     <code>
      screen
     </code>
     命令
    </h4>
    <p>
     <code>
      screen
     </code>
     是一个终端多路复用器，可以让你在断开终端后重新连接到会话。
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 安装 screen</span>
<span class="token function">sudo</span> yum <span class="token function">install</span> <span class="token function">screen</span>

<span class="token comment"># 启动一个新的 screen 会话</span>
<span class="token function">screen</span> <span class="token parameter variable">-S</span> mysession

<span class="token comment"># 在 screen 会话中运行 Python 脚本</span>
python main.py

<span class="token comment"># 按下 Ctrl+A 然后按 D 来分离 screen 会话</span>

<span class="token comment"># 重新连接到 screen 会话</span>
<span class="token function">screen</span> <span class="token parameter variable">-r</span> mysession
</code></pre>
    <h4>
     <a id="3__systemd__33">
     </a>
     3. 使用
     <code>
      systemd
     </code>
     服务
    </h4>
    <p>
     <code>
      systemd
     </code>
     是 CentOS 的初始化系统，可以用来管理守护进程。
    </p>
    <ol>
     <li>
      创建一个新的服务文件：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">vi</span> /etc/systemd/system/my_python_service.service
</code></pre>
    <ol start="2">
     <li>
      在文件中添加以下内容：
     </li>
    </ol>
    <pre><code class="prism language-ini">[Unit]
Description=My Python Script
After=network.target

[Service]
ExecStart=/usr/bin/python3 /path/to/main.py
WorkingDirectory=/path/to/your/script
Restart=always
User=your_username

[Install]
WantedBy=multi-user.target
</code></pre>
    <ul>
     <li>
      <code>
       ExecStart
      </code>
      ：指定要运行的命令。
     </li>
     <li>
      <code>
       WorkingDirectory
      </code>
      ：指定脚本的工作目录。
     </li>
     <li>
      <code>
       Restart=always
      </code>
      ：确保脚本在崩溃或系统重启后自动重启。
     </li>
     <li>
      <code>
       User
      </code>
      ：指定运行脚本的用户。
     </li>
    </ul>
    <ol start="3">
     <li>
      <p>
       保存并退出编辑器。
      </p>
     </li>
     <li>
      <p>
       重新加载
       <code>
        systemd
       </code>
       配置：
      </p>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl daemon-reload
</code></pre>
    <ol start="5">
     <li>
      启动服务：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl start my_python_service
</code></pre>
    <ol start="6">
     <li>
      设置服务开机自启动：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl <span class="token builtin class-name">enable</span> my_python_service
</code></pre>
    <h4>
     <a id="4__tmux__84">
     </a>
     4. 使用
     <code>
      tmux
     </code>
     命令
    </h4>
    <p>
     <code>
      tmux
     </code>
     是另一个终端多路复用器，类似于
     <code>
      screen
     </code>
     。
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 安装 tmux</span>
<span class="token function">sudo</span> yum <span class="token function">install</span> tmux

<span class="token comment"># 启动一个新的 tmux 会话</span>
tmux new <span class="token parameter variable">-s</span> mysession

<span class="token comment"># 在 tmux 会话中运行 Python 脚本</span>
python main.py

<span class="token comment"># 按下 Ctrl+B 然后按 D 来分离 tmux 会话</span>

<span class="token comment"># 重新连接到 tmux 会话</span>
tmux attach <span class="token parameter variable">-t</span> mysession
</code></pre>
    <h4>
     <a id="_103">
     </a>
     总结
    </h4>
    <ul>
     <li>
      <code>
       nohup
      </code>
      是最简单的解决方案，适合临时任务。
     </li>
     <li>
      <code>
       screen
      </code>
      和
      <code>
       tmux
      </code>
      适合需要交互的会话。
     </li>
     <li>
      <code>
       systemd
      </code>
      是最健壮的解决方案，适合长期运行的守护进程。
     </li>
    </ul>
    <p>
     根据你的需求选择合适的方法。
    </p>
