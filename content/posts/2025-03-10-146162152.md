     【linux】【文件】文件权限基础
    </h2>
    <h3>
     <a id="1_Linux__1">
     </a>
     1. Linux 文件权限基础
    </h3>
    <p>
     每个文件或目录都有三类权限：
    </p>
    <ul>
     <li>
      <strong>
       所有者（Owner）
      </strong>
     </li>
     <li>
      <strong>
       用户组（Group）
      </strong>
     </li>
     <li>
      <strong>
       其他用户（Others）
      </strong>
     </li>
    </ul>
    <p>
     每类权限又有三种操作权限：
    </p>
    <ul>
     <li>
      <strong>
       读（r）
      </strong>
      ：读取文件内容或列出目录内容
     </li>
     <li>
      <strong>
       写（w）
      </strong>
      ：修改文件内容或目录（添加、删除文件）
     </li>
     <li>
      <strong>
       执行（x）
      </strong>
      ：执行文件或进入目录
     </li>
    </ul>
    <p>
     这三种权限的数值分别是：
    </p>
    <ul>
     <li>
      写（w） = 2
     </li>
     <li>
      执行（x） = 1
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2___777__19">
     </a>
     2. 数字表示法 —— 777 的含义
    </h3>
    <p>
     使用数字（八进制）表示权限时，三个数字分别对应所有者、用户组和其他用户的权限。
    </p>
    <ul>
     <li>
      <strong>
       7
      </strong>
      = 4 + 2 + 1，即读、写、执行权限都开启。
     </li>
     <li>
      因此，
      <strong>
       777
      </strong>
      表示：
      <ul>
       <li>
        所有者：读、写、执行
       </li>
       <li>
        用户组：读、写、执行
       </li>
       <li>
        其他用户：读、写、执行
       </li>
      </ul>
     </li>
    </ul>
    <p>
     这意味着所有人都对该文件或目录拥有完全的权限，这在某些情况下可能会带来安全风险，因此使用时需谨慎。
    </p>
    <hr/>
    <h3>
     <a id="3__32">
     </a>
     3. 修改文件权限的命令
    </h3>
    <p>
     <strong>
      chmod
     </strong>
     命令用于修改文件或目录的权限。基本语法如下：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> <span class="token punctuation">[</span>权限设置<span class="token punctuation">]</span> 文件或目录名
</code></pre>
    <h4>
     <a id="31__38">
     </a>
     3.1 数字表示法
    </h4>
    <p>
     例如，将文件
     <code>
      file.txt
     </code>
     的权限修改为 777，可以执行：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> <span class="token number">777</span> file.txt
</code></pre>
    <p>
     这条命令将赋予
     <strong>
      所有用户
     </strong>
     （所有者、用户组、其他人）对
     <code>
      file.txt
     </code>
     的读、写、执行权限。
    </p>
    <p>
     只有所有者有读、写权限，其它用户没有任何权限
    </p>
    <pre><code class="prism language-cpp">chmod <span class="token number">600</span> filename
</code></pre>
    <h4>
     <a id="32__51">
     </a>
     3.2 符号表示法
    </h4>
    <p>
     除了数字表示法外，还可以使用符号表示法来修改权限。例如：
    </p>
    <ul>
     <li>
      <strong>
       u
      </strong>
      代表所有者
     </li>
     <li>
      <strong>
       g
      </strong>
      代表用户组
     </li>
     <li>
      <strong>
       o
      </strong>
      代表其他用户
     </li>
     <li>
      <strong>
       a
      </strong>
      代表所有用户（相当于 u、g、o）
     </li>
    </ul>
    <p>
     符号表示法常见操作符：
    </p>
    <ul>
     <li>
      <strong>
       +
      </strong>
      表示添加权限
     </li>
     <li>
      <strong>
       -
      </strong>
      表示去掉权限
     </li>
     <li>
      <strong>
       =
      </strong>
      表示设置精确权限
     </li>
    </ul>
    <p>
     例如，下面的命令将为
     <strong>
      所有者
     </strong>
     添加执行权限，而不给其他用户增加权限：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> u+x file.txt
</code></pre>
    <p>
     而要设置所有用户拥有读、写、执行权限，可以写成：
    </p>
    <pre><code class="prism language-bash"><span class="token function">chmod</span> <span class="token assign-left variable">a</span><span class="token operator">=</span>rwx file.txt
</code></pre>
    <p>
     只有所有者有读、写权限，其它用户没有任何权限（在符号表示法中，go= 表示将用户组和其他用户的权限全部清除。）
    </p>
    <pre><code class="prism language-cpp">chmod u<span class="token operator">=</span>rw<span class="token punctuation">,</span>go<span class="token operator">=</span> filename
</code></pre>
    <hr/>
    <h3>
     <a id="4__81">
     </a>
     4 注意事项
    </h3>
    <ul>
     <li>
      <strong>
       安全性
      </strong>
      ：将文件或目录权限设置为 777 虽然方便，但可能会让系统暴露给未授权的用户，建议仅在必要时使用，且最好只针对特定目录或临时文件使用。
     </li>
     <li>
      <strong>
       用户和用户组管理
      </strong>
      ：有时你可能还需要调整文件的所有者或用户组，这可以通过
      <code>
       chown
      </code>
      和
      <code>
       chgrp
      </code>
      命令来完成，例如：
      <pre><code class="prism language-bash"><span class="token function">chown</span> username file.txt        <span class="token comment"># 修改所有者</span>
<span class="token function">chgrp</span> groupname file.txt        <span class="token comment"># 修改用户组</span>
</code></pre>
     </li>
    </ul>
