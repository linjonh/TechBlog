      </strong>

    </h4>
    <h4>
     <strong>
      1. WebShell的定义与原理
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        定义
       </strong>
       ：WebShell是一种基于Web脚本语言（如PHP、ASP、JSP）编写的恶意后门程序，通常被黑客用于在入侵网站后维持对服务器的控制。它通过Web请求执行命令，支持文件操作、数据库访问、系统命令执行等功能。
      </p>
     </li>
     <li>
      <p>
       <strong>
        原理
       </strong>
       ：通过HTTP请求传递参数（如
       <code>
        $_GET
       </code>
       、
       <code>
        $_POST
       </code>
       ），利用动态执行函数（如
       <code>
        eval()
       </code>
       、
       <code>
        system()
       </code>
       ）或文件包含漏洞执行恶意代码。例如，PHP的一句话木马
       <code>
        &lt;?php @eval($_POST['cmd']);?&gt;
       </code>
       可通过POST参数直接执行系统命令。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      2. WebShell的分类
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        按功能分
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          小马（一句话木马）
         </strong>
         ：体积小，功能单一，如
         <code>
          &lt;%eval request("cmd")%&gt;
         </code>
         ，需配合客户端工具（如中国菜刀）使用。
        </p>
       </li>
       <li>
        <p>
         <strong>
          大马
         </strong>
         ：功能复杂，集成文件管理、数据库操作等模块，可直接通过浏览器操作。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        按脚本语言分
       </strong>
       ：PHP、ASP、JSP、.NET等，不同语言的WebShell利用方式类似，但语法差异较大。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      3. WebShell的常见攻击方式
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        文件上传漏洞
       </strong>
       ：通过绕过上传限制（如修改文件后缀、
       <code>
        %00
       </code>
       截断、MIME类型伪造）上传恶意脚本。例如，利用Nginx解析漏洞将
       <code>
        .jpg
       </code>
       文件解析为PHP执行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数据库注入
       </strong>
       ：通过SQL注入将WebShell写入服务器文件，如MySQL的
       <code>
        SELECT ... INTO OUTFILE
       </code>
       命令313。
      </p>
     </li>
     <li>
      <p>
       <strong>
        配置文件篡改
       </strong>
       ：修改
       <code>
        .htaccess
       </code>
       或
       <code>
        php.ini
       </code>
       ，将图片或日志文件解析为脚本，如
       <code>
        SetHandler application/x-httpd-php
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        CMS漏洞利用
       </strong>
       ：针对开源CMS（如WordPress、PHPCMS）的已知漏洞上传WebShell，或通过后台模板编辑插入恶意代码。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <strong>
      4. WebShell的防御与检测
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        防御措施
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         限制上传文件类型，禁用危险函数（如
         <code>
          eval()
         </code>
         、
         <code>
          system()
         </code>
         ）。
        </p>
       </li>
       <li>
        <p>
         定期检查服务器日志，监控异常文件创建或修改时间。
        </p>
       </li>
       <li>
        <p>
         配置服务器权限，禁止Web目录执行写操作。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        检测工具
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          D盾
         </strong>
         ：专用于查杀WebShell，支持多种脚本语言。
        </p>
       </li>
       <li>
        <p>
         <strong>
          河马查杀
         </strong>
         ：基于AI的WebShell检测工具。
        </p>
       </li>
       <li>
        <p>
         <strong>
          手动排查
         </strong>
         ：通过文件哈希比对、代码审计识别可疑脚本。
        </p>
       </li>
      </ul>
     </li>
    </ul>
