    <h3>
     <a id="_6">
     </a>
     来源不同
    </h3>
    <ul>
     <li>
      <p>
       @Autowired的“爹”是Spring
      </p>
     </li>
     <li>
      <p>
       @Resource的“爹”是Java（JSR-250）
      </p>
      <ul>
       <li>
        这是一个规范，Spring对这个注解进行了支持。
       </li>
       <li>
        如果换了其他的框架，大概率用不了@Autowired（除非兼容了），大概率能用@Resource（毕竟是Java规范，框架应该会支持的）
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      当然了，这一点了解下就行了。毕竟实际开发中用的框架是Spring，这两个注解都能用。
     </p>
    </blockquote>
    <h3>
     <a id="bean_16">
     </a>
     查找bean的方式不同（重点）
    </h3>
    <h4>
     <a id="_18">
     </a>
     简单
    </h4>
    <ul>
     <li>
      <p>
       @Autowired：根据类型去查找bean，如果找到了多个，那么再根据名称去选择注入哪一个。
      </p>
      <ul>
       <li>
        <p>
         找到了多个，名称是helloService，不匹配，因此不知道注入哪一个。
        </p>
        <p>
         ​
         <img alt="image" src="https://i-blog.csdnimg.cn/direct/a00f025c76c94685a88bbfe5a9ebae7b.png">
          ​
         </img>
        </p>
       </li>
       <li>
        <p>
         将名称换成helloServiceImpl1，就能唯一匹配bean了。而且IDEA会提示实际注入了哪个bean（见下图的小红框。）
        </p>
        <p>
         ​
         <img alt="image" src="https://i-blog.csdnimg.cn/direct/4e83bb391cf548ba9d04a66eb26508b2.png">
          ​
         </img>
        </p>
        <blockquote>
         <p>
          虽然cursor很猛，但在Java领域，IDEA还是王者啊。
         </p>
        </blockquote>
       </li>
       <li>
        <p>
         不过，一般会用另一个注解来指定名称：
        </p>
        <pre><code class="prism language-java"><span class="token comment">/**
 * @author: Forrest
 * @since: 2025/3/15
 */</span>
<span class="token annotation punctuation">@RestController</span>
<span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">HelloController</span> <span class="token punctuation">{<!-- --></span>
    <span class="token annotation punctuation">@Qualifier</span><span class="token punctuation">(</span><span class="token string">"helloServiceImpl2"</span><span class="token punctuation">)</span>
    <span class="token annotation punctuation">@Autowired</span>
    <span class="token keyword">private</span> <span class="token class-name">HelloService</span> helloService<span class="token punctuation">;</span>

    <span class="token annotation punctuation">@GetMapping</span><span class="token punctuation">(</span><span class="token string">"/hello"</span><span class="token punctuation">)</span>
    <span class="token keyword">public</span> <span class="token class-name">String</span> <span class="token function">hello</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">return</span> helloService<span class="token punctuation">.</span><span class="token function">sayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
    <span class="token punctuation">}</span>
<span class="token punctuation">}</span>
</code></pre>
        <ul>
         <li>
          这个还是有使用场景的，例如要重构了，从helloServiceImpl1切换为helloServiceImpl2。当然了，如果很多地方都用到了helloService，这样修改是不方便的，应该在helloService的实现中完成路由，即helloServiceImpl1内部切换为helloServiceImpl2。（这样想，Service层的约定：接口 + 实现，非常鸡肋…）
         </li>
        </ul>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     ‍
    </p>
    <ul>
     <li>
      <p>
       @Resource：根据名称去查找bean，未找到，再根据类型去查找，如果找到多个，会抛异常。
      </p>
      <p>
       ​
       <img alt="image" src="https://i-blog.csdnimg.cn/direct/2db89f2734444d97babbb313707d3aa6.png">
        ​

      </p>
      <p>
       ​
       <img alt="image" src="https://i-blog.csdnimg.cn/direct/7cb5d0fa7b0c4125abc9fc0f981e1fbe.png">
        ​
       </img>
      </p>
      <blockquote>
       <p>
        左侧查看实际注入时，会发现有两个bean。
       </p>
      </blockquote>
      <ul>
       <li>
        <p>
         指定名称：
        </p>
        <p>
         ​
         <img alt="image" src="https://i-blog.csdnimg.cn/direct/121426b44ec34bdcbc1d168d09155c2a.png">
          ​
         </img>
        </p>
       </li>
       <li>
        <p>
         用@Qualifier更合适
        </p>
        <pre><code class="prism language-java"><span class="token comment">/**
 * @author: Forrest
 * @since: 2025/3/15
 */</span>
<span class="token annotation punctuation">@RestController</span>

        <span class="token keyword">return</span> helloService<span class="token punctuation">.</span><span class="token function">sayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>

    <h4>
     <a id="_87">
     </a>
     进阶
    </h4>
    <h5>
     <a id="AutowiredResourcePrimarySpring_Boot_337_89">
     </a>
     @Autowired、@Resource都支持@Primary（Spring Boot 3.3.7）
    </h5>
    <ul>
     <li>
      <p>
       上文提到：@Autowired先根据类型去查找bean，如果找到了多个，选哪一个呢？
      </p>
      <ul>
       <li>
        <p>
         如果多个bean中，有的bean被打上了@Primary，那么选这个bean。​
        </p>
        <p>
         <img alt="image" src="https://i-blog.csdnimg.cn/direct/3d0853e43fa348b9a73dc45251318bd4.png">
          ​​
         </img>
        </p>
        <p>
         <img alt="image" src="https://i-blog.csdnimg.cn/direct/8fbb18b1d1a445e7af5a9f15867b5702.png">
          ​
         </img>
        </p>
       </li>
       <li>
        <p>
         如果多个bean都被打上了@Primary呢？那继续根据名称来筛选。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       对于@Resource，优先按照名称去匹配，如果存在多个，那么匹配被打上@Primary的bean。
      </p>
     </li>
    </ul>

    <h3>
     <a id="_104">
     </a>
     使用范围不同（挺重要）
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        @Autowired
       </th>
       <th>
        @Resource
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         注解位置
        </strong>
       </td>
       <td>
        可以注解在字段、构造方法、Setter方法及方法参数上
       </td>
       <td>
        可以注解在字段、Setter方法上
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         集合类型注入
        </strong>
       </td>
       <td>
        支持，可以用于注入集合类型（如
        <code>
         List&lt;XXX&gt;
        </code>
        ​），自动注入所有符合类型的Bean
       </td>
       <td>
        也支持
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         数组类型注入
        </strong>
       </td>
       <td>
        支持，可以用于注入数组类型，自动注入所有符合类型的Bean
       </td>
       <td>
        也支持
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         构造方法注入
        </strong>
       </td>
       <td>
        支持，可以用于构造方法参数的注入
       </td>
       <td>
        不支持构造方法注入
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     <a id="_113">
     </a>
     注解的参数不同
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        @Autowired
       </th>
       <th>
        @Resource
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         支持的参数
        </strong>
       </td>
       <td>
        只支持
        <code>
         required
        </code>
        ​一个参数
       </td>
       <td>
        支持
        <code>
         name
        </code>
        ​、
        <code>
         type
        </code>
        ​等7个参数
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         依赖存在性
        </strong>
       </td>
       <td>
        默认依赖的Bean必须存在，可以通过
        <code>
         required=false
        </code>
        ​设置为非必须
       </td>
       <td>
        默认依赖的Bean必须存在，不存在会抛出异常
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      <p>
       @Autowired(required = false) 不是说就不注入依赖了，而是一旦注入失败了， 那就不注入了。也就是不要因为弱依赖而影响了主流程。
      </p>
     </li>
     <li>
      <p>
       对于@Resource，用的最多的是
       <code>
        name
       </code>
       ​、
       <code>
        type
       </code>
       ​。
      </p>
      <ul>
       <li>
        <p>
         @Resource默认是按名称去查找bean的，因此，最好用name明确这个名称。
        </p>
        <pre><code class="prism language-java"><span class="token annotation punctuation">@Resource</span><span class="token punctuation">(</span>type <span class="token operator">=</span> <span class="token class-name">HelloServiceImpl1</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">)</span>
<span class="token keyword">private</span> <span class="token class-name">HelloService</span> helloService<span class="token punctuation">;</span>
</code></pre>
       </li>
      </ul>
     </li>
    </ul>
