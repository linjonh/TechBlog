     gitsubtree怎么添加新的子仓库
    </h1>

 </div>

  <div class="article_content clearfix" id="article_content">

    <p>
     要使用
     <code>
      git subtree
     </code>
     添加一个新的子仓库，可以按照以下步骤操作：
    </p>
    <h4>
     <a id="1__2">
     </a>
     1. 添加子仓库
    </h4>
    <p>
     使用
     <code>
      git subtree add
     </code>
     命令将子仓库的内容添加到主仓库的指定目录中。命令格式如下：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree <span class="token function">add</span> <span class="token parameter variable">--prefix</span><span class="token operator">=</span><span class="token operator">&lt;</span>子目录路径<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库地址<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库分支<span class="token operator">&gt;</span>
</code></pre>
    <ul>
     <li>
      <code>
       --prefix=&lt;子目录路径&gt;
      </code>
      ：指定子仓库在主仓库中的目录路径。
     </li>
     <li>
      <code>
       &lt;子仓库地址&gt;
      </code>
      ：子仓库的 Git 仓库 URL。
     </li>
     <li>
      <code>
       &lt;子仓库分支&gt;
      </code>
      ：子仓库中要添加的分支名称。
     </li>
    </ul>
    <h5>
     <a id="_11">
     </a>
     示例
    </h5>
    <p>
     假设你希望将
     <code>
      https://github.com/example/subproject.git
     </code>
     的
     <code>
      main
     </code>
     分支作为子仓库添加到主仓库的
     <code>
      subproject
     </code>
     目录中，可以执行以下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree <span class="token function">add</span> <span class="token parameter variable">--prefix</span><span class="token operator">=</span>subproject https://github.com/example/subproject.git main
</code></pre>
    <h4>
     <a id="2__17">
     </a>
     2. 提交更改
    </h4>
    <p>
     添加子仓库后，Git 会自动创建一个提交，将子仓库的内容合并到主仓库中。你可以通过以下命令查看并提交这些更改：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> status
<span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"Add subproject as a subtree"</span>
<span class="token function">git</span> push
</code></pre>
    <h4>
     <a id="3__25">
     </a>
     3. 更新子仓库
    </h4>
    <p>
     如果子仓库有新的更改，可以使用以下命令将这些更改拉取到主仓库中：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree pull <span class="token parameter variable">--prefix</span><span class="token operator">=</span><span class="token operator">&lt;</span>子目录路径<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库地址<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库分支<span class="token operator">&gt;</span>
</code></pre>
    <p>
     例如：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree pull <span class="token parameter variable">--prefix</span><span class="token operator">=</span>subproject https://github.com/example/subproject.git main
</code></pre>
    <p>
     这会将子仓库的最新更改合并到主仓库中。
    </p>
    <h4>
     <a id="4__36">
     </a>
     4. 推送子仓库的更改
    </h4>
    <p>
     如果你在主仓库的子目录中进行了更改，并希望将这些更改推送到子仓库中，可以使用以下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree push <span class="token parameter variable">--prefix</span><span class="token operator">=</span><span class="token operator">&lt;</span>子目录路径<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库地址<span class="token operator">&gt;</span> <span class="token operator">&lt;</span>子仓库分支<span class="token operator">&gt;</span>
</code></pre>
    <p>
     例如：
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> subtree push <span class="token parameter variable">--prefix</span><span class="token operator">=</span>subproject https://github.com/example/subproject.git main
</code></pre>
    <p>
     这会将主仓库中子目录的更改推送到子仓库的指定分支。
    </p>
    <h4>
     <a id="_47">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      如果希望在添加子仓库时简化提交历史，可以使用
      <code>
       --squash
      </code>
      选项。这会将子仓库的提交历史压缩为一个单一的提交。
     </li>
     <li>
      确保子仓库的分支名称和路径正确无误，否则可能会导致命令失败。
     </li>
    </ul>
    <p>
     通过以上步骤，你可以轻松地将一个新的子仓库添加到主仓库中，并在需要时同步更改。
    </p>
