    <h3>
     类的6个默认成员函数
    </h3>
    <p>
     <img alt="" height="530" src="https://i-blog.csdnimg.cn/direct/73a29360f8194b7cba531ba74b3a2c68.png" width="1492"/>
    </p>
    <h2>
     构造函数
    </h2>
    <blockquote>
     <p>
      是对一个对象实例化时的
      <strong>
       初始化
      </strong>
     </p>
    </blockquote>
    <p>
     <strong>
      例如在C语言中写的堆的时候要初始化StackInit，而c++祖师爷写的构造函数本质上就是自动调用初始化。
     </strong>
    </p>
    <blockquote>
     <p>
      <strong>
       构造函数=默认构造函数+自己写的（符合规定的显示表达式）
      </strong>
     </p>
    </blockquote>
    <blockquote>
     <p>
      注：一般情况下构造函数都需要自己写少数情况下，默认生成就可以用，例如队列。
     </p>
    </blockquote>
    <h3>
     写法特点
    </h3>
    <p>
     <strong>
      <span style="background-color:#edf6e8">
       一、构造函数的函数名与类名相同
      </span>
     </strong>
     <br/>
     <strong>
      <span style="background-color:#edf6e8">
       二、构造函数无返回值
      </span>
     </strong>
     <br/>
     这里所说的构造函数无返回值是真的无返回值，而不是说返回值为void。
     <br/>
     <strong>
      <span style="background-color:#edf6e8">
       三、对象实例化时编译器自动调用对应的构造函数
      </span>
     </strong>
     <br/>
     当你用类创建一个对象时，编译器会自动调用该类的构造函数对新创建的变量进行初始化。
     <br/>
     <strong>
      <span style="background-color:#edf6e8">
       四、构造函数支持重载
      </span>
     </strong>
     <br/>
     这意味着你可以有多种初始化对象的方式，编译器会根据你所传递的参数去调用对应的构造函数。
     <br/>
     <strong>
      <span style="background-color:#edf6e8">
       五、无参的构造函数、全缺省的构造函数以及我们不写编译器自动生成的构造函数都称为默认构造函数，并且默认构造函数只能有一个
      </span>
     </strong>
     <br/>
     初学C++时，你可能认为只有当我们不写，编译器自动生成的构造函数才被称为默认构造函数。其实并不是这样的，以下3种都叫做默认构造函数：
     <br/>
     1、我们不写，编译器自动生成的构造函数。
     <br/>
     2、我们自己写的无参的构造函数。
     <br/>
     3、我们自己写的全缺省的构造函数。
     <br/>
     总而言之，无需传参就可以调用的构造函数就是默认构造函数
    </p>
    <p>
     你有没有这种想法，既然在我们不写的情况下，编译器会自动生成一个构造函数，那我们就没有必要自己写构造函数了。这个是❌的。
    </p>
    <p>
     例如：
    </p>
    <pre><code>#include &lt;iostream&gt;
using namespace std;
class Date
{
public:
	void Print()
	{
		cout &lt;&lt; _year &lt;&lt; "年" &lt;&lt; _month &lt;&lt; "月" &lt;&lt; _day &lt;&lt; "日" &lt;&lt; endl;
	}
private:
	int _year;
	int _month;
	int _day;
};
int main()
{
	Date d1; // 编译器将调用自动生成的默认构造函数对d1进行初始化
	d1.Print();
	return 0;
}
</code></pre>
    <p>
     结果是乱码：
    </p>
    <p>
     <img alt="" height="61" src="https://i-blog.csdnimg.cn/direct/9a42f41677bf4425bb76e88ba3934a13.png" width="464"/>
    </p>
    <p>
     <strong>
      <span style="background-color:#d4e9d5">
       六、如果类中没有显示定义构造函数，则C++编译器会自动生成一个无参的默认构造函数，若用户显示定义了，则编译器就不再生成
      </span>
     </strong>
    </p>
    <h3 style="background-color:transparent">
     3.默认构造函数
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        1.⽆参构造函数、全缺省构造函数、我们不写构造时编译器默认⽣成的构造函数，都叫做默认构造函 数。但是这三个函数有且只有⼀个存在，不能同时存在
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        2.⽆参构造函数和全缺省构造函数虽然构成函数重载，但是调⽤时会。存在问题
       </span>
      </strong>
     </span>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       <span style="background-color:#edf6e8">
        在
        <span contenteditable="false" tabindex="-1">
         <a href="https://blog.csdn.net/LJY_CF/article/details/142633268?spm=1001.2014.3001.5502" rel="noopener noreferrer" target="_blank" title="缺省参数（备胎）-CSDN博客">
          缺省参数（备胎）-CSDN博客
         </a>
        </span>
        有相关解释
       </span>
      </span>
     </strong>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        3.总结就是不传实参就能调用的就是就叫默认构造函数
       </span>
      </strong>
     </span>
    </p>
    <h4>
     3.1⽆参构造函数
    </h4>
    <pre><code class="language-cpp">#include&lt;iostream&gt;
using namespace std;
class liu
{
public:
// 1.⽆参构造函数
liu()
{
_year = 1;
_month = 1;
_day = 1;
}
private:
int _year;
int _month;
int _day;
};
</code></pre>
    <h4>
     3.2全缺省构造函数
    </h4>
    <pre><code class="language-cpp">#include&lt;iostream&gt;
using namespace std;
class liu
{
public:
// 3.全缺省构造函数
liu(int year = 1, int month = 1, int day = 1)
{
_year = year;
_month = month;
_day = day;
}
private:
int _year;
int _month;
int _day;
};
</code></pre>
    <h4>
     3.3编译器默认⽣成的构造函数
    </h4>
    <pre><code class="language-cpp">#include&lt;iostream&gt;
using namespace std;
class liu
{
liu(){};
int _year;
int _month;
int _day;
};
</code></pre>
    <pre><code class="language-cpp">int main()
{
liu l1;
return 0;
}</code></pre>
    <blockquote>
     <p>
      注意：如果通过⽆参构造函数创建对象时，
      <strong>
       对象后⾯不⽤跟括号
      </strong>
      ，否则编译器⽆法
     </p>
    </blockquote>
    <blockquote>
     <p>
      这是一个空的函数，即什么都不做。
     </p>
     <p>
      默认构造函数也不是空的，这里只是简化一下，不作探讨。
     </p>
    </blockquote>
    <h2>
     总结：
    </h2>
    <p>
     <strong>
      1、编译器自动生成的构造函数对内置类型不做处理。
      <br/>
      2、对于自定义类型，编译器会再去调用它们自己的默认构造函数。
     </strong>
    </p>
    <hr/>
    <h2 style="background-color:transparent">
     析构函数
    </h2>
    <p>
     析构函数与构造函数功能相反，析构函数不是完成对对象本⾝的销毁，⽐如局部对象是存在栈帧的，函数结束栈帧销毁，他就释放了，不需要我们管，C++规定对象在销毁时会⾃动调⽤析构函数，完成对象中资源的清理释放⼯作。
    </p>
    <h3>
     特点八条
    </h3>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        1. 析构函数名是在类名前加上字符~。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        2. ⽆参数⽆返回值。(这⾥跟构造类似，也不需要加void)
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        3. ⼀个类只能有⼀个析构函数。若未显式定义，系统会⾃动⽣成默认的析构函数。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        4. 对象⽣命周期结束时，系统会⾃动调⽤析构函数。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        5. 跟构造函数类似，我们不写编译器⾃动⽣成的析构函数对内置类型成员不做处理，⾃定类型成员会 调⽤他的析构函数。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        6. 还需要注意的是我们显⽰写析构函数，对于⾃定义类型成员也会调⽤他的析构，也就是说⾃定义类 型成员⽆论什么情况都会⾃动调⽤析构函数。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        7. 如果类中没有申请资源时，析构函数可以不写，直接使⽤编译器⽣成的默认析构函数。
       </span>
      </strong>
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      <strong>
       <span style="background-color:#edf6e8">
        8. ⼀个局部域的多个对象，C++规定后定义的先析构。 给出每一点的实例分析
       </span>
      </strong>
     </span>
    </p>
    <p>
     <strong>
      1. 析构函数名是在类名前加上字符
      <code>
       ~
      </code>
     </strong>
    </p>
    <p>
     析构函数的名称是类名前加上
     <code>
      ~
     </code>
     符号，用于标识该函数是析构函数。
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
public:
    // 析构函数
    ~MyClass() {
        std::cout &lt;&lt; "MyClass 析构函数被调用" &lt;&lt; std::endl;
    }
};

int main() {
    MyClass obj;
    return 0;
}</code></pre>
    <p>
     在上述代码中，
     <code>
      ~MyClass()
     </code>
     就是
     <code>
      MyClass
     </code>
     类的析构函数。
    </p>
    <hr/>
    <p>
     <strong>
      2. ⽆参数⽆返回值
     </strong>
    </p>
    <p>
     析构函数不能有参数，也没有返回值，并且不需要显式
     <strong>
      指定
      <code>
       void
      </code>
     </strong>
     。
    </p>
    <hr/>
    <p>
     <strong>
      3. ⼀个类只能有⼀个析构函数。若未显式定义，系统会⾃动⽣成默认的析构函数
     </strong>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
    // 未显式定义析构函数，编译器会自动生成默认析构函数
};

int main() {
    MyClass obj;
    return 0;
}</code></pre>
    <p>
     在上述代码中，
     <code>
      MyClass
     </code>
     类没有显式定义析构函数，编译器会自动生成一个默认的析构函数。
    </p>
    <hr/>
    <p>
     <strong>
      4. 对象⽣命周期结束时，系统会⾃动调⽤析构函数
     </strong>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
public:
    ~MyClass() {
        std::cout &lt;&lt; "MyClass 析构函数被调用" &lt;&lt; std::endl;
    }
};

void test() {
    MyClass obj; // obj 的生命周期在 test 函数结束时结束
}

int main() {
    test();
    std::cout &lt;&lt; "test 函数调用结束" &lt;&lt; std::endl;
    return 0;
}</code></pre>
    <p>
     在
     <code>
      test
     </code>
     函数中，
     <code>
      obj
     </code>
     对象的生命周期在
     <code>
      test
     </code>
     函数结束时结束，此时系统会自动调用
     <code>
      MyClass
     </code>
     的析构函数。
    </p>
    <hr/>
    <p>
     <strong>
      5. 跟构造函数类似，我们不写编译器⾃动⽣成的析构函数对内置类型成员不做处理，⾃定类型成员会调⽤他的析构函数
     </strong>
    </p>
    <p>
     如果不显式定义析构函数，编译器生成的默认析构函数对内置类型成员不做处理，对自定义类型成员会调用其析构函数。
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class InnerClass {
public:
    ~InnerClass() {
        std::cout &lt;&lt; "InnerClass 析构函数被调用" &lt;&lt; std::endl;
    }
};

class OuterClass {
private:
    int num; // 内置类型成员
    InnerClass inner; // 自定义类型成员
public:
    // 未显式定义析构函数，使用编译器生成的默认析构函数
};

int main() {
    OuterClass obj;
    return 0;
}</code></pre>
    <blockquote>
     <p>
      在上述代码中，
      <code>
       OuterClass
      </code>
      未显式定义析构函数，编译器生成的默认析构函数对
      <code>
       num
      </code>
      不做处理，对
      <code>
       inner
      </code>
      会调用
      <code>
       InnerClass
      </code>
      的析构函数。
     </p>
    </blockquote>
    <p>
    </p>
    <hr/>
    <p>
     <strong>
      6.即使显式定义析构函数，自定义类型成员也会自动调用其析构函数。
     </strong>
    </p>
    <p>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class InnerClass {
public:
    ~InnerClass() {
        std::cout &lt;&lt; "InnerClass 析构函数被调用" &lt;&lt; std::endl;
    }
};

class OuterClass {
private:
    InnerClass inner; // 自定义类型成员
public:
    ~OuterClass() {
        std::cout &lt;&lt; "OuterClass 析构函数被调用" &lt;&lt; std::endl;
    }
};

int main() {
    OuterClass obj;
    return 0;
}</code></pre>
    <p>
     <img alt="" height="69" src="https://i-blog.csdnimg.cn/direct/243eede7b7b34c4ba9461473fae0cd93.png" width="404"/>
    </p>
    <p>
     在上述代码中，
     <code>
      OuterClass
     </code>
     显式定义了析构函数，在
     <code>
      OuterClass
     </code>
     的析构函数调用后，会自动调用
     <code>
      InnerClass
     </code>
     的析构函数。
    </p>
    <p>
    </p>
    <hr/>
    <p>
     <strong>
      7. 如果类中没有申请资源时，析构函数可以不写，直接使⽤编译器⽣成的默认析构函数；如果有资源申请时，⼀定要⾃⼰写析构，否则会造成资源泄漏
     </strong>
    </p>
    <p>
     如果类中没有申请资源，如
     <code>
      Date
     </code>
     类，可使用默认析构函数；如果有资源申请，如
     <code>
      Stack
     </code>
     类，必须自己写析构函数。
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
// Date 类，没有申请资源
class Date {
private:
    int year;
    int month;
    int day;
public:
    // 不需要显式定义析构函数，使用默认析构函数
};

// Stack 类，申请了资源
class Stack {
private:
    int* data;
    int size;
public:
    Stack() {
        data = new int[10];
        size = 0;
    }
    ~Stack() {
        delete[] data; // 释放申请的内存
        std::cout &lt;&lt; "Stack 析构函数被调用，释放内存" &lt;&lt; std::endl;
    }
};

int main() {
    Date d;
    Stack s;
    return 0;
}</code></pre>
    <blockquote>
     <p>
      在上述代码中，
      <code>
       Date
      </code>
      类没有申请资源，使用默认析构函数；
      <code>
       Stack
      </code>
      类申请了动态内存，需要自己写析构函数来释放内存，否则会造成资源泄漏。
     </p>
    </blockquote>
    <hr/>
    <p>
     <strong>
      8. ⼀个局部域的多个对象，C++ 规定后定义的先析构
     </strong>
    </p>
    <p>
     <strong>
      在一个局部域中，后定义的对象先析构。
     </strong>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
private:
    int id;
public:
    MyClass(int i) : id(i) {
        std::cout &lt;&lt; "MyClass " &lt;&lt; id &lt;&lt; " 构造函数被调用" &lt;&lt; std::endl;
    }
    ~MyClass() {
        std::cout &lt;&lt; "MyClass " &lt;&lt; id &lt;&lt; " 析构函数被调用" &lt;&lt; std::endl;
    }
};

int main() {
    MyClass obj1(1);
    MyClass obj2(2);
    return 0;
}</code></pre>
    <p>
     在上述代码中，
     <code>
      obj2
     </code>
     后定义，会先析构，输出结果中
     <code>
      MyClass 2 析构函数被调用
     </code>
     会先于
     <code>
      MyClass 1 析构函数被调用
     </code>
     输出。
    </p>
    <hr/>
    <h2>
     拷贝构造
    </h2>
    <h3>
     特点六条
    </h3>
    <p>
     果⼀个构造函数的第⼀个参数是⾃⾝类类型的引⽤，且任何额外的参数都有默认值，则此构造函数 也叫做拷⻉构造函数，也就是说拷⻉构造是⼀个特殊的构造函数。
    </p>
    <p>
     <strong>
      1. 拷贝构造函数是构造函数的一个重载
     </strong>
    </p>
    <p>
     拷贝构造函数是一种特殊的构造函数，用于创建一个新对象，该对象是另一个同类型对象的副本。它是构造函数的一种重载形式。
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
private:
    int data;
public:
    // 普通构造函数
    MyClass(int val) : data(val) {
        std::cout &lt;&lt; "普通构造函数被调用" &lt;&lt; std::endl;
    }
    // 拷贝构造函数
    MyClass(const MyClass&amp; other) : data(other.data) {
        std::cout &lt;&lt; "拷贝构造函数被调用" &lt;&lt; std::endl;
    }
};

int main() {
    MyClass obj1(10);
    MyClass obj2(obj1); // 调用拷贝构造函数
    return 0;
}</code></pre>
    <p>
     <img alt="" height="92" src="https://i-blog.csdnimg.cn/direct/d87603a1352a4bbb92457946b97588f8.png" width="317"/>
    </p>
    <p>
     <strong>
      2. 拷贝构造函数的第一个参数必须是类类型对象的引用，使用传值方式编译器直接报错，因为语法逻辑上会引发无穷递归调用
     </strong>
    </p>
    <p>
     拷贝构造函数的第一个参数必须是类类型对象的引用，若使用传值方式会导致无穷递归调用。
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
private:
    int data;
public:
    // 错误示例：传值方式作为拷贝构造函数参数，会引发编译错误
    // MyClass(MyClass other) : data(other.data) {}

    // 正确示例：使用引用作为拷贝构造函数参数
    MyClass(const MyClass&amp; other) : data(other.data) {
        std::cout &lt;&lt; "拷贝构造函数被调用" &lt;&lt; std::endl;
    }
};

int main() {
    MyClass obj1(10);
    MyClass obj2(obj1); // 调用拷贝构造函数
    return 0;
}</code></pre>
    <p>
     <strong>
      如果使用传值方式
      <code>
       MyClass(MyClass other)
      </code>
      ，当调用拷贝构造函数时，为了传递
      <code>
       other
      </code>
      参数，又会调用拷贝构造函数，从而导致无穷递归调用。
     </strong>
    </p>
    <p>
     <img alt="" height="481" src="https://i-blog.csdnimg.cn/direct/4176c2c3ecdc42c2bbda99044ce64b27.png" width="901"/>
    </p>
    <p>
     c++规定，拷贝构造之前要传值传参。
    </p>
    <p>
     <strong>
      3.当自定义类型对象进行传值传参或传值返回时，会调用拷贝构造函数。
     </strong>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class MyClass {
private:
    int data;
public:
    MyClass(int val) : data(val) {
        std::cout &lt;&lt; "普通构造函数被调用" &lt;&lt; std::endl;
    }
    MyClass(const MyClass&amp; other) : data(other.data) {
        std::cout &lt;&lt; "拷贝构造函数被调用" &lt;&lt; std::endl;
    }
};

// 传值传参
void func(MyClass obj) {
    // 函数体
}

// 传值返回
MyClass createObj() {
    MyClass obj(10);
    return obj;
}

int main() {
    MyClass obj1(10);
    func(obj1); // 传值传参，调用拷贝构造函数
    MyClass obj2 = createObj(); // 传值返回，调用拷贝构造函数
    return 0;
}</code></pre>
    <p>
     <img alt="" height="94" src="https://i-blog.csdnimg.cn/direct/afd380065f0348149fcda763bd79dec1.png" width="387">
     </img>
    </p>
    <p>
     在上述代码中，
     <code>
      func
     </code>
     函数的传值传参和
     <code>
      createObj
     </code>
     函数的传值返回都会调用拷贝构造函数。
    </p>
    <p>
     <strong>
      4. 若未显式定义拷贝构造，编译器会自动生成拷贝构造函数。
     </strong>
    </p>
    <p>
     自动生成的拷贝构造对内置类型成员变量会完成值拷贝 / 浅拷贝 (一个字节一个字节的拷贝)，对自定义类型成员变量会调用他的拷贝构造
    </p>
    <p>
    </p>
    <pre><code class="hljs">#include &lt;iostream&gt;
class InnerClass {
private:
    int innerData;
public:
    InnerClass(int val) : innerData(val) {
        std::cout &lt;&lt; "InnerClass 普通构造函数被调用" &lt;&lt; std::endl;
    }
    InnerClass(const InnerClass&amp; other) : innerData(other.innerData) {
        std::cout &lt;&lt; "InnerClass 拷贝构造函数被调用" &lt;&lt; std::endl;
    }
};

class OuterClass {
private:
    int outerData;
    InnerClass inner;
public:
    OuterClass(int outerVal, int innerVal) : outerData(outerVal), inner(innerVal) {
        std::cout &lt;&lt; "OuterClass 普通构造函数被调用" &lt;&lt; std::endl;
    }
    // 未显式定义拷贝构造函数，使用编译器自动生成的拷贝构造函数
};

int main() {
    OuterClass obj1(10, 20);
    OuterClass obj2(obj1); // 调用编译器自动生成的拷贝构造函数
    return 0;
}</code></pre>
    <p>
     <img alt="" height="121" src="https://i-blog.csdnimg.cn/direct/7e3cf0ebe67140588164c3e147e50692.png" width="451"/>
    </p>
    <p>
     在上述代码中，
     <code>
      OuterClass
     </code>
     未显式定义拷贝构造函数，编译器自动生成的拷贝构造函数会对
     <code>
      outerData
     </code>
     进行值拷贝，对
     <code>
      inner
     </code>
     调用
     <code>
      InnerClass
     </code>
     的拷贝构造函数。
    </p>
    <p>
     <strong>
      6.并不是所有内置类型都能用拷贝构造的
     </strong>
    </p>
    <p>
     例如：栈Stack，
     <img alt="" height="98" src="https://i-blog.csdnimg.cn/direct/b4496f9deb5e4994aca53b278b8d6870.png" width="191">
      是内置类型；
     </img>
    </p>
    <p>
     现在来拷贝构造，发现报错了，这是为什么呢。
    </p>
    <p>
     <img alt="" height="961" src="https://i-blog.csdnimg.cn/direct/4618b4158eb44f528418beb604b85859.png" width="921"/>
    </p>
    <p style="background-color:transparent">
     <strong>
      错误原因分析
     </strong>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        默认拷贝构造函数的问题
       </strong>
       ：在
       <code>
        main
       </code>
       函数中执行
       <code>
        Stack st2(st1);
       </code>
       时，由于没有自定义拷贝构造函数，编译器会生成一个默认的拷贝构造函数。默认拷贝构造函数只会进行浅拷贝，即简单地将
       <code>
        st1
       </code>
       中成员变量的值复制给
       <code>
        st2
       </code>
       。对于
       <code>
        _a
       </code>
       这个指针成员变量，浅拷贝后
       <code>
        st1._a
       </code>
       和
       <code>
        st2._a
       </code>
       指向同一块堆内存。
      </p>
     </li>
     <li>
      <p>
       <strong>
        析构函数的重复释放
       </strong>
       ：当程序结束时，
       <code>
        st1
       </code>
       和
       <code>
        st2
       </code>
       会依次调用析构函数。由于它们的
       <code>
        _a
       </code>
       指针指向同一块内存，第一次调用析构函数（比如
       <code>
        st1
       </code>
       的析构函数）时释放了这块内存，当
       <code>
        st2
       </code>
       调用析构函数时，再次对已经释放的内存调用
       <code>
        free
       </code>
       ，就会导致运行时错误。
      </p>
     </li>
     <li>
      <img alt="" height="302" src="https://i-blog.csdnimg.cn/direct/e836fbeac670427b90286bc392f943e6.png" width="1092"/>
      需要自己写拷贝构造
      <pre><code class="hljs">// 自定义拷贝构造函数
    Stack(const Stack&amp; other)
    {
        _capacity = other._capacity;
        _top = other._top;
        _a = (int*)malloc(sizeof(int) * _capacity);
        if (nullptr == _a)
        {
            perror("malloc申请空间失败");
            return;
        }
        for (size_t i = 0; i &lt; _top; ++i)
        {
            _a[i] = other._a[i];
        }
    }</code></pre>
      <blockquote>
       <p>
        总结： 传值返回会产生一个临时对象调用拷贝构造，传值引用返回，返回的是返回对象的别名 (引用)，没有产生拷贝。但是如果返回对象是一个当前函数局部域的局部对象，函数结束就销毁了，那么使用引用返回是有问题的，这时的引用相当于一个野引用，类似一个野指针一样。传引用返回可以减少拷贝，但是一定要确保返回对象，在当前函数结束后还在，才能用引用返回。
       </p>
      </blockquote>
     </li>
    </ol>
