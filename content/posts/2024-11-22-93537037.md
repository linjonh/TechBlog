    <h1 class="title-article" id="articleContentId">
     什么是云原生架构
    </h1>
   </div>
  </div>
 </div>

  <div class="article_content clearfix" id="article_content">

    <div class="blogpost-body cnblogs-markdown" id="cnblogs_post_body">
     <blockquote>
      <p>
       本文简要介绍了云原生架构的定义和个人对云原生架构的理解。
       <br/>
       个人认为，云原生架构应该包括2大部分：云原生平台和云原生应用。
      </p>
     </blockquote>
     <p>
      云计算提供了对无限IT资源的按需付费的商业模式，但从技术架构上看，还需要一个用于构建和运行云原生应用的平台，来实践敏捷开发、DevOps、容器编排，微服务和容器化等理论和方法。
     </p>
     <p>
      <img alt="1428428-20181025185543772-30550163.png" src="https://i-blog.csdnimg.cn/blog_migrate/199cbdc0685366fafe5c388fde39c3bc.png"/>
     </p>
     <h2 id="云原生平台">
      云原生平台
     </h2>
     <h4 id="敏捷开发">
      敏捷开发
     </h4>
     <p>
      一种小规模团队的、全栈式的开发方法，要求团队具备快速响应变化，快速迭代开发的能力。
     </p>
     <p>
      <em>
       最佳实践
      </em>
     </p>
     <ul>
      <li>
       scrum
      </li>
      <li>
       xp
      </li>
     </ul>
     <h4 id="devops">
      DevOps
     </h4>
     <p>
      开发和运维之间保持流程连续的协作方法，目标是快速、频繁且更可靠地构建、测试和发布软件。
     </p>
     <p>
      <em>
       最佳实践
      </em>
     </p>
     <ul>
      <li>
       Jenkins
      </li>
      <li>
       GitLab
      </li>
     </ul>
     <h4 id="容器编排">
      容器编排
     </h4>
     <p>
      一种容器资源的管理方法，目标是管理容器集群和调度容器化应用。
     </p>
     <p>
      <em>
       最佳实践
      </em>
     </p>
     <ul>
      <li>
       Kubernetes
      </li>
      <li>
       Docker Swarm
      </li>
      <li>
       Mesos
      </li>
     </ul>
     <h2 id="云原生应用">
      云原生应用
     </h2>
     <h4 id="微服务">
      微服务
     </h4>
     <p>
      是将大型应用作为小型服务集合进行开发的架构方法，其中每个服务都可实现业务功能，在自己的流程中运行并通过 HTTP API 进行通信。每个微服务都可以独立于其它服务进行部署、升级、扩展和重新启动，通常作为自动化系统的一部分运行，可以在不影响最终客户的情况下频繁更新正在使用中的应用。
     </p>
     <p>
      <em>
       最佳实践
      </em>
     </p>
     <ul>
      <li>
       Spring Boot
      </li>
      <li>
       Spring Cloud
      </li>
      <li>
       Jhipster
      </li>
     </ul>
     <h4 id="容器化">
      容器化
     </h4>
     <p>
      与虚拟机相比，容器能同时提供更好的效率和启动速度。每个容器都具有唯一的可写文件系统和资源配额。创建和删除容器的开销较低，在单个虚拟机上能通过容器化充分利用物力资源，这使的容器成为部署微服务的完美工具。
     </p>
     <p>
      <em>
       最佳实践
      </em>
     </p>
     <ul>
      <li>
       Docker Image
      </li>
      <li>
       OCI
      </li>
     </ul>
