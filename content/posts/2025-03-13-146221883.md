    <p>
     在Java中， TypeReference 通常与JSON解析库一起使用，用于在解析JSON数据时指定复杂的泛型类型。以FastJSON为例，当需要将JSON数据解析为具有复杂泛型类型的对象时， TypeReference 能帮助准确地告知解析器目标类型。
    </p>
    <p>
     以下是一个示例：
    </p>
    <p>
     java
    </p>
    <p>
     import com.alibaba.fastjson2.JSON;
     <br/>
     import com.alibaba.fastjson2.TypeReference;
     <br/>
     import java.util.List;
     <br/>
     import java.util.Map;
    </p>
    <p>
     public class TypeReferenceExample {
     <!-- -->
     <br/>
     public static void main(String[] args) {
     <!-- -->
     <br/>
     String json = “[{“name”:“John”,“age”:30},{“name”:“Alice”,“age”:25}]”;
     <br/>
     // 使用TypeReference指定目标类型为List&lt;Map&lt;String, Object&gt;&gt;
     <br/>
     List&lt;Map&lt;String, Object&gt;&gt; list = JSON.parseObject(json, new TypeReference&lt;List&lt;Map&lt;String, Object&gt;&gt;&gt;() {});
     <br/>
     for (Map&lt;String, Object&gt; map : list) {
     <!-- -->
     <br/>
     System.out.println(map.get(“name”) + " - " + map.get(“age”));
     <br/>
     }
     <br/>
     }
     <br/>
     }
    </p>
    <p>
     在上述代码中， TypeReference&lt;List&lt;Map&lt;String, Object&gt;&gt;&gt; 明确指定了要将JSON数据解析为 List ，其中每个元素是 Map&lt;String, Object&gt; 类型。这样，FastJSON就能正确地解析JSON字符串并将其转换为指定的复杂类型。
    </p>
    <p>
     如果不使用 TypeReference ，对于复杂的泛型类型，JSON解析库可能无法准确推断目标类型，从而导致解析结果不正确或出现类型转换错误。
    </p>
