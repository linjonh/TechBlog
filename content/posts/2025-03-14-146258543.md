    <h2>
     <a id="Django__RBAC__0">
     </a>
     Django 自带的用户系统与 RBAC 机制
    </h2>
    <p>
     Django 自带的用户系统（
     <code>
      django.contrib.auth
     </code>
     ）提供了
     <strong>
      身份验证（Authentication）
     </strong>
     和
     <strong>
      权限管理（Authorization）
     </strong>
     ，能够快速实现
     <strong>
      用户管理、权限控制、管理员后台
     </strong>
     等功能，同时具备
     <strong>
      RBAC（基于角色的访问控制，Role-Based Access Control）
     </strong>
     的基本实现。本文将详细介绍 Django 用户系统的功能、管理员账号创建、RBAC 机制及其扩展方案。
    </p>
    <hr/>
    <h3>
     <a id="Django__6">
     </a>
     <strong>
      一、Django 自带用户系统的作用
     </strong>
    </h3>
    <p>
     Django 自带的用户系统主要用于
     <strong>
      用户身份管理、身份验证、权限与授权、用户管理后台
     </strong>
     以及
     <strong>
      扩展性
     </strong>
     ，使开发者能够快速实现用户相关功能，而无需从零编写。
    </p>
    <h4>
     <a id="_10">
     </a>
     <strong>
      （一）用户身份管理
     </strong>
    </h4>
    <p>
     Django 提供了默认的
     <code>
      User
     </code>
     模型，其中包含：
    </p>
    <ul>
     <li>
      <strong>
       基本字段
      </strong>
      ：
      <code>
       username
      </code>
      、
      <code>
       password
      </code>
      、
      <code>
       email
      </code>
      等
     </li>
     <li>
      <strong>
       权限字段
      </strong>
      ：
      <code>
       is_active
      </code>
      （是否激活）、
      <code>
       is_staff
      </code>
      （是否是后台管理员）、
      <code>
       is_superuser
      </code>
      （是否是超级用户）
     </li>
     <li>
      <strong>
       用户创建
      </strong>
      ：
      <ul>
       <li>
        <code>
         User.objects.create_user()
        </code>
        ：创建普通用户
       </li>
       <li>
        <code>
         User.objects.create_superuser()
        </code>
        ：创建超级管理员
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="Authentication_18">
     </a>
     <strong>
      （二）身份验证（Authentication）
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       密码存储
      </strong>
      ：Django 采用 PBKDF2、Argon2、BCrypt 等安全算法存储密码
     </li>
     <li>
      <strong>
       登录与退出
      </strong>
      ：
      <ul>
       <li>
        <code>
         authenticate()
        </code>
        ：验证用户名和密码
       </li>
       <li>
        <code>
         login()
        </code>
        ：记录用户 session
       </li>
       <li>
        <code>
         logout()
        </code>
        ：清除用户 session，注销登录
       </li>
      </ul>
     </li>
     <li>
      <strong>
       自动处理用户 session
      </strong>
      ：可通过
      <code>
       request.user
      </code>
      直接获取当前登录用户
     </li>
    </ul>
    <h4>
     <a id="Permissions__Authorization_26">
     </a>
     <strong>
      （三）权限与授权（Permissions &amp; Authorization）
     </strong>
    </h4>
    <p>
     Django 的权限管理体系包含
     <strong>
      用户权限、组权限、对象权限
     </strong>
     。
    </p>
    <ul>
     <li>
      <strong>
       用户权限
      </strong>
      ：通过
      <code>
       is_staff
      </code>
      和
      <code>
       is_superuser
      </code>
      控制
     </li>
     <li>
      <strong>
       组权限
      </strong>
      ：使用
      <code>
       Group
      </code>
      模型批量管理权限
     </li>
     <li>
      <strong>
       对象权限
      </strong>
      ：支持模型级权限（
      <code>
       add_xxx
      </code>
      、
      <code>
       change_xxx
      </code>
      、
      <code>
       delete_xxx
      </code>
      ）
     </li>
     <li>
      <strong>
       权限检查
      </strong>
      ：
      <ul>
       <li>
        <code>
         @login_required
        </code>
        限制访问
       </li>
       <li>
        <code>
         @permission_required("app_label.permission_name")
        </code>
        控制权限
       </li>
       <li>
        <code>
         request.user.has_perm("app_label.permission_name")
        </code>
        检查权限
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="_36">
     </a>
     <strong>
      （四）用户管理后台
     </strong>
    </h4>
    <p>
     Django 自带
     <code>
      admin
     </code>
     后台（
     <code>
      django.contrib.admin
     </code>
     ），可快速管理
     <strong>
      用户、权限、组
     </strong>
     ，无需额外开发界面。
    </p>
    <h4>
     <a id="_39">
     </a>
     <strong>
      （五）扩展性
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       自定义用户模型
      </strong>
      ：通过
      <code>
       AbstractUser
      </code>
      或
      <code>
       AbstractBaseUser
      </code>
      进行扩展
     </li>
     <li>
      <strong>
       扩展用户字段
      </strong>
      ：如手机号、头像等
     </li>
     <li>
      <strong>
       第三方认证
      </strong>
      ：可集成 OAuth（Google、GitHub 登录）或 JWT（DRF Simple JWT）
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_46">
     </a>
     <strong>
      二、管理员账号的创建与管理
     </strong>
    </h3>
    <h4>
     <a id="_48">
     </a>
     <strong>
      （一）如何创建管理员账号？
     </strong>
    </h4>
    <p>
     Django
     <strong>
      不会自动创建管理员账号
     </strong>
     ，需要手动执行以下命令：
    </p>
    <pre><code class="prism language-bash">python manage.py createsuperuser
</code></pre>
    <p>
     随后输入
     <strong>
      用户名、邮箱、密码
     </strong>
     创建管理员账户。
    </p>
    <h4>
     <a id="_55">
     </a>
     <strong>
      （二）如何使用管理员账号登录？
     </strong>
    </h4>
    <ol>
     <li>
      访问
      <strong>
       Django Admin 后台
      </strong>
      ：
      <pre><code>http://127.0.0.1:8000/admin/
</code></pre>
     </li>
     <li>
      使用
      <code>
       createsuperuser
      </code>
      创建的账号登录。
     </li>
    </ol>
    <h4>
     <a id="_62">
     </a>
     <strong>
      （三）如何查看或修改管理员账号？
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       在数据库中查找超级管理员
      </strong>
      ：
      <pre><code class="prism language-sql"><span class="token keyword">SELECT</span> username<span class="token punctuation">,</span> email <span class="token keyword">FROM</span> auth_user <span class="token keyword">WHERE</span> is_superuser<span class="token operator">=</span><span class="token number">1</span><span class="token punctuation">;</span>
</code></pre>
     </li>
     <li>
      <strong>
       修改管理员密码
      </strong>
      ：
      <pre><code class="prism language-bash">python manage.py changepassword admin
</code></pre>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="Django__RBAC_74">
     </a>
     <strong>
      三、Django 的 RBAC（基于角色的访问控制）机制
     </strong>
    </h3>
    <p>
     Django 自带
     <strong>
      RBAC
     </strong>
     的基础实现，主要通过
     <strong>
      用户（User）、权限（Permission）、组（Group）
     </strong>
     进行管理。
    </p>
    <h4>
     <a id="User_78">
     </a>
     <strong>
      （一）用户（User）
     </strong>
    </h4>
    <ul>
     <li>
      每个用户可以
      <strong>
       独立拥有权限
      </strong>
      或
      <strong>
       归属于某个组（Group）
      </strong>
      ，继承组权限。
     </li>
    </ul>
    <h4>
     <a id="Permission_81">
     </a>
     <strong>
      （二）权限（Permission）
     </strong>
    </h4>
    <p>
     Django 提供了
     <strong>
      基于模型的权限
     </strong>
     ，默认包括：
    </p>
    <ul>
     <li>
      <code>
       add_modelname
      </code>
      （添加权限）
     </li>
     <li>
      <code>
       change_modelname
      </code>
      （修改权限）
     </li>
     <li>
      <code>
       delete_modelname
      </code>
      （删除权限）
     </li>
     <li>
      <code>
       view_modelname
      </code>
      （查看权限）
     </li>
    </ul>
    <h5>
     <a id="_88">
     </a>
     <strong>
      权限检查
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       代码中检查权限
      </strong>
      ：
      <pre><code class="prism language-python"><span class="token keyword">if</span> request<span class="token punctuation">.</span>user<span class="token punctuation">.</span>has_perm<span class="token punctuation">(</span><span class="token string">'app_label.permission_name'</span><span class="token punctuation">)</span><span class="token punctuation">:</span>
    <span class="token comment"># 用户有权限</span>
</code></pre>
     </li>
     <li>
      <strong>
       使用装饰器
      </strong>
      ：
      <pre><code class="prism language-python"><span class="token keyword">from</span> django<span class="token punctuation">.</span>contrib<span class="token punctuation">.</span>auth<span class="token punctuation">.</span>decorators <span class="token keyword">import</span> permission_required

<span class="token decorator annotation punctuation">@permission_required</span><span class="token punctuation">(</span><span class="token string">'app_label.permission_name'</span><span class="token punctuation">,</span> raise_exception<span class="token operator">=</span><span class="token boolean">True</span><span class="token punctuation">)</span>
<span class="token keyword">def</span> <span class="token function">my_view</span><span class="token punctuation">(</span>request<span class="token punctuation">)</span><span class="token punctuation">:</span>
    <span class="token keyword">pass</span>
</code></pre>
     </li>
    </ol>
    <h4>
     <a id="Group_103">
     </a>
     <strong>
      （三）组（Group）
     </strong>
    </h4>
    <p>
     <code>
      Group
     </code>
     允许批量管理权限，方便将权限分配给多个用户。例如：
    </p>
    <pre><code class="prism language-python"><span class="token keyword">from</span> django<span class="token punctuation">.</span>contrib<span class="token punctuation">.</span>auth<span class="token punctuation">.</span>models <span class="token keyword">import</span> Group<span class="token punctuation">,</span> Permission

<span class="token comment"># 创建一个 "编辑" 组</span>

<span class="token comment"># 获取某个权限</span>

<span class="token comment"># 给 "编辑" 组添加权限</span>

    <hr/>
    <h3>
     <a id="Django_RBAC__123">
     </a>
     <strong>
      四、Django RBAC 机制的局限性及扩展
     </strong>
    </h3>
    <p>
     Django 默认的 RBAC
     <strong>
      较为基础
     </strong>
     ，存在以下局限：
    </p>
    <ul>
     <li>
      <strong>
       权限是基于模型的
      </strong>
      ，无法控制对象级权限（如用户只能编辑自己创建的文章）
     </li>
     <li>
      <strong>
       没有层级角色
      </strong>
      ，管理员无法管理某个组
     </li>
     <li>
      <strong>
       没有前端 UI
      </strong>
      ，只能通过 Django Admin 或代码管理
     </li>
    </ul>
    <h4>
     <a id="_Django_RBAC_130">
     </a>
     <strong>
      （一）扩展 Django RBAC
     </strong>
    </h4>
    <h5>
     <a id="1__131">
     </a>
     <strong>
      1. 自定义权限系统
     </strong>
    </h5>
    <ul>
     <li>
      继承
      <code>
       AbstractUser
      </code>
      或
      <code>
       AbstractBaseUser
      </code>
      自定义用户模型
     </li>
     <li>
      设计
      <code>
       UserRole
      </code>
      表，实现
      <strong>
       用户 → 角色 → 权限
      </strong>
      关系
     </li>
    </ul>
    <h5>
     <a id="2__djangoguardian__135">
     </a>
     <strong>
      2. 使用
      <code>
       django-guardian
      </code>
      实现对象级权限
     </strong>
    </h5>
    <ul>
     <li>
      <code>
       django-guardian
      </code>
      允许用户对特定对象（如某篇文章）赋权：
      <pre><code class="prism language-python"><span class="token keyword">from</span> guardian<span class="token punctuation">.</span>shortcuts <span class="token keyword">import</span> assign_perm

assign_perm<span class="token punctuation">(</span><span class="token string">'change_article'</span><span class="token punctuation">,</span> user<span class="token punctuation">,</span> article<span class="token punctuation">)</span>  <span class="token comment"># 允许 user 修改特定文章</span>
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3__djangorules__143">
     </a>
     <strong>
      3. 使用
      <code>
       django-rules
      </code>
      规则引擎
     </strong>
    </h5>
    <ul>
     <li>
      通过函数定义权限规则：
      <pre><code class="prism language-python"><span class="token keyword">import</span> rules

<span class="token decorator annotation punctuation">@rules<span class="token punctuation">.</span>predicate</span>
<span class="token keyword">def</span> <span class="token function">is_author</span><span class="token punctuation">(</span>user<span class="token punctuation">,</span> article<span class="token punctuation">)</span><span class="token punctuation">:</span>
    <span class="token keyword">return</span> article<span class="token punctuation">.</span>author <span class="token operator">==</span> user
</code></pre>
     </li>
    </ul>
