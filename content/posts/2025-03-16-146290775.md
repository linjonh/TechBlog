    <h5>
     🌟 核心需求
    </h5>
    <p>
    </p>
    <p>
     在 Android 13 商显设备开发中，需精简 Recovery 模式的菜单选项（如
     <code>
      Reboot to bootloader
     </code>
     /
     <code>
      Enter rescue
     </code>
     ），但直接修改
     <code>
      g_menu_actions
     </code>
     后在
     <strong>
      User 版本
     </strong>
     出现黑屏卡死问题，需综合方案解决。
    </p>
    <p>
    </p>
    <hr/>
    <p>
    </p>
    <h4>
     🔍 问题根源分析
    </h4>
    <p>
    </p>
    <table>
     <thead>
      <tr>
       <th>
        问题现象
       </th>
       <th>
        触发场景
       </th>
       <th>
        深层原因
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         Recovery 界面黑屏
        </strong>
       </td>
       <td>
        User 版本启动时
       </td>
       <td>
        系统内置的菜单过滤机制与开发者修改冲突
       </td>
      </tr>
      <tr>
       <td>

         选项移除失效
        </strong>
       </td>

        动态分区设备
       </td>

         recovery_main.cpp
        </code>
        中根据硬件特性二次过滤
       </td>

    <h4>
     🔧 终极解决方案（双保险策略）
    </h4>
    <p>
    </p>
    <h5>
     1️⃣
     <strong>
      基础菜单裁剪
     </strong>
    </h5>
    <p>
    </p>
    <p>
     修改
     <code>
      device.cpp
     </code>
     中的默认菜单配置：
    </p>
    <p>
    </p>
    <p>
     cpp
    </p>
    <p>
     Copy
    </p>
    <pre><code>// bootable/recovery/recovery_ui/device.cpp
static std::vector&lt;std::pair&lt;std::string, Device::BuiltinAction&gt;&gt; g_menu_actions{
  { "Reboot system now", Device::REBOOT },   // 仅保留核心选项
  { "Power off", Device::SHUTDOWN },
};
</code></pre>
    <p>
    </p>
    <h5>
     2️⃣
     <strong>
      强制禁用动态过滤
     </strong>
    </h5>
    <p>
    </p>
    <p>
     注释
     <code>
      recovery_main.cpp
     </code>
     中的条件过滤逻辑：
    </p>
    <p>
    </p>
    <p>
     cpp
    </p>
    <p>
     Copy
    </p>
    <pre><code>// bootable/recovery/recovery_main.cpp
// 注释以下代码块，阻止系统二次过滤
/*
if (!has_cache) {
  device-&gt;RemoveMenuItemForAction(Device::WIPE_CACHE);
}
if (!android::base::GetBoolProperty("ro.boot.dynamic_partitions", false)) {
  device-&gt;RemoveMenuItemForAction(Device::ENTER_FASTBOOT);
}
if (!is_ro_debuggable()) {
  device-&gt;RemoveMenuItemForAction(Device::ENTER_RESCUE);
}
*/
</code></pre>
    <p>
    </p>
    <hr/>
    <p>
    </p>
    <h4>
     🚀 进阶避坑指南
    </h4>
    <p>
    </p>
    <h5>
     1.
     <strong>
      版本兼容性处理
     </strong>
    </h5>
    <p>
    </p>
    <p>
     makefile
    </p>
    <p>
     Copy
    </p>
    <pre><code># BoardConfig.mk 增加编译控制
ifneq ($(TARGET_BUILD_VARIANT),user)
    RECOVERY_CUSTOM_MENU := true
endif
</code></pre>
    <p>
    </p>
    <h5>
     2.
     <strong>
      UI 渲染优化
     </strong>
    </h5>
    <p>
    </p>
    <p>
     java
    </p>
    <p>
     Copy
    </p>
    <pre><code>// 强制横屏显示（适用于商显设备）
ui-&gt;SetRotation(RecoveryUI::ROTATION_RIGHT);
</code></pre>
    <p>
    </p>
    <h5>
     3.
     <strong>
      安全加固
     </strong>
    </h5>
    <p>
    </p>
    <p>
     shell
    </p>
    <p>
     Copy
    </p>
    <pre><code># 禁止通过物理按键触发隐藏菜单
adb shell "echo 1 &gt; /sys/class/input/eventX/disable"
</code></pre>
    <p>
    </p>
