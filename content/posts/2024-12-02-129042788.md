    <h2>
     <a id="nano_0">
     </a>
     一、nano命令简介
    </h2>
    <p>
     nano是一个小型、免费、友好的编辑器，旨在取代非免费Pine包中的默认编辑器Pico。nano不仅复制了Pico的外观，还实现了Pico中一些缺失（或默认禁用）的功能，例如“搜索和替换”和“转到行号和列号”。nano是一个字符终端的文本编辑器，有点像DOS下的editor程序。它比vi/vim要简单得多，比较适合Linux初学者使用。某些Linux发行版的默认编辑器就是nano，比如Ubuntu系统默认安装了nano。
    </p>
    <h2>
     <a id="nano_2">
     </a>
     二、nano命令使用示例
    </h2>
    <h3>
     <a id="1_3">
     </a>
     1、命令安装
    </h3>
    <blockquote>
     <p>
      [root@s142 ~]# yum install -y nano
     </p>
    </blockquote>
    <h3>
     <a id="2_5">
     </a>
     2、查看命令版本
    </h3>
    <blockquote>
     <p>
      [root@s142 ~]# nano -V
      <br/>
      GNU nano version 2.3.1 (compiled 04:47:52, Jun 10 2014)
     </p>
    </blockquote>
    <h3>
     <a id="3_8">
     </a>
     3、获取命令帮助
    </h3>
    <blockquote>
     <p>
      [root@s142 ~]# nano --help
     </p>
    </blockquote>
    <h3>
     <a id="4_10">
     </a>
     4、编辑一个文件
    </h3>
    <blockquote>
     <p>
      [wuhs@s142 ~]$ nano hi.txt
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b9c7f95b5f43c779301cbe45701b8b75.png"/>
     </p>
    </blockquote>
    <h3>
     <a id="5_13">
     </a>
     5、不自动换行编辑
    </h3>
    <p>
     nano命令可以打开指定文件进行编辑，默认情况下它会自动断行，即在一行中输入过长的内容时自动拆分成几行，但用这种方式来处理某些文件可能会带来问题，比如Linux系统的/etc/fstab文件，所以非必要情况下建议不使用自动换行功能，使用-w开启编辑模式。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/a2100a36fe140e108c8f3e0596e74420.png"/>
    </p>
    <blockquote>
     <p>
      [wuhs@s142 ~]$ nano -w hi.txt
     </p>
    </blockquote>
    <h3>
     <a id="6_18">
     </a>
     6、搜索字符串
    </h3>
    <p>
     如果我们想从打开的文件中搜索字符串，输入Ctl+W开启搜索框，输入字符串后回车，光标自动定位到搜索到的第一个结果。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/b9f6a3f8acda3a8f1caad9f1bc321d0c.png"/>
    </p>
    <h3>
     <a id="71_21">
     </a>
     7、往编辑文件中插入1个文件
    </h3>
    <p>
     nano支持往编辑的文件中插入一个文件，使用Ctl+R，输入框中输入文件的路径及文件名，默认是当前路径。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/37363f00fecb5e57f0e6179b85f9ee71.png"/>
    </p>
