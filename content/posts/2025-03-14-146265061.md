    <p>
     <strong>
      1.gdb的安装
     </strong>
    </p>
    <p>
     如果你是root用户直接用指令 ：yum install gdb ；如果你是普通用户用指令：sudo yum install gdb；
    </p>
    <p>
     <strong>
      2.gdb调试前可以对你的makefile文件进行编写：
     </strong>
    </p>
    <p>
     下面展示为11.c文件编写的makefile文件：
    </p>
    <pre><code class="language-bash">
code-debug:11.c
        gcc -g 11.c -o code-debug 

.PHONY:clean

clean:
        rm -f code-debug
</code></pre>
    <p>
     <strong>
      3.开始调试：
     </strong>
    </p>
    <p>
     当你的makefile编写完成后，你执行了make指令后，生成了可执行程序code-debug；你输入指令：
     <strong>
      <span style="color:#fe2c24">
       gdb code-debug
      </span>
     </strong>
    </p>
    <p>
     <img alt="" height="393" src="https://i-blog.csdnimg.cn/direct/1a448478525441138c6be12cb7b603e8.png" width="860"/>
    </p>
    <p>
     出现这个界面说明你进入gdb成功！
    </p>
    <p>
     <strong>
      接下来是一些常见的指令：
     </strong>
    </p>
    <h4>
     启动与退出
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        gdb &lt;program&gt;
       </code>
      </strong>
      ：启动 Gdb 并加载指定的可执行文件。例如
      <code>
       gdb main
      </code>
      ，用于调试名为
      <code>
       main
      </code>
      的可执行程序。
     </li>
     <li>
      <strong>
       <code>
        quit
       </code>
       或
       <code>
        q
       </code>
      </strong>
      ：退出 Gdb 调试环境。
     </li>
    </ul>
    <h4>
     程序执行控制
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        run
       </code>
       或
       <code>
        r
       </code>
      </strong>
      ：启动程序的执行，可带参数，如
      <code>
       run arg1 arg2
      </code>
      ，表示使用参数
      <code>
       arg1
      </code>
      和
      <code>
       arg2
      </code>
      运行程序。
     </li>
     <li>
      <strong>
       <code>
        continue
       </code>
       或
       <code>
        c
       </code>
      </strong>
      ：从当前断点处继续执行程序，直到遇到下一个断点或程序结束。
     </li>
     <li>
      <strong>
       <code>
        next
       </code>
       或
       <code>
        n
       </code>
      </strong>
      ：单步执行程序，遇到函数调用时，不会进入函数内部，直接执行函数调用语句的下一行。
     </li>
     <li>
      <strong>
       <code>
        step
       </code>
       或
       <code>
        s
       </code>
      </strong>
      ：单步执行程序，若当前行是函数调用，则会进入函数内部继续单步执行。
      <img alt="" height="135" src="https://i-blog.csdnimg.cn/direct/f24fd5f7dc934c16b3c8d5a2d0011d89.png" width="236"/>
     </li>
     <li>
      <strong>
       <code>
        finish
       </code>
      </strong>
      ：继续执行当前函数，直到函数返回，并显示返回值。
     </li>
     <li>
      <strong>
       <code>
        until
       </code>
      </strong>
      ：不带参数时，执行直到当前函数返回；带参数（如行号或地址）时，程序继续执行直到到达指定位置。
      <img alt="" height="521" src="https://i-blog.csdnimg.cn/direct/d881d8fe762b4cb7b2b76f2d824a9f13.png" width="695"/>
     </li>
    </ul>
    <h4>
     断点操作
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        break &lt;location&gt;
       </code>
       或
       <code>
        b &lt;location&gt;
       </code>
      </strong>
      ：在指定位置设置断点，位置可以是行号（如
      <code>
       b 10
      </code>
      在第 10 行设置断点）、函数名（如
      <code>
       b main
      </code>
      在
      <code>
       main
      </code>
      函数入口设置断点）或文件名 + 行号（如
      <code>
       b main.c:20
      </code>
      在
      <code>
       main.c
      </code>
      的第 20 行设置断点）。
     </li>
     <li>
      <strong>
       <code>
        tbreak &lt;location&gt;
       </code>
      </strong>
      ：设置临时断点，该断点在被命中一次后自动删除。
     </li>
     <li>
      <strong>
       <code>
        info breakpoints
       </code>
       或
       <code>
        info b
       </code>
      </strong>
      ：显示当前设置的所有断点的信息，包括断点编号、位置、是否启用等。
     </li>
     <li>
      <strong>
       <code>
        delete &lt;breakpoint_number&gt;
       </code>
      </strong>
      ：删除指定编号的断点，如
      <code>
       delete 1
      </code>
      删除编号为 1 的断点。
     </li>
     <li>
      <strong>
       <code>
        disable &lt;breakpoint_number&gt;
       </code>
      </strong>
      ：禁用指定编号的断点，使其暂时不起作用，但断点信息仍保留。
      <img alt="" height="511" src="https://i-blog.csdnimg.cn/direct/5b873293186546fbbd557db3a60043aa.png" width="777"/>
     </li>
    </ul>
    <ul>
     <li>
      <strong>
       <code>
        enable &lt;breakpoint_number&gt;
       </code>
      </strong>
      ：启用之前被禁用的指定编号的断点。
     </li>
    </ul>
    <h4>
     查看与打印
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        list
       </code>
       或
       <code>
        l
       </code>
      </strong>
      ：显示当前执行位置附近的源代码，默认每次显示 10 行。也可指定行号（如
      <code>
       l 50
      </code>
      从第 50 行开始显示）或函数名（如
      <code>
       l main
      </code>
      显示
      <code>
       main
      </code>
      函数的代码）来显示特定内容。
      <img alt="" height="268" src="https://i-blog.csdnimg.cn/direct/f8278d179a5c40c7a6e7b1fbccbf4ec1.png" width="403"/>
     </li>
    </ul>
    <ul>
     <li>
      <strong>
       <code>
        print &lt;expression&gt;
       </code>
       或
       <code>
        p &lt;expression&gt;
       </code>
      </strong>
      ：打印指定表达式的值，如
      <code>
       p x
      </code>
      打印变量
      <code>
       x
      </code>
      的值，
      <code>
       p a + b
      </code>
      打印
      <code>
       a + b
      </code>
      的计算结果。
     </li>
     <li>
      <strong>
       <code>
        display &lt;expression&gt;
       </code>
      </strong>
      ：每次程序停止时自动显示指定表达式的值，便于监控变量变化，如
      <code>
       display i
      </code>
      每次程序暂停时都会显示变量
      <code>
       i
      </code>
      的值。
      <img alt="" height="249" src="https://i-blog.csdnimg.cn/direct/bb8801bfc42a486eb115a1d1b1fb299c.png" width="403"/>
     </li>
     <li>
      <strong>
       <code>
        undisplay &lt;display_number&gt;
       </code>
      </strong>
      ：删除指定编号的自动显示设置，编号可通过
      <code>
       info display
      </code>
      查看。
     </li>
     <li>
      <strong>
       <code>
        backtrace
       </code>
       或
       <code>
        bt
       </code>
      </strong>
      ：显当前的函数调用栈信息，帮了解程序的执行路径和各函数间的调用关系。
      <img alt="" height="227" src="https://i-blog.csdnimg.cn/direct/462bfd2298e34a74a1c408ec670d6a72.png" width="536"/>
     </li>
     <li>
      <strong>
       <code>
        info locals
       </code>
      </strong>
      ：查看当前作用域内的局部变量及其值。
     </li>
     <li>
      <strong>
       <code>
        whatis &lt;variable&gt;
       </code>
      </strong>
      ：显示变量的类型。
     </li>
     <li>
      <strong>
       <code>
        ptype &lt;variable&gt;
       </code>
      </strong>
      ：显示变量详细的类型信息，对于结构体、类等复杂类型会展示其成员信息 。
     </li>
    </ul>
    <h4>
     观察点操作
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        watch &lt;expression&gt;
       </code>
      </strong>
      ：设置观察点，当指定的表达式的值发生变化时，程序暂停执行。
     </li>
     <li>
      <strong>
       <code>
        rwatch &lt;expression&gt;
       </code>
      </strong>
      ：设置只读观察点，当程序读取指定表达式的值时，程序暂停执行。
     </li>
     <li>
      <strong>
       <code>
        awatch &lt;expression&gt;
       </code>
      </strong>
      ：设置读写观察点，当程序读取或写入指定表达式的值时，程序暂停执行。
     </li>
    </ul>
    <h4>
     其他常用指令
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        set &lt;variable&gt;=&lt;value&gt;
       </code>
      </strong>
      ：设置变量的值，如
      <code>
       set n = 10
      </code>
      将变量
      <code>
       n
      </code>
      的值设置为 10。
      <img alt="" height="232" src="https://i-blog.csdnimg.cn/direct/f11fa316a5594d4d8ad819c476706c34.png" width="428"/>
     </li>
    </ul>
    <ul>
     <li>
      <strong>
       <code>
        return
       </code>
      </strong>
      ：强制从当前函数返回，可带返回值，如
      <code>
       return 5
      </code>
      使当前函数直接返回 5。
     </li>
     <li>
      <strong>
       <code>
        shell &lt;command&gt;
       </code>
      </strong>
      ：在不离开 Gdb 的情况下，执行 Linux shell 命令，如
      <code>
       shell ls
      </code>
      可列出当前目录下的文件和目录。
     </li>
     <li>
      <strong>
       <code>
        help &lt;command&gt;
       </code>
      </strong>
      ：查看指定 Gdb 命令的帮助信息，若不指定具体命令，会显示 Gdb 的总体帮助信息和命令分类。
     </li>
    </ul>
    <p>
     <span style="color:#511b78">
      <strong>
       Linux系统下调试的时候其实是有一种历史机制的，所以它再调试过程中可以进行直接回车接着进行！
      </strong>
     </span>
    </p>
