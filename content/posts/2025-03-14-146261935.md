    <p>
     <span style="color:#1f2329">
      Spring Web MVC 是基于 Servlet API 构建的原始 Web 框架，从⼀开始就包含在 Spring 框架中。它的正式名称“Spring Web MVC”来⾃其源模块的名称(Spring-webmvc)，但它通常被称为"Spring
     </span>
    </p>
    <p>
     <span style="color:#1f2329">
      MVC"
     </span>
    </p>
    <div>
    </div>
    <h2>
     1.MVC
    </h2>
    <p>
     MVC是Model View Controller的缩写，
     <span style="color:#1f2329">
      它是软件⼯程中的⼀种软件架构设计模式，它把软件系统分
     </span>
    </p>
    <p>
     <span style="color:#1f2329">
      为模型、视图和控制器三个基本部分
     </span>
    </p>
    <p>
    </p>
    <div>
     <img alt="" height="685" src="https://i-blog.csdnimg.cn/direct/835df3ae428a4856accc894295c8fc13.png" width="1109">
     </img>
    </div>
    <p>
     其中：
    </p>
    <p>
     View：用于界面显示，进行人机交互，展示数据
    </p>
    <p>
     Controller：分配资源的分用器，用于处理判断视图传来的请求应该用哪个模型解决，以及处理后的响应应该发给哪个视图
    </p>
    <p>
     Model：
     <span style="color:#1f2329">
      是应用程序的主体部分，⽤来处理程序中数据逻辑的部分
     </span>
    </p>
    <p>
     用生活中的例子说：
    </p>
    <p>
     View：服务员，用于与客户顾客交互的角色
    </p>
    <p>
     Controller：前台，处理服务员从顾客那里拿来的菜单应该给哪个厨师
    </p>
    <p>
     Model：
     <span style="color:#1f2329">
      厨师，真正做菜的角色
     </span>
    </p>
    <h2>
     2.Spring MVC
    </h2>
    <p>
     那么Spring实现MVC后有什么改变呢？下图可能展示的更准确
    </p>
    <p>
     <img alt="" height="690" src="https://i-blog.csdnimg.cn/direct/253e13d08e8a47a98a26bdc6b483b754.png" width="1045"/>
    </p>
    <p>
     <span style="color:#1f2329">
      既然是 Web 框架, 那么当⽤⼾在浏览器中输⼊了 url 之后，我们的 Spring MVC 项⽬就可以感知到⽤户的请求, 并给予响应。
     </span>
    </p>
    <p>
    </p>
    <p>
     <span style="color:#1f2329">
      主要分以下三个⽅⾯:
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#1456f0">
       1.
      </span>
      <span style="color:#1f2329">
       建⽴连接：将用户（浏览器）和 Java 程序连接起来，也就是访问⼀个地址能够调⽤到我们的Spring 程序。
      </span>
     </p>
     <p>
      <span style="color:#1456f0">
       2.
      </span>
      <span style="color:#1f2329">
       请求: 用户请求的时候会带⼀些参数，在程序中要想办法获取到参数, 所以请求这块主要是 获取参数的功能.
      </span>
     </p>
     <p>
      <span style="color:#1456f0">
       3.
      </span>
      <span style="color:#1f2329">
       响应: 执行了业务逻辑之后，要把程序执行的结果返回给用户, 也就是响应
      </span>
     </p>
    </blockquote>
    <h3>
     2.1建立连接
    </h3>
    <p>
     <span style="color:#1f2329">
      在 Spring MVC 中使⽤
     </span>
     <span style="color:#1f2329">
      <strong>
       @RequestMapping
      </strong>
     </span>
     <span style="color:#1f2329">
      来实现 URL 路由映射 ，也就是浏览器连接程序的作⽤
     </span>
    </p>
    <p>
     创建一个HelloController程序，
     <span style="color:#1f2329">
      实现用户通过浏览器和程序的交互
     </span>
    </p>
    <pre><code>import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
@RestController
public class HelloController {
    @RequestMapping("/Hello")
    public String hello() {
        System.out.println("Hello Spring MVC");
        return "Hello Spring MVC!!";
    }

}</code></pre>
    <p>
     启动程序后，接下来在浏览器
     <span style="color:#1f2329">
      访问:
     </span>
     <span style="color:#336df4">
      http://127.0.0.1:8080/Hello
     </span>
     <span style="color:#1f2329">
      , 就可以看到程序返回的数据了
     </span>
    </p>
    <p>
     <img alt="" height="104" src="https://i-blog.csdnimg.cn/direct/8ef848e1258a40cba477556deeba4ee9.png" width="484"/>
    </p>
    <h4>
     2.1.1
     <span style="color:#1f2329">
      <strong>
       @RequestMapping作用
      </strong>
     </span>
    </h4>
    <p>
     <span style="color:#1f2329">
      <strong>
       @RequestMapping
      </strong>
      用与注册接⼝的路由映射的
     </span>
    </p>
    <p>
     <span style="color:#1f2329">
      表示当路径/Hello接收到请求时，就会调用hello（）方法
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#1f2329">
       路由映射：当用户输入url访问服务时，将用户请求定位再某个类的某个方法的过程就是路由映射
      </span>
     </p>
    </blockquote>
    <p>
     <span style="color:#646a73">
      既然
     </span>
     <span style="color:#646a73">
      @RequestMapping
     </span>
     <span style="color:#646a73">
      已经可以达到我们的⽬的了, 我们为什么还要加
     </span>
     <span style="color:#646a73">
      @RestController呢?
     </span>
    </p>
    <p>
     <span style="color:#646a73">
      我们把
     </span>
     <span style="color:#646a73">
      @RestController
     </span>
     <span style="color:#646a73">
      去掉, 再来访问⼀次
     </span>
    </p>
    <p>
     <img alt="" height="250" src="https://i-blog.csdnimg.cn/direct/7b487d2cee474d79b4192fbad8d45d83.png" width="917"/>
    </p>
    <p>
     程序报了404错误，显示找不到界面
    </p>
    <p>
     <span style="color:#646a73">
      Spring会对所有的类进⾏扫描, 如果类加了注解@RestController, Spring才会去看这个类⾥⾯的⽅法有没有加 @RequestMapping
     </span>
     <span style="color:#646a73">
      这个注解
     </span>
    </p>
    <h4>
     <span style="color:#646a73">
      2.1.2@RequestMapping使用
     </span>
    </h4>
    <p>
     <span style="color:#1f2329">
      @RequestMapping 既可修饰类，也可以修饰⽅法 ，当修饰类和⽅法时，访问的地址是类路径 + ⽅法路径.
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#646a73">
       @RequestMapping标识⼀个类：设置映射请求的请求路径的
       <strong>
        初始
       </strong>
       信息
      </span>
     </p>
     <p>
      <span style="color:#646a73">
       @RequestMapping标识⼀个⽅法：设置映射请求请求路径的
       <strong>
        具体
       </strong>
       信息
      </span>
     </p>
    </blockquote>
    <pre><code>import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
@RequestMapping("/r1")
@RestController
public class HelloController {
    @RequestMapping("/Hello")
    public String hello() {
        System.out.println("Hello Spring MVC");
        return "Hello Spring MVC!!";
    }
}
</code></pre>
    <p>
     访问地址：
     <a href="http://127.0.0.1:8080/r1/Hello" rel="nofollow" title="127.0.0.1:8080/r1/Hello">
      127.0.0.1:8080/r1/Hello
     </a>
    </p>
    <p>
     <img alt="" height="99" src="https://i-blog.csdnimg.cn/direct/ed472bb2f9fb4c0cb21ce3a50e00e150.png" width="530"/>
    </p>
    <div>
     <span style="color:#1f2329">
      注意:
     </span>
    </div>
    <div>
     <span style="color:#646a73">
      @RequestMapping 的URL 路径最前⾯加不加
     </span>
     <span style="color:#646a73">
      /
     </span>
     <span style="color:#646a73">
      (斜杠)都可以, Spring程序启动时, 会进⾏判断, 如果前⾯没有加 /
     </span>
     <span style="color:#646a73">
      , Spring会拼接上⼀个
     </span>
     <span style="color:#646a73">
      /
     </span>
    </div>
    <p>
    </p>
    <h4>
     2.1.3@RequestMapping是Get请求还是Post请求
    </h4>
    <p>
     我们通过创造from表单构建请求
    </p>
    <pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="en"&gt;
&lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;title&gt;Title&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;

    &lt;form action="/r1/Hello" method="post"&gt;
        &lt;input type="submit" value="提交"&gt;
    &lt;/form&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre>
    <p>
     放在static包中
    </p>
    <p>
     <img alt="" height="152" src="https://i-blog.csdnimg.cn/direct/fd2c492b9fdc4a3e9d5e9c7db3ea08f6.png" width="469"/>
    </p>
    <p>
     <span style="color:#646a73">
      访问⽅式为：
     </span>
     <span style="color:#336df4">
      http://127.0.0.1:8080/test.htm
     </span>
    </p>
    <p>
     <img alt="" height="149" src="https://i-blog.csdnimg.cn/direct/1584e44c822b4b67ac868c46244c144a.png" width="512"/>
    </p>
    <p>
     如果有多层目录，访问链接从static目录开始
    </p>
    <p>
     <img alt="" height="142" src="https://i-blog.csdnimg.cn/direct/820e0dbf688742beabe3ec0b69015632.png" width="426"/>
    </p>
    <p>
     <span style="color:#646a73">
      如上图, 访问链接为:
     </span>
     <span style="color:#336df4">
      http://127.0.0.1:8080/html/test.html
     </span>
    </p>
    <p>
     <img alt="" height="105" src="https://i-blog.csdnimg.cn/direct/f80fbae68afd4d058f3db3115716b3dc.png" width="549"/>
    </p>
    <p>
     点击提交后跳转到 ：
     <span style="color:#336df4">
      http://127.0.0.1:8080/r1/Hello
     </span>
    </p>
    <p>
     可以通过@RequestMapping
     <span style="color:#1f2329">
      指定GET/POST⽅法类型
     </span>
    </p>
    <pre><code class="hljs">import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.RestController;
@RequestMapping("/r1")
@RestController
public class HelloController {
    @RequestMapping(value = "/Hello",method = RequestMethod.POST)
    public String hello() {
        System.out.println("Hello Spring MVC");
        return "Hello Spring MVC!!";
    }
}
</code></pre>
