    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         摘 要
        </span>
       </strong>
       <span style="font-size: 9pt;">
        自
        <span lang="EN-US">
         80
        </span>
        年代后期以来，相继出现了多种面向对象开发方法，文章主要介绍了其中六种典型的面向对象开发方法，并对它们进行了比较。
        <span lang="EN-US">
         UML
        </span>
        的出现意味着开发方法统一化、标准化的新发展，标志着软件自动化的进程又迈进了一步。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         关键字
        </span>
       </strong>
       <span style="font-size: 9pt;">
        面向对象开发方法 典型 比较
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         一、引言
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        随着计算机科学的发展和应用领域的不断扩大，对计算机技术的要求越来越高。特别是当计算机硬件有了飞速发展之后，各种应用领域对软件提出了更高的要求。结构化程序设计语言和结构化分析与设计已无法满足用户需求的变化。发展软件质量，缩短软件开发周期，发展软件可靠性、可扩充性和可重用性迫使软件界人士不断研究新方法、新技术，探索新途径。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        面向对象的方法是一种分析方法、设计方法和思维方法。面向对象方法学的出发点和所追求的基本目标是使人们分析、设计与实现一个系统的方法尽可能接近人们认识一个系统的方法。也就是使描述问题的问题空间和解决问题的方法空间在结构上尽可能一致。其基本思想是：对问题空间进行自然分割，以更接近人类思维的方式建立问题域模型，以便对客观实体进行结构模拟和行为模拟，从而使设计出的软件尽可能直接地描述现实世界，构造出模块化的、可重用的、维护性好的软件，同时限制软件的复杂性和降低开发维护费用。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        面向对象程序设计方法起源于六十年代末期的语言
        <span lang="EN-US">
         SIMULA’67
        </span>
        ，到了七十年代末期，软件行业正受到软件危机的困扰，结构化的开发方法不能够很好地解决软件危机。面向对象语言
        <span lang="EN-US">
         Smalltalk
        </span>
        的出现，进一步发展和完善了面向对象的程序设计语言，从此面向对象也和开发方法开始结合，出现了面向对象的开发方法。自
        <span lang="EN-US">
         80
        </span>
        年代后期以来，相继出现了多种面向对象开发方法，现将其中几种典型的方法作一个简介和比较。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         二、六种典型的面向对象开发方法
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         1
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          OMT/Rumbaugh
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OMT[1]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Object Modeling Technique
        </span>
        ）方法最早是由
        <span lang="EN-US">
         Loomis
        </span>
        ，
        <span lang="EN-US">
         Shan
        </span>
        和
        <span lang="EN-US">
         Rumbaugh
        </span>
        在
        <span lang="EN-US">
         1987
        </span>
        年提出的，曾扩展应用于关系数据库设计。
        <span lang="EN-US">
         Jim Rumbaugh
        </span>
        在
        <span lang="EN-US">
         1991
        </span>
        年正式把
        <span lang="EN-US">
         OMT
        </span>
        应用于面向对象的分析和设计。这个方法是在实体关系模型上扩展了类、继承和行为而得到的。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OMT
       </span>
       <span style="font-size: 9pt;">
        覆盖了分析、设计和实现三个阶段，它包括一组相互关联的概念：类（
        <span lang="EN-US">
         class
        </span>
        ）、对象（
        <span lang="EN-US">
         object
        </span>
        ）、一般化（
        <span lang="EN-US">
         generalization
        </span>
        ）、继承（
        <span lang="EN-US">
         inheritance
        </span>
        ）、链（
        <span lang="EN-US">
         link
        </span>
        ）、链属性（
        <span lang="EN-US">
         link attribute
        </span>
        ）、聚合（
        <span lang="EN-US">
         aggregation
        </span>
        ）、操作（
        <span lang="EN-US">
         operation
        </span>
        ）、事件（
        <span lang="EN-US">
         event
        </span>
        ）、场景（
        <span lang="EN-US">
         scene
        </span>
        ）、属性（
        <span lang="EN-US">
         attribute
        </span>
        ）、子系统（
        <span lang="EN-US">
         subsystem
        </span>
        ）、模块（
        <span lang="EN-US">
         module
        </span>
        ）等。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OMT
       </span>
       <span style="font-size: 9pt;">
        方法包含分析、系统设计、对象设计和实现四个步骤，它定义了三种模型，这些模型贯穿于每个步骤，在每个步骤中被不断地精化和扩充。这三种模型是：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        对象模型：用类和关系来刻画系统的静态结构；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        动态模型：用事件和对象状态来刻画系统的动态特性；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        功能模型：按照对象的操作来描述如何从输入给出输出结果。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        分析的目的是建立可理解的现实世界模型。系统设计确定高层次的开发策略。对象设计的目的是确定对象的细节，包括定义对象的界面、算法和操作。实现对象则在良好的面向对象编程风格的编码原则指导下进行。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         2
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          OOD/Booch
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOD[3]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Object Oriented Design
        </span>
        ）方法是
        <span lang="EN-US">
         Grady Booch
        </span>
        从
        <span lang="EN-US">
         1983
        </span>
        年开始研究，
        <span lang="EN-US">
         1991
        </span>
        年后走向成熟的一种方法。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOD
       </span>
       <span style="font-size: 9pt;">
        主要包括下述概念：类（
        <span lang="EN-US">
         class
        </span>
        ）、对象（
        <span lang="EN-US">
         object
        </span>
        ）、使用（
        <span lang="EN-US">
         uses
        </span>
        ）、实例化（
        <span lang="EN-US">
         instantiates
        </span>
        ）、继承（
        <span lang="EN-US">
         inherits
        </span>
        ）、元类（
        <span lang="EN-US">
         meta class
        </span>
        ）、类范畴（
        <span lang="EN-US">
         class category
        </span>
        ）、消息（
        <span lang="EN-US">
         message
        </span>
        ）、域（
        <span lang="EN-US">
         field
        </span>
        ）、操作（
        <span lang="EN-US">
         operation
        </span>
        ）、机制（
        <span lang="EN-US">
         mechanism
        </span>
        ）、模块（
        <span lang="EN-US">
         module
        </span>
        ）、子系统（
        <span lang="EN-US">
         subsystem
        </span>
        ）、过程（
        <span lang="EN-US">
         process
        </span>
        ）等。其中，使用及实例化是类间的静态关系，而动态对象之间仅有消息传递的连接。元类是类的类。类范畴是一组类，它们在一定抽象意义上是类同的。物理的一组类用模块来表达。机制是完成一个需求任务的一组类构成的结构。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        Booch
       </span>
       <span style="font-size: 9pt;">
        方法在面向对象的设计中主要强调多次重复和开发者的创造性。方法本身是一组启发性的过程式建议。
        <span lang="EN-US">
         OOD
        </span>
        的一般过程如下：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         1
        </span>
        ）在一定抽象层次上标识类与对象；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         2
        </span>
        ）标识类与对象的语义；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         3
        </span>
        ）标识类与对象之间的关系（如继承、实例化、使用等）；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         4
        </span>
        ）实现类与对象。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         3
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          RDD/Wirfs-Brock
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        RDD[2]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Responsibility-Driven Design
        </span>
        ）方法是
        <span lang="EN-US">
         Wirfs-Brock
        </span>
        在
        <span lang="EN-US">
         1990
        </span>
        年提出的。这是一个按照类、责任以及合作关系对应用进行建模的方法。首先定义系统的类与对象，然后确定系统的责任并划分给类，最后确定对象类之间的合作来完成类的责任。这些设计将进一步按照类层次、子系统和协议来完善。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        RDD
       </span>
       <span style="font-size: 9pt;">
        方法主要包含以下概念：类（
        <span lang="EN-US">
         class
        </span>
        ）、继承（
        <span lang="EN-US">
         inheritance
        </span>
        ）、责任（
        <span lang="EN-US">
         responsibility
        </span>
        ）、合作（
        <span lang="EN-US">
         collaboration
        </span>
        ）、合同（
        <span lang="EN-US">
         contract
        </span>
        ）、子系统（
        <span lang="EN-US">
         subsystem
        </span>
        ）。对每个类都有不同的责任或角色以及动作。合作是为完成责任而需要与之通信的对象集合。责任进一步精化并被分组为合同。合同又进一步按操作精化为协议。子系统是为简化设计而引入的，是一组类和低级子系统，也包含由子系统中的类及子系统支持的合同。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        RDD
       </span>
       <span style="font-size: 9pt;">
        分为探索阶段和精化阶段：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 30pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         1
        </span>
        ）探索阶段：确定类、每个类的责任以及类间的合作；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 30pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         2
        </span>
        ）精化阶段：精化类继承层次、确定子系统、确定协议。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        RDD
       </span>
       <span style="font-size: 9pt;">
        按照类层次图、合作图、类规范、子系统规范、合同规范等设计规范来完成实现。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         4
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          OOAD/Coad-Yourdon
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOAD[3]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Object-Oriented Analysis and Design
        </span>
        ）方法是由
        <span lang="EN-US">
         Peter Coad
        </span>
        和
        <span lang="EN-US">
         Edward Yourdon
        </span>
        在
        <span lang="EN-US">
         1991
        </span>
        年提出的。这是一种逐步进阶的面向对象建模方法。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        在
        <span lang="EN-US">
         OOA
        </span>
        中，分析模型用来描述系统的功能，主要包括以下概念：类（
        <span lang="EN-US">
         class
        </span>
        ）、对象（
        <span lang="EN-US">
         object
        </span>
        ）、属性（
        <span lang="EN-US">
         attribute
        </span>
        ）、服务（
        <span lang="EN-US">
         service
        </span>
        ）、消息（
        <span lang="EN-US">
         message
        </span>
        ）、主题（
        <span lang="EN-US">
         subject
        </span>
        ）、一般
        <span lang="EN-US">
         /
        </span>
        特殊结构（
        <span lang="EN-US">
         Gen-Spec-Structure
        </span>
        ）、全局
        <span lang="EN-US">
         /
        </span>
        部份结构（
        <span lang="EN-US">
         Whole-Part-Structure
        </span>
        ）、实例连接（
        <span lang="EN-US">
         instance connection
        </span>
        ）和消息连接（
        <span lang="EN-US">
         message connection
        </span>
        ）等。其中，主题是指一组特定的类与对象。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOA
       </span>
       <span style="font-size: 9pt;">
        使用了基本的结构化原则，并把它们同面向对象的观点结合起来。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOA
       </span>
       <span style="font-size: 9pt;">
        完成系统分析，包括以下五个步骤：确定类与对象、标识结构、定义主题、定义属性和定义服务。
        <span lang="EN-US">
         OOA
        </span>
        本质上是一种面向对象的方法，适用于小型系统的开发。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOD
       </span>
       <span style="font-size: 9pt;">
        负责系统设计，包括以下四个步骤：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         1
        </span>
        ）设计问题域（细化分析结果）；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         2
        </span>
        ）设计人机交互部分（设计用户界面）；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         3
        </span>
        ）设计任务管理部分（确定系统资源的分配）；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        （
        <span lang="EN-US">
         4
        </span>
        ）设计数据管理部分（确定持久对象的存储）。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         5
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          OOSE/Jacobson
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOSE[3]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Object-Oriented Software Engineering
        </span>
        ）是
        <span lang="EN-US">
         Ivar Jacobson
        </span>
        在
        <span lang="EN-US">
         1992
        </span>
        年提出的一种使用事例驱动的面向对象开发方法。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOSE
       </span>
       <span style="font-size: 9pt;">
        主要包括下列概念：类（
        <span lang="EN-US">
         class
        </span>
        ）、对象（
        <span lang="EN-US">
         object
        </span>
        ）、继承（
        <span lang="EN-US">
         inherits
        </span>
        ）、相识（
        <span lang="EN-US">
         acquaintance
        </span>
        ）、通信（
        <span lang="EN-US">
         communication
        </span>
        ）、激励（
        <span lang="EN-US">
         stimuli
        </span>
        ）、操作（
        <span lang="EN-US">
         operation
        </span>
        ）、属性（
        <span lang="EN-US">
         attribute
        </span>
        ）、参与者（
        <span lang="EN-US">
         actor
        </span>
        ）、使用事例（
        <span lang="EN-US">
         use case
        </span>
        ）、子系统（
        <span lang="EN-US">
         subsystem
        </span>
        ）、服务包（
        <span lang="EN-US">
         service package
        </span>
        ）、块（
        <span lang="EN-US">
         block
        </span>
        ）、对象模块（
        <span lang="EN-US">
         object module
        </span>
        ）。相识表示静态的关联关系，包括聚合关系。激励是通信传送的消息。参与者是与系统交互的事物，它表示所有与系统有信息交换的系统之外的事务，因此不关心它的细节。参与者与用户不同，参与者是用户所充当的角色。参与者的一个实例对系统做一组不同的操作。当用户使用系统时，会执行一个行为相关的事物系列，这个系列是在与系统的会话中完成的，这个特殊的系列称为使用事例，每个使用事例都是使用系统的一条途径。使用事例的一个执行过程可以看作是使用事例的实例。当用户发出一个激励之后，使用事例的实例开始执行，并按照使用事例开始事物。事务包括许多动作，事务在收到用户结束激励后被终止。在这个意义上，使用事例可以被看作是对象类，而使用事例的实例可以被看作是对象。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        OOSE
       </span>
       <span style="font-size: 9pt;">
        开发过程中有以下五种模型，这些模型是自然过渡和紧密耦合的：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        需求模型包括由领域对象模型和界面描述支持的参与者和使用事例。对象模型是系统的概念化的、容易理解的描述。界面描述刻画了系统界面的细节。需求模型从用户的观点上完整地刻画了系统的功能需求，因此按这个模型与最终用户交流比较容易。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        分析模型是在需求模型的基础上建立的。主要目的是要建立在系统生命期中可维护、有逻辑性、健壮的结构。模型中有三种对象。界面对象刻画系统界面。实体对象刻画系统要长期管理的信息和信息上的行为。实体对象生存在一个特别的使用事例中。第三种是按特定的使用事例作面向事务的建模的对象。这三种对象使得需求的改变总是局限于其中一种。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        设计模型进一步精化分析模型并考虑了当前的实现环境。块描述了实现的意图。分析模型通常要根据实现作相应的变化。但分析模型中基本结构要尽可能保留。在设计模型中，块进一步用使用事例模型来阐述界面和块间的通信。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        实现模型主要包括实现块的代码。
        <span lang="EN-US">
         OOSE
        </span>
        并不要求用面向对象语言来完成实现。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        测试模型包括不同程度的保证。这种保证从低层的单元测试延伸到高层的系统测试。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         6
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、
         <span lang="EN-US">
          VMT/IBM
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        VMT[1]
       </span>
       <span style="font-size: 9pt;">
        （
        <span lang="EN-US">
         Visual Modeling Technique
        </span>
        ）方法是
        <span lang="EN-US">
         IBM
        </span>
        公司于
        <span lang="EN-US">
         1996
        </span>
        年公布的。
        <span lang="EN-US">
         VMT
        </span>
        方法结合了
        <span lang="EN-US">
         OMT
        </span>
        、
        <span lang="EN-US">
         OOSE
        </span>
        、
        <span lang="EN-US">
         RDD
        </span>
        等方法的优点，并且结合了可视化编程和原型技术。
        <span lang="EN-US">
         VMT
        </span>
        方法选择
        <span lang="EN-US">
         OMT
        </span>
        方法作为整个方法的框架，并且在表示上也采用了
        <span lang="EN-US">
         OMT
        </span>
        方法的表示。
        <span lang="EN-US">
         VMT
        </span>
        方法用
        <span lang="EN-US">
         RDD
        </span>
        方法中的
        <span lang="EN-US">
         CRC
        </span>
        （
        <span lang="EN-US">
         Class-Responsibility-Collaboration
        </span>
        ）卡片来定义各个对象的责任（操作）以及对象间的合作（关系）。此外，
        <span lang="EN-US">
         VMT
        </span>
        方法引入了
        <span lang="EN-US">
         OOSE
        </span>
        方法中的使用事例概念，用以描述用户与系统之间的相互作用，确定系统为用户提供的服务，从而得到准确的需求模型。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        VMT
       </span>
       <span style="font-size: 9pt;">
        方法的开发过程分为三个阶段：分析、设计和实现。分析阶段的主要任务是建立分析模型。设计阶段包括系统设计、对象设计和永久性对象设计。实现阶段就是用某一种环境来实现系统。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span lang="EN-US" style="font-size: 9pt;">
         7
        </span>
       </strong>
       <strong>
        <span style="font-size: 9pt;">
         、六种方法的比较
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·OMT
       </span>
       <span style="font-size: 9pt;">
        方法覆盖了应用开发的全过程，是一种比较成熟的方法，用几种不同的观念来适应不同的建模场合，它在许多重要观念上受到关系数据库设计的影响，适合于数据密集型的信息系统的开发，是一种比较完善和有效的分析与设计方法。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·Booch
       </span>
       <span style="font-size: 9pt;">
        方法并不是一个开发过程，只是在开发面向对象系统时应遵循的一些技术和 原则。
        <span lang="EN-US">
         Booch
        </span>
        方法是从外部开始，逐步求精每个类直到系统被实现。因此，它是一种分治法，支持循环开发，它的缺点在于不能有效地找出每个对象和类的操作。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·RDD
       </span>
       <span style="font-size: 9pt;">
        是一种用非形式的技术和指导原则开发合适的设计方案的设计技术。它用交互 填写
        <span lang="EN-US">
         CRC
        </span>
        卡片的方法完成设计，对大型系统设计不太适用。
        <span lang="EN-US">
         RDD
        </span>
        采用传统的方法确定对象类，有一定的局限性。另外，均匀地把行为分配给类也十分困难。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·
       </span>
       <span style="font-size: 9pt;">
        在
        <span lang="EN-US">
         OOAD
        </span>
        方法中，
        <span lang="EN-US">
         OOA
        </span>
        把系统横向划分为五个层次，
        <span lang="EN-US">
         OOD
        </span>
        把系统纵向划分为四个部分，从而形成一个清晰的系统模型。
        <span lang="EN-US">
         OOAD
        </span>
        适用于小型系统的开发。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·OOSE
       </span>
       <span style="font-size: 9pt;">
        能够较好地描述系统的需求，是一种实用的面向对象的系统开发方法，适合于商务处理方面的应用开发。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·VMT
       </span>
       <span style="font-size: 9pt;">
        基于现有面向对象方法中的成熟技术，采用这些方法中最好的思想、特色、观点以及技术，并把它们融合成一个完整的开发过程。因此
        <span lang="EN-US">
         VMT
        </span>
        是一种扬长避短的方法，它提供了一种实用的能够处理复杂问题的建模方法和技术。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         三、建模语言
         <span lang="EN-US">
          UML
         </span>
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        建模语言
        <span lang="EN-US">
         UML[4]
        </span>
        最初仅仅是
        <span lang="EN-US">
         OMT
        </span>
        方法、
        <span lang="EN-US">
         Booch
        </span>
        方法的统一。
        <span lang="EN-US">
         1995
        </span>
        年
        <span lang="EN-US">
         10
        </span>
        月，
        <span lang="EN-US">
         Grady Booch
        </span>
        和
        <span lang="EN-US">
         Jim Rumbaugh
        </span>
        联合推出了
        <span lang="EN-US">
         Unified Method 0.8
        </span>
        版本。这个方法力图实现
        <span lang="EN-US">
         OMT
        </span>
        方法和
        <span lang="EN-US">
         Booch
        </span>
        方法的统一。同年秋天，
        <span lang="EN-US">
         Ivar Jacobson
        </span>
        加入了
        <span lang="EN-US">
         Booch
        </span>
        和
        <span lang="EN-US">
         Rumbaugh
        </span>
        所在的
        <span lang="EN-US">
         Rational
        </span>
        软件公司，于是
        <span lang="EN-US">
         OOSE
        </span>
        方法也加入了统一的过程中。
        <span lang="EN-US">
         1997
        </span>
        年
        <span lang="EN-US">
         9
        </span>
        月
        <span lang="EN-US">
         1
        </span>
        日产生了
        <span lang="EN-US">
         UML 1.1
        </span>
        ，并被提交到了
        <span lang="EN-US">
         OMG
        </span>
        （
        <span lang="EN-US">
         Object Management Group
        </span>
        ），同年
        <span lang="EN-US">
         11
        </span>
        月被
        <span lang="EN-US">
         OMG
        </span>
        采纳。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        按照
        <span lang="EN-US">
         UML
        </span>
        文件的说法：
        <span lang="EN-US">
         UML
        </span>
        是一种用于软件系统制品制约的、可视化的构造及建档语言，也可用于业务建模以及其它非软件系统。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        从系统模型这一级别上看，
        <span lang="EN-US">
         UML
        </span>
        表示法由
        <span lang="EN-US">
         9
        </span>
        种图构成，它们是：静态结构图（
        <span lang="EN-US">
         Static Structure Diagram
        </span>
        ），其中包括类图（
        <span lang="EN-US">
         Class Diagram
        </span>
        ）和对象图（
        <span lang="EN-US">
         Object Diagram
        </span>
        ）；使用事例图（
        <span lang="EN-US">
         Use Case Diagram
        </span>
        ）；顺序图（
        <span lang="EN-US">
         Sequence Diagram
        </span>
        ）；协作图（
        <span lang="EN-US">
         Collaboration
        </span>
        ）；状态图（
        <span lang="EN-US">
         Statechart Diagram
        </span>
        ）；活动图（
        <span lang="EN-US">
         Activity Diagram
        </span>
        ）；实现图（
        <span lang="EN-US">
         Implementation Diagram
        </span>
        ），其中包括成分图（
        <span lang="EN-US">
         Component Diagram
        </span>
        ）和展开图（
        <span lang="EN-US">
         Deployment Diagram
        </span>
        ）。
        <span lang="EN-US">
         UML
        </span>
        的通用表示还有：串（
        <span lang="EN-US">
         String
        </span>
        ）、名字（
        <span lang="EN-US">
         Name
        </span>
        ）、标签（
        <span lang="EN-US">
         Label
        </span>
        ）、关键字（
        <span lang="EN-US">
         Keyword
        </span>
        ）、表达式（
        <span lang="EN-US">
         Expression
        </span>
        ）、注释（
        <span lang="EN-US">
         Note
        </span>
        ）等。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        UML
       </span>
       <span style="font-size: 9pt;">
        作为一种建模语言，具有以下特点：
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·UML
       </span>
       <span style="font-size: 9pt;">
        结合了
        <span lang="EN-US">
         Booch
        </span>
        方法、
        <span lang="EN-US">
         OMT
        </span>
        方法和
        <span lang="EN-US">
         OOSE
        </span>
        方法的概念，是一个单一的通用的建模语言；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·UML
       </span>
       <span style="font-size: 9pt;">
        的建模能力比其他面向对象方法更强，不仅适合于一般系统的开发，更擅长于并行、分布式系统的建模；
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        ·UML
       </span>
       <span style="font-size: 9pt;">
        是一种标准的建模语言，而不是一个标准的开发过程，它完全独立于开发过程。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p>
     <span lang="EN-US" style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
       </span>
      </span>
     </span>
    </p>
    <p>
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <strong>
        <span style="font-size: 9pt;">
         四、结束语
        </span>
       </strong>
       <span lang="EN-US" style="font-size: 9pt;">
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-size: 9pt;">
      <span style="font-family: 宋体;">
       <span style="font-size: medium;">
        要想实现软件生产自动化，实现开发方法的统一化、标准化是基本的一条。实现面向对象开发方法的标准化以后，就可以避免不同方法的差异，提高软件重用效率。此外，统一面向对象开发方法，也就是综合每个面向对象方法的优点，最大限度地发挥各种方法的长处。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
    <p style="text-indent: 24pt;">
     <span style="font-family: 宋体;">
      <span style="font-size: medium;">
       <span lang="EN-US" style="font-size: 9pt;">
        UML
       </span>
       <span style="font-size: 9pt;">
        是一种表达能力丰富的建模语言，而不是一种方法，因此，目前它还不能取代现有的各种面向对象的分析与设计方法。但是，随着
        <span lang="EN-US">
         UML
        </span>
        工作的进一步展开，必将有助于实现软件自动化。
        <span lang="EN-US">
        </span>
       </span>
      </span>
     </span>
    </p>
