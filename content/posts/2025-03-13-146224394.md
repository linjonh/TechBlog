     ChromeOS 133 版本更新
    </h2>
    <h4>
     <a id="1__Office__1">
     </a>
     1.
     <strong>
      增强托管用户的 Office 文件处理功能
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 133
     </strong>
     开始，
     <strong>
      托管用户
     </strong>
     现在可以
     <strong>
      无缝打开和编辑 Microsoft Office 文件
     </strong>
     （Word、PowerPoint、Excel），无论他们使用的是
     <strong>
      Microsoft 365（Office for the web）
     </strong>
     还是
     <strong>
      Google Workspace
     </strong>
     。
    </p>
    <h5>
     <a id="11__4">
     </a>
     1.1. 对不同用户的优化
    </h5>
    <p>
     <strong>
      对于使用 Microsoft 365（Office for the web）的企业
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       OneDrive 集成
      </strong>
      ：可直接在
      <strong>
       ChromeOS 文件应用（Files app）
      </strong>
      访问 OneDrive 文件。
     </li>
     <li>
      <strong>
       Microsoft 365 PWA 桌面集成
      </strong>
      ：提供类似桌面应用的体验。
     </li>
     <li>
      <strong>
       单点登录（SSO）
      </strong>
      ：适用于所有必需的 Office 应用。
     </li>
     <li>
      <strong>
       高级策略控制
      </strong>
      ：支持
      <strong>
       预配置策略
      </strong>
      ，优化 Office 文件管理。
     </li>
    </ul>
    <p>
     <strong>
      对于 Google Workspace 用户
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       优化本地文件向 Google Workspace 迁移
      </strong>
      ，让用户可以
      <strong>
       更顺畅地将 Office 文件转换为 Google Docs/Sheets/Slides
      </strong>
      进行编辑。
     </li>
    </ul>
    <h5>
     <a id="12__16">
     </a>
     1.2. 相关策略
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         策略名称
        </strong>
       </th>
       <th>
        <strong>
         作用
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         MicrosoftOneDriveMount
        </strong>
       </td>
       <td>
        控制是否在
        <strong>
         ChromeOS 文件应用（Files app）
        </strong>
        中挂载
        <strong>
         OneDrive
        </strong>
        。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         MicrosoftOneDriveAccountRestrictions
        </strong>
       </td>
       <td>
        限制可以登录 OneDrive 的 Microsoft 账户类型。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         MicrosoftOfficeCloudUpload
        </strong>
       </td>
       <td>
        允许用户将 Office 文件上传到
        <strong>
         Microsoft 365
        </strong>
        云端进行编辑。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         GoogleWorkspaceCloudUpload
        </strong>
       </td>
       <td>
        允许用户将 Office 文件上传到
        <strong>
         Google Workspace
        </strong>
        云端进行编辑。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         QuickOfficeForceFileDownloadEnabled
        </strong>
       </td>
       <td>
        强制 ChromeOS
        <strong>
         下载 Office 文件
        </strong>
        而不是直接在云端编辑。
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     此更新增强了 ChromeOS
     <strong>
      对 Office 文件的兼容性
     </strong>
     ，无论用户使用
     <strong>
      Microsoft 365 还是 Google Workspace
     </strong>
     ，都能更轻松地打开和编辑 Office 文件。
    </p>
    <h4>
     <a id="2__ChromeOS__28">
     </a>
     2.
     <strong>
      在 ChromeOS 设备上强制使用云存储
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 133
     </strong>
     开始，
     <strong>
      引入了真正的“云优先”体验
     </strong>
     ，允许管理员确保
     <strong>
      所有文件都存储在 Google Workspace 或 Microsoft OneDrive
     </strong>
     ，并
     <strong>
      阻止本地存储
     </strong>
     。
    </p>
    <h5>
     <a id="21__31">
     </a>
     2.1. 主要优势
    </h5>
    <ul>
     <li>
      ✅
      <strong>
       防止数据丢失
      </strong>
      ：所有文件均存储在云端，不会因设备故障或丢失而丢失。
     </li>
     <li>
      ✅
      <strong>
       降低安全风险
      </strong>
      ：避免敏感数据存储在本地，提高数据安全性。
     </li>
     <li>
      ✅
      <strong>
       支持法律合规（Legal Hold）
      </strong>
      ：确保所有文件都受到法律保留（Legal Hold）政策的约束。
     </li>
     <li>
      ✅
      <strong>
       适用于共享设备
      </strong>
      ：可防止不同用户在本地存储个人文件，确保数据隔离。
     </li>
    </ul>
    <h5>
     <a id="22__37">
     </a>
     2.2. 详细信息
    </h5>
    <ul>
     <li>
      <strong>
       如果启用此功能
      </strong>
      ，用户将
      <strong>
       无法
      </strong>
      在本地磁盘上存储文件，只能使用 Google Drive 或 OneDrive 进行文件存储和管理。
     </li>
     <li>
      <strong>
       管理员可配置相关策略
      </strong>
      来控制存储行为。
     </li>
    </ul>
    <h5>
     <a id="23__41">
     </a>
     2.3. 新增策略
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         策略名称
        </strong>
       </th>
       <th>
        <strong>
         作用
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         LocalUserFilesAllowed
        </strong>
       </td>
       <td>
        控制是否允许用户在 ChromeOS 设备上存储本地文件。若禁用，则文件必须存储在 Google Drive 或 OneDrive。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         LocalUserFilesMigrationDestination
        </strong>
       </td>
       <td>
        设置本地文件迁移的默认目标（Google Drive 或 OneDrive），用于帮助用户从本地存储迁移到云端。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         DownloadDirectory (OneDrive support)
        </strong>
       </td>
       <td>
        允许管理员将
        <strong>
         默认下载目录
        </strong>
        设置为
        <strong>
         OneDrive
        </strong>
        ，而不是本地存储。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         ScreenCaptureLocation
        </strong>
       </td>
       <td>
        允许管理员
        <strong>
         指定截屏文件的存储位置
        </strong>
        （例如 OneDrive 或 Google Drive）。
       </td>
      </tr>
     </tbody>
    </table>
    <h5>
     <a id="24__50">
     </a>
     2.4.
     <strong>
      如何启用该功能？
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       管理员可通过 LocalUserFilesAllowed 禁用本地存储
      </strong>
      。
     </li>
     <li>
      <strong>
       用户下载的文件、截图、创建的文档都会直接存储在云端
      </strong>
      。
     </li>
     <li>
      <strong>
       可使用 LocalUserFilesMigrationDestination 进行本地文件自动迁移
      </strong>
      。
     </li>
    </ol>
    <h4>
     <a id="3_ChromeOS_Bounce_Keys_55">
     </a>
     3.
     <strong>
      ChromeOS 的回弹键（Bounce Keys）功能
     </strong>
    </h4>
    <p>
     <strong>
      回弹键（Bounce Keys）
     </strong>
     是一项
     <strong>
      辅助功能
     </strong>
     ，旨在帮助
     <strong>
      手部灵活性受限或有震颤（颤抖）
     </strong>
     的用户更轻松地使用计算机。
    </p>
    <ul>
     <li>
      <strong>
       忽略短时间内的重复按键
      </strong>
      ，防止因
      <strong>
       不自主的多次按键
      </strong>
      而输入错误字符。
     </li>
     <li>
      <strong>
       支持自定义时间间隔
      </strong>
      ，用户可以根据个人需求调整
      <strong>
       按键间隔
      </strong>
      ，确保输入准确性。
     </li>
     <li>
      <strong>
       有助于减少误触
      </strong>
      ，适用于
      <strong>
       手部震颤、肌肉控制障碍
      </strong>
      等情况的用户，提高可用性。
     </li>
    </ul>

    </p>

    <h4>
     <a id="5_ChromeOS__73">
     </a>
     5.
     <strong>
      ChromeOS 新增键盘语言管理策略
     </strong>
    </h4>
    <p>
     从
     <strong>
      ChromeOS 133
     </strong>
     开始，新增
     <strong>
      AllowedInputMethodsForceEnabled
     </strong>
     策略，允许管理员
     <strong>
      强制安装
     </strong>
     先前通过
     <strong>
      AllowedInputMethods
     </strong>
     设置的键盘语言。
    </p>
    <ul>
     <li>
      <strong>
       管理员可预设并强制安装键盘语言
      </strong>
      ，用户无法自行更改。
     </li>
     <li>
      <strong>
       当该策略启用时，用户不能添加或删除键盘语言
      </strong>
      ，确保输入法配置符合企业或学校的要求。
     </li>
     <li>
      <strong>
       适用于受管设备
      </strong>
      （如企业或教育机构），确保输入法一致性。
     </li>
    </ul>

      <strong>

      </strong>
      报告。
     </li>

      <strong>
       离线状态更新优化
      </strong>
      ：Kiosk 设备的
      <strong>

      </strong>

     </li>
    </ul>
