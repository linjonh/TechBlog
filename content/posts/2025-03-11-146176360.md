  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>

   <div class="htmledit_views" id="content_views">
    <h4>
     1、json
    </h4>
    <pre><code class="language-java">String json = HttpRequest.post(getUrl(method, "v1", url, userId, appKey))
    .header("Content-type", "application/json") // 设置请求头为 JSON 格式
    .body(JSONUtil.toJsonStr(params)) // 请求体为 JSON 字符串
    .execute()
    .body();</code></pre>
    <h5>
     特点：
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        请求头
       </strong>
       ：
       <code>
        Content-type: application/json
       </code>
       ，表示请求体是 JSON 格式。
      </p>
     </li>
     <li>
      <p>
       <strong>
        请求体
       </strong>
       ：
       <code>
        body(JSONUtil.toJsonStr(params))
       </code>
       ，直接将
       <code>
        params
       </code>
       对象转换为 JSON 字符串作为请求体。
      </p>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：适用于后端接口需要接收 JSON 格式数据的场景，通常是 RESTful API。
      </p>
     </li>
    </ul>
    <p>
    </p>
    <h4>
     2、from表单
    </h4>
    <pre><code class="language-java">String response = HttpRequest.post(getUrl(method, "v1", url, userId, appKey))
    .header("Content-type", "application/x-www-form-urlencoded") // 设置请求头为表单格式
    .form("params", JSONUtil.toJsonStr(params)) // 请求体为表单格式，键值对形式
    .execute()
    .body();</code></pre>
    <h5>
     特点：
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        请求头
       </strong>
       ：
       <code>
        Content-type: application/x-www-form-urlencoded
       </code>
       ，表示请求体是表单格式。
      </p>
     </li>
     <li>
      <p>
       <strong>
        请求体
       </strong>
       ：
       <code>
        form("params", JSONUtil.toJsonStr(params))
       </code>
       ，将
       <code>
        params
       </code>
       对象转换为 JSON 字符串，并将其作为表单的一个字段（键为
       <code>
        params
       </code>
       ，值为 JSON 字符串）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        适用场景
       </strong>
       ：适用于后端接口需要接收表单数据的场景，通常是传统的表单提交或某些特定的 API。
      </p>
     </li>
     <li>
      <p>
       总结
      </p>
     </li>
    </ul>
