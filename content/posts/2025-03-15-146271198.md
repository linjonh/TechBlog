    <h2>
     <a id="you_4">
     </a>
     一、游游的you
    </h2>
    <p>
     题目链接：
     <a href="https://www.nowcoder.com/questionTerminal/cd117803b3364b218a8b3dcc498dee25" rel="nofollow">
      游游的you
     </a>
    </p>
    <h3>
     <a id="_8">
     </a>
     题目解析
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/851dacdd1e5249efa333f2363a87dbe3.png#pic_center"/>
    </p>
    <p>
     题目要求：
    </p>
    <blockquote>
     <p>
      输入
      <code>
       a
      </code>
      ，
      <code>
       b
      </code>
      ，
      <code>
       c
      </code>
      表示
      <code>
       y
      </code>
      、
      <code>
       o
      </code>
      、
      <code>
       u
      </code>
      三个字母的个数；
     </p>
     <p>
      将这些字母连成字符串，并且这里
      <code>
       you
      </code>
      三个字母相邻获得
      <code>
       2
      </code>
      分，两个
      <code>
       o
      </code>
      字母相邻获得
      <code>
       1
      </code>
      分。
     </p>
     <p>
      让我们输出最多可以获得多少分
     </p>
    </blockquote>
    <p>
     这里得分规则我们需要注意
    </p>
    <blockquote>
     <ul>
      <li>
       <code>
        you
       </code>
       相邻获得
       <code>
        2
       </code>
       分
      </li>
      <li>
       <code>
        oo
       </code>
       相邻获得
       <code>
        1
       </code>
       分，如果字符
       <code>
        ooo
       </code>
       可以获得
       <code>
        2
       </code>
       分，
       <code>
        oooo
       </code>
       可以获得
       <code>
        3
       </code>
       分
      </li>
     </ul>
    </blockquote>
    <h3>
     <a id="_26">
     </a>
     算法思路
    </h3>
    <p>
     我们知道了这个得分规则，我们想要获得更高的分数：
    </p>
    <blockquote>
     <ol>
      <li>
       尽可能的让
       <code>
        you
       </code>
       相邻
      </li>
      <li>
       如果不能组成
       <code>
        you
       </code>
       ，则就让所有的
       <code>
        o
       </code>
       相邻
      </li>
     </ol>
     <p>
      这样我们就可以知道如何去求最大得分了，
     </p>
     <ul>
      <li>
       首先就是
       <code>
        you
       </code>
       的个数（就等于
       <code>
        a
       </code>
       、
       <code>
        b
       </code>
       、
       <code>
        c
       </code>
       中最小的那一个）
      </li>
      <li>
       然后就是剩余
       <code>
        o
       </code>
       的个数，让剩余所有的
       <code>
        o
       </code>
       相邻得分就是
       <code>
        o
       </code>
       的个数-1；
      </li>
     </ul>
    </blockquote>
    <p>
     这样最多的得分就可以表示出来了；
    </p>
    <p>
     这里需要注意
     <strong>
      剩余
      <code>
       o
      </code>
      的个数可能为
      <code>
       0
      </code>
      ，这时就需要处理一下，否则就加了
      <code>
       -1
      </code>
     </strong>
    </p>

    <h2>
     <a id="_63">
     </a>
     二、腐烂的苹果
    </h2>
    <p>
     题目链接：
     <a href="https://www.nowcoder.com/practice/54ab9865ce7a45968b126d6968a77f34?tpId=196&amp;tqId=40529&amp;ru=/exam/oj" rel="nofollow">
      腐烂的苹果
     </a>
    </p>
    <h3>
     <a id="_67">
     </a>
     题目解析
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/97d2da90f3f843f5a78216a053b636cc.png#pic_center"/>
    </p>
    <blockquote>
     <p>
      题目：
     </p>
     <p>
      有一个
      <code>
       m*n
      </code>
      的网格，其中每一个格子有三种情况（
      <code>
       没有苹果/0
      </code>
      、
      <code>
       有好的苹果/1
      </code>
      、
      <code>
       有腐烂的苹果/2
      </code>
      )
     </p>
     <p>
      其中，腐烂的苹果每分钟都会向四周（上下左右）传播病菌，相邻的苹果都会腐烂。
     </p>
     <p>
      让我们求出需要多少分钟，会导致所有的苹果变腐烂；（如果有苹果一直不会腐烂，就返回
      <code>
       -1
      </code>
      ）
     </p>
    </blockquote>
    <h3>
     <a id="_80">
     </a>
     算法思路
    </h3>
    <p>
     读懂了题目，现在来看如何去实现这个算法（这里简述一下）
    </p>
    <blockquote>
     <p>
      这道题的标签是
      <code>
       BFS
      </code>
      广度优先遍历
     </p>
     <p>
      所以，这里先遍历给的数组，让所有的
      <code>
       2
      </code>
      （腐烂的苹果）都存放到一个队列当中；
     </p>
     <p>
      再进行广度优先遍历，对队列中腐败的苹果进行扩散
     </p>
     <p>
      这里注意：
      <strong>
       题目要求出时间
      </strong>
      ，所有我们要按时间来进行
     </p>
    </blockquote>
    <p>
     <code>
      BFS
     </code>
     遍历，什么意思呢？
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e17e35e5ef9c4ef7ad5c3c5d7d5da6b5.png#pic_center"/>
    </p>
    <p>
     这里就来捋一捋整个代码的流程
    </p>
    <blockquote>
     <p>
      首先准备工作，定义
      <code>
       dx[4]
      </code>
      、
      <code>
       dy[4]
      </code>
      表示上下左右位置下标变化值；定义
      <code>
       vis
      </code>
      数组来表示每个位置苹果是否腐烂（
      <code>
       true
      </code>
      表示腐败`）；
     </p>
     <ul>
      <li>
       遍历
       <code>
        grid
       </code>
       ，将所有值为
       <code>
        2
       </code>
       的位置的下标入队列；
      </li>
      <li>
       进行
       <code>
        BFS
       </code>
       广度优先遍历，进行腐烂的扩散操作；
      </li>
      <li>
       扩散过程：依次对队列中的每个下标进行上下左右扩散，如果
       <code>
        扩散位置不越界，并且扩散位置有苹果且苹果吗，没有被传入腐败
       </code>
       ，那就修改扩散位置的
       <code>
        vis[x][y]
       </code>
       为
       <code>
        true
       </code>
       ，并让扩散位置下标入队列。
      </li>
      <li>
       最后，遍历
       <code>
        grid
       </code>
       数组，如果存在一个位置有苹果且苹果没有被传染腐败，就返回
       <code>
        -1
       </code>
       否则返回
       <code>
        ret-1
       </code>
       结果。
      </li>
     </ul>
    </blockquote>

    <h2>
     <a id="_164">
     </a>
     三、孩子们的游戏
    </h2>
    <p>
     题目链接：
     <a href="https://www.nowcoder.com/practice/f78a359491e64a50bce2d89cff857eb6?tpId=13&amp;tqId=11199&amp;ru=/exam/oj" rel="nofollow">
      孩子们的游戏
     </a>
    </p>
    <h3>
     <a id="_168">
     </a>
     题目描述
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2a600cd46a7b48bbaf306cdc225a0123.png#pic_center"/>
    </p>
    <p>
     这是一道经典的
     <code>
      约瑟夫
     </code>
     问题，对于
     <code>
      约瑟夫
     </code>
     ，可是早有耳闻；这里就直接提供思路
    </p>
    <blockquote>
     <p>
      这里大致可以将所有的思路/解法分为两种
     </p>
     <ul>
      <li>
       模拟实现
      </li>
      <li>
       使用递推/递归/动态规划
      </li>
     </ul>
     <p>
      为什么可以分为两种呢？
     </p>
     <p>
      第一种就是模拟实现整个过程，可以说是暴力解法了
     </p>
     <p>
      第二种它们都有一个共同的思路，就是将问题一步一步划分，然后寻找子问题的关系；从而简化整个过程。
     </p>
    </blockquote>
    <h3>
     <a id="_186">
     </a>
     算法分析
    </h3>
    <p>
     这里对于两种思路来简单分析一下：
    </p>
    <p>
     <strong>
      首先模拟实现整个过程
     </strong>
    </p>
    <blockquote>
     <p>
      我们可以使用链表或者数组来模拟整个过程。（这里使用数组来模拟）
     </p>
     <ul>
      <li>
       我们定义一个数组(
       <code>
        bool
       </code>
       类型的)，来表示当前位置下标对应的人还是否在圈内
      </li>
      <li>
       定义用来计数的一些变量
       <code>
        i
       </code>
       表示当前遍历位置的下标、
       <code>
        sum
       </code>
       表示圈内剩余的人数、
       <code>
        x
       </code>
       表示当前应该报的数
      </li>
      <li>
       循环遍历整个数组，如果遍历遇到当前位置对应的人不在圈内
       <code>
        v[i] == 0
       </code>
       ，就
       <code>
        i++
       </code>
       并直接执行下一次循环
      </li>
      <li>
       如果当前位置的人在圈内，就继续判断报数是否等于m-1
       <code>
        x == m-1
       </code>
       ，如果等于将让当前位置的人出圈，再继续执行循环遍历
      </li>
      <li>
       知道圈内剩余一个人
       <code>
        sum == 1
       </code>
       ，循环结束，最后返回剩余那个人的下标即可。
      </li>
     </ul>
    </blockquote>
    <p>
     <strong>
      其次，第二种思路就说动态规划
     </strong>
    </p>
    <blockquote>
     <p>
      这种思路主要就在于理清楚
      <code>
       dp[i]
      </code>
      代表了什么、和
      <code>
       dp[i]状态转移方程
      </code>
      。
     </p>
     <ul>
      <li>
       这里
       <code>
        dp[i]
       </code>
       代表当有
       <code>
        i
       </code>
       个人时最后剩余的那个人
      </li>
     </ul>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/92e1a35190a44f8aab3dc6864efca14c.png#pic_center"/>
    </p>
    <p>
     如上图所示，我们找到了第一次报圈和第二次报圈对应下标的关系
     <code>
      dp[n] = (dp[n-1] + n) %b
     </code>
    </p>
    <p>
     我们将
     <code>
      dp[i]
     </code>
     表示有
     <code>
      i
     </code>
     个人报圈最后胜出的人，那么
     <code>
      dp[0] = 0
     </code>
     ；那
     <code>
      i=2
     </code>
     时；
     <code>
      dp[2] = (dp[1] +m) %n
     </code>
     。
    </p>
    <p>
     以此类推，我们就可以直接推出来有
     <code>
      n
     </code>
     个人报圈时，最后胜出的人。
    </p>
