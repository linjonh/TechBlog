    <h2>
     <a id="11_Internet_4">
     </a>
     1.1 什么是Internet（网络）
    </h2>
    <h3>
     <a id="_5">
     </a>
     从“构成”的角度看
    </h3>
    <h4>
     <a id="_6">
     </a>
     计算机网络
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/868afbaecf214335b80d329bd3af2278.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/18ccc85f668f4f3da58614fe6e6808c0.png">
       <br/>
       一个网络有两个要素：
       <strong>
        节点
       </strong>
       和
       <strong>
        边
       </strong>
      </img>
     </img>
    </p>
    <p>
     <strong>
      计算机网络的节点
     </strong>
     ：由多个节点组成的网络，其中，节点包括：主机节点（方框）、数据交换节点（包括中继器、路由器、交换机）（圆圈）
     <br/>
     <strong>
      计算机网络的边
     </strong>
     ：
     <br/>
     有两类：其中主机节点与数据交换节点之间的边称为“接入网链路”，而“数据交换节点”之间的边称为“主干链路”
     <br/>
     （即，有方框参与，就是“接入网链路”；只有圆圈参与，就是“主干链路”）
    </p>
    <p>
     <strong>
      协议
     </strong>
     ：不同厂家的网卡根据同一个协议可以进行通信
    </p>
    <h4>
     <a id="_17">
     </a>
     互联网
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4aa09758ea17489aa71959eb1c57bb7d.png">
      <br/>
      而互联网，就是以数以万计的节点（主机节点、数据交换节点），以及节点与节点之间的链路，外加一簇协议组成的一个大的网络
      <br/>
      （与计算机网络的构成一样，就是数量是巨量的，千家万户的网络连接在一起）
     </img>
    </p>
    <p>
     其中，
     <br/>
     <strong>
      节点：
     </strong>
     <br/>
     主机节点包括PC终端、服务器、手机、应用程序等
     <br/>
     数据交换节点包括路由器交换机等交换设备
    </p>
    <p>
     <strong>
      边：
     </strong>
     <br/>
     逻辑上是分为“接入网链路”、“主干链路”
     <br/>
     物理上，则是由光纤、同轴电缆、无线电等组成，用于链接各个节点，从而构成网络。这些设备的一个重要指标就是传输速率（简称“带宽”）
    </p>
    <h4>
     <a id="_31">
     </a>
     详细介绍“协议”
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/258e52d7fda94a31962c26bf7df1e9cf.png">
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/05a173a9377a4f3b8f53b1512e3f209c.png"/>
     </img>
    </p>
    <p>
     计算机网络不仅有节点和边，还有协议
    </p>
    <p>
     而协议，是
     <strong>
      对等层的实体在通信的过程中应该遵守的规则集合
     </strong>
    </p>
    <p>
     比如，在应用层：这一层的协议规定我的一个报文的格式是怎样的，前多少多少字节代表什么数据内容，一个符号代表什么意思，按照什么格式去解析报文等等，遵守同一份应用层协议，双方才能进行有效的通信（才能听懂对方在说什么）
    </p>
    <p>
     在传输层有TCP、UDP协议，网络层有IP协议，这些协议可以保证不同厂商生产的网卡可以进行无障碍的通信，因为不同的厂商遵守着同一套协议规则
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/95d29aa1496a484d9987fe88ab2720fd.png"/>
    </p>
    <h3>
     <a id="_46">
     </a>
     从“服务”的角度看
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/4c905add91f44842919215e3d9d25853.png"/>
    </p>
    <p>
     从“服务”的角度来看，计算机网络是：
     <strong>
      分布式的应用进程
     </strong>
     （应用层） + 为分布式的进程提供通信服务的
     <strong>
      基础设施
     </strong>
     （一个主机应用层以下的所有设施的软件和硬件、以及支持两个主机通信的网络通信设备（路由器、交换机）的软件和硬件）
    </p>
    <p>
     而
     <strong>
      分布式的应用进程
     </strong>
     ，本质上是一个
     <strong>
      进程
     </strong>
     ，本质上还是在
     <strong>
      运行代码
     </strong>
     ，是我们程序员编程写出来的，所以，我们平时做的高级语言的编程，就是
     <strong>
      处于应用层
     </strong>
    </p>
    <p>
     而
     <strong>
      通信基础设施
     </strong>
     为
     <strong>
      应用进程（应用层，或者说直白点就是编程方面）
      <strong>
       提供的
      </strong>
      服务
     </strong>
     ，具体体现在“提供API接口”（socket编程中的API接口：建立socket、选择TCP还是DUP（传输层提供）、申请建立连接、发送报文、接收报文等API）
    </p>
    <h2>
     <a id="12__56">
     </a>
     1.2 网络的边缘
    </h2>
    <h3>
     <a id="_57">
     </a>
     总览
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f4aa523d6ed14c9e8151d89f039f6ecc.png"/>
     <br/>
     1、对于一个互联网来说，我们的主机节点是最边缘的，所以，我们称主机是网络的
     <strong>
      边缘
     </strong>
     （方框）
    </p>
    <p>
     作用：主机上有我们运行的应用进程，这些进程为我们人类服务（各种各样的APP），这些应用进程是网络存在的理由，我们是为了科技更好的服务人民，所以，这些边缘的作用就是科技更好的服务人民
    </p>
    <p>
     2、而成百上千个数据交换节点（路由器、交换机），是网络的
     <strong>
      核心
     </strong>
     （圆圈）
    </p>
    <p>
     作用：主机与主机之间进行网络通信，不可能就是一对一的拉根线，全球就全是线了，需要一些复用线路，所以，这些数据交换节点就是实现了通信线路的复用
    </p>
    <p>
     3、而从边缘接入到核心的过程，叫
     <strong>
      接入
     </strong>
    </p>
    <p>
     作用：顾名思义，就是将边缘接入核心，从而使得两个边缘可以通信
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/d7e83c616eb64ca299fb7805afd187d0.png"/>
     <br/>
     本小节介绍一下“网络的边缘”，并且侧重于强调网络边缘中的
     <strong>
      应用进程
     </strong>
    </p>
    <h3>
     <a id="_78">
     </a>
     不同“网络边缘（进程）”之间的通信模式
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/446d80df1bfa4073930081f52cae453f.png"/>
     <br/>
     实际上说的是
     <strong>
      网络边缘（主机）
      <strong>
       上的
      </strong>
      应用进程
     </strong>
     （
     <strong>
      但网络边缘是主机，并不是应用进程，主机包含应用进程
     </strong>
     ）
    </p>
    <p>
     第一种：客户端服务器模式（这里的客户端包含CS和BS，是一种广义的CS模式）
     <br/>
     这种模式就是一个服务器，挂着许多个客户端，不同的客户端向服务器请求资源，服务器向客户端们发放资源
    </p>
    <p>
     而第二种，就是对等模式
     <br/>
     就是该产品会有好多个客户端，而每个客户端在前期会向服务器请求一些自己需要的资源，而有了一定的资源积累后，这些客户端们可以共享信息，即客户端1向客户端2请求资源，则此时客户端2就是客户端1的服务端，客户端2将自己的一些资源给客户端1。客户端1会从其他客户端东拼西凑，拼凑出自己想要的完整资源
    </p>
    <h3>
     <a id="TCP_89">
     </a>
     网络边缘（进程）采用基础设施提供的面向连接的服务（TCP）
    </h3>
    <p>
     不管是什么模式，不同网络边缘（主机）上的“应用程序”通信时，基础设施会提供服务，例如应用进程会调用来自传输层的服务，而传输层提供两种协议：TCP、UDP协议，这里介绍TCP协议（面向连接的服务）
    </p>
    <p>
     TCP是面向连接的稳定的连接，且报文内容不会丢不会漏不会错
    </p>
    <p>
     TCP具备有以下三个特性
     <br/>
     1、TCP下面层的服务是不可靠的，TCP靠自己的努力向上层提供
     <strong>
      可靠
     </strong>
     的服务
     <br/>
     2、并且TCP还可以进行
     <strong>
      流量控制
     </strong>
     ，协调发送方与接收方的流量适配
     <br/>
     3、同时，TCP还可以感知两个进程通信时经过的每一跳组成的链路是否拥堵，根据拥堵情况进行发送方与接收方的数据量协调，称为
     <strong>
      拥塞控制
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/ae17909bca41403cbef0db9817e0ecb7.png"/>
    </p>
    <h3>
     <a id="DUP_100">
     </a>
     网络边缘（进程）采用的基础设施提供的无连接的服务（DUP）
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/be071c535ba7484197e61fff74fda14f.png"/>
     <br/>
     DUP没有连接服务，没有可靠，没有流量控制，没有拥塞控制
    </p>
    <p>
     但是！他速度快，这是他最大的优点
    </p>
