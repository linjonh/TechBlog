    <h2 id="Jiuwa" name="Jiuwa" style="background-color:transparent">
     一、注释
    </h2>
    <p id="u12f7c72b">
     在C#中，注释是用来在代码中添加说明性文字的，它们不会被编译器执行，但有助于提高代码的可读性和维护
    </p>
    <p id="u50c894f4">
     性。C#支持三种类型的注释：
    </p>
    <ol>
     <li id="ub78675ba">
      <strong>
       单行注释
      </strong>
     </li>
     <li id="udcf06da3">
      <strong>
       多行注释
      </strong>
     </li>
     <li id="uecb88e2c">
      <strong>
       XML文档注释
      </strong>
     </li>
    </ol>
    <h3 id="h5mD9" name="h5mD9">
     1. 单行注释
    </h3>
    <p id="u2ea7e1cb">
     单行注释使用
     <code>
      //
     </code>
     开头，从
     <code>
      //
     </code>
     开始到该行结束的所有内容都会被视为注释。
    </p>
    <p id="ua03bc27e">
     <strong>
      示例：
     </strong>
    </p>
    <pre id="FyvUW"><code>// 这是一个单行注释
int number = 5; // 这也是单行注释</code></pre>
    <h3 id="NPMRJ" name="NPMRJ">
     2. 多行注释
    </h3>
    <p id="ucae100b9">
     多行注释使用
     <code>
      /*
     </code>
     开头，以
     <code>
      */
     </code>
     结尾。它们可以跨越多行。
    </p>
    <p id="ufe54413b">
     <strong>
      示例：
     </strong>
    </p>
    <pre id="MLbQ5"><code>/* 这是一个多行注释的开始
   这是多行注释的中间部分
   这是多行注释的结束 */
int number = 10; // 这行代码不会被视为多行注释的一部分</code></pre>
    <h3 id="GErrM" name="GErrM">
     3. XML文档注释
    </h3>
    <p id="u052aa7c2">
     XML文档注释以
     <code>
      ///
     </code>
     开头，这种注释通常用于生成文档（例如，使用Visual Studio的XML文档注释工具）。它
    </p>
    <p id="ue5dae895">
     们遵循XML的格式，可以包含标签和属性，用于提供更丰富的代码描述信息。
    </p>
    <p id="u92c6cdd3">
     <strong>
      示例：
     </strong>
    </p>
    <pre id="HCSy3"><code>/// &lt;summary&gt;
/// 这是一个计算两个数字之和的方法。
/// &lt;/summary&gt;
/// &lt;param name="a"&gt;第一个数字&lt;/param&gt;
/// &lt;param name="b"&gt;第二个数字&lt;/param&gt;
/// &lt;returns&gt;返回两个数字的和&lt;/returns&gt;
public int Add(int a, int b)
{
    return a + b;
}</code></pre>
    <h3 id="M0XuT" name="M0XuT">
     4. 使用建议和最佳实践：
    </h3>
    <ul>
     <li id="uae00e4ac">
      <strong>
       保持代码清晰
      </strong>
      ：使用注释来解释复杂的逻辑、算法或者为什么这样实现。
     </li>
     <li id="u2e86f69b">
      <strong>
       避免过度注释
      </strong>
      ：好的代码应该尽可能自解释，过多的注释可能会让代码显得杂乱无章。
     </li>
     <li id="u5d1220ff">
      <strong>
       更新和维护
      </strong>
      ：当代码发生变化时，确保相关的注释也随之更新，避免误导。
     </li>
     <li id="u1aa211ad">
      <strong>
       使用XML文档注释生成文档
      </strong>
      ：对于公开的API或库，使用XML文档注释可以帮助自动生成文档，这对于维护和团队协作非常有帮助。
     </li>
     <li id="u56fd88aa">
      <strong>
       遵循团队的编码规范
      </strong>
      ：如果在一个团队中工作，确保所有成员遵循相同的注释规范和风格。
     </li>
    </ul>
    <p id="u427f0899">
     通过合理使用这些注释技巧，你可以显著提高代码的可读性和维护性。
    </p>
    <h2 id="Cqn2J" name="Cqn2J">
     二、访问修饰符
    </h2>
    <p id="uc807c586">
     在C#中，访问修饰符是用来控制类、方法、属性、事件和索引器等成员的访问权限的。它们决定了这些成员可以
    </p>
    <p id="u21ad0be3">
     被哪些代码访问。C#中共有五种访问修饰符，每种都有其特定的用途和范围。以下是这五种访问修饰符的详细解
    </p>
    <p id="ucd442bf6">
     释：
    </p>
    <h3 id="C2Ttd" name="C2Ttd">
     1. public
    </h3>
    <p id="ua87627a7">
     <strong>
      public
     </strong>
     修饰符表示成员可以被任何其他代码访问，无论是同一个程序集内的代码还是其他程序集的代码。
    </p>
    <p id="ue51ae874">
     示例：
     <code>
      public class MyClass { }
     </code>
     表示这个类可以在任何地方被访问。
    </p>
    <h3 id="jTKLV" name="jTKLV">
     2. private
    </h3>
    <p id="ue9f18aae">
     <strong>
      private
     </strong>
     修饰符表示成员只能被定义它的类内部访问。
    </p>
    <p id="ub8ab44ba">
     这意味着，即使是在同一个程序集内，其他类也无法访问该成员。
    </p>
    <p id="u0b6dac90">
     示例：
     <code>
      private int myVariable;
     </code>
     表示这个变量只能在定义它的类内部被访问。
    </p>
    <h3 id="X5hy4" name="X5hy4">
     3. protected
    </h3>
    <p id="u424c68d6">
     <strong>
      protected
     </strong>
     修饰符表示成员可以被定义它的类以及所有派生类（子类）访问。
    </p>
    <p id="u65a74d50">
     但是，如果这个成员位于不同的程序集，那么只有在派生类位于同一个程序集内时才能访问。
    </p>
    <p id="u20da0253">
     示例：
     <code>
      protected virtual void MyMethod() { }
     </code>
     表示这个方法可以在派生类中被重写和调用。
    </p>
    <h3 id="tp3AV" name="tp3AV">
     4. internal
    </h3>
    <p id="u46296810">
     <strong>
      internal
     </strong>
     或称为
     <strong>
      assembly
     </strong>
     修饰符表示成员可以被同一个程序集内的任何代码访问，但不能被其他程序集的代码
    </p>
    <p id="u1cc3c6a8">
     访问。
    </p>
    <p id="u868c3e56">
     示例：
     <code>
      internal class MyInternalClass { }
     </code>
     表示这个类只能在定义它的程序集内被访问。
    </p>
    <h3 id="V6cQr" name="V6cQr">
     5. protected internal 或 protected and internal
    </h3>
    <p id="u7932546d">
     <strong>
      protected internal
     </strong>
     修饰符是
     <strong>
      protected
     </strong>
     和
     <strong>
      internal
     </strong>
     的组合，表示成员可以被定义它的类、所有派生类以及同
    </p>
    <p id="ue1e3faf9">
     一个程序集内的任何代码访问，但不能被其他程序集的代码访问。
    </p>
    <p id="u326480a8">
     示例：
     <code>
      protected internal string MyProperty { get; set; }
     </code>
    </p>
    <p id="ub439306d">
     表示这个属性可以在派生类和同一个程序集内被访问，但不能被其他程序集访问。
    </p>
    <h3 id="AgnB0" name="AgnB0">
     6. private protected 或 private and protected
    </h3>
    <p id="u02784d07">
     <strong>
      private protected
     </strong>
     是C# 7.2引入的，它结合了
     <strong>
      private
     </strong>
     和
     <strong>
      protected
     </strong>
     的特性，使得成员只能被定义它的类和所
    </p>
    <p id="u47ad014a">
     有派生类（无论是否在同一程序集内）访问。
    </p>
    <p id="ua74b1d38">
     示例：
     <code>
      private protected void MyMethod() { }
     </code>
     表示这个方法只能在定义它的类和所有派生类中被访问。
    </p>
    <h3 id="R16K1" name="R16K1">
     7. 使用场景
    </h3>
    <ul>
     <li id="u2b5272d7">
      <strong>
       public
      </strong>
      ：当你希望类的成员对所有外部代码都可见时使用。
     </li>
     <li id="ue933a049">
      <strong>
       private
      </strong>
      ：当你希望类的成员仅在其定义的类内部可用时使用。
     </li>
     <li id="ud12d07c3">
      <strong>
       protected
      </strong>
      ：当你希望类的成员在其自身和派生类中可用时使用。
     </li>
     <li id="uf7205289">
      <strong>
       internal
      </strong>
      ：当你希望类的成员在同一个程序集内可用时使用。
     </li>
     <li id="u84b23f97">
      <strong>
       protected internal
      </strong>
      ：当你希望类的成员在同一个程序集内或在派生类中可用时使用。
     </li>
     <li id="u8ea27086">
      <strong>
       private protected
      </strong>
      ：当你希望类的成员在定义它的类和所有派生类中可用时使用，无论它们是否在同一程序集内。
     </li>
    </ul>
    <p id="u38d1b622">
     正确选择和使用这些访问修饰符对于设计良好的、可维护的C#代码非常重要。
    </p>
    <h2 id="QYOkr" name="QYOkr">
     三、标识符
    </h2>
    <p id="ub16e30c1">
     在C#中，标识符是用来命名变量、方法、类、接口、事件、属性等元素的名称。
    </p>
    <p id="u67aef84d">
     它们遵循一系列规则和约定，以确保代码的可读性和一致性。
    </p>
    <p id="ub9f7fb2d">
     下面是一些关于C#中标识符的重要规则和细节：
    </p>
    <h3 id="SL32s" name="SL32s">
     1. 命名规则
    </h3>
    <ul>
     <li id="ubb17af0b">
      <strong>
       首字母规则
      </strong>
      ：标识符可以以字母或下划线（_）开头。在C#中，标识符不能以下划线开头。
     </li>
     <li id="u55c6b9c3">
      <strong>
       后续字符规则
      </strong>
      ：标识符可以包含字母、数字、下划线（_）和@符号。但是，不能包含空格或其他特殊字符。
     </li>
     <li id="u1cb858df">
      <strong>
       大小写敏感
      </strong>
      ：C#是大小写敏感的，这意味着
      <code>
       variable
      </code>
      和
      <code>
       Variable
      </code>
      被视为两个不同的标识符。
     </li>
    </ul>
    <h3 id="dsfHx" name="dsfHx">
     2. 命名约定
    </h3>
    <p id="ud3834559">
     C#社区遵循一些命名约定，以提高代码的可读性和维护性：
    </p>
    <ul>
     <li id="u83314885">
      <strong>
       驼峰命名法（Camel Case）
      </strong>
      ：
     </li>
    </ul>
    <ul>
     <li>
      <ul>
       <li id="uc0f02ce0">
        <strong>
         小驼峰命名法（lowerCamelCase）
        </strong>
        ：第一个单词的首字母小写，后续单词的首字母大写。例如：
        <code>
         myVariable
        </code>
        。
       </li>
       <li id="u66055fb5">
        <strong>
         大驼峰命名法（UpperCamelCase或PascalCase）
        </strong>
        ：每个单词的首字母都大写。例如：
        <code>
         MyClass
        </code>
        。
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li id="u4ded6457">
      <strong>
       匈牙利命名法（Hungarian Notation）
      </strong>
      ：尽管不常用，但某些旧代码中仍可见。匈牙利命名法通过在变量名前加上一个或多个字母来表示变量的类型或作用。例如：
      <code>
       iCount
      </code>
      表示一个整型计数器。
     </li>
    </ul>
    <h3 id="cMiba" name="cMiba">
     3. 关键字和保留字
    </h3>
    <p id="u0b6232aa">
     C#有一些关键字和保留字不能用作标识符，因为它们已经被赋予了特定的意义。例如：
    </p>
    <pre id="uOXW3"><code>int class = 5; // 错误，因为'class'是C#的关键字</code></pre>
    <p id="u78fb991c">
     完整的C#关键字列表可以在官方文档中找到，例如：
     <a href="https://docs.microsoft.com/en-us/dotnet/csharp/language-reference/keywords/" rel="nofollow" title="C# Keywords">
      C# Keywords
     </a>
     。
    </p>
    <h3 id="ZIUZF" name="ZIUZF">
     4. 使用@符号
    </h3>
    <p id="u9cc01d36">
     如果你需要使用一个以关键字命名的变量名，或者变量名与C#的保留字冲突，你可以在变量名前加上@符号
    </p>
    <p id="uf1fd6339">
     来“转义”关键字或保留字。例如：
    </p>
    <pre id="Rf0X4"><code>int @class = 5; // 正确，使用@来避免关键字冲突</code></pre>
    <h3 id="HE5yk" name="HE5yk">
     5. 最佳实践
    </h3>
    <ul>
     <li id="u5e7bfece">
      <strong>
       清晰和描述性
      </strong>
      ：选择能够清晰描述变量或对象用途的名称。
     </li>
     <li id="uda22514d">
      <strong>
       一致性
      </strong>
      ：在项目中保持一致的命名约定。
     </li>
     <li id="ua906b84b">
      <strong>
       避免混淆
      </strong>
      ：避免使用可能引起混淆的缩写或过于简短的名称。
     </li>
    </ul>
    <p id="u6de57200">
     <strong>
      示例
     </strong>
    </p>
    <pre id="ktr52"><code>public class MyClass {
    private int myVariable; // 小驼峰命名法
    private int MyProperty { get; set; } // 大驼峰命名法，用于属性
    public static void Main() {
        int iCount = 10; // 小驼峰命名法，表示一个计数器
        Console.WriteLine(iCount);
    }
}</code></pre>
    <p id="u9499b7e6">
     遵循这些规则和最佳实践将有助于编写更清晰、更易于维护的C#代码。
    </p>
    <h2 id="hQySk" name="hQySk">
     四、关键字
    </h2>
    <p id="ube1d4b7f">
     在C#中，关键字是用来表示程序中特定功能的词汇，它们在编程时具有特殊的含义和用途。了解这些关键字对于
    </p>
    <p id="u3875ffec">
     编写有效的C#代码至关重要。C#的关键字可以分为以下几类：
    </p>
    <h3 id="JKGXN" name="JKGXN">
     1. 访问修饰符
    </h3>
    <ul>
     <li id="uc9be6f5f">
      <code>
       public
      </code>
      ：允许访问的类型或成员可以被任何其他代码访问。
     </li>
     <li id="u0303a2da">
      <code>
       private
      </code>
      ：限制访问的类型或成员只能被其所属的类访问。
     </li>
     <li id="uce71d81b">
      <code>
       protected
      </code>
      ：允许访问的类型或成员只能被其所属的类或派生类访问。
     </li>
     <li id="u178d7f00">
      <code>
       internal
      </code>
      ：允许访问的类型或成员只能在同一个程序集内被访问。
     </li>
     <li id="u7e8c0b85">
      <code>
       protected internal
      </code>
      ：允许访问的类型或成员只能在同一个程序集内或派生类中被访问。
     </li>
     <li id="u3dead1cb">
      <code>
       private protected
      </code>
      ：允许访问的类型或成员只能在同一个程序集内的派生类中被访问。
     </li>
    </ul>
    <h3 id="eq0jN" name="eq0jN">
     2. 类型关键字
    </h3>
    <ul>
     <li id="uace1e5f8">
      <code>
       class
      </code>
      ：定义一个类。
     </li>
     <li id="ucbd84583">
      <code>
       struct
      </code>
      ：定义一个值类型结构体。
     </li>
     <li id="u0546b6e2">
      <code>
       interface
      </code>
      ：定义一个接口。
     </li>
     <li id="u2673f238">
      <code>
       enum
      </code>
      ：定义一个枚举类型。
     </li>
     <li id="u2af072fa">
      <code>
       delegate
      </code>
      ：定义一个委托。
     </li>
    </ul>
    <h3 id="GHthL" name="GHthL">
     3. 方法修饰符
    </h3>
    <ul>
     <li id="u890f9567">
      <code>
       static
      </code>
      ：表示方法属于类而不是类的实例。
     </li>
     <li id="ue9ecd0c4">
      <code>
       virtual
      </code>
      ：表示方法可以在派生类中被重写。
     </li>
     <li id="u9339f271">
      <code>
       override
      </code>
      ：表示方法重写基类中的虚方法。
     </li>
     <li id="u452ccf1a">

    <h2 id="wKQNy" name="wKQNy">
     七、方法
    </h2>
    <p id="u76ef8ba9">
     在C#中，方法（也称为函数）是执行特定任务的代码块。它们可以被定义在一个类中，并被其他类或方法调用。
    </p>
    <p id="u00826195">
     C#中的方法可以是静态的（属于类本身）或实例的（属于类的实例）。下面将详细解释C#中方法的各个方面。
    </p>
    <h3 id="JI5S9" name="JI5S9">
     1. 方法的定义
    </h3>
    <p id="u1e9e835b">
     一个C#方法的基本定义格式如下：
    </p>
    <pre id="Cel7d"><code>[访问修饰符] [返回类型] 方法名([参数列表])
{
    // 方法体
}</code></pre>
    <ul>
     <li id="uee404d39">
      <strong>
       访问修饰符
      </strong>
      ：如
      <code>
       public
      </code>
      ,
      <code>
       private
      </code>
      ,
      <code>
       protected
      </code>
      ,
      <code>
       internal
      </code>
      等，控制方法的访问权限。
     </li>
     <li id="u2df432e1">
      <strong>
       返回类型
      </strong>
      ：方法返回的数据类型，如果是
      <code>
       void
      </code>
      则表示不返回任何值。
     </li>
     <li id="ub39cab13">
      <strong>
       方法名
      </strong>
      ：标识符，遵循C#的命名规则。
     </li>
     <li id="ubfa1562e">
      <strong>
       参数列表
      </strong>
      ：括号内包含一个或多个参数，每个参数由类型和名称组成，参数间用逗号分隔。
     </li>
     <li id="uce5646b3">
      <strong>
       方法体
      </strong>
      ：包含执行任务的代码块。
     </li>
    </ul>
    <h3 id="suDUV" name="suDUV">
     2. 示例
    </h3>
    <pre id="sPD1p"><code>public class ExampleClass
{
    // 无参数无返回值的方法
    public void SayHello()
    {
        Console.WriteLine("Hello, World!");
    }
 
    // 有参数有返回值的方法
    public int Add(int a, int b)
    {
        return a + b;
    }
}</code></pre>
    <h3 id="CMb4j" name="CMb4j">
     3. 方法的调用
    </h3>
    <ul>
     <li id="u75cdb03d">
      <strong>
       静态方法
      </strong>
      ：使用类名直接调用。
     </li>
    </ul>
    <pre id="Sqald"><code>ExampleClass.SayHello(); // 需要是静态方法或静态属性访问方式</code></pre>
    <ul>
     <li id="u8378ba79">
      <strong>
       实例方法
      </strong>
      ：首先创建类的实例，然后通过该实例调用。
     </li>
    </ul>
    <pre id="UshcR"><code>ExampleClass example = new ExampleClass();
example.SayHello(); // 调用非静态方法
int result = example.Add(5, 3); // 调用非静态方法并获取返回值</code></pre>
    <h3 id="jw1ae" name="jw1ae">
     4. 参数传递方式
    </h3>
    <p id="u015b3c98">
     C#支持按值传递（默认）和按引用传递（使用
     <code>
      ref
     </code>
     或
     <code>
      out
     </code>
     关键字）。
    </p>
    <ul>
     <li id="u2f7f52de">
      <strong>
       按值传递
      </strong>
      ：传递参数的副本，对副本的修改不会影响原始参数。
     </li>
     <li id="ue9b73f5e">
      <strong>
       按引用传递
      </strong>
      ：直接传递参数的内存地址，对参数的修改会影响原始参数。
      <code>
       ref
      </code>
      用于修改传入的参数值，
      <code>
       out
      </code>
      用于在方法内部给参数赋值。
     </li>
    </ul>
    <h3 id="O3YVu" name="O3YVu">
     5. 方法重载（Overloading）
    </h3>
    <p id="u26e913de">
     同一个类中可以有多个同名方法，只要它们的参数列表不同（参数数量或类型不同）。这称为方法重载。
    </p>
    <pre id="MLlq8"><code>public void Show(int a) { }
public void Show(string text) { }</code></pre>
    <h3 id="zc31l" name="zc31l">
     6. 可选参数和命名参数（C# 4.0及以后）
    </h3>
    <p id="u686ce3c9">
     从C# 4.0开始，可以在方法定义时为参数提供默认值，使其成为可选参数。还可以在调用时使用命名参数。
    </p>
    <pre id="B8KY2"><code>public void Print(string message = "Default Message") { }
Print("Hello"); // 使用默认值和命名参数调用Print方法。</code></pre>
    <h3 id="qMyqT" name="qMyqT">
     7. 表达式体定义方法（C# 6.0及以后）
    </h3>
    <p id="uc85d16ea">
     从C# 6.0开始，可以使用表达式体来定义简单的只返回一个表达式结果的方法。
    </p>
    <pre id="cpgRR"><code>public int Multiply(int a, int b) =&gt; a * b; // 相当于 { return a * b; } 的简写形式。</code></pre>
    <h3 id="GdF6V" name="GdF6V">
     8. 异步方法（Async/Await）
    </h3>
    <p id="u1e61459a">
     C#支持异步编程模式，允许方法在等待操作完成时释放线程，使用
     <code>
      async
     </code>
     和
     <code>
      await
     </code>
     关键字。
    </p>
    <pre id="Z6Vq7"><code>public async Task&lt;int&gt; GetDataAsync() 
{ 
    // 使用await调用异步方法并获取结果 
    return await SomeAsyncMethod(); 
}</code></pre>
    <p id="u87c036d2">
     通过以上介绍，你应该对C#中的方法有了一个全面的理解。
    </p>
    <p id="u02e8257a">
     每种方法和概念都可以帮助你更有效地编写和组织代码。
    </p>

    <h2 id="A7G4a" name="A7G4a">
     八、构造器
    </h2>
    <p id="u17f75897">
     在C#中，构造器（Constructor）是一种特殊的方法，用于在创建对象时初始化对象。构造器与类同名，没有返回
    </p>
    <p id="uf6550e31">
     类型，主要用于分配内存并初始化对象。C#中有两种类型的构造器：实例构造器（用于创建对象实例）和静态构
    </p>
    <p id="udc420f8f">
     造器（用于初始化静态成员）。
    </p>
    <h3 id="r9Mz6" name="r9Mz6">
     1. 实例构造器
    </h3>
    <p id="ue7edc7f2">
     实例构造器用于创建类的实例，并初始化对象的状态。每个类可以有多个构造器，只要它们的参数列表不同。
    </p>
    <p id="u2587e6c6">
     <strong>
      示例
     </strong>
    </p>
    <pre id="KBAAt"><code>public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
 
    // 无参数构造器
    public Person()
    {
        Name = "Unknown";
        Age = 0;
    }
 
    // 带参数的构造器
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }
}</code></pre>
    <h4 id="CyLzw" name="CyLzw">
     1.1. 2. 静态构造器
    </h4>
    <p id="u1456655f">
     静态构造器用于初始化类的静态成员。静态构造器是类的一部分，而不是实例的一部分。静态构造器在类第一次
    </p>
    <p id="uc8b89245">
     被引用时自动调用，仅执行一次。
    </p>
    <p id="u7539a067">
     <strong>
      示例
     </strong>
    </p>
    <pre id="ySsAE"><code>public class DatabaseConnection
{
    private static string connectionString;
 
    // 静态构造器
    static DatabaseConnection()
    {
        connectionString = "Server=myServerAddress;Database=myDataBase;User Id=myUsername;Password=myPassword;";
    }
}</code></pre>
    <h3 id="dJQ7d" name="dJQ7d">
     2. 构造器的特性与使用场景
    </h3>
    <ul>
     <li id="u87b1679f">
      <strong>
       无参数构造器
      </strong>
      ：通常用于提供一个默认的初始化状态。如果类中没有定义任何构造器，编译器会自动提供一个无参数的默认构造器。一旦你定义了任何构造器（无论是带参数的还是无参数的），编译器就不会自动提供默认的无参数构造器。
     </li>
     <li id="ua0ca25bb">
      <strong>
       带参数的构造器
      </strong>
      ：允许在创建对象时传递不同的初始值。这增加了对象的灵活性，使得可以根据需要创建具有不同初始状态的多个对象实例。
     </li>
     <li id="u3cdeaed3">
      <strong>
       构造函数重载
      </strong>
      ：通过定义多个具有不同参数列表的构造器，可以实现构造函数重载。这允许根据提供的参数创建不同类型的对象实例。
     </li>
    </ul>
    <h3 id="Lh5l7" name="Lh5l7">
     3. 构造器的调用顺序
    </h3>
    <ul>
     <li id="u0e5db32a">
      <strong>
       父类构造器的调用
      </strong>
      ：在派生类的构造器中，必须首先调用基类的构造器（除非基类是无参数的默认构造器）。这通常通过使用
      <code>
       :base()
      </code>
      语法完成。例如：
      <code>
       public ChildClass(string name) : base(name)
      </code>
      。
     </li>
     <li id="u68f9a7fc">
      <strong>
       静态构造器的调用
      </strong>
      ：静态构造器在类第一次被引用时自动调用，用于初始化静态成员。它不能被直接调用，也不能带有访问修饰符或参数。
     </li>
    </ul>
    <h3 id="TlDoW" name="TlDoW" style="background-color:transparent">
     4. 构造函数链（Chaining）
    </h3>
    <p id="ud539fc6b">
     在派生类的构造函数中，可以使用
     <code>
      this()
     </code>
     来调用基类的另一个构造函数。例如：
    </p>
    <pre id="WrpRC"><code>public class ChildClass : BaseClass
{
    public ChildClass(string name) : base(name) // 调用基类的带参数的构造器
    {
        // 子类的初始化代码
    }
}</code></pre>
    <p id="ufb87c52e">
     通过以上介绍，你可以更好地理解C#中构造器的使用和它们的重要性。
    </p>
