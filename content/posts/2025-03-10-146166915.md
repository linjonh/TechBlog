    <h2>
     kafka
    </h2>
    <p>
     Apache Kafka 是一个分布式流处理平台，主要用于构建实时数据管道和流应用程序。它具有高吞吐量、低延迟、可扩展性和持久性等特点，广泛应用于日志收集、消息系统、事件溯源、流处理等场景。
    </p>
    <p>
     以下是 Kafka 的基础知识：
    </p>
    <h4>
     1.
     <strong>
      Kafka 的核心概念
     </strong>
    </h4>
    <h5>
     <strong>
      1.1 Topic（主题）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Topic
       </strong>
       是 Kafka 中
       <span style="color:#fe2c24">
        数据存储的逻辑分类
       </span>
       ，类似于数据库中的表。
      </p>
     </li>
     <li>
      <p>
       生产者将消息发布到特定的 Topic，消费者从 Topic 订阅并消费消息。
      </p>
     </li>
     <li>
      <p>
       一个 Kafka 集群可以有多个 Topic。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.2 Partition（分区）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       每个 Topic 可以分为多个
       <strong>
        Partition
       </strong>
       ，分区是 Kafka 实现高吞吐量和并行处理的基础。
      </p>
     </li>
     <li>
      <p>
       每个分区是一个有序的、不可变的消息队列。
      </p>
     </li>
     <li>
      <p>
       分区允许 Topic 的数据分布在多个服务器上，从而实现水平扩展。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.3 Producer（生产者）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Producer
       </strong>
       是向 Kafka Topic 发布消息的客户端。
      </p>
     </li>
     <li>
      <p>
       生产者可以将消息发送到指定的 Topic 和 Partition。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.4 Consumer（消费者）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Consumer
       </strong>
       是从 Kafka Topic 订阅并消费消息的客户端。
      </p>
     </li>
     <li>
      <p>
       消费者可以以组（Consumer Group）的形式工作，组内的消费者共同消费一个 Topic 的消息，实现负载均衡。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.5 Broker（代理）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Broker
       </strong>
       是 Kafka 集群中的单个服务器节点。
      </p>
     </li>
     <li>
      <p>
       每个 Broker 负责存储和管理一个或多个 Partition。
      </p>
     </li>
     <li>
      <p>
       多个 Broker 组成一个 Kafka 集群。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.6 Offset（偏移量）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Offset
       </strong>
       是消息在 Partition 中的唯一标识，表示消息的位置。
      </p>
     </li>
     <li>
      <p>
       消费者通过维护 Offset 来记录自己消费到的位置。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.7 Consumer Group（消费者组）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Consumer Group
       </strong>
       是一组消费者的集合，共同消费一个 Topic 的消息。
      </p>
     </li>
     <li>
      <p>
       Kafka 会将 Topic 的 Partition 分配给组内的消费者，确保每条消息只被组内的一个消费者消费。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      1.8 Replication（副本）
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       Kafka 通过
       <strong>
        Replication
       </strong>
       实现数据的高可用性。
      </p>
     </li>
     <li>
      <p>
       每个 Partition 可以有多个副本，其中一个副本是 Leader，负责处理读写请求，其他副本是 Follower，用于数据备份。
      </p>
     </li>
    </ul>
    <p>
    </p>
    <p>
    </p>
    <h4>
     2.
     <strong>
      Kafka 的架构
     </strong>
    </h4>
    <h5>
     <strong>
      2.1 生产者-消费者模型
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       生产者将消息发布到 Kafka Topic。
      </p>
     </li>
     <li>
      <p>
       消费者从 Topic 订阅并消费消息。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      2.2 分布式存储
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       Kafka 的数据分布在多个 Broker 上，每个 Broker 存储一部分 Partition。
      </p>
     </li>
     <li>
      <p>
       通过分区和副本机制，Kafka 实现了高可用性和负载均衡。
      </p>
     </li>
    </ul>
    <h5>
     <strong>
      2.3 ZooKeeper 的作用
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       Kafka 依赖 ZooKeeper 来管理集群元数据、Broker 状态、消费者 Offset 等。
      </p>
     </li>
     <li>
      <p>
       从 Kafka 2.8.0 开始，Kafka 引入了
       <strong>
        KRaft 模式
       </strong>
       ，可以不再依赖 ZooKeeper。
      </p>
     </li>
    </ul>
