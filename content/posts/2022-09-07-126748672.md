    <h2>
     <a id="_5">
     </a>
     一、态势感知简介
    </h2>
    <h3>
     <a id="1__8">
     </a>
     1. 概念
    </h3>
    <p>
     态势感知是一种基于环境的、动态、整体地洞悉安全风险的能力，是以
     <strong>
      安全大数据
     </strong>
     为基础，从全局视角提升对安全威胁的
     <strong>
      发现识别
     </strong>
     、
     <strong>
      理解分析
     </strong>
     、
     <strong>
      响应处置
     </strong>
     能力的一种方式，最终是为了决策与行动，是安全能力的落地。
    </p>
    <p>
     态势感知的概念最早在军事领域被提出，覆盖
     <strong>
      感知
     </strong>
     、
     <strong>
      理解
     </strong>
     和
     <strong>
      预测
     </strong>
     三个层次。并随着网络的兴起而升级为“网络态势感知（Cyberspace Situation Awareness，CSA）”。旨在大规模网络环境中对能够引起网络态势发生变化的安全要素进行获取、理解、显示以及最近发展趋势的顺延性预测，进而进行决策与行动。（
     <strong>
      获取、理解、显示、预测、响应
     </strong>
     ）
    </p>
    <p>
     态势感知（SA，Situational Awareness or Situation Awareness）是对一定时间和空间内的环境元素进行感知，并对这些元素的含义进行理解，最终预测这些元素在未来的发展状态。当前，大家提到“态势感知”时主要是指“
     <strong>
      网络安全态势感知
     </strong>
     ”，即将态势感知的相关理论和方法应用到网络安全领域中。网络安全态势感知可以使网络安全人员宏观把握整个网络的安全状态，识别出当前网络中存在的问题和异常活动，并作出相应的反馈或改进。通过对一段时间内的网络安全状况进行分析和预测，为高层决策提供有力支撑和参考。
    </p>
    <h3>
     <a id="2__17">
     </a>
     2. 形象举例
    </h3>
    <p>
     态势感知的概念比较抽象，我们举个例子来帮助理解：
     <strong>
      天气预报
     </strong>
     就可以理解为一种“态势感知”。通过对某一地点的持续观测和分析，我们可以预测未来一段时间内的天气。尤其是对重大灾害天气的预测，如台风、雾霾、暴雪等，对我们来讲尤为重要。通过提前进行人员和财产的转移，准备相关抗灾措施，可以大大降低灾害带来的影响，这就是进行“态势感知”的重要目的。
    </p>
    <h3>
     <a id="3__22">
     </a>
     3. 应具备的能力
    </h3>
    <p>
     网络安全态势感知系统应该具备的能力：
    </p>
    <ol>
     <li>
      网络空间安全持续
      <strong>
       监控
      </strong>
      能力，能够及时发现各种攻击威胁与异常；
     </li>
     <li>
      具备威胁调查
      <strong>
       分析及可视化
      </strong>
      能力，可以对威胁相关的影响范围、攻击路径、目的、手段进行快速判别，从而支撑有效的安全决策和响应；
     </li>
     <li>
      能够建立安全
      <strong>
       预警
      </strong>
      机制，来完善风险控制、应急响应和整体安全防护的水平。
     </li>
    </ol>
    <p>
     通俗来讲，态势感知从时间概念上可以被理解为：刚才发生了什么，现在应该做什么，接下来会发生什么，也就是态势感知的根本任务，了解昨天、思考今天，预测明天。
    </p>
    <hr/>
    <h2>
     <a id="_37">
     </a>
     二、为什么要态势感知？
    </h2>
    <p>
     态势感知的目的在于，
     <strong>
      全天候全方位
     </strong>
     地感知网络安全态势。知己知彼，才能百战不殆。没有意识到风险是最大的风险。网络安全具有很强的隐蔽性，一个技术漏洞、安全风险可能隐藏几年都发现不了，结果是“谁进来了不知道、是敌是友不知道、干了什么不知道”，长期“潜伏”在里面，一旦有事就发作了。
    </p>
    <p>
     现阶段面对传统安全防御体系失效的风险，态势感知能够
     <strong>
      全面
     </strong>
     感知网络安全威胁态势、洞悉网络及应用运行健康状态、通过全流量分析技术实现完整的网络攻击溯源取证，帮助安全人员采取针对性响应处置措施。
    </p>
    <h3>
     <a id="_44">
     </a>
     为什么网络安全态势感知很重要？
    </h3>
    <p>
     随着网络与信息技术的不断发展，人们的安全意识在逐步提高。我们已经不再笃定认为自己的网络是绝对安全的，相反的，我们认为网络遭受攻击是必然的、常态化的。我们不能阻止攻击行为，但是可以提前识别和发现攻击行为，尽可能降低损失。也就是说，
     <strong>
      安全防护思想
     </strong>
     已经从过去的
     <strong>
      被动防御
     </strong>
     向
     <strong>
      主动防护
     </strong>
     和
     <strong>
      智能防护
     </strong>
     转变。
    </p>
    <p>
     同时，物联网和云技术的发展也是日新月异，很多颠覆性的新技术也引入了新的安全问题。例如海量终端接入、传统的网络边界消失、网络攻击的隐蔽性和复杂度大大增强等，这都为我们提出了新的挑战，也对网络安全人员的能力也提出了更高的要求。
    </p>
    <p>
     正是在这样的背景下，以网络安全态势感知技术为核心的产品和解决方案得到快速发展。网络安全态势感知技术可以带动整个安全防护体系升级，实现以下三个方面的转变：
    </p>
    <ol>
     <li>
      安全建设的目标从满足合规转变为增强防御和威慑能力，并且更加注重对抗性，这对情报技术提出了更高要求。
     </li>
     <li>
      攻击检测的对象从已知威胁转变为
      <strong>
       未知威胁
      </strong>
      ，通过大数据分析、异常检测、态势感知、机器学习等技术，实现对高级威胁的检测。
     </li>
     <li>
      对威胁的响应从人工分析并处置转变为自动响应闭环，强调应急响应、协同联动，实现安全弹性。
     </li>
    </ol>

     三、态势感知系统的功能
    </h2>
    <ol>
     <li>
      <strong>
       检测
      </strong>
      ：提供网络安全持续监控能力，及时发现各种攻击威胁与异常，特别是针对性攻击。
     </li>
     <li>
      <strong>
       分析、响应
      </strong>
      ：建立威胁可视化及分析能力，对威胁的影响范围、攻击路径、目的、手段进行快速研判，目的是有效的安全决策和响应。
     </li>
     <li>
      <strong>
       预测、预防
      </strong>
      ：建立风险通报和威胁预警机制，全面掌握攻击者目的、技术、攻击工具等信息。
     </li>
     <li>
      <strong>
       防御
      </strong>
      ：利用掌握的攻击者相关目的、技术、攻击工具等情报，完善防御体系。
     </li>
    </ol>

    </p>

      </strong>

     </li>
     <li>
      检测：提供网络安全持续监控能力，
      <strong>
       快速、精准地检测

     </li>

      <strong>

     </li>
     <li>

     </li>
    </ul>

    <h2>
     <a id="_87">
     </a>
     五、什么是态势感知的三个层级
    </h2>
    <p>
     Mica Endsley在“Toward a theory of situation awareness in dynamic systems”（1995）中，仿照人的认知过程提出了一个经典的态势感知模型。这个模型在当前看来虽然比较简单，但却是很多后续理论的基础，人们一般称该模型为Endsley模型（Endsley’s model）。
    </p>
    <p>
     <strong>
      Endsley模型
     </strong>
     将态势感知分为
     <strong>
      三个层级
     </strong>
     ，分别是态势要素感知、态势理解和态势预测。
    </p>
    <ol>
     <li>
      <strong>
       要素感知
      </strong>
      （Level 1）：感知环境中相关要素的状态、属性和动态等信息。
     </li>
     <li>
      <strong>
       态势理解
      </strong>
      （Level 2）：通过识别、解读和评估的过程，将不相关的要素信息联系起来，并关注这些信息对预期目标的影响。
     </li>
     <li>
      <strong>
       态势预测
      </strong>
      （Level 3）：基于对前两级信息的理解，预测未来的发展态势和可能产生的影响。
     </li>
    </ol>

    </p>

    <ol>
     <li>
      态势感知是基于大数据的，数据的丰富程度很重要，信息的收集和整合。
      <strong>
       海量数据的收集
      </strong>
      。需要多数据源的支持。（各种公开数据库（安全事件、日志、流量、漏洞信息、威胁情报），实时数据，情报共享，数据联合）
     </li>
     <li>
      将海量的信息统一到
      <strong>
       单一的管理视图
      </strong>
      当中，新发生的事件也可以加入进来（攻击是不断发展的，实时更新）。通过对多方面数据的实时观测，最大程度减少攻击噪音（减少误报）。
     </li>
     <li>
      业务驱动优先，将
      <strong>
       重要资产
      </strong>
      上的活动的重要级提升上来。
     </li>
     <li>
      信息数据的有效使用。
     </li>
     <li>
      大数据分析需要机器学习技术的支持。（关联分析）认知分析、机器学习帮助我们极大的压缩威胁发现的时间，能够快速识别异常。
     </li>
     <li>
      报告警告的时机很重要。我们要尽量早地发现异常，但报警太早可能造成很多误报。需要持续监测，如果接下来还发生了异常事件就发出警报。
     </li>
    </ol>
