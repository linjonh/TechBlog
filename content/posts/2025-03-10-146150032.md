    <h2>
     <a id="_0">
     </a>
     一、引言
    </h2>
    <p>
     本指导文档旨在帮助测试人员理解和使用自动化测试用例执行框架。该框架通过一个简单而灵活的 CSV 配置文件定义测试步骤，结合 UI 识别和操作技术，实现自动化测试的高效执行。测试人员无需编写复杂的代码，只需通过 CSV 文件配置测试步骤，即可完成对 UI 界面的自动化操作。
    </p>
    <h4>
     <a id="_4">
     </a>
     <strong>
      本文档将基于以下测试用例示例进行说明：
     </strong>
    </h4>
    <pre><code class="prism language-Plain">type,action,value
文本,点击,用户名
文本,点击,密码
图片,点击,登录
</code></pre>
    <hr/>
    <h2>
     <a id="_15">
     </a>
     二、测试用例结构
    </h2>
    <h4>
     <a id="1__CSV__17">
     </a>
     1. 测试用例以 CSV 文件的形式存储，每一行代表一个测试步骤，包含以下三个字段：
    </h4>
    <ul>
     <li>
      <p>
       <code>
        type：
       </code>
       识别类型，表示如何识别 UI 元素。支持以下两种类型：
      </p>
      <ul>
       <li>
        <p>
         “文本”：通过 OCR 技术识别屏幕上的文本。
        </p>
       </li>
       <li>
        <p>
         “图片”：通过图像匹配技术识别屏幕上的图片。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <code>
        action：
       </code>
       操作动作，目前仅支持 “点击”，表示点击识别到的 UI 元素。
      </p>
     </li>
     <li>
      <p>
       <code>
        value：
       </code>
       元素值，根据 type 的不同有不同的含义：
      </p>
      <ul>
       <li>
        <p>
         当 type 为 “文本” 时，value 是要识别并点击的文本内容（例如 “用户名”）。
        </p>
       </li>
       <li>
        <p>
         当 type 为 “图片” 时，value 是图片的关键字（例如 “登录”），用于索引对应的图片路径。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2__35">
     </a>
     2. 编写测试用例
    </h4>
    <ul>
     <li>
      <strong>
       测试用例 CSV 文件的格式如下：
      </strong>
     </li>
    </ul>
    <pre><code class="prism language-Plain">type,action,value
文本,点击,用户名
文本,点击,密码
图片,点击,登录
</code></pre>
    <ul>
     <li>
      <strong>
       字段说明
      </strong>
     </li>
    </ul>
    <ol>
     <li>
      <p>
       对于
       <code>
        type="文本"
       </code>
       ：
      </p>
      <ul>
       <li>
        <p>
         value 是屏幕上需要识别并点击的文本内容。例如，“用户名” 表示系统将在屏幕上寻找包含 “用户名” 的文本区域，并点击其中心点。
        </p>
       </li>
       <li>
        <p>
         系统使用 OCR 技术完成文本识别。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       对于
       <code>
        type="图片"
       </code>
       ：
      </p>
      <ul>
       <li>
        value 是图片的关键字，例如 “登录”。系统通过该关键字查找预配置的图片路径，然后使用图像匹配技术在屏幕上识别该图片，并点击其中心点。
       </li>
      </ul>
     </li>
    </ol>
    <ul>
     <li>
      <p>
       <strong>
        注意事项
       </strong>
      </p>
      <ul>
       <li>
        <p>
         确保
         <code>
          CSV文件格式
         </code>
         正确，字段之间用逗号分隔，无多余空格。
        </p>
       </li>
       <li>
        <p>
         文本和图片的识别依赖屏幕截图的质量和
         <code>
          UI元素
         </code>
         的清晰度。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3__66">
     </a>
     3. 配置图片路径
    </h4>
    <p>
     当 type=“图片” 时，系统需要通过 value（图片关键字）查找对应的图片路径。因此，需要预先配置一个图片路径索引文件（建议使用 YAML 或 JSON 格式），以便系统正确映射关键字到图片路径。
    </p>
    <p>
     <strong>
      示例配置（YAML 格式）yaml
     </strong>
    </p>
    <pre><code class="prism language-YAML">登录: "icons/login_button.png"
</code></pre>
    <ul>
     <li>
      <p>
       在测试用例中，value=“登录” 会映射到图片路径 “icons/login_button.png”。
      </p>
     </li>
     <li>
      <p>
       系统使用该图片进行图像匹配，识别屏幕上的 “登录” 按钮并点击。
      </p>
     </li>
    </ul>
    <p>
     <strong>
      配置步骤
     </strong>
    </p>
    <ol>
     <li>
      <p>
       创建一个配置文件（例如 icon_config.yaml），定义关键字和图片路径的映射。
      </p>
     </li>
     <li>
      <p>
       在测试框架的主配置文件中指定该图片配置文件的路径，以便系统加载。
      </p>
     </li>
    </ol>
    <p>
     <strong>
      注意事项
     </strong>
    </p>
    <ul>
     <li>
      <p>
       确保图片路径准确且图片文件存在。
      </p>
     </li>
     <li>
      <p>
       图片应清晰，尽量与实际屏幕上的 UI 元素一致，以提高匹配成功率。
      </p>
     </li>
    </ul>
