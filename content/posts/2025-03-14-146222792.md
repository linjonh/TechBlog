    <h2>
     Props 校验
    </h2>
    <p>
     vue组件可以更细致地声明对传入的 props 的校验要求。比如类型声明，如果传入的值不满足类型要求，Vue会在浏览器控制台中抛出警告来提醒使用者。这在开发给其他开发者使用的组件时非常有用。
    </p>
    <p>
     要声明对 props 的校验，可以向 props 选项提供一个带有 props 校验选项的对象，例如：
    </p>
    <h4>
     1 基础类型检查
    </h4>
    <p>
     子组件：
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//基础类型检查
			//(给出 'null' 和 'undefined' 值则会跳过任何类型检查)
			aProp:Number
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <p>
     父组件：
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;Sub aProp="3"&gt;&lt;/Sub&gt;
&lt;/template&gt;

&lt;script&gt;
	import Sub from './Sub.vue'
	export default {
		components: {
			Sub,
		},
		data() {
			return {
				
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;

&lt;/style&gt;</code></pre>
    <p>
     结果：
    </p>
    <p>
     报错 [Vue warn]: Invalid prop: type check failed for prop "aProp". Expected Number with value 3, got String with value "3".
    </p>
    <p>
     显示还是会显示
    </p>
    <p>
     <img alt="" height="347" src="https://i-blog.csdnimg.cn/direct/89de47cb11e642df9afbd92940096e0d.png" width="321"/>
    </p>
    <h4>
     2 多种可能的类型
    </h4>
    <p>
     上面的例子，把子组件代码改成这样，就不会收到警告了，
    </p>
    <pre><code class="language-javascript">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//多种可能类型
			aProp:[Number,String]
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <h4>
     3 必传，且为 String 类型
    </h4>
    <p>
     子组件改成这样，
    </p>
    <pre><code class="language-javascript">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//必传，且为 String 类型
			aProp:{
				type:String,
				required:true
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <p>
     父组件不传，报错 Missing required prop: "aProp"
    </p>
    <p>
     父组件传数字类型，也会报错，Invalid prop: type check failed for prop "aProp". Expected String with value "3", got Number with value 3.
    </p>
    <pre><code class="language-html">注意带冒号才算传的数值类型3，不带冒号则表示传的是字符串3
&lt;template&gt;
	&lt;Sub :aProp="3"&gt;&lt;/Sub&gt;
&lt;/template&gt;

&lt;script&gt;
	import Sub from './Sub.vue'
	export default {
		components: {
			Sub,
		},
		data() {
			return {
				
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;

&lt;/style&gt;</code></pre>
    <h4>
     4 必传但可为 null 的字符串
    </h4>
    <p>
     上面3中如果传null，是会报错的，改成这样就不会，
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//必传但可为null的字符串
			aProp:{
				type:[String,null],
				required:true
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <p>
     父组件：
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;Sub :aProp="null"&gt;&lt;/Sub&gt;
&lt;/template&gt;

&lt;script&gt;
	import Sub from './Sub.vue'
	export default {
		components: {
			Sub,
		},
		data() {
			return {
				
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;

&lt;/style&gt;</code></pre>
    <h4>
     5 Number 类型的默认值
    </h4>
    <pre><code class="language-html">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//必传但可为null的字符串
			aProp:{
				type:Number,
				default:101
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <h4>
     6 对象类型的默认值
    </h4>
    <p>
     子组件：
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;text&gt;{<!-- -->{aProp.message}}&lt;/text&gt;
&lt;/template&gt;

&lt;script&gt;
	export default {
		props:{
			//对象类型的默认值
			aProp:{
				type: Object,
				// 对象或者数组应当用工厂函数返回
				// 工厂函数会收到组件所接收的原始 props 作为参数
				default(rawProps) {
					return {message:'子类默认'}
				}
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;
&lt;/style&gt;</code></pre>
    <p>
     父组件：
    </p>
    <pre><code class="language-html">&lt;template&gt;
	&lt;Sub :a-prop="{message:'父组件传入'}"&gt;&lt;/Sub&gt;
&lt;/template&gt;

&lt;script&gt;
	import Sub from './Sub.vue'
	export default {
		components: {
			Sub,
		},
		data() {
			return {
				
			}
		}
	}
&lt;/script&gt;

&lt;style&gt;

&lt;/style&gt;</code></pre>

    <h4>
     7 自定义类型校验函数(不常用)
    </h4>
    <pre><code class="language-javascript">// 自定义类型校验函数
    // 完整的 props 作为第二个参数传入
    propG: {
      validator(value, props) {
        return ['success', 'warning', 'danger'].includes(value)
      }
    },</code></pre>
    <h2>
     一些补充细节：
    </h2>
    <ul>
     <li>
      <p>
       所有 prop 默认都是可选的，除非声明了 required: true
      </p>
     </li>
     <li>
      <p>
       除 Boolean 外的未传递的可选 prop 将会有一个默认值 undefined
      </p>
     </li>
     <li>
      <p>
       Boolean 类型的未传递 prop 将被转换为 false。这可以通过为它设置 default 来更改——例如，设置为 default：undefined 将与非布尔类型的 prop 的行为保持一致
      </p>
     </li>
     <li>
      <p>
       如果声明了 default 值，那么在 prop 的值被解析为 undefined 时，无论 prop 是未被传递还是显式指明的 undefined，都会改为 default 值
      </p>
     </li>
    </ul>
    <p>
     需要注意的是，prop 的校验是在组件实例被创建之前，所以实例的属性比如 data、computed 等将在 default 或 validator 函数中不可用。
    </p>
