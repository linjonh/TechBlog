    <p>
     MyBatis 的核心配置文件（通常命名为
     <code>
      mybatis-config.xml
     </code>
     ）是 MyBatis 应用程序的入口点，它定义了 MyBatis 的全局配置信息 。
    </p>
    <p>
     <strong>
      核心配置文件的作用：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       配置 MyBatis 的运行时行为:
      </strong>
      通过
      <code>
       &lt;settings&gt;
      </code>
      标签设置全局参数，控制 MyBatis 的各种行为，例如是否开启缓存、是否使用延迟加载、是否自动映射等。
     </li>
     <li>
      <strong>
       配置数据源和事务管理器:
      </strong>
      通过
      <code>
       &lt;environments&gt;
      </code>
      标签配置数据库连接信息和事务管理方式。
     </li>
     <li>
      <strong>
       注册类型别名:
      </strong>
      通过
      <code>
       &lt;typeAliases&gt;
      </code>
      标签为 Java 类型定义别名，简化 XML 映射文件中的类型引用。
     </li>
     <li>
      <strong>
       注册类型处理器:
      </strong>
      通过
      <code>
       &lt;typeHandlers&gt;
      </code>
      标签注册自定义的类型处理器，处理 Java 类型与 JDBC 类型之间的转换。
     </li>
     <li>
      <strong>
       注册对象工厂:
      </strong>
      通过
      <code>
       &lt;objectFactory&gt;
      </code>
      标签注册自定义的对象工厂，用于创建结果对象。
     </li>
     <li>
      <strong>
       注册插件:
      </strong>
      通过
      <code>
       &lt;plugins&gt;
      </code>
      标签注册插件，拦截 MyBatis 的核心执行流程。
     </li>
     <li>
      <strong>
       加载映射器:
      </strong>
      通过
      <code>
       &lt;mappers&gt;
      </code>
      标签加载 SQL 映射文件或 Mapper 接口，将 SQL 语句与 Java 代码关联起来。
     </li>
    </ol>

    <p>
     <strong>
      各部分配置详解：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        &lt;configuration&gt;
       </code>
       :
      </strong>
      根元素，所有配置都包含在这个元素中。
     </li>
     <li>
      <strong>
       <code>
        &lt;properties&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        用于加载外部属性文件（例如
        <code>
         db.properties
        </code>
        ），可以在配置文件中引用属性文件中的值。
       </li>
       <li>
        <code>
         resource
        </code>
        属性指定属性文件的路径。
       </li>
       <li>
        也可以直接在
        <code>
         &lt;properties&gt;
        </code>
        标签内部定义属性。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;settings&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        用于配置 MyBatis 的全局设置，控制 MyBatis 的运行时行为。
       </li>
       <li>
        常用的设置包括：
        <ul>
         <li>
          <code>
           cacheEnabled
          </code>
          : 是否开启二级缓存 (默认
          <code>
           true
          </code>
          )。
         </li>
         <li>
          <code>
           lazyLoadingEnabled
          </code>
          : 是否开启延迟加载 (默认
          <code>
           false
          </code>
          )。
         </li>
         <li>
          <code>
           aggressiveLazyLoading
          </code>
          : 是否开启积极的延迟加载 (默认
          <code>
           false
          </code>
          ，MyBatis 3.4.1 及之前版本默认为
          <code>
           true
          </code>
          )。
         </li>
         <li>
          <code>
           multipleResultSetsEnabled
          </code>
          : 是否允许单个语句返回多个结果集 (默认
          <code>
           true
          </code>
          )。
         </li>
         <li>
          <code>
           useColumnLabel
          </code>
          : 是否使用列标签代替列名 (默认
          <code>
           true
          </code>
          )。
         </li>
         <li>
          <code>
           useGeneratedKeys
          </code>
          : 是否允许 JDBC 支持自动生成主键 (默认
          <code>
           false
          </code>
          )。
         </li>
         <li>
          <code>
           autoMappingBehavior
          </code>
          : 自动映射的行为 (默认
          <code>
           PARTIAL
          </code>
          ，可选值：
          <code>
           NONE
          </code>
          、
          <code>
           PARTIAL
          </code>
          、
          <code>
           FULL
          </code>
          )。
         </li>
         <li>
          <code>
           defaultExecutorType
          </code>
          : 默认的执行器类型 (默认
          <code>
           SIMPLE
          </code>
          ，可选值：
          <code>
           SIMPLE
          </code>
          、
          <code>
           REUSE
          </code>
          、
          <code>
           BATCH
          </code>
          )。
         </li>
         <li>
          <code>
           mapUnderscoreToCamelCase
          </code>
          : 是否开启驼峰命名自动映射 (默认
          <code>
           false
          </code>
          )。
         </li>
         <li>
          <code>
           localCacheScope
          </code>
          : 一级缓存的作用域（默认
          <code>
           SESSION
          </code>
          ，可选值
          <code>
           SESSION
          </code>
          或
          <code>
           STATEMENT
          </code>
          ）
         </li>
         <li>
          <code>
           jdbcTypeForNull
          </code>
          : 当传入的参数为
          <code>
           null
          </code>
          时，使用的 JDBC 类型（默认
          <code>
           OTHER
          </code>
          ，通常设置为
          <code>
           NULL
          </code>
          ）。
         </li>
         <li>
          <code>
           defaultStatementTimeout
          </code>
          : 默认的语句超时时间（秒）。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;typeAliases&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        用于为 Java 类型定义别名，简化 XML 映射文件中的类型引用。
       </li>
       <li>
        <code>
         typeAlias
        </code>
        标签定义单个别名，
        <code>
         package
        </code>
        标签扫描指定包下的所有类，并自动生成别名（类名首字母小写）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;typeHandlers&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        用于注册自定义的类型处理器。
       </li>
       <li>
        <code>
         typeHandler
        </code>
        标签注册单个类型处理器，
        <code>
         package
        </code>
        标签扫描指定包下的所有类型处理器。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;objectFactory&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        注册一个自定义对象工厂,MyBatis 每次创建结果对象的新实例时,它都会使用一个对象工厂（ObjectFactory）实例来完成。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;plugins&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        用于注册插件，拦截 MyBatis 的核心执行流程（例如，拦截 SQL 执行、参数设置、结果映射等）。
       </li>
       <li>
        <code>
         plugin
        </code>
        标签注册单个插件，
        <code>
         interceptor
        </code>
        属性指定插件类的全限定名。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;environments&gt;
       </code>
       (必须):
      </strong>
      <ul>
       <li>
        用于配置数据源和事务管理器。
       </li>
       <li>
        <code>
         default
        </code>
        属性指定默认的环境 ID。
       </li>
       <li>
        <code>
         &lt;environment&gt;
        </code>
        标签定义一个环境，
        <code>
         id
        </code>
        属性是环境的唯一标识符。
       </li>
       <li>
        <code>
         &lt;transactionManager&gt;
        </code>
        标签配置事务管理器：
        <ul>
         <li>
          <code>
           type="JDBC"
          </code>
          : 使用 JDBC 的事务管理机制（需要手动提交或回滚事务）。
         </li>
         <li>
          <code>
           type="MANAGED"
          </code>
          : 将事务管理委托给容器（例如 Spring 或 Java EE 容器）。
         </li>
        </ul>
       </li>
       <li>
        <code>
         &lt;dataSource&gt;
        </code>
        标签配置数据源：
        <ul>
         <li>
          <code>
           type="POOLED"
          </code>
          : 使用 MyBatis 内置的连接池。
         </li>
         <li>
          <code>
           type="UNPOOLED"
          </code>
          : 不使用连接池，每次都创建新的连接。
         </li>
         <li>
          <code>
           type="JNDI"
          </code>
          : 使用 JNDI 数据源。
         </li>
         <li>
          <code>
           &lt;property&gt;
          </code>
          标签配置数据源的属性，例如驱动类名、URL、用户名、密码等。
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;databaseIdProvider&gt;
       </code>
       (可选):
      </strong>
      <ul>
       <li>
        MyBatis 可以根据不同的数据库厂商执行不同的 SQL 语句
       </li>
      </ul>
     </li>
     <li>
      <strong>
       <code>
        &lt;mappers&gt;
       </code>
       (必须):
      </strong>
      <ul>
       <li>
        用于加载 SQL 映射文件或 Mapper 接口。
       </li>
       <li>
        <code>
         &lt;mapper resource="..."/&gt;
        </code>
        : 加载 XML 映射文件。
       </li>
       <li>
        <code>
         &lt;mapper class="..."/&gt;
        </code>
        : 加载 Mapper 接口（使用注解配置 SQL）。
       </li>
       <li>
        <code>
         &lt;package name="..."/&gt;
        </code>
        : 扫描指定包下的所有 Mapper 接口或 XML 映射文件（XML 文件需要与 Mapper 接口在同一个包下，且文件名与接口名相同）。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      必须配置的元素：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       <code>
        &lt;environments&gt;
       </code>
       :
      </strong>
      必须配置至少一个数据源环境，否则 MyBatis 无法连接数据库。
     </li>
     <li>
      <strong>
       <code>
        &lt;mappers&gt;
       </code>
       :
      </strong>
      必须加载至少一个 SQL 映射文件或 Mapper 接口，否则 MyBatis 无法执行 SQL 语句。
     </li>
    </ul>
    <p>
     <strong>
      可选配置的元素：
     </strong>
    </p>
    <ul>
     <li>
      <code>
       &lt;properties&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;settings&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;typeAliases&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;typeHandlers&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;objectFactory&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;plugins&gt;
      </code>
     </li>
     <li>
      <code>
       &lt;databaseIdProvider&gt;
      </code>
     </li>
    </ul>
