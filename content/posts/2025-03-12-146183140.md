    <h2>
     11. 关键字和一般标识符有什么不同？
    </h2>
    <blockquote>
     <p>
      C语言中关键字与一般标识符区别：
     </p>
     <ol>
      <li>
       <p>
        <strong>
         定义
        </strong>
        ：关键字是C语言预定义的特殊单词（如
        <code>
         int
        </code>
        、
        <code>
         for
        </code>
        ），有固定含义；标识符是自定义的名称（如变量名、函数名）。
       </p>
      </li>
      <li>
       <p>
        <strong>
         用途
        </strong>
        ：关键字用于语法结构，控制程序逻辑；标识符用于命名变量、函数等对象。
       </p>
      </li>
      <li>
       <p>
        <strong>
         限制
        </strong>
        ：关键字不可用作自定义名称，固定不变；标识符可自由命名，但须遵循规则（如字母开头、数字字母组合）。
       </p>
      </li>
     </ol>
     <p>
      <strong>
       总结
      </strong>
      ：关键字是内置语法元素，标识符是用户定义符号。
     </p>
    </blockquote>
    <h2>
     12. if语句中的条件表达式可以是任意的合法表达式吗？Switch语句中break的作用是什么？
    </h2>
    <blockquote>
     <ol>
      <li>
       <p>
        <strong>
         f语句中的条件表达式可以是任意合法表达式吗？
        </strong>
       </p>
       <ul>
        <li>
         <p>
          是的，if条件可以是任意合法表达式，结果被视为真（非0）或假（0）。例如，
          <code>
           if(1)
          </code>
          、
          <code>
           if(x &gt; 0)
          </code>
          、
          <code>
           if(a &amp;&amp; b)
          </code>
          都有效。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         Switch语句中break的作用是什么？
        </strong>
       </p>
       <ul>
        <li>
         <p>
          <code>
           break
          </code>
          用于跳出switch语句，防止程序继续执行后续case分支。若无break，会发生“贯穿”（fall-through），执行所有后续代码直到结束或遇到break。
         </p>
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
    <h2>
     13.三种循环结构中的条件是循环进行的条件还是循环结束的条件？循环结构中break语句和continue语句的作用是什么，二者有何区别？
    </h2>
    <blockquote>
     <ol>
      <li>
       <p>
        <strong>
         三种循环结构中的条件是循环进行的条件还是循环结束的条件？
        </strong>
       </p>
       <ul>
        <li>
         <p>
          <strong>
           while
          </strong>
          ：条件是循环进行的条件，真（非0）则继续，假（0）则结束。
         </p>
        </li>
        <li>
         <p>
          <strong>
           do-while
          </strong>
          ：同while，条件为继续执行的依据，假则停止，但至少执行一次。
         </p>
        </li>
        <li>
         <p>
          <strong>
           for
          </strong>
          ：条件是循环进行的条件，真则进入循环体，假则终止。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         循环结构中break和continue的作用及区别？
        </strong>
       </p>
       <ul>
        <li>
         <p>
          <strong>
           break
          </strong>
          ：立即跳出整个循环，结束循环执行，转到循环后代码。
         </p>
        </li>
        <li>
         <p>
          <strong>
           continue
          </strong>
          ：跳过本次循环剩余部分，直接进入下一次循环。
         </p>
        </li>
        <li>
         <p>
          <strong>
           区别
          </strong>
          ：break终止循环，continue只跳过当前轮次，继续后续迭代。
         </p>
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
    <h2>
     14.
     <strong>
      字符数组与字符串是否相同？
     </strong>
    </h2>
    <blockquote>
     <ul>
      <li>
       <p>
        不完全相同，二者有区别。
       </p>
      </li>
     </ul>
     <p>
      <strong>
       区别
      </strong>
      ：
     </p>
     <ol>
      <li>
       <p>
        <strong>
         定义
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           字符数组
          </strong>
          ：存储字符的数组，如
          <code>
           char arr[5] = {'a', 'b', 'c'}
          </code>
          ，无需
          <code>
           \0
          </code>
          。
         </p>
        </li>
        <li>
         <p>
          <strong>
           字符串
          </strong>
          ：以
          <code>
           \0
          </code>
          结尾的字符数组，如
          <code>
           char str[] = "abc"
          </code>
          （含
          <code>
           \0
          </code>
          ）。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         结束标志
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           字符数组
          </strong>
          ：没有强制要求结束符，可存任意字符。
         </p>
        </li>
        <li>
         <p>
          <strong>
           字符串
          </strong>
          ：必须以
          <code>
           \0
          </code>
          结束，表示字符串终止。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         使用
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          <strong>
           字符数组
          </strong>
          ：可作为普通数组操作，不一定能用字符串函数。
         </p>
        </li>
        <li>
         <p>
          <strong>
           字符串
          </strong>
          ：专为字符串函数（如
          <code>
           strlen
          </code>
          、
          <code>
           strcpy
          </code>
          ）设计，依赖
          <code>
           \0
          </code>
          。
         </p>
        </li>
       </ul>
      </li>
     </ol>
    </blockquote>
    <h2>
     15. 函数的嵌套调用与递归调用有什么区别？
    </h2>
    <blockquote>
     <p>
      <strong>
       总结
      </strong>
      ：嵌套是多函数层级调用，递归是自我重复调用，机制和目的不同。
     </p>
     <ul>
      <li>
       <p>
        嵌套调用示例：main() -&gt; func1() -&gt; func2()，执行完逐层返回。
       </p>
      </li>
      <li>
       <p>
        递归调用示例：factorial(n)调用factorial(n-1)，需栈管理，易溢出。
       </p>
      </li>
     </ul>
    </blockquote>
    <h2>
     16. 结构体类型与我们前面学过的基本类型有哪些区别？
    </h2>
    <blockquote>
     <p>
      <strong>
       结构体类型与基本类型的区别：
      </strong>
     </p>
     <p>
      <strong>
       从定义上看
      </strong>
      ，基本类型是C语言内置单一数据类型，如int、char、float，而结构体类型是用户自定义复合类型，用struct组合多种数据，如struct {int a; char b;}；
     </p>
     <p>
      <strong>
       从数据结构上看
      </strong>
      ，基本类型存储单一值，结构体类型含多个成员存储一组相关数据；
     </p>
     <p>
      <strong>
       从内存分配上看
      </strong>
      ，基本类型大小固定，如int占4字节，结构体类型大小由成员总和决定并可能有内存对齐；
     </p>
     <p>
      <strong>
       从使用方式上看
      </strong>
      ，基本类型直接操作，如int x = 5，结构体类型通过.或-&gt;访问成员，如struct S s; s.a = 5。
     </p>
    </blockquote>
    <h2>
     17. 指针与地址有什么联系？在使用指针的过程中应注意哪些细节？指针类型对于程序设计有哪些意义？
    </h2>
    <blockquote>
     <h5>
      1. 指针与地址的联系
     </h5>
     <ul>
      <li>
       <p>
        <strong>
         地址
        </strong>
        是内存中某个变量的具体存储位置。
       </p>
      </li>
      <li>
       <p>
        <strong>
         指针
        </strong>
        是一个变量，它的值是另一个变量的地址。简单来说，指针存储的是地址，指向内存中的某个位置。
       </p>
      </li>
     </ul>
     <h5>
      2. 使用指针时应注意的细节
     </h5>
     <p>
      在使用指针时，需要特别注意以下几点，以确保程序的正确性和安全性：
     </p>
     <ul>
      <li>
       <p>
        <strong>
         初始化
        </strong>
        ：指针在使用前必须初始化，否则可能指向随机内存地址，导致程序崩溃或数据损坏。
       </p>
      </li>
      <li>
       <p>
        <strong>
         解引用
        </strong>
        ：通过解引用操作符（
        <code>
         *
        </code>
        ）访问指针指向的内存时，必须确保指针指向的是有效地址。
       </p>
      </li>
      <li>
       <p>
        <strong>
         类型
        </strong>
        ：指针有类型（如
        <code>
         int*
        </code>
        、
        <code>
         char*
        </code>
        ），它决定了所指向变量的类型，影响解引用时数据的解释。
       </p>
      </li>
      <li>
       <p>
        <strong>
         指针算术
        </strong>
        ：可以对指针进行加减运算（如
        <code>
         +1
        </code>
        、
        <code>
         -1
        </code>
        ），移动到相邻内存单元，但要避免超出分配的内存范围。
       </p>
      </li>
      <li>
       <p>
        <strong>
         NULL指针
        </strong>
        ：将指针设为
        <code>
         NULL
        </code>
        表示它不指向任何有效地址，可避免意外解引用。
       </p>
      </li>
     </ul>
     <h5>
      3. 指针类型对程序设计的意义
     </h5>
     <p>
      指针类型在程序设计中具有重要作用，具体体现在以下几个方面：
     </p>
     <ul>
      <li>
       <p>
        <strong>
         动态内存分配
        </strong>
        ：通过
        <code>
         malloc
        </code>
        或
        <code>
         calloc
        </code>
        等函数，指针可以在运行时分配内存，适合处理大小不确定的数据。
       </p>
      </li>
      <li>
       <p>
        <strong>
         函数参数传递
        </strong>
        ：指针实现按引用传递，使函数能够修改调用者的变量。
       </p>
      </li>
      <li>
       <p>
        <strong>
         数据结构
        </strong>
        ：链表、树、图等复杂数据结构依赖指针连接各个节点。
       </p>
      </li>
      <li>
       <p>
        <strong>
         数组和字符串
        </strong>
        ：在C语言中，指针常用于操作数组和字符串，提供灵活性。
       </p>
      </li>
      <li>
       <p>
        <strong>
         硬件访问
        </strong>
        ：在底层编程中，指针可直接访问硬件寄存器或内存映射设备。
       </p>
      </li>
     </ul>
     <h4>
      总结
     </h4>
     <p>
      指针是地址的载体，使用时需关注初始化、解引用和类型等细节。指针类型赋予程序动态性、复杂性和底层操作能力，是C语言中强大且关键的特性。
     </p>
    </blockquote>
    <h2>
     18. 指针 VS 引用
    </h2>
    <blockquote>
     <ul>
      <li>
       <p>
        <strong>
         指针
        </strong>
       </p>
       <ul>
        <li>
         <p>
          存储变量的内存地址，可初始化为
          <code>
           nullptr
          </code>
          ，可重新指向其他对象
         </p>
        </li>
        <li>
         <p>
          通过
          <code>
           *
          </code>
          解引用访问数据，支持指针算术（如
          <code>
           ptr+1
          </code>
          移动到下一元素）
         </p>
        </li>
        <li>
         <p>
          用途：动态内存分配（
          <code>
           new
          </code>
          /
          <code>
           delete
          </code>
          ）、数组操作、实现多态（基类指针指向派生类）
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         引用
        </strong>
       </p>
       <ul>
        <li>
         <p>
          已存在变量的别名，定义时必须绑定对象，无法重新绑定其他对象
         </p>
        </li>
        <li>
         <p>
          直接使用（如
          <code>
           ref = 5
          </code>
          即修改原对象），无需额外操作符，语法简洁
         </p>
        </li>
        <li>
         <p>
          用途：函数按引用传递（修改原数据或避免拷贝）、返回值、操作符重载
         </p>
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
    <h2>
     19. struct和class的区别（C++）
    </h2>
    <blockquote>
     <ol>
      <li>
       <p>
        struct在默认访问权限和继承方式都是public，而class是private；
       </p>
      </li>
      <li>
       <p>
        除了默认访问权限和继承方式外，两者在功能上完全相同，但struct 偏数据，class 偏对象。
       </p>
      </li>
     </ol>
    </blockquote>
    <h2>
     20. c++模板
    </h2>
    <blockquote>
     <ol>
      <li>
       <p>
        <strong>
         泛型编程
        </strong>
        ： 模板允许在编译期生成针对不同数据类型的代码，提高代码复用性和类型安全。
       </p>
      </li>
      <li>
       <p>
        <strong>
         函数模板与类模板
        </strong>
        ：
       </p>
       <ul>
        <li>
         <p>
          函数模板：定义一个模板函数，使得同一函数可以操作多种数据类型。
         </p>
        </li>
        <li>
         <p>
          类模板：定义一个模板类，使得同一个类能处理多种类型的数据。
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         模板特化
        </strong>
        ： 针对特定类型提供定制化实现，分为完全特化和部分特化。
       </p>
      </li>
      <li>
       <p>
        <strong>
         模板元编程
        </strong>
        ： 利用模板在编译期间进行计算，提高程序的灵活性和效率。
       </p>
      </li>
     </ol>
    </blockquote>
