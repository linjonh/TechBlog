    <h2>
     在Linux学习过程中，掌握基础开发工具是迈向深入学习和实际应用的关键一步。以下是我对基础开发工具重点知识的整理，希望能为你的Linux学习之旅提供有价值的参考。
    </h2>
    <h3>
     一、软件包管理器
    </h3>
    <h4>
     1. 什么是软件包
    </h4>
    <p>
     在Linux下，软件包是将预先编译好的软件进行封装形成的文件，类似于Windows上的安装程序。通过包管理器，用户可以方便地获取并安装这些软件包。
    </p>
    <h4>
     2. Linux软件生态
    </h4>
    <p>
     Linux软件生态的丰富性是衡量系统好坏的重要标准之一。许多社区和个人会免费提供软件，并建立云服务器供用户下载，这极大地丰富了Linux的软件资源。
    </p>
    <h4>
     3. 国内镜像源
    </h4>
    <p>
     国内镜像源如阿里云、清华大学开源软件镜像站等，为用户提供丰富的Linux发行版镜像和软件包，方便用户快速下载和更新软件。
    </p>
    <h4>
     4. yum具体操作
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        查看软件包
       </strong>
       ：使用
       <code>
        yum list | grep [软件名]
       </code>
       （Centos）或
       <code>
        apt search [软件名]
       </code>
       （Ubuntu）筛选出关注的软件包。
      </p>
     </li>
     <li>
      <p>
       <strong>
        安装软件
       </strong>
       ：通过
       <code>
        sudo yum install -y [软件名]
       </code>
       （Centos）或
       <code>
        sudo apt install -y [软件名]
       </code>
       （Ubuntu）简单命令完成安装。
      </p>
     </li>
     <li>
      <p>
       <strong>
        卸载软件
       </strong>
       ：使用
       <code>
        sudo yum remove [-y] [软件名]
       </code>
       （Centos）或
       <code>
        sudo apt remove [-y] [软件名]
       </code>
       （Ubuntu）卸载软件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        注意事项
       </strong>
       ：操作必须保证主机网络畅通，安装和卸载过程中避免重复操作，如有报错可自行搜索解决。
      </p>
     </li>
    </ul>
    <h3>
     二、编辑器Vim
    </h3>
    <h4>
     1. 基本概念
    </h4>
    <p>
     Vim是一种多模式编辑器，主要有三种模式：命令模式（Normal mode）、插入模式（Insert mode）和底行模式（Last line mode），各模式功能区分明确。
    </p>
    <h4>
     2. 基本操作
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        进入Vim
       </strong>
       ：在系统提示符输入
       <code>
        vim [文件名]
       </code>
       进入Vim全屏幕编辑画面，初始处于命令模式。
      </p>
     </li>
     <li>
      <p>
       <strong>
        模式切换
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          命令模式到插入模式
         </strong>
         ：按
         <code>
          i
         </code>
         、
         <code>
          a
         </code>
         或
         <code>
          o
         </code>
         键。
        </p>
       </li>
       <li>
        <p>
         <strong>
          插入模式到命令模式
         </strong>
         ：按
         <code>
          ESC
         </code>
         键。
        </p>
       </li>
       <li>
        <p>
         <strong>
          命令模式到末行模式
         </strong>
         ：按
         <code>
          shift + :
         </code>
         键。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     3. 正常模式命令集
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        插入模式
       </strong>
       ：按
       <code>
        i
       </code>
       、
       <code>
        a
       </code>
       、
       <code>
        o
       </code>
       键进入插入模式，进行文字输入。
      </p>
     </li>
     <li>
      <p>
       <strong>
        移动光标
       </strong>
       ：可使用键盘上的光标键或
       <code>
        h
       </code>
       、
       <code>
        j
       </code>
       、
       <code>
        k
       </code>
       、
       <code>
        l
       </code>
       键控制光标移动，
       <code>
        G
       </code>
       、
       <code>
        $
       </code>
       、
       <code>
        ^
       </code>
       等键可快速移动到文件特定位置。
      </p>
     </li>
     <li>
      <p>
       <strong>
        删除文字
       </strong>
       ：
       <code>
        x
       </code>
       键删除光标所在位置的字符，
       <code>
        dd
       </code>
       键删除光标所在行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        复制
       </strong>
       ：
       <code>
        yw
       </code>
       复制光标所在处到字尾的字符，
       <code>
        yy
       </code>
       复制光标所在行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        替换
       </strong>
       ：
       <code>
        r
       </code>
       键替换光标所在处的字符，
       <code>
        R
       </code>
       键持续替换字符直到按下
       <code>
        ESC
       </code>
       键。
      </p>
     </li>
     <li>
      <p>
       <strong>
        撤销上一次操作
       </strong>
       ：
       <code>
        u
       </code>
       键撤销操作，
       <code>
        ctrl + r
       </code>
       恢复撤销。
      </p>
     </li>
     <li>
      <p>
       <strong>
        更改
       </strong>
       ：
       <code>
        cw
       </code>
       更改光标所在处的字到字尾处。
      </p>
     </li>
     <li>
      <p>
       <strong>
        跳至指定的行
       </strong>
       ：
       <code>
        ctrl + g
       </code>
       列出光标所在行的行号，
       <code>
        #G
       </code>
       跳到指定行。
      </p>
     </li>
    </ul>
    <h4>
     4. 底行模式命令集
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        列出行号
       </strong>
       ：输入
       <code>
        set nu
       </code>
       列出文件中每行的行号。
      </p>
     </li>
     <li>
      <p>
       <strong>
        跳到文件中的某一行
       </strong>
       ：输入行号即可跳转到指定行。
      </p>
     </li>
     <li>
      <p>
       <strong>
        查找字符
       </strong>
       ：
       <code>
        /关键字
       </code>
       向后查找，
       <code>
        ?关键字
       </code>
       向前查找。
      </p>
     </li>
     <li>
      <p>
       <strong>
        保存文件
       </strong>
       ：输入
       <code>
        w
       </code>
       保存文件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        离开Vim
       </strong>
       ：输入
       <code>
        q
       </code>
       退出，
       <code>
        wq
       </code>
       保存并退出，
       <code>
        q!
       </code>
       强制退出不保存。
      </p>
     </li>
    </ul>
    <h4>
     5. 简单Vim配置
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        配置文件位置
       </strong>
       ：系统公共配置文件在
       <code>
        /etc/vimrc
       </code>
       ，用户私有配置文件在用户主目录下的
       <code>
        .vimrc
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        常用配置选项
       </strong>
       ：如设置语法高亮（
       <code>
        syntax on
       </code>
       ）、显示行号（
       <code>
        set nu
       </code>
       ）、设置缩进空格数（
       <code>
        set shiftwidth=4
       </code>
       ）等。
      </p>
     </li>
     <li>
      <p>
       <strong>
        使用插件
       </strong>
       ：可安装TagList、WinManager等插件完善Vim配置，提升使用体验。
      </p>
     </li>
    </ul>
    <h3>
     三、编译器gcc/g++
    </h3>
    <h4>
     1. 背景知识
    </h4>
    <p>
     gcc/g++编译器的使用涉及预处理、编译、汇编和连接四个阶段，将源代码转换为可执行文件。
    </p>
    <h4>
     2. gcc编译选项
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        预处理
       </strong>
       ：
       <code>
        -E
       </code>
       选项进行宏替换等预处理操作，生成
       <code>
        .i
       </code>
       文件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        编译
       </strong>
       ：
       <code>
        -S
       </code>
       选项生成汇编代码，不进行汇编和链接。
      </p>
     </li>
     <li>
      <p>
       <strong>
        汇编
       </strong>
       ：
       <code>
        -c
       </code>
       选项将汇编代码转化为目标代码，生成
       <code>
        .o
       </code>
       文件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        连接
       </strong>
       ：将目标文件链接生成可执行文件。
      </p>
     </li>
    </ul>
    <h4>
     3. 动态链接和静态链接
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        静态库
       </strong>
       ：编译链接时将库文件代码加入可执行文件，生成的文件较大，但运行速度快，且不依赖外部库文件，后缀名一般为
       <code>
        .a
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        动态库
       </strong>
       ：编译链接时不加入库文件代码，程序运行时由运行时链接文件加载库，节省系统开销，后缀名一般为
       <code>
        .so
       </code>
       。gcc在编译时默认使用动态库。
      </p>
     </li>
    </ul>
    <h4>
     4. gcc其他常用选项
    </h4>
    <ul>
     <li>
      <p>
       <code>
        -g
       </code>
       ：生成调试信息，供GNU调试器使用。
      </p>
     </li>
     <li>
      <p>
       <code>
        -O0
       </code>
       、
       <code>
        -O1
       </code>
       、
       <code>
        -O2
       </code>
       、
       <code>
        -O3
       </code>
       ：编译器的优化选项，级别依次升高。
      </p>
     </li>
     <li>
      <p>
       <code>
        -w
       </code>
       ：不生成任何警告信息。
      </p>
     </li>
     <li>
      <p>
       <code>
        -Wall
       </code>
       ：生成所有警告信息。
      </p>
     </li>
    </ul>
    <h3>
     四、自动化构建工具make/Makefile
    </h3>
    <h4>
     1. 背景
    </h4>
    <p>
     Makefile定义了一系列规则，指定文件的编译顺序和依赖关系，实现自动化编译，提高软件开发效率。
    </p>
    <h4>
     2. 理解
    </h4>
    <p>
     Makefile通过定义目标文件、依赖文件和命令规则，控制文件的编译和链接过程。目标文件的依赖关系决定了编译的顺序。
    </p>
    <h4>
     3. 基本使用
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        简单实例
       </strong>
       ：通过定义目标文件、依赖文件和编译命令，实现简单程序的自动化编译。
      </p>
     </li>
     <li>
      <p>
       <strong>
        依赖关系
       </strong>
       ：目标文件依赖于源文件，源文件变化时，Makefile会自动重新编译相关的目标文件。
      </p>
     </li>
     <li>
      <p>
       <strong>
        项目清理
       </strong>
       ：使用
       <code>
        clean
       </code>
       目标删除生成的中间文件和可执行文件，方便重新编译。
      </p>
     </li>
    </ul>
    <h4>
     4. 适度扩展语法
    </h4>
    <p>
     通过定义变量、使用函数等，简化Makefile的编写，提高其可读性和可维护性。
    </p>
    <h3>
     五、Linux第一个系统程序−进度条
    </h3>
    <h4>
     1. 行缓冲区
    </h4>
    <p>
     行缓冲区的存在可能导致程序输出行为的差异，理解行缓冲区有助于控制程序的输出时机。
    </p>
    <h4>
     2. 练手−倒计时程序
    </h4>
    <p>
     通过编写倒计时程序，掌握行缓冲区的使用和控制，以及如何实现简单的文本输出控制。
    </p>
    <h4>
     3. 进度条代码
    </h4>
    <p>
     进度条的实现展示了如何通过字符操作和行缓冲区控制，实现动态的进度显示，提升用户交互体验。
    </p>
    <h3>
     六、版本控制器Git
    </h3>
    <h4>
     1. 版本控制器
    </h4>
    <p>
     版本控制器记录文件的历史和开发过程，方便管理不同版本的文件和协同作业，Git是最主流的版本控制器。
    </p>
    <h4>
     2. git简史
    </h4>
    <p>
     Git诞生于2005年，因Linux内核开发需求而创建，具有速度快、支持非线性开发、完全分布式等特点。
    </p>
    <h4>
     3. 安装git
    </h4>
    <p>
     通过
     <code>
      yum install git
     </code>
     命令安装Git，为后续的版本控制操作做好准备。
    </p>
    <h4>
     4. 在Github创建项目
    </h4>
    <p>
     注册Github账号，创建项目并获取项目链接，为代码的上传和管理做准备。
    </p>
    <h4>
     5. 三板斧
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        git add
       </strong>
       ：将代码添加到Git管理中。
      </p>
     </li>
     <li>
      <p>
       <strong>
        git commit
       </strong>
       ：提交改动到本地仓库，需附上提交日志。
      </p>
     </li>
     <li>
      <p>
       <strong>
        git push
       </strong>
       ：将本地仓库的改动同步到远程服务器，如Github。
      </p>
     </li>
    </ul>
    <h4>
     6. 其他
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        git log/status/pull
       </strong>
       ：查看提交历史、当前状态和从远程拉取更新。
      </p>
     </li>
     <li>
      <p>
       <strong>
        .ignore
       </strong>
       ：配置忽略文件，避免不必要的文件被纳入版本控制。
      </p>
     </li>
    </ul>
    <h3>
     七、调试器gdb/cgdb使用
    </h3>
    <h4>
     1. 样例代码
    </h4>
    <p>
     通过简单的C语言代码示例，展示如何使用gdb进行调试。
    </p>
    <h4>
     2. 预备
    </h4>
    <p>
     在编译源代码时添加
     <code>
      -g
     </code>
     选项，生成包含调试信息的可执行文件，为调试做好准备。
    </p>
    <h4>
     3. 常见使用
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        开始调试
       </strong>
       ：使用
       <code>
        gdb binFile
       </code>
       命令启动gdb调试器。
      </p>
     </li>
     <li>
      <p>
       <strong>
        退出调试
       </strong>
       ：按
       <code>
        ctrl + d
       </code>
       或输入
       <code>
        quit
       </code>
       命令退出调试器。
      </p>
     </li>
     <li>
      <p>
       <strong>
        常用调试命令
       </strong>
       ：如
       <code>
        list
       </code>
       查看源代码，
       <code>
        run
       </code>
       开始执行程序，
       <code>
        next
       </code>
       单步执行不进入函数内部，
       <code>
        step
       </code>
       单步执行进入函数内部，
       <code>
        break
       </code>
       设置断点，
       <code>
        info break
       </code>
       查看断点信息等。
      </p>
     </li>
    </ul>
    <h4>
     4. 常见技巧
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        安装cgdb
       </strong>
       ：提升调试体验，实现在终端中查看代码的调试。
      </p>
     </li>
     <li>
      <p>
       <strong>
        watch
       </strong>
       ：监视变量值的变化，定位问题。
      </p>
     </li>
     <li>
      <p>
       <strong>
        set var
       </strong>
       ：修改变量的值，测试程序的行为。
      </p>
     </li>
     <li>
      <p>
       <strong>
