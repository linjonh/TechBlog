    <h3>
     框架概述
    </h3>
    <p>
     一个基本的 PHP 框架通常包含以下几个部分：
    </p>
    <ul>
     <li>
      <p>
       前端控制器（Front Controller）：处理所有的 HTTP 请求，统一入口。
      </p>
     </li>
     <li>
      <p>
       路由器（Router）：解析请求的 URI，并将其映射到相应的控制器和方法。
      </p>
     </li>
     <li>
      <p>
       控制器（Controller）：处理具体的业务逻辑。
      </p>
     </li>
     <li>
      <p>
       视图（View）：负责展示数据（本示例主要关注控制器部分）。
      </p>
     </li>
    </ul>
    <p>
     通过这种结构，框架能够有效地组织代码，提高可维护性和扩展性。
    </p>
    <h3>
     目录结构
    </h3>
    <p>
     假设我们的项目目录结构如下：
    </p>
    <pre>my_framework/
├── index.php
├── Router.php
├── Controller.php
└── controllers/
    └── HomeController.php
</pre>
    <ul>
     <li>
      <p>
       <code>
        index.php
       </code>
       ：前端控制器，所有请求都通过它入口。
      </p>
     </li>
     <li>
      <p>
       <code>
        Router.php
       </code>
       ：路由器类，负责解析 URI 并调用相应的控制器方法。
      </p>
     </li>
     <li>
      <p>
       <code>
        Controller.php
       </code>
       ：控制器基类，其他控制器可以继承它。
      </p>
     </li>
     <li>
      <p>
       <code>
        controllers/
       </code>
       ：存放具体的控制器类文件。
      </p>
     </li>
    </ul>
    <h3>
     实现步骤
    </h3>
    <h4>
     1. 前端控制器（index.php）
    </h4>
    <p>
     前端控制器是框架的入口文件，负责接收所有的 HTTP 请求，并将其传递给路由器处理。
    </p>
    <pre><code>&lt;?php
// index.php

// 启用错误报告（开发阶段使用，生产环境请关闭）
ini_set('display_errors', 1);
error_reporting(E_ALL);

// 自动加载类文件
spl_autoload_register(function ($class) {
    if (file_exists($class . '.php')) {
        require_once $class . '.php';
    } elseif (file_exists('controllers/' . $class . '.php')) {
        require_once 'controllers/' . $class . '.php';
    }
});

// 获取请求的 URI 和方法
$requestUri = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);
$requestMethod = $_SERVER['REQUEST_METHOD'];

// 实例化路由器并添加路由
$router = new Router();

// 定义路由规则
$router-&gt;add('GET', '/', 'HomeController@index');
$router-&gt;add('GET', '/about', 'HomeController@about');
$router-&gt;add('POST', '/submit', 'HomeController@submit');

// 处理请求
$router-&gt;dispatch($requestMethod, $requestUri);
?&gt;</code></pre>
    <p>
     说明：
    </p>
    <ul>
     <li>
      <p>
       错误报告：在开发阶段启用错误报告，便于调试。
      </p>
     </li>
     <li>
      <p>
       自动加载：使用
       <code>
        spl_autoload_register
       </code>
       自动加载类文件，简化类的引入。
      </p>
     </li>
     <li>
      <p>
       获取请求信息：通过
       <code>
        $_SERVER
       </code>
       全局变量获取请求的 URI 和 HTTP 方法。
      </p>
     </li>
     <li>
      <p>
       路由定义：使用
       <code>
        $router-&gt;add
       </code>
       方法添加路由规则，格式为
       <code>
        HTTP_METHOD
       </code>
       ,
       <code>
        URI
       </code>
       ,
       <code>
        Controller@method
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       请求分发：调用
       <code>
        $router-&gt;dispatch
       </code>
       方法，根据请求信息分发到相应的控制器方法。
      </p>
     </li>
    </ul>
    <h4>
     2. 路由器类（Router.php）
    </h4>
    <p>
     路由器类负责解析请求的 URI，并调用相应的控制器和方法。
    </p>
    <pre><code>&lt;?php
// Router.php

class Router
{
    private $routes = [];

    /**
     * 添加路由规则
     *
     * @param string $method HTTP 方法（GET, POST, etc.）
     * @param string $uri 请求的 URI
     * @param string $action 控制器和方法，例如 'HomeController@index'
     */
    public function add($method, $uri, $action)
    {
        $this-&gt;routes[] = [
            'method'  =&gt; strtoupper($method),
            'uri'     =&gt; $uri,
            'action'  =&gt; $action
        ];
    }

    /**
     * 分发请求到相应的控制器方法
     *
     * @param string $requestMethod HTTP 方法
     * @param string $requestUri 请求的 URI
     */
    public function dispatch($requestMethod, $requestUri)
    {
        foreach ($this-&gt;routes as $route) {
            if ($route['method'] === strtoupper($requestMethod) &amp;&amp; $route['uri'] === $requestUri) {
                $this-&gt;executeAction($route['action']);
                return;
            }
        }
        // 如果没有匹配的路由，返回 404
        $this-&gt;sendNotFound();
    }

    /**
     * 执行控制器的方法
     *
     * @param string $action 控制器和方法，例如 'HomeController@index'
     */
    private function executeAction($action)
    {
        list($controllerName, $method) = explode('@', $action);
        if (class_exists($controllerName)) {
            $controller = new $controllerName();
            if (method_exists($controller, $method)) {
                call_user_func([$controller, $method]);
                return;
            }
        }
        // 如果控制器或方法不存在，返回 404
        $this-&gt;sendNotFound();
    }

    /**
     * 发送 404 响应
     */
    private function sendNotFound()
    {
        header("HTTP/1.0 404 Not Found");
        echo "404 Not Found";
    }
}
?&gt;</code></pre>
    <p>
     说明：
    </p>
    <ul>
     <li>
      <p>
       添加路由：
       <code>
        add
       </code>
       方法用于添加新的路由规则。
      </p>
     </li>
     <li>
      <p>
       分发请求：
       <code>
        dispatch
       </code>
       方法遍历所有路由规则，找到匹配的规则后调用相应的控制器方法。
      </p>
     </li>
     <li>
      <p>
       执行动作：
       <code>
        executeAction
       </code>
       方法解析控制器和方法名称，并调用相应的方法。
      </p>
     </li>
     <li>
      <p>
       404 处理：如果没有匹配的路由或控制器方法不存在，返回 404 响应。
      </p>
     </li>
    </ul>

     3. 控制器基类（Controller.php）

    </p>
    <pre>&lt;?php
// Controller.php

class Controller
{
    // 在这里可以添加公共的方法或属性

    <h4>
     4. 示例控制器（HomeController.php）
    </h4>
    <p>
     这是一个示例控制器，包含几个方法来处理不同的请求。
    </p>
    <pre><code>&lt;?php
// controllers/HomeController.php

class HomeController extends Controller
{
    /**
     * 主页方法
     */
    public function index()
    {
        echo "&lt;h1&gt;欢迎来到主页！&lt;/h1&gt;";
    }

    /**
     * 关于页面方法
     */
    public function about()
    {
        echo "&lt;h1&gt;关于我们&lt;/h1&gt;&lt;p&gt;这是关于页面。&lt;/p&gt;";
    }

    /**
     * 处理表单提交的方法
     */
    public function submit()
    {
        // 假设有一个表单提交到 /submit
        // 处理 POST 数据
        $data = $_POST;
        echo "&lt;h1&gt;表单已提交&lt;/h1&gt;";
        echo "&lt;pre&gt;";
        print_r($data);
        echo "&lt;/pre&gt;";
    }
}
?&gt;</code></pre>
    <p>
     说明：
    </p>
    <ul>
     <li>
      <p>
       index 方法：处理主页请求，输出欢迎信息。
      </p>
     </li>
     <li>
      <p>
       about 方法：处理关于页面请求，输出关于信息。
      </p>
     </li>
     <li>
      <p>
       submit 方法：处理表单提交的 POST 请求，输出提交的数据。
      </p>
     </li>
    </ul>

    <h3>
     总结
    </h3>
    <p>
     本文介绍了如何构建一个简单的 PHP 框架，实现将 HTTP 请求映射到类的方法中。通过定义路由规则、创建路由器类和控制器类，可以轻松地管理不同的请求路径和对应的业务逻辑。
    </p>
    <p>
     关键步骤包括：
    </p>
    <ol>
     <li>
      <p>
       前端控制器：作为所有请求的统一入口，负责初始化路由器并分发请求。
      </p>
     </li>
     <li>
      <p>
       路由器类：负责解析请求的 URI 和方法，并调用相应的控制器方法。
      </p>
     </li>
     <li>
      <p>
       控制器类：包含具体的业务逻辑，处理不同的请求。
      </p>
     </li>
    </ol>
    <p>
     扩展建议：
    </p>
    <ul>
     <li>
      <p>
       动态路由：支持带参数的动态路由，例如
       <code>
        /user/{id}
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       中间件：实现中间件机制，处理认证、日志记录等功能。
      </p>
     </li>
     <li>
      <p>
       视图模板：集成模板引擎（如 Twig）以分离视图和逻辑。

      <p>

      <p>
