    <p>
     最近国补 5040 购入了一台 Thinkbook14+，决定将之前在拯救者上的系统迁移过来
    </p>
    <p>
     现有：
    </p>
    <ul>
     <li>
      拯救者 500G 系统盘 + 2T 数据盘
     </li>
     <li>
      thinkbook14+自带 1T 盘
     </li>
    </ul>
    <p>
     目标：把系统迁移到 1T 盘上，数据盘拆装到 thinkbook 上，500G 的盘留在拯救者上
    </p>
    <hr/>
    <p>
     话不多说，拆机！
    </p>
    <p>
     首先吧 500G 的系统盘装上去
    </p>
    <blockquote>
     <p>
      <a href="https://www.bilibili.com/video/BV12S4y1W7Rt" rel="nofollow">
       thinkbook14+ 2025 拆机视频 | BiliBili
      </a>
     </p>
    </blockquote>
    <h3>
     <a id="_1_34">
     </a>
     问题 1-启动黑屏
    </h3>
    <p>
     然后发现无法启动，grub 之后就黑屏了。研究发现可以被 ping 到，说明系统启动了，
    </p>
    <p>
     连接进入检查日志发现如下关键报错
    </p>
    <pre><code>kernel: pci 0000:00:1f.3: deferred probe pending: sof-audio-pci-intel-mtl: init of i915 and HDMI codec failed
</code></pre>
    <p>
     网上询问得到如下临时方案
    </p>
    <p>
     进入系统修改
     <code>
      /etc/default/grub
     </code>
     ，如下行为
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">GRUB_CMDLINE_LINUX_DEFAULT</span><span class="token operator">=</span><span class="token string">"quiet splash nomodeset"</span>
</code></pre>
    <blockquote>
     <p>
      使用“nomodeset”会禁用硬件加速和动态模式设置，用于调试和临时解决启动问题。但可能导致图形性能下降或不支持高级显示特性（比如亮度调节，像素选择等），所以仅建议临时使用。
     </p>
    </blockquote>
    <p>
     成功启动
    </p>
    <h3>
     <a id="_2_56">
     </a>
     问题 2-系统迁移
    </h3>
    <p>
     如何吧 500G 固态上的系统迁移到 1T 的固态上
    </p>
    <p>
     使用 fdisk -l 命令确认旧系统硬盘
     <code>
      /dev/sda
     </code>
     和新硬盘
     <code>
      /dev/sdb
     </code>
     的设备名称。
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">dd</span> <span class="token assign-left variable">if</span><span class="token operator">=</span>/dev/sda <span class="token assign-left variable">of</span><span class="token operator">=</span>/dev/sdb <span class="token assign-left variable">bs</span><span class="token operator">=</span>64K <span class="token assign-left variable">conv</span><span class="token operator">=</span>noerror,sync
</code></pre>
    <p>
     更新引导
    </p>
    <p>
     如果新硬盘大小不同或者需要调整分区结构，可能还需要修改分区 UUID 或重新安装 GRUB。可以挂载新硬盘的系统分区，然后 chroot 进去执行：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">mount</span> /dev/sdbX /mnt   <span class="token comment"># 挂载新系统分区（将 X 替换为具体分区号）</span>
<span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">--bind</span> /dev /mnt/dev
<span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">--bind</span> /proc /mnt/proc
<span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">--bind</span> /sys /mnt/sys
<span class="token function">sudo</span> <span class="token function">chroot</span> /mnt
<span class="token function">update-grub</span>
grub-install /dev/sdb
<span class="token builtin class-name">exit</span>
<span class="token function">sudo</span> <span class="token function">umount</span> /mnt/<span class="token punctuation">{<!-- --></span>dev,proc,sys<span class="token punctuation">}</span>
<span class="token function">sudo</span> <span class="token function">umount</span> /mnt
</code></pre>
    <p>
     然后重启
    </p>
    <p>
     由于是 500G 固态复制到 1T 的固态，分区表也复制上去了，大小也变成了 500G，需要执行下面的命令来恢复
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> partprobe /dev/sdb
</code></pre>
    <h3>
     <a id="_3_91">
     </a>
     问题 3-无法识别分区
    </h3>
    <p>
     虽然更新了分区表，但是启动的时候 bios 还是无法识别到分区，幸好启动其他系统的时候 grub 会识别到这个系统
    </p>
    <p>
     进入 recover 模式，重新执行
    </p>
    <pre><code class="prism language-bash"><span class="token function">update-grub</span>
grub-install /dev/sdb
<span class="token builtin class-name">exit</span>
</code></pre>
    <p>
     解决了
    </p>
    <h3>
     <a id="_4_105">
     </a>
     问题 4-驱动
    </h3>
    <p>
     默认 Kubuntu24.04 安装情况下，如下设备有驱动问题
    </p>
    <ul>
     <li>
      核显 Intel® Arc™ 130T GPU（可以用
      <code>
       nomodeset
      </code>
      解决但会有更多问题，比如：
      <ul>
       <li>
        无法调节亮度
       </li>
       <li>
        无法调节屏幕大小
       </li>
      </ul>
     </li>
     <li>
      触控板
     </li>
     <li>
      声卡
     </li>
     <li>
      指纹
     </li>
    </ul>
    <p>
     找到了完善的解决方案，安装针对硬件兼容优化的内核版本
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> linux-generic-hwe-24.04
</code></pre>
    <blockquote>
     <p>
      <a href="https://zhuanlan.zhihu.com/p/518111547" rel="nofollow">
       Thinkbook14+(2022)安装 Ubuntu20.04 | 知乎
      </a>
     </p>
    </blockquote>
    <p>
     oem 和 HWE 内核的区别
    </p>
    <ol>
     <li>
      oem 内核（OEM Kernel）
      <ul>
       <li>
        oem 内核主要用于 OEM 设备（预装 Ubuntu 的品牌电脑，如 Dell、Lenovo 提供的 Ubuntu 预装机型）。
       </li>
       <li>
        这个内核通常由硬件厂商与 Canonical 合作进行优化，可能包含一些主线 Linux 内核未合并的补丁，以更好地支持特定硬件。
       </li>
       <li>
        适用于特定硬件的驱动优化和稳定性改进。
       </li>
      </ul>
     </li>
     <li>
      HWE 内核（Hardware Enablement Kernel）
      <ul>
       <li>
        HWE 内核是 Ubuntu LTS 版本（长期支持版）的更新内核，通常提供新版本的内核和 X.org 图形栈，以支持新硬件。
       </li>
       <li>
        Ubuntu 每隔一段时间会从较新的 Ubuntu 非 LTS 版本（如 Ubuntu 24.10）中引入内核到 LTS 版本（如 Ubuntu 24.04）。
       </li>
       <li>
        适合希望使用新硬件但不想升级到非 LTS 版本的用户。
       </li>
      </ul>
     </li>
    </ol>
    <p>
     除了指纹，基本的功能都解决了，几乎完美迁移到新电脑上了不用重装什么的也太棒了
    </p>
