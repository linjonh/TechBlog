    <ul>
     <li>
      <strong>
       class(类)是一种
       <u>
        类型
       </u>
       ，用class(类)可以创建
       <span style="color:#fe2c24">
        类变量
       </span>
      </strong>
      ，
      <strong>
       <span style="background-color:#c7e6ea">
        类变量里存储变量与方法
       </span>
      </strong>
      ，创建类变量时都有开辟空间，类变量里的存储的
      <strong>
       <u>
        成员变量都存储在堆区
       </u>
      </strong>
      ，类变量里存储的
      <strong>
       <u>
        成员方法都存储在方法区
       </u>
      </strong>
      ，
      <u>
       类变量本身也是个变量存有空间
      </u>
      ，它里面存储这些成员变量与成员方法，不过类变量开辟的存储空间有部分在堆上(变量)，有部分在方法区上(方法)，是在两个不同区域开辟空间创建来的变量，
      <u>
       <span style="background-color:#c7e6ea">
        类变量创建后相应地它也有
       </span>
       <strong>
        <span style="background-color:#c7e6ea">
         哈希值
        </span>
       </strong>
       <span style="background-color:#c7e6ea">
        ，指向创建的这个类变量，能访问这个类变量的所有
       </span>
      </u>
     </li>
     <li>
      类变量创建好后，
      <span style="color:null">
       <strong>
        类变量也可作为一种类型，以此类变量为类型创建
       </strong>
      </span>
      <span style="color:#fe2c24">
       <strong>
        似复刻变量
       </strong>
      </span>
      ，似复刻变量
      <strong>
       <span style="background-color:#ffd900">
        按照类变量里的非静态变量模具变量创建
        <span style="color:#956fe7">
         复制变量
        </span>
        、在堆上开辟空间存储有类变量里的静态变量、类变量里的所有方法的访问
        <span style="color:#956fe7">
         路径信息
        </span>
       </span>
      </strong>
      ，用类变量创建的似复刻变量在
      <u>
       <strong>
        堆区
       </strong>
      </u>
      ，复刻变量里存储复制变量与访问类变量路径信息两块内容，
      <u>
       <span style="background-color:#c7e6ea">
        创建好似复刻变量后，相应地有指向此似复刻变量的
       </span>
       <strong>
        <span style="background-color:#c7e6ea">
         哈希值
        </span>
       </strong>
       <span style="background-color:#c7e6ea">
        ，能访问与管理到此似复刻变量的所有内容
       </span>
      </u>
      ，
      <strong>
       似复刻变量的存哈希值引用类型与类变量存哈希值引用类型是共用的，因为它们最终的指向对象类型都是相同的
      </strong>
     </li>
    </ul>

     <h2 id="%E4%B8%80%E3%80%81%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E7%9A%84%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E6%80%A7" name="%E4%B8%80%E3%80%81%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E7%9A%84%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E6%80%A7">
      一、数据结构的
      <span style="color:#fe2c24">
       面向对象性
      </span>
     </h2>
    </blockquote>
    <h3 id="1.%E9%9D%A2%E5%90%91%E4%BC%BC%E5%A4%8D%E5%88%BB%E5%8F%98%E9%87%8F%E5%AF%B9%E8%B1%A1%E5%BC%95%E7%94%A8" name="1.%E9%9D%A2%E5%90%91%E4%BC%BC%E5%A4%8D%E5%88%BB%E5%8F%98%E9%87%8F%E5%AF%B9%E8%B1%A1%E5%BC%95%E7%94%A8">
     1.面向
     <span style="color:#fe2c24">
      似复刻变量对象引用
     </span>
    </h3>
    <p>
     以引用的形式面向似复刻变量对象，将它们
     <u>
      <span style="color:#fe2c24">
       <strong>
        引用组群管理起来
       </strong>
      </span>
      的结构就是各种各样的数据结构
     </u>
     ，
     <span style="color:#fe2c24">
      <strong>
       似复刻变量对象引用
      </strong>
     </span>
     是数据存储的目标单位元，以数组直接存放管理似复刻变量对象引用的结构就是顺序表
    </p>
    <hr/>
    <h3 id="2.%E4%B8%8B%E5%B1%82%E5%AD%98%E5%82%A8%E5%BC%95%E7%94%A8%E7%9A%84%E7%89%B9%E7%82%B9" name="2.%E4%B8%8B%E5%B1%82%E5%AD%98%E5%82%A8%E5%BC%95%E7%94%A8%E7%9A%84%E7%89%B9%E7%82%B9">
     2.
     <span style="color:#fe2c24">
      下层存储引用
     </span>
     的特点
    </h3>
    <p>
     同次中全部管理的是相同类变量类型的似复刻变量，因此
     <strong>
      <span style="background-color:#ffd900">
       存储的目标单位元引用都是相同的一个引用类型在一次中
      </span>
      <span style="color:null">
       ，所以用
      </span>
     </strong>
     <span style="color:null">
      <span style="color:null">
       <strong>
        &lt;统一似复刻变量对象引用类型的
       </strong>
      </span>
     </span>
     <span style="color:#ff9900">
      <strong>
       泛型&gt;来统一化结构里的单位元引用类型
      </strong>
     </span>
     ，&lt;泛型化又统一化&gt;是面向似复刻变量对象的数据结构的特点，因为向上转型能保存子类信息所以
     <span style="color:#4da8ee">
      <strong>
       将似复刻变量对象引用的存储统一用Object类变量的似复刻变量引用向上转型式存储
      </strong>
     </span>
     ，
     <strong>
      数据结构的
      <span style="background-color:#38d8f0">
       引用存储格式就能统一化创建存储
      </span>
      了
     </strong>
    </p>
    <hr/>
    <h3 id="3.%E4%B8%8A%E5%B1%82%E7%AE%A1%E7%90%86%E5%BC%95%E7%94%A8" name="3.%E4%B8%8A%E5%B1%82%E7%AE%A1%E7%90%86%E5%BC%95%E7%94%A8">
     3.
     <span style="color:#fe2c24">
      上层管理引用
     </span>
    </h3>
    <p>
     似复刻变量引用作为存储数据对象存储在成组织的个个Object引用变量中，同时需要一个管理存储整体的管理这些存储引用变量的引用，即上层的管理引用管理下层的存储引用，
     <strong>
      <span style="background-color:#ffd900">
       上层的管理引用其实是下层存储引用作为非静态复制变量所对类变量的实例化似复刻变量引用
      </span>
     </strong>
     ，上层的管理引用是似复刻变量，
     <strong>
      <span style="background-color:#38d8f0">
       下层的存储引用是似复刻变量里的非静态成员
      </span>
     </strong>
     ，这样对它管理的，
     <strong>
      所属的类变量就是
      <span style="color:#fe2c24">
       数据结构的类变量
      </span>
     </strong>
     ，
     <u>
      有很多种数据结构有很多种对应的类变量类
     </u>
     ，
     <span style="color:null">
      <strong>
       对着它的一个类变量创建似复刻变量就是创建这个类变量对应的一个数据结构
      </strong>
     </span>
    </p>
    <hr/>
    <h3 id="4.%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E7%9A%84%E7%B1%BB" name="4.%E6%95%B0%E6%8D%AE%E7%BB%93%E6%9E%84%E7%9A%84%E7%B1%BB">
     4.数据结构的
     <span style="color:#fe2c24">
      类
     </span>
    </h3>
    <p>
     <em>
      数据结构的类变量一般有
      <strong>
       继承
      </strong>
      别的一个类、实现很多的
      <strong>
       接口
      </strong>
      ，里面有对应的
      <strong>
       非静态存储引用成员
      </strong>
      作着模板、也有
      <strong>
       静态成员
      </strong>
      、有很多操作非静态存储引用成员的
      <strong>
       方法
      </strong>
      (增删查改这些)，方法有很多是连接接口去
      <strong>
       重写
      </strong>
      对应实现有的，
     </em>
     因为
     <span style="color:#fe2c24">
      <strong>
       存储引用们的类型相互之间必须是统一一种的
      </strong>
     </span>
     ，所以
     <strong>
      类变量的似复刻变量那边就一定有指定一种它里面存储引用的类型—&gt;
      <span style="background-color:#ffd900">
       类&lt;T或具体类型&gt;
      </span>
     </strong>
     ，因为存储引用类型统一了，
     <strong>
      操作存储引用的方法也必须是统一类型的—&gt;
      <span style="background-color:#ffd900">
       方法(E或具体类型)
      </span>
     </strong>
     ，会有很多其它也要跟着统一类型的
    </p>
    <hr/>
    <blockquote>
     <h2 id="%E4%BA%8C%E3%80%81ArrayList%E7%B1%BB%E5%8F%98%E9%87%8F%E9%87%8C%E7%9A%84%E6%96%B9%E6%B3%95" name="%E4%BA%8C%E3%80%81ArrayList%E7%B1%BB%E5%8F%98%E9%87%8F%E9%87%8C%E7%9A%84%E6%96%B9%E6%B3%95">
      二、ArrayList类变量里的方法
     </h2>
    </blockquote>
    <h3 id="1.%E6%9E%84%E9%80%A0" name="1.%E6%9E%84%E9%80%A0" style="text-align:center">
     <span style="color:#0d0016">
      <strong>
       1.构造
      </strong>
     </span>
    </h3>
    <blockquote>
     <h4 id="1.1%E6%97%A0%E5%8F%82%E6%9E%84%E9%80%A0%EF%BC%9A" name="1.1%E6%97%A0%E5%8F%82%E6%9E%84%E9%80%A0%EF%BC%9A">
      <strong>
       1.1
       <span style="color:#fe2c24">
        无参构造
       </span>
      </strong>
      ：
     </h4>
     <p>
      <strong>
       new
       <span style="background-color:#ffd900">
        ArrayList&lt;E&gt;()
       </span>
       ;
      </strong>
     </p>
     <p>
      <strong>
       —&gt; return
       <span style="background-color:#38d8f0">
        ArrayList
       </span>
      </strong>
     </p>
     <p>
      <strong>
       <span style="background-color:#a2e043">
        构造出一个里面存储引用的复制变量0存储引用容量的顺序表的似复刻变量
       </span>
       ，
       <span style="color:#fe2c24">
        0
       </span>
       存储引用容量的复制变量在后面通过调用
       <span style="color:#fe2c24">
        add
       </span>
       等会进行增扩空间放引用
      </strong>
     </p>
    </blockquote>
    <blockquote>
     <h4 id="1.2%E5%B9%B6%E8%A1%A8%E6%9E%84%E9%80%A0%EF%BC%9A" name="1.2%E5%B9%B6%E8%A1%A8%E6%9E%84%E9%80%A0%EF%BC%9A">
      <strong>
       1.2
       <span style="color:#fe2c24">
        并表构造
       </span>
      </strong>
      ：
     </h4>
     <p>
      <strong>
       new
       <span style="background-color:#ffd900">
        ArrayList&lt;E&gt;(Collection)
       </span>
       ;
      </strong>
     </p>
     <p>
      <strong>
       —&gt; return
       <span style="background-color:#38d8f0">
        ArrayList
       </span>
      </strong>
     </p>
     <p>
      <strong>
       <span style="background-color:#a2e043">
        往c里面向下搜索到存储引用的表的存储引用复制并到创的新的存储引用数组成员在此新的构造顺序表的似复刻变量里
       </span>
       ，这里是创建新的存储引用数组的
       <span style="color:#fe2c24">
        浅拷贝
       </span>
       ，存储引用复制到新的存储引用数组里
       <span style="color:#fe2c24">
        还是共享着同一实例
       </span>
      </strong>
     </p>
    </blockquote>
    <p>
     对Collection c的要求：
     <span style="color:#fe2c24">
      <strong>
       ArrayList(Collection&lt;? extends E&gt; c)
      </strong>
     </span>
    </p>
    <p>
     ArrayList类变量里的构造方法设置的
     <strong>
      <span style="background-color:#ffd900">
       形参类型为Collection接口变量
      </span>
     </strong>
     (用来接收传来的Collection及其下面接口、类实现的似复刻变量)，和ArrayList类变量一样，它也有指定统一引用类型的，
     <strong>
      把两个&lt;&gt;写一起一个里面写
     </strong>
     ，
     <strong>
      通配符?是Collection的指定引用类型，E是ArrayList的指定引用类型
     </strong>
     ，extend说明?继承于E即
     <strong>
      <span style="background-color:#ffd900">
       Collection的指定引用类型要继承于ArrayList的指定引用类型的
      </span>
     </strong>
    </p>
    <blockquote>
     <h4 id="1.3%E5%AE%9A%E5%AE%B9%E6%9E%84%E9%80%A0%EF%BC%9A" name="1.3%E5%AE%9A%E5%AE%B9%E6%9E%84%E9%80%A0%EF%BC%9A">

      </strong>
     </p>
    </blockquote>

    <h3 id="2.%E9%81%8D%E5%8E%86" name="2.%E9%81%8D%E5%8E%86" style="text-align:center">
     <span style="color:#0d0016">
      <strong>
       2.遍历
      </strong>
     </span>
    </h3>
    <h4 id="2.1%E8%BF%AD%E4%BB%A3%E5%99%A8%E9%81%8D%E5%8E%86" name="2.1%E8%BF%AD%E4%BB%A3%E5%99%A8%E9%81%8D%E5%8E%86">
     2.1迭代器遍历
    </h4>
    <p>
     <span style="color:#fe2c24">
      <strong>
       实现原理
      </strong>
     </span>
     ：
    </p>
    <p>
     数据结构里的
     <strong>
      所有集合实现类往上最上层都间接连接着
      <span style="color:#fe2c24">
       Iterable接口
      </span>
     </strong>
     ，Iterable(可迭代的)接口里
     <strong>
      有以迭代器接口变量Iterator为返回值
      <span style="color:#fe2c24">
       获得迭代器接口变量的抽象方法
      </span>
     </strong>
     ，
     <u>
      Iterable接口往下继承下去每一层都有对应的重写实现方法来重写上制造出具体对应为本层迭代的迭代器获取方法
     </u>
     (接口继承层没重写继续抽象的就没实现抽象传下去)，到具体的集合实现类(或有实现它的抽象类)时，会在
     <strong>
      要实现迭代器的那层类里定义一个
      <span style="color:#fe2c24">
       主动连接上Iterator接口的内部类
      </span>
     </strong>
     (外部类里是一直都没有往下继承有Iterator接口的)，重写获取Iterator迭代器接口变量方法的部分内部，通过
     <strong>
      <span style="color:#fe2c24">
       实例化
      </span>
      返回此实现上Iterator接口的内部类的似复刻变量去
      <span style="color:#fe2c24">
       返回值接转向上转型
      </span>
      <span style="background-color:#ffd900">
       得到重写对应为本层迭代的迭代器似复刻变量
      </span>
     </strong>
     ，就可用它迭代本层类了
    </p>
    <p>
     在
     <strong>
      <span style="color:#fe2c24">
       List接口
      </span>
      里有多加上
      <span style="color:#fe2c24">
       获取ListIterator列表迭代器接口变量的抽象方法
      </span>
     </strong>
     ，相对应地在后面的所有实现List接口的表数据结构实现类都会有相应的获取列表迭代器的重写实现上方法，在那层实现类里相对应的有定义连接ListIterator列表迭代器接口的内部类，在重写方法里面实例化此内部类返回向上转型成迭代本层类的迭代器
    </p>
    <p>
     <strong>
      ListIterator迭代器接口
      <span style="color:#fe2c24">
       继承于
      </span>
      Iterator迭代器接口
     </strong>
     ，ListIterator里面的抽象方法比Iterator多，对应实现后的ListIterator迭代器接口似复刻变量里的方法自然比Iterator迭代器接口的似复刻变量里的方法多：
    </p>
    <h5 id="2.1.1Iterator%E8%BF%AD%E4%BB%A3%E5%99%A8" name="2.1.1Iterator%E8%BF%AD%E4%BB%A3%E5%99%A8">
     2.1.1Iterator迭代器
    </h5>
    <blockquote>
     <h6 id="2.1.1.1%E8%8E%B7%E5%8F%96%EF%BC%9A" name="2.1.1.1%E8%8E%B7%E5%8F%96%EF%BC%9A">
      <strong>
       2.1.1.1
       <span style="color:#fe2c24">
        获取
       </span>
       ：
      </strong>
     </h6>
     <p>
