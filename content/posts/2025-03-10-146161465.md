    <h2 id="h1_问题现象_1741604421845">
     问题现象
    </h2>
    <p>
     OpenHarmony 5.0版本使用AVPlayer播放mpegts封装格式的H.265(HEVC)编码格式的视频时出现报错导致播放失败
    </p>
    <h2 id="h1_问题原因_1741604421845">
     问题原因
    </h2>
    <p>
     OpenHarmony 5.0版本AVPlayer播放器使用
     <strong>
      histreamer引擎
     </strong>
     ，因为 libav_codec_hevc_parser.z.so 动态库未开源导致H265编码格式视频解析不到正确的宽高，硬件解码器框架层初始化时因为分辨率不合理导致报错
    </p>
    <h2 id="h1_解决方案_1741604421845">
     解决方案
    </h2>
    <h3 id="h2_版本信息_1741604421845">
     版本信息
    </h3>
    <p>
     OpenHarmony-v5.0.1-Release 和 OpenHarmony-v5.0.2-Release 标准系统
    </p>
    <h3 id="h2_方案一：设置默认的分辨率_1741604421845">
     方案一：设置默认的分辨率
    </h3>
    <p>
     修改 foundation/multimedia/av_codec/services/media_engine/plugins/ffmpeg_adapter/demuxer/ffmpeg_format_helper.cpp 文件，"+"符号为新增的代码
    </p>
    <pre><code class="language-javascript"><span class="hljs-literal">void</span> FFmpegFormatHelper<span class="hljs-type">::ParseVideoTrackInfo</span>(const AVStream&amp; avStream, Meta &amp;format,
 const AVFormatContext&amp; avFormatContext)
{
    format.<span class="hljs-built_in">Set</span>&lt;<span class="hljs-built_in">Tag</span><span class="hljs-type">::VIDEO_WIDTH</span>&gt;(static_cast&lt;uint32_t&gt;(avStream.codecpar-&gt;width));
    format.<span class="hljs-built_in">Set</span>&lt;<span class="hljs-built_in">Tag</span><span class="hljs-type">::VIDEO_HEIGHT</span>&gt;(static_cast&lt;uint32_t&gt;(avStream.codecpar-&gt;height));
    format.<span class="hljs-built_in">Set</span>&lt;<span class="hljs-built_in">Tag</span><span class="hljs-type">::VIDEO_DELAY</span>&gt;(static_cast&lt;uint32_t&gt;(avStream.codecpar-&gt;video_delay));

+   <span class="hljs-keyword">if</span> ((avStream.codecpar-&gt;width &lt;=<span class="hljs-number">0</span> || avStream.codecpar-&gt;height &lt;=<span class="hljs-number">0</span>) &amp;&amp; (avStream.codecpar-&gt;codec_id == AV_CODEC_ID_HEVC)) {
+       format.<span class="hljs-built_in">Set</span>&lt;<span class="hljs-built_in">Tag</span><span class="hljs-type">::VIDEO_WIDTH</span>&gt;(<span class="hljs-number">1920</span>);
+       format.<span class="hljs-built_in">Set</span>&lt;<span class="hljs-built_in">Tag</span><span class="hljs-type">::VIDEO_HEIGHT</span>&gt;(<span class="hljs-number">1080</span>);
+   }

    double frameRate = <span class="hljs-number">0</span>;
    <span class="hljs-params">...</span>
}
</code></pre>
    <p>
     如果demuxer未解析到H265视频的分辨率，设置默认值用于初始化解码器，分辨率的默认值需要在codec hcs配置文件的分辨率范围内（minWidth和maxWidth，minHeight和maxHeight之间）
    </p>
    <h3 id="h2_方案二：打开ffmpeg-hevc软解用于解析分辨率_1741604421916">
     方案二：打开ffmpeg hevc软解用于解析分辨率
    </h3>
    <p>
     1、third_party/ffmpeg/BUILD.gn 新增h265 parser和decoder源文件编译
    </p>

     注意：如果是64位平台，还需要添加//third_party/ffmpeg/libavcodec/
     <strong>

     <br/>
     另外可能需要修改libavcodec/hevcdec.c的部分代码以通过编译
     <br/>
     2、third_party/ffmpeg/ohos_config.sh 新增h265 parser和decoder配置
     <br/>
     修改1有两处
    </p>
    <pre><code class="language-javascript">-    <span class="hljs-string">"--enable-parser=h263,h264,mpeg4video,vp8,vp9,mpegvideo"</span>

    <p>
     修改2有两处
    </p>
    <pre><code class="language-javascript">-    <span class="hljs-string">"--enable-decoder=h263,h264,mpeg2video,mpeg4,vp8,vp9"</span>

     结语
    </h2>
    <p>
     修改后验证播放mpegts封装的H.265格式的视频走硬件解码可以正常播放。
