    <h2>
     <a id="K8Snfs_0">
     </a>
     K8S持久化存储之nfs
    </h2>
    <p>
     ​ 在 Kubernetes (k8s) 中使用 NFS（Network File System）作为存储解决方案是一种常见的方式，特别是在需要共享存储的场景中。以下是关于如何在 Kubernetes 中使用 NFS 存储的详细说明：
    </p>
    <h4>
     <a id="1__NFS__4">
     </a>
     1. 准备 NFS 服务器
    </h4>
    <p>
     首先，你需要一个 NFS 服务器来提供共享存储。NFS 服务器可以是一个独立的物理机、虚拟机，或者是一个云服务。
    </p>
    <h5>
     <a id="_Linux__NFS__8">
     </a>
     在 Linux 上设置 NFS 服务器
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        安装 NFS 服务器软件包
       </strong>
       ：
      </p>
      <pre><code>yum install nfs-utils -y
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        创建共享目录
       </strong>
       ：
      </p>
      <pre><code>mkdir -p /data/volumes
chmod 777 /data/volumes
echo 'Hello NFS' &gt; /data/volumes/index.html
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        配置 NFS 导出
       </strong>
       ：
       <br/>
       编辑
       <code>
        /etc/exports
       </code>
       文件，添加以下内容：
      </p>
      <pre><code>/data/volumes *(rw,sync,no_subtree_check)
</code></pre>
      <p>
       这表示将
       <code>
        /data/volumes
       </code>
       目录共享给所有客户端，并赋予读写权限。
      </p>
     </li>
     <li>
      <p>
       <strong>
        重启 NFS 服务
       </strong>
       ：
      </p>
      <pre><code>systemctl restart nfs
</code></pre>
     </li>
    </ol>
    <h4>
     <a id="2__Kubernetes__NFS__39">
     </a>
     2. 在 Kubernetes 中使用 NFS 存储
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        创建 Pod
       </strong>
       ：
       <br/>
       创建一个
       <code>
        nfs-pod.yaml
       </code>
       文件：
      </p>
      <pre><code>apiVersion: v1
kind: Pod
metadata:
  name: nfs-pod
spec:
  containers:
  - name: nfs-container
    image: 172.16.80.140/nginx/nginx:1.26
		imagePullPolicy: ifNotPresent
    volumeMounts:
    - name: nfs-volumes
      mountPath: /usr/share/nginx/html
  volumes:
  - name: nfs-volumes
    nfs:
      path: /data/volumes
      server: 172.16.80.131
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        应用 Pod
       </strong>
       ：
      </p>
      <pre><code>kubectl apply -f nfs-pod.yaml
kubectl get pods -owide
</code></pre>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/6e90efd6559a4485a8d6a00e9cfddded.png#pic_center"/>
    </p>
    <h4>
     <a id="3__NFS__74">
     </a>
     3. 验证 NFS 存储
    </h4>
    <p>
     你可以通过以下步骤验证 NFS 存储是否正常工作：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        查看 Pod 日志
       </strong>
       ：
      </p>
      <pre><code>kubectl logs nfs-pod
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        在 NFS 服务器上验证
       </strong>
       ：
       <br/>
       在 NFS 服务器上，验证nfs是否生效 /data/volumes/index.html
      </p>
      <pre><code>curl 10.244.196.133
</code></pre>
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3cb405b0ec2e45f4a7c353e1661c5cc3.png#pic_left" width="500">
      <br/>
      通过以上步骤，你可以在 Kubernetes 中成功使用 NFS 作为存储解决方案。NFS 特别适用于需要多个 Pod 共享同一存储的场景，例如运行有状态应用或共享配置文件等。
      <br/>
      ​ nfs还可以在集群中作为共享存储使用，比如在另外一台节点创建目录，挂载到nfs服务，就可以实现
     </img>
    </p>
    <pre><code>mkdir /test
mount 172.16.80.131:/data/volumes /test
systemctl restart nfs
</code></pre>
