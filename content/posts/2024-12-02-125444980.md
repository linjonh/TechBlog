## 前言

最近再做小程序，发现微信自带许多bug，讲一下运用van\_dialog 的坑。常规操作van-dialog的引用就三部，在json文件中引用，html文件写组件，最后在js文件中直接调用，一切都准备就绪，可弹框还是没有出现，最后翻阅文档才知道，还要给弹框家show:true 属性，具体如下：

---

`提示：以下是本篇文章正文内容，下面案例可供参考`

## 一、json文件中引入

```c
  "usingComponents": {
    "van-dialog": "@vant/weapp/dialog/index"
  },

```

## 二、html 中写入组件

```c
<van-dialog id="van-dialog" />

```

## 三、js中调用

```c
     Dialog.confirm({
        show:true, //坑，一定要写不否则弹框不显示
        title: '标题',
        message: '弹窗内容'
      }).then(() => {
        // on confirm
      }).catch(() => {
        // on cancel
      });

```
