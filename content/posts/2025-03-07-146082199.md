    <p>
     在 Go 语言中，堆（Heap）和栈（Stack）是内存管理中的两个重要概念，它们在内存分配、数据存储和使用场景等方面存在明显差异。
    </p>
    <h4 id="%E6%A0%88%EF%BC%88Stack%EF%BC%89" name="%E6%A0%88%EF%BC%88Stack%EF%BC%89">
     栈（Stack）
    </h4>
    <p>
     栈是一种具有后进先出（LIFO）特性的数据结构，在程序运行时，系统会为每个线程分配一个栈空间，用于存储函数调用过程中的
     <strong>
      局部变量
     </strong>
     、
     <strong>
      函数参数（形参）
     </strong>
     、
     <strong>
      返回地址
     </strong>
     等（函数调用的上下文）信息。
    </p>
    <h5 id="%E5%B7%A5%E4%BD%9C%E5%8E%9F%E7%90%86" name="%E5%B7%A5%E4%BD%9C%E5%8E%9F%E7%90%86">
     工作原理
    </h5>
    <ul>
     <li>
      <strong>
       入栈（Push）
      </strong>
      ：当一个函数被调用时，会在栈上为该函数的局部变量和参数分配内存空间，这个过程称为入栈操作。
     </li>
     <li>
      <strong>
       出栈（Pop）
      </strong>
      ：当函数执行完毕返回时，系统会自动释放该函数在栈上分配的内存空间，这个过程称为出栈操作。
      <pre><code class="language-Go">package main

func add(a, b int) int {
    // 局部变量sum存储在栈上
    sum := a + b
    return sum
}

func main() {
    x := 1
    y := 2
    // 调用add函数时，参数a、b和局部变量sum会在栈上分配内存
    result := add(x, y)
    println(result)
}</code></pre>
      <p id="%E6%B3%A8%E6%84%8F%EF%BC%9A%E5%BD%A2%E5%8F%82a%E3%80%81b%E6%98%AF%E5%9C%A8%E6%A0%88%E4%B8%8A%EF%BC%8C%C2%A0%C2%A0" name="%E6%B3%A8%E6%84%8F%EF%BC%9A%E5%BD%A2%E5%8F%82a%E3%80%81b%E6%98%AF%E5%9C%A8%E6%A0%88%E4%B8%8A%EF%BC%8C%C2%A0%C2%A0">
       <strong>
        注意
       </strong>
       ：形参a、b是在栈上。
      </p>
     </li>
     <li>
      <h5 id="%E7%89%B9%E7%82%B9" name="%E7%89%B9%E7%82%B9">
       特点
      </h5>
     </li>
     <li>
      <strong>
       自动分配和释放
      </strong>
      ：
      <span style="background-color:#faa572">
       栈上的内存分配和释放是由系统自动完成的
      </span>
      ，函数调用结束后，栈上的内存可以立即回收。不需要程序员手动干预，因此栈的操作效率较高。
     </li>
     <li>
      <strong>
       内存空间连续
      </strong>
      ：
      <span style="background-color:#fbd4d0">
       栈上的内存空间是连续的，这使得栈的访问速度非常快
      </span>
      。
     </li>
     <li>
      <strong>
       大小有限
      </strong>
      ：每个线程的栈空间大小是有限的，如果函数调用层次过深或者局部变量占用的内存过大，可能会导致栈溢出错误。
     </li>
    </ul>
    <blockquote>
     <p id="%C2%A0" name="%C2%A0">
      <span style="color:#e6b223">
       <strong>
        知识点：
       </strong>
      </span>
     </p>
     <p>
      <strong>
       形式参数（简称形参）
      </strong>
      ：定义函数时，函数名后面括号中的变量名。由于它不是实际存在变量，所以又称虚拟变量。
     </p>
     <p>
      例如：当你定义函数void add(int a, int b)的时候,这里的a和b就是形参。
     </p>
     <p>
      <strong>
       实际参数（简称实参）
      </strong>
      ：调用函数时，函数名后面括号中的表达式。是在调用时传递给函数的参数. 实参可以是常量、变量、表达式、函数等。
     </p>
     <p>
      例如：当你调用函数add(1 ,2)的时候，这里的1和2就是实参。
     </p>
    </blockquote>
    <ul>
    </ul>
    <h4 id="%E5%A0%86%EF%BC%88Heap%EF%BC%89" name="%E5%A0%86%EF%BC%88Heap%EF%BC%89">
     堆（Heap）
    </h4>
    <p>
     堆是一块用于动态内存分配的内存区域，程序在运行时可以根据需要在堆上分配和释放内存，通常用于存储生命周期不确定的对象。在 Go 语言中，使用
     <code>
      <span style="background-color:#ed7976">
       new
      </span>
     </code>
     和
     <span style="color:#0d0016">
      <code>
       <span style="background-color:#ffd7b9">
        make
       </span>
      </code>
     </span>
     关键字或者
     <span style="background-color:#9c8ec1">
      直接创建引用类型（如切片、映射、通道等
     </span>
     <span style="background-color:#d7d8d9">
      ）
     </span>
     时，会在堆上分配内存。
    </p>
    <p>
     堆内存的管理相对复杂，需要垃圾回收（GC）来进行清理，回收速度不如栈快。
    </p>
    <h5 id="%E5%B7%A5%E4%BD%9C%E5%8E%9F%E7%90%86" name="%E5%B7%A5%E4%BD%9C%E5%8E%9F%E7%90%86">
     工作原理
    </h5>
    <p>
    </p>
    <h4>
    </h4>
    <p>
    </p>
    <ul>
     <li>
      <strong>
       内存分配
      </strong>
      ：当程序需要在堆上分配内存时，会向操作系统请求一块合适大小的内存空间。
     </li>
     <li>
      <strong>
       垃圾回收
      </strong>
      ：当程序不再使用堆上的某个内存块时，Go 语言的垃圾回收器（GC）会自动回收该内存块，以便后续使用。
      <pre><code class="language-Go">package main

import "fmt"

func main() {
    // 使用new关键字在堆上分配内存
    ptr := new(int)
    *ptr = 10
    fmt.Println(*ptr)

    // 创建切片，切片的底层数组在堆上分配内存
    slice := make([]int, 3)
    slice[0] = 1
    slice[1] = 2
    slice[2] = 3
    fmt.Println(slice)
}</code></pre>
      <h5>
       特点
      </h5>
     </li>
     <li>
      <strong>
       动态分配和释放
      </strong>
      ：堆上的内存分配和释放是动态的，需要程序员手动控制或者由垃圾回收器自动处理。
     </li>
     <li>
      <strong>
       内存空间不连续
      </strong>
      ：堆上的内存空间是不连续的，这使得堆的访问速度相对较慢。
     </li>
    </ul>
    <h4>
     总结
    </h4>
    <p>
     栈主要用于存储函数调用的上下文信息，具有自动分配和释放、内存空间连续、操作效率高但大小有限的特点；堆主要用于动态内存分配，具有动态分配和释放、内存空间不连续、大小灵活但访问速度相对较慢的特点。在 Go 语言中，编译器会根据变量的类型和使用场景自动决定将变量分配到栈上还是堆上。
    </p>
    <h3>
     补充1：内存逃逸
    </h3>
    <p>
     内存逃逸，就是程序在执行过程中，某些本应在栈上分配的变量，被“逃逸”到了堆上。
    </p>
    <p>
     原因：简单来说，就是 Go 的编译器在优化内存分配时，无法确定一个变量的生命周期和作用范围，导致它在堆上分配内存，而不是栈上。
    </p>
    <h4>
     变量逃逸的常见原因：
    </h4>
    <p>
     <strong>
      1、闭包（变量的生命周期超出函数作用域）：
     </strong>
     闭包是导致内存逃逸的一个典型场景。因为闭包会持有外部函数的引用，所以即使外部函数已经返回，闭包内部的变量依然可能在堆上存活。
    </p>
    <pre><code class="language-Go">func main() {
    f := func() int {
        x := 42 // 这个变量 x 会逃逸到堆上
        return x
    }
    fmt.Println(f())
}</code></pre>
    <p>
     这里，
     <code>
      x
     </code>
     是一个局部变量，但因为
     <code>
      f
     </code>
     是一个闭包，Go 编译器不能确定
     <code>
      x
     </code>
     的生命周期是否结束，所以
     <code>
      x
     </code>
     被分配到了堆上。
    </p>
    <p>
     <strong>
      2、指针传递
     </strong>
     ：如果你将一个局部变量的指针传递给了其他函数，Go 编译器会推测这个变量的生命周期已经超出了当前作用域，从而将它分配到堆上。
    </p>
    <pre><code class="language-Go">func foo(ptr *int) {
    fmt.Println(*ptr)
}

func main() {
    x := 42
    foo(&amp;x) // 这里传递了 x 的地址，x 会被分配到堆上
}</code></pre>
    <p>
     在这个例子中，
     <code>
      x
     </code>
     被传递给了
     <code>
      foo
     </code>
     函数，而
     <code>
      foo
     </code>
     是通过指针来访问
     <code>
      x
     </code>
     的。因为 Go 编译器无法确定
     <code>
      x
     </code>
     在
     <code>
      main
     </code>
     函数外部是否会继续使用，所以
     <code>
      x
     </code>
     被分配到了堆上。
    </p>
    <p>
     <strong>
      3、变量大小不确定(数组或切片的引用)
     </strong>
     ：切片类型在传递时会导致数据逃逸。因为切片实际上是对数组的一层封装，如果你在函数外部返回了切片，就可能导致底层的数组逃逸。
    </p>
    <pre><code class="language-Go">func createSlice() []int {
    arr := [3]int{1, 2, 3} // arr 会逃逸到堆上
    return arr[:]
}

func main() {
    slice := createSlice()
    fmt.Println(slice)
}</code></pre>
    <p>
     在这个示例中，使用
     <code>
      make
     </code>
     函数创建的切片 arr 的大小在运行时才能确定，因此 arr会逃逸到堆上。
    </p>
    <p>
     <strong>
      4、函数返回局部变量指针
     </strong>
    </p>
    <pre><code class="language-Go">package main

func escape() *int {
    x := 10
    return &amp;x
}

func main() {
    result := escape()
    println(*result)
}</code></pre>
    <p>
     在这个示例中，
     <code>
      escape
     </code>
     函数返回了局部变量
     <code>
      x
     </code>
     的指针，因此
     <code>
      x
     </code>
     会逃逸到堆上。
    </p>
    <blockquote>
     <p>
      <strong>
       闭包：
      </strong>
      闭包是一个函数，它可以访问并操作其外部作用域中的变量，即使该外部作用域已经执行完毕，这些变量也不会被销毁，如下图，匿名函数且是闭包：
     </p>
     <pre><code class="language-Go">// 这个例子中，outerFunction 返回的匿名函数引用了外部函数的局部变量 count，
// 形成了闭包。每次调用闭包时，count 的值会持续增加
func outerFunction() func() int {
    count := 0
    // 定义一个匿名函数，形成闭包
    return func() int {
        count++
        return count
    }
}

func main() {
    counter := outerFunction()
    fmt.Println(counter()) // 输出: 1
    fmt.Println(counter()) // 输出: 2
}
</code></pre>
     <p>
      <strong>
       匿名函数
      </strong>
      ：强调的是函数没有名称这一特性，常用于实现一些简单的、一次性的功能，比如作为回调函数传递给其他函数，或者在一些临时的逻辑处理中使用。
     </p>
     <pre><code class="language-Go">// 下面的匿名函数没有引用任何外部作用域的变量，所以它只是一个普通的匿名函数，不是闭包。
func main() {
    // 定义一个匿名函数并立即执行
    func() {
        fmt.Println("This is an anonymous function without closure.")
    }()
}</code></pre>
     <h5>
      具名函数作为闭包
     </h5>
     <pre><code class="language-Go">// 下面，inner 是一个具名函数，它引用了外部函数的局部变量 message，形成了闭包
func outer() func() {
    message := "Hello, Closure!"
    // 定义一个具名函数作为闭包
    func inner() {
        fmt.Println(message)
    }
    return inner
}

func main() {
    closure := outer()
    closure() // 输出: Hello, Closure!
}</code></pre>
     <p>
      匿名函数侧重于函数没有名称，而闭包侧重于函数对外部变量的引用和持有。匿名函数可以是闭包，也可以不是闭包；闭包可以是匿名函数，也可以是具名函数。理
     </p>
    </blockquote>
    <h3>
     补充2：new 与 make 的使用区别：
    </h3>
    <p>
     在 Go 语言里，
     <code>
      new
     </code>
     和
     <code>
      make
     </code>
     都用于内存分配，但它们的用途和使用场景存在明显差异
    </p>
    <h4>
     new 函数：
    </h4>
    <p>
     <code>
      new
     </code>
     是一个内置函数，其作用是为类型分配一片零值内存空间（只接收一个参数），并且返回指向该接收参数内存空间的指针。
    </p>
    <pre><code class="language-Go">func new(Type) *Type</code></pre>
    <p>
     这里的
     <code>
      Type
     </code>
     代表任意类型，
     <code>
      new
     </code>
     函数会返回一个指向该类型零值的指针。
    </p>
    <pre><code class="language-Go"> // 使用 new 函数为 int 类型分配内存
    numPtr := new(int)
    // 此时 numPtr 指向的 int 类型变量的值为 0（int 类型的零值）
    fmt.Println(*numPtr) 

    // 修改 numPtr 指向的变量的值
    *numPtr = 10
    fmt.Println(*numPtr) 

    // 使用 new 函数为结构体类型分配内存
    type Person struct {
        Name string
        Age  int
    }
    personPtr := new(Person)
    // 结构体的字段被初始化为零值
    fmt.Printf("Name: %s, Age: %d\n", personPtr.Name, personPtr.Age) </code></pre>
    <h5>
     代码解释
    </h5>
    <ul>
     <li>
      当调用
      <code>
       new(int)
      </code>
      时，会为
      <code>
       int
      </code>
      类型分配一块内存空间，初始值为
      <code>
       0
      </code>
      ，并返回指向该内存空间的指针
      <code>
       numPtr
      </code>
      。
     </li>
     <li>
      对于结构体类型
      <code>
       Person
      </code>
      ，调用
      <code>
       new(Person)
      </code>
      会为该结构体分配内存，结构体的字段会被初始化为各自类型的零值，然后可以通过指针修改这些字段的值。
     </li>
    </ul>
    <h4>
     <code>
      make
     </code>
     函数
    </h4>
    <p>
     <code>
      make
     </code>
     也是一个内置函数，不过它只能用于创建并初始化
     <code>
      slice
     </code>
     （切片）、
     <code>
      map
     </code>
     （映射）和
     <code>
      channel
     </code>
     （通道）这三种引用类型。
    </p>
    <ul>
     <li>
      <pre><code class="language-Go">func make(t Type, size ...IntegerType) Type</code></pre>
      <p>
       这里的
       <code>
        t
       </code>
       必须是
       <code>
        slice
       </code>
       、
       <code>
        map
       </code>
       或
       <code>
        channel
       </code>
       类型，
       <code>
        size
       </code>
       是可选参数，用于指定初始大小
      </p>
     </li>
    </ul>
    <pre><code class="language-Go">package main

import "fmt"

func main() {
    // 创建一个长度为 3，容量为 5 的整数切片
    slice := make([]int, 3, 5)
    fmt.Println("切片长度:", len(slice)) 
    fmt.Println("切片容量:", cap(slice)) 
    fmt.Println(slice) 

    // 修改切片元素的值
    slice[0] = 1
    slice[1] = 2
    slice[2] = 3
    fmt.Println(slice) 


    // 创建一个字符串到整数的映射
    m := make(map[string]int)
    // 向映射中添加键值对
    m["apple"] = 1
    m["banana"] = 2
    fmt.Println(m) 

     // 创建一个无缓冲的整数通道
    ch := make(chan int)
    go func() {
        // 向通道发送数据
        ch &lt;- 10
    }()
    // 从通道接收数据
    num := &lt;-ch
    fmt.Println(num) 

    // 创建一个有缓冲的整数通道，缓冲区大小为 3
    ch2 := make(chan int, 3)
}</code></pre>
    <h4>
     总结
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        new
       </code>
      </strong>
      ：主要用于为任意类型分配零值内存并返回指针，通常用于需要显式管理指针的场景。
     </li>
     <li>
      <strong>
       <code>
        make
       </code>
      </strong>
      ：专门用于创建并初始化
      <code>
       slice
      </code>
      、
      <code>
