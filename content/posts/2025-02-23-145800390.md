     一.MPU6050模块介绍
    </h2>
    <h3 id="%C2%A01.1MPU6050%E7%AE%80%E4%BB%8B" name="%C2%A01.1MPU6050%E7%AE%80%E4%BB%8B">
     1.1MPU6050简介
    </h3>
    <p>
     MPU6050是一款高性能的六轴运动传感器，集成了三轴加速度计和三轴陀螺仪。它能够测量加速度和角速度，广泛应用于姿态测量、运动检测、机器人控制等领域。MPU6050的主要特性如下：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        三轴加速度计
       </strong>
       ：测量范围为
       <u>
        ±2g/±4g/±8g/±16g
       </u>
       （可选 单位g为重力加速度）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        三轴陀螺仪
       </strong>
       ：测量范围为
       <u>
        ±250°/s/±500°/s/±1000°/s/±2000°/s
       </u>
       （可选）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数字运动处理器（DMP）
       </strong>
       ：支持复杂的运动处理算法，如姿态解算。
      </p>
     </li>
     <li>
      <p>
       <strong>
        IIC接口
       </strong>
       ：支持标准IIC通信协议，易于与微控制器连接。
      </p>
     </li>
     <li>
      <p>
       <strong>
        低功耗
       </strong>
       ：适合电池供电的便携式设备。
      </p>
     </li>
     <li>
      <p>
       <strong>
        高精度
       </strong>
       ：能够提供高精度的加速度和角速度数据。
      </p>
     </li>
    </ul>

     2.2对MPU6050寄存器进行读写
    </h3>
    <h4 id="2.2.1%20%E5%86%99%E5%85%A5%E5%AF%84%E5%AD%98%E5%99%A8" name="2.2.1%20%E5%86%99%E5%85%A5%E5%AF%84%E5%AD%98%E5%99%A8">
     2.2.1 写入寄存器
    </h4>
    <p>
     通过IIC接口向MPU6050的寄存器写入数据，可以配置传感器的工作模式、量程、采样率等参数。以下是写入寄存器的代码实现：
    </p>
    <pre><code>#define MPU6050_Addr 0xD0

void MPU6050_WriteReg(uint8_t RegAddress, uint8_t Data) {
    IIC_Start();
    IIC_SendByte(MPU6050_Addr); // 发送设备地址（写操作）
    IIC_ReceiveAck();
    IIC_SendByte(RegAddress); // 发送寄存器地址
    IIC_ReceiveAck();
    IIC_SendByte(Data); // 发送数据
    IIC_ReceiveAck();
    IIC_Stop();
}</code></pre>
    <h4 id="2.2.2%E8%AF%BB%E5%8F%96%E5%AF%84%E5%AD%98%E5%99%A8" name="2.2.2%E8%AF%BB%E5%8F%96%E5%AF%84%E5%AD%98%E5%99%A8">
     2.2.2读取寄存器
    </h4>
    <p>
     通过IIC接口从MPU6050的寄存器读取数据，可以获取传感器的配置状态或测量数据。以下是读取寄存器的代码实现：
    </p>
    <pre><code>uint8_t MPU6050_ReadReg(uint8_t RegAddress) {
    uint8_t Data;
    IIC_Start();
    IIC_SendByte(MPU6050_Addr); // 发送设备地址（写操作）
    IIC_ReceiveAck();
    IIC_SendByte(RegAddress); // 发送寄存器地址
    IIC_ReceiveAck();

    IIC_Start();
    IIC_SendByte(MPU6050_Addr | 0x01); // 发送设备地址（读操作）
    IIC_ReceiveAck();
    Data = IIC_ReceiveByte(); // 读取数据
    IIC_SendAck(1); // 发送NACK，结束读取
    IIC_Stop();

    return Data;
}</code></pre>
    <h3 id="2.3%C2%A0%E5%88%9D%E5%A7%8B%E5%8C%96MPU6050" name="2.3%C2%A0%E5%88%9D%E5%A7%8B%E5%8C%96MPU6050">
     2.3 初始化MPU6050
    </h3>
    <h4 id="2.3.1%20%E8%AE%BE%E7%BD%AE%E5%B7%A5%E4%BD%9C%E6%A8%A1%E5%BC%8F" name="2.3.1%20%E8%AE%BE%E7%BD%AE%E5%B7%A5%E4%BD%9C%E6%A8%A1%E5%BC%8F">
     2.3.1 设置工作模式
    </h4>
    <p>
     通过PWR_MGMT_1寄存器设置MPU6050的工作模式。通常在初始化时，先复位设备，然后将其设置为正常工作模式。
    </p>
    <pre><code>void MPU6050_SetPowerMode(uint8_t mode) 
{
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, mode);
}</code></pre>
    <h4 id="2.3.2%20%E9%85%8D%E7%BD%AE%E9%87%87%E6%A0%B7%E7%8E%87" name="2.3.2%20%E9%85%8D%E7%BD%AE%E9%87%87%E6%A0%B7%E7%8E%87">
     2.3.2 配置采样率
    </h4>
    <p>
     通过SMPLRT_DIV寄存器设置采样率分频值，从而控制数据的采样频率。
    </p>
    <pre><code>void MPU6050_SetSampleRate(uint8_t div) 
{
    MPU6050_WriteReg(MPU6050_SMPLRT_DIV, div);
}</code></pre>
    <h4 id="2.3.3%20%E5%90%AF%E7%94%A8%E4%BC%A0%E6%84%9F%E5%99%A8" name="2.3.3%20%E5%90%AF%E7%94%A8%E4%BC%A0%E6%84%9F%E5%99%A8">
     2.3.3 启用传感器
    </h4>
    <p>
     通过PWR_MGMT_1寄存器启用加速度计和陀螺仪。
    </p>
    <pre><code>void MPU6050_EnableSensors(void) 
{
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, 0x00); // 启用设备
}</code></pre>
    <h3 id="2.4MPU6050%E6%95%B0%E6%8D%AE%E8%AF%BB%E5%8F%96" name="2.4MPU6050%E6%95%B0%E6%8D%AE%E8%AF%BB%E5%8F%96">
     2.4MPU6050数据读取
    </h3>
    <h4 id="2.4.1%20%E8%AF%BB%E5%8F%96%E5%8A%A0%E9%80%9F%E5%BA%A6%E6%95%B0%E6%8D%AE" name="2.4.1%20%E8%AF%BB%E5%8F%96%E5%8A%A0%E9%80%9F%E5%BA%A6%E6%95%B0%E6%8D%AE">
     2.4.1 读取加速度数据
    </h4>
    <p>
     加速度数据存储在寄存器
     <code>
      0x3B
     </code>
     到
     <code>
      0x40
     </code>
     中，分别表示加速度计的X、Y、Z轴数据。
    </p>
    <pre><code>void MPU6050_ReadAccel(int16_t *ax, int16_t *ay, int16_t *az) 
{
    uint8_t buffer[6]; // 创建一个6字节的缓冲区，用于存储从MPU6050读取的加速度数据

    // 逐个读取加速度计的X、Y、Z轴的高字节和低字节数据
    buffer[0] = MPU6050_ReadReg(MPU6050_ACCEL_XOUT_H); // X轴加速度数据的高字节
    buffer[1] = MPU6050_ReadReg(MPU6050_ACCEL_XOUT_L); // X轴加速度数据的低字节
    buffer[2] = MPU6050_ReadReg(MPU6050_ACCEL_YOUT_H); // Y轴加速度数据的高字节
    buffer[3] = MPU6050_ReadReg(MPU6050_ACCEL_YOUT_L); // Y轴加速度数据的低字节
    buffer[4] = MPU6050_ReadReg(MPU6050_ACCEL_ZOUT_H); // Z轴加速度数据的高字节
    buffer[5] = MPU6050_ReadReg(MPU6050_ACCEL_ZOUT_L); // Z轴加速度数据的低字节

    // 将高字节和低字节组合成完整的16位加速度数据
    *ax = (buffer[0] &lt;&lt; 8) | buffer[1]; // 将X轴的高字节左移8位后与低字节进行按位或操作，得到完整的X轴加速度数据
    *ay = (buffer[2] &lt;&lt; 8) | buffer[3]; // 同理，得到Y轴加速度数据
    *az = (buffer[4] &lt;&lt; 8) | buffer[5]; // 同理，得到Z轴加速度数据
}</code></pre>
    <h4 id="2.4.2%20%E8%AF%BB%E5%8F%96%E9%99%80%E8%9E%BA%E4%BB%AA%E6%95%B0%E6%8D%AE" name="2.4.2%20%E8%AF%BB%E5%8F%96%E9%99%80%E8%9E%BA%E4%BB%AA%E6%95%B0%E6%8D%AE">
     2.4.2 读取陀螺仪数据
    </h4>
    <p>
     陀螺仪数据存储在寄存器
     <code>
      0x43
     </code>
     到
     <code>
      0x48
     </code>
     中，分别表示陀螺仪的X、Y、Z轴数据。
    </p>
    <pre><code>void MPU6050_ReadGyro(int16_t *gx, int16_t *gy, int16_t *gz) 
{
    uint8_t buffer[6]; // 创建一个6字节的缓冲区，用于存储从MPU6050读取的陀螺仪数据

    // 逐个读取陀螺仪的X、Y、Z轴的高字节和低字节数据
    buffer[0] = MPU6050_ReadReg(MPU6050_GYRO_XOUT_H); // X轴陀螺仪数据的高字节
    buffer[1] = MPU6050_ReadReg(MPU6050_GYRO_XOUT_L); // X轴陀螺仪数据的低字节
    buffer[2] = MPU6050_ReadReg(MPU6050_GYRO_YOUT_H); // Y轴陀螺仪数据的高字节
    buffer[3] = MPU6050_ReadReg(MPU6050_GYRO_YOUT_L); // Y轴陀螺仪数据的低字节
    buffer[4] = MPU6050_ReadReg(MPU6050_GYRO_ZOUT_H); // Z轴陀螺仪数据的高字节
    buffer[5] = MPU6050_ReadReg(MPU6050_GYRO_ZOUT_L); // Z轴陀螺仪数据的低字节

    // 将高字节和低字节组合成完整的16位陀螺仪数据
    *gx = (buffer[0] &lt;&lt; 8) | buffer[1]; // 将X轴的高字节左移8位后与低字节进行按位或操作，得到完整的X轴陀螺仪数据
    *gy = (buffer[2] &lt;&lt; 8) | buffer[3]; // 同理，得到Y轴陀螺仪数据
    *gz = (buffer[4] &lt;&lt; 8) | buffer[5]; // 同理，得到Z轴陀螺仪数据
}</code></pre>
    <h4 id="2.4.3%20%E6%95%B0%E6%8D%AE%E6%A0%BC%E5%BC%8F%E8%BD%AC%E6%8D%A2" name="2.4.3%20%E6%95%B0%E6%8D%AE%E6%A0%BC%E5%BC%8F%E8%BD%AC%E6%8D%A2">
     2.4.3 数据格式转换
    </h4>
    <p>
     将读取到的原始数据转换为实际的物理量。
    </p>
    <pre><code>float MPU6050_ConvertAccel(int16_t value, uint8_t range) 
{
    float factor; // 用于存储转换因子
    // 根据加速度计的量程选择合适的转换因子
    switch (range) {
        case 0x00: factor = 16384.0; break; // ±2g，转换因子为16384
        case 0x08: factor = 8192.0; break;  // ±4g，转换因子为8192
        case 0x10: factor = 4096.0; break;  // ±8g，转换因子为4096
        case 0x18: factor = 1365.3; break;  // ±16g，转换因子为1365.3
        default: factor = 16384.0; break;    // 默认量程为±2g
    }
    // 将原始加速度值转换为g单位
    return value / factor;
}

float MPU6050_ConvertGyro(int16_t value, uint8_t range) 
{
    float factor; // 用于存储转换因子
    // 根据陀螺仪的量程选择合适的转换因子
    switch (range) {
        case 0x00: factor = 131.0; break;  // ±250°/s，转换因子为131
        case 0x08: factor = 65.5; break;   // ±500°/s，转换因子为65.5
        case 0x10: factor = 32.8; break;   // ±1000°/s，转换因子为32.8
        case 0x18: factor = 16.4; break;   // ±2000°/s，转换因子为16.4
        default: factor = 131.0; break;     // 默认量程为±250°/s
    }
    // 将原始陀螺仪值转换为°/s单位
    return value / factor;
}</code></pre>

     2.5.1 初始化函数
    </h4>
    <p>
     初始化MPU6050，设置工作模式、采样率和传感器量程。
    </p>
    <pre><code>void MPU6050_Init(void) 
{
    // 复位MPU6050
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, 0x80);
    MPU6050_WriteReg(MPU6050_PWR_MGMT_1, 0x00);

    // 设置陀螺计量程为±250°/s
    MPU6050_WriteReg(MPU6050_GYRO_CONFIG, 0x00);

    // 设置加速度计量程为±2g
    MPU6050_WriteReg(MPU6050_ACCEL_CONFIG, 0x00);

    // 设置采样率分频值为0
    MPU6050_WriteReg(MPU6050_SMPLRT_DIV, 0x00);
}</code></pre>
    <h4 id="2.5.2%20%E6%95%B0%E6%8D%AE%E8%AF%BB%E5%8F%96%E5%87%BD%E6%95%B0" name="2.5.2%20%E6%95%B0%E6%8D%AE%E8%AF%BB%E5%8F%96%E5%87%BD%E6%95%B0">
     2.5.2 数据读取函数
    </h4>
    <p>
     读取加速度和陀螺仪数据，并转换为实际的物理量。
    </p>
    <pre><code>void MPU6050_ReadSensors(float *ax, float *ay, float *az, float *gx, float *gy, float *gz) 
{
    int16_t raw_ax, raw_ay, raw_az;  // 用于存储原始加速度数据
    int16_t raw_gx, raw_gy, raw_gz;  // 用于存储原始陀螺仪数据

    // 读取加速度计的原始数据
    MPU6050_ReadAccel(&amp;raw_ax, &amp;raw_ay, &amp;raw_az);

    // 读取陀螺仪的原始数据
    MPU6050_ReadGyro(&amp;raw_gx, &amp;raw_gy, &amp;raw_gz);

    // 将加速度计的原始数据转换为浮点数形式（单位：g）
    *ax = MPU6050_ConvertAccel(raw_ax, 0x00); // ±2g量程
    *ay = MPU6050_ConvertAccel(raw_ay, 0x00); // ±2g量程
    *az = MPU6050_ConvertAccel(raw_az, 0x00); // ±2g量程

    // 将陀螺仪的原始数据转换为浮点数形式（单位：°/s）
    *gx = MPU6050_ConvertGyro(raw_gx, 0x00); // ±250°/s量程
    *gy = MPU6050_ConvertGyro(raw_gy, 0x00); // ±250°/s量程
    *gz = MPU6050_ConvertGyro(raw_gz, 0x00); // ±250°/s量程
}</code></pre>

    <h2 id="%E4%B8%89.%20%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95" name="%E4%B8%89.%20%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95">
     三. 姿态解算
    </h2>
    <p>
     为什么用姿态解算计算设备姿态角👀因为计算设备姿态角上一般所选用的惯性传感器
     <span style="color:#0d0016">
      ，都是mems器件
     </span>
     ，精度相对较差，同时陀螺仪、加速度计、地磁计单个传感器无法得到满意的姿态角信息，所以需要一些融合算法，进行姿态估计。
    </p>
    <p>
     姿态解算的
     <strong>
      目标
     </strong>
     是通过融合MPU6050的加速度计和陀螺仪数据，得到一个准确的姿态估计的👉即计算出设备在三维空间（坐标系）中的姿态角（如俯仰角、横滚角和偏航角）。姿态角是描述物体在空间中旋转状态的三个角度，通常用于
     <u>
      飞控、机器人、VR设备
     </u>
     等领域。
    </p>
    <h3 id="3.1%E5%A7%BF%E6%80%81%E8%A7%92%E4%B8%8E%E5%9D%90%E6%A0%87%E7%B3%BB" name="3.1%E5%A7%BF%E6%80%81%E8%A7%92%E4%B8%8E%E5%9D%90%E6%A0%87%E7%B3%BB">
     3.1姿态角与坐标系
    </h3>
    <h4 id="3.1.1%20%E5%A7%BF%E6%80%81%E8%A7%92%E7%9A%84%E5%AE%9A%E4%B9%89" name="3.1.1%20%E5%A7%BF%E6%80%81%E8%A7%92%E7%9A%84%E5%AE%9A%E4%B9%89">
     3.1.1 姿态角的定义
    </h4>
    <p>
     姿态角描述了设备的机体坐标系相对于惯性坐标系的旋转关系。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        俯仰角（Pitch）
       </strong>
       ：表示设备绕X轴的旋转角度，范围为-90°到+90°。俯仰角描述了设备相对于水平面的上下倾斜程度。
      </p>
     </li>
     <li>
      <p>
       <strong>
        横滚角（Roll）
       </strong>
       ：表示设备绕Y轴的旋转角度，范围为-180°到+180°。横滚角描述了设备相对于水平面的左右倾斜程度。
      </p>
     </li>
     <li>
      <p>
       <strong>
        偏航角（Yaw）
       </strong>
       ：表示设备绕Z轴的旋转角度，范围为-180°到+180°。偏航角描述了设备在水平面上的旋转方向。
      </p>
     </li>
    </ul>
    <p>
     <u>
      <strong>
       <span style="color:#fe2c24">
        注：
       </span>
      </strong>
     </u>
     在姿态解算中，姿态角的定义通常是以惯性坐标系（世界坐标系）作为参考的。惯性坐标系是一个固定在空间中的参考系，通常以地球的重力方向为基准。
    </p>
    <p>
     <img alt="" height="716" src="https://i-blog.csdnimg.cn/direct/b7ca3ae0cf3442f68c65c5fb87c3cbaa.png" width="1105"/>
    </p>
    <h4 id="3.1.2%20%E5%9D%90%E6%A0%87%E7%B3%BB%E7%9A%84%E6%A6%82%E5%BF%B5" name="3.1.2%20%E5%9D%90%E6%A0%87%E7%B3%BB%E7%9A%84%E6%A6%82%E5%BF%B5">
     3.1.2 坐标系的概念
    </h4>
    <p>
     在姿态解算中，坐标系的选择至关重要。通常使用以下两种坐标系：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        惯性坐标系（世界坐标系）
       </strong>
       ：固定在空间中的坐标系，通常以地球的重力方向为Z轴，水平面为XY平面。
      </p>
     </li>
     <li>
      <p>
       <strong>
        机体坐标系（传感器坐标系）
       </strong>
       ：固定在设备上的坐标系，通常以设备的几何中心为原点，X轴指向设备的前方向，Y轴指向设备的右方向，Z轴指向设备的下方向。
      </p>
     </li>
    </ul>
    <p>
     姿态角描述了机体坐标系相对于惯性坐标系的旋转关系。通过姿态解算，可以将机体坐标系下的测量数据转换到惯性坐标系下，从而实现对设备姿态的描述。➡️抽象来说：姿态解算是“
     <strong>
      机体坐标系
     </strong>
     ”与“
     <strong>
      惯性坐标系
     </strong>
     ”之间的转换关系！
    </p>
    <h3 id="3.2%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95" name="3.2%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95">
     3.2姿态解算算法
    </h3>
    <p>
     在姿态解算中，姿态可以通过多种方式表示，常见的有
     <u>
      <strong>
       欧拉角
      </strong>
      、
      <strong>
       四元数
      </strong>
      和
     </u>
     <strong>
      <u>
       旋转矩阵
      </u>
      。
     </strong>
     每种表示方法都有其特点和应用场景，且每个姿态表示方法可以
     <strong>
      相互转换。
     </strong>
    </p>
    <p>
     在姿态解算中，常见的算法包括：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        四元数法
       </strong>
       ：通过四元数融合加速度计和陀螺仪数据，计算出设备的姿态。
      </p>
     </li>
     <li>
      <p>
       <strong>
        互补滤波法
       </strong>
       ：结合加速度计和陀螺仪数据，通过互补滤波器修正姿态角。
      </p>
     </li>
     <li>
      <p>
       <strong>
        卡尔曼滤波法
       </strong>
       ：一种更高级的滤波算法，能够更精确地融合传感器数据，适用于复杂的姿态解算
      </p>
     </li>
    </ol>
    <blockquote>
     <p>
      <span style="background-color:#f9eda6">
       关于上面的所提到的每一个姿态表示方法和姿态解算算法并没有逐个的展开讲解，如果大家感兴趣的话可以自行查阅，这里我们主要讲
      </span>
      <span style="color:#fe2c24">
       <strong>
        <span style="background-color:#f9eda6">
         四元数法
        </span>
       </strong>
      </span>
      <span style="background-color:#f9eda6">
       应用！
      </span>
     </p>
    </blockquote>
    <h4 id="3.2.1%E5%9B%9B%E5%85%83%E6%95%B0%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E5%AE%9E%E7%8E%B0" name="3.2.1%E5%9B%9B%E5%85%83%E6%95%B0%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E5%AE%9E%E7%8E%B0">
     3.2.1四元数姿态解算实现
    </h4>
    <p>
     四元数是一种用于表示三维旋转的数学工具，由四个分量组成：q0​,q1​,q2​,q3​。其中，q0​ 是实部，q1​,q2​,q3​ 是虚部。四元数可以表示为：
     <strong>
      q=q0​+q1​i+q2​j+q3​k
     </strong>
    </p>
    <p>
     相比欧拉角，它具有更好的数值稳定性和抗奇异性的特点，避免欧拉角中的“万向锁”问题。以下是四元数姿态解算的
     <strong>
      基本步骤：
     </strong>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        初始化四元数
       </strong>
       ：初始状态下，四元数设为单位四元数，即 q0​=1,q1​=q2​=q3​=0。
      </p>
     </li>
     <li>
      <p>
       <strong>
        读取传感器数据
       </strong>
       ：从MPU6050读取加速度计和陀螺仪的原始数据，并将陀螺仪数据转换为弧度。
      </p>
     </li>
     <li>
      <p>
       <strong>
        加速度计归一化
       </strong>
       ：将加速度计数据归一化，以提取重力方向。
      </p>
     </li>
     <li>
      <p>
       <strong>
        计算姿态误差
       </strong>
       ：通过加速度计和四元数推导出的重力方向的差异，计算姿态误差。
      </p>
     </li>
     <li>
      <p>
       <strong>
        互补滤波
       </strong>
       ：将姿态误差反馈到四元数更新中，结合陀螺仪数据，通过互补滤波器修正四元数。
      </p>
     </li>
     <li>
      <p>
       <strong>
        四元数更新
       </strong>
       ：通过微分方程更新四元数，并进行归一化处理。
      </p>
     </li>
     <li>
      <p>
       <strong>
        计算欧拉角
       </strong>
       ：将四元数转换为欧拉角，得到俯仰角、横滚角和偏航角。
      </p>
     </li>
    </ol>
    <h4 id="3.2.2%E5%9B%9B%E5%85%83%E6%95%B0%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95" name="3.2.2%E5%9B%9B%E5%85%83%E6%95%B0%E5%A7%BF%E6%80%81%E8%A7%A3%E7%AE%97%E7%AE%97%E6%B3%95">
     3.2.2四元数姿态解算算法
    </h4>
    <h5 id="3.2.2.1%E5%88%9D%E5%A7%8B%E5%8C%96%E5%9B%9B%E5%85%83%E6%95%B0" name="3.2.2.1%E5%88%9D%E5%A7%8B%E5%8C%96%E5%9B%9B%E5%85%83%E6%95%B0">
     3.2.2.1初始化四元数
    </h5>
    <p>
     初始化四元数为单位四元数，表示
     <strong>
      初始姿态为零
     </strong>
     。
    </p>
    <pre><code>//定义一个关于四元数的结构体变量
typedef struct 
{
    float q0, q1, q2, q3;
} Quaternion;

Quaternion q = {1.0f, 0.0f, 0.0f, 0.0f};  // 初始化四元数</code></pre>
    <h5 id="3.2.2.2%E8%AF%BB%E5%8F%96%E4%BC%A0%E6%84%9F%E5%99%A8%E6%95%B0%E6%8D%AE" name="3.2.2.2%E8%AF%BB%E5%8F%96%E4%BC%A0%E6%84%9F%E5%99%A8%E6%95%B0%E6%8D%AE">
     3.2.2.2读取传感器数据
    </h5>
    <p>
     从MPU6050读取加速度计和陀螺仪的原始数据，并将陀螺仪数据
     <strong>
      转换为弧度。
     </strong>
    </p>
    <pre><code>void MPU6050_ReadSensors(float *ax, float *ay, float *az, float *gx, float *gy, float *gz)
{
    int16_t raw_ax, raw_ay, raw_az;
    int16_t raw_gx, raw_gy, raw_gz;

    MPU6050_ReadAccel(&amp;raw_ax, &amp;raw_ay, &amp;raw_az);
    MPU6050_ReadGyro(&amp;raw_gx, &amp;raw_gy, &amp;raw_gz);

    *ax = MPU6050_ConvertAccel(raw_ax, 0x00);  // ±2g
    *ay = MPU6050_ConvertAccel(raw_ay, 0x00);
    *az = MPU6050_ConvertAccel(raw_az, 0x00);

    *gx = MPU6050_ConvertGyro(raw_gx, 0x00);  // ±250°/s
    *gy = MPU6050_ConvertGyro(raw_gy, 0x00);
    *gz = MPU6050_ConvertGyro(raw_gz, 0x00);
}</code></pre>
    <h5 id="3.2.2.3%20%E5%8A%A0%E9%80%9F%E5%BA%A6%E8%AE%A1%E5%BD%92%E4%B8%80%E5%8C%96" name="3.2.2.3%20%E5%8A%A0%E9%80%9F%E5%BA%A6%E8%AE%A1%E5%BD%92%E4%B8%80%E5%8C%96">
     3.2.2.3 加速度计归一化
    </h5>
    <p>
     将加速度计数据归一化，提取重力方向
    </p>
    <pre><code>void NormalizeAccel(float *ax, float *ay, float *az) 
{
    float norm = sqrt(*ax * *ax + *ay * *ay + *az * *az);
    *ax /= norm;
    *ay /= norm;
    *az /= norm;
}</code></pre>
    <h5 id="3.2.2.4%E8%AE%A1%E7%AE%97%E5%A7%BF%E6%80%81%E8%AF%AF%E5%B7%AE" name="3.2.2.4%E8%AE%A1%E7%AE%97%E5%A7%BF%E6%80%81%E8%AF%AF%E5%B7%AE">
     3.2.2.4计算姿态误差
    </h5>
    <p>
     通过加速度计和四元数推导出的重力方向的
     <strong>
      差异
     </strong>
     ，计算姿态误差。
    </p>
    <pre><code>void ComputeError(float ax, float ay, float az, float *error_x, float *error_y, float *error_z) 
{
    float gravity_x = 2 * (q.q1 * q.q3 - q.q0 * q.q2);
    float gravity_y = 2 * (q.q0 * q.q1 + q.q2 * q.q3);
    float gravity_z = 1 - 2 * (q.q1 * q.q1 + q.q2 * q.q2);

    *error_x = (ay * gravity_z - az * gravity_y);
    *error_y = (az * gravity_x - ax * gravity_z);
    *error_z = (ax * gravity_y - ay * gravity_x);
}</code></pre>
    <h5 id="3.2.2.5%E5%9B%9B%E5%85%83%E6%95%B0%E6%9B%B4%E6%96%B0" name="3.2.2.5%E5%9B%9B%E5%85%83%E6%95%B0%E6%9B%B4%E6%96%B0">
     3.2.2.5四元数更新
    </h5>
    <p>
     结合陀螺仪数据，通过
     <strong>
      微分方程
     </strong>
     更新四元数，并进行归一化处理，避免数值误差会越积越大。
    </p>
    <pre><code>void UpdateQuaternion(float gx, float gy, float gz, float error_x, float error_y, float error_z, float dt) 
{
    float Kp = 0.5f;  // 互补滤波系数

    float q0_dot = -q.q1 * gx - q.q2 * gy - q.q3 * gz;
    float q1_dot = q.q0 * gx - q.q3 * gy + q.q2 * gz;
    float q2_dot = q.q3 * gx + q.q0 * gy - q.q1 * gz;
    float q3_dot = -q.q2 * gx + q.q1 * gy + q.q0 * gz;

    q.q0 += (q0_dot + Kp * error_x) * dt;
    q.q1 += (q1_dot + Kp * error_y) * dt;
    q.q2 += (q2_dot + Kp * error_z) * dt;
    q.q3 += (q3_dot) * dt;

    // 归一化四元数，避免数值误差会越积越大
    float norm = sqrt(q.q0 * q.q0 + q.q1 * q.q1 + q.q2 * q.q2 + q.q3 * q.q3);
    q.q0 /= norm;
    q.q1 /= norm;
    q.q2 /= norm;
    q.q3 /= norm;
}</code></pre>
    <h5 id="3.2.2.6%E8%AE%A1%E7%AE%97%E6%AC%A7%E6%8B%89%E8%A7%92" name="3.2.2.6%E8%AE%A1%E7%AE%97%E6%AC%A7%E6%8B%89%E8%A7%92">
     3.2.2.6计算欧拉角
    </h5>
    <p>
     将四元数转换为欧拉角，得到
     <strong>
      俯仰角、横滚角和偏航角。
     </strong>
    </p>
    <pre><code>void ComputeEulerAngles(float *pitch, float *roll, float *yaw) 
{
    *roll = atan2(2 * (q.q2 * q.q3 + q.q0 * q.q1), q.q0 * q.q0 - q.q1 * q.q1 - q.q2 * q.q2 + q.q3 * q.q3);
    *pitch = asin(-2 * (q.q1 * q.q3 - q.q0 * q.q2));
    *yaw = atan2(2 * (q.q1 * q.q2 + q.q0 * q.q3), q.q0 * q.q0 + q.q1 * q.q1 - q.q2 * q.q2 - q.q3 * q.q3);

    *roll *= RtA;  // 弧度转角度
    *pitch *= RtA;
    *yaw *= RtA;
}</code></pre>
    <h5 id="3.2.2.7%E4%B8%BB%E5%87%BD%E6%95%B0" name="3.2.2.7%E4%B8%BB%E5%87%BD%E6%95%B0">
     3.2.2.7主函数
    </h5>
    <p>
     将上述步骤整合到主函数中，
     <strong>
      实现完整的姿态解算。
     </strong>
    </p>
    <pre><code>#include "mpu6050.h"
#include "math.h"

#define PI 3.1415926535f
#define RtA 57.2957795f  // 弧度转角度
#define AtR 0.0174532925f  // 角度转弧度

typedef struct 
{
    float q0, q1, q2, q3;
} Quaternion;

Quaternion q = {1.0f, 0.0f, 0.0f, 0.0f};  // 初始化四元数

void GetAngles(float *pitch, float *roll, float *yaw, float dt) 
{
    float ax, ay, az, gx, gy, gz;
    MPU6050_ReadSensors(&amp;ax, &amp;ay, &amp;az, &amp;gx, &amp;gy, &amp;gz);

    NormalizeAccel(&amp;ax, &amp;ay, &amp;az);

    float error_x, error_y, error_z;
    ComputeError(ax, ay, az, &amp;error_x, &amp;error_y, &amp;error_z);

    UpdateQuaternion(gx, gy, gz, error_x, error_y, error_z, dt);

    ComputeEulerAngles(pitch, roll, yaw);
}

{

        float pitch, roll, yaw;

        // 打印姿态角
        printf("Pitch: %.2f, Roll: %.2f, Yaw: %.2f\r\n", pitch, roll, yaw);

        // 延时
        delay_ms(10);
    }
}</code></pre>

     3.3注意事项
    </h3>
    <p>
     小伙伴们，在
     <strong>
      动手实现姿态解算
     </strong>
     的时候，有几个要点得留心：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        硬件连接要稳
       </strong>
       ：MPU6050和STM32的IIC通信线路（SDA和SCL）一定要接对，不然通信可能会出问题。
      </p>
     </li>
     <li>
      <p>
       <strong>
        传感器得校准
       </strong>
       ：加速度计和陀螺仪用之前最好校准一下，不然数据可能会不准。静止状态下多读几次数据，算个平均值，这样能更准。
      </p>
     </li>
     <li>
      <p>
       <strong>
        数据别急躁
       </strong>
       ：加速度计和陀螺仪的数据可能会有点“毛躁”，用低通滤波器或者滑动平均滤波器给它“梳梳毛”，数据会更平滑。
      </p>
     </li>
     <li>
      <p>
       <strong>
        算法参数要调
       </strong>
       ：四元数解算里的互补滤波器增益（比如
       <code>
        Kp
       </code>
       ）很重要，得根据你的实际需求调整，不然解算出来的姿态可能会“飘”。
      </p>
     </li>
     <li>
      <p>
       <strong>
        代码要细心
       </strong>
       ：四元数更新后一定要归一化，不然数值误差会越积越大。还有，采样频率别太低，不然实时性会受影响。
      </p>
     </li>
     <li>
      <p>
       <strong>
        调试别慌
       </strong>
       ：遇到数据跳变或者通信失败，别急，先检查硬件连接，再看看代码逻辑。慢慢来，总能找到问题所在。
      </p>
     </li>
    </ol>
    <p>
     总之，技术这事儿，耐心和细心是关键。希望这些小提示能帮到你，祝你顺利搞定姿态解算！
