    <p>
     链表是数据结构中的重要组成部分，也是面试中经常考察的内容。本文将详细介绍三道经典的链表问题：

    <h3>
     <a id="1__6">
     </a>
     1. 回文链表
    </h3>
    <h4>
     <a id="11__8">
     </a>
     1.1 问题描述
    </h4>
    <p>
     给定一个单链表的头节点
     <code>
      head
     </code>
     ，请你判断该链表是否为回文链表。如果是，返回
     <code>
      true
     </code>
     ；否则，返回
     <code>
      false
     </code>
     。
    </p>
    <p>
     <strong>
      示例 1：
     </strong>
    </p>
    <p>
     输入：
     <code>
      head = [1,2,2,1]
     </code>
     <br/>
     输出：
     <code>
      true
     </code>
    </p>
    <p>
     <strong>
      示例 2：
     </strong>
    </p>
    <p>
     输入：
     <code>
      head = [1,2]
     </code>
     <br/>
     输出：
     <code>
      false
     </code>
    </p>
    <p>
     <strong>
      提示：
     </strong>
    </p>
    <ul>
     <li>
      链表中节点数目在范围
      <code>
       [1, 10^5]
      </code>
      内
     </li>
     <li>
      <code>
       0 &lt;= Node.val &lt;= 9
      </code>
     </li>
    </ul>
    <h4>
     <a id="12__27">
     </a>
     1.2 解题思路
    </h4>
    <p>
     判断回文链表的关键在于如何高效地比较链表的前半部分和后半部分。常见的方法是使用快慢指针找到链表的中点，然后反转后半部分链表，最后进行比较。这种方法的时间复杂度为 O(n)，空间复杂度为 O(1)。
    </p>
    <p>
     具体步骤如下：
    </p>
    <ol>
     <li>
      <strong>
       找到链表中点
      </strong>
      ：使用快慢指针，快指针每次走两步，慢指针每次走一步。当快指针到达链表末尾时，慢指针恰好位于链表中点。
     </li>
     <li>
      <strong>
       反转后半部分链表
      </strong>
      ：从链表中点开始，反转后半部分链表。
     </li>
     <li>
      <strong>
       比较前后两部分
      </strong>
      ：从链表头部和反转后的后半部分头部开始，逐节点比较。
     </li>
     <li>
      <strong>
       恢复链表
      </strong>
      ：比较完成后，将链表恢复原状（可选）。
     </li>
     <li>
      <strong>
       返回结果
      </strong>
      ：如果所有节点都匹配，则返回
      <code>
       true
      </code>
      ，否则返回
      <code>
       false
      </code>
      。
     </li>
    </ol>

    <h3>
     <a id="2__104">
     </a>
     2. 环形链表
    </h3>
    <h4>
     <a id="21__106">
     </a>
     2.1 问题描述
    </h4>
    <p>
     给定一个链表的头节点
     <code>
      head
     </code>
     ，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪
     <code>
      next
     </code>
     指针再次到达，则链表中存在环。返回
     <code>
      true
     </code>
     表示存在环，否则返回
     <code>
      false
     </code>
     。
    </p>
    <p>
     <strong>
      示例 1：
     </strong>
    </p>
    <p>
     输入：
     <code>
      head = [3,2,0,-4]
     </code>
     ，
     <code>
      pos = 1
     </code>
     <br/>
     输出：
     <code>
      true
     </code>
    </p>
    <p>
     <strong>
      示例 2：
     </strong>
    </p>
    <p>
     输入：
     <code>
      head = [1,2]
     </code>
     ，
     <code>
      pos = 0
     </code>
     <br/>
     输出：
     <code>
      true
     </code>
    </p>
    <p>
     <strong>
      示例 3：
     </strong>
    </p>
    <p>
     输入：
     <code>
      head = [1]
     </code>
     ，
     <code>
      pos = -1
     </code>
     <br/>
     输出：
     <code>
      false
     </code>
    </p>
    <p>
     <strong>
      提示：
     </strong>
    </p>
    <ul>
     <li>
      链表中节点的数目范围是
      <code>
       [0, 10^4]
      </code>
     </li>
     <li>
      <code>
       -10^5 &lt;= Node.val &lt;= 10^5
      </code>
     </li>
    </ul>
    <h4>
     <a id="22__130">
     </a>
     2.2 解题思路
    </h4>
    <p>
     判断链表中是否存在环的经典方法是使用快慢指针。快指针每次走两步，慢指针每次走一步。如果链表中存在环，快指针最终会追上慢指针；如果链表中没有环，快指针会先到达链表末尾。
    </p>

    <h3>
     <a id="3__173">
     </a>
     3. 合并两个有序链表
    </h3>
    <h4>
     <a id="31__175">
     </a>
     3.1 问题描述
    </h4>
    <p>
     将两个升序链表合并为一个新的升序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。
    </p>
    <p>
     <strong>
      示例 1：
     </strong>
    </p>
    <p>
     输入：
     <code>
      l1 = [1,2,4]
     </code>
     ，
     <code>
      l2 = [1,3,4]
     </code>
     <br/>
     输出：
     <code>
      [1,1,2,3,4,4]
     </code>
    </p>
    <p>
     <strong>
      示例 2：
     </strong>
    </p>
    <p>
     输入：
     <code>
      l1 = []
     </code>
     ，
     <code>
      l2 = []
     </code>
     <br/>
     输出：
     <code>
      []
     </code>
    </p>
    <p>
     <strong>
      示例 3：
     </strong>
    </p>
    <p>
     输入：
     <code>
      l1 = []
     </code>
     ，
     <code>
      l2 = [0]
     </code>
     <br/>
     输出：
     <code>
      [0]
     </code>
    </p>
    <p>
     <strong>
      提示：
     </strong>
    </p>
    <ul>
     <li>
      两个链表的节点数目范围是
      <code>
       [0, 50]
      </code>
     </li>
     <li>
      <code>
       -100 &lt;= Node.val &lt;= 100
      </code>
     </li>
     <li>
      <code>
       l1
      </code>
      和
      <code>
       l2
      </code>
      均按非递减顺序排列
     </li>
    </ul>
    <h4>
     <a id="32__200">
     </a>
     3.2 解题思路
    </h4>
    <p>
     合并两个有序链表的关键在于如何高效地将两个链表的节点按顺序拼接。我们可以使用一个虚拟头节点（dummy node）来简化操作，并通过迭代的方式逐节点比较和拼接。
