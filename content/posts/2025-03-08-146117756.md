    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       引言
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Qwen2.5是阿里通义千问团队最新开源的顶尖AI大模型，具备多种参数规模的选项，包括0.5B、1.5B、3B、7B、14B、32B和72B。该模型在预训练时采用了最新的大规模数据集，涵盖了多达
     <span style="color:#ff0000">
      18万亿个tokens
     </span>
     ，使得Qwen2.5在
     <span style="color:#ff0000">
      自然语言理解、文本生成、编程能力和数学能力等
     </span>
     <span style="color:#ff0000">
      （
     </span>
     <span style="color:#ff0000">
      多模态
     </span>
     <span style="color:#ff0000">
      ）
     </span>
     方面都有了显著的提升。Qwen2.5不仅支持长文本处理，能够生成超过8K tokens 的内容，还增强了对系统提示的灵活适应性，提升了角色扮演和机器人的背景设定能力。此外，Qwen2.5支持多达29种语言，包括中文、英文、法文、西班牙文、葡萄牙文和德文等。Qwen2.5-Coder和Qwen2.5-Math是专门针对编程和数学问题优化的模型，在相关领域展现了卓越的性能。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Qwen2.5在MMLU-rudex（通用知识）、MBPP（代码能力）和 MATH（数学能力）基准测试中，分别取得了86.8、88.2和83.1的优异成绩，展示出其在多领域的强大性能。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>

      </strong>
     </strong>
     <strong>
      <strong>

      </strong>
     </strong>
     <strong>
      <strong>
       简介
      </strong>
     </strong>
    </h2>
    <p style="margin-left:0.0001pt; margin-right:0px">
     Qwen2.5是阿里通义千问团队与2024年9月推出的最新开源AI大模型，包括了
     <strong>
      <strong>
       语言模型 Qwen2.5
      </strong>
     </strong>
     ，以及专门针对
     <strong>
      <strong>
       编程的 Qwen2.5-Coder
      </strong>
     </strong>
     和
     <strong>
      <strong>
       数学的 Qwen2.5-Math
      </strong>
     </strong>
     模型。所有开放权重的模型都是稠密的、decoder-only的语言模型，提供多种不同规模的版本，包括：
    </p>
    <ol>
     <li style="text-align:justify">
      Qwen2.5: 0.5B, 1.5B, 3B, 7B, 14B, 32B, 以及72B;
     </li>
     <li style="text-align:justify">
      Qwen2.5-Coder: 1.5B, 7B, 以及即将推出的32B;
     </li>
     <li style="text-align:justify">
      Qwen2.5-Math: 1.5B, 7B, 以及72B。
     </li>
    </ol>

    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       二、模型
      </strong>
     </strong>
     <strong>
      <strong>
       要点
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     就 Qwen2.5 语言模型而言，所有模型都在最新的大规模数据集上进行了预训练，该数据集包含多达
     <strong>
      <strong>
       18T tokens
      </strong>
     </strong>
     （18万亿tokens）。相较于Qwen2，Qwen2.5获得了显著更多的
     <span style="color:#ff0000">
      知识（MMLU：85+）
     </span>
     ，并在
     <span style="color:#ff0000">
      编程能力（HumanEval 85+）
     </span>
     和
     <span style="color:#ff0000">
      数学能力（MATH 80+）
     </span>
     方面有了大幅提升。此外，新模型在指令执行、
     <span style="color:#ff0000">
      生成长文本（超过8K标记）
     </span>
     、理解结构化数据（例如表格）以及生成结构化输出特别是JSON方面取得了显著改进。Qwen2.5模型总体上对各种system prompt更具适应性，增强了角色扮演实现和聊天机器人的条件设置功能。与Qwen2类似，Qwen2.5语言模型支持高达128K tokens，并能生成最多8K tokens的内容。它们同样保持了对包括中文、英文、法文、西班牙文、葡萄牙文、德文、意大利文、俄文、日文、韩文、越南文、泰文、阿拉伯文等 29 种以上语言的支持。 在下表中提供了有关模型的基本信息。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     专业领域的专家语言模型，即用于编程的 Qwen2.5-Coder 和用于数学的 Qwen2.5-Math，相比其前身CodeQwen1.5和Qwen2-Math有了实质性的改进。具体来说，Qwen2.5-Coder在包含5.5 T tokens编程相关数据上进行了训练，使即使较小的编程专用模型也能在编程评估基准测试中表现出媲美大型语言模型的竞争力。 同时，Qwen2.5-Math支持 中文 和 英文，并整合了多种推理方法，包括
     <strong>
      <strong>
       CoT（Chain of Thought）
      </strong>
     </strong>
     、
     <strong>
      <strong>
       PoT（Program of Thought）
      </strong>
     </strong>
     和
     <strong>
      <strong>
       TIR（Tool-Integrated Reasoning）
      </strong>
     </strong>
     。
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/238564060a744c349753a2896e424ff8.png"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     <strong>
      <strong>
       模型要点总结如下：
      </strong>
     </strong>
    </p>
    <ol>
     <li style="text-align:justify">
      <strong>
       <strong>
        多样化模型规模
       </strong>
      </strong>
      ：Qwen2.5提供从0.5B到72B不同参数规模的模型，以满足各种应用需求。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        扩展的预训练数据集
       </strong>
      </strong>
      ：Qwen2.5的预训练数据集从7T tokens扩展到18T tokens，显著提升了模型的知识储备。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        增强的多语言能力
       </strong>
      </strong>
      ：Qwen2.5支持超过29种语言，包括中文和英文，确保了广泛的多语言支持。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        提升的编程和数学能力
       </strong>
      </strong>
      ：Qwen2.5-Coder和Qwen2.5-Math针对编程和数学问题进行了专门优化，提供了更专业的解决方案。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        长文本处理能力
       </strong>
      </strong>
      ：Qwen2.5支持高达128K tokens的上下文长度，能够生成最长8K tokens的内容，增强了对长文本的处理能力。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        结构化数据处理
       </strong>
      </strong>
      ：该模型在理解和生成结构化数据（如表格）方面有显著改善，尤其在 JSON 输出方面表现突出。
     </li>
     <li style="text-align:justify">
      <strong>
       <strong>
        系统提示适应性
       </strong>
      </strong>
      ：Qwen2.5对各种系统提示具有更强的适应性，提升了角色扮演和机器人的条件设置能力。
     </li>
    </ol>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">

    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       三、Qwen-2.5架构
      </strong>
     </strong>
    </h2>
    <h3 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       1. 核心改进
      </strong>
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Qwen-2.5是Qwen系列的最新升级版本，
     <span style="color:#ff0000">
      针对长文本处理、多语言支持和计算效率进行了优化
     </span>
     ，主要改进如下：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）
     <strong>
      <strong>
       动态NTK-aware RoPE
      </strong>
     </strong>
     ：扩展上下文窗口至32K tokens，支持超长文本推理。有关动态NTK-aware RoPE的更多内容可以看的我CSDN文章：
     <a href="https://lzm07.blog.csdn.net/article/details/145865840" rel="nofollow" title="transformer架构嵌入层位置编码之动态NTK-aware位置编码_动态的位置编码嵌入-CSDN博客">
      transformer架构嵌入层位置编码之动态NTK-aware位置编码_动态的位置编码嵌入-CSDN博客
     </a>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）
     <strong>
      <strong>
       混合注意力机制
      </strong>
     </strong>
     ：结合MQA（多查询注意力）和GQA（分组查询注意力），显存占用降低30%。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）
     <strong>
      <strong>
       激活函数优化
      </strong>
     </strong>
     ：采用
     <span style="color:#ff0000">
      SwiGLU v2
     </span>
     ，增强非线性表达能力。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）
     <strong>
      <strong>
       量化感知训练
      </strong>
     </strong>
     ：
     <span style="color:#ff0000">
      原生支持GPTQ-Int4量化
     </span>
     ，精度损失小于1%。
    </p>

2. 代入 f(x) = x²，得 f'(x) = lim_{h→0} [(x+h)² - x²] / h。

3. 展开分子：(x² + 2xh + h²) - x² = 2xh + h²。

4. 化简表达式：(2xh + h²)/h = 2x + h。

5. 取极限 h→0，得 f'(x) = 2x。

6. 因此，f'(2) = 2*2 = 4。

    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在数学专用语言模型方面，相比于Qwen2-Math，
     <span style="color:#ff0000">
      Qwen2.5-Math在更大规模的数学相关数据上进行了预训练
     </span>
     ，包括由Qwen2-Math生成的合成数据。此外，
     <span style="color:#ff0000">
      增加了对中文的支持
     </span>
     ，并通过赋予其进行CoT（Chain of Thought）、PoT（Program of Thought）和TIR（Tool-Integrated Reasoning）的能力来加强其推理能力。 Qwen2.5-Math-72B-Instruct的整体性能超越了Qwen2-Math-72B-Instruct和GPT4-o，甚至是非常小的专业模型如 Qwen2.5-Math-1.5B-Instruct也能在与大型语言模型的竞争中取得高度竞争力的表现。
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/ab8d2a9ff7394e3dbf82b62cd3a37d56.png"/>
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       六、
      </strong>
     </strong>
     <strong>
      <strong>
       使用Qwen2.5
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     最简单的方法使用过
     <span style="color:#4874cb">
      阿里云百炼平台
     </span>
     提供的通义千问API来使用
     <a href="https://www.aliyun.com/product/bailian" rel="nofollow" title="阿里云百炼平台">
      阿里云百炼平台
     </a>
     ，百炼平台已经兼容OpenAI接口规范。
    </p>
    <pre><code class="hljs">from openai import OpenAIimport osclient = OpenAI(    api_key=os.getenv("YOUR_API_KEY"),    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",)completion = client.chat.completions.create(    model="qwen-plus-latest",    messages=[      {'role': 'user', 'content': 'Tell me something about large language models.'}    ])  print(completion.choices[0].message.content)</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     通过 Hugging Face Transformers库来使用，正如在
     <a href="https://huggingface.co/Qwen/Qwen2.5-7B-Instruct" rel="nofollow" title="model card">
      model card
     </a>
     中演示的那样:
    </p>
    <pre><code class="hljs">from transformers import AutoModelForCausalLM, AutoTokenizermodel_name = "Qwen/Qwen2.5-7B-Instruct"model = AutoModelForCausalLM.from_pretrained(    model_name,    torch_dtype="auto",    device_map="auto")tokenizer = AutoTokenizer.from_pretrained(model_name)prompt = "Give me a short introduction to large language model."messages = [    {"role": "user", "content": prompt}]text = tokenizer.apply_chat_template(    messages,    tokenize=False,    add_generation_prompt=True)model_inputs = tokenizer([text], return_tensors="pt").to(model.device)generated_ids = model.generate(    **model_inputs,    max_new_tokens=512)generated_ids = [    output_ids[len(input_ids):] for input_ids, output_ids in zip(model_inputs.input_ids, generated_ids)]response = tokenizer.batch_decode(generated_ids, skip_special_tokens=True)[0]</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     要使用vLLM运行Qwen2.5并部署一个与OpenAI API兼容的服务，可以运行如下命令：
    </p>
    <pre><code class="hljs">python -m vllm.entrypoints.openai.api_server \    --model Qwen/Qwen2.5-7B-Instruct</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     如果你使用的是vllm&gt;=0.5.3，可以使用vllm serve命令。然后你就可以通过curl来与Qwen2.5 进行对话了：
    </p>
    <pre><code class="hljs">curl http://localhost:8000/v1/chat/completions -H "Content-Type: application/json" -d '{  "model": "Qwen/Qwen2.5-7B-Instruct",  "messages": [    {"role": "user", "content": "Tell me something about large language models."}  ],  "temperature": 0.7,  "top_p": 0.8,  "repetition_penalty": 1.05,  "max_tokens": 512}'</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     此外，Qwen2.5支持vLLM内置的工具调用功能。此功能要求 vllm&gt;=0.6。如果你想启用此功能，请使用以下命令启动vLLM的OpenAI API兼容服务：
    </p>
    <pre><code class="hljs">vllm serve Qwen/Qwen2.5-7B-Instruct --enable-auto-tool-choice --tool-call-parser hermes</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     之后你可以像使用
     <a href="https://cookbook.openai.com/examples/how_to_call_functions_with_chat_models" rel="nofollow" title="GPT’s tool calling">
      GPT’s tool calling
     </a>
     那样来使用它。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Qwen2.5同样支持
     <a href="https://ollama.com/blog/tool-support" rel="nofollow" title="Ollama’s tool calling">
      Ollama’s tool calling
     </a>
     .你可以通过启动Ollama的OpenAI兼容服务，并以与使用GPT的工具调用相同的方式来使用它。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     Qwen2.5的聊天模板中也包含了一个工具调用模板，这意味着你可以使用Hugging Face
     <a href="#advanced-tool-use--function-calling" rel="nofollow">
      transformers’ tool calling support
     </a>
     .
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     vLLM / Ollama / Transformers的工具调用支持使用受
     <a href="https://huggingface.co/NousResearch/Hermes-3-Llama-3.1-8B" rel="nofollow" title="Nous’ Hermes">
      Nous’ Hermes
     </a>
     的格式启发的工具调用模板。 此前
     <a href="https://github.com/QwenLM/Qwen-Agent" title="Qwen-Agent">
      Qwen-Agent
     </a>
     提供了使用Qwen2自己的工具调用模板的工具调用支持（这较难与vllm和Ollama集成），而Qwen2.5既保持了与Qwen2模板和Qwen-Agent的兼容性。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
    </p>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       七、应用场景
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （1）机器人和虚拟助手：Qwen2.5 可以作为对话系统的核心，提供自然语言理解和生成，实现用户的交互需求。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （2）内容创作和编辑：能够自动生成文章、故事、诗歌等文本内容，辅助用户进行编辑和创作。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （3）教育和学习辅助：可帮助学生和教师进行语言学习、作业辅导及知识测试。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （4）编程辅助：Qwen2.5-Coder 模型专注于编程任务，提供代码建议和调试支持。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （5）数学问题解决：Qwen2.5-Math 模型支持中英文数学问题的解决，适用于教育和研究领域。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     （6）多语言翻译：可用于生成翻译文本，具备编码器-解码器架构的能力。
    </p>
