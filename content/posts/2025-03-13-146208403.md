    <h3>
     <a id="2C_7">
     </a>
     2.C语言中动态内存管理方式
    </h3>
    <p>
     1.malloc（Memory Allocation）
    </p>
    <blockquote>
     <p>
      <strong>
       定义
      </strong>
      ：void*malloc(size_t size);
      <br/>
      <strong>
       功能
      </strong>
      ：
      <strong>
       分配一块指定大小的内存空间，并返回指向该内存空间的指针
      </strong>
      。
      <br/>
      <strong>
       参数
      </strong>
      ：size_t size：需要分配的内存大小（以字节为单位）。
      <br/>
      <strong>
       返回值
      </strong>
      ：成功时返回指向分配内存的指针。失败时返回 NULL。
      <br/>
      <strong>
       特点
      </strong>
      ：
      <strong>
       分配的内存空间的内容是未初始化的
      </strong>
      ，即内存中的值是随机的。如果分配失败（例如内存不足），返回 NULL。需要处理内存分配失败的情况。
     </p>
    </blockquote>
    <p>
     2.calloc（Clear Memory Allocation）
    </p>
    <blockquote>
     <p>
      <strong>
       定义
      </strong>
      :void* calloc(size_t num, size_t size);
      <br/>
      <strong>
       功能
      </strong>
      ：
      <strong>
       分配一块内存空间，并将该内存空间初始化为零
      </strong>
      。
      <br/>
      <strong>
       参数
      </strong>
      ：size_t num：分配的元素数量。size_t size：每个元素的大小（以字节为单位）。
      <br/>
      <strong>
       返回值
      </strong>
      ：成功时返回指向分配内存的指针。失败时返回 NULL。
      <br/>
      <strong>
       特点
      </strong>
      ：
      <strong>
       分配的内存空间的内容被初始化为零
      </strong>
      。如果分配失败，返回 NULL。实际分配的内存大小为 num * size。
     </p>
    </blockquote>
    <ol start="3">
     <li>
      realloc（Re-Allocation）
     </li>
    </ol>
    <blockquote>
     <p>
      <strong>
       定义
      </strong>
      :void* realloc(void* ptr, size_t size);
      <br/>
      <strong>
       功能
      </strong>
      ：
      <strong>
       重新分配一块内存空间，其大小由 size 参数指定
      </strong>
      。
      <br/>
      <strong>
       参数
      </strong>
      ：void* ptr：指向之前分配的内存空间的指针。
      <br/>
      <strong>
       size_t size
      </strong>
      ：新的内存大小（以字节为单位）。
      <br/>
      <strong>
       返回值
      </strong>
      ：成功时返回指向重新分配内存的指针。失败时返回 NULL，但原内存空间不会被释放。
      <br/>
      <strong>
       特点
      </strong>
      :
      <br/>
      1.如果 ptr 是 NULL，realloc 的行为等同于 malloc。
      <br/>
      2.
      <strong>
       如果 size 为零，realloc 的行为等同于 free
      </strong>
      。
      <br/>
      3.如果
      <strong>
       新的内存大小大于原内存大小
      </strong>
      ，
      <strong>
       超出部分的内容是未定义的
      </strong>
      。
      <br/>
      4.如果新
      <strong>
       的内存大小小于原内存大小
      </strong>
      ，
      <strong>
       超出部分的内容会被截断
      </strong>
      。
      <br/>
      5.
      <strong>
       如果重新分配失败，原内存空间仍然有效
      </strong>
      。
     </p>
    </blockquote>
    <blockquote>
     <p>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8862a511a25345e3b661246a29c80730.png">
       <br/>
       问题1详见上述三个函数介绍
       <br/>
       问题2不需要free(p2)：
       <br/>
       若原地扩容，p2,p3指向同一个地址，只需free一次。
       <br/>
       若异地扩容，p2指向的原地址被释放，原数据已复制到新内存空间中，p3指向该空间地址，只需free(p3)即可。
      </img>
     </p>
    </blockquote>
    <p>
     这里涉及到realloc的扩容问题：
    </p>
    <blockquote>
     <p>
      <strong>
       原地扩容
      </strong>
      ：若原内存块后方有足够大小的连续空闲空间，直接扩展内存区域，保留原有数据，返回原指针。
      <br/>
      <strong>
       异地扩容
      </strong>
      ：若原内存块后空间不足，则
      <strong>
       分配新内存块，复制原有数据到新内存中
      </strong>
      ，释放旧内存，
      <strong>
       返回新指针
      </strong>
      。
     </p>
    </blockquote>

    <h3>
     <a id="5newdelete_123">
     </a>
     5.总结new与delete的实现原理
    </h3>
    <h3>
     <a id="51_124">
     </a>
     5.1内置类型
    </h3>
    <blockquote>
     <p>
      如果申请的是内置类型的空间，new和malloc，delete和free基本类似，不同的地方是：
      <strong>
       new/delete申请和释放的是单个元素的空间
      </strong>
      ，
      <strong>
       new[]和delete[]申请的是连续空间
      </strong>
      ，而且new在申请空间失败时会抛异常，malloc会返回NULL。
     </p>
    </blockquote>
    <h3>
     <a id="52_126">
     </a>
     5.2自定义类型
    </h3>
    <ul>
     <li>
      <strong>
       new的原理
      </strong>
     </li>
    </ul>
    <p>
     1 调用operator new函数申请空间
     <br/>
     2. 在申请的空间上执行构造函数，完成对象的构造
    </p>
    <ul>
     <li>
      <strong>
       delete的原理
      </strong>
     </li>
    </ul>
    <p>
     1.在空间上执行析构函数，完成对象中资源的清理工作
     <br/>
     2.调用operator delete函数释放对象的空间
    </p>
    <ul>
     <li>
      <strong>
       new T[N]的原理
      </strong>
     </li>
    </ul>
    <p>
     1.调用operator new[]函数，在operator new[]中实际调用operator new函数完成N个对象空间的申请
     <br/>
     2. 在申请的空间上执行N次构造函数
    </p>
    <ul>
     <li>
      <strong>
       delete[]的原理
      </strong>
     </li>
    </ul>
    <p>
     1 在释放的对象空间上执行N次析构函数，完成N个对象中资源的清理
     <br/>
     2. 调用operator delete[]释放空间，实际在operator delete[]中调用operator delete来释放空间
    </p>
    <h3>
     <a id="6newplacementnew_145">
     </a>
     6.定位new表达式(placement-new)（了解）
    </h3>
    <p>
     定位new表达式是在
     <strong>
      已分配的原始内存空间
     </strong>
     中
     <strong>
      调用构造函数初始化一个对象
     </strong>
     。
     <br/>
     使用格式：
    </p>
    <blockquote>
     <p>
      <strong>
       new (place_address) type
      </strong>
      或者
      <strong>
       new (place_address) type(initializer-list)
      </strong>
      <br/>
      place_address必须是一个指针，initializer-list是类型的初始化列表
     </p>
    </blockquote>
    <p>
     使用场景：
    </p>
    <blockquote>
     <p>
      定位new表达式在实际中一般是配合内存池使用。因为内存池分配出的内存没有初始化，所以如果是自定义类型的对象，需要使用new的定义表达式进行
      <strong>
       显示调
      </strong>
      构造函数进行初始化。
     </p>
    </blockquote>
    <p>
     内存池是从内存中获取内存，但并没有初始化，在需要频繁申请和释放内存的场景下更方便我们获取内存。除内存池之外，还有线程池，连接池等池，统称为池化技术，后续会深入学习。
    </p>

    <h3>
     <a id="7mallocfreenewdelete_193">
     </a>
     7.malloc/free和new/delete的区别
    </h3>
    <blockquote>
     <p>
      malloc/free和new/delete的共同点是：都是从堆上申请空间，并且需要用户手动释放。不同的地方是：
     </p>
    </blockquote>
    <ol>
     <li>
      malloc和free是
      <strong>
       函数
      </strong>
      ，new和delete是
      <strong>
       操作符
      </strong>
     </li>
     <li>
      malloc申请的空间不会初始化，new可以初始化
     </li>
     <li>
      malloc申请空间时，需要
      <strong>
       手动计算
      </strong>
      空间大小并传递，new
      <strong>
       只需在其后跟上空间的类型即可
      </strong>
      ， 如果是
      <strong>
       多个对象，[]中指定对象个数即可
      </strong>
     </li>
     <li>
      malloc的返回值为void*, 在使用时必须强转，new不需要，因为new后跟的是空间的类型
     </li>
     <li>
      malloc申请空间失败时，返回的是NULL，因此使用时必须判空，new不需要，但是new需要
      <strong>
       捕获异常
      </strong>
     </li>
     <li>
      申请自定义类型对象时，malloc/free只会开辟空间，不会调用构造函数与析构函数，而new在申请空间后会
      <strong>
       调用构造函数
      </strong>
      完成对象的初始化，delete在释放空间前会
      <strong>
       调用析构函数
      </strong>
      完成空间中资源的清理
     </li>
    </ol>

    <h3>
     <a id="8_202">
     </a>
     8.内存泄漏
    </h3>
    <p>
     什么是内存泄漏：
    </p>
    <blockquote>
     <p>
      内存泄漏指因为疏忽或错误造成程序
      <strong>
       未能释放已经不再使用的内存
      </strong>
      的情况。内存泄漏并不是指内存在物理上的消失，而是应用程序分配某段内存后，因为设计错误，
      <strong>
       失去了对该段内存的控制，因而造成了内存的浪费
      </strong>
      。
     </p>
    </blockquote>
    <p>
     内存泄漏的危害：
    </p>
    <blockquote>
     <p>
      普通程序内存泄漏影响不大，进程正常结束会释放资源。
      <br/>
      长期运行的程序出现内存泄漏，影响很大，如操作系统、后台服务等等，出现内存泄漏会导致
      <strong>
       响应越来越慢，最终卡死
      </strong>
      。
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/31737f25733041a2b2fd252a366a2ab2.png"/>
     </p>
    </blockquote>
    <p>
     内存泄漏分类
    </p>
    <ul>
     <li>
      堆内存泄漏(Heap leak)
     </li>
    </ul>
    <blockquote>
     <p>
      堆内存指的是程序执行中依据须要分配通过malloc / calloc / realloc / new等从堆中分配的一块内存，
      <br/>
      用完后必须通过调用相应的 free或者delete 删掉。假设程序的设计错误导致这部分内存没有被释放，那
      <br/>
      么以后这部分空间将无法再被使用，就会产生Heap Leak
     </p>
    </blockquote>
    <ul>
     <li>
      系统资源泄漏
     </li>
    </ul>
    <blockquote>
     <p>
      指程序使用系统分配的资源，比方套接字、文件描述符、管道等没有使用对应的函数释放掉，导致系统资源的浪费，严重可导致系统效能减少，系统执行不稳定。
     </p>
    </blockquote>
