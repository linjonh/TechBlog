    <blockquote>
     <p>
      ​理解子网掩码与CIDR的转换
      <br/>
      子网掩码255.255.255.0对应的二进制形式为11111111.11111111.11111111.00000000，其中连续“1”的个数为24位，因此CIDR表示为/24。
      <br/>
      ​在YAML配置中，子网掩码需与IP地址结合使用，例如192.168.1.10/24，其中/24即为255.255.255.0的CIDR形式。
     </p>
    </blockquote>

    <p>
     以下磁盘的分配来至AI（
     <a href="https://yuanbao.tencent.com/bot/app/share/chat/047059874a01deefc5428bf972b60520" rel="nofollow">
      链接：Ubuntu 24.04.2 服务器存储分配方案（针对 Ollama 和 AnythingLLM）
     </a>
     ），其问法为：“以下服务器，如果分配存储，能合理 Ollama ， AnythingLLM：”
    </p>
    <p>
     <strong>
      Ubuntu 24.04.2 服务器存储分配方案（针对 Ollama 和 AnythingLLM）​
     </strong>
     <br/>
     根据你的磁盘信息（sda 总容量 931.5GB，当前仅分配 100GB 给根目录），建议按以下方案分配存储：
    </p>
    <p>
     1、
     <strong>
      存储现状分析
     </strong>
    </p>
    <ol>
     <li>
      ​可用空间分布
      <br/>
      • ​LVM 剩余空间：卷组 ubuntu-vg 中未分配的物理扩展区（PE）约 ​828.46GB，可通过 LVM 扩展逻辑卷。
      <br/>
      • ​根目录占用：当前根目录 / 使用 12%（11GB），剩余 83GB，适合部署核心系统组件。
      <br/>
      • ​磁盘分区：sda3 已全部加入 LVM，无需额外格式化。
     </li>
     <li>
      ​
      <strong>
       关键限制
      </strong>
      <br/>
      • ​Ollama 需求：模型文件通常较大（7B 模型约 4GB，70B 模型可达 100GB+），需预留充足空间 。
      <br/>
      • ​AnythingLLM 需求：工作区数据、文档索引和本地模型存储需独立分区 。
     </li>
    </ol>
    <p>
     <strong>
      2、存储分配建议 ​
     </strong>
    </p>
    <ol>
     <li>
      扩展 LVM 逻辑卷
      <br/>
      利用 ubuntu-vg 的剩余空间创建两个专用逻辑卷：
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token comment"># 创建 Ollama 模型存储卷（500GB）</span>
<span class="token function">sudo</span> lvcreate <span class="token parameter variable">-L</span> 500G <span class="token parameter variable">-n</span> ollama-data ubuntu-vg

<span class="token comment"># 创建 AnythingLLM 数据存储卷（300GB）</span>
<span class="token function">sudo</span> lvcreate <span class="token parameter variable">-L</span> 300G <span class="token parameter variable">-n</span> anythingllm-data ubuntu-vg
</code></pre>
    <ol start="2">
     <li>
      格式化并挂载
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token comment"># 格式化逻辑卷为 ext4（推荐兼容性）</span>
<span class="token function">sudo</span> mkfs.ext4 /dev/ubuntu-vg/ollama-data
<span class="token function">sudo</span> mkfs.ext4 /dev/ubuntu-vg/anythingllm-data

<span class="token comment"># 创建挂载点</span>
<span class="token function">sudo</span> <span class="token function">mkdir</span> /mnt/ollama_models /mnt/anythingllm_storage

<span class="token comment"># 写入 /etc/fstab 实现持久化</span>
<span class="token builtin class-name">echo</span> <span class="token string">"/dev/ubuntu-vg/ollama-data /mnt/ollama_models ext4 defaults 0 2"</span> <span class="token operator">|</span> <span class="token function">sudo</span> <span class="token function">tee</span> <span class="token parameter variable">-a</span> /etc/fstab
<span class="token builtin class-name">echo</span> <span class="token string">"/dev/ubuntu-vg/anythingllm-data /mnt/anythingllm_storage ext4 defaults 0 2"</span> <span class="token operator">|</span> <span class="token function">sudo</span> <span class="token function">tee</span> <span class="token parameter variable">-a</span> /etc/fstab

<span class="token comment"># 立即挂载</span>
<span class="token function">sudo</span> <span class="token function">mount</span> <span class="token parameter variable">-a</span>
</code></pre>
    <p>
     <strong>
      3、验证与维护
     </strong>
    </p>
    <ol>
     <li>
      验证挂载
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">df</span> <span class="token parameter variable">-hT</span> <span class="token operator">|</span> <span class="token function">grep</span> <span class="token parameter variable">-E</span> <span class="token string">'ollama|anythingllm'</span>
<span class="token comment"># 应显示 /mnt/ollama_models 和 /mnt/anythingllm_storage 的容量</span>
</code></pre>
    <h3>
     <a id="__ollama_195">
     </a>
     二、 安装 ollama
    </h3>
    <p>
     检查系统版本，下载对应ollama压缩包，输出为 “amd64”，当下载
     <a href="https://github.com/ollama/ollama/releases">
      链接：ollama-linux-amd64.tgz
     </a>
     版本。
    </p>
    <pre><code class="prism language-bash">dpkg --print-architecture
</code></pre>
    <p>
     输出：
    </p>
    <pre><code class="prism language-bash">
root@cikkodai:/mnt/ollama_models<span class="token comment"># dpkg --print-architecture</span>
amd64
</code></pre>
    <ol start="2">
     <li>
      将压缩包拷贝至服务器端并解压，解压至 /usr目录。拷贝可以通过搭建局域网服务器，远程工具，命令等。
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">tar</span> <span class="token parameter variable">-C</span> /usr <span class="token parameter variable">-xzf</span> ollama-linux-amd64.tgz
</code></pre>
    <p>
     3.快速定位 ollama 可执行文件及其相关文件的存储路径
    </p>
    <pre><code class="prism language-bash"><span class="token function">whereis</span> ollama
</code></pre>
    <p>
     输出：
    </p>
    <pre><code class="prism language-bash">root@cikkodai:/mnt<span class="token comment"># whereis ollama</span>
ollama: /usr/bin/ollama /usr/lib/ollama
</code></pre>
    <p>
     需要记住 /usr/lib/ollama 路径，此路径在配置文件中会使用。
    </p>
    <ol start="4">
     <li>
      修改ollama配置文件，如果没有可以自行新建。
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">vim</span> /etc/systemd/system/ollama.service
</code></pre>
    <pre><code class="prism language-bash"><span class="token punctuation">[</span>Unit<span class="token punctuation">]</span>
<span class="token assign-left variable">Description</span><span class="token operator">=</span>Ollama Service
<span class="token assign-left variable">After</span><span class="token operator">=</span>network-online.target

<span class="token punctuation">[</span>Service<span class="token punctuation">]</span>
<span class="token assign-left variable">ExecStart</span><span class="token operator">=</span>/usr/bin/ollama serve
<span class="token assign-left variable">User</span><span class="token operator">=</span>root
<span class="token assign-left variable">Group</span><span class="token operator">=</span>root
<span class="token assign-left variable">Restart</span><span class="token operator">=</span>always
<span class="token assign-left variable">RestartSec</span><span class="token operator">=</span><span class="token number">3</span>
<span class="token assign-left variable">Environment</span><span class="token operator">=</span><span class="token string">"OLLAMA_MODELS=/mnt/ollama_models"</span>
<span class="token assign-left variable">Environment</span><span class="token operator">=</span><span class="token string">"OLLAMA_ORIGINS=*"</span>
<span class="token assign-left variable">Environment</span><span class="token operator">=</span><span class="token string">"OLLAMA_HOST=0.0.0.0"</span>

<span class="token punctuation">[</span>Install<span class="token punctuation">]</span>
<span class="token assign-left variable">WantedBy</span><span class="token operator">=</span>default.target

</code></pre>
    <p>
     5.重载配，启动ollama并设置自启和查看状态
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl daemon-reload
<span class="token function">sudo</span> systemctl start ollama
<span class="token function">sudo</span> systemctl <span class="token builtin class-name">enable</span> ollama
<span class="token function">sudo</span> systemctl status ollama

</code></pre>
    <p>
     输出：
    </p>
