     理解 JavaScript 中的浅拷贝与深拷贝

    <blockquote>
     <p>
      在 JavaScript 开发中，我们经常需要复制对象或数组。然而，复制的方式不同，可能会导致不同的结果。本文将详细介绍
      <strong>
       浅拷贝
      </strong>
      和
      <strong>
       深拷贝
      </strong>
      的概念、区别以及实现方式，帮助你更好地理解和使用它们。
     </p>
    </blockquote>
    <h3>
     1. 什么是浅拷贝？
    </h3>
    <h4>
     定义
    </h4>
    <p>
     浅拷贝是指只复制对象的第一层属性。如果对象的属性是引用类型（如对象、数组等），则复制的是引用地址，而不是实际的值。
    </p>
    <h4>
     特点
    </h4>
    <ul>
     <li>
      <p>
       新对象和原对象共享内部的引用类型字段。
      </p>
     </li>
     <li>
      <p>
       修改原对象的引用类型字段，浅拷贝的对象也会受到影响。
      </p>
     </li>
    </ul>
    <p>
     <img alt="" height="2000" src="https://i-blog.csdnimg.cn/direct/429f84b4a7d8427da1068928b7318e55.png" width="4040"/>
    </p>
    <h4>
     实现方式
    </h4>
    <p>
     在 JavaScript 中，浅拷贝可以通过以下方式实现：
    </p>
    <ul>
     <li>
      <p>
       使用
       <code>
        Object.assign()
       </code>
      </p>
     </li>
     <li>
      <p>
       使用扩展运算符
       <code>
        {...obj}
       </code>
      </p>
     </li>
     <li>
      <p>
       使用数组的
       <code>
        slice()
       </code>
       或
       <code>
        concat()
       </code>
       方法（针对数组）
      </p>
     </li>
    </ul>
    <h4>
     示例
    </h4>
    <pre><code>const original = { a: 1, b: { c: 2 } };
const shallowCopied = { ...original }; // 使用扩展运算符实现浅拷贝

// 修改原对象的引用类型字段
original.b.c = 100;

console.log(original);      // { a: 1, b: { c: 100 } }
console.log(shallowCopied); // { a: 1, b: { c: 100 } }，浅拷贝对象也被修改</code></pre>
    <h3>
     2. 什么是深拷贝？
    </h3>
    <h4>
     定义
    </h4>
    <p>
     深拷贝是指递归地复制对象及其内部的所有引用类型字段，生成一个完全独立的新对象。
    </p>
    <h4>
     特点
    </h4>
    <ul>
     <li>
      <p>
       新对象和原对象完全独立。
      </p>
     </li>
     <li>
      <p>
       修改原对象的引用类型字段，深拷贝的对象不会受到影响。
      </p>
     </li>
    </ul>
    <p>
     <img alt="" height="1492" src="https://i-blog.csdnimg.cn/direct/41e265fd546f48d499add6b234f8ea4b.png" width="2536"/>
    </p>
    <h4>
     实现方式
    </h4>
    <p>
     深拷贝的实现方式有多种，以下是常见的几种方法：
    </p>
    <h5>
     方法 1：使用
     <code>
      JSON.parse(JSON.stringify(obj))
     </code>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：简单易用。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         不能处理函数、
         <code>
          undefined
         </code>
         、
         <code>
          Symbol
         </code>
         等特殊类型。
        </p>
       </li>
       <li>
        <p>
         不能处理循环引用（即对象内部属性引用自身）。
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <pre><code>const original = { a: 1, b: { c: 2 } };
const deepCopied = JSON.parse(JSON.stringify(original));

// 修改原对象的引用类型字段
original.b.c = 100;

console.log(original);    // { a: 1, b: { c: 100 } }
console.log(deepCopied);  // { a: 1, b: { c: 2 } }，深拷贝对象不受影响</code></pre>
    <h5>
     方法 2：递归实现深拷贝
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：可以处理更多数据类型（如函数、
       <code>
        undefined
       </code>
       等）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：需要手动处理循环引用。
      </p>
     </li>
    </ul>
    <pre><code>function deepClone(obj, cache = new WeakMap()) {
  // 如果是基本类型或 null，直接返回
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 如果对象已经被拷贝过，直接返回缓存的结果（避免循环引用）
  if (cache.has(obj)) {
    return cache.get(obj);
  }

  // 处理数组
  if (Array.isArray(obj)) {
    const clonedArr = [];
    cache.set(obj, clonedArr);
    obj.forEach((item) =&gt; {
      clonedArr.push(deepClone(item, cache));
    });
    return clonedArr;
  }

  // 处理普通对象
  const clonedObj = {};
  cache.set(obj, clonedObj);
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      clonedObj[key] = deepClone(obj[key], cache);
    }
  }

  return clonedObj;
}

// 测试
const original = { a: 1, b: { c: 2 }, d: [3, 4] };
const deepCopied = deepClone(original);

// 修改原对象
original.b.c = 100;
original.d.push(5);

console.log(original);    // { a: 1, b: { c: 100 }, d: [3, 4, 5] }
console.log(deepCopied);  // { a: 1, b: { c: 2 }, d: [3, 4] }，深拷贝对象不受影响</code></pre>
    <h5>
     方法 3：使用第三方库（如 Lodash）
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        优点
       </strong>
       ：功能强大，支持各种数据类型和循环引用。
      </p>
     </li>
     <li>
      <p>
       <strong>
        缺点
       </strong>
       ：需要引入额外的库。
      </p>
     </li>
    </ul>
    <pre><code>// 安装 lodash：npm install lodash
const _ = require('lodash');

const original = { a: 1, b: { c: 2 } };
const deepCopied = _.cloneDeep(original);

// 修改原对象
original.b.c = 100;

console.log(original);    // { a: 1, b: { c: 100 } }
console.log(deepCopied);  // { a: 1, b: { c: 2 } }，深拷贝对象不受影响</code></pre>
    <h3>
     3. 浅拷贝 vs 深拷贝
    </h3>
    <table>
     <thead>
      <tr>
       <th>
        特性
       </th>
       <th>
        浅拷贝
       </th>
       <th>
        深拷贝
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        复制层级
       </td>
       <td>
        只复制第一层属性
       </td>
       <td>
        递归复制所有层级属性
       </td>
      </tr>
      <tr>
       <td>
        引用类型字段
       </td>
       <td>
        共享引用地址
       </td>
       <td>
        完全独立
       </td>
      </tr>
      <tr>
       <td>
        修改原对象影响
       </td>
       <td>
        影响浅拷贝对象
       </td>
       <td>
        不影响深拷贝对象
       </td>
      </tr>
      <tr>
       <td>
        实现方式
       </td>
       <td>
        <code>
         Object.assign()
        </code>
        、扩展运算符等
       </td>
       <td>
        <code>
         JSON.parse(JSON.stringify())
        </code>
        、递归、Lodash 等
       </td>
      </tr>
     </tbody>
    </table>
    <h3>
     4. 如何选择拷贝方式？
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        浅拷贝
       </strong>
       ：适用于对象结构简单或不需要独立内部引用的情况。
      </p>
     </li>
     <li>
      <p>
       <strong>
        深拷贝
       </strong>
       ：适用于对象结构复杂或需要完全独立副本的情况。
      </p>
     </li>
    </ul>
    <h3>
     5. 总结
    </h3>
    <p>
     在 JavaScript 中，浅拷贝和深拷贝是两种常见的对象复制方式。浅拷贝只复制对象的第一层属性，而深拷贝会递归复制对象及其所有内部引用类型字段。根据实际需求选择合适的拷贝方式，可以避免不必要的 bug 和数据污染。
    </p>
