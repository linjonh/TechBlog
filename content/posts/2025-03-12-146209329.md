    <h3>
     SpringBoot入门-(1) Maven
    </h3>
    <h2>
     动机
    </h2>
    <p>
     对于企业级大项目而言，需要手动导入很大Jar包，费时费力，且Jar包之间也可能存在依赖和冲突，这些关系导致Jar包之间想毛线团一样缠在一起，因此我们需要一个包管理系统帮我们自动下载导入Jar包并且管理好他们之间的依赖和避免冲突，由此Maven诞生。
    </p>
    <ul>
     <li>
      <p>
       <strong>
        Maven
       </strong>
       是一款自动化构建工具，专注服务于
       <strong>
        Java
       </strong>
       平台的
       <strong>
        项目构建和依赖管理
       </strong>
      </p>
     </li>
    </ul>
    <h2>
     安装与配置
    </h2>
    <ol>
     <li>
      <p>
       下载Maven压缩包解压
      </p>
     </li>
     <li>
      <p>
       打开
       <strong>
        MAVEN
       </strong>
       安装目录
       <strong>
        \settings.xml
       </strong>
       文件，修改
       <strong>
        &lt;localRepository&gt;
       </strong>
       标签，设置为新建的本地仓库地址
      </p>
     </li>
     <li>
      <p>
       修改镜像，&lt;mirror&gt;标签添加阿里镜像，下载相关依赖包速度快
      </p>
     </li>
    </ol>
    <h3>
     自动导包原理
    </h3>
    <p>
     从maven中获取Jar包时，maven先从本地仓库中查找，若存在则返回，若不存在从
     <strong>
      中央仓库(镜像)
     </strong>
     取包并保存到本地。
    </p>
    <h3>
     IDEA中配置
    </h3>
    <p>
     在IDEA中配置好相关路径后新建项目即可，如下
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/7a30ac8703044717b23c13c1c8acf52d.png"/>
    </p>
    <p>
     新建Maven项目：
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/7822cea177f649989b2ea078bcac121c.png"/>
    </p>
    <p>
     创建后，项目目录树如下：
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/c51306e08b4943f9b6c6b222cc6d1602.png"/>
    </p>
    <blockquote>
     <ul>
      <li>
       <p>
        main：存放主程序
       </p>
      </li>
      <li>
       <p>
        test：存放测试代码，单元测试等
       </p>
      </li>
      <li>
       <p>
        java：存放源代码
       </p>
      </li>
      <li>
       <p>
        resource：存放配置文件和资源文件
       </p>
      </li>
      <li>
       <p>
        <strong>
         pom.xml
        </strong>
        文件：管理依赖，标识项目
       </p>
      </li>
     </ul>
    </blockquote>
    <h4>
     pom.xml
    </h4>
    <p>
     pom文件中包含我们刚刚新建项目设置的项目坐标，jdk版本等相关信息：
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/22e0bf54ebda48d3b1e664440b9f36af.png"/>
    </p>
    <p>
     我们可以在pom.xml文件中添加依赖，右侧刷新更新依赖即可（自动下载所需依赖包）：
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/79fc613495e847b799be6adfe54f79e3.png"/>
    </p>
    <blockquote>
     <p>
      <strong>
       Project Object Model
      </strong>
      ：项目对象模型。将Java工程的相关信息封装为对象作为便于操作和管理的模型。Maven工程的核心配置。
     </p>
     <p>
      <strong>
       MAVEN
      </strong>
      构建工具可以让对象（项目）和对象之间产生关系
     </p>
    </blockquote>
    <h2>
     相关概念
    </h2>
    <h3>
     坐标
    </h3>
    <ol>
     <li>
      <p>
       使用GAV在Maven仓库中
       <strong>
        唯一
       </strong>
       确定一个Maven工程
      </p>
      <ul>
       <li>
        <p>
         groupId:公司或组织的域名倒序+当前项目名称
        </p>
       </li>
       <li>
        <p>
         artifactId:当前项目的模块名称
        </p>
       </li>
       <li>
        <p>
         version:当前模块的版本信息
        </p>
       </li>
      </ul>
      <pre> &lt;groupId&gt;com.wzu.maven&lt;/groupId&gt;
 &lt;artifactId&gt;Example&lt;/artifactId&gt;
 &lt;version&gt;0.0.1-SNAPSHOT&lt;/version&gt;</pre>
     </li>
    </ol>
    <ol>
     <li>
      <p>
       通过坐标在仓库中查找jar包
      </p>
      <ul>
       <li>
        <p>
         将GAV三个向量连起来即可
        </p>
       </li>
       <li>
        <p>
         com/wzu/maven/Example/0.0.1-SNAPSHOT/Example-0.0.1-SNAPSHOT.jar（相对路径，前面是本地仓库路径）
        </p>
       </li>
       <li>
        <p>
         自己的maven工程要生成jar包供其他项目依赖和使用，执行
         <code>
          mvn install
         </code>
         命令后在对应路径下出现本项目的jar包（已进入仓库）
        </p>
       </li>
      </ul>
     </li>
    </ol>
    <h3>
     maven自动化部署
    </h3>
    <p>
     maven可以自动执行项目构建全流程：清理 -&gt; 编译 -&gt; 测试 -&gt; 报告 -&gt; 打包 -&gt; 部署
    </p>
    <p style="text-align:center">
     <img alt="" src="https://i-blog.csdnimg.cn/direct/bfdd667bc10b49d09fc47667c1644199.png"/>
    </p>
    <h2>
     项目关系
    </h2>
    <ul>
     <li>
      <p>
       依赖关系：标签&lt;dependency&gt;把另一个项目的 jar 引入到当前项目，则maven会自动下载另一个项目的 jar 所依赖的其他项目，不会出现手动导入jar缺失依赖的问题。
      </p>
     </li>
     <li>
      <p>
       聚合关系：
      </p>
      <ul>
       <li>
        <p>
         大项目一般分为模块项目，子项目用&lt;parent&gt;标签继承父项目(pom类型,可删除父工程的
         <strong>
          src
         </strong>
         等文件夹)，此时父项目起到的作用是统一版本等相关信息。
        </p>
       </li>
       <li>
        <p>
         New -&gt; Module 创建子模块，也会生成子模块对应的pom.xml。
        </p>
       </li>
       <li>
        <p>
         父项目的packing类型为pom，子项目一般为jar或war
        </p>
       </li>
       <li>
        <p>
         父项目统一管理子项目的依赖包版本，不真正引入JAR包，依赖放在
         <code>
          &lt;dependencyManagement&gt;
         </code>
         标签中
        </p>
       </li>
       <li>
        <p>
         <strong>
          子项目真正引入
         </strong>
         JAR包，而不用指定版本号,确保各个模块之间版本统一，修改方便
        </p>
       </li>
      </ul>
     </li>
    </ul>
    <blockquote>
     <p>
      如果子项目和
      <code>
       &lt;groupId&gt;
      </code>
      和
      <code>
       &lt;version&gt;
      </code>
      与父项目相同,在子项目中可以不配置
      <code>
       &lt;groupId&gt;
      </code>
      和
      <code>
       &lt;version&gt;
      </code>
     </p>
    </blockquote>
    <h3>
     依赖范围
    </h3>
    <pre> &lt;dependency&gt;
     &lt;groupId&gt;...&lt;/groupId?
     &lt;artifactId&gt;...&lt;/artifactId&gt;
     &lt;scope&gt;...&lt;/scope&gt;
 &lt;/dependency&gt;   </pre>
    <p>
     注意到
     <code>
      &lt;scope&gt;
     </code>
     标签用于控制依赖的范围
    </p>
    <ul>
     <li>
      <p>
       compile （默认）
      </p>
      <ul>
       <li>
        <p>
         main目录下的Java代码可以访问这个范围的依赖
        </p>
       </li>
      </ul>
      <ul>
       <li>
        <p>
         test目录下的Java代码可以访问这个范围的依赖
        </p>
       </li>
      </ul>
      <ul>
       <li>
        <p>
         部署到Tomcat服务器上运行时要放在WEB-INF的lib目录下
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       test
      </p>
      <ul>
       <li>
        <p>
         main目录下的Java代码不能访问这个范围的依赖
        </p>
       </li>
       <li>
        <p>
         test目录下的Java代码可以访问这个范围的依赖
        </p>
       </li>
       <li>
        <p>
         部署到Tomcat服务器上运行时不会放在WEB-INF的lib目录下
        </p>
       </li>
       <li>
        <p>
         例如：对junit的依赖。仅仅是测试程序部分需要。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       provided
      </p>
      <ul>
       <li>
        <p>
         main目录下的Java代码可以访问这个范围的依赖
        </p>
       </li>
       <li>
        <p>
         test目录下的Java代码可以访问这个范围的依赖
        </p>
       </li>
       <li>
        <p>
         部署到Tomcat服务器上运行时不会放在WEB-INF的lib目录下
        </p>
       </li>
       <li>
        <p>
         例如：servlet-api在服务器上运行时，Servlet容器会提供相关API，所以部署的时候不需要
        </p>
       </li>
      </ul>
     </li>
    </ul>
