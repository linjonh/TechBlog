    <h2>
     resultMap
    </h2>
    <p>
     Mybatis中提供了resultMap功能，可以将数据库查询结果映射到Java对象，用于解决
     <span style="color:#fe2c24">
      <strong>
       字段名与属性名不一致
      </strong>
     </span>
     或
     <strong>
      <span style="color:#fe2c24">
       复杂关系（如一对多）
      </span>
     </strong>
     的映射问题。
    </p>
    <p>
     比如一个User类，在它的属性里还有另一个子对象（或者多个），可以是个List或者其他实体类，这个时候就需要用到
    </p>
    <p>
     格式：
    </p>
    <pre><code class="language-XML">&lt;resultMap type="User" id="userOrderMap"&gt;</code></pre>
    <p>
     type:要映射的实体类；id:给该resultMap定义一个唯一标识符。
    </p>
    <p>
    </p>
    <p>
     <strong>
      id、result标签
     </strong>
     ：用于映射指定的属性（不包含除String外的实体类对象），这两者的在映射上没有太多不同。id可以用于标识对象的主键字段，提升性能（MyBatis用主键判断对象是否重复）。
    </p>
    <pre><code class="language-XML">&lt;resultMap id="" type=""&gt;
    &lt;id column="" property=""&gt;&lt;/id&gt;
    &lt;result column="" property=""&gt;&lt;/id&gt;
&lt;/resultMap&gt;</code></pre>
    <p>
     column：对应表的列名
    </p>
    <p>
     property：java对象的属性名
    </p>
    <p>
    </p>
    <p>
     <strong>
      collection标签
     </strong>
     ：用于映射
     <span style="color:#fe2c24">
      <strong>
       一对多的子对象
      </strong>
     </span>
     ，比如将多条数据映射到某个集合中(集合的元素类型是某种实体类)。如果有点晕，就记得：给集合、map等类型的属性赋值，使用collection
    </p>
    <pre><code class="language-XML">&lt;!-- 给list的属性赋值--&gt;
&lt;collection property="" ofType=""&gt;
    &lt;id column="" property=""&gt;&lt;/id&gt;
    &lt;result column="" property=""&gt;&lt;/id&gt;
&lt;/collection&gt;

&lt;!-- 给map属性赋值--&gt;
&lt;collection 
    property=""           
    javaType="java.util.HashMap"  &lt;!-- 固定为某个Map，此处写的是HashMap --&gt;
    ofType=""         
    keyColumn=""           &lt;!-- 指定作为Map键的列 --&gt;
    keyProperty=""              &lt;!-- 键值对中，值对象中作为键的属性 --&gt;
&gt;


    &lt;id column="" property=""/&gt;
    &lt;result column="" property=""/&gt;
&lt;/collection&gt;
</code></pre>
    <p>
     property：父对象中对应的子对象
    </p>
    <p>
     ofType：子对象对应的数据类型，比如该子对象的数据类型是User类，那么此处就填写User。如果是给Map元素赋值，那么该属性的功能是指定 value 的数据类型
    </p>
    <p>
     keyColumn：在映射Map类型数据时，用于指定key 的数据类型
    </p>
    <p>
     给Map映射的内容，光描述比较抽象难懂，下面是个示例：
    </p>
    <pre><code class="language-java">public class Order {
    private Long id;
    private String orderNumber;
    private Map&lt;Long, OrderItem&gt; itemMap; // 键为商品ID，值为OrderItem对象
    
    // Getters &amp; Setters
}

public class OrderItem {
    private Long id;        // 与keyProperty="id"对应
    private String name;
    private Integer quantity;
    
    // Getters &amp; Setters
}

//映射后的数据结构

order.getItemMap() = {
    1001: OrderItem(id=1001, name="商品A", quantity=2),
    1002: OrderItem(id=1002, name="商品B", quantity=5)
}</code></pre>
    <p>
     <strong>
      &lt;association&gt;标签
     </strong>
     ：
    </p>
    <p>
     与使用
     <strong>
      collection标签
     </strong>
     映射List用法一致，只是将collection改成association即可。区别在于，collection是给集合中
     <span style="color:#fe2c24">
      <strong>
       多个
      </strong>
     </span>
     元素的属性进行赋值，association是给父对象的
     <span style="color:#fe2c24">
      <strong>
       某一个子对象元素
      </strong>
     </span>
     中的属性进行赋值。
    </p>
    <p>
     不过javaType在此处应写对应的类类型。
    </p>
    <p>
     示例：
    </p>
    <pre><code class="language-XML">&lt;association property="user" javaType="User"&gt;
  &lt;id column="id" property="id"/&gt;             
  &lt;result column="username" property="name"/&gt;   
  &lt;result column="sex" property="sex"/&gt;        
  &lt;!-- 其他字段映射... --&gt;
&lt;/association&gt;</code></pre>
    <p>
    </p>
