    <h2>
     前言
    </h2>
    <p>
     检测内存错误的工具，有很多个，我今天粗浅的学了一下可在Qt上使用的工具们：
    </p>
    <ul>
     <li>
      Dr.Memory 工具之前我曾在关注的博主上看到相关的博客：
      <a href="https://mahuifa.blog.csdn.net/article/details/142636284?fromshare=blogdetail&amp;sharetype=blogdetail&amp;sharerId=142636284&amp;sharerefer=PC&amp;sharesource=xiaopei_yan&amp;sharefrom=from_link" rel="nofollow" title="C++(Qt)软件调试---内存调试器Dr.Memory（21）_dr. memory-CSDN博客">
       C++(Qt)软件调试---内存调试器Dr.Memory（21）_dr. memory-CSDN博客
      </a>
     </li>
    </ul>
    <p>
     今天尝试了一下，不尽人意。
    </p>
    <ul>
     <li>
      Heob 工具是Qt Creator 的工具栏【分析】下的菜单项有此选项，然后学习了下，简单测试了下，定位是准确的。
     </li>
     <li>
      Vargrind 是用于检测Linux环境内存泄漏的工具。由于当前开发环境是Windows，所以并未测试此工具的可行性。
     </li>
    </ul>
    <p>
    </p>
    <h2>
     环境
    </h2>
    <p>
     win10 + QtCreator 5.15.2 MSVC2019 32bit
    </p>
    <p>
    </p>
    <h2>
     工具
    </h2>
    <h3>
     Dr.Memory 与 Heob的优劣比较
    </h3>
    <blockquote>
     <p>
      ‌
      <strong>
       <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Dr.Memory&amp;rsv_pq=ddb64f0800034f1e&amp;oq=Dr.Memory%20%E4%B8%8E%20Heob%E7%9A%84%E4%BC%98%E5%8A%A3%E6%AF%94%E8%BE%83&amp;rsv_t=9220fVllkXt3A9ffEqc4zxm9awWxyvvah+hVWNbzlvd3qiYmQjpb0471JcKpt4BH5ZJ9&amp;tn=baiduhome_pg&amp;ie=utf-8" rel="nofollow" title="Dr.Memory">
        Dr.Memory
       </a>
       和
       <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Heob&amp;rsv_pq=ddb64f0800034f1e&amp;oq=Dr.Memory%20%E4%B8%8E%20Heob%E7%9A%84%E4%BC%98%E5%8A%A3%E6%AF%94%E8%BE%83&amp;rsv_t=9220fVllkXt3A9ffEqc4zxm9awWxyvvah+hVWNbzlvd3qiYmQjpb0471JcKpt4BH5ZJ9&amp;tn=baiduhome_pg&amp;ie=utf-8" rel="nofollow" title="Heob">
        Heob
       </a>
       各有优缺点，适用于不同的场景和需求。
      </strong>
      ‌
     </p>
     <h4>
      Dr.Memory
     </h4>
     <ul>
      <li>
       <p>
        ‌
        <strong>
         优点
        </strong>
        ‌：
       </p>
       <ul>
        <li>
         ‌
         <strong>
          开源免费
         </strong>
         ‌：Dr.Memory是一个开源工具，用户可以免费使用。
        </li>
        <li>
         ‌
         <strong>
          跨平台支持
         </strong>
         ‌：它支持Windows、Linux等多个操作系统，具有较好的兼容性。
        </li>
        <li>
         ‌
         <strong>
          性能较好
         </strong>
         ‌：相比其他同类工具，Dr.Memory在性能上更有优势，检测速度较快‌。
        </li>
        <li>
         ‌
         <strong>
          功能丰富
         </strong>
         ‌：除了内存泄漏检测，还可以检测内存越界、重复释放内存、空指针、野指针、堆栈溢出等问题‌。
        </li>
       </ul>
      </li>
      <li>
       <p>
        ‌
        <strong>
         缺点
        </strong>
        ‌：
       </p>
       <ul>
        <li>
         ‌
         <strong>
          有时仍较慢
         </strong>
         ‌：尽管比其他工具快，但在某些情况下仍然较慢‌。
        </li>
       </ul>
      </li>
     </ul>
     <h4>
      Heob
     </h4>
     <ul>
      <li>
       <p>
        ‌
        <strong>
         优点
        </strong>
        ‌：
       </p>
       <ul>
        <li>
         ‌
         <strong>
          集成度高
         </strong>
         ‌：Heob特别适用于
         <a href="https://www.baidu.com/s?rsv_dl=re_dqa_generate&amp;sa=re_dqa_generate&amp;wd=Qt%20Creator&amp;rsv_pq=ddb64f0800034f1e&amp;oq=Dr.Memory%20%E4%B8%8E%20Heob%E7%9A%84%E4%BC%98%E5%8A%A3%E6%AF%94%E8%BE%83&amp;rsv_t=9220fVllkXt3A9ffEqc4zxm9awWxyvvah+hVWNbzlvd3qiYmQjpb0471JcKpt4BH5ZJ9&amp;tn=baiduhome_pg&amp;ie=utf-8" rel="nofollow" title="Qt Creator">
          Qt Creator
         </a>
         ，可以很好地与Qt开发环境集成，使用方便‌。
        </li>
        <li>
         ‌
         <strong>
          开源免费
         </strong>
         ‌：Heob是一个开源工具，用户可以免费使用。
        </li>
       </ul>
      </li>
      <li>
       <p>
        ‌
        <strong>
         缺点
        </strong>
        ‌：
       </p>
       <ul>
        <li>
         ‌
         <strong>
          平台限制
         </strong>
         ‌：Heob主要适用于Windows平台，不适用于其他操作系统‌。
        </li>
        <li>
         ‌
         <strong>
          功能限制
         </strong>
         ‌：Heob只能检测动态链接库中的内存泄漏问题，对于其他类型的内存问题检测能力有限‌。
        </li>
       </ul>
      </li>
     </ul>
    </blockquote>
