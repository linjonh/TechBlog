    <h4>
     <span style="color:#000000;">
      Considerations for code search
     </span>
    </h4>
    <p>
     Due to the complexity of searching code, there are a few restrictions on how searches are performed:
    </p>
    <ul>
     <li style="margin-left:30px;">
      Only the default branch is considered. In most cases, this will be the
      <code>
       master
      </code>
      branch.
     </li>
     <li style="margin-left:30px;">
      Only files smaller than 384 KB are searchable.
     </li>
     <li style="margin-left:30px;">
      You must always include at least one search term when searching source code. For example, searching for
      <a href="https://github.com/search?utf8=%E2%9C%93&amp;q=language:go&amp;type=Code" title="language:go">
       language:go
      </a>
      is not valid, while
      <a href="https://github.com/search?utf8=%E2%9C%93&amp;q=amazing+language:go&amp;type=Code" title="amazinglanguage:go">
       amazinglanguage:go
      </a>
      is.
     </li>
     <li style="margin-left:30px;">
      At most, search results can show two fragments from the same file, but there may be more results within the file.
     </li>
     <li style="margin-left:30px;">
      You can't use the following wildcard characters as part of your search query:
      <code>
       ., : ; / \ ` ' " = * ! ? # $ &amp; + ^ | ~ &lt; &gt; ( ) { } []
      </code>
      . The search will simply ignore these symbols.
     </li>
    </ul>
    <p>
     以上是代码搜索中需要注意的一些问题
    </p>
    <p>
     默认搜索是从master分支搜索代码
    </p>
    <p>
     只有小于384k的代码才是可以搜索到的
    </p>
    <p>
     搜索的时候必须包含至少一个搜索关键词
     <a href="https://github.com/search?utf8=%E2%9C%93&amp;q=amazing+language:go&amp;type=Code" title="如amazing language:go">
      如amazing language:go
     </a>
    </p>
    <p>
     搜索语句不能有特殊字符如
     <code>
      ., : ; / \ ` ' " = * ! ? # $ &amp; + ^ | ~ &lt; &gt; ( ) { } []
     </code>
     .
    </p>
    <p>
    </p>
    <h4>
     <span style="color:#000000;">
      Scope the search fields
     </span>
    </h4>
    <p>
     指定搜索方式
    </p>
    <ul>
     <li>
      搜索文件中有octocat的代码----
      <span style="color:#000000;">
       <a href="https://github.com/search?q=octocat+in:file&amp;type=Code" title="octocat in:file">
        octocat in:file
       </a>
       （
      </span>
      Matches code where "octocat" appears in the file contents.
      <span style="color:#000000;">
       ）
      </span>
     </li>
     <li>
      搜索路径中有octocat的代码---
      <span style="color:#000000;">
       <a href="https://github.com/search?q=octocat+in:path&amp;type=Code" title="octocat in:path">
        octocat in:path
       </a>
       （
      </span>
      Matches code where "octocat" appears in the path name.
      <span style="color:#000000;">
       ）
      </span>
     </li>
     <li>
      搜索路径中有octocat的代码或者文件中有octocat的代码---
      <span style="color:#000000;">
       <a href="https://github.com/search?q=octocat+in:file,path&amp;type=Code" title="octocat in:file,path">
        octocat in:file,path
       </a>
       （
      </span>
      Matches code where "octocat" appears in the file contents or the path name.
      <span style="color:#000000;">
       ）
      </span>
     </li>
     <li>
      搜索用scss写的包含display的代码---
      <span style="color:#000000;">
       <a href="https://github.com/search?q=display+language:scss&amp;type=Code" title="display language:scss">
        display language:scss
       </a>
       （
      </span>
      Matches code with the word "display," that's marked as being SCSS.
      <span style="color:#000000;">
       ）
      </span>
     </li>
     <li>
      搜索包含Integer的字段---
      <span style="color:#000000;">
       <a href="https://github.com/search?q=Integer&amp;type=Code" title="Integer">
        Integer
       </a>
       (
      </span>
      Matches code with the word "Integer".
      <span style="color:#000000;">
       )
      </span>
     </li>
    </ul>

     五、加载慢，访问不了
    </h4>
    <p>
     针对Windows下访问GitHub慢，有些图片加载不全的情况，进行如下处理
    </p>
    <p>
     （1）查看IP：打开
     <a href="http://tool.chinaz.com/dns/" rel="nofollow" title="DNS查询网站">
      DNS查询网站
     </a>

    </p>

    <p>
     （2）修改hosts文件
    </p>
    <p>
     利用记事本或者notepad++打开C:\Windows\System32\drivers\etc\hosts（修改前可以先另存为一份，保证出错能恢复），然后添加以下内容

     （3）刷新DNS
    </p>
    <p>
     在cmd中输入
     <code>
      ipconfig /flushdns
     </code>
     即可刷新，然后重启浏览器，不行的话再重启电脑，这样就能成功打开github了，所有图片和内容都能加载。
    </p>
    <p>
     在gitee上有大佬总结了访问策略，具体可查看链接：
    </p>
    <pre><code>https://gitee.com/docmirror/dev-sidecar</code></pre>
