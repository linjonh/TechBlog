    <p>
     <strong>
      3. 性能分析
     </strong>
     <br/>
     （1）优点：
    </p>
    <ul>
     <li>
      稳定排序（相同值保持原顺序）
     </li>
     <li>
      时间复杂度稳定O(n log n)
     </li>
     <li>
      适合链表等非连续存储结构
     </li>
    </ul>
    <p>
     （2）缺点：
    </p>
    <ul>
     <li>
      需要额外O(n)存储空间
     </li>
     <li>
      递归调用栈可能造成内存开销
     </li>
     <li>
      小规模数据效率低于插入排序
     </li>
    </ul>

    <p>

    <p>
     <strong>
      2. 举例说明：
     </strong>
     <br/>
     （1）步骤1：构建“大根堆”（确保“子节点”永远小于其“双亲节点”）
    </p>
    <pre><code class="prism language-bash">初始数组：<span class="token punctuation">[</span><span class="token number">12,11</span>,13,5,6,7<span class="token punctuation">]</span>

  调整节点0（12）→ 与13交换 → <span class="token punctuation">[</span><span class="token number">13,11</span>,12,5,6,7<span class="token punctuation">]</span>
</code></pre>
    <p>
     （2）步骤2：排序阶段，本质是“大根堆”的性质维护
    </p>
    <pre><code class="prism language-bash">第1次交换后：<span class="token punctuation">[</span><span class="token number">7,11</span>,12,5,6,13<span class="token punctuation">]</span>

<span class="token punctuation">..</span>.
</code></pre>
    <p>
     <strong>
      3. 性能分析
     </strong>
     <br/>
     （1）优点：
    </p>
    <ul>
     <li>
      原地排序（空间复杂度O(1)）
     </li>
     <li>
      最坏情况仍保持O(n log n)
     </li>
     <li>
      适合实时数据处理
     </li>
    </ul>
    <p>
     （2）缺点：
    </p>
    <ul>
     <li>
      不稳定排序
     </li>
     <li>
      缓存不友好（跳跃访问）
     </li>
     <li>
      实现复杂度较高
     </li>
    </ul>

     <a id="_171">
     </a>
     三、桶排序
    </h4>
    <p>
     <strong>
      1. 基本步骤：
     </strong>
     <br/>
     （1）步骤1：分桶策略
     <br/>
     示例：数据范围0-49，桶大小10 → 5个桶（0-9,10-19,…40-49）
     <br/>
     （2）步骤2：桶内排序
     <br/>
     使用任意排序算法，对桶内的数据进行排序。因此，桶内排序算法的选择将影响整体性能。
    </p>
    <p>
     <strong>
      2. 性能分析
     </strong>
     <br/>
     （1）优点：
    </p>
    <ul>
     <li>
      线性时间复杂度O(n+k)（k为桶数量）
     </li>
     <li>
      稳定排序（若桶内排序稳定）
     </li>
     <li>
      适合外部排序场景
     </li>
    </ul>
    <p>
     （2）缺点：
    </p>
    <ul>
     <li>
      依赖数据均匀分布
     </li>
     <li>
      需要额外内存存储桶
     </li>
     <li>
      空桶会造成空间浪费
     </li>
    </ul>
