    <h2>
     简述
    </h2>
    <p>
     git传输大于25M的文件时会报错，需要使用 Git LFS进行文件传输。
    </p>
    <p class="img-center">
     <img alt="" height="213" src="https://i-blog.csdnimg.cn/direct/11d3acdb66f940deb29f8c0e79ab40c1.png" width="562"/>
    </p>
    <p>
     Git LFS（Large File Storage）是 GitHub 推荐的方式，可以管理大文件而不会影响 Git 性能。
    </p>
    <h2>
     操作流程
    </h2>
    <pre><code class="language-bash"># 安装 Git LFS
git lfs install

# 将 PDF 文件添加到 Git LFS 追踪
git lfs track "文件名"

# 提交更改
git add .gitattributes
git add 文件名
git commit -m "Add PDF file using LFS"

# 验证网络是否通
ping github.com

# 推送到 GitHub
git push origin main
</code></pre>
