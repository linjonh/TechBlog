    <h3>
     <a id="1_Maven__0">
     </a>
     1.增加 Maven 编译的堆内存
    </h3>
    <p>
     <em>
      <strong>
       我是用这个方法修改成功的
      </strong>
     </em>
     <br/>
     Maven 编译时使用的 JVM 堆内存可以通过设置 MAVEN_OPTS 环境变量来调整。
    </p>
    <p>
     在 IntelliJ IDEA 中设置：
     <br/>
     打开 IntelliJ IDEA 的设置（File -&gt; Settings 或 Ctrl + Alt + S）。
     <br/>
     导航到 Build, Execution, Deployment -&gt; Build Tools -&gt; Maven -&gt; Runner。
    </p>
    <p>
     在 VM Options 字段中，增加堆内存设置，例如：
    </p>
    <pre><code class="prism language-bash"><span class="token parameter variable">-Xmx2048m</span> <span class="token parameter variable">-Xms1024m</span>
</code></pre>
    <p>
     -Xmx2048m：设置最大堆内存为 2048 MB。
     <br/>
     -Xms1024m：设置初始堆内存为 1024 MB。
    </p>
    <p>
     点击 Apply 并重新运行 Maven 编译。
    </p>
    <p>
     在命令行中设置：
     <br/>
     如果你在命令行中使用 Maven，可以通过设置 MAVEN_OPTS 环境变量来增加堆内存：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">MAVEN_OPTS</span><span class="token operator">=</span><span class="token string">"-Xmx2048m -Xms1024m"</span>
mvn clean <span class="token function">install</span>
</code></pre>
    <h3>
     <a id="2__IntelliJ_IDEA__26">
     </a>
     2. 增加 IntelliJ IDEA 的堆内存
    </h3>
    <p>
     如果 IntelliJ IDEA 本身的内存不足，也可能导致编译失败。可以通过以下步骤增加 IDEA 的堆内存：
    </p>
    <p>
     找到 IntelliJ IDEA 的配置文件：
    </p>
    <p>
     Windows: idea64.exe.vmoptions（位于 IDEA 安装目录的 bin 文件夹中）。
    </p>
    <p>
     macOS/Linux: idea.vmoptions（位于 IDEA 安装目录的 bin 文件夹中）。
    </p>
    <p>
     编辑文件，增加以下配置：
    </p>
    <pre><code class="prism language-bash"><span class="token parameter variable">-Xmx2048m</span> <span class="token parameter variable">-Xms1024m</span>
</code></pre>
    <p>
     保存文件并重启 IntelliJ IDEA。
    </p>
    <h3>
     <a id="3__Maven__43">
     </a>
     3. 优化 Maven 编译
    </h3>
    <p>
     如果项目较大，可以尝试优化 Maven 编译过程，减少内存占用：
    </p>
    <p>
     分模块编译
     <br/>
     如果项目是多模块的，可以单独编译某个模块，而不是一次性编译整个项目：
    </p>
    <pre><code class="prism language-bash">mvn clean <span class="token function">install</span> <span class="token parameter variable">-pl</span> 模块名 <span class="token parameter variable">-am</span>
</code></pre>
    <p>
     跳过测试
     <br/>
     测试可能会占用大量内存，可以在编译时跳过测试：
    </p>
    <pre><code class="prism language-bash">mvn clean <span class="token function">install</span> <span class="token parameter variable">-DskipTests</span>
</code></pre>
    <p>
     使用增量编译
     <br/>
     在 IntelliJ IDEA 中，启用增量编译可以减少每次编译的文件数量：
    </p>
    <p>
     打开 File -&gt; Settings -&gt; Build, Execution, Deployment -&gt; Compiler。
    </p>
    <p>
     勾选 Enable incremental compilation。
    </p>
    <h3>
     <a id="4__68">
     </a>
     4. 检查代码和依赖
    </h3>
    <p>
     检查代码：确保没有内存泄漏或不合理的内存使用。
    </p>
    <p>
     减少依赖：检查 pom.xml，移除不必要的依赖。
    </p>
    <p>
     优化资源：如果项目中有大量资源文件（如图片、配置文件等），确保它们没有被错误地包含在编译过程中。
    </p>
    <h3>
     <a id="5__JVM__75">
     </a>
     5. 调整 JVM 参数
    </h3>
    <p>
     如果以上方法仍无法解决问题，可以尝试调整更多的 JVM 参数，例如：
    </p>
    <p>
     -XX:MaxMetaspaceSize：增加元空间大小。
    </p>
    <p>
     -XX:ReservedCodeCacheSize：增加代码缓存大小。
    </p>
    <p>
     示例：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">MAVEN_OPTS</span><span class="token operator">=</span><span class="token string">"-Xmx2048m -Xms1024m -XX:MaxMetaspaceSize=512m -XX:ReservedCodeCacheSize=256m"</span>
</code></pre>
    <p>
     mvn clean install
    </p>
    <h3>
     <a id="6__Maven__JDK_89">
     </a>
     6. 升级 Maven 和 JDK
    </h3>
    <p>
     确保你使用的是最新版本的 Maven 和 JDK。旧版本可能存在内存管理问题。
    </p>
    <p>
     总结
     <br/>
     增加 Maven 和 IntelliJ IDEA 的堆内存是最直接的解决方案。
     <br/>
     优化编译过程和项目结构可以减少内存占用。
     <br/>
     如果问题仍然存在，检查代码和依赖，确保没有内存泄漏或不合理的内存使用。
     <br/>
     通过以上方法，应该可以解决 java.lang.OutOfMemoryError: Java heap space 的问题。
    </p>
