    <h2>
     <a id="_4">
     </a>
     前言
    </h2>
    <p>
     在当今数据驱动的时代，高效且灵活的数据库管理系统对于各类应用程序的开发与运行至关重要。MongoDB作为一款领先的非关系型数据库，以其出色的文档存储模型、高扩展性以及对复杂数据结构的良好支持，在大数据处理、Web应用开发等众多领域得到了广泛应用。
    </p>
    <p>
     无论是Windows系统下的开发环境搭建，还是Linux系统中的服务器部署，掌握MongoDB的安装与配置方法都是迈向数据管理与应用开发征程的关键一步。本教程旨在为读者提供详尽且易懂的Windows和Linux系统中单机MongoDB的安装指南，从下载安装包开始，逐步深入到环境变量配置、目录与文件创建、配置文件调整，再到最后的启动与客户端访问，每一个步骤都配有清晰的操作说明和直观的截图示例，确保无论是初涉数据库领域的新手，还是寻求快速部署方案的专业人士，都能轻松上手，顺利搭建起属于自己的MongoDB运行环境，为后续的数据存储、查询与分析工作奠定坚实基础。
    </p>

      <code>

     </li>
     <li>
      在弹出的“
      <code>
       系统属性
      </code>

      <code>
       环境变量
      </code>

     <code>
      D:\bigdata\mongodb-win32-x86_64-windows-5.0.30\bin

     ，点击“确定”保存更改，然后连续点击每个窗口上的“
     <code>
      确定

net:
  port: 27017
  bindIp: 0.0.0.0
</code></pre>

    <h3>
     <a id="5__119">
     </a>
     5. 启用身份认证（选看）
    </h3>
    <p>
     在MongoDB中，“开启认证”指的是在启动MongoDB实例时启用身份验证，以确保只有经过授权的用户才能访问数据库。这通常涉及到配置MongoDB使用用户名和密码进行认证，并且需要在启动MongoDB服务时指定相应的参数。
    </p>
    <h4>
     <a id="51__123">
     </a>
     5.1 创建管理员用户（如果尚未创建）
    </h4>
    <p>
     在启用了身份验证之前，需要至少有一个拥有适当权限的用户来管理其他用户的访问。这通常是一个管理员用户，具有
     <code>
      userAdminAnyDatabase
     </code>
     角色或类似的管理权限。
    </p>
    <pre><code class="prism language-javascript">use admin

    <span class="token literal-property property">roles</span><span class="token operator">:</span> <span class="token punctuation">[</span> <span class="token punctuation">{<!-- --></span> <span class="token literal-property property">role</span><span class="token operator">:</span> <span class="token string">"userAdminAnyDatabase"</span><span class="token punctuation">,</span> <span class="token literal-property property">db</span><span class="token operator">:</span> <span class="token string">"admin"</span> <span class="token punctuation">}</span> <span class="token punctuation">]</span>

    <h4>
     <a id="52_MongoDB_141">
     </a>
     5.2 启动MongoDB时启用认证
    </h4>
    <p>
     要启动MongoDB并启用认证，可以使用如下命令：
    </p>
    <pre><code class="prism language-shell">mongod <span class="token parameter variable">--auth</span> <span class="token parameter variable">-f</span> D:<span class="token punctuation">\</span>bigdata<span class="token punctuation">\</span>mongodb-win32-x86_64-windows-5.0.30<span class="token punctuation">\</span>conf<span class="token punctuation">\</span>mongodb.conf
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        <code>
         mongod
        </code>
       </strong>
       ：这是MongoDB数据库服务器的主进程命令。运行此命令以启动MongoDB实例。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         --auth
        </code>
       </strong>
       ：这个标志告诉MongoDB要求所有连接进行身份验证。
      </p>
     </li>
     <li>
      <p>
       <strong>
        <code>
         -f
        </code>
        或
        <code>
         --config
        </code>
       </strong>
       ：这个选项用于指定MongoDB配置文件的位置。
      </p>
     </li>
    </ul>

    </p>

</code></pre>

    <p>

     </strong>

    </p>
