    <ul>
     <li>
      图像基础概念
     </li>
    </ul>
    <pre><code class="prism language-csharp">◼像素：像素是一个图片的基本单位，pix是英语单词picture的简写，加上英
语单词“元素element”，就得到了“pixel”，简称px，所以“像素”有“图像元素”
之意。
◼ 分辨率：是指图像的大小或尺寸。比如1920x1080。
◼ 位深：是指在记录数字图像的颜色时，计算机实际上是用每个像素需要的
位深来表示的。比如红色分量用8bit。
◼ 帧率：在<span class="token number">1</span>秒钟时间里传输的图片的帧数，也可以理解为图形处理器每秒钟
能够刷新几次。比如25fps表示一秒有<span class="token number">25</span>张图片。
◼ 码率：视频文件在单位时间内使用的数据流量。比如1Mbps。
◼ Stride：指在内存中每行像素所占的空间。为了实现内存对齐每行像素在
内存中所占的空间并不一定是图像的宽度
</code></pre>
    <ul>
     <li>
      1.1 像素
     </li>
    </ul>
    <pre><code class="prism language-csharp">像素是一个图片的基本单位，例如<span class="token number">2500</span>×<span class="token number">2000</span>的照片就是指横向有<span class="token number">2500</span>个像素点，竖向有<span class="token number">2000</span>个像素点，总共是<span class="token number">500</span>万
个像素，也俗称<span class="token number">500</span>万像素照片
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/54cb094064534b90aa0fabbb26994c43.png"/>
    </p>
    <ul>
     <li>
      1.2 分辨率
     </li>
    </ul>
    <pre><code class="prism language-csharp">图像（或视频）的分辨率是指图像的大小或尺寸。我们通常用像素表示图像的尺寸。
例如<span class="token number">2500</span>×<span class="token number">2000</span>的照片就是指横向<span class="token punctuation">(</span>宽<span class="token punctuation">)</span>有<span class="token number">2500</span>个像素点，竖向<span class="token punctuation">(</span>高<span class="token punctuation">)</span>有<span class="token number">2000</span>个像素点。
常见的分辨率：
<span class="token function">360P</span><span class="token punctuation">(</span>640x360<span class="token punctuation">)</span>、<span class="token function">720P</span><span class="token punctuation">(</span>1280x720<span class="token punctuation">)</span>、<span class="token function">1080P</span><span class="token punctuation">(</span>1920x1080<span class="token punctuation">)</span>、<span class="token function">4K</span><span class="token punctuation">(</span>3840x2160<span class="token punctuation">)</span>、<span class="token function">8K</span><span class="token punctuation">(</span>7680x4320<span class="token punctuation">)</span>
</code></pre>
    <ul>
     <li>
      1.3 位深
     </li>
    </ul>
    <pre><code class="prism language-csharp">我们看到的彩色图片，都有三个通道，分别为红<span class="token punctuation">(</span>R<span class="token punctuation">)</span>、绿<span class="token punctuation">(</span>G<span class="token punctuation">)</span>、蓝<span class="token punctuation">(</span>B<span class="token punctuation">)</span>通道。（如果需
要透明度则还有alpha分量）
通常每个通道用8bit表示，8bit能表示<span class="token number">256</span>种颜色，所以可以组成
<span class="token number">256</span><span class="token operator">*</span><span class="token number">256</span><span class="token operator">*</span><span class="token number">256</span><span class="token operator">=</span><span class="token number">16</span><span class="token punctuation">,</span><span class="token number">777</span><span class="token punctuation">,</span><span class="token number">216</span> <span class="token operator">=</span> <span class="token number">1677</span>万种颜色。
这里的8bit就是我们讲的位深。
每个通道的位深越大，能够表示的颜色值就越大，比如现在高端电视说的10bit色
彩，即是每个通道用10bit表示，每个通道有<span class="token number">1024</span>种颜色。<span class="token number">1024</span><span class="token operator">*</span><span class="token number">1024</span><span class="token operator">*</span><span class="token number">1024</span>约为
<span class="token number">10</span><span class="token punctuation">,</span><span class="token number">7374</span>万色<span class="token operator">=</span><span class="token number">10</span>亿色， 是8bit的<span class="token number">64</span>倍。
常见的颜色还是8bit居多。
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2104171e0a9d4ba98d4a1e5a55b37b71.png"/>
    </p>
    <ul>
     <li>
      1.4 帧率
     </li>
    </ul>
    <pre><code class="prism language-csharp">帧率即 FPS（每秒有多少帧画面），经常玩游戏的同学应该会对这个词很熟悉。我们玩游
戏时，FPS 帧率越高就代表游戏画面越流畅，越低则越卡顿。视频也是如此。
由于视觉图像在视网膜的暂时停留，一般图像帧率能达到<span class="token number">24</span>帧，我们就认为图像是连续
动态的。
电影帧率一般是 24fps（帧每秒）；
电视剧一般是25fps；
监控行业常用 25fps；
音视频通话常用15fps；
帧率越高，画面越流畅，需要的设备性能也越高。
</code></pre>
    <ul>
     <li>
      1.5 码率
     </li>
    </ul>
    <pre><code class="prism language-csharp">◼ 视频文件在单位时间内使用的数据流量。比如1Mbps。
◼ 大多数情况下码率越高 分辨率越高，也就越清晰。但模糊的视频文件大小（码率）也
可以很大，分辨率小的视频文件可能也比分辨率大的视频文件清晰。
◼ 对于同一个原始图像源的时候，同样的编码算法，则码率越高，图像的失真就会越小，
视频画面就会越清晰
</code></pre>
    <ul>
     <li>
      2.1 YUV
     </li>
    </ul>
    <pre><code class="prism language-csharp">YUV也是一种颜色编码方法，它是指将亮度参量（Y：Luminance
或Luma）和色度参量（UV：Chrominance或Chroma）分开进行表示的像素编码格式。
◼ 这样分开的好处就是不但可以避免相互干扰，没有UV信息一样可以显示完整的图像，
因而解决了彩色电视与黑白电视的兼容问题；还可以降低色度的采样率而不会对图像
质量影响太大，降低了视屏信号传输时对频宽（带宽）的要求。
◼ Y Y共用一组UV分量
◼ 打包（packed）格式：将每个像素点的Y、U、V分量交叉排列并以像素点为单元连续的存放
在同一数组中，通常几个相邻的像素组成一个宏像素（macro<span class="token operator">-</span>pixel）
◼ 平面（planar）格式：使用三个数组分开连续的存放Y、U、V三个分量，即Y、U、V分别存
放在各自的数组中
</code></pre>
