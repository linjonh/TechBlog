    <h3>
     1 MEC概念
    </h3>
    <p>
     <strong>
      MEC
     </strong>
     边缘云（
     <strong>
      Multi-Access Edge Cloud
     </strong>
     ）是在靠近人、物或数据源头的网络边缘侧部署云资源池和云平台，它在
     <strong>
      5G
     </strong>
     时代将扮演重要的角色。
     <strong>
      MEC
     </strong>
     边缘云的优势在于，一方面它实现了在更靠近数据源所在的本地网内运算，尽可能地不用将数据回传到云端，减少数据往返云端的等待时间和网络成本，大幅度降低了对运营商核心网和传输网的拥塞与负担，减缓网络带宽压力。另一方面，
     <strong>
      MEC
     </strong>
     边缘云作为一个能力开放平台，不仅提供基本的云资源，同时提供丰富的平台能力。通过开放网络能力与大数据、云计算平台结合，使得第三方应用部署到网络边缘，提升用户业务体验和指标。目前
     <strong>
      3GPP
     </strong>
     协议在23.501和23.502中已经明确将
     <strong>
      5G
     </strong>
     网络支持边缘计算特性作为标准规范，未来在
     <strong>
      5G
     </strong>
     网络建设中，将考虑支持相关业务及设备部署。
    </p>
    <h3>
     <strong>
      2 运营商5G MEC边缘云组网架构
     </strong>
    </h3>
    <h4>
     <strong>
      2.1 5G NSA架构下MEC边缘云组网
     </strong>
    </h4>
    <p>
     由于国内运营商将首先部署
     <strong>
      5G NSA
     </strong>
     组网，因此针对不同的组网架构需要考虑不同的
     <strong>
      MEC
     </strong>
     边缘云组网方案。国内运营商
     <strong>
      NSA
     </strong>
     架构采用
     <strong>
      Option3
     </strong>
     系列组网方案中的
     <strong>
      Option3x
     </strong>
     方案。
    </p>
    <p>
     该系列
     <strong>
      NSA
     </strong>
     组网架构中，核心网采用
     <strong>
      EPC+
     </strong>
     ，是
     <strong>
      4G EPC
     </strong>
     的演进。
     <strong>
      LTE eNB
     </strong>
     和
     <strong>
      5G NR
     </strong>
     同
     <strong>
      EPC+
     </strong>
     的接口为
     <strong>
      S1
     </strong>
     接口，
     <strong>
      S1-C
     </strong>
     为控制面接口，
     <strong>
      S1-U
     </strong>
     为用户面接口。所有的
     <strong>
      Option3
     </strong>
     系列组网方案中，
     <strong>
      S1-C
     </strong>
     控制面信令均走
     <strong>
      LTE eNB
     </strong>
     锚点。根据用户面锚点的情况，
     <strong>
      Option3
     </strong>
     中仅用
     <strong>
      LTE eNB
     </strong>
     作为用户面锚点，
     <strong>
      Option3a
     </strong>
     中
     <strong>
      LTE eNB
     </strong>
     和
     <strong>
      NR
     </strong>
     均提供用户面路径到
     <strong>
      EPC+
     </strong>
     ，而
     <strong>
      Option3x
     </strong>
     用
     <strong>
      NR
     </strong>
     作为唯一的用户面路径。
     <strong>
      NSA
     </strong>
     架构下，终端通过双连接同时接入
     <strong>
      4G
     </strong>
     和
     <strong>
      5G
     </strong>
     网络。
     <strong>
      NSA
     </strong>
     模式下，要求
     <strong>
      5G/4G
     </strong>
     设备同厂商。
    </p>
    <p>
     图1为
     <strong>
      5G NSA Option3x MEC

     边缘云组网架构，
     <strong>
      eNB
     </strong>
     和
     <strong>
      gNB

     分别作为控制面和用户面的锚点，
     <strong>
      MEC
     </strong>
     边缘云需要将
     <strong>
      S1-U
     </strong>
     和
     <strong>
      S1-C
     </strong>

     <strong>
      Option3x
     </strong>
     方案中，由于
     <strong>
      eNB
     </strong>
     和
     <strong>
      gNB
     </strong>

     <strong>
      Option3x
     </strong>
     的方式施工成本最大，新增
     <strong>
      MEC
     </strong>
     边缘云同无线设备的连接无法避免，仅在
     <strong>
      4G
     </strong>
     和
     <strong>
      5G
     </strong>
     共
     <strong>
      BBU
     </strong>
     的时候可以节约建设成本。

     <strong>
      5G NSA
     </strong>
     架构下，
     <strong>
      MEC
     </strong>
     边缘云作为独立网元同其他网元之间的连接。目前建设初期，
     <strong>
      MEC
     </strong>
     边缘云仅处理基本的数据分流、基站信息开放等功能。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/9d93938eaa484b5c6f8d54a3284137cb.png"/>

    <p>
     如图3、图4所示，由于
     <strong>
      5G NSA
     </strong>
     沿袭了
     <strong>
      4G
     </strong>
     的大部分
     <strong>
      EPC
     </strong>
     架构，导致
     <strong>
      MEC
     </strong>
     边缘云的部署仍然无法解决计费、合法监听等问题。解决方法是将
     <strong>
      MEC
     </strong>
     边缘云演进到增强型边缘云，
     <strong>
      MEC
     </strong>
     边缘云具备处理控制面信令的能力，还支持同其他
     <strong>
      4G
     </strong>
     核心网网元的对接，包括
     <strong>
      CG
     </strong>
     （
     <strong>
      Charging GW

     <strong>
      P-GW
     </strong>
     、
     <strong>
      LIG
     </strong>
     （
     <strong>
      Lawful Interception Gateway
     </strong>
     ，警用监听中心）等，以实现对计费、合法监听等功能的支持。业务的支持同
     <strong>
      4G
     </strong>
     完全一致，通过对应的
     <strong>
      Tracking Area
     </strong>
     指向
     <strong>
      MEC
     </strong>
     边缘云，本地流量直接转发到本地应用，非本地流量则与上级机房的
     <strong>
      PGW
     </strong>
     通过
     <strong>
      S5
     </strong>
     接口对接，不需分流的业务通过
     <strong>
      S5
     </strong>
     接口上送，只需要对
     <strong>
      MEC
     </strong>
     边缘云配置好分流规则即可。计费功能：通过与上级机房的
     <strong>
      CG
     </strong>
     通过
     <strong>
      Ga
     </strong>
     接口对接，实现分流业务的计费；合法监听：通过与上级机房的
     <strong>
      LIG
     </strong>
     通过
     <strong>
      Xn（X1/X2/X3）
     </strong>
     接口对接，实现合法监听。
    </p>

    <h3>
     <strong>
      3 MEC边缘云应用场景和案例分析
     </strong>
    </h3>
    <h4>
     <strong>
      3.1 本地分流
     </strong>
    </h4>
    <p>
     目前产业界应用最为广泛的是
     <strong>
      MEC
     </strong>
     边缘云的分流技术，即本地分流（
     <strong>
      LBO, Local Break Out
     </strong>
     ）。如图8所示，本地分流的作用在于流量不出园区，可以将符合分流规则的流量分流到本地网，节约了流量回传（回传网、核心网、
     <strong>
      Internet
     </strong>
     ）造成的时延，同时也能够降低对骨干网的流量压力。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/1a3435815e2df500c4cfd97923866772.png"/>
    </p>
    <p>
     在
     <strong>
      MEC
     </strong>
     边缘云侧配置分流规则，通常采用
     <strong>
      IP
     </strong>
     五元组法和基于
     <strong>
      DNS
     </strong>
     域名解析的分流规则。基于
     <strong>
      IT
     </strong>
     五元组的分流规则：
     <strong>
      IP
     </strong>
     地址+端口号；基于
     <strong>
      DNS
     </strong>
     响应的分流规则：
     <strong>
      DNS
     </strong>
     目标域名。
     <strong>
      MEC
     </strong>
     对所有
     <strong>
      eNB
     </strong>
     覆盖区域流量进行拆包检测，符合分流规则的访问流量直接分流至本地网，不符合分流规则的按照原有路径访问
     <strong>
      Internet
     </strong>
     。
     <strong>
      MEC
     </strong>
     部署层级决定了可接入基站数和覆盖面积；可以接入基站数决定了
     <strong>
      MEC
     </strong>
     需要处理的流量数。
     <strong>
      MEC
     </strong>
     的部署位置属于串接，如果
     <strong>
      MEC
     </strong>
     故障，则流量完全透传，仅无法实现分流，不影响业务。
    </p>
    <h4>
     <strong>
      3.2 安防监控
     </strong>
    </h4>
    <p>
     目前安防监控领域，从部署安装角度，一般传统的监控部署采用有线方式，有线网络覆盖全部的摄像头，布线成本高，效率低，占用大量有线资源。采用
     <strong>
      Wi-Fi
     </strong>
     回传的方式也广泛应用，经济便捷，但是
     <strong>
      Wi-Fi
     </strong>
     稳定性较差，覆盖范围较小，需要补充大量
     <strong>
      AP
     </strong>
     点以保证覆盖和稳定性。
    </p>
    <p>
     安防监控领域 “端-网-边”格局如图9所示。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/7e1389a46798a5ddcc11c51f7ebec62e.png"/>
    </p>
    <p>
     基于上述诉求，可以将监控数据分流到边缘云业务平台，从而有效降低网络传输压力和业务端到端时延。图10为基于边缘云的边缘业务平台的
     <strong>
      AI
     </strong>
     视频分析系统协同架构。边缘云对
     <strong>
      4G
     </strong>
     摄像头采集的视频进行本地分流，降低对核心网及骨干网传输带宽资源的占用，缩短端到端时延。集中云计算中心执行AI的
     <strong>
      Training
     </strong>
     任务，边缘云执行
     <strong>
      AI
     </strong>
     的
     <strong>
      Inference
     </strong>
     ，二者协同可实现本地决策、实时响应，可实现表情识别、行为检测、轨迹跟踪、热点管理、体态属性识别等多种本地AI典型应用。此外，有线摄像头亦可以接入边缘云智能分析平台。
    </p>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/790d528ea74baf8c594ff453e93d4533.png"/>
    </p>
    <h4>
     <strong>
      3.3 工业互联网
     </strong>
    </h4>
    <p>
     面向工业互联网的边缘计算体系架构可以从多种角度进行描述，其中，从部署的角度看，在产线、车间和工厂内均能够部署边缘计算节点设备。这些节点设备均能够提供网络、计算、存储等
     <strong>
      ICT
     </strong>
     资源，但设备形态可以分为边缘控制器，边缘网关和边缘云。
    </p>

     <strong>
      LTE/5G-MEC-IoT
     </strong>
     无线专网，实现工业园区本地分流，保证业务低时延。基于
     <strong>
      MEC
     </strong>
     边缘云技术，本地提供云计算能力并搭载
     <strong>
      AI
     </strong>
     机器视觉能力，面向制衣厂进行印花检测和布料检测。

    <h3>
     <strong>
      4 结论
     </strong>
    </h3>
    <p>
     作为一项新的
     <strong>
      ICT
     </strong>
     融合技术，
     <strong>
      MEC
     </strong>
     边缘云可以整合电信运营商的各类资源，为
     <strong>
