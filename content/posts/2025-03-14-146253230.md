    <h2>
     <a id="_4">
     </a>
     一、核心作用
    </h2>
    <p>
     <code>
      BootstrapRegistryInitializer
     </code>
     是 Spring Boot 启动流程中的
     <strong>
      早期初始化扩展点
     </strong>
     ，主要用于在应用上下文（
     <code>
      ApplicationContext
     </code>
     ）创建之前，向
     <code>
      BootstrapRegistry
     </code>
     注册必要对象或执行自定义逻辑。其核心作用包括：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        早期对象注册
       </strong>
       <br/>
       在 Spring 容器初始化前，预注册需要在后续阶段（如环境准备、配置加载）中使用的对象（如配置中心客户端、数据库连接池原型）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        环境隔离
       </strong>
       <br/>
       通过
       <code>
        BootstrapRegistry
       </code>
       存储的对象独立于主应用上下文，避免未完全初始化的 Bean 污染容器。
      </p>
     </li>
     <li>
      <p>
       <strong>
        生命周期扩展
       </strong>
       <br/>
       允许在应用启动的最初阶段（如加载外部配置、初始化基础设施）插入自定义逻辑，解决传统
       <code>
        @PostConstruct
       </code>
       或
       <code>
        ApplicationRunner
       </code>
       无法覆盖的场景。
      </p>
     </li>
    </ol>

    <h2>
     <a id="_48">
     </a>
     三、典型应用场景
    </h2>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         场景
        </strong>
       </th>
       <th>
        <strong>
         实现方式
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         外部配置预加载
        </strong>
       </td>
       <td>
        注册配置中心客户端（如 Spring Cloud Config Client），提前拉取远程配置。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         基础设施初始化
        </strong>
       </td>
       <td>
        初始化日志系统、监控代理（如 Prometheus Client）或分布式追踪工具（如 Zipkin）。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         动态数据源注册
        </strong>
       </td>
       <td>
        根据环境变量预注册多数据源模板，供后续
        <code>
         DataSourceAutoConfiguration
        </code>
        复用。
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         服务发现预注册
        </strong>
       </td>
       <td>
        提前连接服务注册中心（如 Consul、Nacos），避免主应用启动时的首次调用超时。
       </td>
      </tr>
     </tbody>
    </table>

    <h2>
     <a id="_58">
     </a>
     四、关键执行流程
    </h2>
    <ol>
     <li>
      <p>
       <strong>
        启动阶段
       </strong>
       <br/>
       Spring Boot 在
       <code>
        SpringApplication.run()
       </code>
       的早期阶段（创建
       <code>
        ApplicationContext
       </code>
       前），通过
       <code>
        SpringFactoriesLoader
       </code>
       加载所有
       <code>
        BootstrapRegistryInitializer
       </code>
       实现。
      </p>
     </li>
     <li>
      <p>
       <strong>
        初始化顺序
       </strong>
       <br/>
       初始化器按
       <code>
        @Order
       </code>
       注解或
       <code>
        spring.factories
       </code>
       中的声明顺序执行，确保依赖关系正确。
      </p>
     </li>
     <li>
      <p>
       <strong>
        对象共享
       </strong>
       <br/>
       注册到
       <code>
        BootstrapRegistry
       </code>
       的对象可通过
       <code>
        BootstrapContext
       </code>
       传递给后续阶段（如
       <code>
        EnvironmentPostProcessor
       </code>
       ）。
      </p>
     </li>
    </ol>
    <hr/>
    <h2>
     <a id="_70">
     </a>
     五、注意事项
    </h2>
    <ul>
     <li>
      <p>
       <strong>
        避免依赖注入
       </strong>
       <br/>
       <code>
        initialize
       </code>
       方法执行时 Spring 容器尚未创建，无法使用
       <code>
        @Autowired
       </code>
       或
       <code>
        @Value
       </code>
       注解。
      </p>
     </li>
     <li>
      <p>
       <strong>
        延迟初始化优化
       </strong>
       <br/>
       使用
       <code>
        InstanceSupplier
       </code>
       延迟对象创建，避免启动时不必要的资源消耗：
      </p>
      <pre><code class="prism language-java">registry<span class="token punctuation">.</span><span class="token function">register</span><span class="token punctuation">(</span><span class="token class-name">ExpensiveBean</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">,</span> context <span class="token operator">-&gt;</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">return</span> <span class="token keyword">new</span> <span class="token class-name">ExpensiveBean</span><span class="token punctuation">(</span>context<span class="token punctuation">.</span><span class="token function">get</span><span class="token punctuation">(</span><span class="token class-name">ConfigParams</span><span class="token punctuation">.</span><span class="token keyword">class</span><span class="token punctuation">)</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
     </li>
     <li>
      <p>
       <strong>
        作用域限制
       </strong>
       <br/>
       <code>
        BootstrapRegistry
       </code>
       中的对象默认生命周期为
       <code>
        Scope.REFRESH
       </code>
       ，重启应用上下文时不会重新创建。
      </p>
     </li>
    </ul>
    <hr/>
    <p>
     <strong>
      总结
     </strong>
     ：
     <code>
      BootstrapRegistryInitializer
     </code>
     为 Spring Boot 提供了
     <strong>
      启动初期扩展能力
     </strong>
     ，适合需要早于容器初始化执行的底层逻辑。通过合理设计，可显著提升应用启动效率并解决模块化配置难题。
    </p>
