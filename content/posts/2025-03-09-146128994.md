    <h2>
     <a id="_4">
     </a>
     解析
    </h2>
    <p>
     每个外卖店会在不同的时间点收到订单，我们可以看见测试用例的时间
     <code>
      顺序是不同
     </code>
     的，而且有在
     <code>
      相同时间内有多个订单
     </code>
     的店铺
    </p>
    <p>
     如果我们按输入的顺序来计算，显而易见是求不出来的，所以我们需要
     <code>
      按时间顺序来处理
     </code>
     订单，然后计算优先级，并判断是否在优先缓存中。
    </p>
    <p>
     那么问题来了，我们应该如何对时间进行排序，用数组然后用sort？但是一个id，对应了多个时间点，一维数组显然难以实现，用二维数组？（不知道行不行，但是我知道太复杂了）
    </p>
    <p>
     这里我们会想要一个以id来区分的存储器存储时间点，并且能够对时间点排序
    </p>
    <p>
     没错没错，就只有他了优先队列数组【优先队列还不行，优先队列不能存储id】
    </p>
    <p>
     优先队列的作用：
     <br/>
     1.按时间顺序处理订单，避免手动排序的复杂性。
     <br/>
     2.不同下标的队列数组，内容互不影响。
     <br/>
     3.时间复杂度：每个订单插入和取出操作的时间复杂度为 O(log m)，总复杂度为 O(m log m + n)，适用于大规模数据。
    </p>
    <p>
     找到存取id和时间点的最佳容器后，我们需要遍历每一个id，计算时间点，判断是否在优先缓存中。
    </p>
    <p>
     注意：题目中问的是t时刻，有些店铺在t时刻前就没有订单了，我们还需计算最后一个时刻到t时刻减少的优先级
    </p>

     优先队列
    </h3>
    <pre><code class="prism language-cpp">priority_queue<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token punctuation">,</span> vector<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;</span><span class="token punctuation">,</span> greater<span class="token operator">&lt;</span><span class="token keyword">int</span><span class="token operator">&gt;&gt;</span> h<span class="token punctuation">;</span>

    <p>
     优先队列 (priority_queue)：
     <br/>
     是 C++ 中一种特殊的数据结构，它和普通队列（先进先出）不同，元素的出队顺序由优先级决定。
     <br/>
     它是 C++ STL 中的一种容器，默认是大顶堆（队首元素最大）。
     <br/>
     但这里通过 greater 指定为小顶堆（队首元素最小）。
     <br/>
     priority_queue 的完整模板参数列表如下：
    </p>
    <h4>
     <a id="_45">
     </a>
     如何判断是否使用优先队列？
    </h4>
    <p>
     1.是否要求动态获取最大值/最小值？
     <br/>
     是 → 优先队列。
     <br/>
     2.是否需要按特定顺序处理元素？
     <br/>
     是 → 优先队列。
     <br/>
     3.是否涉及贪心策略（每次选最优解）？
     <br/>
     是 → 优先队列。
    </p>
