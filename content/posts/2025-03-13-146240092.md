    <h2>
     <a id="_0">
     </a>
     装饰器模式
    </h2>
    <p>
     <font color="red">
      装饰器是指能动态地为对象添加额外的功能的一种结构型设计模式。
     </font>
    </p>
    <h3>
     <a id="_3">
     </a>
     特点
    </h3>
    <ul>
     <li>
      不修改原有代码的情况下，动态地扩展一个对象的功能。
     </li>
     <li>
      支持多个装饰器叠加使用
     </li>
     <li>
      透明性，装饰后的对象与原对象保持一致，客户端无需感知装饰过程
     </li>
    </ul>
    <h3>
     <a id="_8">
     </a>
     结构模式与实现
    </h3>
    <p>
     python 中
     <code>
      @decorator
     </code>
     语法就是装饰器模式的简化实现，因此这里就不再做代码示例，详情见
     <a href="https://blog.csdn.net/weixin_47612420/article/details/145579933">
      装饰器模式
     </a>
     <br/>
     要说不同之处，装饰器模式的本质是面向对象的，也就是用于装饰类实例。而python中的
     <code>
      @decorator
     </code>
     语法则是函数式编程的装饰器，其本质是函数式编程中的高阶函数。或者说
     <code>
      @decorator
     </code>
     能起到装饰器模式的作用。
    </p>
    <h3>
     <a id="_12">
     </a>
     适用场景
    </h3>
    <ul>
     <li>
      动态拓展对象功能
     </li>
     <li>
      替代多层继承
     </li>
     <li>
      不可修改的类做功能增强
     </li>
    </ul>
    <h3>
     <a id="_17">
     </a>
     优缺点
    </h3>
    <ul>
     <li>
      优点：
      <ul>
       <li>
        符合开闭原则，扩展性良好
       </li>
       <li>
        符合单一职责原则，每个装饰器只负责一个功能
       </li>
       <li>
        装饰顺序可控，支持自由组合
       </li>
      </ul>
     </li>
     <li>
      缺点：
      <ul>
       <li>
        嵌套装饰，嵌套层级过多，代码可读性差
       </li>
       <li>
        调试困难
       </li>
      </ul>
     </li>
    </ul>
