    <h2>
     一、时间复杂度核心概念
    </h2>
    <h4>
     1.1 为什么要研究时间复杂度
    </h4>
    <p>
     当处理大规模数据时（如计算斐波那契数列第57项），不同算法效率差异巨大：
    </p>
    <ul>
     <li>
      <p>
       <strong>
        递推解法
       </strong>
       ：0.23秒完成
      </p>
     </li>
     <li>
      <p>
       <strong>
        递归解法
       </strong>
       ：需要2369秒（约40分钟）
      </p>
     </li>
    </ul>
    <p>
     时间复杂度是衡量算法执行效率的核心指标，用大O符号表示算法执行时间的
     <strong>
      增长趋势
     </strong>
     ，与具体硬件无关。
    </p>
    <h4>
     1.2 基本定义
    </h4>
    <p>
     时间复杂度（Time Complexity）表示算法中
     <strong>
      基本操作执行次数
     </strong>
     与数据规模n的数学关系，反映算法执行时间的增长速率。记为：T(n) = O(f(n))
    </p>

    <h4>
     2.1 典型代码分析
    </h4>
    <h5>
     线性时间 O(n)
    </h5>
    <pre>for(int i=0; i&lt;n; i++){  // 执行n次
    sum += i;            // 基本操作
}</pre>
    <h5>
     平方时间 O(n²)
    </h5>
    <pre>for(int i=0; i&lt;n; i++){       // 外层n次
    for(int j=0; j&lt;n; j++){   // 内层n次
        if(a[i][j] &gt; max)     // 基本操作
            max = a[i][j];
    }
}</pre>
    <h5>
     对数时间 O(logn)
    </h5>
    <pre>int binarySearch(int arr[], int n, int x){
    int left=0, right=n-1;
    while(left &lt;= right){     // 每次规模减半
        int mid = (left+right)/2;
        if(arr[mid] == x) return mid;
        if(arr[mid] &gt; x) right = mid-1;
        else left = mid+1;
    }
    return -1;               // 执行次数：log2n
}</pre>
    <h3>
     三、斐波那契数列解法对比
    </h3>
    <h4>
     3.1 递推解法（动态规划）
    </h4>
    <pre>long long fib[57] = {1,1};
for(int i=2; i&lt;57; i++){
    fib[i] = fib[i-1] + fib[i-2];  // 基本操作执行56次
}
cout &lt;&lt; fib[56];  // 时间复杂度O(n)</pre>
    <h4>
     3.2 递归解法
    </h4>
    <pre>long long fib(int n){
    if(n==1 || n==2) return 1;
    return fib(n-1) + fib(n-2);  // 时间复杂度O(2ⁿ)
}</pre>
    <h5>
     递归树分析（n=5时）：
    </h5>
    <pre>            fib(5)
          /         \
      fib(4)       fib(3)
     /      \       /    \
  fib(3) fib(2) fib(2) fib(1)
  /    \
fib(2) fib(1)</pre>
    <ul>
     <li>
      <p>
       总调用次数：9次 = 2⁵ - 1 ≈ O(2ⁿ)
      </p>
     </li>
    </ul>
    <h3>
     四、时间复杂度计算实战
    </h3>
    <h4>
     4.1 双重循环特殊场景
    </h4>
    <pre>for(int i=1; i&lt;=n; i++){        // 外层n次
    for(int j=1; j&lt;=n; j+=i){   // 内层n/i次
        sum += a[i][j];         // 总次数：n*(1+1/2+...+1/n)
    }
}</pre>
    <ul>
     <li>
      <p>
       <strong>
        调和级数求和
       </strong>
       ：Σ(1/i) ≈ ln(n) + γ（欧拉常数≈0.577）
      </p>
     </li>
     <li>
      <p>
       <strong>
        时间复杂度
       </strong>
       ：O(nlogn)
      </p>
     </li>
    </ul>
    <h4>
     4.2 常见计算技巧
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        循环次数法
       </strong>
       ：
      </p>
      <pre>for(int i=0; i&lt;n; i*=2){...}  // 执行log2n次</pre>
     </li>
     <li>
      <p>
       <strong>
        摊还分析法
       </strong>
       ：
      </p>
      <pre>vector&lt;int&gt; vec;
for(int i=0; i&lt;n; i++){
    vec.push_back(i);  // 均摊时间复杂度O(1)
}</pre>
     </li>
     <li>
      <p>
       <strong>
        主定理公式
       </strong>
       （分治算法）：
      </p>
      <pre>T(n) = aT(n/b) + O(n^d)</pre>
     </li>
    </ol>
