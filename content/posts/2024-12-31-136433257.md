    <h2>
     <a id="JAVE_2">
     </a>
     JAVE
    </h2>
    <h3>
     <a id="1_4">
     </a>
     1、概述
    </h3>
    <p>
     JAVE (Java Audio Video Encoder)是一个纯Java的音视频编码器和解码器库，它是基于FFmpeg。JAVE库提供了一些简单易用的API，用于音频和视频格式的转换、编码、解码等操作。它对于一些基本的音视频处理任务来说是一个不错的选择。
    </p>
    <p>
     这些库都是基于FFmpeg的，并允许在Java中处理音频和视频文件。使用它们可以避免直接调用外部的FFmpeg命令行工具，而是通过Java API来实现音频格式转换等操作。
    </p>
    <h4>
     <a id="11_9">
     </a>
     1.1、操作系统支持
    </h4>
    <p>
     JAVE 需要Java 8 或更高版本。JAVE 还可以轻松移植到其他操作系统和硬件配置：
    </p>
    <ul>
     <li>
      Windows x32、x64
     </li>
     <li>
      MacOS Intel x64
     </li>
     <li>
      MacOS M1
     </li>
     <li>
      Linux x32、x64
     </li>
     <li>
      Linux ARM32、ARM64
     </li>
    </ul>

    <h3>
     <a id="2_85">
     </a>
     2、编码属性
    </h3>
    <h4>
     <a id="21AudioAttributes_87">
     </a>
     2.1、AudioAttributes
    </h4>
    <p>
     音频编码属性由类的实例表示
     <code>
      ws.schild.jave.AudioAttributes
     </code>
     。此类对象的可用方法有：
    </p>
    <ul>
     <li>
      <strong>
       public void setCodec(String codec)
      </strong>
      ：设置将用于音频流转码的编解码器的名称。必须从
      <code>
       getAudioEncoders()
      </code>
      返回的实例列表中选择一个值。否则可以传递
      <code>
       AudioAttributes.DIRECT_STREAM_COPY
      </code>
      特殊值，这需要从源文件中复制原始音频流。
     </li>
     <li>
      <strong>
       public void setBitRate(Integer bitRate)
      </strong>
      ：它设置新的重新编码的音频流的比特率值。如果未设置比特率值，编码器将选择默认比特率值。该值应以每秒位数表示。例如，如果您想要 128 kb/s 比特率，您应该调用
      <code>
       setBitRate(new Integer(128000))
      </code>
      。
     </li>
     <li>
      <strong>
       public void setSamplingRate(Integer bitRate)
      </strong>
      ：它设置新的重新编码的音频流的采样率。如果未设置采样率值，编码器将选择默认值。该值应以赫兹表示。例如，如果您想要类似 CD 的 44100 Hz 采样率，您应该调用
      <code>
       setSamplingRate(new Integer(44100))
      </code>
      。
     </li>
     <li>
      <strong>
       public void setChannels(Integer channels)
      </strong>
      ：它设置将在重新编码的音频流中使用的音频通道的数量（1 = 单声道，2 = 立体声）。如果未设置通道值，编码器将选择默认通道值。
     </li>
     <li>
      <strong>
       public void setVolume(Integer volume)
      </strong>
      ：可以调用此方法来改变音频流的音量。值 256 表示没有音量变化。因此，小于 256 的值会降低音量，而大于 256 的值会增加音频流的音量。
     </li>
    </ul>
    <h4>
     <a id="22VideoAttributes_98">
     </a>
     2.2、VideoAttributes
    </h4>
    <p>
     视频编码属性由类的实例表示
     <code>
      ws.schild.jave.VideoAttributes
     </code>
     。此类对象的可用方法有：
    </p>
    <ul>
     <li>
      <strong>
       public void setCodec(String codec)
      </strong>
      ：它设置将用于视频流转码的编解码器的名称。必须从
      <code>
       getVideoEncoders()
      </code>
      返回的Encoder 实例列表中选择一个值。否则，您可以传递
      <code>
       VideoAttributes.DIRECT_STREAM_COPY
      </code>
      特殊值，这需要从源文件复制原始视频流。
     </li>
     <li>
      <strong>
       public void setTag(String tag)
      </strong>
      ：它设置与重新编码的视频流关联的 tag/fourcc 值。如果未设置值，编码器将选择默认值。多媒体播放器经常使用标签值来选择在流上运行的视频解码器。例如，带有“DIVX”标签值的 MPEG 4 视频流将使用播放器使用的默认 DivX 解码器进行解码。顺便说一句，这正是 DivX 的本质：带有附加“DIVX”标签/fourcc 值的 MPEG 4 视频流。
     </li>
     <li>
      <strong>
       public void setBitRate(Integer bitRate)
      </strong>
      ：它设置新的重新编码视频流的比特率值。如果未设置比特率值，编码器将选择默认比特率值。该值应以每秒位数表示。例如，如果您想要 360 kb/s 的比特率，您应该调用
      <code>
       setBitRate(new Integer(360000))
      </code>
      。
     </li>
     <li>
      <strong>
       public void setFrameRate(Integer bitRate)
      </strong>
      ：它设置新的重新编码的音频流的帧速率值。如果未设置比特率帧率，编码器将选择默认值。该值应以每秒帧数表示。例如，如果您想要 30 f/s 的帧速率，您应该调用
      <code>
       setFrameRate(new Integer(30))
      </code>
      。
     </li>
     <li>
      <strong>
       pulic void setSize(ws.schild.jave.VideoSize size)
      </strong>
      ：它设置视频流中图像的大小和比例。如果未设置值，编码器将保留原始大小和比例。否则，您可以传递一个ws.schild.java.VideoSize具有您首选大小的实例。您可以设置新编码视频的宽度和高度以及像素值，缩放原始视频。例如，如果您想将视频缩放到宽度为 512 像素，高度为 384 像素，您应该调用
      <code>
       setSize(new VideoSize(512, 384))
      </code>
      。
     </li>
    </ul>

    <h3>
     <a id="4_416">
     </a>
     4、使用方式
    </h3>
    <h4>
     <a id="41_418">
     </a>
     4.1、简单使用
    </h4>
    <p>
     最重要的 JAVE 类是
     <code>
      ws.schild.jave.Encoder
     </code>
     . 编码器对象公开了许多多媒体转码方法。为了使用 JAVE，您始终必须创建一个编码器实例：
    </p>
    <pre><code class="prism language-java"><span class="token class-name">Encoder</span> encoder <span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">Encoder</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     创建实例后，您可以调用encode()方法开始转码：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">encode</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span></span>MultimediaObject</span> source<span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>io<span class="token punctuation">.</span></span>File</span> target<span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncodingAttributes</span> attributes<span class="token punctuation">)</span>
        <span class="token keyword">throws</span> <span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>IllegalArgumentException</span><span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>InputFormatException</span><span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncoderException</span>
</code></pre>
    <ul>
     <li>
      第一个参数source表示要解码的源文件
     </li>
     <li>
      二个参数 target 是将要创建和编码的目标文件
     </li>
     <li>
      属性参数的类型为
      <code>
       ws.schild.jave.EncodingAttributes
      </code>
      ，是包含编码器所需的任何信息的数据结构
     </li>
    </ul>
    <blockquote>
     <p>
      请注意，对 的调用encode()是阻塞调用：只有在转码操作完成（或失败）后该方法才会返回。如果您对监控转码操作感兴趣，请查看“监控转码操作”部分。编码属性
     </p>
    </blockquote>
    <p>
     要指定有关转码操作的首选项，您必须
     <code>
      ws.schild.jave.EncodingAttributes
     </code>
     为调用提供一个实例encode()。您可以创建自己的 EncodingAttributes 实例，并可以使用以下方法填充它：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setAudioAttributes</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>AudioAttributes</span> audioAttributes<span class="token punctuation">)</span>
</code></pre>
    <p>
     它设置音频编码属性。如果从未调用新的 EncodingAttributes 实例，或者给定参数为 null，则编码文件中不会包含任何音频流。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setVideoAttributes</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>AudioAttributes</span> videoAttributes<span class="token punctuation">)</span>
</code></pre>
    <p>
     它设置视频编码属性。如果从未调用新的 EncodingAttributes 实例，或者给定参数为 null，则编码文件中不会包含任何视频流。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setFormat</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>String</span> format<span class="token punctuation">)</span>
</code></pre>
    <p>
     它设置将用于新编码文件的流容器的格式。给定的参数表示格式名称。getSupportedEncodingFormats()仅当编码格式名称出现在由正在使用的 Encoder 实例的方法返回的列表中时，该名称才是有效且受支持的。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setOffset</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>Float</span> offset<span class="token punctuation">)</span>
</code></pre>
    <p>
     它设置转码操作的偏移量。源文件将从其开始后的 offset 秒处开始重新编码。例如，如果您想剪切源文件的前五秒，您应该调用setOffset(5)传递给编码器的 EncodingAttributes 对象。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">setDuration</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>Float</span> duration<span class="token punctuation">)</span>
</code></pre>
    <p>
     它设置转码操作的持续时间。只有源的持续时间秒才会在目标文件中重新编码。例如，如果您想从源中提取并转码 30 秒的一部分，则应调用传递setDuration(30)给编码器的 EncodingAttributes 对象。
    </p>
    <h4>
     <a id="42_472">
     </a>
     4.2、监控转码操作
    </h4>
    <p>
     可以使用侦听器监视转码操作。JAVE定义了
     <code>
      ws.schild.jave.EncoderProgressListener
     </code>
     接口。该接口可以由您的应用程序实现，并且EncoderProgressListener可以将具体实例传递给编码器。每次发生重大事件时，编码器都会调用您的侦听器方法。
    </p>
    <p>
     要将侦听器传递EncoderProgressListener给编码器，您应该使用该encode()方法的定义：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">encode</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span></span>MultimediaObject</span> source<span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>io<span class="token punctuation">.</span></span>File</span> target<span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncodingAttributes</span> attributes<span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncoderProgressListener</span> listener<span class="token punctuation">)</span>
        <span class="token keyword">throws</span> <span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>IllegalArgumentException</span><span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>InputFormatException</span><span class="token punctuation">,</span>
               <span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncoderException</span>
</code></pre>
    <p>
     要实现 EncoderProgressListener 接口，您必须定义以下所有方法：
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">sourceInfo</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>MultimediaInfo</span> info<span class="token punctuation">)</span>
</code></pre>
    <p>
     编码器在分析源文件后调用此方法。info 参数是该类的实例ws.schild.jave.MultimediaInfo，它表示有关源音频和视频流及其容器的信息。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">progress</span><span class="token punctuation">(</span><span class="token keyword">int</span> permil<span class="token punctuation">)</span>
</code></pre>
    <p>
     每次编码操作完成时，编码器都会调用此方法。permil 参数是表示当前操作到达的点的值，其范围是从 0（操作刚刚开始）到 1000（操作完成）。
    </p>
    <pre><code class="prism language-java"><span class="token keyword">public</span> <span class="token keyword">void</span> <span class="token function">message</span><span class="token punctuation">(</span><span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>String</span> message<span class="token punctuation">)</span>
</code></pre>
    <p>
     该方法由编码器调用以通知有关转码操作的消息（通常该消息是警告）。
    </p>
    <h4>
     <a id="43_502">
     </a>
     4.3、异常处理
    </h4>
    <p>
     当然，转码操作可能会失败。然后encode()方法将传播一个异常。根据发生的情况，异常将是以下之一：
    </p>
    <pre><code class="prism language-java"><span class="token class-name"><span class="token namespace">java<span class="token punctuation">.</span>lang<span class="token punctuation">.</span></span>IllegalArgumentException</span>
</code></pre>
    <p>
     由于传递给编码器的编码属性已被识别为无效，因此转码操作从未开始。通常，当提供给编码器的 EncodingAttributes 实例请求对没有音频和视频流的容器进行编码（AudioAttributes 和 VideoAttribues 属性均为 null 或未设置）时，通常会发生这种情况。
    </p>
    <pre><code class="prism language-java"><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>InputFormatException</span>
</code></pre>
    <p>
     源文件无法解码。当解码器不支持源文件容器、视频流格式或音频流格式时，就会发生这种情况。您可以检查支持的容器和插入的解码器调用编码器方法
     <code>
      getSupportedDecodingFormats()
     </code>
     ，
     <code>
      getAudioDecoders()
     </code>
     和
     <code>
      getVideoDecoders()
     </code>
     。
    </p>
    <pre><code class="prism language-java"><span class="token class-name"><span class="token namespace">ws<span class="token punctuation">.</span>schild<span class="token punctuation">.</span>jave<span class="token punctuation">.</span></span>EncoderExpection</span>
</code></pre>
    <p>
     由于内部错误，转码过程中操作失败。您应该检查异常消息，并且还可以使用 EncoderProgressListener 实例来检查编码器发出的任何消息。
    </p>
    <h4>
     <a id="44_519">
     </a>
     4.4、获取多媒体文件信息
    </h4>
    <p>
     可以在对现有多媒体文件进行转码之前调用编码器
     <code>
      getInfo()
     </code>
     方法来获取有关该文件的信息。该getInfo()方法为您提供有关文件使用的容器及其包装的音频和视频流的信息：
    </p>
    <pre><code class="prism language-java"><span class="token class-name">MultimediaObject</span> mmObject<span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">MultimediaObject</span><span class="token punctuation">(</span>myFile<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token class-name">MultimediaInfo</span> infos<span class="token operator">=</span> mmObject<span class="token punctuation">.</span><span class="token function">getInfo</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="prism language-java"><span class="token class-name">MultimediaObject</span> mmObject<span class="token operator">=</span> <span class="token keyword">new</span> <span class="token class-name">MultimediaObject</span><span class="token punctuation">(</span>myURL<span class="token punctuation">)</span><span class="token punctuation">;</span>
<span class="token class-name">MultimediaInfo</span> infos<span class="token operator">=</span> mmObject<span class="token punctuation">.</span><span class="token function">getInfo</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     对象
     <code>
      ws.schild.jave.MultimediaInfo
     </code>
     封装有关整个多媒体内容及其流的信息，使用
     <code>
      ws.schild.jave.AudioInfo
     </code>
     和的实例
     <code>
      ws.schild.jave.VideoInfo
     </code>
     来描述封装的音频和视频。这些对象与 EncodingAttributes、AudioAttributes 和 VideoAttributes 类似，但它们以只读模式工作。
    </p>
