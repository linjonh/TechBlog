    <h3 id="toc-hId--10996238">
     簡介
    </h3>
    <p>
     本文檔介紹了各種常用的思科防火牆術語。本文檔還介紹了如何從一個CLI模式切換到另一個CLI模式。
    </p>
    <h3 id="toc-hId--207509743">
     必要條件
    </h3>
    <h4 id="toc-hId--274940529">
     需求
    </h4>
    <p>
     之前沒有學習此主題的要求。
    </p>
    <h4 id="toc-hId--471454034">
     採用元件
    </h4>
    <p>
     本文中的資訊係根據以下軟體和硬體版本：
    </p>
    <ul>
     <li>
      思科安全防火牆管理中心(FMC)
     </li>
     <li>
      Cisco Firepower威脅防禦(FTD)
     </li>
     <li>
      Cisco Firepower裝置管理(FDM)
     </li>
     <li>
      Firepower eXtensible 作業系統 (FXOS)
     </li>
     <li>
      Firepower Chassis Manager (FCM)
     </li>
     <li>
      調適型安全裝置(ASA)
     </li>
    </ul>
    <p>
     本文中的資訊是根據特定實驗室環境內的裝置所建立。文中使用到的所有裝置皆從已清除（預設）的組態來啟動。如果您的網路運作中，請確保您瞭解任何指令可能造成的影響。
    </p>
    <h3 id="toc-hId--797050258">
    </h3>
    <h3 id="toc-hId--993563763">
     <strong>
      常用技術術語
     </strong>
    </h3>
    <h4 id="toc-hId--1060994549">
     <strong>
      FTD：Firepower威脅防禦
     </strong>
    </h4>
    <p>
     FTD是次世代防火牆，提供超越傳統防火牆的更多功能。它包括入侵防禦系統(IPS)、高級惡意軟體防護(AMP)、URL過濾、安全情報等服務。FTD與ASA （調適型安全裝置）非常類似，但具有附加功能。FTD在2個引擎上執行，LINA和SNORT。
    </p>
    <h4 id="toc-hId--1257508054">
     <strong>
      LINA：
     </strong>
     基於Linux的整合網路架構
    </h4>
    <p>
     我們將ASA稱為FTD裝置中的Lina。LINA只是FTD執行的ASA程式碼。Lina的主要重點是網路層安全。它確實透過其應用檢測和控制功能整合了一些第7層防火牆功能。
    </p>
    <h4 id="toc-hId--1454021559">
     <strong>
      SNORT
     </strong>
    </h4>
    <p>
     Snort引擎是網路入侵檢測和防禦系統。Snort的主要功能包括辨識異常的資料包檢測、基於規則的檢測、即時警報、日誌記錄和分析，以及與其他安全工具的整合。Snort能夠執行L7檢測（應用層流量），不僅基於資料包報頭，還基於資料包的內容。
    </p>
    <p>
     您可以靈活地編寫自己的自定義規則，以在應用層定義特定的模式或簽名，從而增強檢測功能。它透過評估資料包的負載來執行深度資料包檢測。您甚至可以在此處執行加密資料包的解密。
    </p>
    <h4 id="toc-hId--1725766433">
     <strong>
      FXOS： Firepower可擴展作業系統
     </strong>
    </h4>
    <p>
     它是FTD裝置執行的作業系統。根據FXOS用於配置功能、監控機箱狀態和訪問高級故障排除功能的平台。
    </p>
    <p>
     Firepower 4100/9300和Firepower 2100上的FXOS以及平台模式下的自適應安全裝置軟體允許配置更改，而在其他平台中，除特定功能外，FXOS是只讀的。
    </p>
    <h4 id="toc-hId--1922279938">
     <strong>
      FCM：Firepower機箱管理器
     </strong>
    </h4>
    <p>
     FCM是用於管理機箱的GUI。它僅適用於在平台模式下運行ASA的9300、4100、2100。
    </p>
    <p>
    </p>
    <p class="img-center">
     <a href="https://www.cisco.com/c/dam/en/us/td/i/templates/note.gif" rel="nofollow">
      <img alt="附註圖示" src="https://i-blog.csdnimg.cn/img_convert/fc048b3df8e52c294be483188cc01611.gif"/>
     </a>
    </p>
    <p>
     <strong>
      注意
     </strong>
     ：您可以拿筆記型電腦來打個比方。FXOS是在機箱（筆記型電腦）上運行的作業系統（筆記型電腦中的Windows作業系統）。我們可以在其上安裝FTD（應用程式執行個體），其在Lina和Snort（元件）上執行。
    </p>
    <p>
     與ASA不同，您無法通過CLI管理FTD。您需要單獨的基於GUI的管理。此類服務有兩種型別：FDM和FMC。
    </p>
    <h4 id="toc-hId--2118793443">
     <strong>
      FDM：Firepower裝置管理
     </strong>
    </h4>
    <ul>
     <li>
      FDM是機上管理工具。它提供了一個基於Web的介面，用於配置、管理和監控安全策略和系統設定。
     </li>
     <li>
      使用FDM的一大優點是，您不需要額外的授權。
     </li>
     <li>
      您只能使用1個FDM管理1個FTD。
     </li>
    </ul>
    <p>
    </p>

     <strong>
      FMC：Firepower管理中心
     </strong>
    </h4>
    <ul>
     <li>
      FMC是思科FTD裝置（具備Firepower服務的思科ASA裝置）的集中管理解決方案。此功能也提供可用於設定、管理及監控FTD裝置的GUI。
     </li>
     <li>
      您可以使用硬體FMC裝置或虛擬FMC裝置。
     </li>
     <li>
      這需要單獨的許可證才能運行。
     </li>
     <li>
      FMC的一個優點是您可以使用1個FMC裝置來管理多個FTD裝置。
     </li>
    </ul>
    <p>
    </p>

     <strong>
      附註
     </strong>

    </p>
    <h4 id="toc-hId--98793958">
     <strong>
      CLISH：指令行介面Shell
     </strong>
    </h4>
    <p>
     CLISH是Cisco Firepower威脅防禦(FTD)裝置中使用的命令列介面。您可以使用此CLISH模式在FTD上執行指令。
    </p>
    <h4 id="toc-hId--295307463">
     <strong>
      診斷管理
     </strong>
    </h4>
    <p>
     FTD裝置中有兩個管理介面、診斷管理介面和FTD管理介面。如果必須存取LINA引擎，則會使用診斷管理介面。如果必須存取SNORT引擎，則使用FTD管理介面。兩者都是不同的介面，需要不同的介面IP地址。
    </p>

    <h4 id="toc-hId-798784125">
     <strong>
      ASA平台模式
     </strong>
    </h4>
    <ol>
     <li>
      在Platform模式下，您必須在FXOS中配置基本操作引數和硬體介面設定，例如啟用介面、建立EtherChannel、NTP、映像管理等。
     </li>
     <li>
      所有其他配置必須透過ASA CLI/ASDM完成。
     </li>
     <li>
      您擁有FCM存取權。
     </li>
    </ol>
    <h4 id="toc-hId-602270620">
     <strong>
      ASA裝置模式
     </strong>
    </h4>
    <ol>
     <li>
      在Firepower 2100中，從第9.13（包括）版開始引入裝置模式下的ASA。
     </li>
     <li>
      裝置模式允許您配置ASA中的所有設定。FXOS CLI僅提供高級故障排除命令。
     </li>
     <li>
      此模式中沒有FCM。
     </li>
    </ol>
