    <h2>
     <a id="_2">
     </a>
     获取源码
    </h2>
    <pre><code class="prism language-shell"><span class="token function">git</span> clone https://github.com/open-webui/open-webui <span class="token operator">&amp;&amp;</span> <span class="token builtin class-name">cd</span> open-webui
</code></pre>
    <h2>
     <a id="_6">
     </a>
     启动后端服务
    </h2>
    <pre><code class="prism language-shell"><span class="token builtin class-name">cd</span> backend
conda create <span class="token parameter variable">--name</span> open-webui <span class="token assign-left variable">python</span><span class="token operator">=</span><span class="token number">3.11</span>
conda activate open-webui
pip <span class="token function">install</span> <span class="token parameter variable">-r</span> requirements.txt <span class="token parameter variable">-U</span>
<span class="token function">sh</span> dev.sh
</code></pre>
    <p>
     没有
     <code>
      conda
     </code>
     的要先安装下，启动成功会后监听
     <code>
      8080
     </code>
     端口。
    </p>
    <h2>
     <a id="_15">
     </a>
     启动前端服务
    </h2>
    <p>
     回到代码根目录
    </p>
    <pre><code class="prism language-shell"><span class="token function">npm</span> i
<span class="token function">npm</span> run dev
</code></pre>
    <p>
     启动成功后会监听
     <code>
      5173
     </code>
     端口，在浏览器就可以访问
     <code>
      http://127.0.0.1:5173
     </code>
     。
    </p>
    <h2>
     <a id="_23">
     </a>
     常见问题解决
    </h2>
    <h4>
     <a id="1logo_24">
     </a>
     1.在访问前端服务时显示logo后会显示空白页面
    </h4>
    <blockquote>
     <p>
      这个问题是因为后端在获取模型列表，在获取模型列表时会访问openai，在国内无法访问，可以通过修改后端代码屏蔽掉这个逻辑，代码如下：
     </p>
    </blockquote>
    <pre><code class="prism language-python">代码路径 backend<span class="token operator">/</span>open_webui<span class="token operator">/</span>utils<span class="token operator">/</span>models<span class="token punctuation">.</span>py
</code></pre>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1ee493dddfab475fae3e2db3ee95b0eb.png">
      <br/>
      注释掉以上代码后可进入webui界面。
     </img>
    </p>
    <h4>
     <a id="2_32">
     </a>
     2.在相关界面有可能会遇到跨域问题，导致无法访问后端工程服务
    </h4>
    <blockquote>
     <p>
      解决问题思路是避免跨域，可以使用nginx代理合并端口，把
      <code>
       8080
      </code>
      和
      <code>
       5173
      </code>
      合并到
      <code>
       8888
      </code>
      进行访问，这样就规避了跨域问题。nginx配置参考如下：
     </p>
    </blockquote>
    <pre><code>    server {
        listen       8888;
        server_name  localhost;

        location / {
             proxy_pass http://127.0.0.1:5173/;
        }

        location /api/ {
          proxy_pass http://127.0.0.1:8080/api/;
        }

        location /ollama/ {
          proxy_pass http://127.0.0.1:8080/ollama/;
        }

        location /openai/ {
          proxy_pass http://127.0.0.1:8080/openai/;
        }
    }
</code></pre>
    <h4>
     <a id="ollama_57">
     </a>
     如何修改ollama监听地址和端口
    </h4>
    <pre><code class="prism language-shell"><span class="token function">vi</span> /etc/profile
添加如下一行
<span class="token builtin class-name">export</span> <span class="token assign-left variable">OLLAMA_HOST</span><span class="token operator">=</span>http://0.0.0.0:11434
<span class="token builtin class-name">source</span> /etc/profile
然后重启ollama服务
</code></pre>
