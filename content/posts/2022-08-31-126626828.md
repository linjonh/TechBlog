    <h1 class="title-article" id="articleContentId">
     Windows下Jenkins的运行环境由Java8 升级为Java11
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p>
     开源 Devops 工具 Jenkins 在官方博客宣布，从 6 月 28 日发布的 Jenkins 2.357 和将于 9 月发布的 LTS 版本开始，Jenkins 需要 Java 11 才能使用，将放弃 Java 8。
    </p>
    <h4 id="%E6%AD%A5%E9%AA%A4%EF%BC%9A">
     <strong>
      步骤：
     </strong>
    </h4>
    <p>
     1、安装java11
    </p>
    <p>
     1）官网下载安装包：
     <a href="https://www.oracle.com/java/technologies/downloads/archive/" rel="nofollow" title="Java Archive | Oracle">
      Java Archive | Oracle
     </a>
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/d57aa0d1a5547ef7c072decac0a020f1.png"/>
    </p>
    <p>
     下载exe安装包：
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/41695b57fbe87a2094fedbf16c636894.png"/>
    </p>
    <p>
     2）双击下载好的exe安装包进行安装
    </p>
    <p>
     3）运行CMD，输入命令java -version，回车，如出现以下内容则说明安装成功,否则需要自己手动配置环境变量：
    </p>
    <p class="img-center">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/d65ec0b34ef76d75f60599bb5933d244.png"/>
    </p>
    <p>
     4）停止jenkins服务，以管理员身份运行CMD，输入net stop jenkins，回车；
    </p>
    <p>
     5）查看jenkins_home下的jenkins.xml文件，将excutable里面的值改为java11安装的路径，保存关闭。例如博主的是C:\Program Files\Java\jdk-11.0.15\bin\java.exe：
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/8e83578341f9935b6fc110ba0d032dcf.png"/>
    </p>
    <p>
     6）重新启动jenkins服务，以管理员身份运行CMD，输入net start jenkins，回车，jenkins服务启动完成；
    </p>
    <p>
     7）查看jenkins的配置
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/e166d22668d0e4e94b86555b4092ffc5.png"/>
    </p>
    <p style="text-align:center;">
     <img alt="" src="https://i-blog.csdnimg.cn/blog_migrate/9437166cf78dd1d1f92bc02613639013.png"/>
    </p>
    <p>
     java.home显示刚才配置的java11路径 。
    </p>
    <h4>
     <strong>
      jenkins系统属性含义
     </strong>
    </h4>
    <ul>
     <li>
      java.home Java        java安装目录
     </li>
     <li>
      java.io.tmpdir         默认的临时文件路径
     </li>
     <li>
      java.library.path        加载库时搜索的路径列表
     </li>
     <li>
      java.runtime.name        java的运行环境名称
     </li>
     <li>
      java.runtime.version        java运行时的版本
     </li>
     <li>
      java.specification.name        Java 虚拟机规范名称
     </li>
     <li>
      java.specification.vendor        虚拟机规范供应商
     </li>
     <li>
      java.specification.version        虚拟机规范版本
     </li>
     <li>
      java.vendor        Java 运行时环境供应商
     </li>
     <li>
      java.vendor.url        供应商的URL
     </li>
     <li>
      java.vendor.url.bug        java发行者的url
     </li>
     <li>
      java.vendor.version        发行版本
     </li>
     <li>
      java.version Java        运行时环境版本
     </li>
     <li>
      java.version.date        发行日期
     </li>
    </ul>
