    <h4>
     <a id="1Queue__15">
     </a>
     1.Queue 的介绍
    </h4>
    <p>
     <code>
      Queue
     </code>
     ，中文名叫
     <code>
      队列
     </code>
     ，无论现实生活中还是计算机的世界中，都是一个很重要的角色哦~
    </p>
    <p>
     Queue 是一种
     <code>
      数据结构
     </code>
     ，大家可以把我想象成一个数组，元素从我的一头进入、从另外一头出去，称为 FIFO 原则（先进先出原则）。
    </p>
    <p>
     Queue 有两个亲兄弟：
     <code>
      List
     </code>
     （列表）、
     <code>
      Set
     </code>
     （集），他们都是
     <code>
      Collection
     </code>
     的儿子，Queue 还有一个远房亲戚：
     <code>
      Map
     </code>
     （映射）。他们都是
     <code>
      java.util
     </code>
     包这个大家庭的成员哦~
    </p>

    <h4>
     <a id="3Queue__47">
     </a>
     3.Queue 的核心方法
    </h4>
    <ul>
     <li>
      add
     </li>
     <li>
      element
     </li>
     <li>
      offer
     </li>
     <li>
      peek
     </li>
     <li>
      poll
     </li>
     <li>
      remove
     </li>
    </ul>
    <p>
     <code>
      Queue的核心方法:
     </code>
    </p>
    <table>
     <thead>
      <tr>
       <th align="left">
        动作
       </th>
       <th align="left">
        抛异常
       </th>
       <th align="left">
        返回特殊值
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td align="left">
        Insert
       </td>
       <td align="left">
        add(e)
       </td>
       <td align="left">
        offer(e)
       </td>
      </tr>
      <tr>
       <td align="left">
        Remove
       </td>
       <td align="left">
        remove()
       </td>
       <td align="left">
        poll
       </td>
      </tr>
      <tr>
       <td align="left">
        Examine
       </td>
       <td align="left">
        element()
       </td>
       <td align="left">
        peek()
       </td>
      </tr>
     </tbody>
    </table>
    <ul>
     <li>
      比如
      <code>
       添加（Insert）
      </code>
      元素的动作，会有两种方式：
      <code>
       add(e)
      </code>
      和
      <code>
       offer(e)
      </code>
      。如果调用 add(e)方法时，添加失败，则会
      <code>
       抛异常
      </code>
      ，而如果调用的是 offer(e)方法失败时，则会
      <code>
       返回false
      </code>
      。offer 方法用于异常是正常的情况下使用，比如在有界队列中，优先使用 offer 方法。假如队列满了，不能添加元素，offer 方法返回 false，这样我们就知道是队列满了，而不是去 handle 运行时抛出的异常。
     </li>
     <li>
      同理，移除（Remove）元素的动作，队列为空时，remove 方法抛异常，而 poll 返回 null。如果移除头部的元素成功，则返回移除的元素。
     </li>
     <li>
      同理，检测（Examine）元素的动作，返回头部元素（最开始加入的元素），但不删除元素， 如果队列为空，则 element()方法抛异常，而 peek()返回 false。
     </li>
     <li>
      Queue 接口没有定义阻塞队列的方法，这些方法在 BlockQueue 接口中定义了。
     </li>
     <li>
      Queue 实现类通常不允许插入 null 元素，尽管一些实现类比如 LinkedList 不禁止插入 null，但是还是不建议插入 null，因为 null 也被用在 poll 方法的特殊返回值，以说明队列不包含元素。
     </li>
    </ul>
