    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     当Adobe、Microsoft、Sun等一系列巨头开始表现出对”开源”的青睐时，”开源”的时代即将到来！
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     最初来自：sinoprise.com/read.php?tid-662-page-e-fpage-1.html（遗憾的是这个链接已经打不开了），我基本未改动，只是进行了一些排版和整理。
     <br style="padding:0px; margin:0px"/>
     参考文献：
     <a href="http://www.fsf.org/licensing/licenses/" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      http://www.fsf.org/licensing/licenses/
     </a>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     现今存在的开源协议很多，而经过Open Source Initiative组织通过批准的开源协议目前有58种（
     <a href="http://www.opensource.org/licenses/alphabetical" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      http://www.opensource.org/licenses/alphabetical
     </a>
     ）。我们在常见的开源协议如BSD, GPL, LGPL,MIT等都是OSI批准的协议。如果要开源自己的代码，最好也是选择这些被批准的开源协议。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     这里我们来看四种最常用的开源协议及它们的适用范围，供那些准备开源或者使用开源产品的开发人员/厂家参考。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <span style="padding:0px; margin:0px">
      BSD开源协议（
     </span>
     <a href="http://www.fsf.org/licensing/licenses/index_html#OriginalBSD" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      original BSD license
     </a>
     <span style="padding:0px; margin:0px">
      、
     </span>
     <a href="http://www.freebsd.org/copyright/freebsd-license.html" id="FreeBSD" name="FreeBSD" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      FreeBSD license
     </a>
     <span style="padding:0px; margin:0px">
      、
     </span>
     <a href="http://www.xfree86.org/3.3.6/COPYRIGHT2.html#6" id="OriginalBSD" name="OriginalBSD" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      Original BSD license
     </a>
     <span style="padding:0px; margin:0px">
      ）
     </span>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     BSD开源协议是一个给于使用者很大自由的协议。基本上使用者可以”为所欲为”,可以自由的使用，修改源代码，也可以将修改后的代码作为开源或者专有软件再发布。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     但”为所欲为”的前提当你发布使用了BSD协议的代码，或则以BSD协议代码为基础做二次开发自己的产品时，需要满足三个条件：
    </p>
    <ol style="padding:0px; margin:1.2em 0px 1.2em 2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <li style="padding:0px; margin:0px 0px 0.2em">
      如果再发布的产品中包含源代码，则在源代码中必须带有原来代码中的BSD协议。
     </li>
     <li style="padding:0px; margin:0px 0px 0.2em">
      如果再发布的只是二进制类库/软件，则需要在类库/软件的文档和版权声明中包含原来代码中的BSD协议。
     </li>
     <li style="padding:0px; margin:0px 0px 0.2em">
      不可以用开源代码的作者/机构名字和原来产品的名字做市场推广。
     </li>
    </ol>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     BSD 代码鼓励代码共享，但需要尊重代码作者的著作权。BSD由于允许使用者修改和重新发布代码，也允许使用或在BSD代码上开发商业软件发布和销售，因此是对商业集成很友好的协议。而很多的公司企业在选用开源产品的时候都首选BSD协议，因为可以完全控制这些第三方的代码，在必要的时候可以修改或者二次开发。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <span style="padding:0px; margin:0px">
      Apache Licence 2.0（
     </span>
     <a href="http://www.apache.org/licenses/LICENSE-2.0" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      Apache License, Version 2.0
     </a>
     、
     <a href="http://www.apache.org/LICENSE-1.1" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      Apache License, Version 1.1
     </a>
     、
     <a href="http://www.apache.org/LICENSE-1.0" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      Apache License, Version 1.0
     </a>
     <span style="padding:0px; margin:0px">
      ）
     </span>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     Apache Licence是著名的非盈利开源组织Apache采用的协议。该协议和BSD类似，同样鼓励代码共享和尊重原作者的著作权，同样允许代码修改，再发布（作为开源或商业软件）。需要满足的条件也和BSD类似：
    </p>
    <ol style="padding:0px; margin:1.2em 0px 1.2em 2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <li style="padding:0px; margin:0px 0px 0.2em">
      需要给代码的用户一份Apache Licence
     </li>
     <li style="padding:0px; margin:0px 0px 0.2em">
      如果你修改了代码，需要再被修改的文件中说明。
     </li>
     <li style="padding:0px; margin:0px 0px 0.2em">
      在延伸的代码中（修改和有源代码衍生的代码中）需要带有原来代码中的协议，商标，专利声明和其他原来作者规定需要包含的说明。
     </li>
     <li style="padding:0px; margin:0px 0px 0.2em">
      如果再发布的产品中包含一个Notice文件，则在Notice文件中需要带有Apache Licence。你可以在Notice中增加自己的许可，但不可以表现为对Apache Licence构成更改。
     </li>
    </ol>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     Apache Licence也是对商业应用友好的许可。使用者也可以在需要的时候修改代码来满足需要并作为开源或商业产品发布/销售。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <span style="padding:0px; margin:0px">
      GPL（
     </span>
     <a href="http://www.fsf.org/licensing/licenses/gpl.html" id="GNUGPL" name="GNUGPL" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      GNU General Public License
     </a>
     <span style="padding:0px; margin:0px">
      ）
     </span>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     我们很熟悉的Linux就是采用了GPL。GPL协议和BSD, Apache Licence等鼓励代码重用的许可很不一样。GPL的出发点是代码的开源/免费使用和引用/修改/衍生代码的开源/免费使用，但不允许修改后和衍生的代码做为闭源的商业软件发布和销售。这也就是为什么我们能用免费的各种linux，包括商业公司的linux和linux上各种各样的由个人，组织，以及商业软件公司开发的免费软件了。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     GPL协议的主要内容是只要在一个软件中使用(“使用”指类库引用，修改后的代码或者衍生代码)GPL 协议的产品，则该软件产品必须也采用GPL协议，既必须也是开源和免费。
     <span style="padding:0px; margin:0px">
      这就是所谓的”传染性”
     </span>
     。GPL协议的产品作为一个单独的产品使用没有任何问题，还可以享受免费的优势。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     由于GPL严格要求使用了GPL类库的软件产品必须使用GPL协议，对于使用GPL协议的开源代码，商业软件或者对代码有保密要求的部门就不适合集成/采用作为类库和二次开发的基础。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     其它细节如再发布的时候需要伴随GPL协议等和BSD/Apache等类似。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <span style="padding:0px; margin:0px">
      LGPL（
     </span>
     <a href="http://www.fsf.org/licensing/licenses/lgpl.html" id="LGPL" name="LGPL" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
      GNU Lesser General Public License
     </a>
     <span style="padding:0px; margin:0px">
      ）
     </span>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     LGPL是GPL的一个为主要为类库使用设计的开源协议。和GPL要求任何使用/修改/衍生之GPL类库的的软件必须采用GPL协议不同。LGPL允许商业软件通过类库引用(link)方式使用LGPL类库而不需要开源商业软件的代码。这使得采用LGPL协议的开源代码可以被商业软件作为类库引用并发布和销售。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     但是如果修改LGPL协议的代码或者衍生，则所有修改的代码，涉及修改部分的额外代码和衍生的代码都必须采用LGPL协议。因此LGPL协议的开源代码很适合作为第三方类库被商业软件引用，但不适合希望以LGPL协议代码为基础，通过修改和衍生的方式做二次开发的商业软件采用。
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     GPL/LGPL都保障原作者的知识产权，避免有人利用开源代码复制并开发类似的产品
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     <span style="padding:0px; margin:0px">
      MIT（
      <a href="http://www.opensource.org/licenses/mit-license.php" rel="nofollow" style="padding:0px; margin:0px; color:rgb(71,119,37); text-decoration:none; border-bottom-width:1px; border-bottom-style:dashed; border-bottom-color:rgb(0,102,0)">
       MIT
      </a>
      ）
     </span>
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     MIT是和BSD一样宽范的许可协议,作者只想保留版权,而无任何其他了限制.也就是说,你必须在你的发行版里包含原许可协议的声明,无论你是以二进制发布的还是以源代码发布的.
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
     ————————————————————————————————————————————————————————————————————————————————————————————
    </p>
    <p style="padding-top:0px; padding-bottom:0px; margin-top:1.2em; margin-bottom:1.2em; line-height:1.5em; color:rgb(0,34,0); font-family:georgia">
    </p>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <span style="line-height:normal; border-collapse:separate; font-size:14px">
      <strong>
       关于开源许可
      </strong>
     </span>
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     现今存在的开源协议很多，而经过
     <span style="text-decoration:underline">
      Open Source Initiative
     </span>
     组织通过批准的开源协议目前有60多种（
     <a href="http://www.opensource.org/licenses/alphabetical" rel="nofollow" style="color:#108ac6">
      http://www.opensource.org/licenses/alphabetical
     </a>
     ）。我们在常见的开源协议如BSD, GPL, LGPL,MIT等都是OSI批准的协议。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <strong>
      <br/>
     </strong>
    </div>
    <p>
    </p>
    <p style="padding-bottom:0px; line-height:25px; margin-top:0px; margin-bottom:0px; font-family:Helvetica,Tahoma,Arial,sans-serif; padding-top:0px">
     <strong>
      基本概念
     </strong>
    </p>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <span style="color:#2d4fc9">
      1.Contributors 和 Recipients
     </span>
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Contributors（贡献者）
     </em>
     ——指的是对某个开源软件或项目提供了代码（包括最初的或者修改过）的人或实体（退队、公司、组织等）。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     按照贡献的先后可分为"创始人"（an initial Contributor）和"参与者"（subsequent Contributors）。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Recipients（获取者）
     </em>
     ——指的是开源软件或项目的使用者。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     显然，subsequent Contributors也属于Recipients之列。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <span style="color:#2d4fc9">
      2.Source Code 和 Object Code
     </span>
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Source Code
     </em>
     ——指的是由各种语言写成的
     <span style="text-decoration:underline">
      源代码
     </span>
     。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Object Code
     </em>
     ——指的是Source Code经过编译后，生成的类似“类库”一样的，提供了各种接口供他人使用的
     <span style="text-decoration:underline">
      目标代码
     </span>
     （就如,DLL、JAR等）。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <span style="color:#2d4fc9">
      3.Derivative Module 和 Separate Module
     </span>
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Derivative Module（衍生模块）
     </em>
     ——指的是，依托或包含“最初的”或者“从别人处获取的”开源代码而产生的代码，是对“源代码模块”的增强、改善和延续。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <em>
      Separate Module（独立模块）
     </em>
     ——指的是，参考或借助“源代码”开发出来的独立的，不包含、不依赖于原“源代码模块”的功能模块。
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
    </div>
    <div style="line-height:25px; font-family:Helvetica,Tahoma,Arial,sans-serif">
     <strong>
      在OSI网站上
      <a href="http://www.opensource.org/licenses/category" rel="nofollow" style="color:#108ac6">
       被列为主流及被广泛使用的许可
      </a>
      有：
     </strong>
    </div>
