    <h4 id="1%E3%80%81%3C%3E%3C%2F%3E%E6%98%AF%E4%BB%80%E4%B9%88" name="1%E3%80%81%3C%3E%3C%2F%3E%E6%98%AF%E4%BB%80%E4%B9%88">
     1、&lt;&gt;&lt;/&gt;是什么
    </h4>
    <p>
     在React中，
     <code>
      &lt;&gt;&lt;/&gt;
     </code>
     （称为Fragment）是一种用于包裹多个子元素而不添加额外DOM节点的语法
    </p>
    <h4 id="2%E3%80%81%E4%B8%BA%E4%BB%80%E4%B9%88%E8%A6%81%E4%BD%BF%E7%94%A8%3C%3E%3C%2F%3E%EF%BC%9F" name="2%E3%80%81%E4%B8%BA%E4%BB%80%E4%B9%88%E8%A6%81%E4%BD%BF%E7%94%A8%3C%3E%3C%2F%3E%EF%BC%9F">
     <strong>
      2、为什么要使用
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      ？
     </strong>
    </h4>
    <p>
     <strong>
      避免冗余DOM节点
     </strong>
     ：
     <br/>
     React要求组件返回的JSX必须有一个根元素。若用
     <code>
      &lt;div&gt;
     </code>
     包裹多个元素，会增加不必要的DOM层级，可能破坏布局（如表格结构、Flex/Grid布局）。Fragment解决了这一问题，不会生成实际节点。
    </p>
    <p>
     <strong>
      保持结构简洁
     </strong>
     ：
     <br/>
     当不需要包裹元素的属性（如
     <code>
      className
     </code>
     或事件）时，Fragment让代码更简洁，同时满足JSX语法要求。
    </p>
    <p>
     <strong>
      性能优化
     </strong>
     ：
     <br/>
     减少不必要的DOM节点，对复杂组件或大型应用有一定优化作用。
    </p>
    <h4 id="3%E3%80%81%E5%A6%82%E4%BD%95%E4%BD%BF%E7%94%A8%3C%3E%3C%2F%3E%EF%BC%9F" name="3%E3%80%81%E5%A6%82%E4%BD%95%E4%BD%BF%E7%94%A8%3C%3E%3C%2F%3E%EF%BC%9F">
     <strong>
      3、如何使用
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      ？
     </strong>
    </h4>
    <h5 id="%E5%9F%BA%E6%9C%AC%E7%94%A8%E6%B3%95" name="%E5%9F%BA%E6%9C%AC%E7%94%A8%E6%B3%95">
     基本用法
    </h5>
    <p>
     直接包裹多个同级元素：
    </p>
    <pre><code class="language-javascript">function App() {
  return (
    &lt;&gt;
      &lt;Header /&gt;
      &lt;Content /&gt;
      &lt;Footer /&gt;
    &lt;/&gt;
  );
}</code></pre>
    <h5 id="%E9%9C%80%E8%A6%81%E4%BC%A0%E9%80%92%E5%B1%9E%E6%80%A7%E6%97%B6%EF%BC%88%E5%A6%82key%EF%BC%89" name="%E9%9C%80%E8%A6%81%E4%BC%A0%E9%80%92%E5%B1%9E%E6%80%A7%E6%97%B6%EF%BC%88%E5%A6%82key%EF%BC%89">
     需要传递属性时（如
     <code>
      key
     </code>
     ）
    </h5>
    <p>
     使用显式
     <code>
      &lt;React.Fragment&gt;
     </code>
     语法：
    </p>
    <pre><code class="language-javascript">function List({ items }) {
  return items.map(item =&gt; (
    &lt;React.Fragment key={item.id}&gt;
      &lt;li&gt;{item.name}&lt;/li&gt;
      &lt;li&gt;{item.description}&lt;/li&gt;
    &lt;/React.Fragment&gt;
  ));
}</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        注意
       </strong>
       ：空标签
       <code>
        &lt;&gt;&lt;/&gt;
       </code>
       不支持任何属性（如
       <code>
        key
       </code>
       ），此时必须用
       <code>
        &lt;React.Fragment&gt;
       </code>
       。
      </p>
     </li>
    </ul>
    <h5 id="%E4%BD%BF%E7%94%A8%E6%95%88%E6%9E%9C" name="%E4%BD%BF%E7%94%A8%E6%95%88%E6%9E%9C">
     <strong>
      使用效果
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        渲染结果
       </strong>
       ：
       <br/>
       Fragment的子元素会直接挂载到父节点，
       <strong>
        不会产生额外DOM层级
       </strong>
       。例如：
      </p>
      <pre><code class="language-javascript">&lt;&gt;
  &lt;div&gt;A&lt;/div&gt;
  &lt;div&gt;B&lt;/div&gt;
&lt;/&gt;</code></pre>
      <p>
       渲染结果为：
      </p>
      <pre><code class="language-html">&lt;div&gt;A&lt;/div&gt;
&lt;div&gt;B&lt;/div&gt;</code></pre>
     </li>
     <li>
      <p>
       <strong>
        与
        <code>
         &lt;div&gt;
        </code>
        的对比
       </strong>
       ：
       <br/>
       使用
       <code>
        &lt;div&gt;
       </code>
       包裹会多出一个节点，可能影响布局或样式：
      </p>
      <pre><code class="language-html">&lt;div&gt;
  &lt;div&gt;A&lt;/div&gt;
  &lt;div&gt;B&lt;/div&gt;
&lt;/div&gt;</code></pre>
     </li>
    </ul>
    <h5 id="%E6%B3%A8%E6%84%8F%E4%BA%8B%E9%A1%B9" name="%E6%B3%A8%E6%84%8F%E4%BA%8B%E9%A1%B9">
     <strong>
      注意事项
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        版本兼容性
       </strong>
       ：
       <br/>
       Fragment在React 16.2+支持，确保项目React版本足够新，且Babel配置正确。
      </p>
     </li>
     <li>
      <p>
       <strong>
        工具支持
       </strong>
       ：
       <br/>
       部分IDE或工具可能对空标签的语法高亮或格式化支持不完善，但通常不影响功能。
      </p>
     </li>
     <li>
      <p>
       <strong>
        条件渲染
       </strong>
       ：
       <br/>
       Fragment可以包裹条件渲染的内容：
      </p>
      <pre><code class="language-javascript">&lt;&gt;
  {isLoading &amp;&amp; &lt;Spinner /&gt;}
  &lt;Content /&gt;
&lt;/&gt;</code></pre>
     </li>
    </ol>
    <blockquote>
     <h5 id="%E6%80%BB%E7%BB%93" name="%E6%80%BB%E7%BB%93">
      <strong>
       总结
      </strong>
     </h5>
     <p>
      <strong>
       使用场景
      </strong>
      ：
      <br/>
      需返回多个元素但不想添加冗余DOM节点时（如表格行、列表项、布局组件）。
     </p>
     <p>
      <strong>
       语法选择
      </strong>
      ：
     </p>
     <p>
      无属性需求 → 用
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      （简洁）。
     </p>
     <p>
      需要传递
      <code>
       key
      </code>
      或其他属性 → 用
      <code>
       &lt;React.Fragment&gt;
      </code>
      。
     </p>
     <p>
      <strong>
       优势
      </strong>
      ：
      <br/>
      代码简洁、DOM结构干净、避免布局问题。
     </p>
    </blockquote>
    <p>
    </p>
    <h4 id="4%E3%80%81React%20Fragment%20%E4%B8%8E%E7%A9%BA%E6%A0%87%E7%AD%BE%EF%BC%88%3C%3E%3C%2F%3E%EF%BC%89%E8%AF%A6%E8%A7%A3" name="4%E3%80%81React%20Fragment%20%E4%B8%8E%E7%A9%BA%E6%A0%87%E7%AD%BE%EF%BC%88%3C%3E%3C%2F%3E%EF%BC%89%E8%AF%A6%E8%A7%A3">
     <strong>
      4、React Fragment 与空标签（
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      ）详解
     </strong>
    </h4>
    <p>
     在 React 中，
     <strong>
      Fragment
     </strong>
     和
     <strong>
      空标签（
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      ）
     </strong>
     都是用于包裹多个子元素而不引入额外 DOM 节点的语法结构。它们的主要目的是解决 JSX 必须返回单个根元素的限制。
    </p>
    <h5 id="1.%20Fragment%20%E7%9A%84%E7%94%A8%E6%B3%95" name="1.%20Fragment%20%E7%9A%84%E7%94%A8%E6%B3%95">
     <strong>
      1. Fragment 的用法
     </strong>
    </h5>
    <h6 id="%E5%9F%BA%E6%9C%AC%E5%AE%9A%E4%B9%89" name="%E5%9F%BA%E6%9C%AC%E5%AE%9A%E4%B9%89">
     <strong>
      基本定义
     </strong>
    </h6>
    <blockquote>
     <p>
      <strong>
       Fragment
      </strong>
      是一个虚拟容器，允许将多个子元素分组而不影响 DOM 结构。
     </p>
     <p>
      <strong>
       语法
      </strong>
      ：使用
      <code>
       &lt;React.Fragment&gt;
      </code>
      或简写的空标签
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      。
     </p>
    </blockquote>
    <h6 id="%E6%98%BE%E5%BC%8F%20Fragment%EF%BC%88%3CReact.Fragment%3E%EF%BC%89" name="%E6%98%BE%E5%BC%8F%20Fragment%EF%BC%88%3CReact.Fragment%3E%EF%BC%89">
     <strong>
      显式 Fragment（
      <code>
       &lt;React.Fragment&gt;
      </code>
      ）
     </strong>
    </h6>
    <p>
     <strong>
      支持属性
     </strong>
     ：可以传递
     <code>
      key
     </code>
     或其他属性。
    </p>
    <p>
     <strong>
      适用场景
     </strong>
     ：需要在循环中包裹元素或添加
     <code>
      key
     </code>
     时
    </p>
    <pre><code class="language-javascript">import React from 'react';

function List({ items }) {
  return items.map((item) =&gt; (
    &lt;React.Fragment key={item.id}&gt;
      &lt;li&gt;{item.name}&lt;/li&gt;
    &lt;/React.Fragment&gt;
  ));
}</code></pre>
    <h6 id="%E9%9A%90%E5%BC%8F%20Fragment%EF%BC%88%E7%A9%BA%E6%A0%87%E7%AD%BE%C2%A0%3C%3E%3C%2F%3E%EF%BC%89" name="%E9%9A%90%E5%BC%8F%20Fragment%EF%BC%88%E7%A9%BA%E6%A0%87%E7%AD%BE%C2%A0%3C%3E%3C%2F%3E%EF%BC%89">
     <strong>
      隐式 Fragment（空标签
      <code>
       &lt;&gt;&lt;/&gt;
      </code>
      ）
     </strong>
    </h6>
    <p>
     <strong>
      简洁语法
     </strong>
     ：无属性需求时的简化写法。
    </p>
    <pre><code class="language-javascript">function App() {
  return (
    &lt;&gt;
      &lt;Header /&gt;
      &lt;Content /&gt;
    &lt;/&gt;
  );
}</code></pre>
    <h5 id="2.%20Fragment%20%E4%B8%8E%E7%A9%BA%E6%A0%87%E7%AD%BE%E7%9A%84%E5%8C%BA%E5%88%AB" name="2.%20Fragment%20%E4%B8%8E%E7%A9%BA%E6%A0%87%E7%AD%BE%E7%9A%84%E5%8C%BA%E5%88%AB">
     <strong>
      2. Fragment 与空标签的区别
     </strong>
    </h5>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         特性
        </strong>
       </th>
       <th>
        <strong>
         显式 Fragment (
         <code>
          &lt;React.Fragment&gt;
         </code>
         )
        </strong>
       </th>
       <th>
        <strong>
         空标签 (
         <code>
          &lt;&gt;&lt;/&gt;
         </code>
         )
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         语法简洁性
        </strong>
       </td>
       <td>
        需要显式引入
        <code>
         React.Fragment
        </code>
       </td>
       <td>
        直接使用
        <code>
         &lt;&gt;...&lt;/&gt;
        </code>
        ，更简洁
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         支持属性
        </strong>
       </td>
       <td>
        ✅ 支持
        <code>
         key
        </code>
        、
        <code>
         className
        </code>
        等属性
       </td>
       <td>
        ❌ 不支持任何属性
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         适用场景
        </strong>
       </td>
       <td>
        需要传递属性（如列表循环中的
        <code>
         key
        </code>
        ）
       </td>
       <td>
        简单包裹元素，无需属性
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         编译结果
        </strong>
       </td>
       <td>
        相同（均生成
        <code>
         React.Fragment
        </code>
        元素）
       </td>
       <td>
        相同
       </td>
      </tr>
     </tbody>
    </table>
    <h5 id="3.%20%E4%BD%BF%E7%94%A8%E5%9C%BA%E6%99%AF%E5%AF%B9%E6%AF%94" name="3.%20%E4%BD%BF%E7%94%A8%E5%9C%BA%E6%99%AF%E5%AF%B9%E6%AF%94">
     <strong>
      3. 使用场景对比
     </strong>
    </h5>
    <h6 id="%E5%9C%BA%E6%99%AF%201%EF%BC%9A%E5%BE%AA%E7%8E%AF%E5%88%97%E8%A1%A8" name="%E5%9C%BA%E6%99%AF%201%EF%BC%9A%E5%BE%AA%E7%8E%AF%E5%88%97%E8%A1%A8">
     <strong>
      场景 1：循环列表
     </strong>
    </h6>
    <p>
     <strong>
      必须使用显式 Fragment
     </strong>
     ：当需要为列表项添加
     <code>
      key
     </code>
     时。
    </p>
    <pre><code class="language-javascript">function UserList({ users }) {
  return users.map((user) =&gt; (
    &lt;React.Fragment key={user.id}&gt;
      &lt;span&gt;{user.name}&lt;/span&gt;
      &lt;span&gt;{user.email}&lt;/span&gt;
    &lt;/React.Fragment&gt;
  ));
}</code></pre>
    <h6 id="%E5%9C%BA%E6%99%AF%202%EF%BC%9A%E7%AE%80%E5%8D%95%E5%88%86%E7%BB%84" name="%E5%9C%BA%E6%99%AF%202%EF%BC%9A%E7%AE%80%E5%8D%95%E5%88%86%E7%BB%84">
     <strong>
      场景 2：简单分组
     </strong>
    </h6>
    <p>
     <strong>
      优先使用空标签
     </strong>
     ：无属性需求时更简洁。
    </p>
    <pre><code class="language-javascript">function Layout() {
  return (
    &lt;&gt;
      &lt;Header /&gt;
      &lt;MainContent /&gt;
      &lt;Footer /&gt;
    &lt;/&gt;
  );
}</code></pre>
    <h5 id="4.%20%E6%B8%B2%E6%9F%93%E7%BB%93%E6%9E%9C%E5%88%86%E6%9E%90" name="4.%20%E6%B8%B2%E6%9F%93%E7%BB%93%E6%9E%9C%E5%88%86%E6%9E%90">
     <strong>
      4. 渲染结果分析
     </strong>
    </h5>
    <p>
     无论是显式 Fragment 还是空标签，编译后的结果均为
     <code>
      React.Fragment
     </code>
     ，不会生成实际的 DOM 节点。
    </p>
    <p>
     输入 JSX
    </p>
    <pre><code class="language-javascript">&lt;&gt;
  &lt;div&gt;A&lt;/div&gt;
  &lt;div&gt;B&lt;/div&gt;
&lt;/&gt;</code></pre>
    <p>
     编译后的 JavaScript
    </p>
    <pre><code class="language-javascript">React.createElement(
  React.Fragment,
  null,
  React.createElement("div", null, "A"),
  React.createElement("div", null, "B")
);</code></pre>
    <p>
     DOM 输出
    </p>
    <pre><code class="language-html">&lt;div&gt;A&lt;/div&gt;
&lt;div&gt;B&lt;/div&gt;</code></pre>
    <h5 id="5.%20%E6%B3%A8%E6%84%8F%E4%BA%8B%E9%A1%B9" name="5.%20%E6%B3%A8%E6%84%8F%E4%BA%8B%E9%A1%B9">
     <strong>
      5. 注意事项
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        属性限制
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         空标签无法传递任何属性（如
         <code>
          key
         </code>
         、
         <code>
          className
         </code>
         ），此时必须使用显式 Fragment。
        </p>
       </li>
       <li>
        <p>
         显式 Fragment 的
         <code>
          key
         </code>
         是唯一支持的属性（其他属性如
         <code>
          className
         </code>
         会被忽略）。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        版本兼容性
       </strong>
       ：
      </p>
      <ul>
       <li>
        <p>
         Fragment 在 React 16.2+ 中支持。
        </p>
       </li>
       <li>
        <p>
         空标签语法需要 Babel 7+ 或 TypeScript 3.2+ 支持。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        工具链支持
       </strong>
       ：
      </p>
     </li>
    </ol>
    <p>
     某些 IDE 或代码格式化工具可能对空标签的高亮或缩进支持不完善，但功能不受影响。
    </p>
    <h5 id="6.%20%E6%9C%80%E4%BD%B3%E5%AE%9E%E8%B7%B5" name="6.%20%E6%9C%80%E4%BD%B3%E5%AE%9E%E8%B7%B5">
     <strong>
      6. 最佳实践
     </strong>
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        优先使用空标签
       </strong>
       ：在不需要属性的场景下保持代码简洁。
      </p>
     </li>
     <li>
      <p>
       <strong>
        显式 Fragment 用于列表或属性需求
       </strong>
       ：在循环中必须为每个 Fragment 添加
       <code>
        key
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        避免冗余包裹
       </strong>
       ：仅在需要分组多个元素时使用，避免不必要的嵌套。
      </p>
     </li>
    </ol>
    <blockquote>
     <p>
      <strong>
       总结
      </strong>
     </p>
     <p>
      <strong>
       Fragment
      </strong>
      是 React 中解决 JSX 多根节点问题的标准方案。
     </p>
     <p>
      <strong>
       空标签
      </strong>
      是 Fragment 的语法糖，适用于无属性需求的场景。
     </p>
     <p>
      <strong>
       显式 Fragment
      </strong>
      必须用于需要传递
      <code>
       key
      </code>
      或其他属性的情况（如循环列表）
     </p>
    </blockquote>
