    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       1.如何优化一个慢查询？你会使用哪些工具和方法？
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      使用EXPLAIN分析查询执行计划。
     </li>
     <li style="text-align:justify">
      添加合适的索引来加速查询。
     </li>
     <li style="text-align:justify">
      避免全表扫描，优化查询条件。
     </li>
     <li style="text-align:justify">
      使用慢速查询日志，定位问题查询。
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       2.请解释MySQL中的索引是如何工作的？什么情况下索引会失效？
      </strong>
     </strong>
    </h2>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     索引通过B+树或哈希表加速数据查找。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     索引失效的情况：
    </p>
    <ol>
     <li style="text-align:justify">
      查询条件中使用了函数或表达式（如WHERE YEAR(column) = 2083）。
     </li>
     <li style="text-align:justify">
      使用了OR条件且部分列没有索引
     </li>
     <li style="text-align:justify">
      数据分布不均匀，导致优化器选择全表扫描。
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       3.如何监控MySQL的性能？你会关注哪些关键指标？
      </strong>
     </strong>
    </h2>
    <ul>
     <li style="text-align:justify">
      使用SHOW STATUS和SHOW VARIABLES查看MySQL状态和配置。
     </li>
     <li style="text-align:justify">
      使用工具如pt-query-digest分析慢查询日志。
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     关键指标：
    </p>
    <ol>
     <li style="text-align:justify">
      查询响应时间
     </li>
     <li style="text-align:justify">
      连接数
     </li>
     <li style="text-align:justify">
      缓存命中率
     </li>
    </ol>
    <h2 style="margin-left:0.0001pt; margin-right:0px; text-align:justify">
     <strong>
      <strong>
       4.如何解决MySQL中的死锁问题？
      </strong>
     </strong>
    </h2>
    <ol>
     <li style="text-align:justify">
      使用SHOW ENGINE INNODB STATUS可以查看死锁的信息
     </li>
     <li style="text-align:justify">
      优化事务逻辑，减少锁竞争
     </li>
     <li style="text-align:justify">
      使用innodb_lock_wait_timeout设置锁等待超时时间
     </li>
    </ol>
