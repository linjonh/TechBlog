     1.1.什么是小程序？
    </h3>
    <blockquote>
     <ol>
      <li>
       2017 年度百度百科十大热词之一
      </li>
      <li>
       微信小程序，简称小程序，英文名 Mini Program，是一种不需要下载安装即可使用的应用
       <br/>
       ( 张小龙对其的定义是无需安装，用完即走，实际上是需要安装的，只不过小程序的体积特
       <br/>
       别小， 下载速度很快，用户感觉不到下载的过程 )
      </li>
      <li>
       小程序刚发布的时候要求压缩包的体积不能大于 1M,，否则无法通过，在 2017 年 4 月做 了改进，由原来的 1M 提升到 2M；
      </li>
      <li>
       2017 年 1 月 9 日 0 点，万众瞩目的微信第一批小程序正式低调上线。
      </li>
     </ol>
    </blockquote>

     1.2. 微信⼩程序历史
    </h3>
    <blockquote>
     <p>
      （1）2016年1⽉11⽇，微信之⽗张⼩⻰时隔多年的公开亮相，解读了微信的四⼤价值观。张⼩⻰指出，
      <br/>
      越来越多产品通过公众号来做，因为这⾥开发、获取⽤⼾和传播成本更低。拆分出来的服务号并没
      <br/>
      有提供更好的服务，所以微信内部正在研究新的形态，叫「微信⼩程序」 需要注意的是，之前是叫
      <br/>
      做 应⽤号
      <br/>
      （2）2016年9⽉21⽇，微信⼩程序正式开启内测。在微信⽣态下，触⼿可及、⽤完即⾛的微信⼩程序引
      <br/>
      起⼴泛关注。腾讯云正式上线微信⼩程序解决⽅案，提供⼩程序在云端服务器的技术⽅案。
      <br/>
      （3）2017年1⽉9⽇，微信推出的“⼩程序”正式上线。“⼩程序”是⼀种⽆需安装，即可使⽤的⼿
      <br/>
      机“应⽤”。不需要像往常⼀样下载App，⽤⼾在微信中“⽤完即⾛”。
     </p>
    </blockquote>

     </strong>
    </p>
    <blockquote>
     <p>
      设备独立像素(也叫密度无关像素)，可以认为是计算机坐标系统中的一个点，这个点代
      <br/>
      表一个可以由程序使用并控制的虚拟像素(比如：CSS 像素,只是在 android 机中 CSS 像素就
      <br/>
      不叫”CSS 像素”了而是叫”设备独立像素”)，然后由相关系统转换为物理像素
     </p>
    </blockquote>

     <strong>
      4.2.3. dpr 比 &amp; DPI &amp; PPI
     </strong>
    </p>

     <li>
      PPI: 一英寸显示屏上的像素点个数
     </li>
     <li>
      DPI：最早指的是打印机在单位面积上打印的墨点数，墨点越多越清晰

     <strong>
      8.8. 小程序分包流程
     </strong>
     <br/>
     <strong>
      8.8.1. 为什么要分包
     </strong>
    </p>
    <ol>
     <li>
      小程序要求压缩包体积不能大于 2M，否则无法发布
     </li>
     <li>
      实际开发中小程序体积如果大于 2M 就需要使用分包机制进行发布上传
     </li>
     <li>
      分包后可解决 2M 限制，并且能分包加载内容，提高性能
     </li>
     <li>
      分包后单个包的体积不能大于 2M
     </li>
     <li>
      分包后所有包的体积不能大于 16M
     </li>
    </ol>
    <p>
     <strong>
      8.8.2. 分包形式
     </strong>
    </p>
    <ol>
     <li>

     </li>

     <li>

     </li>
    </ol>
    <p>
     <strong>
      8.8.3. 常规分包
     </strong>
    </p>
    <ol>
     <li>
      开发者通过在 app.json subpackages 字段声明项目分包结构
     </li>
     <li>
      特点：
      <br/>
      a) 加载小程序的时候先加载主包，当需要访问分包的页面时候才加载分包内容
      <br/>
      b) 分包的页面可以访问主包的文件，数据，图片等资源
      <br/>
      c) 主包：
      <br/>
      i. 主包来源： 除了分包以外的内容都会被打包到主包中
      <br/>
      ii. 通常放置启动页/tabBar 页面
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/ce175071fb66d98025cbdb6979706765.png"/>

      8.8.4. 独立分包
     </strong>

     </li>
     <li>
      特点:
      <br/>
      a) 独立分包可单独访问分包的内容，不需要下载主包
      <br/>
      b) 独立分包不能依赖主包或者其他包的内容
     </li>

      <br/>
      a) 通常某些页面和当前小程序的其他页面关联不大的时候可进行独立分包

    <p>
     <strong>
      9.1. 支付流程官网图解
     </strong>
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/e74e6f11a066a768eaf1f555a71bbde8.png"/>
    </p>
    <p>
     <strong>
      9.2. 支付流程详细说明
     </strong>
    </p>
    <ol>
     <li>
      用户在小程序客服端下单(包含用户及商品信息)
     </li>
     <li>
      小程序客户端发送下单支付请求给商家服务器
     </li>
     <li>
      商家服务器同微信服务器对接获取唯一标识 openID
     </li>
     <li>
      商家服务器根据 openId 生成商户订单(包含商户信息)
     </li>
     <li>
      商家服务器发送请求调用统一下单 API 获取预支付订单信息
      <br/>
      a)
      <a href="https://api.mch.weixin.qq.com/pay/unifiedorder" rel="nofollow">
       接口地址
      </a>
      https://api.mch.weixin.qq.com/pay/unifiedorder
     </li>
     <li>
      商家对预支付信息签名加密后返回给小程序客户端
      <br/>
      a) 签名方式： MD5
      <br/>
      b) 签名字段：小程序 ID, 时间戳， 随机串，数据包，签名方式
      <br/>
      c)
      <a href="https://pay.weixin.qq.com/wiki/doc/api/wxa/wxa_api.php?chapter=7_7&amp;index=3" rel="nofollow">
       参考地址
      </a>
      <br/>
      https://pay.weixin.qq.com/wiki/doc/api/wxa/wxa_api.php?chapter=7_7&amp;index=3
     </li>
     <li>
      用户确认支付（鉴权调起支付）
      <br/>
      a) API: wx.requestPayment()
     </li>
     <li>
      微信服务器返回支付结果给小程序客户端
     </li>
     <li>
      微信服务器推送支付结果给商家服务器端
     </li>
    </ol>
