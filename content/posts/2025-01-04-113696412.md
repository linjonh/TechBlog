     <p style="text-align: center">
      <strong>
       前言
      </strong>
      <br/>
     </p>
     <p>
      使用 SpringCloud 架构后，我们希望所有的请求都需要经过网关才能访问。在不作任何处理的情况下，我们是可以绕过网关直接访问后端服务的。如下，我们绕过网关直接访问后端服务也是可以获取到数据的。
      <br/>
      <br/>
     </p>
     <p style="text-align: center">
      <img src="https://i-blog.csdnimg.cn/blog_migrate/e6a35d45fc351e0a6d76fc369bc99aaf.png"/>
     </p>
     <p>
      那我们今天的议题就是
      <strong>
       如何防止请求绕过网关直接访问后端服务？
      </strong>
     </p>
     <h4>
      <br/>
     </h4>
     <p style="text-align: center">
      <img src="https://i-blog.csdnimg.cn/blog_migrate/ee679d0db6d97d91090b537e6ab55f33.png"/>
     </p>
     <h3>
      <strong>
       解决方案
      </strong>
      <br/>
     </h3>
     <p>
      我觉得防止绕过网关直接请求后端服务的解决方案主要有三种：
     </p>
     <ul>
      <li>
       <p>
        使用 Kubernetes 部署
       </p>
       <p>
        在使用 Kubernetes 部署 SpringCloud 架构时，我们给网关的 Service 配置 NodePort，其他后端服务的 Service 使用 ClusterIp，这样在集群外就只能访问到网关了。
       </p>
      </li>
      <li>
       <p>
        网络隔离
       </p>
       <p>
        后端普通服务都部署在内网，通过防火墙策略限制只允许网关应用访问后端服务。
       </p>
      </li>
      <li>
       <p>
        应用层拦截
       </p>
       <p>
        请求后端服务时通过拦截器校验请求是否来自网关，如果不来自网关则提示不允许访问。
       </p>
      </li>
     </ul>
     <p>
      这里我们着重关注在
      <strong>
       应用层拦截
      </strong>
      这种解决方案。
     </p>
     <h4>
      <br/>
     </h4>
     <p style="text-align: center">
      <img src="https://i-blog.csdnimg.cn/blog_migrate/9ef615a857dc2e74ec5f26d3a4d9165a.png"/>
     </p>
     <h4>
      <strong>
       实现
      </strong>
      <br/>
     </h4>
     <p>
      <strong>
       实现思路
      </strong>
     </p>
     <p>
      实现思路其实也很简单，
      <strong>
       在请求经过网关的时候给请求头中增加一个额外的 Header，在后端服务中写一个拦截器，判断请求头是否与在网关设置的请求 Header 一致，如果不一致则不允许访问并给出提示。
      </strong>
     </p>
     <p>
      当然为了防止在每个后端服务都需要编写这个拦截器，我们可以将其写在一个公共的 starter 中，让后端服务引用即可。而且为了灵活，可以通过配置决定是否只允许后端服务访问。
     </p>
     <p>
      接下来我们看看核心代码。
     </p>
     <h4>
      <strong>
       实现过程
      </strong>
     </h4>
     <p>
      在网关 cloud-gateway 模块编写网关过滤器
     </p>
     <pre class="has"><code class="language-go">@Component@Order(0)public class GatewayRequestFilter implements GlobalFilter {
    @Override    public Mono&lt;Void&gt; filter(ServerWebExchange exchange, GatewayFilterChain chain) {        byte[] token = Base64Utils.encode((CloudConstant.GATEWAY_TOKEN_VALUE).getBytes());        String[] headerValues = {new String(token)};        ServerHttpRequest build = exchange.getRequest()                .mutate()                .header(CloudConstant.GATEWAY_TOKEN_HEADER, headerValues)                .build();
        ServerWebExchange newExchange = exchange.mutate().request(build).build();        return chain.filter(newExchange);    }
}
</code></pre>
     <p>
      在请求经过网关时添加额外的 Header，为了方便这里直接设置成固定值。
     </p>
     <p>
      建立公共 Starter 模块 cloud-component-security-starter
     </p>
     <p style="text-align: center">
      <img src="https://i-blog.csdnimg.cn/blog_migrate/d26711cecb9b64059079f4b39768a528.png"/>
     </p>
     <p>
      编写配置类，用于灵活控制服务是否允许绕过网关
     </p>
     <pre class="has"><code class="language-kotlin">@Data
@ConfigurationProperties(prefix = "javadaily.cloud")
public class CloudSecurityProperties {

    /**
     * 是否只能通过网关获取资源
     * 默认为True
     */
    private Boolean onlyFetchByGateway = Boolean.TRUE;

}

</code></pre>
<p>
编写拦截器，用于校验请求是否经过网关
</p>
<pre class="has"><code class="language-go">public class ServerProtectInterceptor implements HandlerInterceptor {
private CloudSecurityProperties properties;
@Override public boolean preHandle(@NonNull HttpServletRequest request, @NonNull HttpServletResponse response, @NonNull Object handler){
if (!properties.getOnlyFetchByGateway()) { return true; }
String token = request.getHeader(CloudConstant.GATEWAY_TOKEN_HEADER);
String gatewayToken = new String(Base64Utils.encode(CloudConstant.GATEWAY_TOKEN_VALUE.getBytes()));
if (StringUtils.equals(gatewayToken, token)) { return true; } else { ResultData&lt;String&gt; resultData = new ResultData&lt;&gt;(); resultData.setSuccess(false); resultData.setStatus(HttpServletResponse.SC_FORBIDDEN); resultData.setMessage("请通过网关访问资源"); WebUtils.writeJson(response,resultData); return false; } }
public void setProperties(CloudSecurityProperties properties) { this.properties = properties; }}
</code></pre>
<p>
配置拦截器
</p>
<pre class="has"><code class="language-go">public class CloudSecurityInterceptorConfigure implements WebMvcConfigurer {
private CloudSecurityProperties properties;
@Autowired public void setProperties(CloudSecurityProperties properties) { this.properties = properties; }
@Bean public HandlerInterceptor serverProtectInterceptor() { ServerProtectInterceptor interceptor = new ServerProtectInterceptor(); interceptor.setProperties(properties); return interceptor; }
@Override public void addInterceptors(InterceptorRegistry registry) { registry.addInterceptor(serverProtectInterceptor()); }}
</code></pre>
<p>
编写 starter 装载类
</p>
<pre class="has"><code class="language-go">@EnableConfigurationProperties(CloudSecurityProperties.class)public class CloudSecurityAutoConfigure{
@Bean public CloudSecurityInterceptorConfigure cloudSecurityInterceptorConfigure() { return new CloudSecurityInterceptorConfigure(); }
}
</code></pre>
<p>
建立资源文件 spring.factories，配置 Bean 的自动加载
</p>
<pre class="has"><code class="language-properties">org.springframework.boot.autoconfigure.EnableAutoConfiguration=\   com.javadaily.component.security.configure.CloudSecurityAutoConfigure
</code></pre>
<p>
在后端服务配置文件中添加属性配置，默认只能通过网关访问
</p>
<pre class="has"><code class="language-go">javadaily: cloud: onlyFetchByGateway: true
</code></pre>
<p>
经过以上几步，一个公共的 Starter 模块就构建完成了。
</p>
<p>
后端服务引用此公共 Starter 模块即可，以 account-service 为例
</p>
<pre class="has"><code class="language-go">&lt;dependency&gt; &lt;groupId&gt;com.jianzh5.cloud&lt;/groupId&gt; &lt;artifactId&gt;cloud-component-security-starter&lt;/artifactId&gt;&lt;/dependency&gt;
</code></pre>
