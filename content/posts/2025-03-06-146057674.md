     在
     <code>
      better-sqlite3

     中，
     <code>

     </code>

    </p>

    <h4>
     <a id="_83">
     </a>
     详细解释
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        读取 SQL 文件
       </strong>
       ：
      </p>
      <ul>
       <li>
        使用
        <code>
         fs.readFileSync()
        </code>
        读取 SQL 文件的内容。这里我们使用
        <code>
         path.join()
        </code>
        来确保路径的兼容性。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        开始事务
       </strong>
       ：
      </p>
      <ul>
       <li>
        在执行 SQL 语句之前，首先调用
        <code>
         db.exec('BEGIN TRANSACTION;')
        </code>
        开始一个事务。这可以确保所有的 SQL 语句要么全部成功，要么全部失败，从而保持数据的一致性。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        执行 SQL 文件中的所有语句
       </strong>
       ：
      </p>
      <ul>
       <li>
        使用
        <code>
         db.exec(migration)
        </code>
        执行从文件中读取的所有 SQL 语句。注意，
        <code>
         .exec()
        </code>
        可以执行包含多个 SQL 语句的字符串。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        提交事务
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果所有 SQL 语句都成功执行，则调用
        <code>
         db.exec('COMMIT;')
        </code>
        提交事务。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        错误处理和事务回滚
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果在执行 SQL 语句的过程中发生错误，捕获异常并调用
        <code>
         db.exec('ROLLBACK;')
        </code>
        回滚事务，以防止部分更新导致的数据不一致问题。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        关闭数据库连接
       </strong>
       ：
      </p>
      <ul>
       <li>
        最后，调用
        <code>
         db.close()
        </code>
        关闭数据库连接。
       </li>
      </ul>
     </li>
    </ol>

    <h4>
     <a id="_129">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        安全性
       </strong>
       ：由于
       <code>
        .exec()
       </code>
       直接执行 SQL 字符串，因此存在 SQL 注入的风险。尽量避免直接将用户输入插入到
       <code>
        .exec()
       </code>
       调用中。如果必须这样做，请先进行严格的验证和清理。
      </p>
     </li>
     <li>
      <p>
       <strong>
        性能
       </strong>
       ：与预编译语句相比，
       <code>
        .exec()
       </code>
       的性能较差。如果可能的话，尽量使用预编译语句来提高性能和安全性。
      </p>
     </li>
     <li>
      <p>
       <strong>
        事务管理
       </strong>
       ：当执行多个 SQL 语句时，务必使用事务来确保数据一致性。如果没有使用事务，部分语句的成功执行可能会导致数据库处于不一致状态。
      </p>
     </li>
    </ul>
