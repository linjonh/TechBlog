    <p>
     常见的数据集Pascal VOC/COCO
    </p>
    <p>
     DataLoader
    </p>
    <p>
     DataLoader( dataset,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：指定要加载的数据集，它必须是
       <code>
        torch.utils.data.Dataset
       </code>
       类的子类实例。
       <code>
        Dataset
       </code>
       类定义了如何获取单个样本以及数据集的长度，
       <code>
        DataLoader
       </code>
       会基于这个数据集进行数据的批量加载。
      </span>
     </li>
    </ul>
    <p>
     batch_size=1,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        <code>
         batch_size
        </code>
        ：
       </strong>
      </span>
      <ul>
       <li>
        <span style="color:#ed7976">
         <strong>
          含义：每个批次加载的样本数量，默认值为 1。在训练神经网络时，通常会将多个样本组成一个批次一起输入到模型中，以提高训练效率和稳定性。
         </strong>
        </span>
       </li>
       <li>
        <span style="color:#ed7976">
         <strong>
          示例：
          <code>
           dataloader = DataLoader(dataset, batch_size=10)
          </code>
         </strong>
        </span>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     shuffle=False,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：布尔类型参数，若设置为
       <code>
        True
       </code>
       ，则在每个 epoch 开始时打乱数据集的顺序，有助于模型更好地学习数据的分布，提高模型的泛化能力，默认值为
       <code>
        False
       </code>
       。
      </span>
     </li>
     <li>
      <span style="color:#ed7976">
       <strong>
        示例
       </strong>
       ：
       <code>
        dataloader = DataLoader(dataset, batch_size=10, shuffle=True)
       </code>
      </span>
     </li>
    </ul>
    <p>
     sampler=None,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        <code>
         sampler
        </code>
       </strong>
       ：
      </span>
     </li>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：自定义采样器，用于指定如何从数据集中采样样本。如果指定了
       <code>
        sampler
       </code>
       ，则
       <code>
        shuffle
       </code>
       参数将被忽略。采样器必须是
       <code>
        torch.utils.data.Sampler
       </code>
       类的子类实例。
      </span>
     </li>
    </ul>
    <p>
     batch_sampler=None,
    </p>
    <p>
     <span style="color:#ed7976">
      <strong>
       <code>
        batch_sampler
       </code>
      </strong>
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：自定义批次采样器，用于指定如何将样本组合成批次。如果指定了
       <code>
        batch_sampler
       </code>
       ，则
       <code>
        batch_size
       </code>
       、
       <code>
        shuffle
       </code>
       、
       <code>
        sampler
       </code>
       和
       <code>
        drop_last
       </code>
       参数将被忽略。批次采样器必须是
       <code>
        torch.utils.data.BatchSampler
       </code>
       类的子类实例。
      </span>
     </li>
    </ul>
    <p>
     num_workers=0,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        <code>
         num_workers
        </code>
        ：
       </strong>
      </span>
      <ul>
       <li>
        <span style="color:#ed7976">
         <strong>
          含义：用于数据加载的子进程数量，设置为大于 0 的值可以实现并行数据加载，提高数据加载速度。默认值为 0，表示在主进程中加载数据。在使用多进程加载时，需要注意内存和 CPU 资源的使用情况。
         </strong>
        </span>
       </li>
       <li>
        <span style="color:#ed7976">
         <strong>
          示例：
          <code>
           dataloader = DataLoader(dataset, batch_size=10, num_workers=4)
          </code>
         </strong>
        </span>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     collate_fn=None,
    </p>
    <p>
     <span style="color:#ed7976">
      <strong>
       <code>
        collate_fn
       </code>
      </strong>
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：用于将多个样本组合成一个批次的函数。默认情况下，
       <code>
        DataLoader
       </code>
       会使用一个简单的默认函数将样本组合成批次，但在处理一些特殊的数据类型或结构时，可能需要自定义
       <code>
        collate_fn
       </code>
       函数。
      </span>
     </li>
    </ul>
    <p>
     pin_memory=False, drop_last=False,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        <code>
         pin_memory
        </code>
       </strong>
       ：
      </span>
      <ul>
       <li>
        <span style="color:#ed7976">
         <strong>
          含义
         </strong>
         ：布尔类型参数，若设置为
         <code>
          True
         </code>
         ，则会将数据样本在返回前固定到 CUDA 页锁定内存中，这样可以加快数据从 CPU 到 GPU 的传输速度，适用于使用 GPU 进行训练的场景，默认值为
         <code>
          False
         </code>
         。
        </span>
       </li>
       <li>
        <span style="color:#ed7976">
         <strong>
          示例
         </strong>
         ：
         <code>
          dataloader = DataLoader(dataset, batch_size=10, pin_memory=True)
         </code>
        </span>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     timeout=0, worker_init_fn=None,
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        <code>
         timeout
        </code>
       </strong>
       ：
      </span>
      <ul>
       <li>
        <span style="color:#ed7976">
         <strong>
          含义
         </strong>
         ：在多进程数据加载时，等待从工作进程获取数据的超时时间（单位：秒）。如果在指定的时间内没有获取到数据，则会抛出
         <code>
          TimeoutError
         </code>
         异常。默认值为 0，表示不设置超时时间。
        </span>
       </li>
       <li>
        <span style="color:#ed7976">
         <strong>
          示例
         </strong>
         ：
         <code>
          dataloader = DataLoader(dataset, batch_size=10, num_workers=4, timeout=1)
         </code>
        </span>
       </li>
      </ul>
     </li>
    </ul>
    <p>
     multiprocessing_context=None,
    </p>
    <p>
     <span style="color:#ed7976">
      <strong>
       <code>
        multiprocessing_context
       </code>
      </strong>
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：指定多进程的上下文，可以是
       <code>
        'fork'
       </code>
       、
       <code>
        'spawn'
       </code>
       或
       <code>
        'forkserver'
       </code>
       等。不同的上下文适用于不同的操作系统和场景，默认情况下会根据操作系统自动选择合适的上下文。
      </span>
     </li>
    </ul>
    <p>
     generator=None,
    </p>
    <p>
     <span style="color:#ed7976">
      <strong>
       <code>
        generator
       </code>
      </strong>
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：用于生成随机数的生成器，可用于控制数据的打乱顺序等随机操作。可以传入一个
       <code>
        torch.Generator
       </code>
       实例，方便复现实验结果。
      </span>
     </li>
    </ul>
    <p>
     prefetch_factor=2,
    </p>
    <p>
     <span style="color:#ed7976">
      <strong>
       <code>
        prefetch_factor
       </code>
      </strong>
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：每个工作进程预取的样本数量，默认值为 2。在多进程数据加载时，工作进程会预先加载一定数量的样本，以提高数据加载的连续性。
      </span>
     </li>
    </ul>
    <p>
     persistent_workers=False )
    </p>
    <p>
     <strong>
      <code>
       p
       <span style="color:#ed7976">
        ersistent_workers
       </span>
      </code>
     </strong>
     <span style="color:#ed7976">
      ：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#ed7976">
       <strong>
        含义
       </strong>
       ：布尔类型参数，若设置为
       <code>
        True
       </code>
       ，则在每个 epoch 结束后，工作进程不会被销毁，而是保持存活状态，这样可以减少进程创建和销毁的开销，提高数据加载效率，默认值为
       <code>
        False
       </code>
       。
      </span>
     </li>
