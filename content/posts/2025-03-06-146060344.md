    <h3 id="sTQ1C">
     1、_.isPlainObject(value)
    </h3>
    <h5 id="3jv8x">
     参数
    </h5>
    1. `value` _(*)_: 要检查的值。
    <h5 id="HnwzN">
     返回
    </h5>
    _(boolean)_: 如果 `value` 为一个普通对象，那么返回 `true`，否则返回 `false`。
    <h5 id="SkEQj">
     例子
    </h5>
    ```plain
    <p>
     function Foo() {
     <!-- -->
     <br/>
     this.a = 1;
     <br/>
     }
     <br/>
     _.isPlainObject(new Foo);
     <br/>
     // =&gt; false
     <br/>
     _.isPlainObject([1, 2, 3]);
     <br/>
     // =&gt; false
     <br/>
     _.isPlainObject({ ‘x’: 0, ‘y’: 0 });
     <br/>
     // =&gt; true
     <br/>
     _.isPlainObject(Object.create(null));
     <br/>
     // =&gt; true
    </p>

&lt;font style="color:#1C1E21;"&gt;检查 &lt;/font&gt;`value`&lt;font style="color:#1C1E21;"&gt; 是否为 &lt;/font&gt;`Object`&lt;font style="color:#1C1E21;"&gt; 的&lt;/font&gt;[language type](http://www.ecma-international.org/ecma-262/6.0/#sec-ecmascript-language-types)&lt;font style="color:#1C1E21;"&gt;。 &lt;/font&gt;_(例如： arrays, functions, objects, regexes,_`_new Number(0)_`_, 以及 _`_new String('')_`_)_

&lt;h4 id="bqOJk"&gt;参数&lt;/h4&gt;
1. `value` _(*)_: 要检查的值。

&lt;h4 id="TBgWw"&gt;返回&lt;/h4&gt;
_(boolean)_: 如果 `value` 为一个对象，那么返回 `true`，否则返回 `false`。

_.isObject({});

    <h3 id="wp3Kf">
     2、_.get(object, path, [defaultValue])
    </h3>
    根据 `object` 
对象的`path` 
路径获取值。 如果解析 value 是 `undefined` 
 会以 `defaultValue` 
 取代。
    <h5 id="hxhVe">
     参数
    </h5>
    1. `object` _(Object)_: 要检索的对象。 2. `path` _(Array|string)_: 要获取属性的路径。 3. `[defaultValue]` _(*)_: 如果解析值是 `undefined` ，这值会被返回。
    <h5 id="JPtrm">
     返回
    </h5>
    _(*)_: 返回解析的值。
    <h5 id="7aPlj">
     例子
    </h5>
    ```plain var object = { 'a': [{ 'b': { 'c': 3 } }] }; _.get(object, 'a[0].b.c'); // =&gt; 3 _.get(object, ['a', '0', 'b', 'c']); // =&gt; 3 _.get(object, 'a.b.c', 'default'); // =&gt; 'default' ```
    <h3 id="3Vzuu">
     3、_.hasIn(object, path)
    </h3>
    检查 `path` 
 是否是`object` 
对象的直接或继承属性。
    <h5 id="zr6yp">
     参数
    </h5>
    1. `object` _(Object)_: 要检索的对象。 2. `path` _(Array|string)_: 要检查的路径`path`。
    <h5 id="o3ZXd">
     返回
    </h5>
    _(boolean)_: 如果`path`存在，那么返回 `true` ，否则返回 `false`。
    <h5 id="IBcgO">
     例子
    </h5>
    ```plain var object = _.create({ 'a': _.create({ 'b': 2 }) });
    <p>
     _.hasIn(object, ‘a’);
     <br/>
     // =&gt; true
    </p>
    <p>
     _.hasIn(object, ‘a.b’);
     <br/>
     // =&gt; true
    </p>
    <p>
     _.hasIn(object, [‘a’, ‘b’]);
     <br/>
     // =&gt; true
    </p>
    <p>
     _.hasIn(object, ‘b’);
     <br/>
     // =&gt; false
    </p>
    <pre><code>
xfront项目中的相关使用例子如下：

```plain
 const feedback = response =&gt; {
    const type = get(response.data, 'type', 0);
    const msg = getMsg(response);
    const hasFrontMsg = hasIn(response.config, 'msg');
    switch (type) {
      case 0:
        if (hasFrontMsg) { // 此处的用处是否可以理解为判断response.config里是否包含msg属性
          Message.success(msg);
        }
        break;
      case -1:
        Message.error(msg);
        break;
      case -2:
        if (response.config.responseType !== 'blob') {
          showError({
            ...response.data,
            msg
          });
        }
        break;
    }
  };
</code></pre>
    <h4 id="UUhai">
     4、`_.defaultTo(value, defaultValue)`
    </h4>
    检查`value` 
，以确定一个默认值是否应被返回。如果`value` 
为`NaN` 
, `null` 
, 或者 `undefined` 
，那么返回`defaultValue` 
默认值。
    <h5 id="DOTap">
     参数
    </h5>
    1. `value` _(*)_: 要检查的值。 2. `defaultValue` _(*)_: 默认值。
    <h5 id="Se99Q">
     返回
    </h5>
    _(*)_: 返回 resolved 值。
    <h5 id="ZjBJb">
     例子
    </h5>
    ```plain _.defaultTo(1, 10); // =&gt; 1
    <p>
     _.defaultTo(undefined, 10);
     <br/>
     // =&gt; 10
    </p>
