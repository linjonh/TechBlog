    <p>
     在 Android 开发中，我们经常需要对字符串进行格式化处理，比如动态填充数字、日期、字符等。
     <br/>
     其中，
     <code>
      %d
     </code>
     和
     <code>
      1$%d
     </code>
     都是格式化占位符，但它们在使用上有一些不同。
     <br/>
     本文将详细解析这两者的区别，并结合 Kotlin 代码示例帮助你理解。
    </p>
    <h3>
     <a id="1_d__1d__7">
     </a>
     1.
     <code>
      %d
     </code>
     和
     <code>
      1$%d
     </code>
     的基本概念
    </h3>
    <ul>
     <li>
      <code>
       %d
      </code>
      ：表示整数占位符，按照传入参数的顺序依次填充。
     </li>
     <li>
      <code>
       1$%d
      </code>
      ：表示带索引的占位符，
      <code>
       1$
      </code>
      代表第一个参数，
      <code>
       2$
      </code>
      代表第二个，以此类推。
     </li>
    </ul>

     </a>

    <h3>
     <a id="3__40">
     </a>
     3. 区别解析
    </h3>
    <h4>
     <a id="31_d__42">
     </a>
     3.1
     <code>
      %d
     </code>
     的工作原理
    </h4>
    <ul>
     <li>
      直接按照传入的参数顺序进行替换。
     </li>
     <li>
      如果参数顺序变了，结果也会相应改变。
     </li>
    </ul>
    <h4>
     <a id="32_1d__47">
     </a>
     3.2
     <code>
      1$%d
     </code>
     的工作原理
    </h4>
    <ul>
     <li>
      <code>
       1$
      </code>
      明确指定了参数的位置，即便参数顺序变动，引用的仍然是指定位置的值。
     </li>
     <li>
      适用于多语言（国际化）情况，可以根据语言调整顺序，而不用改变代码。
     </li>
    </ul>
    <h3>
     <a id="4__1d_52">
     </a>
     4. 为什么需要
     <code>
      1$%d
     </code>
    </h3>
    <p>
     在多语言（Localization）场景下，不同语言的语序可能不同，比如：
    </p>
    <ul>
     <li>
      <strong>
       中文
      </strong>
      ：2025 年 3 月 11 日
     </li>
     <li>
      <strong>
       英语
      </strong>
      ：March 11, 2025
     </li>
    </ul>
    <p>
     如果我们只用
     <code>
      %d
     </code>
     ，就需要改动代码传参顺序，而
     <code>
      1$%d
     </code>
     允许我们调整字符串资源中的格式，而无需改变代码。
    </p>
    <h3>
     <a id="5__Android__61">
     </a>
     5. 在 Android 开发中的使用
    </h3>
    <p>
     在 Android 的
     <code>
      strings.xml
     </code>
     中，我们通常这样定义格式化字符串：
    </p>
    <pre><code class="prism language-xml"><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>string</span> <span class="token attr-name">name</span><span class="token attr-value"><span class="token punctuation attr-equals">=</span><span class="token punctuation">"</span>date_format<span class="token punctuation">"</span></span><span class="token punctuation">&gt;</span></span>%1$d 年 %2$d 月 %3$d 日<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>string</span><span class="token punctuation">&gt;</span></span>

    <p>
     然后在代码中使用：
    </p>
    <pre><code class="prism language-kotlin"><span class="token keyword">val</span> formattedDate <span class="token operator">=</span> context<span class="token punctuation">.</span><span class="token function">getString</span><span class="token punctuation">(</span>R<span class="token punctuation">.</span>string<span class="token punctuation">.</span>date_format<span class="token punctuation">,</span> <span class="token number">2025</span><span class="token punctuation">,</span> <span class="token number">3</span><span class="token punctuation">,</span> <span class="token number">11</span><span class="token punctuation">)</span>

    <h3>
     <a id="6__76">
     </a>
     6. 总结
    </h3>
    <ul>
     <li>
      <code>
       %d
      </code>
      适用于简单的格式化，不涉及参数顺序调整。
     </li>
     <li>
      <code>
       1$%d
      </code>
      适用于国际化（i18n）等需要调整参数顺序的场景。
     </li>
     <li>
      在 Android 开发中，推荐在
      <code>
       strings.xml
      </code>
      里使用
      <code>
       1$%d
      </code>
      ，确保不同语言的兼容性。
     </li>
    </ul>
    <p>
     这样在适配不同语言时，只需要修改
     <code>
      strings.xml
     </code>
     ，而不需要调整代码的参数传递顺序。
    </p>
