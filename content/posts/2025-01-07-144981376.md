    <p>
     当前主流用于 AI 的数据库可以分为几大类：关系型数据库、非关系型数据库、时序数据库和专门为 AI/ML 优化的数据库。
    </p>
    <h4>
     <strong>
      1. 关系型数据库 (RDBMS)
     </strong>
    </h4>
    <h5>
     主流数据库：
    </h5>
    <ul>
     <li>
      <strong>
       MySQL
      </strong>
      : 开源、高效，适合中小型应用。
     </li>
     <li>
      <strong>
       PostgreSQL
      </strong>
      : 高度可扩展，支持复杂查询和事务。
     </li>
     <li>
      <strong>
       Oracle Database
      </strong>
      : 商业化数据库，功能全面，企业级支持。
     </li>
     <li>
      <strong>
       SQL Server
      </strong>
      : 微软产品，紧密集成于 Windows 系统中。
     </li>
    </ul>
    <h5>
     功能特点：
    </h5>
    <ul>
     <li>
      支持结构化数据（表格格式）。
     </li>
     <li>
      强大的事务支持（ACID特性）。
     </li>
     <li>
      适用于需要严格一致性和关系操作的应用，如财务、客户管理。
     </li>
    </ul>
    <h5>
     使用场景：
    </h5>
    <p>
     用于结构化数据处理，例如存储用户信息、产品信息、订单等。
    </p>
    <h4>
     <strong>
      2. NoSQL 数据库
     </strong>
    </h4>
    <h5>
     主流数据库：
    </h5>
    <ul>
     <li>
      <strong>
       MongoDB
      </strong>
      : 文档型数据库，支持灵活的 JSON 数据。
     </li>
     <li>
      <strong>
       Redis
      </strong>
      : 内存数据库，极高的读写速度，支持键值存储。
     </li>
     <li>
      <strong>
       Cassandra
      </strong>
      : 分布式数据库，适合大规模写入和高可用。
     </li>
     <li>
      <strong>
       Elasticsearch
      </strong>
      : 搜索引擎，适合全文搜索和实时数据分析。
     </li>
    </ul>
    <h5>
     功能特点：
    </h5>
    <ul>
     <li>
      适合非结构化或半结构化数据。
     </li>
     <li>
      更灵活的存储模式（文档、键值、列族、图）。
     </li>
     <li>
      强大的扩展性和高性能。
     </li>
    </ul>
    <h5>
     使用场景：
    </h5>
    <ul>
     <li>
      MongoDB 用于存储灵活的用户数据（如社交媒体内容）。
     </li>
     <li>
      Redis 用于缓存层和实时排名。
     </li>
     <li>
      Cassandra 适用于分布式日志存储。
     </li>
     <li>
      Elasticsearch 用于日志分析或构建搜索功能。
     </li>
    </ul>
    <h4>
     <strong>
      3. 图数据库
     </strong>
    </h4>
    <h5>
     主流数据库：
    </h5>
    <ul>
     <li>
      <strong>
       Neo4j
      </strong>
      : 面向关系图的分析，查询语言为 Cypher。
     </li>
     <li>
      <strong>
       TigerGraph
      </strong>
      : 支持大规模图计算。
     </li>
     <li>
      <strong>
       ArangoDB
      </strong>
      : 多模型数据库，支持图、文档和键值存储。
     </li>
    </ul>
    <h5>
     功能特点：
    </h5>
    <ul>
     <li>
      专注于处理图状数据，如节点和边。
     </li>
     <li>
      支持复杂关系查询。
     </li>
     <li>
      高效的路径分析和推荐。
     </li>
    </ul>
    <h5>
     使用场景：
    </h5>
    <ul>
     <li>
      推荐系统（如社交网络好友推荐）。
     </li>
     <li>
      诈骗检测（如银行的交易网络分析）。
     </li>
     <li>
      知识图谱构建。
     </li>
    </ul>
    <h4>
     <strong>
      4. 时序数据库
     </strong>
    </h4>
    <h5>
     主流数据库：
    </h5>
    <ul>
     <li>
      <strong>
       InfluxDB
      </strong>
      : 专门为时序数据设计，支持高效的时间点存储。
     </li>
     <li>
      <strong>
       TimescaleDB
      </strong>
      : 基于 PostgreSQL，支持时序扩展。
     </li>
     <li>
      <strong>
       OpenTSDB
      </strong>
      : 基于 HBase 构建，适合大规模时序数据。
     </li>
    </ul>
    <h5>
     功能特点：
    </h5>
    <ul>
     <li>
      处理时间序列数据，如传感器数据、监控指标。
     </li>
     <li>
      高效存储和查询时间相关的数据。
     </li>
     <li>
      支持数据压缩和聚合。
     </li>
    </ul>
    <h5>
     使用场景：
    </h5>
    <ul>
     <li>
      物联网（IoT）设备监控。
     </li>
     <li>
      系统性能指标分析。
     </li>
     <li>
      股票市场数据存储和分析。
     </li>
    </ul>
    <h4>
     <strong>
      5. 数据仓库
     </strong>
    </h4>
    <h5>
     主流数据库：
    </h5>
    <ul>
     <li>
      <strong>
       Snowflake
      </strong>
      : 云原生数据仓库，扩展性强。
     </li>
     <li>
      <strong>
       BigQuery
      </strong>
      : Google 提供的服务器无关的数据仓库。
     </li>
     <li>
      <strong>
       Amazon Redshift
      </strong>
      : AWS 的数据仓库服务。
     </li>
    </ul>
    <h5>
     功能特点：
    </h5>
    <ul>
     <li>
      专注于大规模数据分析。
     </li>
     <li>
      支持复杂查询和多维分析。
     </li>
     <li>
      集成 ETL 工具和 BI 工具。
     </li>
    </ul>
    <h5>
     使用场景：
    </h5>
    <ul>
     <li>
      数据分析和商业智能（BI）。
     </li>
     <li>
      数据湖建设和历史数据存储。
     </li>
     <li>
      实现大规模机器学习模型训练的基础数据支持。
     </li>
    </ul>
    <h4>
     <strong>
      数据库选择和使用建议
     </strong>
    </h4>
    <h5>
     如何选择：
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        结构化 vs 非结构化数据
       </strong>
       ：
      </p>
      <ul>
       <li>
        如果是结构化数据，使用关系型数据库。
       </li>
       <li>
        如果是非结构化或半结构化数据，优选 NoSQL。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        实时 vs 离线分析
       </strong>
       ：
      </p>
      <ul>
       <li>
        实时处理（如缓存、推荐）可选 Redis。
       </li>
       <li>
        离线分析（如大数据处理）选用数据仓库或 Hadoop 生态。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        特定需求
       </strong>
       ：
      </p>
      <ul>
       <li>
        图数据库用于关系分析。
       </li>
       <li>
        时序数据库用于监控和日志存储。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     如何使用：
    </h5>
    <ul>
     <li>
      <strong>
       安装与部署
      </strong>
      ：
      <ul>
       <li>
        大多数数据库提供云托管版本（如 MongoDB Atlas、AWS RDS），也支持本地安装。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       集成到 AI 项目
      </strong>
      ：
      <ul>
       <li>
        通过编程语言的客户端库（如
        <code>
         pymongo
        </code>
        、
        <code>
         redis-py
        </code>
        ）访问和操作数据库。
       </li>
       <li>
        使用 SQL 或特定查询语言执行查询。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       优化性能
      </strong>
      ：
      <ul>
       <li>
        添加索引、使用缓存、优化查询结构。
       </li>
      </ul>
     </li>
    </ul>

     对比总结表：

      <tr>

       <th>
        <strong>
         适用场景
        </strong>
       </th>
       <th>
        <strong>
         优势
        </strong>
       </th>
       <th>
        <strong>

      <tr>
       <td>

       </td>

       </td>

       </td>

       </td>
       <td>

       </td>

       </td>
       <td>
        高效向量处理和相似度计算
       </td>
       <td>
        对传统数据支持较差
       </td>
      </tr>
      <tr>
       <td>
        分布式系统
       </td>
       <td>
        Spark, Hadoop
       </td>
       <td>

       </td>
       <td>
        可处理大规模数据，强扩展性
       </td>

       </td>
