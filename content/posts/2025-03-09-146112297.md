     1.  Linux编辑器的使用--vim使用
    </h2>
    <p>
     vi/vim的区别简单点来说，它们都是多模式编辑器，不同的是vim是vi的升级版本，它不仅兼容vi的所有指令，而且还有一些新的特性在里面。例如语法加亮，可视化操作不仅可以在终端运行，也可以运行于x window、 mac os、windows。
    </p>
    <p>
    </p>
    <h3 id="1.1%C2%A0%20vim%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%A6%82%E5%BF%B5%C2%A0" name="1.1%C2%A0%20vim%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%A6%82%E5%BF%B5%C2%A0">
     1.1  vim的基本概念
    </h3>
    <p>
     vim其实有好多模式，目前掌握这3种即可),分别是命令模式（command mode）、插入模式（Insert mode）和底行模式（last line mode）。
    </p>
    <p>
     <img alt="" height="750" src="https://i-blog.csdnimg.cn/direct/cafd0ebada2c4af1a32ba476ccb99aa1.png" width="1594"/>
    </p>
    <p>
    </p>
    <p>
     <span style="color:#b95514">
      <strong>
       各模式的功能区分如下：
      </strong>
     </span>
    </p>
    <p>
     <strong>
      1.  正常/普通/命令模式(Normal mode)
     </strong>
     <br/>
     控制屏幕光标的移动，字符、字或行的删除，移动复制某区段及进入Insert mode下，或者到 last line mode
    </p>
    <p>
     <strong>
      2.  插入模式(Insert mode)
     </strong>
     <br/>
     只有在Insert mode下，才可以做文字输入，按「ESC」键可回到命令行模式。该模式是我们后面用的最频繁的编辑模式。
    </p>
    <p>
     <strong>
      3.  末行模式(last line mode)
     </strong>
     <br/>
     文件保存或退出，也可以进行文件替换，找字符串，列出行号等操作。 在命令模式下，shift+: 即可进入该模式。要查看你的所有模式：打开vim，底行模式直接输入。
    </p>
    <p>
    </p>
    <h3 id="1.2%C2%A0%20vim%E5%9F%BA%E6%9C%AC%E6%93%8D%E4%BD%9C%C2%A0" name="1.2%C2%A0%20vim%E5%9F%BA%E6%9C%AC%E6%93%8D%E4%BD%9C%C2%A0">
     1.2  vim基本操作
    </h3>
    <p>
     进入vim,在系统提示符号输入vim及文件名称后，就进入vim全屏幕编辑画面:
    </p>
    <p>
     <img alt="" height="125" src="https://i-blog.csdnimg.cn/direct/bee8178fa10d445aa75b2e042bf585a7.png" width="1168"/>
    </p>
    <p>
     <span style="color:#1c7331">
      <strong>
       [正常模式]切换至[插入模式]
      </strong>
     </span>
     <br/>
     输入a
     <br/>
     入i
     <br/>
     入o
    </p>
    <p>
     <br/>
     <span style="color:#1c7331">
      <strong>
       [插入模式]切换至[正常模式]
      </strong>
     </span>
     <br/>
     目前处于[插入模式]，就只能一直输入文字，如果发现输错了字,想用光标键往回移动，将该字删除，可以先按一下「ESC」键转到[正常模式]再删除文字。当然，也可以直接删除。
    </p>
    <p>
     <br/>
     <span style="color:#1c7331">
      <strong>
       [正常模式]切换至[末行模式]
      </strong>
     </span>
     <br/>
     「shift + ;」, 其实就是输入「:」
     <br/>
     退出vim及保存文件,在[正常模式]下，按一下「:」冒号键进入「Last line mode」,例如:
     <br/>
     : w （保存当前文件）
    </p>
    <p>
     ： wq (输入「wq」,存盘并退出vim)
     <br/>
     ： q! (输入q!,不存盘强制退出vim)
    </p>
    <p>
    </p>

     1.3  vim正常模式（指令模式）命令集
    </h3>
    <p>
     <span style="color:#1c7892">
      <strong>
       插入模式
      </strong>
     </span>
     <br/>
     按「i」切换进入插入模式「insert mode」，按“i”进入插入模式后是从光标当前位置开始输入文件；
     <br/>
     按「a」进入插入模式后，是从目前光标所在位置的下一个位置开始输入文字；
     <br/>
     「o」进入插入模式后，是插入新的一行，从行首开始输入文字。
    </p>
    <p>
     <br/>
     <span style="color:#1c7892">
      <strong>
       从插入模式切换为命令模式
      </strong>
     </span>

     按「ESC」键。

     <br/>

     </span>

     按「G」：移动到文章的最后

     2.3  gcc选择项
    </h3>
    <p>
     <strong>
      -E 只激活预处理,这个不生成文件,你需要把它重定向到一个输出文件里面
      <br/>
      -S 编译到汇编语言不进行汇编和链接
      <br/>
      -c 编译到目标代码
      <br/>
      -o 文件输出到 文件
      <br/>
      -static 此选项对生成的文件采用静态链接
      <br/>
      -g 生成调试信息。GNU 调试器可利用该信息。
      <br/>
      -shared 此选项将尽量使用动态库，所以生成文件比较小，但是需要系统由动态库.
      <br/>
      O0
      <br/>
      -O1
      <br/>
      -O2
      <br/>
      -O3 编译器的优化选项的4个级别，-O0表示没有优化,-O1为缺省值，-O3优化级别最高
      <br/>
      w 不生成任何警告信息。
      <br/>
      -Wall 生成所有警告信息。
     </strong>
    </p>
