    <p>
     在负载均衡（Load Balancing）中，
     <strong>
      四层（L4）负载均衡
     </strong>
     和
     <strong>
      七层（L7）负载均衡
     </strong>
     是两种常见的方式，它们的区别主要体现在工作层级、处理方式、功能特性等方面。
    </p>
    <hr/>
    <h3>
     <a id="1_L4_6">
     </a>
     <strong>
      1. L4（四层）负载均衡
     </strong>
    </h3>
    <p>
     <strong>
      基于 TCP/UDP 传输层的负载均衡
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       协议层
      </strong>
      ：工作在
      <strong>
       OSI 模型的第 4 层（传输层）
      </strong>
      。
     </li>
     <li>
      <strong>
       工作方式
      </strong>
      ：
      <ul>
       <li>
        主要依据
        <strong>
         IP 地址
        </strong>
        和
        <strong>
         端口号
        </strong>
        进行流量分发。
       </li>
       <li>
        不解析 HTTP/HTTPS 请求，不检查 URL、Cookie、Header 等应用层数据。
       </li>
       <li>
        负载均衡器
        <strong>
         仅作为数据包的转发器
        </strong>
        ，不修改内容。
       </li>
       <li>
        适用于
        <strong>
         TCP（如 HTTP、HTTPS）和 UDP（如 DNS、视频流）
        </strong>
        协议的流量分发。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       常见四层负载均衡器
      </strong>
      ：
      <ul>
       <li>
        Nginx（Stream 模块）
       </li>
       <li>
        LVS（Linux Virtual Server）
       </li>
       <li>
        HAProxy（TCP 模式）
       </li>
       <li>
        F5 BIG-IP LTM（L4 模式）
       </li>
      </ul>
     </li>
     <li>
      <strong>
       优点
      </strong>
      ：
      <ul>
       <li>
        性能高，转发数据包时几乎不消耗 CPU。
       </li>
       <li>
        支持
        <strong>
         所有 TCP/UDP 应用
        </strong>
        ，不仅限于 HTTP/HTTPS。
       </li>
       <li>
        适用于高吞吐量和低延迟的应用，如数据库、游戏、流媒体等。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       缺点
      </strong>
      ：
      <ul>
       <li>
        不能识别 HTTP 请求的具体内容，如 URL、Header、Cookie 等。
       </li>
       <li>
        不能进行智能流量调度（如 URL 路由、基于用户身份的分流）。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="L4_Nginx_Stream_27">
     </a>
     <strong>
      示例：L4 负载均衡（Nginx Stream）
     </strong>
    </h4>
    <pre><code class="prism language-nginx">stream {
    upstream backend {
        server 192.168.1.100:3306;
        server 192.168.1.101:3306;
    }
    server {
        listen 3306;
        proxy_pass backend;
    }
}
</code></pre>
    <ul>
     <li>
      这个 Nginx
      <strong>
       L4 负载均衡
      </strong>
      方案用于数据库（MySQL）。
     </li>
     <li>
      监听
      <strong>
       3306 端口
      </strong>
      ，将流量转发到后端数据库服务器。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="2_L7_45">
     </a>
     <strong>
      2. L7（七层）负载均衡
     </strong>
    </h3>
    <p>
     <strong>
      基于 HTTP/HTTPS（应用层）的负载均衡
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       协议层
      </strong>
      ：工作在
      <strong>
       OSI 模型的第 7 层（应用层）
      </strong>
      。
     </li>
     <li>
      <strong>
       工作方式
      </strong>
      ：
      <ul>
       <li>
        能解析
        <strong>
         HTTP 请求
        </strong>
        ，可基于
        <strong>
         URL、Host、Cookie、Header
        </strong>
        等进行流量调度。
       </li>
       <li>
        适用于
        <strong>
         Web 应用
        </strong>
        ，可以进行
        <strong>
         智能路由
        </strong>
        （如
        <code>
         /api
        </code>
        请求转发到 API 服务器）。
       </li>
       <li>
        负载均衡器可以修改 HTTP 请求内容，如增加 Header、URL 重写等。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       常见七层负载均衡器
      </strong>
      ：
      <ul>
       <li>
        Nginx（HTTP 反向代理）
       </li>
       <li>
        HAProxy（HTTP 模式）
       </li>
       <li>
        Traefik
       </li>
       <li>
        F5 BIG-IP LTM（L7 模式）
       </li>
       <li>
        AWS ALB（Application Load Balancer）
       </li>
      </ul>
     </li>
     <li>
      <strong>
       优点
      </strong>
      ：
      <ul>
       <li>
        <strong>
         智能流量调度
        </strong>
        ，可根据 URL、Cookie、Header 进行分流。
       </li>
       <li>
        支持
        <strong>
         A/B 测试、灰度发布、蓝绿部署
        </strong>
        等高级策略。
       </li>
       <li>
        适用于
        <strong>
         Web 应用、微服务架构
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       缺点
      </strong>
      ：
      <ul>
       <li>
        需要解析 HTTP 请求，
        <strong>
         性能开销较大
        </strong>
        。
       </li>
       <li>
        只能处理
        <strong>
         HTTP/HTTPS/WebSocket
        </strong>
        流量，
        <strong>
         不适用于 TCP/UDP 应用
        </strong>
        。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="L7_Nginx_HTTP__66">
     </a>
     <strong>
      示例：L7 负载均衡（Nginx HTTP 反向代理）
     </strong>
    </h4>
    <pre><code class="prism language-nginx">http {
    upstream backend {
        server web1.example.com;
        server web2.example.com;
    }

    server {
        listen 80;

        location /api/ {
            proxy_pass http://backend;
        }
    }
}
</code></pre>
    <ul>
     <li>
      这里的 Nginx
      <strong>
       L7 负载均衡
      </strong>
      用于 HTTP 请求。
     </li>
     <li>
      <code>
       /api/
      </code>
      的请求会被转发到
      <code>
       web1
      </code>
      或
      <code>
       web2
      </code>
      服务器。
     </li>
    </ul>

    <h3>
     <a id="4__L4_L7_101">
     </a>
     <strong>
      4. 什么时候用 L4？什么时候用 L7？
     </strong>
    </h3>
    <p>
     ✅
     <strong>
      L4 适用场景
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       高并发、高吞吐
      </strong>
      ：数据库、流媒体、游戏服务器。
     </li>
     <li>
      <strong>
       不需要解析 HTTP 请求
      </strong>
      ，只进行 TCP/UDP 层转发。
     </li>
     <li>
      <strong>
       内部负载均衡
      </strong>
      ，如数据库集群 MySQL、Redis、PostgreSQL。
     </li>
    </ul>
    <p>
     ✅
     <strong>
      L7 适用场景
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       Web 服务器、API 网关
      </strong>
      （Nginx、Node.js、Spring Boot）。
     </li>
     <li>
      <strong>
       基于 URL、Cookie、Header 进行流量调度
      </strong>
      （如 A/B 测试、灰度发布）。
     </li>
     <li>
      <strong>
       需要 HTTPS 终结（SSL Offloading）
      </strong>
      ，在负载均衡器上处理 TLS 证书。
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_114">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <ul>
     <li>
      <strong>
       L4 负载均衡
      </strong>
      ：基于
      <strong>
       TCP/UDP 连接
      </strong>
      ，速度快，但无法解析 HTTP 请求，适用于
      <strong>
       数据库、流媒体、游戏
      </strong>
      。
     </li>
     <li>
      <strong>
       L7 负载均衡
      </strong>
      ：基于
      <strong>
       HTTP/HTTPS
      </strong>
      ，可以做
      <strong>
       智能路由、灰度发布
      </strong>
      ，适用于
      <strong>
       Web 服务器、API 网关
      </strong>
      。
     </li>
     <li>
      <strong>
       混合使用
      </strong>
      ：有些架构会同时使用
      <strong>
       L4 + L7
      </strong>
      ，比如 L4 负责 TCP 连接调度，L7 负责 HTTP 请求分发。
     </li>
    </ul>
    <p>
     🚀
     <strong>
      如果你要做 Web 服务器或 API 网关，优先使用 L7；如果你要做数据库、游戏等高吞吐场景，优先使用 L4！
     </strong>
     🚀
    </p>
