    <h2 id="%E4%BB%80%E4%B9%88%E6%98%AF%E6%95%B0%E6%8D%AE%E5%BA%93%E8%A7%A6%E5%8F%91%E5%99%A8%EF%BC%9F" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      什么是数据库触发器？
     </span>
    </h2>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      数据库触发器是在数据库中发生特定操作时运行的特殊
     </span>
     存储过程
     <span style="color:#111111;">
      。大多数触发器定义为在对表的数据进行更改时运行。触发器可以定义为替代或在
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      （数据操纵语言）
     </span>
     <span style="color:#111111;">
      -
     </span>
     <span style="color:#111111;">
      如
     </span>
     <span style="color:#111111;">
      INSERT, UPDATE, and DELETE
     </span>
     <span style="color:#111111;">
      -
     </span>
     <span style="color:#111111;">
      之后来运行。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器可帮助数据库设计人员确保完成某些操作（如维护审计文件），而不管哪个程序或用户对数据进行了更改。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      程序被称为触发器，因为事件（例如向表中添加记录）会触发它们的执行。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器及其实现特定于数据库供应商。在本文中，我们将重点介绍
     </span>
     <span style="color:#111111;">
      Microsoft SQL
     </span>
     <span style="color:#111111;">
      服务器
     </span>
     <span style="color:#111111;">
      ;
     </span>
     <span style="color:#111111;">
      但是，
     </span>
     <span style="color:#111111;">
      Oracle
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      MySQL
     </span>
     <span style="color:#111111;">
      中的概念相同或相似。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      注意：本课程的所有示例均基于
     </span>
     <span style="color:#111111;">
      Microsoft SQL Server Management Studio
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      AdventureWorks2012
     </span>
     <span style="color:#111111;">
      数据库。
     </span>
    </p>
    <h2 id="%E4%BA%8B%E4%BB%B6" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      事件
     </span>
    </h2>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器可以在
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      操作之后发生，也可以不发生。触发器与数据库
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      操作
     </span>
     <span style="color:#111111;">
      INSERT
     </span>
     <span style="color:#111111;">
      ，
     </span>
     <span style="color:#111111;">
      UPDATE
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      DELETE
     </span>
     <span style="color:#111111;">
      相关联。触发器定义为在特定表上执行这些操作时运行。
     </span>
    </p>
    <h3 id="AFTER%E8%A7%A6%E5%8F%91%E5%99%A8" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      AFTER
     </span>
     <span style="color:#ff9900;">
      触发器
     </span>
    </h3>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      一旦
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      操作（例如
     </span>
     <span style="color:#111111;">
      INSERT
     </span>
     <span style="color:#111111;">
      ）完成，就会执行
     </span>
     <span style="color:#111111;">
      AFTER
     </span>
     <span style="color:#111111;">
      触发器。以下是
     </span>
     <span style="color:#111111;">
      AFTER
     </span>
     <span style="color:#111111;">
      触发器的一些关键特性：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#111111;">
       在
      </span>
      <span style="color:#111111;">
       DML
      </span>
      <span style="color:#111111;">
       操作之后运行触发器之后，例如
      </span>
      <span style="color:#111111;">
       INSERT
      </span>
      <span style="color:#111111;">
       语句和任何随后的参考级联操作和约束检查已运行。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       您无法使用
      </span>
      <span style="color:#111111;">
       AFTER
      </span>
      <span style="color:#111111;">
       触发器取消数据库操作。这是因为该行动已经完成。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       可以在表上定义每个动作一个或多个
      </span>
      <span style="color:#111111;">
       AFTER
      </span>
      <span style="color:#111111;">
       触发器，但为了简单起见，我建议仅定义一个。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       您无法在视图上定义
      </span>
      <span style="color:#111111;">
       AFTER
      </span>
      <span style="color:#111111;">
       触发器。
      </span>
     </li>
    </ul>
    <h3 id="INSTEAD%20OF%E8%A7%A6%E5%8F%91%E5%99%A8" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      INSTEAD OF
     </span>
     <span style="color:#ff9900;">
      触发器
     </span>
    </h3>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      顾名思义，
     </span>
     <span style="color:#111111;">
      INSTEAD OF
     </span>
     <span style="color:#111111;">
      触发器代替导致它们触发的
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      动作。使用
     </span>
     <span style="color:#111111;">
      INSTEAD OF
     </span>
     <span style="color:#111111;">
      触发器时要考虑的事项包括：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#111111;">
       INSTEAD OF
      </span>
      <span style="color:#111111;">
       触发器会覆盖触发操作。如果定义一个
      </span>
      <span style="color:#111111;">
       INSTEAD OF
      </span>
      <span style="color:#111111;">
       触发器在
      </span>
      <span style="color:#111111;">
       INSERT
      </span>
      <span style="color:#111111;">
       语句上执行，那么一旦
      </span>
      <span style="color:#111111;">
       INSERT
      </span>
      <span style="color:#111111;">
       语句尝试运行，控制就会立即传递给
      </span>
      <span style="color:#111111;">
       INSTEAD OF
      </span>
      <span style="color:#111111;">
       触发器。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       最多可以为表的每个操作定义一个
      </span>
      <span style="color:#111111;">
       INSTEAD OF
      </span>
      <span style="color:#111111;">
       触发器。这是有道理的，就好像你必须为插入一个
      </span>
      <span style="color:#111111;">
       “INSTEAD OF”
      </span>
      <span style="color:#111111;">
       触发器一样，哪一个应该运行？
      </span>
     </li>
    </ul>
    <h3 id="%E7%89%B9%E6%AE%8A%E6%95%B0%E6%8D%AE%E5%BA%93%E5%AF%B9%E8%B1%A1" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      特殊数据库对象
     </span>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器使用两个特殊的数据库对象
     </span>
     <span style="color:#111111;">
      INSERTED
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      DELETED
     </span>
     <span style="color:#111111;">
      来访问受数据库操作影响的行。在触发器的范围内，
     </span>
     <span style="color:#111111;">
      INSERTED
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      DELETE
     </span>
     <span style="color:#111111;">
      对象具有与触发器表相同的列。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      INSERTED
     </span>
     <span style="color:#111111;">
      表包含所有新值
     </span>
     <span style="color:#111111;">
      ;
     </span>
     <span style="color:#111111;">
      而
     </span>
     <span style="color:#111111;">
      DELETED
     </span>
     <span style="color:#111111;">
      表包含旧值。以下是表格的使用方法：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#111111;">
       INSERT
      </span>
      <span style="color:#111111;">
       ——使用
      </span>
      <span style="color:#111111;">
       INSERTED
      </span>
      <span style="color:#111111;">
       表确定将哪些行添加到表中。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       DELETE
      </span>
      <span style="color:#111111;">
       ——使用
      </span>
      <span style="color:#111111;">
       DELETED
      </span>
      <span style="color:#111111;">
       表查看从表中删除的行。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       UPDATE
      </span>
      <span style="color:#111111;">
       ——使用
      </span>
      <span style="color:#111111;">
       INSERTED
      </span>
      <span style="color:#111111;">
       表检查新值或更新值，使用
      </span>
      <span style="color:#111111;">
       DELETED
      </span>
      <span style="color:#111111;">
       表查看更新前的值。
      </span>
     </li>
    </ul>
    <h2 id="%E5%AE%9A%E4%B9%89" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      定义
     </span>
    </h2>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      为特定表和一个或多个事件定义触发器。在大多数数据库管理系统中，每个表只能定义一个触发器。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      以下是
     </span>
     <span style="color:#111111;">
      AdventureWorks2012
     </span>
     <span style="color:#111111;">
      数据库的示例触发器。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <img alt="" class="has" height="1090" src="https://i-blog.csdnimg.cn/blog_migrate/ba16c64673c07f5ef105ac64ddf76921.png" width="792"/>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      您会注意到触发器的语法与存储过程的语法非常相似。事实上，触发器使用相同的语言来实现其逻辑，就像
     </span>
     存储过程一样
     <span style="color:#111111;">
      。在
     </span>
     <span style="color:#111111;">
      MS SQL
     </span>
     <span style="color:#111111;">
      中，这是
     </span>
     <span style="color:#111111;">
      <a href="https://technet.microsoft.com/en-us/library/ms189826%28v=sql.90%29.aspx" rel="nofollow">
       <span style="color:#800080;">
        T-SQL
       </span>
      </a>
      ;
     </span>
     <span style="color:#111111;">
      而在
     </span>
     <span style="color:#111111;">
      Oracle
     </span>
     <span style="color:#111111;">
      中则是
     </span>
     <span style="color:#111111;">
      <a href="https://www.google.com/url?sa=t&amp;rct=j&amp;q=&amp;esrc=s&amp;source=web&amp;cd=2&amp;cad=rja&amp;uact=8&amp;ved=0CCUQFjAB&amp;url=http%3A%2F%2Fwww.oracle.com%2Ftechnetwork%2Fdatabase%2Ffeatures%2Fplsql%2F&amp;ei=WZzDVLGCL4-wyASgr4LAAw&amp;usg=AFQjCNF8JC4VvVtLZzbFspbVJq0lRMf6mw&amp;sig2=ZkWawFM7T0Ws9tXMb8G33g&amp;bvm=bv.84349003,d.aWw" rel="nofollow">
       <span style="color:#800080;">
        PL/SQL
       </span>
      </a>
     </span>
     <span style="color:#111111;">
      。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      以下是触发器的一些重要部分：
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#111111;">
       CREATE
      </span>
      <span style="color:#111111;">
       语句——它定义哪个表与触发器相关联。此外，此语句用于指定触发器何时执行（例如插入后）。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       实际的程序。在该示例中，只要将一行或多行插入
      </span>
      <span style="color:#111111;">
       WorkOrder
      </span>
      <span style="color:#111111;">
       表，该程序就会运行。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       特殊数据库对象——触发器使用特殊定义的数据库对象（如
      </span>
      <span style="color:#111111;">
       INSERTED
      </span>
      <span style="color:#111111;">
       或
      </span>
      <span style="color:#111111;">
       DELETED
      </span>
      <span style="color:#111111;">
       ）来访问受数据库操作影响的记录。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       在此示例中，触发器使用
      </span>
      <span style="color:#111111;">
       INSERTED
      </span>
      <span style="color:#111111;">
       对象来访问新创建的行。
      </span>
      <span style="color:#111111;">
       INSERT
      </span>
      <span style="color:#111111;">
       语句用于表示这些行并将它们添加到历史表中。
      </span>
     </li>
    </ol>
    <h2 id="%E7%94%A8%E4%BA%8E%E8%A7%A6%E5%8F%91%E5%99%A8" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      用于触发器
     </span>
    </h2>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      以下是触发器的一些常见用法：
     </span>
    </p>
    <h3 id="%E5%A4%8D%E6%9D%82%E7%9A%84%E5%AE%A1%E8%AE%A1" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      复杂的审计
     </span>
    </h3>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      您可以使用触发器来跟踪对表所做的更改。在上面的示例中，对
     </span>
     <span style="color:#111111;">
      WorkOrder
     </span>
     <span style="color:#111111;">
      表所做的更改将记录在
     </span>
     <span style="color:#111111;">
      TransactionHistory
     </span>
     <span style="color:#111111;">
      表中。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      通常在创建审计跟踪时，您将使用
     </span>
     <span style="color:#111111;">
      AFTER
     </span>
     <span style="color:#111111;">
      触发器。
     </span>
    </p>
    <p style="text-indent:0;">
     <span style="color:#111111;">
      您可能认为这是多余的，因为数据库日志中记录了许多更改，但日志用于数据库恢复，用户程序无法轻松访问。
     </span>
     <span style="color:#111111;">
      TransactionHistory
     </span>
     <span style="color:#111111;">
      表易于引用，可以合并到最终用户报告中。
     </span>
    </p>
    <h3 id="%E6%89%A7%E8%A1%8C%E4%B8%9A%E5%8A%A1%E8%A7%84%E5%88%99" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      执行业务规则
     </span>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器可用于在执行
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      操作之前检查所有数据。您可以使用
     </span>
     <span style="color:#111111;">
      INSTEAD OF
     </span>
     <span style="color:#111111;">
      触发器
     </span>
     <span style="color:#111111;">
      “
     </span>
     <span style="color:#111111;">
      拦截
     </span>
     <span style="color:#111111;">
      ”
     </span>
     <span style="color:#111111;">
      挂起的
     </span>
     <span style="color:#111111;">
      DML
     </span>
     <span style="color:#111111;">
      操作，应用任何业务规则，并最终完成事务。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      示例业务规则可以是客户状态定义为：
     </span>
    </p>
    <ul>
     <li>
      <span style="color:#111111;">
       黄金——过去
      </span>
      <span style="color:#111111;">
       12
      </span>
      <span style="color:#111111;">
       个月的购买量超过
      </span>
      <span style="color:#111111;">
       1,000,000
      </span>
      <span style="color:#111111;">
       美元。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       白银——在过去
      </span>
      <span style="color:#111111;">
       12
      </span>
      <span style="color:#111111;">
       个月内购买
      </span>
      <span style="color:#111111;">
       500,000
      </span>
      <span style="color:#111111;">
       美元至
      </span>
      <span style="color:#111111;">
       1,000,000
      </span>
      <span style="color:#111111;">
       美元。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       青铜——所有其他购买水平。
      </span>
     </li>
    </ul>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      可以定义
     </span>
     <span style="color:#111111;">
      INSTEAD OF
     </span>
     <span style="color:#111111;">
      触发器，以便在每次添加或修改客户记录时检查客户状态。状态检查将涉及创建所有客户购买的总和，并确保新状态与过去
     </span>
     <span style="color:#111111;">
      12
     </span>
     <span style="color:#111111;">
      个月的购买总和相对应。
     </span>
    </p>
    <h3 id="%E6%B4%BE%E7%94%9F%E5%88%97%E5%80%BC" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      派生列值
     </span>
    </h3>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      触发器可用于计算列值。例如，对于每个客户，您可能希望在客户记录上维护
     </span>
     <span style="color:#111111;">
      TotalSales
     </span>
     <span style="color:#111111;">
      列。当然，为了保持准确，每次销售时都必须更新。
     </span>
    </p>
    <p style="text-indent:33px;">
     <span style="color:#111111;">
      这可以使用
     </span>
     <span style="color:#111111;">
      Sales
     </span>
     <span style="color:#111111;">
      表的
     </span>
     <span style="color:#111111;">
      INSERT
     </span>
     <span style="color:#111111;">
      ，
     </span>
     <span style="color:#111111;">
      UPDATE
     </span>
     <span style="color:#111111;">
      和
     </span>
     <span style="color:#111111;">
      DELETE
     </span>
     <span style="color:#111111;">
      语句上的
     </span>
     <span style="color:#111111;">
      AFTER
     </span>
     <span style="color:#111111;">
      触发器来完成。
     </span>
    </p>
    <h2 id="%E8%A7%A6%E5%8F%91%E5%99%A8%E5%BE%88%E6%A3%98%E6%89%8B%EF%BC%81" style="margin-left:0cm;">
     <span style="color:#ff9900;">
      触发器很棘手！
     </span>
    </h2>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      一般来说，我的建议是避免使用触发器，除非绝对必要。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      您应该避免使用触发器代替内置功能。例如，不是依靠触发器来强制执行引用完整性，而是最好使用
     </span>
     <a href="https://msdn.microsoft.com/en-us/library/ms189049.aspx" rel="nofollow">
      <span style="color:#800080;">
       关系
      </span>
     </a>
     <span style="color:#111111;">
      。
     </span>
    </p>
    <p style="margin-left:0cm;">
     <span style="color:#111111;">
      以下是我回避他们的一些原因：
     </span>
    </p>
    <ol>
     <li>
      <span style="color:#111111;">
       他们可能很难排除故障。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       触发器可以触发其他触发器。两个表
      </span>
      <span style="color:#111111;">
       A
      </span>
      <span style="color:#111111;">
       和
      </span>
      <span style="color:#111111;">
       B
      </span>
      <span style="color:#111111;">
       都有一个
      </span>
      <span style="color:#111111;">
       AFTER UPDATE
      </span>
      <span style="color:#111111;">
       触发器。如果表
      </span>
      <span style="color:#111111;">
       A
      </span>
      <span style="color:#111111;">
       上的
      </span>
      <span style="color:#111111;">
       AFTER UPDATE
      </span>
      <span style="color:#111111;">
       触发器更新表
      </span>
      <span style="color:#111111;">
       B
      </span>
      <span style="color:#111111;">
       ，则更新表
      </span>
      <span style="color:#111111;">
       A
      </span>
      <span style="color:#111111;">
       会导致触发，然后
      </span>
      <span style="color:#111111;">
       B
      </span>
      <span style="color:#111111;">
       触发。
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       你必须确保不会造成触发风暴！你能想象表
      </span>
      <span style="color:#111111;">
       B
      </span>
      <span style="color:#111111;">
       由于某种原因更新了表
      </span>
      <span style="color:#111111;">
       A
      </span>
      <span style="color:#111111;">
       吗？现在你有一个循环参考
      </span>
      <span style="color:#111111;">
       ......
      </span>
      <span style="color:#111111;">
       轰！
      </span>
     </li>
     <li>
      <span style="color:#111111;">
       我尝试将尽可能多的逻辑移动到存储过程中，并让应用程序通过它们而不是直接的
      </span>
      <span style="color:#111111;">
       SQL
      </span>
      <span style="color:#111111;">
       语句来更改数据库。
      </span>
     </li>
    </ol>
