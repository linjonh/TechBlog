    <h3 style="text-align:justify">
     <strong>
      驱动
     </strong>
    </h3>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     驱动，即操作系统用来操作硬件的逻辑代码
    </p>
    <h3 style="text-align:justify">
     <strong>
      Linux下的驱动
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      特点
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     底层驱动要配合应用层才能完成对底层硬件的获取/操作
    </p>
    <ul>
     <li>
      流程
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     向内核插入驱动xxx.ko -&gt; 生成一个设备文件 /dev/xxx -&gt; 执行应用层代码的可执行程序main---&gt;实现对硬件设备的操作
    </p>
    <h3 style="text-align:justify">
     <strong>
      设备文件
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      本质
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     内核驱动需要生成设备文件作为上层应用与底层的桥梁。底层需要一直编写接口，以便于上层的操作。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     在Linux下，一切皆文件。==所有设备在Linux下都是设备文件。
    </p>
    <ul>
     <li>
      特点
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     驱动设备文件是由内核生成的特殊文件，存放在/dev目录下，在Ubuntu下呈黄色字体，对其操作时需要使用非缓冲区操作（open、close、read、write）。每个设备文件都有唯一的设备号（uint32_t的数字），其中高12bit为主设备号，低20bit为次设备号。
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     设备文件分为3类：
    </p>
    <ol>
     <li style="text-align:justify">
      块设备（block）：硬件、内存条、Flash
     </li>
     <li style="text-align:justify">
      网络设备：WiFi、蓝牙、网卡
     </li>
     <li style="text-align:justify">
      字符设备（char）：除以上设备外都属于字符设备。
     </li>
    </ol>
    <blockquote>
     <p style="text-align:justify">
      字符设备有三种开发方法：杂项开发、经典开发、Linux2.6开发
     </p>
    </blockquote>
    <h3 style="text-align:justify">
     <strong>
      杂项驱动设备文件
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      特点
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     杂项类单独占用一个主设备号（10），且最多允许255个设备。在注册时会自动占用空闲设备号。
    </p>
    <ul>
     <li>
      注册杂项设备接口函数
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数原型：
    </p>
    <pre><code class="language-cpp">int misc_register(struct miscdevice *misc);</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数功能：注册一个杂项设备，并生成一个对应的设备文件
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数头文件：#include "linux/miscdevice.h"
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数参数：misc：注册结构体
    </p>
    <pre><code class="language-cpp">struct miscdevice  {
 int minor; //次设备号，填入255可以由系统自动分配次设备号。杂项设备的主设备号为10
 const char *name; //设备文件名
 const struct file_operations *fops; //当前设备文件在底层的接口函数结构体
 struct list_head list; //余下参数可以不填写
 struct device *parent;
 struct device *this_device;
 const struct attribute_group **groups;
 const char *nodename;
 umode_t mode;
};</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     fops参数中至少需要填入三个参数：
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     owner ：填入THIS_MODULE
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     open  ：int (*open) (struct inode *, struct file *);// 自主编写本设备文件的打开操作函数
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     release： int (*release) (struct inode *, struct file *);//自主编写本设备文件的关闭操作函数
    </p>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数返回值：成功返回0，失败返回负数。
    </p>
    <ul>
     <li>
      注销杂项设备接口函数
     </li>
    </ul>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数原型：
    </p>
    <pre><code class="language-cpp">void misc_deregister(struct miscdevice *misc);</code></pre>
    <p style="margin-left:.0001pt; margin-right:0; text-align:justify">
     函数参数：已经注册的设备文件结构体。
    </p>
    <h3 style="text-align:justify">
     <strong>
      杂项开发驱动蜂鸣器与led灯
     </strong>
    </h3>
    <ul>
     <li style="text-align:justify">
      驱动开发程序
     </li>
    </ul>
    <pre><code class="language-cpp">#include "linux/module.h"
#include "linux/kernel.h"
#include "linux/miscdevice.h"
#include "linux/gpio.h"
struct miscdevice mymisc0, mymisc1, mymisc2;
struct file_operations ops0, ops1, ops2;
//我自己在内核层写了一个 open
/**
 * 当我的上层的生成的设备文件-&gt;/dev/test06beep被上层人员打开了 open
 * 内核层就会调用该函数一次！
 */
int my_beep_open(struct inode * i,struct file *f_delown)
{
    printk("蜂鸣器被打开\r\n");
    gpio_set_value(36,1);
    return 0;
}
int my_beep_close(struct inode * i,struct file * f_delown){
    printk("蜂鸣器被关闭\r\n");
    gpio_set_value(36,0);
    return 0;
}
//加载函数-&gt;驱动入口
static int __init test_init(void)
{
    printk("hello  入口函数！\r\n");
    gpio_request(36,"mybeep");
    gpio_direction_output(36,0);
    ops0.owner = THIS_MODULE;
    ops0.open = my_beep_open;
    ops0.release = my_beep_close;
    mymisc0.minor = 255;
    mymisc0.name  = "test06beep";
    mymisc0.fops = &amp;ops0;
    misc_register(&amp;mymisc0);
    return 0;
}         
//卸载函数-&gt;驱动的出口
static  void __exit test_exit(void)
{
    printk("hello  出口函数！\r\n");
    gpio_free(36);
    misc_deregister(&amp;mymisc0);
}
//内核层的声明
module_init(test_init);
module_exit(test_exit);
MODULE_LICENSE("GPL");//必须遵循开源协议</code></pre>

    <p>
     这里的所有设备文件都单独驱动，不建议一个驱动里写好多个设备。
    </p>
    <ul>
     <li style="margin-left: 0.0001pt; margin-right: 0px; text-align: justify;">
      应用层程序
     </li>
    </ul>
    <pre><code class="language-cpp">#include "stdio.h"
#include "unistd.h"
#include "fcntl.h"
#include "sys/types.h"
#include "sys/stat.h"

    char buf[10]={0};
    int fd0 =0 ,fd1 = 0,fd2 = 0;
    while(1){
        fd0 = open("/dev/test06beep",O_RDONLY);
        sleep(1);
        close(fd0);
        fd1 = open("/dev/test06led1",O_RDONLY);
        sleep(1);
        close(fd1);
