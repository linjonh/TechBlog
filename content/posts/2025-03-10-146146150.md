    <h3>
     <a id="SVN__1">
     </a>
     <strong>
      一、SVN 的优缺点
     </strong>
    </h3>
    <h4>
     <a id="_2">
     </a>
     <strong>
      优点
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        集中式管理
       </strong>
      </p>
      <ul>
       <li>
        所有代码和历史记录存储在中央服务器，权限控制严格，适合企业级项目管理。
       </li>
       <li>
        简单直观：用户只需要关注当前工作目录，无需处理复杂的本地仓库。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        目录版本控制
       </strong>
      </p>
      <ul>
       <li>
        支持对文件和目录的版本管理（包括重命名、移动等操作）。
       </li>
       <li>
        可以单独对目录进行版本控制。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        原子性提交
       </strong>
      </p>
      <ul>
       <li>
        提交操作要么完全成功，要么完全失败，避免部分提交导致的代码不一致。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        适合二进制文件
       </strong>
      </p>
      <ul>
       <li>
        对二进制文件（如图片、视频）的版本管理支持较好（Git 对大文件支持较弱）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        严格的权限控制
       </strong>
      </p>
      <ul>
       <li>
        通过
        <code>
         svnserve
        </code>
        或 Apache 插件，可实现目录级的读写权限控制。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_22">
     </a>
     <strong>
      缺点
     </strong>
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        依赖中央服务器
       </strong>
      </p>
      <ul>
       <li>
        必须联网才能提交代码或查看历史记录（无法离线工作）。
       </li>
       <li>
        服务器宕机时，团队协作会中断。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        分支和合并效率低
       </strong>
      </p>
      <ul>
       <li>
        分支是目录的完整拷贝，占用存储空间。
       </li>
       <li>
        合并分支时需要手动跟踪变更历史，容易出错（早期版本）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        版本号全局递增
       </strong>
      </p>
      <ul>
       <li>
        版本号是全局的整数（如
        <code>
         r123
        </code>
        ），而非 Git 的哈希值，可能引发冲突。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        历史追溯不够灵活
       </strong>
      </p>
      <ul>
       <li>
        查看历史需要连接服务器，且操作速度较慢。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="SVN__39">
     </a>
     <strong>
      二、SVN 基础用法示例
     </strong>
    </h3>
    <h4>
     <a id="1__40">
     </a>
     <strong>
      1. 安装与配置
     </strong>
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        安装 SVN
       </strong>
       <br/>
       下载地址：https://subversion.apache.org/
      </p>
      <ul>
       <li>
        Windows 推荐使用
        <strong>
         TortoiseSVN
        </strong>
        （图形化工具）。
       </li>
       <li>
        Linux/macOS 使用包管理器：
        <pre><code class="prism language-bash"><span class="token comment"># Ubuntu/Debian</span>
<span class="token function">sudo</span> <span class="token function">apt-get</span> <span class="token function">install</span> subversion

<span class="token comment"># macOS</span>
brew <span class="token function">install</span> subversion
</code></pre>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        创建仓库（服务端）
       </strong>
      </p>
      <pre><code class="prism language-bash">svnadmin create /path/to/repo  <span class="token comment"># 创建一个新的 SVN 仓库</span>
</code></pre>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2__60">
     </a>
     <strong>
      2. 基础操作
     </strong>
    </h4>
    <h5>
     <a id="1_Checkout_61">
     </a>
     <strong>
      (1) 检出代码（Checkout）
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn checkout http://svn.example.com/repo/trunk  <span class="token comment"># 检出仓库的 trunk 目录</span>
<span class="token comment"># 简写为 svn co</span>
</code></pre>
    <h5>
     <a id="2__67">
     </a>
     <strong>
      (2) 添加文件并提交
     </strong>
    </h5>
    <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token string">"Hello SVN"</span> <span class="token operator">&gt;</span> file.txt
svn <span class="token function">add</span> file.txt          <span class="token comment"># 添加文件到版本控制</span>
svn commit <span class="token parameter variable">-m</span> <span class="token string">"Add file.txt"</span>  <span class="token comment"># 提交到服务器</span>
</code></pre>
    <h5>
     <a id="3_Update_74">
     </a>
     <strong>
      (3) 更新代码（Update）
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn update   <span class="token comment"># 拉取服务器最新代码（简写为 svn up）</span>
</code></pre>
    <h5>
     <a id="4__79">
     </a>
     <strong>
      (4) 查看状态与历史
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn status    <span class="token comment"># 查看工作区状态（简写为 svn st）</span>
svn log       <span class="token comment"># 查看提交历史</span>
svn <span class="token function">diff</span>      <span class="token comment"># 查看当前修改的差异</span>
</code></pre>
    <hr/>
    <h4>
     <a id="3__88">
     </a>
     <strong>
      3. 分支与合并
     </strong>
    </h4>
    <h5>
     <a id="1__89">
     </a>
     <strong>
      (1) 创建分支
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn copy http://svn.example.com/repo/trunk <span class="token punctuation">\</span>
         http://svn.example.com/repo/branches/feature-login <span class="token punctuation">\</span>
         <span class="token parameter variable">-m</span> <span class="token string">"Create branch for login feature"</span>
</code></pre>
    <h5>
     <a id="2__96">
     </a>
     <strong>
      (2) 切换分支
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn switch http://svn.example.com/repo/branches/feature-login  <span class="token comment"># 切换到分支</span>
</code></pre>
    <h5>
     <a id="3__101">
     </a>
     <strong>
      (3) 合并分支
     </strong>
    </h5>
    <pre><code class="prism language-bash">svn merge http://svn.example.com/repo/branches/feature-login  <span class="token comment"># 将分支合并到当前目录</span>
svn commit <span class="token parameter variable">-m</span> <span class="token string">"Merge feature-login to trunk"</span>
</code></pre>
    <hr/>
    <h4>
     <a id="4__109">
     </a>
     <strong>
      4. 解决冲突
     </strong>
    </h4>
    <p>
     当多人修改同一文件时可能发生冲突：
    </p>
    <ol>
     <li>
      执行
      <code>
       svn update
      </code>
      后提示冲突。
     </li>
     <li>
      手动编辑文件，解决冲突标记（
      <code>
       &lt;&lt;&lt;&lt;&lt;&lt;&lt;
      </code>
      ,
      <code>
       =======
      </code>
      ,
      <code>
       &gt;&gt;&gt;&gt;&gt;&gt;&gt;
      </code>
      ）。
     </li>
     <li>
      标记冲突已解决：
      <pre><code class="prism language-bash">svn resolve <span class="token parameter variable">--accept</span> working file.txt
</code></pre>
     </li>
     <li>
      提交修改：
      <pre><code class="prism language-bash">svn commit <span class="token parameter variable">-m</span> <span class="token string">"Resolve conflict in file.txt"</span>
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="5_Tag_124">
     </a>
     <strong>
      5. 标签（Tag）
     </strong>
    </h4>
    <p>
     标签通常用于标记发布版本：
    </p>
    <pre><code class="prism language-bash">svn copy http://svn.example.com/repo/trunk <span class="token punctuation">\</span>
         http://svn.example.com/repo/tags/v1.0.0 <span class="token punctuation">\</span>
         <span class="token parameter variable">-m</span> <span class="token string">"Tag version 1.0.0"</span>
</code></pre>
    <hr/>
    <h3>
     <a id="SVN__134">
     </a>
     <strong>
      三、SVN 目录结构规范
     </strong>
    </h3>
    <p>
     SVN 仓库通常按以下目录组织：
    </p>
    <ul>
     <li>
      <strong>
       <code>
        trunk
       </code>
      </strong>
      ：主开发分支，存放当前稳定代码。
     </li>
     <li>
      <strong>
       <code>
        branches
       </code>
      </strong>
      ：功能分支或修复分支。
     </li>
     <li>
      <strong>
       <code>
        tags
       </code>
      </strong>
      ：发布版本的快照（只读）。
     </li>
    </ul>
    <p>
     示例仓库结构：
    </p>
    <pre><code>repo/
├── trunk/
├── branches/
│   ├── feature-login/
│   └── hotfix-2023/
└── tags/
    ├── v1.0.0/
    └── v1.1.0/
</code></pre>
    <hr/>
    <h3>
     <a id="SVN__154">
     </a>
     <strong>
      四、SVN 最佳实践
     </strong>
    </h3>
    <ol>
     <li>
      <p>
       <strong>
        频繁更新与提交
       </strong>
      </p>
      <ul>
       <li>
        提交前先执行
        <code>
         svn update
        </code>
        ，避免覆盖他人修改。
       </li>
       <li>
        提交时写明清晰的日志（如 “Fix: resolve login page CSS issue”）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        合理使用分支
       </strong>
      </p>
      <ul>
       <li>
        长期功能开发使用分支，避免直接修改
        <code>
         trunk
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        避免提交临时文件
       </strong>
      </p>
      <ul>
       <li>
        通过
        <code>
         svn:ignore
        </code>
        属性忽略临时文件（如
        <code>
         .class
        </code>
        ,
        <code>
         .log
        </code>
        ）。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        定期备份仓库
       </strong>
      </p>
      <ul>
       <li>
        集中式架构的服务器需定期备份，防止数据丢失。
       </li>
      </ul>
     </li>
    </ol>

    <h3>
     <a id="_182">
     </a>
     <strong>
      六、适用场景
     </strong>
    </h3>
    <ul>
     <li>
      <p>
       <strong>
        选择 SVN
       </strong>
       ：
      </p>
      <ul>
       <li>
        企业级项目，需严格权限控制。
       </li>
       <li>
        项目以二进制文件为主（如游戏资源、设计稿）。
       </li>
       <li>
        团队习惯集中式管理。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        选择 Git
       </strong>
       ：
      </p>
      <ul>
       <li>
        开源项目或分布式团队协作。
       </li>
       <li>
        需要频繁分支和合并的场景。
       </li>
       <li>
        开发者需离线工作。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <h3>
     <a id="_195">
     </a>
     <strong>
      总结
     </strong>
    </h3>
    <p>
     SVN 作为集中式版本控制的代表，适合对权限管理和目录结构要求严格的场景，但分支和合并效率较低。建议根据团队需求和项目特点选择工具。如果想深入学习现代协作流程，Git 仍是更主流的选择！
    </p>
