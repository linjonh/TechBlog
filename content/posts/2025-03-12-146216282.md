    <h2>
     <a id="3_35">
     </a>
     3.尝试理解示例代码
    </h2>
    <ul>
     <li>
      示例代码和预览效果：
      <br/>
      <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/b71996a37f394739821d8d56740657ee.png">
       <br/>
       -下面我们对这段 SwiftUI 代码做一个非常详细、底层语法级别的解释，逐行剖析每个语法结构和细节。
      </img>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      模块导入
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token keyword">import</span> <span class="token class-name">SwiftUI</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
       <br/>
       使用
       <code>
        import SwiftUI
       </code>
       导入了 SwiftUI 框架。SwiftUI 是苹果提供的声明式 UI 框架，包含了大量用于构建用户界面的类型和工具。
      </p>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         import
        </code>
        是 Swift 语言的关键字，用于引入模块（module），让代码能够使用该模块中定义的所有公共接口和类型。
       </li>
       <li>
        模块名称
        <code>
         SwiftUI
        </code>
        指定了需要加载的库，这样可以使用 SwiftUI 提供的视图、布局、控件和其他相关功能。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      定义视图结构体
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token keyword">struct</span> <span class="token class-name">ContentView</span><span class="token punctuation">:</span> <span class="token class-name">View</span> <span class="token punctuation">{<!-- --></span>
    <span class="token operator">...</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
       <br/>
       定义了一个名为
       <code>
        ContentView
       </code>
       的结构体，并声明其遵循
       <code>
        View
       </code>
       协议，从而成为一个可以被 SwiftUI 渲染的视图。
      </p>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         struct
        </code>
        关键字定义了一个结构体（值类型），在 Swift 中结构体是一种常用的自定义数据类型。
       </li>
       <li>
        <code>
         ContentView
        </code>
        是结构体的名称。命名时遵循大驼峰命名法。
       </li>
       <li>
        冒号
        <code>
         :
        </code>
        后面跟着的是一个协议名称
        <code>
         View
        </code>
        。这意味着
        <code>
         ContentView
        </code>
        必须满足
        <code>
         View
        </code>
        协议的所有要求。
       </li>
       <li>
        Swift 中的协议（protocol）定义了一系列方法、属性或其他要求，
        <code>
         View
        </code>
        协议规定了一个视图必须实现的接口，最主要的是必须提供一个
        <code>
         body
        </code>
        属性。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      实现视图的主体
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token keyword">var</span> body<span class="token punctuation">:</span> <span class="token keyword">some</span> <span class="token class-name">View</span> <span class="token punctuation">{<!-- --></span>
    <span class="token class-name">VStack</span> <span class="token punctuation">{<!-- --></span>
        <span class="token operator">...</span>
    <span class="token punctuation">}</span>
    <span class="token punctuation">.</span><span class="token function">padding</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        定义了一个名为
        <code>
         body
        </code>
        的计算属性，它必须返回符合
        <code>
         View
        </code>
        协议的内容。SwiftUI 使用该属性来确定如何绘制这个视图的界面。
       </li>
       <li>
        使用了 Swift 的
        <strong>
         不透明返回类型
        </strong>
        （
        <code>
         some View
        </code>
        ），表示返回值符合
        <code>
         View
        </code>
        协议，但具体类型对调用者是隐藏的。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         var
        </code>
        声明了一个变量属性。由于
        <code>
         body
        </code>
        是只读计算属性（没有
        <code>
         set
        </code>
        方法），它实际上表现得像常量。
       </li>
       <li>
        <code>
         body
        </code>
        的类型被声明为
        <code>
         some View
        </code>
        ，其中
        <code>
         some
        </code>
        关键字利用了 Swift 的
        <strong>
         不透明返回类型
        </strong>
        特性，允许隐藏具体的视图类型细节，只需要保证返回的内容满足
        <code>
         View
        </code>
        协议即可。
       </li>
       <li>
        大括号
        <code>
         { ... }
        </code>
        内部定义了该属性的计算逻辑，也就是返回值。Swift 中的计算属性不需要显式返回语句（如果只有一行表达式，可以省略
        <code>
         return
        </code>
        ）。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      垂直堆叠视图（VStack）
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token class-name">VStack</span> <span class="token punctuation">{<!-- --></span>
    <span class="token class-name">Image</span><span class="token punctuation">(</span>systemName<span class="token punctuation">:</span> <span class="token string-literal"><span class="token string">"globe"</span></span><span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">imageScale</span><span class="token punctuation">(</span><span class="token punctuation">.</span>large<span class="token punctuation">)</span>
        <span class="token punctuation">.</span><span class="token function">foregroundStyle</span><span class="token punctuation">(</span><span class="token punctuation">.</span>tint<span class="token punctuation">)</span>
    <span class="token class-name">Text</span><span class="token punctuation">(</span><span class="token string-literal"><span class="token string">"Hello, world!"</span></span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         VStack
        </code>
        是一种容器视图，用于将其子视图垂直排列。
       </li>
       <li>
        该容器中包含两个子视图：一个图像和一个文本。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         VStack { ... }
        </code>
        使用了
        <strong>
         尾随闭包
        </strong>
        语法，这在 Swift 中非常常见，用于将一段代码块作为函数的参数传递。
       </li>
       <li>
        括号内的代码块定义了
        <code>
         VStack
        </code>
        的子视图。SwiftUI 内部使用
        <strong>
         视图构建器
        </strong>
        （ViewBuilder）机制，将闭包中的多个表达式组合成一个复合视图。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      Image 视图
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token class-name">Image</span><span class="token punctuation">(</span>systemName<span class="token punctuation">:</span> <span class="token string-literal"><span class="token string">"globe"</span></span><span class="token punctuation">)</span>
    <span class="token punctuation">.</span><span class="token function">imageScale</span><span class="token punctuation">(</span><span class="token punctuation">.</span>large<span class="token punctuation">)</span>
    <span class="token punctuation">.</span><span class="token function">foregroundStyle</span><span class="token punctuation">(</span><span class="token punctuation">.</span>tint<span class="token punctuation">)</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        使用
        <code>
         Image(systemName: "globe")
        </code>
        创建了一个系统符号图标，这里使用的是 SF Symbols 中的
        <code>
         "globe"
        </code>
        图标。
       </li>
       <li>
        随后通过链式调用了两个视图修饰符（modifiers）：
        <code>
         .imageScale(.large)
        </code>
        和
        <code>
         .foregroundStyle(.tint)
        </code>
        ，分别调整图像的缩放比例和前景样式。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <strong>
         构造函数调用：
        </strong>
        <ul>
         <li>
          <code>
           Image(systemName: "globe")
          </code>
          是对
          <code>
           Image
          </code>
          结构体的初始化调用，传入的参数
          <code>
           systemName
          </code>
          是一个标签，用于查找对应的系统图标。
         </li>
        </ul>
       </li>
       <li>
        <strong>
         链式调用（Method Chaining）：
        </strong>
        <ul>
         <li>
          <code>
           .imageScale(.large)
          </code>
          调用返回一个经过修改的
          <code>
           Image
          </code>
          实例。
          <code>
           .large
          </code>
          是一个枚举值，用于指定图像的显示大小。
         </li>
         <li>
          <code>
           .foregroundStyle(.tint)
          </code>
          应用前景颜色样式，这里使用
          <code>
           .tint
          </code>
          表示使用当前环境中的主色调或强调色。
         </li>
        </ul>
       </li>
       <li>
        每个修饰符都是 SwiftUI 提供的函数，这些函数通常会返回一个新的视图实例，而不改变原有视图。这种设计符合 SwiftUI 的
        <strong>
         声明式编程
        </strong>
        和
        <strong>
         不可变性
        </strong>
        原则。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      Text 视图
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token class-name">Text</span><span class="token punctuation">(</span><span class="token string-literal"><span class="token string">"Hello, world!"</span></span><span class="token punctuation">)</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        创建一个文本视图，显示字符串
        <code>
         "Hello, world!"
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         Text
        </code>
        是 SwiftUI 中的一个结构体，用于在界面上渲染文本内容。
       </li>
       <li>
        构造函数直接传入一个字符串字面量。Swift 中的字符串是
        <code>
         String
        </code>
        类型，支持多种文本处理功能。
       </li>
       <li>
        这里没有额外的修饰符，意味着文本使用默认样式显示。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      应用修饰符到 VStack
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token punctuation">.</span><span class="token function">padding</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         .padding()
        </code>
        是一个视图修饰符，作用是在
        <code>
         VStack
        </code>
        外围添加默认的内边距，使其与其它视图之间保持一定的间隔。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        这也是一个链式调用。修饰符会返回一个新的视图实例，其中包含了原始
        <code>
         VStack
        </code>
        的布局信息及额外的内边距效果。
       </li>
       <li>
        SwiftUI 的修饰符设计是函数式的，每个调用都是不可变操作，生成一个全新的视图描述。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      预览代码
     </li>
    </ul>
    <pre><code class="prism language-swift"><span class="token other-directive property">#Preview</span> <span class="token punctuation">{<!-- --></span>
    <span class="token class-name">ContentView</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <ul>
     <li>
      <p>
       <strong>
        功能说明：
       </strong>
      </p>
      <ul>
       <li>
        这段代码使用 SwiftUI 的预览功能，在 Xcode 中可以实时预览
        <code>
         ContentView
        </code>
        的界面效果。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        底层语法细节：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         #Preview
        </code>
        是一个特殊的编译器指令或宏（具体细节可能会随 Xcode 版本变化），用于标记预览内容。
       </li>
       <li>
        花括号
        <code>
         { ... }
        </code>
        内部包含一个表达式
        <code>
         ContentView()
        </code>
        ，这实际上是调用
        <code>
         ContentView
        </code>
        结构体的默认初始化器生成一个视图实例。
       </li>
       <li>
        与传统的
        <code>
         PreviewProvider
        </code>
        协议实现方式相比，这种写法更加简洁，符合 SwiftUI 追求声明式和直观的设计风格。
       </li>
      </ul>
     </li>
    </ul>
    <hr/>
    <ul>
     <li>
      <p>
       代码总结
      </p>
     </li>
     <li>
      <p>
       <strong>
        模块导入与结构体定义：
       </strong>
       <br/>
       使用
       <code>
        import
       </code>
       导入 SwiftUI 模块，并定义了一个遵循
       <code>
        View
       </code>
       协议的结构体
       <code>
        ContentView
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        计算属性和不透明返回类型：
       </strong>
       <br/>
       <code>
        body
       </code>
       属性使用
       <code>
        some View
       </code>
       作为返回类型，隐藏了具体的实现类型，但确保返回内容符合
       <code>
        View
       </code>
       协议。
      </p>
     </li>
     <li>
      <p>
       <strong>
        视图构建与尾随闭包：
       </strong>
       <br/>
       使用
       <code>
        VStack
       </code>
       将
       <code>
        Image
       </code>
       和
       <code>
        Text
       </code>
       视图垂直排列，通过尾随闭包语法简化代码结构。
      </p>
     </li>
     <li>
      <p>
       <strong>
        链式调用的修饰符：
       </strong>
       <br/>
       图像和容器均通过链式调用应用了各自的修饰符，这种方式符合 SwiftUI 的函数式和不可变性设计。
      </p>
     </li>
     <li>
      <p>
       <strong>
        预览机制：
       </strong>
       <br/>
       最后的
       <code>
        #Preview
       </code>
       代码块为开发者提供了实时预览界面的能力，便于调试和视觉调整。
      </p>
     </li>
    </ul>

    </ul>
