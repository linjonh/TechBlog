      <h2>
       <span style="font-size:18px">
        <span style="color:black">
         黑盒测试用例设计方法包括等价类划分法、边界值分析法、错误推测法、因果图法、判定表驱动法、正交试验设计法、功能图法等

      <div>
       <div>
        <div id="cnblogs_post_body">
         <p>
          <span style="color:black">
           等价类划分法是把程序的输入域划分成若干部分（子集），然后从每个部分中选取少数代表性数据作为测试用例。每一类的代表性数据在测试中的作用等价于这一类中的其他值。
          </span>
         </p>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          1.
          <span style="color:black">
           等价类是指某个输入域的子集合。在该子集合中，各个输入数据对于揭露程序中的错误都是等效的，并合理地假定：测试某等价类的代表值就等于对这一类其它值的测试
          </span>
          <span style="color:black">
           .因此
          </span>
          ,可以把全部输入数据合理划分为若干等价类,在每一个等价类中取一个数据作为测试的输入条件,就可以用少量代表性的测试数据.取得较好的测试结果.等价类划分可有两种不同的情况:有效等价类和无效等价类。
         </div>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           有效等价类
          </span>
          <span style="color:black">
           :是指对于程序的规格说明来说是合理的
          </span>
          ,有意义的输入数据构成的集合.利用有效等价类可检验程序是否实现了规格说明中所规定的功能和性能。
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           无效等价类
          </span>
          <span style="color:black">
           :与有效等价类的定义恰巧相
          </span>
          <span style="color:black">
           反。
          </span>
         </p>
         <p>
          <span style="color:black">
           设计测试用例时
          </span>
          <span style="color:black">
           ,要同时考虑这两种等价类
          </span>
          .因为,软件不仅要能接收合理的数据,也要能经受意外的考验.这样的测试才能确保软件具有更高的可靠
          <span style="color:black">
           性。
          </span>
         </p>
         <p>
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           划分等价类的六大原则：
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在输入条件规定了取值范围或值的个数的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           则可以确立一个有效等价类和两个无效等价类
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p>
          <span style="color:black">
           例：输入值是学生成绩，范围是
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           ～
          </span>
          <span style="color:black">
           100
          </span>
          <span style="color:black">
           ：
          </span>
         </p>
         <p style="background:white">
          <img alt="" border="0" height="122" src="https://i-blog.csdnimg.cn/blog_migrate/41f950f753f863c8ee185348c1233cfa.gif" width="555"/>
         </p>
         <p>
          <span style="color:black">
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在输入条件规定了输入值的集合或者规定了
          </span>
          <span style="color:black">
           “
          </span>
          <span style="color:black">
           必须如何
          </span>
          <span style="color:black">
           ”
          </span>
          <span style="color:black">
           的条件的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           可确立一个有效等价类和一个无效等价类
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在输入条件是一个布尔量的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           可确定一个有效等价类和一个无效等价类
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:#cc0033">
           布尔量
          </span>
          是一个二值枚举类型, 一个
          <span style="color:#cc0033">
           布尔量
          </span>
          具有两种状态: true 和 false 。
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在规定了输入数据的一组值（假定
          </span>
          <span style="color:black">
           n
          </span>
          <span style="color:black">
           个）
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           并且程序要对每一个输入值分别处理的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           可确立
          </span>
          <span style="color:black">
           n
          </span>
          <span style="color:black">
           个有效等价类和一个无效等价类
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p>
          <span style="color:black">
           例：输入条件说明
          </span>
          <span style="color:black">
           输入字符为
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           中文
          </span>
          <span style="color:black">
           、
          </span>
          <span style="color:black">
           英文
          </span>
          <span style="color:black">
           、
          </span>
          <span style="color:black">
           阿拉伯文三
          </span>
          <span style="color:black">
           种之一，则分别取这
          </span>
          <span style="color:black">
           三
          </span>
          <span style="color:black">
           种这
          </span>
          <span style="color:black">
           三
          </span>
          <span style="color:black">
           个值作为
          </span>
          <span style="color:black">
           三
          </span>
          <span style="color:black">
           个有效等价类，另外把
          </span>
          <span style="color:black">
           三
          </span>
          <span style="color:black">

           之外的任何
          </span>
          <span style="color:black">
           字符
          </span>
          <span style="color:black">
           作为无效等价类。
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在规定了输入数据必须遵守的规则的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           可确立一个有效等价类（符合规则）和若干个无效等价类（从不同角度违反规则）
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           在确知已划分的等价类中各元素在程序处理中的方式不同的情况下
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           则应再将该等价类进一步的划分为更小的等价类
          </span>
         </p>
         <p>
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           将等价类转化成测试用例：
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           按照
          </span>
          <span style="color:black">
           [输入条件
          </span>
          ][有效等价类][无效等价
          <span style="color:black">
           类
          </span>
          <span style="color:black">
           ]
          </span>
          <span style="color:black">
           建立等价类表
          </span>
          ,列出所有划分出的等价
          <span style="color:black">
           类
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           为每一个等价类规定一个唯一的编号
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           设计一个新的测试用例
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           使其尽可能多地覆盖尚未被覆盖地有效等价类
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           重复这一步
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           直到所有的有效等价类都被覆盖为止
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p>
          <span style="color:black">
           ·
          </span>
          <span style="color:black">
           设计一个新的测试用例
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           使其仅覆盖一个尚未被覆盖的无效等价类
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           重复这一步
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           直到所有的无效等价类都被覆盖为止
          </span>
          <span style="color:black">
           .
          </span>
         </p>

            <span style="font-size:14px">

           <strong>
            <span style="font-size:24px">
             3.
            </span>
            <span style="font-size:24px">
             边界值分析法
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260578">
            <span style="font-size:14px">
             3.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           边界值分析法就是对输入或输出的边界值进行测试的一种黑盒测试方法。通常边界值分析法是作为对等价类划分法的补充，这种情况下，其测试用例来自等价类的边界。
          </span>
          <span style="color:black">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260579">
            <span style="font-size:14px">
             3.2.
            </span>
            <span style="font-size:14px">
             边界值分析法的应用
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           根据大量的
          </span>
          <span style="color:black">
           测试
          </span>
          <span style="color:black">
           统计数据
          </span>
          <span style="color:black">
           ，
          </span>
          <span style="color:black">
           很多
          </span>
          <span style="color:black">
           错误是发生在输入或输出范围的边界上，而不是发生在输入
          </span>
          <span style="color:black">
           /
          </span>
          <span style="color:black">
           输出范围的
          </span>
          <span style="color:black">
           中间区域
          </span>
          <span style="color:black">
           。因此针对各种边界情况设计测试用例，可以查出更多的错误。
          </span>
         </p>
         <p>
          <span style="color:black">
           使用边界值分析方法设计测试用例，首先应确定边界情况。通常输入和输出等价类的边界，就是应着重测试的边界情况。应当选取正好等于，刚刚大于或刚刚小于边界的值作为测试数据，而不是选取等价类中的典型值或任意值作为测试数据
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          1.     边界值分析法与等价类分析法的区别：
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           边界值分析不是从某等价类中随便挑一个作为代表，而是使这个等
          </span>
          <span style="color:black">
           价类的每个边界都要作为测试条件。
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           边界值分析不仅考虑输入条件，还要考虑输出空间产生的测试情况
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          <span style="color:black">
           例：测试计算平方根的函数
          </span>
         </p>
         <p>
          <span style="color:black">
           --
          </span>
          <span style="color:black">
           输入：实数
          </span>
         </p>
         <p>
          <span style="color:black">
           --
          </span>
          <span style="color:black">
           输出：实数
          </span>
         </p>
         <p>
          <span style="color:black">
           --
          </span>
          <span style="color:black">
           需求说明：当输入一个
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           或比
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           大的数的时候，返回其正平方根；当输入一个小于
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           的数时，显示错误信息
          </span>
          <span style="color:black">
           "
          </span>
          <span style="color:black">
           平方根非法
          </span>
          <span style="color:black">
           -
          </span>
          <span style="color:black">
           输入值小于
          </span>
          <span style="color:black">
           0"
          </span>
          <span style="color:black">
           并返回
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           ；库函数
          </span>
          <span style="color:black">
           Print-Line
          </span>
          <span style="color:black">
           可以用来输出错误信息。
          </span>
         </p>
         <p>
          <span style="color:black">

          </span>
          <span style="color:black">

          </span>
          <span style="color:black">

          </span>

          </span>
         </p>

          <span style="color:black">
           5.
          </span>
          <span style="color:black">
           基于边界值分析方法选择测试用例的原则
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           如果输入条件规定了值的范围
          </span>
          <span style="color:black">
           ,则应取刚达到这个范围的边界的值
          </span>
          ,以及刚刚超越这个范围边界的值作为测试输入数据。
         </p>
         <p>
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           例如，如果程序的规格说明中规定：
          </span>
          <span style="color:black">
           "重量在
          </span>
          10公斤至50公斤范围内的邮件，其邮费计算公式为……"。作为测试用例，我们应取10及50，还应取10.01,49.99,9.99及50.01等。
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           如果输入条件规定了值的个数
          </span>
          <span style="color:black">
           ,则用最大个数
          </span>
          ,最小个数,比最小个数少一,比最大个数多一的数作为测试数据。
         </p>
         <p>
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           例如，一个输入文件应包括
          </span>
          <span style="color:black">
           1~255个记录，则测试用例可取
          </span>
          1和255，还应取0及256等。
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           将规则
          </span>
          <span style="color:black">
           1）和
          </span>
          2）应用于输出条件，即设计测试用例使输出值达到边界值及其左右的值。
         </p>
         <p>
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           例如，某程序的规格说明要求计算出
          </span>
          <span style="color:black">
           "每月保险金扣除额为
          </span>
          0至1165.25元"，其测试用例可取0.00及1165.24、还可取一0.01及1165．26等。
         </p>
         <p>
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           再如一程序属于情报检索系统，要求每次
          </span>
          <span style="color:black">
           "最少显示
          </span>
          1条、最多显示4条情报摘要"，这时我们应考虑的测试用例包括1和4，还应包括0和5等
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           如果程序的规格说明给出的输入域或输出域是有序集合
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           则应选取集合的第一个元素和最后一个元素作为测试用例。
          </span>
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           如果程序中使用了一个内部数据结构
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           则应当选择这个内部数据结构的边界上的值作为测试用例。
          </span>
         </p>
         <p>
          <span style="color:black">
           6)
          </span>
          <span style="color:black">
           分析规格说明
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           找出其它可能的边界条件。
          </span>
         </p>

         <h2>
          <a name="_Toc223260581">
           <strong>
            <span style="font-size:24px">
             4.
            </span>
            <span style="font-size:24px">
             错误推断法
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260582">
            <span style="font-size:14px">
             4.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           基于经验和直觉推测程序中所有可能存在的各种错误, 从而有针对性的设计测试用例的方法。
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260583">
            <span style="font-size:14px">
             4.2.
            </span>
            <span style="font-size:14px">
             错误推断法的应用
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           基本思想：列举出程序中所有可能有的错误和容易发生错误的特殊情况
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           根据他们选择测试用例。
          </span>
         </p>
         <p>
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           例如
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           输入数据和输出数据为
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           的情况；输入表格为空格或输入表格只有一行。
          </span>
          <span style="color:black">
           这些都是容易发生错误的情况。可选择这些情况下的例子作为测试用例。
          </span>
         </p>
         <p>
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           例如，前面例子中成绩报告的程序，采用错误推测法还可补充设计一些测试用例：
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           程序是否把空格作为回答
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           在回答记录中混有标准答案记录
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           除了标题记录外，还有一些的记录最后一个字符即不是
          </span>
          <span style="color:black">
           2
          </span>
          <span style="color:black">
           也不是
          </span>
          <span style="color:black">
           3
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           有两个学生的学号相同
          </span>
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           试题数是负数
          </span>
         </p>
         <p>
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           例如，测试一个对线性表（比如数组）进行排序的程序，可推测列出以下几项需要特别测试的情况：
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           输入的线性表为空表；
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           表中只含有一个元素；
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           输入表中所有元素已排好序；
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           输入表已按逆序排好；
          </span>
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           输入表中部分或全部元素相同。
          </span>
         </p>
         <p>
          <span style="color:black">
           4.
          </span>
          <span style="color:black">
           例如，测试手机终端的通话功能，可以设计各种通话失败的情况来补充测试用例：
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           无
          </span>
          <span style="color:black">
           SIM 卡插入时进行呼出（非紧急呼叫）
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           插入已欠费
          </span>
          <span style="color:black">
           SIM卡进行呼出
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           射频器件损坏或无信号区域插入有效
          </span>
          <span style="color:black">
           SIM卡呼出
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           网络正常，插入有效
          </span>
          <span style="color:black">
           SIM卡，呼出无效号码（如
          </span>
          1、888、333333、不输入任何号码等）
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           网络正常，插入有效
          </span>
          <span style="color:black">
           SIM卡，使用“快速拨号”功能呼出设置无效号码的数字
          </span>
         </p>
         <p>
          <span style="color:black">
          </span>
         </p>
         <p>
          <span style="color:black">
          </span>
         </p>
         <p>
          <span style="color:black">

         <h2>
          <a name="_Toc130718048">
          </a>
          <a name="_Toc223260584">
           <strong>
            <span style="font-size:24px">
             5.
            </span>
            <span style="font-size:24px">
             因果图法
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260585">
            <span style="font-size:14px">
             5.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           因果图法
          </span>
          <span style="color:black">
           是一种利用图解法分析输入的各种组合情况，从而设计测试用例的方法，它适合于检查程序输入条件的各种组合情况。
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260586">
            <span style="font-size:14px">
             5.2.
            </span>
            <span style="font-size:14px">
             因果图法的应用
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p style="background:white">
          <span style="color:black">
           等价类划分法和边界值分析方法都是着重考虑输入条件，但没有考虑输入条件的各种组合、输入条件之间的相互制约关系。这样虽然各种输入条件可能出错的情况已经测试到了，但多个输入条件组合起来可能出错的情况却被忽视了。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           如果在测试时必须考虑输入条件的各种组合，则可能的组合数目将是天文数字，因此必须考虑采用一种适合于描述多种条件的组合、相应产生多个动作的形式来进行测试用例的设计，这就需要利用因果图（逻辑模型）。
          </span>
         </p>
         <p>
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           因果图介绍
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           4种符号分别表示了规格说明中向
          </span>
          4种因果关系。
         </p>
         <p align="center" style="background:white">
          <span style="color:black">
           <img alt="" border="0" height="354" src="https://i-blog.csdnimg.cn/blog_migrate/e1ff73a37f6991b2a59d4dd18ff905cf.gif" width="486"/>
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           因果图中使用了简单的逻辑符号，以直线联接左右结点。左结点表示输入状态（或称原因），右结点表示输出状态（或称结果）。
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           C1
          </span>
          <span style="color:black">
           表示原因，通常置于图的左部；
          </span>
          <span style="color:black">
           e1
          </span>
          <span style="color:black">

          </span>
          <span style="color:black">
           C1
          </span>
          <span style="color:black">
           和
          </span>
          <span style="color:black">
           e1
          </span>
          <span style="color:black">
           均可取值
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           或
          </span>
          <span style="color:black">
           1
          </span>
          <span style="color:black">
           ，
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           表示某状态不出现，
          </span>
          <span style="color:black">
           1
          </span>
          <span style="color:black">
           表示某状态出现。
          </span>
         </p>
         <p>
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           因果图涉及的概念
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           关系
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           恒等：若
          </span>

          </span>

          <span style="color:black">

          </span>
          <span style="color:black">
           ；否则
          </span>
          <span style="color:black">
           e1
          </span>
          <span style="color:black">
           为
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           非：若
          </span>
          <span style="color:black">
           c1
          </span>
          <span style="color:black">
           是
          </span>
          <span style="color:black">
           1
          </span>
          <span style="color:black">
           ，则
          </span>
          <span style="color:black">
           e1
          </span>
          <span style="color:black">
           是
          </span>
          <span style="color:black">
           0
          </span>
          <span style="color:black">
           ；否则
          </span>
          <span style="color:black">
           e1
          </span>
          <span style="color:black">
           是
          </span>
          <span style="color:black">

          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           或：若

            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           判定表是分析和表达多逻辑条件下执行不同操作的情况的工具。
          </span>
         </p>

         <p>
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           判定表的优点
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           能够将复杂的问题按照各种可能的情况全部列举出来，简明并避免遗漏。因此，利用判定表能够设计出完整的测试用例集合。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           在一些数据处理问题当中，某些操作的
          </span>
          <span style="color:black">
           实施依赖于多个逻辑条件的组合，即：针对不同逻辑条件的组合值，分别执行不同的操作。判定表很适合于处理这类问题。

         <p>
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           判定表通常由四个部分组成如下图所示。
          </span>
         </p>
         <p align="center" style="background:white">
          <span style="color:black">
           <img alt="" border="0" height="257" src="https://i-blog.csdnimg.cn/blog_migrate/e13fe94208c2fc1fafa492ec122fddb7.gif" width="507"/>
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           条件桩（
          </span>
          <span style="color:black">
           Condition Stub）：列出了问题得所有条件。通常认为列出的条件的次序无关紧要。
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           动作桩（
          </span>
          <span style="color:black">
           Action Stub）：列出了问题规定可能采取的操作。这些操作的排列顺序没有约束。
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           条件项（
          </span>
          <span style="color:black">
           Condition Entry）：列出针对它左列条件的取值。在所有可能情况下的真假值。
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           动作项（
          </span>
          <span style="color:black">
           Action Entry）：列出在条件项的各种取值情况下应该采取的动作。
          </span>
         </p>
         <p>
          <span style="color:black">
           4.
          </span>
          <span style="color:black">
           规则及规则合并
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           规则：任何一个条件组合的特定取值及其相应要执行的操作称为规则。在判定表中贯穿条件项和动作项的一列就是一条规则。显然
          </span>
          <span style="color:black">
           ,判定表中列出多少组条件取值
          </span>
          ,也就有多少条规则,既条件项和动作项有多少列。
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           化简：就是规则合并有两条或多条规则具有相同的动作，并且其条件项之间存在着极为相似的关系。
          </span>
         </p>
         <p>

          </span>
          <span style="color:black">
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           判定表的优点和缺点
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           优点：它能把复杂的问题按各种可能的情况一一列举出来，简明而易于理解，也可避免遗漏。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           缺点：不能表达重复执行的动作，例如循环结构。
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           B. Beizer 指出了适合使用判定表设计测试用例的条件：
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           规格说明以判定表形式给出
          </span>
          <span style="color:black">
           ,或很容易转换成判定表。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           条件的排列顺序不会也不影响执行哪些操作。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           规则的排列顺序不会也不影响执行哪些操作。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           每当某一规则的条件已经满足
          </span>
          <span style="color:black">
           ,并确定要执行的操作后
          </span>
          ,不必检验别的规则。
         </p>
         <p style="background:white">
          <span style="color:black">
           Ø
          </span>
          <span style="color:black">
           如果某一规则得到满足要执行多个操作
          </span>
          <span style="color:black">
           ,这些操作的执行顺序无关紧要。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           B. Beizer提出这
          </span>
          5个必要条件的目的是为了使操作的执行完全依赖于条件的组合。其实对于某些不满足这几条的判定表，同样可以借以设计测试用例，只不过尚需增加其它的测试用例罢了。
         </p>

          <a name="_Toc223260592">
           <strong>
            <span style="font-size:24px">
             7.
            </span>
            <span style="font-size:24px">
             正交试验法
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260593">
            <span style="font-size:14px">
             7.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           依据
          </span>
          <span style="color:black">
           Galois
          </span>
          <span style="color:black">
           理论
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           从大量的（实验）数据（测试例）中挑选适量的
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           有代表性的点（例）
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           从而合理地安排实验（测试）的一种科学实验设计方法
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           类似的方法有
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           聚类分析方法
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           因子方法方法等
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260594">
            <span style="font-size:14px">
             7.2.
            </span>
            <span style="font-size:14px">
             正交试验法
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           利用因果图来设计测试用例时
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           作为输入条件的原因与输出结果之间的因果关系,有时很难从软件需求规格说明中得到。往往因果关系非常庞大,以至于据此因果图而得到的测试用例数目多的惊人，给软件测试带来沉重的负担，为了有效地,合理地减少测试的工时与费用,可利用正交实验设计方法进行测试用例的设计
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           利用正交实验设计测试用例的步骤：
          </span>
         </p>
         <p>
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           提取功能说明
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           构造因子
          </span>
          <span style="color:black">
           --
          </span>
          <span style="color:black">
           状态表
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           把影响实验指标的条件称为因子
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           而影响实验因子的条件叫因子的状态
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           利用正交实验设计方法来设计测试用例时
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           首先要根据被测试软件的规格说明书找出影响其功能实现的操作对象和外部因素
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           把他们当作因子
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           而把各个因子的取值当作状态
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           对软件需求规格说明中的功能要求进行划分
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           把整体的概要性的功能要求进行层层分解与展开
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           分解成具体的有相对独立性的基本的功能要求
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           这样就可以把被测试软件中所有的因子都确定下来
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           并为确定个因子的权值提供参考的依据
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           确定因子与状态是设计测试用例的关键
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           因此要求尽可能全面的正确的确定取值
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           以确保测试用例的设计作到完整与有效。
          </span>
         </p>
         <p>
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           加权筛选
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           生成因素分析表
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           对因子与状态的选择可按其重要程度分别加权
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           可根据各个因子及状态的作用大小
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           出现频率的大小以及测试的需要
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           确定权值的大小。
          </span>
         </p>
         <p>
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           利用正交表构造测试数据集
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           正交表的推导依据
          </span>
          <span style="color:black">
           Galois
          </span>
          <span style="color:black">
           理论（这里省略
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           需要时可查数理统计方面的教材）。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           利用正交实验设计方法设计测试用例
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           比使用等价类划分
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           边界值分析
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           因果图等方法有以下优点
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           节省测试工作工时；可控制生成的测试用例数量；测试用例具有一定的覆盖率。
          </span>
         </p>
         <p>
         </p>
         <br/>
         <h2>
          <a name="_Toc223260595">
           <strong>
            <span style="font-size:24px">
             8.
            </span>
            <span style="font-size:24px">
             功能图法
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260596">
            <span style="font-size:14px">
             8.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
         </p>
         <p style="background:white">
          <span style="color:black">
           功能图由状态迁移图和布尔函数组成
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           状态迁移图用状态和迁移来描述
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           一个状态指出数据输入的位置（或时间）
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           而迁移则指明状态的改变
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           同时要依靠判定表或因果图表示的逻辑功能
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           例
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           一个简化的自动出纳机
          </span>
          <span style="color:black">
           ATM
          </span>
          <span style="color:black">
           的功能图。
          </span>
         </p>
         <div>
          <h3>
           <a name="_Toc223260597">
            <span style="font-size:14px">
             8.2.
            </span>
            <span style="font-size:14px">
             功能图法的应用
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p style="background:white">
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           功能图介绍
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           一个程序的功能说明通常由动态说明和静态说明组成
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           动态说明描述了输入数据的次序或转移的次序
          </span>
          <span style="color:black">
           .
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           静态说明描述了输入条件与输出条件之间的对应关系
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           对于较复杂的
          </span>
          <span style="color:black">
           程序,由于存在大量的组合情况,因此,仅用静态说明组成的规格说明对于测试来说往往是不够的.必须用动态说明来补充功能说明.功能图方法是用功能图FD形式化地表示程序的功能说明,并机械地生成功能图的测试用例.
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           功能图模型由状态迁移图和逻辑功能模型构成.状态迁移图用于表示输入数据序列以及相应的输出数据.在状态迁移图中,由输入数据和当前状态决定输出数据和后续状态.逻辑功能模型用于表示在状态中输入条件和输出条件之间的对应关系.逻辑功能模型只适合于描述静态说明,输出数据仅由输入数据决定.测试用例则是由测试中经过的一系列状态和在每个状态中必须依靠输入/输出数据满足的一对条件组成.功能图方法其实是是一种黑盒白盒混合用例设计方法
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           （功能图方法中
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           要用到逻辑覆盖和路径测试的概念和方法
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           其属白盒测试方法中
          </span>
          <span style="color:black">
           的内容.逻辑覆盖是以程序内部的逻辑结构为基础的测试用例设计方法.该方法要求测试人员对程序的逻辑结构有清楚的了解.由于覆盖测试的目标不同,逻辑覆盖可分为:语句覆盖,判定覆盖,判定-条件覆盖,条件组合覆盖及路径覆盖.下面我们指的逻辑覆盖和路径是功能或系统水平上的,以区别与白盒测试中的程序内部的.
          </span>
          <span style="color:black">
           ）
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           测试用例生成方法
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           从功能图生成测试用例,得到的测试用例数是可接受的. 问题的关键的是如何从状态迁移图中选取测试用例. 若用节点代替状态,用弧线代替迁移,则状态迁移图就可转化成一个程序的控制流程图形式.问题就转化为程序的路径测试问题（如白盒测试）问题了.
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           测试用例生成规则
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           为了把状态迁移（测试路径）的测试用例与逻辑模型（局部测试用例）的测试用例组合起来,从功能图生成实用的测试用例,须定义下面的规则.在一个结构化的状态迁移（SST）中,定义三种形式的循环:顺序,选择和重复.但分辨一个状态迁移中的所有循环是有困难的.（其表示图形省略）。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           4.
          </span>
          <span style="color:black">
           从功能图生成测试用例的过程
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           生成局部测试用例
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           在每个状态中
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           从因果图生成局部测试用例
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           局部测试用例由原因值（输入数据）组合与对应的结果值（输出数据或状态）构成。
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           测试路径生成
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           利用上面的规则（三种）生成从初始状态到最后状态的测试路径。
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           测试用例合成
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           合成测试路径与功能图中每个状态中的局部测试用例
          </span>
          <span style="color:black">
           .
          </span>
          <span style="color:black">
           结果是初始状态到最后状态的一个状态序列
          </span>
          <span style="color:black">
           ,
          </span>
          <span style="color:black">
           以及每个状态中输入数据与对应输出数据的组合。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           5.
          </span>
          <span style="color:black">
           测试用例的合成算法
          </span>
          <span style="color:black">
           :
          </span>
          <span style="color:black">
           采用条件构造树
          </span>
          <span style="color:black">
           .
          </span>
         </p>

           <a name="_Toc223260599">
            <span style="font-size:14px">
             9.1.
            </span>
            <span style="font-size:14px">
             概念
            </span>
           </a>
          </h3>
         </div>
         <p>
         </p>
         <p>
          <span style="color:black">
           现在的软件几乎都是用事件触发来控制流程的，事件触发时的情景便形成了场景，而同一事件不同的触发顺序和处理结果就形成事件流。这种在软件设计方面的思想也可以引入到软件测试中，可以比较生动地描绘出事件触发时的情景，有利于测试设计者设计测试用例，同时使测试用例更容易理解和执行。

          </span>

         <h2>
          <a name="_Toc223260602">
           <strong>
            <span style="font-size:24px">
             10.
            </span>
            <span style="font-size:24px">
             测试用例设计综合策略
            </span>
           </strong>
          </a>
         </h2>
         <p>
          <span style="font-family:Times New Roman">
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           1.
          </span>
          <span style="color:black">
           Myers提出了使用各种测试方法的综合策略
          </span>
          <span style="color:black">
           ：
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           在任何情况下都必须使用边界值分析方法，经验表明用这种方法设计出测试用例发现程序错误的能力最强。
          </span>
          <span style="color:white">
           】
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           必要时用等价类划分方法补充一些测试用例
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           用错误推测法再追加一些测试用例
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           对照程序逻辑，检查已设计出的测试用例的逻辑覆盖程度，如果没有达到要求的覆盖标准，应当再补充足够的测试用例
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           如果程序的功能说明中含有输入条件的组合情况，则一开始就可选用因果图法
          </span>
          <span style="color:black">
           。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           2.
          </span>
          <span style="color:black">
           测试用例的设计步骤
          </span>
          <span style="color:white">
           】
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           构造根据设计规格得出的基本功能测试用例
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           边界值测试用例
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           状态转换测试用例
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           4)
          </span>
          <span style="color:black">
           错误猜测测试用例
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           5)
          </span>
          <span style="color:black">
           异常测试用例；
          </span>
          <span style="color:white">
           】
          </span>
         </p>
         <p>
          <span style="color:black">
           6)
          </span>
          <span style="color:black">
           性能测试用例；
          </span>
         </p>
         <p>
          <span style="color:black">
           7)
          </span>
          <span style="color:black">
           压力测试用例。
          </span>
         </p>
         <p style="background:white">
          <span style="color:black">
           3.
          </span>
          <span style="color:black">
           优化测试用例的方
          </span>
          <span style="color:black">
           法
          </span>
         </p>
         <p>
          <span style="color:black">
           1)
          </span>
          <span style="color:black">
           利用设计测试用例的8种方法不断的对测试用例进行分解与合并
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           2)
          </span>
          <span style="color:black">
           采用遗传算法理论进化测试用例
          </span>
          <span style="color:black">
           ；
          </span>
         </p>
         <p>
          <span style="color:black">
           3)
          </span>
          <span style="color:black">
           在测试时利用发散思维构造测试用例

          <span style="color:black">
           ；
