    <h5>
     <a id="_0">
     </a>
     问题背景
    </h5>
    <p>
     在实际业务中，我们常会遇到数据冗余问题。例如，一个公司表（
     <code>
      sys_company
     </code>
     ）中存在多条相同公司名的记录，但只有一条有效（
     <code>
      del_flag=0
     </code>
     ），其余需要删除。删除前需将关联表（如合同草稿表、发票表等）的外键字段（如
     <code>
      purchaser_id
     </code>
     ）替换为保留记录的ID。这类问题通常涉及多表、多字段的动态更新，如何高效且安全地实现？
    </p>
    <hr/>
    <h5>
     <a id="_5">
     </a>
     解决方案
    </h5>
    <p>
     我们将通过以下步骤实现：
    </p>
    <ol>
     <li>
      <strong>
       配置化驱动
      </strong>
      ：用配置类声明需要处理的表和字段，避免硬编码。
     </li>
     <li>
      <strong>
       动态SQL更新
      </strong>
      ：通过MyBatis XML实现批量更新和删除。
     </li>
     <li>
      <strong>
       事务一致性
      </strong>
      ：确保所有操作原子化执行。
     </li>
    </ol>
    <hr/>

                <span class="token comment">// 4. 删除冗余公司记录</span>
                companyCleanMapper<span class="token punctuation">.</span><span class="token function">deleteCompanies</span><span class="token punctuation">(</span>idsToDelete<span class="token punctuation">)</span><span class="token punctuation">;</span>

    <h4>
     <a id="_190">
     </a>
     关键设计说明
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        实体类与数据映射
       </strong>
       <br/>
       •
       <code>
        CompanyRetainedInfo
       </code>
       通过别名（
       <code>
        AS retainedId
       </code>
       ）直接映射查询结果，避免额外转换。
       <br/>
       •
       <code>
        companyName
       </code>
       和
       <code>
        retainedName
       </code>
       字段值相同，但保留后者以明确语义。
      </p>
     </li>
     <li>
      <p>
       <strong>
        XML动态SQL优势
       </strong>
       <br/>
       •
       <strong>
        <code>
         &lt;choose&gt;
        </code>
       </strong>
       ：根据配置动态决定是否更新名称字段。
       <br/>
       •
       <strong>
        <code>
         &lt;foreach&gt;
        </code>
       </strong>
       ：自动展开ID列表为
       <code>
        IN (id1, id2...)
       </code>
       ，支持批量操作。
       <br/>
       •
       <strong>
        <code>
         ${}
        </code>
        占位符
       </strong>
       ：安全引用配置的表名和字段名（非用户输入，无注入风险）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        事务与性能优化
       </strong>
       <br/>
       •
       <strong>
        <code>
         @Transactional
        </code>
       </strong>
       ：保证“更新外键”和“删除公司”操作的原子性。
       <br/>
       •
       <strong>
        索引建议
       </strong>
       ：对
       <code>
        sys_company.company_name
       </code>
       和关联表的外键字段添加索引。
      </p>
     </li>
    </ol>

    <h4>
     <a id="_207">
     </a>
     总结
    </h4>
    <p>
     通过
     <strong>
      实体类封装
     </strong>
     、
     <strong>
      配置化表关系
     </strong>
     和
     <strong>
      MyBatis动态SQL
     </strong>
     ，我们实现了一套可扩展的多表数据清洗方案。这种模式的核心在于：
    </p>
    <ol>
     <li>
      <strong>
       抽象变化部分
      </strong>
      ：将表和字段的差异收敛到配置类中。
     </li>
     <li>
      <strong>
       复用不变逻辑
      </strong>
      ：批量更新和删除操作由统一服务驱动。
     </li>
     <li>
      <strong>
       最小化侵入性
      </strong>
      ：新增表只需修改配置，无需改动核心逻辑。
     </li>
    </ol>
    <p>
     该方案适用于用户中心、商品系统等存在外键关联的冗余数据处理场景，读者可结合实际需求调整配置和SQL逻辑。
    </p>
