     <span style="color:#0d0016">
      一、 进程创建
     </span>
    </h2>
    <blockquote>
     <h3 id="fork%E5%87%BD%E6%95%B0" name="fork%E5%87%BD%E6%95%B0">
      <span style="color:#0d0016">
       fork函数
      </span>
     </h3>
     <span style="color:#0d0016">
      在linux中fork函数时非常重要的函数，它从已存在进程中创建一个新进程。新进程为子进程，而原进程为父进程。
     </span>
     <p>
     </p>
     <pre><code class="language-cpp">#include &lt;unistd.h&gt;
pid_t fork(void);
</code></pre>
    </blockquote>
    <p>
     <span style="color:#fe2c24">
      进程：内核的相关管理数据结构(tast_struct + mm_struct + 页表)  + 代码和数据
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       <strong>
        进程调用fork，当控制转移到内核中的fork代码后，内核做：
       </strong>
      </span>
     </p>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       1.分配新的内存块和内核数据结构给子进程
       <br/>
       2.将父进程部分数据结构内容拷贝至子进程
       <br/>
       3.添加子进程到系统进程列表当中
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       4.fork返回，开始调度器调度
      </span>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       <strong>
        写时拷贝
       </strong>
      </span>
     </p>
     <p>
      <img alt="" height="420" src="https://i-blog.csdnimg.cn/direct/910a879fd99c4c28a6c899bb69dd9dbf.png" width="1137"/>
     </p>
     <p>
      <span style="color:#0d0016">
       fork函数具体使用和写时拷贝，可以查看我的上一篇博客：
      </span>
     </p>
     <p>
      <a href="https://blog.csdn.net/2303_77756141/article/details/146186902?spm=1011.2124.3001.6209" title="linux操作系统内存管理核心解析：地址空间、页表与进程调度原理-CSDN博客">
       linux操作系统内存管理核心解析：地址空间、页表与进程调度原理-CSDN博客
      </a>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <strong>
       <span style="color:#0d0016">
        返回值：子进程中返回0，父进程返回子进程id，出错返回-1
       </span>
      </strong>
     </p>
     <p>
      <span style="color:#0d0016">
       因为return的本质就是向父进程进行写入，写入发生写时拷贝。
      </span>
     </p>
     <p>
      <strong>
       <span style="color:#fe2c24">
        为什么给父进程返回的是子进程的PID，子进程返回的是0？
       </span>
      </strong>
     </p>
     <p>
      <span style="color:#0d0016">
       方便父进程对子进程进行标识，从而对子进程进行管理，通过0值知道子进程创建成功
      </span>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       <strong>
        fork常规用法
       </strong>
      </span>
     </p>
     <p>
     </p>
     <span style="color:#0d0016">
      1.一个父进程希望复制自己，使父子进程同时执行不同的代码段。例如，父进程等待客户端请求，生成子进程来处理请求。
     </span>
     <p>
     </p>
     <span style="color:#0d0016">
      2.一个进程要执行一个不同的程序。例如子进程从fork返回后，调用exec函数。
     </span>
    </blockquote>
    <blockquote>
     <strong>
      <span style="color:#0d0016">
       fork调用失败的原因
      </span>
     </strong>
     <p>
     </p>
     <span style="color:#0d0016">
      1.系统中有太多的进程
     </span>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       2.实际用户的进程数超过了限制
      </span>
     </p>
    </blockquote>
    <h2 id="%E4%BA%8C%E3%80%81%E8%BF%9B%E7%A8%8B%E7%9A%84%E7%BB%88%E6%AD%A2%EF%BC%9A" name="%E4%BA%8C%E3%80%81%E8%BF%9B%E7%A8%8B%E7%9A%84%E7%BB%88%E6%AD%A2%EF%BC%9A">
     <span style="color:#0d0016">
      二、进程的终止：
     </span>
    </h2>
    <h3 id="1.%20%E6%83%B3%E6%98%8E%E7%99%BD%EF%BC%9A%E7%BB%88%E6%AD%A2%E6%98%AF%E5%9C%A8%E5%81%9A%E4%BB%80%E4%B9%88%EF%BC%9F" name="1.%20%E6%83%B3%E6%98%8E%E7%99%BD%EF%BC%9A%E7%BB%88%E6%AD%A2%E6%98%AF%E5%9C%A8%E5%81%9A%E4%BB%80%E4%B9%88%EF%BC%9F">
     <span style="color:#0d0016">
      1. 想明白：终止是在做什么？
     </span>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       创建进程是先创建好内核的相关管理数据结构还是先创建好代码和数据？
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       <strong>
        先 创建好内核的相关管理数据结构  再 创建好代 码和数据
       </strong>
      </span>
     </p>
     <p>
     </p>
     <p>
      <span style="color:#fe2c24">
       <strong>
        终止是在释放曾经的代码和数据所占据的空间。再释放内核数据结构
       </strong>
      </span>
     </p>
    </blockquote>
    <hr/>
    <h3 id="%C2%A02.%E8%BF%9B%E7%A8%8B%E7%BB%88%E6%AD%A2%E7%9A%843%E7%A7%8D%E6%83%85%E5%86%B5%EF%BC%9F" name="%C2%A02.%E8%BF%9B%E7%A8%8B%E7%BB%88%E6%AD%A2%E7%9A%843%E7%A7%8D%E6%83%85%E5%86%B5%EF%BC%9F">
     <span style="color:#fe2c24">
      2.进程终止的3种情况？
     </span>
    </h3>
    <h3 id="a.%E9%80%80%E5%87%BA%E7%A0%81%E6%98%AF%E4%BB%80%E4%B9%88%EF%BC%9F%E5%AD%98%E5%9C%A8%E5%8E%9F%E5%9B%A0%EF%BC%9F%E4%B8%BA%E4%BB%80%E4%B9%88int%20main%EF%BC%88%EF%BC%89return%200%3F" name="a.%E9%80%80%E5%87%BA%E7%A0%81%E6%98%AF%E4%BB%80%E4%B9%88%EF%BC%9F%E5%AD%98%E5%9C%A8%E5%8E%9F%E5%9B%A0%EF%BC%9F%E4%B8%BA%E4%BB%80%E4%B9%88int%20main%EF%BC%88%EF%BC%89return%200%3F">
     <span style="color:#0d0016">
      a.退出码是什么？存在原因？为什么int main（）return 0?
     </span>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       使用vim编译器写一段代码：
      </span>
     </p>
     <pre><code class="language-cpp">#include&lt;stdio.h&gt;

#include&lt;unistd.h&gt;

int main()
{
	printf("i'm a process, pid: %d, ppid: %d\n",getpid(),getppid());
	sleep(1);
	return 0;
}</code></pre>
     <p>
      <span style="color:#0d0016">
       这是我两次运行打印出的结果：
      </span>
     </p>
     <pre><code class="language-bash">i'm a process, pid: 562487, ppid: 553748

 
i'm a process, pid: 562539, ppid: 553748
</code></pre>
     <p>
      <span style="color:#0d0016">
       现在将main函数的返回值改为100
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       再次运行程序的结果：
      </span>
     </p>
     <pre><code class="language-bash">i'm a process, pid: 563668, ppid: 553748


i'm a process, pid: 563689, ppid: 553748
</code></pre>
     <p>
      <span style="color:#0d0016">
       此时在命令行输入echo $?,输出结果100
      </span>
     </p>
     <pre><code class="language-bash">echo $?
100
</code></pre>
     <p>
      <span style="color:#0d0016">
       变量'?'
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       在shell中存在一个变量'?'    ----&gt;使用‘$’ 查看
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       echo 是 内建命令，打印的都是bash内部的变量数据。
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       ?   ----&gt;  父进程bash获取到的，
      </span>
      <span style="color:#fe2c24">
       <strong>
        最近一个子进程退出的退出码
       </strong>
      </span>
      <span style="color:#0d0016">
       ，退出码为0表示成功，!0表示失败。
      </span>
     </p>
     <p style="text-align:center">
      <span style="color:#fe2c24">
       <strong>
        进程的退出码是告诉关心方（父进程），任务完成的情况，失败的原因是什么。
       </strong>
      </span>
     </p>
     <p>
      <img alt="" height="501" src="https://i-blog.csdnimg.cn/direct/82bb0d860da24bb1a1e0bf5e88c181a6.png" width="884"/>
     </p>
     <p>
      <span style="color:#0d0016">
       不是说
       <strong>
        echo $？
       </strong>
       是查看最近一次进程的退出码吗？
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       因为
       <strong>
        echo $?
       </strong>
       也是一个进程
      </span>
     </p>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       不同的非0值，一方面表示失败，一方面表示失败原因，每一个数字都有对应的错误描述（字符串类型）。
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       运行以下代码：
      </span>
     </p>
     <pre><code class="language-cpp">#include&lt;stdio.h&gt;

#include&lt;unistd.h&gt;

#include&lt;string.h&gt;

int main()
{       
	for(int errcode = 0; errcode &lt;= 255; errcode++)
	{
		printf("errcode=%d, %s\n",errcode, strerror(errcode));
	}
	printf("i'm a process, pid: %d, ppid: %d\n",getpid(),getppid());
	sleep(1);
	return 0;
}
</code></pre>
     <p>
      <span style="color:#0d0016">
       其中的strerror:把错误码转换成一段描述错误信息的字符串。
      </span>
     </p>
     <p>
      <img alt="" height="61" src="https://i-blog.csdnimg.cn/direct/0ba8cc037aff472cbc5e2b6675523e9a.png" width="661"/>
     </p>
     <p>
      <span style="color:#0d0016">
       运行程序：一共有133条错误码的提示。我们发现在进行指令操作或者运行程序失败后，系统会直接告诉我们错误的原因。
      </span>
     </p>
     <p>
      <img alt="" height="815" src="https://i-blog.csdnimg.cn/direct/c20e3efa0692415fb8c186a683626ed4.png" width="1159"/>
     </p>
     <p>
      <span style="color:#0d0016">
       <strong>
        因此，进程的退出码是告诉关心方（父进程），任务完成的情况，失败的原因是什么。bash是为用户负责
       </strong>
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       运行以下代码：
      </span>
     </p>
     <pre><code class="language-cpp">#include&lt;stdio.h&gt;

#include&lt;unistd.h&gt;

#include&lt;string.h&gt;


int Div(int x, int y)
{
        if( 0 == y)
        {
                return -1;
        }
        else
        {
                return x / y;
        }
}

int main()
{       
        int result = 0;
        result = Div(10, 0);
        printf("result = %d\n", result);
        sleep(1);
        return 0;
}</code></pre>
     <p>
      <span style="color:#0d0016">
       运行结果：
      </span>
     </p>
     <pre><code class="language-bash">result = -1
</code></pre>
     <p>
      <span style="color:#0d0016">
       虽然我知道代码是什么，但看到result = -1, 会认为是结果为-1，还是因为错误的 -1
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       再写以下代码：
      </span>
     </p>
     <pre><code class="language-cpp">#include&lt;stdio.h&gt;

#include&lt;unistd.h&gt;

#include&lt;string.h&gt;

//自定义枚举常量
enum
{
	Success = 0,
	DIV_ZERO,
	Mod_Zero,
};
int exit_code = Success;

const char* CodeToErrString(int code)
{
	switch(code)
	{
		case Success:
			return "Success!";
		case DIV_ZERO:
			return "div zero!";
		case Mod_Zero:
			return "mod zero!";
		default:
			return "unknown error!";
	}
}

int Div(int x, int y)
{
	if( 0 == y)
	{
		exit_code = DIV_ZERO;
		return -1;
	}
	else
	{
		return x / y;
	}
}

int main()
{
 	int result = 0;
	result = Div(10, 5);
	printf("result = %d [%s]\n", result, CodeToErrString(exit_code));
	result = Div(10, 0);
	printf("result = %d [%s]\n", result, CodeToErrString(exit_code));
	sleep(1);
	return 0;
}
</code></pre>
     <p>
      <span style="color:#0d0016">
       输出结果：
      </span>
     </p>
     <pre><code class="language-bash">result = 2 [Success!]
result = -1 [div zero!]
</code></pre>
     <p>
      <span style="color:#0d0016">
       此时的结果就是正确的
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       进程终止的前两种情况：
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       <span style="background-color:#fef2f0">
        1.代码跑完，结果正确

     <p>
      <strong>
       <span style="color:#fe2c24">

      <span style="color:#0d0016">
       exit() （c库函数）vs _exit()（系统调用）：
      </span>
     </h3>
     <p>
      <span style="color:#0d0016">

      </span>
     </p>

          <strong>
           <span style="color:#0d0016">

           </span>
          </strong>
         </p>
        </li>
        <li>
         <p>
          <strong>
           <span style="color:#0d0016">
            用户空间的缓冲区（由
            <code>
             exit()
            </code>
            处理，
            <code>
             _exit()
            </code>
            不处理）。
           </span>
          </strong>
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         <span style="color:#0d0016">
          释放内核空间资源：
         </span>
        </strong>
       </p>
       <ul>
        <li>
         <p>
          <strong>
           <span style="color:#0d0016">
            关闭所有打开的文件描述符（由内核自动处理，无论是否调用
            <code>
             exit()
            </code>
            ）。
           </span>
          </strong>
         </p>
        </li>
        <li>
         <p>
          <strong>
           <span style="color:#0d0016">
            释放进程的页表、信号处理结构、定时器等内核数据结构。
           </span>
          </strong>
         </p>
        </li>
        <li>
         <p>
          <strong>
           <span style="color:#0d0016">
            PCB（进程控制块）：内核会保留 PCB 直到父进程通过
            <code>
             wait()
            </code>
            读取子进程的退出状态（
            <code>
             ZOMBIE
            </code>
            状态）。之后 PCB 才会被彻底释放。
           </span>
          </strong>
         </p>
        </li>
       </ul>
      </li>
      <li>
       <p>
        <strong>
         <span style="color:#0d0016">
          通知父进程：通过
          <code>
           SIGCHLD
          </code>
          信号和退出状态码。
         </span>
        </strong>
       </p>

     <ul>
      <li>
       <p>
        <span style="color:#0d0016">
         <strong>
          <code>
           task_struct
          </code>
          （进程控制块，PCB）
         </strong>

     <strong>
      <span style="color:#0d0016">
       进程等待必要性
      </span>
     </strong>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       之前讲过，子进程退出，父进程如果不管不顾，就可能造成‘僵尸进程’的问题，进而造成内存泄漏。另外，进程一旦变成僵尸状态，那就刀枪不入，“杀人不眨眼”的kill -9 也无能为力，因为谁也没有办法杀死一个已经死去的进程。
      </span>
     </p>
     <p>
     </p>
     <span style="color:#0d0016">
      最后，父进程派给子进程的任务完成的如何，我们需要知道。如，子进程运行完成，结果对还是不对，或者是否正常退出。
     </span>
     <p>
     </p>
     <p>
      <span style="color:#fe2c24">
       <strong>
        父进程通过进程等待的方式：
       </strong>
      </span>
     </p>
     <p>
     </p>

      <strong>
       <span style="color:#0d0016">
        1.回收子进程资源
       </span>
      </strong>
     </p>
     <p>
     </p>
     <p>
      <strong>
       <span style="color:#0d0016">
        2. 获取子进程退出信息。
       </span>
      </strong>
     </p>
    </blockquote>
    <h3 id="wait()" name="wait()">
     wait()
    </h3>
    <p>
     <img alt="" height="1281" src="https://i-blog.csdnimg.cn/direct/7507e483a4834a299a87a59f36dd13f2.png" width="2534"/>
    </p>
    <p>
     <span style="color:#0d0016">
      pid_t wait(int* status);
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      wait函数成功后，会返回进程的pid，等待父进程中，任意一个子进程退出。
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       如果子进程已经退出，调用wait/waitpid时，wait/waitpid会立即返回，并且释放资源，获得子进程退出信息。
      </span>
     </p>
     <p>
     </p>
     <span style="color:#0d0016">
      如果在任意时刻调用wait/waitpid，子进程存在且正常运行，则进程可能阻塞。
     </span>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       如果不存在该子进程，则立即出错返回。
      </span>
     </p>
    </blockquote>

    <h3 id="%E9%9D%9E%E9%98%BB%E5%A1%9E%E7%AD%89%E5%BE%85" name="%E9%9D%9E%E9%98%BB%E5%A1%9E%E7%AD%89%E5%BE%85">
     <span style="color:#0d0016">
      非阻塞等待
     </span>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       pid_t waitpid(pid_t pid, int *status, int options) ---&gt;当 options = 0就默认是阻塞等待
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       宏：WNOHANG  ---&gt;  以非阻塞等待    ，HANG --&gt; hang(服务器卡住了，服务器hang住了)
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       服务器hang很久，计算机挂掉  ----&gt; 服务器宕机了。WNOHANG --&gt;wait no hang，等待的时候不要hang住。
      </span>
     </p>
    </blockquote>
    <p>
     <span style="color:#0d0016">
      非阻塞等待的特点（优点）：
     </span>
    </p>
    <p>
     <span style="color:#0d0016">
      1. 检测子进程状态的变化，是否就绪
     </span>
    </p>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       阻塞等待
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       pid_t &gt; 0 : 等待成功，子进程退出了，并且父进程回收成功，并退出
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       pid_t &lt; 0 :等待失败了
      </span>
     </p>
    </blockquote>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       非阻塞等待：
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       pid_t  ==  0:检测是成功的，只不过子进程还没有退出，需要你下一次进行重复等待。
      </span>
     </p>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       非阻塞等待接口 + 循环  =  非阻塞轮询方案
      </span>
     </p>
    </blockquote>
    <p>
     <span style="color:#0d0016">
      2. 在非阻塞轮询的时候允许父进程做其他的事情
     </span>

    <h3 id="%E6%9B%BF%E6%8D%A2%E5%8E%9F%E7%90%86" name="%E6%9B%BF%E6%8D%A2%E5%8E%9F%E7%90%86">
     <span style="color:#0d0016">
      替换原理
     </span>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       用fork创建子进程后执行的是和父进程相同的程序(但有可能执行不同的代码分支),子进程往往要调用一种exec函数以执行另一个程序。当进程调用一种exec函数时,该进程的用户空间代码和数据完全被新程序替换,从新程序的启动例程开始执行。调用exec并不创建新进程,所以调用exec前后该进程的id并未改变。
      </span>
     </p>
    </blockquote>
    <h3 id="1.%20%E9%80%9A%E8%BF%87%E4%BB%A3%E7%A0%81%E7%9C%8B%E7%8E%B0%E8%B1%A1" name="1.%20%E9%80%9A%E8%BF%87%E4%BB%A3%E7%A0%81%E7%9C%8B%E7%8E%B0%E8%B1%A1">

    <p>
     <strong>

     </p>

    <h3 id="2.%20%E8%A7%A3%E9%87%8A%E5%8E%9F%E7%90%86" name="2.%20%E8%A7%A3%E9%87%8A%E5%8E%9F%E7%90%86">
     <span style="color:#0d0016">
      2. 解释原理
     </span>
    </h3>
    <p>
     <span style="color:#0d0016">
      <img alt="" height="414" src="https://i-blog.csdnimg.cn/direct/e4ef0ea500914207ab72a919ee6fc8db.png" width="1075"/>
     </span>
    </p>
    <p>
    </p>
    <blockquote>
     <p style="text-align:center">
      <span style="color:#fe2c24">
       <strong>
        进程 = 内核数据结构 + 代码和数据
       </strong>
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       程序替换：在执行进程的时候，保留原有的内核数据结构，结构本身不变(部分属性值更改)，再将新程序的代码和数据和原有的在物理内存上的代码和数据进行替换覆盖，取代原有的代码和数据。
      </span>
     </p>
     <p>
     </p>
     <p>
      <span style="color:#0d0016">
       在进行替换的时候，没有创建新的进程。
      </span>
     </p>
    </blockquote>
    <p>
    </p>
    <p>
     <span style="color:#0d0016">
      查看此图：
     </span>
    </p>
    <p>
     <img alt="" height="661" src="https://i-blog.csdnimg.cn/direct/21d359a32ec8424cbb8be842bd029cac.png" width="1601"/>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       程序运行之前，会先被加载到内存里，根据冯诺依曼定理，我的程序最总是会被cpu访问的，cpu只能就近访问内存，是通过 exec* 函数（类似于一种Linux上的加载函数）加载的。
      </span>
     </strong>
    </p>
    <p>
     <strong>
      <span style="color:#0d0016">
       exec* 函数的返回值不需要关心，只要替代成功就不会运行后面的代码，反之，只要后面的代码运行了，就没有替换成功，因此不需要通过返回值来判断是否替换成功。
      </span>
     </strong>
    </p>
    <hr/>
    <p>
     <strong>
      <span style="color:#0d0016">
       想要进行程序替换但是不想要影响父进程本身：
      </span>
     </strong>
    </p>
    <h3 id="3.%20%E5%B0%86%E4%BB%A3%E7%A0%81%E6%94%B9%E6%88%90%E5%A4%9A%E8%BF%9B%E7%A8%8B%E7%89%88" name="3.%20%E5%B0%86%E4%BB%A3%E7%A0%81%E6%94%B9%E6%88%90%E5%A4%9A%E8%BF%9B%E7%A8%8B%E7%89%88">
     <span style="color:#0d0016">
      3. 将代码改成多进程版
     </span>
    </h3>
    <blockquote>
     <p>
      <span style="color:#0d0016">
       检测一次失败的替换：
      </span>
     </p>

    exit(1);

  int status = 0;
  pid_t rid = waitpid(id, &amp;status, 0);

  if(rid == id)
  {
    printf("father wait success, child exit code : %d\n", WEXITSTATUS(status));
  }
  printf("testexec ... end\n");
  return 0;

     <pre><code class="language-cpp">#include&lt;iostream&gt;

{
  cout &lt;&lt; "hello c++, i'm c++ pragma!" &lt;&lt; endl;
  cout &lt;&lt; "hello c++, i'm c++ pragma!" &lt;&lt; endl;
  cout &lt;&lt; "hello c++, i'm c++ pragma!" &lt;&lt; endl;
  cout &lt;&lt; "hello c++, i'm c++ pragma!" &lt;&lt; endl;
  cout &lt;&lt; "hello c++, i'm c++ pragma!" &lt;&lt; endl;

       修改makefile文件让他能一次性执行两个程序：

     <p>
      <span style="color:#0d0016">
       运行后，却只执行了mypragma,因为makefile在执行程序时，从上至下匹配时，只会默认形成第一个目标文件所对应的可执行程序，推导他的依赖关系。
      </span>
     </p>
     <p>
      <span style="color:#0d0016">

     <p>
      <span style="color:#0d0016">
       解决办法以及实现原理：
      </span>
     </p>
     <p>
      <span style="color:#0d0016">
       不能让其中的一个成为第一个目标文件，使用.PHONY，定义一个伪目标all，all所依赖的是这两个目标文件，有依赖关系，不写依赖方法，makefile在从上往下扫描时，先扫描到的是all，这个all 就成为第一个目标文件，其中并没有依赖方法，makefile就会推all其中mypragma，testexec的依赖关系：
      </span>
     </p>
     <pre><code class="language-bash">.PHONY:all
all:mypragma testexec

mypragma:mypragma.cc
	g++ -o $@ $^ -std=c++11

testexec:testexec.c
	gcc -o $@ $^
.PHONY:clean
clean:
	rm -f testexec mypragma
