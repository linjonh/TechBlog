    <h4>
     实现BottomNavigationItem
    </h4>
    <p>
     我们这里整体的底部是一个
     <code>
      BottomNavigation
     </code>
     ，他是由四个
     <code>
      BottomNavigationItem
     </code>
     组合实现。首先定义一个实体类，用于存贮底部导航栏对象信息。
    </p>
    <pre><code class="hljs">export class BottomNavigationEntity {
  /**
   * 底部导航tab标题
   */
  title: Resource;

  /**
   * 底部导航tab图片
   */
  image: Resource;

  /**
   * 底部导航tab图片，未选中
   */
  unCheckImage: Resource;

  /**
   * tab类型标志位
   */
  tag: number;

  constructor(tag: number, title: Resource, image: Resource, unCheckImage: Resource) {
    this.tag = tag;
    this.title = title;
    this.image = image;
    this.unCheckImage = unCheckImage;
  }
}
</code></pre>
    <p>
     接下来的
    </p>
    <p class="img-center">
     <img alt="" height="61" src="https://i-blog.csdnimg.cn/blog_migrate/b475716b87cc7c328c16e5d3440d205d.png" width="67"/>
    </p>
    <p>
     组成是一个图标+一个文字组合而成，第一反应我们应该行到
     <code>
      Column
     </code>
     组件。
    </p>
    <p>
     <code>
      Column
     </code>
     组件中，用于处理组件内容对其方式使用的话flex方式。
     <code>
      alignItems(value: HorizontalAlign): ColumnAttribute; # 水平方向
     </code>
     <code>
      justifyContent(value: FlexAlign): ColumnAttribute; # 垂直方向
     </code>
     了解了这些之后，接下来看具体
     <code>
      BottomNavigationItem
     </code>
     的封装代码。
    </p>
    <pre><code class="hljs">@Preview  # 方便单个view直接预览
@Component  # 标记是一个组件，可供其他组件引用
export default struct BottomNavigationItem {
  private navigationItem: BottomNavigationEntity;
  
  # 这里的Link是用于父组件和子组件进行通信	
  @Link currentIndex: number;

  build() {
    Column({ space: 5 }) {
    # 这里判断如果当前选中的item是当前的这个，则使用选中状态图片
      Image(this.currentIndex === this.navigationItem.tag ? this.navigationItem.image : this.navigationItem.unCheckImage)
        .width(24)
        .height(24)
      Text(this.navigationItem.title)
        .fontSize(14)
        .fontColor(this.currentIndex === this.navigationItem.tag ? Color.Green : 0x333333)
    }
  }
}
</code></pre>
    <p>
     代码是不是非常简单。对于
     <code>
      @Link
     </code>
     你如果现在不太清楚，也没有关系，我最终会专门进行一个讲解。
    </p>
    <h4>
     实现BottomNavigation
    </h4>
    <pre><code class="hljs">@Preview
@Component
export default struct BottomNavigation {
  @Link currentItemIndex: number;

  build() {
    Row({ space: 5 }) {
      //  这里通过对结合遍历，生成BottomNavigationItem进行填充BottomNavigation
      ForEach(navigationViewModel.getNavigationList(), (item: BottomNavigationEntity, index: number) =&gt; {
        # 对于这里的$currentItemIndex写法可以先将疑问留着，后续结合Link一并说明
        BottomNavigationItem({ navigationItem: item, currentIndex: $currentItemIndex })
          .onClick(() =&gt; {
          	#  点击后更新选中的item，以实现刷新界面的效果
            this.currentItemIndex = index
          })
      })
    }
    .width('100%')
    .height(65)
    .padding({
      top: 5,
      bottom: 5
    })
    .justifyContent(FlexAlign.SpaceAround)
    .backgroundColor(0xF3EEEA)
  }
}
</code></pre>
    <h4>
     实现WechatMainFrame
    </h4>
    <p>
     整体的界面组合使用
     <code>
      RelativeContainer
     </code>
     进行组合，将
     <code>
      BottomNavigation
     </code>
     固定于屏幕的底部，内容区域底部在
     <code>
      BottomNavigation
     </code>
     之上，顶部和屏幕顶部对其，使其填充满
     <code>
      BottomNavigation
     </code>
     之上的部分。内容区域使用
     <code>
      Stack
     </code>
     将所有的内容层叠展示，切换到哪个展示，则使用
     <code>
      visibility
     </code>
     方法设置该页面展示即可。
    </p>
    <pre><code class="hljs">@Entry
@Component
struct WechatMainFrame {
  @State currentCheckIndex: number = 0;

  build() {
    RelativeContainer() {
      BottomNavigation({ currentItemIndex: $currentCheckIndex })
        .alignRules({
          bottom: { anchor: "__container__", align: VerticalAlign.Bottom },
          left: { anchor: "__container__", align: HorizontalAlign.Start }
        })
        .id("bottomNavigation")

      Stack() {
        HomeFragment().visibility(this.currentCheckIndex == 0 ? Visibility.Visible : Visibility.Hidden)
        ContactFragment().visibility(this.currentCheckIndex == 1 ? Visibility.Visible : Visibility.Hidden)
        DiscoverFragment().visibility(this.currentCheckIndex == 2 ? Visibility.Visible : Visibility.Hidden)
        MeFragment().visibility(this.currentCheckIndex == 3 ? Visibility.Visible : Visibility.Hidden)
      }
      .width('100%')
      .height('100%')
      .alignRules({
        left: { anchor: "__container__", align: HorizontalAlign.Start },
        right: { anchor: "__container__", align: HorizontalAlign.End },
        bottom: { anchor: "bottomNavigation", align: VerticalAlign.Top },
        top: { anchor: "__container__", align: VerticalAlign.Top }
      })
      .id("contentPanel")
    }
    .width('100%').height('100%')
  }
}
</code></pre>

    <h4>
     关于@Link相关的说明
    </h4>
    <p>
     我们对于视图更新，可以使用
     <code>
      @State
     </code>
     标记变量，但是
     <code>
      @State
     </code>
     不能进行跨文件使用。这个时候
     <code>
      @Link
     </code>
     的实现就弥补了
     <code>
      @State
     </code>
     的不足。使用
     <code>
      @Link
     </code>
     的话。子组件中被
     <code>
      @Link
     </code>
     装饰的变量与其父组件中对应的数据源建立双向数据绑定。
    </p>
    <ul>
     <li>
      @Link装饰的变量与其父组件中的数据源共享相同的值。
     </li>
     <li>
      @Link装饰器不能在@Entry装饰的自定义组件中使用。
     </li>
     <li>
      @Link子组件从父组件初始化@State的语法为Comp({ aLink: this.aState })。同样Comp({aLink: $aState})也支持。
     </li>
    </ul>
    <p>
     <img alt="" height="280" src="https://i-blog.csdnimg.cn/blog_migrate/b556da6e1f1bb783736ca950b00d94af.png" width="1199"/>
    </p>
    <p>
     下面我们回到上面的代码中。结合代码进行分析。 当我们在
     <code>
      BottomNavigation
     </code>
     中
     <code>
      .onClick(() =&gt; { this.currentItemIndex = index })
     </code>
     在点击之后，会更改
     <code>
      @Link currentItemIndex: number;
     </code>
     触发界面ui的更改。而
     <code>
      BottomNavigationItem({ navigationItem: item, currentIndex: $currentItemIndex })
     </code>
     中，我们需要把选中的item的
     <code>
      index
     </code>
     值传递给
     <code>
      BottomNavigationItem
     </code>
     本身。而作为传递的值，则需要使用
     <code>
      \$
     </code>
     标记。这样点击之后会将
     <code>
      BottomNavigationItem
     </code>
     的值也触发更改，以达到更改布局效果。
     <code>
      BottomNavigationItem\
     </code>
     的判断也会根据这个值变化而变化。
    </p>
    <p>
     点击之后，除了对
     <code>
      BottomNavigation
     </code>
     的状态更新之外，还需要对内容区域进行判断展示不同的界面。因此
     <code>
      BottomNavigation
     </code>
     的
     <code>
      @Link currentItemIndex: number;
     </code>
     又要和
     <code>
      WechatMainFrame
     </code>
     的
     <code>
      @State currentCheckIndex: number = 0;
     </code>
     进行双向绑定
     <code>
      BottomNavigation({ currentItemIndex: $currentCheckIndex })
     </code>
     。最终当我们点击
     <code>
      BottomNavigation
     </code>
     的
     <code>
      onclick
     </code>
     的时候，就会向上和
     <code>
      WechatMainFrame
     </code>
     双向绑定更改内容区域，也会和
     <code>
      BottomNavigationItem
     </code>
     双向绑定更改底部导航展示。
    </p>
