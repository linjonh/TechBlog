    <h2>
     <a id="_git_github_0">
     </a>
     错误记录: git 无法连接到github
    </h2>
    <p>
     今天, 新建了一个
     <code>
      github
     </code>
     仓库, 但从本地怎么都
     <code>
      push
     </code>
     不上去.并报错
    </p>
    <pre><code>git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.
</code></pre>
    <p>
     于是开始尝试各种方法.
    </p>
    <p>
     一开始尝试重新创建添加
     <code>
      ssh公钥
     </code>
     .
    </p>
    <p>
     详见
     <a href="https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent" rel="nofollow">
      github官方文档
     </a>
    </p>
    <p>
     然后按照
     <a href="https://docs.github.com/en/authentication/troubleshooting-ssh/error-permission-denied-publickey" rel="nofollow">
      github troubleshooting-ssh
     </a>
     进行了
     <code>
      ssh
     </code>
     的连接测试.
     <br/>
     报错, 大概是这样(这个不是我的报错)
    </p>
    <pre><code>Permissions 0777 for '/Users/username/.ssh/id_rsa' are too open.
It is recommended that your private key files are NOT accessible by others.
This private key will be ignored.
</code></pre>
    <p>
     按照
     <a href="https://stackoverflow.com/questions/9270734/ssh-permissions-are-too-open" rel="nofollow">
      stackoverflow
     </a>
     上的回答的进行了文件权限的修改. 使
     <code>
      ssh
     </code>
     连接测试可以通过了.
     <br/>
     然而,
     <code>
      git
     </code>
     依然连接不上.
    </p>
    <p>
     后来我尝试启动
     <code>
      ssh-agent
     </code>
     , 分表用
     <code>
      git-bash
     </code>
     和
     <code>
      powershell
     </code>
     开启了
     <code>
      ssh-agent
     </code>
     ,并运行了
     <code>
      ssh-add
     </code>
     命令. 这次成功了. 从其他的
     <code>
      terminal
     </code>
     也可以正常
     <code>
      push
     </code>
     , 我执行
     <code>
      taskkill /f /im ssh-agent.exe
     </code>
     杀掉所有
     <code>
      ssh-agent
     </code>
     之后, 依然可以正常执行.
    </p>
    <p>
     不解. 删掉新建的各种
     <code>
      key pair
     </code>
     之后, 决定重新创建
     <code>
      key pair
     </code>
     , 重走一遍流程. 然而这次又不行了.
     <br/>
     这次的表现依然是,
     <code>
      ssh -T git@githb.com
     </code>
     可以正常连接, 但是
     <code>
      git
     </code>
     无法连接.
    </p>
    <p>
     我不禁怀疑, 它们用的是两套
     <code>
      .ssh路径
     </code>
     .
     <br/>
     然后在
     <a href="https://stackoverflow.com/questions/37518346/how-does-git-know-which-ssh-key-to-use-for-its-operations" rel="nofollow">
      这篇回答
     </a>
     的评论中, 找到了调试
     <code>
      git
     </code>
     调用
     <code>
      ssh
     </code>
     的方法.
     <br/>
     按照这种方法, 调试之后, 果然, 用的是两套路径.
     <br/>
     在
     <code>
      git
     </code>
     调用
     <code>
      ssh
     </code>
     的时候, 环境中添加了一个名称为
     <code>
      HOME
     </code>
     的环境变量. 如
     <a href="https://stackoverflow.com/questions/84096/setting-the-default-ssh-key-location" rel="nofollow">
      这个回答
     </a>
     所言.
    </p>
    <p>
     最终问题解决了. 无需用到
     <code>
      ssh-agent
     </code>
     服务
     <br/>
     然而, 为解决的两个问题是:
    </p>
    <ol>
     <li>
      为什么之前可以用, 现在忽然就不能用了, 是我在什么时候修改了什么配置导致的?
     </li>
     <li>
      <code>
       git
      </code>
      为什么在调用
      <code>
       ssh
      </code>
      的时候, 会添加
      <code>
       HOME
      </code>
      环境变量, 该值是
      <code>
       git
      </code>
      从哪里获取的?
     </li>
    </ol>
