    <h2>
     一、Docker的介绍
    </h2>
    <p>
     Docker 是一个开源的应用容器引擎，基于 Go 语言 并遵从Apache2.0协议开源。
    </p>
    <p>
     Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。
    </p>
    <p>
     容器是完全使用沙箱机制，相互之间不会有任何接口（类似 iPhone 的 app）,更重要的是容器性能开销极低
    </p>
    <h2>
     二、Docker的安装
    </h2>
    <h3>
     1. Windows系统的安装
    </h3>
    <p>
     win7、win8 等需要利用 docker toolbox 来安装，国内可以使用阿里云的镜像来下载，
    </p>
    <p>
     下载地址：
     <a class="link-info" href="http://mirrors.aliyun.com/docker-toolbox/windows/docker-toolbox/" rel="nofollow" title="http://mirrors.aliyun.com/docker-toolbox/windows/docker-toolbox/">
      http://mirrors.aliyun.com/docker-toolbox/windows/docker-toolbox/
     </a>
    </p>
    <p>
     安装好后会有3个软件：
    </p>
    <ul>
     <li>
      <p>
       Docker QuickStart
      </p>
      <ul>
       <li>
        shell. 这是一个已经配置好Docker的命令行环境
       </li>
      </ul>
     </li>
     <li>
      <p>
       Kitematic
      </p>
      <ul>
       <li>
        这是Docker的GUI版本
       </li>
      </ul>
     </li>
     <li>
      <p>
       Oracle VM Virtualbox
      </p>
      <ul>
       <li>
        虚拟机
       </li>
      </ul>
     </li>
    </ul>
    <p>
     win10 系统
    </p>
    <p>
     现在 Docker 有专门的 Win10 专业版系统的安装包，需要开启Hyper-V
    </p>
    <p>
     下载地址：
     <a class="link-info" href="http://%20https://www.docker.com/get-docker" rel="nofollow" title=" https://www.docker.com/get-docker">
      https://www.docker.com/get-docker
     </a>
    </p>
    <p>
     安装完成后，Docker 会自动启动。通知栏上会出现个小鲸鱼的图标
    </p>
    <h3>
     2. CentOS系统的安装
    </h3>
    <p>
     Docker支持以下的CentOS版本：
    </p>
    <ul>
     <li>
      CentOS 7 (64-bit)
     </li>
     <li>
      CentOS 6.5 (64-bit) 或更高的版本
     </li>
    </ul>
    <h4>
     1）前提条件
    </h4>
    <p>
     目前，CentOS 仅发行版本中的内核支持 Docker。
    </p>
    <p>
     Docker 运行在 CentOS 7 上，要求系统为64位、系统内核版本为 3.10 以上。
    </p>
    <p>
     Docker 运行在 CentOS-6.5 或更高的版本的 CentOS 上，要求系统为64位、系统内核版本为 2.6.32-431 或者更高版本。
    </p>
    <p>
     如果系统内核不符合，可以通过以下命令升级
    </p>
    <pre><code>uname -r #查看版本号
​
rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org #导入公钥数字证书
​
rpm -ivh http://www.elrepo.org/elrepo-release-6-5.el6.elrepo.noarch.rpm #安装ELRepo
​
yum --enablerepo=elrepo-kernel install kernel-lt -y #安装kernel长期版本 lt表示long-term的意思，长期维护版本，也可以将kernel-lt改为kernel-ml，安装主线版本
​
#编辑grub.conf文件，修改Grub引导顺序，确认刚安装好的内核在哪个位置，然后设置default值（从0开始），一般新安装的内核在第一个位置，所以设置default=0
​
vim /etc/grub.conf
​
# 以下是/etc/grub.conf的内容
default=0        # 修改该值即可
timeout=5
splashimage=(hd0,0)/grub/splash.xpm.gz
hiddenmenu
title CentOS (3.10.103-1.el6.elrepo.x86_64)
​
# 重启并查看内核版本，将会发现内核已经更新</code></pre>
    <h3>
     3. 软件的安装
    </h3>
    <p>
     Docker 要求 CentOS 系统的内核版本高于 3.10 ，查看本页面的前提条件来验证你的CentOS 版本是否支持 Docker 。
    </p>
    <p>
     通过 uname -r 命令查看你当前的内核版本
    </p>
    <pre><code>安装命令： yum install -y docker</code></pre>
    <h2>
     三、软件的启动
    </h2>
    <p>
     1. windows
    </p>
    <p>
     可以直接图标直接启动
    </p>
    <p>
     2. liunx
    </p>
    <p>
     启动命令：
     <code>
      sudo systemctl start docker
     </code>
    </p>
    <p>
    </p>
    <h2>
     四、docker的常用命令
    </h2>
    <ul>
     <li>
      <p>
       查询镜像
      </p>
      <ul>
       <li>
        docker search 镜像名
       </li>
      </ul>
     </li>
     <li>
      <p>
       下载镜像
      </p>
      <ul>
       <li>
        docker pull 镜像名:
       </li>
      </ul>
     </li>
     <li>
      <p>
       查看当前有什么镜像
      </p>
      <ul>
       <li>
        docker images
       </li>
      </ul>
     </li>
     <li>
      <p>
       创建容器
      </p>
      <ul>
       <li>
        docker run -tid image_id(前4位就可)
       </li>
      </ul>
     </li>
     <li>
      <p>
       进入容器
      </p>
      <ul>
       <li>
        docker attach container_id(前4位就可)
       </li>
      </ul>
     </li>
     <li>
      <p>
       查看所有容器运行状态
      </p>
      <ul>
       <li>
        docker ps -s
       </li>
      </ul>
     </li>
     <li>
      <p>
       查看所有容器
      </p>
      <ul>
       <li>
        docker ps -a
       </li>
      </ul>
     </li>
     <li>
      <p>
       容器的名称
      </p>
      <ul>
       <li>
        docker run -tid --name 容器名 image_id(前4位就可)
       </li>
      </ul>
     </li>
     <li>
      <p>
       封装新容器
      </p>
      <ul>
       <li>
        docker commit container_id 镜像名：镜像版本号
       </li>
      </ul>
     </li>
     <li>
      <p>
       容器之间的通信
      </p>
      <ul>
       <li>
        <p>
         docker run -tid --name 容器名 --link 被链接的容器 image_id
        </p>
       </li>
       <li>
        <p>
         查看如何通信
        </p>
        <ul>
         <li>
          进入容器 查看/etc/hosts
         </li>
        </ul>
       </li>
      </ul>
     </li>
     <li>
      <p>
       容器的删除
      </p>
      <ul>
       <li>
        docker rm -f container_id(前4位就可)
       </li>
      </ul>
     </li>
    </ul>
