    <h3 id="%E5%89%8D%E8%A8%80">
     前言
    </h3>
    <p>
     规范的注释是一个良好的编程习惯。
    </p>
    <p>
     <a href="https://so.csdn.net/so/search?q=Doxygen&amp;spm=1001.2101.3001.7020" title="Doxygen">
      Doxygen
     </a>
     可以直接将注释提取为程序文档，便于开发人员使用
    </p>
    <p>
     本文主要介绍了Doxygen的使用方法，更多细节请阅读
     <a href="https://www.doxygen.nl/manual/index.html" rel="nofollow" title="官方文档">
      官方文档
     </a>
    </p>
    <p>
     一、Doxygen是什么？
     <br/>
     doxygen是一个文档生成工具，用于为源码生成文档，我们可以在代码中按照doxygen要求的语法编写代码注释，然后使用doxygen将程序中的注释提取出来生成一个文档
    </p>
    <p>
     一般都把注释写在代码中，所以使用doxygen生成的文档也可以非常方便地更新。doxygen还会在文档中引用源码文件，故我们也可以在文档中方便地查看源码
    </p>
    <p>
    </p>

    <h3 id="RgIGi">
     四、Doxygen语法
    </h3>
    <p>
     1.特殊命令
     <br/>
     命令
    </p>
    <p>
     字段名
    </p>
    <p>
     语法
    </p>
    <p>
     @file
    </p>
    <p>
     文件名
    </p>
    <p>
     file [&lt; name &gt;]
    </p>
    <p>
     @brief
    </p>
    <p>
     简介
    </p>
    <p>
     brief { brief description }
    </p>
    <p>
     @author
    </p>
    <p>
     作者
    </p>
    <p>
     author { list of authors }
    </p>
    <p>
     @mainpage
    </p>
    <p>
     主页信息
    </p>
    <p>
     mainpage [(title)]
    </p>
    <p>
     @date
    </p>
    <p>
     年-月-日
    </p>
    <p>
     date { date description }
    </p>
    <p>
     @author
    </p>
    <p>
     版本号
    </p>
    <p>
     version { version number }
    </p>
    <p>
     @copyright
    </p>
    <p>
     版权
    </p>
    <p>
     copyright { copyright description }
    </p>
    <p>
     @param
    </p>
    <p>
     参数
    </p>
    <p>
     param [(dir)] &lt; parameter-name&gt; { parameter description }
    </p>
    <p>
     @return
    </p>
    <p>
     返回
    </p>
    <p>
     return { description of the return value }
    </p>
    <p>
     @retval
    </p>
    <p>
     返回值
    </p>
    <p>
     retval { description }
    </p>
    <p>
     @bug
    </p>
    <p>
     漏洞
    </p>
    <p>
     bug { bug description }
    </p>
    <p>
     @details
    </p>
    <p>
     细节
    </p>
    <p>
     details { detailed description }
    </p>
    <p>
     @pre
    </p>
    <p>
     前提条件
    </p>
    <p>
     pre { description of the precondition }
    </p>
    <p>
     @post
    </p>
    <p>
     说明代码项之后的使用条件
    </p>
    <p>
     { description of the postcondition }
    </p>
    <p>
     @see
    </p>
    <p>
     参考
    </p>
    <p>
     see { references }
    </p>
    <p>
     @link
    </p>
    <p>
     连接(与@see类库，{@link www.google.com})
    </p>
    <p>
     link &lt; link-object&gt;
    </p>
    <p>
     @throw
    </p>
    <p>
     异常描述
    </p>
    <p>
     throw &lt; exception-object&gt; { exception description }
    </p>
    <p>
     @exception
    </p>
    <p>
     对一个异常对象进行注释
    </p>
    <p>
     {exception description}
    </p>
    <p>
     @todo
    </p>
    <p>
     待处理
    </p>
    <p>
     todo { paragraph describing what is to be done }
    </p>
    <p>
     @warning
    </p>
    <p>
     警告信息
    </p>
    <p>
     warning { warning message }
    </p>
    <p>
     @deprecated
    </p>
    <p>
     弃用说明。可用于描述替代方案，预期寿命等
    </p>
    <p>
     deprecated { description }
    </p>
    <p>
     @example
    </p>
    <p>
     弃用说明。可用于描述替代方案，预期寿命等
    </p>
    <p>
     deprecated { description }
    </p>
    <p>
     @code
    </p>
    <p>
     在注释中开始说明一段代码，直到@endcode命令。
    </p>
    <p>
     @endcode
    </p>
    <p>
     注释中代码段的结束
    </p>
    <p>
     @since
    </p>
    <p>
     通常用来说明从什么版本、时间写此部分代码。
    </p>
    <p>
     {text}
    </p>
    <p>
     @attention
    </p>
    <p>
     注意
    </p>
    <p>
     2.注释风格
     <br/>
     (1).文件注释
     <br/>
     注：位于文件开头
    </p>
    <p>
     /**
     <br/>
     * @file 文件名
     <br/>
     * @brief 简介
     <br/>
     * @details 细节
     <br/>
     * @author 作者
     <br/>
     * @version 版本号
     <br/>
     * @date 年-月-日
     <br/>
     * @copyright 版权
     <br/>
     */
     <br/>
     (2).函数注释
     <br/>
     /**
     <br/>
     * @brief 函数描述
     <br/>
     * @param 参数描述
     <br/>
     * @return 返回描述
     <br/>
     * @retval 返回值描述
     <br/>
     */
     <br/>
     (3).常量/变量注释
     <br/>
     一般常量/变量可以有两种形式：
    </p>
    <p>
     常量/变量上一行注释
     <br/>
     常量/变量后注释
     <br/>
     ///定义一个变量
     <br/>
     int a;
     <br/>
     int a; ///&lt; 定义一个变量
    </p>
    <p>
     <br/>
     参考文献：https://blog.csdn.net/LiftHong/article/details/139939151
    </p>
