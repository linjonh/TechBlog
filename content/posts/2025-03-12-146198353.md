    <h3>
     <a id="_10">
     </a>
     解决方案
    </h3>
    <hr/>
    <h4>
     <a id="_13">
     </a>
     临时修改
    </h4>
    <p>
     如果你是临时修改，可以直接运行：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token number">0</span> <span class="token operator">&gt;</span> /proc/sys/net/ipv4/ip_forward
</code></pre>
    <hr/>
    <h4>
     <a id="_21">
     </a>
     永久生效
    </h4>
    <p>
     要永久将
     <code>
      /proc/sys/net/ipv4/ip_forward
     </code>
     置为 0，需要修改
     <code>
      sysctl
     </code>
     配置文件并使其生效。
     <br/>
     <strong>
      步骤：
     </strong>
    </p>
    <ol>
     <li>
      <strong>
       编辑 sysctl 配置文件
      </strong>
      <br/>
      使用文本编辑器修改
      <code>
       /etc/sysctl.conf
      </code>
      文件：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">vi</span> /etc/sysctl.conf
</code></pre>
      找到或添加以下配置：
      <pre><code class="prism language-bash">net.ipv4.ip_forward <span class="token operator">=</span> <span class="token number">0</span>
</code></pre>
     </li>
     <li>
      <strong>
       使配置生效
      </strong>
      <br/>
      运行以下命令，使更改立即生效：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">sysctl</span> <span class="token parameter variable">-p</span>
</code></pre>
     </li>
     <li>
      <strong>
       确保开机自动生效
      </strong>
      <br/>
      sysctl 读取 /etc/sysctl.conf 进行配置，因此上述修改会在每次系统启动时生效，无需额外操作。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_42">
     </a>
     验证配置
    </h4>
    <p>
     你可以使用以下命令确认 ip_forward 是否被成功设置为 0：
    </p>
    <pre><code class="prism language-bash"><span class="token function">cat</span> /proc/sys/net/ipv4/ip_forward
</code></pre>
    <p>
     如果返回 0，说明已生效。
    </p>
    <hr/>
    <h2>
     <a id="_50">
     </a>
     影响
    </h2>
    <ul>
     <li>
      <strong>
       禁用路由功能
      </strong>
      ：服务器无法在不同网卡之间转发流量，影响 NAT、VPN、Docker 等依赖 IP 转发的功能。
     </li>
     <li>
      <strong>
       影响 Docker、Kubernetes
      </strong>
      ：如果使用了 Docker 或 K8s，可能会导致 Pod 之间或容器之间的网络通信失败。
     </li>
     <li>
      <strong>
       影响 VPN 服务
      </strong>
      ：如 OpenVPN、IPSec VPN 需要 IP 转发，禁用后可能导致 VPN 连接异常。
     </li>
     <li>
      <strong>
       影响网关服务器
      </strong>
      ：如果服务器用作网关，禁用 IP 转发会阻止内部网络通过该服务器访问外部网络。
     </li>
    </ul>
    <p>
     如果你有 NAT 或 VPN 需求，建议保持 ip_forward=1，否则请确保该更改不会影响你的网络环境。
    </p>
