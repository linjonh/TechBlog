    <p>
     <code>
      @synthesize
     </code>
     和
     <code>
      @dynamic
     </code>
     是 Objective-C 中用于控制属性合成的两个关键字，它们的作用和用法有显著区别。以下是它们的详细说明：
    </p>
    <hr/>
    <h4>
     <a id="1_synthesize_4">
     </a>
     <strong>
      1.
      <code>
       @synthesize
      </code>
     </strong>
    </h4>
    <p>
     <code>
      @synthesize
     </code>
     用于显式声明编译器为属性生成对应的实例变量（ivar）和访问器方法（getter 和 setter）。
    </p>
    <h5>
     <a id="_7">
     </a>
     <strong>
      作用：
     </strong>
    </h5>
    <ul>
     <li>
      告诉编译器自动生成属性的 getter 和 setter 方法。
     </li>
     <li>
      如果未显式指定实例变量名，编译器会生成一个默认的实例变量（通常以下划线
      <code>
       _
      </code>
      开头，例如
      <code>
       _propertyName
      </code>
      ）。
     </li>
     <li>
      可以自定义实例变量的名称。
     </li>
    </ul>
    <h5>
     <a id="_12">
     </a>
     <strong>
      语法：
     </strong>
    </h5>
    <pre><code class="prism language-objc"><span class="token keyword">@synthesize</span> propertyName <span class="token operator">=</span> _instanceVariableName<span class="token punctuation">;</span>
</code></pre>
    <h5>
     <a id="_17">
     </a>
     <strong>
      示例：
     </strong>
    </h5>
    <pre><code class="prism language-objc"><span class="token keyword">@interface</span> MyClass <span class="token punctuation">:</span> NSObject
<span class="token keyword">@property</span> <span class="token punctuation">(</span>nonatomic<span class="token punctuation">,</span> strong<span class="token punctuation">)</span> NSString <span class="token operator">*</span>name<span class="token punctuation">;</span>
<span class="token keyword">@end</span>

<span class="token keyword">@implementation</span> MyClass
<span class="token keyword">@synthesize</span> name <span class="token operator">=</span> _customName<span class="token punctuation">;</span> <span class="token comment">// 自定义实例变量名</span>
<span class="token keyword">@end</span>
</code></pre>
    <ul>
     <li>
      如果没有显式使用
      <code>
       @synthesize
      </code>
      ，编译器会自动生成以下代码：
      <pre><code class="prism language-objc"><span class="token keyword">@synthesize</span> name <span class="token operator">=</span> _name<span class="token punctuation">;</span>
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="_33">
     </a>
     <strong>
      使用场景：
     </strong>
    </h5>
    <ul>
     <li>
      需要自定义实例变量名时。
     </li>
     <li>
      在早期的 Objective-C 版本中（Xcode 4.4 之前），必须显式使用
      <code>
       @synthesize
      </code>
      来生成访问器方法。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_dynamic_39">
     </a>
     <strong>
      2.
      <code>
       @dynamic
      </code>
     </strong>
    </h4>
    <p>
     <code>
      @dynamic
     </code>
     用于告诉编译器不要为属性生成实例变量和访问器方法，开发者会在运行时动态提供这些方法的实现。
    </p>
    <h5>
     <a id="_42">
     </a>
     <strong>
      作用：
     </strong>
    </h5>
    <ul>
     <li>
      禁止编译器自动生成 getter 和 setter 方法。
     </li>
     <li>
      开发者需要在运行时通过其他方式（如消息转发机制）提供这些方法的实现。
     </li>
     <li>
      常用于 Core Data 或动态方法解析的场景。
     </li>
    </ul>
    <h5>
     <a id="_47">
     </a>
     <strong>
      语法：
     </strong>
    </h5>
    <pre><code class="prism language-objc"><span class="token keyword">@dynamic</span> propertyName<span class="token punctuation">;</span>
</code></pre>
    <h5>
     <a id="_52">
     </a>
     <strong>
      示例：
     </strong>
    </h5>
    <pre><code class="prism language-objc"><span class="token keyword">@interface</span> MyClass <span class="token punctuation">:</span> NSObject
<span class="token keyword">@property</span> <span class="token punctuation">(</span>nonatomic<span class="token punctuation">,</span> strong<span class="token punctuation">)</span> NSString <span class="token operator">*</span>name<span class="token punctuation">;</span>
<span class="token keyword">@end</span>

<span class="token keyword">@implementation</span> MyClass
<span class="token keyword">@dynamic</span> name<span class="token punctuation">;</span> <span class="token comment">// 告诉编译器不要生成 getter 和 setter</span>

<span class="token comment">// 在运行时动态解析方法</span>
<span class="token operator">-</span> <span class="token punctuation">(</span>id<span class="token punctuation">)</span>forwardingTargetForSelector<span class="token punctuation">:</span><span class="token punctuation">(</span>SEL<span class="token punctuation">)</span>aSelector <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">if</span> <span class="token punctuation">(</span>aSelector <span class="token operator">==</span> <span class="token keyword">@selector</span><span class="token punctuation">(</span>setName<span class="token punctuation">:</span><span class="token punctuation">)</span> <span class="token operator">||</span> aSelector <span class="token operator">==</span> <span class="token keyword">@selector</span><span class="token punctuation">(</span>name<span class="token punctuation">)</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        <span class="token keyword">return</span> someOtherObject<span class="token punctuation">;</span> <span class="token comment">// 将方法调用转发给其他对象</span>
    <span class="token punctuation">}</span>
    <span class="token keyword">return</span> <span class="token punctuation">[</span><span class="token keyword">super</span> forwardingTargetForSelector<span class="token punctuation">:</span>aSelector<span class="token punctuation">]</span><span class="token punctuation">;</span>
<span class="token punctuation">}</span>
<span class="token keyword">@end</span>
</code></pre>
    <h5>
     <a id="_71">
     </a>
     <strong>
      使用场景：
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       Core Data
      </strong>
      ：Core Data 会为
      <code>
       NSManagedObject
      </code>
      子类的属性动态生成访问器方法。
     </li>
     <li>
      <strong>
       动态方法解析
      </strong>
      ：在运行时通过
      <code>
       resolveInstanceMethod:
      </code>
      或消息转发机制动态提供方法实现。
     </li>
     <li>
      <strong>
       桥接其他语言或框架
      </strong>
      ：例如通过桥接方式调用 Swift 或 C 代码。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3__78">

     <strong>
      3. 区别对比
     </strong>
    </h4>

       </td>

      <tr>
       <td>
        <strong>

      <tr>
       <td>

       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     <a id="4__90">
     </a>
     <strong>
      4. 默认行为
     </strong>
    </h4>
    <ul>
     <li>
      在现代 Objective-C 中（Xcode 4.4 及以后），如果没有显式使用
      <code>
       @synthesize
      </code>
      或
      <code>
       @dynamic
      </code>
      ，编译器会默认生成以下代码：
      <pre><code class="prism language-objc"><span class="token keyword">@synthesize</span> propertyName <span class="token operator">=</span> _propertyName<span class="token punctuation">;</span>
</code></pre>
      即自动生成实例变量和访问器方法。
     </li>
    </ul>
    <hr/>
    <h4>

     <strong>
      5. 示例对比

    <hr/>
    <h4>
     <a id="6__148">
     </a>
     <strong>
      6. 总结
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       <code>
        @synthesize
       </code>
      </strong>
      ：用于显式生成属性的实例变量和访问器方法，适合普通属性或需要自定义实例变量名的场景。
     </li>
     <li>
      <strong>
       <code>
        @dynamic
       </code>
      </strong>
      ：用于禁止编译器生成访问器方法，适合需要在运行时动态提供方法实现的场景（如 Core Data 或消息转发）。
     </li>
    </ul>
