    <h2>
     1.
     <strong>
      泛型编程：编写与类型无关的通用代码，是代码复用的一种手段。模板是泛型编程的基础
     </strong>
    </h2>
    <p>
    </p>
    <h2>
     2.函数模板的格式
    </h2>
    <blockquote>
     <p>
      template&lt;typename T1,typename T2,…,typename Tn&gt;
      <br/>
      返回类型 函数名(参数列表)
      <br/>
      {
      <!-- -->
      <br/>
      //函数体
      <br/>
      }
     </p>
    </blockquote>
    <h2>
     3.template&lt;class T1,class T2,…,class Tn&gt;
    </h2>
    <blockquote>
     <p>
      template&lt;class T1,class T2,…,class Tn&gt;
     </p>
     <p>
      class 类模板名
      <br/>
      {
      <!-- -->
      <br/>
      //类内成员声明
      <br/>
      };
     </p>
    </blockquote>
    <h2>
     <strong>
      4.
     </strong>
     <strong>
      使用模板时，编译器一般不会进行类型转换操作
     </strong>
    </h2>
    <p>
     以下代码将不能通过编译：
    </p>
    <pre><code class="hljs">T Add(const T&amp; x, const T&amp; y)
{
	return x + y;
}
int main()
{
    int a = 10;
    double b = 1.1;
    int c = Add(a, b);
	return 0;
}
</code></pre>
    <blockquote>
     <p>
      <br/>
      因为在编译期间，编译器根据实参推演模板参数的实际类型时，根据实参a将T推演为int，根据实参b将T推演为double，但是模板参数列表中只有一个T，编译器无法确定此处应该将T确定为int还是double。
     </p>
    </blockquote>
    <h2>
     5.类模板中的成员函数若是放在类外定义时，需要加模板参数列表
    </h2>
    <pre><code class="hljs">template&lt;class T&gt;
class Date
{
public:
	void Print();
private:
	T _year;
	T _month;
	T _day;
};
//类模板中的成员函数在类外定义，需要加模板参数列表
template&lt;class T&gt;
void Score&lt;T&gt;::Print()
{
	cout &lt;&lt; _year &lt;&lt; "-" &lt;&lt;  _month &lt;&lt; "-"&lt;&lt; _day &lt;&lt; endl;
}
</code></pre>
    <h2>
     6.类模板不支持分离编译，即声明在xxx.hpp文件中，而定义却在xxx.cpp文件中
