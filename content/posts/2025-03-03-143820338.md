     <div style="margin-left:auto">
      <p style="text-align:left">
       红外传感器由一个红外发射管  和一个红外接收管组成。LED 发出的红外光线照射到表面后反射回接收管输出相应的数值，亮表面”的数值较高，“黑/暗表面”的数值较低。
      </p>
     </div>
    </div>
    <div style="margin-left:auto; text-align:left">
     <p style="margin-left:0">
      对于使用的传感器，模块上的集成电路会生成一个简单的数字信号作为输出（高：暗；低：亮）。安装在模块上的电位器（见图）将调整正确的光级，以被视为“暗”或“亮”。它的工作原理是，当反射光颜色为黑色/暗时，其输出端会产生高（“1”）数字电平，而对于另一种较浅的颜色，则会产生低（“0”）。我在这里使用了一个带有 4 个传感器的集成模块和一个带有单个传感器的额外模块（形状不同，但逻辑相同）。我发现这种组合是 5 个传感器的阵列，可以实现良好而流畅的控制，如下所述。
     </p>
     <p style="margin-left:0">
      5 个传感器阵列的安装方式是，如果只有一个传感器相对于黑线居中，则只有该特定传感器会产生高电平。另一方面，应计算传感器之间的空间，以允许 2 个传感器同时覆盖黑线的整个宽度，并在两个传感器上产生高电平。
     </p>
     <p style="margin-left:0">
      沿线行驶时可能的传感器阵列输出为：
     </p>
     <ul style="margin-left:0">
      <li>
       0 0 0 0 1//偏左
      </li>
      <li>
       0 0 0 1 1//偏左
      </li>
      <li>
       0 0 0 1 0//偏左
      </li>
      <li>
       0 0 1 1 0//偏左
      </li>
      <li>
       0 0 1 0 0//居中
      </li>
      <li>
       0 1 1 0 0//偏右
      </li>
      <li>
       0 1 0 0 0//偏右
      </li>
      <li>
       1 1 0 0 0//偏右
      </li>
      <li>
       1 0 0 0 0//偏右
      </li>
     </ul>
     <p style="margin-left:0">
      拥有 5 个传感器，可以生成“error 变量”，这将有助于确定机器人在线上的位置，如下所示。
     </p>
     <p style="margin-left:0">
      假设黑线刚好位于机器人“中间传感器”（sersor 2）下方。数组的输出将是：0 0 1 0 0，在这种情况下，“error”将为“零”，也就是说没有误差。 与传感器状态相关的error变量值为：
     </p>
     <ul style="margin-left:0">
      <li>
       0 0 0 0 1 ==&gt; error= 4
      </li>
      <li>
       0 0 0 1 1 ==&gt; error = 3
      </li>
      <li>
       0 0 0 1 0 ==&gt; error = 2
      </li>
      <li>
       0 0 1 1 0 ==&gt; error = 1
      </li>
      <li>
       <strong>
        0 0 1 0 0 ==&gt;
       </strong>
       error
       <strong>
        = 0  正好处在中间传感器sersor2下方
       </strong>
      </li>
      <li>
       0 1 1 0 0 ==&gt; error = -1
      </li>
      <li>
       0 1 0 0 0 ==&gt; error = -2
      </li>
      <li>
       1 1 0 0 0 ==&gt; error = -3
      </li>
      <li>
       1 0 0 0 0 ==&gt; error = -4
      </li>

    <div style="margin-left:auto; text-align:left">
     <p style="margin-left:0">
      此时，我们的机器人已组装完毕并可运行。您应该对电机进行一些基本测试，读取传感器的输出并通过线路进行测试。下面我们将实现一个控制逻辑，以确保机器人能够一直沿着线路运行。
     </p>
     <p style="margin-left:0">
      假设机器人正在跑过一条线，传感器阵列输出为：
      <strong>
       “0 0 1 0 0 ”
      </strong>
      。对应的误差为“0”。在这种情况下，两个电机都应该以恒定速度向前运行。
     </p>
     <p style="margin-left:0">
      例如：
      <strong>
       <em>
        iniMotorSpeed = 250
       </em>
      </strong>
      ；表示左舵机将接收 1250us 的脉冲右舵机将接收 1750us 的脉冲。使用这些参数，机器人将以半速向前移动。
     </p>
     <pre style="text-align:center">rightServo.writeMicroseconds(1500 + iniMotorPower);
leftServo.writeMicroseconds(1500 - iniMotorPower);</pre>
     <p style="margin-left:0">
      现在假设机器人行驶中传感器2和3检测到黑色，数组输出将为：
      <strong>
       “0 0 1 1 0”
      </strong>
      ，
      <strong>
       <em>
        error= 1。也就是小车偏左了，
       </em>
      </strong>
      在这种情况下，您需要将机器人向右转。要做到这一点，您必须降低右侧舵机的速度，这意味着减少脉冲的长度。此外，必须增加左侧舵机的速度，这意味着减少左侧舵机脉冲的长度。为此，我们需要更改电机控制功能：
     </p>
     <pre style="margin-left:0">rightServo.writeMicroseconds(1500 + iniMotorPower - error); ==&gt; 正误差：降低速度
leftServo.writeMicroseconds(1500 - iniMotorPower - error); ==&gt; 正误差：增加速度
</pre>
     <p style="margin-left:0">
      上述逻辑是正确的，但很容易理解的是，在脉冲长度上增加或减去“1”不会产生所需的校正。下面我们需要将error调整成和速度相当的一个值，以便使用它来调整机器人两个舵机的速度。我们使用的方法是将“error”乘以一个常数（我们称之为“K”），将error映射成一个与速度相当的值。这个用来“放大error”的值我们定义成Kp。
     </p>
     <p style="margin-left:0">
      int Kp = 50;
     </p>
     <p style="margin-left:0">
      rightServo.writeMicroseconds（1500 + iniMotorPower-Kp*error）;
     </p>
     <p style="margin-left:0">
      leftServo.writeMicroseconds（1500-iniMotorPower-Kp*error）;
     </p>
     <p style="margin-left:0">
      传感器阵列的不同情况对应两个舵机的速度例如：
     </p>
     <ul style="margin-left:0">
      <li>
       传感器阵列：
       <strong>
        0 0 1 0 0
       </strong>
       ==&gt;error
       <strong>
        = 0
       </strong>
       ==&gt; 右舵机脉冲长度 = 1500+250-50*0=1,750us ==&gt; 左舵机脉冲长度 = 1500-250+50*0=1250us（两个电机速度相同）
      </li>
      <li>
       传感器阵列：
       <strong>
        0 0 1 1 0
       </strong>
       ==&gt;error
       <strong>
        = 1
       </strong>
       ==&gt; 右舵机脉冲长度 = 1,700us（较慢）==&gt; 左舵机脉冲长度 = 1,200us（较快）
      </li>
     </ul>
     <p style="margin-left:0">
      如果情况相反，机器人向右转，则误差将为“负”，并且舵机的速度应该改变：
     </p>
     <ul style="margin-left:0">
      <li>
       传感器阵列：
       <strong>
        0 0 1 0 0
       </strong>
       ==&gt;error
       <strong>
        = 0
       </strong>
       ==&gt; 右舵机脉冲长度 = 1,750us ==&gt; 左舵机脉冲长度 = 1,250us（两个电机速度相同）
      </li>
      <li>
       传感器阵列：
       <strong>
        0 1 1 0 0
       </strong>
       ==&gt;error
       <strong>
        = -1
       </strong>
       ==&gt; 右舵机脉冲长度 = 1,800us（较快）==&gt; 左舵机脉冲长度 = 1,300us（较慢）
      </li>
     </ul>
     <p style="margin-left:0">
      此时很明显，机器人被推向一侧的程度越大，误差就越大，它必须
      <strong>
       更快
      </strong>
      地返回中心。机器人对误差的反应速度与误差成正比。这称为
      <strong>
       “比例控制”
      </strong>
      ，即更复杂的控制网络 PDI（比例、微分、积分）的“P”组件。
     </p>
    </div>

     <div>
      <div style="margin-left:auto">
       <div>

    <div style="margin-left:auto; text-align:left">
     <p style="margin-left:0">
      如果您想跳过此部分，也可以。您可以继续使用上一步中解释的比例控制，或者花点脑子在机器人中实现更复杂的控制系统，这是您的选择。
     </p>
     <p style="margin-left:0">
      PID（比例、积分和微分）是最常见的控制方案之一。大多数工业控制回路都使用某种形式的 PID 控制。有许多方法可以调整 PID 回路，包括本例中使用的手动技术。如果你想更加详细的了解PID可以看
      <a class="link-info" href="https://blog.csdn.net/sxstj/article/details/144701519?spm=1001.2014.3001.5501" title="Arduino PID 控制教程">
       Arduino PID 控制教程
      </a>
      。
     </p>
     <p style="margin-left:0">
      在巡线机器人的运行中如果一旦机器人与线发生了偏差，我们就要让他以最快的速度回到线上来，而不会等待误差的积累后再处理，所以需要微分项D来调整，而积分项I在巡线机器人中一般是没有用处的，一般为了保证公式的完整性我们设置KI=0。
     </p>
     <pre>void calculatePID()
{
  P = error;
  I = I + error;
  D = error-previousError;
  PIDvalue = (Kp*P) + (Ki*I) + (Kd*D);
  previousError = error;
}</pre>
     <pre>void motorPIDcontrol()
{
  int leftMotorSpeed = 1500 - iniMotorPower - PIDvalue;
  int rightMotorSpeed = 1500 + iniMotorPower - PIDvalue;
  
  leftServo.writeMicroseconds(leftMotorSpeed);
  rightServo.writeMicroseconds(rightMotorSpeed);
}</pre>

     <p style="margin-left:0">
      循环程序如下：
     </p>
     <pre style="margin-left:0">void loop ()

     <p style="margin-left:0">
      但为了实现更完整、更真实的操作，让我们引入一个新变量：“mode”。我们将为该变量定义 3 个状态：
     </p>
     <p style="margin-left:0">
      模式：
     </p>
     <ul style="margin-left:0">
      <li>
       #define STOPPED 0        //停止
      </li>
      <li>
       #define FOLLOWING_LINE 1    //巡线状态
      </li>
      <li>
       #define NO_LINE 2        //小车脱离了线
      </li>
     </ul>
     <p style="margin-left:0">
      如果所有传感器都发现黑线，则传感器阵列输出将为：1 1 1 1 1。在这种情况下，我们可以将模式定义为“停止”，机器人应执行“完全停止”。
     </p>
     <pre>if((LFSensor[0]== 1 )&amp;&amp;(LFSensor[1]== 1 )&amp;&amp;(LFSensor[2]== 1 )&amp;&amp;(LFSensor[3]== 1 )&amp;&amp;(LFSensor[4]== 1 )) 
{ 
  mode = STOPPED;
}</pre>
     <p style="margin-left:0">
      跟随线机器人的其他常见情况是发现“NO_LINE”也就是脱离了线，传感器检测不到黑线，传感器阵列输出为：0 0 0 0 0。在这种情况下，我们可以对其进行编程，使其原地旋转 ，直到找到线并恢复正常的巡线状态。
     </p>
     <pre>else if((LFSensor[0]== 0 )&amp;&amp;(LFSensor[1]== 0 )&amp;&amp;(LFSensor[2]== 0 )&amp;&amp;(LFSensor[3]== 0 )&amp;&amp;(LFSensor[4]== 0 )) 
{ 
  mode = NO_LINE;
}</pre>
     <p style="margin-left:0">
      完整的loop()将是：
     </p>
     <pre style="margin-left:0">void loop() 
{ 
  readLFSsensors(); 
  switch (mode) 
  { 
    case STOPPED: 
       motorStop(); 
       break; 
    case NO_LINE: 
       motorStop(); 
       motorTurn(LEFT, 180); 
       break; 
    case FOLLOWING_LINE: 
       calculatePID(); 
       motorPIDcontrol(); 
       break; 
  } 
}</pre>
