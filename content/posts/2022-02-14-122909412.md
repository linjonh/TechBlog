     <em>
      <strong>
       前言
      </strong>
      ：前段时间学习了软件测试技术课程，课程作业要求对某个网站进行测试，但因为自己只对单元测试与移动应用测试比较熟悉，对 Web 应用测试不甚了解，因此最近自学了一下 Web 自动化测试技术。由于找到的在线资源大多已经过时，自己在学习的过程中还是碰到了很多问题，所以在此记录一下过程。

    <h2>
     <a id="Selenium__3">
     </a>
     Selenium 安装与使用
    </h2>
    <h3>
     <a id="Selenium__4">
     </a>
     Selenium 介绍
    </h3>
    <p>
     Selenium 是一个用于 Web 应用程序测试的工具，是目前最广泛使用的开源 Web UI 自动化测试套件之一。Selenium 支持多种浏览器与编程语言，提供了大量函数对网页元素进行定位和操作。
    </p>
    <h3>
     <a id="Selenium__6">
     </a>
     Selenium 安装
    </h3>
    <p>
     进入 Selenium 的下载官网：https://www.selenium.dev/downloads/，在 Selenium Server (Grid) 右边点击最新版本下载即可。个人在此遇到了一个问题，就是最新版本的 Selenium 并不适合自己的 Eclipse 版本，在后续的配置中一直报类似于“NoModuleFound”的错误，因此在下方的 Previous Releases 中选择了较老的 3.14.0 版本，如下：
    </p>
    <p>
     <img alt="Selenium 安装" src="https://i-blog.csdnimg.cn/blog_migrate/ca1fcf00617a52afdb43e50bc532ed3a.png#pic_center" width="700">
      <br/>
      下载后得到一个 selenium-java-3.14.0.zip 文件，将其解压即可。
     </img>
    </p>
    <h3>
     <a id="Selenium__12">
     </a>
     Selenium 配置
    </h3>
    <p>
     接下来在 Eclipse中 添加 Selenium 插件，方法如下：
    </p>
    <ol>
     <li>
      <p>
       右击项目（在个人的例子中，项目名为 WebTest）→ Build Path → Configure Build Path → Libraries 选项卡，进入如下界面；
      </p>
     </li>
     <li>
      <p>
       点击 Classpath，再点击右侧的 Add External Jars，将刚才解压的 selenium-java-3.14.0 下 libs 文件夹中的 jar 文件，以及 selenium-java-3.14.0 下的 jar 文件，全部添加到 Classpath 中（
       <strong>
        注意是根目录以及 libs 目录下所有的 jar 文件
       </strong>
       ），完成后点击 Apply and Close 关闭。
      </p>
     </li>
    </ol>
    <p>
     <img alt="Selenium 配置" src="https://i-blog.csdnimg.cn/blog_migrate/ff84cca9aea0a29489e7b101c5262288.png#pic_center" width="700">
      <br/>
      配置成功后，可看到项目目录下出现了 Reference Libraries，可以看到 selenium-java-3.14.0 中的所有 jar 文件都在其中：
     </img>
    </p>
    <p>
     <img alt="参考库" src="https://i-blog.csdnimg.cn/blog_migrate/54cbf681ec0585ccbcbaa4c85c3989a9.png#pic_center" width="300">
      <br/>
      可以通过以下方式简单验证是否配置成功：新建任意 class 文件，输入代码：
     </img>
    </p>
    <pre><code class="prism language-java"><span class="token keyword">import</span> <span class="token namespace">org<span class="token punctuation">.</span>openqa<span class="token punctuation">.</span>selenium<span class="token punctuation">.</span></span><span class="token class-name">WebElement</span><span class="token punctuation">;</span>
</code></pre>
    <p>
     如果输入时 Eclipse 提供补全功能，并且运行能成功导入不报错，那么就配置成功了。
    </p>
    <h3>
     <a id="Selenium__28">
     </a>
     Selenium 使用
    </h3>
    <p>
     接下来以 Edge 浏览器为例介绍 Selenium 的使用。
    </p>
    <h4>
     <a id="_30">
     </a>
     浏览器驱动
    </h4>
    <p>
     首先需要下载浏览器驱动，用于 Web 测试，步骤如下：
    </p>
    <ol>
     <li>
      点击 Edge 浏览器右上角 … → 帮助与反馈 → 关于 Microsoft Edge，查看自己的 Edge 浏览器版本号；
     </li>
     <li>
      进入驱动下载网站：https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/，选择对应版本的驱动下载即可，下载后解压，会得到一个 msedgedriver.exe 文件。
     </li>
    </ol>

<h2>
<a id="TestNG__89">
</a>
TestNG 安装与使用
</h2>
<h3>
<a id="TestNG__90">
</a>
TestNG 介绍
</h3>
<p>
TestNG 是一个测试框架，继承自单元测试框架 Junit 。其功能更加强大，提供包括单元测试、功能测试、集成测试等功能。TestNG 提供各种方法注释，并支持多线程执行测试用例、数据驱动等功能。
</p>
<h3>
<a id="TestNG__92">
</a>
TestNG 安装与配置
</h3>
<p>
在 Eclipse 中按如下安装并配置 TestNG：
</p>
<ol>
<li>
在 Eclipse 上方选择 Help → Install New Software，进入如下界面；
</li>
<li>
点击 Add，在 Name 中输入 TestNG，Location 中输入 https://testng.org/testng-eclipse-update-site，点击 Add 后一路 Next 即可。
</li>
</ol>
<p>
<img alt="TestNG 安装" src="https://i-blog.csdnimg.cn/blog_migrate/a79986eceb2e38683ce5f05ed9756f3c.png#pic_center">
<br/>
验证是否安装成功的方式很简单，如下图所示：
</img>
</p>
<ol>
<li>
新建文件 → 选择 Others → TestNG → TestNG class → Next 创建。
</li>
</ol>
<p>
<img alt="TestNG 验证" src="https://i-blog.csdnimg.cn/blog_migrate/03b900848e020ecd8eea8f81bb084181.png#pic_center" width="400">
<br/>
如果能成功创建，则说明安装成功。
</img>
</p>

<p>
其中，由 @Test 注释的方法就是需要执行的测试用例，最后通过 assertEquals() 函数验证实际值与预期值。每个测试用例可对应一个 @Test 方法。由 @BeforeClass/@AfterClass 注释的方法则是分别是在当前测试类的第一个测试方法执行之前运行、最后一个测试方法执行之后运行的方法。
</p>
