    <h2 id="E684g">
     1. 项目介绍
    </h2>
    <p id="u50b1cd4f">
     <strong>
      JSFScan
     </strong>
     是一款专为
     <strong>
      Bug Bounty
     </strong>
     和
     <strong>
      渗透测试
     </strong>
     设计的自动化工具，专注于对目标网站的
     <strong>
      JavaScript 文件
     </strong>
     进行深度侦察。其通过多维度分析 JS 文件，快速提取敏感信息、潜在漏洞点和攻击面，适用于红队渗透、漏洞挖掘及资产测绘等场景。
    </p>
    <p id="u487fd58c">
     <strong>
      核心功能亮点
     </strong>
     ：
    </p>
    <ol>
     <li id="uadf6af3e">
      <strong>
       JS 文件收集
      </strong>
      ：从目标子域名列表自动爬取所有关联的 JS 文件链接（支持 Hakrawler、Gau 等工具集成）
     </li>
     <li id="uf4d94f9d">
      <strong>
       端点提取
      </strong>
      ：解析 JS 文件中的 API 接口、隐藏路径及 URL 端点，为漏洞探测提供关键数据。
     </li>
     <li id="u23f2172f">
      <strong>
       敏感信息发现
      </strong>
      ：检测 JS 中的硬编码密钥（如 AWS Access Key、API Token）、调试接口等。
     </li>
     <li id="u7a12e55b">
      <strong>
       XSS 探测
      </strong>
      ：识别潜在的 DOM XSS 漏洞点，并提取变量名辅助手动验证。
     </li>
     <li id="u3da7fd57">
      <strong>
       数据整合
      </strong>
      ：生成结构化报告（HTML/文本）、自定义词表，支持本地存储 JS 文件供后续分析。
     </li>
    </ol>
    <p id="u3637bff0">
     <strong>
      适用场景
     </strong>
     ：
    </p>
    <ul>
     <li id="u76d5a724">
      <strong>
       红队渗透
      </strong>
      ：快速定位 Web 应用的未授权接口或敏感数据泄露点。
     </li>
     <li id="u7477d38e">
      <strong>
       漏洞挖掘
      </strong>
      ：通过 JS 文件中的隐藏路径发现 SQL 注入、SSRF 等漏洞。
     </li>
     <li id="uc49b165c">
      <strong>
       资产测绘
      </strong>
      ：梳理企业 Web 应用的 JavaScript 依赖库及第三方服务。
     </li>
    </ul>
    <h2 id="Wznpj">
     2. 安装教程
    </h2>
    <h3 id="zzr6O">
     2.1. 环境要求
    </h3>
    <ul>
     <li id="u602418c8">
      <strong>
       操作系统
      </strong>
      ：Linux/macOS（推荐 Kali Linux 或 Ubuntu）。
     </li>
     <li id="ufdf1f8bc">
      <strong>
       依赖工具
      </strong>
      ：Go 1.16+、Docker（可选）、Python 3.6+。
     </li>
    </ul>
    <h3 id="eOLjA">
     2.2. 安装方法
    </h3>
    <p id="uf2d3bcab">
     <strong>
      方法一：本地安装（推荐）
     </strong>
    </p>
    <pre id="U06iQ"><code>git clone https://github.com/KathanP19/JSFScan.sh  
cd JSFScan.sh  
sudo chmod +x install.sh  
./install.sh  # 自动安装 Hakrawler、Gau、SecretFinder 等依赖  </code></pre>
    <p id="u27f27966">
     <strong>
      注意
     </strong>
     ：若遇到权限问题，确保已正确配置 Go 环境变量
    </p>
    <p id="u6d3e9836">
     <strong>
      方法二：Docker 容器
     </strong>
    </p>
    <pre id="MQm2A"><code>docker build -t jsfscan .  
docker run -it jsfscan /bin/bash  # 进入容器环境  </code></pre>
    <p id="u3ad3cb07">
     <strong>
      优势
     </strong>
     ：避免依赖冲突，适合快速部署
    </p>
    <h2 id="FIvxa">
     3. 基础使用教程
    </h2>
    <h3 id="mpTk6">
     3.1. 准备目标列表
    </h3>
    <ul>
     <li id="ud0a52dbc">
      创建
      <code>
       targets.txt
      </code>
      文件，每行一个子域名（需包含协议头）：
     </li>
    </ul>
    <pre id="LShzU"><code>https://example.com  
http://test.example.com  </code></pre>
    <h3 id="rQcgk">
     3.2. 常用命令示例
    </h3>
    <ol>
     <li id="ueb8f3bed">
      <strong>
       收集 JS 文件链接
      </strong>
      ：
     </li>
    </ol>
    <pre id="rFUg4"><code>./JSFScan.sh -l -i targets.txt -o output  </code></pre>
    <ul>
     <li>
      <ul>
       <li id="u7d7ac978">
        <code>
         -l
        </code>
        ：启用 JS 链接收集模块。
       </li>
       <li id="ubb76f7ba">
        <code>
         -o
        </code>
        ：指定输出目录。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="u9da7d2d9">
      <strong>
       提取端点与敏感信息
      </strong>
      ：
     </li>
    </ol>
    <pre id="ppSrA"><code>./JSFScan.sh -e -s -i targets.txt  </code></pre>
    <ul>
     <li>
      <ul>
       <li id="u2bcf68b3">
        <code>
         -e
        </code>
        ：提取 JS 中的 URL 端点。
       </li>
       <li id="u9b596a60">
        <code>
         -s
        </code>
        ：扫描硬编码密钥（如 AWS Key、数据库凭证）。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="u9472c9f0">
      <strong>
       生成 HTML 报告
      </strong>
      ：
     </li>
    </ol>
    <pre id="UsIUq"><code>./JSFScan.sh --all -i targets.txt -r report.html  </code></pre>
    <ul>
     <li>
      <ul>
       <li id="u58dad678">
        <code>
         --all
        </code>
        ：启用全部功能模块。
       </li>
       <li id="ua39d518c">
        <code>
         -r
        </code>
        ：生成可视化报告
       </li>
      </ul>
     </li>
    </ul>
    <h2 id="v545g">
     4. 进阶使用教程
    </h2>
    <h3 id="ntC1R">
     4.1. 自定义 Cookie 与请求头
    </h3>
    <ul>
     <li id="u41347788">
      编辑脚本文件，修改 Hakrawler 的请求参数（如添加认证 Cookie）：
     </li>
    </ul>
    <pre id="QJSua"><code># 修改 JSFScan.sh 第 23 行  
hakrawler -js -cookie "session=123abc" -depth 3  </code></pre>
    <h3 id="DFEBA">
     4.2. 生成自定义词表
    </h3>
    <ul>
     <li id="u6be5472d">
      从 JS 文件中提取高频词汇，用于目录爆破或密码猜测：
     </li>
    </ul>
    <pre id="QD0Wu"><code> ./JSFScan.sh -w -i targets.txt -o custom_wordlist.txt  </code></pre>
    <h3 id="mBTx7">
     4.3. 变量名提取与 XSS 检测
    </h3>
    <ul>
     <li id="u0f07420e">
      提取 JS 变量名辅助 XSS 漏洞挖掘：
     </li>
    </ul>
    <pre id="Ybrxl"><code> ./JSFScan.sh -v -d -i targets.txt  </code></pre>
    <ul>
     <li>
      <ul>
       <li id="u3e7f6ded">
        <code>
         -v
        </code>
        ：提取变量名（如
        <code>
         userInput
        </code>
        、
        <code>
         redirectUrl
        </code>
        ）。
       </li>
       <li id="ufdbac871">
        <code>
         -d
        </code>
        ：扫描潜在的 DOM XSS 漏洞点
       </li>
      </ul>
     </li>
    </ul>
    <h3 id="xLlHh">
     4.4. 联动其他工具
    </h3>
    <ul>
     <li id="u524746f4">
      将提取的端点导入
      <strong>
       Burp Suite
      </strong>
      或
      <strong>
       Nuclei
      </strong>
      进行深度测试：
     </li>
    </ul>
    <pre id="noPIz"><code> cat output/endpoints.txt | nuclei -t ~/nuclei-templates/  </code></pre>
