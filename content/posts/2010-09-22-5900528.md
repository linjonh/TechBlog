# Ubuntu硬件峰会，软硬件开发者大团圆

<!-- @page { margin: 2cm } P { margin-bottom: 0.21cm } -->

9



月



25



日整整一天（上午



9



点开始），在台北市维多利亚（



Grand Victoria



）大酒店举行一年一度的



Ubuntu



硬件峰会（



UHS



），今年是第二届。为什么说这次峰会是软硬件开发者的“大团圆”聚会？这与这次大会的性质及其热烈的氛围有关。

今年全球


SFD


活动的主要赞助商是


Canonical


公司，该公司也是


Ubuntu


发行版及其国际开发社区的主要赞助商。去年的台北


UHS


峰会也是在维多利亚大酒店（三层


Victoria Ballroom


）举行的，峰会的规格绝对是超一流的，峰会的学术水平也是世界一流的，此事震动了整个台湾业界。按理说，这种会议的注册费不会很低，除了会议场地费以外，光是在维多利亚大酒店吃一顿会议午餐的费用就不会很少。这一切费用都由


Canonical


全部“包圆儿”了，


UHS


会议注册费全免，包括午餐、饮料和甜食等费用全免。谁不想去呢？别急，不是什么人都能注册参与此次会议，与会者必须是


ODM


和


OEM


以及一些嵌入式设备制造商的产品主管和工程技术人员才行，任何不相干的人要想钻进去蹭一顿饭吃那可不行。（注：今年


UHS


注册人数现已经超过


180


人。）

言归正传。今年已经决定参加


UHS


峰会的有：



American Megatrends Inc. (AMI), Phoenix, Compal, Dell, Foxconn, Intel, MSI, Marvell,


以及



Quanta


，阵容不算小。这些单位不是被邀请（求他们参会），而是不请自到（有这种需求），他们都想与



Canonical“


拉近乎”，学到点新东西。有什么新东西好学呢？已经确定的大会议题有：多点触摸技术，云计算中的服务器，



Linux


内核



Bug


处理，



Ubuntu


核心团队介绍，



UTouch


手势语言包，语音技术以及系统质量保证等方面。

一个软件公司为什么要搞



UHS


硬件峰会？



UHS


峰会的真正使命是什么？是



Canonical


想推销自己的



Linux


商业版吗？与会者需要为



Canonical


的这些“技术创新”付费吗？表面上看来似乎是这样的，



Canonical


在



Linux


商业版中注入“创新价值“，使其成为需要付费的



Linux


商业版。实际上，非也。这是“误区”作者散布的一种谬论。



Canonical


走的是另外一条路线：紧密与硬件厂商合作，使其硬件产品注入



GNU/Ubuntu


自由软件包之后，硬软件“一体化”之后的整体能够增值，并且能够快速投放市场，极大地提升原生产厂商的经济效益，然后再与



Canonical


从市场收益中合理分成，根本不需要将



Ubuntu Linux


商品化，搞成需要付费的



Linux


商业版。简而言之，



Google


与网络运营商进行收益分成，



Canonical


与



OEM


硬件制造商进行收益分成（具体分成比例属于商业机密）。单纯的硬件是没有活力的，只有注入相应的软件才能被激活，整体得以增值，体现了软件原来潜在的价值；单纯的软件也是没有活力的，只有附着在相应的硬件上才能运行起来，证明自己存在的价值。从物理上来讲，对于硬件而言，软件只是其内部电子介质上的某种有序变化的电磁状态，单独存在并无意义。说什么把二次开发的工程化经验注入到



Linux


发行版（纯粹的软件包）里面，提升其“商业价值”，云云，都是扑风捉影、毫无根据、别有用心的说辞。
