#### Amazon EC2使用

> 1、启动 EC2 实例，转到 EC2 选项卡，通过用户右上角的 搜索栏中搜索"EC2" 或从控制台主页中的服务中选择
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/2bf1cfa2fa2b495ca448550f2aaa5897.png#pic_center)
>
> 2、启动实例（完成此步骤后，在帐户中看到已分配的 EC2 实例，然后点击 启动实例”即可开始配置新的 EC2 实例）
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/2d1a3d22a1a645289447d2324bac4d0c.png#pic_center)
>
> 3、选择 AMI（Amazon 系统映像）
>
> * AMI 是云端计算机的预配置蓝图，包含操作系统和所需软件，选 AMI 如同为实例选入门套件，类似租用已装 Windows 或 Linux 的计算机。
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/8d1fef8b9a77481a87d5f2c856acd83e.png#pic_center)
>
> 4、选择实例类型
>
> * 接下来需根据工作负载选择实例类型，亚马逊云科技提供多种选项，如通用 / 突发性能的 T2 实例、计算密集型的 C5 实例，此步骤类似确定租用计算机的硬件规格（CPU、内存、网络性能等）
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/637905ab1827404fb6ef1b67cf9b5897.png#pic_center)
>
> 5、配置密钥对
>
> * 要安全访问实例需创建密钥对（公钥 / 私钥），用于通过 SSH 或 RDP 连接实例，可选择现有密钥对或点击 “创建新密钥对” 生成新的
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/f15450788d064985a2141159281b34da.png#pic_center)
>
> 6、配置网络
>
> * 然后需通过选择虚拟私有云（VPC）和子网确定实例位置，这将决定网络环境（如实例是否分配公有 IP 以支持互联网访问或保持私有），类似在安全数据中心内选择计算机的部署位置
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/c10f2614408b4a3f88704e7119911e04.png#pic_center)
>
> 7、设置安全性（安全组）-添加存储
>
> * 安全组类似虚拟防火墙，可通过定义规则（如 SSH、HTTP 等流量类型）控制实例访问权限，例如限定特定 IP 地址的 SSH 访问以限制用户访问方式。可点击 “创建安全组” 新建或选择现有安全组
> * 可将存储卷（如 Amazon EBS 卷）附加到实例（类似挂载硬盘），还能为实例添加标签以便识别管理，如同为计算机设置存储空间并标记便于后续访问
>
> ![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/99ac7b0d68634f86abc4d98fc4049fb9.png#pic_center)
>
> 8、配置完成启动
>
> * 最后检查实例配置确保无误，确认满意后点击 “启动” 启动实例，类似开机前检查计算机设置。

#### Amazon EC2 实例类型详解

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/d83f2cda51b64515b851a37f137066d2.png#pic_center)

| **类型** | **典型实例** | **核心配置** | **适用场景** |
| --- | --- | --- | --- |
| **通用型** | T4g（ARM） | Graviton2 处理器，2vCPU/512MB 起，25Gbps 带宽，低成本轻量级负载 | Web 服务器、微服务、开发测试 |
| **计算优化型** | C6g（ARM） | Graviton2 处理器，1vCPU/2G 起，25Gbps 带宽，3.8TB 存储 | 批处理、媒体转码、高性能计算 |
| **内存优化型** | R6g（ARM） | Graviton2 处理器，1vCPU/8GB 起，25Gbps 带宽，512GB 内存 | 大数据分析、内存数据库（如 Redis） |

| **存储优化型** | I3 | Intel Xeon 处理器，2vCPU/15.25G 起，16TB NVMe SSD，高 IOPS | 数据库、日志处理、实时数据存储 |

##### 通用计算型实例

> 平衡计算、内存与联网资源，适用 Web 服务器等多元负载：
>
> * **Mac1**：Apple 硬件，6 物理核 + 12 逻辑核 + 32GiB 内存，专属 Mac 开发测试裸机环境。
> * **T4g（ARM）**：Graviton2 处理器，2vCPU/512MB 起至 64vCPU/256G 内存，25Gbps 带宽，部分机型 0 元开机，轻量级通用首选。
> * **M6i（Intel）**：第 3 代至强处理器，2vCPU/8G 起至 128vCPU/512G 内存，50Gbps 带宽 + 内存加密，适配企业级应用。
> * **M5a（AMD）**：EPYC 7000 系列处理器，2vCPU/8G 起至 96vCPU/384G 内存，20Gbps 带宽 + 3.6TB NVMe 存储，兼顾通用与存储需求。

##### 计算优化型实例

> 聚焦高性能计算，适用于批处理、媒体转码等密集场景：
>
> * **C6g（ARM）**：Graviton2 处理器，1vCPU/2G 起至 64vCPU/128G 内存，25Gbps 带宽 + 3.8TB 存储，高效支撑计算任务。
> * **C5n（联网增强）**：3.0GHz Intel Xeon Platinum 处理器，1vCPU/2G 起至 72vCPU/192G 内存，100Gbps 带宽 + 38Gbps EBS 带宽，网络敏感型首选。
> * **C5（Intel）**：第二代至强处理器，2vCPU/4G 起至 96vCPU/192G 内存，25Gbps 带宽 + 3.6T 存储，适配常规计算密集应用。
> * **C5a（AMD）**：EPYC 7002 系列处理器，2vCPU/4G 起至 96vCPU/192G 内存，20Gbps 带宽 + 3.8T 存储，提供稳定算力。

##### 内存优化型实例

> 专注内存中大型数据集处理，提升负载性能：
>
> * **U 系列（内存增强）**：Intel Xeon Platinum 处理器，224vCPU/6T 内存起至 448vCPU/24T 内存，裸机 / 虚拟化形式满足超大规模内存需求。
> * **R6g（ARM）**：Graviton2 处理器，1vCPU/8GB 起至 64vCPU/512G 内存，25Gbps 带宽 + 3.8TB 存储，适配 ARM 内存密集应用。
> * **X1e（Intel）**：Xeon E7-8880 v3 处理器，4vCPU/122GB 起至 128vCPU/4T 内存，25Gbps 带宽 + 3.8TB 存储，助力内存敏感业务。
> * **R5a（AMD）**：EPYC 7000 系列处理器，2vCPU/16GB 起至 96vCPU/768G 内存，20Gbps 带宽 + 3.6TB 存储，处理 AMD 架构内存任务。

##### 加速计算型实例

> 借助硬件加速器提升浮点、图形处理效率：
>
> * **P4（GPU 服务器）**：8×NVIDIA A100 GPU + 第二代至强处理器，96vCPU/1152G 内存，400Gbps 联网 + 8TB 存储，赋能深度学习训练。
> * **Inf（Inferentia 芯片）**：最多 16 个 Amazon Inferentia 芯片 + 第二代至强处理器，4vCPU/8GB 起至 96vCPU/192G 内存，100Gbps 带宽，加速推理任务。
> * **g4dn（Intel）**：Xeon Cascade Lake 处理器 + NVIDIA T4 GPU，1GPU/4vCPU/16G 起至 8GPU/96vCPU/128G 内存，100Gbps 带宽 + 1.8TB 存储，适配多场景加速。
> * **F1（FPGA 加速）**：Xeon E5 处理器 + Xilinx FPGA，1FPGA/8vCPU/122G 起至 8FPGA/64vCPU/976G 内存，250 万逻辑元素 + 6800 DSP 引擎，满足定制化加速需求。

##### 存储优化型实例

> 优化本地存储高速读写，支持数万低延迟 I/O 操作：
>
> * **H1**：Intel Xeon E5 处理器，8vCPU/32G 起至 64vCPU/256G 内存，25Gbps 带宽 + 16TB HDD 存储，适配大容量读写任务。
> * **I3**：低成本高 IOPS，Intel Xeon E5 处理器，2vCPU/15.25G 起至 72vCPU/512G 内存，25Gbps 带宽 + 16TB NVMe SSD 存储，高 IOPS 服务存储业务。
> * **D3**：Intel Xeon Cascade Lake 处理器，4vCPU/32G 起至 32vCPU/256G 内存，25Gbps 带宽 + 48TB HDD 存储，满足大容量 HDD 需求。
> * **D3en**：Intel Xeon Cascade Lake 处理器，4vCPU/16G 起至 48vCPU/192G 内存，75Gbps 带宽 + 336TB HDD 存储，适配超大规模 HDD 读写场景。

#### 总结

> Amazon EC2 凭借丰富的实例类型、全球基础设施及灵活的管理工具，成为云计算场景下的核心计算平台。通过合理选择实例类型、优化网络配置与成本管理，用户可高效构建安全、可扩展的应用架构，适用于从中小型企业到大型企业的各类工作负载，建议结合官方文档与实践操作，进一步探索 EC2 与其他亚马逊云科技服务（如 S3、RDS、Lambda）的集成，释放云计算的全场景价值。
>
> **成本控制**
>
> * 利用亚马逊云科技免费套餐（如 750 小时 / 月 t2.micro 实例）降低入门成本
> * 对长期负载使用预留实例（RI）或Savings Plans，节省 30%~72% 费用
>
> **安全与合规**
>
> * 最小化安全组规则，限制仅必要 IP 地址访问（如 SSH 仅限管理 IP）
> * 使用 AWS Identity and Access Management（IAM）创建子用户，避免直接使用根账户操作
>
> **高可用性架构**
>
> * 将实例分布在多个可用区（AZ），结合弹性负载均衡（ELB）和自动扩展组（ASG），提升系统容错能力
