    <h4 id="33b714ca">
     NPM（Node Package Manager）是 Node.js 默认的包管理工具，主要用于管理 JavaScript 项目的依赖包。以下是常用的 NPM 命令，涵盖安装、卸载、更新、查看等操作。
    </h4>
    <hr id="SLQRN"/>
    <h3 id="03878aab">
     <strong>
      📌
     </strong>
     <strong>
      1. NPM 初始化
     </strong>
    </h3>
    <pre id="rUNgt"><code>npm init</code></pre>
    <ul>
     <li id="udeb3f7aa">
      逐步询问项目信息，并生成
      <code>
       package.json
      </code>
      。
     </li>
     <li id="ua9c9a71b">
      可使用
      <code>
       -y
      </code>
      跳过交互，直接生成默认配置：
     </li>
    </ul>
    <pre id="M7v5k"><code>npm init -y</code></pre>
    <hr id="kKGTL"/>
    <h3 id="28eb70f7">
     <strong>
      📌
     </strong>
     <strong>
      2. 安装依赖
     </strong>
    </h3>
    <h4 id="dd0770bb">
     <strong>
      🔹
     </strong>
     <strong>
      （1）安装单个依赖
     </strong>
    </h4>
    <pre id="kDUjN"><code>npm install package-name</code></pre>
    <p id="uece01f00">
     🔹
     <strong>
      简写
     </strong>
     ：
    </p>
    <pre id="UZtUS"><code>npm i package-name</code></pre>
    <p id="u7091fecc">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="DWGZa"><code>npm i express</code></pre>
    <p id="ua3ff85fe">
     安装
     <code>
      express
     </code>
     并添加到
     <code>
      package.json
     </code>
     的
     <code>
      dependencies
     </code>
     。
    </p>
    <h4 id="4d8260e6">
     <strong>
      🔹
     </strong>
     <strong>
      （2）安装开发依赖
     </strong>
    </h4>
    <pre id="aPiUU"><code>npm install package-name --save-dev</code></pre>
    <p id="u24ee3dfd">
     🔹
     <strong>
      简写
     </strong>
     ：
    </p>
    <pre id="ibkVr"><code>npm i package-name -D</code></pre>
    <p id="u1da4534c">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="Bq53w"><code>npm i nodemon -D</code></pre>
    <p id="u30479e0f">
     仅在开发环境使用
     <code>
      nodemon
     </code>
     。
    </p>
    <h4 id="10789500">
     <strong>
      🔹
     </strong>
     <strong>
      （3）全局安装
     </strong>
    </h4>
    <pre id="GK1Wl"><code>npm install package-name -g</code></pre>
    <p id="u6e54ab7d">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="YxNAr"><code>npm i nodemon -g</code></pre>
    <p id="u7b24ed15">
     在全局范围安装
     <code>
      nodemon
     </code>
     ，可在终端直接使用。
    </p>
    <h4 id="f1a37a49">
     <strong>
      🔹
     </strong>
     <strong>
      （4）安装 package.json 中所有依赖
     </strong>
    </h4>
    <pre id="YDEKl"><code>npm install</code></pre>
    <p id="uac765f27">
     <strong>
      通常用于克隆项目后，自动安装所有依赖
     </strong>
     。
    </p>
    <hr id="D8n4B"/>
    <h3 id="0cc9cf09">
     <strong>
      📌
     </strong>
     <strong>
      3. 卸载依赖
     </strong>
    </h3>
    <h4 id="7d1b5f81">
     <strong>
      🔹
     </strong>
     <strong>
      （1）卸载已安装的包
     </strong>
    </h4>
    <pre id="UZibq"><code>npm uninstall package-name</code></pre>
    <p id="u048f8990">
     🔹
     <strong>
      简写
     </strong>
     ：
    </p>
    <pre id="kt1Nq"><code>npm remove package-name</code></pre>
    <p id="ufcccac0a">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="aBUYM"><code>npm uninstall express</code></pre>
    <p id="udd67e60c">
     从
     <code>
      dependencies
     </code>
     中移除
     <code>
      express
     </code>
     。
    </p>
    <h4 id="522bc5b9">
     <strong>
      🔹
     </strong>
     <strong>
      （2）卸载开发依赖
     </strong>
    </h4>
    <pre id="JoGq9"><code>npm uninstall package-name --save-dev</code></pre>
    <p id="u63df7fa6">
     🔹
     <strong>
      简写
     </strong>
     ：
    </p>
    <pre id="VtR1O"><code>npm rm package-name -D</code></pre>
    <p id="u7644f9ec">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="qVszI"><code>npm rm nodemon -D</code></pre>
    <p id="uc8ccbc64">
     从
     <code>
      devDependencies
     </code>
     中移除
     <code>
      nodemon
     </code>
     。
    </p>
    <h4 id="a50c437d">
     <strong>
      🔹
     </strong>
     <strong>
      （3）全局卸载
     </strong>
    </h4>
    <pre id="VDZkf"><code>npm uninstall package-name -g</code></pre>
    <p id="u0bdbfad5">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="CakCU"><code>npm uninstall nodemon -g</code></pre>
    <p id="u67c60c9c">
     卸载全局安装的
     <code>
      nodemon
     </code>
     。
    </p>
    <hr id="IVhnc"/>
    <h3 id="b74606bc">
     <strong>
      📌
     </strong>
     <strong>
      4. 更新依赖
     </strong>
    </h3>
    <h4 id="143c389b">
     <strong>
      🔹
     </strong>
     <strong>
      （1）更新单个依赖
     </strong>
    </h4>
    <pre id="A579p"><code>npm update package-name</code></pre>
    <p id="ud41e55b5">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="sZxrg"><code>npm update lodash</code></pre>
    <p id="u2aa734ac">
     更新
     <code>
      lodash
     </code>
     依赖。
    </p>
    <h4 id="099f8fc4">
     <strong>
      🔹
     </strong>
     <strong>
      （2）更新所有依赖
     </strong>
    </h4>
    <pre id="sM2EI"><code>npm update</code></pre>
    <p id="u4f033602">
     <strong>
      更新
     </strong>
     <code>
      <strong>
       package.json
      </strong>
     </code>
     <strong>
      里所有的依赖
     </strong>
     。
    </p>
    <h4 id="018ec4cc">
     <strong>
      🔹
     </strong>
     <strong>
      （3）更新到最新版本（可能引入不兼容变更）
     </strong>
    </h4>
    <pre id="KpLHV"><code>npm install package-name@latest</code></pre>
    <p id="u23c69ef4">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="Q1AJO"><code>npm i react@latest</code></pre>
    <p id="u7d6c5e74">
     安装最新的
     <code>
      react
     </code>
     版本。
    </p>
    <hr id="lYdKg"/>
    <h3 id="381bd891">
     <strong>
      📌
     </strong>
     <strong>
      5. 查看信息
     </strong>
    </h3>
    <h4 id="a43a346f">
     <strong>
      🔹
     </strong>
     <strong>
      （1）查看已安装的依赖
     </strong>
    </h4>
    <pre id="xOkZ4"><code>npm list</code></pre>
    <p id="u2818811c">
     <strong>
      查看当前项目已安装的依赖及版本
     </strong>
     。
    </p>
    <pre id="XCEGq"><code>npm list -g --depth=0</code></pre>
    <p id="u0e760436">
     <strong>
      查看全局已安装的 npm 包（仅显示顶级包）
     </strong>
     。
    </p>
    <h4 id="c30c66f1">
     <strong>
      🔹
     </strong>
     <strong>
      （2）查看依赖信息
     </strong>
    </h4>
    <pre id="Qq1Li"><code>npm info package-name</code></pre>
    <p id="u16cce3b9">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="zvo1x"><code>npm info express</code></pre>
    <p id="u2e7a7e2e">
     显示
     <code>
      express
     </code>
     包的详细信息。
    </p>
    <hr id="cttqR"/>
    <h3 id="2adc9251">
     <strong>
      📌
     </strong>
     <strong>
      6. 清理 &amp; 修复
     </strong>
    </h3>
    <h4 id="c258e9b8">
     <strong>
      🔹
     </strong>
     <strong>
      （1）清理缓存
     </strong>
    </h4>
    <pre id="WTjtE"><code>npm cache clean --force</code></pre>
    <p id="uceff8f3e">
     <strong>
      清除 npm 缓存，解决依赖安装问题
     </strong>
     。
    </p>
    <h4 id="cab94867">
     <strong>
      🔹
     </strong>
     <strong>
      （2）修复 npm 权限问题
     </strong>
    </h4>
    <pre id="W8gA1"><code>npm doctor</code></pre>
    <p id="u3da6e363">
     <strong>
      检测 npm 运行状态，修复可能的问题
     </strong>
     。
    </p>
    <pre id="Mbw8L"><code>npm audit fix</code></pre>
    <p id="uff472d7f">
     <strong>
      自动修复安全漏洞
     </strong>
     。
    </p>
    <hr id="bUpU4"/>
    <h3 id="02529c75">
     <strong>
      📌
     </strong>
     <strong>
      7. 版本管理
     </strong>
    </h3>
    <h4 id="ec1879ad">
     <strong>
      🔹
     </strong>
     <strong>
      （1）查看 NPM 版本
     </strong>
    </h4>
    <pre id="RxfL0"><code>npm -v</code></pre>
    <p id="u6a67793c">
     <strong>
      查看本机安装的 npm 版本
     </strong>
     。
    </p>
    <h4 id="58ed9153">
     <strong>
      🔹
     </strong>
     <strong>
      （2）更新 NPM
     </strong>
    </h4>
    <pre id="VtfeX"><code>npm install -g npm@latest</code></pre>
    <p id="u8760c0a5">
     <strong>
      更新 npm 到最新版本
     </strong>
     。
    </p>
    <h4 id="b476e75e">
     <strong>
      🔹
     </strong>
     <strong>
      （3）切换 Node 版本（需安装
     </strong>
     <code>
      <strong>
       n
      </strong>
     </code>
     <strong>
      ）
     </strong>
    </h4>
    <pre id="V77s0"><code>npm install -g n
n stable</code></pre>
    <p id="u35d3e5b5">
     <strong>
      安装 Node.js 的最新稳定版
     </strong>
     。
    </p>
    <hr id="HqlWx"/>
    <h3 id="4cf66061">
     <strong>
      📌
     </strong>
     <strong>
      8. 运行脚本
     </strong>
    </h3>
    <h4 id="bd2d6408">
     <strong>
      🔹
     </strong>
     <strong>
      （1）运行
     </strong>
     <code>
      <strong>
       package.json
      </strong>
     </code>
     <strong>
      中定义的脚本
     </strong>
    </h4>
    <pre id="YKqd9"><code>npm run script-name</code></pre>
    <p id="u76c274a5">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="Pwy7E"><code>{
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  }
}</code></pre>
    <p id="u5c6f5852">
     运行：
    </p>
    <pre id="tE1N9"><code>npm run start</code></pre>
    <p id="ud1b29af9">
     或：
    </p>
    <pre id="MFwUb"><code>npm run dev</code></pre>
    <h4 id="0436ed91">
     <strong>
      🔹
     </strong>
     <strong>
      （2）使用
     </strong>
     <code>
      <strong>
       npx
      </strong>
     </code>
     <strong>
      运行包
     </strong>
    </h4>
    <pre id="uOIYP"><code>npx package-name</code></pre>
    <p id="u2834156b">
     <strong>
      示例
     </strong>
     ：
    </p>
    <pre id="qVEyv"><code>npx create-react-app my-app</code></pre>
    <p id="u45b61152">
     使用
     <code>
      npx
     </code>
     运行
     <code>
      create-react-app
     </code>
     （无需全局安装）。
    </p>
    <hr id="AHf2E"/>
    <h3 id="7f47c287">
     <strong>
      📌
     </strong>
     <strong>
      9. 版本号规则（SemVer）
     </strong>
    </h3>
    <p id="u618b92a4">
     NPM 版本号格式为
     <code>
      MAJOR.MINOR.PATCH
     </code>
     ：
    </p>
    <ul>
     <li id="u4a19646e">
      <code>
       1.0.0
      </code>
      ：主版本（MAJOR）变化，可能有不兼容变更。
     </li>
     <li id="u84e46c44">
      <code>
       1.1.0
      </code>
      ：次版本（MINOR）变化，新增功能，向后兼容。
     </li>
     <li id="ud44ce800">
      <code>
       1.1.1
      </code>
      ：补丁版本（PATCH）变化，修复 bug，向后兼容。
     </li>
    </ul>
    <h4 id="74972c90">
     <strong>
      🔹
     </strong>
     <strong>
      （1）安装特定版本
     </strong>
    </h4>
    <pre id="kmhju"><code>npm install package-name@1.2.3</code></pre>
    <h4 id="7a4f1c45">
     <strong>
      🔹
     </strong>
     <strong>
      （2）安装指定范围
     </strong>
    </h4>
    <pre id="IrPK0"><code>npm install package-name@^1.2.0  # 允许 1.x 版本升级，但不超过 2.0.0
npm install package-name@~1.2.0  # 允许 1.2.x 版本升级，但不超过 1.3.0</code></pre>
    <hr id="YMFmg"/>
    <h3 id="c50df5df">
     <strong>
      📌
     </strong>
     <strong>
      10. 切换 NPM 镜像
     </strong>
    </h3>
    <h4 id="e139f40d">
     <strong>
      🔹
     </strong>
     <strong>
      （1）使用淘宝镜像
     </strong>
    </h4>
    <pre id="hOm91"><code>npm config set registry https://registry.npmmirror.com</code></pre>
    <p id="uf46bb1b8">
     <strong>
      切换 npm 源，提高安装速度
     </strong>
     。
    </p>
    <h4 id="4d88386b">
     <strong>
      🔹
     </strong>
     <strong>
      （2）恢复官方镜像
     </strong>
    </h4>
    <pre id="WGHGh"><code>npm config set registry https://registry.npmjs.org</code></pre>
