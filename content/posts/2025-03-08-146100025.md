    <p>
     MyBatis SQL 映射文件定义了 SQL 语句以及如何将 SQL 语句的参数和结果映射到 Java 对象。
     <br/>
     <strong>
      一、 作用 (Purpose)
     </strong>
    </p>
    <p>
     MyBatis SQL 映射文件（通常命名为
     <code>
      XXXMapper.xml
     </code>
     ）的主要作用是：
    </p>
    <ol>
     <li>
      <strong>
       定义 SQL 语句：
      </strong>
      在 XML 映射文件中编写 SQL 语句，包括 SELECT, INSERT, UPDATE, DELETE 等各种类型的 SQL 语句。
     </li>
     <li>
      <strong>
       参数映射 (Parameter Mapping)：
      </strong>
      定义如何将 Java 方法的参数映射到 SQL 语句中的参数。 可以使用
      <code>
       #
      </code>
      和
      <code>
       $
      </code>
      占位符，并指定参数类型、jdbcType 等信息。
     </li>
     <li>
      <strong>
       结果映射 (Result Mapping)：
      </strong>
      定义如何将 SQL 语句的查询结果映射到 Java 对象。 可以使用
      <code>
       resultType
      </code>
      或
      <code>
       resultMap
      </code>
      ，并定义一对一、一对多等关系映射。
     </li>
     <li>
      <strong>
       动态 SQL (Dynamic SQL)：
      </strong>
      使用 MyBatis 提供的动态 SQL 元素 (例如
      <code>
       &lt;if&gt;
      </code>
      ,
      <code>
       &lt;choose&gt;
      </code>
      ,
      <code>
       &lt;when&gt;
      </code>
      ,
      <code>
       &lt;otherwise&gt;
      </code>
      ,
      <code>
       &lt;where&gt;
      </code>
      ,
      <code>
       &lt;set&gt;
      </code>
      ,
      <code>
       &lt;foreach&gt;
      </code>
      )，根据不同的条件动态生成 SQL 语句。
     </li>
     <li>
      <strong>
       缓存配置 (Cache Configuration)：
      </strong>
      配置 SQL 语句的缓存行为，提高查询性能。
     </li>
     <li>
      <strong>
       与 Mapper 接口关联：
      </strong>
      将 SQL 映射文件与 Java Mapper 接口关联起来，使得可以通过调用 Mapper 接口的方法来执行 SQL 语句。
     </li>
    </ol>
    <p>
     <strong>
      总而言之，SQL 映射文件是 MyBatis 将 SQL 语句与 Java 代码解耦的关键，它允许开发者专注于 SQL 语句的编写和优化，而无需关心 JDBC 的细节。
     </strong>
    </p>
    <p>
     <strong>
      二、 结构 (Structure)
     </strong>
    </p>

     MyBatis SQL 映射文件的根元素是
     <code>

    <p>
     <strong>
      三、各个元素详解
     </strong>
    </p>
    <ol>
     <li>
      <p>
       <strong>
        <code>
         &lt;mapper&gt;
        </code>
        (根元素)：
       </strong>
      </p>
      <ul>
       <li>
        <code>
         namespace
        </code>
        属性：指定 Mapper 接口的完全限定名。 MyBatis 会根据这个命名空间来查找 SQL 映射文件，并将 SQL 语句与 Mapper 接口的方法关联起来。 这是将 SQL 映射文件与 Java 代码关联的关键。
        <br/>
        例如：
        <code>
         &lt;mapper namespace="com.example.mapper.UserMapper"&gt;
        </code>
        对应着
        <code>
         UserMapper.java
        </code>
        接口文件
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         &lt;cache&gt;
        </code>
        (缓存)：
       </strong>
      </p>
      <ul>
       <li>
        配置该 Mapper 对应的二级缓存。
       </li>
       <li>
        <code>
         eviction
        </code>
        属性：指定缓存的淘汰策略 (LRU, FIFO, SOFT, WEAK)。
       </li>
       <li>
        <code>
         flushInterval
        </code>
        属性：指定缓存的刷新间隔 (毫秒)。
       </li>
       <li>
        <code>
         size
        </code>
        属性：指定缓存的最大容量。
       </li>
       <li>
        <code>
         readOnly
        </code>
        属性：指定缓存是否只读 (
        <code>
         true
        </code>
        或
        <code>
         false
        </code>
        )。
       </li>
       <li>
        <code>
         &lt;cache type="org.mybatis.cache.impl.PerpetualCache"/&gt;
        </code>
        使用自定义缓存
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         &lt;cache-ref&gt;
        </code>
        (缓存引用)：
       </strong>
      </p>
      <ul>
       <li>
        引用其他 Mapper 的缓存配置。
       </li>
       <li>
        <code>
         namespace
        </code>
        属性：指定要引用的 Mapper 的命名空间。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        <code>
         &lt;resultMap&gt;
        </code>
        (结果映射)：
       </strong>
      </p>
      <ul>
       <li>
        <p>
         <strong>
          作用：
         </strong>
         定义如何将 SQL 语句的查询结果映射到 Java 对象。
        </p>
       </li>
       <li>
        <p>
         <code>
          id
         </code>
         属性：指定
         <code>
          resultMap
         </code>
         的唯一标识。
        </p>
       </li>
       <li>
        <p>
         <code>
          type
         </code>
         属性：指定要映射的 Java 类型。
        </p>
       </li>
       <li>
        <p>
         <strong>
          子元素：
         </strong>
        </p>
        <ul>
         <li>
          <code>
           &lt;id&gt;
          </code>
          ：定义主键映射。
          <ul>
           <li>
            <code>
             property
            </code>
            属性：指定 Java 对象的属性名。
           </li>
           <li>
            <code>
             column
            </code>
            属性：指定数据库表中的列名。
           </li>
          </ul>
         </li>
         <li>
          <code>
           &lt;result&gt;
          </code>
          ：定义普通属性映射。
          <ul>
           <li>
            <code>
             property
            </code>
            属性：指定 Java 对象的属性名。
           </li>
           <li>
            <code>
             column
            </code>
            属性：指定数据库表中的列名。
           </li>
          </ul>
         </li>
         <li>
          <code>
           &lt;association&gt;
          </code>
          ：定义一对一关系映射。
          <ul>
           <li>
            <code>
             property
            </code>
            属性：指定 Java 对象的属性名。
           </li>
           <li>
            <code>
             column
            </code>
            属性：指定数据库表中的列名。
           </li>
           <li>
            <code>
             javaType
            </code>
            属性：指定关联对象的 Java 类型。
           </li>
           <li>
            <code>
             select
            </code>
            属性：指定用于查询关联对象的 SQL 语句的 ID。
           </li>
          </ul>
         </li>
         <li>
          <code>
           &lt;collection&gt;
          </code>
          ：定义一对多关系映射。
          <ul>
           <li>
            <code>
             property
            </code>
            属性：指定 Java 对象的属性名。
           </li>
           <li>
            <code>
             column
            </code>
            属性：指定数据库表中的列名。
           </li>
           <li>
            <code>
             ofType
            </code>
            属性：指定集合中元素的 Java 类型。
           </li>
           <li>
            <code>
             select
            </code>
            属性：指定用于查询集合元素的 SQL 语句的 ID。
           </li>
          </ul>
         </li>
         <li>
          <code>
           &lt;discriminator&gt;
          </code>
          ：定义鉴别器，根据不同的条件选择不同的映射规则。
          <ul>
           <li>
            <code>
             column
            </code>
            属性：指定用于鉴别的列名。
           </li>
           <li>
            <code>
             &lt;case&gt;
            </code>
            元素：定义不同的映射规则。
            <ul>
             <li>
              <code>
               value
              </code>
              属性：指定鉴别值。
             </li>
             <li>
              <code>
               resultMap
              </code>
              属性：指定要使用的
              <code>
               resultMap
              </code>
              。
             </li>
            </ul>
           </li>
          </ul>
         </li>
        </ul>
       </li>
       <li>
        <p>
         <strong>
          示例：
         </strong>
