    <h4>
     <a id="1_GitHub_Actions__3">
     </a>
     <strong>
      1. GitHub Actions 基础
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       GitHub Actions
      </strong>
      是 GitHub 提供的自动化工具，用于构建、测试和部署代码。
     </li>
     <li>
      <strong>
       使用场景
      </strong>
      ：自动化代码构建、测试、部署、代码审查、分支管理等。
     </li>
     <li>
      <strong>
       成本
      </strong>
      ：GitHub 提供免费和付费计划，免费计划每月有 2,000 分钟的运行时间。
     </li>
     <li>
      <strong>
       何时使用
      </strong>
      ：当项目需要自动化 CI/CD 流程时，GitHub Actions 是一个很好的选择。
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：GitHub Actions 是 GitHub 的自动化工具，用于代码构建、测试和部署。它适用于需要自动化流程的项目，且提供免费和付费计划。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_GitHub_Actions__12">
     </a>
     <strong>
      2. GitHub Actions 入门
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Starter Workflows
      </strong>
      ：GitHub 提供了多种预定义的工作流模板，适用于代码、问题、拉取请求、安全等场景。
     </li>
     <li>
      <strong>
       示例工作流
      </strong>
      ：
      <ul>
       <li>
        <strong>
         部署 Node.js 到 Azure Web App
        </strong>
        ：构建 Node.js 项目并部署到 Azure。
       </li>
       <li>
        <strong>
         部署到 Amazon ECS
        </strong>
        ：将容器部署到 Amazon ECS 服务。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：GitHub 提供了多种预定义的工作流模板，帮助用户快速开始自动化流程，例如部署 Node.js 到 Azure 或 Amazon ECS。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_GitHub_Actions__21">
     </a>
     <strong>
      3. GitHub Actions 市场
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       市场
      </strong>
      ：GitHub Actions 市场提供了丰富的第三方 Actions，用于扩展工作流功能。
     </li>
     <li>
      <strong>
       常见 Actions
      </strong>
      ：
      <ul>
       <li>
        <strong>
         Setup Go environment
        </strong>
        ：设置 Go 环境。
       </li>
       <li>
        <strong>
         Setup Node.js environment
        </strong>
        ：设置 Node.js 环境。
       </li>
       <li>
        <strong>
         Upload a Build Artifact
        </strong>
        ：上传构建产物。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：GitHub Actions 市场提供了许多第三方 Actions，用户可以根据需要选择并集成到工作流中。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="4_GitHub_Actions__31">
     </a>
     <strong>
      4. GitHub Actions 定价计划
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       免费计划
      </strong>
      ：每月 2,000 分钟，500 MB 存储。
     </li>
     <li>
      <strong>
       Pro 计划
      </strong>
      ：每月 3,000 分钟，1 GB 存储。
     </li>
     <li>
      <strong>
       企业计划
      </strong>
      ：每月 50,000 分钟，50 GB 存储。
     </li>
     <li>
      <strong>
       操作系统成本
      </strong>
      ：
      <ul>
       <li>
        <strong>
         Linux
        </strong>
        ：每分钟 $0.008。
       </li>
       <li>
        <strong>
         macOS
        </strong>
        ：每分钟 $0.08。
       </li>
       <li>
        <strong>
         Windows
        </strong>
        ：每分钟 $0.016。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：GitHub Actions 提供多种定价计划，免费计划适合小型项目，企业计划适合大型团队。不同操作系统的运行成本不同。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="5_GitHub_Actions__43">
     </a>
     <strong>
      5. GitHub Actions 组件
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       事件（Event）
      </strong>
      ：触发工作流的事件，例如代码推送（push）或拉取请求（pull request）。
     </li>
     <li>
      <strong>
       工作流（Workflow）
      </strong>
      ：由多个任务（Jobs）组成，每个任务包含多个步骤（Steps）。
     </li>
     <li>
      <strong>
       运行器（Runner）
      </strong>
      ：执行工作流的机器，可以是 GitHub 托管的或自托管的。
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：GitHub Actions 的核心组件包括事件、工作流、任务、步骤和运行器。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="6__51">
     </a>
     <strong>
      6. 触发工作流
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       触发方式
      </strong>
      ：
      <ul>
       <li>
        <strong>
         单一事件
        </strong>
        ：例如代码推送（push）。
       </li>
       <li>
        <strong>
         多个事件
        </strong>
        ：例如代码推送和拉取请求（push, pull_request）。
       </li>
       <li>
        <strong>
         定时任务
        </strong>
        ：使用 cron 语法定时触发。
       </li>
       <li>
        <strong>
         手动触发
        </strong>
        ：例如通过
        <code>
         workflow-dispatch
        </code>
        手动触发。
       </li>
       <li>
        <strong>
         从其他工作流调用
        </strong>
        ：通过
        <code>
         workflowcall
        </code>
        触发。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：工作流可以通过多种方式触发，包括代码推送、拉取请求、定时任务或手动触发。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="7__62">
     </a>
     <strong>
      7. 工作流步骤
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       步骤示例
      </strong>
      ：
      <ul>
       <li>
        <strong>
         检出代码
        </strong>
        ：
        <code>
         uses: actions/checkout@v3
        </code>
        。
       </li>
       <li>
        <strong>
         设置 Go 环境
        </strong>
        ：
        <code>
         uses: actions/setup-go@v2
        </code>
        。
       </li>
       <li>
        <strong>
         运行命令
        </strong>
        ：
        <code>
         run: go run helloworld.go
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：工作流的每个步骤可以执行特定的操作，例如检出代码、设置环境或运行命令。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="8__71">
     </a>
     <strong>
      8. 工作流运行器和任务
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       运行器
      </strong>
      ：指定运行任务的机器，例如
      <code>
       runs-on: ubuntu-latest
      </code>
      。
     </li>
     <li>
      <strong>
       任务
      </strong>
      ：每个任务可以包含多个步骤，例如构建、测试和部署。
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：运行器是执行任务的机器，任务是由多个步骤组成的工作单元。
     </li>
    </ul>
    <hr/>

     </a>

    <hr/>
    <h4>
     <a id="10_GitHub_Actions__101">
     </a>
     <strong>
      10. GitHub Actions 配置与安全
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       配置
      </strong>
      ：工作流文件（YAML）定义了自动化流程的步骤和触发条件。
     </li>
     <li>
      <strong>
       安全
      </strong>
      ：GitHub Actions 支持加密的环境变量和权限控制，确保工作流的安全性。
     </li>
     <li>
      <strong>
       翻译
      </strong>
      ：通过 YAML 文件配置工作流，GitHub Actions 还提供了安全功能，例如加密的环境变量。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_108">
     </a>
     <strong>
      初学者如何理解？
     </strong>
    </h4>
    <ol>
     <li>
      <strong>
       GitHub Actions
      </strong>
      是一个自动化工具，可以帮助你自动构建、测试和部署代码。
     </li>
     <li>
      <strong>
       工作流
      </strong>
      是由多个步骤组成的自动化流程，可以通过代码推送、拉取请求等事件触发。
     </li>
     <li>
      <strong>
       运行器
      </strong>
      是执行工作流的机器，可以是 GitHub 托管的或自托管的。
     </li>
     <li>
      <strong>
       市场
      </strong>
      提供了许多第三方 Actions，可以扩展工作流的功能。
     </li>
     <li>
      <strong>
       定价计划
      </strong>
      ：GitHub 提供免费和付费计划，初学者可以从免费计划开始。
     </li>
    </ol>
