    <p>
     VS Code 是当前非常流行的开发工具. 本文讲述如何配置 VS Code 作为 C++开发环境. 本文将按照如下步骤来介绍如何配置 VS Code 作为 C++开发环境.
    </p>
    <ol>
     <li>
      安装编译器
     </li>
     <li>
      安装插件
     </li>
     <li>
      配置工作区
     </li>
    </ol>
    <p>
     第一个步骤的具体操作会因为系统不同或者方案不同而有不同的选择.
    </p>
    <h3>
     <a id="_8">
     </a>
     环境要求
    </h3>
    <p>
     首先需要立即 VS Code 主要的功能是编辑器, 它并没有编译器能力. 因此我们要想 VS Code 作为 C++开发环境, 我们需要安装好编译器.
    </p>
    <ol>
     <li>
      VS Code. 如果没有请去
      <a href="https://code.visualstudio.com/download" rel="nofollow">
       官网
      </a>
      下载并安装.
     </li>
     <li>
      C++ 编译器. 根据系统的不同, 可以选择不同的编译器. 读者请根据自己所用系统选择对应的编译器.
     </li>
    </ol>
    <h4>
     <a id="Windows__C_15">
     </a>
     Windows 下安装 C++编译器
    </h4>
    <p>
     Windows 下有很多选择, 简单来说有如下选择:
    </p>
    <ul>
     <li>
      使用原生的编译器:
      <a href="https://visualstudio.microsoft.com/zh-hans/downloads/" rel="nofollow">
       Microsoft Visual Studio
      </a>
      . Visual Studio 本身就是一款 IDE, 但是相比起 VS Code 来说, 软件体积很大.
     </li>
     <li>
      如果是 Windows 11 使用 WSL 来安装 C++编译器. Windows 10 应该也可以, 但是我本地环境是 Windows 11, 所以这里只介绍 Windows 11.
     </li>
     <li>
      如果本地有 Docker, 则可以用 dev container 方案. 有多个语言开发环境, 或者有多个编译器版本的用户可以考虑使用 dev container 方案. 可以参考我的这篇文章:
      <a href="https://blog.csdn.net/arong_xu/article/details/136900608">
       VS Code Dev Containers 教程: 从基础到进阶配置
      </a>
     </li>
    </ul>
    <h4>
     <a id="Linux__C_23">
     </a>
     Linux 下安装 C++编译器
    </h4>
    <p>
     Linux 下可以使用
     <a href="https://gcc.gnu.org/" rel="nofollow">
      gcc
     </a>
     ,
     <a href="https://clang.llvm.org/" rel="nofollow">
      clang
     </a>
     . 通过系统的包管理器来安装.
    </p>
    <ul>
     <li>
      <p>
       对于 Ubuntu 来说, 可以使用如下命令来安装:
      </p>
      <ul>
       <li>
        gcc:
        <code>
         sudo apt install gcc
        </code>
       </li>
       <li>
        clang:
        <code>
         sudo apt install clang
        </code>
       </li>
      </ul>
     </li>
     <li>
      <p>
       对于 CentOS 来说, 可以使用如下命令来安装:
      </p>
      <ul>
       <li>
        gcc:
        <code>
         sudo yum install gcc
        </code>
       </li>
       <li>
        clang:
        <code>
         sudo yum install clang
        </code>
       </li>
      </ul>
     </li>
    </ul>
    <h4>
     <a id="MacOS__C_36">
     </a>
     MacOS 下安装 C++编译器
    </h4>
    <p>
     对于 MacOS 来说, 如果有 Homebrew, 可以使用如下命令来安装:
    </p>
    <pre><code class="prism language-bash">brew <span class="token function">install</span> clang
</code></pre>
    <h3>
     <a id="_44">
     </a>
     安装插件
    </h3>
    <p>
     我们主要需要安装如下插件
     <a href="https://marketplace.visualstudio.com/items?itemName=ms-vscode.cpptools-extension-pack" rel="nofollow">
      C/C++ Extension Pack
     </a>
     .
    </p>
    <p>
     它是一个扩展包, 包含了 C/C++ 的所有功能. 包括:
    </p>
    <ul>
     <li>
      <a href="https://marketplace.visualstudio.com/items?itemName=ms-vscode.cpptools" rel="nofollow">
       C/C++
      </a>
     </li>
     <li>
      <a href="https://marketplace.visualstudio.com/items?itemName=ms-vscode.cpptools-themes" rel="nofollow">
       C/C++ Themes
      </a>
     </li>
     <li>
      <a href="https://marketplace.visualstudio.com/items?itemName=ms-vscode.cmake-tools" rel="nofollow">
       CMake Tools
      </a>
     </li>
    </ul>
    <h3>
     <a id="_54">
     </a>
     配置工作区
    </h3>
    <p>
     我们使用 CMake 来配置我们的项目. 这里我使用一个
     <a href="https://gitee.com/aronic/cmake-project-2024" rel="nofollow">
      示例项目
     </a>
     来演示如何配置工作区.
    </p>
    <p>
     这个项目使用 CMake 来构建, 并且使用 CMakePreset.json 来配置工作区. 如果对这两个工具不熟悉, 请参考我的文章:
     <a href="https://blog.csdn.net/arong_xu/article/details/145123224">
      CMake 教程: 从基础到进阶配置
     </a>
    </p>
    <p>
     操作步骤如下:
    </p>
    <ol>
     <li>
      <p>
       clone 项目:
      </p>
      <pre><code class="prism language-bash"><span class="token function">git</span> clone https://gitee.com/aronic/cmake-project-2024.git
</code></pre>
     </li>
     <li>
      <p>
       用 VS Code 打开项目.
      </p>
     </li>
     <li>
      <p>
       选择 CMake Preset: 按下
       <code>
        shift + cmd + p
       </code>
       , 输入
       <code>
        CMake: Select a CMake Preset
       </code>
       , 选择对应的 preset. 目前只有两个, 分别针对 Linux 和 Windows.
      </p>
      <p>
       <img alt="select preset" src="https://i-blog.csdnimg.cn/direct/8a8d3499705142ec9df8a07958dfec98.jpeg#pic_center" width="500"/>
      </p>
     </li>
     <li>
      <p>
       运行构建: 按下
       <code>
        shift + cmd + p
       </code>
       , 输出
       <code>
        CMake: Build
       </code>
       , 运行构建.
      </p>
      <p>
       <img alt="cmake build" src="https://i-blog.csdnimg.cn/direct/461c59fae10045dbbfe35967788c9d9e.jpeg#pic_center" width="500"/>
      </p>
      <p>
       构建结果:
      </p>
      <p>
       <img alt="build result" src="https://i-blog.csdnimg.cn/direct/5ee577cd408840058226b16713a74854.jpeg#pic_center" width="600"/>
      </p>
     </li>
     <li>
      <p>
       运行测试: 按下
       <code>
        shift + cmd + p
       </code>
       , 输出
       <code>
        CMake: Run Test
       </code>
       , 运行测试.
      </p>
      <p>
       <img alt="run test" src="https://i-blog.csdnimg.cn/direct/97c9c3e122214d71bce949b85f469cc4.jpeg#pic_center" width="500"/>
      </p>
      <p>
       测试结果:
      </p>
      <p>
       <img alt="run test result" src="https://i-blog.csdnimg.cn/direct/7e8ac10a0e11437f85d98c3fa90f90c7.jpeg#pic_center" width="600"/>
      </p>
     </li>
    </ol>
    <p>
     上述操作在底层状态栏也可以操作.
    </p>
    <p>
     <img alt="status bar" src="https://i-blog.csdnimg.cn/direct/5bbbf3fd63d441e2aa51a95e8741b37a.jpeg#pic_center" width="600"/>
    </p>
    <p>
     至此, 我们已经完成了 C++开发环境的搭建.
    </p>
