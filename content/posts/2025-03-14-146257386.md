    <h2>
     <a id="_42">
     </a>
     二、分析属性区别
    </h2>
    <h3>
     <a id="1_Name__43">
     </a>
     1.
     <code>
      Name
     </code>
     属性
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      : 获取类型的简单名称，即不包含命名空间的类型名称。
     </li>
     <li>
      <strong>
       示例
      </strong>
      : 对于一个名为
      <code>
       ReflectionDemo.User
      </code>
      的类，
      <code>
       Name
      </code>
      属性将返回
      <code>
       "User"
      </code>
      。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 当你需要获取类型不带命名空间的名称时使用。
     </li>
    </ul>
    <h3>
     <a id="2__FullName_48">
     </a>
     2.
     <code>
      FullName
     </code>
     属性
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      : 获取该类型的完全限定名称，包括其命名空间，但不包括程序集。
     </li>
     <li>
      <strong>
       示例
      </strong>
      : 对于一个名为
      <code>
       ReflectionDemo.User
      </code>
      的类，
      <code>
       FullName
      </code>
      属性将返回
      <code>
       ReflectionDemo.User
      </code>
      。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 当你需要唯一标识一个类型（在其所在的程序集中）而不关心其所属程序集时使用。
     </li>
    </ul>
    <h3>
     <a id="3_Namespace_53">
     </a>
     3.
     <code>
      Namespace
     </code>
     属性
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      : 获取声明该类型的命名空间。
     </li>
     <li>
      <strong>
       示例
      </strong>
      : 对于一个名为
      <code>
       ReflectionDemo.User
      </code>
      的类，
      <code>
       Namespace
      </code>
      属性将返回
      <code>
       "ReflectionDemo"
      </code>
      。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 当你需要知道某个类型属于哪个命名空间时使用，这对于避免命名冲突非常有用。
     </li>
    </ul>
    <h3>
     <a id="4_AssemblyQualifiedName_58">
     </a>
     4.
     <code>
      AssemblyQualifiedName
     </code>
     属性
    </h3>
    <ul>
     <li>
      <strong>
       定义
      </strong>
      : 获取类型的程序集限定名，包括命名空间、程序集名称、版本号、文化信息和公钥标记等。这是类型在程序集中的唯一标识。
     </li>
     <li>
      <strong>
       示例
      </strong>
      : 对于一个名为
      <code>
       ReflectionDemo.User
      </code>
      且位于
      <code>
       ReflectionDemo
      </code>
      程序集中的类，
      <br/>
      <code>
       AssemblyQualifiedName
      </code>
      会返回类似
      <code>
       ReflectionDemo.User, ReflectionDemo, Version=1.0.0.0, Culture=neutral, PublicKeyToken=null
      </code>
      的字符串。
     </li>
     <li>
      <strong>
       用途
      </strong>
      : 当你需要加载类型或进行反射操作时，通常需要使用
      <code>
       AssemblyQualifiedName
      </code>
      来精确指定类型的位置和版本信息。这对于
      <strong>
       跨程序集识别类型
      </strong>
      特别重要。
     </li>
    </ul>
    <h2>
     <a id="AssemblyQualifiedName__64">
     </a>
     三、AssemblyQualifiedName 详解
    </h2>
    <h3>
     <a id="1__65">
     </a>
     1. 定义
    </h3>
    <p>
     <code>
      AssemblyQualifiedName
     </code>
     是类型的程序集限定名，是一个字符串，这是类型在程序集中的唯一标识。它不仅包含类型的完全限定名（命名空间 + 类型名称），还包含了程序集的详细信息（如名称、版本、文化、公钥令牌等）。这使得
     <code>
      AssemblyQualifiedName
     </code>
     可以在不同程序集中准确地定位和加载特定类型。
    </p>
    <h3>
     <a id="2__68">
     </a>
     2. 格式
    </h3>
    <p>
     <code>
      AssemblyQualifiedName
     </code>
     的格式如下：
    </p>
    <pre><code>Namespace.TypeName, AssemblyName, Version=Major.Minor.Build.Revision, Culture=culture, PublicKeyToken=publicKeyToken
</code></pre>
    <ul>
     <li>
      <strong>
       Namespace.TypeName
      </strong>
      : 类型的完全限定名（包括命名空间）。
     </li>
     <li>
      <strong>
       AssemblyName
      </strong>
      : 程序集的名称（不包括扩展名）。
     </li>
     <li>
      <strong>
       Version
      </strong>
      : 程序集的版本号（例如：1.0.0.0）。
     </li>
     <li>
      <strong>
       Culture
      </strong>
      : 程序集的文化信息（通常是 “neutral”，除非程序集是特定文化的）。
     </li>
     <li>
      <strong>
       PublicKeyToken
      </strong>
      : 程序集的公钥令牌，用于验证程序集的发布者身份。
     </li>
    </ul>
    <p>
     <strong>
      示例
     </strong>
     ：
     <code>
      MyNamespace.MyClass, MyLibrary, Version=1.0.0.0, Culture=neutral, PublicKeyToken=abcdef1234567890
     </code>
    </p>
    <h3>
     <a id="3__83">
     </a>
     3. 类型的程序集限定名获取方式
    </h3>
    <ul>
     <li>
      <p>
       通过Type 实例的
       <code>
        AssemblyQualifiedName
       </code>
       属性获取
      </p>
      <pre><code class="prism language-csharp">type<span class="token punctuation">.</span>AssemblyQualifiedName
</code></pre>
     </li>
     <li>
      <p>
       通过手动按格式组装定义获取
      </p>
      <pre><code class="prism language-csharp"><span class="token class-name"><span class="token keyword">string</span></span> assemblyQualifiedName <span class="token operator">=</span> <span class="token string">"MyNamespace.MyClass, MyLibrary, Version=1.0.0.0, Culture=neutral, PublicKeyToken=abcdef1234567890"</span><span class="token punctuation">;</span>
</code></pre>
     </li>
    </ul>
    <h3>
     <a id="4__96">
     </a>
     4. 程序集的强名称和弱名称
    </h3>
    <h4>
     <a id="1Strong_Name_97">
     </a>
     1）强名称（Strong Name）
    </h4>
    <p>
     强名称是一个程序集的唯一标识符，由以下部分组成：
    </p>
    <ul>
     <li>
      <strong>
       简单名称
      </strong>
      ：程序集的文件名（不包括扩展名）。
     </li>
     <li>
      <strong>
       版本号
      </strong>
      ：程序集的版本信息（例如：1.0.0.0）。
     </li>
     <li>
      <strong>
       文化标识
      </strong>
      ：通常为 “neutral”，除非程序集是特定于某种文化的。
     </li>
     <li>
      <strong>
       公钥/令牌
      </strong>
      ：用于验证程序集的发布者身份。公钥令牌是公钥的缩写形式。
     </li>
    </ul>
    <p>
     强名称确保了程序集的唯一性，并且允许 .NET 运行时加载正确的程序集版本，防止不同版本之间的冲突。强名称通过使用私钥签名生成，并可以通过公钥令牌进行验证。
    </p>
    <h4>
     <a id="2Weak_Name_107">
     </a>
     2）弱名称（Weak Name）
    </h4>
    <p>
     弱名称指的是没有强名称签名的程序集。这种程序集没有版本控制和唯一性的保障，因此不能保证跨应用程序的一致性和安全性。
     <strong>
      弱名称的程序集仅包含其文件名和可能的版本信息，但不包含公钥令牌和文化标识等强名称元素
     </strong>
     。
    </p>
    <blockquote>
     <p>
      我们创建的一个类库，如果没有特殊操作都属于是是弱名称程序集
     </p>
    </blockquote>
    <h3>
     <a id="5_AssemblyQualifiedName__112">
     </a>
     5. AssemblyQualifiedName 和程序集的强名称和弱名称的关系
    </h3>
    <h4>
     <a id="1__114">
     </a>
     1）
     <strong>
      强名称程序集
     </strong>
    </h4>
    <ul>
     <li>
      使用
      <code>
       AssemblyQualifiedName
      </code>
      时，如果程序集具有强名称，则必须包含版本、文化、公钥令牌等信息。
     </li>
     <li>
      示例：
      <code>
       MyNamespace.MyClass, MyLibrary, Version=1.0.0.0, Culture=neutral, PublicKeyToken=abcdef1234567890
      </code>
     </li>
     <li>
      这种情况下，
      <code>
       AssemblyQualifiedName
      </code>
      可以唯一标识并加载特定版本的程序集中的类型。
     </li>
    </ul>
    <h4>
     <a id="2_119">
     </a>
     2）
     <strong>
      弱名称程序集
     </strong>
    </h4>
    <ul>
     <li>
      如果程序集没有强名称，则
      <code>
       AssemblyQualifiedName
      </code>
      中可以省略版本、文化和公钥令牌等信息。
     </li>
     <li>
      示例：
      <code>
       MyNamespace.MyClass, MyLibrary
      </code>
     </li>
     <li>
      在这种情况下，程序集的唯一性依赖于其文件名和路径，而不是强名称提供的额外标识信息。
     </li>
    </ul>
    <h3>
     <a id="6_AssemblyQualifiedName__126">
     </a>
     6. AssemblyQualifiedName 的应用
    </h3>
    <h4>
     <a id="1_127">
     </a>
     1）应用
    </h4>
    <p>
     将
     <code>
      AssemblyQualifiedName
     </code>
     作为
     <code>
      Type.GetType
     </code>
     方法的参数，动态加载指定类型。
    </p>
    <pre><code class="prism language-csharp"><span class="token comment">// 类型的程序集限定名 - 强名称</span>

<span class="token class-name"><span class="token keyword">var</span></span> type <span class="token operator">=</span> Type<span class="token punctuation">.</span><span class="token function">GetType</span><span class="token punctuation">(</span>assemblyQualifiedName1<span class="token punctuation">)</span><span class="token punctuation">;</span>

</code></pre>
    <h4>
     <a id="2_140">
     </a>
     2）示例代码
    </h4>
    <p>
     <code>
      AssemblyQualifiedName
     </code>
     在实际应用中主要用于通过反射加载类型，尤其是在跨程序集的情况下。以下是一个具体的案例，展示如何使用
     <code>
      AssemblyQualifiedName
     </code>
     来动态加载类型并创建实例。
    </p>
    <h5>
     <a id="_144">
     </a>
     案例背景
    </h5>
    <p>
     假设你有两个项目：一个类库（
     <code>
      MyLibrary.dll
     </code>
     ）和一个控制台应用程序（
     <code>
      MyConsoleApp.exe
     </code>
     ）。类库中定义了一个名为
     <code>
      MyNamespace.MyClass
     </code>
     的类，并且你想在控制台应用程序中动态加载这个类并调用其方法。
    </p>
    <h5>
     <a id="_147">
     </a>
     步骤说明
    </h5>
    <ol>
     <li>
      <strong>
       定义类库项目
      </strong>
      (
      <code>
       MyLibrary.dll
      </code>
      )
     </li>
     <li>
      <strong>
       编写控制台应用程序
      </strong>
      (
      <code>
       MyConsoleApp.exe
      </code>
      ) 使用
      <code>
       AssemblyQualifiedName
      </code>
      动态加载并使用类库中的类型。
     </li>
    </ol>
    <h5>
     <a id="_MyLibrarydll_152">
     </a>
     类库代码 (
     <code>
      MyLibrary.dll
     </code>
     )
    </h5>
    <p>
     首先，我们定义一个简单的类库项目，包含一个类
     <code>
      MyClass
     </code>
     。
    </p>
    <pre><code class="prism language-csharp"><span class="token comment">// MyLibrary/MyClass.cs</span>
<span class="token keyword">namespace</span> <span class="token namespace">MyNamespace</span>
<span class="token punctuation">{<!-- --></span>
    <span class="token keyword">public</span> <span class="token keyword">class</span> <span class="token class-name">MyClass</span>
    <span class="token punctuation">{<!-- --></span>

    <h5>
     <a id="_218">
     </a>
     详细步骤解释
    </h5>
    <ol>
     <li>
      <p>
       <strong>
        获取
        <code>
         AssemblyQualifiedName
        </code>
       </strong>
       :
      </p>
      <ul>
       <li>
        在上面的示例中，
        <code>
         assemblyQualifiedName
        </code>
        包含了完整的类型名称、命名空间、程序集名称及其版本信息。
       </li>
       <li>
        注意：你需要确保程序集名称和版本信息与实际的类库匹配。如果类库是强命名的，还需要包括
        <code>
         PublicKeyToken
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        加载类型
       </strong>
       :
      </p>
      <ul>
       <li>
        使用
        <code>
         Type.GetType(assemblyQualifiedName)
        </code>
        方法来加载指定的类型。如果类型或程序集找不到，该方法将返回
        <code>
         null
        </code>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        创建实例
       </strong>
       :
      </p>
      <ul>
       <li>
        使用
        <code>
         Activator.CreateInstance(myType)
        </code>
        方法创建目标类型的实例。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        调用方法
       </strong>
       :
      </p>
      <ul>
       <li>
        使用
        <code>
         GetMethods()
        </code>
        或
        <code>
         GetMethod()
        </code>
        方法获取要调用的方法的信息。
       </li>
       <li>
        使用
        <code>
         Invoke()
        </code>
        方法调用该方法。
       </li>
      </ul>
     </li>
    </ol>

    <h2>
     <a id="_246">
     </a>
     四、汇总
    </h2>
    <ul>
     <li>
      <strong>
       Name
      </strong>
      : 类型的简单名称。
     </li>
     <li>
      <strong>
       FullName
      </strong>
      : 类型的完全限定名称，包括其命名空间，但不包括程序集。
     </li>
     <li>
      <strong>
       Namespace
      </strong>
      : 类型所在命名空间的名称。
     </li>
     <li>
      <strong>
       <code>
        AssemblyQualifiedName
       </code>
      </strong>
      : 获取类型的程序集限定名，包含类型及其所在程序集的详细信息，
      <strong>
       适用于通过反射加载类型
      </strong>
      。
     </li>
    </ul>
    <p>
     这些属性各有用途，选择哪一个取决于你的具体需求。例如，如果你只是想显示类型的名称给用户看，可能只需要
     <code>
      Name
     </code>
     或
     <code>
      FullName
     </code>
     ；而如果你需要动态加载类型，则可能需要用到
     <code>
      AssemblyQualifiedName
     </code>
     。
    </p>
