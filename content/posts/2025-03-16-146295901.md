    <hr/>
    <h4>
     <a id="1_Caddy__5">
     </a>
     <strong>
      1. Caddy 是什么？
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       Caddy
      </strong>
      是一个现代化的 Web 服务器，以其简单易用和自动化的 HTTPS 支持而闻名。
     </li>
     <li>
      它内置了 TLS 支持，可以自动为域名配置 HTTPS 证书。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="2_TLS__11">
     </a>
     <strong>
      2. TLS 是什么？
     </strong>
    </h4>
    <ul>
     <li>
      <strong>
       TLS（Transport Layer Security）
      </strong>
      是一种加密协议，用于保护网络通信的安全。
     </li>
     <li>
      它确保客户端和服务器之间的数据传输是加密的，防止被窃听或篡改。
     </li>
     <li>
      HTTPS 就是基于 TLS 的 HTTP 协议。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="3_Caddy__TLS__18">
     </a>
     <strong>
      3. Caddy 的 TLS 功能
     </strong>
    </h4>
    <h5>
     <a id="1__TLS__20">
     </a>
     <strong>
      1. 默认的 TLS 设置是安全的
     </strong>
    </h5>
    <ul>
     <li>
      Caddy 的默认 TLS 配置已经足够安全，通常不需要手动调整。
     </li>
     <li>
      初学者可以依赖默认设置，避免因配置错误导致安全问题。
     </li>
    </ul>
    <h5>
     <a id="2__24">
     </a>
     <strong>
      2. 本地/内部主机名使用自签名证书
     </strong>
    </h5>
    <ul>
     <li>
      对于本地或内部使用的域名（如
      <code>
       localhost
      </code>
      或内部网络域名），Caddy 会使用
      <strong>
       自签名证书
      </strong>
      。
     </li>
     <li>
      <strong>
       自签名证书
      </strong>
      ：由服务器自己生成的证书，不需要第三方机构签发。浏览器可能会提示不安全，但在内部网络中可以使用。
     </li>
    </ul>
    <h5>
     <a id="3__DNS__ACME_CA__28">
     </a>
     <strong>
      3. 公共 DNS 名称使用公共 ACME CA 证书
     </strong>
    </h5>
    <ul>
     <li>
      对于公共域名（如
      <code>
       example.com
      </code>
      ），Caddy 会从
      <strong>
       公共 ACME CA
      </strong>
      获取证书。
     </li>
     <li>
      <strong>
       ACME（Automated Certificate Management Environment）
      </strong>
      ：一种自动化证书管理协议，用于获取和更新 TLS 证书。
     </li>
     <li>
      Caddy 支持从
      <strong>
       Let’s Encrypt
      </strong>
      和
      <strong>
       ZeroSSL
      </strong>
      获取免费的 TLS 证书。
      <ul>
       <li>
        <strong>
         Let’s Encrypt
        </strong>
        和
        <strong>
         ZeroSSL
        </strong>
        是两家受信任的机构，提供免费的 TLS 证书。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="4__34">
     </a>
     <strong>
      4. 证书签发过程
     </strong>
    </h5>
    <ul>
     <li>
      为了获取证书，Caddy 会在端口 80 上发布一个临时的加密资源。
     </li>
     <li>
      <strong>
       CA（证书颁发机构）
      </strong>
      会检查这个资源，确认域名的所有权。
     </li>
     <li>
      如果验证通过，CA 会签发 TLS 证书，Caddy 会自动配置并使用该证书。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="4__41">
     </a>
     <strong>
      4. 初学者如何理解这些技术？
     </strong>
    </h4>
    <h5>
     <a id="1_TLS__43">
     </a>
     <strong>
      1. TLS 的作用
     </strong>
    </h5>
    <ul>
     <li>
      想象你在寄一封信，TLS 就像给信封上锁，只有收件人才能打开。
     </li>
     <li>
      HTTPS 就是基于 TLS 的“上锁”通信方式，确保数据在传输过程中不被窃听或篡改。
     </li>
    </ul>
    <h5>
     <a id="2_Caddy__47">
     </a>
     <strong>
      2. Caddy 的自动化
     </strong>
    </h5>
    <ul>
     <li>
      Caddy 就像一个“自动化助手”，它会自动为你处理 TLS 证书的申请、配置和更新。
     </li>
     <li>
      你只需要告诉 Caddy 你的域名，它就会帮你搞定一切。
     </li>
    </ul>
    <h5>
     <a id="3__vs__CA__51">
     </a>
     <strong>
      3. 自签名证书 vs 公共 CA 证书
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       自签名证书
      </strong>
      ：适合本地测试或内部网络，但浏览器可能会提示不安全。
     </li>
     <li>
      <strong>
       公共 CA 证书
      </strong>
      ：适合公共网站，浏览器会信任这些证书。
     </li>
    </ul>
    <h5>
     <a id="4_Lets_Encrypt__ZeroSSL_55">
     </a>
     <strong>
      4. Let’s Encrypt 和 ZeroSSL
     </strong>
    </h5>
    <ul>
     <li>
      这两家机构提供免费的 TLS 证书，帮助网站启用 HTTPS。
     </li>
     <li>
      它们通过 ACME 协议自动化证书的签发和更新。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="5__61">
     </a>
     <strong>
      5. 初学者如何实践？
     </strong>
    </h4>
    <h5>
     <a id="1__Caddy_63">
     </a>
     <strong>
      1. 安装 Caddy
     </strong>
    </h5>
    <ul>
     <li>
      在服务器上安装 Caddy：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> <span class="token parameter variable">-y</span> debian-keyring debian-archive-keyring apt-transport-https
<span class="token function">curl</span> <span class="token parameter variable">-1sLf</span> <span class="token string">'https://dl.cloudsmith.io/public/caddy/stable/gpg.key'</span> <span class="token operator">|</span> <span class="token function">sudo</span> gpg <span class="token parameter variable">--dearmor</span> <span class="token parameter variable">-o</span> /usr/share/keyrings/caddy-stable-archive-keyring.gpg
<span class="token function">curl</span> <span class="token parameter variable">-1sLf</span> <span class="token string">'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt'</span> <span class="token operator">|</span> <span class="token function">sudo</span> <span class="token function">tee</span> /etc/apt/sources.list.d/caddy-stable.list
<span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> caddy
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="2__Caddy_73">
     </a>
     <strong>
      2. 配置 Caddy
     </strong>
    </h5>
    <ul>
     <li>
      编辑 Caddy 配置文件（
      <code>
       /etc/caddy/Caddyfile
      </code>
      ）：
      <pre><code class="prism language-plaintext">example.com {
    respond "Hello, world!"
}
</code></pre>
     </li>
     <li>
      将
      <code>
       example.com
      </code>
      替换为你的域名。
     </li>
    </ul>
    <h5>
     <a id="3__Caddy_82">
     </a>
     <strong>
      3. 启动 Caddy
     </strong>
    </h5>
    <ul>
     <li>
      启动 Caddy 服务：
      <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl start caddy
<span class="token function">sudo</span> systemctl <span class="token builtin class-name">enable</span> caddy
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="4__HTTPS_89">
     </a>
     <strong>
      4. 测试 HTTPS
     </strong>
    </h5>
    <ul>
     <li>
      在浏览器中访问
      <code>
       https://example.com
      </code>
      ，确认网站已启用 HTTPS。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="6__94">
     </a>
     <strong>
      6. 总结
     </strong>
    </h4>
    <ul>
     <li>
      Caddy 是一个现代化的 Web 服务器，内置了自动化的 TLS 支持。
     </li>
     <li>
      它可以使用自签名证书保护本地/内部域名，也可以从 Let’s Encrypt 或 ZeroSSL 获取免费的 TLS 证书保护公共域名。
     </li>
     <li>
      初学者可以通过安装和配置 Caddy，快速实现 HTTPS 支持。
     </li>
    </ul>
