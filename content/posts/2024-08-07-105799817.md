    <h2>
     <a id="PycharmPython_Console_0">
     </a>
     解决Pycharm运行文件时总是使用Python Console执行的问题
    </h2>
    <h3>
     <a id="_2">
     </a>
     问题描述
    </h3>
    <p>
     近期发现Pycharm运行文件时没有使用以前的run窗口，而是使用Python Console来执行，用它执行代码有时会出现意外的行为（主线程结束时守护线程仍然不会结束），所以我希望只有需要交互模式的时候才会使用它来运行程序。
    </p>
    <h3>
     <a id="_5">
     </a>
     解决方案
    </h3>
    <h4>
     <a id="__RunDebug_Configuration__6">
     </a>
     第一步 打开 “
     <strong>
      Run/Debug Configuration
     </strong>
     ” 窗口
    </h4>
    <p>
     找到右上角的用于运行和测试区域，单击下拉菜单，然后再单击
     <code>
      Edit Configuration
     </code>
     。
     <br/>
     <img alt="点击Edit Configuration" src="https://i-blog.csdnimg.cn/blog_migrate/0617d46fbb9471cad13f37d82e5d5b3f.png"/>
    </p>
    <h4>
     <a id="__9">
     </a>
     第二步 编辑配置模板
    </h4>
    <p>
     在左上角找到这个小扳手并单击。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/887405e32c051fecf005820908dbdfa7.png"/>
    </p>
    <h4>
     <a id="_Run_with_Python_Console_13">
     </a>
     第三步 取消复选框“Run with Python Console”
    </h4>
    <p>
     把
     <code>
      Run with Python Console
     </code>
     复选框取消，然后点击右下角的
     <code>
      OK
     </code>
     保存配置，你会发现右键运行文件时就不会使用Python Console来执行了。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/cd09dfaffa5812441917a758d878f6c3.png"/>
    </p>
