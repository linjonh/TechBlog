    <h2>
     <a id="_11">
     </a>
     什么是单元测试
    </h2>
    <p>
     <strong>
      单元测试
     </strong>
     （unit testing）是指对软件中的最小可测试单元进行检查和验证。 单元测试中单元的含义，单元就是人为规定的最小的被测功能模块，如C语言中单元指一个函数，Java里单元指一个类，图形化的软件中可以指一个窗口或一个菜单等。
    </p>
    <h2>
     <a id="_14">
     </a>
     什么是单元测试框架
    </h2>
    <p>
     理论上来说，不使用单元测试框架也能进行单元测试。但如果用于单元测试的代码（即测试用例）增多，在没有测试框架的情况下会变得拥挤、不可管理，这个时候引入测试框架就变得尤为重要。
    </p>
    <p>
     单元测试框架提供了一种统一的编程模型，可以将测试定义为一些简单的类，这些类中的方法可以调用希望测试的应用程序代码。利用单元测试框架，可以很轻松地插入、设置和分解有关测试的功能，可以直观方便地管理测试用例。
    </p>
    <p>
     主流的单元测试框架，如Java的Junit、python的Unittest、Pytest
    </p>
    <p>
     单元测试框架作用：
    </p>
    <ul>
     <li>
      提供用例组织与执行
     </li>
     <li>
      提供丰富的断言方法
     </li>
     <li>
      提供丰富的日志与测试结果
     </li>
    </ul>
    <h2>
     <a id="Unittest__27">
     </a>
     Unittest 测试框架
    </h2>
    <p>
     Unittest是Python自带的单元测试框架，不仅适用于单元测试，还可用于Web、Appium、接口自动化测试用例的开发与执行。该测试框架可组织执行测试用例，并且提供丰富的断言方法，判断测试用例是否通过，并最终生成测试结果。
    </p>
    <p>
     Unittest官方文档：
     <a href="https://docs.python.org/3/library/unittest.html" rel="nofollow">
      unittest — Unit testing framework — Python 3.10.4 documentation
     </a>
    </p>
    <h3>
     <a id="Unittest_33">
     </a>
     Unittest核心要素
    </h3>
    <h4>
     <a id="TestCase_34">
     </a>
     TestCase
    </h4>
    <p>
     即测试用例，Unittest提供testCase类来编写测试用例，一个TestCase的实例就是一个测试用例。一条测试用例就是一个完整的测试流程，包括测试前准备环境的搭建(setUp)，执行测试代码(run)，以及测试后环境的还原(tearDown)，通过运行一条测试用例，可以对某一个问题进行验证。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8f11b708d2d34a7fac9da39c0b9ca0eb.png"/>
    </p>
    <h4>
     <a id="Fixture_38">
     </a>
     Fixture
    </h4>
    <p>
     即测试固件，用于测试用例环境的搭建和销毁。在测试步骤执行前需要为该测试用例准备环境（SetUp），如启动app或打开浏览器，测试步骤执行后需要恢复环境 （TearDown），如关闭app或浏览器，这时候就需要用到Fixture，使代码更简洁。
    </p>
    <h4>
     <a id="TestSuite_41">
     </a>
     TestSuite
    </h4>
    <p>
     即测试套件，把需要执行的测试用例集合在一起就是TestSuite。使用TestLoader来加载TestCase到TestSuite中。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2bf7172af9ab4e29ae47ac908b5d34b9.png"/>
    </p>
    <h4>
     <a id="TextTestRunner_44">
     </a>
     TextTestRunner
    </h4>
    <p>
     即测试执行器，用于执行测试套件。该模块中提供run方法执行TestSuite中的测试用例，并返回测试用例的执行结果，如运行的用例总数、用例通过数、用例失败数。
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/1091e5c0ba8144b3b373e71b5e2a6993.png"/>
    </p>
    <h4>
     <a id="report_48">
     </a>
     report
    </h4>
    <p>
     即测试报告。unittest框架没有自带的用于生成测试报告的模块或接口，需要使用第三方的扩展模块。
    </p>
    <h3>
     <a id="Unittest__52">
     </a>
     Unittest 断言
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/7a26db46c64345bdb843ee0151f1e7f2.png">
      <br/>
      补充:
      <br/>
      如果断言成功则该条测试用例通过，断言失败则该条测试用例执行失败，且会抛出AssertionError错误。
      <br/>
      以上提供的断言方法中，都有一个msg参数，默认为None。如果msg参数有对应的值，则断言失败后该msg的值会作为失败信息返回，如 assertEqual(a, b, msg=“a与b不相等！”) 。
     </img>
    </p>

     <a id="TestCase_80">

     编写TestCase（测试用例）
    </h3>
    <p>
     在Unittest框架下创建测试用例，步骤如下：
    </p>
    <ul>
     <li>
      1)，导入unittest模块。
     </li>
     <li>
      2)，创建测试类。测试类的命名不做要求，但需要继承unittest.TestCase类。
     </li>
     <li>
      3）添加setUp()、tearDown()函数，即测试固件。（非必须）
     </li>
     <li>
      4)，定义测试方法，即测试用例。测试方法名称必须以test开头，否则测试时该方法将不会被执行。测试方法里需要添加断言。
     </li>
     <li>
      5)，调试执行测试用例。执行当前模块的测试用例时，调用unittest.main()方法，该方法会搜索该模块下所有以test开头的测试用例方法，并执行。

    <h3>
     <a id="Fixture_142">
     </a>
     在测试用例中添加Fixture（测试夹具）
    </h3>
    <p>
     在Unittest框架下的测试用例中，使用Fixture有三种方法：
    </p>
    <p>
     setUp()、tearDown()，作用于测试方法。即测试类下的每个测试方法执行前先运行setUp()，每个测试方法执行完毕后都要执行tearDown()方法，如testSum.py示例。
     <br/>
     setUpClass()、tearDownClass()，作用于测试类。即只在整个测试类执行开始时运行setUpClass()，测试类下所有测试方法执行完后运行tearDownClass()。
     <strong>
      且需要使用装饰器@classmethod
     </strong>
     <br/>
     setUpModule()、tearDownModule()，作用于测试模块（可以理解为整个TestCase.py文件）。即只在整个测试模块执行开始时运行setUpModule()，测试类下所有测试模块执行完后运行tearDownModule()。
    </p>
    <p>
     <strong>
      以setUpClass()、tearDownClass() 举例
     </strong>
     <br/>
     test_sum.py修改如下，运行

     <a id="TestSuite_209">

     将测试用例添加至TestSuite（测试套件）
    </h3>
    <p>
     在testSum.py模块中，使用了unittest.main()方法执行当前模块里的测试用例。除此之外，Unittest还可以通过测试套件构造测试用例集，再执行测试用例。构造TestSuite常用的方法如下：
    </p>
    <p>
     4.1，方法一：加载测试用例
    </p>
    <ul>
     <li>
      <p>
       1)，先通过unittest.TestSuite() 创建测试套件实例对象。
      </p>
     </li>
     <li>
      <p>
       2)，再通过addTest() 往测试套件里添加单个测试用例，或通过addTests([…]) 添加多个测试用例（列表中为用例方法名）。
      </p>
     </li>
     <li>
      <p>
       3)，执行测试套件里的测试用例
      </p>
     </li>
    </ul>
    <p>
     run.py示例：
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">import</span> <span class="token module">unittest</span>

    <h3>
     <a id="_322">
     </a>
     输出测试报告
    </h3>
    <p>
     unittest框架执行测试用例完成后会在控制台输出如上的结果，但实际测试过程中，我们需要输出测试报告，这个时候我们需要使用第三方模块。例如 HTMLTestRunner，如果对该模块使用有需求可查看该博客
     <a href="https://blog.csdn.net/qaqqqqqaq_/article/details/135389040">
      软件测试——Unittest单元测试框架详解
     </a>
     ，或者查看该模块的官方文档。
    </p>
    <p>
     本文介绍的是另外的一个第三方模块，
     <strong>
      unittestreport
     </strong>
     。
    </p>
    <p>
     UnitTestReport 是一个基于 Python 的开源库，专为改进 unittest 框架的测试报告体验而生。它不仅解决了标准 unittest 报告样式单一且不够直观的问题，还添加了许多实用特性，如生成丰富的 HTML 测试报告、支持数据驱动测试、测试用例的失败重试、多线程并发执行、以及自动化测试结果的通知功能，如发送至邮箱或企业协作平台如钉钉和企业微信。
    </p>
    <p>
     该项目由木森发起并维护，旨在通过提供详细且易于理解的测试报告，提升软件开发过程中的测试质量和效率。无论是小型项目还是大型企业级应用的测试流程，UnitTestReport都能助力团队更有效地管理和分析测试结果。
    </p>
    <p>
     github地址：
     <a href="https://github.com/musen123/UnitTestReport">
      UnitTestReport
     </a>
     <br/>
     使用文档：
     <a href="https://unittestreport.readthedocs.io/en/latest/" rel="nofollow">
      UnitTestReport使用文档
     </a>
    </p>
    <p>
     目前UnitTestReport实现了以下功能：
    </p>
    <ul>
     <li>
      HTML测试报告生成
     </li>
     <li>
      unittest数据驱动
     </li>
     <li>
      测试用例失败重运行
     </li>
     <li>
      多线程并发执行用例
     </li>
     <li>
      发送测试结果及报告到邮箱
     </li>
     <li>
      测试结果推送到钉钉
     </li>
     <li>
      测试结构推送到企业微信
     </li>
     <li>
      使用pytest生成unittestreport的html报告（最新）
     </li>
    </ul>

     <strong>

    <h3>
     <a id="ddt_384">
     </a>
     ddt模块的实现步骤
    </h3>
    <p>
     我们设计测试用例时，会出现测试步骤一样，只是其中的测试数据有变化的情况，比如测试登录时的账号密码。这个时候，如果我们依然使用一条case一个方法的话，会出现大量的代码冗余，而且效率也会大大降低。此时，ddt模块就能帮助我们解决这个问题。
    </p>
    <p>
     ddt(data-driven test),顾名思义，数据驱动测试。这个模块是第三方库，需要我们自己下载。或者直接在命令行输入pip install ddt。
    </p>
    <p>
     使用方法：
    </p>
    <ol>
     <li>
      使用数据驱动，要在class前加上修饰器 @ddt
     </li>
    </ol>
    <pre><code class="prism language-cpp"><span class="token keyword">import</span> <span class="token module">unittest</span>
from ddt <span class="token keyword">import</span> <span class="token module">ddt</span><span class="token punctuation">,</span> data
 
@ddt  
<span class="token keyword">class</span> <span class="token class-name">TestDemo</span><span class="token punctuation">(</span>unittest<span class="token punctuation">.</span>TestCase<span class="token punctuation">)</span><span class="token operator">:</span>
    # 单一参数
    @<span class="token function">data</span><span class="token punctuation">(</span><span class="token char">'17611110000'</span><span class="token punctuation">,</span> <span class="token char">'17611112222'</span><span class="token punctuation">)</span>
    def <span class="token function">test_1</span><span class="token punctuation">(</span>self<span class="token punctuation">,</span> phone<span class="token punctuation">)</span><span class="token operator">:</span>
        <span class="token function">print</span><span class="token punctuation">(</span><span class="token char">'测试一电话号码：'</span><span class="token punctuation">,</span> phone<span class="token punctuation">)</span>
        
<span class="token keyword">if</span> __name__ <span class="token operator">==</span> <span class="token char">'__main__'</span><span class="token operator">:</span>
    unittest<span class="token punctuation">.</span><span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token keyword">else</span><span class="token operator">:</span>
    pass
</code></pre>
    <ol start="2">
     <li>
      在实际中不一定是单一参数进行传参，将会使用多个参数进行传参：
     </li>
    </ol>
    <p>
     注意事项：
     <br/>
     1）、多个数据传参的时候@data里面是要用列表形式
     <br/>
     2）、会用到 @unpack 装饰器 进行拆包，把对应的内容传入对应的参数；
    </p>
    <pre><code class="prism language-cpp"><span class="token keyword">import</span> <span class="token module">unittest</span>
from ddt <span class="token keyword">import</span> <span class="token module">ddt</span><span class="token punctuation">,</span> data<span class="token punctuation">,</span> unpack
 
@ddt
<span class="token keyword">class</span> <span class="token class-name">TestDemo</span><span class="token punctuation">(</span>unittest<span class="token punctuation">.</span>TestCase<span class="token punctuation">)</span><span class="token operator">:</span>
    # 多参数数据驱动
    @<span class="token function">data</span><span class="token punctuation">(</span><span class="token punctuation">[</span><span class="token char">'admin'</span><span class="token punctuation">,</span> <span class="token char">'123456'</span><span class="token punctuation">]</span><span class="token punctuation">)</span>
    <span class="token macro property"><span class="token directive-hash">#</span> <span class="token directive keyword">unpack</span> <span class="token expression">是进行拆包，不然会把列表里面的数据全部传到username这个一个参数，我们要实现列表中的两个数据分别传入对应的变量中</span></span>
    @unpack
    def <span class="token function">test_2</span><span class="token punctuation">(</span>self<span class="token punctuation">,</span> username<span class="token punctuation">,</span> password<span class="token punctuation">)</span><span class="token operator">:</span>
        <span class="token function">print</span><span class="token punctuation">(</span><span class="token char">'测试二：'</span><span class="token punctuation">,</span> username<span class="token punctuation">,</span> password<span class="token punctuation">)</span>
 
 
<span class="token keyword">if</span> __name__ <span class="token operator">==</span> <span class="token char">'__main__'</span><span class="token operator">:</span>
    unittest<span class="token punctuation">.</span><span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token keyword">else</span><span class="token operator">:</span>
    pass
</code></pre>
    <p>
     但是以上步骤都是数据在代码当中的，假如要测试n个手机号这样的数据，全部写在 @data 装饰器里面就很麻烦，这就引出了数据驱动里面的代码和数据的分离。
     <br/>
     将数据放入一个文本文件中，从文件读取数据， 如JSON、 excel、 xml、 txt等格式文件
    </p>

    pass

     注意事项：
     <br/>
     1、with open里面默认是 ”r“
