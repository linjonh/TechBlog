    <h4>
     1、同步代码和异步代码
    </h4>
    <p>
     同步代码：浏览器按照书写代码的顺序一行一行去执行代码，逐行执行，需要原地等待结果后，才继续向下执行
    </p>
    <p>
     异步代码：调用后耗时，不阻塞代码继续执行，在将来完成后触发一个回调函数
    </p>
    <h4>
     2、回调函数地狱和Promise链式调用
    </h4>
    <p>
     概念：在回调函数中嵌套回调函数，从而形成回调函数地狱
    </p>
    <p>
     缺点：可读性差，异常无法捕获，耦合性严重，牵一发动全身
    </p>
    <p>
     promise-链式调用
    </p>
    <p>
     概念：依靠then()方法会返回一个新生成的Promise对象特性，继续串联下一环任务，直到结束
    </p>
    <p>
     细节：then()回调函数中的返回值，会影响新生成的Promise对象最终状态和结果
    </p>
    <p>
     例如：
    </p>
    <pre><code class="language-javascript">    &lt;script&gt;

    const p=new Promise((resolve,reject) =&gt; {
        setTimeout(() =&gt; {
            //获取省份列表
            resolve('湖北省')
        },500)
    })

    p.then(res =&gt; {
        console.log(res)
        return new Promise((resolve,reject) =&gt; {
            setTimeout(() =&gt; {
                //获取省份列表
                resolve(res+'武汉市')
            },500)
        })
    }).then(res =&gt; {
        console.log(res)
        return new Promise((resolve,reject) =&gt; {
            setTimeout(() =&gt; {
                //获取省份列表
                resolve(res+'黄陂区')
            },500)
        })
    }).then(res =&gt; {
        console.log(res)
    })</code></pre>
    <p>
     也就是将代码的嵌套换成了串式，不会形成回调地狱
    </p>
    <p>
     每个promise对象中管理一个异步任务，用then返回promise对象，串联起来
    </p>
    <h4>
     3、async和await使用
    </h4>
    <p>
     async和await关键字让我们可以用一种更简洁的方式写出基于Promise的异步行为，而无需刻意地链式调用Promise
    </p>
    <p>
     async关键字只能用在函数前，给函数进行修饰
    </p>
    <p>
     await必须结合async使用
    </p>
    <p>
     例如：
    </p>
    <pre><code class="language-javascript">        async function getPro() {
            const res=await axios({url:'http://ajax-api.itheima.net/api/province'})
            const app=new Vue({
                el:'#province',
                data:{
                    pList:res.data.data,
                    pname:res.data.data[0]
                }
            })
        }

        getPro()</code></pre>
    <h5>
     * 异常捕获
    </h5>
    <p>
     使用：try...catch
    </p>
    <h4>
     4、事件循环-EventLoop
    </h4>
    <p>
     概念：事件循环负责执行代码、收集和处理事件以及执行队列中的子任务
    </p>
    <p>
     原因：JavaScript单线程，为了让耗时的代码不阻塞其他代码的执行，设计了事件循环模型
    </p>
    <h5>
     * 宏任务于微任务
    </h5>
    <p>
     异步任务分为：
    </p>
    <p>
     宏任务：由浏览器环境执行的异步代码
    </p>
    <p>
     微任务：由JS引擎环境执行的异步代码
    </p>

    <p>
     微任务：Promise对象.then()
    </p>
    <p>
     Promise本身是同步的，而then和catch回调函数是异步的
    </p>
    <h4 style="background-color:transparent">
     5、Promise.all静态方法
    </h4>
    <p>
     概念：合并多个Promise对象，等待所有同时成功完成（或某一个失败），做后续逻辑
    </p>
    <p>
     语法：
    </p>
    <pre><code class="language-javascript">        const p=Promise.all([Promise对象,Promise对象,...])
        p.then(result =&gt; {
            //result结果：... 成功
        }).catch(error =&gt; {
            //error结果：第一个失败的对象
        })</code></pre>
    <p>
     可以实现，在依次发出请求后，实现同时渲染的功能，返回的结果为一个数组
    </p>
