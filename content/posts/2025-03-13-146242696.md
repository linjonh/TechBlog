     <strong>
      经典场景
     </strong>
     ：
     <br/>
     • 全局弹窗/模态框
     <br/>
     • 全屏Loading
     <br/>
     • 通知提示

     高频面试题
    </h4>
    <ol>
     <li>
      <p>
       Teleport组件能否和父组件通信？如何实现？
      </p>
      <p>
       <strong>
        答案
       </strong>
       ：
       <br/>
       Teleport 组件
       <strong>
        可以
       </strong>
       与父组件正常通信，因为它的
       <strong>
        逻辑作用域仍然属于父组件
       </strong>
       ，只是
       <strong>
        DOM 结构被渲染到其他位置
       </strong>
       。通信方式与普通组件完全一致：
      </p>
      <ul>
       <li>
        <strong>
         Props 传递
        </strong>
        ：父组件通过
        <code>
         props
        </code>
        向 Teleport 内的子组件传值
       </li>
       <li>
        <strong>
         事件触发
        </strong>
        ：子组件通过
        <code>
         emit
        </code>
        触发父组件的事件
       </li>
       <li>
        <strong>
         依赖注入
        </strong>
        ：使用
        <code>
         provide/inject
        </code>
        跨层级通信
       </li>
      </ul>
     </li>
     <li>
      <p>
       Suspense是否可以捕获子组件内的所有异步操作？
       <br/>
       <strong>
        答案
       </strong>
       ：
       <br/>
       Suspense ​
       <strong>
        只能捕获以下两种异步场景
       </strong>
       ：
      </p>
      <ol>
       <li>
        <strong>
         异步组件加载
        </strong>
        （
        <code>
         defineAsyncComponent
        </code>
        加载的组件）
       </li>
       <li>
        <strong>
         组件顶层
         <code>
          async setup()
         </code>
        </strong>
        （组件
        <code>
         setup
        </code>
        函数返回 Promise）
       </li>
      </ol>
      <p>
       <strong>
        无法捕获的场景
       </strong>
       ：
      </p>
      <ul>
       <li>
        子组件内的生命周期钩子中的异步操作（如
        <code>
         onMounted
        </code>
        中的
        <code>
         setTimeout
        </code>
        ）
       </li>
       <li>
        事件处理函数中的异步操作（如按钮点击后的
        <code>
         fetch
        </code>
        请求）
       </li>
       <li>
        非顶层
        <code>
         setup
        </code>
        中的异步逻辑
       </li>
      </ul>
      <p>
       <strong>
        解决方案
       </strong>
       ：
       <br/>
       对于其他异步场景，需自行实现 Loading 状态（如使用
       <code>
        v-if
       </code>
       + 加载状态变量）。
      </p>
     </li>

      </p>

          </strong>

          </strong>

         </td>
        </tr>
        <tr>
         <td align="center">
          <strong>
           数据流
          </strong>
         </td>

         </td>

          </strong>
         </td>
         <td align="center">

         </td>

         </td>
        </tr>
        <tr>
         <td align="center">

          </strong>
         </td>
         <td align="center">

         </td>
        </tr>

      <p>
       <strong>
        使用场景对比
       </strong>

      </p>

          <code>
           v-click-outside
          </code>
          )
         </li>
         <li>

          <code>
           v-infinite-scroll
          </code>
          )
         </li>
        </ul>
       </li>
       <li>
        <strong>
         适合组件的场景

        <ul>

       </li>

       </li>

        <strong>

        或
        <strong>

        </strong>
        时用组件
       </li>

     </li>
