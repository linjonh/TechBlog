    <h3>
     <a id="_LeetCode_2208__5">
     </a>
     🌟 LeetCode 2208. 将数组和减半的最少操作次数：贪心策略与大根堆的完美结合
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f1b912307914441c9b9cace558d4a366.png"/>
    </p>
    <h4>
     <a id="__8">
     </a>
     🔍 问题描述
    </h4>
    <p>
     给定一个正整数数组
     <code>
      nums
     </code>
     ，每次操作可以选择任意一个元素将其减半（可对同一元素多次操作）。要求找到使数组总和
     <strong>
      至少减少一半
     </strong>
     所需的
     <strong>
      最少操作次数
     </strong>
     。
    </p>
    <h5>
     <a id="_11">
     </a>
     示例分析
    </h5>
    <p>
     以示例
     <code>
      nums = [5,19,8,1]
     </code>
     为例：
    </p>
    <ul>
     <li>
      <strong>
       初始总和
      </strong>
      ：33 →
      <strong>
       目标减少量
      </strong>
      ：16.5
     </li>
     <li>
      <strong>
       操作路径
      </strong>
      ：19→9.5→4.75、8→4
     </li>
     <li>
      <strong>
       最终总和
      </strong>
      ：14.75（减少量18.25 ≥ 16.5）
     </li>
     <li>
      <strong>
       最少操作次数
      </strong>
      ：3
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="__20">
     </a>
     💡 算法思路
    </h4>
    <h5>
     <a id="1__21">
     </a>
     1. 贪心策略的选择
    </h5>
    <p>
     每次选择当前数组中的
     <strong>
      最大值
     </strong>
     进行减半操作，这是最优策略的数学基础：
    </p>
    <ul>
     <li>
      <strong>
       交换论证法
      </strong>
      ：假设某次未选最大值而选较小值，后续操作需补偿更多次数。例如，若未减半最大值
      <code>
       x
      </code>
      而选较小值
      <code>
       y
      </code>
      ，后续仍需处理
      <code>
       x
      </code>
      ，总操作次数必然更多。
     </li>
     <li>
      <strong>
       直观理解
      </strong>
      ：最大值贡献的减少量最大（
      <code>
       x/2
      </code>
      ），优先处理能最快逼近目标。
     </li>
    </ul>
    <h5>
     <a id="2__26">
     </a>
     2. 大根堆的运用
    </h5>
    <p>
     直接遍历数组找最大值的时间复杂度为 O(n²)，会超时（如
     <code>
      n=1e5
     </code>
     时）。使用**优先队列（大根堆）**优化：
    </p>
    <ul>
     <li>
      <strong>
       堆特性
      </strong>
      ：取最大值 O(1)，插入 O(log n)。
     </li>
     <li>
      <strong>
       动态维护
      </strong>
      ：每次减半后重新插入堆，保持堆的有效性。
     </li>
    </ul>

     <a id="__43">

     🛠 代码实现解析
    </h4>
    <pre><code class="prism language-cpp"><span class="token keyword">class</span> <span class="token class-name">Solution</span> <span class="token punctuation">{<!-- --></span>
<span class="token keyword">public</span><span class="token operator">:</span>
