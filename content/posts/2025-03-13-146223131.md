    <p>
     <strong>
      虚拟化数据恢复环境&amp;故障：
     </strong>
     <br/>
     VMware虚拟化平台
     <br/>
     vmfs文件系统
     <br/>
     工作人员误操作重装操作系统，服务器崩溃。
     <br/>
     重装系统会导致文件系统元文件被覆盖。要恢复数据，必须找到&amp;提取重装系统前的文件系统残留信息，通过提取出来的元文件信息恢复虚拟磁盘。通过拼接虚拟磁盘来恢复服务器数据。
    </p>
    <p>
     <strong>
      虚拟化数据恢复过程：
     </strong>
     <br/>
     1、将服务器中所有硬盘标记后取出，以只读方式进行全盘镜像。镜像完成后将所有硬盘按照原样还原到原服务器中。后续的数据分析和数据恢复操作都基于镜像文件进行，避免对原始磁盘数据造成二次破坏。
     <br/>
     基于镜像文件分析底层数据，计算原始分区信息。
     <br/>
     分析服务器上的LVM信息，计算出被重装系统的PV原始分区起始位置和大小等信息。
     <br/>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/44fe3edf67bc49c8b7331af8adeef2c6.jpeg">
      <br/>
      2、经过分析发现LVM信息破坏，北亚企安数据恢复工程师根据各个元文件entry标志将原文件系统内的所有残留的元文件信息全部提取出来，才能进行重组并解析文件系统。
      <br/>
      <img alt="" src="https://i-blog.csdnimg.cn/direct/a8831fed56da4505a8d10fdb208f19d4.jpeg">
       <img alt="" src="https://i-blog.csdnimg.cn/direct/e4cacb69b45141a19bb72da0cb14f7fc.jpeg">
        <br/>
        3、提取pbc中所有类型为3的子块。
        <br/>
        <img alt="" src="https://i-blog.csdnimg.cn/direct/340e2fc894ac455e966505f41d91643d.jpeg">
         <img alt="" src="https://i-blog.csdnimg.cn/direct/ba5935b06cca41efa2b8458d52440764.jpeg">
          <br/>
          4、扫描&amp;解析fdc元文件中的有效节点，提取虚拟机。
          <br/>
          <img alt="" src="https://i-blog.csdnimg.cn/direct/4f6bbca3a7234fa5802614e7677d2d7e.jpeg">
           <img alt="" src="https://i-blog.csdnimg.cn/direct/721265c60a5f44ec93faba98541b7ec9.jpeg">
            <br/>
            5、提取pbc元文件中类型为1的指针块，扫描指针并记录位置。北亚企安数据恢复工程师根据文件系统的连续性拼接虚拟机。
            <br/>
            <img alt="" src="https://i-blog.csdnimg.cn/direct/e053e773b8f142ba92335438873e2b5f.jpeg">
             <img alt="" src="https://i-blog.csdnimg.cn/direct/e88cf9eeec8844c688e0e3999505ca56.jpeg">
              <img alt="" src="https://i-blog.csdnimg.cn/direct/b8805b0ad89a491fa33037394b1897c8.jpeg">
               <br/>
               6、对拼接后的虚拟机进行文件系统验证，确认无误后挂载虚拟磁盘并重启即可恢复虚拟机。
               <br/>
               7、经过用户方工程师的验证，确认恢复数据完整有效。
