    <h4>
     一、光栅化 (Rasterization)
    </h4>
    <p>
     <strong>
      1.1 原理
     </strong>
    </p>
    <p>
     光栅化是实时渲染中最常用的技术。它的核心思想是将 3D 场景中的几何体投影到 2D 屏幕上，并逐个像素地计算颜色值。具体步骤如下：
    </p>
    <ol>
     <li>
      <strong>
       顶点处理
      </strong>
      : 将 3D 模型的顶点坐标转换到屏幕空间。
     </li>
     <li>
      <strong>
       三角形遍历
      </strong>
      : 将三角形拆分成像素大小的片段。
     </li>
     <li>
      <strong>
       片段处理
      </strong>
      : 对每个片段进行着色计算，包括纹理采样、光照计算等。
     </li>
     <li>
      <strong>
       深度测试
      </strong>
      : 比较片段的深度值，决定是否更新像素颜色。
     </li>
     <li>
      <strong>
       颜色混合
      </strong>
      : 将片段颜色与帧缓冲区中的颜色进行混合。
     </li>
    </ol>
    <p>
     <strong>
      1.2 优缺点
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       优点
      </strong>
      :
      <ul>
       <li>
        <strong>
         速度快
        </strong>
        : 光栅化经过多年优化，在硬件上运行效率极高，能够满足实时渲染的需求。
       </li>
       <li>
        <strong>
         硬件支持
        </strong>
        : 几乎所有显卡都支持光栅化，兼容性好。
       </li>
       <li>
        <strong>
         易于实现
        </strong>
        : Unity3D 提供了完善的渲染管线，开发者可以轻松实现各种渲染效果。
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <ul>
     <li>
      <strong>
       缺点
      </strong>
      :
      <ul>
       <li>
        <strong>
         光照效果有限
        </strong>
        : 光栅化通常使用简化光照模型，难以实现全局光照、阴影等复杂效果。
       </li>
       <li>
        <strong>
         几何细节不足
        </strong>
        : 光栅化对几何细节的处理有限，容易出现锯齿等问题。
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      1.3 代码实现
     </strong>
    </p>
    <p>
     Unity3D 默认使用光栅化进行渲染。以下是一个简单的
     <a href="https://zhida.zhihu.com/search?content_id=254691210&amp;content_type=Article&amp;match_order=1&amp;q=Shader&amp;zhida_source=entity" rel="nofollow" title="Shader">
      Shader
     </a>
     示例，展示了如何使用光栅化进行漫反射光照计算：
    </p>
    <pre><code>Shader "Custom/DiffuseShader"
{
    Properties
    {
        _MainTex ("Texture", 2D) = "white" {}
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 200

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag

            #include "UnityCG.cginc"

            struct appdata
            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
                float3 normal : NORMAL;
            };

            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
                float3 worldNormal : NORMAL;
            };

            sampler2D _MainTex;

            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = v.uv;
                o.worldNormal = UnityObjectToWorldNormal(v.normal);
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                // 漫反射光照计算
                float3 lightDir = normalize(_WorldSpaceLightPos0.xyz);
                float3 normal = normalize(i.worldNormal);
                float diff = max(dot(normal, lightDir), 0.0);
                fixed4 col = tex2D(_MainTex, i.uv);
                col.rgb *= diff;
                return col;
            }
            ENDCG
        }
    }
}</code></pre>
    <h4>
     二、光线追踪 (Ray Tracing)
    </h4>
    <p>
     <strong>
      2.1 原理
     </strong>
    </p>
    <p>
     光线追踪是一种模拟光线传播路径的渲染技术。它通过追踪从摄像机出发的光线，计算光线与场景中物体的交点，并根据交点处的材质属性、光照信息等计算像素颜色。具体步骤如下：
    </p>
    <ol>
     <li>
      <strong>
       光线生成
      </strong>
      : 从摄像机出发，为每个像素生成一条光线。
     </li>
     <li>
      <strong>
       光线求交
      </strong>
      : 计算光线与场景中所有物体的交点，找到最近的交点。
     </li>
     <li>
      <strong>
       着色计算
      </strong>
      : 根据交点处的材质属性、光照信息等计算像素颜色。
     </li>
     <li>
      <strong>
       递归追踪
      </strong>
      : 根据需要，递归地追踪反射、折射等光线，直到达到最大递归深度或光线离开场景。
     </li>
    </ol>
    <p>
     <strong>
      2.2 优缺点
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       优点
      </strong>
      :
      <ul>
       <li>
        <strong>
         渲染质量高
        </strong>
        : 光线追踪能够模拟真实的光线传播，实现全局光照、阴影、反射、折射等复杂效果，渲染质量极高。
       </li>
       <li>
        <strong>
         物理准确
        </strong>
        : 光线追踪基于物理光学原理，能够更真实地模拟现实世界的光照效果。
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <ul>
     <li>
      <strong>
       缺点
      </strong>
      :
      <ul>
       <li>
        <strong>
         计算量大
        </strong>
        : 光线追踪需要计算大量光线与物体的交点，计算量巨大，难以满足实时渲染的需求。
       </li>
       <li>
        <strong>
         硬件要求高
        </strong>
        : 光线追踪需要强大的硬件支持，目前只有高端显卡才能较好地支持光线追踪。
       </li>
      </ul>
     </li>
    </ul>
    <p>
    </p>
    <p>
     <strong>
      2.3 代码实现
     </strong>
    </p>
    <p>
     Unity3D 从 2019.3 版本开始支持光线追踪。以下是一个简单的 Shader 示例，展示了如何使用光线追踪进行镜面反射计算：
    </p>
    <pre><code>Shader "Custom/RayTracingShader"
{
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 200

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma raytracing

            #include "UnityCG.cginc"
            #include "UnityRaytracingMeshUtils.cginc"

            struct appdata
            {
                float4 vertex : POSITION;
            };

            struct v2f
            {
                float4 vertex : SV_POSITION;
            };

            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                return o;
            }

            float4 frag (v2f i) : SV_Target
            {
                // 获取屏幕坐标
                float2 screenPos = i.vertex.xy / _ScreenParams.xy;

                // 生成光线
                RayDesc ray;
                ray.Origin = _WorldSpaceCameraPos;
                ray.Direction = normalize(float3(screenPos * 2.0 - 1.0, 1.0));

                // 光线追踪
                RayIntersection rayIntersection;
                rayIntersection.t = 1.#INF;
                rayIntersection.primitiveID = -1;
                rayIntersection.instanceID = -1;
                rayIntersection.barycentricCoordinates = float2(0.0, 0.0);

                TraceRay(_<a href="https://zhida.zhihu.com/search?content_id=254691210&amp;content_type=Article&amp;match_order=1&amp;q=RaytracingAccelerationStructure&amp;zhida_source=entity" rel="nofollow" title="RaytracingAccelerationStructure">RaytracingAccelerationStructure</a>, RAY_FLAG_NONE, 0xFF, 0, 0, 0, ray, rayIntersection);

                // 镜面反射
                if (rayIntersection.primitiveID != -1)
                {
                    float3 normal = UnityRayTracingGetTriangleNormal(rayIntersection);
                    float3 reflectedDir = reflect(ray.Direction, normal);
                    ray.Direction = reflectedDir;
                    ray.Origin = rayIntersection.worldPosition + normal * 1e-4;

                    TraceRay(_RaytracingAccelerationStructure, RAY_FLAG_NONE, 0xFF, 0, 0, 0, ray, rayIntersection);

                    if (rayIntersection.primitiveID != -1)
                    {
                        return float4(0.5, 0.5, 0.5, 1.0);
                    }
                }

                return float4(0.0, 0.0, 0.0, 1.0);
            }
            ENDCG
        }
    }
}</code></pre>
    <h4>
     三、总结
    </h4>
    <p>
     光栅化和光线追踪是 Unity3D 中两种重要的渲染技术。光栅化速度快、效率高，适合实时渲染；光线追踪渲染质量高、物理准确，适合离线渲染和对画质要求极高的场景。随着硬件性能的不断提升，光线追踪在实时渲染中的应用也越来越广泛。开发者可以根据具体需求选择合适的渲染技术，并结合 Unity3D 提供的强大工具，创造出令人惊叹的视觉效果。
    </p>
