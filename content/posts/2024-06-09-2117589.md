    <h2>
     <span style="font-family: 宋体;">
      软件工程
     </span>
     <span lang="EN-US">
      (
     </span>
     <span style="font-family: 宋体;">
      第一章
     </span>
     <span lang="EN-US">
      )
     </span>
    </h2>
    <h3>
     <span style="font-family: 黑体;">
      软件
     </span>
    </h3>
    <p class="MsoNormal">
     <span lang="EN-US">
      1983
     </span>
     <span style="font-family: 宋体;">
      年
     </span>
     <span lang="EN-US">
      IEEE
     </span>
     <span style="font-family: 宋体;">
      这么定义：软件是计算机程序、方法、规则、相关的文档资料和在计算机上运行程序所必需的数据。
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      软件工程
     </span>
    </h3>
    <p class="MsoNormal">
     <span lang="EN-US">
      1983
     </span>
     <span style="font-family: 宋体;">
      年
     </span>
     <span lang="EN-US">
      IEEE
     </span>
     <span style="font-family: 宋体;">
      这么定义：软件工程是软件开发、运行、维护和修复软件的系统方法。
     </span>
    </p>
    <p class="MsoNormal">
     <span lang="EN-US">
      1993
     </span>
     <span style="font-family: 宋体;">
      年
     </span>
     <span lang="EN-US">
      IEEE
     </span>
     <span style="font-family: 宋体;">
      这么定义：软件工程是：
     </span>
     <span lang="EN-US">
      1
     </span>
     <span style="font-family: 宋体;">
      ）把系统化的、规范的、可度量的途径应用于软件开发、运行和维护的过程，也就是把工程化应用于软件中；
     </span>
     <span lang="EN-US">
      2
     </span>
     <span style="font-family: 宋体;">
      ）研究
     </span>
     <span lang="EN-US">
      1
     </span>
     <span style="font-family: 宋体;">
      ）中提到的途径。
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      软件危机
     </span>
    </h3>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      指在计算机软件的开发和维护过程中所遇到的一系列严重问题。
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      产生原因：一是和软件本身特点有关，另一方面也和软件开发与维护的方法不正确有关。
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      解决途径：从技术（方法和工具）上和管理上努力。
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      软件工程七大原理
     </span>
    </h3>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       1）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      用分阶段的生命周期计划严格管理
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       2）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      坚持进行阶段评审
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       3）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      实行严格的产品控制
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       4）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      采用现代程序设计技术
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       5）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      结果应能清楚地审查
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       6）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      开发小组的人员应该少而精
     </span>
    </p>
    <p class="MsoNormal" style="margin-left: 18pt; text-indent: -18pt;">
     <span lang="EN-US" style="">
      <span style="">
       7）
       <span style='font-family: "Times New Roman"; font-style: normal; font-variant: normal; font-weight: normal; font-size: 7pt; line-height: normal; font-size-adjust: none; font-stretch: normal;'>
       </span>
      </span>
     </span>
     <span style="font-family: 宋体;">
      承认不断改进软件实践的必要性
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      软件工程三要素
     </span>
    </h3>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      方法、工具、过程
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      使用最广泛的软件工程方法学
     </span>
    </h3>
    <h4>
     <span style="font-family: 宋体;">
      传统方法学
     </span>
    </h4>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      又称生命周期方法学或结构化范型（范型与方法学在软件工程同义）。
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      特征：要等到前一阶段完成，后一阶段才开始。
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      缺点：当软件的需求随着时间的变化时，不适合用此范型，往往不成功。这种方法数据和方法想脱离。
     </span>
    </p>
    <h4>
     <span style="font-family: 宋体;">
      面向对象方法学
     </span>
    </h4>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      又称面向对象范型
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      特征：面向对象方法
     </span>
     <span lang="EN-US">
      =
     </span>
     <span style="font-family: 宋体;">
      对象
     </span>
     <span lang="EN-US">
      +
     </span>
     <span style="font-family: 宋体;">
      类
     </span>
     <span lang="EN-US">
      +
     </span>
     <span style="font-family: 宋体;">
      继承
     </span>
     <span lang="EN-US">
      +
     </span>
     <span style="font-family: 宋体;">
      用消息通信
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      扩展：如果只有对象和消息，只能说是基于对象的方法（如
     </span>
     <span lang="EN-US">
      javascript
     </span>
     <span style="font-family: 宋体;">
      ），如果只有对象、类、消息只能说是基于类的方法，当满足公式时才能说是面向对象的方法。
     </span>
    </p>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      优点：将数据和行为结合在了一起。
     </span>
    </p>
    <h3>
     <span style="font-family: 黑体;">
      扩展
     </span>
    </h3>
    <p class="MsoNormal">
     <span style="font-family: 宋体;">
      软件开发设计过程从抽象层次上分析，往往是从行为域开始，以低层的结构域（结构化方法）结束。
     </span>
    </p>
