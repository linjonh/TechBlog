     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      二、
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       视频缓存池
      </strong>
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      ：向媒体业务提供大块内存管理功能。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      三、
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       视频编码模块VENC
      </strong>
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      支持以下协议编码：（1）H.264 （2）JPEG （3）MJPEG （4）MPEG-4
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      四、
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       编码模块的输入源包括三类：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （1）
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98); background-color:rgb(255,165,0)">
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(255,0,0); background-color:rgb(255,165,0)">
      <strong>
       用户态读取图像文件向编码模块发送数据
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （2）     视频输入（VIU）模块采集的图像经过视频处理子系统（VPSS）发送到编码模块
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （3）     视频输入（VIU）模块采集的图像直接发送到编码模块
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       五、视频前处理单元VPSS（Video Process Sub-System）
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （1）     支持对一幅输入图像进行统一处理，然后进行缩放输出。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （2）     一进五出（0~4）
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （3）
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(255,0,0); background-color:rgb(255,165,0)">
      <strong>
       一幅图像最多输出4种不同分辨率的预处理后图像。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      0:主码流    1：辅码流   2：JPEG   3：预览
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       六、  典型的编码流程包括：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （1）   输入图像的接收
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （2）   图像内容的遮挡和覆盖
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （3）   图像的编码
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （4）   码流的输入
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       七、编码通道：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
     </span>
     <span style="font-size:13px; font-family:Verdana; background-color:rgb(0,176,80)">
     </span>
     <span style="font-family:Verdana; background-color:rgb(0,176,80)">
      <strong>
       完成图像转化为码流的功能，具体由码率控制器和编码器协同完成。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （1）
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       码率控制器
      </strong>
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      ：提供了对编码参数的控制和调整，从而对输出码率进行控制。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      （2）
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       编码器
      </strong>
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      ：只完成编码功能。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       八、编码通道组：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      指芯片启动一次，能够同时处理的编码通道的集合。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
     </span>
     <span style="font-size:13px; font-family:Verdana">
      <strong>
       Hi3531/Hi3532编码通道组支持启动一次，仅编码一路主码流通道。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      九、
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      <strong>
       系统绑定
      </strong>
     </span>
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      ：
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      通过数据接收者绑定数据源来建立两者之间的关联关系。绑定后，数据源生成的数据将自动发给接收者。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
      十、一些名词简写：
     </span>
    </div>

      十一、Hi3531解码器提供

     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">

     </span>
     <span style="font-family:Verdana">
      <strong>

     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98); background-color:rgb(31,73,125)">

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:18px; color:rgb(255,0,0)">
      <strong>
       一、基本概念
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       1）ES
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       ES--Elementary  Streams  (
      </strong>
     </span>
     <span style="color:rgb(99,36,35); background-color:rgb(0,176,80)">
      <strong>
       原始流
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       )
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      是直接从编码器出来的数据流，可以是编码过的视频数据流（H.264,MJPEG等），音频数据流（AAC），或其他编码数据流的统称。
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      ES流经过PES打包器之后，被转换成PES包
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
      ES是
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      只包含一种内容的数据流
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ，如只含视频或只含音频等，打包之后的PES也是只含一种性质的ES,如只含视频ES的PES,只含音频ES的PES等。
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      每个ES都由若干个
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       存取单元（AU）
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      组成
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ，每个视频AU或音频AU都是由
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      <strong>
       头部和编码数据
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      两部分组成，
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      1个AU相当于编码的1幅视频图像或1个音频帧
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ，也可以说，每个AU实际上是编码数据流的显示单元，即相当于解码的1幅视频图像或1个音频帧的取样。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       2）PES
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       PES--Packetized  Elementary Streams  (分组的ES)，
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ES形成的分组称为PES分组，是用来传递ES的一种
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      数据结构
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      。PES流是ES流经过PES打包器处理后形成的数据流，在这个过程中完成了将
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      ES流分组、打包、加入包头信息
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      等操作（对ES流的第一次打包）。PES流的基本单位是PES包。PES包由
     </span>
     <span style="font-size:12px; color:rgb(0,128,0)">
      <strong>
       包头和payload组成
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(0,0,255)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       3）PTS、DTS
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       PTS--PresentationTime Stamp（显示时间标记）
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      表示显示单元出现在系统目标解码器（H.264、MJPEG等）的时间。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       DTS--Decoding Time Stamp（解码时间标记）
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      表示将存取单元全部字节从解码缓存器移走的时间。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       PTS/DTS是打在PES包的包头里面的，这两个参数是解决音视频同步显示
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       ，防
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      止解码器输入缓存上溢或下溢的关键。每一个I（关键帧）、P（预测帧）、B（双向预测 帧）帧的包头都有一个PTS和DTS，但PTS与DTS对于B帧不一样，无需标出B帧的DTS，对于I帧和P帧，显示前一定要存储于视频解码器的
     </span>
     <span style="font-size:12px; color:rgb(99,36,35)">
      <strong>
       重新排序缓存器
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      中，经过延迟（重新排序）后再显示，所以一定要分别标明PTS和DTS。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       4）PS
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       PS--Program Stream(节目流)
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      PS流由PS包组成，而一个PS包又由若干个PES包组成（
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      到这里，ES经过了两层的封装
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ）。PS包的包头中包含了同步信息与时钟恢复信息。一个PS包最多可包含具有同一时钟基准的16个视频PES包和32个音频PES包。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       5）TS
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       TS--Transport Stream
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       （传输流）
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      由定长的TS包组成（188字节）
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      ，而TS包是对PES包的一个重新封装（到这里，ES也经过了两层的封装）。PES包的包头信息依然存在于TS包中。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-weight:bold">
      TS流与PS流的区别
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      在于
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      TS流的包结构是
     </span>
     <span style="font-size:12px; color:rgb(192,0,0); background-color:rgb(0,176,80)">
      <strong>
       固定长度
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      的,而PS流的包结构是
     </span>
     <span style="font-size:12px; color:rgb(192,0,0); background-color:rgb(0,176,80)">
      <strong>
       可变长度的
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      。PS包由于长度是变化的,一旦丢失某一PS包的同步信息,接收机就会进入失步状态,从而导致严重的信息丢失事件。而TS码流由于采用了固定长度的包结构,当传输误码破坏了某一TS包的同步信息时,接收机可在固定的位置检测它后面包中的同步信息,从而恢复同步,避免了信息丢失。因此在信道环境较为恶劣、传输误码较高时一般采用TS码流,而在信环境较好、传输误码较低时一般采用PS码流。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(0,0,255)">
      <strong>
       6）TS单一码流、混合码流
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       单一性：
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      TS流的基本组成单位是长度为188字节的TS包。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      <strong>
       混合性：
      </strong>
     </span>
     <span style="font-size:12px; color:rgb(51,51,51)">
      TS流由多种数据组合而成，一个TS包中的数据可以是视频数据，音频数据，填充数据，PSI/SI表格数据等（唯一的PID对应）。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:18px; color:rgb(255,0,0)">
      <strong>
       二、基本流程
      </strong>
     </span>
    </div>
    <div style="float:none">
     <img alt="" src="" style="width:620px">
     </img>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <br/>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:18px; color:rgb(51,51,51)">
      1）
     </span>
     <span style="font-size:18px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       A/D转换后，通过MPEG-2压缩编码得到的ES基本流
      </strong>
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      。这个数据流很大，并且只是I，P，B的这些视频帧或音频取样信息。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:18px; color:rgb(51,51,51)">
      2）通过
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      <strong>
       PES
      </strong>
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      打包器，打包并在每个帧中插入 PTS/DTS标志，变成PES。原来是流的格式，现在成了数据包的分割形式。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:18px; color:rgb(51,51,51)">
      3）PES根据需要打包成
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      <strong>
       PS或TS包
      </strong>
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      进行
     </span>
     <span style="font-size:18px; color:rgb(0,128,0)">
      存储（DVD）或传输(DVB)
     </span>
     <span style="font-size:18px; color:rgb(51,51,51)">
      。因每路音/视频只包含一路的编码数据流，所以每路PES也只包含相应的数据流。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <br/>
    </div>

     </span>

     <span style="font-size:13px; font-family:Verdana; color:rgb(123,125,98)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       8.NALU
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      <strong>
       NALU(Network Abstract Layer Unit)：
      </strong>
     </span>
     <span style="color:rgb(192,0,0)">
      <strong>
       H264标准中的比特流是以NAL为单位,
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      每个NAL单元包含一个RBSP（rawbyte sequence payload，
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       原始字节序列载荷
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      ），NALU的头信息定义了RBSP所属类型。类型一般包括序列参数集（SPS）、图像参数集（PPS）、增强信息（SEI）、条带（Slice）等，其中，SPS和PPS属于参数集，两标准采用参数集机制是为了将一些主要的序列、图像参数（解码图像尺寸、片组数、参考帧数、量化和滤波参数标记等）与其他参数分离，通过
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      解码器先解码出来
     </span>
     <span style="color:rgb(51,51,51)">
      。此外，为了增强图像的清晰度，AVS-M添加了图像头（Picturehead）信息。读取NALU流程中，
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      每个NALU前有一个起始码0x000001
     </span>
     <span style="color:rgb(51,51,51)">
      ，为防止内部0x000001序列竞争，H.264编码器在
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       最后一字节前插入一个新的字节——0x03
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      ，所以解码器检测到该序列时，需将0x03删掉，而AVS-M只需识别出起始码0x000001。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       9.主码流与次码流
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
     </span>
     <span style="color:rgb(51,51,51)">
      hi3515中，
     </span>
     <span style="color:rgb(0,0,255)">
      大小码流使用相同的源图像，大码流直接由源图像编码得到，小码流由源图像缩小后编码得到 ,
     </span>
     <span style="color:rgb(51,51,51)">
      小码流编码图像支持的最大尺寸为 CIF。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
     <span style="color:rgb(0,0,255)">
      主码流用于本地存储，子码流适用于图像在低带宽网络上传输。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <br/>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,255)">
      <strong>
       10.ES流、PES流、PS流以及TS流
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       ES流
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      （Elementary Stream）由三部分组成：
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      1）经MPEG-2视频编码器编码后的图像数据流；
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      2）经MPEG-2音频编码器编码后的声音数据流；
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      3）其他编码数据流；
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       PES流
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      （Packetized ElementaryStream）：PES流是ES流经过
     </span>
     <span style="color:rgb(0,0,255)">
      PES打包器
     </span>
     <span style="color:rgb(51,51,51)">
      处理后形成的数据流，在这个过程中完成了将ES流
     </span>
     <span style="color:rgb(0,0,255)">
      分组、打包、加入包头信息
     </span>
     <span style="color:rgb(51,51,51)">
      等操作（对ES流的第一次打包）。PES流的基本单位是PES包。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       节目流（Program Stream简称PS）与传输流（Transport Stream简称TS）
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      是数字视频系统中的两类数据码流，由MPEG-2系统层定义，
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       H.264采用与MPEG-2相同的系统层
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      ，因而也采用PS/TS两类码流，TS流与PS流的
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       区别
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      在于TS流的包结构是
     </span>
     <span style="color:rgb(0,0,255)">
      固定长度
     </span>
     <span style="color:rgb(51,51,51)">
      的,而PS流的包结构是
     </span>
     <span style="color:rgb(0,0,255)">
      可变长度
     </span>
     <span style="color:rgb(51,51,51)">
      的。PS包由于长度是变化的,一旦丢失某一PS包的同步信息,接收机就会进入失步状态,从而导致严重的信息丢失事件。而TS码流由于采用了固定长度的包结构,当传输误码破坏了某一TS包的同步信息时,接收机可在固定的位置检测它后面包中的同步信息,从而恢复同步,避免了信息丢失。因此在信道环境较为恶劣、传输误码较高时一般采用TS码流,而在信环境较好、传输误码较低时一般采用PS码流。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(34,34,34)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       11.视频输入接口模式
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      Hi3520/Hi3515 芯片支持
     </span>
     <span style="color:rgb(0,0,255)">
      多种视频接口协议
     </span>
     <span style="color:rgb(51,51,51)">
      ，且提供灵活多样的配置与各种外围codec 对接。支持的
     </span>
     <span style="color:rgb(0,0,255)">
      接口时序
     </span>
     <span style="color:rgb(51,51,51)">
      包括
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       ITU-R BT.656、ITU-R BT.601、ITU-R BT.1120、
      </strong>
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       digital camera
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      等。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      4 个视频输入设备
     </span>
     <span style="color:rgb(0,0,255)">
      都能支持
     </span>
     <span style="color:rgb(51,51,51)">
      ITU-R BT.656接口模式，
     </span>
     <span style="color:rgb(0,0,255)">
      但只有
     </span>
     <span style="color:rgb(51,51,51)">
      设备 0 和设备2才支持其他几种接口模式。
     </span>
     <span style="color:rgb(0,0,255)">
      支持BT.656接口模式时，又能支持几种不同的多路复用工作模式，例如 54M2D1、54M 4CIF以及 108M 4D1。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       12.视频输入通道
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      视频输入通道与物理上的
     </span>
     <span style="color:rgb(0,0,255)">
      <strong>
       实际视频采集通道
      </strong>
     </span>
     <span style="color:rgb(0,0,255)">
      一一对应
     </span>
     <span style="color:rgb(51,51,51)">
      ，视频输入通道归属于视频输入设备，
     </span>
     <span style="color:rgb(255,0,0)">
      视频输入设备的接口模式决定了此设备下能支持的通道个数
     </span>
     <span style="color:rgb(51,51,51)">
      ，一个设备
     </span>
     <span style="color:rgb(255,0,0)">
      最大支持
     </span>
     <span style="color:rgb(51,51,51)">
      4 个通道，依次用 ViChn0、ViChn1、ViChn2、ViChn3 标示；
     </span>
     <span style="color:rgb(0,0,255)">
      除了
     </span>
     <span style="color:rgb(51,51,51)">
      有多路复用功能的 BT.656接口模式能支持多个通道
     </span>
     <span style="color:rgb(0,0,255)">
      外
     </span>
     <span style="color:rgb(51,51,51)">
      ，其他接口模式时每个设备下
     </span>
     <span style="color:rgb(0,0,255)">
      只能支持
     </span>
     <span style="color:rgb(51,51,51)">
      1个通道。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       13.分辨率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      分辨率主要有以下 3 种概念：
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      −
     </span>
     <span style="color:rgb(0,0,255)">
      设备分辨率
     </span>
     <span style="color:rgb(51,51,51)">
      指该设备的输出有效像素点数，由设备时序决定。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      −
     </span>
     <span style="color:rgb(0,0,255)">
      显示分辨率
     </span>
     <span style="color:rgb(51,51,51)">
      指画面在显示设备上的有效显示区域。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      −
     </span>
     <span style="color:rgb(0,0,255)">
      图像分辨率
     </span>
     <span style="color:rgb(51,51,51)">
      指图像本身的有效像素点数。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      指图像中存储的信息量，是每
     </span>
     <a href="http://baike.baidu.com/view/261514.htm" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-size:12px; font-family:Arial; color:rgb(19,110,194)">
       英寸
      </span>
     </a>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      图像内有多少个像素点，分辨率的单位为PPI(Pixels Per Inch)，通常叫做：像素每英寸。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255); background-color:rgb(0,176,80)">
      <strong>
       14.通道优先级
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      当同一输出设备上有多个通道
     </span>
     <span style="color:rgb(0,0,255); background-color:rgb(0,176,80)">
      同时输出
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      显示时，按照优先级顺序对输出图像进行叠加。当各个通道的画面有重叠区域时，优先级高的图像显示在上层。如果各个通道
     </span>
     <span style="color:rgb(0,0,255); background-color:rgb(0,176,80)">
      优先级一致
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      ，则通道号越大的默认优先级越高。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       15.通道组
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
     <span style="color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       通道组是指芯片能够同时处理的编码通道的集合
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      ，
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       相当于一个容器
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      。一个通道组
     </span>
     <span style="color:rgb(51,51,51)">
      <strong>
       最多可同时包含
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      1路主码流（H.264/MJPEG）、1路次码流（H.264/MJPEG） ，或者仅包含1路 JPEG抓拍（即 JPEG抓拍时，不允许包含任何其他通道） ，或者 1 路MPEG4 编码通道。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       16. Overlay Region
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      视频叠加区域，针对码流里打的叠加区域，即通常所说的
     </span>
     <span style="color:rgb(255,0,0)">
      <strong>
       码流OSD
      </strong>
     </span>
     <span style="color:rgb(51,51,51)">
      ，即往视频上打字/Logo。同时，还有视频遮挡区域和软件视频叠加区域的选择。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       17.宏块
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      将图像划分为
     </span>
     <span style="color:rgb(0,0,255)">
      16*16(以像素为单位)大小的块
     </span>
     <span style="color:rgb(51,51,51)">
      ，每一块称为一个宏块。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <br/>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(0,0,255)">
      <strong>
       18.移动侦测（Motion Detection）
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      又称运动检测，是检测
     </span>
     <span style="color:rgb(0,0,255)">
      正在视频编码
     </span>
     <span style="color:rgb(51,51,51)">
      的图像是否发生
     </span>
     <span style="color:rgb(0,0,255)">
      亮度变化以及相应的运动向量
     </span>
     <span style="color:rgb(51,51,51)">
      。移动侦测功能以
     </span>
     <span style="color:rgb(0,0,255)">
      宏块
     </span>
     <span style="color:rgb(51,51,51)">
      为最小单位，计算指定图像的宏块在指定图像
     </span>
     <span style="color:rgb(0,0,255)">
      间隔内
     </span>
     <span style="color:rgb(51,51,51)">
      的亮度变化和运动向量。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       码流 /
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       码率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       码流(Data Rate)
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      是指视频文件
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       在单位时间内使用的数据流量
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，也叫码率或码流率，
     </span>
     <span style="font-family:Arial; color:rgb(192,0,0); background-color:rgb(0,176,80)">
      <strong>
       就是取样率,
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      是视频编码中画面质量控制中最重要的部分，一般的单位是
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       kb/s
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      或者
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      <strong>
       Mb/s
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      。一般同样分辨率下，
     </span>
     <span style="font-family:Arial; color:rgb(112,48,160)">
      <strong>
       视频文件的码流越大，压缩比就越小，画面质量就越高
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      。码流越大，说明单位时间内取样率越大，数据流精度就越高，处理出来的文件就越接近原始文件，图像质量越好，画质越清晰，
     </span>
     <span style="font-family:Arial; color:rgb(49,133,155)">
      <strong>
       要求播放设备的解码能力也越高
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,32,96)">
      <strong>
       计算文件体积公式:   文件体积=时间X码率/8
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       常见的一部90分钟1Mbps码流的720P RMVB文件，其体积就=5400秒×1Mb/8=675MB。
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      通常一个视频文件包括了画面及声音,并且同一个视频文件音频和视频的
     </span>
     <span style="font-family:Arial; color:rgb(49,133,155)">
      <strong>
       采样方式和比特率并不是一样
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      的。而我们所说的一个视频文件码流率大小，一般是
     </span>
     <span style="font-family:Arial; color:rgb(49,133,155)">
      <strong>
       指视频文件中音频及视频信息码流率的总和
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,32,96)">
      <strong>
       RMVB视频文件为：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      RMVB中的VB，指的是VBR，即Variable Bit Rate的缩写，中文含义是可变比特率，它表示RMVB采用的是动态编码的方式，把
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       较高的采样率用于复杂的动态画面
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      (歌舞、飞车、战争、动作等)，而
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       把较低的采样率用于静态画面，合理利用资源
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，达到画质与体积可兼得的效果。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      在介绍:
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       位率又称为“码率”。指单位时间内，单个录像通道所产生的数据量,可以根据录像的时间与位率估算出一定时间内的录像文件大小，是一个可调参数。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       影响因素:
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,204,51)">
      <strong>
       分辨率、
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,153,0)">
      <strong>
       场景(画面的抖动情况)、存储空间 。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       采样率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      采样率是指将
     </span>
     <span style="font-family:Arial; color:rgb(247,150,70)">
      <strong>
       模拟信号转换成数字信号
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      时的采样频率，也就是单位时间内采样多少点。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      电影的采样率是24赫兹，PAL制式的采样率是25赫兹，NTSC制式的采样率是30赫兹。当我们把采样到的一个个静止画面再以采样率同样的速度回放时，看到的就是连续的画面，采样率越高越连贯，再高就区别不了。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:16px; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       采样位数：
      </strong>
     </span>
     <span style="font-size:16px; color:rgb(51,51,51)">
      可以理解为
     </span>
     <a href="https://www.baidu.com/s?wd=%E9%87%87%E9%9B%86%E5%8D%A1&amp;tn=44039180_cpr&amp;fenlei=mv6quAkxTZn0IZRqIHckPjm4nH00T1YkmHfkmHckmWcLuWR1uHRk0ZwV5Hcvrjm3rH6sPfKWUMw85HfYnjn4nH6sgvPsT6KdThsqpZwYTjCEQLGCpyw9Uz4Bmy-bIi4WUvYETgN-TLwGUv3EPHTYrj0vPjcs" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-size:16px; color:rgb(63,136,191)">
       采集卡
      </span>
     </a>
     <span style="font-size:16px; color:rgb(51,51,51)">
      处理声音的解析度。这个数值越大，解析度就越高，录制和回放的声音就越真实;
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <a href="https://www.baidu.com/s?wd=%E9%87%87%E6%A0%B7%E9%A2%91%E7%8E%87&amp;tn=44039180_cpr&amp;fenlei=mv6quAkxTZn0IZRqIHckPjm4nH00T1YkmHfkmHckmWcLuWR1uHRk0ZwV5Hcvrjm3rH6sPfKWUMw85HfYnjn4nH6sgvPsT6KdThsqpZwYTjCEQLGCpyw9Uz4Bmy-bIi4WUvYETgN-TLwGUv3EPHTYrj0vPjcs" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-size:16px; color:rgb(63,136,191)">
       采样频率
      </span>
     </a>
     <span style="font-size:16px; color:rgb(51,51,51)">
      是指录音设备在一秒钟内对声音信号的采样次数，
     </span>
     <a href="https://www.baidu.com/s?wd=%E9%87%87%E6%A0%B7%E9%A2%91%E7%8E%87&amp;tn=44039180_cpr&amp;fenlei=mv6quAkxTZn0IZRqIHckPjm4nH00T1YkmHfkmHckmWcLuWR1uHRk0ZwV5Hcvrjm3rH6sPfKWUMw85HfYnjn4nH6sgvPsT6KdThsqpZwYTjCEQLGCpyw9Uz4Bmy-bIi4WUvYETgN-TLwGUv3EPHTYrj0vPjcs" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-size:16px; color:rgb(63,136,191)">
       采样频率
      </span>
     </a>
     <span style="font-size:16px; color:rgb(51,51,51)">
      越高声音的还原就越真实越自然。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       比特率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       一个
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(247,150,70)">
      <strong>
       采样点数据有
      </strong>
     </span>
     <span style="font-family:Arial">
      <strong>
       多少
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(247,150,70)">
      <strong>
       个比特
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      。比特率是指每秒传送的比特(bit)数。单位为 bps(Bit Per Second)，比特率越高，传送的数据越大，音质越好
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      .比特率 =
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      采样率 x 采用位数 x声道数.
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      以电话为例，每秒3000
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      次取样，每个取样是7比特，那么电话的比特率是21000。 而
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      CD是每秒 44100
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      次取样，两个声道，每个取样是13位PCM编码，所以CD的比特率是44100*2*13=1146600，也就是说CD每秒的数据量大约是 144KB，而一张CD的容量是74分等于4440秒，就是639360KB＝640MB。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       在视频领域,比特率常翻译为码率，
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      比特率
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77); background-color:rgb(149,179,215)">
      <strong>
       表示经过编码（压缩）后的音、视频数据每秒钟需要用多少个比特来表示
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，而比特就是二进制里面最小的单位，要么是0，要么是1。比特率越高，音、视频的质量就越好，文件越大；比特率越少则情况刚好相反。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      常见编码模式：
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      VBR（Variable Bitrate）
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       动态比特率
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ,没有固定的比特率，压缩软件在压缩时根据音频数据即时确定使用什么比特率，这是以质量为前提兼顾文件大小的方式.
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ABR（Average Bitrate）
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       平均比特率
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      是VBR的一种插值参数。LAME针对CBR不佳的文件体积比和VBR生成文件大小不定的特点独创了这种编码模式。ABR在指定的文件大小内，以每50帧（30帧约1秒）为一段，低频和不敏感频率使用相对低的流量，高频和大动态表现时使用高流量，可以做为VBR和CBR的一种折衷选择。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      CBR（Constant Bitrate）
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      ，常数比特率 指文都是一种位速率。相对于VBR和ABR来讲，它压缩出来的文件体积很大，而且音质相对于VBR和ABR不会有明显的件从头到尾提高。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       帧速率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      帧速率也称为FPS(Frames PerSecond)的缩写——帧/秒。指
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       每秒钟刷新的图片的帧数
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，可以理解为图形处理器每秒钟能够刷新几次。越高的帧速率可以得到更流畅、更逼真的动画。每秒钟帧数(FPS)越多，所显示的动作就会越流畅。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       分辨率
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      就是帧大小(
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       一幅图像的大小
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ),每一帧就是一副图像。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       一个音频编码率为128Kbps，视频编码率为800Kbps的文件，其总编码率为928Kbps，
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      意思是经过编码后的数据每秒钟需要用928K比特来表示。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      <strong>
       高清视频:
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       720和1080
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       代表视频流的分辨率，前者1280*720，后者1920*1080，不同的编码需要不同的系统资源，大概可以认为是H.264&gt;VC-1&gt;MPEG2。
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       总的来说，从压缩比上来看，H.264
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       的压缩比率更高一些，也就是同样的视频，通过H.264编码
      </strong>
     </span>
     <a href="http://lib.csdn.net/base/datastructure" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-family:Arial; color:rgb(223,52,52)">
       <strong>
        算法
       </strong>
      </span>
     </a>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       压出来的视频容量要比VC-1的更小，
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      但是VC-1 格式的视频在解码计算方面则更小一些。目前的高清视频编码格式主要有H.264、VC-1、MPEG-2、MPEG-4、DivX、XviD、WMA-HD以及X264。事实上，现在网络上流传的高清视频主要以两类文件的方式存在：一
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      类是经过MPEG-2
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      标准压缩，以tp和ts为后缀的视频流文件;一类是经过WMV-HD(Windows Media Video HighDefinition)标准压缩过的wmv文件，还有少数文件后缀为avi或mpg，其性质与wmv是一样的。
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      <strong>
       真正效果好的高清视频更多地以H.264
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       与VC-1这两种主流的编码格式流传。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       一般来说，H.264
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       格式以“.avi”、“.mkv”以及“.ts”封装比较常见。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      <strong>
       像素，
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       就是CCD/CMOS上光电感应元件的数量，
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      一个感光元件经过感光，光电信号转换，A/D转换等步骤以后，在输出的照片上就形成一个点，我们如果把影像放大数倍，会
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       发现这些连续色调其实是由许多色彩相近的小方点所组成，这些小方点就是构成影像的最小单位“像素”（Pixel）。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      <strong>
       像素格式相关
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       4:3的标准格式有：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       PAL     720x576   25.00fps 像素比1.067
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       NTSC   720x480   29.97fps  像素比0.9
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       16:9的标准格式有
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      PAL     720x576  25.00fps   像素比1.422
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      NTSC    720x480  29.97fps  像素比1.2
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      720p    1280x720 25.00fps   像素比1.0
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      1080p/i 1440x108025.00fps     像素比1.333
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       像素比:
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       表示像素的形状，即像素的宽高之比。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <br/>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,0,255)">
      很多朋友总觉得16:9的画面横向像素数要比4:3的画面更多，其实它们的像素数量是一样的，4:3与16:9并不是由像素的数量来区分的？
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      原因:
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      就拿PAL制的标清DV来说，无论4:3还是16:9，画面的像素数都是720*576。
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      其实它们的
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(0,176,80)">
      <strong>
       像素形状是不同
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      的。细心
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      的朋友可能注意过在很多软件里的分辨率设定时，除了有4:3、16:9，制式、高清还是标清以外，还有一个不可选的参数--
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(0,0,255); background-color:rgb(255,255,0)">
      -像素比
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      ，以PAL制的DV格式为例，选4:3的时候，像素比是1.0667，选16:9的时候是1.4222。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; font-family:Arial; color:rgb(255,0,0)">
      <strong>
       画面的长度 = 横向像素数*每个像素长度，
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(192,80,77)">
      <strong>
       宽度
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(192,80,77)">
      <strong>
       =
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(192,80,77)">
      <strong>
       纵向像素数
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(255,0,0)">
      <strong>
       *
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(0,176,80)">
      <strong>
       每个像素宽度
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(255,0,0)">
      <strong>
       。
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      我们假设每个像素宽度为1，那么，4:3的画面中，每个像素的长度就是1.0667，则整个画面的长宽比就是(720*1.0667)/(576*1)=4/3。同样，在16:9的画面中，每个像素长度就是1.4222，则整个画面的长宽比就是(720*1.4222)/(576*1)=16/9。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,176,80)">
      <strong>
       显示设备分辨率:
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px; font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      <strong>
       分辨率
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(255,0,0)">
      <strong>
       ,就是屏幕图像的精密度，是指显示器所能显示的像素的多少。
      </strong>
     </span>
     <span style="font-size:12px; font-family:Arial; color:rgb(51,51,51)">
      由于屏幕上的点、线和面都是由像素组成的，显示器可显示的像素越多，画面就越精细，同样的屏幕区域内能显示的信息也越多，所以分辨率是个非常重要的性能指标之一。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      <strong>
       6,什么是视频信号?
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       数字视频信号、模拟视频信号？
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       视频信号可分为
      </strong>
     </span>
     <span style="font-family:Arial; background-color:rgb(75,172,198)">
      <strong>
       模拟视频信号
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       和
      </strong>
     </span>
     <span style="font-family:Arial; background-color:rgb(146,205,220)">
      <strong>
       数字视频信
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       号两大类：模拟视频信号和数字视频信号。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      模拟视频是指每一帧图像是实时获取的自然景物的真实图像信号。我们在日常生活中看到的电视、电影都属于模拟视频的范畴。模拟视频信号具有
     </span>
     <span style="font-family:Arial">
      <strong>
       <em>
        成本低和还原性好
       </em>
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(128,128,0)">
      等优点，视频画面往往会给人一种身临其境的感觉。但它的最大缺点是不论被记录的图像信号有多好，经过长时间的存放之后，信号和画面的质量将大大的降低；或者
     </span>
     <span style="font-family:Arial">
      <strong>
       经过多次复制之后，画面的失真
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(128,128,0)">
      就会很明显。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       数字视频信号是基于数字技术以及其他更为拓展的图像显示标准的视频信息，数字视频与模拟视频相比有以下特点：
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       （1
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ）数字视频可以可以不失真的进行无数次复制，而模拟视频信号每转录一次，就会有一次误差积累，产生信号失真。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       （2
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ）模拟视频长时间存放后视频质量会降低，而数字视频便于长时间的存放。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       （3
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ）可以对数字视频进行非线性编辑，并可增加特技效果等。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(128,128,0)">
      <strong>
       （4
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ）数字视频数据量大，在存储与传输的过程中必须进行压缩编码。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      <strong>
       模拟视频信号每帧的图像信息是连续获取的，用连续的电信号表示；数字视频信号从摄像机开始就数字化了，光电器件转换的模拟电信号，通过一定速率的模数转换器（A/D
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ）取样、量化、编码，转换成离散的数字信号，至此，数字视频信号就可以在计算机里做进一步的处理，根据不同的数据压缩编码方案，形成不同格式的数据流
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(51,51,51)">
      -----------------------------------------------------------------------------------
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      H264、Xvid等就是视频编码格式，MP3、AAC等就是音频编码格式。将一个Xvid视频编码文件和一个MP3音频编码文件按AVI封装标准封装以后，就得到一个AVI后缀的视频文件，这个就是我们常见的AVI视频文件了。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      部分技术先进的容器还可以同时封装多个视频、音频编码文件，甚至同时封装进字幕，如MKV封装格式。
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       MKV文件可以做到一个文件包括多语种发音、多语种字幕，适合不同人的需要。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      如：MKV文件只要制作的时候同时加入国语和粤语发音的音轨和对应的简体、繁体字幕，播放的时候，你可以独立选择国语或粤语发音，并根据自己需要选择简体或繁体字幕，也可以选择不显示字幕。相当方便。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      一个完整的视频转换设置都至少包括
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      <strong>
       3个步骤:
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="color:rgb(255,0,0)">
      A：设置需要的
     </span>
     <span style="color:rgb(99,36,35)">
      视频编码
     </span>
     <span style="color:rgb(255,0,0)">
      、B：设置需要的
     </span>
     <span style="color:rgb(99,36,35)">
      音频编码
     </span>
     <span style="color:rgb(255,0,0)">
      、C：选择需要的
     </span>
     <span style="color:rgb(99,36,35)">
      容器封装
     </span>
     <span style="color:rgb(255,0,0)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      常用的编码格式有
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      Xvid，H264，MPEG1，MPEG2。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      Xvid：与RMVB格式差不多的压缩率，通用性很强，特别是用于家用DVD和便携式MP4等设备。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      H264：压缩率最高的视频压缩格式，与其他编码格式相比，同等画面质量，文件体积最小，远超RMVB 编
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      码格式
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，电脑都可以播放，部分便携式视频设备也支持，如苹果播放器。PDA/PPC等设备也可以使用
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(255,0,0)">
      MPEG1：其实就是
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      <strong>
       VCD编码格式
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      MPEG2：
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      <strong>
       DVD编码格式
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      。比MPEG1强，与MPEG1一样，已经落后的编码格式，压缩率都不高，编码后的文件体积大
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，多用于希望把网上下载的文件转换为VCD或DVD碟的时候。
     </span>
    </div>

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">

     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      事实上，很多封装容器对音频编码和视频编码的组合方式放的很开，如AVI还可以使用H264+AAC
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      <strong>
       编码组合
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ，可以在具体使用中自己体会。尤其是MKV封装容器，基本无论什么样的组合都可以！但一
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">

     </span>
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">

     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">

     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">

    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">
      首先要分清楚
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      媒体文件和编码的区别
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      ：
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      文件是既包括视频又包括音频、甚至还带有脚本的一个集合，也可以叫容器
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      ；文件当中的视频和音频的
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35); background-color:rgb(255,255,0)">
      压缩
     </span>
     <a href="http://lib.csdn.net/base/datastructure" rel="nofollow noopener noreferrer" target="_blank">
      <span style="font-family:Arial; color:rgb(0,56,132); background-color:rgb(255,255,0)">
       <strong>
        算法
       </strong>
      </span>
     </a>
     <span style="font-family:Arial; color:rgb(255,0,0); background-color:rgb(255,255,0)">
      才是具体的
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35); background-color:rgb(255,255,0)">
      <strong>
       编码
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(255,0,0)">
      。
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      *AVI :
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      音视频交互存储，最常见的音频视频容器。
     </span>
     <span style="font-family:Arial; color:rgb(99,36,35)">
      支持的视频音频编码也是最多的
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,0,255); background-color:rgb(255,255,0)">
      <strong>
       *MPG :
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       MPEG编码采用的音频视频容器，具有流的特性。里面又分为 PS
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       ，TS 等，PS 主要用于 DVD 存储，TS 主要用于 HDTV。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      *VOB
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      :DVD采用的音频视频容器格式（即视频MPEG-2，音频用AC3或者DTS），支持多视频多音轨多字幕章节等。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(0,0,255); background-color:rgb(255,255,0)">
      <strong>
       *MP4 :M
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(0,0,255)">
      <strong>
       PEG-4编码采用的音频视频容器，基于 QuickTime MOV
      </strong>
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">
      <strong>
       开发，具有许多先进特性。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51); background-color:rgb(255,255,0)">
      *3GP :
     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">

     </span>
    </div>

     </span>
     <span style="font-family:Arial; color:rgb(51,51,51)">

     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-family:Arial; color:rgb(51,51,51)">

     </span>
    </div>

     </span>

    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:16px">
      <strong>
       HDcctv是什么?
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
      由HDcctv联盟(HDcctv Alliance®)推出的HDcctv标准是通过同轴电缆传输非压缩的无损高清视频信号的一个物理电气接口标准，是世界上唯一的综合性高清监控录像的电气标准。该标准是由国际标准SMPTE-292M派生，也就是在现有“高清晰度串行数字接口”或HD-SDI的基础上，以它作为核心的传输方式，并在此基础上进行一些改动、增减，最后形成了一个适用于视频监控领域的标准。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
      HD-SDI已经应用于广电领域接近25年，其技术较为成熟，H
     </span>
     <span style="font-size:12px">
      <strong>
       Dcctv有别于HD-SDI的方面是，它增加了兼容性测试以保证其产品的互联性
      </strong>
     </span>
     <span style="font-size:12px">
      。HDcctv所有的产品都可以保证产品的兼容性，但是HD-SDI却不能，HD-SDI的应用领域小、量少，对兼容性的要求不是很高。而监控却是要求重复性大量生产，兼容性非常重要。生产厂家可以很容易的把HDcctv以及HD-SDI采纳，但是HD-SDI不一定能保证产品的互联性，HDcctv却能够保证产品的互联性。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
     </span>
     <span style="font-size:12px; background-color:rgb(255,165,0)">
      <strong>
       HDcctv标准与我们较为熟悉的H.246以及SVAC标准区别很大，一个是信道，一个是信源
      </strong>
     </span>
     <span style="font-size:12px">
      ，像H.264和SVAC是一种编码标准，在信源上对图像进行压缩再传输，而
     </span>
     <span style="font-size:12px; color:rgb(146,208,80)">
      <strong>
       <u>
        经过压缩后所传输的信号变成了数字信号，已经不再是视频信号
       </u>
      </strong>
     </span>
     <span style="font-size:12px">
      。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
     </span>
     <span style="font-size:12px; color:rgb(192,0,0); background-color:rgb(146,208,80)">
      <strong>
       HDcctv是一个传输标准，是通过同轴电缆传输无损的高清数字视频信号，传输进去的时候是视频信号，出来后依旧是视频信号。
      </strong>
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
      此外，基于HDcctv标准的HDcctv系统，更新以及安装都非常方便。把线缆保留下来，只需用更换前端的模拟摄像机、后端的存储设备以及配套的显示设备，就可以把系统由模拟变成高清的。同时它也可以避免比较复杂的管理，跟IT架构相比， HDcctv电缆系统是一个非常简单的系统。
     </span>
    </div>
    <div style="white-space:pre-wrap; line-height:1.75; font-size:14px">
     <span style="font-size:12px">
      HDcctv的标准在不断发展，HDcctv联盟会已发布2.0的版本，包括兼容性、运用XR技术、基于远距离传输的一些技术。到2012年，会增加双向语音、数字通讯和同轴供电等相关技术，把它整体的可用性、覆盖面以及整体的适用能力提的更高。3.0的版本，还会加双向语音的技术，传输距离也可以由原来的一百多米到将近三百米。后续HDcctv还可以达到每帧一千万像素的高度，包括3D影像也会逐渐的被考虑进去。随着整个HDcctv的不断发展，在更多的层面会相较于原来绝大部分要用到压缩的IP而言有更多的优势。
