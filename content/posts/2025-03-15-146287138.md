    <h2>
     <a id="_1">
     </a>
     介绍
    </h2>
    <h3>
     <a id="_runfile__2">
     </a>
     方法一：使用 runfile 手动安装（推荐）
    </h3>
    <ol>
     <li>
      <p>
       根据
       <a href="https://developer.nvidia.com/cuda-11-8-0-download-archive?target_os=Linux&amp;target_arch=x86_64&amp;Distribution=WSL-Ubuntu&amp;target_version=2.0&amp;target_type=runfile_local" rel="nofollow">
        CUDA Toolkit 11.8 Downloads | NVIDIA Developer
       </a>
       指示完成安装。
      </p>
     </li>
     <li>
      <p>
       设置环境变量：
      </p>
      <ul>
       <li>
        <code>
         PATH
        </code>
        ：将 CUDA 的可执行文件路径添加到系统路径。
       </li>
       <li>
        <code>
         LD_LIBRARY_PATH
        </code>
        ：将 CUDA 的库文件路径添加到动态链接库路径。
       </li>
      </ul>
      <pre><code class="prism language-bash"><span class="token builtin class-name">export</span> <span class="token assign-left variable">CUDA_HOME</span><span class="token operator">=</span>/usr/local/cuda-11.8
<span class="token builtin class-name">export</span> <span class="token assign-left variable"><span class="token environment constant">PATH</span></span><span class="token operator">=</span><span class="token variable">$CUDA_HOME</span>/bin:<span class="token environment constant">$PATH</span>
<span class="token builtin class-name">export</span> <span class="token assign-left variable">LD_LIBRARY_PATH</span><span class="token operator">=</span><span class="token variable">$CUDA_HOME</span>/lib64:<span class="token variable">$LD_LIBRARY_PATH</span>
</code></pre>
      <p>
       或者：
      </p>
      <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token string">'export CUDA_HOME=/usr/local/cuda-11.8'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">echo</span> <span class="token string">'export PATH=$CUDA_HOME/bin:$PATH'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">echo</span> <span class="token string">'export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
     </li>
     <li>
      <p>
       验证 CUDA 安装:
      </p>
      <pre><code class="prism language-bash">nvcc <span class="token parameter variable">--version</span>       <span class="token comment"># 应输出 11.8</span>
nvidia-smi           <span class="token comment"># 查看驱动和 GPU 状态</span>
/usr/local/cuda-11.8/extras/demo_suite/deviceQuery <span class="token comment"># 运行测试程序</span>
</code></pre>
     </li>
    </ol>
    <h3>
     <a id="apt_32">
     </a>
     方法二：通过apt安装
    </h3>
    <ol>
     <li>
      <p>
       配置 CUDA 仓库(需要注意不同系统配置不同)
      </p>
      <p>
       Unbuntu 24.04版本：
      </p>
      <pre><code class="prism language-bash"><span class="token comment"># 修改为适配 24.04 的路径</span>
<span class="token function">wget</span> https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-ubuntu2404.pin
<span class="token function">sudo</span> <span class="token function">mv</span> cuda-ubuntu2404.pin /etc/apt/preferences.d/cuda-repository-pin-600

<span class="token comment"># 添加 GPG 密钥（需确认密钥是否更新）</span>
<span class="token function">sudo</span> apt-key adv --fetch-keys https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/3bf863cc.pub

<span class="token comment"># 添加仓库</span>
<span class="token function">sudo</span> add-apt-repository <span class="token string">"deb https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/ /"</span>
</code></pre>
      <p>
       Ubuntu 22.04版本：
      </p>
      <pre><code class="prism language-bash"><span class="token function">wget</span> https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/cuda-wsl-ubuntu.pin
<span class="token function">sudo</span> <span class="token function">mv</span> cuda-wsl-ubuntu.pin /etc/apt/preferences.d/cuda-repository-pin-600
<span class="token function">sudo</span> apt-key adv --fetch-keys https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/3bf863cc.pub
<span class="token function">sudo</span> add-apt-repository <span class="token string">"deb https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/ /"</span>
</code></pre>
     </li>
     <li>
      <p>
       执行安装：
      </p>
      <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> cuda-toolkit-11-8
</code></pre>
     </li>
     <li>
      <p>
       配置环境：
      </p>
      <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token string">'export PATH=/usr/local/cuda-11.8/bin${PATH:+:${PATH}}'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">echo</span> <span class="token string">'export LD_LIBRARY_PATH=/usr/local/cuda-11.8/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}'</span> <span class="token operator">&gt;&gt;</span> ~/.bashrc
<span class="token builtin class-name">source</span> ~/.bashrc
</code></pre>
     </li>
    </ol>
    <h3>
     <a id="_CUDA_78">
     </a>
     其他：升级到更高版本 CUDA
    </h3>
    <pre><code class="prism language-bash"><span class="token comment"># 移除旧 CUDA 仓库</span>
<span class="token comment"># sudo rm /etc/apt/sources.list.d/cuda*.list</span>

<span class="token comment"># 添加新版本 CUDA 仓库， 按上述方法</span>

<span class="token comment"># 执行安装</span>
<span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> nvidia-cuda-toolkit
</code></pre>
    <p>
     使用
     <code>
      sudo apt install nvidia-cuda-toolkit
     </code>
     安装 CUDA 后，默认会将 CUDA 工具包安装到系统的标准路径中。为了正确配置环境变量（如
     <code>
      PATH
     </code>
     和
     <code>
      LD_LIBRARY_PATH
     </code>
     ），以下是详细步骤：
    </p>
    <hr/>
    <h4>
     <a id="1__CUDA__92">
     </a>
     1. 确认 CUDA 安装路径
    </h4>
    <p>
     通过
     <code>
      apt
     </code>
     安装的 CUDA 工具包通常会将文件分散到系统的多个目录中，而不是集中在一个目录（如
     <code>
      /usr/local/cuda
     </code>
     ）。以下是主要路径：
    </p>
    <ul>
     <li>
      <strong>
       CUDA 工具
      </strong>
      ：
      <code>
       /usr/bin
      </code>
     </li>
     <li>
      <strong>
       CUDA 库
      </strong>
      ：
      <code>
       /usr/lib/x86_64-linux-gnu
      </code>
     </li>
     <li>
      <strong>
       头文件
      </strong>
      ：
      <code>
       /usr/include
      </code>
     </li>
    </ul>
    <p>
     运行以下命令确认
     <code>
      nvcc
     </code>
     的路径：
    </p>
    <pre><code class="prism language-bash"><span class="token function">which</span> nvcc
</code></pre>
    <p>
     输出示例：
    </p>
    <pre><code>/usr/bin/nvcc
</code></pre>
    <hr/>
    <h4>
     <a id="2__109">
     </a>
     2. 验证环境变量
    </h4>
    <p>
     运行以下命令检查环境变量是否设置成功：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">echo</span> <span class="token environment constant">$PATH</span>
<span class="token builtin class-name">echo</span> <span class="token variable">$LD_LIBRARY_PATH</span>
</code></pre>
    <p>
     输出应包含
     <code>
      /usr/bin
     </code>
     和
     <code>
      /usr/lib/x86_64-linux-gnu
     </code>
     。
    </p>
    <p>
     同时，检查
     <code>
      nvcc
     </code>
     是否可用：
    </p>
    <pre><code class="prism language-bash">nvcc <span class="token parameter variable">--version</span>
</code></pre>
    <p>
     输出应显示 CUDA 版本信息。
    </p>
    <hr/>
    <h4>
     <a id="_125">
     </a>
     注意事项
    </h4>
    <ul>
     <li>
      <strong>
       多版本 CUDA
      </strong>
      ：如果系统中安装了多个 CUDA 版本，可以通过修改
      <code>
       PATH
      </code>
      和
      <code>
       LD_LIBRARY_PATH
      </code>
      切换版本。
     </li>
     <li>
      <strong>
       WSL2 环境
      </strong>
      ：在 WSL2 中，CUDA 的路径通常与 Linux 系统一致，但需确保 Windows 宿主机的 NVIDIA 驱动已正确安装。
     </li>
    </ul>
