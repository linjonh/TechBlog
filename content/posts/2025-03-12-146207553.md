    <h3>
     <a id="_0">
     </a>
     <strong>
      摘要
     </strong>
    </h3>
    <p>
     Git 是最流行的分布式版本控制系统。本指南详细整理了 Git 的
     <strong>
      基础操作、高级技巧、远程仓库管理、代理配置、子模块管理
     </strong>
     以及
     <strong>
      进阶使用技巧
     </strong>
     ，帮助你全面掌握 Git。
    </p>
    <hr/>
    <h3>
     <a id="1_Git__6">
     </a>
     <strong>
      1. Git 基础命令
     </strong>
    </h3>
    <h4>
     <a id="11__Git__8">
     </a>
     <strong>
      1.1 配置 Git 用户信息
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> user.name <span class="token string">"你的用户名"</span>
<span class="token function">git</span> config <span class="token parameter variable">--global</span> user.email <span class="token string">"你的邮箱"</span>
</code></pre>
    <p>
     💡
     <strong>
      作用
     </strong>
     ：设置全局用户名和邮箱，这些信息会出现在提交记录中。
    </p>
    <h4>
     <a id="12__Git__15">
     </a>
     <strong>
      1.2 初始化 Git 仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> init
</code></pre>
    <p>
     💡
     <strong>
      作用
     </strong>
     ：初始化一个新的 Git 仓库。
    </p>
    <h4>
     <a id="13__21">
     </a>
     <strong>
      1.3 克隆远程仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> clone <span class="token operator">&lt;</span>仓库URL<span class="token operator">&gt;</span>
</code></pre>
    <p>
     💡
     <strong>
      作用
     </strong>
     ：从远程仓库克隆代码到本地。
    </p>
    <h4>
     <a id="14__27">
     </a>
     <strong>
      1.4 添加文件到暂存区
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> <span class="token function">add</span> <span class="token operator">&lt;</span>文件名<span class="token operator">&gt;</span>   <span class="token comment"># 添加单个文件</span>
<span class="token function">git</span> <span class="token function">add</span> <span class="token builtin class-name">.</span>          <span class="token comment"># 添加所有更改的文件</span>
</code></pre>
    <h4>
     <a id="15__33">
     </a>
     <strong>
      1.5 提交更改
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> commit <span class="token parameter variable">-m</span> <span class="token string">"提交描述"</span>
</code></pre>
    <h4>
     <a id="16__Git__38">
     </a>
     <strong>
      1.6 查看 Git 状态
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> status
</code></pre>
    <h4>
     <a id="17__43">
     </a>
     <strong>
      1.7 查看提交历史
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> log
</code></pre>
    <h4>
     <a id="18__48">
     </a>
     <strong>
      1.8 查看文件修改
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> <span class="token function">diff</span>            <span class="token comment"># 查看未暂存的更改</span>
<span class="token function">git</span> <span class="token function">diff</span> <span class="token parameter variable">--staged</span>   <span class="token comment"># 查看已暂存但未提交的更改</span>
</code></pre>
    <h4>
     <a id="19__54">
     </a>
     <strong>
      1.9 撤销更改
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> checkout -- <span class="token operator">&lt;</span>文件名<span class="token operator">&gt;</span>    <span class="token comment"># 放弃未提交的更改</span>
<span class="token function">git</span> reset HEAD <span class="token operator">&lt;</span>文件名<span class="token operator">&gt;</span>     <span class="token comment"># 取消暂存</span>
<span class="token function">git</span> reset <span class="token parameter variable">--hard</span> HEAD       <span class="token comment"># 丢弃所有未提交的更改</span>
</code></pre>
    <hr/>
    <h3>
     <a id="2_Git__63">
     </a>
     <strong>
      2. Git 远程操作
     </strong>
    </h3>
    <h4>
     <a id="21__65">
     </a>
     <strong>
      2.1 查看远程仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> remote <span class="token parameter variable">-v</span>
</code></pre>
    <h4>
     <a id="22__70">
     </a>
     <strong>
      2.2 添加远程仓库
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> remote <span class="token function">add</span> origin <span class="token operator">&lt;</span>仓库URL<span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="23__75">
     </a>
     <strong>
      2.3 修改远程仓库地址
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> remote set-url origin <span class="token operator">&lt;</span>新的仓库URL<span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="24__80">
     </a>
     <strong>
      2.4 推送代码到远程
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> push origin <span class="token operator">&lt;</span>分支名<span class="token operator">&gt;</span>
<span class="token function">git</span> push <span class="token parameter variable">-u</span> origin <span class="token operator">&lt;</span>分支名<span class="token operator">&gt;</span>   <span class="token comment"># 设为默认上游分支</span>
</code></pre>
    <h4>
     <a id="25__86">
     </a>
     <strong>
      2.5 拉取代码
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> pull origin <span class="token operator">&lt;</span>分支名<span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="26__91">
     </a>
     <strong>
      2.6 获取远程更新
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> fetch origin
</code></pre>
    <h4>
     <a id="27__96">
     </a>
     <strong>
      2.7 删除远程分支
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> push origin <span class="token parameter variable">--delete</span> <span class="token operator">&lt;</span>分支名<span class="token operator">&gt;</span>
</code></pre>
    <h4>
     <a id="28__101">
     </a>
     <strong>
      2.8 删除远程仓库关联
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> remote remove origin
</code></pre>
    <hr/>
    <h3>
     <a id="3_Git__108">
     </a>
     <strong>
      3. Git 代理配置
     </strong>
    </h3>
    <h4>
     <a id="31__HTTP__110">
     </a>
     <strong>
      3.1 设置全局 HTTP 代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> http.proxy http://127.0.0.1:7897
<span class="token function">git</span> config <span class="token parameter variable">--global</span> https.proxy http://127.0.0.1:7897
</code></pre>
    <h4>
     <a id="32__116">
     </a>
     <strong>
      3.2 取消全局代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> http.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> https.proxy
</code></pre>
    <h4>
     <a id="33__122">
     </a>
     <strong>
      3.3 查看当前代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--get</span> http.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--get</span> https.proxy
</code></pre>
    <h4>
     <a id="34__GitHub__128">
     </a>
     <strong>
      3.4 仅对 GitHub 设置代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> http.https://github.com.proxy http://127.0.0.1:7897
<span class="token function">git</span> config <span class="token parameter variable">--global</span> https.https://github.com.proxy http://127.0.0.1:7897
</code></pre>
    <h4>
     <a id="35__GitHub__134">
     </a>
     <strong>
      3.5 取消仅对 GitHub 的代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> http.https://github.com.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> https.https://github.com.proxy
</code></pre>
    <h4>
     <a id="36__SOCKS5__140">
     </a>
     <strong>
      3.6 使用全局 SOCKS5 代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> http.proxy socks5://127.0.0.1:7897
<span class="token function">git</span> config <span class="token parameter variable">--global</span> https.proxy socks5://127.0.0.1:7897
</code></pre>
    <h4>
     <a id="37__SOCKS5__146">
     </a>
     <strong>
      3.7 取消全局 SOCKS5 代理
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> http.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> https.proxy
</code></pre>
    <h4>
     <a id="38__Git__152">
     </a>
     <strong>
      3.8 让公司内网 Git 不走代理
     </strong>
    </h4>
    <p>
     如果你在公司内网 Git (git.company.com) 访问不需要代理，确保 ~/.gitconfig 里没有它的代理配置，或者执行：
    </p>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> http.https://git.company.com.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> https.https://git.company.com.proxy
</code></pre>
    <h4>
     <a id="39__159">
     </a>
     <strong>
      3.9 针对单个远程仓库设置代理
     </strong>
    </h4>
    <p>
     如果你希望
     <code>
      只对某个仓库
     </code>
     使用代理，而不影响全局
     <code>
      Git
     </code>
     配置，可以在该仓库的
     <code>
      config
     </code>
     文件中进行本地设置。
    </p>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> http.https://git.company.com.proxy
<span class="token function">git</span> config <span class="token parameter variable">--global</span> <span class="token parameter variable">--unset</span> https.https://git.company.com.proxy
</code></pre>
    <h5>
     <a id="391__Git__166">
     </a>
     <strong>
      3.9.1 进入目标 Git 仓库
     </strong>
    </h5>
    <pre><code class="prism language-sh"><span class="token builtin class-name">cd</span> /path/to/your/repo
</code></pre>
    <h5>
     <a id="392____171">
     </a>
     <strong>
      3.9.2 设置
      <code>
       仅针对该仓库
      </code>
      走代理
     </strong>
     ：
    </h5>
    <pre><code class="prism language-sh"><span class="token function">git</span> config http.proxy http://127.0.0.1:7897
<span class="token function">git</span> config https.proxy http://127.0.0.1:7897
</code></pre>
    <p>
     💡 此配置只会影响当前仓库，不会影响其他 Git 仓库的网络访问。
    </p>
    <h5>
     <a id="393__178">
     </a>
     <strong>
      3.9.3 验证是否生效
     </strong>
    </h5>
    <pre><code class="prism language-sh"><span class="token function">git</span> config <span class="token parameter variable">--get</span> http.proxy
<span class="token function">git</span> config <span class="token parameter variable">--get</span> https.proxy
</code></pre>
    <p>
     如果返回 http://127.0.0.1:7897，说明该仓库代理已生效。
    </p>

</code></pre>

<span class="token function">git</span> submodule <span class="token function">sync</span>

    <h3>
     <a id="5__Git__224">
     </a>
     <strong>
      5. 进阶 Git 技巧
     </strong>
    </h3>
    <h4>
     <a id="51_Git_stash_226">
     </a>
     <strong>
      5.1 Git stash（存储未提交的修改）
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> stash           <span class="token comment"># 暂存所有未提交的更改</span>
<span class="token function">git</span> stash pop       <span class="token comment"># 恢复最近一次存储的更改</span>
<span class="token function">git</span> stash list      <span class="token comment"># 查看存储的更改列表</span>
<span class="token function">git</span> stash drop      <span class="token comment"># 删除最近一次存储的更改</span>
<span class="token function">git</span> stash <span class="token function">clear</span>     <span class="token comment"># 清空所有存储的更改</span>
</code></pre>
    <h4>
     <a id="52__235">
     </a>
     <strong>
      5.2 修改最近一次提交
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> commit <span class="token parameter variable">--amend</span> <span class="token parameter variable">-m</span> <span class="token string">"新的提交描述"</span>
</code></pre>
    <h4>
     <a id="53__240">
     </a>
     <strong>
      5.3 软重置、硬重置
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> reset <span class="token parameter variable">--soft</span> HEAD~1  <span class="token comment"># 撤回最近一次提交，保留更改</span>
<span class="token function">git</span> reset <span class="token parameter variable">--hard</span> HEAD~1  <span class="token comment"># 撤回最近一次提交并删除所有更改</span>
</code></pre>
    <h4>
     <a id="54__246">
     </a>
     <strong>
      5.4 交互式变基
     </strong>
    </h4>
    <pre><code class="prism language-sh"><span class="token function">git</span> rebase <span class="token parameter variable">-i</span> HEAD~3   <span class="token comment"># 交互式变基，修改最近 3 次提交</span>
</code></pre>
    <h3>
     <a id="6_Git__251">
     </a>
     <strong>
      6. Git 配置文件
     </strong>
    </h3>
    <p>
     Git 的配置文件按
     <strong>
      优先级
     </strong>
     分为 3 层：
    </p>
    <table>
     <thead>
      <tr>
       <th>
        <strong>
         级别
        </strong>
       </th>
       <th>
        <strong>
         文件路径
        </strong>
       </th>
       <th>
        <strong>
         作用范围
        </strong>
       </th>
       <th>
        <strong>
         优先级（高 → 低）
        </strong>
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         仓库级（Local）
        </strong>
       </td>
       <td>
        <code>
         .git/config
        </code>
        （当前仓库）
       </td>
       <td>
        仅影响当前 Git 仓库
       </td>
       <td>
        ⭐⭐⭐⭐⭐
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         用户级（Global）
        </strong>
       </td>
       <td>
        <code>
         ~/.gitconfig
        </code>
        或
        <code>
         ~/.config/git/config
        </code>
       </td>
       <td>
        影响当前用户的所有 Git 仓库
       </td>
       <td>
        ⭐⭐⭐
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         系统级（System）
        </strong>
       </td>
       <td>
        <code>
         /etc/gitconfig
        </code>
       </td>
       <td>
        影响整个系统所有 Git 用户
       </td>
       <td>
        ⭐
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     💡
     <strong>
      Git 读取配置时，优先使用更高优先级的配置
     </strong>
     。
     <br/>
     比如：如果
     <code>
      ~/.gitconfig
     </code>
     里定义了代理，而
     <code>
      .git/config
     </code>
     里没有，那 Git 会使用全局代理。
    </p>
    <h4>
     <a id="61_Linux__macOS__262">
     </a>
     <strong>
      6.1 Linux / macOS 下配置文件存在的路径
     </strong>
    </h4>
    <ul>
     <li>
      修改 ~/.gitconfig 影响当前用户。
     </li>
     <li>
      修改 /etc/gitconfig 影响所有用户。
     </li>
     <li>
      修改 .git/config 影响单个 Git 仓库。
     </li>
    </ul>
    <h4>
     <a id="62_Windows__267">
     </a>
     <strong>
      6.2 Windows 下配置文件存在的路径
     </strong>
    </h4>
    <ul>
     <li>
      修改 %USERPROFILE%\.gitconfig 影响当前用户。
     </li>
     <li>
      修改 C:\ProgramData\Git\config 影响所有用户。
     </li>
     <li>
      修改 C:\path\to\repo\.git\config 影响单个 Git 仓库。
     </li>
    </ul>
    <h4>
     <a id="63__Git__272">
     </a>
     <strong>
      6.3 全局代理（适用于所有 Git 服务器）
     </strong>
    </h4>
    <pre><code class="prism language-ini">[http]
    proxy = http://127.0.0.1:7897

[https]
    proxy = http://127.0.0.1:7897
</code></pre>

[https "https://github.com"]
