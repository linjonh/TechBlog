    <h2 id="%E4%B8%80.%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%B3%BB%E7%BB%9F%E6%A6%82%E8%BF%B0">
     一.计算机系统概述
    </h2>
    <h3 id="1.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%EF%BC%88%E8%BD%AF%E4%BB%B6%EF%BC%89%E6%A6%82%E5%BF%B5">
     1.操作系统（软件）概念
    </h3>
    <p>
     <strong>
      操作系统（OS）
     </strong>
     是指控制和管理整个计算机系统的硬件与软件资源，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合。
    </p>
    <h3 id="2.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E5%9F%BA%E6%9C%AC%E7%89%B9%E5%BE%81">
     2.操作系统基本特征
    </h3>
    <ul>
     <li>
      并发：指两个或多个事件在同一时间间隔内发生；并行（在同一时刻发生）。
     </li>
     <li>
      共享：指系统中的资源可供内存中多个并发执行的进程共同使用。
     </li>
     <li>
      虚拟：指把一个物理上的实体变为若干逻辑上的对应物。
     </li>
     <li>
      异步：多道程序环境允许多个程序并发执行，但由于资源有限，进程的执行并不是一贯到底的，而是走走停停的，它以不可预知的速度向前推进。
     </li>
    </ul>
    <h3 id="3.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E7%9A%84%E7%9B%AE%E6%A0%87%E5%92%8C%E5%8A%9F%E8%83%BD">
     3.操作系统的目标和功能
    </h3>
    <h4 id="%C2%A0%203.1.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E4%BD%9C%E4%B8%BA%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%B3%BB%E7%BB%9F%E8%B5%84%E6%BA%90%E7%9A%84%E7%AE%A1%E7%90%86%E8%80%85">
     3.1.操作系统作为计算机系统资源的管理者
    </h4>
    <ul>
     <li>
      处理机管理
     </li>
     <li>
      存储器管理
     </li>
     <li>
      文件管理
     </li>
     <li>
      设备管理
     </li>
    </ul>
    <h4 id="%C2%A0%203.2.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E4%BD%9C%E4%B8%BA%E7%94%A8%E6%88%B7%E4%B8%8E%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%A1%AC%E4%BB%B6%E7%B3%BB%E7%BB%9F%E4%B9%8B%E9%97%B4%E7%9A%84%E6%8E%A5%E5%8F%A3">
     3.2.操作系统作为用户与计算机硬件系统之间的接口
    </h4>
    <p>
     <strong>
      （1）命令接口
     </strong>
    </p>
    <ul>
     <li>
      联机命令接口（适用于分时或实时系统的接口，由一组键盘操作命令组成）
     </li>
     <li>
      脱机命令接口（适用于批处理系统，由一组作业控制命令组成）
     </li>
    </ul>
    <p>
     <strong>
      （2）程序接口（由一组系统调用组成）
     </strong>
    </p>
    <h4 id="%C2%A0%203.3.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E5%AE%9E%E7%8E%B0%E4%BA%86%E5%AF%B9%E8%AE%A1%E7%AE%97%E6%9C%BA%E8%B5%84%E6%BA%90%E7%9A%84%E6%89%A9%E5%85%85">
     3.3.操作系统实现了对计算机资源的扩充

    <h4 id="%C2%A0%204.6.%E4%B8%AA%E4%BA%BA%E8%AE%A1%E7%AE%97%E6%9C%BA%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F">

     5.操作系统运行环境
    </h3>
    <h4 id="%C2%A0%205.1.%E5%A4%84%E7%90%86%E6%9C%BA%E8%BF%90%E8%A1%8C%E6%A8%A1%E5%BC%8F">
     5.1.处理机运行模式
    </h4>
    <ul>
     <li>
      时钟管理
     </li>
     <li>
      中断机制
     </li>
     <li>
      原语
     </li>
    </ul>
    <h4 id="%C2%A0%205.2.%E4%B8%AD%E6%96%AD%E5%92%8C%E5%BC%82%E5%B8%B8">
     5.2.中断和异常
    </h4>
    <ul>
     <li>
      中断（外中断）：I/O结束中断，时钟中断
     </li>
     <li>
      异常（内中断）：程序的非法操作码、地址越界、运算溢出、虚拟系统的缺页及专门的陷入指令。
     </li>
    </ul>
    <h4 id="%C2%A0%205.3.%E7%B3%BB%E7%BB%9F%E8%B0%83%E7%94%A8">
     5.3.系统调用
    </h4>
    <ul>
     <li>
      用户程序可执行
      <strong>
       陷入指令（访管指令或trap指令）
      </strong>
      来发起系统调用，请求操作系统提供服务。
     </li>
    </ul>
    <h3 id="6.%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E7%BB%93%E6%9E%84">
     6.操作系统结构
    </h3>
    <h4 id="%C2%A0%206.1.%E5%88%86%E5%B1%82%E6%B3%95">
     <strong>
      6.1.分层法
     </strong>
    </h4>
    <p>
     <strong>
      最底层（层0）为硬件，最高层（层N）为用户接口，每层只能调用紧邻它的低层的功能和服务。
     </strong>
    </p>
    <h4 id="%C2%A0%206.2.%E6%A8%A1%E5%9D%97%E5%8C%96">
     6.2.模块化
    </h4>
    <p>
     <strong>
      将操作系统按功能划分为若干具有一定独立性的模块。
     </strong>
    </p>
    <ul>
     <li>
      内聚性
     </li>
     <li>
      耦合度
     </li>
    </ul>
    <h4 id="%C2%A0%206.3.%E5%AE%8F%E5%86%85%E6%A0%B8">
     6.3.宏内核
    </h4>
    <p>
     <strong>
      将系统的主要功能模块都作为一个紧密联系的整体运行在核心态
     </strong>
     ，从而为用户程序提供高性能的系统服务。
    </p>
    <h4 id="%C2%A0%206.4.%E5%BE%AE%E5%86%85%E6%A0%B8">
     6.4.微内核
    </h4>
    <p>
     <strong>
      指将内核中最基本的功能保留在内核
     </strong>
     ，而将那些不需要在核心态执行的功能移到用户态执行，从而降低内核的设计复杂性。
    </p>
    <h4 id="%C2%A0%206.5.%E5%A4%96%E6%A0%B8">
     6.5.外核
    </h4>
    <p>
     <strong>
      对机器进行分区，给每个用户整个资源的一个子集。
     </strong>
    </p>
    <h2 id="%E4%BA%8C.%E8%BF%9B%E7%A8%8B%E4%B8%8E%E7%BA%BF%E7%A8%8B">
     二.进程与线程
    </h2>
    <h3 id="1.%E8%BF%9B%E7%A8%8B%E4%B8%8E%E7%BA%BF%E7%A8%8B">
     1.进程与线程
    </h3>
    <h4 id="%C2%A0%201.1.%E8%BF%9B%E7%A8%8B%E7%9A%84%E6%A6%82%E5%BF%B5%E4%B8%8E%E7%89%B9%E5%BE%81">
     1.1.进程的概念与特征
    </h4>
    <p>
     <strong>
      （1）进程实体：
     </strong>
     由程序段、相关数据段和PCB三部分构成。
    </p>
    <p>
     <strong>
      （2）概念：
     </strong>
     进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位。
    </p>
    <p>
     <strong>
      （3）特征
     </strong>
    </p>
    <ul>
     <li>
      动态性：是进程最基本的特征。
     </li>
     <li>
      并发性：指多个进程实体同存于内存中，能在一段时间内同时运行。
     </li>
     <li>
      独立性
     </li>
     <li>
      异步性
     </li>
    </ul>
    <h4 id="%C2%A0%201.2.%E8%BF%9B%E7%A8%8B%E7%9A%84%E7%8A%B6%E6%80%81%E4%B8%8E%E8%BD%AC%E6%8D%A2">
     1.2.进程的状态与转换
    </h4>
    <ul>
     <li>
      运行态：程序正在处理机上运行。
     </li>
     <li>
      就绪态：进程获得了除处理机外的一切所需资源，一旦得到处理机，便可立即运行。
     </li>
     <li>
      阻塞态：进程正在等待某一事件而暂停运行，如等待某资源为可用（不包括处理机）或等待输入输出完成。
     </li>
     <li>
      创建态：进程正在被创建，尚未转到就绪态。
     </li>
     <li>
      结束态：进程正从系统中消失，可能是进程正常结束或其他原因退出运行。
     </li>
    </ul>
    <h4 id="%C2%A0%201.3.%E8%BF%9B%E7%A8%8B%E7%9A%84%E7%BB%84%E7%BB%87">
     1.3.进程的组织
    </h4>
    <p>
     <strong>
      （1）进程控制块（PCB）
     </strong>
    </p>
    <ul>
     <li>
      进程描述信息
     </li>
     <li>
      进程控制和管理信息
     </li>
     <li>
      资源分配清单
     </li>
     <li>
      处理机相关信息
     </li>
    </ul>
    <p>
     <strong>
      （2）程序段
     </strong>
    </p>
    <p>
     <strong>
      （3）数据段
     </strong>
    </p>
    <h4 id="%C2%A0%201.4.%E8%BF%9B%E7%A8%8B%E6%8E%A7%E5%88%B6">
     1.4.进程控制
    </h4>
    <ul>
     <li>
      进程的创建
     </li>
     <li>
      进程的终止
     </li>
     <li>
      进程的唤醒和阻塞
     </li>
    </ul>
    <h4 id="%C2%A0%201.5.%E8%BF%9B%E7%A8%8B%E7%9A%84%E9%80%9A%E4%BF%A1">
     1.5.进程的通信
    </h4>
    <p>
     <strong>
      （1）共享存储
     </strong>
    </p>
    <p>
     <strong>
      （2）消息传递（以格式化的信息为单位）
     </strong>
    </p>
    <ul>
     <li>
      直接通信方式
     </li>
     <li>
      间接通信方式：发送进程把消息发送到某个中间实体（信箱）。
     </li>
    </ul>
    <p>
     <strong>
      （3）管道通信
     </strong>
    </p>
    <h4 id="%C2%A0%201.6.%E7%BA%BF%E7%A8%8B%E5%92%8C%E5%A4%9A%E7%BA%BF%E7%A8%8B%E6%A8%A1%E5%9E%8B">
     1.6.线程和多线程模型
    </h4>
    <p>
     <strong>
      （1）线程概念
     </strong>
     ：线程是进程中的一个实体，是被系统独立调度和分派的基本单位，不拥有系统资源。
    </p>
    <p>
     <strong>
      （2）线程的实现方式
     </strong>
    </p>
    <ul>
     <li>
      用户级线程（ULT）
     </li>
     <li>
      内核级线程（KLT）
     </li>
    </ul>
    <p>
     <strong>
      （3）多线程模型
     </strong>
    </p>
    <ul>
     <li>
      多对一模型
     </li>
     <li>
      一对一模型
     </li>
     <li>
      多对多模型
     </li>
    </ul>
    <h3 id="2.%E5%A4%84%E7%90%86%E6%9C%BA%E8%B0%83%E5%BA%A6">
     2.处理机调度
    </h3>
    <h4 id="%C2%A0%202.1.%E8%B0%83%E5%BA%A6%E7%9A%84%E5%B1%82%E6%AC%A1">
     2.1.调度的层次
    </h4>
    <ul>
     <li>
      高级调度（作业调度）：外存-&gt;内存（面向作业），无-&gt;创建态-&gt;就绪态。
     </li>
     <li>
      中级调度（内存调度）：外存-&gt;内存（面向进程），挂起态-&gt;就绪态。
     </li>
     <li>
      低级调度（进程调度）：内存-&gt;CPU，就绪态-&gt;运行态。
     </li>
    </ul>
    <h4 id="%C2%A0%202.2.%E8%B0%83%E5%BA%A6%E7%9A%84%E7%9B%AE%E6%A0%87">
     2.2.调度的目标
    </h4>
    <ul>
     <li>
      CPU利用率
     </li>
     <li>
      系统吞吐量：表示单位时间内CPU完成作业的数量。
     </li>
     <li>
      周转时间：指从作业提交到作业完成所经历的时间。
     </li>
     <li>
      等待时间
     </li>
     <li>
      响应时间：从用户提交请求到系统首次产生响应所用的时间。
     </li>
    </ul>
    <h4 id="%C2%A0%202.3.%E8%BF%9B%E7%A8%8B%E8%B0%83%E5%BA%A6%E7%9A%84%E6%96%B9%E5%BC%8F">
     2.3.进程调度的方式
    </h4>
    <ul>
     <li>
      非抢占调度方式：只能由当前运行的进程主动放弃CPU。
     </li>
     <li>
      抢占调度方式：可由操作系统剥夺当前进程的CPU使用权。
     </li>
    </ul>
    <h4 id="%C2%A0%202.4.%E7%BB%8F%E5%85%B8%E8%B0%83%E5%BA%A6%E7%AE%97%E6%B3%95">
     2.4.经典调度算法
    </h4>
    <p>
     <strong>
      （1）先来先服务（FCFS）调度算法：
     </strong>
     按照到达的先后顺序调度。
    </p>
    <p>
     <strong>
      （2）短作业优先（SJF）调度算法：
     </strong>
     每次调度时选择当前已经到达且运行时间最短的作业/进程。
     <strong>
      （会导致饥饿现象）
     </strong>
    </p>
    <p>
     <strong>
      （3）优先级调度算法：
     </strong>
     适用于实时操作系统，可能发生饥饿现象。
    </p>
    <ul>
     <li>
      静态优先级：在创建进程时确定的，且在进程的整个运行期间保持不变。
     </li>
     <li>
      动态优先级：在进程运行过程中，根据进程情况的变化动态调整优先级。
     </li>
     <li>
      优先级
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       系统进程&gt;用户进程
      </strong>
     </li>
     <li>
      <strong>
       交互型进程&gt;非交互型进程（或前台进程&gt;后台进程）
      </strong>
     </li>
     <li>
      <strong>
       I/O型进程&gt;计算型进程
      </strong>
     </li>
    </ol>
    <p>
     <strong>
      （4）高响应比优先调度算法（非抢占式调度算法）：
     </strong>
     响应比=（等待时间+要求服务时间）/  要求服务时间。
    </p>
    <p>
     <strong>
      （5）时间片轮转调度算法：
     </strong>
     适用于分时系统。
    </p>
    <p>
     <strong>
      （6）多级队列调度算法
     </strong>
    </p>
    <p>
     <strong>
      （7）多级反馈队列调度算法
     </strong>
    </p>
    <ul>
     <li>
      设置多个就绪队列，并为每个队列赋予不同的优先级。
     </li>
     <li>
      各队列的进程运行时间片的大小各不相同，优先级越高的队列中，每个进程的时间片就越小。
     </li>
     <li>
      每个队列都采用FCFS算法。
     </li>
     <li>
      按队列优先级调度。
     </li>
    </ul>
    <h3 id="3.%E5%90%8C%E6%AD%A5%E4%B8%8E%E4%BA%92%E6%96%A5">
     3.同步与互斥
    </h3>
    <h4 id="%C2%A0%203.1.%E5%90%8C%E6%AD%A5%E4%B8%8E%E4%BA%92%E6%96%A5%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%A6%82%E5%BF%B5">
     3.1.同步与互斥的基本概念
    </h4>
    <p>
     <strong>
      （1）临界资源
     </strong>
    </p>
    <ul>
     <li>
      进入区：设置正在访问临界区的标志。
     </li>
     <li>
      临界区：进程中访问临界资源的那段代码。
     </li>
     <li>
      退出区：将正在访问临界区的标志驱除。
     </li>
     <li>
      剩余区：代码中的其余部分。
     </li>
    </ul>
    <p>
     <strong>
      （2）同步
     </strong>
    </p>
    <ul>
     <li>
      空闲让进
     </li>
     <li>
      忙则等待
     </li>
     <li>
      有限等待
     </li>
     <li>
      让权等待
     </li>
    </ul>
    <p>
     <strong>
      （3）互斥：
     </strong>
     也称间接制约关系。
    </p>
    <h4 id="%C2%A0%203.2.%E5%AE%9E%E7%8E%B0%E4%B8%B4%E7%95%8C%E5%8C%BA%E4%BA%92%E6%96%A5%E7%9A%84%E5%9F%BA%E6%9C%AC%E6%96%B9%E6%B3%95">
     3.2.实现临界区互斥的基本方法
    </h4>
    <p>
     <strong>
      （1）软件实现方法
     </strong>
    </p>
    <ul>
     <li>
      单标志法（违背“空闲让进”）
     </li>
     <li>
      双标志法先检查（违背“忙则等待”）
     </li>
     <li>
      双标志法后检查（违背“空闲让进”、“有限等待”，导致饥饿现象）
     </li>
     <li>
      Peterson's Algorithm（违背“让权等待”）
     </li>
    </ul>
    <p>
     <strong>
      （2）硬件实现方法
     </strong>
    </p>
    <ul>
     <li>
      中断屏蔽方法：不适合多处理机，只用于操作系统内核进程。
     </li>
     <li>
      硬件指令方法
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       TestAndSet指令：执行该代码时不允许被中断。
      </strong>
     </li>
     <li>
      <strong>
       Swap指令：交换两个字（字节）的内容。
      </strong>
     </li>
    </ol>
    <h4 id="%C2%A0%203.3.%E4%BA%92%E6%96%A5%E9%94%81%EF%BC%88%E9%80%9A%E5%B8%B8%E9%87%87%E7%94%A8%E7%A1%AC%E4%BB%B6%E6%9C%BA%E5%88%B6%E6%9D%A5%E5%AE%9E%E7%8E%B0%EF%BC%89">
     3.3.互斥锁（通常采用硬件机制来实现）
    </h4>
    <h4 id="%C2%A0%203.4.%E4%BF%A1%E5%8F%B7%E9%87%8F">
     3.4.信号量
    </h4>
    <ul>
     <li>
      wait(S)：P操作
     </li>
     <li>
      signal(S)：V操作
     </li>
     <li>
      利用信号量实现同步：必须保证“一前一后”执行两个操作。
     </li>
    </ul>
    <ol>
     <li>
      <strong>
       在“前操作”之后执行V(S)
      </strong>
     </li>
     <li>
      <strong>
       在“后操作”之前执行P(S)
      </strong>
     </li>
    </ol>
    <ul>
     <li>
      利用信号量实现进程互斥：临界区之间执行P操作，临界区之后执行V操作。
     </li>
     <li>
      利用信号量实现前驱关系
     </li>
     <li>
      对于不同的临界资源需要设置不同的互斥信号量。
     </li>
    </ul>
    <h4 id="%C2%A0%203.5.%E7%AE%A1%E7%A8%8B">
     3.5.管程
    </h4>
    <ul>
     <li>
      管程的名称
     </li>
     <li>
      局部于管程内部的共享数据结构说明。
     </li>
     <li>
      对该数据结构进行操作的一组过程（或函数）--管程内的过程。
     </li>
     <li>
      对局部于管程内部的共享数据设置初始值的语句。
     </li>
    </ul>
    <h4 id="%C2%A0%203.6.%E7%BB%8F%E5%85%B8%E5%90%8C%E6%AD%A5%E9%97%AE%E9%A2%98">
     3.6.经典同步问题
    </h4>
    <ul>
     <li>
      生产者-消费者问题：实现互斥的P操作需在实现同步的P操作之后。
     </li>
     <li>
      读者-写者问题
     </li>
     <li>
      哲学家进餐问题
     </li>
     <li>
      吸烟者问题
     </li>
    </ul>
    <h3 id="4.%E6%AD%BB%E9%94%81">

    <h4 id="%C2%A0%204.1.%E6%AD%BB%E9%94%81%E7%9A%84%E5%AE%9A%E4%B9%89">
     4.1.死锁的定义
    </h4>
    <p>
     <strong>
      指多个进程因竞争资源而造成的一种僵局（互相等待），若无外力作用，这些进程都将无法向前推进。
     </strong>
    </p>
    <h4 id="%C2%A0%204.2.%E6%AD%BB%E9%94%81%E4%BA%A7%E7%94%9F%E7%9A%84%E5%8E%9F%E5%9B%A0">
     4.2.死锁产生的原因
    </h4>
    <ul>
     <li>
      系统资源的竞争
     </li>
     <li>
      进程推进顺序非法
     </li>
     <li>
      信号量使用不当
     </li>
    </ul>
    <h4 id="%C2%A0%204.3.%E6%AD%BB%E9%94%81%E4%BA%A7%E7%94%9F%E7%9A%84%E5%BF%85%E8%A6%81%E6%9D%A1%E4%BB%B6">
     4.3.死锁产生的必要条件
    </h4>
    <ul>
     <li>
      互斥条件
     </li>
     <li>
      不剥夺条件
     </li>
     <li>
      请求并保持条件
     </li>
     <li>
      循环等待条件
     </li>
    </ul>
    <h4 id="%C2%A0%204.4.%E6%AD%BB%E9%94%81%E7%9A%84%E5%A4%84%E7%90%86%E7%AD%96%E7%95%A5">
     4.4.死锁的处理策略
    </h4>
    <p>
     <strong>
      （1）死锁预防
     </strong>
    </p>

      破坏互斥条件

     </li>
     <li>
      破坏循环等待条件：采用顺序资源分配法
     </li>
    </ul>
    <p>
     <strong>
      （2）死锁避免：银行家算法
     </strong>
    </p>
    <p>
     <strong>
      （3）死锁的检测及解除
     </strong>
    </p>
    <ul>

     </li>
     <li>
      死锁解除
     </li>

    <ol>
     <li>
      <strong>
       资源剥夺法
      </strong>
     </li>

      <strong>

    <h4 id="%C2%A0%201.7.%E5%86%85%E5%AD%98%E5%88%86%E9%85%8D%E4%B8%8E%E5%9B%9E%E6%94%B6">
     1.7.内存分配与回收
    </h4>
    <h3 id="2.%E8%A6%86%E7%9B%96%E4%B8%8E%E4%BA%A4%E6%8D%A2">
     2.覆盖与交换
    </h3>
    <ul>
     <li>
      覆盖：内存中能更新的地方只有覆盖区的段，不在覆盖区中的段会常驻内存（对用户不透明）。
     </li>
     <li>
      交换：把处于等待状态的程序从内存移到辅存，把准备好竞争CPU运行的程序从辅存移到内存。
     </li>
    </ul>
    <h3 id="3.%E8%BF%9E%E7%BB%AD%E5%88%86%E9%85%8D%E7%AE%A1%E7%90%86%E6%96%B9%E5%BC%8F%EF%BC%88%E4%B8%BA%E7%94%A8%E6%88%B7%E7%A8%8B%E5%BA%8F%E5%88%86%E9%85%8D%E4%B8%80%E4%B8%AA%E8%BF%9E%E7%BB%AD%E7%9A%84%E5%86%85%E5%AD%98%E7%A9%BA%E9%97%B4%EF%BC%89">
     3.连续分配管理方式（为用户程序分配一个连续的内存空间）
    </h3>
    <h4 id="%C2%A0%203.1.%E5%8D%95%E4%B8%80%E8%BF%9E%E7%BB%AD%E5%88%86%E9%85%8D">
     3.1.单一连续分配
    </h4>
    <ul>
     <li>
      无外部碎片，仅有一道用户程序。
     </li>
    </ul>
    <h4 id="%C2%A0%203.2.%E5%9B%BA%E5%AE%9A%E5%88%86%E5%8C%BA%E5%88%86%E9%85%8D">
     3.2.固定分区分配
    </h4>
    <ul>
     <li>
      无外部碎片，用于多道程序存储管理方式。
     </li>
    </ul>
    <h4 id="%C2%A0%203.3.%E5%8A%A8%E6%80%81%E5%88%86%E5%8C%BA%E5%88%86%E9%85%8D">
     3.3.动态分区分配
    </h4>
    <ul>
     <li>
      首次适应算法：空闲分区以地址递增的次序链接。
     </li>
     <li>
      邻近适应算法：分配内存时从上次查找结束的位置开始继续查找。
     </li>
     <li>
      最佳适应算法：空闲分区按容量递增的次序形成空闲分区链。（会产生外部碎片）
     </li>
     <li>
      最坏适应算法：空闲分区以容量递减的次序链接。
     </li>
    </ul>
    <h3 id="4.%E5%9F%BA%E6%9C%AC%E5%88%86%E9%A1%B5%E5%AD%98%E5%82%A8%E7%AE%A1%E7%90%86%EF%BC%88%E4%B8%80%E7%BB%B4%EF%BC%89">

    <h4 id="%C2%A0%204.6.%E4%B8%A4%E7%BA%A7%E9%A1%B5%E8%A1%A8">
     4.6.两级页表

     7.1.虚拟存储的基本概念
    </h4>
    <p>
     <strong>
      （1）局部性原理
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       时间局部性：
      </strong>
      程序中的某条指令一旦执行，不久后该指令可能再次执行；某数据被访问过，不久后该数据可能再次被访问。（产生原因是程序中存在着大量的循环操作）
     </li>
     <li>
      <strong>
       空间局部性：
      </strong>
      一旦程序访问了某个存储单元，在不久后，其附近的存储单元也将被访问，即程序在一段时间内所访问的地址，可能集中在一定的范围之内，因为指令通常是顺序存放、顺序执行的，数据也一般是以向量、数组、表等形式簇聚存储的。
     </li>
    </ul>
    <p>
     <strong>
      （2）虚拟存储器的特征
     </strong>
    </p>
    <ul>
     <li>
      多次性
     </li>
     <li>
      对换性
     </li>
     <li>
      虚拟性
     </li>
    </ul>

    <h4 id="%C2%A0%201.1.I%2FO%E8%AE%BE%E5%A4%87">
     1.1.I/O设备
    </h4>
    <ul>
     <li>
      块设备：信息交换以数据块为单位。
     </li>
     <li>
      字符设备：信息交换以字符为单位。
     </li>
     <li>
      低速设备：鼠标、键盘。
     </li>
     <li>
      中速设备：激光打印机。
     </li>
     <li>
      高速设备：磁盘机、光盘机。
     </li>
    </ul>
    <h4 id="%C2%A0%201.2.I%2FO%E6%8E%A5%E5%8F%A3%EF%BC%88%E4%BD%8D%E4%BA%8ECPU%E4%B8%8E%E8%AE%BE%E5%A4%87%E4%B9%8B%E9%97%B4%EF%BC%89">
     1.2.I/O接口（位于CPU与设备之间）
    </h4>
    <p>
     <strong>
      （1）设备控制器与CPU接口
     </strong>
    </p>
    <ul>
     <li>
      数据线
     </li>
     <li>
      地址线
     </li>
     <li>
      控制线
     </li>
    </ul>
    <p>
     <strong>
      （2）设备控制器与设备的接口
     </strong>
    </p>
    <p>
     <strong>
      （3）I/O逻辑：用于实现对设备的控制。
     </strong>
    </p>
    <h4 id="%C2%A0%201.3.I%2FO%E7%AB%AF%E5%8F%A3%EF%BC%9A%E6%8C%87%E8%AE%BE%E5%A4%87%E6%8E%A7%E5%88%B6%E5%99%A8%E4%B8%AD%E5%8F%AF%E8%A2%ABCPU%E7%9B%B4%E6%8E%A5%E8%AE%BF%E9%97%AE%E7%9A%84%E5%AF%84%E5%AD%98%E5%99%A8%E3%80%82">
     1.3.I/O端口：指设备控制器中可被CPU直接访问的寄存器。
    </h4>
    <ul>
     <li>
      数据寄存器
     </li>
     <li>
      状态寄存器
     </li>
     <li>
      控制寄存器
     </li>
    </ul>
    <h4 id="%C2%A0%201.4.I%2FO%20%E6%8E%A7%E5%88%B6%E6%96%B9%E5%BC%8F">
     1.4.I/O 控制方式
    </h4>
    <p>
     <strong>
      （1）程序直接控制方式
     </strong>
    </p>
    <p>
     <strong>
      （2）中断驱动方式
     </strong>
    </p>
    <p>
     <strong>
      （3）DMA方式
     </strong>
    </p>
    <p>
     <strong>
      （4）通道控制方式
     </strong>
    </p>
    <ul>
     <li>
      I/O通道指专门负责输入/输出的处理机（硬件）。
     </li>
     <li>
      一个通道可控制多个设备控制器，每个设备控制器可以控制多个设备。
     </li>
     <li>
      通道与CPU共享内存。
     </li>
    </ul>
    <h4 id="%C2%A0%201.5.I%2FO%E8%BD%AF%E4%BB%B6%E5%B1%82%E6%AC%A1%E7%BB%93%E6%9E%84">
     1.5.I/O软件层次结构
    </h4>
    <ul>
     <li>
      用户层I/O软件（实现与用户交互的接口）
     </li>
     <li>
      设备独立性软件：用于实现用户程序与设备驱动器的统一接口、设备命令、设备的保护及设备的分配与回收。
     </li>
     <li>
      设备驱动程序（每类设备配置一个）：与硬件直接相关，负责具体实现系统对设备发出的操作指令，驱动I/O设备工作的驱动程序。
     </li>
     <li>
      中断处理程序：用于保存被中断进程的CPU环境，转入相应的中断处理程序进行处理，处理完毕再恢复被中断进程的现场后，返回到被中断进程。
     </li>
    </ul>

     2.1.高速缓冲与缓冲区
    </h4>
    <p>
     <strong>
      （1）磁盘高速缓存

     <strong>
      （2）缓冲区
     </strong>

     <li>
      单缓冲

     </li>

    <h4 id="%C2%A0%202.2.%E8%AE%BE%E5%A4%87%E5%88%86%E9%85%8D%E4%B8%8E%E5%9B%9E%E6%94%B6">
     2.2.设备分配与回收
    </h4>
    <p>
     <strong>
      （1）设备分配
     </strong>
    </p>
    <ul>
     <li>
      独占式使用设备
     </li>
     <li>
      分时式共享使用设备
     </li>
     <li>
      以SPOOLing方式使用外部设备（实现了对设备的I/O操作的批处理）
     </li>
    </ul>
    <p>
     <strong>
      （2）设备分配的数据结构
     </strong>
    </p>
    <ul>
     <li>
      设备控制表（DCT）
     </li>
     <li>
      控制器设备表（COCT）
     </li>
     <li>
      通道控制表（CHCT）
     </li>
     <li>
      系统设备表（SDT）：整个系统只有一张。
     </li>
    </ul>
    <h4 id="%C2%A0%202.3.SPOOLing%E6%8A%80%E6%9C%AF%EF%BC%88%E5%81%87%E8%84%B1%E6%9C%BA%E6%8A%80%E6%9C%AF%EF%BC%89">
     2.3.SPOOLing技术（假脱机技术）
    </h4>
    <ul>
     <li>
      输入井和输出井（在磁盘中开辟两个存储区域）
     </li>
     <li>
      输入缓冲区和输出缓冲区
     </li>
     <li>
      输入进程和输出进程
     </li>
     <li>
      特点
     </li>
    </ul>
    <ol>
     <li>
      提高了I/O的速度
     </li>
     <li>
      将独占设备改造为共享设备
     </li>
     <li>
      实现了虚拟设备功能
     </li>
    </ol>
