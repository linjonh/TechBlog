    <h3>
     <a id="1__0">
     </a>
     <strong>
      1. 简介
     </strong>
    </h3>
    <p>
     Powerlevel10k 是 Oh My Zsh 最流行的终端主题，它不仅美观，还提供
     <strong>
      Git 状态显示、命令执行时间、网络状态、Python 虚拟环境指示等
     </strong>
     实用功能。相比其他主题，Powerlevel10k
     <strong>
      速度更快、可定制性更强
     </strong>
     。
    </p>
    <p>
     本教程将详细介绍如何安装 Powerlevel10k，并
     <strong>
      完整解读每个配置选项
     </strong>
     ，帮助你打造个性化的终端体验。
    </p>
    <hr/>
    <h3>
     <a id="2__Powerlevel10k_7">
     </a>
     <strong>
      2. 安装 Powerlevel10k
     </strong>
    </h3>
    <h4>
     <a id="21__aptyumpacman__9">
     </a>
     <strong>
      2.1 通过 apt/yum/pacman 安装（适用于部分发行版）
     </strong>
    </h4>
    <p>
     在部分 Linux 发行版（如 Arch Linux 和一些带有 Powerlevel10k 的 Ubuntu 变种）中，可以直接通过包管理器安装 Powerlevel10k：
    </p>
    <p>
     <strong>
      Ubuntu/Debian
     </strong>
     （可能没有最新版本）
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> <span class="token function">install</span> zsh-theme-powerlevel10k <span class="token parameter variable">-y</span>
</code></pre>
    <p>
     <strong>
      Arch Linux（推荐使用官方仓库）
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> pacman <span class="token parameter variable">-S</span> zsh-theme-powerlevel10k
</code></pre>
    <p>
     但是，使用 Git 安装通常可以获取最新版本。
    </p>
    <h4>
     <a id="22__Git__24">
     </a>
     <strong>
      2.2 通过 Git 安装（推荐方式）
     </strong>
    </h4>
    <p>
     Powerlevel10k 可以通过 Git 克隆到 Oh My Zsh 的主题目录。
    </p>
    <p>
     <strong>
      第一种方式
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone <span class="token parameter variable">--depth</span><span class="token operator">=</span><span class="token number">1</span> https://github.com/romkatv/powerlevel10k.git <span class="token variable">${ZSH_CUSTOM<span class="token operator">:-</span>$HOME<span class="token operator">/</span>.oh-my-zsh<span class="token operator">/</span>custom}</span>/themes/powerlevel10k
</code></pre>
    <ul>
     <li>
      更加兼容，不管你的 ZSH_CUSTOM 在哪里，它都能正确安装 Powerlevel10k。
     </li>
     <li>
      适用于不同环境（macOS / Linux / WSL），无需担心 Oh My Zsh 目录位置不同的问题。
     </li>
     <li>
      避免路径写死（硬编码），提高可维护性。
     </li>
    </ul>
    <p>
     <strong>
      第二种方式（推荐）
     </strong>
    </p>
    <pre><code class="prism language-bash"><span class="token function">git</span> clone <span class="token parameter variable">--depth</span><span class="token operator">=</span><span class="token number">1</span> https://github.com/romkatv/powerlevel10k.git ~/.oh-my-zsh/custom/themes/powerlevel10k
</code></pre>
    <ul>
     <li>
      适用于默认安装路径，但如果你曾修改过 Oh My Zsh 目录，就可能安装失败。
     </li>
    </ul>
    <p>
     如果你的服务器无法直接访问 GitHub，可以在其他设备下载后手动传输。
    </p>
    <h4>
     <a id="23__Powerlevel10k__44">
     </a>
     <strong>
      2.3 配置 Powerlevel10k 主题
     </strong>
    </h4>
    <p>
     编辑
     <code>
      ~/.zshrc
     </code>
     ，找到
     <code>
      ZSH_THEME
     </code>
     这一行，并修改为：
    </p>
    <pre><code class="prism language-bash"><span class="token assign-left variable">ZSH_THEME</span><span class="token operator">=</span><span class="token string">"powerlevel10k/powerlevel10k"</span>
</code></pre>
    <p>
     然后
     <strong>
      应用更改
     </strong>
     ：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> ~/.zshrc
</code></pre>
    <p>
     如果你正确安装了 Powerlevel10k，
     <strong>
      终端会自动启动配置向导
     </strong>
     。
    </p>
    <hr/>
    <h3>
     <a id="3__Powerlevel10k_61">
     </a>
     <strong>
      3. 更新 Powerlevel10k
     </strong>
    </h3>
    <p>
     如果你通过 Git 安装了 Powerlevel10k，可以使用以下命令更新到最新版本：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">cd</span> ~/.oh-my-zsh/custom/themes/powerlevel10k
<span class="token function">git</span> pull
</code></pre>
    <p>
     然后重新加载 Zsh 使更改生效：
    </p>
    <pre><code class="prism language-bash"><span class="token builtin class-name">source</span> ~/.zshrc
</code></pre>
    <p>
     如果你通过
     <code>
      apt
     </code>
     或
     <code>
      pacman
     </code>
     安装的 Powerlevel10k，可以使用包管理器进行更新：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">apt</span> update <span class="token operator">&amp;&amp;</span> <span class="token function">sudo</span> <span class="token function">apt</span> upgrade <span class="token parameter variable">-y</span>  <span class="token comment"># Ubuntu/Debian</span>
<span class="token function">sudo</span> pacman <span class="token parameter variable">-Syu</span>  <span class="token comment"># Arch Linux</span>
</code></pre>
    <hr/>
    <h3>
     <a id="4_Powerlevel10k__82">
     </a>
     <strong>
      4. Powerlevel10k 重大版本升级
     </strong>
    </h3>
    <p>
     有时 Powerlevel10k 进行大版本更新，可能会导致配置文件
     <code>
      ~/.p10k.zsh
     </code>
     失效，建议在升级后重新进行配置。
    </p>
    <ol>
     <li>
      <strong>
       删除旧配置（可选）
      </strong>
     </li>
    </ol>
    <pre><code class="prism language-bash"><span class="token function">mv</span> ~/.p10k.zsh ~/.p10k.zsh.bak
</code></pre>
    <ol start="2">
     <li>
      <strong>
       运行配置向导
      </strong>
     </li>
    </ol>
    <pre><code class="prism language-bash">p10k configure
</code></pre>
    <ol start="3">
     <li>
      <strong>
       重新设置 Powerlevel10k 并应用新配置
      </strong>
     </li>
    </ol>
    <p>
     如果遇到问题，可以参考官方文档：
     <a href="https://github.com/romkatv/powerlevel10k">
      Powerlevel10k 官方 GitHub
     </a>
    </p>
    <hr/>
    <h3>
     <a id="5__Powerlevel10k_99">
     </a>
     <strong>
      5. 配置 Powerlevel10k
     </strong>
    </h3>
    <p>
     Powerlevel10k 的首次运行会引导你进行
     <strong>
      交互式配置
     </strong>
     。以下是每个选项的解读和推荐选择。
    </p>
    <h4>
     <a id="51_Prompt_Style_103">
     </a>
     <strong>
      5.1 选择提示符风格（Prompt Style）
     </strong>
    </h4>
    <p>
     你会看到三种风格：
    </p>
    <ul>
     <li>
      <strong>
       Lean（精简模式）
      </strong>
      <ul>
       <li>
        仅显示基本信息（路径、分支）。
       </li>
       <li>
        适合喜欢简洁风格的用户。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Classic（经典模式）
      </strong>
      <ul>
       <li>
        传统 robbyrussell 风格，带有分支信息。
       </li>
       <li>
        适合不想要太多装饰但仍然希望有一些增强的用户。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Rainbow（彩色模式）
      </strong>
      <ul>
       <li>
        最推荐！带有 颜色分隔 和 状态指示，可读性更高。
       </li>
       <li>
        适合喜欢酷炫终端界面的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      3
     </code>
     （Rainbow），它拥有最丰富的颜色和状态信息，适合需要 Git 状态、网络状态等信息的用户。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/3518392af5bf4c63901c785fb252d9bf.png"/>
    </p>
    <h4>
     <a id="52_Prompt_Height_119">
     </a>
     <strong>
      5.2 选择提示符高度（Prompt Height）
     </strong>
    </h4>
    <p>
     你可以选择：
    </p>
    <ul>
     <li>
      <strong>
       One line（单行）
      </strong>
      <ul>
       <li>
        终端提示符信息（路径、分支、状态等）全部显示在一行。
       </li>
       <li>
        适合小窗口或喜欢节省空间的用户。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       Two lines（双行）
      </strong>
      <ul>
       <li>
        第一行显示路径、Git 分支、状态。
       </li>
       <li>
        第二行只显示命令输入区域 &gt;。
       </li>
       <li>
        适合喜欢清晰分隔信息的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      2
     </code>
     （Two lines），这样你可以获得更清晰的终端信息。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/284ed4ce69b34c41a52e85849eab650b.png"/>
    </p>
    <h4>
     <a id="53_Prompt_Connection_133">
     </a>
     <strong>
      5.3 选择提示符连接样式（Prompt Connection）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Disconnected
      </code>
      （无连接线）
      <ul>
       <li>
        无任何分隔线，每一行的提示符都是独立的。
       </li>
       <li>
        适合喜欢简洁风格的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Dotted
      </code>
      （虚线）
      <ul>
       <li>
        使用点线连接多行提示符，使界面更整齐。
       </li>
       <li>
        适合喜欢有一定视觉分隔但又不想太突出的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Solid
      </code>
      （实线）
      <ul>
       <li>
        使用实线连接提示符，使界面更有条理。
       </li>
       <li>
        适合喜欢清晰、结构化终端界面的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      3
     </code>
     （Solid），这样整个终端看起来更加整洁有序。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/8620c9a13d6246ca9238d61b50187ade.png"/>
    </p>
    <h4>
     <a id="54_Connection_Color_148">
     </a>
     <strong>
      5.4 选择连接颜色（Connection Color）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Lightest
      </code>
      <ul>
       <li>
        分隔线颜色非常淡，几乎不可见。
       </li>
       <li>
        适合喜欢 低对比度、极简 终端界面的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Light
      </code>
      <ul>
       <li>
        分隔线颜色较浅，不会太显眼，但仍然清晰可见。
       </li>
       <li>
        适合希望 稍微有一点视觉分隔，但不突出的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Dark
      </code>
      <ul>
       <li>
        分隔线颜色较深，与终端背景形成 明显对比。
       </li>
       <li>
        适合喜欢 清晰分隔 的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Darkest
      </code>
      <ul>
       <li>
        分隔线颜色最深，与终端背景 对比最强烈。
       </li>
       <li>
        适合在 黑色背景下增强分隔，或者 高对比度 偏好的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      3
     </code>
     （Dark），避免过亮影响阅读。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/030476b5535744f692dcea5b520f2885.png"/>
    </p>
    <h4>
     <a id="55_Prompt_Spacing_166">
     </a>
     <strong>
      5.5 选择提示符间距（Prompt Spacing）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Compact
      </code>
      （紧凑）
      <ul>
       <li>
        提示符行之间间距较小，节省屏幕空间。
       </li>
       <li>
        适合小屏幕设备或者希望尽量减少滚屏的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Sparse
      </code>
      （宽松）
      <ul>
       <li>
        提示符行之间增加额外的空行，提高可读性。
       </li>
       <li>
        适合大屏幕或喜欢清晰分隔的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      1
     </code>
     （Compact），可以减少屏幕占用，提高可读性。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/2b4fc1550f734ad9af046e6e84b151fc.png"/>
    </p>
    <h4>
     <a id="56_Prompt_Flow_178">
     </a>
     <strong>
      5.6 选择提示符流畅度（Prompt Flow）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Concise
      </code>
      （简洁模式）
      <ul>
       <li>
        直接显示必要的信息，如 路径、Git 分支、执行时间。
       </li>
       <li>
        不使用额外的文字描述。
       </li>
       <li>
        适合 希望界面简洁、信息紧凑 的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Fluent
      </code>
      （流畅模式，显示额外提示信息）
      <ul>
       <li>
        在信息前加了一些自然语言描述。
       </li>
       <li>
        适合 喜欢直观、带有语义提示 的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      2
     </code>
     （Fluent），可以获得更完整的命令执行信息。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/75c7c408e53a495ab1f237c182003b7a.png"/>
    </p>
    <h4>
     <a id="57_Enable_Transient_Prompt_191">
     </a>
     <strong>
      5.7 是否启用即时提示（Enable Transient Prompt?）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Yes
      </code>
      （推荐）：清理旧的提示符，只保留当前命令的提示。
      <ul>
       <li>
        运行命令后，之前的提示符会被简化，只保留 &gt;，不显示路径、Git 分支等信息。
       </li>
       <li>
        让终端更整洁，不会被旧的提示符占据太多空间。
       </li>
       <li>
        适合喜欢简洁、紧凑风格的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       No
      </code>
      ：保留所有历史提示符。
      <ul>
       <li>
        运行命令后，所有旧提示符都会完整保留，包括路径、Git 分支、时间等。
       </li>
       <li>
        适合希望保留完整历史信息，便于回溯的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      y
     </code>
     （Yes），这样终端不会显得杂乱。
    </p>
    <h4>
     <a id="58_Instant_Prompt_Mode_202">
     </a>
     <strong>
      5.8 是否启用即时加载（Instant Prompt Mode）
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Verbose
      </code>
      （推荐）：加速终端加载，并提供调试信息。
      <ul>
       <li>
        启用 Instant Prompt，并在出现问题时提供详细的调试信息。
       </li>
       <li>
        适合 大多数用户，因为它可以加快终端启动速度，同时保证你可以看到潜在的错误信息。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Quiet
      </code>
      ：只启用加速，不显示调试信息。
      <ul>
       <li>
        启用 Instant Prompt，但不会显示调试信息。
       </li>
       <li>
        适合已经熟悉 Instant Prompt 并确信不会有兼容性问题的用户。
       </li>
      </ul>
     </li>
     <li>
      <code>
       Off
      </code>
      ：完全关闭即时加载。
      <ul>
       <li>
        禁用 Instant Prompt，Zsh 将等待所有插件加载完成后再显示提示符。
       </li>
       <li>
        可能会导致终端启动变慢，但适合如果你的 .zshrc 里有不兼容的配置。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      1
     </code>
     （Verbose），这样你可以享受更快的终端启动速度。
    </p>

     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/241f970305ca41bf8f7ab3b3bcc8b876.png"/>
    </p>
    <h4>
     <a id="59__zshrc_217">
     </a>
     <strong>
      5.9 是否应用配置到
      <code>
       ~/.zshrc
      </code>
     </strong>
    </h4>
    <ul>
     <li>
      <code>
       Yes
      </code>
      （推荐）：自动保存 Powerlevel10k 配置。
      <ul>
       <li>
        自动将所有选择的 Powerlevel10k 配置 写入 ~/.zshrc 文件。
       </li>
       <li>
        无需手动修改配置文件，安装后可直接使用。
       </li>
      </ul>
     </li>
     <li>
      <code>
       No
      </code>
      ：手动修改
      <code>
       ~/.zshrc
      </code>
      。
      <ul>
       <li>
        不自动写入 ~/.zshrc，你需要手动编辑 ~/.zshrc 文件来应用你的配置。
       </li>
       <li>
        适合高级用户，或有特定自定义需求的用户。
       </li>
      </ul>
     </li>
    </ul>
    <p>
     <strong>
      推荐：
     </strong>
     选择
     <code>
      y
     </code>
     （Yes），这样你无需手动编辑配置文件。
    </p>
