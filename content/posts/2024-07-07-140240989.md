    <p>
     xmind2testcase是一个非常好用的测试用例工具，可以快速将xmind8文件转换为可导入禅道、testlink的用例文件。无系统限制，安装python3后即可安装工具。
    </p>
    <h2 style="text-align:center;">
     <strong>
      一、安装方式
     </strong>
    </h2>
    <p>
     Windows或Mac系统都可以使用，首先安装python3（安装教程可百度），安装python后使用pip命令或pip3命令安装xmind2testcase。
    </p>
    <pre><code>pip3 install xmind2testcase</code></pre>
    <h2 style="text-align:center;">
     <strong>
      二、版本升级
     </strong>
    </h2>
    <pre><code>pip3 install -U xmind2testcase</code></pre>
    <h2 style="text-align:center;">
     <strong>
      三、使用方式
     </strong>
    </h2>
    <h3>
     方法1、命令行调用
    </h3>
    <pre><code>命令:</code><code> xmind2testcase [path_to_xmind_file] [-csv] [-xml] [-json]</code>
<code>举例:</code><code> xmind2testcase /path/to/testcase.xmind        =&gt; output testcase.csv、testcase.xml、testcase.json</code><code> xmind2testcase /path/to/testcase.xmind -csv   =&gt; output testcase.csv</code><code> xmind2testcase /path/to/testcase.xmind -xml   =&gt; output testcase.xml</code><code> xmind2testcase /path/to/testcase.xmind -json  =&gt; output testcase.json</code></pre>
    <h5>
    </h5>
    <h3>
     方法2、网页操作（推荐）
    </h3>
    <p>
     <strong>
      1、2种启动方式
     </strong>
    </p>
    <ol>
     <li>
      不指定端口（默认端口5001）：
      <code>
       xmind2testcase webtool
      </code>
     </li>
     <li>
      指定端口(如9000)：
      <code>
       xmind2testcase webtool 9000
      </code>
     </li>
    </ol>
    <p>
     <strong>
      2、访问
     </strong>
    </p>
    <ol>
     <li>
      不指定端口，电脑浏览器输入网址：
      <code>
       http://127.0.0.1:5001/
      </code>
      或
      <code>
       http://localhost:5001/
      </code>
     </li>
     <li>
      有指定端口，使用指定端口访问：
      <code>
       http://127.0.0.1:9000/
      </code>
      或
      <code>
       http://localhost:9000/
      </code>
     </li>
    </ol>

    <h4>
     3、具体解析规则
    </h4>
    <p>
    </p>
    <p class="img-center">
     <img alt="图片" height="483" src="https://i-blog.csdnimg.cn/blog_migrate/8534120829c0b7317455f4033d5bc341.png" width="915"/>

    <p>
     <strong>
      注意，注意，注意，重要的事情说三遍！！！
     </strong>
    </p>
    <p>
     第六条规则，我们使用标注
     <strong>
      优先级图标
     </strong>
     作为”测试标题”与”测试步骤”界线，如果解析过程没有遇到
     <strong>
      优先级图标
     </strong>
     ，则TestSuite后面的子主题链作为一条测试用例。
    </p>
    <p>
     一条测试用例支持只有标题，没有测试步骤和预期结果，因为实际测试过程中，我们常常通过用例标题就可以明确测试点了。
    </p>
    <p>
     之所以有第六条规则这样设计，因为实际测试用例设计过程中，我们所测产品往往有非常多的模块和层级。
    </p>
