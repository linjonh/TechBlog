# 解决5：解决狂神说\_Linux配置java环境变量后出现命令不能使用的问题，形如：-bash: ls: command not found

#### 解决5：解决狂神说\_Linux配置java环境变量后出现命令不能使用的问题，形如：-bash: ls: command not found

---

**问题描述：**
  
![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/dcb9b5568dbc13fab0250931cc7d86f1.png)

---

**解决方法一：**

```
export PATH=/bin:/usr/bin:$PATH

```

但是该命令在重启后就会失效，没有从根本解决问题。

---

**解决方法二：**

感觉还是之前更改的配置有问题，打开
`vim /etc/profile`
,以下是之前的配置：

```
JAVA_HOME=/usr/java/jdk1.8.0_221-amd64
CLASSPATH=%JAVA_HOME%/lib;%JAVA_HOME%/jre/lib
PATH=$JAVA_HOME/bin;$JAVA_HOME/jre/bin
export PATH CLASSPATH JAVA_HOME

```

更改为

```
export JAVA_HOME=/usr/java/jdk1.8.0_221-amd64
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH
export PATH CLASSPATH JRE_HOME JAVA_HOME

```

执行
`source /etc/profile`
,之后发现就问题圆满解决。
