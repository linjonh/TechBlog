    <h3 id="背景">
     背景
    </h3>
    <ul>
     <li>
      在最近的工作开发之中，慢慢习惯了很多Java8中的Stream的用法，很方便而且也可以并行的去执行这个流，这边去写一下昨天遇到的一个list转map的场景。
      <br/>
     </li>

    <h3 id="list转map在java8中stream的应用">
     list转map在Java8中stream的应用
    </h3>
    <h4 id="常用方式">
     常用方式
    </h4>
    <p>
     1.利用Collectors.toMap方法进行转换
    </p>
    <pre class="prettyprint"><code class="hljs lasso"><span class="hljs-keyword">public</span> <span class="hljs-built_in">Map</span><span class="hljs-subst">&lt;</span>Long, <span class="hljs-built_in">String</span><span class="hljs-subst">&gt;</span> getIdNameMap(<span class="hljs-built_in">List</span><span class="hljs-subst">&lt;</span>Account<span class="hljs-subst">&gt;</span> accounts) {
    <span class="hljs-keyword">return</span> accounts<span class="hljs-built_in">.</span>stream()<span class="hljs-built_in">.</span>collect(Collectors<span class="hljs-built_in">.</span>toMap(Account<span class="hljs-tag">::getId</span>, Account<span class="hljs-tag">::getUsername</span>));
}</code></pre>
    <p>
     其中第一个参数就是可以，第二个参数就是value的值。
    </p>
    <p>
     2.收集对象实体本身
     <br/>
     - 在开发过程中我们也需要有时候对自己的list中的实体按照其中的一个字段进行分组（比如 id -&gt;List），这时候要设置map的value值是实体本身。
    </p>
    <pre class="prettyprint"><code class="hljs lasso"><span class="hljs-keyword">public</span> <span class="hljs-built_in">Map</span><span class="hljs-subst">&lt;</span>Long, Account<span class="hljs-subst">&gt;</span> getIdAccountMap(<span class="hljs-built_in">List</span><span class="hljs-subst">&lt;</span>Account<span class="hljs-subst">&gt;</span> accounts) {
    <span class="hljs-keyword">return</span> accounts<span class="hljs-built_in">.</span>stream()<span class="hljs-built_in">.</span>collect(Collectors<span class="hljs-built_in">.</span>toMap(Account<span class="hljs-tag">::getId</span>, account <span class="hljs-subst">-&gt; </span>account));
}</code></pre>
    <p>
     account -&gt; account是一个返回本身的lambda表达式，其实还可以使用Function接口中的一个默认方法 Function.identity()，这个方法返回自身对象，更加简洁
    </p>
    <ol>
     <li>
      重复key的情况。
      <br/>
      <ul>
       <li>
        在list转为map时，作为key的值有可能重复，这时候流的处理会抛出个异常：Java.lang.IllegalStateException:Duplicate key。这时候就要在toMap方法中指定当key冲突时key的选择。(这里是选择第二个key覆盖第一个key)
       </li>
      </ul>
     </li>
    </ol>
    <pre class="prettyprint"><code class="hljs coffeescript">public Map&lt;String, Account&gt; getNameAccountMap(List&lt;Account&gt; accounts) {
    <span class="hljs-keyword">return</span> accounts.stream<span class="hljs-function"><span class="hljs-params">()</span>.<span class="hljs-title">collect</span><span class="hljs-params">(Collectors.toMap(Account::getUsername, Function.identity(), (key1, key2) -&gt; key2))</span>;
}</span></code></pre>
    <ol>
     <li>
      用groupingBy 或者 partitioningBy进行分组
      <br/>
      <ul>
       <li>
        根据一个字段或者属性分组也可以直接用groupingBy方法，很方便。
       </li>
      </ul>
     </li>
    </ol>
    <pre class="prettyprint"><code class="hljs avrasm">Map&lt;Integer, List&lt;Person&gt;&gt; personGroups = Stream<span class="hljs-preprocessor">.generate</span>(new PersonSupplier()).
 limit(<span class="hljs-number">100</span>).
 collect(Collectors<span class="hljs-preprocessor">.groupingBy</span>(Person::getAge))<span class="hljs-comment">;</span>
Iterator it = personGroups<span class="hljs-preprocessor">.entrySet</span>()<span class="hljs-preprocessor">.iterator</span>()<span class="hljs-comment">;</span>
while (it<span class="hljs-preprocessor">.hasNext</span>()) {
 Map<span class="hljs-preprocessor">.Entry</span>&lt;Integer, List&lt;Person&gt;&gt; persons = (Map<span class="hljs-preprocessor">.Entry</span>) it<span class="hljs-preprocessor">.next</span>()<span class="hljs-comment">;</span>
 System<span class="hljs-preprocessor">.out</span><span class="hljs-preprocessor">.println</span>(<span class="hljs-string">"Age "</span> + persons<span class="hljs-preprocessor">.getKey</span>() + <span class="hljs-string">" = "</span> + persons<span class="hljs-preprocessor">.getValue</span>()<span class="hljs-preprocessor">.size</span>())<span class="hljs-comment">;</span>
}</code></pre>
    <ul>
     <li>
      partitioningBy可以理解为特殊的groupingBy，key值为true和false，当然此时方法中的参数为一个判断语句（用于判断的函数式接口）
     </li>
    </ul>
    <pre class="prettyprint"><code class="hljs avrasm">Map&lt;Boolean, List&lt;Person&gt;&gt; children = Stream<span class="hljs-preprocessor">.generate</span>(new PersonSupplier()).
 limit(<span class="hljs-number">100</span>).
 collect(Collectors<span class="hljs-preprocessor">.partitioningBy</span>(p -&gt; p<span class="hljs-preprocessor">.getAge</span>() &lt; <span class="hljs-number">18</span>))<span class="hljs-comment">;</span>
System<span class="hljs-preprocessor">.out</span><span class="hljs-preprocessor">.println</span>(<span class="hljs-string">"Children number: "</span> + children<span class="hljs-preprocessor">.get</span>(true)<span class="hljs-preprocessor">.size</span>())<span class="hljs-comment">;</span>
