    <p>
     根据微软CEO史蒂夫·鲍尔默在2013年7月11日的备忘录的内容，微软将实现全局战略，即所谓的“一个微软”。微软的将主要关注与四个工程领域，操作系统(OS)、应用(App)、云计算和设备。具体改动如下：
    </p>
    <p>
     windows desktop和wp还有xbox合并为一个大部门，由硬件业务部门和操作系统部门交叉管理。简而言之目的就是合并生态。
    </p>
    <p>
     将bing，office，skype合并为应用部门。
    </p>
    <p>
     将云计算，azure，还有企业解决方案服务合并云计算和企业部门。
    </p>
    <p>
     当然微软还有设立在全球各地的研究院。下面摘自维基百科：
    </p>
    <ul>
     <li>
      <strong>
       微软雷德蒙德研究院1991年设立于美国
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%8F%AF%E7%9B%9B%E9%A0%93%E5%B7%9E" rel="nofollow noopener noreferrer" target="_blank" title="华盛顿州">
       <strong>
        华盛顿州
       </strong>
      </a>
      <strong>
       微软雷德蒙德园区（Redmond，微软总公司所在地），雷德蒙德研究院常与来自英国剑桥、中国北京、旧金山的研究员合作交流。
      </strong>
     </li>
     <li>
      <a href="http://zh.wikipedia.org/wiki/%E5%BE%AE%E8%BD%AF%E4%BA%9A%E6%B4%B2%E7%A0%94%E7%A9%B6%E9%99%A2" rel="nofollow noopener noreferrer" target="_blank" title="微软亚洲研究院">
       <strong>
        微软亚洲研究院
       </strong>
      </a>
      <strong>
       （MSRA）1998年在北京成立，有上百位来自全球的研究人员与科学家，还有200多位客座研究员与学生支持研究，主要研究改良的
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E4%BD%BF%E7%94%A8%E8%80%85%E4%BB%8B%E9%9D%A2" rel="nofollow noopener noreferrer" target="_blank" title="用户接口">
       <strong>
        用户接口
       </strong>
      </a>
      <strong>
       、网络与
      </strong>
      <a href="http://zh.wikipedia.org/wiki/%E7%84%A1%E7%B7%9A%E9%80%9A%E8%A8%8A" rel="nofollow noopener noreferrer" target="_blank" title="无线通信">
       <strong>
        无线通信
       </strong>
      </a>
      <strong>
       ，和新世代多媒体与亚洲消息处理技术等。
      </strong>
     </li>
     <li>
      <strong>
       微软英国剑桥研究院1997年成立于英国剑桥，有超过百名的研究人员，主要研究机械学习、
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%B3%87%E8%A8%8A%E5%AE%89%E5%85%A8" rel="nofollow noopener noreferrer" target="_blank" title="信息安全">
       <strong>
        信息安全
       </strong>
      </a>
      <strong>
       、
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E8%B3%87%E6%96%99%E6%8E%A2%E5%8B%98" rel="nofollow noopener noreferrer" target="_blank" title="数据挖掘">
       <strong>
        数据挖掘
       </strong>
      </a>
      <strong>
       、
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E4%BD%9C%E6%A5%AD%E7%B3%BB%E7%B5%B1" rel="nofollow noopener noreferrer" target="_blank" title="操作系统">
       <strong>
        操作系统
       </strong>
      </a>
      <strong>
       、编程与网络技术等。微软英国研究所也与
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%8A%8D%E6%A9%8B%E5%A4%A7%E5%AD%B8" rel="nofollow noopener noreferrer" target="_blank" title="剑桥大学">
       <strong>
        剑桥大学
       </strong>
      </a>
      <strong>
       的计算机研究室有密切的联系。
      </strong>
     </li>
     <li>
      <strong>
       微软硅谷研究院2001年成立于
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E7%9F%BD%E8%B0%B7" rel="nofollow noopener noreferrer" target="_blank" title="硅谷">
       <strong>
        硅谷
       </strong>
      </a>
      <strong>
       。研究人员约25人，主要研究
      </strong>
      <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E5%88%86%E5%B8%83%E5%BC%8F%E9%81%8B%E7%AE%97&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" target="_blank" title="分布式运算（页面不存在）">
       <strong>
        分布式运算
       </strong>
      </a>
      <strong>
       ，还有包括信息安全、
      </strong>
      <a class="mw-redirect" href="http://zh.wikipedia.org/wiki/%E5%8F%AF%E9%9D%A0%E6%80%A7" rel="nofollow noopener noreferrer" target="_blank" title="可靠性">
       <strong>
        可靠性
       </strong>
      </a>
      <strong>
       、协议、
      </strong>
      <a class="new" href="http://zh.wikipedia.org/w/index.php?title=%E8%B3%87%E6%BA%90%E5%AE%9A%E5%9D%80%E5%99%A8&amp;action=edit&amp;redlink=1" rel="nofollow noopener noreferrer" target="_blank" title="资源寻址器（页面不存在）">
       <strong>
        资源寻址器
       </strong>
      </a>
      <strong>
       等研究。2006年8月与微软湾区研究中心（BARC）合并。
      </strong>
     </li>
     <li>
      <strong>
       微软印度研究院2005年1月成立于
      </strong>
      <a href="http://zh.wikipedia.org/wiki/%E7%8F%AD%E5%8A%A0%E7%BE%85%E7%88%BE" rel="nofollow noopener noreferrer" target="_blank" title="班加罗尔">
       <strong>
        班加罗尔
       </strong>
      </a>
      <strong>
       ，研究人员约50人，并有大批实习生支持研究。主要再进行多领域的基础研究及应用研究，并与多所学校与研究机构有密切交流
      </strong>
      <p>
       在中国，微软成立了微软亚太研发集团，微软亚洲研究院、微软亚洲工程院、微软（亚洲）互联网工程院、微软亚洲Office事业部、微软亚洲硬件技术中心、微软服务器与开发工具（中国）、微软亚太研发集团战略合作与外包事业部等机构组成。微软亚太研发集团应该属于微软的子公司。（一个集团是公司的子公司。。）
      </p>
      <p>
       最重要的使我们要了解微软为开发者准备的开发资料的分布：
      </p>
      <p>
       <span class="g">
        <span style="color:#008000">
         <a href="http://www.itellyou.cn/" rel="nofollow noopener noreferrer" target="_blank">
          www.
          <strong>
           itellyou
          </strong>
          .cn/
         </a>
         这是一个私人搭建的，为中国人服务的镜像站，主要提供下载链接
        </span>
       </span>
      </p>
      <p>
       <span class="g">
        <a href="https://msdn.microsoft.com/zh-cn/" rel="nofollow noopener noreferrer" target="_blank">
         https://msdn.microsoft.com/zh-cn/
        </a>
        是中文的主要网站是微软提供的中文技术支持网站。
       </span>
      </p>
      <p>
       <span class="g">
        在该网站上，提供技术（最新技术简介），下载，计划（主要是一些培训计划），社区（博客，论坛），还有文档和示例代码
       </span>
       <span class="g">
        。
       </span>
      </p>
      <p>
       <span class="g">
        示例代码可以按照平台，VS版本，贡献者，编程语言，关键技术，主题进行选择，比如说MFC应该是一个关键技术，但是由于比较老，已经找不到相关的选项，可以直接进行搜索。
       </span>
      </p>
      <p>
       <span class="g">
        下面重点来讲解一下文档区
       </span>
      </p>
      <p>
       <span class="g">
        这边可以在将显示格式转换为库目录。普通目录就是把微软认为用户可能感兴趣的部分提取出来，放在表面上（也就是一个桌面的功能），库目录相当于一个文件系统。
       </span>
      </p>
      <p>
       <span class="g">
        假设我们要寻找MFC的相关文档，选择VS2013，选择VC++，选择MFC和ATL，在选择MFC桌面应用程序，就可以获取所有和MFC相关的文档了。如果不选择MFC和ATL，而是选择win32程序的话，就可以访问windows api的相关功能了。
       </span>
      </p>
      <p>
       <span class="g">
        在这里，唯一的遗憾是微软组织好了文档，但是你在搜索时，他只能把你的搜索当成一个关键字而不是领域来搜索，所以你可能会发现不同角度，不同领域的多个搜索结果。需要你人工筛选你究竟想要的参考文档。
       </span>
      </p>
      <p>
       <span class="g">
        进入页面上又链接你可进入微软开发者中心页面，然后可以根据平台和工具来选择具体前往对象，windows选项会将你送至
        <a href="http://dev.windows.com/zh-cn" rel="nofollow noopener noreferrer" target="_blank">
         http://dev.windows.com/zh-cn
        </a>
        页面，该页面包括一些简单的新闻内容，点击该页面上的链接你会回到msdn中。所以你就忽略到整个开发者中心就好了，这不是一种好的索引途径。其他一些选项会将你送至一些页面，最终的有用内容都被包含在msdn的文档区。在开发者中心的页面上也会有实例检索的功能，但是搜索到的实例比msdn搜索到的实例少。
       </span>
      </p>
      <p>
       <span class="g">
        <span style="color:#008000">
         总体而言，微软的网站搭建的很乱，但是msdn中的文档参考资料确实非常完整，搜索起来也非常简单，索引逻辑完全是依照关键字算是清晰。
        </span>
       </span>
      </p>
