    <h3 id="h_709930403_0">
     一、背景
    </h3>
    <p>
     前文已经介绍了利用
     <a href="https://zhida.zhihu.com/search?content_id=245893974&amp;content_type=Article&amp;match_order=1&amp;q=busybox&amp;zhida_source=entity" rel="nofollow" title="busybox">
      busybox
     </a>
     构建一个简易的qemu调试环境：
     <a href="https://zhuanlan.zhihu.com/p/667525514" rel="nofollow" title="无人知晓：qemu搭建arm64 linux kernel环境">
      无人知晓：qemu搭建arm64 linux kernel环境
     </a>
    </p>
    <p>
     但是busybox的功能比较少，难以满足我们的调试和实验需求，在商用产品上也很少使用；本文将介绍如何在qemu下构建一个ubuntu的rootfs（基于arm64平台）。
    </p>
    <h3 id="h_709930403_1">
     二、ubuntu 基础包下载
    </h3>
    <p>
     下载对应版本的base版本：
    </p>
    <p>
     <a href="https://link.zhihu.com/?target=http%3A//cdimage.ubuntu.com/ubuntu-base/releases/" rel="nofollow" title="Index of /ubuntu-base/releases">
      Index of /ubuntu-base/releases
     </a>
    </p>
    <p>
    </p>
    <p class="img-center">
     <img alt="" height="719" src="https://i-blog.csdnimg.cn/img_convert/4f4969e7aca8fb2f91f3829ead6cd28f.jpeg" width="1067"/>
    </p>
    <pre><code>命令行下载ubuntu基础包
wget http://cdimage.ubuntu.com/ubuntu-base/releases/22.04/release/ubuntu-base-22.04-base-arm64.tar.gz</code></pre>
    <h3 id="h_709930403_2">
     三、基于ubuntu基础包构建根文件系统
    </h3>
    <p>
     <strong>
      1、创建img，并将ubuntu-base解压到img
     </strong>
    </p>
    <pre><code>#创建img，4G大小
dd if=/dev/zero of=ubuntu.img bs=1M count=4096 
#将img格式化为ext4格式
mkfs.ext4 ubuntu.img

#将ubuntu-base包解压到img中
mkdir -p rootfs
sudo mount -t ext4 ubuntu.img rootfs/
sudo tar -xzf ubuntu-base-22.04-base-arm64.tar.gz -C rootfs/

#利用qemu-aarch64-static 运行ubuntu_arm64下指令来在X86上完成ubuntu-base-22.04-base-arm64
sudo apt-get install <a href="https://zhida.zhihu.com/search?content_id=245893974&amp;content_type=Article&amp;match_order=1&amp;q=qemu-user-static&amp;zhida_source=entity" rel="nofollow" title="qemu-user-static">qemu-user-static</a></code></pre>
    <p>
     <strong>
      2、切换更新ubuntu-base包的基础环境
     </strong>
    </p>
    <pre><code>#<a href="https://zhida.zhihu.com/search?content_id=245893974&amp;content_type=Article&amp;match_order=1&amp;q=chroot&amp;zhida_source=entity" rel="nofollow" title="chroot">chroot</a> 到rootfs目录，后续你用qemu-user-static更新arm64 依赖软件
sudo ./chroot_mount.sh rootfs/ mount</code></pre>
    <p>
     依赖的chroot_mount.sh 脚本
    </p>
    <pre><code>#!/bin/bash

if [ -z "${1}" ]; then
    echo "Usage: $0 CHROOT_DIR [BRANCH_NAME|umount]"
    exit 1
fi

chrootdir="${1}"
branchname="${2:-}"

if [ ! -d "${chrootdir}" ]; then
    echo "Not dir ${chrootdir}"
    exit 1
fi

if [ "$(df -T "${chrootdir}" | tail -n1 | awk '{print $2}')" = "sffs" ]; then
    echo "Should not use sffs"
    exit 1
fi

function mountfs()
{
    mountpoint -q "${chrootdir}/dev" || mount --bind /dev "${chrootdir}/dev"
    mountpoint -q "${chrootdir}/dev/pts" || mount -t devpts pts "${chrootdir}/dev/pts"
    mountpoint -q "${chrootdir}/proc" || mount -t proc proc "${chrootdir}/proc"
    mountpoint -q "${chrootdir}/sys" || mount -t sysfs sys "${chrootdir}/sys"
}

function umountfs()
{
    mountpoint -q "${chrootdir}/dev/pts" &amp;&amp; umount "${chrootdir}/dev/pts"
    mountpoint -q "${chrootdir}/dev" &amp;&amp; umount "${chrootdir}/dev"
    mountpoint -q "${chrootdir}/proc" &amp;&amp; umount "${chrootdir}/proc"
    mountpoint -q "${chrootdir}/sys" &amp;&amp; umount "${chrootdir}/sys"
}

function chroot_run()
{
    env -i - LC_ALL=C.UTF-8 LANG=en_US.UTF-8 LANGUAGE=en_US: LROOT=/root TERM=${TERM} USER=root TZ=Asia/Shanghai HOME=/root _=/usr/bin/env /usr/sbin/chroot "${chrootdir}" "$@"
}

if [ "${branchname}" = "umount" ]; then
    umountfs
    exit 0
fi

if [ ! -s "${chrootdir}/etc/hosts" ]; then
    cat &gt;"${chrootdir}/etc/hosts" &lt;&lt; EOF
127.0.0.1       localhost
EOF
fi

if [ ! -s "${chrootdir}/etc/resolv.conf" ]; then
    cat &gt;"${chrootdir}/etc/resolv.conf" &lt;&lt; EOF
nameserver 114.114.114.114
nameserver 8.8.8.8
EOF
fi

mountfs

echo "Enter chroot dir ${chrootdir}"
chroot_run bash -c "cd /root; exec bash --login -i"</code></pre>
    <p>
     <strong>
      3、安装必要的软件，添加用户和密码
     </strong>
    </p>
    <pre><code>apt update
apt install udev vim net-tools ssh ethtool ifupdown rsyslog bash-completion nfs-common telnetd sudo inetutils-ping</code></pre>
    <p>
     添加用户和密码，这里添加了一个geek用户名：
    </p>
    <pre><code>#add geek user, and to sudo group
useradd -s '/bin/bash' -m -G adm,sudo geek
passwd geek
#设置geek用户密码
passwd root
#设置root用户密码</code></pre>
    <p>
     <strong>
      4、卸载img
     </strong>
    </p>
    <pre><code>sudo ./chroot_mount.sh rootfs/ umount
sudo umount rootfs</code></pre>
    <p>
     <strong>
      5、配置网络user mode
     </strong>
    </p>
    <p>
     qemu增加网络配置cmdline:
    </p>
    <pre><code>-netdev user,id=eth0,hostfwd=tcp::5022-:22,hostfwd=tcp::5080-:80 -device virtio-net-device,netdev=eth0 \</code></pre>
    <p>
     将22，80 端口分别转发到Host PC的5022口和5080口
    </p>
    <p>
     这里可能会遇到的报错处理：
    </p>
    <pre><code>qemu-system-aarch64: -netdev user,id=eth0,xxxxxx network backend 'user' is not compiled into this binary
原因是自己构建的qemu版本： 默认是没有 "<a href="https://zhida.zhihu.com/search?content_id=245893974&amp;content_type=Article&amp;match_order=1&amp;q=slirp&amp;zhida_source=entity" rel="nofollow" title="slirp">slirp</a>" 模块的
sudo apt-get install libslirp-dev
qemu编译前,
./configure --enable-slirp --target-list=aarch64-softmmu</code></pre>
    <p>
     <strong>
      6、qemu启动命令
     </strong>
    </p>
    <pre><code>/home/geek/workspace/linux/qemu_src/qemu/build/qemu-system-aarch64 \
    -machine virt,virtualization=true,gic-version=3 \
    -nographic \
    -m size=2048M \
    -cpu cortex-a72 \
    -smp 8 \ 
    -kernel Image \
    -netdev user,id=eth0,hostfwd=tcp::5022-:22,hostfwd=tcp::5080-:80 -device virtio-net-device,netdev=eth0 \
    -drive format=raw,file=ubuntu.img \
    -append "noinitrd root=/dev/vda rw nokaslr console=ttyAMA0 loglevel=8 " \
    -s</code></pre>
    <p>
     <strong>
      7、ubuntu网络配置
     </strong>
    </p>
    <p>
     此时登录到ubuntu还有网络连接问题，需要在ubuntu的/etc/network/interfaces中配置（直接编辑文件添加）：
    </p>
    <pre><code>auto eth0
iface eth0 inet dhcp</code></pre>
    <p>
     <strong>
      8、其他
     </strong>
    </p>
    <p>
     ssh 登录：
    </p>
    <pre><code>ssh -p 5022 geek@127.0.0.1
或者
ifconfig查看qemu启动ip地址
ssh -p 5022 geek@192.168.0.106</code></pre>
    <p>
     scp拷贝:
    </p>
    <pre><code>scp -P 5022  geek@127.0.0.1:/home/geek/workspace/test/* ./</code></pre>
