     你的团队为什么成长慢
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    你的团队已经组建了三四个月了，但依旧无法形成战斗力。看到别的公司的团队生机勃勃，创意层出，你是不是心急如焚？你在想是不是有什么灵丹妙药可以立竿见影起死回生。别急，让我们一起找下原因。然后才能对照下药药到病除。
    <br/>
    那到底是什么原因导致团队成长慢呢？分析起来有如下原因：
    <br/>
    1、
    <span style="white-space:pre">
    </span>
    团队中知识源数量不足和质量不佳。
    <br/>
    你的团队毫无疑问新手占了一大部分，这大部分的人该从什么地方获取他们要的知识？如何学习才能把学到的知识及时用到你团队负责项目中？这两个问题直接就决定了你团队提升和创造价值的基础。假如你团队里有一个老师傅带一群新人，但这个老师傅的技能不怎么的，那么带出来的徒弟很遗憾也不会太好。但假如你的团队里的师傅是好的，但数量不足也同样很慢。
    <br/>
    要解决这个问题，就是必须给新团队注入适当数量的技术能力好的熟手。
    <br/>
    2、
    <span style="white-space:pre">
    </span>
    知识传播速度慢
    <br/>
    有了师傅后，如何与新人进行交互、指导也是一个值得重视的问题。师傅的工作重心要适当的往新人的培养上放，不要一味只埋头苦干自己的工作。在敏捷开发过程中，结对编程和松结对编程方法可以使用。这样可以快速的实现知识技能往新人方向的传递。还可以进行定期的代码评审、培训等，营造好的团队学习氛围。
    <br/>
    3、
    <span style="white-space:pre">
    </span>
    没有定期总结，对知识实现定期的系统梳理、回顾和下阶段学习规划，团队提倡学习的氛围不佳，学习动力不足
    <br/>
    适当的总结和系统的梳理对学习的作用不言而喻，在学习中起到画龙点睛的作用。团队中定期总结也可以促进大家学习的积极性，也有部分监督作用。下一阶段的规划可以给团队成员的学习指明方向。让团队成员的学习可以真正的学以致用，促进团队成员的积极性。
    <br/>
    <br/>
    <br/>
    团队的快速提高需要团队中高质量知识的快速传播。确保学以致用和不断改进是刺激团队不断学习新知识积极性的关键。学不能致用可能会导致人才流失。这当然是后话。
