      <p>
       <code>
        @RequestParam
       </code>
       注解共包含四个属性如下所示：
      </p>
      <ol>
       <li>
        name：请求中的参数名，与value等价。
       </li>
       <li>
        value：请求中的参数名，与name等价。
       </li>
       <li>
        required：请求参数是否必须，默认为true。
       </li>
       <li>
        defaultValue：请求参数的默认值，该如果配置该属性会使required属性自动设置为false。
       </li>
      </ol>

    <h2>
     <a id="2__143">
     </a>
     2. 响应数据
    </h2>
    <table>
     <thead>
      <tr>
       <th>
        名称
       </th>
       <th>
        @ResponseBody
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        类型
       </td>
       <td>
        <mark>
         方法\类注解
        </mark>
       </td>
      </tr>
      <tr>
       <td>
        位置
       </td>
       <td>
        SpringMVC控制器方法定义上方和控制类上
       </td>
      </tr>
      <tr>
       <td>
        作用
       </td>
       <td>
        设置当前控制器返回值作为响应体,
        <br/>
        写在类上，该类的所有方法都有该注解功能
       </td>
      </tr>
      <tr>
       <td>
        相关属性
       </td>
       <td>
        pattern：指定日期时间格式字符串
       </td>
      </tr>
     </tbody>
    </table>
    <p>
     <strong>
      说明:
     </strong>
    </p>
    <ul>
     <li>
      该注解可以写在类上或者方法上
     </li>
     <li>
      写在类上就是该类下的所有方法都有
      <code>
       @ReponseBody
      </code>
      功能：
      <code>
       @RestController
      </code>
      =
      <code>
       @Controller
      </code>
      +
      <code>
       @ResponseBody
      </code>
      。
     </li>
     <li>
      当方法上有
      <code>
       @ReponseBody
      </code>
      注解后
      <ul>
       <li>
        方法的返回值为字符串，会将其作为文本内容直接响应给前端
       </li>
       <li>
        方法的返回值为对象，会将对象转换成JSON响应给前端
       </li>
      </ul>
     </li>
    </ul>
    <h2>
     <a id="3_REST_162">
     </a>
     3. REST风格
    </h2>
    <ul>
     <li>
      <p>
       REST（Representational State Transfer）
       <strong>
        表现形式状态转换
       </strong>
       ，它是一种软件架构
       <mark>
        风格
       </mark>
       ，它充分利用HTTP协议的特性，为我们规定了一套统一的资源获取方式，以实现不同终端之间（客户端与服务端）的数据访问与交互。
      </p>
     </li>
     <li>
      <p>
       REST风格：根据REST风格对资源进行访问称为
       <mark>
        RESTful
       </mark>
       ，通过URL定位要操作的资源，通过HTTP动词(请求方式)来描述具体的操作。在REST风格的URL中，通过四种请求方式，来操作数据的增删改查。
      </p>
      <ul>
       <li>
        GET ： 查询
       </li>
       <li>
        POST ：新增
       </li>
       <li>
        PUT ：修改
       </li>
       <li>
        DELETE ：删除
       </li>
      </ul>
     </li>
     <li>
      <p>
       下述示例代码分别展示采用REST风格对用户资源进行增删改查。
      </p>
