    <h2>
     <a id="_VS_Code__GitHub_Copilot_Chat__claude37__0">
     </a>
     解决 VS Code 中 GitHub Copilot Chat 遇到的
     <code>
      claude-3.7
     </code>
     模型不支持问题
    </h2>
    <p>
     最近在 VS Code 中使用 Cline、Roo Code 插件的时候，尝试切换到
     <code>
      claude-3.7
     </code>
     模型时遇到了如下错误：
    </p>
    <pre><code>API 请求失败
请求失败： 400 {"error":{"message":"此请求不支持模型","param":"model","code":"model_not_supported","type":"invalid_request_error"}}
</code></pre>
    <p>
     然而，当选择
     <code>
      claude-3.5
     </code>
     模型时，功能正常。本文将介绍问题的原因以及如何手动解决该问题。
    </p>
    <hr/>
    <h3>
     <a id="_13">
     </a>
     问题描述
    </h3>
    <p>
     Cline、Roo Code 等插件可以使用 VS Code LM API 来工作， VS Code LM API 提供了多种语言模型选项，但我发现，虽然插件中启用了
     <code>
      claude-3.7
     </code>
     模型，但在实际使用时却提示模型不被支持。而其他模型（如
     <code>
      claude-3.5
     </code>
     ）则可以正常使用。
    </p>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/285bdccaa29b418ab8d42f2daa33f15c.png"/>
    </p>
    <hr/>
    <h3>
     <a id="_20">
     </a>
     原因分析
    </h3>
    <p>
     经过进一步调查，这一问题是由 Microsoft 引发的，具体表现为 GitHub Copilot Chat 在发送 API 请求时包含了特定的头信息（
     <code>
      x-onbehalf-extension-id
     </code>
     ），导致 API 服务器拒绝了对
     <code>
      claude-3.7
     </code>
     模型的支持。简单来说，Microsoft 的服务尚未完全准备好支持
     <code>
      claude-3.7
     </code>
     模型。
    </p>
    <hr/>
    <h3>
     <a id="_26">
     </a>
     解决方案
    </h3>
    <p>
     尽管这是由 Microsoft 引发的问题，但我们可以采取一些手动措施来绕过限制，从而启用
     <code>
      claude-3.7
     </code>
     模型。
    </p>
    <p>
     以下是详细步骤：
    </p>
    <h4>
     <a id="_1_32">
     </a>
     <strong>
      步骤 1：找到扩展文件夹
     </strong>
    </h4>
    <h5>
     <a id="Windows_34">
     </a>
     Windows:
    </h5>
    <ol>
     <li>
      打开文件资源管理器并导航到：
      <pre><code>%USERPROFILE%\.vscode\extensions\
</code></pre>
     </li>
     <li>
      查找类似以下名称的文件夹：
      <pre><code>github.copilot-chat-&lt;版本号&gt;
</code></pre>
     </li>
    </ol>
    <h5>
     <a id="macOSLinux_44">
     </a>
     macOS/Linux:
    </h5>
    <ol>
     <li>
      打开终端并导航到：
      <pre><code>~/.vscode/extensions/
</code></pre>
     </li>
     <li>
      找到类似以下名称的文件夹：
      <pre><code>github.copilot-chat-&lt;版本号&gt;
</code></pre>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_2_extensionjs__56">
     </a>
     <strong>
      步骤 2：打开
      <code>
       extension.js
      </code>
      文件
     </strong>
    </h4>
    <ol>
     <li>
      进入文件夹
      <code>
       github.copilot-chat-&lt;版本号&gt;/dist/
      </code>
      。
     </li>
     <li>
      使用你喜欢的文本编辑器（如 VS Code）打开文件
      <code>
       extension.js
      </code>
      。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_3_63">
     </a>
     <strong>
      步骤 3：定位标头代码
     </strong>
    </h4>
    <p>
     使用编辑器的搜索功能（快捷键通常是
     <code>
      Ctrl+F
     </code>
     或
     <code>
      Cmd+F
     </code>
     ）搜索以下内容：
    </p>
    <pre><code>"x-onbehalf-extension-id"
</code></pre>
    <hr/>
    <h4>
     <a id="_4_72">
     </a>
     <strong>
      步骤 4：移除或注释掉标头代码
     </strong>
    </h4>
    <p>
     找到包含以下内容的代码段：
    </p>
    <pre><code class="prism language-javascript"><span class="token constant">S</span> <span class="token operator">===</span> <span class="token string">"getExtraHeaders"</span> <span class="token operator">?</span> <span class="token keyword">function</span> <span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token keyword">return</span> <span class="token punctuation">{<!-- --></span>
        <span class="token operator">...</span>f<span class="token punctuation">.</span>getExtraHeaders<span class="token operator">?.</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token operator">??</span> <span class="token punctuation">{<!-- --></span><span class="token punctuation">}</span><span class="token punctuation">,</span>
        <span class="token string-property property">"x-onbehalf-extension-id"</span><span class="token operator">:</span> <span class="token template-string"><span class="token template-punctuation string">`</span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span><span class="token constant">A</span><span class="token interpolation-punctuation punctuation">}</span></span><span class="token string">/</span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span>c<span class="token interpolation-punctuation punctuation">}</span></span><span class="token template-punctuation string">`</span></span>

</code></pre>
    <h5>
     <a id="__84">
     </a>
     解决方法 ：直接删除标头代码
    </h5>
    <p>
     完全删除以下部分：
    </p>
    <pre><code class="prism language-javascript"><span class="token string-property property">"x-onbehalf-extension-id"</span><span class="token operator">:</span> <span class="token template-string"><span class="token template-punctuation string">`</span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span><span class="token constant">A</span><span class="token interpolation-punctuation punctuation">}</span></span><span class="token string">/</span><span class="token interpolation"><span class="token interpolation-punctuation punctuation">${<!-- --></span>c<span class="token interpolation-punctuation punctuation">}</span></span><span class="token template-punctuation string">`</span></span>

    <p>
     并确保删除逗号以保持语法正确。
    </p>
    <hr/>
    <h4>
     <a id="_5_VS_Code_93">
     </a>
     <strong>
      步骤 5：保存并重启 VS Code
     </strong>
    </h4>
    <ol>
     <li>
      保存对
      <code>
       extension.js
      </code>
      文件的修改。
     </li>
     <li>
      关闭所有正在运行的 VS Code 实例。
     </li>
     <li>
      重新打开 VS Code，以使更改生效。
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="_6_101">
     </a>
     <strong>
      步骤 6：验证更改
     </strong>
    </h4>
    <ol>
     <li>
      打开你常用的代码文件。
     </li>
     <li>
      启动 Cline、Roo Code并尝试使用
      <code>
       claude-3.7
      </code>
      模型。
     </li>
     <li>
      如果一切正常，你应该能够看到模型的响应。
     </li>
    </ol>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/e09ee06ad9664abd87961d069e822b8c.png"/>
    </p>
    <hr/>
    <h3>
     <a id="_112">
     </a>
     注意事项
    </h3>
    <ol>
     <li>
      <strong>
       备份文件
      </strong>
      ：在修改扩展文件之前，请务必备份原始的
      <code>
       extension.js
      </code>
      文件，以免出现问题时无法恢复。
     </li>
     <li>
      <strong>
       更新风险
      </strong>
      ：如果你更新了 GitHub Copilot Chat 插件，你的更改可能会被覆盖。
     </li>
     <li>
      <strong>
       安全性
      </strong>
      ：尽管 GitHub 官方确认这些操作是安全的，但仍然建议仅在了解风险的情况下进行此类修改。
     </li>
    </ol>
    <hr/>
    <h3>
     <a id="_120">
     </a>
     总结
    </h3>
    <p>
     通过手动移除不必要的头信息，我们可以暂时解决 VS Code 中 Cline、Roo Code 对
     <code>
      claude-3.7
     </code>
     模型不支持的问题。希望 Microsoft 能尽快修复此问题，让所有用户都能无缝使用最新模型！
    </p>
