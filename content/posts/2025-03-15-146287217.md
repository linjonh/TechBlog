    <p>
     Zed是当前新流行的一种编辑器，支持MCP（模型上下文协议）
    </p>
    <p>
     Mac下安装Zed比较简单，直接有安装包，在这里：
    </p>
    <pre><code class="hljs">brew install --cask zed
</code></pre>
    <p>
     Mac Monterey下是可以安装上的，亲测有效。
    </p>
    <p>
    </p>
    <h2>
     配置
    </h2>
    <p>
     使用Ctrl+Shift+P调出AI，然后设置使用的模型
    </p>
    <p>
     可以使用deepseek，但是没有找到使用自建服务器的设置方法，有些遗憾。
    </p>
    <p>
    </p>
    <h2>
     附加学习
    </h2>
    <p>
    </p>
    <p>
     关于Zed里面的MCP部分，手册：
     <a href="https://zed.dev/docs/assistant/model-context-protocol?highlight=mcp#model-context-protocol" rel="nofollow" title="Model Context Protocol - Zed">
      Model Context Protocol - Zed
     </a>
    </p>
    <p>
     也就是有个Postgres数据库的MCP Server的扩展，代码在：
     <a href="https://github.com/zed-extensions/postgres-context-server" title="https://github.com/zed-extensions/postgres-context-server">
      https://github.com/zed-extensions/postgres-context-server
     </a>
    </p>
    <p>
     配置和使用，见后面
    </p>
    <h2>
     Zed Postgres Context Server
    </h2>
    <p>
    </p>
    <p>
     This extension provides a Model Context Server for Postgres, for use with the Zed AI assistant.
    </p>
    <p>
     It adds a
     <code>
      /pg-schema
     </code>
     slash command to the Assistant Panel.
    </p>
    <h3>
     Configuration
    </h3>
    <p>
    </p>
    <p>
     To use the extension, you will need to point the context server at a Postgres database by setting the
     <code>
      database_url
     </code>
     in your Zed
     <code>
      settings.json
     </code>
     :
    </p>
    <pre>{
  "context_servers": {
    "postgres-context-server": {
      "settings": {
        "database_url": "postgresql://myuser:mypassword@localhost:5432/mydatabase"
      }
    }
  }
}</pre>
    <h3>
     Usage
    </h3>
    <p>
    </p>
    <ul>
     <li>
      <code>
       /pg-schema &lt;table-name&gt;
      </code>
      : Retrieve the schema for the table with the given name.
     </li>
     <li>
      <code>
       /pg-schema all-tables
      </code>
      : Retrieve the schemas for all tables in the database.
     </li>
    </ul>
    <p>
    </p>
    <p>
     设置参考这里：
     <a href="https://zed.dev/docs/extensions/context-servers" rel="nofollow" title="Context Server Extensions - Zed">
      Context Server Extensions - Zed
     </a>
    </p>
    <p>
    </p>
    <h2 id="model-context-protocol">
     <a href="https://zed.dev/docs/assistant/model-context-protocol?highlight=mcp#model-context-protocol" rel="nofollow" title="Model Context Protocol">
      Model Context Protocol
     </a>
    </h2>
    <p>
     Zed uses the
     <a href="https://modelcontextprotocol.io/" rel="nofollow" title="Model Context Protocol">
      Model Context Protocol
     </a>
     to interact with
     <a href="https://zed.dev/docs/assistant/context-servers.html" rel="nofollow" title="context servers">
      context servers
     </a>
     :
    </p>
    <blockquote>
     <p>
      The Model Context Protocol (MCP) is an open protocol that enables seamless integration between LLM applications and external data sources and tools. Whether you're building an AI-powered IDE, enhancing a chat interface, or creating custom AI workflows, MCP provides a standardized way to connect LLMs with the context they need.
     </p>
    </blockquote>
    <p>
     Check out the
     <a href="https://www.anthropic.com/news/model-context-protocol" rel="nofollow" title="Anthropic news post">
      Anthropic news post
     </a>
     and the
     <a href="https://zed.dev/blog/mcp" rel="nofollow" title="Zed blog post">
      Zed blog post
     </a>
     for an introduction to MCP.
    </p>
    <h3 id="try-it-out">
     <a href="https://zed.dev/docs/assistant/model-context-protocol?highlight=mcp#try-it-out" rel="nofollow" title="Try it out">
      Try it out
     </a>
    </h3>
    <p>
     Want to try it for yourself?
    </p>
    <p>
     The following context servers are available today as Zed extensions:
    </p>
    <ul>
     <li>
      <a href="https://github.com/zed-extensions/postgres-context-server" title="Postgres Context Server">
       Postgres Context Server
      </a>
     </li>
    </ul>
    <h3 id="bring-your-own-context-server">
     <a href="https://zed.dev/docs/assistant/model-context-protocol?highlight=mcp#bring-your-own-context-server" rel="nofollow" title="Bring your own context server">
      Bring your own context server
     </a>
    </h3>
    <p>
     If there's an existing context server you'd like to bring to Zed, check out the
     <a href="https://zed.dev/docs/extensions/context-servers.html" rel="nofollow" title="context server extension docs">
      context server extension docs
     </a>
     for how to make it available as an extension.
    </p>
    <p>
     If you are interested in building your own context server, check out the
     <a href="https://modelcontextprotocol.io/introduction#get-started-with-mcp" rel="nofollow" title="Model Context Protocol docs">
      Model Context Protocol docs
     </a>
     to get started.
    </p>
    <p>
    </p>
    <h2 id="context-servers">
     <a href="https://zed.dev/docs/extensions/context-servers#context-servers" rel="nofollow" title="Context Servers">
      Context Servers
     </a>
    </h2>
    <p>
     Extensions may provide
     <a href="https://zed.dev/docs/assistant/context-servers.html" rel="nofollow" title="context servers">
      context servers
     </a>
     for use in the Assistant.
    </p>
    <h3 id="example-extension">
     <a href="https://zed.dev/docs/extensions/context-servers#example-extension" rel="nofollow" title="Example extension">
      Example extension
     </a>
    </h3>
    <p>
     To see a working example of an extension that provides context servers, check out the
     <a href="https://github.com/zed-extensions/postgres-context-server" title="postgres-context-server extension">
      postgres-context-server extension
     </a>
     .
    </p>
    <p>
     This extension can be
     <a href="https://zed.dev/docs/extensions/developing-extensions.html#developing-an-extension-locally" rel="nofollow" title="installed as a dev extension">
      installed as a dev extension
     </a>
     if you want to try it out for yourself.
    </p>
    <h3 id="defining-context-servers">
     <a href="https://zed.dev/docs/extensions/context-servers#defining-context-servers" rel="nofollow" title="Defining context servers">
      Defining context servers
     </a>
    </h3>
    <p>
     A given extension may provide one or more context servers. Each context server must be registered in the
     <code>
      extension.toml
     </code>
     :
    </p>
    <pre></pre>
    <p>
    </p>
    <p>
     <code>
      [context_servers.my-context-server]
     </code>
    </p>
    <p>
     Then, in the Rust code for your extension, implement the
     <code>
      context_server_command
     </code>
     method on your extension:
    </p>
    <pre></pre>
    <pre></pre>
    <p>
    </p>
    <p>
     <code>
      impl zed::Extension for MyExtension { fn context_server_command( &amp;mut self, context_server_id: &amp;ContextServerId, project: &amp;zed::Project, ) -&gt; Result&lt;zed::Command&gt; { Ok(zed::Command { command: get_path_to_context_server_executable()?, args: get_args_for_context_server()?, env: get_env_for_context_server()?, }) } }
     </code>
    </p>
    <p>
     This method should return the command to start up a context server, along with any arguments or environment variables necessary for it to function.
    </p>
    <p>
     If you need to download the context server from an external source—like GitHub Releases or npm—you can also do this here.
    </p>
