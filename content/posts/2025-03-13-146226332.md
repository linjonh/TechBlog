     数据库(Database) DB
    </p>
    <p id="ud10f1585">
     存储数据的仓库，本质是一个文件系统。按照特定格式将数据存储起来。
    </p>
    <p id="u1a872367">
     数据库管理系统(Database Management System) DBMS
    </p>
    <p id="ubbac499d">
     是一种操纵和管理数据库的大型软件，用于建立、使用和维护数据库
    </p>
    <p id="uc755cac4">
     <img alt="" id="u6217aae8" src="https://i-blog.csdnimg.cn/img_convert/8fbc9104166a2208360c6c85793036e4.png" width="500"/>

    <p id="u6d316b63">
     关系型数据库与非关系型数据库
    </p>
    <p id="u86a2f5f6">
     关系型数据库.
    </p>
    <p id="ue592797b">
     关系型数据库:指采用了关系模型来组织数据的数据库。
    </p>
    <p id="u526e5227">
     关系模型指的就是二维表格模型，而一个关系型数据库就是由二维表及其之间的联系所组成的一个数据组织。
    </p>

    <h3 id="uWkaV">
     一、分类
    </h3>
    <p id="u81d09983">
     根据SQL指令完成的数据库操作的不同，可以将SQL指令分为四类
    </p>
    <ul>
     <li id="u190be89b">
      DDL Data Defintion language 数据库定义语言
     </li>
    </ul>
    <ul>
     <li>
      <ul>
       <li id="uc7b5096e">
        用于完成对数据库对象(数据表,数据库,视图,索引)的创建，删除，修改
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li id="u53bb7700">
      DML Data Manipulation language 数据操作语言
     </li>
    </ul>
    <ul>
     <li>
      <ul>
       <li id="u76391285">
        用于完成对数据表中的数据添加，删除，修改
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li id="u398f84e9">
      DQL Data Query language 数据查询语言
     </li>
    </ul>
    <ul>
     <li>
      <ul>
       <li id="ubb0625f8">
        用于将数据表中的数据查询出来
       </li>
      </ul>
     </li>
    </ul>
    <ul>
     <li id="ua243d754">
      DCL Data Control Laguage 数据控制语言
     </li>
    </ul>
    <ul>
     <li>
      <ul>
       <li id="ua1a6d1f2">
        用于完成事务管理等控制型操作
       </li>
      </ul>
     </li>
    </ul>

    <h3 id="c9niy">

    <pre id="TdB2N"><code>CREATE TABLE 表名称
(
  列名称1 数据类型,
  列名称2 数据类型,
  列名称3 数据类型,

);</code></pre>
    <h4 id="D0Frf">
     4.1字段约束
    </h4>
    <p id="u54ba5ecb">
     字段常见的约束？
    </p>
    <ul>
     <li id="uc7cc1c9d">
      非空约束(not null):限制此列的值必须提供,不能null
     </li>
     <li id="u6bf98271">
      唯一约束(unique):在表中的多条数据,此列的值不能重复
     </li>
     <li id="u870c1096">
      主键约束(primary key):非空+唯一,能够唯一标示数据表中的一条数据.
     </li>
     <li id="u86332d8c">
      外键约束(foreign key):建立不同表之间的关联关系.
     </li>
    </ul>
    <h5 id="UhYBP">
     主键约束
    </h5>
    <p id="ufbf62530">
     主键–就是数据表中的记录中的唯一标识，在一张表中只能有一个主键(主键可以是一个字段,也可以是多个列组合)
    </p>
    <p id="uc21a7f22">
     当一个字段声明为主键之后，添加数据时;
    </p>
    <ul>
     <li id="u70330cd8">
      此字段数据不能为null;
     </li>
     <li id="u6a89770f">
      此字段数据不能重复;
     </li>
    </ul>
    <h4 id="dssTn">
     添加单列主键
    </h4>
    <p id="u37d26063">
     <strong>
      创建表时添加主键约束：
     </strong>
    </p>
    <pre id="M4eDd"><code>create table books(
    book_isbn char(4),
    book_name varchar(20) primary key
);</code></pre>
    <p id="u79b54bf2">
     <strong>
      或者
     </strong>
    </p>
    <pre id="vshSl"><code>create table books(
    book_isbn char(4);
    book_name varchar(20),
    primary key(book_name)
);</code></pre>
    <p id="u72d43186">
     <strong>
      追加主键：
     </strong>
    </p>
    <pre id="PBKtQ"><code>## 创建表时没有添加主键约束
create table books(
    book_isbn char(4),
    book_name varchar(10) 
);

## 创建表之后添加主键约束
alter table books modify book_isbn char(4) primary key;</code></pre>
    <p id="u69e5d29a">
     <strong>
      删除主键：
     </strong>
    </p>
    <pre id="ekppC"><code>alter table books drop primary key;</code></pre>
    <h4 id="JQdhH">
     添加多列主键
    </h4>
    <pre id="uO43k"><code>ALTER TABLE Enrollments
ADD CONSTRAINT PK_Enrollments PRIMARY KEY (StudentID, CourseID);</code></pre>
    <p id="u29d9de48">
     <strong>
      主键自增长：
     </strong>
    </p>
    <p id="u67ff11fb">
     在我们创建一张数据表时，如果数据表中有可以作为主键(列如:学生表的学号,图书表的isbn)我们可以直接设为这个字段为主键,
    </p>
    <p id="ue03082e4">
     当有些数据没有合适的字段作为主键的时候，我们可以定义一个与记录无关的列(ID)作为主键,此数据无具体含义主要作为标识唯一,在mysql中我们可以将此定义为int ，同时设置为
     <strong>
      自动增长
     </strong>
    </p>
    <p id="u45cf8f8a">
     当我们想数据表中新曾一条数据时，无需提供ID列的值，它会自动增长.
    </p>
    <p id="u52dd03e6">
     <strong>
      定义主键自动增长：
     </strong>
    </p>
    <pre id="kcEYw"><code>create table types(
    type_id double primary key auto_increment,
    type_name varchar(20) not null,
    type_reamrk varchar(20)
);</code></pre>
    <p id="u86871a6f">
     <strong>
      联合主键
     </strong>
    </p>
    <p id="uea7fe76a">
     用两个主键包含的字段作为主键，这个组合在数据表中是唯一，且加了主键索引。这两个字段有一个字段不相同就可以区别两个数据，例如：你的订单表里有很多字段，一般情况只要有个订单号bill_no做主键就可以了，但是，现在要求可能会有补 充订单，使用相同的订单号，
     <br/>
     那么这时单独使用订单号就不可以了，因为会有重复。那么你可以再使用个订单序列号bill_seq来 作为区别。把bill_no和bill_seq设成联合主键。即使bill_no相同，bill_seq不同也是可以的。
    </p>
    <pre id="BegMb"><code>create table grades(
    stu_num char(8),
    course_id int,
    score int,
    primary key(stu_num,course_id)

);</code></pre>
    <h5 id="qGfJM">
     外键约束
    </h5>
    <p id="u2007e5af">
     如果一个实体的某个字段指向另一个实体的

     </strong>

     </strong>
    </p>
    <ul>
     <li id="u915577f0">
      笛卡尔积(A集合&amp;B集合):使用A中的每个记录依次关联B中每个记录,笛卡尔集的总数=A总数*B总数
     </li>
     <li id="ub27027cc">
      如果直接执行select … from tableNmae1 inner join tableName2; 会获取两张数据表中的数据集合的笛卡尔积(依次使用tableName1表中的每条数据 去 匹配tableName2的每条数据)
     </li>
    </ul>
    <p id="udaec8a3a">
     <strong>
      内连接条件
     </strong>
    </p>
    <p id="uc862633d">
     两张表同时用inner join连接查询之后产生笛卡尔积数据很多是无意义的,我们如何消除无意义的数据----添加两张进行连接的查询时的条件
    </p>
    <ul>
     <li id="u7a130d2c">
      使用on设置两张表连接查询的匹配条件
     </li>
    </ul>

where students.cid=classes.class_id;
-- 使用ON设置连接查询条件:先判断连接条件是否成立,如果成立两张表的数据进行组合在生成一条结果记录
select * from students 
inner join classes 
on students.cid=classes.class_id;
</code></pre>

     <strong>
      左连接 LEFT JOIN
     </strong>

    <p id="uded5aca8">
     左连接:显示左表中的所有数据,如果在右表中满足条件的数据,则进行匹配;如果右表中不存在匹配数据,则显示为null

    <p id="u40afc30e">
     <strong>
      右连接 RIGHT JOIN

    </p>
    <p id="u38a00c2c">

    </p>

    </p>
    <ul>
     <li id="u86a37943">
      子查询:

    </ul>

select class_id from classes where class_name like 'Java%';

     查询上述结果相反的结果，可以使用 NOT：
    </p>
    <pre id="ADeZZ"><code>SELECT * FROM Persons WHERE LastName NOT BETWEEN 'Adams' AND 'Carter';</code></pre>

     8.13 AS – 别名
    </h5>
    <p id="u93a1d83c">
     通过使用 SQL，可以为列名称和表名称指定别名（Alias），别名使查询程序更易阅读和书写。
    </p>
    <p id="u90eb2e05">
     表别名
    </p>
    <pre id="lV3Bt"><code>SELECT 列名称/(*) FROM 表名称 AS 别名;、

SELECT p.LastName, p.FirstName

     8.14 IN – 锁定多个值
    </h5>
    <p id="uffea7858">
     IN 操作符允许我们在 WHERE 子句中规定多个值。
    </p>
    <pre id="zQDvz"><code>SELECT 列名/(*) FROM 表名称 WHERE 列名称 IN (值1,值2,值3);</code></pre>

    </p>
    <pre id="jsORo"><code>SELECT * FROM Persons WHERE LastName IN ('Adams','Carter');</code></pre>
    <h5 id="kBw8E">
     8.15 分页查询
    </h5>
    <p id="u9057b588">
     当数据表中的记录比较多的时候,如果一次性全部查询出来的显示给用户,用户的可读性/体验性就不太好,因为此我们可以将这些数据分页进行展示.
    </p>
    <pre id="XujQ6"><code>select ... from ... where ... limit param1,param2;</code></pre>
    <ul>
     <li id="ub072fbdc">
      param1 int 获取查询语句的结果的第一条数据的索引(索引从0开始)
     </li>
     <li id="u3f1f99cd">
      param2 int 获取查询语句的结果的条数(如果剩下的数据条数&lt;pram2,则返回剩下的记录)
     </li>
    </ul>
    <p id="u22fddeb4">
     对数据表中的学生信息进行分页显示,总共59条数据,我们每页显示20条
    </p>
    <p id="u5961ec82">
     总记录数: count 59
    </p>
    <p id="uab9994be">
     每页显示 page 20
    </p>
    <p id="u57987326">
     总页数 pageCount = cpimt%page ?count/page:count/page +1;
    </p>
    <pre id="KMEMk"><code># 查询第一页:
select * from stu limit 0,20;
# 查询第二页
select * from stu limit 20,20;
# 查询第三页
select * from stu limit 40,20;
# 如果一张表中,pageNum表示查询的页码,pageSize表示查询每页的条数:                                                                                                                                                       select * from stu limit (pageNum-1)*pageSize,pageSize;</code></pre>
    <h3 id="W3uT4">
     九、VIEW – 视图
    </h3>
    <p id="uea7f3af4">
     在 SQL 中，视图是基于 SQL 语句的结果集的可视化的表。
    </p>
    <p id="ua822aca1">
     视图包含行和列，就像一个真实的表。视图中的字段就是来自一个或多个数据库中的真实的表中的字段。我们可以向视图添加 SQL 函数、WHERE 以及 JOIN 语句，我们也可以提交数据，就像这些来自于某个单一的表。
    </p>
    <pre id="W3idd"><code>CREATE VIEW 视图名 AS
SELECT 列名
FROM 表名
WHERE 查询条件;</code></pre>
    <p id="u6dc4235b">
     视图总是显示最近的数据。每当用户查询视图时，数据库引擎通过使用 SQL 语句来重建数据。
    </p>
    <p id="ua99fb5f1">
     将 Persons 表中住在 Beijing 的人筛选出来创建视图：
    </p>
    <pre id="Q1Or3"><code>create view persons_beijing as
select * from persons where city='Beijing';</code></pre>
    <p id="u1175d85a">
     如果需要更新视图中的列或者其他信息，无需删除，使用 CREATE OR REPLACE VIEW 选项：
    </p>
    <pre id="My4sZ"><code>CREATE OR REPLACE VIEW 视图名 AS
SELECT 列名
FROM 表名
WHERE 查询条件;</code></pre>

    <h5 id="snoZl">
     10.2 delimiter
    </h5>
    <p id="u3ef2ea46">
     在 SQL 中，
     <code>
      DELIMITER
     </code>
     命令用于更改语句结束符（通常是分号
     <code>
      ;
     </code>
     ），以便能够在数据库管理系统中编写包含多个语句的复杂 SQL 脚本。这个功能在编写存储过程、触发器、函数等时特别有用，因为这些结构内可能包含多个 SQL 语句。
    </p>
    <h6 id="UpBZm">
     为什么需要
     <code>
      DELIMITER
     </code>
    </h6>
    <p id="u40015f84">
     默认情况下，SQL 语句以分号（
     <code>
      ;
     </code>
     ）结尾。但是，当我们编写包含多个 SQL 语句的复杂结构（如存储过程、触发器等）时，我们需要一种方式来区分单个 SQL 语句的结尾和整个结构的结尾。此时，
     <code>
      DELIMITER
     </code>
     命令就派上用场了。
    </p>
    <h6 id="R65wV">
     示例
    </h6>
    <p id="uff9aeb88">
     假设我们要创建一个存储过程，其中包含多个 SQL 语句。以下是如何使用
     <code>
      DELIMITER
     </code>
     命令的示例。
    </p>
    <h6 id="wtGjy">
     1. 更改语句结束符
    </h6>
    <p id="u6ede261b">
     首先，告诉 SQL 解释器我们要使用不同的结束符（例如
     <code>
      $$
     </code>
     ）：
    </p>
    <pre id="59c3d1b5"><code>DELIMITER $$</code></pre>
    <h6 id="aycMT">
     2. 编写存储过程
    </h6>
    <p id="u47e445b5">
     在更改结束符之后，我们可以编写存储过程：
    </p>
    <pre id="8f0b6629"><code>CREATE PROCEDURE SampleProcedure()
BEGIN
    DECLARE v INT;
    SET v = 1;
    SELECT v;
END$$</code></pre>
    <h6 id="AjTDW">
     3. 恢复默认结束符
    </h6>
    <p id="ue237c1ba">
     最后，恢复默认的语句结束符（分号
     <code>
      ;
     </code>
     ）：
    </p>
    <pre id="ebaea682"><code>DELIMITER ;</code></pre>
    <h6 id="VCVUL">
     完整示例
    </h6>
    <p id="u571c6005">
     下面是一个完整的示例，用于创建和调用一个简单的存储过程：
    </p>
    <pre id="0c8def89"><code>-- 更改语句结束符为 $$
DELIMITER $$

-- 创建存储过程
CREATE PROCEDURE SampleProcedure()
BEGIN
    DECLARE v INT;
    SET v = 1;
    SELECT v;
END$$

-- 恢复默认语句结束符为 ;
DELIMITER ;

-- 调用存储过程
CALL SampleProcedure();</code></pre>
    <h6 id="ye3Nd">
     使用场景
    </h6>
    <ul>
     <li id="u6ed97b21">
      <strong>
       存储过程
      </strong>
      ：编写和创建包含多个 SQL 语句的存储过程。
     </li>
     <li id="u1865b809">
      <strong>
       触发器
      </strong>
      ：定义包含多个 SQL 语句的触发器。
     </li>
     <li id="ud8c7e8b8">
      <strong>
       函数
      </strong>
      ：编写和创建包含多个 SQL 语句的用户定义函数。
     </li>
    </ul>

    <h3 id="NOT6N">
     十二、触发器
    </h3>
    <p id="u0526ef59">
     触发器（Trigger）是数据库中的一种特殊的存储过程，它在指定的表上执行特定的事件（INSERT、UPDATE 或 DELETE）时自动触发。触发器可以用于执行复杂的业务逻辑、维护数据一致性和完整性。
    </p>
    <h5 id="wkYyc">
     12.1 创建触发器的语法
    </h5>
    <pre id="333eb7e7"><code>CREATE TRIGGER trigger_name
{BEFORE | AFTER} {INSERT | UPDATE | DELETE}
ON table_name
FOR EACH ROW
BEGIN
    -- 触发器的逻辑
END;</code></pre>
    <ul>
     <li id="u85e70de5">
      <code>
       trigger_name
      </code>
      ：触发器的名称。
     </li>
     <li id="ubf954884">
      <code>
       {BEFORE | AFTER}
      </code>
      ：指定触发器是在事件之前（BEFORE）还是之后（AFTER）触发。
     </li>
     <li id="u7ce5e49c">
      <code>
       {INSERT | UPDATE | DELETE}
      </code>
      ：指定触发器的事件类型。
     </li>
     <li id="u0f92ea30">
      <code>
       table_name
      </code>
      ：触发器所作用的表名。
     </li>
     <li id="u5fc96bac">
      <code>
       FOR EACH ROW
      </code>
      ：指定触发器的操作是针对每一行数据。
     </li>
     <li id="u7af34392">
      <code>
       BEGIN ... END
      </code>
      ：触发器的主体部分，包含具体的业务逻辑。
     </li>
    </ul>
    <h5 id="DFwUL">
     示例
    </h5>
    <p id="u9d0f1536">
     假设我们有一个表
     <code>
      employees
     </code>
     ，并希望在插入新记录时自动记录日志到
     <code>
      employee_log
     </code>
     表中。
    </p>
    <h5 id="ecdc5771">
     创建触发器
    </h5>
    <p id="u5ae31df6">
     创建一个在插入新员工记录之前自动记录日志的触发器：
    </p>
    <pre id="e79fb761"><code>DELIMITER $$

CREATE TRIGGER before_employee_insert
BEFORE INSERT ON employees
FOR EACH ROW
BEGIN
    INSERT INTO employee_log (employee_id, action)
    VALUES (NEW.employee_id, 'INSERT');
END$$

DELIMITER ;</code></pre>
    <h5 id="0ea78e42">
     查看日志
    </h5>
    <pre id="c085edf5"><code>log_id  employee_id  action  log_date
------  ------------ ------- -------------------
1       1            INSERT  2023-05-21 12:34:56</code></pre>
    <h5 id="GQC62">
     12.2 触发器的事件类型
    </h5>
    <p id="u74ba94ca">
     触发器可以根据不同的事件类型进行定义：
    </p>
    <ol>
     <li id="ud3b17342">
      <strong>
       INSERT 触发器
      </strong>
      ：在表上执行插入操作时触发。
     </li>
     <li id="u8c4a4e32">
      <strong>
       UPDATE 触发器
      </strong>
      ：在表上执行更新操作时触发。
     </li>
     <li id="u055447ae">
      <strong>
       DELETE 触发器
      </strong>
      ：在表上执行删除操作时触发。
     </li>
    </ol>
    <h5 id="aDdXT">
     BEFORE 和 AFTER
    </h5>
    <ul>
     <li id="u513bb43e">
      <strong>
       BEFORE 触发器
      </strong>
      ：在事件发生之前触发，常用于验证或修改即将插入或更新的数据。
     </li>
     <li id="u02436ef0">
      <strong>
       AFTER 触发器
      </strong>
      ：在事件发生之后触发，常用于日志记录或级联操作。
     </li>
    </ul>
    <h5 id="tkGjo">
     12.3 删除触发器
    </h5>
    <p id="u3e35b6d1">
     如果需要删除已创建的触发器，可以使用
     <code>
      DROP TRIGGER
     </code>
     语句：
    </p>
    <pre id="a389b94a"><code>DROP TRIGGER IF EXISTS trigger_name;</code></pre>
    <p id="u1841903d">
     例如，删除
     <code>
      before_employee_insert
     </code>
     触发器：
    </p>
    <pre id="0c09ed80"><code>DROP TRIGGER IF EXISTS before_employee_insert;</code></pre>
    <p id="u32327533">
     通过上述步骤，您可以在MySQL中创建、使用和管理触发器，以便更好地执行复杂的业务逻辑和维护数据的一致性和完整性。
    </p>
    <h3 id="dRhP3">
     十三、流程控制语句
    </h3>
    <p id="uba6bdefe">
     在 SQL 中，
     <code>
      CASE WHEN THEN
     </code>
     是一个用于条件表达式的语句，可以根据不同的条件返回不同的值。它在查询中非常有用，可以用于创建有条件的计算或显示。
    </p>
    <h5 id="KCNmN">
     13.1 语法
    </h5>
    <p id="ub0e1ab6d">
     <code>
      CASE
     </code>
     语句有两种主要形式：简单
     <code>
      CASE
     </code>
     表达式和搜索
     <code>
      CASE
     </code>
     表达式。
    </p>
    <h6 id="mYVgW">
     简单 CASE 表达式
    </h6>
    <pre id="636c35bb"><code>CASE expression
    WHEN value1 THEN result1
    WHEN value2 THEN result2
    ...
    ELSE resultN
END</code></pre>
    <h5 id="0c0c4b36">
     搜索 CASE 表达式
    </h5>
    <pre id="79e4f042"><code>CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    ...
    ELSE resultN
END</code></pre>
    <h5 id="LdHiD">
     13.2 示例
    </h5>
    <h5 id="417a0c2c-1">
     简单 CASE 表达式
    </h5>
    <p id="ua410e13a">
     假设我们有一个
     <code>
      students
     </code>
     表，包含学生的 ID 和他们的成绩。我们想要创建一个查询，根据成绩显示相应的等级（A, B, C, D, F）。
    </p>
    <pre id="67e6fba1"><code>CREATE TABLE students (
    student_id INT,
    score INT
);

INSERT INTO students (student_id, score) VALUES
(1, 95),
(2, 85),
(3, 75),
(4, 65),
(5, 55);</code></pre>
    <p id="u8936052f">
     使用简单
     <code>
      CASE
     </code>
     表达式：
    </p>
    <pre id="f9fc8d32"><code>SELECT student_id, score,
CASE score
    WHEN 90 THEN 'A'
    WHEN 80 THEN 'B'
    WHEN 70 THEN 'C'
    WHEN 60 THEN 'D'
    ELSE 'F'
END AS grade
FROM students;</code></pre>
    <h5 id="0c0c4b36-1">
     搜索 CASE 表达式
    </h5>
    <p id="ucf445996">
     使用搜索
     <code>
      CASE
     </code>
     表达式，我们可以处理更复杂的条件。例如，我们可以根据不同的分数范围来计算等级：
    </p>
    <pre id="dda0150b"><code>SELECT student_id, score,
CASE
    WHEN score &gt;= 90 THEN 'A'
    WHEN score &gt;= 80 THEN 'B'
    WHEN score &gt;= 70 THEN 'C'
    WHEN score &gt;= 60 THEN 'D'
    ELSE 'F'
END AS grade
FROM students;</code></pre>
    <h5 id="toyoM">
     示例解释
    </h5>
    <ol>
     <li id="ud1b71683">
      <strong>
       简单 CASE 表达式
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="u22226f27">
        通过
        <code>
         CASE score
        </code>
        ，检查
        <code>
         score
        </code>
        的值是否匹配
        <code>
         WHEN
        </code>
        子句中的某个值。
       </li>
       <li id="u3919f42a">
        如果匹配，则返回对应的
        <code>
         THEN
        </code>
        结果。
       </li>
       <li id="udc83ad54">
        如果没有匹配任何
        <code>
         WHEN
        </code>
        子句，返回
        <code>
         ELSE
        </code>
        结果。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="u97e83605">
      <strong>
       搜索 CASE 表达式
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="u69cbc68a">
        通过
        <code>
         CASE
        </code>
        关键字，依次检查
        <code>
         WHEN
        </code>
        子句中的条件。
       </li>
       <li id="u57eda1ef">
        如果某个条件为
        <code>
         TRUE
        </code>
        ，则返回对应的
        <code>
         THEN
        </code>
        结果。
       </li>
       <li id="u4bda6eff">
        如果没有任何条件为
        <code>
         TRUE
        </code>
        ，返回
        <code>
         ELSE
        </code>
        结果。
       </li>
      </ul>
     </li>
    </ul>

    </ul>
    <p id="u10897fc4">
     流程控制语句在数据库存储过程中非常重要，用于控制SQL语句的执行顺序和条件。常见的流程控制语句包括条件语句（IF...THEN）、循环语句（LOOP, WHILE, REPEAT）和跳转语句（LEAVE, ITERATE）。这些语句使得存储过程可以实现更复杂的逻辑。
    </p>
    <h4 id="0aae9e97">
     条件语句
    </h4>
    <h5 id="IF...THEN...ELSE">
     IF...THEN...ELSE
    </h5>
    <p id="uf400239a">
     用于根据条件执行不同的代码块。
    </p>
    <pre id="43753de6"><code>IF condition THEN
    -- statements
ELSE
    -- statements
END IF;</code></pre>
    <h5 id="1a63ac23">
     示例
    </h5>
    <pre id="ba114310"><code>DELIMITER $$

CREATE PROCEDURE check_salary (IN employee_id INT, OUT result VARCHAR(50))
BEGIN
    DECLARE emp_salary DECIMAL(10, 2);

    -- 获取员工的工资
    SELECT salary INTO emp_salary
    FROM employees
    WHERE employee_id = employee_id;

    -- 检查工资
    IF emp_salary &gt; 100000 THEN
        SET result = 'High salary';
    ELSE
        SET result = 'Normal salary';
    END IF;
END$$

DELIMITER ;</code></pre>
    <h4 id="CASE">
     CASE
    </h4>
    <p id="u288f0bf1">
     用于根据多个条件之一执行代码块。
    </p>
    <pre id="15e0ade9"><code>CASE
    WHEN condition1 THEN
        -- statements
    WHEN condition2 THEN
        -- statements
    ELSE
        -- statements
END CASE;</code></pre>
    <h5 id="1a63ac23-1">
     示例
    </h5>
    <pre id="942b16bb"><code>DELIMITER $$

CREATE PROCEDURE check_position (IN employee_id INT, OUT position_desc VARCHAR(100))
BEGIN
    DECLARE emp_position VARCHAR(100);

    -- 获取员工的职位
    SELECT position INTO emp_position
    FROM employees
    WHERE employee_id = employee_id;

    -- 检查职位
    CASE emp_position
        WHEN 'Manager' THEN
            SET position_desc = 'Manages the team';
        WHEN 'Developer' THEN
            SET position_desc = 'Writes code';
        ELSE
            SET position_desc = 'Other position';
    END CASE;
END$$

DELIMITER ;</code></pre>
    <p id="uc6b9ab73">
    </p>
    <h4 id="97928077">
     循环语句
    </h4>
    <h5 id="LOOP">
     LOOP
    </h5>
    <p id="uf08a5749">
     无限循环，需要使用
     <code>
      LEAVE
     </code>
     语句跳出循环。
    </p>
    <pre id="04f464f8"><code>[loop_label:] LOOP
    -- statements
    LEAVE loop_label;
END LOOP loop_label;</code></pre>
    <h5 id="1a63ac23-2">
     示例
    </h5>
    <pre id="fa029f6a"><code>DELIMITER $$

CREATE PROCEDURE loop_example ()
BEGIN
    DECLARE i INT DEFAULT 0;
    loop_label: LOOP
        SET i = i + 1;
        IF i &gt;= 10 THEN
            LEAVE loop_label;
        END IF;
    END LOOP loop_label;
END$$

DELIMITER ;</code></pre>
    <h5 id="WHILE">
     WHILE
    </h5>
    <p id="u31be87d9">
     基于条件的循环，在每次迭代前检查条件。
    </p>
    <pre id="d279bd27"><code>WHILE condition DO
    -- statements
END WHILE;</code></pre>
    <h5 id="1a63ac23-3">
     示例
    </h5>
    <pre id="bf68b1f3"><code>DELIMITER $$

CREATE PROCEDURE while_example ()
BEGIN
    DECLARE i INT DEFAULT 0;
    WHILE i &lt; 10 DO
        SET i = i + 1;
    END WHILE;
END$$

DELIMITER ;</code></pre>
    <h5 id="REPEAT">
     REPEAT
    </h5>
    <p id="ud2747563">
     基于条件的循环，在每次迭代后检查条件。
    </p>
    <pre id="def0703f"><code>[repeat_label:] REPEAT
    -- statements
UNTIL condition
END REPEAT repeat_label;</code></pre>
    <h5 id="1a63ac23-4">
     示例
    </h5>
    <pre id="41e435b3"><code>DELIMITER $$

CREATE PROCEDURE repeat_example ()
BEGIN
    DECLARE i INT DEFAULT 0;
    repeat_label: REPEAT
        SET i = i + 1;
    UNTIL i &gt;= 10
    END REPEAT repeat_label;
END$$

DELIMITER ;</code></pre>
    <h4 id="e51202e3">
     跳转语句
    </h4>
    <h5 id="LEAVE">
     LEAVE
    </h5>
    <p id="uc867323e">
     用于跳出循环。
    </p>
    <pre id="983f6d59"><code>LEAVE label;</code></pre>
    <h5 id="ITERATE">
     ITERATE
    </h5>
    <p id="uf7f6916d">
     用于跳过当前迭代并开始下一次迭代。
    </p>
    <pre id="3da36b1f"><code>ITERATE label;</code></pre>
    <h5 id="1a63ac23-5">
     示例
    </h5>
    <pre id="32d952b8"><code>DELIMITER $$

CREATE PROCEDURE iterate_example ()
BEGIN
    DECLARE i INT DEFAULT 0;
    loop_label: LOOP
        SET i = i + 1;
        IF i &lt; 5 THEN
            ITERATE loop_label;
        END IF;
        IF i &gt;= 10 THEN
            LEAVE loop_label;
        END IF;
    END LOOP loop_label;
END$$

DELIMITER ;</code></pre>
    <h4 id="8ac1c821">
     综合示例
    </h4>
    <p id="u3c8b1c60">
     结合条件语句和循环语句的综合示例：
    </p>
    <pre id="a5ebc2c0"><code>DELIMITER $$

CREATE PROCEDURE comprehensive_example (IN max_count INT)
BEGIN
    DECLARE i INT DEFAULT 0;
    DECLARE result VARCHAR(100) DEFAULT '';
    
    WHILE i &lt; max_count DO
        SET i = i + 1;
        CASE
            WHEN i MOD 2 = 0 THEN
                SET result = CONCAT(result, 'Even, ');
            ELSE
                SET result = CONCAT(result, 'Odd, ');
        END CASE;
    END WHILE;
    
    SELECT TRIM(TRAILING ', ' FROM result) AS result;
END$$

DELIMITER ;</code></pre>

     结论

    <p id="u257b65fb">
     通过使用这些流程控制语句，您可以在 MySQL 中创建功能强大且灵活的存储过程。这些语句允许您在数据库层实现复杂的业务逻辑，从而提高应用程序的性能和安全性。

    <h3 id="bJRwM">
     十四、存储过程
    </h3>
    <h4 id="e860651e">
     14.1 什么是存储过程
    </h4>
    <p id="uba289e54">
     存储过程（Stored Procedure）是一组预编译的 SQL 语句，这些语句在数据库中以一个单元存储，并可以在需要时被调用执行。存储过程通常用于封装复杂的业务逻辑、执行重复的操作、提高性能、确保数据一致性和安全性。
    </p>
    <h4 id="eb681904">
     14.2 存储过程的优点
    </h4>
    <ol>
     <li id="u3ad064ac">
      <strong>
       提高性能
      </strong>
      ：存储过程在数据库服务器端执行，减少了客户端和服务器之间的数据传输，提高了执行效率。
     </li>
     <li id="u1f555d7e">
      <strong>
       封装业务逻辑
      </strong>
      ：将复杂的业务逻辑封装在存储过程中，使得应用程序更简洁、维护更方便。
     </li>
     <li id="uea31b805">
      <strong>
       代码复用
      </strong>
      ：一次编写，可以多次调用，减少了代码的重复。
     </li>
     <li id="ue6d5ead0">
      <strong>
       安全性
      </strong>
      ：通过授予对存储过程的执行权限，可以控制对底层数据的访问，增强了数据的安全性。
     </li>
     <li id="u1ba7b6e7">
      <strong>
       减少网络流量
      </strong>
      ：存储过程在服务器端执行，只返回最终结果，减少了网络传输的数据量。
     </li>
    </ol>
    <h4 id="fe201e4e">
     14.3 存储过程的结构
    </h4>
    <p id="u6720b2fa">
     存储过程的基本结构包括过程名、参数列表、过程体（包含SQL语句）和返回值。存储过程可以接受输入参数（IN）、输出参数（OUT）或输入输出参数（INOUT）。
    </p>
    <h4 id="3854d0b0">
     14.4 存储过程的语法
    </h4>
    <pre id="0a99b33d"><code>CREATE PROCEDURE procedure_name (parameter_list)
BEGIN
    -- Procedure body
    -- SQL statements
END;</code></pre>
    <ul>
     <li id="u0fcbb8e6">
      <code>
       procedure_name
      </code>
      ：存储过程的名称。
     </li>
     <li id="uf2ba8089">
      <code>
       parameter_list
      </code>
      ：存储过程的参数列表，格式为
      <code>
       IN
      </code>
      、
      <code>
       OUT
      </code>
      或
      <code>
       INOUT
      </code>
      参数。
     </li>
     <li id="u21d9a4f2">
      <code>
       BEGIN ... END
      </code>
      ：存储过程的主体部分，包含具体的 SQL 逻辑。
     </li>
    </ul>
    <h4 id="rcdqX">
     14.5 示例
    </h4>
    <h5 id="jKyQV">
     表结构
    </h5>
    <p id="ud26af9df">
     假设我们有一个员工表
     <code>
      employees
     </code>
     和一个日志表
     <code>
      employee_log
     </code>
     。
    </p>
    <p id="u2d5c056e">
    </p>
    <pre id="66872324"><code>CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(100),
    salary DECIMAL(10, 2)
);

CREATE TABLE employee_log (
    log_id INT AUTO_INCREMENT PRIMARY KEY,
    employee_id INT,
    action VARCHAR(50),
    log_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);</code></pre>
    <h5 id="850987e8">
     创建存储过程
    </h5>
    <p id="u0b47d1b6">
     创建一个名为
     <code>
      update_salary
     </code>
     的存储过程，用于更新员工的工资并记录日志。
    </p>
    <pre id="675be291"><code>DELIMITER $$

CREATE PROCEDURE update_salary (
    IN p_employee_id INT,
    IN p_new_salary DECIMAL(10, 2)
)
BEGIN
    -- 更新员工工资
    UPDATE employees
    SET salary = p_new_salary
    WHERE employee_id = p_employee_id;
    
    -- 记录日志
    INSERT INTO employee_log (employee_id, action)
    VALUES (p_employee_id, 'UPDATE SALARY');
END$$

DELIMITER ;</code></pre>
    <h5 id="GcUww">
     解释
    </h5>
    <ol>
     <li id="u5a82b15c">
      <code>
       <strong>
        DELIMITER $$
       </strong>
      </code>
      ：更改语句结束符为
      <code>
       $$
      </code>
      ，以便可以包含多个语句。
     </li>
     <li id="uf20805b4">
      <code>
       <strong>
        CREATE PROCEDURE update_salary
       </strong>
      </code>
      ：定义一个名为
      <code>
       update_salary
      </code>
      的存储过程，包含两个输入参数
      <code>
       p_employee_id
      </code>
      和
      <code>
       p_new_salary
      </code>
      。
     </li>
     <li id="u404679d6">
      <code>
       <strong>
        BEGIN ... END
       </strong>
      </code>
      ：存储过程的主体部分，首先执行
      <code>
       UPDATE
      </code>
      语句更新员工的工资，然后执行
      <code>
       INSERT
      </code>
      语句将更新操作记录到
      <code>
       employee_log
      </code>
      表中。
     </li>
     <li id="u420ecd28">
      <code>
       <strong>
        DELIMITER ;
       </strong>
      </code>
      ：恢复默认语句结束符
      <code>
       ;
      </code>
      。
     </li>
    </ol>
    <h5 id="c8abaad1">
     调用存储过程
    </h5>
    <p id="u09b5ab2e">
     使用
     <code>
      CALL
     </code>
     语句调用存储过程：
    </p>
    <pre id="d5fa1cdb"><code>CALL update_salary(1, 95000.00);</code></pre>
    <h4 id="c7e344c4">
     14.6 使用 IN、OUT 和 INOUT 参数
    </h4>
    <p id="u1d75546a">
     以下示例展示了如何使用
     <code>
      IN
     </code>
     、
     <code>
      OUT
     </code>
     和
     <code>
      INOUT
     </code>
     参数：
    </p>
    <pre id="bda3c6e8"><code>DELIMITER $$

CREATE PROCEDURE example_procedure (
    IN in_param INT,
    OUT out_param INT,
    INOUT inout_param INT
)
BEGIN
    SET out_param = in_param * 2;
    SET inout_param = inout_param + in_param;
END$$

DELIMITER ;</code></pre>
    <p id="u8e169709">
     调用存储过程并获取输出参数：
    </p>
    <pre id="6b3a2ef5"><code>SET @out_value = 0;
SET @inout_value = 5;
CALL example_procedure(10, @out_value, @inout_value);
SELECT @out_value, @inout_value;</code></pre>
    <h4 id="2f8bd5ec">
     14.7 删除存储过程
    </h4>
    <p id="u1517b446">
     如果需要删除已创建的存储过程，可以使用
     <code>
      DROP PROCEDURE
     </code>
     语句：
    </p>
    <pre id="94989a5c"><code>DROP PROCEDURE IF EXISTS update_salary;</code></pre>

     14.8 注意事项
    </h4>
    <ol>
     <li id="u66c794b8">
      <strong>
       权限
      </strong>
      ：创建和执行存储过程需要特定的权限，如果没有权限，可以联系数据库管理员。
     </li>
     <li id="u79a3ea6c">
      <strong>
       调试
      </strong>
      ：调试存储过程可能会比较困难，可以通过日志记录或使用调试工具来辅助调试。
     </li>
     <li id="uc7e2b7b6">
      <strong>
       性能
      </strong>
      ：存储过程可以提高性能，因为它们在数据库服务器上编译并执行，减少了网络传输和客户端处理的开销。
     </li>
    </ol>
    <p id="uf91afaf7">
     通过上述步骤，您可以在 MySQL 中创建、使用和管理存储过程，以便更好地封装业务逻辑、提高性能和维护数据的一致性和完整性。

    <h3 id="LgtE2">
     十五、索引
    </h3>
    <h4 id="30927593">
     什么是索引
    </h4>
    <p id="u46dc0548">
     索引（Index）是数据库中的一种数据结构，用于快速查询和检索数据库表中的数据。索引通过对表中的一个或多个列进行排序和组织，使得查询操作的效率大大提高。索引类似于书的目录，可以帮助你快速找到所需的信息。
    </p>
    <h4 id="c433af88">
     索引的类型
    </h4>
    <ol>
     <li id="u5e527824">
      <strong>
       主键索引（Primary Key Index）
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="ue933f1c9">
        自动创建在定义主键时。
       </li>
       <li id="ub985fe0a">
        唯一性和非空性。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="ua5c65dbc">
      <strong>
       唯一索引（Unique Index）
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="ueacd0340">
        确保索引列中的所有值都是唯一的。
       </li>
       <li id="u4fbc9648">
        允许列包含空值（NULL）。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="ube312134">
      <strong>
       普通索引（Non-Unique Index）
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="u3110186a">
        提高查询效率，不要求唯一性。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="u72390256">
      <strong>
       全文索引（Full-Text Index）
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="uc4a19052">
        用于全文搜索，如在文本字段中查找特定单词或短语。
       </li>
       <li id="u6a32942d">
        仅在支持的存储引擎（如 MyISAM）上使用。
       </li>
      </ul>
     </li>
    </ul>
    <ol>
     <li id="u847b2512">
      <strong>
       组合索引（Composite Index）
      </strong>
      ：
     </li>
    </ol>
    <ul>
     <li>
      <ul>
       <li id="u378b8792">
        包含多个列的索引。
       </li>
       <li id="u1436c204">
        提高多列查询的效率。
       </li>
      </ul>
     </li>
    </ul>
    <h4 id="de4e5908">
     创建索引
    </h4>
    <h5 id="df1f8b21">
     创建主键索引
    </h5>
    <p id="u20f22f34">
     主键索引在创建主键时自动创建：
    </p>
    <pre id="3c8bc1ad"><code>CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(100),
    salary DECIMAL(10, 2)
);</code></pre>
    <h5 id="6b67e840">
     创建唯一索引
    </h5>
    <p id="u893e5176">
     创建唯一索引确保列值唯一：
    </p>
    <pre id="6f2286e0"><code>CREATE UNIQUE INDEX idx_unique_name ON employees (name);</code></pre>
    <p id="u8820d708">
     或者在创建表时定义唯一约束：
    </p>
    <pre id="efdc5c42"><code>CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    name VARCHAR(100) UNIQUE,
    position VARCHAR(100),
    salary DECIMAL(10, 2)
);</code></pre>
    <h5 id="781183da">
     创建普通索引
    </h5>
    <p id="u855a4ea6">
     创建普通索引提高查询效率：
    </p>
    <pre id="af416e2a"><code>CREATE INDEX idx_position ON employees (position);</code></pre>
    <h5 id="735ea731">
     创建全文索引
    </h5>
    <p id="u3a679c2d">
     创建全文索引用于全文搜索：
    </p>
    <pre id="527c4c9a"><code>CREATE FULLTEXT INDEX idx_fulltext_name ON employees (name);</code></pre>
    <h5 id="2c5a5336">
     创建组合索引
    </h5>
    <p id="ua70a01fc">
     创建组合索引用于多列查询：
    </p>
    <pre id="5ebd43a5"><code>CREATE INDEX idx_name_position ON employees (name, position);</code></pre>
    <h4 id="592d9b36">
     使用索引
    </h4>
    <h5 id="24b1d7db">
     查询使用索引
    </h5>
    <p id="u5002e770">
     使用索引进行查询：
    </p>
    <pre id="62ed2694"><code>SELECT * FROM employees WHERE name = 'John Doe';</code></pre>
    <p id="ua3082b92">
     使用组合索引进行多列查询：
    </p>
    <pre id="eef0dc33"><code>SELECT * FROM employees WHERE name = 'John Doe' AND position = 'Manager';</code></pre>
    <h4 id="00d3d13d">
     查看索引
    </h4>
    <p id="ua21b9375">
     查看表的索引：
    </p>
    <pre id="0134f071"><code>SHOW INDEX FROM employees;</code></pre>
    <h4 id="af9dfbc0">
     删除索引
    </h4>
    <p id="ub22f4ffd">
     删除索引：
    </p>
    <pre id="397e5f6b"><code>DROP INDEX idx_position ON employees;</code></pre>
    <h4 id="8e3e9729">
     索引的注意事项
    </h4>
    <ol>
     <li id="uab0072b4">
      <strong>
       空间消耗
      </strong>
      ：索引会占用额外的存储空间。
     </li>
     <li id="u4b763f44">
      <strong>
       维护成本
      </strong>
      ：插入、更新和删除操作可能需要更新索引，增加了维护成本。
     </li>
     <li id="u62cdbcea">
      <strong>
       合理使用
      </strong>
      ：在需要频繁查询的列上创建索引，不要在更新频繁的列上过多使用索引。
     </li>
    </ol>
    <h4 id="9dbde6c2">
     示例：索引的创建与使用
    </h4>
    <h5 id="fd03fde1">
     示例表结构
    </h5>
    <p id="u973b77b8">
     创建一个简单的
     <code>
      students
     </code>
     表：
    </p>
    <pre id="921e12d3"><code>CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(100),
    age INT,
    email VARCHAR(100),
    major VARCHAR(100)
);</code></pre>
    <h5 id="de4e5908-1">
     创建索引
    </h5>
    <p id="u02602b17">
     为
     <code>
      email
     </code>
     列创建唯一索引：
    </p>
    <pre id="316f9108"><code>CREATE UNIQUE INDEX idx_unique_email ON students (email);</code></pre>
    <p id="u19d63937">
     为
     <code>
      major
     </code>
     列创建普通索引：
    </p>
    <pre id="e8bfc01c"><code>CREATE INDEX idx_major ON students (major);</code></pre>
    <p id="ud48dcb99">
     为
     <code>
      name
     </code>
     和
     <code>
      age
     </code>
     列创建组合索引：
    </p>
    <pre id="ffb57980"><code>CREATE INDEX idx_name_age ON students (name, age);</code></pre>
    <h5 id="2ff06ba4">
     使用索引进行查询
    </h5>
    <p id="uc71c175a">
     查询使用
     <code>
      email
     </code>
     列上的唯一索引：
    </p>
    <pre id="4a10edc6"><code>SELECT * FROM students WHERE email = 'john.doe@example.com';</code></pre>
    <p id="u24a91487">
     查询使用
     <code>
      major
     </code>
     列上的普通索引：
    </p>
    <pre id="a80a6f33"><code>SELECT * FROM students WHERE major = 'Computer Science';</code></pre>
    <p id="u0b052cbf">
     查询使用组合索引：
    </p>
    <pre id="818e2260"><code>SELECT * FROM students WHERE name = 'John Doe' AND age = 21;</code></pre>
    <h4 id="25f9c7fa">
     总结
    </h4>
    <p id="u56a4c881">
     索引是数据库优化的关键工具，可以显著提高查询性能。然而，索引也有其成本，包括占用额外的存储空间和增加维护开销。因此，在设计数据库时，应根据实际需求合理创建和使用索引，以平衡查询性能和系统开销。
    </p>
    <h3 id="cCIS8">
     十五、索引失效的情况
    </h3>
    <p id="u690aaa41">
     索引是提高数据库查询性能的重要工具，但在某些情况下，索引可能会失效，导致查询性能下降。了解索引失效的情况有助于更好地设计和优化数据库。以下是一些常见的索引失效情况：
    </p>
    <h4 id="39ecc1ce">
     1. 使用函数或表达式
    </h4>
    <p id="u6499335e">
     在查询中对索引列使用函数或表达式会导致索引失效，因为数据库无法直接使用索引进行查找。
    </p>
    <h5 id="FC7jD">
     示例
    </h5>
    <pre id="56b8f72d"><code>SELECT * FROM employees WHERE UPPER(name) = 'JOHN DOE';</code></pre>
    <p id="ue6dc7237">
     这种情况可以通过使用函数索引来解决（如果数据库支持）或在应用程序层处理函数调用。
    </p>
    <h4 id="06313a4e">
     2. 隐式类型转换
    </h4>
    <p id="u63319e2b">
     如果索引列的数据类型与查询条件的数据类型不一致，数据库可能会进行隐式类型转换，导致索引失效。
    </p>
    <h5 id="cyhaI">
     示例
    </h5>
    <pre id="e9cfaef9"><code>SELECT * FROM employees WHERE employee_id = '123';</code></pre>
    <p id="u69578462">
     这种情况可以通过确保查询条件的数据类型与索引列的数据类型一致来解决。
    </p>
    <h4 id="07714268">
     3. 使用不等于操作符（&lt;&gt;）
    </h4>
    <p id="ucfbe9ae4">
     在查询中使用不等于操作符（&lt;&gt; 或 !=）会导致索引失效，因为数据库无法使用索引进行范围查找。
    </p>
    <h5 id="meVme">
     示例
    </h5>
    <pre id="070ff4e5"><code>SELECT * FROM employees WHERE salary &lt;&gt; 50000;</code></pre>
    <h4 id="508f29ae">
     4. 使用 IS NULL 或 IS NOT NULL
    </h4>
    <p id="u18a8d59e">
     在查询中使用 IS NULL 或 IS NOT NULL 可能导致索引失效，因为 NULL 值通常不会被索引。
    </p>
    <h5 id="AcxIP">
     示例
    </h5>
    <pre id="0e183c68"><code>SELECT * FROM employees WHERE salary IS NULL;</code></pre>
    <h4 id="889ce368">
     5. 使用 LIKE 模式匹配
    </h4>
    <p id="ubea9f8fe">
     在查询中使用 LIKE 模式匹配时，如果模式以通配符（% 或 _）开头，索引将失效。
    </p>
    <h5 id="Hz1cJ">
     示例
    </h5>
    <pre id="25f8df57"><code>SELECT * FROM employees WHERE name LIKE '%John';</code></pre>
    <p id="u8bc6d47b">
     这种情况可以通过避免在模式开头使用通配符来解决。
    </p>
    <h4 id="81725776">
     6. 使用 OR 条件
    </h4>
    <p id="u4431ef82">
     在查询中使用 OR 条件，如果 OR 条件中的某一列没有索引，则索引会失效。
    </p>
    <h5 id="ggVQc">
     示例
    </h5>
    <pre id="433fa3f7"><code>SELECT * FROM employees WHERE employee_id = 1 OR salary = 50000;</code></pre>
    <p id="u0cdd76be">
     这种情况可以通过使用联合索引或将 OR 条件拆分为多个查询来解决。
    </p>
    <h4 id="72973668">
     7. 不满足最左前缀原则（对于复合索引）
    </h4>
    <p id="ucc92e516">
     对于复合索引，查询条件必须包含最左前缀，否则索引将失效。
    </p>
    <h5 id="1a63ac23-6">
     示例
    </h5>
    <pre id="ea7fc238"><code>CREATE INDEX idx_name_position ON employees (name, position);

-- 仅使用position列，索引失效
SELECT * FROM employees WHERE position = 'Manager';

-- 使用name和position列，索引有效
SELECT * FROM employees WHERE name = 'John Doe' AND position = 'Manager';</code></pre>
    <h4 id="30cb8160">
     8. 使用低选择性列
    </h4>
    <p id="ud0a22e07">
     在选择性较低的列上创建索引可能导致索引失效，因为索引的效果有限。选择性是指列中不同值的数量与总行数的比率。
    </p>
    <h5 id="1a63ac23-7">
     示例
    </h5>
    <pre id="2df8395c"><code>SELECT * FROM employees WHERE gender = 'M';</code></pre>
    <h4 id="16a4dffc">
     9. 查询数据量过大
    </h4>
    <p id="ued643265">
     如果查询返回的数据量占表中总数据量的大部分，数据库可能会选择进行全表扫描而不是使用索引。
    </p>
    <h5 id="1a63ac23-8">
     示例
    </h5>
    <pre id="ad3a0696"><code>SELECT * FROM employees WHERE salary &gt; 1000;</code></pre>
    <h4 id="c115df0f">
     10. 更新频繁的列
    </h4>
    <p id="u0db27096">
     在频繁更新的列上创建索引可能会导致索引失效，因为索引维护成本高，影响查询性能。
    </p>
    <h4 id="be52abda">
     示例汇总
    </h4>
    <p id="u0c1f1d57">
     以下是一个综合示例，展示了各种索引失效的情况：
    </p>
    <h5 id="dsj7a">
     示例表结构
    </h5>
    <pre id="6729b47a"><code>CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(100),
    salary DECIMAL(10, 2),
    gender CHAR(1),
    hire_date DATE
);

CREATE INDEX idx_name_position ON employees (name, position);
CREATE INDEX idx_salary ON employees (salary);
CREATE INDEX idx_hire_date ON employees (hire_date);</code></pre>
    <h5 id="38a8b0ea">
     示例查询
    </h5>
    <pre id="a144d7e7"><code>-- 使用函数，索引失效
SELECT * FROM employees WHERE UPPER(name) = 'JOHN DOE';

-- 隐式类型转换，索引失效
SELECT * FROM employees WHERE employee_id = '123';

-- 使用不等于操作符，索引失效
SELECT * FROM employees WHERE salary &lt;&gt; 50000;

-- 使用 IS NULL，索引失效
SELECT * FROM employees WHERE salary IS NULL;

-- LIKE 模式匹配，索引失效
SELECT * FROM employees WHERE name LIKE '%John';

-- OR 条件，索引失效
SELECT * FROM employees WHERE employee_id = 1 OR salary = 50000;

-- 不满足最左前缀原则，索引失效
SELECT * FROM employees WHERE position = 'Manager';

-- 低选择性列，索引失效
SELECT * FROM employees WHERE gender = 'M';

-- 查询数据量过大，索引失效
SELECT * FROM employees WHERE salary &gt; 1000;

-- 更新频繁的列，索引失效（假设 salary 是更新频繁的列）
UPDATE employees SET salary = salary + 1000 WHERE employee_id = 1;</code></pre>
    <h4 id="czk6p">
     总结
    </h4>
    <p id="u56fe8576">
     索引是数据库优化的重要工具，但在某些情况下可能会失效。理解这些索引失效的情况可以帮助你更好地设计和优化数据库结构和查询，提高数据库的性能。
    </p>
