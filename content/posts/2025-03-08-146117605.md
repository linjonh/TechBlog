    <h3>
     一、什么是交换机？
    </h3>
    <p>
     交换机（Switch）是局域网（LAN）中的核心设备，负责在
     <strong>
      数据链路层
     </strong>
     （OSI第二层）高效转发数据帧。它像一位“智能交通警察”，根据设备的
     <strong>
      MAC地址
     </strong>
     精准引导数据流向，避免网络拥堵。
    </p>
    <h4>
     对比其他设备：
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        集线器（Hub）
       </strong>
       ：无脑广播所有数据，导致冲突和低效（已淘汰）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        路由器（Router）
       </strong>
       ：在
       <strong>
        网络层
       </strong>
       （OSI第三层）跨网络转发数据，基于IP地址寻址。
       <img alt="" src="https://i-blog.csdnimg.cn/direct/c70ee9917a264100935505a389f0283a.png"/>
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     二、交换机的作用与核心功能
    </h3>
    <h4>
     1. 核心作用
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        智能转发
       </strong>
       ：仅将数据发送给目标设备，而非广播。
      </p>
     </li>
     <li>
      <p>
       <strong>
        隔离冲突域
       </strong>
       ：每个端口独立冲突域，减少数据碰撞。
      </p>
     </li>
     <li>
      <p>
       <strong>
        提升带宽利用率
       </strong>
       ：全双工通信，支持多设备同时收发数据。
      </p>
     </li>
    </ul>
    <h4>
     2. 三大核心功能
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        学习（Learning）
       </strong>
       ：记录设备的MAC地址与端口的映射关系，构建
       <strong>
        MAC地址表
       </strong>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        转发（Forwarding）
       </strong>
       ：根据MAC地址表精准转发数据帧。
      </p>
     </li>
     <li>
      <p>
       <strong>
        泛洪（Flooding）
       </strong>
       ：若目标MAC未知，向所有端口广播（除来源端口）。
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     三、交换机的工作原理
    </h3>
    <h4>
     1. MAC地址表的学习过程
    </h4>
    <ol>
     <li>
      <p>
       <strong>
        初始状态
       </strong>
       ：交换机的MAC地址表为空。
      </p>
     </li>
     <li>
      <p>
       <strong>
        接收数据帧
       </strong>
       ：当PC1向PC2发送数据时，交换机会记录：
      </p>
      <ul>
       <li>
        <p>
         <strong>
          源MAC
         </strong>
         （PC1的MAC） →
         <strong>
          端口1
         </strong>
         。
        </p>
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        持续更新
       </strong>
       ：随着更多设备通信，表项逐步完善。
      </p>
     </li>
    </ol>
    <h4>
     2. 数据帧转发规则
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        已知目标MAC
       </strong>
       ：直接转发到对应端口。
      </p>
     </li>
     <li>
      <p>
       <strong>
        未知目标MAC
       </strong>
       ：泛洪到所有端口。
      </p>
     </li>
     <li>
      <p>
       <strong>
        目标MAC为广播地址
       </strong>
       （如
       <code>
        FF-FF-FF-FF-FF-FF
       </code>
       ）：泛洪
       <img alt="" src="https://i-blog.csdnimg.cn/direct/10d63b1393514f8488beac9801d0218b.png"/>
      </p>
     </li>
    </ul>
    <hr/>
    <h3>
     四、交换机的类型
    </h3>
    <h4>
     1. 根据网络层级划分
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        类型
       </th>
       <th>
        工作层级
       </th>
       <th>
        功能特点
       </th>
       <th>
        应用场景
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <strong>
         二层交换机
        </strong>
       </td>
       <td>
        数据链路层
       </td>
       <td>
        基于MAC地址转发，不支持VLAN
       </td>
       <td>
        小型局域网
       </td>
      </tr>
      <tr>
       <td>
        <strong>
         三层交换机
        </strong>
       </td>
       <td>
        网络层
       </td>
       <td>
        支持IP路由、VLAN间通信
       </td>
       <td>
        企业级核心网络
       </td>
      </tr>
     </tbody>
    </table>
    <h4>
     2. 根据管理方式划分
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        非网管交换机
       </strong>
       ：即插即用，无法配置（适合家庭网络）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        网管交换机
       </strong>
       ：支持VLAN、QoS、端口镜像等高级功能（企业级网络）
       <img alt="" src="https://i-blog.csdnimg.cn/direct/74bf518d305d496d94f5c18ca557c252.png"/>
      </p>
     </li>
    </ul>
