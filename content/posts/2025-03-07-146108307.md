    <h4>
     <a id="SHA256_0">
     </a>
     SHA-256简介
    </h4>
    <p>
     SHA-256 是 **SHA-2（Secure Hash Algorithm 2）**家族中的一种哈希算法，由美国国家安全局设计，并于 2001 年发布。它能够将任意长度的数据映射为一个固定长度256 位，即 32 字节的哈希值，通常以 64 个十六进制字符的形式表示。SHA-256 被广泛用于数据完整性验证、数字签名、区块链等领。
    </p>
    <h4>
     <a id="SHA256__4">
     </a>
     SHA-256 的特点
    </h4>
    <ol>
     <li>
      <strong>
       固定输出长度
      </strong>
      ：
      <ul>
       <li>
        SHA-256 生成的哈希值始终是 256 位（32 字节）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       不可逆性
      </strong>
      ：
      <ul>
       <li>
        从哈希值无法反推出原始数据。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       雪崩效应
      </strong>
      ：
      <ul>
       <li>
        输入数据的微小变化会导致输出哈希值的巨大变化。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       抗碰撞性
      </strong>
      ：
      <ul>
       <li>
        设计上极难找到两个不同的输入生成相同的哈希值。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       高效性
      </strong>
      ：
      <ul>
       <li>
        计算速度快，适合处理大量数据。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       安全性
      </strong>
      ：
      <ul>
       <li>
        目前尚未发现有效的攻击方法，被广泛认为是安全的。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="SHA256__21">
     </a>
     SHA-256 的应用场景
    </h4>
    <ol>
     <li>
      <strong>
       数据完整性验证
      </strong>
      ：
      <ul>
       <li>
        通过比较文件的 SHA-256 值，可以验证文件是否被篡改。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       数字签名
      </strong>
      ：
      <ul>
       <li>
        用于生成数据的唯一标识，确保数据的真实性和完整性。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       区块链
      </strong>
      ：
      <ul>
       <li>
        比特币和其他加密货币使用 SHA-256 进行区块哈希计算。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       密码存储
      </strong>
      ：
      <ul>
       <li>
        用于存储用户密码的哈希值（通常与盐值结合使用）。
       </li>
      </ul>
     </li>
     <li>
      <strong>
       证书和密钥
      </strong>
      ：
      <ul>
       <li>
        用于生成 SSL/TLS 证书和密钥。
       </li>
      </ul>
     </li>
    </ol>
    <hr/>
    <h4>
     <a id="SHA256__36">
     </a>
     SHA-256 的工作原理
    </h4>
    <p>
     SHA-256 的计算过程可以分为以下几个步骤：
    </p>
    <h5>
     <a id="1__40">
     </a>
     1.
     <strong>
      填充数据
     </strong>
    </h5>
    <ul>
     <li>
      将输入数据的长度填充至 512 位（64 字节）的倍数。
     </li>
     <li>
      填充方式：在数据末尾添加一个
      <code>
       1
      </code>
      ，然后添加若干个
      <code>
       0
      </code>
      ，最后添加一个 64 位的整数，表示原始数据的长度。
     </li>
    </ul>
    <h5>
     <a id="2__45">
     </a>
     2.
     <strong>
      初始化哈希值
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       SHA-256 使用 8 个 32 位的状态变量（A, B, C, D, E, F, G, H），初始值为以下常量：
      </p>
      <pre><code class="prism language-ini">A = 0x6A09E667
B = 0xBB67AE85
C = 0x3C6EF372
D = 0xA54FF53A
E = 0x510E527F
F = 0x9B05688C
G = 0x1F83D9AB
H = 0x5BE0CD19
</code></pre>
     </li>
    </ul>
    <h5>
     <a id="3__60">
     </a>
     3.
     <strong>
      分块处理
     </strong>
    </h5>
    <ul>
     <li>
      将填充后的数据分割成若干个 512 位（64 字节）的块。
     </li>
     <li>
      对每个块进行 64 轮处理。
     </li>
    </ul>
    <h5>
     <a id="4_64__65">
     </a>
     4.
     <strong>
      64 轮变换
     </strong>
    </h5>
    <ul>
     <li>
      每轮使用以下非线性函数和常量：
      <ul>
       <li>
        <strong>
         Ch
        </strong>
        :
        <code>
         Ch(x, y, z) = (x &amp; y) ^ (~x &amp; z)
        </code>
       </li>
       <li>
        <strong>
         Maj
        </strong>
        :
        <code>
         Maj(x, y, z) = (x &amp; y) ^ (x &amp; z) ^ (y &amp; z)
        </code>
       </li>
       <li>
        <strong>
         Σ0
        </strong>
        :
        <code>
         Σ0(x) = (x &gt;&gt;&gt; 2) ^ (x &gt;&gt;&gt; 13) ^ (x &gt;&gt;&gt; 22)
        </code>
       </li>
       <li>
        <strong>
         Σ1
        </strong>
        :
        <code>
         Σ1(x) = (x &gt;&gt;&gt; 6) ^ (x &gt;&gt;&gt; 11) ^ (x &gt;&gt;&gt; 25)
        </code>
       </li>
       <li>
        <strong>
         常量表
        </strong>
        ：使用 64 个预定义的常量值。
       </li>
      </ul>
     </li>
    </ul>
    <h5>
     <a id="5__74">
     </a>
     5.
     <strong>
      更新哈希值
     </strong>
    </h5>
    <ul>
     <li>
      每轮处理后，更新状态变量（A, B, C, D, E, F, G, H）。
     </li>
    </ul>
    <h5>
     <a id="6__78">
     </a>
     6.
     <strong>
      输出哈希值
     </strong>
    </h5>
    <ul>
     <li>
      将最终的 8 个状态变量（A, B, C, D, E, F, G, H）按大端序拼接，生成 256 位的哈希值。
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="SHA256__84">
     </a>
     SHA-256 的伪代码
    </h4>
    <p>
     以下是 SHA-256 算法的简化伪代码：
    </p>
    <pre><code class="prism language-plaintext">Input: 数据 data
Output: 256 位的哈希值

1. 填充数据：
   - 在 data 末尾添加 1 和若干个 0，使其长度 ≡ 448 mod 512
   - 在末尾添加 64 位的原始数据长度

2. 初始化哈希值：
   A = 0x6A09E667
   B = 0xBB67AE85
   C = 0x3C6EF372
   D = 0xA54FF53A
   E = 0x510E527F
   F = 0x9B05688C
   G = 0x1F83D9AB
   H = 0x5BE0CD19

3. 对每个 512 位块进行处理：
   - 将块分割为 16 个 32 位字
   - 扩展为 64 个 32 位字
   - 进行 64 轮变换

4. 输出哈希值：
   - 将 A, B, C, D, E, F, G, H 按大端序拼接
   - 转换为 64 个十六进制字符
</code></pre>

    <h4>
     <a id="SHA256__168">
     </a>
     SHA-256 的优缺点
    </h4>
    <h5>
     <a id="_170">
     </a>
     优点：
    </h5>
    <ol>
     <li>
      <strong>
       安全性高
      </strong>
      ：抗碰撞性强，尚未发现有效的攻击方法。
     </li>
     <li>
      <strong>
       固定输出长度
      </strong>
      ：便于存储和比较。
     </li>
     <li>
      <strong>
       高效性
      </strong>
      ：计算速度快，适合处理大量数据。
     </li>
    </ol>
    <h5>
     <a id="_176">
     </a>
     缺点：
    </h5>
    <ol>
     <li>
      <strong>
       计算资源消耗较大
      </strong>
      ：相比 MD5，SHA-256 的计算开销更大。
     </li>
     <li>
      <strong>
       不适合密码存储
      </strong>
      ：虽然安全，但通常需要与盐值结合使用。
     </li>
