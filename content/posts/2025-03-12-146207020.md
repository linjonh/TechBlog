    <p>
     DNS ​
     <strong>
      默认使用UDP协议
     </strong>
     ​（端口53）进行通信，但在以下场景中会切换到
     <strong>
      TCP协议
     </strong>
     ​（端口53）：
    </p>
    <h4>
     ​
     <strong>
      1. 响应数据过大（超过512字节）​
     </strong>
    </h4>
    <ul>
     <li>
      ​
      <strong>
       UDP限制
      </strong>
      ：DNS的UDP协议默认限制单个数据包大小为
      <strong>
       512字节
      </strong>
      ​。如果响应数据（如包含多条记录的查询结果）超过此限制，服务器会强制使用TCP。
     </li>
     <li>
      ​
      <strong>
       机制
      </strong>
      ：
      <ul>
       <li>
        当UDP响应被截断（Truncated）时，客户端会重新发起TCP请求。
       </li>
       <li>
        例如：查询包含大量记录的域（如大型企业的子域名列表）。
       </li>
      </ul>
     </li>
    </ul>

    <h4>
     ​
     <strong>
      2. 区域传输（Zone Transfer）​
     </strong>
    </h4>
    <ul>
     <li>
      ​
      <strong>
       全量传输（AXFR）和增量传输（IXFR）​
      </strong>
      ：主从DNS服务器之间的区域传输
      <strong>
       必须使用TCP
      </strong>
      。
     </li>
     <li>
      ​
      <strong>
       原因
      </strong>
      ：
      <ul>
       <li>
        区域文件通常较大（可能包含数千条记录），需要可靠传输。
       </li>
       <li>
        TCP的流控、重传机制保证数据完整性。
       </li>
      </ul>
     </li>
    </ul>
    <h4>
    </h4>
    <h4>
     <strong>
      3. 高可靠性场景
     </strong>
    </h4>
    <ul>
     <li>
      ​
      <strong>
       TCP的重传机制
      </strong>
      ：在不可靠网络环境中，若UDP请求多次失败（如丢包），客户端可能改用TCP重试。
     </li>
     <li>
      ​
      <strong>
       应用场景
      </strong>
      ：关键业务域名解析（如金融、政务系统）要求高可靠性。
     </li>
    </ul>
