    <h4>
     <a id="_sse_1">
     </a>
     问题: sse接口重复执行
    </h4>
    <blockquote>
     <p>
      对接
      <strong>
       Deepseek
      </strong>
      时，使用@microsoft/fetch-event-source 联调SSE接口
      <br/>
      后端同事提出个奇怪的问题，
      <strong>
       接口老是重复执行
      </strong>
      ，我又没有复现，经过来回测试，
      <br/>
      终于复现了，
      <strong>
       切换浏览器页签
      </strong>
      时，就会多一个sse接口调用，如下图：
     </p>
    </blockquote>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/f5185aa7d16b420e9540e5bb0d0553af.png"/>
    </p>
    <h4>
     <a id="_8">
     </a>
     分析：
    </h4>
    <ol>
     <li>
      <p>
       调用栈Initator里不是我们的代码引起的，而是@microsoft/fetch-event-source库，由此可见是该库的
       <strong>
        某配置
       </strong>
       或者sse事件的
       <strong>
        特性
       </strong>
      </p>
     </li>
     <li>
      <p>
       查阅@microsoft/fetch-event-source库的配置项参数，发现有
       <strong>
        openWhenHidden
       </strong>
      </p>
     </li>
    </ol>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/b6e916efcb324c85bb18f933143c143b.png">
      <br/>
      3. 尝试配置为true，代码生效，再次切换浏览器页签，也不重复调用
     </img>
    </p>
    <h4>
     <a id="_17">
     </a>
     溯源找原因
    </h4>
    <ol>
     <li>
      查阅
      <a href="https://developer.mozilla.org/zh-CN/docs/Web/API/EventSource" rel="nofollow">
       mdn EventSource
      </a>
      资料，并未发现有openWhenHidden配置
     </li>
    </ol>
    <p>
     <img alt="" src="https://i-blog.csdnimg.cn/direct/afd9beed82f2480cb0f0be75512e1c52.png">
      <br/>
      2. 查阅
      <a href="https://github.com/Azure/fetch-event-source/blob/main/src/fetch.ts">
       @microsoft/fetch-event-source
      </a>
      库源码，发现是绑定了
      <strong>
       visibilitychange
      </strong>
      事件触发的
     </img>
    </p>
