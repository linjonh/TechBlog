    <h2>
     <a id="_7">
     </a>
     一：四大基本概念
    </h2>
    <h3>
     <a id="1Data_8">
     </a>
     （1）数据(Data)
    </h3>
    <p>
     <strong>
      <font color="ff0000">
       概念
      </font>
      ：是数据库中存储的基本对象，是描述事物的符号记录
     </strong>
    </p>
    <p>
     <strong>
      <font color="ff0000">
       种类
      </font>
      ：文本、图形、图像、音频、视频、学生的档案记录、货物的运输情况等
     </strong>
    </p>
    <p>
     <strong>
      <font color="ff0000">
       特点
      </font>
      ：数据与其语义是不可分的
     </strong>
    </p>
    <h3>
     <a id="2DataBaseDB_15">
     </a>
     （2）数据库(DataBase,DB)
    </h3>
    <p>
     <strong>
      <font color="ff0000">
       概念
      </font>
      ：是永久存储在
      <font color="0000ff">
       计算机内，有组织、可共享的
      </font>
      大量数据的集合
     </strong>
    </p>
    <p>
     <strong>
      <font color="ff0000">
       特征
      </font>
      ：
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       数据按一定的数据模型组织、描述和储存
      </strong>
     </li>
     <li>
      <strong>
       可为各种用户共享
      </strong>
     </li>
     <li>
      <strong>
       冗余度较小
      </strong>
     </li>
     <li>
      <strong>
       数据独立性较高
      </strong>
     </li>
     <li>
      <strong>
       易扩展
      </strong>
     </li>
    </ul>
    <h3>
     <a id="3DataBase_Management_SystemDBMS_27">
     </a>
     （3）数据库管理系统(DataBase Management System,DBMS)
    </h3>
    <p>
     MYSQL就是一张DBMS
    </p>
    <hr/>
    <p>
     <strong>
      <font color="ff0000">
       概念
      </font>
      ：是位于
      <font color="0000ff">
       用户与操作系统之间
      </font>
      的一层数据管理软件。和操作系统一样是计算机的基础软件
     </strong>
    </p>
    <ul>
     <li>
      数据库是一个仓库，但是对数据库的操作是依靠DBMS完成的
     </li>
    </ul>
    <p>
     <strong>
      <font color="ff0000">
       功能
      </font>
      ：
     </strong>
    </p>
    <p>
     <strong>
      1：数据定义功能
     </strong>
    </p>
    <ul>
     <li>
      提供
      <strong>
       数据定义语言(DDL)
      </strong>
     </li>
     <li>
      定义数据库中的数据对象
     </li>
    </ul>
    <p>
     <strong>
      2：数据组织、存储和管理功能
     </strong>
    </p>
    <ul>
     <li>
      分类组织、存储和管理各种数据
     </li>
     <li>
      确定组织数据的文件结构和存取方式
     </li>
     <li>
      实现数据之间的联系
     </li>
     <li>
      提供多种存取方法提高存取效率
     </li>
    </ul>
    <p>
     <strong>
      3：数据操作功能
     </strong>
    </p>
    <ul>
     <li>
      提供
      <strong>
       数据操纵语言(DML)
      </strong>
     </li>
     <li>
      实现对数据库的基本操作 (查询、插入、删除和修改)
     </li>
    </ul>
    <p>
     <strong>
      4：数据库的事物管理和运行管理
     </strong>
    </p>
    <ul>
     <li>
      数据库在建立、运行和维护时由DBMS统一管理和控制
     </li>
     <li>
      保证数据的安全性、完整性、多用户对数据的并发使用
     </li>
     <li>
      发生故障后的系统恢复
     </li>
    </ul>
    <p>
     <strong>
      5：数据库的建立和维护功能（实用程序）
     </strong>
    </p>
    <ul>
     <li>
      数据库初始数据装载转换
     </li>
     <li>
      数据库转储
     </li>
     <li>
      介质故障恢复
     </li>
     <li>
      数据库的重组织
     </li>
     <li>
      性能监视分析等
     </li>
    </ul>
    <p>
     其他功能（如通信、数据转换、互访等）
    </p>
    <h3>
     <a id="4Database_SystemDBS_74">
     </a>
     （4）数据库系统(Database System，DBS)
    </h3>
    <p>
     <strong>
      <font color="ff0000">
       概念
      </font>
      ：在计算后的系统构成
     </strong>
    </p>
    <p>
     <strong>
      <font color="ff0000">
       组成
      </font>
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       数据库 Database
      </strong>
     </li>
     <li>
      <strong>
       数据库管理系统（及其开发工具）Database Management System
      </strong>
     </li>
     <li>
      <strong>
       应用系统(app)
      </strong>
     </li>
     <li>
      <strong>
       数据库管理员 Database Administrator(DBA)
      </strong>
     </li>
    </ul>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/blog_migrate/94a552f4f8a1b84aeec8015375c23d14.png">
      <br/>
      <strong>
       <font color="ff0000">
        特点
       </font>
       ：
      </strong>
      <br/>
      <strong>
       1：数据结构化（数据库系统与文件系统的本质区别）
      </strong>
     </img>
    </p>
    <ul>
     <li>
      <strong>
       整体结构化
      </strong>
      是数据库的主要特征之一
     </li>
     <li>
      所谓整体结构化就是指它不针对某一个应用，而是面向全组织。不仅数据
      <strong>
       内部结构化并且数据之间也具有连续
      </strong>
     </li>
     <li>
      数据的结构用
      <strong>
       数据模型描述
      </strong>
      ，无需程序定义和解释；数据可以变长；数据的最小存取单位是
      <strong>
       数据项
      </strong>
     </li>
    </ul>
    <p>
     <strong>
      2：数据的共享性高，冗余度低，易扩充
     </strong>
    </p>
    <p>
     <strong>
      3：数据独立性高（由DBMS的二级映像功能保证）
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       物理独立性
      </strong>
      ：用户的应用程序与存储在磁盘上的数据库中数据是独立的。当数据的物理存储改变，应用程序不改变
     </li>
     <li>
      <strong>
       逻辑独立性
      </strong>
      ：指用户的应用程序与数据库的逻辑结构是相互独立的。数据的逻辑结构改变了，用户程序也可以不变
     </li>
    </ul>
    <p>
     <strong>
      4：数据由DBMS统一管理和控制
     </strong>
    </p>
    <ul>
     <li>
      <strong>
       数据的安全性（Security）保护
      </strong>
      ：保护数据，以防止不合法的使用造成的数据的泄密和破坏。
     </li>
     <li>
      <strong>
       数据的完整性（Integrity）检查
      </strong>
      ：将数据控制在有效的范围内，或保证数据之间满足一定的关系。
     </li>
     <li>
      <strong>
       并发（Concurrency）控制
      </strong>
      ：对多用户的并发操作加以控制和协调，防止相互干扰而得到错误的结果。
     </li>
     <li>
      <strong>
       数据库恢复（Recovery）
      </strong>
      ：将数据库从错误状态恢复到某一已知的正确状态。
     </li>
    </ul>
    <h2>
     <a id="_109">
     </a>
     二：数据库发展阶段
    </h2>
    <p>
     <strong>
      <font color="0000ff">
       人工管理阶段(20世纪40年代中–50年代中)
      </font>
     </strong>
     ：数据不保存、不共享、不具有独立性
    </p>
    <p>
     <strong>
      <font color="0000ff">
       文件系统阶段(20世纪50年代末–60年代中)
      </font>
     </strong>
     ：可保存，但共享性差，冗余度大，独立性差
    </p>
    <p>
     <strong>
      <font color="0000ff">
       数据库系统阶段(20世纪60年代末–现在)
      </font>
     </strong>
     ：数据库系统的出现使信息系统以
     <strong>
      加工数据的程序
     </strong>
     为中心转向
     <strong>
      围绕共享的数据库
     </strong>
     为中心的新阶段
    </p>
