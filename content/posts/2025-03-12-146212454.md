     我的创作纪念日：730天的技术写作之旅
    </h2>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/75e583258c34407493847a09bdc739c8.png"/>

    <h3>
     <a id="_3">
     </a>
     机缘
    </h3>
    <h4>
     <a id="_4">
     </a>
     从一篇案例分析开始
    </h4>
    <p>
     2023年3月13日，我写下了第一篇技术博客《软考高级-系统分析师-案例分析-系统维护与设计模式》。那时的初心很简单：
    </p>
    <ol>
     <li>
      <strong>
       沉淀实战经验
      </strong>
      ——在备考软考系统分析师时，发现许多知识需要结合实践才能深入理解。
     </li>
     <li>
      <strong>
       技术交流
      </strong>
      ——希望通过文章与同行探讨设计模式的应用场景，避免“纸上谈兵”。
     </li>
     <li>
      <strong>
       自我鞭策
      </strong>
      ——用公开写作倒逼自己系统化梳理知识体系。
     </li>
    </ol>
    <p>
     没想到这一写，就走过了两年的时光。
    </p>
    <hr/>
    <h3>
     <a id="_14">
     </a>
     收获
    </h3>
    <p>
     创作带来的成长远超预期：
    </p>
    <ol>
     <li>
      <strong>
       关注与互动
      </strong>
      ：累计收获了
      <strong>
       800+ 粉丝，希望尽快上千
      </strong>
      ，文章总阅读量突破
      <strong>
       10W+
      </strong>
      ，收到过许多读者的私信感谢。
     </li>
     <li>
      <strong>
       领域深耕
      </strong>
      ：通过评论区交流，结识了多位系统架构师和软考备考伙伴。
     </li>
     <li>
      <strong>
       个人品牌
      </strong>
      ：被CSDN认证为“优质创作者”。
     </li>
    </ol>
    <h3>
     <a id="_22">
     </a>
     最珍贵的收获是发现：
     <strong>
      输出的过程，才是最高效的学习
     </strong>
     。
    </h3>
    <h3>
     <a id="_25">
     </a>
     日常
    </h3>
    <h4>
     <a id="_26">
     </a>
     创作与工作的平衡术
    </h4>
    <ol>
     <li>
      <strong>
       碎片化整理
      </strong>
      ：工作中遇到的典型问题，会即时记录到备忘录，周末整理成文。例如，一次线上事务锁超时的排查过程，最终成了一篇《分布式事务死锁诊断实战》。
     </li>
     <li>
      <strong>
       主题式学习
      </strong>
      ：每月设定一个技术主题（如“云原生监控体系”），边学边写，避免零散输出。
     </li>
     <li>
      <strong>
       工具加持
      </strong>
      ：用Markdown+Git管理技术笔记，配合流程图工具快速绘制架构图，提升效率。
     </li>
    </ol>
    <p>
     如今，每周3小时的写作时间已像健身一样，成为生活的一部分。
    </p>
    <hr/>

     未来的方向

    <ol>
