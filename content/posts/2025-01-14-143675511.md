    <h3>
     <a id="_ssh__16">
     </a>
     一.修改 ssh 配置
    </h3>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/73b63639814b4492919c793b17805f85.png#pic_center"/>
    </p>
    <p>
     VNC 服务器不能使用 SSH 22 端口登录的问题可能由多种原因引起，以下是一些可能的原因和解决方案：
    </p>
    <ol>
     <li>
      <p>
       <strong>
        SSH 服务未开启或配置异常
       </strong>
       ：如果 SSH 服务（
       <code>
        sshd
       </code>
       ）未开启或配置文件（通常是
       <code>
        /etc/ssh/sshd_config
       </code>
       ）权限被修改，可能会导致无法通过 SSH 22 端口登录。你可以通过 VNC 登录到服务器，检查
       <code>
        sshd
       </code>
       进程是否正常运行，并检查配置文件权限。
      </p>
     </li>
     <li>
      <p>
       <strong>
        SSH 服务未设置开机自启动
       </strong>
       ：如果 SSH 服务未设置为开机自启动，重启实例后 SSH 服务可能不会自动启动，导致无法通过 SSH 远程连接。可以通过 VNC 登录实例，然后检查并设置 SSH 服务开机自启动。
      </p>
     </li>
     <li>
      <p>
       <strong>
        安全组规则限制
       </strong>
       ：如果你的服务器在云平台上，可能需要检查安全组规则，确保允许 SSH（22 端口）的入站规则已经设置。如果没有正确设置，SSH 连接会被阻止。
      </p>
     </li>
     <li>
      <p>
       <strong>
        防火墙设置
       </strong>
       ：服务器上的防火墙可能阻止了 SSH 端口。需要检查防火墙状态，并确保 SSH 端口（22）是开放的。
      </p>
     </li>
     <li>
      <p>
       <strong>
        SSH 配置文件错误
       </strong>
       ：检查
       <code>
        /etc/ssh/sshd_config
       </code>
       文件，确保 PermitRootLogin 和 PasswordAuthentication 等参数设置正确。例如，将
       <code>
        PermitRootLogin no
       </code>
       修改为
       <code>
        PermitRootLogin yes
       </code>
       ，将
       <code>
        PasswordAuthentication no
       </code>
       修改为
       <code>
        PasswordAuthentication yes
       </code>
       。
      </p>
     </li>
     <li>
      <p>
       <strong>
        SELinux 设置
       </strong>
       ：如果 SELinux 设置为 enforcing 模式，可能会阻止 SSH 服务。可以尝试将 SELinux 设置为 disabled 或 permissive 模式。
      </p>
     </li>
     <li>
      <p>
       <strong>
        VNC 配置问题
       </strong>
       ：如果 VNC 配置不正确，也可能导致无法通过 SSH 连接。需要检查 VNC 配置文件，确保 VNC 服务正确启动，并且端口（通常是 5901 及以上）没有被防火墙阻止。
       <br/>
       <img alt="" src="https://img-blog.csdnimg.cn/img_convert/073d14fb5a4fac0dc4539df8f90f621e.png#pic_center"/>
      </p>
     </li>
    </ol>
    <h3>
     <a id="_ssh__35">
     </a>
     二.重启 ssh 服务
    </h3>
    <p>
     修改 SSH 配置文件
     <code>
      /etc/ssh/sshd_config
     </code>
     后，要使配置生效，你需要执行以下步骤：
    </p>
    <h4>
     <a id="1_39">
     </a>
     1.保存配置文件
    </h4>
    <p>
     确保你已经保存了对
     <code>
      sshd_config
     </code>
     文件所做的更改。如果你使用的是命令行编辑器（如
     <code>
      vi
     </code>
     或
     <code>
      nano
     </code>
     ），请确保保存并退出编辑器。
    </p>
    <h4>
     <a id="2_SSH__43">
     </a>
     2.重启 SSH 服务
    </h4>
    <p>
     为了让新的配置生效，你需要重启 SSH 服务。这可以通过以下命令完成：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl restart sshd
</code></pre>
    <p>
     或者，如果你的系统使用的是较旧的 init 系统，可以使用：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">service</span> <span class="token function">ssh</span> restart
</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> /etc/init.d/ssh restart
</code></pre>
    <h4>
     <a id="3_SSH__63">
     </a>
     3.检查 SSH 服务状态
    </h4>
    <p>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/9beb0575d4864c70bcafb72772e247dd.png#pic_center"/>
    </p>
    <p>
     重启 SSH 服务后，你可以检查服务的状态以确保它正在运行：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> systemctl status sshd
</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">service</span> <span class="token function">ssh</span> status
</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> /etc/init.d/ssh status
</code></pre>
    <h4>
     <a id="4_84">
     </a>
     4.检查配置文件语法
    </h4>
    <p>
     在重启 SSH 服务之前，建议检查
     <code>
      sshd_config
     </code>
     文件的语法是否正确，以避免因配置错误导致 SSH 服务启动失败。可以使用以下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> sshd -t
</code></pre>
    <p>
     或者：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> <span class="token function">ssh</span> -T -a localhost
</code></pre>
    <p>
     这些命令会检查
     <code>
      sshd_config
     </code>
     文件的语法，并报告任何错误。
    </p>
    <h4>
     <a id="5_SSH__100">
     </a>
     5.确保防火墙允许 SSH 端口
    </h4>
    <p>
     如果你更改了 SSH 的默认端口（从 22 改为其他端口），确保防火墙规则允许新的端口。如果你使用的是
     <code>
      iptables
     </code>
     ，可以使用以下命令允许新的端口（假设新端口为 2222）：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> iptables -A INPUT -p tcp --dport <span class="token number">2222</span> -j ACCEPT
<span class="token function">sudo</span> iptables -A OUTPUT -p tcp --sport <span class="token number">2222</span> -j ACCEPT
</code></pre>
    <p>
     如果你使用的是
     <code>
      firewalld
     </code>
     ，可以使用以下命令：
    </p>
    <pre><code class="prism language-bash"><span class="token function">sudo</span> firewall-cmd --permanent --add-port<span class="token operator">=</span><span class="token number">2222</span>/tcp
<span class="token function">sudo</span> firewall-cmd --reload
</code></pre>
    <h4>
     <a id="6_116">
     </a>
     6.重新尝试连接
    </h4>
    <p>
     在完成上述步骤后，尝试重新通过 SSH 连接到服务器，以验证配置更改是否生效。
    </p>
