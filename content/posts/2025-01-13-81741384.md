    <h1 class="title-article" id="articleContentId">
     Ubuntu14.04 64位机上配置OpenCV3.4.2+OpenCV_Contrib3.4.2+Python3.4.3操作步骤
    </h1>
   </div>
  </div>
 </div>
 <article class="baidu_pl">
  <div class="article_content clearfix" id="article_content">
   <link href="../../assets/css/kdoc_html_views-1a98987dfd.css" rel="stylesheet"/>
   <link href="../../assets/css/ck_htmledit_views-704d5b9767.css" rel="stylesheet"/>
   <div class="htmledit_views" id="content_views">
    <p style="text-indent:50px;">
     Ubuntu 14.04 64位上默认安装了两个版本的python，一个是python2.7.6,另外一个是python3.4.3。这里使用OpenCV最新的稳定版本3.4.2在Ubuntu上安装python3.4.3支持OpenCV的操作步骤如下：
    </p>
    <p style="text-indent:50px;">
     1. 更新包，执行：
    </p>
    <pre class="has"><code class="language-bash">sudo apt-get update
sudo apt-get upgrade</code></pre>
    <p style="text-indent:50px;">
     2. 安装依赖(若下面的依赖库本地已安装则略过)，执行
    </p>
    <pre class="has"><code class="language-bash">sudo apt-get install build-essential cmake git pkg-config
sudo apt-get install libjpeg8-dev libtiff4-dev libjasper-dev libpng12-dev
sudo apt-get install libavcodec-dev libavformat-dev libswscale-dev libv4l-dev
sudo apt-get install libgtk2.0-dev
sudo apt-get install libatlas-base-dev gfortran</code></pre>
    <p style="text-indent:50px;">
     3. 从GitHub中下载最新opencv及opencv_contrib源码3.4.2： opencv-3.4.2.zip, opencv_contrib-3.4.2.zip
    </p>
    <p style="text-indent:50px;">
     4. 将它们存放到/home/spring/Soft/opencv3.4.2目录下，解压缩,并改名，执行：
    </p>
    <pre class="has"><code class="language-bash">unzip opencv-3.4.2.zip; mv opencv-3.4.2 opencv
unzip opencv_contrib-3.4.2.zip; mv opencv_contrib-3.4.2 opencv_contrib</code></pre>
    <p style="text-indent:50px;">
     5. opencv的python库需要依赖Numpy，安装Numpy步骤可以参考：
     <a href="https://blog.csdn.net/fengbingchun/article/details/81590102">
      https://blog.csdn.net/fengbingchun/article/details/81590102
     </a>
    </p>
    <p style="text-indent:50px;">
     6. 编译不带cuda的opencv，因为本地/usr/local/lib目录下已存在opencv2.4.13的库，这里将其安装到/opt/opencv3.4.2目录下，进入到opencv目录下，依次执行：
    </p>
    <pre class="has"><code class="language-bash">mkdir build
cd build
cmake -DCMAKE_BUILD_TYPE=RELEASE -DCMAKE_INSTALL_PREFIX=/opt/opencv3.4.2 -DINSTALL_C_EXAMPLES=ON -DINSTALL_PYTHON_EXAMPLES=ON -DOPENCV_EXTRA_MODULES_PATH=../../opencv_contrib/modules -DBUILD_EXAMPLES=ON ..</code></pre>
    <p style="text-indent:50px;">
     执行后结果如下：
    </p>
    <p style="text-align:center;">
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/4643ef3e5c99cbed3d2b25205bdb4290.png"/>
    </p>
    <p style="text-indent:50px;">
     执行：
    </p>
    <pre class="has"><code class="language-bash">make -j4</code></pre>
    <p style="text-indent:50px;">
     执行后结果如下：
    </p>
    <p style="text-align:center;">
     <img alt="" class="has" src="https://i-blog.csdnimg.cn/blog_migrate/a317dc04d0caca97e69f50beea983473.png"/>
    </p>
    <p style="text-indent:50px;">
     注意：
    </p>
    <p style="text-indent:50px;">
     (1). 为了加快编译速度，可以将编译examples选项关闭，即
    </p>
    <pre class="has"><code class="language-bash">-DINSTALL_C_EXAMPLES=OFF -DINSTALL_PYTHON_EXAMPLES=OFF -DBUILD_EXAMPLES=OFF</code></pre>
    <p style="text-indent:50px;">
     (2). 在编译stitching模块时会提示找不到cuda.hpp即matchers.hpp:52:42: fatal error: opencv2/xfeatures2d/cuda.hpp: No such file or directory，解决方法是在opencv/modules/stitching/CMakeLists.txt文件中加入一条语句使其include opencv_contrib/modules/xfeatures2d/include，可以是绝对路径，如
    </p>
    <pre class="has"><code class="language-bash">INCLUDE_DIRECTORIES("/home/spring/Soft/opencv3.4.2/opencv_contrib/modules/xfeatures2d/include")</code></pre>
    <p style="text-indent:50px;">
     (3). 在编译opencv_contrib/modules/xfeatures2d时会提示找不到boostdesc_bgm.i，即boostdesc.cpp:653:37: fatal error: boostdesc_bgm.i: No such file or directory，解决方法是手动下载所需的*.i文件，下载地址
     <a href="http://answers.opencv.org/question/113942/cmake-failing-with-hash-mismatch/%E4%B8%AD%E6%9C%89%E7%9B%B8%E5%BA%94%E7%9A%84" rel="nofollow">
      http://answers.opencv.org/question/113942/cmake-failing-with-hash-mismatch/%E4%B8%AD%E6%9C%89%E7%9B%B8%E5%BA%94%E7%9A%84
     </a>
     ，然后将这些*.i文件拷贝到opencv_contrib/modules/xfeatures2d/src目录下；或者更新CMake到最新3.12.1，在重装CMake之前，先安装curl,执行
    </p>
    <pre class="has"><code class="language-bash">sudo apt-get install libcurl4-gnutls-dev</code></pre>
    <p style="text-indent:50px;">
     CMake源码的安装步骤可以参考：
     <a href="https://blog.csdn.net/fengbingchun/article/details/51159348">
      https://blog.csdn.net/fengbingchun/article/details/51159348
     </a>
    </p>
    <p style="text-indent:50px;">
     安装CMake时添加--system-curl选项，即
    </p>
    <pre class="has"><code class="language-bash">./bootstrap --system-curl</code></pre>
    <p style="text-indent:50px;">
     强烈推荐后面的方法,因为除了下载*.i文件，还需要用同样的方法下载其它所需文件。
    </p>
    <p style="text-indent:50px;">
     执行：
    </p>
    <pre class="has"><code class="language-bash">sudo make install</code></pre>
    <p style="text-indent:50px;">
     执行完后将会在/opt/opencv3.4.2目录下产生4个目录：bin, include, lib, share,在lib/python3.4/dist-packages目录下会生成cv2.cpython-34m.so
    </p>
    <p style="text-indent:50px;">
     7. 验证
    </p>
    <p style="text-indent:50px;">
     将cv2.cpython-34m.so软链接到/usr/lib/python3/dist-packages目录下，执行：
    </p>
    <p style="text-indent:0;">
    </p>
    <pre class="has"><code class="language-bash">sudo ln -s /opt/opencv3.4.2/lib/python3.4/dist-packages/cv2.cpython-34m.so /usr/lib/python3/dist-packages</code></pre>
    <p style="text-indent:50px;">
     test_opencv_1.py内容如下：
    </p>
    <pre class="has"><code class="language-python">import cv2

print(cv2.__version__)</code></pre>
    <p style="text-indent:50px;">
     执行结果如下：
    </p>
