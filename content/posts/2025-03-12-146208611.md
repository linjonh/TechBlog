    </p>
    <pre><code># 下载方式1-创建一个环境
uv venv --python 3.12.0

# 下载方式2-命令行执行
uv run --python pypy@3.8 -- python

# 使用
uv python pin 3.11
</code></pre>

     2）安装多个 Python 版本：
    </p>
    <pre><code>uv python install 3.10 3.11 3.12
</code></pre>

    <h5>
     <a id="_61">
     </a>
     开展项目
    </h5>
    <p>
     uv 支持管理 Python 项目，在文件中定义其依赖关系
     <code>
      pyproject.toml
     </code>
     。
    </p>
    <h5>
     <a id="_65">
     </a>
     创建新项目
    </h5>
    <p>
     您可以使用以下命令创建一个新的 Python 项目
     <code>
      uv init
     </code>
     ：
    </p>
    <pre><code>uv init hello-world
cd hello-world
</code></pre>
    <p>
     或者，您可以在工作目录中初始化一个项目：
    </p>
    <pre><code>mkdir hello-world
cd hello-world
uv init
</code></pre>
    <p>
     uv 将创建以下文件：
    </p>
    <pre><code>.
├── .python-version
├── README.md
├── main.py
└── pyproject.toml
</code></pre>
    <p>
     该
     <code>
      main.py
     </code>
     文件包含一个简单的“Hello world”程序。尝试一下
     <code>
      uv run
     </code>
     ：
    </p>
    <pre><code>uv run main.py
</code></pre>
    <h5>
     <a id="_98">
     </a>
     项目结构
    </h5>
    <p>
     项目由几个重要的部分组成，它们协同工作并允许 uv 管理您的项目。除了 创建的文件之外， 当您第一次运行项目命令（即、 或
     <br/>
     ）时，
     <code>
      uv init
     </code>
     uv 还会在项目的根目录中创建虚拟环境和文件。
     <code>
      uv.lock``uv run``uv sync``uv lock
     </code>
    </p>
    <p>
     完整列表如下：
    </p>
    <pre><code>.
├── .venv
│   ├── bin
│   ├── lib
│   └── pyvenv.cfg
├── .python-version
├── README.md
├── main.py
├── pyproject.toml
└── uv.lock
</code></pre>
    <hr/>
    <h5>
     <a id="Managing_dependencies_120">
     </a>
     Managing dependencies
    </h5>
    <p>
     You can add dependencies to your
     <code>
      pyproject.toml
     </code>
     with the
     <code>
      uv add
     </code>
     command. This will also update the lockfile and
     <br/>
     project environment:
    </p>
    <pre><code>uv add requests
</code></pre>
    <p>
     You can also specify version constraints or alternative sources:
    </p>
    <pre><code># Specify a version constraint
uv add 'requests==2.31.0'

# Add a git dependency
uv add git+https://github.com/psf/requestsxxxxxxxxxx # Specify a version constraintuv add 'requests==2.31.0'# Add a git dependencyuv add git+https://github.com/psf/requestsuv add requests
</code></pre>
    <p>
     To remove a package, you can use
     <code>
      uv remove
     </code>
     :
    </p>
    <pre><code>uv remove requests
</code></pre>
    <p>
     To upgrade a package, run
     <code>
      uv lock
     </code>
     with the
     <code>
      --upgrade-package
     </code>
     flag:
    </p>
    <pre><code>uv lock --upgrade-package requests
</code></pre>
    <p>
     The
     <code>
      --upgrade-package
     </code>
     flag will attempt to update the specified package to the latest compatible version, while
     <br/>
     keeping the rest of the lockfile intact.
    </p>
    <p>
     See the documentation on
     <a href="https://docs.astral.sh/uv/concepts/projects/dependencies/" rel="nofollow">
      managing dependencies
     </a>
     for more
     <br/>
     details.
    </p>
    <h5>
     <a id="Running_commands_157">
     </a>
     Running commands
    </h5>
    <p>
     <code>
      uv run
     </code>
     can be used to run arbitrary scripts or commands in your project environment.
    </p>
    <p>
     Prior to every
     <code>
      uv run
     </code>
     invocation, uv will verify that the lockfile is up-to-date with the
     <code>
      pyproject.toml
     </code>
     , and that
     <br/>
     the environment is up-to-date with the lockfile, keeping your project in-sync without the need for manual
     <br/>
     intervention.
     <code>
      uv run
     </code>
     guarantees that your command is run in a consistent, locked environment.
    </p>
    <p>
     For example, to use
     <code>
      flask
     </code>
     :
    </p>
    <pre><code>uv add flask
uv run -- flask run -p 3000
</code></pre>
    <p>
     Or, to run a script:
    </p>
    <p>
     example.py
    </p>
    <pre><code># Require a project dependency
import flask

print("hello world")
</code></pre>
    <p>
     Alternatively, you can use
     <code>
      uv sync
     </code>
     to manually update the environment then activate it before executing a command:
    </p>
    <pre><code>uv sync
source .venv/bin/activate
flask run -p 3000
python example.py
</code></pre>
    <p>
     Note
    </p>
    <p>
     虚拟环境必须处于活动状态才能在项目中运行脚本和命令
     <code>
      uv run
     </code>
     。虚拟环境的激活因 shell 和平台而异。
    </p>
    <p>
     有关更多详细信息，请参阅有关在项目中
     <a href="https://docs.astral.sh/uv/concepts/projects/run/" rel="nofollow">
      运行命令和脚本
     </a>
     的文档。
    </p>
    <h5>
     <a id="_198">
     </a>
     构建发行版
    </h5>
    <p>
     <code>
      uv build
     </code>
     可用于为您的项目构建源分布和二进制分布（wheel）。
    </p>
    <p>
     默认情况下，
     <code>
      uv build
     </code>

     <code>
      dist/
     </code>
     子目录中：
    </p>
    <pre><code>uv build
ls dist/
</code></pre>

    <h4>
     <a id="1_uv__pyprojecttoml__213">
     </a>
     1.
     <code>
      uv
     </code>
     与
     <code>
      pyproject.toml
     </code>
     的关系
    </h4>
    <h5>
     <a id="_uv_215">
     </a>
     理解
     <code>
      uv
     </code>
    </h5>
    <p>
     <code>
      uv
     </code>
     通常指的是
     <code>
      ultra-venv
     </code>
     ，它是一个用于创建和管理 Python 虚拟环境的工具。虚拟环境可以为不同的项目隔离 Python
     <br/>
     解释器和依赖包，避免不同项目之间的依赖冲突。
    </p>
    <h5>
     <a id="_pyprojecttoml_220">
     </a>
     理解
     <code>
      pyproject.toml
     </code>
    </h5>
    <p>
     <code>
      pyproject.toml
     </code>
     是 Python 项目的配置文件，在 Python PEP 518
     <br/>
     中被引入。它主要用于指定项目的元数据（如项目名称、版本、作者等）以及构建后端（如
     <code>
      setuptools
     </code>
     、
     <code>
      poetry
     </code>
     <br/>
     等）。通过
     <code>
      pyproject.toml
     </code>
     ，可以清晰地定义项目的依赖项、构建脚本等信息。
    </p>
    <h5>
     <a id="_226">
     </a>
     二者关系
    </h5>
    <ul>
     <li>
      <strong>
       无直接关系
      </strong>
      ：
      <code>
       uv
      </code>
      主要专注于虚拟环境的创建和管理，而
      <code>
       pyproject.toml
      </code>
      是关于项目配置和依赖管理的。它们本身没有直接的功能联系。
     </li>
     <li>
      <strong>
       可间接协同
      </strong>
      ：在一个项目中，你可以使用
      <code>
       uv
      </code>
      创建虚拟环境，然后在这个虚拟环境中根据
      <code>
       pyproject.toml
      </code>
      <br/>
      里定义的依赖信息来安装项目所需的包。例如，如果你使用
      <code>
       poetry
      </code>
      作为构建后端，
      <code>
       poetry
      </code>
      会读取
      <code>
       pyproject.toml
      </code>
      <br/>
      文件中的依赖列表，然后在
      <code>
       uv
      </code>
      创建的虚拟环境中安装这些依赖。
     </li>
    </ul>
    <p>
     以下是一个简单示例：
    </p>
    <pre><code># 使用 uv 创建虚拟环境
uv create my_project_env

# 激活虚拟环境
source my_project_env/bin/activate

# 在项目根目录下有 pyproject.toml 文件，使用 poetry 安装依赖
poetry install
</code></pre>
    <h4>
     <a id="2_uv_Anaconda__246">
     </a>
     2. uv的版本管理和 Anaconda 的区别
    </h4>
    <h5>
     <a id="uv_248">
     </a>
     uv的版本管理
    </h5>
    <ul>
     <li>
      <strong>
       专注虚拟环境
      </strong>
      ：
      <code>
       uv
      </code>
      主要用于创建和管理 Python 虚拟环境。它可以方便地创建、激活、停用和删除虚拟环境，但对于 Python
      <br/>
      解释器的版本管理功能相对较弱。通常需要手动下载和安装不同版本的 Python 解释器，然后在创建虚拟环境时指定使用哪个版本的解释器。
     </li>
     <li>
      <strong>
       轻量级
      </strong>
      ：
      <code>
       uv
      </code>
      是一个轻量级的工具，不会像 Anaconda 那样安装大量的预打包库和工具，因此占用的磁盘空间较小，安装和使用也相对简单。
     </li>
    </ul>
    <h5>
     <a id="Anaconda__254">
     </a>
     Anaconda 的版本管理
    </h5>
    <ul>
     <li>
      <strong>
       全面的版本管理
      </strong>
      ：Anaconda 是一个集成的 Python 发行版，它不仅可以管理虚拟环境，还可以方便地管理 Python
      <br/>
      解释器的版本。通过
      <code>
       conda
      </code>
      命令，可以轻松地安装、切换不同版本的 Python 解释器，以及安装各种科学计算和数据处理相关的库。
     </li>
     <li>
      <strong>
       丰富的包管理
      </strong>
      ：Anaconda 自带了大量的预打包库，如 NumPy、Pandas、Matplotlib 等。使用
      <code>
       conda
      </code>
      <br/>
      命令可以方便地安装、更新和卸载这些库，并且可以解决库之间的依赖关系。
     </li>
     <li>
      <strong>
       跨平台支持
      </strong>
      ：Anaconda 支持多种操作系统（如 Windows、Linux、macOS），并且在不同平台上的使用方式基本一致。
     </li>
    </ul>

    <h4>
     <a id="2uvx__uv_273">
     </a>
     2-uvx 和 uv是什么关系
    </h4>
    <blockquote>
     <p>
      uvx (an alias for uv tool run)
     </p>
    </blockquote>
    <h4>
     <a id="1__277">
     </a>
     1. 基本关系概述
    </h4>
    <p>
     <code>
      uvx
     </code>
     是
     <code>
      uv
     </code>
     工具运行的别名（an alias for uv tool run）。这意味着
     <code>
      uvx
     </code>
     本质上和使用
     <code>
      uv run
     </code>
     <br/>
     命令的功能是等价的，它是为了方便用户操作而提供的一种快捷方式。
    </p>
    <h4>
     <a id="2__282">
     </a>
     2. 使用方式举例
    </h4>
    <p>
     假设我们有一个
     <code>
      uv
     </code>
     环境，并且想要在这个环境中运行某个 Python 脚本。
    </p>
    <h5>
     <a id="_uv_run_286">
     </a>
     使用
     <code>
      uv run
     </code>
    </h5>
    <p>
     通常情况下，如果使用
     <code>
      uv
     </code>
     自带的
     <code>
      run
     </code>
     命令，我们可能会这样操作：
    </p>
    <pre><code class="prism language-bash"><span class="token comment"># 假设我们要运行一个名为 script.py 的 Python 脚本</span>
uv run python script.py
</code></pre>
    <p>
     这里的
     <code>
      uv run
     </code>
     命令会在
     <code>
      uv
     </code>
     管理的虚拟环境上下文中执行后面跟着的命令（在这个例子中是
     <code>
      python script.py
     </code>
     ）。
    </p>
    <h5>
     <a id="_uvx_297">
     </a>
     使用
     <code>
      uvx
     </code>
    </h5>
    <p>
     而使用
     <code>
      uvx
     </code>
     时，操作会更简洁：
    </p>
    <pre><code class="prism language-bash">uvx python script.py
</code></pre>
    <p>
     这行命令和上面使用
     <code>
      uv run
     </code>
     的命令效果是一样的，都是在
     <code>
      uv
     </code>
     虚拟环境中运行
     <code>
      script.py
     </code>
     脚本。
    </p>
    <h4>
     <a id="3__307">
     </a>
     3. 优势
    </h4>
    <p>
     使用
     <code>
      uvx
     </code>
     作为别名的优势在于减少了输入量，提高了操作效率。特别是对于那些需要频繁在
     <code>
      uv
     </code>
     <br/>
     环境中运行命令的用户来说，每次输入
     <code>
      uv run
     </code>
     可能会比较繁琐，而使用
     <code>
      uvx
     </code>
     可以让命令输入更加简洁、快速。
    </p>
