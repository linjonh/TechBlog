     源码分析
    </h4>
    <p>
     <strong>
      简述（口述太难，建议去听课）：
     </strong>
    </p>
    <ol>
     <li>
      <p>
       最开始分两种情况：添加元素扩容到10没满（一） &amp; 添加元素扩容到10但还是超出了（二）
      </p>
     </li>
     <li>
      <p>
       第一种：先进行扩容到10的操作，由grow通过改变size+1（箭头往后移一位）执行：开始扩容，
       <br/>
       这里又分两种情况：初始为0只需要默认扩容到10（一） &amp; 原本已经扩容到10需要继续进行1.5倍扩容（二）。
       <br/>
       这里是第一种情况，所以执行扩容到10的else语句（看图），到此grow执行完毕。执行后箭头后移指向添加好的元素。
      </p>
     </li>
     <li>
      <p>
       第二种：前部分和第一种一样，到扩容的时候，因为超出需要扩容的10（一次直接超出或者多次添加超出是一样的，反正就是超了），所以执行if语句（看图），调用右下角的方法：得到最终的容量（至少和默认的容量作比较，因为两者之间的数值可以大于小于等于）返回（这就是实际长度，因为超出的根本没有规律，只能填多少加多少视情况而定），然后返回右上的方法，同时把老数组的元素复制到了新的数组，最后和第一种一样。

    <h3>
     <a id="3_36">
     </a>
     3.泛型
    </h3>
    <p>
     可以在编译阶段约束操作的数据类型，并进行检查。
     <br/>
     格式：
     <strong>
      &lt;数据类型&gt;
     </strong>
     <br/>
     注意：泛型只能支持引用数据类型。
    </p>
    <p>
     如果我们没有给集合指定类型，默认认为所有的数据类型都是Object类型。
     <br/>
     此时可以往集合添加任意的数据类型。
     <br/>
     带来一个坏处：我们在获取数据的时候，无法使用他的特有行为。
    </p>
    <p>
     此时推出了泛型，可以在添加数据的时候就把类型进行统一。
     <br/>
     而且我们在获取数据的时候，也省得强转了，非常方便。
    </p>
    <h4>
     <a id="1_48">
     </a>
     1.好处
    </h4>
    <ul>
     <li>
      <strong>
       统一数据类型。
      </strong>
     </li>
     <li>
      把运行时期的问题提前到了编译期间，避免了强制类型转换可能出现的异常，因为在编译阶段类型就能确认下来。
     </li>
    </ul>
    <p>
     扩展：Java中的泛型是伪泛型。
    </p>
    <p>
     先检查添加的是否是String，但添加后会统一转为Object，使用的时候再统一转为String。
     <br/>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/2e44e637305e46eebadb59137d81dc02.jpeg"/>
    </p>
    <p>
     <img alt="请添加图片描述" src="https://i-blog.csdnimg.cn/direct/a144dc4dd35540008662f58c965f1815.jpeg"/>
    </p>
    <h4>
     <a id="2_59">
     </a>
     2.细节
    </h4>
    <ol>
     <li>
      泛型不能写基本数据类型。
     </li>
     <li>
      指定泛型的具体类型后，传递数据时，可以传入该类类型或者其子类类型。
     </li>
     <li>
      如果不写泛型，类型默认是Object。
     </li>
    </ol>

    <h5>
     <a id="4_82">
     </a>
     4.通配符
    </h5>
    <p>
     泛型不具备继承性，但是数据具备继承性。
     <br/>
     泛型里面写的是什么类型，那么只能传递什么类型的数据。
    </p>
    <p>
     <strong>
      泛型的通配符：？
     </strong>
     <br/>
     ？也表示不确定的类型
     <br/>
     他可以进行类型的限定
     <br/>
     ？extends E：表示可以传递E或者E所有的子类类型
     <br/>
     ？super E：表示可以传递E或者E所有的父类类型
    </p>
    <p>
     应用场景：
    </p>
    <ol>
     <li>
      如果我们在定义类、方法、接口的时候，如果类型不确定，就可以定义泛型类、泛型方法、泛型接口。
     </li>
     <li>
      如果类型不确定，但是能知道以后
      <strong>
       只能传递某个继承体系中
      </strong>
      的，就可以使用泛型的
      <strong>
       通配符
      </strong>
      。
     </li>
    </ol>

     1.前序遍历
    </h5>
    <p>
     从根节点开始，然后按照
     <strong>
      当前节点
     </strong>
     ，左子节点，右子节点的顺序进行遍历。

    <p>

     <strong>
      当前节点
     </strong>
     ，右子节点的顺序进行遍历。
    </p>
    <h5>
     <a id="3_325">
     </a>
     3.后序遍历
    </h5>
    <p>
     从根节点开始，然后按照左子节点，右子节点，
     <strong>
      当前节点
     </strong>
     的顺序进行遍历。
