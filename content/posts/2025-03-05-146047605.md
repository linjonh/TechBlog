    <p>
     HackBar V2 是一款功能强大的浏览器渗透测试工具，主要用于测试 SQL 注入、XSS 漏洞、POST 传参等安全场景。以下是其核心功能、用法及实际案例操作的综合介绍：
    </p>
    <hr/>
    <h4>
     <a id="_4">
     </a>
     <strong>
      一、核心功能与用法详解
     </strong>
    </h4>
    <h5>
     <a id="1__6">
     </a>
     <strong>
      1. 基础操作
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        Load URL
       </strong>
       <br/>
       功能：将当前浏览器地址栏的 URL 加载到 HackBar 输入框中，便于直接修改参数。
       <br/>
       用法：点击
       <code>
        Load
       </code>
       按钮或使用快捷键
       <code>
        Alt+A
       </code>
       。
       <br/>
       <strong>
        案例
       </strong>
       ：在 DVWA 的 SQL 注入低安全级别下，输入
       <code>
        id=1
       </code>
       提交后，通过
       <code>
        Load
       </code>
       加载 URL 至输入框，便于后续注入操作。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Split URL
       </strong>
       <br/>
       功能：自动分割 URL 中的参数，快速定位可修改的注入点。
       <br/>
       用法：点击
       <code>
        Split
       </code>
       按钮或使用快捷键
       <code>
        Alt+S
       </code>
       。
       <br/>
       <strong>
        案例
       </strong>
       ：对 URL
       <code>
        http://example.com/news.php?id=1&amp;category=2
       </code>
       使用 Split 后，参数
       <code>
        id
       </code>
       和
       <code>
        category
       </code>
       会被拆分显示，便于逐个修改。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Execute
       </strong>
       <br/>
       功能：执行修改后的 URL，等同于刷新页面（
       <code>
        F5
       </code>
       ）。
       <br/>
       用法：点击
       <code>
        Execute
       </code>
       或使用快捷键
       <code>
        Alt+X
       </code>
       /
       <code>
        Ctrl+Enter
       </code>
       。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="2_POST__23">
     </a>
     <strong>
      2. POST 请求功能
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        POST Data 传参
       </strong>
       <br/>
       功能：模拟 POST 请求，用于测试表单提交漏洞。
       <br/>
       用法：勾选
       <code>
        Enable Post Data
       </code>
       ，输入参数（如
       <code>
        id=1&amp;Submit=Submit
       </code>
       ），点击
       <code>
        Execute
       </code>
       发送请求。
       <br/>
       <strong>
        案例
       </strong>
       ：在 DVWA 的中级 SQL 注入关卡，通过抓取表单参数并修改
       <code>
        id
       </code>
       值为注入语句（如
       <code>
        id=1' UNION SELECT 1,2,3 --
       </code>
       ），实现注入攻击。
      </p>
     </li>
     <li>
      <p>
       <strong>
        表单数据抓取
       </strong>
       <br/>
       功能：通过浏览器开发者工具的“网络”标签抓取 POST 请求内容，直接复制到 HackBar 的 Post Data 框。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="3__34">
     </a>
     <strong>
      3. 编码与加密工具
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        URL 编码/解码
       </strong>
       <br/>
       功能：对特殊字符（如空格、引号）进行 URL 编码（
       <code>
        %20
       </code>
       ）或解码。
       <br/>
       用法：选中文本后点击
       <code>
        Encoding
       </code>
       中的
       <code>
        URLencode
       </code>
       或
       <code>
        URLdecode
       </code>
       。
       <br/>
       <strong>
        案例
       </strong>
       ：注入语句
       <code>
        admin' OR 1=1 --
       </code>
       需编码为
       <code>
        admin%27%20OR%201%3D1%20--%20
       </code>
       以绕过过滤。
      </p>
     </li>
     <li>
      <p>
       <strong>
        哈希加密
       </strong>
       <br/>
       功能：支持 MD5、SHA-1、SHA-256 等哈希算法。
       <br/>
       用法：选中文本后选择
       <code>
        MD5 Hash
       </code>
       等选项生成哈希值。
       <br/>
       <strong>
        案例
       </strong>
       ：测试密码爆破时，将常见密码（如
       <code>
        password
       </code>
       ）转换为 MD5 哈希（
       <code>
        5f4dcc3b5aa765d61d8327deb882cf99
       </code>
       ）进行比对。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="4_SQL__47">
     </a>
     <strong>
      4. SQL 注入辅助
     </strong>
    </h5>
    <ul>
     <li>
      <p>
       <strong>
        联合查询语句生成
       </strong>
       <br/>
       功能：提供 MySQL、MS SQL、Oracle 的联合查询模板。
       <br/>
       用法：点击
       <code>
        SQL
       </code>
       菜单选择对应数据库的联合查询语句，自动填充字段数。
       <br/>
       <strong>
        案例
       </strong>
       ：猜解字段数后，生成
       <code>
        UNION SELECT 1,2,3,4,5
       </code>
       语句，替换数字位为
       <code>
        database()
       </code>
       、
       <code>
        user()
       </code>
       等函数获取数据库信息。
      </p>
     </li>
     <li>
      <p>
       <strong>
        Hex 编码转换
       </strong>
       <br/>
       功能：将表名或字段名转换为 16 进制，绕过 WAF 过滤。
       <br/>
       用法：使用在线工具（如 hex2str）转换后，输入至注入语句。
       <br/>
       <strong>
        案例
       </strong>
       ：注入
       <code>
        admin_account
       </code>
       表时，转换为
       <code>
        0x61646d696e5f6163636f756e74
       </code>
       ，避免引号被拦截。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="5_XSS__60">
     </a>
     <strong>
      5. XSS 与漏洞测试
     </strong>
    </h5>
    <ul>
     <li>
      <strong>
       XSS 语句生成
      </strong>
      <br/>
      功能：提供常见 XSS 攻击向量（如
      <code>
       &lt;script&gt;alert(1)&lt;/script&gt;
      </code>
      ）。
      <br/>
      用法：点击
      <code>
       XSS
      </code>
      菜单选择预设语句。
      <br/>
      <strong>
       案例
      </strong>
      ：在搜索框输入
      <code>
       &lt;script&gt;alert(document.cookie)&lt;/script&gt;
      </code>
      ，测试反射型 XSS。
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="6_LFI_67">
     </a>
     <strong>
      6. LFI（本地文件包含）测试辅助
     </strong>
    </h5>
    <h6>
     <a id="1__69">
     </a>
     <strong>
      1. 功能适配与用法
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       <strong>
        路径遍历构造
       </strong>
       <br/>
       HackBar 的 URL 输入框支持快速修改参数值，通过
       <code>
        ../
       </code>
       路径遍历尝试读取系统文件。
       <br/>
       <strong>
        用法
       </strong>
       ：在 URL 参数中直接输入文件路径（如
       <code>
        file=../../../../etc/passwd
       </code>
       ），结合 URL 编码绕过过滤。
       <br/>
       <strong>
        案例
       </strong>
       ：
      </p>
      <ul>
       <li>
        目标 URL 为
        <code>
         http://example.com/view.php?file=index.html
        </code>
        。
       </li>
       <li>
        使用
        <code>
         Split
        </code>
        拆分参数后，将
        <code>
         file
        </code>
        值修改为
        <code>
         ../../../../etc/passwd
        </code>
        。
       </li>
       <li>
        若服务器过滤
        <code>
         ../
        </code>
        ，可使用
        <code>
         %2e%2e%2f
        </code>
        （
        <code>
         ../
        </code>
        的 URL 编码）绕过。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        PHP 包装器利用
       </strong>
       <br/>
       通过 HackBar 快速输入 PHP 伪协议（如
       <code>
        php://filter
       </code>
       ）读取文件源码或执行代码。
       <br/>
       <strong>
        案例
       </strong>
       ：
      </p>
      <ul>
       <li>
        修改参数为
        <code>
         file=php://filter/convert.base64-encode/resource=config.php
        </code>
        。
       </li>
       <li>
        执行后获取 Base64 编码的源码，使用 HackBar 的
        <strong>
         Base64 解码工具
        </strong>
        还原明文。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        编码转换
       </strong>
       <br/>
       使用
       <code>
        URLencode
       </code>
       或
       <code>
        Hex
       </code>
       编码绕过关键词过滤（如
       <code>
        etc/passwd
       </code>
       转换为
       <code>
        %65%74%63%2f%70%61%73%73%77%64
       </code>
       ）。
       <br/>
       <strong>
        快捷键
       </strong>
       ：选中文本后点击
       <code>
        Encoding
       </code>
       菜单选择编码方式。
      </p>
     </li>
    </ul>
    <hr/>
    <h5>
     <a id="7XXEXML__90">
     </a>
     <strong>
      7.XXE（XML 外部实体注入）测试辅助
     </strong>
    </h5>
    <h6>
     <a id="1__92">
     </a>
     <strong>
      1. 功能适配与用法
     </strong>
    </h6>
    <ul>
     <li>
      <p>
       <strong>
        POST 数据构造
       </strong>
       <br/>
       HackBar 的
       <strong>
        POST Data
       </strong>
       功能可直接提交 XML Payload，测试 XXE 漏洞。
       <br/>
       <strong>
        步骤
       </strong>
       ：
      </p>
      <ol>
       <li>
        抓取目标请求的 XML 格式参数（如通过浏览器开发者工具）。
       </li>
       <li>
        在 HackBar 中启用
        <code>
         Enable Post Data
        </code>
        ，输入恶意 XML 实体。
       </li>
       <li>
        点击
        <code>
         Execute
        </code>
        发送请求，观察响应是否包含注入内容。
       </li>
      </ol>
     </li>
     <li>
      <p>
       <strong>
        XXE Payload 模板
       </strong>
       <br/>
       虽然 HackBar 未内置 XXE 生成器，但可手动输入以下常见 Payload：
      </p>
      <pre><code class="prism language-xml"><span class="token prolog">&lt;?xml version="1.0"?&gt;</span>
<span class="token doctype"><span class="token punctuation">&lt;!</span><span class="token doctype-tag">DOCTYPE</span> <span class="token name">foo</span> <span class="token punctuation">[</span><span class="token internal-subset">
  &lt;!ENTITY xxe SYSTEM "file:///etc/passwd"&gt;
</span><span class="token punctuation">]</span><span class="token punctuation">&gt;</span></span>
<span class="token tag"><span class="token tag"><span class="token punctuation">&lt;</span>data</span><span class="token punctuation">&gt;</span></span><span class="token entity named-entity" title="&amp;xxe;">&amp;xxe;</span><span class="token tag"><span class="token tag"><span class="token punctuation">&lt;/</span>data</span><span class="token punctuation">&gt;</span></span>
</code></pre>
      <p>
       <strong>
        案例
       </strong>
       ：
      </p>
      <ul>
       <li>
        目标接口接收 XML 数据（如
        <code>
         http://example.com/api/upload
        </code>
        ）。
       </li>
       <li>
        在 Post Data 中插入上述 Payload，尝试读取
        <code>
         /etc/passwd
        </code>
        文件内容。
       </li>
       <li>
        若服务器禁用外部实体，可尝试
        <strong>
         报错型 XXE
        </strong>
        或
        <strong>
         OOB（带外数据外传）
        </strong>
        。
       </li>
      </ul>
     </li>
     <li>
      <p>
       <strong>
        编码辅助
       </strong>
       <br/>
       若 XML 中特殊字符（如
       <code>
        &lt;
       </code>
       ,
       <code>
        &gt;
       </code>
       ）被过滤，可使用 HackBar 的
       <code>
        HTML Encode
       </code>
       或
       <code>
        Hex
       </code>
       编码功能绕过。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     <a id="_119">
     </a>
     <strong>
      二、实际操作案例
     </strong>
    </h4>
    <h5>
     <a id="_1DVWA__SQL__121">
     </a>
     <strong>
      案例 1：DVWA 中级 SQL 注入
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       目标
      </strong>
      ：绕过下拉菜单限制，注入 POST 请求。
     </li>
     <li>
      <strong>
       步骤
      </strong>
      ：
      <ul>
       <li>
        使用浏览器开发者工具抓取提交表单的 POST 数据（如
        <code>
         id=1&amp;Submit=Submit
        </code>
        ）。
       </li>
       <li>
        在 HackBar 中启用
        <code>
         Post Data
        </code>
        ，输入
        <code>
         id=1' UNION SELECT 1,2,3 -- &amp;Submit=Submit
        </code>
        。
       </li>
       <li>
        点击
        <code>
         Execute
        </code>
        执行，页面返回字段位（如
        <code>
         2
        </code>
        ），替换为
        <code>
         database()
        </code>
        获取数据库名。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_2CTF__POST__128">
     </a>
     <strong>
      案例 2：CTF 题目 POST 传参
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       题目要求
      </strong>
      ：通过 POST 传递参数
      <code>
       A=RUNNER
      </code>
      和
      <code>
       B=BLAD
      </code>
      。
     </li>
     <li>
      <strong>
       步骤
      </strong>
      ：
      <ul>
       <li>
        加载目标 URL 至 HackBar。
       </li>
       <li>
        启用
        <code>
         Post Data
        </code>
        ，输入
        <code>
         A=RUNNER&amp;B=BLAD
        </code>
        。
       </li>
       <li>
        点击
        <code>
         Execute
        </code>
        ，页面返回成功提示或弹窗。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_3LFI_DVWA__135">
     </a>
     <strong>
      案例 3：LFI 读取敏感文件（DVWA 靶场）
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       目标
      </strong>
      ：通过 LFI 漏洞读取
      <code>
       /etc/passwd
      </code>
      。
     </li>
     <li>
      <strong>
       步骤
      </strong>
      ：
      <ul>
       <li>
        访问
        <code>
         http://dvwa/vulnerabilities/fi/?page=include.php
        </code>
        。
       </li>
       <li>
        使用
        <code>
         Split
        </code>
        拆分参数，将
        <code>
         page
        </code>
        值改为
        <code>
         ../../../../etc/passwd
        </code>
        。
       </li>
       <li>
        若回显错误，尝试 URL 编码路径：
        <code>
         %2e%2e%2f%2e%2e%2fetc/passwd
        </code>
        。
       </li>
       <li>
        成功时页面显示系统用户列表。
       </li>
      </ul>
     </li>
    </ol>
    <h5>
     <a id="_4XXE_CTF__143">
     </a>
     <strong>
      案例 4：XXE 读取系统文件（CTF 题目）
     </strong>
    </h5>
    <ol>
     <li>
      <strong>
       目标
      </strong>
      ：通过 XML 上传接口读取
      <code>
       /flag.txt
      </code>
      。
     </li>
     <li>
      <strong>
       步骤
      </strong>
      ：
      <ul>
       <li>
        抓取正常 XML 请求（如
        <code>
         &lt;data&gt;test&lt;/data&gt;
        </code>
        ）。
       </li>
       <li>
        在 HackBar 的 Post Data 中输入：
        <pre><code class="prism language-xml"><span class="token prolog">&lt;?xml version="1.0"?&gt;</span>
<span class="token doctype"><span class="token punctuation">&lt;!</span><span class="token doctype-tag">DOCTYPE</span> <span class="token name">foo</span> <span class="token punctuation">[</span><span class="token internal-subset">&lt;!ENTITY xxe SYSTEM "file:///flag.txt"&gt;</span><span class="token punctuation">]</span><span class="token punctuation">&gt;</span></span>
