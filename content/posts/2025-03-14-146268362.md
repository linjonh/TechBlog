     go中实现子模块调用main包中函数的方法
    </h1>
   </div>
  </div>

    <p>
     你提到的“import cycle not allowed”错误是 Go 语言中一个常见的问题，表示在包的导入中存在循环依赖。在 Go 中，一个包不能直接或间接导入自己，否则就会报这个错误。
    </p>
    <p>
     在你提到的第二个例子中，
     <code>
      main
     </code>
     包和
     <code>
      submodule
     </code>
     包相互导入，形成了一个循环依赖：
    </p>
    <ul>
     <li>
      <code>
       main
      </code>
      包导入了
      <code>
       submodule
      </code>
      包。
     </li>
     <li>
      <code>
       submodule
      </code>
      包又导入了
      <code>
       main
      </code>
      包。
     </li>
    </ul>
    <p>
     这种结构是不被允许的，因为 Go 语言的设计哲学是避免循环依赖，以保持代码的清晰性和可维护性。
    </p>
    <h4>
     <a id="_9">
     </a>
     解决方法
    </h4>
    <h5>
     <a id="_1_11">
     </a>
     方法 1：重构代码，避免循环依赖
    </h5>
    <p>
     最简单的方法是重新设计代码结构，避免
     <code>
      submodule
     </code>
     包直接依赖
     <code>
      main
     </code>
     包。例如，可以将需要共享的逻辑提取到一个独立的包中，然后让
     <code>
      main
     </code>
     包和
     <code>
      submodule
     </code>
     包都依赖这个独立的包。
    </p>
    <p>
     假设项目结构如下：
    </p>
    <pre><code>myapp/
├── main.go
├── submodule/
│   └── submodule.go
├── shared/
│   └── shared.go
</code></pre>
    <p>
     在
     <code>
      shared/shared.go
     </code>
     中定义共享逻辑：
    </p>
    <pre><code class="prism language-go"><span class="token comment">// shared/shared.go</span>
<span class="token keyword">package</span> shared

<span class="token keyword">import</span> <span class="token string">"fmt"</span>

<span class="token keyword">func</span> <span class="token function">SayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    fmt<span class="token punctuation">.</span><span class="token function">Println</span><span class="token punctuation">(</span><span class="token string">"Hello from shared package!"</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在
     <code>
      main.go
     </code>
     中调用共享逻辑：
    </p>
    <pre><code class="prism language-go"><span class="token comment">// main.go</span>
<span class="token keyword">package</span> main

<span class="token keyword">import</span> <span class="token punctuation">(</span>
    <span class="token string">"./shared"</span>
    <span class="token string">"./submodule"</span>
<span class="token punctuation">)</span>

<span class="token keyword">func</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    shared<span class="token punctuation">.</span><span class="token function">SayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
    submodule<span class="token punctuation">.</span><span class="token function">CallSharedFunction</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在
     <code>
      submodule/submodule.go
     </code>
     中调用共享逻辑：
    </p>
    <pre><code class="prism language-go"><span class="token comment">// submodule/submodule.go</span>
<span class="token keyword">package</span> submodule

<span class="token keyword">import</span> <span class="token punctuation">(</span>
    <span class="token string">"fmt"</span>
    <span class="token string">"./shared"</span>
<span class="token punctuation">)</span>

<span class="token keyword">func</span> <span class="token function">CallSharedFunction</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    shared<span class="token punctuation">.</span><span class="token function">SayHello</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
    fmt<span class="token punctuation">.</span><span class="token function">Println</span><span class="token punctuation">(</span><span class="token string">"Called from submodule!"</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这样，
     <code>
      main
     </code>
     包和
     <code>
      submodule
     </code>
     包都依赖
     <code>
      shared
     </code>
     包，而不会形成循环依赖。
    </p>
    <h5>
     <a id="_2_70">
     </a>
     方法 2：使用回调函数
    </h5>
    <p>
     如果确实需要在
     <code>
      submodule
     </code>
     中调用
     <code>
      main
     </code>
     包中的函数，可以通过回调函数的方式传递。这样可以避免直接导入
     <code>
      main
     </code>
     包。
    </p>
    <p>
     假设项目结构如下：
    </p>
    <pre><code>myapp/
├── main.go
├── submodule/
│   └── submodule.go
</code></pre>
    <p>
     在
     <code>
      main.go
     </code>
     中定义回调函数：
    </p>
    <pre><code class="prism language-go"><span class="token comment">// main.go</span>
<span class="token keyword">package</span> main

<span class="token keyword">import</span> <span class="token punctuation">(</span>
    <span class="token string">"fmt"</span>
    <span class="token string">"./submodule"</span>
<span class="token punctuation">)</span>

<span class="token keyword">func</span> <span class="token function">main</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    submodule<span class="token punctuation">.</span><span class="token function">CallFunction</span><span class="token punctuation">(</span><span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
        fmt<span class="token punctuation">.</span><span class="token function">Println</span><span class="token punctuation">(</span><span class="token string">"Hello from main package!"</span><span class="token punctuation">)</span>
    <span class="token punctuation">}</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     在
     <code>
      submodule/submodule.go
     </code>
     中定义一个函数，接受回调函数作为参数：
    </p>
    <pre><code class="prism language-go"><span class="token comment">// submodule/submodule.go</span>
<span class="token keyword">package</span> submodule

<span class="token keyword">import</span> <span class="token string">"fmt"</span>

<span class="token keyword">func</span> <span class="token function">CallFunction</span><span class="token punctuation">(</span>callback <span class="token keyword">func</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">)</span> <span class="token punctuation">{<!-- --></span>
    <span class="token function">callback</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
    fmt<span class="token punctuation">.</span><span class="token function">Println</span><span class="token punctuation">(</span><span class="token string">"Called from submodule!"</span><span class="token punctuation">)</span>
<span class="token punctuation">}</span>
</code></pre>
    <p>
     这种方式通过回调函数传递逻辑，避免了直接导入
     <code>
      main
     </code>
     包，从而解决了循环依赖问题。
    </p>
    <h4>
     <a id="_113">
     </a>
     总结
    </h4>
    <ul>
     <li>
      <strong>
       推荐方法
      </strong>
      ：将共享逻辑提取到独立的包中，避免循环依赖。
     </li>
     <li>
      <strong>
       替代方法
      </strong>
      ：使用回调函数传递逻辑，避免直接导入
      <code>
       main
      </code>
      包。
     </li>
    </ul>
    <p>
     在实际开发中，尽量避免在子模块中直接调用
     <code>
      main
     </code>
     包的逻辑，这样可以保持代码的清晰性和模块化。
    </p>
