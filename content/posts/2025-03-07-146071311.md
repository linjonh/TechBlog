    <h4>
     hvigor主要提供了两种方式来实现插件
    </h4>
    <p>
     （1）基于hvigorfile脚本开发插件。
    </p>
    <p>
     （2）基于typescript项目开发。
    </p>
    <p>
    </p>
    <h3>
     一、基于hvigorfile脚本开发
    </h3>
    <p>
     基于hvigorfile.ts脚本开发的方式，其优点是可实现快速开发，直接编辑工程或者模块下hvigorfile.ts即可编写插件代码，不足之处是在多个项目中，无法方便的进行插件代码的复用和共享开发。
    </p>
    <h5>
     1、导入模块依赖，更多接口参考
     <a class="link-info" href="https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-hvigor-apis-V5" rel="nofollow" title="扩展构建API。">
      扩展构建API。
     </a>
    </h5>
    <pre><code class="hljs">// 导入接口
import { HvigorPlugin, HvigorNode } from '@ohos/hvigor';</code></pre>
    <h5>
     2、编写插件代码。
    </h5>
    <p>
     在hvigorfile.ts中定义插件方法，实现HvigorPlugin接口。
    </p>
    <pre><code class="hljs">// 实现自定义插件
function customPlugin(): HvigorPlugin {
    return {
        pluginId: 'customPlugin',
        apply(node: HvigorNode) {
            // 插件主体
            console.log('hello customPlugin!');
        }
    }
}</code></pre>
    <h5>
     3、在导出声明中使用插件
    </h5>
    <pre><code class="hljs">export default {
    system: appTasks,
    plugins:[
        customPlugin()  // 应用自定义Plugin
    ]
}</code></pre>
    <h5>
     4、执行hvigor命令。
    </h5>
    <p>
     执行hvigor命令时，在hvigor生命周期配置阶段执行插件中的apply方法。
    </p>
    <p>
     <img alt="" height="133" src="https://i-blog.csdnimg.cn/direct/ccfd44ceffdf48248b79beff53cffbcb.png" width="513"/>
    </p>
    <p>
    </p>
    <p>
     下面咱们按照上面的步骤操作一下：
    </p>
    <h3>
     二、实操（基于hvigorfile脚本开发）

     1、创建一个新的工程，比如命名ProjectName为 MyPluginProject
    </h5>
    <h5>
     <br/>
     <img alt="" height="1320" src="https://i-blog.csdnimg.cn/direct/a8b2bd45c5e44d189934fab41e57b134.png" width="1988"/>
    </h5>
    <h5>
     2、在新的项目工程的工程目录下找到hvigorfile.ts文件，打开该文件按上述步骤进行编辑
    </h5>
    <p>
     <img alt="" height="882" src="https://i-blog.csdnimg.cn/direct/9f99a9335d144a6aa14cbacf67fb1023.png" width="3122"/>
    </p>
    <p>

     </img>
    </p>
    <h4>
     3、在Terminal中执行hvigorw --sync命令
    </h4>
    <p>
     <img alt="" height="1132" src="https://i-blog.csdnimg.cn/direct/4d731526d0d742efa8992aa0baee9ce4.png" width="3390"/>
    </p>
    <p>
     可以在控制台看到我们在自定义插件customPlugin中通过console打印的log："hello customPlugin!"，这样自定义插件就完成，是不是so easy。如果想实现你自己的逻辑，你可以在自定义插件的 apply里面编写。
    </p>
    <h5>
     弊端：第一种方法到这里就介绍完了，看起来很简单，不过我们在实际开发中，如果所有的自定义插件都是以这种方式编写在hvigorfile.ts文件中，会导致hvigorfile.ts中代码量很大，易读性差，而且有多个工程项目想用同一个插件，还得把代码拷贝来拷贝去，很繁琐，所以咱们下面介绍第二种方法。
    </h5>
    <p>
    </p>
    <h3>
     三、基于typescript项目开发
    </h3>
    <p>
     <span style="color:#b95514">
      基于typescript项目开发较好地弥补了上一小节中使用hvigorfile脚本方式编写插件代码不易复用和共享分发的问题。因此通常情况下推荐此方式开发。
     </span>
    </p>
    <h4>
     1、初始化typescript项目
    </h4>
    <p>
     （1）创建一个空目录。
    </p>
    <p>
     在命令行工具中使用cd命令进入空目录下。
    </p>
    <p>
     （2）安装typescript模块。
    </p>
    <pre><code class="hljs">// 全局安装TypeScript
npm install typescript -g</code></pre>
    <p>
     （3）初始化npm项目
    </p>
    <p>
     执行如下命令，
     <span style="color:#ff9900">
      根据命令行指示配置项目
     </span>
     。
    </p>
    <pre><code class="hljs">// 初始化一个npm项目
npm init</code></pre>
    <p>
     （4）生成typescript配置文件。
    </p>
    <p>
     执行如下命令生成tsconfig.json文件。
    </p>
    <pre><code class="hljs">// 初始化TypeScript配置文件
tsc --init</code></pre>
    <p>
     （5）typescript项目初始化完成。
    </p>
    <p>
     <img alt="" height="108" src="https://i-blog.csdnimg.cn/direct/84909032864d4d7b8e68d2ac9ec4ab91.png" width="248"/>
    </p>
    <h4>
     2、开发插件
    </h4>
    <p>
     （1）配置npm镜像仓库地址。
    </p>
    <p>
     在用户目录下创建或打开.npmrc文件，配置如下信息：
    </p>
    <pre><code class="hljs">registry=https://repo.huaweicloud.com/repository/npm/
@ohos:registry=https://repo.harmonyos.com/npm/</code></pre>
    <p>
     （2）添加依赖声明。
    </p>
    <p>
     打开package.json 添加devDependencies配置。
    </p>
    <pre><code class="hljs">"devDependencies": {
    "@ohos/hvigor": "5.2.2"
}</code></pre>
    <p>
     （3）安装依赖
    </p>
    <p>
     执行如下命令安装依赖。
    </p>
    <pre><code class="hljs">npm install</code></pre>
    <p>
     （4）编写插件代码。
    </p>
    <p>
     创建 custom-plugin.ts文件，编写插件代码。
    </p>
    <pre><code class="hljs">import { HvigorNode, HvigorPlugin } from '@ohos/hvigor';

export function customPlugin(): HvigorPlugin {
    return {
        pluginId: 'customPlugin',
        apply(node: HvigorNode) {
            console.log('hello customPlugin!');
        }
    }
}</code></pre>
    <p>
     （5）创建index.ts文件，并在该文件中声明插件方法的导出。
    </p>
    <pre><code class="hljs">export { customPlugin } from './src/plugin/custom-plugin';</code></pre>
    <h4>
     3、发布插件
    </h4>
    <p>
     typescript项目本质上是一种npm项目，插件发布流程遵循npm发布规范。
    </p>
    <p>
     发布npm包流程：
    </p>
    <p>
     （1）配置registry。
    </p>
    <p>
     在用户目录下创建.npmrc文件，配置您需要发布的镜像仓库。
    </p>
    <pre><code class="hljs">registry=[npm镜像仓库地址]</code></pre>
    <p>
     （2）生成AccessToken。
    </p>
    <p>
     执行如下命令，注册并登录npm仓库，在用户目录下.npmrc文件中自动生成token信息。
    </p>
    <pre><code class="hljs">npm login</code></pre>
    <p>
     （3）发布nmp包
    </p>
    <p>
     执行如下命令，将npm项目打包并发布至镜像仓库。
    </p>
    <pre><code class="hljs">npm publish</code></pre>
    <h4>
     4、使用插件
    </h4>
    <p>
     （1）添加依赖。
    </p>
    <p>
     在工程下hvigor/hvigor-config.json5中添加自定义插件依赖，依赖项支持离线插件配置。
    </p>
    <pre><code class="hljs"> "dependencies": {
    "custom-plugin": "1.0.0"   // 添加自定义插件依赖
  }</code></pre>
    <p>
     （2）安装依赖
    </p>
    <ul>
     <li>
      方式1：执行编辑区右上角Install Now 或执行菜单File-&gt;Sync and Refresh Project进行工程Sync后，DevEco Studio将会根据hvigor-config.json5中的依赖配置自动安装。
     </li>
     <li>
      方式2：使用hvigorw命令行工具执行任一命令，命令行工具会自动执行安装构建依赖。
      <pre><code class="hljs">hvigorw --sync</code></pre>
     </li>
    </ul>
    <p>
     （3）导入插件。
    </p>
    <p>
     根据插件编写时基于的node节点，确定导入的节点所在的hvigorfile.ts文件，在hvigorfile.ts中导入插件。
    </p>
    <pre><code class="hljs">import { customPlugin } from 'custom-plugin';</code></pre>
    <p>
     （4）使用插件
    </p>
    <p>
     将自定义插件添加到export default的plugins中。
    </p>
    <pre><code class="hljs">export default {
    system: appTasks,
    plugins:[
        customPlugin()  // 应用自定义插件
    ]
}</code></pre>
