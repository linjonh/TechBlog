    <h3>
     <a id="51__2">
     </a>
     5.1 语义化标签的「身份认证」
    </h3>
    <h4>
     <a id="_4">
     </a>
     一、语义化概念理解
    </h4>
    <p>
     在 HTML 的旧时代，我们就像一群建筑工人，只能用一些通用的
     <code>
      &lt;div&gt;
     </code>
     和
     <code>
      &lt;span&gt;
     </code>
     标签来搭建网页，就好比用千篇一律的砖块盖房子，虽然能把房子盖起来，但很难从外观上看出房子的各个部分是做什么用的。而 HTML5 引入的语义化标签，就像是给每一块砖块都赋予了特殊的形状和用途，让网页的结构变得一目了然，就像给每个房间都贴上了清晰的标签，告诉搜索引擎和屏幕阅读器等辅助设备，这里是客厅，那里是卧室。
    </p>
    <p>
     语义化标签不仅能让代码的可读性大大提高，方便开发者维护和理解，还能对搜索引擎优化（SEO）起到积极的作用。搜索引擎就像一个聪明的访客，它能根据这些语义化标签更好地理解网页的内容，从而提高网页在搜索结果中的排名。同时，对于视力障碍等需要使用屏幕阅读器的用户来说，语义化标签能让他们更轻松地了解网页的结构和内容。
    </p>

      &lt;ul&gt;

     <code>
      &lt;ol&gt;
     </code>
     ，列表中的每个项
     <code>
      &lt;li&gt;
     </code>
     就是一个导航链接。

    <h5>
     <a id="3_article_44">
     </a>
     3.
     <code>
      &lt;article&gt;
     </code>
     ：文章小宇宙
    </h5>
    <p>
     <code>
      &lt;article&gt;
     </code>

     <code>
      &lt;article&gt;
     </code>
     标签内的内容应该是可以独立存在和被理解的，就像一篇单独的文章可以在不同的地方被阅读。

    <h4>
     <a id="_286">
     </a>
     二、离线应用
    </h4>
    <p>
     HTML5 的离线应用功能就像是给网页穿上了一件“离线防护服”，让网页在没有网络连接的情况下也能正常访问和使用。要实现离线应用，需要创建一个
     <code>
      manifest
     </code>
     文件。
    </p>
    <h5>
     <a id="1__manifest__289">
     </a>
     1. 创建
     <code>
      manifest
     </code>
     文件
    </h5>
    <p>
     <code>
      manifest
     </code>
     文件是一个文本文件，用于指定哪些资源需要被缓存。以下是一个简单的
     <code>
      manifest
     </code>
     文件示例：
    </p>
    <pre><code>CACHE MANIFEST
# 版本号，每次更新资源后修改版本号可强制刷新缓存
# 2025-03-10 v1

# 需要缓存的资源
CACHE:
index.html
styles.css
script.js

# 在线时使用的资源
NETWORK:
*

# 离线时的备用页面
FALLBACK:
/ /offline.html
</code></pre>
    <ul>
     <li>
      <code>
       CACHE:
      </code>
      部分列出了需要缓存的资源，包括 HTML 文件、CSS 文件、JavaScript 文件等。
     </li>
     <li>
      <code>
       NETWORK:
      </code>
      部分指定了在线时使用的资源，
      <code>
       *
      </code>
      表示所有资源。
     </li>
     <li>
      <code>
       FALLBACK:
      </code>
      部分指定了离线时的备用页面，这里表示当任何页面无法访问时，显示
      <code>
       offline.html
      </code>
      页面。
     </li>
    </ul>
    <h5>
     <a id="2__HTML__manifest__314">
     </a>
     2. 在 HTML 文件中引用
     <code>
      manifest
     </code>
     文件
    </h5>
    <p>
     在 HTML 文件的
     <code>
      &lt;html&gt;
     </code>
     标签中添加
     <code>
      manifest
     </code>
     属性，引用
     <code>
      manifest
     </code>
     文件。
    </p>
    <pre><code class="prism language-html"><span class="token doctype"><span class="token punctuation">&lt;!</span><span class="token doctype-tag">DOCTYPE</span> <span class="token name">html</span><span class="token punctuation">&gt;</span></span>

     这样，浏览器会根据
     <code>
      manifest
