    <p>
     <a href="https://blog.csdn.net/xhzq1986/article/details/145986523">
      Ubuntu 安装 stable-diffusion-webui-docker 常见问题处理方法
     </a>
     这篇文章介绍了在 Ubuntu 上安装 stable-diffusion-webui-docker，运行
     <code>
      docker compose --profile auto up --build
     </code>
     构建出的界面是 stable-diffusion-webui，如果运行
     <code>
      docker compose --profile comfy up --build
     </code>
     ，那么就是生成的 comfy-ui 界面，如下图所示
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/21528a415776408b8088f8ff84058061.png"/>
    </p>
    <p>
     同理，如需指定 comfy 运行的gpu，可以修改
     <code>
      docker-compose.yaml
     </code>
     ，如下，指定
     <code>
      --cuda-device 0
     </code>
     ：
    </p>
    <pre><code>  comfy: &amp;comfy
    &lt;&lt;: *base_service
    profiles: ["comfy"]
    build: ./services/comfy/
    image: sd-comfy:7
    environment:
      - CLI_ARGS=--cuda-device 0
</code></pre>
    <p>
     在
     <code>
      /stable-diffusion-webui-docker/services/comfy/Dockerfile
     </code>
     文件中需要注意两点：
     <br/>
     1、
     <code>
      FROM pytorch/pytorch:2.3.0-cuda12.1-cudnn8-runtime
     </code>
     pytorch 镜像需要和本地环境保持一致
     <br/>
     2、需要删除
     <code>
      git reset --hard 276f8fce9f5a80b500947fb5745a4dde9e84622d &amp;&amp; \
     </code>
     这一行，直接使用最新的 comfyui 即可
     <br/>
     如果你还用到了文生视频等功能，还需要放开一些目录映射，详见文件
     <code>
      stable-diffusion-webui-docker/services/comfy/extra_model_paths.yaml
     </code>
     ，如下：
     <br/>
     <img alt="在这里插入图片描述" src="https://i-blog.csdnimg.cn/direct/a5d518680ee7458589d54f44807ff65e.png">
      <br/>
      我统一修改为：
     </img>
    </p>
    <pre><code>style_models: models/style_models
t2i_adapter: models/t2i_adapter
clip_vision: models/clip_vision
diffusers: models/diffusers
text_encoders: models/text_encoders
diffusion_models: models/Stable-diffusion
</code></pre>
