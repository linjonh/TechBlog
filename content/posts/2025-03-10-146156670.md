    <h4>
     引言
    </h4>
    <p>
     “为什么我传个文件到服务器要折腾半小时？”
     <br/>
     如果你也曾在Linux系统中为文件传输抓狂，今天这篇保姆级教程就是你的救星！
     <strong>
      SCP命令
     </strong>
     ——一个基于SSH协议的高效传输工具，只需5分钟，彻底告别FTP客户端和繁琐操作！
    </p>
    <h4>
     一、什么是SCP？为什么必须学它？
    </h4>
    <p>
     <strong>
      SCP（Secure Copy Protocol）
     </strong>
     是Linux系统中基于SSH协议的文件传输命令，优势直接拉满：
    </p>
    <ul>
     <li>
      <p>
       ✅
       <strong>
        安全加密
       </strong>
       ：所有数据传输全程加密，不怕被截获。
      </p>
     </li>
     <li>
      <p>
       ✅
       <strong>
        无需额外工具
       </strong>
       ：一条命令搞定上传/下载，无需安装FTP。
      </p>
     </li>
     <li>
      <p>
       ✅
       <strong>
        跨平台兼容
       </strong>
       ：支持Linux、Mac，甚至Windows（借助WinSCP或PowerShell）。
      </p>
     </li>
    </ul>
    <hr/>
    <h4>
     二、SCP命令超简公式
    </h4>
    <p>
     <strong>
      记住这个万能模板，秒变文件传输高手！
     </strong>
    </p>
    <pre><code class="language-bash">scp [参数] 源文件路径 目标路径  </code></pre>
    <h5>
     1️⃣
     <strong>
      本地 → 远程服务器
     </strong>
     （上传文件）
    </h5>
    <pre><code class="language-bash">scp /本地/照片.jpg user@远程IP:/远程/目录/  </code></pre>
    <p>
     <strong>
      执行后会提示输入密码
     </strong>
     ，输入远程服务器的用户密码即可！
    </p>
    <h5>
     2️⃣
     <strong>
      远程服务器 → 本地
     </strong>
     （下载文件）
    </h5>
    <pre><code class="language-bash">scp user@远程IP:/远程/报告.pdf /本地/下载/  </code></pre>
    <h5>
     3️⃣
     <strong>
      远程 ↔ 远程
     </strong>
     （服务器间传输）
    </h5>
    <pre><code class="language-bash">scp user1@服务器AIP:/文件.txt user2@服务器BIP:/目标目录/  </code></pre>
    <h4>
     三、小白必学的4个高频参数
    </h4>
    <table>
     <thead>
      <tr>
       <th>
        参数
       </th>
       <th>
        作用
       </th>
       <th>
        示例场景
       </th>
      </tr>
     </thead>
     <tbody>
      <tr>
       <td>
        <code>
         -P
        </code>
       </td>
       <td>
        指定SSH端口（默认22）
       </td>
       <td>
        <code>
         scp -P 2222 文件 user@IP:/目录
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -r
        </code>
       </td>
       <td>
        递归传输整个目录
       </td>
       <td>
        <code>
         scp -r 本地文件夹/ user@IP:/远程目录
        </code>
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -v
        </code>
       </td>
       <td>
        显示详细传输过程
       </td>
       <td>
        调试时查看问题原因
       </td>
      </tr>
      <tr>
       <td>
        <code>
         -C
        </code>
       </td>
       <td>
        启用压缩加速传输
       </td>
       <td>
        大文件传输省时间！
       </td>
      </tr>
     </tbody>
    </table>
    <hr/>
    <h4>
     四、实战！手把手教你跑通SCP
    </h4>
    <h5>
     场景1：从本地上传代码到服务器
    </h5>
    <pre><code class="language-bash">scp -P 2022 -r ~/projects/myapp/ root@192.168.1.100:/var/www/  </code></pre>
    <p>
     <strong>
      效果
     </strong>
     ：将本地
     <code>
      myapp
     </code>
     目录压缩后传到服务器的
     <code>
      /var/www/
     </code>
     ，使用非默认端口
     <code>
      2022
     </code>
     。
    </p>
    <h5>
     场景2：从服务器下载日志文件
    </h5>
    <pre><code class="language-bash">scp admin@10.0.0.5:/var/log/nginx/error.log ./Desktop/  </code></pre>
    <h4>
     <strong>
      效果
     </strong>
     ：把服务器的Nginx错误日志下载到电脑桌面。
    </h4>
    <p>
    </p>
    <h4>
     五、安全警告！90%人踩过的坑
    </h4>
    <ul>
     <li>
      <p>
       <strong>
        ⚠️ 密码暴露风险
       </strong>
       ：SCP默认用密码验证，容易被历史命令记录。
       <br/>
       <strong>
        解决方案
       </strong>
       ：改用SSH密钥对（下期手把手教你配置！）。
      </p>
     </li>
     <li>
      <p>
       <strong>
        ⚠️ 文件权限问题
       </strong>
       ：传输后文件权限可能变化，用
       <code>
        chmod
       </code>
       修复。
      </p>
     </li>
     <li>
      <p>
       <strong>
        ⚠️ 网络中断重传
       </strong>
       ：大文件传输中断需重新开始，建议用
       <code>
        rsync
       </code>
       （下期详解）。
      </p>
     </li>
    </ul>
